Control UI 中的每個討論串都有兩個面向：你熟悉的對話，以及一個
**儀表板**——由你的代理程式為你建立、包含即時小工具的網格。沒有
小工具的討論串就只是聊天。一旦釘選小工具，標頭中就會出現 **聊天 | 儀表板**
切換選項，而儀表板會成為主要介面，聊天則停駐在旁邊。

不需要進行任何設定，也不必設定其他應用程式：儀表板是
核心功能，由討論串擁有、隨代理程式儲存，並且會在
`/new` 和 `/reset` 後保留（對話情境會清除，但面板會保留）。

## 透過提出要求建立儀表板

告訴代理程式你想看到什麼：

> 建立名為 revenue-graph 的小工具：顯示每月
> 營收的互動式長條圖。新增可切換檢視的「長條」和「趨勢」按鈕。將它釘選到我的
> 儀表板。

代理程式會先在聊天中以內嵌方式呈現小工具，讓你能在它移至其他位置前
先查看。接著：

- **由你釘選**：將游標停在內嵌小工具上，然後選擇 **釘選到儀表板**。
- **或由代理程式直接釘選**，並在你提出要求時依
  名稱更新——小工具有穩定的名稱，因此「使用六月的
  數字更新 revenue-graph」會就地取代內容，而面板維持原位。

小工具是自成一體的小型應用程式（在嚴格沙箱中的 HTML/JS/SVG）。小工具內的按鈕
和檢視切換功能可以立即運作——切換圖表檢視
完全不需要代理程式。

## 面板

- **流動式網格。** 使用控點拖曳小工具；所有內容都會自動重新排列並
  壓縮。使用控點調整大小，或從小工具選單中選取大小預設值（小、
  中、大、特大）。沒有人需要指定像素位置——
  你不需要，代理程式也不需要。
- **分頁。** 一個面板可以有多個頁面——例如總覽分頁，以及
  只有一個大型小工具的聚焦分頁。每個分頁都會記住自己的聊天停駐
  位置。
- **停駐聊天。** 在儀表板介面上，對話可停駐於
  左側、右側或底部，能像側邊欄一樣調整大小，也能完全
  隱藏——當你重新顯示聊天時，代理程式仍能聽到你的訊息。
- **代理程式對等操作。** 你能執行的所有操作，代理程式都能使用其
  `dashboard` 工具執行：新增、更新、移動、調整大小及移除小工具、管理
  分頁、切換可見分頁，以及移動或隱藏聊天停駐區。提出「將聊天放在
  左側並顯示財務分頁」，即可看到操作立即完成。

## 小工具可執行的操作

僅進行轉譯的小工具不需要核准——它會立即出現，就像
內嵌聊天小工具一樣，而且其網路存取權會完全停用。

想要取得**存取能力**的小工具必須加以宣告，而你只需為每個小工具
輕觸一次即可授予：

- **網路**（`net`）：直接從沙箱擷取已宣告的 HTTPS 來源——
  例如，可從 API 自行重新整理的天氣卡片。
- **閘道資料**（`data`）：工作階段、用量或排程
  狀態等唯讀資訊流，由閘道解析——小工具永遠不會持有你的權杖。
- **自動化**（`actions`）：觸發特定的排程工作，讓按鈕可以執行
  真正的任務（可能使用較小的模型），而不會喚醒你的主要
  對話。
- **提示**（`prompt`）：將訊息傳送至你的討論串，無須像未核准的
  小工具一樣，每次點擊都要確認。

已啟用的外掛可以將自己的具名唯讀資訊流和動作加入這些功能清單；停用外掛會移除這些整合。

授權會綁定至你審閱過的小工具確切位元組與修訂版本。如果
代理程式變更小工具，並要求的權限比你核准的_更多_，它會回到
待處理狀態；在相同權限範圍內重新整理內容則會保留授權。
代理程式應知道的小工具互動（你點選的篩選條件、你切換的檢視）
會以工作階段通知的形式悄悄傳達給它——讓它掌握狀況而不受
中斷。

## 面板上的 MCP 應用程式

如果你的閘道已設定 MCP 伺服器，聊天中出現的互動式 MCP 應用程式
可以像任何小工具一樣釘選。釘選的應用程式會在面板上以新的工作階段
重新啟動；預設情況下只能顯示內容，而授予小工具其宣告的伺服器工具後，
即可獲得完整互動功能——使用與其他所有項目相同的
單次輕觸、綁定修訂版本核准機制。

## 注意事項

- 重設含有面板的討論串時，系統會要求確認，並保留
  面板。
- 刪除討論串時，也會刪除其面板。
- 面板存放在你的閘道上（位於所屬代理程式的資料庫中），並會顯示在
  你連線的每一部裝置上。
- 安全模型、儲存詳細資訊和設計理由記載於
  [儀表板架構](/zh-TW/web/dashboard-architecture)，其中也包括
  已記錄的沙箱取捨。
