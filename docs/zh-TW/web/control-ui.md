Control UI 是由閘道提供的小型 **Vite + Lit** 單頁應用程式：

- 預設：`http://<host>:18789/`
- 選用前綴：設定 `gateway.controlUi.basePath`（例如 `/openclaw`）

它會透過相同連接埠**直接連線至閘道 WebSocket**。

## 快速開啟（本機）

如果閘道在同一台電腦上執行，請開啟 [http://127.0.0.1:18789/](http://127.0.0.1:18789/)（或 [http://localhost:18789/](http://localhost:18789/)）。

如果頁面無法載入，請先啟動閘道：`openclaw gateway`。

<Note>
在原生 Windows LAN 繫結上，即使 `127.0.0.1` 可在閘道主機上運作，Windows 防火牆或組織管理的群組原則仍可能封鎖公布的 LAN URL。請在 Windows 主機上執行 `openclaw gateway status --deep`；它會回報可能遭封鎖的連接埠、設定檔不符，以及原則可能忽略的本機防火牆規則。
</Note>

驗證會在 WebSocket 交握期間透過下列方式提供：

- `connect.params.auth.token`
- `connect.params.auth.password`
- 當 `gateway.auth.allowTailscale: true` 時，使用 Tailscale Serve 身分標頭
- 當 `gateway.auth.mode: "trusted-proxy"` 時，使用受信任 Proxy 身分標頭

儀表板設定面板會保留目前瀏覽器分頁工作階段的權杖及所選的閘道 URL；密碼不會持久保存。初次連線時，初始設定通常會產生閘道權杖，以進行共用祕密驗證；但當 `gateway.auth.mode` 為 `"password"` 時，也可使用密碼驗證。

## 裝置配對（首次連線）

從新的瀏覽器或裝置連線時，通常需要進行**一次性配對核准**，顯示為 `disconnected (1008): pairing required`。

<Steps>
  <Step title="列出待處理的要求">
    ```bash
    openclaw devices list
    ```
  </Step>
  <Step title="依要求 ID 核准">
    ```bash
    openclaw devices approve <requestId>
    ```
  </Step>
</Steps>

如果瀏覽器使用已變更的驗證詳細資料（角色／範圍／公開金鑰）重試配對，先前的待處理要求會被取代，並建立新的 `requestId`；核准前請重新執行 `openclaw devices list`。

將已配對瀏覽器從讀取存取權切換為寫入／管理員存取權，會被視為核准升級，而不是無提示地重新連線：OpenClaw 會保留舊核准有效、封鎖權限更廣的重新連線，並要求你明確核准新的範圍組合。

核准後，系統會記住該裝置，除非你使用 `openclaw devices revoke --device <id> --role <role>` 撤銷核准，否則不需重新核准。如需瞭解權杖輪替、撤銷，以及 Paperclip／`openclaw_gateway` 首次執行核准流程，請參閱[裝置命令列介面](/zh-TW/cli/devices)。

<Note>
- 直接的本機回送瀏覽器連線（`127.0.0.1`／`localhost`）會自動核准。
- 當 `gateway.auth.allowTailscale: true`、Tailscale 身分驗證通過，且瀏覽器提供其裝置身分時，Tailscale Serve 可讓 Control UI 操作員工作階段略過配對往返流程。沒有裝置身分的瀏覽器和節點角色連線仍會遵循一般裝置檢查。
- 直接 Tailnet 繫結、LAN 瀏覽器連線，以及沒有裝置身分的瀏覽器設定檔，仍需明確核准。
- 每個瀏覽器設定檔都會產生唯一的裝置 ID，因此切換瀏覽器或清除瀏覽器資料後需要重新配對。

</Note>

## 配對行動裝置

已配對的管理員可以建立 iOS／Android 連線 QR Code，而不必開啟終端機：

<Steps>
  <Step title="開啟行動裝置配對">
    選取 **Devices**，然後在 **Devices** 卡片中按一下 **Pair mobile device**。
  </Step>
  <Step title="連線手機">
    在 OpenClaw 行動應用程式中，開啟 **Settings** → **Gateway** 並掃描 QR Code。你也可以改為複製並貼上設定碼。
  </Step>
  <Step title="確認連線">
    官方 iOS／Android 應用程式會自動連線。如果 **Pending approval** 顯示要求，請先檢查其角色與範圍再核准。
  </Step>
</Steps>

建立設定碼需要 `operator.admin`；不具備該權限的工作階段會停用此按鈕。設定碼包含短期有效的啟動認證資訊，因此在有效期間，請將 QR Code 和複製的程式碼視同密碼處理。若要進行遠端配對，閘道必須解析為 `wss://`（例如透過 Tailscale Serve/Funnel）；單純的 `ws://` 僅限回送和私人 LAN 位址。如需完整的安全性和後援詳細資料，請參閱[配對](/zh-TW/channels/pairing#pair-from-the-control-ui-recommended)。

## 個人身分（瀏覽器本機）

Control UI 支援附加至外送訊息的每瀏覽器個人身分（顯示名稱和頭像），以便在共用工作階段中標明來源。它儲存在瀏覽器儲存空間中，範圍限定於目前的瀏覽器設定檔，不會同步至其他裝置，也不會在伺服器端持久保存；但你傳送的訊息仍會保留一般對話記錄作者中繼資料。清除網站資料或切換瀏覽器會將其重設為空白。

助理頭像覆寫採用相同的瀏覽器本機模式：上傳的覆寫會在本機覆蓋由閘道解析的身分，且絕不會透過 `config.patch` 往返傳輸。共用的 `ui.assistant.avatar` 設定欄位仍可供直接寫入該欄位的非 UI 用戶端使用。

## 執行階段設定端點

Control UI 會從 `/control-ui-config.json` 擷取其執行階段設定，並相對於閘道的 Control UI 基底路徑解析（例如基底路徑 `/__openclaw__/` 下的 `/__openclaw__/control-ui-config.json`）。該端點受到與其餘 HTTP 介面相同的閘道驗證保護：未驗證的瀏覽器無法擷取它，而成功擷取需要有效的閘道權杖／密碼、Tailscale Serve 身分，或受信任 Proxy 身分。

## 閘道主機狀態

開啟 **Settings → General**，即可查看 **Gateway Host** 卡片，其中包含閘道機器、LAN 位址、作業系統、執行階段、運作時間、CPU 負載、記憶體和狀態磁碟區空間。卡片顯示時會每 10 秒透過 `system.info` 閘道 RPC 重新整理，這需要 `operator.read` 範圍。較舊的閘道和不具備該範圍的連線不會顯示此卡片。

## 語言支援

Control UI 首次載入時會根據你的瀏覽器地區設定進行本地化。若要稍後覆寫，請開啟 **Settings -> General -> Language**（選擇器位於 General 頁面，而不是 Appearance 下方）。

- 支援的地區設定：`en`、`ar`、`de`、`es`、`fa`、`fr`、`hi`、`id`、`it`、`ja-JP`、`ko`、`nl`、`pl`、`pt-BR`、`ru`、`th`、`tr`、`uk`、`vi`、`zh-CN`、`zh-TW`
- 非英文翻譯會在瀏覽器中延遲載入。
- 所選地區設定會儲存在瀏覽器儲存空間中，並在日後造訪時重複使用。
- 缺少的翻譯鍵會退回英文。

文件翻譯會針對相同的非英文地區設定集合產生，但文件網站內建的 Mintlify 語言選擇器只會列出 Mintlify 接受的地區設定代碼。泰文（`th`）和波斯文（`fa`）文件仍會在發布存放庫中產生；在 Mintlify 支援這些代碼之前，它們可能不會出現在該選擇器中。

## 外觀主題

Appearance 面板內建 Claw、Knot 和 Dash 主題（預設為 Claw），另有一個瀏覽器本機 tweakcn 匯入插槽。若要匯入主題，請開啟 [tweakcn 編輯器](https://tweakcn.com/editor/theme)，選擇或建立主題，按一下 **Share**，然後將複製的連結貼到 Appearance。匯入工具也接受 `https://tweakcn.com/r/themes/<id>` 登錄 URL、類似 `https://tweakcn.com/editor/theme?theme=amethyst-haze` 的編輯器 URL、相對 `/themes/<id>` 路徑、原始主題 ID，以及 `amethyst-haze` 等預設主題名稱。

匯入的主題只會儲存在目前的瀏覽器設定檔中；它們不會寫入閘道設定，也不會跨裝置同步。取代匯入的主題會更新該本機插槽；如果匯入的主題當時為使用中，清除它會切換回 Claw。

Appearance 也有 Text size 設定。它會套用至聊天文字、撰寫器文字、工具卡片和聊天側邊欄，並讓文字輸入欄至少保持 16px，避免行動版 Safari 在聚焦時自動縮放。

主題、主題模式、文字大小、語言和聊天顯示偏好設定會透過閘道設定（`ui.prefs`）同步，因此可跨裝置沿用，且代理程式可透過核准閘門變更這些設定——已連線的用戶端會透過閘道的 `config.changed` 通知即時套用變更。每個瀏覽器都會保留本機鏡像，以便立即啟動；無法寫入設定的用戶端（檢視者範圍、離線）會將變更保留在裝置本機。如需詳細資訊，請參閱[設定參考](/zh-TW/gateway/configuration-reference#ui)。

## OpenClaw 系統維護

開啟 **Settings → Ask OpenClaw**，與系統設定和修復代理程式交談。在初始設定之外，此頁面每次造訪最多可顯示一個可關閉的事件晶片。對於例行閘道流量，它會保持靜默，且僅對以下健康狀態快照做出反應：回報設定重新載入器已停用、已設定的頻道中斷連線／效能下降、頻道探測失敗，或頻道認證資訊無法使用。只有當較新的事件更嚴重時，才會取代待處理的晶片；關閉或使用晶片後，該次造訪不再顯示事件提示。按一下晶片會將其診斷問題作為真正的 `openclaw.chat` 訊息傳送，因此對話記錄會記錄該要求，而 OpenClaw 會執行診斷。初始設定期間絕不會顯示這些事件晶片。

## 管理外掛

開啟側邊欄中的 **Plugins**，或使用相對於已設定 Control UI 基底路徑的
`/settings/plugins`，即可在不離開 Control UI 的情況下瀏覽及管理外掛。
例如，基底路徑 `/openclaw` 會使用 `/openclaw/settings/plugins`。
即使所有選用外掛都已停用，此頁面也一律可用。

Plugins 是具有四個分頁的中心：**Installed** 和 **Discover** 在
`/settings/plugins` 管理外掛程式碼，**Skills** 在 `/skills`
提供每代理程式的技能管理員，而 **Workshop** 在 `/skills/workshop`
提供 Skill Workshop 提案審查。每個分頁都保有自己的 URL，而側邊欄
會為所有分頁顯示單一 Plugins 項目。

**Installed** 分頁會依類別分組顯示完整的本機清單，並附上概覽計數。
每一列都可開啟詳細資料檢視；其溢位（`…`）選單可啟用
或停用外掛，並為外部安裝的外掛提供 **Remove**。它也會列出已設定的
[MCP 伺服器](/zh-TW/cli/mcp)，並支援直接新增、停用及移除。相同的伺服器控制項
也位於 **Settings → MCP**。**Discover** 分頁就是商店：其中包含 OpenClaw
隨附的精選外掛、官方外部外掛，以及熱門服務的一鍵式 MCP 連接器。
在搜尋方塊中輸入文字會直接查詢 [ClawHub](https://clawhub.ai/plugins)，
並附加一個 **From ClawHub** 區段，其中包含下載次數和來源驗證徽章。
深層連結可使用 `/settings/plugins?tab=discover` 直接指向商店。

**Skills** 分頁會保留技能狀態報告、啟用／停用切換、API 金鑰輸入，
以及行內 ClawHub 技能搜尋，範圍限定於所選代理程式。**Workshop**
分頁會保留 Skill Workshop 看板和 Today 的[技能提案](/zh-TW/tools/skill-workshop)
審查流程。**Find skill ideas** 會由新到舊審查一個有界範圍內的重要工作階段，
並將所有結果保留為待處理提案。面板會顯示累計涵蓋範圍；**Scan earlier work**
會從持久保存的游標繼續，較舊的歷史記錄用盡後則會變成 **Scan new work**。
停用自主自我學習時仍可手動審查歷史記錄，並使用所選代理程式已設定的模型。

隨附外掛已存在於閘道上，並會顯示 **Enable** 或 **Disable**，而不是
**Install**。例如，Workboard 隨 OpenClaw 提供，但預設停用，因此其動作為
**Enable**。綑綁外掛無法移除，只能停用。

讀取目錄和搜尋 ClawHub 需要 `operator.read`。安裝、
啟用、停用或移除外掛，以及變更 MCP 伺服器需要
`operator.admin`；唯讀操作員無法執行這些動作。

ClawHub 安裝會透過閘道執行，並採用與其他由閘道中介的安裝相同的信任、完整性
和外掛安裝原則檢查。安裝
或移除外掛程式碼需要重新啟動閘道。若外掛和目前的
閘道執行階段支援，啟用或停用已安裝的外掛可在不重新啟動的情況下套用；否則 UI 會回報
需要重新啟動。由 OAuth 支援的 MCP 連接器在新增後，需要從命令列介面執行一次
`openclaw mcp login <name>`。

此頁面刻意著重於清單、探索、安裝、啟用
和移除。若要使用任意 npm、git 或
本機路徑來源、更新及進階外掛設定，請參閱 [`openclaw plugins`](/zh-TW/cli/plugins)。

## 應用程式與擴充功能

從側邊欄的 **More** 選單、命令選擇區或
側邊欄代理程式選單（**Get the apps**）開啟 **Apps**，或使用相對於
已設定 Control UI 基底路徑的 `/apps`。此頁面彙整每個
OpenClaw 夥伴介面的安裝連結：[iOS](/zh-TW/platforms/ios) 和
[Android](/zh-TW/platforms/android) 應用程式、與其綑綁的 Apple Watch 和 Wear OS 夥伴應用程式、
[macOS](/zh-TW/platforms/macos)、[Windows](/zh-TW/platforms/windows)
和 [Linux](/zh-TW/platforms/linux) 桌面應用程式、
[Chrome 擴充功能](/zh-TW/tools/chrome-extension)、含

## 側邊欄導覽

側邊欄以代理程式為核心組織所有內容。頂端的身分列是使用中的代理程式；其下方的 **Pages** 區段從 **Home** 開始——這是代理程式持續進行的主要工作階段，並以徽章顯示其未讀或執行中狀態——接著是釘選的目的地（預設為 **Usage**、**Automations** 和 **Plugins**）。Pages 標題上的自訂控制項會開啟一個選單，其中包含所有其他目的地（包括外掛提供的分頁）以及 **Edit pinned items**；在導覽區域按一下滑鼠右鍵可直接開啟釘選編輯器。下方的工作階段清單分為不同區域：**Threads** 用於代理程式的聊天工作階段（主要工作階段位於 Home 之後；由其產生的工作階段會在此顯示為頂層討論串，而具名討論串不會顯示類型前綴）、**Groups** 用於群組和聊天室對話，以及 **Coding** 用於繫結至受管理工作樹或執行節點的工作階段（資料列會顯示一行 `repo ⎇ branch` 加上節點主機）、由 ACP 支援的測試框架工作階段，以及 Codex/Claude 命令列介面目錄。Coding 在首次執行時預設收合，並會記住你的選擇；收合的標題會保留正確數量，且所含工作階段執行時會顯示執行中指示器。自訂群組（工作階段 `category`）和 **Pinned** 資料列位於 Threads 上方，而將工作階段指派至自訂群組時，一律優先於自動區域分類。Threads 標題包含排序控制項（Created 或 Last updated，以及 Group by 切換開關）和可開啟新工作階段頁面的 **+**。開啟工作階段會移動選取醒目提示，但不會重新排列資料列。近期有子執行的父工作階段會顯示展開控制項和子項數量；展開後即可檢查巢狀子工作階段、即時或終止狀態及執行階段，而不必離開側邊欄。選取子工作階段會開啟其聊天，並自動顯示其祖先路徑。子資料列不參與根層級群組、釘選、拖曳、多重選取和分頁；收合的區域不會占用可見頁面額度。自上次讀取後有新活動的工作階段會顯示未讀圓點，開啟後即標示為已讀。代理程式也可以發布一行簡短且會到期的狀態，並可選擇以精選的琥珀色圖示要求注意；當你開啟工作階段、傳送下一則訊息、明確清除該宣告，或其 TTL 到期時，該宣告就會清除。雲端工作者生命週期狀態使用地球徽章；本機和已收回的工作階段不顯示位置徽章，因為本機執行是預設方式。每個根工作階段資料列都有內容選單（三點按鈕或按一下滑鼠右鍵），其中包含 Pin/Unpin、Mark as unread/read、Rename、Fork、Move to group（包括 New group 和 Remove from group）、Archive 和 Delete；觸控版面配置會讓直接釘選和選單控制項保持可見。按住 Cmd/Ctrl 並按一下可切換根資料列的多重選取狀態，按住 Shift 並按一下則會依可見順序延伸選取範圍；之後在選取的資料列上開啟選單，便會提供批次動作（Mark N as unread/read、Move N to group、Archive N、Delete N），並套用至每個選取的工作階段，批次刪除只需確認一次。將根工作階段拖曳到 **Pinned** 即可釘選，或拖曳到自訂群組即可移動。自訂群組標題可以收合、展開或拖曳以重新排序；群組名稱及其順序會儲存在閘道（`sessions.groups.*`）中，因此可跨瀏覽器沿用，而收合狀態則保留在瀏覽器設定檔中。群組標題也有選單（三點按鈕或按一下滑鼠右鍵），其中包含 Rename group、New group 和 Delete group；重新命名或刪除群組時，會在伺服器端更新每個成員工作階段（包括已封存的工作階段），而刪除群組會保留其中的工作階段，並將其移回 Threads。

## 新工作階段頁面

側邊欄工作階段清單標題中的 **+** 會在 `/new` 開啟全頁草稿：在你傳送第一則訊息前不會建立任何內容。訊息方塊上方的目標列可選擇工作階段的運作位置：代理程式（多代理程式設定）、執行作業的執行位置（**Gateway · local** 或公開 `system.run` 的已配對節點；需要 `operator.admin`）、資料夾（預設為代理程式工作區；其他絕對閘道路徑需要 `operator.admin` 和工作樹），以及選用的 **Worktree** 切換開關，其中包含基礎分支選擇器（由 `worktrees.branches` 支援，因此不會執行擷取）和選用的工作樹名稱（分支會變成 `openclaw/<name>`）。撰寫器頁尾可選擇新工作階段的模型和推理層級，而雲端啟動會在將工作階段分派給工作者前保存這兩項選擇。資料夾標籤的瀏覽按鈕會開啟由僅限管理員使用的 `fs.listDir` 方法支援的內嵌目錄選擇器。其頂層會顯示閘道和每個已知節點；離線節點和不支援目錄瀏覽的節點仍會顯示，但會停用。選取閘道會從目前資料夾或閘道主目錄開始。選取具備功能的節點會瀏覽該節點的主機檔案系統、將執行繫結至該節點，並直接使用所選的絕對節點路徑（受管理工作樹仍僅限閘道）。提交會使用第一則訊息呼叫 `sessions.create`，因此執行會在同一次往返中開始，且 UI 會跳至新工作階段的聊天。如果閘道建立工作階段，但拒絕第一次傳送，聊天會在重新載入後保留提示和錯誤；**Retry** 會透過已建立的工作階段重新傳送，而不會另建一個工作階段。

在 **Settings** 中，專用側邊欄包含 **Ask OpenClaw**，並以 **Search settings** 欄位開頭，方便快速尋找設定區段。

側邊欄頂端的 **Search** 欄位會開啟命令選擇區（⌘K）。按一下側邊欄頂端的代理程式身分列會開啟代理程式選單；**Home** 會開啟主要工作階段。當某些項目需要處理時——失敗或逾期的排程工作、即將到期或已到期的模型驗證——側邊欄頁尾上方會顯示精簡的注意事項標籤，按一下即可前往其所屬頁面。身分列會顯示代理程式的頭像（身分圖片或表情符號）、名稱、連線圓點和即時副標題。按一下後會開啟代理程式選單：代理程式切換器（多代理程式設定）、「What can this agent do?」、**Agent settings**、**Settings**、行動裝置配對、**Docs**、組建標籤，以及色彩模式切換開關。代理程式超過十個的名冊會顯示篩選欄位，並優先列出已釘選的代理程式；可從 Agents 設定頁面釘選或取消釘選代理程式，釘選集合會儲存在瀏覽器設定檔中。選擇代理程式會將 Chat、Usage、Automations、Tasks、Workboard 和 Sessions 的範圍限定於該代理程式。每個限定範圍的頁面都會提供 **Agent** 控制項，並以 **All agents** 作為退出方式；這會擴大共用頁面的範圍，但不會變更實際的聊天代理程式，而直接工作階段連結仍會開啟其目標。Agents 設定頁面會保留自己的 `?agent=` 選取項目，且不會跟隨共用頁面範圍。頁尾列包含產品標誌、組建標籤、閘道連線圓點和 Settings 捷徑。當閘道從原始碼簽出執行，且所在分支不是 `main` 時，頁尾也會以紅色顯示該分支名稱，讓人一眼就能看出這是非發布版閘道（發布版安裝永遠不會顯示）。按 Shift-Command-Comma 會開啟 **Settings**，且不會覆寫瀏覽器的 Command-Comma 捷徑。側邊欄標題也包含收合切換開關（⌘B）；收合後會完全隱藏側邊欄，以提供全寬工作區，而浮動展開控制項（或 ⌘B）可將其帶回；macOS 應用程式則會改在標題列中原生託管該切換開關。側邊欄是桌面版唯一的導覽介面，沒有頂端列。狹窄檢視區會將側邊欄替換為滑入式抽屜，並隱藏在精簡標題列後方；該標題列包含抽屜切換開關、品牌和命令選擇區搜尋；在手機上，Chat 會將該導覽列整合至其標題列，選單和搜尋控制項位於工作階段標題旁。在 macOS 應用程式中，獨立的標題列會將標題列淨空空間整合成視窗控制項旁的單一精簡區帶。導覽使用一般瀏覽器歷程記錄，因此瀏覽器的上一頁／下一頁按鈕可以在其中巡覽；macOS 應用程式還會在視窗控制項旁新增原生側邊欄切換開關和觸控板滑動手勢，側邊欄展開時，其右側邊緣會顯示上一頁／下一頁按鈕，而收合時則會顯示原生搜尋（命令選擇區）和新工作階段按鈕。

待處理的核准也會在側邊欄頁尾上方產生注意事項標籤；
選取後即可開啟其所屬的 Approvals 頁面。

    - 透過閘道 WS 與模型聊天（`chat.history`、`chat.send`、`chat.abort`、`chat.inject`）。
    - 聊天記錄重新整理時，會請求有範圍限制的近期時間窗，並限制每則訊息的文字量，因此大型工作階段不會迫使瀏覽器在聊天可用前先算繪完整的逐字記錄承載資料。

    - 透過瀏覽器即時工作階段進行語音交談。OpenAI 使用直接 WebRTC，Google Live 使用透過 WebSocket 傳送、受限且僅能使用一次的瀏覽器權杖，而僅限後端的即時語音外掛則使用閘道中繼傳輸。支援視訊的瀏覽器工作階段可在「設定」中選擇裝置本機攝影機，或從即時預覽切換攝影機；瀏覽器會為即時供應商擷取 JPEG 畫面，而不會透過閘道串流攝影機視訊。由用戶端擁有的供應商工作階段以 `talk.client.create` 啟動；閘道中繼工作階段以 `talk.session.create` 啟動。中繼會將供應商認證資訊保留在閘道上，同時瀏覽器透過 `talk.session.appendAudio` 串流麥克風 PCM，並透過 `talk.client.toolCall` 轉送 `openclaw_agent_consult` 供應商工具呼叫，以套用閘道政策及使用已設定的較大型 OpenClaw 模型，且透過 `talk.client.steer` 或 `talk.session.steer` 路由執行中工作的語音引導。
    - 在聊天中串流工具呼叫與即時工具輸出卡片（代理程式事件）。工具活動會依類型顯示為不同資料列：Shell 命令會顯示具有語法醒目提示的命令及終端機樣式輸出；支援的編輯與寫入呼叫會顯示有範圍限制的行內差異、可用時的行號，以及 `+added -removed` 統計資料；連續呼叫則會摺疊成摘要，例如「執行了 13 個命令、讀取了 6 個檔案、編輯了 9 個檔案」。工作執行期間，最新的執行中呼叫會作為群組標頭名稱。展開資料列即可檢查其餘引數與原始輸出。
    - 複雜工具呼叫（長 Shell 命令、引數繁多的外掛工具）可選用 AI 用途標題，透過 `gateway.controlUi.toolTitles: true` 啟用（預設關閉）。標題來自透過標準公用模型路由批次處理的 `chat.toolTitles` 方法：優先使用明確指定的 `utilityModel`（由操作員選擇的供應商，與其他公用工作相同），否則使用工作階段供應商宣告的預設小型模型；並依代理程式在閘道端快取。若未選擇啟用，或沒有可用的低成本模型，資料列會保留其確定性標籤，且不會進行模型呼叫。
    - 啟動或關閉由模型建議的暫時性後續工作；接受建議後，會以提議的提示詞開啟新的受管理工作樹工作階段。
    - 活動分頁會根據現有的 `session.tool`／工具事件傳遞，顯示僅儲存於瀏覽器本機、優先遮蔽敏感資訊的即時工具活動摘要。

## 匯入助理記憶

開啟 **設定** → **匯入記憶**，將本機 Codex 或 Claude Code 記憶
帶入 OpenClaw 代理程式。閘道會自行探索其所在主機上支援的本機記憶，
因此遠端控制 UI 會從閘道電腦匯入，而非瀏覽器電腦。

1. 選擇目的地代理程式。
2. 檢視偵測到的來源集合及 Markdown 檔名。檔案內容
   不會在規劃回應中傳送，也不會顯示於頁面上。
3. 選取要匯入的集合並確認。套用會在寫入前重新建置規劃，
   使過時的選擇能安全地失敗。
4. 如果檔案已存在，請啟用 **取代現有匯入項目**、重新整理
   預覽，然後確認取代。

Codex 僅匯入其整合後的 `MEMORY.md` 和 `memory_summary.md`。Claude
Code 會從專案自動記憶目錄及已設定的
`autoMemoryDirectory` 匯入 Markdown；它不會透過此頁面匯入工作階段、設定、指示或
認證資訊。檔案會複製到所選工作區中的 `memory/imports/` 下方，
作用中的記憶外掛可在該處為其建立索引。來源絕不會變更。

規劃和套用需要 `operator.admin`。每次套用都會在有狀態資料時建立經驗證的
OpenClaw 備份、寫入已遮蔽的遷移報告，並在取代現有目的地檔案前保留
項目層級的備份。路徑和
回憶行為請參閱[記憶概觀](/zh-TW/concepts/memory#import-from-coding-assistants)。

## MCP 頁面

專屬 MCP 頁面是位於 `mcp.servers` 下、供操作員檢視由 OpenClaw 管理之 MCP 伺服器的介面。它本身不會啟動 MCP 傳輸；請用它檢查及編輯已儲存的設定，並在需要即時伺服器證明時使用 `openclaw mcp doctor --probe`。

典型工作流程：

1. 從側邊欄開啟 **MCP**。
2. 檢查摘要卡片中的伺服器總數、已啟用、OAuth 及已篩選伺服器數量。
3. 檢視每個伺服器列的傳輸方式、啟用狀態、驗證、篩選器、逾時及命令提示。
4. 直接在 MCP 頁面新增、啟用、停用或移除伺服器。請明確選擇 Streamable HTTP、SSE 或 stdio；stdio 命令列接受以引號括住的引數，例如包含空格的路徑。單鍵連接器和探索功能請使用 **外掛** 頁面。
5. 編輯限定範圍的 `mcp` 設定區段，以設定環境變數、工作目錄、標頭、TLS/mTLS 路徑、OAuth 中繼資料、工具篩選器及 Codex 投影中繼資料等進階伺服器欄位。
6. 使用 **儲存** 寫入設定；若要讓執行中的閘道套用變更後的設定，則使用 **儲存並發布**。
7. 從終端機執行 `openclaw mcp status --verbose`、`openclaw mcp doctor --probe` 或 `openclaw mcp reload`，以進行靜態診斷、即時證明或清除快取的執行階段。

此頁面會在轉譯前遮蔽包含認證資訊且類似 URL 的值，並在命令片段中以引號括住伺服器名稱，因此複製的命令即使包含空格或 Shell 中繼字元仍可運作。完整的命令列介面和設定參考：[MCP](/zh-TW/cli/mcp)。

## 活動分頁

「活動」分頁位於 **設定 › 系統**，緊鄰「日誌」和「偵錯」。它是暫時性的瀏覽器本機即時工具活動觀察器，衍生自支援聊天工具卡片的相同閘道 `session.tool`／工具事件串流。它不會新增其他閘道事件系列、端點、永久活動儲存區、指標摘要或外部觀察器串流。

活動項目僅保留經過清理的摘要，以及已遮蔽、截斷的輸出預覽。工具引數值不會儲存在活動狀態中；UI 會顯示引數已隱藏，且只記錄引數欄位數量。記憶體內的清單跟隨目前的瀏覽器分頁，在控制 UI 內瀏覽時會保留，並於重新載入頁面、切換工作階段或按下 **清除** 時重設。

## 操作員終端機

可停駐的操作員終端機預設為停用。若要啟用，請設定 `gateway.terminal.enabled: true` 並重新啟動閘道。終端機需要 `operator.admin` 連線，並會在作用中代理程式工作區開啟主機 PTY。新分頁會跟隨目前選取的聊天代理程式。

<Warning>
終端機是不受限制的主機 shell，並會繼承閘道程序環境。僅在受信任的操作員部署中啟用。OpenClaw 會拒絕為使用 `sandbox.mode: "all"` 的代理程式建立終端機工作階段；將作用中的代理程式變更為該模式，會關閉其現有及進行中的終端機工作階段。
</Warning>

使用 **Ctrl + 反引號** 切換停駐面板。版面配置支援停駐於底部和右側、隨瀏覽器檢視區調整大小，並保留多個 shell 分頁。關於 `gateway.terminal.enabled` 和選用的 `gateway.terminal.shell` 覆寫，請參閱[閘道設定](/zh-TW/gateway/configuration-reference#gateway)。

經擁有者授權且未受沙箱限制的代理程式，可以使用 `terminal` 工具執行應由操作員監看的長時間或互動式工作。每次工具呼叫都可以開啟、讀取、寫入、調整大小、關閉或列出代理程式自己的閘道 PTY。新工作階段預設會開啟共同附加的控制介面分頁，讓代理程式與操作員共享輸出，且雙方皆可輸入或調整大小。代理程式存取權的範圍精確限制於特定工作階段：代理程式無法讀取或控制由操作員建立的終端機，或由另一個代理程式工作階段開啟的終端機。

將一個或多個檔案拖曳至作用中的終端機，或使用迴紋針按鈕選擇檔案。OpenClaw 會在擁有該 PTY 的機器上暫存各檔案，並在游標處貼上經 shell 引號處理的絕對路徑；它絕不會按下 Enter 或執行輸入。精簡的批次指示器會顯示目前檔案和已完成數量。取消會停止剩餘批次且不貼上路徑；失敗的傳輸會保持可見，因此你可以從該檔案重試，而不必重新上傳已完成的檔案。支援圖片、PDF、封存檔和其他檔案類型，每個檔案上限為 16 MiB。暫存檔案在 POSIX 主機上會使用私有的系統暫存目錄（目錄模式 `0700`、檔案模式 `0600`），在 Windows 上則使用位於使用者設定檔 ACL 邊界下的目錄，並設有 24 小時清理計時器，因此請移動或複製任何需要保留的內容。

路徑插入支援 PowerShell、`cmd.exe`，以及可辨識的 POSIX shell（`sh`、Bash、Dash、Ash、Ksh、Zsh 和 Fish），包括 Windows 上的 Git Bash。系統會拒絕其他 shell 覆寫，因為無法安全推斷其引號規則；若要使用原生 WSL 終端機和 Linux 上傳路徑，請在 WSL 內執行閘道。包含 `%` 或 `!` 的 `cmd.exe` 路徑也會遭到拒絕，因為該 shell 即使在雙引號內仍會展開這些字元。

在工作階段側邊欄中探索到的 Codex 和 Claude Code 工作階段，可以在同一個終端機面板內以其原生命令列介面開啟。在 **Settings › Chat** 中，將 **Open Codex/Claude threads in** 設為 **Terminal**，即可讓一般的資料列點擊開啟 `codex resume` 或 `claude --resume`；預設仍為唯讀的 OpenClaw 檢視器。資料列的右鍵選單或三點選單一律提供這兩種選項；當該工作階段符合資格時，檢視器標頭也會包含 **Open in terminal**。

資格是針對各工作階段和各主機分別判定。閘道本機工作階段會在閘道主機上啟動由提供者擁有的繼續執行命令。配對節點工作階段會在所屬節點上啟動允許清單中的提供者命令，並且只轉送該 PTY 的輸出、輸入和調整大小事件；這不會公開一般節點 shell，也不接受瀏覽器提供的命令。檔案上傳會使用獨立且有大小限制的 `terminal.upload` 節點命令，並持續繫結至已開啟的終端機工作階段。該命令首次出現時，請核准節點配對升級。未公布相符終端機繼續執行命令的節點，包括不具雙工串流功能的內嵌工作節點橋接器，仍可使用檢視器，並會顯示無法開啟終端機；較舊的節點仍可執行終端機，但無法接收拖曳的檔案。

由連線擁有的工作階段會在中斷連線後持續存在：頁面重新載入、筆記型電腦進入睡眠或網路短暫中斷時，工作階段會在閘道上分離，而非遭到終止；重新連線後，同一個瀏覽器分頁會重新附加，並重播最近的輸出。分離的連線擁有工作階段會在 `gateway.terminal.detachedSessionTimeoutSeconds` 後遭到終止（預設為 300 秒；`0` 會恢復為中斷連線時即終止）。附加至其中一個工作階段時，仍會採用 tmux 風格的接管方式。

代理程式擁有的工作階段不會繫結至瀏覽器連線。`terminal.attach` 會將每個瀏覽器新增為檢視者，而不接管擁有權；關閉檢視器分頁只會分離該瀏覽器。PTY 會持續存在，直到所屬代理程式將其關閉、其程序結束、政策停用它，或閘道關閉。`terminal.list` 會將每個項目標示為連線擁有或代理程式擁有，而 `terminal.text` 可讓管理員連線讀取最近的純文字輸出，而不需附加。

終端機也可在 `/?view=terminal` 作為全螢幕、僅含終端機的文件使用。iOS 和 Android 應用程式會在其終端機畫面內嵌此頁面，並重複使用已儲存的閘道認證資訊；可用性受相同的 `gateway.terminal.enabled` 和 `operator.admin` 閘門控制，且當所連線的閘道未提供終端機時，頁面會顯示通知。

## 瀏覽器面板

控制介面隨附可停駐的瀏覽器面板，可在任何一般網頁瀏覽器中呈現由閘道控制的瀏覽器（即代理程式透過[瀏覽器工具](/zh-TW/tools/browser-control)操控的同一個瀏覽器），不需要原生 WebView。當所連線的閘道向 `operator.admin` 連線公布 `browser.request` 時，該面板便會顯示；討論串工作區邊欄中的地球按鈕可切換該面板。面板會顯示即時頁面快照，並提供分頁、可編輯的 URL 列、上一頁／下一頁／重新載入，以及在你的瀏覽器中開啟等功能；它可停駐於右側或底部，並將點擊、滾輪捲動和基本輸入轉送至遠端頁面。

兩種擷取模式可為代理程式封裝頁面內容：

- **註記（鉛筆）**：在頁面上手繪標記。**傳送至聊天**會將筆畫合成至螢幕截圖中，把圖片附加至作用中的聊天撰寫區，並預先填入描述頁面 URL、標題和每個標記區域的提示，讓代理程式確切知道你圈選了什麼。
- **檢查（指標）**：將游標停留於元素上，即可查看游標下方的元素（選擇器、無障礙名稱、角色、大小）；點擊後會透過相同的撰寫區流程，傳送該元素的詳細資料和加上醒目標示的螢幕截圖。檢查、滾輪捲動和上一頁／下一頁需要 `browser.evaluateEnabled`（預設開啟）。

macOS 應用程式會保留其原生連結瀏覽器側邊欄，用於顯示在儀表板中點擊的連結；瀏覽器面板也可在其中運作，並且是在其他所有平台上註記頁面的方式。

## 聊天行為

  <Accordion title="傳送與歷史記錄語意">
    - `chat.send` 為**非阻塞式**：它會立即以 `{ runId, status: "started" }` 確認，回應則透過 `chat` 事件串流傳送。受信任的 Control UI 用戶端也可能收到選用的 ACK 計時中繼資料，以供本機診斷。
    - 聊天上傳支援圖片及非影片檔案。圖片會保留原生圖片路徑；其他檔案則儲存為受管理的媒體，並在歷史記錄中顯示為附件連結。
    - 執行期間以相同的 `idempotencyKey` 重新傳送會傳回 `{ status: "in_flight" }`，完成後則傳回 `{ status: "ok" }`。
    - `chat.history` 回應設有大小上限，以確保 UI 安全。當逐字記錄項目過大時，閘道可能會截斷過長的文字欄位、省略龐大的中繼資料區塊，並以預留位置取代超出大小限制的訊息（`[chat.history omitted: message too large]`）。
    - 當可見的助理訊息在 `chat.history` 中遭到截斷時，側邊閱讀器可依需求透過 `chat.message.get`，使用 `sessionKey`、需要時的有效 `agentId`，以及逐字記錄 `messageId`，擷取經顯示正規化的完整逐字記錄項目。如果閘道仍無法傳回更多內容，閱讀器會顯示明確的無法使用狀態，而不會默默重複遭截斷的預覽。
    - 助理產生的圖片會保存為受管理的媒體參照，並透過經驗證的閘道媒體 URL 提供，因此重新載入不需仰賴原始 base64 圖片承載資料持續留在聊天歷史記錄回應中。
    - 呈現 `chat.history` 時，Control UI 會從可見的助理文字中移除僅供顯示的行內指令標籤（例如 `[[reply_to_*]]` 和 `[[audio_as_voice]]`）、純文字工具呼叫 XML 承載資料（包括 `<tool_call>...</tool_call>`、`<function_call>...</function_call>`、`<tool_calls>...</tool_calls>`、`<function_calls>...</function_calls>`，以及遭截斷的工具呼叫區塊），以及外洩的 ASCII／全形模型控制權杖。如果助理項目的全部可見文字僅為完全相符的靜默權杖 `NO_REPLY`／`no_reply` 或心跳確認權杖 `HEARTBEAT_OK`，則會省略該項目。
    - 在進行中的傳送及最後一次歷史記錄重新整理期間，如果 `chat.history` 短暫傳回較舊的快照，聊天檢視仍會顯示本機樂觀更新的使用者／助理訊息；閘道歷史記錄追上進度後，標準逐字記錄會取代這些本機訊息。
    - 即時 `chat` 事件代表傳遞狀態，而 `chat.history` 則根據持久化的工作階段逐字記錄重建。工具最終事件發生後，Control UI 會重新載入歷史記錄，且只合併一小段樂觀更新的尾端內容；逐字記錄邊界記載於 [WebChat](/zh-TW/web/webchat)。
    - `chat.inject` 會將助理備註附加至工作階段逐字記錄，並廣播 `chat` 事件以供僅限 UI 的更新使用（不執行代理程式、不傳遞至頻道）。
    - 側邊欄會依代理程式區段列出每個已載入的有效工作階段，並分為已釘選／頻道／工作／自訂／聊天分類，且只有一個「新增工作階段」動作可開啟草稿對話方塊。開啟可見列只會移動醒目提示。可將工作階段拖曳至「已釘選」加以釘選，或拖曳至自訂群組或「聊天」以移動；自訂群組可收合並支援拖曳重新排序，群組名稱與順序會透過閘道同步，而收合狀態則保留在瀏覽器中。新的儀表板工作階段會非同步根據第一則非命令訊息產生簡潔標題；明確名稱與經驗證的傳送者身分會保持分離，因此絕不會以帳戶名稱作為產生的標題。設定 `agents.defaults.utilityModel`（或 `agents.list[].utilityModel`）可將這個獨立的模型呼叫導向成本較低的模型；如果該獨立模型失敗，標題產生會使用主要模型重試一次。展開另一個代理程式區段即可瀏覽該代理程式的工作階段，而不會離開目前開啟的聊天。
    - 對話串搜尋位於命令選擇區中（⌘K，或側邊欄頂端的「搜尋」欄位）：輸入查詢後，系統會在代理程式間檢索有限數量的相符頁面、篩除內部子項目／排程列，並在導覽命令旁列出可見的相符項目。「對話串」頁面則保留可使用篩選條件的完整可搜尋清單。
    - 每個側邊欄列都保留直接釘選功能，以及包含未讀狀態、重新命名、分支、分組、封存與刪除功能的完整快顯功能表。多選的列（按住 Cmd/Ctrl 並按一下，或按住 Shift 並按一下以選取範圍）會顯示批次功能表，涵蓋未讀狀態、分組、封存與刪除；除非每個選取的工作階段皆可封存，否則批次封存／刪除功能會維持停用。進行中的執行作業與代理程式的主要工作階段無法封存。封存或刪除目前選取的工作階段時，「聊天」會切回該代理程式的主要工作階段。
    - 在 macOS 應用程式中，OpenClaw 標誌會使用視窗控制項旁原本空白的原生標題列區域，而不占用側邊欄的一列。
    - 在桌面寬度下，聊天控制項會維持在單一緊湊列中，並在向下捲動逐字記錄時收合；向上捲動、返回頂端或抵達底部時，控制項會恢復顯示。
    - 連續且重複的純文字訊息會呈現為單一對話泡泡，並附有數量徽章。包含圖片、附件、工具輸出或畫布預覽的訊息不會收合。
    - 使用者訊息泡泡包含逐字記錄動作：游標停留時顯示的倒轉按鈕（確認彈出視窗包含「不要再詢問」選項），以及按右鍵顯示的**倒轉至此處**和**從此處建立分支**。倒轉會將工作階段重新指向該訊息之前的狀態，並將其文字放回編輯器以供編輯及重新傳送（`sessions.rewind`、`operator.admin`）；建立分支會根據該訊息之前的有效路徑前綴建立新工作階段、開啟該工作階段，並以相同文字預先填入其編輯器（`sessions.fork`、`operator.write`）。代理程式工作時，這兩個動作都會停用並顯示說明工具提示；它們僅適用於已保存的使用者訊息，且外部代理程式執行框架擁有對話的工作階段會拒絕這些動作。倒轉只會移動聊天內容——檔案與其他工具的副作用不會還原——且倒轉前的逐字記錄仍會保留在僅附加的工作階段儲存區中。當該儲存區包含多個逐字記錄分支時，聊天標題列會顯示分支功能表，其中包含每個分支的最新訊息、訊息數量與距今時間；選取非有效分支會將目前工作階段切回該保留路徑（`sessions.branches.list`、`operator.read`；`sessions.branches.switch`、`operator.admin`）。代理程式工作時也無法切換分支，而在 RPC 邊界選取已有效的分支會產生具型別的無操作錯誤。使用者泡泡上的獨立隱藏動作只會在目前瀏覽器中隱藏訊息；訊息仍會留在逐字記錄中，代理程式也仍可看到它。
    - 當工作階段的簽出內容位於 GitHub 儲存庫的非預設分支時，聊天檢視會在編輯器上方釘選 PR 方塊：PR 編號、儲存庫、分支、差異數量、CI 狀態膠囊，以及草稿／已合併／已關閉狀態，每一項都會連結至該 PR。該列最多顯示兩個方塊——即時（開啟／草稿）PR 優先——而「顯示更多」按鈕會顯示已收合的已合併／已關閉歷史記錄。CI 狀態膠囊會開啟小型 CI 監控彈出視窗，其中包含通過／失敗／執行中／已略過的檢查數量，以及前往 PR 檢查頁面的連結。偵測會透過 `controlUi.sessionPullRequests` 在伺服器端執行，並在有設定時重複使用閘道的 `GH_TOKEN`／`GITHUB_TOKEN`。達到 GitHub API 速率限制時，方塊會保留最後已知狀態，並顯示狀態可能已過期的警告；關閉方塊後，會在目前瀏覽器設定檔中針對該工作階段隱藏它。在任何 PR 存在之前，該列會顯示分支本身——儲存庫、分支名稱，以及相對於預設分支合併基準的差異 +/− 大小（包括已提交與未提交的工作）。推送的分支有可供比較的提交後，該列會新增「建立 PR」按鈕，以開啟 GitHub 的新建 PR 頁面；在此之前，只要工作階段含有已變更的檔案（已提交、未提交或未追蹤），仍會顯示該列，但不會顯示按鈕。存在開啟或草稿 PR 時，該列會自行隱藏。分支列僅來自本機 git，因此即使 GitHub 受到速率限制時仍可使用；它也會顯示相同的狀態過期警告，因為在限制重設前無法確信「找不到 PR」。
    - 工作階段差異面板會顯示工作階段簽出內容實際變更的項目：工作區側軌或聊天標題列中的分支按鈕會開啟詳細資料面板，以逐檔差異顯示相對於簽出內容預設分支合併基準的分支、未提交及未追蹤工作——包括狀態圓點、重新命名箭頭、逐檔 +/− 數量、可收合檔案，以及差異區塊之間的「N 行未修改」標記。差異會透過 `sessions.diff` 閘道方法（`operator.read` 範圍）在伺服器端計算；二進位檔案與超大檔案會降級為僅顯示統計資料的項目，且只有在連線的閘道宣告 `sessions.diff` 時才會顯示該按鈕。
    - 每個「聊天」窗格都有標題列。按一下工作階段標題即可重新命名；工作區方塊可複製簽出路徑或分支，亦可在主機的檔案管理員中顯示本機閘道工作區。遠端與執行節點工作階段會保留複製動作，但隱藏顯示動作。
    - 每個「聊天」窗格中的對話串工作區側軌會列出對話串檔案、專案檔案及成品。預設停駐於窗格右側邊緣；拖曳其標頭（或使用停駐按鈕）可將其移至底部，且此選擇會儲存在目前瀏覽器設定檔中。收合的側軌完全不占空間：使用 ⇧⌘B 或標題列中的檔案切換按鈕重新開啟，該按鈕會顯示已變更檔案數量徽章。獨立的檔案、工具與畫布詳細資料面板不受影響。
    - 按一下聊天中的檔案參照、已展開讀取／編輯／寫入工具卡片中的檔案路徑，或工作區側軌中的檔案列，即會開啟檔案詳細資料面板：以 CodeMirror 為基礎的程式碼檢視，具備語法醒目提示、行號、跳至指定行、檔案內搜尋、複製動作，以及在外部編輯器中開啟的功能表。當閘道向 `operator.admin` 連線宣告 `sessions.files.set` 時，面板會新增具備未儲存變更追蹤與 Cmd/Ctrl-S 儲存功能的「編輯」模式；未儲存的草稿會在目前瀏覽器分頁中跨檔案、面板及工作階段導覽持續保留，直到明確儲存或捨棄。儲存會根據 `sessions.files.get` 傳回的內容雜湊執行比較後交換：如果檔案自載入後已在磁碟上變更（例如代理程式仍持續工作），面板會顯示衝突通知，並提供「重新載入」（採用最新內容）與「覆寫」（保留本機編輯）動作。寫入會通過與讀取相同且保障檔案系統安全的工作區防護機制——路徑包含範圍、拒絕符號連結／硬式連結，以及 256 KB UTF-8 上限——且只能覆寫現有檔案；編輯器絕不會建立或刪除檔案。
    - 每個「聊天」窗格中的背景工作側軌會列出目前代理程式的背景工作與子代理程式（`tasks.list` 依代理程式設定範圍，並由 `task` 事件保持即時狀態）：執行中的工作會顯示即時經過時間計時器、工具使用次數、目前使用中的工具，以及停止控制項；可收合的已完成區段則會新增執行持續時間；「檢視逐字記錄」連結會在窗格中開啟該工作的子工作階段。可使用標題列的活動切換按鈕開啟；工作快照會預先載入，因此無須先開啟側軌即可顯示執行中數量徽章。「工作」頁面仍會保留完整的跨代理程式記錄。
    - 工作區側欄、背景工作側欄和詳細資料面板會依各窗格本身的寬度調整，而不是依視窗寬度調整：在狹窄窗格或精簡視窗中，兩個側欄都會顯示為底部橫列（側邊停駐控制項會隱藏，直到窗格變寬；當空間僅容納一欄時，工作區側欄會優先占用側邊位置），詳細資料面板則會堆疊在線程下方，並使用水平調整大小控點，而不是與線程共用同一列。手機大小的檢視區仍會以全螢幕開啟詳細資料面板。
    - 聊天標頭中的模型與思考選擇器會透過 `sessions.patch` 立即修補作用中工作階段；它們是持續有效的工作階段覆寫設定，而非僅適用單次傳送的選項。
    - **分割檢視：**從聊天標題列開啟（位於線程差異、背景工作和線程檔案切換鈕旁），接著將作用中窗格向右或向下分割，直到容納所需數量的窗格。每個窗格都有各自的線程、文字記錄、編輯器和工具串流。
    - 具有 `screen` 工具的代理程式，可在具備相應功能的控制介面連線時，要求進行相同的窗格、側邊欄、終端機、瀏覽器、焦點和導覽變更。協定 v1 會將命令套用至每個已連線且具備相應功能的控制介面；請參閱[螢幕](/zh-TW/tools/screen)。
    - 將工作階段從側邊欄拖曳至聊天區，即可在窗格中開啟。動畫式放置預覽會在各區域間滑動並標示結果——在新窗格將占用的確切半區上顯示「分割」，在整個窗格上顯示「在此開啟」——而且也能從單窗格模式進行放置。
    - 作用中的分割窗格會決定側邊欄的選取項目和 URL。其標題列會新增分割與關閉控制項；分隔線可調整各欄與堆疊窗格的大小，而瀏覽器會在本機儲存版面配置，重新載入後仍會保留。
    - 在窄螢幕上，分割檢視會保留版面配置，但只呈現作用中的窗格，包括含有關閉控制項的標頭。
    - 如果在同一工作階段的模型選擇器變更仍在儲存時傳送訊息，編輯器會先等待該工作階段修補完成，再呼叫 `chat.send`，以確保使用所選模型傳送訊息。
    - 輸入 `/new` 會建立並切換至與「新聊天」相同的全新儀表板工作階段，但若已設定 `session.dmScope: "main"`，且目前的父工作階段是代理程式的主要工作階段，則會改為就地重設主要工作階段。輸入 `/reset` 則會沿用閘道對目前工作階段明確執行的就地重設。
    - 聊天模型選擇器會要求取得閘道所設定的模型檢視。如果 `agents.defaults.modelPolicy.allow` 非空白，選擇器將依該原則運作，包括讓供應商範圍目錄維持動態的 `provider/*` 項目。否則，選擇器會顯示已設定的項目，以及具有可用驗證的供應商；`agents.defaults.models` 下的別名和設定不會限制選擇器。完整目錄仍可透過偵錯用的 `models.list` RPC 搭配 `view: "all"` 取得。

  </Accordion>
  <Accordion title="對話模式（瀏覽器即時）">

    **視訊對話** 適用於 OpenAI Realtime WebRTC 與 Google Live 瀏覽器工作階段。按一下相機按鈕、允許存取相機與麥克風，並確認本機預覽。當 `describe_view` 要求視覺內容時，OpenAI 會透過其瀏覽器資料通道傳送一個大小受限的 JPEG 畫面。Google Live 會以支援的每秒最多一個畫面速率，將大小受限的 JPEG 畫面直接從瀏覽器傳送至供應商，並以相機串流狀態回應 `describe_view` 函式呼叫。相機畫面永遠不會經過閘道。停止對話模式會關閉預覽並釋放兩個媒體軌道。供應商線路合約請參閱 Google 的 [Live API 功能](https://ai.google.dev/gemini-api/docs/live-api/capabilities#video)與[函式呼叫指南](https://ai.google.dev/gemini-api/docs/live-api/tools)。

## 連線中斷與重新連線

工作階段建立後，閘道連線中斷不會將你登出。用戶端使用退避機制（從 800 ms 到最多 15 s）自動重試時，儀表板會保持顯示，並在頂端列下方顯示浮動的琥珀色「閘道連線中斷 — 正在重新連線…」膠囊提示。即時更新與即時／工作階段動作會暫停，直到連線恢復；膠囊提示中的 **Retry now** 會強制立即嘗試。聊天內容仍可編輯：一般文字與附件傳送會保留在目前分頁以閘道／工作階段為範圍的瀏覽器儲存空間中，顯示為等待重新連線，並在閘道恢復後自動傳送。離線時，即時控制項與斜線命令仍無法使用。

當此瀏覽器已持有認證資訊（已設定的權杖／密碼或已核准的裝置權杖）時，首次開啟與重新載入會在建立連線期間顯示小型動畫 OpenClaw 標誌，而不會短暫閃現登入閘門。只有在尚未儲存認證資訊，或閘道主動拒絕認證資訊（錯誤的權杖／密碼、已撤銷的配對）時，才會顯示登入閘門——這些狀態需要你輸入資訊，而非等待。

## PWA 安裝與網頁推播

控制使用者介面隨附 `manifest.webmanifest` 與服務工作者，因此現代瀏覽器可將其安裝為獨立 PWA。即使分頁或瀏覽器視窗未開啟，網頁推播仍可讓閘道透過通知喚醒已安裝的 PWA。

在 macOS 應用程式中，「通知」設定頁面顯示的是應用程式的原生通知權限，而非瀏覽器推播，因為應用程式會以原生方式傳送通知。

如果頁面在 OpenClaw 更新後立即顯示 **通訊協定不相符**，請先使用 `openclaw dashboard` 重新開啟儀表板並強制重新整理。如果仍然失敗，請清除儀表板來源的網站資料，或在私人瀏覽器視窗中測試；舊分頁或瀏覽器服務工作者快取可能持續執行更新前的控制使用者介面套件，並將其連線至較新的閘道。

| 介面                                               | 功能                                                                         |
| -------------------------------------------------- | ---------------------------------------------------------------------------- |
| `ui/public/manifest.webmanifest`                   | PWA 資訊清單。可存取後，瀏覽器會提供「Install app」。                        |
| `ui/public/sw.js`                                  | 處理 `push` 事件與通知點擊的服務工作者。                         |
| `state/openclaw.sqlite` → `web_push_vapid_keys`    | 用於簽署網頁推播承載資料的自動產生 VAPID 金鑰組。                            |
| `state/openclaw.sqlite` → `web_push_subscriptions` | 持久儲存的瀏覽器訂閱端點、金鑰及註冊時間戳記。                               |

從已淘汰的 `push/vapid-keys.json` 與 `push/web-push-subscriptions.json` 儲存區升級時，`openclaw doctor --fix` 會匯入其中的資料。執行該修復前請停止閘道，以免較舊的處理程序在匯入期間重新建立已淘汰的狀態。升級後使用網頁推播前，請先執行修復；只要任一已淘汰來源仍存在，或 Doctor 宣告的處理遭到中斷，註冊、傳送、刪除與金鑰解析都會拒絕繼續。閘道執行階段只會讀寫 SQLite。

若要固定金鑰（多主機部署、密鑰輪替或測試），請透過閘道處理程序的環境變數覆寫 VAPID 金鑰組：

- `OPENCLAW_VAPID_PUBLIC_KEY`
- `OPENCLAW_VAPID_PRIVATE_KEY`
- `OPENCLAW_VAPID_SUBJECT`（預設為 `https://openclaw.ai`）

控制使用者介面使用以下受範圍限制的閘道方法註冊及測試瀏覽器訂閱：

- `push.web.vapidPublicKey` 會擷取作用中的 VAPID 公開金鑰。
- `push.web.subscribe` 會註冊 `endpoint` 加上 `keys.p256dh`/`keys.auth`。
- `push.web.unsubscribe` 會移除已註冊的端點。
- `push.web.test` 會將測試通知傳送至呼叫端的訂閱。

<Note>
網頁推播與 iOS APNS 轉送路徑（轉送式推播請參閱[設定](/zh-TW/gateway/configuration)）以及以原生行動裝置配對為目標的 `push.test` 方法彼此獨立。
</Note>

## 託管嵌入內容

助理訊息可以使用 `[embed ...]` 短代碼，在行內呈現託管的網頁內容。iframe 沙箱原則由 `gateway.controlUi.embedSandbox` 控制：

核心 [`show_widget`](/zh-TW/tools/show-widget) 工具會直接從工具呼叫呈現獨立的 SVG 或 HTML。瀏覽器與支援的原生聊天用戶端會宣告 `inline-widgets` 閘道功能，產生的 Canvas 文件會在重新載入聊天記錄後繼續保留。Discord Activities 在 Discord 上提供相同的工具名稱；來自其他頻道的執行則不會收到此工具。

<Tabs>
  <Tab title="嚴格">
    停用託管嵌入內容內的指令碼執行。
  </Tab>
  <Tab title="指令碼（預設）">
    允許互動式嵌入內容，同時維持來源隔離；通常足以用於獨立的瀏覽器遊戲／小工具。
  </Tab>
  <Tab title="受信任">
    在 `allow-scripts` 之上加入 `allow-same-origin`，供有意需要較高權限的同網站文件使用。
  </Tab>
</Tabs>

```json5
{
  gateway: {
    controlUi: {
      embedSandbox: "scripts",
    },
  },
}
```

<Warning>
只有在嵌入文件確實需要同源行為時，才使用 `trusted`。對大多數由代理程式產生的遊戲與互動式畫布而言，`scripts` 是更安全的選擇。
</Warning>

預設仍會封鎖絕對外部 `http(s)` 嵌入網址。若要讓 `[embed url="https://..."]` 載入第三方頁面，請設定 `gateway.controlUi.allowExternalEmbedUrls: true`。

## 聊天訊息寬度

聊天逐字稿使用與撰寫區對齊、置中的易讀框架。在該框架內，助理與工具輸出維持靠左對齊，使用者對話框則維持靠右對齊。寬螢幕部署可設定 `gateway.controlUi.chatMessageMaxWidth`，在不修補隨附 CSS 的情況下覆寫逐字稿寬度：

```json5
{
  gateway: {
    controlUi: {
      chatMessageMaxWidth: "min(1280px, 82%)",
    },
  },
}
```

此值會在傳送至瀏覽器前進行驗證。支援的格式包括 `960px` 或 `82%` 等純長度與百分比，以及受限制的 `min(...)`、`max(...)`、`clamp(...)`、`calc(...)` 和 `fit-content(...)` 寬度運算式。

## Tailnet 存取（建議）

<Tabs>
  <Tab title="整合式 Tailscale Serve（偏好）">
    讓閘道保持在回送介面上，並由 Tailscale Serve 使用 HTTPS 進行代理：

    ```bash
    openclaw gateway --tailscale serve
    ```

    開啟 `https://<magicdns>/`（或你設定的 `gateway.controlUi.basePath`）。

    依預設，Control UI/WebSocket Serve 要求可透過 Tailscale 身分標頭（`tailscale-user-login`）進行驗證，前提是 `gateway.auth.allowTailscale` 為 `true`。OpenClaw 會使用 `tailscale whois` 解析 `x-forwarded-for` 位址，並將結果與標頭比對以驗證身分；只有當要求透過回送介面進入，且帶有 Tailscale 的 `x-forwarded-*` 標頭時，才會接受這些要求。對於具備瀏覽器裝置身分的 Control UI 操作者工作階段，這條已驗證的 Serve 路徑也會略過裝置配對往返流程；沒有裝置身分的瀏覽器與節點角色連線仍會執行一般裝置檢查。若想要求即使是 Serve 流量也必須提供明確的共用密鑰認證資訊，請設定 `gateway.auth.allowTailscale: false`，然後使用 `gateway.auth.mode: "token"` 或 `"password"`。

    <Warning>
    無權杖的 Serve 驗證假設閘道主機值得信任。若該主機上可能執行不受信任的本機程式碼，請要求權杖／密碼驗證。
    </Warning>

## 不安全的 HTTP

若透過純 HTTP（`http://<lan-ip>` 或 `http://<tailscale-ip>`）開啟儀表板，瀏覽器會在**非安全內容**中執行並封鎖 WebCrypto。依預設，OpenClaw 會**封鎖**沒有裝置身分的 Control UI 連線。

文件中列出的例外：

- 僅限 localhost 的不安全 HTTP 相容性，搭配 `gateway.controlUi.allowInsecureAuth=true`
- 透過 `gateway.auth.mode: "trusted-proxy"` 成功完成的操作者 Control UI 驗證
- 緊急備援 `gateway.controlUi.dangerouslyDisableDeviceAuth=true`

**建議的修正方式：**使用 HTTPS（Tailscale Serve），或在本機透過 `https://<magicdns>/`（Serve）或 `http://127.0.0.1:18789/`（位於閘道主機上）開啟 UI。

<AccordionGroup>
  <Accordion title="不安全驗證切換開關的行為">
    ```json5
    {
      gateway: {
        controlUi: { allowInsecureAuth: true },
        bind: "tailnet",
        auth: { mode: "token", token: "replace-me" },
      },
    }
    ```

    `allowInsecureAuth` 僅是本機相容性切換開關：

    - 它允許 localhost Control UI 工作階段在非安全 HTTP 內容中，無須裝置身分即可繼續。
    - 它不會略過配對檢查。
    - 它不會放寬遠端（非 localhost）的裝置身分要求。

  </Accordion>
  <Accordion title="僅限緊急備援">
    ```json5
    {
      gateway: {
        controlUi: { dangerouslyDisableDeviceAuth: true },
        bind: "tailnet",
        auth: { mode: "token", token: "replace-me" },
      },
    }
    ```

    <Warning>
    `dangerouslyDisableDeviceAuth` 會停用 Control UI 裝置身分檢查，造成嚴重的安全性降級。緊急使用後請儘速還原。
    </Warning>

  </Accordion>
  <Accordion title="受信任 Proxy 注意事項">
    - 成功的受信任 Proxy 驗證可以允許沒有裝置身分的**操作者** Control UI 工作階段進入。
    - 這**不會**延伸到節點角色 Control UI 工作階段。
    - 同一主機上的回送反向 Proxy 仍無法滿足受信任 Proxy 驗證；請參閱[受信任 Proxy 驗證](/zh-TW/gateway/trusted-proxy-auth)。

  </Accordion>
</AccordionGroup>

如需 HTTPS 設定指引，請參閱 [Tailscale](/zh-TW/gateway/tailscale)。

## 內容安全政策

Control UI 隨附嚴格的 `img-src` 政策：只允許**同源**資產、`data:` URL，以及本機產生的 `blob:` URL。瀏覽器會拒絕遠端 `http(s)` 與協定相對圖片 URL，且絕不會發出網路擷取要求。

實際行為如下：

- 透過相對路徑提供的頭像與圖片（例如 `/avatars/<id>`）仍可顯示，包括 UI 擷取後轉換成本機 `blob:` URL 的已驗證頭像路由。
- 內嵌 `data:image/...` URL 仍可顯示。
- 由 Control UI 建立的本機 `blob:` URL 仍可顯示。
- GitHub 連結預覽頭像由閘道從 GitHub 的固定頭像主機擷取，並以大小受限的 `data:` URL 傳回；操作者的瀏覽器絕不會連絡遠端頭像主機。
- 頻道中繼資料所產生的遠端頭像 URL 會由 Control UI 的頭像輔助函式移除，並替換為內建標誌／徽章，因此遭入侵或惡意的頻道無法強迫操作者瀏覽器擷取任意遠端圖片。

此功能一律啟用，無法設定。

## 頭像路由驗證

設定閘道驗證後，Control UI 頭像端點需要與 API 其他部分相同的閘道權杖：

- `GET /avatar/<agentId>` 僅向已驗證的呼叫端傳回頭像圖片。`GET /avatar/<agentId>?meta=1` 依相同規則傳回頭像中繼資料。
- 對任一路由提出的未驗證要求都會遭到拒絕（與同層的助理媒體路由一致），因此在其他部分受到保護的主機上，頭像路由無法洩漏代理程式身分。
- Control UI 擷取頭像時，會將閘道權杖當作 bearer 標頭轉送，並使用已驗證的 blob URL，讓圖片仍可顯示在儀表板中。

若停用閘道驗證（不建議在共用主機上這麼做），頭像路由也會與閘道其他部分一致，變成不需要驗證。

## 助理媒體路由驗證

設定閘道驗證後，助理的本機媒體預覽會使用兩步驟路由：

- `GET /__openclaw__/assistant-media?meta=1&source=<path>` 需要一般的 Control UI 操作者驗證；瀏覽器檢查可用性時，會將閘道權杖當作 bearer 標頭傳送。
- 成功的中繼資料回應會包含一個短效的 `mediaTicket`，其範圍限定於該確切來源路徑。
- 由瀏覽器顯示的圖片、音訊、影片和文件 URL 會使用 `mediaTicket=<ticket>`，而不是使用中的閘道權杖或密碼。票證會快速到期，且無法授權不同的來源。

這可在不將可重複使用的閘道認證資訊放入可見媒體 URL 的情況下，讓媒體顯示功能與瀏覽器原生媒體元素相容。

## 核准連結

操作者核准通知可以深層連結至保留的 `${controlUiBasePath}/approve/{approvalId}` 命名空間下所提供的獨立核准文件（例如 `/approve/<approvalId>`，或使用已設定基礎路徑時的 `/openclaw/approve/<approvalId>`）。該 URL 在核准的存續期間保持穩定，且可安全地在你自己的裝置之間轉送：它只識別核准，絕不會授權核准。

- 閘道會在外掛 HTTP 路由之前，為**所有** HTTP 方法保留單一區段的 `/approve/<approvalId>` 命名空間，因此外掛路由絕不可能遮蔽或攔截核准文件。
- 開啟核准文件需要與 Control UI 其他部分相同的閘道驗證（權杖／密碼、Tailscale Serve 身分，或受信任 Proxy 身分）；認證資訊絕不會成為核准 URL 的一部分。
- 停用 Control UI 服務時，對該命名空間的要求會傳回 `404`，而不會繼續傳遞至外掛處理常式。
- 在核准文件上登入只會暫時作用於該頁面：它不會覆寫相同瀏覽器中完整 Control UI 所儲存的閘道選擇或設定。

閘道會從 `dist/control-ui` 提供靜態檔案：

```bash
pnpm ui:build
```

選用的絕對基底（固定資產 URL）：

```bash
OPENCLAW_CONTROL_UI_BASE_PATH=/openclaw/ pnpm ui:build
```

本機開發（獨立開發伺服器）：

```bash
pnpm ui:dev
```

接著將 UI 指向你的閘道 WS URL（例如 `ws://127.0.0.1:18789`）。

## 空白的 Control UI 頁面

若瀏覽器載入空白儀表板，且開發人員工具未顯示有用的錯誤，可能是某個擴充功能或提早執行的內容指令碼阻止了 JavaScript 模組應用程式求值。若啟動後尚未註冊 `<openclaw-app>`，靜態頁面會顯示純 HTML 復原面板。

變更瀏覽器環境後，請使用面板中的 **Try again** 動作；或完成以下檢查後手動重新載入：

- 停用會注入所有頁面的擴充功能，尤其是具有 `<all_urls>` 內容指令碼的擴充功能。
- 嘗試使用私密視窗、全新的瀏覽器設定檔或其他瀏覽器。
- 讓閘道保持執行，並在變更瀏覽器後驗證相同的儀表板 URL。

## 偵錯／測試：開發伺服器 + 遠端閘道

Control UI 是靜態檔案；WebSocket 目標可設定，且可以與 HTTP 來源不同。若想在本機執行 Vite 開發伺服器，但讓閘道在其他位置執行，這會很方便。

<Steps>
  <Step title="啟動 UI 開發伺服器">
    ```bash
    pnpm ui:dev
    ```
  </Step>
  <Step title="使用 gatewayUrl 開啟">
    ```text
    http://localhost:5173/?gatewayUrl=ws%3A%2F%2F<gateway-host>%3A18789
    ```

    選用的一次性驗證（若需要）：

    ```text
    http://localhost:5173/?gatewayUrl=wss%3A%2F%2F<gateway-host>%3A18789#token=<gateway-token>
    ```

  </Step>
</Steps>

<AccordionGroup>
  <Accordion title="注意事項">
    - `gatewayUrl` 會在載入後儲存至 localStorage，並從 URL 中移除。
    - 若透過 `gatewayUrl` 傳入完整的 `ws://` 或 `wss://` 端點，請對該值進行 URL 編碼，讓瀏覽器正確剖析查詢字串。
    - 只要可行，應透過 URL 片段（`#token=...`）傳入 `token`。片段不會傳送至伺服器，可避免要求日誌與 Referer 洩漏。為了相容性，舊版 `?token=` 查詢參數仍會匯入一次，但僅作為備援，並會在啟動程序完成後立即移除。
    - `password` 僅保留在記憶體中。
    - 設定 `gatewayUrl` 後，UI 不會改用設定或環境認證資訊。請明確提供 `token`（或 `password`）；缺少明確認證資訊會被視為錯誤。
    - 當閘道位於 TLS 後方（Tailscale Serve、HTTPS Proxy 等）時，請使用 `wss://`。
    - `gatewayUrl` 只會在頂層視窗中接受（不得嵌入），以防止點擊劫持。
    - 公開且非回送的 Control UI 部署必須明確設定 `gateway.controlUi.allowedOrigins`（完整來源）。從回送、RFC1918／連結本機、`.local`、`.ts.net` 或 Tailscale CGNAT 主機進行的私有同源 LAN／Tailnet 載入，無須啟用 Host 標頭備援即可接受。
    - 閘道啟動時可能會根據有效的執行階段繫結與連接埠，預先填入 `http://localhost:<port>` 和 `http://127.0.0.1:<port>` 等本機來源，但遠端瀏覽器來源仍需要明確的項目。
    - 除非是受到嚴格控制的本機測試，否則不要使用 `gateway.controlUi.allowedOrigins: ["*"]`；其含義是允許任何瀏覽器來源，而不是“符合我目前使用的任何主機”。
    - `gateway.controlUi.dangerouslyAllowHostHeaderOriginFallback=true` 會啟用 Host 標頭來源備援模式，但這是危險的安全模式。

  </Accordion>
</AccordionGroup>

```json5
{
  gateway: {
    controlUi: {
      allowedOrigins: ["http://localhost:5173"],
    },
  },
}
```
