Control UI 是由閘道提供的小型 **Vite + Lit** 單頁應用程式：

- 預設值：`http://<host>:18789/`
- 選用前綴：設定 `gateway.controlUi.basePath`（例如 `/openclaw`）

它會透過相同連接埠**直接連線至閘道 WebSocket**。

當你查看執行中的工作階段時，閘道可使用該代理程式的工具模型產生精簡的狀態摘要。聊天介面會將其顯示為單行狀態標籤，展開後則呈現包含評估、計畫進度、PR 與經過時間的卡片。當執行陷入停滯或需要輸入時，卡片可自動展開一次；`/btw` 側邊聊天的優先順序高於展開的卡片。

展開的卡片也接受關於該次執行的簡短問題。回答只會使用觀察器目前的摘要與經過清理且有大小限制的筆記，僅在該工作階段中保留於瀏覽器內，而且絕不會進入或中斷主要代理程式執行。如果觀察內容沒有包含答案，觀察器會表示無法得知。

第一份摘要送達後，它會取代啟發式即時活動，成為該次執行的側邊欄副標題。最終的完成或失敗摘要會在工作階段仍為未讀時保持可見，之後該列會恢復為正常的工作副標題。

工作階段觀察預設為啟用。在 **Settings > Appearance > Sidebar** 中，你可以在整個閘道停用此功能、檢視解析後的小型模型及其來源，或選擇自動路由、停用工具任務，或明確選取 `agents.defaults.utilityModel`。對應的設定控制項為 `gateway.controlUi.sessionObserver: false` 與 `agents.defaults.utilityModel: ""`。

## 快速開啟（本機）

如果閘道正在同一部電腦上執行，請開啟 [http://127.0.0.1:18789/](http://127.0.0.1:18789/)（或 [http://localhost:18789/](http://localhost:18789/)）。

如果頁面無法載入，請先啟動閘道：`openclaw gateway`。

<Note>
在原生 Windows LAN 繫結中，即使 `127.0.0.1` 可在閘道主機上運作，Windows Firewall 或組織管理的 Group Policy 仍可能封鎖公告的 LAN URL。請在 Windows 主機上執行 `openclaw gateway status --deep`；它會回報可能遭封鎖的連接埠、設定檔不相符情況，以及原則可能忽略的本機防火牆規則。
</Note>

驗證會在 WebSocket 交握期間透過下列方式提供：

- `connect.params.auth.token`
- `connect.params.auth.password`
- 當 `gateway.auth.allowTailscale: true` 時使用 Tailscale Serve 身分標頭
- 當 `gateway.auth.mode: "trusted-proxy"` 時使用受信任 Proxy 身分標頭

儀表板設定面板會為目前的瀏覽器分頁工作階段與所選的閘道 URL 保留權杖，但不會保存密碼。初始設定通常會在第一次連線時產生閘道權杖，以供共用密鑰驗證使用；但當 `gateway.auth.mode` 為 `"password"` 時，也可使用密碼驗證。

## 裝置配對（首次連線）

從新的瀏覽器或裝置連線時，通常需要**一次性配對核准**，顯示為 `disconnected (1008): pairing required`。

<Steps>
  <Step title="列出待處理的要求">
    ```bash
    openclaw devices list
    ```
  </Step>
  <Step title="依要求 ID 核准">
    ```bash
    openclaw devices approve <requestId>
    ```
  </Step>
</Steps>

如果瀏覽器使用變更後的驗證詳細資料（角色／範圍／公開金鑰）重試配對，先前的待處理要求會被取代，並建立新的 `requestId`；請在核准前重新執行 `openclaw devices list`。

將已配對瀏覽器從讀取存取權切換為寫入／管理員存取權，會視為核准升級，而非無提示重新連線：OpenClaw 會保持舊核准有效、封鎖權限更廣的重新連線，並要求你明確核准新的範圍集合。

核准後，系統會記住該裝置，除非你使用 `openclaw devices revoke --device <id> --role <role>` 撤銷，否則不需要再次核准。如需瞭解權杖輪替、撤銷，以及 Paperclip／`openclaw_gateway` 首次執行核准流程，請參閱[裝置命令列介面](/zh-TW/cli/devices)。

<Note>
- 直接從本機回送位址進行的瀏覽器連線（`127.0.0.1`／`localhost`）會自動核准。
- 當 `gateway.auth.allowTailscale: true`、Tailscale 身分驗證通過，且瀏覽器提供其裝置身分時，Tailscale Serve 可讓 Control UI 操作員工作階段略過配對往返。沒有裝置身分的瀏覽器與節點角色連線仍會遵循一般裝置檢查。
- 直接 Tailnet 繫結、LAN 瀏覽器連線，以及沒有裝置身分的瀏覽器設定檔仍需明確核准。
- 每個瀏覽器設定檔都會產生唯一的裝置 ID，因此切換瀏覽器或清除瀏覽器資料後需要重新配對。

</Note>

## 配對行動裝置

已配對的管理員無須開啟終端機，即可建立 iOS／Android 連線 QR Code：

<Steps>
  <Step title="開啟行動裝置配對">
    選取 **Devices**，然後按一下 **Devices** 卡片中的 **Pair mobile device**。
  </Step>
  <Step title="連線手機">
    在 OpenClaw 行動應用程式中，開啟 **Settings** → **Gateway** 並掃描 QR Code。也可以改為複製並貼上設定碼。
  </Step>
  <Step title="確認連線">
    官方 iOS／Android 應用程式會自動連線。如果 **Pending approval** 顯示要求，請先檢查其角色與範圍再予以核准。
  </Step>
</Steps>

建立設定碼需要 `operator.admin`；不具備此權限的工作階段會停用該按鈕。設定碼包含短效的啟動認證資訊，因此在 QR Code 與複製的程式碼有效期間，請將它們視同密碼保管。若要遠端配對，閘道必須解析為 `wss://`（例如透過 Tailscale Serve/Funnel）；純 `ws://` 僅限於回送位址與私人 LAN 位址。如需完整的安全性與後援詳細資料，請參閱[配對](/zh-TW/channels/pairing#pair-from-the-control-ui-recommended)。

## 個人身分（瀏覽器本機）

Control UI 支援為每個瀏覽器設定個人身分（顯示名稱與虛擬人像），並將其附加至外送訊息，以便在共用工作階段中標示來源。此身分儲存在瀏覽器儲存空間中，範圍僅限目前的瀏覽器設定檔，不會同步到其他裝置；除了你傳送之訊息所含的一般對話記錄作者中繼資料外，也不會持續儲存於伺服器端。清除網站資料或切換瀏覽器會將其重設為空白。

助理虛擬人像覆寫採用相同的瀏覽器本機模式：上傳的覆寫會在本機疊加於閘道解析的身分上，絕不會透過 `config.patch` 往返傳輸。共用的 `ui.assistant.avatar` 設定欄位仍可供直接寫入該欄位的非 UI 用戶端使用。

## 執行階段設定端點

Control UI 會從 `/control-ui-config.json` 擷取其執行階段設定，並以閘道的 Control UI 基底路徑為基準進行解析（例如基底路徑 `/__openclaw__/` 下的 `/__openclaw__/control-ui-config.json`）。該端點受與其餘 HTTP 介面相同的閘道驗證保護：未驗證的瀏覽器無法擷取；若要成功擷取，則需要有效的閘道權杖／密碼、Tailscale Serve 身分，或受信任 Proxy 身分。

## 閘道主機狀態

開啟 **Settings → General**，即可查看 **Gateway Host** 卡片，其中包含閘道機器、LAN 位址、作業系統、執行階段、運作時間、CPU 負載、記憶體，以及狀態磁碟區的磁碟空間。卡片在顯示時會每 10 秒透過 `system.info` 閘道 RPC 重新整理，且需要 `operator.read` 範圍。舊版閘道與沒有該範圍的連線不會顯示此卡片。

## 語言支援

Control UI 首次載入時會根據你的瀏覽器地區設定進行本地化。若要稍後覆寫，請開啟 **Settings -> General -> Language**（選擇器位於 General 頁面，而非 Appearance 下）。

- 支援的地區設定：`en`、`ar`、`de`、`es`、`fa`、`fr`、`hi`、`id`、`it`、`ja-JP`、`ko`、`nl`、`pl`、`pt-BR`、`ru`、`th`、`tr`、`uk`、`vi`、`zh-CN`、`zh-TW`
- 非英文翻譯會在瀏覽器中延遲載入。
- 所選的地區設定會儲存於瀏覽器儲存空間中，並於日後造訪時重複使用。
- 缺少的翻譯鍵會退回使用英文。

文件翻譯會針對相同的非英文地區設定集合產生，但文件網站內建的 Mintlify 語言選擇器只會列出 Mintlify 接受的地區設定代碼。泰文（`th`）與波斯文（`fa`）文件仍會在發布儲存庫中產生；在 Mintlify 支援這些代碼之前，它們可能不會出現在該選擇器中。

## 外觀主題

Appearance 面板提供內建的 Claw、Knot 與 Dash 主題（預設為 Claw），另有一個瀏覽器本機的 tweakcn 匯入欄位。若要匯入主題，請開啟 [tweakcn 編輯器](https://tweakcn.com/editor/theme)，選擇或建立主題，按一下 **Share**，然後將複製的連結貼到 Appearance 中。匯入工具也接受 `https://tweakcn.com/r/themes/<id>` 登錄 URL、如 `https://tweakcn.com/editor/theme?theme=amethyst-haze` 的編輯器 URL、相對 `/themes/<id>` 路徑、原始主題 ID，以及如 `amethyst-haze` 的預設主題名稱。

匯入的主題只會儲存在目前的瀏覽器設定檔中；不會寫入閘道設定，也不會跨裝置同步。取代匯入的主題會更新該單一本機欄位；若清除時匯入的主題正處於啟用狀態，則會切換回 Claw。

Appearance 也提供 Text size 設定。此設定會套用至聊天文字、撰寫器文字、工具卡片與聊天側邊欄，並確保文字輸入欄位至少為 16px，以防止行動版 Safari 在聚焦時自動縮放。

主題、主題模式、文字大小、語言與聊天顯示偏好設定會透過閘道設定（`ui.prefs`）同步，因此可在不同裝置間跟隨你，代理程式也能透過核准閘門變更這些設定——已連線的用戶端會透過閘道的 `config.changed` 通知即時套用變更。每個瀏覽器都會保留本機鏡像，以便立即啟動；無法寫入設定的用戶端（檢視者範圍、離線）則會將變更保留於裝置本機。請參閱[設定參考資料](/zh-TW/gateway/configuration-reference#ui)。

## OpenClaw 系統維護

開啟 **Settings → Ask OpenClaw**，即可與系統設定及修復代理程式交談。在初始設定之外，此頁面每次造訪最多可顯示一個可關閉的事件標籤。它不會對例行閘道流量做出反應，只有在健康狀態快照回報設定重新載入器已停用、已設定的頻道中斷連線／降級、頻道探測失敗，或頻道認證資訊無法使用時才會回應。只有在嚴重程度更高時，較新的事件才會取代待處理的標籤；關閉或使用標籤後，該次造訪期間將不再顯示事件提示。按一下標籤會將其診斷問題作為真正的 `openclaw.chat` 訊息傳送，因此對話記錄會記下該要求，且 OpenClaw 會執行診斷。初始設定期間絕不會顯示這些事件標籤。

## 管理外掛

在側邊欄中開啟**外掛**，或使用相對於已設定 Control UI 基底路徑的
`/settings/plugins`，即可在不離開 Control UI 的情況下瀏覽及管理外掛。
例如，基底路徑 `/openclaw` 會使用 `/openclaw/settings/plugins`。
即使所有選用外掛都已停用，此頁面仍一律可用。

外掛是具有四個分頁的中樞：**已安裝**與**探索**會在
`/settings/plugins` 管理外掛程式碼；**Skills** 會在
`/skills` 提供每個代理程式的 Skill 管理工具；**工作坊**則會在
`/skills/workshop` 提供 Skill Workshop 提案審查。每個分頁都保有自己的 URL，
而側邊欄會顯示單一的「外掛」項目來涵蓋所有分頁。

**已安裝**分頁會顯示依類別分組的完整本機清單，並附上概覽計數。每一列都可開啟詳細資料檢視；其更多選項（`…`）選單可啟用或停用外掛，並為外部安裝的外掛提供**移除**選項。此分頁也會列出已設定的 [MCP 伺服器](/zh-TW/cli/mcp)，並支援直接新增、停用及移除。相同的伺服器控制項也位於**設定 → MCP**。**探索**分頁是外掛商店：其中包含 OpenClaw 隨附的精選外掛、官方外部外掛，以及適用於熱門服務的一鍵式 MCP 連接器。在搜尋框中輸入內容會直接查詢 [ClawHub](https://clawhub.ai/plugins)，並附加一個**來自 ClawHub**區段，其中包含下載次數與來源驗證徽章。深層連結可透過 `/settings/plugins?tab=discover` 直接指向商店。

**Skills** 分頁保留技能狀態報告、啟用／停用切換開關、API 金鑰輸入，以及直接在 ClawHub 搜尋技能的功能，範圍限定於所選代理程式。**工作坊**分頁保留 Skill Workshop 看板，以及用於[技能提案](/zh-TW/tools/skill-workshop)的今日審查流程。**尋找技能構想**會由新至舊審查數量受限的實質工作階段，並將所有結果保留為待處理提案。面板會顯示累計涵蓋範圍；**掃描更早的工作**會從已保存的游標繼續，較舊的歷史記錄全部掃描完畢後，則會變成**掃描新工作**。停用自主自我學習時，仍可手動審查歷史記錄，並會使用所選代理程式已設定的模型。

隨附的外掛已存在於閘道上，因此會顯示**啟用**或**停用**，而非**安裝**。例如，Workboard 隨 OpenClaw 提供，但預設為停用，因此其動作為**啟用**。內建外掛無法移除，只能停用。

讀取目錄及搜尋 ClawHub 需要 `operator.read`。安裝、啟用、停用或移除外掛，以及變更 MCP 伺服器，需要 `operator.admin`；唯讀操作員無法使用這些動作。

ClawHub 安裝會透過閘道執行，並沿用其他由閘道媒介之安裝所採用的信任、完整性與外掛安裝原則檢查。安裝或移除外掛程式碼需要重新啟動閘道。若外掛與目前的閘道執行階段支援，啟用或停用已安裝的外掛可在不重新啟動的情況下套用；否則，UI 會回報需要重新啟動。新增採用 OAuth 的 MCP 連接器後，需要從命令列介面執行一次 `openclaw mcp login <name>`。

此頁面刻意著重於清單、探索、安裝、啟用與移除。若要使用任意 npm、git 或本機路徑來源、更新及進階外掛設定，請使用 [`openclaw plugins`](/zh-TW/cli/plugins)。

## 應用程式與擴充功能

## 側邊欄導覽

側邊欄以代理程式為中心組織所有內容。頂端的身分列代表目前使用中的代理程式；其下方的**頁面**區段以**首頁**開頭——這是代理程式持續進行的主要工作階段，會以徽章顯示未讀或執行中狀態——之後則是釘選的目的地（預設為**用量**、**自動化**和**外掛**）。頁面標題上的自訂控制項會開啟包含所有其他目的地的選單，包括外掛提供的分頁，以及**編輯釘選項目**；在導覽區域按一下滑鼠右鍵可直接開啟釘選編輯器。下方的工作階段清單分為數個區域：**討論串**用於代理程式的聊天工作階段（主要工作階段仍位於首頁之後；由其產生的工作階段會以頂層討論串顯示於此，而具名討論串不會顯示類型前綴）、**群組**用於群組與聊天室對話，以及**程式設計**用於綁定至受管理工作樹或執行節點的工作階段（每一列會顯示一行 `repo ⎇ branch` 加上節點主機）、採用 ACP 的操作框架工作階段，以及 Codex／Claude 命令列介面目錄。程式設計區域在首次執行時預設收合，並會記住你的選擇；其收合標題會保留實際數量，且其中的工作階段運作時會顯示執行中指示器。自訂群組（工作階段 `category`）和**已釘選**列位於討論串上方，且將工作階段指派至自訂群組永遠優先於自動區域分類。討論串標題包含排序控制項（建立時間或最後更新時間、分組依據，以及針對使用中、已封存或全部的持久化**狀態**篩選器），以及用於開啟新工作階段頁面的 **+**。已封存列仍會以暗淡樣式內嵌顯示，並附有封存字符；它們不會計入未讀或需注意狀態，也不會參與譜系提升。開啟工作階段只會移動選取醒目提示，不會重新排列各列。近期有子執行的父工作階段會顯示展開控制項和子項目數量；展開後即可在不離開側邊欄的情況下檢視巢狀子工作階段、即時或終止狀態，以及執行階段。選取子工作階段會開啟其聊天，並自動顯示其祖先路徑。子列不參與根層級分組、釘選、拖曳、多選和分頁；收合的區域不會占用可見頁面額度。自上次讀取後有新活動的工作階段會顯示未讀圓點，開啟後即標示為已讀。代理程式也可發布一行短暫且會到期的狀態，並可選擇以特選的琥珀色圖示要求注意；當你開啟工作階段、傳送下一則訊息、明確清除該宣告，或其 TTL 到期時，此宣告便會清除。雲端工作程式生命週期狀態使用地球徽章；本機及已回收的工作階段不顯示位置徽章，因為本機執行是預設行為。每個根工作階段列都有內容選單（烤肉串按鈕或按一下滑鼠右鍵），包含釘選／取消釘選、標示為未讀／已讀、重新命名、分叉、移至群組（包括新群組及從群組移除）、封存或取消封存，以及刪除；觸控版面會讓直接釘選和選單控制項保持可見。按住 Cmd／Ctrl 並按一下，可切換根列的多選狀態；按住 Shift 並按一下，則可依可見順序延伸選取範圍；接著在所選列開啟選單時，會提供批次動作（將 N 個項目標示為未讀／已讀、將 N 個項目移至群組、封存 N 個項目、刪除 N 個項目），並套用至每個所選工作階段；批次刪除只需確認一次。將根工作階段拖曳至**已釘選**即可釘選，或拖曳至自訂群組即可移動。自訂群組標題可收合、展開或拖曳以重新排序；群組名稱及順序儲存於閘道（`sessions.groups.*`）中，因此會跨瀏覽器跟隨你，而收合狀態則保留在瀏覽器設定檔中。群組標題也有選單（烤肉串按鈕或按一下滑鼠右鍵），包含重新命名群組、新群組和刪除群組；重新命名或刪除群組時，伺服器端的每個成員工作階段都會更新，包括已封存的工作階段；刪除群組會保留其中的工作階段，並將其移回討論串。

## 新工作階段頁面

側邊欄工作階段清單標題中的 **+** 會在 `/new` 開啟全頁草稿：在你傳送第一則訊息前，不會建立任何內容。統一的**位置**選擇器可選擇工作資料夾，管理員操作員還可選擇執行目的地：**閘道 · 本機**、公開 `system.run` 的已配對節點，或可用的雲端設定檔。資料夾預設為代理程式工作區；其他閘道絕對路徑需要 `operator.admin`，但即使不是 Git 簽出也能直接執行。當所選的閘道資料夾是 Git 簽出時，相同的選擇器會提供選用的**工作樹**隔離功能，包含由 `worktrees.branches` 支援的基礎分支選擇器（不擷取），以及選用的工作樹名稱（分支會變成 `openclaw/<name>`）。雲端工作程式需要該受管理工作樹路徑；已配對節點絕不會公開此選項。撰寫器頁尾可選擇新工作階段的模型和推理層級；雲端啟動會先保存這兩項選擇，再將工作階段分派至其工作程式。

**瀏覽資料夾**會開啟位置選擇器的內嵌目錄瀏覽器；此瀏覽器由僅限管理員使用的 `fs.listDir` 方法支援，範圍限定於所選的閘道或節點。閘道及支援瀏覽的節點會列出其檔案系統；不具 `fs.listDir` 的可執行節點仍接受手動輸入的絕對路徑。最近使用的位置可一併還原資料夾及其所屬節點，而不會跨主機沿用路徑。提交時會以第一則訊息呼叫 `sessions.create`，因此執行會在同一次往返中開始，且 UI 會跳至新工作階段的聊天。如果閘道建立了工作階段，但拒絕首次傳送，聊天會在重新載入後保留提示與錯誤；**重試**會透過已建立的工作階段傳送，而不會再建立一個工作階段。

在**設定**中，專用側邊欄包含**詢問 OpenClaw**，並以**搜尋設定**欄位開頭，可快速尋找設定區段。

側邊欄頂端的**搜尋**欄位會開啟命令選擇區（⌘K）。按一下側邊欄頂端的代理程式身分列會開啟代理程式選單；**首頁**會開啟主要工作階段。當有項目需要處理時——例如失敗或逾期的排程工作、即將到期或已到期的模型驗證——側邊欄頁尾上方會出現精簡的注意提示方塊，按一下即可前往所屬頁面。身分列會顯示代理程式的虛擬人偶（身分圖片或表情符號）、名稱、連線圓點，以及即時副標題。按一下後會開啟代理程式選單：代理程式切換器（多代理程式設定）、“此代理程式能做什麼？”、**代理程式設定**、**設定**、行動裝置配對、**文件**、組建資訊方塊，以及色彩模式切換開關。代理程式數量超過十個時，名冊會提供篩選欄位，並優先列出已釘選的代理程式；你可從代理程式設定頁面釘選或取消釘選代理程式，且釘選集合會儲存在瀏覽器設定檔中。選擇代理程式後，聊天、用量、自動化、任務、Workboard 和工作階段的範圍都會限定於該代理程式。每個限定範圍的頁面都會提供**代理程式**控制項，並以**所有代理程式**作為退出限定範圍的選項；這會擴大共用頁面範圍，而不會變更具體的聊天代理程式，且直接工作階段連結仍會開啟其目標。代理程式設定頁面會保留自己的 `?agent=` 選取項目，不會跟隨共用頁面範圍。頁尾列包含產品標誌、組建資訊方塊、閘道連線圓點，以及設定捷徑。當閘道從來源簽出執行，且所在分支不是 `main` 時，頁尾也會以紅色顯示該分支名稱，讓非發行版閘道一目瞭然（發行版安裝絕不會顯示）。按下 Shift-Command-Comma 可開啟**設定**，而不會覆寫瀏覽器的 Command-Comma 捷徑。側邊欄標題也包含收合切換開關（⌘B）；收合會完全隱藏側邊欄，以提供全寬工作區，浮動展開控制項（或 ⌘B）則可將其帶回；macOS 應用程式改為在標題列中以原生方式提供該切換開關。在桌面版中，側邊欄是唯一的導覽框架，沒有頂端列。狹窄的檢視區會以滑出式抽屜取代側邊欄，並在精簡標題列後方提供抽屜切換開關、品牌標誌和命令選擇區搜尋；在手機上，聊天會將該導覽列整合至其標題列，選單和搜尋控制項位於工作階段標題旁。在 macOS 應用程式中，獨立的標題列會將標題列間距整合為視窗控制項旁的單一精簡橫條。導覽會使用一般瀏覽器歷史記錄，因此瀏覽器的上一頁／下一頁按鈕可逐一瀏覽；macOS 應用程式會在視窗控制項旁新增原生側邊欄切換開關和觸控式軌跡板滑動手勢，側邊欄展開時，其右邊緣會有上一頁／下一頁按鈕；側邊欄收合時，則會有原生搜尋（命令選擇區）和新工作階段按鈕。

待核准項目也會在側邊欄頁尾上方顯示提醒標記；
選取它即可開啟所屬的核准頁面。

## 匯入助理記憶

開啟**設定** → **匯入記憶**，將本機 Codex 或 Claude Code 記憶
匯入 OpenClaw 代理程式。閘道會自行探索其所在主機上支援的本機記憶，
因此遠端控制介面會從閘道電腦匯入，而不是從瀏覽器電腦匯入。

1. 選擇目的地代理程式。
2. 檢視偵測到的來源集合和 Markdown 檔名。檔案內容
   不會在規劃回應中傳送，也不會顯示於頁面中。
3. 選取要匯入的集合並確認。「套用」會在寫入前重建規劃，
   使過時的選取項目能安全失敗。
4. 如果檔案已存在，請啟用**取代現有匯入項目**、重新整理
   預覽，然後確認取代。

Codex 僅匯入其彙整後的 `MEMORY.md` 和 `memory_summary.md`。Claude
Code 會從專案自動記憶目錄和已設定的
`autoMemoryDirectory` 匯入 Markdown；此頁面不會匯入工作階段、設定、指示或
認證資訊。檔案會複製到所選工作區的 `memory/imports/` 下，
作用中的記憶外掛可在該處建立索引。來源永遠不會變更。

規劃與套用需要 `operator.admin`。每次套用都會在狀態存在時建立經驗證的
OpenClaw 備份、寫入遮蔽後的遷移報告，並在取代現有目的地檔案前
保留項目層級的備份。路徑和回想行為請參閱
[記憶概覽](/zh-TW/concepts/memory#import-from-coding-assistants)。

## MCP 頁面

專屬 MCP 頁面是針對 `mcp.servers` 下由 OpenClaw 管理的 MCP 伺服器所提供的操作員檢視。它本身不會啟動 MCP 傳輸；請使用此頁面檢查和編輯已儲存設定，並在需要即時伺服器驗證時使用 `openclaw mcp doctor --probe`。

一般工作流程：

1. 從側邊欄開啟 **MCP**。
2. 檢查摘要卡片中的伺服器總數、已啟用、OAuth 和已篩選伺服器數量。
3. 檢視每個伺服器列的傳輸方式、啟用狀態、驗證、篩選器、逾時和命令提示。
4. 直接在 MCP 頁面新增、啟用、停用或移除伺服器。請明確選擇「可串流 HTTP」、SSE 或 stdio；stdio 命令列可接受加上引號的引數，例如含有空格的路徑。單鍵連接器與探索功能請使用**外掛**頁面。
5. 編輯限定範圍的 `mcp` 設定區段，以設定環境變數、工作目錄、標頭、TLS/mTLS 路徑、OAuth 中繼資料、工具篩選器和 Codex 投射中繼資料等進階伺服器欄位。
6. 使用**儲存**寫入設定，或在需要讓執行中的閘道套用已變更設定時使用**儲存並發布**。
7. 從終端機執行 `openclaw mcp status --verbose`、`openclaw mcp doctor --probe` 或 `openclaw mcp reload`，以進行靜態診斷、即時驗證或清除快取的執行階段。

此頁面會在呈現前遮蔽帶有認證資訊的類 URL 值，並在命令片段中為伺服器名稱加上引號，讓複製的命令在名稱含有空格或 Shell 中繼字元時仍可正常運作。完整的命令列介面與設定參考：[MCP](/zh-TW/cli/mcp)。

## 活動分頁

「活動」分頁位於**設定 › 系統**中，緊鄰「日誌」和「偵錯」。它是暫存於瀏覽器本機的即時工具活動觀察器，衍生自為聊天工具卡片提供資料的同一個閘道 `session.tool`／工具事件串流。它不會新增其他閘道事件系列、端點、永久活動儲存區、指標資料流或外部觀察器串流。

活動項目只保留經過清理的摘要和已遮蔽、截斷的輸出預覽。工具引數值不會儲存在活動狀態中；UI 會顯示引數已隱藏，且只記錄引數欄位數量。記憶體內清單會跟隨目前的瀏覽器分頁、在控制介面內導覽時保留，並於頁面重新載入、切換工作階段或按下**清除**時重設。

## 操作員終端機

可停駐的操作員終端機預設為停用。若要啟用，請設定 `gateway.terminal.enabled: true` 並重新啟動閘道。終端機需要 `operator.admin` 連線，並會在作用中代理程式工作區內開啟主機 PTY。新分頁會跟隨目前選取的聊天代理程式。

<Warning>
終端機是不受限制的主機 Shell，並會繼承閘道處理程序的環境。請只在受信任的操作員部署中啟用。OpenClaw 會拒絕為具有 `sandbox.mode: "all"` 的代理程式建立終端機工作階段；將作用中的代理程式變更為該模式，會關閉其現有及進行中的終端機工作階段。
</Warning>

使用 **Ctrl + 反引號** 切換停駐面板。版面配置支援停駐於底部或右側、隨瀏覽器檢視區調整大小，並可保留多個 Shell 分頁。請參閱[閘道設定](/zh-TW/gateway/configuration-reference#gateway)，以瞭解 `gateway.terminal.enabled` 及選用的 `gateway.terminal.shell` 覆寫設定。

經擁有者授權且未受沙箱限制的代理程式，可以使用 `terminal` 工具執行操作員應監看的長時間或互動式工作。每次工具呼叫都能開啟、讀取、寫入、調整大小、關閉或列出代理程式自己的閘道 PTY。新工作階段預設會開啟共同附加的控制介面分頁，讓代理程式與操作員共享輸出，且雙方都能輸入或調整大小。代理程式的存取權限精確限定於工作階段：代理程式無法讀取或控制操作員建立的終端機，或由其他代理程式工作階段開啟的終端機。

將一個或多個檔案拖曳到作用中的終端機，或使用迴紋針按鈕選擇檔案。OpenClaw 會將每個檔案暫存到 PTY 所在的機器，並在游標處貼上經 Shell 引號處理的絕對路徑；它絕不會按下 Enter 或執行輸入內容。精簡的批次指示器會顯示目前檔案及已完成數量。取消會停止批次中的剩餘項目而不貼上路徑；傳輸失敗的項目會保持可見，讓你能從該檔案重試，而不必重新上傳已完成的檔案。每個影像、PDF、封存檔及其他檔案類型的大小上限為 16 MiB。暫存檔案在 POSIX 主機上會使用私有的系統暫存目錄（目錄模式 `0700`、檔案模式 `0600`），在 Windows 上則使用位於使用者設定檔 ACL 邊界內的目錄，並設有 24 小時清理計時器，因此請移動或複製任何需要保留的內容。

路徑插入支援 PowerShell、`cmd.exe` 及可辨識的 POSIX Shell（`sh`、Bash、Dash、Ash、Ksh、Zsh 和 Fish），包括 Windows 上的 Git Bash。系統會拒絕其他 Shell 覆寫設定，因為無法安全推斷其引號規則；若要使用原生 WSL 終端機及 Linux 上傳路徑，請在 WSL 內執行閘道。系統也會拒絕包含 `%` 或 `!` 的 `cmd.exe` 路徑，因為該 Shell 即使在雙引號內仍會展開這些字元。

在工作階段側邊欄中找到的 Codex 與 Claude Code 工作階段，可在同一個終端機面板內以其原生命令列介面開啟。在 **Settings › Chat** 中，將 **Open Codex/Claude threads in** 設為 **Terminal**，即可讓一般的資料列點擊動作開啟 `codex resume` 或 `claude --resume`；預設仍會開啟唯讀的 OpenClaw 檢視器。資料列的右鍵或烤肉串選單一律會提供這兩種選項；工作階段符合條件時，檢視器標頭也會包含 **Open in terminal**。

是否符合條件會依個別工作階段及主機而定。閘道本機工作階段會在閘道主機上啟動供應商所擁有的繼續執行命令。配對節點工作階段會在其所屬節點上啟動允許清單中的供應商命令，並只轉送該 PTY 的輸出、輸入及大小調整事件；這不會公開一般節點 Shell，也不會接受瀏覽器提供的命令。檔案上傳使用獨立且有大小限制的 `terminal.upload` 節點命令，並持續綁定至已開啟的終端機工作階段。該命令首次出現時，請核准節點配對升級。未宣告相符終端機繼續執行命令的節點（包括不具雙向串流功能的內嵌工作節點橋接器）仍可使用檢視器，但會將開啟終端機顯示為不可用；較舊的節點仍可執行終端機，但無法接收拖曳的檔案。

由連線擁有的工作階段可在中斷連線後繼續存續：頁面重新載入、筆電休眠或網路短暫中斷時，工作階段只會在閘道上分離，而不會遭到終止；重新連線後，同一個瀏覽器分頁會重新附加，並重播近期輸出。已分離且由連線擁有的工作階段會在 `gateway.terminal.detachedSessionTimeoutSeconds` 後終止（預設為 300 秒；`0` 會恢復中斷連線時立即終止的行為）。附加到其中一個工作階段時，仍採用 tmux 式接管。

代理程式擁有的工作階段不會綁定至瀏覽器連線。`terminal.attach` 會將每個瀏覽器加入為檢視者，而不取得擁有權；關閉檢視器分頁只會分離該瀏覽器。PTY 會持續存在，直到擁有它的代理程式將其關閉、其處理程序結束、政策將其停用，或閘道關閉為止。`terminal.list` 會將每個項目標示為由連線或代理程式擁有，而 `terminal.text` 可讓管理員連線在不附加的情況下讀取近期的純文字輸出。

終端機也可在 `/?view=terminal` 以全螢幕、僅終端機的文件形式使用。iOS 與 Android 應用程式會將此頁面嵌入其 Terminal 畫面，並重複使用已儲存的閘道認證資訊；可用性遵循相同的 `gateway.terminal.enabled` 與 `operator.admin` 閘門，且當連線的閘道未提供終端機時，頁面會顯示通知。

## 瀏覽器面板

控制介面隨附可停駐的瀏覽器面板，可在任何一般網頁瀏覽器中呈現由閘道控制的瀏覽器（也就是代理程式透過[瀏覽器工具](/zh-TW/tools/browser-control)操作的同一個瀏覽器），不需要原生 WebView。當連線的閘道向 `operator.admin` 連線宣告 `browser.request` 時，此面板便會出現；討論串工作區欄中的地球按鈕可切換面板。面板會顯示即時頁面快照，並提供分頁、可編輯的網址列、上一頁／下一頁／重新載入，以及在你的瀏覽器中開啟等功能；面板可停駐於右側或底部，並會將點擊、滾輪捲動及基本輸入轉送至遠端頁面。

有兩種擷取模式可為代理程式封裝頁面情境：

- **Annotate (pencil)**：在頁面上手繪標記。**Send to chat** 會將筆畫合成到螢幕擷取畫面中、把影像附加至作用中的聊天撰寫區，並預先填入描述頁面網址、標題及每個標記區域的提示，讓代理程式確切知道你圈選了什麼。
- **Inspect (pointer)**：將游標停留在元素上，即可查看游標下方的元素（選取器、無障礙名稱、角色、大小）；按一下即可透過相同的撰寫區流程傳送該元素的詳細資料及醒目標示的螢幕擷取畫面。Inspect、滾輪捲動及上一頁／下一頁需要 `browser.evaluateEnabled`（預設開啟）。

macOS 應用程式會保留其原生連結瀏覽器側邊欄，用於顯示在儀表板中點擊的連結；瀏覽器面板也可在其中運作，且是在其他所有平台上為頁面加註的方式。

## 聊天行為

<AccordionGroup>
  <Accordion title="Send and history semantics">
    - `chat.send` 為**非阻塞**：它會立即以 `{ runId, status: "started" }` 確認，並透過 `chat` 事件串流回應。受信任的 Control UI 用戶端也可能收到選用的 ACK 計時中繼資料，以供本機診斷。
    - 聊天上傳支援圖片及非影片檔案。圖片保留原生圖片路徑；其他檔案則儲存為受管理媒體，並在歷史記錄中顯示為附件連結。
    - 使用相同的 `idempotencyKey` 重新傳送時，執行期間會傳回 `{ status: "in_flight" }`，完成後則傳回 `{ status: "ok" }`。
    - `chat.history` 回應會限制大小，以確保 UI 安全。當逐字稿項目過大時，閘道可能截斷較長的文字欄位、省略龐大的中繼資料區塊，並以預留位置（`[chat.history omitted: message too large]`）取代超出大小限制的訊息。
    - 當可見的助理訊息在 `chat.history` 中遭到截斷時，側邊閱讀器可依需要透過 `chat.message.get`，使用 `sessionKey`、必要時的作用中 `agentId`，以及逐字稿 `messageId`，擷取完整且經顯示正規化的逐字稿項目。如果閘道仍無法傳回更多內容，閱讀器會顯示明確的無法使用狀態，而不會無提示地重複截斷的預覽。
    - 助理／產生的圖片會以受管理媒體參照持久保存，並透過經驗證的閘道媒體 URL 提供，因此重新載入時不必依賴原始 base64 圖片承載資料持續保留在聊天歷史記錄回應中。
    - 轉譯 `chat.history` 時，Control UI 會從可見的助理文字中移除僅供顯示的行內指示詞標籤（例如 `[[reply_to_*]]` 和 `[[audio_as_voice]]`）、純文字工具呼叫 XML 承載資料（包括 `<tool_call>...</tool_call>`、`<function_call>...</function_call>`、`<tool_calls>...</tool_calls>`、`<function_calls>...</function_calls>` 及遭截斷的工具呼叫區塊），以及洩漏的 ASCII／全形模型控制權杖。若助理項目的全部可見文字僅為完全相符的靜默權杖 `NO_REPLY`／`no_reply`，或心跳偵測確認權杖 `HEARTBEAT_OK`，則會省略該項目。
    - 在傳送進行期間及最後重新整理歷史記錄時，如果 `chat.history` 短暫傳回較舊的快照，聊天檢視仍會顯示本機樂觀加入的使用者／助理訊息；閘道歷史記錄追上後，標準逐字稿便會取代這些本機訊息。
    - 即時 `chat` 事件代表傳遞狀態，而 `chat.history` 則從持久的工作階段逐字稿重建。工具最終事件發生後，Control UI 會重新載入歷史記錄，且只合併一小段樂觀加入的尾端內容；逐字稿邊界記錄於 [WebChat](/zh-TW/web/webchat)。
    - `chat.inject` 會將助理備註附加至工作階段逐字稿，並廣播 `chat` 事件，以進行僅限 UI 的更新（不執行代理程式、不傳遞至頻道）。
    - 側邊欄會依代理程式區段，以及已釘選／頻道／工作／自訂／聊天分類，列出每個已載入的作用中工作階段，並提供單一的「新增工作階段」動作以開啟草稿對話框。開啟可見的資料列只會移動醒目標示。工作階段可拖放至「已釘選」以釘選，也可拖放至自訂群組或「聊天」以移動；自訂群組可收合並能以拖曳重新排序，群組名稱與順序會透過閘道同步，而收合狀態則保留在瀏覽器中。新的儀表板工作階段會非同步地根據第一則非命令訊息產生簡潔標題；明確名稱與經驗證的傳送者身分會保持分離，因此帳戶名稱絕不會用作產生的標題。設定 `agents.defaults.utilityModel`（或 `agents.entries.*.utilityModel`），可將這個獨立的模型呼叫路由至成本較低的模型；如果該獨立模型失敗，標題產生功能會改用主要模型重試一次。展開另一個代理程式區段時，可瀏覽該代理程式的工作階段，而不會離開目前開啟的聊天。
    - 對話串搜尋位於命令選擇區（⌘K，或側邊欄頂端的「搜尋」欄位）：輸入查詢後，會跨代理程式搜尋數量受限的相符頁面、篩除內部子項目／排程資料列，並在導覽命令旁列出可見的相符項目。「對話串」頁面則保留可使用篩選條件的完整搜尋清單。
    - 每個側邊欄資料列都保留直接釘選功能，以及涵蓋未讀狀態、重新命名、分支、分組、封存和刪除的完整內容選單。多選的資料列（Cmd/Ctrl + 按一下，Shift + 按一下可選取範圍）會顯示批次選單，涵蓋未讀狀態、分組、封存和刪除；除非每個選取的工作階段都可封存，否則批次封存／刪除會維持停用。作用中的執行及代理程式的主要工作階段無法封存。封存或刪除目前選取的工作階段後，「聊天」會切回該代理程式的主要工作階段。
    - 在 macOS 應用程式中，OpenClaw 標誌會使用視窗控制項旁原本空白的原生標題列區帶，而不會占用側邊欄資料列。
    - 在桌面寬度下，聊天控制項會維持在單一精簡資料列中，並在向下捲動逐字稿時收合；向上捲動、返回頂端或到達底部時，控制項便會恢復。
    - 連續且重複的純文字訊息會轉譯為一個帶有數量徽章的對話泡泡。包含圖片、附件、工具輸出或畫布預覽的訊息則不會收合。
    - 使用者訊息泡泡提供逐字稿動作：游標停留時顯示倒轉按鈕（確認彈出視窗含有 "Don't ask again" 選項），並在按一下滑鼠右鍵時提供**倒轉至此處**和**從此處建立分支**。倒轉會將工作階段重新指向該訊息之前的狀態，並將其文字送回編輯器以供編輯及重新傳送（`sessions.rewind`、`operator.admin`）；建立分支則會從該訊息之前作用中路徑的前綴建立新工作階段、將其開啟，並以相同文字預先填入其編輯器（`sessions.fork`、`operator.write`）。代理程式工作期間，這兩個動作都會停用並顯示說明工具提示；它們僅適用於已持久保存的使用者訊息，且若工作階段的對話由外部代理程式控制架構擁有，則會遭到拒絕。倒轉只會移動聊天內容脈絡，檔案及其他工具副作用不會還原，而倒轉前的逐字稿仍會保留在僅附加的工作階段儲存區中。當該儲存區包含多個逐字稿分支時，聊天標題列會顯示分支選單，其中包含每個分支的最新訊息、訊息數量與新近程度；選取非作用中分支時，會將目前工作階段切回該保留路徑（`sessions.branches.list`、`operator.read`；`sessions.branches.switch`、`operator.admin`）。代理程式工作期間同樣無法切換分支，而選取已作用中的分支會在 RPC 邊界產生具型別的無操作錯誤。使用者泡泡上的獨立隱藏動作只會在目前瀏覽器中隱藏訊息；訊息仍會保留在逐字稿中，代理程式也仍能看見。
    - 當工作階段的簽出位於 GitHub 儲存庫的非預設分支時，聊天檢視會將 PR 方塊釘選在編輯器上方：PR 編號、儲存庫、分支、差異計數、CI 膠囊及草稿／已合併／已關閉狀態，且每個項目都會連結至該 PR。該資料列最多顯示兩個方塊，優先顯示即時（開啟／草稿）PR，而 "Show more" 按鈕可顯示已收合的已合併／已關閉歷史記錄。CI 膠囊會開啟小型 CI 監控彈出視窗，其中包含通過／失敗／執行中／已略過的檢查數量，以及 PR 檢查頁面的連結。偵測會透過 `controlUi.sessionPullRequests` 在伺服器端執行，並在已設定時重複使用閘道的 `GH_TOKEN`／`GITHUB_TOKEN`。觸及 GitHub API 速率限制時，方塊會保留最後已知狀態，並顯示狀態可能已過時的警告；關閉方塊會針對目前瀏覽器設定檔中的該工作階段將其隱藏。在任何 PR 存在之前，該資料列會顯示分支本身，包括儲存庫、分支名稱，以及相對於預設分支合併基準的差異 +/− 大小（已提交及未提交的工作）。當已推送的分支有可供比較的提交後，資料列會新增 Create PR 按鈕，以開啟 GitHub 的新 PR 頁面；在此之前，只要工作階段有變更的檔案（已提交、未提交或未追蹤），仍會顯示該資料列，但不會顯示按鈕。存在開啟或草稿 PR 時，該資料列會自行隱藏。分支資料列僅來自本機 git，因此即使 GitHub 受到速率限制，它仍可使用，並會顯示相同的過時狀態警告，因為在限制重設前，無法信任「找不到 PR」的結果。
    - 工作階段差異面板會顯示工作階段簽出實際變更的內容：工作區導軌或聊天標題列中的分支按鈕會開啟詳細資料面板，針對相對於簽出預設分支合併基準的分支、未提交及未追蹤工作顯示逐檔差異，包括狀態圓點、重新命名箭頭、逐檔 +/− 計數、可收合檔案，以及差異區塊之間的 "N unmodified lines" 標記。差異會透過 `sessions.diff` 閘道方法（`operator.read` 範圍）在伺服器端計算；二進位及超出大小限制的檔案會降級為僅顯示統計資料的項目，而按鈕僅會在連線的閘道宣告支援 `sessions.diff` 時顯示。
    - 每個「聊天」窗格都有標題列。按一下工作階段標題即可重新命名；工作區方塊可複製簽出路徑或分支，並可在主機檔案管理員中顯示本機閘道工作區。遠端及執行節點工作階段會保留複製動作，但隱藏顯示功能。
    - 每個「聊天」窗格中的對話串工作區導軌會列出對話串檔案、專案檔案及成品。預設會停駐在窗格右側邊緣；拖曳其標頭（或使用停駐按鈕）可將其移至底部，且此選擇會儲存在目前瀏覽器設定檔中。收合的導軌完全不占用空間：使用 ⇧⌘B 或標題列中的檔案切換按鈕重新開啟；該按鈕會顯示已變更檔案數量徽章。獨立的檔案、工具和畫布詳細資料面板不受影響。
    - 按一下聊天中的檔案參照、展開的讀取／編輯／寫入工具卡片中的檔案路徑，或工作區導軌中的檔案資料列，會開啟檔案詳細資料面板：以 CodeMirror 為基礎的程式碼檢視，提供語法醒目提示、行號、跳至指定行、檔案內搜尋、複製動作及在外部編輯器中開啟的選單。當閘道向 `operator.admin` 連線宣告支援 `sessions.files.set` 時，面板會新增「編輯」模式，提供變更追蹤及 Cmd/Ctrl-S 儲存功能；未儲存的草稿會在目前瀏覽器分頁中跨檔案、面板及工作階段導覽保留，直到明確儲存或捨棄為止。儲存作業會針對 `sessions.files.get` 傳回的內容雜湊執行比較並交換：如果檔案自載入後已在磁碟上變更（例如代理程式持續工作所致），面板會顯示衝突通知，並提供 Reload（採用最新內容）及 Overwrite（保留本機編輯）動作。寫入作業使用與讀取相同且符合檔案系統安全性的工作區防護措施，包括路徑包含範圍、拒絕符號連結／硬連結，以及 256 KB UTF-8 上限，且只能覆寫現有檔案；編輯器絕不會建立或刪除檔案。
    - 每個「聊天」窗格中的背景工作導軌會列出目前代理程式的背景工作及子代理程式（`tasks.list` 依代理程式設定範圍，並由 `task` 事件保持即時更新）：執行中的工作會顯示即時經過時間計時器、工具使用次數、目前使用中的工具及停止控制項；可收合的已完成區段則會新增執行持續時間；「檢視逐字稿」連結會在窗格中開啟該工作的子工作階段。使用標題列活動切換按鈕開啟；工作快照會預先載入，因此無須先開啟導軌就會顯示執行中數量徽章。「工作」頁面仍是完整的跨代理程式總帳。
    - 工作區側欄、背景工作側欄與詳細資料面板會依各窗格本身的寬度調整，而非依視窗寬度：在狹窄窗格或緊湊視窗中，兩個側欄都會顯示為底部橫列（側邊停駐控制項會隱藏，直到窗格變寬；若空間僅容納一欄，工作區側欄會優先占用側邊位置），而詳細資料面板會堆疊在線程下方，並提供水平調整大小控點，而非與線程並排顯示。手機尺寸的檢視區仍會以全螢幕開啟詳細資料面板。
    - 聊天標頭中的模型與思考選擇器會透過 `sessions.patch` 立即更新作用中的工作階段；它們是持續生效的工作階段覆寫設定，而非僅適用於單次傳送的選項。
    - **分割檢視：**從聊天標題列開啟（位於線程差異、背景工作和線程檔案切換鈕旁），然後將作用中窗格向右或向下分割，直到達到可容納的窗格數量。每個窗格都有各自的線程、文字記錄、撰寫器和工具串流。
    - 具備 `screen` 工具的代理程式，可在具備相應功能的控制介面連線時，要求執行相同的窗格、側邊欄、終端機、瀏覽器、焦點及導覽變更。協定 v1 會將命令套用至每個已連線且具備相應功能的控制介面；請參閱[螢幕](/zh-TW/tools/screen)。
    - 將工作階段從側邊欄拖曳至聊天區，即可在窗格中開啟。動畫放置預覽會在各區域間滑動，並標示結果——在新窗格將占用的確切半區上顯示「分割」，在整個窗格上顯示「在此開啟」——單一窗格模式也支援放置操作。

    - 如果你在同一工作階段的模型選擇器變更仍在儲存時傳送訊息，撰寫器會先等待該工作階段更新完成，再呼叫 `chat.send`，以確保傳送時使用所選模型。
    - 輸入 `/new` 會建立並切換至與「新增聊天」相同的全新儀表板工作階段；但若已設定 `session.dmScope: "main"`，且目前父項是代理程式的主要工作階段，則會就地重設該主要工作階段。輸入 `/reset` 則會保留閘道針對目前工作階段明確執行的就地重設。
    - 聊天模型選擇器會向閘道要求已設定的模型檢視。若 `agents.defaults.modelPolicy.allow` 非空，該原則會決定選擇器內容，包括讓限定於供應商範圍的目錄維持動態的 `provider/*` 項目。否則，選擇器會顯示已設定的項目，以及具有可用驗證的供應商；`agents.defaults.models` 下的別名與設定不會限制其內容。完整目錄仍可透過偵錯用的 `models.list` RPC 搭配 `view: "all"` 取得。

## 連線中斷與重新連線

建立工作階段後，閘道連線中斷不會將你登出。用戶端自動採用退避機制重試時（從 800 ms 到最多 15 s），儀表板會保持顯示，並在頂端列下方顯示浮動的琥珀色「閘道連線已中斷 — 正在重新連線…」膠囊提示。即時更新與即時通訊／工作階段動作會暫停，直到連線恢復；膠囊提示中的 **Retry now** 會強制立即嘗試連線。聊天仍可編輯：一般文字與附件傳送內容會保留在目前分頁中依閘道／工作階段劃分的瀏覽器儲存空間、顯示為等待重新連線，並在閘道恢復後自動傳送。離線時，即時控制項與斜線命令仍無法使用。

當此瀏覽器已持有認證資訊（已設定的權杖／密碼或已核准的裝置權杖）時，首次開啟及重新載入會在建立連線期間顯示小型動畫 OpenClaw 標誌，而不會短暫顯示登入閘門。只有在尚未儲存認證資訊，或閘道主動拒絕認證資訊（錯誤的權杖／密碼、已撤銷的配對）時，才會顯示登入閘門——這些狀態需要你提供輸入，而非繼續等待。

## 安裝 PWA 與網頁推播

控制介面隨附 `manifest.webmanifest` 和服務工作者，因此現代瀏覽器可將其安裝為獨立 PWA。即使分頁或瀏覽器視窗未開啟，網頁推播也能讓閘道透過通知喚醒已安裝的 PWA。

在 macOS 應用程式內，「通知」設定頁面會顯示應用程式的原生通知權限，而不是瀏覽器推播，因為應用程式會以原生方式傳遞通知。

如果頁面在 OpenClaw 更新後立即顯示 **通訊協定不相符**，請先使用 `openclaw dashboard` 重新開啟儀表板並強制重新整理。如果仍然失敗，請清除儀表板來源的網站資料，或在私密瀏覽器視窗中測試；舊分頁或瀏覽器服務工作者快取可能會繼續執行更新前的控制介面套件，並與較新的閘道搭配使用。

| 介面                                               | 功能說明                                                                     |
| -------------------------------------------------- | ---------------------------------------------------------------------------- |
| `ui/public/manifest.webmanifest`                   | PWA 資訊清單。可供存取後，瀏覽器會提供「Install app」。                     |
| `ui/public/sw.js`                                  | 處理 `push` 事件與通知點擊的服務工作者。           |
| `state/openclaw.sqlite` → `web_push_vapid_keys`    | 用於簽署網頁推播承載資料的自動產生 VAPID 金鑰組。                 |
| `state/openclaw.sqlite` → `web_push_subscriptions` | 持久保存的瀏覽器訂閱端點、金鑰及註冊時間戳記。 |

從已淘汰的 `push/vapid-keys.json` 和 `push/web-push-subscriptions.json` 儲存區升級時，資料會由 `openclaw doctor --fix` 匯入。執行該修復前請停止閘道，以免舊處理程序在匯入期間重新建立已淘汰的狀態。升級後使用網頁推播前，請先執行修復；只要任一已淘汰的來源仍存在，或 Doctor 宣告的處理曾遭中斷，註冊、傳遞、刪除與金鑰解析都會拒絕繼續。閘道執行階段只會讀寫 SQLite。

如需固定金鑰（多主機部署、秘密輪替或測試），請透過閘道處理程序的環境變數覆寫 VAPID 金鑰組：

- `OPENCLAW_VAPID_PUBLIC_KEY`
- `OPENCLAW_VAPID_PRIVATE_KEY`
- `OPENCLAW_VAPID_SUBJECT`（預設為 `https://openclaw.ai`）

控制介面使用以下受範圍限制的閘道方法，註冊及測試瀏覽器訂閱：

- `push.web.vapidPublicKey` 會取得作用中的 VAPID 公開金鑰。
- `push.web.subscribe` 會註冊一個 `endpoint`，以及 `keys.p256dh`/`keys.auth`。
- `push.web.unsubscribe` 會移除已註冊的端點。
- `push.web.test` 會將測試通知傳送至呼叫端的訂閱。

<Note>
網頁推播與 iOS APNS 中繼路徑（中繼支援的推播請參閱[設定](/zh-TW/gateway/configuration)）及 `push.test` 方法相互獨立；後者的目標是原生行動裝置配對。
</Note>

## 託管嵌入內容

助理訊息可使用 `[embed ...]` 短代碼，在訊息內直接呈現託管網頁內容。iframe 沙箱原則由 `gateway.controlUi.embedSandbox` 控制：

核心 [`show_widget`](/zh-TW/tools/show-widget) 工具會直接從工具呼叫呈現獨立完整的 SVG 或 HTML。瀏覽器及支援的原生聊天用戶端會宣告 `inline-widgets` 閘道功能，聊天歷程重新載入後，產生的 Canvas 文件仍可使用。Discord Activities 在 Discord 上提供相同的工具名稱；源自其他頻道的執行作業不會收到此工具。

<Tabs>
  <Tab title="嚴格">
    停用託管嵌入內容中的指令碼執行。
  </Tab>
  <Tab title="指令碼（預設）">
    允許互動式嵌入內容，同時維持來源隔離；通常足以用於獨立完整的瀏覽器遊戲／小工具。
  </Tab>
  <Tab title="信任">
    在 `allow-scripts` 之上加入 `allow-same-origin`，供刻意需要較高權限的同站台文件使用。
  </Tab>
</Tabs>

```json5
{
  gateway: {
    controlUi: {
      embedSandbox: "scripts",
    },
  },
}
```

<Warning>
只有在嵌入文件確實需要同源行為時，才使用 `trusted`。對於大多數代理程式產生的遊戲與互動式畫布，`scripts` 是較安全的選擇。
</Warning>

預設會封鎖絕對外部 `http(s)` 嵌入 URL。如要允許 `[embed url="https://..."]` 載入第三方頁面，請設定 `gateway.controlUi.allowExternalEmbedUrls: true`。

## 聊天訊息寬度

聊天逐字稿使用與編輯器對齊、置中且便於閱讀的框架。助理與工具輸出在該框架內維持靠左對齊，使用者訊息泡泡則維持靠右對齊。寬螢幕部署可設定 `ui.prefs.chatMessageMaxWidth` 來覆寫逐字稿寬度，無須修改隨附的 CSS：

```json5
{
  gateway: {
    controlUi: {
      chatMessageMaxWidth: "min(1280px, 82%)",
    },
  },
}
```

此值會先經過驗證，才傳送至瀏覽器。支援的格式包括 `960px` 或 `82%` 等純長度與百分比，以及受限的 `min(...)`、`max(...)`、`clamp(...)`、`calc(...)` 和 `fit-content(...)` 寬度運算式。

## Tailnet 存取（建議）

<Tabs>
  <Tab title="整合式 Tailscale Serve（首選）">
    將閘道保留在迴路介面上，並讓 Tailscale Serve 使用 HTTPS 代理：

    ```bash
    openclaw gateway --tailscale serve
    ```

    開啟 `https://<magicdns>/`（或你設定的 `gateway.controlUi.basePath`）。

    依預設，Control UI/WebSocket Serve 請求可在 `gateway.auth.allowTailscale` 為 `true` 時，透過 Tailscale 身分標頭（`tailscale-user-login`）進行驗證。OpenClaw 會使用 `tailscale whois` 解析 `x-forwarded-for` 位址，並將結果與標頭比對以驗證身分；只有當請求透過迴送介面抵達，且帶有 Tailscale 的 `x-forwarded-*` 標頭時，才會接受這些標頭。對於具有瀏覽器裝置身分的 Control UI 操作者工作階段，這個已驗證的 Serve 路徑也會略過裝置配對的往返流程；無裝置身分的瀏覽器和節點角色連線仍會遵循一般裝置檢查。如果你希望即使是 Serve 流量也必須提供明確的共用密鑰認證資訊，請設定 `gateway.auth.allowTailscale: false`，然後使用 `gateway.auth.mode: "token"` 或 `"password"`。

    對於該非同步 Serve 身分路徑，來自相同用戶端 IP 且屬於相同驗證範圍的失敗驗證嘗試，會在寫入速率限制資料前依序處理。因此，來自同一瀏覽器的並行錯誤重試可能會讓第二個請求顯示 `retry later`，而不是讓兩次單純的不相符結果並行競爭。

    <Warning>
    無權杖的 Serve 驗證假設閘道主機可信任。如果不受信任的本機程式碼可能在該主機上執行，請要求權杖／密碼驗證。
    </Warning>

  </Tab>

    ```bash
    openclaw gateway --bind tailnet --token "$(openssl rand -hex 32)"
    ```

    開啟 `http://<tailscale-ip>:18789/`（或你設定的 `gateway.controlUi.basePath`）。

    將相符的共用密鑰貼到 UI 設定中（以 `connect.params.auth.token` 或 `connect.params.auth.password` 傳送）。

## 不安全的 HTTP

如果你透過純 HTTP（`http://<lan-ip>` 或 `http://<tailscale-ip>`）開啟儀表板，瀏覽器會在**非安全情境**中執行並封鎖 WebCrypto。依預設，OpenClaw 會**封鎖**沒有裝置身分的 Control UI 連線。

支援的無裝置身分例外，是透過 `gateway.auth.mode: "trusted-proxy"` 成功完成 Control UI 操作者驗證。沒有可停用裝置身分的持久性設定開關。

**建議修正方式：**使用 HTTPS（Tailscale Serve），或在本機透過 `https://<magicdns>/`（Serve）或 `http://127.0.0.1:18789/`（在閘道主機上）開啟 UI。

<AccordionGroup>
  <Accordion title="受信任 Proxy 注意事項">
    - 成功的受信任 Proxy 驗證可以允許沒有裝置身分的 **operator** Control UI 工作階段。
    - 這**不會**延伸至節點角色的 Control UI 工作階段。
    - 同一主機上的迴送反向 Proxy 仍不符合受信任 Proxy 驗證的條件；請參閱[受信任 Proxy 驗證](/zh-TW/gateway/trusted-proxy-auth)。

  </Accordion>
</AccordionGroup>

如需 HTTPS 設定指引，請參閱 [Tailscale](/zh-TW/gateway/tailscale)。

## 內容安全政策

Control UI 隨附嚴格的 `img-src` 政策：僅允許**同源**資產、`data:` URL，以及在本機產生的 `blob:` URL。遠端 `http(s)` 和通訊協定相對圖片 URL 會遭瀏覽器拒絕，且絕不會發出網路擷取請求。

實際行為如下：

- 透過相對路徑提供的頭像和圖片（例如 `/avatars/<id>`）仍可顯示，包括由 UI 擷取並轉換成本機 `blob:` URL、需要驗證的頭像路由。
- 行內 `data:image/...` URL 仍可顯示。
- 由 Control UI 建立的本機 `blob:` URL 仍可顯示。
- GitHub 連結預覽頭像會由閘道從 GitHub 的固定頭像主機擷取，並以受大小限制的 `data:` URL 傳回；操作者的瀏覽器絕不會連線至遠端頭像主機。
- 頻道中繼資料所產生的遠端頭像 URL，會由 Control UI 的頭像輔助程式移除，並改用內建標誌／徽章，因此遭入侵或惡意的頻道無法強迫操作者的瀏覽器任意擷取遠端圖片。

此功能一律啟用，且無法設定。

## 頭像路由驗證

設定閘道驗證後，Control UI 頭像端點需要使用與 API 其餘部分相同的閘道權杖：

- `GET /avatar/<agentId>` 僅向已驗證的呼叫端傳回頭像圖片。`GET /avatar/<agentId>?meta=1` 依相同規則傳回頭像中繼資料。
- 對任一路由提出的未驗證請求都會遭拒絕（與同層的助理媒體路由一致），因此頭像路由無法在其他部分受到保護的主機上洩漏代理程式身分。
- Control UI 在擷取頭像時，會以 Bearer 標頭轉送閘道權杖，並使用經驗證的 blob URL，讓圖片仍可顯示於儀表板中。

如果你停用閘道驗證（不建議在共用主機上這麼做），頭像路由也會與閘道其餘部分一致，變成不需要驗證。

## 助理媒體路由驗證

設定閘道驗證後，助理的本機媒體預覽會使用兩階段路由：

- `GET /__openclaw__/assistant-media?meta=1&source=<path>` 需要一般的 Control UI 操作者驗證；瀏覽器在檢查可用性時，會以 Bearer 標頭傳送閘道權杖。
- 成功的中繼資料回應會包含一個短效的 `mediaTicket`，其範圍限定於該確切來源路徑。
- 由瀏覽器呈現的圖片、音訊、影片和文件 URL 會使用 `mediaTicket=<ticket>`，而非使用中的閘道權杖或密碼。票證會很快到期，且無法授權不同來源。

這可讓媒體呈現與瀏覽器原生媒體元素保持相容，同時避免將可重複使用的閘道認證資訊放入可見的媒體 URL 中。

## 核准連結

操作者核准通知可深層連結至保留的 `${controlUiBasePath}/approve/{approvalId}` 命名空間下所提供的獨立核准文件（例如 `/approve/<approvalId>`，或在已設定基底路徑時使用 `/openclaw/approve/<approvalId>`）。該 URL 在核准的有效期間內保持穩定，且可安全地轉傳至你自己的裝置：它只會識別該核准，絕不會授權核准。

- 單一區段的 `/approve/<approvalId>` 命名空間會由閘道優先於外掛 HTTP 路由，保留給**所有** HTTP 方法，因此外掛路由絕不可能遮蔽或攔截核准文件。
- 開啟核准文件需要使用與 Control UI 其餘部分相同的閘道驗證（權杖／密碼、Tailscale Serve 身分或受信任 Proxy 身分）；認證資訊絕不會成為核准 URL 的一部分。
- 停用 Control UI 服務時，對該命名空間的請求會傳回 `404`，而不會落入外掛處理常式。
- 在核准文件上登入只會暫時套用於該頁面：它不會覆寫同一瀏覽器中由完整 Control UI 儲存的閘道選擇或設定。

閘道從 `dist/control-ui` 提供靜態檔案：

```bash
pnpm ui:build
```

選用的絕對基底（固定資產 URL）：

```bash
OPENCLAW_CONTROL_UI_BASE_PATH=/openclaw/ pnpm ui:build
```

本機開發（獨立開發伺服器）：

```bash
pnpm ui:dev
```

接著將 UI 指向你的閘道 WS URL（例如 `ws://127.0.0.1:18789`）。

## Control UI 空白頁面

如果瀏覽器載入空白儀表板，且 DevTools 未顯示有用的錯誤，可能是擴充功能或提早執行的內容指令碼阻止 JavaScript 模組應用程式進行求值。當啟動後未註冊 `<openclaw-app>` 時，靜態頁面會顯示純 HTML 復原面板。

變更瀏覽器環境後，使用面板上的 **Try again** 動作，或在完成以下檢查後手動重新載入：

- 停用會注入所有頁面的擴充功能，尤其是具有 `<all_urls>` 內容指令碼的擴充功能。
- 嘗試使用私密視窗、乾淨的瀏覽器設定檔或其他瀏覽器。
- 保持閘道執行，並在變更瀏覽器後驗證相同的儀表板 URL。

## 偵錯／測試：開發伺服器 + 遠端閘道

Control UI 是靜態檔案；WebSocket 目標可設定，且可與 HTTP 來源不同。當你想在本機執行 Vite 開發伺服器、但閘道在其他位置執行時，這會很方便。

<Steps>
  <Step title="啟動 UI 開發伺服器">
    ```bash
    pnpm ui:dev
    ```
  </Step>
  <Step title="使用 gatewayUrl 開啟">
    ```text
    http://localhost:5173/?gatewayUrl=ws%3A%2F%2F<gateway-host>%3A18789
    ```

    選用的一次性驗證（如有需要）：

    ```text
    http://localhost:5173/?gatewayUrl=wss%3A%2F%2F<gateway-host>%3A18789#token=<gateway-token>
    ```

  </Step>
</Steps>

<AccordionGroup>
  <Accordion title="注意事項">
    - `gatewayUrl` 會在載入後儲存於 localStorage，並從 URL 中移除。
    - 如果你透過 `gatewayUrl` 傳入完整的 `ws://` 或 `wss://` 端點，請對值進行 URL 編碼，讓瀏覽器正確剖析查詢字串。
    - 只要可行，`token` 應透過 URL 片段（`#token=...`）傳遞。片段不會傳送至伺服器，因此可避免洩漏至請求記錄和 Referer。為了相容性，舊版 `?token=` 查詢參數仍會匯入一次，但僅作為備援，並會在啟動程序完成後立即移除。
    - `password` 僅保留在記憶體中。
    - 設定 `gatewayUrl` 時，UI 不會改用設定或環境中的認證資訊。請明確提供 `token`（或 `password`）；缺少明確的認證資訊會導致錯誤。
    - 當閘道位於 TLS（Tailscale Serve、HTTPS Proxy 等）後方時，請使用 `wss://`。
    - `gatewayUrl` 僅在最上層視窗中接受（不可內嵌），以防止點擊劫持。
    - 公開且非迴送的 Control UI 部署必須明確設定 `gateway.controlUi.allowedOrigins`（完整來源）。從迴送、RFC1918／連結本機、`.local`、`.ts.net` 或 Tailscale CGNAT 主機載入的私有同源 LAN／Tailnet 請求，無須啟用 Host 標頭備援即可接受。
    - 閘道啟動時，可能會根據實際執行階段的繫結與連接埠植入 `http://localhost:<port>` 和 `http://127.0.0.1:<port>` 等本機來源，但遠端瀏覽器來源仍需要明確的項目。
    - 除非是受到嚴格控制的本機測試，否則請勿使用 `gateway.controlUi.allowedOrigins: ["*"]`；它表示允許任何瀏覽器來源，而不是“比對我正在使用的任何主機”。
    - `gateway.controlUi.dangerouslyAllowHostHeaderOriginFallback=true` 會啟用 Host 標頭來源備援模式，但這是一種危險的安全模式。

  </Accordion>
</AccordionGroup>

```json5
{
  gateway: {
    controlUi: {
      allowedOrigins: ["http://localhost:5173"],
    },
  },
}
```
