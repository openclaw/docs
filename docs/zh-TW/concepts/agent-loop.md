代理迴圈是依工作階段序列化執行的流程，會將訊息轉換為
動作與回覆：接收、組合上下文、模型推論、工具
執行、串流、持久化。

## 進入點

- 閘道 RPC：`agent` 和 `agent.wait`。
- 命令列介面：`openclaw agent`。

## 執行順序

1. `agent` RPC 會驗證參數、解析工作階段（`sessionKey`/`sessionId`）、持久化工作階段中繼資料，並立即傳回 `{ runId, acceptedAt }`。
2. `agentCommand` 會執行該輪次：解析模型及思考／詳細／追蹤的預設值、載入 Skills 快照、呼叫 `runEmbeddedAgent`；若內嵌迴圈尚未發出事件，則發出備援的**生命週期結束／錯誤**事件。
3. `runEmbeddedAgent`：透過每工作階段及全域佇列序列化執行、解析模型與驗證設定檔、建立 OpenClaw 工作階段、訂閱執行階段事件、串流助理／工具增量、強制執行逾時（到期時中止），並傳回承載資料及用量中繼資料。對於 Codex 應用程式伺服器輪次，若已接受的輪次在終止事件前不再產生應用程式伺服器進度，也會將其中止。
4. `subscribeEmbeddedAgentSession` 會將執行階段事件橋接至 `agent` 串流：工具事件傳至 `stream: "tool"`、助理增量傳至 `stream: "assistant"`、生命週期事件傳至 `stream: "lifecycle"`（`phase: "start" | "end" | "error"`）。
5. `agent.wait`（`waitForAgentRun`）會在 `runId` 上等候**生命週期結束／錯誤**，並傳回 `{ status: ok|error|timeout, startedAt, endedAt, error? }`。

## 佇列與並行處理

執行會依工作階段金鑰（工作階段通道）序列化，並可選擇再通過全域通道，以防止工具／工作階段競爭。訊息通道會選擇將內容送入此通道系統的佇列模式（steer/followup/collect/interrupt）；請參閱[命令佇列](/zh-TW/concepts/queue)。

逐字稿寫入還會受到工作階段檔案上的工作階段寫入鎖保護。此鎖可感知處理程序且以檔案為基礎，因此能捕捉略過處理程序內佇列或來自其他處理程序的寫入者。寫入者預設最多等候 60 秒（可透過環境變數 `OPENCLAW_SESSION_WRITE_LOCK_ACQUIRE_TIMEOUT_MS` 覆寫），之後便會回報工作階段忙碌中。

工作階段寫入鎖預設不可重入。若輔助程式刻意巢狀取得同一把鎖，同時維持單一邏輯寫入者，則必須使用 `allowReentrant: true` 明確選擇啟用。

## 工作階段與工作區準備

- 解析並建立工作區；沙箱化執行可能會重新導向至沙箱工作區根目錄。
- 載入 Skills（或從快照重複使用），並注入環境與提示詞。
- 解析啟動／上下文檔案，並注入系統提示詞。
- 在串流開始前，會取得工作階段寫入鎖並準備工作階段逐字稿目標。之後任何逐字稿重寫、壓縮或截斷路徑，在變更 SQLite 逐字稿資料列前，都必須取得相同的鎖。

## 提示詞組合

系統提示詞由 OpenClaw 的基礎提示詞、Skills 提示詞、啟動上下文及每次執行的覆寫內容建構而成。會強制套用模型特定限制及壓縮保留 Token。關於模型會看到的內容，請參閱[系統提示詞](/zh-TW/concepts/system-prompt)。

## 鉤子

OpenClaw 有兩套鉤子系統：

- **內部鉤子**（閘道鉤子）：用於命令與生命週期事件的事件驅動指令碼。
- **外掛鉤子**：代理／工具生命週期及閘道流水線內的擴充點。

### 內部鉤子（閘道鉤子）

- **`agent:bootstrap`**：在系統提示詞定稿前建立啟動檔案時執行。可用來新增或移除啟動上下文檔案。
- **命令鉤子**：`/new`、`/reset`、`/stop` 及其他命令事件（請參閱鉤子文件）。

設定方式與範例請參閱[鉤子](/zh-TW/automation/hooks)。

### 外掛鉤子

這些鉤子會在代理迴圈或閘道流水線內執行：

| 鉤子                                                    | 執行時機                                                                                                                                                                                                                                                                                        |
| ------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `before_model_resolve`                                  | 工作階段前（無 `messages`），用來在解析前以確定性的方式覆寫供應商／模型。                                                                                                                                                                                                |
| `before_prompt_build`                                   | 載入工作階段後（含 `messages`），在提交前注入 `prependContext`、`systemPrompt`、`prependSystemContext` 或 `appendSystemContext`。每輪動態文字請使用 `prependContext`；屬於系統提示詞範疇的穩定指引則使用系統上下文欄位。 |
| `before_agent_reply`                                    | 內嵌動作後、呼叫 LLM 前。讓外掛接管該輪次，並傳回合成回覆或使其完全靜默。                                                                                                                                                                |
| `agent_end`                                             | 完成後執行，並提供最終訊息清單及執行中繼資料。                                                                                                                                                                                                                             |
| `before_compaction` / `after_compaction`                | 觀察或註記壓縮週期。                                                                                                                                                                                                                                                      |
| `before_tool_call` / `after_tool_call`                  | 攔截工具參數／結果。                                                                                                                                                                                                                                                              |
| `before_install`                                        | 操作者安裝政策執行後，在已暫存的 Skill／外掛安裝材料上，且目前處理程序已載入外掛鉤子時執行。                                                                                                                                                           |
| `tool_result_persist`                                   | 在工具結果寫入 OpenClaw 所擁有的工作階段逐字稿前，以同步方式進行轉換。                                                                                                                                                                                      |
| `message_received` / `message_sending` / `message_sent` | 傳入與傳出訊息鉤子。                                                                                                                                                                                                                                                         |
| `session_start` / `session_end`                         | 工作階段生命週期邊界。                                                                                                                                                                                                                                                               |
| `gateway_start` / `gateway_stop`                        | 閘道生命週期事件。                                                                                                                                                                                                                                                                   |

傳出／工具防護的鉤子決策規則：

- `before_tool_call`：`{ block: true }` 為終止結果，並會停止優先順序較低的處理常式。`{ block: false }` 不執行任何操作，也不會清除先前的封鎖。
- `before_install`：具有與上述相同的終止／不執行語意。對於必須涵蓋命令列介面安裝與更新路徑、由操作者擁有的安裝允許／封鎖決策，請使用 `security.installPolicy`，而非 `before_install`。
- `message_sending`：`{ cancel: true }` 為終止結果，並會停止優先順序較低的處理常式。`{ cancel: false }` 不執行任何操作，也不會清除先前的取消。

鉤子 API 與註冊詳細資訊請參閱[外掛鉤子](/zh-TW/plugins/hooks)。

測試框架可調整這些鉤子。Codex 應用程式伺服器測試框架會將 OpenClaw 外掛鉤子保留為已記錄鏡像介面的相容性合約；Codex 原生鉤子則是另一套較低階的 Codex 機制。

## 串流

- 助理增量會從代理執行階段以 `assistant` 事件串流送出。
- 區塊串流可在 `text_end` 或 `message_end` 時發出部分回覆。
- 推理串流可作為獨立串流或區塊回覆。
- 關於分塊及區塊回覆行為，請參閱[串流](/zh-TW/concepts/streaming)。

## 工具執行

- 工具開始／更新／結束事件會在 `tool` 串流上發出。
- 工具結果會先針對大小及圖片承載資料進行清理，再記錄／發出。
- 會追蹤訊息工具傳送，以抑制重複的助理確認訊息。

## 回覆塑形

最終承載資料由助理文字（以及選用的推理內容）、內嵌工具摘要（啟用詳細模式且允許時），以及模型發生錯誤時的助理錯誤文字組成。

- 傳出承載資料會過濾完全相符的靜默 Token `NO_REPLY`。
- 最終承載資料清單會移除重複的訊息工具內容。
- 若沒有任何可算繪的承載資料，且工具發生錯誤，除非訊息工具已傳送使用者可見的回覆，否則會發出備援的工具錯誤回覆。

## 壓縮與重試

自動壓縮會發出 `compaction` 串流事件，且可能觸發重試。重試時，記憶體內緩衝區及工具摘要會重設，以避免重複輸出。請參閱[壓縮](/zh-TW/concepts/compaction)。

## 事件串流

- `lifecycle`：由 `subscribeEmbeddedAgentSession` 發出（亦可由 `agentCommand` 作為備援發出）。
- `assistant`：來自代理執行階段的串流增量。
- `tool`：來自代理執行階段的串流工具事件。

閘道會將生命週期及工具開始／終止事件投影至有界、
僅含中繼資料的[稽核帳本](/zh-TW/cli/audit)。此投影會記錄來源及
結果代碼，而不會將提示詞、訊息、工具引數、工具結果
或原始錯誤複製到逐字稿／執行階段路徑之外。

## 聊天通道處理

助理增量會緩衝至聊天 `delta` 訊息中。發生**生命週期結束／錯誤**時，會發出聊天 `final`。

### 卡住的工作階段診斷

啟用診斷後，內建的兩分鐘門檻會將長時間未觀察到回覆、工具、狀態、阻塞或 ACP 進度的 `processing` 工作階段分類：

- 作用中的內嵌執行、模型呼叫與工具呼叫會回報為 `session.long_running`。由擁有者管理的靜默模型呼叫在達到中止門檻前會維持為 `session.long_running`，避免過早將緩慢或非串流供應商標記為停滯。
- 近期沒有進度的作用中工作會回報為 `session.stalled`。由擁有者管理的模型呼叫會在達到或超過中止門檻時切換為 `session.stalled`；沒有擁有者的過期模型／工具活動不會被隱藏為長時間執行。
- `session.stuck` 保留供可復原的過期工作階段簿記使用，包括具有過期無擁有者模型／工具活動的閒置佇列工作階段。

中止門檻至少為 5 分鐘，且為警告門檻的 3 倍。過期工作階段簿記會在復原閘門通過後立即釋放受影響的工作階段執行通道；停滯的內嵌執行僅會在中止門檻後才進行中止清空，因此佇列工作可以恢復，而不會截斷只是緩慢的執行。復原會發出結構化的已要求／已完成結果；只有在相同的處理世代仍為目前世代時，診斷狀態才會標記為閒置，而在工作階段維持不變時，重複的 `session.stuck` 診斷會採取退避。

## 可能提前結束的位置

- 代理程式逾時（中止）
- AbortSignal（取消）
- 閘道中斷連線或 RPC 逾時
- `agent.wait` 逾時（僅等待，不會停止代理程式）
