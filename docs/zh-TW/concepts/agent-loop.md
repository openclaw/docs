代理程式迴圈是每個工作階段各自依序執行的運作流程，會將訊息轉換為動作與回覆：接收、組裝脈絡、模型推論、工具執行、串流及持久化。

## 進入點

- 閘道 RPC：`agent` 和 `agent.wait`。
- 命令列介面：`openclaw agent`。

## 執行順序

1. `agent` RPC 會驗證參數、解析工作階段（`sessionKey`/`sessionId`）、持久化工作階段中繼資料，並立即傳回 `{ runId, acceptedAt }`。
2. `agentCommand` 會執行該輪：解析模型與思考／詳細／追蹤的預設值、載入 Skills 快照、呼叫 `runEmbeddedAgent`，並在內嵌迴圈尚未發出時，發出備援的**生命週期結束／錯誤**事件。
3. `runEmbeddedAgent`：透過每個工作階段與全域佇列依序執行工作、解析模型與驗證設定檔、建置 OpenClaw 工作階段、訂閱執行階段事件、串流助理／工具增量、強制執行工作逾時（到期時中止），並傳回承載資料與使用量中繼資料。對於 Codex app-server 輪次，若已接受的輪次在終止事件前停止產生 app-server 進度，也會將其中止。
4. `subscribeEmbeddedAgentSession` 會將執行階段事件橋接至 `agent` 串流：工具事件送至 `stream: "tool"`、助理增量送至 `stream: "assistant"`、生命週期事件送至 `stream: "lifecycle"`（`phase: "start" | "end" | "error"`）。
5. `agent.wait`（`waitForAgentRun`）會在 `runId` 上等待**生命週期結束／錯誤**，並傳回 `{ status: ok|error|timeout, startedAt, endedAt, error? }`。

## 佇列與並行處理

工作會依各工作階段鍵（工作階段通道）依序執行，並可選擇再通過全域通道，以避免工具／工作階段競爭。訊息通道會選擇一種佇列模式（引導／後續／收集／中斷），再將其送入此通道系統；請參閱[命令佇列](/zh-TW/concepts/queue)。

對話記錄寫入還會由工作階段檔案上的工作階段寫入鎖額外保護。此鎖會感知處理程序且以檔案為基礎，因此能攔截略過處理程序內佇列或來自其他處理程序的寫入者。寫入者預設最多等待 60 秒（可透過環境變數 `OPENCLAW_SESSION_WRITE_LOCK_ACQUIRE_TIMEOUT_MS` 覆寫），之後便會回報工作階段忙碌中。

工作階段寫入鎖預設不可重入。若輔助程式刻意巢狀取得同一把鎖，同時維持單一邏輯寫入者，則必須透過 `allowReentrant: true` 明確啟用。

## 工作階段與工作區準備

- 解析並建立工作區；沙箱化工作可能會重新導向至沙箱工作區根目錄。
- 載入 Skills（或重用快照），並將其注入環境與提示詞。
- 解析啟動／脈絡檔案，並將其注入系統提示詞。
- 在串流開始前取得工作階段寫入鎖，並準備工作階段對話記錄目標。後續任何對話記錄重寫、壓縮或截斷路徑，都必須在變更 SQLite 對話記錄資料列前取得相同的鎖。

## 提示詞組裝

系統提示詞由 OpenClaw 的基礎提示詞、Skills 提示詞、啟動脈絡及每次工作的覆寫設定建置而成。系統會強制套用模型特定限制與壓縮保留權杖。模型所見內容請參閱[系統提示詞](/zh-TW/concepts/system-prompt)。

## 掛鉤

OpenClaw 有兩套掛鉤系統：

- **內部掛鉤**（閘道掛鉤）：用於命令與生命週期事件的事件驅動指令碼。
- **外掛掛鉤**：代理程式／工具生命週期與閘道流水線內的擴充點。

### 內部掛鉤（閘道掛鉤）

- **`agent:bootstrap`**：在系統提示詞定案前建置啟動檔案時執行。可用它新增或移除啟動脈絡檔案。
- **命令掛鉤**：`/new`、`/reset`、`/stop` 及其他命令事件（請參閱掛鉤文件）。

設定與範例請參閱[掛鉤](/zh-TW/automation/hooks)。

### 外掛掛鉤

這些掛鉤會在代理程式迴圈或閘道流水線內執行：

| 掛鉤                                                    | 執行時機                                                                                                                                                                                                                                                                                        |
| ------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `before_model_resolve`                                  | 工作階段前（沒有 `messages`），可在解析前以確定性方式覆寫供應商／模型。                                                                                                                                                                                                |
| `before_prompt_build`                                   | 載入工作階段後（具有 `messages`），可在提交前注入 `prependContext`、`systemPrompt`、`prependSystemContext` 或 `appendSystemContext`。每輪動態文字請使用 `prependContext`；屬於系統提示詞空間的穩定指引則使用系統脈絡欄位。 |
| `before_agent_reply`                                    | 內嵌動作之後、呼叫 LLM 之前。讓外掛接管該輪並傳回合成回覆，或完全不回覆。                                                                                                                                                                |
| `agent_end`                                             | 完成後，提供最終訊息清單與工作中繼資料。                                                                                                                                                                                                                             |
| `before_compaction` / `after_compaction`                | 觀察或註記壓縮週期。                                                                                                                                                                                                                                                      |
| `before_tool_call` / `after_tool_call`                  | 攔截工具參數／結果。                                                                                                                                                                                                                                                              |
| `before_install`                                        | 操作者安裝政策執行後，針對暫存的 Skill／外掛安裝素材執行；此時外掛掛鉤必須已載入目前處理程序。                                                                                                                                                           |
| `tool_result_persist`                                   | 在工具結果寫入 OpenClaw 所擁有的工作階段對話記錄前，同步轉換工具結果。                                                                                                                                                                                      |
| `message_received` / `message_sending` / `message_sent` | 傳入與傳出訊息掛鉤。                                                                                                                                                                                                                                                         |
| `session_start` / `session_end`                         | 工作階段生命週期邊界。                                                                                                                                                                                                                                                               |
| `gateway_start` / `gateway_stop`                        | 閘道生命週期事件。                                                                                                                                                                                                                                                                   |

傳出／工具防護的掛鉤決策規則：

- `before_tool_call`：`{ block: true }` 是終止決策，會停止優先順序較低的處理常式。`{ block: false }` 不執行任何操作，也不會清除先前的封鎖。
- `before_install`：終止／不執行任何操作的語意同上。對於操作者所擁有且必須涵蓋命令列介面安裝與更新路徑的安裝允許／封鎖決策，請使用 `security.installPolicy`，而非 `before_install`。
- `message_sending`：`{ cancel: true }` 是終止決策，會停止優先順序較低的處理常式。`{ cancel: false }` 不執行任何操作，也不會清除先前的取消。

掛鉤 API 與註冊詳細資訊請參閱[外掛掛鉤](/zh-TW/plugins/hooks)。

測試框架可以調整這些掛鉤。Codex app-server 測試框架將 OpenClaw 外掛掛鉤保留為已記錄鏡像介面的相容性合約；Codex 原生掛鉤則是另一套更底層的 Codex 機制。

## 串流

- 助理增量會以 `assistant` 事件的形式從代理程式執行階段串流傳送。
- 區塊串流可在 `text_end` 或 `message_end` 時發出部分回覆。
- 推理串流可以是獨立串流或區塊回覆。
- 分塊與區塊回覆行為請參閱[串流](/zh-TW/concepts/streaming)。

## 工具執行

- 工具開始／更新／結束事件會在 `tool` 串流上發出。
- 工具結果會先針對大小與影像承載資料進行淨化，再記錄／發出。
- 系統會追蹤訊息工具的傳送，以抑制重複的助理確認。

## 回覆塑形

最終承載資料會由助理文字（以及選用的推理）、內嵌工具摘要（詳細模式啟用且允許時），以及模型發生錯誤時的助理錯誤文字組裝而成。

- 完全相符的靜默權杖 `NO_REPLY` 會從傳出承載資料中篩除。
- 最終承載資料清單會移除訊息工具的重複項目。
- 若未留下任何可呈現的承載資料，且有工具發生錯誤，便會發出備援工具錯誤回覆；但若訊息工具已傳送使用者可見的回覆，則不會發出。

## 壓縮與重試

自動壓縮會發出 `compaction` 串流事件，且可觸發重試。重試時，記憶體內緩衝區與工具摘要會重設，以避免重複輸出。請參閱[壓縮](/zh-TW/concepts/compaction)。

## 事件串流

- `lifecycle`：由 `subscribeEmbeddedAgentSession` 發出（`agentCommand` 也會作為備援發出）。
- `assistant`：來自代理程式執行階段的串流增量。
- `tool`：來自代理程式執行階段的串流工具事件。

閘道會將生命週期及工具開始／終止事件投影至有界且僅含中繼資料的[稽核帳本](/zh-TW/cli/audit)。此投影會記錄來源與結果代碼，而不會將提示詞、訊息、工具引數、工具結果或原始錯誤複製到對話記錄／執行階段路徑之外。

## 聊天通道處理

助理增量會緩衝至聊天 `delta` 訊息中。發生**生命週期結束／錯誤**時，會發出聊天 `final`。

### 工作階段卡住診斷

啟用診斷後，內建的兩分鐘門檻會對長時間沒有觀察到回覆、工具、狀態、區塊或 ACP 進度的 `processing` 工作階段進行分類：

- 進行中的內嵌執行、模型呼叫和工具呼叫會回報為 `session.long_running`。由擁有者管理且沒有輸出的模型呼叫，在達到中止門檻前會維持 `session.long_running`，避免過早將緩慢或非串流供應商標記為停滯。
- 近期沒有進度的進行中工作會回報為 `session.stalled`。由擁有者管理的模型呼叫會在達到或超過中止門檻時切換為 `session.stalled`；沒有擁有者的過期模型／工具活動不會被隱藏為長時間執行。
- `session.stuck` 保留給可復原的過期工作階段簿記，包括具有過期且沒有擁有者之模型／工具活動的閒置佇列工作階段。

中止門檻至少為 5 分鐘，且為警告門檻的 3 倍。復原閘門通過後，過期工作階段簿記會立即釋放受影響的工作階段通道；停滯的內嵌執行只有在達到中止門檻後才會進行中止排空，因此佇列工作可繼續執行，而不會切斷只是速度較慢的執行。復原會發出結構化的已要求／已完成結果；只有在相同的處理世代仍為目前世代時，診斷狀態才會標記為閒置，而且在工作階段維持不變時，重複的 `session.stuck` 診斷會採用退避機制。

## 可能提早結束的情況

- Agent 逾時（中止）
- AbortSignal（取消）
- 閘道中斷連線或 RPC 逾時
- `agent.wait` 逾時（僅等待，不會停止 Agent）
