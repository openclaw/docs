# 遙測

ClawHub 使用**最小化遙測**來計算**安裝數**（實際正在使用的項目），並支援更好的排序/篩選。
這是以 CLI `clawhub sync` 指令為基礎。

## 何時收集遙測

只有在以下情況才會傳送遙測：

- 你已在 CLI 中**登入**（我們已要求同步/發布流程需要驗證）。
- 你執行 `clawhub sync`。
- 遙測**未停用**（請參閱下方「如何停用」）。

如果你尚未登入，不會回報任何內容。

## 我們收集的內容

每次執行 `clawhub sync` 時，CLI 會回報它找到內容的**完整快照**，並依掃描根目錄（「資料夾/根目錄」）分組。

我們會為每個根目錄儲存：

- `rootId`：正規化根目錄路徑的 **SHA-256 雜湊**（伺服器永遠不會看到原始路徑）。
- `label`：從最後兩個路徑片段衍生的人類可讀標籤（家目錄路徑會以 `~` 顯示）。
- `firstSeenAt`、`lastSeenAt`，以及選用的 `expiredAt`。

我們會為根目錄底下找到的每個技能儲存：

- `skillId`（由短代稱解析；只會追蹤存在於註冊庫中的技能）。
- `firstSeenAt`、`lastSeenAt`。
- `lastVersion`（盡力而為；目前若已知，則為註冊庫相符的版本）。
- 當先前回報的安裝從根目錄消失時，選用的 `removedAt`。

### 我們_不會_收集的內容

- 不收集原始絕對資料夾路徑（只收集雜湊後的 `rootId` + 簡短顯示標籤）。
- 不收集檔案內容。
- 不收集每次執行的日誌、提示或其他 CLI 輸出。
- 不追蹤未上傳至註冊庫的技能（未知短代稱會被忽略）。

## 安裝數

我們為每個技能維護兩個計數器：

- `installsCurrent`：目前在至少一個有效根目錄中安裝該技能的唯一使用者數。
- `installsAllTime`：曾經回報安裝該技能的唯一使用者數。

### 多個根目錄

如果你從多個資料夾同步，我們會獨立處理每個掃描根目錄。只要技能存在於**任何**有效根目錄中，就會視為「目前已安裝」。

### 解除安裝偵測

因為 `sync` 會回報每個根目錄的完整集合：

- 如果某個技能在下一次同步時從根目錄消失，我們會將它標記為已從該根目錄移除。
- 如果該技能已從你的所有根目錄移除，它就不再計入 `installsCurrent`。
- 除非你刪除遙測（見下方），否則 `installsAllTime` 永遠不會減少。

### 過期（120 天）

若根目錄有 **120 天**未回報遙測，會被標記為過期，其安裝項目也會停止計入 `installsCurrent`。
這會採延遲評估（在下一次遙測回報時）以避免背景作業。

## 透明度 + 使用者控制

ClawHub 會在你自己的個人檔案中提供私人的「已安裝」分頁：

- 顯示我們儲存的確切根目錄 + 已安裝技能。
- 包含 **JSON 匯出**檢視。
- 包含**刪除遙測資料**動作，可移除你帳戶的所有已儲存遙測。

其他人只會看到**彙總安裝計數器**；沒有其他人能看到你的根目錄/資料夾。

刪除你的帳戶也會刪除你的遙測資料。

## 如何停用遙測

設定環境變數：

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

設定後，CLI 在執行 `clawhub sync` 時不會傳送遤測。
