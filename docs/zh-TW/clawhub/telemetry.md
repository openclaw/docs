# 遙測

ClawHub 使用**最少量的遙測**來計算**安裝計數**（實際正在使用的項目），並提供更好的排序/篩選功能。
這是以 CLI `clawhub sync` 命令為基礎。

## 何時收集遙測

只有在以下情況才會傳送遙測：

- 你已在 CLI 中**登入**（同步/發布流程本來就需要驗證）。
- 你執行 `clawhub sync`。
- 遙測**未被停用**（請參閱下方「如何停用」）。

如果你尚未登入，則不會回報任何內容。

## 我們收集什麼

每次執行 `clawhub sync` 時，CLI 都會回報它找到的項目的**完整快照**，並依掃描根目錄（「資料夾/根目錄」）分組。

針對每個根目錄，我們會儲存：

- `rootId`：標準根目錄路徑的 **SHA-256 雜湊**（伺服器永遠不會看到原始路徑）。
- `label`：從最後兩個路徑區段衍生的人類可讀標籤（家目錄路徑會以 `~` 顯示）。
- `firstSeenAt`、`lastSeenAt`、選用的 `expiredAt`。

針對在根目錄下找到的每個技能，我們會儲存：

- `skillId`（由 slug 解析；只追蹤登錄檔中存在的技能）。
- `firstSeenAt`、`lastSeenAt`。
- `lastVersion`（盡力而為；目前為已知時與登錄檔相符的版本）。
- 當先前回報的安裝項目從根目錄消失時，選用的 `removedAt`。

### 我們_不會_收集什麼

- 不收集原始絕對資料夾路徑（只收集經雜湊的 `rootId` + 簡短顯示標籤）。
- 不收集檔案內容。
- 不收集每次執行的記錄、提示詞或其他 CLI 輸出。
- 不追蹤未上傳至登錄檔的技能（未知的 slug 會被忽略）。

## 安裝計數

我們為每個技能維護兩個計數器：

- `installsCurrent`：目前在至少一個作用中根目錄安裝該技能的唯一使用者。
- `installsAllTime`：曾經回報安裝該技能的唯一使用者。

### 多個根目錄

如果你從多個資料夾同步，我們會獨立處理每個掃描根目錄。只要某個技能存在於**任何**作用中根目錄，就會被視為「目前已安裝」。

### 解除安裝偵測

因為 `sync` 會回報每個根目錄的完整集合：

- 如果某個技能在下一次同步時從根目錄消失，我們會將它標記為該根目錄已移除。
- 如果該技能已從你的所有根目錄移除，它就不再計入 `installsCurrent`。
- 除非你刪除遙測（見下方），否則 `installsAllTime` 永遠不會減少。

### 陳舊狀態（120 天）

**120 天**內未回報遙測的根目錄會被標記為陳舊，其安裝項目也會停止計入 `installsCurrent`。
這會以延遲方式評估（在下一次遙測回報時），以避免背景作業。

## 透明度 + 使用者控制

ClawHub 會在你自己的個人檔案上提供一個私人「已安裝」分頁：

- 顯示我們儲存的確切根目錄 + 已安裝技能。
- 包含 **JSON 匯出**檢視。
- 包含**刪除遙測**操作，可移除你帳戶所有已儲存的遙測。

其他所有人只會看到**彙總後的安裝計數器**；沒有人可以看到你的根目錄/資料夾。

刪除你的帳戶也會刪除你的遙測資料。

## 如何停用遙測

設定環境變數：

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

設定後，CLI 在執行 `clawhub sync` 期間不會傳送遙測。
