# 安全稽核

ClawHub 安全稽核可協助你判斷某個技能或外掛是否足夠安全、可以安裝。它們會顯示某個版本會做什麼、要求哪些權限，以及在它能存取檔案、帳號、憑證、程式碼或外部服務前，是否有任何事項需要額外注意。

稽核是很強的安全訊號，但不保證某個版本完全沒有風險。授予敏感存取權前，請務必自行判斷。

另請參閱[安全性](/zh-TW/clawhub/security)、[可接受使用](/zh-TW/clawhub/acceptable-usage)，以及[審核與帳號安全](/zh-TW/clawhub/moderation)。

## 安裝前要檢查什麼

安裝前，請檢視：

- 整體稽核狀態
- 風險等級
- 任何列出的發現項目
- 必要的憑證、權限或環境變數
- 擁有者、來源、版本、變更記錄、下載次數、星標數，以及其他信任訊號

只安裝你理解且信任的內容。

## 稽核狀態

稽核狀態會告訴你應如何回應稽核結果：

| 狀態        | 含義                                                                     |
| ----------- | ------------------------------------------------------------------------ |
| `Pass`      | 未發現高於低風險的可見問題。                                             |
| `Review`    | 安裝前請閱讀發現項目。該版本仍可能是合法的。                             |
| `Warn`      | 請格外謹慎。ClawHub 發現高影響疑慮或警示訊號。                           |
| `Malicious` | 請勿安裝。                                                               |
| `Pending`   | 稽核尚未完成。                                                           |
| `Error`     | 無法完成稽核。                                                           |

`Pass` 令人安心，但不能取代你自己的判斷。對於能發佈內容、編輯資料、執行命令、讀取檔案或存取正式環境系統的工具，這一點尤其重要。

## 風險等級

風險等級描述影響範圍：如果你依預期使用此版本，它看起來擁有多大的能力。

| 風險等級 | 含義                                                                         |
| -------- | ---------------------------------------------------------------------------- |
| `Low`    | 發現的敏感權限或使用者影響很少。                                             |
| `Medium` | 該版本具有實質權限，例如帳號存取或資料變更。                                 |
| `High`   | 該版本具有高影響權限、嚴重發現項目或惡意訊號。                               |

風險等級與稽核狀態回答的是不同問題：

- 風險等級問的是：「這裡有多大的能力？」
- 稽核狀態問的是：「我應如何處理這個結果？」

例如，某個發佈技能可能顯示 `Review` 且風險為 `Medium`。這不表示它是惡意的。這表示該技能看起來符合用途，但能以有意義的帳號權限採取行動。

## 發現項目

發現項目會說明為什麼顯示某個稽核結果。每個發現項目通常包括：

- 它代表什麼
- 為什麼被標記
- 相關的技能或外掛內容
- 建議

發現項目可能標示為 `Info`、`Low`、`Medium`、`High` 或 `Critical`。嚴重程度越高的發現項目，對風險等級與稽核狀態的影響越大。

低信心的發現項目會從公開稽核彙總中隱藏，讓頁面聚焦於有用的證據。

## ClawHub 會檢查什麼

ClawHub 會稽核提交的版本成品，包括：

- 技能指示或外掛中繼資料
- 宣告的環境變數與權限
- 安裝指示與套件中繼資料
- 內含檔案與檔案清單
- 相容性與能力中繼資料

主要問題是一致性：名稱、摘要、中繼資料、要求的權限與實際內容，是否符合使用者合理預期？

強大的行為不會自動變成壞事。許多有用工具需要憑證、本機命令、供應商 API 或套件安裝。稽核會檢查這些能力是否符合預期、已揭露且相稱。

成品頁面會連結到完整稽核：

```text
/<owner>/skills/<slug>/security-audit
```

稽核頁面結合：

1. SkillSpector
2. VirusTotal
3. 風險分析

## VirusTotal

ClawHub 在稽核堆疊中使用 VirusTotal 作為惡意軟體遙測。VirusTotal 是檔案聲譽與惡意軟體掃描的可信業界標準，而我們的合作關係讓 ClawHub 能將更廣泛的安全情報加入技能與外掛審查。

VirusTotal 對已知惡意成品、引擎命中，以及補充 ClawHub 代理感知審查的聲譽訊號特別有用。當供應商引擎計數可用時，稽核會以白話摘要，例如：

```text
62/62 vendors flagged this skill as clean.
```

或：

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

當 ClawHub 沒有可摘要的供應商計數遙測時，稽核會顯示：

```text
No VirusTotal findings
```

VirusTotal 仍然是遙測。它不會取代 ClawHub 自己的成品感知風險分析。

## 風險分析

風險分析由 ClawHub 自有的安全稽核系統 ClawScan 在內部提供支援。它會將每個版本作為面向代理的成品進行審查：指示、中繼資料、宣告權限、檔案、能力訊號、靜態掃描訊號、SkillSpector 發現項目、VirusTotal 遙測，以及發佈者提供的背景脈絡。靜態掃描訊號是此審查的內部脈絡；它們不是獨立的公開稽核區段，也不是阻止安裝的裁定。

風險分析使用
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
作為檢視風險的框架，例如提示注入、工具濫用、憑證暴露、不安全執行、記憶或上下文污染，以及過度自主性。

ClawScan 不會把看起來可怕的能力自動視為惡意。它會詢問該能力是否已揭露、符合用途，並受到該版本所述使用案例的支持。
