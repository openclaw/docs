# 安全稽核

ClawHub 安全稽核可協助你判斷某項技能或外掛是否足夠安全，適合安裝。稽核會顯示版本的功能、要求取得的權限，以及在它能夠存取檔案、帳號、認證資訊、程式碼或外部服務之前，是否有任何需要特別注意的事項。

稽核是重要的安全指標，但不保證版本毫無風險。授予敏感存取權之前，請務必自行判斷。

另請參閱[安全性](/clawhub/security)、[可接受的使用方式](/zh-TW/clawhub/acceptable-usage)和[內容管理與帳號安全](/clawhub/moderation)。

## 安裝前的檢查事項

安裝前，請檢查：

- 整體稽核狀態
- 風險等級
- 列出的所有發現
- 所需的認證資訊、權限或環境變數
- 擁有者、來源、版本、變更記錄、下載次數、星號數，以及其他信任指標

僅安裝你瞭解並信任的內容。

## 稽核狀態

稽核狀態會告訴你應如何因應稽核結果：

| 狀態      | 意義                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 未發現高於低風險的明顯問題。                                |
| `Review`    | 安裝前請閱讀發現。此版本仍可能是正當的。 |
| `Warn`      | 請格外謹慎。ClawHub 發現了高影響性的疑慮或警示訊號。 |
| `Malicious` | 請勿安裝。                                                           |
| `Pending`   | 稽核尚未完成。                                             |
| `Error`     | 無法完成稽核。                                         |

`Pass`雖然令人安心，但不能取代你自己的判斷。對於能夠發布內容、編輯資料、執行命令、讀取檔案或存取正式環境系統的工具，這點尤其重要。

## 風險等級

風險等級描述影響範圍：若你依預期用途使用此版本，它看來擁有多大的權力。

| 風險等級 | 意義                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | 未發現多少敏感權限或使用者影響。                          |
| `Medium`   | 此版本擁有實質權限，例如帳號存取權或變更資料的能力。 |
| `High`     | 此版本擁有高影響性的權限、嚴重發現或惡意訊號。 |

風險等級和稽核狀態回答的是不同問題：

- 風險等級問的是：“這裡有多大的權力？”
- 稽核狀態問的是：“我應該如何處理這項結果？”

例如，發布技能可能顯示 `Review`，同時具有 `Medium` 風險。這不表示它具有惡意，而是表示此技能的功能看來與用途相符，但能以實質的帳號權限執行操作。

## 發現

發現會說明顯示特定稽核結果的原因。每項發現通常包括：

- 其意義
- 遭到標記的原因
- 相關的技能或外掛內容
- 建議

發現可能標記為 `Info`、`Low`、`Medium`、`High` 或 `Critical`。嚴重性越高的發現，對風險等級與稽核狀態的影響越大。

低可信度的發現不會顯示在公開稽核摘要中，讓頁面聚焦於實用的證據。

## ClawHub 的檢查項目

ClawHub 會稽核提交的版本成品，包括：

- 技能指示或外掛中繼資料
- 宣告的環境變數和權限
- 安裝指示和套件中繼資料
- 包含的檔案和檔案清單
- 相容性和功能中繼資料

主要問題在於一致性：名稱、摘要、中繼資料、要求的權限和實際內容，是否符合使用者的合理預期？

強大的行為不一定是壞事。許多實用工具需要認證資訊、本機命令、供應商 API 或安裝套件。稽核會檢查這些權力是否符合預期、已充分揭露且程度相稱。

成品頁面會連結至完整稽核，網址如下：

```text
/<owner>/skills/<slug>/security-audit
```

稽核頁面結合了：

1. SkillSpector
2. VirusTotal
3. 風險分析

## VirusTotal

ClawHub 在稽核架構中使用 VirusTotal 作為惡意軟體遙測工具。VirusTotal 是檔案信譽評估和惡意軟體掃描領域值得信賴的業界標準，而雙方的合作可讓 ClawHub 在技能和外掛審查中加入更廣泛的安全情報。

VirusTotal 對於已知惡意成品、掃描引擎命中結果，以及補充 ClawHub 代理程式感知審查的信譽訊號尤其有用。如果有供應商引擎的統計數據，稽核會以淺顯易懂的文字加以摘要，例如：

```text
62/62 家供應商將此技能標記為安全。
```

或：

```text
2/64 家供應商將此技能標記為惡意，1/64 家將其標記為可疑，61/64 家將其標記為安全。
```

當 ClawHub 沒有可供摘要的供應商數量遙測資料時，稽核會顯示：

```text
沒有 VirusTotal 發現
```

VirusTotal 仍屬遙測資訊，無法取代 ClawHub 自有的成品感知風險分析。

## 風險分析

風險分析由 ClawHub 自有的安全稽核系統 ClawScan 在內部提供支援。它會將每個版本視為面向代理程式的成品進行審查，包括指示、中繼資料、宣告的權限、檔案、功能訊號、靜態掃描訊號、SkillSpector 發現、VirusTotal 遙測資料，以及發布者提供的背景資訊。靜態掃描訊號是此審查的內部背景資訊；它們不是獨立的公開稽核章節，也不是阻止安裝的判定。

風險分析使用
[OWASP 代理式技能十大風險](https://owasp.org/www-project-agentic-skills-top-10/)
作為評估提示詞注入、工具誤用、認證資訊暴露、不安全執行、記憶或情境污染，以及過度自主性等風險的視角。

ClawScan 不會自動將看似可怕的功能視為惡意。它會檢查該功能是否已揭露、符合用途，並受到版本所述使用案例的支持。
