# 安全性稽核

ClawHub 安全性稽核可協助你判斷某個 skill 或外掛是否足夠安全、可以安裝。它們會顯示某個發行版會做什麼、要求哪些權限，以及在它能存取檔案、帳號、憑證、程式碼或外部服務之前，是否有任何事項需要額外注意。

稽核是強而有力的安全訊號，但不保證某個發行版毫無風險。授予敏感存取權之前，請務必自行判斷。

另請參閱[安全性](/clawhub/security)、[可接受使用](/zh-TW/clawhub/acceptable-usage)和[審核與帳號安全](/clawhub/moderation)。

## 安裝前要檢查什麼

安裝前，請檢閱：

- 整體稽核狀態
- 風險等級
- 任何列出的發現
- 必要的憑證、權限或環境變數
- 擁有者、來源、版本、變更記錄、下載數、星標數和其他信任訊號

只安裝你理解且信任的內容。

## 稽核狀態

稽核狀態會告訴你該如何回應稽核結果：

| 狀態        | 意義                                                                      |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 未發現高於低風險的可見問題。                                              |
| `Review`    | 安裝前請閱讀發現。此發行版仍可能是合法的。                                |
| `Warn`      | 請格外謹慎。ClawHub 發現高影響疑慮或警示訊號。                            |
| `Malicious` | 不要安裝。                                                                |
| `Pending`   | 稽核尚未完成。                                                            |
| `Error`     | 無法完成稽核。                                                            |

`Pass` 令人安心，但不能取代你自己的判斷。對於能夠發布內容、編輯資料、執行命令、讀取檔案或存取生產系統的工具，這一點最重要。

## 風險等級

風險等級描述影響範圍：如果你按預期方式使用，此發行版看起來擁有多少能力。

| 風險等級 | 意義                                                                          |
| -------- | ----------------------------------------------------------------------------- |
| `Low`    | 發現的敏感權限或使用者影響很少。                                              |
| `Medium` | 此發行版具有實質權限，例如帳號存取或資料變更。                                |
| `High`   | 此發行版具有高影響權限、嚴重發現或惡意訊號。                                  |

風險等級和稽核狀態回答的是不同問題：

- 風險等級問的是：「這裡有多少能力？」
- 稽核狀態問的是：「我該如何處理這個結果？」

例如，某個發布 skill 可能顯示為 `Review` 且風險為 `Medium`。這不表示它是惡意的。這表示該 skill 看起來符合目的，但能以實質帳號權限行動。

## 發現

發現會說明為什麼顯示某個稽核結果。每個發現通常包含：

- 它的意思
- 它為什麼被標記
- 相關的 skill 或外掛內容
- 建議

發現可能標示為 `Info`、`Low`、`Medium`、`High` 或 `Critical`。嚴重程度較高的發現，對風險等級和稽核狀態的影響更強。

低信心發現會從公開稽核摘要中隱藏，讓頁面聚焦於有用的證據。

## ClawHub 會檢查什麼

ClawHub 會稽核提交的發行成品，包括：

- skill 指示或外掛中繼資料
- 宣告的環境變數和權限
- 安裝指示和套件中繼資料
- 包含的檔案和檔案清單
- 相容性和能力中繼資料

主要問題是一致性：名稱、摘要、中繼資料、要求的權限和實際內容，是否符合使用者合理期待？

強大的行為不會自動就是壞事。許多實用工具需要憑證、本機命令、提供者 API 或套件安裝。稽核會檢查該能力是否符合預期、已揭露且合乎比例。

成品頁面會連結到完整稽核：

```text
/<owner>/skills/<slug>/security-audit
```

稽核頁面結合：

1. SkillSpector
2. VirusTotal
3. 風險分析

## VirusTotal

ClawHub 在稽核堆疊中使用 VirusTotal 作為惡意軟體遙測。VirusTotal 是檔案聲譽和惡意軟體掃描方面受信任的業界標準，而我們的合作關係讓 ClawHub 能為 skill 和外掛審查加入更廣泛的安全情報。

VirusTotal 對已知惡意成品、引擎命中，以及補充 ClawHub 代理感知審查的聲譽訊號特別有用。當可用供應商引擎計數時，稽核會以白話摘要它們，例如：

```text
62/62 vendors flagged this skill as clean.
```

或：

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

當 ClawHub 沒有可摘要的供應商計數遙測時，稽核會說：

```text
No VirusTotal findings
```

VirusTotal 仍然是遙測。它不會取代 ClawHub 自己的成品感知風險分析。

## 風險分析

風險分析內部由 ClawScan 提供支援，ClawScan 是 ClawHub 自有的安全性稽核系統。它會將每個發行版作為面向代理的成品來審查：指示、中繼資料、宣告的權限、檔案、能力訊號、靜態掃描訊號、SkillSpector 發現、VirusTotal 遙測，以及發布者提供的背景脈絡。靜態掃描訊號是此審查的內部背景脈絡；它們不是獨立的公開稽核區段，也不是阻止安裝的裁定。

風險分析使用 [OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/) 作為檢視風險的視角，例如提示注入、工具誤用、憑證暴露、不安全執行、記憶或脈絡污染，以及過度自主性。

ClawScan 不會把看起來嚇人的能力自動視為惡意。它會詢問該能力是否已揭露、符合目的，並且受到發行版所述使用案例的支持。
