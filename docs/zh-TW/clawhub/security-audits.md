# 安全稽核

ClawHub 安全稽核可協助你判斷某個技能或外掛是否足夠安全、可以安裝。它們會顯示某個發行版本會做什麼、要求哪些權限，以及在它能存取檔案、帳號、憑證、程式碼或外部服務之前，是否有任何需要額外注意的事項。

稽核是很強的安全訊號，但不保證某個發行版本完全沒有風險。授予敏感存取權之前，請一律自行判斷。

另請參閱[安全](/zh-TW/clawhub/security)、[可接受使用](/zh-TW/clawhub/acceptable-usage)，以及[內容審核與帳號安全](/zh-TW/clawhub/moderation)。

## 安裝前要檢查什麼

安裝前，請檢查：

- 整體稽核狀態
- 風險等級
- 任何列出的發現
- 所需憑證、權限或環境變數
- 擁有者、來源、版本、變更記錄、下載次數、星標，以及其他信任訊號

只安裝你理解且信任的內容。

## 稽核狀態

稽核狀態會告訴你應如何回應稽核結果：

| 狀態        | 含義                                                                      |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 未發現高於低風險的可見問題。                                              |
| `Review`    | 安裝前請閱讀發現。該發行版本仍可能是正當的。                              |
| `Warn`      | 請格外謹慎。ClawHub 發現高影響疑慮或警示訊號。                           |
| `Malicious` | 請勿安裝。                                                                |
| `Pending`   | 稽核尚未完成。                                                            |
| `Error`     | 無法完成稽核。                                                            |

`Pass` 令人安心，但不能取代你自己的判斷。對於能發佈內容、編輯資料、執行命令、讀取檔案或存取正式環境系統的工具，這一點最為重要。

## 風險等級

風險等級描述影響範圍：如果你按預期使用該發行版本，它看起來擁有多大的能力。

| 風險等級 | 含義                                                                          |
| -------- | ----------------------------------------------------------------------------- |
| `Low`    | 發現的敏感權限或使用者影響很少。                                              |
| `Medium` | 該發行版本具有有意義的權限，例如帳號存取或資料變更。                          |
| `High`   | 該發行版本具有高影響權限、嚴重發現或惡意訊號。                                |

風險等級和稽核狀態回答的是不同問題：

- 風險等級問的是：「這裡有多大的能力？」
- 稽核狀態問的是：「我該如何處理這個結果？」

例如，某個發佈技能可能顯示 `Review` 並具有 `Medium` 風險。這並不表示它是惡意的。這表示該技能看起來符合用途，但能以有意義的帳號權限執行操作。

## 發現

發現會說明為何顯示某個稽核結果。每個發現通常包含：

- 它代表什麼
- 為何被標記
- 相關的技能或外掛內容
- 建議

發現可能標記為 `Info`、`Low`、`Medium`、`High` 或 `Critical`。嚴重程度較高的發現，對風險等級和稽核狀態的影響更強。

低信心的發現會從公開稽核彙總中隱藏，讓頁面聚焦在有用的證據上。

## ClawHub 會檢查什麼

ClawHub 會稽核提交的發行成品，包括：

- 技能說明或外掛中繼資料
- 宣告的環境變數和權限
- 安裝說明和套件中繼資料
- 包含的檔案和檔案清單
- 相容性和能力中繼資料

主要問題是一致性：名稱、摘要、中繼資料、要求的權限，以及實際內容，是否符合使用者合理預期？

強大的行為不會自動代表不好。許多有用工具都需要憑證、本機命令、供應商 API 或套件安裝。稽核會檢查這些能力是否符合預期、已揭露且相稱。

成品頁面會連結到完整稽核，位置如下：

```text
/<owner>/skills/<slug>/security-audit
```

稽核頁面結合了：

1. SkillSpector
2. VirusTotal
3. 風險分析

## VirusTotal

ClawHub 在稽核堆疊中使用 VirusTotal 作為惡意軟體遙測。VirusTotal 是檔案信譽和惡意軟體掃描方面受信任的業界標準，而我們的合作關係讓 ClawHub 能將更廣泛的安全情報加入技能和外掛審查。

VirusTotal 對於已知惡意成品、引擎命中，以及補足 ClawHub 具備代理情境感知審查的信譽訊號特別有用。當供應商引擎計數可用時，稽核會以白話摘要，例如：

```text
62/62 vendors flagged this skill as clean.
```

或：

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

當 ClawHub 沒有可供摘要的供應商計數遙測時，稽核會顯示：

```text
No VirusTotal findings
```

VirusTotal 仍然只是遙測。它不會取代 ClawHub 自身具備成品情境感知的風險分析。

## 風險分析

風險分析由 ClawHub 自有的安全稽核系統 ClawScan 在內部提供支援。它會將每個發行版本作為面向代理的成品來審查：說明、中繼資料、宣告的權限、檔案、能力訊號、靜態掃描訊號、SkillSpector 發現、VirusTotal 遙測，以及發佈者提供的情境。靜態掃描訊號是這項審查的內部情境；它們不是獨立的公開稽核章節，也不是阻擋安裝的裁定。

風險分析使用 [OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/) 作為檢視風險的角度，例如提示注入、工具誤用、憑證暴露、不安全執行、記憶或情境污染，以及過度自主性。

ClawScan 不會把看起來嚇人的能力自動視為惡意。它會詢問該能力是否已揭露、符合用途，並由發行版本聲明的使用情境支撐。
