# 安全稽核

ClawHub 安全稽核可協助你判斷某個 skill 或外掛是否足夠安全，可以安裝。它們會顯示某個版本會做什麼、要求哪些權限，以及在它能存取檔案、帳號、憑證、程式碼或外部服務之前，是否有任何事項需要額外注意。

稽核是強力的安全訊號，但不保證某個版本毫無風險。在授予敏感存取權之前，請一律自行判斷。

另請參閱[安全性](/clawhub/security)、[可接受使用方式](/zh-TW/clawhub/acceptable-usage)與[審核與帳號安全](/clawhub/moderation)。

## 安裝前應檢查的事項

安裝前，請檢視：

- 整體稽核狀態
- 風險等級
- 任何列出的發現
- 必要的憑證、權限或環境變數
- 擁有者、來源、版本、變更日誌、下載次數、星標數，以及其他信任訊號

只安裝你理解且信任的內容。

## 稽核狀態

稽核狀態會告訴你應如何回應稽核結果：

| 狀態        | 含義                                                                       |
| ----------- | -------------------------------------------------------------------------- |
| `Pass`      | 未發現高於低風險的可見問題。                                               |
| `Review`    | 安裝前請閱讀發現。該版本仍可能是正當的。                                   |
| `Warn`      | 請格外謹慎。ClawHub 發現高影響疑慮或警告訊號。                             |
| `Malicious` | 不要安裝。                                                                 |
| `Pending`   | 稽核尚未完成。                                                             |
| `Error`     | 無法完成稽核。                                                             |

`Pass` 令人安心，但不能取代你自己的判斷。這對於能發布內容、編輯資料、執行命令、讀取檔案或存取正式環境系統的工具尤其重要。

## 風險等級

風險等級描述影響範圍：若你依預期方式使用該版本，它看起來擁有多少權力。

| 風險等級 | 含義                                                                         |
| -------- | ---------------------------------------------------------------------------- |
| `Low`    | 未發現太多敏感權限或使用者影響。                                             |
| `Medium` | 該版本具有有意義的權限，例如帳號存取或資料變更。                             |
| `High`   | 該版本具有高影響權限、嚴重發現或惡意訊號。                                   |

風險等級與稽核狀態回答的是不同問題：

- 風險等級問的是：「這裡有多少權力？」
- 稽核狀態問的是：「我應如何處理這個結果？」

例如，某個發布用 skill 可能顯示 `Review` 且風險為 `Medium`。這並不表示它是惡意的。這表示該 skill 看起來符合用途，但能以有意義的帳號權限行動。

## 發現

發現會說明為什麼顯示某個稽核結果。每個發現通常包含：

- 它代表什麼
- 為什麼被標記
- 相關的 skill 或外掛內容
- 建議

發現可能標記為 `Info`、`Low`、`Medium`、`High` 或 `Critical`。嚴重程度越高的發現，對風險等級與稽核狀態的影響越大。

低信心的發現會從公開稽核彙總中隱藏，讓頁面聚焦於有用的證據。

## ClawHub 檢查的內容

ClawHub 會稽核提交的版本成品，包括：

- skill 指令或外掛中繼資料
- 宣告的環境變數與權限
- 安裝指令與套件中繼資料
- 內含檔案與檔案清單
- 相容性與功能中繼資料

主要問題是一致性：名稱、摘要、中繼資料、要求的權限與實際內容，是否符合使用者合理預期？

強大的行為不會自動代表不好。許多實用工具需要憑證、本機命令、供應商 API 或套件安裝。稽核會檢查這些權力是否符合預期、是否已揭露，且是否相稱。

成品頁面會連結到完整稽核：

```text
/<owner>/skills/<slug>/security-audit
```

稽核頁面結合：

1. SkillSpector
2. VirusTotal
3. 風險分析

## VirusTotal

ClawHub 在稽核堆疊中使用 VirusTotal 作為惡意軟體遙測。VirusTotal 是檔案聲譽與惡意軟體掃描的可信業界標準，而我們的合作夥伴關係讓 ClawHub 能為 skill 與外掛審查加入更廣泛的安全情報。

VirusTotal 對於已知惡意成品、引擎命中，以及可補足 ClawHub 具備 agent 感知能力審查的聲譽訊號特別有用。當供應商引擎計數可用時，稽核會以白話摘要呈現，例如：

```text
62/62 vendors flagged this skill as clean.
```

或：

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

當 ClawHub 沒有可摘要的供應商計數遙測時，稽核會顯示：

```text
No VirusTotal findings
```

VirusTotal 仍然是遙測。它不能取代 ClawHub 自己具備成品感知能力的風險分析。

## 風險分析

風險分析由 ClawHub 自有的安全稽核系統 ClawScan 在內部提供支援。它會將每個版本作為面向 agent 的成品來審查：指令、中繼資料、宣告的權限、檔案、功能訊號、靜態掃描訊號、SkillSpector 發現、VirusTotal 遙測，以及發布者提供的脈絡。靜態掃描訊號是此審查的內部脈絡；它們不是獨立的公開稽核章節，也不是阻止安裝的判定。

風險分析使用
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
作為檢視風險的視角，例如提示注入、工具誤用、憑證暴露、不安全執行、記憶或脈絡污染，以及過度自主性。

ClawScan 不會把看起來嚇人的功能自動視為惡意。它會詢問該功能是否已揭露、是否符合目的，並且是否受到該版本所述使用案例的支持。
