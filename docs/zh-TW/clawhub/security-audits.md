# 安全性稽核

ClawHub 安全性稽核可協助你判斷 Skill 或外掛是否安全到足以
安裝。稽核會顯示某個版本的作用、它要求哪些權限，以及在它能夠存取檔案、帳戶、
認證資訊、程式碼或外部服務之前，是否有任何事項值得格外留意。

稽核是重要的安全性指標，但不保證版本
毫無風險。授予敏感存取權之前，務必自行審慎判斷。

另請參閱[安全性](/clawhub/security)、[可接受的使用方式](/clawhub/acceptable-usage)
及[內容審核與帳戶安全](/clawhub/moderation)。

## 安裝前應檢查的事項

安裝前，請檢查：

- 整體稽核狀態
- 風險等級
- 列出的任何發現項目
- 所需的認證資訊、權限或環境變數
- 擁有者、來源、版本、變更記錄、下載次數、星號數及其他信任指標

只安裝你瞭解且信任的內容。

## 稽核狀態

稽核狀態會告訴你應如何因應稽核結果：

| 狀態      | 含義                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 未發現高於低風險的明顯問題。                                |
| `Review`    | 安裝前請閱讀發現項目。此版本仍可能是正當的。 |
| `Warn`      | 請格外謹慎。ClawHub 發現高影響性的疑慮或警示訊號。 |
| `Malicious` | 請勿安裝。                                                           |
| `Pending`   | 稽核尚未完成。                                             |
| `Error`     | 無法完成稽核。                                         |

`Pass` 令人安心，但不能取代你自己的判斷。這對能夠發佈內容、編輯資料、執行命令、讀取檔案或
存取正式環境系統的工具尤其重要。

## 風險等級

風險等級描述影響範圍：如果你依預期用途使用此版本，它看起來擁有
多大的權限。

| 風險等級 | 含義                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | 未發現多少敏感權限或使用者影響。                          |
| `Medium`   | 此版本擁有實質權限，例如帳戶存取權或變更資料的能力。 |
| `High`     | 此版本擁有高影響性權限、嚴重發現項目或惡意訊號。 |

風險等級和稽核狀態回答的是不同問題：

- 風險等級問的是：“這裡有多大的權限？”
- 稽核狀態問的是：“我該如何處理這項結果？”

例如，發佈用的 Skill 可能顯示 `Review`，風險為 `Medium`。這並不
表示它具有惡意，而是表示該 Skill 看起來符合其用途，但能夠
運用實質的帳戶權限。

## 發現項目

發現項目會說明顯示某項稽核結果的原因。每個發現項目通常包括：

- 其含義
- 標記原因
- 相關的 Skill 或外掛內容
- 建議

發現項目可能標示為 `Info`、`Low`、`Medium`、`High` 或 `Critical`。嚴重程度越高的
發現項目，對風險等級和稽核狀態的影響越大。

低信心度的發現項目不會顯示於公開稽核摘要中，讓頁面
聚焦於有用的證據。

## ClawHub 檢查的內容

ClawHub 會稽核提交的版本成品，包括：

- Skill 指示或外掛中繼資料
- 宣告的環境變數和權限
- 安裝說明和套件中繼資料
- 內含檔案和檔案清單
- 相容性和功能中繼資料

主要問題在於一致性：名稱、摘要、中繼資料、要求的
權限和實際內容，是否符合使用者的合理預期？

強大的行為並不一定有害。許多實用工具需要認證資訊、
本機命令、供應商 API 或套件安裝。稽核會檢查這些
權限是否符合預期、已揭露且合乎比例。

成品頁面會連結至完整稽核：

```text
/<owner>/skills/<slug>/security-audit
```

稽核頁面整合了：

1. SkillSpector
2. VirusTotal
3. 風險分析

## VirusTotal

ClawHub 在稽核堆疊中使用 VirusTotal 作為惡意軟體遙測來源。VirusTotal 是
檔案信譽與惡意軟體掃描領域中值得信賴的業界標準，而雙方的
合作讓 ClawHub 能為 Skill 與外掛審查加入更廣泛的安全性情報。

VirusTotal 對已知惡意成品、引擎命中結果，以及
可補充 ClawHub 代理程式感知審查的信譽訊號特別有用。當供應商
引擎計數可用時，稽核會以簡明文字摘要，例如：

```text
62/62 家供應商將此 Skill 標示為安全。
```

或：

```text
2/64 家供應商將此 Skill 標示為惡意，1/64 標示為可疑，61/64 標示為安全。
```

當 ClawHub 沒有可供摘要的供應商計數遙測資料時，稽核會顯示：

```text
沒有 VirusTotal 發現項目
```

VirusTotal 仍只是遙測資料，無法取代 ClawHub 自身具成品感知能力的
風險分析。

## 風險分析

風險分析由 ClawHub 自有的安全性稽核系統 ClawScan 在內部提供支援。
它會將每個版本視為面向代理程式的成品進行審查，包括：指示、
中繼資料、宣告的權限、檔案、功能訊號、靜態掃描訊號、
SkillSpector 發現項目、VirusTotal 遙測資料，以及發佈者提供的背景資訊。
靜態掃描訊號是此審查的內部背景資訊；它們並非
獨立的公開稽核區段，也不是阻止安裝的判定結果。

風險分析使用
[OWASP 代理式 Skills 十大風險](https://owasp.org/www-project-agentic-skills-top-10/)
作為檢視提示詞注入、工具誤用、認證資訊暴露、
不安全執行、記憶或情境污染，以及過度自主性等風險的框架。

ClawScan 不會自動將看似危險的功能視為惡意。
它會判斷該功能是否已揭露、符合用途，且有此版本所述的使用案例
作為依據。
