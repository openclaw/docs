# 安全稽核

ClawHub 安全稽核可協助你判斷某項 skill 或外掛是否足夠安全，適合安裝。稽核會顯示某個版本的作用、它要求的權限，以及在它能夠存取檔案、帳號、認證資訊、程式碼或外部服務之前，是否有任何需要額外留意之處。

稽核是強而有力的安全指標，但不保證版本毫無風險。授予敏感存取權之前，請務必自行判斷。

另請參閱[安全性](/clawhub/security)、[可接受的使用方式](/zh-TW/clawhub/acceptable-usage)及[內容審核與帳號安全](/clawhub/moderation)。

## 安裝前應檢查的事項

安裝前，請檢視：

- 整體稽核狀態
- 風險等級
- 列出的所有發現
- 所需的認證資訊、權限或環境變數
- 擁有者、來源、版本、變更記錄、下載次數、星號數及其他信任指標

只安裝你了解且信任的內容。

## 稽核狀態

稽核狀態會告訴你應如何因應稽核結果：

| 狀態        | 意義                                                                 |
| ----------- | -------------------------------------------------------------------- |
| `Pass`      | 未發現明顯高於低風險的問題。                                         |
| `Review`    | 安裝前請閱讀相關發現。此版本仍可能是正當的。                         |
| `Warn`      | 請格外謹慎。ClawHub 發現了影響重大的疑慮或警告訊號。                 |
| `Malicious` | 請勿安裝。                                                           |
| `Pending`   | 稽核尚未完成。                                                       |
| `Error`     | 無法完成稽核。                                                       |

`Pass` 能讓人較為放心，但不能取代你自己的判斷。對於能夠發布內容、編輯資料、執行命令、讀取檔案或存取正式環境系統的工具，這一點尤其重要。

## 風險等級

風險等級描述影響範圍：如果你按預期用途使用此版本，它看起來擁有多大的權限。

| 風險等級 | 意義                                                                 |
| -------- | -------------------------------------------------------------------- |
| `Low`    | 未發現多少敏感權限或對使用者的影響。                                 |
| `Medium` | 此版本擁有實質權限，例如帳號存取權或變更資料的能力。                 |
| `High`   | 此版本擁有影響重大的權限、嚴重發現或惡意訊號。                       |

風險等級與稽核狀態回答的是不同問題：

- 風險等級問的是：“這裡有多大的權限？”
- 稽核狀態問的是：“我應如何處理這項結果？”

例如，發布用的 skill 可能顯示 `Review`，風險為 `Medium`。這並不表示它具有惡意，而是表示此 skill 看起來符合其用途，但能以實質的帳號權限執行操作。

## 發現

發現會說明顯示某項稽核結果的原因。每項發現通常包括：

- 它代表什麼
- 為何遭到標記
- 相關的 skill 或外掛內容
- 建議

發現可能標示為 `Info`、`Low`、`Medium`、`High` 或 `Critical`。嚴重性較高的發現，對風險等級與稽核狀態的影響較大。

信心度較低的發現不會顯示在公開稽核摘要中，讓頁面能聚焦於有用的證據。

## ClawHub 檢查的內容

ClawHub 會稽核提交的版本成品，包括：

- skill 指示或外掛中繼資料
- 宣告的環境變數與權限
- 安裝指示與套件中繼資料
- 內含的檔案與檔案清單
- 相容性與功能中繼資料

主要問題在於一致性：名稱、摘要、中繼資料、要求的權限與實際內容，是否符合使用者的合理預期？

強大的行為不一定是壞事。許多實用工具需要認證資訊、本機命令、供應商 API 或安裝套件。稽核會檢查這些權限是否符合預期、已充分揭露且合乎比例。

成品頁面會連結至以下位置的完整稽核：

```text
/<owner>/skills/<slug>/security-audit
```

稽核頁面整合了：

1. SkillSpector
2. VirusTotal
3. 風險分析

## VirusTotal

ClawHub 在稽核架構中使用 VirusTotal 作為惡意軟體遙測資料。VirusTotal 是檔案信譽與惡意軟體掃描方面值得信賴的業界標準，而雙方的合作讓 ClawHub 能在審查 skill 與外掛時納入更廣泛的安全情報。

VirusTotal 對於已知的惡意成品、引擎偵測結果及信譽訊號尤其有用，能補充 ClawHub 針對代理程式特性的審查。當有供應商引擎的數量資料可用時，稽核會以淺白文字加以摘要，例如：

```text
62/62 家供應商將此 skill 標記為安全。
```

或：

```text
2/64 家供應商將此 skill 標記為惡意，1/64 家將其標記為可疑，而 61/64 家將其標記為安全。
```

當 ClawHub 沒有可供摘要的供應商數量遙測資料時，稽核會顯示：

```text
沒有 VirusTotal 發現
```

VirusTotal 仍然只是遙測資料，無法取代 ClawHub 自己針對成品特性的風險分析。

## 風險分析

風險分析在內部由 ClawHub 自有的安全稽核系統 ClawScan 提供支援。它會將每個版本視為面向代理程式的成品進行審查，包括指示、中繼資料、宣告的權限、檔案、功能訊號、靜態掃描訊號、SkillSpector 發現、VirusTotal 遙測資料及發布者提供的背景資訊。靜態掃描訊號是此審查的內部背景資訊；它們不是獨立的公開稽核區段，也不是阻止安裝的判定。

風險分析使用
[OWASP 代理式 Skills 十大風險](https://owasp.org/www-project-agentic-skills-top-10/)
作為檢視提示注入、工具誤用、認證資訊暴露、不安全執行、記憶或情境毒化及過度自主性等風險的框架。

ClawScan 不會自動將看似危險的功能視為惡意。它會檢查該功能是否已揭露、符合用途，並且受到此版本所述使用案例的支持。
