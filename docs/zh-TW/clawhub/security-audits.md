# 安全稽核

ClawHub 安全稽核可協助你判斷某個技能或外掛是否足夠安全，可以安裝。它們會顯示某個版本會做什麼、要求哪些權限，以及在它能存取檔案、帳號、憑證、程式碼或外部服務之前，是否有任何事項值得額外注意。

稽核是強力的安全訊號，但不保證某個版本完全沒有風險。授予敏感存取權前，請務必自行判斷。

另請參閱[安全性](/clawhub/security)、[可接受使用方式](/zh-TW/clawhub/acceptable-usage)和[審核與帳號安全](/clawhub/moderation)。

## 安裝前要檢查什麼

安裝前，請檢閱：

- 整體稽核狀態
- 風險等級
- 任何列出的發現
- 必要的憑證、權限或環境變數
- 擁有者、來源、版本、變更記錄、下載數、星標數，以及其他信任訊號

只安裝你理解且信任的內容。

## 稽核狀態

稽核狀態會告訴你應如何回應稽核結果：

| 狀態        | 含義                                                                      |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 未發現高於低風險的可見問題。                                              |
| `Review`    | 安裝前請閱讀發現。此版本仍可能是正當的。                                  |
| `Warn`      | 請格外謹慎。ClawHub 發現了高影響疑慮或警告訊號。                         |
| `Malicious` | 請勿安裝。                                                                |
| `Pending`   | 稽核尚未完成。                                                            |
| `Error`     | 無法完成稽核。                                                            |

`Pass` 令人安心，但它無法取代你自己的判斷。對於可發布內容、編輯資料、執行命令、讀取檔案或存取生產系統的工具，這點尤其重要。

## 風險等級

風險等級描述影響範圍：如果你按預期使用此版本，它看起來擁有多大的權力。

| 風險等級 | 含義                                                                          |
| -------- | ----------------------------------------------------------------------------- |
| `Low`    | 幾乎沒有發現敏感權限或使用者影響。                                             |
| `Medium` | 此版本具有有意義的權限，例如帳號存取或資料變更。                              |
| `High`   | 此版本具有高影響權限、嚴重發現或惡意訊號。                                    |

風險等級和稽核狀態回答不同問題：

- 風險等級問的是：「這裡有多大的權力？」
- 稽核狀態問的是：「我應該如何處理這個結果？」

例如，發布技能可能會顯示 `Review`，且風險為 `Medium`。這不代表它是惡意的。這表示該技能看起來符合用途，但能以有意義的帳號權限行動。

## 發現

發現會說明為什麼顯示某個稽核結果。每項發現通常包含：

- 它代表什麼
- 為什麼被標記
- 相關技能或外掛內容
- 建議

發現可能標示為 `Info`、`Low`、`Medium`、`High` 或 `Critical`。嚴重性較高的發現會更強烈地影響風險等級和稽核狀態。

低信心發現會從公開稽核彙總中隱藏，讓頁面聚焦於有用證據。

## ClawHub 會檢查什麼

ClawHub 會稽核提交的版本成品，包括：

- 技能指示或外掛中繼資料
- 宣告的環境變數和權限
- 安裝指示和套件中繼資料
- 內含檔案和檔案清單
- 相容性和能力中繼資料

主要問題是一致性：名稱、摘要、中繼資料、要求的權限和實際內容，是否符合使用者可合理預期的內容？

強大的行為並不會自動變成壞事。許多有用的工具需要憑證、本機命令、提供者 API 或套件安裝。稽核會檢查這些權力是否符合預期、是否已揭露，以及是否合乎比例。

成品頁面會連結到完整稽核，位於：

```text
/<owner>/skills/<slug>/security-audit
```

稽核頁面結合：

1. SkillSpector
2. VirusTotal
3. 風險分析

## VirusTotal

ClawHub 在稽核堆疊中使用 VirusTotal 作為惡意軟體遙測。VirusTotal 是檔案聲譽和惡意軟體掃描的可信業界標準，而我們的合作關係讓 ClawHub 能將更廣泛的安全情報加入技能和外掛審查。

VirusTotal 對已知惡意成品、引擎命中，以及補充 ClawHub 代理感知審查的聲譽訊號特別有用。當供應商引擎計數可用時，稽核會以白話摘要它們，例如：

```text
62/62 vendors flagged this skill as clean.
```

或：

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

當 ClawHub 沒有可摘要的供應商計數遙測時，稽核會顯示：

```text
No VirusTotal findings
```

VirusTotal 仍然是遙測。它不會取代 ClawHub 自己的成品感知風險分析。

## 風險分析

風險分析在內部由 ClawScan 驅動，這是 ClawHub 自有的安全稽核系統。它會將每個版本作為面向代理的成品進行審查：指示、中繼資料、宣告的權限、檔案、能力訊號、靜態掃描訊號、SkillSpector 發現、VirusTotal 遙測，以及發布者提供的情境。靜態掃描訊號是此審查的內部情境；它們不是獨立的公開稽核章節，也不是阻止安裝的判定。

風險分析使用
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
作為檢視風險的視角，例如提示注入、工具誤用、憑證暴露、不安全執行、記憶或情境污染，以及過度代理權限。

ClawScan 不會將看起來可怕的能力視為自動具有惡意。它會詢問該能力是否已揭露、是否符合用途，以及是否受到版本所述使用案例的支持。
