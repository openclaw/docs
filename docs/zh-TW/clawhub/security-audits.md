# 安全性稽核

ClawHub 安全性稽核可協助你判斷某項 Skill 或外掛是否足夠安全，適合安裝。稽核會顯示某個版本會執行哪些操作、要求哪些權限，以及在它能夠存取檔案、帳號、認證資訊、程式碼或外部服務之前，是否有任何需要額外留意之處。

稽核是重要的安全指標，但不保證版本完全沒有風險。授予敏感存取權之前，請務必自行判斷。

另請參閱[安全性](/clawhub/security)、[可接受的使用方式](/zh-TW/clawhub/acceptable-usage)和[內容審核與帳號安全](/clawhub/moderation)。

## 安裝前應檢查的項目

安裝前，請檢查：

- 整體稽核狀態
- 風險等級
- 列出的任何發現
- 必要的認證資訊、權限或環境變數
- 擁有者、來源、版本、變更記錄、下載次數、星號數及其他信任指標

只安裝你瞭解且信任的內容。

## 稽核狀態

稽核狀態會告訴你應如何因應稽核結果：

| 狀態        | 意義                                                             |
| ----------- | ---------------------------------------------------------------- |
| `Pass`      | 未發現明顯高於低風險的問題。                                     |
| `Review`    | 安裝前請閱讀發現。此版本仍可能是正當的。                           |
| `Warn`      | 請格外謹慎。ClawHub 發現了高影響疑慮或警告訊號。                   |
| `Malicious` | 請勿安裝。                                                       |
| `Pending`   | 稽核尚未完成。                                                   |
| `Error`     | 無法完成稽核。                                                   |

`Pass` 能讓人較為放心，但不能取代你自己的判斷。對於可發布內容、編輯資料、執行命令、讀取檔案或存取正式環境系統的工具，這點尤其重要。

## 風險等級

風險等級描述影響範圍：依照預期方式使用此版本時，它看起來擁有多大的權力。

| 風險等級 | 意義                                                               |
| -------- | ------------------------------------------------------------------ |
| `Low`    | 未發現多少敏感權限或使用者影響。                                   |
| `Medium` | 此版本擁有實質權限，例如帳號存取或資料變更。                       |
| `High`   | 此版本擁有高影響權限、嚴重發現或惡意訊號。                         |

風險等級和稽核狀態回答的是不同問題：

- 風險等級問的是：“這裡有多大的權力？”
- 稽核狀態問的是：“我應如何處理這個結果？”

例如，某項發布用 Skill 可能顯示 `Review`，風險為 `Medium`。這不代表它具有惡意，而是表示該 Skill 看起來符合其用途，但能以實質帳號權限執行操作。

## 發現

發現會說明為何顯示此稽核結果。每項發現通常包括：

- 其意義
- 被標記的原因
- 相關的 Skill 或外掛內容
- 建議

發現可能標示為 `Info`、`Low`、`Medium`、`High` 或 `Critical`。嚴重程度越高的發現，對風險等級和稽核狀態的影響越大。

低可信度的發現不會顯示於公開稽核摘要中，讓頁面聚焦於有用的證據。

## ClawHub 的檢查項目

ClawHub 會稽核提交的版本成品，包括：

- Skill 指示或外掛中繼資料
- 宣告的環境變數和權限
- 安裝指示和套件中繼資料
- 內含檔案和檔案清單
- 相容性和能力中繼資料

主要問題在於一致性：名稱、摘要、中繼資料、要求的權限和實際內容，是否符合使用者的合理預期？

強大的行為不一定有害。許多實用工具需要認證資訊、本機命令、供應商 API 或安裝套件。稽核會檢查這些權力是否符合預期、已充分揭露且合乎比例。

成品頁面會連結至完整稽核，路徑為：

```text
/<owner>/skills/<slug>/security-audit
```

稽核頁面會整合：

1. SkillSpector
2. VirusTotal
3. 風險分析

## VirusTotal

ClawHub 在稽核架構中使用 VirusTotal 作為惡意軟體遙測。VirusTotal 是檔案信譽和惡意軟體掃描領域中值得信賴的業界標準；透過雙方合作，ClawHub 能在 Skill 和外掛審查中加入更廣泛的安全情報。

VirusTotal 對已知惡意成品、掃描引擎命中結果和信譽訊號尤其有用，可補充 ClawHub 考量代理程式特性的審查。若有供應商引擎數量可用，稽核會以淺白文字摘要呈現，例如：

```text
62/62 家供應商將此 Skill 標記為安全。
```

或：

```text
2/64 家供應商將此 Skill 標記為惡意，1/64 家標記為可疑，61/64 家標記為安全。
```

當 ClawHub 沒有可供摘要的供應商數量遙測資料時，稽核會顯示：

```text
沒有 VirusTotal 發現
```

VirusTotal 仍屬於遙測，不能取代 ClawHub 自身針對成品進行的風險分析。

## 風險分析

風險分析由 ClawHub 自有的安全性稽核系統 ClawScan 在內部提供支援。它會將每個版本視為面向代理程式的成品進行審查，包括：指示、中繼資料、宣告的權限、檔案、能力訊號、靜態掃描訊號、SkillSpector 發現、VirusTotal 遙測，以及發布者提供的背景資訊。靜態掃描訊號是此審查的內部背景資訊；它們不是獨立的公開稽核區段，也不是阻止安裝的判定。

風險分析以
[OWASP 代理式 Skills 十大風險](https://owasp.org/www-project-agentic-skills-top-10/)
作為評估提示注入、工具誤用、認證資訊外洩、不安全執行、記憶或情境污染及代理權過度等風險的分析視角。

ClawScan 不會自動將看似危險的能力視為惡意。它會判斷該能力是否已揭露、符合用途，並且有此版本所述的使用案例作為依據。
