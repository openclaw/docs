# 安全稽核

ClawHub 安全稽核可協助你判斷某個 Skills 或外掛是否足夠安全，適合安裝。稽核會顯示版本的功能、所要求的權限，以及在其存取檔案、帳戶、憑證、程式碼或外部服務之前，是否有任何值得額外留意的事項。

稽核是重要的安全指標，但不保證版本毫無風險。授予敏感存取權之前，請務必自行審慎判斷。

另請參閱[安全性](/clawhub/security)、[可接受的使用方式](/clawhub/acceptable-usage)及[內容審核與帳戶安全](/clawhub/moderation)。

## 安裝前應檢查的項目

安裝前，請檢視：

- 整體稽核狀態
- 風險等級
- 所有列出的發現
- 所需的憑證、權限或環境變數
- 擁有者、來源、版本、變更記錄、下載次數、星號及其他信任指標

只安裝你瞭解且信任的內容。

## 稽核狀態

稽核狀態會告訴你應如何因應稽核結果：

| 狀態        | 意義                                                         |
| ----------- | ------------------------------------------------------------ |
| `Pass`      | 未發現明顯超過低風險的問題。                                 |
| `Review`    | 安裝前請閱讀發現。此版本仍可能是正當的。                     |
| `Warn`      | 請格外謹慎。ClawHub 發現了高影響疑慮或警告訊號。             |
| `Malicious` | 請勿安裝。                                                   |
| `Pending`   | 稽核尚未完成。                                               |
| `Error`     | 無法完成稽核。                                               |

`Pass` 能讓人較為放心，但不能取代你自己的判斷。對於能夠發布內容、編輯資料、執行命令、讀取檔案或存取正式環境系統的工具，這點尤其重要。

## 風險等級

風險等級描述影響範圍：依預期方式使用此版本時，它看起來擁有多大的權力。

| 風險等級 | 意義                                                               |
| -------- | ------------------------------------------------------------------ |
| `Low`    | 未發現多少敏感權限或對使用者的影響。                               |
| `Medium` | 此版本擁有實質權限，例如存取帳戶或變更資料。                       |
| `High`   | 此版本擁有高影響權限、嚴重發現或惡意訊號。                         |

風險等級與稽核狀態回答的是不同問題：

- 風險等級問的是：「這裡有多大的權力？」
- 稽核狀態問的是：「我應如何處理這個結果？」

例如，發布型 Skills 可能顯示 `Review`，風險為 `Medium`。這並不代表它具有惡意，而是表示該 Skills 看似符合其用途，但能以實質的帳戶權限執行操作。

## 發現

發現會說明顯示該稽核結果的原因。每項發現通常包括：

- 其意義
- 被標記的原因
- 相關的 Skills 或外掛內容
- 建議

發現可能標示為 `Info`、`Low`、`Medium`、`High` 或 `Critical`。嚴重性越高的發現，對風險等級與稽核狀態的影響越大。

低可信度的發現不會顯示在公開的稽核摘要中，讓頁面聚焦於有用的證據。

## ClawHub 檢查的內容

ClawHub 會稽核提交的版本成品，包括：

- Skills 指示或外掛中繼資料
- 宣告的環境變數與權限
- 安裝指示與套件中繼資料
- 所包含的檔案與檔案清單
- 相容性與能力中繼資料

主要問題在於一致性：名稱、摘要、中繼資料、所要求的權限及實際內容，是否符合使用者的合理預期？

強大的行為不一定是壞事。許多實用工具都需要憑證、本機命令、供應商 API 或安裝套件。稽核會檢查這些權力是否符合預期、已充分揭露且合乎比例。

成品頁面會連結至完整稽核：

```text
/<owner>/skills/<slug>/security-audit
```

稽核頁面整合：

1. SkillSpector
2. VirusTotal
3. 風險分析

## VirusTotal

ClawHub 在稽核技術架構中使用 VirusTotal 作為惡意軟體遙測資訊。VirusTotal 是檔案信譽與惡意軟體掃描領域廣受信任的業界標準，而雙方的合作讓 ClawHub 能在 Skills 與外掛審查中加入更廣泛的安全情報。

VirusTotal 對於已知的惡意成品、掃描引擎命中結果及信譽訊號尤其有用，可補充 ClawHub 針對代理程式特性的審查。若有供應商掃描引擎的統計資料，稽核會以淺白文字摘要，例如：

```text
62/62 vendors flagged this skill as clean.
```

或：

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

當 ClawHub 沒有可供摘要的供應商統計遙測資訊時，稽核會顯示：

```text
No VirusTotal findings
```

VirusTotal 仍屬於遙測資訊，無法取代 ClawHub 本身針對成品的風險分析。

## 風險分析

風險分析由 ClawHub 自有的安全稽核系統 ClawScan 在內部提供支援。它會將每個版本視為供代理程式使用的成品進行審查，涵蓋指示、中繼資料、宣告的權限、檔案、能力訊號、靜態掃描訊號、SkillSpector 發現、VirusTotal 遙測資訊，以及發布者提供的背景資訊。靜態掃描訊號是這項審查的內部背景資訊，並非獨立的公開稽核區段，也不是阻止安裝的判定。

風險分析以 [OWASP 代理式 Skills 十大風險](https://owasp.org/www-project-agentic-skills-top-10/)為檢視框架，用於評估提示詞注入、工具濫用、憑證暴露、不安全執行、記憶或上下文污染，以及過度自主權等風險。

ClawScan 不會將看似危險的能力自動視為惡意。它會判斷該能力是否已揭露、符合用途，並且有版本所述的使用情境作為依據。
