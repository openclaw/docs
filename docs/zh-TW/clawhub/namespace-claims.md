# 組織與命名空間宣告

ClawHub 使用擁有者識別名稱、組織識別名稱、技能代稱、外掛套件名稱，以及
套件作用域作為公開命名空間。如果某個命名空間看起來屬於
現實世界的專案、品牌、套件生態系或組織，但在 ClawHub 上已被
宣告、保留、具誤導性或存在爭議，請要求工作人員透過
[組織／命名空間宣告議題表單](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)進行審查。

請將此路徑用於公開、非敏感的所有權審查。請勿將產品內
回報或帳戶申訴表單用於命名空間宣告。

## 何時開啟宣告

當你認為 ClawHub 工作人員應審查某個命名空間是否應因現實世界的所有權而
被保留、轉讓、重新命名、隱藏、隔離、建立別名，
或以其他方式變更時，請開啟命名空間宣告。

範例包括：

- 與你的 GitHub 組織、專案、公司或社群相符的組織識別名稱
- 像 `@example-org/*` 這樣的套件作用域，應該只由相符的
  ClawHub 擁有者發布
- 看起來在冒充某個專案的技能代稱或外掛套件名稱
- 品牌、商標、專案重新命名或套件歷史爭議
- 已刪除、未活動或無法聯絡的擁有者，阻擋了合法命名空間
  擁有者

如果該列表除了所有權爭議之外還不安全、惡意或具誤導性，
也請遵循相關的管理或安全指引。命名空間宣告
表單用於所有權審查，而非緊急弱點揭露。

## 提交前

請先確認你是使用與命名空間相符的擁有者進行發布。
對於外掛套件，像 `@example-org/example-plugin` 這樣的作用域名稱必須
以相符的 `example-org` 擁有者發布。

如果你可以管理目前的擁有者，請透過發布、
重新命名、轉讓、隱藏或刪除受影響的資源，直接修正命名空間。當你無法管理目前的擁有者，或需要工作人員解決
爭議時，請使用宣告。

## 要包含的證據

請使用公開、非敏感的證據。有幫助的證明包括：

- GitHub 組織、儲存庫、發布版本或維護者歷史
- 指明該命名空間的官方專案文件
- 網域或官方電子郵件網域證明
- npm、PyPI、crates.io 或其他套件登錄檔的作用域控制權
- 可公開討論的商標、品牌或專案所有權證據
- 原始碼儲存庫歷史、套件歷史或公開重新命名通知
- 指向有爭議的 ClawHub 擁有者、技能、外掛、套件或議題的連結

請說明每個連結證明了什麼。工作人員應該能夠在不需要私人憑證或機密的情況下理解其
關係。

## 不應包含的內容

請勿在公開 GitHub 議題中放入機密或私人證明。請勿包含：

- API 權杖、簽署金鑰或憑證
- DNS 挑戰權杖
- 私人法律檔案或合約
- 個人身分文件
- 私人電子郵件、私人安全回報或機密客戶資料

宣告表單會詢問敏感證據是否需要私人工作人員通道。
請使用該選項，而不是公開發布敏感資料。

## 可能結果

根據證據與風險，ClawHub 工作人員可能會保留命名空間、
轉讓所有權、重新命名資源、隱藏或隔離現有列表、
新增別名或重新導向、要求更多證明，或拒絕請求。

命名空間審查不保證每個相符名稱都會被轉讓。
工作人員會權衡公開證據、現有使用情況、安全風險與使用者影響。
