# 組織與命名空間申請

ClawHub 將擁有者代稱、組織代稱、Skill 短名稱、外掛套件名稱及套件命名範圍用作公開命名空間。如果某個命名空間看似屬於現實世界中的專案、品牌、套件生態系或組織，但在 ClawHub 上已被申請、保留、以誤導方式使用或存在爭議，請透過
[組織／命名空間申請問題表單](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
要求工作人員審查。

此途徑適用於公開且不涉及敏感資訊的所有權審查。請勿使用產品內檢舉功能或帳號申訴表單提出命名空間申請。

## 何時提出申請

如果你認為基於現實世界中的所有權，ClawHub 工作人員應審查是否需要保留、移轉、重新命名、隱藏、隔離、設定別名或以其他方式變更某個命名空間，請提出命名空間申請。

相關情況包括：

- 與你的 GitHub 組織、專案、公司或社群相符的組織代稱
- 應僅允許相符的 ClawHub 擁有者發布的套件命名範圍，例如 `@example-org/*`
- 看似冒充某個專案的 Skill 短名稱或外掛套件名稱
- 品牌、商標、專案重新命名或套件歷史紀錄爭議
- 已刪除、停止活動或無法聯絡，且阻礙正當命名空間擁有者的現有擁有者

如果除了所有權爭議外，該項目還具有不安全、惡意或誤導性，亦請遵循相關的內容管理或安全性指引。命名空間申請表單用於所有權審查，不適用於緊急漏洞揭露。

## 提交前準備

請先確認你使用的發布擁有者與命名空間相符。對於外掛套件，`@example-org/example-plugin` 等具命名範圍的名稱必須由相符的 `example-org` 擁有者發布。

如果你可以管理目前的擁有者，請透過發布、重新命名、移轉、隱藏或刪除受影響的資源，直接修正命名空間。當你無法管理目前的擁有者，或需要工作人員解決爭議時，才提出申請。

## 應附上的證據

請使用公開且不涉及敏感資訊的證據。實用的證明包括：

- GitHub 組織、儲存庫、版本或維護者歷史紀錄
- 指明該命名空間的官方專案文件
- 網域或官方電子郵件網域的證明
- 對 npm、PyPI、crates.io 或其他套件登錄庫命名範圍的控制權
- 適合公開討論的商標、品牌或專案所有權證據
- 原始碼儲存庫歷史、套件歷史或公開的重新命名公告
- 爭議所涉及之 ClawHub 擁有者、Skill、外掛、套件或問題的連結

請說明每個連結可證明的事項。工作人員應能在不需要私人認證資訊或密鑰的情況下，理解其中的關聯。

## 不應附上的內容

請勿在公開的 GitHub 問題中放入密鑰或私人證明。請勿包含：

- API 權杖、簽署金鑰或認證資訊
- DNS 驗證權杖
- 私人法律文件或合約
- 個人身分證明文件
- 私人電子郵件、非公開安全性報告或機密客戶資料

申請表單會詢問敏感證據是否需要透過工作人員的私人管道提供。請使用該選項，而不要公開張貼敏感資料。

## 可能的處理結果

ClawHub 工作人員可能根據證據和風險保留命名空間、移轉所有權、重新命名資源、隱藏或隔離現有項目、新增別名或重新導向、要求提供更多證明，或拒絕申請。

命名空間審查不保證所有相符名稱都會移轉。工作人員會衡量公開證據、現有使用情況、安全性風險及對使用者的影響。
