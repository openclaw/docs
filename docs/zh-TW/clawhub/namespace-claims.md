# 組織與命名空間主張

ClawHub 使用擁有者代號、組織代號、Skill 短名稱、外掛套件名稱及
套件範圍作為公開命名空間。如果某個命名空間看似屬於現實世界中的
專案、品牌、套件生態系或組織，但在 ClawHub 上已遭主張、保留、具有誤導性或存在爭議，請透過
[組織／命名空間主張問題表單](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
要求工作人員審查。

此管道適用於公開且不涉及敏感資訊的所有權審查。請勿使用產品內
檢舉功能或帳號申訴表單提出命名空間主張。

## 何時提出主張

如果你認為 ClawHub 工作人員應根據現實世界中的所有權，審查是否應
保留、移轉、重新命名、隱藏、隔離、設定別名或以其他方式變更某個
命名空間，請提出命名空間主張。

範例包括：

- 與你的 GitHub 組織、專案、公司或社群相符的組織代號
- 如 `@example-org/*` 的套件範圍，且只應由相符的
  ClawHub 擁有者發布
- 疑似冒充某個專案的 Skill 短名稱或外掛套件名稱
- 品牌、商標、專案更名或套件歷史爭議
- 已刪除、停用或無法聯絡，因而阻礙正當命名空間
  擁有者的現有擁有者

如果該項目除了所有權爭議外，也不安全、具有惡意或誤導性，
亦請遵循相關的內容管理或安全性指引。命名空間主張
表單用於所有權審查，而非緊急漏洞揭露。

## 提交前準備

請先確認你是使用與該命名空間相符的擁有者身分進行發布。
對於外掛套件，如 `@example-org/example-plugin` 的範圍名稱必須
以相符的 `example-org` 擁有者身分發布。

如果你可以管理目前的擁有者，請透過發布、重新命名、移轉、隱藏或刪除
受影響的資源，直接修正命名空間。當你無法管理目前的擁有者，或需要
工作人員解決爭議時，才提出主張。

## 應附上的證據

請使用公開且不涉及敏感資訊的證據。實用的證明包括：

- GitHub 組織、儲存庫、版本或維護者歷史
- 載明該命名空間的官方專案文件
- 網域或官方電子郵件網域證明
- npm、PyPI、crates.io 或其他套件登錄檔的範圍控制權
- 可安全公開討論的商標、品牌或專案所有權證據
- 原始碼儲存庫歷史、套件歷史或公開更名公告
- 爭議所涉及之 ClawHub 擁有者、Skill、外掛、套件或議題的連結

請說明每個連結所證明的內容。工作人員應能在不需要私人認證資訊或
祕密的情況下理解其中的關係。

## 不應附上的內容

請勿在公開的 GitHub 議題中放入祕密或非公開證明。請勿包含：

- API 權杖、簽署金鑰或認證資訊
- DNS 挑戰權杖
- 非公開的法律文件或合約
- 個人身分證明文件
- 私人電子郵件、非公開安全報告或機密客戶資料

主張表單會詢問敏感證據是否需要透過工作人員的私人管道提交。
請使用該選項，而不要公開張貼敏感資料。

## 可能的處理結果

視證據與風險而定，ClawHub 工作人員可能會保留命名空間、
移轉所有權、重新命名資源、隱藏或隔離現有項目、
新增別名或重新導向、要求更多證明，或拒絕該要求。

命名空間審查不保證所有相符的名稱都會被移轉。
工作人員會衡量公開證據、既有使用情形、安全風險及使用者影響。
