# 組織與命名空間聲明

ClawHub 使用擁有者代號、組織代號、Skill slug、外掛套件名稱及套件範圍作為公開命名空間。如果某個命名空間看似屬於現實世界中的專案、品牌、套件生態系或組織，但已在 ClawHub 上遭到占用、保留、冒用或引發爭議，請使用
[組織／命名空間聲明問題表單](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
要求工作人員審查。

此途徑適用於公開且不涉及敏感資訊的所有權審查。命名空間聲明請勿使用產品內檢舉功能或帳號申訴表單。

## 何時提出聲明

如果你認為因現實世界中的所有權問題，ClawHub 工作人員應審查是否需要保留、移轉、重新命名、隱藏、隔離命名空間、為其設定別名或以其他方式變更，請提出命名空間聲明。

範例包括：

- 與你的 GitHub 組織、專案、公司或社群相符的組織代號
- `@example-org/*` 之類應僅能由相符的 ClawHub 擁有者發布的套件範圍
- 看似冒充某個專案的 Skill slug 或外掛套件名稱
- 品牌、商標、專案重新命名或套件歷史爭議
- 已刪除、停用或無法聯絡的擁有者，阻礙命名空間的正當擁有者

如果清單除了所有權爭議外，還有不安全、惡意或誤導性內容，也請遵循相關的內容管理或安全性指引。命名空間聲明表單用於所有權審查，而非緊急漏洞揭露。

## 提交前須知

請先確認你是使用與命名空間相符的擁有者進行發布。對於外掛套件，`@example-org/example-plugin` 之類的範圍名稱必須由相符的 `example-org` 擁有者發布。

如果你可以管理目前的擁有者，請透過發布、重新命名、移轉、隱藏或刪除受影響的資源，直接修正命名空間。當你無法管理目前的擁有者，或需要由工作人員解決爭議時，才使用聲明。

## 應提供的證據

請使用公開且不涉及敏感資訊的證據。實用的證明包括：

- GitHub 組織、儲存庫、版本發布或維護者歷史
- 提及該命名空間的官方專案文件
- 網域或官方電子郵件網域的證明
- npm、PyPI、crates.io 或其他套件登錄服務的範圍控制權
- 可安全公開討論的商標、品牌或專案所有權證據
- 原始碼儲存庫歷史、套件歷史或公開的重新命名公告
- 爭議所涉及的 ClawHub 擁有者、Skill、外掛、套件或問題連結

請說明每個連結能證明什麼。工作人員應能在不需要私人認證資訊或機密資料的情況下理解其關係。

## 不應提供的內容

請勿在公開的 GitHub 問題中放入機密資料或非公開證明。請勿包含：

- API 權杖、簽署金鑰或認證資訊
- DNS 驗證權杖
- 私人法律文件或合約
- 個人身分證明文件
- 私人電子郵件、非公開安全性報告或機密客戶資料

聲明表單會詢問敏感證據是否需要透過私人工作人員管道提供。請使用該選項，而不要公開發布敏感資料。

## 可能的處理結果

根據證據與風險，ClawHub 工作人員可能會保留命名空間、移轉所有權、重新命名資源、隱藏或隔離現有清單、新增別名或重新導向、要求提供更多證明，或拒絕請求。

命名空間審查不保證所有相符名稱都會被移轉。工作人員會衡量公開證據、現有使用情況、安全風險及對使用者的影響。
