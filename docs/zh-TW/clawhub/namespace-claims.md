# 組織與命名空間申請

ClawHub 使用擁有者代號、組織代號、技能 slug、外掛套件名稱，以及
套件 scope 作為公開命名空間。如果某個命名空間看起來屬於
真實世界的專案、品牌、套件生態系或組織，但在 ClawHub 上已被
申請、保留、具有誤導性，或存在爭議，請要求工作人員透過
[組織／命名空間申請議題表單](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
進行審查。

此途徑適用於公開、非敏感的所有權審查。請勿將產品內回報或帳號申訴表單
用於命名空間申請。

## 何時提出申請

當你認為 ClawHub 工作人員應審查某個命名空間是否應因真實世界的所有權而
被保留、移轉、重新命名、隱藏、隔離、設為別名，或以其他方式變更時，
請提出命名空間申請。

範例包括：

- 與你的 GitHub 組織、專案、公司或社群相符的組織代號
- 像 `@example-org/*` 這樣應只由相符 ClawHub 擁有者發布的套件 scope
- 看起來冒充某個專案的技能 slug 或外掛套件名稱
- 品牌、商標、專案重新命名，或套件歷史爭議
- 已刪除、非活躍或無法聯絡的擁有者，阻擋了正當的命名空間擁有者

如果該清單在所有權爭議之外還具有不安全、惡意或誤導性，
也請遵循相關的審核或安全指引。命名空間申請表單用於所有權審查，
不是緊急漏洞揭露管道。

## 提交之前

請先確認你是以符合該命名空間的擁有者身分發布。對於外掛套件，
像 `@example-org/example-plugin` 這類 scope 名稱，必須由相符的
`example-org` 擁有者發布。

如果你可以管理目前的擁有者，請透過發布、重新命名、移轉、隱藏或刪除
受影響資源，直接修正命名空間。當你無法管理目前的擁有者，或需要工作人員
解決爭議時，再使用申請。

## 應包含的證據

請使用公開、非敏感的證據。有幫助的證明包括：

- GitHub 組織、儲存庫、發行版本或維護者歷史
- 命名該命名空間的官方專案文件
- 網域或官方電子郵件網域證明
- npm、PyPI、crates.io 或其他套件登錄檔的 scope 控制權
- 可公開討論的商標、品牌或專案所有權證據
- 原始碼儲存庫歷史、套件歷史或公開重新命名通知
- 指向有爭議的 ClawHub 擁有者、技能、外掛、套件或議題的連結

請說明每個連結證明了什麼。工作人員應能在不需要私人憑證或秘密的情況下
理解其關聯。

## 不應包含的內容

請勿在公開的 GitHub 議題中放入秘密或私人證明。請勿包含：

- API 權杖、簽署金鑰或憑證
- DNS 驗證權杖
- 私人法律文件或合約
- 個人身分文件
- 私人電子郵件、私人安全報告或機密客戶資料

申請表單會詢問敏感證據是否需要私人工作人員管道。
請使用該選項，而不是公開張貼敏感資料。

## 可能結果

視證據與風險而定，ClawHub 工作人員可能會保留命名空間、
移轉所有權、重新命名資源、隱藏或隔離現有清單、加入別名或重新導向、
要求更多證明，或拒絕請求。

命名空間審查不保證每個相符名稱都會被移轉。工作人員會權衡公開證據、
現有使用情況、安全風險與使用者影響。
