# 組織與命名空間聲明

ClawHub 使用擁有者代號、組織代號、技能短名、外掛套件名稱及套件作用域作為公開命名空間。如果某個命名空間看似屬於現實世界中的專案、品牌、套件生態系或組織，但在 ClawHub 上已遭占用、保留、具有誤導性或存在爭議，請透過
[組織／命名空間聲明議題表單](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
要求工作人員審查。

此途徑適用於公開且不涉及敏感資訊的所有權審查。請勿使用產品內檢舉功能或帳號申訴表單提出命名空間聲明。

## 何時提出聲明

如果您認為 ClawHub 工作人員應根據現實世界中的所有權，審查是否應保留、移轉、重新命名、隱藏、隔離、設定別名或以其他方式變更某個命名空間，請提出命名空間聲明。

範例包括：

- 與您的 GitHub 組織、專案、公司或社群相符的組織代號
- `@example-org/*` 之類的套件作用域，且應僅能由相符的 ClawHub 擁有者發布
- 看似冒充某個專案的技能短名或外掛套件名稱
- 品牌、商標、專案重新命名或套件歷史爭議
- 已刪除、停止活動或無法聯絡，因而妨礙正當命名空間擁有者的現有擁有者

如果清單除了所有權爭議外，還涉及不安全、惡意或誤導性內容，請一併遵循相關的內容管理或安全性指引。命名空間聲明表單用於所有權審查，而非緊急漏洞揭露。

## 提交前須知

請先確認您是使用與命名空間相符的擁有者身分進行發布。若為外掛套件，`@example-org/example-plugin` 之類的作用域名稱必須由相符的 `example-org` 擁有者發布。

如果您可以管理目前的擁有者，請直接透過發布、重新命名、移轉、隱藏或刪除受影響的資源來修正命名空間。當您無法管理目前的擁有者，或需要工作人員解決爭議時，再提出聲明。

## 應附上的證據

請使用公開且不涉及敏感資訊的證據。實用的證明包括：

- GitHub 組織、儲存庫、版本發布或維護者歷史記錄
- 載明該命名空間的官方專案文件
- 網域或官方電子郵件網域的證明
- 對 npm、PyPI、crates.io 或其他套件登錄庫作用域的控制權
- 可安全公開討論的商標、品牌或專案所有權證據
- 原始碼儲存庫歷史、套件歷史或公開的重新命名公告
- 有爭議的 ClawHub 擁有者、技能、外掛、套件或議題連結

請說明每個連結可證明的事項。工作人員應能在不需要私人憑證或機密資訊的情況下理解其中的關係。

## 不應附上的內容

請勿在公開的 GitHub 議題中放入機密資訊或非公開證明。請勿附上：

- API 權杖、簽署金鑰或憑證
- DNS 驗證權杖
- 非公開的法律文件或合約
- 個人身分證明文件
- 私人電子郵件、非公開安全性報告或機密客戶資料

聲明表單會詢問敏感證據是否需要透過工作人員的私人管道提交。請使用該選項，而不要公開張貼敏感資料。

## 可能的處理結果

ClawHub 工作人員可能會根據證據與風險保留命名空間、移轉所有權、重新命名資源、隱藏或隔離現有清單、新增別名或重新導向、要求更多證明，或駁回請求。

命名空間審查不保證所有相符名稱都會被移轉。工作人員會衡量公開證據、現有使用情況、安全風險及對使用者的影響。
