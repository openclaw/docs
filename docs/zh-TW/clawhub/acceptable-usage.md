# 可接受使用

ClawHub 代管 OpenClaw 的 Skills、外掛、套件和市集中繼資料。
使用本頁判斷內容或發布行為是否適合放在
ClawHub。

這些規則適用於清單項目的功能、要求使用者執行的內容、如何
呈現自身，以及發布者如何使用 ClawHub 的探索、安裝和
信任介面。如需審核狀態和帳號信譽，請參閱
[審核與帳號安全](/clawhub/moderation)。如需著作權或其他權利
主張，請參閱[內容權利請求](/clawhub/content-rights)。

## 允許的內容

ClawHub 歡迎有用、易於理解且本著善意發布的內容。

| 類別                                         | 允許條件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開發者生產力                           | 清單項目能協助使用者建置、測試、遷移、偵錯、撰寫文件或維運軟體。                                               |
| UI、資料與自動化工作流程               | 範圍清楚、必要憑證明確，且風險動作包含審查、dry-run、預覽或確認路徑。 |
| 防禦性安全、審核與濫用審查 | 工具以授權審查為定位、保留證據，並清楚維持人工核准邊界。                          |
| 個人或團隊工作流程                       | 工作流程使用基於同意的帳號、透明的設定流程和明確的權限。                                            |
| 維護中的目錄                              | 每個清單項目都彼此不同、有用、描述準確，並受到合理維護。                                                |

情境很重要。同一主題在狹義的防禦性或
基於同意的設定中可能可以接受，但若被包裝成濫用工作流程則不可接受。

## 不允許的市集行為

ClawHub 也會審查發布者如何使用市集。請勿使用 ClawHub
操縱探索、指標、信任訊號、審核系統或使用者
注意力。

不允許的市集行為包括：

- 大量發布大量低投入、重複、占位或
  機器生成且看不出具有真實使用者價值的清單項目
- 以近乎相同的 Skills 或外掛灌滿搜尋或分類介面
- 發布數百個幾乎沒有或完全沒有使用量、維護、來源
  清晰度或有意義差異化的清單項目
- 透過自動化、自我安裝循環、假帳號、協同行動、
  付費互動或其他非自然行為，人為灌高安裝、下載、星標或其他互動
  指標
- 建立或輪替帳號以規避審核、封鎖、發布者限制或
  市集審查
- 在所有權、來源、能力、安全狀態、
  安裝要求，或與其他專案或發布者的關聯方面誤導使用者
- 在未修正根本問題的情況下，反覆上傳已被隱藏、移除或封鎖的
  內容

高量發布不會自動構成濫用。當清單項目具有實質差異、描述準確、受到維護，
並且由真實使用者使用時，大型目錄可以接受。當
數量伴隨內容薄弱、重複、誤導、未維護或
人為推廣的清單項目時，大型目錄就會成為信任與安全問題。

## 內容權利

如果你認為 ClawHub 上的內容侵犯你的著作權或其他權利，請使用
[內容權利請求](/clawhub/content-rights)。除非清單項目同時不安全、
惡意或誤導，否則請勿使用一般市集回報提出著作權或權利主張。

## 審查與執行

ClawHub 可能使用自動檢查、統計濫用訊號、使用者回報和
工作人員審查來識別不安全內容或濫用發布行為。訊號
本身不能證明濫用；它會協助 ClawHub 判斷哪些內容需要審查。

我們可能會：

- 隱藏、暫停、移除、軟刪除，或在資源類型支援時
  硬刪除違規清單項目
- 封鎖不安全版本的下載或安裝
- 撤銷 API 權杖
- 軟刪除相關內容
- 限制發布存取
- 封鎖重複或嚴重違規者

對於明顯濫用，我們不保證先警告再執行。請參閱
[審核與帳號安全](/clawhub/moderation)，了解回報、審核暫停、
隱藏清單項目、封鎖和帳號信譽。
