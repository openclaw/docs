# 可接受使用

本頁說明 ClawHub 接受哪些類型的 Skills 與內容，以及它不會託管的濫用工作流程。

這些規則刻意保持務實。我們最在意的是端到端的濫用工作流程，而不只是孤立的關鍵字。如果某個 Skill 是為了規避防禦、濫用平台、詐騙他人、侵犯隱私，或促成未經同意的行為而建立，它就不屬於 ClawHub。

## 我們明確接受的近期模式

- 使用真實元件、語意權杖、無障礙狀態，以及經測試使用者流程的前端與設計系統工作。
- 使用已安裝的原始碼元件、專案別名和文件化變體，而不是一次性標記的 shadcn/ui 組合。
- 保留註解、使用具體 UI5 型別，並讓產生的控制項介面可供審查的 UI5 JavaScript-to-TypeScript 轉換。
- 防禦性安全審查、審核工具，以及顯示證據並清楚保留人工核准邊界的濫用偵測提示。
- 針對個人或團隊帳號、以明確憑證、透明設定，以及 dry-run 或預覽模式為基礎的同意式工作流程自動化。
- 範圍限定於其支援軟體的文件、遷移操作手冊、開發者工具，以及測試 fixtures。

## 不接受

- 安全繞過或未授權存取工作流程。
  - 範例：auth 繞過、帳號接管、CAPTCHA 繞過、Cloudflare 或反機器人規避、速率限制繞過、設計用來擊敗保護機制的隱匿式爬取、即時通話或代理接管、可重複使用的工作階段竊取、為未核准使用者自動核准配對流程。

- 平台濫用與封鎖規避。
  - 範例：被封鎖後的隱匿帳號、帳號暖身/養號、虛假互動、karma 或追蹤者培養、多帳號自動化、大量發文、垃圾訊息機器人、設計用來避免偵測的市集或社群自動化。

- 詐欺、詐騙，以及欺騙性金融工作流程。
  - 範例：偽造證書、偽造發票、欺騙性付款流程、詐騙外展、虛假社會證明、在沒有明確人工核准和透明控制的情況下促成支出或收費的工具，或為詐欺建立帳號而設計的合成身分工作流程。

- 侵犯隱私的爬取、資料補全或監控。
  - 範例：為垃圾訊息而大規模爬取聯絡資料、開盒、跟蹤、搭配未經請求外展的潛在客戶擷取、祕密監控、未經明確同意使用的人臉搜尋或生物特徵比對，或購買、發布、下載，或將外洩資料或資料外洩 dump 操作化。

- 未經同意的冒名或欺騙性身分操控。
  - 範例：換臉、數位分身、假人物、複製的網紅，或其他用於冒名或誤導的身分操控工具。

- 露骨性內容與停用安全機制的成人內容生成。
  - 範例：NSFW 圖像/影片/內容生成、包裝第三方 API 的成人內容工具，或主要目的為露骨性內容的 Skills。

- 隱藏、不安全或誤導性的執行需求。
  - 範例：混淆的安裝命令、`curl | sh`、未宣告的祕密需求、未宣告的私鑰使用、沒有清楚可審查性的遠端 `npx @latest` 執行、隱藏該 Skill 實際執行需求的誤導性中繼資料。

## 我們明確不接受的近期模式

- 「在市集封鎖後建立隱匿賣家帳號。」
- 「修改 Telegram 配對，讓未核准使用者自動收到配對代碼。」
- 「以不可偵測的自動化培養 Reddit/Twitter 帳號。」
- 「產生可任意使用的專業證書或發票。」
- 「停用安全檢查來產生 NSFW 內容。」
- 「爬取潛在客戶、補全聯絡人資料，並大規模發起陌生外展。」
- 「購買、發布或下載外洩資料或資料外洩 dump。」
- 「使用合成身分或 CAPTCHA 解題大量建立電子郵件或社群帳號。」

## 給審查者的注意事項

- 脈絡很重要。同一主題在範圍狹窄的防禦性或同意式情境中可能是正當的，但如果被包裝成濫用工作流程，則不可接受。
- 當某個 Skill 明顯針對規避、欺騙或未經同意的使用最佳化時，我們應傾向採取行動。
- 在這些類別中重複上傳，是隱藏內容並封鎖帳號的理由。

## 執行

- 我們可能會隱藏、移除或硬刪除違規的 Skills。
- 我們可能會撤銷權杖、軟刪除相關內容，並封鎖重複或嚴重違規者。
- 對於明顯濫用，我們不保證會先警告再執法。
