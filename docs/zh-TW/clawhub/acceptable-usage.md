# 可接受使用

ClawHub 代管 OpenClaw 的 Skills、外掛、套件與市集中繼資料。
使用本頁判斷內容或發布行為是否適合 ClawHub。

這些規則適用於刊登項目的功能、要求使用者執行的內容、呈現自身的方式，以及發布者如何使用 ClawHub 的探索、安裝與信任介面。關於審核狀態與帳號狀態，請參閱
[審核與帳號安全](/clawhub/moderation)。關於著作權或其他權利主張，請參閱[內容權利請求](/clawhub/content-rights)。

## 允許的內容

ClawHub 歡迎有用、易於理解且以善意發布的內容。

| 類別                                         | 允許條件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開發者生產力                           | 刊登項目可協助使用者建置、測試、遷移、除錯、文件化或營運軟體。                                               |
| UI、資料與自動化工作流程               | 範圍明確、所需憑證清楚說明，且高風險動作包含審查、試跑、預覽或確認路徑。 |
| 防禦性安全、審核與濫用審查 | 工具以授權審查為定位、保留證據，並清楚維持人工核准邊界。                          |
| 個人或團隊工作流程                       | 工作流程使用以同意為基礎的帳號、透明設定，以及明確權限。                                            |
| 維護中的目錄                              | 每個刊登項目都具備區別性、有用、描述準確，且受到合理維護。                                                |

情境很重要。同一主題在狹義的防禦性或同意基礎情境中可以接受，但若被包裝成濫用工作流程則不可接受。

## 禁止的市集行為

ClawHub 也會審查發布者如何使用市集。請勿使用 ClawHub 操縱探索、指標、信任訊號、審核系統或使用者注意力。

禁止的市集行為包括：

- 大量發布大量低投入、重複、佔位或機器生成的刊登項目，而這些項目看起來沒有真實使用者價值
- 以幾乎相同的 Skills 或外掛淹沒搜尋或類別介面
- 發布數百個幾乎或完全沒有使用量、維護、來源清晰度或有意義差異化的刊登項目
- 透過自動化、自我安裝循環、假帳號、協同行動、付費互動或其他非自然行為，人為膨脹安裝、下載、星標或其他互動指標
- 建立或輪替帳號以規避審核、封禁、發布者限制或市集審查
- 在所有權、來源、能力、安全態勢、安裝需求，或與另一個專案或發布者的關係上誤導使用者
- 在未修正根本問題的情況下，重複上傳已被隱藏、移除或封鎖的內容

高量發布不會自動構成濫用。當刊登項目具有有意義的差異、描述準確、受到維護，且由真實使用者使用時，大型目錄是可以接受的。當數量搭配薄弱、重複、誤導、未維護或人為推廣的刊登項目時，大型目錄就會成為信任與安全問題。

## 內容權利

如果你認為 ClawHub 上的內容侵害你的著作權或其他權利，請使用[內容權利請求](/clawhub/content-rights)。除非該刊登項目同時不安全、惡意或具誤導性，否則請勿使用一般市集檢舉來提出著作權或權利主張。

## 審查與執行

ClawHub 可能使用自動化檢查、統計濫用訊號、使用者檢舉與員工審查，以識別不安全內容或濫用性發布行為。單一訊號本身並不能證明濫用；它會協助 ClawHub 判斷哪些項目需要審查。

我們可能會：

- 隱藏、暫停、移除、軟刪除，或在資源類型支援時，硬刪除違規刊登項目
- 封鎖不安全版本的下載或安裝
- 撤銷 API 權杖
- 軟刪除相關內容
- 限制發布存取權
- 封禁重複或嚴重違規者

對於明顯濫用，我們不保證會先警告再執行處置。關於檢舉、審核暫停、隱藏的刊登項目、封禁與帳號狀態，請參閱[審核與帳號安全](/clawhub/moderation)。
