# 可接受使用

ClawHub 託管 OpenClaw 的技能、外掛、套件與市集 metadata。
使用此頁面判斷內容或發布行為是否適合放在
ClawHub。

這些規則適用於列表項目的用途、要求使用者執行的內容、如何
呈現自身，以及發布者如何使用 ClawHub 的探索、安裝與
信任介面。關於審核狀態與帳號狀態，請參閱
[審核與帳號安全](/clawhub/moderation)。關於著作權或其他權利
主張，請參閱[內容權利請求](/clawhub/content-rights)。

## 允許的內容

ClawHub 歡迎有用、易懂且以善意發布的內容。

| 類別                                         | 允許條件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開發者生產力                           | 列表項目可協助使用者建置、測試、遷移、除錯、撰寫文件或營運軟體。                                               |
| 介面、資料與自動化工作流程               | 範圍清楚、必要憑證明確，且高風險操作包含審查、試執行、預覽或確認流程。 |
| 防禦性安全、審核與濫用審查 | 工具定位為授權審查用途、保留證據，並清楚維持人工核准界線。                          |
| 個人或團隊工作流程                       | 工作流程使用基於同意的帳號、透明的設定流程與明確權限。                                            |
| 維護中的目錄                              | 每個列表項目都不同、有用、描述準確，並有合理維護。                                                |

情境很重要。同一主題在狹義的防禦性或
基於同意的情境中可能可以接受，但若被包裝成濫用工作流程則不可接受。

## 不允許的內容

ClawHub 不託管主要目的為濫用、欺騙、不安全
執行或侵害權利的內容。

| 類別                                                    | 不允許                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 未授權存取或安全繞過                      | Auth 繞過、帳號接管、濫用速率限制、即時通話或代理接管、可重複使用的工作階段竊取，或為未核准使用者自動核准配對流程。                                                                                                                                                   |
| 平台濫用與規避停權                              | 停權後的隱蔽帳號、帳號養成或農場化、假互動、多帳號自動化、大量發布、垃圾訊息機器人，或為規避偵測而建置的自動化。                                                                                                                                          |
| 詐欺、詐騙與欺騙性金融工作流程             | 假憑證或發票、欺騙性付款流程、詐騙外展、假社會證明、用於詐欺的合成身份工作流程，或未有清楚人工核准的花費/收款工具。                                                                                                                    |
| 侵犯隱私的強化資料或監控                 | 為垃圾訊息抓取聯絡人、公開個人隱私、跟蹤、搭配未請求外展的潛在客戶擷取、隱蔽監控、未經同意的生物特徵比對，或使用外洩資料或資料外洩 dump。                                                                                                                  |
| 未經同意的冒充或身份操弄       | 換臉、數位分身、複製的網紅、假人格，或其他用於冒充或誤導的工具。                                                                                                                                                                                                 |
| 明確性內容或停用安全機制的成人內容生成 | NSFW 圖片、影片或內容生成；圍繞第三方 API 的成人內容 wrapper；或主要目的為明確性內容的列表項目。                                                                                                                                                       |
| 隱藏、不安全或誤導性的執行要求        | 混淆的安裝命令、pipe-to-shell 安裝程式，例如在沒有清楚可審查性的情況下用 `sh` 或 `bash` 執行下載內容、未宣告的 secret 或私密金鑰要求、沒有清楚可審查性的遠端 `npx @latest` 執行，或隱藏列表項目實際執行需求的 metadata。 |
| 侵害著作權或違反權利的素材           | 未經許可重新發布他人的技能、外掛、文件、品牌資產或專有程式碼；違反授權條款；或冒充原作者或發布者。                                                                                                                            |

## 不允許的市集行為

ClawHub 也會審查發布者如何使用市集。請勿使用 ClawHub
操弄探索、指標、信任訊號、審核系統或使用者
注意力。

不允許的市集行為包括：

- 批量發布大量低投入、重複、佔位或
  機器生成的列表項目，且這些項目看不出有真實使用者價值
- 用幾乎相同的技能或外掛灌滿搜尋或分類介面
- 發布數百個幾乎沒有使用量、維護、來源
  清楚度或有意義差異化的列表項目
- 透過自動化、自我安裝迴圈、假帳號、協同行動、
  付費互動或其他非自然行為，人為灌高安裝數、下載數、星標或其他互動
  指標
- 建立或輪換帳號以規避審核、停權、發布者限制或
  市集審查
- 在所有權、來源、能力、安全狀態、
  安裝要求或與其他專案或發布者的關聯方面誤導使用者
- 反覆上傳已被隱藏、移除或封鎖的內容，
  且未修正根本問題

大量發布不會自動構成濫用。當列表項目有實質差異、
描述準確、有維護，並由真實使用者使用時，大型目錄是可以接受的。
當大量項目伴隨內容薄弱、重複、誤導、缺乏維護或
人為推廣的列表項目時，大型目錄就會成為信任與安全問題。

## 內容權利

如果你認為 ClawHub 上的內容侵害你的著作權或其他權利，請使用
[內容權利請求](/clawhub/content-rights)。除非該列表項目同時不安全、
惡意或具誤導性，否則不要使用一般市集
檢舉來提出著作權或權利主張。

## 審查與執行

ClawHub 可能使用自動檢查、統計濫用訊號、使用者檢舉與
工作人員審查來識別不安全內容或濫用發布行為。單一訊號
本身並不能證明濫用；它協助 ClawHub 判斷需要審查的內容。

我們可能會：

- 隱藏、保留、移除、軟刪除，或在資源類型支援時，
  硬刪除違規列表項目
- 封鎖不安全 release 的下載或安裝
- 撤銷 API token
- 軟刪除相關內容
- 限制發布存取權
- 停權屢犯或嚴重違規者

對於明顯濫用，我們不保證先警告再執行。請參閱
[審核與帳號安全](/clawhub/moderation)，了解檢舉、審核保留、
隱藏列表項目、停權與帳號狀態。
