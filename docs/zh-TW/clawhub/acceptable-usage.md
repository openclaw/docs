# 可接受使用規範

ClawHub 代管 OpenClaw 的 Skills、外掛、套件與市集中繼資料。
請使用本頁判斷內容或發布行為是否適合放在 ClawHub 上。

這些規則適用於上架項目的功能、要求使用者執行的內容、其自我呈現方式，以及發布者如何使用 ClawHub 的探索、安裝與信任介面。關於審核狀態與帳號信譽，請參閱[審核與帳號安全](/clawhub/moderation)。關於著作權或其他權利主張，請參閱[內容權利請求](/zh-TW/clawhub/content-rights)。

## 允許的內容

ClawHub 歡迎有用、易懂，且以善意發布的內容。

| 類別 | 允許條件 |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開發者生產力 | 上架項目能協助使用者建置、測試、遷移、偵錯、撰寫文件或營運軟體。 |
| UI、資料與自動化工作流程 | 範圍清楚、明確列出所需憑證，且高風險動作包含審查、試執行、預覽或確認路徑。 |
| 防禦性安全、審核與濫用檢視 | 工具定位為授權檢視用途、保留證據，並清楚維持人工核准邊界。 |
| 個人或團隊工作流程 | 工作流程使用基於同意的帳號、透明的設定流程與明確權限。 |
| 維護中的目錄 | 每個上架項目都彼此不同、有用、描述準確，並受到合理維護。 |

情境很重要。同一主題在狹義的防禦性或基於同意的情境中可能可以接受，但若包裝成濫用工作流程則不可接受。

## 禁止的內容

ClawHub 不代管主要目的為濫用、欺騙、不安全執行或侵犯權利的內容。

| 類別 | 不允許 |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 未授權存取或安全繞過 | 身分驗證繞過、帳號接管、濫用速率限制、即時通話或代理接管、可重複使用的工作階段竊取，或為未核准使用者自動核准配對流程。 |
| 平台濫用與規避封鎖 | 遭封鎖後使用隱匿帳號、帳號養成或耕作、虛假互動、多帳號自動化、大量發文、垃圾訊息機器人，或為避免偵測而建置的自動化。 |
| 詐欺、詐騙與欺騙性金融工作流程 | 偽造憑證或發票、欺騙性付款流程、詐騙推廣、虛假社會證明、用於詐欺的合成身分工作流程，或沒有清楚人工核准的支出／收款工具。 |
| 侵犯隱私的資料擴充或監控 | 為垃圾訊息抓取聯絡人、起底、跟蹤、將名單擷取搭配未經請求的推廣、秘密監控、未經同意的生物特徵比對，或使用外洩資料或資料外洩傾印。 |
| 未經同意的冒充或身分操弄 | 換臉、數位分身、複製網紅、虛假人格，或其他用於冒充或誤導的工具。 |
| 明示性性內容或停用安全措施的成人內容生成 | NSFW 圖片、影片或內容生成；包裝第三方 API 的成人內容工具；或主要目的為明示性性內容的上架項目。 |
| 隱藏、不安全或誤導性的執行需求 | 混淆的安裝命令、下載內容後以 `sh` 或 `bash` 執行且缺乏清楚可審查性的 pipe-to-shell 安裝器、未宣告的密鑰或私密金鑰需求、缺乏清楚可審查性的遠端 `npx @latest` 執行，或隱藏上架項目實際執行需求的中繼資料。 |
| 侵犯著作權或違反權利的素材 | 未經許可重新發布他人的 Skills、外掛、文件、品牌素材或專有程式碼；違反授權條款；或冒充原作者或發布者。 |

## 禁止的市集行為

ClawHub 也會審查發布者如何使用市集。請勿使用 ClawHub 操弄探索、指標、信任訊號、審核系統或使用者注意力。

禁止的市集行為包括：

- 大量發布低投入、重複、佔位或機器生成的上架項目，且這些項目看起來沒有實際使用者價值
- 以幾乎相同的 Skills 或外掛淹沒搜尋或分類介面
- 發布數百個幾乎沒有使用量、維護、來源清楚度或有意義差異化的上架項目
- 透過自動化、自行安裝迴圈、假帳號、協同行動、付費互動或其他非自然行為，人為灌高安裝、下載、星標或其他互動指標
- 建立或輪換帳號以規避審核、封鎖、發布者限制或市集審查
- 在所有權、來源、能力、安全狀態、安裝需求，或與其他專案或發布者的關聯方面誤導使用者
- 在未修正根本問題的情況下，反覆上傳已被隱藏、移除或封鎖的內容

大量發布不會自動構成濫用。若上架項目有實質差異、描述準確、受到維護，並由真實使用者使用，大型目錄是可以接受的。當數量伴隨單薄、重複、誤導、缺乏維護或人為推廣的上架項目時，大型目錄就會成為信任與安全問題。

## 內容權利

如果你認為 ClawHub 上的內容侵犯你的著作權或其他權利，請使用[內容權利請求](/zh-TW/clawhub/content-rights)。除非該上架項目同時不安全、惡意或具誤導性，否則請勿使用一般市集檢舉來提出著作權或權利主張。

## 審查與執行

ClawHub 可能使用自動化檢查、統計濫用訊號、使用者檢舉與人員審查，識別不安全內容或濫用發布行為。單一訊號本身不能證明濫用；它會協助 ClawHub 判斷哪些內容需要審查。

我們可能會：

- 隱藏、暫停、移除、軟刪除，或在資源類型支援時硬刪除違規上架項目
- 封鎖不安全版本的下載或安裝
- 撤銷 API 權杖
- 軟刪除相關內容
- 限制發布存取權
- 封鎖重複或嚴重違規者

對於明顯濫用，我們不保證會先警告再執行處置。關於檢舉、審核暫停、隱藏的上架項目、封鎖與帳號信譽，請參閱[審核與帳號安全](/clawhub/moderation)。
