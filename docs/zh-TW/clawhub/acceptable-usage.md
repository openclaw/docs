# 可接受使用

本頁說明 ClawHub 可接受的技能與內容類型，以及它不會代管的濫用工作流程。

這些規則刻意保持務實。我們最重視的是端到端濫用工作流程，而不只是孤立的關鍵字。如果某項技能是為了規避防護、濫用平台、詐騙他人、侵犯隱私，或促成未經同意的行為而打造，它就不適合出現在 ClawHub 上。

## 我們明確接受的近期模式

- 使用真實元件、語意化權杖、可存取狀態與經測試使用者流程的前端與設計系統工作。
- 使用已安裝原始碼元件、專案別名與已文件化變體，而不是一次性標記的 shadcn/ui 組合。
- 保留註解、使用具體 UI5 型別，並讓產生的控制項介面可供審閱的 UI5 JavaScript 至 TypeScript 轉換。
- 防禦性安全審查、審核工具，以及顯示證據並保持人工核准邊界清楚的濫用偵測提示。
- 針對個人或團隊帳號，具備明確憑證、透明設定，以及試執行或預覽模式的同意式工作流程自動化。
- 文件、遷移執行手冊、開發者工具，以及範圍限於其支援軟體的測試固定資料。

## 不接受

- 安全繞過或未授權存取工作流程。
  - 範例：驗證繞過、帳號接管、CAPTCHA 繞過、Cloudflare 或反機器人規避、速率限制繞過、旨在擊敗保護措施的隱蔽爬取、即時通話或代理接管、可重複使用的工作階段竊取、為未核准使用者自動核准配對流程。

- 平台濫用與封鎖規避。
  - 範例：遭封鎖後建立隱蔽帳號、帳號暖身/養號、假互動、karma 或追蹤者培養、多帳號自動化、大量發文、垃圾訊息機器人、為避免偵測而打造的市集或社群自動化。

- 詐欺、詐騙與欺瞞性金融工作流程。
  - 範例：假證書、假發票、欺瞞性付款流程、詐騙式外聯、假社會證明、在缺乏清楚人工核准與透明控制的情況下促成支出或收款的工具，或為詐欺建立帳號而打造的合成身分工作流程。

- 侵犯隱私的爬取、資料補全或監控。
  - 範例：為垃圾訊息大規模爬取聯絡資訊、肉搜、跟蹤、搭配未經邀請外聯的潛在客戶擷取、隱蔽監控、未經明確同意使用的人臉搜尋或生物特徵比對，或購買、發布、下載或操作化外洩資料或資料外洩傾印。

- 未經同意的冒充或欺瞞性身分操弄。
  - 範例：換臉、數位分身、假人物、複製網紅，或其他用於冒充或誤導的身分操弄工具。

- 露骨色情內容與停用安全機制的成人內容生成。
  - 範例：NSFW 圖片/影片/內容生成、包裝第三方 API 的成人內容外殼，或主要用途是露骨色情內容的技能。

- 隱藏、不安全或具誤導性的執行需求。
  - 範例：混淆的安裝命令、`curl | sh`、未宣告的機密需求、未宣告的私密金鑰使用、缺乏清楚可審閱性的遠端 `npx @latest` 執行、隱藏技能實際執行需求的誤導性中繼資料。

## 我們明確不接受的近期模式

- 「在市集封鎖後建立隱蔽賣家帳號。」
- 「修改 Telegram 配對，讓未核准使用者自動收到配對碼。」
- 「以無法偵測的自動化培養 Reddit/Twitter 帳號。」
- 「產生可任意使用的專業證書或發票。」
- 「在停用安全檢查的情況下產生 NSFW 內容。」
- 「爬取潛在客戶、補全聯絡人資料，並大規模發起冷外聯。」
- 「購買、發布或下載外洩資料或資料外洩傾印。」
- 「使用合成身分或 CAPTCHA 解題大量建立電子郵件或社群帳號。」

## 給審閱者的注意事項

- 情境很重要。同一主題在狹窄的防禦性或同意式設定中可能是正當的，但若包裝成濫用工作流程則不可接受。
- 當某項技能明顯針對規避、欺瞞或未經同意的使用而最佳化時，我們應傾向採取行動。
- 重複上傳這些類別的內容，足以構成隱藏內容與封鎖帳號的理由。
