# 可接受使用

本頁說明 ClawHub 可以接受哪些類型的 Skills 與內容，以及它不會託管的濫用工作流程。

這些規則刻意保持務實。我們最重視端到端濫用工作流程，而不只是孤立的關鍵字。如果某個 Skill 是為了規避防護、濫用平台、詐騙他人、侵犯隱私，或促成未經同意的行為而建置，它就不適合出現在 ClawHub 上。

## 我們明確接受的近期模式

- 使用真實元件、語義化權杖、無障礙狀態，以及經測試使用者流程的前端與設計系統工作。
- 使用已安裝來源元件、專案別名與文件化變體，而不是一次性標記的 shadcn/ui 組合。
- 保留註解、使用具體 UI5 型別，並讓產生的控制項介面可供審查的 UI5 JavaScript 至 TypeScript 轉換。
- 防禦性安全審查、審核工具，以及顯示證據並清楚保留人工核准界線的濫用偵測提示。
- 針對個人或團隊帳戶、具備明確憑證、透明設定，以及 dry-run 或預覽模式的同意式工作流程自動化。
- 限定於其所支援軟體範圍內的文件、遷移執行手冊、開發者工具與測試夾具。

## 不接受

- 安全繞過或未授權存取工作流程。
  - 範例：驗證繞過、帳戶接管、CAPTCHA 繞過、Cloudflare 或反機器人規避、速率限制繞過、為擊敗防護而設計的隱匿式爬取、即時通話或代理接管、可重用的工作階段竊取、自動核准未核准使用者的配對流程。

- 平台濫用與封鎖規避。
  - 範例：遭封鎖後建立隱匿帳戶、帳戶養號/農場化、虛假互動、karma 或追蹤者培養、多帳戶自動化、大量發文、垃圾訊息機器人、為避免偵測而建置的市集或社群自動化。

- 詐欺、詐騙與欺騙性金融工作流程。
  - 範例：假證書、假發票、欺騙性付款流程、詐騙外聯、虛假社會證明、在沒有清楚人工核准與透明控制的情況下促成支出或收款的工具，或為詐欺建立帳戶而建置的合成身分工作流程。

- 侵犯隱私的爬取、資料擴充或監控。
  - 範例：大規模爬取聯絡資訊以發送垃圾訊息、公開個人隱私、跟蹤、搭配未經請求外聯的潛在客戶擷取、隱蔽監控、未經明確同意使用的人臉搜尋或生物特徵比對，或購買、發布、下載或操作化外洩資料或資料外洩傾印。

- 未經同意的冒充或欺騙性身分操控。
  - 範例：換臉、數位分身、假人物、複製的網紅，或其他用於冒充或誤導的身分操控工具。

- 明示性性內容與停用安全機制的成人內容生成。
  - 範例：NSFW 圖片/影片/內容生成、圍繞第三方 API 的成人內容包裝器，或主要目的為明示性性內容的 Skills。

- 隱藏、不安全或誤導性的執行需求。
  - 範例：混淆的安裝命令、`curl | sh`、未宣告的祕密需求、未宣告的私密金鑰使用、未提供清楚可審查性的遠端 `npx @latest` 執行，或隱藏 Skill 真正執行需求的誤導性中繼資料。

## 我們明確不接受的近期模式

- 「在市集封鎖後建立隱匿賣家帳戶。」
- 「修改 Telegram 配對，讓未核准使用者自動收到配對碼。」
- 「以不可偵測的自動化培養 Reddit/Twitter 帳戶。」
- 「產生可任意使用的專業證書或發票。」
- 「在停用安全檢查的情況下產生 NSFW 內容。」
- 「爬取潛在客戶、擴充聯絡人資料，並大規模發起冷外聯。」
- 「購買、發布或下載外洩資料或資料外洩傾印。」
- 「使用合成身分或 CAPTCHA 解題大量建立電子郵件或社群帳戶。」

## 審查者注意事項

- 情境很重要。同一主題在狹義的防禦性或同意式設定中可能是合法的，但若被包裝成濫用工作流程則不可接受。
- 當某個 Skill 明顯針對規避、欺騙或未經同意使用最佳化時，我們應傾向採取行動。
- 反覆上傳這些類別的內容，可構成隱藏內容並封鎖帳戶的理由。

## 執行

- 我們可能會隱藏、移除或硬刪除違規 Skills。
- 我們可能會撤銷權杖、軟刪除相關內容，並封鎖重複或嚴重違規者。
- 對於明顯濫用，我們不保證會先警告再執行處置。
