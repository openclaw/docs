# 可接受的使用方式

ClawHub 託管 OpenClaw 的 Skills、外掛、套件及市集詮釋資料。
請使用此頁面判斷內容或發布行為是否適合放在 ClawHub 上。

這些規則適用於項目所執行的操作、要求使用者執行的內容、呈現自身的方式，以及發布者如何使用 ClawHub 的探索、安裝與信任機制。關於審核狀態與帳戶狀態，請參閱[審核與帳戶安全](/clawhub/moderation)。關於著作權或其他權利主張，請參閱[內容權利請求](/clawhub/content-rights)。

## 允許的內容

ClawHub 歡迎實用、易於理解且秉持善意發布的內容。

| 類別                                             | 符合以下條件時允許                                                                                                              |
| ------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------- |
| 開發者生產力                                     | 項目可協助使用者建置、測試、遷移、偵錯、撰寫文件或操作軟體。                                                                    |
| 使用者介面、資料與自動化工作流程                 | 範圍清楚、明確列出必要的憑證，且高風險操作提供審查、試執行、預覽或確認流程。                                                      |
| 防禦性安全、審核與濫用行為檢查                   | 工具定位為用於經授權的檢查、會保留證據，並清楚維持人工核准的界線。                                                              |
| 個人或團隊工作流程                               | 工作流程使用基於同意的帳戶、透明的設定程序及明確的權限。                                                                        |
| 持續維護的目錄                                   | 每個項目皆具獨特性與實用性、說明準確，並獲得合理維護。                                                                          |

情境相當重要。同一主題在範圍明確的防禦性或基於同意的情境中可能可以接受，但若包裝為濫用工作流程，則不可接受。

## 禁止的市集行為

ClawHub 也會審查發布者使用市集的方式。請勿使用 ClawHub 操弄探索結果、指標、信任訊號、審核系統或使用者注意力。

禁止的市集行為包括：

- 大量發布看似不具實際使用者價值的低投入、重複、預留位置或
  機器生成項目
- 以近乎相同的 Skills 或外掛大量灌入搜尋結果或類別頁面
- 發布數百個幾乎或完全沒有使用量、維護、來源透明度或實質差異的
  項目
- 透過自動化、自我安裝循環、虛假帳戶、協同行動、付費互動或其他
  非自然行為，人為提高安裝次數、下載次數、星號或其他互動指標
- 建立或輪替帳戶，以規避審核、封禁、發布者限制或
  市集審查
- 在所有權、來源、功能、安全狀態、安裝要求，或與其他專案或發布者的
  關聯方面誤導使用者
- 未修正根本問題，便反覆上傳已遭隱藏、移除或封鎖的
  內容

大量發布不一定是濫用。若項目之間具有實質差異、說明準確、持續維護，且確實有真實使用者使用，則大型目錄可以接受。當大量項目同時具有內容單薄、重複、誤導、缺乏維護或經人為推廣等情況時，大型目錄就會成為信任與安全問題。

## 內容權利

如果您認為 ClawHub 上的內容侵犯您的著作權或其他權利，請使用[內容權利請求](/clawhub/content-rights)。除非項目同時具有不安全、惡意或誤導性，否則請勿使用一般市集檢舉功能提出著作權或其他權利主張。

## 審查與處置

ClawHub 可能使用自動化檢查、統計性濫用訊號、使用者檢舉及工作人員審查，識別不安全內容或濫用性發布行為。單一訊號本身無法證明濫用；它能協助 ClawHub 判斷哪些內容需要審查。

我們可能會：

- 隱藏、暫緩、移除、軟刪除違規項目，或在資源類型支援的情況下
  永久刪除違規項目
- 封鎖不安全版本的下載或安裝
- 撤銷 API 權杖
- 軟刪除相關內容
- 限制發布權限
- 封禁屢次違規或情節嚴重的違規者

對於明顯的濫用行為，我們不保證會先行警告再採取處置。關於檢舉、審核暫緩、隱藏項目、封禁及帳戶狀態，請參閱[審核與帳戶安全](/clawhub/moderation)。
