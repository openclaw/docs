# 可接受使用

本頁說明 ClawHub 接受哪些類型的技能與內容，以及它不會託管哪些濫用工作流程。

這些規則刻意保持務實。我們最在意的是端對端的濫用工作流程，而不只是孤立的關鍵字。如果某個技能是為了規避防禦、濫用平台、詐騙他人、侵犯隱私，或促成未經同意的行為而建置，就不屬於 ClawHub。

## 我們明確接受的近期模式

- 使用真實元件、語意化權杖、可存取狀態，以及經測試使用者流程的前端與設計系統工作。
- 使用已安裝的原始碼元件、專案別名與文件化變體，而非一次性標記的 shadcn/ui 組合。
- 保留註解、使用具體 UI5 型別，並讓產生的控制項介面可供審查的 UI5 JavaScript 到 TypeScript 轉換。
- 防禦性安全審查、審核工具，以及會顯示證據並明確保留人工核准邊界的濫用偵測提示。
- 針對個人或團隊帳號、以同意為基礎的工作流程自動化，並具備明確憑證、透明設定，以及試跑或預覽模式。
- 文件、遷移操作手冊、開發者工具，以及限定於其所支援軟體範圍內的測試夾具。

## 不接受

- 安全繞過或未授權存取工作流程。
  - 範例：驗證繞過、帳號接管、CAPTCHA 繞過、Cloudflare 或反機器人規避、速率限制繞過、設計用來擊敗保護機制的隱匿式抓取、即時通話或代理接管、可重複使用的工作階段竊取、為未核准使用者自動核准配對流程。

- 平台濫用與封鎖規避。
  - 範例：遭封鎖後建立隱匿帳號、帳號養成/農場、虛假互動、karma 或追蹤者培養、多帳號自動化、大量發文、垃圾訊息機器人、為避免偵測而建置的市集或社群自動化。

- 詐欺、詐騙與欺瞞性金融工作流程。
  - 範例：偽造證書、偽造發票、欺瞞性付款流程、詐騙推廣、虛假社會證明、在缺乏明確人工核准與透明控制的情況下促成支出或收費的工具，或為詐欺建立帳號而建置的合成身分工作流程。

- 侵犯隱私的抓取、擴充或監控。
  - 範例：大規模抓取聯絡資訊以發送垃圾訊息、肉搜、跟蹤、搭配未經請求外聯的潛在客戶擷取、隱蔽監控、未經明確同意使用的人臉搜尋或生物特徵比對，或購買、發布、下載或操作化外洩資料或外洩資料庫。

- 未經同意的冒充或欺瞞性身分操弄。
  - 範例：換臉、數位分身、虛假人設、複製網紅，或其他用於冒充或誤導的身分操弄工具。

- 明示性性內容與停用安全機制的成人內容生成。
  - 範例：NSFW 圖像/影片/內容生成、圍繞第三方 API 的成人內容包裝器，或主要目的為明示性性內容的技能。

- 隱藏、不安全或誤導性的執行需求。
  - 範例：混淆的安裝命令、`curl | sh`、未宣告的祕密需求、未宣告的私鑰使用、缺乏明確可審查性的遠端 `npx @latest` 執行、隱藏技能實際執行需求的誤導性中繼資料。

## 我們明確不接受的近期模式

- 「在市集封鎖後建立隱匿賣家帳號。」
- 「修改 Telegram 配對，讓未核准使用者自動收到配對代碼。」
- 「以無法偵測的自動化培養 Reddit/Twitter 帳號。」
- 「產生可任意使用的專業證書或發票。」
- 「在停用安全檢查的情況下產生 NSFW 內容。」
- 「抓取潛在客戶、擴充聯絡資訊，並大規模發起冷外聯。」
- 「購買、發布或下載外洩資料或外洩資料庫。」
- 「使用合成身分或 CAPTCHA 解題大量建立電子郵件或社群帳號。」

## 給審查者的注意事項

- 情境很重要。同一主題在狹義防禦或以同意為基礎的設定中可能是正當的，但若被包裝成濫用工作流程，則不可接受。
- 當某個技能明顯針對規避、欺瞞或未經同意的用途最佳化時，我們應傾向採取行動。
- 反覆上傳這些類別的內容，可作為隱藏內容並封鎖帳號的依據。

## 執行

- 我們可能會隱藏、移除或硬刪除違規技能。
- 我們可能會撤銷權杖、軟刪除相關內容，並封鎖重複或嚴重違規者。
- 對於明顯濫用，我們不保證會先警告再執行。
