# 可接受使用

本頁說明 ClawHub 可接受的技能與內容類型，以及它不會託管的濫用工作流程。

這些規則刻意保持務實。我們最重視的是端到端的濫用工作流程，而不只是孤立的關鍵字。如果某項技能是為了規避防禦、濫用平台、詐騙他人、侵犯隱私，或促成未經同意的行為而建置，它就不適合出現在 ClawHub 上。

## 我們明確接受的近期模式

- 使用真實元件、語意化權杖、無障礙狀態，以及經測試使用者流程的前端與設計系統工作。
- 使用已安裝的原始碼元件、專案別名與文件化變體，而不是一次性標記的 shadcn/ui 組合。
- 保留註解、使用具體 UI5 型別，並讓產生的控制項介面可審查的 UI5 JavaScript 至 TypeScript 轉換。
- 防禦性安全審查、審核工具，以及顯示證據並清楚保留人工核准邊界的濫用偵測提示。
- 針對個人或團隊帳戶、具備明確憑證、透明設定，以及試執行或預覽模式的同意式工作流程自動化。
- 限定於其支援軟體範圍內的文件、遷移操作手冊、開發者工具與測試夾具。

## 不可接受

- 安全性繞過或未授權存取工作流程。
  - 範例：驗證繞過、帳戶接管、CAPTCHA 繞過、Cloudflare 或反機器人規避、速率限制繞過、旨在擊敗防護的隱蔽式爬取、即時通話或代理接管、可重複使用的工作階段竊取、為未核准使用者自動核准配對流程。

- 平台濫用與封鎖規避。
  - 範例：遭封鎖後建立隱蔽帳戶、帳戶養成/農場化、假互動、karma 或追蹤者培養、多帳戶自動化、大量發文、垃圾訊息機器人、為避免偵測而建置的市集或社群自動化。

- 詐欺、詐騙與欺瞞性金融工作流程。
  - 範例：假證書、假發票、欺瞞性付款流程、詐騙推廣、假社會證明、在缺乏明確人工核准與透明控制的情況下促成支出或收款的工具，或為詐欺建立帳戶而建置的合成身分工作流程。

- 侵犯隱私的爬取、資料強化或監控。
  - 範例：大規模爬取聯絡資訊以發送垃圾訊息、肉搜、跟蹤、搭配未經請求推廣的潛在客戶擷取、祕密監控、在缺乏明確同意下使用的人臉搜尋或生物特徵比對，或購買、發布、下載或操作化外洩資料或資料外洩傾印。

- 未經同意的冒充或欺瞞性身分操控。
  - 範例：換臉、數位分身、假人設、複製的網紅，或其他用於冒充或誤導的身分操控工具。

- 露骨性內容與停用安全機制的成人內容生成。
  - 範例：NSFW 圖片/影片/內容生成、包裝第三方 API 的成人內容工具，或主要目的為露骨性內容的技能。

- 隱藏、不安全或誤導性的執行需求。
  - 範例：混淆的安裝命令、`curl | sh`、未宣告的祕密需求、未宣告的私鑰使用、缺乏清楚可審查性的遠端 `npx @latest` 執行、隱藏技能實際執行需求的誤導性中繼資料。

## 我們明確不接受的近期模式

- 「在市集封鎖後建立隱蔽賣家帳戶。」
- 「修改 Telegram 配對，讓未核准使用者自動收到配對代碼。」
- 「以無法偵測的自動化培養 Reddit/Twitter 帳戶。」
- 「產生可任意使用的專業證書或發票。」
- 「在停用安全檢查的情況下產生 NSFW 內容。」
- 「爬取潛在客戶、強化聯絡人資料，並大規模啟動冷推廣。」
- 「購買、發布或下載外洩資料或資料外洩傾印。」
- 「使用合成身分或 CAPTCHA 解題大量建立電子郵件或社群帳戶。」

## 給審查者的注意事項

- 情境很重要。同一主題在狹義的防禦性或同意式設定中可能是正當的，但若被包裝成濫用工作流程則不可接受。
- 當某項技能明顯針對規避、欺瞞或未經同意的使用而最佳化時，我們應傾向採取行動。
- 在這些類別中重複上傳，是隱藏內容並封鎖帳戶的理由。

## 執行

- 我們可能隱藏、移除或硬刪除違規技能。
- 我們可能撤銷權杖、軟刪除相關內容，並封鎖重複或嚴重違規者。
- 對於明顯濫用，我們不保證會先警告再執行處置。
