# 可接受使用政策

ClawHub 託管 OpenClaw 的 Skills、外掛、套件與市集中繼資料。
請使用本頁判斷內容或發布行為是否適合出現在
ClawHub 上。

這些規則適用於清單的作用、要求使用者執行的內容、它如何
呈現自身，以及發布者如何使用 ClawHub 的探索、安裝與
信任介面。關於審核狀態與帳號信譽，請參閱
[審核與帳號安全](/clawhub/moderation)。關於著作權或其他權利
主張，請參閱[內容權利請求](/clawhub/content-rights)。

## 允許的內容

ClawHub 歡迎有用、易懂且以善意發布的內容。

| 類別                                             | 允許條件                                                                                                                          |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開發者生產力                                     | 清單協助使用者建置、測試、遷移、除錯、記錄或運作軟體。                                                                           |
| UI、資料與自動化工作流程                         | 範圍清楚、所需憑證明確，且有風險的動作包含審查、試執行、預覽或確認路徑。                                                       |
| 防禦性安全、審核與濫用檢閱                       | 工具以授權檢閱為定位、保留證據，並清楚維持人工核准邊界。                                                                         |
| 個人或團隊工作流程                               | 工作流程使用基於同意的帳號、透明的設定流程與明確權限。                                                                           |
| 維護中的目錄                                     | 每個清單都具備獨特性、有用、描述準確，且受到合理維護。                                                                           |

情境很重要。同一主題在狹義的防禦性或
基於同意的情境中可能可以接受，但若包裝成濫用工作流程則不可接受。

## 禁止的市集行為

ClawHub 也會檢閱發布者如何使用市集。請勿使用 ClawHub
操縱探索、指標、信任訊號、審核系統或使用者
注意力。

禁止的市集行為包括：

- 批量發布大量低投入、重複、佔位或
  機器生成且看不出具有實際使用者價值的清單
- 以近乎相同的 Skills 或外掛灌滿搜尋或分類介面
- 發布數百個幾乎或完全沒有使用量、維護、來源
  清晰度或有意義差異化的清單
- 透過自動化、自我安裝循環、假帳號、協同行為、
  付費互動或其他非自然行為，人為膨脹安裝數、下載數、星標或其他互動
  指標
- 建立或輪替帳號以規避審核、封鎖、發布者限制或
  市集檢閱
- 對所有權、來源、能力、安全狀態、
  安裝需求，或與其他專案或發布者的關係誤導使用者
- 在未修正根本問題的情況下，反覆上傳已被隱藏、移除或封鎖
  的內容

高量發布不會自動構成濫用。大型目錄在清單之間有實質差異、
描述準確、受到維護，且由真實使用者使用時是可以接受的。
當高量發布伴隨內容薄弱、重複、誤導、缺乏維護或
人為推廣的清單時，大型目錄就會成為信任與安全問題。

## 內容權利

如果你認為 ClawHub 上的內容侵害你的著作權或其他權利，請使用
[內容權利請求](/clawhub/content-rights)。除非該清單同時不安全、
惡意或具誤導性，否則請勿使用一般市集
檢舉來提出著作權或權利主張。

## 檢閱與執行

ClawHub 可能使用自動化檢查、統計濫用訊號、使用者檢舉與
工作人員檢閱，以識別不安全內容或濫用性發布行為。訊號
本身不能證明濫用；它會協助 ClawHub 判斷哪些內容需要檢閱。

我們可能會：

- 隱藏、暫停、移除、軟刪除，或在資源類型支援時
  硬刪除違規清單
- 封鎖不安全版本的下載或安裝
- 撤銷 API 權杖
- 軟刪除相關內容
- 限制發布權限
- 封鎖重複或嚴重違規者

對於明顯濫用，我們不保證會先警告再執行處置。請參閱
[審核與帳號安全](/clawhub/moderation)，了解檢舉、審核暫停、
隱藏清單、封鎖與帳號信譽。
