# 可接受使用

本頁說明 ClawHub 可接受的 Skills 與內容類型，以及它不會代管的濫用工作流程。

這些規則刻意保持務實。我們最重視的是端到端的濫用工作流程，而不只是孤立的關鍵字。如果某項 skill 是為了規避防禦、濫用平台、詐騙他人、侵犯隱私，或促成非合意行為而建置，它就不適合出現在 ClawHub。

## 我們明確接受的近期模式

- 使用真實元件、語意化權杖、無障礙狀態，以及經測試使用者流程的前端與設計系統工作。
- 使用已安裝的來源元件、專案別名與文件化變體，而不是一次性標記的 shadcn/ui 組合。
- 保留註解、使用具體 UI5 型別，並讓產生的控制項介面可供審查的 UI5 JavaScript-to-TypeScript 轉換。
- 防禦性安全審查、審核工具，以及顯示證據並清楚保留人工核准界線的濫用偵測提示。
- 針對個人或團隊帳戶、以同意為基礎的工作流程自動化，具備明確憑證、透明設定，以及試執行或預覽模式。
- 文件、遷移作業手冊、開發者工具，以及限定於其支援軟體範圍內的測試夾具。

## 不接受

- 安全繞過或未授權存取工作流程。
  - 範例：驗證繞過、帳戶接管、CAPTCHA 繞過、Cloudflare 或反機器人規避、速率限制繞過、設計用來擊敗保護機制的隱蔽擷取、即時通話或代理接管、可重複使用的工作階段竊取、為未核准使用者自動核准配對流程。

- 平台濫用與規避封鎖。
  - 範例：遭封鎖後建立隱蔽帳戶、帳戶養號/農場、虛假互動、karma 或追蹤者培養、多帳戶自動化、大量發文、垃圾訊息機器人、為避免偵測而建置的市集或社交自動化。

- 詐欺、詐騙與欺瞞性金融工作流程。
  - 範例：假證書、假發票、欺瞞性付款流程、詐騙外聯、虛假社會證明、在缺乏明確人工核准與透明控制的情況下促成花費或收款的工具，或為詐欺建立帳戶而建置的合成身分工作流程。

- 侵犯隱私的擷取、資料豐富化或監控。
  - 範例：大規模擷取聯絡資訊用於垃圾訊息、公開個人隱私、跟蹤、搭配未經請求外聯的潛在客戶擷取、秘密監控、未經明確同意使用的人臉搜尋或生物特徵比對，或購買、發布、下載或實際運用外洩資料或資料外洩傾印。

- 非合意冒充或欺瞞性身分操弄。
  - 範例：換臉、數位分身、假人設、複製的影響者，或其他用於冒充或誤導的身分操弄工具。

- 明確性內容與停用安全措施的成人內容產生。
  - 範例：NSFW 圖片/影片/內容產生、包裝第三方 API 的成人內容工具，或主要用途為明確性內容的 Skills。

- 隱藏、不安全或誤導性的執行需求。
  - 範例：混淆的安裝指令、`curl | sh`、未宣告的祕密需求、未宣告的私密金鑰使用、在缺乏清楚可審查性的情況下遠端執行 `npx @latest`、隱藏 skill 實際執行需求的誤導性中繼資料。

## 我們明確不接受的近期模式

- 「在市集封鎖後建立隱蔽賣家帳戶。」
- 「修改 Telegram 配對，使未核准使用者自動收到配對代碼。」
- 「以無法偵測的自動化培養 Reddit/Twitter 帳戶。」
- 「產生可任意使用的專業證書或發票。」
- 「在停用安全檢查的情況下產生 NSFW 內容。」
- 「擷取潛在客戶、豐富化聯絡資訊，並大規模發起冷外聯。」
- 「購買、發布或下載外洩資料或資料外洩傾印。」
- 「以合成身分或 CAPTCHA 解題大量建立電子郵件或社交帳戶。」

## 給審查者的注意事項

- 情境很重要。同一主題在狹義的防禦性或以同意為基礎的設定中可能是正當的，但在被包裝成濫用工作流程時則不可接受。
- 當某項 skill 明顯針對規避、欺瞞或非合意用途進行最佳化時，我們應傾向採取行動。
- 重複上傳這些類別的內容，是隱藏內容並封鎖帳戶的理由。

## 執行

- 我們可能會隱藏、移除或硬刪除違規的 Skills。
- 我們可能會撤銷權杖、軟刪除相關內容，並封鎖重複或嚴重違規者。
- 對於明顯濫用，我們不保證會先行警告再執行處置。
