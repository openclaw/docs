# 可接受使用

本頁說明 ClawHub 可接受的技能與內容類型，以及它不會託管的濫用工作流程。

這些規則刻意保持務實。我們最重視端到端的濫用工作流程，而不只是孤立的關鍵字。如果某項技能是為了規避防護、濫用平台、詐騙他人、侵犯隱私，或促成非自願行為而建置，它就不適合放在 ClawHub 上。

## 我們明確接受的近期模式

- 使用真實元件、語意化權杖、無障礙狀態，以及經測試使用者流程的前端與設計系統工作。
- 使用已安裝來源元件、專案別名與文件化變體，而非一次性標記的 shadcn/ui 組合。
- 保留註解、使用具體 UI5 型別，並讓產生的控制項介面可審查的 UI5 JavaScript 至 TypeScript 轉換。
- 防禦性安全審查、審核工具，以及顯示證據並明確保留人工核准邊界的濫用偵測提示。
- 針對個人或團隊帳號，具備明確憑證、透明設定，以及 dry-run 或預覽模式的同意式工作流程自動化。
- 文件、遷移作業手冊、開發者工具，以及範圍限於其支援軟體的測試夾具。

## 不可接受

- 安全繞過或未授權存取工作流程。
  - 範例：繞過驗證、帳號接管、繞過 CAPTCHA、規避 Cloudflare 或反機器人機制、繞過速率限制、設計用來擊敗防護的隱蔽爬取、即時通話或代理程式接管、可重複使用的工作階段竊取、為未核准使用者自動核准配對流程。

- 平台濫用與規避封鎖。
  - 範例：遭封鎖後建立隱蔽帳號、帳號暖身/養號、虛假互動、培養 karma 或追蹤者、多帳號自動化、大量發文、垃圾訊息機器人、為避免偵測而建置的市集或社群自動化。

- 詐欺、詐騙與欺瞞性金融工作流程。
  - 範例：假證書、假發票、欺瞞性付款流程、詐騙式外展、虛假社會認同、在缺乏明確人工核准與透明控制的情況下促成支出或收款的工具，或為詐欺建立帳號而建置的合成身分工作流程。

- 侵犯隱私的爬取、資料補全或監控。
  - 範例：大規模爬取聯絡資訊以發送垃圾訊息、doxxing、跟蹤、搭配未經請求外展的潛在客戶擷取、隱蔽監控、未經明確同意使用的人臉搜尋或生物特徵比對，或購買、發布、下載或操作化外洩資料或資料外洩傾印。

- 非自願冒充或欺瞞性身分操控。
  - 範例：換臉、數位分身、假人物設定、複製網紅，或其他用於冒充或誤導的身分操控工具。

- 露骨性內容與停用安全機制的成人內容生成。
  - 範例：NSFW 圖像/影片/內容生成、包裝第三方 API 的成人內容包裝器，或主要用途為露骨性內容的技能。

- 隱藏、不安全或誤導性的執行需求。
  - 範例：混淆的安裝指令、`curl | sh`、未宣告的祕密需求、未宣告的私鑰使用、缺乏明確可審查性的遠端 `npx @latest` 執行、隱藏技能真正執行需求的誤導性中繼資料。

## 我們明確不接受的近期模式

- 「在市集封鎖後建立隱蔽賣家帳號。」
- 「修改 Telegram 配對，讓未核准使用者自動收到配對碼。」
- 「以不可偵測的自動化培養 Reddit/Twitter 帳號。」
- 「為任意用途產生專業證書或發票。」
- 「在停用安全檢查的情況下產生 NSFW 內容。」
- 「爬取潛在客戶、補全聯絡資料，並大規模發動陌生外展。」
- 「購買、發布或下載外洩資料或資料外洩傾印。」
- 「使用合成身分或 CAPTCHA 解題大量建立電子郵件或社群帳號。」

## 給審查者的注意事項

- 情境很重要。同一主題在狹義的防禦性或同意式設定中可能是正當的，但若被包裝成濫用工作流程則不可接受。
- 當某項技能明顯針對規避、欺瞞或非自願使用最佳化時，我們應傾向採取行動。
- 重複上傳這些類別的內容，可作為隱藏內容與封鎖帳號的理由。

## 執行

- 我們可能隱藏、移除或硬刪除違規技能。
- 我們可能撤銷權杖、軟刪除相關內容，並封鎖重複或嚴重違規者。
- 對於明顯濫用，我們不保證會先警告再執行處置。
