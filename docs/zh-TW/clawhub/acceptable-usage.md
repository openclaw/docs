# 可接受使用

ClawHub 為 OpenClaw 託管技能、外掛、套件與市集後設資料。
使用本頁判斷內容或發布行為是否適合出現在
ClawHub。

這些規則適用於清單的功能、它要求使用者執行的內容、它如何
呈現自身，以及發布者如何使用 ClawHub 的探索、安裝與
信任介面。若要了解審核狀態與帳號信譽，請參閱
[審核與帳號安全](/clawhub/moderation)。若要提出著作權或其他權利
主張，請參閱[內容權利請求](/clawhub/content-rights)。

## 允許的內容

ClawHub 歡迎有用、易懂且以善意發布的內容。

| 類別                                             | 允許條件                                                                                                                          |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開發者生產力                                     | 清單能協助使用者建置、測試、遷移、偵錯、記錄或營運軟體。                                               |
| 介面、資料與自動化工作流程                       | 範圍清楚、所需憑證明確，且高風險動作包含審查、試執行、預覽或確認路徑。 |
| 防禦性安全、審核與濫用檢視                       | 工具定位為授權檢視用途，會保留證據，並清楚維持人工核准邊界。                          |
| 個人或團隊工作流程                               | 工作流程使用基於同意的帳號、透明的設定流程與明確的權限。                                            |
| 維護中的目錄                                     | 每個清單都彼此區分、有用、描述準確，且有合理維護。                                                |

情境很重要。同一個主題在狹義的防禦性或
基於同意的情境中可能可以接受，但若被包裝成濫用工作流程則不可接受。

## 禁止的內容

ClawHub 不託管主要目的為濫用、欺騙、不安全
執行或侵犯權利的內容。

| 類別                                                        | 不允許                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 未授權存取或安全繞過                                      | 驗證繞過、帳號接管、濫用速率限制、即時通話或代理接管、可重複使用的工作階段竊取，或為未核准使用者自動核准配對流程。                                                                                                                                                   |
| 平台濫用與規避封鎖                                        | 被封鎖後使用隱匿帳號、帳號養成或農場、虛假互動、多帳號自動化、大量發布、垃圾訊息機器人，或為避免偵測而建置的自動化。                                                                                                                                          |
| 詐欺、詐騙與欺騙性金融工作流程                             | 假憑證或發票、欺騙性付款流程、詐騙式外聯、虛假社會認同、用於詐欺的合成身分工作流程，或沒有明確人工核准的支出／收款工具。                                                                                                                    |
| 侵犯隱私的強化資料或監控                                   | 為垃圾訊息抓取聯絡人、公開個資、跟蹤、與未經請求外聯搭配的名單擷取、隱密監控、未經同意的生物特徵比對，或使用外洩資料或外洩資料庫。                                                                                                                  |
| 未經同意的冒充或身分操控                                   | 換臉、數位分身、複製網紅、假人設，或其他用於冒充或誤導的工具。                                                                                                                                                                                                 |
| 明確性內容或停用安全限制的成人內容生成                     | 成人限制級圖片、影片或內容生成；包裝第三方 API 的成人內容工具；或主要目的為明確性內容的清單。                                                                                                                                                       |
| 隱藏、不安全或誤導性的執行需求                             | 混淆的安裝命令、例如下載內容後在未提供清楚可審查性的情況下以 `sh` 或 `bash` 執行的管線至 shell 安裝器、未宣告的祕密或私鑰需求、未提供清楚可審查性的遠端 `npx @latest` 執行，或隱藏清單實際執行需求的後設資料。 |
| 侵犯著作權或違反權利的素材                                 | 未經許可重新發布他人的技能、外掛、文件、品牌素材或專有程式碼；違反授權條款；或冒充原作者或發布者。                                                                                                                            |

## 禁止的市集行為

ClawHub 也會審查發布者如何使用市集。請勿使用 ClawHub
操控探索、指標、信任訊號、審核系統或使用者
注意力。

禁止的市集行為包括：

- 大量發布低投入、重複、佔位或
  機器產生且看起來沒有實際使用者價值的大量清單
- 以近乎相同的技能或外掛淹沒搜尋或分類介面
- 發布數百個幾乎沒有或完全沒有使用量、維護、來源
  清楚度或有意義差異化的清單
- 透過自動化、自我安裝迴圈、假帳號、協同行動、
  付費互動或其他非自然行為，人為提高安裝、下載、星號或其他互動
  指標
- 建立或輪換帳號以規避審核、封鎖、發布者限制或
  市集審查
- 在所有權、來源、功能、安全狀態、
  安裝需求，或與其他專案或發布者的關係方面誤導使用者
- 在未修正根本問題的情況下，重複上傳已被隱藏、移除或封鎖的
  內容

高量發布不會自動構成濫用。當清單之間有實質差異、描述準確、受到維護，
並由真實使用者使用時，大型目錄可以接受。當數量與空泛、重複、誤導、
缺乏維護或
人為推廣的清單相結合時，大型目錄就會成為信任與安全問題。

## 內容權利

如果你認為 ClawHub 上的內容侵犯你的著作權或其他權利，請使用
[內容權利請求](/clawhub/content-rights)。除非清單同時不安全、
惡意或具誤導性，否則請勿使用一般市集
檢舉來提出著作權或權利主張。

## 審查與執法

ClawHub 可能使用自動檢查、統計濫用訊號、使用者檢舉與
員工審查，以識別不安全內容或濫用性的發布行為。單一訊號
本身並不能證明濫用；它會協助 ClawHub 判斷哪些內容需要審查。

我們可能會：

- 隱藏、暫停、移除、軟刪除，或在資源類型支援時
  硬刪除違規清單
- 阻擋不安全版本的下載或安裝
- 撤銷 API 權杖
- 軟刪除相關內容
- 限制發布權限
- 封鎖重複或嚴重違規者

對於明顯濫用，我們不保證會先警告再執法。請參閱
[審核與帳號安全](/clawhub/moderation)，了解檢舉、審核保留、
隱藏清單、封鎖與帳號信譽。
