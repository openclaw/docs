# 可接受使用

ClawHub 託管 OpenClaw 的 Skills、外掛、套件與市集詮釋資料。
請使用本頁判斷內容或發布行為是否適合放在
ClawHub。

這些規則適用於清單的行為、要求使用者執行的內容、呈現自身的方式，以及發布者如何使用 ClawHub 的探索、安裝與信任介面。關於審核狀態與帳號狀態，請參閱
[審核與帳號安全](/clawhub/moderation)。關於著作權或其他權利
主張，請參閱[內容權利請求](/clawhub/content-rights)。

## 允許的內容

ClawHub 歡迎實用、易懂且以善意發布的內容。

| 類別 | 允許條件 |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開發者生產力 | 清單協助使用者建置、測試、遷移、偵錯、撰寫文件或營運軟體。 |
| UI、資料與自動化工作流程 | 範圍明確、所需憑證清楚列明，且高風險動作包含審查、演練、預覽或確認路徑。 |
| 防禦性安全、審核與濫用審查 | 工具以授權審查為定位、保留證據，並清楚維持人工核准邊界。 |
| 個人或團隊工作流程 | 工作流程使用基於同意的帳號、透明的設定流程與明確權限。 |
| 維護中的目錄 | 每個清單都各自獨特、實用、描述準確，且有合理維護。 |

情境很重要。同一主題在狹義的防禦性或
基於同意的情境中可以接受，但若包裝成濫用工作流程則不可接受。

## 不允許的內容

ClawHub 不託管主要目的為濫用、欺騙、不安全
執行或侵害權利的內容。

| 類別 | 不允許 |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 未授權存取或安全繞過 | 驗證繞過、帳號接管、速率限制濫用、即時通話或代理接管、可重複使用的工作階段竊取，或為未核准使用者自動核准配對流程。 |
| 平台濫用與規避封鎖 | 被封鎖後使用隱匿帳號、帳號養成或批量培養、虛假互動、多帳號自動化、大量發布、垃圾訊息機器人，或為避免偵測而建立的自動化。 |
| 詐欺、詐騙與欺騙性金融工作流程 | 假證書或發票、欺騙性付款流程、詐騙外聯、虛假社會證明、用於詐欺的合成身分工作流程，或缺乏明確人工核准的花費/收費工具。 |
| 侵犯隱私的資料補強或監控 | 為垃圾訊息抓取聯絡人、公開個人隱私資訊、跟蹤、搭配未經請求外聯的名單擷取、秘密監控、未經同意的生物辨識比對，或使用外洩資料或資料外洩轉儲。 |
| 未經同意的冒充或身分操弄 | 換臉、數位分身、複製的網紅、假人設，或其他用於冒充或誤導的工具。 |
| 明確性內容或停用安全保護的成人內容生成 | NSFW 圖像、影片或內容生成；包裝第三方 API 的成人內容封裝；或主要目的為明確性內容的清單。 |
| 隱藏、不安全或誤導性的執行需求 | 混淆的安裝命令、pipe-to-shell 安裝程式，例如下載內容在缺乏清楚可審查性的情況下用 `sh` 或 `bash` 執行、未聲明的祕密或私密金鑰需求、缺乏清楚可審查性的遠端 `npx @latest` 執行，或隱藏清單實際執行需求的詮釋資料。 |
| 侵害著作權或違反權利的素材 | 未經許可重新發布他人的 skill、外掛、文件、品牌資產或專有程式碼；違反授權條款；或冒充原作者或發布者。 |

## 不允許的市集行為

ClawHub 也會審查發布者如何使用市集。請勿使用 ClawHub
操縱探索、指標、信任訊號、審核系統或使用者
注意力。

不允許的市集行為包括：

- 大量發布低投入、重複、佔位或
  機器生成的清單，且看不出具有真實使用者價值
- 以幾乎相同的 Skills 或外掛灌爆搜尋或分類介面
- 發布數百個清單，但幾乎或完全沒有使用量、維護、來源
  清晰度或有意義的差異化
- 透過自動化、自我安裝迴圈、假帳號、協同行動、付費互動或其他非自然行為，人為灌高安裝、下載、星標或其他互動
  指標
- 建立或輪換帳號以規避審核、封鎖、發布者限制或
  市集審查
- 在所有權、來源、能力、安全狀態、
  安裝需求，或與其他專案或發布者的關係上誤導使用者
- 在未修正根本問題的情況下，反覆上傳已被隱藏、移除或封鎖
  的內容

高量發布不會自動構成濫用。當清單具有有意義的差異、描述準確、受到維護，
且由真實使用者使用時，大型目錄可以接受。當
數量伴隨單薄、重複、誤導、無人維護或
人為推廣的清單時，大型目錄就會成為信任與安全問題。

## 內容權利

如果你認為 ClawHub 上的內容侵犯你的著作權或其他權利，請使用
[內容權利請求](/clawhub/content-rights)。除非清單同時不安全、
惡意或誤導，否則請勿使用一般市集
檢舉來提出著作權或權利主張。

## 審查與執行

ClawHub 可能使用自動化檢查、統計濫用訊號、使用者檢舉與
工作人員審查來識別不安全內容或濫用性發布行為。單一訊號
本身不證明濫用；它會協助 ClawHub 判斷需要審查的事項。

我們可能會：

- 隱藏、暫停、移除、軟刪除，或在資源類型支援時，
  硬刪除違規清單
- 封鎖不安全版本的下載或安裝
- 撤銷 API 權杖
- 軟刪除相關內容
- 限制發布存取權
- 封鎖重複或嚴重違規者

對於明顯濫用，我們不保證先警告再執行。請參閱
[審核與帳號安全](/clawhub/moderation)，了解檢舉、審核暫停、
隱藏清單、封鎖與帳號狀態。
