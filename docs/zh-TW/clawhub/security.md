# 安全性 + 審核

ClawHub 開放發布，但公開列表仍會通過信任、掃描、回報與審核控制。目標很務實：協助使用者檢查他們安裝的內容、為發布者提供誤判的復原路徑，並讓濫用套件遠離公開探索。

另請參閱[可接受使用](/zh-TW/clawhub/acceptable-usage)。

## 使用者可檢查的內容

安裝技能或 plugin 前，請檢查其 ClawHub 列表中的：

- 擁有者與來源歸屬
- 最新版本與變更日誌
- 必要的環境變數或權限
- plugins 的相容性中繼資料
- 掃描或審核狀態
- 顯示時的回報、留言、星標、下載與安裝訊號

只安裝你理解並信任的內容。

## 掃描狀態

ClawHub 可能會在公開頁面與擁有者可見的診斷中顯示掃描或審核結果。

常見結果包括：

- `clean`：未發現阻擋性問題。
- `suspicious`：此版本需要謹慎或審查。
- `malicious`：此版本被視為不安全。
- `pending`：檢查尚未完成。
- `held`、`quarantined`、`revoked` 或 `hidden`：此版本未完全提供於公開安裝介面。

確切措辭可能因介面而異，但實際含義相同：如果版本被保留或阻擋，使用者不應安裝，直到擁有者解決問題或審核恢復該版本。

## Skills

技能掃描會檢查已發布的技能套件組合、中繼資料、宣告的需求與可疑指令。

ClawHub 特別注意技能宣告內容與其看似執行行為之間的不一致。例如，參照必要 API 金鑰的技能應在 `SKILL.md` 中宣告該需求，讓使用者可在安裝前看見。

掃描發現以成品為基礎。預期的提供者行為，例如宣告的 API 憑證、localhost OAuth 回呼、範圍限定的解除安裝清理、Basic Auth 編碼，或使用者選擇將檔案上傳到所述提供者，會與隱藏式憑證轉送、廣泛存取私有檔案、無關網路目的地或隱蔽瀏覽器濫用區別處理。

請參閱[技能格式](/zh-TW/clawhub/skill-format)。

## Plugins

Plugin 版本包含套件中繼資料、來源歸屬、相容性欄位與成品完整性資訊。

OpenClaw 會在安裝 ClawHub 託管的 plugins 前檢查相容性。套件記錄也可能公開摘要中繼資料，讓 OpenClaw 可驗證下載的成品。ClawScan 在審查 plugin 版本時會納入宣告的套件 `openclaw.environment` 環境/設定中繼資料，以便將宣告的執行階段需求與觀察到的行為進行比較。

## 回報

已登入的使用者可以回報技能、套件與留言。

回報應具體且可採取行動。濫用回報本身也可能導致帳號處置。

回報範例：

- 誤導性中繼資料
- 未宣告的憑證或權限需求
- 可疑安裝指令
- 詐騙留言或冒充
- 惡意註冊或商標濫用
- 違反[可接受使用](/zh-TW/clawhub/acceptable-usage)的內容

## 惡意或商標回報

ClawHub 對惡意註冊、冒充與商標相關爭議使用相同的回報與工作人員審核流程。這些回報需要足夠背景，讓工作人員能識別主張者、爭議列表與請求的處置。

請包含：

- 標準 ClawHub 技能或套件 URL 與擁有者帳號
- 涉及的商標、專案、公司或產品名稱
- 主張者擁有權或授權的公開證據
- 為何目前擁有者未獲授權以該名稱發布
- 請求的處置，例如在待審期間隱藏、轉移所有權、重新命名或移除

不要在公開回報中放入私人秘密或敏感法律文件。需要時，請開啟 GitHub issue，提供非敏感證據，並向維護者詢問私下交接路徑。

## 申訴與重新掃描

當擁有者認為技能或套件被錯誤保留或標記時，可以要求重新掃描。平台審核者與管理員在處理回報或支援請求時，可以要求重新掃描任何技能或套件：

```bash
clawhub skill rescan <slug>
clawhub package rescan <name>
```

對於受審核內容，擁有者或許可以從擁有者可見的 ClawHub 介面提交申訴。申訴應說明變更了什麼，或為何該標記不正確。

## 審核保留

當靜態掃描器將上傳的技能標記為惡意時，發布者會自動被置於審核保留狀態（使用者上設定 `requiresModerationAt`）。這會隱藏該發布者的所有技能，使未來發布一開始即為隱藏，並建立 `user.moderation.auto` 稽核日誌項目。

靜態可疑發現會保留為給審核者的檔案/行號證據，但它們本身不會隱藏內容，也不會決定公開掃描判定。新的上傳會維持在審查/待處理狀態，直到 LLM 審查完成。靜態掃描只會對惡意特徵立即阻擋。VirusTotal 引擎命中仍會作為可見的安全證據，但 VirusTotal Code Insight/Palm 判定屬於參考性質，本身不會隱藏技能。ClawScan LLM 審查會保留與目的相符的備註作為指引。中等審查發現會維持在成品上可見，而可疑篩選則保留給高影響 LLM 疑慮、惡意發現，或經 AV 引擎偵測佐證的結果。

管理員可以解除誤判保留：

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

這會清除 `requiresModerationAt` 與 `requiresModerationReason`，恢復因使用者層級保留而被隱藏的技能，並寫入 `user.moderation.lift` 稽核日誌項目。因其他原因被隱藏，或其自身靜態掃描仍為惡意的技能，會維持隱藏。

## 封鎖與帳號狀態

違反 ClawHub 政策的帳號可能失去發布權限。嚴重濫用可能導致帳號封鎖、權杖撤銷、內容隱藏或列表移除。

已刪除、被封鎖或停用的帳號無法使用 ClawHub API 權杖。如果 CLI 驗證在帳號處置後開始失敗，請登入網頁 UI 查看帳號狀態，或透過預期的專案支援管道聯絡維護者。

## 發布者指引

若要減少誤判並提升使用者信任：

- 保持名稱、摘要、標籤與變更日誌準確
- 宣告必要的環境變數與權限
- 避免混淆化的安裝命令
- 盡可能連結到來源
- 發布 plugins 前使用 dry runs
- 如果使用者或審核者詢問套件行為，請清楚回應
