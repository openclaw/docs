# 審核與帳戶安全

ClawHub 開放發布，但公開探索與安裝介面仍然
需要防護措施。檢舉、審核保留、隱藏列表和帳戶處置，
有助於在發布版本或帳戶看似不安全、具誤導性或違反
政策時保護使用者。

本頁說明審核與帳戶狀態。如需 `Pass`、`Review`、`Warn`、
`Malicious` 等稽核標籤及風險等級，請參閱
[安全稽核](/zh-TW/clawhub/security-audits)。

另請參閱[安全性](/zh-TW/clawhub/security)和
[可接受使用](/zh-TW/clawhub/acceptable-usage)。如有著作權或其他內容
權利相關疑慮，請使用[內容權利請求](/zh-TW/clawhub/content-rights)。

## 檢舉

已登入的使用者可以檢舉技能、外掛和套件。

ClawHub 檢舉僅用於不安全的市集內容，例如：

- 惡意列表
- 具誤導性的中繼資料
- 未宣告的憑證或權限需求
- 可疑的安裝指示
- 冒充身分
- 惡意註冊或商標濫用
- 違反[可接受使用](/zh-TW/clawhub/acceptable-usage)的內容

在技能頁面上使用**檢舉技能**按鈕，或針對套件使用套件檢舉
命令/API。

請勿使用 ClawHub 檢舉來回報第三方技能或
外掛自身原始碼中的弱點。請直接向列表中連結的發布者或原始碼
儲存庫回報。ClawHub 不維護或修補
第三方技能或外掛程式碼。

`openclaw/clawhub` 的 GitHub Security Advisories 適用於
ClawHub 本身的弱點。範例包括網站、API、命令列介面、登錄檔、驗證、
掃描、審核或下載/安裝信任邊界中的錯誤。請勿使用 ClawHub
advisories 來回報第三方技能或外掛中的弱點。

良好的檢舉應具體且可採取行動。濫用檢舉本身也可能導致
帳戶處置。

## 組織與命名空間主張

組織、品牌、套件範圍、擁有者帳號或命名空間所有權爭議，應
使用[組織與命名空間主張](/zh-TW/clawhub/namespace-claims)流程，而不是
產品內檢舉流程或帳戶申訴表單。

當你需要 ClawHub 工作人員審查非敏感證明，以判定某個
命名空間是否應被保留、轉移、重新命名、隱藏、隔離、設為別名，
或以其他方式審查時，請使用該流程。請勿在
公開議題中包含祕密、私人文件、私人法律文件、
個人身分文件、API 權杖或 DNS challenge 權杖。

## 審核保留

某些嚴重發現或政策問題可能會讓發布者或列表進入
審核保留狀態。發生這種情況時，受影響的內容可能會從公開
探索中隱藏，或未來發布可能會先以隱藏狀態開始，直到問題
完成審查。

審核保留的目的，是在 ClawHub 解決高風險
案例期間保護使用者。確認為誤判時，也可以解除保留。

## 隱藏或封鎖的列表

列表可能會被保留、隱藏、隔離、撤銷，或以其他方式無法在
公開安裝介面使用。

如果你看到其中一種狀態，除非擁有者解決問題或審核恢復它，
否則請勿安裝該發布版本。

擁有者仍可看到自己被保留或隱藏列表的診斷資訊。這些
診斷資訊有助於說明發生了什麼，以及列表在恢復到公開介面之前
需要變更什麼。

## 禁用與帳戶狀態

違反 ClawHub 政策的帳戶可能會失去發布權限。嚴重濫用可能
導致帳戶禁用、權杖撤銷、內容隱藏或列表移除。
發布者濫用壓力訊號會每日檢查。達到
ClawHub 潛在禁用門檻的訊號可能觸發自動警告。如果警告期限後的下一次
符合資格掃描仍將發布者置於
潛在禁用門檻，ClawHub 可能會自動套用帳戶處置。
較低信心度與有時間範圍限制的審查訊號不會進入自動
執行。

已刪除、被禁用或停用的帳戶無法使用 ClawHub API 權杖。如果命令列介面驗證
在帳戶處置後開始失敗，請登入網頁 UI 以查看帳戶
狀態。如果登入或一般命令列介面存取因禁用或停用帳戶而被封鎖，
請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/)進行復原審查。

如果掃描器觸發的電子郵件將某個技能或外掛版本標示為惡意，
請下載被封鎖提交版本的已儲存掃描結果：
`clawhub scan download <slug> --version <version>`。對於外掛，請加上
`--kind plugin`。檢閱掃描輸出、修正列表、遞增版本
號碼，然後上傳修正後的版本。

## 發布者指南

若要減少誤判並提升使用者信任：

- 保持名稱、摘要、標籤和變更記錄準確
- 宣告必要的環境變數和權限
- 避免混淆處理過的安裝命令
- 盡可能連結到原始碼
- 發布外掛前使用 dry runs
- 如果使用者或審核人員詢問發布版本行為，請清楚回應
