# 審核與帳號安全

ClawHub 開放發布，但公開探索與安裝介面仍然需要防護措施。檢舉、審核暫停、隱藏清單與帳號處置，有助於在某個版本或帳號看起來不安全、具誤導性或不符合政策時保護使用者。

本頁說明審核與帳號狀態。如需了解 `Pass`、`Review`、`Warn`、`Malicious` 與風險等級等稽核標籤，請參閱 [安全性稽核](/clawhub/security-audits)。

另請參閱 [安全性](/clawhub/security) 與 [可接受使用](/clawhub/acceptable-usage)。若有著作權或其他內容權利疑慮，請使用 [內容權利請求](/clawhub/content-rights)。

## 檢舉

已登入的使用者可以檢舉 Skills、外掛與套件。

ClawHub 檢舉僅用於不安全的市集內容，例如：

- 惡意清單
- 具誤導性的中繼資料
- 未宣告的憑證或權限需求
- 可疑的安裝指示
- 冒充
- 惡意註冊或商標濫用
- 違反 [可接受使用](/clawhub/acceptable-usage) 的內容

在 Skill 頁面使用 **檢舉 Skill** 按鈕，或對套件使用套件檢舉命令/API。

請勿使用 ClawHub 檢舉來通報第三方 Skill 或外掛自身原始碼中的漏洞。請直接向清單中連結的發布者或原始碼儲存庫通報。ClawHub 不維護或修補第三方 Skill 或外掛程式碼。

`openclaw/clawhub` 的 GitHub 安全性公告適用於 ClawHub 本身的漏洞。範例包括網站、API、命令列介面、登錄檔、驗證、掃描、審核或下載/安裝信任邊界中的錯誤。請勿將 ClawHub 公告用於第三方 Skills 或外掛中的漏洞。

好的檢舉應具體且可採取行動。濫用檢舉本身也可能導致帳號處置。

## 組織與命名空間聲明

組織、品牌、套件範圍、擁有者帳號或命名空間所有權爭議，應使用 [組織與命名空間聲明](/clawhub/namespace-claims) 流程，而不是產品內檢舉流程或帳號申訴表單。

當你需要 ClawHub 工作人員審查非敏感證明，以判定某個命名空間應被保留、移轉、重新命名、隱藏、隔離、設為別名或以其他方式審查時，請使用該流程。請勿在公開 issue 中包含秘密、私人文件、私人法律文件、個人身分文件、API token 或 DNS challenge token。

## 審核暫停

某些嚴重發現或政策問題可能會讓發布者或清單進入審核暫停狀態。發生這種情況時，受影響的內容可能會從公開探索中隱藏，或未來發布可能會先以隱藏狀態開始，直到問題完成審查。

審核暫停旨在 ClawHub 解決高風險案例時保護使用者。若確認為誤判，也可以解除暫停。

## 隱藏或封鎖的清單

清單可能會被暫停、隱藏、隔離、撤銷，或以其他方式無法出現在公開安裝介面上。

如果你看到其中一種狀態，除非擁有者解決問題或審核恢復清單，否則請勿安裝該版本。

擁有者仍可能看見其自身被暫停或隱藏清單的診斷資訊。這些診斷有助於說明發生了什麼，以及清單回到公開介面之前需要變更什麼。

## 封鎖與帳號狀態

違反 ClawHub 政策的帳號可能會失去發布權限。嚴重濫用可能導致帳號封鎖、token 撤銷、內容隱藏或清單移除。發布者濫用壓力訊號每日檢查。達到 ClawHub 潛在封鎖門檻的訊號可能會觸發自動警告。如果警告期限後下一次符合條件的掃描仍將發布者置於潛在封鎖門檻內，ClawHub 可能會自動套用帳號處置。較低信心與有時間範圍限制的審查訊號不會納入自動執法。

已刪除、遭封鎖或停用的帳號無法使用 ClawHub API token。如果命令列介面驗證在帳號處置後開始失敗，請登入網頁 UI 檢視帳號狀態。如果登入或一般命令列介面存取因封鎖或帳號停用而受阻，請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/) 進行復原審查。

如果掃描器觸發的電子郵件指出某個 Skill 或外掛版本為惡意，請下載被封鎖提交版本的已儲存掃描結果：
`clawhub scan download <slug> --version <version>`。若為外掛，請加上 `--kind plugin`。檢視掃描輸出，修正清單，提高版本號，並上傳修正後的版本。

## 發布者指南

若要減少誤判並提升使用者信任：

- 保持名稱、摘要、標籤與變更記錄準確
- 宣告必要的環境變數與權限
- 避免混淆處理過的安裝命令
- 盡可能連結至原始碼
- 發布外掛前先使用 dry runs
- 如果使用者或審核人員詢問版本行為，請清楚回應
