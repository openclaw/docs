# 管理措施與帳號安全

ClawHub 開放發布內容，但公開探索與安裝介面仍需要防護措施。檢舉、管理暫停、隱藏上架項目及帳號處置，有助於在版本或帳號疑似不安全、具誤導性或違反政策時保護使用者。

本頁說明管理措施與帳號狀態。若要瞭解 `Pass`、`Review`、`Warn`、`Malicious` 等稽核標籤及風險等級，請參閱[安全稽核](/clawhub/security-audits)。

另請參閱[安全性](/clawhub/security)與[可接受使用規範](/clawhub/acceptable-usage)。若對著作權或其他內容權利有疑慮，請使用[內容權利請求](/clawhub/content-rights)。

## 檢舉

已登入的使用者可以檢舉 Skills、外掛及套件。

ClawHub 檢舉僅適用於不安全的市集內容，例如：

- 惡意上架項目
- 具誤導性的中繼資料
- 未申明的認證資訊或權限需求
- 可疑的安裝指示
- 冒充他人
- 惡意搶註或濫用商標
- 違反[可接受使用規範](/clawhub/acceptable-usage)的內容

請使用 Skills 頁面上的 **Report skill** 按鈕，或套件的檢舉命令／API。

請勿使用 ClawHub 檢舉來通報第三方 Skills 或外掛本身原始碼中的弱點。請直接向上架項目所連結的發布者或原始碼儲存庫通報。ClawHub 不會維護或修補第三方 Skills 或外掛的程式碼。

`openclaw/clawhub` 的 GitHub Security Advisories 適用於 ClawHub 本身的弱點，例如網站、API、命令列介面、登錄檔、驗證、掃描、管理措施或下載／安裝信任邊界中的錯誤。請勿使用 ClawHub 安全性公告通報第三方 Skills 或外掛中的弱點。

良好的檢舉應具體且可據以處理。濫用檢舉功能本身也可能導致帳號處置。

## 組織與命名空間申索

組織、品牌、套件範圍、擁有者代號或命名空間的所有權爭議，應使用[組織與命名空間申索](/clawhub/namespace-claims)流程，而非產品內的檢舉流程或帳號申訴表單。

若你需要 ClawHub 工作人員審查非敏感證明，以判定命名空間是否應予保留、移轉、重新命名、隱藏、隔離、設為別名或接受其他審查，請使用該流程。請勿在公開議題中提供密鑰、私人文件、非公開法律文件、個人身分證明文件、API 權杖或 DNS 驗證權杖。

## 管理暫停

某些嚴重的調查結果或政策問題，可能使發布者或上架項目遭到管理暫停。發生此情況時，受影響的內容可能會從公開探索中隱藏，或之後發布的內容在問題完成審查前皆預設為隱藏。

管理暫停旨在讓 ClawHub 處理高風險案例期間保護使用者。確認為誤判後，也可以解除管理暫停。

## 隱藏或封鎖的上架項目

上架項目可能會遭到暫停、隱藏、隔離、撤銷，或以其他方式無法從公開安裝介面取得。

若你看到其中任一狀態，除非擁有者已解決問題或管理人員已恢復該版本，否則請勿安裝。

擁有者仍可能看到自己遭暫停或隱藏之上架項目的診斷資訊。這些診斷資訊有助於說明發生了什麼事，以及必須進行哪些變更，上架項目才能重新出現在公開介面。

## 封禁與帳號狀態

違反 ClawHub 政策的帳號可能會失去發布權限。嚴重濫用可能導致帳號遭封禁、權杖遭撤銷、內容遭隱藏或上架項目遭移除。系統每天都會檢查發布者的濫用壓力訊號。達到 ClawHub 潛在封禁門檻的訊號可能觸發自動警告。若警告期限過後的下一次符合條件之掃描，仍將發布者判定為達到潛在封禁門檻，ClawHub 可能會自動執行帳號處置。可信度較低及有時間範圍限制的審查訊號不會納入自動執行處置。

已刪除、遭封禁或停用的帳號無法使用 ClawHub API 權杖。若在帳號處置後，命令列介面驗證開始失敗，請登入網頁使用者介面查看帳號狀態。若因帳號遭封禁或停用而無法登入或正常使用命令列介面，請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/)申請復原審查。

若掃描器觸發的電子郵件指出某個 Skills 或外掛版本具有惡意行為，請下載遭封鎖之已提交版本所儲存的掃描結果：`clawhub scan download <slug> --version <version>`。若為外掛，請加上 `--kind plugin`。檢閱掃描輸出、修正上架項目、增加版本號碼，然後上傳修正後的版本。

## 發布者指南

若要減少誤判並提升使用者信任：

- 確保名稱、摘要、標籤及變更記錄正確
- 申明必要的環境變數與權限
- 避免使用經過混淆的安裝命令
- 可行時連結至原始碼
- 發布外掛前先使用試執行
- 若使用者或管理人員詢問版本行為，請清楚回覆
