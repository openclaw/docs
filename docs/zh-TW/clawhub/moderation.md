# 管理審核與帳號安全

ClawHub 開放發布，但公開探索與安裝介面仍需要防護措施。檢舉、管理審核暫停、隱藏列表項目與帳號處置，有助於在發行版本或帳號看似不安全、誤導或違反政策時保護使用者。

本頁說明管理審核與帳號狀態。若要了解 `Pass`、`Review`、`Warn`、`Malicious` 等稽核標籤與風險等級，請參閱
[安全稽核](/clawhub/security-audits)。

另請參閱[安全](/clawhub/security)與
[可接受使用](/clawhub/acceptable-usage)。如有著作權或其他內容權利疑慮，請使用[內容權利請求](/clawhub/content-rights)。

## 檢舉

已登入的使用者可以檢舉 Skills、外掛與套件。

ClawHub 檢舉僅適用於不安全的市集內容，例如：

- 惡意列表項目
- 誤導性中繼資料
- 未宣告的憑證或權限需求
- 可疑的安裝指示
- 冒名
- 惡意註冊或商標濫用
- 違反[可接受使用](/clawhub/acceptable-usage)的內容

請在 skill 頁面上使用**檢舉 skill**按鈕，或使用套件的檢舉命令/API。

請勿使用 ClawHub 檢舉來通報第三方 skill 或外掛自身原始碼中的弱點。請直接通報給列表項目連結的發布者或原始碼儲存庫。ClawHub 不維護或修補第三方 skill 或外掛程式碼。

`openclaw/clawhub` 的 GitHub Security Advisories 適用於 ClawHub 本身的弱點。範例包括網站、API、命令列介面、登錄檔、驗證、掃描、管理審核或下載/安裝信任邊界中的錯誤。請勿將 ClawHub 安全公告用於第三方 Skills 或外掛中的弱點。

好的檢舉應具體且可執行。濫用檢舉本身也可能導致帳號處置。

## 組織與命名空間主張

組織、品牌、套件範圍、擁有者控制代碼或命名空間擁有權爭議，應使用[組織與命名空間主張](/clawhub/namespace-claims)流程，而不是產品內檢舉流程或帳號申訴表單。

當你需要 ClawHub 工作人員審查非敏感證明，以判定某個命名空間是否應保留、轉移、重新命名、隱藏、隔離、設為別名或以其他方式審查時，請使用該流程。請勿在公開 issue 中包含密鑰、私人文件、私人法律文件、個人身分文件、API 權杖或 DNS challenge token。

## 管理審核暫停

某些嚴重發現或政策問題可能會讓發布者或列表項目進入管理審核暫停狀態。發生這種情況時，受影響內容可能會從公開探索中隱藏，或未來發布可能會先以隱藏狀態開始，直到問題完成審查。

管理審核暫停旨在 ClawHub 解決高風險案例期間保護使用者。當確認為誤判時，也可以解除暫停。

## 隱藏或封鎖的列表項目

列表項目可能會被暫停、隱藏、隔離、撤銷，或在公開安裝介面上以其他方式無法使用。

如果你看到其中一種狀態，除非擁有者解決問題或管理審核恢復該項目，否則請勿安裝該發行版本。

擁有者仍可能看到自己被暫停或隱藏列表項目的診斷資訊。這些診斷資訊有助於說明發生了什麼，以及列表項目返回公開介面前需要變更什麼。

## 封禁與帳號狀態

違反 ClawHub 政策的帳號可能會失去發布存取權。嚴重濫用可能導致帳號封禁、權杖撤銷、內容隱藏或列表項目移除。發布者濫用壓力訊號會每日檢查。達到 ClawHub 潛在封禁門檻的訊號可能會觸發自動警告。如果警告期限後下一次符合資格的掃描仍將發布者置於潛在封禁門檻，ClawHub 可能會自動套用帳號處置。較低信心度與有界限時間性的審查訊號不會納入自動執法。

已刪除、遭封禁或停用的帳號無法使用 ClawHub API 權杖。如果帳號處置後命令列介面驗證開始失敗，請登入網頁 UI 檢視帳號狀態。如果登入或一般命令列介面存取因封禁或停用帳號而遭封鎖，請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/) 進行復原審查。

如果掃描器觸發的電子郵件將某個 skill 或外掛版本標示為惡意，請下載遭封鎖提交版本的已儲存掃描結果：
`clawhub scan download <slug> --version <version>`。若為外掛，請加上
`--kind plugin`。檢閱掃描輸出、修正列表項目、遞增版本號，並上傳修正後的版本。

## 發布者指南

為了減少誤判並提升使用者信任：

- 保持名稱、摘要、標籤與變更記錄準確
- 宣告必要的環境變數與權限
- 避免混淆的安裝命令
- 盡可能連結至原始碼
- 發布外掛前使用 dry run
- 如果使用者或審核員詢問發行版本行為，請清楚回應
