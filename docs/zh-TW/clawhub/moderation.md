# 內容審核與帳號安全

ClawHub 開放發布內容，但公開探索與安裝介面仍需要防護措施。檢舉、內容審核暫停、隱藏項目與帳號處置，有助於在版本或帳號看似不安全、具有誤導性或不符合政策時保護使用者。

本頁說明內容審核與帳號狀態。如需瞭解 `Pass`、`Review`、`Warn`、`Malicious` 與風險等級等稽核標籤，請參閱
[安全稽核](/clawhub/security-audits)。

另請參閱[安全性](/clawhub/security)與
[可接受的使用方式](/zh-TW/clawhub/acceptable-usage)。如有著作權或其他內容權利疑慮，請使用[內容權利申請](/zh-TW/clawhub/content-rights)。

## 檢舉

已登入的使用者可以檢舉 Skills、外掛與套件。

ClawHub 檢舉僅適用於不安全的市集內容，例如：

- 惡意項目
- 誤導性中繼資料
- 未宣告的認證資訊或權限要求
- 可疑的安裝指示
- 冒充行為
- 惡意註冊或濫用商標
- 違反[可接受的使用方式](/zh-TW/clawhub/acceptable-usage)的內容

請使用 Skill 頁面上的 **檢舉 Skill** 按鈕，或使用套件檢舉命令/API 來檢舉套件。

請勿使用 ClawHub 檢舉功能回報第三方 Skill 或外掛本身原始碼中的漏洞。請直接向發布者或項目中連結的原始碼儲存庫回報。ClawHub 不會維護或修補第三方 Skill 或外掛程式碼。

`openclaw/clawhub` 的 GitHub Security Advisories 適用於 ClawHub 本身的漏洞。例如網站、API、命令列介面、登錄檔、驗證、掃描、內容審核或下載/安裝信任邊界中的錯誤。請勿使用 ClawHub 安全性公告回報第三方 Skills 或外掛中的漏洞。

良好的檢舉應具體且可採取行動。濫用檢舉功能本身也可能導致帳號處置。

## 組織與命名空間聲明

組織、品牌、套件範圍、擁有者帳號名稱或命名空間的所有權爭議，應使用[組織與命名空間聲明](/clawhub/namespace-claims)流程，而非產品內的檢舉流程或帳號申訴表單。

需要 ClawHub 工作人員審查命名空間是否應予保留、移轉、重新命名、隱藏、隔離、設定別名或進行其他處理時，請使用該流程提交非敏感的證明。請勿在公開議題中包含祕密、私人文件、非公開法律文件、個人身分證明文件、API 權杖或 DNS 挑戰權杖。

## 內容審核暫停

某些嚴重發現或政策問題可能會使發布者或項目遭到內容審核暫停。發生這種情況時，受影響的內容可能會從公開探索中隱藏，或在問題獲得審查前，後續發布的內容一開始即為隱藏狀態。

內容審核暫停旨在 ClawHub 處理高風險案例期間保護使用者。如果確認為誤判，也可以解除暫停。

## 隱藏或封鎖的項目

項目可能會遭到暫停、隱藏、隔離、撤銷，或以其他方式無法透過公開安裝介面取得。

如果看到其中任何一種狀態，除非擁有者解決問題或內容審核人員恢復該版本，否則請勿安裝。

擁有者仍可能看到自己遭暫停或隱藏項目的診斷資訊。這些診斷資訊有助於說明發生了什麼事，以及該項目必須進行哪些變更才能重新出現在公開介面。

## 封禁與帳號狀態

違反 ClawHub 政策的帳號可能會失去發布權限。嚴重濫用可能導致帳號遭封禁、權杖被撤銷、內容遭隱藏或項目遭移除。系統每日檢查發布者濫用壓力訊號。達到 ClawHub 潛在封禁門檻的訊號可能會觸發自動警告。如果警告期限後的下一次合格掃描仍將發布者判定為達到潛在封禁門檻，ClawHub 可能會自動執行帳號處置。信心較低且限於特定時間範圍的審查訊號不會納入自動執法。

已刪除、遭封禁或停用的帳號無法使用 ClawHub API 權杖。如果帳號遭處置後，命令列介面驗證開始失敗，請登入網頁 UI 查看帳號狀態。如果因帳號遭封禁或停用而無法登入或正常使用命令列介面，請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/)申請復原審查。

如果掃描器觸發的電子郵件指出某個 Skill 或外掛版本具有惡意，請下載遭封鎖之已提交版本的儲存掃描結果：
`clawhub scan download <slug> --version <version>`。若為外掛，請加上
`--kind plugin`。檢查掃描輸出、修正項目、遞增版本號，然後上傳修正後的版本。

## 發布者指南

若要減少誤判並提高使用者信任：

- 確保名稱、摘要、標籤與變更記錄準確無誤
- 宣告必要的環境變數與權限
- 避免使用經過混淆的安裝命令
- 盡可能連結至原始碼
- 發布外掛前先使用試執行
- 如果使用者或內容審核人員詢問版本行為，請清楚回覆
