# 審核與帳號安全

ClawHub 開放發布，但公開探索與安裝介面仍然需要防護措施。回報、審核暫停、隱藏列表與帳號處置，有助於在發行版本或帳號看似不安全、具誤導性或違反政策時保護使用者。

本頁說明審核與帳號狀態。若要了解 `Pass`、`Review`、`Warn`、`Malicious` 等稽核標籤與風險等級，請參閱[安全稽核](/zh-TW/clawhub/security-audits)。

另請參閱[安全性](/zh-TW/clawhub/security)與[可接受使用](/zh-TW/clawhub/acceptable-usage)。若有著作權或其他內容權利疑慮，請使用[內容權利請求](/zh-TW/clawhub/content-rights)。

## 回報

已登入的使用者可以回報 Skills、外掛與套件。

僅將 ClawHub 回報用於不安全的市集內容，例如：

- 惡意列表
- 具誤導性的中繼資料
- 未宣告的憑證或權限需求
- 可疑的安裝指示
- 冒名
- 惡意註冊或商標濫用
- 違反[可接受使用](/zh-TW/clawhub/acceptable-usage)的內容

在 Skills 頁面使用 **回報 Skill** 按鈕，或針對套件使用套件回報命令/API。

請勿使用 ClawHub 回報第三方 Skill 或外掛自身原始碼中的漏洞。請直接向發布者或列表中連結的原始碼儲存庫回報。ClawHub 不維護或修補第三方 Skill 或外掛程式碼。

`openclaw/clawhub` 的 GitHub Security Advisories 用於 ClawHub 本身的漏洞。範例包括網站、API、命令列介面、登錄、驗證、掃描、審核或下載/安裝信任邊界中的錯誤。請勿使用 ClawHub advisories 回報第三方 Skills 或外掛中的漏洞。

良好的回報應具體且可執行。濫用回報功能本身也可能導致帳號處置。

## 組織與命名空間主張

組織、品牌、套件範圍、擁有者帳號或命名空間所有權爭議，應使用[組織與命名空間主張](/zh-TW/clawhub/namespace-claims)流程，而不是產品內回報流程或帳號申訴表單。

當你需要 ClawHub 工作人員審查非敏感證明，以判斷命名空間是否應被保留、轉移、重新命名、隱藏、隔離、設定別名或以其他方式審查時，請使用該流程。請勿在公開議題中包含秘密、私人文件、私人法律文件、個人身分文件、API 權杖或 DNS challenge 權杖。

## 審核暫停

某些嚴重發現或政策問題，可能會讓發布者或列表進入審核暫停狀態。發生這種情況時，受影響內容可能會從公開探索中隱藏，或未來發布可能會先以隱藏狀態開始，直到問題完成審查。

審核暫停旨在 ClawHub 處理高風險案例時保護使用者。若確認為誤判，也可以解除暫停。

## 隱藏或封鎖的列表

列表可能會被暫停、隱藏、隔離、撤銷，或以其他方式無法在公開安裝介面上取得。

如果你看到其中一種狀態，請勿安裝該發行版本，除非擁有者解決問題或審核恢復該版本。

擁有者仍可查看其被暫停或隱藏列表的診斷資訊。這些診斷資訊有助於說明發生了什麼，以及列表在回到公開介面前需要變更什麼。

## 封鎖與帳號狀態

違反 ClawHub 政策的帳號可能會失去發布存取權。嚴重濫用可能導致帳號封鎖、權杖撤銷、內容隱藏或列表移除。發布者濫用壓力訊號會每日檢查。達到 ClawHub 潛在封鎖門檻的訊號可能會觸發自動警告。如果警告期限後下一次符合條件的掃描仍使發布者落在潛在封鎖門檻內，ClawHub 可能會自動套用帳號處置。較低信心與有時間範圍限制的審查訊號不會納入自動執行。

已刪除、被封鎖或停用的帳號無法使用 ClawHub API 權杖。如果帳號處置後命令列介面驗證開始失敗，請登入網頁 UI 查看帳號狀態。如果登入或一般命令列介面存取因封鎖或停用帳號而受阻，請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/) 進行復原審查。

如果掃描器觸發的電子郵件將某個 Skill 或外掛版本標示為惡意，請下載被封鎖提交版本的已儲存掃描結果：
`clawhub scan download <slug> --version <version>`。若為外掛，請加上
`--kind plugin`。檢查掃描輸出，修正列表，遞增版本號，並上傳修正後的版本。

## 發布者指引

為了減少誤判並提升使用者信任：

- 保持名稱、摘要、標籤與變更記錄準確
- 宣告必要的環境變數與權限
- 避免混淆過的安裝命令
- 盡可能連結到原始碼
- 發布外掛前使用 dry run
- 如果使用者或審核人員詢問發行版本行為，請清楚回應
