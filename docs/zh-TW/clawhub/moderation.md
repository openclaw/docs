# 審核與帳號安全

ClawHub 開放發布，但公開探索與安裝介面仍然
需要防護措施。檢舉、審核暫停、隱藏列表與帳號處置
有助於在某個版本或帳號看起來不安全、具誤導性或違反
政策時保護使用者。

本頁說明審核與帳號狀態。若要了解稽核標籤，例如
`Pass`、`Review`、`Warn`、`Malicious` 與風險等級，請參閱
[安全稽核](/clawhub/security-audits)。

另請參閱[安全性](/clawhub/security)與
[可接受使用](/clawhub/acceptable-usage)。若有著作權或其他內容
權利疑慮，請使用[內容權利請求](/clawhub/content-rights)。

## 檢舉

已登入的使用者可以檢舉技能、外掛與套件。

ClawHub 檢舉僅適用於不安全的市集內容，例如：

- 惡意列表
- 具誤導性的中繼資料
- 未宣告的憑證或權限需求
- 可疑的安裝指示
- 冒充
- 惡意註冊或商標濫用
- 違反[可接受使用](/clawhub/acceptable-usage)的內容

請在技能頁面使用 **檢舉技能** 按鈕，或使用套件的檢舉
命令/API。

請勿使用 ClawHub 檢舉來回報第三方技能或
外掛自身原始碼中的漏洞。請直接向列表中連結的發布者或原始碼
儲存庫回報。ClawHub 不維護或修補
第三方技能或外掛程式碼。

`openclaw/clawhub` 的 GitHub Security Advisories 適用於
ClawHub 本身的漏洞。例如網站、API、命令列介面、登錄檔、驗證、
掃描、審核或下載/安裝信任邊界中的錯誤。請勿使用 ClawHub
advisories 回報第三方技能或外掛中的漏洞。

良好的檢舉應具體且可執行。濫用檢舉本身也可能導致
帳號處置。

## 組織與命名空間主張

組織、品牌、套件範圍、擁有者帳號或命名空間所有權爭議，應
使用[組織與命名空間主張](/clawhub/namespace-claims)流程，而不是
產品內檢舉流程或帳號申訴表單。

當你需要 ClawHub 工作人員審查非敏感證明，以判定某個
命名空間是否應保留、轉移、重新命名、隱藏、隔離、設為別名
或以其他方式審查時，請使用該流程。請勿在
公開 issue 中包含祕密、私人文件、私人法律文件、個人身分文件、
API token 或 DNS challenge token。

## 審核暫停

某些嚴重發現或政策問題可能會使發布者或列表進入
審核暫停狀態。發生這種情況時，受影響的內容可能會從公開
探索中隱藏，或未來發布可能會先以隱藏狀態開始，直到問題經過審查。

審核暫停旨在讓 ClawHub 解決高風險
案件時保護使用者。若確認為誤判，也可以解除審核暫停。

## 隱藏或封鎖的列表

列表可能會被暫停、隱藏、隔離、撤銷，或以其他方式無法在
公開安裝介面使用。

如果看到這些狀態之一，除非擁有者
解決問題或審核恢復該版本，否則請勿安裝該版本。

擁有者仍可看到自己被暫停或隱藏列表的診斷資訊。這些
診斷資訊有助於說明發生了什麼，以及列表返回公開介面前
需要變更哪些內容。

## 停權與帳號狀態

違反 ClawHub 政策的帳號可能會失去發布權限。嚴重濫用可能會
導致帳號停權、token 撤銷、內容隱藏或列表移除。
發布者濫用壓力訊號會每日檢查。達到
ClawHub 潛在停權門檻的訊號可能會觸發自動警告。若警告期限後的下一次
符合資格掃描仍將發布者置於
潛在停權門檻，ClawHub 可能會自動套用帳號處置。
較低信心度與有時間界限的審查訊號不會納入自動
執行。

已刪除、被停權或停用的帳號無法使用 ClawHub API token。如果命令列介面驗證
在帳號處置後開始失敗，請登入網頁 UI 檢查帳號
狀態。如果因停權或帳號停用而無法登入或使用一般命令列介面存取，
請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/) 進行復原審查。

如果掃描器觸發的電子郵件將某個技能或外掛版本標示為惡意，
請下載被封鎖提交版本的已儲存掃描結果：
`clawhub scan download <slug> --version <version>`。對於外掛，請加上
`--kind plugin`。檢查掃描輸出、修正列表、遞增版本
號碼，並上傳修正後的版本。

## 發布者指南

若要減少誤判並提升使用者信任：

- 保持名稱、摘要、標籤與變更記錄準確
- 宣告所需的環境變數與權限
- 避免使用混淆的安裝命令
- 可行時連結到原始碼
- 發布外掛前使用試跑
- 如果使用者或審核人員詢問版本行為，請清楚回應
