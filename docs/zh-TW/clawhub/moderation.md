# 內容審核與帳戶安全

ClawHub 開放發布，但公開探索與安裝介面仍需要防護機制。回報、內容審核暫停、隱藏上架項目與帳戶處置，可在某個發布版本或帳戶看起來不安全、具誤導性或違反政策時協助保護使用者。

本頁說明內容審核與帳戶狀態。若要了解 `Pass`、`Review`、`Warn`、`Malicious` 等稽核標籤與風險等級，請參閱 [安全性稽核](/clawhub/security-audits)。

另請參閱 [安全性](/clawhub/security) 與 [可接受使用](/clawhub/acceptable-usage)。如有著作權或其他內容權利疑慮，請使用 [內容權利請求](/clawhub/content-rights)。

## 回報

已登入的使用者可以回報 Skills、外掛與套件。

ClawHub 回報僅適用於不安全的市集內容，例如：

- 惡意上架項目
- 具誤導性的中繼資料
- 未聲明的憑證或權限要求
- 可疑的安裝指示
- 冒充
- 惡意註冊或商標誤用
- 違反 [可接受使用](/clawhub/acceptable-usage) 的內容

請在 Skills 頁面上使用 **回報 Skills** 按鈕，或針對套件使用套件回報命令/API。

請勿使用 ClawHub 回報第三方 Skills 或外掛自身原始碼中的弱點。請直接向發布者或上架項目連結的原始碼儲存庫回報。ClawHub 不維護或修補第三方 Skills 或外掛程式碼。

`openclaw/clawhub` 的 GitHub Security Advisories 適用於 ClawHub 本身的弱點。範例包括網站、API、命令列介面、登錄檔、驗證、掃描、內容審核或下載/安裝信任邊界中的錯誤。請勿使用 ClawHub advisories 回報第三方 Skills 或外掛中的弱點。

良好的回報應具體且可行。濫用回報本身也可能導致帳戶處置。

## 組織與命名空間主張

組織、品牌、套件範圍、擁有者帳號或命名空間所有權爭議，應使用 [組織與命名空間主張](/clawhub/namespace-claims) 流程，而非產品內回報流程或帳戶申訴表單。

當你需要 ClawHub 工作人員審查非敏感證明，以判定某個命名空間是否應保留、轉移、重新命名、隱藏、隔離、設定別名或以其他方式審查時，請使用該流程。請勿在公開議題中包含密鑰、私人文件、私人法律文件、個人身分文件、API 權杖或 DNS challenge 權杖。

## 內容審核暫停

某些嚴重發現或政策問題可能會使發布者或上架項目進入內容審核暫停狀態。發生此情況時，受影響的內容可能會從公開探索中隱藏，或未來發布可能會先以隱藏狀態開始，直到問題完成審查。

內容審核暫停的目的，是在 ClawHub 處理高風險案例期間保護使用者。若確認為誤判，也可以解除暫停。

## 隱藏或封鎖的上架項目

上架項目可能會被暫停、隱藏、隔離、撤銷，或以其他方式在公開安裝介面上無法使用。

如果你看到其中一種狀態，除非擁有者已解決問題或內容審核已恢復該項目，否則請勿安裝該發布版本。

擁有者仍可查看其被暫停或隱藏上架項目的診斷資訊。這些診斷可協助說明發生了什麼，以及上架項目恢復到公開介面之前需要變更什麼。

## 封鎖與帳戶狀態

違反 ClawHub 政策的帳戶可能會失去發布權限。嚴重濫用可能導致帳戶封鎖、權杖撤銷、內容隱藏或上架項目移除。發布者濫用壓力訊號會每日檢查。達到 ClawHub 潛在封鎖門檻的訊號，可能會觸發自動警告。如果警告期限後的下一次符合資格掃描仍將該發布者置於潛在封鎖門檻，ClawHub 可能會自動套用帳戶處置。較低信心度與有時間範圍的審查訊號，不會進入自動執行。

已刪除、被封鎖或停用的帳戶無法使用 ClawHub API 權杖。如果帳戶處置後命令列介面驗證開始失敗，請登入網頁 UI 查看帳戶狀態。如果登入或一般命令列介面存取因封鎖或停用帳戶而受阻，請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/) 進行復原審查。

如果掃描器觸發的電子郵件將某個 Skills 或外掛版本標記為惡意，請下載被封鎖提交版本的已儲存掃描結果：
`clawhub scan download <slug> --version <version>`。外掛請加上 `--kind plugin`。檢閱掃描輸出、修正上架項目、遞增版本號，然後上傳已修正版本。

## 發布者指南

為降低誤判並提升使用者信任：

- 保持名稱、摘要、標籤與變更日誌準確
- 聲明所需的環境變數與權限
- 避免混淆處理過的安裝命令
- 盡可能連結到原始碼
- 發布外掛前使用試執行
- 如果使用者或內容審核人員詢問發布版本行為，請清楚回應
