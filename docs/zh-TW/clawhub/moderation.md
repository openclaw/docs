# 審核與帳戶安全

ClawHub 開放發布，但公開探索與安裝介面仍需要防護機制。檢舉、審核暫停、隱藏清單與帳戶處置，可在版本或帳戶看似不安全、具誤導性或違反政策時協助保護使用者。

本頁說明審核與帳戶狀態。若要了解 `Pass`、`Review`、`Warn`、`Malicious` 與風險等級等稽核標籤，請參閱[安全稽核](/zh-TW/clawhub/security-audits)。

另請參閱[安全性](/zh-TW/clawhub/security)與[可接受使用](/zh-TW/clawhub/acceptable-usage)。若有著作權或其他內容權利疑慮，請使用[內容權利請求](/zh-TW/clawhub/content-rights)。

## 檢舉

已登入的使用者可以檢舉 skill、外掛與套件。

ClawHub 檢舉僅適用於不安全的市集內容，例如：

- 惡意清單
- 具誤導性的中繼資料
- 未宣告的憑證或權限需求
- 可疑的安裝指示
- 冒名
- 惡意註冊或商標濫用
- 違反[可接受使用](/zh-TW/clawhub/acceptable-usage)的內容

在 skill 頁面使用 **檢舉 skill** 按鈕，或針對套件使用套件檢舉命令/API。

請勿使用 ClawHub 檢舉來通報第三方 skill 或外掛自身原始碼中的漏洞。請直接通報給清單連結的發布者或原始碼儲存庫。ClawHub 不維護或修補第三方 skill 或外掛程式碼。

`openclaw/clawhub` 的 GitHub Security Advisories 適用於 ClawHub 本身的漏洞。例如網站、API、命令列介面、登錄服務、認證、掃描、審核或下載/安裝信任邊界中的錯誤。請勿使用 ClawHub advisory 通報第三方 skill 或外掛中的漏洞。

良好的檢舉應具體且可執行。濫用檢舉本身也可能導致帳戶處置。

## 組織與命名空間主張

組織、品牌、套件範圍、擁有者帳號或命名空間所有權爭議，應使用[組織與命名空間主張](/zh-TW/clawhub/namespace-claims)流程，而不是產品內檢舉流程或帳戶申訴表單。

當你需要 ClawHub 工作人員審查非敏感證據，以判斷某個命名空間是否應被保留、移轉、重新命名、隱藏、隔離、建立別名或以其他方式審查時，請使用該流程。請勿在公開 issue 中包含密鑰、私人文件、私人法律檔案、個人身分文件、API 權杖或 DNS challenge 權杖。

## 審核暫停

某些嚴重發現或政策問題可能會讓發布者或清單進入審核暫停狀態。發生這種情況時，受影響內容可能會從公開探索中隱藏，或未來發布可能會先以隱藏狀態開始，直到問題完成審查。

審核暫停旨在於 ClawHub 處理高風險案例期間保護使用者。若確認為誤判，也可以解除審核暫停。

## 隱藏或封鎖的清單

清單可能會被暫停、隱藏、隔離、撤銷，或以其他方式無法在公開安裝介面上使用。

如果你看到其中一種狀態，除非擁有者解決問題或審核恢復該清單，否則請勿安裝該版本。

擁有者仍可能看到自己被暫停或隱藏清單的診斷資訊。這些診斷資訊有助於說明發生了什麼，以及清單在回到公開介面之前需要做出哪些變更。

## 封鎖與帳戶狀態

違反 ClawHub 政策的帳戶可能會失去發布權限。嚴重濫用可能導致帳戶封鎖、權杖撤銷、內容隱藏或清單移除。發布者濫用壓力訊號會每日檢查。達到 ClawHub 潛在封鎖門檻的訊號可能會觸發自動警告。如果警告期限後下一次符合資格的掃描仍讓發布者落在潛在封鎖門檻內，ClawHub 可能會自動套用帳戶處置。較低信心與有時間界限的審查訊號不會納入自動執行。

已刪除、被封鎖或停用的帳戶無法使用 ClawHub API 權杖。如果帳戶處置後命令列介面認證開始失敗，請登入網頁 UI 查看帳戶狀態。如果因封鎖或停用帳戶而無法登入或正常使用命令列介面，請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/)申請復原審查。

如果掃描器觸發的電子郵件將某個 skill 或外掛版本標示為惡意，請下載被封鎖提交版本的已儲存掃描結果：
`clawhub scan download <slug> --version <version>`。若為外掛，請加上
`--kind plugin`。檢閱掃描輸出、修正清單、遞增版本號，然後上傳修正後的版本。

## 發布者指南

若要降低誤判並提升使用者信任：

- 保持名稱、摘要、標籤與變更記錄準確
- 宣告必要的環境變數與權限
- 避免混淆的安裝命令
- 可行時連結至原始碼
- 發布外掛前使用試執行
- 如果使用者或審核人員詢問版本行為，請清楚回應
