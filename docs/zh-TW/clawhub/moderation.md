# 管理與帳戶安全

ClawHub 開放發布，但公開探索與安裝介面仍然需要防護機制。檢舉、管理暫停、隱藏列表和帳戶處置，有助於在某個版本或帳戶看起來不安全、具誤導性或不符合政策時保護使用者。

本頁說明管理與帳戶狀態。若要了解 `Pass`、`Review`、`Warn`、`Malicious` 等稽核標籤與風險等級，請參閱
[安全稽核](/clawhub/security-audits)。

另請參閱[安全性](/clawhub/security)與
[可接受使用方式](/clawhub/acceptable-usage)。如有著作權或其他內容權利疑慮，請使用[內容權利請求](/clawhub/content-rights)。

## 檢舉

已登入的使用者可以檢舉 Skills、外掛和套件。

僅在遇到不安全的市集內容時使用 ClawHub 檢舉，例如：

- 惡意列表
- 具誤導性的中繼資料
- 未聲明的憑證或權限需求
- 可疑的安裝指示
- 冒充
- 惡意註冊或商標濫用
- 違反[可接受使用方式](/clawhub/acceptable-usage)的內容

在技能頁面上使用**檢舉 skill**按鈕，或針對套件使用套件檢舉命令/API。

請勿使用 ClawHub 檢舉來回報第三方 skill 或外掛自身原始碼中的漏洞。請直接向發布者或列表中連結的原始碼儲存庫回報。ClawHub 不維護或修補第三方 skill 或外掛程式碼。

`openclaw/clawhub` 的 GitHub Security Advisories 適用於 ClawHub 本身的漏洞。例如網站、API、命令列介面、登錄檔、驗證、掃描、管理或下載/安裝信任邊界中的錯誤。請勿使用 ClawHub advisories 回報第三方 Skills 或外掛中的漏洞。

良好的檢舉應具體且可採取行動。濫用檢舉本身也可能導致帳戶處置。

## 組織與命名空間主張

組織、品牌、套件範圍、擁有者帳號或命名空間所有權爭議，應使用[組織與命名空間主張](/clawhub/namespace-claims)流程，而不是產品內檢舉流程或帳戶申訴表單。

當你需要 ClawHub 工作人員審查非敏感證明，以判定某個命名空間是否應被保留、轉移、重新命名、隱藏、隔離、建立別名或以其他方式審查時，請使用該流程。請勿在公開議題中包含密鑰、私人文件、私人法律文件、個人身分證明文件、API 權杖或 DNS challenge 權杖。

## 管理暫停

某些嚴重發現或政策問題可能會讓發布者或列表進入管理暫停狀態。發生這種情況時，受影響的內容可能會從公開探索中隱藏，或未來發布可能會先以隱藏狀態開始，直到問題經過審查。

管理暫停旨在 ClawHub 解決高風險案例期間保護使用者。若確認為誤判，也可以解除暫停。

## 隱藏或封鎖的列表

列表可能會被暫停、隱藏、隔離、撤銷，或以其他方式在公開安裝介面上無法使用。

如果你看到其中一種狀態，除非擁有者解決問題或管理方恢復該版本，否則請勿安裝該版本。

擁有者仍可能會看到其被暫停或隱藏列表的診斷資訊。這些診斷有助於說明發生了什麼，以及列表在返回公開介面前需要變更什麼。

## 停權與帳戶狀態

違反 ClawHub 政策的帳戶可能會失去發布權限。嚴重濫用可能導致帳戶停權、權杖撤銷、內容隱藏或列表移除。發布者濫用壓力訊號每日檢查。達到 ClawHub 潛在停權門檻的訊號可能會觸發自動警告。如果警告期限後下一次符合資格的掃描仍將發布者置於潛在停權門檻內，ClawHub 可能會自動套用帳戶處置。較低信心度和有時間範圍限制的審查訊號不會納入自動執法。

已刪除、被停權或停用的帳戶無法使用 ClawHub API 權杖。如果帳戶處置後命令列介面驗證開始失敗，請登入 Web UI 查看帳戶狀態。如果因停權或停用帳戶而無法登入或正常使用命令列介面，請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/)進行復原審查。

如果由掃描器觸發的電子郵件將某個 skill 或外掛版本標記為惡意，請下載被封鎖提交版本的已儲存掃描結果：
`clawhub scan download <slug> --version <version>`。對於外掛，請加上
`--kind plugin`。檢閱掃描輸出、修正列表、遞增版本號碼，並上傳修正後的版本。

## 發布者指南

為減少誤判並提升使用者信任：

- 保持名稱、摘要、標籤與變更紀錄準確
- 聲明必要的環境變數與權限
- 避免混淆的安裝命令
- 盡可能連結到原始碼
- 發布外掛前使用 dry runs
- 如果使用者或管理員詢問版本行為，請清楚回應
