# 審核與帳號安全

ClawHub 開放發布，但公開探索與安裝介面仍然需要防護機制。檢舉、審核保留、隱藏上架項目與帳號處置，可在某個發布版本或帳號看似不安全、具誤導性或違反政策時，協助保護使用者。

本頁說明審核與帳號狀態。如需了解 `Pass`、`Review`、`Warn`、`Malicious` 等稽核標籤與風險等級，請參閱 [安全稽核](/clawhub/security-audits)。

另請參閱 [安全性](/clawhub/security) 與 [可接受使用](/clawhub/acceptable-usage)。若涉及著作權或其他內容權利疑慮，請使用 [內容權利請求](/clawhub/content-rights)。

## 檢舉

已登入的使用者可以檢舉技能、外掛和套件。

僅將 ClawHub 檢舉用於不安全的市集內容，例如：

- 惡意上架項目
- 具誤導性的中繼資料
- 未宣告的憑證或權限需求
- 可疑的安裝指示
- 冒充
- 惡意註冊或商標濫用
- 違反 [可接受使用](/clawhub/acceptable-usage) 的內容

在技能頁面上使用 **檢舉技能** 按鈕，或對套件使用套件檢舉命令/API。

請勿將 ClawHub 檢舉用於第三方技能或外掛自身原始碼中的漏洞。請直接向發布者或上架項目連結的原始碼儲存庫回報。ClawHub 不維護或修補第三方技能或外掛程式碼。

`openclaw/clawhub` 的 GitHub Security Advisories 是用於 ClawHub 本身的漏洞。範例包括網站、API、命令列介面、登錄檔、驗證、掃描、審核或下載/安裝信任邊界中的錯誤。請勿將 ClawHub advisories 用於第三方技能或外掛中的漏洞。

良好的檢舉具體且可執行。濫用檢舉本身也可能導致帳號處置。

## 組織與命名空間聲明

組織、品牌、套件範圍、擁有者帳號或命名空間所有權爭議，應使用 [組織與命名空間聲明](/clawhub/namespace-claims) 流程，而不是產品內檢舉流程或帳號申訴表單。

當你需要 ClawHub 工作人員審查非敏感證明，以判斷某個命名空間是否應保留、轉移、重新命名、隱藏、隔離、設為別名或以其他方式審查時，請使用該流程。請勿在公開議題中包含秘密、私人文件、私人法律檔案、個人身分文件、API 權杖或 DNS 驗證權杖。

## 審核保留

某些嚴重發現或政策問題，可能會讓發布者或上架項目進入審核保留狀態。發生這種情況時，受影響的內容可能會從公開探索中隱藏，或之後發布時可能會先以隱藏狀態開始，直到問題完成審查。

審核保留旨在於 ClawHub 解決高風險案例期間保護使用者。若確認為誤判，也可以解除審核保留。

## 隱藏或封鎖的上架項目

上架項目可能會被保留、隱藏、隔離、撤銷，或以其他方式無法在公開安裝介面上使用。

如果你看到其中一種狀態，除非擁有者解決問題或審核恢復該項目，否則請勿安裝該發布版本。

擁有者仍可能看到自己被保留或隱藏上架項目的診斷資訊。這些診斷資訊有助於說明發生了什麼，以及上架項目回到公開介面前需要變更什麼。

## 封鎖與帳號狀態

違反 ClawHub 政策的帳號可能會失去發布權限。嚴重濫用可能導致帳號封鎖、權杖撤銷、內容隱藏或上架項目移除。發布者濫用壓力訊號每日檢查一次。達到 ClawHub 潛在封鎖門檻的訊號，可能會觸發自動警告。如果警告期限後的下一次符合條件掃描，仍將發布者置於潛在封鎖門檻內，ClawHub 可能會自動套用帳號處置。可信度較低且有時間範圍限制的審查訊號，不會納入自動執行。

已刪除、遭封鎖或停用的帳號無法使用 ClawHub API 權杖。如果帳號處置後命令列介面驗證開始失敗，請登入網頁 UI 檢視帳號狀態。如果登入或一般命令列介面存取因封鎖或停用帳號而受阻，請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/) 進行復原審查。

如果掃描器觸發的電子郵件將某個技能或外掛版本標記為惡意，請下載被封鎖提交版本的已儲存掃描結果：`clawhub scan download <slug> --version <version>`。對於外掛，請加入 `--kind plugin`。檢視掃描輸出、修正上架項目、遞增版本號碼，然後上傳修正後的版本。

## 發布者指南

若要減少誤判並提升使用者信任：

- 保持名稱、摘要、標籤和變更記錄準確
- 宣告必要的環境變數與權限
- 避免混淆處理的安裝命令
- 盡可能連結到原始碼
- 發布外掛前使用試跑
- 如果使用者或審核者詢問發布版本行為，請清楚回應
