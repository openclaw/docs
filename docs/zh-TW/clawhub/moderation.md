# 審核與帳戶安全

ClawHub 開放發布，但公開探索和安裝介面仍然需要防護措施。回報、審核暫停、隱藏清單和帳戶處置，有助於在版本或帳戶看似不安全、具誤導性或違反政策時保護使用者。

本頁涵蓋審核與帳戶狀態。如需 `Pass`、`Review`、`Warn`、`Malicious` 等稽核標籤與風險等級，請參閱 [安全稽核](/clawhub/security-audits)。

另請參閱 [安全性](/clawhub/security) 和 [可接受使用方式](/clawhub/acceptable-usage)。如有著作權或其他內容權利疑慮，請使用 [內容權利請求](/clawhub/content-rights)。

## 回報

已登入的使用者可以回報技能、外掛和套件。

ClawHub 回報僅用於不安全的市集內容，例如：

- 惡意清單
- 具誤導性的中繼資料
- 未宣告的認證資訊或權限需求
- 可疑的安裝指示
- 冒充
- 惡意註冊或商標濫用
- 違反 [可接受使用方式](/clawhub/acceptable-usage) 的內容

在技能頁面上使用 **回報技能** 按鈕，或使用套件的回報命令/API。

請勿使用 ClawHub 回報來通報第三方技能或外掛自身原始碼中的漏洞。請直接向清單中連結的發布者或原始碼儲存庫回報。ClawHub 不維護或修補第三方技能或外掛程式碼。

`openclaw/clawhub` 的 GitHub Security Advisories 用於 ClawHub 本身的漏洞。範例包括網站、API、命令列介面、登錄、驗證、掃描、審核，或下載/安裝信任邊界中的錯誤。請勿使用 ClawHub advisories 來通報第三方技能或外掛中的漏洞。

良好的回報應具體且可採取行動。濫用回報機制本身也可能導致帳戶處置。

## 組織與命名空間聲明

組織、品牌、套件範圍、擁有者帳號或命名空間所有權爭議，應使用 [組織與命名空間聲明](/clawhub/namespace-claims) 流程，而不是產品內回報流程或帳戶申訴表單。

當你需要 ClawHub 工作人員審查非敏感證明，以判定命名空間應被保留、轉移、重新命名、隱藏、隔離、設為別名或以其他方式審查時，請使用該流程。請勿在公開議題中包含機密、私人文件、私人法律文件、個人身分文件、API token 或 DNS challenge token。

## 審核暫停

某些嚴重發現或政策問題可能會使發布者或清單進入審核暫停狀態。發生這種情況時，受影響的內容可能會從公開探索中隱藏，或未來發布可能會先以隱藏狀態開始，直到問題完成審查。

審核暫停旨在 ClawHub 處理高風險案例期間保護使用者。確認為誤判時，也可以解除審核暫停。

## 隱藏或封鎖的清單

清單可能會被暫停、隱藏、隔離、撤銷，或以其他方式在公開安裝介面上無法使用。

如果你看到其中一種狀態，除非擁有者解決問題或審核恢復該清單，否則請勿安裝該版本。

擁有者仍可查看其自身被暫停或隱藏清單的診斷資訊。這些診斷有助於說明發生了什麼，以及清單返回公開介面前需要變更的內容。

## 停權與帳戶狀態

違反 ClawHub 政策的帳戶可能會失去發布權限。嚴重濫用可能導致帳戶停權、token 撤銷、內容隱藏或清單移除。發布者濫用壓力訊號會每日檢查。達到 ClawHub 潛在停權門檻的訊號可能會觸發自動警告。如果警告期限後下一次符合條件的掃描仍將發布者置於潛在停權門檻內，ClawHub 可能會自動套用帳戶處置。較低信心度和有時間範圍限制的審查訊號不會納入自動執行。

已刪除、已停權或已停用的帳戶無法使用 ClawHub API token。如果帳戶處置後命令列介面驗證開始失敗，請登入網頁介面以檢查帳戶狀態。如果因停權或帳戶停用而無法登入或正常使用命令列介面，請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/) 申請恢復審查。

如果掃描器觸發的電子郵件將某個技能或外掛版本標示為惡意，請下載被封鎖提交版本的已儲存掃描結果：
`clawhub scan download <slug> --version <version>`。對於外掛，請加上 `--kind plugin`。檢視掃描輸出、修正清單、遞增版本號，並上傳修正後的版本。

## 發布者指南

若要減少誤判並提升使用者信任：

- 保持名稱、摘要、標籤和變更記錄準確
- 宣告必要的環境變數和權限
- 避免混淆的安裝命令
- 可行時連結至原始碼
- 發布外掛前使用試執行
- 如果使用者或審核者詢問版本行為，請清楚回應
