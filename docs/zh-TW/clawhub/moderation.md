# 內容審核與帳戶安全

ClawHub 開放發布，但公開探索與安裝介面仍然需要防護措施。回報、內容審核保留、隱藏列表，以及帳戶處置，有助於在某個版本或帳戶看起來不安全、具誤導性或違反政策時保護使用者。

本頁涵蓋內容審核與帳戶狀態。如需 `Pass`、`Review`、`Warn`、`Malicious` 等稽核標籤與風險等級，請參閱
[安全稽核](/clawhub/security-audits)。

另請參閱[安全性](/clawhub/security)和
[可接受使用](/clawhub/acceptable-usage)。如有著作權或其他內容權利疑慮，請使用[內容權利請求](/clawhub/content-rights)。

## 回報

已登入的使用者可以回報 Skills、外掛和套件。

僅將 ClawHub 回報用於不安全的市集內容，例如：

- 惡意列表
- 具誤導性的中繼資料
- 未宣告的憑證或權限需求
- 可疑的安裝指示
- 冒名
- 惡意註冊或商標濫用
- 違反[可接受使用](/clawhub/acceptable-usage)的內容

在 Skills 頁面上使用 **回報 Skills** 按鈕，或對套件使用套件回報命令/API。

請勿使用 ClawHub 回報來通報第三方 Skills 或外掛自身原始碼中的漏洞。請直接向發布者或列表中連結的原始碼儲存庫回報。ClawHub 不維護或修補第三方 Skills 或外掛程式碼。

`openclaw/clawhub` 的 GitHub 安全公告用於 ClawHub 本身的漏洞。範例包括網站、API、命令列介面、登錄檔、驗證、掃描、內容審核，或下載/安裝信任邊界中的錯誤。請勿將 ClawHub 公告用於第三方 Skills 或外掛中的漏洞。

良好的回報應具體且可執行。濫用回報本身也可能導致帳戶處置。

## 組織與命名空間主張

組織、品牌、套件範圍、擁有者帳號，或命名空間所有權爭議，應使用[組織與命名空間主張](/clawhub/namespace-claims)流程，而不是產品內回報流程或帳戶申訴表單。

當你需要 ClawHub 工作人員審查非敏感證明，以判斷某個命名空間是否應被保留、轉移、重新命名、隱藏、隔離、設為別名，或以其他方式審查時，請使用該流程。請勿在公開 issue 中包含秘密、私人文件、私人法律檔案、個人身分文件、API 權杖，或 DNS challenge 權杖。

## 內容審核保留

某些嚴重發現或政策問題可能會讓發布者或列表進入內容審核保留狀態。發生這種情況時，受影響的內容可能會從公開探索中隱藏，或未來發布可能會先以隱藏狀態開始，直到問題完成審查。

內容審核保留旨在 ClawHub 處理高風險案例時保護使用者。若確認為誤判，也可以解除保留。

## 隱藏或封鎖的列表

列表可能會被保留、隱藏、隔離、撤銷，或以其他方式在公開安裝介面上無法使用。

如果你看到其中一種狀態，除非擁有者解決問題或內容審核恢復該版本，否則請勿安裝該版本。

擁有者仍可能看到自己被保留或隱藏列表的診斷資訊。這些診斷資訊有助於說明發生了什麼，以及列表回到公開介面前需要變更什麼。

## 封鎖與帳戶狀態

違反 ClawHub 政策的帳戶可能會失去發布存取權。嚴重濫用可能導致帳戶封鎖、權杖撤銷、內容隱藏，或列表移除。發布者濫用壓力訊號每日檢查。達到 ClawHub 潛在封鎖門檻的訊號可能會觸發自動警告。如果警告期限後下一次符合條件的掃描仍將發布者置於潛在封鎖門檻內，ClawHub 可能會自動套用帳戶處置。較低信心度與有時間範圍限制的審查訊號不會納入自動執行。

已刪除、被封鎖或已停用的帳戶無法使用 ClawHub API 權杖。如果帳戶處置後命令列介面驗證開始失敗，請登入網頁介面查看帳戶狀態。如果因封鎖或停用帳戶而無法登入或正常使用命令列介面，請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/) 進行復原審查。

如果掃描器觸發的電子郵件將某個 Skills 或外掛版本標記為惡意，請下載被封鎖提交版本的已儲存掃描結果：
`clawhub scan download <slug> --version <version>`。對於外掛，請加上
`--kind plugin`。檢閱掃描輸出、修正列表、遞增版本號，然後上傳修正後的版本。

## 發布者指引

若要減少誤判並提升使用者信任：

- 保持名稱、摘要、標籤和變更記錄準確
- 宣告必要的環境變數和權限
- 避免混淆難懂的安裝命令
- 盡可能連結到原始碼
- 發布外掛前使用試執行
- 如果使用者或內容審核人員詢問版本行為，請清楚回應
