# 內容審核與帳號安全

ClawHub 開放內容發布，但公開探索與安裝介面仍需要防護措施。檢舉、內容審核保留、隱藏刊登項目及帳號處置，有助於在某個版本或帳號疑似不安全、具誤導性或違反政策時保護使用者。

本頁說明內容審核與帳號狀態。若要瞭解 `Pass`、`Review`、`Warn`、`Malicious` 等稽核標籤及風險等級，請參閱[安全稽核](/clawhub/security-audits)。

另請參閱[安全性](/zh-TW/clawhub/security)與[可接受使用規範](/clawhub/acceptable-usage)。如有著作權或其他內容權利相關疑慮，請使用[內容權利請求](/clawhub/content-rights)。

## 檢舉

已登入的使用者可以檢舉 Skills、外掛和套件。

ClawHub 檢舉僅適用於不安全的市集內容，例如：

- 惡意刊登項目
- 誤導性中繼資料
- 未聲明的憑證或權限要求
- 可疑的安裝指示
- 冒用身分
- 惡意註冊或濫用商標
- 違反[可接受使用規範](/clawhub/acceptable-usage)的內容

在 Skills 頁面上使用 **Report skill** 按鈕，或針對套件使用套件檢舉命令/API。

請勿使用 ClawHub 檢舉功能通報第三方 Skills 或外掛本身原始碼中的弱點。請直接向發布者或刊登項目所連結的原始碼儲存庫通報。ClawHub 不維護或修補第三方 Skills 或外掛程式碼。

`openclaw/clawhub` 的 GitHub Security Advisories 僅用於 ClawHub 本身的弱點。例如網站、API、命令列介面、登錄檔、驗證、掃描、內容審核或下載/安裝信任邊界中的錯誤。請勿使用 ClawHub 安全公告通報第三方 Skills 或外掛中的弱點。

良好的檢舉應具體且可採取行動。濫用檢舉功能本身也可能導致帳號處置。

## 組織與命名空間所有權聲明

組織、品牌、套件範圍、擁有者帳號名稱或命名空間的所有權爭議，應使用[組織與命名空間所有權聲明](/clawhub/namespace-claims)流程，而非產品內的檢舉流程或帳號申訴表單。

若需要 ClawHub 工作人員審查非敏感證明，以判定某個命名空間是否應予保留、移轉、重新命名、隱藏、隔離、設定別名或以其他方式審查，請使用該流程。請勿在公開議題中包含密鑰、私人文件、非公開法律文件、個人身分證明文件、API 權杖或 DNS 驗證權杖。

## 內容審核保留

某些嚴重發現或政策問題可能導致發布者或刊登項目受到內容審核保留。發生此情況時，受影響的內容可能會從公開探索中隱藏，或後續發布的版本可能會先以隱藏狀態上架，直到問題經過審查。

內容審核保留旨在 ClawHub 處理高風險案例期間保護使用者。確認為誤判後，也可以解除保留。

## 隱藏或封鎖的刊登項目

刊登項目可能處於保留、隱藏、隔離、撤銷或其他無法在公開安裝介面中取得的狀態。

如果看到上述任一狀態，除非擁有者已解決問題或內容審核已恢復該版本，否則請勿安裝該版本。

擁有者可能仍可查看自己遭保留或隱藏之刊登項目的診斷資訊。這些診斷資訊有助於說明發生了什麼情況，以及在刊登項目恢復至公開介面前需要進行哪些變更。

## 封禁與帳號狀態

違反 ClawHub 政策的帳號可能會失去發布權限。嚴重濫用可能導致帳號遭封禁、權杖被撤銷、內容遭隱藏或刊登項目遭移除。系統每天都會檢查發布者濫用壓力訊號。達到 ClawHub 潛在封禁門檻的訊號可能會觸發自動警告。如果警告期限過後的下一次符合條件之掃描仍將該發布者判定為達到潛在封禁門檻，ClawHub 可能會自動套用帳號處置。可信度較低及時間範圍有限的審查訊號不會納入自動執行機制。

已刪除、遭封禁或停用的帳號無法使用 ClawHub API 權杖。如果命令列介面驗證在帳號處置後開始失敗，請登入網頁介面查看帳號狀態。如果因帳號遭封禁或停用而無法登入或正常使用命令列介面，請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/)申請復原審查。

如果由掃描器觸發的電子郵件指出某個 Skills 或外掛版本具有惡意行為，請下載遭封鎖之已提交版本所儲存的掃描結果：`clawhub scan download <slug> --version <version>`。若為外掛，請加上 `--kind plugin`。檢查掃描輸出、修正刊登項目、增加版本號碼，然後上傳修正後的版本。

## 發布者指南

若要減少誤判並提升使用者信任：

- 確保名稱、摘要、標籤與變更記錄準確
- 聲明必要的環境變數與權限
- 避免使用經過混淆的安裝命令
- 盡可能連結至原始碼
- 發布外掛前先執行試運行
- 如果使用者或內容審核人員詢問版本行為，請清楚回應
