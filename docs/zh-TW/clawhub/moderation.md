# 審核與帳號安全

ClawHub 開放發布，但公開探索與安裝介面仍需要防護措施。回報、審核保留、隱藏清單與帳號處置，有助於在版本或帳號看似不安全、具誤導性或違反政策時保護使用者。

本頁說明審核與帳號狀態。若要了解 `Pass`、`Review`、`Warn`、`Malicious` 等稽核標籤與風險等級，請參閱
[安全稽核](/clawhub/security-audits)。

另請參閱 [安全性](/clawhub/security) 與
[可接受使用](/clawhub/acceptable-usage)。若有著作權或其他內容權利疑慮，請使用 [內容權利請求](/clawhub/content-rights)。

## 回報

已登入的使用者可以回報技能、外掛與套件。

僅針對不安全的市集內容使用 ClawHub 回報，例如：

- 惡意清單
- 具誤導性的中繼資料
- 未宣告的憑證或權限需求
- 可疑的安裝指示
- 冒充身分
- 惡意註冊或商標濫用
- 違反 [可接受使用](/clawhub/acceptable-usage) 的內容

在技能頁面上使用 **回報技能** 按鈕，或針對套件使用套件回報命令/API。

請勿將 ClawHub 回報用於第三方技能或外掛自身原始碼中的漏洞。請直接回報給清單連結的發布者或原始碼儲存庫。ClawHub 不維護或修補第三方技能或外掛程式碼。

`openclaw/clawhub` 的 GitHub 安全公告適用於 ClawHub 本身的漏洞。範例包括網站、API、命令列介面、登錄檔、驗證、掃描、審核或下載/安裝信任邊界中的錯誤。請勿將 ClawHub 公告用於第三方技能或外掛中的漏洞。

好的回報應該具體且可執行。濫用回報本身也可能導致帳號處置。

## 組織與命名空間聲明

組織、品牌、套件範圍、擁有者帳號或命名空間所有權爭議，應使用 [組織與命名空間聲明](/clawhub/namespace-claims) 流程，而不是產品內回報流程或帳號申訴表單。

當你需要 ClawHub 工作人員審查非敏感證明，以判斷命名空間是否應被保留、轉移、重新命名、隱藏、隔離、設為別名或以其他方式審查時，請使用該流程。請勿在公開議題中包含機密、私人文件、私人法律檔案、個人身分文件、API 權杖或 DNS 挑戰權杖。

## 審核保留

某些嚴重發現或政策問題可能會使發布者或清單進入審核保留狀態。發生這種情況時，受影響的內容可能會從公開探索中隱藏，或之後發布時可能先以隱藏狀態開始，直到問題完成審查。

審核保留旨在 ClawHub 解決高風險案例期間保護使用者。當確認為誤判時，也可以解除保留。

## 隱藏或封鎖的清單

清單可能會被保留、隱藏、隔離、撤銷，或以其他方式無法在公開安裝介面上使用。

如果你看到這些狀態之一，除非擁有者解決問題或審核恢復該版本，否則請勿安裝該版本。

擁有者仍可能看到自己被保留或隱藏清單的診斷資訊。這些診斷有助於說明發生了什麼，以及清單重新回到公開介面前需要做哪些變更。

## 封鎖與帳號狀態

違反 ClawHub 政策的帳號可能會失去發布權限。嚴重濫用可能導致帳號封鎖、權杖撤銷、內容隱藏或清單移除。發布者濫用壓力訊號會每日檢查。達到 ClawHub 潛在封鎖門檻的訊號可能會觸發自動警告。如果警告期限後下一次符合資格的掃描仍將發布者置於潛在封鎖門檻內，ClawHub 可能會自動套用帳號處置。較低信心與有時間界限的審查訊號不會進入自動執行。

已刪除、被封鎖或停用的帳號無法使用 ClawHub API 權杖。如果帳號處置後命令列介面驗證開始失敗，請登入網頁 UI 檢查帳號狀態。如果因封鎖或停用帳號而無法登入或正常使用命令列介面，請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/) 進行復原審查。

如果由掃描器觸發的電子郵件將某個技能或外掛版本標記為惡意，請下載被封鎖提交版本的已儲存掃描結果：
`clawhub scan download <slug> --version <version>`。對於外掛，請加入
`--kind plugin`。檢查掃描輸出，修正清單，遞增版本號，然後上傳修正後的版本。

## 發布者指南

若要降低誤判並提升使用者信任：

- 保持名稱、摘要、標籤與變更記錄準確
- 宣告必要的環境變數與權限
- 避免混淆過的安裝命令
- 盡可能連結到原始碼
- 發布外掛前使用 dry runs
- 如果使用者或審核人員詢問版本行為，請清楚回覆
