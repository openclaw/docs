# 審核與帳號安全

ClawHub 開放發佈，但公開探索與安裝介面仍需要防護措施。檢舉、審核暫停、隱藏的上架項目，以及帳號處置，有助於在某個版本或帳號看似不安全、具誤導性或違反政策時保護使用者。

本頁說明審核與帳號狀態。關於 `Pass`、`Review`、`Warn`、`Malicious` 等稽核標籤與風險等級，請參閱 [安全稽核](/clawhub/security-audits)。

另請參閱 [安全性](/clawhub/security) 和 [可接受使用方式](/clawhub/acceptable-usage)。若有著作權或其他內容權利疑慮，請使用 [內容權利請求](/clawhub/content-rights)。

## 檢舉

已登入的使用者可以檢舉 Skills、外掛和套件。

請僅針對不安全的市集內容使用 ClawHub 檢舉，例如：

- 惡意上架項目
- 具誤導性的中繼資料
- 未宣告的憑證或權限需求
- 可疑的安裝指示
- 冒名
- 惡意註冊或商標濫用
- 違反 [可接受使用方式](/clawhub/acceptable-usage) 的內容

在 Skill 頁面上使用 **檢舉 Skill** 按鈕，或對套件使用套件檢舉命令/API。

請勿使用 ClawHub 檢舉來回報第三方 Skill 或外掛自身原始碼中的漏洞。請直接向上架項目連結的發佈者或原始碼儲存庫回報。ClawHub 不維護或修補第三方 Skill 或外掛程式碼。

`openclaw/clawhub` 的 GitHub Security Advisories 用於 ClawHub 本身的漏洞。例如網站、API、命令列介面、登錄檔、驗證、掃描、審核，或下載/安裝信任邊界中的錯誤。請勿使用 ClawHub advisories 來回報第三方 Skills 或外掛中的漏洞。

好的檢舉應具體且可採取行動。濫用檢舉本身也可能導致帳號處置。

## 組織與命名空間聲明

組織、品牌、套件範圍、擁有者帳號，或命名空間所有權爭議，應使用 [組織與命名空間聲明](/clawhub/namespace-claims) 流程，而不是產品內檢舉流程或帳號申訴表單。

當你需要 ClawHub 人員審查非敏感證明，以判定某個命名空間是否應保留、轉移、重新命名、隱藏、隔離、別名化，或以其他方式審查時，請使用該流程。請勿在公開 issue 中包含秘密、私人文件、私人法律檔案、個人身分文件、API token，或 DNS challenge token。

## 審核暫停

某些嚴重發現或政策問題，可能會使發佈者或上架項目進入審核暫停狀態。發生這種情況時，受影響的內容可能會從公開探索中隱藏，或未來發佈可能會先以隱藏狀態開始，直到問題完成審查。

審核暫停的目的，是在 ClawHub 處理高風險案例期間保護使用者。若確認為誤判，也可以解除暫停。

## 隱藏或封鎖的上架項目

上架項目可能會被暫停、隱藏、隔離、撤銷，或以其他方式無法在公開安裝介面取得。

如果你看到其中一種狀態，除非擁有者解決問題或審核恢復該項目，否則請勿安裝該版本。

擁有者仍可能看到自己被暫停或隱藏上架項目的診斷資訊。這些診斷有助於說明發生了什麼事，以及上架項目恢復到公開介面前需要變更什麼。

## 停權與帳號狀態

違反 ClawHub 政策的帳號可能會失去發佈權限。嚴重濫用可能導致帳號停權、token 撤銷、內容隱藏，或上架項目移除。發佈者濫用壓力訊號會每日檢查。達到 ClawHub 潛在停權門檻的訊號，可能觸發自動警告。如果警告期限後下一次符合條件的掃描，仍將該發佈者置於潛在停權門檻，ClawHub 可能會自動套用帳號處置。較低信心度與有時間範圍限制的審查訊號，不會納入自動執法。

已刪除、停權或停用的帳號無法使用 ClawHub API token。如果命令列介面驗證在帳號處置後開始失敗，請登入 Web UI 查看帳號狀態。如果因停權或停用帳號而無法登入或正常使用命令列介面，請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/) 進行復原審查。

如果由掃描器觸發的電子郵件指出某個 Skill 或外掛版本為惡意，請下載被封鎖提交版本的已儲存掃描結果：`clawhub scan download <slug> --version <version>`。對於外掛，請加上 `--kind plugin`。檢閱掃描輸出，修正上架項目，遞增版本號，然後上傳修正後的版本。

## 發佈者指引

若要減少誤判並提升使用者信任：

- 保持名稱、摘要、標籤和變更記錄準確
- 宣告必要的環境變數與權限
- 避免混淆的安裝命令
- 盡可能連結到原始碼
- 發佈外掛前使用 dry runs
- 如果使用者或審核人員詢問版本行為，請清楚回應
