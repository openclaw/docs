# 審核與帳號安全

ClawHub 開放發布內容，但公開探索與安裝介面仍需要防護措施。檢舉、審核保留、隱藏刊登項目與帳號處置，有助於在版本或帳號疑似不安全、具誤導性或違反政策時保護使用者。

本頁說明審核與帳號狀態。若要瞭解 `Pass`、`Review`、`Warn`、`Malicious` 等稽核標籤與風險等級，請參閱[安全稽核](/clawhub/security-audits)。

另請參閱[安全性](/clawhub/security)與[可接受的使用方式](/clawhub/acceptable-usage)。若有著作權或其他內容權利疑慮，請使用[內容權利申請](/clawhub/content-rights)。

## 檢舉

已登入的使用者可以檢舉 Skills、外掛與套件。

ClawHub 檢舉僅適用於不安全的市集內容，例如：

- 惡意刊登項目
- 具誤導性的中繼資料
- 未聲明的認證資訊或權限要求
- 可疑的安裝指示
- 冒充他人
- 惡意搶註或濫用商標
- 違反[可接受的使用方式](/clawhub/acceptable-usage)的內容

請使用 Skill 頁面上的 **Report skill** 按鈕，或套件的檢舉命令/API。

請勿使用 ClawHub 檢舉來回報第三方 Skill 或外掛自身原始碼中的漏洞。請直接向發布者或刊登項目所連結的原始碼儲存庫回報。ClawHub 不維護或修補第三方 Skill 或外掛的程式碼。

`openclaw/clawhub` 的 GitHub Security Advisories 適用於 ClawHub 本身的漏洞。例如網站、API、命令列介面、登錄檔、驗證、掃描、審核或下載／安裝信任邊界中的錯誤。請勿使用 ClawHub 安全公告來回報第三方 Skills 或外掛中的漏洞。

良好的檢舉應具體且可採取行動。濫用檢舉功能本身也可能導致帳號處置。

## 組織與命名空間主張

組織、品牌、套件範圍、擁有者帳號名稱或命名空間的所有權爭議，應使用[組織與命名空間主張](/clawhub/namespace-claims)程序，而非產品內檢舉流程或帳號申訴表單。

當你需要 ClawHub 工作人員審查非敏感證明，以判斷命名空間是否應予保留、移轉、重新命名、隱藏、隔離、設定別名或以其他方式進行審查時，請使用該程序。請勿在公開議題中包含機密資訊、私人文件、非公開法律文件、個人身分證明文件、API 權杖或 DNS 驗證權杖。

## 審核保留

部分嚴重發現或政策問題可能使發布者或刊登項目進入審核保留狀態。發生此情況時，受影響的內容可能會從公開探索中隱藏，或後續發布的內容可能會先以隱藏狀態建立，直到問題經過審查為止。

審核保留旨在於 ClawHub 處理高風險案例期間保護使用者。若確認為誤判，也可以解除審核保留。

## 隱藏或封鎖的刊登項目

刊登項目可能會遭到保留、隱藏、隔離、撤銷，或因其他原因而無法出現在公開安裝介面。

若你看到其中一種狀態，請勿安裝該版本，除非擁有者已解決問題或審核人員已恢復該版本。

擁有者仍可能看見自己遭保留或隱藏之刊登項目的診斷資訊。這些診斷資訊有助於說明發生的情況，以及刊登項目重新出現在公開介面之前需要進行哪些變更。

## 封禁與帳號狀態

違反 ClawHub 政策的帳號可能會失去發布權限。嚴重濫用可能導致帳號遭封禁、權杖遭撤銷、內容遭隱藏或刊登項目遭移除。系統每天都會檢查發布者濫用壓力訊號。達到 ClawHub 潛在封禁門檻的訊號可能會觸發自動警告。如果警告期限後的下一次合格掃描仍判定發布者達到潛在封禁門檻，ClawHub 可能會自動套用帳號處置。信心度較低且時間範圍有限的審查訊號不會納入自動執法。

已刪除、遭封禁或停用的帳號無法使用 ClawHub API 權杖。如果帳號遭處置後命令列介面驗證開始失敗，請登入網頁使用者介面以檢視帳號狀態。如果因帳號遭封禁或停用而無法登入或正常使用命令列介面，請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/)申請復原審查。

如果掃描器觸發的電子郵件將某個 Skill 或外掛版本標示為惡意版本，請下載遭封鎖之已提交版本的已儲存掃描結果：
`clawhub scan download <slug> --version <version>`。若為外掛，請加上
`--kind plugin`。檢查掃描輸出、修正刊登項目、遞增版本號，然後上傳修正後的版本。

## 發布者指南

若要減少誤判並提高使用者信任：

- 確保名稱、摘要、標籤與變更記錄正確無誤
- 聲明必要的環境變數與權限
- 避免使用經過混淆的安裝命令
- 盡可能提供原始碼連結
- 發布外掛前先使用試執行
- 若使用者或審核人員詢問版本行為，請清楚回覆
