# 內容審核與帳戶安全

ClawHub 開放發布，但公開探索與安裝介面仍然需要防護措施。回報、內容審核暫停、隱藏列表項目與帳戶處置，可在某個版本或帳戶看起來不安全、具誤導性或違反政策時，協助保護使用者。

本頁說明內容審核與帳戶狀態。如需了解 `Pass`、`Review`、`Warn`、`Malicious` 等稽核標籤與風險等級，請參閱
[安全稽核](/clawhub/security-audits)。

另請參閱[安全性](/clawhub/security)與
[可接受使用方式](/clawhub/acceptable-usage)。若有著作權或其他內容權利疑慮，請使用[內容權利請求](/clawhub/content-rights)。

## 回報

已登入的使用者可以回報 Skills、外掛與套件。

僅將 ClawHub 回報用於不安全的市集內容，例如：

- 惡意列表項目
- 具誤導性的中繼資料
- 未宣告的憑證或權限需求
- 可疑的安裝指示
- 冒充
- 惡意註冊或商標誤用
- 違反[可接受使用方式](/clawhub/acceptable-usage)的內容

在 Skill 頁面使用 **回報 Skill** 按鈕，或對套件使用套件回報命令/API。

請勿使用 ClawHub 回報第三方 Skill 或外掛自身原始碼中的弱點。請直接回報給列表項目連結的發布者或原始碼儲存庫。ClawHub 不維護或修補第三方 Skill 或外掛程式碼。

`openclaw/clawhub` 的 GitHub Security Advisories 用於 ClawHub 本身的弱點。例如網站、API、命令列介面、登錄檔、驗證、掃描、內容審核，或下載/安裝信任邊界中的錯誤。請勿將 ClawHub advisories 用於第三方 Skills 或外掛中的弱點。

好的回報應具體且可執行。濫用回報本身也可能導致帳戶處置。

## 組織與命名空間聲明

組織、品牌、套件範圍、擁有者帳號或命名空間擁有權爭議，應使用[組織與命名空間聲明](/clawhub/namespace-claims)流程，而不是產品內回報流程或帳戶申訴表單。

當你需要 ClawHub 工作人員審查非敏感證明，以判定某個命名空間應保留、轉移、重新命名、隱藏、隔離、建立別名或以其他方式審查時，請使用該流程。請勿在公開 issue 中包含秘密、私人文件、私人法律檔案、個人身分文件、API 權杖或 DNS 挑戰權杖。

## 內容審核暫停

某些嚴重發現或政策問題可能使發布者或列表項目進入內容審核暫停狀態。發生這種情況時，受影響內容可能會從公開探索中隱藏，或未來發布可能會在審查完成前先以隱藏狀態開始。

內容審核暫停旨在讓 ClawHub 處理高風險案例期間保護使用者。確認為誤判時，也可以解除暫停。

## 隱藏或封鎖的列表項目

列表項目可能會被暫停、隱藏、隔離、撤銷，或以其他方式在公開安裝介面上無法取得。

如果你看到這些狀態之一，除非擁有者解決問題或內容審核將其恢復，否則請勿安裝該版本。

擁有者可能仍會看到自己被暫停或隱藏列表項目的診斷資訊。這些診斷可協助說明發生了什麼，以及列表項目在返回公開介面前需要變更哪些內容。

## 停權與帳戶狀態

違反 ClawHub 政策的帳戶可能會失去發布權限。嚴重濫用可能導致帳戶停權、權杖撤銷、內容隱藏或列表項目移除。發布者濫用壓力訊號每日檢查。達到 ClawHub 潛在停權門檻的訊號可能會觸發自動警告。如果警告期限後的下一次合格掃描仍將發布者置於潛在停權門檻內，ClawHub 可能會自動套用帳戶處置。信心較低與有時間範圍限制的審查訊號不會進入自動執行。

已刪除、停權或停用的帳戶無法使用 ClawHub API 權杖。如果命令列介面驗證在帳戶處置後開始失敗，請登入網頁介面檢查帳戶狀態。如果登入或一般命令列介面存取因停權或停用帳戶而受阻，請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/) 進行復原審查。

如果掃描器觸發的電子郵件指出某個 Skill 或外掛版本為惡意，請下載被封鎖提交版本的已儲存掃描結果：
`clawhub scan download <slug> --version <version>`。對於外掛，請加上
`--kind plugin`。檢閱掃描輸出，修正列表項目，遞增版本號碼，並上傳修正後的版本。

## 發布者指引

為了減少誤判並提升使用者信任：

- 保持名稱、摘要、標籤與變更記錄準確
- 宣告必要的環境變數與權限
- 避免混淆過的安裝命令
- 盡可能連結到原始碼
- 發布外掛前使用試執行
- 若使用者或審核人員詢問版本行為，請清楚回應
