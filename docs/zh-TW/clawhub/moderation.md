# 審核與帳戶安全

ClawHub 開放發布，但公開探索與安裝介面仍需要防護措施。回報、審核保留、隱藏清單與帳戶處置，可在某個版本或帳戶看似不安全、具誤導性或違反政策時，協助保護使用者。

本頁說明審核與帳戶狀態。關於 `Pass`、`Review`、`Warn`、`Malicious` 等稽核標籤與風險等級，請參閱 [安全稽核](/clawhub/security-audits)。

另請參閱 [安全性](/clawhub/security) 與 [可接受使用](/clawhub/acceptable-usage)。若涉及著作權或其他內容權利疑慮，請使用 [內容權利請求](/clawhub/content-rights)。

## 回報

已登入的使用者可以回報技能、外掛與套件。

ClawHub 回報僅應用於不安全的市集內容，例如：

- 惡意清單
- 具誤導性的中繼資料
- 未宣告的憑證或權限需求
- 可疑的安裝指示
- 冒名
- 惡意註冊或商標濫用
- 違反 [可接受使用](/clawhub/acceptable-usage) 的內容

請在技能頁面使用 **回報技能** 按鈕，或對套件使用套件回報命令/API。

請勿使用 ClawHub 回報來通報第三方技能或外掛自身原始碼中的漏洞。請直接向清單連結的發布者或原始碼儲存庫回報。ClawHub 不維護或修補第三方技能或外掛程式碼。

`openclaw/clawhub` 的 GitHub 安全公告是用於 ClawHub 本身的漏洞。例如網站、API、命令列介面、登錄檔、驗證、掃描、審核，或下載/安裝信任邊界中的錯誤。請勿使用 ClawHub 公告來通報第三方技能或外掛中的漏洞。

好的回報應具體且可執行。濫用回報本身也可能導致帳戶處置。

## 組織與命名空間主張

組織、品牌、套件範圍、擁有者帳號或命名空間所有權爭議，應使用 [組織與命名空間主張](/clawhub/namespace-claims) 流程，而不是產品內回報流程或帳戶申訴表單。

當你需要 ClawHub 工作人員審查非敏感證明，以判定某個命名空間是否應保留、轉移、重新命名、隱藏、隔離、建立別名或以其他方式審查時，請使用該流程。請勿在公開 issue 中包含秘密、私人文件、私人法律檔案、個人身分文件、API token 或 DNS challenge token。

## 審核保留

某些嚴重發現或政策問題，可能會使發布者或清單進入審核保留狀態。發生這種情況時，受影響的內容可能會從公開探索中隱藏，或未來發布可能會先以隱藏狀態開始，直到問題完成審查。

審核保留的目的是在 ClawHub 解決高風險案例時保護使用者。若確認為誤判，也可以解除保留。

## 隱藏或封鎖的清單

清單可能會被保留、隱藏、隔離、撤銷，或以其他方式在公開安裝介面上無法使用。

如果你看到其中一種狀態，除非擁有者解決問題或審核恢復該版本，否則請勿安裝該版本。

擁有者仍可能看見自己被保留或隱藏清單的診斷資訊。這些診斷資訊有助於說明發生了什麼事，以及清單返回公開介面前需要變更什麼。

## 封鎖與帳戶狀態

違反 ClawHub 政策的帳戶可能會失去發布存取權。嚴重濫用可能導致帳戶封鎖、token 撤銷、內容隱藏或清單移除。發布者濫用風險訊號會每日檢查。達到 ClawHub 潛在封鎖門檻的訊號可能會觸發自動警告。如果警告期限後下一次符合條件的掃描，仍將發布者置於潛在封鎖門檻內，ClawHub 可能會自動套用帳戶處置。信心較低且有時間範圍限制的審查訊號，不會進入自動執行。

已刪除、被封鎖或停用的帳戶無法使用 ClawHub API token。如果命令列介面驗證在帳戶處置後開始失敗，請登入網頁 UI 以檢視帳戶狀態。如果登入或一般命令列介面存取因封鎖或帳戶停用而受阻，請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/) 申請復原審查。

如果掃描器觸發的電子郵件指出某個技能或外掛版本為惡意，請下載被封鎖提交版本的已儲存掃描結果：`clawhub scan download <slug> --version <version>`。對於外掛，請加上 `--kind plugin`。檢閱掃描輸出、修正清單、遞增版本號，然後上傳已修正版本。

## 發布者指引

若要減少誤判並提升使用者信任：

- 保持名稱、摘要、標籤與變更記錄準確
- 宣告必要的環境變數與權限
- 避免混淆的安裝命令
- 盡可能連結到原始碼
- 發布外掛前使用 dry run
- 若使用者或審核者詢問版本行為，請清楚回應
