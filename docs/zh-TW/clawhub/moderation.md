# 內容審查與帳號安全

ClawHub 開放發布內容，但公開探索與安裝介面仍需要防護機制。檢舉、內容審查保留、隱藏的項目，以及帳號處置，有助於在版本或帳號疑似不安全、具有誤導性或違反政策時保護使用者。

本頁涵蓋內容審查與帳號狀態。關於 `Pass`、`Review`、`Warn`、`Malicious` 等稽核標籤與風險等級，請參閱[安全稽核](/clawhub/security-audits)。

另請參閱[安全性](/clawhub/security)與[可接受的使用方式](/clawhub/acceptable-usage)。如有著作權或其他內容權利疑慮，請使用[內容權利請求](/clawhub/content-rights)。

## 檢舉

已登入的使用者可以檢舉 Skills、外掛和套件。

ClawHub 檢舉僅適用於不安全的市集內容，例如：

- 惡意項目
- 誤導性的中繼資料
- 未聲明的認證資訊或權限要求
- 可疑的安裝指示
- 冒充他人
- 惡意搶註或濫用商標
- 違反[可接受的使用方式](/clawhub/acceptable-usage)的內容

請使用 Skill 頁面上的 **Report skill** 按鈕，或套件的檢舉命令/API。

請勿使用 ClawHub 檢舉來通報第三方 Skill 或外掛本身原始碼中的漏洞。請直接向發布者或項目所連結的原始碼儲存庫通報。ClawHub 不會維護或修補第三方 Skill 或外掛的程式碼。

`openclaw/clawhub` 的 GitHub Security Advisories 僅用於 ClawHub 本身的漏洞，例如網站、API、命令列介面、登錄檔、驗證、掃描、內容審查或下載/安裝信任邊界中的錯誤。請勿使用 ClawHub Advisories 來通報第三方 Skills 或外掛中的漏洞。

良好的檢舉應具體且可供採取行動。濫用檢舉功能本身也可能導致帳號處置。

## 組織與命名空間聲明

組織、品牌、套件範圍、擁有者代號或命名空間的所有權爭議，應使用[組織與命名空間聲明](/clawhub/namespace-claims)流程，而非產品內檢舉流程或帳號申訴表單。

當你需要 ClawHub 工作人員審查非敏感證明，以判定某個命名空間是否應予保留、移轉、重新命名、隱藏、隔離、設定別名或接受其他審查時，請使用此流程。請勿在公開 Issue 中加入機密、私人文件、非公開法律文件、個人身分證明文件、API 權杖或 DNS 驗證權杖。

## 內容審查保留

某些嚴重的發現或政策問題可能會使發布者或項目進入內容審查保留狀態。發生此情況時，受影響的內容可能會從公開探索中隱藏，或之後發布的內容可能一開始即為隱藏狀態，直到問題完成審查。

內容審查保留旨在 ClawHub 解決高風險案例期間保護使用者。確認為誤判後，也可以解除保留。

## 隱藏或封鎖的項目

項目可能會遭到保留、隱藏、隔離、撤銷，或以其他方式無法出現在公開安裝介面上。

如果你看到其中一種狀態，除非擁有者解決問題或內容審查人員恢復該版本，否則請勿安裝。

擁有者可能仍會看到自己遭保留或隱藏項目的診斷資訊。這些診斷資訊有助於說明發生的情況，以及項目恢復至公開介面前需要進行哪些變更。

## 封禁與帳號狀態

違反 ClawHub 政策的帳號可能會失去發布權限。嚴重濫用可能導致帳號遭封禁、權杖遭撤銷、內容遭隱藏或項目遭移除。系統會每日檢查發布者的濫用壓力訊號。達到 ClawHub 潛在封禁門檻的訊號可能會觸發自動警告。如果警告期限後的下一次合格掃描仍將發布者判定為達到潛在封禁門檻，ClawHub 可能會自動執行帳號處置。信心較低且限於特定時間範圍的審查訊號，不會用於自動執法。

已刪除、遭封禁或停用的帳號無法使用 ClawHub API 權杖。如果命令列介面的驗證在帳號處置後開始失敗，請登入網頁 UI 檢查帳號狀態。如果因帳號遭封禁或停用而無法登入或使用一般命令列介面存取，請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/)申請復原審查。

如果由掃描器觸發的電子郵件將某個 Skill 或外掛版本標示為惡意，請下載遭封鎖之已提交版本的已儲存掃描結果：`clawhub scan download <slug> --version <version>`。若為外掛，請加上 `--kind plugin`。檢查掃描輸出、修正項目、遞增版本號碼，然後上傳已修正的版本。

## 發布者指南

若要減少誤判並提升使用者信任：

- 確保名稱、摘要、標籤與變更記錄正確
- 聲明必要的環境變數與權限
- 避免經過混淆處理的安裝命令
- 盡可能連結至原始碼
- 發布外掛前先執行模擬運行
- 如果使用者或內容審查人員詢問版本行為，請清楚回覆
