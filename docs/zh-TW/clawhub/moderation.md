# 審核與帳戶安全

ClawHub 開放發布，但公開探索與安裝介面仍然需要防護措施。回報、審核保留、隱藏條目與帳戶處置，可在某個版本或帳戶看起來不安全、具誤導性或違反政策時協助保護使用者。

本頁說明審核與帳戶狀態。若要了解 `Pass`、`Review`、`Warn`、`Malicious` 等稽核標籤與風險等級，請參閱[安全性稽核](/clawhub/security-audits)。

另請參閱[安全性](/clawhub/security)與[可接受使用](/clawhub/acceptable-usage)。若有著作權或其他內容權利疑慮，請使用[內容權利請求](/clawhub/content-rights)。

## 回報

已登入的使用者可以回報技能、外掛與套件。

請僅將 ClawHub 回報用於不安全的市集內容，例如：

- 惡意條目
- 具誤導性的中繼資料
- 未聲明的憑證或權限要求
- 可疑的安裝指示
- 冒充
- 惡意註冊或商標濫用
- 違反[可接受使用](/clawhub/acceptable-usage)的內容

在技能頁面上使用**回報技能**按鈕，或使用套件回報命令/API 回報套件。

請勿使用 ClawHub 回報第三方技能或外掛自身原始碼中的漏洞。請直接向條目連結的發布者或原始碼儲存庫回報。ClawHub 不維護或修補第三方技能或外掛程式碼。

`openclaw/clawhub` 的 GitHub 安全性公告用於 ClawHub 本身的漏洞。例如網站、API、命令列介面、登錄檔、驗證、掃描、審核或下載/安裝信任邊界中的錯誤。請勿使用 ClawHub 公告回報第三方技能或外掛中的漏洞。

良好的回報應具體且可採取行動。濫用回報本身也可能導致帳戶處置。

## 組織與命名空間主張

組織、品牌、套件範圍、擁有者帳號或命名空間所有權爭議，應使用[組織與命名空間主張](/clawhub/namespace-claims)流程，而不是產品內回報流程或帳戶申訴表單。

當你需要 ClawHub 工作人員審查非敏感證明，以判斷某個命名空間是否應保留、移轉、重新命名、隱藏、隔離、設定別名或以其他方式審查時，請使用該流程。請勿在公開議題中包含秘密、私人文件、私人法律檔案、個人身分文件、API 權杖或 DNS 驗證權杖。

## 審核保留

某些嚴重發現或政策問題可能會使發布者或條目進入審核保留狀態。發生這種情況時，受影響的內容可能會從公開探索中隱藏，或未來發布可能會先以隱藏狀態開始，直到問題完成審查。

審核保留旨在於 ClawHub 解決高風險案例時保護使用者。當確認為誤判時，也可以解除保留。

## 隱藏或封鎖的條目

條目可能會被保留、隱藏、隔離、撤銷，或以其他方式無法在公開安裝介面上取得。

如果你看到其中一種狀態，除非擁有者解決問題或審核恢復該條目，否則請勿安裝該版本。

擁有者仍可看到自己被保留或隱藏條目的診斷資訊。這些診斷資訊有助於說明發生了什麼，以及條目返回公開介面之前需要變更哪些內容。

## 停權與帳戶狀態

違反 ClawHub 政策的帳戶可能會失去發布權限。嚴重濫用可能導致帳戶停權、權杖撤銷、內容隱藏或條目移除。發布者濫用壓力訊號會每日檢查。達到 ClawHub 潛在停權門檻的訊號可能會觸發自動警告。如果警告期限後的下一次符合資格掃描仍將發布者置於潛在停權門檻，ClawHub 可能會自動套用帳戶處置。較低信心度與有時間界限的審查訊號不會納入自動執行。

已刪除、停權或停用的帳戶無法使用 ClawHub API 權杖。如果命令列介面驗證在帳戶處置後開始失敗，請登入網頁 UI 以檢視帳戶狀態。如果因停權或停用帳戶而無法登入或使用一般命令列介面存取，請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/)進行復原審查。

如果由掃描器觸發的電子郵件將某個技能或外掛版本標示為惡意，請下載被封鎖提交版本的已儲存掃描結果：
`clawhub scan download <slug> --version <version>`。若是外掛，請加上 `--kind plugin`。檢閱掃描輸出、修正條目、遞增版本號，並上傳修正後的版本。

## 發布者指引

為減少誤判並提升使用者信任：

- 保持名稱、摘要、標籤與變更記錄準確
- 聲明必要的環境變數與權限
- 避免模糊化的安裝命令
- 可行時連結至原始碼
- 發布外掛前使用試執行
- 如果使用者或審核人員詢問版本行為，請清楚回應
