# 審核與帳號安全

ClawHub 開放發布內容，但公開探索與安裝介面仍需有防護措施。檢舉、審核保留、隱藏上架項目與帳號處置，可在版本或帳號疑似不安全、具誤導性或違反政策時協助保護使用者。

本頁說明審核與帳號狀態。關於 `Pass`、`Review`、`Warn`、`Malicious` 等稽核標籤及風險等級，請參閱[安全稽核](/clawhub/security-audits)。

另請參閱[安全性](/clawhub/security)與[可接受使用規範](/clawhub/acceptable-usage)。若有著作權或其他內容權利疑慮，請使用[內容權利申請](/clawhub/content-rights)。

## 檢舉

已登入的使用者可以檢舉 Skills、外掛與套件。

ClawHub 檢舉僅適用於不安全的市集內容，例如：

- 惡意上架項目
- 具誤導性的中繼資料
- 未聲明的認證資訊或權限要求
- 可疑的安裝指示
- 冒充他人
- 惡意搶註或濫用商標
- 違反[可接受使用規範](/clawhub/acceptable-usage)的內容

請使用 Skill 頁面上的 **Report skill** 按鈕，或使用套件檢舉命令/API 來檢舉套件。

請勿使用 ClawHub 檢舉功能通報第三方 Skill 或外掛自身原始碼中的漏洞。請直接向發布者或上架項目所連結的原始碼儲存庫通報。ClawHub 不維護或修補第三方 Skill 或外掛的程式碼。

`openclaw/clawhub` 的 GitHub Security Advisories 適用於 ClawHub 本身的漏洞，例如網站、API、命令列介面、登錄檔、驗證、掃描、審核或下載/安裝信任邊界中的錯誤。請勿使用 ClawHub 安全公告通報第三方 Skills 或外掛中的漏洞。

良好的檢舉應具體且可採取行動。濫用檢舉功能本身也可能導致帳號處置。

## 組織與命名空間權利主張

組織、品牌、套件範圍、擁有者代號或命名空間的所有權爭議，應使用[組織與命名空間權利主張](/clawhub/namespace-claims)程序，而非產品內檢舉流程或帳號申訴表單。

當你需要 ClawHub 工作人員審查非敏感證明，以判斷命名空間是否應予保留、移轉、重新命名、隱藏、隔離、設定別名或接受其他審查時，請使用該程序。請勿在公開議題中提供密鑰、私人文件、非公開法律檔案、個人身分文件、API 權杖或 DNS 驗證權杖。

## 審核保留

部分嚴重發現或政策問題可能使發布者或上架項目進入審核保留狀態。發生此情況時，受影響的內容可能會從公開探索中隱藏，或後續發布內容在問題完成審查前可能預設為隱藏。

審核保留旨在 ClawHub 處理高風險案件期間保護使用者。確認為誤判後，也可解除審核保留。

## 隱藏或封鎖的上架項目

上架項目可能遭到保留、隱藏、隔離、撤銷，或以其他方式無法在公開安裝介面使用。

若你看到其中任一狀態，除非擁有者解決問題或審核人員將其恢復，否則請勿安裝該版本。

擁有者可能仍可查看自己遭保留或隱藏之上架項目的診斷資訊。這些診斷資訊有助於說明發生了什麼情況，以及上架項目返回公開介面前必須變更的事項。

## 封禁與帳號狀態

違反 ClawHub 政策的帳號可能失去發布權限。嚴重濫用可能導致帳號遭封禁、權杖遭撤銷、內容遭隱藏或上架項目遭移除。系統每天都會檢查發布者的濫用壓力訊號。達到 ClawHub 潛在封禁門檻的訊號可能觸發自動警告。如果警告期限過後的下一次符合條件之掃描，仍將該發布者判定為達到潛在封禁門檻，ClawHub 可能會自動採取帳號處置。信心較低且具有時間範圍限制的審查訊號不會納入自動執行措施。

已刪除、遭封禁或停用的帳號無法使用 ClawHub API 權杖。如果命令列介面驗證在帳號處置後開始失敗，請登入網頁 UI 查看帳號狀態。如果封禁或帳號停用導致無法登入或無法正常存取命令列介面，請使用 [ClawHub 申訴表單](https://appeals.openclaw.ai/)申請復原審查。

如果掃描器觸發的電子郵件將某個 Skill 或外掛版本標示為惡意，請下載遭封鎖之提交版本所儲存的掃描結果：
`clawhub scan download <slug> --version <version>`。若為外掛，請加入
`--kind plugin`。檢閱掃描輸出、修正上架項目、遞增版本號碼，然後上傳修正後的版本。

## 發布者指南

若要減少誤判並提升使用者信任：

- 確保名稱、摘要、標籤與變更記錄正確無誤
- 聲明必要的環境變數與權限
- 避免經過混淆處理的安裝命令
- 盡可能連結至原始碼
- 發布外掛前先使用試執行
- 若使用者或審核人員詢問版本行為，請清楚回覆
