`/btw`（別名 `/side`）可針對**目前工作階段**快速詢問支線問題，而不會將其加入對話歷程。其設計參考 Claude Code 的 `/btw`，並配合 OpenClaw 的閘道與多頻道架構調整。

```text
/btw 有哪些變更？
/side 這個錯誤是什麼意思？
```

## 功能說明

1. 擷取目前工作階段的快照作為背景脈絡（包括任何進行中的主要執行提示）。
2. 執行獨立的一次性支線查詢，要求模型僅回答支線問題，不要繼續或引導主要任務。
3. 將答案以即時支線結果傳送，而非一般助理訊息。
4. 絕不將問題或答案寫入工作階段歷程或 `chat.history`。

如果主要執行仍在進行，將完全不受影響。

對於 Codex 控制框架工作階段，BTW 會將作用中的 Codex app-server 執行緒分叉為暫時性的子執行緒，而不是另外呼叫提供者。這可維持 Codex OAuth 及原生工具／執行緒行為不變，分叉的執行緒也會保留父執行緒目前的核准政策、沙箱及原生工具介面。分叉的執行緒會收到一個邊界提示，告知模型邊界之前的所有內容都是繼承的參考脈絡，而非作用中的指示，且只有邊界之後的訊息才是即時訊息。`/btw` 需要既有的 Codex 執行緒；請先傳送一般訊息。

對於命令列介面執行階段別名，BTW 會以一次性支線問題模式叫用其所屬的命令列介面後端：它會將經過清理的對話脈絡植入新的命令列介面叫用、停用工具套裝與可重複使用的工作階段狀態，並加入後端支援的任何禁止繼續／禁止工具旗標。直接（非命令列介面）執行階段則改用直接的一次性提供者呼叫。

## 不會執行的動作

`/btw` 不會建立持久工作階段、繼續尚未完成的主要任務、將問題／答案資料保存到文字記錄歷程，也無法在重新載入後保留。

## 傳送模型

一般助理聊天使用閘道 `chat` 事件。BTW 使用獨立的 `chat.side_result` 事件，因此用戶端不會將其誤認為一般對話歷程。由於不會從 `chat.history` 重播，因此重新載入後就會消失。

## 各介面行為

| 介面              | 行為                                                                                                                                                                                                                                                                                    |
| ----------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 終端介面          | 內嵌顯示於聊天記錄中，外觀與一般回覆明顯不同，可按 `Enter` 或 `Esc` 關閉。                                                                                                                                                                                                               |
| 外部頻道          | 以清楚標示的一次性回覆傳送（Telegram、WhatsApp、Discord 沒有本機暫時性浮動層）。                                                                                                                                                                                                         |
| 控制介面／網頁    | 顯示為固定在線程上的浮動「支線聊天」面板。答案會逐輪累積，而「繼續提問」輸入欄可用來詢問下一個支線問題。關閉（按 `Esc` 或 X）會保留對話，並在收到下一個答案時重新開啟；垃圾桶按鈕則會將其捨棄並停止待處理的執行。 |

## 選取快顯視窗（控制介面）

在控制介面的聊天訊息中反白文字時，會開啟一個含有兩個動作的小型選取快顯視窗：

- **更多詳細資料**會立即傳送一個隱含的 `/btw` 問題，要求模型根據目前工作階段的脈絡解釋反白文字。答案會顯示在浮動的支線聊天面板中。
- **在支線聊天中詢問**會在撰寫器中預先填入一份引用反白文字的 `/btw` 草稿，讓你可以輸入自己的相關問題。

這兩個動作都遵循一般 `/btw` 語意：問題與答案不會進入工作階段歷程，主要執行也不受影響。

## 使用時機

當你需要快速釐清、在長時間執行仍進行時取得事實性的支線答案，或取得不應進入未來工作階段脈絡的暫時性答案時，請使用 `/btw`。

```text
/btw 我們正在編輯哪個檔案？
/btw 用一句話摘要目前的任務
/btw 17 * 19 是多少？
```

如果你希望內容成為工作階段未來工作脈絡的一部分，請改在主要工作階段中正常提問。
