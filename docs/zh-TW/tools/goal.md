# 目標

**目標**是附加至目前 OpenClaw 工作階段的一項持久性目的。
它為代理程式與操作者提供長時間工作所需的共同標的，
而不會將該標的轉變為背景任務、提醒、排程工作或
常設指令。

目標屬於工作階段狀態：它們會隨工作階段金鑰移動、在程序
重新啟動後繼續存在，並顯示於 `/goal`、模型所使用的目標工具，以及終端介面
頁尾中。

分離的命令完成結果會傳回原本面向使用者的討論串，因此
即使命令執行使用了不同的沙箱政策工作階段，下一輪仍會看到
相同的目標。

## 快速開始

```text
/goal start 讓 PR 87469 的 CI 通過並推送修正
/goal
/goal edit 讓 PR 87469 的 CI 通過、推送修正，並更新文件
/goal pause 等待 CI
/goal resume
/goal complete 已推送並驗證
/goal clear
```

`start` 為選用：`/goal get CI green for PR 87469` 也會建立目標，
因為 `/goal` 之後任何不是已知動作詞的文字，都會被視為
新的目的。

## 目標的用途

當工作階段具有應在多輪互動中持續可見的具體成果時，
請使用目標：

- PR 收尾：修正、驗證、自動審查、推送，以及開啟或更新 PR。
- 偵錯流程：重現錯誤、找出所屬介面、修補，並
  證明修正有效。
- 文件作業：閱讀相關文件、撰寫新頁面、加入交叉連結，並
  驗證文件建置。
- 維護任務：檢查目前狀態、進行範圍明確的變更、執行
  正確的檢查，並報告變更內容。

目標不是任務佇列。若工作應分離執行、
依排程重複、展開為受管理的子工作，或持續作為政策，請使用
[Task Flow](/zh-TW/automation/taskflow)、[任務](/zh-TW/automation/tasks)、[排程工作](/zh-TW/automation/cron-jobs)或
[常設指令](/zh-TW/automation/standing-orders)。

## 命令參考

不帶引數的 `/goal` 會顯示目前的目標摘要：

```text
目標
狀態：進行中
目的：讓 PR 87469 的 CI 通過並推送修正
已使用權杖：12k
權杖預算：12k/50k

命令：/goal edit <objective>、/goal pause、/goal complete、/goal clear
```

| 命令                                                | 效果                                                                     |
| --------------------------------------------------- | ------------------------------------------------------------------------ |
| `/goal` 或 `/goal status`            | 顯示目前的目標。                                                         |
| `/goal start <objective>`                                  | 為目前工作階段建立新目標。                                               |
| `/goal set <objective>`、`/goal create <objective>`              | `start` 的別名。                                              |
| `/goal <objective>`                                  | 也會建立新目標（任何不是已辨識動作詞的文字）。                           |
| `/goal edit <objective>`                                  | 重新表述目前的目的；狀態與權杖計量保持不變。                             |
| `/goal pause [note]`                                  | 暫停進行中的目標。                                                       |
| `/goal resume [note]`                                  | 繼續已暫停、受阻、受用量限制或受預算限制的目標。                         |
| `/goal complete [note]`                                  | 將目標標示為已達成。                                                     |
| `/goal done [note]`                                  | `complete` 的別名。                                              |
| `/goal block [note]`                                  | 將目標標示為受阻。                                                       |
| `/goal blocked [note]`                                  | `block` 的別名。                                              |
| `/goal clear`                                  | 從工作階段移除目標。                                                     |

每個工作階段一次只能有一個目標。在目前目標清除前，啟動第二個目標會失敗，
並顯示 `Goal error: goal already exists`。

`/goal start` 不接受權杖預算旗標；預算只能透過
模型所使用的 `create_goal` 工具設定。

## 狀態

- `active`：工作階段正在執行目標。
- `paused`：操作者已暫停目標；`/goal resume` 會使其再次進入進行中
  狀態。
- `blocked`：代理程式或操作者回報了實際的阻礙；當有新的資訊或狀態可用時，
  `/goal resume` 會使其再次進入進行中狀態。
- `budget_limited`：已達設定的權杖預算；`/goal resume`
  會從相同目的重新開始執行，並使用新的預算區間。
- `usage_limited`：保留供未來的用量限制停止狀態使用；`/goal
resume` 會以相同方式重新開始執行。
- `complete`：目標已達成。已完成的目標是終止狀態；開始另一個目標前，請先使用 `/goal
clear`。

`/new` 和 `/reset` 會清除目前工作階段的目標，因為它們會刻意
以全新的工作階段內容開始。

## 權杖預算

目標可以設定選用的正數權杖預算，透過
`create_goal` 工具的 `token_budget` 參數設定。預算從
建立目標時工作階段的最新權杖計數開始計量。如果目標開始時，
工作階段只有過時或未知的權杖快照，OpenClaw 會等待
下一個最新快照並以其作為基準，因此目標存在前所使用的權杖
不會計入其中。

當用量達到預算時，目標會移至 `budget_limited`。這不會
刪除目標或清除目的；它會告知操作者與
代理程式，除非目標已繼續或清除，否則目前不再主動執行該目標。
繼續目標會從目前最新的權杖
計數開始新的預算區間。

權杖預算是工作階段目標的防護界線，而非計費上限。供應商
配額、成本報告和內容視窗行為仍會使用一般的
OpenClaw 用量與模型控制。

## 模型工具

OpenClaw 向代理程式執行框架提供三個目標工具：

| 工具              | 用途                                                                                                                        |
| ----------------- | --------------------------------------------------------------------------------------------------------------------------- |
| `get_goal` | 讀取目前的工作階段目標：狀態、目的、權杖用量和權杖預算。                                                                   |
| `create_goal` | 僅在使用者或系統指示明確要求時建立目標。若工作階段已有目標，則會失敗。                                                       |
| `update_goal` | 將目標標示為 `complete` 或 `blocked`。                                                                     |

模型無法暗中暫停、繼續、清除或取代目標。這些仍是透過
`/goal` 和重設命令進行的操作者／工作階段控制，因此代理程式
可以回報已達成或實際的阻礙，而不會悄悄移動
標的。

`update_goal` 只有在目的
確實達成時，才應將目標標示為 `complete`。只有在相同的
阻礙條件連續至少三個目標回合重複發生後，才應將目標標示為 `blocked`，而不是因為
一般的困難或缺少潤飾。

## 每一輪的目標內容

每個具有進行中目標的使用者／聊天回合，都會包含這一行使用者角色內容：

```text
進行中的目標：<objective> — 推進目標或更新其狀態（get_goal/update_goal）。
```

OpenClaw 會截斷過長的目的，讓此行保持精簡。已暫停、
受阻、受預算限制、受用量限制及已完成的目標不會被注入，
因此操作者的停止狀態會持續有效，直到目標繼續為止。

## 控制介面

網頁控制介面會在聊天輸入框上方以精簡膠囊顯示目標：
包含狀態圖示、狀態標籤（例如 `Pursuing goal`）、截斷後的
目的，以及即時經過時間計時器。

膠囊提供行內控制項：

- **鉛筆**會在輸入框預填 `/goal edit <objective>`，以便
  重新表述並提交目的。
- **暫停／繼續**會依據
  目前狀態在 `/goal pause` 和 `/goal resume` 之間切換。
- **垃圾桶**會傳送 `/goal clear`。
- **箭頭**會展開膠囊，以顯示完整目的、最新狀態
  備註、權杖用量和經過時間。

當輸入框無法傳送時（例如
閘道連線中斷），動作按鈕會隱藏；展開箭頭仍可運作。

## 終端介面

終端介面頁尾會在權杖／模式指示器之前，將進行中工作階段的目標顯示在代理程式、
工作階段及模型欄位旁。

頁尾範例：

- `Pursuing goal (12k/50k)` 表示具有權杖預算的進行中目標。
- `Goal paused (/goal resume)` 表示已暫停的目標。
- `Goal blocked (/goal resume)` 表示受阻的目標。
- `Goal hit usage limits (/goal resume)` 表示受用量限制的目標。
- `Goal unmet (50k/50k)` 表示受預算限制的目標。
- `Goal achieved (42k)` 表示已完成的目標。

頁尾刻意保持精簡。請使用 `/goal` 查看完整目的、
備註、權杖預算及可用命令。

## 頻道行為

`/goal` 可在支援命令的 OpenClaw 工作階段中運作，包括終端介面和
允許文字命令的聊天介面。目標狀態附加至
工作階段金鑰，而不是傳輸方式，因此共享工作階段金鑰的兩個介面會看到
相同的目標。

目標狀態不是傳遞指令：它不會強制透過某個
頻道回覆、變更佇列行為、核准工具或排程工作。

## 疑難排解

| 訊息                                   | 意義                                                                                                                                                     |
| -------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Goal error: goal already exists`                     | 工作階段已有目標。使用 `/goal` 檢查它；若已完成，請使用 `/goal complete`；或在開始不同的目的前使用 `/goal clear`。                      |
| `Goal error: goal not found`                     | 工作階段尚無目標。使用 `/goal start <objective>` 啟動目標。                                                                                                    |
| `Goal error: goal is already complete`                     | 目標已進入終止狀態。開始或繼續另一個目的前，請先將其清除。                                                                                               |

如果權杖用量顯示 `0` 或看起來已過時，進行中的工作階段可能尚無
最新的權杖快照。當 OpenClaw 記錄工作階段用量
及根據逐字稿計算的總量時，用量便會重新整理。
