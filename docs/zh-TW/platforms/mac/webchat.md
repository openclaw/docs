macOS 選單列 App 將 WebChat UI 內嵌為原生 SwiftUI 檢視。它會連線至閘道，並預設使用所選代理程式的主要工作階段（`main`，或當 `session.scope` 為 `global` 時使用 `global`）。

完整聊天視窗是原生分割檢視：

- **工作階段側邊欄**：可搜尋的工作階段清單，包含已釘選、由閘道支援的群組及最近使用區段。在每個區段中，衍生的子工作階段會巢狀顯示於其父工作階段下方；收合的父工作階段會摘要執行中、失敗及未讀的後代工作階段。內容選單支援工作階段資訊、重新命名、釘選、分支、標示已讀／未讀、封存／還原、複製工作階段金鑰及刪除。主要的新工作階段動作（或 Cmd-N）會立即透過 `sessions.create` 建立工作階段；其相鄰的選項彈出式視窗可選取代理程式，並要求建立含選用基底參照的受管理工作樹。
- **視窗工具列**：內容用量環（權杖與工作階段成本，並提供精簡動作）、模型控制項，以及工作階段動作選單。模型依提供者分組，預設提供者排在最前面，而已釘選和最近使用的模型仍會置頂。控制項可繼承或覆寫模型的思考層級、選擇工具呼叫的詳細程度，以及切換快速回應。選單可重新命名或分支目前的工作階段，並更新其釘選、已讀或封存狀態。**工作階段…**（Shift-Cmd-S）會開啟「使用中／已封存」管理器，以進行閘道搜尋、群組管理、工作階段檢查、重新命名、釘選、封存及還原。選取模式可將釘選、取消釘選、封存或刪除套用至多個使用中的工作階段，同時讓個別失敗項目保持可見。個別選單核取記號可顯示或隱藏助理推理與工具活動；兩者預設皆開啟，並會在不同啟動階段之間記住設定。
- **對話記錄與編輯器**：助理訊息會以附有頭像的純文字呈現，使用者訊息則呈現為強調色對話泡泡。待處理的代理程式問題會呈現為原生卡片，包含單選或多選選項、可自由輸入文字的**其他**答案、到期倒數計時，以及共用的終止狀態。空白聊天會提供桌面版起始提示。在輸入 `/` 時，會開啟由 `commands.list` 支援的斜線命令自動完成，並可使用方向鍵／Tab／Return／Escape 進行鍵盤導覽。在訊息上按一下右鍵，可複製其可見的 Markdown，而不包含隱藏的推理。遭截斷的助理訊息也會提供**開啟完整訊息**，載入可選取文字的 Markdown 閱讀器。使用**聆聽**可透過閘道進行 TTS，並以本機語音作為備援。
- **語音控制項**：編輯器可啟動或停止現有的 macOS 對話模式，而不會取代其選單列覆疊介面。對話模式啟用時，編輯器會顯示其聆聽／思考／說話狀態、即時音訊活動，以及可展開的滾動對話記錄。在「對話」按鈕上按一下右鍵，可選擇**系統預設值**或已連線的麥克風；這與語音喚醒及按住說話所使用的麥克風選項相同。若選取的麥克風中斷連線，進行中的對話工作階段會改用系統預設值，並在下次啟動對話模式時再次嘗試使用該選項。當對話模式未占用音訊擷取時，另有一個麥克風動作可錄製語音留言。

選單列中的錨定精簡聊天面板會維持精簡的單欄版面配置，並在行內提供相同的模型、思考、詳細程度與快速控制項，以及起始提示、對話模式、語音留言和聆聽功能。助理推理與工具活動在此精簡介面中仍會保持隱藏。

## 快速聊天列

按下 Option-Space（⌥Space），或從選單列選單中選擇 **Quick Chat**，即可開啟主要工作階段的浮動編輯器。可使用 **Settings → General → Quick Chat shortcut** 中的錄製器變更全域快速鍵。

快速聊天會顯示目標代理程式（頭像或表情符號，並以代理程式名稱作為預留位置），且會傳送至該代理程式的主要工作階段。Return 接受傳送後，聊天列會保持開啟，並向下展開以顯示串流 Markdown 回覆與最近的對話記錄。聊天列輸入欄仍作為編輯器使用。按下 Command-Return 可傳送訊息，並在完整聊天視窗中開啟相同目標；按下 Shift-Return 可換行，或按下 Escape 關閉整個聊天列與回覆區域。按一下外部區域也會將其關閉。若缺少相關的 macOS 權限，附加的橫幅會提供 **Grant** 和 **Not now** 動作。

使用麥克風按鈕將語音聽寫至編輯器。部分語音辨識結果會即時取代聽寫範圍，同時保留編輯器中原有的文字。再次按下該按鈕、Return 或 Escape 即可停止；傳送、隱藏快速聊天或使其失去焦點，也會釋放麥克風。首次使用時會要求 macOS 的「麥克風」與「語音辨識」存取權限。

精簡模型控制項會顯示目標工作階段目前的模型與推理層級。模型選項會更新該工作階段，因此會保留於該處；推理選項則僅套用至目前快速聊天顯示期間所傳送的每則訊息。聊天列隱藏時，本機選項會重設。切換代理程式或選擇最近的工作階段時，會保留明確選擇的項目，但會重新載入新目標工作階段的基礎模型狀態。

按一下歷史記錄按鈕，可從最近更新的五個工作階段中選擇，或返回**傳送新訊息給 &lt;agent&gt;**。選取最近的工作階段後，訊息會傳送至該確切工作階段，並將預留位置變更為**在 &lt;session&gt; 中回覆**。隱藏快速聊天會將此暫時目標重設為所選代理程式的主要工作階段；從頭像選單切換代理程式也會清除該目標。

Command-Return 會開啟接收該次傳送之代理程式的對話，即使工作階段範圍是全域亦同。

相機按鈕會開啟包含**擷取視窗…**或**擷取區域…**的選單。視窗擷取會標示每個可見視窗；區域擷取則會在你拖曳區域時調暗每個顯示器，並顯示該區域的即時大小。選取的螢幕截圖會連同任何輸入文字作為說明文字，傳送給所選代理程式。首次使用時會要求 macOS 的「螢幕錄製」存取權限。按下 Escape、按一下空白處，或按一下但未拖曳出具實際範圍的區域，都會取消操作。

使用文件文字按鈕，可附加焦點 App 之焦點視窗中的文字。快速聊天會將結果顯示為可移除的內容標籤，而非將擷取的文字放入編輯器；傳送時會將標籤文字附加至傳出訊息，然後將其清除。此功能需要 macOS 的「輔助使用」權限。每當快速聊天關閉時，附加文字也會清除，因此某次顯示期間的內容不會洩漏到稍後傳送的訊息中。

回覆完成後，選擇**貼到 &lt;app&gt;**，即可將其可見的助理文字（不包含隱藏的推理）複製到一般剪貼簿，並貼入先前位於最前方的 App。此功能需要 macOS 的「輔助使用」權限。此動作會取代目前的剪貼簿內容，然後隱藏快速聊天。

使用 **Settings → General → Quick Chat** 可完全停用此功能；快速鍵錄製器也位於同一區段。

- **本機模式**：直接連線至本機閘道 WebSocket。
- **遠端模式**：透過 SSH 轉送閘道控制連接埠，並使用該通道作為資料平面。

## 啟動與偵錯

- 手動：Lobster 選單 ->「開啟聊天」。
- 測試時自動開啟：

  ```bash
  dist/OpenClaw.app/Contents/MacOS/OpenClaw --chat
  ```

  （`--webchat` 可作為舊版別名使用。）

- 記錄：`./scripts/clawlog.sh`（子系統 `ai.openclaw`，類別 `WebChatSwiftUI`）。

## 運作方式

- 資料平面：閘道 WS 方法 `chat.history`、`chat.message.get`、`chat.send`、`chat.abort`、`chat.inject`，以及 `question.list` 和 `question.resolve`，另有事件 `chat`、`agent`、`presence`、`tick`、`health`；問題卡片會遵循 `question.requested` 和 `question.resolved` 事件，並在重新連線後從 `question.list` 重新整理。
- `chat.history` 會傳回經顯示正規化的對話記錄：從可見文字中移除行內指令標籤、純文字工具呼叫 XML 承載資料（`<tool_call>`、`<function_call>`、`<tool_calls>`、`<function_calls>`，包括遭截斷的區塊）及洩漏的模型控制權杖；省略只包含靜默權杖的助理列，例如完全等於 `NO_REPLY`/`no_reply` 的列；過大的列則可替換為截斷預留位置。
- 工作階段：如上所述，預設使用主要工作階段；UI 可在不同工作階段之間切換。
- 工作階段群組：`sessions.groups.list`、`sessions.groups.put`、`sessions.groups.rename` 和 `sessions.groups.delete` 負責管理群組目錄。成員資格是透過 `sessions.patch` 更新的工作階段 `category`。
- 未讀狀態：工作階段啟用且其即時歷史記錄成功載入後，App 會清除該工作階段的未讀標記。歷史記錄載入失敗時不會清除；暫時性的修補失敗會在下次啟用時重試。
- 初始設定會使用專用工作階段，以將首次執行設定分開。
- 離線快取：App 會為每個閘道保留近期聊天工作階段與對話記錄的小型唯讀快取（`~/Library/Application Support/OpenClaw/chat-cache.sqlite`）：冷啟動時會立即顯示最後已知的對話記錄，並在閘道回應後重新整理；中斷連線時仍可瀏覽最近的聊天（傳送功能會保持停用，直到連線恢復）。

## 安全性介面

- 遠端模式僅透過 SSH 轉送閘道 WebSocket 控制連接埠。

## 已知限制

- 此 UI 針對聊天工作階段最佳化，而非完整的瀏覽器沙箱。
