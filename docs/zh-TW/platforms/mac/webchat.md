macOS 選單列 App 會將 WebChat UI 內嵌為原生 SwiftUI 檢視。它會連線至閘道，並預設使用所選代理程式的主要工作階段（`main`，或當 `session.scope` 為 `global` 時使用 `global`）。

完整聊天視窗是原生分割檢視：

- **工作階段側邊欄**：可搜尋的工作階段清單，包含已釘選、閘道支援的群組和最近使用區段。衍生的子工作階段會在各區段中巢狀顯示於其父工作階段下方；摺疊的父工作階段會摘要執行中、失敗和未讀的後代工作階段。內容選單支援工作階段資訊、重新命名、釘選、分支、已讀/未讀、封存/還原、複製工作階段金鑰和刪除。主要的新工作階段動作（或 Shift-Cmd-N）會立即透過 `sessions.create` 建立；其旁的選項彈出視窗可選擇代理程式，並要求建立具有選用基底參照的受管理工作樹。
- **視窗工具列**：內容使用量環形圖（權杖與工作階段成本，並附精簡動作）、模型控制項和工作階段動作選單。模型會依供應商分組，預設供應商排在最前面，而已釘選和最近使用的模型則維持在頂端。控制項可繼承或覆寫模型的思考層級、選擇工具呼叫詳細程度，並切換快速回應。選單可重新命名目前的工作階段或為其建立分支，並更新其釘選、已讀或封存狀態。**工作階段⋯**（Shift-Cmd-S）會開啟「使用中/已封存」管理器，以進行閘道搜尋、群組管理、工作階段檢查、重新命名、釘選、封存和還原。選取模式可對多個使用中的工作階段套用釘選、取消釘選、封存或刪除，同時保持各別失敗狀態可見。獨立的選單勾選標記可顯示或隱藏助理推理與工具活動；兩者預設皆開啟，且會跨啟動記住設定。
- **對話記錄與撰寫器**：助理訊息會以附有頭像的純文字呈現，使用者訊息則以強調色對話框呈現。待處理的代理程式問題會顯示為原生卡片，包含單選或多選選項、自由文字的**其他**答案、到期倒數計時和共用終止狀態。空白聊天會提供桌面版入門提示。在輸入 `/` 時，會開啟由 `commands.list` 支援的斜線命令自動完成，並支援方向鍵/Tab/Return/Escape 鍵盤導覽。在訊息上按右鍵，即可複製其可見的 Markdown，而不含隱藏的推理。遭截斷的助理訊息也會提供**開啟完整訊息**，以載入可選取文字的 Markdown 閱讀器。使用**聆聽**即可透過閘道進行 TTS，並以本機語音作為備援。
- **語音控制項**：撰寫器可啟動或停止現有的 macOS 對話模式，而不取代其選單列覆疊介面。對話模式啟用時，撰寫器會顯示其聆聽/思考/說話狀態、即時音訊活動，以及可展開的滾動對話記錄。在「對話」按鈕上按右鍵，即可選擇**系統預設值**或已連線的麥克風；這與語音喚醒和按住說話所使用的麥克風選擇相同。如果所選麥克風中斷連線，使用中的對話工作階段會改用系統預設值，並在下次啟動對話模式時再次嘗試該選項。當對話模式未占用音訊擷取時，獨立的麥克風動作可錄製語音留言。

選單列的錨定式精簡聊天面板會保留精簡的單欄版面配置，並在行內提供相同的模型、思考、詳細程度和快速控制項，另包含入門提示、對話模式、語音留言和聆聽功能。助理推理和工具活動在此精簡介面中仍會隱藏。

## 多個閘道視窗

開啟**設定 → 閘道**以新增或移除可重複使用的閘道設定檔。每個
設定檔都包含私人網路 `ws://` 或安全的 `wss://` 端點及其
選用權杖或密碼；認證資訊會儲存在 macOS 鑰匙圈中。
安全設定檔會維護自己的首次使用憑證釘選，並由系統信任機制控管，
且不會從主要閘道繼承 `gateway.remote.tlsFingerprint`。
移除設定檔也會關閉其已開啟的視窗，並關閉其次要
連線。

選擇**檔案 → 新增閘道視窗⋯**或按下 Cmd-N，然後選取其中一個
已儲存的設定檔。選擇器會記住最近使用的設定檔。每次
選取都會建立一個新的獨立視窗，因此同一個閘道可以顯示在
多個視窗中，並具有不同的使用中工作階段和導覽狀態。

每個已儲存的設定檔都擁有一個共用閘道連線、裝置驗證範圍、
對話記錄快取、離線寄件匣和路由租約。該設定檔的各個視窗
會重複使用這些資源，同時維持可獨立導覽。不同設定檔的視窗
會保持連線並同時執行聊天。

選單列 App 所設定的閘道仍是 Mac 節點
功能和對話模式的擁有者。額外的閘道視窗僅供操作者使用，因此
第二個閘道無法在未提示的情況下重新指定全域麥克風或裝置控制項。
聆聽/TTS 和一般聊天動作會使用該視窗自己的閘道連線。

## 快速聊天列

按下 Option-Space（⌥Space），或從選單列選單中選擇**快速聊天**，即可開啟主要工作階段的浮動撰寫器。使用**設定 → 一般 → 快速聊天快捷鍵**中的錄製器變更全域快捷鍵。

快速聊天會顯示目標代理程式（頭像或表情符號，並以代理程式名稱作為預留位置），並傳送至該代理程式的主要工作階段。按下 Return 接受傳送後，聊天列會保持開啟，並向下展開以顯示串流 Markdown 回覆和最近的對話記錄。聊天列輸入區仍是撰寫器。按下 Command-Return 可傳送訊息，並在完整聊天視窗中開啟相同目標；按 Shift-Return 可換行；按 Escape 則會關閉整個聊天列和回覆區域。按一下外部區域也會將其關閉。缺少相關 macOS 權限時，附加的提示列會提供**授予**和**暫時不要**動作。

使用麥克風按鈕將語音聽寫至撰寫器。部分語音結果會即時取代聽寫範圍，同時保留撰寫器中原有的文字。再次按下按鈕、Return 或 Escape 即可停止；傳送、隱藏快速聊天或使其失去焦點，也會釋放麥克風。首次使用時會要求 macOS 麥克風和語音辨識存取權。快速聊天使用 Apple 語音功能，並可能使用其網路服務；只有被動式語音喚醒需要裝置端辨識。

精簡模型控制項會顯示目標工作階段目前的模型和推理層級。模型選項會更新該工作階段，因此會持續保留；推理選項則只會套用至目前快速聊天介面傳送的每則訊息。聊天列隱藏時，本機選項會重設。切換代理程式或選擇最近的工作階段時，會保留明確設定的選項，但重新載入新目標工作階段的基礎模型狀態。

按一下歷史記錄按鈕，即可從最近更新的五個工作階段中選擇，或返回**傳送新訊息給 &lt;agent&gt;**。選擇最近的項目後，會將訊息傳送至該特定工作階段，並將預留位置變更為**在 &lt;session&gt; 中回覆**。隱藏快速聊天會將這個暫時目標重設為所選代理程式的主要工作階段；從頭像選單切換代理程式也會將其清除。

Command-Return 會開啟收到該次傳送之代理程式的對話，包括工作階段範圍為全域時。

相機按鈕會開啟選單，提供**擷取視窗⋯**或**擷取區域⋯**。擷取視窗時會標示每個可見視窗；擷取區域時，會在你拖曳區域的過程中調暗各個顯示器，並顯示即時尺寸。所選螢幕截圖會連同任何輸入文字作為說明文字，傳送給所選代理程式。首次使用時會要求 macOS 螢幕錄製存取權。按下 Escape、按一下空白區域，或未實際拖曳出有效區域便按一下，都會取消操作。

使用文件文字按鈕附加目前聚焦 App 之聚焦視窗中的文字。快速聊天會將結果顯示為可移除的內容標籤，而不會將擷取的文字放入撰寫器；傳送時會將標籤文字附加到送出的訊息，然後將其清除。此功能需要 macOS 輔助使用權限。每當快速聊天關閉時，附加的文字也會清除，因此某次顯示中的內容不會洩漏到後續傳送。

回覆完成後，選擇**貼到 &lt;app&gt;**，即可將其可見的助理文字（不含隱藏的推理）複製到一般剪貼板，並貼入先前位於最前方的 App。此功能需要 macOS 輔助使用權限。此動作會取代目前的剪貼板內容，然後隱藏快速聊天。

使用**設定 → 一般 → 快速聊天**可完全停用此功能；同一區段也包含快捷鍵錄製器。

- **本機模式**：直接連線至本機閘道 WebSocket。
- **遠端模式**：使用已設定的直接 `ws://`/`wss://` 路由，或由 App 管理的 SSH 通道作為資料平面。

## 啟動與偵錯

- 手動：Lobster 選單 -> “開啟聊天”。
- 測試時自動開啟：

  ```bash
  dist/OpenClaw.app/Contents/MacOS/OpenClaw --chat
  ```

  （`--webchat` 可作為舊版別名使用。）

- 記錄：`./scripts/clawlog.sh`（子系統 `ai.openclaw`，類別 `WebChatSwiftUI`）。

## 連接方式

- 資料平面：閘道 WS 方法 `chat.history`、`chat.message.get`、`chat.send`、`chat.abort`、`chat.inject`，以及 `question.list` 和 `question.resolve`；另有事件 `chat`、`agent`、`presence`、`tick`、`health`；問題卡片會依循 `question.requested` 和 `question.resolved` 事件，並在重新連線後從 `question.list` 重新整理。
- `chat.history` 會傳回經過顯示正規化的對話記錄：從可見文字中移除行內指令標籤；移除純文字工具呼叫 XML 承載內容（`<tool_call>`、`<function_call>`、`<tool_calls>`、`<function_calls>`，包括遭截斷的區塊）及外洩的模型控制權杖；省略僅含靜默權杖的助理列，例如完全符合 `NO_REPLY`/`no_reply` 的列；過大的列則可替換為截斷預留位置。
- 工作階段：如上所述，預設使用主要工作階段；UI 可在不同工作階段之間切換。
- 工作階段群組：`sessions.groups.list`、`sessions.groups.put`、`sessions.groups.rename` 和 `sessions.groups.delete` 擁有群組目錄。成員資格是透過 `sessions.patch` 更新的工作階段 `category`。
- 未讀狀態：工作階段啟用且即時歷史記錄成功載入後，App 會清除該工作階段的未讀標記。歷史記錄載入失敗時不會清除；暫時性的修補失敗會在下次啟用時重試。
- 初始設定會使用專用工作階段，將首次執行設定分開處理。
- 離線快取：App 會為每個閘道保留少量唯讀的最近聊天工作階段和對話記錄快取（`~/Library/Application Support/OpenClaw/chat-cache.sqlite`）：冷啟動時會立即呈現最後已知的對話記錄，並在閘道回應後重新整理；中斷連線時仍可瀏覽最近的聊天（在連線恢復前，傳送功能會保持停用）。

## 安全介面

- 遠端模式只會透過 SSH 轉送閘道 WebSocket 控制連接埠。

## 已知限制

- 此 UI 針對聊天工作階段最佳化，而不是完整的瀏覽器沙箱。
