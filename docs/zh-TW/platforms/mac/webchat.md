macOS 選單列 App 將 WebChat UI 嵌入為原生 SwiftUI 檢視。它會連線至閘道，並預設使用所選代理程式的主要工作階段（`main`；當 `session.scope` 為 `global` 時則為 `global`）。

完整聊天視窗是原生分割檢視：

- **工作階段側邊欄**：可搜尋的工作階段清單，分為已釘選、由閘道支援的群組及最近使用區段。衍生的子工作階段會在各區段中巢狀顯示於其父工作階段下方；收合的父工作階段會彙整執行中、失敗及未讀的後代工作階段。內容選單支援工作階段資訊、重新命名、釘選、分叉、標示已讀／未讀、封存／還原、複製工作階段金鑰及刪除。主要的新工作階段動作（或 Shift-Cmd-N）會立即透過 `sessions.create` 建立工作階段；相鄰的選項彈出視窗可選取代理程式，並要求建立具有選用基底參照的受管理工作樹。
- **視窗工具列**：內容用量環（權杖和工作階段成本，附精簡動作）、模型控制項及工作階段動作選單。模型依供應商分組，預設供應商列於最前，而已釘選和最近使用的模型仍會置頂。控制項可繼承或覆寫模型的思考層級、選擇工具呼叫的詳細程度，以及切換快速回應。選單可重新命名或分叉目前的工作階段，並更新其釘選、已讀或封存狀態。**Sessions…**（Shift-Cmd-S）會開啟 Active/Archived 管理器，以進行閘道搜尋、群組管理、工作階段檢查、重新命名、釘選、封存及還原。選取模式可對多個作用中的工作階段套用釘選、取消釘選、封存或刪除，同時保持個別失敗項目可見。個別選單勾選項可顯示或隱藏助理推理與工具活動；兩者預設皆開啟，且會跨啟動記住設定。
- **對話記錄與撰寫器**：助理訊息會搭配頭像以純文字呈現，使用者訊息則以強調色泡泡呈現。待處理的代理程式問題會顯示為原生卡片，包含單選或多選選項、自由文字 **Other** 答案、到期倒數及共用終止狀態。空白聊天會提供桌面版入門提示。輸入 `/` 會開啟由 `commands.list` 支援的斜線命令自動完成，並可使用方向鍵／Tab／Return／Escape 操作鍵盤導覽。在訊息上按一下右鍵，即可複製其可見的 Markdown，而不包含隱藏的推理內容。遭截斷的助理訊息也會提供 **Open Full Message**，以載入可選取文字的 Markdown 閱讀器。使用 **Listen** 可透過閘道執行 TTS，並以本機語音作為備援。
- **語音控制項**：撰寫器可啟動或停止既有的 macOS Talk Mode，而不取代其選單列覆疊層。Talk Mode 啟用時，撰寫器會顯示其聆聽／思考／說話狀態、即時音訊活動及可展開的滾動對話記錄。在 Talk 按鈕上按一下右鍵，以選擇 **System Default** 或已連線的麥克風；這與 Voice Wake 和按住說話所使用的麥克風選項相同。如果所選麥克風中斷連線，作用中的 Talk 工作階段會改用系統預設值，並在下次啟動 Talk Mode 時再次嘗試該選項。當 Talk Mode 未占用音訊擷取時，另一個麥克風動作可錄製語音留言。

選單列的錨定精簡聊天面板會保留精簡的單欄版面配置，內嵌相同的模型、思考、詳細程度及快速控制項，並提供入門提示、Talk Mode、語音留言和 Listen。助理推理與工具活動在此精簡介面中仍會隱藏。

## 多個閘道視窗

開啟 **Settings → Gateways**，以新增或移除可重複使用的閘道設定檔。每個
設定檔包含一個 `ws://` 或 `wss://` 端點，以及其選用的權杖或
密碼；認證資訊會儲存在 macOS Keychain 中。移除設定檔
也會關閉其已開啟的視窗，並中止其次要連線。

選擇 **File → New Gateway Window…** 或按下 Cmd-N，然後選取其中一個
已儲存的設定檔。選取器會記住最近使用的設定檔。每次
選取都會建立新的獨立視窗，因此相同的閘道可顯示在
多個視窗中，並各自擁有不同的作用中工作階段及導覽狀態。

每個已儲存的設定檔都擁有一個共用的閘道連線、裝置驗證範圍、
對話記錄快取、離線寄件匣及路由租約。該設定檔的視窗
會重複使用這些資源，同時仍可彼此獨立導覽。不同
設定檔的視窗會保持連線，並同時執行聊天。

選單列 App 所設定的閘道仍是 Mac 節點
功能和 Talk Mode 的擁有者。其他閘道視窗僅供操作人員使用，因此
第二個閘道無法在未明示的情況下重新指定全域麥克風或裝置控制項。
Listen／TTS 和一般聊天動作會使用視窗本身的閘道連線。

## Quick Chat 列

按下 Option-Space（⌥Space），或從選單列選單選擇 **Quick Chat**，以開啟主要工作階段的浮動撰寫器。使用 **Settings → General → Quick Chat shortcut** 中的錄製器變更全域快速鍵。

Quick Chat 會顯示目標代理程式（頭像或表情符號，並以代理程式名稱作為預留位置），且會傳送至該代理程式的主要工作階段。Return 接受傳送後，此列會維持開啟並向下展開，顯示串流傳回的 Markdown 回覆及最近的對話記錄。此列的輸入欄位仍是撰寫器。按下 Command-Return 可傳送並在完整聊天視窗中開啟相同目標；按下 Shift-Return 可換行，或按下 Escape 關閉整個列與回覆區域。按一下外部也會關閉。若缺少相關的 macOS 權限，附加的提示列會提供 **Grant** 和 **Not now** 動作。

使用麥克風按鈕將聽寫內容輸入撰寫器。部分語音辨識結果會即時取代聽寫範圍，同時保留撰寫器中原有的文字。再次按下按鈕、Return 或 Escape 即可停止；傳送、隱藏 Quick Chat 或使其失去焦點也會釋放麥克風。首次使用時會要求 macOS 麥克風和語音辨識存取權。Quick Chat 使用 Apple Speech，且可能使用其網路服務；只有被動式 Voice Wake 要求使用裝置端辨識。

精簡模型控制項會顯示目標工作階段目前的模型與推理層級。模型選項會更新該工作階段，因此會在該處持續保留；推理選項則只會套用至從目前 Quick Chat 顯示介面傳送的每則訊息。此列隱藏時，本機選項會重設。切換代理程式或選擇最近的工作階段時，會保留明確選取的項目，但重新載入新目標工作階段的基礎模型狀態。

按一下歷程記錄按鈕，即可從最近更新的五個工作階段中選擇，或返回 **New message to &lt;agent&gt;**。選擇最近的工作階段後，訊息會傳送至該特定工作階段，且預留位置會變更為 **Reply in &lt;session&gt;**。隱藏 Quick Chat 會將此暫時目標重設為所選代理程式的主要工作階段；從頭像選單切換代理程式也會清除該目標。

Command-Return 會開啟接收該次傳送之代理程式的對話，包括工作階段範圍為全域時。

相機按鈕會開啟選單，其中包含 **Capture Window…** 或 **Capture Area…**。視窗擷取會為每個可見視窗加上標籤；區域擷取則會在你拖曳區域時調暗各個顯示器，並顯示即時尺寸。選取的螢幕截圖會連同任何已輸入文字作為說明文字，傳送給所選代理程式。首次使用時會要求 macOS 螢幕錄製存取權。按下 Escape、按一下空白處，或未實際拖曳出有意義的區域便按一下，都會取消操作。

使用文件文字按鈕附加焦點 App 之焦點視窗中的文字。Quick Chat 會將結果顯示為可移除的內容籌碼，而非將擷取的文字放入撰寫器；傳送時會將籌碼文字附加至送出的訊息，然後清除籌碼。這需要 macOS 輔助使用權限。Quick Chat 關閉時也會清除附加文字，因此一次顯示中的內容不會洩漏至之後的傳送。

回覆完成後，選擇 **Paste to &lt;app&gt;**，即可將其可見的助理文字（不含隱藏推理）複製至一般剪貼簿，並貼到先前位於最前方的 App 中。這需要 macOS 輔助使用權限。此動作會取代目前的剪貼簿內容，然後隱藏 Quick Chat。

使用 **Settings → General → Quick Chat** 可完全停用此功能；同一區段也包含快速鍵錄製器。

- **本機模式**：直接連線至本機閘道 WebSocket。
- **遠端模式**：透過 SSH 轉送閘道控制連接埠，並使用該通道作為資料平面。

## 啟動與偵錯

- 手動：Lobster 選單 -> "Open Chat"。
- 測試時自動開啟：

  ```bash
  dist/OpenClaw.app/Contents/MacOS/OpenClaw --chat
  ```

  （`--webchat` 可作為舊版別名使用。）

- 記錄：`./scripts/clawlog.sh`（子系統 `ai.openclaw`，類別 `WebChatSwiftUI`）。

## 運作方式

- 資料平面：閘道 WS 方法 `chat.history`、`chat.message.get`、`chat.send`、`chat.abort`、`chat.inject`，加上 `question.list` 和 `question.resolve`，以及事件 `chat`、`agent`、`presence`、`tick`、`health`；問題卡片會依循 `question.requested` 和 `question.resolved` 事件，並在重新連線後從 `question.list` 重新整理。
- `chat.history` 會傳回經顯示正規化的對話記錄：從可見文字中移除內嵌指令標籤、移除純文字工具呼叫 XML 承載資料（`<tool_call>`、`<function_call>`、`<tool_calls>`、`<function_calls>`，包括遭截斷的區塊）及洩漏的模型控制權杖、省略僅包含無聲權杖的助理資料列（例如完全相符的 `NO_REPLY`/`no_reply`），且過大的資料列可由截斷預留位置取代。
- 工作階段：如上所述，預設為主要工作階段；UI 可在工作階段之間切換。
- 工作階段群組：`sessions.groups.list`、`sessions.groups.put`、`sessions.groups.rename` 和 `sessions.groups.delete` 負責群組目錄。成員資格是透過 `sessions.patch` 更新的工作階段 `category`。
- 未讀狀態：工作階段啟用且其即時歷程記錄成功載入後，App 會清除該工作階段的未讀標記。歷程記錄載入失敗時不會清除；暫時性的修補失敗會在下次啟用時重試。
- 初始設定會使用專用工作階段，使首次執行設定保持獨立。
- 離線快取：App 會為每個閘道保留一份最近聊天工作階段和對話記錄的小型唯讀快取（`~/Library/Application Support/OpenClaw/chat-cache.sqlite`）：冷啟動時會立即呈現最後已知的對話記錄，並在閘道回應後重新整理；中斷連線時仍可瀏覽最近的聊天（傳送功能會維持停用，直到連線恢復）。

## 安全性介面

- 遠端模式只會透過 SSH 轉送閘道 WebSocket 控制連接埠。

## 已知限制

- 此 UI 針對聊天工作階段最佳化，而非完整的瀏覽器沙箱。
