macOS 選單列 App 將 WebChat UI 內嵌為原生 SwiftUI 檢視。它會連線至閘道，並預設使用所選代理程式的主要工作階段（`main`；當 `session.scope` 為 `global` 時則為 `global`）。

完整聊天視窗是原生分割檢視：

- **工作階段側邊欄**：可搜尋的工作階段清單，包含已釘選、由閘道支援的群組，以及最近使用區段。在每個區段中，衍生的子工作階段會巢狀顯示於其父工作階段下方；收合的父工作階段會彙整執行中、失敗及未讀的後代工作階段。內容選單支援工作階段資訊、重新命名、釘選、分支、標示已讀／未讀、封存／還原、複製工作階段金鑰，以及刪除。主要的新工作階段動作（或 Shift-Cmd-N）會立即透過 `sessions.create` 建立工作階段；旁邊的選項彈出視窗可選取代理程式，並要求建立受管理的工作樹，且可選擇指定基底參照。
- **視窗工具列**：內容用量環形圖（權杖與工作階段成本，並附精簡動作）、模型控制項，以及工作階段動作選單。模型依提供者分組，預設提供者排在最前面，而已釘選與最近使用的模型維持在頂端。控制項可繼承或覆寫模型的思考層級、選擇工具呼叫詳細程度，以及切換快速回應。選單可重新命名或分支目前的工作階段，並更新其釘選、已讀或封存狀態。**工作階段…**（Shift-Cmd-S）會開啟使用中／已封存管理器，以進行閘道搜尋、群組管理、工作階段檢查、重新命名、釘選、封存及還原。選取模式可將釘選、取消釘選、封存或刪除套用至多個使用中的工作階段，同時保留各別失敗項目的可見性。個別的選單勾選標記可顯示或隱藏助理推理與工具活動；兩者預設皆為開啟，且會在重新啟動後保留設定。
- **對話記錄與撰寫器**：助理訊息會以含頭像的純文字呈現，使用者訊息則以強調色對話框呈現。待處理的代理程式問題會呈現為原生卡片，包含單選或多選選項、可自由輸入文字的**其他**回答、到期倒數，以及共用終止狀態。空白聊天會提供桌面版起始提示。輸入 `/` 會開啟由 `commands.list` 支援的斜線命令自動完成，並支援方向鍵／Tab／Return／Escape 鍵盤導覽。在訊息上按一下右鍵，可複製其可見的 Markdown，而不包含隱藏的推理。遭截斷的助理訊息也會提供**開啟完整訊息**，以載入可選取文字的 Markdown 閱讀器。使用**聆聽**可透過閘道執行 TTS，並以本機語音作為備援。
- **語音控制項**：撰寫器可啟動或停止現有的 macOS 對話模式，而不取代其選單列覆疊介面。對話模式啟用時，撰寫器會顯示其聆聽／思考／說話狀態、即時音訊活動，以及可展開的捲動式逐字稿。在「對話」按鈕上按一下右鍵，可選擇**系統預設**或已連線的麥克風；這與語音喚醒和按住說話所使用的麥克風選項相同。如果選取的麥克風中斷連線，作用中的對話工作階段會改用系統預設，並在下次啟動對話模式時再次嘗試使用該選項。當對話模式未占用音訊擷取時，另一個麥克風動作可錄製語音留言。

選單列中的錨定精簡聊天面板會維持精簡的單欄版面配置，並在列內提供相同的模型、思考、詳細程度與快速控制項，另加上起始提示、對話模式、語音留言及聆聽功能。助理推理與工具活動在此精簡介面中仍會保持隱藏。

## 多個閘道視窗

開啟**設定 → 閘道**以新增或移除可重複使用的閘道設定檔。每個
設定檔都包含一個 `ws://` 或 `wss://` 端點，以及選用的權杖或
密碼；認證資訊會儲存在 macOS 鑰匙圈中。移除設定檔
也會關閉其已開啟的視窗，並關閉其次要連線。

選擇**檔案 → 新增閘道視窗…**或按下 Cmd-N，然後選取其中一個
已儲存的設定檔。選擇器會記住最近使用的設定檔。每次
選取都會建立新的獨立視窗，因此同一個閘道可顯示於
多個視窗中，並各自具有不同的作用中工作階段與導覽狀態。

每個已儲存的設定檔都擁有一個共用的閘道連線、裝置驗證範圍、
對話記錄快取、離線寄件匣，以及路由租約。該設定檔的視窗
會重複使用這些資源，同時維持各自獨立的導覽狀態。不同
設定檔的視窗會維持連線並同時執行聊天。

選單列 App 所設定的閘道仍是 Mac 節點
功能與對話模式的擁有者。其他閘道視窗僅供操作人員使用，因此
第二個閘道無法在未提示的情況下重新指定全域麥克風或裝置控制項。
聆聽／TTS 與一般聊天動作會使用視窗本身的閘道連線。

## 快速聊天列

按下 Option-Space（⌥Space）或從選單列選單選擇**快速聊天**，即可開啟供主要工作階段使用的浮動撰寫器。可使用**設定 → 一般 → 快速聊天快速鍵**中的錄製器變更全域快速鍵。

快速聊天會顯示目標代理程式（頭像或表情符號，並以代理程式名稱作為預留文字），並將訊息傳送至該代理程式的主要工作階段。按下 Return 接受傳送後，聊天列會保持開啟並向下展開，顯示串流傳入的 Markdown 回覆與最近的對話記錄。聊天列輸入框仍是撰寫器。按下 Command-Return 可傳送訊息，並在完整聊天視窗中開啟相同目標；按下 Shift-Return 可換行；按下 Escape 則關閉整個聊天列與回覆區域。按一下外部區域也會將其關閉。缺少相關 macOS 權限時，附加的提示列會提供**授予**與**暫時不要**動作。

使用麥克風按鈕可將語音聽寫至撰寫器。部分語音辨識結果會即時取代聽寫範圍，同時保留撰寫器中原有的文字。再次按下按鈕、Return 或 Escape 可停止；傳送、隱藏快速聊天或讓其失去焦點也會釋放麥克風。第一次使用時會要求 macOS 麥克風與語音辨識存取權。

精簡模型控制項會顯示目標工作階段目前的模型與推理層級。模型選項會更新該工作階段，因此會持續保留於其中；推理選項則僅套用至目前快速聊天介面所傳送的每則訊息。聊天列隱藏時，本機選項會重設。切換代理程式或選擇最近使用的工作階段時，會保留明確選取的選項，但重新載入新目標工作階段的基礎模型狀態。

按一下歷史記錄按鈕，可從最近更新的五個工作階段中選擇，或返回**傳送新訊息給 &lt;agent&gt;**。選取最近使用的工作階段後，會將訊息傳送至該確切工作階段，並將預留文字變更為**在 &lt;session&gt; 中回覆**。隱藏快速聊天會將此暫時目標重設為所選代理程式的主要工作階段；從頭像選單切換代理程式也會將其清除。

Command-Return 會開啟收到傳送內容之代理程式的對話，包括工作階段範圍為全域時。

相機按鈕會開啟包含**擷取視窗…**或**擷取區域…**的選單。擷取視窗會標示所有可見視窗；擷取區域會調暗每個顯示器，讓你拖曳選取區域，並顯示其即時尺寸。所選螢幕截圖會傳送至選定的代理程式，並將任何輸入的文字作為其說明文字。第一次使用時會要求 macOS 螢幕錄製存取權。按下 Escape、按一下空白區域，或只按一下而未拖曳出具有實際範圍的區域，都會取消操作。

使用文件文字按鈕，可附加目前焦點 App 之焦點視窗中的文字。快速聊天會將結果顯示為可移除的內容晶片，而不會將擷取的文字放入撰寫器；傳送時會將晶片文字附加至傳出訊息，然後將其清除。這需要 macOS 輔助使用權限。每當快速聊天關閉時，附加的文字也會清除，因此某次顯示中的內容不會洩漏至之後傳送的訊息。

回覆完成後，選擇**貼到 &lt;app&gt;**，可將其可見的助理文字（不含隱藏的推理）複製到一般剪貼簿，並貼入先前位於最前方的 App。這需要 macOS 輔助使用權限。此動作會取代目前的剪貼簿內容，然後隱藏快速聊天。

可使用**設定 → 一般 → 快速聊天**完全停用此功能；快速鍵錄製器也位於同一區段。

- **本機模式**：直接連線至本機閘道 WebSocket。
- **遠端模式**：透過 SSH 轉送閘道控制連接埠，並使用該通道作為資料平面。

## 啟動與偵錯

- 手動：Lobster 選單 -> "Open Chat"。
- 測試時自動開啟：

  ```bash
  dist/OpenClaw.app/Contents/MacOS/OpenClaw --chat
  ```

  （`--webchat` 可作為舊版別名使用。）

- 記錄：`./scripts/clawlog.sh`（子系統 `ai.openclaw`，類別 `WebChatSwiftUI`）。

## 連接方式

- 資料平面：閘道 WS 方法 `chat.history`、`chat.message.get`、`chat.send`、`chat.abort`、`chat.inject`，以及 `question.list` 和 `question.resolve`，另有事件 `chat`、`agent`、`presence`、`tick`、`health`；問題卡片會依循 `question.requested` 與 `question.resolved` 事件，並在重新連線後從 `question.list` 重新整理。
- `chat.history` 會傳回經顯示正規化的對話記錄：從可見文字中移除列內指令標籤、純文字工具呼叫 XML 承載內容（`<tool_call>`、`<function_call>`、`<tool_calls>`、`<function_calls>`，包括遭截斷的區塊）及外洩的模型控制權杖；省略僅包含靜默權杖的助理資料列，例如完全相符的 `NO_REPLY`/`no_reply`；過大的資料列則可替換為截斷預留文字。
- 工作階段：預設使用上述主要工作階段；UI 可在工作階段之間切換。
- 工作階段群組：`sessions.groups.list`、`sessions.groups.put`、`sessions.groups.rename` 與 `sessions.groups.delete` 擁有群組目錄。成員資格是透過 `sessions.patch` 更新的工作階段 `category`。
- 未讀狀態：工作階段啟用且其即時歷史記錄成功載入後，App 會清除該工作階段的未讀標記。歷史記錄載入失敗時不會清除；暫時性的修補失敗會在下次啟用時重試。
- 初始設定會使用專用工作階段，將首次執行設定與其他工作階段分開。
- 離線快取：App 會為每個閘道保留最近聊天工作階段與對話記錄的小型唯讀快取（`~/Library/Application Support/OpenClaw/chat-cache.sqlite`）：冷啟動時會立即顯示最後已知的對話記錄，並在閘道回應後重新整理；中斷連線時仍可瀏覽最近的聊天（在連線恢復前，傳送功能會保持停用）。

## 安全面

- 遠端模式只會透過 SSH 轉送閘道 WebSocket 控制連接埠。

## 已知限制

- 此 UI 針對聊天工作階段最佳化，而非完整的瀏覽器沙箱。
