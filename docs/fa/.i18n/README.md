# دارایی‌های i18n مستندات OpenClaw

این پوشه پیکربندی ترجمه را برای مخزن مستندات مبدأ نگه می‌دارد.

صفحه‌های محلی‌سازی تولیدشده و حافظه ترجمه زنده محلی‌سازی اکنون در مخزن انتشار (`openclaw/docs`، checkout هم‌سطح محلی `~/Projects/openclaw-docs`) قرار دارند.

## فایل‌ها

- `glossary.<lang>.json` — نگاشت‌های اصطلاحات ترجیحی (در راهنمایی پرامپت استفاده می‌شود).
- `<lang>.tm.jsonl` — حافظه ترجمه (cache) که با گردش‌کار + مدل + هش متن کلیدگذاری شده است. در این مخزن، فایل‌های TM محلی‌سازی بر اساس نیاز تولید می‌شوند.

## قالب واژه‌نامه

`glossary.<lang>.json` آرایه‌ای از ورودی‌ها است:

```json
{
  "source": "troubleshooting",
  "target": "故障排除",
  "ignore_case": true,
  "whole_word": false
}
```

فیلدها:

- `source`: عبارت انگلیسی (یا مبدأ) که باید ترجیح داده شود.
- `target`: خروجی ترجمه ترجیحی.

## نکته‌ها

- ورودی‌های واژه‌نامه به‌عنوان **راهنمایی پرامپت** به مدل داده می‌شوند (بدون بازنویسی‌های قطعی).
- `scripts/docs-i18n` همچنان مالک تولید ترجمه است.
- مخزن مبدأ، مستندات انگلیسی را به مخزن انتشار همگام‌سازی می‌کند؛ تولید محلی‌سازی برای هر زبان در همان‌جا هنگام push، زمان‌بندی، و ارسال انتشار اجرا می‌شود.
