ClawRouter برای چندین ارائه‌دهنده بالادستی مدل، یک کلید با دامنه محدود به سیاست در اختیار OpenClaw قرار می‌دهد. Plugin همراه `clawrouter` فقط مدل‌های مجاز برای آن کلید را کشف می‌کند، هر مدل را از طریق پروتکل اعلام‌شده آن مسیریابی می‌کند و بودجه کلید و مصرف تجمیعی آن را در سطوح مصرف OpenClaw گزارش می‌دهد.

اعتبارنامه‌های بالادستی و فورواردینگ ویژه هر ارائه‌دهنده در ClawRouter باقی می‌مانند؛ بنابراین هرگز لازم نیست Plugin هر ارائه‌دهنده بالادستی را روی میزبان OpenClaw نصب یا احراز هویت کنید. این Plugin همراه OpenClaw (`enabledByDefault: true`) عرضه می‌شود؛ فقط به یک اعتبارنامه صادرشده ClawRouter نیاز دارید.

| ویژگی      | مقدار                                    |
| ------------- | ---------------------------------------- |
| ارائه‌دهنده      | `clawrouter`                             |
| Plugin        | همراه (در OpenClaw گنجانده شده است)           |
| احراز هویت          | `CLAWROUTER_API_KEY`                     |
| URL پیش‌فرض   | `https://clawrouter.openclaw.ai`         |
| کاتالوگ مدل | محدود به دامنه اعتبارنامه از طریق `/v1/catalog`      |
| سهمیه‌ها        | بودجه و مصرف ماهانه از طریق `/v1/usage` |

## شروع به کار

<Steps>
  <Step title="دریافت اعتبارنامه با دامنه محدود">
    از مدیر ClawRouter خود اعتبارنامه‌ای بخواهید که سیاست آن شامل
    ارائه‌دهندگان، مدل‌ها و بودجه ماهانه‌ای باشد که باید استفاده کنید. اعتبارنامه‌ها
    هنگام صدور فقط یک‌بار نمایش داده می‌شوند.
  </Step>
  <Step title="پیکربندی OpenClaw">
    ```bash
    export CLAWROUTER_API_KEY="..."
    openclaw onboard --auth-choice clawrouter-api-key
    openclaw plugins enable clawrouter
    ```

    `clawrouter` همراه است و به‌طور پیش‌فرض فعال می‌شود. اگر پیکربندی شما
    `plugins.allow` را تنظیم می‌کند، پیش از فعال‌سازی، `clawrouter` را به آن فهرست اضافه کنید. برای یک
    استقرار سفارشی، `models.providers.clawrouter.baseUrl` را روی مبدأ
    ClawRouter تنظیم کنید؛ مقدار پیش‌فرض `https://clawrouter.openclaw.ai` است.

  </Step>
  <Step title="فهرست‌کردن مدل‌های اعطاشده">
    ```bash
    openclaw models list --all --provider clawrouter
    ```

    ارجاع‌های مدل بازگردانده‌شده را دقیقاً همان‌طور که نمایش داده شده‌اند استفاده کنید. آن‌ها فضای نام
    بالادستی را حفظ می‌کنند؛ مانند `clawrouter/openai/gpt-5.5`،
    `clawrouter/anthropic/claude-sonnet-4-6` یا
    `clawrouter/google/gemini-3.5-flash`. اگر `agents.defaults.models` در
    پیکربندی شما یک فهرست مجاز است، هر ارجاع انتخاب‌شده ClawRouter را به آن اضافه کنید.

  </Step>
  <Step title="انتخاب مدل">
    ```bash
    openclaw models set clawrouter/<provider>/<model>
    ```

    همچنین می‌توانید یک مدل بازگردانده‌شده را برای یک اجرا با
    `openclaw agent --model clawrouter/<provider>/<model> --message "..."` انتخاب کنید.

  </Step>
</Steps>

## استقرار مدیریت‌شده غیرتعاملی

کلید پراکسی را در تزریق راز بار کاری نگه دارید و در `openclaw.json` فقط یک
SecretRef ذخیره کنید. فیلدهای استاندارد مدیریت‌شده عبارت‌اند از:

| کاربرد       | فیلد پیکربندی یا محیطی                                              |
| ------------- | ------------------------------------------------------------------------ |
| مبدأ مسیریاب | `models.providers.clawrouter.baseUrl`                                    |
| اعتبارنامه    | `models.providers.clawrouter.apiKey` -> SecretRef محیطی                    |
| مقدار راز  | `CLAWROUTER_API_KEY` در محیط فرایند Gateway                  |
| مدل پیش‌فرض | `agents.defaults.model.primary` -> `clawrouter/<provider>/<model>`       |
| برچسب بار کاری  | `models.providers.clawrouter.headers.X-ClawRouter-Project-Id` (اختیاری) |

برای نمونه، یک کنترل‌کننده استقرار می‌تواند مالک این وصله JSON5 باشد:

```json5
{
  plugins: {
    entries: { clawrouter: { enabled: true } },
  },
  models: {
    providers: {
      clawrouter: {
        baseUrl: "https://clawrouter.internal.example",
        apiKey: {
          source: "env",
          provider: "default",
          id: "CLAWROUTER_API_KEY",
        },
        headers: {
          "X-ClawRouter-Project-Id": "fakeco",
        },
      },
    },
  },
  agents: {
    defaults: {
      model: { primary: "clawrouter/openai/gpt-5.5" },
    },
  },
}
```

اگر استقرار `plugins.allow` را تنظیم می‌کند، ورودی‌های موجود آن را حفظ و
`clawrouter` را اضافه کنید. بدون راهنمای تعاملی اعتبارسنجی و اعمال کنید:

```bash
openclaw config patch --file ./clawrouter.patch.json5 --dry-run --json
openclaw config patch --file ./clawrouter.patch.json5
```

اجرای آزمایشی SecretRef را حل می‌کند، اما هرگز مقدار آن را چاپ نمی‌کند. برای چرخش
اعتبارنامه، Secret خارجی تأمین‌کننده `CLAWROUTER_API_KEY` را به‌روزرسانی کنید و
بار کاری Gateway را راه‌اندازی مجدد کنید تا محیط فرایند جدید بارگذاری شود. فایل
پیکربندی و ارجاع مدل تغییر نمی‌کنند.

برای یک Gateway مستقل Docker که از منبع ساخته شده است، ClawRouter از پیش در
زمان اجرای ریشه گنجانده شده است. فقط Plugin کانالی را انتخاب کنید که به بسته‌بندی جداگانه نیاز دارد؛
مانند `OPENCLAW_EXTENSIONS=clickclack`، `slack` یا `msteams`؛
[تصاویر ساخته‌شده از منبع با Pluginهای انتخاب‌شده](/fa/install/docker#source-built-images-with-selected-plugins) را ببینید.
استقرارهای آرشیوی/دستگاهی باید همان منبع نهایی‌شده را از طریق
خط لوله مصنوع خود بسته‌بندی کنند، نه اینکه تصویر OCI را مصرف کنند.

## آمادگی و اثبات زنده

این بررسی‌ها مرزهای متفاوتی را اثبات می‌کنند؛ یکی را جایگزین دیگری نکنید:

```bash
# فقط سلامت فرایند ClawRouter؛ هیچ اعتبارنامه یا مدل بالادستی آزمایش نمی‌شود.
curl -fsS https://clawrouter.internal.example/v1/health

# فقط آمادگی راه‌اندازی Gateway در OpenClaw؛ هیچ فراخوانی مدلی انجام نمی‌شود.
curl -fsS http://127.0.0.1:18789/readyz

# کشف کاتالوگ محدود به دامنه اعتبارنامه.
openclaw models list --all --provider clawrouter --json

# کاوش حداقلی استنتاج واقعی از طریق ارائه‌دهنده پیکربندی‌شده ClawRouter.
openclaw models status --probe --probe-provider clawrouter --probe-max-tokens 8 --json

# کانری بار کاری با استفاده از ارجاع دقیق مدل اعطاشده.
openclaw agent --agent main \
  --model clawrouter/openai/gpt-5.5 \
  --message "دقیقاً پاسخ دهید: CLAWROUTER_CANARY_OK" \
  --json
```

به‌جای کپی کورکورانه مدل نمونه، از مدلی استفاده کنید که کاتالوگ محدودشده بازگردانده است.
پاسخ موفق `/readyz` یعنی Gateway می‌تواند به
درخواست‌ها خدمت‌رسانی کند؛ این پاسخ ادعا نمی‌کند که ClawRouter، اعتبارنامه آن یا یک
ارائه‌دهنده بالادستی آماده است. کاوش مدل و کانری عامل، اثبات‌های استنتاج هستند.

برای عیب‌یابی زنده، کانری را صادر و لاگ‌های استاندارد Gateway را بررسی کنید.
تشخیص‌های موجود و صرفاً مبتنی بر فراداده حمل‌ونقل مدل، خط‌هایی با شکل زیر منتشر می‌کنند:

```text
[model-fetch] شروع provider=clawrouter api=openai-responses model=openai/gpt-5.5 method=POST url=https://clawrouter.internal.example/v1/responses
[model-fetch] پاسخ provider=clawrouter api=openai-responses model=openai/gpt-5.5 status=200
```

هنگامی که آن شناسه‌ها در دسترس باشند، Plugin سرآیندهای کران‌دار
`X-ClawRouter-Client`، `X-ClawRouter-Agent-Id` و
`X-ClawRouter-Session-Id` را ارسال می‌کند. همچنین
`callId` تشخیصی فراخوانی مدل (`<run-id>:model:<n>`) را به
`X-Request-ID` نگاشت می‌کند تا یک رویداد فراخوانی مدل OpenClaw به ردپای ممیزی
صرفاً مبتنی بر فراداده ClawRouter پیوند داده شود. مقادیری که در بودجه 128 نویسه‌ای شناسه درخواست
جای می‌گیرند، یکسان‌اند. مقادیر طولانی‌تر پسوند `:model:<n>` و یک هش
قطعی را حفظ می‌کنند تا فراخوانی‌های متمایز همچنان کران‌دار و قابل‌پیوند باقی بمانند. فراداده ایستای استقرار
مانند `X-ClawRouter-Project-Id` را می‌توان در نگاشت `headers` ارائه‌دهنده تنظیم کرد.
سرآیندهای انتساب عامل و نشست محدودیت جداگانه 256 نویسه‌ای خود را
حفظ می‌کنند. شناسه‌های خودکار درخواست که شامل نویسه‌های خارج از مجموعه شناسه ASCII
ClawRouter هستند، از همان قالب قطعی و کران‌دار استفاده می‌کنند.
سرآیندهای صریحاً پیکربندی‌شده، از جمله هر گونه تفاوت حروف بزرگ و کوچک در `X-Request-ID`، بر
مقادیر خودکار اولویت دارند. تشخیص حمل‌ونقل، فراداده مسیریابی و پاسخ را
ثبت می‌کند؛ اعتبارنامه‌ها، شناسه‌های درخواست، پرامپت‌ها یا تکمیل‌ها را ثبت نمی‌کند.
رویداد ممیزی خود ClawRouter ارائه‌دهنده بالادستی انتخاب‌شده و
وضعیت نگهداشت محتوا را فراهم می‌کند.

## کشف مدل

`GET /v1/catalog` مقدار `{ providers: [...] }` را بازمی‌گرداند که در آن هر ورودی ارائه‌دهنده،
`models[]` خود (با شناسه بالادستی، قابلیت‌ها و قیمت‌گذاری) و
مسیرهای درخواست پشتیبانی‌شده‌اش را فهرست می‌کند. OpenClaw فهرست ثابت دومی از
مدل‌های ClawRouter ارائه نمی‌کند. یک مدل کاتالوگ هنگامی به‌عنوان مدل OpenClaw معرفی می‌شود که:

- سیاست اعتبارنامه، ارائه‌دهنده آن را مجاز کند؛
- مدل کاتالوگ یک قابلیت LLM پشتیبانی‌شده (`llm.responses`،
  `llm.chat`، `llm.messages` یا `llm.stream` با یک مسیر استریم منطبق)
  را اعلام کند؛ و
- ارائه‌دهنده برای یکی از حمل‌ونقل‌های زیر مسیری منطبق ارائه کند.

افزودن مدل به یک ارائه‌دهنده پشتیبانی‌شده ClawRouter به انتشار OpenClaw نیاز ندارد:
تازه‌سازی بعدی کاتالوگ (با کش 60 ثانیه‌ای برای هر دامنه اعتبارنامه)
آن را کشف می‌کند. مدلی که به پروتکل سیمی جدیدی نیاز دارد، ابتدا به پشتیبانی Plugin نیاز دارد.

## پروتکل و Pluginهای ارائه‌دهنده

ClawRouter مالک اعتبارنامه‌های بالادستی است؛ کاتالوگ آن به OpenClaw می‌گوید از کدام
حمل‌ونقل استفاده کند، بنابراین هرگز لازم نیست Plugin احراز هویت همه شرکت‌های بالادستی را نصب کنید.

| قابلیت / مسیر کاتالوگ                               | حمل‌ونقل OpenClaw     |
| -------------------------------------------------------- | ---------------------- |
| `llm.responses` (ارائه‌دهنده سازگار با OpenAI)             | `openai-responses`     |
| `llm.chat` (ارائه‌دهنده سازگار با OpenAI)                  | `openai-completions`   |
| مسیر `llm.messages` + `anthropic.messages`              | `anthropic-messages`   |
| مسیر `llm.stream` + استریم `google.generate_content` | `google-generative-ai` |

این Plugin همچنین سیاست‌های منطبق بازپخش و طرح‌واره ابزار را برای آن
خانواده‌ها اعمال می‌کند (سازگاری طرح‌واره ابزار OpenAI/DeepSeek/Gemini/Perplexity؛ سیاست‌های
بازپخش بومی Anthropic و Google Gemini). مدل‌های Perplexity یک بازنویسی سخت‌گیرانه
طرح‌واره دریافت می‌کنند: `patternProperties` و `additionalProperties` حذف می‌شوند و
هر طرح‌واره شیء `properties` را اعلام می‌کند، زیرا Perplexity طرح‌واره‌های ابزار
فاقد آن‌ها را رد می‌کند. ارائه‌دهنده‌ای در کاتالوگ که فقط یک
قالب درخواست پشتیبانی‌نشده ارائه کند، عمداً به‌عنوان مدل متنی OpenClaw معرفی نمی‌شود.
به‌جای ارسال محموله ناسازگار، آن ارائه‌دهندگان را در
ClawRouter به یکی از قراردادهای پشتیبانی‌شده نرمال‌سازی کنید.

## سهمیه‌ها و مصرف

پاسخ `/v1/usage` از ClawRouter سطوح عادی مصرف ارائه‌دهنده OpenClaw را
تغذیه می‌کند: مجموع درخواست، توکن و هزینه، به‌علاوه پنجره بودجه ماهانه هنگامی که
کلید دارای محدودیت است. کلیدهای بدون اندازه‌گیری همچنان مصرف تجمیعی را بدون
پنجره درصدی نمایش می‌دهند.

جست‌وجوی سهمیه از همان کلید محدود به دامنه کشف مدل استفاده می‌کند. شکست در جست‌وجوی
سهمیه مانع اجرای مدل نمی‌شود.

نمای زنده را با این دستورات بررسی کنید:

```bash
openclaw status --usage
openclaw models status
```

همان نمای ارائه‌دهنده برای `/status` در گفت‌وگو و رابط کاربری
مصرف OpenClaw در دسترس است. بودجه در سراسر سیاست مشترک است؛ بنابراین درخواست‌های
کلاینت دیگری که از همان سیاست ClawRouter استفاده می‌کند می‌تواند درصد باقی‌مانده را تغییر دهد.

## عیب‌یابی

| نشانه                                  | بررسی                                                                                                                                          |
| ---------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| هیچ مدل ClawRouter وجود ندارد                     | تأیید کنید Plugin فعال است و `plugins.allow` آن را مجاز می‌داند؛ سپس بررسی کنید که اعتبارنامه فعال است و دست‌کم یک ارائه‌دهنده آماده را مجاز می‌کند. |
| یک مدل پیکربندی‌شده ClawRouter وجود ندارد | قابلیت `/v1/catalog` و پشتیبانی مسیر آن را بررسی کنید. قراردادهای حمل‌ونقل پشتیبانی‌نشده عمداً فیلتر می‌شوند.                            |
| `Unknown model: clawrouter/...`          | هنگامی که آن نگاشت پیکربندی به‌عنوان فهرست مجاز استفاده می‌شود، ارجاع دقیق کاتالوگ را به `agents.defaults.models` اضافه کنید.                               |
| `401` یا `403` از کاتالوگ یا مصرف     | اعتبارنامه ClawRouter را دوباره صادر کنید یا دامنه آن را تغییر دهید؛ OpenClaw به کلیدهای ارائه‌دهنده بالادستی بازنمی‌گردد.                                          |
| فراخوانی مدل پس از کشف شکست می‌خورد         | اتصال ارائه‌دهنده و سلامت بالادستی را در ClawRouter بررسی کنید، سپس پس از بازیابی وضعیت آمادگی آن دوباره تلاش کنید.                                |
| مصرف دارای مجموع است اما درصد ندارد       | سیاست بدون اندازه‌گیری است؛ برای نمایش پنجره درصدی، در ClawRouter بودجه ماهانه‌ای اضافه کنید.                                                     |

## رفتار امنیتی

- کشف کاتالوگ به کلید پراکسی پیکربندی‌شده محدود است و برای هر دامنه اعتبارنامه (دایرکتوری عامل، دایرکتوری فضای کاری، شناسه پروفایل احراز هویت و URL پایه) در حافظه نهان ذخیره می‌شود.
- کلید پراکسی فقط هنگام ارسال درخواست پیوست می‌شود؛ این کلید در فراداده مدل ذخیره نمی‌شود.
- مقادیر انتساب خودکار و هم‌بستگی درخواست پیش از ارسال برش داده می‌شوند و در صورت وجود نویسه‌های کنترلی رد می‌شوند. مقادیر انتساب به 256 نویسه و شناسه‌های درخواست به 128 نویسه محدود هستند.
- اطلاعات تشخیصی انتقال مدل فقط شامل فراداده است و هرگز کلید پراکسی یا محتوای مدل را در بر نمی‌گیرد.
- شناسه‌های بومی مدل Anthropic و Gemini فقط هنگام ارسال به شناسه‌های بالادستی آن‌ها بازنویسی می‌شوند.
- ردیف‌های پشتیبانی‌نشده یا فاقد مجوز کاتالوگ به‌صورت بسته شکست می‌خورند و قابل انتخاب نیستند.
