GMI Cloud یک پلتفرم استنتاج میزبانی‌شده برای مدل‌های پیشرو و دارای وزن‌های باز است
که پشت یک API سازگار با OpenAI قرار دارد. در OpenClaw، این پلتفرم یک Plugin رسمی
ارائه‌دهنده خارجی است: آن را یک‌بار نصب کنید، اطلاعات احراز هویت را از طریق
احراز هویت معمول مدل ذخیره کنید و از ارجاع‌های مدل مانند
`gmi/google/gemini-3.1-flash-lite` استفاده کنید.

زمانی از GMI استفاده کنید که برای چندین خانواده مدل میزبانی‌شده، از جمله مسیرهای
Anthropic، DeepSeek، Google، Moonshot، OpenAI و Z.AI که کاتالوگ GMI ارائه می‌دهد،
یک کلید API واحد می‌خواهید. می‌توان از آن به‌عنوان ارائه‌دهنده ثانویه برای جایگزینی
مدل هنگام خطا، مقایسه مسیرهای میزبانی‌شده میان عرضه‌کنندگان، یا زمانی استفاده کرد
که مدلی در GMI زودتر از ارائه‌دهنده اصلی شما در دسترس قرار می‌گیرد. OpenClaw
مالک شناسه ارائه‌دهنده، نمایه احراز هویت، نام‌های مستعار، داده‌های اولیه کاتالوگ مدل
و نشانی پایه است؛ GMI مالک وضعیت زنده دسترس‌پذیری مدل‌ها، صورت‌حساب، محدودیت‌های نرخ
و هرگونه سیاست مسیریابی سمت ارائه‌دهنده است.

| ویژگی              | مقدار                                    |
| ------------------ | ---------------------------------------- |
| شناسه ارائه‌دهنده  | `gmi` (نام‌های مستعار: `gmi-cloud`، `gmicloud`) |
| بسته                | `@openclaw/gmi-provider`                 |
| متغیر محیطی احراز هویت | `GMI_API_KEY`                        |
| API                 | سازگار با OpenAI (`openai-completions`) |
| نشانی پایه          | `https://api.gmi-serving.com/v1`         |
| مدل پیش‌فرض         | `gmi/google/gemini-3.1-flash-lite`       |

## راه‌اندازی

Plugin را نصب و Gateway را راه‌اندازی مجدد کنید، سپس در GMI Cloud
(`https://www.gmicloud.ai/`) یک کلید API بسازید:

```bash
openclaw plugins install @openclaw/gmi-provider
openclaw gateway restart
```

سپس اجرا کنید:

```bash
openclaw onboard --auth-choice gmi-api-key
```

در راه‌اندازی‌های غیرتعاملی می‌توانید `--gmi-api-key <key>` را ارائه دهید، یا این
متغیر را تنظیم کنید:

```bash
export GMI_API_KEY="<your-gmi-api-key>" # pragma: allowlist secret
```

## چه زمانی GMI را انتخاب کنید

- به‌جای سرور محلی مدل، یک نقطه پایانی میزبانی‌شده سازگار با OpenAI می‌خواهید.
- می‌خواهید چندین خانواده مدل تجاری و دارای وزن‌های باز را از طریق یک حساب
  ارائه‌دهنده امتحان کنید.
- ارائه‌دهنده‌ای جایگزین با مسیریابی بالادستی متفاوت از DeepInfra، OpenRouter،
  Together یا APIهای مستقیم عرضه‌کنندگان می‌خواهید.
- به شناسه‌های مدل، قیمت‌گذاری یا کنترل‌های حساب مختص GMI نیاز دارید.

زمانی ارائه‌دهنده مستقیم عرضه‌کننده را انتخاب کنید که به قابلیت‌های بومی آن
عرضه‌کننده نیاز دارید و GMI آن‌ها را از طریق مسیر سازگار با OpenAI خود ارائه
نمی‌کند. زمانی یک ارائه‌دهنده محلی مانند LM Studio، Ollama، SGLang یا vLLM را
انتخاب کنید که محلی‌بودن داده‌ها یا کنترل GPU محلی از سهولت میزبانی‌شده مهم‌تر است.

## مدل‌ها

کاتالوگ Plugin شامل داده‌های اولیه شناسه‌های مسیر رایج GMI Cloud است:

| ارجاع مدل                           | ورودی        | پنجره زمینه | حداکثر خروجی |
| ----------------------------------- | ------------ | ----------- | ------------- |
| `gmi/anthropic/claude-sonnet-4.6`  | متن + تصویر  | 200,000     | 64,000        |
| `gmi/deepseek-ai/DeepSeek-V3.2`    | متن          | 163,840     | 65,536        |
| `gmi/google/gemini-3.1-flash-lite` | متن + تصویر  | 1,048,576   | 65,536        |
| `gmi/moonshotai/Kimi-K2.5`         | متن + تصویر  | 262,144     | 65,536        |
| `gmi/openai/gpt-5.4`               | متن + تصویر  | 400,000     | 128,000       |
| `gmi/zai-org/GLM-5.1-FP8`          | متن          | 202,752     | 65,536        |

این کاتالوگ فقط داده اولیه است و تضمین نمی‌کند که هر حسابی بتواند در هر زمانی
هر مدلی را فراخوانی کند. مواردی را که ارائه‌دهنده پیکربندی‌شده در محیط شما گزارش
می‌کند فهرست کنید:

```bash
openclaw models list --provider gmi
```

## عیب‌یابی

- `401` یا `403`: بررسی کنید که `GMI_API_KEY` برای فرایندی که OpenClaw را اجرا
  می‌کند تنظیم شده باشد، یا راه‌اندازی اولیه را دوباره اجرا کنید تا کلید در نمایه
  احراز هویت ارائه‌دهنده ذخیره شود.
- خطاهای مدل ناشناخته: تأیید کنید که مدل در حساب GMI شما وجود دارد و از ارجاع کامل
  `gmi/<route-id>` نمایش‌داده‌شده توسط `openclaw models list --provider gmi`
  استفاده کنید.
- خطاهای متناوب ارائه‌دهنده: مسیر دیگری از GMI را امتحان کنید یا GMI را به‌عنوان
  ارائه‌دهنده جایگزین پیکربندی کنید، نه تنها ارائه‌دهنده اصلی مدل.
