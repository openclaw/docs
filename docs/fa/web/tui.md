## شروع سریع

### حالت Gateway

1. Gateway را شروع کنید.

```bash
openclaw gateway
```

2. TUI را باز کنید.

```bash
openclaw tui
```

3. پیامی تایپ کنید و Enter را فشار دهید.

Gateway راه‌دور:

```bash
openclaw tui --url ws://<host>:<port> --token <gateway-token>
```

اگر Gateway شما از احراز هویت با گذرواژه استفاده می‌کند، از `--password` استفاده کنید.

### حالت محلی

TUI را بدون Gateway اجرا کنید:

```bash
openclaw chat
# or
openclaw tui --local
```

نکات:

- `openclaw chat` و `openclaw terminal` نام‌های مستعار `openclaw tui --local` هستند.
- `--local` را نمی‌توان با `--url`، `--token` یا `--password` ترکیب کرد.
- حالت محلی مستقیماً از runtime عامل تعبیه‌شده استفاده می‌کند. بیشتر ابزارهای محلی کار می‌کنند، اما قابلیت‌های مخصوص Gateway در دسترس نیستند.
- `openclaw` و `openclaw crestodian` نیز از این پوسته TUI استفاده می‌کنند، با Crestodian به‌عنوان بک‌اند گفت‌وگوی راه‌اندازی و تعمیر محلی.

## آنچه می‌بینید

- سرآیند: URL اتصال، عامل فعلی، نشست فعلی.
- گزارش گفت‌وگو: پیام‌های کاربر، پاسخ‌های دستیار، اعلان‌های سیستم، کارت‌های ابزار.
- خط وضعیت: وضعیت اتصال/اجرا (در حال اتصال، در حال اجرا، در حال پخش، بیکار، خطا).
- پانوشت: وضعیت اتصال + عامل + نشست + مدل + think/fast/verbose/trace/reasoning + شمارش توکن‌ها + deliver.
- ورودی: ویرایشگر متن با تکمیل خودکار.

## مدل ذهنی: عامل‌ها + نشست‌ها

- عامل‌ها slugهای یکتا هستند (مثلاً `main`، `research`). Gateway فهرست را ارائه می‌کند.
- نشست‌ها متعلق به عامل فعلی هستند.
- کلیدهای نشست به‌صورت `agent:<agentId>:<sessionKey>` ذخیره می‌شوند.
  - اگر `/session main` را تایپ کنید، TUI آن را به `agent:<currentAgent>:main` گسترش می‌دهد.
  - اگر `/session agent:other:main` را تایپ کنید، صریحاً به آن نشست عامل جابه‌جا می‌شوید.
- دامنه نشست:
  - `per-sender` (پیش‌فرض): هر عامل نشست‌های زیادی دارد.
  - `global`: TUI همیشه از نشست `global` استفاده می‌کند (ممکن است انتخاب‌گر خالی باشد).
- عامل + نشست فعلی همیشه در پانوشت قابل مشاهده‌اند.

## ارسال + تحویل

- پیام‌ها به Gateway ارسال می‌شوند؛ تحویل به ارائه‌دهندگان به‌صورت پیش‌فرض خاموش است.
- تحویل را روشن کنید:
  - `/deliver on`
  - یا پنل تنظیمات
  - یا با `openclaw tui --deliver` شروع کنید

## انتخاب‌گرها + لایه‌های رویی

- انتخاب‌گر مدل: مدل‌های موجود را فهرست کنید و override نشست را تنظیم کنید.
- انتخاب‌گر عامل: عامل دیگری انتخاب کنید.
- انتخاب‌گر نشست: فقط نشست‌های عامل فعلی را نشان می‌دهد.
- تنظیمات: تغییر وضعیت deliver، گسترش خروجی ابزار، و دیده‌شدن تفکر.

## میان‌برهای صفحه‌کلید

- Enter: ارسال پیام
- Esc: لغو اجرای فعال
- Ctrl+C: پاک‌کردن ورودی (برای خروج دوبار فشار دهید)
- Ctrl+D: خروج
- Ctrl+L: انتخاب‌گر مدل
- Ctrl+G: انتخاب‌گر عامل
- Ctrl+P: انتخاب‌گر نشست
- Ctrl+O: تغییر وضعیت گسترش خروجی ابزار
- Ctrl+T: تغییر وضعیت دیده‌شدن تفکر (تاریخچه را دوباره بارگذاری می‌کند)

## فرمان‌های Slash

هسته:

- `/help`
- `/status`
- `/agent <id>` (یا `/agents`)
- `/session <key>` (یا `/sessions`)
- `/model <provider/model>` (یا `/models`)

کنترل‌های نشست:

- `/think <off|minimal|low|medium|high>`
- `/fast <status|on|off>`
- `/verbose <on|full|off>`
- `/trace <on|off>`
- `/reasoning <on|off|stream>`
- `/usage <off|tokens|full>`
- `/elevated <on|off|ask|full>` (نام مستعار: `/elev`)
- `/activation <mention|always>`
- `/deliver <on|off>`

چرخه عمر نشست:

- `/new` یا `/reset` (بازنشانی نشست)
- `/abort` (لغو اجرای فعال)
- `/settings`
- `/exit`

فقط حالت محلی:

- `/auth [provider]` جریان احراز هویت/ورود ارائه‌دهنده را داخل TUI باز می‌کند.

سایر فرمان‌های Slash مربوط به Gateway (برای مثال، `/context`) به Gateway ارسال می‌شوند و به‌عنوان خروجی سیستم نشان داده می‌شوند. [فرمان‌های Slash](/fa/tools/slash-commands) را ببینید.

## فرمان‌های پوسته محلی

- برای اجرای یک فرمان پوسته محلی روی میزبان TUI، ابتدای یک خط `!` بگذارید.
- TUI در هر نشست یک‌بار برای اجازه اجرای محلی درخواست می‌دهد؛ رد کردن، `!` را برای آن نشست غیرفعال نگه می‌دارد.
- فرمان‌ها در یک پوسته تازه و غیرتعاملی در دایرکتوری کاری TUI اجرا می‌شوند (بدون `cd`/env پایدار).
- فرمان‌های پوسته محلی در محیط خود `OPENCLAW_SHELL=tui-local` دریافت می‌کنند.
- یک `!` تنها به‌عنوان پیام عادی ارسال می‌شود؛ فاصله‌های ابتدایی اجرای محلی را فعال نمی‌کنند.

## تعمیر پیکربندی‌ها از TUI محلی

زمانی از حالت محلی استفاده کنید که پیکربندی فعلی از قبل اعتبارسنجی می‌شود و می‌خواهید عامل تعبیه‌شده آن را روی همان دستگاه بررسی کند، آن را با مستندات مقایسه کند، و بدون وابستگی به Gateway در حال اجرا به تعمیر drift کمک کند.

اگر `openclaw config validate` از قبل شکست می‌خورد، ابتدا با `openclaw configure` یا `openclaw doctor --fix` شروع کنید. `openclaw chat` guard پیکربندی نامعتبر را دور نمی‌زند.

چرخه معمول:

1. حالت محلی را شروع کنید:

```bash
openclaw chat
```

2. از عامل بپرسید چه چیزی را می‌خواهید بررسی کند، برای مثال:

```text
Compare my gateway auth config with the docs and suggest the smallest fix.
```

3. برای شواهد دقیق و اعتبارسنجی از فرمان‌های پوسته محلی استفاده کنید:

```text
!openclaw config file
!openclaw docs gateway auth token secretref
!openclaw config validate
!openclaw doctor
```

4. تغییرات محدود را با `openclaw config set` یا `openclaw configure` اعمال کنید، سپس `!openclaw config validate` را دوباره اجرا کنید.
5. اگر Doctor یک مهاجرت یا تعمیر خودکار پیشنهاد کرد، آن را بازبینی کنید و `!openclaw doctor --fix` را اجرا کنید.

نکته‌ها:

- به‌جای ویرایش دستی `openclaw.json`، `openclaw config set` یا `openclaw configure` را ترجیح دهید.
- `openclaw docs "<query>"` فهرست مستندات زنده را از همان دستگاه جست‌وجو می‌کند.
- `openclaw config validate --json` زمانی مفید است که خطاهای ساخت‌یافته schema و SecretRef/قابلیت resolve شدن را می‌خواهید.

## خروجی ابزار

- فراخوانی‌های ابزار به‌صورت کارت‌هایی با آرگومان‌ها + نتایج نمایش داده می‌شوند.
- Ctrl+O بین نماهای جمع‌شده/گسترش‌یافته جابه‌جا می‌شود.
- هنگام اجرای ابزارها، به‌روزرسانی‌های جزئی در همان کارت پخش می‌شوند.

## رنگ‌های ترمینال

- TUI متن بدنه دستیار را با رنگ پیش‌زمینه پیش‌فرض ترمینال شما نگه می‌دارد تا ترمینال‌های تیره و روشن هر دو خوانا بمانند.
- اگر ترمینال شما از پس‌زمینه روشن استفاده می‌کند و تشخیص خودکار اشتباه است، پیش از اجرای `openclaw tui` مقدار `OPENCLAW_THEME=light` را تنظیم کنید.
- برای اجبار به استفاده از پالت تیره اصلی، مقدار `OPENCLAW_THEME=dark` را تنظیم کنید.

## تاریخچه + پخش

- هنگام اتصال، TUI آخرین تاریخچه را بارگذاری می‌کند (پیش‌فرض 200 پیام).
- پاسخ‌های پخش‌شونده تا زمان نهایی‌شدن درجا به‌روزرسانی می‌شوند.
- TUI همچنین برای کارت‌های ابزار غنی‌تر به رویدادهای ابزار عامل گوش می‌دهد.

## جزئیات اتصال

- TUI با `mode: "tui"` در Gateway ثبت می‌شود.
- اتصال‌های مجدد یک پیام سیستمی نشان می‌دهند؛ gapهای رویداد در گزارش نمایش داده می‌شوند.

## گزینه‌ها

- `--local`: اجرا در برابر runtime عامل تعبیه‌شده محلی
- `--url <url>`: URL WebSocket مربوط به Gateway (به‌صورت پیش‌فرض از پیکربندی یا `ws://127.0.0.1:<port>`)
- `--token <token>`: توکن Gateway (در صورت نیاز)
- `--password <password>`: گذرواژه Gateway (در صورت نیاز)
- `--session <key>`: کلید نشست (پیش‌فرض: `main`، یا `global` زمانی که دامنه global است)
- `--deliver`: تحویل پاسخ‌های دستیار به ارائه‌دهنده (پیش‌فرض خاموش)
- `--thinking <level>`: override سطح تفکر برای ارسال‌ها
- `--message <text>`: ارسال یک پیام اولیه پس از اتصال
- `--timeout-ms <ms>`: مهلت زمانی عامل بر حسب ms (به‌صورت پیش‌فرض از `agents.defaults.timeoutSeconds`)
- `--history-limit <n>`: ورودی‌های تاریخچه برای بارگذاری (پیش‌فرض `200`)

<Warning>
وقتی `--url` را تنظیم می‌کنید، TUI به پیکربندی یا اعتبارنامه‌های محیطی fallback نمی‌کند. `--token` یا `--password` را صریحاً پاس بدهید. نبود اعتبارنامه‌های صریح یک خطاست. در حالت محلی، `--url`، `--token` یا `--password` را پاس ندهید.
</Warning>

## عیب‌یابی

پس از ارسال پیام خروجی‌ای وجود ندارد:

- در TUI فرمان `/status` را اجرا کنید تا تأیید شود Gateway متصل و بیکار/مشغول است.
- گزارش‌های Gateway را بررسی کنید: `openclaw logs --follow`.
- تأیید کنید عامل می‌تواند اجرا شود: `openclaw status` و `openclaw models status`.
- اگر انتظار پیام در یک کانال گفت‌وگو را دارید، تحویل را فعال کنید (`/deliver on` یا `--deliver`).

## عیب‌یابی اتصال

- `disconnected`: مطمئن شوید Gateway در حال اجراست و `--url/--token/--password` شما درست هستند.
- نبود عامل در انتخاب‌گر: `openclaw agents list` و پیکربندی مسیریابی خود را بررسی کنید.
- انتخاب‌گر نشست خالی: ممکن است در دامنه global باشید یا هنوز هیچ نشستی نداشته باشید.
