## شروع سریع

### حالت Gateway

1. Gateway را شروع کنید.

```bash
openclaw gateway
```

2. TUI را باز کنید.

```bash
openclaw tui
```

3. پیامی تایپ کنید و Enter را فشار دهید.

Gateway راه دور:

```bash
openclaw tui --url ws://<host>:<port> --token <gateway-token>
```

اگر Gateway شما از احراز هویت با گذرواژه استفاده می‌کند، از `--password` استفاده کنید.

### حالت محلی

TUI را بدون Gateway اجرا کنید:

```bash
openclaw chat
# or
openclaw tui --local
```

نکات:

- `openclaw chat` و `openclaw terminal` نام‌های مستعار `openclaw tui --local` هستند.
- `--local` را نمی‌توان با `--url`، `--token` یا `--password` ترکیب کرد.
- حالت محلی مستقیماً از زمان‌اجرای تعبیه‌شده عامل استفاده می‌کند. بیشتر ابزارهای محلی کار می‌کنند، اما قابلیت‌های مخصوص Gateway در دسترس نیستند.
- پس از آنکه یک فایل پیکربندی تنظیماتی داشته باشد، `openclaw` و `openclaw crestodian` نیز از این پوسته TUI استفاده می‌کنند، با Crestodian به‌عنوان بک‌اند گفت‌وگوی راه‌اندازی و تعمیر محلی.

## آنچه می‌بینید

- سربرگ: URL اتصال، عامل فعلی، نشست فعلی.
- گزارش گفت‌وگو: پیام‌های کاربر، پاسخ‌های دستیار، اعلان‌های سیستم، کارت‌های ابزار.
- خط وضعیت: وضعیت اتصال/اجرا (در حال اتصال، در حال اجرا، در حال پخش جریانی، بیکار، خطا).
- پابرگ: عامل + نشست + مدل + وضعیت هدف + فکر/سریع/پرحرف/ردیابی/استدلال + شمارش توکن + تحویل. وقتی `tui.footer.showRemoteHost` فعال باشد، اتصال‌های Gateway راه دور میزبان اتصال را هم نشان می‌دهند.
- ورودی: ویرایشگر متن با تکمیل خودکار.

## مدل ذهنی: عامل‌ها + نشست‌ها

- عامل‌ها اسلاگ‌های یکتا هستند (مثلاً `main`، `research`). Gateway فهرست را ارائه می‌کند.
- نشست‌ها متعلق به عامل فعلی هستند.
- کلیدهای نشست به‌شکل `agent:<agentId>:<sessionKey>` ذخیره می‌شوند.
  - اگر `/session main` را تایپ کنید، TUI آن را به `agent:<currentAgent>:main` گسترش می‌دهد.
  - اگر `/session agent:other:main` را تایپ کنید، صراحتاً به آن نشست عامل جابه‌جا می‌شوید.
- دامنه نشست:
  - `per-sender` (پیش‌فرض): هر عامل نشست‌های زیادی دارد.
  - `global`: TUI همیشه از نشست `global` استفاده می‌کند (انتخاب‌گر ممکن است خالی باشد).
- عامل + نشست فعلی همیشه در پابرگ دیده می‌شوند.
- برای نمایش میزبان Gateway برای اتصال‌های غیرمحلی مبتنی بر URL، با این دستور آن را فعال کنید:

  ```bash
  openclaw config set tui.footer.showRemoteHost true
  ```

  اتصال‌های loopback و محلی تعبیه‌شده هرگز برچسب میزبان نشان نمی‌دهند.

- اگر نشست یک [هدف](/fa/tools/goal) داشته باشد، پابرگ وضعیت فشرده آن را نشان می‌دهد،
  مانند `Pursuing goal`، `Goal paused (/goal resume)` یا
  `Goal achieved`.
- وقتی بدون `--session` شروع شود، TUI در حالت Gateway آخرین نشست انتخاب‌شده را برای همان Gateway، عامل و دامنه نشست از سر می‌گیرد، اگر آن نشست هنوز وجود داشته باشد. ارسال `--session`، `/session`، `/new` یا `/reset` همچنان صریح است.

## ارسال + تحویل

- پیام‌ها به Gateway ارسال می‌شوند؛ تحویل به ارائه‌دهندگان به‌صورت پیش‌فرض خاموش است.
- TUI یک سطح منبع داخلی مانند WebChat است، نه یک کانال خروجی عمومی. هارنس‌هایی که برای پاسخ‌های قابل مشاهده به `tools.message` نیاز دارند می‌توانند نوبت فعال TUI را با یک `message.send` بدون هدف برآورده کنند؛ تحویل صریح ارائه‌دهنده همچنان از کانال‌های پیکربندی‌شده معمول استفاده می‌کند و هرگز به `lastChannel` بازنمی‌گردد.
- روشن کردن تحویل:
  - `/deliver on`
  - یا پنل تنظیمات
  - یا شروع با `openclaw tui --deliver`

## انتخاب‌گرها + پوشش‌ها

- انتخاب‌گر مدل: مدل‌های موجود را فهرست کنید و بازنویسی نشست را تنظیم کنید.
- انتخاب‌گر عامل: عامل دیگری انتخاب کنید.
- انتخاب‌گر نشست: تا ۵۰ نشست را برای عامل فعلی که در ۷ روز گذشته به‌روزرسانی شده‌اند نشان می‌دهد. برای رفتن به یک نشست قدیمی‌ترِ شناخته‌شده از `/session <key>` استفاده کنید.
- تنظیمات: تحویل، گسترش خروجی ابزار و نمایانی فکر کردن را تغییر دهید.

## میانبرهای صفحه‌کلید

- Enter: ارسال پیام
- Esc: لغو اجرای فعال
- Ctrl+C: پاک کردن ورودی (برای خروج دو بار فشار دهید)
- Ctrl+D: خروج
- Ctrl+L: انتخاب‌گر مدل
- Ctrl+G: انتخاب‌گر عامل
- Ctrl+P: انتخاب‌گر نشست
- Ctrl+O: تغییر گسترش خروجی ابزار
- Ctrl+T: تغییر نمایانی فکر کردن (تاریخچه را دوباره بارگذاری می‌کند)

## دستورهای اسلش

هسته:

- `/help`
- `/status`
- `/agent <id>` (یا `/agents`)
- `/session <key>` (یا `/sessions`)
- `/model <provider/model>` (یا `/models`)

کنترل‌های نشست:

- `/think <off|minimal|low|medium|high>`
- `/fast <status|on|off>`
- `/verbose <on|full|off>`
- `/trace <on|off>`
- `/reasoning <on|off|stream>`
- `/usage <off|tokens|full|reset>` (`reset`/`inherit`/`clear`/`default` بازنویسی نشست را پاک می‌کند)
- `/goal [status] | /goal start <objective> | /goal pause|resume|complete|block|clear`
- `/elevated <on|off|ask|full>` (نام مستعار: `/elev`)
- `/activation <mention|always>`
- `/deliver <on|off>`

چرخه عمر نشست:

- `/new` یا `/reset` (بازنشانی نشست)
- `/abort` (لغو اجرای فعال)
- `/settings`
- `/exit`

فقط حالت محلی:

- `/auth [provider]` جریان احراز هویت/ورود ارائه‌دهنده را داخل TUI باز می‌کند.

سایر دستورهای اسلش Gateway (برای مثال، `/context`) به Gateway ارسال می‌شوند و به‌عنوان خروجی سیستم نشان داده می‌شوند. [دستورهای اسلش](/fa/tools/slash-commands) را ببینید.

## دستورهای پوسته محلی

- برای اجرای یک دستور پوسته محلی روی میزبان TUI، ابتدای خط را با `!` شروع کنید.
- TUI برای اجازه اجرای محلی در هر نشست یک‌بار درخواست تأیید می‌کند؛ رد کردن، `!` را برای نشست غیرفعال نگه می‌دارد.
- دستورها در یک پوسته تازه و غیرتعاملی در پوشه کاری TUI اجرا می‌شوند (بدون `cd`/env پایدار).
- دستورهای پوسته محلی در محیط خود `OPENCLAW_SHELL=tui-local` دریافت می‌کنند.
- یک `!` تنها به‌عنوان پیام عادی ارسال می‌شود؛ فاصله‌های ابتدایی اجرای محلی را فعال نمی‌کنند.

## تعمیر پیکربندی‌ها از TUI محلی

وقتی پیکربندی فعلی از قبل معتبر است و می‌خواهید عامل تعبیه‌شده
آن را روی همان دستگاه بررسی کند، با مستندات مقایسه کند
و بدون وابستگی به Gateway در حال اجرا به تعمیر انحراف کمک کند، از حالت محلی استفاده کنید.

اگر `openclaw config validate` از قبل شکست می‌خورد، ابتدا با `openclaw configure`
یا `openclaw doctor --fix` شروع کنید. `openclaw chat` گارد پیکربندی نامعتبر را دور نمی‌زند.

حلقه معمول:

1. حالت محلی را شروع کنید:

```bash
openclaw chat
```

2. از عامل بخواهید آنچه می‌خواهید بررسی شود را انجام دهد، برای مثال:

```text
Compare my gateway auth config with the docs and suggest the smallest fix.
```

3. برای شواهد و اعتبارسنجی دقیق از دستورهای پوسته محلی استفاده کنید:

```text
!openclaw config file
!openclaw docs gateway auth token secretref
!openclaw config validate
!openclaw doctor
```

4. تغییرات محدود را با `openclaw config set` یا `openclaw configure` اعمال کنید، سپس دوباره `!openclaw config validate` را اجرا کنید.
5. اگر Doctor یک مهاجرت یا تعمیر خودکار پیشنهاد کرد، آن را بازبینی کنید و `!openclaw doctor --fix` را اجرا کنید.

نکته‌ها:

- به‌جای ویرایش دستی `openclaw.json`، `openclaw config set` یا `openclaw configure` را ترجیح دهید.
- `openclaw docs "<query>"` نمایه مستندات زنده را از همان دستگاه جست‌وجو می‌کند.
- `openclaw config validate --json` وقتی خروجی ساختاریافته خطاهای اسکیما و SecretRef/قابلیت رفع را می‌خواهید مفید است.

## خروجی ابزار

- فراخوانی‌های ابزار به‌صورت کارت‌هایی با آرگومان‌ها + نتایج نشان داده می‌شوند.
- Ctrl+O بین نماهای جمع‌شده/گسترش‌یافته جابه‌جا می‌شود.
- هنگام اجرای ابزارها، به‌روزرسانی‌های جزئی در همان کارت پخش جریانی می‌شوند.

## رنگ‌های ترمینال

- TUI متن بدنه دستیار را در پیش‌زمینه پیش‌فرض ترمینال شما نگه می‌دارد تا ترمینال‌های تیره و روشن هر دو خوانا بمانند.
- اگر ترمینال شما پس‌زمینه روشن دارد و تشخیص خودکار اشتباه است، پیش از اجرای `openclaw tui` مقدار `OPENCLAW_THEME=light` را تنظیم کنید.
- برای اجبار به استفاده از پالت تیره اصلی، به‌جای آن `OPENCLAW_THEME=dark` را تنظیم کنید.

## تاریخچه + پخش جریانی

- هنگام اتصال، TUI آخرین تاریخچه را بارگذاری می‌کند (پیش‌فرض ۲۰۰ پیام).
- پاسخ‌های پخش جریانی تا نهایی شدن درجا به‌روزرسانی می‌شوند.
- TUI همچنین برای کارت‌های ابزار غنی‌تر به رویدادهای ابزار عامل گوش می‌دهد.

## جزئیات اتصال

- TUI با `mode: "tui"` در Gateway ثبت می‌شود.
- اتصال‌های مجدد یک پیام سیستم نشان می‌دهند؛ فاصله‌های رویداد در گزارش نمایش داده می‌شوند.

## گزینه‌ها

- `--local`: اجرا در برابر زمان‌اجرای عامل تعبیه‌شده محلی
- `--url <url>`: URL وب‌سوکت Gateway (پیش‌فرض از پیکربندی یا `ws://127.0.0.1:<port>`)
- `--token <token>`: توکن Gateway (در صورت نیاز)
- `--password <password>`: گذرواژه Gateway (در صورت نیاز)
- `--session <key>`: کلید نشست (پیش‌فرض: `main`، یا وقتی دامنه global است `global`)
- `--deliver`: تحویل پاسخ‌های دستیار به ارائه‌دهنده (پیش‌فرض خاموش)
- `--thinking <level>`: بازنویسی سطح فکر کردن برای ارسال‌ها
- `--message <text>`: ارسال یک پیام اولیه پس از اتصال
- `--timeout-ms <ms>`: مهلت زمانی عامل بر حسب میلی‌ثانیه (پیش‌فرض از `agents.defaults.timeoutSeconds`)
- `--history-limit <n>`: ورودی‌های تاریخچه برای بارگذاری (پیش‌فرض `200`)

<Warning>
وقتی `--url` را تنظیم می‌کنید، TUI به اعتبارنامه‌های پیکربندی یا محیط بازنمی‌گردد. `--token` یا `--password` را صریح ارسال کنید. نبود اعتبارنامه‌های صریح خطا است. در حالت محلی، `--url`، `--token` یا `--password` را ارسال نکنید.
</Warning>

## عیب‌یابی

پس از ارسال پیام خروجی‌ای ندارید:

- برای تأیید اینکه Gateway متصل و بیکار/مشغول است، `/status` را در TUI اجرا کنید.
- گزارش‌های Gateway را بررسی کنید: `openclaw logs --follow`.
- تأیید کنید عامل می‌تواند اجرا شود: `openclaw status` و `openclaw models status`.
- اگر انتظار پیام‌ها را در یک کانال گفت‌وگو دارید، تحویل را فعال کنید (`/deliver on` یا `--deliver`).

## عیب‌یابی اتصال

- `disconnected`: مطمئن شوید Gateway در حال اجرا است و `--url/--token/--password` شما درست هستند.
- نبود عامل در انتخاب‌گر: `openclaw agents list` و پیکربندی مسیریابی خود را بررسی کنید.
- انتخاب‌گر نشست خالی: ممکن است در دامنه global باشید یا هنوز هیچ نشستی نداشته باشید.
