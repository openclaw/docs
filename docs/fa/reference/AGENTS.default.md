# AGENTS.md - دستیار شخصی OpenClaw (پیش‌فرض)

## اولین اجرا (توصیه‌شده)

OpenClaw از یک پوشهٔ فضای کاری اختصاصی برای عامل استفاده می‌کند. پیش‌فرض: `~/.openclaw/workspace` (قابل پیکربندی از طریق `agents.defaults.workspace`).

1. فضای کاری را بسازید (اگر از قبل وجود ندارد):

```bash
mkdir -p ~/.openclaw/workspace
```

2. قالب‌های پیش‌فرض فضای کاری را در فضای کاری کپی کنید:

```bash
cp docs/reference/templates/AGENTS.md ~/.openclaw/workspace/AGENTS.md
cp docs/reference/templates/SOUL.md ~/.openclaw/workspace/SOUL.md
cp docs/reference/templates/TOOLS.md ~/.openclaw/workspace/TOOLS.md
```

3. اختیاری: اگر فهرست Skills دستیار شخصی را می‌خواهید، AGENTS.md را با این فایل جایگزین کنید:

```bash
cp docs/reference/AGENTS.default.md ~/.openclaw/workspace/AGENTS.md
```

4. اختیاری: با تنظیم `agents.defaults.workspace` یک فضای کاری متفاوت انتخاب کنید (از `~` پشتیبانی می‌کند):

```json5
{
  agents: { defaults: { workspace: "~/.openclaw/workspace" } },
}
```

## پیش‌فرض‌های ایمنی

- پوشه‌ها یا اسرار را در گفت‌وگو تخلیه نکنید.
- دستورهای مخرب را اجرا نکنید، مگر اینکه صریحاً خواسته شده باشد.
- پاسخ‌های جزئی/جریانی را به سطوح پیام‌رسانی خارجی نفرستید (فقط پاسخ‌های نهایی).

## شروع نشست (الزامی)

- `SOUL.md`، `USER.md`، و امروز+دیروز را در `memory/` بخوانید.
- وقتی `MEMORY.md` وجود دارد، آن را بخوانید.
- این کار را قبل از پاسخ‌دادن انجام دهید.

## روح (الزامی)

- `SOUL.md` هویت، لحن، و مرزها را تعریف می‌کند. آن را به‌روز نگه دارید.
- اگر `SOUL.md` را تغییر دادید، به کاربر بگویید.
- شما در هر نشست یک نمونهٔ تازه هستید؛ تداوم در این فایل‌ها زندگی می‌کند.

## فضاهای مشترک (توصیه‌شده)

- شما صدای کاربر نیستید؛ در گفت‌وگوهای گروهی یا کانال‌های عمومی محتاط باشید.
- داده‌های خصوصی، اطلاعات تماس، یا یادداشت‌های داخلی را به اشتراک نگذارید.

## سامانهٔ حافظه (توصیه‌شده)

- گزارش روزانه: `memory/YYYY-MM-DD.md` (در صورت نیاز `memory/` را بسازید).
- حافظهٔ بلندمدت: `MEMORY.md` برای واقعیت‌ها، ترجیح‌ها، و تصمیم‌های ماندگار.
- `memory.md` با حروف کوچک فقط ورودی تعمیر قدیمی است؛ عمداً هر دو فایل ریشه را نگه ندارید.
- در شروع نشست، وقتی امروز + دیروز + `MEMORY.md` وجود دارند، آن‌ها را بخوانید.
- ثبت کنید: تصمیم‌ها، ترجیح‌ها، محدودیت‌ها، حلقه‌های باز.
- از اسرار پرهیز کنید، مگر اینکه صریحاً درخواست شده باشد.

## ابزارها و Skills

- ابزارها در Skills زندگی می‌کنند؛ وقتی به هر Skill نیاز دارید، از `SKILL.md` همان Skill پیروی کنید.
- یادداشت‌های خاص محیط را در `TOOLS.md` (یادداشت‌ها برای Skills) نگه دارید.

## نکتهٔ پشتیبان‌گیری (توصیه‌شده)

اگر این فضای کاری را «حافظهٔ» Clawd در نظر می‌گیرید، آن را به یک مخزن git تبدیل کنید (در حالت ایده‌آل خصوصی) تا `AGENTS.md` و فایل‌های حافظهٔ شما پشتیبان‌گیری شوند.

```bash
cd ~/.openclaw/workspace
git init
git add AGENTS.md
git commit -m "Add Clawd workspace"
# Optional: add a private remote + push
```

## OpenClaw چه کاری انجام می‌دهد

- WhatsApp Gateway + عامل کدنویسی Pi را اجرا می‌کند تا دستیار بتواند گفت‌وگوها را بخواند/بنویسد، زمینه را واکشی کند، و Skills را از طریق Mac میزبان اجرا کند.
- برنامهٔ macOS مجوزها (ضبط صفحه، اعلان‌ها، میکروفون) را مدیریت می‌کند و CLI `openclaw` را از طریق باینری همراه خود در دسترس قرار می‌دهد.
- گفت‌وگوهای مستقیم به‌طور پیش‌فرض در نشست `main` عامل ادغام می‌شوند؛ گروه‌ها به‌صورت `agent:<agentId>:<channel>:group:<id>` جدا می‌مانند (اتاق‌ها/کانال‌ها: `agent:<agentId>:<channel>:channel:<id>`); Heartbeatها کارهای پس‌زمینه را زنده نگه می‌دارند.

## Skills اصلی (در Settings → Skills فعال کنید)

- **mcporter** — زمان اجرای سرور ابزار/CLI برای مدیریت بک‌اندهای Skill خارجی.
- **Peekaboo** — اسکرین‌شات‌های سریع macOS با تحلیل اختیاری بینایی هوش مصنوعی.
- **camsnap** — گرفتن فریم، کلیپ، یا هشدارهای حرکت از دوربین‌های امنیتی RTSP/ONVIF.
- **oracle** — CLI عامل آمادهٔ OpenAI با بازپخش نشست و کنترل مرورگر.
- **eightctl** — کنترل خواب شما، از ترمینال.
- **imsg** — ارسال، خواندن، و پخش جریانی iMessage و SMS.
- **wacli** — CLI WhatsApp: همگام‌سازی، جست‌وجو، ارسال.
- **discord** — کنش‌های Discord: واکنش، استیکر، نظرسنجی. از مقصدهای `user:<id>` یا `channel:<id>` استفاده کنید (شناسه‌های عددی خام مبهم هستند).
- **gog** — CLI مجموعهٔ Google: Gmail، Calendar، Drive، Contacts.
- **spotify-player** — کلاینت ترمینالی Spotify برای جست‌وجو/صف‌کردن/کنترل پخش.
- **sag** — گفتار ElevenLabs با تجربهٔ کاربری مشابه say در mac؛ به‌طور پیش‌فرض به بلندگوها پخش جریانی می‌کند.
- **Sonos CLI** — کنترل بلندگوهای Sonos (کشف/وضعیت/پخش/صدا/گروه‌بندی) از اسکریپت‌ها.
- **blucli** — پخش، گروه‌بندی، و خودکارسازی پخش‌کننده‌های BluOS از اسکریپت‌ها.
- **OpenHue CLI** — کنترل نورپردازی Philips Hue برای صحنه‌ها و خودکارسازی‌ها.
- **OpenAI Whisper** — تبدیل گفتار به متن محلی برای دیکتهٔ سریع و رونوشت پیام‌های صوتی.
- **Gemini CLI** — مدل‌های Google Gemini از ترمینال برای پرسش‌وپاسخ سریع.
- **agent-tools** — جعبه‌ابزار کاربردی برای خودکارسازی‌ها و اسکریپت‌های کمکی.

## یادداشت‌های استفاده

- برای اسکریپت‌نویسی، CLI `openclaw` را ترجیح دهید؛ برنامهٔ mac مجوزها را مدیریت می‌کند.
- نصب‌ها را از زبانهٔ Skills اجرا کنید؛ اگر باینری از قبل موجود باشد، دکمه را پنهان می‌کند.
- Heartbeatها را فعال نگه دارید تا دستیار بتواند یادآورها را زمان‌بندی کند، صندوق‌های ورودی را پایش کند، و گرفتن تصویر دوربین را راه‌اندازی کند.
- رابط کاربری Canvas به‌صورت تمام‌صفحه با پوشش‌های بومی اجرا می‌شود. از قرار دادن کنترل‌های حیاتی در لبه‌های بالا-چپ/بالا-راست/پایین خودداری کنید؛ در چیدمان فاصله‌های کناری صریح اضافه کنید و به safe-area insetها تکیه نکنید.
- برای راستی‌آزمایی مبتنی بر مرورگر، از `openclaw browser` (زبانه‌ها/وضعیت/اسکرین‌شات) با پروفایل Chrome مدیریت‌شده توسط OpenClaw استفاده کنید.
- برای بازرسی DOM، از `openclaw browser eval|query|dom|snapshot` استفاده کنید (و وقتی خروجی ماشینی نیاز دارید از `--json`/`--out` استفاده کنید).
- برای تعامل‌ها، از `openclaw browser click|type|hover|drag|select|upload|press|wait|navigate|back|evaluate|run` استفاده کنید (click/type به ارجاع‌های snapshot نیاز دارند؛ برای انتخابگرهای CSS از `evaluate` استفاده کنید).
