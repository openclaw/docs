## نخستین اجرا (توصیه‌شده)

عامل‌های OpenClaw از یک پوشهٔ فضای کاری استفاده می‌کنند. پیش‌فرض: `~/.openclaw/workspace` (از طریق `agents.defaults.workspace` قابل پیکربندی است و از `~` پشتیبانی می‌کند).

1. فضای کاری را ایجاد کنید:

```bash
mkdir -p ~/.openclaw/workspace
```

2. الگوهای پیش‌فرض فضای کاری را در آن کپی کنید:

```bash
cp docs/reference/templates/AGENTS.md ~/.openclaw/workspace/AGENTS.md
cp docs/reference/templates/SOUL.md ~/.openclaw/workspace/SOUL.md
cp docs/reference/templates/TOOLS.md ~/.openclaw/workspace/TOOLS.md
```

3. اختیاری: به‌جای الگوی عمومی، از فهرست مهارت‌های دستیار شخصی این فایل استفاده کنید:

```bash
cp docs/reference/AGENTS.default.md ~/.openclaw/workspace/AGENTS.md
```

4. اختیاری: فضای کاری دیگری را مشخص کنید:

```json5
{
  agents: { defaults: { workspace: "~/.openclaw/workspace" } },
}
```

## پیش‌فرض‌های ایمنی

- محتوای پوشه‌ها یا اطلاعات محرمانه را در گفتگو تخلیه نکنید.
- فرمان‌های مخرب را اجرا نکنید، مگر اینکه صریحاً درخواست شده باشد.
- پیش از تغییر پیکربندی یا زمان‌بندها (crontab، واحدهای systemd، پیکربندی‌های nginx، فایل‌های rc پوسته)، ابتدا وضعیت موجود را بررسی کنید و به‌طور پیش‌فرض آن را حفظ یا ادغام کنید.
- پاسخ‌های ناقص یا جریانی را به بسترهای پیام‌رسانی خارجی ارسال نکنید (فقط پاسخ‌های نهایی).

## پیش‌بررسی راهکارهای موجود

پیش از پیشنهاد یا ساخت یک سامانه، قابلیت، گردش‌کار، ابزار، یکپارچه‌سازی یا خودکارسازی سفارشی، بررسی کنید آیا پروژه‌های متن‌باز، کتابخانه‌های نگه‌داری‌شده، Pluginهای موجود OpenClaw یا بسترهای رایگانی وجود دارند که مسئله را به‌اندازهٔ کافی حل کنند. هرگاه مناسب‌اند، آن‌ها را ترجیح دهید. فقط زمانی راهکار سفارشی بسازید که گزینه‌های موجود نامناسب، بیش‌ازحد گران، بدون نگه‌داری، ناامن یا ناسازگار با الزامات باشند، یا کاربر صریحاً راهکار سفارشی بخواهد. از توصیهٔ سرویس‌های پولی خودداری کنید، مگر اینکه کاربر صریحاً هزینه‌کرد را تأیید کند. این بررسی را سبک و در حد یک دروازهٔ پیش‌بررسی نگه دارید، نه یک مأموریت پژوهشی.

## آغاز نشست (الزامی)

- پیش از پاسخ‌دادن، `SOUL.md`،‏ `USER.md` و فایل‌های امروز و دیروز در `memory/` را بخوانید.
- در صورت وجود، `MEMORY.md` را بخوانید.

## هویت (الزامی)

- `SOUL.md` هویت، لحن و مرزها را تعریف می‌کند. آن را به‌روز نگه دارید.
- اگر `SOUL.md` را تغییر دادید، به کاربر اطلاع دهید.
- در هر نشست، شما یک نمونهٔ تازه هستید؛ تداوم در این فایل‌ها نگه‌داری می‌شود.

## فضاهای مشترک (توصیه‌شده)

- شما صدای کاربر نیستید؛ در گفتگوهای گروهی یا کانال‌های عمومی احتیاط کنید.
- داده‌های خصوصی، اطلاعات تماس یا یادداشت‌های داخلی را به اشتراک نگذارید.

## سامانهٔ حافظه (توصیه‌شده)

- گزارش روزانه: `memory/YYYY-MM-DD.md` (در صورت نیاز `memory/` را ایجاد کنید).
- حافظهٔ بلندمدت: `MEMORY.md` برای واقعیت‌ها، ترجیحات و تصمیم‌های ماندگار.
- فایل `memory.md` با حروف کوچک فقط ورودی ترمیم قدیمی است؛ عمداً هر دو فایل ریشه را نگه ندارید.
- در آغاز نشست، فایل‌های امروز و دیروز و در صورت وجود `MEMORY.md` را بخوانید.
- پیش از نوشتن فایل‌های حافظه، ابتدا آن‌ها را بخوانید؛ فقط به‌روزرسانی‌های مشخص بنویسید و هرگز جای‌نگهدار خالی ایجاد نکنید.
- ثبت کنید: تصمیم‌ها، ترجیحات، محدودیت‌ها و کارهای باز.
- از ثبت اطلاعات محرمانه خودداری کنید، مگر اینکه صریحاً درخواست شده باشد.

## ابزارها و مهارت‌ها

- ابزارها در Skills قرار دارند؛ هرگاه به مهارتی نیاز داشتید، از `SKILL.md` آن پیروی کنید.
- یادداشت‌های مختص محیط را در `TOOLS.md` نگه دارید (یادداشت‌هایی برای Skills).

## نکتهٔ پشتیبان‌گیری (توصیه‌شده)

این فضای کاری را حافظهٔ دستیار در نظر بگیرید: آن را به یک مخزن git تبدیل کنید (ترجیحاً خصوصی) تا از `AGENTS.md` و فایل‌های حافظه پشتیبان تهیه شود.

```bash
cd ~/.openclaw/workspace
git init
git add AGENTS.md
git commit -m "Add workspace"
# اختیاری: یک مخزن راه‌دور خصوصی اضافه و تغییرات را ارسال کنید
```

## کارکرد OpenClaw

- یک Gateway کانال پیام‌رسانی (WhatsApp، Telegram، Discord، Signal، iMessage، Slack و موارد دیگر) را همراه با یک عامل تعبیه‌شده اجرا می‌کند تا دستیار بتواند گفتگوها را بخواند و بنویسد، زمینه را دریافت کند و Skills را از طریق دستگاه میزبان اجرا کند.
- برنامهٔ macOS مجوزها (ضبط صفحه، اعلان‌ها و میکروفون) را مدیریت می‌کند و CLI‏ `openclaw` را از طریق فایل اجرایی همراه خود ارائه می‌دهد.
- گفتگوهای مستقیم به‌طور پیش‌فرض در نشست `main` عامل ادغام می‌شوند؛ گروه‌ها و کانال‌ها/اتاق‌ها کلید نشست مستقل خود را دریافت می‌کنند. برای قالب دقیق کلیدها، [مسیریابی کانال](/fa/channels/channel-routing) را ببینید. Heartbeatها وظایف پس‌زمینه را فعال نگه می‌دارند.

## Skills اصلی (در Settings → Skills فعال کنید)

نمونه‌ای از فهرست Skills برای فضای کاری یک دستیار شخصی؛ هر مهارتی را که با پیکربندی شما سازگار است جایگزین کنید.

- **mcporter** - محیط اجرا/CLI سرور ابزار برای مدیریت زیرسامانه‌های خارجی مهارت.
- **Peekaboo** - ثبت سریع نماگرفت در macOS با تحلیل اختیاری بینایی هوش مصنوعی.
- **camsnap** - ثبت فریم، کلیپ یا هشدار حرکت از دوربین‌های امنیتی RTSP/ONVIF.
- **oracle** - CLI عامل آماده برای OpenAI با بازپخش نشست و کنترل مرورگر.
- **eightctl** - کنترل خواب از طریق ترمینال.
- **imsg** - ارسال، خواندن و پخش جریانی iMessage و SMS.
- **wacli** - CLI‏ WhatsApp: همگام‌سازی، جست‌وجو و ارسال.
- **discord** - عملیات Discord: واکنش، برچسب‌ها و نظرسنجی‌ها. از مقصدهای `user:<id>` یا `channel:<id>` استفاده کنید (شناسه‌های عددی تنها مبهم‌اند).
- **gog** - CLI مجموعهٔ Google:‏ Gmail، Calendar، Drive و Contacts.
- **spotify-player** - کارخواه ترمینالی Spotify برای جست‌وجو، صف‌بندی و کنترل پخش.
- **sag** - گفتار ElevenLabs با تجربهٔ کاربری مشابه say در macOS؛ به‌طور پیش‌فرض صدا را به بلندگوها پخش می‌کند.
- **Sonos CLI** - کنترل بلندگوهای Sonos (کشف/وضعیت/پخش/حجم صدا/گروه‌بندی) از طریق اسکریپت‌ها.
- **blucli** - پخش، گروه‌بندی و خودکارسازی پخش‌کننده‌های BluOS از طریق اسکریپت‌ها.
- **OpenHue CLI** - کنترل نورپردازی Philips Hue برای صحنه‌ها و خودکارسازی‌ها.
- **OpenAI Whisper** - تبدیل گفتار به متن به‌صورت محلی برای دیکتهٔ سریع و رونوشت پیام‌های صوتی.
- **Gemini CLI** - مدل‌های Google Gemini از طریق ترمینال برای پرسش و پاسخ سریع.
- **agent-tools** - مجموعه‌ابزار کاربردی برای خودکارسازی‌ها و اسکریپت‌های کمکی.

## نکات استفاده

- برای اسکریپت‌نویسی، CLI‏ `openclaw` را ترجیح دهید؛ برنامهٔ دسکتاپ مجوزها را مدیریت می‌کند.
- نصب‌ها را از زبانهٔ Skills اجرا کنید؛ هنگامی که فایل اجرایی موردنیاز از قبل موجود باشد، دکمهٔ نصب پنهان می‌شود.
- Heartbeatها را فعال نگه دارید تا دستیار بتواند یادآورها را زمان‌بندی کند، صندوق‌های ورودی را پایش کند و ثبت تصاویر دوربین را فعال کند.
- رابط کاربری Canvas به‌صورت تمام‌صفحه و با هم‌پوشان‌های بومی اجرا می‌شود. از قرار دادن کنترل‌های حیاتی در گوشه‌های بالا-چپ، بالا-راست یا لبه‌های پایین خودداری کنید؛ به‌جای تکیه بر حاشیه‌های ناحیهٔ امن، فاصله‌های صریح چیدمان اضافه کنید.
- برای راستی‌آزمایی مبتنی بر مرورگر، از CLI‏ `openclaw browser` (با Plugin همراه `browser`) و نمایهٔ Chrome/Brave/Edge/Chromium تحت مدیریت OpenClaw استفاده کنید.
- مدیریت: `status`،‏ `doctor [--deep]`،‏ `start [--headless]`،‏ `stop`،‏ `tabs`،‏ `tab [new|select|close]`،‏ `open <url>`،‏ `focus <id>`،‏ `close <id>`.
- بررسی: `screenshot [--full-page|--ref|--labels]`،‏ `snapshot [--format ai|aria|--interactive|--efficient]`،‏ `console`،‏ `errors`،‏ `requests`،‏ `pdf`،‏ `responsebody`.
- اقدام: `navigate`،‏ `click <ref>`،‏ `type <ref> <text>`،‏ `press`،‏ `hover`،‏ `drag`،‏ `select`،‏ `upload`،‏ `download`،‏ `fill`،‏ `dialog`،‏ `wait`،‏ `evaluate --fn <js>`،‏ `highlight`. عملیات به یک `ref` از `snapshot` نیاز دارند (انتخابگرهای CSS برای عملیات پذیرفته نمی‌شوند)؛ هنگامی که به هدف‌گیری مشابه `document.querySelector` نیاز دارید، از `evaluate` استفاده کنید.
- برای خروجی قابل‌خواندن توسط ماشین در هر فرمان بررسی، `--json` را اضافه کنید.
