QQ Bot از طریق QQ Bot API رسمی (Gateway وب‌سوکت) به OpenClaw متصل می‌شود.
گفت‌وگوی خصوصی C2C و منشن‌های `@` در گروه، انواع اصلی گفت‌وگو هستند و از
رسانه‌های غنی (تصویر، صدا، ویدئو و فایل) پشتیبانی می‌کنند. پیام‌های کانال انجمن فقط برای
متن و تصاویر دارای URL راه دور پشتیبانی می‌شوند؛ صدا، ویدئو، بارگذاری فایل و تصاویر
محلی/Base64 در کانال‌های انجمن در دسترس نیستند. واکنش‌ها و رشته‌ها در هیچ‌جا
پشتیبانی نمی‌شوند.

وضعیت: Plugin رسمی قابل‌دانلود.

## نصب

```bash
openclaw plugins install @openclaw/qqbot
```

## راه‌اندازی

1. به [پلتفرم باز QQ](https://q.qq.com/) بروید و برای ثبت‌نام / ورود، کد QR را با
   QQ تلفن خود اسکن کنید.
2. برای ایجاد یک ربات QQ جدید، روی **Create Bot** کلیک کنید.
3. در صفحه تنظیمات ربات، **AppID** و **AppSecret** را پیدا و کپی کنید.

<Note>
AppSecret به‌صورت متن ساده ذخیره نمی‌شود. اگر بدون ذخیره‌کردن صفحه را ترک کنید، باید یک مورد جدید بازتولید کنید.
</Note>

4. کانال را اضافه کنید:

```bash
openclaw channels add --channel qqbot --token "AppID:AppSecret"
```

5. Gateway را راه‌اندازی مجدد کنید.

راه‌اندازی تعاملی:

```bash
openclaw channels add
```

جادوگر همچنین اتصال با کد QR را به‌عنوان جایگزینی برای واردکردن دستی AppID/AppSecret
ارائه می‌دهد: برای تکمیل اتصال، کد را با برنامه تلفن متصل به QQ Bot مقصد اسکن کنید.
OpenClaw اعتبارنامه‌های بازگردانده‌شده را در محدوده پیکربندی حساب
ذخیره می‌کند.

## پیکربندی

پیکربندی حداقلی:

```json5
{
  channels: {
    qqbot: {
      enabled: true,
      appId: "YOUR_APP_ID",
      clientSecret: "YOUR_APP_SECRET",
    },
  },
}
```

متغیرهای محیطی حساب پیش‌فرض (فقط حساب سطح‌بالا):

- `QQBOT_APP_ID`
- `QQBOT_CLIENT_SECRET`

AppSecret مبتنی بر فایل:

```json5
{
  channels: {
    qqbot: {
      enabled: true,
      appId: "YOUR_APP_ID",
      clientSecretFile: "/path/to/qqbot-secret.txt",
    },
  },
}
```

AppSecret از نوع SecretRef محیطی:

```json5
{
  channels: {
    qqbot: {
      enabled: true,
      appId: "YOUR_APP_ID",
      clientSecret: { source: "env", provider: "default", id: "QQBOT_CLIENT_SECRET" },
    },
  },
}
```

نکات:

- `openclaw channels add --channel qqbot --token-file ...` فقط AppSecret را تنظیم می‌کند؛
  `appId` باید از قبل در پیکربندی یا `QQBOT_APP_ID` تنظیم شده باشد.
- `clientSecret` یک رشته متن ساده، مسیر فایل (`clientSecretFile`)
  یا یک شیء ساخت‌یافته SecretRef را می‌پذیرد.
- رشته‌های نشانگر قدیمی `secretref:...` / `secretref-env:...` برای
  `clientSecret` رد می‌شوند؛ به‌جای آن از یک شیء ساخت‌یافته SecretRef استفاده کنید.

### پخش جریانی

```json5
{
  channels: {
    qqbot: {
      streaming: {
        mode: "partial", // پخش جریانی بلوکی: "partial" (پیش‌فرض) یا "off"
        nativeTransport: true, // استفاده از API رسمی stream_messages متعلق به QQ برای پیام‌های خصوصی C2C
      },
    },
  },
}
```

- `streaming.mode: "off"` پخش جریانی بلوکی را برای حساب غیرفعال می‌کند.
- `streaming.nativeTransport: true` پاسخ‌های C2C (پیام خصوصی) را از طریق
  API رسمی `stream_messages` متعلق به QQ پخش می‌کند؛ مقصدهای گروه/کانال تغییری نمی‌کنند.
- مقادیر اسکالر قدیمی `streaming: true|false` و کلید `streaming.c2cStreamApi`
  از طریق `openclaw doctor --fix` به این ساختار مهاجرت می‌کنند.
- `/bot-streaming on|off` همین پیکربندی را از طریق یک پیام خصوصی تغییر می‌دهد.

### سیاست دسترسی

- `allowFrom` / `groupAllowFrom` تعیین می‌کنند چه کسانی می‌توانند در زمینه‌های C2C /
  گروه با ربات گفت‌وگو کنند. `dmPolicy` / `groupPolicy` ‏(`open` | `allowlist` | `disabled`)
  حالت اعمال را کنترل می‌کنند. پس از آنکه `allowFrom` دارای یک ورودی مشخص
  (غیرعام) شود، مقدار پیش‌فرض `dmPolicy` برابر `allowlist` است؛ در غیر این صورت
  `open`. پس از آنکه `groupAllowFrom` یا
  `allowFrom` دارای یک ورودی مشخص شود، مقدار پیش‌فرض `groupPolicy` برابر
  `allowlist` است؛ در غیر این صورت `open`.
- دستورهای اسلش «احراز هویت: فهرست مجاز» صرف‌نظر از
  `dmPolicy` / `groupPolicy`، به یک ورودی صریح غیرعام در
  `allowFrom` (یا برای فراخوانی‌های گروهی در `groupAllowFrom`) نیاز دارند — به [دستورهای اسلش](#slash-commands) مراجعه کنید.

### راه‌اندازی چندحسابی

چند ربات QQ را در یک نمونه OpenClaw اجرا کنید:

```json5
{
  channels: {
    qqbot: {
      enabled: true,
      appId: "111111111",
      clientSecret: "secret-of-bot-1",
      accounts: {
        bot2: {
          enabled: true,
          appId: "222222222",
          clientSecret: "secret-of-bot-2",
        },
      },
    },
  },
}
```

هر حساب دارای اتصال وب‌سوکت، کلاینت API و حافظه نهان توکن مجزایی است که با
`appId` کلیدگذاری می‌شود. خطوط گزارش با شناسه حساب مالک برچسب‌گذاری می‌شوند تا
هنگام اجرای چند ربات تحت یک Gateway، عیب‌یابی‌ها از هم تفکیک‌پذیر بمانند.

افزودن ربات دوم از طریق CLI:

```bash
openclaw channels add --channel qqbot --account bot2 --token "222222222:secret-of-bot-2"
```

### گفت‌وگوهای گروهی

پشتیبانی گروه از OpenIDهای گروه QQ استفاده می‌کند، نه نام‌های نمایشی. ربات را به یک
گروه اضافه کنید، سپس آن را منشن کنید یا گروه را طوری پیکربندی کنید که بدون منشن اجرا شود.

```json5
{
  channels: {
    qqbot: {
      groupPolicy: "allowlist",
      groupAllowFrom: ["member_openid"],
      groups: {
        "*": {
          requireMention: true,
          commandLevel: "all",
          historyLimit: 50,
          tools: { deny: ["exec", "read", "write"] },
        },
        GROUP_OPENID: {
          name: "Release room",
          requireMention: false,
          ignoreOtherMentions: true,
          commandLevel: "safety",
          historyLimit: 20,
          prompt: "Keep replies short and operational.",
        },
      },
    },
  },
}
```

`groups["*"]` مقادیر پیش‌فرض همه گروه‌ها را تنظیم می‌کند؛ یک ورودی مشخص
`groups.GROUP_OPENID` این مقادیر پیش‌فرض را برای یک گروه بازنویسی می‌کند. تنظیمات گروه:

| فیلد                 | پیش‌فرض          | توضیحات                                                                                        |
| --------------------- | ---------------- | -------------------------------------------------------------------------------------------------- |
| `requireMention`      | `true`           | پیش از پاسخ‌دادن ربات، به یک منشن `@` نیاز دارد.                                                     |
| `commandLevel`        | `all`            | تعیین می‌کند کدام دستورهای اسلش داخلی می‌توانند در گروه اجرا شوند (پایین را ببینید).                                    |
| `ignoreOtherMentions` | `false`          | پیام‌هایی را که فرد دیگری را منشن می‌کنند اما ربات را نه، حذف می‌کند.                                           |
| `historyLimit`        | `50`             | پیام‌های اخیر بدون منشن که به‌عنوان زمینه برای نوبت منشن‌شده بعدی نگه‌داری می‌شوند. `0` تاریخچه را غیرفعال می‌کند.     |
| `tools`               | —                | ابزارها را برای کل گروه مجاز/غیرمجاز می‌کند.                                                              |
| `toolsBySender`       | —                | بازنویسی ابزارها به‌ازای هر فرستنده؛ به [گروه‌ها](/fa/channels/groups#groupchannel-tool-restrictions-optional) مراجعه کنید. |
| `name`                | پیشوند openid    | برچسب خوانایی که در گزارش‌ها و زمینه گروه استفاده می‌شود.                                                     |
| `prompt`              | پیش‌فرض داخلی | پرامپت رفتاری مخصوص گروه که به زمینه عامل افزوده می‌شود.                                           |

`commandLevel` موارد زیر را می‌پذیرد:

| سطح    | رفتار                                                                                                                                      |
| -------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| `all`    | دستورهای داخلی موجود همچنان در دسترس می‌مانند. برخی از منوها پنهان می‌مانند، اما کاربران مجاز همچنان می‌توانند آن‌ها را در گروه اجرا کنند.                  |
| `safety` | `/help`، `/btw`، `/stop` در گروه قابل‌مشاهده می‌مانند؛ دستورهای حساس (`/config`، `/tools`، `/bash` و غیره) باید در گفت‌وگوی خصوصی اجرا شوند.      |
| `strict` | فقط کنترل‌های نشست گروهی لازم برای عملیات سخت‌گیرانه مجاز هستند. `/stop` همچنان کار می‌کند تا فرستنده مجاز بتواند یک اجرای فعال را متوقف کند. |

ورودی‌های قدیمی `toolPolicy` در QQBot بازنشسته شده‌اند. برای مهاجرت آن‌ها به `tools`، دستور `openclaw doctor --fix` را اجرا کنید.

حالت‌های فعال‌سازی `mention` و `always` هستند. `requireMention: true` به
`mention` نگاشت می‌شود؛ `requireMention: false` به `always` نگاشت می‌شود. اگر بازنویسی
فعال‌سازی در سطح نشست وجود داشته باشد، بر پیکربندی اولویت دارد.

صف ورودی به‌ازای هر همتاست. همتاهای گروهی سقف صف بزرگ‌تری دارند (50 در برابر 20
برای همتاهای مستقیم)، هنگام پرشدن ابتدا پیام‌های نوشته‌شده توسط ربات را پیش از پیام‌های انسانی
حذف می‌کنند و توالی سریع پیام‌های عادی گروه را در یک نوبت دارای انتساب ادغام می‌کنند. دستورهای اسلش
یکی‌یکی و مستقل از هر دسته ادغام اجرا می‌شوند.

### صدا (STT / TTS)

STT و TTS از پیکربندی دوسطحی همراه با بازگشت اولویت‌دار پشتیبانی می‌کنند:

| تنظیم | مخصوص Plugin                                          | بازگشت چارچوب            |
| ------- | -------------------------------------------------------- | ----------------------------- |
| STT     | `channels.qqbot.stt`                                     | `tools.media.audio.models[0]` |
| TTS     | `channels.qqbot.tts`، `channels.qqbot.accounts.<id>.tts` | `messages.tts`                |

```json5
{
  channels: {
    qqbot: {
      stt: {
        provider: "your-provider",
        model: "your-stt-model",
      },
      tts: {
        provider: "your-provider",
        model: "your-tts-model",
        voice: "your-voice",
      },
      accounts: {
        "qq-main": {
          tts: {
            providers: {
              openai: { voice: "shimmer" },
            },
          },
        },
      },
    },
  },
}
```

برای غیرفعال‌کردن، `enabled: false` را روی هرکدام تنظیم کنید. بازنویسی‌های TTS در سطح حساب از
همان ساختار `messages.tts` استفاده می‌کنند و به‌صورت عمیق روی پیکربندی TTS کانال/سراسری ادغام می‌شوند.

درخواست‌های STT به‌طور پیش‌فرض پس از 60 ثانیه مهلتشان تمام می‌شود. STT مخصوص Plugin از
بازنویسی انتخاب‌شده `models.providers.<id>.timeoutSeconds` استفاده می‌کند. STT صوتی چارچوب
ابتدا از `tools.media.audio.models[0].timeoutSeconds`، سپس
`tools.media.audio.timeoutSeconds` و بعد از بازنویسی ارائه‌دهنده انتخاب‌شده استفاده می‌کند.

پیوست‌های صوتی ورودی QQ به‌صورت فراداده رسانه صوتی در اختیار عامل‌ها قرار می‌گیرند،
درحالی‌که فایل‌های خام صوتی از `MediaPaths` عمومی خارج نگه داشته می‌شوند. وجود `[[audio_as_voice]]`
در یک پاسخ متن ساده، در صورت پیکربندی TTS، صوت را با TTS تولید و یک پیام صوتی بومی QQ ارسال می‌کند.

رفتار بارگذاری/تبدیل کدگذاری صوت خروجی را نیز می‌توان با
`channels.qqbot.audioFormatPolicy` تنظیم کرد:

- `sttDirectFormats`
- `uploadDirectFormats`
- `transcodeEnabled`

## قالب‌های مقصد

| قالب                     | توضیحات        |
| -------------------------- | ------------------ |
| `qqbot:c2c:OPENID`         | گفت‌وگوی خصوصی (C2C) |
| `qqbot:group:GROUP_OPENID` | گفت‌وگوی گروهی         |
| `qqbot:channel:CHANNEL_ID` | کانال انجمن      |

<Note>
هر ربات مجموعه OpenIDهای کاربری مختص خود را دارد. OpenID دریافت‌شده توسط ربات A **نمی‌تواند** برای ارسال پیام از طریق ربات B استفاده شود.
</Note>

## دستورهای اسلش

دستورهای داخلی که پیش از صف هوش مصنوعی رهگیری می‌شوند:

| فرمان              | احراز هویت      | دامنه        | توضیحات                                                                    |
| -------------------- | --------- | ------------ | ------------------------------------------------------------------------------ |
| `/bot-ping`          | —         | همه          | آزمون تأخیر                                                                   |
| `/bot-help`          | —         | همه          | فهرست‌کردن همه فرمان‌ها                                                              |
| `/bot-me`            | —         | فقط خصوصی | نمایش شناسه کاربری QQ فرستنده (openid) برای راه‌اندازی `allowFrom` / `groupAllowFrom` |
| `/bot-version`       | —         | فقط خصوصی | نمایش نسخه چارچوب OpenClaw و نسخه Plugin                         |
| `/bot-upgrade`       | —         | فقط خصوصی | نمایش پیوند راهنمای ارتقای QQBot                                              |
| `/bot-approve`       | فهرست مجاز | فقط خصوصی | مدیریت پیکربندی تأیید اجرای فرمان (روشن / خاموش / همیشه / بازنشانی / وضعیت)  |
| `/bot-logs`          | فهرست مجاز | فقط خصوصی | خروجی‌گرفتن از گزارش‌های اخیر Gateway به‌صورت فایل                                           |
| `/bot-clear-storage` | فهرست مجاز | فقط خصوصی | حذف بارگیری‌های ذخیره‌شده در حافظه نهان از پوشه رسانه QQBot                        |
| `/bot-streaming`     | فهرست مجاز | فقط خصوصی | فعال یا غیرفعال‌کردن پاسخ‌های جریانی C2C                                                   |
| `/bot-group-allways` | فهرست مجاز | فقط خصوصی | تغییر حالت پیش‌فرض فعال‌سازی گروه (نیازمند اشاره در برابر همیشه فعال)      |

برای دریافت راهنمای استفاده، `?` را به هر فرمانی بیفزایید (برای نمونه `/bot-upgrade ?`).

فرمان‌های دارای «احراز هویت: فهرست مجاز» علاوه بر این، مستلزم آن‌اند که openid فرستنده در یک
فهرست صریح و بدون نویسه عام `allowFrom` قرار داشته باشد (`groupAllowFrom` برای
فرمان‌های صادرشده در گروه اولویت دارد و در صورت نبود، از `allowFrom` استفاده می‌شود). نویسه عام
`allowFrom: ["*"]` اجازه گفت‌وگو می‌دهد، اما اجازه اجرای این فرمان‌ها را نمی‌دهد. اجرای یکی از آن‌ها
خارج از گفت‌وگوی خصوصی یا بدون مجوز، به‌جای
نادیده‌گرفتن بی‌سروصدای پیام، یک راهنما برمی‌گرداند.

`/bot-me`، `/bot-version` و `/bot-upgrade` فقط در گفت‌وگوی خصوصی قابل اجرا هستند، اما
به فهرست مجاز نیاز ندارند — هر فرستنده C2C می‌تواند آن‌ها را اجرا کند.

وقتی تأییدهای اجرای QQ Bot از جایگزین پیش‌فرض همان گفت‌وگو استفاده می‌کنند، کلیک روی دکمه‌های بومی تأیید
از همان فهرست مجاز صریح و بدون نویسه عام فرمان‌ها پیروی می‌کند. برای
اعطای دسترسی صرفاً جهت تأیید، بدون دسترسی گسترده‌تر به فرمان‌ها،
`channels.qqbot.execApprovals.approvers` را پیکربندی کنید. تأییدهای بومی اجرا به‌طور
پیش‌فرض فعال هستند.

## رسانه و ذخیره‌سازی

- رسانه‌های ورودی، خروجی و پل Gateway یک ریشه محموله مشترک در
  `~/.openclaw/media/qqbot` دارند (و در صورت تنظیم، `OPENCLAW_HOME` را رعایت می‌کنند) تا بارگذاری‌ها،
  بارگیری‌ها و حافظه‌های نهان تبدیل کدگذاری همگی در یک پوشه محافظت‌شده باقی بمانند.
- تحویل رسانه غنی به مقصدهای C2C و گروهی از یک مسیر واحد `sendMedia`
  عبور می‌کند. فایل‌های محلی و بافرهای درون‌حافظه‌ای با اندازه 5&nbsp;MiB یا بیشتر از نقاط پایانی
  بارگذاری قطعه‌ای QQ استفاده می‌کنند؛ محموله‌های کوچک‌تر و منابع URL راه‌دور/Base64 از
  API بارگذاری یک‌مرحله‌ای استفاده می‌کنند.
- اگر ارتقای گرم، Gateway را پیش از پایان نوشتن
  `openclaw.json` متوقف کند، Plugin در شروع بعدی آخرین `appId` / `clientSecret`
  شناخته‌شده برای آن حساب را از یک تصویر لحظه‌ای داخلی بازیابی می‌کند (و هرگز
  تغییر عمدی پیکربندی را بازنویسی نمی‌کند)، بنابراین نیازی به اسکن دوباره کد QR
  نیست.

## عیب‌یابی

- **Gateway شروع نمی‌شود / هیچ پیام ورودی دریافت نمی‌شود:** بررسی کنید `appId` و
  `clientSecret` درست باشند و ربات در QQ Open Platform فعال شده باشد.
  نبود اطلاعات احراز هویت با پیام "QQBot پیکربندی نشده است (appId یا
  clientSecret موجود نیست)" نمایش داده می‌شود.
- **راه‌اندازی با `--token-file` همچنان وضعیت پیکربندی‌نشده را نشان می‌دهد:** `--token-file` فقط
  AppSecret را تنظیم می‌کند. `appId` همچنان باید در پیکربندی یا `QQBOT_APP_ID` تنظیم شود.
- **پاسخ‌های گروهی انبوه با هم تداخل دارند:** وقتی صف یک همتا پر می‌شود، صف ورودی
  پیام‌های نوشته‌شده توسط ربات را پیش از پیام‌های انسانی بیرون می‌اندازد و
  موج پیام‌های عادی (غیرفرمانی) گروه را در یک نوبت انتساب‌داده‌شده ادغام می‌کند؛ بنابراین
  سیل گفت‌وگوی ربات نباید مانع رسیدگی به پیام‌های انسانی شود.
- **پیام‌های پیش‌دستانه نمی‌رسند:** اگر کاربر اخیراً تعاملی نداشته باشد،
  QQ ممکن است پیام‌های آغازشده توسط ربات را مسدود کند.
- **صدا رونویسی نمی‌شود:** مطمئن شوید STT پیکربندی شده و ارائه‌دهنده
  در دسترس است.
