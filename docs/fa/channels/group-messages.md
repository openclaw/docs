هدف: اجازه دهید Clawd در گروه‌های WhatsApp حضور داشته باشد، فقط وقتی پینگ شد فعال شود، و آن رشته گفتگو را از نشست پیام مستقیم شخصی جدا نگه دارد.

<Note>
`agents.list[].groupChat.mentionPatterns` همچنین توسط Telegram، Discord، Slack و iMessage استفاده می‌شود. این سند بر رفتار ویژه WhatsApp تمرکز دارد. برای چیدمان‌های چندعاملی، `agents.list[].groupChat.mentionPatterns` را برای هر عامل تنظیم کنید، یا از `messages.groupChat.mentionPatterns` به‌عنوان جایگزین سراسری استفاده کنید.
</Note>

## پیاده‌سازی فعلی (2025-12-03)

- حالت‌های فعال‌سازی: `mention` (پیش‌فرض) یا `always`. `mention` به یک پینگ نیاز دارد (منشن‌های واقعی WhatsApp با `mentionedJids`، الگوهای regex امن، یا E.164 ربات در هر جای متن). `always` عامل را با هر پیام فعال می‌کند، اما عامل فقط وقتی باید پاسخ دهد که بتواند ارزش معناداری اضافه کند؛ در غیر این صورت، توکن سکوت دقیق `NO_REPLY` / `no_reply` را برمی‌گرداند. پیش‌فرض‌ها را می‌توان در پیکربندی (`channels.whatsapp.groups`) تنظیم کرد و برای هر گروه از طریق `/activation` بازنویسی کرد. وقتی `channels.whatsapp.groups` تنظیم شده باشد، به‌عنوان فهرست مجاز گروه‌ها نیز عمل می‌کند (برای مجاز کردن همه، `"*"` را اضافه کنید).
- سیاست گروه: `channels.whatsapp.groupPolicy` کنترل می‌کند که آیا پیام‌های گروه پذیرفته شوند یا نه (`open|disabled|allowlist`). `allowlist` از `channels.whatsapp.groupAllowFrom` استفاده می‌کند (جایگزین: `channels.whatsapp.allowFrom` صریح). پیش‌فرض `allowlist` است (تا زمانی که فرستنده‌ها را اضافه نکنید، مسدود است).
- نشست‌های جداگانه برای هر گروه: کلیدهای نشست مانند `agent:<agentId>:whatsapp:group:<jid>` هستند، بنابراین دستورهایی مانند `/verbose on`، `/trace on`، یا `/think high` (که به‌صورت پیام‌های مستقل فرستاده می‌شوند) محدود به همان گروه هستند؛ وضعیت پیام مستقیم شخصی دست‌نخورده می‌ماند. Heartbeatها برای رشته گفتگوهای گروهی نادیده گرفته می‌شوند.
- تزریق زمینه: پیام‌های گروهی **فقط در انتظار** (پیش‌فرض 50) که باعث اجرای عامل _نشده‌اند_، زیر `[Chat messages since your last reply - for context]` قرار می‌گیرند و خط آغازگر زیر `[Current message - respond to this]` می‌آید. پیام‌هایی که از قبل در نشست هستند دوباره تزریق نمی‌شوند.
- نمایش فرستنده: هر دسته پیام گروهی اکنون با `[from: Sender Name (+E164)]` پایان می‌یابد تا Pi بداند چه کسی صحبت می‌کند.
- پیام‌های موقت/یک‌بارمشاهده: پیش از استخراج متن/منشن‌ها آن‌ها را باز می‌کنیم، بنابراین پینگ‌های داخل آن‌ها همچنان فعال‌سازی را انجام می‌دهند.
- پرامپت سیستمی گروه: در نوبت اول یک نشست گروهی (و هر زمان که `/activation` حالت را تغییر دهد)، یک متن کوتاه مانند این را به پرامپت سیستمی تزریق می‌کنیم: `You are replying inside the WhatsApp group "<subject>". Group members: Alice (+44...), Bob (+43...), … Activation: trigger-only … Address the specific sender noted in the message context.` اگر فراداده در دسترس نباشد، همچنان به عامل می‌گوییم که این یک گفتگوی گروهی است.

## نمونه پیکربندی (WhatsApp)

یک بلوک `groupChat` به `~/.openclaw/openclaw.json` اضافه کنید تا پینگ‌های نام نمایشی حتی وقتی WhatsApp نشانه بصری `@` را از بدنه متن حذف می‌کند کار کنند:

```json5
{
  channels: {
    whatsapp: {
      groups: {
        "*": { requireMention: true },
      },
    },
  },
  agents: {
    list: [
      {
        id: "main",
        groupChat: {
          historyLimit: 50,
          mentionPatterns: ["@?openclaw", "\\+?15555550123"],
        },
      },
    ],
  },
}
```

نکته‌ها:

- regexها به حروف کوچک و بزرگ حساس نیستند و از همان محافظ‌های safe-regex استفاده می‌کنند که در سایر سطوح regex پیکربندی وجود دارد؛ الگوهای نامعتبر و تکرارهای تودرتوی ناامن نادیده گرفته می‌شوند.
- WhatsApp همچنان وقتی کسی روی مخاطب می‌زند، منشن‌های استاندارد را از طریق `mentionedJids` می‌فرستد، بنابراین جایگزین شماره به‌ندرت لازم می‌شود، اما یک شبکه اطمینان مفید است.

### دستور فعال‌سازی (فقط مالک)

از دستور گفتگوی گروهی استفاده کنید:

- `/activation mention`
- `/activation always`

فقط شماره مالک (از `channels.whatsapp.allowFrom`، یا E.164 خود ربات وقتی تنظیم نشده باشد) می‌تواند این را تغییر دهد. برای دیدن حالت فعال‌سازی فعلی، `/status` را به‌صورت پیام مستقل در گروه بفرستید.

## نحوه استفاده

1. حساب WhatsApp خود را (همان حسابی که OpenClaw را اجرا می‌کند) به گروه اضافه کنید.
2. بگویید `@openclaw …` (یا شماره را اضافه کنید). فقط فرستنده‌های فهرست مجاز می‌توانند آن را فعال کنند، مگر اینکه `groupPolicy: "open"` را تنظیم کنید.
3. پرامپت عامل شامل زمینه اخیر گروه به‌همراه نشانگر پایانی `[from: …]` خواهد بود تا بتواند فرد درست را خطاب قرار دهد.
4. دستورالعمل‌های سطح نشست (`/verbose on`، `/trace on`، `/think high`، `/new` یا `/reset`، `/compact`) فقط روی نشست همان گروه اعمال می‌شوند؛ آن‌ها را به‌صورت پیام‌های مستقل بفرستید تا ثبت شوند. نشست پیام مستقیم شخصی شما مستقل می‌ماند.

## آزمایش / تأیید

- آزمون دستی سریع:
  - یک پینگ `@openclaw` در گروه بفرستید و تأیید کنید پاسخی می‌آید که به نام فرستنده اشاره می‌کند.
  - یک پینگ دوم بفرستید و بررسی کنید بلوک تاریخچه گنجانده شده و سپس در نوبت بعدی پاک می‌شود.
- لاگ‌های Gateway را بررسی کنید (با `--verbose` اجرا کنید) تا ورودی‌های `inbound web message` را ببینید که `from: <groupJid>` و پسوند `[from: …]` را نشان می‌دهند.

## ملاحظات شناخته‌شده

- Heartbeatها عمداً برای گروه‌ها نادیده گرفته می‌شوند تا از پخش‌های پرسر‌وصدا جلوگیری شود.
- جلوگیری از اکو از رشته ترکیبی دسته پیام استفاده می‌کند؛ اگر متن یکسانی را دو بار بدون منشن بفرستید، فقط اولی پاسخ می‌گیرد.
- ورودی‌های محل ذخیره نشست به‌صورت `agent:<agentId>:whatsapp:group:<jid>` در محل ذخیره نشست ظاهر می‌شوند (`~/.openclaw/agents/<agentId>/sessions/sessions.json` به‌صورت پیش‌فرض)؛ نبودن یک ورودی فقط یعنی آن گروه هنوز اجرای عامل را فعال نکرده است.
- نشانگرهای تایپ در گروه‌ها از `agents.defaults.typingMode` پیروی می‌کنند. وقتی پاسخ‌های قابل مشاهده از حالت پیش‌فرض فقط-ابزار-پیام استفاده می‌کنند، تایپ به‌صورت پیش‌فرض بلافاصله شروع می‌شود تا اعضای گروه ببینند عامل در حال کار است، حتی اگر هیچ پاسخ نهایی خودکاری ارسال نشود. پیکربندی صریح حالت تایپ همچنان اولویت دارد.
