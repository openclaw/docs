Tlon یک پیام‌رسان غیرمتمرکز است که بر پایه Urbit ساخته شده است. OpenClaw به کشتی Urbit شما وصل می‌شود و می‌تواند
به پیام‌های مستقیم و پیام‌های گفت‌وگوی گروهی پاسخ دهد. پاسخ‌های گروهی به‌طور پیش‌فرض به اشاره با @ نیاز دارند و می‌توانند
از طریق فهرست‌های مجاز محدودتر شوند.

وضعیت: Plugin همراه. پیام‌های مستقیم، اشاره‌های گروهی، پاسخ‌های رشته‌ای، قالب‌بندی متن غنی، و
بارگذاری تصویر پشتیبانی می‌شوند. واکنش‌ها و نظرسنجی‌ها هنوز پشتیبانی نمی‌شوند.

## Plugin همراه

Tlon در نسخه‌های فعلی OpenClaw به‌عنوان یک Plugin همراه ارائه می‌شود، بنابراین بیلدهای بسته‌بندی‌شده معمولی
به نصب جداگانه نیاز ندارند.

اگر از یک بیلد قدیمی‌تر یا نصب سفارشی استفاده می‌کنید که Tlon را شامل نمی‌شود، وقتی بسته npm
فعلی منتشر شد آن را نصب کنید:

نصب از طریق CLI (رجیستری npm، وقتی بسته فعلی وجود داشته باشد):

```bash
openclaw plugins install @openclaw/tlon
```

اگر npm گزارش داد که بسته متعلق به OpenClaw منسوخ شده است، تا زمانی که بسته npm جدیدتری
منتشر شود، از یک بیلد بسته‌بندی‌شده فعلی OpenClaw یا مسیر checkout محلی استفاده کنید.

checkout محلی (هنگام اجرا از یک مخزن git):

```bash
openclaw plugins install ./path/to/local/tlon-plugin
```

جزئیات: [Plugins](/fa/tools/plugin)

## راه‌اندازی

1. مطمئن شوید Plugin مربوط به Tlon در دسترس است.
   - نسخه‌های بسته‌بندی‌شده فعلی OpenClaw آن را از پیش همراه دارند.
   - نصب‌های قدیمی‌تر/سفارشی می‌توانند آن را به‌صورت دستی با فرمان‌های بالا اضافه کنند.
2. URL کشتی و کد ورود خود را گردآوری کنید.
3. `channels.tlon` را پیکربندی کنید.
4. Gateway را راه‌اندازی مجدد کنید.
5. به ربات پیام مستقیم بدهید یا در یک کانال گروهی به آن اشاره کنید.

پیکربندی حداقلی (یک حساب):

```json5
{
  channels: {
    tlon: {
      enabled: true,
      ship: "~sampel-palnet",
      url: "https://your-ship-host",
      code: "lidlut-tabwed-pillex-ridrup",
      ownerShip: "~your-main-ship", // recommended: your ship, always allowed
    },
  },
}
```

## کشتی‌های خصوصی/LAN

OpenClaw به‌طور پیش‌فرض برای محافظت در برابر SSRF نام‌های میزبان و بازه‌های IP خصوصی/داخلی را مسدود می‌کند.
اگر کشتی شما روی یک شبکه خصوصی اجرا می‌شود (localhost، IP شبکه LAN، یا نام میزبان داخلی)،
باید صریحا فعال‌سازی کنید:

```json5
{
  channels: {
    tlon: {
      url: "http://localhost:8080",
      allowPrivateNetwork: true,
    },
  },
}
```

این برای URLهایی مانند موارد زیر اعمال می‌شود:

- `http://localhost:8080`
- `http://192.168.x.x:8080`
- `http://my-ship.local:8080`

⚠️ این گزینه را فقط در صورتی فعال کنید که به شبکه محلی خود اعتماد دارید. این تنظیم محافظت‌های SSRF را
برای درخواست‌ها به URL کشتی شما غیرفعال می‌کند.

## کانال‌های گروهی

کشف خودکار به‌طور پیش‌فرض فعال است. همچنین می‌توانید کانال‌ها را به‌صورت دستی ثابت کنید:

```json5
{
  channels: {
    tlon: {
      groupChannels: ["chat/~host-ship/general", "chat/~host-ship/support"],
    },
  },
}
```

غیرفعال کردن کشف خودکار:

```json5
{
  channels: {
    tlon: {
      autoDiscoverChannels: false,
    },
  },
}
```

## کنترل دسترسی

فهرست مجاز پیام مستقیم (خالی = هیچ پیام مستقیمی مجاز نیست، برای جریان تأیید از `ownerShip` استفاده کنید):

```json5
{
  channels: {
    tlon: {
      dmAllowlist: ["~zod", "~nec"],
    },
  },
}
```

مجوزدهی گروهی (به‌طور پیش‌فرض محدود):

```json5
{
  channels: {
    tlon: {
      defaultAuthorizedShips: ["~zod"],
      authorization: {
        channelRules: {
          "chat/~host-ship/general": {
            mode: "restricted",
            allowedShips: ["~zod", "~nec"],
          },
          "chat/~host-ship/announcements": {
            mode: "open",
          },
        },
      },
    },
  },
}
```

## مالک و سامانه تأیید

یک کشتی مالک تنظیم کنید تا وقتی کاربران غیرمجاز تلاش می‌کنند تعامل کنند، درخواست‌های تأیید را دریافت کند:

```json5
{
  channels: {
    tlon: {
      ownerShip: "~your-main-ship",
    },
  },
}
```

کشتی مالک **به‌طور خودکار در همه‌جا مجاز است** — دعوت‌های پیام مستقیم به‌طور خودکار پذیرفته می‌شوند و
پیام‌های کانال همیشه مجاز هستند. لازم نیست مالک را به `dmAllowlist` یا
`defaultAuthorizedShips` اضافه کنید.

وقتی تنظیم شده باشد، مالک برای موارد زیر اعلان پیام مستقیم دریافت می‌کند:

- درخواست‌های پیام مستقیم از کشتی‌هایی که در فهرست مجاز نیستند
- اشاره‌ها در کانال‌هایی که مجوزدهی ندارند
- درخواست‌های دعوت گروهی

## تنظیمات پذیرش خودکار

پذیرش خودکار دعوت‌های پیام مستقیم (برای کشتی‌های موجود در `dmAllowlist`):

```json5
{
  channels: {
    tlon: {
      autoAcceptDmInvites: true,
    },
  },
}
```

پذیرش خودکار دعوت‌های گروهی:

```json5
{
  channels: {
    tlon: {
      autoAcceptGroupInvites: true,
    },
  },
}
```

## مقصدهای تحویل (CLI/Cron)

این‌ها را با `openclaw message send` یا تحویل Cron استفاده کنید:

- پیام مستقیم: `~sampel-palnet` یا `dm/~sampel-palnet`
- گروه: `chat/~host-ship/channel` یا `group:~host-ship/channel`

## مهارت همراه

Plugin مربوط به Tlon شامل یک مهارت همراه ([`@tloncorp/tlon-skill`](https://github.com/tloncorp/tlon-skill))
است که دسترسی CLI به عملیات Tlon را فراهم می‌کند:

- **مخاطبان**: دریافت/به‌روزرسانی نمایه‌ها، فهرست کردن مخاطبان
- **کانال‌ها**: فهرست کردن، ایجاد، ارسال پیام‌ها، دریافت تاریخچه
- **گروه‌ها**: فهرست کردن، ایجاد، مدیریت اعضا
- **پیام‌های مستقیم**: ارسال پیام‌ها، واکنش به پیام‌ها
- **واکنش‌ها**: افزودن/حذف واکنش‌های ایموجی به پست‌ها و پیام‌های مستقیم
- **تنظیمات**: مدیریت مجوزهای Plugin از طریق فرمان‌های slash

این مهارت وقتی Plugin نصب باشد به‌طور خودکار در دسترس است.

## قابلیت‌ها

| قابلیت | وضعیت |
| --------------- | --------------------------------------- |
| پیام‌های مستقیم | ✅ پشتیبانی می‌شود |
| گروه‌ها/کانال‌ها | ✅ پشتیبانی می‌شود (به‌طور پیش‌فرض با اشاره کنترل می‌شود) |
| رشته‌ها | ✅ پشتیبانی می‌شود (پاسخ‌های خودکار در رشته) |
| متن غنی | ✅ Markdown به قالب Tlon تبدیل می‌شود |
| تصاویر | ✅ در فضای ذخیره‌سازی Tlon بارگذاری می‌شوند |
| واکنش‌ها | ✅ از طریق [مهارت همراه](#bundled-skill) |
| نظرسنجی‌ها | ❌ هنوز پشتیبانی نمی‌شود |
| فرمان‌های بومی | ✅ پشتیبانی می‌شود (به‌طور پیش‌فرض فقط مالک) |

## عیب‌یابی

ابتدا این نردبان را اجرا کنید:

```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
```

خرابی‌های رایج:

- **پیام‌های مستقیم نادیده گرفته می‌شوند**: فرستنده در `dmAllowlist` نیست و هیچ `ownerShip` برای جریان تأیید پیکربندی نشده است.
- **پیام‌های گروهی نادیده گرفته می‌شوند**: کانال کشف نشده یا فرستنده مجاز نیست.
- **خطاهای اتصال**: بررسی کنید URL کشتی در دسترس باشد؛ برای کشتی‌های محلی `allowPrivateNetwork` را فعال کنید.
- **خطاهای احراز هویت**: بررسی کنید کد ورود فعلی باشد (کدها چرخش دارند).

## مرجع پیکربندی

پیکربندی کامل: [پیکربندی](/fa/gateway/configuration)

گزینه‌های ارائه‌دهنده:

- `channels.tlon.enabled`: فعال/غیرفعال کردن راه‌اندازی کانال.
- `channels.tlon.ship`: نام کشتی Urbit ربات (مثلا `~sampel-palnet`).
- `channels.tlon.url`: URL کشتی (مثلا `https://sampel-palnet.tlon.network`).
- `channels.tlon.code`: کد ورود کشتی.
- `channels.tlon.allowPrivateNetwork`: اجازه دادن به URLهای localhost/LAN (دور زدن SSRF).
- `channels.tlon.ownerShip`: کشتی مالک برای سامانه تأیید (همیشه مجاز).
- `channels.tlon.dmAllowlist`: کشتی‌هایی که اجازه پیام مستقیم دارند (خالی = هیچ‌کدام).
- `channels.tlon.autoAcceptDmInvites`: پذیرش خودکار پیام‌های مستقیم از کشتی‌های موجود در فهرست مجاز.
- `channels.tlon.autoAcceptGroupInvites`: پذیرش خودکار همه دعوت‌های گروهی.
- `channels.tlon.autoDiscoverChannels`: کشف خودکار کانال‌های گروهی (پیش‌فرض: true).
- `channels.tlon.groupChannels`: آشیانه‌های کانال که به‌صورت دستی ثابت شده‌اند.
- `channels.tlon.defaultAuthorizedShips`: کشتی‌های مجاز برای همه کانال‌ها.
- `channels.tlon.authorization.channelRules`: قواعد احراز مجوز برای هر کانال.
- `channels.tlon.showModelSignature`: افزودن نام مدل به پیام‌ها.

## یادداشت‌ها

- پاسخ‌های گروهی برای پاسخ دادن به یک اشاره نیاز دارند (مثلا `~your-bot-ship`).
- پاسخ‌های رشته‌ای: اگر پیام ورودی در یک رشته باشد، OpenClaw در همان رشته پاسخ می‌دهد.
- متن غنی: قالب‌بندی Markdown (ضخیم، ایتالیک، کد، سرصفحه‌ها، فهرست‌ها) به قالب بومی Tlon تبدیل می‌شود.
- تصاویر: URLها در فضای ذخیره‌سازی Tlon بارگذاری می‌شوند و به‌عنوان بلوک‌های تصویر جاسازی می‌شوند.
