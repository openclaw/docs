## نردبان فرمان‌ها

ابتدا این‌ها را به‌ترتیب اجرا کنید:

```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
openclaw channels status --probe
```

مبنای سالم:

- `Runtime: running`
- `Connectivity probe: ok`
- `Capability: read-only`, `write-capable`, یا `admin-capable`
- بررسی کانال نشان می‌دهد انتقال متصل است و، در مواردی که پشتیبانی می‌شود، `works` یا `audit ok`

## WhatsApp

### نشانه‌های خرابی WhatsApp

| نشانه | سریع‌ترین بررسی | رفع |
| ------------------------------- | --------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| متصل است اما پاسخی به DM نمی‌دهد | `openclaw pairing list whatsapp` | فرستنده را تأیید کنید یا سیاست/فهرست مجاز DM را تغییر دهید. |
| پیام‌های گروه نادیده گرفته می‌شوند | `requireMention` + الگوهای منشن در پیکربندی را بررسی کنید | ربات را منشن کنید یا سیاست منشن را برای آن گروه آسان‌تر کنید. |
| ورود با QR با 408 تمام می‌شود | env مربوط به `HTTPS_PROXY` / `HTTP_PROXY` در Gateway را بررسی کنید | یک پروکسی قابل دسترسی تنظیم کنید؛ از `NO_PROXY` فقط برای دورزدن‌ها استفاده کنید. |
| قطع اتصال/ورود دوباره تصادفی در حلقه | `openclaw channels status --probe` + لاگ‌ها | اتصال‌های دوباره اخیر حتی وقتی اکنون متصل هستند علامت‌گذاری می‌شوند؛ لاگ‌ها را دنبال کنید، Gateway را راه‌اندازی دوباره کنید، سپس اگر نوسان ادامه داشت دوباره پیوند دهید. |

عیب‌یابی کامل: [عیب‌یابی WhatsApp](/fa/channels/whatsapp#troubleshooting)

## Telegram

### نشانه‌های خرابی Telegram

| نشانه | سریع‌ترین بررسی | رفع |
| ------------------------------------ | ------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------- |
| `/start` اما جریان پاسخ قابل استفاده‌ای وجود ندارد | `openclaw pairing list telegram` | pairing را تأیید کنید یا سیاست DM را تغییر دهید. |
| ربات آنلاین است اما گروه ساکت می‌ماند | الزام منشن و حالت حریم خصوصی ربات را بررسی کنید | برای دیده‌شدن گروهی، حالت حریم خصوصی را غیرفعال کنید یا ربات را منشن کنید. |
| خطاهای ارسال با خطاهای شبکه | لاگ‌ها را برای خرابی‌های فراخوانی API در Telegram بررسی کنید | مسیریابی DNS/IPv6/proxy به `api.telegram.org` را درست کنید. |
| راه‌اندازی گزارش می‌دهد `getMe returned 401` | منبع توکن پیکربندی‌شده را بررسی کنید | توکن BotFather را دوباره کپی یا بازتولید کنید و `botToken`، `tokenFile`، یا `TELEGRAM_BOT_TOKEN` حساب پیش‌فرض را به‌روزرسانی کنید. |
| polling متوقف می‌شود یا به‌آرامی دوباره وصل می‌شود | `openclaw logs --follow` برای تشخیص‌های polling | ارتقا دهید؛ اگر راه‌اندازی‌های دوباره مثبت کاذب هستند، `pollingStallThresholdMs` را تنظیم کنید. توقف‌های پایدار همچنان به proxy/DNS/IPv6 اشاره دارند. |
| `setMyCommands` هنگام راه‌اندازی رد می‌شود | لاگ‌ها را برای `BOT_COMMANDS_TOO_MUCH` بررسی کنید | فرمان‌های Telegram مربوط به plugin/skill/سفارشی را کاهش دهید یا منوهای بومی را غیرفعال کنید. |
| ارتقا داده‌اید و فهرست مجاز شما را مسدود می‌کند | `openclaw security audit` و فهرست‌های مجاز پیکربندی | `openclaw doctor --fix` را اجرا کنید یا `@username` را با شناسه‌های عددی فرستنده جایگزین کنید. |

عیب‌یابی کامل: [عیب‌یابی Telegram](/fa/channels/telegram#troubleshooting)

## Discord

### نشانه‌های خرابی Discord

| نشانه | سریع‌ترین بررسی | رفع |
| ----------------------------------------- | ---------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ربات آنلاین است اما پاسخی در guild نمی‌دهد | `openclaw channels status --probe` | guild/channel را مجاز کنید و intent محتوای پیام را بررسی کنید. |
| پیام‌های گروه نادیده گرفته می‌شوند | لاگ‌ها را برای حذف‌های ناشی از gating منشن بررسی کنید | ربات را منشن کنید یا `requireMention: false` را برای guild/channel تنظیم کنید. |
| مصرف تایپ/توکن وجود دارد اما پیام Discord نیست | لاگ جلسه متن دستیار را با `didSendViaMessagingTool: false` نشان می‌دهد | مدل به‌جای فراخوانی ابزار پیام، خصوصی پاسخ داده است. از مدلی استفاده کنید که فراخوانی ابزار آن قابل اتکا باشد، یا `messages.groupChat.visibleReplies: "automatic"` را برای ارسال خودکار تنظیم کنید. |
| پاسخ‌های DM وجود ندارند | `openclaw pairing list discord` | pairing مربوط به DM را تأیید کنید یا سیاست DM را تنظیم کنید. |

عیب‌یابی کامل: [عیب‌یابی Discord](/fa/channels/discord#troubleshooting)
