## نردبان دستورها

ابتدا این‌ها را به‌ترتیب اجرا کنید:

```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
openclaw channels status --probe
```

خط پایه سالم:

- `Runtime: running`
- `Connectivity probe: ok`
- `Capability: read-only`، `write-capable`، یا `admin-capable`
- بررسی کانال نشان می‌دهد که انتقال وصل است و، در مواردی که پشتیبانی می‌شود، `works` یا `audit ok`

## WhatsApp

### نشانه‌های خرابی WhatsApp

| نشانه | سریع‌ترین بررسی | رفع مشکل |
| ------------------------------- | --------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| وصل است اما پاسخی به DMها داده نمی‌شود | `openclaw pairing list whatsapp` | فرستنده را تأیید کنید یا سیاست/فهرست مجاز DM را تغییر دهید. |
| پیام‌های گروه نادیده گرفته می‌شوند | `requireMention` و الگوهای اشاره را در پیکربندی بررسی کنید | ربات را منشن کنید یا سیاست منشن را برای آن گروه آسان‌تر کنید. |
| ورود با QR با 408 منقضی می‌شود | متغیرهای محیطی `HTTPS_PROXY` / `HTTP_PROXY` مربوط به Gateway را بررسی کنید | یک پراکسی در دسترس تنظیم کنید؛ از `NO_PROXY` فقط برای عبورها استفاده کنید. |
| قطع اتصال/ورود مجدد تصادفی تکرار می‌شود | `openclaw channels status --probe` + لاگ‌ها | اتصال‌های مجدد اخیر حتی وقتی اکنون وصل هستند علامت‌گذاری می‌شوند؛ لاگ‌ها را دنبال کنید، Gateway را بازراه‌اندازی کنید، سپس اگر نوسان ادامه داشت دوباره پیوند دهید. |

عیب‌یابی کامل: [عیب‌یابی WhatsApp](/fa/channels/whatsapp#troubleshooting)

## Telegram

### نشانه‌های خرابی Telegram

| نشانه | سریع‌ترین بررسی | رفع مشکل |
| ------------------------------------ | ------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------- |
| `/start` اما جریان پاسخ قابل استفاده‌ای نیست | `openclaw pairing list telegram` | جفت‌سازی را تأیید کنید یا سیاست DM را تغییر دهید. |
| ربات آنلاین است اما گروه ساکت می‌ماند | الزام منشن و حالت حریم خصوصی ربات را بررسی کنید | برای دیده‌شدن در گروه، حالت حریم خصوصی را غیرفعال کنید یا ربات را منشن کنید. |
| خرابی ارسال همراه با خطاهای شبکه | لاگ‌ها را برای خرابی‌های فراخوانی API مربوط به Telegram بررسی کنید | مسیریابی DNS/IPv6/پراکسی به `api.telegram.org` را اصلاح کنید. |
| راه‌اندازی گزارش می‌دهد `getMe returned 401` | منبع توکن پیکربندی‌شده را بررسی کنید | توکن BotFather را دوباره کپی یا بازتولید کنید و `botToken`، `tokenFile`، یا `TELEGRAM_BOT_TOKEN` حساب پیش‌فرض را به‌روزرسانی کنید. |
| Polling متوقف می‌شود یا کند دوباره وصل می‌شود | `openclaw logs --follow` برای عیب‌یابی polling | ارتقا دهید؛ اگر بازراه‌اندازی‌ها مثبت کاذب هستند، `pollingStallThresholdMs` را تنظیم کنید. توقف‌های پایدار همچنان به پراکسی/DNS/IPv6 اشاره دارند. |
| `setMyCommands` هنگام راه‌اندازی رد می‌شود | لاگ‌ها را برای `BOT_COMMANDS_TOO_MUCH` بررسی کنید | دستورهای Plugin/Skill/سفارشی Telegram را کاهش دهید یا منوهای بومی را غیرفعال کنید. |
| پس از ارتقا، فهرست مجاز شما را مسدود می‌کند | `openclaw security audit` و فهرست‌های مجاز پیکربندی | `openclaw doctor --fix` را اجرا کنید یا `@username` را با شناسه‌های عددی فرستنده جایگزین کنید. |

عیب‌یابی کامل: [عیب‌یابی Telegram](/fa/channels/telegram#troubleshooting)
