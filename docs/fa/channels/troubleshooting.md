وقتی یک کانال متصل می‌شود اما رفتارش نادرست است، از این صفحه استفاده کنید.

## ترتیب فرمان‌ها

ابتدا این‌ها را به‌ترتیب اجرا کنید:

```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
openclaw channels status --probe
```

وضعیت پایهٔ سالم:

- `Runtime: running`
- `Connectivity probe: ok`
- `Capability: read-only`، `write-capable` یا `admin-capable`
- کاوش کانال نشان می‌دهد که انتقال متصل است و در موارد پشتیبانی‌شده، `works` یا `audit ok` نمایش داده می‌شود

## پس از به‌روزرسانی

وقتی Telegram، iMessage، پیکربندی‌های دورهٔ BlueBubbles یا کانال Plugin دیگری پس از به‌روزرسانی ناپدید می‌شود، از این بخش استفاده کنید.

```bash
openclaw status --all
openclaw doctor --fix
openclaw gateway restart
openclaw status --all
```

در خروجی `openclaw status --all` به‌دنبال `plugin load failed: dependency tree corrupted; run openclaw doctor --fix` بگردید. این یعنی کانال پیکربندی شده است، اما راه‌اندازی یا بارگذاری Plugin به‌جای ثبت کانال با یک درخت وابستگی خراب مواجه شده است. `openclaw doctor --fix` پیوندهای نمادین قدیمی وابستگی‌های زمان اجرای Plugin و سایه‌های قدیمی احراز هویت را پاک می‌کند، سپس `openclaw gateway restart` وضعیت پاک را دوباره بارگذاری می‌کند.

## WhatsApp

### نشانه‌های خرابی WhatsApp

| نشانه | سریع‌ترین بررسی | راه‌حل |
| ----------------------------------- | --------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| متصل است اما به پیام‌های مستقیم پاسخ نمی‌دهد | `openclaw pairing list whatsapp` | فرستنده را تأیید کنید یا سیاست/فهرست مجاز پیام مستقیم را تغییر دهید. |
| پیام‌های گروه نادیده گرفته می‌شوند | `requireMention` و الگوهای اشاره را در پیکربندی بررسی کنید | به ربات اشاره کنید یا سیاست اشاره را برای آن گروه آسان‌گیرانه‌تر کنید. |
| ورود با کد QR با خطای 408 منقضی می‌شود | متغیرهای محیطی `HTTPS_PROXY` / `HTTP_PROXY` مربوط به Gateway را بررسی کنید | یک پراکسی قابل‌دسترسی تنظیم کنید؛ از `NO_PROXY` فقط برای دور زدن پراکسی استفاده کنید. |
| قطع اتصال/ورود مجدد به‌شکل تصادفی و حلقه‌ای رخ می‌دهد | `openclaw channels status --probe` و گزارش‌ها | اتصال‌های مجدد اخیر حتی هنگام اتصال فعلی نیز علامت‌گذاری می‌شوند؛ گزارش‌ها را زیر نظر بگیرید، Gateway را راه‌اندازی مجدد کنید و اگر ناپایداری ادامه یافت دوباره پیوند دهید. |
| حلقهٔ `status=408 Request Time-out` | کاوش، گزارش‌ها، doctor و سپس وضعیت Gateway | ابتدا اتصال/زمان‌بندی میزبان را اصلاح کنید؛ اگر حلقه ادامه داشت، از احراز هویت پشتیبان بگیرید و حساب را دوباره پیوند دهید. |
| پاسخ‌ها با چند ثانیه/دقیقه تأخیر می‌رسند | `openclaw doctor --fix` | Doctor کلاینت‌های محلی و قدیمی TUI را که فرسودگی آن‌ها تأیید شده و حلقهٔ رویداد Gateway را مختل می‌کنند، متوقف می‌کند. |

عیب‌یابی کامل: [عیب‌یابی WhatsApp](/fa/channels/whatsapp#troubleshooting)

## Telegram

### نشانه‌های خرابی Telegram

| نشانه | سریع‌ترین بررسی | راه‌حل |
| ------------------------------------ | ------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------- |
| `/start` اجرا می‌شود اما روند پاسخ‌دهی قابل‌استفاده‌ای وجود ندارد | `openclaw pairing list telegram` | جفت‌سازی را تأیید کنید یا سیاست پیام مستقیم را تغییر دهید. |
| ربات آنلاین است اما گروه ساکت می‌ماند | الزام اشاره و حالت حریم خصوصی ربات را بررسی کنید | برای مشاهده‌پذیری در گروه، حالت حریم خصوصی را غیرفعال کنید یا به ربات اشاره کنید. |
| ارسال با خطاهای شبکه ناموفق است | گزارش‌ها را برای خطاهای فراخوانی API مربوط به Telegram بررسی کنید | مسیریابی DNS/IPv6/پراکسی به `api.telegram.org` را اصلاح کنید. |
| هنگام راه‌اندازی `getMe returned 401` گزارش می‌شود | منبع توکن پیکربندی‌شده را بررسی کنید | توکن BotFather را دوباره کپی یا تولید کنید و `botToken`، `tokenFile` یا `TELEGRAM_BOT_TOKEN` حساب پیش‌فرض را به‌روزرسانی کنید. |
| نمونه‌برداری متوقف می‌شود یا اتصال مجدد کند است | برای تشخیص مشکلات نمونه‌برداری از `openclaw logs --follow` استفاده کنید | ارتقا دهید؛ اگر راه‌اندازی‌های مجدد تشخیص‌های مثبت کاذب هستند، `pollingStallThresholdMs` را تنظیم کنید. توقف‌های مداوم همچنان نشان‌دهندهٔ مشکل پراکسی/DNS/IPv6 هستند. |
| `setMyCommands` هنگام راه‌اندازی رد می‌شود | گزارش‌ها را برای `BOT_COMMANDS_TOO_MUCH` بررسی کنید | فرمان‌های Plugin، Skills یا سفارشی Telegram را کاهش دهید یا منوهای بومی را غیرفعال کنید. |
| پس از ارتقا فهرست مجاز دسترسی شما را مسدود می‌کند | `openclaw security audit` و فهرست‌های مجاز پیکربندی | `openclaw doctor --fix` را اجرا کنید یا `@username` را با شناسه‌های عددی فرستنده جایگزین کنید. |

عیب‌یابی کامل: [عیب‌یابی Telegram](/fa/channels/telegram#troubleshooting)

### نشانه‌های خرابی Discord

| نشانه | سریع‌ترین بررسی | راه‌حل |
| ----------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ربات آنلاین است اما در سرور پاسخ نمی‌دهد | `openclaw channels status --probe` | سرور/کانال را مجاز کنید و فعال بودن دسترسی به محتوای پیام را بررسی کنید. |
| پیام‌های گروه نادیده گرفته می‌شوند | گزارش‌ها را برای حذف پیام‌ها به‌دلیل دروازهٔ اشاره بررسی کنید | به ربات اشاره کنید یا `requireMention: false` را برای سرور/کانال تنظیم کنید. |

| پاسخ‌های پیام مستقیم وجود ندارند | `openclaw pairing list discord` | جفت‌سازی پیام مستقیم را تأیید کنید یا سیاست پیام مستقیم را تغییر دهید. |
