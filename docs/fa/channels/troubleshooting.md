## نردبان فرمان

ابتدا این‌ها را به‌ترتیب اجرا کنید:

```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
openclaw channels status --probe
```

خط مبنای سالم:

- `Runtime: running`
- `Connectivity probe: ok`
- `Capability: read-only`، `write-capable`، یا `admin-capable`
- بررسی کانال نشان می‌دهد ترابری وصل است و، در صورت پشتیبانی، `works` یا `audit ok`

## WhatsApp

### امضاهای خرابی WhatsApp

| نشانه                              | سریع‌ترین بررسی                                      | رفع                                                                                                                              |
| ---------------------------------- | ---------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| وصل است اما پاسخ DM نمی‌آید        | `openclaw pairing list whatsapp`                     | فرستنده را تأیید کنید یا سیاست/allowlist مربوط به DM را تغییر دهید.                                                            |
| پیام‌های گروه نادیده گرفته می‌شوند | `requireMention` و الگوهای اشاره را در پیکربندی بررسی کنید | ربات را منشن کنید یا سیاست منشن را برای آن گروه آسان‌تر کنید.                                                                   |
| ورود QR با 408 منقضی می‌شود        | متغیرهای محیطی `HTTPS_PROXY` / `HTTP_PROXY` مربوط به Gateway را بررسی کنید | یک پراکسی قابل دسترس تنظیم کنید؛ از `NO_PROXY` فقط برای دورزدن‌ها استفاده کنید.                                                |
| قطع اتصال/ورود دوباره تصادفی       | `openclaw channels status --probe` + لاگ‌ها           | اتصال‌های مجدد اخیر حتی وقتی اکنون وصل هستند علامت‌گذاری می‌شوند؛ لاگ‌ها را ببینید، Gateway را بازراه‌اندازی کنید، سپس اگر نوسان ادامه داشت دوباره لینک کنید. |
| پاسخ‌ها با چند ثانیه/دقیقه تأخیر می‌رسند | `openclaw doctor --fix`                              | Doctor کلاینت‌های محلی TUI مانده و تأییدشده را که حلقه رویداد Gateway را تضعیف می‌کنند متوقف می‌کند.                           |

عیب‌یابی کامل: [عیب‌یابی WhatsApp](/fa/channels/whatsapp#troubleshooting)

## Telegram

### امضاهای خرابی Telegram

| نشانه                              | سریع‌ترین بررسی                                    | رفع                                                                                                                        |
| ---------------------------------- | -------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| `/start` اما جریان پاسخ قابل استفاده نیست | `openclaw pairing list telegram`                   | جفت‌سازی را تأیید کنید یا سیاست DM را تغییر دهید.                                                                         |
| ربات آنلاین است اما گروه ساکت می‌ماند | الزام منشن و حالت حریم خصوصی ربات را تأیید کنید     | برای دیده‌شدن در گروه، حالت حریم خصوصی را غیرفعال کنید یا ربات را منشن کنید.                                            |
| شکست ارسال با خطاهای شبکه          | لاگ‌ها را برای شکست‌های فراخوانی API مربوط به Telegram بررسی کنید | مسیریابی DNS/IPv6/پراکسی به `api.telegram.org` را اصلاح کنید.                                                            |
| هنگام راه‌اندازی `getMe returned 401` گزارش می‌شود | منبع توکن پیکربندی‌شده را بررسی کنید               | توکن BotFather را دوباره کپی یا بازتولید کنید و `botToken`، `tokenFile`، یا `TELEGRAM_BOT_TOKEN` حساب پیش‌فرض را به‌روزرسانی کنید. |
| polling متوقف می‌شود یا دیر دوباره وصل می‌شود | `openclaw logs --follow` برای تشخیص‌های polling     | ارتقا دهید؛ اگر بازراه‌اندازی‌ها مثبت کاذب هستند، `pollingStallThresholdMs` را تنظیم کنید. توقف‌های پایدار همچنان به پراکسی/DNS/IPv6 اشاره دارند. |
| `setMyCommands` هنگام راه‌اندازی رد می‌شود | لاگ‌ها را برای `BOT_COMMANDS_TOO_MUCH` بررسی کنید  | فرمان‌های Plugin/skill/سفارشی Telegram را کاهش دهید یا منوهای بومی را غیرفعال کنید.                                      |
| ارتقا داده‌اید و allowlist شما را مسدود می‌کند | `openclaw security audit` و allowlistهای پیکربندی  | `openclaw doctor --fix` را اجرا کنید یا `@username` را با شناسه‌های عددی فرستنده جایگزین کنید.                            |

عیب‌یابی کامل: [عیب‌یابی Telegram](/fa/channels/telegram#troubleshooting)

| پاسخ‌های DM موجود نیستند                | `openclaw pairing list discord`                                          | جفت‌سازی DM را تأیید کنید یا سیاست DM را تنظیم کنید.                                                                                                                    |
