## نردبان دستورها

ابتدا این‌ها را به‌ترتیب اجرا کنید:

```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
openclaw channels status --probe
```

خط مبنای سالم:

- `Runtime: running`
- `Connectivity probe: ok`
- `Capability: read-only`، `write-capable` یا `admin-capable`
- پروب کانال نشان می‌دهد انتقال وصل است و، در مواردی که پشتیبانی می‌شود، `works` یا `audit ok`

## پس از به‌روزرسانی

وقتی Telegram، iMessage، پیکربندی‌های دوره BlueBubbles، یا کانال Plugin دیگری
پس از به‌روزرسانی ناپدید می‌شود، از این استفاده کنید.

```bash
openclaw status --all
openclaw doctor --fix
openclaw gateway restart
openclaw status --all
```

در `openclaw status --all` دنبال `plugin load failed: dependency tree corrupted; run openclaw doctor
--fix` بگردید. این یعنی کانال پیکربندی شده است، اما
مسیر راه‌اندازی/بارگذاری Plugin به‌جای ثبت کانال، به یک درخت وابستگی خراب
برخورده است. `openclaw doctor --fix` دایرکتوری‌های مرحله‌بندی وابستگی Plugin
قدیمی و سایه‌های احراز هویت قدیمی را حذف می‌کند، سپس `openclaw gateway restart` وضعیت
پاک را دوباره بارگذاری می‌کند.

## WhatsApp

### امضاهای خرابی WhatsApp

| نشانه | سریع‌ترین بررسی | رفع |
| ----------------------------------- | --------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| وصل است اما پاسخی به پیام مستقیم نمی‌دهد | `openclaw pairing list whatsapp` | فرستنده را تأیید کنید یا سیاست/فهرست مجاز پیام مستقیم را تغییر دهید. |
| پیام‌های گروه نادیده گرفته می‌شوند | `requireMention` و الگوهای منشن را در پیکربندی بررسی کنید | ربات را منشن کنید یا سیاست منشن را برای آن گروه آسان‌تر کنید. |
| ورود با QR با 408 مهلت‌اش تمام می‌شود | متغیرهای محیطی `HTTPS_PROXY` / `HTTP_PROXY` در Gateway را بررسی کنید | یک پراکسی قابل دسترس تنظیم کنید؛ از `NO_PROXY` فقط برای میان‌برها استفاده کنید. |
| قطع اتصال/ورود دوباره تصادفی تکرار می‌شود | `openclaw channels status --probe` + لاگ‌ها | اتصال‌های مجدد اخیر حتی وقتی اکنون وصل هستند علامت‌گذاری می‌شوند؛ لاگ‌ها را ببینید، Gateway را بازراه‌اندازی کنید، سپس اگر نوسان ادامه داشت دوباره پیوند دهید. |
| حلقه `status=408 Request Time-out` | پروب، لاگ‌ها، doctor، سپس وضعیت Gateway | ابتدا اتصال/زمان‌بندی میزبان را رفع کنید؛ اگر حلقه ادامه داشت از احراز هویت پشتیبان بگیرید و حساب را دوباره پیوند دهید. |
| پاسخ‌ها چند ثانیه/دقیقه دیر می‌رسند | `openclaw doctor --fix` | `doctor` کلاینت‌های محلی TUI قدیمی تأییدشده را وقتی حلقه رویداد Gateway را تخریب می‌کنند متوقف می‌کند. |

راهنمای کامل عیب‌یابی: [عیب‌یابی WhatsApp](/fa/channels/whatsapp#troubleshooting)

## Telegram

### امضاهای خرابی Telegram

| نشانه | سریع‌ترین بررسی | رفع |
| ------------------------------------ | ------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------- |
| `/start` اما بدون جریان پاسخ قابل استفاده | `openclaw pairing list telegram` | جفت‌سازی را تأیید کنید یا سیاست پیام مستقیم را تغییر دهید. |
| ربات آنلاین است اما گروه ساکت می‌ماند | الزام منشن و حالت حریم خصوصی ربات را بررسی کنید | برای دیده‌شدن در گروه، حالت حریم خصوصی را غیرفعال کنید یا ربات را منشن کنید. |
| خطاهای ارسال با خطاهای شبکه | لاگ‌ها را برای شکست‌های فراخوانی API Telegram بررسی کنید | مسیریابی DNS/IPv6/پراکسی به `api.telegram.org` را رفع کنید. |
| راه‌اندازی گزارش می‌دهد `getMe returned 401` | منبع توکن پیکربندی‌شده را بررسی کنید | توکن BotFather را دوباره کپی یا بازتولید کنید و `botToken`، `tokenFile`، یا `TELEGRAM_BOT_TOKEN` حساب پیش‌فرض را به‌روزرسانی کنید. |
| polling متوقف می‌شود یا اتصال مجدد کند است | `openclaw logs --follow` برای تشخیص‌های polling | ارتقا دهید؛ اگر بازراه‌اندازی‌ها مثبت کاذب هستند، `pollingStallThresholdMs` را تنظیم کنید. توقف‌های پایدار همچنان به پراکسی/DNS/IPv6 اشاره دارند. |
| `setMyCommands` هنگام راه‌اندازی رد می‌شود | لاگ‌ها را برای `BOT_COMMANDS_TOO_MUCH` بررسی کنید | فرمان‌های Telegram مربوط به Plugin/skill/سفارشی را کم کنید یا منوهای بومی را غیرفعال کنید. |
| ارتقا داده‌اید و فهرست مجاز شما را مسدود می‌کند | `openclaw security audit` و فهرست‌های مجاز پیکربندی | `openclaw doctor --fix` را اجرا کنید یا `@username` را با شناسه‌های عددی فرستنده جایگزین کنید. |

راهنمای کامل عیب‌یابی: [عیب‌یابی Telegram](/fa/channels/telegram#troubleshooting)

| ----------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| پاسخ‌های پیام مستقیم نمی‌رسند | `openclaw pairing list discord` | جفت‌سازی پیام مستقیم را تأیید کنید یا سیاست پیام مستقیم را تنظیم کنید. |
