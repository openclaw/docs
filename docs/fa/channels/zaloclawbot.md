OpenClaw از طریق Plugin خارجی `@zalo-platforms/openclaw-zaloclawbot` که در فهرست کاتالوگ قرار دارد، به Zalo ClawBot متصل می‌شود. ورود با کد QR یک Zalo Mini App انجام می‌شود؛ شناسهٔ Plugin در پیکربندی `openclaw-zaloclawbot` است.

## سازگاری

| نسخهٔ Plugin | نسخهٔ OpenClaw | برچسب توزیع npm | وضعیت       |
| ------------ | --------------- | --------------- | ------------ |
| 0.1.4        | >=2026.4.10     | `latest`        | فعال / بتا   |

## پیش‌نیازها

- Node.js >= 22
- نصب بودن [OpenClaw](https://docs.openclaw.ai/install) (در دسترس بودن CLI ‏`openclaw`)
- یک حساب Zalo روی دستگاه همراه برای اسکن کد QR ورود

## نصب با فرایند راه‌اندازی اولیه (پیشنهادی)

```bash
openclaw onboard
```

از منوی کانال، **Zalo ClawBot** را انتخاب کنید. راهنما Plugin را از کاتالوگ رسمی نصب می‌کند (با تأیید یکپارچگی)، کد QR ورود را در ترمینال نمایش می‌دهد و پس از اسکن آن با برنامهٔ Zalo، راه‌اندازی کانال را تکمیل می‌کند.

## نصب دستی

برای افزودن کانال به Gatewayی که قبلاً راه‌اندازی اولیه شده است:

### ۱. نصب Plugin

```bash
openclaw plugins install "@zalo-platforms/openclaw-zaloclawbot@0.1.4"
```

از نسخهٔ دقیق و ثابت‌شده استفاده کنید تا OpenClaw هنگام نصب، بسته را با هش یکپارچگی کاتالوگ تطبیق دهد.

### ۲. فعال‌سازی Plugin در پیکربندی

```bash
openclaw config set plugins.entries.openclaw-zaloclawbot.enabled true
```

### ۳. تولید کد QR و ورود

```bash
openclaw channels login --channel openclaw-zaloclawbot
```

کد QR نمایش‌داده‌شده در ترمینال را با برنامهٔ همراه Zalo اسکن کنید، شرایط استفاده را در Zalo Mini App بپذیرید و به نشست مجوز دهید.

### ۴. راه‌اندازی مجدد Gateway

```bash
openclaw gateway restart
```

## نحوهٔ کار

برخلاف کانال استاندارد Zalo که به ثبت Zalo Official Account ‏(OA) شخصی و پیکربندی اعتبارنامه‌های ثابت توسعه‌دهنده نیاز دارد، Zalo ClawBot یک **دستیار شخصی وابسته به مالک** روی زیرساخت رسمی مشترک است:

1. **راه‌اندازی اولیه:** کد QR به یک Zalo Mini App هدایت می‌شود که یک ربات خصوصی و تازه ایجادشده را زیر یک OA رسمی مشترک، مستقیماً به شناسهٔ کاربری Zalo شما متصل می‌کند.
2. **حریم خصوصی وابسته به مالک:** ربات فقط با مالک خود ارتباط برقرار می‌کند. پیام‌های کاربران دیگر در سطح پلتفرم کنار گذاشته می‌شوند.
3. **مسیر رسمی API:** این Plugin از APIهای Zalo Bot Platform استفاده می‌کند، نه خودکارسازی مرورگر یا نشست وب.

## جزئیات داخلی

Plugin از طریق یک حلقهٔ پایدار نظرسنجی طولانی (`getUpdates`) با Zalo ارتباط برقرار می‌کند. Webhookها به‌طور پیش‌فرض برای اجرای محلی Gateway در رایانهٔ رومیزی یا ترمینال غیرفعال هستند. پیام‌ها در سمت کارخواه پردازش و به محیط اجرای عامل محلی شما نگاشت می‌شوند.

Plugin اعتبارنامه‌های ربات را در پوشهٔ وضعیت OpenClaw مدیریت می‌کند. این پوشه را حساس در نظر بگیرید و همان خط‌مشی کنترل دسترسی و پشتیبان‌گیری سایر وضعیت‌های OpenClaw را برای آن نیز اعمال کنید.

محیط اجرای این Plugin کاملاً در بستهٔ خارجی `@zalo-platforms/openclaw-zaloclawbot` قرار دارد؛ جزئیات رفتاری زیر، فراتر از نصب و پیکربندی، بر اساس گزارش نگه‌دارندگان Plugin است و با کد منبع هستهٔ OpenClaw تأیید نشده است.

## عیب‌یابی

- **پایان مهلت ورود با QR:** توکن ورود (`zbsk`) برای حفظ امنیت پس از ۵ دقیقه منقضی می‌شود. اگر کد QR پیش از اسکن منقضی شد، فرمان ورود را دوباره اجرا کنید تا کد جدیدی تولید شود.
- **Gateway بارگذاری نمی‌شود:** تأیید کنید نسخهٔ میزبان OpenClaw شما `2026.4.10` یا بالاتر است. نسخه‌های قدیمی‌تر از دفتر ثبت نصب Plugin خارجی npm که این شناسه به آن نیاز دارد، پشتیبانی نمی‌کنند.
