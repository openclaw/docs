این سند جریان راه‌اندازی اولین اجرا در وضعیت **فعلی** را توضیح می‌دهد. هدف، تجربه‌ای روان در «روز ۰» است: انتخاب کنید Gateway کجا اجرا شود، احراز هویت را متصل کنید، ویزارد را اجرا کنید، و اجازه دهید عامل خودش را راه‌اندازی اولیه کند.
برای نمای کلی مسیرهای آماده‌سازی اولیه، [نمای کلی آماده‌سازی اولیه](/fa/start/onboarding-overview) را ببینید.

مدل اعتماد امنیتی:

- به‌طور پیش‌فرض، OpenClaw یک عامل شخصی است: یک مرز اپراتور مورد اعتماد.
- راه‌اندازی‌های اشتراکی/چندکاربره به قفل‌سازی نیاز دارند (مرزهای اعتماد را جدا کنید، دسترسی ابزارها را حداقلی نگه دارید، و از [امنیت](/fa/gateway/security) پیروی کنید).
- آماده‌سازی اولیه محلی اکنون پیکربندی‌های جدید را به‌طور پیش‌فرض روی `tools.profile: "coding"` می‌گذارد تا راه‌اندازی‌های محلی تازه، ابزارهای فایل‌سیستم/زمان اجرا را بدون اجبار به پروفایل نامحدود `full` حفظ کنند.
- اگر هوک‌ها/Webhookها یا دیگر خوراک‌های محتوای نامطمئن فعال هستند، از یک رده مدل مدرن و قوی استفاده کنید و سیاست ابزار/سندباکسینگ سخت‌گیرانه را حفظ کنید.

</Step>
<Step title="محلی در برابر راه‌دور">
<Frame>
<img src="/assets/macos-onboarding/04-choose-gateway.png" alt="" />
</Frame>

**Gateway** کجا اجرا می‌شود؟

- **این Mac (فقط محلی):** آماده‌سازی اولیه می‌تواند احراز هویت را پیکربندی کند و اعتبارنامه‌ها را به‌صورت محلی بنویسد.
- **راه‌دور (از طریق SSH/Tailnet):** آماده‌سازی اولیه احراز هویت محلی را پیکربندی **نمی‌کند**؛ اعتبارنامه‌ها باید روی میزبان Gateway وجود داشته باشند. فیلد توکن Gateway راه‌دور، توکنی را ذخیره می‌کند که برنامه macOS برای اتصال به آن Gateway استفاده می‌کند؛ مقادیر غیرمتن‌ساده موجود در `gateway.remote.token` تا زمانی که آن‌ها را جایگزین کنید حفظ می‌شوند.
- **بعداً پیکربندی شود:** راه‌اندازی را رد کنید و برنامه را بدون پیکربندی باقی بگذارید.

<Tip>
**نکته احراز هویت Gateway:**

- ویزارد اکنون حتی برای local loopback هم یک **توکن** تولید می‌کند، بنابراین کلاینت‌های WS محلی باید احراز هویت کنند.
- اگر احراز هویت را غیرفعال کنید، هر فرایند محلی می‌تواند متصل شود؛ این را فقط روی ماشین‌های کاملاً مورد اعتماد استفاده کنید.
- برای دسترسی چندماشینه یا bindهای غیر local loopback از یک **توکن** استفاده کنید.

</Tip>
</Step>

آماده‌سازی اولیه مجوزهای TCC مورد نیاز برای این موارد را درخواست می‌کند:

- خودکارسازی (AppleScript)
- اعلان‌ها
- دسترس‌پذیری
- ضبط صفحه
- میکروفون
- تشخیص گفتار
- دوربین
- موقعیت مکانی

</Step>
<Step title="CLI">
  <Info>این مرحله اختیاری است</Info>
  برنامه می‌تواند CLI سراسری `openclaw` را از طریق npm، pnpm، یا bun نصب کند.
  ابتدا npm را ترجیح می‌دهد، سپس pnpm، و سپس bun را اگر تنها مدیر بسته شناسایی‌شده باشد.
  برای زمان اجرای Gateway، Node همچنان مسیر توصیه‌شده است.
</Step>
<Step title="چت آماده‌سازی اولیه (نشست اختصاصی)">
  پس از راه‌اندازی، برنامه یک نشست چت اختصاصی برای آماده‌سازی اولیه باز می‌کند تا عامل بتواند
  خودش را معرفی کند و گام‌های بعدی را راهنمایی کند. این کار راهنمایی اولین اجرا را از
  گفت‌وگوی عادی شما جدا نگه می‌دارد. برای اینکه در میزبان Gateway طی اولین اجرای عامل چه رخ می‌دهد،
  [راه‌اندازی اولیه](/fa/start/bootstrapping) را ببینید.
</Step>
