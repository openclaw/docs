این سند جریان راه‌اندازی نخستین اجرای **فعلی** را شرح می‌دهد. هدف، تجربه‌ای روان برای «روز ۰» است: انتخاب محل اجرای Gateway، اتصال احراز هویت، اجرای راهنما، و اجازه دادن به عامل برای راه‌اندازی اولیه خودش.
برای نمای کلی مسیرهای ورود، [نمای کلی ورود](/fa/start/onboarding-overview) را ببینید.

مدل اعتماد امنیتی:

- به‌طور پیش‌فرض، OpenClaw یک عامل شخصی است: یک مرز اپراتور قابل اعتماد.
- راه‌اندازی‌های اشتراکی/چندکاربره به قفل‌سازی نیاز دارند (مرزهای اعتماد را جدا کنید، دسترسی ابزارها را حداقلی نگه دارید، و از [امنیت](/fa/gateway/security) پیروی کنید).
- ورود محلی اکنون به‌طور پیش‌فرض پیکربندی‌های جدید را روی `tools.profile: "coding"` تنظیم می‌کند تا راه‌اندازی‌های محلی تازه، ابزارهای فایل‌سیستم/زمان اجرا را بدون اجبار به پروفایل نامحدود `full` نگه دارند.
- اگر hooks/webhooks یا دیگر خوراک‌های محتوای غیرقابل اعتماد فعال هستند، از یک سطح مدل مدرن و قدرتمند استفاده کنید و سیاست ابزار/سندباکسینگ سخت‌گیرانه را حفظ کنید.

</Step>
<Step title="محلی در برابر راه‌دور">

**Gateway** کجا اجرا می‌شود؟

- **این Mac (فقط محلی):** ورود می‌تواند احراز هویت را پیکربندی کند و اعتبارنامه‌ها را به‌صورت محلی بنویسد.
- **راه‌دور (از طریق SSH/Tailnet):** ورود احراز هویت محلی را پیکربندی **نمی‌کند**؛ اعتبارنامه‌ها باید روی میزبان Gateway وجود داشته باشند.
- **بعداً پیکربندی شود:** راه‌اندازی را رد کنید و برنامه را بدون پیکربندی رها کنید.

<Tip>
**نکته احراز هویت Gateway:**

- راهنما اکنون حتی برای loopback نیز یک **توکن** تولید می‌کند، پس کلاینت‌های WS محلی باید احراز هویت شوند.
- اگر احراز هویت را غیرفعال کنید، هر فرایند محلی می‌تواند متصل شود؛ از این حالت فقط روی ماشین‌های کاملاً قابل اعتماد استفاده کنید.
- برای دسترسی چندماشینه یا bindهای غیر loopback از یک **توکن** استفاده کنید.

</Tip>
</Step>
<Step title="مجوزها">

ورود مجوزهای TCC موردنیاز برای موارد زیر را درخواست می‌کند:

- اتوماسیون (AppleScript)
- اعلان‌ها
- دسترسی‌پذیری
- ضبط صفحه
- میکروفون
- تشخیص گفتار
- دوربین
- موقعیت مکانی

</Step>
<Step title="CLI">
  <Info>این مرحله اختیاری است</Info>
  برنامه می‌تواند CLI سراسری `openclaw` را از طریق npm، pnpm یا bun نصب کند.
  ابتدا npm را ترجیح می‌دهد، سپس pnpm، و سپس bun را اگر تنها مدیر بسته شناسایی‌شده باشد.
  برای زمان اجرای Gateway، Node همچنان مسیر پیشنهادی است.
</Step>
<Step title="گفت‌وگوی ورود (نشست اختصاصی)">
  پس از راه‌اندازی، برنامه یک نشست گفت‌وگوی اختصاصی ورود باز می‌کند تا عامل بتواند
  خودش را معرفی کند و مراحل بعدی را راهنمایی کند. این کار راهنمایی نخستین اجرا را از
  گفت‌وگوی معمول شما جدا نگه می‌دارد. برای آنچه در میزبان Gateway هنگام نخستین اجرای عامل رخ می‌دهد،
  [راه‌اندازی اولیه](/fa/start/bootstrapping) را ببینید.
</Step>
