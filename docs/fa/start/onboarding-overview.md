OpenClaw دو مسیر راه‌اندازی اولیه دارد. هر دو احراز هویت، Gateway، و
کانال‌های چت اختیاری را پیکربندی می‌کنند؛ تفاوتشان فقط در نحوه تعامل شما با فرایند راه‌اندازی است.

## از کدام مسیر باید استفاده کنم؟

|                | راه‌اندازی اولیه CLI                         | راه‌اندازی اولیه اپلیکیشن macOS      |
| -------------- | -------------------------------------- | ------------------------- |
| **پلتفرم‌ها**  | macOS، Linux، Windows (بومی یا WSL2) | فقط macOS                |
| **واسط**  | راهنمای تعاملی ترمینال                        | رابط کاربری هدایت‌شده در اپلیکیشن      |
| **مناسب برای**   | سرورها، محیط‌های بدون رابط گرافیکی، کنترل کامل        | Mac دسکتاپ، راه‌اندازی دیداری |
| **خودکارسازی** | `--non-interactive` برای اسکریپت‌ها        | فقط دستی               |
| **فرمان**    | `openclaw onboard`                     | اجرای اپلیکیشن            |

بیشتر کاربران باید با **راه‌اندازی اولیه CLI** شروع کنند؛ این روش همه‌جا کار می‌کند و
بیشترین کنترل را به شما می‌دهد.

## راه‌اندازی اولیه چه چیزهایی را پیکربندی می‌کند

صرف‌نظر از اینکه کدام مسیر را انتخاب می‌کنید، راه‌اندازی اولیه این موارد را تنظیم می‌کند:

1. **ارائه‌دهنده مدل و احراز هویت** — کلید API، OAuth، یا توکن راه‌اندازی برای ارائه‌دهنده انتخابی شما
2. **فضای کاری** — دایرکتوری فایل‌های عامل، قالب‌های راه‌انداز، و حافظه
3. **Gateway** — پورت، نشانی bind، حالت احراز هویت
4. **کانال‌ها** (اختیاری) — کانال‌های چت داخلی و همراه مانند
   iMessage، Discord، Feishu، Google Chat، Mattermost، Microsoft Teams،
   Telegram، WhatsApp، و موارد دیگر
5. **دیمون** (اختیاری) — سرویس پس‌زمینه تا Gateway به‌صورت خودکار شروع شود

## راه‌اندازی اولیه CLI

در هر ترمینالی اجرا کنید:

```bash
openclaw onboard
```

برای نصب سرویس پس‌زمینه در همان مرحله، `--install-daemon` را اضافه کنید.

مرجع کامل: [راه‌اندازی اولیه (CLI)](/fa/start/wizard)
مستندات فرمان CLI: [`openclaw onboard`](/fa/cli/onboard)

## راه‌اندازی اولیه اپلیکیشن macOS

اپلیکیشن OpenClaw را باز کنید. راهنمای اجرای نخستین، شما را با یک رابط دیداری از همان مراحل عبور می‌دهد.

مرجع کامل: [راه‌اندازی اولیه (اپلیکیشن macOS)](/fa/start/onboarding)

## ارائه‌دهندگان سفارشی یا فهرست‌نشده

اگر ارائه‌دهنده شما در راه‌اندازی اولیه فهرست نشده است، **ارائه‌دهنده سفارشی** را انتخاب کنید و
این موارد را وارد کنید:

- حالت سازگاری API (سازگار با OpenAI، سازگار با Anthropic، یا تشخیص خودکار)
- URL پایه و کلید API
- شناسه مدل و نام مستعار اختیاری

چندین endpoint سفارشی می‌توانند هم‌زمان وجود داشته باشند؛ هرکدام شناسه endpoint خودش را می‌گیرد.
