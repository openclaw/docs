OpenClaw دارای فرایند راه‌اندازی اولیه در ترمینال و برنامه macOS است. هر دو ابتدا استنتاج را برقرار می‌کنند:
دسترسی موجود به هوش مصنوعی را شناسایی می‌کنند، یک تکمیل زنده را الزامی می‌دانند و تنها پس از آن
Crestodian را برای پیکربندی ادامه راه‌اندازی اجرا می‌کنند. اگر یک Gateway قابل دسترس و پیکربندی‌شده
وجود داشته باشد که عامل پیش‌فرض آن از قبل مدل پیکربندی‌شده‌ای دارد، فرایند راه‌اندازی اولیه رد می‌شود و
رابط کاربری عادی عامل باز می‌شود. جریان ترمینال برای راه‌اندازی دقیق، راهنمای گام‌به‌گام کلاسیک کامل را نیز
ارائه می‌دهد.

## از کدام مسیر باید استفاده کنم؟

|                | راه‌اندازی اولیه CLI                         | راه‌اندازی اولیه برنامه macOS             |
| -------------- | -------------------------------------- | -------------------------------- |
| **پلتفرم‌ها**  | macOS، Linux، Windows (بومی یا WSL2) | فقط macOS                       |
| **رابط**  | راه‌اندازی استنتاج، سپس Crestodian       | راه‌اندازی استنتاج، سپس Crestodian |
| **مناسب برای**   | سرورها، محیط‌های بدون رابط گرافیکی، کنترل کامل        | رایانه رومیزی Mac، راه‌اندازی بصری        |
| **خودکارسازی** | `--non-interactive` برای اسکریپت‌ها        | فقط دستی                      |
| **دستور**    | `openclaw onboard`                     | اجرای برنامه                   |

بیشتر کاربران باید با **راه‌اندازی اولیه CLI** شروع کنند — این روش همه‌جا کار می‌کند و
بیشترین کنترل را در اختیار شما می‌گذارد.

## مواردی که راه‌اندازی اولیه پیکربندی می‌کند

مرحله هدایت‌شده استنتاج فقط موارد زیر را برقرار می‌کند:

1. **ارائه‌دهنده مدل و احراز هویت** — دسترسی شناسایی‌شده یا کلید API تأییدشده
2. **استنتاج تأییدشده** — یک تکمیل واقعی با مدل مؤثر عامل پیش‌فرض

پس از موفقیت آن تکمیل، Crestodian می‌تواند فضای کاری، Gateway،
سرویس Gateway، کانال‌ها، عامل‌ها، Pluginها و سایر قابلیت‌های اختیاری را پیکربندی کند.

راهنمای گام‌به‌گام کلاسیک CLI می‌تواند موارد زیر را نیز پیکربندی کند:

1. **کانال‌ها** (اختیاری) — کانال‌های گفت‌وگوی داخلی و همراه، مانند
   Discord، Feishu، Google Chat، iMessage، Mattermost، Microsoft Teams،
   Telegram، WhatsApp و موارد دیگر
2. **کنترل‌های پیشرفته Gateway** — حالت راه دور، تنظیمات شبکه و گزینه‌های سرویس پس‌زمینه

## راه‌اندازی اولیه CLI

در هر ترمینالی اجرا کنید:

```bash
openclaw onboard
```

جریان هدایت‌شده دسترسی موجود به هوش مصنوعی را شناسایی می‌کند، گزینه‌ها را به‌ترتیب به‌صورت زنده
آزمایش می‌کند، در صورت شکست به گزینه بعدی می‌رود و امکان وارد کردن دستی و پوشیده کلید را ارائه می‌دهد. مدل
و اعتبارنامه را فقط پس از تکمیل موفق ذخیره می‌کند، سپس Crestodian را
برای پیکربندی فضای کاری، Gateway، کانال‌ها، عامل‌ها، Pluginها و سایر
قابلیت‌های اختیاری اجرا می‌کند. هیچ Crestodian پیش از استنتاج، مسیر رد کردن هوش مصنوعی یا
انتقال درون‌جریانی به حالت کلاسیک وجود ندارد. اگر راهنمای گام‌به‌گام کلاسیک را
می‌خواهید، خارج شوید و `openclaw onboard --classic` را اجرا کنید.

پس از موفقیت استنتاج، Crestodian می‌تواند راه‌اندازی کانال را به یک راهنمای گام‌به‌گام ترمینالی
با ورودی پوشیده واگذار کند. این فرایند راه‌اندازی هدایت‌شده یا کلاسیک ارائه‌دهنده را باز نمی‌کند؛ برای
تغییر ارائه‌دهنده مدل یا احراز هویت آن، از Crestodian خارج شوید و
`openclaw onboard` را اجرا کنید.

برای راه‌اندازی دقیق مدل/احراز هویت، کانال، مهارت، Gateway راه دور یا
واردسازی، از `openclaw onboard --classic` استفاده کنید. افزودن `--install-daemon` نیز
جریان کلاسیک را انتخاب می‌کند و سرویس پس‌زمینه را در یک مرحله نصب می‌کند. برای راه‌اندازی و تعمیر
مکالمه‌ایِ غیرمرتبط با استنتاج، از `openclaw crestodian` استفاده کنید. `openclaw
onboard --modern` یک نام مستعار سازگاری است که از همان دروازه استنتاج زنده
استفاده می‌کند.

## راه‌اندازی اولیه برنامه macOS

برنامه OpenClaw را باز کنید. اگر Gateway محلی یا راه دور پیکربندی‌شده آن قابل دسترس باشد
و عامل پیش‌فرض از قبل مدل پیکربندی‌شده‌ای داشته باشد، برنامه راه‌اندازی اولیه
و Crestodian را رد می‌کند و بلافاصله رابط کاربری عادی عامل را باز می‌کند.

برای یک Gateway جدید یا ناقص، جریان اجرای نخست دسترسی موجود به هوش مصنوعی
(Claude Code، Codex یا کلیدهای API) را شناسایی می‌کند، بهترین
گزینه را به‌صورت زنده آزمایش می‌کند و تنها پس از دریافت یک پاسخ واقعی آن را ذخیره می‌کند — در صورت نیاز به‌طور خودکار
به گزینه جایگزین می‌رود و وقتی چیزی یافت نشود، مرحله تأییدشده ورود دستی کلید API را
ارائه می‌دهد. اعتبارنامه‌های حساس با ورودی پوشیده دریافت می‌شوند. پس از موفقیت استنتاج، Crestodian اجرا می‌شود و
به پیکربندی سایر بخش‌ها کمک می‌کند.

Gemini CLI پس از راه‌اندازی همچنان برای عامل‌های عادی در دسترس است، اما برای این
دروازه استنتاج ارائه نمی‌شود، زیرا نمی‌تواند کاوش بدون ابزار را الزامی کند.

مرجع کامل: [راه‌اندازی اولیه (برنامه macOS)](/fa/start/onboarding)

## ارائه‌دهندگان سفارشی یا فهرست‌نشده

اگر ارائه‌دهنده شما در فهرست نیست، `openclaw onboard --classic` را اجرا کنید، گزینه
**ارائه‌دهنده سفارشی** را انتخاب کنید و موارد زیر را وارد کنید:

- سازگاری نقطه پایانی: سازگار با OpenAI (`/chat/completions`)، سازگار با OpenAI Responses (`/responses`)، سازگار با Anthropic (`/messages`) یا ناشناخته (هر سه را کاوش و به‌طور خودکار شناسایی می‌کند)
- نشانی پایه و کلید API (اگر نقطه پایانی به کلید نیاز نداشته باشد، کلید API اختیاری است)
- شناسه مدل و نام مستعار اختیاری مدل

چندین نقطه پایانی سفارشی می‌توانند هم‌زمان وجود داشته باشند — هرکدام شناسه نقطه پایانی مخصوص خود را دریافت می‌کنند.
