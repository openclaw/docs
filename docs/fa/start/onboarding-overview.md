OpenClaw دارای راه‌اندازی اولیه در ترمینال و برنامه macOS است. هر دو ابتدا استنتاج را برقرار می‌کنند:
دسترسی موجود به هوش مصنوعی را تشخیص می‌دهند، تکمیل زنده را الزامی می‌کنند و تنها پس از آن
OpenClaw را برای پیکربندی باقی مراحل راه‌اندازی اجرا می‌کنند. اگر یک Gateway قابل‌دسترسی و پیکربندی‌شده
وجود داشته باشد که عامل پیش‌فرض آن از قبل مدل پیکربندی‌شده‌ای دارد، راه‌اندازی اولیه رد می‌شود و
رابط کاربری عادی عامل باز می‌شود. جریان ترمینال همچنین برای
راه‌اندازی دقیق، راهنمای کلاسیک کامل را ارائه می‌دهد.

## باید از کدام مسیر استفاده کنم؟

|                | راه‌اندازی اولیه CLI                         | راه‌اندازی اولیه برنامه macOS           |
| -------------- | -------------------------------------- | ------------------------------ |
| **پلتفرم‌ها**  | macOS، Linux، Windows (بومی یا WSL2) | فقط macOS                     |
| **رابط**  | راه‌اندازی استنتاج، سپس OpenClaw         | راه‌اندازی استنتاج، سپس OpenClaw |
| **مناسب برای**   | سرورها، محیط‌های بدون رابط گرافیکی، کنترل کامل        | Mac رومیزی، راه‌اندازی بصری      |
| **خودکارسازی** | `--non-interactive` برای اسکریپت‌ها        | فقط دستی                    |
| **فرمان**    | `openclaw onboard`                     | اجرای برنامه                 |

بیشتر کاربران بهتر است با **راه‌اندازی اولیه CLI** شروع کنند — این روش همه‌جا کار می‌کند و
بیشترین کنترل را در اختیارتان می‌گذارد.

## مواردی که راه‌اندازی اولیه پیکربندی می‌کند

مرحله هدایت‌شده استنتاج فقط موارد زیر را برقرار می‌کند:

1. **ارائه‌دهنده مدل و احراز هویت** — دسترسی تشخیص‌داده‌شده یا ورود تأییدشده به ارائه‌دهنده،
   کلید API یا توکن
2. **استنتاج تأییدشده** — یک تکمیل واقعی روی مدل مؤثر
   عامل پیش‌فرض

پس از موفقیت آن تکمیل، OpenClaw می‌تواند فضای کاری، Gateway،
سرویس Gateway، کانال‌ها، عامل‌ها، Pluginها و دیگر قابلیت‌های اختیاری را پیکربندی کند.

راهنمای کلاسیک CLI همچنین می‌تواند موارد زیر را پیکربندی کند:

1. **کانال‌ها** (اختیاری) — کانال‌های گفت‌وگوی داخلی و همراه مانند
   Discord، Feishu، Google Chat، iMessage، Mattermost، Microsoft Teams،
   Telegram، WhatsApp و موارد دیگر
2. **کنترل‌های پیشرفته Gateway** — حالت راه‌دور، تنظیمات شبکه و گزینه‌های دیمن

## راه‌اندازی اولیه CLI

در هر ترمینالی اجرا کنید:

```bash
openclaw onboard
```

جریان هدایت‌شده دسترسی موجود به هوش مصنوعی را تشخیص می‌دهد، گزینه‌ها را به‌ترتیب به‌صورت زنده
آزمایش می‌کند و در صورت شکست به گزینه بعدی می‌رود. اگر تشخیص هیچ نتیجه‌ای نداشته باشد، ابتدا OpenAI،
Anthropic، xAI (Grok)، Google و OpenRouter را نمایش می‌دهد. **بیشتر…** شامل
سایر ارائه‌دهندگان در گروه‌های ارائه‌دهنده است و مناطق، طرح‌ها و روش‌های پشتیبانی‌شده
مرورگر، دستگاه، کلید API یا توکن را در منوی دوم نشان می‌دهد. مدل
و اعتبارنامه تنها پس از یک تکمیل موفق ذخیره می‌شوند؛ سپس OpenClaw برای
پیکربندی فضای کاری، Gateway، کانال‌ها، عامل‌ها، Pluginها و دیگر
قابلیت‌های اختیاری اجرا می‌شود. **فعلاً رد کردن** بدون اجرای OpenClaw خارج می‌شود. درون این
جریان، انتقالی به راهنمای کلاسیک وجود ندارد؛ اگر راهنمای کلاسیک را می‌خواهید، خارج شوید و
`openclaw onboard --classic` را اجرا کنید.

پس از موفقیت استنتاج، OpenClaw می‌تواند راه‌اندازی کانال را به یک راهنمای ترمینالی
با ورودی پوشیده واگذار کند. این راه‌اندازی هدایت‌شده یا کلاسیک ارائه‌دهنده را باز نمی‌کند؛ برای
تغییر ارائه‌دهنده مدل یا احراز هویت آن، از OpenClaw خارج شوید و
`openclaw onboard` را اجرا کنید.

برای راه‌اندازی دقیق مدل/احراز هویت، کانال، skill،
Gateway راه‌دور یا واردکردن، از `openclaw onboard --classic` استفاده کنید. افزودن `--install-daemon` همچنین
جریان کلاسیک را انتخاب می‌کند و سرویس پس‌زمینه را در یک مرحله نصب می‌کند. برای راه‌اندازی و تعمیر مکالمه‌ایِ غیرمرتبط با استنتاج، از `openclaw
openclaw` استفاده کنید. `openclaw
onboard --modern` یک نام مستعار سازگاری است که از همان
دروازه استنتاج زنده استفاده می‌کند.

## راه‌اندازی اولیه برنامه macOS

برنامه OpenClaw را باز کنید. اگر Gateway محلی یا راه‌دور پیکربندی‌شده آن قابل‌دسترسی باشد
و عامل پیش‌فرض از قبل مدل پیکربندی‌شده‌ای داشته باشد، برنامه راه‌اندازی اولیه
و OpenClaw را رد می‌کند و بلافاصله رابط کاربری عادی عامل را باز می‌کند.

برای یک Gateway تازه یا ناقص، جریان اجرای نخست دسترسی موجود به هوش مصنوعی
(Claude Code، Codex یا کلیدهای API) را تشخیص می‌دهد، بهترین
گزینه را به‌صورت زنده آزمایش می‌کند و فقط پس از دریافت پاسخی واقعی آن را ذخیره می‌کند — در صورت نیاز به‌طور خودکار
به گزینه جایگزین می‌رود و اگر چیزی پیدا نشود، مرحله دستی و تأییدشده کلید API را
ارائه می‌دهد. اعتبارنامه‌های حساس با ورودی پوشیده دریافت می‌شوند. پس از موفقیت استنتاج، OpenClaw اجرا می‌شود و
به پیکربندی سایر موارد کمک می‌کند.

Gemini CLI پس از راه‌اندازی برای عامل‌های عادی همچنان در دسترس است، اما برای
این دروازه استنتاج ارائه نمی‌شود، زیرا نمی‌تواند اجرای کاوش بدون ابزار را تضمین کند.

مرجع کامل: [راه‌اندازی اولیه (برنامه macOS)](/fa/start/onboarding)

## ارائه‌دهندگان سفارشی یا فهرست‌نشده

اگر ارائه‌دهنده شما در فهرست نیست، `openclaw onboard --classic` را اجرا کنید، گزینه
**ارائه‌دهنده سفارشی** را انتخاب کنید و موارد زیر را وارد کنید:

- سازگاری نقطه پایانی: سازگار با OpenAI ‏(`/chat/completions`)، سازگار با OpenAI Responses ‏(`/responses`)، سازگار با Anthropic ‏(`/messages`) یا ناشناخته (هر سه را کاوش و به‌طور خودکار تشخیص می‌دهد)
- نشانی پایه و کلید API (اگر نقطه پایانی به کلید نیاز نداشته باشد، کلید API اختیاری است)
- شناسه مدل و نام مستعار اختیاری مدل

چندین نقطه پایانی سفارشی می‌توانند هم‌زمان وجود داشته باشند — هرکدام شناسه نقطه پایانی مخصوص خود را دریافت می‌کنند.
