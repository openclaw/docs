OpenClaw دو مسیر راه‌اندازی اولیه دارد. هر دو احراز هویت، Gateway و
کانال‌های چت اختیاری را پیکربندی می‌کنند؛ تفاوتشان فقط در نحوهٔ تعامل شما با راه‌اندازی است.

## از کدام مسیر استفاده کنم؟

|                | راه‌اندازی اولیه با CLI                | راه‌اندازی اولیه با برنامهٔ macOS |
| -------------- | -------------------------------------- | ------------------------- |
| **سکوها**      | macOS، Linux، Windows (بومی یا WSL2)  | فقط macOS                 |
| **رابط**       | جادوگر ترمینال                         | رابط کاربری راهنمایی‌شده در برنامه |
| **بهترین برای** | سرورها، محیط‌های بدون رابط، کنترل کامل | Mac رومیزی، راه‌اندازی بصری |
| **خودکارسازی** | `--non-interactive` برای اسکریپت‌ها    | فقط دستی                  |
| **فرمان**      | `openclaw onboard`                     | اجرای برنامه              |

بیشتر کاربران باید با **راه‌اندازی اولیه با CLI** شروع کنند؛ همه‌جا کار می‌کند و
بیشترین کنترل را به شما می‌دهد.

## راه‌اندازی اولیه چه چیزهایی را پیکربندی می‌کند

صرف‌نظر از مسیری که انتخاب می‌کنید، راه‌اندازی اولیه این موارد را تنظیم می‌کند:

1. **ارائه‌دهندهٔ مدل و احراز هویت** — کلید API، OAuth، یا توکن راه‌اندازی برای ارائه‌دهندهٔ انتخابی شما
2. **فضای کاری** — پوشه‌ای برای فایل‌های عامل، قالب‌های بوت‌استرپ، و حافظه
3. **Gateway** — پورت، نشانی bind، حالت احراز هویت
4. **کانال‌ها** (اختیاری) — کانال‌های چت داخلی و همراه مانند
   BlueBubbles، Discord، Feishu، Google Chat، Mattermost، Microsoft Teams،
   Telegram، WhatsApp و موارد بیشتر
5. **دیمون** (اختیاری) — سرویس پس‌زمینه تا Gateway به‌صورت خودکار شروع شود

## راه‌اندازی اولیه با CLI

در هر ترمینالی اجرا کنید:

```bash
openclaw onboard
```

برای نصب سرویس پس‌زمینه در همان یک مرحله، `--install-daemon` را هم اضافه کنید.

مرجع کامل: [راه‌اندازی اولیه (CLI)](/fa/start/wizard)
مستندات فرمان CLI: [`openclaw onboard`](/fa/cli/onboard)

## راه‌اندازی اولیه با برنامهٔ macOS

برنامهٔ OpenClaw را باز کنید. جادوگر اجرای اول، شما را با یک رابط بصری از همان مراحل عبور می‌دهد.

مرجع کامل: [راه‌اندازی اولیه (برنامهٔ macOS)](/fa/start/onboarding)

## ارائه‌دهنده‌های سفارشی یا فهرست‌نشده

اگر ارائه‌دهندهٔ شما در راه‌اندازی اولیه فهرست نشده است، **ارائه‌دهندهٔ سفارشی** را انتخاب کنید و
وارد کنید:

- حالت سازگاری API (سازگار با OpenAI، سازگار با Anthropic، یا تشخیص خودکار)
- نشانی پایه و کلید API
- شناسهٔ مدل و نام مستعار اختیاری

چندین نقطهٔ پایانی سفارشی می‌توانند هم‌زمان وجود داشته باشند؛ هرکدام شناسهٔ نقطهٔ پایانی خود را دریافت می‌کند.
