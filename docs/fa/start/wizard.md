```bash
openclaw onboard
```

راه‌اندازی اولیه از طریق CLI، مسیر پیشنهادی برای تنظیم در ترمینال روی macOS، Linux و
Windows (به‌صورت بومی یا WSL2) است. این فرایند به‌طور پیش‌فرض دسترسی موجود به هوش مصنوعی روی
دستگاه را شناسایی می‌کند، آن را با یک تکمیل واقعی می‌آزماید و OpenClaw را برای
پیکربندی فضای کاری، Gateway و قابلیت‌های اختیاری راه‌اندازی می‌کند. `openclaw setup` همین فرایند را اجرا می‌کند ([راه‌اندازی](/fa/cli/setup)
گونهٔ صرفاً پیکربندیِ `--baseline` را پوشش می‌دهد). کاربران دسکتاپ Windows همچنین می‌توانند
از [Windows Hub](/fa/platforms/windows) شروع کنند.

راه‌اندازی هدایت‌شده ابتدا استنتاج را برقرار می‌کند. این فرایند دسترسی موجود به هوش مصنوعی را شناسایی می‌کند،
یک تکمیل واقعی را الزامی می‌داند و تنها پس از آن [OpenClaw](/cli/openclaw) را
برای پیکربندی سایر بخش‌های OpenClaw راه‌اندازی می‌کند. انتخاب **Skip for now** بدون
راه‌اندازی OpenClaw از فرایند راه‌اندازی اولیه خارج می‌شود.

ویزارد کلاسیک برای ارائه‌دهندگان سفارشی، تنظیم Gateway راه‌دور،
جفت‌سازی کانال، کنترل‌های سرویس پس‌زمینه، مهارت‌ها و درون‌ریزی‌ها همچنان در دسترس است. آن را صراحتاً
با `openclaw onboard --classic` اجرا کنید؛ انتخاب‌گر هدایت‌شدهٔ استنتاج، فرایند را
به آن واگذار نمی‌کند. پس از موفقیت استنتاج، OpenClaw می‌تواند با استفاده از `open channel wizard for
<channel>` تنظیم کانال‌هایی را که به اطلاعات محرمانه نیاز دارند به یک ویزارد ترمینالی با ورودی پوشانده‌شده بسپارد.
برای تغییر ارائه‌دهندهٔ مدل یا احراز هویت آن، از OpenClaw خارج شوید و
`openclaw onboard` را اجرا کنید؛ OpenClaw فرایندهای هدایت‌شده یا کلاسیک ارائه‌دهنده را باز نمی‌کند.

<Info>
سریع‌ترین راه برای نخستین گفت‌وگو: تنظیم هدایت‌شده را کامل کنید، `openclaw dashboard` را اجرا کنید و از طریق
رابط کنترل در مرورگر گفت‌وگو کنید. مستندات: [داشبورد](/fa/web/dashboard).
</Info>

## زبان و منطقه

ویزارد، متن‌های ثابت راه‌اندازی اولیه را بومی‌سازی می‌کند. ترتیب تشخیص: `OPENCLAW_LOCALE`،
`LC_ALL`، `LC_MESSAGES`، `LANG` و سپس انگلیسی. زبان‌ها و مناطق پشتیبانی‌شده: `en`،
`zh-CN`، `zh-TW`.

```bash
OPENCLAW_LOCALE=zh-CN openclaw onboard
```

نام محصولات، فرمان‌ها، کلیدهای پیکربندی، نشانی‌های اینترنتی، شناسه‌های ارائه‌دهنده، شناسه‌های مدل و
برچسب‌های افزونه/کانال، مستقل از زبان و منطقه به انگلیسی باقی می‌مانند.

برای پیکربندی مجدد تنظیمات غیراستنتاجی در آینده:

```bash
openclaw configure
openclaw agents add <name>
```

<Note>
`--json` به‌معنای حالت غیرتعاملی نیست. برای اسکریپت‌ها از `--non-interactive` استفاده کنید (به [خودکارسازی CLI](/fa/start/wizard-cli-automation) مراجعه کنید).
</Note>

<Tip>
ویزارد کلاسیک شامل مرحله‌ای برای جست‌وجوی وب است که در آن می‌توانید یک ارائه‌دهنده انتخاب کنید: Brave،
DuckDuckGo، Exa، Firecrawl، Gemini، Grok، Kimi، MiniMax Search، Ollama Web
Search، Perplexity، SearXNG یا Tavily. برخی به کلید API نیاز دارند و برخی
بدون کلید کار می‌کنند. بعداً این مورد را با `openclaw configure --section web` پیکربندی کنید. مستندات:
[ابزارهای وب](/fa/tools/web).
</Tip>

## حالت هدایت‌شدهٔ پیش‌فرض

اجرای سادهٔ `openclaw onboard` این مسیر را دنبال می‌کند:

1. اعلان امنیتی را بپذیرید.
2. مدل‌های پیکربندی‌شده، متغیرهای محیطی کلید API، CLIهای محلی پشتیبانی‌شدهٔ هوش مصنوعی
   و مدل‌های دارای قابلیت استفاده از ابزار را که از قبل روی سرورهای در دسترس Ollama یا LM
   Studio در میزبان Gateway نصب شده‌اند، شناسایی کنید. این بررسی فقط‌خواندنی هرگز مدلی را
   بارگیری نمی‌کند. نصب‌های Gemini CLI و Antigravity گزارش می‌شوند، اما به‌طور خودکار آزمایش
   نمی‌شوند، زیرا نمی‌توانند اجرای آزمایشی بدون ابزار را تضمین کنند.
3. نخستین گزینهٔ شناسایی‌شده را با یک تکمیل واقعی آزمایش کنید. در صورت شکست، دلیل را
   نمایش دهید و به گزینهٔ قابل‌استفادهٔ بعدی بروید.
4. اگر همهٔ گزینه‌های شناسایی‌شده بررسی شدند، OpenAI، Anthropic، xAI (Grok)، Google یا
   OpenRouter را انتخاب کنید، یا برای سایر ارائه‌دهندگان **More…** را برگزینید. منطقه‌ها،
   طرح‌ها و روش‌های پشتیبانی‌شدهٔ مرورگر، دستگاه، کلید API یا توکن هر ارائه‌دهنده
   در منوی دومی ظاهر می‌شوند و با همان تکمیل واقعی آزمایش می‌شوند.
   برای خروج بدون راه‌اندازی OpenClaw، **Skip for now** را انتخاب کنید.
5. فقط مسیر مدل تأییدشده و هرگونه وضعیت اعتبارنامه/افزونهٔ موردنیاز آن را
   ذخیره کنید. تنظیمات فضای کاری و Gateway بدون تغییر باقی می‌مانند.
6. OpenClaw را با مدل تأییدشده راه‌اندازی کنید تا بتواند فضای کاری،
   Gateway، کانال‌ها، عامل‌ها، افزونه‌ها و سایر تنظیمات اختیاری را پیکربندی کند.

اجرای مجدد فرمان روی یک نصب پیکربندی‌شده، ابتدا مدل پیش‌فرض فعلی را
آزمایش می‌کند و فرایند هدایت‌شده را به یک مرحلهٔ بررسی و تعمیر تبدیل می‌کند. شکست
بررسی هرگز مدل پیکربندی‌شده را به‌طور خودکار جایگزین نمی‌کند؛ راه‌اندازی اولیه متوقف می‌شود و
دربارهٔ نحوهٔ ادامه پرسش می‌کند. برای افزودن موارد غیراستنتاجی در آینده، `openclaw channels add` یا `openclaw configure` را اجرا کنید؛
برای تغییر مسیر ارائه‌دهنده یا احراز هویت از `openclaw onboard` استفاده کنید.

## ویزارد کلاسیک: QuickStart در برابر Advanced

برای باز کردن ویزارد کامل، `openclaw onboard --classic` را اجرا کنید. این ویزارد با
انتخاب میان **QuickStart** (پیش‌فرض‌ها) و **Advanced** (کنترل کامل) آغاز می‌شود. برای انتخاب
فرایند کلاسیک و رد شدن از آن پرسش، `--flow quickstart` یا `--flow advanced` (نام مستعار `manual`) را وارد کنید.

<Tabs>
  <Tab title="QuickStart (پیش‌فرض‌ها)">
    - Gateway محلی، اتصال loopback
    - فضای کاری پیش‌فرض (یا فضای کاری موجود)
    - درگاه Gateway برابر با **18789**
    - احراز هویت Gateway با **Token** (حتی روی loopback به‌طور خودکار تولید می‌شود)
    - سیاست ابزار: `tools.profile: "coding"` برای تنظیمات جدید (نمایهٔ صریح موجود حفظ می‌شود)
    - جداسازی پیام مستقیم: `session.dmScope: "per-channel-peer"` برای تنظیمات جدید. جزئیات: [مرجع تنظیم CLI](/fa/start/wizard-cli-reference#outputs-and-internals)
    - دسترسی از طریق Tailscale برابر با **Off**
    - پیام‌های مستقیم Telegram و WhatsApp به‌طور پیش‌فرض روی **allowlist** هستند: Telegram یک شناسهٔ عددی کاربر Telegram و WhatsApp یک شماره‌تلفن درخواست می‌کند

  </Tab>
  <Tab title="Advanced (کنترل کامل)">
    - همهٔ مراحل را در دسترس قرار می‌دهد: حالت، فضای کاری، Gateway، کانال‌ها، سرویس پس‌زمینه، مهارت‌ها

  </Tab>
</Tabs>

حالت راه‌دور (`--mode remote`) همیشه از فرایند پیشرفته استفاده می‌کند؛ این حالت فقط
این دستگاه را برای اتصال به یک Gateway در مکانی دیگر پیکربندی می‌کند و هرگز چیزی را
روی میزبان راه‌دور نصب یا تغییر نمی‌دهد.

## مواردی که راه‌اندازی اولیهٔ کلاسیک پیکربندی می‌کند

حالت محلی (پیش‌فرض) این مراحل را طی می‌کند:

1. **مدل/احراز هویت** - یک فرایند احراز هویت ارائه‌دهنده را انتخاب کنید (کلید API، OAuth یا
   احراز هویت دستی مخصوص ارائه‌دهنده)، از جمله ارائه‌دهندهٔ سفارشی
   (سازگار با OpenAI، سازگار با OpenAI Responses، سازگار با Anthropic یا
   تشخیص خودکار Unknown). یک مدل پیش‌فرض انتخاب کنید.
   تنظیم جدید کلید API ‏OpenAI به‌طور پیش‌فرض از `openai/gpt-5.6` استفاده می‌کند (شناسهٔ سادهٔ API مستقیم
   به Sol نگاشت می‌شود)؛ تنظیم جدید ChatGPT/Codex به‌طور پیش‌فرض از
   `openai/gpt-5.6-sol` استفاده می‌کند. اجرای مجدد تنظیم، مدل صریح موجود را حفظ می‌کند،
   از جمله `openai/gpt-5.5`. اگر حساب به GPT-5.6 دسترسی ندارد،
   `openai/gpt-5.5` را صراحتاً انتخاب کنید.
   نکتهٔ امنیتی: اگر این عامل ابزارها را اجرا می‌کند یا محتوای webhook/hook را
   پردازش می‌کند، قوی‌ترین مدل نسل جدید موجود را ترجیح دهید و سیاست
   ابزار را سخت‌گیرانه نگه دارید؛ سطوح ضعیف‌تر یا قدیمی‌تر در برابر تزریق پرامپت آسیب‌پذیرترند.
   برای اجراهای غیرتعاملی، `--secret-input-mode ref` ارجاع‌های مبتنی بر محیط را
   به‌جای مقادیر متنی سادهٔ کلید API ذخیره می‌کند؛ متغیر محیطی مورد ارجاع باید از قبل
   تنظیم شده باشد، وگرنه راه‌اندازی اولیه بی‌درنگ شکست می‌خورد. حالت تعاملی ارجاع اطلاعات محرمانه می‌تواند
   به یک متغیر محیطی یا ارجاع پیکربندی‌شدهٔ ارائه‌دهنده (`file` یا
   `exec`) اشاره کند و پیش از ذخیره‌سازی، بررسی مقدماتی سریعی انجام می‌شود. پس از تنظیم مدل/احراز هویت،
   ویزارد یک آزمایش اختیاری تکمیل زنده ارائه می‌دهد؛ در صورت شکست می‌توان یک‌بار به
   تنظیم مدل/احراز هویت بازگشت یا بدون مسدود کردن ادامهٔ
   ویزارد کلاسیک، شکست را نادیده گرفت. نادیده گرفتن آن OpenClaw را فعال نمی‌کند؛ تنظیم
   مکالمه همچنان به بررسی موفق استنتاج نیاز دارد.
2. **فضای کاری** - پوشهٔ فایل‌های عامل (پیش‌فرض `~/.openclaw/workspace`). فایل‌های راه‌انداز اولیه را ایجاد می‌کند.
3. **Gateway** - درگاه، نشانی اتصال، حالت احراز هویت، دسترسی Tailscale. در
   حالت تعاملی توکن، ذخیره‌سازی متنی سادهٔ توکن (پیش‌فرض) را انتخاب کنید یا
   استفاده از SecretRef را برگزینید. مسیر غیرتعاملی SecretRef: ‏`--gateway-token-ref-env <ENV_VAR>`.
4. **کانال‌ها** - کانال‌های گفت‌وگوی داخلی و افزونه‌های رسمی، از جمله
   Discord، Feishu، Google Chat، iMessage، Mattermost، Microsoft Teams،
   QQ Bot، Signal، Slack، Telegram، WhatsApp و موارد دیگر.
5. **سرویس پس‌زمینه** - یک LaunchAgent در macOS، یک واحد کاربری systemd
   در Linux/WSL2 یا یک Windows Scheduled Task بومی با مسیر جایگزین پوشهٔ
   Startup برای هر کاربر نصب می‌کند.
   اگر احراز هویت توکنی لازم باشد و `gateway.auth.token` توسط SecretRef مدیریت شود،
   نصب سرویس پس‌زمینه آن را اعتبارسنجی می‌کند، اما توکن حل‌شده را در
   فرادادهٔ محیط سرویس ناظر ذخیره نمی‌کند؛ SecretRef حل‌نشده با ارائهٔ
   راهنمایی، نصب را مسدود می‌کند. اگر `gateway.auth.token` و
   `gateway.auth.password` هر دو تنظیم شده باشند و `gateway.auth.mode` تنظیم نشده باشد، نصب
   تا زمانی که حالت را صراحتاً تعیین کنید مسدود می‌شود.
6. **بررسی سلامت** - Gateway را راه‌اندازی می‌کند و دردسترس‌بودن آن را می‌آزماید.
7. **مهارت‌ها** - مهارت‌های پیشنهادی و وابستگی‌های اختیاری آن‌ها را نصب می‌کند.

<Note>
اجرای مجدد راه‌اندازی اولیه هیچ‌چیز را پاک **نمی‌کند**، مگر اینکه صراحتاً
**Reset** را انتخاب کنید (یا `--reset` را وارد کنید). فرمان CLI ‏`--reset` به‌طور پیش‌فرض پیکربندی، اعتبارنامه‌ها
و نشست‌ها را حذف می‌کند؛ برای حذف فضای کاری نیز از `--reset-scope full` استفاده کنید. اگر
پیکربندی نامعتبر باشد یا کلیدهای قدیمی داشته باشد، راه‌اندازی اولیه از شما می‌خواهد ابتدا
`openclaw doctor` را اجرا کنید.
</Note>

`--flow import` به‌جای تنظیم جدید، یک فرایند مهاجرت شناسایی‌شده (برای نمونه Hermes) را در
ویزارد کلاسیک اجرا می‌کند؛ به [مهاجرت](/fa/cli/migrate) و راهنماهای مهاجرت در بخش
[نصب](/fa/install/migrating-hermes) مراجعه کنید. `openclaw onboard --modern` یک
نام مستعار سازگاری برای [OpenClaw](/cli/openclaw) است. این فرمان از همان
دروازهٔ استنتاج `openclaw setup` استفاده می‌کند: استنتاج تأییدشده
دستیار را راه‌اندازی می‌کند، درحالی‌که شکست تعاملی کاربر را به تنظیم هدایت‌شدهٔ استنتاج بازمی‌گرداند.

## افزودن عامل دیگر

برای ایجاد عاملی جداگانه با فضای کاری، نشست‌ها و
نمایه‌های احراز هویت مختص خود، از `openclaw agents add <name>` استفاده کنید. اجرای آن بدون `--workspace` یک
فرایند تعاملی برای نام، فضای کاری، احراز هویت، کانال‌ها و اتصال‌ها آغاز می‌کند؛ این
ویزارد کامل `openclaw onboard` نیست.

مواردی که تنظیم می‌کند:

- `agents.list[].name`
- `agents.list[].workspace`
- `agents.list[].agentDir`

نکات:

- فضای کاری پیش‌فرض: `~/.openclaw/workspace-<agentId>` (یا زیر
  `agents.defaults.workspace`، اگر تنظیم شده باشد).
- برای هدایت پیام‌های ورودی به این عامل، `bindings` را اضافه کنید (راه‌اندازی اولیه می‌تواند این کار را برای شما انجام دهد).
- پرچم‌های غیرتعاملی: `--model`، `--agent-dir`، `--bind`، `--non-interactive`.
