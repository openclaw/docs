آماده‌سازی CLI مسیر **توصیه‌شده** برای راه‌اندازی ترمینالی OpenClaw روی
macOS، Linux، یا Windows است. کاربران دسکتاپ Windows همچنین می‌توانند با
[هاب Windows](/fa/platforms/windows) شروع کنند.
این مسیر یک Gateway محلی یا اتصال به Gateway راه‌دور، به‌همراه کانال‌ها، Skills،
و پیش‌فرض‌های فضای کاری را در یک جریان راهنمایی‌شده پیکربندی می‌کند.

```bash
openclaw onboard
```

شروع سریع معمولا فقط چند دقیقه طول می‌کشد، اما آماده‌سازی کامل ممکن است زمانی بیشتر طول بکشد
که ورود به ارائه‌دهنده، جفت‌سازی کانال، نصب دیمون، دانلودهای شبکه،
Skills، یا Pluginهای اختیاری به راه‌اندازی بیشتری نیاز داشته باشند. ویزارد این زمان‌بندی را از ابتدا نشان می‌دهد،
و مراحل اختیاری را می‌توان رد کرد و بعدا با
`openclaw configure` دوباره انجام داد.

## زبان و منطقه

ویزارد CLI متن ثابت آماده‌سازی را بومی‌سازی می‌کند. زبان و منطقه را از
`OPENCLAW_LOCALE`، سپس `LC_ALL`، سپس `LC_MESSAGES`، سپس `LANG` تشخیص می‌دهد،
و در نهایت به انگلیسی برمی‌گردد. زبان‌های پشتیبانی‌شده ویزارد `en`، `zh-CN`، و `zh-TW` هستند.

```bash
OPENCLAW_LOCALE=zh-CN openclaw onboard
```

نام‌ها و شناسه‌های پایدار به‌صورت تحت‌اللفظی باقی می‌مانند: `OpenClaw`، `Gateway`، `Tailscale`،
دستورها، کلیدهای پیکربندی، URLها، شناسه‌های ارائه‌دهنده، شناسه‌های مدل، و برچسب‌های Plugin/کانال
ترجمه نمی‌شوند.

<Info>
سریع‌ترین چت اول: Control UI را باز کنید (به راه‌اندازی کانال نیازی نیست). اجرا کنید
`openclaw dashboard` و در مرورگر چت کنید. مستندات: [داشبورد](/fa/web/dashboard).
</Info>

برای پیکربندی دوباره در آینده:

```bash
openclaw configure
openclaw agents add <name>
```

<Note>
`--json` به معنی حالت غیرتعاملی نیست. برای اسکریپت‌ها، از `--non-interactive` استفاده کنید.
</Note>

<Tip>
آماده‌سازی CLI شامل یک مرحله جست‌وجوی وب است که در آن می‌توانید ارائه‌دهنده‌ای
مانند Brave، DuckDuckGo، Exa، Firecrawl، Gemini، Grok، Kimi، MiniMax Search،
Ollama Web Search، Perplexity، SearXNG، یا Tavily را انتخاب کنید. برخی ارائه‌دهنده‌ها به
کلید API نیاز دارند، در حالی که برخی دیگر بدون کلید کار می‌کنند. همچنین می‌توانید این را بعدا با
`openclaw configure --section web` پیکربندی کنید. مستندات: [ابزارهای وب](/fa/tools/web).
</Tip>

## شروع سریع در برابر پیشرفته

آماده‌سازی با **شروع سریع** (پیش‌فرض‌ها) در برابر **پیشرفته** (کنترل کامل) آغاز می‌شود.

<Tabs>
  <Tab title="شروع سریع (پیش‌فرض‌ها)">
    - Gateway محلی (loopback)
    - پیش‌فرض فضای کاری (یا فضای کاری موجود)
    - پورت Gateway **18789**
    - احراز هویت Gateway با **Token** (تولید خودکار، حتی روی loopback)
    - پیش‌فرض سیاست ابزار برای راه‌اندازی‌های محلی جدید: `tools.profile: "coding"` (نمایه صریح موجود حفظ می‌شود)
    - پیش‌فرض ایزوله‌سازی DM: آماده‌سازی محلی وقتی مقدار تنظیم نشده باشد `session.dmScope: "per-channel-peer"` را می‌نویسد. جزئیات: [مرجع راه‌اندازی CLI](/fa/start/wizard-cli-reference#outputs-and-internals)
    - نمایش از طریق Tailscale **خاموش**
    - DMهای Telegram + WhatsApp به‌طور پیش‌فرض روی **فهرست مجاز** هستند (برای شماره تلفن از شما پرسیده می‌شود)

  </Tab>
  <Tab title="پیشرفته (کنترل کامل)">
    - همه مراحل را نمایش می‌دهد (حالت، فضای کاری، Gateway، کانال‌ها، دیمون، Skills).

  </Tab>
</Tabs>

## آماده‌سازی چه چیزهایی را پیکربندی می‌کند

**حالت محلی (پیش‌فرض)** شما را در این مراحل راهنمایی می‌کند:

1. **مدل/احراز هویت** — هر ارائه‌دهنده/جریان احراز هویت پشتیبانی‌شده را انتخاب کنید (کلید API، OAuth، یا احراز هویت دستی مخصوص ارائه‌دهنده)، از جمله ارائه‌دهنده سفارشی
   (سازگار با OpenAI، سازگار با Anthropic، یا تشخیص خودکار ناشناخته). یک مدل پیش‌فرض انتخاب کنید.
   نکته امنیتی: اگر این عامل قرار است ابزارها را اجرا کند یا محتوای webhook/hooks را پردازش کند، قوی‌ترین مدل نسل جدید موجود را ترجیح دهید و سیاست ابزار را سخت‌گیرانه نگه دارید. رده‌های ضعیف‌تر/قدیمی‌تر راحت‌تر دچار prompt injection می‌شوند.
   برای اجراهای غیرتعاملی، `--secret-input-mode ref` به‌جای مقادیر متنی کلید API، ارجاع‌های مبتنی بر env را در نمایه‌های احراز هویت ذخیره می‌کند.
   در حالت غیرتعاملی `ref`، متغیر env ارائه‌دهنده باید تنظیم شده باشد؛ ارسال پرچم‌های کلید inline بدون آن متغیر env سریع شکست می‌خورد.
   در اجراهای تعاملی، انتخاب حالت ارجاع محرمانه به شما اجازه می‌دهد به یک متغیر محیطی یا یک ref ارائه‌دهنده پیکربندی‌شده (`file` یا `exec`) اشاره کنید، همراه با اعتبارسنجی پیش‌بررسی سریع قبل از ذخیره.
   برای Anthropic، آماده‌سازی/پیکربندی تعاملی **Anthropic Claude CLI** را به‌عنوان مسیر محلی ترجیحی و **کلید API Anthropic** را به‌عنوان مسیر تولید توصیه‌شده ارائه می‌دهد. Anthropic setup-token نیز همچنان به‌عنوان مسیر احراز هویت توکنی پشتیبانی‌شده در دسترس است.
2. **فضای کاری** — محل فایل‌های عامل (پیش‌فرض `~/.openclaw/workspace`). فایل‌های bootstrap را seed می‌کند.
3. **Gateway** — پورت، آدرس bind، حالت احراز هویت، نمایش از طریق Tailscale.
   در حالت توکن تعاملی، ذخیره‌سازی پیش‌فرض توکن plaintext را انتخاب کنید یا SecretRef را برگزینید.
   مسیر SecretRef توکن غیرتعاملی: `--gateway-token-ref-env <ENV_VAR>`.
4. **کانال‌ها** — کانال‌های چت داخلی و Plugin رسمی مانند iMessage، Discord، Feishu، Google Chat، Mattermost، Microsoft Teams، QQ Bot، Signal، Slack، Telegram، WhatsApp، و موارد بیشتر.
5. **دیمون** — یک LaunchAgent (macOS)، واحد کاربری systemd (Linux/WSL2)، یا Windows Scheduled Task بومی را با fallback پوشه Startup برای هر کاربر نصب می‌کند.
   اگر احراز هویت توکنی به توکن نیاز داشته باشد و `gateway.auth.token` با SecretRef مدیریت شود، نصب دیمون آن را اعتبارسنجی می‌کند اما توکن resolve‌شده را در metadata محیط سرویس supervisor پایدار نمی‌کند.
   اگر احراز هویت توکنی به توکن نیاز داشته باشد و SecretRef توکن پیکربندی‌شده resolve نشده باشد، نصب دیمون با راهنمایی قابل‌اقدام مسدود می‌شود.
   اگر هر دو `gateway.auth.token` و `gateway.auth.password` پیکربندی شده باشند و `gateway.auth.mode` تنظیم نشده باشد، نصب دیمون تا زمانی که حالت صریحا تنظیم شود مسدود می‌شود.
6. **بررسی سلامت** — Gateway را راه‌اندازی می‌کند و اجرای آن را تایید می‌کند.
7. **Skills** — Skills توصیه‌شده و وابستگی‌های اختیاری را نصب می‌کند.

<Note>
اجرای دوباره آماده‌سازی هیچ چیزی را پاک **نمی‌کند** مگر اینکه صریحا **بازنشانی** را انتخاب کنید (یا `--reset` را ارسال کنید).
CLI `--reset` به‌طور پیش‌فرض شامل پیکربندی، اعتبارنامه‌ها، و نشست‌ها می‌شود؛ برای شامل کردن فضای کاری از `--reset-scope full` استفاده کنید.
اگر پیکربندی نامعتبر باشد یا کلیدهای قدیمی داشته باشد، آماده‌سازی از شما می‌خواهد ابتدا `openclaw doctor` را اجرا کنید.
</Note>

**حالت راه‌دور** فقط کلاینت محلی را برای اتصال به Gateway در جای دیگر پیکربندی می‌کند.
این حالت هیچ چیزی را روی میزبان راه‌دور نصب یا تغییر **نمی‌دهد**.

## افزودن عامل دیگر

از `openclaw agents add <name>` برای ایجاد یک عامل جداگانه با فضای کاری،
نشست‌ها، و نمایه‌های احراز هویت خودش استفاده کنید. اجرا بدون `--workspace` آماده‌سازی را راه‌اندازی می‌کند.

آنچه تنظیم می‌کند:

- `agents.list[].name`
- `agents.list[].workspace`
- `agents.list[].agentDir`

نکات:

- فضاهای کاری پیش‌فرض از `~/.openclaw/workspace-<agentId>` پیروی می‌کنند.
- برای مسیریابی پیام‌های ورودی، `bindings` اضافه کنید (آماده‌سازی می‌تواند این کار را انجام دهد).
- پرچم‌های غیرتعاملی: `--model`، `--agent-dir`، `--bind`، `--non-interactive`.
