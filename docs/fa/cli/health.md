# `openclaw health`

یک تصویر لحظه‌ای از سلامت Gateway در حال اجرا را از طریق RPC مبتنی بر WebSocket دریافت می‌کند (بدون سوکت مستقیم کانال از CLI).

## گزینه‌ها

| پرچم             | پیش‌فرض | توضیحات                                                                                               |
| ---------------- | ------- | ----------------------------------------------------------------------------------------------------- |
| `--json`         | `false` | به‌جای متن، JSON قابل‌خواندن توسط ماشین را چاپ می‌کند.                                                |
| `--timeout <ms>` | `10000` | مهلت زمانی اتصال بر حسب میلی‌ثانیه.                                                                   |
| `--verbose`      | `false` | یک بررسی زنده را اجباری می‌کند و خروجی را برای همه حساب‌ها و عامل‌های پیکربندی‌شده گسترش می‌دهد.       |
| `--debug`        | `false` | نام مستعار `--verbose` است.                                                                           |

مثال‌ها:

```bash
openclaw health
openclaw health --json
openclaw health --timeout 2500
openclaw health --verbose
openclaw health --debug
```

## رفتار

- بدون `--verbose`، Gateway می‌تواند یک تصویر لحظه‌ای ذخیره‌شده در حافظه نهان را بازگرداند (که تا ۶۰ ثانیه تازه است و با وضعیت زنده زمان اجرای کانال تفاوتی ندارد) و آن را در پس‌زمینه برای فراخواننده بعدی تازه‌سازی کند.
- `--verbose` یک بررسی زنده را اجباری می‌کند (بررسی حساب‌ها به‌ازای هر کانال)، جزئیات اتصال Gateway را چاپ می‌کند و خروجی خوانا برای انسان را به‌جای محدود کردن به عامل پیش‌فرض، برای همه حساب‌ها و عامل‌های پیکربندی‌شده گسترش می‌دهد.
- `--json` همیشه تصویر لحظه‌ای کامل را بازمی‌گرداند: کانال‌ها، بررسی‌های هر حساب، وضعیت بارگذاری Plugin، وضعیت قرنطینه موتور زمینه، وضعیت حافظه نهان قیمت‌گذاری مدل، سلامت حلقه رویداد و مخازن نشست هر عامل.
