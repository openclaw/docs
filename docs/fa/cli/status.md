عیب‌یابی کانال‌ها + نشست‌ها.

```bash
openclaw status
openclaw status --all
openclaw status --deep
openclaw status --usage
```

| پرچم                    | توضیحات                                                                                                                        |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| `--all`                 | عیب‌یابی کامل (فقط‌خواندنی، قابل جای‌گذاری). شامل ممیزی امنیتی، سازگاری Plugin و کاوش‌های بردار حافظه است.                       |
| `--deep`                | کاوش‌های زنده را اجرا می‌کند (WhatsApp Web + Telegram + Discord + Slack + Signal). همچنین ممیزی امنیتی را فعال می‌کند.          |
| `--usage`               | بازه‌های نرمال‌شدهٔ مصرف ارائه‌دهنده را به‌شکل `X% left` چاپ می‌کند.                                                            |
| `--json`                | خروجی قابل‌خواندن برای ماشین.                                                                                                  |
| `--verbose` / `--debug` | تفکیک خام مقصد Gateway را نیز پیش از گزارش چاپ می‌کند.                                                                          |

دستور سادهٔ `openclaw status` در مسیر سریع و فقط‌خواندنی باقی می‌ماند و هنگامی که بررسی حافظه را رد می‌کند، به‌جای دردسترس‌نبودن، حافظه را با وضعیت
`not checked` علامت‌گذاری می‌کند. ممیزی امنیتی سنگین، سازگاری Plugin و کاوش‌های بردار حافظه به
`openclaw status --all`،‏ `openclaw status --deep`،‏ `openclaw security audit`
و `openclaw memory status --deep` واگذار می‌شوند.

## تفکیک نشست و مدل

- خروجی وضعیت نشست، `Execution:` را از `Runtime:` جدا می‌کند. `Execution`
  مسیر سندباکس (`direct`،‏ `docker/*`) است، درحالی‌که `Runtime` مشخص می‌کند
  آیا نشست از `OpenClaw Default`،‏ `OpenAI Codex`، یک بک‌اند CLI
  یا یک بک‌اند ACP مانند `codex (acp/acpx)` استفاده می‌کند. برای آشنایی با
  تفاوت ارائه‌دهنده/مدل/محیط اجرا، به
  [محیط‌های اجرای عامل](/fa/concepts/agent-runtimes) مراجعه کنید.
- هنگامی که اسنپ‌شات نشست فعلی داده‌های کمی دارد، `/status` می‌تواند شمارنده‌های توکن
  و کش را از جدیدترین گزارش مصرف رونوشت بازیابی کند. مقادیر زندهٔ غیرصفر موجود
  همچنان بر مقادیر جایگزین رونوشت اولویت دارند.
- سازوکار جایگزین رونوشت می‌تواند برچسب مدل فعال محیط اجرا را نیز هنگامی بازیابی کند
  که ورودی نشست زنده فاقد آن باشد. اگر آن مدل رونوشت با مدل انتخاب‌شده متفاوت باشد،
  وضعیت به‌جای مدل انتخاب‌شده، پنجرهٔ زمینه را بر اساس مدل بازیابی‌شدهٔ محیط اجرا
  تفکیک می‌کند.
- برای محاسبهٔ اندازهٔ پرامپت، هنگامی که فرادادهٔ نشست موجود نیست یا کوچک‌تر است،
  سازوکار جایگزین رونوشت مجموع بزرگ‌تر و معطوف به پرامپت را ترجیح می‌دهد تا
  نمایش توکن نشست‌های ارائه‌دهندهٔ سفارشی به `0` تقلیل نیابد.
- هنگامی که یک نشست به مدلی سنجاق شده است که با مدل اصلی پیکربندی‌شده تفاوت دارد،
  وضعیت هر دو مقدار، دلیل (`session override`) و راهنمای `/model default` را
  چاپ می‌کند. مدل اصلی پیکربندی‌شده برای نشست‌های جدید یا سنجاق‌نشده اعمال می‌شود؛
  نشست‌های سنجاق‌شدهٔ موجود تا زمان پاک‌شدن، انتخاب نشست خود را حفظ می‌کنند.
- هنگامی که چند عامل پیکربندی شده باشند، خروجی شامل مخازن نشست هر عامل است.

## مصرف و سهمیه

- `--usage` بازه‌های نرمال‌شدهٔ مصرف ارائه‌دهنده را به‌شکل `X% left` چاپ می‌کند.
- فیلدهای خام `usage_percent` / `usagePercent` در MiniMax نشان‌دهندهٔ سهمیهٔ باقی‌مانده‌اند،
  بنابراین OpenClaw پیش از نمایش آن‌ها را معکوس می‌کند؛ فیلدهای مبتنی بر شمارش، در صورت
  وجود، اولویت دارند. پاسخ‌های `model_remains` ورودی مدل گفتگو را ترجیح می‌دهند، در صورت
  نیاز برچسب بازه را از مُهرهای زمانی استخراج می‌کنند و نام مدل را در برچسب طرح می‌گنجانند.
- خطاهای تازه‌سازی قیمت‌گذاری مدل به‌صورت هشدارهای اختیاری قیمت‌گذاری نمایش داده می‌شوند.
  این خطاها به‌معنای ناسالم‌بودن Gateway یا کانال‌ها نیستند.

## نمای کلی و وضعیت به‌روزرسانی

- نمای کلی، در صورت وجود، شامل وضعیت نصب/اجرای سرویس میزبان Gateway + Node است و
  همچنین زمان فعالیت فشردهٔ فرایند Gateway و زمان فعالیت سیستم میزبان را نشان می‌دهد.
- نمای کلی شامل کانال به‌روزرسانی + SHA گیت (برای نسخه‌های دریافت‌شده از کد منبع) است.
- اطلاعات به‌روزرسانی در نمای کلی نمایش داده می‌شود؛ اگر به‌روزرسانی موجود باشد، وضعیت
  راهنمای اجرای `openclaw update` را چاپ می‌کند (به [به‌روزرسانی](/fa/install/updating) مراجعه کنید).

## اسرار

- سطوح وضعیت فقط‌خواندنی (`status`،‏ `status --json`،‏ `status --all`)
  در صورت امکان، SecretRefهای پشتیبانی‌شده را برای مسیرهای پیکربندی هدف خود تفکیک می‌کنند.
- اگر SecretRef پشتیبانی‌شده‌ای برای یک کانال پیکربندی شده باشد اما در مسیر فرمان فعلی
  در دسترس نباشد، وضعیت فقط‌خواندنی باقی می‌ماند و به‌جای ازکارافتادن، خروجی تنزل‌یافته
  را گزارش می‌کند. خروجی انسانی هشدارهایی مانند «توکن پیکربندی‌شده در این مسیر فرمان
  در دسترس نیست» را نشان می‌دهد و خروجی JSON شامل `secretDiagnostics` است.
- هنگامی که تفکیک SecretRef محلی فرمان موفق باشد، وضعیت اسنپ‌شات تفکیک‌شده را ترجیح
  می‌دهد و نشانگرهای گذرای «سرّ در دسترس نیست» کانال را از خروجی نهایی پاک می‌کند.
- `status --all` شامل یک ردیف نمای کلی اسرار و یک بخش عیب‌یابی است که تشخیص‌های اسرار
  را بدون توقف تولید گزارش خلاصه می‌کند (برای خوانایی کوتاه‌شده).

## حافظه

`status --json --all` جزئیات حافظه را از محیط اجرای Plugin حافظهٔ فعال که توسط
`plugins.slots.memory` انتخاب شده است، گزارش می‌کند. Pluginهای حافظهٔ سفارشی می‌توانند
`agents.defaults.memorySearch.enabled` داخلی را غیرفعال نگه دارند و همچنان وضعیت
فایل‌ها، قطعه‌ها، بردار و FTS خود را گزارش کنند.
