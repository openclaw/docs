# `openclaw system`

ابزارهای کمکی سطح سیستم برای Gateway: افزودن رویدادهای سیستمی به صف، کنترل Heartbeatها و مشاهدهٔ وضعیت حضور.

همهٔ زیرفرمان‌های `system` از RPC مربوط به Gateway استفاده می‌کنند و پرچم‌های مشترک کلاینت را می‌پذیرند:

| پرچم              | پیش‌فرض                              | توضیحات                                                                                                                                                                                            |
| ----------------- | ------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `--url <url>`     | در صورت پیکربندی، `gateway.remote.url` | نشانی WebSocket مربوط به Gateway.                                                                                                                                                                                 |
| `--token <token>` | هیچ‌کدام                                 | توکن Gateway (در صورت نیاز).                                                                                                                                                                           |
| `--timeout <ms>`  | `30000`                              | مهلت زمانی RPC بر حسب میلی‌ثانیه.                                                                                                                                                                           |
| `--expect-final`  | غیرفعال                                  | انتظار برای پاسخ نهایی (عامل).                                                                                                                                                                       |
| `--json`          | غیرفعال                                  | خروجی JSON. فرمان‌های `heartbeat last/enable/disable` و `system presence` صرف‌نظر از این پرچم، همیشه محتوای خام JSON مربوط به RPC را چاپ می‌کنند؛ `system event` از آن برای جابه‌جایی بین JSON و یک خط سادهٔ `ok` استفاده می‌کند. |

## فرمان‌های رایج

```bash
openclaw system event --text "Check for urgent follow-ups" --mode now
openclaw system event --text "Check for urgent follow-ups" --url ws://127.0.0.1:18789 --token "$OPENCLAW_GATEWAY_TOKEN"
openclaw system heartbeat enable
openclaw system heartbeat last
openclaw system presence
```

## `system event`

به‌طور پیش‌فرض، یک رویداد سیستمی را در نشست **اصلی** به صف اضافه می‌کند. Heartbeat بعدی آن را به‌شکل یک خط `System:` در پرامپت درج می‌کند. برای فعال‌کردن فوری Heartbeat از `--mode now` استفاده کنید؛ `next-heartbeat` (پیش‌فرض) تا تیک زمان‌بندی‌شدهٔ بعدی منتظر می‌ماند.

برای هدف‌گرفتن یک نشست مشخص، `--session-key` را ارسال کنید؛ برای مثال، جهت بازگرداندن تکمیل یک وظیفهٔ ناهمگام به کانالی که آن را آغاز کرده است.

<Note>
**استثنای زمان‌بندی با `--session-key`:** هنگامی که `--session-key` ارائه شود،
`--mode next-heartbeat` به‌جای انتظار برای تیک زمان‌بندی‌شدهٔ بعدی، به یک بیدارسازی هدفمند فوری تبدیل می‌شود. بیدارسازی‌های هدفمند از قصد Heartbeat با مقدار
`immediate` استفاده می‌کنند تا از دروازهٔ «هنوز موعد نرسیده» در اجراکننده عبور کنند؛ در غیر این صورت، این دروازه یک بیدارسازی با قصد `event` را به تعویق می‌انداخت (و عملاً حذف می‌کرد). اگر تحویل با تأخیر می‌خواهید،
`--session-key` را حذف کنید تا رویداد در نشست اصلی قرار گیرد و همراه Heartbeat عادی بعدی اجرا شود.
</Note>

پرچم‌ها:

- `--text <text>`: متن الزامی رویداد سیستمی.
- `--mode <mode>`: `now` یا `next-heartbeat` (پیش‌فرض).
- `--session-key <sessionKey>`: اختیاری؛ به‌جای نشست اصلی عامل، یک نشست مشخص عامل را هدف می‌گیرد. کلیدهایی که به عامل شناسایی‌شده تعلق ندارند، به نشست اصلی عامل بازمی‌گردند.

## `system heartbeat last|enable|disable`

- `last`: آخرین رویداد Heartbeat را نمایش می‌دهد.
- `enable`: Heartbeatها را دوباره فعال می‌کند (اگر غیرفعال شده‌اند، از این گزینه استفاده کنید).
- `disable`: Heartbeatها را متوقف می‌کند.

## `system presence`

ورودی‌های فعلی حضور سیستم را که Gateway از آن‌ها اطلاع دارد فهرست می‌کند (Nodeها، نمونه‌ها و خطوط وضعیت مشابه).

## نکات

- به یک Gateway در حال اجرا نیاز دارد که از طریق پیکربندی فعلی شما (محلی یا راه‌دور) قابل دسترسی باشد.
- رویدادهای سیستمی موقتی هستند و پس از راه‌اندازی مجدد حفظ نمی‌شوند.
