# `openclaw setup`

`openclaw setup` نقطه ورود عامل سیستم است. در یک سیستم پیکربندی‌شده، اجرای صرف
`openclaw setup` یک گفت‌وگوی تعاملی OpenClaw را باز می‌کند. در یک سیستم تازه، به
راه‌اندازی هدایت‌شده منتقل می‌شود. برای یک درخواست از `-m`/`--message` استفاده کنید یا
برای مقداردهی اولیه پوشه‌های پیکربندی/فضای کاری بدون جادوگر، از `--baseline` استفاده کنید.

ترتیب مسیریابی:

1. هر گزینه راه‌اندازی (`--wizard`، `--baseline`، فضای کاری، بازنشانی،
   غیرتعاملی، جریان، حالت، Gateway، دیمن، رد کردن، درون‌ریزی، راه دور، یا گزینه‌های
   احراز هویت) راه‌اندازی را دقیقاً مانند `openclaw onboard` اجرا می‌کند.
2. `-m`/`--message` یا `--yes` عامل سیستم را اجرا می‌کند.
3. بدون گزینه مسیریابی، یک سیستم تعاملی پیکربندی‌شده OpenClaw را باز می‌کند. یک
   سیستم تازه راه‌اندازی را اجرا می‌کند. در یک سیستم پیکربندی‌شده، `--json` حتی بدون TTY
   نمای کلی سیستم را چاپ می‌کند؛ یک گزینه راه‌اندازی، خلاصه JSON راه‌اندازی را
   حفظ می‌کند.

در حالت هدایت‌شده، `--workspace <dir>` فضای کاری پیشنهادی به OpenClaw است؛
این مقدار تنها پس از تأیید پیشنهاد ذخیره می‌شود. راه‌اندازی پایه، کلاسیک و
غیرتعاملی، فضای کاری ارائه‌شده را از طریق جریان عادی خود ذخیره می‌کنند.

تشخیص استنتاج هدایت‌شده روی میزبان Gateway در macOS یا Linux اجرا می‌شود. CLI
و برنامه macOS همان تشخیص‌دهنده تحت مالکیت Gateway را فراخوانی می‌کنند که مدل‌های
پیکربندی‌شده، ورودهای CLI پشتیبانی‌شده، متغیرهای محیطی کلید API و مدل‌های از پیش
نصب‌شده Ollama یا LM Studio را بررسی می‌کند. در این مرحله خودکار، مدل‌های محلی هرگز
بارگیری نمی‌شوند؛ گزینه انتخاب‌شده باید پیش از ذخیره پیکربندی ارائه‌دهنده و مدل،
به یک تکمیل واقعی پاسخ دهد.

`setup` همان پرچم‌های راه‌اندازی `openclaw onboard` را می‌پذیرد، از جمله
احراز هویت (`--auth-choice`، `--token`، پرچم‌های کلید ارائه‌دهنده)، Gateway
(`--gateway-port`، `--gateway-bind`، `--gateway-auth`، `--install-daemon`)،
Tailscale (`--tailscale`)، بازنشانی (`--reset`، `--reset-scope`)، جریان
(`--flow quickstart|advanced|manual|import`) و پرچم‌های رد کردن
(`--skip-channels`، `--skip-skills`، `--skip-bootstrap`، `--skip-search`،
`--skip-health`، `--skip-ui`، `--skip-hooks`). برای مرجع کامل پرچم‌ها و
نمونه‌های غیرتعاملی، به [راه‌اندازی](/fa/cli/onboard) و
[اتوماسیون CLI](/fa/start/wizard-cli-automation) مراجعه کنید. `openclaw onboard --modern` همچنان یک ورودی
سازگاری برای همان دستیار OpenClaw وابسته به استنتاج باقی می‌ماند.

<Note>
`openclaw setup` برای نصب‌هایی با پیکربندی قابل‌تغییر است. در حالت Nix ‏(`OPENCLAW_NIX_MODE=1`)، OpenClaw از نوشتن تغییرات راه‌اندازی خودداری می‌کند، زیرا فایل پیکربندی توسط Nix مدیریت می‌شود. از [شروع سریع nix-openclaw](https://github.com/openclaw/nix-openclaw#quick-start) رسمی یا پیکربندی منبع معادل برای بسته Nix دیگری استفاده کنید.
</Note>

## گزینه‌ها

| پرچم                       | توضیحات                                                                                           |
| -------------------------- | ----------------------------------------------------------------------------------------------------- |
| `-m, --message <text>`     | اجرای یک درخواست OpenClaw.                                                                             |
| `--yes`                    | تأیید نوشتن پایدار پیکربندی برای یک درخواست `--message`.                                         |
| `--workspace <dir>`        | پیشنهاد فضای کاری در حالت هدایت‌شده؛ در راه‌اندازی پایه، کلاسیک و غیرتعاملی مستقیماً ذخیره می‌شود. |
| `--baseline`               | ایجاد پوشه‌های پایه پیکربندی/فضای کاری/نشست بدون راه‌اندازی.                                  |
| `--wizard`                 | اجبار راه‌اندازی تعاملی.                                                                         |
| `--non-interactive`        | اجرای راه‌اندازی بدون اعلان.                                                                       |
| `--accept-risk`            | تأیید خطر دسترسی عامل به کل سیستم؛ همراه با `--non-interactive` الزامی است.                         |
| `--mode <mode>`            | حالت راه‌اندازی: `local` یا `remote`.                                                                 |
| `--flow <flow>`            | جریان راه‌اندازی: `quickstart`، `advanced`، `manual` یا `import`.                                        |
| `--reset`                  | بازنشانی پیکربندی + اعتبارنامه‌ها + نشست‌ها پیش از راه‌اندازی (فضای کاری فقط با `--reset-scope full`).   |
| `--reset-scope <scope>`    | دامنه بازنشانی: `config`، `config+creds+sessions` یا `full`.                                            |
| `--import-from <provider>` | ارائه‌دهنده مهاجرت که هنگام راه‌اندازی اجرا می‌شود.                                                          |
| `--import-source <path>`   | خانه عامل مبدأ برای `--import-from`.                                                                |
| `--import-secrets`         | درون‌ریزی رازهای پشتیبانی‌شده هنگام مهاجرت راه‌اندازی.                                                 |
| `--remote-url <url>`       | نشانی WebSocket راه دور Gateway.                                                                         |
| `--remote-token <token>`   | توکن Gateway راه دور (اختیاری).                                                                      |
| `--json`                   | سیستم پیکربندی‌شده: نمای کلی OpenClaw. مسیر راه‌اندازی: خلاصه راه‌اندازی.                           |

`--classic` و `--non-interactive` ناسازگارند: حالت کلاسیک جادوگر
اعلان‌محور را باز می‌کند، درحالی‌که راه‌اندازی غیرتعاملی از مسیر اتوماسیون استفاده می‌کند.

### حالت پایه

`openclaw setup --baseline` رفتار قدیمی مختص حالت پایه را حفظ می‌کند: این حالت
دایرکتوری‌های پیکربندی، فضای کاری و نشست را ایجاد می‌کند و سپس بدون
اجرای راه‌اندازی خارج می‌شود.

## نمونه‌ها

```bash
openclaw setup
openclaw setup -m "status"
openclaw setup -m "restart gateway" --yes
openclaw setup --json
openclaw setup --wizard
openclaw setup --baseline
openclaw setup --workspace ~/.openclaw/workspace
openclaw setup --import-from hermes --import-source ~/.hermes
openclaw setup --non-interactive --accept-risk --mode remote --remote-url wss://gateway-host:18789 --remote-token <token>
```

## نکته‌ها

- پس از راه‌اندازی پایه، برای فرایند کامل هدایت‌شده `openclaw onboard`، برای تغییرات هدفمند `openclaw configure`، یا برای افزودن حساب‌های کانال `openclaw channels add` را اجرا کنید.
- اگر وضعیت Hermes شناسایی شود، راه‌اندازی تعاملی می‌تواند مهاجرت را به‌طور خودکار پیشنهاد دهد. راه‌اندازی درون‌ریزی به یک تنظیم تازه نیاز دارد؛ برای طرح‌های اجرای آزمایشی، پشتیبان‌گیری‌ها و حالت بازنویسی خارج از راه‌اندازی، از [مهاجرت](/fa/cli/migrate) استفاده کنید.
