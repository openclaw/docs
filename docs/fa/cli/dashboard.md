# `openclaw dashboard`

رابط کاربری کنترل را با احراز هویت فعلی خود باز کنید.

```bash
openclaw dashboard
openclaw dashboard --no-open
openclaw dashboard --json
openclaw dashboard --yes
```

- `--no-open`: نشانی URL را چاپ می‌کند، اما مرورگر را اجرا نمی‌کند.
- `--json`: بدون بازکردن مرورگر، استفاده از کلیپ‌بورد، نمایش درخواست ورودی یا راه‌اندازی Gateway، یک شیء اتصال ماشین‌خوان را چاپ می‌کند.
- `--yes`: در صورت نیاز، Gateway را بدون نمایش درخواست ورودی راه‌اندازی/نصب می‌کند.

## خروجی ماشین‌خوان

برای یکپارچه‌سازی‌های دسکتاپ و اسکریپت‌هایی که به نشانی URL حل‌شدهٔ رابط کاربری کنترل نیاز دارند، از `--json` استفاده کنید:

```bash
openclaw dashboard --json
```

پاسخ شامل `url`، `httpUrl`، `wsUrl`، `port` و `tokenIncluded` است. اگر Gateway آماده نباشد، فرمان `{"ok":false,"reason":"..."}` را برمی‌گرداند و با کد غیرصفر خارج می‌شود. توکن‌های مدیریت‌شده با SecretRef هرگز در `url` گنجانده نمی‌شوند.

نکته‌ها:

- در صورت امکان، SecretRefهای پیکربندی‌شدهٔ `gateway.auth.token` را حل می‌کند.
- از `gateway.tls.enabled` پیروی می‌کند: Gatewayهای دارای TLS نشانی‌های URL رابط کاربری کنترل `https://` را چاپ/باز می‌کنند و از طریق `wss://` متصل می‌شوند.
- برای اتصال `lan` یا اتصال wildcard با `custom`، اجراها روی همان میزبان همیشه از loopback استفاده می‌کنند، زیرا wildcard مقصد مرورگر نیست. اتصال‌های متن سادهٔ `tailnet` و `custom` نیز از `127.0.0.1` استفاده می‌کنند تا مرورگر زمینه‌ای امن داشته باشد؛ میزبان‌های مشخص دارای TLS نشانی پیکربندی‌شده را حفظ می‌کنند تا نام‌های گواهی مطابقت داشته باشند.
- پیش از ارائهٔ یک نشانی URL احرازشدهٔ loopback برای اتصال به یک رابط مشخص، فرمان رابط پیکربندی‌شده را بررسی می‌کند و تأیید می‌کند که آن رابط و `127.0.0.1` تحت مالکیت همان فرایند Gateway هستند. اگر مالکیت شنونده مبهم باشد، عملیات به‌صورت بسته و همراه با راهنمایی وضعیت شکست می‌خورد.
- برای توکن‌های مدیریت‌شده با SecretRef (حل‌شده یا حل‌نشده)، نشانی URL چاپ‌شده/کپی‌شده/بازشده هرگز شامل توکن نیست؛ بنابراین رازهای خارجی در خروجی ترمینال، تاریخچهٔ کلیپ‌بورد یا آرگومان‌های اجرای مرورگر افشا نمی‌شوند.
- اگر `gateway.auth.token` با SecretRef مدیریت شود اما حل‌نشده باشد، فرمان به‌جای یک جای‌نگهدار نامعتبر توکن، یک نشانی URL بدون توکن و راهنمای رفع مشکل چاپ می‌کند.
- اگر تحویل از طریق کلیپ‌بورد/مرورگر برای یک نشانی URL با احراز هویت توکنی شکست بخورد، فرمان بدون چاپ مقدار توکن، یک راهنمای امن برای احراز هویت دستی ثبت می‌کند که `OPENCLAW_GATEWAY_TOKEN`، `gateway.auth.token` و کلید بخش URL یعنی `token` را نام می‌برد.
