# `openclaw configure`

اعلان تعاملی برای تغییرات هدفمند در یک راه‌اندازی موجود: اعتبارنامه‌ها، دستگاه‌ها، پیش‌فرض‌های عامل، Gateway، کانال‌ها، Plugin‌ها، Skills و بررسی‌های سلامت.

برای مسیر کامل راهنمایی‌شده در اولین اجرا از `openclaw onboard` یا `openclaw setup` استفاده کنید، برای فقط پیکربندی/فضای کاری پایه از `openclaw setup --baseline`، و وقتی فقط به راه‌اندازی حساب کانال نیاز دارید از `openclaw channels add` استفاده کنید.

<Note>
بخش **مدل** شامل یک انتخاب چندگانه برای فهرست مجاز `agents.defaults.models` است (آنچه در `/model` و انتخاب‌گر مدل نمایش داده می‌شود). انتخاب‌های راه‌اندازی محدود به ارائه‌دهنده، مدل‌های انتخاب‌شده خود را به‌جای جایگزین کردن ارائه‌دهندگان نامرتبطی که از قبل در پیکربندی هستند، در فهرست مجاز موجود ادغام می‌کنند.

اجرای دوباره احراز هویت ارائه‌دهنده از configure، مقدار موجود `agents.defaults.model.primary` را حفظ می‌کند، حتی وقتی مرحله احراز هویت ارائه‌دهنده یک وصله پیکربندی با مدل پیش‌فرض پیشنهادی خودش برمی‌گرداند. این یعنی افزودن یا احراز هویت دوباره xAI، OpenRouter یا ارائه‌دهنده‌ای دیگر باید مدل جدید را بدون گرفتن جای مدل اصلی فعلی شما در دسترس کند. وقتی عمداً می‌خواهید مدل پیش‌فرض را تغییر دهید، از `openclaw models auth login --provider <id> --set-default` یا `openclaw models set <model>` استفاده کنید.
</Note>

وقتی configure از یک انتخاب احراز هویت ارائه‌دهنده شروع می‌شود، انتخاب‌گرهای مدل پیش‌فرض و فهرست مجاز به‌طور خودکار همان ارائه‌دهنده را ترجیح می‌دهند. برای ارائه‌دهندگان جفت‌شده مانند Volcengine و BytePlus، همین ترجیح با گونه‌های coding-plan آن‌ها نیز مطابقت دارد (`volcengine-plan/*`، `byteplus-plan/*`). اگر فیلتر ارائه‌دهنده ترجیحی فهرستی خالی تولید کند، configure به‌جای نمایش انتخاب‌گری خالی، به کاتالوگ فیلترنشده برمی‌گردد.

<Tip>
`openclaw config` بدون زیرفرمان همان جادوگر را باز می‌کند. برای ویرایش‌های غیرتعاملی از `openclaw config get|set|unset` استفاده کنید.
</Tip>

برای جست‌وجوی وب، `openclaw configure --section web` به شما امکان می‌دهد یک ارائه‌دهنده انتخاب کنید
و اعتبارنامه‌های آن را پیکربندی کنید. برخی ارائه‌دهندگان همچنین اعلان‌های پیگیری
ویژه ارائه‌دهنده را نمایش می‌دهند:

- **Grok** می‌تواند راه‌اندازی اختیاری `x_search` را با همان نمایه OAuth مربوط به xAI
  یا کلید API پیشنهاد کند و اجازه دهد یک مدل `x_search` انتخاب کنید.
- **Kimi** می‌تواند منطقه API مربوط به Moonshot (`api.moonshot.ai` در برابر
  `api.moonshot.cn`) و مدل پیش‌فرض جست‌وجوی وب Kimi را بپرسد.

## گزینه‌ها

- `--section <section>`: فیلتر بخش تکرارپذیر

بخش‌های موجود:

- `workspace`
- `model`
- `web`
- `gateway`
- `daemon`
- `channels`
- `plugins`
- `skills`
- `health`

نکته‌ها:

- جادوگر کامل و بخش‌های مرتبط با Gateway می‌پرسند Gateway کجا اجرا می‌شود و `gateway.mode` را به‌روزرسانی می‌کنند. فیلترهای بخشی که شامل `gateway`، `daemon` یا `health` نیستند مستقیماً به راه‌اندازی درخواست‌شده می‌روند.
- پس از نوشتن پیکربندی محلی، configure وقتی مسیر راه‌اندازی انتخاب‌شده به آن‌ها نیاز داشته باشد، Plugin‌های قابل دانلود انتخاب‌شده را نصب می‌کند. پیکربندی Gateway راه‌دور بسته‌های Plugin محلی را نصب نمی‌کند.
- سرویس‌های کانال‌محور (Slack/Discord/Matrix/Microsoft Teams) هنگام راه‌اندازی، فهرست‌های مجاز کانال/اتاق را درخواست می‌کنند. می‌توانید نام‌ها یا شناسه‌ها را وارد کنید؛ جادوگر در صورت امکان نام‌ها را به شناسه‌ها تبدیل می‌کند.
- اگر مرحله نصب daemon را اجرا کنید، احراز هویت توکنی به توکن نیاز داشته باشد، و `gateway.auth.token` توسط SecretRef مدیریت شود، configure مقدار SecretRef را اعتبارسنجی می‌کند اما مقدارهای توکن متن ساده حل‌شده را در فراداده محیط سرویس supervisor ذخیره نمی‌کند.
- اگر احراز هویت توکنی به توکن نیاز داشته باشد و SecretRef توکن پیکربندی‌شده حل‌نشده باشد، configure نصب daemon را با راهنمایی اصلاحی قابل اقدام مسدود می‌کند.
- اگر هر دو `gateway.auth.token` و `gateway.auth.password` پیکربندی شده باشند و `gateway.auth.mode` تنظیم نشده باشد، configure نصب daemon را تا زمانی که حالت به‌صراحت تنظیم شود مسدود می‌کند.

## نمونه‌ها

```bash
openclaw configure
openclaw configure --section web
openclaw configure --section model --section channels
openclaw configure --section gateway --section daemon
```
