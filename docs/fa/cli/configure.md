# `openclaw configure`

پرسش‌های تعاملی برای اعمال تغییرات هدفمند در یک راه‌اندازی موجود: اطلاعات احراز هویت، دستگاه‌ها، پیش‌فرض‌های عامل، Gateway، کانال‌ها، Pluginها، Skills و بررسی‌های سلامت.

برای فرایند کامل و هدایت‌شدهٔ اجرای نخست از `openclaw onboard` یا `openclaw setup`، فقط برای پیکربندی پایه و فضای کاری از `openclaw setup --baseline` و هنگامی که تنها به راه‌اندازی حساب کانال نیاز دارید از `openclaw channels add` استفاده کنید.

<Tip>
اجرای `openclaw config` بدون زیرفرمان، همین راهنما را باز می‌کند. برای ویرایش‌های غیرتعاملی از `openclaw config get|set|unset` استفاده کنید.
</Tip>

## گزینه‌ها

`--section <section>`: فیلتر تکرارپذیر بخش‌ها. بخش‌های موجود:

`workspace`، `model`، `web`، `gateway`، `daemon`، `channels`، `plugins`، `skills`، `health`

```bash
openclaw configure
openclaw configure --section web
openclaw configure --section model --section channels
openclaw configure --section gateway --section daemon
```

انتخاب `gateway`، `daemon` یا `health` (یا اجرای راهنمای کامل بدون `--section`) دربارهٔ محل اجرای Gateway پرسش می‌کند و `gateway.mode` را به‌روزرسانی می‌کند. فیلترهای بخشی که هر سه مورد را نادیده می‌گیرند، بدون پرسش دربارهٔ حالت Gateway مستقیماً به راه‌اندازی درخواستی می‌روند. انتخاب حالت Gateway راه‌دور، پیکربندی راه‌دور را می‌نویسد و بلافاصله خارج می‌شود؛ مراحل مختص اجرای محلی، مانند نصب Pluginها، اجرا نمی‌شوند.

<Note>
`openclaw configure` به یک پایانهٔ تعاملی نیاز دارد (هم ورودی استاندارد و هم خروجی استاندارد باید TTY باشند). بدون آن، به‌جای اجرای ناقص، فرمان‌های غیرتعاملی معادل `openclaw config get|set|patch|validate` را چاپ می‌کند و با خطا خارج می‌شود.
</Note>

## بخش مدل

<Note>
**مدل** شامل یک انتخاب‌گر چندگزینه‌ای برای فهرست مجاز `agents.defaults.models` است (مواردی که در `/model` و انتخاب‌گر مدل نمایش داده می‌شوند). گزینه‌های راه‌اندازی مختص ارائه‌دهنده، مدل‌های انتخاب‌شدهٔ خود را با فهرست مجاز موجود ادغام می‌کنند و ارائه‌دهندگان نامرتبطی را که از قبل در پیکربندی هستند جایگزین نمی‌کنند.

اجرای دوبارهٔ احراز هویت ارائه‌دهنده از طریق پیکربندی، مقدار موجود `agents.defaults.model.primary` را حفظ می‌کند؛ حتی زمانی که مرحلهٔ احراز هویت ارائه‌دهنده وصلهٔ پیکربندی‌ای را با مدل پیش‌فرض پیشنهادی خودش بازمی‌گرداند. افزودن یا احراز هویت دوبارهٔ یک ارائه‌دهنده، مدل‌های آن را بدون جایگزین کردن مدل اصلی فعلی شما در دسترس قرار می‌دهد. برای تغییر عمدی مدل پیش‌فرض از `openclaw models auth login --provider <id> --set-default` یا `openclaw models set <model>` استفاده کنید.
</Note>

هنگامی که پیکربندی از گزینهٔ احراز هویت یک ارائه‌دهنده آغاز می‌شود، انتخاب‌گرهای مدل پیش‌فرض و فهرست مجاز به‌طور خودکار آن ارائه‌دهنده را ترجیح می‌دهند. برای ارائه‌دهندگان جفت‌شده‌ای مانند Volcengine و BytePlus، همین ترجیح گونه‌های طرح برنامه‌نویسی آن‌ها (`volcengine-plan/*` و `byteplus-plan/*`) را نیز تطبیق می‌دهد. اگر فیلتر ارائه‌دهندهٔ ترجیحی به فهرستی خالی منجر شود، پیکربندی به‌جای نمایش انتخاب‌گری خالی، از کاتالوگ فیلترنشده استفاده می‌کند.

## بخش وب

`openclaw configure --section web` یک ارائه‌دهندهٔ جست‌وجوی وب را انتخاب و اطلاعات احراز هویت آن را پیکربندی می‌کند. برخی ارائه‌دهندگان گزینه‌های تکمیلی مختص خود را نمایش می‌دهند:

- **Grok** می‌تواند راه‌اندازی اختیاری `x_search` را با همان نمایهٔ OAuth یا کلید API متعلق به xAI ارائه دهد و امکان انتخاب یک مدل `x_search` را فراهم کند.
- **Kimi** می‌تواند دربارهٔ منطقهٔ API متعلق به Moonshot (`api.moonshot.ai` در برابر `api.moonshot.cn`) و مدل پیش‌فرض جست‌وجوی وب Kimi سؤال کند.

## نکات دیگر

- پس از نوشتن پیکربندی محلی، اگر مسیر راه‌اندازی انتخاب‌شده به Pluginهای قابل بارگیری نیاز داشته باشد، پیکربندی آن‌ها را نصب می‌کند. پیکربندی Gateway راه‌دور بسته‌های محلی Plugin را نصب نمی‌کند.
- سرویس‌های مبتنی بر کانال (Slack/Discord/Matrix/Microsoft Teams) هنگام راه‌اندازی، فهرست مجاز کانال‌ها یا اتاق‌ها را درخواست می‌کنند. می‌توانید نام یا شناسه وارد کنید؛ راهنما در صورت امکان نام‌ها را به شناسه تبدیل می‌کند.
- اگر مرحلهٔ نصب دیمن را اجرا کنید، احراز هویت توکنی به توکن نیاز دارد. اگر `gateway.auth.token` با SecretRef مدیریت شود، پیکربندی SecretRef را اعتبارسنجی می‌کند، اما مقادیر متن سادهٔ توکنِ حل‌شده را در فرادادهٔ محیط سرویس ناظر ذخیره نمی‌کند؛ اگر SecretRef حل‌نشده باشد، پیکربندی نصب دیمن را مسدود می‌کند و راهنمای عملی برای رفع مشکل ارائه می‌دهد.
- اگر هم `gateway.auth.token` و هم `gateway.auth.password` پیکربندی شده باشند و `gateway.auth.mode` تنظیم نشده باشد، پیکربندی تا زمانی که حالت را صریحاً تعیین نکنید، نصب دیمن را مسدود می‌کند.
