# ثبت گزارش

برای نمای کلی کاربرمحور (CLI + رابط کاربری کنترل + پیکربندی)، [/logging](/fa/logging) را ببینید.

OpenClaw دو «سطح» ثبت گزارش دارد:

- **خروجی کنسول** (چیزی که در ترمینال / رابط کاربری اشکال‌زدایی می‌بینید).
- **گزارش‌های فایلی** (خطوط JSON) که توسط ثبت‌کنندهٔ گزارش Gateway نوشته می‌شوند.

## ثبت‌کنندهٔ گزارش مبتنی بر فایل

- فایل گزارش چرخشی پیش‌فرض زیر `/tmp/openclaw/` است (یک فایل برای هر روز): `openclaw-YYYY-MM-DD.log`
  - تاریخ از منطقهٔ زمانی محلی میزبان Gateway استفاده می‌کند.
- فایل‌های گزارش فعال در `logging.maxFileBytes` می‌چرخند (پیش‌فرض: 100 MB)، تا
  پنج بایگانی شماره‌گذاری‌شده را نگه می‌دارند و نوشتن در یک فایل فعال تازه را ادامه می‌دهند.
- مسیر و سطح فایل گزارش را می‌توان از طریق `~/.openclaw/openclaw.json` پیکربندی کرد:
  - `logging.file`
  - `logging.level`

قالب فایل، یک شیء JSON در هر خط است.

زبانهٔ گزارش‌های رابط کاربری کنترل این فایل را از طریق Gateway دنبال می‌کند (`logs.tail`).
CLI هم می‌تواند همین کار را انجام دهد:

```bash
openclaw logs --follow
```

**پرحرفی در برابر سطح‌های گزارش**

- **گزارش‌های فایلی** فقط توسط `logging.level` کنترل می‌شوند.
- `--verbose` فقط بر **پرحرفی کنسول** (و سبک گزارش WS) اثر می‌گذارد؛ **سطح گزارش فایل را**
  افزایش نمی‌دهد.
- برای ثبت جزئیات فقط-پرحرف در گزارش‌های فایلی، `logging.level` را روی `debug` یا
  `trace` تنظیم کنید.

## ضبط کنسول

CLI موارد `console.log/info/warn/error/debug/trace` را ضبط می‌کند و آن‌ها را در گزارش‌های فایلی می‌نویسد،
در حالی که همچنان در stdout/stderr چاپ می‌کند.

می‌توانید پرحرفی کنسول را به‌صورت مستقل از این طریق تنظیم کنید:

- `logging.consoleLevel` (پیش‌فرض `info`)
- `logging.consoleStyle` (`pretty` | `compact` | `json`)

## پوشاندن اطلاعات حساس

OpenClaw می‌تواند توکن‌های حساس را پیش از خروج خروجی گزارش یا رونوشت از
فرایند ماسک کند. این سیاست پوشاندن اطلاعات حساس در گزارش‌گیری روی خروجی‌های کنسول، فایل گزارش، رکورد گزارش OTLP
و متن رونوشت نشست اعمال می‌شود، بنابراین مقادیر محرمانهٔ منطبق
پیش از نوشته‌شدن خطوط JSONL یا پیام‌ها روی دیسک ماسک می‌شوند.

- `logging.redactSensitive`: `off` | `tools` (پیش‌فرض: `tools`)
- `logging.redactPatterns`: آرایه‌ای از رشته‌های regex (پیش‌فرض‌ها را بازنویسی می‌کند)
  - از رشته‌های regex خام (خودکار `gi`) استفاده کنید، یا اگر به پرچم‌های سفارشی نیاز دارید از `/pattern/flags` استفاده کنید.
  - تطابق‌ها با نگه‌داشتن 6 نویسهٔ نخست + 4 نویسهٔ آخر ماسک می‌شوند (طول >= 18)، وگرنه `***`.
  - پیش‌فرض‌ها انتساب‌های کلید رایج، پرچم‌های CLI، فیلدهای JSON، سرآیندهای bearer، بلوک‌های PEM و پیشوندهای توکن محبوب را پوشش می‌دهند.

برخی مرزهای ایمنی همیشه بدون توجه به `logging.redactSensitive` پوشانده می‌شوند.
این شامل رویدادهای فراخوانی ابزار رابط کاربری کنترل، خروجی ابزار `sessions_history`،
خروجی‌های پشتیبانی عیب‌یابی، مشاهده‌های خطای ارائه‌دهنده، نمایش فرمان تأیید exec
و گزارش‌های پروتکل WebSocket Gateway است. این سطوح ممکن است همچنان از
`logging.redactPatterns` به‌عنوان الگوهای اضافی استفاده کنند، اما `redactSensitive: "off"`
باعث نمی‌شود اسرار خام را منتشر کنند.

## گزارش‌های WebSocket Gateway

Gateway گزارش‌های پروتکل WebSocket را در دو حالت چاپ می‌کند:

- **حالت عادی (بدون `--verbose`)**: فقط نتایج RPC «جالب» چاپ می‌شوند:
  - خطاها (`ok=false`)
  - فراخوانی‌های کند (آستانهٔ پیش‌فرض: `>= 50ms`)
  - خطاهای تجزیه
- **حالت پرحرف (`--verbose`)**: همهٔ ترافیک درخواست/پاسخ WS را چاپ می‌کند.

### سبک گزارش WS

`openclaw gateway` از یک سوییچ سبک برای هر Gateway پشتیبانی می‌کند:

- `--ws-log auto` (پیش‌فرض): حالت عادی بهینه‌سازی شده است؛ حالت پرحرف از خروجی فشرده استفاده می‌کند
- `--ws-log compact`: خروجی فشرده (درخواست/پاسخ جفت‌شده) هنگام پرحرفی
- `--ws-log full`: خروجی کامل برای هر فریم هنگام پرحرفی
- `--compact`: نام مستعار برای `--ws-log compact`

مثال‌ها:

```bash
# optimized (only errors/slow)
openclaw gateway

# show all WS traffic (paired)
openclaw gateway --verbose --ws-log compact

# show all WS traffic (full meta)
openclaw gateway --verbose --ws-log full
```

## قالب‌بندی کنسول (ثبت گزارش زیرسامانه)

قالب‌بند کنسول **آگاه از TTY** است و خطوط سازگار و پیشونددار چاپ می‌کند.
ثبت‌کننده‌های گزارش زیرسامانه خروجی را گروه‌بندی‌شده و قابل مرور نگه می‌دارند.

رفتار:

- **پیشوندهای زیرسامانه** روی هر خط (مانند `[gateway]`، `[canvas]`، `[tailscale]`)
- **رنگ‌های زیرسامانه** (پایدار برای هر زیرسامانه) همراه با رنگ‌آمیزی سطح
- **رنگ وقتی خروجی TTY است یا محیط شبیه یک ترمینال غنی است** (`TERM`/`COLORTERM`/`TERM_PROGRAM`)، با رعایت `NO_COLOR`
- **پیشوندهای کوتاه‌شدهٔ زیرسامانه**: `gateway/` + `channels/` ابتدایی را حذف می‌کند و 2 بخش آخر را نگه می‌دارد (مانند `whatsapp/outbound`)
- **زیرثبت‌کننده‌ها بر اساس زیرسامانه** (پیشوند خودکار + فیلد ساخت‌یافتهٔ `{ subsystem }`)
- **`logRaw()`** برای خروجی QR/UX (بدون پیشوند، بدون قالب‌بندی)
- **سبک‌های کنسول** (مانند `pretty | compact | json`)
- **سطح گزارش کنسول** جدا از سطح گزارش فایل (وقتی `logging.level` روی `debug`/`trace` تنظیم شده باشد، فایل جزئیات کامل را نگه می‌دارد)
- **بدنهٔ پیام‌های WhatsApp** در سطح `debug` ثبت می‌شوند (برای دیدن آن‌ها از `--verbose` استفاده کنید)

این کار گزارش‌های فایلی موجود را پایدار نگه می‌دارد و در عین حال خروجی تعاملی را قابل مرور می‌کند.
