OpenClaw از [SearXNG](https://docs.searxng.org/) به‌عنوان ارائه‌دهنده `web_search` **خودمیزبان و
بدون کلید** پشتیبانی می‌کند. SearXNG یک موتور فراجست‌وجوی متن‌باز است
که نتایج را از Google، Bing، DuckDuckGo و منابع دیگر گردآوری می‌کند.

مزایا:

- **رایگان و نامحدود** -- بدون نیاز به کلید API یا اشتراک تجاری
- **حریم خصوصی / جداسازی شبکه‌ای** -- پرس‌وجوها هرگز از شبکه شما خارج نمی‌شوند
- **قابل استفاده در همه‌جا** -- بدون محدودیت منطقه‌ای در APIهای جست‌وجوی تجاری

## راه‌اندازی

<Steps>
  <Step title="نصب Plugin">
    ```bash
    openclaw plugins install @openclaw/searxng-plugin
    ```
  </Step>
  <Step title="اجرای یک نمونه SearXNG">
    ```bash
    docker run -d -p 8888:8080 searxng/searxng
    ```

    یا از هر استقرار SearXNG موجودی که به آن دسترسی دارید استفاده کنید. برای راه‌اندازی تولید،
    [مستندات SearXNG](https://docs.searxng.org/) را ببینید.

  </Step>
  <Step title="پیکربندی">
    ```bash
    openclaw configure --section web
    # Select "searxng" as the provider
    ```

    یا متغیر محیطی را تنظیم کنید و بگذارید تشخیص خودکار آن را پیدا کند:

    ```bash
    export SEARXNG_BASE_URL="http://localhost:8888"
    ```

  </Step>
</Steps>

## پیکربندی

```json5
{
  tools: {
    web: {
      search: {
        provider: "searxng",
      },
    },
  },
}
```

تنظیمات سطح Plugin برای نمونه SearXNG:

```json5
{
  plugins: {
    entries: {
      searxng: {
        config: {
          webSearch: {
            baseUrl: "http://localhost:8888",
            categories: "general,news", // optional
            language: "en", // optional
          },
        },
      },
    },
  },
}
```

فیلد `baseUrl` همچنین اشیای SecretRef را می‌پذیرد.

قواعد انتقال:

- `https://` برای میزبان‌های عمومی یا خصوصی SearXNG کار می‌کند
- `http://` فقط برای میزبان‌های مورداعتمادِ شبکه خصوصی یا loopback پذیرفته می‌شود
- میزبان‌های عمومی SearXNG باید از `https://` استفاده کنند
- میزبان‌های خصوصی/داخلی از محافظ شبکه خودمیزبان استفاده می‌کنند؛ میزبان‌های عمومی `https://`
  روی محافظ سخت‌گیرانه جست‌وجوی وب می‌مانند و نمی‌توانند به نشانی‌های خصوصی
  تغییرمسیر دهند

## متغیر محیطی

`SEARXNG_BASE_URL` را به‌عنوان جایگزین پیکربندی تنظیم کنید:

```bash
export SEARXNG_BASE_URL="http://localhost:8888"
```

وقتی `SEARXNG_BASE_URL` تنظیم شده باشد و هیچ ارائه‌دهنده صریحی پیکربندی نشده باشد، تشخیص خودکار
SearXNG را به‌طور خودکار انتخاب می‌کند (با پایین‌ترین اولویت -- هر ارائه‌دهنده متکی به API که
کلید داشته باشد ابتدا برنده می‌شود).

## مرجع پیکربندی Plugin

| فیلد         | توضیح                                                               |
| ------------ | ------------------------------------------------------------------ |
| `baseUrl`    | URL پایه نمونه SearXNG شما (الزامی)                                |
| `categories` | دسته‌های جداشده با ویرگول مانند `general`، `news`، یا `science`    |
| `language`   | کد زبان برای نتایج، مانند `en`، `de`، یا `fr`                      |

## نکات

- **JSON API** -- از نقطه پایانی بومی `format=json` در SearXNG استفاده می‌کند، نه استخراج از HTML
- **URLهای نتایج تصویر** -- نتایج دسته تصویر وقتی SearXNG
  یک URL مستقیم تصویر برگرداند شامل `img_src` هستند
- **بدون کلید API** -- با هر نمونه SearXNG بدون تنظیمات اضافی کار می‌کند
- **اعتبارسنجی URL پایه** -- `baseUrl` باید یک URL معتبر `http://` یا `https://`
  باشد؛ میزبان‌های عمومی باید از `https://` استفاده کنند
- **محافظ شبکه** -- نقاط پایانی خصوصی/داخلی SearXNG دسترسی
  شبکه خصوصی را صریحاً فعال می‌کنند؛ نقاط پایانی عمومی `https://` مربوط به SearXNG حفاظت سخت‌گیرانه SSRF
  را حفظ می‌کنند
- **ترتیب تشخیص خودکار** -- SearXNG بعد از ارائه‌دهندگان متکی به API
  با کلیدهای پیکربندی‌شده بررسی می‌شود (ترتیب 200). ارائه‌دهندگان بدون کلید مانند DuckDuckGo یا
  Ollama Web Search بدون انتخاب صریح ارائه‌دهنده به‌طور خودکار انتخاب نمی‌شوند
- **خودمیزبان** -- شما نمونه، پرس‌وجوها، و موتورهای جست‌وجوی بالادستی را کنترل می‌کنید
- **دسته‌ها** وقتی پیکربندی نشده باشند به‌طور پیش‌فرض `general` هستند
- **بازگشت دسته** -- اگر درخواست دسته‌ای غیر از `general` موفق شود اما
  هیچ نتیجه‌ای برنگرداند، OpenClaw همان پرس‌وجو را یک بار با `general`
  دوباره امتحان می‌کند و سپس مجموعه نتیجه خالی را برمی‌گرداند

<Tip>
  برای اینکه SearXNG JSON API کار کند، مطمئن شوید نمونه SearXNG شما قالب `json`
  را در `settings.yml` زیر `search.formats` فعال کرده است.
</Tip>
