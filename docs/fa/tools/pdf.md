`pdf` یک یا چند سند PDF را تحلیل می‌کند و متن برمی‌گرداند.

رفتار سریع:

- حالت ارائه‌دهنده بومی برای ارائه‌دهندگان مدل Anthropic و Google.
- حالت جایگزینِ استخراج برای ارائه‌دهندگان دیگر (ابتدا متن را استخراج می‌کند، سپس در صورت نیاز تصاویر صفحه را).
- از ورودی تکی (`pdf`) یا چندتایی (`pdfs`) پشتیبانی می‌کند، با حداکثر ۱۰ PDF در هر فراخوانی.

## در دسترس بودن

این ابزار فقط زمانی ثبت می‌شود که OpenClaw بتواند یک پیکربندی مدلِ دارای قابلیت PDF را برای عامل حل کند:

1. `agents.defaults.pdfModel`
2. جایگزینی به `agents.defaults.imageModel`
3. جایگزینی به مدل جلسه/پیش‌فرضِ حل‌شده عامل
4. اگر ارائه‌دهندگان PDF بومی با احراز هویت پشتیبانی شوند، آن‌ها را پیش از نامزدهای جایگزین تصویر عمومی ترجیح می‌دهد

اگر هیچ مدل قابل استفاده‌ای قابل حل نباشد، ابزار `pdf` ارائه نمی‌شود.

نکات در دسترس بودن:

- زنجیره جایگزین نسبت به احراز هویت آگاه است. یک `provider/model` پیکربندی‌شده فقط زمانی حساب می‌شود که
  OpenClaw واقعا بتواند آن ارائه‌دهنده را برای عامل احراز هویت کند.
- ارائه‌دهندگان PDF بومی در حال حاضر **Anthropic** و **Google** هستند.
- اگر ارائه‌دهنده جلسه/پیش‌فرضِ حل‌شده از قبل یک مدل بینایی/PDF
  پیکربندی‌شده داشته باشد، ابزار PDF پیش از جایگزینی به ارائه‌دهندگان دیگرِ دارای
  پشتیبانی احراز هویت، همان را دوباره استفاده می‌کند.

## مرجع ورودی

<ParamField path="pdf" type="string">
یک مسیر یا URL مربوط به PDF.
</ParamField>

<ParamField path="pdfs" type="string[]">
چند مسیر یا URL مربوط به PDF، در مجموع تا ۱۰ مورد.
</ParamField>

<ParamField path="prompt" type="string" default="Analyze this PDF document.">
درخواست تحلیل.
</ParamField>

<ParamField path="pages" type="string">
فیلتر صفحه مانند `1-5` یا `1,3,7-9`.
</ParamField>

<ParamField path="model" type="string">
بازنویسی اختیاری مدل در قالب `provider/model`.
</ParamField>

<ParamField path="maxBytesMb" type="number">
سقف اندازه برای هر PDF بر حسب MB. مقدار پیش‌فرض `agents.defaults.pdfMaxBytesMb` یا `10` است.
</ParamField>

نکات ورودی:

- `pdf` و `pdfs` پیش از بارگذاری ادغام می‌شوند و موارد تکراری‌شان حذف می‌شود.
- اگر هیچ ورودی PDF ارائه نشود، ابزار خطا می‌دهد.
- `pages` به‌عنوان شماره صفحه‌های مبتنی بر ۱ تجزیه می‌شود، موارد تکراری حذف می‌شوند، مرتب می‌شود، و به حداکثر صفحه‌های پیکربندی‌شده محدود می‌شود.
- مقدار پیش‌فرض `maxBytesMb` برابر با `agents.defaults.pdfMaxBytesMb` یا `10` است.

## ارجاع‌های PDF پشتیبانی‌شده

- مسیر فایل محلی (شامل گسترش `~`)
- URL از نوع `file://`
- URL از نوع `http://` و `https://`
- ارجاع‌های ورودی مدیریت‌شده توسط OpenClaw مانند `media://inbound/<id>`

نکات ارجاع:

- طرح‌های URI دیگر (برای مثال `ftp://`) با `unsupported_pdf_reference` رد می‌شوند.
- در حالت سندباکس، URLهای راه‌دور `http(s)` رد می‌شوند.
- با فعال بودن سیاست فایل فقط-فضای‌کار، مسیرهای فایل محلی بیرون از ریشه‌های مجاز رد می‌شوند.
- ارجاع‌های ورودی مدیریت‌شده و مسیرهای بازپخش‌شده زیر انبار رسانه ورودی OpenClaw با سیاست فایل فقط-فضای‌کار مجاز هستند.

## حالت‌های اجرا

### حالت ارائه‌دهنده بومی

حالت بومی برای ارائه‌دهنده `anthropic` و `google` استفاده می‌شود.
این ابزار بایت‌های خام PDF را مستقیما به APIهای ارائه‌دهنده می‌فرستد.

محدودیت‌های حالت بومی:

- `pages` پشتیبانی نمی‌شود. اگر تنظیم شود، ابزار خطا برمی‌گرداند.
- ورودی چند PDF پشتیبانی می‌شود؛ هر PDF پیش از پرامپت، به‌صورت یک بلوک سند بومی /
  بخش PDF درون‌خطی ارسال می‌شود.

### حالت جایگزین استخراج

حالت جایگزین برای ارائه‌دهندگان غیربومی استفاده می‌شود.

جریان کار:

1. متن را از صفحه‌های انتخاب‌شده استخراج کنید (تا سقف `agents.defaults.pdfMaxPages`، پیش‌فرض `20`).
2. اگر طول متن استخراج‌شده کمتر از `200` نویسه باشد، صفحه‌های انتخاب‌شده را به تصویر PNG رندر کنید و آن‌ها را اضافه کنید.
3. محتوای استخراج‌شده به‌همراه پرامپت را به مدل انتخاب‌شده ارسال کنید.

جزئیات جایگزین:

- استخراج تصویر صفحه از بودجهٔ پیکسلی `4,000,000` استفاده می‌کند.
- اگر مدل مقصد از ورودی تصویر پشتیبانی نکند و هیچ متن قابل‌استخراجی وجود نداشته باشد، ابزار خطا می‌دهد.
- اگر استخراج متن موفق شود اما استخراج تصویر روی یک مدل فقط‌متنی به بینایی نیاز داشته باشد، OpenClaw تصویرهای رندرشده را حذف می‌کند و با متن استخراج‌شده ادامه می‌دهد.
- جایگزین استخراج از Plugin همراه `document-extract` استفاده می‌کند. مالک `pdfjs-dist` خود Plugin است؛ `@napi-rs/canvas` فقط زمانی استفاده می‌شود که جایگزین رندر تصویر در دسترس باشد.

## پیکربندی

```json5
{
  agents: {
    defaults: {
      pdfModel: {
        primary: "anthropic/claude-opus-4-6",
        fallbacks: ["openai/gpt-5.4-mini"],
      },
      pdfMaxBytesMb: 10,
      pdfMaxPages: 20,
    },
  },
}
```

برای جزئیات کامل فیلدها، [مرجع پیکربندی](/fa/gateway/configuration-reference) را ببینید.

## جزئیات خروجی

ابزار متن را در `content[0].text` و فرادادهٔ ساختاریافته را در `details` برمی‌گرداند.

فیلدهای رایج `details`:

- `model`: ارجاع مدل حل‌شده (`provider/model`)
- `native`: برای حالت ارائه‌دهندهٔ بومی `true`، برای جایگزین `false`
- `attempts`: تلاش‌های جایگزینی که پیش از موفقیت شکست خورده‌اند

فیلدهای مسیر:

- ورودی PDF تکی: `details.pdf`
- ورودی‌های PDF متعدد: `details.pdfs[]` با ورودی‌های `pdf`
- فرادادهٔ بازنویسی مسیر sandbox (در صورت کاربرد): `rewrittenFrom`

## رفتار خطا

- ورودی PDF موجود نیست: `pdf required: provide a path or URL to a PDF document` را پرتاب می‌کند
- تعداد PDFها بیش از حد است: خطای ساختاریافته را در `details.error = "too_many_pdfs"` برمی‌گرداند
- طرح ارجاع پشتیبانی‌نشده: `details.error = "unsupported_pdf_reference"` را برمی‌گرداند
- حالت بومی با `pages`: خطای واضح `pages is not supported with native PDF providers` را پرتاب می‌کند

## نمونه‌ها

PDF تکی:

```json
{
  "pdf": "/tmp/report.pdf",
  "prompt": "Summarize this report in 5 bullets"
}
```

چند PDF:

```json
{
  "pdfs": ["/tmp/q1.pdf", "/tmp/q2.pdf"],
  "prompt": "Compare risks and timeline changes across both documents"
}
```

مدل جایگزین با فیلتر صفحه:

```json
{
  "pdf": "https://example.com/report.pdf",
  "pages": "1-3,7",
  "model": "openai/gpt-5.4-mini",
  "prompt": "Extract only customer-impacting incidents"
}
```
