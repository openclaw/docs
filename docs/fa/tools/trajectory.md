ضبط مسیر حرکت، جعبه‌سیاه هر نشست در OpenClaw است. این قابلیت یک
خط زمانی ساخت‌یافته برای هر اجرای agent ثبت می‌کند، سپس `/export-trajectory`
نشست فعلی را در یک بستهٔ پشتیبانی پاک‌سازی‌شده بسته‌بندی می‌کند.

وقتی نیاز دارید به پرسش‌هایی مثل موارد زیر پاسخ دهید، از آن استفاده کنید:

- چه پرامپت، پرامپت سیستم، و ابزارهایی به مدل ارسال شدند؟
- کدام پیام‌های رونویس و فراخوانی‌های ابزار به این پاسخ منجر شدند؟
- آیا اجرا timeout شد، لغو شد، Compaction انجام داد، یا به خطای ارائه‌دهنده برخورد کرد؟
- کدام مدل، Plugin‌ها، Skills، و تنظیمات runtime فعال بودند؟
- ارائه‌دهنده چه فراداده‌ای دربارهٔ مصرف و حافظهٔ نهان پرامپت برگرداند؟

اگر در حال ثبت یک گزارش پشتیبانی گسترده برای مشکل زندهٔ Gateway هستید، با
[`/diagnostics`](/fa/gateway/diagnostics#chat-command) شروع کنید. Diagnostics بستهٔ
پاک‌سازی‌شدهٔ Gateway را جمع‌آوری می‌کند و، برای نشست‌های OpenAI Codex harness،
می‌تواند پس از تأیید، بازخورد Codex را نیز به سرورهای OpenAI ارسال کند. وقتی
به‌طور مشخص به خط زمانی تفصیلی پرامپت، ابزار، و رونویس برای هر نشست نیاز دارید
از `/export-trajectory` استفاده کنید.

## شروع سریع

این را در نشست فعال ارسال کنید:

```text
/export-trajectory
```

نام مستعار:

```text
/trajectory
```

OpenClaw بسته را زیر workspace می‌نویسد:

```text
.openclaw/trajectory-exports/openclaw-trajectory-<session>-<timestamp>/
```

می‌توانید یک نام دایرکتوری خروجی نسبی انتخاب کنید:

```text
/export-trajectory bug-1234
```

مسیر سفارشی داخل `.openclaw/trajectory-exports/` resolve می‌شود. مسیرهای مطلق
و مسیرهای `~` رد می‌شوند.

بسته‌های مسیر حرکت می‌توانند شامل پرامپت‌ها، پیام‌های مدل، schemaهای ابزار،
نتایج ابزار، رویدادهای runtime، و مسیرهای محلی باشند. بنابراین فرمان slash در
chat هر بار از مسیر تأیید exec عبور می‌کند. وقتی قصد ایجاد بسته را دارید، export
را یک‌بار تأیید کنید؛ از allow-all استفاده نکنید. در chatهای گروهی، OpenClaw
پرامپت تأیید و نتیجهٔ export را به‌صورت خصوصی برای مالک می‌فرستد، نه اینکه
جزئیات مسیر حرکت را به اتاق مشترک برگرداند.

برای بررسی محلی یا جریان‌های کاری پشتیبانی، می‌توانید مسیر فرمان تأییدشده را
مستقیماً نیز اجرا کنید:

```bash
openclaw sessions export-trajectory --session-key "agent:main:telegram:direct:123" --workspace .
```

## دسترسی

export مسیر حرکت یک فرمان مالک است. فرستنده باید بررسی‌های عادی مجوز فرمان و
بررسی‌های مالک برای channel را بگذراند.

## چه چیزهایی ثبت می‌شود

ضبط مسیر حرکت به‌صورت پیش‌فرض برای اجراهای agent در OpenClaw فعال است.

رویدادهای runtime شامل این موارد هستند:

- `session.started`
- `trace.metadata`
- `context.compiled`
- `prompt.submitted`
- `model.fallback_step`، شامل مدل مبدأ، مدل بعدی، دلیل/جزئیات شکست، موقعیت در زنجیره، و اینکه fallback زنجیره را جلو برد، موفق شد، یا زنجیره را تمام کرد
- `model.completed`
- `trace.artifacts`
- `session.ended`

رویدادهای رونویس نیز از شاخهٔ نشست فعال بازسازی می‌شوند:

- پیام‌های کاربر
- پیام‌های assistant
- فراخوانی‌های ابزار
- نتایج ابزار
- Compactionها
- تغییرات مدل
- labelها و ورودی‌های سفارشی نشست

رویدادها به‌صورت JSON Lines با این نشانگر schema نوشته می‌شوند:

```json
{
  "traceSchema": "openclaw-trajectory",
  "schemaVersion": 1
}
```

## فایل‌های بسته

یک بستهٔ export‌شده می‌تواند شامل این موارد باشد:

| فایل                  | محتوا                                                                                       |
| --------------------- | ---------------------------------------------------------------------------------------------- |
| `manifest.json`       | schema بسته، فایل‌های مبدأ، شمارش رویدادها، و فهرست فایل‌های تولیدشده                             |
| `events.jsonl`        | خط زمانی مرتب‌شدهٔ runtime و رونویس                                                        |
| `session-branch.json` | شاخهٔ رونویس فعال پاک‌سازی‌شده و header نشست                                           |
| `metadata.json`       | نسخهٔ OpenClaw، OS/runtime، مدل، snapshot پیکربندی، Plugin‌ها، Skills، و فرادادهٔ پرامپت     |
| `artifacts.json`      | وضعیت نهایی، خطاها، مصرف، حافظهٔ نهان پرامپت، تعداد Compaction، متن assistant، و فرادادهٔ ابزار |
| `prompts.json`        | پرامپت‌های ارسال‌شده و جزئیات انتخاب‌شدهٔ ساخت پرامپت                                         |
| `system-prompt.txt`   | آخرین پرامپت سیستم کامپایل‌شده، وقتی ضبط شده باشد                                                   |
| `tools.json`          | تعریف‌های ابزار ارسال‌شده به مدل، وقتی ضبط شده باشد                                              |

`manifest.json` فایل‌های موجود در آن بسته را فهرست می‌کند. وقتی نشست دادهٔ
runtime متناظر را ضبط نکرده باشد، بعضی فایل‌ها حذف می‌شوند.

## محل ضبط

به‌صورت پیش‌فرض، رویدادهای مسیر حرکت runtime کنار فایل نشست نوشته می‌شوند:

```text
<session>.trajectory.jsonl
```

OpenClaw همچنین یک فایل اشاره‌گر best-effort کنار نشست می‌نویسد:

```text
<session>.trajectory-path.json
```

برای ذخیرهٔ sidecarهای مسیر حرکت runtime در یک دایرکتوری اختصاصی،
`OPENCLAW_TRAJECTORY_DIR` را تنظیم کنید:

```bash
export OPENCLAW_TRAJECTORY_DIR=/var/lib/openclaw/trajectories
```

وقتی این متغیر تنظیم شده باشد، OpenClaw برای هر شناسهٔ نشست یک فایل JSONL در
آن دایرکتوری می‌نویسد.

نگهداری نشست‌ها، sidecarهای مسیر حرکت را وقتی ورودی نشست مالک آن‌ها prune،
capped، یا به‌دلیل بودجهٔ دیسک نشست‌ها evict شود حذف می‌کند. فایل‌های runtime
بیرون از دایرکتوری نشست‌ها فقط وقتی حذف می‌شوند که هدف اشاره‌گر هنوز ثابت کند
به همان نشست تعلق دارد.

## غیرفعال کردن ضبط

پیش از شروع OpenClaw، `OPENCLAW_TRAJECTORY=0` را تنظیم کنید:

```bash
export OPENCLAW_TRAJECTORY=0
```

این کار ضبط مسیر حرکت runtime را غیرفعال می‌کند. `/export-trajectory` همچنان
می‌تواند شاخهٔ رونویس را export کند، اما فایل‌های فقط-runtime مانند context
کامپایل‌شده، artifactهای ارائه‌دهنده، و فرادادهٔ پرامپت ممکن است وجود نداشته
باشند.

## حریم خصوصی و محدودیت‌ها

بسته‌های مسیر حرکت برای پشتیبانی و اشکال‌زدایی طراحی شده‌اند، نه انتشار عمومی.
OpenClaw پیش از نوشتن فایل‌های export، مقدارهای حساس را پاک‌سازی می‌کند:

- اعتبارنامه‌ها و فیلدهای payload شناخته‌شده شبیه secret
- داده‌های تصویر
- مسیرهای state محلی
- مسیرهای workspace، جایگزین‌شده با `$WORKSPACE_DIR`
- مسیرهای دایرکتوری home، در صورت تشخیص

exporter اندازهٔ ورودی را نیز محدود می‌کند:

- فایل‌های sidecar runtime: 50 MiB
- فایل‌های نشست: 50 MiB
- رویدادهای runtime: 200,000
- مجموع رویدادهای export‌شده: 250,000
- خط‌های رویداد runtime منفرد بالاتر از 256 KiB کوتاه می‌شوند

پیش از اشتراک‌گذاری بسته‌ها بیرون از تیم خود، آن‌ها را بازبینی کنید. پاک‌سازی
best-effort است و نمی‌تواند همهٔ secretهای خاص هر برنامه را بشناسد.

## عیب‌یابی

اگر export هیچ رویداد runtime ندارد:

- تأیید کنید OpenClaw بدون `OPENCLAW_TRAJECTORY=0` شروع شده است
- بررسی کنید آیا `OPENCLAW_TRAJECTORY_DIR` به یک دایرکتوری قابل نوشتن اشاره می‌کند
- یک پیام دیگر در نشست اجرا کنید، سپس دوباره export کنید
- `manifest.json` را برای `runtimeEventCount` بررسی کنید

اگر فرمان مسیر خروجی را رد می‌کند:

- از یک نام نسبی مثل `bug-1234` استفاده کنید
- `/tmp/...` یا `~/...` ارسال نکنید
- export را داخل `.openclaw/trajectory-exports/` نگه دارید

اگر export با خطای اندازه شکست بخورد، نشست یا sidecar از محدودیت‌های ایمنی
export فراتر رفته است. یک نشست جدید شروع کنید یا یک بازتولید کوچک‌تر export کنید.
