# ورود مرورگر + ارسال در X/Twitter

## ورود دستی (توصیه‌شده)

وقتی سایتی به ورود نیاز دارد، **به‌صورت دستی وارد شوید** در نمایهٔ مرورگر **میزبان** (مرورگر openclaw).

اطلاعات ورود خود را به مدل ندهید. ورودهای خودکار اغلب دفاع‌های ضدبات را فعال می‌کنند و می‌توانند حساب را قفل کنند.

بازگشت به مستندات اصلی مرورگر: [مرورگر](/fa/tools/browser).

## کدام نمایهٔ Chrome استفاده می‌شود؟

OpenClaw یک **نمایهٔ اختصاصی Chrome** را کنترل می‌کند (با نام `openclaw`، رابط کاربری با ته‌رنگ نارنجی). این نمایه از نمایهٔ مرورگر روزمرهٔ شما جدا است.

برای فراخوانی‌های ابزار مرورگر عامل:

- انتخاب پیش‌فرض: عامل باید از مرورگر ایزولهٔ `openclaw` خودش استفاده کند.
- فقط وقتی از `profile="user"` استفاده کنید که نشست‌های واردشدهٔ موجود اهمیت دارند و کاربر پشت رایانه حاضر است تا هر اعلان اتصال را کلیک/تأیید کند.
- اگر چند نمایهٔ مرورگر کاربر دارید، به‌جای حدس‌زدن، نمایه را صریحاً مشخص کنید.

دو راه ساده برای دسترسی به آن:

1. **از عامل بخواهید مرورگر را باز کند** و سپس خودتان وارد شوید.
2. **آن را از طریق CLI باز کنید**:

```bash
openclaw browser start
openclaw browser open https://x.com
```

اگر چند نمایه دارید، `--browser-profile <name>` را ارسال کنید (پیش‌فرض `openclaw` است).

## X/Twitter: جریان پیشنهادی

- **خواندن/جست‌وجو/رشته‌ها:** از مرورگر **میزبان** استفاده کنید (ورود دستی).
- **ارسال به‌روزرسانی‌ها:** از مرورگر **میزبان** استفاده کنید (ورود دستی).

## سندباکس‌سازی + دسترسی به مرورگر میزبان

نشست‌های مرورگر سندباکس‌شده **احتمال بیشتری** دارند که تشخیص بات را فعال کنند. برای X/Twitter (و سایت‌های سخت‌گیر دیگر)، مرورگر **میزبان** را ترجیح دهید.

اگر عامل سندباکس‌شده باشد، ابزار مرورگر به‌طور پیش‌فرض از سندباکس استفاده می‌کند. برای اجازه‌دادن به کنترل میزبان:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

سپس مرورگر میزبان را هدف بگیرید:

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

یا سندباکس‌سازی را برای عاملی که به‌روزرسانی‌ها را ارسال می‌کند غیرفعال کنید.
