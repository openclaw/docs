## ورود دستی (توصیه‌شده)

هنگامی که سایتی نیاز به ورود دارد، به‌صورت دستی در پروفایل `openclaw` مرورگر میزبان وارد شوید. اطلاعات احراز هویت خود را در اختیار مدل قرار ندهید: ورودهای خودکار اغلب سازوکارهای دفاعی ضدربات را فعال می‌کنند و ممکن است باعث قفل‌شدن حساب شوند.

برای خواندن (جست‌وجو/رشته‌گفت‌وگوها) و انتشار مطلب در X/Twitter و دیگر سایت‌های حساس به ربات، از مرورگر میزبان (با ورود دستی) استفاده کنید. نشست‌های مرورگر سندباکس‌شده احتمال بیشتری دارد که سامانه‌های تشخیص ربات را فعال کنند.

بازگشت به مستندات اصلی مرورگر: [مرورگر](/fa/tools/browser).

## از کدام پروفایل Chrome استفاده می‌شود؟

OpenClaw یک پروفایل اختصاصی Chrome با نام `openclaw` (با رابط کاربری نارنجی‌رنگ) را کنترل می‌کند که از پروفایل روزمره مرورگر شما جدا است.

برای فراخوانی‌های ابزار مرورگر عامل:

- انتخاب پیش‌فرض: عامل از مرورگر ایزوله `openclaw` خود استفاده می‌کند.
- فقط زمانی از `profile="user"` استفاده کنید که نشست‌های موجودِ واردشده اهمیت دارند و برای کلیک‌کردن/تأیید هرگونه درخواست اتصال، پشت رایانه حضور دارید.
- اگر چند پروفایل مرورگر کاربر دارید، به‌جای حدس‌زدن، پروفایل را صریحاً مشخص کنید.

دو روش برای دسترسی به پروفایل `openclaw`:

1. از عامل بخواهید مرورگر را باز کند، سپس خودتان وارد شوید.
2. آن را از طریق CLI باز کنید:

```bash
openclaw browser start
openclaw browser open https://x.com
```

برای پروفایلی غیر از پیش‌فرض، `--browser-profile <name>` را پیش از زیرفرمان قرار دهید (پیش‌فرض `openclaw` است):

```bash
openclaw browser --browser-profile <name> open https://x.com
```

## سندباکس‌سازی: اجازه دسترسی به مرورگر میزبان

اگر عامل در سندباکس اجرا شود، فراخوانی‌های ابزار `browser` آن به‌طور پیش‌فرض مرورگر سندباکس را هدف قرار می‌دهند، نه مرورگر میزبان. برای اینکه عامل بتواند به‌جای آن مرورگر میزبان را هدف قرار دهد:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

فراخوانی‌های CLI همیشه مرورگر میزبان را هدف قرار می‌دهند، نه سندباکس؛ بنابراین صرف‌نظر از این تنظیم، می‌توانید خودتان مرورگر میزبان را باز کنید:

```bash
openclaw browser --browser-profile openclaw open https://x.com
```

پس از تنظیم `sandbox.browser.allowHostControl: true`، فراخوانی‌های ابزار `browser` عامل نیز می‌توانند مرورگر میزبان را هدف قرار دهند. همچنین می‌توانید سندباکس‌سازی را برای عاملی که به‌روزرسانی‌ها را منتشر می‌کند غیرفعال کنید.
