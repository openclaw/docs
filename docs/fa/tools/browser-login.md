## ورود دستی (توصیه‌شده)

وقتی سایتی به ورود نیاز دارد، در نمایهٔ مرورگر **میزبان** (مرورگر openclaw) **به‌صورت دستی وارد شوید**.

**اعتبارنامه‌های خود را به مدل ندهید**. ورودهای خودکار اغلب سامانه‌های ضدربات را فعال می‌کنند و ممکن است حساب را قفل کنند.

بازگشت به مستندات اصلی مرورگر: [مرورگر](/fa/tools/browser).

## از کدام نمایهٔ Chrome استفاده می‌شود؟

OpenClaw یک **نمایهٔ اختصاصی Chrome** را کنترل می‌کند (با نام `openclaw` و رابط کاربری با ته‌رنگ نارنجی). این نمایه از نمایهٔ مرورگر روزمرهٔ شما جدا است.

برای فراخوانی‌های ابزار مرورگر عامل:

- انتخاب پیش‌فرض: عامل باید از مرورگر ایزولهٔ `openclaw` خودش استفاده کند.
- فقط زمانی از `profile="user"` استفاده کنید که نشست‌های واردشدهٔ موجود مهم باشند و کاربر پشت رایانه باشد تا هر اعلان اتصال را کلیک/تأیید کند.
- اگر چند نمایهٔ مرورگر کاربر دارید، به‌جای حدس زدن، نمایه را صریحاً مشخص کنید.

دو روش ساده برای دسترسی به آن:

1. **از عامل بخواهید مرورگر را باز کند** و سپس خودتان وارد شوید.
2. **آن را از طریق CLI باز کنید**:

```bash
openclaw browser start
openclaw browser open https://x.com
```

اگر چند نمایه دارید، `--browser-profile <name>` را ارسال کنید (پیش‌فرض `openclaw` است).

## X/Twitter: روند توصیه‌شده

- **خواندن/جست‌وجو/رشته‌ها:** از مرورگر **میزبان** استفاده کنید (ورود دستی).
- **ارسال به‌روزرسانی‌ها:** از مرورگر **میزبان** استفاده کنید (ورود دستی).

## Sandboxing + دسترسی به مرورگر میزبان

نشست‌های مرورگر sandboxed **با احتمال بیشتری** تشخیص ربات را فعال می‌کنند. برای X/Twitter (و سایت‌های سخت‌گیر دیگر)، مرورگر **میزبان** را ترجیح دهید.

اگر عامل sandboxed باشد، ابزار مرورگر به‌صورت پیش‌فرض از sandbox استفاده می‌کند. برای اجازه دادن به کنترل میزبان:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

سپس مرورگر میزبان را هدف بگیرید:

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

یا sandboxing را برای عاملی که به‌روزرسانی‌ها را ارسال می‌کند غیرفعال کنید.
