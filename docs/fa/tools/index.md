هر کاری که عامل فراتر از تولید متن انجام می‌دهد از طریق **ابزارها** رخ می‌دهد.
ابزارها روشی هستند که عامل با آن‌ها فایل‌ها را می‌خواند، دستورها را اجرا می‌کند، وب را مرور می‌کند، پیام‌ها را ارسال می‌کند و با دستگاه‌ها تعامل دارد.

## ابزارها، Skills و Pluginها

OpenClaw سه لایه دارد که با هم کار می‌کنند:

<Steps>
  <Step title="ابزارها چیزهایی هستند که عامل فراخوانی می‌کند">
    ابزار یک تابع نوع‌دار است که عامل می‌تواند آن را فراخوانی کند (برای مثال `exec`، `browser`،
    `web_search`، `message`). OpenClaw مجموعه‌ای از **ابزارهای داخلی** ارائه می‌کند و
    Pluginها می‌توانند موارد بیشتری ثبت کنند.

    عامل ابزارها را به‌صورت تعریف‌های تابع ساختاریافته می‌بیند که به API مدل ارسال می‌شوند.

  </Step>

  <Step title="Skills به عامل یاد می‌دهند چه زمانی و چگونه">

    استفاده مؤثر از ابزارها می‌دهند. Skills در فضای کاری شما، در پوشه‌های مشترک،
    یا داخل Pluginها قرار می‌گیرند.

    [مرجع Skills](/fa/tools/skills) | [ایجاد Skills](/fa/tools/creating-skills)

  </Step>

  <Step title="Pluginها همه‌چیز را با هم بسته‌بندی می‌کنند">

    صدای بلادرنگ، فهم رسانه، تولید تصویر، تولید ویدیو،

## ابزارهای داخلی

این ابزارها همراه OpenClaw ارائه می‌شوند و بدون نصب هیچ Pluginی در دسترس هستند:

| ابزار                                      | کاری که انجام می‌دهد                                                | صفحه                                                        |
| ------------------------------------------ | ------------------------------------------------------------------- | ----------------------------------------------------------- |
| `exec` / `process`                         | اجرای دستورهای شل، مدیریت فرایندهای پس‌زمینه                       | [Exec](/fa/tools/exec)، [تأییدهای Exec](/fa/tools/exec-approvals) |
| `code_execution`                           | اجرای تحلیل Python راه دور در محیط سندباکس                         | [اجرای کد](/fa/tools/code-execution)                           |
| `browser`                                  | کنترل مرورگر Chromium (ناوبری، کلیک، نماگرفت)                      | [مرورگر](/fa/tools/browser)                                    |
| `web_search` / `x_search` / `web_fetch`    | جست‌وجوی وب، جست‌وجوی پست‌های X، دریافت محتوای صفحه                | [وب](/fa/tools/web)، [دریافت وب](/fa/tools/web-fetch)             |
| `read` / `write` / `edit`                  | ورودی/خروجی فایل در فضای کاری                                      |                                                             |
| `apply_patch`                              | وصله‌های چندبخشی فایل                                               | [اعمال وصله](/fa/tools/apply-patch)                            |
| `message`                                  | ارسال پیام در همه کانال‌ها                                          | [ارسال عامل](/fa/tools/agent-send)                             |
| `nodes`                                    | کشف و هدف‌گیری دستگاه‌های جفت‌شده                                   |                                                             |
| `cron` / `gateway`                         | مدیریت کارهای زمان‌بندی‌شده؛ بررسی، وصله، راه‌اندازی مجدد، یا به‌روزرسانی Gateway |                                                             |
| `image` / `image_generate`                 | تحلیل یا تولید تصویر                                                | [تولید تصویر](/fa/tools/image-generation)                      |
| `music_generate`                           | تولید قطعه‌های موسیقی                                               | [تولید موسیقی](/fa/tools/music-generation)                     |
| `video_generate`                           | تولید ویدیوها                                                       | [تولید ویدیو](/fa/tools/video-generation)                      |
| `tts`                                      | تبدیل یک‌باره متن به گفتار                                          | [TTS](/fa/tools/tts)                                           |
| `sessions_*` / `subagents` / `agents_list` | مدیریت نشست، وضعیت، و هماهنگ‌سازی زیرعامل‌ها                       | [زیرعامل‌ها](/fa/tools/subagents)                              |
| `session_status`                           | بازخوانی سبک به سبک `/status` و override مدل نشست                  | [ابزارهای نشست](/fa/concepts/session-tool)                    |

برای کار با تصویر، از `image` برای تحلیل و از `image_generate` برای تولید یا ویرایش استفاده کنید. اگر `openai/*`، `google/*`، `fal/*`، یا ارائه‌دهنده تصویر غیردیفالت دیگری را هدف می‌گیرید، ابتدا احراز هویت/کلید API آن ارائه‌دهنده را پیکربندی کنید.

برای کار با موسیقی، از `music_generate` استفاده کنید. اگر `google/*`، `minimax/*`، یا ارائه‌دهنده موسیقی غیردیفالت دیگری را هدف می‌گیرید، ابتدا احراز هویت/کلید API آن ارائه‌دهنده را پیکربندی کنید.

برای کار با ویدیو، از `video_generate` استفاده کنید. اگر `qwen/*` یا ارائه‌دهنده ویدیوی غیردیفالت دیگری را هدف می‌گیرید، ابتدا احراز هویت/کلید API آن ارائه‌دهنده را پیکربندی کنید.

برای تولید صوت مبتنی بر گردش‌کار، زمانی از `music_generate` استفاده کنید که Pluginی مانند
ComfyUI آن را ثبت می‌کند. این با `tts` که متن‌به‌گفتار است جداست.

`session_status` ابزار سبک وضعیت/بازخوانی در گروه نشست‌ها است.
به پرسش‌های به سبک `/status` درباره نشست جاری پاسخ می‌دهد و می‌تواند
به‌صورت اختیاری برای هر نشست یک override مدل تنظیم کند؛ `model=default` آن
override را پاک می‌کند. مانند `/status`، می‌تواند شمارنده‌های پراکنده توکن/کش و برچسب
مدل زمان اجرای فعال را از آخرین ورودی مصرف رونوشت تکمیل کند.

`gateway` ابزار زمان اجرای فقط مالک برای عملیات Gateway است:

- `config.schema.lookup` برای یک زیردرخت پیکربندی محدود به مسیر پیش از ویرایش‌ها
- `config.get` برای اسنپ‌شات پیکربندی جاری + هش
- `config.patch` برای به‌روزرسانی‌های جزئی پیکربندی همراه با راه‌اندازی مجدد
- `config.apply` فقط برای جایگزینی کامل پیکربندی
- `update.run` برای خودبه‌روزرسانی صریح + راه‌اندازی مجدد

برای تغییرات جزئی، ابتدا `config.schema.lookup` و سپس `config.patch` را ترجیح دهید. فقط زمانی از
`config.apply` استفاده کنید که عمداً کل پیکربندی را جایگزین می‌کنید.
برای مستندات گسترده‌تر پیکربندی، [پیکربندی](/fa/gateway/configuration) و
[مرجع پیکربندی](/fa/gateway/configuration-reference) را بخوانید.
این ابزار همچنین از تغییر `tools.exec.ask` یا `tools.exec.security` خودداری می‌کند؛
نام‌های مستعار قدیمی `tools.bash.*` به همان مسیرهای exec محافظت‌شده نرمال‌سازی می‌شوند.

### ابزارهای ارائه‌شده توسط Plugin

Pluginها می‌توانند ابزارهای بیشتری ثبت کنند. چند نمونه:

- [Canvas](/fa/plugins/reference/canvas) — Plugin آزمایشی بسته‌بندی‌شده برای کنترل Canvas گره و رندر A2UI
- [Diffs](/fa/tools/diffs) — نمایشگر و رندرکننده diff
- [LLM Task](/fa/tools/llm-task) — گام LLM فقط JSON برای خروجی ساختاریافته
- [Lobster](/fa/tools/lobster) — زمان اجرای گردش‌کار نوع‌دار با تأییدهای قابل ازسرگیری
- [تولید موسیقی](/fa/tools/music-generation) — ابزار مشترک `music_generate` با ارائه‌دهندگان پشتیبانی‌شده توسط گردش‌کار
- [OpenProse](/fa/prose) — هماهنگ‌سازی گردش‌کار با اولویت مارک‌داون
- [Tokenjuice](/fa/tools/tokenjuice) — فشرده‌سازی نتایج پرنویز ابزارهای `exec` و `bash`

ابزارهای Plugin همچنان با `api.registerTool(...)` نوشته می‌شوند و در
فهرست `contracts.tools` مانیفست Plugin اعلان می‌شوند. OpenClaw توصیفگر ابزار
اعتبارسنجی‌شده را هنگام کشف ثبت می‌کند و آن را بر اساس منبع و قرارداد Plugin کش می‌کند، بنابراین
برنامه‌ریزی بعدی ابزار می‌تواند بارگذاری زمان اجرای Plugin را رد کند. اجرای ابزار همچنان
Plugin مالک را بارگذاری می‌کند و پیاده‌سازی ثبت‌شده زنده را فراخوانی می‌کند.

## پیکربندی ابزار

### فهرست‌های مجاز و غیرمجاز

کنترل کنید عامل کدام ابزارها را می‌تواند از طریق `tools.allow` / `tools.deny` در
پیکربندی فراخوانی کند. غیرمجاز همیشه بر مجاز غلبه می‌کند.

```json5
{
  tools: {
    allow: ["group:fs", "browser", "web_search"],
    deny: ["exec"],
  },
}
```

OpenClaw وقتی یک allowlist صریح به هیچ ابزار قابل‌فراخوانی‌ای نرسد، بسته شکست می‌خورد.
برای مثال، `tools.allow: ["query_db"]` فقط زمانی کار می‌کند که یک Plugin بارگذاری‌شده واقعاً
`query_db` را ثبت کند. اگر هیچ ابزار داخلی، Plugin، یا ابزار MCP بسته‌بندی‌شده‌ای با
allowlist مطابقت نداشته باشد، اجرا پیش از فراخوانی مدل متوقف می‌شود، به‌جای آن‌که به‌صورت
اجرای فقط متنی ادامه دهد که ممکن است نتایج ابزار را جعل کند.

### پروفایل‌های ابزار

`tools.profile` پیش از اعمال `allow`/`deny` یک allowlist پایه تنظیم می‌کند.
override برای هر عامل: `agents.list[].tools.profile`.

| پروفایل     | شامل چه چیزهایی است                                                                                                                               |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| `full`      | همه ابزارهای هسته‌ای و اختیاری Plugin؛ خط پایه نامحدود برای دسترسی گسترده‌تر به فرمان/کنترل                                                     |
| `coding`    | `group:fs`, `group:runtime`, `group:web`, `group:sessions`, `group:memory`, `cron`, `image`, `image_generate`, `music_generate`, `video_generate` |
| `messaging` | `group:messaging`, `sessions_list`, `sessions_history`, `sessions_send`, `session_status`                                                         |
| `minimal`   | فقط `session_status`                                                                                                                              |

<Note>
`tools.profile: "messaging"` عمداً برای عامل‌های متمرکز بر کانال محدود است.
ابزارهای گسترده‌تر فرمان/کنترل مانند فایل‌سیستم، زمان اجرا،
مرورگر، canvas، nodes، cron، و کنترل Gateway را کنار می‌گذارد. برای دسترسی گسترده‌تر
فرمان/کنترل، از `tools.profile: "full"` به‌عنوان خط پایه نامحدود استفاده کنید، سپس در صورت نیاز
دسترسی را با `tools.allow` / `tools.deny` محدود کنید.
</Note>

`coding` شامل ابزارهای سبک وب (`web_search`، `web_fetch`، `x_search`) است
اما ابزار کامل کنترل مرورگر را شامل نمی‌شود. اتوماسیون مرورگر می‌تواند نشست‌های واقعی
و پروفایل‌های واردشده را هدایت کند، بنابراین آن را صراحتاً با
`tools.alsoAllow: ["browser"]` یا برای هر عامل با
`agents.list[].tools.alsoAllow: ["browser"]` اضافه کنید.

<Note>
پیکربندی `tools.exec` یا `tools.fs` زیر یک پروفایل محدود (`messaging`، `minimal`) به‌صورت ضمنی allowlist پروفایل را گسترده‌تر نمی‌کند. وقتی می‌خواهید یک پروفایل محدود از آن بخش‌های پیکربندی‌شده استفاده کند، ورودی‌های صریح `tools.alsoAllow` اضافه کنید (برای مثال `["exec", "process"]` برای exec، یا `["read", "write", "edit"]` برای fs). وقتی یک بخش پیکربندی بدون مجوز متناظر `alsoAllow` وجود داشته باشد، OpenClaw هنگام راه‌اندازی یک هشدار ثبت می‌کند.
</Note>

پروفایل‌های `coding` و `messaging` همچنین ابزارهای MCP بسته‌بندی‌شده پیکربندی‌شده را
زیر کلید Plugin با نام `bundle-mcp` مجاز می‌کنند. وقتی می‌خواهید
یک پروفایل داخلی‌های عادی خود را نگه دارد اما همه ابزارهای MCP پیکربندی‌شده را پنهان کند، `tools.deny: ["bundle-mcp"]` را اضافه کنید.
پروفایل `minimal` شامل ابزارهای MCP بسته‌بندی‌شده نیست.

نمونه (گسترده‌ترین سطح ابزار به‌صورت پیش‌فرض):

```json5
{
  tools: {
    profile: "full",
  },
}
```

### گروه‌های ابزار

از کوتاه‌نویسی‌های `group:*` در فهرست‌های مجاز/غیرمجاز استفاده کنید:

| گروه               | ابزارها                                                                                                    |
| ------------------ | --------------------------------------------------------------------------------------------------------- |
| `group:runtime`    | exec, process, code_execution (`bash` به‌عنوان نام مستعار برای `exec` پذیرفته می‌شود)                     |
| `group:fs`         | read, write, edit, apply_patch                                                                            |
| `group:sessions`   | sessions_list, sessions_history, sessions_send, sessions_spawn, sessions_yield, subagents, session_status |
| `group:memory`     | memory_search, memory_get                                                                                 |
| `group:web`        | web_search, x_search, web_fetch                                                                           |
| `group:ui`         | browser، canvas هنگامی که Plugin همراه Canvas فعال باشد                                                   |
| `group:automation` | heartbeat_respond, cron, gateway                                                                          |
| `group:messaging`  | message                                                                                                   |
| `group:nodes`      | nodes                                                                                                     |
| `group:agents`     | agents_list, update_plan                                                                                  |
| `group:media`      | image, image_generate, music_generate, video_generate, tts                                                |
| `group:openclaw`   | همهٔ ابزارهای داخلی OpenClaw (شامل ابزارهای Plugin نمی‌شود)                                              |

`sessions_history` یک نمای یادآوری محدود و فیلترشده از نظر ایمنی برمی‌گرداند. این فرمان
برچسب‌های تفکر، داربست `<relevant-memories>`، محموله‌های XML فراخوانی ابزار به‌صورت متن ساده
(از جمله `<tool_call>...</tool_call>`،
`<function_call>...</function_call>`، `<tool_calls>...</tool_calls>`،
`<function_calls>...</function_calls>` و بلوک‌های کوتاه‌شدهٔ فراخوانی ابزار)،
داربست تنزل‌یافتهٔ فراخوانی ابزار، توکن‌های کنترلی مدل ASCII/تمام‌عرض افشاشده،
و XMLهای ناقص فراخوانی ابزار MiniMax را از متن دستیار حذف می‌کند، سپس
به‌جای عمل کردن مانند تخلیهٔ خام رونوشت، ویرایش/کوتاه‌سازی و در صورت نیاز
جای‌نگهدارهای سطرهای بیش‌ازحد بزرگ را اعمال می‌کند.

### محدودیت‌های ویژهٔ ارائه‌دهنده

از `tools.byProvider` برای محدود کردن ابزارها برای ارائه‌دهندگان مشخص بدون
تغییر پیش‌فرض‌های سراسری استفاده کنید:

```json5
{
  tools: {
    profile: "coding",
    byProvider: {
      "google-antigravity": { profile: "minimal" },
    },
  },
}
```
