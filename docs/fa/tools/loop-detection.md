OpenClaw می‌تواند مانع گیر افتادن عامل‌ها در الگوهای تکراری فراخوانی ابزار شود.
این محافظ به‌صورت **پیش‌فرض غیرفعال** است.

آن را فقط جایی فعال کنید که لازم است، زیرا با تنظیمات سخت‌گیرانه می‌تواند فراخوانی‌های تکراریِ مشروع را مسدود کند.

## چرا این وجود دارد

- شناسایی توالی‌های تکراری که پیشرفتی ایجاد نمی‌کنند.
- شناسایی حلقه‌های پرتکرارِ بدون نتیجه (همان ابزار، همان ورودی‌ها، خطاهای تکراری).
- شناسایی الگوهای خاصِ فراخوانی تکراری برای ابزارهای شناخته‌شده‌ی نظرسنجی.

## بلوک پیکربندی

پیش‌فرض‌های سراسری:

```json5
{
  tools: {
    loopDetection: {
      enabled: false,
      historySize: 30,
      warningThreshold: 10,
      criticalThreshold: 20,
      globalCircuitBreakerThreshold: 30,
      detectors: {
        genericRepeat: true,
        knownPollNoProgress: true,
        pingPong: true,
      },
    },
  },
}
```

بازنویسی برای هر عامل (اختیاری):

```json5
{
  agents: {
    list: [
      {
        id: "safe-runner",
        tools: {
          loopDetection: {
            enabled: true,
            warningThreshold: 8,
            criticalThreshold: 16,
          },
        },
      },
    ],
  },
}
```

### رفتار فیلدها

- `enabled`: کلید اصلی. `false` یعنی هیچ شناسایی حلقه‌ای انجام نمی‌شود.
- `historySize`: تعداد فراخوانی‌های اخیر ابزار که برای تحلیل نگه داشته می‌شوند.
- `warningThreshold`: آستانه پیش از طبقه‌بندی یک الگو به‌عنوان فقط-هشدار.
- `criticalThreshold`: آستانه برای مسدود کردن الگوهای حلقه‌ای تکراری.
- `globalCircuitBreakerThreshold`: آستانه‌ی مدارشکن سراسریِ بدون پیشرفت.
- `detectors.genericRepeat`: الگوهای تکراریِ ابزار یکسان + پارامترهای یکسان را شناسایی می‌کند.
- `detectors.knownPollNoProgress`: الگوهای شناخته‌شده‌ی شبیه نظرسنجی را بدون تغییر وضعیت شناسایی می‌کند.
- `detectors.pingPong`: الگوهای متناوب پینگ‌پنگ را شناسایی می‌کند.

برای `exec`، بررسی‌های بدون پیشرفت خروجی‌های پایدار فرمان را مقایسه می‌کنند و فراداده‌های ناپایدار زمان اجرا مانند مدت‌زمان، PID، شناسه نشست و دایرکتوری کاری را نادیده می‌گیرند.
وقتی شناسه اجرا در دسترس باشد، تاریخچه اخیر فراخوانی ابزار فقط در همان اجرا ارزیابی می‌شود تا چرخه‌های زمان‌بندی‌شده‌ی Heartbeat و اجراهای تازه، شمارش حلقه‌های کهنه را از اجراهای قبلی به ارث نبرند.

## راه‌اندازی پیشنهادی

- با `enabled: true` و بدون تغییر پیش‌فرض‌ها شروع کنید.
- آستانه‌ها را به‌ترتیب `warningThreshold < criticalThreshold < globalCircuitBreakerThreshold` نگه دارید.
- اگر مثبت‌های کاذب رخ داد:
  - `warningThreshold` و/یا `criticalThreshold` را افزایش دهید
  - (اختیاری) `globalCircuitBreakerThreshold` را افزایش دهید
  - فقط آشکارسازی را که مشکل ایجاد می‌کند غیرفعال کنید
  - برای زمینه تاریخی کمتر سخت‌گیرانه، `historySize` را کاهش دهید

## گزارش‌ها و رفتار مورد انتظار

وقتی حلقه‌ای شناسایی شود، OpenClaw یک رویداد حلقه را گزارش می‌کند و بسته به شدت، چرخه ابزار بعدی را مسدود یا تضعیف می‌کند.
این کار از کاربران در برابر مصرف بی‌مهار توکن و قفل‌شدن محافظت می‌کند، در حالی که دسترسی عادی به ابزار حفظ می‌شود.

- ابتدا هشدار و سرکوب موقت را ترجیح دهید.
- فقط وقتی شواهد تکراری انباشته شد، تشدید کنید.

## نکته‌ها

- `tools.loopDetection` با بازنویسی‌های سطح عامل ادغام می‌شود.
- پیکربندی هر عامل، مقادیر سراسری را به‌طور کامل بازنویسی یا گسترش می‌دهد.
- اگر پیکربندی‌ای وجود نداشته باشد، محافظ‌ها خاموش می‌مانند.
