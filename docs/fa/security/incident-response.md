# پاسخ به رخداد

## 1. شناسایی و تریاژ

ما سیگنال‌های امنیتی را از این منابع پایش می‌کنیم:

- اعلان‌های امنیتی GitHub (GHSA) و گزارش‌های خصوصی آسیب‌پذیری.
- مسئله‌ها/بحث‌های عمومی GitHub وقتی گزارش‌ها حساس نیستند.
- سیگنال‌های خودکار (برای مثال Dependabot، CodeQL، هشدارهای npm و اسکن اسرار).

تریاژ اولیه:

1. مؤلفه، نسخه و اثر مرز اعتمادِ تحت تأثیر را تأیید کنید.
2. با استفاده از دامنه و قواعد خارج از دامنه در `SECURITY.md` مخزن، آن را به‌عنوان مسئله امنیتی در برابر سخت‌سازی/بدون‌اقدام طبقه‌بندی کنید.
3. مالک رخداد متناسب با آن پاسخ می‌دهد.

## 2. ارزیابی

راهنمای شدت:

- **بحرانی:** مصالحه بسته/انتشار/مخزن، بهره‌برداری فعال، یا دور زدن بدون احراز هویتِ مرز اعتماد با کنترل پراثر یا افشای داده.
- **زیاد:** دور زدن تأییدشده مرز اعتماد که به پیش‌شرط‌های محدود نیاز دارد (برای مثال اقدام پراثرِ احراز هویت‌شده اما غیرمجاز)، یا افشای اعتبارنامه‌های حساس متعلق به OpenClaw.
- **متوسط:** ضعف امنیتی مهم با اثر عملی اما بهره‌برداری محدودشده یا پیش‌نیازهای قابل‌توجه.
- **کم:** یافته‌های دفاع در عمق، انکار سرویس با دامنه محدود، یا شکاف‌های سخت‌سازی/برابری بدون دور زدن اثبات‌شده مرز اعتماد.

## 3. پاسخ

1. دریافت گزارش را به گزارش‌دهنده تأیید کنید (در موارد حساس، به‌صورت خصوصی).
2. روی انتشارهای پشتیبانی‌شده و آخرین `main` بازتولید کنید، سپس وصله‌ای با پوشش رگرسیون پیاده‌سازی و اعتبارسنجی کنید.
3. برای رخدادهای بحرانی/زیاد، انتشار(های) وصله‌شده را تا حد عملی با بیشترین سرعت آماده کنید.
4. برای رخدادهای متوسط/کم، در جریان عادی انتشار وصله کنید و راهنمای کاهش اثر را مستند کنید.

## 4. ارتباطات

ما از این مسیرها ارتباط برقرار می‌کنیم:

- اعلان‌های امنیتی GitHub در مخزن تحت تأثیر.
- یادداشت‌های انتشار/ورودی‌های changelog برای نسخه‌های رفع‌شده.
- پیگیری مستقیم با گزارش‌دهنده درباره وضعیت و رفع مسئله.

سیاست افشا:

- رخدادهای بحرانی/زیاد باید افشای هماهنگ دریافت کنند، همراه با صدور CVE در موارد مناسب.
- یافته‌های سخت‌سازی کم‌خطر، بسته به اثر و میزان مواجهه کاربران، ممکن است در یادداشت‌های انتشار یا اعلان‌ها بدون CVE مستند شوند.

## 5. بازیابی و پیگیری

پس از ارسال رفع مشکل:

1. اصلاحات را در CI و مصنوعات انتشار تأیید کنید.
2. یک بازبینی کوتاه پس از رخداد اجرا کنید (خط زمانی، علت ریشه‌ای، شکاف شناسایی، برنامه پیشگیری).
3. کارهای پیگیری برای سخت‌سازی/آزمون‌ها/مستندات را اضافه کنید و تا تکمیل پیگیری کنید.
