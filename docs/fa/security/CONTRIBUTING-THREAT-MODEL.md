# مشارکت در مدل تهدید OpenClaw

از شما برای کمک به امن‌تر کردن OpenClaw سپاسگزاریم. این مدل تهدید یک سند زنده است و از مشارکت همه استقبال می‌کنیم - لازم نیست متخصص امنیت باشید.

## راه‌های مشارکت

### افزودن تهدید

یک بردار حمله یا ریسکی دیده‌اید که پوشش نداده‌ایم؟ یک issue در [openclaw/trust](https://github.com/openclaw/trust/issues) باز کنید و آن را با کلمات خودتان توصیف کنید. لازم نیست هیچ چارچوبی را بشناسید یا همه فیلدها را پر کنید - فقط سناریو را توضیح دهید.

**موارد مفید برای درج (اما الزامی نیست):**

- سناریوی حمله و اینکه چگونه می‌تواند مورد سوءاستفاده قرار گیرد
- کدام بخش‌های OpenClaw تحت تأثیر قرار می‌گیرند (CLI، Gateway، کانال‌ها، ClawHub، سرورهای MCP و غیره)
- فکر می‌کنید شدت آن چقدر است (کم / متوسط / زیاد / بحرانی)
- هر پیوندی به پژوهش‌های مرتبط، CVEها، یا نمونه‌های دنیای واقعی

ما نگاشت ATLAS، شناسه‌های تهدید، و ارزیابی ریسک را هنگام بازبینی انجام می‌دهیم. اگر می‌خواهید این جزئیات را اضافه کنید، عالی است - اما انتظار نمی‌رود.

> **این برای افزودن به مدل تهدید است، نه گزارش آسیب‌پذیری‌های زنده.** اگر یک آسیب‌پذیری قابل بهره‌برداری پیدا کرده‌اید، برای دستورالعمل‌های افشای مسئولانه به [صفحه اعتماد](https://trust.openclaw.ai) ما مراجعه کنید.

### پیشنهاد یک کاهش ریسک

ایده‌ای برای رسیدگی به یک تهدید موجود دارید؟ یک issue یا PR باز کنید و به آن تهدید ارجاع دهید. کاهش ریسک‌های مفید مشخص و قابل اقدام هستند - برای نمونه، «محدودسازی نرخ به‌ازای هر فرستنده به ۱۰ پیام در دقیقه در Gateway» بهتر از «محدودسازی نرخ را پیاده‌سازی کنید» است.

### پیشنهاد یک زنجیره حمله

زنجیره‌های حمله نشان می‌دهند چگونه چند تهدید با هم در یک سناریوی حمله واقع‌گرایانه ترکیب می‌شوند. اگر ترکیب خطرناکی می‌بینید، مراحل را توضیح دهید و بگویید مهاجم چگونه آن‌ها را به هم زنجیر می‌کند. یک روایت کوتاه از اینکه حمله در عمل چگونه پیش می‌رود، ارزشمندتر از یک قالب رسمی است.

### اصلاح یا بهبود محتوای موجود

اشکال‌های تایپی، شفاف‌سازی‌ها، اطلاعات قدیمی، نمونه‌های بهتر - از PRها استقبال می‌کنیم، نیازی به issue نیست.

## آنچه استفاده می‌کنیم

### MITRE ATLAS

این مدل تهدید بر پایه [MITRE ATLAS](https://atlas.mitre.org/) (چشم‌انداز تهدید خصمانه برای سامانه‌های هوش مصنوعی)، چارچوبی که به‌طور ویژه برای تهدیدهای AI/ML مانند تزریق پرامپت، سوءاستفاده از ابزار، و بهره‌برداری از عامل طراحی شده، ساخته شده است. برای مشارکت لازم نیست ATLAS را بشناسید - ما ارسال‌ها را هنگام بازبینی به چارچوب نگاشت می‌کنیم.

### شناسه‌های تهدید

هر تهدید یک شناسه مانند `T-EXEC-003` دریافت می‌کند. دسته‌ها عبارت‌اند از:

| کد     | دسته                                      |
| ------- | ------------------------------------------ |
| RECON   | شناسایی - گردآوری اطلاعات                 |
| ACCESS  | دسترسی اولیه - ورود گرفتن                 |
| EXEC    | اجرا - اجرای اقدامات مخرب                 |
| PERSIST | پایداری - حفظ دسترسی                      |
| EVADE   | گریز از دفاع - اجتناب از شناسایی          |
| DISC    | کشف - آگاهی از محیط                       |
| EXFIL   | برون‌برد - سرقت داده‌ها                   |
| IMPACT  | اثرگذاری - آسیب یا اختلال                 |

شناسه‌ها هنگام بازبینی توسط نگه‌دارندگان اختصاص داده می‌شوند. لازم نیست یکی انتخاب کنید.

### سطوح ریسک

| سطح         | معنا                                                             |
| ------------ | ----------------------------------------------------------------- |
| **بحرانی** | سازش کامل سامانه، یا احتمال زیاد + اثر بحرانی                  |
| **زیاد**   | احتمال آسیب قابل توجه، یا احتمال متوسط + اثر بحرانی            |
| **متوسط** | ریسک متوسط، یا احتمال کم + اثر زیاد                             |
| **کم**     | بعید و با اثر محدود                                              |

اگر درباره سطح ریسک مطمئن نیستید، فقط اثر را توصیف کنید و ما آن را ارزیابی می‌کنیم.

## فرایند بازبینی

1. **تریاژ** - ارسال‌های جدید را ظرف ۴۸ ساعت بازبینی می‌کنیم
2. **ارزیابی** - امکان‌پذیری را راستی‌آزمایی می‌کنیم، نگاشت ATLAS و شناسه تهدید را اختصاص می‌دهیم، سطح ریسک را اعتبارسنجی می‌کنیم
3. **مستندسازی** - اطمینان می‌دهیم همه چیز قالب‌بندی شده و کامل است
4. **ادغام** - به مدل تهدید و نمایش بصری افزوده می‌شود
