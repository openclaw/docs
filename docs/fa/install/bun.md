<Warning>
استفاده از Bun برای زمان اجرای Gateway توصیه نمی‌شود (به‌دلیل مشکلات شناخته‌شده با WhatsApp و Telegram). برای محیط عملیاتی از Node استفاده کنید.
</Warning>

Bun یک زمان اجرای محلی اختیاری برای اجرای مستقیم TypeScript است (`bun run ...`، `bun --watch ...`). مدیر بستهٔ پیش‌فرض همچنان `pnpm` است که کاملاً پشتیبانی می‌شود و ابزارهای مستندسازی از آن استفاده می‌کنند. Bun نمی‌تواند از `pnpm-lock.yaml` استفاده کند و آن را نادیده می‌گیرد.

## نصب

<Steps>
  <Step title="نصب وابستگی‌ها">
    ```sh
    bun install
    ```

    فایل‌های `bun.lock` و `bun.lockb` توسط git نادیده گرفته می‌شوند، بنابراین تغییری در مخزن ایجاد نمی‌شود. برای جلوگیری کامل از نوشتن فایل قفل:

    ```sh
    bun install --no-save
    ```

  </Step>
  <Step title="ساخت و آزمایش">
    ```sh
    bun run build
    bun run vitest run
    ```
  </Step>
</Steps>

## اسکریپت‌های چرخهٔ حیات

Bun اسکریپت‌های چرخهٔ حیات وابستگی‌ها را مسدود می‌کند، مگر اینکه صراحتاً مورد اعتماد قرار گیرند. در این مخزن، اسکریپت‌هایی که معمولاً مسدود می‌شوند ضروری نیستند:

- `baileys` `preinstall`: بررسی می‌کند که نسخهٔ اصلی Node برابر با ۲۰ یا بالاتر باشد (OpenClaw به Node 22.19+ یا 23.11+ نیاز دارد و Node 24 توصیه می‌شود)
- `protobufjs` `postinstall`: دربارهٔ طرح‌های نسخه‌بندی ناسازگار هشدارهایی صادر می‌کند (بدون مصنوعات ساخت)

اگر با مشکلی در زمان اجرا مواجه شدید که به این اسکریپت‌ها نیاز دارد، صراحتاً به آن‌ها اعتماد کنید:

```sh
bun pm trust baileys protobufjs
```

## ملاحظات

برخی اسکریپت‌های بسته، `pnpm` را به‌صورت ثابت در خود تعریف کرده‌اند (برای مثال `check:docs`، `ui:*` و `protocol:check`). اجرای آن‌ها از طریق `bun run` همچنان `pnpm` را در پوسته اجرا می‌کند؛ بنابراین این اسکریپت‌ها را مستقیماً با `pnpm` اجرا کنید.
