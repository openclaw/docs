<Warning>
Bun نمی‌تواند CLI یا Gateway متعلق به OpenClaw را اجرا کند، زیرا API موردنیاز `node:sqlite` را فراهم نمی‌کند. برای همه فرمان‌های زمان اجرای OpenClaw، یک نسخه پشتیبانی‌شده Node نصب کنید.
</Warning>

Bun همچنان می‌تواند به‌عنوان نصب‌کننده اختیاری وابستگی‌ها و اجراکننده اسکریپت‌های بسته استفاده شود. مدیر بسته پیش‌فرض همچنان `pnpm` است که کاملاً پشتیبانی می‌شود و ابزارهای مستندات از آن استفاده می‌کنند. Bun نمی‌تواند از `pnpm-lock.yaml` استفاده کند و آن را نادیده می‌گیرد.

## نصب

<Steps>
  <Step title="نصب وابستگی‌ها">
    ```sh
    bun install
    ```

    `bun.lock` / `bun.lockb` در git نادیده گرفته می‌شوند، بنابراین تغییری ناخواسته در مخزن ایجاد نمی‌شود. برای صرف‌نظر کامل از نوشتن فایل قفل:

    ```sh
    bun install --no-save
    ```

  </Step>
  <Step title="ساخت و آزمایش">
    ```sh
    bun run build
    bun run vitest run
    ```

    فرمان‌هایی که خود OpenClaw را راه‌اندازی می‌کنند، همچنان باید از طریق Node اجرا شوند.

  </Step>
</Steps>

## اسکریپت‌های چرخه عمر

Bun اسکریپت‌های چرخه عمر وابستگی‌ها را مسدود می‌کند، مگر اینکه صراحتاً مورد اعتماد قرار گیرند. برای این مخزن، اسکریپت‌هایی که معمولاً مسدود می‌شوند ضروری نیستند:

- `baileys` `preinstall`: بررسی می‌کند که نسخه اصلی Node برابر یا بزرگ‌تر از 20 باشد (OpenClaw به Node 22.22.3+، 24.15+ یا 25.9+ نیاز دارد و Node 24 توصیه می‌شود)
- `protobufjs` `postinstall`: هشدارهایی درباره طرح‌های نسخه‌بندی ناسازگار منتشر می‌کند (بدون مصنوعات ساخت)

اگر با مشکلی در زمان اجرا مواجه شدید که به این اسکریپت‌ها نیاز دارد، صراحتاً به آن‌ها اعتماد کنید:

```sh
bun pm trust baileys protobufjs
```

## ملاحظات

برخی اسکریپت‌های بسته، `pnpm` را به‌صورت ثابت در داخل خود دارند (برای مثال `check:docs`، `ui:*`، `protocol:check`). اجرای آن‌ها از طریق `bun run` همچنان `pnpm` را از طریق پوسته اجرا می‌کند؛ بنابراین آن‌ها را مستقیماً از طریق `pnpm` اجرا کنید.
