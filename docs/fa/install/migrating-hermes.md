ارائه‌دهنده مهاجرت Hermes که همراه برنامه ارائه می‌شود، وضعیت را در `~/.hermes` شناسایی می‌کند، پیش از اعمال هر تغییر پیش‌نمایش آن را نشان می‌دهد، اطلاعات محرمانه را در طرح‌ها و گزارش‌ها پوشیده می‌کند و پیش از دست‌زدن به هر چیزی، یک نسخه پشتیبان تأییدشده از OpenClaw می‌نویسد.

<Note>
درون‌ریزی به یک راه‌اندازی تازه OpenClaw نیاز دارد. اگر از قبل وضعیت محلی OpenClaw دارید، ابتدا پیکربندی، اعتبارنامه‌ها، نشست‌ها و فضای کاری را بازنشانی کنید؛ یا پس از بررسی طرح، مستقیماً از `openclaw migrate apply hermes` همراه با `--overwrite` استفاده کنید.
</Note>

## دو روش برای درون‌ریزی

<Tabs>
  <Tab title="جادوگر راه‌اندازی اولیه">
    Hermes را در `~/.hermes` شناسایی می‌کند و پیش از اعمال، پیش‌نمایشی نشان می‌دهد.

    ```bash
    openclaw onboard --flow import
    ```

    یا یک منبع مشخص را تعیین کنید:

    ```bash
    openclaw onboard --import-from hermes --import-source ~/.hermes
    ```

  </Tab>
  <Tab title="CLI">
    برای اجراهای اسکریپتی یا تکرارپذیر از `openclaw migrate` استفاده کنید. برای مرجع کامل، به [`openclaw migrate`](/fa/cli/migrate) مراجعه کنید.

    ```bash
    openclaw migrate hermes --dry-run    # فقط پیش‌نمایش
    openclaw migrate apply hermes --yes  # اعمال بدون درخواست تأیید
    ```

    اگر Hermes خارج از `~/.hermes` قرار دارد، `--from <path>` را اضافه کنید.

  </Tab>
</Tabs>

## مواردی که درون‌ریزی می‌شوند

<AccordionGroup>
  <Accordion title="پیکربندی مدل">
    - انتخاب مدل پیش‌فرض از `config.yaml` در Hermes.
    - ارائه‌دهندگان مدل پیکربندی‌شده و نقاط پایانی سفارشی سازگار با OpenAI از `providers` و `custom_providers`.

  </Accordion>
  <Accordion title="سرورهای MCP">
    تعریف‌های سرور MCP از `mcp_servers` یا `mcp.servers`.
  </Accordion>
  <Accordion title="فایل‌های فضای کاری">
    - فایل‌های `SOUL.md` و `AGENTS.md` در فضای کاری عامل OpenClaw کپی می‌شوند.
    - فایل‌های `memories/MEMORY.md` و `memories/USER.md` به‌جای بازنویسی، به فایل‌های حافظه متناظر OpenClaw **افزوده** می‌شوند.

  </Accordion>
  <Accordion title="پیکربندی حافظه">
    پیش‌فرض‌های پیکربندی حافظه برای حافظه فایلی OpenClaw. ارائه‌دهندگان حافظه خارجی مانند Honcho به‌عنوان موارد بایگانی یا نیازمند بررسی دستی ثبت می‌شوند تا بتوانید آن‌ها را آگاهانه منتقل کنید.
  </Accordion>
  <Accordion title="Skills">
    Skills دارای فایل `SKILL.md` در مسیر `skills/<name>/`، همراه با مقادیر پیکربندی مختص هر Skill از `skills.config` کپی می‌شوند.
  </Accordion>
  <Accordion title="اعتبارنامه‌های احراز هویت">
    اجرای تعاملی `openclaw migrate` پیش از درون‌ریزی اعتبارنامه‌های احراز هویت سؤال می‌کند و گزینه بله به‌صورت پیش‌فرض انتخاب شده است. با پذیرش، ورودی‌های OpenCode OpenAI OAuth و GitHub Copilot از `auth.json` متعلق به OpenCode، به‌همراه [کلیدهای پشتیبانی‌شده `.env` در Hermes](/fa/cli/migrate#supported-env-keys) درون‌ریزی می‌شوند. ورودی‌های OAuth در `auth.json` خود Hermes وضعیت قدیمی محسوب می‌شوند: این موارد به‌جای درون‌ریزی در احراز هویت فعال، به‌عنوان موردی برای احراز هویت مجدد یا بررسی دستی با Doctor نمایش داده می‌شوند. برای درون‌ریزی اعتبارنامه‌ها در اجرای غیرتعاملی از `--include-secrets`، برای ردکردن کامل درون‌ریزی اعتبارنامه‌ها از `--no-auth-credentials`، یا در جادوگر راه‌اندازی اولیه از پرچم `--import-secrets` استفاده کنید.
  </Accordion>
</AccordionGroup>

## مواردی که فقط در بایگانی باقی می‌مانند

ارائه‌دهنده این موارد را برای بررسی دستی در پوشه گزارش مهاجرت کپی می‌کند، اما آن‌ها را در پیکربندی یا اعتبارنامه‌های فعال OpenClaw بارگذاری **نمی‌کند**:

- `plugins/`
- `sessions/`
- `logs/`
- `cron/`
- `mcp-tokens/`
- `state.db`

OpenClaw از اجرای خودکار یا اعتماد به این وضعیت خودداری می‌کند، زیرا قالب‌ها و فرضیات اعتماد ممکن است میان سامانه‌ها تغییر کرده باشند. پس از بررسی بایگانی، موارد موردنیاز را به‌صورت دستی منتقل کنید.

## روند پیشنهادی

<Steps>
  <Step title="پیش‌نمایش طرح">
    ```bash
    openclaw migrate hermes --dry-run
    ```

    طرح همه مواردی را که تغییر خواهند کرد فهرست می‌کند؛ از جمله تداخل‌ها، موارد ردشده و موارد حساس. کلیدهای تو‌در‌تویی که محرمانه به نظر می‌رسند در خروجی پوشیده می‌شوند.

  </Step>
  <Step title="اعمال همراه با نسخه پشتیبان">
    ```bash
    openclaw migrate apply hermes --yes
    ```

    OpenClaw پیش از اعمال، یک نسخه پشتیبان ایجاد و تأیید می‌کند. این نمونه غیرتعاملی فقط وضعیت غیرمحرمانه را درون‌ریزی می‌کند. برای پاسخ تعاملی به درخواست اعتبارنامه، فرمان را بدون `--yes` اجرا کنید؛ یا برای دربرگرفتن اعتبارنامه‌های پشتیبانی‌شده در اجرای بدون نظارت، `--include-secrets` را اضافه کنید.

  </Step>
  <Step title="اجرای Doctor">
    ```bash
    openclaw doctor
    ```

    [Doctor](/fa/gateway/doctor) مهاجرت‌های پیکربندی معلق را دوباره اعمال می‌کند و مشکلات ایجادشده هنگام درون‌ریزی را بررسی می‌کند.

  </Step>
  <Step title="راه‌اندازی مجدد و تأیید">
    ```bash
    openclaw gateway restart
    openclaw status
    ```

    تأیید کنید که Gateway سالم است و مدل، حافظه و Skills درون‌ریزی‌شده شما بارگذاری شده‌اند.

  </Step>
</Steps>

## مدیریت تداخل‌ها

هنگامی که طرح تداخل گزارش می‌دهد ــ یعنی فایل یا مقدار پیکربندی از قبل در مقصد وجود دارد ــ عملیات اعمال از ادامه خودداری می‌کند.

<Warning>
فقط زمانی با `--overwrite` دوباره اجرا کنید که جایگزینی مقصد موجود عمدی باشد. ارائه‌دهندگان ممکن است همچنان برای فایل‌های بازنویسی‌شده، نسخه‌های پشتیبان در سطح هر مورد را در پوشه گزارش مهاجرت بنویسند.
</Warning>

تداخل‌ها در نصب تازه غیرمعمول‌اند. آن‌ها معمولاً زمانی ظاهر می‌شوند که درون‌ریزی را دوباره روی راه‌اندازی‌ای اجرا کنید که از قبل ویرایش‌های کاربر را دارد.

اگر تداخلی هنگام اعمال ظاهر شود ــ برای مثال، رقابتی پیش‌بینی‌نشده روی یک فایل پیکربندی ــ Hermes موارد پیکربندی وابسته باقی‌مانده را به‌جای نوشتن ناقص، با وضعیت `skipped` و دلیل `blocked by earlier apply conflict` علامت‌گذاری می‌کند. گزارش مهاجرت هر مورد مسدودشده را ثبت می‌کند تا بتوانید تداخل اصلی را برطرف کرده و درون‌ریزی را دوباره اجرا کنید.

## اطلاعات محرمانه

اجرای تعاملی `openclaw migrate` می‌پرسد آیا اعتبارنامه‌های احراز هویت شناسایی‌شده درون‌ریزی شوند یا نه و گزینه بله به‌صورت پیش‌فرض انتخاب شده است.

- با پذیرش، ورودی‌های OpenCode OpenAI OAuth و GitHub Copilot از `auth.json` متعلق به OpenCode، به‌همراه [کلیدهای پشتیبانی‌شده `.env`](/fa/cli/migrate#supported-env-keys) درون‌ریزی می‌شوند. ورودی‌های OAuth در `auth.json` خود Hermes در عوض برای احراز هویت مجدد دستی OpenAI یا ترمیم با Doctor گزارش می‌شوند.
- برای درون‌ریزی فقط وضعیت غیرمحرمانه، از `--no-auth-credentials` استفاده کنید یا در درخواست پاسخ منفی دهید.
- برای درون‌ریزی اعتبارنامه‌ها در اجرای بدون نظارت همراه با `--yes`، از `--include-secrets` استفاده کنید.
- برای درون‌ریزی اعتبارنامه‌ها از طریق جادوگر راه‌اندازی اولیه، از پرچم `--import-secrets` استفاده کنید.

## خروجی JSON برای خودکارسازی

```bash
openclaw migrate hermes --dry-run --json
openclaw migrate apply hermes --json --yes
```

با `--json` و بدون `--yes`، عملیات اعمال طرح را چاپ می‌کند و وضعیت را تغییر نمی‌دهد؛ این امن‌ترین حالت برای CI و اسکریپت‌های اشتراکی است.

  </Accordion>
</AccordionGroup>
