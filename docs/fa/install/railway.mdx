OpenClaw را با یک قالب تک‌کلیکی روی Railway مستقر کنید و از طریق رابط کاربری وب Control UI به آن دسترسی داشته باشید. این ساده‌ترین مسیر «بدون ترمینال روی سرور» است: Railway، Gateway را برای شما اجرا می‌کند.

## استقرار تک‌کلیکی

<a href="https://railway.com/deploy/clawdbot-railway-template" target="_blank" rel="noreferrer">
  استقرار روی Railway
</a>

<Steps>
  <Step title="استقرار قالب">
    روی **Deploy on Railway** در بالا کلیک کنید.
  </Step>

<Step title="افزودن یک دیسک">
  یک دیسک با نقطهٔ اتصال `/data` متصل کنید (برای ماندگاری وضعیت الزامی است).
</Step>

  <Step title="تنظیم متغیرها">
    **Variables** الزامی زیر را در سرویس تنظیم کنید:

    - `OPENCLAW_GATEWAY_PORT=8080` (الزامی -- باید با درگاه موجود در Public Networking مطابقت داشته باشد)
    - `OPENCLAW_GATEWAY_TOKEN` (الزامی؛ آن را یک راز مدیریتی در نظر بگیرید)
    - `OPENCLAW_STATE_DIR=/data/.openclaw` (توصیه‌شده)
    - `OPENCLAW_WORKSPACE_DIR=/data/workspace` (توصیه‌شده)

  </Step>

<Step title="فعال‌کردن شبکهٔ عمومی">
  در بخش **Public Networking**، گزینهٔ **HTTP Proxy** را برای سرویس روی درگاه `8080` فعال کنید.
</Step>

  <Step title="اتصال">
    نشانی عمومی خود را در **Railway -> your service -> Settings -> Domains** پیدا کنید -- یا یک دامنهٔ تولیدشده (معمولاً `https://<something>.up.railway.app`) یا دامنهٔ سفارشی متصل‌شدهٔ خودتان.

    `https://<your-railway-domain>/openclaw` را باز کنید و با استفاده از راز مشترک پیکربندی‌شده متصل شوید. قالب به‌طور پیش‌فرض از `OPENCLAW_GATEWAY_TOKEN` استفاده می‌کند؛ اگر آن را با احراز هویت مبتنی بر گذرواژه جایگزین کردید، به‌جای آن از همان گذرواژه استفاده کنید.

  </Step>
</Steps>

## آنچه دریافت می‌کنید

- Gateway میزبانی‌شدهٔ OpenClaw به‌همراه Control UI
- فضای ذخیره‌سازی ماندگار از طریق Railway Volume (`/data`)، به‌طوری‌که `openclaw.json`، فایل `auth-profiles.json` هر عامل، وضعیت کانال/ارائه‌دهنده، نشست‌ها و فضای کاری پس از استقرار مجدد حفظ می‌شوند

## اتصال یک کانال

برای دستورالعمل‌های راه‌اندازی کانال، از Control UI در مسیر `/openclaw` استفاده کنید یا `openclaw onboard` را از طریق پوستهٔ Railway اجرا کنید:

- [Discord](/fa/channels/discord)
- [Telegram](/fa/channels/telegram) (سریع‌ترین روش -- فقط به یک توکن ربات نیاز دارد)
- [همهٔ کانال‌ها](/fa/channels)

## پشتیبان‌گیری و مهاجرت

وضعیت، پیکربندی، نمایه‌های احراز هویت و فضای کاری خود را صادر کنید:

```bash
openclaw backup create
```

این فرمان یک بایگانی پشتیبان قابل‌انتقال شامل وضعیت OpenClaw و هر فضای کاری پیکربندی‌شده ایجاد می‌کند. برای جزئیات، به [پشتیبان‌گیری](/fa/cli/backup) مراجعه کنید.
