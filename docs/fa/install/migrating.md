OpenClaw از سه مسیر مهاجرت پشتیبانی می‌کند: وارد کردن از یک سامانه عامل دیگر، انتقال یک نصب موجود به یک ماشین جدید، و ارتقای یک Plugin در همان محل.

## وارد کردن از یک سامانه عامل دیگر

از ارائه‌دهندگان مهاجرت همراه استفاده کنید تا دستورالعمل‌ها، سرورهای MCP، Skills، پیکربندی مدل، و کلیدهای API (با انتخاب صریح) را به OpenClaw بیاورید. طرح‌ها پیش از هر تغییری پیش‌نمایش می‌شوند، اسرار در گزارش‌ها پوشانده می‌شوند، و اعمال تغییرات با یک نسخه پشتیبان تاییدشده پشتیبانی می‌شود.

نقطه ورود CLI، [`openclaw migrate`](/fa/cli/migrate) است. راه‌اندازی اولیه نیز می‌تواند وقتی یک منبع شناخته‌شده را تشخیص می‌دهد، مهاجرت را پیشنهاد کند (`openclaw onboard --flow import`).

## انتقال OpenClaw به یک ماشین جدید

**دایرکتوری وضعیت** (`~/.openclaw/` به‌صورت پیش‌فرض) و **فضای کاری** خود را کپی کنید تا این موارد حفظ شوند:

- **پیکربندی** — `openclaw.json` و همه تنظیمات Gateway.
- **احراز هویت** — `auth-profiles.json` برای هر عامل (کلیدهای API به‌همراه OAuth)، به‌علاوه هر وضعیت کانال یا ارائه‌دهنده در `credentials/`.
- **نشست‌ها** — تاریخچه گفت‌وگو و وضعیت عامل.
- **وضعیت کانال** — ورود WhatsApp، نشست Telegram، و موارد مشابه.
- **فایل‌های فضای کاری** — `MEMORY.md`، `USER.md`، Skills، و اعلان‌ها.

<Tip>
روی ماشین قدیمی `openclaw status` را اجرا کنید تا مسیر دایرکتوری وضعیت خود را تایید کنید. پروفایل‌های سفارشی از `~/.openclaw-<profile>/` یا مسیری که از طریق `OPENCLAW_STATE_DIR` تنظیم شده است استفاده می‌کنند.
</Tip>

### مراحل مهاجرت

<Steps>
  <Step title="Gateway را متوقف کنید و نسخه پشتیبان بگیرید">
    روی ماشین **قدیمی**، Gateway را متوقف کنید تا فایل‌ها هنگام کپی تغییر نکنند، سپس آرشیو بگیرید:

    ```bash
    openclaw gateway stop
    cd ~
    tar -czf openclaw-state.tgz .openclaw
    ```

    اگر از چند پروفایل استفاده می‌کنید (برای مثال `~/.openclaw-work`)، هرکدام را جداگانه آرشیو کنید.

  </Step>

  <Step title="OpenClaw را روی ماشین جدید نصب کنید">
    CLI (و در صورت نیاز Node) را روی ماشین جدید [نصب کنید](/fa/install). اگر راه‌اندازی اولیه یک `~/.openclaw/` تازه ایجاد کند مشکلی ندارد. در مرحله بعد آن را بازنویسی می‌کنید.
  </Step>

  <Step title="دایرکتوری وضعیت و فضای کاری را کپی کنید">
    آرشیو را از طریق `scp`، `rsync -a`، یا یک درایو خارجی منتقل کنید، سپس آن را استخراج کنید:

    ```bash
    cd ~
    tar -xzf openclaw-state.tgz
    ```

    مطمئن شوید دایرکتوری‌های مخفی شامل شده‌اند و مالکیت فایل با کاربری که Gateway را اجرا خواهد کرد همخوانی دارد.

  </Step>

  <Step title="Doctor را اجرا و بررسی کنید">
    روی ماشین جدید، برای اعمال مهاجرت‌های پیکربندی و تعمیر سرویس‌ها [Doctor](/fa/gateway/doctor) را اجرا کنید:

    ```bash
    openclaw doctor
    openclaw gateway restart
    openclaw status
    ```

  </Step>
</Steps>

### مشکلات رایج

<AccordionGroup>
  <Accordion title="ناسازگاری پروفایل یا دایرکتوری وضعیت">
    اگر Gateway قدیمی از `--profile` یا `OPENCLAW_STATE_DIR` استفاده می‌کرد و Gateway جدید این کار را نمی‌کند، کانال‌ها به‌صورت خارج‌شده از حساب نمایش داده می‌شوند و نشست‌ها خالی خواهند بود. Gateway را با **همان** پروفایل یا دایرکتوری وضعیتی که مهاجرت داده‌اید اجرا کنید، سپس دوباره `openclaw doctor` را اجرا کنید.
  </Accordion>

  <Accordion title="کپی کردن فقط openclaw.json">
    فایل پیکربندی به‌تنهایی کافی نیست. پروفایل‌های احراز هویت مدل در `agents/<agentId>/agent/auth-profiles.json` قرار دارند، و وضعیت کانال و ارائه‌دهنده در `credentials/` قرار دارد. همیشه **کل** دایرکتوری وضعیت را مهاجرت دهید.
  </Accordion>

  <Accordion title="مجوزها و مالکیت">
    اگر به‌عنوان root کپی کرده‌اید یا کاربر را تغییر داده‌اید، ممکن است Gateway نتواند اعتبارنامه‌ها را بخواند. مطمئن شوید دایرکتوری وضعیت و فضای کاری متعلق به کاربری هستند که Gateway را اجرا می‌کند.
  </Accordion>

  <Accordion title="حالت راه‌دور">
    اگر رابط کاربری شما به یک Gateway **راه‌دور** اشاره می‌کند، میزبان راه‌دور مالک نشست‌ها و فضای کاری است. خود میزبان Gateway را مهاجرت دهید، نه لپ‌تاپ محلی خود را. [پرسش‌های متداول](/fa/help/faq#where-things-live-on-disk) را ببینید.
  </Accordion>

  <Accordion title="اسرار در نسخه‌های پشتیبان">
    دایرکتوری وضعیت شامل پروفایل‌های احراز هویت، اعتبارنامه‌های کانال، و وضعیت سایر ارائه‌دهندگان است. نسخه‌های پشتیبان را رمزگذاری‌شده نگه دارید، از کانال‌های انتقال ناامن پرهیز کنید، و اگر به افشا شدن مشکوک هستید کلیدها را بچرخانید.
  </Accordion>
</AccordionGroup>

### چک‌لیست بررسی

روی ماشین جدید، تایید کنید:

- [ ] `openclaw status` نشان می‌دهد Gateway در حال اجراست.
- [ ] کانال‌ها همچنان متصل هستند (نیازی به جفت‌سازی دوباره نیست).
- [ ] داشبورد باز می‌شود و نشست‌های موجود را نشان می‌دهد.
- [ ] فایل‌های فضای کاری (حافظه، پیکربندی‌ها) حاضر هستند.

## ارتقای یک Plugin در همان محل

ارتقاهای درجا برای Plugin همان شناسه Plugin و کلیدهای پیکربندی را حفظ می‌کنند، اما ممکن است وضعیت روی دیسک را به چیدمان فعلی منتقل کنند. راهنماهای ارتقای ویژه هر Plugin کنار کانال‌های آن قرار دارند:

- [مهاجرت Matrix](/fa/channels/matrix-migration): محدودیت‌های بازیابی وضعیت رمزگذاری‌شده، رفتار عکس‌برداری خودکار، و فرمان‌های بازیابی دستی.
