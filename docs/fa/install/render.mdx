OpenClaw را با استفاده از Blueprint موجود در فایل `render.yaml` مخزن روی [Render](https://render.com) مستقر کنید. این فایل سرویس، دیسک و متغیرهای محیطی را در یک فایل تعریف می‌کند.

## پیش‌نیازها

- یک [حساب Render](https://render.com) (سطح رایگان در دسترس است)
- یک کلید API از [ارائه‌دهنده مدل](/fa/providers) دلخواه شما

## استقرار

[استقرار در Render](https://render.com/deploy?repo=https://github.com/openclaw/openclaw)

این کار یک سرویس Render از روی `render.yaml` ایجاد می‌کند، ایمیج Docker را می‌سازد و آن را مستقر می‌کند. نشانی سرویس شما از الگوی `https://<service-name>.onrender.com` پیروی می‌کند.

## Blueprint

```yaml
services:
  - type: web
    name: openclaw
    runtime: docker
    plan: starter
    healthCheckPath: /health
    envVars:
      - key: OPENCLAW_GATEWAY_PORT
        value: "8080"
      - key: OPENCLAW_STATE_DIR
        value: /data/.openclaw
      - key: OPENCLAW_WORKSPACE_DIR
        value: /data/workspace
      - key: OPENCLAW_GATEWAY_TOKEN
        generateValue: true # یک توکن امن را به‌طور خودکار ایجاد می‌کند
    disk:
      name: openclaw-data
      mountPath: /data
      sizeGB: 1
```

| قابلیت                 | کاربرد                                                        |
| --------------------- | ---------------------------------------------------------- |
| `runtime: docker`     | از Dockerfile مخزن می‌سازد                                    |
| `healthCheckPath`     | Render مسیر `/health` را پایش و نمونه‌های ناسالم را راه‌اندازی مجدد می‌کند |
| `generateValue: true` | یک مقدار امن رمزنگاری‌شده را به‌طور خودکار ایجاد می‌کند             |
| `disk`                | فضای ذخیره‌سازی ماندگاری که پس از استقرار مجدد باقی می‌ماند          |

## انتخاب طرح

| طرح       | توقف خودکار           | دیسک          | مناسب برای                         |
| --------- | ----------------- | ------------- | ----------------------------- |
| رایگان    | پس از ۱۵ دقیقه بی‌کاری | در دسترس نیست | آزمایش و نمایش آزمایشی              |
| Starter   | هرگز                | ۱ گیگابایت یا بیشتر | استفاده شخصی و تیم‌های کوچک          |
| Standard+ | هرگز                | ۱ گیگابایت یا بیشتر | محیط عملیاتی و چندین کانال           |

Blueprint به‌طور پیش‌فرض از `starter` استفاده می‌کند. برای استفاده از سطح رایگان، در فایل `render.yaml` فورک خود مقدار را به `plan: free` تغییر دهید — توجه داشته باشید که بدون دیسک ماندگار، وضعیت OpenClaw با هر استقرار بازنشانی می‌شود.

## پس از استقرار

### دسترسی به رابط کاربری کنترل

داشبورد وب در `https://<your-service>.onrender.com/` در دسترس است. با استفاده از راز مشترک متصل شوید: `OPENCLAW_GATEWAY_TOKEN` که به‌طور خودکار ایجاد شده است (آن را در **Dashboard → your service → Environment** پیدا کنید)، یا اگر به احراز هویت با گذرواژه تغییر داده‌اید، از گذرواژه خود استفاده کنید.

### گزارش‌ها

**Dashboard → your service → Logs** گزارش‌های ساخت (ایجاد ایمیج Docker)، گزارش‌های استقرار (راه‌اندازی سرویس) و گزارش‌های زمان اجرا (خروجی برنامه) را نمایش می‌دهد.

### دسترسی به پوسته

**Dashboard → your service → Shell** یک نشست پوسته باز می‌کند. دیسک ماندگار در `/data` متصل شده است.

### متغیرهای محیطی

متغیرها را در **Dashboard → your service → Environment** ویرایش کنید. تغییرات باعث استقرار مجدد خودکار می‌شوند.

### استقرار خودکار

وقتی commit جدیدی به شاخه مخزن متصل اضافه شود، Render به‌طور خودکار دوباره مستقر می‌کند. اگر به‌جای فورک خودتان مستقیماً از `openclaw/openclaw` مستقر کرده‌اید، دسترسی push برای آغاز این فرایند ندارید؛ بنابراین برای به‌روزرسانی، همگام‌سازی دستی Blueprint را از Dashboard اجرا کنید یا سرویس را به فورک خودتان متصل کنید.

## دامنه سفارشی

1. **Dashboard → your service → Settings → Custom Domains**
2. دامنه خود را اضافه کنید
3. DNS را طبق دستورالعمل پیکربندی کنید (CNAME به `*.onrender.com`)
4. Render به‌طور خودکار گواهی TLS صادر می‌کند

## مقیاس‌پذیری

- **عمودی**: برای CPU/RAM بیشتر، طرح را تغییر دهید. این گزینه معمولاً برای OpenClaw کافی است.
- **افقی**: تعداد نمونه‌ها را افزایش دهید (طرح Standard و بالاتر). از آنجا که OpenClaw وضعیت زمان اجرا را روی دیسک محلی نگه می‌دارد، این کار به نشست‌های چسبنده یا مدیریت وضعیت خارجی نیاز دارد.

## پشتیبان‌گیری و مهاجرت

از پوسته Render Dashboard، در هر زمان وضعیت، پیکربندی، نمایه‌های احراز هویت و فضای کاری را صادر کنید:

```bash
openclaw backup create
```

این دستور یک بایگانی پشتیبان قابل‌انتقال ایجاد می‌کند. به [پشتیبان‌گیری](/fa/cli/backup) مراجعه کنید.

## عیب‌یابی

### سرویس راه‌اندازی نمی‌شود

گزارش‌های استقرار را در Render Dashboard بررسی کنید. مشکلات رایج:

- نبود `OPENCLAW_GATEWAY_TOKEN` — بررسی کنید که در **Dashboard → Environment** تنظیم شده باشد
- ناهماهنگی پورت — مطمئن شوید `OPENCLAW_GATEWAY_PORT=8080` است تا Gateway به پورتی متصل شود که Render انتظار دارد

### شروع سرد کُند (سطح رایگان)

سرویس‌های سطح رایگان پس از ۱۵ دقیقه عدم فعالیت متوقف می‌شوند؛ نخستین درخواست پس از توقف، تا هنگام راه‌اندازی کانتینر چند ثانیه طول می‌کشد. برای فعالیت دائمی، به Starter ارتقا دهید.

### ازدست‌رفتن داده پس از استقرار مجدد

این اتفاق در سطح رایگان رخ می‌دهد (دیسک ماندگار وجود ندارد). به یک طرح پولی ارتقا دهید یا به‌طور منظم با اجرای `openclaw backup create` از پوسته Render یک نسخه پشتیبان صادر کنید.

### شکست بررسی سلامت

اگر ساخت‌ها موفق هستند اما استقرارها شکست می‌خورند، ممکن است راه‌اندازی سرویس بیش از حد طول بکشد یا `/health` در دسترس نباشد. موارد زیر را بررسی کنید:

- گزارش‌های ساخت برای یافتن خطاها
- اینکه آیا کانتینر با `docker build && docker run` به‌صورت محلی اجرا می‌شود
