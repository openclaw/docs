# عیب‌یابی

## `clawhub login` مرورگر را باز می‌کند اما هرگز کامل نمی‌شود

CLI هنگام ورود با مرورگر، یک سرور کوتاه‌عمرِ کال‌بک محلی راه‌اندازی می‌کند.

- مطمئن شوید مرورگرتان می‌تواند به `http://127.0.0.1:<port>/callback` دسترسی پیدا کند.
- اگر کال‌بک هرگز نمی‌رسد، قوانین فایروال محلی، VPN و پراکسی را بررسی کنید.
- در محیط‌های بدون رابط گرافیکی، در رابط وب ClawHub یک توکن API بسازید و اجرا کنید:

```bash
clawhub login --token clh_...
```

## `whoami` یا `publish` مقدار `Unauthorized` (401) برمی‌گرداند

- دوباره با `clawhub login` وارد شوید.
- اگر از مسیر پیکربندی سفارشی استفاده می‌کنید، تأیید کنید `CLAWHUB_CONFIG_PATH` به
  فایلی اشاره می‌کند که توکن فعلی شما را دارد.
- اگر از توکن API استفاده می‌کنید، تأیید کنید که در رابط وب لغو نشده باشد.

## جست‌وجو یا نصب مقدار `Rate limit exceeded` (429) برمی‌گرداند

اطلاعات تلاش دوباره را در پاسخ بخوانید:

- `Retry-After`: تعداد ثانیه‌هایی که باید پیش از تلاش دوباره صبر کنید.
- `RateLimit-Limit`: محدودیتی که برای این درخواست اعمال شده است.
- `RateLimit-Remaining`: بودجه دقیق باقی‌مانده شما وقتی این هدر وجود دارد. در `429`، مقدار آن `0` است.
- `RateLimit-Reset` یا `X-RateLimit-Reset`: زمان‌بندی بازنشانی.

اگر کاربران زیادی یک IP خروجی مشترک داشته باشند، محدودیت‌های IP ناشناس ممکن است حتی وقتی هر
نفر فقط چند درخواست می‌فرستد، فعال شوند. هرجا ممکن است وارد شوید و پس از تأخیر
گزارش‌شده دوباره تلاش کنید.

## جست‌وجو یا نصب پشت پراکسی شکست می‌خورد

CLI متغیرهای استاندارد پراکسی را رعایت می‌کند:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

نام‌های پشتیبانی‌شده شامل `HTTPS_PROXY`، `HTTP_PROXY`، `https_proxy` و
`http_proxy` هستند.

## یک مهارت در جست‌وجو ظاهر نمی‌شود

- اگر نامک دقیق یا صفحه مالک را می‌دانید، آن را بررسی کنید.
- تأیید کنید انتشار عمومی است و به‌دلیل اسکن یا بازبینی نگه داشته نشده است.
- اگر مالک مهارت هستید، وارد شوید و آن را بررسی کنید:

```bash
clawhub inspect @openclaw/demo
```

عیب‌یابی‌های قابل‌مشاهده برای مالک ممکن است وضعیت اسکن، گیت بارگذاری یا بازبینی را توضیح دهند.

## انتشار به‌دلیل نبود فراداده الزامی شکست می‌خورد

برای مهارت‌ها، frontmatter در `SKILL.md` را بررسی کنید. متغیرهای محیطی و
ابزارهای الزامی باید اعلام شوند تا کاربران و اسکنرها بتوانند بسته را درک کنند.

برای پلاگین‌ها، فراداده سازگاری در `package.json` را بررسی کنید. انتشارهای پلاگین کد
به فیلدهای سازگاری OpenClaw مانند `openclaw.compat.pluginApi` و
`openclaw.build.openclawVersion` نیاز دارند.

ابتدا بارِ انتشار را پیش‌نمایش کنید:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## انتشار با خطای مالک GitHub یا منبع شکست می‌خورد

ClawHub از هویت GitHub و انتساب منبع استفاده می‌کند تا بسته‌ها را به
ناشرانشان متصل کند.

- مطمئن شوید با حساب GitHub وارد شده‌اید که مالک بسته است یا می‌تواند آن را
  منتشر کند.
- بررسی کنید URL منبع عمومی باشد یا برای ClawHub قابل دسترسی باشد.
- برای منابع GitHub، از `owner/repo`، `owner/repo@ref` یا URL کامل GitHub استفاده کنید.

## انتشار به‌دلیل ادعا یا رزرو بودن یک فضای نام شکست می‌خورد

اگر انتشار به این دلیل شکست می‌خورد که شناسه مالک، فضای نام سازمان، محدوده بسته، نامک مهارت
یا نام بسته از قبل ادعا یا رزرو شده است، ابتدا تأیید کنید که با مالکی منتشر می‌کنید
که با فضای نام مطابقت دارد. برای بسته‌های پلاگین،
نام‌های scoped مانند `@example-org/example-plugin` باید به‌عنوان مالک
مطابق `example-org` منتشر شوند.

اگر باور دارید سازمان، پروژه یا برند شما مالک برحق فضای نام است اما
نمی‌توانید مالک فعلی ClawHub را مدیریت کنید، یک
[مسئله ادعای سازمان / فضای نام](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
با مدرک عمومی و غیرحساس باز کنید. برای راهنمایی درباره مدارک و اینکه چه چیزهایی
نباید در مسائل عمومی قرار گیرند، [ادعاهای سازمان و فضای نام](/clawhub/namespace-claims) را ببینید.

## `sync` می‌گوید هیچ مهارتی پیدا نشد

`sync` به‌دنبال پوشه‌هایی می‌گردد که شامل `SKILL.md` یا `skill.md` هستند.

آن را به ریشه‌هایی که می‌خواهید اسکن شوند اشاره دهید:

```bash
clawhub sync --root /path/to/skills
```

اگر مطمئن نیستید چه چیزی منتشر خواهد شد، ابتدا پیش‌نمایش بگیرید:

```bash
clawhub sync --all --dry-run --no-input
```

## `update` به‌دلیل تغییرات محلی امتناع می‌کند

فایل‌های محلی با هیچ نسخه‌ای که ClawHub می‌شناسد مطابقت ندارند. یکی را انتخاب کنید:

- ویرایش‌های محلی را نگه دارید و به‌روزرسانی را رد کنید.
- با نسخه منتشرشده بازنویسی کنید:

```bash
clawhub update @openclaw/demo --force
```

- نسخه ویرایش‌شده خود را به‌عنوان نامک یا فورک جدید منتشر کنید.

## نصب یک پلاگین در OpenClaw شکست می‌خورد

- از منبع صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

- صفحه جزئیات بسته را برای وضعیت اسکن و فراداده سازگاری بررسی کنید.
- تأیید کنید نسخه OpenClaw شما بازه سازگاری اعلام‌شده بسته را برآورده می‌کند.
- اگر بسته پنهان، نگه‌داشته‌شده یا مسدود باشد، ممکن است تا زمانی که
  مالک مشکل را حل نکند قابل نصب نباشد.

## درخواست‌های API عمومی شکست می‌خورند

- هدرهای تلاش دوباره `429` را رعایت کنید و پاسخ‌های فهرست/جست‌وجوی عمومی را cache کنید.
- کاربران را به فهرست رسمی ClawHub برگردانید.
- محتوای پنهان، خصوصی، نگه‌داشته‌شده یا مسدودشده به‌دلیل بازبینی را خارج از
  سطح API عمومی mirror نکنید.

برای جزئیات endpoint، [API HTTP](/clawhub/http-api) را ببینید.
