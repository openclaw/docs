# عیب‌یابی

## `clawhub login` مرورگر را باز می‌کند اما هرگز کامل نمی‌شود

CLI هنگام ورود از طریق مرورگر، یک سرور محلی کوتاه‌عمر برای دریافت فراخوان بازگشتی راه‌اندازی می‌کند.

- مطمئن شوید مرورگر شما می‌تواند به `http://127.0.0.1:<port>/callback` دسترسی پیدا کند.
- اگر فراخوان بازگشتی هرگز نمی‌رسد، قواعد دیوار آتش محلی، VPN و پروکسی را بررسی کنید.
- در محیط‌های بدون رابط گرافیکی، یک توکن API در رابط وب ClawHub ایجاد کنید و دستور زیر را اجرا کنید:

```bash
clawhub login --token clh_...
```

## `whoami` یا `publish` خطای `Unauthorized` (401) برمی‌گرداند

- با `clawhub login` دوباره وارد شوید.
- اگر از مسیر پیکربندی سفارشی استفاده می‌کنید، مطمئن شوید `CLAWHUB_CONFIG_PATH` به
  فایلی اشاره می‌کند که توکن فعلی شما را در بر دارد.
- اگر از توکن API استفاده می‌کنید، مطمئن شوید در رابط وب لغو نشده است.

## جست‌وجو یا نصب خطای `Rate limit exceeded` (429) برمی‌گرداند

اطلاعات تلاش مجدد را در پاسخ بخوانید:

- `Retry-After`: تعداد ثانیه‌هایی که باید پیش از تلاش مجدد منتظر بمانید.
- `RateLimit-Limit`: محدودیت اعمال‌شده بر این درخواست.
- `RateLimit-Remaining`: سهمیه دقیق باقی‌مانده شما، در صورتی که این سرآیند وجود داشته باشد. در پاسخ `429`، مقدار آن `0` است.
- `RateLimit-Reset` یا `X-RateLimit-Reset`: زمان بازنشانی.

اگر کاربران زیادی از یک IP خروجی مشترک استفاده کنند، ممکن است محدودیت‌های IP ناشناس حتی زمانی اعمال شوند که هر
فرد فقط چند درخواست ارسال می‌کند. در صورت امکان وارد حساب شوید و پس از
تأخیر اعلام‌شده دوباره تلاش کنید.

## جست‌وجو یا نصب پشت پروکسی ناموفق است

CLI متغیرهای استاندارد پروکسی را رعایت می‌کند:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

نام‌های پشتیبانی‌شده شامل `HTTPS_PROXY`، `HTTP_PROXY`، `https_proxy` و
`http_proxy` هستند.

## یک مهارت در جست‌وجو ظاهر نمی‌شود

- اگر نامک دقیق یا صفحه مالک را می‌دانید، آن را بررسی کنید.
- مطمئن شوید انتشار عمومی است و به‌دلیل اسکن یا نظارت متوقف نشده است.
- اگر مالک مهارت هستید، وارد حساب شوید و آن را بررسی کنید:

```bash
clawhub inspect @openclaw/demo
```

اطلاعات تشخیصی قابل‌مشاهده برای مالک ممکن است وضعیت اسکن، دروازه بارگذاری یا نظارت را توضیح دهد.

## انتشار به‌دلیل نبود فراداده الزامی ناموفق است

برای مهارت‌ها، فراداده ابتدایی `SKILL.md` را بررسی کنید. متغیرهای محیطی و
ابزارهای الزامی باید اعلام شوند تا کاربران و اسکنرها بتوانند بسته را درک کنند.

برای Pluginها، فراداده سازگاری `package.json` را بررسی کنید. انتشار Pluginهای کدی
به فیلدهای سازگاری OpenClaw مانند `openclaw.compat.pluginApi` و
`openclaw.build.openclawVersion` نیاز دارد.

ابتدا پیش‌نمایش محموله انتشار را ببینید:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## انتشار به‌دلیل خطای مالک یا منبع GitHub ناموفق است

ClawHub از هویت GitHub و انتساب منبع برای متصل‌کردن بسته‌ها به
ناشرانشان استفاده می‌کند.

- مطمئن شوید با حساب GitHub که مالک بسته است یا اجازه انتشار
  آن را دارد وارد شده‌اید.
- بررسی کنید که نشانی اینترنتی منبع عمومی یا برای ClawHub قابل‌دسترسی باشد.
- برای منابع GitHub، از `owner/repo`،‏ `owner/repo@ref` یا نشانی اینترنتی کامل GitHub استفاده کنید.

## انتشار به‌دلیل تصاحب یا رزرو بودن فضای نام ناموفق است

اگر انتشار به‌دلیل اینکه شناسه مالک، فضای نام سازمان، محدوده بسته، نامک مهارت
یا نام بسته قبلاً تصاحب یا رزرو شده است ناموفق بود، ابتدا مطمئن شوید که با
مالکی انتشار می‌دهید که با فضای نام مطابقت دارد. برای بسته‌های Plugin،
نام‌های محدوده‌دار مانند `@example-org/example-plugin` باید با مالک
مطابق `example-org` منتشر شوند.

اگر معتقدید سازمان، پروژه یا برند شما مالک به‌حق فضای نام است اما
نمی‌توانید مالک فعلی ClawHub را مدیریت کنید، یک
[درخواست تصاحب سازمان / فضای نام](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
با شواهد عمومی و غیرحساس باز کنید. برای راهنمای شواهد و مواردی که
نباید در درخواست‌های عمومی درج شوند، به
[ادعاهای سازمان و فضای نام](/clawhub/namespace-claims) مراجعه کنید.

## `sync` اعلام می‌کند هیچ مهارتی یافت نشد

`sync` پوشه‌های حاوی `SKILL.md` یا `skill.md` را جست‌وجو می‌کند.

ریشه‌هایی را که می‌خواهید اسکن شوند به آن بدهید:

```bash
clawhub sync --root /path/to/skills
```

اگر مطمئن نیستید چه چیزی منتشر خواهد شد، ابتدا پیش‌نمایش را ببینید:

```bash
clawhub sync --all --dry-run --no-input
```

## `update` به‌دلیل تغییرات محلی ادامه نمی‌دهد

فایل‌های محلی با هیچ‌یک از نسخه‌هایی که ClawHub می‌شناسد مطابقت ندارند. یکی را انتخاب کنید:

- ویرایش‌های محلی را نگه دارید و به‌روزرسانی را رد کنید.
- فایل‌ها را با نسخه منتشرشده بازنویسی کنید:

```bash
clawhub update @openclaw/demo --force
```

- نسخه ویرایش‌شده خود را با نامک جدید یا به‌صورت انشعاب منتشر کنید.

## نصب یک Plugin در OpenClaw ناموفق است

- از یک منبع صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

- وضعیت اسکن و فراداده سازگاری را در صفحه جزئیات بسته بررسی کنید.
- مطمئن شوید نسخه OpenClaw شما با محدوده سازگاری اعلام‌شده
  بسته مطابقت دارد.
- اگر بسته پنهان، متوقف یا مسدود شده باشد، ممکن است تا زمانی که
  مالک مشکل را برطرف نکند قابل‌نصب نباشد.

## درخواست‌های API عمومی ناموفق هستند

- سرآیندهای تلاش مجدد `429` را رعایت و پاسخ‌های عمومی فهرست و جست‌وجو را ذخیره موقت کنید.
- کاربران را به فهرست رسمی ClawHub هدایت کنید.
- محتوای پنهان، خصوصی، متوقف‌شده یا مسدودشده توسط نظارت را خارج از
  سطح API عمومی بازنشر نکنید.

برای جزئیات نقاط پایانی، به [API مبتنی بر HTTP](/clawhub/http-api) مراجعه کنید.
