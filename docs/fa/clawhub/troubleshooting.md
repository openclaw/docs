# عیب‌یابی

## `clawhub login` مرورگر را باز می‌کند اما هرگز کامل نمی‌شود

CLI هنگام ورود با مرورگر، یک سرور بازگشت محلی کوتاه‌عمر را راه‌اندازی می‌کند.

- مطمئن شوید مرورگر شما می‌تواند به `http://127.0.0.1:<port>/callback` دسترسی پیدا کند.
- اگر بازگشت هرگز نمی‌رسد، قوانین فایروال محلی، VPN و پراکسی را بررسی کنید.
- در محیط‌های بدون رابط گرافیکی، یک توکن API در رابط وب ClawHub بسازید و اجرا کنید:

```bash
clawhub login --token clh_...
```

## `whoami` یا `publish` مقدار `Unauthorized` (401) برمی‌گرداند

- دوباره با `clawhub login` وارد شوید.
- اگر از مسیر پیکربندی سفارشی استفاده می‌کنید، تأیید کنید `CLAWHUB_CONFIG_PATH` به
  فایلی اشاره می‌کند که توکن فعلی شما را دارد.
- اگر از توکن API استفاده می‌کنید، تأیید کنید که در رابط وب لغو نشده است.

## جست‌وجو یا نصب مقدار `Rate limit exceeded` (429) برمی‌گرداند

اطلاعات تلاش مجدد را در پاسخ بخوانید:

- `Retry-After`: تعداد ثانیه‌هایی که باید پیش از تلاش مجدد صبر کنید.
- `RateLimit-Remaining` و `RateLimit-Limit`: بودجه فعلی شما.
- `RateLimit-Reset` یا `X-RateLimit-Reset`: زمان‌بندی بازنشانی.

اگر کاربران زیادی یک IP خروجی مشترک داشته باشند، ممکن است محدودیت‌های IP ناشناس حتی وقتی هر
فرد فقط چند درخواست می‌فرستد نیز اعمال شوند. در صورت امکان وارد شوید و پس از تأخیر
گزارش‌شده دوباره تلاش کنید.

## جست‌وجو یا نصب پشت پراکسی شکست می‌خورد

CLI از متغیرهای استاندارد پراکسی پیروی می‌کند:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

نام‌های پشتیبانی‌شده شامل `HTTPS_PROXY`، `HTTP_PROXY`، `https_proxy` و
`http_proxy` هستند.

## یک مهارت در جست‌وجو ظاهر نمی‌شود

- اگر اسلاگ دقیق یا صفحه مالک را می‌دانید، آن را بررسی کنید.
- تأیید کنید انتشار عمومی است و به دلیل اسکن یا بازبینی نگه داشته نشده است.
- اگر مالک مهارت هستید، وارد شوید و آن را بررسی کنید:

```bash
clawhub inspect <skill-slug>
```

عیب‌یابی‌های قابل مشاهده برای مالک ممکن است وضعیت اسکن، دروازه بارگذاری یا بازبینی را توضیح دهند.

## انتشار به دلیل نبود فراداده الزامی شکست می‌خورد

برای مهارت‌ها، frontmatter فایل `SKILL.md` را بررسی کنید. متغیرهای محیطی و
ابزارهای لازم باید اعلام شوند تا کاربران و اسکنرها بتوانند بسته را درک کنند.

برای plugins، فراداده سازگاری `package.json` را بررسی کنید. انتشار code-plugin
به فیلدهای سازگاری OpenClaw مانند `openclaw.compat.pluginApi` و
`openclaw.build.openclawVersion` نیاز دارد.

ابتدا payload انتشار را پیش‌نمایش کنید:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## انتشار با خطای مالک GitHub یا منبع شکست می‌خورد

ClawHub از هویت GitHub و انتساب منبع برای اتصال بسته‌ها به
ناشرانشان استفاده می‌کند.

- مطمئن شوید با حساب GitHub که مالک بسته است یا می‌تواند آن را منتشر کند
  وارد شده‌اید.
- بررسی کنید که URL منبع عمومی باشد یا برای ClawHub قابل دسترسی باشد.
- برای منابع GitHub، از `owner/repo`، `owner/repo@ref` یا یک URL کامل GitHub استفاده کنید.

## `sync` می‌گوید هیچ مهارتی پیدا نشد

`sync` به‌دنبال پوشه‌هایی می‌گردد که `SKILL.md` یا `skill.md` دارند.

آن را به ریشه‌هایی که می‌خواهید اسکن شوند اشاره دهید:

```bash
clawhub sync --root /path/to/skills
```

اگر مطمئن نیستید چه چیزی منتشر خواهد شد، ابتدا پیش‌نمایش بگیرید:

```bash
clawhub sync --all --dry-run --no-input
```

## `update` به دلیل تغییرات محلی امتناع می‌کند

فایل‌های محلی با هیچ نسخه‌ای که ClawHub می‌شناسد مطابقت ندارند. یکی را انتخاب کنید:

- ویرایش‌های محلی را نگه دارید و به‌روزرسانی را رد کنید.
- با نسخه منتشرشده بازنویسی کنید:

```bash
clawhub update <slug> --force
```

- نسخه ویرایش‌شده خود را به‌عنوان یک اسلاگ یا فورک جدید منتشر کنید.

## نصب یک plugin در OpenClaw شکست می‌خورد

- از یک منبع صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

- صفحه جزئیات بسته را برای وضعیت اسکن و فراداده سازگاری بررسی کنید.
- تأیید کنید نسخه OpenClaw شما با بازه سازگاری اعلام‌شده بسته
  مطابقت دارد.
- اگر بسته مخفی، نگه داشته‌شده یا مسدود شده باشد، ممکن است تا زمانی که
  مالک مشکل را برطرف نکند قابل نصب نباشد.

## درخواست‌های API عمومی شکست می‌خورند

- سرآیندهای تلاش مجدد `429` را رعایت کنید و پاسخ‌های فهرست/جست‌وجوی عمومی را کش کنید.
- کاربران را به فهرست رسمی ClawHub برگردانید.
- محتوای مخفی، خصوصی، نگه داشته‌شده یا مسدودشده توسط بازبینی را خارج از
  سطح API عمومی آینه نکنید.

برای جزئیات endpointها، [HTTP API](/fa/clawhub/http-api) را ببینید.
