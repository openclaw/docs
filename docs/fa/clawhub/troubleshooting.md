# عیب‌یابی

## `clawhub login` مرورگر را باز می‌کند اما هرگز کامل نمی‌شود

CLI هنگام ورود با مرورگر، یک سرور callback محلی کوتاه‌عمر راه‌اندازی می‌کند.

- مطمئن شوید مرورگر شما می‌تواند به `http://127.0.0.1:<port>/callback` دسترسی داشته باشد.
- اگر callback هرگز نمی‌رسد، قوانین فایروال محلی، VPN و پراکسی را بررسی کنید.
- در محیط‌های بدون رابط گرافیکی، در رابط وب ClawHub یک توکن API بسازید و اجرا کنید:

```bash
clawhub login --token clh_...
```

## `whoami` یا `publish` مقدار `Unauthorized` (401) برمی‌گرداند

- دوباره با `clawhub login` وارد شوید.
- اگر از مسیر پیکربندی سفارشی استفاده می‌کنید، تأیید کنید `CLAWHUB_CONFIG_PATH` به
  فایلی اشاره می‌کند که توکن فعلی شما را در بر دارد.
- اگر از توکن API استفاده می‌کنید، تأیید کنید که در رابط وب لغو نشده باشد.

## جستجو یا نصب مقدار `Rate limit exceeded` (429) برمی‌گرداند

اطلاعات تلاش دوباره را در پاسخ بخوانید:

- `Retry-After`: تعداد ثانیه‌هایی که باید پیش از تلاش دوباره صبر کنید.
- `RateLimit-Limit`: محدودیتی که برای این درخواست اعمال شده است.
- `RateLimit-Remaining`: بودجه دقیق باقی‌مانده شما وقتی این سرآیند وجود دارد. در `429`، مقدار آن `0` است.
- `RateLimit-Reset` یا `X-RateLimit-Reset`: زمان‌بندی بازنشانی.

اگر کاربران زیادی یک IP خروجی مشترک داشته باشند، ممکن است محدودیت‌های IP ناشناس فعال شوند، حتی وقتی هر
فرد فقط چند درخواست ارسال می‌کند. تا حد امکان وارد شوید و پس از تأخیر
گزارش‌شده دوباره تلاش کنید.

## جستجو یا نصب پشت پراکسی شکست می‌خورد

CLI به متغیرهای استاندارد پراکسی احترام می‌گذارد:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

نام‌های پشتیبانی‌شده شامل `HTTPS_PROXY`، `HTTP_PROXY`، `https_proxy` و
`http_proxy` هستند.

## یک skill در جستجو ظاهر نمی‌شود

- اگر slug دقیق یا صفحه مالک را می‌دانید، آن را بررسی کنید.
- تأیید کنید انتشار عمومی است و به‌دلیل اسکن یا moderation متوقف نشده است.
- اگر مالک skill هستید، وارد شوید و آن را بررسی کنید:

```bash
clawhub inspect @openclaw/demo
```

تشخیص‌های قابل مشاهده برای مالک ممکن است وضعیت اسکن، upload-gate یا moderation را توضیح دهند.

## انتشار شکست می‌خورد چون فراداده لازم وجود ندارد

برای skills، frontmatter فایل `SKILL.md` را بررسی کنید. متغیرهای محیطی و
ابزارهای لازم باید اعلام شوند تا کاربران و اسکنرها بتوانند بسته را درک کنند.

برای plugins، فراداده سازگاری در `package.json` را بررسی کنید. انتشارهای code-plugin
به فیلدهای سازگاری OpenClaw مانند `openclaw.compat.pluginApi` و
`openclaw.build.openclawVersion` نیاز دارند.

ابتدا payload انتشار را پیش‌نمایش کنید:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## انتشار به‌دلیل خطای مالک GitHub یا منبع شکست می‌خورد

ClawHub از هویت GitHub و انتساب منبع استفاده می‌کند تا بسته‌ها را به
ناشرانشان متصل کند.

- مطمئن شوید با حساب GitHubی وارد شده‌اید که مالک بسته است یا می‌تواند آن را
  منتشر کند.
- بررسی کنید URL منبع عمومی است یا برای ClawHub قابل دسترسی است.
- برای منابع GitHub، از `owner/repo`، `owner/repo@ref` یا یک URL کامل GitHub استفاده کنید.

## انتشار شکست می‌خورد چون یک namespace ادعا یا رزرو شده است

اگر انتشار شکست می‌خورد چون شناسه مالک، namespace سازمان، scope بسته، slug مهارت
یا نام بسته از قبل ادعا یا رزرو شده است، ابتدا تأیید کنید که با مالکی
منتشر می‌کنید که با namespace مطابقت دارد. برای بسته‌های plugin،
نام‌های scoped مانند `@example-org/example-plugin` باید با مالک
مطابق `example-org` منتشر شوند.

اگر فکر می‌کنید سازمان، پروژه یا برند شما مالک برحق namespace است اما
نمی‌توانید مالک فعلی ClawHub را مدیریت کنید، یک
[مسئله ادعای سازمان / namespace](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
با مدارک عمومی و غیرحساس باز کنید. برای راهنمایی درباره شواهد و اینکه چه چیزهایی
نباید در مسائل عمومی قرار گیرند، [ادعاهای سازمان و Namespace](/clawhub/namespace-claims) را ببینید.

## `sync` می‌گوید هیچ skillsی پیدا نشد

`sync` به‌دنبال پوشه‌هایی می‌گردد که `SKILL.md` یا `skill.md` دارند.

آن را به ریشه‌هایی که می‌خواهید اسکن شوند اشاره دهید:

```bash
clawhub sync --root /path/to/skills
```

اگر مطمئن نیستید چه چیزی منتشر می‌شود، ابتدا پیش‌نمایش بگیرید:

```bash
clawhub sync --all --dry-run --no-input
```

## `update` به‌دلیل تغییرات محلی رد می‌کند

فایل‌های محلی با هیچ نسخه‌ای که ClawHub می‌شناسد مطابقت ندارند. یکی را انتخاب کنید:

- ویرایش‌های محلی را نگه دارید و به‌روزرسانی را رد کنید.
- با نسخه منتشرشده بازنویسی کنید:

```bash
clawhub update @openclaw/demo --force
```

- نسخه ویرایش‌شده خود را به‌عنوان یک slug یا fork جدید منتشر کنید.

## نصب یک plugin در OpenClaw شکست می‌خورد

- از یک منبع ClawHub صریح استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

- صفحه جزئیات بسته را برای وضعیت اسکن و فراداده سازگاری بررسی کنید.
- تأیید کنید نسخه OpenClaw شما بازه سازگاری اعلام‌شده بسته را برآورده می‌کند.
- اگر بسته پنهان، متوقف یا مسدود شده باشد، ممکن است تا زمانی که
  مالک مشکل را حل نکند قابل نصب نباشد.

## درخواست‌های API عمومی شکست می‌خورند

- به سرآیندهای تلاش دوباره `429` احترام بگذارید و پاسخ‌های فهرست/جستجوی عمومی را cache کنید.
- کاربران را به فهرست canonical ClawHub برگردانید.
- محتوای پنهان، خصوصی، متوقف‌شده یا مسدودشده با moderation را خارج از
  سطح API عمومی mirror نکنید.

برای جزئیات endpointها، [HTTP API](/clawhub/http-api) را ببینید.
