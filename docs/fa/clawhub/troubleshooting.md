# عیب‌یابی

## `clawhub login` مرورگر را باز می‌کند اما هرگز کامل نمی‌شود

CLI هنگام ورود از طریق مرورگر، یک سرور callback محلی کوتاه‌عمر راه‌اندازی می‌کند.

- مطمئن شوید مرورگر شما می‌تواند به `http://127.0.0.1:<port>/callback` دسترسی پیدا کند.
- اگر callback هرگز نمی‌رسد، قوانین فایروال محلی، VPN و پراکسی را بررسی کنید.
- در محیط‌های بدون رابط گرافیکی، در رابط وب ClawHub یک توکن API بسازید و اجرا کنید:

```bash
clawhub login --token clh_...
```

## `whoami` یا `publish` مقدار `Unauthorized` (401) برمی‌گرداند

- دوباره با `clawhub login` وارد شوید.
- اگر از مسیر پیکربندی سفارشی استفاده می‌کنید، مطمئن شوید `CLAWHUB_CONFIG_PATH` به
  فایلی اشاره می‌کند که توکن فعلی شما را دارد.
- اگر از توکن API استفاده می‌کنید، مطمئن شوید در رابط وب لغو نشده است.

## جستجو یا نصب مقدار `Rate limit exceeded` (429) برمی‌گرداند

اطلاعات تلاش مجدد را در پاسخ بخوانید:

- `Retry-After`: تعداد ثانیه‌هایی که پیش از تلاش مجدد باید صبر کنید.
- `RateLimit-Limit`: محدودیتی که برای این درخواست اعمال شده است.
- `RateLimit-Remaining`: بودجه دقیق باقی‌مانده شما وقتی این سرآیند وجود دارد. در `429`، مقدار آن `0` است.
- `RateLimit-Reset` یا `X-RateLimit-Reset`: زمان‌بندی بازنشانی.

اگر کاربران زیادی یک IP خروجی مشترک داشته باشند، ممکن است محدودیت‌های IP ناشناس حتی وقتی هر
فرد فقط چند درخواست می‌فرستد هم فعال شوند. در صورت امکان وارد شوید و پس از تأخیر
گزارش‌شده دوباره تلاش کنید.

## جستجو یا نصب پشت پراکسی ناموفق می‌شود

CLI متغیرهای استاندارد پراکسی را رعایت می‌کند:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

نام‌های پشتیبانی‌شده شامل `HTTPS_PROXY`، `HTTP_PROXY`، `https_proxy` و
`http_proxy` هستند.

## یک مهارت در جستجو نمایش داده نمی‌شود

- اگر slug دقیق یا صفحه مالک را می‌دانید، آن را بررسی کنید.
- مطمئن شوید انتشار عمومی است و به‌دلیل اسکن یا نظارت نگه داشته نشده است.
- اگر مالک مهارت هستید، وارد شوید و آن را بررسی کنید:

```bash
clawhub inspect @openclaw/demo
```

تشخیص‌های قابل مشاهده برای مالک ممکن است وضعیت اسکن، دروازه بارگذاری یا نظارت را توضیح دهند.

## انتشار به‌دلیل نبود فراداده لازم ناموفق می‌شود

برای مهارت‌ها، frontmatter فایل `SKILL.md` را بررسی کنید. متغیرهای محیطی و
ابزارهای لازم باید اعلام شوند تا کاربران و اسکنرها بتوانند بسته را درک کنند.

برای Pluginها، فراداده سازگاری در `package.json` را بررسی کنید. انتشارهای code-plugin
به فیلدهای سازگاری OpenClaw مانند `openclaw.compat.pluginApi` و
`openclaw.build.openclawVersion` نیاز دارند.

ابتدا payload انتشار را پیش‌نمایش کنید:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## انتشار با خطای مالک GitHub یا منبع ناموفق می‌شود

ClawHub از هویت GitHub و انتساب منبع استفاده می‌کند تا بسته‌ها را به
ناشرانشان متصل کند.

- مطمئن شوید با حساب GitHub که مالک بسته است یا می‌تواند آن را منتشر کند
  وارد شده‌اید.
- بررسی کنید URL منبع عمومی است یا برای ClawHub قابل دسترسی است.
- برای منابع GitHub، از `owner/repo`، `owner/repo@ref` یا یک URL کامل GitHub استفاده کنید.

## انتشار به‌دلیل ادعا یا رزرو بودن یک namespace ناموفق می‌شود

اگر انتشار به‌دلیل اینکه handle مالک، namespace سازمان، scope بسته، slug مهارت
یا نام بسته قبلاً ادعا یا رزرو شده است ناموفق شد، ابتدا مطمئن شوید که با مالکی
منتشر می‌کنید که با namespace مطابقت دارد. برای بسته‌های Plugin،
نام‌های scoped مانند `@example-org/example-plugin` باید با مالک
مطابق `example-org` منتشر شوند.

اگر باور دارید سازمان، پروژه یا برند شما مالک برحق namespace است اما
نمی‌توانید مالک فعلی ClawHub را مدیریت کنید، یک
[درخواست ادعای سازمان / Namespace](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
با مدرک عمومی و غیرحساس باز کنید. برای راهنمایی درباره شواهد و اینکه چه چیزهایی
را نباید در issueهای عمومی قرار دهید، [ادعاهای سازمان و Namespace](/fa/clawhub/namespace-claims) را ببینید.

## `sync` می‌گوید هیچ مهارتی پیدا نشد

`sync` به‌دنبال پوشه‌هایی می‌گردد که `SKILL.md` یا `skill.md` دارند.

آن را به ریشه‌هایی که می‌خواهید اسکن شوند اشاره دهید:

```bash
clawhub sync --root /path/to/skills
```

اگر مطمئن نیستید چه چیزی منتشر خواهد شد، ابتدا پیش‌نمایش کنید:

```bash
clawhub sync --all --dry-run --no-input
```

## `update` به‌دلیل تغییرات محلی امتناع می‌کند

فایل‌های محلی با هیچ نسخه‌ای که ClawHub می‌شناسد مطابقت ندارند. یکی را انتخاب کنید:

- ویرایش‌های محلی را نگه دارید و به‌روزرسانی را رد کنید.
- با نسخه منتشرشده بازنویسی کنید:

```bash
clawhub update @openclaw/demo --force
```

- نسخه ویرایش‌شده خود را به‌عنوان یک slug یا fork جدید منتشر کنید.

## نصب یک Plugin در OpenClaw ناموفق می‌شود

- از یک منبع صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

- صفحه جزئیات بسته را برای وضعیت اسکن و فراداده سازگاری بررسی کنید.
- مطمئن شوید نسخه OpenClaw شما بازه سازگاری اعلام‌شده بسته را برآورده می‌کند.
- اگر بسته پنهان، نگه‌داشته‌شده یا مسدود شده باشد، ممکن است تا زمانی که
  مالک مشکل را حل نکند قابل نصب نباشد.

## درخواست‌های API عمومی ناموفق می‌شوند

- سرآیندهای تلاش مجدد `429` را رعایت کنید و پاسخ‌های عمومی فهرست/جستجو را cache کنید.
- کاربران را به فهرست canonical در ClawHub برگردانید.
- محتوای پنهان، خصوصی، نگه‌داشته‌شده یا مسدودشده به‌دلیل نظارت را خارج از
  سطح API عمومی mirror نکنید.

برای جزئیات endpointها، [API HTTP](/fa/clawhub/http-api) را ببینید.
