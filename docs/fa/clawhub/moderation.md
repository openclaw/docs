# نظارت و ایمنی حساب

ClawHub برای انتشار باز است، اما سطوح کشف عمومی و نصب همچنان به
حفاظ‌های کنترلی نیاز دارند. گزارش‌ها، توقف‌های نظارتی، فهرست‌های پنهان و اقدامات حساب
زمانی که یک انتشار یا حساب ناامن، گمراه‌کننده یا خارج از خط‌مشی به نظر برسد،
به محافظت از کاربران کمک می‌کنند.

این صفحه نظارت و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های ممیزی مانند
`Pass`، `Review`، `Warn`، `Malicious` و سطح ریسک، به
[ممیزی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

همچنین [امنیت](/clawhub/security) و
[استفاده قابل قبول](/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های مربوط به حق نشر یا سایر
حقوق محتوا، از [درخواست‌های حقوق محتوا](/clawhub/content-rights) استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند Skills، Pluginها و بسته‌ها را گزارش کنند.

از گزارش‌های ClawHub فقط برای محتوای ناامن بازارچه استفاده کنید، مانند:

- فهرست‌های مخرب
- فراداده گمراه‌کننده
- اعتبارنامه‌ها یا الزامات مجوز اعلام‌نشده
- دستورالعمل‌های نصب مشکوک
- جعل هویت
- ثبت‌های بدخواهانه یا سوءاستفاده از نشان تجاری
- محتوایی که [استفاده قابل قبول](/clawhub/acceptable-usage) را نقض می‌کند

از دکمه **گزارش skill** در صفحه یک skill، یا فرمان/API گزارش بسته
برای بسته‌ها استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خود یک skill یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبعی
که از فهرست پیوند شده گزارش کنید. ClawHub کد skill یا Plugin شخص ثالث را
نگهداری یا وصله نمی‌کند.

GitHub Security Advisories برای `openclaw/clawhub` مربوط به آسیب‌پذیری‌های
خود ClawHub هستند. نمونه‌ها شامل باگ‌هایی در وب‌سایت، API، CLI، رجیستری، احراز هویت،
اسکن، نظارت، یا مرزهای اعتماد دانلود/نصب است. از advisories ClawHub
برای آسیب‌پذیری‌های موجود در Skills یا Pluginهای شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و قابل اقدام هستند. سوءاستفاده از گزارش‌دهی نیز می‌تواند به
اقدام حساب منجر شود.

## ادعاهای سازمان و فضای نام

اختلاف‌های مالکیت سازمان، برند، محدوده بسته، شناسه مالک، یا فضای نام باید
از فرایند [ادعاهای سازمان و فضای نام](/clawhub/namespace-claims) استفاده کنند، نه
جریان گزارش درون‌محصولی یا فرم درخواست تجدیدنظر حساب.

زمانی از آن فرایند استفاده کنید که لازم است کارکنان ClawHub شواهد غیرحساس را بررسی کنند که نشان می‌دهد یک
فضای نام باید رزرو، منتقل، تغییرنام داده، پنهان، قرنطینه، مستعارسازی،
یا به شکل دیگری بررسی شود. در یک issue عمومی، اسرار، اسناد خصوصی، پرونده‌های حقوقی خصوصی،
اسناد هویت شخصی، توکن‌های API، یا توکن‌های چالش DNS را وارد نکنید.

## توقف‌های نظارتی

برخی یافته‌های شدید یا مسائل خط‌مشی می‌توانند یک ناشر یا فهرست را تحت
توقف نظارتی قرار دهند. وقتی این اتفاق می‌افتد، محتوای تحت تأثیر ممکن است از کشف عمومی
پنهان شود یا انتشارهای آینده تا زمان بررسی مسئله به‌صورت پنهان شروع شوند.

هدف توقف‌های نظارتی محافظت از کاربران در زمانی است که ClawHub پرونده‌های پرریسک را
حل‌وفصل می‌کند. همچنین وقتی یک مثبت کاذب تأیید شود، می‌توانند برداشته شوند.

## فهرست‌های پنهان یا مسدودشده

یک فهرست ممکن است متوقف، پنهان، قرنطینه، لغوشده، یا به شکل دیگری در
سطوح نصب عمومی در دسترس نباشد.

اگر یکی از این وضعیت‌ها را می‌بینید، انتشار را نصب نکنید مگر اینکه مالک
مسئله را حل کند یا نظارت آن را بازیابی کند.

مالکان ممکن است همچنان تشخیص‌هایی را برای فهرست‌های متوقف یا پنهان خود ببینند. این
تشخیص‌ها کمک می‌کنند توضیح داده شود چه اتفاقی افتاده و پیش از بازگشت
فهرست به سطوح عمومی چه چیزی باید تغییر کند.

## ممنوعیت‌ها و وضعیت حساب

حساب‌هایی که خط‌مشی ClawHub را نقض می‌کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفاده شدید می‌تواند
به ممنوعیت حساب، لغو توکن، محتوای پنهان، یا حذف فهرست‌ها منجر شود.
سیگنال‌های فشار سوءاستفاده ناشر روزانه بررسی می‌شوند. سیگنال‌هایی که به
آستانه ممنوعیت احتمالی ClawHub برسند می‌توانند یک هشدار خودکار را فعال کنند. اگر اسکن واجد شرایط بعدی
پس از مهلت هشدار همچنان ناشر را در
آستانه ممنوعیت احتمالی قرار دهد، ClawHub ممکن است اقدام حساب را به‌صورت خودکار اعمال کند.
سیگنال‌های بررسی با اطمینان پایین‌تر و محدود به بازه زمانی از اجرای خودکار
خارج می‌مانند.

حساب‌های حذف‌شده، ممنوع‌شده یا غیرفعال نمی‌توانند از توکن‌های API ClawHub استفاده کنند. اگر احراز هویت CLI
پس از اقدام حساب شروع به شکست کرد، برای بررسی وضعیت حساب
وارد رابط وب شوید. اگر ورود یا دسترسی عادی CLI به‌دلیل ممنوعیت یا حساب غیرفعال
مسدود شده است، از [فرم درخواست تجدیدنظر ClawHub](https://appeals.openclaw.ai/) برای بررسی بازیابی استفاده کنید.

اگر یک ایمیل فعال‌شده توسط اسکنر نسخه‌ای از یک skill یا Plugin را مخرب نام‌گذاری کند،
نتایج اسکن ذخیره‌شده برای نسخه ارسالی مسدودشده را دانلود کنید:
`clawhub scan download <slug> --version <version>`. برای Pluginها، اضافه کنید
`--kind plugin`. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید، شماره نسخه را
افزایش دهید و نسخه اصلاح‌شده را بارگذاری کنید.

## راهنمایی ناشر

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربران:

- نام‌ها، خلاصه‌ها، برچسب‌ها و changelogها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع پیوند دهید
- پیش از انتشار Pluginها از اجرای آزمایشی استفاده کنید
- اگر کاربران یا ناظران درباره رفتار انتشار سؤال کردند، شفاف پاسخ دهید
