# نظارت و ایمنی حساب

ClawHub برای انتشار باز است، اما سطوح کشف و نصب عمومی همچنان
به محافظت نیاز دارند. گزارش‌ها، توقف‌های نظارتی، فهرست‌های پنهان، و اقدامات حساب
کمک می‌کنند از کاربران محافظت شود وقتی یک انتشار یا حساب ناامن، گمراه‌کننده، یا خارج
از سیاست به نظر می‌رسد.

این صفحه نظارت و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های حسابرسی مانند
`Pass`، `Review`، `Warn`، `Malicious`، و سطح ریسک، ببینید
[حسابرسی‌های امنیتی](/clawhub/security-audits).

همچنین [امنیت](/clawhub/security) و
[استفاده قابل قبول](/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های حق نشر یا دیگر
حقوق محتوا، از [درخواست‌های حقوق محتوا](/clawhub/content-rights) استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند skills، plugins، و بسته‌ها را گزارش کنند.

از گزارش‌های ClawHub فقط برای محتوای ناامن بازار استفاده کنید، مانند:

- فهرست‌های مخرب
- فراداده گمراه‌کننده
- اعتبارنامه‌ها یا الزامات مجوز اعلام‌نشده
- دستورالعمل‌های نصب مشکوک
- جعل هویت
- ثبت‌های بدخواهانه یا سوءاستفاده از علامت تجاری
- محتوایی که [استفاده قابل قبول](/clawhub/acceptable-usage) را نقض می‌کند

از دکمه **گزارش skill** در صفحه یک skill، یا فرمان/API گزارش بسته
برای بسته‌ها استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌ها در کد منبع خودِ یک skill یا
plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیم به ناشر یا مخزن منبع
پیوندشده از فهرست گزارش کنید. ClawHub کد skill یا plugin
شخص ثالث را نگهداری یا وصله نمی‌کند.

GitHub Security Advisories برای `openclaw/clawhub` مربوط به آسیب‌پذیری‌ها در
خود ClawHub هستند. نمونه‌ها شامل باگ‌ها در وب‌سایت، API، CLI، رجیستری، احراز هویت،
اسکن، نظارت، یا مرزهای اعتماد دانلود/نصب است. از advisories ClawHub
برای آسیب‌پذیری‌ها در skills یا plugins شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و قابل اقدام هستند. سوءاستفاده از گزارش‌دهی می‌تواند خودش به
اقدام حساب منجر شود.

## ادعاهای سازمان و فضای نام

اختلاف‌های مالکیت سازمان، برند، دامنه بسته، شناسه مالک، یا فضای نام باید
از فرایند [ادعاهای سازمان و فضای نام](/clawhub/namespace-claims) استفاده کنند، نه
جریان گزارش درون محصول یا فرم درخواست تجدیدنظر حساب.

از آن فرایند زمانی استفاده کنید که لازم است کارکنان ClawHub اثبات غیرحساس را بررسی کنند که یک
فضای نام باید رزرو، منتقل، تغییر نام داده، پنهان، قرنطینه، نام مستعار داده،
یا به‌شکلی دیگر بررسی شود. اسرار، اسناد خصوصی، پرونده‌های حقوقی خصوصی،
اسناد هویت شخصی، توکن‌های API، یا توکن‌های چالش DNS را در یک
issue عمومی وارد نکنید.

## توقف‌های نظارتی

برخی یافته‌های شدید یا مسائل سیاستی می‌توانند یک ناشر یا فهرست را تحت
توقف نظارتی قرار دهند. وقتی این اتفاق می‌افتد، محتوای تحت تأثیر ممکن است از کشف عمومی
پنهان شود یا انتشارهای آینده ممکن است تا زمان بررسی مسئله به‌صورت پنهان شروع شوند.

توقف‌های نظارتی برای محافظت از کاربران در زمانی هستند که ClawHub پرونده‌های پرریسک
را حل می‌کند. همچنین وقتی مثبت کاذب تأیید شود، می‌توانند برداشته شوند.

## فهرست‌های پنهان یا مسدودشده

یک فهرست ممکن است روی سطوح نصب عمومی نگه داشته، پنهان، قرنطینه، لغو،
یا به‌شکلی دیگر در دسترس نباشد.

اگر یکی از این وضعیت‌ها را می‌بینید، انتشار را نصب نکنید مگر اینکه مالک
مسئله را حل کند یا نظارت آن را بازگرداند.

مالکان ممکن است همچنان عیب‌یابی‌های فهرست‌های نگه‌داشته‌شده یا پنهان خود را ببینند. این
عیب‌یابی‌ها کمک می‌کنند توضیح دهند چه اتفاقی افتاده و پیش از اینکه
فهرست بتواند به سطوح عمومی برگردد، چه چیزی باید تغییر کند.

## محرومیت‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض می‌کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفاده شدید می‌تواند
به محرومیت حساب، لغو توکن، محتوای پنهان، یا فهرست‌های حذف‌شده منجر شود.
سیگنال‌های فشار سوءاستفاده ناشر روزانه بررسی می‌شوند. سیگنال‌هایی که به
آستانه احتمال محرومیت ClawHub می‌رسند می‌توانند یک هشدار خودکار ایجاد کنند. اگر اسکن
واجد شرایط بعدی پس از مهلت هشدار همچنان ناشر را در
آستانه احتمال محرومیت قرار دهد، ClawHub ممکن است اقدام حساب را به‌صورت خودکار اعمال کند.
سیگنال‌های بازبینی زمانی با اطمینان پایین‌تر و محدود از اجرای خودکار
خارج می‌مانند.

حساب‌های حذف‌شده، محروم، یا غیرفعال نمی‌توانند از توکن‌های API ClawHub استفاده کنند. اگر احراز هویت CLI
پس از اقدام حساب شروع به شکست کرد، وارد UI وب شوید تا وضعیت حساب را
بررسی کنید. اگر ورود یا دسترسی عادی CLI به‌دلیل محرومیت یا حساب غیرفعال مسدود شده است،
از [فرم درخواست تجدیدنظر ClawHub](https://appeals.openclaw.ai/) برای بررسی بازیابی استفاده کنید.

اگر ایمیلی که توسط اسکنر فعال شده، یک نسخه skill یا plugin را مخرب معرفی کند،
نتایج اسکن ذخیره‌شده برای نسخه ارسالی مسدودشده را دانلود کنید:
`clawhub scan download <slug> --version <version>`. برای plugins،
`--kind plugin` را اضافه کنید. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید، شماره نسخه
را افزایش دهید، و نسخه اصلاح‌شده را بارگذاری کنید.

## راهنمای ناشر

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربر:

- نام‌ها، خلاصه‌ها، برچسب‌ها، و تغییرنامه‌ها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- از فرمان‌های نصب مبهم‌سازی‌شده اجتناب کنید
- در صورت امکان به منبع پیوند دهید
- پیش از انتشار plugins از اجرای آزمایشی استفاده کنید
- اگر کاربران یا ناظران درباره رفتار انتشار پرسیدند، شفاف پاسخ دهید
