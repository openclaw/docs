# تعدیل و ایمنی حساب

ClawHub برای انتشار باز است، اما سطوح کشف عمومی و نصب همچنان
به حفاظ‌ها نیاز دارند. گزارش‌ها، توقف‌های تعدیل، فهرست‌های پنهان و اقدامات حساب
کمک می‌کنند از کاربران محافظت شود، زمانی که یک انتشار یا حساب ناایمن، گمراه‌کننده یا خارج
از سیاست به نظر می‌رسد.

این صفحه تعدیل و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های ممیزی مانند
`Pass`، `Review`، `Warn`، `Malicious` و سطح ریسک، ببینید
[ممیزی‌های امنیتی](/clawhub/security-audits).

همچنین [امنیت](/clawhub/security) و
[استفاده قابل قبول](/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های مربوط به حق نشر یا سایر
حقوق محتوا، از [درخواست‌های حقوق محتوا](/clawhub/content-rights) استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند skills، plugins و packages را گزارش کنند.

از گزارش‌های ClawHub فقط برای محتوای ناایمن بازار استفاده کنید، مانند:

- فهرست‌های مخرب
- فراداده گمراه‌کننده
- اعتبارنامه‌ها یا الزامات مجوز اعلام‌نشده
- دستورالعمل‌های نصب مشکوک
- جعل هویت
- ثبت‌های بدخواهانه یا سوءاستفاده از نشان تجاری
- محتوایی که [استفاده قابل قبول](/clawhub/acceptable-usage) را نقض می‌کند

از دکمه **گزارش skill** در صفحه skill، یا فرمان/API گزارش package
برای packages استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ یک skill یا
plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبع
پیوندشده از فهرست گزارش کنید. ClawHub کد skill یا plugin
شخص ثالث را نگهداری یا وصله نمی‌کند.

GitHub Security Advisories برای `openclaw/clawhub` مخصوص آسیب‌پذیری‌های خودِ
ClawHub است. نمونه‌ها شامل باگ‌ها در وب‌سایت، API، CLI، registry، احراز هویت،
اسکن، تعدیل، یا مرزهای اعتماد دانلود/نصب هستند. از advisories مربوط به ClawHub
برای آسیب‌پذیری‌های skills یا plugins شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و قابل اقدام هستند. سوءاستفاده از گزارش‌دهی می‌تواند خودش به
اقدام حساب منجر شود.

## ادعاهای سازمان و namespace

اختلاف‌های مالکیت org، برند، package-scope، owner-handle یا namespace باید
از فرایند [ادعاهای سازمان و Namespace](/clawhub/namespace-claims) استفاده کنند، نه
جریان گزارش درون محصول یا فرم درخواست تجدیدنظر حساب.

وقتی به کارکنان ClawHub نیاز دارید تا مدرک غیرحساسی را بررسی کنند که نشان می‌دهد یک
namespace باید رزرو، منتقل، تغییرنام، پنهان، قرنطینه، alias
یا به شکل دیگری بررسی شود، از آن فرایند استفاده کنید. اسرار، اسناد خصوصی، پرونده‌های حقوقی خصوصی،
اسناد هویت شخصی، توکن‌های API یا توکن‌های چالش DNS را در یک
issue عمومی قرار ندهید.

## توقف‌های تعدیل

برخی یافته‌های شدید یا مسائل سیاستی می‌توانند یک ناشر یا فهرست را تحت
توقف تعدیل قرار دهند. وقتی این اتفاق می‌افتد، محتوای متاثر ممکن است از کشف عمومی
پنهان شود یا انتشارهای آینده تا زمان بررسی مسئله به‌صورت پنهان آغاز شوند.

هدف توقف‌های تعدیل محافظت از کاربران است، در حالی که ClawHub موارد پرریسک
را حل‌وفصل می‌کند. همچنین وقتی مثبت کاذب تایید شود، می‌توانند برداشته شوند.

## فهرست‌های پنهان یا مسدودشده

یک فهرست ممکن است نگه داشته، پنهان، قرنطینه، لغو یا به شکل دیگری در
سطوح نصب عمومی unavailable باشد.

اگر یکی از این وضعیت‌ها را می‌بینید، انتشار را نصب نکنید مگر اینکه مالک
مسئله را حل کند یا تعدیل آن را بازیابی کند.

مالکان ممکن است همچنان diagnostics مربوط به فهرست‌های نگه‌داشته‌شده یا پنهان خود را ببینند. این
diagnostics کمک می‌کنند توضیح داده شود چه اتفاقی افتاده و پیش از آنکه
فهرست بتواند به سطوح عمومی بازگردد، چه چیزی باید تغییر کند.

## ممنوعیت‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفاده شدید می‌تواند
به ممنوعیت حساب، لغو توکن، محتوای پنهان یا حذف فهرست‌ها منجر شود.
سیگنال‌های فشار سوءاستفاده ناشر روزانه بررسی می‌شوند. سیگنال‌هایی که به
آستانه ممنوعیت احتمالی ClawHub برسند می‌توانند یک هشدار خودکار ایجاد کنند. اگر اسکن واجد شرایط بعدی
پس از مهلت هشدار همچنان ناشر را در آستانه ممنوعیت احتمالی قرار دهد،
ClawHub ممکن است اقدام حساب را به‌صورت خودکار اعمال کند.
سیگنال‌های بررسی با اطمینان پایین‌تر و محدود به بازه زمانی از اجرای خودکار
خارج می‌مانند.

حساب‌های حذف‌شده، ممنوع‌شده یا غیرفعال نمی‌توانند از توکن‌های API ClawHub استفاده کنند. اگر احراز هویت CLI
پس از اقدام حساب شروع به شکست کند، برای بررسی وضعیت حساب
به رابط وب وارد شوید. اگر ورود یا دسترسی عادی CLI به دلیل ممنوعیت یا حساب غیرفعال مسدود شده است،
برای بررسی بازیابی از [فرم درخواست تجدیدنظر ClawHub](https://appeals.openclaw.ai/) استفاده کنید.

اگر ایمیلی که توسط scanner ایجاد شده، نسخه‌ای از skill یا plugin را مخرب معرفی می‌کند،
نتایج اسکن ذخیره‌شده برای نسخه ارسالی مسدودشده را دانلود کنید:
`clawhub scan download <slug> --version <version>`. برای plugins،
`--kind plugin` را اضافه کنید. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید، شماره نسخه
را افزایش دهید و نسخه اصلاح‌شده را بارگذاری کنید.

## راهنمای ناشر

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربران:

- نام‌ها، خلاصه‌ها، برچسب‌ها و changelogها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- هر زمان ممکن است به منبع پیوند دهید
- پیش از انتشار plugins از اجرای آزمایشی استفاده کنید
- اگر کاربران یا moderators درباره رفتار انتشار سوال کردند، شفاف پاسخ دهید
