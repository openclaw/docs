# ایمنی نظارت و حساب

ClawHub برای انتشار باز است، اما سطوح کشف عمومی و نصب همچنان
به حفاظ‌ها نیاز دارند. گزارش‌ها، توقف‌های نظارتی، فهرست‌های پنهان، و اقدامات حساب
کمک می‌کنند از کاربران محافظت شود وقتی یک انتشار یا حساب ناامن، گمراه‌کننده، یا خارج
از خط‌مشی به نظر می‌رسد.

این صفحه نظارت و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های حسابرسی مانند
`Pass`، `Review`، `Warn`، `Malicious`، و سطح ریسک، به
[حسابرسی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

همچنین [امنیت](/clawhub/security) و
[استفاده قابل قبول](/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های حق نشر یا دیگر
حقوق محتوا، از [درخواست‌های حقوق محتوا](/clawhub/content-rights) استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند مهارت‌ها، Pluginها، و بسته‌ها را گزارش کنند.

از گزارش‌های ClawHub فقط برای محتوای ناامن بازار استفاده کنید، مانند:

- فهرست‌های مخرب
- فراداده گمراه‌کننده
- الزامات اعلام‌نشده برای اعتبارنامه‌ها یا مجوزها
- دستورالعمل‌های نصب مشکوک
- جعل هویت
- ثبت‌نام‌های بدخواهانه یا سوءاستفاده از نشان تجاری
- محتوایی که [استفاده قابل قبول](/clawhub/acceptable-usage) را نقض می‌کند

از دکمه **گزارش مهارت** در صفحه مهارت، یا دستور/API گزارش بسته
برای بسته‌ها استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌ها در کد منبع خودِ یک مهارت یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبعی
که از فهرست پیوند شده گزارش کنید. ClawHub کد مهارت یا Plugin شخص ثالث را
نگهداری یا وصله نمی‌کند.

GitHub Security Advisories برای `openclaw/clawhub` مربوط به آسیب‌پذیری‌ها در
خود ClawHub است. نمونه‌ها شامل باگ‌ها در وب‌سایت، API، CLI، رجیستری، احراز هویت،
اسکن، نظارت، یا مرزهای اعتماد دانلود/نصب هستند. از advisoryهای ClawHub
برای آسیب‌پذیری‌ها در مهارت‌ها یا Pluginهای شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و قابل اقدام هستند. سوءاستفاده از گزارش‌دهی می‌تواند خود
به اقدام حساب منجر شود.

## ادعاهای سازمان و فضای نام

اختلاف‌های مالکیت سازمان، برند، محدوده بسته، دسته مالک، یا فضای نام باید
از فرایند [ادعاهای سازمان و فضای نام](/clawhub/namespace-claims) استفاده کنند، نه
جریان گزارش درون‌محصولی یا فرم اعتراض حساب.

وقتی نیاز دارید کارکنان ClawHub مدرک غیرحساس را بررسی کنند که نشان می‌دهد یک
فضای نام باید رزرو، منتقل، تغییرنام، پنهان، قرنطینه، دارای alias،
یا به شکل دیگری بررسی شود، از آن فرایند استفاده کنید. اسرار، اسناد خصوصی، فایل‌های حقوقی خصوصی،
اسناد هویت شخصی، توکن‌های API، یا توکن‌های چالش DNS را در یک
issue عمومی قرار ندهید.

## توقف‌های نظارتی

برخی یافته‌های شدید یا مسائل خط‌مشی می‌توانند یک ناشر یا فهرست را تحت
توقف نظارتی قرار دهند. وقتی این اتفاق می‌افتد، محتوای تحت تأثیر ممکن است از کشف عمومی
پنهان شود یا انتشارهای آینده ممکن است تا زمان بررسی مسئله به صورت پنهان شروع شوند.

توقف‌های نظارتی برای محافظت از کاربران طراحی شده‌اند تا ClawHub پرونده‌های پرریسک را
حل کند. آن‌ها همچنین می‌توانند وقتی مثبت کاذب تأیید شود برداشته شوند.

## فهرست‌های پنهان یا مسدودشده

یک فهرست ممکن است متوقف، پنهان، قرنطینه، لغوشده، یا به شکل دیگری در
سطوح نصب عمومی در دسترس نباشد.

اگر یکی از این وضعیت‌ها را می‌بینید، انتشار را نصب نکنید مگر اینکه مالک
مسئله را حل کند یا نظارت آن را بازیابی کند.

مالکان ممکن است همچنان عیب‌یابی‌های فهرست‌های متوقف یا پنهان خود را ببینند. این
عیب‌یابی‌ها کمک می‌کنند توضیح داده شود چه اتفاقی افتاده و پیش از بازگشت
فهرست به سطوح عمومی چه چیزی باید تغییر کند.

## ممنوعیت‌ها و وضعیت حساب

حساب‌هایی که خط‌مشی ClawHub را نقض می‌کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفاده شدید می‌تواند
به ممنوعیت حساب، ابطال توکن، محتوای پنهان، یا فهرست‌های حذف‌شده منجر شود.
سیگنال‌های فشار سوءاستفاده ناشر روزانه بررسی می‌شوند. سیگنال‌هایی که به
آستانه ممنوعیت احتمالی ClawHub برسند می‌توانند یک هشدار خودکار را فعال کنند. اگر اسکن
واجد شرایط بعدی پس از مهلت هشدار همچنان ناشر را در
آستانه ممنوعیت احتمالی قرار دهد، ClawHub ممکن است اقدام حساب را به‌صورت خودکار اعمال کند.
سیگنال‌های بررسی زمانی با اطمینان کمتر و محدود خارج از اجرای خودکار
می‌مانند.

حساب‌های حذف‌شده، ممنوع‌شده، یا غیرفعال نمی‌توانند از توکن‌های API ClawHub استفاده کنند. اگر احراز هویت CLI
پس از اقدام حساب شروع به شکست کند، برای بررسی وضعیت حساب به رابط وب وارد شوید.
اگر ورود یا دسترسی عادی CLI به دلیل ممنوعیت یا حساب غیرفعال مسدود شده است،
برای بررسی بازیابی از [فرم اعتراض ClawHub](https://appeals.openclaw.ai/) استفاده کنید.

اگر یک ایمیل فعال‌شده توسط اسکنر نسخه‌ای از یک مهارت یا Plugin را مخرب نام‌گذاری کند،
نتایج اسکن ذخیره‌شده را برای نسخه ارسالی مسدودشده دانلود کنید:
`clawhub scan download <slug> --version <version>`. برای Pluginها،
`--kind plugin` را اضافه کنید. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید، شماره نسخه را افزایش دهید،
و نسخه اصلاح‌شده را بارگذاری کنید.

## راهنمای ناشر

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربر:

- نام‌ها، خلاصه‌ها، برچسب‌ها، و تغییرنامه‌ها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای مورد نیاز را اعلام کنید
- از دستورهای نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع پیوند دهید
- پیش از انتشار Pluginها از اجراهای آزمایشی استفاده کنید
- اگر کاربران یا ناظران درباره رفتار انتشار سؤال کردند، شفاف پاسخ دهید
