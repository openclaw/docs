# نظارت و ایمنی حساب

ClawHub برای انتشار باز است، اما سطح‌های کشف عمومی و نصب همچنان به
حفاظ‌ها نیاز دارند. گزارش‌ها، توقف‌های نظارتی، فهرست‌های پنهان و اقدامات حساب
وقتی یک انتشار یا حساب ناامن، گمراه‌کننده، یا خارج از سیاست به نظر می‌رسد،
به حفاظت از کاربران کمک می‌کنند.

این صفحه نظارت و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های ممیزی مانند
`Pass`، `Review`، `Warn`، `Malicious` و سطح ریسک، [ممیزی‌های امنیتی](/clawhub/security-audits)
را ببینید.

همچنین [امنیت](/clawhub/security) و
[استفاده قابل قبول](/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های مربوط به حق نشر یا
دیگر حقوق محتوا، از [درخواست‌های حقوق محتوا](/clawhub/content-rights) استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند Skills، Pluginها و بسته‌ها را گزارش کنند.

گزارش‌های ClawHub را فقط برای محتوای ناامن بازارگاه استفاده کنید، مانند:

- فهرست‌های مخرب
- فراداده گمراه‌کننده
- اعتبارنامه‌ها یا الزامات مجوز اعلام‌نشده
- دستورالعمل‌های نصب مشکوک
- جعل هویت
- ثبت‌های بدخواهانه یا سوءاستفاده از نشان تجاری
- محتوایی که [استفاده قابل قبول](/clawhub/acceptable-usage) را نقض می‌کند

از دکمه **گزارش Skill** در صفحه Skill، یا دستور/API گزارش بسته برای بسته‌ها استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ یک Skill یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبعی که از
فهرست پیوند شده گزارش کنید. ClawHub کد Skill یا Plugin شخص ثالث را نگهداری یا وصله نمی‌کند.

GitHub Security Advisories برای `openclaw/clawhub` مخصوص آسیب‌پذیری‌های خود
ClawHub است. نمونه‌ها شامل باگ‌های وب‌سایت، API، CLI، رجیستری، احراز هویت،
اسکن، نظارت، یا مرزهای اعتماد دانلود/نصب هستند. از advisoryهای ClawHub
برای آسیب‌پذیری‌های Skills یا Pluginهای شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و قابل اقدام هستند. سوءاستفاده از گزارش‌دهی می‌تواند خود
به اقدام حساب منجر شود.

## ادعاهای سازمان و namespace

اختلاف‌های مربوط به مالکیت سازمان، برند، محدوده بسته، شناسه مالک، یا namespace
باید از فرایند [ادعاهای سازمان و Namespace](/clawhub/namespace-claims) استفاده کنند، نه از
جریان گزارش درون محصول یا فرم فرجام‌خواهی حساب.

وقتی نیاز دارید کارکنان ClawHub شواهد غیرحساس را بررسی کنند که نشان می‌دهد یک
namespace باید رزرو، منتقل، تغییر نام داده، پنهان، قرنطینه، مستعارسازی، یا به‌گونه‌ای دیگر
بررسی شود، از آن فرایند استفاده کنید. اسرار، اسناد خصوصی، پرونده‌های حقوقی خصوصی،
اسناد هویتی شخصی، توکن‌های API، یا توکن‌های چالش DNS را در یک issue عمومی قرار ندهید.

## توقف‌های نظارتی

برخی یافته‌های شدید یا مسائل سیاستی می‌توانند یک ناشر یا فهرست را تحت
توقف نظارتی قرار دهند. وقتی این اتفاق می‌افتد، محتوای تحت تأثیر ممکن است از
کشف عمومی پنهان شود یا انتشارهای آینده تا زمان بررسی مسئله به‌صورت پنهان شروع شوند.

توقف‌های نظارتی برای حفاظت از کاربران در زمانی است که ClawHub پرونده‌های پرریسک
را حل‌وفصل می‌کند. همچنین وقتی مثبت کاذب تأیید شود، می‌توان آن‌ها را برداشت.

## فهرست‌های پنهان یا مسدودشده

یک فهرست ممکن است روی سطح‌های نصب عمومی متوقف، پنهان، قرنطینه، لغوشده، یا
به‌گونه‌ای دیگر ناموجود باشد.

اگر یکی از این وضعیت‌ها را می‌بینید، انتشار را نصب نکنید مگر اینکه مالک
مسئله را حل کند یا نظارت آن را بازگرداند.

مالکان ممکن است همچنان عیب‌یابی‌های مربوط به فهرست‌های متوقف یا پنهان خود را ببینند.
این عیب‌یابی‌ها توضیح می‌دهند چه اتفاقی افتاده و پیش از بازگشت فهرست به سطح‌های عمومی
چه چیزی باید تغییر کند.

## مسدودسازی‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفاده شدید می‌تواند
به مسدودسازی حساب، لغو توکن، محتوای پنهان، یا حذف فهرست‌ها منجر شود.
سیگنال‌های فشار سوءاستفاده ناشر روزانه بررسی می‌شوند. سیگنال‌هایی که به
آستانه مسدودسازی احتمالی ClawHub برسند می‌توانند یک هشدار خودکار ایجاد کنند. اگر اسکن
واجد شرایط بعدی پس از مهلت هشدار همچنان ناشر را در آستانه مسدودسازی احتمالی قرار دهد،
ClawHub ممکن است اقدام حساب را به‌طور خودکار اعمال کند.
سیگنال‌های بازبینی زمانیِ کم‌اطمینان‌تر و محدود از اجرای خودکار خارج می‌مانند.

حساب‌های حذف‌شده، مسدودشده، یا غیرفعال‌شده نمی‌توانند از توکن‌های API ClawHub استفاده کنند. اگر احراز هویت CLI
پس از اقدام حساب شروع به شکست کرد، برای بررسی وضعیت حساب وارد UI وب شوید.
اگر ورود یا دسترسی عادی CLI به‌دلیل مسدودسازی یا حساب غیرفعال‌شده مسدود شده است،
برای بررسی بازیابی از [فرم فرجام‌خواهی ClawHub](https://appeals.openclaw.ai/) استفاده کنید.

اگر یک ایمیل فعال‌شده توسط اسکنر، نسخه‌ای از Skill یا Plugin را مخرب معرفی کند،
نتایج اسکن ذخیره‌شده برای نسخه ارسال‌شده مسدودشده را دانلود کنید:
`clawhub scan download <slug> --version <version>`. برای Pluginها،
`--kind plugin` را اضافه کنید. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید، شماره نسخه
را افزایش دهید، و نسخه اصلاح‌شده را بارگذاری کنید.

## راهنمای ناشر

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربر:

- نام‌ها، خلاصه‌ها، برچسب‌ها و changelogها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- از دستورهای نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع پیوند دهید
- پیش از انتشار Pluginها از اجرای آزمایشی استفاده کنید
- اگر کاربران یا ناظران درباره رفتار انتشار سؤال کردند، شفاف پاسخ دهید
