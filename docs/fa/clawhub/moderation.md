# نظارت و ایمنی حساب

ClawHub برای انتشار باز است، اما سطوح کشف عمومی و نصب همچنان
به حفاظ‌هایی نیاز دارند. گزارش‌ها، توقف‌های نظارتی، فهرست‌های پنهان، و اقدامات حساب
وقتی یک انتشار یا حساب ناامن، گمراه‌کننده، یا خارج از خط‌مشی به نظر برسد،
به محافظت از کاربران کمک می‌کنند.

این صفحه نظارت و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های حسابرسی مانند
`Pass`، `Review`، `Warn`، `Malicious` و سطح ریسک، به
[حسابرسی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

همچنین [امنیت](/clawhub/security) و
[استفاده قابل قبول](/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های مربوط به حق نشر یا سایر حقوق محتوا،
از [درخواست‌های حقوق محتوا](/clawhub/content-rights) استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند Skills، Plugin‌ها، و بسته‌ها را گزارش کنند.

از گزارش‌های ClawHub فقط برای محتوای ناامن marketplace استفاده کنید، مانند:

- فهرست‌های مخرب
- فراداده گمراه‌کننده
- اعتبارنامه‌ها یا الزامات مجوز اعلام‌نشده
- دستورالعمل‌های نصب مشکوک
- جعل هویت
- ثبت‌نام‌های بدخواهانه یا سوءاستفاده از علامت تجاری
- محتوایی که [استفاده قابل قبول](/clawhub/acceptable-usage) را نقض می‌کند

از دکمه **گزارش Skill** در صفحه Skill، یا فرمان/API گزارش بسته
برای بسته‌ها استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ Skill یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبع
پیوندشده از فهرست گزارش کنید. ClawHub کد Skill یا Plugin شخص ثالث را
نگهداری یا وصله نمی‌کند.

GitHub Security Advisories برای `openclaw/clawhub` مخصوص آسیب‌پذیری‌های خود
ClawHub هستند. نمونه‌ها شامل باگ‌ها در وب‌سایت، API، CLI، رجیستری، احراز هویت،
اسکن، نظارت، یا مرزهای اعتماد دانلود/نصب هستند. از advisories مربوط به ClawHub
برای آسیب‌پذیری‌های موجود در Skills یا Plugin‌های شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و قابل اقدام هستند. سوءاستفاده از گزارش‌دهی می‌تواند خود
به اقدام حساب منجر شود.

## ادعاهای سازمان و namespace

اختلاف‌های مربوط به مالکیت سازمان، برند، محدوده بسته، شناسه مالک، یا namespace باید
از فرایند [ادعاهای سازمان و Namespace](/clawhub/namespace-claims) استفاده کنند، نه
جریان گزارش درون محصول یا فرم درخواست تجدیدنظر حساب.

وقتی نیاز دارید کارکنان ClawHub مدرک غیرحساس را بررسی کنند که نشان می‌دهد یک
namespace باید رزرو، منتقل، تغییرنام‌داده، پنهان، قرنطینه، مستعار، یا به‌نحوی دیگر بررسی شود،
از آن فرایند استفاده کنید. اسرار، اسناد خصوصی، فایل‌های حقوقی خصوصی،
اسناد هویت شخصی، توکن‌های API، یا توکن‌های چالش DNS را در یک issue عمومی
قرار ندهید.

## توقف‌های نظارتی

برخی یافته‌های شدید یا مسائل خط‌مشی می‌توانند یک ناشر یا فهرست را تحت
توقف نظارتی قرار دهند. وقتی این اتفاق می‌افتد، محتوای تحت تأثیر ممکن است از
کشف عمومی پنهان شود یا انتشارهای آینده تا زمان بررسی مسئله به‌صورت پنهان شروع شوند.

توقف‌های نظارتی برای محافظت از کاربران در زمانی هستند که ClawHub موارد پرریسک
را حل‌وفصل می‌کند. همچنین وقتی مثبت کاذب تأیید شود، می‌توان آن‌ها را برداشت.

## فهرست‌های پنهان یا مسدود

یک فهرست ممکن است در وضعیت متوقف، پنهان، قرنطینه، لغوشده، یا به‌شکل دیگری در
سطوح نصب عمومی در دسترس نباشد.

اگر یکی از این وضعیت‌ها را می‌بینید، انتشار را نصب نکنید مگر اینکه مالک
مسئله را حل کند یا نظارت آن را بازگرداند.

مالکان ممکن است همچنان عیب‌یابی‌های مربوط به فهرست‌های متوقف یا پنهان خود را ببینند. این
عیب‌یابی‌ها کمک می‌کنند توضیح داده شود چه اتفاقی افتاده و پیش از اینکه
فهرست بتواند به سطوح عمومی برگردد چه چیزی باید تغییر کند.

## ممنوعیت‌ها و وضعیت حساب

حساب‌هایی که خط‌مشی ClawHub را نقض کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفاده شدید می‌تواند
به ممنوعیت حساب، ابطال توکن، محتوای پنهان، یا فهرست‌های حذف‌شده منجر شود.
سیگنال‌های فشار سوءاستفاده ناشر روزانه بررسی می‌شوند. سیگنال‌هایی که به
آستانه ممنوعیت احتمالی ClawHub برسند می‌توانند یک هشدار خودکار را فعال کنند. اگر اسکن واجد شرایط بعدی
پس از مهلت هشدار همچنان ناشر را در آستانه ممنوعیت احتمالی قرار دهد،
ClawHub ممکن است اقدام حساب را به‌صورت خودکار اعمال کند.
سیگنال‌های بررسی زمانی کم‌اطمینان‌تر و محدود از اجرای خودکار
خارج می‌مانند.

حساب‌های حذف‌شده، ممنوع‌شده، یا غیرفعال نمی‌توانند از توکن‌های API ClawHub استفاده کنند. اگر احراز هویت CLI
پس از اقدام حساب شروع به شکست کرد، برای بررسی وضعیت حساب وارد رابط وب شوید.
اگر ورود یا دسترسی معمول CLI به‌دلیل ممنوعیت یا حساب غیرفعال مسدود شده است،
برای بررسی بازیابی از [فرم درخواست تجدیدنظر ClawHub](https://appeals.openclaw.ai/) استفاده کنید.

اگر ایمیلی که توسط اسکنر فعال شده، نسخه‌ای از Skill یا Plugin را مخرب نام‌گذاری کند،
نتایج اسکن ذخیره‌شده برای نسخه ارسالی مسدودشده را دانلود کنید:
`clawhub scan download <slug> --version <version>`. برای Plugin‌ها،
`--kind plugin` را اضافه کنید. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید، شماره نسخه
را افزایش دهید، و نسخه اصلاح‌شده را بارگذاری کنید.

## راهنمایی ناشر

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربران:

- نام‌ها، خلاصه‌ها، برچسب‌ها، و changelogها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- از فرمان‌های نصب مبهم‌سازی‌شده اجتناب کنید
- در صورت امکان به منبع پیوند دهید
- پیش از انتشار Plugin‌ها از dry run استفاده کنید
- اگر کاربران یا ناظران درباره رفتار انتشار سؤال کردند، شفاف پاسخ دهید
