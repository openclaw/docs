# نظارت و ایمنی حساب

ClawHub برای انتشار باز است، اما سطوح کشف عمومی و نصب همچنان
به حفاظ‌های کنترلی نیاز دارند. گزارش‌ها، نگه‌داشت‌های نظارتی، فهرست‌های پنهان، و اقدامات حساب
به محافظت از کاربران کمک می‌کنند، زمانی که یک انتشار یا حساب ناامن، گمراه‌کننده، یا خارج
از سیاست به نظر می‌رسد.

این صفحه نظارت و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های ممیزی مانند
`Pass`، `Review`، `Warn`، `Malicious`، و سطح ریسک، به
[ممیزی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

همچنین [امنیت](/clawhub/security) و
[استفاده قابل قبول](/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های مربوط به حق نشر یا سایر
حقوق محتوا، از [درخواست‌های حقوق محتوا](/clawhub/content-rights) استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند skills، plugins، و بسته‌ها را گزارش کنند.

از گزارش‌های ClawHub فقط برای محتوای ناامن بازار استفاده کنید، مانند:

- فهرست‌های مخرب
- فراداده گمراه‌کننده
- نیازمندی‌های اعلام‌نشده برای اعتبارنامه‌ها یا مجوزها
- دستورالعمل‌های نصب مشکوک
- جعل هویت
- ثبت‌نام‌های با سوءنیت یا سوءاستفاده از نشان تجاری
- محتوایی که [استفاده قابل قبول](/clawhub/acceptable-usage) را نقض می‌کند

از دکمه **گزارش skill** در صفحه یک skill، یا فرمان/API گزارش‌دهی بسته
برای بسته‌ها استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ یک skill یا
plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبع
پیوندشده از فهرست گزارش کنید. ClawHub کد skill یا plugin شخص ثالث را نگهداری یا وصله
نمی‌کند.

GitHub Security Advisories برای `openclaw/clawhub` مخصوص آسیب‌پذیری‌های خود
ClawHub هستند. نمونه‌ها شامل باگ‌ها در وب‌سایت، API، CLI، رجیستری، احراز هویت،
اسکن، نظارت، یا مرزهای اعتماد دانلود/نصب می‌شود. از مشاوره‌های ClawHub
برای آسیب‌پذیری‌های skills یا plugins شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و قابل اقدام هستند. سوءاستفاده از گزارش‌دهی می‌تواند خودش به
اقدام علیه حساب منجر شود.

## ادعاهای سازمان و فضای نام

اختلافات مالکیت سازمان، برند، محدوده بسته، شناسه مالک، یا فضای نام باید
از فرایند [ادعاهای سازمان و فضای نام](/clawhub/namespace-claims) استفاده کنند، نه
جریان گزارش درون محصول یا فرم درخواست تجدیدنظر حساب.

وقتی نیاز دارید کارکنان ClawHub مدرک غیرحساسی را بررسی کنند که نشان می‌دهد یک
فضای نام باید رزرو، منتقل، تغییرنام، پنهان، قرنطینه، مستعارسازی،
یا به شکل دیگری بررسی شود، از آن فرایند استفاده کنید. اسرار، اسناد خصوصی، پرونده‌های حقوقی خصوصی،
مدارک هویتی شخصی، توکن‌های API، یا توکن‌های چالش DNS را در یک
issue عمومی قرار ندهید.

## نگه‌داشت‌های نظارتی

برخی یافته‌های شدید یا مسائل سیاستی می‌توانند یک ناشر یا فهرست را تحت
نگه‌داشت نظارتی قرار دهند. وقتی این اتفاق می‌افتد، محتوای تحت تأثیر ممکن است از کشف
عمومی پنهان شود یا انتشارهای آینده تا زمان بررسی مسئله به‌صورت پنهان شروع شوند.

نگه‌داشت‌های نظارتی برای محافظت از کاربران طراحی شده‌اند، در حالی که ClawHub پرونده‌های پرریسک را
حل می‌کند. همچنین وقتی مثبت کاذب تأیید شود، می‌توانند برداشته شوند.

## فهرست‌های پنهان یا مسدود

یک فهرست ممکن است نگه‌داشته، پنهان، قرنطینه، لغو، یا به شکل دیگری در
سطوح نصب عمومی در دسترس نباشد.

اگر یکی از این وضعیت‌ها را دیدید، انتشار را نصب نکنید مگر اینکه مالک
مسئله را حل کند یا نظارت آن را بازگرداند.

مالکان ممکن است همچنان عیب‌یابی‌های مربوط به فهرست‌های نگه‌داشته‌شده یا پنهان خود را ببینند. این
عیب‌یابی‌ها کمک می‌کنند توضیح داده شود چه اتفاقی افتاده و پیش از بازگشت
فهرست به سطوح عمومی چه چیزی باید تغییر کند.

## مسدودسازی‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفاده شدید می‌تواند
به مسدودسازی حساب، لغو توکن، محتوای پنهان، یا فهرست‌های حذف‌شده منجر شود.
سیگنال‌های فشار سوءاستفاده ناشر روزانه بررسی می‌شوند. سیگنال‌هایی که به
آستانه مسدودسازی احتمالی ClawHub برسند می‌توانند یک هشدار خودکار ایجاد کنند. اگر اسکن
واجد شرایط بعدی پس از مهلت هشدار همچنان ناشر را در
آستانه مسدودسازی احتمالی قرار دهد، ClawHub ممکن است اقدام حساب را به‌صورت خودکار اعمال کند.
سیگنال‌های کم‌اطمینان‌تر و سیگنال‌های بررسی زمانی محدود از اجرای خودکار
خارج می‌مانند.

حساب‌های حذف‌شده، مسدودشده، یا غیرفعال نمی‌توانند از توکن‌های API ClawHub استفاده کنند. اگر احراز هویت CLI
پس از اقدام حساب شروع به شکست کرد، برای بررسی وضعیت حساب وارد UI وب شوید.
اگر ورود یا دسترسی عادی CLI به دلیل مسدودسازی یا حساب غیرفعال مسدود شده است،
برای بررسی بازیابی از [فرم درخواست تجدیدنظر ClawHub](https://appeals.openclaw.ai/) استفاده کنید.

اگر ایمیلی که توسط اسکنر ایجاد شده، نسخه یک skill یا plugin را مخرب اعلام می‌کند،
نتایج اسکن ذخیره‌شده را برای نسخه ارسالی مسدودشده دانلود کنید:
`clawhub scan download <slug> --version <version>`. برای plugins، گزینه
`--kind plugin` را اضافه کنید. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید، شماره نسخه
را افزایش دهید، و نسخه اصلاح‌شده را بارگذاری کنید.

## راهنمای ناشر

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربر:

- نام‌ها، خلاصه‌ها، برچسب‌ها، و changelogها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای موردنیاز را اعلام کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع پیوند دهید
- پیش از انتشار plugins از اجرای آزمایشی استفاده کنید
- اگر کاربران یا ناظران درباره رفتار انتشار سؤال کردند، شفاف پاسخ دهید
