# نظارت و ایمنی حساب

ClawHub برای انتشار باز است، اما سطوح کشف عمومی و نصب همچنان به
حفاظ‌های کنترلی نیاز دارند. گزارش‌ها، نگه‌داشت‌های نظارتی، فهرست‌های پنهان و اقدامات حساب
وقتی یک انتشار یا حساب ناامن، گمراه‌کننده، یا خارج از سیاست به نظر می‌رسد،
به محافظت از کاربران کمک می‌کنند.

این صفحه نظارت و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های حسابرسی مانند
`Pass`، `Review`، `Warn`، `Malicious`، و سطح ریسک، به
[حسابرسی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

همچنین [امنیت](/clawhub/security) و
[استفاده قابل قبول](/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های مربوط به حق نشر یا دیگر
حقوق محتوا، از [درخواست‌های حقوق محتوا](/clawhub/content-rights) استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند Skills، Pluginها و بسته‌ها را گزارش کنند.

از گزارش‌های ClawHub فقط برای محتوای ناامن بازار استفاده کنید، مانند:

- فهرست‌های مخرب
- فرادادهٔ گمراه‌کننده
- اطلاعات محرمانه یا الزامات مجوز اعلام‌نشده
- دستورالعمل‌های نصب مشکوک
- جعل هویت
- ثبت‌نام‌های بدخواهانه یا سوءاستفاده از نشان تجاری
- محتوایی که [استفاده قابل قبول](/clawhub/acceptable-usage) را نقض می‌کند

از دکمهٔ **گزارش Skill** در صفحهٔ یک Skill، یا فرمان/API گزارش بسته
برای بسته‌ها استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ Skill یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبع
پیوندشده از فهرست گزارش کنید. ClawHub کد Skill یا Plugin شخص ثالث را
نگه‌داری یا وصله نمی‌کند.

GitHub Security Advisories برای `openclaw/clawhub` مربوط به آسیب‌پذیری‌های خود
ClawHub هستند. نمونه‌ها شامل باگ‌ها در وب‌سایت، API، CLI، رجیستری، احراز هویت،
اسکن، نظارت، یا مرزهای اعتماد دانلود/نصب هستند. از توصیه‌نامه‌های ClawHub
برای آسیب‌پذیری‌های Skills یا Pluginهای شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و قابل اقدام هستند. سوءاستفاده از گزارش‌دهی می‌تواند خود
به اقدام حساب منجر شود.

## ادعاهای سازمان و فضای نام

اختلاف‌های مالکیت سازمان، برند، دامنهٔ بسته، دستهٔ مالک، یا فضای نام باید
از فرایند [ادعاهای سازمان و فضای نام](/clawhub/namespace-claims) استفاده کنند، نه
جریان گزارش درون‌محصولی یا فرم اعتراض حساب.

وقتی نیاز دارید کارکنان ClawHub مدرک غیرحساسی را بررسی کنند که نشان می‌دهد یک
فضای نام باید رزرو، منتقل، تغییرنام، پنهان، قرنطینه، مستعارسازی،
یا به‌شکل دیگری بررسی شود، از آن فرایند استفاده کنید. در یک issue عمومی،
رازها، اسناد خصوصی، پرونده‌های حقوقی خصوصی، اسناد هویت شخصی، توکن‌های API،
یا توکن‌های چالش DNS را وارد نکنید.

## نگه‌داشت‌های نظارتی

برخی یافته‌های شدید یا مسائل سیاستی می‌توانند یک ناشر یا فهرست را تحت
نگه‌داشت نظارتی قرار دهند. وقتی این اتفاق می‌افتد، محتوای تحت تأثیر ممکن است از
کشف عمومی پنهان شود یا انتشارهای آینده تا زمان بررسی مسئله به‌صورت پنهان شروع شوند.

نگه‌داشت‌های نظارتی برای محافظت از کاربران در زمانی هستند که ClawHub پرونده‌های
پرریسک را حل‌وفصل می‌کند. همچنین وقتی مثبت کاذب تأیید شود، می‌توانند برداشته شوند.

## فهرست‌های پنهان یا مسدود

یک فهرست ممکن است نگه داشته، پنهان، قرنطینه، لغو، یا به‌شکل دیگری در
سطوح نصب عمومی ناموجود شود.

اگر یکی از این وضعیت‌ها را می‌بینید، انتشار را نصب نکنید مگر اینکه مالک
مسئله را حل کند یا نظارت آن را بازیابی کند.

مالکان ممکن است همچنان عیب‌یابی‌های فهرست‌های نگه‌داشته یا پنهان خود را ببینند. این
عیب‌یابی‌ها کمک می‌کنند توضیح داده شود چه اتفاقی افتاده و پیش از اینکه
فهرست بتواند به سطوح عمومی بازگردد، چه چیزی باید تغییر کند.

## محرومیت‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفادهٔ شدید می‌تواند
به محرومیت حساب، لغو توکن، محتوای پنهان، یا فهرست‌های حذف‌شده منجر شود.
سیگنال‌های فشار سوءاستفادهٔ ناشر روزانه بررسی می‌شوند. سیگنال‌هایی که به
آستانهٔ محرومیت احتمالی ClawHub برسند می‌توانند یک هشدار خودکار را فعال کنند. اگر اسکن
واجد شرایط بعدی پس از مهلت هشدار همچنان ناشر را در
آستانهٔ محرومیت احتمالی قرار دهد، ClawHub ممکن است اقدام حساب را به‌صورت خودکار اعمال کند.
سیگنال‌های بررسی زمانی با اطمینان کمتر و محدودشده خارج از اجرای خودکار
می‌مانند.

حساب‌های حذف‌شده، محروم، یا غیرفعال نمی‌توانند از توکن‌های API ClawHub استفاده کنند. اگر احراز هویت CLI
پس از اقدام حساب شروع به شکست کرد، برای بررسی وضعیت حساب وارد UI وب شوید.
اگر ورود یا دسترسی عادی CLI به‌دلیل محرومیت یا حساب غیرفعال مسدود شده است،
برای بررسی بازیابی از [فرم اعتراض ClawHub](https://appeals.openclaw.ai/) استفاده کنید.

اگر ایمیل فعال‌شده توسط اسکنر، نسخهٔ یک Skill یا Plugin را مخرب نام‌گذاری می‌کند،
نتایج اسکن ذخیره‌شده برای نسخهٔ ارسالی مسدودشده را دانلود کنید:
`clawhub scan download <slug> --version <version>`. برای Pluginها،
`--kind plugin` را اضافه کنید. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید، شمارهٔ نسخه را
افزایش دهید، و نسخهٔ اصلاح‌شده را بارگذاری کنید.

## راهنمایی ناشران

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربر:

- نام‌ها، خلاصه‌ها، برچسب‌ها و changelogها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع پیوند دهید
- پیش از انتشار Pluginها از اجرای آزمایشی استفاده کنید
- اگر کاربران یا ناظران دربارهٔ رفتار انتشار سؤال کردند، روشن پاسخ دهید
