# تعدیل محتوا و ایمنی حساب

ClawHub برای انتشار باز است، اما سطوح کشف عمومی و نصب همچنان
به محافظ نیاز دارند. گزارش‌ها، نگه‌داشت‌های تعدیل محتوا، فهرست‌های پنهان و اقدامات حساب
وقتی یک انتشار یا حساب ناامن، گمراه‌کننده یا خارج از سیاست به نظر برسد
به محافظت از کاربران کمک می‌کنند.

این صفحه تعدیل محتوا و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های حسابرسی مانند
`Pass`، `Review`، `Warn`، `Malicious` و سطح ریسک، به
[حسابرسی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

همچنین [امنیت](/clawhub/security) و
[استفاده قابل قبول](/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های مربوط به حق نشر یا دیگر
حقوق محتوا، از [درخواست‌های حقوق محتوا](/clawhub/content-rights) استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند Skills، Pluginها و بسته‌ها را گزارش کنند.

از گزارش‌های ClawHub فقط برای محتوای ناامن بازارچه استفاده کنید، مانند:

- فهرست‌های مخرب
- فراداده گمراه‌کننده
- اعتبارنامه‌ها یا الزامات مجوز اعلام‌نشده
- دستورالعمل‌های نصب مشکوک
- جعل هویت
- ثبت‌های با سوءنیت یا سوءاستفاده از نشان تجاری
- محتوایی که [استفاده قابل قبول](/clawhub/acceptable-usage) را نقض می‌کند

از دکمه **گزارش Skills** در صفحه Skills، یا فرمان/API گزارش بسته
برای بسته‌ها استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ Skills یا
Plugin شخص ثالث استفاده نکنید. این موارد را مستقیما به ناشر یا مخزن منبعی که
از فهرست لینک شده گزارش کنید. ClawHub کد Skills یا Plugin شخص ثالث را نگهداری
یا وصله نمی‌کند.

هشدارهای امنیتی GitHub برای `openclaw/clawhub` مربوط به آسیب‌پذیری‌های خود
ClawHub هستند. نمونه‌ها شامل باگ‌ها در وب‌سایت، API، CLI، رجیستری، احراز هویت،
اسکن، تعدیل محتوا، یا مرزهای اعتماد دانلود/نصب می‌شوند. از هشدارهای ClawHub
برای آسیب‌پذیری‌های Skills یا Pluginهای شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و قابل اقدام هستند. سوءاستفاده از گزارش‌دهی می‌تواند خود
به اقدام علیه حساب منجر شود.

## ادعاهای سازمان و فضای نام

اختلاف‌های مالکیت سازمان، برند، دامنه بسته، شناسه مالک یا فضای نام باید
از فرایند [ادعاهای سازمان و فضای نام](/clawhub/namespace-claims) استفاده کنند، نه از
جریان گزارش درون محصول یا فرم درخواست تجدیدنظر حساب.

وقتی لازم دارید کارکنان ClawHub شواهد غیرحساس را بررسی کنند که نشان می‌دهد یک
فضای نام باید رزرو، منتقل، تغییرنام، پنهان، قرنطینه، مستعارسازی یا به شکل دیگری
بررسی شود، از آن فرایند استفاده کنید. اسرار، اسناد خصوصی، فایل‌های حقوقی خصوصی،
مدارک هویتی شخصی، توکن‌های API یا توکن‌های چالش DNS را در یک issue عمومی
قرار ندهید.

## نگه‌داشت‌های تعدیل محتوا

برخی یافته‌های شدید یا مسائل سیاستی می‌توانند یک ناشر یا فهرست را تحت
نگه‌داشت تعدیل محتوا قرار دهند. وقتی این اتفاق می‌افتد، محتوای متاثر ممکن است از
کشف عمومی پنهان شود یا انتشارهای آینده تا زمان بررسی مسئله به صورت پنهان شروع شوند.

نگه‌داشت‌های تعدیل محتوا برای محافظت از کاربران در زمانی است که ClawHub
پرونده‌های پرریسک را حل‌وفصل می‌کند. همچنین وقتی مثبت کاذب تایید شود، می‌توانند برداشته شوند.

## فهرست‌های پنهان یا مسدود

یک فهرست ممکن است در حالت نگه‌داشت، پنهان، قرنطینه، لغوشده یا به شکل دیگری در
سطوح نصب عمومی در دسترس نباشد.

اگر یکی از این وضعیت‌ها را می‌بینید، انتشار را نصب نکنید مگر اینکه مالک
مسئله را حل کند یا تعدیل محتوا آن را بازگرداند.

مالکان ممکن است همچنان عیب‌یابی‌های مربوط به فهرست‌های نگه‌داشته‌شده یا پنهان خود را ببینند. این
عیب‌یابی‌ها کمک می‌کنند توضیح دهند چه اتفاقی افتاده و پیش از بازگشت فهرست
به سطوح عمومی چه چیزی باید تغییر کند.

## ممنوعیت‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض می‌کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفاده شدید می‌تواند
به ممنوعیت حساب، لغو توکن، محتوای پنهان یا فهرست‌های حذف‌شده منجر شود.
سیگنال‌های فشار سوءاستفاده ناشر روزانه بررسی می‌شوند. سیگنال‌هایی که به
آستانه ممنوعیت احتمالی ClawHub می‌رسند می‌توانند یک هشدار خودکار فعال کنند. اگر اسکن واجدشرایط بعدی
پس از مهلت هشدار همچنان ناشر را در آستانه ممنوعیت احتمالی قرار دهد،
ClawHub ممکن است اقدام حساب را به صورت خودکار اعمال کند.
سیگنال‌های بررسی زمانی با اطمینان پایین‌تر و محدود از اجرای خودکار
خارج می‌مانند.

حساب‌های حذف‌شده، ممنوع‌شده یا غیرفعال نمی‌توانند از توکن‌های API ClawHub استفاده کنند. اگر احراز هویت CLI
پس از اقدام حساب شروع به شکست کرد، برای بررسی وضعیت حساب وارد رابط وب شوید.
اگر ورود یا دسترسی عادی CLI به دلیل ممنوعیت یا حساب غیرفعال مسدود شده است،
برای بررسی بازیابی از [فرم درخواست تجدیدنظر ClawHub](https://appeals.openclaw.ai/) استفاده کنید.

اگر ایمیلی که توسط اسکنر فعال شده نسخه‌ای از Skills یا Plugin را مخرب نام‌گذاری می‌کند،
نتایج اسکن ذخیره‌شده برای نسخه ارسالی مسدودشده را دانلود کنید:
`clawhub scan download <slug> --version <version>`. برای Pluginها،
`--kind plugin` را اضافه کنید. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید، شماره نسخه
را افزایش دهید و نسخه اصلاح‌شده را بارگذاری کنید.

## راهنمای ناشران

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربران:

- نام‌ها، خلاصه‌ها، برچسب‌ها و changelogها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع لینک دهید
- پیش از انتشار Pluginها از اجرای آزمایشی استفاده کنید
- اگر کاربران یا ناظران درباره رفتار انتشار سؤال کردند، روشن پاسخ دهید
