# تعدیل محتوا و ایمنی حساب

ClawHub برای انتشار باز است، اما سطوح کشف عمومی و نصب همچنان
به حفاظ‌ها نیاز دارند. گزارش‌ها، توقف‌های تعدیل محتوا، فهرست‌های پنهان، و اقدامات حساب
به محافظت از کاربران کمک می‌کنند وقتی یک انتشار یا حساب ناایمن، گمراه‌کننده، یا خارج
از خط‌مشی به نظر می‌رسد.

این صفحه تعدیل محتوا و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های ممیزی مانند
`Pass`، `Review`، `Warn`، `Malicious`، و سطح ریسک، ببینید
[ممیزی‌های امنیتی](/fa/clawhub/security-audits).

همچنین [امنیت](/fa/clawhub/security) و
[استفاده قابل قبول](/fa/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های مربوط به حق نشر یا دیگر
حقوق محتوا، از [درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند Skills، plugins، و بسته‌ها را گزارش کنند.

از گزارش‌های ClawHub فقط برای محتوای ناایمن بازارچه استفاده کنید، مانند:

- فهرست‌های مخرب
- فراداده گمراه‌کننده
- الزامات اعلام‌نشده برای اعتبارنامه‌ها یا مجوزها
- دستورالعمل‌های نصب مشکوک
- جعل هویت
- ثبت‌نام‌های با نیت بد یا سوءاستفاده از نشان تجاری
- محتوایی که [استفاده قابل قبول](/fa/clawhub/acceptable-usage) را نقض می‌کند

از دکمه **گزارش skill** در صفحه یک skill، یا فرمان/API گزارش بسته برای بسته‌ها استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌ها در کد منبع خود یک skill یا
plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبع
پیوندشده از فهرست گزارش کنید. ClawHub کد skill یا plugin
شخص ثالث را نگهداری یا وصله نمی‌کند.

GitHub Security Advisories برای `openclaw/clawhub` برای آسیب‌پذیری‌ها در
خود ClawHub هستند. نمونه‌ها شامل باگ‌هایی در وب‌سایت، API، CLI، رجیستری، احراز هویت،
اسکن، تعدیل محتوا، یا مرزهای اعتماد دانلود/نصب می‌شوند. از advisories مربوط به ClawHub
برای آسیب‌پذیری‌ها در Skills یا plugins شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و قابل اقدام هستند. سوءاستفاده از گزارش‌دهی می‌تواند خود باعث
اقدام علیه حساب شود.

## ادعاهای سازمان و namespace

اختلاف‌های مالکیت سازمان، برند، scope بسته، handle مالک، یا namespace باید
از فرایند [ادعاهای سازمان و Namespace](/fa/clawhub/namespace-claims) استفاده کنند، نه جریان گزارش
داخل محصول یا فرم درخواست تجدیدنظر حساب.

وقتی به کارکنان ClawHub نیاز دارید تا مدرک غیرحساسی را بررسی کنند که نشان می‌دهد یک
namespace باید رزرو، منتقل، تغییرنام، پنهان، قرنطینه، alias،
یا به شکل دیگری بررسی شود، از آن فرایند استفاده کنید. در یک issue عمومی، اسرار، اسناد خصوصی، پرونده‌های حقوقی خصوصی،
اسناد هویتی شخصی، توکن‌های API، یا توکن‌های چالش DNS را وارد نکنید.

## توقف‌های تعدیل محتوا

برخی یافته‌های شدید یا مسائل خط‌مشی می‌توانند ناشر یا فهرست را تحت
توقف تعدیل محتوا قرار دهند. وقتی این اتفاق می‌افتد، محتوای متاثر ممکن است از کشف عمومی
پنهان شود یا انتشارهای آینده ممکن است تا زمان بررسی مسئله به صورت پنهان شروع شوند.

توقف‌های تعدیل محتوا برای محافظت از کاربران در حالی است که ClawHub موارد پرریسک
را حل می‌کند. وقتی مثبت کاذب تایید شود نیز می‌توانند برداشته شوند.

## فهرست‌های پنهان یا مسدودشده

یک فهرست ممکن است نگه‌داشته، پنهان، قرنطینه، لغو، یا به شکل دیگری روی
سطوح نصب عمومی در دسترس نباشد.

اگر یکی از این وضعیت‌ها را می‌بینید، انتشار را نصب نکنید مگر اینکه مالک
مسئله را حل کند یا تعدیل محتوا آن را بازیابی کند.

مالکان ممکن است همچنان تشخیص‌ها را برای فهرست‌های نگه‌داشته یا پنهان خود ببینند. این
تشخیص‌ها کمک می‌کنند توضیح دهند چه اتفاقی افتاده و پیش از اینکه
فهرست بتواند به سطوح عمومی بازگردد چه چیزی باید تغییر کند.

## محرومیت‌ها و وضعیت حساب

حساب‌هایی که خط‌مشی ClawHub را نقض کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفاده شدید می‌تواند
به محرومیت حساب، لغو توکن، محتوای پنهان، یا فهرست‌های حذف‌شده منجر شود.
سیگنال‌های فشار سوءاستفاده ناشر روزانه بررسی می‌شوند. سیگنال‌هایی که به
آستانه محرومیت احتمالی ClawHub برسند می‌توانند یک هشدار خودکار را فعال کنند. اگر اسکن
واجد شرایط بعدی پس از مهلت هشدار همچنان ناشر را در
آستانه محرومیت احتمالی قرار دهد، ClawHub ممکن است اقدام حساب را به‌طور خودکار اعمال کند.
سیگنال‌های بازبینی با اطمینان کمتر و محدود به بازه زمانی از اجرای خودکار
خارج می‌مانند.

حساب‌های حذف‌شده، محروم‌شده، یا غیرفعال نمی‌توانند از توکن‌های API ClawHub استفاده کنند. اگر احراز هویت CLI
پس از اقدام حساب شروع به شکست کرد، برای بررسی وضعیت حساب وارد UI وب شوید.
اگر ورود یا دسترسی عادی CLI به دلیل محرومیت یا حساب غیرفعال مسدود شده است،
از [فرم درخواست تجدیدنظر ClawHub](https://appeals.openclaw.ai/) برای بازبینی بازیابی استفاده کنید.

اگر یک ایمیل فعال‌شده توسط اسکنر نسخه‌ای از یک skill یا plugin را مخرب نامید،
نتایج اسکن ذخیره‌شده را برای نسخه ارسالی مسدودشده دانلود کنید:
`clawhub scan download <slug> --version <version>`. برای plugins،
`--kind plugin` را اضافه کنید. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید، شماره نسخه
را افزایش دهید، و نسخه اصلاح‌شده را بارگذاری کنید.

## راهنمای ناشر

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربر:

- نام‌ها، خلاصه‌ها، برچسب‌ها، و تغییرنامه‌ها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع پیوند دهید
- پیش از انتشار plugins از dry runها استفاده کنید
- اگر کاربران یا ناظران درباره رفتار انتشار می‌پرسند، روشن پاسخ دهید
