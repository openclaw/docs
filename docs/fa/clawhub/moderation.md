# تعدیل محتوا و ایمنی حساب

ClawHub برای انتشار باز است، اما سطوح کشف عمومی و نصب همچنان به
حفاظت‌های کنترلی نیاز دارند. گزارش‌ها، توقف‌های تعدیل، فهرست‌های پنهان، و اقدامات حساب
زمانی که یک انتشار یا حساب ناامن، گمراه‌کننده، یا خارج از سیاست به نظر برسد،
به محافظت از کاربران کمک می‌کنند.

این صفحه تعدیل محتوا و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های حسابرسی مانند
`Pass`، `Review`، `Warn`، `Malicious` و سطح ریسک، ببینید
[حسابرسی‌های امنیتی](/clawhub/security-audits).

همچنین [امنیت](/clawhub/security) و
[استفاده قابل قبول](/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های مربوط به حق نشر یا سایر
حقوق محتوا، از [درخواست‌های حقوق محتوا](/clawhub/content-rights) استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند Skills، Pluginها، و بسته‌ها را گزارش کنند.

از گزارش‌های ClawHub فقط برای محتوای ناامن بازارگاه استفاده کنید، مانند:

- فهرست‌های مخرب
- فراداده گمراه‌کننده
- الزامات اعلام‌نشده برای اعتبارنامه‌ها یا مجوزها
- دستورالعمل‌های نصب مشکوک
- جعل هویت
- ثبت‌نام‌های سوءنیت‌آمیز یا سوءاستفاده از نشان تجاری
- محتوایی که [استفاده قابل قبول](/clawhub/acceptable-usage) را نقض می‌کند

از دکمه **گزارش Skills** در صفحه Skills، یا فرمان/API گزارش بسته
برای بسته‌ها استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌ها در کد منبع خود Skills یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبعی
گزارش کنید که از فهرست پیوند داده شده است. ClawHub کد Skills یا Plugin
شخص ثالث را نگهداری یا وصله نمی‌کند.

GitHub Security Advisories برای `openclaw/clawhub` مخصوص آسیب‌پذیری‌ها در
خود ClawHub هستند. نمونه‌ها شامل باگ‌ها در وب‌سایت، API، CLI، رجیستری، احراز هویت،
اسکن، تعدیل محتوا، یا مرزهای اعتماد دانلود/نصب می‌شوند. از مشاوره‌های ClawHub
برای آسیب‌پذیری‌ها در Skills یا Pluginهای شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و قابل اقدام هستند. سوءاستفاده از گزارش‌دهی می‌تواند خودش به
اقدام حساب منجر شود.

## ادعاهای سازمان و فضای نام

اختلاف‌های مالکیت سازمان، برند، دامنه بسته، شناسه مالک، یا فضای نام باید
از فرایند [ادعاهای سازمان و فضای نام](/clawhub/namespace-claims) استفاده کنند، نه از
جریان گزارش درون محصول یا فرم درخواست تجدیدنظر حساب.

وقتی نیاز دارید کارکنان ClawHub مدرک غیرحساسی را بررسی کنند که نشان می‌دهد یک
فضای نام باید رزرو، منتقل، تغییرنام، پنهان، قرنطینه، مستعارسازی،
یا به شکل دیگری بازبینی شود، از آن فرایند استفاده کنید. اسرار، اسناد خصوصی، پرونده‌های حقوقی خصوصی،
مدارک هویتی شخصی، توکن‌های API، یا توکن‌های چالش DNS را در یک
issue عمومی قرار ندهید.

## توقف‌های تعدیل محتوا

برخی یافته‌های شدید یا مسائل سیاستی می‌توانند یک ناشر یا فهرست را تحت
توقف تعدیل محتوا قرار دهند. وقتی این اتفاق می‌افتد، محتوای تحت تأثیر ممکن است از
کشف عمومی پنهان شود یا انتشارهای آینده تا زمان بازبینی مسئله به‌صورت پنهان شروع شوند.

توقف‌های تعدیل محتوا برای محافظت از کاربران در زمانی هستند که ClawHub موارد پرریسک را
حل‌وفصل می‌کند. همچنین وقتی مثبت کاذب تأیید شود، می‌توان آن‌ها را برداشت.

## فهرست‌های پنهان یا مسدودشده

یک فهرست ممکن است نگه داشته، پنهان، قرنطینه، لغو، یا به شکل دیگری در
سطوح نصب عمومی در دسترس نباشد.

اگر یکی از این وضعیت‌ها را می‌بینید، انتشار را نصب نکنید مگر اینکه مالک
مسئله را حل کند یا تعدیل محتوا آن را بازگرداند.

مالکان ممکن است همچنان عیب‌یابی‌های فهرست‌های نگه‌داشته‌شده یا پنهان خود را ببینند. این
عیب‌یابی‌ها کمک می‌کنند توضیح داده شود چه اتفاقی افتاده و پیش از اینکه
فهرست بتواند به سطوح عمومی برگردد، چه چیزی باید تغییر کند.

## ممنوعیت‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض می‌کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفاده شدید می‌تواند
به ممنوعیت حساب، لغو توکن، محتوای پنهان، یا حذف فهرست‌ها منجر شود.
سیگنال‌های فشار سوءاستفاده ناشر روزانه بررسی می‌شوند. سیگنال‌هایی که به
آستانه ممنوعیت احتمالی ClawHub می‌رسند می‌توانند یک هشدار خودکار را فعال کنند. اگر اسکن
واجد شرایط بعدی پس از مهلت هشدار همچنان ناشر را در
آستانه ممنوعیت احتمالی قرار دهد، ClawHub ممکن است اقدام حساب را به‌صورت خودکار اعمال کند.
سیگنال‌های بازبینی با اطمینان پایین‌تر و محدود به زمان از اجرای خودکار
بیرون می‌مانند.

حساب‌های حذف‌شده، ممنوع‌شده، یا غیرفعال‌شده نمی‌توانند از توکن‌های API ClawHub استفاده کنند. اگر احراز هویت CLI
پس از اقدام حساب شروع به شکست کرد، وارد رابط وب شوید تا وضعیت حساب را
بررسی کنید. اگر ورود یا دسترسی عادی CLI به‌دلیل ممنوعیت یا حساب غیرفعال‌شده مسدود شده است،
برای بازبینی بازیابی از [فرم درخواست تجدیدنظر ClawHub](https://appeals.openclaw.ai/) استفاده کنید.

اگر ایمیلی که توسط اسکنر فعال شده است نسخه‌ای از Skills یا Plugin را مخرب نام می‌برد،
نتایج اسکن ذخیره‌شده برای نسخه ارسالی مسدودشده را دانلود کنید:
`clawhub scan download <slug> --version <version>`. برای Pluginها،
`--kind plugin` را اضافه کنید. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید، شماره نسخه را
افزایش دهید، و نسخه اصلاح‌شده را بارگذاری کنید.

## راهنمایی ناشران

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربران:

- نام‌ها، خلاصه‌ها، برچسب‌ها، و تغییرنامه‌ها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- هرجا ممکن است به منبع پیوند دهید
- پیش از انتشار Pluginها از اجرای آزمایشی استفاده کنید
- اگر کاربران یا ناظران درباره رفتار انتشار سؤال کردند، روشن پاسخ دهید
