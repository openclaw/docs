# نظارت و ایمنی حساب

ClawHub برای انتشار باز است، اما سطوح کشف عمومی و نصب همچنان به
محافظ‌ها نیاز دارند. گزارش‌ها، نگه‌داشت‌های نظارتی، فهرست‌های پنهان و اقدامات
حساب کمک می‌کنند وقتی یک انتشار یا حساب ناامن، گمراه‌کننده یا خارج از سیاست به
نظر می‌رسد، از کاربران محافظت شود.

این صفحه نظارت و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های ممیزی مانند
`Pass`، `Review`، `Warn`، `Malicious` و سطح ریسک، به
[ممیزی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

همچنین [امنیت](/fa/clawhub/security) و
[استفاده قابل قبول](/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های
حق نشر یا دیگر حقوق محتوا، از [درخواست‌های حقوق محتوا](/clawhub/content-rights)
استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند Skills، Pluginها و بسته‌ها را گزارش کنند.

از گزارش‌های ClawHub فقط برای محتوای ناامن بازارگاه استفاده کنید، مانند:

- فهرست‌های مخرب
- فراداده گمراه‌کننده
- نیازمندی‌های اعلام‌نشده برای اعتبارنامه‌ها یا مجوزها
- دستورالعمل‌های نصب مشکوک
- جعل هویت
- ثبت‌نام‌های بدخواهانه یا سوءاستفاده از نشان تجاری
- محتوایی که [استفاده قابل قبول](/clawhub/acceptable-usage) را نقض می‌کند

از دکمه **گزارش Skill** در صفحه یک Skill، یا دستور/API گزارش بسته برای
بسته‌ها استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ یک Skill یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیما به ناشر یا مخزن منبعی که از
فهرست لینک شده است گزارش کنید. ClawHub کد Skill یا Plugin شخص ثالث را نگهداری
یا وصله نمی‌کند.

GitHub Security Advisories برای `openclaw/clawhub` مخصوص آسیب‌پذیری‌های خود
ClawHub هستند. نمونه‌ها شامل اشکال‌ها در وب‌سایت، API، CLI، رجیستری، احراز
هویت، اسکن، نظارت، یا مرزهای اعتماد دانلود/نصب است. از مشاوره‌های ClawHub برای
آسیب‌پذیری‌های Skills یا Pluginهای شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و قابل اقدام هستند. سوءاستفاده از گزارش‌دهی می‌تواند خودش
به اقدام حساب منجر شود.

## ادعاهای سازمان و فضای نام

اختلاف‌های مالکیت سازمان، برند، محدوده بسته، شناسه مالک، یا فضای نام باید از
فرایند [ادعاهای سازمان و فضای نام](/clawhub/namespace-claims) استفاده کنند، نه
جریان گزارش درون محصول یا فرم تجدیدنظر حساب.

وقتی نیاز دارید کارکنان ClawHub مدرک غیرحساسی را بررسی کنند که نشان می‌دهد یک
فضای نام باید رزرو، منتقل، تغییرنام، پنهان، قرنطینه، دارای نام مستعار، یا به
شکل دیگری بررسی شود، از آن فرایند استفاده کنید. اسرار، اسناد خصوصی، فایل‌های
حقوقی خصوصی، مدارک هویت شخصی، توکن‌های API، یا توکن‌های چالش DNS را در یک
مسئله عمومی قرار ندهید.

## نگه‌داشت‌های نظارتی

برخی یافته‌های شدید یا مسائل سیاستی می‌توانند یک ناشر یا فهرست را تحت
نگه‌داشت نظارتی قرار دهند. وقتی این اتفاق می‌افتد، محتوای متاثر ممکن است از
کشف عمومی پنهان شود یا انتشارهای آینده تا زمان بررسی مسئله به‌صورت پنهان شروع
شوند.

نگه‌داشت‌های نظارتی برای محافظت از کاربران طراحی شده‌اند تا ClawHub موارد
پرریسک را حل کند. همچنین وقتی یک مثبت کاذب تایید شود، می‌توان آن‌ها را برداشت.

## فهرست‌های پنهان یا مسدودشده

یک فهرست ممکن است روی سطوح نصب عمومی نگه‌داشته، پنهان، قرنطینه، لغو، یا به
شکل دیگری ناموجود شود.

اگر یکی از این وضعیت‌ها را دیدید، انتشار را نصب نکنید مگر اینکه مالک مسئله را
حل کند یا نظارت آن را بازگرداند.

مالکان ممکن است همچنان عیب‌یابی‌های فهرست‌های نگه‌داشته‌شده یا پنهان خود را
ببینند. این عیب‌یابی‌ها کمک می‌کنند توضیح داده شود چه اتفاقی افتاده و پیش از
بازگشت فهرست به سطوح عمومی چه چیزی باید تغییر کند.

## مسدودسازی‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض کنند ممکن است دسترسی انتشار را از دست
بدهند. سوءاستفاده شدید می‌تواند به مسدودسازی حساب، لغو توکن، محتوای پنهان، یا
فهرست‌های حذف‌شده منجر شود. سیگنال‌های فشار سوءاستفاده ناشر روزانه بررسی
می‌شوند. سیگنال‌هایی که به آستانه احتمال مسدودسازی ClawHub برسند می‌توانند یک
هشدار خودکار را فعال کنند. اگر اسکن واجد شرایط بعدی پس از مهلت هشدار همچنان
ناشر را در آستانه احتمال مسدودسازی قرار دهد، ClawHub ممکن است اقدام حساب را
به‌صورت خودکار اعمال کند. سیگنال‌های بررسی زمانی با اطمینان کمتر و محدود، از
اجرای خودکار خارج می‌مانند.

حساب‌های حذف‌شده، مسدودشده، یا غیرفعال نمی‌توانند از توکن‌های API ClawHub
استفاده کنند. اگر احراز هویت CLI پس از اقدام حساب شروع به شکست کرد، وارد رابط
وب شوید تا وضعیت حساب را بررسی کنید. اگر ورود یا دسترسی عادی CLI به دلیل
مسدودسازی یا حساب غیرفعال مسدود شده است، از
[فرم تجدیدنظر ClawHub](https://appeals.openclaw.ai/) برای بررسی بازیابی استفاده
کنید.

اگر یک ایمیل فعال‌شده توسط اسکنر نسخه‌ای از Skill یا Plugin را مخرب نامید،
نتایج اسکن ذخیره‌شده برای نسخه ارسال‌شده مسدودشده را دانلود کنید:
`clawhub scan download <slug> --version <version>`. برای Pluginها،
`--kind plugin` را اضافه کنید. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید،
شماره نسخه را افزایش دهید و نسخه اصلاح‌شده را بارگذاری کنید.

## راهنمایی ناشر

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربر:

- نام‌ها، خلاصه‌ها، برچسب‌ها و تغییرنامه‌ها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای مورد نیاز را اعلام کنید
- از دستورهای نصب مبهم‌سازی‌شده پرهیز کنید
- هر زمان ممکن است به منبع لینک دهید
- پیش از انتشار Pluginها از اجرای آزمایشی استفاده کنید
- اگر کاربران یا ناظران درباره رفتار انتشار پرسیدند، شفاف پاسخ دهید
