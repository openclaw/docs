# نظارت و ایمنی حساب

ClawHub برای انتشار باز است، اما سطوح کشف عمومی و نصب همچنان به
حفاظت‌های کنترلی نیاز دارند. گزارش‌ها، توقف‌های نظارتی، فهرست‌های پنهان، و اقدامات حساب
کمک می‌کنند وقتی یک انتشار یا حساب ناامن، گمراه‌کننده، یا خارج از خط‌مشی به نظر می‌رسد،
از کاربران محافظت شود.

این صفحه نظارت و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های حسابرسی مانند
`Pass`، `Review`، `Warn`، `Malicious`، و سطح ریسک، [ممیزی‌های امنیتی](/fa/clawhub/security-audits) را ببینید.

همچنین [امنیت](/fa/clawhub/security) و
[استفاده قابل‌قبول](/fa/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های مربوط به حق نشر یا دیگر
حقوق محتوا، از [درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند Skills، pluginها، و بسته‌ها را گزارش کنند.

از گزارش‌های ClawHub فقط برای محتوای ناامن بازار استفاده کنید، مانند:

- فهرست‌های مخرب
- فراداده گمراه‌کننده
- الزامات اعلام‌نشده برای اعتبارنامه‌ها یا مجوزها
- دستورالعمل‌های نصب مشکوک
- جعل هویت
- ثبت‌های با سوءنیت یا سوءاستفاده از علامت تجاری
- محتوایی که [استفاده قابل‌قبول](/fa/clawhub/acceptable-usage) را نقض می‌کند

از دکمه **گزارش skill** در صفحه skill، یا فرمان/API گزارش بسته
برای بسته‌ها استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌ها در کد منبع خودِ یک skill یا
plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبعی
گزارش کنید که از فهرست پیوند شده است. ClawHub کد skill یا plugin
شخص ثالث را نگه‌داری یا وصله نمی‌کند.

GitHub Security Advisories برای `openclaw/clawhub` مخصوص آسیب‌پذیری‌ها در
خود ClawHub است. نمونه‌ها شامل باگ‌ها در وب‌سایت، API، CLI، رجیستری، احراز هویت،
اسکن، نظارت، یا مرزهای اعتماد دانلود/نصب است. از advisoryهای ClawHub
برای آسیب‌پذیری‌ها در Skills یا pluginهای شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و قابل اقدام هستند. سوءاستفاده از گزارش‌دهی می‌تواند خود
به اقدام علیه حساب منجر شود.

## ادعاهای سازمان و namespace

اختلافات مالکیت سازمان، برند، دامنه بسته، شناسه مالک، یا namespace باید
از فرایند [ادعاهای سازمان و Namespace](/fa/clawhub/namespace-claims) استفاده کنند، نه
جریان گزارش درون‌محصول یا فرم درخواست تجدیدنظر حساب.

وقتی به کارکنان ClawHub نیاز دارید تا مدرک غیرحساسی را بررسی کنند که نشان می‌دهد یک
namespace باید رزرو، منتقل، تغییرنام، پنهان، قرنطینه، دارای alias،
یا به‌شکل دیگری بررسی شود، از آن فرایند استفاده کنید. اسرار، اسناد خصوصی، پرونده‌های حقوقی خصوصی،
اسناد هویت شخصی، توکن‌های API، یا توکن‌های چالش DNS را در یک
issue عمومی وارد نکنید.

## توقف‌های نظارتی

برخی یافته‌های شدید یا مسائل خط‌مشی می‌توانند یک ناشر یا فهرست را تحت
توقف نظارتی قرار دهند. وقتی این اتفاق می‌افتد، محتوای متاثر ممکن است از
کشف عمومی پنهان شود یا انتشارهای آینده تا زمان بررسی مسئله به‌صورت پنهان شروع شوند.

توقف‌های نظارتی برای محافظت از کاربران در زمانی هستند که ClawHub پرونده‌های پرریسک
را حل‌وفصل می‌کند. همچنین وقتی مثبت کاذب تایید شود، می‌توان آن‌ها را برداشت.

## فهرست‌های پنهان یا مسدود

یک فهرست ممکن است در حالت توقف، پنهان، قرنطینه‌شده، لغوشده، یا به‌شکل دیگری روی
سطوح نصب عمومی در دسترس نباشد.

اگر یکی از این وضعیت‌ها را می‌بینید، انتشار را نصب نکنید مگر اینکه مالک
مسئله را حل کند یا نظارت آن را بازگرداند.

مالکان ممکن است همچنان برای فهرست‌های خود که در حالت توقف یا پنهان هستند، عیب‌یابی‌ها را ببینند. این
عیب‌یابی‌ها کمک می‌کنند توضیح داده شود چه اتفاقی افتاده و پیش از بازگشت
فهرست به سطوح عمومی چه چیزی باید تغییر کند.

## ممنوعیت‌ها و وضعیت حساب

حساب‌هایی که خط‌مشی ClawHub را نقض کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفاده شدید می‌تواند
به ممنوعیت حساب، لغو توکن، محتوای پنهان، یا حذف فهرست‌ها منجر شود.
سیگنال‌های فشار سوءاستفاده ناشر به‌صورت روزانه بررسی می‌شوند. سیگنال‌هایی که به
آستانه ممنوعیت احتمالی ClawHub برسند می‌توانند یک هشدار خودکار را فعال کنند. اگر اسکن
واجد شرایط بعدی پس از مهلت هشدار همچنان ناشر را در
آستانه ممنوعیت احتمالی قرار دهد، ClawHub ممکن است اقدام حساب را به‌صورت خودکار اعمال کند.
سیگنال‌های بررسی با اطمینان پایین‌تر و محدود به بازه زمانی از اجرای خودکار
خارج می‌مانند.

حساب‌های حذف‌شده، ممنوع‌شده، یا غیرفعال‌شده نمی‌توانند از توکن‌های API ClawHub استفاده کنند. اگر احراز هویت CLI
پس از اقدام حساب شروع به شکست خوردن کرد، برای بررسی وضعیت حساب
وارد رابط وب شوید. اگر ورود یا دسترسی عادی CLI به‌دلیل ممنوعیت یا حساب غیرفعال‌شده مسدود است،
برای بررسی بازیابی از [فرم درخواست تجدیدنظر ClawHub](https://appeals.openclaw.ai/) استفاده کنید.

اگر یک ایمیل فعال‌شده توسط اسکنر نسخه‌ای از skill یا plugin را مخرب نامید،
نتایج اسکن ذخیره‌شده را برای نسخه ارسالی مسدودشده دانلود کنید:
`clawhub scan download <slug> --version <version>`. برای pluginها،
`--kind plugin` را اضافه کنید. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید، شماره نسخه
را افزایش دهید، و نسخه اصلاح‌شده را بارگذاری کنید.

## راهنمای ناشران

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربران:

- نام‌ها، خلاصه‌ها، برچسب‌ها، و changelogها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع پیوند دهید
- پیش از انتشار pluginها از اجرای آزمایشی استفاده کنید
- اگر کاربران یا ناظران درباره رفتار انتشار پرسیدند، شفاف پاسخ دهید
