# نظارت و ایمنی حساب

ClawHub برای انتشار باز است، اما سطوح کشف عمومی و نصب همچنان به
حفاظت‌گر نیاز دارند. گزارش‌ها، توقف‌های نظارتی، فهرست‌های پنهان و اقدامات
حساب کمک می‌کنند زمانی که یک انتشار یا حساب ناامن، گمراه‌کننده یا خارج از
سیاست به نظر می‌رسد، از کاربران محافظت شود.

این صفحه نظارت و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های ممیزی مانند
`Pass`، `Review`، `Warn`، `Malicious` و سطح ریسک، [ممیزی‌های امنیتی](/clawhub/security-audits) را ببینید.

همچنین [امنیت](/clawhub/security) و
[استفاده قابل قبول](/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های
کپی‌رایت یا سایر حقوق محتوا، از [درخواست‌های حقوق محتوا](/clawhub/content-rights) استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند Skills، Pluginها و بسته‌ها را گزارش کنند.

از گزارش‌های ClawHub فقط برای محتوای ناامن بازارچه استفاده کنید، مانند:

- فهرست‌های مخرب
- فراداده گمراه‌کننده
- نیازمندی‌های اعلام‌نشده برای اعتبارنامه‌ها یا مجوزها
- دستورالعمل‌های نصب مشکوک
- جعل هویت
- ثبت‌های با سوءنیت یا سوءاستفاده از نشان تجاری
- محتوایی که [استفاده قابل قبول](/clawhub/acceptable-usage) را نقض می‌کند

از دکمه **گزارش Skill** در صفحه Skill، یا فرمان/API گزارش بسته برای بسته‌ها استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ Skill یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبعی که از
فهرست پیوند شده گزارش کنید. ClawHub کد Skill یا Plugin شخص ثالث را نگهداری یا وصله نمی‌کند.

GitHub Security Advisories برای `openclaw/clawhub` مربوط به آسیب‌پذیری‌های
خود ClawHub است. نمونه‌ها شامل باگ‌ها در وب‌سایت، API، CLI، رجیستری، احراز
هویت، اسکن، نظارت، یا مرزهای اعتماد دانلود/نصب هستند. از advisoryهای ClawHub
برای آسیب‌پذیری‌های Skills یا Pluginهای شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و قابل اقدام هستند. سوءاستفاده از گزارش‌دهی می‌تواند خود
به اقدام حساب منجر شود.

## ادعاهای سازمان و فضای نام

اختلاف‌های مالکیت سازمان، برند، دامنه بسته، شناسه مالک، یا فضای نام باید از
فرآیند [ادعاهای سازمان و فضای نام](/clawhub/namespace-claims) استفاده کنند،
نه جریان گزارش درون‌محصولی یا فرم فرجام‌خواهی حساب.

وقتی لازم است کارکنان ClawHub مدرک غیرحساسی را بررسی کنند که نشان می‌دهد یک
فضای نام باید رزرو، منتقل، تغییرنام، پنهان، قرنطینه، مستعارسازی یا به شکل
دیگری بررسی شود، از آن فرآیند استفاده کنید. اسرار، اسناد خصوصی، فایل‌های
حقوقی خصوصی، مدارک هویتی شخصی، توکن‌های API، یا توکن‌های چالش DNS را در یک
issue عمومی قرار ندهید.

## توقف‌های نظارتی

برخی یافته‌های شدید یا مسائل سیاستی می‌توانند ناشر یا فهرست را تحت توقف
نظارتی قرار دهند. وقتی این اتفاق می‌افتد، محتوای تحت تأثیر ممکن است از کشف
عمومی پنهان شود یا انتشارهای آینده تا زمان بررسی مسئله به صورت پنهان آغاز شوند.

هدف توقف‌های نظارتی محافظت از کاربران در زمانی است که ClawHub پرونده‌های
پرریسک را حل‌وفصل می‌کند. همچنین وقتی مثبت کاذب تأیید شود، می‌توان آن‌ها را برداشت.

## فهرست‌های پنهان یا مسدودشده

یک فهرست ممکن است در حالت توقف، پنهان، قرنطینه، لغوشده یا به شکل دیگری در
سطوح نصب عمومی unavailable باشد.

اگر یکی از این وضعیت‌ها را می‌بینید، انتشار را نصب نکنید مگر اینکه مالک مسئله
را حل کند یا نظارت آن را بازگرداند.

مالکان ممکن است همچنان برای فهرست‌های متوقف یا پنهان خودشان عیب‌یابی ببینند.
این عیب‌یابی‌ها کمک می‌کنند توضیح داده شود چه اتفاقی افتاده و پیش از بازگشت
فهرست به سطوح عمومی چه چیزی باید تغییر کند.

## ممنوعیت‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض می‌کنند ممکن است دسترسی انتشار را از دست
بدهند. سوءاستفاده شدید می‌تواند به ممنوعیت حساب، لغو توکن، محتوای پنهان یا
فهرست‌های حذف‌شده منجر شود. سیگنال‌های فشار سوءاستفاده ناشر روزانه بررسی
می‌شوند. سیگنال‌هایی که به آستانه ممنوعیت احتمالی ClawHub برسند می‌توانند یک
هشدار خودکار را فعال کنند. اگر اسکن واجد شرایط بعدی پس از مهلت هشدار همچنان
ناشر را در آستانه ممنوعیت احتمالی قرار دهد، ClawHub ممکن است اقدام حساب را
به‌صورت خودکار اعمال کند. سیگنال‌های بررسی با اطمینان کمتر و زمان‌مند محدود
از اجرای خودکار کنار گذاشته می‌شوند.

حساب‌های حذف‌شده، ممنوع‌شده یا غیرفعال‌شده نمی‌توانند از توکن‌های API در
ClawHub استفاده کنند. اگر احراز هویت CLI پس از اقدام حساب شروع به شکست کرد،
برای بررسی وضعیت حساب وارد رابط وب شوید. اگر ورود یا دسترسی عادی CLI به دلیل
ممنوعیت یا حساب غیرفعال‌شده مسدود است، برای بررسی بازیابی از [فرم فرجام‌خواهی ClawHub](https://appeals.openclaw.ai/) استفاده کنید.

اگر ایمیلی که توسط اسکنر فعال شده، نسخه‌ای از Skill یا Plugin را به‌عنوان
مخرب نام ببرد، نتایج اسکن ذخیره‌شده را برای نسخه ارسالی مسدودشده دانلود کنید:
`clawhub scan download <slug> --version <version>`. برای Pluginها، `--kind plugin` را اضافه کنید. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید، شماره
نسخه را افزایش دهید و نسخه اصلاح‌شده را بارگذاری کنید.

## راهنمای ناشر

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربر:

- نام‌ها، خلاصه‌ها، برچسب‌ها و changelogها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع پیوند دهید
- پیش از انتشار Pluginها از اجراهای آزمایشی استفاده کنید
- اگر کاربران یا ناظران درباره رفتار انتشار سؤال کردند، روشن پاسخ دهید
