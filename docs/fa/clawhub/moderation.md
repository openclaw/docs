# تعدیل محتوا و ایمنی حساب

ClawHub برای انتشار باز است، اما سطح‌های کشف عمومی و نصب همچنان به
محافظ نیاز دارند. گزارش‌ها، تعلیق‌های تعدیل محتوا، فهرست‌های پنهان، و اقدامات حساب
به محافظت از کاربران کمک می‌کنند، هنگامی که یک انتشار یا حساب ناامن، گمراه‌کننده، یا
خارج از سیاست به نظر می‌رسد.

این صفحه تعدیل محتوا و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های ممیزی مانند
`Pass`، `Review`، `Warn`، `Malicious`، و سطح ریسک، ببینید
[ممیزی‌های امنیتی](/fa/clawhub/security-audits).

همچنین [امنیت](/fa/clawhub/security) و
[استفاده قابل قبول](/fa/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های مربوط به حق نشر یا سایر
حقوق محتوا، از [درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند Skillها، Pluginها، و بسته‌ها را گزارش کنند.

از گزارش‌های ClawHub فقط برای محتوای ناامن بازار استفاده کنید، مانند:

- فهرست‌های مخرب
- فراداده گمراه‌کننده
- نیازمندی‌های اعلام‌نشده برای اعتبارنامه‌ها یا مجوزها
- دستورالعمل‌های نصب مشکوک
- جعل هویت
- ثبت‌نام‌های با سوءنیت یا سوءاستفاده از علامت تجاری
- محتوایی که [استفاده قابل قبول](/fa/clawhub/acceptable-usage) را نقض می‌کند

از دکمه **گزارش Skill** در صفحه Skill، یا فرمان/API گزارش بسته
برای بسته‌ها استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ Skill یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیما به ناشر یا مخزن منبع
پیوند داده‌شده در فهرست گزارش کنید. ClawHub کد Skill یا Plugin
شخص ثالث را نگهداری یا وصله نمی‌کند.

GitHub Security Advisories برای `openclaw/clawhub` مخصوص آسیب‌پذیری‌های خود
ClawHub است. نمونه‌ها شامل باگ‌ها در وب‌سایت، API، CLI، رجیستری، احراز هویت،
اسکن، تعدیل محتوا، یا مرزهای اعتماد دانلود/نصب هستند. از advisories مربوط به ClawHub
برای آسیب‌پذیری‌های Skillها یا Pluginهای شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و قابل اقدام هستند. سوءاستفاده از گزارش‌دهی می‌تواند خود باعث
اقدام علیه حساب شود.

## ادعاهای سازمان و namespace

اختلاف‌های مربوط به مالکیت سازمان، برند، محدوده بسته، شناسه مالک، یا namespace باید
از فرایند [ادعاهای سازمان و Namespace](/fa/clawhub/namespace-claims) استفاده کنند، نه
جریان گزارش درون محصول یا فرم اعتراض حساب.

وقتی به کارکنان ClawHub نیاز دارید تا مدرک غیرحساسی را بررسی کنند که نشان می‌دهد یک
namespace باید رزرو، منتقل، تغییر نام، پنهان، قرنطینه، دارای alias،
یا به‌نحوی دیگر بررسی شود، از آن فرایند استفاده کنید. اسرار، اسناد خصوصی، فایل‌های حقوقی خصوصی،
مدارک هویت شخصی، توکن‌های API، یا توکن‌های چالش DNS را در یک
issue عمومی قرار ندهید.

## تعلیق‌های تعدیل محتوا

برخی یافته‌های شدید یا مسائل سیاستی می‌توانند یک ناشر یا فهرست را تحت
تعلیق تعدیل محتوا قرار دهند. وقتی این اتفاق می‌افتد، محتوای متاثر ممکن است از کشف عمومی
پنهان شود یا انتشارهای آینده ممکن است تا زمان بررسی مسئله به‌صورت پنهان شروع شوند.

تعلیق‌های تعدیل محتوا برای محافظت از کاربران هستند، تا وقتی ClawHub پرونده‌های پرریسک را
حل می‌کند. همچنین هنگامی که مثبت کاذب تایید شود، می‌توان آن‌ها را برداشت.

## فهرست‌های پنهان یا مسدودشده

یک فهرست ممکن است در حالت تعلیق، پنهان، قرنطینه، لغوشده، یا به‌نحوی دیگر روی
سطح‌های نصب عمومی در دسترس نباشد.

اگر یکی از این حالت‌ها را می‌بینید، انتشار را نصب نکنید، مگر اینکه مالک
مسئله را حل کند یا تعدیل محتوا آن را بازگرداند.

مالکان ممکن است همچنان تشخیص‌های مربوط به فهرست‌های تعلیق‌شده یا پنهان خود را ببینند. این
تشخیص‌ها کمک می‌کنند توضیح دهند چه اتفاقی افتاده و چه چیزی باید تغییر کند تا
فهرست بتواند به سطح‌های عمومی بازگردد.

## ممنوعیت‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض می‌کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفاده شدید می‌تواند
به ممنوعیت حساب، لغو توکن، محتوای پنهان، یا فهرست‌های حذف‌شده منجر شود.
سیگنال‌های فشار سوءاستفاده ناشر روزانه بررسی می‌شوند. سیگنال‌هایی که به
آستانه احتمال ممنوعیت ClawHub می‌رسند، می‌توانند یک هشدار خودکار ایجاد کنند. اگر اسکن
واجد شرایط بعدی پس از مهلت هشدار همچنان ناشر را در
آستانه احتمال ممنوعیت قرار دهد، ClawHub ممکن است اقدام حساب را به‌صورت خودکار اعمال کند.
سیگنال‌های بررسی با اطمینان پایین‌تر و زمان‌مند محدود، وارد اجرای خودکار نمی‌شوند.

حساب‌های حذف‌شده، ممنوع‌شده، یا غیرفعال نمی‌توانند از توکن‌های API ClawHub استفاده کنند. اگر احراز هویت CLI
پس از اقدام حساب شروع به شکست کرد، برای بررسی وضعیت حساب وارد رابط وب شوید.
اگر ورود یا دسترسی عادی CLI به دلیل ممنوعیت یا حساب غیرفعال مسدود شده است،
برای بررسی بازیابی از [فرم اعتراض ClawHub](https://appeals.openclaw.ai/) استفاده کنید.

اگر ایمیل فعال‌شده توسط اسکنر نسخه‌ای از Skill یا Plugin را مخرب نام‌گذاری می‌کند،
نتایج اسکن ذخیره‌شده برای نسخه ارسالی مسدودشده را دانلود کنید:
`clawhub scan download <slug> --version <version>`. برای Pluginها، اضافه کنید
`--kind plugin`. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید، شماره نسخه را افزایش دهید،
و نسخه اصلاح‌شده را بارگذاری کنید.

## راهنمای ناشر

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربران:

- نام‌ها، خلاصه‌ها، برچسب‌ها، و changelogها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای موردنیاز را اعلام کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع پیوند دهید
- پیش از انتشار Pluginها از اجراهای آزمایشی استفاده کنید
- اگر کاربران یا ناظران درباره رفتار انتشار پرسیدند، شفاف پاسخ دهید
