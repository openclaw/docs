# مدیریت محتوا و ایمنی حساب

ClawHub برای انتشار باز است، اما سطوح کشف عمومی و نصب همچنان به
حفاظت نیاز دارند. گزارش‌ها، توقف‌های مدیریت محتوا، فهرست‌های پنهان، و اقدام‌های حساب
به محافظت از کاربران کمک می‌کنند وقتی یک انتشار یا حساب ناامن، گمراه‌کننده، یا خارج
از سیاست به نظر می‌رسد.

این صفحه مدیریت محتوا و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های ممیزی مانند
`Pass`، `Review`، `Warn`، `Malicious`، و سطح ریسک، به
[ممیزی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

همچنین [امنیت](/clawhub/security) و
[استفاده قابل‌قبول](/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های مربوط به حق نشر یا دیگر
حقوق محتوا، از [درخواست‌های حقوق محتوا](/clawhub/content-rights) استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند Skills، Pluginها، و بسته‌ها را گزارش کنند.

از گزارش‌های ClawHub فقط برای محتوای ناامن بازارچه استفاده کنید، مانند:

- فهرست‌های مخرب
- فراداده گمراه‌کننده
- اعتبارنامه‌ها یا الزامات مجوز اعلام‌نشده
- دستورالعمل‌های نصب مشکوک
- جعل هویت
- ثبت‌نام‌های بدخواهانه یا سوءاستفاده از نشان تجاری
- محتوایی که [استفاده قابل‌قبول](/clawhub/acceptable-usage) را نقض می‌کند

از دکمه **گزارش Skill** در صفحه Skill، یا فرمان/API گزارش بسته
برای بسته‌ها استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ یک Skill یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبع
پیوندشده از فهرست گزارش کنید. ClawHub کد Skill یا Plugin شخص ثالث را نگهداری یا وصله
نمی‌کند.

GitHub Security Advisories برای `openclaw/clawhub` مربوط به آسیب‌پذیری‌های خود
ClawHub است. نمونه‌ها شامل باگ‌ها در وب‌سایت، API، CLI، رجیستری، احراز هویت،
اسکن، مدیریت محتوا، یا مرزهای اعتماد دانلود/نصب هستند. از advisories مربوط به ClawHub
برای آسیب‌پذیری‌های موجود در Skills یا Pluginهای شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و قابل اقدام هستند. سوءاستفاده از گزارش‌دهی نیز می‌تواند به
اقدام حساب منجر شود.

## ادعاهای سازمان و namespace

اختلاف‌های مالکیت سازمان، برند، package-scope، owner-handle، یا namespace باید
از فرایند [ادعاهای سازمان و Namespace](/clawhub/namespace-claims) استفاده کنند، نه
جریان گزارش داخل محصول یا فرم درخواست تجدیدنظر حساب.

وقتی نیاز دارید کارکنان ClawHub مدرک غیرحساس را بررسی کنند که نشان می‌دهد یک
namespace باید رزرو، منتقل، تغییرنام، پنهان، قرنطینه، alias، یا به شکل دیگری
بررسی شود، از آن فرایند استفاده کنید. اسرار، اسناد خصوصی، پرونده‌های حقوقی خصوصی،
اسناد هویت شخصی، توکن‌های API، یا توکن‌های چالش DNS را در یک issue عمومی
قرار ندهید.

## توقف‌های مدیریت محتوا

برخی یافته‌های شدید یا مسائل سیاستی می‌توانند یک ناشر یا فهرست را تحت
توقف مدیریت محتوا قرار دهند. وقتی این اتفاق می‌افتد، محتوای تحت تأثیر ممکن است از
کشف عمومی پنهان شود یا انتشارهای آینده ممکن است تا زمان بررسی مسئله به‌صورت پنهان شروع شوند.

توقف‌های مدیریت محتوا برای محافظت از کاربران طراحی شده‌اند، در حالی که ClawHub موارد پرریسک
را حل‌وفصل می‌کند. همچنین وقتی مثبت کاذب تأیید شود، می‌توان آن‌ها را برداشت.

## فهرست‌های پنهان یا مسدودشده

یک فهرست ممکن است در وضعیت توقف، پنهان، قرنطینه، باطل‌شده، یا به شکل دیگری در
سطوح نصب عمومی در دسترس نباشد.

اگر یکی از این وضعیت‌ها را دیدید، انتشار را نصب نکنید مگر اینکه مالک
مسئله را حل کند یا مدیریت محتوا آن را بازیابی کند.

مالکان ممکن است همچنان تشخیص‌ها را برای فهرست‌های متوقف یا پنهان خود ببینند. این
تشخیص‌ها کمک می‌کنند توضیح داده شود چه اتفاقی افتاده و پیش از بازگشت
فهرست به سطوح عمومی چه چیزی باید تغییر کند.

## مسدودیت‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفاده شدید می‌تواند
به مسدودیت حساب، ابطال توکن، محتوای پنهان، یا حذف فهرست‌ها منجر شود.
سیگنال‌های فشار سوءاستفاده ناشر روزانه بررسی می‌شوند. سیگنال‌هایی که به
آستانه احتمال مسدودیت ClawHub برسند می‌توانند یک هشدار خودکار را فعال کنند. اگر اسکن
واجد شرایط بعدی پس از مهلت هشدار همچنان ناشر را در
آستانه احتمال مسدودیت قرار دهد، ClawHub ممکن است اقدام حساب را به‌صورت خودکار اعمال کند.
سیگنال‌های بررسی کم‌اطمینان‌تر و محدود به بازه زمانی از اجرای خودکار
خارج می‌مانند.

حساب‌های حذف‌شده، مسدودشده، یا غیرفعال نمی‌توانند از توکن‌های API ClawHub استفاده کنند. اگر احراز هویت CLI
پس از اقدام حساب شروع به شکست کرد، برای بررسی وضعیت حساب وارد UI وب شوید.
اگر ورود یا دسترسی عادی CLI به دلیل مسدودیت یا حساب غیرفعال مسدود شده است،
از [فرم درخواست تجدیدنظر ClawHub](https://appeals.openclaw.ai/) برای بررسی بازیابی استفاده کنید.

اگر یک ایمیل فعال‌شده توسط اسکنر، نسخه‌ای از Skill یا Plugin را مخرب معرفی کرد،
نتایج اسکن ذخیره‌شده را برای نسخه ارسالی مسدودشده دانلود کنید:
`clawhub scan download <slug> --version <version>`. برای Pluginها،
`--kind plugin` را اضافه کنید. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید، شماره نسخه
را افزایش دهید، و نسخه اصلاح‌شده را بارگذاری کنید.

## راهنمای ناشر

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربران:

- نام‌ها، خلاصه‌ها، برچسب‌ها، و changelogها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع پیوند دهید
- پیش از انتشار Pluginها از اجرای آزمایشی استفاده کنید
- اگر کاربران یا ناظران درباره رفتار انتشار پرسیدند، شفاف پاسخ دهید
