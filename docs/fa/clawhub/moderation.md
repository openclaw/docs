# ایمنی تعدیل محتوا و حساب

ClawHub برای انتشار باز است، اما سطوح کشف عمومی و نصب همچنان به
محافظت‌ها نیاز دارند. گزارش‌ها، نگه‌داشت‌های تعدیل محتوا، فهرست‌های پنهان و اقدام‌های حساب
به محافظت از کاربران کمک می‌کنند وقتی یک انتشار یا حساب ناامن، گمراه‌کننده یا خارج
از سیاست به نظر می‌رسد.

این صفحه تعدیل محتوا و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های ممیزی مانند
`Pass`، `Review`، `Warn`، `Malicious` و سطح ریسک، به
[ممیزی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

همچنین [امنیت](/clawhub/security) و
[استفاده قابل قبول](/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های مربوط به حق نشر یا سایر
حقوق محتوا، از [درخواست‌های حقوق محتوا](/clawhub/content-rights) استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند Skills، Pluginها و بسته‌ها را گزارش کنند.

گزارش‌های ClawHub را فقط برای محتوای ناامن بازارچه به کار ببرید، مانند:

- فهرست‌های مخرب
- فراداده گمراه‌کننده
- اعتبارنامه‌ها یا الزامات مجوز اعلام‌نشده
- دستورالعمل‌های نصب مشکوک
- جعل هویت
- ثبت‌های با سوءنیت یا سوءاستفاده از نشان تجاری
- محتوایی که [استفاده قابل قبول](/clawhub/acceptable-usage) را نقض می‌کند

از دکمه **گزارش Skills** در صفحه Skills، یا فرمان/API گزارش بسته
برای بسته‌ها استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ Skills یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیما به ناشر یا مخزن منبع
پیوندشده از فهرست گزارش کنید. ClawHub کد Skills یا Plugin شخص ثالث را
نگهداری یا وصله نمی‌کند.

GitHub Security Advisories برای `openclaw/clawhub` مربوط به آسیب‌پذیری‌های خود
ClawHub هستند. نمونه‌ها شامل اشکال‌ها در وب‌سایت، API، CLI، رجیستری، احراز هویت،
اسکن، تعدیل محتوا، یا مرزهای اعتماد دانلود/نصب هستند. از advisoriesهای ClawHub
برای آسیب‌پذیری‌های Skills یا Pluginهای شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و قابل اقدام هستند. سوءاستفاده از گزارش‌دهی می‌تواند خودش به
اقدام حساب منجر شود.

## ادعاهای سازمان و فضای نام

اختلاف‌های مالکیت سازمان، برند، محدوده بسته، شناسه مالک یا فضای نام باید
از فرایند [ادعاهای سازمان و فضای نام](/clawhub/namespace-claims) استفاده کنند، نه
جریان گزارش درون محصول یا فرم درخواست تجدیدنظر حساب.

وقتی نیاز دارید کارکنان ClawHub مدرک غیرحساسی را بررسی کنند که نشان می‌دهد یک
فضای نام باید رزرو، منتقل، تغییرنام، پنهان، قرنطینه، نام‌مستعاردهی یا به‌نحو دیگری
بررسی شود، از آن فرایند استفاده کنید. رازها، اسناد خصوصی، پرونده‌های حقوقی خصوصی،
اسناد هویت شخصی، توکن‌های API یا توکن‌های چالش DNS را در یک issue عمومی
قرار ندهید.

## نگه‌داشت‌های تعدیل محتوا

برخی یافته‌های شدید یا مسائل سیاستی می‌توانند یک ناشر یا فهرست را تحت
نگه‌داشت تعدیل محتوا قرار دهند. وقتی این اتفاق می‌افتد، محتوای تحت تأثیر ممکن است از
کشف عمومی پنهان شود یا انتشارهای آینده تا زمان بررسی مسئله به‌صورت پنهان شروع شوند.

نگه‌داشت‌های تعدیل محتوا برای محافظت از کاربران طراحی شده‌اند تا ClawHub موارد پرریسک
را حل کند. همچنین وقتی مثبت کاذب تأیید شود، می‌توانند برداشته شوند.

## فهرست‌های پنهان یا مسدودشده

یک فهرست ممکن است نگه داشته، پنهان، قرنطینه، لغو یا به‌نحو دیگری در
سطوح نصب عمومی ناموجود شود.

اگر یکی از این وضعیت‌ها را دیدید، انتشار را نصب نکنید مگر اینکه مالک
مسئله را حل کند یا تعدیل محتوا آن را بازگرداند.

مالکان ممکن است همچنان عیب‌یابی‌های مربوط به فهرست‌های نگه‌داشته‌شده یا پنهان خودشان را ببینند. این
عیب‌یابی‌ها کمک می‌کنند توضیح داده شود چه اتفاقی افتاده و پیش از بازگشت
فهرست به سطوح عمومی چه چیزی باید تغییر کند.

## مسدودسازی‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفاده شدید می‌تواند
به مسدودسازی حساب، لغو توکن، محتوای پنهان یا فهرست‌های حذف‌شده منجر شود.
سیگنال‌های فشار سوءاستفاده ناشر روزانه بررسی می‌شوند. سیگنال‌هایی که به
آستانه مسدودسازی احتمالی ClawHub برسند می‌توانند یک هشدار خودکار را فعال کنند. اگر اسکن
واجد شرایط بعدی پس از مهلت هشدار همچنان ناشر را در
آستانه مسدودسازی احتمالی قرار دهد، ClawHub ممکن است اقدام حساب را به‌صورت خودکار اعمال کند.
سیگنال‌های بررسی با اطمینان پایین‌تر و محدود به بازه زمانی از اجرای خودکار
خارج می‌مانند.

حساب‌های حذف‌شده، مسدودشده یا غیرفعال نمی‌توانند از توکن‌های API ClawHub استفاده کنند. اگر احراز هویت CLI
پس از اقدام حساب شروع به شکست کرد، وارد UI وب شوید تا وضعیت حساب را
بررسی کنید. اگر ورود یا دسترسی عادی CLI به دلیل مسدودسازی یا حساب غیرفعال مسدود شده است،
برای بررسی بازیابی از [فرم درخواست تجدیدنظر ClawHub](https://appeals.openclaw.ai/) استفاده کنید.

اگر یک ایمیل فعال‌شده توسط اسکنر، نسخه‌ای از Skills یا Plugin را به‌عنوان مخرب نام برد،
نتایج اسکن ذخیره‌شده را برای نسخه ارسالی مسدودشده دانلود کنید:
`clawhub scan download <slug> --version <version>`. برای Pluginها،
`--kind plugin` را اضافه کنید. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید، شماره نسخه
را افزایش دهید و نسخه اصلاح‌شده را بارگذاری کنید.

## راهنمای ناشران

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربران:

- نام‌ها، خلاصه‌ها، برچسب‌ها و تغییرات انتشار را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع پیوند دهید
- پیش از انتشار Pluginها از اجرای آزمایشی استفاده کنید
- اگر کاربران یا تعدیل‌کنندگان درباره رفتار انتشار پرسیدند، روشن پاسخ دهید
