# نظارت و ایمنی حساب

ClawHub برای انتشار باز است، اما سطوح کشف عمومی و نصب همچنان به
حفاظ‌ها نیاز دارند. گزارش‌ها، نگه‌داشت‌های نظارتی، فهرست‌های پنهان، و اقدام‌های حساب
به محافظت از کاربران کمک می‌کنند وقتی یک انتشار یا حساب ناامن، گمراه‌کننده، یا خارج
از خط‌مشی به نظر برسد.

این صفحه نظارت و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های ممیزی مانند
`Pass`، `Review`، `Warn`، `Malicious`، و سطح ریسک، ببینید
[ممیزی‌های امنیتی](/clawhub/security-audits).

همچنین [امنیت](/clawhub/security) و
[استفاده پذیرفتنی](/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های مربوط به حق نشر یا سایر
حقوق محتوا، از [درخواست‌های حقوق محتوا](/clawhub/content-rights) استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند Skills، plugins، و بسته‌ها را گزارش کنند.

از گزارش‌های ClawHub فقط برای محتوای ناامن بازار استفاده کنید، مانند:

- فهرست‌های مخرب
- فراداده گمراه‌کننده
- اعتبارنامه‌ها یا نیازمندی‌های مجوز اعلام‌نشده
- دستورهای نصب مشکوک
- جعل هویت
- ثبت‌های بدخواهانه یا سوءاستفاده از نشان تجاری
- محتوایی که [استفاده پذیرفتنی](/clawhub/acceptable-usage) را نقض می‌کند

از دکمه **گزارش skill** در صفحه skill، یا فرمان/API گزارش بسته برای بسته‌ها استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌ها در کد منبع خودِ skill یا
plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبعی
که از فهرست پیوند شده گزارش کنید. ClawHub کد skill یا plugin شخص ثالث را نگهداری یا وصله
نمی‌کند.

GitHub Security Advisories برای `openclaw/clawhub` مربوط به آسیب‌پذیری‌ها در
خود ClawHub هستند. نمونه‌ها شامل باگ‌ها در وب‌سایت، API، CLI، رجیستری، احراز هویت،
اسکن، نظارت، یا مرزهای اعتماد دانلود/نصب هستند. از advisories مربوط به ClawHub
برای آسیب‌پذیری‌ها در Skills یا plugins شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و قابل اقدام هستند. سوءاستفاده از گزارش‌دهی می‌تواند خود
به اقدام حساب منجر شود.

## ادعاهای سازمان و فضای نام

اختلاف‌های مالکیت سازمان، برند، محدوده بسته، شناسه مالک، یا فضای نام باید
از فرایند [ادعاهای سازمان و فضای نام](/clawhub/namespace-claims) استفاده کنند، نه
جریان گزارش درون محصول یا فرم اعتراض حساب.

از آن فرایند زمانی استفاده کنید که نیاز دارید کارکنان ClawHub شواهد غیرحساس را بررسی کنند مبنی بر اینکه یک
فضای نام باید رزرو، منتقل، تغییر نام، پنهان، قرنطینه، مستعارسازی،
یا به شکل دیگری بررسی شود. اسرار، اسناد خصوصی، پرونده‌های حقوقی خصوصی،
اسناد هویت شخصی، توکن‌های API، یا توکن‌های چالش DNS را در یک
issue عمومی وارد نکنید.

## نگه‌داشت‌های نظارتی

برخی یافته‌های شدید یا مسائل خط‌مشی می‌توانند یک ناشر یا فهرست را تحت
نگه‌داشت نظارتی قرار دهند. وقتی این اتفاق می‌افتد، محتوای تحت تأثیر ممکن است از کشف
عمومی پنهان شود یا انتشارهای آینده تا زمان بررسی مسئله به‌صورت پنهان شروع شوند.

نگه‌داشت‌های نظارتی برای محافظت از کاربران تا زمانی است که ClawHub موارد پرریسک را
حل کند. همچنین وقتی مثبت کاذب تأیید شود می‌توانند برداشته شوند.

## فهرست‌های پنهان یا مسدود

یک فهرست ممکن است در حالت نگه‌داشت، پنهان، قرنطینه، ابطال‌شده، یا به شکل دیگری در
سطوح نصب عمومی در دسترس نباشد.

اگر یکی از این وضعیت‌ها را می‌بینید، انتشار را نصب نکنید مگر اینکه مالک
مسئله را حل کند یا نظارت آن را بازیابی کند.

مالکان ممکن است همچنان تشخیص‌ها را برای فهرست‌های نگه‌داشته‌شده یا پنهان خود ببینند. این
تشخیص‌ها کمک می‌کنند توضیح دهند چه اتفاقی افتاده و پیش از آنکه
فهرست بتواند به سطوح عمومی بازگردد چه چیزهایی باید تغییر کند.

## ممنوعیت‌ها و وضعیت حساب

حساب‌هایی که خط‌مشی ClawHub را نقض می‌کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفاده شدید می‌تواند
به ممنوعیت حساب، ابطال توکن، محتوای پنهان، یا فهرست‌های حذف‌شده منجر شود.
سیگنال‌های فشار سوءاستفاده ناشر روزانه بررسی می‌شوند. سیگنال‌هایی که به
آستانه ممنوعیت احتمالی ClawHub برسند می‌توانند یک هشدار خودکار را فعال کنند. اگر اسکن
واجد شرایط بعدی پس از مهلت هشدار همچنان ناشر را در
آستانه ممنوعیت احتمالی قرار دهد، ClawHub ممکن است اقدام حساب را به‌صورت خودکار اعمال کند.
سیگنال‌های بازبینی با اطمینان پایین‌تر و محدود به زمان از اجرای خودکار
خارج می‌مانند.

حساب‌های حذف‌شده، ممنوع‌شده، یا غیرفعال نمی‌توانند از توکن‌های API ClawHub استفاده کنند. اگر احراز هویت CLI
پس از اقدام حساب شروع به شکست کرد، برای بررسی وضعیت حساب
وارد UI وب شوید. اگر ورود یا دسترسی معمول CLI به‌دلیل ممنوعیت یا حساب غیرفعال مسدود شده است،
از [فرم اعتراض ClawHub](https://appeals.openclaw.ai/) برای بررسی بازیابی استفاده کنید.

اگر ایمیلی که توسط اسکنر فعال شده نسخه‌ای از skill یا plugin را مخرب نام می‌برد،
نتایج اسکن ذخیره‌شده را برای نسخه ارسالی مسدودشده دانلود کنید:
`clawhub scan download <slug> --version <version>`. برای plugins، اضافه کنید
`--kind plugin`. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید، شماره نسخه را افزایش دهید
و نسخه اصلاح‌شده را بارگذاری کنید.

## راهنمای ناشر

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربران:

- نام‌ها، خلاصه‌ها، برچسب‌ها، و changelogs را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع پیوند دهید
- پیش از انتشار plugins از اجراهای آزمایشی استفاده کنید
- اگر کاربران یا ناظران درباره رفتار انتشار پرسیدند، شفاف پاسخ دهید
