# ایمنی تعدیل و حساب

ClawHub برای انتشار باز است، اما سطوح کشف عمومی و نصب همچنان به محافظت نیاز دارند. گزارش‌ها، توقف‌های تعدیل، فهرست‌های پنهان و اقدامات حساب کمک می‌کنند از کاربران محافظت شود، وقتی یک انتشار یا حساب ناامن، گمراه‌کننده، یا خارج از خط‌مشی به نظر می‌رسد.

این صفحه تعدیل و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های ممیزی مانند `Pass`، `Review`، `Warn`، `Malicious` و سطح ریسک، [ممیزی‌های امنیتی](/clawhub/security-audits) را ببینید.

همچنین [امنیت](/clawhub/security) و [استفاده قابل قبول](/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های مربوط به کپی‌رایت یا سایر حقوق محتوا، از [درخواست‌های حقوق محتوا](/clawhub/content-rights) استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند Skills، Plugin‌ها و بسته‌ها را گزارش کنند.

از گزارش‌های ClawHub فقط برای محتوای ناامن بازارچه استفاده کنید، مانند:

- فهرست‌های مخرب
- فراداده گمراه‌کننده
- اطلاعات محرمانه یا الزامات مجوز اعلام‌نشده
- دستورالعمل‌های نصب مشکوک
- جعل هویت
- ثبت‌های با سوءنیت یا سوءاستفاده از علامت تجاری
- محتوایی که [استفاده قابل قبول](/clawhub/acceptable-usage) را نقض می‌کند

از دکمه **گزارش skill** در صفحه یک skill، یا فرمان/API گزارش بسته برای بسته‌ها استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ یک skill یا plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبعی که از فهرست پیوند شده گزارش کنید. ClawHub کد skill یا plugin شخص ثالث را نگهداری یا وصله نمی‌کند.

Security Advisoryهای GitHub برای `openclaw/clawhub` مربوط به آسیب‌پذیری‌های خود ClawHub هستند. نمونه‌ها شامل باگ‌هایی در وب‌سایت، API، CLI، رجیستری، احراز هویت، اسکن، تعدیل، یا مرزهای اعتماد دانلود/نصب است. از Advisoryهای ClawHub برای آسیب‌پذیری‌های موجود در Skills یا Plugin‌های شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و قابل اقدام هستند. سوءاستفاده از گزارش‌دهی می‌تواند خودش به اقدام علیه حساب منجر شود.

## ادعاهای سازمان و فضای نام

اختلاف‌های مالکیت سازمان، برند، دامنه بسته، شناسه مالک، یا فضای نام باید از فرایند [ادعاهای سازمان و فضای نام](/clawhub/namespace-claims) استفاده کنند، نه جریان گزارش درون‌محصولی یا فرم درخواست تجدیدنظر حساب.

وقتی نیاز دارید کارکنان ClawHub شواهد غیرحساس را بررسی کنند که نشان می‌دهد یک فضای نام باید رزرو، منتقل، تغییر نام داده، پنهان، قرنطینه، دارای alias، یا به‌گونه‌ای دیگر بررسی شود، از آن فرایند استفاده کنید. اسرار، اسناد خصوصی، پرونده‌های حقوقی خصوصی، مدارک هویت شخصی، توکن‌های API، یا توکن‌های چالش DNS را در یک issue عمومی قرار ندهید.

## توقف‌های تعدیل

برخی یافته‌های شدید یا مسائل خط‌مشی می‌توانند یک ناشر یا فهرست را تحت توقف تعدیل قرار دهند. وقتی این اتفاق می‌افتد، محتوای تحت تأثیر ممکن است از کشف عمومی پنهان شود یا انتشارهای آینده ممکن است تا زمان بررسی مسئله به‌صورت پنهان شروع شوند.

توقف‌های تعدیل برای محافظت از کاربران در زمانی هستند که ClawHub پرونده‌های پرریسک را حل می‌کند. وقتی مثبت کاذب تأیید شود، می‌توان آن‌ها را نیز برداشت.

## فهرست‌های پنهان یا مسدود

یک فهرست ممکن است در حالت توقف، پنهان، قرنطینه، لغوشده، یا به‌گونه‌ای دیگر در سطوح نصب عمومی در دسترس نباشد.

اگر یکی از این وضعیت‌ها را می‌بینید، انتشار را نصب نکنید مگر اینکه مالک مسئله را حل کند یا تعدیل آن را بازگرداند.

مالکان ممکن است همچنان عیب‌یابی‌های فهرست‌های خودشان را که در توقف یا پنهان هستند ببینند. این عیب‌یابی‌ها کمک می‌کنند توضیح داده شود چه اتفاقی افتاده و چه چیزی باید تغییر کند تا فهرست بتواند به سطوح عمومی بازگردد.

## ممنوعیت‌ها و وضعیت حساب

حساب‌هایی که خط‌مشی ClawHub را نقض می‌کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفاده شدید می‌تواند به ممنوعیت حساب، لغو توکن، محتوای پنهان، یا حذف فهرست‌ها منجر شود. سیگنال‌های فشار سوءاستفاده ناشر روزانه بررسی می‌شوند. سیگنال‌هایی که به آستانه ممنوعیت احتمالی ClawHub برسند می‌توانند یک هشدار خودکار را فعال کنند. اگر اسکن واجد شرایط بعدی پس از مهلت هشدار همچنان ناشر را در آستانه ممنوعیت احتمالی قرار دهد، ClawHub ممکن است اقدام حساب را به‌صورت خودکار اعمال کند. سیگنال‌های بازبینی زمانی با اطمینان پایین‌تر و محدود از اجرای خودکار کنار گذاشته می‌شوند.

حساب‌های حذف‌شده، ممنوع‌شده، یا غیرفعال نمی‌توانند از توکن‌های API ClawHub استفاده کنند. اگر احراز هویت CLI پس از اقدام حساب شروع به شکست خوردن کرد، برای بررسی وضعیت حساب وارد رابط کاربری وب شوید. اگر ورود یا دسترسی عادی CLI به‌دلیل ممنوعیت یا حساب غیرفعال مسدود شده است، برای بررسی بازیابی از [فرم درخواست تجدیدنظر ClawHub](https://appeals.openclaw.ai/) استفاده کنید.

اگر ایمیلی که توسط اسکنر فعال شده است یک نسخه skill یا plugin را مخرب می‌نامد، نتایج اسکن ذخیره‌شده را برای نسخه ارسالی مسدودشده دانلود کنید: `clawhub scan download <slug> --version <version>`. برای Plugin‌ها، `--kind plugin` را اضافه کنید. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید، شماره نسخه را افزایش دهید و نسخه اصلاح‌شده را بارگذاری کنید.

## راهنمایی ناشران

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربران:

- نام‌ها، خلاصه‌ها، برچسب‌ها و changelogها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع پیوند دهید
- پیش از انتشار Plugin‌ها از dry run استفاده کنید
- اگر کاربران یا ناظران درباره رفتار انتشار پرسیدند، شفاف پاسخ دهید
