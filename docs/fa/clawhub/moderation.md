# مدیریت محتوا و ایمنی حساب

ClawHub برای انتشار باز است، اما سطوح کشف عمومی و نصب همچنان به
محافظت‌های لازم نیاز دارند. گزارش‌ها، توقف‌های مدیریتی، فهرست‌های پنهان، و اقدامات حساب
وقتی یک انتشار یا حساب ناامن، گمراه‌کننده، یا خارج از سیاست به نظر می‌رسد، به محافظت از کاربران
کمک می‌کنند.

این صفحه مدیریت محتوا و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های ممیزی مانند
`Pass`، `Review`، `Warn`، `Malicious`، و سطح ریسک، ببینید
[ممیزی‌های امنیتی](/fa/clawhub/security-audits).

همچنین [امنیت](/fa/clawhub/security) و
[استفاده قابل قبول](/fa/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های مربوط به حق نشر یا سایر
حقوق محتوا، از [درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند Skills، Pluginها، و بسته‌ها را گزارش کنند.

از گزارش‌های ClawHub فقط برای محتوای ناامن بازار استفاده کنید، مانند:

- فهرست‌های مخرب
- فراداده گمراه‌کننده
- الزامات اعلام‌نشده برای اعتبارنامه‌ها یا مجوزها
- دستورالعمل‌های نصب مشکوک
- جعل هویت
- ثبت‌نام‌های بدخواهانه یا سوءاستفاده از نشان تجاری
- محتوایی که [استفاده قابل قبول](/fa/clawhub/acceptable-usage) را نقض می‌کند

از دکمه **گزارش Skill** در صفحه Skill، یا فرمان/API گزارش‌دهی بسته
برای بسته‌ها استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ یک Skill یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبعی که
از فهرست پیوند داده شده گزارش کنید. ClawHub کد Skill یا Plugin شخص ثالث را نگهداری یا وصله
نمی‌کند.

GitHub Security Advisories برای `openclaw/clawhub` مربوط به آسیب‌پذیری‌های خود
ClawHub هستند. نمونه‌ها شامل باگ‌ها در وب‌سایت، API، CLI، رجیستری، احراز هویت،
اسکن، مدیریت محتوا، یا مرزهای اعتماد دانلود/نصب است. از توصیه‌نامه‌های ClawHub
برای آسیب‌پذیری‌های موجود در Skills یا Pluginهای شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و قابل اقدام هستند. سوءاستفاده از گزارش‌دهی می‌تواند خود
به اقدام علیه حساب منجر شود.

## ادعاهای سازمان و فضای نام

اختلاف‌های مالکیت سازمان، برند، دامنه بسته، شناسه مالک، یا فضای نام باید
از فرایند [ادعاهای سازمان و فضای نام](/fa/clawhub/namespace-claims) استفاده کنند، نه از
جریان گزارش درون‌محصولی یا فرم اعتراض حساب.

وقتی لازم است کارکنان ClawHub مدرک غیرحساسی را بررسی کنند که نشان می‌دهد یک
فضای نام باید رزرو، منتقل، تغییرنام داده، پنهان، قرنطینه، مستعارسازی،
یا به شکل دیگری بررسی شود، از این فرایند استفاده کنید. در یک مسئله عمومی
رازها، اسناد خصوصی، فایل‌های حقوقی خصوصی، مدارک هویتی شخصی، توکن‌های API، یا توکن‌های چالش DNS
را قرار ندهید.

## توقف‌های مدیریتی

برخی یافته‌های شدید یا مسائل سیاستی می‌توانند یک ناشر یا فهرست را تحت
توقف مدیریتی قرار دهند. وقتی این اتفاق می‌افتد، محتوای متاثر ممکن است از کشف عمومی
پنهان شود یا انتشارهای آینده تا زمان بررسی مسئله به صورت پنهان شروع شوند.

توقف‌های مدیریتی برای محافظت از کاربران در زمانی هستند که ClawHub پرونده‌های پرریسک
را حل‌وفصل می‌کند. همچنین وقتی مثبت کاذب تایید شود، می‌توانند برداشته شوند.

## فهرست‌های پنهان یا مسدودشده

یک فهرست ممکن است در حالت توقف، پنهان، قرنطینه، لغوشده، یا به شکل دیگری روی
سطوح نصب عمومی در دسترس نباشد.

اگر یکی از این وضعیت‌ها را دیدید، انتشار را نصب نکنید مگر اینکه مالک
مسئله را حل کند یا مدیریت محتوا آن را بازگرداند.

مالکان ممکن است همچنان برای فهرست‌های متوقف‌شده یا پنهان خود عیب‌یابی‌ها را ببینند. این
عیب‌یابی‌ها کمک می‌کنند توضیح دهند چه اتفاقی افتاده و پیش از بازگشت
فهرست به سطوح عمومی چه چیزی باید تغییر کند.

## مسدودسازی‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفاده شدید می‌تواند
به مسدودسازی حساب، ابطال توکن، محتوای پنهان، یا حذف فهرست‌ها منجر شود.
سیگنال‌های فشار سوءاستفاده ناشر روزانه بررسی می‌شوند. سیگنال‌هایی که به
آستانه مسدودسازی احتمالی ClawHub برسند می‌توانند یک هشدار خودکار ایجاد کنند. اگر اسکن
واجد شرایط بعدی پس از مهلت هشدار همچنان ناشر را در
آستانه مسدودسازی احتمالی قرار دهد، ClawHub ممکن است اقدام حساب را به صورت خودکار اعمال کند.
سیگنال‌های بازبینی با اطمینان پایین‌تر و زمان‌مند محدود خارج از اجرای خودکار
می‌مانند.

حساب‌های حذف‌شده، مسدودشده، یا غیرفعال‌شده نمی‌توانند از توکن‌های API ClawHub استفاده کنند. اگر احراز هویت CLI
پس از اقدام حساب شروع به خطا دادن کرد، برای بررسی وضعیت حساب وارد رابط وب
شوید. اگر ورود یا دسترسی عادی CLI به دلیل مسدودسازی یا حساب غیرفعال‌شده مسدود شده است،
برای بررسی بازیابی از [فرم اعتراض ClawHub](https://appeals.openclaw.ai/) استفاده کنید.

اگر یک ایمیل فعال‌شده توسط اسکنر نسخه‌ای از Skill یا Plugin را مخرب نام می‌برد،
نتایج اسکن ذخیره‌شده برای نسخه ارسالی مسدودشده را دانلود کنید:
`clawhub scan download <slug> --version <version>`. برای Pluginها، اضافه کنید
`--kind plugin`. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید، شماره نسخه
را افزایش دهید، و نسخه اصلاح‌شده را بارگذاری کنید.

## راهنمایی ناشر

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربران:

- نام‌ها، خلاصه‌ها، برچسب‌ها، و تغییرنامه‌ها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای موردنیاز را اعلام کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- هر زمان ممکن است به منبع پیوند دهید
- پیش از انتشار Pluginها از اجرای آزمایشی استفاده کنید
- اگر کاربران یا مدیران محتوا درباره رفتار انتشار سوال کردند، شفاف پاسخ دهید
