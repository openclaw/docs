# تعدیل و ایمنی حساب

ClawHub برای انتشار باز است، اما سطوح کشف عمومی و نصب همچنان به
محافظ‌هایی نیاز دارند. گزارش‌ها، توقف‌های تعدیل، فهرست‌های پنهان، و اقدامات
حساب کمک می‌کنند وقتی یک انتشار یا حساب ناامن، گمراه‌کننده، یا خارج از خط‌مشی
به نظر می‌رسد، از کاربران محافظت شود.

این صفحه تعدیل و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های ممیزی مانند
`Pass`، `Review`، `Warn`، `Malicious` و سطح ریسک، [ممیزی‌های امنیتی](/fa/clawhub/security-audits)
را ببینید.

همچنین [امنیت](/fa/clawhub/security) و
[استفاده قابل قبول](/fa/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های
مربوط به حق نشر یا سایر حقوق محتوا، از [درخواست‌های حقوق محتوا](/fa/clawhub/content-rights)
استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند مهارت‌ها، Pluginها و بسته‌ها را گزارش کنند.

از گزارش‌های ClawHub فقط برای محتوای ناامن بازارچه استفاده کنید، مانند:

- فهرست‌های مخرب
- فراداده گمراه‌کننده
- اعتبارنامه‌ها یا الزامات مجوز اعلام‌نشده
- دستورالعمل‌های نصب مشکوک
- جعل هویت
- ثبت‌های با سوءنیت یا سوءاستفاده از علامت تجاری
- محتوایی که [استفاده قابل قبول](/fa/clawhub/acceptable-usage) را نقض می‌کند

از دکمه **گزارش مهارت** در صفحه مهارت، یا فرمان/API گزارش بسته برای بسته‌ها
استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خود یک مهارت یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبعی که از
فهرست لینک شده است گزارش کنید. ClawHub کد مهارت یا Plugin شخص ثالث را نگهداری
یا وصله نمی‌کند.

GitHub Security Advisories برای `openclaw/clawhub` مخصوص آسیب‌پذیری‌های خود
ClawHub هستند. نمونه‌ها شامل باگ‌های وب‌سایت، API، CLI، رجیستری، احراز هویت،
اسکن، تعدیل، یا مرزهای اعتماد دانلود/نصب می‌شوند. از توصیه‌نامه‌های ClawHub
برای آسیب‌پذیری‌های مهارت‌ها یا Pluginهای شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و قابل اقدام هستند. سوءاستفاده از گزارش‌دهی می‌تواند خود
به اقدام حساب منجر شود.

## ادعاهای سازمان و فضای نام

اختلاف‌های مربوط به مالکیت سازمان، برند، دامنه بسته، شناسه مالک، یا فضای نام
باید از فرایند [ادعاهای سازمان و فضای نام](/fa/clawhub/namespace-claims) استفاده
کنند، نه جریان گزارش درون محصول یا فرم اعتراض حساب.

وقتی نیاز دارید کارکنان ClawHub مدرک غیرحساسی را بررسی کنند که نشان می‌دهد یک
فضای نام باید رزرو، منتقل، تغییرنام داده، پنهان، قرنطینه، مستعارسازی، یا به
شکل دیگری بررسی شود، از آن فرایند استفاده کنید. در یک issue عمومی، اسرار،
اسناد خصوصی، فایل‌های حقوقی خصوصی، مدارک هویت شخصی، توکن‌های API، یا توکن‌های
چالش DNS را وارد نکنید.

## توقف‌های تعدیل

برخی یافته‌های شدید یا مسائل خط‌مشی می‌توانند یک ناشر یا فهرست را تحت توقف
تعدیل قرار دهند. وقتی این اتفاق می‌افتد، محتوای تحت تأثیر ممکن است از کشف
عمومی پنهان شود یا انتشارهای آینده تا زمان بررسی مسئله به صورت پنهان آغاز
شوند.

توقف‌های تعدیل برای محافظت از کاربران در زمانی هستند که ClawHub موارد پرریسک
را حل‌وفصل می‌کند. همچنین وقتی مثبت کاذب تأیید شود، می‌توان آن‌ها را برداشت.

## فهرست‌های پنهان یا مسدودشده

یک فهرست ممکن است در حالت توقف، پنهان، قرنطینه، لغوشده، یا به شکلی دیگر روی
سطوح نصب عمومی در دسترس نباشد.

اگر یکی از این وضعیت‌ها را می‌بینید، انتشار را نصب نکنید مگر اینکه مالک مسئله
را حل کند یا تعدیل آن را بازگرداند.

مالکان ممکن است همچنان تشخیص‌های مربوط به فهرست‌های در توقف یا پنهان خود را
ببینند. این تشخیص‌ها کمک می‌کنند توضیح دهند چه اتفاقی افتاده و پیش از بازگشت
فهرست به سطوح عمومی چه چیزی باید تغییر کند.

## ممنوعیت‌ها و وضعیت حساب

حساب‌هایی که خط‌مشی ClawHub را نقض می‌کنند ممکن است دسترسی انتشار را از دست
بدهند. سوءاستفاده شدید می‌تواند به ممنوعیت حساب، لغو توکن، محتوای پنهان، یا
حذف فهرست‌ها منجر شود. سیگنال‌های فشار سوءاستفاده ناشر روزانه بررسی می‌شوند.
سیگنال‌هایی که به آستانه ممنوعیت احتمالی ClawHub برسند می‌توانند یک هشدار
خودکار ایجاد کنند. اگر اسکن واجد شرایط بعدی پس از مهلت هشدار همچنان ناشر را
در آستانه ممنوعیت احتمالی قرار دهد، ClawHub ممکن است اقدام حساب را به‌صورت
خودکار اعمال کند. سیگنال‌های بازبینی زمانی با اطمینان پایین‌تر و محدود از
اجرای خودکار خارج می‌مانند.

حساب‌های حذف‌شده، ممنوع‌شده، یا غیرفعال‌شده نمی‌توانند از توکن‌های API ClawHub
استفاده کنند. اگر احراز هویت CLI پس از اقدام حساب شروع به شکست کرد، وارد UI
وب شوید تا وضعیت حساب را بررسی کنید. اگر ورود یا دسترسی عادی CLI به دلیل
ممنوعیت یا حساب غیرفعال مسدود شده است، برای بررسی بازیابی از
[فرم اعتراض ClawHub](https://appeals.openclaw.ai/) استفاده کنید.

اگر ایمیلی که توسط اسکنر ایجاد شده، نسخه‌ای از یک مهارت یا Plugin را مخرب
معرفی کند، نتایج اسکن ذخیره‌شده برای نسخه ارسالی مسدودشده را دانلود کنید:
`clawhub scan download <slug> --version <version>`. برای Pluginها،
`--kind plugin` را اضافه کنید. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید،
شماره نسخه را افزایش دهید، و نسخه اصلاح‌شده را بارگذاری کنید.

## راهنمایی ناشران

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربران:

- نام‌ها، خلاصه‌ها، برچسب‌ها و تغییرنامه‌ها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع لینک دهید
- پیش از انتشار Pluginها از اجرای آزمایشی استفاده کنید
- اگر کاربران یا ناظران درباره رفتار انتشار سؤال کردند، روشن پاسخ دهید
