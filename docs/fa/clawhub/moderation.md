# تعدیل و ایمنی حساب

ClawHub برای انتشار باز است، اما سطوح کشف عمومی و نصب همچنان
به محافظ‌ها نیاز دارند. گزارش‌ها، نگه‌داشت‌های تعدیل، فهرست‌های پنهان و اقدامات حساب
هنگامی که یک انتشار یا حساب ناامن، گمراه‌کننده یا خارج از خط‌مشی به نظر می‌رسد،
به محافظت از کاربران کمک می‌کنند.

این صفحه تعدیل و وضعیت حساب را پوشش می‌دهد. برای برچسب‌های ممیزی مانند
`Pass`، `Review`، `Warn`، `Malicious` و سطح ریسک، [ممیزی‌های امنیتی](/clawhub/security-audits) را ببینید.

همچنین [امنیت](/clawhub/security) و
[استفاده قابل‌قبول](/clawhub/acceptable-usage) را ببینید. برای نگرانی‌های حق نشر یا سایر حقوق محتوا،
از [درخواست‌های حقوق محتوا](/clawhub/content-rights) استفاده کنید.

## گزارش‌ها

کاربران واردشده می‌توانند Skills، Pluginها و بسته‌ها را گزارش کنند.

از گزارش‌های ClawHub فقط برای محتوای ناامن بازارچه استفاده کنید، مانند:

- فهرست‌های مخرب
- فراداده گمراه‌کننده
- الزامات اعلام‌نشده برای اعتبارنامه‌ها یا مجوزها
- دستورالعمل‌های نصب مشکوک
- جعل هویت
- ثبت‌های بدخواهانه یا سوءاستفاده از نشان تجاری
- محتوایی که [استفاده قابل‌قبول](/clawhub/acceptable-usage) را نقض می‌کند

از دکمه **گزارش skill** در صفحه یک skill، یا فرمان/API گزارش بسته برای بسته‌ها استفاده کنید.

از گزارش‌های ClawHub برای آسیب‌پذیری‌ها در کد منبع خودِ یک skill یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبع
پیوندشده از فهرست گزارش کنید. ClawHub کد skill یا Plugin شخص ثالث را نگه‌داری یا وصله نمی‌کند.

GitHub Security Advisories برای `openclaw/clawhub` مربوط به آسیب‌پذیری‌ها در
خود ClawHub هستند. نمونه‌ها شامل باگ‌ها در وب‌سایت، API، CLI، رجیستری، احراز هویت،
اسکن، تعدیل، یا مرزهای اعتماد دانلود/نصب هستند. از توصیه‌نامه‌های ClawHub
برای آسیب‌پذیری‌ها در Skills یا Pluginهای شخص ثالث استفاده نکنید.

گزارش‌های خوب مشخص و اقدام‌پذیر هستند. سوءاستفاده از گزارش‌دهی می‌تواند خودش به
اقدام حساب منجر شود.

## ادعاهای سازمان و فضای نام

اختلاف‌های مربوط به مالکیت سازمان، برند، دامنه بسته، شناسه مالک، یا فضای نام باید
از فرایند [ادعاهای سازمان و فضای نام](/clawhub/namespace-claims) استفاده کنند، نه
جریان گزارش درون‌محصول یا فرم درخواست تجدیدنظر حساب.

وقتی لازم است کارکنان ClawHub مدرک غیرحساس را بررسی کنند که نشان می‌دهد یک
فضای نام باید رزرو، منتقل، تغییرنام، پنهان، قرنطینه، نام‌مستعاردهی، یا به شکل دیگری بررسی شود،
از آن فرایند استفاده کنید. در یک مسئله عمومی، اسرار، اسناد خصوصی، فایل‌های حقوقی خصوصی،
اسناد هویت شخصی، توکن‌های API، یا توکن‌های چالش DNS را وارد نکنید.

## نگه‌داشت‌های تعدیل

برخی یافته‌های شدید یا مسائل خط‌مشی می‌توانند یک ناشر یا فهرست را تحت
نگه‌داشت تعدیل قرار دهند. وقتی این اتفاق می‌افتد، محتوای تحت تأثیر ممکن است از
کشف عمومی پنهان شود یا انتشارهای آینده تا زمان بررسی مسئله به‌صورت پنهان شروع شوند.

نگه‌داشت‌های تعدیل برای محافظت از کاربران در حالی طراحی شده‌اند که ClawHub پرونده‌های پرریسک را حل می‌کند.
همچنین وقتی یک مثبت کاذب تأیید شود، می‌توانند برداشته شوند.

## فهرست‌های پنهان یا مسدودشده

یک فهرست ممکن است نگه داشته، پنهان، قرنطینه، لغو، یا به شکلی دیگر در
سطوح نصب عمومی دردسترس نباشد.

اگر یکی از این وضعیت‌ها را می‌بینید، انتشار را نصب نکنید مگر اینکه مالک
مسئله را حل کند یا تعدیل آن را بازگرداند.

مالکان ممکن است همچنان تشخیص‌ها را برای فهرست‌های نگه‌داشته‌شده یا پنهان خودشان ببینند. این
تشخیص‌ها کمک می‌کنند توضیح دهند چه اتفاقی افتاده و پیش از بازگشت فهرست
به سطوح عمومی چه چیزی باید تغییر کند.

## ممنوعیت‌ها و وضعیت حساب

حساب‌هایی که خط‌مشی ClawHub را نقض می‌کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفاده شدید می‌تواند
به ممنوعیت حساب، لغو توکن، محتوای پنهان، یا فهرست‌های حذف‌شده منجر شود.
سیگنال‌های فشار سوءاستفاده ناشر روزانه بررسی می‌شوند. سیگنال‌هایی که به
آستانه ممنوعیت احتمالی ClawHub برسند می‌توانند یک هشدار خودکار را فعال کنند. اگر اسکن واجد شرایط بعدی
پس از مهلت هشدار همچنان ناشر را در
آستانه ممنوعیت احتمالی قرار دهد، ClawHub ممکن است اقدام حساب را به‌صورت خودکار اعمال کند.
سیگنال‌های بازبینی با اطمینان پایین‌تر و محدود از نظر زمانی از اجرای خودکار
خارج می‌مانند.

حساب‌های حذف‌شده، ممنوع‌شده، یا غیرفعال نمی‌توانند از توکن‌های API ClawHub استفاده کنند. اگر احراز هویت CLI
پس از اقدام حساب شروع به شکست کرد، برای بررسی وضعیت حساب وارد رابط وب شوید.
اگر ورود یا دسترسی عادی CLI به‌دلیل ممنوعیت یا حساب غیرفعال مسدود شده است،
برای بررسی بازیابی از [فرم درخواست تجدیدنظر ClawHub](https://appeals.openclaw.ai/) استفاده کنید.

اگر ایمیلی که توسط اسکنر فعال شده است نسخه‌ای از skill یا Plugin را مخرب نامید،
نتایج اسکن ذخیره‌شده را برای نسخه ارسالی مسدودشده دانلود کنید:
`clawhub scan download <slug> --version <version>`. برای Pluginها،
`--kind plugin` را اضافه کنید. خروجی اسکن را بررسی کنید، فهرست را اصلاح کنید، شماره نسخه را افزایش دهید،
و نسخه اصلاح‌شده را بارگذاری کنید.

## راهنمایی ناشر

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربر:

- نام‌ها، خلاصه‌ها، برچسب‌ها و گزارش‌های تغییر را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع پیوند دهید
- پیش از انتشار Pluginها از اجرای آزمایشی استفاده کنید
- اگر کاربران یا ناظران درباره رفتار انتشار پرسیدند، روشن پاسخ دهید
