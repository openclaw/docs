# ممیزی‌های امنیتی

ممیزی‌های امنیتی ClawHub به شما کمک می‌کنند تصمیم بگیرید آیا یک skill یا plugin برای نصب به‌اندازه کافی امن است یا نه. آن‌ها نشان می‌دهند یک انتشار چه کاری انجام می‌دهد، چه اختیاراتی درخواست می‌کند، و آیا پیش از دسترسی به فایل‌ها، حساب‌ها، اعتبارنامه‌ها، کد، یا سرویس‌های خارجی، موردی نیازمند توجه بیشتر است یا نه.

ممیزی‌ها نشانه‌های ایمنی قوی هستند، اما تضمین نمی‌کنند که یک انتشار بدون ریسک است. همیشه پیش از اعطای دسترسی حساس، قضاوت خود را به کار بگیرید.

همچنین [امنیت](/fa/clawhub/security)، [استفاده قابل‌قبول](/fa/clawhub/acceptable-usage)،
و [مدیریت محتوا و ایمنی حساب](/fa/clawhub/moderation) را ببینید.

## پیش از نصب چه چیزهایی را بررسی کنید

پیش از نصب، این موارد را مرور کنید:

- وضعیت کلی ممیزی
- سطح ریسک
- هر یافته فهرست‌شده
- اعتبارنامه‌ها، مجوزها، یا متغیرهای محیطی لازم
- مالک، منبع، نسخه، تغییرات، دانلودها، ستاره‌ها، و سایر نشانه‌های اعتماد

فقط محتوایی را نصب کنید که آن را می‌فهمید و به آن اعتماد دارید.

## وضعیت ممیزی

وضعیت ممیزی به شما می‌گوید چگونه به نتیجه ممیزی واکنش نشان دهید:

| وضعیت      | معنی                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | هیچ مشکل قابل‌مشاهده‌ای بالاتر از ریسک پایین پیدا نشد.                                |
| `Review`    | پیش از نصب، یافته‌ها را بخوانید. انتشار ممکن است همچنان مشروع باشد. |
| `Warn`      | احتیاط بیشتری به خرج دهید. ClawHub یک نگرانی پراثر یا نشانه هشدار پیدا کرده است. |
| `Malicious` | نصب نکنید.                                                           |
| `Pending`   | ممیزی‌ها هنوز تمام نشده‌اند.                                             |
| `Error`     | ممیزی تکمیل نشد.                                         |

`Pass` اطمینان‌بخش است، اما جایگزین قضاوت خود شما نمی‌شود. این موضوع برای ابزارهایی که می‌توانند محتوا منتشر کنند، داده‌ها را ویرایش کنند، فرمان اجرا کنند، فایل‌ها را بخوانند، یا به سامانه‌های تولید دسترسی پیدا کنند، بیشترین اهمیت را دارد.

## سطح ریسک

سطح ریسک شعاع اثر را توصیف می‌کند: اگر انتشار را همان‌طور که در نظر گرفته شده استفاده کنید، ظاهرا چه میزان قدرت دارد.

| سطح ریسک | معنی                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | اختیار حساس یا اثرگذاری کاربری کمی پیدا شد.                          |
| `Medium`   | انتشار اختیار معناداری دارد، مانند دسترسی به حساب یا تغییر داده‌ها. |
| `High`     | انتشار اختیار پراثر، یافته‌های شدید، یا نشانه‌های مخرب دارد. |

سطح ریسک و وضعیت ممیزی به پرسش‌های متفاوتی پاسخ می‌دهند:

- سطح ریسک می‌پرسد: «اینجا چقدر قدرت وجود دارد؟»
- وضعیت ممیزی می‌پرسد: «با این نتیجه چه کار باید بکنم؟»

برای مثال، یک skill انتشار ممکن است `Review` با ریسک `Medium` نشان دهد. این به معنای مخرب بودن آن نیست. یعنی skill ظاهرا با هدف خود همسو است، اما می‌تواند با اختیار معنادار حساب عمل کند.

## یافته‌ها

یافته‌ها توضیح می‌دهند چرا یک نتیجه ممیزی نمایش داده شده است. هر یافته معمولا شامل این موارد است:

- معنی آن چیست
- چرا علامت‌گذاری شده است
- محتوای skill یا plugin مرتبط
- یک توصیه

یافته‌ها ممکن است با `Info`، `Low`، `Medium`، `High`، یا `Critical` برچسب‌گذاری شوند. یافته‌های با شدت بالاتر، سهم قوی‌تری در سطح ریسک و وضعیت ممیزی دارند.

یافته‌های با اطمینان پایین از خلاصه عمومی ممیزی پنهان می‌شوند تا صفحه روی شواهد مفید متمرکز بماند.

## ClawHub چه چیزهایی را بررسی می‌کند

ClawHub مصنوعات انتشار ارسال‌شده را ممیزی می‌کند، از جمله:

- دستورالعمل‌های skill یا فراداده plugin
- متغیرهای محیطی و مجوزهای اعلام‌شده
- دستورالعمل‌های نصب و فراداده بسته
- فایل‌های گنجانده‌شده و manifestهای فایل
- فراداده سازگاری و قابلیت

پرسش اصلی، انسجام است: آیا نام، خلاصه، فراداده، اختیار درخواستی، و محتوای واقعی با آنچه کاربران به‌طور معقول انتظار دارند هم‌راستا هستند؟

رفتار قدرتمند به‌خودی‌خود بد نیست. بسیاری از ابزارهای مفید به اعتبارنامه‌ها، فرمان‌های محلی، APIهای ارائه‌دهنده، یا نصب بسته نیاز دارند. ممیزی بررسی می‌کند که آیا آن قدرت مورد انتظار، افشاشده، و متناسب است یا نه.

صفحه‌های مصنوع به ممیزی کامل در این مسیر پیوند می‌دهند:

```text
/<owner>/skills/<slug>/security-audit
```

صفحه ممیزی این موارد را ترکیب می‌کند:

1. SkillSpector
2. VirusTotal
3. تحلیل ریسک

## VirusTotal

ClawHub از VirusTotal به‌عنوان دورسنجی بدافزار در پشته ممیزی استفاده می‌کند. VirusTotal یک استاندارد صنعتی مورد اعتماد برای اعتبار فایل و پویش بدافزار است، و همکاری ما به ClawHub اجازه می‌دهد هوش امنیتی گسترده‌تری را به بازبینی skill و plugin اضافه کند.

VirusTotal به‌ویژه برای مصنوعات مخرب شناخته‌شده، تشخیص‌های موتور، و نشانه‌های اعتبار که بازبینی آگاه از عامل ClawHub را تکمیل می‌کنند مفید است. وقتی شمارش‌های موتورهای فروشندگان در دسترس باشد، ممیزی آن‌ها را با زبان ساده خلاصه می‌کند، مانند:

```text
62/62 vendors flagged this skill as clean.
```

یا:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

وقتی ClawHub دورسنجی شمارش فروشنده‌ای برای خلاصه کردن نداشته باشد، ممیزی می‌گوید:

```text
No VirusTotal findings
```

VirusTotal همچنان دورسنجی است. جایگزین تحلیل ریسک آگاه از مصنوع خود ClawHub نمی‌شود.

## تحلیل ریسک

تحلیل ریسک در داخل توسط ClawScan، سامانه ممیزی امنیتی خود ClawHub، پشتیبانی می‌شود. این سامانه هر انتشار را به‌عنوان یک مصنوع روبه‌روی عامل بررسی می‌کند: دستورالعمل‌ها، فراداده، مجوزهای اعلام‌شده، فایل‌ها، نشانه‌های قابلیت، نشانه‌های پویش ایستا، یافته‌های SkillSpector، دورسنجی VirusTotal، و زمینه ارائه‌شده از سوی ناشر. نشانه‌های پویش ایستا زمینه داخلی برای این بازبینی هستند؛ آن‌ها یک بخش ممیزی عمومی مستقل یا حکم مسدودکننده نصب نیستند.

تحلیل ریسک از
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
به‌عنوان لنزی برای ریسک‌هایی مانند تزریق پرامپت، سوءاستفاده از ابزار، افشای اعتبارنامه، اجرای ناامن، مسموم‌سازی حافظه یا زمینه، و عاملیت بیش‌ازحد استفاده می‌کند.

ClawScan یک قابلیت ترسناک‌به‌نظررسنده را به‌طور خودکار مخرب تلقی نمی‌کند. می‌پرسد آیا قابلیت افشاشده، همسو با هدف، و پشتیبانی‌شده توسط مورد استفاده اعلام‌شده انتشار است یا نه.
