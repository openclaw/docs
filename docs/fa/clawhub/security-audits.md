# ممیزی‌های امنیتی

ممیزی‌های امنیتی ClawHub به شما کمک می‌کنند تصمیم بگیرید آیا یک Skill یا Plugin برای نصب به‌اندازه کافی امن است یا نه. آن‌ها نشان می‌دهند یک انتشار چه کاری انجام می‌دهد، چه اختیاری درخواست می‌کند، و آیا پیش از دسترسی به فایل‌ها، حساب‌ها، اعتبارنامه‌ها، کد، یا سرویس‌های خارجی چیزی نیازمند توجه بیشتر است یا نه.

ممیزی‌ها نشانه‌های ایمنی قوی هستند، اما تضمین نمی‌کنند که یک انتشار بدون ریسک است. همیشه پیش از اعطای دسترسی حساس، قضاوت خودتان را به‌کار بگیرید.

همچنین ببینید [امنیت](/clawhub/security)، [استفاده قابل قبول](/clawhub/acceptable-usage)،
و [مدیریت محتوا و ایمنی حساب](/clawhub/moderation).

## پیش از نصب چه چیزهایی را بررسی کنید

پیش از نصب، این موارد را بازبینی کنید:

- وضعیت کلی ممیزی
- سطح ریسک
- هر یافته فهرست‌شده
- اعتبارنامه‌ها، مجوزها، یا متغیرهای محیطی موردنیاز
- مالک، منبع، نسخه، تغییرات انتشار، دانلودها، ستاره‌ها، و دیگر نشانه‌های اعتماد

فقط محتوایی را نصب کنید که می‌فهمید و به آن اعتماد دارید.

## وضعیت ممیزی

وضعیت ممیزی به شما می‌گوید در برابر نتیجه ممیزی چگونه واکنش نشان دهید:

| وضعیت | معنی |
| ----------- | ------------------------------------------------------------------------- |
| `Pass` | هیچ مسئله آشکاری بالاتر از ریسک پایین پیدا نشد. |
| `Review` | پیش از نصب، یافته‌ها را بخوانید. انتشار ممکن است همچنان معتبر باشد. |
| `Warn` | با احتیاط بیشتر عمل کنید. ClawHub یک نگرانی با اثر زیاد یا نشانه هشدار پیدا کرده است. |
| `Malicious` | نصب نکنید. |
| `Pending` | ممیزی‌ها هنوز تمام نشده‌اند. |
| `Error` | ممیزی قابل تکمیل نبود. |

یک `Pass` اطمینان‌بخش است، اما جایگزین قضاوت خودتان نمی‌شود. این موضوع بیش از همه برای ابزارهایی اهمیت دارد که می‌توانند محتوا منتشر کنند، داده را ویرایش کنند، فرمان اجرا کنند، فایل‌ها را بخوانند، یا به سامانه‌های تولید دسترسی داشته باشند.

## سطح ریسک

سطح ریسک شعاع اثر را توصیف می‌کند: اگر انتشار را طبق هدفش استفاده کنید، به نظر می‌رسد چقدر قدرت دارد.

| سطح ریسک | معنی |
| ---------- | ----------------------------------------------------------------------------- |
| `Low` | اختیار حساس یا اثر کاربری اندکی پیدا شد. |
| `Medium` | انتشار اختیار معناداری دارد، مانند دسترسی به حساب یا تغییر داده. |
| `High` | انتشار اختیار با اثر زیاد، یافته‌های شدید، یا نشانه‌های مخرب دارد. |

سطح ریسک و وضعیت ممیزی به پرسش‌های متفاوتی پاسخ می‌دهند:

- سطح ریسک می‌پرسد: «اینجا چقدر قدرت وجود دارد؟»
- وضعیت ممیزی می‌پرسد: «با این نتیجه چه کاری باید انجام دهم؟»

برای مثال، یک Skill انتشار محتوا ممکن است با ریسک `Medium` وضعیت `Review` نشان دهد. این به معنی مخرب بودن آن نیست. یعنی Skill به نظر با هدفش همسو است، اما می‌تواند با اختیار معنادار حساب عمل کند.

## یافته‌ها

یافته‌ها توضیح می‌دهند چرا یک نتیجه ممیزی نمایش داده شده است. هر یافته معمولا شامل این موارد است:

- معنای آن
- علت علامت‌گذاری آن
- محتوای مرتبط Skill یا Plugin
- یک توصیه

یافته‌ها ممکن است با `Info`، `Low`، `Medium`، `High`، یا `Critical` برچسب‌گذاری شوند. یافته‌های با شدت بالاتر اثر قوی‌تری بر سطح ریسک و وضعیت ممیزی دارند.

یافته‌های با اطمینان پایین از خلاصه عمومی ممیزی پنهان می‌شوند تا صفحه روی شواهد مفید متمرکز بماند.

## ClawHub چه چیزهایی را بررسی می‌کند

ClawHub مصنوعات انتشار ارسال‌شده را ممیزی می‌کند، از جمله:

- دستورالعمل‌های Skill یا فراداده Plugin
- متغیرهای محیطی و مجوزهای اعلام‌شده
- دستورالعمل‌های نصب و فراداده بسته
- فایل‌های گنجانده‌شده و مانیفست‌های فایل
- فراداده سازگاری و قابلیت

پرسش اصلی انسجام است: آیا نام، خلاصه، فراداده، اختیار درخواست‌شده، و محتوای واقعی با چیزی که کاربران به‌طور منطقی انتظار دارند هم‌خوانی دارد؟

رفتار قدرتمند به‌خودی‌خود بد نیست. بسیاری از ابزارهای مفید به اعتبارنامه‌ها، فرمان‌های محلی، APIهای ارائه‌دهنده، یا نصب بسته نیاز دارند. ممیزی بررسی می‌کند که آیا این قدرت مورد انتظار، افشا‌شده، و متناسب است یا نه.

صفحه‌های مصنوعه به ممیزی کامل در این مسیر پیوند می‌دهند:

```text
/<owner>/skills/<slug>/security-audit
```

صفحه ممیزی این موارد را ترکیب می‌کند:

1. SkillSpector
2. VirusTotal
3. تحلیل ریسک

## VirusTotal

ClawHub از VirusTotal به‌عنوان دورسنجی بدافزار در پشته ممیزی استفاده می‌کند. VirusTotal یک استاندارد صنعتی مورد اعتماد برای اعتبار فایل و اسکن بدافزار است، و همکاری ما به ClawHub اجازه می‌دهد هوش امنیتی گسترده‌تری را به بازبینی Skill و Plugin اضافه کند.

VirusTotal به‌ویژه برای مصنوعات مخرب شناخته‌شده، Trefferهای موتور، و نشانه‌های اعتبار که بازبینی آگاه از عامل ClawHub را تکمیل می‌کنند مفید است. وقتی شمارش موتورهای فروشنده در دسترس باشد، ممیزی آن‌ها را با زبان ساده خلاصه می‌کند، مانند:

```text
62/62 vendors flagged this skill as clean.
```

یا:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

وقتی ClawHub دورسنجی شمارش فروشنده‌ای برای خلاصه کردن ندارد، ممیزی می‌گوید:

```text
No VirusTotal findings
```

VirusTotal همچنان دورسنجی است. جایگزین تحلیل ریسک آگاه از مصنوعه خود ClawHub نمی‌شود.

## تحلیل ریسک

تحلیل ریسک به‌صورت داخلی توسط ClawScan، سامانه ممیزی امنیتی خود ClawHub، پشتیبانی می‌شود. این سامانه هر انتشار را به‌عنوان یک مصنوعه روبه‌عامل بازبینی می‌کند: دستورالعمل‌ها، فراداده، مجوزهای اعلام‌شده، فایل‌ها، نشانه‌های قابلیت، نشانه‌های اسکن ایستا، یافته‌های SkillSpector، دورسنجی VirusTotal، و زمینه ارائه‌شده توسط ناشر. نشانه‌های اسکن ایستا زمینه داخلی برای این بازبینی هستند؛ آن‌ها یک بخش ممیزی عمومی مستقل یا رأی مسدودکننده نصب نیستند.

تحلیل ریسک از
[۱۰ مورد برتر OWASP برای Agentic Skills](https://owasp.org/www-project-agentic-skills-top-10/)
به‌عنوان لنزی برای ریسک‌هایی مانند تزریق پرامپت، سوءاستفاده از ابزار، افشای اعتبارنامه، اجرای ناامن، مسموم‌سازی حافظه یا زمینه، و عاملیت بیش از حد استفاده می‌کند.

ClawScan یک قابلیت ترسناک‌به‌نظررس را به‌طور خودکار مخرب تلقی نمی‌کند. می‌پرسد آیا قابلیت افشا شده، با هدف همسو است، و با مورد استفاده اعلام‌شده انتشار پشتیبانی می‌شود یا نه.
