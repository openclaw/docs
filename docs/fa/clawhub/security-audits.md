# ممیزی‌های امنیتی

ممیزی‌های امنیتی ClawHub به شما کمک می‌کنند تصمیم بگیرید آیا یک مهارت یا Plugin به‌اندازه کافی برای نصب امن است یا نه. آن‌ها نشان می‌دهند یک انتشار چه کاری انجام می‌دهد، چه اختیاری درخواست می‌کند، و آیا پیش از دسترسی به فایل‌ها، حساب‌ها، اعتبارنامه‌ها، کد، یا سرویس‌های خارجی، چیزی نیازمند توجه بیشتر است یا نه.

ممیزی‌ها نشانه‌های ایمنی قوی هستند، اما تضمین نمی‌کنند که یک انتشار بدون ریسک است. همیشه پیش از اعطای دسترسی حساس، با قضاوت خود تصمیم بگیرید.

همچنین [امنیت](/clawhub/security)، [استفاده پذیرفتنی](/fa/clawhub/acceptable-usage)،
و [مدیریت محتوا و ایمنی حساب](/clawhub/moderation) را ببینید.

## پیش از نصب چه چیزهایی را بررسی کنید

پیش از نصب، این موارد را بررسی کنید:

- وضعیت کلی ممیزی
- سطح ریسک
- هر یافته فهرست‌شده
- اعتبارنامه‌ها، مجوزها، یا متغیرهای محیطی موردنیاز
- مالک، منبع، نسخه، گزارش تغییرات، دانلودها، ستاره‌ها، و دیگر نشانه‌های اعتماد

فقط محتوایی را نصب کنید که آن را می‌فهمید و به آن اعتماد دارید.

## وضعیت ممیزی

وضعیت ممیزی به شما می‌گوید در برابر نتیجه ممیزی چه واکنشی نشان دهید:

| وضعیت      | معنا                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | هیچ مشکل آشکاری بالاتر از ریسک پایین پیدا نشد.                                |
| `Review`    | پیش از نصب، یافته‌ها را بخوانید. انتشار هنوز ممکن است معتبر باشد. |
| `Warn`      | احتیاط بیشتری به‌خرج دهید. ClawHub یک نگرانی پراثر یا نشانه هشدار پیدا کرده است. |
| `Malicious` | نصب نکنید.                                                           |
| `Pending`   | ممیزی‌ها هنوز تمام نشده‌اند.                                             |
| `Error`     | ممیزی کامل نشد.                                         |

`Pass` اطمینان‌بخش است، اما جایگزین قضاوت خود شما نمی‌شود. این موضوع بیش از همه برای ابزارهایی اهمیت دارد که می‌توانند محتوا منتشر کنند، داده‌ها را ویرایش کنند، فرمان اجرا کنند، فایل‌ها را بخوانند، یا به سامانه‌های تولید دسترسی داشته باشند.

## سطح ریسک

سطح ریسک دامنه اثر را توصیف می‌کند: اگر از انتشار طبق هدفش استفاده کنید، ظاهرا چه میزان قدرت دارد.

| سطح ریسک | معنا                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | اختیار حساس یا اثر کاربری اندکی پیدا شد.                          |
| `Medium`   | انتشار اختیار معناداری دارد، مانند دسترسی حساب یا تغییرات داده. |
| `High`     | انتشار اختیار پراثر، یافته‌های شدید، یا نشانه‌های مخرب دارد. |

سطح ریسک و وضعیت ممیزی به پرسش‌های متفاوتی پاسخ می‌دهند:

- سطح ریسک می‌پرسد: «اینجا چقدر قدرت وجود دارد؟»
- وضعیت ممیزی می‌پرسد: «با این نتیجه چه کار باید بکنم؟»

برای مثال، یک مهارت انتشار ممکن است با ریسک `Medium` وضعیت `Review` نشان دهد. این به معنای مخرب بودن آن نیست. یعنی مهارت ظاهرا با هدف خود همسو است، اما می‌تواند با اختیار معنادار حساب عمل کند.

## یافته‌ها

یافته‌ها توضیح می‌دهند چرا یک نتیجه ممیزی نمایش داده شده است. هر یافته معمولا شامل این موارد است:

- معنای آن
- دلیل علامت‌گذاری آن
- محتوای مرتبط مهارت یا Plugin
- یک توصیه

یافته‌ها ممکن است با `Info`، `Low`، `Medium`، `High`، یا `Critical` برچسب بخورند. یافته‌هایی با شدت بالاتر، سهم بیشتری در سطح ریسک و وضعیت ممیزی دارند.

یافته‌های کم‌اطمینان از خلاصه عمومی ممیزی پنهان می‌شوند تا صفحه بر شواهد مفید متمرکز بماند.

## ClawHub چه چیزهایی را بررسی می‌کند

ClawHub مصنوعات انتشار ارسالی را ممیزی می‌کند، از جمله:

- دستورالعمل‌های مهارت یا فراداده Plugin
- متغیرهای محیطی و مجوزهای اعلام‌شده
- دستورالعمل‌های نصب و فراداده بسته
- فایل‌های همراه و فهرست فایل‌ها
- فراداده سازگاری و قابلیت

پرسش اصلی انسجام است: آیا نام، خلاصه، فراداده، اختیار درخواستی، و محتوای واقعی با چیزی که کاربران منطقا انتظار دارند همخوانی دارد؟

رفتار قدرتمند به‌صورت خودکار بد نیست. بسیاری از ابزارهای مفید به اعتبارنامه‌ها، فرمان‌های محلی، APIهای ارائه‌دهنده، یا نصب بسته نیاز دارند. ممیزی بررسی می‌کند که آیا این قدرت مورد انتظار، افشاشده، و متناسب است یا نه.

صفحه‌های مصنوع به ممیزی کامل در این مسیر پیوند می‌دهند:

```text
/<owner>/skills/<slug>/security-audit
```

صفحه ممیزی این موارد را ترکیب می‌کند:

1. SkillSpector
2. VirusTotal
3. تحلیل ریسک

## VirusTotal

ClawHub از VirusTotal به‌عنوان دورسنجی بدافزار در پشته ممیزی استفاده می‌کند. VirusTotal یک استاندارد صنعتی مورد اعتماد برای اعتبارسنجی فایل و اسکن بدافزار است، و همکاری ما به ClawHub امکان می‌دهد اطلاعات امنیتی گسترده‌تری را به بررسی مهارت و Plugin اضافه کند.

VirusTotal به‌ویژه برای مصنوعات مخرب شناخته‌شده، تشخیص‌های موتور، و نشانه‌های اعتبار که مکمل بررسی آگاه از عامل ClawHub هستند مفید است. وقتی شمارش موتورهای فروشنده در دسترس باشد، ممیزی آن‌ها را با زبان ساده خلاصه می‌کند، مانند:

```text
62/62 vendors flagged this skill as clean.
```

یا:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

وقتی ClawHub دورسنجی شمارش فروشنده‌ای برای خلاصه‌سازی نداشته باشد، ممیزی می‌گوید:

```text
No VirusTotal findings
```

VirusTotal همچنان دورسنجی است. این جایگزین تحلیل ریسک آگاه از مصنوع خود ClawHub نمی‌شود.

## تحلیل ریسک

تحلیل ریسک به‌صورت داخلی توسط ClawScan، سامانه ممیزی امنیتی خود ClawHub، پشتیبانی می‌شود. این سامانه هر انتشار را به‌عنوان یک مصنوع روبه‌روی عامل بررسی می‌کند: دستورالعمل‌ها، فراداده، مجوزهای اعلام‌شده، فایل‌ها، نشانه‌های قابلیت، نشانه‌های اسکن ایستا، یافته‌های SkillSpector، دورسنجی VirusTotal، و زمینه ارائه‌شده توسط ناشر. نشانه‌های اسکن ایستا زمینه داخلی برای این بررسی هستند؛ آن‌ها یک بخش ممیزی عمومی مستقل یا حکم مسدودکننده نصب نیستند.

تحلیل ریسک از
[۱۰ مورد برتر Skills عامل‌محور OWASP](https://owasp.org/www-project-agentic-skills-top-10/)
به‌عنوان لنزی برای ریسک‌هایی مانند تزریق پرامپت، سوءاستفاده از ابزار، افشای اعتبارنامه، اجرای ناامن، مسموم‌سازی حافظه یا زمینه، و عاملیت بیش از حد استفاده می‌کند.

ClawScan یک قابلیت ترسناک‌به‌نظررسیده را به‌صورت خودکار مخرب تلقی نمی‌کند. این سامانه می‌پرسد آیا قابلیت افشاشده، همسو با هدف، و پشتیبانی‌شده توسط مورد استفاده اعلام‌شده انتشار است یا نه.
