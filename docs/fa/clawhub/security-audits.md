# ممیزی‌های امنیتی

ممیزی‌های امنیتی ClawHub به شما کمک می‌کنند تصمیم بگیرید آیا یک مهارت یا Plugin به‌اندازه کافی
برای نصب امن است یا نه. آن‌ها نشان می‌دهند یک انتشار چه کاری انجام می‌دهد، چه اختیاری درخواست می‌کند، و
آیا پیش از دسترسی به فایل‌ها، حساب‌ها، اعتبارنامه‌ها، کد، یا سرویس‌های خارجی
چیزی نیازمند توجه بیشتر است یا نه.

ممیزی‌ها نشانه‌های ایمنی قوی هستند، اما تضمین نمی‌کنند که یک انتشار
کاملاً بی‌خطر باشد. همیشه پیش از اعطای دسترسی حساس، با قضاوت خود تصمیم بگیرید.

همچنین [امنیت](/clawhub/security)، [استفاده قابل قبول](/clawhub/acceptable-usage)،
و [نظارت و ایمنی حساب](/clawhub/moderation) را ببینید.

## پیش از نصب چه چیزهایی را بررسی کنید

پیش از نصب، این موارد را مرور کنید:

- وضعیت کلی ممیزی
- سطح ریسک
- هر یافته فهرست‌شده
- اعتبارنامه‌ها، مجوزها، یا متغیرهای محیطی لازم
- مالک، منبع، نسخه، تغییرات انتشار، دانلودها، ستاره‌ها، و سایر نشانه‌های اعتماد

فقط محتوایی را نصب کنید که آن را می‌فهمید و به آن اعتماد دارید.

## وضعیت ممیزی

وضعیت ممیزی به شما می‌گوید نسبت به نتیجه ممیزی چگونه واکنش نشان دهید:

| وضعیت      | معنی                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | هیچ مشکل آشکاری بالاتر از ریسک پایین پیدا نشد.                                |
| `Review`    | پیش از نصب، یافته‌ها را بخوانید. انتشار ممکن است همچنان مشروع باشد. |
| `Warn`      | با احتیاط بیشتر استفاده کنید. ClawHub یک نگرانی پراثر یا نشانه هشدار پیدا کرده است. |
| `Malicious` | نصب نکنید.                                                           |
| `Pending`   | ممیزی‌ها هنوز تمام نشده‌اند.                                             |
| `Error`     | ممیزی کامل نشد.                                         |

`Pass` اطمینان‌بخش است، اما جایگزین قضاوت خود شما نمی‌شود. این موضوع
بیشتر برای ابزارهایی اهمیت دارد که می‌توانند محتوا منتشر کنند، داده را ویرایش کنند، فرمان اجرا کنند، فایل بخوانند، یا
به سامانه‌های تولید دسترسی داشته باشند.

## سطح ریسک

سطح ریسک شعاع اثر را توصیف می‌کند: اگر از انتشار طبق هدفش استفاده کنید، ظاهراً
چه میزان قدرت دارد.

| سطح ریسک | معنی                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | اختیار حساس یا اثر کاربری اندکی پیدا شد.                          |
| `Medium`   | انتشار اختیار معناداری دارد، مانند دسترسی حساب یا تغییرات داده. |
| `High`     | انتشار اختیار پراثر، یافته‌های شدید، یا نشانه‌های مخرب دارد. |

سطح ریسک و وضعیت ممیزی به پرسش‌های متفاوتی پاسخ می‌دهند:

- سطح ریسک می‌پرسد: «اینجا چقدر قدرت وجود دارد؟»
- وضعیت ممیزی می‌پرسد: «با این نتیجه چه کار باید بکنم؟»

برای مثال، یک مهارت انتشار محتوا ممکن است با ریسک `Medium` وضعیت `Review` را نشان دهد. این
به معنی مخرب بودن آن نیست. یعنی مهارت ظاهراً با هدفش هم‌راستا است، اما می‌تواند
با اختیار معنادار حساب عمل کند.

## یافته‌ها

یافته‌ها توضیح می‌دهند چرا یک نتیجه ممیزی نشان داده شده است. هر یافته معمولاً شامل این موارد است:

- معنی آن چیست
- چرا علامت‌گذاری شده است
- محتوای مرتبط مهارت یا Plugin
- یک توصیه

یافته‌ها ممکن است با `Info`، `Low`، `Medium`، `High`، یا `Critical` برچسب‌گذاری شوند. یافته‌هایی با
شدت بالاتر، اثر قوی‌تری بر سطح ریسک و وضعیت ممیزی دارند.

یافته‌های با اطمینان پایین از جمع‌بندی عمومی ممیزی پنهان می‌شوند تا صفحه
روی شواهد مفید متمرکز بماند.

## ClawHub چه چیزهایی را بررسی می‌کند

ClawHub مصنوعات انتشار ارسال‌شده را ممیزی می‌کند، از جمله:

- دستورالعمل‌های مهارت یا فراداده Plugin
- متغیرهای محیطی و مجوزهای اعلام‌شده
- دستورالعمل‌های نصب و فراداده بسته
- فایل‌های گنجانده‌شده و فهرست‌های فایل
- فراداده سازگاری و قابلیت

پرسش اصلی انسجام است: آیا نام، خلاصه، فراداده، اختیار درخواست‌شده،
و محتوای واقعی با چیزی که کاربران به‌طور معقول انتظار دارند هم‌خوان است؟

رفتار قدرتمند به‌خودی‌خود بد نیست. بسیاری از ابزارهای مفید به اعتبارنامه‌ها،
فرمان‌های محلی، APIهای ارائه‌دهنده، یا نصب بسته نیاز دارند. ممیزی بررسی می‌کند که آیا این
قدرت مورد انتظار، اعلام‌شده، و متناسب است یا نه.

صفحه‌های مصنوع به ممیزی کامل در این مسیر پیوند می‌دهند:

```text
/<owner>/skills/<slug>/security-audit
```

صفحه ممیزی این موارد را ترکیب می‌کند:

1. SkillSpector
2. VirusTotal
3. تحلیل ریسک

## VirusTotal

ClawHub از VirusTotal به‌عنوان دورسنجی بدافزار در پشته ممیزی استفاده می‌کند. VirusTotal یک
استاندارد مورد اعتماد صنعت برای اعتبار فایل و اسکن بدافزار است، و
همکاری ما به ClawHub امکان می‌دهد هوش امنیتی گسترده‌تری به بازبینی مهارت و Plugin
اضافه کند.

VirusTotal به‌ویژه برای مصنوعات مخرب شناخته‌شده، تشخیص‌های موتور، و
نشانه‌های اعتبار مفید است که بازبینی آگاه از عامل ClawHub را تکمیل می‌کنند. وقتی شمارش
موتورهای فروشنده در دسترس باشد، ممیزی آن‌ها را به زبان ساده خلاصه می‌کند، مانند:

```text
62/62 vendors flagged this skill as clean.
```

یا:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

وقتی ClawHub هیچ دورسنجی شمارش فروشنده‌ای برای خلاصه‌سازی نداشته باشد، ممیزی می‌گوید:

```text
No VirusTotal findings
```

VirusTotal همچنان دورسنجی است. جایگزین تحلیل ریسک آگاه از مصنوع خود
ClawHub نمی‌شود.

## تحلیل ریسک

تحلیل ریسک به‌صورت داخلی توسط ClawScan، سامانه ممیزی امنیتی خود ClawHub،
توان‌دهی می‌شود. این سامانه هر انتشار را به‌عنوان یک مصنوع روبه‌روی عامل بازبینی می‌کند: دستورالعمل‌ها،
فراداده، مجوزهای اعلام‌شده، فایل‌ها، نشانه‌های قابلیت، نشانه‌های اسکن ایستا،
یافته‌های SkillSpector، دورسنجی VirusTotal، و زمینه ارائه‌شده توسط ناشر.
نشانه‌های اسکن ایستا زمینه داخلی برای این بازبینی هستند؛ آن‌ها یک
بخش ممیزی عمومی مستقل یا حکم مسدودکننده نصب نیستند.

تحلیل ریسک از
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
به‌عنوان لنزی برای ریسک‌هایی مانند تزریق پرامپت، سوءاستفاده از ابزار، افشای اعتبارنامه،
اجرای ناامن، مسموم‌سازی حافظه یا زمینه، و اختیار بیش‌ازحد استفاده می‌کند.

ClawScan یک قابلیت ترسناک‌نما را به‌طور خودکار مخرب تلقی نمی‌کند.
می‌پرسد آیا قابلیت اعلام شده، با هدف هم‌راستا است، و توسط
مورد استفاده اعلام‌شده انتشار پشتیبانی می‌شود یا نه.
