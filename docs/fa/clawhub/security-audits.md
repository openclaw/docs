# ممیزی‌های امنیتی

ممیزی‌های امنیتی ClawHub به شما کمک می‌کنند تصمیم بگیرید آیا یک skill یا Plugin برای نصب به‌اندازه کافی امن است یا نه. آن‌ها نشان می‌دهند یک انتشار چه کاری انجام می‌دهد، چه اختیاری درخواست می‌کند، و آیا پیش از دسترسی به فایل‌ها، حساب‌ها، اعتبارنامه‌ها، کد، یا سرویس‌های خارجی، چیزی نیازمند توجه بیشتر است یا نه.

ممیزی‌ها نشانه‌های ایمنی قوی هستند، اما تضمین نمی‌کنند که یک انتشار بدون ریسک است. همیشه پیش از اعطای دسترسی حساس، با قضاوت خود تصمیم بگیرید.

همچنین [امنیت](/clawhub/security)، [استفاده پذیرفتنی](/clawhub/acceptable-usage)،
و [نظارت و ایمنی حساب](/clawhub/moderation) را ببینید.

## پیش از نصب چه چیزهایی را بررسی کنید

پیش از نصب، این موارد را بررسی کنید:

- وضعیت کلی ممیزی
- سطح ریسک
- هر یافته فهرست‌شده
- اعتبارنامه‌ها، مجوزها، یا متغیرهای محیطی موردنیاز
- مالک، منبع، نسخه، changelog، دانلودها، ستاره‌ها، و دیگر نشانه‌های اعتماد

فقط محتوایی را نصب کنید که آن را می‌فهمید و به آن اعتماد دارید.

## وضعیت ممیزی

وضعیت ممیزی به شما می‌گوید چگونه به نتیجه ممیزی واکنش نشان دهید:

| وضعیت      | معنا                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | هیچ مشکل قابل مشاهده‌ای بالاتر از ریسک پایین پیدا نشد.                                |
| `Review`    | پیش از نصب یافته‌ها را بخوانید. انتشار ممکن است همچنان معتبر باشد. |
| `Warn`      | با احتیاط بیشتر استفاده کنید. ClawHub یک نگرانی با اثر زیاد یا نشانه هشدار پیدا کرده است. |
| `Malicious` | نصب نکنید.                                                           |
| `Pending`   | ممیزی‌ها هنوز تمام نشده‌اند.                                             |
| `Error`     | ممیزی کامل نشد.                                         |

یک `Pass` اطمینان‌بخش است، اما جایگزین قضاوت خود شما نمی‌شود. این موضوع بیش از همه برای ابزارهایی مهم است که می‌توانند محتوا منتشر کنند، داده‌ها را ویرایش کنند، دستورها را اجرا کنند، فایل‌ها را بخوانند، یا به سامانه‌های production دسترسی داشته باشند.

## سطح ریسک

سطح ریسک شعاع اثر را توصیف می‌کند: اگر انتشار را مطابق هدفش استفاده کنید، ظاهرا چه میزان قدرت دارد.

| سطح ریسک | معنا                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | اختیار حساس یا اثر کاربری اندکی پیدا شد.                          |
| `Medium`   | انتشار اختیار معناداری دارد، مانند دسترسی به حساب یا تغییر داده‌ها. |
| `High`     | انتشار اختیار با اثر زیاد، یافته‌های شدید، یا نشانه‌های مخرب دارد. |

سطح ریسک و وضعیت ممیزی به پرسش‌های متفاوتی پاسخ می‌دهند:

- سطح ریسک می‌پرسد: «اینجا چه مقدار قدرت وجود دارد؟»
- وضعیت ممیزی می‌پرسد: «با این نتیجه چه کار باید بکنم؟»

برای مثال، یک skill انتشار ممکن است `Review` با ریسک `Medium` نشان دهد. این به معنای مخرب بودن آن نیست. یعنی skill ظاهرا با هدف خود همسو است، اما می‌تواند با اختیار حساب معنادار عمل کند.

## یافته‌ها

یافته‌ها توضیح می‌دهند چرا یک نتیجه ممیزی نمایش داده شده است. هر یافته معمولا شامل این موارد است:

- معنای آن چیست
- چرا علامت‌گذاری شده است
- محتوای مرتبط skill یا Plugin
- یک توصیه

یافته‌ها ممکن است با `Info`، `Low`، `Medium`، `High`، یا `Critical` برچسب‌گذاری شوند. یافته‌های با شدت بالاتر، اثر قوی‌تری بر سطح ریسک و وضعیت ممیزی دارند.

یافته‌های با اطمینان پایین از خلاصه عمومی ممیزی پنهان می‌شوند تا صفحه بر شواهد مفید متمرکز بماند.

## ClawHub چه چیزهایی را بررسی می‌کند

ClawHub مصنوعات انتشار ارسال‌شده را ممیزی می‌کند، از جمله:

- دستورالعمل‌های skill یا metadata مربوط به Plugin
- متغیرهای محیطی و مجوزهای اعلام‌شده
- دستورالعمل‌های نصب و metadata بسته
- فایل‌های شامل‌شده و manifestهای فایل
- metadata سازگاری و قابلیت

پرسش اصلی انسجام است: آیا نام، خلاصه، metadata، اختیار درخواست‌شده، و محتوای واقعی با آنچه کاربران منطقا انتظار دارند هم‌خوان است؟

رفتار قدرتمند به‌طور خودکار بد نیست. بسیاری از ابزارهای مفید به اعتبارنامه‌ها، دستورهای محلی، APIهای ارائه‌دهنده، یا نصب بسته نیاز دارند. ممیزی بررسی می‌کند که آیا آن قدرت مورد انتظار، افشاشده، و متناسب است یا نه.

صفحه‌های artifact به ممیزی کامل در این مسیر پیوند می‌دهند:

```text
/<owner>/skills/<slug>/security-audit
```

صفحه ممیزی این موارد را ترکیب می‌کند:

1. SkillSpector
2. VirusTotal
3. تحلیل ریسک

## VirusTotal

ClawHub از VirusTotal به‌عنوان دورسنجی بدافزار در پشته ممیزی استفاده می‌کند. VirusTotal یک استاندارد صنعتی قابل اعتماد برای اعتبار فایل و اسکن بدافزار است، و همکاری ما به ClawHub امکان می‌دهد هوش امنیتی گسترده‌تری را به بررسی skill و Plugin اضافه کند.

VirusTotal به‌ویژه برای مصنوعات مخرب شناخته‌شده، تشخیص‌های موتور، و نشانه‌های اعتبار که بررسی آگاه از عامل ClawHub را تکمیل می‌کنند مفید است. وقتی شمارش موتورهای فروشنده موجود باشد، ممیزی آن‌ها را با زبان ساده خلاصه می‌کند، مانند:

```text
62/62 vendors flagged this skill as clean.
```

یا:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

وقتی ClawHub دورسنجی شمارش فروشنده برای خلاصه کردن نداشته باشد، ممیزی می‌گوید:

```text
No VirusTotal findings
```

VirusTotal همچنان دورسنجی است. جایگزین تحلیل ریسک آگاه از artifact خود ClawHub نمی‌شود.

## تحلیل ریسک

تحلیل ریسک به‌صورت داخلی توسط ClawScan، سامانه ممیزی امنیتی خود ClawHub، پشتیبانی می‌شود. این سامانه هر انتشار را به‌عنوان یک artifact روبه‌روی عامل بررسی می‌کند: دستورالعمل‌ها، metadata، مجوزهای اعلام‌شده، فایل‌ها، نشانه‌های قابلیت، نشانه‌های اسکن ایستا، یافته‌های SkillSpector، دورسنجی VirusTotal، و زمینه ارائه‌شده توسط ناشر. نشانه‌های اسکن ایستا زمینه داخلی برای این بررسی هستند؛ آن‌ها یک بخش ممیزی عمومی مستقل یا حکم مسدودکننده نصب نیستند.

تحلیل ریسک از
[۱۰ مورد برتر OWASP Agentic Skills](https://owasp.org/www-project-agentic-skills-top-10/)
به‌عنوان لنزی برای ریسک‌هایی مانند تزریق prompt، سوءاستفاده از ابزار، افشای اعتبارنامه، اجرای ناامن، مسموم‌سازی حافظه یا زمینه، و عاملیت بیش از حد استفاده می‌کند.

ClawScan یک قابلیت ترسناک‌به‌نظررسنده را به‌طور خودکار مخرب تلقی نمی‌کند. می‌پرسد آیا قابلیت افشاشده، همسو با هدف، و پشتیبانی‌شده توسط مورد استفاده اعلام‌شده انتشار است یا نه.
