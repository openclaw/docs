# ممیزی‌های امنیتی

ممیزی‌های امنیتی ClawHub به شما کمک می‌کنند تصمیم بگیرید آیا یک skill یا plugin برای نصب به‌اندازه کافی ایمن است یا نه. آن‌ها نشان می‌دهند یک انتشار چه کاری انجام می‌دهد، چه اختیاراتی درخواست می‌کند، و آیا پیش از دسترسی به فایل‌ها، حساب‌ها، اعتبارنامه‌ها، کد، یا سرویس‌های خارجی چیزی نیازمند توجه بیشتر هست یا نه.

ممیزی‌ها نشانه‌های ایمنی قوی هستند، اما تضمین نمی‌کنند که یک انتشار بدون ریسک است. همیشه پیش از اعطای دسترسی حساس از قضاوت خود استفاده کنید.

همچنین [امنیت](/clawhub/security)، [استفاده قابل قبول](/fa/clawhub/acceptable-usage)، و [نظارت و ایمنی حساب](/clawhub/moderation) را ببینید.

## پیش از نصب چه چیزهایی را بررسی کنید

پیش از نصب، موارد زیر را بررسی کنید:

- وضعیت کلی ممیزی
- سطح ریسک
- هر یافته فهرست‌شده
- اعتبارنامه‌ها، مجوزها، یا متغیرهای محیطی موردنیاز
- مالک، منبع، نسخه، گزارش تغییرات، دانلودها، ستاره‌ها، و دیگر نشانه‌های اعتماد

فقط محتوایی را نصب کنید که آن را می‌فهمید و به آن اعتماد دارید.

## وضعیت ممیزی

وضعیت ممیزی به شما می‌گوید چگونه به نتیجه ممیزی واکنش نشان دهید:

| وضعیت      | معنا                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | هیچ مشکل قابل مشاهده‌ای بالاتر از ریسک پایین پیدا نشد.                                |
| `Review`    | پیش از نصب یافته‌ها را بخوانید. انتشار هنوز ممکن است مشروع باشد. |
| `Warn`      | احتیاط بیشتری به خرج دهید. ClawHub یک نگرانی با اثرگذاری بالا یا نشانه هشدار پیدا کرده است. |
| `Malicious` | نصب نکنید.                                                           |
| `Pending`   | ممیزی‌ها هنوز تمام نشده‌اند.                                             |
| `Error`     | ممیزی قابل تکمیل نبود.                                         |

یک `Pass` اطمینان‌بخش است، اما جایگزین قضاوت خود شما نمی‌شود. این موضوع بیش از همه برای ابزارهایی اهمیت دارد که می‌توانند محتوا منتشر کنند، داده‌ها را ویرایش کنند، فرمان‌ها را اجرا کنند، فایل‌ها را بخوانند، یا به سامانه‌های تولید دسترسی پیدا کنند.

## سطح ریسک

سطح ریسک شعاع اثر را توصیف می‌کند: اگر انتشار را طبق قصدش استفاده کنید، ظاهرا چه میزان قدرت دارد.

| سطح ریسک | معنا                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | اختیار حساس یا اثرگذاری کاربری کمی پیدا شد.                          |
| `Medium`   | انتشار اختیار قابل توجهی دارد، مانند دسترسی حساب یا تغییرات داده. |
| `High`     | انتشار اختیار با اثرگذاری بالا، یافته‌های شدید، یا نشانه‌های مخرب دارد. |

سطح ریسک و وضعیت ممیزی به پرسش‌های متفاوتی پاسخ می‌دهند:

- سطح ریسک می‌پرسد: «اینجا چه میزان قدرت وجود دارد؟»
- وضعیت ممیزی می‌پرسد: «با این نتیجه چه کار باید بکنم؟»

برای مثال، یک skill انتشار ممکن است `Review` با ریسک `Medium` نشان دهد. این به معنای مخرب بودن آن نیست. یعنی skill با هدف اعلام‌شده هم‌راستا به نظر می‌رسد، اما می‌تواند با اختیار معنادار حساب عمل کند.

## یافته‌ها

یافته‌ها توضیح می‌دهند چرا یک نتیجه ممیزی نمایش داده شده است. هر یافته معمولا شامل موارد زیر است:

- معنای آن چیست
- چرا پرچم‌گذاری شده است
- محتوای skill یا plugin مربوطه
- یک توصیه

یافته‌ها ممکن است با `Info`، `Low`، `Medium`، `High`، یا `Critical` برچسب‌گذاری شوند. یافته‌های با شدت بالاتر سهم بیشتری در سطح ریسک و وضعیت ممیزی دارند.

یافته‌های کم‌اطمینان از جمع‌بندی عمومی ممیزی پنهان می‌شوند تا صفحه بر شواهد مفید متمرکز بماند.

## ClawHub چه چیزهایی را بررسی می‌کند

ClawHub مصنوعات انتشار ارسال‌شده را ممیزی می‌کند، از جمله:

- دستورالعمل‌های skill یا فراداده plugin
- متغیرهای محیطی و مجوزهای اعلام‌شده
- دستورالعمل‌های نصب و فراداده بسته
- فایل‌های گنجانده‌شده و فهرست فایل‌ها
- فراداده سازگاری و قابلیت

پرسش اصلی انسجام است: آیا نام، خلاصه، فراداده، اختیار درخواستی، و محتوای واقعی با چیزی که کاربران منطقا انتظار دارند هم‌راستا هستند؟

رفتار قدرتمند به‌خودی‌خود بد نیست. بسیاری از ابزارهای مفید به اعتبارنامه‌ها، فرمان‌های محلی، APIهای ارائه‌دهنده، یا نصب بسته‌ها نیاز دارند. ممیزی بررسی می‌کند که آیا آن قدرت مورد انتظار، افشاشده، و متناسب است یا نه.

صفحه‌های مصنوعات به ممیزی کامل در این مسیر پیوند می‌دهند:

```text
/<owner>/skills/<slug>/security-audit
```

صفحه ممیزی ترکیبی از موارد زیر است:

1. SkillSpector
2. VirusTotal
3. تحلیل ریسک

## VirusTotal

ClawHub از VirusTotal به‌عنوان تله‌متری بدافزار در پشته ممیزی استفاده می‌کند. VirusTotal یک استاندارد معتبر صنعتی برای اعتبارسنجی فایل و اسکن بدافزار است، و همکاری ما به ClawHub اجازه می‌دهد هوشمندی امنیتی گسترده‌تری را به بازبینی skill و plugin اضافه کند.

VirusTotal به‌ویژه برای مصنوعات مخرب شناخته‌شده، تشخیص‌های موتور، و نشانه‌های اعتبار که بازبینی آگاه از عامل ClawHub را تکمیل می‌کنند مفید است. وقتی شمارش موتورهای فروشنده در دسترس باشد، ممیزی آن‌ها را با زبان ساده خلاصه می‌کند، مانند:

```text
62/62 vendors flagged this skill as clean.
```

یا:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

وقتی ClawHub تله‌متری شمارش فروشنده‌ای برای خلاصه کردن نداشته باشد، ممیزی می‌گوید:

```text
No VirusTotal findings
```

VirusTotal همچنان تله‌متری است. جایگزین تحلیل ریسک آگاه از مصنوع خود ClawHub نمی‌شود.

## تحلیل ریسک

تحلیل ریسک به‌صورت داخلی توسط ClawScan، سامانه ممیزی امنیتی خود ClawHub، پشتیبانی می‌شود. این سامانه هر انتشار را به‌عنوان یک مصنوع روبه‌عامل بازبینی می‌کند: دستورالعمل‌ها، فراداده، مجوزهای اعلام‌شده، فایل‌ها، نشانه‌های قابلیت، نشانه‌های اسکن ایستا، یافته‌های SkillSpector، تله‌متری VirusTotal، و زمینه ارائه‌شده توسط ناشر. نشانه‌های اسکن ایستا زمینه داخلی برای این بازبینی هستند؛ آن‌ها یک بخش مستقل ممیزی عمومی یا حکم مسدودکننده نصب نیستند.

تحلیل ریسک از
[۱۰ مورد برتر Skills عاملی OWASP](https://owasp.org/www-project-agentic-skills-top-10/)
به‌عنوان دریچه‌ای برای ریسک‌هایی مانند تزریق پرامپت، سوءاستفاده از ابزار، افشای اعتبارنامه، اجرای ناامن، مسموم‌سازی حافظه یا زمینه، و عاملیت بیش از حد استفاده می‌کند.

ClawScan یک قابلیت ترسناک‌نما را به‌طور خودکار مخرب تلقی نمی‌کند. می‌پرسد آیا قابلیت افشاشده، هم‌راستا با هدف، و با مورد استفاده اعلام‌شده انتشار پشتیبانی شده است یا نه.
