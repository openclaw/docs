# ممیزی‌های امنیتی

ممیزی‌های امنیتی ClawHub به شما کمک می‌کنند تصمیم بگیرید آیا یک مهارت یا plugin برای نصب به‌اندازه کافی امن است یا نه. آن‌ها نشان می‌دهند یک انتشار چه کاری انجام می‌دهد، چه اختیاری درخواست می‌کند، و آیا پیش از دسترسی به فایل‌ها، حساب‌ها، اعتبارنامه‌ها، کد یا سرویس‌های خارجی چیزی نیازمند توجه بیشتر است یا نه.

ممیزی‌ها نشانه‌های ایمنی قوی هستند، اما تضمین نمی‌کنند که یک انتشار بدون ریسک باشد. همیشه پیش از اعطای دسترسی حساس، از قضاوت خود استفاده کنید.

همچنین [امنیت](/fa/clawhub/security)، [استفاده قابل قبول](/fa/clawhub/acceptable-usage)،
و [نظارت و ایمنی حساب](/fa/clawhub/moderation) را ببینید.

## پیش از نصب چه چیزهایی را بررسی کنید

پیش از نصب، این موارد را مرور کنید:

- وضعیت کلی ممیزی
- سطح ریسک
- هر یافته فهرست‌شده
- اعتبارنامه‌ها، مجوزها یا متغیرهای محیطی موردنیاز
- مالک، منبع، نسخه، تغییرات، دانلودها، ستاره‌ها و دیگر نشانه‌های اعتماد

فقط محتوایی را نصب کنید که آن را می‌فهمید و به آن اعتماد دارید.

## وضعیت ممیزی

وضعیت ممیزی به شما می‌گوید چگونه به نتیجه ممیزی واکنش نشان دهید:

| وضعیت      | معنی                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | هیچ مشکل آشکاری بالاتر از ریسک پایین یافت نشد.                                |
| `Review`    | پیش از نصب، یافته‌ها را بخوانید. انتشار ممکن است همچنان معتبر باشد. |
| `Warn`      | با احتیاط بیشتر استفاده کنید. ClawHub یک نگرانی پُراثر یا نشانه هشدار پیدا کرده است. |
| `Malicious` | نصب نکنید.                                                           |
| `Pending`   | ممیزی‌ها هنوز تمام نشده‌اند.                                             |
| `Error`     | ممیزی کامل نشد.                                         |

یک `Pass` اطمینان‌بخش است، اما جایگزین قضاوت خودتان نمی‌شود. این موضوع بیشترین اهمیت را برای ابزارهایی دارد که می‌توانند محتوا منتشر کنند، داده را ویرایش کنند، فرمان اجرا کنند، فایل‌ها را بخوانند یا به سامانه‌های تولیدی دسترسی داشته باشند.

## سطح ریسک

سطح ریسک شعاع اثر را توصیف می‌کند: اگر انتشار را طبق هدفش استفاده کنید، ظاهراً چقدر قدرت دارد.

| سطح ریسک | معنی                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | اختیار حساس یا اثر کاربری اندکی یافت شد.                          |
| `Medium`   | انتشار اختیار معناداری دارد، مانند دسترسی حساب یا تغییرات داده. |
| `High`     | انتشار اختیار پُراثر، یافته‌های شدید یا نشانه‌های مخرب دارد. |

سطح ریسک و وضعیت ممیزی به پرسش‌های متفاوتی پاسخ می‌دهند:

- سطح ریسک می‌پرسد: «اینجا چقدر قدرت وجود دارد؟»
- وضعیت ممیزی می‌پرسد: «با این نتیجه چه کار باید بکنم؟»

برای مثال، یک مهارت انتشار ممکن است `Review` با ریسک `Medium` نشان دهد. این به معنی مخرب بودن آن نیست. یعنی مهارت ظاهراً با هدفش همسو است، اما می‌تواند با اختیار معنادار حساب عمل کند.

## یافته‌ها

یافته‌ها توضیح می‌دهند چرا یک نتیجه ممیزی نمایش داده شده است. هر یافته معمولاً شامل این موارد است:

- معنی آن چیست
- چرا علامت‌گذاری شده است
- محتوای مرتبط مهارت یا plugin
- یک توصیه

یافته‌ها ممکن است با `Info`، `Low`، `Medium`، `High` یا `Critical` برچسب‌گذاری شوند. یافته‌های با شدت بالاتر سهم قوی‌تری در سطح ریسک و وضعیت ممیزی دارند.

یافته‌های با اطمینان پایین از جمع‌بندی عمومی ممیزی پنهان می‌شوند تا صفحه بر شواهد مفید متمرکز بماند.

## ClawHub چه چیزهایی را بررسی می‌کند

ClawHub مصنوعات انتشار ارسال‌شده را ممیزی می‌کند، از جمله:

- دستورالعمل‌های مهارت یا فراداده plugin
- متغیرهای محیطی و مجوزهای اعلام‌شده
- دستورالعمل‌های نصب و فراداده بسته
- فایل‌های گنجانده‌شده و فهرست‌های فایل
- فراداده سازگاری و قابلیت

پرسش اصلی انسجام است: آیا نام، خلاصه، فراداده، اختیار درخواست‌شده و محتوای واقعی با آنچه کاربران به‌طور معقول انتظار دارند هم‌راستا هستند؟

رفتار قدرتمند به‌صورت خودکار بد نیست. بسیاری از ابزارهای مفید به اعتبارنامه‌ها، فرمان‌های محلی، APIهای ارائه‌دهنده یا نصب بسته نیاز دارند. ممیزی بررسی می‌کند که آیا آن قدرت مورد انتظار، افشاشده و متناسب است یا نه.

صفحه‌های مصنوع به ممیزی کامل در این نشانی پیوند می‌دهند:

```text
/<owner>/skills/<slug>/security-audit
```

صفحه ممیزی این موارد را ترکیب می‌کند:

1. SkillSpector
2. VirusTotal
3. تحلیل ریسک

## VirusTotal

ClawHub از VirusTotal به‌عنوان دورسنجی بدافزار در پشته ممیزی استفاده می‌کند. VirusTotal یک استاندارد صنعتی مورداعتماد برای اعتبار فایل و اسکن بدافزار است، و همکاری ما به ClawHub امکان می‌دهد هوشمندی امنیتی گسترده‌تری به بررسی مهارت و plugin اضافه کند.

VirusTotal به‌ویژه برای مصنوعات مخرب شناخته‌شده، تشخیص‌های موتور و نشانه‌های اعتبار که بررسی آگاه از عامل ClawHub را تکمیل می‌کنند مفید است. وقتی شمارش موتورهای فروشنده در دسترس باشد، ممیزی آن‌ها را با زبان ساده خلاصه می‌کند، مانند:

```text
62/62 vendors flagged this skill as clean.
```

یا:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

وقتی ClawHub دورسنجی شمارش فروشنده‌ای برای خلاصه‌سازی نداشته باشد، ممیزی می‌گوید:

```text
No VirusTotal findings
```

VirusTotal همچنان دورسنجی است. جایگزین تحلیل ریسک آگاه از مصنوع خود ClawHub نمی‌شود.

## تحلیل ریسک

تحلیل ریسک در داخل توسط ClawScan، سامانه ممیزی امنیتی خود ClawHub، پشتیبانی می‌شود. این سامانه هر انتشار را به‌عنوان یک مصنوع روبه‌روی عامل بررسی می‌کند: دستورالعمل‌ها، فراداده، مجوزهای اعلام‌شده، فایل‌ها، نشانه‌های قابلیت، نشانه‌های اسکن ایستا، یافته‌های SkillSpector، دورسنجی VirusTotal، و زمینه ارائه‌شده توسط ناشر. نشانه‌های اسکن ایستا زمینه داخلی برای این بررسی هستند؛ آن‌ها بخش مستقل ممیزی عمومی یا حکم مسدودکننده نصب نیستند.

تحلیل ریسک از
[۱۰ مورد برتر مهارت‌های عاملی OWASP](https://owasp.org/www-project-agentic-skills-top-10/)
به‌عنوان دریچه‌ای برای ریسک‌هایی مانند تزریق پرامپت، سوءاستفاده از ابزار، افشای اعتبارنامه، اجرای ناامن، مسموم‌سازی حافظه یا زمینه، و عاملیت بیش‌ازحد استفاده می‌کند.

ClawScan یک قابلیت ترسناک‌به‌نظررسیده را به‌صورت خودکار مخرب تلقی نمی‌کند. می‌پرسد آیا قابلیت افشا شده، با هدف همسو است، و با مورد استفاده اعلام‌شده انتشار پشتیبانی می‌شود یا نه.
