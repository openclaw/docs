# ممیزی‌های امنیتی

ممیزی‌های امنیتی ClawHub به شما کمک می‌کنند تصمیم بگیرید آیا یک skill یا Plugin برای نصب به‌اندازه کافی امن است یا نه. آن‌ها نشان می‌دهند یک انتشار چه کاری انجام می‌دهد، چه اختیاری درخواست می‌کند، و آیا پیش از دسترسی به فایل‌ها، حساب‌ها، اعتبارنامه‌ها، کد، یا سرویس‌های خارجی چیزی نیازمند توجه بیشتر است یا نه.

ممیزی‌ها نشانه‌های ایمنی قوی هستند، اما تضمین نمی‌کنند که یک انتشار بدون ریسک است. همیشه پیش از اعطای دسترسی حساس از قضاوت خود استفاده کنید.

همچنین [امنیت](/clawhub/security)، [استفاده قابل‌قبول](/clawhub/acceptable-usage)،
و [مدیریت محتوا و ایمنی حساب](/clawhub/moderation) را ببینید.

## پیش از نصب چه چیزهایی را بررسی کنید

پیش از نصب، موارد زیر را مرور کنید:

- وضعیت کلی ممیزی
- سطح ریسک
- هر یافته فهرست‌شده
- اعتبارنامه‌ها، مجوزها، یا متغیرهای محیطی لازم
- مالک، منبع، نسخه، تغییرات، دانلودها، ستاره‌ها، و دیگر نشانه‌های اعتماد

فقط محتوایی را نصب کنید که می‌فهمید و به آن اعتماد دارید.

## وضعیت ممیزی

وضعیت ممیزی به شما می‌گوید چگونه به نتیجه ممیزی واکنش نشان دهید:

| وضعیت      | معنی                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | هیچ مسئله قابل‌مشاهده‌ای بالاتر از ریسک پایین پیدا نشد.                                |
| `Review`    | پیش از نصب، یافته‌ها را بخوانید. انتشار همچنان ممکن است مشروع باشد. |
| `Warn`      | با احتیاط بیشتر عمل کنید. ClawHub یک نگرانی با اثر زیاد یا نشانه هشدار پیدا کرده است. |
| `Malicious` | نصب نکنید.                                                           |
| `Pending`   | ممیزی‌ها هنوز تمام نشده‌اند.                                             |
| `Error`     | ممیزی قابل تکمیل نبود.                                         |

یک `Pass` اطمینان‌بخش است، اما جایگزین قضاوت خودتان نمی‌شود. این موضوع بیش از همه برای ابزارهایی اهمیت دارد که می‌توانند محتوا منتشر کنند، داده‌ها را ویرایش کنند، فرمان اجرا کنند، فایل‌ها را بخوانند، یا به سامانه‌های تولید دسترسی داشته باشند.

## سطح ریسک

سطح ریسک شعاع اثر را توصیف می‌کند: اگر انتشار را مطابق هدفش استفاده کنید، ظاهرا چه مقدار قدرت دارد.

| سطح ریسک | معنی                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | اختیار حساس یا اثرگذاری بر کاربر اندکی پیدا شد.                          |
| `Medium`   | انتشار اختیار معناداری دارد، مانند دسترسی به حساب یا تغییر داده‌ها. |
| `High`     | انتشار اختیار با اثر زیاد، یافته‌های شدید، یا نشانه‌های مخرب دارد. |

سطح ریسک و وضعیت ممیزی به پرسش‌های متفاوتی پاسخ می‌دهند:

- سطح ریسک می‌پرسد: «اینجا چقدر قدرت وجود دارد؟»
- وضعیت ممیزی می‌پرسد: «با این نتیجه چه کار باید بکنم؟»

برای نمونه، یک skill انتشار ممکن است `Review` با ریسک `Medium` نشان دهد. این به معنای مخرب بودن آن نیست. یعنی skill ظاهرا با هدف خود همسو است، اما می‌تواند با اختیار معنادار حساب عمل کند.

## یافته‌ها

یافته‌ها توضیح می‌دهند چرا یک نتیجه ممیزی نمایش داده شده است. هر یافته معمولا شامل این موارد است:

- معنی آن چیست
- چرا علامت‌گذاری شده است
- محتوای مرتبط skill یا Plugin
- یک توصیه

یافته‌ها ممکن است با `Info`، `Low`، `Medium`، `High`، یا `Critical` برچسب‌گذاری شوند. یافته‌های با شدت بالاتر، اثر قوی‌تری بر سطح ریسک و وضعیت ممیزی دارند.

یافته‌های با اطمینان پایین از جمع‌بندی عمومی ممیزی پنهان می‌شوند تا صفحه بر شواهد مفید متمرکز بماند.

## ClawHub چه چیزهایی را بررسی می‌کند

ClawHub آرتیفکت‌های انتشار ارسال‌شده را ممیزی می‌کند، از جمله:

- دستورالعمل‌های skill یا فراداده Plugin
- متغیرهای محیطی و مجوزهای اعلام‌شده
- دستورالعمل‌های نصب و فراداده بسته
- فایل‌های گنجانده‌شده و manifestهای فایل
- فراداده سازگاری و قابلیت

پرسش اصلی انسجام است: آیا نام، خلاصه، فراداده، اختیار درخواست‌شده، و محتوای واقعی با آنچه کاربران به‌طور منطقی انتظار دارند همخوان است؟

رفتار قدرتمند به‌طور خودکار بد نیست. بسیاری از ابزارهای مفید به اعتبارنامه‌ها، فرمان‌های محلی، APIهای ارائه‌دهنده، یا نصب بسته نیاز دارند. ممیزی بررسی می‌کند که آیا این قدرت مورد انتظار، افشاشده، و متناسب است یا نه.

صفحه‌های آرتیفکت به ممیزی کامل در این مسیر پیوند می‌دهند:

```text
/<owner>/skills/<slug>/security-audit
```

صفحه ممیزی این موارد را ترکیب می‌کند:

1. SkillSpector
2. VirusTotal
3. تحلیل ریسک

## VirusTotal

ClawHub از VirusTotal به‌عنوان تله‌متری بدافزار در پشته ممیزی استفاده می‌کند. VirusTotal یک استاندارد معتبر صنعتی برای اعتبارسنجی فایل و اسکن بدافزار است، و همکاری ما به ClawHub امکان می‌دهد هوشمندی امنیتی گسترده‌تری را به بازبینی skill و Plugin اضافه کند.

VirusTotal به‌ویژه برای آرتیفکت‌های مخرب شناخته‌شده، تشخیص‌های موتور، و نشانه‌های اعتبار که بازبینی عامل‌آگاه ClawHub را تکمیل می‌کنند مفید است. وقتی شمارش موتورهای فروشنده در دسترس باشد، ممیزی آن‌ها را با زبان ساده خلاصه می‌کند، مانند:

```text
62/62 vendors flagged this skill as clean.
```

یا:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

وقتی ClawHub تله‌متری شمارش فروشنده‌ای برای خلاصه‌سازی نداشته باشد، ممیزی می‌گوید:

```text
No VirusTotal findings
```

VirusTotal همچنان تله‌متری است. این ابزار جایگزین تحلیل ریسک آرتیفکت‌آگاه خود ClawHub نمی‌شود.

## تحلیل ریسک

تحلیل ریسک به‌صورت داخلی توسط ClawScan، سامانه ممیزی امنیتی خود ClawHub، پشتیبانی می‌شود. این سامانه هر انتشار را به‌عنوان یک آرتیفکت روبه‌روی عامل بازبینی می‌کند: دستورالعمل‌ها، فراداده، مجوزهای اعلام‌شده، فایل‌ها، نشانه‌های قابلیت، نشانه‌های اسکن ایستا، یافته‌های SkillSpector، تله‌متری VirusTotal، و زمینه ارائه‌شده توسط ناشر. نشانه‌های اسکن ایستا زمینه داخلی این بازبینی هستند؛ آن‌ها یک بخش مستقل ممیزی عمومی یا حکم مسدودکننده نصب نیستند.

تحلیل ریسک از
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
به‌عنوان لنزی برای ریسک‌هایی مانند تزریق پرامپت، سوءاستفاده از ابزار، افشای اعتبارنامه، اجرای ناامن، مسموم‌سازی حافظه یا زمینه، و اختیار بیش از حد استفاده می‌کند.

ClawScan یک قابلیت ترسناک‌به‌نظررسنده را به‌طور خودکار مخرب تلقی نمی‌کند. می‌پرسد آیا قابلیت افشاشده، همسو با هدف، و پشتیبانی‌شده توسط مورد استفاده اعلام‌شده انتشار است یا نه.
