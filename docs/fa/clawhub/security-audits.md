# ممیزی‌های امنیتی

ممیزی‌های امنیتی ClawHub به شما کمک می‌کنند تصمیم بگیرید آیا یک مهارت یا Plugin برای نصب به‌اندازه کافی امن است یا نه. آن‌ها نشان می‌دهند یک انتشار چه کاری انجام می‌دهد، چه اختیاری درخواست می‌کند، و آیا پیش از دسترسی به فایل‌ها، حساب‌ها، اعتبارنامه‌ها، کد یا سرویس‌های خارجی چیزی نیازمند توجه بیشتر است یا خیر.

ممیزی‌ها نشانه‌های ایمنی قدرتمندی هستند، اما تضمین نمی‌کنند که یک انتشار بی‌خطر باشد. همیشه پیش از اعطای دسترسی حساس با قضاوت خود تصمیم بگیرید.

همچنین [امنیت](/clawhub/security)، [استفاده قابل قبول](/fa/clawhub/acceptable-usage)،
و [نظارت و ایمنی حساب](/clawhub/moderation) را ببینید.

## پیش از نصب چه چیزهایی را بررسی کنید

پیش از نصب، موارد زیر را بررسی کنید:

- وضعیت کلی ممیزی
- سطح ریسک
- هر یافته فهرست‌شده
- اعتبارنامه‌ها، مجوزها یا متغیرهای محیطی موردنیاز
- مالک، منبع، نسخه، تغییرات انتشار، دانلودها، ستاره‌ها و دیگر نشانه‌های اعتماد

فقط محتوایی را نصب کنید که آن را می‌فهمید و به آن اعتماد دارید.

## وضعیت ممیزی

وضعیت ممیزی به شما می‌گوید چگونه به نتیجه ممیزی واکنش نشان دهید:

| وضعیت      | معنی                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | هیچ مشکل قابل مشاهده‌ای بالاتر از ریسک پایین یافت نشد.                                |
| `Review`    | پیش از نصب یافته‌ها را بخوانید. انتشار ممکن است همچنان معتبر باشد. |
| `Warn`      | با احتیاط بیشتر استفاده کنید. ClawHub یک نگرانی پراثر یا نشانه هشدار پیدا کرده است. |
| `Malicious` | نصب نکنید.                                                           |
| `Pending`   | ممیزی‌ها هنوز تمام نشده‌اند.                                             |
| `Error`     | ممیزی تکمیل نشد.                                         |

یک `Pass` اطمینان‌بخش است، اما جایگزین قضاوت خود شما نمی‌شود. این موضوع بیش از همه برای ابزارهایی اهمیت دارد که می‌توانند محتوا منتشر کنند، داده‌ها را ویرایش کنند، فرمان اجرا کنند، فایل‌ها را بخوانند یا به سیستم‌های تولید دسترسی داشته باشند.

## سطح ریسک

سطح ریسک دامنه اثر را توصیف می‌کند: اگر انتشار را مطابق هدف استفاده کنید، ظاهرا چه مقدار قدرت دارد.

| سطح ریسک | معنی                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | اختیار حساس یا اثر کاربری کمی یافت شد.                          |
| `Medium`   | انتشار اختیار معناداری دارد، مانند دسترسی به حساب یا تغییر داده‌ها. |
| `High`     | انتشار اختیار پراثر، یافته‌های شدید یا نشانه‌های مخرب دارد. |

سطح ریسک و وضعیت ممیزی به پرسش‌های متفاوتی پاسخ می‌دهند:

- سطح ریسک می‌پرسد: «اینجا چه مقدار قدرت وجود دارد؟»
- وضعیت ممیزی می‌پرسد: «با این نتیجه چه کار باید بکنم؟»

برای مثال، یک مهارت انتشار ممکن است `Review` با ریسک `Medium` نشان دهد. این به معنای مخرب بودن آن نیست. یعنی مهارت ظاهرا با هدف خود هم‌راستا است، اما می‌تواند با اختیار معنادار حساب عمل کند.

## یافته‌ها

یافته‌ها توضیح می‌دهند چرا یک نتیجه ممیزی نمایش داده شده است. هر یافته معمولا شامل این موارد است:

- معنی آن چیست
- چرا علامت‌گذاری شده است
- محتوای مهارت یا Plugin مربوطه
- یک توصیه

یافته‌ها ممکن است با `Info`، `Low`، `Medium`، `High` یا `Critical` برچسب‌گذاری شوند. یافته‌های با شدت بالاتر اثر قوی‌تری بر سطح ریسک و وضعیت ممیزی دارند.

یافته‌های با اطمینان پایین از خلاصه عمومی ممیزی پنهان می‌شوند تا صفحه بر شواهد مفید متمرکز بماند.

## ClawHub چه چیزهایی را بررسی می‌کند

ClawHub مصنوعات انتشار ارسال‌شده را ممیزی می‌کند، از جمله:

- دستورالعمل‌های مهارت یا فراداده Plugin
- متغیرهای محیطی و مجوزهای اعلام‌شده
- دستورالعمل‌های نصب و فراداده بسته
- فایل‌های گنجانده‌شده و مانیفست‌های فایل
- فراداده سازگاری و قابلیت

پرسش اصلی انسجام است: آیا نام، خلاصه، فراداده، اختیار درخواست‌شده و محتوای واقعی با چیزی که کاربران به‌طور معقول انتظار دارند هم‌خوانی دارد؟

رفتار قدرتمند به‌طور خودکار بد نیست. بسیاری از ابزارهای مفید به اعتبارنامه‌ها، فرمان‌های محلی، APIهای ارائه‌دهنده یا نصب بسته نیاز دارند. ممیزی بررسی می‌کند آیا آن قدرت مورد انتظار، افشاشده و متناسب است یا نه.

صفحه‌های مصنوعات به ممیزی کامل در این مسیر پیوند می‌دهند:

```text
/<owner>/skills/<slug>/security-audit
```

صفحه ممیزی این موارد را ترکیب می‌کند:

1. SkillSpector
2. VirusTotal
3. تحلیل ریسک

## VirusTotal

ClawHub از VirusTotal به‌عنوان دورسنجی بدافزار در پشته ممیزی استفاده می‌کند. VirusTotal یک استاندارد مورداعتماد صنعت برای اعتبارسنجی فایل و اسکن بدافزار است، و همکاری ما به ClawHub امکان می‌دهد هوشمندی امنیتی گسترده‌تری را به بازبینی مهارت و Plugin اضافه کند.

VirusTotal به‌ویژه برای مصنوعات مخرب شناخته‌شده، نتایج موتورهای اسکن، و نشانه‌های اعتبار که بازبینی آگاه از عامل ClawHub را تکمیل می‌کنند مفید است. وقتی شمارش موتورهای فروشنده در دسترس باشد، ممیزی آن‌ها را به زبان ساده خلاصه می‌کند، مانند:

```text
62/62 vendors flagged this skill as clean.
```

یا:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

وقتی ClawHub دورسنجی شمارش فروشنده‌ای برای خلاصه کردن نداشته باشد، ممیزی می‌گوید:

```text
No VirusTotal findings
```

VirusTotal همچنان دورسنجی است. جایگزین تحلیل ریسک آگاه از مصنوعات خود ClawHub نمی‌شود.

## تحلیل ریسک

تحلیل ریسک در داخل توسط ClawScan، سامانه ممیزی امنیتی خود ClawHub، انجام می‌شود. این سامانه هر انتشار را به‌عنوان یک مصنوع روبه‌روی عامل بازبینی می‌کند: دستورالعمل‌ها، فراداده، مجوزهای اعلام‌شده، فایل‌ها، نشانه‌های قابلیت، نشانه‌های اسکن ایستا، یافته‌های SkillSpector، دورسنجی VirusTotal، و زمینه ارائه‌شده توسط ناشر. نشانه‌های اسکن ایستا زمینه داخلی برای این بازبینی هستند؛ آن‌ها یک بخش مستقل ممیزی عمومی یا حکم مسدودکننده نصب نیستند.

تحلیل ریسک از
[۱۰ مورد برتر مهارت‌های عاملی OWASP](https://owasp.org/www-project-agentic-skills-top-10/)
به‌عنوان لنزی برای ریسک‌هایی مانند تزریق پرامپت، سوءاستفاده از ابزار، افشای اعتبارنامه، اجرای ناامن، مسموم‌سازی حافظه یا زمینه، و عاملیت بیش‌ازحد استفاده می‌کند.

ClawScan یک قابلیت ترسناک‌نما را به‌طور خودکار مخرب تلقی نمی‌کند. می‌پرسد آیا قابلیت افشا شده، با هدف هم‌راستا است، و با مورد استفاده اعلام‌شده انتشار پشتیبانی می‌شود یا نه.
