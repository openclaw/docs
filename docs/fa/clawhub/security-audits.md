# ممیزی‌های امنیتی

ممیزی‌های امنیتی ClawHub به شما کمک می‌کنند تصمیم بگیرید آیا یک مهارت یا Plugin برای
نصب به اندازه کافی امن است یا نه. آن‌ها نشان می‌دهند یک انتشار چه کاری انجام می‌دهد، چه اختیاری درخواست می‌کند، و
آیا پیش از آنکه بتواند به فایل‌ها، حساب‌ها،
اعتبارنامه‌ها، کد، یا سرویس‌های خارجی دسترسی پیدا کند، چیزی نیازمند توجه بیشتر است یا نه.

ممیزی‌ها نشانه‌های ایمنی قوی هستند، اما تضمین نمی‌کنند که یک انتشار
کاملا بدون ریسک است. همیشه پیش از اعطای دسترسی حساس، با قضاوت خود عمل کنید.

همچنین [امنیت](/clawhub/security)، [استفاده قابل‌قبول](/clawhub/acceptable-usage)،
و [نظارت و ایمنی حساب](/clawhub/moderation) را ببینید.

## پیش از نصب چه چیزهایی را بررسی کنید

پیش از نصب، این موارد را مرور کنید:

- وضعیت کلی ممیزی
- سطح ریسک
- هر یافته فهرست‌شده
- اعتبارنامه‌ها، مجوزها، یا متغیرهای محیطی موردنیاز
- مالک، منبع، نسخه، تغییرنامه، دانلودها، ستاره‌ها، و سایر نشانه‌های اعتماد

فقط محتوایی را نصب کنید که می‌فهمید و به آن اعتماد دارید.

## وضعیت ممیزی

وضعیت ممیزی به شما می‌گوید چگونه به نتیجه ممیزی واکنش نشان دهید:

| وضعیت      | معنا                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | هیچ مشکل آشکاری بالاتر از ریسک پایین پیدا نشد.                                |
| `Review`    | پیش از نصب، یافته‌ها را بخوانید. انتشار هنوز ممکن است مشروع باشد. |
| `Warn`      | با احتیاط بیشتری عمل کنید. ClawHub یک نگرانی پراثر یا نشانه هشدار پیدا کرده است. |
| `Malicious` | نصب نکنید.                                                           |
| `Pending`   | ممیزی‌ها هنوز تمام نشده‌اند.                                             |
| `Error`     | ممیزی کامل نشد.                                         |

یک `Pass` اطمینان‌بخش است، اما جایگزین قضاوت خودتان نمی‌شود. این موضوع
برای ابزارهایی که می‌توانند محتوا منتشر کنند، داده‌ها را ویرایش کنند، فرمان اجرا کنند، فایل‌ها را بخوانند، یا
به سامانه‌های تولید دسترسی پیدا کنند، بیشترین اهمیت را دارد.

## سطح ریسک

سطح ریسک شعاع اثر را توصیف می‌کند: اگر انتشار را طبق هدفش استفاده کنید،
به نظر می‌رسد چه مقدار قدرت دارد.

| سطح ریسک | معنا                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | اختیار حساس یا اثر کاربری اندکی پیدا شد.                          |
| `Medium`   | انتشار اختیار معناداری دارد، مانند دسترسی حساب یا تغییرات داده. |
| `High`     | انتشار اختیار پراثر، یافته‌های شدید، یا نشانه‌های مخرب دارد. |

سطح ریسک و وضعیت ممیزی به پرسش‌های متفاوتی پاسخ می‌دهند:

- سطح ریسک می‌پرسد: «اینجا چقدر قدرت وجود دارد؟»
- وضعیت ممیزی می‌پرسد: «با این نتیجه چه کار باید بکنم؟»

برای مثال، یک مهارت انتشار ممکن است `Review` با ریسک `Medium` نشان دهد. این
به معنای مخرب بودن آن نیست. یعنی مهارت ظاهرا با هدف خود همسو است، اما می‌تواند
با اختیار معنادار حساب عمل کند.

## یافته‌ها

یافته‌ها توضیح می‌دهند چرا یک نتیجه ممیزی نمایش داده شده است. هر یافته معمولا شامل این موارد است:

- معنای آن چیست
- چرا علامت‌گذاری شده است
- محتوای مرتبط مهارت یا Plugin
- یک توصیه

یافته‌ها ممکن است با `Info`، `Low`، `Medium`، `High`، یا `Critical` برچسب‌گذاری شوند. یافته‌های با شدت بالاتر
با قدرت بیشتری در سطح ریسک و وضعیت ممیزی اثر می‌گذارند.

یافته‌های با اطمینان پایین از جمع‌بندی عمومی ممیزی پنهان می‌شوند تا صفحه
بر شواهد مفید متمرکز بماند.

## ClawHub چه چیزهایی را بررسی می‌کند

ClawHub آرتیفکت‌های انتشار ارسال‌شده را ممیزی می‌کند، از جمله:

- دستورالعمل‌های مهارت یا فراداده Plugin
- متغیرهای محیطی و مجوزهای اعلام‌شده
- دستورالعمل‌های نصب و فراداده بسته
- فایل‌های همراه و فهرست فایل‌ها
- فراداده سازگاری و قابلیت

پرسش اصلی انسجام است: آیا نام، خلاصه، فراداده، اختیار درخواست‌شده،
و محتوای واقعی با چیزی که کاربران به طور معقول انتظار دارند همخوانی دارد؟

رفتار قدرتمند به‌خودی‌خود بد نیست. بسیاری از ابزارهای مفید به اعتبارنامه‌ها،
فرمان‌های محلی، APIهای ارائه‌دهنده، یا نصب بسته‌ها نیاز دارند. ممیزی بررسی می‌کند آیا آن
قدرت مورد انتظار، افشاشده، و متناسب است یا نه.

صفحه‌های آرتیفکت به ممیزی کامل در این مسیر پیوند می‌دهند:

```text
/<owner>/skills/<slug>/security-audit
```

صفحه ممیزی این موارد را ترکیب می‌کند:

1. SkillSpector
2. VirusTotal
3. تحلیل ریسک

## VirusTotal

ClawHub از VirusTotal به‌عنوان دورسنجی بدافزار در پشته ممیزی استفاده می‌کند. VirusTotal یک
استاندارد صنعتی مورداعتماد برای اعتبار فایل و اسکن بدافزار است، و
همکاری ما به ClawHub امکان می‌دهد هوشمندی امنیتی گسترده‌تری به بررسی مهارت و Plugin
اضافه کند.

VirusTotal به‌ویژه برای آرتیفکت‌های مخرب شناخته‌شده، هشدارهای موتور، و
نشانه‌های اعتبار که بررسی آگاه از عامل ClawHub را تکمیل می‌کنند مفید است. وقتی تعدادهای موتورهای فروشنده
در دسترس باشد، ممیزی آن‌ها را به زبان ساده خلاصه می‌کند، مانند:

```text
62/62 vendors flagged this skill as clean.
```

یا:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

وقتی ClawHub هیچ دورسنجی شمارش فروشنده برای خلاصه‌سازی نداشته باشد، ممیزی می‌گوید:

```text
No VirusTotal findings
```

VirusTotal همچنان دورسنجی است. این ابزار جایگزین تحلیل ریسک آگاه از آرتیفکت خود ClawHub
نمی‌شود.

## تحلیل ریسک

تحلیل ریسک در داخل توسط ClawScan، سامانه ممیزی امنیتی خود ClawHub،
قدرت می‌گیرد. این سامانه هر انتشار را به‌عنوان یک آرتیفکت روبه‌روی عامل بررسی می‌کند: دستورالعمل‌ها،
فراداده، مجوزهای اعلام‌شده، فایل‌ها، نشانه‌های قابلیت، نشانه‌های اسکن ایستا،
یافته‌های SkillSpector، دورسنجی VirusTotal، و زمینه ارائه‌شده توسط ناشر.
نشانه‌های اسکن ایستا زمینه داخلی برای این بررسی هستند؛ آن‌ها یک
بخش ممیزی عمومی مستقل یا رأی مسدودکننده نصب نیستند.

تحلیل ریسک از
[۱۰ مورد برتر OWASP برای Agentic Skills](https://owasp.org/www-project-agentic-skills-top-10/)
به‌عنوان لنزی برای ریسک‌هایی مانند تزریق پرامپت، سوءاستفاده از ابزار، افشای اعتبارنامه،
اجرای ناامن، مسموم‌سازی حافظه یا زمینه، و عاملیت بیش‌ازحد استفاده می‌کند.

ClawScan یک قابلیت ترسناک‌نما را به‌طور خودکار مخرب تلقی نمی‌کند.
می‌پرسد آیا قابلیت افشاشده، همسو با هدف، و پشتیبانی‌شده توسط
مورد استفاده اعلام‌شده انتشار است یا نه.
