# ممیزی‌های امنیتی

ممیزی‌های امنیتی ClawHub به شما کمک می‌کنند تصمیم بگیرید آیا یک مهارت یا Plugin
برای نصب به‌اندازه کافی امن است یا نه. آن‌ها نشان می‌دهند یک انتشار چه کاری انجام می‌دهد، چه اختیاری درخواست می‌کند، و
آیا چیزی پیش از دسترسی به فایل‌ها، حساب‌ها،
اعتبارنامه‌ها، کد، یا سرویس‌های خارجی به توجه بیشتر نیاز دارد یا نه.

ممیزی‌ها نشانه‌های ایمنی قوی هستند، اما تضمین نمی‌کنند که یک انتشار
بی‌خطر است. همیشه پیش از اعطای دسترسی حساس، با قضاوت خود تصمیم بگیرید.

همچنین [امنیت](/fa/clawhub/security)، [استفاده قابل قبول](/fa/clawhub/acceptable-usage)،
و [مدیریت محتوا و ایمنی حساب](/fa/clawhub/moderation) را ببینید.

## پیش از نصب چه چیزهایی را بررسی کنید

پیش از نصب، این موارد را مرور کنید:

- وضعیت کلی ممیزی
- سطح ریسک
- هر یافته فهرست‌شده
- اعتبارنامه‌ها، مجوزها، یا متغیرهای محیطی موردنیاز
- مالک، منبع، نسخه، گزارش تغییرات، دانلودها، ستاره‌ها، و سایر نشانه‌های اعتماد

فقط محتوایی را نصب کنید که آن را می‌فهمید و به آن اعتماد دارید.

## وضعیت ممیزی

وضعیت ممیزی به شما می‌گوید در برابر نتیجه ممیزی چگونه واکنش نشان دهید:

| وضعیت       | معنا                                                                      |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | هیچ مشکل آشکاری بالاتر از ریسک پایین پیدا نشد.                           |
| `Review`    | پیش از نصب، یافته‌ها را بخوانید. انتشار همچنان ممکن است معتبر باشد.      |
| `Warn`      | با احتیاط بیشتر عمل کنید. ClawHub یک نگرانی پراثر یا نشانه هشدار پیدا کرد. |
| `Malicious` | نصب نکنید.                                                               |
| `Pending`   | ممیزی‌ها هنوز تمام نشده‌اند.                                             |
| `Error`     | ممیزی کامل نشد.                                                          |

یک `Pass` اطمینان‌بخش است، اما جایگزین قضاوت خود شما نمی‌شود. این موضوع
بیشتر برای ابزارهایی اهمیت دارد که می‌توانند محتوا منتشر کنند، داده‌ها را ویرایش کنند، دستورها را اجرا کنند، فایل‌ها را بخوانند، یا
به سامانه‌های تولید دسترسی داشته باشند.

## سطح ریسک

سطح ریسک دامنه اثر را توصیف می‌کند: اگر انتشار را مطابق هدفش استفاده کنید،
به نظر می‌رسد چه میزان قدرت داشته باشد.

| سطح ریسک | معنا                                                                            |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | اختیار حساس یا اثرگذاری کاربری کمی پیدا شد.                                   |
| `Medium`   | انتشار اختیار معناداری دارد، مانند دسترسی حساب یا تغییرات داده.              |
| `High`     | انتشار اختیار پراثر، یافته‌های شدید، یا نشانه‌های مخرب دارد.                 |

سطح ریسک و وضعیت ممیزی به پرسش‌های متفاوتی پاسخ می‌دهند:

- سطح ریسک می‌پرسد: «اینجا چقدر قدرت وجود دارد؟»
- وضعیت ممیزی می‌پرسد: «با این نتیجه چه کار باید بکنم؟»

برای مثال، یک مهارت انتشار محتوا ممکن است با ریسک `Medium` وضعیت `Review` نشان دهد. این
به معنای مخرب بودن آن نیست. یعنی به نظر می‌رسد مهارت با هدفش همسو است، اما می‌تواند
با اختیار معنادار حساب عمل کند.

## یافته‌ها

یافته‌ها توضیح می‌دهند چرا یک نتیجه ممیزی نمایش داده شده است. هر یافته معمولاً شامل این موارد است:

- معنای آن
- دلیل علامت‌گذاری آن
- محتوای مرتبط مهارت یا Plugin
- یک توصیه

یافته‌ها ممکن است با برچسب‌های `Info`، `Low`، `Medium`، `High`، یا `Critical` مشخص شوند. یافته‌های با شدت بالاتر
اثر قوی‌تری بر سطح ریسک و وضعیت ممیزی دارند.

یافته‌های کم‌اطمینان از خلاصه عمومی ممیزی پنهان می‌شوند تا صفحه
بر شواهد مفید متمرکز بماند.

## ClawHub چه چیزهایی را بررسی می‌کند

ClawHub آرتیفکت‌های انتشار ارسال‌شده را ممیزی می‌کند، از جمله:

- دستورالعمل‌های مهارت یا فراداده Plugin
- متغیرهای محیطی و مجوزهای اعلام‌شده
- دستورالعمل‌های نصب و فراداده بسته
- فایل‌های شامل‌شده و فهرست‌های فایل
- فراداده سازگاری و قابلیت

پرسش اصلی انسجام است: آیا نام، خلاصه، فراداده، اختیار درخواستی
و محتوای واقعی با چیزی که کاربران به‌طور معقول انتظار دارند هم‌راستا هستند؟

رفتار قدرتمند به‌خودی‌خود بد نیست. بسیاری از ابزارهای مفید به اعتبارنامه‌ها،
دستورهای محلی، APIهای ارائه‌دهنده، یا نصب بسته‌ها نیاز دارند. ممیزی بررسی می‌کند آیا آن
قدرت مورد انتظار، افشاشده، و متناسب است یا نه.

صفحه‌های آرتیفکت به ممیزی کامل در این مسیر پیوند می‌دهند:

```text
/<owner>/skills/<slug>/security-audit
```

صفحه ممیزی این موارد را ترکیب می‌کند:

1. SkillSpector
2. VirusTotal
3. تحلیل ریسک

## VirusTotal

ClawHub از VirusTotal به‌عنوان تله‌متری بدافزار در زنجیره ممیزی استفاده می‌کند. VirusTotal یک
استاندارد صنعتی مورد اعتماد برای اعتبار فایل و اسکن بدافزار است، و
همکاری ما به ClawHub اجازه می‌دهد هوشمندی امنیتی گسترده‌تری را به بررسی مهارت و Plugin
اضافه کند.

VirusTotal به‌ویژه برای آرتیفکت‌های مخرب شناخته‌شده، تشخیص‌های موتور، و
نشانه‌های اعتبار که بررسی آگاه از عامل ClawHub را تکمیل می‌کنند مفید است. وقتی شمارش‌های
موتورهای فروشندگان در دسترس باشند، ممیزی آن‌ها را به زبان ساده خلاصه می‌کند، مانند:

```text
62/62 vendors flagged this skill as clean.
```

یا:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

وقتی ClawHub هیچ تله‌متری شمارش فروشنده برای خلاصه‌سازی نداشته باشد، ممیزی می‌گوید:

```text
No VirusTotal findings
```

VirusTotal همچنان تله‌متری است. جایگزین تحلیل ریسک آگاه از آرتیفکت
خود ClawHub نمی‌شود.

## تحلیل ریسک

تحلیل ریسک به‌صورت داخلی توسط ClawScan، سامانه ممیزی امنیتی خود
ClawHub، پشتیبانی می‌شود. این سامانه هر انتشار را به‌عنوان یک آرتیفکت روبه‌عامل بررسی می‌کند: دستورالعمل‌ها،
فراداده، مجوزهای اعلام‌شده، فایل‌ها، نشانه‌های قابلیت، نشانه‌های اسکن ایستا،
یافته‌های SkillSpector، تله‌متری VirusTotal، و زمینه ارائه‌شده توسط ناشر.
نشانه‌های اسکن ایستا زمینه داخلی برای این بررسی هستند؛ آن‌ها یک
بخش ممیزی عمومی مستقل یا رأی مسدودکننده نصب نیستند.

تحلیل ریسک از
[۱۰ ریسک برتر مهارت‌های عامل‌محور OWASP](https://owasp.org/www-project-agentic-skills-top-10/)
به‌عنوان لنزی برای ریسک‌هایی مانند تزریق پرامپت، سوءاستفاده از ابزار، افشای اعتبارنامه،
اجرای ناامن، مسموم‌سازی حافظه یا زمینه، و اختیار بیش‌ازحد استفاده می‌کند.

ClawScan یک قابلیت ترسناک‌نما را به‌صورت خودکار مخرب تلقی نمی‌کند.
می‌پرسد آیا قابلیت افشا شده، با هدف همسو است، و توسط
مورد استفاده اعلام‌شده انتشار پشتیبانی می‌شود یا نه.
