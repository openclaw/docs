# ممیزی‌های امنیتی

ممیزی‌های امنیتی ClawHub به شما کمک می‌کنند تصمیم بگیرید آیا یک مهارت یا Plugin برای نصب به‌اندازه کافی امن است یا نه. آن‌ها نشان می‌دهند یک انتشار چه کاری انجام می‌دهد، چه اختیاری درخواست می‌کند، و آیا پیش از دسترسی به فایل‌ها، حساب‌ها، اعتبارنامه‌ها، کد، یا سرویس‌های خارجی چیزی نیازمند توجه بیشتر است یا نه.

ممیزی‌ها نشانه‌های ایمنی قدرتمندی هستند، اما تضمین نمی‌کنند که یک انتشار بدون ریسک است. همیشه پیش از اعطای دسترسی حساس، قضاوت خودتان را به کار بگیرید.

همچنین ببینید [امنیت](/clawhub/security)، [استفاده قابل قبول](/clawhub/acceptable-usage)،
و [مدیریت محتوا و ایمنی حساب](/clawhub/moderation).

## پیش از نصب چه چیزهایی را بررسی کنید

پیش از نصب، این موارد را بررسی کنید:

- وضعیت کلی ممیزی
- سطح ریسک
- هر یافته فهرست‌شده
- اعتبارنامه‌ها، مجوزها، یا متغیرهای محیطی لازم
- مالک، منبع، نسخه، تغییرات انتشار، دانلودها، ستاره‌ها، و سایر نشانه‌های اعتماد

فقط محتوایی را نصب کنید که آن را می‌فهمید و به آن اعتماد دارید.

## وضعیت ممیزی

وضعیت ممیزی به شما می‌گوید چگونه به نتیجه ممیزی واکنش نشان دهید:

| وضعیت      | معنی                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | هیچ مشکل قابل مشاهده‌ای بالاتر از ریسک پایین پیدا نشد.                                |
| `Review`    | پیش از نصب، یافته‌ها را بخوانید. انتشار ممکن است همچنان معتبر باشد. |
| `Warn`      | احتیاط بیشتری به خرج دهید. ClawHub یک نگرانی با اثر زیاد یا نشانه هشدار پیدا کرده است. |
| `Malicious` | نصب نکنید.                                                           |
| `Pending`   | ممیزی‌ها هنوز تمام نشده‌اند.                                             |
| `Error`     | ممیزی قابل تکمیل نبود.                                         |

`Pass` اطمینان‌بخش است، اما جایگزین قضاوت خودتان نمی‌شود. این موضوع بیشتر برای ابزارهایی اهمیت دارد که می‌توانند محتوا منتشر کنند، داده‌ها را ویرایش کنند، فرمان اجرا کنند، فایل‌ها را بخوانند، یا به سامانه‌های تولید دسترسی داشته باشند.

## سطح ریسک

سطح ریسک شعاع اثر را توصیف می‌کند: اگر انتشار را مطابق هدفش استفاده کنید، ظاهرا چه میزان قدرت دارد.

| سطح ریسک | معنی                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | اختیار حساس یا اثر کاربری کمی پیدا شد.                          |
| `Medium`   | انتشار اختیار معناداری دارد، مانند دسترسی حساب یا تغییر داده‌ها. |
| `High`     | انتشار اختیار با اثر زیاد، یافته‌های شدید، یا نشانه‌های مخرب دارد. |

سطح ریسک و وضعیت ممیزی به پرسش‌های متفاوتی پاسخ می‌دهند:

- سطح ریسک می‌پرسد: «اینجا چقدر قدرت وجود دارد؟»
- وضعیت ممیزی می‌پرسد: «با این نتیجه چه کنم؟»

برای مثال، یک مهارت انتشار محتوا ممکن است با ریسک `Medium` وضعیت `Review` نشان دهد. این به معنی مخرب بودن آن نیست. یعنی مهارت ظاهرا با هدف خود هم‌راستا است، اما می‌تواند با اختیار معنادار حساب عمل کند.

## یافته‌ها

یافته‌ها توضیح می‌دهند چرا یک نتیجه ممیزی نمایش داده شده است. هر یافته معمولا شامل این موارد است:

- معنی آن چیست
- چرا علامت‌گذاری شده است
- محتوای مرتبط مهارت یا Plugin
- یک توصیه

یافته‌ها ممکن است با `Info`، `Low`، `Medium`، `High`، یا `Critical` برچسب‌گذاری شوند. یافته‌های با شدت بالاتر، اثر قوی‌تری بر سطح ریسک و وضعیت ممیزی دارند.

یافته‌های با اطمینان پایین از خلاصه عمومی ممیزی پنهان می‌شوند تا صفحه بر شواهد مفید متمرکز بماند.

## ClawHub چه چیزهایی را بررسی می‌کند

ClawHub مصنوعات انتشار ارسال‌شده را ممیزی می‌کند، از جمله:

- دستورالعمل‌های مهارت یا فراداده Plugin
- متغیرهای محیطی و مجوزهای اعلام‌شده
- دستورالعمل‌های نصب و فراداده بسته
- فایل‌های گنجانده‌شده و مانیفست‌های فایل
- فراداده سازگاری و قابلیت

پرسش اصلی انسجام است: آیا نام، خلاصه، فراداده، اختیار درخواستی، و محتوای واقعی با چیزی که کاربران منطقا انتظار دارند هم‌خوان است؟

رفتار قدرتمند به‌طور خودکار بد نیست. بسیاری از ابزارهای مفید به اعتبارنامه‌ها، فرمان‌های محلی، APIهای ارائه‌دهنده، یا نصب بسته نیاز دارند. ممیزی بررسی می‌کند آیا این قدرت مورد انتظار، افشاشده، و متناسب است یا نه.

صفحه‌های مصنوع به ممیزی کامل در این مسیر پیوند می‌دهند:

```text
/<owner>/skills/<slug>/security-audit
```

صفحه ممیزی این موارد را ترکیب می‌کند:

1. SkillSpector
2. VirusTotal
3. تحلیل ریسک

## VirusTotal

ClawHub از VirusTotal به‌عنوان دورسنجی بدافزار در مجموعه ممیزی استفاده می‌کند. VirusTotal یک استاندارد صنعتی قابل اعتماد برای اعتبارسنجی فایل و اسکن بدافزار است، و همکاری ما به ClawHub امکان می‌دهد هوش امنیتی گسترده‌تری را به بازبینی مهارت و Plugin اضافه کند.

VirusTotal به‌ویژه برای مصنوعات مخرب شناخته‌شده، تشخیص‌های موتور، و نشانه‌های اعتبار مفید است که بازبینی آگاه از عامل ClawHub را تکمیل می‌کنند. وقتی شمارش موتورهای فروشندگان در دسترس باشد، ممیزی آن‌ها را با زبان ساده خلاصه می‌کند، مانند:

```text
62/62 vendors flagged this skill as clean.
```

یا:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

وقتی ClawHub دورسنجی شمارش فروشنده‌ای برای خلاصه‌سازی نداشته باشد، ممیزی می‌گوید:

```text
No VirusTotal findings
```

VirusTotal همچنان دورسنجی است. این جایگزین تحلیل ریسک آگاه از مصنوع خود ClawHub نمی‌شود.

## تحلیل ریسک

تحلیل ریسک به‌صورت داخلی توسط ClawScan، سامانه ممیزی امنیتی خود ClawHub، پشتیبانی می‌شود. این سامانه هر انتشار را به‌عنوان مصنوعی روبه‌روی عامل بازبینی می‌کند: دستورالعمل‌ها، فراداده، مجوزهای اعلام‌شده، فایل‌ها، نشانه‌های قابلیت، نشانه‌های اسکن ایستا، یافته‌های SkillSpector، دورسنجی VirusTotal، و زمینه ارائه‌شده توسط ناشر. نشانه‌های اسکن ایستا زمینه داخلی برای این بازبینی هستند؛ آن‌ها یک بخش مستقل ممیزی عمومی یا حکم مسدودکننده نصب نیستند.

تحلیل ریسک از
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
به‌عنوان لنزی برای ریسک‌هایی مانند تزریق پرامپت، سوءاستفاده از ابزار، افشای اعتبارنامه، اجرای ناامن، مسموم‌سازی حافظه یا زمینه، و عاملیت بیش‌ازحد استفاده می‌کند.

ClawScan یک قابلیت ترسناک‌به‌نظررسنده را به‌طور خودکار مخرب تلقی نمی‌کند. این سامانه می‌پرسد آیا قابلیت افشا شده، با هدف هم‌راستا است، و توسط مورد استفاده اعلام‌شده انتشار پشتیبانی می‌شود یا نه.
