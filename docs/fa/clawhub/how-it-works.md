# نحوه کار ClawHub

ClawHub لایهٔ رجیستری برای Skills و Pluginهای OpenClaw است. این لایه به کاربران
جایی برای کشف بسته‌ها، به ناشران جایی برای انتشار نسخه‌ها، و به OpenClaw
فرادادهٔ کافی برای نصب و به‌روزرسانی ایمن آن بسته‌ها می‌دهد.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است با:

- مالک و اسلاگ یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها، و انتساب منبع
- اطلاعات تغییرات و برچسب مانند `latest`
- سیگنال‌های دانلود، نصب، ستاره، و نظر
- وضعیت اسکن امنیتی و تعدیل محتوا

صفحهٔ فهرست، مکان مرجع برای کاربران است تا پیش از نصب بررسی کنند یک مهارت یا
Plugin ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

یک مهارت یک بستهٔ متنی نسخه‌بندی‌شده با محوریت `SKILL.md` است. می‌تواند شامل
فایل‌های پشتیبان، مثال‌ها، قالب‌ها، و اسکریپت‌ها باشد.

ClawHub فرانت‌متر `SKILL.md` را می‌خواند تا نام مهارت، توضیحات، الزامات،
متغیرهای محیطی، و فراداده را بفهمد. فرادادهٔ دقیق مهم است، چون به کاربران کمک
می‌کند تصمیم بگیرند آیا مهارت را نصب کنند یا نه، و به اسکن‌های خودکار کمک
می‌کند ناسازگاری‌های میان رفتار اعلام‌شده و مشاهده‌شده را تشخیص دهند.

ببینید [قالب مهارت](/fa/clawhub/skill-format).

## Pluginها

Pluginها افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته،
اطلاعات سازگاری، پیوندهای منبع، آرتیفکت‌ها، و رکوردهای نسخه را ذخیره می‌کند.

وقتی OpenClaw یک Plugin را از ClawHub نصب می‌کند، پیش از نصب فرادادهٔ سازگاری
اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API، حداقل
نسخهٔ Gateway، اهداف میزبان، الزامات محیطی، و چکیده‌های آرتیفکت باشند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده
کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار یک رکورد نسخهٔ جدید و تغییرناپذیر ایجاد می‌کند. ناشران از CLI
`clawhub` برای گردش‌کارهای رجیستری احرازهویت‌شده استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

از اجرای آزمایشی استفاده کنید تا پیش از بارگذاری، محتوای نهایی حل‌شده را پیش‌نمایش
کنید. سپس صفحه‌های عمومی فرادادهٔ منتشرشده، فایل‌ها، انتساب منبع، و وضعیت اسکن
را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

دستورهای نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بعداً بتوانند همان
بستهٔ رجیستری را پیدا کنند. CLI ClawHub همچنین از گردش‌کارهای نصب و
به‌روزرسانی مستقیم مهارت برای کاربرانی پشتیبانی می‌کند که پوشه‌های مهارت
مدیریت‌شده توسط رجیستری را بیرون از یک فضای کاری کامل OpenClaw می‌خواهند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما انتشارها همچنان مشمول دروازه‌های بارگذاری،
بررسی‌های خودکار، گزارش‌های کاربران، و اقدام ناظر هستند.

صفحه‌های عمومی، در صورت موجود بودن، خلاصه‌های اسکن را نشان می‌دهند. محتوایی که
نگه داشته شده، پنهان شده، یا مسدود شده است ممکن است از جست‌وجوی عمومی و
جریان‌های نصب ناپدید شود، در حالی که برای عیب‌یابی یا درخواست تجدیدنظر همچنان
برای مالک قابل مشاهده می‌ماند.

ببینید [امنیت + تعدیل محتوا](/fa/clawhub/security) و
[استفادهٔ قابل‌قبول](/fa/clawhub/acceptable-usage).

## دسترسی API

ClawHub APIهای خواندن عمومی را برای کشف، جست‌وجو، جزئیات بسته، و دانلودها
ارائه می‌کند. کاتالوگ‌های شخص ثالث می‌توانند از این APIها استفاده کنند، به شرطی
که به فهرست مرجع ClawHub پیوند بدهند، محدودیت‌های نرخ را رعایت کنند، و از القای
تأیید ضمنی پرهیز کنند.

ببینید [API عمومی](/fa/clawhub/api) و [HTTP API](/fa/clawhub/http-api).
