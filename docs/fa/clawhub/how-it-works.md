# ClawHub چگونه کار می‌کند

ClawHub لایهٔ رجیستری برای Skills و Pluginهای OpenClaw است. این لایه به کاربران
جایی برای کشف بسته‌ها، به ناشران جایی برای انتشار نسخه‌ها، و به
OpenClaw فرادادهٔ کافی برای نصب و به‌روزرسانی ایمن آن بسته‌ها می‌دهد.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است با:

- مالک و اسلاگ یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها، و انتساب منبع
- اطلاعات تغییرات و برچسب مانند `latest`
- سیگنال‌های دانلود، نصب، و ستاره
- وضعیت اسکن امنیتی و نظارت

صفحهٔ فهرست، مکان مرجع برای کاربران است تا پیش از نصب بررسی کنند یک مهارت یا
Plugin ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

یک مهارت یک بستهٔ متنی نسخه‌دار است که حول `SKILL.md` شکل می‌گیرد. می‌تواند شامل
فایل‌های پشتیبان، مثال‌ها، قالب‌ها، و اسکریپت‌ها باشد.

ClawHub فرانت‌متر `SKILL.md` را می‌خواند تا نام مهارت، توضیحات،
نیازمندی‌ها، متغیرهای محیطی، و فراداده را بفهمد. فرادادهٔ دقیق مهم است، چون
به کاربران کمک می‌کند تصمیم بگیرند آیا مهارت را نصب کنند یا نه و به اسکن‌های
خودکار کمک می‌کند ناسازگاری‌ها میان رفتار اعلام‌شده و مشاهده‌شده را تشخیص دهند.

[قالب مهارت](/fa/clawhub/skill-format) را ببینید.

## Pluginها

Pluginها افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته،
اطلاعات سازگاری، پیوندهای منبع، آرتیفکت‌ها، و رکوردهای نسخه را ذخیره می‌کند.

وقتی OpenClaw یک Plugin را از ClawHub نصب می‌کند، پیش از نصب فرادادهٔ سازگاری
اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API،
حداقل نسخهٔ Gateway، هدف‌های میزبان، نیازمندی‌های محیطی، و چکیده‌های آرتیفکت
باشند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار یک رکورد نسخهٔ تغییرناپذیر جدید ایجاد می‌کند. ناشران برای گردش‌کارهای
رجیستری احراز هویت‌شده از CLI `clawhub` استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

از اجرای آزمایشی برای پیش‌نمایش محمولهٔ حل‌شده پیش از بارگذاری استفاده کنید. سپس
صفحه‌های عمومی فرادادهٔ منتشرشده، فایل‌ها، انتساب منبع، و وضعیت اسکن را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

فرمان‌های نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بعداً بتوانند همان
بستهٔ رجیستری را حل کنند. CLI ClawHub همچنین برای کاربرانی که پوشه‌های مهارت
مدیریت‌شده توسط رجیستری را بیرون از یک فضای کاری کامل OpenClaw می‌خواهند،
گردش‌کارهای نصب و به‌روزرسانی مستقیم مهارت را پشتیبانی می‌کند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما انتشارها همچنان تابع دروازه‌های بارگذاری،
بررسی‌های خودکار، گزارش‌های کاربران، و اقدام ناظر هستند.

صفحه‌های عمومی در صورت موجود بودن، خلاصه‌های اسکن را نشان می‌دهند. محتوایی که
نگه داشته، پنهان، یا مسدود شده است ممکن است از جریان‌های جست‌وجو و نصب عمومی
ناپدید شود، در حالی که برای عیب‌یابی همچنان برای مالک قابل مشاهده بماند.

[امنیت](/clawhub/security)، [ممیزی‌های امنیتی](/clawhub/security-audits)،
[نظارت و ایمنی حساب](/fa/clawhub/moderation)، و
[استفادهٔ قابل قبول](/clawhub/acceptable-usage) را ببینید.

## دسترسی API

ClawHub APIهای خواندن عمومی را برای کشف، جست‌وجو، جزئیات بسته، و
دانلودها ارائه می‌کند. کاتالوگ‌های شخص ثالث می‌توانند وقتی به فهرست مرجع
ClawHub پیوند می‌دهند، محدودیت‌های نرخ را رعایت می‌کنند، و از القای تأیید رسمی
خودداری می‌کنند، از این APIها استفاده کنند.

[API عمومی](/fa/clawhub/api) و [API HTTP](/clawhub/http-api) را ببینید.
