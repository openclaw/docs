# ClawHub چگونه کار می‌کند

ClawHub لایهٔ رجیستری برای Skills و پلاگین‌های OpenClaw است. به کاربران
جایی برای کشف بسته‌ها می‌دهد، به ناشران جایی برای انتشار نسخه‌ها می‌دهد، و
فرادادهٔ کافی در اختیار OpenClaw می‌گذارد تا آن بسته‌ها را با ایمنی نصب و به‌روزرسانی کند.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است با:

- مالک و نامک یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها، و انتساب منبع
- اطلاعات changelog و برچسب‌هایی مانند `latest`
- سیگنال‌های دانلود، نصب، ستاره، و نظر
- وضعیت اسکن امنیتی و مدیریت محتوا

صفحهٔ فهرست، جایگاه مرجع برای کاربران است تا پیش از نصب، بررسی کنند یک Skill یا
پلاگین ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

یک Skill بستهٔ متنی نسخه‌گذاری‌شده‌ای است که حول `SKILL.md` شکل می‌گیرد. می‌تواند شامل
فایل‌های پشتیبان، مثال‌ها، قالب‌ها، و اسکریپت‌ها باشد.

ClawHub فرانت‌متر `SKILL.md` را می‌خواند تا نام Skill،
توضیح، نیازمندی‌ها، متغیرهای محیطی، و فراداده را بفهمد. فرادادهٔ دقیق
مهم است، چون به کاربران کمک می‌کند تصمیم بگیرند آیا Skill را نصب کنند یا نه و
به اسکن‌های خودکار کمک می‌کند ناسازگاری‌های میان رفتار اعلام‌شده و مشاهده‌شده را تشخیص دهند.

[قالب Skill](/fa/clawhub/skill-format) را ببینید.

## پلاگین‌ها

پلاگین‌ها افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته،
اطلاعات سازگاری، پیوندهای منبع، مصنوعات، و رکوردهای نسخه را نگه می‌دارد.

وقتی OpenClaw یک پلاگین را از ClawHub نصب می‌کند، پیش از نصب، فرادادهٔ
سازگاری اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API،
حداقل نسخهٔ Gateway، اهداف میزبان، نیازمندی‌های محیطی، و چکیده‌های مصنوع باشند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار، یک رکورد نسخهٔ تغییرناپذیر جدید ایجاد می‌کند. ناشران از CLI `clawhub`
برای گردش‌کارهای احراز هویت‌شدهٔ رجیستری استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

از اجرای آزمایشی برای پیش‌نمایش بارگذاری حل‌شده پیش از آپلود استفاده کنید. سپس صفحه‌های عمومی
فرادادهٔ منتشرشده، فایل‌ها، انتساب منبع، و وضعیت اسکن را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

دستورهای نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بتوانند بعداً همان
بستهٔ رجیستری را حل کنند. CLI مربوط به ClawHub همچنین از گردش‌کارهای نصب و
به‌روزرسانی مستقیم Skill برای کاربرانی پشتیبانی می‌کند که پوشه‌های Skill مدیریت‌شده با رجیستری را خارج از
یک فضای کاری کامل OpenClaw می‌خواهند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما انتشارها همچنان مشمول دروازه‌های آپلود،
بررسی‌های خودکار، گزارش‌های کاربران، و اقدام ناظران هستند.

صفحه‌های عمومی، وقتی موجود باشد، خلاصه‌های اسکن را نشان می‌دهند. محتوایی که نگه داشته، پنهان،
یا مسدود شده است ممکن است از جست‌وجوی عمومی و جریان‌های نصب ناپدید شود، در حالی که همچنان
برای مالک جهت عیب‌یابی یا اعتراض قابل مشاهده بماند.

[امنیت + مدیریت محتوا](/fa/clawhub/security) و
[استفادهٔ قابل قبول](/fa/clawhub/acceptable-usage) را ببینید.

## دسترسی API

ClawHub APIهای خواندن عمومی را برای کشف، جست‌وجو، جزئیات بسته، و
دانلودها ارائه می‌کند. کاتالوگ‌های شخص ثالث می‌توانند از این APIها استفاده کنند، به شرطی که به
فهرست مرجع ClawHub پیوند بدهند، محدودیت‌های نرخ را رعایت کنند، و از القای تأیید شدن پرهیز کنند.

[API عمومی](/fa/clawhub/api) و [HTTP API](/fa/clawhub/http-api) را ببینید.
