# ClawHub چگونه کار می‌کند

ClawHub لایهٔ رجیستری برای Skills و Pluginهای OpenClaw است. به کاربران
جایی برای کشف بسته‌ها می‌دهد، به ناشران جایی برای انتشار نسخه‌ها می‌دهد، و
فرادادهٔ کافی در اختیار OpenClaw می‌گذارد تا آن بسته‌ها را با اطمینان نصب و به‌روزرسانی کند.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است با:

- مالک و اسلاگ یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها، و انتساب منبع
- اطلاعات تغییرات و برچسب مانند `latest`
- سیگنال‌های دانلود، نصب، ستاره، و نظر
- وضعیت پویش امنیتی و تعدیل محتوا

صفحهٔ فهرست، جای canonical برای کاربران است تا پیش از نصب بررسی کنند یک skill یا
plugin ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

یک skill یک بستهٔ متنی نسخه‌گذاری‌شده است که حول `SKILL.md` شکل گرفته است. می‌تواند شامل
فایل‌های پشتیبان، نمونه‌ها، قالب‌ها، و اسکریپت‌ها باشد.

ClawHub فرانت‌متر `SKILL.md` را می‌خواند تا نام skill،
توضیح، نیازمندی‌ها، متغیرهای محیطی، و فراداده را بفهمد. فرادادهٔ دقیق
مهم است، چون به کاربران کمک می‌کند تصمیم بگیرند آیا skill را نصب کنند یا نه و
به پویش‌های خودکار کمک می‌کند ناسازگاری‌ها را میان رفتار اعلام‌شده و مشاهده‌شده تشخیص دهند.

ببینید: [قالب skill](/fa/clawhub/skill-format).

## Pluginها

Pluginها افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته،
اطلاعات سازگاری، پیوندهای منبع، artifactها، و رکوردهای نسخه را ذخیره می‌کند.

وقتی OpenClaw یک plugin را از ClawHub نصب می‌کند، پیش از نصب،
فرادادهٔ سازگاری اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API،
حداقل نسخهٔ gateway، هدف‌های میزبان، نیازمندی‌های محیطی، و digestهای artifact باشند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار، یک رکورد نسخهٔ immutable جدید می‌سازد. ناشران از CLI `clawhub`
برای گردش‌کارهای رجیستری احراز هویت‌شده استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

برای پیش‌نمایش payload حل‌شده پیش از بارگذاری، از اجرای خشک استفاده کنید. سپس صفحه‌های عمومی
فرادادهٔ منتشرشده، فایل‌ها، انتساب منبع، و وضعیت پویش را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

دستورهای نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بتوانند همان
بستهٔ رجیستری را بعداً resolve کنند. CLI ClawHub همچنین برای کاربرانی که می‌خواهند پوشه‌های skill مدیریت‌شده توسط رجیستری را خارج از یک فضای کاری کامل OpenClaw داشته باشند، گردش‌کارهای نصب و
به‌روزرسانی مستقیم skill را پشتیبانی می‌کند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما نسخه‌ها همچنان مشمول gateهای بارگذاری،
بررسی‌های خودکار، گزارش‌های کاربران، و اقدام تعدیل‌گران هستند.

صفحه‌های عمومی، وقتی موجود باشد، خلاصه‌های پویش را نشان می‌دهند. محتوایی که نگه داشته، پنهان،
یا مسدود شده است ممکن است از جریان‌های جست‌وجو و نصب عمومی ناپدید شود، در حالی که برای عیب‌یابی
برای مالک همچنان قابل مشاهده بماند.

ببینید: [امنیت + تعدیل محتوا](/fa/clawhub/security) و
[استفادهٔ قابل قبول](/fa/clawhub/acceptable-usage).

## دسترسی API

ClawHub APIهای خواندن عمومی را برای کشف، جست‌وجو، جزئیات بسته، و
دانلودها ارائه می‌کند. کاتالوگ‌های شخص ثالث می‌توانند از این APIها استفاده کنند، به شرطی که به
فهرست canonical ClawHub پیوند دهند، محدودیت‌های نرخ را رعایت کنند، و از القای تأیید ضمنی خودداری کنند.

ببینید: [API عمومی](/fa/clawhub/api) و [API HTTP](/fa/clawhub/http-api).
