# نحوهٔ کار ClawHub

ClawHub لایهٔ رجیستری برای Skills و pluginهای OpenClaw است. به کاربران
جایی برای کشف بسته‌ها می‌دهد، به ناشران جایی برای انتشار نسخه‌ها می‌دهد، و
به OpenClaw فرادادهٔ کافی می‌دهد تا آن بسته‌ها را با ایمنی نصب و به‌روزرسانی کند.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است با:

- مالک و slug یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها، و انتساب منبع
- اطلاعات changelog و tag مانند `latest`
- سیگنال‌های دانلود، نصب، ستاره، و نظر
- وضعیت اسکن امنیتی و تعدیل

صفحهٔ فهرست، جایگاه مرجع برای کاربران است تا پیش از نصب بررسی کنند یک skill یا
plugin ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

یک skill یک بستهٔ متنی نسخه‌دار است که حول `SKILL.md` شکل گرفته است. می‌تواند شامل
فایل‌های پشتیبان، مثال‌ها، قالب‌ها، و اسکریپت‌ها باشد.

ClawHub فرانت‌متر `SKILL.md` را می‌خواند تا نام skill،
توضیح، نیازمندی‌ها، متغیرهای محیطی، و فراداده را بفهمد. فرادادهٔ دقیق
مهم است، چون به کاربران کمک می‌کند تصمیم بگیرند skill را نصب کنند یا نه و
به اسکن‌های خودکار کمک می‌کند ناسازگاری‌ها میان رفتار اعلام‌شده و مشاهده‌شده را تشخیص دهند.

نگاه کنید به [قالب skill](/fa/clawhub/skill-format).

## Pluginها

Pluginها افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته،
اطلاعات سازگاری، پیوندهای منبع، artifactها، و رکوردهای نسخه را ذخیره می‌کند.

وقتی OpenClaw یک plugin را از ClawHub نصب می‌کند، پیش از نصب فرادادهٔ سازگاری
اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API،
حداقل نسخهٔ gateway، هدف‌های میزبان، نیازمندی‌های محیطی، و digestهای artifact باشند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار، یک رکورد نسخهٔ تغییرناپذیر جدید ایجاد می‌کند. ناشران از CLI `clawhub`
برای گردش‌کارهای رجیستری احرازهویت‌شده استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

از dry runها برای پیش‌نمایش payload حل‌شده پیش از بارگذاری استفاده کنید. سپس
صفحه‌های عمومی فرادادهٔ منتشرشده، فایل‌ها، انتساب منبع، و وضعیت اسکن را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

دستورهای نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بعداً بتوانند همان
بستهٔ رجیستری را حل کنند. CLI ClawHub همچنین از گردش‌کارهای نصب و
به‌روزرسانی مستقیم skill برای کاربرانی پشتیبانی می‌کند که پوشه‌های skill مدیریت‌شده توسط رجیستری را بیرون از یک
فضای کاری کامل OpenClaw می‌خواهند.

## وضعیت امنیت

ClawHub برای انتشار باز است، اما نسخه‌ها همچنان مشمول دروازه‌های بارگذاری،
بررسی‌های خودکار، گزارش‌های کاربران، و اقدام ناظر هستند.

صفحه‌های عمومی، وقتی موجود باشد، خلاصه‌های اسکن را نشان می‌دهند. محتوایی که نگه داشته، پنهان،
یا مسدود شده باشد ممکن است از جست‌وجوی عمومی و جریان‌های نصب ناپدید شود، در حالی که برای
مالک جهت عیب‌یابی قابل مشاهده باقی می‌ماند.

نگاه کنید به [امنیت + تعدیل](/fa/clawhub/security) و
[استفادهٔ قابل قبول](/fa/clawhub/acceptable-usage).

## دسترسی API

ClawHub APIهای خواندن عمومی را برای کشف، جست‌وجو، جزئیات بسته، و
دانلودها ارائه می‌کند. کاتالوگ‌های شخص ثالث می‌توانند از این APIها استفاده کنند، به شرطی که به
فهرست مرجع ClawHub پیوند بدهند، محدودیت‌های نرخ را رعایت کنند، و از القای تأیید خودداری کنند.

نگاه کنید به [API عمومی](/fa/clawhub/api) و [HTTP API](/fa/clawhub/http-api).
