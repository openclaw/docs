# ClawHub چگونه کار می‌کند

ClawHub لایهٔ رجیستری برای Skills و Pluginهای OpenClaw است. به کاربران
جایی برای کشف بسته‌ها می‌دهد، به ناشران جایی برای انتشار نسخه‌ها می‌دهد، و
به OpenClaw فرادادهٔ کافی می‌دهد تا آن بسته‌ها را با ایمنی نصب و به‌روزرسانی کند.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است با:

- مالک و اسلاگ یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها، و انتساب منبع
- اطلاعات تغییرات و برچسب مانند `latest`
- سیگنال‌های دانلود، نصب، ستاره، و نظر
- وضعیت اسکن امنیتی و نظارت

صفحهٔ فهرست، مکان مرجع برای کاربران است تا پیش از نصب بررسی کنند یک Skill یا
Plugin ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

یک Skill بستهٔ متنی نسخه‌دار است که حول `SKILL.md` شکل گرفته است. می‌تواند
شامل فایل‌های پشتیبان، نمونه‌ها، قالب‌ها، و اسکریپت‌ها باشد.

ClawHub فرانت‌متر `SKILL.md` را می‌خواند تا نام Skill،
توضیح، الزامات، متغیرهای محیطی، و فراداده را درک کند. فرادادهٔ دقیق
مهم است چون به کاربران کمک می‌کند تصمیم بگیرند آیا Skill را نصب کنند و
به اسکن‌های خودکار کمک می‌کند ناسازگاری میان رفتار اعلام‌شده و مشاهده‌شده را تشخیص دهند.

[قالب Skill](/fa/clawhub/skill-format) را ببینید.

## Pluginها

Pluginها افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته،
اطلاعات سازگاری، پیوندهای منبع، آرتیفکت‌ها، و رکوردهای نسخه را ذخیره می‌کند.

وقتی OpenClaw یک Plugin را از ClawHub نصب می‌کند، پیش از نصب فرادادهٔ
سازگاری اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API،
حداقل نسخهٔ Gateway، هدف‌های میزبان، الزامات محیطی، و چکیده‌های آرتیفکت باشند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار، یک رکورد نسخهٔ تغییرناپذیر جدید ایجاد می‌کند. ناشران از CLI `clawhub`
برای گردش‌کارهای رجیستری احراز هویت‌شده استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

از اجرای آزمایشی برای پیش‌نمایش محمولهٔ نهایی پیش از بارگذاری استفاده کنید. سپس صفحات عمومی
فرادادهٔ منتشرشده، فایل‌ها، انتساب منبع، و وضعیت اسکن را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

دستورهای نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بعداً بتوانند همان
بستهٔ رجیستری را پیدا کنند. CLI ClawHub همچنین از گردش‌کارهای نصب و
به‌روزرسانی مستقیم Skill برای کاربرانی پشتیبانی می‌کند که پوشه‌های Skill مدیریت‌شده توسط رجیستری را بیرون از
یک فضای کاری کامل OpenClaw می‌خواهند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما انتشارها همچنان مشمول دروازه‌های بارگذاری،
بررسی‌های خودکار، گزارش‌های کاربر، و اقدام ناظر هستند.

صفحات عمومی، در صورت موجود بودن، خلاصه‌های اسکن را نشان می‌دهند. محتوایی که نگه داشته، پنهان،
یا مسدود شده است ممکن است از جست‌وجوی عمومی و جریان‌های نصب ناپدید شود، در حالی که برای
مالک جهت عیب‌یابی قابل مشاهده باقی می‌ماند.

[امنیت + نظارت](/fa/clawhub/security) و
[استفادهٔ قابل قبول](/fa/clawhub/acceptable-usage) را ببینید.

## دسترسی API

ClawHub APIهای خواندنی عمومی برای کشف، جست‌وجو، جزئیات بسته، و
دانلودها ارائه می‌دهد. کاتالوگ‌های شخص ثالث می‌توانند از این APIها استفاده کنند، مشروط بر اینکه به
فهرست مرجع ClawHub پیوند دهند، محدودیت‌های نرخ را رعایت کنند، و از القای تأیید شدن خودداری کنند.

[API عمومی](/fa/clawhub/api) و [HTTP API](/fa/clawhub/http-api) را ببینید.
