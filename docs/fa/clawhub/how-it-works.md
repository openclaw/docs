# ClawHub چگونه کار می‌کند

ClawHub لایهٔ رجیستری برای Skills و Pluginهای OpenClaw است. این لایه به کاربران
جایی برای کشف بسته‌ها، به ناشران جایی برای انتشار نسخه‌ها، و به
OpenClaw فرادادهٔ کافی برای نصب و به‌روزرسانی ایمن آن بسته‌ها می‌دهد.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است با:

- مالک و slug یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها و انتساب منبع
- اطلاعات تغییرات و برچسب مانند `latest`
- سیگنال‌های دانلود، نصب و ستاره
- وضعیت اسکن امنیتی و نظارت

صفحهٔ فهرست، مکان مرجع برای کاربران است تا پیش از نصب، بررسی کنند یک Skill یا
Plugin ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

یک Skill یک بستهٔ متنی نسخه‌دار است که حول `SKILL.md` ساخته می‌شود. می‌تواند شامل
فایل‌های پشتیبان، نمونه‌ها، الگوها و اسکریپت‌ها باشد.

ClawHub برای فهمیدن نام Skill، توضیحات، نیازمندی‌ها، متغیرهای محیطی و فراداده،
frontmatter فایل `SKILL.md` را می‌خواند. فرادادهٔ دقیق اهمیت دارد، چون به کاربران
کمک می‌کند تصمیم بگیرند آیا Skill را نصب کنند یا نه و به اسکن‌های خودکار کمک
می‌کند ناسازگاری میان رفتار اعلام‌شده و رفتار مشاهده‌شده را تشخیص دهند.

نگاه کنید به [قالب Skill](/fa/clawhub/skill-format).

## Pluginها

Pluginها افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته،
اطلاعات سازگاری، لینک‌های منبع، artifactها و رکوردهای نسخه را ذخیره می‌کند.

وقتی OpenClaw یک Plugin را از ClawHub نصب می‌کند، پیش از نصب، فرادادهٔ سازگاری
اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API،
حداقل نسخهٔ Gateway، هدف‌های میزبان، نیازمندی‌های محیطی و چکیده‌های artifact
باشند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار، یک رکورد نسخهٔ تغییرناپذیر جدید ایجاد می‌کند. ناشران برای گردش‌کارهای
رجیستری احرازهویت‌شده از CLI `clawhub` استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

برای پیش‌نمایش payload حل‌شده پیش از بارگذاری، از اجرای آزمایشی استفاده کنید.
سپس صفحه‌های عمومی فرادادهٔ منتشرشده، فایل‌ها، انتساب منبع و وضعیت اسکن را
نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

دستورهای نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بتوانند بعداً همان
بستهٔ رجیستری را حل کنند. CLI ClawHub همچنین از گردش‌کارهای نصب و به‌روزرسانی
مستقیم Skill برای کاربرانی پشتیبانی می‌کند که پوشه‌های Skill مدیریت‌شده توسط
رجیستری را بیرون از یک فضای کاری کامل OpenClaw می‌خواهند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما نسخه‌های منتشرشده همچنان مشمول دروازه‌های
بارگذاری، بررسی‌های خودکار، گزارش‌های کاربران و اقدام ناظر هستند.

صفحه‌های عمومی، وقتی موجود باشد، خلاصه‌های اسکن را نشان می‌دهند. محتوایی که
نگه‌داشته، پنهان یا مسدود شده است ممکن است از جست‌وجوی عمومی و جریان‌های نصب
ناپدید شود، در حالی که برای عیب‌یابی همچنان برای مالک قابل مشاهده می‌ماند.

نگاه کنید به [امنیت](/clawhub/security)، [ممیزی‌های امنیتی](/clawhub/security-audits)،
[نظارت و ایمنی حساب](/fa/clawhub/moderation)، و
[استفادهٔ قابل قبول](/clawhub/acceptable-usage).

## دسترسی API

ClawHub برای کشف، جست‌وجو، جزئیات بسته و دانلودها APIهای خواندنی عمومی ارائه
می‌کند. کاتالوگ‌های شخص ثالث می‌توانند از این APIها استفاده کنند، به شرطی که به
فهرست مرجع ClawHub لینک دهند، محدودیت‌های نرخ را رعایت کنند، و از القای تأیید
رسمی پرهیز کنند.

نگاه کنید به [API عمومی](/fa/clawhub/api) و [HTTP API](/clawhub/http-api).
