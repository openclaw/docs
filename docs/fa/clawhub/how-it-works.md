# ClawHub چگونه کار می‌کند

ClawHub لایهٔ رجیستری برای Skills و Pluginهای OpenClaw است. این لایه به کاربران
جایی برای کشف بسته‌ها می‌دهد، به منتشرکنندگان جایی برای انتشار نسخه‌ها می‌دهد، و
به OpenClaw فرادادهٔ کافی می‌دهد تا آن بسته‌ها را به‌صورت امن نصب و به‌روزرسانی کند.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است با:

- مالک و اسلاگ یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها، و انتساب منبع
- اطلاعات تغییرات و برچسب مانند `latest`
- سیگنال‌های دانلود، نصب، ستاره، و نظر
- وضعیت اسکن امنیتی و تعدیل

صفحهٔ فهرست، مکان مرجع برای کاربران است تا پیش از نصب بررسی کنند یک Skill یا
Plugin ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

یک Skill یک بستهٔ متنی نسخه‌دار است که بر محور `SKILL.md` قرار دارد. می‌تواند
شامل فایل‌های پشتیبان، مثال‌ها، قالب‌ها، و اسکریپت‌ها باشد.

ClawHub فرانت‌متر `SKILL.md` را می‌خواند تا نام Skill،
توضیحات، الزامات، متغیرهای محیطی، و فراداده را بفهمد. فرادادهٔ دقیق مهم است،
زیرا به کاربران کمک می‌کند تصمیم بگیرند آیا Skill را نصب کنند یا نه و به اسکن‌های
خودکار کمک می‌کند ناسازگاری‌ها میان رفتار اعلام‌شده و مشاهده‌شده را تشخیص دهند.

نگاه کنید به [قالب Skill](/fa/clawhub/skill-format).

## Plugins

Pluginها افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته،
اطلاعات سازگاری، پیوندهای منبع، آرتیفکت‌ها، و رکوردهای نسخه را نگه‌داری می‌کند.

وقتی OpenClaw یک Plugin را از ClawHub نصب می‌کند، پیش از نصب فرادادهٔ سازگاری
اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API،
حداقل نسخهٔ Gateway، هدف‌های میزبان، الزامات محیطی، و چکیده‌های آرتیفکت باشند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار، یک رکورد نسخهٔ تغییرناپذیر جدید ایجاد می‌کند. منتشرکنندگان از CLI
`clawhub` برای گردش‌کارهای رجیستری احراز هویت‌شده استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

از اجرای آزمایشی برای پیش‌نمایش محمولهٔ حل‌شده پیش از بارگذاری استفاده کنید. سپس
صفحه‌های عمومی، فرادادهٔ منتشرشده، فایل‌ها، انتساب منبع، و وضعیت اسکن را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

فرمان‌های نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بتوانند بعدا همان
بستهٔ رجیستری را حل کنند. CLI مربوط به ClawHub همچنین برای کاربرانی که پوشه‌های
Skill مدیریت‌شده با رجیستری را بیرون از یک فضای کاری کامل OpenClaw می‌خواهند،
از گردش‌کارهای نصب و به‌روزرسانی مستقیم Skill پشتیبانی می‌کند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما انتشارها همچنان مشمول دروازه‌های بارگذاری،
بررسی‌های خودکار، گزارش‌های کاربران، و اقدام مدیران تعدیل هستند.

صفحه‌های عمومی در صورت موجود بودن، خلاصه‌های اسکن را نشان می‌دهند. محتوایی که
نگه داشته، پنهان، یا مسدود شده است ممکن است از جست‌وجوی عمومی و جریان‌های نصب
ناپدید شود، در حالی که برای عیب‌یابی همچنان برای مالک قابل مشاهده بماند.

نگاه کنید به [امنیت + تعدیل](/fa/clawhub/security) و
[استفادهٔ قابل قبول](/fa/clawhub/acceptable-usage).

## دسترسی API

ClawHub APIهای خواندن عمومی را برای کشف، جست‌وجو، جزئیات بسته، و دانلودها ارائه
می‌کند. کاتالوگ‌های شخص ثالث می‌توانند از این APIها استفاده کنند، به شرطی که به
فهرست مرجع ClawHub پیوند بدهند، محدودیت‌های نرخ را رعایت کنند، و از القای تأیید
رسمی پرهیز کنند.

نگاه کنید به [API عمومی](/fa/clawhub/api) و [HTTP API](/fa/clawhub/http-api).
