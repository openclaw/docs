# نحوه کار ClawHub

ClawHub لایه رجیستری برای Skills و Pluginهای OpenClaw است. به کاربران
جایی برای کشف بسته‌ها، به ناشران جایی برای انتشار نسخه‌ها، و به
OpenClaw فراداده کافی برای نصب و به‌روزرسانی ایمن آن بسته‌ها می‌دهد.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است با:

- یک مالک و نامک یا نام بسته
- یک یا چند نسخه منتشرشده
- فراداده، خلاصه، فایل‌ها، و انتساب منبع
- اطلاعات تغییرات و برچسب مانند `latest`
- سیگنال‌های دانلود، نصب، و ستاره
- وضعیت اسکن امنیتی و نظارت

صفحه فهرست، جایگاه مرجع برای کاربران است تا پیش از نصب، ادعای یک skill یا
plugin درباره کارکردش را بررسی کنند.

## Skills

یک skill یک بسته متنی نسخه‌دار است که حول `SKILL.md` شکل گرفته است. می‌تواند شامل
فایل‌های پشتیبان، نمونه‌ها، قالب‌ها، و اسکریپت‌ها باشد.

ClawHub فرانت‌متر `SKILL.md` را می‌خواند تا نام skill،
توضیح، نیازمندی‌ها، متغیرهای محیطی، و فراداده را بفهمد. فراداده دقیق
مهم است، چون به کاربران کمک می‌کند تصمیم بگیرند آیا skill را نصب کنند و
به اسکن‌های خودکار کمک می‌کند ناسازگاری‌های بین رفتار اعلام‌شده و مشاهده‌شده را تشخیص دهند.

ببینید [قالب skill](/fa/clawhub/skill-format).

## Pluginها

Pluginها افزونه‌های بسته‌بندی‌شده OpenClaw هستند. ClawHub فراداده بسته،
اطلاعات سازگاری، پیوندهای منبع، آرتیفکت‌ها، و رکوردهای نسخه را ذخیره می‌کند.

وقتی OpenClaw یک plugin را از ClawHub نصب می‌کند، پیش از نصب، فراداده سازگاری
اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API،
حداقل نسخه gateway، اهداف میزبان، نیازمندی‌های محیطی، و digestهای آرتیفکت باشند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار، یک رکورد نسخه تغییرناپذیر جدید ایجاد می‌کند. ناشران از CLI `clawhub`
برای گردش‌کارهای احراز هویت‌شده رجیستری استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

از dry runها برای پیش‌نمایش payload حل‌شده پیش از upload استفاده کنید. سپس صفحه‌های عمومی
فراداده منتشرشده، فایل‌ها، انتساب منبع، و وضعیت اسکن را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

دستورهای نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw فراداده منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بتوانند بعداً همان
بسته رجیستری را resolve کنند. CLI ClawHub همچنین از گردش‌کارهای نصب و
به‌روزرسانی مستقیم skill برای کاربرانی پشتیبانی می‌کند که پوشه‌های skill مدیریت‌شده توسط رجیستری را خارج از
یک workspace کامل OpenClaw می‌خواهند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما انتشارها همچنان مشمول gateهای upload،
بررسی‌های خودکار، گزارش‌های کاربران، و اقدام ناظران هستند.

صفحه‌های عمومی، وقتی خلاصه‌های اسکن در دسترس باشند، آن‌ها را نشان می‌دهند. محتوایی که نگه داشته،
پنهان، یا مسدود شده است ممکن است از جریان‌های جست‌وجو و نصب عمومی ناپدید شود، در حالی که برای
مالک جهت عیب‌یابی قابل مشاهده باقی می‌ماند.

ببینید [امنیت](/clawhub/security)، [ممیزی‌های امنیتی](/clawhub/security-audits)،
[نظارت و ایمنی حساب](/fa/clawhub/moderation)، و
[استفاده قابل قبول](/clawhub/acceptable-usage).

## دسترسی API

ClawHub APIهای خواندن عمومی را برای کشف، جست‌وجو، جزئیات بسته، و
دانلودها ارائه می‌کند. کاتالوگ‌های شخص ثالث می‌توانند از این APIها استفاده کنند، وقتی به
فهرست مرجع ClawHub پیوند دهند، محدودیت‌های نرخ را رعایت کنند، و از القای تأیید ضمنی پرهیز کنند.

ببینید [API عمومی](/fa/clawhub/api) و [API HTTP](/clawhub/http-api).
