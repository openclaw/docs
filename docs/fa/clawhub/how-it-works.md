# ClawHub چگونه کار می‌کند

ClawHub لایهٔ رجیستری برای Skills و Pluginهای OpenClaw است. این لایه به کاربران
جایی برای کشف بسته‌ها می‌دهد، به ناشران جایی برای انتشار نسخه‌ها می‌دهد، و به
OpenClaw فرادادهٔ کافی می‌دهد تا آن بسته‌ها را با ایمنی نصب و به‌روزرسانی کند.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است با:

- مالک و slug یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها، و انتساب منبع
- اطلاعات changelog و برچسب مانند `latest`
- سیگنال‌های دانلود، نصب، و ستاره
- وضعیت اسکن امنیتی و نظارت

صفحهٔ فهرست، مکان مرجع برای کاربران است تا پیش از نصب، بررسی کنند یک skill یا
plugin ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

یک skill یک بستهٔ متنی نسخه‌گذاری‌شده با محوریت `SKILL.md` است. می‌تواند شامل
فایل‌های پشتیبان، نمونه‌ها، قالب‌ها، و اسکریپت‌ها باشد.

ClawHub frontmatter فایل `SKILL.md` را می‌خواند تا نام skill،
توضیح، نیازمندی‌ها، متغیرهای محیطی، و فراداده را بفهمد. فرادادهٔ دقیق اهمیت
دارد، چون به کاربران کمک می‌کند تصمیم بگیرند آیا skill را نصب کنند و به اسکن‌های
خودکار کمک می‌کند ناسازگاری‌ها میان رفتار اعلام‌شده و مشاهده‌شده را تشخیص دهند.

[قالب skill](/fa/clawhub/skill-format) را ببینید.

## Pluginها

Pluginها افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته،
اطلاعات سازگاری، پیوندهای منبع، artifactها، و رکوردهای نسخه را ذخیره می‌کند.

وقتی OpenClaw یک plugin را از ClawHub نصب می‌کند، پیش از نصب، فرادادهٔ سازگاری
اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API،
حداقل نسخهٔ gateway، هدف‌های میزبان، نیازمندی‌های محیط، و digestهای artifact
باشند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار، یک رکورد نسخهٔ تغییرناپذیر جدید ایجاد می‌کند. ناشران از CLI `clawhub`
برای گردش‌کارهای رجیستری احراز هویت‌شده استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

از اجرای آزمایشی برای پیش‌نمایش payload حل‌شده پیش از بارگذاری استفاده کنید.
سپس صفحه‌های عمومی فراداده، فایل‌ها، انتساب منبع، و وضعیت اسکن منتشرشده را
نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

دستورهای نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بتوانند همان بستهٔ
رجیستری را بعداً حل کنند. CLI ClawHub همچنین از گردش‌کارهای نصب و به‌روزرسانی
مستقیم skill برای کاربرانی پشتیبانی می‌کند که پوشه‌های skill مدیریت‌شده با
رجیستری را خارج از یک فضای کاری کامل OpenClaw می‌خواهند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما انتشارها همچنان تابع دروازه‌های بارگذاری،
بررسی‌های خودکار، گزارش‌های کاربران، و اقدام ناظران هستند.

صفحه‌های عمومی، در صورت وجود، خلاصه‌های اسکن را نشان می‌دهند. محتوایی که نگه
داشته، پنهان، یا مسدود شده است ممکن است از جریان‌های جست‌وجو و نصب عمومی ناپدید
شود، درحالی‌که برای تشخیص عیب همچنان برای مالک قابل مشاهده می‌ماند.

[امنیت](/clawhub/security)، [ممیزی‌های امنیتی](/clawhub/security-audits)،
[نظارت و ایمنی حساب](/fa/clawhub/moderation)، و
[استفادهٔ پذیرفتنی](/clawhub/acceptable-usage) را ببینید.

## دسترسی API

ClawHub APIهای خواندن عمومی را برای کشف، جست‌وجو، جزئیات بسته، و دانلودها ارائه
می‌کند. کاتالوگ‌های شخص ثالث می‌توانند زمانی از این APIها استفاده کنند که به
فهرست مرجع ClawHub پیوند بدهند، محدودیت‌های نرخ را رعایت کنند، و از القای تأیید
ضمنی خودداری کنند.

[API عمومی](/fa/clawhub/api) و [API HTTP](/clawhub/http-api) را ببینید.
