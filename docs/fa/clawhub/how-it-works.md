# ClawHub چگونه کار می‌کند

ClawHub لایهٔ رجیستری برای Skills و Pluginهای OpenClaw است. به کاربران
جایی برای کشف بسته‌ها، به ناشران جایی برای انتشار نسخه‌ها، و به
OpenClaw فرادادهٔ کافی برای نصب و به‌روزرسانی ایمن آن بسته‌ها می‌دهد.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است که شامل موارد زیر است:

- مالک و slug یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها، و انتساب منبع
- اطلاعات تغییرات و برچسب مانند `latest`
- سیگنال‌های دانلود، نصب، و ستاره
- وضعیت اسکن امنیتی و نظارت

صفحهٔ فهرست، محل مرجع برای کاربران است تا پیش از نصب، بررسی کنند یک Skill یا
Plugin ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

یک Skill یک بستهٔ متنی نسخه‌دار است که حول `SKILL.md` ساخته می‌شود. می‌تواند
شامل فایل‌های پشتیبان، نمونه‌ها، قالب‌ها، و اسکریپت‌ها باشد.

ClawHub فرانت‌متر `SKILL.md` را می‌خواند تا نام Skill،
توضیح، نیازمندی‌ها، متغیرهای محیطی، و فراداده را بفهمد. فرادادهٔ دقیق
مهم است، چون به کاربران کمک می‌کند تصمیم بگیرند آیا Skill را نصب کنند یا نه و
به اسکن‌های خودکار کمک می‌کند ناسازگاری‌ها میان رفتار اعلام‌شده و مشاهده‌شده را تشخیص دهند.

ببینید: [قالب Skill](/fa/clawhub/skill-format).

## Pluginها

Pluginها افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته،
اطلاعات سازگاری، پیوندهای منبع، artifactها، و رکوردهای نسخه را ذخیره می‌کند.

وقتی OpenClaw یک Plugin را از ClawHub نصب می‌کند، پیش از نصب، فرادادهٔ سازگاری
اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API،
حداقل نسخهٔ Gateway، هدف‌های میزبان، نیازمندی‌های محیطی، و digestهای artifact باشند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار، یک رکورد نسخهٔ تغییرناپذیر جدید ایجاد می‌کند. ناشران از CLI مربوط به `clawhub`
برای گردش‌کارهای رجیستری احرازهویت‌شده استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

برای پیش‌نمایش payload نهایی‌شده پیش از بارگذاری، از اجرای خشک استفاده کنید. سپس صفحه‌های عمومی
فرادادهٔ منتشرشده، فایل‌ها، انتساب منبع، و وضعیت اسکن را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

دستورهای نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بتوانند بعدا همان
بستهٔ رجیستری را حل کنند. CLI مربوط به ClawHub همچنین از گردش‌کارهای نصب و
به‌روزرسانی مستقیم Skill برای کاربرانی پشتیبانی می‌کند که پوشه‌های Skill مدیریت‌شده با رجیستری را بیرون از یک
فضای کاری کامل OpenClaw می‌خواهند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما انتشارها همچنان مشمول gateهای بارگذاری،
بررسی‌های خودکار، گزارش‌های کاربران، و اقدام ناظر هستند.

صفحه‌های عمومی، وقتی خلاصه‌های اسکن در دسترس باشند، آن‌ها را نشان می‌دهند. محتوایی که نگه داشته، پنهان،
یا مسدود شده است ممکن است از جریان‌های جست‌وجو و نصب عمومی ناپدید شود، در حالی که برای
مالک جهت عیب‌یابی قابل مشاهده می‌ماند.

ببینید: [امنیت](/fa/clawhub/security)، [ممیزی‌های امنیتی](/fa/clawhub/security-audits)،
[نظارت و ایمنی حساب](/fa/clawhub/moderation)، و
[استفادهٔ قابل قبول](/fa/clawhub/acceptable-usage).

## دسترسی API

ClawHub APIهای خواندن عمومی را برای کشف، جست‌وجو، جزئیات بسته، و
دانلودها ارائه می‌کند. کاتالوگ‌های شخص ثالث می‌توانند از این APIها استفاده کنند، به شرطی که به
فهرست مرجع ClawHub پیوند بدهند، محدودیت‌های نرخ را رعایت کنند، و از القای تأیید رسمی خودداری کنند.

ببینید: [API عمومی](/fa/clawhub/api) و [HTTP API](/fa/clawhub/http-api).
