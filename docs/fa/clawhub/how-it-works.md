# ClawHub چگونه کار می‌کند

ClawHub لایهٔ رجیستری برای Skills و Pluginهای OpenClaw است. به کاربران
جایی برای کشف بسته‌ها می‌دهد، به ناشران جایی برای انتشار نسخه‌ها می‌دهد، و
فرادادهٔ کافی در اختیار OpenClaw می‌گذارد تا آن بسته‌ها را به‌صورت ایمن نصب و به‌روزرسانی کند.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است با:

- مالک و اسلاگ یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها و انتساب منبع
- اطلاعات تغییرات و برچسب‌هایی مانند `latest`
- سیگنال‌های دانلود، نصب و ستاره
- وضعیت اسکن امنیتی و نظارت

صفحهٔ فهرست، جایگاه مرجع برای کاربران است تا پیش از نصب بررسی کنند یک مهارت یا
Plugin ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

یک مهارت یک بستهٔ متنی نسخه‌دار است که حول `SKILL.md` شکل گرفته است. می‌تواند
شامل فایل‌های پشتیبان، مثال‌ها، قالب‌ها و اسکریپت‌ها باشد.

ClawHub فرانت‌متر `SKILL.md` را می‌خواند تا نام مهارت،
توضیح، نیازمندی‌ها، متغیرهای محیطی و فراداده را درک کند. فرادادهٔ دقیق
مهم است، چون به کاربران کمک می‌کند تصمیم بگیرند آیا مهارت را نصب کنند یا نه و
به اسکن‌های خودکار کمک می‌کند ناسازگاری‌های میان رفتار اعلام‌شده و مشاهده‌شده را تشخیص دهند.

[قالب مهارت](/fa/clawhub/skill-format) را ببینید.

## Pluginها

Pluginها افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته،
اطلاعات سازگاری، پیوندهای منبع، آرتیفکت‌ها و رکوردهای نسخه را ذخیره می‌کند.

وقتی OpenClaw یک Plugin را از ClawHub نصب می‌کند، پیش از نصب، فرادادهٔ سازگاری
اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API،
حداقل نسخهٔ Gateway، اهداف میزبان، نیازمندی‌های محیطی و چکیده‌های آرتیفکت باشند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار، یک رکورد نسخهٔ تغییرناپذیر جدید ایجاد می‌کند. ناشران از CLI `clawhub`
برای گردش‌کارهای رجیستری احراز هویت‌شده استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

از اجرای آزمایشی استفاده کنید تا پیش از بارگذاری، payload نهایی‌شده را پیش‌نمایش کنید. سپس صفحه‌های عمومی
فرادادهٔ منتشرشده، فایل‌ها، انتساب منبع و وضعیت اسکن را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

دستورهای نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بتوانند بعدا همان
بستهٔ رجیستری را پیدا کنند. CLI ClawHub همچنین گردش‌کارهای نصب و
به‌روزرسانی مستقیم مهارت را برای کاربرانی پشتیبانی می‌کند که پوشه‌های مهارت مدیریت‌شده با رجیستری را خارج از یک
فضای کاری کامل OpenClaw می‌خواهند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما نسخه‌های منتشرشده همچنان مشمول گیت‌های بارگذاری،
بررسی‌های خودکار، گزارش‌های کاربران و اقدام ناظران هستند.

صفحه‌های عمومی، در صورت موجود بودن، خلاصه‌های اسکن را نشان می‌دهند. محتوایی که نگه داشته، پنهان
یا مسدود شده است ممکن است از جریان‌های جست‌وجو و نصب عمومی ناپدید شود، در حالی که برای
عیب‌یابی همچنان برای مالک قابل مشاهده است.

[امنیت](/clawhub/security)، [ممیزی‌های امنیتی](/clawhub/security-audits)،
[نظارت و ایمنی حساب](/fa/clawhub/moderation)، و
[استفادهٔ پذیرفتنی](/clawhub/acceptable-usage) را ببینید.

## دسترسی API

ClawHub APIهای خواندن عمومی را برای کشف، جست‌وجو، جزئیات بسته و
دانلودها ارائه می‌کند. کاتالوگ‌های شخص ثالث می‌توانند از این APIها استفاده کنند، به شرطی که به
فهرست مرجع ClawHub پیوند بدهند، محدودیت‌های نرخ را رعایت کنند و از القای تأیید رسمی پرهیز کنند.

[API عمومی](/fa/clawhub/api) و [HTTP API](/clawhub/http-api) را ببینید.
