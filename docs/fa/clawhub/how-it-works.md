# سازوکار ClawHub

ClawHub لایهٔ رجیستری برای Skills و Pluginهای OpenClaw است. این سامانه فضایی برای یافتن بسته‌ها در اختیار کاربران، محلی برای انتشار نسخه‌ها در اختیار ناشران، و فرادادهٔ کافی برای نصب و به‌روزرسانی ایمن این بسته‌ها در اختیار OpenClaw قرار می‌دهد.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است که شامل موارد زیر می‌شود:

- یک مالک و نامک یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها و انتساب منبع
- اطلاعات تغییرات و برچسب‌ها، مانند `latest`
- شاخص‌های دانلود، نصب و ستاره
- وضعیت پویش امنیتی و نظارت

صفحهٔ فهرست، مرجع اصلی کاربران برای بررسی قابلیت‌هایی است که یک Skills یا Plugin پیش از نصب ادعا می‌کند ارائه می‌دهد.

## Skills

هر Skills یک بستهٔ متنی نسخه‌بندی‌شده با محوریت `SKILL.md` است. این بسته می‌تواند شامل فایل‌های پشتیبان، نمونه‌ها، الگوها و اسکریپت‌ها باشد.

ClawHub برای درک نام، توضیحات، الزامات، متغیرهای محیطی و فرادادهٔ Skills، فرادادهٔ ابتدایی `SKILL.md` را می‌خواند. دقت فراداده اهمیت دارد، زیرا به کاربران در تصمیم‌گیری برای نصب Skills کمک می‌کند و امکان تشخیص مغایرت میان رفتار اعلام‌شده و رفتار مشاهده‌شده را برای پویش‌های خودکار فراهم می‌سازد.

به [قالب Skills](/clawhub/skill-format) مراجعه کنید.

## Pluginها

Pluginها افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته، اطلاعات سازگاری، پیوندهای منبع، مصنوعات و رکوردهای نسخه را ذخیره می‌کند.

هنگامی که OpenClaw یک Plugin را از ClawHub نصب می‌کند، پیش از نصب فرادادهٔ سازگاری اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API، حداقل نسخهٔ Gateway، میزبان‌های هدف، الزامات محیطی و چکیده‌های مصنوعات باشند.

وقتی می‌خواهید رجیستری مرجع نهایی باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار، یک رکورد نسخهٔ تغییرناپذیر جدید ایجاد می‌کند. ناشران برای گردش‌کارهای احراز هویت‌شدهٔ رجیستری از CLI ‏`clawhub` استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

برای پیش‌نمایش بار دادهٔ نهایی پیش از بارگذاری، از اجرای آزمایشی استفاده کنید. سپس صفحات عمومی، فرادادهٔ منتشرشده، فایل‌ها، انتساب منبع و وضعیت پویش را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

فرمان‌های نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بتوانند بعداً همان بستهٔ رجیستری را پیدا کنند. CLI ‏ClawHub همچنین برای کاربرانی که پوشه‌های Skills مدیریت‌شده توسط رجیستری را خارج از یک فضای کاری کامل OpenClaw می‌خواهند، از گردش‌کارهای نصب و به‌روزرسانی مستقیم Skills پشتیبانی می‌کند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما نسخه‌ها همچنان مشمول دروازه‌های بارگذاری، بررسی‌های خودکار، گزارش‌های کاربران و اقدامات ناظران هستند.

صفحات عمومی در صورت موجود بودن، خلاصهٔ پویش‌ها را نمایش می‌دهند. محتوای متوقف‌شده، پنهان‌شده یا مسدودشده ممکن است از جست‌وجوی عمومی و گردش‌کارهای نصب ناپدید شود، اما برای اهداف عیب‌یابی همچنان برای مالک قابل مشاهده باشد.

به [امنیت](/fa/clawhub/security)، [ممیزی‌های امنیتی](/clawhub/security-audits)، [نظارت و ایمنی حساب](/fa/clawhub/moderation) و [استفادهٔ قابل‌قبول](/clawhub/acceptable-usage) مراجعه کنید.

## دسترسی API

ClawHub برای کشف، جست‌وجو، جزئیات بسته و دانلودها، APIهای خواندنی عمومی ارائه می‌دهد. فهرست‌های شخص ثالث می‌توانند هنگامی از این APIها استفاده کنند که به فهرست مرجع ClawHub پیوند دهند، محدودیت نرخ را رعایت کنند و از القای تأییدشدگی بپرهیزند.

به [API عمومی](/clawhub/api) و [API ‏HTTP](/clawhub/http-api) مراجعه کنید.
