# ClawHub چگونه کار می‌کند

ClawHub لایهٔ رجیستری برای Skills و Pluginهای OpenClaw است. این لایه به کاربران جایی برای کشف بسته‌ها، به ناشران جایی برای انتشار نسخه‌ها، و به OpenClaw فرادادهٔ کافی برای نصب و به‌روزرسانی امن آن بسته‌ها می‌دهد.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است با:

- مالک و نامک یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها، و انتساب منبع
- اطلاعات changelog و برچسب مانند `latest`
- سیگنال‌های دانلود، نصب، و ستاره
- وضعیت اسکن امنیتی و تعدیل محتوا

صفحهٔ فهرست، محل مرجع برای کاربران است تا پیش از نصب، بررسی کنند یک skill یا plugin ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

یک skill یک بستهٔ متنی نسخه‌دار با محوریت `SKILL.md` است. می‌تواند شامل فایل‌های پشتیبان، نمونه‌ها، قالب‌ها، و اسکریپت‌ها باشد.

ClawHub فرانت‌متر `SKILL.md` را می‌خواند تا نام skill، توضیحات، الزامات، متغیرهای محیطی، و فراداده را درک کند. فرادادهٔ دقیق مهم است، چون به کاربران کمک می‌کند تصمیم بگیرند آیا skill را نصب کنند یا نه و به اسکن‌های خودکار کمک می‌کند ناهماهنگی‌ها میان رفتار اعلام‌شده و مشاهده‌شده را تشخیص دهند.

ببینید [قالب skill](/fa/clawhub/skill-format).

## Pluginها

Pluginها افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته، اطلاعات سازگاری، پیوندهای منبع، آرتیفکت‌ها، و رکوردهای نسخه را ذخیره می‌کند.

وقتی OpenClaw یک plugin را از ClawHub نصب می‌کند، پیش از نصب فرادادهٔ سازگاری اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API، حداقل نسخهٔ gateway، اهداف میزبان، الزامات محیطی، و digestهای آرتیفکت باشند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار، یک رکورد نسخهٔ تغییرناپذیر جدید ایجاد می‌کند. ناشران از CLI `clawhub` برای گردش‌کارهای رجیستری احراز هویت‌شده استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

از اجرای آزمایشی استفاده کنید تا payload حل‌شده را پیش از بارگذاری پیش‌نمایش کنید. سپس صفحه‌های عمومی فرادادهٔ منتشرشده، فایل‌ها، انتساب منبع، و وضعیت اسکن را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

دستورهای نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بتوانند همان بستهٔ رجیستری را بعداً حل کنند. CLI ClawHub همچنین از گردش‌کارهای نصب و به‌روزرسانی مستقیم skill برای کاربرانی پشتیبانی می‌کند که پوشه‌های skill مدیریت‌شده توسط رجیستری را بیرون از یک workspace کامل OpenClaw می‌خواهند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما انتشارها همچنان مشمول دروازه‌های بارگذاری، بررسی‌های خودکار، گزارش‌های کاربر، و اقدام ناظر هستند.

صفحه‌های عمومی، در صورت موجود بودن، خلاصه‌های اسکن را نشان می‌دهند. محتوایی که نگه داشته، پنهان، یا مسدود شده است ممکن است از جریان‌های جست‌وجوی عمومی و نصب ناپدید شود، در حالی که برای عیب‌یابی همچنان برای مالک قابل مشاهده می‌ماند.

ببینید [امنیت](/clawhub/security)، [ممیزی‌های امنیتی](/clawhub/security-audits)، [تعدیل محتوا و ایمنی حساب](/fa/clawhub/moderation)، و [استفادهٔ قابل قبول](/clawhub/acceptable-usage).

## دسترسی API

ClawHub APIهای خواندن عمومی را برای کشف، جست‌وجو، جزئیات بسته، و دانلودها ارائه می‌کند. کاتالوگ‌های شخص ثالث می‌توانند از این APIها استفاده کنند، به شرطی که به فهرست مرجع ClawHub پیوند بدهند، محدودیت‌های نرخ را رعایت کنند، و از القای تأیید رسمی خودداری کنند.

ببینید [API عمومی](/fa/clawhub/api) و [HTTP API](/clawhub/http-api).
