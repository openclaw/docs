# ClawHub چگونه کار می‌کند

ClawHub لایهٔ رجیستری برای Skills و Pluginهای OpenClaw است. به کاربران
جایی برای کشف بسته‌ها، به منتشرکنندگان جایی برای انتشار نسخه‌ها، و به
OpenClaw فرادادهٔ کافی می‌دهد تا آن بسته‌ها را با ایمنی نصب و به‌روزرسانی کند.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است با:

- مالک و slug یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها، و انتساب منبع
- اطلاعات changelog و برچسب مانند `latest`
- نشانه‌های دانلود، نصب، و ستاره‌گذاری
- وضعیت اسکن امنیتی و نظارت

صفحهٔ فهرست، مکان معیار برای کاربران است تا پیش از نصب، بررسی کنند یک Skills یا
Plugin ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

یک Skills یک بستهٔ متنی نسخه‌بندی‌شده است که محور آن `SKILL.md` است. می‌تواند شامل
فایل‌های پشتیبان، مثال‌ها، قالب‌ها، و اسکریپت‌ها باشد.

ClawHub فرادادهٔ ابتدایی `SKILL.md` را می‌خواند تا نام Skills،
توضیح، نیازمندی‌ها، متغیرهای محیطی، و فراداده را بفهمد. فرادادهٔ دقیق
مهم است، چون به کاربران کمک می‌کند تصمیم بگیرند آیا Skills را نصب کنند یا نه و
به اسکن‌های خودکار کمک می‌کند ناهماهنگی‌های میان رفتار اعلام‌شده و مشاهده‌شده را تشخیص دهند.

ببینید [قالب Skills](/fa/clawhub/skill-format).

## Pluginها

Pluginها افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته،
اطلاعات سازگاری، پیوندهای منبع، آرتیفکت‌ها، و رکوردهای نسخه را ذخیره می‌کند.

وقتی OpenClaw یک Plugin را از ClawHub نصب می‌کند، پیش از نصب، فرادادهٔ
سازگاری اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API،
حداقل نسخهٔ Gateway، هدف‌های میزبان، نیازمندی‌های محیطی، و digestهای آرتیفکت باشند.

وقتی می‌خواهید رجیستری منبع معتبر باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار، یک رکورد نسخهٔ تغییرناپذیر جدید ایجاد می‌کند. منتشرکنندگان از CLI `clawhub`
برای گردش‌کارهای رجیستری احراز هویت‌شده استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

از اجرای آزمایشی استفاده کنید تا پیش از بارگذاری، payload نهایی‌شده را پیش‌نمایش کنید. سپس صفحه‌های عمومی
فرادادهٔ منتشرشده، فایل‌ها، انتساب منبع، و وضعیت اسکن را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

دستورهای نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بتوانند بعداً همان
بستهٔ رجیستری را resolve کنند. CLI ClawHub همچنین از گردش‌کارهای نصب و
به‌روزرسانی مستقیم Skills برای کاربرانی پشتیبانی می‌کند که پوشه‌های Skills مدیریت‌شده توسط رجیستری را خارج از یک
workspace کامل OpenClaw می‌خواهند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما releaseها همچنان تابع gateهای بارگذاری،
بررسی‌های خودکار، گزارش‌های کاربران، و اقدام ناظر هستند.

صفحه‌های عمومی در صورت موجود بودن، خلاصه‌های اسکن را نشان می‌دهند. محتوایی که نگه داشته، پنهان،
یا مسدود شده است، ممکن است از جریان‌های جست‌وجو و نصب عمومی ناپدید شود، در حالی که برای
تشخیص عیب همچنان برای مالک قابل مشاهده می‌ماند.

ببینید [امنیت](/clawhub/security)، [ممیزی‌های امنیتی](/clawhub/security-audits)،
[نظارت و ایمنی حساب](/fa/clawhub/moderation)، و
[استفادهٔ قابل قبول](/clawhub/acceptable-usage).

## دسترسی API

ClawHub APIهای خواندنی عمومی برای کشف، جست‌وجو، جزئیات بسته، و
دانلودها ارائه می‌کند. کاتالوگ‌های شخص ثالث می‌توانند از این APIها استفاده کنند، به‌شرطی که به
فهرست معیار ClawHub پیوند بدهند، محدودیت‌های نرخ را رعایت کنند، و از القای تأیید رسمی پرهیز کنند.

ببینید [API عمومی](/clawhub/api) و [HTTP API](/clawhub/http-api).
