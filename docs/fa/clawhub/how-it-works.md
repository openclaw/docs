# ClawHub چگونه کار می‌کند

ClawHub لایهٔ رجیستری برای skills و plugins در OpenClaw است. این لایه به کاربران
جایی برای کشف بسته‌ها، به ناشران جایی برای انتشار نسخه‌ها، و به
OpenClaw فرادادهٔ کافی می‌دهد تا آن بسته‌ها را با اطمینان نصب و به‌روزرسانی کند.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است با:

- یک مالک و slug یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها، و انتساب منبع
- اطلاعات changelog و tag مانند `latest`
- سیگنال‌های دانلود، نصب، ستاره، و دیدگاه
- وضعیت اسکن امنیتی و نظارت

صفحهٔ فهرست، مرجع اصلی برای کاربران است تا پیش از نصب بررسی کنند یک skill یا
plugin ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

یک skill یک بستهٔ متنی نسخه‌بندی‌شده است که حول `SKILL.md` ساخته شده است. می‌تواند
شامل فایل‌های پشتیبان، نمونه‌ها، قالب‌ها، و اسکریپت‌ها باشد.

ClawHub برای درک نام skill، توضیح،
نیازمندی‌ها، متغیرهای محیطی، و فراداده، frontmatter فایل `SKILL.md` را می‌خواند. فرادادهٔ دقیق
مهم است، چون به کاربران کمک می‌کند تصمیم بگیرند آیا skill را نصب کنند یا نه و
به اسکن‌های خودکار کمک می‌کند ناسازگاری‌های بین رفتار اعلام‌شده و مشاهده‌شده را تشخیص دهند.

ببینید [قالب Skill](/fa/clawhub/skill-format).

## Plugins

Plugins افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته،
اطلاعات سازگاری، پیوندهای منبع، artifacts، و رکوردهای نسخه را ذخیره می‌کند.

وقتی OpenClaw یک plugin را از ClawHub نصب می‌کند، پیش از نصب فرادادهٔ سازگاری
اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API،
حداقل نسخهٔ Gateway، اهداف میزبان، نیازمندی‌های محیط، و digestهای artifact باشند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار، یک رکورد نسخهٔ جدید و تغییرناپذیر ایجاد می‌کند. ناشران برای گردش‌کارهای رجیستری احراز هویت‌شده از CLI
`clawhub` استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

از اجرای آزمایشی استفاده کنید تا پیش از بارگذاری، payload حل‌شده را پیش‌نمایش کنید. سپس صفحه‌های عمومی
فرادادهٔ منتشرشده، فایل‌ها، انتساب منبع، و وضعیت اسکن را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

فرمان‌های نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بتوانند بعداً همان
بستهٔ رجیستری را resolve کنند. CLI مربوط به ClawHub همچنین برای کاربرانی که پوشه‌های skill مدیریت‌شده توسط رجیستری را بیرون از یک
workspace کامل OpenClaw می‌خواهند، از گردش‌کارهای نصب و به‌روزرسانی مستقیم skill پشتیبانی می‌کند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما انتشارها همچنان مشمول gateهای بارگذاری،
بررسی‌های خودکار، گزارش‌های کاربران، و اقدام ناظران هستند.

صفحه‌های عمومی، در صورت موجود بودن، خلاصه‌های اسکن را نشان می‌دهند. محتوایی که نگه داشته، پنهان،
یا مسدود شده است ممکن است از جریان‌های جست‌وجو و نصب عمومی ناپدید شود، در حالی که برای عیب‌یابی
برای مالک قابل مشاهده باقی می‌ماند.

ببینید [امنیت + نظارت](/fa/clawhub/security) و
[استفادهٔ قابل قبول](/fa/clawhub/acceptable-usage).

## دسترسی API

ClawHub APIهای خواندن عمومی را برای کشف، جست‌وجو، جزئیات بسته، و
دانلودها ارائه می‌کند. کاتالوگ‌های شخص ثالث می‌توانند از این APIها استفاده کنند، به‌شرطی که به
فهرست مرجع ClawHub پیوند بدهند، محدودیت‌های نرخ را رعایت کنند، و از القای تأیید رسمی خودداری کنند.

ببینید [API عمومی](/fa/clawhub/api) و [HTTP API](/fa/clawhub/http-api).
