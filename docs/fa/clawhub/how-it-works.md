# ClawHub چگونه کار می‌کند

ClawHub لایهٔ رجیستری برای Skills و Pluginهای OpenClaw است. به کاربران
جایی برای کشف بسته‌ها می‌دهد، به ناشران جایی برای انتشار نسخه‌ها می‌دهد، و
فرادادهٔ کافی در اختیار OpenClaw می‌گذارد تا آن بسته‌ها را با ایمنی نصب و به‌روزرسانی کند.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است با:

- مالک و slug یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها، و انتساب منبع
- اطلاعات changelog و برچسب مانند `latest`
- سیگنال‌های دانلود، نصب، و ستاره
- وضعیت اسکن امنیتی و مدیریت محتوا

صفحهٔ فهرست، جایگاه مرجع برای کاربران است تا پیش از نصب، بررسی کنند یک skill یا
Plugin ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

یک skill یک بستهٔ متنی نسخه‌بندی‌شده است که حول `SKILL.md` شکل گرفته است. می‌تواند شامل
فایل‌های پشتیبان، مثال‌ها، الگوها، و اسکریپت‌ها باشد.

ClawHub frontmatter فایل `SKILL.md` را می‌خواند تا نام skill،
توضیح، نیازمندی‌ها، متغیرهای محیطی، و فراداده را بفهمد. فرادادهٔ دقیق
اهمیت دارد، چون به کاربران کمک می‌کند تصمیم بگیرند آیا skill را نصب کنند و
به اسکن‌های خودکار کمک می‌کند ناسازگاری میان رفتار اعلام‌شده و مشاهده‌شده را تشخیص دهند.

[قالب Skill](/fa/clawhub/skill-format) را ببینید.

## Pluginها

Pluginها افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته،
اطلاعات سازگاری، پیوندهای منبع، artifactها، و رکوردهای نسخه را نگه می‌دارد.

وقتی OpenClaw یک Plugin را از ClawHub نصب می‌کند، پیش از نصب فرادادهٔ سازگاری
اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API،
حداقل نسخهٔ gateway، هدف‌های میزبان، نیازمندی‌های محیط، و digestهای artifact باشند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار یک رکورد نسخهٔ تغییرناپذیر جدید ایجاد می‌کند. ناشران از CLI `clawhub`
برای گردش‌کارهای رجیستری احراز هویت‌شده استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

از اجراهای dry run برای پیش‌نمایش payload نهایی پیش از بارگذاری استفاده کنید. سپس صفحه‌های عمومی
فرادادهٔ منتشرشده، فایل‌ها، انتساب منبع، و وضعیت اسکن را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

فرمان‌های نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بعداً بتوانند همان
بستهٔ رجیستری را resolve کنند. CLI ClawHub همچنین از گردش‌کارهای نصب و
به‌روزرسانی مستقیم skill برای کاربرانی پشتیبانی می‌کند که پوشه‌های skill مدیریت‌شده با رجیستری را خارج از یک
فضای کاری کامل OpenClaw می‌خواهند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما انتشارها همچنان مشمول gateهای بارگذاری،
بررسی‌های خودکار، گزارش‌های کاربران، و اقدام مدیران محتوا هستند.

صفحه‌های عمومی، وقتی موجود باشد، خلاصه‌های اسکن را نشان می‌دهند. محتوایی که نگه داشته، پنهان،
یا مسدود شده است ممکن است از جست‌وجوی عمومی و گردش‌کارهای نصب ناپدید شود، در حالی که برای
مالک جهت عیب‌یابی قابل مشاهده باقی می‌ماند.

[امنیت](/clawhub/security)، [ممیزی‌های امنیتی](/clawhub/security-audits)،
[مدیریت محتوا و ایمنی حساب](/fa/clawhub/moderation)، و
[کاربرد قابل قبول](/clawhub/acceptable-usage) را ببینید.

## دسترسی API

ClawHub APIهای خواندن عمومی را برای کشف، جست‌وجو، جزئیات بسته، و
دانلودها ارائه می‌کند. کاتالوگ‌های شخص ثالث می‌توانند از این APIها استفاده کنند، به شرطی که به
فهرست مرجع ClawHub پیوند دهند، محدودیت‌های نرخ را رعایت کنند، و از القای تأیید رسمی پرهیز کنند.

[API عمومی](/fa/clawhub/api) و [HTTP API](/clawhub/http-api) را ببینید.
