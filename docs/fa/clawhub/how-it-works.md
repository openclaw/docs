# ClawHub چگونه کار می‌کند

ClawHub لایهٔ رجیستری برای Skills و Pluginهای OpenClaw است. این سرویس به کاربران
جایی برای کشف بسته‌ها، به ناشران جایی برای انتشار نسخه‌ها، و به
OpenClaw فرادادهٔ کافی برای نصب و به‌روزرسانی ایمن آن بسته‌ها می‌دهد.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است با:

- مالک و slug یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها و انتساب منبع
- اطلاعات changelog و tag مانند `latest`
- سیگنال‌های دانلود، نصب و ستاره
- وضعیت اسکن امنیتی و نظارت

صفحهٔ فهرست، محل مرجع برای کاربران است تا پیش از نصب بررسی کنند یک مهارت یا
Plugin ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

یک مهارت یک بستهٔ متنی نسخه‌دار است که حول `SKILL.md` شکل گرفته است. این بسته می‌تواند
فایل‌های پشتیبان، نمونه‌ها، قالب‌ها و اسکریپت‌ها را شامل شود.

ClawHub برای درک نام مهارت، توضیحات، نیازمندی‌ها، متغیرهای محیطی و فراداده،
frontmatter فایل `SKILL.md` را می‌خواند. فرادادهٔ دقیق مهم است، چون به کاربران کمک می‌کند
تصمیم بگیرند آیا مهارت را نصب کنند یا نه، و به اسکن‌های خودکار کمک می‌کند ناسازگاری‌های
میان رفتار اعلام‌شده و رفتار مشاهده‌شده را تشخیص دهند.

[قالب مهارت](/fa/clawhub/skill-format) را ببینید.

## Pluginها

Pluginها افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته،
اطلاعات سازگاری، پیوندهای منبع، artifactها و رکوردهای نسخه را ذخیره می‌کند.

وقتی OpenClaw یک Plugin را از ClawHub نصب می‌کند، پیش از نصب فرادادهٔ سازگاری
اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند سازگاری API،
حداقل نسخهٔ Gateway، هدف‌های میزبان، نیازمندی‌های محیطی و digestهای artifact
را شامل شوند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار، یک رکورد نسخهٔ immutable جدید ایجاد می‌کند. ناشران برای گردش‌کارهای رجیستری احرازشده
از CLI مربوط به `clawhub` استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

از dry runها برای پیش‌نمایش payload حل‌شده پیش از upload استفاده کنید. سپس صفحه‌های عمومی
فرادادهٔ منتشرشده، فایل‌ها، انتساب منبع و وضعیت اسکن را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

دستورهای نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بتوانند همان بستهٔ
رجیستری را بعداً resolve کنند. CLI مربوط به ClawHub همچنین از گردش‌کارهای نصب و
به‌روزرسانی مستقیم مهارت برای کاربرانی پشتیبانی می‌کند که پوشه‌های مهارت مدیریت‌شده با رجیستری را بیرون از یک
workspace کامل OpenClaw می‌خواهند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما انتشارها همچنان مشمول gateهای upload،
بررسی‌های خودکار، گزارش‌های کاربران و اقدام ناظر هستند.

صفحه‌های عمومی، هر زمان در دسترس باشد، خلاصه‌های اسکن را نشان می‌دهند. محتوایی که نگه داشته، پنهان
یا مسدود شده باشد ممکن است از جست‌وجوی عمومی و جریان‌های نصب ناپدید شود، در حالی که برای
مالک جهت عیب‌یابی قابل مشاهده باقی می‌ماند.

[امنیت](/clawhub/security)، [ممیزی‌های امنیتی](/clawhub/security-audits)،
[نظارت و ایمنی حساب](/fa/clawhub/moderation)، و
[استفادهٔ قابل قبول](/clawhub/acceptable-usage) را ببینید.

## دسترسی API

ClawHub APIهای خواندن عمومی را برای کشف، جست‌وجو، جزئیات بسته و
دانلودها ارائه می‌کند. کاتالوگ‌های شخص ثالث می‌توانند از این APIها استفاده کنند، به شرطی که به فهرست
مرجع ClawHub پیوند دهند، محدودیت‌های نرخ را رعایت کنند و از القای تأیید رسمی پرهیز کنند.

[API عمومی](/clawhub/api) و [HTTP API](/clawhub/http-api) را ببینید.
