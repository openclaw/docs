# ClawHub چگونه کار می‌کند

ClawHub لایهٔ رجیستری برای Skills و plugins در OpenClaw است. این لایه به کاربران
جایی برای کشف بسته‌ها، به ناشران جایی برای انتشار نسخه‌ها، و به
OpenClaw فرادادهٔ کافی برای نصب و به‌روزرسانی ایمن آن بسته‌ها می‌دهد.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است که شامل موارد زیر است:

- مالک و slug یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها، و انتساب منبع
- اطلاعات changelog و برچسب مانند `latest`
- سیگنال‌های دانلود، نصب، و ستاره
- وضعیت اسکن امنیتی و نظارت

صفحهٔ فهرست، مکان مرجع برای کاربران است تا پیش از نصب بررسی کنند یک skill یا
plugin ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

یک skill یک بستهٔ متنی نسخه‌بندی‌شده است که حول `SKILL.md` شکل می‌گیرد. می‌تواند شامل
فایل‌های پشتیبان، نمونه‌ها، الگوها، و اسکریپت‌ها باشد.

ClawHub بخش frontmatter در `SKILL.md` را می‌خواند تا نام skill،
توضیح، نیازمندی‌ها، متغیرهای محیطی، و فراداده را بفهمد. فرادادهٔ دقیق
مهم است، زیرا به کاربران کمک می‌کند تصمیم بگیرند آیا skill را نصب کنند و
به اسکن‌های خودکار کمک می‌کند ناسازگاری‌ها بین رفتار اعلام‌شده و مشاهده‌شده را تشخیص دهند.

[قالب Skill](/fa/clawhub/skill-format) را ببینید.

## Plugins

Plugins افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته،
اطلاعات سازگاری، پیوندهای منبع، مصنوعات، و رکوردهای نسخه را ذخیره می‌کند.

وقتی OpenClaw یک plugin را از ClawHub نصب می‌کند، پیش از نصب، فرادادهٔ سازگاری
اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API،
حداقل نسخهٔ Gateway، اهداف میزبان، نیازمندی‌های محیطی، و digest مصنوعات باشند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار یک رکورد نسخهٔ تغییرناپذیر جدید ایجاد می‌کند. ناشران از CLI `clawhub`
برای گردش‌کارهای رجیستری احراز هویت‌شده استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

از dry runها برای پیش‌نمایش payload حل‌شده پیش از بارگذاری استفاده کنید. سپس صفحه‌های عمومی
فرادادهٔ منتشرشده، فایل‌ها، انتساب منبع، و وضعیت اسکن را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

فرمان‌های نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بتوانند همان
بستهٔ رجیستری را بعداً حل کنند. CLI مربوط به ClawHub همچنین از گردش‌کارهای نصب و
به‌روزرسانی مستقیم skill برای کاربرانی پشتیبانی می‌کند که پوشه‌های skill مدیریت‌شده توسط رجیستری را بیرون از
یک فضای کاری کامل OpenClaw می‌خواهند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما releaseها همچنان تابع دروازه‌های بارگذاری،
بررسی‌های خودکار، گزارش‌های کاربر، و اقدام ناظر هستند.

صفحه‌های عمومی، در صورت موجود بودن، خلاصه‌های اسکن را نشان می‌دهند. محتوایی که نگه داشته، پنهان،
یا مسدود شده است ممکن است از جریان‌های جست‌وجو و نصب عمومی ناپدید شود، در حالی که برای
مالک جهت عیب‌یابی همچنان قابل مشاهده بماند.

[امنیت](/fa/clawhub/security)، [ممیزی‌های امنیتی](/fa/clawhub/security-audits)،
[نظارت و ایمنی حساب](/fa/clawhub/moderation)، و
[استفادهٔ قابل قبول](/fa/clawhub/acceptable-usage) را ببینید.

## دسترسی API

ClawHub APIهای خواندن عمومی را برای کشف، جست‌وجو، جزئیات بسته، و
دانلودها ارائه می‌کند. کاتالوگ‌های شخص ثالث می‌توانند از این APIها استفاده کنند، به شرطی که به
فهرست مرجع ClawHub پیوند دهند، محدودیت‌های نرخ را رعایت کنند، و از القای تأیید خودداری کنند.

[API عمومی](/fa/clawhub/api) و [HTTP API](/fa/clawhub/http-api) را ببینید.
