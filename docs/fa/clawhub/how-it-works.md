# ClawHub چگونه کار می‌کند

ClawHub لایهٔ رجیستری برای Skills و Pluginهای OpenClaw است. به کاربران
جایی برای کشف بسته‌ها می‌دهد، به ناشران جایی برای انتشار نسخه‌ها می‌دهد، و
به OpenClaw فرادادهٔ کافی می‌دهد تا آن بسته‌ها را با ایمنی نصب و به‌روزرسانی کند.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است با:

- مالک و اسلاگ یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها، و انتساب منبع
- اطلاعات changelog و tag مانند `latest`
- سیگنال‌های دانلود، نصب، ستاره، و دیدگاه
- وضعیت اسکن امنیتی و تعدیل

صفحهٔ فهرست، محل مرجع برای کاربران است تا پیش از نصب، بررسی کنند یک Skills یا
Plugin ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

یک Skills بستهٔ متنی نسخه‌دار است که بر `SKILL.md` متمرکز است. می‌تواند شامل
فایل‌های پشتیبان، نمونه‌ها، الگوها، و اسکریپت‌ها باشد.

ClawHub فرانت‌متر `SKILL.md` را می‌خواند تا نام Skills،
شرح، الزامات، متغیرهای محیطی، و فراداده را بفهمد. فرادادهٔ دقیق
اهمیت دارد، چون به کاربران کمک می‌کند تصمیم بگیرند آیا Skills را نصب کنند و
به اسکن‌های خودکار کمک می‌کند ناسازگاری‌های میان رفتار اعلام‌شده و مشاهده‌شده را تشخیص دهند.

ببینید [قالب Skills](/fa/clawhub/skill-format).

## Pluginها

Pluginها افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته،
اطلاعات سازگاری، پیوندهای منبع، مصنوعات، و رکوردهای نسخه را ذخیره می‌کند.

وقتی OpenClaw یک Plugin را از ClawHub نصب می‌کند، پیش از نصب فرادادهٔ سازگاری
اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API،
حداقل نسخهٔ Gateway، اهداف میزبان، الزامات محیطی، و چکیده‌های مصنوع باشند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار یک رکورد نسخهٔ تغییرناپذیر جدید ایجاد می‌کند. ناشران برای گردش‌کارهای رجیستری احرازهویت‌شده از CLI
`clawhub` استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

برای پیش‌نمایش payload حل‌شده پیش از بارگذاری، از اجرای آزمایشی استفاده کنید. سپس صفحه‌های عمومی
فرادادهٔ منتشرشده، فایل‌ها، انتساب منبع، و وضعیت اسکن را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

فرمان‌های نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بتوانند همان
بستهٔ رجیستری را بعداً حل کنند. CLI مربوط به ClawHub همچنین از گردش‌کارهای نصب و
به‌روزرسانی مستقیم Skills برای کاربرانی پشتیبانی می‌کند که پوشه‌های Skills مدیریت‌شده توسط رجیستری را خارج از
یک فضای کاری کامل OpenClaw می‌خواهند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما انتشارها همچنان مشمول gateهای بارگذاری،
بررسی‌های خودکار، گزارش‌های کاربران، و اقدام ناظران هستند.

صفحه‌های عمومی در صورت موجود بودن، خلاصه‌های اسکن را نشان می‌دهند. محتوایی که نگه‌داشته، پنهان،
یا مسدود شده است ممکن است از جست‌وجوی عمومی و گردش‌کارهای نصب ناپدید شود، در حالی که برای
مالک جهت عیب‌یابی قابل مشاهده باقی می‌ماند.

ببینید [امنیت + تعدیل](/fa/clawhub/security) و
[استفادهٔ پذیرفتنی](/fa/clawhub/acceptable-usage).

## دسترسی API

ClawHub APIهای خواندنی عمومی برای کشف، جست‌وجو، جزئیات بسته، و
دانلودها ارائه می‌کند. کاتالوگ‌های شخص ثالث می‌توانند از این APIها استفاده کنند، به شرطی که به
فهرست مرجع ClawHub پیوند دهند، محدودیت‌های نرخ را رعایت کنند، و از القای تأیید رسمی خودداری کنند.

ببینید [API عمومی](/fa/clawhub/api) و [HTTP API](/fa/clawhub/http-api).
