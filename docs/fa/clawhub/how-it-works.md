# نحوه کار ClawHub

ClawHub لایه رجیستری برای مهارت‌ها و افزونه‌های OpenClaw است. به کاربران
جایی برای کشف بسته‌ها می‌دهد، به ناشران جایی برای انتشار نسخه‌ها می‌دهد، و
فراداده کافی در اختیار OpenClaw می‌گذارد تا آن بسته‌ها را با ایمنی نصب و به‌روزرسانی کند.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است با:

- مالک و اسلاگ یا نام بسته
- یک یا چند نسخه منتشرشده
- فراداده، خلاصه، فایل‌ها، و انتساب منبع
- اطلاعات تغییرات و برچسب مانند `latest`
- سیگنال‌های دانلود، نصب، و ستاره
- وضعیت اسکن امنیتی و نظارت

صفحه فهرست، جایگاه مرجع برای کاربران است تا پیش از نصب، بررسی کنند یک مهارت یا
افزونه ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

مهارت یک بسته متنی نسخه‌بندی‌شده با محوریت `SKILL.md` است. می‌تواند شامل
فایل‌های پشتیبان، نمونه‌ها، قالب‌ها، و اسکریپت‌ها باشد.

ClawHub فرانت‌متر `SKILL.md` را می‌خواند تا نام مهارت،
توضیح، الزامات، متغیرهای محیطی، و فراداده را بفهمد. فراداده دقیق
مهم است، چون به کاربران کمک می‌کند تصمیم بگیرند آیا مهارت را نصب کنند و
به اسکن‌های خودکار کمک می‌کند ناسازگاری‌های میان رفتار اعلام‌شده و مشاهده‌شده را تشخیص دهند.

ببینید [قالب مهارت](/fa/clawhub/skill-format).

## افزونه‌ها

افزونه‌ها اکستنشن‌های بسته‌بندی‌شده OpenClaw هستند. ClawHub فراداده بسته،
اطلاعات سازگاری، پیوندهای منبع، آرتیفکت‌ها، و رکوردهای نسخه را ذخیره می‌کند.

وقتی OpenClaw افزونه‌ای را از ClawHub نصب می‌کند، پیش از نصب، فراداده سازگاری
اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API،
حداقل نسخه Gateway، هدف‌های میزبان، الزامات محیطی، و چکیده‌های آرتیفکت باشند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار، یک رکورد نسخه تغییرناپذیر جدید ایجاد می‌کند. ناشران از CLI `clawhub`
برای گردش‌کارهای رجیستری احراز هویت‌شده استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

از اجرای آزمایشی برای پیش‌نمایش محموله حل‌شده پیش از بارگذاری استفاده کنید. سپس
صفحه‌های عمومی، فراداده منتشرشده، فایل‌ها، انتساب منبع، و وضعیت اسکن را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

فرمان‌های نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw فراداده منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بتوانند بعدا همان
بسته رجیستری را حل کنند. CLI ClawHub همچنین از گردش‌کارهای نصب و
به‌روزرسانی مستقیم مهارت برای کاربرانی پشتیبانی می‌کند که پوشه‌های مهارت مدیریت‌شده توسط رجیستری را بیرون از
یک فضای کاری کامل OpenClaw می‌خواهند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما انتشارها همچنان مشمول دروازه‌های بارگذاری،
بررسی‌های خودکار، گزارش‌های کاربران، و اقدام ناظر هستند.

صفحه‌های عمومی، در صورت وجود، خلاصه‌های اسکن را نشان می‌دهند. محتوایی که نگه داشته شده، پنهان شده،
یا مسدود شده است ممکن است از جست‌وجوی عمومی و جریان‌های نصب ناپدید شود، در حالی که
برای تشخیص عیب همچنان برای مالک قابل مشاهده می‌ماند.

ببینید [امنیت](/clawhub/security)، [ممیزی‌های امنیتی](/clawhub/security-audits)،
[نظارت و ایمنی حساب](/fa/clawhub/moderation)، و
[استفاده قابل قبول](/clawhub/acceptable-usage).

## دسترسی API

ClawHub APIهای خواندن عمومی را برای کشف، جست‌وجو، جزئیات بسته، و
دانلودها ارائه می‌کند. کاتالوگ‌های شخص ثالث می‌توانند از این APIها استفاده کنند، به شرط آنکه به
فهرست مرجع ClawHub پیوند دهند، محدودیت‌های نرخ را رعایت کنند، و از القای تأیید رسمی خودداری کنند.

ببینید [API عمومی](/clawhub/api) و [API HTTP](/clawhub/http-api).
