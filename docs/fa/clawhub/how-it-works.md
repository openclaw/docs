# ClawHub چگونه کار می‌کند

ClawHub لایهٔ رجیستری برای Skills و Pluginهای OpenClaw است. به کاربران
جایی برای کشف بسته‌ها می‌دهد، به ناشران جایی برای انتشار نسخه‌ها می‌دهد، و
فرادادهٔ کافی در اختیار OpenClaw می‌گذارد تا آن بسته‌ها را ایمن نصب و به‌روزرسانی کند.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است که شامل این موارد است:

- مالک و slug یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها، و انتساب منبع
- اطلاعات تغییرات و برچسب مانند `latest`
- سیگنال‌های دانلود، نصب، و ستاره
- وضعیت اسکن امنیتی و نظارت

صفحهٔ فهرست، مکان مرجع برای کاربران است تا پیش از نصب، بررسی کنند یک skill یا
plugin ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

یک skill یک بستهٔ متنی نسخه‌دار است که حول `SKILL.md` شکل گرفته است. می‌تواند
شامل فایل‌های پشتیبان، مثال‌ها، قالب‌ها، و اسکریپت‌ها باشد.

ClawHub فرانت‌متر `SKILL.md` را می‌خواند تا نام skill،
توضیح، نیازمندی‌ها، متغیرهای محیطی، و فراداده را بفهمد. فرادادهٔ دقیق
مهم است، چون به کاربران کمک می‌کند تصمیم بگیرند آیا skill را نصب کنند و
به اسکن‌های خودکار کمک می‌کند ناسازگاری‌های بین رفتار اعلام‌شده و مشاهده‌شده را تشخیص دهند.

[قالب skill](/fa/clawhub/skill-format) را ببینید.

## Pluginها

Pluginها افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته،
اطلاعات سازگاری، پیوندهای منبع، آرتیفکت‌ها، و رکوردهای نسخه را ذخیره می‌کند.

وقتی OpenClaw یک plugin را از ClawHub نصب می‌کند، پیش از نصب، فرادادهٔ
سازگاری اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API،
حداقل نسخهٔ gateway، هدف‌های میزبان، نیازمندی‌های محیطی، و digestهای آرتیفکت باشند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار یک رکورد نسخهٔ تغییرناپذیر جدید ایجاد می‌کند. ناشران از CLI `clawhub`
برای جریان‌های کاری رجیستری احرازشده استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

از اجرای آزمایشی برای پیش‌نمایش payload حل‌شده پیش از بارگذاری استفاده کنید. سپس صفحه‌های عمومی
فرادادهٔ منتشرشده، فایل‌ها، انتساب منبع، و وضعیت اسکن را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

دستورهای نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بتوانند بعداً همان
بستهٔ رجیستری را resolve کنند. CLI ClawHub همچنین برای کاربرانی که پوشه‌های skill
مدیریت‌شده توسط رجیستری را بیرون از یک workspace کامل OpenClaw می‌خواهند، از جریان‌های کاری نصب و
به‌روزرسانی مستقیم skill پشتیبانی می‌کند.

## وضعیت امنیت

ClawHub برای انتشار باز است، اما انتشارها همچنان مشمول gateهای بارگذاری،
بررسی‌های خودکار، گزارش‌های کاربران، و اقدام ناظر هستند.

صفحه‌های عمومی، در صورت موجود بودن، خلاصه‌های اسکن را نشان می‌دهند. محتوایی که نگه داشته، پنهان،
یا مسدود شده است ممکن است از جست‌وجوی عمومی و جریان‌های نصب ناپدید شود، درحالی‌که برای
تشخیص عیب همچنان برای مالک قابل مشاهده می‌ماند.

[امنیت](/fa/clawhub/security)، [ممیزی‌های امنیتی](/fa/clawhub/security-audits)،
[نظارت و ایمنی حساب](/fa/clawhub/moderation)، و
[استفادهٔ پذیرفتنی](/fa/clawhub/acceptable-usage) را ببینید.

## دسترسی API

ClawHub APIهای خواندنی عمومی را برای کشف، جست‌وجو، جزئیات بسته، و
دانلودها ارائه می‌کند. کاتالوگ‌های شخص ثالث می‌توانند از این APIها استفاده کنند، به شرطی که به
فهرست مرجع ClawHub پیوند بدهند، محدودیت‌های نرخ را رعایت کنند، و از القای تأیید ضمنی پرهیز کنند.

[API عمومی](/fa/clawhub/api) و [HTTP API](/fa/clawhub/http-api) را ببینید.
