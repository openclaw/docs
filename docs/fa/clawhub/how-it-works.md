# ClawHub چگونه کار می‌کند

ClawHub لایهٔ رجیستری برای Skills و Pluginهای OpenClaw است. به کاربران
جایی برای کشف بسته‌ها می‌دهد، به ناشران جایی برای انتشار نسخه‌ها می‌دهد، و
فرادادهٔ کافی در اختیار OpenClaw می‌گذارد تا آن بسته‌ها را به‌صورت ایمن نصب و به‌روزرسانی کند.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است با:

- مالک و اسلاگ یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها، و انتساب منبع
- اطلاعات changelog و برچسب‌هایی مانند `latest`
- سیگنال‌های دانلود، نصب، و ستاره
- وضعیت اسکن امنیتی و مدیریت محتوا

صفحهٔ فهرست، مکان مرجع برای کاربران است تا پیش از نصب، بررسی کنند که یک skill یا
plugin ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

یک skill یک بستهٔ متنی نسخه‌دار است که حول `SKILL.md` شکل گرفته است. می‌تواند
شامل فایل‌های پشتیبان، مثال‌ها، الگوها، و اسکریپت‌ها باشد.

ClawHub برای فهمیدن نام skill، توضیحات،
نیازمندی‌ها، متغیرهای محیطی، و فراداده، frontmatter فایل `SKILL.md` را می‌خواند. فرادادهٔ دقیق
مهم است، زیرا به کاربران کمک می‌کند تصمیم بگیرند آیا skill را نصب کنند یا نه، و
به اسکن‌های خودکار کمک می‌کند ناهماهنگی‌ها میان رفتار اعلام‌شده و مشاهده‌شده را تشخیص دهند.

ببینید [فرمت Skill](/fa/clawhub/skill-format).

## Pluginها

Pluginها افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته،
اطلاعات سازگاری، پیوندهای منبع، آرتیفکت‌ها، و رکوردهای نسخه را ذخیره می‌کند.

وقتی OpenClaw یک plugin را از ClawHub نصب می‌کند، پیش از نصب، فرادادهٔ سازگاری
اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API،
حداقل نسخهٔ gateway، اهداف میزبان، نیازمندی‌های محیطی، و digestهای آرتیفکت باشند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار، یک رکورد نسخهٔ تغییرناپذیر تازه ایجاد می‌کند. ناشران از CLI `clawhub`
برای گردش‌کارهای رجیستری احراز هویت‌شده استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

از اجرای آزمایشی برای پیش‌نمایش payload حل‌شده پیش از بارگذاری استفاده کنید. سپس صفحه‌های عمومی
فرادادهٔ منتشرشده، فایل‌ها، انتساب منبع، و وضعیت اسکن را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

فرمان‌های نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بعداً بتوانند همان
بستهٔ رجیستری را resolve کنند. CLI ClawHub همچنین از گردش‌کارهای نصب و
به‌روزرسانی مستقیم skill برای کاربرانی پشتیبانی می‌کند که می‌خواهند پوشه‌های skill مدیریت‌شده توسط رجیستری را بیرون از
یک فضای کاری کامل OpenClaw داشته باشند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما انتشارها همچنان مشمول gateهای بارگذاری،
بررسی‌های خودکار، گزارش‌های کاربر، و اقدام ناظر هستند.

صفحه‌های عمومی، وقتی موجود باشد، خلاصه‌های اسکن را نشان می‌دهند. محتوایی که نگه داشته شده، پنهان شده،
یا مسدود شده است ممکن است از جست‌وجوی عمومی و جریان‌های نصب ناپدید شود، در حالی که برای
مالک جهت عیب‌یابی قابل مشاهده می‌ماند.

ببینید [امنیت](/clawhub/security)، [ممیزی‌های امنیتی](/clawhub/security-audits)،
[مدیریت محتوا و ایمنی حساب](/fa/clawhub/moderation)، و
[استفادهٔ قابل قبول](/clawhub/acceptable-usage).

## دسترسی API

ClawHub APIهای خواندن عمومی را برای کشف، جست‌وجو، جزئیات بسته، و
دانلودها ارائه می‌کند. کاتالوگ‌های شخص ثالث می‌توانند از این APIها استفاده کنند، به‌شرطی که به
فهرست مرجع ClawHub پیوند دهند، محدودیت‌های نرخ را رعایت کنند، و از القای تأیید ضمنی پرهیز کنند.

ببینید [API عمومی](/fa/clawhub/api) و [API HTTP](/clawhub/http-api).
