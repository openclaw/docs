# ClawHub چگونه کار می‌کند

ClawHub لایهٔ رجیستری برای Skills و Pluginهای OpenClaw است. این لایه به کاربران
جایی برای کشف بسته‌ها، به ناشران جایی برای انتشار نسخه‌ها، و به
OpenClaw فرادادهٔ کافی می‌دهد تا آن بسته‌ها را با ایمنی نصب و به‌روزرسانی کند.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است با:

- مالک و نامک یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها و انتساب منبع
- اطلاعات تغییرات و برچسب مانند `latest`
- سیگنال‌های دانلود، نصب و ستاره
- وضعیت پویش امنیتی و نظارت

صفحهٔ فهرست، مکان مرجع برای کاربران است تا پیش از نصب بررسی کنند یک Skill یا
Plugin ادعا می‌کند چه کاری انجام می‌دهد.

## Skills

یک Skill یک بستهٔ متنی نسخه‌بندی‌شده است که حول `SKILL.md` شکل گرفته است. می‌تواند
شامل فایل‌های پشتیبان، نمونه‌ها، الگوها و اسکریپت‌ها باشد.

ClawHub فرانت‌متر `SKILL.md` را می‌خواند تا نام Skill،
توضیح، نیازمندی‌ها، متغیرهای محیطی و فراداده را بفهمد. فرادادهٔ دقیق
مهم است، چون به کاربران کمک می‌کند تصمیم بگیرند آیا Skill را نصب کنند و
به پویش‌های خودکار کمک می‌کند ناسازگاری‌های میان رفتار اعلام‌شده و مشاهده‌شده را تشخیص دهند.

[قالب Skill](/fa/clawhub/skill-format) را ببینید.

## Pluginها

Pluginها افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته،
اطلاعات سازگاری، پیوندهای منبع، مصنوعات و رکوردهای نسخه را ذخیره می‌کند.

وقتی OpenClaw یک Plugin را از ClawHub نصب می‌کند، پیش از نصب فرادادهٔ سازگاری
اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API،
حداقل نسخهٔ Gateway، اهداف میزبان، نیازمندی‌های محیطی و چکیده‌های مصنوع باشند.

وقتی می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار، یک رکورد نسخهٔ تغییرناپذیر جدید ایجاد می‌کند. ناشران از CLI `clawhub`
برای گردش‌کارهای رجیستری احرازشده استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

از اجرای آزمایشی برای پیش‌نمایش بار نهایی پیش از بارگذاری استفاده کنید. سپس صفحه‌های عمومی
فرادادهٔ منتشرشده، فایل‌ها، انتساب منبع و وضعیت پویش را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

فرمان‌های نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بعداً بتوانند همان
بستهٔ رجیستری را resolve کنند. CLI ClawHub همچنین برای کاربرانی که پوشه‌های Skill
مدیریت‌شده با رجیستری را بیرون از یک فضای کاری کامل OpenClaw می‌خواهند، از گردش‌کارهای نصب و
به‌روزرسانی مستقیم Skill پشتیبانی می‌کند.

## وضعیت امنیتی

ClawHub برای انتشار باز است، اما انتشارها همچنان مشمول دروازه‌های بارگذاری،
بررسی‌های خودکار، گزارش‌های کاربران و اقدام ناظر هستند.

صفحه‌های عمومی، در صورت موجود بودن، خلاصه‌های پویش را نشان می‌دهند. محتوایی که نگه‌داشته، پنهان
یا مسدود شده باشد ممکن است از جست‌وجوی عمومی و جریان‌های نصب ناپدید شود، درحالی‌که برای مالک
جهت عیب‌یابی قابل مشاهده می‌ماند.

[امنیت](/clawhub/security)، [ممیزی‌های امنیتی](/clawhub/security-audits)،
[نظارت و ایمنی حساب](/fa/clawhub/moderation)، و
[استفادهٔ قابل قبول](/clawhub/acceptable-usage) را ببینید.

## دسترسی API

ClawHub APIهای خواندن عمومی را برای کشف، جست‌وجو، جزئیات بسته و
دانلودها ارائه می‌کند. کاتالوگ‌های شخص ثالث می‌توانند از این APIها استفاده کنند، به شرطی که به
فهرست مرجع ClawHub پیوند دهند، محدودیت‌های نرخ را رعایت کنند و از القای تأیید ضمنی پرهیز کنند.

[API عمومی](/fa/clawhub/api) و [HTTP API](/clawhub/http-api) را ببینید.
