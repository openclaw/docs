# ClawHub چگونه کار می‌کند

ClawHub لایهٔ رجیستری برای Skills و Pluginهای OpenClaw است. این سامانه فضایی برای کاربران فراهم می‌کند تا بسته‌ها را کشف کنند، فضایی برای ناشران فراهم می‌کند تا نسخه‌ها را منتشر کنند و فرادادهٔ کافی در اختیار OpenClaw می‌گذارد تا آن بسته‌ها را با ایمنی نصب و به‌روزرسانی کند.

## رکوردهای رجیستری

هر فهرست عمومی یک رکورد رجیستری است که شامل موارد زیر می‌شود:

- یک مالک و نامک یا نام بسته
- یک یا چند نسخهٔ منتشرشده
- فراداده، خلاصه، فایل‌ها و انتساب منبع
- اطلاعات تغییرات و برچسب، مانند `latest`
- نشانه‌های دانلود، نصب و ستاره
- وضعیت اسکن امنیتی و نظارت

صفحهٔ فهرست، مرجع اصلی کاربران برای بررسی کارکرد ادعایی یک Skill یا Plugin پیش از نصب آن است.

## Skills

یک Skill بستهٔ متنی نسخه‌بندی‌شده‌ای است که حول `SKILL.md` شکل می‌گیرد. این بسته می‌تواند شامل فایل‌های پشتیبان، نمونه‌ها، الگوها و اسکریپت‌ها باشد.

ClawHub فرادادهٔ آغازین `SKILL.md` را می‌خواند تا نام، توضیحات، الزامات، متغیرهای محیطی و فرادادهٔ Skill را درک کند. دقت فراداده اهمیت دارد، زیرا به کاربران کمک می‌کند دربارهٔ نصب Skill تصمیم بگیرند و به اسکن‌های خودکار کمک می‌کند مغایرت‌های میان رفتار اعلام‌شده و مشاهده‌شده را تشخیص دهند.

به [قالب Skill](/fa/clawhub/skill-format) مراجعه کنید.

## Pluginها

Pluginها افزونه‌های بسته‌بندی‌شدهٔ OpenClaw هستند. ClawHub فرادادهٔ بسته، اطلاعات سازگاری، پیوندهای منبع، مصنوعات و رکوردهای نسخه را ذخیره می‌کند.

هنگامی که OpenClaw یک Plugin را از ClawHub نصب می‌کند، پیش از نصب، فرادادهٔ سازگاری اعلام‌شده را بررسی می‌کند. رکوردهای بسته می‌توانند شامل سازگاری API، حداقل نسخهٔ Gateway، میزبان‌های هدف، الزامات محیطی و چکیده‌های مصنوعات باشند.

هنگامی که می‌خواهید رجیستری منبع حقیقت باشد، از یک منبع نصب صریح ClawHub استفاده کنید:

```bash
openclaw plugins install clawhub:<package>
```

## انتشار

انتشار، یک رکورد نسخهٔ تغییرناپذیر جدید ایجاد می‌کند. ناشران برای گردش‌کارهای احرازهویت‌شدهٔ رجیستری از CLI ‏`clawhub` استفاده می‌کنند:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

از اجرای آزمایشی برای پیش‌نمایش محمولهٔ نهایی پیش از بارگذاری استفاده کنید. سپس صفحات عمومی، فراداده، فایل‌ها، انتساب منبع و وضعیت اسکن منتشرشده را نمایش می‌دهند.

## نصب‌ها و به‌روزرسانی‌ها

فرمان‌های نصب OpenClaw از ClawHub به‌عنوان منبع بسته استفاده می‌کنند:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw فرادادهٔ منبع نصب را ثبت می‌کند تا به‌روزرسانی‌ها بتوانند بعداً همان بستهٔ رجیستری را پیدا کنند. CLI ‏ClawHub همچنین از گردش‌کارهای نصب و به‌روزرسانی مستقیم Skill برای کاربرانی پشتیبانی می‌کند که پوشه‌های Skill مدیریت‌شده توسط رجیستری را خارج از یک فضای کاری کامل OpenClaw می‌خواهند.

## وضعیت امنیتی

ClawHub برای انتشار آزاد است، اما نسخه‌ها همچنان مشمول دروازه‌های بارگذاری، بررسی‌های خودکار، گزارش‌های کاربران و اقدامات ناظران هستند.

صفحات عمومی در صورت موجود بودن، خلاصه‌های اسکن را نمایش می‌دهند. محتوایی که معلق، پنهان یا مسدود شده است، ممکن است از جست‌وجوی عمومی و گردش‌کارهای نصب حذف شود، اما برای عیب‌یابی همچنان در دید مالک باقی بماند.

به [امنیت](/clawhub/security)، [ممیزی‌های امنیتی](/clawhub/security-audits)، [نظارت و ایمنی حساب](/fa/clawhub/moderation) و [استفادهٔ قابل‌قبول](/clawhub/acceptable-usage) مراجعه کنید.

## دسترسی API

ClawHub برای کشف، جست‌وجو، جزئیات بسته و دانلودها، APIهای خواندن عمومی ارائه می‌دهد. کاتالوگ‌های شخص ثالث می‌توانند هنگامی از این APIها استفاده کنند که به فهرست مرجع ClawHub پیوند دهند، محدودیت نرخ را رعایت کنند و از القای تأییدشدگی بپرهیزند.

به [API عمومی](/clawhub/api) و [HTTP API](/clawhub/http-api) مراجعه کنید.
