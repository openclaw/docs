# Telemetry

ClawHub از **Telemetry حداقلی** برای محاسبهٔ **تعداد نصب‌ها** (آنچه واقعاً در حال استفاده است) و برای بهبود مرتب‌سازی/فیلترکردن استفاده می‌کند.
این بر پایهٔ فرمان CLI به نام `clawhub sync` است.

## چه زمانی Telemetry جمع‌آوری می‌شود

Telemetry فقط زمانی ارسال می‌شود که:

- در CLI **وارد حساب شده باشید** (ما همین حالا هم برای جریان‌های sync/publish احراز هویت را لازم داریم).
- `clawhub sync` را اجرا کنید.
- Telemetry **غیرفعال نشده باشد** (بخش «چگونه غیرفعال کنیم» را در پایین ببینید).

اگر وارد حساب نشده باشید، هیچ چیزی گزارش نمی‌شود.

## چه چیزهایی جمع‌آوری می‌کنیم

در هر `clawhub sync`، CLI یک **نماگرفت کامل** از چیزهایی که پیدا کرده گزارش می‌دهد، که بر اساس ریشهٔ اسکن («پوشه/ریشه») گروه‌بندی شده‌اند.

برای هر ریشه ذخیره می‌کنیم:

- `rootId`: یک **هش SHA-256** از مسیر متعارف ریشه (سرور هرگز مسیر خام را نمی‌بیند).
- `label`: یک برچسب خوانا برای انسان که از دو بخش پایانی مسیر به دست می‌آید (مسیرهای خانه با `~` نشان داده می‌شوند).
- `firstSeenAt`، `lastSeenAt`، و `expiredAt` اختیاری.

برای هر Skill پیدا شده زیر یک ریشه ذخیره می‌کنیم:

- `skillId` (بر اساس slug حل می‌شود؛ فقط Skillsی که در registry وجود دارند ردیابی می‌شوند).
- `firstSeenAt`، `lastSeenAt`.
- `lastVersion` (در حد بهترین تلاش؛ در حال حاضر نسخهٔ منطبق با registry اگر شناخته شده باشد).
- `removedAt` اختیاری، وقتی نصبی که قبلاً گزارش شده از یک ریشه ناپدید شود.

### چه چیزهایی را _جمع‌آوری نمی‌کنیم_

- هیچ مسیر مطلق خام پوشه‌ای (فقط `rootId` هش‌شده + یک برچسب نمایشی کوتاه).
- هیچ محتوای فایلی.
- هیچ لاگ هر اجرا، prompt، یا خروجی دیگر CLI.
- هیچ ردیابی برای Skillsی که در registry بارگذاری نشده‌اند (slugهای ناشناخته نادیده گرفته می‌شوند).

## تعداد نصب‌ها

برای هر Skill دو شمارنده نگه می‌داریم:

- `installsCurrent`: کاربران یکتایی که در حال حاضر Skill را دست‌کم در یک ریشهٔ فعال نصب دارند.
- `installsAllTime`: کاربران یکتایی که تاکنون گزارش کرده‌اند Skill نصب است.

### چند ریشه

اگر از چند پوشه sync کنید، هر ریشهٔ اسکن را مستقل در نظر می‌گیریم. یک Skill زمانی «در حال حاضر نصب‌شده» است که در **هر** ریشهٔ فعالی وجود داشته باشد.

### تشخیص حذف نصب

چون `sync` مجموعهٔ کامل هر ریشه را گزارش می‌دهد:

- اگر یک Skill در sync بعدی از یک ریشه ناپدید شود، آن را برای آن ریشه حذف‌شده علامت‌گذاری می‌کنیم.
- اگر Skill از همهٔ ریشه‌های شما حذف شود، دیگر در `installsCurrent` حساب نمی‌شود.
- `installsAllTime` هرگز کاهش نمی‌یابد مگر اینکه Telemetry را حذف کنید (پایین را ببینید).

### کهنگی (۱۲۰ روز)

ریشه‌هایی که برای **۱۲۰ روز** Telemetry گزارش نکنند، کهنه علامت‌گذاری می‌شوند و نصب‌هایشان دیگر در `installsCurrent` حساب نمی‌شود.
این کار به‌صورت تنبل ارزیابی می‌شود (در گزارش بعدی Telemetry) تا از کارهای پس‌زمینه جلوگیری شود.

## شفافیت + کنترل‌های کاربر

ClawHub یک زبانهٔ خصوصی «نصب‌شده» در پروفایل خودتان ارائه می‌کند:

- ریشه‌های دقیق + Skills نصب‌شده‌ای را که ذخیره می‌کنیم نشان می‌دهد.
- شامل نمای **خروجی JSON** است.
- شامل کنش **حذف Telemetry** برای حذف همهٔ Telemetry ذخیره‌شده برای حساب شماست.

دیگران فقط **شمارنده‌های نصب تجمیع‌شده** را می‌بینند؛ هیچ‌کس دیگری نمی‌تواند ریشه‌ها/پوشه‌های شما را ببیند.

حذف حساب شما، داده‌های Telemetry شما را نیز حذف می‌کند.

## چگونه Telemetry را غیرفعال کنیم

متغیر محیطی را تنظیم کنید:

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

با تنظیم این مقدار، CLI هنگام `clawhub sync` Telemetry ارسال نخواهد کرد.
