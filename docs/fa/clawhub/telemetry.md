# تله‌متری

ClawHub از **تله‌متری حداقلی** برای محاسبهٔ **تعداد نصب‌ها** (آنچه واقعاً در حال استفاده است) و برای فراهم کردن مرتب‌سازی/فیلتر کردن بهتر استفاده می‌کند.
این بر اساس فرمان CLI یعنی `clawhub sync` است.

## چه زمانی تله‌متری جمع‌آوری می‌شود

تله‌متری فقط زمانی ارسال می‌شود که:

- شما در CLI **وارد شده باشید** (ما از قبل برای جریان‌های sync/publish احراز هویت را الزامی می‌کنیم).
- شما `clawhub sync` را اجرا کنید.
- تله‌متری **غیرفعال نشده باشد** (بخش «نحوهٔ غیرفعال کردن» را در ادامه ببینید).

اگر وارد نشده باشید، هیچ چیزی گزارش نمی‌شود.

## چه چیزی جمع‌آوری می‌کنیم

در هر `clawhub sync`، CLI یک **تصویر کامل** از آنچه پیدا کرده گزارش می‌کند، گروه‌بندی‌شده بر اساس ریشهٔ اسکن («پوشه/ریشه»).

برای هر ریشه ذخیره می‌کنیم:

- `rootId`: یک **هش SHA-256** از مسیر ریشهٔ canonical (سرور هرگز مسیر خام را نمی‌بیند).
- `label`: یک برچسب قابل خواندن برای انسان که از دو بخش آخر مسیر مشتق شده است (مسیرهای home با `~` نمایش داده می‌شوند).
- `firstSeenAt`، `lastSeenAt`، و `expiredAt` اختیاری.

برای هر skill که زیر یک ریشه پیدا شود ذخیره می‌کنیم:

- `skillId` (بر اساس slug resolve می‌شود؛ فقط skillsهایی که در رجیستری وجود دارند ردیابی می‌شوند).
- `firstSeenAt`، `lastSeenAt`.
- `lastVersion` (تا حد امکان؛ در حال حاضر نسخهٔ تطبیق‌یافته با رجیستری، اگر شناخته شده باشد).
- `removedAt` اختیاری، زمانی که نصبی که قبلاً گزارش شده از یک ریشه ناپدید شود.

### چه چیزی را جمع‌آوری _نمی‌کنیم_

- هیچ مسیر پوشهٔ absolute خامی جمع‌آوری نمی‌شود (فقط `rootId` هش‌شده + یک برچسب نمایشی کوتاه).
- هیچ محتوای فایلی جمع‌آوری نمی‌شود.
- هیچ گزارش هر اجرا، prompt، یا خروجی دیگر CLI جمع‌آوری نمی‌شود.
- هیچ ردیابی برای skillsهایی که در رجیستری بارگذاری نشده‌اند انجام نمی‌شود (slugهای ناشناخته نادیده گرفته می‌شوند).

## تعداد نصب‌ها

برای هر skill دو شمارنده نگه می‌داریم:

- `installsCurrent`: کاربران یکتایی که در حال حاضر skill را در دست‌کم یک ریشهٔ فعال نصب دارند.
- `installsAllTime`: کاربران یکتایی که تا به حال نصب بودن skill را گزارش کرده‌اند.

### چندین ریشه

اگر از چند پوشه sync کنید، هر ریشهٔ اسکن را مستقل در نظر می‌گیریم. یک skill زمانی «در حال حاضر نصب‌شده» است که در **هر** ریشهٔ فعالی وجود داشته باشد.

### تشخیص حذف نصب

از آنجا که `sync` مجموعهٔ کامل را برای هر ریشه گزارش می‌کند:

- اگر یک skill در sync بعدی از یک ریشه ناپدید شود، آن را برای آن ریشه حذف‌شده علامت‌گذاری می‌کنیم.
- اگر skill از همهٔ ریشه‌های شما حذف شود، دیگر در `installsCurrent` شمرده نمی‌شود.
- `installsAllTime` هرگز کاهش نمی‌یابد، مگر اینکه تله‌متری را حذف کنید (در ادامه ببینید).

### کهنگی (۱۲۰ روز)

ریشه‌هایی که به مدت **۱۲۰ روز** تله‌متری گزارش نکنند، کهنه علامت‌گذاری می‌شوند و نصب‌های آن‌ها دیگر در `installsCurrent` شمرده نمی‌شود.
این ارزیابی به‌صورت lazy انجام می‌شود (در گزارش تله‌متری بعدی) تا از کارهای پس‌زمینه جلوگیری شود.

## شفافیت + کنترل‌های کاربر

ClawHub یک زبانهٔ خصوصی «نصب‌شده» در پروفایل خودتان ارائه می‌کند:

- ریشه‌ها + skills نصب‌شده‌ای را که ذخیره می‌کنیم دقیقاً نشان می‌دهد.
- شامل نمای **خروجی JSON** است.
- شامل یک اقدام **حذف تله‌متری** برای حذف همهٔ تله‌متری ذخیره‌شده برای حساب شما است.

دیگران فقط **شمارنده‌های نصب تجمیع‌شده** را می‌بینند؛ هیچ‌کس دیگری نمی‌تواند ریشه‌ها/پوشه‌های شما را ببیند.

حذف حساب شما، داده‌های تله‌متری شما را نیز حذف می‌کند.

## نحوهٔ غیرفعال کردن تله‌متری

متغیر محیطی را تنظیم کنید:

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

با تنظیم این مورد، CLI هنگام `clawhub sync` تله‌متری ارسال نخواهد کرد.
