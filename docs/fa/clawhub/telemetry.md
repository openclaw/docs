# تله‌متری

ClawHub از **تله‌متری حداقلی** برای محاسبهٔ **تعداد نصب‌ها** (آنچه واقعاً در حال استفاده است) و فراهم‌کردن مرتب‌سازی/فیلترگذاری بهتر استفاده می‌کند.
این کار بر پایهٔ فرمان `clawhub sync` در CLI انجام می‌شود.

## چه زمانی تله‌متری جمع‌آوری می‌شود

تله‌متری فقط زمانی ارسال می‌شود که:

- در CLI **وارد حساب شده باشید** (ما همین حالا هم برای جریان‌های sync/publish احراز هویت را لازم داریم).
- `clawhub sync` را اجرا کنید.
- تله‌متری **غیرفعال نشده باشد** (بخش «نحوهٔ غیرفعال‌کردن» را در پایین ببینید).

اگر وارد حساب نشده باشید، چیزی گزارش نمی‌شود.

## چه چیزهایی را جمع‌آوری می‌کنیم

در هر `clawhub sync`، CLI یک **نمایهٔ کامل** از آنچه پیدا کرده گزارش می‌کند که بر اساس ریشهٔ اسکن («پوشه/ریشه») گروه‌بندی شده است.

برای هر ریشه این موارد را ذخیره می‌کنیم:

- `rootId`: یک **هش SHA-256** از مسیر متعارف ریشه (سرور هرگز مسیر خام را نمی‌بیند).
- `label`: برچسبی خوانا برای انسان که از دو بخش آخر مسیر به‌دست می‌آید (مسیرهای خانه با `~` نمایش داده می‌شوند).
- `firstSeenAt`، `lastSeenAt`، و `expiredAt` اختیاری.

برای هر Skill که زیر یک ریشه پیدا شود، این موارد را ذخیره می‌کنیم:

- `skillId` (بر اساس slug حل می‌شود؛ فقط Skillsهایی که در رجیستری وجود دارند رهگیری می‌شوند).
- `firstSeenAt`، `lastSeenAt`.
- `lastVersion` (با بهترین تلاش؛ در حال حاضر نسخهٔ منطبق با رجیستری، اگر شناخته‌شده باشد).
- `removedAt` اختیاری، زمانی که یک نصبِ قبلاً گزارش‌شده از یک ریشه ناپدید شود.

### چه چیزهایی را جمع‌آوری _نمی‌کنیم_

- هیچ مسیر مطلق خامی برای پوشه‌ها (فقط `rootId` هش‌شده + یک برچسب نمایش کوتاه).
- هیچ محتوای فایلی.
- هیچ لاگِ هر اجرا، پرامپت، یا خروجی دیگر CLI.
- هیچ رهگیری برای Skillsهایی که در رجیستری بارگذاری نشده‌اند (slugهای ناشناخته نادیده گرفته می‌شوند).

## تعداد نصب‌ها

برای هر Skill دو شمارنده نگه می‌داریم:

- `installsCurrent`: کاربران یکتایی که در حال حاضر Skill را در دست‌کم یک ریشهٔ فعال نصب کرده‌اند.
- `installsAllTime`: کاربران یکتایی که تاکنون نصب بودن Skill را گزارش کرده‌اند.

### چند ریشه

اگر از چند پوشه sync کنید، هر ریشهٔ اسکن را مستقل در نظر می‌گیریم. یک Skill زمانی «در حال حاضر نصب‌شده» است که در **هر** ریشهٔ فعالی وجود داشته باشد.

### تشخیص حذف نصب

از آنجا که `sync` مجموعهٔ کامل هر ریشه را گزارش می‌کند:

- اگر یک Skill در sync بعدی از یک ریشه ناپدید شود، آن را برای آن ریشه حذف‌شده علامت‌گذاری می‌کنیم.
- اگر Skill از همهٔ ریشه‌های شما حذف شود، دیگر در `installsCurrent` حساب نمی‌شود.
- `installsAllTime` هرگز کاهش نمی‌یابد، مگر اینکه تله‌متری را حذف کنید (پایین را ببینید).

### کهنگی (۱۲۰ روز)

ریشه‌هایی که به مدت **۱۲۰ روز** تله‌متری گزارش نکنند، کهنه علامت‌گذاری می‌شوند و نصب‌هایشان دیگر در `installsCurrent` حساب نمی‌شود.
این ارزیابی به‌صورت تنبل انجام می‌شود (در گزارش تله‌متری بعدی) تا از jobهای پس‌زمینه اجتناب شود.

## شفافیت + کنترل‌های کاربر

ClawHub یک تب خصوصی «نصب‌شده‌ها» در پروفایل خودتان ارائه می‌کند:

- ریشه‌های دقیق + Skills نصب‌شده‌ای را که ذخیره می‌کنیم نشان می‌دهد.
- شامل نمای **خروجی JSON** است.
- شامل کنش **حذف تله‌متری** برای حذف همهٔ تله‌متری ذخیره‌شده برای حساب شما است.

دیگران فقط **شمارنده‌های تجمیعی نصب** را می‌بینند؛ هیچ‌کس دیگر نمی‌تواند ریشه‌ها/پوشه‌های شما را ببیند.

حذف حساب شما داده‌های تله‌متری شما را نیز حذف می‌کند.

## نحوهٔ غیرفعال‌کردن تله‌متری

متغیر محیطی را تنظیم کنید:

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

با تنظیم این مقدار، CLI هنگام `clawhub sync` تله‌متری ارسال نخواهد کرد.
