# تله‌متری

ClawHub از **تله‌متری حداقلی** برای محاسبه **شمار نصب‌ها** (آنچه واقعاً در حال استفاده است) و برای بهبود مرتب‌سازی/فیلترکردن استفاده می‌کند.
این بر پایه فرمان CLI به نام `clawhub sync` است.

## تله‌متری چه زمانی جمع‌آوری می‌شود

تله‌متری فقط زمانی ارسال می‌شود که:

- در CLI **وارد شده باشید** (ما از قبل برای جریان‌های sync/publish احراز هویت را لازم داریم).
- `clawhub sync` را اجرا کنید.
- تله‌متری **غیرفعال نشده باشد** (بخش «روش غیرفعال‌سازی» را در پایین ببینید).

اگر وارد نشده باشید، هیچ چیزی گزارش نمی‌شود.

## چه چیزهایی جمع‌آوری می‌کنیم

در هر اجرای `clawhub sync`، CLI یک **نمای کامل** از چیزهایی که پیدا کرده گزارش می‌کند، که بر اساس ریشه اسکن («پوشه/ریشه») گروه‌بندی شده است.

برای هر ریشه، این موارد را ذخیره می‌کنیم:

- `rootId`: یک **هش SHA-256** از مسیر ریشه canonical (سرور هرگز مسیر خام را نمی‌بیند).
- `label`: برچسبی خوانا برای انسان که از دو بخش پایانی مسیر ساخته می‌شود (مسیرهای خانه با `~` نمایش داده می‌شوند).
- `firstSeenAt`، `lastSeenAt`، و `expiredAt` اختیاری.

برای هر skill پیدا شده زیر یک ریشه، این موارد را ذخیره می‌کنیم:

- `skillId` (با slug حل می‌شود؛ فقط skillهایی که در رجیستری وجود دارند ردیابی می‌شوند).
- `firstSeenAt`، `lastSeenAt`.
- `lastVersion` (با بهترین تلاش؛ در حال حاضر نسخه منطبق با رجیستری، اگر شناخته‌شده باشد).
- `removedAt` اختیاری، وقتی نصبی که قبلاً گزارش شده بود از یک ریشه ناپدید شود.

### چه چیزهایی را جمع‌آوری _نمی‌کنیم_

- هیچ مسیر خام و مطلق پوشه‌ای (فقط `rootId` هش‌شده + یک برچسب نمایشی کوتاه).
- هیچ محتوای فایلی.
- هیچ لاگ هر اجرا، پرامپت، یا خروجی دیگر CLI.
- هیچ ردیابی برای skillهایی که در رجیستری بارگذاری نشده‌اند (slugهای ناشناخته نادیده گرفته می‌شوند).

## شمار نصب‌ها

ما برای هر skill دو شمارنده نگه می‌داریم:

- `installsCurrent`: کاربران یکتایی که در حال حاضر skill را در دست‌کم یک ریشه فعال نصب‌شده دارند.
- `installsAllTime`: کاربران یکتایی که تاکنون نصب بودن skill را گزارش کرده‌اند.

### چندین ریشه

اگر از چند پوشه sync کنید، هر ریشه اسکن را مستقل در نظر می‌گیریم. یک skill زمانی «در حال حاضر نصب‌شده» است که در **هر** ریشه فعال وجود داشته باشد.

### تشخیص حذف نصب

چون `sync` مجموعه کامل را برای هر ریشه گزارش می‌کند:

- اگر در sync بعدی یک skill از یک ریشه ناپدید شود، آن را برای همان ریشه حذف‌شده علامت‌گذاری می‌کنیم.
- اگر skill از همه ریشه‌های شما حذف شود، دیگر در `installsCurrent` شمرده نمی‌شود.
- `installsAllTime` هرگز کاهش نمی‌یابد، مگر اینکه تله‌متری را حذف کنید (پایین را ببینید).

### کهنگی (۱۲۰ روز)

ریشه‌هایی که به مدت **۱۲۰ روز** تله‌متری گزارش نکنند، stale علامت‌گذاری می‌شوند و نصب‌های آن‌ها دیگر در `installsCurrent` شمرده نمی‌شود.
این مورد به‌صورت تنبل ارزیابی می‌شود (در گزارش تله‌متری بعدی) تا از کارهای پس‌زمینه جلوگیری شود.

## شفافیت + کنترل‌های کاربر

ClawHub یک زبانه خصوصی «نصب‌شده» در پروفایل خودتان ارائه می‌کند:

- ریشه‌های دقیق + skillهای نصب‌شده‌ای را که ذخیره می‌کنیم نشان می‌دهد.
- شامل نمای **خروجی JSON** است.
- شامل یک کنش **حذف تله‌متری** برای حذف همه تله‌متری ذخیره‌شده حساب شما است.

دیگران فقط **شمارنده‌های تجمیعی نصب** را می‌بینند؛ هیچ‌کس دیگری نمی‌تواند ریشه‌ها/پوشه‌های شما را ببیند.

حذف حساب شما، داده‌های تله‌متری شما را نیز حذف می‌کند.

## روش غیرفعال‌سازی تله‌متری

متغیر محیطی را تنظیم کنید:

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

با تنظیم این مقدار، CLI هنگام اجرای `clawhub sync` تله‌متری ارسال نمی‌کند.
