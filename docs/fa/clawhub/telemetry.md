# Telemetry

ClawHub از **حداقل Telemetry** برای محاسبه‌ی **تعداد نصب‌ها** (آنچه واقعاً در حال استفاده است) و برای فراهم کردن مرتب‌سازی/فیلترگذاری بهتر استفاده می‌کند.
این بر پایه‌ی دستور CLI یعنی `clawhub sync` است.

## زمان جمع‌آوری Telemetry

Telemetry فقط زمانی ارسال می‌شود که:

- شما در CLI **وارد شده باشید** (ما از قبل برای جریان‌های sync/publish نیاز به احراز هویت داریم).
- شما `clawhub sync` را اجرا کنید.
- Telemetry **غیرفعال نشده باشد** (بخش «نحوه‌ی غیرفعال‌سازی» را در پایین ببینید).

اگر وارد نشده باشید، هیچ چیزی گزارش نمی‌شود.

## چه چیزی را جمع‌آوری می‌کنیم

در هر `clawhub sync`، CLI یک **اسنپ‌شات کامل** از آنچه پیدا کرده است گزارش می‌دهد، که بر اساس ریشه‌ی اسکن («پوشه/ریشه») گروه‌بندی می‌شود.

برای هر ریشه ذخیره می‌کنیم:

- `rootId`: یک **هش SHA-256** از مسیر ریشه‌ی کانونیکال (سرور هرگز مسیر خام را نمی‌بیند).
- `label`: یک برچسب خوانا برای انسان که از دو بخش پایانی مسیر مشتق شده است (مسیرهای home با `~` نمایش داده می‌شوند).
- `firstSeenAt`، `lastSeenAt`، و `expiredAt` اختیاری.

برای هر skill پیدا‌شده زیر یک ریشه ذخیره می‌کنیم:

- `skillId` (بر اساس slug حل می‌شود؛ فقط skillهایی که در registry وجود دارند ردیابی می‌شوند).
- `firstSeenAt`، `lastSeenAt`.
- `lastVersion` (با بهترین تلاش؛ در حال حاضر نسخه‌ی منطبق با registry در صورت شناخته بودن).
- `removedAt` اختیاری، زمانی که یک نصبِ قبلاً گزارش‌شده از یک ریشه ناپدید شود.

### چه چیزی را _جمع‌آوری نمی‌کنیم_

- هیچ مسیر پوشه‌ی مطلق خامی وجود ندارد (فقط `rootId` هش‌شده + یک برچسب نمایشی کوتاه).
- هیچ محتوای فایلی وجود ندارد.
- هیچ لاگ هر اجرا، پرامپت، یا خروجی دیگر CLI وجود ندارد.
- هیچ ردیابی برای skillهایی که در registry بارگذاری نشده‌اند وجود ندارد (slugهای ناشناخته نادیده گرفته می‌شوند).

## تعداد نصب‌ها

برای هر skill دو شمارنده نگه می‌داریم:

- `installsCurrent`: کاربران یکتایی که در حال حاضر skill را حداقل در یک ریشه‌ی فعال نصب‌شده دارند.
- `installsAllTime`: کاربران یکتایی که تاکنون نصب بودن skill را گزارش کرده‌اند.

### چند ریشه

اگر از چند پوشه sync کنید، هر ریشه‌ی اسکن را مستقل در نظر می‌گیریم. یک skill اگر در **هر** ریشه‌ی فعال وجود داشته باشد، «در حال حاضر نصب‌شده» محسوب می‌شود.

### تشخیص حذف نصب

چون `sync` مجموعه‌ی کامل را برای هر ریشه گزارش می‌دهد:

- اگر یک skill در sync بعدی از یک ریشه ناپدید شود، آن را برای آن ریشه حذف‌شده علامت‌گذاری می‌کنیم.
- اگر skill از همه‌ی ریشه‌های شما حذف شود، دیگر در `installsCurrent` حساب نمی‌شود.
- `installsAllTime` هرگز کاهش نمی‌یابد، مگر اینکه Telemetry را حذف کنید (پایین را ببینید).

### کهنگی (۱۲۰ روز)

ریشه‌هایی که به مدت **۱۲۰ روز** Telemetry گزارش نکنند، کهنه علامت‌گذاری می‌شوند و نصب‌های آن‌ها دیگر در `installsCurrent` حساب نمی‌شود.
این به‌صورت تنبل ارزیابی می‌شود (در گزارش Telemetry بعدی) تا از کارهای پس‌زمینه جلوگیری شود.

## شفافیت + کنترل‌های کاربر

ClawHub یک زبانه‌ی خصوصی «نصب‌شده» در پروفایل خودتان فراهم می‌کند:

- ریشه‌ها + skillهای نصب‌شده‌ی دقیقی را که ذخیره می‌کنیم نشان می‌دهد.
- شامل نمای **خروجی JSON** است.
- شامل کنش **حذف Telemetry** برای حذف همه‌ی Telemetry ذخیره‌شده برای حساب شماست.

دیگران فقط **شمارنده‌های تجمیعی نصب** را می‌بینند؛ هیچ‌کس دیگری نمی‌تواند ریشه‌ها/پوشه‌های شما را ببیند.

حذف حساب شما، داده‌های Telemetry شما را نیز حذف می‌کند.

## نحوه‌ی غیرفعال‌سازی Telemetry

متغیر محیطی را تنظیم کنید:

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

با تنظیم این مورد، CLI هنگام `clawhub sync`، Telemetry ارسال نخواهد کرد.
