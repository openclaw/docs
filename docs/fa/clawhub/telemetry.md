# تله‌متری

ClawHub از **تله‌متری حداقلی** برای محاسبهٔ **تعداد نصب‌ها** (آنچه واقعاً در حال استفاده است) و برای بهبود مرتب‌سازی/فیلتر کردن استفاده می‌کند.
این بر پایهٔ فرمان CLI یعنی `clawhub sync` است.

## چه زمانی تله‌متری جمع‌آوری می‌شود

تله‌متری فقط زمانی ارسال می‌شود که:

- در CLI **وارد حساب شده باشید** (ما از قبل برای جریان‌های همگام‌سازی/انتشار احراز هویت را الزامی می‌کنیم).
- `clawhub sync` را اجرا کنید.
- تله‌متری **غیرفعال نشده باشد** (بخش «نحوهٔ غیرفعال کردن» را در ادامه ببینید).

اگر وارد حساب نشده باشید، چیزی گزارش نمی‌شود.

## چه چیزهایی را جمع‌آوری می‌کنیم

در هر `clawhub sync`، CLI یک **عکس فوری کامل** از آنچه پیدا کرده گزارش می‌دهد که بر اساس ریشهٔ اسکن («پوشه/ریشه») گروه‌بندی شده است.

برای هر ریشه این موارد را ذخیره می‌کنیم:

- `rootId`: یک **هش SHA-256** از مسیر متعارف ریشه (سرور هرگز مسیر خام را نمی‌بیند).
- `label`: یک برچسب خوانا برای انسان که از دو بخش پایانی مسیر مشتق می‌شود (مسیرهای خانه با `~` نمایش داده می‌شوند).
- `firstSeenAt`، `lastSeenAt`، و `expiredAt` اختیاری.

برای هر Skill که زیر یک ریشه پیدا شود، این موارد را ذخیره می‌کنیم:

- `skillId` (بر اساس slug حل می‌شود؛ فقط Skillهایی که در رجیستری وجود دارند ردیابی می‌شوند).
- `firstSeenAt`، `lastSeenAt`.
- `lastVersion` (در حد بهترین تلاش؛ در حال حاضر نسخهٔ مطابق با رجیستری، اگر شناخته‌شده باشد).
- `removedAt` اختیاری، وقتی نصبی که قبلاً گزارش شده بود از یک ریشه ناپدید شود.

### چه چیزهایی را جمع‌آوری _نمی‌کنیم_

- هیچ مسیر خام مطلق پوشه‌ای (فقط `rootId` هش‌شده + یک برچسب کوتاه نمایشی).
- هیچ محتوای فایلی.
- هیچ لاگ هر اجرا، پرامپت، یا خروجی دیگر CLI.
- هیچ ردیابی برای Skillهایی که در رجیستری آپلود نشده‌اند (slugهای ناشناخته نادیده گرفته می‌شوند).

## تعداد نصب‌ها

برای هر Skill دو شمارنده نگه می‌داریم:

- `installsCurrent`: کاربران یکتایی که در حال حاضر Skill را در دست‌کم یک ریشهٔ فعال نصب‌شده دارند.
- `installsAllTime`: کاربران یکتایی که تا به حال نصب بودن Skill را گزارش کرده‌اند.

### چندین ریشه

اگر از چند پوشه همگام‌سازی کنید، هر ریشهٔ اسکن را مستقل در نظر می‌گیریم. یک Skill زمانی «در حال حاضر نصب‌شده» است که در **هرکدام** از ریشه‌های فعال وجود داشته باشد.

### تشخیص حذف نصب

چون `sync` مجموعهٔ کامل هر ریشه را گزارش می‌دهد:

- اگر یک Skill در همگام‌سازی بعدی از یک ریشه ناپدید شود، آن را برای آن ریشه حذف‌شده علامت‌گذاری می‌کنیم.
- اگر Skill از همهٔ ریشه‌های شما حذف شود، دیگر در `installsCurrent` شمرده نمی‌شود.
- `installsAllTime` هرگز کاهش نمی‌یابد، مگر اینکه تله‌متری را حذف کنید (در ادامه ببینید).

### کهنگی (۱۲۰ روز)

ریشه‌هایی که برای **۱۲۰ روز** تله‌متری گزارش نکنند کهنه علامت‌گذاری می‌شوند و نصب‌هایشان دیگر در `installsCurrent` شمرده نمی‌شود.
این به‌صورت تنبل ارزیابی می‌شود (در گزارش تله‌متری بعدی) تا از کارهای پس‌زمینه جلوگیری شود.

## شفافیت + کنترل‌های کاربر

ClawHub یک زبانهٔ خصوصی «نصب‌شده» در پروفایل خودتان ارائه می‌دهد:

- ریشه‌های دقیق + Skillهای نصب‌شده‌ای را که ذخیره می‌کنیم نشان می‌دهد.
- یک نمای **خروجی JSON** دارد.
- یک اقدام **حذف تله‌متری** برای حذف همهٔ تله‌متری ذخیره‌شدهٔ حساب شما دارد.

دیگران فقط **شمارنده‌های تجمیعی نصب** را می‌بینند؛ هیچ‌کس جز شما نمی‌تواند ریشه‌ها/پوشه‌هایتان را ببیند.

حذف حساب شما داده‌های تله‌متری شما را نیز حذف می‌کند.

## نحوهٔ غیرفعال کردن تله‌متری

متغیر محیطی را تنظیم کنید:

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

با تنظیم این مقدار، CLI هنگام `clawhub sync` تله‌متری ارسال نخواهد کرد.
