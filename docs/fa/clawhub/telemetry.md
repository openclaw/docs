# تله‌متری

ClawHub از **تله‌متری حداقلی** برای محاسبهٔ **شمار نصب‌ها** (چیزی که واقعاً در حال استفاده است) و برای بهتر کردن مرتب‌سازی/فیلترکردن استفاده می‌کند.
این بر پایهٔ فرمان CLI یعنی `clawhub sync` است.

## چه زمانی تله‌متری جمع‌آوری می‌شود

تله‌متری فقط زمانی ارسال می‌شود که:

- شما در CLI **وارد شده باشید** (ما از قبل برای جریان‌های همگام‌سازی/انتشار احراز هویت را لازم می‌دانیم).
- `clawhub sync` را اجرا کنید.
- تله‌متری **غیرفعال نشده باشد** (در ادامه، «نحوهٔ غیرفعال‌سازی» را ببینید).

اگر وارد نشده باشید، هیچ چیزی گزارش نمی‌شود.

## چه چیزهایی جمع‌آوری می‌کنیم

در هر اجرای `clawhub sync`، CLI یک **نمای کامل** از آنچه پیدا کرده گزارش می‌دهد، گروه‌بندی‌شده بر اساس ریشهٔ اسکن («پوشه/ریشه»).

برای هر ریشه ذخیره می‌کنیم:

- `rootId`: یک **هش SHA-256** از مسیر ریشهٔ کانونی (سرور هرگز مسیر خام را نمی‌بیند).
- `label`: یک برچسب خوانا برای انسان که از دو بخش پایانی مسیر ساخته می‌شود (مسیرهای خانه با `~` نشان داده می‌شوند).
- `firstSeenAt`، `lastSeenAt`، و `expiredAt` اختیاری.

برای هر Skill که زیر یک ریشه پیدا شود ذخیره می‌کنیم:

- `skillId` (بر اساس اسلاگ حل می‌شود؛ فقط Skills موجود در رجیستری ردیابی می‌شوند).
- `firstSeenAt`، `lastSeenAt`.
- `lastVersion` (در حد بهترین تلاش؛ در حال حاضر نسخهٔ منطبق با رجیستری، اگر معلوم باشد).
- `removedAt` اختیاری، وقتی نصبی که قبلاً گزارش شده بود از یک ریشه ناپدید شود.

### چه چیزهایی را جمع‌آوری _نمی‌کنیم_

- هیچ مسیر پوشهٔ مطلق خامی وجود ندارد (فقط `rootId` هش‌شده + یک برچسب نمایشی کوتاه).
- هیچ محتوای فایلی وجود ندارد.
- هیچ لاگ اجرای جداگانه، پرامپت، یا خروجی دیگر CLI وجود ندارد.
- هیچ ردیابی‌ای برای Skills که در رجیستری بارگذاری نشده‌اند انجام نمی‌شود (اسلاگ‌های ناشناخته نادیده گرفته می‌شوند).

## شمار نصب‌ها

برای هر Skill دو شمارنده نگه می‌داریم:

- `installsCurrent`: کاربران یکتایی که در حال حاضر Skill را در دست‌کم یک ریشهٔ فعال نصب دارند.
- `installsAllTime`: کاربران یکتایی که تاکنون نصب بودن Skill را گزارش کرده‌اند.

### چند ریشه

اگر از چند پوشه همگام‌سازی کنید، هر ریشهٔ اسکن را مستقل در نظر می‌گیریم. یک Skill زمانی «در حال حاضر نصب‌شده» است که در **هرکدام** از ریشه‌های فعال وجود داشته باشد.

### تشخیص حذف نصب

چون `sync` مجموعهٔ کامل هر ریشه را گزارش می‌کند:

- اگر یک Skill در همگام‌سازی بعدی از یک ریشه ناپدید شود، آن را برای آن ریشه حذف‌شده علامت‌گذاری می‌کنیم.
- اگر Skill از همهٔ ریشه‌های شما حذف شود، دیگر در `installsCurrent` شمرده نمی‌شود.
- `installsAllTime` هرگز کاهش پیدا نمی‌کند، مگر اینکه تله‌متری را حذف کنید (پایین را ببینید).

### کهنگی (۱۲۰ روز)

ریشه‌هایی که به مدت **۱۲۰ روز** تله‌متری گزارش نکنند کهنه علامت‌گذاری می‌شوند و نصب‌های آن‌ها دیگر در `installsCurrent` شمرده نمی‌شود.
این ارزیابی به‌صورت تنبل انجام می‌شود (در گزارش تله‌متری بعدی) تا از کارهای پس‌زمینه جلوگیری شود.

## شفافیت + کنترل‌های کاربر

ClawHub در نمایهٔ خودتان یک زبانهٔ خصوصی «نصب‌شده» ارائه می‌دهد:

- ریشه‌های دقیق + Skills نصب‌شده‌ای را که ذخیره می‌کنیم نشان می‌دهد.
- یک نمای **خروجی JSON** دارد.
- یک اقدام **حذف تله‌متری** برای حذف همهٔ تله‌متری ذخیره‌شده برای حساب شما دارد.

دیگران فقط **شمارنده‌های نصب تجمیع‌شده** را می‌بینند؛ هیچ‌کس دیگر نمی‌تواند ریشه‌ها/پوشه‌های شما را ببیند.

حذف حساب شما داده‌های تله‌متری شما را نیز حذف می‌کند.

## نحوهٔ غیرفعال‌سازی تله‌متری

متغیر محیطی را تنظیم کنید:

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

با تنظیم این مقدار، CLI هنگام `clawhub sync` تله‌متری ارسال نمی‌کند.
