# احراز هویت

ClawHub برای ورود به وب از GitHub استفاده می‌کند. CLI از توکن‌های API متعلق به ClawHub استفاده می‌کند که
از طریق حساب واردشده ایجاد می‌شوند.

## ورود به وب

برای ورود در [clawhub.ai](https://clawhub.ai) از GitHub استفاده کنید.

حساب‌های حذف‌شده، مسدودشده یا غیرفعال نمی‌توانند فرایند عادی ورود به ClawHub را تکمیل کنند.
اگر پس از ورود دوباره به وضعیت خارج‌شده از حساب بازگردانده شدید، ممکن است حساب شما در
وضعیت مناسبی نباشد. اگر حساب شما مسدود یا غیرفعال شده است و تصور می‌کنید این
یک اشتباه است، از [فرم اعتراض ClawHub](https://appeals.openclaw.ai/) استفاده کنید.

## ورود به CLI

جریان پیش‌فرض ورود به CLI مرورگر را باز می‌کند:

```bash
clawhub login
clawhub whoami
```

آنچه رخ می‌دهد:

1. CLI یک سرور موقت بازگشت فراخوانی را روی `127.0.0.1` راه‌اندازی می‌کند.
2. مرورگر صفحه ورود ClawHub را باز می‌کند.
3. پس از ورود با GitHub، ‏ClawHub یک توکن API ایجاد می‌کند.
4. مرورگر به بازگشت فراخوانی محلی هدایت می‌شود.
5. CLI توکن را در فایل پیکربندی ClawHub ذخیره می‌کند.

اگر مرورگر به‌دلیل قوانین دیوار آتش، VPN یا پروکسی نتواند به بازگشت فراخوانی محلی دسترسی پیدا کند،
از جریان توکن بدون رابط گرافیکی استفاده کنید.

## ورود بدون رابط گرافیکی

در رابط وب ClawHub یک توکن ایجاد کنید، سپس آن را به CLI بدهید:

```bash
clawhub login --token clh_...
```

از این جریان برای سرورها، کارهای CI یا محیط‌های صرفاً مبتنی بر ترمینال استفاده کنید.

برای پوسته‌های راه دور که می‌توانید مرورگر را در جای دیگری باز کنید، اجرا کنید:

```bash
clawhub login --device
```

CLI یک کد یک‌بارمصرف نمایش می‌دهد و تا زمانی که آن را در
`https://clawhub.ai/cli/device` مجاز کنید، منتظر می‌ماند.

## ذخیره‌سازی توکن

مسیرهای پیش‌فرض پیکربندی:

- macOS: `~/Library/Application Support/clawhub/config.json`
- Linux/XDG: `$XDG_CONFIG_HOME/clawhub/config.json` یا `~/.config/clawhub/config.json`
- Windows: `%APPDATA%\\clawhub\\config.json`

مسیر را با دستور زیر بازنویسی کنید:

```bash
export CLAWHUB_CONFIG_PATH=/path/to/config.json
```

برای راه‌اندازی CI، توکن ذخیره‌شده را با دستور زیر نمایش دهید:

```bash
clawhub token
```

## لغو

می‌توانید توکن‌های API را در رابط وب ClawHub لغو کنید.

توکن‌های لغوشده، نامعتبر یا موجودنبودن توکن باعث بازگرداندن `401 Unauthorized` می‌شوند. دوباره
با `clawhub login` وارد شوید یا با `clawhub login --token` یک توکن تازه ارائه دهید.

حساب‌های حذف‌شده، مسدودشده یا غیرفعال نمی‌توانند به استفاده از توکن‌های API موجود ادامه دهند.
اگر حساب شما مسدود یا غیرفعال شده است و تصور می‌کنید این
یک اشتباه است، از [فرم اعتراض ClawHub](https://appeals.openclaw.ai/) استفاده کنید.
