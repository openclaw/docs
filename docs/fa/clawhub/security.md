# امنیت

مسائل امنیتی ClawHub را می‌توان از طریق GitHub Security Advisories برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از GitHub Security Advisories استفاده کنید. گزارش‌های
هشداری خوب برای ClawHub شامل باگ‌هایی در این موارد هستند:

- وب‌سایت، API یا CLI مربوط به ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها، یا یکپارچگی مصنوعات
- احراز هویت، مجوزدهی، یا توکن‌های API
- اسکن، تعدیل محتوا، یا رسیدگی به گزارش‌ها

از هشدارهای ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ یک مهارت یا
افزونه شخص ثالث استفاده نکنید. آن‌ها را مستقیما به ناشر یا مخزن منبعی که در
فهرست ClawHub پیوند شده است گزارش کنید.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌صورت پیش‌فرض عمومی افشا نمی‌شوند. آن‌ها زمانی عمومی افشا می‌شوند که شواهدی
از اثر واقعی بر کاربران وجود داشته باشد یا وقتی کاربران لازم باشد اقدامی انجام دهند.

نمونه‌های اثر واقعی بر کاربران شامل بهره‌برداری تاییدشده، افشای داده‌ها یا
اسرار کاربران، رسیدن محتوای مخرب به کاربران به‌دلیل خرابی پلتفرم، یا هر مسئله‌ای
است که کاربران را ملزم می‌کند اعتبارنامه‌ها را چرخش دهند، نرم‌افزار محلی را
به‌روزرسانی کنند، یا اقدام حفاظتی دیگری انجام دهند.

آسیب‌پذیری‌های نرم‌افزار نصب‌شده توسط کاربر به‌صورت عمومی افشا می‌شوند، مانند
بسته‌های CLI مربوط به ClawHub، باینری‌ها، کتابخانه‌ها، یا سایر مصنوعات انتشار
که کاربران باید به‌صورت محلی به‌روزرسانی کنند.
