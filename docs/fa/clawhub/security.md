# امنیت + مدیریت محتوا

ClawHub برای انتشار باز است، اما فهرست‌های عمومی همچنان از کنترل‌های اعتماد،
اسکن، گزارش‌دهی و مدیریت محتوا عبور می‌کنند. هدف عملی است: کمک به کاربران برای
بررسی آنچه نصب می‌کنند، ارائه مسیر بازیابی به ناشران برای موارد مثبت کاذب،
و دور نگه داشتن بسته‌های سوءاستفاده‌گر از کشف عمومی.

همچنین [استفاده قابل قبول](/fa/clawhub/acceptable-usage) را ببینید.

## آنچه کاربران می‌توانند بررسی کنند

پیش از نصب یک skill یا plugin، فهرست ClawHub آن را برای موارد زیر بررسی کنید:

- مالک و انتساب منبع
- آخرین نسخه و changelog
- متغیرهای محیطی یا مجوزهای لازم
- فراداده سازگاری برای plugins
- وضعیت اسکن یا مدیریت محتوا
- گزارش‌ها، دیدگاه‌ها، ستاره‌ها، دانلودها و نشانه‌های نصب، در صورت نمایش

فقط محتوایی را نصب کنید که آن را می‌فهمید و به آن اعتماد دارید.

## وضعیت‌های اسکن

ClawHub ممکن است نتایج اسکن یا مدیریت محتوا را در صفحه‌های عمومی و عیب‌یابی‌های
قابل مشاهده برای مالک نشان دهد.

نتایج رایج شامل این‌ها هستند:

- `clean`: هیچ مشکل مسدودکننده‌ای پیدا نشد.
- `suspicious`: این انتشار به احتیاط یا بازبینی نیاز دارد.
- `malicious`: این انتشار ناامن تلقی می‌شود.
- `pending`: بررسی‌ها هنوز تمام نشده‌اند.
- `held`، `quarantined`، `revoked`، یا `hidden`: این انتشار روی سطوح نصب عمومی
  کاملا در دسترس نیست.

عبارت دقیق ممکن است بسته به سطح متفاوت باشد، اما معنای عملی یکسان است: اگر یک
انتشار نگه داشته یا مسدود شده باشد، کاربران نباید آن را نصب کنند تا زمانی که
مالک مشکل را حل کند یا مدیریت محتوا آن را بازگرداند.

## Skills

اسکن‌های skill به بسته skill منتشرشده، فراداده، الزامات اعلام‌شده و دستورالعمل‌های
مشکوک نگاه می‌کنند.

ClawHub توجه ویژه‌ای به ناهماهنگی میان آنچه یک skill اعلام می‌کند و آنچه به نظر
می‌رسد انجام می‌دهد دارد. برای مثال، skillای که به یک کلید API لازم اشاره می‌کند
باید آن الزام را در `SKILL.md` اعلام کند تا کاربران بتوانند پیش از نصب آن را ببینند.

یافته‌های اسکن بر پایه artifact هستند. رفتار مورد انتظار provider، مانند اعتبارنامه‌های
API اعلام‌شده، callbackهای OAuth روی localhost، پاک‌سازی uninstall محدود به scope،
رمزگذاری Basic Auth، یا بارگذاری فایل‌های انتخاب‌شده توسط کاربر به provider اعلام‌شده،
با ارسال پنهانی اعتبارنامه‌ها، دسترسی گسترده به فایل‌های خصوصی، مقصدهای شبکه
نامرتبط، یا سوءاستفاده پنهانی از مرورگر متفاوت برخورد می‌شود.

[قالب Skill](/fa/clawhub/skill-format) را ببینید.

## Plugins

انتشارهای Plugin شامل فراداده بسته، انتساب منبع، فیلدهای سازگاری و اطلاعات یکپارچگی
artifact هستند.

OpenClaw پیش از نصب plugins میزبانی‌شده در ClawHub سازگاری را بررسی می‌کند. رکوردهای
بسته همچنین ممکن است فراداده digest را در معرض دید بگذارند تا OpenClaw بتواند
artifactهای دانلودشده را راستی‌آزمایی کند. ClawScan هنگام بازبینی انتشارهای Plugin،
فراداده env/config اعلام‌شده بسته `openclaw.environment` را شامل می‌کند تا الزامات
runtime اعلام‌شده با رفتار مشاهده‌شده مقایسه شوند.

## گزارش‌ها

کاربران واردشده می‌توانند skills، بسته‌ها و دیدگاه‌ها را گزارش کنند.

گزارش‌ها باید مشخص و قابل اقدام باشند. سوءاستفاده از گزارش‌دهی خود می‌تواند به
اقدام علیه حساب منجر شود.

نمونه‌های گزارش:

- فراداده گمراه‌کننده
- الزامات اعلام‌نشده برای اعتبارنامه یا مجوز
- دستورالعمل‌های نصب مشکوک
- دیدگاه‌های کلاهبردارانه یا جعل هویت
- ثبت‌های با سوءنیت یا سوءاستفاده از نشان تجاری
- محتوایی که [استفاده قابل قبول](/fa/clawhub/acceptable-usage) را نقض می‌کند

## یادداشت‌های ClawScan ناشر

ناشران می‌توانند هنگام انتشار یک skill یا plugin یک یادداشت اختیاری ClawScan ارائه کنند.
این یادداشت به ClawScan برای رفتاری زمینه می‌دهد که در غیر این صورت ممکن است غیرعادی
به نظر برسد، مانند دسترسی شبکه، دسترسی host بومی، یا اعتبارنامه‌های مخصوص provider.

## نگه‌داشت‌های مدیریت محتوا

وقتی اسکنر ایستا یک skill بارگذاری‌شده را malicious علامت‌گذاری می‌کند، ناشر به‌طور
خودکار تحت نگه‌داشت مدیریت محتوا قرار می‌گیرد (`requiresModerationAt` روی کاربر
تنظیم می‌شود). این کار همه skills ناشر را پنهان می‌کند، باعث می‌شود انتشارهای آینده
به‌صورت hidden شروع شوند، و یک ورودی audit log با عنوان `user.moderation.auto`
ایجاد می‌کند.

یافته‌های suspicious ایستا به‌عنوان شواهد file/line برای moderators نگه داشته می‌شوند،
اما به‌تنهایی محتوا را پنهان نمی‌کنند یا verdict اسکن عمومی را تعیین نمی‌کنند.
بارگذاری‌های جدید تا زمان نهایی شدن بازبینی LLM در وضعیت review/pending باقی می‌مانند.
اسکن ایستا فقط برای امضاهای malicious فورا مسدود می‌کند. hitهای موتور VirusTotal
همچنان شواهد امنیتی قابل مشاهده باقی می‌مانند، اما verdictهای VirusTotal Code Insight/Palm
مشورتی هستند و به‌تنهایی skills را پنهان نمی‌کنند. بازبینی‌های LLM در ClawScan
یادداشت‌های همسو با هدف را به‌عنوان راهنما نگه می‌دارند. یافته‌های بازبینی متوسط
روی artifact قابل مشاهده باقی می‌مانند، در حالی که فیلتر suspicious برای نگرانی‌های
LLM با اثر بالا، یافته‌های malicious، یا تشخیص‌های تاییدشده موتور AV رزرو شده است.

مدیران می‌توانند یک نگه‌داشت مثبت کاذب را بردارند:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

این کار `requiresModerationAt` و `requiresModerationReason` را پاک می‌کند، skills
پنهان‌شده توسط نگه‌داشت سطح کاربر را بازمی‌گرداند، و یک ورودی audit log با عنوان
`user.moderation.lift` می‌نویسد. Skills که به دلایل دیگر پنهان شده‌اند، یا اسکن
ایستای خودشان همچنان malicious است، پنهان می‌مانند.

## مسدودسازی‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض کنند ممکن است دسترسی انتشار را از دست بدهند.
سوءاستفاده شدید می‌تواند به مسدودسازی حساب، لغو token، محتوای hidden، یا حذف
فهرست‌ها منجر شود.

حساب‌های حذف‌شده، مسدودشده یا غیرفعال نمی‌توانند از tokenهای API در ClawHub استفاده
کنند. اگر احراز هویت CLI پس از اقدام علیه حساب شروع به شکست کرد، برای بازبینی
وضعیت حساب وارد UI وب شوید. اگر ورود یا دسترسی عادی CLI مسدود شده است، برای
بازبینی بازیابی با security@openclaw.ai تماس بگیرید.

## راهنمای ناشر

برای کاهش موارد مثبت کاذب و بهبود اعتماد کاربر:

- نام‌ها، خلاصه‌ها، tags و changelogها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- وقتی یک انتشار رفتار غیرعادی اما عمدی دارد، یک یادداشت ClawScan ناشر اضافه کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع لینک دهید
- پیش از انتشار plugins از dry run استفاده کنید
- اگر کاربران یا moderators درباره رفتار بسته پرسیدند، روشن پاسخ دهید
