# امنیت

مشکلات امنیتی ClawHub را می‌توان از طریق «هشدارهای امنیتی GitHub» برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از «هشدارهای امنیتی GitHub» استفاده کنید. گزارش‌های
مناسب هشدار ClawHub شامل اشکال‌های موارد زیر هستند:

- وب‌سایت، API یا CLI مربوط به ClawHub
- انتشار در رجیستری، بارگیری‌ها، نصب‌ها یا یکپارچگی مصنوعات
- احراز هویت، مجوزدهی یا توکن‌های API
- اسکن، نظارت بر محتوا یا رسیدگی به گزارش‌ها

برای آسیب‌پذیری‌های موجود در کد منبع خود یک Skill یا Plugin شخص ثالث، از هشدارهای ClawHub
استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبعی که در فهرست ClawHub پیوند داده شده است
گزارش کنید.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌طور پیش‌فرض به‌صورت عمومی افشا نمی‌شوند. زمانی افشای عمومی انجام می‌شود که
شواهدی از تأثیر واقعی بر کاربران وجود داشته باشد یا کاربران نیاز به انجام اقدامی داشته باشند.

نمونه‌های تأثیر واقعی بر کاربران شامل بهره‌برداری تأییدشده، افشای داده‌ها
یا اسرار کاربران، رسیدن محتوای مخرب به کاربران به‌دلیل نقص پلتفرم،
یا هر مشکلی است که کاربران را ملزم به تعویض اطلاعات اصالت‌سنجی، به‌روزرسانی نرم‌افزار محلی یا
انجام اقدامات حفاظتی دیگر کند.

آسیب‌پذیری‌های نرم‌افزارهای نصب‌شده توسط کاربر، مانند بسته‌های CLI مربوط به
ClawHub، فایل‌های دودویی، کتابخانه‌ها یا سایر مصنوعات انتشار که کاربران
باید آن‌ها را به‌صورت محلی به‌روزرسانی کنند، به‌صورت عمومی افشا می‌شوند.

## صفحات مرتبط

برای برچسب‌های ممیزی هنگام نصب، سطوح خطر، یافته‌ها و تفسیر، به
[ممیزی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

برای گزارش‌های بازار، تعلیق‌های ناشی از نظارت بر محتوا، فهرست‌های پنهان، مسدودسازی‌ها و وضعیت
حساب، به [نظارت بر محتوا و امنیت حساب](/clawhub/moderation) مراجعه کنید.
