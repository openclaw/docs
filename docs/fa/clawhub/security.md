# امنیت

مشکلات امنیتی ClawHub را می‌توان از طریق GitHub Security Advisories برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از GitHub Security Advisories استفاده کنید. گزارش‌های
مشورتی خوب ClawHub شامل باگ‌هایی در این موارد هستند:

- وب‌سایت، API، یا CLI مربوط به ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها، یا یکپارچگی آرتیفکت‌ها
- احراز هویت، مجوزدهی، یا توکن‌های API
- اسکن، نظارت، یا رسیدگی به گزارش‌ها

از مشاوره‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خود یک skill شخص ثالث یا
plugin استفاده نکنید. این موارد را مستقیما به ناشر یا مخزن منبعی که از فهرست ClawHub لینک شده است گزارش کنید.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌طور پیش‌فرض به‌صورت عمومی افشا نمی‌شوند. آن‌ها زمانی به‌صورت عمومی افشا می‌شوند که
شواهدی از تأثیر واقعی بر کاربر وجود داشته باشد یا کاربران نیاز به اقدام داشته باشند.

نمونه‌های تأثیر واقعی بر کاربر شامل بهره‌برداری تأییدشده، افشای داده‌ها یا اسرار کاربر، رسیدن محتوای مخرب به کاربران به‌دلیل شکست پلتفرم،
یا هر مشکلی است که کاربران را ملزم به چرخش اعتبارنامه‌ها، به‌روزرسانی نرم‌افزار محلی، یا
انجام اقدام حفاظتی دیگر می‌کند.

آسیب‌پذیری‌های موجود در نرم‌افزار نصب‌شده توسط کاربر، مانند
بسته‌های CLI مربوط به ClawHub، باینری‌ها، کتابخانه‌ها، یا دیگر آرتیفکت‌های انتشار که کاربران
باید به‌صورت محلی به‌روزرسانی کنند، به‌صورت عمومی افشا می‌شوند.

## صفحات مرتبط

برای برچسب‌های ممیزی هنگام نصب، سطوح ریسک، یافته‌ها، و تفسیر، به
[ممیزی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

برای گزارش‌های بازارچه، توقف‌های نظارتی، فهرست‌های پنهان، ممنوعیت‌ها، و وضعیت حساب،
به [نظارت و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
