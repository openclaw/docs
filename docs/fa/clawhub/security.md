# امنیت

مشکلات امنیتی ClawHub را می‌توان از طریق GitHub Security Advisories برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از GitHub Security Advisories استفاده کنید. گزارش‌های مشاوره‌ای خوب
ClawHub شامل باگ‌ها در این موارد هستند:

- وب‌سایت، API یا CLI مربوط به ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها یا یکپارچگی آرتیفکت
- احراز هویت، مجوزدهی یا توکن‌های API
- اسکن، نظارت یا رسیدگی به گزارش‌ها

از مشاوره‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خود یک skill یا
plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیما به ناشر یا مخزن منبعی که در فهرست ClawHub پیوند شده است گزارش کنید.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌طور پیش‌فرض به‌صورت عمومی افشا نمی‌شوند. آن‌ها زمانی به‌صورت عمومی افشا می‌شوند که
شواهدی از اثر واقعی بر کاربر وجود داشته باشد یا کاربران لازم باشد اقدامی انجام دهند.

نمونه‌های اثر واقعی بر کاربر شامل بهره‌برداری تاییدشده، افشای داده‌ها یا اسرار کاربر، رسیدن محتوای مخرب به کاربران به‌دلیل خرابی پلتفرم،
یا هر مشکلی است که کاربران را ملزم به چرخش اعتبارنامه‌ها، به‌روزرسانی نرم‌افزار محلی یا
انجام اقدام حفاظتی دیگر کند.

آسیب‌پذیری‌های موجود در نرم‌افزار نصب‌شده توسط کاربر به‌صورت عمومی افشا می‌شوند، مانند
بسته‌های CLI، باینری‌ها، کتابخانه‌ها یا سایر آرتیفکت‌های انتشار ClawHub که کاربران
باید به‌صورت محلی به‌روزرسانی کنند.

## صفحه‌های مرتبط

برای برچسب‌های ممیزی هنگام نصب، سطوح ریسک، یافته‌ها و تفسیر، به
[ممیزی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

برای گزارش‌های بازارچه، توقف‌های نظارتی، فهرست‌های پنهان، ممنوعیت‌ها و وضعیت حساب،
به [نظارت و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
