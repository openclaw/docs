# امنیت

مشکلات امنیتی ClawHub را می‌توان از طریق GitHub Security Advisories برای
`openclaw/clawhub` گزارش کرد.

از GitHub Security Advisories برای آسیب‌پذیری‌های خود ClawHub استفاده کنید. گزارش‌های
مشاوره‌ای خوب برای ClawHub شامل باگ‌ها در موارد زیر هستند:

- وب‌سایت، API، یا CLI مربوط به ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها، یا یکپارچگی آرتیفکت‌ها
- احراز هویت، مجوزدهی، یا توکن‌های API
- اسکن، نظارت، یا رسیدگی به گزارش‌ها

از مشاوره‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خود یک skill یا
plugin شخص ثالث استفاده نکنید. این موارد را مستقیماً به ناشر یا مخزن منبعی
که از فهرست ClawHub لینک شده است گزارش کنید.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌طور پیش‌فرض به‌صورت عمومی افشا نمی‌شوند. زمانی به‌صورت عمومی افشا می‌شوند
که شواهدی از تأثیر واقعی بر کاربران وجود داشته باشد یا کاربران نیاز به اقدام داشته باشند.

نمونه‌هایی از تأثیر واقعی بر کاربران شامل بهره‌برداری تأییدشده، افشای داده‌ها
یا اسرار کاربران، رسیدن محتوای مخرب به کاربران به‌دلیل خرابی پلتفرم، یا هر
مشکلی است که کاربران را ملزم می‌کند اعتبارنامه‌ها را چرخش دهند، نرم‌افزار
محلی را به‌روزرسانی کنند، یا اقدام محافظتی دیگری انجام دهند.

آسیب‌پذیری‌های موجود در نرم‌افزار نصب‌شده توسط کاربر به‌صورت عمومی افشا می‌شوند؛
مانند بسته‌های CLI مربوط به ClawHub، باینری‌ها، کتابخانه‌ها، یا سایر آرتیفکت‌های
انتشار که کاربران باید به‌صورت محلی به‌روزرسانی کنند.

## صفحه‌های مرتبط

برای برچسب‌های ممیزی هنگام نصب، سطح‌های ریسک، یافته‌ها، و تفسیر، [ممیزی‌های امنیتی](/fa/clawhub/security-audits) را ببینید.

برای گزارش‌های بازارچه، تعلیق‌های نظارتی، فهرست‌های پنهان، ممنوعیت‌ها، و وضعیت
حساب، [نظارت و ایمنی حساب](/fa/clawhub/moderation) را ببینید.
