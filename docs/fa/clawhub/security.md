# امنیت + نظارت

ClawHub برای انتشار باز است، اما فهرست‌های عمومی همچنان از کنترل‌های اعتماد،
اسکن، گزارش‌دهی و نظارت عبور می‌کنند. هدف عملی است: به کاربران کمک شود
آنچه نصب می‌کنند را بررسی کنند، برای ناشران مسیر بازیابی در موارد مثبت کاذب
فراهم شود، و بسته‌های سوءاستفاده‌گر از کشف عمومی دور نگه داشته شوند.

همچنین [استفادهٔ قابل‌قبول](/fa/clawhub/acceptable-usage) را ببینید.

## آنچه کاربران می‌توانند بررسی کنند

پیش از نصب یک skill یا plugin، فهرست ClawHub آن را برای موارد زیر بررسی کنید:

- مالک و انتساب منبع
- آخرین نسخه و تغییرات
- متغیرهای محیطی یا مجوزهای لازم
- فرادادهٔ سازگاری برای plugins
- وضعیت اسکن یا نظارت
- گزارش‌ها، دیدگاه‌ها، ستاره‌ها، دانلودها و نشانه‌های نصب در جاهایی که نمایش داده می‌شوند

فقط محتوایی را نصب کنید که آن را می‌فهمید و به آن اعتماد دارید.

## وضعیت‌های اسکن

ClawHub ممکن است نتایج اسکن یا نظارت را در صفحه‌های عمومی و عیب‌یابی‌های
قابل‌مشاهده برای مالک نمایش دهد.

نتایج رایج شامل این موارد است:

- `clean`: هیچ مشکل مسدودکننده‌ای پیدا نشد.
- `suspicious`: انتشار نیازمند احتیاط یا بازبینی است.
- `malicious`: انتشار ناامن تلقی می‌شود.
- `pending`: بررسی‌ها هنوز تمام نشده‌اند.
- `held`، `quarantined`، `revoked` یا `hidden`: انتشار به‌طور کامل در سطح‌های نصب عمومی
  در دسترس نیست.

عبارت دقیق ممکن است بسته به سطح متفاوت باشد، اما معنای عملی یکسان است: اگر
انتشاری نگه داشته یا مسدود شده باشد، کاربران نباید آن را نصب کنند تا زمانی که مالک
مشکل را رفع کند یا نظارت آن را بازگرداند.

## Skills

اسکن‌های skill بستهٔ skill منتشرشده، فراداده، الزامات اعلام‌شده،
و دستورالعمل‌های مشکوک را بررسی می‌کنند.

ClawHub به ناهماهنگی‌ها میان آنچه یک skill اعلام می‌کند و آنچه ظاهراً انجام
می‌دهد توجه ویژه دارد. برای نمونه، skillای که به یک کلید API لازم اشاره می‌کند
باید آن الزام را در `SKILL.md` اعلام کند تا کاربران پیش از نصب آن را ببینند.

یافته‌های اسکن مبتنی بر artifact هستند. رفتار موردانتظار provider، مانند
اعتبارنامه‌های API اعلام‌شده، callbackهای OAuth روی localhost، پاک‌سازی scoped
هنگام uninstall، کدگذاری Basic Auth، یا بارگذاری فایل‌های انتخاب‌شده توسط کاربر
به provider اعلام‌شده، با ارسال پنهانی اعتبارنامه، دسترسی گسترده به فایل‌های خصوصی،
مقصدهای شبکهٔ نامرتبط، یا سوءاستفادهٔ پنهانی از مرورگر متفاوت برخورد می‌شود.

[قالب Skill](/fa/clawhub/skill-format) را ببینید.

## Plugins

انتشارهای Plugin شامل فرادادهٔ بسته، انتساب منبع، فیلدهای سازگاری،
و اطلاعات یکپارچگی artifact هستند.

OpenClaw پیش از نصب plugins میزبانی‌شده در ClawHub سازگاری را بررسی می‌کند. رکوردهای بسته
ممکن است فرادادهٔ digest را نیز در معرض بگذارند تا OpenClaw بتواند artifactهای
دانلودشده را راستی‌آزمایی کند. ClawScan هنگام بازبینی انتشارهای plugin، فرادادهٔ
env/config اعلام‌شدهٔ بستهٔ `openclaw.environment` را شامل می‌کند تا الزامات runtime
اعلام‌شده با رفتار مشاهده‌شده مقایسه شوند.

## گزارش‌ها

کاربران واردشده می‌توانند skills، بسته‌ها و دیدگاه‌ها را گزارش کنند.

گزارش‌ها باید مشخص و قابل اقدام باشند. سوءاستفاده از گزارش‌دهی می‌تواند خودش
به اقدام علیه حساب منجر شود.

نمونه‌های گزارش:

- فرادادهٔ گمراه‌کننده
- الزامات اعلام‌نشدهٔ اعتبارنامه یا مجوز
- دستورالعمل‌های نصب مشکوک
- دیدگاه‌های کلاهبردارانه یا جعل هویت
- ثبت‌نام‌های بدخواهانه یا سوءاستفاده از نشان تجاری
- محتوایی که [استفادهٔ قابل‌قبول](/fa/clawhub/acceptable-usage) را نقض می‌کند

## یادداشت‌های ClawScan ناشر

ناشران می‌توانند هنگام انتشار یک skill یا plugin یک یادداشت اختیاری ClawScan
ارائه کنند. این یادداشت به ClawScan برای رفتاری که ممکن است در غیر این صورت
غیرمعمول به نظر برسد زمینه می‌دهد، مانند دسترسی شبکه، دسترسی native host،
یا اعتبارنامه‌های مخصوص provider.

## نگه‌داشت‌های نظارتی

وقتی اسکنر ایستا یک skill بارگذاری‌شده را به‌عنوان مخرب علامت‌گذاری می‌کند، ناشر
به‌طور خودکار تحت نگه‌داشت نظارتی قرار می‌گیرد (`requiresModerationAt` روی
کاربر تنظیم می‌شود). این کار همهٔ skills ناشر را پنهان می‌کند، باعث می‌شود
انتشارهای آینده به‌صورت پنهان شروع شوند، و یک ورودی audit log با
`user.moderation.auto` ایجاد می‌کند.

یافته‌های مشکوک ایستا به‌عنوان شواهد فایل/خط برای ناظران نگه داشته می‌شوند،
اما به‌تنهایی محتوا را پنهان نمی‌کنند یا رأی اسکن عمومی را تعیین نمی‌کنند.
بارگذاری‌های تازه تا زمانی که بازبینی LLM نهایی شود در وضعیت review/pending می‌مانند. اسکن ایستا
فقط برای امضاهای مخرب بلافاصله مسدود می‌کند. hitهای موتور VirusTotal همچنان
به‌عنوان شواهد امنیتی قابل‌مشاهده می‌مانند، اما رأی‌های VirusTotal Code Insight/Palm
مشورتی هستند و به‌تنهایی skills را پنهان نمی‌کنند. بازبینی‌های ClawScan LLM
یادداشت‌های همسو با هدف را به‌عنوان راهنما نگه می‌دارند. یافته‌های بازبینی متوسط
روی artifact قابل‌مشاهده می‌مانند، در حالی که فیلتر مشکوک برای نگرانی‌های
پرتأثیر LLM، یافته‌های مخرب، یا تشخیص‌های تأییدشدهٔ موتور AV محفوظ است.

مدیران می‌توانند نگه‌داشت مثبت کاذب را بردارند:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

این کار `requiresModerationAt` و `requiresModerationReason` را پاک می‌کند،
skills پنهان‌شده توسط نگه‌داشت سطح کاربر را بازمی‌گرداند، و یک ورودی audit
log با `user.moderation.lift` می‌نویسد. Skillsای که به دلایل دیگر پنهان شده‌اند،
یا اسکن ایستای خودشان همچنان مخرب است، پنهان می‌مانند.

## ممنوعیت‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض کنند ممکن است دسترسی انتشار را از دست بدهند.
سوءاستفادهٔ شدید می‌تواند به ممنوعیت حساب، ابطال token، محتوای پنهان، یا حذف
فهرست‌ها منجر شود.

حساب‌های حذف‌شده، ممنوع‌شده، یا غیرفعال نمی‌توانند از tokenهای API ClawHub
استفاده کنند. اگر احراز هویت CLI پس از اقدام روی حساب شروع به شکست کرد، برای
بررسی وضعیت حساب وارد UI وب شوید. اگر ورود یا دسترسی عادی CLI مسدود شده است،
برای بازبینی بازیابی با security@openclaw.ai تماس بگیرید.

## راهنمایی ناشران

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربران:

- نام‌ها، خلاصه‌ها، برچسب‌ها و تغییرات را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- وقتی یک انتشار رفتار غیرمعمول اما عمدی دارد، یادداشت ClawScan ناشر اضافه کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع پیوند دهید
- پیش از انتشار plugins از dry run استفاده کنید
- اگر کاربران یا ناظران دربارهٔ رفتار بسته پرسیدند، شفاف پاسخ دهید
