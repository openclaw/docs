# امنیت + نظارت

ClawHub برای انتشار باز است، اما فهرست‌های عمومی همچنان از کنترل‌های اعتماد،
اسکن، گزارش‌دهی و نظارت عبور می‌کنند. هدف عملی است: کمک به کاربران برای
بررسی چیزی که نصب می‌کنند، دادن مسیر بازیابی به ناشران برای مثبت‌های کاذب،
و دور نگه داشتن بسته‌های سوءاستفاده‌گر از کشف عمومی.

همچنین [استفاده قابل قبول](/fa/clawhub/acceptable-usage) را ببینید.

## آنچه کاربران می‌توانند بررسی کنند

پیش از نصب یک مهارت یا Plugin، فهرست ClawHub آن را برای موارد زیر بررسی کنید:

- انتساب مالک و منبع
- آخرین نسخه و تغییرات
- متغیرهای محیطی یا مجوزهای لازم
- فراداده سازگاری برای Pluginها
- وضعیت اسکن یا نظارت
- گزارش‌ها، نظرها، ستاره‌ها، دانلودها و سیگنال‌های نصب در جاهایی که نمایش داده می‌شوند

فقط محتوایی را نصب کنید که می‌فهمید و به آن اعتماد دارید.

## وضعیت‌های اسکن

ClawHub ممکن است نتایج اسکن یا نظارت را در صفحه‌های عمومی و عیب‌یابی‌های
قابل مشاهده برای مالک نمایش دهد.

نتایج رایج شامل این موارد است:

- `clean`: هیچ مشکل مسدودکننده‌ای پیدا نشد.
- `suspicious`: انتشار نیازمند احتیاط یا بازبینی است.
- `malicious`: انتشار ناامن در نظر گرفته می‌شود.
- `pending`: بررسی‌ها هنوز تمام نشده‌اند.
- `held`، `quarantined`، `revoked`، یا `hidden`: انتشار به‌طور کامل روی
  سطوح نصب عمومی در دسترس نیست.

عبارت دقیق ممکن است بسته به سطح متفاوت باشد، اما معنای عملی یکسان است: اگر یک
انتشار نگه داشته یا مسدود شده باشد، کاربران نباید آن را نصب کنند تا زمانی که
مالک مشکل را حل کند یا نظارت آن را بازیابی کند.

## Skills

اسکن‌های مهارت، بسته منتشرشده مهارت، فراداده، نیازمندی‌های اعلام‌شده و
دستورالعمل‌های مشکوک را بررسی می‌کنند.

ClawHub به ناهماهنگی میان چیزی که یک مهارت اعلام می‌کند و کاری که به نظر
می‌رسد انجام می‌دهد توجه ویژه‌ای دارد. برای مثال، مهارتی که به یک کلید API
لازم ارجاع می‌دهد، باید آن نیازمندی را در `SKILL.md` اعلام کند تا کاربران
پیش از نصب آن را ببینند.

یافته‌های اسکن مبتنی بر آرتیفکت هستند. رفتار مورد انتظار ارائه‌دهنده، مانند
اعتبارنامه‌های API اعلام‌شده، callbackهای OAuth روی localhost، پاک‌سازی
حذف نصب محدوده‌دار، کدگذاری Basic Auth، یا بارگذاری فایل‌های انتخاب‌شده توسط
کاربر به ارائه‌دهنده اعلام‌شده، متفاوت از ارسال مخفیانه اعتبارنامه‌ها،
دسترسی گسترده به فایل‌های خصوصی، مقصدهای شبکه نامرتبط، یا سوءاستفاده پنهانی
از مرورگر تلقی می‌شود.

[قالب مهارت](/fa/clawhub/skill-format) را ببینید.

## Plugins

انتشارهای Plugin شامل فراداده بسته، انتساب منبع، فیلدهای سازگاری و اطلاعات
یکپارچگی آرتیفکت هستند.

OpenClaw پیش از نصب Pluginهای میزبانی‌شده در ClawHub، سازگاری را بررسی
می‌کند. رکوردهای بسته همچنین ممکن است فراداده digest را ارائه کنند تا OpenClaw
بتواند آرتیفکت‌های دانلودشده را راستی‌آزمایی کند. ClawScan هنگام بازبینی
انتشارهای Plugin، فراداده env/config اعلام‌شده در `openclaw.environment` بسته
را شامل می‌کند تا نیازمندی‌های زمان اجرای اعلام‌شده با رفتار مشاهده‌شده
مقایسه شوند.

## گزارش‌ها

کاربران واردشده می‌توانند مهارت‌ها، بسته‌ها و نظرها را گزارش کنند.

گزارش‌ها باید مشخص و قابل اقدام باشند. سوءاستفاده از گزارش‌دهی می‌تواند خودش
به اقدام علیه حساب منجر شود.

نمونه‌های گزارش:

- فراداده گمراه‌کننده
- نیازمندی‌های اعلام‌نشده برای اعتبارنامه یا مجوز
- دستورالعمل‌های نصب مشکوک
- نظرهای کلاهبردارانه یا جعل هویت
- ثبت‌های با نیت بد یا سوءاستفاده از نشان تجاری
- محتوایی که [استفاده قابل قبول](/fa/clawhub/acceptable-usage) را نقض می‌کند

## یادداشت‌های ClawScan ناشر

ناشران می‌توانند هنگام انتشار یک مهارت یا Plugin، یک یادداشت اختیاری ClawScan
ارائه کنند. این یادداشت به ClawScan برای رفتاری که ممکن است در غیر این صورت
غیرعادی به نظر برسد زمینه می‌دهد، مانند دسترسی شبکه، دسترسی به میزبان بومی،
یا اعتبارنامه‌های ویژه ارائه‌دهنده.

## نگه‌داشت‌های نظارتی

وقتی اسکنر ایستا یک مهارت بارگذاری‌شده را به‌عنوان مخرب علامت‌گذاری می‌کند،
ناشر به‌طور خودکار تحت نگه‌داشت نظارتی قرار می‌گیرد (`requiresModerationAt`
روی کاربر تنظیم می‌شود). این کار همه مهارت‌های ناشر را پنهان می‌کند، باعث
می‌شود انتشارهای آینده به‌صورت پنهان شروع شوند، و یک ورودی لاگ حسابرسی
`user.moderation.auto` ایجاد می‌کند.

یافته‌های مشکوک ایستا به‌عنوان شواهد فایل/خط برای ناظران نگه داشته می‌شوند،
اما به‌تنهایی محتوا را پنهان نمی‌کنند یا رأی عمومی اسکن را تعیین نمی‌کنند.
بارگذاری‌های جدید تا زمانی که بازبینی LLM نهایی شود در وضعیت بازبینی/در انتظار
می‌مانند. اسکن ایستا فقط برای امضاهای مخرب بلافاصله مسدود می‌کند. برخوردهای
موتور VirusTotal همچنان به‌عنوان شواهد امنیتی قابل مشاهده می‌مانند، اما
رأی‌های VirusTotal Code Insight/Palm مشورتی هستند و به‌تنهایی مهارت‌ها را
پنهان نمی‌کنند. بازبینی‌های LLM در ClawScan یادداشت‌های همسو با هدف را به‌عنوان
راهنما نگه می‌دارند. یافته‌های بازبینی متوسط روی آرتیفکت قابل مشاهده می‌مانند،
در حالی که فیلتر مشکوک برای نگرانی‌های پراثر LLM، یافته‌های مخرب، یا تشخیص‌های
تأییدشده موتور AV رزرو شده است.

مدیران می‌توانند یک نگه‌داشت مثبت کاذب را بردارند:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

این کار `requiresModerationAt` و `requiresModerationReason` را پاک می‌کند،
مهارت‌هایی را که به‌دلیل نگه‌داشت سطح کاربر پنهان شده بودند بازیابی می‌کند،
و یک ورودی لاگ حسابرسی `user.moderation.lift` می‌نویسد. مهارت‌هایی که به دلایل
دیگر پنهان شده‌اند، یا اسکن ایستای خودشان همچنان مخرب است، پنهان می‌مانند.

## ممنوعیت‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض کنند ممکن است دسترسی انتشار را از دست بدهند.
سوءاستفاده شدید می‌تواند به ممنوعیت حساب، لغو token، محتوای پنهان، یا حذف
فهرست‌ها منجر شود.

حساب‌های حذف‌شده، ممنوع‌شده، یا غیرفعال نمی‌توانند از tokenهای API ClawHub
استفاده کنند. اگر پس از اقدام علیه حساب، احراز هویت CLI شروع به شکست کرد، برای
بررسی وضعیت حساب وارد UI وب شوید. اگر ورود یا دسترسی عادی CLI مسدود است، برای
بازبینی بازیابی با security@openclaw.ai تماس بگیرید.

## راهنمای ناشر

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربر:

- نام‌ها، خلاصه‌ها، برچسب‌ها و تغییرات را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- وقتی یک انتشار رفتار غیرعادی اما عمدی دارد، یک یادداشت ClawScan ناشر اضافه کنید
- از دستورهای نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع لینک دهید
- پیش از انتشار Pluginها از اجراهای آزمایشی استفاده کنید
- اگر کاربران یا ناظران درباره رفتار بسته سؤال کردند، شفاف پاسخ دهید
