# امنیت

مسائل امنیتی ClawHub را می‌توان از طریق GitHub Security Advisories برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از GitHub Security Advisories استفاده کنید. گزارش‌های
مشاوره‌ای خوب ClawHub شامل باگ‌ها در این موارد هستند:

- وب‌سایت، API، یا CLI ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها، یا یکپارچگی آرتیفکت
- احراز هویت، مجوزدهی، یا توکن‌های API
- اسکن، تعدیل، یا رسیدگی به گزارش‌ها

از مشاوره‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ یک Skill یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبعی که از
فهرست ClawHub پیوند شده است گزارش کنید.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌طور پیش‌فرض به‌صورت عمومی افشا نمی‌شوند. آن‌ها زمانی به‌صورت عمومی افشا
می‌شوند که شواهدی از اثر واقعی بر کاربران وجود داشته باشد یا وقتی کاربران باید
اقدامی انجام دهند.

نمونه‌های اثر واقعی بر کاربران شامل بهره‌برداری تأییدشده، افشای داده‌ها یا
اسرار کاربران، رسیدن محتوای مخرب به کاربران به دلیل خرابی پلتفرم، یا هر مسئله‌ای
است که کاربران را ملزم می‌کند اعتبارنامه‌ها را بچرخانند، نرم‌افزار محلی را
به‌روزرسانی کنند، یا اقدام حفاظتی دیگری انجام دهند.

آسیب‌پذیری‌ها در نرم‌افزارهای نصب‌شده توسط کاربر به‌صورت عمومی افشا می‌شوند؛
مانند بسته‌های CLI ClawHub، باینری‌ها، کتابخانه‌ها، یا سایر آرتیفکت‌های انتشار
که کاربران باید به‌صورت محلی به‌روزرسانی کنند.

## صفحه‌های مرتبط

برای برچسب‌های ممیزی زمان نصب، سطوح ریسک، یافته‌ها، و تفسیر، به
[ممیزی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

برای گزارش‌های بازارچه، توقف‌های تعدیل، فهرست‌های پنهان، ممنوعیت‌ها، و وضعیت
حساب، به [تعدیل و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
