# امنیت

مسائل امنیتی ClawHub را می‌توان از طریق هشدارهای امنیتی GitHub برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از هشدارهای امنیتی GitHub استفاده کنید. گزارش‌های
مشاوره‌ای خوب ClawHub شامل باگ‌هایی در موارد زیر هستند:

- وب‌سایت، API، یا CLI مربوط به ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها، یا یکپارچگی آرتیفکت
- احراز هویت، مجوزدهی، یا توکن‌های API
- اسکن، مدیریت محتوا، یا رسیدگی به گزارش‌ها

از هشدارهای ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خود یک مهارت یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبعی گزارش
کنید که از فهرست ClawHub به آن پیوند داده شده است.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌صورت پیش‌فرض عمومی افشا نمی‌شوند. این موارد زمانی عمومی افشا می‌شوند که
شواهدی از تأثیر واقعی بر کاربر وجود داشته باشد یا کاربران نیاز به انجام اقدامی
داشته باشند.

نمونه‌های تأثیر واقعی بر کاربر شامل بهره‌برداری تأییدشده، افشای داده‌ها یا
اسرار کاربران، رسیدن محتوای مخرب به کاربران به‌دلیل شکست پلتفرم، یا هر مسئله‌ای
است که کاربران را ملزم به چرخش اعتبارنامه‌ها، به‌روزرسانی نرم‌افزار محلی، یا
انجام اقدام حفاظتی دیگری کند.

آسیب‌پذیری‌های موجود در نرم‌افزارهای نصب‌شده توسط کاربر به‌صورت عمومی افشا
می‌شوند، مانند بسته‌های CLI، باینری‌ها، کتابخانه‌ها، یا سایر آرتیفکت‌های انتشار
ClawHub که کاربران باید به‌صورت محلی به‌روزرسانی کنند.

## صفحه‌های مرتبط

برای برچسب‌های ممیزی هنگام نصب، سطح‌های ریسک، یافته‌ها، و تفسیر، به
[ممیزی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

برای گزارش‌های بازار، نگه‌داشت‌های مدیریت محتوا، فهرست‌های پنهان، ممنوعیت‌ها، و
وضعیت حساب، به [مدیریت محتوا و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
