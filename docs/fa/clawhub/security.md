# امنیت

مشکلات امنیتی ClawHub را می‌توان از طریق GitHub Security Advisories برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از GitHub Security Advisories استفاده کنید. گزارش‌های
مشورتی خوب برای ClawHub شامل باگ‌هایی در موارد زیر هستند:

- وب‌سایت، API، یا CLI مربوط به ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها، یا یکپارچگی آرتیفکت‌ها
- احراز هویت، مجوزدهی، یا توکن‌های API
- اسکن، مدیریت محتوا، یا رسیدگی به گزارش‌ها

از هشدارهای امنیتی ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خود یک مهارت یا
Plugin شخص ثالث استفاده نکنید. این موارد را مستقیماً به ناشر یا مخزن منبعی گزارش
کنید که از فهرست ClawHub به آن پیوند داده شده است.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌طور پیش‌فرض به‌صورت عمومی افشا نمی‌شوند. زمانی به‌صورت عمومی افشا می‌شوند که
شواهدی از اثرگذاری واقعی بر کاربران وجود داشته باشد یا کاربران لازم باشد اقدامی
انجام دهند.

نمونه‌های اثرگذاری واقعی بر کاربران شامل بهره‌برداری تأییدشده، افشای داده‌ها یا
اسرار کاربران، رسیدن محتوای مخرب به کاربران به‌دلیل خرابی پلتفرم، یا هر مشکلی است
که کاربران را ملزم کند اعتبارنامه‌ها را بچرخانند، نرم‌افزار محلی را به‌روزرسانی
کنند، یا اقدام حفاظتی دیگری انجام دهند.

آسیب‌پذیری‌های موجود در نرم‌افزار نصب‌شده توسط کاربر به‌صورت عمومی افشا می‌شوند؛
مانند بسته‌های CLI، باینری‌ها، کتابخانه‌ها، یا سایر آرتیفکت‌های انتشار ClawHub که
کاربران باید به‌صورت محلی به‌روزرسانی کنند.

## صفحه‌های مرتبط

برای برچسب‌های ممیزی هنگام نصب، سطوح ریسک، یافته‌ها، و تفسیر، به
[ممیزی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

برای گزارش‌های بازارچه، توقف‌های مدیریت محتوا، فهرست‌های پنهان، ممنوعیت‌ها، و
وضعیت حساب، به [مدیریت محتوا و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
