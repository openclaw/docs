# امنیت

مسائل امنیتی ClawHub را می‌توان از طریق GitHub Security Advisories برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از GitHub Security Advisories استفاده کنید. گزارش‌های خوب
مشاوره امنیتی ClawHub شامل باگ‌هایی در موارد زیر هستند:

- وب‌سایت، API، یا CLI ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها، یا یکپارچگی آرتیفکت‌ها
- احراز هویت، مجوزدهی، یا توکن‌های API
- اسکن، تعدیل محتوا، یا رسیدگی به گزارش‌ها

از مشاوره‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ Skills یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبعی گزارش کنید
که از فهرست ClawHub لینک شده است.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌طور پیش‌فرض عمومی افشا نمی‌شوند. آن‌ها زمانی به‌صورت عمومی افشا می‌شوند که
شواهدی از تأثیر واقعی بر کاربران وجود داشته باشد یا کاربران لازم باشد اقدامی انجام دهند.

نمونه‌های تأثیر واقعی بر کاربران شامل بهره‌برداری تأییدشده، افشای داده‌ها یا
اسرار کاربران، رسیدن محتوای مخرب به کاربران به‌دلیل خرابی پلتفرم،
یا هر مسئله‌ای است که کاربران را ملزم می‌کند اعتبارنامه‌ها را چرخش دهند، نرم‌افزار محلی را به‌روزرسانی کنند، یا
اقدام حفاظتی دیگری انجام دهند.

آسیب‌پذیری‌های نرم‌افزار نصب‌شده توسط کاربر به‌صورت عمومی افشا می‌شوند، مانند
بسته‌های CLI، باینری‌ها، کتابخانه‌ها، یا دیگر آرتیفکت‌های انتشار ClawHub که کاربران
باید به‌صورت محلی به‌روزرسانی کنند.

## صفحه‌های مرتبط

برای برچسب‌های ممیزی هنگام نصب، سطوح ریسک، یافته‌ها، و تفسیر، به
[ممیزی‌های امنیتی](/fa/clawhub/security-audits) مراجعه کنید.

برای گزارش‌های بازارچه، توقف‌های تعدیل محتوا، فهرست‌های پنهان، ممنوعیت‌ها، و وضعیت
حساب، به [تعدیل محتوا و ایمنی حساب](/fa/clawhub/moderation) مراجعه کنید.
