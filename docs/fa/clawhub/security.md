# امنیت + مدیریت محتوا

ClawHub برای انتشار باز است، اما فهرست‌های عمومی همچنان از کنترل‌های اعتماد،
اسکن، گزارش‌دهی و مدیریت محتوا عبور می‌کنند. هدف عملی است: کمک به کاربران برای
بررسی آنچه نصب می‌کنند، دادن مسیر بازیابی به ناشران برای موارد مثبت کاذب،
و دور نگه داشتن بسته‌های سوءاستفاده‌گر از کشف عمومی.

همچنین ببینید: [استفاده قابل قبول](/fa/clawhub/acceptable-usage).

## آنچه کاربران می‌توانند بررسی کنند

پیش از نصب یک skill یا plugin، فهرست ClawHub آن را برای موارد زیر بررسی کنید:

- مالک و انتساب منبع
- آخرین نسخه و changelog
- متغیرهای محیطی یا مجوزهای لازم
- فراداده سازگاری برای plugins
- وضعیت اسکن یا مدیریت محتوا
- گزارش‌ها، دیدگاه‌ها، ستاره‌ها، دانلودها و سیگنال‌های نصب، در صورت نمایش

فقط محتوایی را نصب کنید که آن را می‌فهمید و به آن اعتماد دارید.

## وضعیت‌های اسکن

ClawHub ممکن است نتایج اسکن یا مدیریت محتوا را در صفحه‌های عمومی و عیب‌یابی‌های
قابل مشاهده برای مالک نشان دهد.

نتایج رایج شامل این موارد است:

- `clean`: هیچ مشکل مسدودکننده‌ای پیدا نشد.
- `suspicious`: این انتشار نیازمند احتیاط یا بازبینی است.
- `malicious`: این انتشار ناامن تلقی می‌شود.
- `pending`: بررسی‌ها هنوز تمام نشده‌اند.
- `held`، `quarantined`، `revoked` یا `hidden`: این انتشار به‌طور کامل در
  سطوح نصب عمومی در دسترس نیست.

عبارت دقیق ممکن است بسته به سطح متفاوت باشد، اما معنای عملی یکسان است: اگر یک
انتشار نگه داشته یا مسدود شده باشد، کاربران نباید آن را نصب کنند تا زمانی که مالک
مشکل را حل کند یا مدیریت محتوا آن را بازگرداند.

## Skills

اسکن‌های skill به بسته منتشرشده skill، فراداده، نیازمندی‌های اعلام‌شده،
و دستورالعمل‌های مشکوک نگاه می‌کنند.

ClawHub به ناسازگاری میان آنچه یک skill اعلام می‌کند و کاری که ظاهرا انجام
می‌دهد توجه ویژه‌ای دارد. برای مثال، skillی که به یک کلید API لازم اشاره می‌کند
باید آن نیازمندی را در `SKILL.md` اعلام کند تا کاربران بتوانند پیش از نصب آن را
ببینند.

یافته‌های اسکن مبتنی بر آرتیفکت هستند. رفتار مورد انتظار provider، مانند
اعتبارنامه‌های API اعلام‌شده، callbackهای OAuth روی localhost، پاک‌سازی uninstall
محدود، کدگذاری Basic Auth، یا آپلود فایل‌های انتخاب‌شده توسط کاربر به provider
اعلام‌شده، با ارسال پنهانی اعتبارنامه‌ها، دسترسی گسترده به فایل‌های خصوصی،
مقصدهای شبکه‌ای نامرتبط، یا سوءاستفاده پنهانی از مرورگر متفاوت برخورد می‌شود.

ببینید: [قالب Skill](/fa/clawhub/skill-format).

## Plugins

انتشارهای Plugin شامل فراداده بسته، انتساب منبع، فیلدهای سازگاری،
و اطلاعات یکپارچگی آرتیفکت هستند.

OpenClaw پیش از نصب plugins میزبانی‌شده در ClawHub، سازگاری را بررسی می‌کند.
رکوردهای بسته همچنین ممکن است فراداده digest را در معرض نمایش بگذارند تا OpenClaw
بتواند آرتیفکت‌های دانلودشده را تأیید کند. ClawScan هنگام بازبینی انتشارهای
plugin، فراداده env/config اعلام‌شده بسته در `openclaw.environment` را شامل می‌کند
تا نیازمندی‌های runtime اعلام‌شده با رفتار مشاهده‌شده مقایسه شوند.

## گزارش‌ها

کاربران واردشده می‌توانند skills، بسته‌ها و دیدگاه‌ها را گزارش کنند.

گزارش‌ها باید مشخص و قابل اقدام باشند. سوءاستفاده از گزارش‌دهی می‌تواند خودش به
اقدام روی حساب منجر شود.

نمونه‌های گزارش:

- فراداده گمراه‌کننده
- نیازمندی‌های اعلام‌نشده برای اعتبارنامه یا مجوز
- دستورالعمل‌های نصب مشکوک
- دیدگاه‌های کلاهبردارانه یا جعل هویت
- ثبت‌نام‌های با سوءنیت یا سوءاستفاده از نشان تجاری
- محتوایی که [استفاده قابل قبول](/fa/clawhub/acceptable-usage) را نقض می‌کند

## یادداشت‌های ناشر برای ClawScan

ناشران می‌توانند هنگام انتشار یک skill یا plugin، یک یادداشت اختیاری برای ClawScan
ارائه کنند. این یادداشت به ClawScan برای رفتاری که در غیر این صورت ممکن است غیرعادی
به نظر برسد زمینه می‌دهد، مانند دسترسی شبکه، دسترسی native host، یا اعتبارنامه‌های
خاص provider.

## نگه‌داشت‌های مدیریت محتوا

وقتی اسکنر static یک skill آپلودشده را به‌عنوان مخرب علامت‌گذاری می‌کند، ناشر
به‌طور خودکار تحت نگه‌داشت مدیریت محتوا قرار می‌گیرد (`requiresModerationAt` روی
کاربر تنظیم می‌شود). این کار همه skills ناشر را پنهان می‌کند، باعث می‌شود
انتشارهای آینده به‌صورت پنهان شروع شوند، و یک ورودی audit log با
`user.moderation.auto` ایجاد می‌کند.

یافته‌های مشکوک static به‌عنوان شواهد فایل/خط برای مدیران محتوا نگه داشته
می‌شوند، اما به‌تنهایی محتوا را پنهان نمی‌کنند یا رأی اسکن عمومی را تعیین
نمی‌کنند. آپلودهای جدید تا زمان نهایی شدن بازبینی LLM در وضعیت review/pending
می‌مانند. اسکن static فقط برای امضاهای مخرب بلافاصله مسدود می‌کند. hitهای موتور
VirusTotal همچنان شواهد امنیتی قابل مشاهده هستند، اما verdictهای VirusTotal Code
Insight/Palm مشورتی‌اند و به‌تنهایی skills را پنهان نمی‌کنند. بازبینی‌های LLM در
ClawScan یادداشت‌های همسو با هدف را به‌عنوان راهنما نگه می‌دارند. یافته‌های
بازبینی medium روی آرتیفکت قابل مشاهده می‌مانند، در حالی که فیلتر suspicious برای
نگرانی‌های LLM با اثر بالا، یافته‌های مخرب، یا شناسایی‌های تأییدشده موتور AV
رزرو شده است.

مدیران می‌توانند یک نگه‌داشت مثبت کاذب را بردارند:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

این کار `requiresModerationAt` و `requiresModerationReason` را پاک می‌کند،
skills پنهان‌شده به دلیل نگه‌داشت سطح کاربر را بازمی‌گرداند، و یک ورودی audit log
با `user.moderation.lift` می‌نویسد. skillsی که به دلایل دیگر پنهان شده‌اند، یا
اسکن static خودشان همچنان مخرب است، پنهان می‌مانند.

## ممنوعیت‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض کنند ممکن است دسترسی انتشار را از دست بدهند.
سوءاستفاده شدید می‌تواند به ممنوعیت حساب، لغو token، محتوای پنهان، یا حذف
فهرست‌ها منجر شود.

حساب‌های حذف‌شده، ممنوع‌شده یا غیرفعال نمی‌توانند از tokenهای API ClawHub استفاده
کنند. اگر احراز هویت CLI پس از اقدام روی حساب شروع به شکست کرد، برای بررسی وضعیت
حساب وارد رابط وب شوید. اگر ورود یا دسترسی عادی CLI مسدود است، برای بازبینی
بازیابی با security@openclaw.ai تماس بگیرید.

## راهنمای ناشر

برای کاهش موارد مثبت کاذب و بهبود اعتماد کاربران:

- نام‌ها، خلاصه‌ها، tagها و changelogها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- وقتی یک انتشار رفتاری غیرمعمول اما عمدی دارد، یک یادداشت ClawScan ناشر اضافه کنید
- از دستورهای نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع پیوند دهید
- پیش از انتشار plugins از dry run استفاده کنید
- اگر کاربران یا مدیران محتوا درباره رفتار بسته پرسیدند، شفاف پاسخ دهید
