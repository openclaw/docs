# امنیت + مدیریت محتوا

ClawHub برای انتشار باز است، اما فهرست‌های عمومی همچنان از کنترل‌های اعتماد،
اسکن، گزارش‌دهی و مدیریت محتوا عبور می‌کنند. هدف عملی است: کمک به کاربران برای
بررسی چیزی که نصب می‌کنند، دادن مسیر بازیابی به ناشران برای مثبت‌های کاذب،
و دور نگه داشتن بسته‌های سوءاستفاده‌گر از کشف عمومی.

همچنین [کاربرد قابل قبول](/fa/clawhub/acceptable-usage) را ببینید.

## آنچه کاربران می‌توانند بررسی کنند

پیش از نصب یک Skills یا Plugin، فهرست ClawHub آن را برای موارد زیر بررسی کنید:

- مالک و انتساب منبع
- آخرین نسخه و تغییرات
- متغیرهای محیطی یا مجوزهای لازم
- فراداده سازگاری برای Pluginها
- وضعیت اسکن یا مدیریت محتوا
- گزارش‌ها، دیدگاه‌ها، ستاره‌ها، دانلودها و سیگنال‌های نصب در جاهایی که نمایش داده می‌شوند

فقط محتوایی را نصب کنید که آن را می‌فهمید و به آن اعتماد دارید.

## وضعیت‌های اسکن

ClawHub ممکن است نتایج اسکن یا مدیریت محتوا را در صفحه‌های عمومی و
عیب‌یابی‌های قابل مشاهده برای مالک نشان دهد.

نتایج رایج شامل این موارد است:

- `clean`: هیچ مشکل مسدودکننده‌ای پیدا نشد.
- `suspicious`: انتشار نیازمند احتیاط یا بازبینی است.
- `malicious`: انتشار ناامن تلقی می‌شود.
- `pending`: بررسی‌ها هنوز تمام نشده‌اند.
- `held`، `quarantined`، `revoked`، یا `hidden`: انتشار به‌طور کامل در
  سطوح نصب عمومی در دسترس نیست.

عبارت دقیق ممکن است بسته به سطح متفاوت باشد، اما معنای عملی یکسان است: اگر یک
انتشار نگه داشته یا مسدود شده باشد، کاربران نباید آن را نصب کنند تا زمانی که
مالک مشکل را حل کند یا مدیریت محتوا آن را بازگرداند.

## Skills

اسکن‌های Skills بسته منتشرشده Skills، فراداده، نیازمندی‌های اعلام‌شده
و دستورالعمل‌های مشکوک را بررسی می‌کنند.

ClawHub به ناهماهنگی بین آنچه یک Skills اعلام می‌کند و آنچه به نظر می‌رسد انجام
می‌دهد توجه ویژه‌ای دارد. برای مثال، Skillsای که به یک کلید API لازم ارجاع
می‌دهد باید آن نیازمندی را در `SKILL.md` اعلام کند تا کاربران بتوانند پیش از
نصب آن را ببینند.

یافته‌های اسکن مبتنی بر آرتیفکت هستند. رفتار مورد انتظار ارائه‌دهنده، مانند
اعتبارنامه‌های API اعلام‌شده، callbackهای OAuth روی localhost، پاک‌سازی حذف
نصب محدود به دامنه، کدگذاری Basic Auth، یا بارگذاری فایل‌های انتخاب‌شده توسط
کاربر به ارائه‌دهنده اعلام‌شده، متفاوت از ارسال پنهانی اعتبارنامه‌ها،
دسترسی گسترده به فایل‌های خصوصی، مقصدهای شبکه نامرتبط، یا سوءاستفاده پنهانی
از مرورگر در نظر گرفته می‌شود.

[قالب Skills](/fa/clawhub/skill-format) را ببینید.

## Pluginها

انتشارهای Plugin شامل فراداده بسته، انتساب منبع، فیلدهای سازگاری
و اطلاعات یکپارچگی آرتیفکت هستند.

OpenClaw پیش از نصب Pluginهای میزبانی‌شده در ClawHub سازگاری را بررسی می‌کند.
رکوردهای بسته همچنین ممکن است فراداده digest را ارائه کنند تا OpenClaw بتواند
آرتیفکت‌های دانلودشده را تأیید کند. ClawScan هنگام بازبینی انتشارهای Plugin،
فراداده env/config اعلام‌شده بسته `openclaw.environment` را شامل می‌کند تا
نیازمندی‌های runtime اعلام‌شده با رفتار مشاهده‌شده مقایسه شوند.

## گزارش‌ها

کاربران واردشده می‌توانند Skills، بسته‌ها و دیدگاه‌ها را گزارش کنند.

گزارش‌ها باید مشخص و قابل اقدام باشند. سوءاستفاده از گزارش‌دهی خود می‌تواند
به اقدام علیه حساب منجر شود.

نمونه‌های گزارش:

- فراداده گمراه‌کننده
- نیازمندی‌های اعلام‌نشده برای اعتبارنامه یا مجوز
- دستورالعمل‌های نصب مشکوک
- دیدگاه‌های کلاهبردارانه یا جعل هویت
- ثبت‌نام‌های بدخواهانه یا سوءاستفاده از نشان تجاری
- محتوایی که [کاربرد قابل قبول](/fa/clawhub/acceptable-usage) را نقض می‌کند

## یادداشت‌های ClawScan ناشر

ناشران می‌توانند هنگام انتشار یک Skills یا Plugin یک یادداشت اختیاری ClawScan
ارائه کنند. این یادداشت به ClawScan برای رفتاری که ممکن است در غیر این صورت
غیرعادی به نظر برسد زمینه می‌دهد، مانند دسترسی شبکه، دسترسی میزبان بومی،
یا اعتبارنامه‌های ویژه ارائه‌دهنده.

## نگه‌داشت‌های مدیریت محتوا

وقتی اسکنر ایستا یک Skills بارگذاری‌شده را مخرب علامت‌گذاری می‌کند، ناشر به‌طور
خودکار تحت نگه‌داشت مدیریت محتوا قرار می‌گیرد (`requiresModerationAt` روی
کاربر تنظیم می‌شود). این کار همه Skillsهای ناشر را پنهان می‌کند، باعث می‌شود
انتشارهای آینده به‌صورت پنهان شروع شوند، و یک ورودی audit log
`user.moderation.auto` ایجاد می‌کند.

یافته‌های مشکوک ایستا به‌عنوان شواهد فایل/خط برای ناظران نگه‌داری می‌شوند،
اما به‌تنهایی محتوا را پنهان نمی‌کنند یا verdict اسکن عمومی را تعیین نمی‌کنند.
بارگذاری‌های جدید تا زمانی که بازبینی LLM نهایی شود در وضعیت بازبینی/در انتظار
می‌مانند. اسکن ایستا فقط برای امضاهای مخرب بلافاصله مسدود می‌کند. hitهای موتور
VirusTotal همچنان شواهد امنیتی قابل مشاهده باقی می‌مانند، اما verdictهای
VirusTotal Code Insight/Palm مشورتی هستند و به‌تنهایی Skills را پنهان نمی‌کنند.
بازبینی‌های LLM در ClawScan یادداشت‌های همسو با هدف را به‌عنوان راهنما نگه
می‌دارند. یافته‌های بازبینی با شدت متوسط روی آرتیفکت قابل مشاهده می‌مانند، در
حالی که فیلتر مشکوک برای نگرانی‌های پراثر LLM، یافته‌های مخرب، یا تشخیص‌های
تأییدشده موتور AV رزرو شده است.

مدیران می‌توانند یک نگه‌داشت مثبت کاذب را بردارند:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

این کار `requiresModerationAt` و `requiresModerationReason` را پاک می‌کند،
Skillsهای پنهان‌شده توسط نگه‌داشت سطح کاربر را بازمی‌گرداند، و یک ورودی
audit log با نام `user.moderation.lift` می‌نویسد. Skillsهایی که به دلایل دیگر
پنهان شده‌اند، یا اسکن ایستای خودشان همچنان مخرب است، پنهان می‌مانند.

## ممنوعیت‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض کنند ممکن است دسترسی انتشار را از دست بدهند.
سوءاستفاده شدید می‌تواند به ممنوعیت حساب، ابطال توکن، محتوای پنهان، یا حذف
فهرست‌ها منجر شود.

حساب‌های حذف‌شده، ممنوع‌شده، یا غیرفعال نمی‌توانند از توکن‌های API مربوط به
ClawHub استفاده کنند. اگر auth در CLI پس از اقدام علیه حساب شروع به شکست کرد،
برای بازبینی وضعیت حساب به UI وب وارد شوید. اگر ورود یا دسترسی عادی CLI مسدود
است، برای بازبینی بازیابی با security@openclaw.ai تماس بگیرید.

## راهنمای ناشران

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربران:

- نام‌ها، خلاصه‌ها، برچسب‌ها و تغییرات را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- وقتی یک انتشار رفتار غیرعادی اما عمدی دارد، یادداشت ClawScan ناشر را اضافه کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع لینک دهید
- پیش از انتشار Pluginها از dry run استفاده کنید
- اگر کاربران یا ناظران درباره رفتار بسته پرسیدند، روشن پاسخ دهید
