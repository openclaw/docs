# امنیت

مشکلات امنیتی ClawHub را می‌توان از طریق GitHub Security Advisories برای
`openclaw/clawhub` گزارش کرد.

از GitHub Security Advisories برای آسیب‌پذیری‌های خود ClawHub استفاده کنید. گزارش‌های خوب
مشاوره امنیتی ClawHub شامل باگ‌هایی در این موارد هستند:

- وب‌سایت، API، یا CLI ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها، یا یکپارچگی آرتیفکت
- احراز هویت، مجوزدهی، یا توکن‌های API
- اسکن، تعدیل محتوا، یا رسیدگی به گزارش‌ها

از مشاوره‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خود یک مهارت یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبعی گزارش کنید
که از فهرست ClawHub لینک شده است.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌طور پیش‌فرض به‌صورت عمومی افشا نمی‌شوند. زمانی به‌صورت عمومی افشا می‌شوند که
شواهدی از تأثیر واقعی بر کاربر وجود داشته باشد یا کاربران لازم باشد اقدامی انجام دهند.

نمونه‌های تأثیر واقعی بر کاربر شامل بهره‌برداری تأییدشده، افشای داده‌ها یا اسرار
کاربر، رسیدن محتوای مخرب به کاربران به‌دلیل خرابی پلتفرم، یا هر مشکلی است که
کاربران را ملزم می‌کند اعتبارنامه‌ها را چرخش دهند، نرم‌افزار محلی را به‌روزرسانی کنند، یا
اقدام حفاظتی دیگری انجام دهند.

آسیب‌پذیری‌های موجود در نرم‌افزارهای نصب‌شده توسط کاربر به‌صورت عمومی افشا می‌شوند، مانند
بسته‌های CLI ClawHub، باینری‌ها، کتابخانه‌ها، یا سایر آرتیفکت‌های انتشار که کاربران
باید آن‌ها را به‌صورت محلی به‌روزرسانی کنند.

## صفحه‌های مرتبط

برای برچسب‌های ممیزی زمان نصب، سطوح ریسک، یافته‌ها، و تفسیر، [ممیزی‌های امنیتی](/fa/clawhub/security-audits) را ببینید.

برای گزارش‌های بازارچه، توقف‌های تعدیل محتوا، فهرست‌های پنهان، ممنوعیت‌ها، و وضعیت حساب،
[تعدیل محتوا و ایمنی حساب](/fa/clawhub/moderation) را ببینید.
