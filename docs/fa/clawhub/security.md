# امنیت

مسائل امنیتی ClawHub را می‌توان از طریق هشدارهای امنیتی GitHub برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از هشدارهای امنیتی GitHub استفاده کنید. گزارش‌های
خوب هشدار امنیتی ClawHub شامل اشکال‌ها در این موارد هستند:

- وب‌سایت، API، یا CLI ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها، یا یکپارچگی آرتیفکت
- احراز هویت، مجوزدهی، یا توکن‌های API
- اسکن، نظارت، یا رسیدگی به گزارش‌ها

از هشدارهای ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خود یک skill یا
plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبعی گزارش کنید
که از فهرست ClawHub به آن پیوند داده شده است.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌صورت پیش‌فرض عمومی افشا نمی‌شوند. زمانی عمومی افشا می‌شوند که شواهدی از اثر
واقعی بر کاربر وجود داشته باشد یا کاربران لازم باشد اقدامی انجام دهند.

نمونه‌های اثر واقعی بر کاربر شامل بهره‌برداری تأییدشده، افشای داده‌ها یا اسرار
کاربر، رسیدن محتوای مخرب به کاربران به‌دلیل خرابی پلتفرم، یا هر مسئله‌ای است که
کاربران را ملزم به چرخش اعتبارنامه‌ها، به‌روزرسانی نرم‌افزار محلی، یا انجام
اقدام محافظتی دیگر می‌کند.

آسیب‌پذیری‌های نرم‌افزار نصب‌شده توسط کاربر، مانند بسته‌های CLI، باینری‌ها،
کتابخانه‌ها، یا دیگر آرتیفکت‌های انتشار ClawHub که کاربران باید به‌صورت محلی
به‌روزرسانی کنند، به‌صورت عمومی افشا می‌شوند.

## صفحه‌های مرتبط

برای برچسب‌های ممیزی هنگام نصب، سطوح ریسک، یافته‌ها، و تفسیر، به
[ممیزی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

برای گزارش‌های بازارچه، تعلیق‌های نظارتی، فهرست‌های پنهان، ممنوعیت‌ها، و وضعیت
حساب، به [نظارت و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
