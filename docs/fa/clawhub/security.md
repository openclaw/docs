# امنیت

مشکلات امنیتی ClawHub را می‌توان از طریق مشاوره‌های امنیتی GitHub برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از مشاوره‌های امنیتی GitHub استفاده کنید. گزارش‌های
خوب مشاوره امنیتی ClawHub شامل باگ‌ها در این موارد هستند:

- وب‌سایت، API، یا CLI مربوط به ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها، یا یکپارچگی آرتیفکت‌ها
- احراز هویت، مجوزدهی، یا توکن‌های API
- اسکن، مدیریت محتوا، یا رسیدگی به گزارش‌ها

از مشاوره‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ یک مهارت یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبعی گزارش کنید
که از فهرست ClawHub لینک شده است.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌طور پیش‌فرض عمومی افشا نمی‌شوند. آن‌ها زمانی عمومی افشا می‌شوند که شواهدی از
اثر واقعی بر کاربران وجود داشته باشد یا کاربران نیاز به انجام اقدامی داشته باشند.

نمونه‌های اثر واقعی بر کاربران شامل بهره‌برداری تأییدشده، افشای داده‌ها یا اسرار
کاربر، رسیدن محتوای مخرب به کاربران به‌دلیل خرابی پلتفرم، یا هر مشکلی است که
کاربران را ملزم می‌کند اعتبارنامه‌ها را چرخش دهند، نرم‌افزار محلی را به‌روزرسانی
کنند، یا اقدام محافظتی دیگری انجام دهند.

آسیب‌پذیری‌های موجود در نرم‌افزار نصب‌شده توسط کاربر به‌صورت عمومی افشا می‌شوند،
مانند بسته‌های CLI مربوط به ClawHub، فایل‌های باینری، کتابخانه‌ها، یا دیگر
آرتیفکت‌های انتشار که کاربران باید به‌صورت محلی به‌روزرسانی کنند.

## صفحه‌های مرتبط

برای برچسب‌های ممیزی زمان نصب، سطح‌های ریسک، یافته‌ها، و تفسیر، به
[ممیزی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

برای گزارش‌های بازارچه، توقف‌های مربوط به مدیریت محتوا، فهرست‌های پنهان، ممنوعیت‌ها،
و وضعیت حساب، به [مدیریت محتوا و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
