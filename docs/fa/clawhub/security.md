# امنیت

مشکلات امنیتی ClawHub را می‌توان از طریق هشدارهای امنیتی GitHub برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از هشدارهای امنیتی GitHub استفاده کنید. گزارش‌های
خوب هشدار امنیتی ClawHub شامل باگ‌هایی در این موارد هستند:

- وب‌سایت، API یا CLI ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها یا یکپارچگی آرتیفکت‌ها
- احراز هویت، مجوزدهی یا توکن‌های API
- اسکن، بازبینی یا رسیدگی به گزارش‌ها

از هشدارهای ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ یک مهارت یا افزونه
شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبعی گزارش کنید که در
فهرست ClawHub پیوند داده شده است.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌صورت پیش‌فرض به‌طور عمومی افشا نمی‌شوند. آن‌ها زمانی به‌طور عمومی افشا می‌شوند
که شواهدی از تأثیر واقعی بر کاربر وجود داشته باشد یا کاربران لازم باشد اقدامی
انجام دهند.

نمونه‌هایی از تأثیر واقعی بر کاربر شامل بهره‌برداری تأییدشده، افشای داده‌ها یا
اسرار کاربران، رسیدن محتوای مخرب به کاربران به‌دلیل خرابی پلتفرم، یا هر مشکلی است
که کاربران را ملزم کند اعتبارنامه‌ها را چرخش دهند، نرم‌افزار محلی را به‌روزرسانی
کنند یا اقدام حفاظتی دیگری انجام دهند.

آسیب‌پذیری‌های نرم‌افزار نصب‌شده توسط کاربر، مانند بسته‌های CLI، باینری‌ها،
کتابخانه‌ها یا سایر آرتیفکت‌های انتشار ClawHub که کاربران باید به‌صورت محلی
به‌روزرسانی کنند، به‌طور عمومی افشا می‌شوند.

## صفحات مرتبط

برای برچسب‌های ممیزی هنگام نصب، سطوح ریسک، یافته‌ها و تفسیر، به
[ممیزی‌های امنیتی](/fa/clawhub/security-audits) مراجعه کنید.

برای گزارش‌های بازارچه، توقف‌های بازبینی، فهرست‌های پنهان، ممنوعیت‌ها و وضعیت
حساب، به [بازبینی و ایمنی حساب](/fa/clawhub/moderation) مراجعه کنید.
