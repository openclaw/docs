# امنیت

مشکلات امنیتی ClawHub را می‌توان از طریق هشدارهای امنیتی GitHub برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از هشدارهای امنیتی GitHub استفاده کنید. گزارش‌های
خوب هشدار امنیتی ClawHub شامل باگ‌هایی در موارد زیر هستند:

- وب‌سایت، API یا CLI مربوط به ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها یا یکپارچگی آرتیفکت‌ها
- احراز هویت، مجوزدهی یا توکن‌های API
- اسکن، نظارت محتوایی یا رسیدگی به گزارش‌ها

از هشدارهای امنیتی ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خود یک skill یا
plugin شخص ثالث استفاده نکنید. این موارد را مستقیماً به ناشر یا مخزن منبعی که در
فهرست ClawHub پیوند داده شده است گزارش کنید.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌طور پیش‌فرض عمومی افشا نمی‌شوند. این موارد زمانی عمومی افشا می‌شوند که شواهدی
از اثر واقعی بر کاربران وجود داشته باشد یا کاربران نیاز به اقدام داشته باشند.

نمونه‌های اثر واقعی بر کاربران شامل بهره‌برداری تأییدشده، افشای داده‌ها یا
اسرار کاربران، رسیدن محتوای مخرب به کاربران به دلیل نقص پلتفرم، یا هر مشکلی است
که کاربران را ملزم به چرخش اعتبارنامه‌ها، به‌روزرسانی نرم‌افزار محلی، یا انجام
اقدام حفاظتی دیگر کند.

آسیب‌پذیری‌های موجود در نرم‌افزار نصب‌شده توسط کاربر به‌صورت عمومی افشا می‌شوند،
مانند بسته‌های CLI مربوط به ClawHub، باینری‌ها، کتابخانه‌ها، یا دیگر آرتیفکت‌های
انتشار که کاربران باید آن‌ها را به‌صورت محلی به‌روزرسانی کنند.

## صفحه‌های مرتبط

برای برچسب‌های ممیزی هنگام نصب، سطوح ریسک، یافته‌ها و تفسیر، به
[ممیزی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

برای گزارش‌های بازارچه، توقف‌های نظارتی، فهرست‌های پنهان، ممنوعیت‌ها و وضعیت
حساب، به [نظارت محتوایی و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
