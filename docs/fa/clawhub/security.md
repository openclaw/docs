# امنیت

مسائل امنیتی ClawHub را می‌توان از طریق مشاوره‌های امنیتی GitHub برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از مشاوره‌های امنیتی GitHub استفاده کنید. گزارش‌های مشاوره امنیتی خوب برای ClawHub شامل باگ‌هایی در موارد زیر هستند:

- وب‌سایت، API، یا CLI مربوط به ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها، یا یکپارچگی آرتیفکت‌ها
- احراز هویت، مجوزدهی، یا توکن‌های API
- اسکن، نظارت محتوایی، یا رسیدگی به گزارش‌ها

از مشاوره‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ یک Skills یا Plugin شخص ثالث استفاده نکنید. این موارد را مستقیماً به ناشر یا مخزن منبعی گزارش کنید که از فهرست ClawHub پیوند داده شده است.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub به‌طور پیش‌فرض به‌صورت عمومی افشا نمی‌شوند. زمانی به‌صورت عمومی افشا می‌شوند که شواهدی از تأثیر واقعی بر کاربر وجود داشته باشد یا کاربران لازم باشد اقدامی انجام دهند.

نمونه‌هایی از تأثیر واقعی بر کاربر شامل بهره‌برداری تأییدشده، افشای داده‌ها یا اسرار کاربران، رسیدن محتوای مخرب به کاربران به‌دلیل نقص پلتفرم، یا هر مسئله‌ای است که کاربران را ملزم به چرخش اعتبارنامه‌ها، به‌روزرسانی نرم‌افزار محلی، یا انجام اقدام حفاظتی دیگری کند.

آسیب‌پذیری‌های موجود در نرم‌افزار نصب‌شده توسط کاربر، مانند بسته‌های CLI مربوط به ClawHub، باینری‌ها، کتابخانه‌ها، یا سایر آرتیفکت‌های انتشار که کاربران باید به‌صورت محلی به‌روزرسانی کنند، به‌صورت عمومی افشا می‌شوند.

## صفحه‌های مرتبط

برای برچسب‌های ممیزی زمان نصب، سطح‌های ریسک، یافته‌ها، و تفسیر، [ممیزی‌های امنیتی](/clawhub/security-audits) را ببینید.

برای گزارش‌های بازارچه، توقف‌های نظارتی، فهرست‌های پنهان، مسدودسازی‌ها، و وضعیت حساب، [نظارت محتوایی و ایمنی حساب](/clawhub/moderation) را ببینید.
