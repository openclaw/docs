# امنیت + نظارت

ClawHub برای انتشار باز است، اما فهرست‌های عمومی همچنان از کنترل‌های اعتماد، اسکن، گزارش‌دهی و نظارت عبور می‌کنند. هدف کاربردی است: کمک به کاربران برای بررسی چیزی که نصب می‌کنند، فراهم‌کردن مسیر بازیابی برای ناشران در موارد مثبت کاذب، و دور نگه‌داشتن بسته‌های سوءاستفاده‌گر از کشف عمومی.

همچنین ببینید [استفادهٔ قابل قبول](/fa/clawhub/acceptable-usage).

## آنچه کاربران می‌توانند بررسی کنند

پیش از نصب یک مهارت یا Plugin، فهرست ClawHub آن را از نظر موارد زیر بررسی کنید:

- مالک و انتساب منبع
- آخرین نسخه و گزارش تغییرات
- متغیرهای محیطی یا مجوزهای موردنیاز
- فرادادهٔ سازگاری برای Pluginها
- وضعیت اسکن یا نظارت
- گزارش‌ها، دیدگاه‌ها، ستاره‌ها، دانلودها، و نشانه‌های نصب در جاهایی که نمایش داده می‌شوند

فقط محتوایی را نصب کنید که آن را می‌فهمید و به آن اعتماد دارید.

## وضعیت‌های اسکن

ClawHub ممکن است نتایج اسکن یا نظارت را در صفحه‌های عمومی و عیب‌یابی‌های قابل مشاهده برای مالک نشان دهد.

نتایج رایج شامل این موارد هستند:

- `clean`: هیچ مشکل مسدودکننده‌ای پیدا نشد.
- `suspicious`: انتشار نیازمند احتیاط یا بازبینی است.
- `malicious`: انتشار ناامن تلقی می‌شود.
- `pending`: بررسی‌ها هنوز تمام نشده‌اند.
- `held`، `quarantined`، `revoked`، یا `hidden`: انتشار به‌طور کامل در سطوح نصب عمومی در دسترس نیست.

عبارت دقیق ممکن است بسته به سطح متفاوت باشد، اما معنای عملی یکسان است: اگر یک انتشار نگه داشته یا مسدود شده باشد، کاربران نباید آن را نصب کنند تا زمانی که مالک مشکل را برطرف کند یا نظارت آن را بازیابی کند.

## Skills

اسکن‌های مهارت، بستهٔ مهارت منتشرشده، فراداده، نیازمندی‌های اعلام‌شده، و دستورالعمل‌های مشکوک را بررسی می‌کنند.

ClawHub توجه ویژه‌ای به ناهماهنگی میان چیزی که یک مهارت اعلام می‌کند و چیزی که به نظر می‌رسد انجام می‌دهد دارد. برای مثال، مهارتی که به یک کلید API موردنیاز اشاره می‌کند باید آن نیازمندی را در `SKILL.md` اعلام کند تا کاربران بتوانند پیش از نصب آن را ببینند.

یافته‌های اسکن مبتنی بر مصنوع هستند. رفتار موردانتظار ارائه‌دهنده، مانند اعتبارنامه‌های API اعلام‌شده، callbackهای OAuth روی localhost، پاک‌سازی محدود هنگام حذف نصب، رمزگذاری Basic Auth، یا بارگذاری فایل‌های انتخاب‌شده توسط کاربر به ارائه‌دهندهٔ اعلام‌شده، با ارسال پنهانی اعتبارنامه‌ها، دسترسی گسترده به فایل‌های خصوصی، مقصدهای شبکهٔ نامرتبط، یا سوءاستفادهٔ مخفیانه از مرورگر متفاوت برخورد می‌شود.

ببینید [قالب مهارت](/fa/clawhub/skill-format).

## Pluginها

انتشارهای Plugin شامل فرادادهٔ بسته، انتساب منبع، فیلدهای سازگاری، و اطلاعات یکپارچگی مصنوع هستند.

OpenClaw پیش از نصب Pluginهای میزبانی‌شده در ClawHub سازگاری را بررسی می‌کند. رکوردهای بسته همچنین ممکن است فرادادهٔ digest را آشکار کنند تا OpenClaw بتواند مصنوع‌های دانلودشده را تأیید کند. ClawScan هنگام بازبینی انتشارهای Plugin، فرادادهٔ env/config اعلام‌شدهٔ بستهٔ `openclaw.environment` را شامل می‌کند تا نیازمندی‌های زمان اجرای اعلام‌شده با رفتار مشاهده‌شده مقایسه شوند.

## گزارش‌ها

کاربران واردشده می‌توانند مهارت‌ها، بسته‌ها، و دیدگاه‌ها را گزارش کنند.

گزارش‌ها باید مشخص و قابل اقدام باشند. سوءاستفاده از گزارش‌دهی نیز می‌تواند به اقدام علیه حساب منجر شود.

نمونه‌های گزارش:

- فرادادهٔ گمراه‌کننده
- نیازمندی‌های اعلام‌نشدهٔ اعتبارنامه یا مجوز
- دستورالعمل‌های نصب مشکوک
- دیدگاه‌های کلاهبردارانه یا جعل هویت
- ثبت‌های با سوءنیت یا سوءاستفاده از نشان تجاری
- محتوایی که [استفادهٔ قابل قبول](/fa/clawhub/acceptable-usage) را نقض می‌کند

## یادداشت‌های ClawScan ناشر

ناشران می‌توانند هنگام انتشار یک مهارت یا Plugin، یک یادداشت اختیاری ClawScan ارائه کنند. این یادداشت به ClawScan برای رفتاری که در غیر این صورت ممکن است غیرمعمول به نظر برسد زمینه می‌دهد، مانند دسترسی شبکه، دسترسی native host، یا اعتبارنامه‌های ویژهٔ ارائه‌دهنده.

## نگه‌داشت‌های نظارتی

وقتی اسکنر ایستا یک مهارت بارگذاری‌شده را به‌عنوان مخرب علامت‌گذاری می‌کند، ناشر به‌طور خودکار تحت نگه‌داشت نظارتی قرار می‌گیرد (`requiresModerationAt` روی کاربر تنظیم می‌شود). این کار همهٔ مهارت‌های ناشر را پنهان می‌کند، باعث می‌شود انتشارهای آینده به‌صورت پنهان شروع شوند، و یک ورودی گزارش حسابرسی `user.moderation.auto` ایجاد می‌کند.

یافته‌های مشکوک ایستا به‌عنوان شواهد فایل/خط برای ناظران نگه داشته می‌شوند، اما به‌تنهایی محتوا را پنهان نمی‌کنند یا دربارهٔ رأی اسکن عمومی تصمیم نمی‌گیرند. بارگذاری‌های جدید تا زمان نهایی‌شدن بازبینی LLM در وضعیت بازبینی/در انتظار می‌مانند. اسکن ایستا فقط برای امضاهای مخرب فوراً مسدود می‌کند. اصابت‌های موتور VirusTotal همچنان به‌عنوان شواهد امنیتی قابل مشاهده می‌مانند، اما رأی‌های VirusTotal Code Insight/Palm مشورتی هستند و به‌تنهایی مهارت‌ها را پنهان نمی‌کنند. بازبینی‌های LLM در ClawScan یادداشت‌های همسو با هدف را به‌عنوان راهنما نگه می‌دارند. یافته‌های بازبینی با شدت متوسط روی مصنوع قابل مشاهده می‌مانند، در حالی که فیلتر مشکوک برای نگرانی‌های LLM با اثرگذاری بالا، یافته‌های مخرب، یا تشخیص‌های تأییدشده توسط موتورهای AV محفوظ است.

ادمین‌ها می‌توانند یک نگه‌داشت مثبت کاذب را بردارند:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

این کار `requiresModerationAt` و `requiresModerationReason` را پاک می‌کند، مهارت‌های پنهان‌شده توسط نگه‌داشت سطح کاربر را بازیابی می‌کند، و یک ورودی گزارش حسابرسی `user.moderation.lift` می‌نویسد. مهارت‌هایی که به دلایل دیگر پنهان شده‌اند، یا اسکن ایستای خودشان همچنان مخرب است، پنهان می‌مانند.

## مسدودسازی‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفادهٔ شدید می‌تواند به مسدودسازی حساب، لغو توکن، محتوای پنهان، یا حذف فهرست‌ها منجر شود.

حساب‌های حذف‌شده، مسدودشده، یا غیرفعال نمی‌توانند از توکن‌های API ClawHub استفاده کنند. اگر احراز هویت CLI پس از اقدام روی حساب شروع به شکست کرد، برای بررسی وضعیت حساب وارد رابط وب شوید. اگر ورود یا دسترسی عادی CLI مسدود است، برای بازبینی بازیابی با security@openclaw.ai تماس بگیرید.

## راهنمای ناشر

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربران:

- نام‌ها، خلاصه‌ها، برچسب‌ها، و گزارش‌های تغییرات را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای موردنیاز را اعلام کنید
- وقتی یک انتشار رفتاری غیرمعمول اما عمدی دارد، یک یادداشت ClawScan ناشر اضافه کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع پیوند دهید
- پیش از انتشار Pluginها از dry run استفاده کنید
- اگر کاربران یا ناظران دربارهٔ رفتار بسته پرسیدند، شفاف پاسخ دهید
