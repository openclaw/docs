# امنیت

مسائل امنیتی ClawHub را می‌توان از طریق GitHub Security Advisories برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از GitHub Security Advisories استفاده کنید. گزارش‌های
راهنمای امنیتی خوب برای ClawHub شامل اشکال‌هایی در این موارد هستند:

- وب‌سایت، API یا CLI مربوط به ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها یا یکپارچگی آرتیفکت‌ها
- احراز هویت، مجوزدهی یا توکن‌های API
- اسکن، تعدیل‌گری یا رسیدگی به گزارش‌ها

از راهنماهای امنیتی ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خود یک مهارت یا
Plugin شخص ثالث استفاده نکنید. این موارد را مستقیما به ناشر یا مخزن منبعی که از
فهرست ClawHub لینک شده است گزارش کنید.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌طور پیش‌فرض به‌صورت عمومی افشا نمی‌شوند. زمانی به‌صورت عمومی افشا می‌شوند که
شواهدی از اثر واقعی بر کاربر وجود داشته باشد یا کاربران لازم باشد اقدامی انجام دهند.

نمونه‌های اثر واقعی بر کاربر شامل بهره‌برداری تأییدشده، افشای داده‌ها یا اسرار
کاربر، رسیدن محتوای مخرب به کاربران به‌دلیل خرابی پلتفرم، یا هر مسئله‌ای است که
کاربران را ملزم به چرخاندن اعتبارنامه‌ها، به‌روزرسانی نرم‌افزار محلی یا انجام
اقدام محافظتی دیگری کند.

آسیب‌پذیری‌های موجود در نرم‌افزار نصب‌شده توسط کاربر به‌صورت عمومی افشا می‌شوند،
مانند بسته‌های CLI مربوط به ClawHub، فایل‌های اجرایی، کتابخانه‌ها یا دیگر
آرتیفکت‌های انتشار که کاربران باید به‌صورت محلی به‌روزرسانی کنند.

## صفحه‌های مرتبط

برای برچسب‌های ممیزی هنگام نصب، سطح‌های ریسک، یافته‌ها و تفسیر، به
[ممیزی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

برای گزارش‌های بازار، توقف‌های تعدیل‌گری، فهرست‌های پنهان، ممنوعیت‌ها و وضعیت
حساب، به [تعدیل‌گری و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
