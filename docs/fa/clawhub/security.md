# امنیت

مشکلات امنیتی ClawHub را می‌توان از طریق GitHub Security Advisories برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از GitHub Security Advisories استفاده کنید. گزارش‌های
مشاورهٔ خوب برای ClawHub شامل باگ‌ها در این موارد هستند:

- وب‌سایت، API، یا CLI مربوط به ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها، یا یکپارچگی آرتیفکت
- احراز هویت، مجوزدهی، یا توکن‌های API
- اسکن، تعدیل، یا رسیدگی به گزارش‌ها

برای آسیب‌پذیری‌های موجود در کد منبع خود یک Skills یا Plugin شخص ثالث، از مشاوره‌های ClawHub استفاده نکنید. آن‌ها را مستقیماً به منتشرکننده یا مخزن منبعی که در فهرست ClawHub پیوند شده است گزارش کنید.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامهٔ ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌طور پیش‌فرض به‌صورت عمومی افشا نمی‌شوند. زمانی به‌صورت عمومی افشا می‌شوند که
شواهدی از تأثیر واقعی بر کاربر وجود داشته باشد یا کاربران لازم باشد اقدامی انجام دهند.

نمونه‌های تأثیر واقعی بر کاربر شامل بهره‌برداری تأییدشده، افشای داده‌ها یا اسرار کاربر، رسیدن محتوای مخرب به کاربران به‌دلیل نقص پلتفرم، یا هر مشکلی است که کاربران را ملزم می‌کند اعتبارنامه‌ها را چرخش دهند، نرم‌افزار محلی را به‌روزرسانی کنند، یا اقدام محافظتی دیگری انجام دهند.

آسیب‌پذیری‌های موجود در نرم‌افزار نصب‌شده توسط کاربر به‌صورت عمومی افشا می‌شوند، مانند
بسته‌های CLI مربوط به ClawHub، باینری‌ها، کتابخانه‌ها، یا سایر آرتیفکت‌های انتشار که کاربران
باید به‌صورت محلی به‌روزرسانی کنند.

## صفحه‌های مرتبط

برای برچسب‌های حسابرسی هنگام نصب، سطوح ریسک، یافته‌ها، و تفسیر، [حسابرسی‌های امنیتی](/clawhub/security-audits) را ببینید.

برای گزارش‌های بازارچه، توقف‌های تعدیل، فهرست‌های پنهان، محرومیت‌ها، و وضعیت حساب، [تعدیل و ایمنی حساب](/clawhub/moderation) را ببینید.
