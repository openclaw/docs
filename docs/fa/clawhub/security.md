# امنیت + تعدیل محتوا

ClawHub برای انتشار باز است، اما فهرست‌های عمومی همچنان از کنترل‌های اعتماد،
اسکن، گزارش‌دهی و تعدیل محتوا عبور می‌کنند. هدف عملی است: کمک به کاربران برای
بررسی چیزی که نصب می‌کنند، دادن مسیر بازیابی به ناشران برای مثبت‌های کاذب،
و دور نگه داشتن بسته‌های سوءاستفاده‌گر از کشف عمومی.

همچنین ببینید: [استفاده قابل قبول](/fa/clawhub/acceptable-usage).

## آنچه کاربران می‌توانند بررسی کنند

پیش از نصب یک مهارت یا Plugin، فهرست ClawHub آن را برای موارد زیر بررسی کنید:

- مالک و انتساب منبع
- آخرین نسخه و changelog
- متغیرهای محیطی یا مجوزهای لازم
- فراداده سازگاری برای Pluginها
- وضعیت اسکن یا تعدیل محتوا
- گزارش‌ها، نظرها، ستاره‌ها، دانلودها و نشانه‌های نصب در جاهایی که نمایش داده می‌شوند

فقط محتوایی را نصب کنید که آن را می‌فهمید و به آن اعتماد دارید.

## وضعیت‌های اسکن

ClawHub ممکن است نتیجه‌های اسکن یا تعدیل محتوا را در صفحه‌های عمومی و
عیب‌یابی‌های قابل مشاهده برای مالک نشان دهد.

نتیجه‌های رایج شامل موارد زیر است:

- `clean`: هیچ مشکل مسدودکننده‌ای پیدا نشده است.
- `suspicious`: انتشار به احتیاط یا بازبینی نیاز دارد.
- `malicious`: انتشار ناامن در نظر گرفته می‌شود.
- `pending`: بررسی‌ها هنوز تمام نشده‌اند.
- `held`، `quarantined`، `revoked`، یا `hidden`: انتشار به طور کامل در سطح‌های نصب عمومی
  در دسترس نیست.

عبارت دقیق ممکن است بسته به سطح متفاوت باشد، اما معنای عملی همان است: اگر یک
انتشار نگه داشته یا مسدود شده باشد، کاربران نباید آن را نصب کنند تا زمانی که مالک
مشکل را برطرف کند یا تعدیل محتوا آن را بازگرداند.

## Skills

اسکن‌های مهارت، بسته مهارت منتشرشده، فراداده، نیازمندی‌های اعلام‌شده
و دستورالعمل‌های مشکوک را بررسی می‌کنند.

ClawHub به ناهماهنگی بین چیزی که یک مهارت اعلام می‌کند و کاری که به نظر می‌رسد
انجام می‌دهد توجه ویژه دارد. برای مثال، مهارتی که به یک کلید API لازم اشاره می‌کند
باید آن نیازمندی را در `SKILL.md` اعلام کند تا کاربران بتوانند پیش از نصب آن را ببینند.

یافته‌های اسکن مبتنی بر artifact هستند. رفتار مورد انتظار provider، مانند
اعتبارنامه‌های API اعلام‌شده، callbackهای OAuth روی localhost، پاک‌سازی uninstall
محدودشده، رمزگذاری Basic Auth، یا upload فایل‌های انتخاب‌شده توسط کاربر به provider
اعلام‌شده، متفاوت از ارسال پنهانی اعتبارنامه، دسترسی گسترده به فایل‌های خصوصی،
مقصدهای شبکه نامرتبط، یا سوءاستفاده پنهانی از مرورگر تلقی می‌شود.

ببینید: [قالب مهارت](/fa/clawhub/skill-format).

## Pluginها

انتشارهای Plugin شامل فراداده بسته، انتساب منبع، فیلدهای سازگاری،
و اطلاعات یکپارچگی artifact هستند.

OpenClaw پیش از نصب Pluginهای میزبانی‌شده در ClawHub سازگاری را بررسی می‌کند. رکوردهای
بسته همچنین ممکن است فراداده digest را ارائه کنند تا OpenClaw بتواند artifactهای
دانلودشده را راستی‌آزمایی کند. ClawScan هنگام بازبینی انتشارهای Plugin، فراداده
env/config اعلام‌شده بسته `openclaw.environment` را شامل می‌کند تا نیازمندی‌های
runtime اعلام‌شده با رفتار مشاهده‌شده مقایسه شوند.

## گزارش‌ها

کاربران واردشده می‌توانند مهارت‌ها، بسته‌ها و نظرها را گزارش کنند.

گزارش‌ها باید مشخص و قابل اقدام باشند. سوءاستفاده از گزارش‌دهی می‌تواند خودش
به اقدام علیه حساب منجر شود.

نمونه‌های گزارش:

- فراداده گمراه‌کننده
- نیازمندی‌های اعتبارنامه یا مجوز اعلام‌نشده
- دستورالعمل‌های نصب مشکوک
- نظرهای کلاه‌بردارانه یا جعل هویت
- ثبت‌های با سوءنیت یا سوءاستفاده از علامت تجاری
- محتوایی که [استفاده قابل قبول](/fa/clawhub/acceptable-usage) را نقض می‌کند

## یادداشت‌های ClawScan ناشر

ناشران می‌توانند هنگام انتشار یک مهارت یا Plugin یک یادداشت اختیاری ClawScan ارائه کنند.
این یادداشت برای رفتاری که در غیر این صورت ممکن است غیرعادی به نظر برسد، مانند دسترسی
شبکه، دسترسی native host، یا اعتبارنامه‌های خاص provider، به ClawScan زمینه می‌دهد.

## نگه‌داشت‌های تعدیل محتوا

وقتی اسکنر ایستا یک مهارت بارگذاری‌شده را به عنوان مخرب علامت‌گذاری می‌کند، ناشر
به طور خودکار تحت نگه‌داشت تعدیل محتوا قرار می‌گیرد (`requiresModerationAt` روی
کاربر تنظیم می‌شود). این کار همه مهارت‌های ناشر را پنهان می‌کند، باعث می‌شود
انتشارهای آینده به صورت پنهان شروع شوند، و یک ورودی audit log با عنوان
`user.moderation.auto` ایجاد می‌کند.

یافته‌های مشکوک ایستا به عنوان شواهد فایل/خط برای ناظران نگه داشته می‌شوند،
اما به تنهایی محتوا را پنهان نمی‌کنند یا verdict اسکن عمومی را تعیین نمی‌کنند.
بارگذاری‌های جدید تا زمانی که بازبینی LLM تعیین تکلیف شود در وضعیت بازبینی/در انتظار
می‌مانند. اسکن ایستا فقط برای امضاهای مخرب بلافاصله مسدود می‌کند. hitهای engine
در VirusTotal همچنان به عنوان شواهد امنیتی قابل مشاهده می‌مانند، اما verdictهای
VirusTotal Code Insight/Palm مشورتی هستند و به تنهایی مهارت‌ها را پنهان نمی‌کنند.
بازبینی‌های LLM در ClawScan یادداشت‌های همسو با هدف را به عنوان راهنما نگه می‌دارند.
یافته‌های بازبینی متوسط روی artifact قابل مشاهده می‌مانند، در حالی که فیلتر مشکوک
برای نگرانی‌های پراثر LLM، یافته‌های مخرب، یا شناسایی‌های تأییدشده توسط engineهای AV
رزرو شده است.

ادمین‌ها می‌توانند یک نگه‌داشت مثبت کاذب را بردارند:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

این کار `requiresModerationAt` و `requiresModerationReason` را پاک می‌کند، مهارت‌های
پنهان‌شده توسط نگه‌داشت سطح کاربر را بازمی‌گرداند، و یک ورودی audit log با عنوان
`user.moderation.lift` می‌نویسد. مهارت‌هایی که به دلایل دیگر پنهان شده‌اند، یا اسکن
ایستای خودشان همچنان مخرب است، پنهان می‌مانند.

## مسدودسازی‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض کنند ممکن است دسترسی انتشار را از دست بدهند.
سوءاستفاده شدید می‌تواند به مسدود شدن حساب، لغو token، پنهان شدن محتوا، یا حذف
فهرست‌ها منجر شود.

حساب‌های حذف‌شده، مسدودشده یا غیرفعال نمی‌توانند از tokenهای API در ClawHub استفاده کنند.
اگر auth در CLI پس از اقدام علیه حساب شروع به شکست کند، برای بررسی وضعیت حساب وارد
رابط وب شوید. اگر ورود یا دسترسی معمول CLI مسدود شده است، برای بازبینی بازیابی با
security@openclaw.ai تماس بگیرید.

## راهنمای ناشر

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربر:

- نام‌ها، خلاصه‌ها، tagها و changelogها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- وقتی یک انتشار رفتار غیرعادی اما عمدی دارد، یک یادداشت ClawScan ناشر اضافه کنید
- از دستورهای نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع لینک دهید
- پیش از انتشار Pluginها از dry run استفاده کنید
- اگر کاربران یا ناظران درباره رفتار بسته سؤال کردند، شفاف پاسخ دهید
