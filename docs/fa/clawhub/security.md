# امنیت

مسائل امنیتی ClawHub را می‌توان از طریق مشاوره‌های امنیتی GitHub برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از مشاوره‌های امنیتی GitHub استفاده کنید. گزارش‌های خوب
مشاوره امنیتی ClawHub شامل باگ‌ها در این موارد هستند:

- وب‌سایت، API، یا CLI مربوط به ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها، یا یکپارچگی آرتیفکت‌ها
- احراز هویت، مجوزدهی، یا توکن‌های API
- اسکن، نظارت، یا رسیدگی به گزارش‌ها

از مشاوره‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خود یک مهارت یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیما به ناشر یا مخزن منبعی
گزارش کنید که از فهرست ClawHub پیوند شده است.

## افشای آسیب‌پذیری

چون ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌طور پیش‌فرض به‌صورت عمومی افشا نمی‌شوند. وقتی شواهدی از تأثیر واقعی بر
کاربران وجود داشته باشد یا کاربران لازم باشد اقدامی انجام دهند، آن‌ها به‌صورت
عمومی افشا می‌شوند.

نمونه‌های تأثیر واقعی بر کاربران شامل بهره‌برداری تأییدشده، افشای داده‌ها یا
اسرار کاربران، رسیدن محتوای مخرب به کاربران به‌دلیل خرابی پلتفرم، یا هر
مسئله‌ای است که کاربران را ملزم می‌کند اعتبارنامه‌ها را بچرخانند، نرم‌افزار
محلی را به‌روزرسانی کنند، یا اقدام حفاظتی دیگری انجام دهند.

آسیب‌پذیری‌های موجود در نرم‌افزار نصب‌شده توسط کاربر، مانند بسته‌های CLI،
باینری‌ها، کتابخانه‌ها، یا سایر آرتیفکت‌های انتشار ClawHub که کاربران باید
به‌صورت محلی به‌روزرسانی کنند، به‌صورت عمومی افشا می‌شوند.

## صفحه‌های مرتبط

برای برچسب‌های ممیزی هنگام نصب، سطوح ریسک، یافته‌ها، و تفسیر، به
[ممیزی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

برای گزارش‌های بازارچه، توقف‌های نظارتی، فهرست‌های پنهان، مسدودسازی‌ها، و
وضعیت حساب، به [نظارت و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
