# امنیت + تعدیل محتوا

ClawHub برای انتشار باز است، اما فهرست‌های عمومی همچنان از کنترل‌های اعتماد،
اسکن، گزارش‌دهی و تعدیل محتوا عبور می‌کنند. هدف عملی است: کمک به کاربران برای
بررسی آنچه نصب می‌کنند، دادن مسیر بازیابی به ناشران برای موارد مثبت کاذب،
و دور نگه داشتن بسته‌های سوءاستفاده‌گر از کشف عمومی.

همچنین ببینید [استفاده قابل قبول](/fa/clawhub/acceptable-usage).

## آنچه کاربران می‌توانند بررسی کنند

پیش از نصب یک skill یا Plugin، فهرست ClawHub آن را برای موارد زیر بررسی کنید:

- مالک و انتساب منبع
- آخرین نسخه و changelog
- متغیرهای محیطی یا مجوزهای لازم
- فراداده سازگاری برای Pluginها
- وضعیت اسکن یا تعدیل محتوا
- گزارش‌ها، دیدگاه‌ها، ستاره‌ها، دانلودها و سیگنال‌های نصب در جاهایی که نمایش داده می‌شوند

فقط محتوایی را نصب کنید که آن را می‌فهمید و به آن اعتماد دارید.

## وضعیت‌های اسکن

ClawHub ممکن است نتایج اسکن یا تعدیل محتوا را در صفحه‌های عمومی و عیب‌یابی‌های
قابل مشاهده برای مالک نشان دهد.

نتایج رایج شامل موارد زیر است:

- `clean`: هیچ مشکل مسدودکننده‌ای پیدا نشد.
- `suspicious`: این انتشار نیازمند احتیاط یا بازبینی است.
- `malicious`: این انتشار ناامن محسوب می‌شود.
- `pending`: بررسی‌ها هنوز تمام نشده‌اند.
- `held`، `quarantined`، `revoked` یا `hidden`: این انتشار به‌طور کامل روی
  سطوح نصب عمومی در دسترس نیست.

واژگان دقیق ممکن است بسته به سطح متفاوت باشد، اما معنای عملی یکسان است: اگر یک
انتشار نگه داشته یا مسدود شده باشد، کاربران نباید آن را نصب کنند تا زمانی که
مالک مشکل را حل کند یا تعدیل محتوا آن را بازگرداند.

## Skills

اسکن‌های skill به بسته skill منتشرشده، فراداده، نیازمندی‌های اعلام‌شده،
و دستورالعمل‌های مشکوک نگاه می‌کنند.

ClawHub به ناهماهنگی میان آنچه یک skill اعلام می‌کند و آنچه ظاهرا انجام می‌دهد
توجه ویژه دارد. برای مثال، skillای که به یک کلید API لازم اشاره می‌کند باید آن
نیازمندی را در `SKILL.md` اعلام کند تا کاربران بتوانند پیش از نصب آن را ببینند.

یافته‌های اسکن مبتنی بر artifact هستند. رفتار مورد انتظار provider، مانند
اعتبارنامه‌های API اعلام‌شده، callbackهای OAuth روی localhost، پاک‌سازی حذف نصب
محدود به دامنه، رمزگذاری Basic Auth، یا بارگذاری فایل‌های انتخاب‌شده توسط کاربر
به provider اعلام‌شده، متفاوت از ارسال پنهانی اعتبارنامه‌ها، دسترسی گسترده به
فایل‌های خصوصی، مقصدهای شبکه نامرتبط، یا سوءاستفاده پنهانی از مرورگر تلقی
می‌شود.

ببینید [قالب skill](/fa/clawhub/skill-format).

## Pluginها

انتشارهای Plugin شامل فراداده بسته، انتساب منبع، فیلدهای سازگاری،
و اطلاعات یکپارچگی artifact هستند.

OpenClaw پیش از نصب Pluginهای میزبانی‌شده در ClawHub، سازگاری را بررسی می‌کند.
رکوردهای بسته همچنین ممکن است فراداده digest را آشکار کنند تا OpenClaw بتواند
artifactهای دانلودشده را تأیید کند. ClawScan هنگام بازبینی انتشارهای Plugin،
فراداده env/config اعلام‌شده بسته `openclaw.environment` را هم شامل می‌کند تا
نیازمندی‌های runtime اعلام‌شده با رفتار مشاهده‌شده مقایسه شوند.

## گزارش‌ها

کاربران واردشده می‌توانند Skills، بسته‌ها و دیدگاه‌ها را گزارش کنند.

گزارش‌ها باید مشخص و قابل اقدام باشند. سوءاستفاده از گزارش‌دهی می‌تواند خود به
اقدام علیه حساب منجر شود.

نمونه‌های گزارش:

- فراداده گمراه‌کننده
- نیازمندی‌های اعلام‌نشده برای اعتبارنامه یا مجوز
- دستورالعمل‌های نصب مشکوک
- دیدگاه‌های کلاهبردارانه یا جعل هویت
- ثبت‌نام‌های بدخواهانه یا سوءاستفاده از علامت تجاری
- محتوایی که [استفاده قابل قبول](/fa/clawhub/acceptable-usage) را نقض می‌کند

## یادداشت‌های ClawScan ناشر

ناشران می‌توانند هنگام انتشار یک skill یا Plugin، یک یادداشت اختیاری ClawScan
ارائه کنند. این یادداشت به ClawScan برای رفتاری که ممکن است در غیر این صورت
غیرمعمول به نظر برسد زمینه می‌دهد، مانند دسترسی شبکه، دسترسی native host، یا
اعتبارنامه‌های مخصوص provider.

## نگه‌داشت‌های تعدیل محتوا

وقتی اسکنر ایستا یک skill بارگذاری‌شده را به‌عنوان مخرب علامت‌گذاری می‌کند،
ناشر به‌طور خودکار تحت نگه‌داشت تعدیل محتوا قرار می‌گیرد (`requiresModerationAt`
روی کاربر تنظیم می‌شود). این کار همه Skills ناشر را پنهان می‌کند، باعث می‌شود
انتشارهای آینده به‌صورت پنهان شروع شوند، و یک ورودی audit log با
`user.moderation.auto` ایجاد می‌کند.

یافته‌های مشکوک ایستا به‌عنوان شواهد فایل/خط برای moderators نگه داشته می‌شوند،
اما به‌تنهایی محتوا را پنهان نمی‌کنند یا رأی اسکن عمومی را تعیین نمی‌کنند.
بارگذاری‌های جدید تا زمانی که بازبینی LLM تعیین تکلیف شود در وضعیت
بازبینی/در انتظار می‌مانند. اسکن ایستا فقط برای امضاهای مخرب بلافاصله مسدود
می‌کند. تشخیص‌های موتور VirusTotal به‌عنوان شواهد امنیتی قابل مشاهده باقی
می‌مانند، اما رأی‌های VirusTotal Code Insight/Palm مشورتی هستند و به‌تنهایی
Skills را پنهان نمی‌کنند. بازبینی‌های ClawScan LLM یادداشت‌های همسو با هدف را
به‌عنوان راهنما نگه می‌دارند. یافته‌های بازبینی متوسط روی artifact قابل مشاهده
می‌مانند، در حالی که فیلتر مشکوک برای نگرانی‌های پراثر LLM، یافته‌های مخرب، یا
تشخیص‌های تأییدشده موتور AV کنار گذاشته شده است.

مدیران می‌توانند یک نگه‌داشت مثبت کاذب را بردارند:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

این کار `requiresModerationAt` و `requiresModerationReason` را پاک می‌کند،
Skills پنهان‌شده توسط نگه‌داشت سطح کاربر را بازمی‌گرداند، و یک ورودی audit log
با `user.moderation.lift` می‌نویسد. Skillsی که به دلایل دیگر پنهان شده‌اند، یا
اسکن ایستای خودشان همچنان مخرب است، پنهان می‌مانند.

## ممنوعیت‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض کنند ممکن است دسترسی انتشار را از دست بدهند.
سوءاستفاده شدید می‌تواند به ممنوعیت حساب، لغو token، محتوای پنهان، یا حذف
فهرست‌ها منجر شود.

حساب‌های حذف‌شده، ممنوع‌شده، یا غیرفعال‌شده نمی‌توانند از tokenهای API
ClawHub استفاده کنند. اگر احراز هویت CLI پس از اقدام روی حساب شروع به شکست
کرد، برای بازبینی وضعیت حساب وارد رابط وب شوید. اگر ورود یا دسترسی عادی CLI
مسدود شده است، برای بازبینی بازیابی با security@openclaw.ai تماس بگیرید.

## راهنمایی ناشر

برای کاهش موارد مثبت کاذب و بهبود اعتماد کاربر:

- نام‌ها، خلاصه‌ها، برچسب‌ها و changelogها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- وقتی یک انتشار رفتار غیرمعمول اما عمدی دارد، یادداشت ClawScan ناشر اضافه کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع لینک دهید
- پیش از انتشار Pluginها از dry run استفاده کنید
- اگر کاربران یا moderators درباره رفتار بسته پرسیدند، روشن پاسخ دهید
