# امنیت

مشکلات امنیتی ClawHub را می‌توان از طریق GitHub Security Advisories برای
`openclaw/clawhub` گزارش کرد.

از GitHub Security Advisories برای آسیب‌پذیری‌های خود ClawHub استفاده کنید. گزارش‌های
هشدار امنیتی خوب برای ClawHub شامل خطاهایی در موارد زیر هستند:

- وب‌سایت، API، یا CLI مربوط به ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها، یا یکپارچگی آرتیفکت‌ها
- احراز هویت، مجوزدهی، یا توکن‌های API
- اسکن، مدیریت محتوا، یا رسیدگی به گزارش‌ها

از هشدارهای ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ یک مهارت یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیما به ناشر یا مخزن منبعی گزارش
کنید که از فهرست ClawHub به آن لینک شده است.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌طور پیش‌فرض به‌صورت عمومی افشا نمی‌شوند. آن‌ها زمانی به‌صورت عمومی افشا
می‌شوند که شواهدی از تاثیر واقعی بر کاربر وجود داشته باشد یا کاربران لازم باشد
اقدامی انجام دهند.

نمونه‌هایی از تاثیر واقعی بر کاربر شامل بهره‌برداری تاییدشده، افشای داده‌ها یا
اسرار کاربران، رسیدن محتوای مخرب به کاربران به‌دلیل خرابی پلتفرم، یا هر مشکلی
است که کاربران را ملزم می‌کند اعتبارنامه‌ها را چرخش دهند، نرم‌افزار محلی را
به‌روزرسانی کنند، یا اقدام حفاظتی دیگری انجام دهند.

آسیب‌پذیری‌های موجود در نرم‌افزار نصب‌شده توسط کاربر به‌صورت عمومی افشا می‌شوند،
مانند بسته‌های CLI مربوط به ClawHub، باینری‌ها، کتابخانه‌ها، یا سایر آرتیفکت‌های
انتشار که کاربران باید آن‌ها را به‌صورت محلی به‌روزرسانی کنند.

## صفحه‌های مرتبط

برای برچسب‌های ممیزی هنگام نصب، سطوح ریسک، یافته‌ها، و تفسیر، به
[ممیزی‌های امنیتی](/fa/clawhub/security-audits) مراجعه کنید.

برای گزارش‌های بازارچه، نگه‌داشتن در انتظار مدیریت محتوا، فهرست‌های پنهان،
ممنوعیت‌ها، و وضعیت حساب، به [مدیریت محتوا و ایمنی حساب](/fa/clawhub/moderation)
مراجعه کنید.
