# امنیت

مسائل امنیتی ClawHub را می‌توان از طریق GitHub Security Advisories برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از GitHub Security Advisories استفاده کنید. گزارش‌های
خوبِ مشاوره امنیتی ClawHub شامل باگ‌ها در این موارد هستند:

- وب‌سایت، API یا CLI مربوط به ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها یا یکپارچگی آرتیفکت
- احراز هویت، مجوزدهی یا توکن‌های API
- اسکن، نظارت یا رسیدگی به گزارش‌ها

از مشاوره‌های امنیتی ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ یک مهارت یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبعی گزارش کنید
که از فهرست ClawHub لینک شده است.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌طور پیش‌فرض به‌صورت عمومی افشا نمی‌شوند. زمانی به‌صورت عمومی افشا می‌شوند که
شواهدی از تأثیر واقعی بر کاربر وجود داشته باشد یا کاربران لازم باشد اقدامی انجام دهند.

نمونه‌های تأثیر واقعی بر کاربر شامل بهره‌برداری تأییدشده، افشای داده‌ها یا اسرار
کاربر، رسیدن محتوای مخرب به کاربران به‌دلیل شکست پلتفرم، یا هر مسئله‌ای است که
کاربران را ملزم کند اعتبارنامه‌ها را چرخش دهند، نرم‌افزار محلی را به‌روزرسانی کنند یا
اقدام محافظتی دیگری انجام دهند.

آسیب‌پذیری‌های موجود در نرم‌افزار نصب‌شده توسط کاربر به‌صورت عمومی افشا می‌شوند،
مانند بسته‌های CLI، باینری‌ها، کتابخانه‌ها یا دیگر آرتیفکت‌های انتشار ClawHub که
کاربران باید به‌صورت محلی به‌روزرسانی کنند.

## صفحه‌های مرتبط

برای برچسب‌های ممیزی هنگام نصب، سطح‌های ریسک، یافته‌ها و تفسیر، به
[ممیزی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

برای گزارش‌های بازارچه، توقف‌های نظارتی، فهرست‌های پنهان، مسدودسازی‌ها و وضعیت
حساب، به [نظارت و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
