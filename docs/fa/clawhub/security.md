# امنیت + نظارت

ClawHub برای انتشار باز است، اما فهرست‌های عمومی همچنان از کنترل‌های اعتماد،
اسکن، گزارش‌دهی و نظارت عبور می‌کنند. هدف عملی است: کمک به کاربران برای
بررسی آنچه نصب می‌کنند، ارائه مسیر بازیابی به منتشرکنندگان برای مثبت‌های کاذب،
و دور نگه داشتن بسته‌های سوءاستفاده‌گر از کشف عمومی.

همچنین [استفاده قابل قبول](/fa/clawhub/acceptable-usage) را ببینید.

## آنچه کاربران می‌توانند بررسی کنند

پیش از نصب یک مهارت یا Plugin، فهرست ClawHub آن را از نظر موارد زیر بررسی کنید:

- مالک و انتساب منبع
- آخرین نسخه و changelog
- متغیرهای محیطی یا مجوزهای موردنیاز
- فراداده سازگاری برای Pluginها
- وضعیت اسکن یا نظارت
- گزارش‌ها، دیدگاه‌ها، ستاره‌ها، دانلودها و نشانه‌های نصب در صورت نمایش

فقط محتوایی را نصب کنید که آن را می‌فهمید و به آن اعتماد دارید.

## وضعیت‌های اسکن

ClawHub ممکن است نتایج اسکن یا نظارت را در صفحه‌های عمومی و عیب‌یابی‌های قابل مشاهده برای مالک
نمایش دهد.

نتایج رایج شامل موارد زیر است:

- `clean`: هیچ مشکل مسدودکننده‌ای پیدا نشد.
- `suspicious`: این انتشار نیازمند احتیاط یا بازبینی است.
- `malicious`: این انتشار ناامن تلقی می‌شود.
- `pending`: بررسی‌ها هنوز تمام نشده‌اند.
- `held`، `quarantined`، `revoked`، یا `hidden`: این انتشار به‌طور کامل
  در سطح‌های نصب عمومی در دسترس نیست.

عبارت دقیق ممکن است بسته به سطح متفاوت باشد، اما معنای عملی یکسان است: اگر یک
انتشار نگه داشته یا مسدود شده باشد، کاربران نباید آن را نصب کنند تا زمانی که مالک
مشکل را حل کند یا نظارت آن را بازیابی کند.

## Skills

اسکن‌های مهارت، بسته مهارت منتشرشده، فراداده، نیازمندی‌های اعلام‌شده،
و دستورالعمل‌های مشکوک را بررسی می‌کنند.

ClawHub به ناهماهنگی میان آنچه یک مهارت اعلام می‌کند و آنچه به نظر می‌رسد انجام می‌دهد
توجه ویژه دارد. برای مثال، مهارتی که به یک کلید API موردنیاز اشاره می‌کند
باید آن نیازمندی را در `SKILL.md` اعلام کند تا کاربران پیش از
نصب بتوانند آن را ببینند.

یافته‌های اسکن مبتنی بر artifact هستند. رفتار مورد انتظار provider، مانند
اعتبارنامه‌های API اعلام‌شده، callbackهای OAuth در localhost، پاک‌سازی محدودشده هنگام حذف نصب،
کدگذاری Basic Auth، یا بارگذاری فایل‌های انتخاب‌شده توسط کاربر به provider اعلام‌شده،
با ارسال پنهانی اعتبارنامه‌ها، دسترسی گسترده به فایل‌های خصوصی،
مقصدهای شبکه نامرتبط، یا سوءاستفاده پنهانی از مرورگر متفاوت در نظر گرفته می‌شود.

[قالب مهارت](/fa/clawhub/skill-format) را ببینید.

## Pluginها

انتشارهای Plugin شامل فراداده بسته، انتساب منبع، فیلدهای سازگاری،
و اطلاعات یکپارچگی artifact هستند.

OpenClaw پیش از نصب Pluginهای میزبانی‌شده در ClawHub، سازگاری را بررسی می‌کند. رکوردهای بسته
ممکن است فراداده digest را نیز نمایش دهند تا OpenClaw بتواند artifactهای دانلودشده را
راستی‌آزمایی کند. ClawScan هنگام بازبینی انتشارهای Plugin، فراداده env/config اعلام‌شده
در `openclaw.environment` بسته را شامل می‌کند تا نیازمندی‌های runtime اعلام‌شده
با رفتار مشاهده‌شده مقایسه شوند.

## گزارش‌ها

کاربران واردشده می‌توانند مهارت‌ها، بسته‌ها و دیدگاه‌ها را گزارش کنند.

گزارش‌ها باید مشخص و قابل اقدام باشند. سوءاستفاده از گزارش‌دهی می‌تواند خود به
اقدام علیه حساب منجر شود.

نمونه‌های گزارش:

- فراداده گمراه‌کننده
- نیازمندی‌های اعلام‌نشده برای اعتبارنامه یا مجوز
- دستورالعمل‌های نصب مشکوک
- دیدگاه‌های کلاهبردارانه یا جعل هویت
- ثبت‌نام‌های با سوءنیت یا سوءاستفاده از علامت تجاری
- محتوایی که [استفاده قابل قبول](/fa/clawhub/acceptable-usage) را نقض می‌کند

## یادداشت‌های ClawScan منتشرکننده

منتشرکنندگان می‌توانند هنگام انتشار یک مهارت یا Plugin، یک یادداشت اختیاری ClawScan ارائه کنند.
این یادداشت به ClawScan برای رفتاری که ممکن است در غیر این صورت غیرمعمول به نظر برسد
زمینه می‌دهد، مانند دسترسی شبکه، دسترسی host بومی، یا اعتبارنامه‌های ویژه provider.

## نگه‌داشت‌های نظارت

وقتی اسکنر ایستا یک مهارت بارگذاری‌شده را به‌عنوان مخرب علامت‌گذاری می‌کند، منتشرکننده
به‌طور خودکار تحت نگه‌داشت نظارتی قرار می‌گیرد (`requiresModerationAt` روی
کاربر تنظیم می‌شود). این کار همه مهارت‌های منتشرکننده را پنهان می‌کند، باعث می‌شود انتشارهای آینده
به‌صورت پنهان شروع شوند، و یک ورودی گزارش audit با عنوان `user.moderation.auto` ایجاد می‌کند.

یافته‌های مشکوک ایستا به‌عنوان شواهد فایل/خط برای ناظران نگه داشته می‌شوند،
اما به‌تنهایی محتوا را پنهان نمی‌کنند یا verdict اسکن عمومی را تعیین نمی‌کنند.
بارگذاری‌های جدید تا زمان قطعی شدن بازبینی LLM در وضعیت بازبینی/در انتظار باقی می‌مانند. اسکن
ایستا فقط برای امضاهای مخرب فوراً مسدود می‌کند. hitهای موتور VirusTotal
همچنان به‌عنوان شواهد امنیتی قابل مشاهده باقی می‌مانند، اما verdictهای VirusTotal Code Insight/Palm
مشورتی هستند و به‌تنهایی مهارت‌ها را پنهان نمی‌کنند. بازبینی‌های LLM در ClawScan
یادداشت‌های همسو با هدف را به‌عنوان راهنما نگه می‌دارند. یافته‌های بازبینی متوسط روی
artifact قابل مشاهده باقی می‌مانند، در حالی که فیلتر مشکوک برای نگرانی‌های LLM
با اثرگذاری بالا، یافته‌های مخرب، یا تشخیص‌های تاییدشده موتورهای AV نگه داشته می‌شود.

مدیران می‌توانند یک نگه‌داشت مثبت کاذب را بردارند:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

این کار `requiresModerationAt` و `requiresModerationReason` را پاک می‌کند، مهارت‌های
پنهان‌شده به‌دلیل نگه‌داشت در سطح کاربر را بازیابی می‌کند، و یک ورودی گزارش audit
با عنوان `user.moderation.lift` می‌نویسد. مهارت‌هایی که به دلایل دیگر پنهان شده‌اند،
یا اسکن ایستای خودشان همچنان مخرب است، پنهان باقی می‌مانند.

## ممنوعیت‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفاده شدید
می‌تواند به ممنوعیت حساب، لغو توکن، محتوای پنهان، یا حذف
فهرست‌ها منجر شود.

حساب‌های حذف‌شده، ممنوع‌شده، یا غیرفعال‌شده نمی‌توانند از توکن‌های API ClawHub استفاده کنند. اگر احراز هویت CLI
پس از اقدام علیه حساب شروع به خطا دادن کرد، برای بررسی وضعیت حساب
وارد رابط کاربری وب شوید. اگر ورود یا دسترسی معمول CLI مسدود است، برای بازبینی بازیابی با
security@openclaw.ai تماس بگیرید.

## راهنمای منتشرکننده

برای کاهش مثبت‌های کاذب و بهبود اعتماد کاربر:

- نام‌ها، خلاصه‌ها، برچسب‌ها، و changelogها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای موردنیاز را اعلام کنید
- وقتی یک انتشار رفتاری غیرمعمول اما عمدی دارد، یک یادداشت ClawScan منتشرکننده اضافه کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع پیوند دهید
- پیش از انتشار Pluginها از اجرای آزمایشی استفاده کنید
- اگر کاربران یا ناظران درباره رفتار بسته سؤال کردند، شفاف پاسخ دهید
