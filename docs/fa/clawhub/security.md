# امنیت

مشکلات امنیتی ClawHub را می‌توان از طریق مشاوره‌های امنیتی GitHub برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از مشاوره‌های امنیتی GitHub استفاده کنید. گزارش‌های
مشاوره‌ای خوب برای ClawHub شامل باگ‌ها در موارد زیر هستند:

- وب‌سایت، API، یا CLI مربوط به ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها، یا یکپارچگی آرتیفکت‌ها
- احراز هویت، مجوزدهی، یا توکن‌های API
- اسکن، تعدیل، یا رسیدگی به گزارش‌ها

از مشاوره‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ Skills یا Plugin
شخص ثالث استفاده نکنید. آن موارد را مستقیماً به ناشر یا مخزن منبعی گزارش کنید که
از فهرست ClawHub پیوند داده شده است.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌طور پیش‌فرض به‌صورت عمومی افشا نمی‌شوند. زمانی به‌صورت عمومی افشا می‌شوند که
شواهدی از تأثیر واقعی بر کاربر وجود داشته باشد یا کاربران لازم باشد اقدامی انجام دهند.

نمونه‌هایی از تأثیر واقعی بر کاربر شامل بهره‌برداری تأییدشده، افشای داده‌ها یا اسرار
کاربر، رسیدن محتوای مخرب به کاربران به‌دلیل شکست پلتفرم، یا هر مشکلی است که کاربران را
ملزم به چرخش اعتبارنامه‌ها، به‌روزرسانی نرم‌افزار محلی، یا انجام اقدام محافظتی دیگر می‌کند.

آسیب‌پذیری‌های موجود در نرم‌افزار نصب‌شده توسط کاربر، مانند بسته‌های CLI ClawHub،
باینری‌ها، کتابخانه‌ها، یا سایر آرتیفکت‌های انتشار که کاربران باید به‌صورت محلی
به‌روزرسانی کنند، به‌صورت عمومی افشا می‌شوند.

## صفحه‌های مرتبط

برای برچسب‌های ممیزی هنگام نصب، سطوح ریسک، یافته‌ها، و تفسیر، [ممیزی‌های امنیتی](/clawhub/security-audits) را ببینید.

برای گزارش‌های بازارچه، توقف‌های تعدیل، فهرست‌های پنهان، ممنوعیت‌ها، و وضعیت حساب،
[تعدیل و ایمنی حساب](/clawhub/moderation) را ببینید.
