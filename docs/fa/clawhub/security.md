# امنیت

مشکلات امنیتی ClawHub را می‌توان از طریق GitHub Security Advisories برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از GitHub Security Advisories استفاده کنید. گزارش‌های
امنیتی خوب ClawHub شامل باگ‌هایی در این موارد هستند:

- وب‌سایت، API، یا CLI مربوط به ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها، یا یکپارچگی مصنوعات
- احراز هویت، مجوزدهی، یا توکن‌های API
- اسکن، نظارت محتوایی، یا رسیدگی به گزارش‌ها

از هشدارهای امنیتی ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ یک skill یا
plugin شخص ثالث استفاده نکنید. این موارد را مستقیماً به ناشر یا مخزن منبعی که از
فهرست ClawHub لینک شده است گزارش کنید.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌طور پیش‌فرض به‌صورت عمومی افشا نمی‌شوند. این موارد زمانی به‌صورت عمومی افشا
می‌شوند که شواهدی از اثر واقعی بر کاربران وجود داشته باشد یا کاربران لازم باشد
اقدامی انجام دهند.

نمونه‌های اثر واقعی بر کاربران شامل بهره‌برداری تأییدشده، افشای داده‌ها یا
اسرار کاربران، رسیدن محتوای مخرب به کاربران به‌دلیل نقص پلتفرم، یا هر مشکلی است
که کاربران را ملزم به چرخش اعتبارنامه‌ها، به‌روزرسانی نرم‌افزار محلی، یا انجام
اقدام محافظتی دیگری کند.

آسیب‌پذیری‌های موجود در نرم‌افزار نصب‌شده توسط کاربر، مانند بسته‌های CLI مربوط
به ClawHub، باینری‌ها، کتابخانه‌ها، یا دیگر مصنوعات انتشار که کاربران باید
به‌صورت محلی به‌روزرسانی کنند، به‌صورت عمومی افشا می‌شوند.

## صفحه‌های مرتبط

برای برچسب‌های ممیزی هنگام نصب، سطح‌های ریسک، یافته‌ها، و تفسیر، به
[ممیزی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

برای گزارش‌های بازارچه، نگه‌داشت‌های نظارت محتوایی، فهرست‌های پنهان، مسدودسازی‌ها،
و وضعیت حساب، به [نظارت محتوایی و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
