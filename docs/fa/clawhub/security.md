# امنیت

مشکلات امنیتی ClawHub را می‌توان از طریق مشاوره‌های امنیتی GitHub برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از مشاوره‌های امنیتی GitHub استفاده کنید. گزارش‌های
مشاوره‌ای خوب ClawHub شامل باگ‌هایی در موارد زیر هستند:

- وب‌سایت، API یا CLI مربوط به ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها، یا یکپارچگی آرتیفکت‌ها
- احراز هویت، مجوزدهی، یا توکن‌های API
- اسکن، نظارت محتوایی، یا رسیدگی به گزارش‌ها

از مشاوره‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ یک مهارت یا
Plugin شخص ثالث استفاده نکنید. این موارد را مستقیماً به ناشر یا مخزن منبعی که در
فهرست ClawHub لینک شده است گزارش کنید.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌صورت پیش‌فرض به‌طور عمومی افشا نمی‌شوند. زمانی به‌طور عمومی افشا می‌شوند که
شواهدی از اثر واقعی بر کاربران وجود داشته باشد یا کاربران لازم باشد اقدامی انجام دهند.

نمونه‌هایی از اثر واقعی بر کاربران شامل سوءاستفاده تأییدشده، افشای داده‌ها یا
اسرار کاربران، رسیدن محتوای مخرب به کاربران به‌دلیل نقص پلتفرم، یا هر مشکلی است
که کاربران را ملزم به چرخش اعتبارنامه‌ها، به‌روزرسانی نرم‌افزار محلی، یا انجام
اقدام حفاظتی دیگری کند.

آسیب‌پذیری‌های موجود در نرم‌افزار نصب‌شده توسط کاربر به‌طور عمومی افشا می‌شوند؛
مانند بسته‌های CLI مربوط به ClawHub، باینری‌ها، کتابخانه‌ها، یا سایر آرتیفکت‌های
انتشاری که کاربران باید به‌صورت محلی به‌روزرسانی کنند.

## صفحه‌های مرتبط

برای برچسب‌های ممیزی هنگام نصب، سطوح ریسک، یافته‌ها، و تفسیر، به
[ممیزی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

برای گزارش‌های بازارچه، تعلیق‌های نظارتی، فهرست‌های پنهان، ممنوعیت‌ها، و وضعیت
حساب، به [نظارت محتوایی و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
