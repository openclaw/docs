# امنیت

مشکلات امنیتی ClawHub را می‌توان از طریق GitHub Security Advisories برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از GitHub Security Advisories استفاده کنید. گزارش‌های
مشاوره‌ای خوب ClawHub شامل باگ‌هایی در این موارد هستند:

- وب‌سایت، API، یا CLI ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها، یا یکپارچگی آرتیفکت
- احراز هویت، مجوزدهی، یا توکن‌های API
- اسکن، تعدیل محتوا، یا رسیدگی به گزارش‌ها

از مشاوره‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع خودِ یک skill یا
plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبعی گزارش کنید
که از فهرست ClawHub لینک شده است.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌طور پیش‌فرض به‌صورت عمومی افشا نمی‌شوند. زمانی به‌صورت عمومی افشا می‌شوند که
شواهدی از اثر واقعی بر کاربر وجود داشته باشد یا کاربران لازم باشد اقدامی انجام دهند.

نمونه‌های اثر واقعی بر کاربر شامل بهره‌برداری تأییدشده، افشای داده‌ها یا اسرار
کاربر، رسیدن محتوای مخرب به کاربران به دلیل خرابی پلتفرم، یا هر مشکلی است که
کاربران را ملزم کند اعتبارنامه‌ها را چرخش دهند، نرم‌افزار محلی را به‌روزرسانی کنند،
یا اقدام حفاظتی دیگری انجام دهند.

آسیب‌پذیری‌ها در نرم‌افزار نصب‌شده توسط کاربر به‌صورت عمومی افشا می‌شوند، مانند
بسته‌های CLI، باینری‌ها، کتابخانه‌ها، یا سایر آرتیفکت‌های انتشار ClawHub که کاربران
باید به‌صورت محلی به‌روزرسانی کنند.

## صفحه‌های مرتبط

برای برچسب‌های ممیزی هنگام نصب، سطح‌های ریسک، یافته‌ها، و تفسیر، به
[ممیزی‌های امنیتی](/fa/clawhub/security-audits) مراجعه کنید.

برای گزارش‌های بازارچه، توقف‌های تعدیل محتوا، فهرست‌های پنهان، ممنوعیت‌ها، و وضعیت
حساب، به [تعدیل محتوا و ایمنی حساب](/fa/clawhub/moderation) مراجعه کنید.
