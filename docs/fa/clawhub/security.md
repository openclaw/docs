# امنیت

مشکلات امنیتی ClawHub را می‌توان از طریق GitHub Security Advisories برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از GitHub Security Advisories استفاده کنید. گزارش‌های مشورتی خوب
ClawHub شامل باگ‌هایی در این موارد هستند:

- وب‌سایت، API، یا CLI مربوط به ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها، یا یکپارچگی آرتیفکت‌ها
- احراز هویت، مجوزدهی، یا توکن‌های API
- اسکن، نظارت محتوایی، یا رسیدگی به گزارش‌ها

از گزارش‌های مشورتی ClawHub برای آسیب‌پذیری‌های موجود در کد منبع یک Skills یا
Plugin شخص ثالث استفاده نکنید. این موارد را مستقیما به ناشر یا مخزن منبعی
گزارش کنید که از فهرست ClawHub لینک شده است.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌صورت پیش‌فرض عمومی افشا نمی‌شوند. این موارد زمانی به‌صورت عمومی افشا می‌شوند
که شواهدی از تاثیر واقعی بر کاربر وجود داشته باشد یا کاربران لازم باشد اقدامی انجام دهند.

نمونه‌های تاثیر واقعی بر کاربر شامل بهره‌برداری تاییدشده، افشای داده‌ها یا اسرار کاربر،
رسیدن محتوای مخرب به کاربران به‌دلیل خرابی پلتفرم، یا هر مشکلی است که کاربران را
ملزم به چرخش اعتبارنامه‌ها، به‌روزرسانی نرم‌افزار محلی، یا انجام اقدام محافظتی دیگری کند.

آسیب‌پذیری‌های موجود در نرم‌افزار نصب‌شده توسط کاربر به‌صورت عمومی افشا می‌شوند، مانند
بسته‌های CLI مربوط به ClawHub، باینری‌ها، کتابخانه‌ها، یا سایر آرتیفکت‌های انتشار که کاربران
باید آن‌ها را به‌صورت محلی به‌روزرسانی کنند.

## صفحه‌های مرتبط

برای برچسب‌های ممیزی هنگام نصب، سطح‌های ریسک، یافته‌ها، و تفسیر، به
[ممیزی‌های امنیتی](/fa/clawhub/security-audits) مراجعه کنید.

برای گزارش‌های بازارچه، توقف‌های نظارت محتوایی، فهرست‌های پنهان، مسدودسازی‌ها، و وضعیت حساب،
به [نظارت محتوایی و ایمنی حساب](/fa/clawhub/moderation) مراجعه کنید.
