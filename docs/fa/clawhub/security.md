# امنیت

مشکلات امنیتی ClawHub را می‌توان از طریق GitHub Security Advisories برای
`openclaw/clawhub` گزارش کرد.

برای آسیب‌پذیری‌های خود ClawHub از GitHub Security Advisories استفاده کنید. گزارش‌های
مشاوره‌ای خوب ClawHub شامل باگ‌هایی در این موارد هستند:

- وب‌سایت، API یا CLI مربوط به ClawHub
- انتشار در رجیستری، دانلودها، نصب‌ها یا یکپارچگی آرتیفکت‌ها
- احراز هویت، مجوزدهی یا توکن‌های API
- اسکن، مدیریت محتوا یا رسیدگی به گزارش‌ها

از مشاوره‌های ClawHub برای آسیب‌پذیری‌های موجود در کد منبع متعلق به یک skill یا
Plugin شخص ثالث استفاده نکنید. آن‌ها را مستقیماً به ناشر یا مخزن منبعی که از فهرست
ClawHub پیوند شده است گزارش کنید.

## افشای آسیب‌پذیری

از آنجا که ClawHub یک برنامه ابری میزبانی‌شده است، آسیب‌پذیری‌های سرویس ClawHub
به‌طور پیش‌فرض عمومی افشا نمی‌شوند. زمانی عمومی افشا می‌شوند که شواهدی از تأثیر
واقعی بر کاربر وجود داشته باشد یا کاربران لازم باشد اقدامی انجام دهند.

نمونه‌های تأثیر واقعی بر کاربر شامل بهره‌برداری تأییدشده، افشای داده‌ها یا اسرار
کاربر، رسیدن محتوای مخرب به کاربران به دلیل نقص پلتفرم، یا هر مشکلی است که
کاربران را ملزم به چرخش اعتبارنامه‌ها، به‌روزرسانی نرم‌افزار محلی یا انجام اقدام
محافظتی دیگری کند.

آسیب‌پذیری‌های موجود در نرم‌افزار نصب‌شده توسط کاربر، مانند بسته‌های CLI مربوط به
ClawHub، باینری‌ها، کتابخانه‌ها یا سایر آرتیفکت‌های انتشار که کاربران باید به‌صورت
محلی به‌روزرسانی کنند، به‌صورت عمومی افشا می‌شوند.

## صفحه‌های مرتبط

برای برچسب‌های ممیزی زمان نصب، سطوح ریسک، یافته‌ها و تفسیر، به
[ممیزی‌های امنیتی](/clawhub/security-audits) مراجعه کنید.

برای گزارش‌های بازارچه، نگه‌داشت‌های مدیریت محتوا، فهرست‌های پنهان، ممنوعیت‌ها و
وضعیت حساب، به [مدیریت محتوا و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
