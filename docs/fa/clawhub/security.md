# امنیت + مدیریت محتوا

ClawHub برای انتشار باز است، اما فهرست‌های عمومی همچنان از کنترل‌های اعتماد،
اسکن، گزارش‌دهی، و مدیریت محتوا عبور می‌کنند. هدف عملی است: کمک به کاربران
برای بررسی آنچه نصب می‌کنند، فراهم کردن مسیر بازیابی برای ناشران در موارد مثبت کاذب،
و دور نگه داشتن بسته‌های سوءاستفاده‌گر از کشف عمومی.

همچنین [کاربری قابل قبول](/fa/clawhub/acceptable-usage) را ببینید.

## آنچه کاربران می‌توانند بررسی کنند

پیش از نصب یک Skills یا Plugin، فهرست ClawHub آن را برای موارد زیر بررسی کنید:

- مالک و انتساب منبع
- آخرین نسخه و changelog
- متغیرهای محیطی یا مجوزهای لازم
- فراداده سازگاری برای Pluginها
- وضعیت اسکن یا مدیریت محتوا
- گزارش‌ها، دیدگاه‌ها، ستاره‌ها، دانلودها، و سیگنال‌های نصب در جاهایی که نمایش داده می‌شوند

فقط محتوایی را نصب کنید که آن را می‌فهمید و به آن اعتماد دارید.

## وضعیت‌های اسکن

ClawHub ممکن است نتایج اسکن یا مدیریت محتوا را در صفحه‌های عمومی و عیب‌یابی‌های
قابل مشاهده برای مالک نمایش دهد.

نتایج رایج شامل موارد زیر است:

- `clean`: هیچ مشکل مسدودکننده‌ای پیدا نشد.
- `suspicious`: انتشار نیازمند احتیاط یا بازبینی است.
- `malicious`: انتشار ناامن تلقی می‌شود.
- `pending`: بررسی‌ها هنوز تمام نشده‌اند.
- `held`، `quarantined`، `revoked`، یا `hidden`: انتشار به طور کامل در سطح‌های
  نصب عمومی در دسترس نیست.

عبارت دقیق ممکن است بسته به سطح متفاوت باشد، اما معنای عملی یکسان است: اگر
انتشاری نگه داشته یا مسدود شده باشد، کاربران نباید آن را نصب کنند تا زمانی که مالک
مشکل را رفع کند یا مدیریت محتوا آن را بازگرداند.

## Skills

اسکن‌های Skills بسته منتشرشده Skills، فراداده، الزامات اعلام‌شده،
و دستورالعمل‌های مشکوک را بررسی می‌کنند.

ClawHub به ناسازگاری بین آنچه یک Skills اعلام می‌کند و آنچه به نظر می‌رسد انجام می‌دهد
توجه ویژه دارد. برای مثال، Skillsی که به یک کلید API لازم اشاره می‌کند
باید آن نیازمندی را در `SKILL.md` اعلام کند تا کاربران بتوانند پیش از
نصب آن را ببینند.

یافته‌های اسکن مبتنی بر artifact هستند. رفتار مورد انتظار provider، مانند اعتبارنامه‌های
API اعلام‌شده، بازفراخوانی‌های OAuth روی localhost، پاک‌سازی حذف نصب محدودشده، کدگذاری Basic Auth،
یا بارگذاری فایل‌های انتخاب‌شده توسط کاربر به provider اعلام‌شده، با ارسال پنهانی
اعتبارنامه‌ها، دسترسی گسترده به فایل‌های خصوصی، مقصدهای شبکه نامرتبط، یا سوءاستفاده پنهانی
از مرورگر متفاوت برخورد می‌شود.

[قالب Skills](/fa/clawhub/skill-format) را ببینید.

## Pluginها

انتشارهای Plugin شامل فراداده بسته، انتساب منبع، فیلدهای سازگاری،
و اطلاعات یکپارچگی artifact هستند.

OpenClaw پیش از نصب Pluginهای میزبانی‌شده در ClawHub سازگاری را بررسی می‌کند. رکوردهای بسته
ممکن است فراداده digest را نیز در معرض نمایش بگذارند تا OpenClaw بتواند artifactهای
دانلودشده را راستی‌آزمایی کند. ClawScan هنگام بازبینی انتشارهای Plugin، فراداده env/config
اعلام‌شده در `openclaw.environment` بسته را شامل می‌کند تا الزامات runtime اعلام‌شده
با رفتار مشاهده‌شده مقایسه شوند.

## گزارش‌ها

کاربران واردشده می‌توانند Skills، بسته‌ها، و دیدگاه‌ها را گزارش کنند.

گزارش‌ها باید مشخص و قابل اقدام باشند. سوءاستفاده از گزارش‌دهی خود می‌تواند به
اقدام علیه حساب منجر شود.

نمونه‌های گزارش:

- فراداده گمراه‌کننده
- الزامات اعلام‌نشده اعتبارنامه یا مجوز
- دستورالعمل‌های نصب مشکوک
- دیدگاه‌های کلاهبردارانه یا جعل هویت
- ثبت‌نام‌های با سوءنیت یا سوءاستفاده از نشان تجاری
- محتوایی که [کاربری قابل قبول](/fa/clawhub/acceptable-usage) را نقض می‌کند

## یادداشت‌های ClawScan ناشر

ناشران هنگام انتشار یک Skills یا Plugin می‌توانند یک یادداشت اختیاری ClawScan ارائه کنند.
این یادداشت به ClawScan برای رفتاری که ممکن است در غیر این صورت غیرعادی به نظر برسد،
مانند دسترسی شبکه، دسترسی native host، یا اعتبارنامه‌های اختصاصی provider،
زمینه می‌دهد.

## نگه‌داشت‌های مدیریت محتوا

وقتی اسکنر ایستا یک Skills بارگذاری‌شده را به عنوان مخرب علامت‌گذاری می‌کند، ناشر
به طور خودکار تحت نگه‌داشت مدیریت محتوا قرار می‌گیرد (`requiresModerationAt` روی
کاربر تنظیم می‌شود). این کار همه Skillsهای ناشر را پنهان می‌کند، باعث می‌شود انتشارهای آینده
به صورت پنهان شروع شوند، و یک ورودی audit log با `user.moderation.auto` ایجاد می‌کند.

یافته‌های مشکوک ایستا به عنوان شواهد فایل/خط برای مدیران محتوا نگه داشته می‌شوند،
اما به‌تنهایی محتوا را پنهان نمی‌کنند یا رأی اسکن عمومی را تعیین نمی‌کنند.
بارگذاری‌های جدید تا زمان نهایی شدن بازبینی LLM در وضعیت بازبینی/pending باقی می‌مانند. اسکن
ایستا فقط برای امضاهای مخرب بلافاصله مسدود می‌کند. اصابت‌های موتور VirusTotal
همچنان به عنوان شواهد امنیتی قابل مشاهده باقی می‌مانند، اما رأی‌های VirusTotal Code Insight/Palm
مشورتی هستند و به‌تنهایی Skills را پنهان نمی‌کنند. بازبینی‌های ClawScan LLM
یادداشت‌های همسو با هدف را به عنوان راهنما نگه می‌دارند. یافته‌های بازبینی متوسط روی
artifact قابل مشاهده باقی می‌مانند، در حالی که فیلتر مشکوک برای نگرانی‌های LLM
با اثرگذاری بالا، یافته‌های مخرب، یا تشخیص‌های تأییدشده موتور AV رزرو شده است.

مدیران می‌توانند یک نگه‌داشت مثبت کاذب را بردارند:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

این کار `requiresModerationAt` و `requiresModerationReason` را پاک می‌کند، Skillsهای
پنهان‌شده توسط نگه‌داشت سطح کاربر را بازمی‌گرداند، و یک ورودی audit log با
`user.moderation.lift` می‌نویسد. Skillsهایی که به دلایل دیگر پنهان شده‌اند، یا اسکن
ایستای خودشان همچنان مخرب است، پنهان می‌مانند.

## مسدودسازی‌ها و وضعیت حساب

حساب‌هایی که سیاست ClawHub را نقض کنند ممکن است دسترسی انتشار را از دست بدهند. سوءاستفاده شدید
می‌تواند به مسدودسازی حساب، ابطال token، محتوای پنهان، یا حذف
فهرست‌ها منجر شود.

حساب‌های حذف‌شده، مسدودشده، یا غیرفعال نمی‌توانند از tokenهای API ClawHub استفاده کنند. اگر احراز هویت CLI
پس از اقدام علیه حساب شروع به خطا دادن کرد، برای بازبینی وضعیت حساب
به UI وب وارد شوید. اگر ورود یا دسترسی معمول CLI مسدود است، برای بازبینی بازیابی با
security@openclaw.ai تماس بگیرید.

## راهنمایی ناشر

برای کاهش مثبت‌های کاذب و افزایش اعتماد کاربر:

- نام‌ها، خلاصه‌ها، tagها، و changelogها را دقیق نگه دارید
- متغیرهای محیطی و مجوزهای لازم را اعلام کنید
- وقتی یک انتشار رفتار غیرمعمول اما عمدی دارد، یادداشت ClawScan ناشر اضافه کنید
- از فرمان‌های نصب مبهم‌سازی‌شده پرهیز کنید
- در صورت امکان به منبع پیوند دهید
- پیش از انتشار Pluginها از dry run استفاده کنید
- اگر کاربران یا مدیران محتوا درباره رفتار بسته پرسیدند، روشن پاسخ دهید
