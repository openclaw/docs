# استفاده قابل قبول

ClawHub میزبان Skills، plugins، بسته‌ها و فراداده بازارچه برای OpenClaw است.
از این صفحه استفاده کنید تا تصمیم بگیرید آیا محتوا یا رفتار انتشار به
ClawHub تعلق دارد یا نه.

این قواعد درباره کاری که یک مدخل انجام می‌دهد، چیزی که از کاربران می‌خواهد اجرا کنند، نحوه معرفی خودش، و نحوه استفاده ناشران از سطوح کشف، نصب، و اعتماد ClawHub اعمال می‌شوند. برای وضعیت‌های نظارت و جایگاه حساب، به
[نظارت و ایمنی حساب](/fa/clawhub/moderation) مراجعه کنید. برای ادعاهای حق نشر یا سایر حقوق،
[درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) را ببینید.

## محتوای مجاز

ClawHub از محتوایی استقبال می‌کند که مفید، قابل فهم، و با حسن نیت منتشر شده باشد.

| دسته | زمانی مجاز است که |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| بهره‌وری توسعه‌دهنده | مدخل به کاربران کمک کند نرم‌افزار بسازند، آزمایش کنند، مهاجرت دهند، اشکال‌زدایی کنند، مستندسازی کنند، یا اجرا و نگهداری کنند. |
| گردش‌کارهای UI، داده، و خودکارسازی | دامنه روشن باشد، اعتبارنامه‌های لازم صریح باشند، و اقدامات پرریسک شامل مسیرهای بازبینی، اجرای آزمایشی، پیش‌نمایش، یا تأیید باشند. |
| امنیت دفاعی، نظارت، و بازبینی سوءاستفاده | ابزار برای بازبینی مجاز تعریف شده باشد، شواهد را حفظ کند، و مرزهای تأیید انسانی را روشن نگه دارد. |
| گردش‌کارهای شخصی یا تیمی | گردش‌کار از حساب‌های مبتنی بر رضایت، راه‌اندازی شفاف، و مجوزهای صریح استفاده کند. |
| کاتالوگ‌های نگهداری‌شده | هر مدخل متمایز، مفید، دقیقاً توصیف‌شده، و به‌طور معقول نگهداری‌شده باشد. |

زمینه مهم است. یک موضوع یکسان می‌تواند در محیطی محدود، دفاعی، یا مبتنی بر رضایت قابل قبول باشد و وقتی به‌عنوان گردش‌کار سوءاستفاده بسته‌بندی شود غیرقابل قبول باشد.

| دسته | مجاز نیست |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| سوءاستفاده از پلتفرم و دور زدن ممنوعیت | حساب‌های پنهان‌کار پس از ممنوعیت، آماده‌سازی یا مزرعه‌سازی حساب، تعامل جعلی، خودکارسازی چندحسابی، ارسال انبوه، ربات‌های اسپم، یا خودکارسازی ساخته‌شده برای اجتناب از شناسایی. |
| تقلب، کلاهبرداری، و گردش‌کارهای مالی فریبکارانه | گواهی‌ها یا صورت‌حساب‌های جعلی، جریان‌های پرداخت فریبکارانه، ارتباطات کلاهبردارانه، اثبات اجتماعی جعلی، گردش‌کارهای هویت مصنوعی برای تقلب، یا ابزارهای خرج کردن/دریافت وجه بدون تأیید انسانی روشن. |
| غنی‌سازی یا نظارت ناقض حریم خصوصی | برداشت مخاطبان برای اسپم، افشای اطلاعات شخصی، تعقیب، استخراج سرنخ همراه با ارتباطات ناخواسته، پایش پنهانی، تطبیق زیست‌سنجی بدون رضایت، یا استفاده از داده‌های افشاشده یا دامپ‌های رخنه. |

| الزامات اجرای پنهان، ناامن، یا گمراه‌کننده | فرمان‌های نصب مبهم‌سازی‌شده، نصب‌کننده‌های pipe-to-shell مانند محتوای دانلودشده‌ای که بدون قابلیت بازبینی روشن با `sh` یا `bash` اجرا می‌شود، الزامات اعلام‌نشده برای رازها یا کلیدهای خصوصی، اجرای راه‌دور `npx @latest` بدون قابلیت بازبینی روشن، یا فراداده‌ای که آنچه مدخل واقعاً برای اجرا نیاز دارد را پنهان می‌کند. |

## رفتار غیرمجاز در بازارچه

ClawHub همچنین نحوه استفاده ناشران از بازارچه را بررسی می‌کند. از ClawHub برای دست‌کاری کشف، سنجه‌ها، نشانه‌های اعتماد، سامانه‌های نظارت، یا توجه کاربر استفاده نکنید.

رفتار غیرمجاز در بازارچه شامل این موارد است:

- انتشار انبوه تعداد زیادی مدخل کم‌زحمت، تکراری، جایگزین‌نما، یا
  ماشین‌تولید که به نظر نمی‌رسد ارزش واقعی برای کاربر داشته باشند
- انباشتن سطوح جست‌وجو یا دسته‌بندی با Skills یا plugins تقریباً یکسان
- انتشار صدها مدخل با استفاده، نگهداری، شفافیت منبع، یا تمایز معنادار کم یا صفر
- افزایش مصنوعی نصب‌ها، دانلودها، ستاره‌ها، یا دیگر سنجه‌های تعامل
  از طریق خودکارسازی، حلقه‌های خودنصبی، حساب‌های جعلی، فعالیت هماهنگ،
  تعامل پولی، یا رفتارهای غیرارگانیک دیگر
- ایجاد یا چرخاندن حساب‌ها برای دور زدن نظارت، ممنوعیت‌ها، محدودیت‌های ناشر، یا
  بازبینی بازارچه
- گمراه کردن کاربران درباره مالکیت، منبع، قابلیت‌ها، وضعیت امنیتی،
  الزامات نصب، یا وابستگی به پروژه یا ناشر دیگر
- بارگذاری مکرر محتوایی که قبلاً پنهان، حذف، یا مسدود شده است
  بدون رفع مشکل زیربنایی

انتشار با حجم بالا به‌خودی‌خود سوءاستفاده نیست. کاتالوگ‌های بزرگ زمانی قابل قبول‌اند که مدخل‌ها به‌طور معنادار متفاوت، دقیقاً توصیف‌شده، نگهداری‌شده، و توسط کاربران واقعی استفاده شوند. کاتالوگ‌های بزرگ وقتی به مشکل اعتماد و ایمنی تبدیل می‌شوند که حجم با مدخل‌های کم‌مایه، تکراری، گمراه‌کننده، نگهداری‌نشده، یا به‌طور مصنوعی ترویج‌شده همراه باشد.

## حقوق محتوا

اگر باور دارید محتوایی در ClawHub حق نشر یا سایر حقوق شما را نقض می‌کند، از
[درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) استفاده کنید. برای ادعاهای حق نشر یا حقوق از گزارش‌های عادی بازارچه استفاده نکنید، مگر اینکه مدخل همچنین ناامن، مخرب، یا گمراه‌کننده باشد.

## بازبینی و اجرا

ClawHub ممکن است از بررسی‌های خودکار، نشانه‌های آماری سوءاستفاده، گزارش‌های کاربران، و بازبینی کارکنان برای شناسایی محتوای ناامن یا رفتار انتشار سوءاستفاده‌آمیز استفاده کند. یک نشانه به‌تنهایی سوءاستفاده را ثابت نمی‌کند؛ به ClawHub کمک می‌کند تصمیم بگیرد چه چیزی به بازبینی نیاز دارد.

ممکن است ما:

- مدخل‌های ناقض را پنهان، نگه‌داری، حذف، حذف نرم، یا، در مواردی که برای نوع منبع پشتیبانی می‌شود،
  حذف کامل کنیم
- دانلودها یا نصب‌ها را برای انتشارهای ناامن مسدود کنیم
- توکن‌های API را لغو کنیم
- محتوای مرتبط را حذف نرم کنیم
- دسترسی انتشار را محدود کنیم
- متخلفان تکراری یا شدید را ممنوع کنیم

ما اجرای ابتدا-با-هشدار را برای سوءاستفاده آشکار تضمین نمی‌کنیم. برای گزارش‌ها، نگه‌داشت‌های نظارتی، مدخل‌های پنهان، ممنوعیت‌ها، و جایگاه حساب، به
[نظارت و ایمنی حساب](/fa/clawhub/moderation) مراجعه کنید.
