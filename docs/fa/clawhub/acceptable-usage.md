# استفاده قابل قبول

ClawHub میزبان Skills، Pluginها، بسته‌ها و فراداده بازارچه برای OpenClaw است.
از این صفحه برای تصمیم‌گیری درباره اینکه آیا محتوا یا رفتار انتشار به
ClawHub تعلق دارد یا نه استفاده کنید.

این قوانین درباره کاری که یک فهرست انجام می‌دهد، چیزی که از کاربران می‌خواهد اجرا کنند، نحوه
معرفی خودش، و نحوه استفاده ناشران از سطوح کشف، نصب و اعتماد ClawHub اعمال می‌شوند. برای وضعیت‌های نظارت و اعتبار حساب، به
[نظارت و ایمنی حساب](/clawhub/moderation) مراجعه کنید. برای ادعاهای حق نشر یا سایر حقوق،
[درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) را ببینید.

## محتوای مجاز

ClawHub از محتوایی استقبال می‌کند که مفید، قابل فهم و با حسن نیت منتشر شده باشد.

| دسته                                         | مجاز است وقتی                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| بهره‌وری توسعه‌دهنده                           | فهرست به کاربران کمک کند نرم‌افزار بسازند، آزمایش کنند، مهاجرت دهند، اشکال‌زدایی کنند، مستندسازی کنند یا بهره‌برداری کنند.                                               |
| گردش‌کارهای رابط کاربری، داده و خودکارسازی               | دامنه روشن باشد، اعتبارنامه‌های لازم صریح باشند، و اقدامات پرخطر مسیرهای بازبینی، اجرای آزمایشی، پیش‌نمایش یا تأیید داشته باشند. |
| امنیت دفاعی، نظارت و بازبینی سوءاستفاده | ابزار برای بازبینی مجاز تعریف شده باشد، شواهد را حفظ کند و مرزهای تأیید انسانی را روشن نگه دارد.                          |
| گردش‌کارهای شخصی یا تیمی                       | گردش‌کار از حساب‌های مبتنی بر رضایت، راه‌اندازی شفاف و مجوزهای صریح استفاده کند.                                            |
| کاتالوگ‌های نگهداری‌شده                              | هر فهرست متمایز، مفید، دقیقاً توصیف‌شده و به شکل معقولی نگهداری‌شده باشد.                                                |

زمینه اهمیت دارد. یک موضوع یکسان می‌تواند در یک محیط دفاعی محدود یا
مبتنی بر رضایت قابل قبول باشد و وقتی به‌عنوان گردش‌کار سوءاستفاده بسته‌بندی شود غیرقابل قبول باشد.

## رفتار غیرمجاز در بازارچه

ClawHub همچنین نحوه استفاده ناشران از بازارچه را بازبینی می‌کند. از ClawHub برای
دستکاری کشف، معیارها، سیگنال‌های اعتماد، سامانه‌های نظارت یا توجه کاربر
استفاده نکنید.

رفتار غیرمجاز در بازارچه شامل این موارد است:

- انتشار انبوه تعداد زیادی فهرست کم‌زحمت، تکراری، جای‌نگهدار یا
  تولیدشده با ماشین که به نظر نمی‌رسد ارزش واقعی برای کاربر داشته باشند
- پر کردن سطوح جست‌وجو یا دسته‌بندی با Skills یا Pluginهای تقریباً یکسان
- انتشار صدها فهرست با استفاده، نگهداری، شفافیت منبع
  یا تمایز معنادار اندک یا صفر
- افزایش مصنوعی نصب‌ها، دانلودها، ستاره‌ها یا سایر معیارهای تعامل
  از طریق خودکارسازی، حلقه‌های خودنصب، حساب‌های جعلی، فعالیت هماهنگ،
  تعامل پولی یا سایر رفتارهای غیرارگانیک
- ایجاد یا چرخاندن حساب‌ها برای دور زدن نظارت، ممنوعیت‌ها، محدودیت‌های ناشر یا
  بازبینی بازارچه
- گمراه‌کردن کاربران درباره مالکیت، منبع، قابلیت‌ها، وضعیت امنیتی،
  الزامات نصب، یا وابستگی به پروژه یا ناشر دیگر
- بارگذاری مکرر محتوایی که قبلاً پنهان، حذف یا مسدود شده است
  بدون رفع مسئله زیربنایی

انتشار پرحجم به‌طور خودکار سوءاستفاده نیست. کاتالوگ‌های بزرگ وقتی قابل قبول‌اند
که فهرست‌ها به‌طور معنادار متفاوت، دقیقاً توصیف‌شده، نگهداری‌شده
و توسط کاربران واقعی استفاده‌شده باشند. کاتالوگ‌های بزرگ زمانی به مسئله اعتماد و ایمنی تبدیل می‌شوند که
حجم با فهرست‌های کم‌محتوا، تکراری، گمراه‌کننده، نگهداری‌نشده یا
به‌طور مصنوعی تبلیغ‌شده همراه شود.

## حقوق محتوا

اگر باور دارید محتوایی در ClawHub حق نشر یا سایر حقوق شما را نقض می‌کند، از
[درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) استفاده کنید. برای ادعاهای حق نشر یا حقوق از گزارش‌های عادی بازارچه
استفاده نکنید، مگر اینکه فهرست همچنین ناایمن،
مخرب یا گمراه‌کننده باشد.

## بازبینی و اجرا

ClawHub ممکن است از بررسی‌های خودکار، سیگنال‌های آماری سوءاستفاده، گزارش‌های کاربران و
بازبینی کارکنان برای شناسایی محتوای ناایمن یا رفتار انتشار سوءاستفاده‌آمیز استفاده کند. یک سیگنال
به‌تنهایی سوءاستفاده را ثابت نمی‌کند؛ به ClawHub کمک می‌کند تصمیم بگیرد چه چیزی نیازمند بازبینی است.

ما ممکن است:

- فهرست‌های ناقض را پنهان کنیم، نگه داریم، حذف کنیم، حذف نرم کنیم، یا، جایی که برای نوع منبع پشتیبانی می‌شود،
  حذف سخت کنیم
- دانلودها یا نصب‌ها را برای انتشارهای ناایمن مسدود کنیم
- توکن‌های API را باطل کنیم
- محتوای مرتبط را حذف نرم کنیم
- دسترسی انتشار را محدود کنیم
- متخلفان تکراری یا شدید را ممنوع کنیم

ما اجرای مبتنی بر هشدار اولیه را برای سوءاستفاده آشکار تضمین نمی‌کنیم. برای گزارش‌ها، نگه‌داشت‌های نظارتی،
فهرست‌های پنهان، ممنوعیت‌ها و اعتبار حساب، به
[نظارت و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
