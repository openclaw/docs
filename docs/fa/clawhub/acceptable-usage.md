# استفاده قابل قبول

ClawHub میزبان Skills، Pluginها، بسته‌ها و فراداده بازارچه برای OpenClaw است.
از این صفحه استفاده کنید تا تصمیم بگیرید آیا محتوا یا رفتار انتشار به
ClawHub تعلق دارد یا نه.

این قوانین درباره کاری که یک فهرست انجام می‌دهد، چیزی که از کاربران می‌خواهد اجرا کنند، نحوه
معرفی خودش، و نحوه استفاده ناشران از سطوح کشف، نصب و
اعتماد ClawHub اعمال می‌شوند. برای وضعیت‌های نظارت و وضعیت حساب، به
[نظارت و ایمنی حساب](/clawhub/moderation) مراجعه کنید. برای ادعاهای حق نشر یا حقوق دیگر،
به [درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) مراجعه کنید.

## محتوای مجاز

ClawHub از محتوایی استقبال می‌کند که مفید، قابل فهم، و با حسن نیت منتشر شده باشد.

| دسته‌بندی | مجاز است وقتی |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| بهره‌وری توسعه‌دهنده | فهرست به کاربران کمک می‌کند نرم‌افزار را بسازند، آزمایش کنند، مهاجرت دهند، اشکال‌زدایی کنند، مستند کنند، یا عملیاتی نگه دارند. |
| جریان‌های کاری UI، داده و خودکارسازی | دامنه روشن است، اعتبارنامه‌های لازم صریح هستند، و اقدامات پرریسک شامل مسیرهای بازبینی، اجرای آزمایشی، پیش‌نمایش، یا تأیید هستند. |
| امنیت دفاعی، نظارت، و بازبینی سوءاستفاده | ابزار برای بازبینی مجاز چارچوب‌بندی شده، شواهد را حفظ می‌کند، و مرزهای تأیید انسانی را روشن نگه می‌دارد. |
| جریان‌های کاری شخصی یا تیمی | جریان کاری از حساب‌های مبتنی بر رضایت، راه‌اندازی شفاف، و مجوزهای صریح استفاده می‌کند. |
| کاتالوگ‌های نگهداری‌شده | هر فهرست متمایز، مفید، دقیق توصیف‌شده، و به شکل معقولی نگهداری‌شده است. |

زمینه اهمیت دارد. یک موضوع یکسان می‌تواند در یک محیط محدود دفاعی یا
مبتنی بر رضایت قابل قبول باشد و وقتی به‌عنوان جریان کاری سوءاستفاده بسته‌بندی شود غیرقابل قبول باشد.

## محتوای غیرمجاز

ClawHub محتوایی را میزبانی نمی‌کند که هدف اصلی آن سوءاستفاده، فریب، اجرای ناامن،
یا نقض حقوق باشد.

| دسته‌بندی | غیرمجاز |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| دسترسی غیرمجاز یا دور زدن امنیت | دور زدن احراز هویت، تصاحب حساب، سوءاستفاده از محدودیت نرخ، تصاحب تماس زنده یا عامل، سرقت نشست قابل استفاده مجدد، یا تأیید خودکار جریان‌های جفت‌سازی برای کاربران تأییدنشده. |
| سوءاستفاده از پلتفرم و دور زدن مسدودسازی | حساب‌های مخفی پس از مسدودسازی، گرم‌کردن یا پرورش حساب، تعامل جعلی، خودکارسازی چندحسابی، ارسال انبوه، ربات‌های هرزنامه، یا خودکارسازی ساخته‌شده برای دور زدن شناسایی. |
| تقلب، کلاهبرداری، و جریان‌های کاری مالی فریبنده | گواهی‌ها یا فاکتورهای جعلی، جریان‌های پرداخت فریبنده، ارتباط‌گیری کلاهبردارانه، اثبات اجتماعی جعلی، جریان‌های کاری هویت مصنوعی برای تقلب، یا ابزارهای خرج‌کردن/شارژکردن بدون تأیید انسانی روشن. |
| غنی‌سازی یا پایش ناقض حریم خصوصی | استخراج مخاطب برای هرزنامه، افشای اطلاعات شخصی، تعقیب، استخراج سرنخ همراه با ارتباط‌گیری ناخواسته، پایش پنهانی، تطبیق زیست‌سنجی بدون رضایت، یا استفاده از داده‌های افشاشده یا دامپ‌های رخنه. |
| جعل هویت یا دست‌کاری هویت بدون رضایت | تعویض چهره، دوقلوهای دیجیتال، اینفلوئنسرهای شبیه‌سازی‌شده، شخصیت‌های جعلی، یا ابزارهای دیگری که برای جعل هویت یا گمراه‌کردن استفاده می‌شوند. |
| محتوای جنسی صریح یا تولید محتوای بزرگسال با ایمنی غیرفعال | تولید تصویر، ویدئو، یا محتوا NSFW؛ پوشش‌های محتوای بزرگسال پیرامون APIهای شخص ثالث؛ یا فهرست‌هایی که هدف اصلی آن‌ها محتوای جنسی صریح است. |
| الزامات اجرای پنهان، ناامن، یا گمراه‌کننده | دستورهای نصب مبهم‌سازی‌شده، نصب‌کننده‌های pipe-to-shell مانند محتوای دانلودشده که با `sh` یا `bash` بدون قابلیت بازبینی روشن اجرا می‌شود، الزامات اعلام‌نشده secret یا private-key، اجرای راه‌دور `npx @latest` بدون قابلیت بازبینی روشن، یا فراداده‌ای که پنهان می‌کند فهرست واقعاً برای اجرا به چه چیزی نیاز دارد. |
| مواد ناقض حق نشر یا ناقض حقوق | بازنشر Skill، Plugin، مستندات، دارایی‌های برند، یا کد اختصاصی شخص دیگر بدون اجازه؛ نقض شرایط مجوز؛ یا جعل هویت نویسنده یا ناشر اصلی. |

## رفتار غیرمجاز بازارچه

ClawHub همچنین نحوه استفاده ناشران از بازارچه را بازبینی می‌کند. از ClawHub برای
دست‌کاری کشف، سنجه‌ها، سیگنال‌های اعتماد، سامانه‌های نظارت، یا توجه کاربر
استفاده نکنید.

رفتار غیرمجاز بازارچه شامل این موارد است:

- انتشار انبوه تعداد زیادی فهرست کم‌تلاش، تکراری، جای‌نگهدار، یا
  تولیدشده توسط ماشین که به نظر نمی‌رسد ارزش واقعی برای کاربر داشته باشند
- اشباع سطوح جست‌وجو یا دسته‌بندی با Skills یا Pluginهای تقریباً یکسان
- انتشار صدها فهرست با استفاده، نگهداری، شفافیت منبع،
  یا تمایز معنادار اندک یا بدون آن
- افزایش مصنوعی نصب‌ها، دانلودها، ستاره‌ها، یا سایر سنجه‌های تعامل
  از طریق خودکارسازی، چرخه‌های نصب توسط خود، حساب‌های جعلی، فعالیت
  هماهنگ، تعامل پولی، یا رفتار غیرارگانیک دیگر
- ایجاد یا چرخاندن حساب‌ها برای دور زدن نظارت، مسدودسازی‌ها، محدودیت‌های ناشر، یا
  بازبینی بازارچه
- گمراه‌کردن کاربران درباره مالکیت، منبع، قابلیت‌ها، وضعیت امنیتی،
  الزامات نصب، یا وابستگی به پروژه یا ناشر دیگر
- بارگذاری مکرر محتوایی که قبلاً پنهان، حذف، یا مسدود شده است
  بدون رفع مسئله زیربنایی

انتشار با حجم بالا به‌طور خودکار سوءاستفاده نیست. کاتالوگ‌های بزرگ زمانی قابل قبول هستند
که فهرست‌ها به‌طور معنادار متفاوت، دقیق توصیف‌شده، نگهداری‌شده،
و توسط کاربران واقعی استفاده‌شده باشند. کاتالوگ‌های بزرگ وقتی به مشکل اعتماد و ایمنی تبدیل می‌شوند که
حجم با فهرست‌های کم‌مایه، تکراری، گمراه‌کننده، نگهداری‌نشده، یا
به‌صورت مصنوعی ترویج‌شده همراه باشد.

## حقوق محتوا

اگر باور دارید محتوایی در ClawHub حق نشر یا سایر حقوق شما را نقض می‌کند، از
[درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) استفاده کنید. برای ادعاهای حق نشر یا حقوق، از گزارش‌های عادی بازارچه
استفاده نکنید مگر اینکه فهرست همچنین ناامن،
مخرب، یا گمراه‌کننده باشد.

## بازبینی و اعمال مقررات

ClawHub ممکن است از بررسی‌های خودکار، سیگنال‌های آماری سوءاستفاده، گزارش‌های کاربران، و
بازبینی کارکنان برای شناسایی محتوای ناامن یا رفتار انتشار سوءاستفاده‌آمیز استفاده کند. یک سیگنال
به‌تنهایی سوءاستفاده را ثابت نمی‌کند؛ به ClawHub کمک می‌کند تصمیم بگیرد چه چیزی نیاز به بازبینی دارد.

ممکن است:

- فهرست‌های ناقض را پنهان کنیم، نگه داریم، حذف کنیم، soft-delete کنیم، یا، در مواردی که برای نوع منبع پشتیبانی می‌شود،
  hard-delete کنیم
- دانلودها یا نصب‌ها را برای انتشارهای ناامن مسدود کنیم
- توکن‌های API را لغو کنیم
- محتوای مرتبط را soft-delete کنیم
- دسترسی انتشار را محدود کنیم
- متخلفان تکراری یا شدید را مسدود کنیم

ما برای سوءاستفاده آشکار، اعمال مقررات با هشدار اولیه را تضمین نمی‌کنیم. برای گزارش‌ها، نگه‌داشت‌های نظارتی،
فهرست‌های پنهان، مسدودسازی‌ها، و وضعیت حساب، به
[نظارت و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
