# استفادهٔ قابل قبول

ClawHub میزبان Skills، Pluginها، بسته‌ها، و فرادادهٔ بازارچه برای OpenClaw است.
از این صفحه استفاده کنید تا تصمیم بگیرید آیا محتوا یا رفتار انتشار به
ClawHub تعلق دارد یا نه.

این قواعد بر کاری که یک فهرست انجام می‌دهد، چیزی که از کاربران می‌خواهد اجرا کنند، نحوهٔ
معرفی خودش، و شیوهٔ استفادهٔ ناشران از سطوح کشف، نصب، و
اعتماد ClawHub اعمال می‌شوند. برای وضعیت‌های نظارت و جایگاه حساب، به
[نظارت و ایمنی حساب](/clawhub/moderation) مراجعه کنید. برای ادعاهای حق نشر یا سایر حقوق،
به [درخواست‌های حقوق محتوا](/clawhub/content-rights) مراجعه کنید.

## محتوای مجاز

ClawHub از محتوایی استقبال می‌کند که مفید، قابل فهم، و با حسن نیت
منتشر شده باشد.

| دسته‌بندی | مجاز است وقتی |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| بهره‌وری توسعه‌دهنده | فهرست به کاربران کمک کند نرم‌افزار بسازند، آزمایش کنند، مهاجرت دهند، اشکال‌زدایی کنند، مستند کنند، یا اجرا و نگهداری کنند. |
| گردش‌کارهای UI، داده، و خودکارسازی | دامنه روشن باشد، اعتبارنامه‌های موردنیاز صریح باشند، و کنش‌های پرخطر مسیرهای بازبینی، اجرای آزمایشی، پیش‌نمایش، یا تأیید داشته باشند. |
| امنیت دفاعی، نظارت، و بازبینی سوءاستفاده | ابزار برای بازبینی مجاز چارچوب‌بندی شده باشد، شواهد را حفظ کند، و مرزهای تأیید انسانی را روشن نگه دارد. |
| گردش‌کارهای شخصی یا تیمی | گردش‌کار از حساب‌های مبتنی بر رضایت، راه‌اندازی شفاف، و مجوزهای صریح استفاده کند. |
| کاتالوگ‌های نگهداری‌شده | هر فهرست متمایز، مفید، دقیقاً توصیف‌شده، و به‌طور معقول نگهداری‌شده باشد. |

زمینه اهمیت دارد. یک موضوع یکسان می‌تواند در یک موقعیت دفاعی محدود یا
مبتنی بر رضایت قابل قبول باشد و وقتی به‌عنوان گردش‌کار سوءاستفاده بسته‌بندی شود غیرقابل قبول باشد.

ClawHub محتوایی را میزبانی نمی‌کند که هدف اصلی آن سوءاستفاده، فریب، اجرای ناامن،
یا نقض حقوق باشد.

| دسته‌بندی | غیرمجاز |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| دسترسی غیرمجاز یا دور زدن امنیت | دور زدن احراز هویت، تصاحب حساب، سوءاستفاده از محدودیت نرخ، تصاحب تماس زنده یا agent، سرقت نشست قابل استفادهٔ مجدد، یا تأیید خودکار جریان‌های جفت‌سازی برای کاربران تأییدنشده. |
| سوءاستفاده از پلتفرم و دور زدن ممنوعیت | حساب‌های مخفی پس از ممنوعیت، گرم‌کردن یا مزرعه‌سازی حساب، تعامل جعلی، خودکارسازی چندحسابی، ارسال انبوه، ربات‌های هرزنامه، یا خودکارسازی ساخته‌شده برای اجتناب از شناسایی. |
| کلاهبرداری، فریب، و گردش‌کارهای مالی گمراه‌کننده | گواهی‌ها یا فاکتورهای جعلی، جریان‌های پرداخت گمراه‌کننده، تماس‌های کلاهبردارانه، اثبات اجتماعی جعلی، گردش‌کارهای هویت مصنوعی برای کلاهبرداری، یا ابزارهای خرج‌کردن/دریافت هزینه بدون تأیید روشن انسانی. |
| غنی‌سازی ناقض حریم خصوصی یا نظارت | خزش مخاطبان برای هرزنامه، افشای اطلاعات شخصی، تعقیب، استخراج سرنخ همراه با تماس ناخواسته، پایش پنهانی، تطبیق زیست‌سنجی بدون رضایت، یا استفاده از داده‌های افشاشده یا dumpهای رخنه. |
| جعل هویت بدون رضایت یا دست‌کاری هویت | تعویض چهره، دوقلوهای دیجیتال، اینفلوئنسرهای شبیه‌سازی‌شده، شخصیت‌های جعلی، یا ابزارهای دیگری که برای جعل هویت یا گمراه‌کردن استفاده می‌شوند. |
| محتوای جنسی صریح یا تولید محتوای بزرگسال با ایمنی غیرفعال | تولید تصویر، ویدئو، یا محتواهای NSFW؛ wrapperهای محتوای بزرگسال پیرامون APIهای شخص ثالث؛ یا فهرست‌هایی که هدف اصلی آن‌ها محتوای جنسی صریح است. |
| الزامات اجرای پنهان، ناامن، یا گمراه‌کننده | فرمان‌های نصب مبهم‌سازی‌شده، نصب‌کننده‌های pipe-to-shell مانند محتوای دانلودشده که با `sh` یا `bash` بدون امکان بازبینی روشن اجرا می‌شود، الزامات اعلام‌نشدهٔ secret یا کلید خصوصی، اجرای راه‌دور `npx @latest` بدون امکان بازبینی روشن، یا فراداده‌ای که نیاز واقعی فهرست برای اجرا را پنهان می‌کند. |
| مطالب ناقض حق نشر یا ناقض حقوق | بازنشر skill، plugin، مستندات، دارایی‌های برند، یا کد اختصاصی شخص دیگر بدون اجازه؛ نقض شروط مجوز؛ یا جعل هویت نویسنده یا ناشر اصلی. |

## رفتار غیرمجاز بازارچه

ClawHub همچنین نحوهٔ استفادهٔ ناشران از بازارچه را بازبینی می‌کند. از ClawHub برای
دست‌کاری کشف، سنجه‌ها، نشانه‌های اعتماد، سامانه‌های نظارت، یا توجه
کاربر استفاده نکنید.

رفتار غیرمجاز بازارچه شامل این موارد است:

- انتشار انبوه تعداد زیادی فهرست کم‌زحمت، تکراری، جای‌نگهدار، یا
  تولیدشده با ماشین که به نظر نمی‌رسد ارزش واقعی برای کاربر داشته باشند
- پر کردن سطوح جست‌وجو یا دسته‌بندی با skills یا plugins تقریباً یکسان
- انتشار صدها فهرست با استفاده، نگهداری، شفافیت منبع،
  یا تمایز معنادار اندک یا بدون آن
- افزایش مصنوعی نصب‌ها، دانلودها، ستاره‌ها، یا سایر سنجه‌های
  تعامل از طریق خودکارسازی، حلقه‌های نصب توسط خود، حساب‌های جعلی، فعالیت
  هماهنگ، تعامل پولی، یا رفتارهای غیرارگانیک دیگر
- ایجاد یا چرخش حساب‌ها برای دور زدن نظارت، ممنوعیت‌ها، محدودیت‌های ناشر، یا
  بازبینی بازارچه
- گمراه‌کردن کاربران دربارهٔ مالکیت، منبع، قابلیت‌ها، وضعیت امنیتی،
  الزامات نصب، یا وابستگی به پروژه یا ناشر دیگر
- بارگذاری مکرر محتوایی که قبلاً پنهان، حذف، یا مسدود شده است
  بدون رفع مشکل زیربنایی

انتشار با حجم بالا به‌طور خودکار سوءاستفاده نیست. کاتالوگ‌های بزرگ وقتی قابل قبول‌اند
که فهرست‌ها به‌طور معناداری متفاوت، دقیقاً توصیف‌شده، نگهداری‌شده،
و توسط کاربران واقعی استفاده شوند. کاتالوگ‌های بزرگ وقتی به مشکل اعتماد و ایمنی تبدیل می‌شوند که
حجم با فهرست‌های کم‌محتوا، تکراری، گمراه‌کننده، نگهداری‌نشده، یا
به‌طور مصنوعی ترویج‌شده همراه باشد.

## حقوق محتوا

اگر باور دارید محتوایی در ClawHub حق نشر یا سایر حقوق شما را نقض می‌کند، از
[درخواست‌های حقوق محتوا](/clawhub/content-rights) استفاده کنید. از گزارش‌های عادی بازارچه
برای ادعاهای حق نشر یا حقوق استفاده نکنید، مگر اینکه فهرست همچنین ناامن،
مخرب، یا گمراه‌کننده باشد.

## بازبینی و اجرا

ClawHub ممکن است از بررسی‌های خودکار، نشانه‌های آماری سوءاستفاده، گزارش‌های کاربر، و
بازبینی کارکنان برای شناسایی محتوای ناامن یا رفتار انتشار سوءاستفاده‌گرانه استفاده کند. یک نشانه
به‌تنهایی سوءاستفاده را ثابت نمی‌کند؛ به ClawHub کمک می‌کند تصمیم بگیرد چه چیزی به بازبینی نیاز دارد.

ممکن است:

- فهرست‌های متخلف را پنهان، نگه‌داری، حذف، حذف نرم، یا، در مواردی که برای نوع منبع پشتیبانی می‌شود،
  حذف سخت کنیم
- دانلودها یا نصب‌ها را برای انتشارهای ناامن مسدود کنیم
- توکن‌های API را لغو کنیم
- محتوای مرتبط را حذف نرم کنیم
- دسترسی انتشار را محدود کنیم
- متخلفان تکراری یا شدید را ممنوع کنیم

ما اجرای اخطار-اول را برای سوءاستفادهٔ آشکار تضمین نمی‌کنیم. برای گزارش‌ها، نگه‌داری‌های نظارتی،
فهرست‌های پنهان، ممنوعیت‌ها، و جایگاه حساب به
[نظارت و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
