# استفاده قابل قبول

ClawHub میزبان Skills، Pluginها، بسته‌ها و فراداده بازارچه برای OpenClaw است.
از این صفحه برای تصمیم‌گیری درباره اینکه محتوا یا رفتار انتشار به
ClawHub تعلق دارد یا نه استفاده کنید.

این قوانین درباره کاری که یک فهرست انجام می‌دهد، آنچه از کاربران می‌خواهد اجرا کنند، نحوه
معرفی خودش، و شیوه استفاده ناشران از سطوح کشف، نصب، و
اعتماد ClawHub اعمال می‌شود. برای وضعیت‌های نظارت و اعتبار حساب، به
[نظارت و ایمنی حساب](/clawhub/moderation) مراجعه کنید. برای ادعاهای حق نشر یا سایر حقوق،
به [درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) مراجعه کنید.

## محتوای مجاز

ClawHub از محتوایی استقبال می‌کند که مفید، قابل فهم، و با حسن نیت منتشر شده باشد.

| دسته‌بندی                                         | مجاز است وقتی                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| بهره‌وری توسعه‌دهنده                           | فهرست به کاربران کمک کند نرم‌افزار بسازند، آزمایش کنند، مهاجرت دهند، اشکال‌زدایی کنند، مستند کنند، یا عملیاتی کنند.                                               |
| گردش‌کارهای UI، داده، و خودکارسازی               | دامنه روشن باشد، اعتبارنامه‌های لازم صریح باشند، و اقدامات پرخطر شامل مسیرهای بازبینی، اجرای آزمایشی، پیش‌نمایش، یا تأیید باشند. |
| امنیت دفاعی، نظارت، و بازبینی سوءاستفاده | ابزار برای بازبینی مجاز چارچوب‌بندی شده باشد، شواهد را حفظ کند، و مرزهای تأیید انسانی را روشن نگه دارد.                          |
| گردش‌کارهای شخصی یا تیمی                       | گردش‌کار از حساب‌های مبتنی بر رضایت، راه‌اندازی شفاف، و مجوزهای صریح استفاده کند.                                            |
| کاتالوگ‌های نگهداری‌شده                              | هر فهرست متمایز، مفید، با توصیف دقیق، و به‌طور معقول نگهداری‌شده باشد.                                                |

زمینه اهمیت دارد. یک موضوع مشابه می‌تواند در یک چارچوب دفاعی محدود یا
مبتنی بر رضایت قابل قبول باشد و وقتی به‌عنوان گردش‌کار سوءاستفاده بسته‌بندی شود غیرقابل قبول باشد.

## رفتار غیرمجاز در بازارچه

ClawHub همچنین نحوه استفاده ناشران از بازارچه را بازبینی می‌کند. از ClawHub برای
دستکاری کشف، معیارها، نشانه‌های اعتماد، سامانه‌های نظارت، یا توجه
کاربر استفاده نکنید.

رفتار غیرمجاز در بازارچه شامل موارد زیر است:

- انتشار انبوه تعداد زیادی فهرست کم‌زحمت، تکراری، جایگزین‌نما، یا
  تولیدشده با ماشین که به نظر نمی‌رسد ارزش واقعی برای کاربر داشته باشند
- سیل‌کردن سطوح جست‌وجو یا دسته‌بندی با Skills یا Pluginهای تقریباً یکسان
- انتشار صدها فهرست با استفاده، نگهداری، شفافیت منبع،
  یا تمایز معنادار اندک یا بدون آن
- افزایش مصنوعی نصب‌ها، دانلودها، ستاره‌ها، یا سایر معیارهای
  تعامل از طریق خودکارسازی، حلقه‌های خودنصبی، حساب‌های جعلی، فعالیت
  هماهنگ، تعامل پولی، یا رفتار غیرارگانیک دیگر
- ایجاد یا چرخاندن حساب‌ها برای دور زدن نظارت، ممنوعیت‌ها، محدودیت‌های ناشر، یا
  بازبینی بازارچه
- گمراه‌کردن کاربران درباره مالکیت، منبع، قابلیت‌ها، وضعیت امنیتی،
  الزامات نصب، یا وابستگی به پروژه یا ناشر دیگر
- بارگذاری مکرر محتوایی که قبلاً پنهان، حذف، یا مسدود شده است
  بدون رفع مشکل زیربنایی

انتشار با حجم بالا به‌طور خودکار سوءاستفاده نیست. کاتالوگ‌های بزرگ زمانی قابل قبول‌اند
که فهرست‌ها به‌طور معناداری متفاوت، با توصیف دقیق، نگهداری‌شده،
و مورد استفاده کاربران واقعی باشند. کاتالوگ‌های بزرگ وقتی به مشکل اعتماد و ایمنی تبدیل می‌شوند که
حجم با فهرست‌های کم‌مایه، تکراری، گمراه‌کننده، نگهداری‌نشده، یا
به‌طور مصنوعی ترویج‌شده همراه باشد.

## حقوق محتوا

اگر باور دارید محتوایی در ClawHub حق نشر یا سایر حقوق شما را نقض می‌کند، از
[درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) استفاده کنید. از گزارش‌های عادی بازارچه
برای ادعاهای حق نشر یا حقوق استفاده نکنید مگر اینکه فهرست همچنین ناامن،
مخرب، یا گمراه‌کننده باشد.

## بازبینی و اجرا

ClawHub ممکن است از بررسی‌های خودکار، نشانه‌های آماری سوءاستفاده، گزارش‌های کاربران، و
بازبینی کارکنان برای شناسایی محتوای ناامن یا رفتار انتشار سوءاستفاده‌گرانه استفاده کند. یک نشانه
به‌تنهایی سوءاستفاده را ثابت نمی‌کند؛ به ClawHub کمک می‌کند تصمیم بگیرد چه چیزی نیاز به بازبینی دارد.

ممکن است:

- فهرست‌های ناقض را پنهان کنیم، نگه داریم، حذف کنیم، نرم‌حذف کنیم، یا، در جایی که برای نوع منبع پشتیبانی می‌شود،
  سخت‌حذف کنیم
- دانلودها یا نصب‌ها را برای انتشارهای ناامن مسدود کنیم
- توکن‌های API را لغو کنیم
- محتوای مرتبط را نرم‌حذف کنیم
- دسترسی انتشار را محدود کنیم
- متخلفان تکراری یا شدید را ممنوع کنیم

ما اجرای ابتدا-با-هشدار را برای سوءاستفاده آشکار تضمین نمی‌کنیم. برای گزارش‌ها، توقف‌های نظارتی،
فهرست‌های پنهان، ممنوعیت‌ها، و اعتبار حساب، به
[نظارت و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
