# استفادهٔ قابل قبول

ClawHub میزبان Skills، Pluginها، بسته‌ها و فرادادهٔ بازارچه برای OpenClaw است.
از این صفحه استفاده کنید تا تصمیم بگیرید آیا محتوا یا رفتار انتشار به
ClawHub تعلق دارد یا نه.

این قوانین دربارهٔ این است که یک مدخل چه کاری انجام می‌دهد، از کاربران می‌خواهد چه چیزی را اجرا کنند، چگونه
خود را معرفی می‌کند، و ناشران چگونه از سطوح کشف، نصب و اعتماد
ClawHub استفاده می‌کنند. برای وضعیت‌های نظارت و اعتبار حساب، به
[نظارت و ایمنی حساب](/clawhub/moderation) مراجعه کنید. برای ادعاهای حق نشر یا سایر حقوق،
[درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) را ببینید.

## محتوای مجاز

ClawHub از محتوایی استقبال می‌کند که مفید، قابل فهم، و با حسن نیت منتشر شده باشد.

| دسته | مجاز است وقتی |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| بهره‌وری توسعه‌دهنده | مدخل به کاربران کمک کند نرم‌افزار بسازند، آزمایش کنند، مهاجرت دهند، اشکال‌زدایی کنند، مستندسازی کنند، یا اجرا و نگهداری کنند. |
| گردش‌کارهای UI، داده، و خودکارسازی | دامنه روشن باشد، اعتبارنامه‌های لازم صریح باشند، و اقدامات پرخطر شامل مسیرهای بازبینی، اجرای آزمایشی، پیش‌نمایش، یا تأیید باشند. |
| امنیت دفاعی، نظارت، و بررسی سوءاستفاده | ابزار برای بازبینی مجاز چارچوب‌بندی شده باشد، شواهد را حفظ کند، و مرزهای تأیید انسانی را روشن نگه دارد. |
| گردش‌کارهای شخصی یا تیمی | گردش‌کار از حساب‌های مبتنی بر رضایت، راه‌اندازی شفاف، و مجوزهای صریح استفاده کند. |
| کاتالوگ‌های نگهداری‌شده | هر مدخل متمایز، مفید، دقیقاً توصیف‌شده، و به‌طور معقول نگهداری‌شده باشد. |

زمینه اهمیت دارد. یک موضوع واحد می‌تواند در یک چارچوب دفاعی محدود یا
مبتنی بر رضایت قابل قبول باشد و وقتی به‌عنوان گردش‌کار سوءاستفاده بسته‌بندی شود غیرقابل قبول باشد.

| دسته | مجاز نیست |

| سوءاستفاده از پلتفرم و دور زدن ممنوعیت | حساب‌های مخفی پس از ممنوعیت، گرم‌کردن یا پرورش حساب، تعامل جعلی، خودکارسازی چندحسابی، ارسال انبوه، ربات‌های هرزنامه، یا خودکارسازی ساخته‌شده برای اجتناب از شناسایی. |

| غنی‌سازی یا پایش ناقض حریم خصوصی | استخراج مخاطبان برای هرزنامه، افشای اطلاعات شخصی، تعقیب، استخراج سرنخ همراه با ارتباط‌گیری ناخواسته، پایش پنهانی، تطبیق زیست‌سنجی بدون رضایت، یا استفاده از داده‌های نشت‌کرده یا دامپ‌های رخنه. |

## رفتار غیرمجاز در بازارچه

ClawHub همچنین بررسی می‌کند ناشران چگونه از بازارچه استفاده می‌کنند. از ClawHub برای
دستکاری کشف، معیارها، نشانه‌های اعتماد، سامانه‌های نظارت، یا توجه کاربران
استفاده نکنید.

رفتار غیرمجاز در بازارچه شامل این موارد است:

- انتشار انبوه تعداد زیادی مدخل کم‌زحمت، تکراری، جای‌نگهدار، یا
  تولیدشده با ماشین که به‌نظر نمی‌رسد ارزش واقعی برای کاربر داشته باشند
- پر کردن سطوح جست‌وجو یا دسته‌بندی با Skills یا Pluginهای تقریباً یکسان
- انتشار صدها مدخل با استفاده، نگهداری، وضوح منبع،
  یا تمایز معنادار اندک یا بدون آن
- افزایش مصنوعی نصب‌ها، دانلودها، ستاره‌ها، یا سایر معیارهای تعامل
  از طریق خودکارسازی، حلقه‌های خودنصب، حساب‌های جعلی، فعالیت هماهنگ،
  تعامل پولی، یا سایر رفتارهای غیرارگانیک
- ایجاد یا چرخاندن حساب‌ها برای دور زدن نظارت، ممنوعیت‌ها، محدودیت‌های ناشر، یا
  بازبینی بازارچه
- گمراه‌کردن کاربران دربارهٔ مالکیت، منبع، قابلیت‌ها، وضعیت امنیتی،
  الزامات نصب، یا وابستگی به پروژه یا ناشر دیگر
- بارگذاری مکرر محتوایی که پیش‌تر پنهان، حذف، یا مسدود شده است
  بدون رفع مشکل زیربنایی

انتشار با حجم بالا به‌طور خودکار سوءاستفاده نیست. کاتالوگ‌های بزرگ زمانی قابل قبول‌اند
که مدخل‌ها به‌طور معنادار متفاوت، دقیقاً توصیف‌شده، نگهداری‌شده،
و مورد استفادهٔ کاربران واقعی باشند. کاتالوگ‌های بزرگ وقتی به مشکل اعتماد و ایمنی تبدیل می‌شوند که
حجم با مدخل‌های کم‌محتوا، تکراری، گمراه‌کننده، نگهداری‌نشده، یا
به‌طور مصنوعی ترویج‌شده همراه باشد.

## حقوق محتوا

اگر باور دارید محتوایی در ClawHub حق نشر یا سایر حقوق شما را نقض می‌کند، از
[درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) استفاده کنید. برای ادعاهای حق نشر یا حقوق، از گزارش‌های عادی بازارچه
استفاده نکنید مگر اینکه مدخل همچنین ناامن،
مخرب، یا گمراه‌کننده باشد.

## بازبینی و اجرا

ClawHub ممکن است از بررسی‌های خودکار، نشانه‌های آماری سوءاستفاده، گزارش‌های کاربران، و
بازبینی کارکنان برای شناسایی محتوای ناامن یا رفتار انتشار سوءاستفاده‌آمیز استفاده کند. یک نشانه
به‌تنهایی سوءاستفاده را اثبات نمی‌کند؛ به ClawHub کمک می‌کند تصمیم بگیرد چه چیزی نیاز به بازبینی دارد.

ممکن است ما:

- مدخل‌های ناقض را پنهان کنیم، نگه داریم، حذف کنیم، نیمه‌حذف کنیم، یا، در مواردی که برای نوع منبع پشتیبانی می‌شود،
  حذف قطعی کنیم
- دانلودها یا نصب‌ها را برای انتشارهای ناامن مسدود کنیم
- توکن‌های API را لغو کنیم
- محتوای مرتبط را نیمه‌حذف کنیم
- دسترسی انتشار را محدود کنیم
- متخلفان تکراری یا شدید را ممنوع کنیم

ما اجرای همراه با هشدار اولیه را برای سوءاستفادهٔ آشکار تضمین نمی‌کنیم. برای گزارش‌ها، توقف‌های نظارتی،
مدخل‌های پنهان، ممنوعیت‌ها، و اعتبار حساب به
[نظارت و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
