# استفادهٔ قابل قبول

ClawHub میزبان Skills، Pluginها، بسته‌ها و فرادادهٔ بازارچه برای OpenClaw است.
از این صفحه برای تصمیم‌گیری دربارهٔ اینکه آیا محتوا یا رفتار انتشار به
ClawHub تعلق دارد یا نه استفاده کنید.

این قواعد دربارهٔ کاری که یک فهرست انجام می‌دهد، چیزی که از کاربران می‌خواهد اجرا کنند، شیوه‌ای که
خود را معرفی می‌کند، و نحوهٔ استفادهٔ ناشران از سطوح کشف، نصب و اعتماد
ClawHub اعمال می‌شوند. برای وضعیت‌های نظارت و وضعیت حساب، به
[نظارت و ایمنی حساب](/clawhub/moderation) مراجعه کنید. برای ادعاهای حق نشر یا سایر حقوق،
به [درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) مراجعه کنید.

## محتوای مجاز

ClawHub از محتوایی استقبال می‌کند که مفید، قابل فهم و با حسن نیت منتشر شده باشد.

| دسته‌بندی | مجاز است وقتی |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| بهره‌وری توسعه‌دهنده | فهرست به کاربران کمک می‌کند نرم‌افزار بسازند، آزمایش کنند، مهاجرت دهند، اشکال‌زدایی کنند، مستندسازی کنند یا اجرا و نگهداری کنند. |
| جریان‌های کاری رابط کاربری، داده و خودکارسازی | دامنه روشن است، اعتبارنامه‌های لازم صریح هستند، و اقدامات پرخطر مسیرهای بازبینی، اجرای آزمایشی، پیش‌نمایش یا تأیید دارند. |
| امنیت دفاعی، نظارت و بازبینی سوءاستفاده | ابزار برای بازبینی مجاز چارچوب‌بندی شده است، شواهد را حفظ می‌کند و مرزهای تأیید انسانی را روشن نگه می‌دارد. |
| جریان‌های کاری شخصی یا تیمی | جریان کاری از حساب‌های مبتنی بر رضایت، راه‌اندازی شفاف و مجوزهای صریح استفاده می‌کند. |
| کاتالوگ‌های نگهداری‌شده | هر فهرست متمایز، مفید، دقیقاً توصیف‌شده و به‌طور معقول نگهداری‌شده است. |

زمینه اهمیت دارد. همان موضوع می‌تواند در یک محیط دفاعی محدود یا
مبتنی بر رضایت قابل قبول باشد و وقتی به‌عنوان جریان کاری سوءاستفاده بسته‌بندی شود غیرقابل قبول باشد.

## محتوای غیرمجاز

ClawHub محتوایی را که هدف اصلی آن سوءاستفاده، فریب، اجرای ناایمن
یا نقض حقوق باشد میزبانی نمی‌کند.

| دسته‌بندی | غیرمجاز |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| دسترسی غیرمجاز یا دور زدن امنیت | دور زدن احراز هویت، تصاحب حساب، سوءاستفاده از محدودیت نرخ، تصاحب تماس زنده یا عامل، سرقت نشست قابل استفادهٔ مجدد، یا تأیید خودکار جریان‌های جفت‌سازی برای کاربران تأییدنشده. |
| سوءاستفاده از پلتفرم و دور زدن ممنوعیت | حساب‌های مخفیانه پس از ممنوعیت، گرم‌کردن یا پرورش حساب، تعامل جعلی، خودکارسازی چندحسابی، ارسال انبوه، ربات‌های هرزنامه، یا خودکارسازی ساخته‌شده برای پرهیز از شناسایی. |
| کلاهبرداری، فریب و جریان‌های کاری مالی گمراه‌کننده | گواهی‌ها یا فاکتورهای جعلی، جریان‌های پرداخت گمراه‌کننده، ارتباط‌گیری کلاهبردارانه، اعتبار اجتماعی جعلی، جریان‌های کاری هویت مصنوعی برای کلاهبرداری، یا ابزارهای خرج‌کردن/شارژکردن بدون تأیید انسانی روشن. |
| غنی‌سازی یا نظارت ناقض حریم خصوصی | استخراج مخاطبان برای هرزنامه، افشای اطلاعات شخصی، تعقیب، استخراج سرنخ همراه با ارتباط‌گیری ناخواسته، پایش پنهانی، تطبیق زیست‌سنجی بدون رضایت، یا استفاده از داده‌های درزکرده یا مجموعه‌داده‌های رخنه. |
| جعل هویت یا دست‌کاری هویت بدون رضایت | تعویض چهره، دوقلوهای دیجیتال، اینفلوئنسرهای شبیه‌سازی‌شده، شخصیت‌های جعلی، یا ابزارهای دیگری که برای جعل هویت یا گمراه‌کردن استفاده می‌شوند. |
| محتوای جنسی صریح یا تولید محتوای بزرگسال با ایمنی غیرفعال | تولید تصویر، ویدئو یا محتوای NSFW؛ پوشش‌های محتوای بزرگسال پیرامون APIهای شخص ثالث؛ یا فهرست‌هایی که هدف اصلی آن‌ها محتوای جنسی صریح است. |
| الزامات اجرای پنهان، ناایمن یا گمراه‌کننده | فرمان‌های نصب مبهم‌سازی‌شده، نصب‌کننده‌های انتقال مستقیم به پوسته مانند محتوای دانلودشده‌ای که بدون بازبینی‌پذیری روشن با `sh` یا `bash` اجرا می‌شود، الزامات اعلام‌نشدهٔ راز یا کلید خصوصی، اجرای راه‌دور `npx @latest` بدون بازبینی‌پذیری روشن، یا فراداده‌ای که نیاز واقعی فهرست برای اجرا را پنهان می‌کند. |
| مواد ناقض حق نشر یا ناقض حقوق | بازنشر Skills، Plugin، مستندات، دارایی‌های برند یا کد اختصاصی شخص دیگر بدون اجازه؛ نقض شرایط مجوز؛ یا جعل هویت نویسنده یا ناشر اصلی. |

## رفتار غیرمجاز بازارچه

ClawHub همچنین نحوهٔ استفادهٔ ناشران از بازارچه را بازبینی می‌کند. از ClawHub برای
دست‌کاری کشف، معیارها، سیگنال‌های اعتماد، سامانه‌های نظارت یا توجه کاربر
استفاده نکنید.

رفتار غیرمجاز بازارچه شامل موارد زیر است:

- انتشار انبوه تعداد زیادی فهرست کم‌تلاش، تکراری، جایگزین‌نما یا
  تولیدشده با ماشین که به نظر نمی‌رسد ارزش واقعی برای کاربر داشته باشند
- پرکردن سطوح جستجو یا دسته‌بندی با Skills یا Pluginهای تقریباً یکسان
- انتشار صدها فهرست با استفاده، نگهداری، شفافیت منبع
  یا تمایز معنادار کم یا هیچ
- افزایش مصنوعی نصب‌ها، دانلودها، ستاره‌ها یا سایر معیارهای تعامل
  از طریق خودکارسازی، حلقه‌های نصب توسط خود، حساب‌های جعلی، فعالیت
  هماهنگ، تعامل پولی یا رفتار غیرارگانیک دیگر
- ایجاد یا چرخاندن حساب‌ها برای دور زدن نظارت، ممنوعیت‌ها، محدودیت‌های ناشر یا
  بازبینی بازارچه
- گمراه‌کردن کاربران دربارهٔ مالکیت، منبع، قابلیت‌ها، وضعیت امنیتی،
  الزامات نصب یا وابستگی به پروژه یا ناشر دیگر
- بارگذاری مکرر محتوایی که قبلاً پنهان، حذف یا مسدود شده است
  بدون رفع مشکل زیربنایی

انتشار با حجم بالا به‌خودی‌خود سوءاستفاده نیست. کاتالوگ‌های بزرگ زمانی قابل قبول هستند
که فهرست‌ها به‌طور معنادار متفاوت، دقیقاً توصیف‌شده، نگهداری‌شده
و توسط کاربران واقعی استفاده شوند. کاتالوگ‌های بزرگ زمانی به مشکل اعتماد و ایمنی تبدیل می‌شوند که
حجم با فهرست‌های کم‌محتوا، تکراری، گمراه‌کننده، نگهداری‌نشده یا
به‌طور مصنوعی ترویج‌شده همراه باشد.

## حقوق محتوا

اگر باور دارید محتوایی در ClawHub حق نشر یا سایر حقوق شما را نقض می‌کند، از
[درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) استفاده کنید. از گزارش‌های عادی بازارچه
برای ادعاهای حق نشر یا حقوق استفاده نکنید، مگر اینکه فهرست همچنین ناایمن،
مخرب یا گمراه‌کننده باشد.

## بازبینی و اجرا

ClawHub ممکن است از بررسی‌های خودکار، سیگنال‌های آماری سوءاستفاده، گزارش‌های کاربران و
بازبینی کارکنان برای شناسایی محتوای ناایمن یا رفتار انتشار سوءاستفاده‌گرانه استفاده کند. یک سیگنال
به‌تنهایی سوءاستفاده را ثابت نمی‌کند؛ به ClawHub کمک می‌کند تصمیم بگیرد چه چیزی به بازبینی نیاز دارد.

ممکن است:

- فهرست‌های ناقض را پنهان، متوقف، حذف، نرم‌حذف، یا، در مواردی که برای نوع منبع پشتیبانی می‌شود،
  سخت‌حذف کنیم
- دانلودها یا نصب‌ها را برای انتشارهای ناایمن مسدود کنیم
- توکن‌های API را لغو کنیم
- محتوای مرتبط را نرم‌حذف کنیم
- دسترسی انتشار را محدود کنیم
- متخلفان تکراری یا شدید را ممنوع کنیم

ما اجرای ابتدا-با-هشدار را برای سوءاستفادهٔ آشکار تضمین نمی‌کنیم. برای گزارش‌ها، توقف‌های نظارتی،
فهرست‌های پنهان، ممنوعیت‌ها و وضعیت حساب به
[نظارت و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
