# استفاده قابل قبول

ClawHub میزبان Skills، Plugin‌ها، بسته‌ها و فراداده بازارچه برای OpenClaw است.
از این صفحه برای تصمیم‌گیری درباره اینکه محتوا یا رفتار انتشار به
ClawHub تعلق دارد یا نه استفاده کنید.

این قواعد درباره کاری که یک فهرست انجام می‌دهد، چیزی که از کاربران می‌خواهد اجرا کنند، نحوه
معرفی خود، و شیوه استفاده ناشران از سطوح کشف، نصب و اعتماد ClawHub اعمال می‌شوند.
برای وضعیت‌های نظارت و وضعیت حساب، به
[نظارت و ایمنی حساب](/fa/clawhub/moderation) مراجعه کنید. برای ادعاهای حق نشر یا سایر حقوق،
به [درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) مراجعه کنید.

## محتوای مجاز

ClawHub از محتوایی استقبال می‌کند که مفید، قابل فهم و با حسن نیت منتشر شده باشد.

| دسته‌بندی                                         | مجاز است وقتی                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| بهره‌وری توسعه‌دهنده                           | فهرست به کاربران کمک کند نرم‌افزار را بسازند، آزمایش کنند، مهاجرت دهند، اشکال‌زدایی کنند، مستند کنند یا به کار بگیرند.                                               |
| گردش‌کارهای رابط کاربری، داده و خودکارسازی               | دامنه روشن باشد، اعتبارنامه‌های لازم صریح باشند، و اقدامات پرخطر شامل مسیرهای بازبینی، اجرای آزمایشی، پیش‌نمایش یا تأیید باشند. |
| امنیت دفاعی، نظارت و بازبینی سوءاستفاده | ابزار برای بازبینی مجاز تعریف شده باشد، شواهد را حفظ کند، و مرزهای تأیید انسانی را روشن نگه دارد.                          |
| گردش‌کارهای شخصی یا تیمی                       | گردش‌کار از حساب‌های مبتنی بر رضایت، راه‌اندازی شفاف و مجوزهای صریح استفاده کند.                                            |
| کاتالوگ‌های نگهداری‌شده                              | هر فهرست متمایز، مفید، دقیق توصیف‌شده و به شکل معقولی نگهداری‌شده باشد.                                                |

زمینه مهم است. یک موضوع یکسان می‌تواند در محیطی محدود، دفاعی یا
مبتنی بر رضایت قابل قبول باشد و وقتی به‌عنوان گردش‌کار سوءاستفاده بسته‌بندی شود غیرقابل قبول باشد.

## رفتار غیرمجاز بازارچه

ClawHub همچنین نحوه استفاده ناشران از بازارچه را بازبینی می‌کند. از ClawHub برای
دستکاری کشف، سنجه‌ها، سیگنال‌های اعتماد، سامانه‌های نظارت یا توجه کاربران استفاده نکنید.

رفتار غیرمجاز بازارچه شامل این موارد است:

- انتشار انبوه تعداد زیادی فهرست کم‌زحمت، تکراری، placeholder یا
  تولیدشده توسط ماشین که به نظر نمی‌رسد ارزش واقعی برای کاربر داشته باشند
- پر کردن سطوح جست‌وجو یا دسته‌بندی با Skills یا Plugin‌های تقریباً یکسان
- انتشار صدها فهرست با استفاده، نگهداری، شفافیت منبع
  یا تمایز معنادار اندک یا بدون آن
- افزایش مصنوعی نصب‌ها، دانلودها، ستاره‌ها یا سایر سنجه‌های تعامل
  از طریق خودکارسازی، حلقه‌های نصب توسط خود، حساب‌های جعلی، فعالیت هماهنگ،
  تعامل پولی یا رفتار غیرارگانیک دیگر
- ایجاد یا چرخاندن حساب‌ها برای دور زدن نظارت، ممنوعیت‌ها، محدودیت‌های ناشر یا
  بازبینی بازارچه
- گمراه‌کردن کاربران درباره مالکیت، منبع، قابلیت‌ها، وضعیت امنیتی،
  نیازمندی‌های نصب یا وابستگی به پروژه یا ناشر دیگر
- بارگذاری مکرر محتوایی که قبلاً پنهان، حذف یا مسدود شده است
  بدون رفع مشکل زیربنایی

انتشار با حجم بالا به‌خودی‌خود سوءاستفاده نیست. کاتالوگ‌های بزرگ وقتی قابل قبول‌اند
که فهرست‌ها به شکل معناداری متفاوت، دقیق توصیف‌شده، نگهداری‌شده
و توسط کاربران واقعی استفاده شوند. کاتالوگ‌های بزرگ زمانی به مشکل اعتماد و ایمنی تبدیل می‌شوند که
حجم با فهرست‌های کم‌محتوا، تکراری، گمراه‌کننده، نگهداری‌نشده یا
به‌صورت مصنوعی ترویج‌شده همراه باشد.

## حقوق محتوا

اگر باور دارید محتوایی در ClawHub حق نشر یا سایر حقوق شما را نقض می‌کند، از
[درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) استفاده کنید. برای ادعاهای حق نشر یا حقوق از گزارش‌های عادی بازارچه
استفاده نکنید، مگر اینکه فهرست همچنین ناامن،
مخرب یا گمراه‌کننده باشد.

## بازبینی و اجرا

ClawHub ممکن است از بررسی‌های خودکار، سیگنال‌های آماری سوءاستفاده، گزارش‌های کاربران و
بازبینی کارکنان برای شناسایی محتوای ناامن یا رفتار انتشار سوءاستفاده‌آمیز استفاده کند. یک سیگنال
به‌تنهایی سوءاستفاده را ثابت نمی‌کند؛ به ClawHub کمک می‌کند تصمیم بگیرد چه چیزی نیاز به بازبینی دارد.

ممکن است:

- فهرست‌های ناقض را پنهان، نگه‌داری، حذف، نرم‌حذف، یا، در مواردی که برای نوع منبع پشتیبانی می‌شود،
  سخت‌حذف کنیم
- دانلودها یا نصب‌ها را برای انتشارهای ناامن مسدود کنیم
- توکن‌های API را لغو کنیم
- محتوای مرتبط را نرم‌حذف کنیم
- دسترسی انتشار را محدود کنیم
- متخلفان تکراری یا شدید را ممنوع کنیم

ما اجرای هشدار-ابتدا را برای سوءاستفاده آشکار تضمین نمی‌کنیم. برای گزارش‌ها، نگه‌داشت‌های نظارتی،
فهرست‌های پنهان، ممنوعیت‌ها و وضعیت حساب، به
[نظارت و ایمنی حساب](/fa/clawhub/moderation) مراجعه کنید.
