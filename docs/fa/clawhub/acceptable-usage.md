# استفاده قابل قبول

ClawHub میزبان Skills، Pluginها، بسته‌ها و فراداده بازارچه برای OpenClaw است.
از این صفحه استفاده کنید تا تصمیم بگیرید آیا محتوا یا رفتار انتشار به
ClawHub تعلق دارد یا نه.

این قوانین درباره این است که یک فهرست چه کاری انجام می‌دهد، از کاربران می‌خواهد چه چیزی را اجرا کنند، چگونه
خود را معرفی می‌کند، و ناشران چگونه از سطوح کشف، نصب و
اعتماد ClawHub استفاده می‌کنند. برای وضعیت‌های تعدیل و وضعیت حساب، به
[تعدیل و ایمنی حساب](/clawhub/moderation) مراجعه کنید. برای ادعاهای حق نشر یا سایر حقوق،
به [درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) مراجعه کنید.

## محتوای مجاز

ClawHub از محتوایی استقبال می‌کند که مفید، قابل فهم، و با حسن نیت منتشر شده
باشد.

| دسته‌بندی                                         | زمانی مجاز است که                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| بهره‌وری توسعه‌دهنده                           | فهرست به کاربران کمک کند نرم‌افزار بسازند، آزمایش کنند، مهاجرت دهند، اشکال‌زدایی کنند، مستند کنند، یا اجرا و نگهداری کنند.                                               |
| گردش‌کارهای رابط کاربری، داده، و خودکارسازی               | دامنه روشن باشد، اعتبارنامه‌های لازم صریح باشند، و اقدام‌های پرریسک شامل مسیرهای بازبینی، اجرای آزمایشی، پیش‌نمایش، یا تأیید باشند. |
| امنیت دفاعی، تعدیل، و بازبینی سوءاستفاده | ابزار برای بازبینی مجاز چارچوب‌بندی شده باشد، شواهد را حفظ کند، و مرزهای تأیید انسانی را روشن نگه دارد.                          |
| گردش‌کارهای شخصی یا تیمی                       | گردش‌کار از حساب‌های مبتنی بر رضایت، راه‌اندازی شفاف، و مجوزهای صریح استفاده کند.                                            |
| کاتالوگ‌های نگهداری‌شده                              | هر فهرست متمایز، مفید، دقیقاً توصیف‌شده، و به‌طور معقول نگهداری‌شده باشد.                                                |

زمینه مهم است. همان موضوع می‌تواند در یک چارچوب دفاعی محدود یا
مبتنی بر رضایت قابل قبول باشد و وقتی به‌عنوان گردش‌کار سوءاستفاده بسته‌بندی شود غیرقابل قبول باشد.

## رفتار غیرمجاز در بازارچه

ClawHub همچنین بررسی می‌کند که ناشران چگونه از بازارچه استفاده می‌کنند. از ClawHub برای
دستکاری کشف، معیارها، نشانه‌های اعتماد، سامانه‌های تعدیل، یا توجه کاربران
استفاده نکنید.

رفتار غیرمجاز در بازارچه شامل موارد زیر است:

- انتشار انبوه تعداد زیادی فهرست کم‌زحمت، تکراری، جایگزین‌نما، یا
  تولیدشده با ماشین که به نظر نمی‌رسد ارزش واقعی برای کاربر داشته باشند
- پر کردن سطوح جست‌وجو یا دسته‌بندی با Skills یا Pluginهای تقریباً یکسان
- انتشار صدها فهرست با استفاده، نگهداری، شفافیت منبع،
  یا تمایز معنادار اندک یا بدون آن
- افزایش مصنوعی نصب‌ها، دانلودها، ستاره‌ها، یا سایر معیارهای
  تعامل از طریق خودکارسازی، حلقه‌های نصب توسط خود، حساب‌های جعلی، فعالیت
  هماهنگ، تعامل پولی، یا رفتار غیرطبیعی دیگر
- ایجاد یا چرخش حساب‌ها برای دور زدن تعدیل، ممنوعیت‌ها، محدودیت‌های ناشر، یا
  بازبینی بازارچه
- گمراه‌کردن کاربران درباره مالکیت، منبع، توانایی‌ها، وضعیت امنیتی،
  الزامات نصب، یا وابستگی به پروژه یا ناشر دیگر
- بارگذاری مکرر محتوایی که پیش‌تر پنهان، حذف، یا مسدود شده است
  بدون رفع مشکل زیربنایی

انتشار با حجم بالا به‌طور خودکار سوءاستفاده نیست. کاتالوگ‌های بزرگ زمانی قابل قبول‌اند
که فهرست‌ها به‌طور معناداری متفاوت، دقیقاً توصیف‌شده، نگهداری‌شده،
و توسط کاربران واقعی استفاده شوند. کاتالوگ‌های بزرگ زمانی به مشکل اعتماد و ایمنی تبدیل می‌شوند که
حجم با فهرست‌های کم‌محتوا، تکراری، گمراه‌کننده، نگهداری‌نشده، یا
به‌طور مصنوعی تبلیغ‌شده همراه باشد.

## حقوق محتوا

اگر باور دارید محتوایی در ClawHub حق نشر یا سایر حقوق شما را نقض می‌کند، از
[درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) استفاده کنید. از گزارش‌های عادی بازارچه
برای ادعاهای حق نشر یا حقوق استفاده نکنید، مگر اینکه فهرست همچنین ناامن،
مخرب، یا گمراه‌کننده باشد.

## بازبینی و اجرا

ClawHub ممکن است از بررسی‌های خودکار، نشانه‌های آماری سوءاستفاده، گزارش‌های کاربران، و
بازبینی کارکنان برای شناسایی محتوای ناامن یا رفتار انتشار سوءاستفاده‌آمیز استفاده کند. یک نشانه
به‌تنهایی سوءاستفاده را ثابت نمی‌کند؛ به ClawHub کمک می‌کند تصمیم بگیرد چه چیزی نیاز به بازبینی دارد.

ممکن است:

- فهرست‌های ناقض را پنهان، نگه‌داری، حذف، حذف نرم، یا، در مواردی که برای نوع منبع پشتیبانی می‌شود،
  حذف سخت کنیم
- دانلودها یا نصب‌ها را برای انتشارهای ناامن مسدود کنیم
- توکن‌های API را لغو کنیم
- محتوای مرتبط را حذف نرم کنیم
- دسترسی انتشار را محدود کنیم
- متخلفان تکراری یا شدید را ممنوع کنیم

ما اجرای ابتدا-با-هشدار را برای سوءاستفاده آشکار تضمین نمی‌کنیم. برای گزارش‌ها، نگه‌داری‌های تعدیل،
فهرست‌های پنهان، ممنوعیت‌ها، و وضعیت حساب، به
[تعدیل و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
