# استفاده قابل قبول

ClawHub میزبان Skills، Pluginها، بسته‌ها و فرادادهٔ بازارچه برای OpenClaw است.
از این صفحه استفاده کنید تا تصمیم بگیرید آیا محتوا یا رفتار انتشار به
ClawHub تعلق دارد یا نه.

این قواعد دربارهٔ کاری که یک فهرست انجام می‌دهد، آنچه از کاربران می‌خواهد اجرا کنند، نحوهٔ
معرفی خودش، و چگونگی استفادهٔ ناشران از سطوح کشف، نصب و
اعتماد ClawHub اعمال می‌شود. برای وضعیت‌های تعدیل و وضعیت حساب، به
[تعدیل و ایمنی حساب](/clawhub/moderation) مراجعه کنید. برای ادعاهای کپی‌رایت یا سایر حقوق،
[درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) را ببینید.

## محتوای مجاز

ClawHub از محتوایی استقبال می‌کند که مفید، قابل فهم، و با حسن نیت منتشر شده باشد.

| دسته‌بندی                                         | مجاز است وقتی                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| بهره‌وری توسعه‌دهنده                           | فهرست به کاربران کمک می‌کند نرم‌افزار بسازند، آزمایش کنند، مهاجرت دهند، اشکال‌زدایی کنند، مستندسازی کنند، یا اداره کنند.                                               |
| گردش‌کارهای UI، داده، و خودکارسازی               | دامنه روشن است، اعتبارنامه‌های لازم صریح هستند، و کنش‌های پرخطر مسیرهای بازبینی، اجرای آزمایشی، پیش‌نمایش، یا تأیید دارند. |
| امنیت دفاعی، تعدیل، و بازبینی سوءاستفاده | ابزار برای بازبینی مجاز چارچوب‌بندی شده، شواهد را حفظ می‌کند، و مرزهای تأیید انسانی را روشن نگه می‌دارد.                          |
| گردش‌کارهای شخصی یا تیمی                       | گردش‌کار از حساب‌های مبتنی بر رضایت، راه‌اندازی شفاف، و مجوزهای صریح استفاده می‌کند.                                            |
| کاتالوگ‌های نگهداری‌شده                              | هر فهرست متمایز، مفید، با توصیف دقیق، و به‌طور معقول نگهداری‌شده است.                                                |

زمینه مهم است. یک موضوع واحد می‌تواند در یک محیط دفاعی محدود یا
مبتنی بر رضایت قابل قبول باشد و وقتی به‌عنوان گردش‌کار سوءاستفاده بسته‌بندی شود غیرقابل قبول باشد.

## رفتار غیرمجاز بازارچه

ClawHub همچنین نحوهٔ استفادهٔ ناشران از بازارچه را بازبینی می‌کند. از ClawHub برای
دستکاری کشف، معیارها، نشانه‌های اعتماد، سامانه‌های تعدیل، یا توجه
کاربران استفاده نکنید.

رفتار غیرمجاز بازارچه شامل این موارد است:

- انتشار انبوه تعداد زیادی فهرست کم‌تلاش، تکراری، جایگزین‌نما، یا
  تولیدشده با ماشین که به نظر نمی‌رسد ارزش واقعی برای کاربر داشته باشند
- پر کردن سطوح جست‌وجو یا دسته‌بندی با Skills یا Pluginهای تقریباً یکسان
- انتشار صدها فهرست با استفاده، نگهداری، شفافیت منبع،
  یا تمایز معنادار کم یا بدون آن
- افزایش مصنوعی نصب‌ها، دانلودها، ستاره‌ها، یا سایر معیارهای
  تعامل از طریق خودکارسازی، حلقه‌های خودنصب، حساب‌های جعلی، فعالیت
  هماهنگ، تعامل پولی، یا رفتار غیرارگانیک دیگر
- ایجاد یا چرخش حساب‌ها برای دور زدن تعدیل، مسدودیت‌ها، محدودیت‌های ناشر، یا
  بازبینی بازارچه
- گمراه‌کردن کاربران دربارهٔ مالکیت، منبع، قابلیت‌ها، وضعیت امنیتی،
  نیازمندی‌های نصب، یا وابستگی به پروژه یا ناشر دیگر
- بارگذاری مکرر محتوایی که قبلاً پنهان، حذف، یا مسدود شده است
  بدون رفع مسئلهٔ زیربنایی

انتشار با حجم بالا به‌طور خودکار سوءاستفاده نیست. کاتالوگ‌های بزرگ زمانی قابل قبول هستند
که فهرست‌ها به‌طور معنادار متفاوت، با توصیف دقیق، نگهداری‌شده،
و توسط کاربران واقعی استفاده‌شده باشند. کاتالوگ‌های بزرگ زمانی به مسئلهٔ اعتماد و ایمنی تبدیل می‌شوند که
حجم با فهرست‌های سطحی، تکراری، گمراه‌کننده، نگهداری‌نشده، یا
به‌طور مصنوعی ترویج‌شده همراه باشد.

## حقوق محتوا

اگر باور دارید محتوایی در ClawHub کپی‌رایت یا سایر حقوق شما را نقض می‌کند، از
[درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) استفاده کنید. از گزارش‌های عادی بازارچه
برای ادعاهای کپی‌رایت یا حقوق استفاده نکنید، مگر اینکه فهرست همچنین ناایمن،
مخرب، یا گمراه‌کننده باشد.

## بازبینی و اجرا

ClawHub ممکن است از بررسی‌های خودکار، نشانه‌های آماری سوءاستفاده، گزارش‌های کاربران، و
بازبینی کارکنان برای شناسایی محتوای ناایمن یا رفتار انتشار سوءاستفاده‌آمیز استفاده کند. یک نشانه
به‌تنهایی سوءاستفاده را ثابت نمی‌کند؛ به ClawHub کمک می‌کند تصمیم بگیرد چه چیزی نیازمند بازبینی است.

ممکن است:

- فهرست‌های ناقض را پنهان، نگه‌داری، حذف، حذف نرم، یا، در جایی که برای نوع منبع پشتیبانی می‌شود،
  حذف سخت کنیم
- دانلودها یا نصب‌ها را برای انتشارهای ناایمن مسدود کنیم
- توکن‌های API را لغو کنیم
- محتوای مرتبط را حذف نرم کنیم
- دسترسی انتشار را محدود کنیم
- متخلفان تکراری یا شدید را مسدود کنیم

برای سوءاستفادهٔ آشکار، اجرای همراه با هشدار اولیه را تضمین نمی‌کنیم. برای گزارش‌ها، نگه‌داشت‌های تعدیل،
فهرست‌های پنهان، مسدودیت‌ها، و وضعیت حساب، به
[تعدیل و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
