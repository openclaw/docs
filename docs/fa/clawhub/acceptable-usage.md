# استفادهٔ قابل قبول

ClawHub میزبان Skills، Plugin‌ها، بسته‌ها، و فرادادهٔ بازارچه برای OpenClaw است.
از این صفحه استفاده کنید تا تصمیم بگیرید آیا محتوا یا رفتار انتشار به
ClawHub تعلق دارد یا نه.

این قواعد دربارهٔ کاری که یک فهرست انجام می‌دهد، چیزی که از کاربران می‌خواهد اجرا کنند، نحوه‌ای که
خود را معرفی می‌کند، و شیوهٔ استفادهٔ ناشران از سطوح کشف، نصب، و
اعتماد ClawHub اعمال می‌شود. برای وضعیت‌های نظارت و اعتبار حساب، ببینید
[نظارت و ایمنی حساب](/clawhub/moderation). برای ادعاهای حق نشر یا حقوق دیگر،
ببینید [درخواست‌های حقوق محتوا](/fa/clawhub/content-rights).

## محتوای مجاز

ClawHub از محتوایی استقبال می‌کند که مفید، قابل فهم، و با حسن نیت منتشر شده باشد.

| دسته‌بندی                                         | مجاز است وقتی                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| بهره‌وری توسعه‌دهنده                           | فهرست به کاربران کمک کند نرم‌افزار بسازند، آزمایش کنند، مهاجرت دهند، اشکال‌زدایی کنند، مستند کنند، یا اجرا و نگهداری کنند.                                               |
| گردش‌کارهای رابط کاربری، داده، و خودکارسازی               | دامنه روشن باشد، اعتبارنامه‌های لازم صریح باشند، و کنش‌های پرخطر مسیرهای بازبینی، اجرای آزمایشی، پیش‌نمایش، یا تأیید داشته باشند. |
| امنیت دفاعی، نظارت، و بازبینی سوءاستفاده | ابزار برای بازبینی مجاز چارچوب‌بندی شده باشد، شواهد را حفظ کند، و مرزهای تأیید انسانی را روشن نگه دارد.                          |

| کاتالوگ‌های نگهداری‌شده                              | هر فهرست متمایز، مفید، دقیق توصیف شده، و به‌طور معقول نگهداری شده باشد.                                                |

زمینه اهمیت دارد. همان موضوع می‌تواند در یک محیط دفاعی محدود یا
مبتنی بر رضایت قابل قبول باشد و وقتی به‌عنوان گردش‌کار سوءاستفاده بسته‌بندی شود غیرقابل قبول باشد.

## رفتار غیرمجاز بازارچه

ClawHub همچنین نحوهٔ استفادهٔ ناشران از بازارچه را بازبینی می‌کند. از ClawHub برای
دستکاری کشف، سنجه‌ها، نشانه‌های اعتماد، سامانه‌های نظارت، یا توجه کاربران
استفاده نکنید.

رفتار غیرمجاز بازارچه شامل این موارد است:

- انتشار انبوه تعداد زیادی فهرست کم‌زحمت، تکراری، جای‌نگهدار، یا
  تولیدشده با ماشین که به نظر نمی‌رسد ارزش واقعی برای کاربر داشته باشند
- پر کردن سطوح جست‌وجو یا دسته‌بندی با Skills یا Plugin‌های تقریباً یکسان
- انتشار صدها فهرست با استفاده، نگهداری، شفافیت منبع،
  یا تمایز معنادار کم یا بدون آن
- افزایش مصنوعی نصب‌ها، دانلودها، ستاره‌ها، یا سایر سنجه‌های
  تعامل از طریق خودکارسازی، حلقه‌های نصب توسط خود، حساب‌های جعلی، فعالیت
  هماهنگ، تعامل پولی، یا سایر رفتارهای غیرارگانیک
- ایجاد یا چرخاندن حساب‌ها برای دور زدن نظارت، ممنوعیت‌ها، محدودیت‌های ناشر، یا
  بازبینی بازارچه
- گمراه‌کردن کاربران دربارهٔ مالکیت، منبع، قابلیت‌ها، وضعیت امنیتی،
  الزامات نصب، یا وابستگی به پروژه یا ناشر دیگر
- بارگذاری مکرر محتوایی که قبلاً پنهان، حذف، یا مسدود شده است
  بدون رفع مشکل زیربنایی

انتشار پرحجم به‌طور خودکار سوءاستفاده نیست. کاتالوگ‌های بزرگ زمانی قابل قبول هستند
که فهرست‌ها به‌طور معنادار متفاوت، دقیق توصیف شده، نگهداری‌شده،
و توسط کاربران واقعی استفاده شوند. کاتالوگ‌های بزرگ زمانی به مشکل اعتماد و ایمنی تبدیل می‌شوند که
حجم با فهرست‌های کم‌محتوا، تکراری، گمراه‌کننده، نگهداری‌نشده، یا
به‌طور مصنوعی ترویج‌شده همراه باشد.

## حقوق محتوا

اگر باور دارید محتوایی در ClawHub حق نشر یا حقوق دیگر شما را نقض می‌کند، از
[درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) استفاده کنید. برای ادعاهای حق نشر یا حقوق، از گزارش‌های عادی بازارچه
استفاده نکنید مگر اینکه فهرست همچنین ناامن،
مخرب، یا گمراه‌کننده باشد.

## بازبینی و اجرا

ClawHub ممکن است از بررسی‌های خودکار، نشانه‌های آماری سوءاستفاده، گزارش‌های کاربر، و
بازبینی کارکنان برای شناسایی محتوای ناامن یا رفتار انتشار سوءاستفاده‌آمیز استفاده کند. یک نشانه
به‌تنهایی سوءاستفاده را اثبات نمی‌کند؛ به ClawHub کمک می‌کند تصمیم بگیرد چه چیزی نیاز به بازبینی دارد.

ممکن است:

- فهرست‌های ناقض را پنهان، نگه‌داری، حذف، soft-delete، یا، در جایی که برای نوع منبع پشتیبانی می‌شود،
  hard-delete کنیم
- دانلودها یا نصب‌ها را برای انتشارهای ناامن مسدود کنیم
- توکن‌های API را لغو کنیم
- محتوای مرتبط را soft-delete کنیم
- دسترسی انتشار را محدود کنیم
- متخلفان تکراری یا شدید را ممنوع کنیم

ما اجرای ابتدا-با-هشدار را برای سوءاستفادهٔ آشکار تضمین نمی‌کنیم. برای گزارش‌ها، نگه‌داشت‌های نظارتی،
فهرست‌های پنهان، ممنوعیت‌ها، و اعتبار حساب، ببینید
[نظارت و ایمنی حساب](/clawhub/moderation).
