# استفادهٔ قابل قبول

ClawHub میزبان Skills، Pluginها، بسته‌ها و فرادادهٔ بازار برای OpenClaw است.
از این صفحه استفاده کنید تا تصمیم بگیرید آیا محتوا یا رفتار انتشار به
ClawHub تعلق دارد یا نه.

این قواعد دربارهٔ کاری که یک فهرست انجام می‌دهد، چیزی که از کاربران می‌خواهد اجرا کنند، نحوهٔ
معرفی خود، و نحوهٔ استفادهٔ ناشران از سطوح کشف، نصب و اعتماد ClawHub اعمال می‌شود. برای وضعیت‌های مدیریت محتوا و اعتبار حساب، [مدیریت محتوا و ایمنی حساب](/fa/clawhub/moderation) را ببینید. برای ادعاهای حق نشر یا سایر حقوق،
[درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) را ببینید.

## محتوای مجاز

ClawHub از محتوایی استقبال می‌کند که مفید، قابل فهم، و با حسن نیت منتشر شده باشد.

| دسته | مجاز است وقتی |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| بهره‌وری توسعه‌دهنده | فهرست به کاربران کمک کند نرم‌افزار بسازند، آزمایش کنند، مهاجرت دهند، اشکال‌زدایی کنند، مستندسازی کنند، یا بهره‌برداری کنند. |
| گردش‌کارهای رابط کاربری، داده و خودکارسازی | دامنه روشن باشد، اعتبارنامه‌های لازم صریح باشند، و اقدامات پرخطر مسیرهای بازبینی، اجرای آزمایشی، پیش‌نمایش، یا تأیید داشته باشند. |
| امنیت دفاعی، مدیریت محتوا، و بازبینی سوءاستفاده | ابزار برای بازبینی مجاز چارچوب‌بندی شده باشد، شواهد را حفظ کند، و مرزهای تأیید انسانی را روشن نگه دارد. |
| گردش‌کارهای شخصی یا تیمی | گردش‌کار از حساب‌های مبتنی بر رضایت، راه‌اندازی شفاف، و مجوزهای صریح استفاده کند. |
| کاتالوگ‌های نگهداری‌شده | هر فهرست متمایز، مفید، دقیقاً توصیف‌شده، و به‌طور معقول نگهداری‌شده باشد. |

زمینه اهمیت دارد. همان موضوع می‌تواند در یک محیط دفاعی محدود یا
مبتنی بر رضایت قابل قبول باشد و وقتی به‌عنوان گردش‌کار سوءاستفاده بسته‌بندی شود، غیرقابل قبول باشد.

## محتوای غیرمجاز

ClawHub محتوایی را میزبانی نمی‌کند که هدف اصلی آن سوءاستفاده، فریب، اجرای ناامن،
یا نقض حقوق باشد.

| دسته | غیرمجاز |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| دسترسی غیرمجاز یا دور زدن امنیت | دور زدن احراز هویت، تصاحب حساب، سوءاستفاده از محدودیت نرخ، تصاحب تماس زنده یا عامل، سرقت نشست قابل استفادهٔ مجدد، یا تأیید خودکار جریان‌های جفت‌سازی برای کاربران تأییدنشده. |
| سوءاستفاده از پلتفرم و دور زدن ممنوعیت | حساب‌های پنهان پس از ممنوعیت، آماده‌سازی یا مزرعه‌سازی حساب، تعامل جعلی، خودکارسازی چندحسابی، ارسال انبوه، ربات‌های هرزنامه، یا خودکارسازی ساخته‌شده برای اجتناب از شناسایی. |
| کلاهبرداری، تقلب، و گردش‌کارهای مالی فریبنده | گواهی‌ها یا فاکتورهای جعلی، جریان‌های پرداخت فریبنده، ارتباط‌گیری کلاهبردارانه، اثبات اجتماعی جعلی، گردش‌کارهای هویت مصنوعی برای تقلب، یا ابزارهای خرج‌کردن/شارژکردن بدون تأیید انسانی روشن. |
| غنی‌سازی ناقض حریم خصوصی یا نظارت | استخراج مخاطب برای هرزنامه، افشای اطلاعات شخصی، تعقیب، استخراج سرنخ همراه با ارتباط‌گیری ناخواسته، پایش پنهانی، تطبیق زیستی بدون رضایت، یا استفاده از داده‌های نشت‌کرده یا دامپ‌های رخنه. |
| جعل هویت بدون رضایت یا دستکاری هویت | تعویض چهره، دوقلوهای دیجیتال، اینفلوئنسرهای شبیه‌سازی‌شده، شخصیت‌های جعلی، یا ابزارهای دیگری که برای جعل هویت یا گمراه‌کردن استفاده می‌شوند. |
| محتوای صریح جنسی یا تولید بزرگسالانه با ایمنی غیرفعال | تولید تصویر، ویدئو، یا محتوا NSFW؛ wrapperهای محتوای بزرگسالانه پیرامون APIهای شخص ثالث؛ یا فهرست‌هایی که هدف اصلی آن‌ها محتوای صریح جنسی است. |
| الزامات اجرای پنهان، ناامن، یا گمراه‌کننده | فرمان‌های نصب مبهم‌سازی‌شده، نصب‌کننده‌های pipe-to-shell مانند محتوای دانلودشده که با `sh` یا `bash` بدون قابلیت بازبینی روشن اجرا می‌شود، الزامات اعلام‌نشدهٔ secret یا کلید خصوصی، اجرای راه‌دور `npx @latest` بدون قابلیت بازبینی روشن، یا فراداده‌ای که نیاز واقعی فهرست برای اجرا را پنهان می‌کند. |
| مواد ناقض حق نشر یا نقض‌کنندهٔ حقوق | بازنشر skill، Plugin، مستندات، دارایی‌های برند، یا کد اختصاصی شخص دیگر بدون اجازه؛ نقض شرایط مجوز؛ یا جعل هویت نویسنده یا ناشر اصلی. |

## رفتار غیرمجاز در بازار

ClawHub همچنین نحوهٔ استفادهٔ ناشران از بازار را بازبینی می‌کند. از ClawHub برای
دستکاری کشف، معیارها، نشانه‌های اعتماد، سامانه‌های مدیریت محتوا، یا توجه کاربران
استفاده نکنید.

رفتار غیرمجاز در بازار شامل این موارد است:

- انتشار انبوه تعداد زیادی فهرست کم‌تلاش، تکراری، جایگزین‌نما، یا
  تولیدشده با ماشین که به نظر نمی‌رسد ارزش واقعی برای کاربر داشته باشند
- پرکردن سطوح جستجو یا دسته‌بندی با Skills یا Pluginهای تقریباً یکسان
- انتشار صدها فهرست با استفاده، نگهداری، شفافیت منبع،
  یا تمایز معنادار اندک یا هیچ
- افزایش مصنوعی نصب‌ها، دانلودها، ستاره‌ها، یا سایر معیارهای تعامل
  از طریق خودکارسازی، حلقه‌های خودنصب، حساب‌های جعلی، فعالیت هماهنگ،
  تعامل پولی، یا رفتار غیرارگانیک دیگر
- ایجاد یا چرخاندن حساب‌ها برای دور زدن مدیریت محتوا، ممنوعیت‌ها، محدودیت‌های ناشر، یا
  بازبینی بازار
- گمراه‌کردن کاربران دربارهٔ مالکیت، منبع، قابلیت‌ها، وضعیت امنیتی،
  الزامات نصب، یا وابستگی به پروژه یا ناشر دیگر
- بارگذاری مکرر محتوایی که قبلاً پنهان، حذف، یا مسدود شده است
  بدون رفع مشکل زیربنایی

انتشار پرحجم به‌طور خودکار سوءاستفاده نیست. کاتالوگ‌های بزرگ زمانی قابل قبول‌اند
که فهرست‌ها به‌طور معنادار متفاوت، دقیقاً توصیف‌شده، نگهداری‌شده،
و توسط کاربران واقعی استفاده شوند. کاتالوگ‌های بزرگ وقتی به مسئلهٔ اعتماد و ایمنی تبدیل می‌شوند که
حجم با فهرست‌های کم‌محتوا، تکراری، گمراه‌کننده، نگهداری‌نشده، یا
به‌طور مصنوعی ترویج‌شده همراه باشد.

## حقوق محتوا

اگر باور دارید محتوایی در ClawHub حق نشر یا سایر حقوق شما را نقض می‌کند، از
[درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) استفاده کنید. از گزارش‌های عادی بازار
برای ادعاهای حق نشر یا حقوق استفاده نکنید مگر اینکه فهرست همچنین ناامن،
مخرب، یا گمراه‌کننده باشد.

## بازبینی و اعمال مقررات

ClawHub ممکن است از بررسی‌های خودکار، نشانه‌های آماری سوءاستفاده، گزارش‌های کاربران، و
بازبینی کارکنان برای شناسایی محتوای ناامن یا رفتار انتشار سوءاستفاده‌آمیز استفاده کند. یک نشانه
به‌تنهایی سوءاستفاده را ثابت نمی‌کند؛ به ClawHub کمک می‌کند تصمیم بگیرد چه چیزی نیاز به بازبینی دارد.

ممکن است:

- فهرست‌های ناقض را پنهان، نگه‌داری، حذف، حذف نرم، یا، در جایی که برای نوع منبع پشتیبانی می‌شود،
  حذف سخت کنیم
- دانلودها یا نصب‌ها را برای نسخه‌های ناامن مسدود کنیم
- توکن‌های API را لغو کنیم
- محتوای مرتبط را حذف نرم کنیم
- دسترسی انتشار را محدود کنیم
- متخلفان تکراری یا شدید را ممنوع کنیم

ما اعمال مقررات همراه با هشدار اولیه را برای سوءاستفادهٔ آشکار تضمین نمی‌کنیم. برای گزارش‌ها، نگه‌داشت‌های مدیریت محتوا،
فهرست‌های پنهان، ممنوعیت‌ها، و اعتبار حساب، [مدیریت محتوا و ایمنی حساب](/fa/clawhub/moderation) را ببینید.
