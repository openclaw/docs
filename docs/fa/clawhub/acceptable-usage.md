# استفادهٔ قابل قبول

ClawHub میزبان Skills، Pluginها، بسته‌ها و فرادادهٔ بازارچه برای OpenClaw است.
از این صفحه استفاده کنید تا تصمیم بگیرید آیا محتوا یا رفتار انتشار به
ClawHub تعلق دارد یا نه.

این قواعد دربارهٔ کاری که یک فهرست انجام می‌دهد، چیزی که از کاربران می‌خواهد اجرا کنند، نحوهٔ
معرفی خودش، و شیوهٔ استفادهٔ ناشران از سطوح کشف، نصب و
اعتماد ClawHub اعمال می‌شوند. برای وضعیت‌های تعدیل و اعتبار حساب، ببینید
[تعدیل و ایمنی حساب](/clawhub/moderation). برای ادعاهای حق نشر یا حقوق دیگر،
ببینید [درخواست‌های حقوق محتوا](/fa/clawhub/content-rights).

## محتوای مجاز

ClawHub از محتوایی استقبال می‌کند که مفید، قابل‌فهم و با حسن نیت منتشر شده باشد.

| دسته | مجاز است وقتی |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| بهره‌وری توسعه‌دهنده | فهرست به کاربران کمک کند نرم‌افزار بسازند، آزمایش کنند، مهاجرت دهند، اشکال‌زدایی کنند، مستند کنند یا اجرا و نگهداری کنند. |
| گردش‌کارهای UI، داده و خودکارسازی | دامنه روشن باشد، اعتبارنامه‌های لازم صریح باشند، و اقدامات پرخطر مسیرهای بازبینی، اجرای آزمایشی، پیش‌نمایش یا تأیید داشته باشند. |
| امنیت دفاعی، تعدیل و بازبینی سوءاستفاده | ابزار برای بازبینی مجاز چارچوب‌بندی شده باشد، شواهد را حفظ کند و مرزهای تأیید انسانی را روشن نگه دارد. |
| گردش‌کارهای شخصی یا تیمی | گردش‌کار از حساب‌های مبتنی بر رضایت، راه‌اندازی شفاف و مجوزهای صریح استفاده کند. |
| کاتالوگ‌های نگه‌داری‌شده | هر فهرست متمایز، مفید، دقیق توصیف‌شده و به‌طور معقول نگه‌داری‌شده باشد. |

زمینه مهم است. یک موضوع واحد می‌تواند در یک چارچوب دفاعی محدود یا
مبتنی بر رضایت قابل قبول باشد و وقتی به‌عنوان گردش‌کار سوءاستفاده بسته‌بندی شود غیرقابل قبول باشد.

## محتوای غیرمجاز

ClawHub محتوایی را میزبانی نمی‌کند که هدف اصلی آن سوءاستفاده، فریب، اجرای ناامن
یا نقض حقوق باشد.

| دسته | مجاز نیست |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| دسترسی غیرمجاز یا دور زدن امنیت | دور زدن احراز هویت، تصاحب حساب، سوءاستفاده از محدودیت نرخ، تصاحب تماس زنده یا عامل، سرقت نشست قابل استفادهٔ مجدد، یا تأیید خودکار جریان‌های جفت‌سازی برای کاربران تأییدنشده. |
| سوءاستفاده از پلتفرم و دور زدن ممنوعیت | حساب‌های پنهان پس از ممنوعیت‌ها، گرم‌کردن یا پرورش حساب، تعامل جعلی، خودکارسازی چندحسابی، ارسال انبوه، ربات‌های اسپم، یا خودکارسازی ساخته‌شده برای اجتناب از شناسایی. |
| تقلب، کلاهبرداری و گردش‌کارهای مالی فریبنده | گواهی‌ها یا فاکتورهای جعلی، جریان‌های پرداخت فریبنده، outreach کلاهبردارانه، اثبات اجتماعی جعلی، گردش‌کارهای هویت مصنوعی برای تقلب، یا ابزارهای خرج‌کردن/شارژکردن بدون تأیید انسانی روشن. |
| غنی‌سازی یا پایش ناقض حریم خصوصی | خراشیدن مخاطبان برای اسپم، افشای اطلاعات شخصی، تعقیب، استخراج سرنخ همراه با outreach ناخواسته، پایش پنهانی، تطبیق زیستی بدون رضایت، یا استفاده از داده‌های افشاشده یا dumpهای رخنه. |
| جعل هویت یا دست‌کاری هویت بدون رضایت | تعویض چهره، دوقلوهای دیجیتال، اینفلوئنسرهای شبیه‌سازی‌شده، پرسوناهای جعلی، یا ابزارهای دیگری که برای جعل هویت یا گمراه‌کردن استفاده می‌شوند. |
| محتوای جنسی صریح یا تولید محتوای بزرگسال با ایمنی غیرفعال | تولید تصویر، ویدیو یا محتوای NSFW؛ پوشش‌های محتوای بزرگسال پیرامون APIهای شخص ثالث؛ یا فهرست‌هایی که هدف اصلی‌شان محتوای جنسی صریح است. |
| الزامات اجرای پنهان، ناامن یا گمراه‌کننده | فرمان‌های نصب مبهم‌سازی‌شده، نصب‌کننده‌های pipe-to-shell مانند محتوای دانلودشده که با `sh` یا `bash` اجرا می‌شود بدون قابلیت بازبینی روشن، الزامات اعلام‌نشدهٔ secret یا کلید خصوصی، اجرای راه‌دور `npx @latest` بدون قابلیت بازبینی روشن، یا فراداده‌ای که آنچه فهرست واقعاً برای اجرا نیاز دارد را پنهان می‌کند. |
| مواد ناقض حق نشر یا ناقض حقوق | بازنشر skill، Plugin، مستندات، دارایی‌های برند یا کد اختصاصی شخص دیگر بدون اجازه؛ نقض شرایط مجوز؛ یا جعل هویت نویسنده یا ناشر اصلی. |

## رفتار غیرمجاز در بازارچه

ClawHub همچنین نحوهٔ استفادهٔ ناشران از بازارچه را بررسی می‌کند. از ClawHub برای
دست‌کاری کشف، معیارها، سیگنال‌های اعتماد، سامانه‌های تعدیل یا توجه
کاربر استفاده نکنید.

رفتار غیرمجاز در بازارچه شامل این موارد است:

- انتشار انبوه تعداد زیادی فهرست کم‌زحمت، تکراری، placeholder یا
  تولیدشده با ماشین که به نظر نمی‌رسد ارزش واقعی برای کاربر داشته باشند
- سیل‌آسا کردن سطوح جست‌وجو یا دسته‌بندی با Skills یا Pluginهای تقریباً یکسان
- انتشار صدها فهرست با استفاده، نگه‌داری، شفافیت منبع
  یا تمایز معنادار اندک یا بدون آن
- افزایش مصنوعی نصب‌ها، دانلودها، ستاره‌ها یا سایر معیارهای
  تعامل از طریق خودکارسازی، حلقه‌های خودنصب، حساب‌های جعلی، فعالیت
  هماهنگ، تعامل پولی یا رفتار غیرارگانیک دیگر
- ایجاد یا چرخاندن حساب‌ها برای دور زدن تعدیل، ممنوعیت‌ها، محدودیت‌های ناشر یا
  بازبینی بازارچه
- گمراه‌کردن کاربران دربارهٔ مالکیت، منبع، قابلیت‌ها، وضعیت امنیتی،
  الزامات نصب یا وابستگی به پروژه یا ناشر دیگر
- بارگذاری مکرر محتوایی که قبلاً پنهان، حذف یا مسدود شده است
  بدون رفع مسئلهٔ زیربنایی

انتشار با حجم بالا به‌طور خودکار سوءاستفاده نیست. کاتالوگ‌های بزرگ وقتی قابل قبول‌اند
که فهرست‌ها به‌طور معنادار متفاوت، دقیق توصیف‌شده، نگه‌داری‌شده
و توسط کاربران واقعی استفاده شوند. کاتالوگ‌های بزرگ وقتی به مسئلهٔ اعتماد و ایمنی تبدیل می‌شوند که
حجم با فهرست‌های نازک، تکراری، گمراه‌کننده، نگه‌داری‌نشده یا
به‌طور مصنوعی ترویج‌شده همراه باشد.

## حقوق محتوا

اگر باور دارید محتوایی در ClawHub حق نشر یا حقوق دیگر شما را نقض می‌کند، از
[درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) استفاده کنید. برای ادعاهای حق نشر یا حقوق، از گزارش‌های عادی بازارچه
استفاده نکنید مگر اینکه فهرست همچنین ناامن،
مخرب یا گمراه‌کننده باشد.

## بازبینی و اجرا

ClawHub ممکن است از بررسی‌های خودکار، سیگنال‌های آماری سوءاستفاده، گزارش‌های کاربران و
بازبینی کارکنان برای شناسایی محتوای ناامن یا رفتار انتشار سوءاستفاده‌آمیز استفاده کند. یک سیگنال
به‌تنهایی سوءاستفاده را ثابت نمی‌کند؛ به ClawHub کمک می‌کند تصمیم بگیرد چه چیزی نیاز به بازبینی دارد.

ما ممکن است:

- فهرست‌های ناقض را پنهان، نگه‌دار، حذف، soft-delete، یا، در مواردی که برای نوع منبع پشتیبانی می‌شود،
  hard-delete کنیم
- دانلودها یا نصب‌ها را برای انتشارهای ناامن مسدود کنیم
- توکن‌های API را لغو کنیم
- محتوای مرتبط را soft-delete کنیم
- دسترسی انتشار را محدود کنیم
- متخلفان تکراری یا شدید را ممنوع کنیم

ما اجرای هشدار-اول را برای سوءاستفادهٔ آشکار تضمین نمی‌کنیم. ببینید
[تعدیل و ایمنی حساب](/clawhub/moderation) برای گزارش‌ها، نگه‌داشت‌های تعدیل،
فهرست‌های پنهان، ممنوعیت‌ها و اعتبار حساب.
