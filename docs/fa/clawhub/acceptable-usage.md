# استفاده قابل‌قبول

ClawHub میزبان Skills، Pluginها، بسته‌ها، و فرادادهٔ بازارگاه برای OpenClaw است.
از این صفحه استفاده کنید تا تصمیم بگیرید آیا محتوا یا رفتار انتشار به
ClawHub تعلق دارد یا نه.

این قواعد بر کاری که یک فهرست انجام می‌دهد، چیزی که از کاربران می‌خواهد اجرا کنند، نحوه‌ای که
خود را معرفی می‌کند، و نحوهٔ استفادهٔ ناشران از سطوح کشف، نصب، و اعتماد
ClawHub اعمال می‌شود. برای وضعیت‌های تعدیل و وضعیت حساب، به
[تعدیل و ایمنی حساب](/clawhub/moderation) مراجعه کنید. برای ادعاهای حق نشر یا سایر حقوق،
به [درخواست‌های حقوق محتوا](/clawhub/content-rights) مراجعه کنید.

## محتوای مجاز

ClawHub از محتوایی استقبال می‌کند که مفید، قابل‌فهم، و با حسن نیت
منتشر شده باشد.

| دسته‌بندی                                         | مجاز است وقتی                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| بهره‌وری توسعه‌دهنده                           | فهرست به کاربران کمک می‌کند نرم‌افزار بسازند، آزمایش کنند، مهاجرت دهند، اشکال‌زدایی کنند، مستندسازی کنند، یا راهبری کنند.                                               |
| گردش‌کارهای UI، داده، و اتوماسیون               | دامنه روشن است، اعتبارنامه‌های لازم صریح هستند، و اقدامات پرریسک شامل مسیرهای بازبینی، اجرای خشک، پیش‌نمایش، یا تأیید هستند. |
| امنیت دفاعی، تعدیل، و بازبینی سوءاستفاده | ابزار برای بازبینی مجاز چارچوب‌بندی شده، شواهد را حفظ می‌کند، و مرزهای تأیید انسانی را روشن نگه می‌دارد.                          |
| گردش‌کارهای شخصی یا تیمی                       | گردش‌کار از حساب‌های مبتنی بر رضایت، راه‌اندازی شفاف، و مجوزهای صریح استفاده می‌کند.                                            |
| کاتالوگ‌های نگهداری‌شده                              | هر فهرست متمایز، مفید، دقیق توصیف‌شده، و به‌طور معقول نگهداری‌شده است.                                                |

زمینه مهم است. یک موضوع یکسان می‌تواند در چارچوبی محدود، دفاعی، یا
مبتنی بر رضایت قابل‌قبول باشد و وقتی به‌صورت گردش‌کار سوءاستفاده بسته‌بندی شود غیرقابل‌قبول باشد.

## رفتار غیرمجاز بازارگاه

ClawHub همچنین نحوهٔ استفادهٔ ناشران از بازارگاه را بازبینی می‌کند. از ClawHub برای
دستکاری کشف، سنجه‌ها، نشانگرهای اعتماد، سامانه‌های تعدیل، یا توجه
کاربر استفاده نکنید.

رفتار غیرمجاز بازارگاه شامل این موارد است:

- انتشار انبوه تعداد زیادی فهرست کم‌زحمت، تکراری، جای‌نگهدار، یا
  تولیدشده توسط ماشین که به نظر نمی‌رسد ارزش واقعی برای کاربر داشته باشند
- پر کردن سطوح جست‌وجو یا دسته‌بندی با Skills یا Pluginهای تقریباً یکسان
- انتشار صدها فهرست با استفاده، نگهداری، شفافیت منبع،
  یا تمایز معنادار اندک یا بدون آن
- افزایش مصنوعی نصب‌ها، دانلودها، ستاره‌ها، یا سایر سنجه‌های تعامل
  از طریق اتوماسیون، چرخه‌های خودنصب، حساب‌های جعلی، فعالیت هماهنگ،
  تعامل پولی، یا سایر رفتارهای غیرطبیعی
- ایجاد یا چرخاندن حساب‌ها برای دور زدن تعدیل، مسدودسازی‌ها، محدودیت‌های ناشر، یا
  بازبینی بازارگاه
- گمراه‌کردن کاربران دربارهٔ مالکیت، منبع، قابلیت‌ها، وضعیت امنیتی،
  الزامات نصب، یا وابستگی به پروژه یا ناشر دیگر
- بارگذاری مکرر محتوایی که قبلاً پنهان، حذف، یا مسدود شده است
  بدون رفع مشکل زیربنایی

انتشار با حجم بالا به‌طور خودکار سوءاستفاده نیست. کاتالوگ‌های بزرگ زمانی قابل‌قبول هستند
که فهرست‌ها به‌طور معنادار متفاوت، دقیق توصیف‌شده، نگهداری‌شده،
و مورد استفادهٔ کاربران واقعی باشند. کاتالوگ‌های بزرگ وقتی به مشکل اعتماد و ایمنی تبدیل می‌شوند
که حجم با فهرست‌های کم‌محتوا، تکراری، گمراه‌کننده، نگهداری‌نشده، یا
به‌طور مصنوعی ترویج‌شده همراه باشد.

## حقوق محتوا

اگر معتقدید محتوایی در ClawHub حق نشر یا سایر حقوق شما را نقض می‌کند، از
[درخواست‌های حقوق محتوا](/clawhub/content-rights) استفاده کنید. از گزارش‌های عادی بازارگاه
برای ادعاهای حق نشر یا حقوق استفاده نکنید، مگر اینکه فهرست همچنین ناامن،
مخرب، یا گمراه‌کننده باشد.

## بازبینی و اجرا

ClawHub ممکن است از بررسی‌های خودکار، نشانگرهای آماری سوءاستفاده، گزارش‌های کاربران، و
بازبینی کارکنان برای شناسایی محتوای ناامن یا رفتار انتشار سوءاستفاده‌گرانه استفاده کند. یک نشانگر
به‌تنهایی سوءاستفاده را ثابت نمی‌کند؛ بلکه به ClawHub کمک می‌کند تصمیم بگیرد چه چیزی نیازمند بازبینی است.

ممکن است:

- فهرست‌های ناقض را پنهان، نگه‌داری، حذف، حذف نرم، یا، در مواردی که برای نوع منبع پشتیبانی می‌شود،
  حذف سخت کنیم
- دانلودها یا نصب‌ها را برای انتشارهای ناامن مسدود کنیم
- توکن‌های API را لغو کنیم
- محتوای مرتبط را حذف نرم کنیم
- دسترسی انتشار را محدود کنیم
- متخلفان تکراری یا شدید را مسدود کنیم

ما اجرای ابتدا-هشدار را برای سوءاستفادهٔ آشکار تضمین نمی‌کنیم. برای گزارش‌ها، توقف‌های تعدیل،
فهرست‌های پنهان، مسدودسازی‌ها، و وضعیت حساب به
[تعدیل و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
