# استفادهٔ قابل‌قبول

ClawHub میزبان Skills، Pluginها، بسته‌ها و فرادادهٔ بازارچه برای OpenClaw است.
از این صفحه برای تصمیم‌گیری استفاده کنید که آیا محتوا یا رفتار انتشار به
ClawHub تعلق دارد یا نه.

این قوانین دربارهٔ این اعمال می‌شوند که یک فهرست چه کاری انجام می‌دهد، از کاربران می‌خواهد چه چیزی را اجرا کنند، چگونه
خود را معرفی می‌کند، و ناشران چگونه از سطوح کشف، نصب و
اعتماد ClawHub استفاده می‌کنند. برای وضعیت‌های نظارت و اعتبار حساب، به
[نظارت و ایمنی حساب](/fa/clawhub/moderation) مراجعه کنید. برای ادعاهای حق نشر یا سایر حقوق،
[درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) را ببینید.

## محتوای مجاز

ClawHub از محتوایی استقبال می‌کند که مفید، قابل‌فهم و با حسن نیت منتشر شده
باشد.

| دسته | مجاز است وقتی |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| بهره‌وری توسعه‌دهنده | فهرست به کاربران کمک کند نرم‌افزار را بسازند، آزمایش کنند، مهاجرت دهند، اشکال‌زدایی کنند، مستندسازی کنند یا اجرا و اداره کنند. |
| گردش‌کارهای UI، داده و اتوماسیون | دامنه روشن باشد، اعتبارنامه‌های لازم صریح باشند، و اقدامات پرخطر شامل مسیرهای بازبینی، اجرای آزمایشی، پیش‌نمایش یا تأیید باشند. |
| امنیت دفاعی، نظارت و بازبینی سوءاستفاده | ابزار برای بازبینی مجاز چارچوب‌بندی شده باشد، شواهد را حفظ کند، و مرزهای تأیید انسانی را روشن نگه دارد. |
| گردش‌کارهای شخصی یا تیمی | گردش‌کار از حساب‌های مبتنی بر رضایت، راه‌اندازی شفاف و مجوزهای صریح استفاده کند. |
| کاتالوگ‌های نگهداری‌شده | هر فهرست متمایز، مفید، دقیقاً توصیف‌شده و به‌طور معقول نگهداری‌شده باشد. |

زمینه اهمیت دارد. یک موضوع یکسان می‌تواند در یک چارچوب محدود دفاعی یا
مبتنی بر رضایت قابل‌قبول باشد و وقتی به‌صورت گردش‌کار سوءاستفاده بسته‌بندی شود غیرقابل‌قبول باشد.

## محتوای غیرمجاز

ClawHub محتوایی را میزبانی نمی‌کند که هدف اصلی آن سوءاستفاده، فریب، اجرای ناایمن
یا نقض حقوق باشد.

| دسته | غیرمجاز |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| دسترسی غیرمجاز یا دور زدن امنیت | دور زدن احراز هویت، تصاحب حساب، سوءاستفاده از محدودیت نرخ، تصاحب تماس زنده یا عامل، سرقت نشست قابل‌استفادهٔ مجدد، یا تأیید خودکار جریان‌های جفت‌سازی برای کاربران تأییدنشده. |
| سوءاستفاده از پلتفرم و دور زدن ممنوعیت | حساب‌های مخفی پس از ممنوعیت، گرم‌کردن یا مزرعه‌سازی حساب، تعامل جعلی، اتوماسیون چندحسابی، ارسال انبوه، ربات‌های هرزنامه، یا اتوماسیونی که برای پرهیز از شناسایی ساخته شده است. |
| کلاهبرداری، فریب‌کاری و گردش‌کارهای مالی گمراه‌کننده | گواهی‌ها یا فاکتورهای جعلی، جریان‌های پرداخت گمراه‌کننده، ارتباط‌گیری کلاهبردارانه، اثبات اجتماعی جعلی، گردش‌کارهای هویت مصنوعی برای کلاهبرداری، یا ابزارهای خرج‌کردن/شارژکردن بدون تأیید انسانی روشن. |
| غنی‌سازی یا نظارت ناقض حریم خصوصی | برداشت مخاطبان برای هرزنامه، افشای اطلاعات شخصی، تعقیب، استخراج سرنخ همراه با ارتباط‌گیری ناخواسته، پایش پنهانی، تطبیق زیست‌سنجی بدون رضایت، یا استفاده از داده‌های افشاشده یا دامپ‌های رخنه. |
| جعل هویت یا دست‌کاری هویت بدون رضایت | تعویض چهره، دوقلوهای دیجیتال، اینفلوئنسرهای شبیه‌سازی‌شده، شخصیت‌های جعلی، یا ابزارهای دیگری که برای جعل هویت یا گمراه‌کردن استفاده می‌شوند. |
| محتوای جنسی صریح یا تولید بزرگسالان با ایمنی غیرفعال | تولید تصویر، ویدئو یا محتوای NSFW؛ پوشش‌های محتوای بزرگسالان پیرامون APIهای شخص ثالث؛ یا فهرست‌هایی که هدف اصلی آن‌ها محتوای جنسی صریح است. |
| الزامات اجرای پنهان، ناایمن یا گمراه‌کننده | فرمان‌های نصب مبهم‌سازی‌شده، نصب‌کننده‌های pipe-to-shell مانند محتوای دانلودشده که با `sh` یا `bash` بدون قابلیت بازبینی روشن اجرا می‌شود، الزامات اعلام‌نشدهٔ راز یا کلید خصوصی، اجرای راه دور `npx @latest` بدون قابلیت بازبینی روشن، یا فراداده‌ای که پنهان می‌کند فهرست واقعاً برای اجرا به چه چیزهایی نیاز دارد. |
| مواد ناقض حق نشر یا ناقض حقوق | بازنشر skill، plugin، مستندات، دارایی‌های برند یا کد اختصاصی شخص دیگر بدون اجازه؛ نقض شرایط مجوز؛ یا جعل هویت نویسنده یا ناشر اصلی. |

## رفتار غیرمجاز در بازارچه

ClawHub همچنین نحوهٔ استفادهٔ ناشران از بازارچه را بازبینی می‌کند. از ClawHub برای
دست‌کاری کشف، معیارها، نشانه‌های اعتماد، سامانه‌های نظارت یا توجه
کاربران استفاده نکنید.

رفتار غیرمجاز در بازارچه شامل این موارد است:

- انتشار انبوه تعداد زیادی فهرست کم‌زحمت، تکراری، جای‌نگهدار یا
  تولیدشده با ماشین که به نظر نمی‌رسد ارزش واقعی برای کاربر داشته باشند
- غرق‌کردن سطوح جست‌وجو یا دسته‌بندی با Skills یا Pluginهای تقریباً یکسان
- انتشار صدها فهرست با استفاده، نگهداری، شفافیت منبع یا
  تمایز معنادار اندک یا بدون آن
- افزایش مصنوعی نصب‌ها، دانلودها، ستاره‌ها یا سایر معیارهای تعامل
  از طریق اتوماسیون، حلقه‌های خودنصب، حساب‌های جعلی، فعالیت هماهنگ،
  تعامل پولی یا رفتار غیرارگانیک دیگر
- ایجاد یا چرخاندن حساب‌ها برای دور زدن نظارت، ممنوعیت‌ها، محدودیت‌های ناشر یا
  بازبینی بازارچه
- گمراه‌کردن کاربران دربارهٔ مالکیت، منبع، قابلیت‌ها، وضعیت امنیتی،
  الزامات نصب، یا وابستگی به پروژه یا ناشر دیگر
- بارگذاری مکرر محتوایی که پیش‌تر پنهان، حذف یا مسدود شده است
  بدون رفع مشکل زیربنایی

انتشار با حجم بالا به‌طور خودکار سوءاستفاده نیست. کاتالوگ‌های بزرگ زمانی قابل‌قبول‌اند
که فهرست‌ها به‌طور معنادار متفاوت، دقیقاً توصیف‌شده، نگهداری‌شده
و توسط کاربران واقعی استفاده شوند. کاتالوگ‌های بزرگ وقتی به مشکل اعتماد و ایمنی تبدیل می‌شوند که
حجم با فهرست‌های کم‌مایه، تکراری، گمراه‌کننده، نگهداری‌نشده یا
به‌طور مصنوعی ترویج‌شده همراه باشد.

## حقوق محتوا

اگر باور دارید محتوایی در ClawHub حق نشر یا سایر حقوق شما را نقض می‌کند، از
[درخواست‌های حقوق محتوا](/fa/clawhub/content-rights) استفاده کنید. برای ادعاهای حق نشر یا حقوق از گزارش‌های عادی بازارچه
استفاده نکنید مگر آن‌که فهرست همچنین ناایمن،
مخرب یا گمراه‌کننده باشد.

## بازبینی و اجرا

ClawHub ممکن است از بررسی‌های خودکار، نشانه‌های آماری سوءاستفاده، گزارش‌های کاربران و
بازبینی کارکنان برای شناسایی محتوای ناایمن یا رفتار انتشار سوءاستفاده‌گرانه استفاده کند. یک نشانه
به‌خودی‌خود سوءاستفاده را ثابت نمی‌کند؛ بلکه به ClawHub کمک می‌کند تصمیم بگیرد چه چیزی نیاز به بازبینی دارد.

ممکن است:

- فهرست‌های ناقض را پنهان کنیم، نگه داریم، حذف کنیم، soft-delete کنیم، یا در جایی که برای نوع منبع پشتیبانی می‌شود،
  hard-delete کنیم
- دانلودها یا نصب‌ها را برای انتشارهای ناایمن مسدود کنیم
- توکن‌های API را باطل کنیم
- محتوای مرتبط را soft-delete کنیم
- دسترسی انتشار را محدود کنیم
- متخلفان تکراری یا شدید را ممنوع کنیم

ما اجرای هشدار-اول را برای سوءاستفادهٔ آشکار تضمین نمی‌کنیم. برای گزارش‌ها، نگه‌داشت‌های نظارتی،
فهرست‌های پنهان، ممنوعیت‌ها و اعتبار حساب، به
[نظارت و ایمنی حساب](/fa/clawhub/moderation) مراجعه کنید.
