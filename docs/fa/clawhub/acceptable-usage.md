# استفاده قابل قبول

ClawHub میزبان مهارت‌ها، افزونه‌ها، بسته‌ها، و فراداده بازارچه برای OpenClaw است.
از این صفحه برای تصمیم‌گیری درباره اینکه آیا محتوا یا رفتار انتشار به
ClawHub تعلق دارد یا نه استفاده کنید.

این قوانین درباره کاری که یک فهرست انجام می‌دهد، آنچه از کاربران می‌خواهد اجرا کنند، نحوه
معرفی خودش، و چگونگی استفاده ناشران از سطوح کشف، نصب، و اعتماد
ClawHub اعمال می‌شوند. برای وضعیت‌های نظارت و جایگاه حساب، به
[نظارت و ایمنی حساب](/clawhub/moderation) مراجعه کنید. برای ادعاهای حق نشر یا سایر حقوق،
به [درخواست‌های حقوق محتوا](/clawhub/content-rights) مراجعه کنید.

## محتوای مجاز

ClawHub از محتوایی استقبال می‌کند که مفید، قابل فهم، و با حسن نیت منتشر شده باشد.

| دسته | مجاز است وقتی |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| بهره‌وری توسعه‌دهنده | فهرست به کاربران کمک کند نرم‌افزار بسازند، آزمایش کنند، مهاجرت دهند، اشکال‌زدایی کنند، مستندسازی کنند، یا به‌کار بگیرند. |
| گردش‌کارهای UI، داده، و خودکارسازی | دامنه روشن باشد، اعتبارنامه‌های لازم صریح باشند، و اقدامات پرخطر مسیرهای بازبینی، اجرای آزمایشی، پیش‌نمایش، یا تأیید داشته باشند. |
| امنیت دفاعی، نظارت، و بازبینی سوءاستفاده | ابزار برای بازبینی مجاز ارائه شده باشد، شواهد را حفظ کند، و مرزهای تأیید انسانی را روشن نگه دارد. |
| گردش‌کارهای شخصی یا تیمی | گردش‌کار از حساب‌های مبتنی بر رضایت، راه‌اندازی شفاف، و مجوزهای صریح استفاده کند. |
| کاتالوگ‌های نگهداری‌شده | هر فهرست متمایز، مفید، دقیقاً توصیف‌شده، و به‌طور معقول نگهداری‌شده باشد. |

زمینه اهمیت دارد. یک موضوع یکسان می‌تواند در یک محیط دفاعی محدود یا
مبتنی بر رضایت قابل قبول باشد و وقتی به‌عنوان گردش‌کار سوءاستفاده بسته‌بندی شود غیرقابل قبول باشد.

یا نقض حقوق باشد میزبانی نمی‌کند.

| دسته | غیرمجاز |

| سوءاستفاده از پلتفرم و دور زدن ممنوعیت | حساب‌های مخفی پس از ممنوعیت، گرم‌کردن یا پرورش حساب، تعامل جعلی، خودکارسازی چندحسابی، ارسال انبوه، ربات‌های هرزنامه، یا خودکارسازی ساخته‌شده برای اجتناب از شناسایی. |
| تقلب، کلاهبرداری، و گردش‌کارهای مالی فریبنده | گواهی‌ها یا فاکتورهای جعلی، جریان‌های پرداخت فریبنده، ارتباط‌گیری کلاهبردارانه، اعتبار اجتماعی جعلی، گردش‌کارهای هویت مصنوعی برای تقلب، یا ابزارهای خرج‌کردن/دریافت هزینه بدون تأیید انسانی روشن. |
| غنی‌سازی یا پایش ناقض حریم خصوصی | جمع‌آوری مخاطب برای هرزنامه، افشای اطلاعات شخصی، تعقیب، استخراج سرنخ همراه با ارتباط‌گیری ناخواسته، پایش پنهانی، تطبیق زیستی بدون رضایت، یا استفاده از داده‌های درزکرده یا بسته‌های داده رخنه. |
| جعل هویت یا دستکاری هویت بدون رضایت | تعویض چهره، دوقلوهای دیجیتال، اینفلوئنسرهای شبیه‌سازی‌شده، شخصیت‌های جعلی، یا ابزارهای دیگری که برای جعل هویت یا گمراه‌کردن استفاده می‌شوند. |
| محتوای جنسی صریح یا تولید بزرگسال با ایمنی غیرفعال | تولید تصویر، ویدئو، یا محتوای نامناسب برای محیط کار؛ پوشش‌دهنده‌های محتوای بزرگسال روی رابط‌های برنامه‌نویسی کاربردی شخص ثالث؛ یا فهرست‌هایی که هدف اصلی آن‌ها محتوای جنسی صریح است. |
| نیازمندی‌های اجرای پنهان، ناامن، یا گمراه‌کننده | دستورهای نصب مبهم‌سازی‌شده، نصب‌کننده‌های انتقال مستقیم به پوسته مانند محتوای دانلودشده‌ای که با `sh` یا `bash` بدون قابلیت بازبینی روشن اجرا می‌شود، نیازمندی‌های اعلام‌نشده برای راز یا کلید خصوصی، اجرای راه‌دور `npx @latest` بدون قابلیت بازبینی روشن، یا فراداده‌ای که آنچه فهرست واقعاً برای اجرا نیاز دارد را پنهان می‌کند. |
| ماده ناقض حق نشر یا ناقض حقوق | بازنشر مهارت، افزونه، مستندات، دارایی‌های برند، یا کد اختصاصی شخص دیگر بدون اجازه؛ نقض شرایط مجوز؛ یا جعل هویت نویسنده یا ناشر اصلی. |

## رفتار غیرمجاز بازارچه

ClawHub همچنین نحوه استفاده ناشران از بازارچه را بازبینی می‌کند. از ClawHub برای
دستکاری کشف، معیارها، نشانه‌های اعتماد، سامانه‌های نظارت، یا توجه کاربر
استفاده نکنید.

رفتار غیرمجاز بازارچه شامل موارد زیر است:

- انتشار انبوه تعداد زیادی فهرست کم‌زحمت، تکراری، جای‌نگهدار، یا
  تولیدشده با ماشین که به نظر نمی‌رسد ارزش واقعی برای کاربر داشته باشند
- پر کردن سطوح جست‌وجو یا دسته‌بندی با مهارت‌ها یا افزونه‌های تقریباً یکسان
- انتشار صدها فهرست با استفاده، نگهداری، شفافیت منبع،
  یا تمایز معنادار اندک یا بدون آن
- افزایش مصنوعی نصب‌ها، دانلودها، ستاره‌ها، یا سایر معیارهای تعامل
  از طریق خودکارسازی، چرخه‌های نصب توسط خود، حساب‌های جعلی، فعالیت
  هماهنگ، تعامل پولی، یا سایر رفتارهای غیرطبیعی
- ایجاد یا چرخاندن حساب‌ها برای دور زدن نظارت، ممنوعیت‌ها، محدودیت‌های ناشر، یا
  بازبینی بازارچه
- گمراه‌کردن کاربران درباره مالکیت، منبع، توانایی‌ها، وضعیت امنیتی،
  نیازمندی‌های نصب، یا وابستگی به پروژه یا ناشر دیگر
- بارگذاری مکرر محتوایی که قبلاً پنهان، حذف، یا مسدود شده است
  بدون رفع مشکل زیربنایی

انتشار با حجم بالا به‌طور خودکار سوءاستفاده نیست. کاتالوگ‌های بزرگ زمانی قابل قبول‌اند
که فهرست‌ها به‌طور معنادار متفاوت، دقیقاً توصیف‌شده، نگهداری‌شده،
و توسط کاربران واقعی استفاده شوند. کاتالوگ‌های بزرگ زمانی به مشکل اعتماد و ایمنی تبدیل می‌شوند که
حجم با فهرست‌های کم‌مایه، تکراری، گمراه‌کننده، نگهداری‌نشده، یا
به‌طور مصنوعی ترویج‌شده همراه باشد.

## حقوق محتوا

اگر باور دارید محتوایی در ClawHub حق نشر یا سایر حقوق شما را نقض می‌کند، از
[درخواست‌های حقوق محتوا](/clawhub/content-rights) استفاده کنید. از گزارش‌های معمول بازارچه
برای ادعاهای حق نشر یا حقوق استفاده نکنید، مگر اینکه فهرست همچنین ناامن،
مخرب، یا گمراه‌کننده باشد.

## بازبینی و اجرا

ClawHub ممکن است از بررسی‌های خودکار، نشانه‌های آماری سوءاستفاده، گزارش‌های کاربران، و
بازبینی کارکنان برای شناسایی محتوای ناامن یا رفتار انتشار سوءاستفاده‌آمیز استفاده کند. یک نشانه
به‌تنهایی سوءاستفاده را ثابت نمی‌کند؛ به ClawHub کمک می‌کند تصمیم بگیرد چه چیزی به بازبینی نیاز دارد.

ما ممکن است:

- فهرست‌های ناقض را پنهان کنیم، نگه داریم، حذف کنیم، حذف نرم کنیم، یا، در جایی که برای نوع منبع پشتیبانی می‌شود،
  حذف سخت کنیم
- دانلودها یا نصب‌ها را برای انتشارهای ناامن مسدود کنیم
- توکن‌های رابط برنامه‌نویسی کاربردی را باطل کنیم
- محتوای مرتبط را حذف نرم کنیم
- دسترسی انتشار را محدود کنیم
- متخلفان تکراری یا شدید را ممنوع کنیم

ما اجرای اخطار-اول را برای سوءاستفاده آشکار تضمین نمی‌کنیم. برای گزارش‌ها، نگه‌داشت‌های نظارتی،
فهرست‌های پنهان، ممنوعیت‌ها، و جایگاه حساب به
[نظارت و ایمنی حساب](/clawhub/moderation) مراجعه کنید.
