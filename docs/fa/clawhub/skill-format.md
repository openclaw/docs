# قالب Skill

## روی دیسک

هر Skill یک پوشه است.

الزامی:

- `SKILL.md` (یا `skill.md`؛ قالب قدیمی `skills.md` نیز پذیرفته می‌شود)

اختیاری:

- هر فایل پشتیبان _مبتنی بر متن_ (به «فایل‌های مجاز» مراجعه کنید)
- `.clawhubignore` (الگوهای نادیده‌گیری برای انتشار، قالب قدیمی `.clawdhubignore`)
- `.gitignore` (این مورد نیز رعایت می‌شود)

## درون‌ریزی از GitHub

درون‌ریز وب GitHub از انتشار/همگام‌سازی محلی سخت‌گیرانه‌تر است. این ابزار فقط فایل‌های
`SKILL.md` یا قالب قدیمی `skills.md` را در مخزن‌های عمومی و غیرفورک که متعلق به
حساب واردشده GitHub هستند، شناسایی می‌کند. این ابزار مخزن‌های خصوصی، فورک‌ها،
مخزن‌های بایگانی‌شده/غیرفعال یا مخزن‌های عمومی شخص ثالث را درون‌ریزی نمی‌کند.

فراداده نصب محلی (نوشته‌شده توسط CLI):

- `<skill>/.clawhub/origin.json` (قالب قدیمی `.clawdhub`)

وضعیت نصب پوشه کاری (نوشته‌شده توسط CLI):

- `<workdir>/.clawhub/lock.json` (قالب قدیمی `.clawdhub`)

## `SKILL.md`

- Markdown با frontmatter اختیاری YAML.
- سرور هنگام انتشار، فراداده را از frontmatter استخراج می‌کند.
- `description` به‌عنوان خلاصه Skill در رابط کاربری/جست‌وجو استفاده می‌شود.

برای Agent Skills قابل‌انتقال، `name` باید با پوشه والد مطابقت داشته باشد و از
1 تا 64 حرف کوچک، عدد یا خط تیره تشکیل شود. ClawHub اسلاگ قابل‌مسیریابی و
نام نمایشی کاتالوگ را جدا نگه می‌دارد؛ بنابراین نام‌های موجود از کلاینت‌های دیگر همچنان
قابل‌انتشار می‌مانند و بی‌سروصدا بازنویسی نمی‌شوند. فهرست‌های کاتالوگ ممکن است نام‌های طولانی را
بدون تغییر نام ذخیره‌شده، از نظر بصری کوتاه کنند.

## فراداده frontmatter

فراداده Skill در frontmatter از نوع YAML در ابتدای `SKILL.md` تعریف می‌شود. این بخش به رجیستری (و تحلیل امنیتی) اعلام می‌کند Skill برای اجرا به چه چیزهایی نیاز دارد.

### frontmatter پایه

```yaml
---
name: my-skill
description: خلاصه‌ای کوتاه از کاری که این Skill انجام می‌دهد.
version: 1.0.0
---
```

### فراداده زمان اجرا (`metadata.openclaw`)

نیازمندی‌های زمان اجرای Skill را زیر `metadata.openclaw` تعریف کنید (نام‌های مستعار: `metadata.clawdbot`، `metadata.clawdis`).

```yaml
---
name: my-skill
description: مدیریت وظایف از طریق API سرویس Todoist.
metadata:
  openclaw:
    requires:
      env:
        - TODOIST_API_KEY
      bins:
        - curl
    primaryEnv: TODOIST_API_KEY
---
```

برای متغیرهای محیطی که باید پیش از اجرای Skill موجود باشند، از `requires.env` استفاده کنید. هنگامی که برای هر متغیر به فراداده نیاز دارید، از جمله متغیرهای اختیاری دارای `required: false`، از `envVars` استفاده کنید.

### مرجع کامل فیلدها

| فیلد              | نوع       | توضیحات                                                                                                                                  |
| ------------------ | ---------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| `requires.env`     | `string[]` | متغیرهای محیطی الزامی مورد انتظار Skill.                                                                                           |
| `requires.bins`    | `string[]` | فایل‌های اجرایی CLI که باید همگی نصب شده باشند.                                                                                                     |
| `requires.anyBins` | `string[]` | فایل‌های اجرایی CLI که دست‌کم یکی از آن‌ها باید موجود باشد.                                                                                                  |
| `requires.config`  | `string[]` | مسیر فایل‌های پیکربندی که Skill می‌خواند.                                                                                                          |
| `primaryEnv`       | `string`   | متغیر محیطی اصلی اعتبارنامه برای Skill.                                                                                                  |
| `envVars`          | `array`    | تعریف متغیرهای محیطی با `name`، `required` اختیاری و `description` اختیاری. برای متغیرهای محیطی اختیاری، `required: false` را تنظیم کنید. |
| `always`           | `boolean`  | اگر `true` باشد، Skill همیشه فعال است (به نصب صریح نیازی نیست).                                                                              |
| `skillKey`         | `string`   | کلید فراخوانی Skill را بازنویسی می‌کند.                                                                                                         |
| `emoji`            | `string`   | ایموجی نمایشی Skill.                                                                                                                 |
| `homepage`         | `string`   | نشانی URL صفحه اصلی یا مستندات Skill.                                                                                                         |
| `os`               | `string[]` | محدودیت‌های سیستم‌عامل (برای مثال `["macos"]`، `["linux"]`).                                                                                             |
| `install`          | `array`    | مشخصات نصب وابستگی‌ها (در ادامه ببینید).                                                                                                  |
| `nix`              | `object`   | مشخصات Plugin مربوط به Nix (به README مراجعه کنید).                                                                                                                |
| `config`           | `object`   | مشخصات پیکربندی Clawdbot (به README مراجعه کنید).                                                                                                           |

### مشخصات نصب

اگر Skill به نصب وابستگی‌ها نیاز دارد، آن‌ها را در آرایه `install` تعریف کنید:

```yaml
metadata:
  openclaw:
    install:
      - kind: brew
        formula: jq
        bins: [jq]
      - kind: node
        package: typescript
        bins: [tsc]
```

انواع نصب پشتیبانی‌شده: `brew`، `node`، `go`، `uv`.

### متغیرهای محیطی اختیاری

متغیرهای محیطی اختیاری را زیر `metadata.openclaw.envVars` تعریف و `required: false` را تنظیم کنید. ورودی‌های اختیاری را به `requires.env` اضافه نکنید، زیرا `requires.env` به این معناست که Skill بدون آن‌ها قابل‌اجرا نیست.

```yaml
metadata:
  openclaw:
    primaryEnv: TODOIST_API_KEY
    envVars:
      - name: TODOIST_API_KEY
        required: true
        description: توکن API سرویس Todoist که برای درخواست‌های احراز هویت‌شده استفاده می‌شود.
      - name: TODOIST_PROJECT_ID
        required: false
        description: شناسه اختیاری پروژه پیش‌فرض برای زمانی که کاربر پروژه‌ای را مشخص نمی‌کند.
```

### اهمیت این موضوع

تحلیل امنیتی ClawHub بررسی می‌کند که موارد تعریف‌شده توسط Skill با عملکرد واقعی آن مطابقت داشته باشند. اگر کد به `TODOIST_API_KEY` ارجاع دهد، اما frontmatter آن را زیر `requires.env`، `primaryEnv` یا `envVars` تعریف نکرده باشد، تحلیل یک عدم تطابق فراداده را علامت‌گذاری می‌کند. دقیق نگه‌داشتن تعریف‌ها به Skill کمک می‌کند بررسی را با موفقیت پشت سر بگذارد و به کاربران کمک می‌کند بدانند چه چیزی را نصب می‌کنند.

### نمونه: frontmatter کامل

```yaml
---
name: todoist-cli
description: مدیریت وظایف، پروژه‌ها و برچسب‌های Todoist از خط فرمان.
version: 1.2.0
metadata:
  openclaw:
    requires:
      env:
        - TODOIST_API_KEY
      bins:
        - curl
    primaryEnv: TODOIST_API_KEY
    envVars:
      - name: TODOIST_API_KEY
        required: true
        description: توکن API سرویس Todoist.
      - name: TODOIST_PROJECT_ID
        required: false
        description: شناسه اختیاری پروژه پیش‌فرض.
    emoji: "\u2705"
    homepage: https://github.com/example/todoist-cli
---
```

## فایل‌های مجاز

هنگام انتشار، فقط فایل‌های «مبتنی بر متن» پذیرفته می‌شوند.

- فهرست مجاز پسوندها در `packages/schema/src/textFiles.ts` (`TEXT_FILE_EXTENSIONS`) قرار دارد.
- فایل‌های اسکریپت پس از بارگذاری همچنان اسکن می‌شوند؛ فایل‌های PowerShell با پسوندهای `.ps1`، `.psm1` و `.psd1` به‌عنوان متن پذیرفته می‌شوند.
- انواع محتوایی که با `text/` آغاز می‌شوند، به‌عنوان متن در نظر گرفته می‌شوند؛ افزون بر این، فهرست مجاز کوچکی نیز وجود دارد (JSON/YAML/TOML/JS/TS/Markdown/SVG).

محدودیت‌ها (سمت سرور):

- اندازه کل بسته: 50MB.
- متن تعبیه‌سازی شامل `SKILL.md` به‌همراه حداکثر حدود 40 فایل غیر `.md` است (سقف بر مبنای بیشترین تلاش).

## اسلاگ‌ها

- به‌طور پیش‌فرض از نام پوشه ساخته می‌شود.
- محدوده‌های بسته باید دقیقاً با شناسه ناشر در ClawHub مطابقت داشته باشند. شناسه‌های ناشر می‌توانند شامل حروف کوچک، اعداد، خط تیره، نقطه و زیرخط باشند؛ آن‌ها باید با یک حرف کوچک یا عدد شروع و پایان یابند.
- اسلاگ‌های بسته باید با حروف کوچک و برای npm ایمن باشند؛ برای مثال `@example.tools/demo-plugin` یا `demo-plugin`.

## نسخه‌بندی و برچسب‌ها

- هر انتشار یک نسخه جدید ایجاد می‌کند (semver).
- برچسب‌ها اشاره‌گرهای رشته‌ای به یک نسخه هستند؛ `latest` معمولاً استفاده می‌شود.

## مجوز

- همه Skillهای منتشرشده در ClawHub تحت مجوز `MIT-0` عرضه می‌شوند.
- همه می‌توانند Skillهای منتشرشده را، از جمله برای مقاصد تجاری، استفاده، تغییر و بازتوزیع کنند.
- ذکر منبع الزامی نیست.
- شرایط مجوز متعارض را به `SKILL.md` اضافه نکنید؛ ClawHub از بازنویسی مجوز برای هر Skill پشتیبانی نمی‌کند.

## Skillهای پولی

- ClawHub از Skillهای پولی، قیمت‌گذاری برای هر Skill، دیوار پرداخت یا تقسیم درآمد پشتیبانی نمی‌کند.
- فراداده قیمت‌گذاری را به `SKILL.md` اضافه نکنید؛ این مورد بخشی از قالب Skill نیست و یک Skill منتشرشده را پولی نمی‌کند.
- اگر Skill با یک سرویس پولی شخص ثالث یکپارچه می‌شود، هزینه خارجی و حساب الزامی را به‌روشنی در دستورالعمل‌های Skill و تعریف‌های متغیر محیطی مستند کنید (`requires.env` برای متغیرهای الزامی، یا `envVars` همراه با `required: false` برای متغیرهای اختیاری).
