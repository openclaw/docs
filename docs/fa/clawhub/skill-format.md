# قالب Skill

## روی دیسک

یک Skill یک پوشه است.

الزامی:

- `SKILL.md` (یا `skill.md`؛ قالب قدیمی `skills.md` نیز پذیرفته می‌شود)

اختیاری:

- هرگونه فایل پشتیبان _مبتنی بر متن_ (به «فایل‌های مجاز» مراجعه کنید)
- `.clawhubignore` (الگوهای نادیده‌گرفتن برای انتشار، قالب قدیمی `.clawdhubignore`)
- `.gitignore` (این فایل نیز رعایت می‌شود)

## درون‌ریزی از GitHub

درون‌ریز وب GitHub از انتشار/همگام‌سازی محلی سخت‌گیرانه‌تر است. این ابزار فقط فایل‌های
`SKILL.md` یا قالب قدیمی `skills.md` را در مخزن‌های عمومی و غیرفورک که متعلق به
حساب واردشده GitHub هستند، پیدا می‌کند. مخزن‌های خصوصی، فورک‌ها،
مخزن‌های بایگانی‌شده/غیرفعال یا مخزن‌های عمومی اشخاص ثالث را درون‌ریزی نمی‌کند.

فراداده نصب محلی (نوشته‌شده توسط CLI):

- `<skill>/.clawhub/origin.json` (قالب قدیمی `.clawdhub`)

وضعیت نصب در دایرکتوری کاری (نوشته‌شده توسط CLI):

- `<workdir>/.clawhub/lock.json` (قالب قدیمی `.clawdhub`)

## `SKILL.md`

- Markdown با فرانت‌متر اختیاری YAML.
- سرور هنگام انتشار، فراداده را از فرانت‌متر استخراج می‌کند.
- از `description` به‌عنوان خلاصه Skill در رابط کاربری/جست‌وجو استفاده می‌شود.

برای Agent Skills قابل‌انتقال، `name` باید با دایرکتوری والد مطابقت داشته باشد و شامل
۱ تا ۶۴ حرف کوچک، عدد یا خط تیره باشد. ClawHub نامک قابل‌مسیریابی و
نام نمایشی کاتالوگ را جدا نگه می‌دارد؛ بنابراین نام‌های موجود از کلاینت‌های دیگر همچنان
قابل‌انتشار هستند و بدون اطلاع بازنویسی نمی‌شوند. فهرست‌های کاتالوگ ممکن است نام‌های طولانی را
بدون تغییر نام ذخیره‌شده، به‌صورت بصری کوتاه کنند.

## فراداده فرانت‌متر

فراداده Skill در فرانت‌متر YAML بالای فایل `SKILL.md` شما اعلام می‌شود. این بخش به رجیستری (و تحلیل امنیتی) می‌گوید Skill شما برای اجرا به چه چیزهایی نیاز دارد.

### فرانت‌متر پایه

```yaml
---
name: my-skill
description: خلاصه‌ای کوتاه از کاری که این Skill انجام می‌دهد.
version: 1.0.0
---
```

### فراداده زمان اجرا (`metadata.openclaw`)

نیازمندی‌های زمان اجرای Skill خود را زیر `metadata.openclaw` اعلام کنید (نام‌های مستعار: `metadata.clawdbot`، `metadata.clawdis`).

```yaml
---
name: my-skill
description: مدیریت وظایف از طریق API سرویس Todoist.
metadata:
  openclaw:
    requires:
      env:
        - TODOIST_API_KEY
      bins:
        - curl
    primaryEnv: TODOIST_API_KEY
---
```

برای متغیرهای محیطی که باید پیش از اجرای Skill موجود باشند، از `requires.env` استفاده کنید. هنگامی که به فراداده جداگانه برای هر متغیر نیاز دارید، از جمله متغیرهای اختیاری با `required: false`، از `envVars` استفاده کنید.

### مرجع کامل فیلدها

| فیلد               | نوع        | توضیحات                                                                                                                                                     |
| ------------------ | ---------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `requires.env`     | `string[]` | متغیرهای محیطی الزامی که Skill شما انتظار دارد.                                                                                                            |
| `requires.bins`    | `string[]` | فایل‌های اجرایی CLI که همگی باید نصب شده باشند.                                                                                                             |
| `requires.anyBins` | `string[]` | فایل‌های اجرایی CLI که دست‌کم یکی از آن‌ها باید موجود باشد.                                                                                                 |
| `requires.config`  | `string[]` | مسیر فایل‌های پیکربندی که Skill شما می‌خواند.                                                                                                               |
| `primaryEnv`       | `string`   | متغیر محیطی اصلی حاوی اعتبارنامه برای Skill شما.                                                                                                            |
| `envVars`          | `array`    | اعلان متغیرهای محیطی با `name`، فیلد اختیاری `required` و فیلد اختیاری `description`. برای متغیرهای محیطی اختیاری، `required: false` را تنظیم کنید. |
| `always`           | `boolean`  | اگر `true` باشد، Skill همیشه فعال است (به نصب صریح نیازی نیست).                                                                                              |
| `skillKey`         | `string`   | بازنویسی کلید فراخوانی Skill.                                                                                                                               |
| `emoji`            | `string`   | ایموجی نمایشی Skill.                                                                                                                                        |
| `homepage`         | `string`   | نشانی URL صفحه اصلی یا مستندات Skill.                                                                                                                       |
| `os`               | `string[]` | محدودیت‌های سیستم‌عامل (برای مثال `["macos"]`، `["linux"]`).                                                                                                |
| `install`          | `array`    | مشخصات نصب وابستگی‌ها (به بخش زیر مراجعه کنید).                                                                                                             |
| `nix`              | `object`   | مشخصات Plugin برای Nix (به README مراجعه کنید).                                                                                                             |
| `config`           | `object`   | مشخصات پیکربندی Clawdbot (به README مراجعه کنید).                                                                                                           |

### مشخصات نصب

اگر Skill شما به نصب وابستگی‌ها نیاز دارد، آن‌ها را در آرایه `install` اعلام کنید:

```yaml
metadata:
  openclaw:
    install:
      - kind: brew
        formula: jq
        bins: [jq]
      - kind: node
        package: typescript
        bins: [tsc]
```

انواع نصب پشتیبانی‌شده: `brew`، `node`، `go`، `uv`.

### متغیرهای محیطی اختیاری

متغیرهای محیطی اختیاری را زیر `metadata.openclaw.envVars` اعلام کنید و `required: false` را تنظیم کنید. ورودی‌های اختیاری را به `requires.env` اضافه نکنید، زیرا `requires.env` یعنی Skill بدون آن‌ها نمی‌تواند اجرا شود.

```yaml
metadata:
  openclaw:
    primaryEnv: TODOIST_API_KEY
    envVars:
      - name: TODOIST_API_KEY
        required: true
        description: توکن API سرویس Todoist که برای درخواست‌های احراز هویت‌شده استفاده می‌شود.
      - name: TODOIST_PROJECT_ID
        required: false
        description: شناسه اختیاری پروژه پیش‌فرض، هنگامی که کاربر پروژه‌ای را مشخص نمی‌کند.
```

### اهمیت این موضوع

تحلیل امنیتی ClawHub بررسی می‌کند آنچه Skill شما اعلام کرده با کاری که واقعاً انجام می‌دهد مطابقت داشته باشد. اگر کد شما به `TODOIST_API_KEY` ارجاع دهد، اما فرانت‌متر آن را زیر `requires.env`، `primaryEnv` یا `envVars` اعلام نکرده باشد، تحلیل یک عدم تطابق فراداده را علامت‌گذاری می‌کند. دقیق نگه‌داشتن اعلان‌ها به Skill شما کمک می‌کند بررسی را با موفقیت بگذراند و به کاربران کمک می‌کند بدانند چه چیزی را نصب می‌کنند.

### نمونه: فرانت‌متر کامل

```yaml
---
name: todoist-cli
description: مدیریت وظایف، پروژه‌ها و برچسب‌های Todoist از خط فرمان.
version: 1.2.0
metadata:
  openclaw:
    requires:
      env:
        - TODOIST_API_KEY
      bins:
        - curl
    primaryEnv: TODOIST_API_KEY
    envVars:
      - name: TODOIST_API_KEY
        required: true
        description: توکن API سرویس Todoist.
      - name: TODOIST_PROJECT_ID
        required: false
        description: شناسه اختیاری پروژه پیش‌فرض.
    emoji: "\u2705"
    homepage: https://github.com/example/todoist-cli
---
```

## فایل‌های مجاز

فقط فایل‌های «مبتنی بر متن» برای انتشار پذیرفته می‌شوند.

- فهرست مجاز پسوندها در `packages/schema/src/textFiles.ts` (`TEXT_FILE_EXTENSIONS`) قرار دارد.
- فایل‌های اسکریپت پس از بارگذاری نیز اسکن می‌شوند؛ فایل‌های PowerShell با پسوندهای `.ps1`، `.psm1` و `.psd1` به‌عنوان متن پذیرفته می‌شوند.
- انواع محتوایی که با `text/` شروع می‌شوند، متن در نظر گرفته می‌شوند؛ افزون بر آن‌ها، یک فهرست مجاز کوچک نیز وجود دارد (JSON/YAML/TOML/JS/TS/Markdown/SVG).

محدودیت‌ها (سمت سرور):

- اندازه کل بسته: 50MB.
- متن تعبیه‌شده شامل `SKILL.md` به‌علاوه حداکثر حدود ۴۰ فایل غیر`.md` است (سقف به‌صورت بهترین تلاش اعمال می‌شود).

## نامک‌ها

- به‌طور پیش‌فرض از نام پوشه مشتق می‌شوند.
- محدوده‌های بسته باید دقیقاً با شناسه ناشر ClawHub مطابقت داشته باشند. شناسه‌های ناشر می‌توانند از حروف کوچک، اعداد، خط تیره، نقطه و زیرخط استفاده کنند؛ آن‌ها باید با یک حرف کوچک یا عدد شروع و پایان یابند.
- نامک‌های بسته باید با حروف کوچک و برای npm ایمن باشند؛ برای مثال `@example.tools/demo-plugin` یا `demo-plugin`.

## نسخه‌بندی و برچسب‌ها

- هر انتشار یک نسخه جدید ایجاد می‌کند (semver).
- برچسب‌ها اشاره‌گرهای رشته‌ای به یک نسخه هستند؛ `latest` معمولاً استفاده می‌شود.

## مجوز

- همه Skillهای منتشرشده در ClawHub تحت مجوز `MIT-0` هستند.
- هر فردی می‌تواند Skillهای منتشرشده را استفاده، اصلاح و بازتوزیع کند، از جمله برای مقاصد تجاری.
- ذکر منبع الزامی نیست.
- شرایط مجوز متعارض را به `SKILL.md` اضافه نکنید؛ ClawHub از بازنویسی مجوز برای هر Skill پشتیبانی نمی‌کند.

## Skillهای پولی

- ClawHub از Skillهای پولی، قیمت‌گذاری جداگانه برای هر Skill، دیوار پرداخت یا تقسیم درآمد پشتیبانی نمی‌کند.
- فراداده قیمت‌گذاری را به `SKILL.md` اضافه نکنید؛ این فراداده بخشی از قالب Skill نیست و باعث پولی‌شدن Skill منتشرشده نمی‌شود.
- اگر Skill شما با یک سرویس پولی شخص ثالث یکپارچه می‌شود، هزینه خارجی و حساب الزامی را در دستورالعمل‌های Skill و اعلان‌های محیطی به‌روشنی مستند کنید (`requires.env` برای متغیرهای الزامی، یا `envVars` با `required: false` برای متغیرهای اختیاری).
