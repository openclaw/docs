# حالت‌های آیکون نوار منو

دامنه: برنامه macOS (`apps/macos`). رندر: `CritterIconRenderer.makeIcon(...)`. اتصال پویانمایی/حالت: `CritterStatusLabel` + `CritterStatusLabel+Behavior.swift`.

## حالت‌ها

| حالت                 | محرک                                   | ظاهر                                                                                              |
| --------------------- | ----------------------------------------- | --------------------------------------------------------------------------------------------------- |
| بی‌کار                  | پیش‌فرض                                   | پویانمایی عادی پلک‌زدن/جنبیدن؛ چشم‌های باز درخشش براق خود را حفظ می‌کنند                                        |
| متوقف                | `isPaused=true`                           | شاخک‌ها با چشم‌های باز آویزان می‌شوند («خارج از خدمت»)؛ بدون حرکت                                               |
| خوابیده              | Gateway قطع یا پیکربندی‌نشده         | شاخک‌ها آویزان و چشم‌ها به پلک‌های `⌣ ⌣` بسته می‌شوند؛ بدون حرکت                                            |
| جشن                  | پیام ارسال شد (`sendCelebrationTick`)      | چشم‌ها برای حدود 0.9s به‌شکل کمان‌های شاد `∩ ∩` چشمک می‌زنند و پا نیز لگدی می‌زند                                               |
| بیدارباش صوتی (گوش‌های بزرگ) | واژه بیدارباش شنیده شد                           | شاخک‌ها صاف و بلندتر (`earScale=1.9`) سیخ می‌شوند؛ پس از سکوت پایین می‌آیند                          |
| در حال کار               | `isWorking=true` یا یک `IconState` فعال | جنبش سریع‌تر پاها (`legWiggle` تا `1.0`) همراه با جابه‌جایی افقی کوچک؛ به جنبش حالت بی‌کار افزوده می‌شود |

وقتی یک نشست کار یا ابزار فعالی دارد، نشان فعالیت ابزار (یک دایره با نماد SF، مثلاً `chevron.left.slash.chevron.right` برای اجرا) می‌تواند روی همان آیکون موجود رندر شود. این نشان از `IconState`/`ActivityKind` می‌آید؛ برای مدل کامل حالت‌ها، [نوار منو](/fa/platforms/mac/menu-bar) را ببینید.

## گوش‌های بیدارباش صوتی

- محرک: `AppStateStore.shared.triggerVoiceEars(ttl: nil)`، فراخوانی‌شده از خط لوله ضبط بیدارباش صوتی (`VoiceWakeRuntime`) و ابزارهای اشکال‌زدایی/آزمایش بیدارباش صوتی (`VoiceWakeTester`، `VoiceWakeOverlayController`).
- توقف: `stopVoiceEars()`، هنگام نهایی‌شدن ضبط فراخوانی می‌شود.
- پنجره سکوت پیش از نهایی‌سازی: معمولاً `2.0s` و اگر فقط واژه محرک شنیده شده باشد و گفتار دیگری پس از آن نیاید، `5.0s` (`VoiceWakeRuntime.silenceWindow` / `triggerOnlySilenceWindow`).
- هنگام تقویت، زمان‌سنج‌های پلک‌زدن/جنبیدن/پا/گوش در حالت بی‌کار تعلیق می‌شوند (`earBoostActive` وظیفه پویانمایی را در `CritterStatusLabel+Behavior` کنترل می‌کند).

## شکل‌ها و اندازه‌ها

- بوم: تصویر الگوی 18x18pt که در یک حافظه پشتیبان بیت‌مپ 36x36px (2x) رندر می‌شود تا آیکون روی Retina واضح بماند.
- مقیاس گوش به‌طور پیش‌فرض `1.0` است؛ تقویت صوتی بدون تغییر قاب کلی آن را روی `earScale=1.9` تنظیم می‌کند.
- `antennaDroop` (0-1) شاخک‌ها را برای حالت‌های متوقف و خوابیده به پایین خم می‌کند.
- حرکت تند پاها از `legWiggle` تا `1.0` همراه با تکان افقی کوچکی انجام می‌شود.

## نکات رفتاری

- هیچ کلید خارجی CLI/واسطی برای گوش‌ها یا حالت کار وجود ندارد؛ هر دو به‌صورت داخلی با سیگنال‌های برنامه (`AppState.setWorking`، `AppState.triggerVoiceEars`) هدایت می‌شوند تا از بالا و پایین رفتن ناخواسته جلوگیری شود.
- هر TTL جدید را کوتاه نگه دارید (بسیار کمتر از 10s) تا اگر کاری متوقف ماند، آیکون سریع به حالت پایه بازگردد.
