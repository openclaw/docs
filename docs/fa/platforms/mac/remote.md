# OpenClaw راه‌دور (macOS ⇄ میزبان راه‌دور)

این جریان به برنامه macOS اجازه می‌دهد مانند یک کنترل راه‌دور کامل برای یک OpenClaw gateway که روی میزبان دیگری (دسکتاپ/سرور) اجرا می‌شود عمل کند. این همان قابلیت **راه‌دور از طریق SSH** (اجرای راه‌دور) برنامه است. همه قابلیت‌ها، از جمله بررسی‌های سلامت، ارسال Voice Wake و چت وب، از همان پیکربندی SSH راه‌دور در _تنظیمات → عمومی_ دوباره استفاده می‌کنند.

## حالت‌ها

- **محلی (این Mac)**: همه‌چیز روی لپ‌تاپ اجرا می‌شود. SSH در کار نیست.
- **راه‌دور از طریق SSH (پیش‌فرض)**: فرمان‌های OpenClaw روی میزبان راه‌دور اجرا می‌شوند. برنامه mac با `-o BatchMode` به‌همراه هویت/کلید انتخابی شما و یک پورت‌فوروارد محلی، اتصال SSH باز می‌کند.
- **راه‌دور مستقیم (ws/wss)**: بدون تونل SSH. برنامه mac مستقیماً به نشانی Gateway وصل می‌شود (برای مثال، از طریق Tailscale Serve یا یک پراکسی معکوس HTTPS عمومی).

## انتقال‌های راه‌دور

حالت راه‌دور از دو روش انتقال پشتیبانی می‌کند:

- **تونل SSH** (پیش‌فرض): از `ssh -N -L ...` برای فوروارد کردن پورت Gateway به localhost استفاده می‌کند. Gateway نشانی IP Node را `127.0.0.1` می‌بیند، چون تونل loopback است.
- **مستقیم (ws/wss)**: مستقیماً به نشانی Gateway وصل می‌شود. Gateway نشانی IP واقعی کلاینت را می‌بیند.

در حالت تونل SSH، نام‌های میزبان کشف‌شده در LAN/tailnet به‌صورت
`gateway.remote.sshTarget` ذخیره می‌شوند. برنامه `gateway.remote.url` را روی
نقطه پایانی تونل محلی نگه می‌دارد، برای مثال `ws://127.0.0.1:18789`، تا CLI، چت وب و
سرویس میزبان Node محلی همگی از همان انتقال امن loopback استفاده کنند.

خودکارسازی مرورگر در حالت راه‌دور متعلق به میزبان Node در CLI است، نه Node
برنامه بومی macOS. برنامه تا حد امکان سرویس میزبان Node نصب‌شده را شروع می‌کند؛
اگر از همان Mac به کنترل مرورگر نیاز دارید، آن را با
`openclaw node install ...` و `openclaw node start` نصب/شروع کنید (یا
`openclaw node run ...` را در پیش‌زمینه اجرا کنید)، سپس همان Node دارای قابلیت مرورگر
را هدف بگیرید.

## پیش‌نیازها روی میزبان راه‌دور

1. Node + pnpm را نصب کنید و OpenClaw CLI را بسازید/نصب کنید (`pnpm install && pnpm build && pnpm link --global`).
2. مطمئن شوید `openclaw` برای شل‌های غیرتعاملی روی PATH است (در صورت نیاز به `/usr/local/bin` یا `/opt/homebrew/bin` symlink کنید).
3. SSH را با احراز هویت کلیدی باز کنید. برای دسترسی پایدار خارج از LAN، نشانی‌های IP مربوط به **Tailscale** را پیشنهاد می‌کنیم.

## راه‌اندازی برنامه macOS

1. _تنظیمات → عمومی_ را باز کنید.
2. زیر **OpenClaw اجرا می‌شود**، **راه‌دور از طریق SSH** را انتخاب کنید و تنظیم کنید:
   - **انتقال**: **تونل SSH** یا **مستقیم (ws/wss)**.
   - **هدف SSH**: `user@host` (به‌صورت اختیاری `:port`).
     - اگر Gateway روی همان LAN است و Bonjour را تبلیغ می‌کند، آن را از فهرست کشف‌شده انتخاب کنید تا این فیلد به‌صورت خودکار پر شود.
   - **نشانی Gateway** (فقط مستقیم): `wss://gateway.example.ts.net` (یا `ws://...` برای محلی/LAN).
   - **فایل هویت** (پیشرفته): مسیر کلید شما.
   - **ریشه پروژه** (پیشرفته): مسیر checkout راه‌دور که برای فرمان‌ها استفاده می‌شود.
   - **مسیر CLI** (پیشرفته): مسیر اختیاری به یک نقطه ورود/باینری قابل اجرای `openclaw` (هنگام تبلیغ شدن به‌صورت خودکار پر می‌شود).
3. **آزمایش راه‌دور** را بزنید. موفقیت نشان می‌دهد که `openclaw status --json` راه‌دور درست اجرا می‌شود. شکست‌ها معمولاً به معنی مشکلات PATH/CLI هستند؛ کد خروج 127 یعنی CLI در راه‌دور پیدا نشده است.
4. بررسی‌های سلامت و چت وب اکنون به‌صورت خودکار از طریق این تونل SSH اجرا می‌شوند.

## چت وب

- **تونل SSH**: چت وب از طریق پورت کنترل WebSocket فورواردشده (پیش‌فرض 18789) به Gateway وصل می‌شود.
- **مستقیم (ws/wss)**: چت وب مستقیماً به نشانی Gateway پیکربندی‌شده وصل می‌شود.
- دیگر هیچ سرور HTTP جداگانه‌ای برای WebChat وجود ندارد.

## مجوزها

- میزبان راه‌دور به همان تأییدهای TCC حالت محلی نیاز دارد (Automation، Accessibility، Screen Recording، Microphone، Speech Recognition، Notifications). روی همان ماشین onboarding را اجرا کنید تا یک‌بار آن‌ها را اعطا کنید.
- Nodeها وضعیت مجوز خود را از طریق `node.list` / `node.describe` اعلام می‌کنند تا agentها بدانند چه چیزهایی در دسترس است.

## نکات امنیتی

- اتصال‌های loopback را روی میزبان راه‌دور ترجیح دهید و از طریق SSH یا Tailscale وصل شوید.
- تونل‌زنی SSH از بررسی سخت‌گیرانه کلید میزبان استفاده می‌کند؛ ابتدا به کلید میزبان اعتماد کنید تا در `~/.ssh/known_hosts` وجود داشته باشد.
- اگر Gateway را به یک رابط غیر loopback bind می‌کنید، احراز هویت معتبر Gateway را الزامی کنید: token، password، یا یک پراکسی معکوس آگاه از هویت با `gateway.auth.mode: "trusted-proxy"`.
- [امنیت](/fa/gateway/security) و [Tailscale](/fa/gateway/tailscale) را ببینید.

## جریان ورود WhatsApp (راه‌دور)

- `openclaw channels login --verbose` را **روی میزبان راه‌دور** اجرا کنید. QR را با WhatsApp روی تلفن خود اسکن کنید.
- اگر احراز هویت منقضی شد، ورود را دوباره روی همان میزبان اجرا کنید. بررسی سلامت مشکلات پیوند را نشان می‌دهد.

## عیب‌یابی

- **exit 127 / پیدا نشد**: `openclaw` برای شل‌های غیر login روی PATH نیست. آن را به `/etc/paths`، rc شل خود اضافه کنید، یا به `/usr/local/bin`/`/opt/homebrew/bin` symlink کنید.
- **کاوش سلامت شکست خورد**: دسترسی‌پذیری SSH، PATH و وارد بودن Baileys را بررسی کنید (`openclaw status --json`).
- **چت وب گیر کرده است**: تأیید کنید Gateway روی میزبان راه‌دور اجرا می‌شود و پورت فورواردشده با پورت WS مربوط به Gateway مطابقت دارد؛ UI به یک اتصال سالم WS نیاز دارد.
- **نشانی IP Node مقدار 127.0.0.1 را نشان می‌دهد**: با تونل SSH مورد انتظار است. اگر می‌خواهید Gateway نشانی IP واقعی کلاینت را ببیند، **انتقال** را به **مستقیم (ws/wss)** تغییر دهید.
- **Voice Wake**: عبارت‌های فعال‌سازی در حالت راه‌دور به‌صورت خودکار فوروارد می‌شوند؛ به فورواردکننده جداگانه نیازی نیست.

## صداهای اعلان

برای هر اعلان، صداها را از اسکریپت‌ها با `openclaw` و `node.invoke` انتخاب کنید، برای مثال:

```bash
openclaw nodes notify --node <id> --title "Ping" --body "Remote gateway ready" --sound Glass
```

دیگر در برنامه هیچ کلید سراسری «صدای پیش‌فرض» وجود ندارد؛ فراخوان‌ها برای هر درخواست یک صدا (یا هیچ صدا) انتخاب می‌کنند.
