برنامه macOS، **همراه نوار منوی** OpenClaw است: رابط کاربری بومی سینی سیستم، درخواست‌های مجوز macOS، اعلان‌ها، WebChat، ورودی صوتی، Canvas و
ابزارهای Node میزبانی‌شده روی Mac مانند `system.run`.

فقط به CLI و Gateway نیاز دارید؟ با [شروع به کار](/fa/start/getting-started) آغاز کنید.

## دانلود

نسخه‌های برنامه macOS را از [انتشارهای GitHub مربوط به OpenClaw](https://github.com/openclaw/openclaw/releases) دریافت کنید.
وقتی یک انتشار شامل دارایی‌های برنامه macOS است، به‌دنبال موارد زیر بگردید:

- `OpenClaw-<version>.dmg` (ترجیحی)
- `OpenClaw-<version>.zip`

برخی انتشارها فقط شامل CLI، شواهد یا دارایی‌های Windows هستند. اگر جدیدترین انتشار
دارایی برنامه macOS ندارد، از جدیدترین انتشاری که آن را دارد استفاده کنید یا با
[راه‌اندازی محیط توسعه macOS](/fa/platforms/mac/dev-setup) از منبع بسازید.

## نخستین اجرا

1. **OpenClaw.app** را نصب و اجرا کنید.
2. برای Gateway محلی، **این Mac** را انتخاب کنید یا به یک Gateway راه‌دور متصل شوید.
3. صبر کنید تا برنامه زمان‌اجرای CLI منطبق را نصب کند. در حالت محلی، همچنین
   Gateway را نصب و راه‌اندازی می‌کند.
4. با بررسی زنده مدل، استنتاج را برقرار کنید. پس از موفقیت آن، OpenClaw
   ادامه راه‌اندازی را انجام می‌دهد.
5. فهرست بررسی مجوزهای macOS را تکمیل و پیام آزمایشی آغاز به کار را ارسال کنید.

اگر برنامه به Gateway موجودی برسد که عامل پیش‌فرض آن مدل پیکربندی‌شده‌ای دارد،
آن Gateway را از قبل راه‌اندازی‌شده در نظر می‌گیرد، فرایند آغاز به کار ارائه‌دهنده و
OpenClaw را نادیده می‌گیرد و داشبورد را باز می‌کند. اگر اتصال به Gateway ممکن نباشد یا
عامل پیش‌فرض آن مدلی نداشته باشد، آغاز به کار استنتاج برای
بازیابی همچنان در دسترس می‌ماند.

برای مسیر راه‌اندازی CLI/Gateway، از [شروع به کار](/fa/start/getting-started) استفاده کنید.
برای بازیابی مجوزها، به [مجوزهای macOS](/fa/platforms/mac/permissions) مراجعه کنید.

## به‌روزرسانی‌ها

کارت به‌روزرسانی داشبورد مشخص می‌کند برنامه چه چیزی را به‌روزرسانی خواهد کرد:

- **به‌روزرسانی برنامه Mac و Gateway** یعنی برنامه امضاشده مالک Gateway محلی
  مبتنی بر launchd است. Sparkle ابتدا برنامه را به‌روزرسانی می‌کند؛ پس از اجرای مجدد، برنامه به‌طور خودکار
  Gateway خود را به نسخه منطبق به‌روزرسانی و بازراه‌اندازی می‌کند، سپس
  اتصال را تأیید می‌کند.
- **به‌روزرسانی Gateway** یعنی برنامه به یک Gateway راه‌دور، یک Gateway محلی با
  مدیریت دستی یا نصب دیگری که برنامه مالک آن نیست متصل است. این دکمه
  به‌جای تغییر برنامه Mac، فرایند عادی به‌روزرسانی آن Gateway را اجرا می‌کند.

به‌روزرسانی هماهنگ ناموفق، در پنجره‌ای به سبک راه‌اندازی باقی می‌ماند و گزینه‌های تلاش مجدد،
[راهنمای به‌روزرسانی](/fa/install/updating) و Discord را ارائه می‌دهد. تعمیر خودکار هرگز
یک Gateway جدیدتر را تنزل نسخه نمی‌دهد یا پین کانال `extended-stable` را لغو نمی‌کند.

پس از به‌روزرسانی موفق، برنامه آخرین نشست مستقیم سطح‌بالا را که اخیراً توسط انسان استفاده شده است
پیدا می‌کند و یک رویداد یک‌باره به‌روزرسانی به آن عامل می‌دهد. فعالیت Heartbeat
و Cron بر این انتخاب تأثیری ندارد. سپس عامل می‌تواند از مکالمه‌ای که به‌احتمال زیاد
در حال استفاده از آن بودید، بازگشت شما را خوشامد بگوید. در حالت راه‌دور، برنامه
فقط زمان‌اجرای Node محلی Mac را به‌روزرسانی می‌کند و وقتی
Gateway راه‌دور قدیمی‌تر از برنامه باشد، اعلان را نادیده می‌گیرد.

Sparkle از تنظیم `update.channel` مربوط به Gateway پیروی می‌کند. `beta` و `dev` استفاده از
نسخه‌های بتای برنامه را فعال می‌کنند؛ `stable`، `extended-stable` و مقادیر ناموجود یا ناشناخته
روی نسخه‌های پایدار برنامه باقی می‌مانند.

## باز کردن پیوندهای داشبورد

در داشبورد تعبیه‌شده برنامه macOS، کلیک روی یک پیوند وب خارجی، آن را در نوار کناری مرورگر با قابلیت تغییر اندازه و با نصف عرض پنجره باز می‌کند، درحالی‌که پیمایش داشبورد قابل‌مشاهده باقی می‌ماند. برای انتخاب عرضی دیگر، جداکننده را بکشید؛ برنامه آن را به خاطر می‌سپارد. هر پیوند در زبانه‌ای جداگانه باز می‌شود، وقتی چند صفحه باز باشند نوار زبانه ظاهر می‌شود و کلیک دوباره روی همان پیوند از زبانه موجود آن استفاده می‌کند. برای تغییر ترتیب زبانه‌ها آن‌ها را بکشید، با دکمه بستن زبانه یا کلیک میانی ببندید و برای گزینه‌های **باز کردن در مرورگر پیش‌فرض**، **کپی پیوند**، **بارگذاری مجدد**، **بستن زبانه** و **بستن سایر زبانه‌ها** روی زبانه کلیک راست کنید. کنترل‌های عقب/جلو در نوار عنوان پنجره و کشیدن انگشت روی ترک‌پد، در تاریخچه داشبورد پیمایش می‌کنند؛ کنترل‌های عقب/جلوی خود نوار کناری در تاریخچه زبانه فعال پیمایش می‌کنند. نوار کناری همچنین کنترل‌های بارگذاری مجدد، باز کردن در مرورگر پیش‌فرض و بستن را دارد.

کنترل‌های نوار عنوان از نوار کناری برنامه پیروی می‌کنند: وقتی باز است، دکمه‌های عقب/جلو در لبه راست آن و کنار کلید تغییر وضعیت نوار کناری قرار می‌گیرند؛ وقتی جمع شده است، جای خود را به دکمه جست‌وجو (برای باز کردن پالت فرمان) و دکمه نشست جدید می‌دهند.

برای انتخاب **باز کردن در نوار کناری**، **باز کردن در مرورگر پیش‌فرض** یا **کپی پیوند** روی یک پیوند خارجی کلیک راست کنید. کلیک‌های همراه با کلید اصلاح‌گر و پیوندهای پنجره جدیدی که کاربر در داشبورد فعال می‌کند، همچنان در مرورگر پیش‌فرض باز می‌شوند؛ پیوندهای پنجره جدید داخل نوار کناری به‌صورت زبانه‌های جدید نوار کناری باز می‌شوند. صفحات عادی رابط کنترل که در مرورگر میزبانی می‌شوند، رفتار معمول پیوند و منوی زمینه مرورگر را حفظ می‌کنند.

## وارد کردن ورودهای مرورگر

نخستین بار که نوار کناری مرورگر درحالی باز می‌شود که برنامه با یک Gateway محلی اجرا شده است، اگر نمایه‌ای از خانواده Chrome دارای کوکی روی Mac وجود داشته باشد، داشبورد یک بنر قابل‌بستن نمایش می‌دهد. بنر امکان کپی‌کردن آن کوکی‌ها را در یک نمایه مدیریت‌شده و ایزوله فراهم می‌کند که عامل‌ها برای مرور استفاده می‌کنند. از کنترل **وارد کردن** آن یک نمایه انتخاب کنید (ممکن است Touch ID لازم باشد)؛ پیشرفت و تعداد کوکی‌های واردشده در همان محل ظاهر می‌شوند و فقط کوکی‌ها کپی می‌شوند — گذرواژه‌ها هرگز مرورگر مبدأ را ترک نمی‌کنند. بستن بنر این انتخاب را ثبت می‌کند؛ **تنظیمات ← عمومی ← ورود مرورگر ← وارد کردن…** در هر زمان دوباره آن را ارائه می‌دهد. برای فرایند زیربنایی وارد کردن و محدودیت `browser.allowSystemProfileImport` به [مرورگر](/fa/cli/browser) مراجعه کنید.

## انتخاب حالت Gateway

| حالت   | زمانی استفاده کنید که                                                       | صفحه جزئیات                                        |
| ------ | --------------------------------------------------------------------------- | -------------------------------------------------- |
| محلی   | این Mac باید Gateway را اجرا کند و با launchd فعال نگه دارد.                | [Gateway در macOS](/fa/platforms/mac/bundled-gateway) |
| راه‌دور | میزبان دیگری Gateway را اجرا می‌کند؛ این Mac آن را از طریق SSH، LAN یا Tailnet کنترل می‌کند. | [کنترل راه‌دور](/fa/platforms/mac/remote)            |

هر دو حالت به CLI نصب‌شده `openclaw` نیاز دارند، زیرا برنامه از زمان‌اجرای میزبان Node آن
دوباره استفاده می‌کند. در یک Mac تازه، برنامه CLI منطبق را به‌طور خودکار نصب می‌کند؛ سپس حالت
محلی راهنمای Gateway را آغاز می‌کند، درحالی‌که حالت راه‌دور بدون راه‌اندازی
Gateway محلی دوم به Gateway انتخاب‌شده متصل می‌شود.
برای بازیابی دستی، به [Gateway در macOS](/fa/platforms/mac/bundled-gateway) مراجعه کنید.

## موارد تحت مالکیت برنامه

- وضعیت نوار منو، اعلان‌ها، سلامت و WebChat.
- درخواست‌های مجوز macOS برای صفحه‌نمایش، میکروفون، گفتار، خودکارسازی و دسترس‌پذیری.
- یک Node مربوط به Mac که Canvas بومی، ثبت دوربین/صفحه‌نمایش، اعلان‌ها،
  موقعیت مکانی و کنترل رایانه را با فرمان‌های سیستم، مرورگر،
  Plugin، مهارت و MCP میزبان Node مربوط به CLI ترکیب می‌کند.
- درخواست‌های تأیید اجرا برای فرمان‌های میزبانی‌شده روی Mac.
- اجرای فرمان‌های تأییدشده پوسته در زمینه برنامه، با حفظ انتساب مجوزهای macOS
  به برنامه، درحالی‌که زمان‌اجرای CLI مالک خط‌مشی مشترک Node است.
- تونل‌های SSH حالت راه‌دور یا اتصال‌های مستقیم Gateway.

برنامه جایگزین مستندات Gateway یا مستندات عمومی CLI **نمی‌شود**. پیکربندی
Gateway، ارائه‌دهندگان، Pluginها، کانال‌ها، ابزارها و امنیت، مستندات
مخصوص خود را دارند.
