# فهرست Plugin

این صفحه از `extensions/*/package.json`، `openclaw.plugin.json`،
و استثناهای `files` در بسته npm ریشه تولید می‌شود. آن را با دستور زیر بازتولید کنید:

```bash
pnpm plugins:inventory:gen
```

## تعاریف

- **بسته npm هسته:** در بسته npm `openclaw` ساخته شده و بدون نصب جداگانه Plugin در دسترس است.
- **بسته خارجی رسمی:** Plugin نگهداری‌شده توسط OpenClaw که از بسته npm هسته حذف شده، در این فهرست رسمی نگه داشته می‌شود، و در صورت نیاز از طریق ClawHub و/یا npm نصب می‌شود.
- **فقط checkout منبع:** Plugin محلیِ repo که از آرتیفکت‌های npm منتشرشده حذف شده و به‌عنوان بسته قابل نصب تبلیغ نمی‌شود.

checkoutهای منبع با نصب‌های npm متفاوت‌اند: پس از `pnpm install`، Pluginهای همراه
از `extensions/<id>` بارگذاری می‌شوند تا ویرایش‌های محلی و وابستگی‌های workspace محلیِ بسته
در دسترس باشند.

## نصب یک Plugin

از ستون **توزیع** استفاده کنید تا تصمیم بگیرید آیا نصب لازم است یا نه. Pluginهایی که
می‌گویند `included in OpenClaw` از قبل در بسته هسته وجود دارند. بسته‌های خارجی
رسمی به یک نصب نیاز دارند و سپس به restart کردن Gateway.

برای مثال، Discord یک بسته خارجی رسمی است:

```bash
openclaw plugins install @openclaw/discord
openclaw gateway restart
openclaw plugins inspect discord --runtime --json
```

مشخصات بسته بدون پیشوند ابتدا ClawHub را امتحان می‌کنند، سپس به npm fallback می‌کنند. برای اجبار به استفاده از یک منبع، از
`clawhub:@openclaw/discord` یا `npm:@openclaw/discord` استفاده کنید. پس از نصب، راهنمای راه‌اندازی
Plugin، مانند [Discord](/fa/channels/discord)، را دنبال کنید تا اعتبارنامه‌ها
و پیکربندی کانال را اضافه کنید. برای فرمان‌های به‌روزرسانی،
حذف نصب، و انتشار، [مدیریت Pluginها](/fa/plugins/manage-plugins) را ببینید.
