# موجودی Plugin

این صفحه از `extensions/*/package.json`، `openclaw.plugin.json`،
و موارد حذف‌شده از `files` در بسته ریشه npm تولید می‌شود. آن را با دستور زیر بازتولید کنید:

```bash
pnpm plugins:inventory:gen
```

## تعریف‌ها

- **بسته اصلی npm:** در بسته npm مربوط به `openclaw` ساخته شده و بدون نصب جداگانه Plugin در دسترس است.
- **بسته رسمی خارجی:** Plugin نگهداری‌شده توسط OpenClaw که از بسته اصلی npm حذف شده، در این موجودی رسمی نگه داشته می‌شود، و بر اساس نیاز از طریق ClawHub و/یا npm نصب می‌شود.
- **فقط checkout سورس:** Plugin محلی مخزن که از آرتیفکت‌های منتشرشده npm حذف شده و به‌عنوان بسته قابل نصب معرفی نمی‌شود.

checkoutهای سورس با نصب‌های npm تفاوت دارند: پس از `pnpm install`، Pluginهای همراه
از `extensions/<id>` بارگذاری می‌شوند تا ویرایش‌های محلی و وابستگی‌های workspace محلی بسته
در دسترس باشند.

## نصب یک Plugin

از ستون **توزیع** استفاده کنید تا مشخص شود آیا نصب لازم است یا نه. Pluginهایی که
می‌گویند `included in OpenClaw` از قبل در بسته اصلی وجود دارند. بسته‌های رسمی
خارجی به یک نصب نیاز دارند و سپس باید Gateway راه‌اندازی مجدد شود.

برای مثال، Discord یک بسته رسمی خارجی است:

```bash
openclaw plugins install @openclaw/discord
openclaw gateway restart
openclaw plugins inspect discord --runtime --json
```

مشخصات بسته بدون پیشوند ابتدا ClawHub را امتحان می‌کند، سپس به npm fallback می‌کند. برای اجبار به استفاده از یک منبع، از
`clawhub:@openclaw/discord` یا `npm:@openclaw/discord` استفاده کنید. پس از نصب، سند راه‌اندازی
Plugin، مانند [Discord](/fa/channels/discord)، را دنبال کنید تا اطلاعات احراز هویت
و پیکربندی کانال را اضافه کنید. برای دستورهای به‌روزرسانی،
حذف نصب، و انتشار، [مدیریت Pluginها](/fa/plugins/manage-plugins) را ببینید.
