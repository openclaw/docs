OpenClaw می‌تواند بسته‌ها را از سه زیست‌بوم خارجی نصب کند: **Codex**، **Claude**،
و **Cursor**. به این‌ها **بسته‌ها** گفته می‌شود — بسته‌های محتوا و فراداده که
OpenClaw آن‌ها را به قابلیت‌های بومی مانند Skills، هوک‌ها و ابزارهای MCP نگاشت می‌کند.

<Info>
  بسته‌ها همان Pluginهای بومی OpenClaw **نیستند**. Pluginهای بومی به‌صورت
  درون‌فرایندی اجرا می‌شوند و می‌توانند هر قابلیتی را ثبت کنند. بسته‌ها بسته‌های
  محتوا با نگاشت گزینشی قابلیت و مرز اعتماد محدودتر هستند.
</Info>

## چرا بسته‌ها وجود دارند

بسیاری از Pluginهای مفید در قالب Codex، Claude یا Cursor منتشر می‌شوند. به‌جای
اینکه از نویسندگان خواسته شود آن‌ها را به‌صورت Pluginهای بومی OpenClaw بازنویسی کنند،
OpenClaw این قالب‌ها را تشخیص می‌دهد و محتوای پشتیبانی‌شده‌ی آن‌ها را به مجموعه
قابلیت‌های بومی نگاشت می‌کند. یعنی می‌توانید یک بسته فرمان Claude یا یک بسته Skill
برای Codex را نصب کنید و بلافاصله از آن استفاده کنید.

## نصب یک بسته

<Steps>
  <Step title="نصب از یک دایرکتوری، آرشیو یا marketplace">
    ```bash
    # Local directory
    openclaw plugins install ./my-bundle

    # Archive
    openclaw plugins install ./my-bundle.tgz

    # Claude marketplace
    openclaw plugins marketplace list <marketplace-name>
    openclaw plugins install <plugin-name>@<marketplace-name>
    ```

  </Step>

  <Step title="تأیید تشخیص">
    ```bash
    openclaw plugins list
    openclaw plugins inspect <id>
    ```

    بسته‌ها به‌صورت `Format: bundle` با زیرنوع `codex`، `claude` یا `cursor` نمایش داده می‌شوند.

  </Step>

  <Step title="راه‌اندازی دوباره و استفاده">
    ```bash
    openclaw gateway restart
    ```

    قابلیت‌های نگاشت‌شده (Skills، هوک‌ها، ابزارهای MCP، پیش‌فرض‌های LSP) در نشست بعدی در دسترس هستند.

  </Step>
</Steps>

## OpenClaw چه چیزهایی را از بسته‌ها نگاشت می‌کند

امروز همه قابلیت‌های بسته در OpenClaw اجرا نمی‌شوند. در اینجا آمده است چه چیزهایی کار می‌کنند و چه چیزهایی
تشخیص داده می‌شوند اما هنوز وصل نشده‌اند.

### اکنون پشتیبانی می‌شود

| قابلیت       | نحوه نگاشت                                                                                 | قابل اعمال به     |
| ------------- | ------------------------------------------------------------------------------------------- | -------------- |
| محتوای Skill | ریشه‌های Skill بسته مانند Skills معمول OpenClaw بارگذاری می‌شوند                                           | همه قالب‌ها    |
| فرمان‌ها      | `commands/` و `.cursor/commands/` به‌عنوان ریشه‌های Skill در نظر گرفته می‌شوند                                  | Claude، Cursor |
| بسته‌های هوک    | چیدمان‌های سبک OpenClaw با `HOOK.md` + `handler.ts`                                             | Codex          |
| ابزارهای MCP     | پیکربندی MCP بسته در تنظیمات Pi تعبیه‌شده ادغام می‌شود؛ سرورهای stdio و HTTP پشتیبانی‌شده بارگذاری می‌شوند | همه قالب‌ها    |
| سرورهای LSP   | فایل `.lsp.json` در Claude و `lspServers` اعلام‌شده در مانیفست در پیش‌فرض‌های LSP مربوط به Pi تعبیه‌شده ادغام می‌شوند  | Claude         |
| تنظیمات      | فایل `settings.json` در Claude به‌عنوان پیش‌فرض‌های Pi تعبیه‌شده وارد می‌شود                                     | Claude         |

#### محتوای Skill

- ریشه‌های Skill بسته مانند ریشه‌های Skill معمول OpenClaw بارگذاری می‌شوند
- ریشه‌های `commands` در Claude به‌عنوان ریشه‌های Skill اضافی در نظر گرفته می‌شوند
- ریشه‌های `.cursor/commands` در Cursor به‌عنوان ریشه‌های Skill اضافی در نظر گرفته می‌شوند

یعنی فایل‌های فرمان Markdown در Claude از مسیر بارگذار Skill معمول OpenClaw کار می‌کنند.
Markdown فرمان Cursor از همان مسیر کار می‌کند.

#### بسته‌های هوک

- ریشه‌های هوک بسته **فقط** زمانی کار می‌کنند که از چیدمان معمول بسته هوک OpenClaw
  استفاده کنند. امروز این عمدتاً حالت سازگار با Codex است:
  - `HOOK.md`
  - `handler.ts` یا `handler.js`

#### MCP برای Pi

- بسته‌های فعال می‌توانند پیکربندی سرور MCP اضافه کنند
- OpenClaw پیکربندی MCP بسته را به‌صورت `mcpServers` در تنظیمات مؤثر Pi تعبیه‌شده ادغام می‌کند
- OpenClaw ابزارهای MCP پشتیبانی‌شده‌ی بسته را در طول نوبت‌های عامل Pi تعبیه‌شده، با
  راه‌اندازی سرورهای stdio یا اتصال به سرورهای HTTP، در معرض استفاده قرار می‌دهد
- پروفایل‌های ابزار `coding` و `messaging` به‌طور پیش‌فرض ابزارهای MCP بسته را شامل می‌شوند؛ برای انصراف در سطح یک عامل یا Gateway از `tools.deny: ["bundle-mcp"]` استفاده کنید
- تنظیمات Pi محلی پروژه همچنان پس از پیش‌فرض‌های بسته اعمال می‌شوند، بنابراین تنظیمات
  فضای کاری می‌توانند در صورت نیاز ورودی‌های MCP بسته را بازنویسی کنند
- کاتالوگ‌های ابزار MCP بسته پیش از ثبت به‌صورت قطعی مرتب می‌شوند، بنابراین
  تغییرات ترتیب `listTools()` در بالادست، بلوک‌های ابزار کش پرامپت را بی‌ثبات نمی‌کند

##### انتقال‌ها

سرورهای MCP می‌توانند از انتقال stdio یا HTTP استفاده کنند:

**Stdio** یک فرایند فرزند را راه‌اندازی می‌کند:

```json
{
  "mcp": {
    "servers": {
      "my-server": {
        "command": "node",
        "args": ["server.js"],
        "env": { "PORT": "3000" }
      }
    }
  }
}
```

**HTTP** به‌طور پیش‌فرض از طریق `sse` به یک سرور MCP در حال اجرا متصل می‌شود، یا در صورت درخواست از `streamable-http` استفاده می‌کند:

```json
{
  "mcp": {
    "servers": {
      "my-server": {
        "url": "http://localhost:3100/mcp",
        "transport": "streamable-http",
        "headers": {
          "Authorization": "Bearer ${MY_SECRET_TOKEN}"
        },
        "connectionTimeoutMs": 30000
      }
    }
  }
}
```

- `transport` می‌تواند روی `"streamable-http"` یا `"sse"` تنظیم شود؛ اگر حذف شود، OpenClaw از `sse` استفاده می‌کند
- `type: "http"` یک شکل پایین‌دستی بومی CLI است؛ در پیکربندی OpenClaw از `transport: "streamable-http"` استفاده کنید. `openclaw mcp set` و `openclaw doctor --fix` نام مستعار رایج را نرمال‌سازی می‌کنند.
- فقط طرح‌های URL با `http:` و `https:` مجاز هستند
- مقادیر `headers` از درون‌یابی `${ENV_VAR}` پشتیبانی می‌کنند
- ورودی سروری که هم `command` و هم `url` داشته باشد رد می‌شود
- اعتبارنامه‌های URL (userinfo و پارامترهای query) از توضیحات ابزار
  و لاگ‌ها حذف می‌شوند
- `connectionTimeoutMs` زمان پایان اتصال پیش‌فرض ۳۰ ثانیه‌ای را برای
  هر دو انتقال stdio و HTTP بازنویسی می‌کند

##### نام‌گذاری ابزار

OpenClaw ابزارهای MCP بسته را با نام‌های امن برای ارائه‌دهنده در قالب
`serverName__toolName` ثبت می‌کند. برای مثال، سروری با کلید `"vigil-harbor"` که یک
ابزار `memory_search` ارائه می‌دهد، به‌صورت `vigil-harbor__memory_search` ثبت می‌شود.

- نویسه‌های خارج از `A-Za-z0-9_-` با `-` جایگزین می‌شوند
- پیشوندهای سرور حداکثر ۳۰ نویسه هستند
- نام‌های کامل ابزار حداکثر ۶۴ نویسه هستند
- نام‌های خالی سرور به `mcp` برمی‌گردند
- نام‌های پاک‌سازی‌شده‌ی متداخل با پسوندهای عددی از هم متمایز می‌شوند
- ترتیب نهایی ابزارهای در معرض استفاده بر اساس نام امن قطعی است تا نوبت‌های تکراری Pi
  از نظر کش پایدار بمانند
- فیلتر پروفایل با همه ابزارهای یک سرور MCP بسته به‌عنوان متعلق به Plugin توسط
  `bundle-mcp` رفتار می‌کند، بنابراین allowlistها و deny listهای پروفایل می‌توانند
  نام ابزارهای در معرض استفاده‌ی جداگانه یا کلید Plugin یعنی `bundle-mcp` را شامل شوند

#### تنظیمات Pi تعبیه‌شده

- وقتی بسته فعال باشد، فایل `settings.json` در Claude به‌عنوان تنظیمات پیش‌فرض Pi تعبیه‌شده وارد می‌شود
- OpenClaw کلیدهای بازنویسی shell را پیش از اعمال پاک‌سازی می‌کند

کلیدهای پاک‌سازی‌شده:

- `shellPath`
- `shellCommandPrefix`

#### LSP تعبیه‌شده‌ی Pi

- بسته‌های فعال Claude می‌توانند پیکربندی سرور LSP اضافه کنند
- OpenClaw فایل `.lsp.json` به‌همراه هر مسیر `lspServers` اعلام‌شده در مانیفست را بارگذاری می‌کند
- پیکربندی LSP بسته در پیش‌فرض‌های مؤثر LSP مربوط به Pi تعبیه‌شده ادغام می‌شود
- امروز فقط سرورهای LSP پشتیبانی‌شده با پشتوانه stdio قابل اجرا هستند؛ انتقال‌های
  پشتیبانی‌نشده همچنان در `openclaw plugins inspect <id>` نمایش داده می‌شوند

### تشخیص داده‌شده اما اجرا نمی‌شود

این موارد شناسایی می‌شوند و در عیب‌یابی نمایش داده می‌شوند، اما OpenClaw آن‌ها را اجرا نمی‌کند:

- `agents`، خودکارسازی `hooks.json` و `outputStyles` در Claude
- `.cursor/agents`، `.cursor/hooks.json` و `.cursor/rules` در Cursor
- فراداده درون‌خطی/برنامه در Codex فراتر از گزارش قابلیت

## تقدم تشخیص

OpenClaw ابتدا قالب Plugin بومی را بررسی می‌کند:

1. `openclaw.plugin.json` یا `package.json` معتبر با `openclaw.extensions` — به‌عنوان **Plugin بومی** در نظر گرفته می‌شود
2. نشانگرهای بسته (`.codex-plugin/`، `.claude-plugin/` یا چیدمان پیش‌فرض Claude/Cursor) — به‌عنوان **بسته** در نظر گرفته می‌شود

اگر یک دایرکتوری هر دو را داشته باشد، OpenClaw از مسیر بومی استفاده می‌کند. این کار از
نصب ناقص بسته‌های دوقالبی به‌عنوان بسته جلوگیری می‌کند.

## وابستگی‌های زمان اجرا و پاک‌سازی

- وابستگی‌های زمان اجرای Plugin بسته‌بندی‌شده داخل بسته OpenClaw در مسیر
  `dist/*` ارائه می‌شوند. OpenClaw هنگام راه‌اندازی برای Pluginهای بسته‌بندی‌شده
  `npm install` را اجرا **نمی‌کند**؛ خط انتشار مسئول ارائه یک محموله کامل
  وابستگی‌های بسته‌بندی‌شده است (قاعده تأیید پس از انتشار را در
  [انتشار](/fa/reference/RELEASING) ببینید).

## امنیت

بسته‌ها نسبت به Pluginهای بومی مرز اعتماد محدودتری دارند:

- OpenClaw ماژول‌های دلخواه زمان اجرای بسته را به‌صورت درون‌فرایندی بارگذاری **نمی‌کند**
- مسیرهای Skills و بسته هوک باید داخل ریشه Plugin باقی بمانند (با بررسی مرز)
- فایل‌های تنظیمات با همان بررسی‌های مرزی خوانده می‌شوند
- سرورهای MCP پشتیبانی‌شده‌ی stdio ممکن است به‌عنوان زیرفرایند راه‌اندازی شوند

این باعث می‌شود بسته‌ها به‌طور پیش‌فرض امن‌تر باشند، اما همچنان باید بسته‌های شخص ثالث
را برای قابلیت‌هایی که در معرض استفاده قرار می‌دهند، محتوای مورد اعتماد در نظر بگیرید.
