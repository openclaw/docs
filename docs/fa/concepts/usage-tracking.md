## چیست

- مصرف/سهمیهٔ ارائه‌دهنده را مستقیماً از نقاط پایانی مصرف آن‌ها می‌گیرد.
- بدون هزینه‌های تخمینی؛ فقط پنجره‌هایی که ارائه‌دهنده گزارش می‌کند.
- خروجی وضعیت خوانا برای انسان به `X% left` نرمال‌سازی می‌شود، حتی وقتی یک
  API بالادستی سهمیهٔ مصرف‌شده، سهمیهٔ باقی‌مانده، یا فقط شمارش‌های خام را گزارش کند.
- `/status` و `session_status` در سطح نشست می‌توانند وقتی نمای فوری نشست زنده کم‌جزئیات است، به آخرین
  ورودی مصرف در رونوشت بازگردند. این
  مسیر جایگزین شمارنده‌های مفقود توکن/کش را پر می‌کند، می‌تواند برچسب مدل زمان اجرای فعال را بازیابی کند،
  و وقتی فرادادهٔ نشست مفقود یا کوچک‌تر است، کل بزرگ‌ترِ متمرکز بر پرامپت را ترجیح می‌دهد. مقدارهای زندهٔ غیرصفر موجود همچنان اولویت دارند.

## کجا نمایش داده می‌شود

- `/status` در چت‌ها: کارت وضعیت پربار از نظر ایموجی با توکن‌های نشست + هزینهٔ تخمینی (فقط کلید API). مصرف ارائه‌دهنده برای **ارائه‌دهندهٔ مدل فعلی** در صورت وجود، به‌صورت پنجرهٔ نرمال‌شدهٔ `X% left` نمایش داده می‌شود.
- `/usage off|tokens|full` در چت‌ها: پابرگ مصرف برای هر پاسخ (OAuth فقط توکن‌ها را نشان می‌دهد).
- `/usage cost` در چت‌ها: خلاصهٔ هزینهٔ محلی تجمیع‌شده از گزارش‌های نشست OpenClaw.
- CLI: `openclaw status --usage` یک تفکیک کامل برای هر ارائه‌دهنده چاپ می‌کند.
- CLI: `openclaw channels list` همان نمای فوری مصرف را کنار پیکربندی ارائه‌دهنده چاپ می‌کند (برای رد کردن از `--no-usage` استفاده کنید).
- نوار منوی macOS: بخش «مصرف» زیر Context (فقط در صورت موجود بودن).

## ارائه‌دهنده‌ها + اعتبارنامه‌ها

- **Anthropic (Claude)**: توکن‌های OAuth در پروفایل‌های احراز هویت.
- **GitHub Copilot**: توکن‌های OAuth در پروفایل‌های احراز هویت.
- **Gemini CLI**: توکن‌های OAuth در پروفایل‌های احراز هویت.
  - مصرف JSON به `stats` بازمی‌گردد؛ `stats.cached` به
    `cacheRead` نرمال‌سازی می‌شود.
- **OpenAI Codex**: توکن‌های OAuth در پروفایل‌های احراز هویت (در صورت وجود از accountId استفاده می‌شود).
- **MiniMax**: کلید API یا پروفایل احراز هویت OAuth برای MiniMax. OpenClaw
  `minimax`، `minimax-cn` و `minimax-portal` را سطح سهمیهٔ یکسان MiniMax در نظر می‌گیرد،
  وقتی OAuth ذخیره‌شدهٔ MiniMax وجود داشته باشد آن را ترجیح می‌دهد، و در غیر این صورت به
  `MINIMAX_CODE_PLAN_KEY`، `MINIMAX_CODING_API_KEY` یا `MINIMAX_API_KEY` بازمی‌گردد.
  فیلدهای خام `usage_percent` / `usagePercent` در MiniMax به‌معنای سهمیهٔ **باقی‌مانده**
  هستند، بنابراین OpenClaw پیش از نمایش آن‌ها را معکوس می‌کند؛ فیلدهای مبتنی بر شمارش، در صورت
  وجود، اولویت دارند.
  - برچسب‌های پنجرهٔ طرح کدنویسی، در صورت وجود، از فیلدهای ساعت/دقیقهٔ ارائه‌دهنده می‌آیند و سپس
    به بازهٔ `start_time` / `end_time` بازمی‌گردند.
  - اگر نقطهٔ پایانی طرح کدنویسی `model_remains` را برگرداند، OpenClaw ورودی
    مدل چت را ترجیح می‌دهد، وقتی فیلدهای صریح
    `window_hours` / `window_minutes` وجود ندارند برچسب پنجره را از برچسب‌های زمانی استخراج می‌کند، و نام مدل را
    در برچسب طرح می‌آورد.
- **Xiaomi MiMo**: کلید API از طریق env/config/auth store (`XIAOMI_API_KEY`).
- **z.ai**: کلید API از طریق env/config/auth store.

وقتی هیچ احراز هویت قابل استفاده‌ای برای مصرف ارائه‌دهنده قابل حل نباشد، مصرف پنهان می‌شود. ارائه‌دهنده‌ها
می‌توانند منطق احراز هویت مصرف مخصوص Plugin ارائه دهند؛ در غیر این صورت OpenClaw به
اعتبارنامه‌های OAuth/کلید API منطبق از پروفایل‌های احراز هویت، متغیرهای محیطی،
یا پیکربندی بازمی‌گردد.
