لاین تضمین کیفیت Matrix، Plugin همراه `@openclaw/matrix` را در برابر یک سرور خانگی یک‌بارمصرف Tuwunel در Docker اجرا می‌کند؛ همراه با حساب‌های موقت راه‌انداز، سامانه تحت آزمون و ناظر، به‌علاوه اتاق‌های ازپیش‌مقداردهی‌شده. این لاین، پوشش زنده و مبتنی بر انتقال واقعی برای Matrix است.

ابزار ویژه نگه‌دارندگان. نسخه‌های بسته‌بندی‌شده OpenClaw شامل `qa-lab` نیستند؛ بنابراین `openclaw qa` فقط از یک وارسی کد منبع اجرا می‌شود که اجراکننده همراه را مستقیماً و بدون مرحله نصب Plugin بارگذاری می‌کند.

برای آشنایی با چارچوب گسترده‌تر تضمین کیفیت، [نمای کلی تضمین کیفیت](/fa/concepts/qa-e2e-automation) را ببینید.

## شروع سریع

```bash
pnpm openclaw qa matrix --profile fast --fail-fast
```

اجرای ساده `pnpm openclaw qa matrix` از `--profile all` استفاده می‌کند و با نخستین شکست متوقف نمی‌شود. با استفاده از `--profile transport|media|e2ee-smoke|e2ee-deep|e2ee-cli` کل فهرست را میان کارهای موازی تقسیم کنید.

## کارکرد لاین

1. یک سرور خانگی یک‌بارمصرف Tuwunel را در Docker (تصویر پیش‌فرض `ghcr.io/matrix-construct/tuwunel:v1.5.1`، نام سرور `matrix-qa.test`، درگاه `28008`) پشت یک ضبط‌کننده محدود و اطلاعات‌زدای درخواست/پاسخ فراهم می‌کند.
2. سه کاربر موقت ثبت می‌کند: `driver` (ترافیک ورودی را ارسال می‌کند)، `sut` (حساب Matrix متعلق به OpenClaw که تحت آزمون است)، `observer` (ترافیک شخص ثالث را ثبت می‌کند).
3. اتاق‌های موردنیاز سناریوهای انتخاب‌شده را ازپیش‌مقداردهی می‌کند (اصلی، رشته گفتگو، رسانه، راه‌اندازی مجدد، ثانویه، فهرست مجاز، E2EE، پیام مستقیم تأیید و غیره).
4. کاوشگر پروتکل مستقل از زیرساخت `matrix-qa-v1` را در برابر مرز ضبط‌شده Tuwunel اجرا می‌کند. آزمون‌های واحد، قرارداد کاوشگر را با داده ثابت پروتکل Matrix اثبات می‌کنند؛ میزبان استاندارد تطبیق‌دهنده انتقال تضمین کیفیت در [#99707](https://github.com/openclaw/openclaw/pull/99707) مسئول سیم‌کشی هدف واقعی Crabline است.
5. یک Gateway فرزند OpenClaw را با Plugin واقعی Matrix و محدود به حساب سامانه تحت آزمون راه‌اندازی می‌کند.
6. سناریوها را به‌ترتیب اجرا می‌کند، رویدادها را از طریق کارخواه‌های Matrix راه‌انداز/ناظر مشاهده می‌کند و انتظارات مسیریابی/وضعیت را از ترافیک ضبط‌شده به‌دست می‌آورد.
7. سرور خانگی را جمع‌آوری می‌کند، گزارش و مصنوع‌های شواهد را می‌نویسد و سپس خارج می‌شود.

## CLI

```text
pnpm openclaw qa matrix [options]
```

### پرچم‌های رایج

| پرچم                  | پیش‌فرض                                       | توضیح                                                                                                                                   |
| --------------------- | --------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| `--profile <profile>` | `all`                                         | نمایه سناریو. [نمایه‌ها](#profiles) را ببینید.                                                                                                  |
| `--fail-fast`         | خاموش                                           | پس از نخستین بررسی یا سناریوی ناموفق متوقف می‌شود.                                                                                                |
| `--scenario <id>`     | -                                             | فقط این سناریو را اجرا می‌کند. تکرارپذیر است. [سناریوها](#scenarios) را ببینید.                                                                              |
| `--output-dir <path>` | `<repo>/.artifacts/qa-e2e/matrix-<timestamp>` | محل نوشتن گزارش‌ها، خلاصه، فهرست مسیریابی/وضعیت، رویدادهای مشاهده‌شده و گزارش خروجی. مسیرهای نسبی بر مبنای `--repo-root` حل می‌شوند. |
| `--repo-root <path>`  | `process.cwd()`                               | ریشه مخزن هنگام فراخوانی از یک پوشه کاری خنثی.                                                                               |
| `--sut-account <id>`  | `sut`                                         | شناسه حساب Matrix درون پیکربندی Gateway تضمین کیفیت.                                                                                               |

### پرچم‌های ارائه‌دهنده

این لاین از انتقال واقعی Matrix استفاده می‌کند، اما ارائه‌دهنده مدل قابل پیکربندی است:

| پرچم                     | پیش‌فرض          | توضیح                                                                                                                               |
| ------------------------ | ---------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| `--provider-mode <mode>` | `live-frontier`  | `mock-openai` برای اعزام ساختگی قطعی یا `live-frontier` برای ارائه‌دهندگان زنده مرزی. نام مستعار قدیمی `live-openai` همچنان کار می‌کند. |
| `--model <ref>`          | پیش‌فرض ارائه‌دهنده | ارجاع اصلی `provider/model`.                                                                                                             |
| `--alt-model <ref>`      | پیش‌فرض ارائه‌دهنده | ارجاع جایگزین `provider/model` برای مواردی که سناریوها در میانه اجرا تغییر می‌کنند.                                                                            |
| `--fast`                 | خاموش              | حالت سریع ارائه‌دهنده را در صورت پشتیبانی فعال می‌کند.                                                                                                |

تضمین کیفیت Matrix گزینه‌های `--credential-source` یا `--credential-role` را نمی‌پذیرد. این لاین کاربران یک‌بارمصرف را به‌صورت محلی فراهم می‌کند؛ هیچ مخزن مشترکی از اطلاعات اصالت‌سنجی برای اجاره وجود ندارد.

## نمایه‌ها

| نمایه         | کاربرد                                                                                                                                                                                                            |
| --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `all` (پیش‌فرض) | فهرست کامل. کند اما جامع.                                                                                                                                                                                    |
| `fast`          | زیرمجموعه دروازه انتشار که قرارداد ضروری انتقال زنده را می‌آزماید: کنترل اشاره، مسدودسازی فهرست مجاز، شکل پاسخ، ادامه پس از راه‌اندازی مجدد، مشاهده واکنش، تحویل فراداده تأیید اجرا و پاسخ پایه E2EE. |
| `transport`     | سناریوهای رشته گفتگو در سطح انتقال، پیام مستقیم، اتاق، پیوستن خودکار، اشاره/فهرست مجاز، تأیید و واکنش.                                                                                                                   |
| `media`         | پوشش پیوست تصویر، صدا، ویدئو، PDF و EPUB.                                                                                                                                                                   |
| `e2ee-smoke`    | حداقل پوشش E2EE: پاسخ رمزگذاری‌شده پایه، ادامه رشته گفتگو، موفقیت راه‌اندازی اولیه.                                                                                                                                    |
| `e2ee-deep`     | سناریوهای جامع ازدست‌رفتن وضعیت، پشتیبان‌گیری، کلید و بازیابی E2EE.                                                                                                                                                      |
| `e2ee-cli`      | سناریوهای CLI برای `openclaw matrix encryption setup` و `verify *` که از طریق مهار تضمین کیفیت هدایت می‌شوند.                                                                                                                        |

نگاشت دقیق در `extensions/qa-matrix/src/runners/contract/scenario-catalog.ts` قرار دارد.

## سناریوها

تطبیق‌دهنده مشترک Matrix این سناریوهای استاندارد YAML را از طریق `openclaw qa suite --channel-driver live --channel matrix` ارائه می‌کند:

- `channel-chat-baseline`
- `thread-follow-up`
- `thread-isolation`
- `thread-reply-override`
- `dm-shared-session`
- `dm-per-room-session`

`subagent-thread-spawn` همچنان از طریق انتخاب صریح `--scenario subagent-thread-spawn`
در دسترس است، اما تا زمانی که اثبات زنده تکمیل فرزند پایدار نشود، بخشی از مجموعه مشترک پیش‌فرض Matrix نیست.

فهرست باقی‌مانده شناسه‌های سناریوی دستوری، اجتماع `MatrixQaScenarioId` در `extensions/qa-matrix/src/runners/contract/scenario-catalog.ts` است. دسته‌ها:

- رشته گفتگو: `matrix-thread-root-preservation`، `matrix-thread-nested-reply-shape`
- سطح بالا / پیام مستقیم / اتاق: `matrix-top-level-reply-shape`، `matrix-room-*`، `matrix-dm-*`
- جریان‌دهی و پیشرفت ابزار: `matrix-room-partial-streaming-preview`، `matrix-room-quiet-streaming-preview`، `matrix-room-tool-progress-*`، `matrix-room-block-streaming`
- رسانه: `matrix-media-type-coverage`، `matrix-room-image-understanding-attachment`، `matrix-attachment-only-ignored`، `matrix-unsupported-media-safe`
- مسیریابی: `matrix-room-autojoin-invite`، `matrix-secondary-room-*`
- واکنش‌ها: `matrix-reaction-*`
- تأییدها: `matrix-approval-*` (فراداده اجرا/Plugin، حالت جایگزین قطعه‌بندی‌شده، واکنش‌های رد، رشته‌های گفتگو و مسیریابی `target: "both"`)
- راه‌اندازی مجدد و بازپخش: `matrix-restart-*`، `matrix-stale-sync-replay-dedupe`، `matrix-room-membership-loss`، `matrix-homeserver-restart-resume`، `matrix-initial-catchup-then-incremental`
- کنترل اشاره، ربات‌به‌ربات و فهرست‌های مجاز: `matrix-mention-*`، `matrix-allowbots-*`، `matrix-allowlist-*`، `matrix-multi-actor-ordering`، `matrix-inbound-edit-*`، `matrix-mxid-prefixed-command-block`، `matrix-observer-allowlist-override`
- E2EE: `matrix-e2ee-*` (پاسخ پایه، ادامه رشته گفتگو، راه‌اندازی اولیه، چرخه عمر کلید بازیابی، گونه‌های ازدست‌رفتن وضعیت، رفتار پشتیبان‌گیری سرور، بهداشت دستگاه، تأیید SAS / QR / پیام مستقیم، راه‌اندازی مجدد، اطلاعات‌زدایی مصنوع‌ها)
- CLI مربوط به E2EE: `matrix-e2ee-cli-*` (راه‌اندازی رمزگذاری، راه‌اندازی تکرارپذیر، شکست راه‌اندازی اولیه، چرخه عمر کلید بازیابی، چندحسابی، رفت‌وبرگشت پاسخ Gateway، خودتأییدی)

برای اجرای مجموعه‌ای دست‌چین‌شده، `--scenario <id>` را ارسال کنید (تکرارپذیر است)؛ برای نادیده‌گرفتن کنترل نمایه، آن را با `--profile all` ترکیب کنید.

## متغیرهای محیطی

| متغیر                                  | پیش‌فرض                                  | اثر                                                                                                                                                                                                 |
| --------------------------------------- | ----------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `OPENCLAW_QA_MATRIX_TIMEOUT_MS`         | `1800000` (۳۰ دقیقه)                      | سقف زمانی قطعی برای کل اجرا.                                                                                                                                                                        |
| `OPENCLAW_QA_MATRIX_CANARY_TIMEOUT_MS`  | `45000`                                   | محدودیت زمانی پاسخ اولیهٔ قناری. CI انتشار این مقدار را روی اجراکننده‌های اشتراکی افزایش می‌دهد تا کندی نخستین نوبت Gateway پیش از آغاز پوشش سناریوها باعث شکست نشود.                              |
| `OPENCLAW_QA_MATRIX_NO_REPLY_WINDOW_MS` | `8000`                                    | بازهٔ سکوت برای ادعاهای منفیِ عدم پاسخ. این مقدار حداکثر تا `<=` زمان انقضای اجرا محدود می‌شود.                                                                                                    |
| `OPENCLAW_QA_MATRIX_CLEANUP_TIMEOUT_MS` | `90000`                                   | محدودیت زمانی برچیدن Docker. خروجی خطا شامل فرمان بازیابی `docker compose ... down --remove-orphans` است.                                                                                          |
| `OPENCLAW_QA_MATRIX_TUWUNEL_IMAGE`      | `ghcr.io/matrix-construct/tuwunel:v1.5.1` | هنگام اعتبارسنجی با نسخه‌ای متفاوت از Tuwunel، تصویر سرور خانگی را بازنویسی می‌کند.                                                                                                                |
| `OPENCLAW_QA_MATRIX_PROGRESS`           | روشن                                      | `0` خطوط پیشرفت `[matrix-qa] ...` را در stderr خاموش می‌کند. `1` نمایش آن‌ها را اجباری می‌کند.                                                                                                    |
| `OPENCLAW_QA_MATRIX_CAPTURE_CONTENT`    | حذف‌شده                                   | `1` بدنهٔ پیام و `formatted_body` را در `matrix-qa-observed-events.json` نگه می‌دارد. به‌طور پیش‌فرض، برای ایمن ماندن مصنوعات CI محتوا حذف می‌شود.                                                 |
| `OPENCLAW_QA_MATRIX_DISABLE_FORCE_EXIT` | خاموش                                     | `1` اجرای قطعی `process.exit` پس از نوشتن مصنوع را نادیده می‌گیرد. حالت پیش‌فرض خروج را اجباری می‌کند، زیرا دسته‌های رمزنگاری بومی matrix-js-sdk ممکن است پس از تکمیل مصنوع، حلقهٔ رویداد را فعال نگه دارند. |
| `OPENCLAW_RUN_NODE_OUTPUT_LOG`          | تنظیم‌نشده                                | هنگامی که یک راه‌انداز بیرونی (برای مثال `scripts/run-node.mjs`) آن را تنظیم کند، QA ماتریکس به‌جای آغاز tee اختصاصی خود، از همان مسیر گزارش استفاده می‌کند.                                      |

## مصنوعات خروجی

در `--output-dir` نوشته می‌شوند (پیش‌فرض `<repo>/.artifacts/qa-e2e/matrix-<timestamp>` است تا اجراهای پیاپی یکدیگر را بازنویسی نکنند):

- `matrix-qa-report.md`: گزارش پروتکل Markdown (موارد موفق، ناموفق و ردشده، به‌همراه دلیل).
- `matrix-qa-summary.json`: خلاصهٔ ساختاریافتهٔ مناسب برای تجزیه در CI و داشبوردها.
- `matrix-qa-route-state-manifest.json`: فهرست پویای `matrix-qa-v1` با کلید شناسهٔ سناریو. این فایل شکل‌های پالایش‌شدهٔ مسیر/بدنه، ترتیب درخواست‌ها، تلاش‌های مجدد مشاهده‌شده، خطاها، پیوستگی توکن همگام‌سازی و خانواده‌های وضعیت دستگاه/کلید/رسانه/پشتیبان مشاهده‌شده در آن اجرا را ثبت می‌کند. این مدرک اجرایی است، نه یک خط مبنای ثبت‌شده در مخزن.
- `matrix-qa-observed-events.json`: رویدادهای Matrix مشاهده‌شده از کارخواه‌های هدایتگر و ناظر. بدنه‌ها حذف می‌شوند، مگر اینکه `OPENCLAW_QA_MATRIX_CAPTURE_CONTENT=1` باشد؛ فرادادهٔ تأیید با فیلدهای امن منتخب و پیش‌نمایش کوتاه‌شدهٔ فرمان خلاصه می‌شود.
- `matrix-qa-output.log`: ترکیب stdout/stderr اجرا. اگر `OPENCLAW_RUN_NODE_OUTPUT_LOG` تنظیم شده باشد، به‌جای آن از گزارش راه‌انداز بیرونی استفاده می‌شود.

## نکات عیب‌یابی

- **اجرا نزدیک پایان متوقف می‌ماند:** دسته‌های رمزنگاری بومی `matrix-js-sdk` ممکن است بیش از عمر چارچوب آزمایش فعال بمانند. حالت پیش‌فرض پس از نوشتن مصنوع، یک `process.exit` پاک را اجباری می‌کند؛ اگر `OPENCLAW_QA_MATRIX_DISABLE_FORCE_EXIT=1` را تنظیم کنید، انتظار داشته باشید فرایند برای مدتی باقی بماند.
- **خطای پاک‌سازی:** فرمان بازیابی چاپ‌شده (فراخوانی `docker compose ... down --remove-orphans`) را پیدا و برای آزادسازی درگاه سرور خانگی به‌صورت دستی اجرا کنید.
- **بازه‌های ناپایدار ادعای منفی در CI:** وقتی CI سریع است، `OPENCLAW_QA_MATRIX_NO_REPLY_WINDOW_MS` (پیش‌فرض ۸ ثانیه) را کاهش دهید؛ روی اجراکننده‌های اشتراکی کند، آن را افزایش دهید.
- **نیاز به بدنه‌های پالایش‌شده برای گزارش اشکال:** با `OPENCLAW_QA_MATRIX_CAPTURE_CONTENT=1` دوباره اجرا کنید و `matrix-qa-observed-events.json` را پیوست کنید. مصنوع حاصل را حساس در نظر بگیرید.
- **نسخهٔ متفاوت Tuwunel:** `OPENCLAW_QA_MATRIX_TUWUNEL_IMAGE` را به نسخهٔ تحت آزمایش اشاره دهید. تنها تصویر پیش‌فرض سنجاق‌شده در مسیر آزمون ثبت می‌شود.

## قرارداد انتقال زنده

Matrix یکی از سه مسیر انتقال زنده (Matrix، Telegram و Discord) است که فهرست بررسی قراردادی مشترکی دارند؛ این فهرست در [نمای کلی QA: پوشش انتقال زنده](/fa/concepts/qa-e2e-automation#live-transport-coverage) تعریف شده است. `qa-channel` همچنان مجموعهٔ مصنوعی گسترده است و عمداً بخشی از آن ماتریس نیست.
