OpenClaw برای هر اجرای عامل، یک پرامپت سیستمی سفارشی می‌سازد. این پرامپت **متعلق به OpenClaw** است و از پرامپت پیش‌فرض pi-coding-agent استفاده نمی‌کند.

پرامپت توسط OpenClaw مونتاژ می‌شود و به هر اجرای عامل تزریق می‌شود.

Provider Pluginها می‌توانند راهنمایی پرامپت آگاه از کش را بدون جایگزین کردن
کل پرامپت متعلق به OpenClaw اضافه کنند. runtime ارائه‌دهنده می‌تواند:

- مجموعه کوچکی از بخش‌های هسته نام‌گذاری‌شده را جایگزین کند (`interaction_style`,
  `tool_call_style`, `execution_bias`)
- یک **پیشوند پایدار** را بالای مرز کش پرامپت تزریق کند
- یک **پسوند پویا** را پایین مرز کش پرامپت تزریق کند

از مشارکت‌های متعلق به ارائه‌دهنده برای تنظیم اختصاصی خانواده مدل استفاده کنید. جهش پرامپت قدیمی
`before_prompt_build` را برای سازگاری یا تغییرات واقعاً سراسری پرامپت نگه دارید،
نه رفتار عادی ارائه‌دهنده.

لایه OpenAI GPT-5 family قانون اجرای هسته را کوچک نگه می‌دارد و
راهنمایی اختصاصی مدل را برای چسبندگی پرسونا، خروجی موجز، انضباط ابزار،
جست‌وجوی موازی، پوشش تحویل‌دادنی‌ها، راستی‌آزمایی، زمینه مفقود، و
بهداشت ابزار ترمینال اضافه می‌کند.

## ساختار

پرامپت عمداً فشرده است و از بخش‌های ثابت استفاده می‌کند:

- **ابزارها**: یادآور منبع حقیقت ابزار ساختاریافته به‌همراه راهنمای استفاده از ابزار در runtime.
- **سوگیری اجرا**: راهنمای فشرده برای پیگیری تا پایان: روی درخواست‌های قابل اقدام
  در همان نوبت عمل کن، تا انجام شدن یا مسدود شدن ادامه بده، از نتایج ضعیف ابزار
  بازیابی کن، وضعیت تغییرپذیر را زنده بررسی کن، و پیش از نهایی‌سازی راستی‌آزمایی کن.
- **ایمنی**: یادآور کوتاه guardrail برای پرهیز از رفتار قدرت‌طلبانه یا دور زدن نظارت.
- **Skills** (وقتی در دسترس باشد): به مدل می‌گوید چگونه دستورالعمل‌های skill را در صورت نیاز بارگذاری کند.
- **خودبه‌روزرسانی OpenClaw**: نحوه بازرسی امن پیکربندی با
  `config.schema.lookup`، وصله کردن پیکربندی با `config.patch`، جایگزینی کامل
  پیکربندی با `config.apply`، و اجرای `update.run` فقط با درخواست صریح کاربر.
  ابزار فقط-مالک `gateway` همچنین از بازنویسی
  `tools.exec.ask` / `tools.exec.security`، از جمله نام‌های مستعار قدیمی `tools.bash.*`
  که به آن مسیرهای exec محافظت‌شده نرمال می‌شوند، خودداری می‌کند.
- **فضای کاری**: دایرکتوری کاری (`agents.defaults.workspace`).
- **مستندات**: مسیر محلی به مستندات OpenClaw (repo یا بسته npm) و زمان خواندن آن‌ها.
- **فایل‌های فضای کاری (تزریق‌شده)**: نشان می‌دهد فایل‌های bootstrap در پایین گنجانده شده‌اند.
- **Sandbox** (وقتی فعال باشد): runtime سندباکس‌شده، مسیرهای sandbox، و اینکه exec ارتقایافته در دسترس است یا نه را نشان می‌دهد.
- **تاریخ و زمان فعلی**: زمان محلی کاربر، منطقه زمانی، و قالب زمان.
- **برچسب‌های پاسخ**: نحو اختیاری برچسب پاسخ برای ارائه‌دهندگان پشتیبانی‌شده.
- **Heartbeatها**: پرامپت Heartbeat و رفتار ack، وقتی Heartbeatها برای عامل پیش‌فرض فعال باشند.
- **Runtime**: میزبان، سیستم‌عامل، Node، مدل، ریشه repo (وقتی تشخیص داده شود)، سطح thinking (یک خط).
- **استدلال**: سطح دید فعلی + راهنمای toggle برای /reasoning.

OpenClaw محتوای پایدار بزرگ، از جمله **زمینه پروژه**، را بالای
مرز داخلی کش پرامپت نگه می‌دارد. بخش‌های فرّار کانال/نشست مانند
راهنمای embed رابط کنترل، **پیام‌رسانی**، **صدا**، **زمینه گفت‌وگوی گروهی**،
**واکنش‌ها**، **Heartbeatها**، و **Runtime** پایین آن مرز افزوده می‌شوند
تا backendهای محلی با کش‌های پیشوند بتوانند پیشوند پایدار فضای کاری را
در نوبت‌های کانال دوباره استفاده کنند. توضیحات ابزار نیز باید از قرار دادن نام‌های
فعلی کانال خودداری کنند، وقتی schema پذیرفته‌شده همین جزئیات runtime را حمل می‌کند.

بخش ابزارها همچنین شامل راهنمای runtime برای کارهای طولانی‌مدت است:

- برای پیگیری آینده (`check back later`، یادآورها، کارهای تکرارشونده)
  از cron استفاده کن، نه از حلقه‌های خواب `exec`، ترفندهای تأخیر `yieldMs`، یا polling مکرر `process`
- از `exec` / `process` فقط برای دستورهایی استفاده کن که اکنون شروع می‌شوند و
  در پس‌زمینه به اجرا ادامه می‌دهند
- وقتی wake تکمیل خودکار فعال است، دستور را یک‌بار شروع کن و به مسیر wake مبتنی بر push
  تکیه کن وقتی خروجی منتشر می‌کند یا شکست می‌خورد
- وقتی لازم است یک دستور در حال اجرا را بررسی کنی، برای logها، وضعیت، ورودی، یا مداخله
  از `process` استفاده کن
- اگر کار بزرگ‌تر است، `sessions_spawn` را ترجیح بده؛ تکمیل sub-agent
  مبتنی بر push است و به‌طور خودکار به درخواست‌کننده اعلام می‌شود
- فقط برای انتظار تکمیل، `subagents list` / `sessions_list` را در حلقه poll نکن

وقتی ابزار آزمایشی `update_plan` فعال باشد، ابزارها همچنین به مدل می‌گوید
فقط برای کارهای چندمرحله‌ای غیرساده از آن استفاده کند، دقیقاً یک مرحله
`in_progress` نگه دارد، و پس از هر به‌روزرسانی کل برنامه را تکرار نکند.

guardrailهای ایمنی در پرامپت سیستمی توصیه‌ای هستند. آن‌ها رفتار مدل را هدایت می‌کنند اما policy را enforce نمی‌کنند. برای enforce سخت‌گیرانه از policy ابزار، تأییدهای exec، sandboxing، و allowlistهای کانال استفاده کنید؛ operatorها طبق طراحی می‌توانند این‌ها را غیرفعال کنند.

در کانال‌هایی با کارت‌ها/دکمه‌های تأیید بومی، پرامپت runtime اکنون به
عامل می‌گوید ابتدا به همان رابط تأیید بومی تکیه کند. فقط وقتی نتیجه ابزار می‌گوید
تأییدهای chat در دسترس نیستند یا تأیید دستی تنها مسیر است، باید یک دستور دستی
`/approve` را اضافه کند.

## حالت‌های پرامپت

OpenClaw می‌تواند پرامپت‌های سیستمی کوچک‌تری برای sub-agentها render کند. runtime برای هر اجرا یک
`promptMode` تنظیم می‌کند (نه یک پیکربندی کاربرنما):

- `full` (پیش‌فرض): همه بخش‌های بالا را شامل می‌شود.
- `minimal`: برای sub-agentها استفاده می‌شود؛ **Skills**، **Memory Recall**، **خودبه‌روزرسانی OpenClaw**،
  **نام‌های مستعار مدل**، **هویت کاربر**، **برچسب‌های پاسخ**،
  **پیام‌رسانی**، **پاسخ‌های خاموش**، و **Heartbeatها** را حذف می‌کند. ابزارها، **ایمنی**،
  فضای کاری، Sandbox، تاریخ و زمان فعلی (وقتی شناخته شده باشد)، Runtime، و زمینه تزریق‌شده
  در دسترس می‌مانند.
- `none`: فقط خط هویت پایه را برمی‌گرداند.

وقتی `promptMode=minimal` باشد، پرامپت‌های تزریق‌شده اضافی به‌جای **زمینه گفت‌وگوی گروهی**
با **زمینه Subagent** برچسب‌گذاری می‌شوند.

برای اجراهای پاسخ خودکار کانال، OpenClaw می‌تواند بخش عمومی **پاسخ‌های خاموش**
را حذف کند وقتی زمینه مستقیم/گروهی chat از قبل رفتار اختصاصی مکالمه
`NO_REPLY` را شامل می‌شود. این کار از تکرار سازوکارهای token هم در پرامپت سیستمی سراسری
و هم در زمینه کانال جلوگیری می‌کند.

## snapshotهای پرامپت

OpenClaw snapshotهای پرامپت commit‌شده را برای مسیر موفق runtime Codex زیر
`test/fixtures/agents/prompt-snapshots/codex-runtime-happy-path/` نگه می‌دارد. آن‌ها
پارامترهای thread/turn منتخب app-server به‌همراه یک stack بازسازی‌شده لایه پرامپت متصل به مدل
را برای نوبت‌های مستقیم Telegram، گروه Discord، و Heartbeat render می‌کنند. آن stack
شامل یک fixture پرامپت مدل Codex `gpt-5.5` پین‌شده است که از شکل catalog/cache مدل Codex،
متن توسعه‌دهنده permission مسیر موفق Codex،
دستورالعمل‌های توسعه‌دهنده OpenClaw، دستورالعمل‌های collaboration-mode scoped به نوبت
وقتی OpenClaw آن‌ها را فراهم می‌کند، ورودی نوبت کاربر، و ارجاع‌ها به specهای پویای ابزار
تولید شده است.

fixture پرامپت مدل Codex پین‌شده را با
`pnpm prompt:snapshots:sync-codex-model` تازه‌سازی کنید. به‌طور پیش‌فرض، اسکریپت به‌دنبال
کش runtime Codex در `$CODEX_HOME/models_cache.json`، سپس
`~/.codex/models_cache.json` می‌گردد، و فقط بعد از آن به قرارداد checkout نگه‌دارنده Codex
در `~/code/codex/codex-rs/models-manager/models.json` fallback می‌کند. اگر
هیچ‌یک از این منابع وجود نداشته باشند، دستور بدون تغییر دادن fixture commit‌شده خارج می‌شود.
برای تازه‌سازی از یک فایل مشخص `models_cache.json` یا `models.json`، `--catalog <path>` را ارسال کنید.

این snapshotها همچنان capture خام byte-for-byte درخواست OpenAI نیستند. Codex
می‌تواند زمینه فضای کاری متعلق به runtime مانند `AGENTS.md`، زمینه محیط،
memoryها، دستورالعمل‌های app/plugin، و دستورالعمل‌های داخلی Default
collaboration-mode را داخل runtime Codex پس از ارسال پارامترهای thread و turn توسط OpenClaw اضافه کند.

آن‌ها را با `pnpm prompt:snapshots:gen` بازتولید کنید و drift را با
`pnpm prompt:snapshots:check` راستی‌آزمایی کنید. CI بررسی drift را در shard مرزی اضافی
اجرا می‌کند تا تغییرات پرامپت و به‌روزرسانی‌های snapshot به همان PR متصل بمانند.

## تزریق bootstrap فضای کاری

فایل‌های bootstrap کوتاه می‌شوند و زیر **زمینه پروژه** افزوده می‌شوند تا مدل بدون نیاز به read صریح، زمینه هویت و profile را ببیند:

- `AGENTS.md`
- `SOUL.md`
- `TOOLS.md`
- `IDENTITY.md`
- `USER.md`
- `HEARTBEAT.md`
- `BOOTSTRAP.md` (فقط در فضاهای کاری کاملاً جدید)
- `MEMORY.md` وقتی موجود باشد

همه این فایل‌ها در هر نوبت **به پنجره context تزریق می‌شوند** مگر اینکه
gate اختصاصی فایل اعمال شود. `HEARTBEAT.md` در اجراهای عادی حذف می‌شود وقتی
Heartbeatها برای عامل پیش‌فرض غیرفعال باشند یا
`agents.defaults.heartbeat.includeSystemPromptSection` برابر false باشد. فایل‌های تزریق‌شده
را موجز نگه دارید — به‌ویژه `MEMORY.md`، که می‌تواند در طول زمان رشد کند و به
مصرف context غیرمنتظره بالا و Compaction مکررتر منجر شود.

وقتی یک نشست روی harness بومی Codex اجرا می‌شود، Codex فایل `AGENTS.md`
را از طریق discovery اسناد پروژه خودش بارگذاری می‌کند. OpenClaw همچنان فایل‌های bootstrap باقی‌مانده
را resolve می‌کند و آن‌ها را به‌عنوان دستورالعمل‌های پیکربندی Codex forward می‌کند، بنابراین `SOUL.md`،
`TOOLS.md`، `IDENTITY.md`، `USER.md`، `HEARTBEAT.md`، `BOOTSTRAP.md`، و
`MEMORY.md` همان نقش زمینه فضای کاری را بدون duplicating
`AGENTS.md` حفظ می‌کنند.

<Note>
فایل‌های روزانه `memory/*.md` بخشی از زمینه پروژه bootstrap عادی **نیستند**. در نوبت‌های معمولی، آن‌ها در صورت نیاز از طریق ابزارهای `memory_search` و `memory_get` دسترسی پیدا می‌کنند، بنابراین مگر اینکه مدل صراحتاً آن‌ها را بخواند، از پنجره context سهمی نمی‌گیرند. نوبت‌های bare `/new` و `/reset` استثنا هستند: runtime می‌تواند memory روزانه اخیر را به‌عنوان یک بلوک startup-context یک‌باره برای همان نوبت اول prepend کند.
</Note>

فایل‌های بزرگ با یک marker کوتاه می‌شوند. اندازه حداکثری هر فایل توسط
`agents.defaults.bootstrapMaxChars` کنترل می‌شود (پیش‌فرض: 12000). کل محتوای bootstrap تزریق‌شده
در همه فایل‌ها با `agents.defaults.bootstrapTotalMaxChars`
محدود می‌شود (پیش‌فرض: 60000). فایل‌های مفقود یک marker کوتاه فایل-مفقود تزریق می‌کنند. وقتی truncation
رخ می‌دهد، OpenClaw می‌تواند یک notice هشدار موجز پرامپت سیستمی تزریق کند؛ این را با
`agents.defaults.bootstrapPromptTruncationWarning` کنترل کنید (`off`, `once`, `always`;
پیش‌فرض: `once`). شمارش‌های خام/تزریق‌شده تفصیلی در diagnostics مانند
`/context`، `/status`، doctor، و logها می‌مانند.

نشست‌های sub-agent فقط `AGENTS.md` و `TOOLS.md` را تزریق می‌کنند (فایل‌های bootstrap دیگر
فیلتر می‌شوند تا زمینه sub-agent کوچک بماند).

hookهای داخلی می‌توانند از طریق `agent:bootstrap` این مرحله را intercept کنند تا فایل‌های bootstrap تزریق‌شده
را mutate یا replace کنند (برای مثال تعویض `SOUL.md` با یک پرسونا جایگزین).

اگر می‌خواهید صدای عامل کمتر عمومی باشد، با
[راهنمای شخصیت SOUL.md](/fa/concepts/soul) شروع کنید.

برای بازرسی اینکه هر فایل تزریق‌شده چه مقدار سهم دارد (خام در برابر تزریق‌شده، truncation، به‌علاوه سربار schema ابزار)، از `/context list` یا `/context detail` استفاده کنید. [Context](/fa/concepts/context) را ببینید.

## مدیریت زمان

پرامپت سیستمی وقتی منطقه زمانی کاربر شناخته شده باشد، یک بخش اختصاصی **تاریخ و زمان فعلی** دارد. برای پایدار نگه داشتن کش پرامپت، اکنون فقط
**time zone** را شامل می‌شود (بدون ساعت پویا یا قالب زمان).

وقتی عامل به زمان فعلی نیاز دارد از `session_status` استفاده کنید؛ کارت status
یک خط timestamp دارد. همان ابزار می‌تواند به‌صورت اختیاری override مدل در سطح نشست تنظیم کند
(`model=default` آن را پاک می‌کند).

پیکربندی با:

- `agents.defaults.userTimezone`
- `agents.defaults.timeFormat` (`auto` | `12` | `24`)

برای جزئیات کامل رفتار، [تاریخ و زمان](/fa/date-time) را ببینید.

## Skills

وقتی Skills واجد شرایط وجود داشته باشند، OpenClaw یک **فهرست skills موجود** فشرده
(`formatSkillsForPrompt`) تزریق می‌کند که شامل **مسیر فایل** برای هر skill است. پرامپت
به مدل دستور می‌دهد از `read` برای بارگذاری SKILL.md در مکان فهرست‌شده
(فضای کاری، مدیریت‌شده، یا bundled) استفاده کند. اگر هیچ skill واجد شرایطی وجود نداشته باشد،
بخش Skills حذف می‌شود.

واجد شرایط بودن شامل gateهای metadata skill، بررسی‌های runtime environment/config،
و allowlist مؤثر skill عامل است وقتی `agents.defaults.skills` یا
`agents.list[].skills` پیکربندی شده باشد.

Skills بسته‌شده همراه Plugin فقط وقتی واجد شرایط هستند که Plugin مالک آن‌ها فعال باشد.
این به Pluginهای ابزار اجازه می‌دهد راهنماهای عملیاتی عمیق‌تر را بدون قرار دادن همه
آن راهنمایی‌ها مستقیماً در هر توضیح ابزار ارائه کنند.

```
<available_skills>
  <skill>
    <name>...</name>
    <description>...</description>
    <location>...</location>
  </skill>
</available_skills>
```

این کار پرامپت پایه را کوچک نگه می‌دارد و در عین حال استفاده هدفمند از skill را ممکن می‌کند.

بودجه فهرست skills متعلق به subsystem skills است:

- پیش‌فرض سراسری: `skills.limits.maxSkillsPromptChars`
- بازنویسی برای هر عامل: `agents.list[].skillsLimits.maxSkillsPromptChars`

گزیده‌های محدود عمومی زمان اجرا از سطح متفاوتی استفاده می‌کنند:

- `agents.defaults.contextLimits.*`
- `agents.list[].contextLimits.*`

این تفکیک، اندازه‌بندی Skills را از اندازه‌بندی خواندن/تزریق زمان اجرا، مانند `memory_get`، نتایج زنده ابزار، و تازه‌سازی‌های AGENTS.md پس از Compaction جدا نگه می‌دارد.

## مستندات

پرامپت سیستم شامل بخش **مستندات** است. وقتی مستندات محلی در دسترس باشند، به پوشه مستندات محلی OpenClaw اشاره می‌کند (`docs/` در یک Git checkout یا مستندات بسته npm همراه). اگر مستندات محلی در دسترس نباشند، به [https://docs.openclaw.ai](https://docs.openclaw.ai) برمی‌گردد.

همان بخش همچنین شامل محل منبع OpenClaw است. Git checkoutها ریشه منبع محلی را در دسترس می‌گذارند تا عامل بتواند کد را مستقیماً بررسی کند. نصب‌های بسته شامل URL منبع GitHub هستند و به عامل می‌گویند هر زمان مستندات ناقص یا قدیمی بودند، منبع را آنجا بررسی کند. پرامپت همچنین به آینه مستندات عمومی، Discord جامعه، و ClawHub ([https://clawhub.ai](https://clawhub.ai)) برای کشف Skills اشاره می‌کند. به مدل می‌گوید برای رفتار، فرمان‌ها، پیکربندی، یا معماری OpenClaw ابتدا به مستندات مراجعه کند و در صورت امکان خودش `openclaw status` را اجرا کند (فقط وقتی دسترسی ندارد از کاربر بپرسد). به‌طور خاص برای پیکربندی، عامل‌ها را برای مستندات و محدودیت‌های دقیق در سطح فیلد به کنش ابزار `gateway` یعنی `config.schema.lookup` ارجاع می‌دهد، سپس برای راهنمایی گسترده‌تر به `docs/gateway/configuration.md` و `docs/gateway/configuration-reference.md`.
