OpenClaw برای هر اجرای عامل یک پرامپت سیستم سفارشی می‌سازد. پرامپت **متعلق به OpenClaw** است و از پرامپت پیش‌فرض pi-coding-agent استفاده نمی‌کند.

پرامپت توسط OpenClaw سرهم‌بندی و به هر اجرای عامل تزریق می‌شود.

Pluginهای ارائه‌دهنده می‌توانند بدون جایگزین کردن کل پرامپت متعلق به OpenClaw، راهنمای پرامپت آگاه از کش اضافه کنند. runtime ارائه‌دهنده می‌تواند:

- مجموعه کوچکی از بخش‌های اصلی نام‌دار را جایگزین کند (`interaction_style`،
  `tool_call_style`، `execution_bias`)
- یک **پیشوند پایدار** بالای مرز کش پرامپت تزریق کند
- یک **پسوند پویا** پایین مرز کش پرامپت تزریق کند

از مشارکت‌های متعلق به ارائه‌دهنده برای تنظیمات اختصاصی خانواده مدل استفاده کنید. جهش پرامپت قدیمی
`before_prompt_build` را برای سازگاری یا تغییرات واقعا سراسری پرامپت نگه دارید، نه رفتار عادی ارائه‌دهنده.

لایه پوششی خانواده OpenAI GPT-5 قانون اجرای اصلی را کوچک نگه می‌دارد و
راهنمای اختصاصی مدل را برای چسبندگی پرسونا، خروجی موجز، انضباط ابزار،
جست‌وجوی موازی، پوشش خروجی قابل‌تحویل، راستی‌آزمایی، زمینه مفقود، و
بهداشت ابزار ترمینال اضافه می‌کند.

## ساختار

پرامپت عمدا فشرده است و از بخش‌های ثابت استفاده می‌کند:

- **ابزاردهی**: یادآور منبع حقیقت ابزار ساختاریافته به‌همراه راهنمای استفاده از ابزار در runtime.
- **سوگیری اجرا**: راهنمای فشرده برای پیگیری تا پایان: روی درخواست‌های قابل اقدام در همان نوبت عمل کن،
  تا انجام شدن یا مسدود شدن ادامه بده، از نتایج ضعیف ابزار بازیابی کن،
  وضعیت تغییرپذیر را زنده بررسی کن، و پیش از نهایی‌سازی راستی‌آزمایی کن.
- **ایمنی**: یادآور کوتاه گاردریل برای پرهیز از رفتار قدرت‌طلبانه یا دور زدن نظارت.
- **Skills** (در صورت موجود بودن): به مدل می‌گوید چگونه دستورالعمل‌های Skills را هنگام نیاز بارگذاری کند.
- **خودبه‌روزرسانی OpenClaw**: نحوه بازرسی ایمن پیکربندی با
  `config.schema.lookup`، وصله کردن پیکربندی با `config.patch`، جایگزینی کامل
  پیکربندی با `config.apply`، و اجرای `update.run` فقط با درخواست صریح کاربر.
  ابزار فقط-مالک `gateway` همچنین از بازنویسی
  `tools.exec.ask` / `tools.exec.security` خودداری می‌کند، از جمله نام‌های مستعار قدیمی `tools.bash.*`
  که به آن مسیرهای محافظت‌شده exec نرمال‌سازی می‌شوند.
- **فضای کاری**: دایرکتوری کاری (`agents.defaults.workspace`).
- **مستندات**: مسیر محلی به مستندات OpenClaw (مخزن یا بسته npm) و زمان خواندن آن‌ها.
- **فایل‌های فضای کاری (تزریق‌شده)**: نشان می‌دهد فایل‌های bootstrap در ادامه گنجانده شده‌اند.
- **Sandbox** (در صورت فعال بودن): runtime سندباکس‌شده، مسیرهای سندباکس، و اینکه exec ارتقایافته در دسترس است یا نه را نشان می‌دهد.
- **تاریخ و زمان فعلی**: زمان محلی کاربر، منطقه زمانی، و قالب زمان.
- **برچسب‌های پاسخ**: نحو اختیاری برچسب پاسخ برای ارائه‌دهندگان پشتیبانی‌شده.
- **Heartbeatها**: پرامپت Heartbeat و رفتار تایید، وقتی Heartbeatها برای عامل پیش‌فرض فعال هستند.
- **Runtime**: میزبان، سیستم‌عامل، Node، مدل، ریشه مخزن (در صورت تشخیص)، سطح فکر کردن (یک خط).
- **استدلال**: سطح دیدپذیری فعلی + راهنمای کلید /reasoning.

OpenClaw محتوای پایدار بزرگ، از جمله **زمینه پروژه**، را بالای
مرز داخلی کش پرامپت نگه می‌دارد. بخش‌های ناپایدار کانال/جلسه مانند
راهنمای embed رابط کنترل، **پیام‌رسانی**، **صدا**، **زمینه گفت‌وگوی گروهی**،
**واکنش‌ها**، **Heartbeatها**، و **Runtime** پایین آن مرز افزوده می‌شوند
تا backendهای محلی دارای کش پیشوند بتوانند پیشوند پایدار فضای کاری را
در نوبت‌های کانال بازاستفاده کنند. توضیحات ابزار نیز باید از درج نام‌های فعلی
کانال خودداری کنند وقتی schema پذیرفته‌شده از قبل آن جزئیات runtime را حمل می‌کند.

بخش ابزاردهی همچنین شامل راهنمای runtime برای کارهای طولانی‌مدت است:

- برای پیگیری آینده (`check back later`، یادآورها، کارهای تکرارشونده) به‌جای
  حلقه‌های خواب `exec`، ترفندهای تاخیر `yieldMs`، یا polling تکراری `process`
  از Cron استفاده کن
- از `exec` / `process` فقط برای فرمان‌هایی استفاده کن که اکنون شروع می‌شوند و
  در پس‌زمینه به اجرا ادامه می‌دهند
- وقتی بیدارسازی تکمیل خودکار فعال است، فرمان را یک‌بار شروع کن و به
  مسیر بیدارسازی مبتنی بر push تکیه کن وقتی خروجی منتشر می‌کند یا شکست می‌خورد
- وقتی لازم است یک فرمان در حال اجرا را بررسی کنی، برای لاگ‌ها، وضعیت، ورودی، یا مداخله
  از `process` استفاده کن
- اگر کار بزرگ‌تر است، `sessions_spawn` را ترجیح بده؛ تکمیل زیرعامل
  مبتنی بر push است و به‌طور خودکار به درخواست‌کننده اعلام می‌شود
- فقط برای انتظار تکمیل، `subagents list` / `sessions_list` را در حلقه poll نکن

وقتی ابزار آزمایشی `update_plan` فعال است، ابزاردهی همچنین به مدل می‌گوید
فقط برای کار چندمرحله‌ای غیرساده از آن استفاده کند، دقیقا یک گام
`in_progress` نگه دارد، و از تکرار کل برنامه پس از هر به‌روزرسانی خودداری کند.

گاردریل‌های ایمنی در پرامپت سیستم جنبه راهنمایی دارند. آن‌ها رفتار مدل را هدایت می‌کنند اما سیاست را اعمال نمی‌کنند. برای اعمال سخت‌گیرانه از سیاست ابزار، تاییدهای exec، سندباکس، و allowlistهای کانال استفاده کنید؛ اپراتورها بنا بر طراحی می‌توانند این‌ها را غیرفعال کنند.

در کانال‌هایی با کارت‌ها/دکمه‌های تایید بومی، پرامپت runtime اکنون به
عامل می‌گوید ابتدا به همان UI تایید بومی تکیه کند. فقط وقتی باید یک فرمان دستی
`/approve` را درج کند که نتیجه ابزار بگوید تاییدهای chat در دسترس نیستند یا
تایید دستی تنها مسیر است.

## حالت‌های پرامپت

OpenClaw می‌تواند پرامپت‌های سیستم کوچک‌تری برای زیرعامل‌ها render کند. runtime برای هر اجرا یک
`promptMode` تنظیم می‌کند (نه یک پیکربندی روبه‌روی کاربر):

- `full` (پیش‌فرض): همه بخش‌های بالا را شامل می‌شود.
- `minimal`: برای زیرعامل‌ها استفاده می‌شود؛ **Skills**، **فراخوانی حافظه**، **خودبه‌روزرسانی OpenClaw**،
  **نام‌های مستعار مدل**، **هویت کاربر**، **برچسب‌های پاسخ**،
  **پیام‌رسانی**، **پاسخ‌های بی‌صدا**، و **Heartbeatها** را حذف می‌کند. ابزاردهی، **ایمنی**،
  فضای کاری، Sandbox، تاریخ و زمان فعلی (در صورت معلوم بودن)، Runtime، و زمینه تزریق‌شده
  همچنان در دسترس می‌مانند.
- `none`: فقط خط هویت پایه را برمی‌گرداند.

وقتی `promptMode=minimal` باشد، پرامپت‌های تزریق‌شده اضافی با برچسب **زمینه زیرعامل**
به‌جای **زمینه گفت‌وگوی گروهی** مشخص می‌شوند.

برای اجراهای پاسخ خودکار کانال، OpenClaw می‌تواند بخش عمومی **پاسخ‌های بی‌صدا**
را حذف کند وقتی زمینه گفت‌وگوی مستقیم/گروهی از قبل رفتار resolved
اختصاصی مکالمه `NO_REPLY` را شامل می‌شود. این کار از تکرار مکانیک توکن
هم در پرامپت سیستم سراسری و هم در زمینه کانال جلوگیری می‌کند.

## تزریق bootstrap فضای کاری

فایل‌های bootstrap کوتاه و زیر **زمینه پروژه** افزوده می‌شوند تا مدل بدون نیاز به خواندن‌های صریح، زمینه هویت و پروفایل را ببیند:

- `AGENTS.md`
- `SOUL.md`
- `TOOLS.md`
- `IDENTITY.md`
- `USER.md`
- `HEARTBEAT.md`
- `BOOTSTRAP.md` (فقط در فضاهای کاری کاملا جدید)
- `MEMORY.md` در صورت وجود

همه این فایل‌ها در هر نوبت **به پنجره زمینه تزریق می‌شوند** مگر اینکه
gate اختصاصی فایل اعمال شود. `HEARTBEAT.md` در اجراهای عادی وقتی
Heartbeatها برای عامل پیش‌فرض غیرفعال باشند یا
`agents.defaults.heartbeat.includeSystemPromptSection` برابر false باشد حذف می‌شود. فایل‌های تزریق‌شده
را موجز نگه دارید، به‌خصوص `MEMORY.md` که می‌تواند با گذشت زمان رشد کند و به
مصرف غیرمنتظره بالای زمینه و Compactionهای مکررتر منجر شود.

<Note>
فایل‌های روزانه `memory/*.md` بخشی از زمینه پروژه bootstrap عادی نیستند. در نوبت‌های معمولی، آن‌ها هنگام نیاز از طریق ابزارهای `memory_search` و `memory_get` دسترسی می‌شوند، بنابراین مگر اینکه مدل صراحتا آن‌ها را بخواند، در پنجره زمینه حساب نمی‌شوند. نوبت‌های خام `/new` و `/reset` استثنا هستند: runtime می‌تواند حافظه روزانه اخیر را به‌عنوان یک بلوک زمینه آغازین یک‌باره برای همان نوبت اول prepend کند.
</Note>

فایل‌های بزرگ با یک marker کوتاه می‌شوند. حداکثر اندازه هر فایل توسط
`agents.defaults.bootstrapMaxChars` کنترل می‌شود (پیش‌فرض: 12000). کل محتوای bootstrap تزریق‌شده
در همه فایل‌ها توسط `agents.defaults.bootstrapTotalMaxChars` محدود می‌شود
(پیش‌فرض: 60000). فایل‌های مفقود یک marker کوتاه فایل مفقود تزریق می‌کنند. وقتی کوتاه‌سازی
رخ می‌دهد، OpenClaw می‌تواند یک بلوک هشدار در زمینه پروژه تزریق کند؛ این را با
`agents.defaults.bootstrapPromptTruncationWarning` کنترل کنید (`off`، `once`، `always`؛
پیش‌فرض: `once`).

جلسه‌های زیرعامل فقط `AGENTS.md` و `TOOLS.md` را تزریق می‌کنند (فایل‌های bootstrap دیگر
برای کوچک نگه داشتن زمینه زیرعامل فیلتر می‌شوند).

hookهای داخلی می‌توانند این گام را از طریق `agent:bootstrap` رهگیری کنند تا
فایل‌های bootstrap تزریق‌شده را تغییر دهند یا جایگزین کنند (برای مثال جایگزین کردن `SOUL.md` با یک پرسونا جایگزین).

اگر می‌خواهید عامل کمتر عمومی به نظر برسد، از
[راهنمای شخصیت SOUL.md](/fa/concepts/soul) شروع کنید.

برای بررسی اینکه هر فایل تزریق‌شده چقدر سهم دارد (خام در برابر تزریق‌شده، کوتاه‌سازی، به‌علاوه سربار schema ابزار)، از `/context list` یا `/context detail` استفاده کنید. [زمینه](/fa/concepts/context) را ببینید.

## مدیریت زمان

پرامپت سیستم وقتی منطقه زمانی کاربر معلوم باشد، یک بخش اختصاصی **تاریخ و زمان فعلی** دارد.
برای پایدار ماندن کش پرامپت، اکنون فقط **منطقه زمانی** را شامل می‌شود
(بدون ساعت پویا یا قالب زمان).

وقتی عامل به زمان فعلی نیاز دارد از `session_status` استفاده کنید؛ کارت وضعیت
یک خط timestamp دارد. همان ابزار می‌تواند به‌صورت اختیاری override مدل برای هر جلسه را تنظیم کند
(`model=default` آن را پاک می‌کند).

پیکربندی با:

- `agents.defaults.userTimezone`
- `agents.defaults.timeFormat` (`auto` | `12` | `24`)

برای جزئیات کامل رفتار، [تاریخ و زمان](/fa/date-time) را ببینید.

## Skills

وقتی Skills واجد شرایط وجود داشته باشند، OpenClaw یک **فهرست Skills موجود** فشرده
(`formatSkillsForPrompt`) تزریق می‌کند که شامل **مسیر فایل** برای هر skill است. پرامپت
به مدل دستور می‌دهد از `read` برای بارگذاری SKILL.md در مکان فهرست‌شده
(فضای کاری، مدیریت‌شده، یا bundled) استفاده کند. اگر هیچ Skills واجد شرایطی نباشد،
بخش Skills حذف می‌شود.

واجد شرایط بودن شامل gateهای metadata مربوط به skill، بررسی‌های محیط/runtime و پیکربندی،
و allowlist موثر skill عامل وقتی `agents.defaults.skills` یا
`agents.list[].skills` پیکربندی شده باشد است.

Skillsهای bundled همراه Plugin فقط وقتی واجد شرایط هستند که Plugin مالک آن‌ها فعال باشد.
این به Pluginهای ابزار اجازه می‌دهد راهنماهای عملیاتی عمیق‌تری عرضه کنند بدون اینکه همه
آن راهنما مستقیما در هر توضیح ابزار embed شود.

```
<available_skills>
  <skill>
    <name>...</name>
    <description>...</description>
    <location>...</location>
  </skill>
</available_skills>
```

این کار پرامپت پایه را کوچک نگه می‌دارد و هم‌زمان استفاده هدفمند از Skills را ممکن می‌کند.

بودجه فهرست Skills متعلق به زیرسیستم Skills است:

- پیش‌فرض سراسری: `skills.limits.maxSkillsPromptChars`
- override برای هر عامل: `agents.list[].skillsLimits.maxSkillsPromptChars`

گزیده‌های bounded عمومی runtime از سطح دیگری استفاده می‌کنند:

- `agents.defaults.contextLimits.*`
- `agents.list[].contextLimits.*`

این جداسازی، اندازه‌گذاری Skills را از اندازه‌گذاری خواندن/تزریق runtime مانند
`memory_get`، نتایج زنده ابزار، و refreshهای AGENTS.md پس از Compaction جدا نگه می‌دارد.

## مستندات

پرامپت سیستم شامل یک بخش **مستندات** است. وقتی مستندات محلی در دسترس باشد،
به دایرکتوری محلی مستندات OpenClaw اشاره می‌کند (`docs/` در یک checkout گیت یا مستندات بسته npm
bundled). اگر مستندات محلی در دسترس نباشد، به
[https://docs.openclaw.ai](https://docs.openclaw.ai) fallback می‌کند.

همان بخش همچنین شامل مکان source OpenClaw است. checkoutهای گیت ریشه source محلی را ارائه می‌کنند
تا عامل بتواند code را مستقیم بررسی کند. نصب‌های بسته شامل URL source در GitHub هستند
و به عامل می‌گویند هرگاه مستندات ناقص یا stale بود، source را آنجا بررسی کند.
پرامپت همچنین آینه مستندات عمومی، Discord جامعه، و ClawHub
([https://clawhub.ai](https://clawhub.ai)) را برای کشف Skills یادآوری می‌کند. به مدل می‌گوید
برای رفتار، فرمان‌ها، پیکربندی، یا معماری OpenClaw ابتدا به مستندات رجوع کند، و
وقتی ممکن است خودش `openclaw status` را اجرا کند (فقط وقتی دسترسی ندارد از کاربر بپرسد).
به‌طور خاص برای پیکربندی، عامل‌ها را به action ابزار `gateway` یعنی
`config.schema.lookup` برای مستندات و محدودیت‌های دقیق سطح فیلد، سپس به
`docs/gateway/configuration.md` و `docs/gateway/configuration-reference.md`
برای راهنمایی گسترده‌تر ارجاع می‌دهد.
