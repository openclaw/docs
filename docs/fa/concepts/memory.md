OpenClaw با نوشتن فایل‌های Markdown ساده در فضای کاری عامل شما
(پیش‌فرض `~/.openclaw/workspace`) موارد را به خاطر می‌سپارد. مدل فقط چیزهایی را به خاطر می‌سپارد که
روی دیسک ذخیره شده‌اند؛ هیچ حالت پنهانی وجود ندارد.

## نحوه کار

عامل شما سه فایل مرتبط با حافظه دارد:

- **`MEMORY.md`** — حافظه بلندمدت. واقعیت‌ها، ترجیحات و
  تصمیم‌های ماندگار. در آغاز یک نشست بارگذاری می‌شود.
- **`memory/YYYY-MM-DD.md`** (یا `memory/YYYY-MM-DD-<slug>.md`) — یادداشت‌های روزانه.
  زمینه جاری و مشاهدات. یادداشت‌های تاریخ‌دار امروز و دیروز با اجرای صرف
  `/new` یا `/reset` به‌طور خودکار بارگذاری می‌شوند؛ گونه‌های دارای نامک، مانند مواردی که
  هوک حافظه نشست همراه می‌نویسد، در کنار فایلِ فقط‌تاریخ دریافت می‌شوند.
- **`DREAMS.md`** (اختیاری) — دفترچه رؤیا و خلاصه‌های پیمایش Dreaming برای
  بازبینی انسانی، شامل ورودی‌های تکمیل سابقه مبتنی بر شواهد.

<Tip>
اگر می‌خواهید عاملتان چیزی را به خاطر بسپارد، کافی است از آن بخواهید: «به خاطر بسپار که
من TypeScript را ترجیح می‌دهم.» عامل یادداشت را در فایل مناسب می‌نویسد.
</Tip>

## چه چیزی کجا قرار می‌گیرد

`MEMORY.md` لایه فشرده و گزینش‌شده است: واقعیت‌ها، ترجیحات، تصمیم‌های
پایدار و خلاصه‌های کوتاهی که باید در آغاز یک نشست در دسترس باشند. این فایل
رونوشت خام، گزارش روزانه یا بایگانی جامع نیست.

فایل‌های `memory/YYYY-MM-DD.md` لایه کاری هستند: یادداشت‌های روزانه تفصیلی،
مشاهدات، خلاصه‌های نشست و زمینه خامی که ممکن است بعداً همچنان مفید باشند.
این فایل‌ها برای `memory_search` و `memory_get` نمایه‌سازی می‌شوند، اما در هر نوبت
به اعلان راه‌اندازی تزریق نمی‌شوند.

عامل به‌مرور مطالب مفید را از یادداشت‌های روزانه استخراج و در
`MEMORY.md` ذخیره می‌کند و ورودی‌های بلندمدت منسوخ را حذف می‌کند. دستورالعمل‌های
تولیدشده فضای کاری و جریان Heartbeat این کار را به‌صورت دوره‌ای انجام می‌دهند؛ لازم نیست
برای هر جزئیات `MEMORY.md` را دستی ویرایش کنید.

اگر `MEMORY.md` از بودجه فایل راه‌اندازی فراتر برود، OpenClaw فایل روی
دیسک را دست‌نخورده نگه می‌دارد، اما نسخه تزریق‌شده به زمینه را کوتاه می‌کند. این وضعیت را
نشانه‌ای برای انتقال مطالب تفصیلی به `memory/*.md`، نگه‌داشتن فقط یک
خلاصه ماندگار در `MEMORY.md`، یا افزایش محدودیت‌های راه‌اندازی در صورت تمایل به مصرف
بودجه بیشتر اعلان در نظر بگیرید. برای مشاهده اندازه‌های خام در برابر اندازه‌های تزریق‌شده و وضعیت کوتاه‌سازی، از
`/context list`، `/context detail` یا `openclaw doctor` استفاده کنید.

## واردکردن از دستیارهای کدنویسی

رابط کنترل می‌تواند حافظه محلی موجود را از Codex و Claude Code وارد کند.
**Settings** → **Import Memory** را باز کنید، عامل مقصد را انتخاب کنید، فایل‌های
شناسایی‌شده را بازبینی کنید و واردکردن را تأیید کنید. OpenClaw فقط حافظه Markdown را کپی می‌کند:

- Codex: فایل‌های یکپارچه `MEMORY.md` و `memory_summary.md` در
  `~/.codex/memories` (یا `CODEX_HOME/memories`). فایل‌های خام اجرای مرحله‌ای و رونوشت
  وارد نمی‌شوند.
- Claude Code: فایل‌های Markdown از پوشه حافظه خودکار هر پروژه در
  `~/.claude/projects/*/memory`، به‌علاوه `autoMemoryDirectory` پیکربندی‌شده توسط کاربر
  در صورت وجود. دستورالعمل‌های پروژه، نشست‌ها، تنظیمات
  و اعتبارنامه‌ها بخشی از این عملیات مختص حافظه نیستند.

فایل‌های واردشده در `memory/imports/codex/` و
`memory/imports/claude-code/` در فضای کاری عامل انتخاب‌شده جدا باقی می‌مانند. آن‌ها برای
`memory_search` نمایه‌سازی می‌شوند و از طریق `memory_get` در دسترس‌اند؛ با
`MEMORY.md` راه‌اندازی عامل ادغام نمی‌شوند. فایل‌های مبدأ بدون تغییر باقی می‌مانند.

پیش‌نمایش تداخل‌های مقصد را مشخص می‌کند. برای جایگزینی آن فایل‌ها، **Replace existing imports** را
فعال کنید؛ اعمال عملیات، یک پشتیبان تأییدشده پیش از واردکردن ایجاد می‌کند و
نسخه‌های مجزای فایل‌های بازنویسی‌شده را در گزارش مهاجرت نگه می‌دارد.

## حافظه‌های حساس به اقدام

بیشتر حافظه‌ها یادداشت‌های معمولی Markdown هستند. برخی بر کاری که عامل باید
بعداً انجام دهد اثر می‌گذارند؛ برای این موارد، فقط خود واقعیت را ثبت نکنید، بلکه زمان امن‌بودن
اقدام بر اساس یادداشت را نیز ثبت کنید.

هنگامی که یک یادداشت شامل موارد زیر است، آن مرز اقدام را ثبت کنید:

- الزامات تأیید یا مجوز،
- محدودیت‌های موقت،
- واگذاری به نشست، رشته یا شخصی دیگر،
- شرایط انقضا،
- زمان امن برای اقدام،
- مرجع منبع یا مالک،
- دستورالعمل‌های پرهیز از اقدامی وسوسه‌انگیز.

یک حافظه حساس به اقدامِ مفید، موارد زیر را روشن می‌کند:

- چه چیزی رفتار آینده را تغییر می‌دهد،
- چه زمانی یا تحت چه شرایطی اعمال می‌شود،
- چه زمانی منقضی می‌شود، یا چه چیزی اجازه اقدام را صادر می‌کند،
- عامل باید از انجام چه کاری پرهیز کند،
- منبع یا مالک چه کسی است، اگر این موضوع بر اعتماد یا اختیار اثر می‌گذارد.

حافظه می‌تواند زمینه تأیید را حفظ کند، اما سیاست را اعمال نمی‌کند. برای
کنترل‌های عملیاتی سخت‌گیرانه از تنظیمات تأیید OpenClaw، محیط ایزوله و وظایف زمان‌بندی‌شده
استفاده کنید.

مثال:

```md
مهاجرت API در نشست دیگری در حال طراحی است. نوبت‌های آینده نباید
پیاده‌سازی API را از این رشته ویرایش کنند؛ تا زمان نهایی‌شدن طرح مهاجرت، از یافته‌های اینجا فقط به‌عنوان
ورودی طراحی استفاده کنید.
```

مثالی دیگر:

```md
گزارشی از یک منبع نامطمئن پیش از ترویج به بازبینی نیاز دارد. نوبت‌های آینده
باید آن را فقط به‌عنوان مدرک در نظر بگیرند؛ تا زمانی که یک بازبین مورد اعتماد محتوا را
تأیید نکرده است، آن را به‌عنوان حافظه ماندگار ذخیره نکنید.
```

این یک طرح‌واره الزامی برای همه حافظه‌ها نیست؛ واقعیت‌های ساده می‌توانند مختصر باقی بمانند.
وقتی ازدست‌رفتن زمینه زمان‌بندی، اختیار، انقضا یا
امن‌بودن اقدام ممکن است باعث شود عامل بعداً کار اشتباهی انجام دهد، از مرزهای حساس به اقدام استفاده کنید.

برای پیگیری‌های استنباط‌شده و کوتاه‌مدت از [تعهدات](/fa/concepts/commitments) استفاده کنید.
برای یادآوری‌های دقیق، بررسی‌های زمان‌بندی‌شده و
کارهای تکرارشونده از [وظایف زمان‌بندی‌شده](/fa/automation/cron-jobs) استفاده کنید. حافظه همچنان می‌تواند زمینه ماندگار پیرامون
هر یک از این مسیرها را خلاصه کند.

## تعهدات استنباط‌شده

برخی پیگیری‌های آینده واقعیت‌های ماندگار نیستند. اگر به مصاحبه‌ای
در فردا اشاره کنید، حافظه مفید ممکن است «پس از مصاحبه پیگیری کن» باشد، نه «این را
برای همیشه در `MEMORY.md` ذخیره کن.»

[تعهدات](/fa/concepts/commitments) حافظه‌های پیگیری اختیاری و کوتاه‌مدت
برای چنین مواردی هستند. OpenClaw آن‌ها را در یک گذر پس‌زمینه پنهان استنباط می‌کند،
دامنه‌شان را به همان عامل و کانال محدود می‌کند و پیگیری‌های سررسیدشده را از طریق
Heartbeat تحویل می‌دهد. یادآوری‌های صریح همچنان از [وظایف زمان‌بندی‌شده](/fa/automation/cron-jobs) استفاده می‌کنند.

## ابزارهای حافظه

عامل دو ابزار برای کار با حافظه دارد:

- **`memory_search`** — با استفاده از جست‌وجوی معنایی، یادداشت‌های مرتبط را حتی زمانی که
  عبارت‌بندی با متن اصلی متفاوت است پیدا می‌کند.
- **`memory_get`** — یک فایل حافظه یا محدوده‌ای از خطوط مشخص را می‌خواند.

هر دو ابزار را Plugin فعال حافظه فراهم می‌کند (پیش‌فرض: `memory-core`).

## جست‌وجوی حافظه

وقتی یک ارائه‌دهنده تعبیه پیکربندی شده باشد، `memory_search` از جست‌وجوی ترکیبی استفاده می‌کند:
شباهت برداری (معنای معنایی) همراه با تطبیق کلیدواژه‌ها (عبارت‌های دقیق
مانند شناسه‌ها و نمادهای کد). این قابلیت با یک کلید API
برای هر ارائه‌دهنده پشتیبانی‌شده، بدون پیکربندی اضافی کار می‌کند.

<Info>
OpenClaw به‌طور پیش‌فرض از تعبیه‌های OpenAI استفاده می‌کند. برای استفاده از Gemini، Voyage،
Mistral، Bedrock، DeepInfra، GGUF محلی، Ollama، LM Studio، GitHub Copilot یا
یک نقطه پایانی عمومی سازگار با OpenAI، مقدار
`agents.defaults.memorySearch.provider` را صریحاً تنظیم کنید.
</Info>

برای آشنایی با نحوه کار جست‌وجو، گزینه‌های تنظیم دقیق
و راه‌اندازی ارائه‌دهنده، به [جست‌وجوی حافظه](/fa/concepts/memory-search) مراجعه کنید.

## لایه ویکی دانش

اگر می‌خواهید حافظه ماندگار بیشتر شبیه یک پایگاه دانش نگهداری‌شده
عمل کند تا یادداشت‌های خام، از Plugin همراه `memory-wiki` استفاده کنید. این Plugin دانش ماندگار
را در یک مخزن ویکی با ساختار قطعی صفحه، ادعاها و شواهد ساخت‌یافته،
ردیابی تناقض و تازگی، داشبوردهای تولیدشده،
چکیده‌های گردآوری‌شده و ابزارهای بومی ویکی (`wiki_status`،
`wiki_search`، `wiki_get`، `wiki_apply`، `wiki_lint`) گردآوری می‌کند.

`memory-wiki` جایگزین Plugin فعال حافظه نمی‌شود؛ Plugin فعال حافظه
همچنان مالک بازیابی، ترویج و Dreaming است. `memory-wiki` یک
لایه دانش غنی از منشأ را در کنار آن اضافه می‌کند.

## تخلیه خودکار حافظه

پیش از آنکه [Compaction](/fa/concepts/compaction) گفت‌وگوی شما را خلاصه کند،
OpenClaw یک نوبت بی‌صدا اجرا می‌کند که به عامل یادآوری می‌کند زمینه مهم را
در فایل‌های حافظه ذخیره کند. این قابلیت به‌طور پیش‌فرض فعال است؛ برای غیرفعال‌کردن آن
`agents.defaults.compaction.memoryFlush.enabled: false` را تنظیم کنید.

برای نگه‌داشتن آن نوبت نگهداری روی یک مدل محلی، یک بازنویسی دقیق تنظیم کنید که
فقط بر نوبت تخلیه حافظه اعمال شود (این تنظیم زنجیره جایگزین مدل
نشست فعال را به ارث نمی‌برد):

```json
{
  "agents": {
    "defaults": {
      "compaction": {
        "memoryFlush": {
          "model": "ollama/qwen3:8b"
        }
      }
    }
  }
}
```

<Tip>
تخلیه حافظه از ازدست‌رفتن زمینه هنگام Compaction جلوگیری می‌کند. اگر عامل شما
واقعیت‌های مهمی در گفت‌وگو داشته باشد که هنوز در فایلی نوشته نشده‌اند،
پیش از ایجاد خلاصه به‌طور خودکار ذخیره می‌شوند.
</Tip>

## Dreaming

Dreaming یک گذر اختیاری تثبیت پس‌زمینه برای حافظه است. این قابلیت
سیگنال‌های بازیابی کوتاه‌مدت را جمع‌آوری می‌کند، به گزینه‌ها امتیاز می‌دهد و فقط موارد واجد شرایط
را به حافظه بلندمدت (`MEMORY.md`) ترویج می‌کند:

- **اختیاری**: به‌طور پیش‌فرض غیرفعال است.
- **زمان‌بندی‌شده**: در صورت فعال‌بودن، `memory-core` یک کار Cron تکرارشونده
  را برای پیمایش کامل Dreaming به‌طور خودکار مدیریت می‌کند.
- **آستانه‌دار**: ترویج‌ها باید از دروازه‌های امتیاز، بسامد بازیابی و
  تنوع پرس‌وجو عبور کنند.
- **قابل بازبینی**: خلاصه‌های مرحله و ورودی‌های دفترچه در
  `DREAMS.md` برای بازبینی انسانی نوشته می‌شوند.

برای رفتار مرحله‌ها، سیگنال‌های امتیازدهی و جزئیات دفترچه رؤیا، به
[Dreaming](/fa/concepts/dreaming) مراجعه کنید.

## تکمیل سابقه مبتنی بر شواهد و ترویج زنده

سامانه Dreaming دو مسیر بازبینی مرتبط دارد:

- **Dreaming زنده** با مخزن کوتاه‌مدت Dreaming در
  `memory/.dreams/` کار می‌کند و مرحله عمیق معمولی از آن برای تصمیم‌گیری درباره مواردی که
  به `MEMORY.md` راه می‌یابند استفاده می‌کند.
- **تکمیل سابقه مبتنی بر شواهد** یادداشت‌های تاریخی `memory/YYYY-MM-DD.md` را به‌عنوان
  فایل‌های روز مستقل می‌خواند و خروجی بازبینی ساخت‌یافته را در `DREAMS.md` می‌نویسد.

تکمیل سابقه مبتنی بر شواهد برای بازپخش یادداشت‌های قدیمی‌تر و بررسی مواردی که
سامانه ماندگار می‌داند مفید است، بدون اینکه لازم باشد `MEMORY.md` را دستی ویرایش کنید.

```bash
openclaw memory rem-backfill --path ./memory --stage-short-term
```

پرچم `--stage-short-term` گزینه‌های ماندگار مبتنی بر شواهد را در همان
مخزن کوتاه‌مدت Dreaming قرار می‌دهد که مرحله عمیق معمولی از قبل استفاده می‌کند؛ آن‌ها را
مستقیماً ترویج نمی‌کند. بنابراین:

- `DREAMS.md` سطح بازبینی انسانی باقی می‌ماند.
- مخزن کوتاه‌مدت سطح رتبه‌بندی ویژه ماشین باقی می‌ماند.
- `MEMORY.md` همچنان فقط با ترویج عمیق نوشته می‌شود.

برای لغو یک بازپخش بدون دست‌زدن به ورودی‌های معمول دفترچه یا وضعیت عادی بازیابی:

```bash
openclaw memory rem-backfill --rollback
openclaw memory rem-backfill --rollback-short-term
```

## CLI

```bash
openclaw memory status          # بررسی وضعیت نمایه و ارائه‌دهنده
openclaw memory search "query"  # جست‌وجو از خط فرمان
openclaw memory index --force   # بازسازی نمایه
```
