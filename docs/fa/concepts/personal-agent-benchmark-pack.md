بستهٔ معیار سنجش عامل شخصی، بسته‌ای کوچک از سناریوهای تضمین کیفیت با پشتوانهٔ مخزن برای
گردش‌کارهای دستیار شخصی محلی است. این بسته یک معیار سنجش عمومی مدل نیست و
به اجراکنندهٔ جدیدی نیاز ندارد: از پشتهٔ خصوصی تضمین کیفیت ([نمای کلی تضمین کیفیت](/fa/concepts/qa-e2e-automation))،
[کانال تضمین کیفیت](/fa/channels/qa-channel) مصنوعی و کاتالوگ YAML موجود در
`qa/scenarios` دوباره استفاده می‌کند.

## سناریوها

ده سناریو که در `qa/scenarios/personal/*.yaml` تعریف شده‌اند:

| شناسهٔ سناریو                              | بررسی‌ها                                                                                               |
| ------------------------------------------ | ------------------------------------------------------------------------------------------------------ |
| `personal-reminder-roundtrip`              | یادآوری‌های شخصی ساختگی از طریق تحویل Cron محلی                                                        |
| `personal-channel-thread-reply`            | مسیریابی پیام مستقیم ساختگی و پاسخ رشته از طریق `qa-channel`                                          |
| `personal-memory-preference-recall`        | بازیابی ترجیح ساختگی از فایل‌های حافظهٔ فضای کاری موقت تضمین کیفیت                                    |
| `personal-redaction-no-secret-leak`        | بررسی‌های ساختگی عدم بازگویی اطلاعات محرمانه                                                          |
| `personal-tool-safety-followthrough`       | پیگیری ایمن ابزار با پشتوانهٔ خواندن، پس از یک نوبت کوتاه به سبک تأیید                                 |
| `personal-approval-denial-stop`            | رفتار توقف هنگام رد تأیید برای درخواست حساس خواندن محلی                                               |
| `personal-task-followthrough-status`       | گزارش وضعیت وظیفه با پشتوانهٔ شواهد که موارد در انتظار، مسدود و انجام‌شده را جدا نگه می‌دارد           |
| `personal-share-safe-diagnostics-artifact` | مصنوعات عیب‌یابی ایمن برای اشتراک‌گذاری که ضمن حذف محتوای شخصی خام، وضعیت مفید را حفظ می‌کنند          |
| `personal-no-fake-progress`                | ادعاهای تکمیل با پشتوانهٔ شواهد که پیش از وجود شواهد محلی، از نمایش پیشرفت ساختگی خودداری می‌کنند      |
| `personal-failure-recovery`                | بازیابی از شکست که وضعیت جزئی را گزارش می‌کند و مرزهای تلاش مجدد را روشن نگه می‌دارد                   |

فرادادهٔ ماشین‌خوان بسته (فهرست شناسه‌ها، عنوان و توضیحات) با نام
`QA_PERSONAL_AGENT_SCENARIO_IDS` در `extensions/qa-lab/src/scenario-packs.ts` قرار دارد.
بسته را با `--pack personal-agent` اجرا کنید:

```bash
OPENCLAW_ENABLE_PRIVATE_QA_CLI=1 pnpm openclaw qa suite \
  --provider-mode mock-openai \
  --pack personal-agent \
  --concurrency 1
```

`--pack` همراه با پرچم‌های تکرارشوندهٔ `--scenario` به‌صورت افزایشی عمل می‌کند. ابتدا سناریوهای صریح اجرا
می‌شوند، سپس سناریوهای بسته به ترتیب `QA_PERSONAL_AGENT_SCENARIO_IDS`
و پس از حذف موارد تکراری اجرا می‌شوند.

این بسته `qa-channel` را با `mock-openai` یا یک مسیر ارائه‌دهندهٔ محلی دیگر برای تضمین کیفیت
هدف قرار می‌دهد. آن را به سرویس‌های گفت‌وگوی زنده یا حساب‌های شخصی واقعی متصل نکنید.

## مدل حریم خصوصی

سناریوها فقط از کاربران ساختگی، ترجیحات ساختگی، اطلاعات محرمانهٔ ساختگی و
فضای کاری موقت Gateway تضمین کیفیت که مجموعه ایجاد می‌کند استفاده می‌کنند. آن‌ها نباید حافظهٔ کاربر،
نشست‌ها، اطلاعات احراز هویت، عامل‌های راه‌اندازی، پیکربندی‌های سراسری یا وضعیت زندهٔ Gateway
در OpenClaw را بخوانند یا بنویسند.

مصنوعات در پوشهٔ مصنوعات موجود مجموعهٔ تضمین کیفیت باقی می‌مانند و مانند
خروجی آزمون با آن‌ها رفتار می‌شود. بررسی‌های حذف اطلاعات حساس از نشانگرهای ساختگی استفاده می‌کنند تا بررسی
و ثبت شکست‌ها در مسائل ایمن باشد.

## گسترش بسته

موارد جدید `.yaml` را زیر `qa/scenarios/personal/` اضافه کنید، سپس شناسهٔ سناریو را
به `QA_PERSONAL_AGENT_SCENARIO_IDS` بیفزایید. هر مورد را کوچک، محلی، قطعی
در `mock-openai` و متمرکز بر یک رفتار دستیار شخصی نگه دارید.

گزینه‌های مناسب برای ادامه: بررسی‌های برون‌بری مسیر اجرای حذف‌شده از اطلاعات حساس و بررسی‌های گردش‌کار
Plugin صرفاً محلی.

تا زمانی که کاتالوگ سناریو موارد پایدار کافی برای توجیه چنین سطحی ندارد، از افزودن اجراکننده، Plugin،
وابستگی، انتقال زنده یا داور مدل جدید خودداری کنید.
