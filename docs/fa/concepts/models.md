یک ارجاع مدل (`provider/model`) ارائه‌دهنده و مدل را انتخاب می‌کند، نه زمان‌اجرای سطح‌پایین
عامل را. وقتی سیاست زمان‌اجرا تنظیم نشده یا `auto` باشد، سیاست مسیریابی متعلق به ارائه‌دهنده OpenAI
ممکن است Codex را فقط برای یک مسیر رسمی دقیق HTTPS Platform
Responses یا ChatGPT Responses بدون بازنویسی درخواست از سوی نویسنده انتخاب کند؛ پیشوند
`openai/*` به‌تنهایی هرگز Codex را انتخاب نمی‌کند. سازگارکننده‌های Completions، نقاط پایانی سفارشی
و رفتار درخواست تعریف‌شده روی OpenClaw باقی می‌مانند. نقاط پایانی رسمی
HTTP متن ساده رد می‌شوند. [زمان‌اجرای ضمنی عامل OpenAI](/fa/providers/openai#implicit-agent-runtime) را ببینید.

ارجاع‌های اشتراکی Copilot (`github-copilot/*`) را می‌توان برای استفاده از Plugin خارجی
زمان‌اجرای عامل GitHub Copilot فعال کرد، اما این مسیر همیشه صریح است (و هرگز
با `auto` انتخاب نمی‌شود). بازنویسی‌های زمان‌اجرا باید در سیاست ارائه‌دهنده/مدل باشند، نه در
کل عامل یا نشست. انتخاب زمان‌اجرا صورت‌حساب را تعیین نمی‌کند:
اعتبارنامه‌های کلید API در OpenAI و اشتراک ChatGPT/Codex از یکدیگر متمایز می‌مانند. به
[زمان‌های اجرای عامل](/fa/concepts/agent-runtimes) و
[زمان‌اجرای عامل GitHub Copilot](/fa/plugins/copilot) مراجعه کنید.

## ترتیب انتخاب

<Steps>
  <Step title="مدل اصلی">
    `agents.defaults.model.primary` (یا `agents.defaults.model` به‌صورت یک رشته ساده).
  </Step>
  <Step title="مدل‌های جایگزین">
    `agents.defaults.model.fallbacks`، که به‌ترتیب امتحان می‌شوند.
  </Step>
  <Step title="جابه‌جایی اضطراری احراز هویت">
    چرخش پروفایل احراز هویت درون یک ارائه‌دهنده انجام می‌شود، پیش از آنکه OpenClaw به مدل جایگزین بعدی برود.
  </Step>
</Steps>

سطوح مرتبط پیکربندی مدل:

- `agents.defaults.models` فهرست مجاز/کاتالوگ مدل‌هایی است که OpenClaw می‌تواند استفاده کند، به‌همراه نام‌های مستعار. برای مجاز کردن همه مدل‌های کشف‌شده از یک ارائه‌دهنده بدون فهرست‌کردن تک‌تک آن‌ها، از ورودی‌های `provider/*` استفاده کنید.
- `agents.defaults.utilityModel` یک مدل اختیاری کم‌هزینه‌تر برای وظایف داخلی کوتاه مانند عنوان‌های تولیدشده برای نشست‌های داشبورد، عنوان‌های رشته/موضوع کانال‌های پشتیبانی‌شده و روایت پیشرفت است. مقدار `agents.list[].utilityModel` مختص هر عامل آن را بازنویسی می‌کند. وقتی تنظیم نشده باشد، OpenClaw در صورت وجود از مدل کوچک پیش‌فرض اعلام‌شده ارائه‌دهنده اصلی استفاده می‌کند (OpenAI ← `gpt-5.6-luna`، Anthropic ← `claude-haiku-4-5`)؛ در غیر این صورت از مدل اصلی عامل استفاده می‌کند. برای غیرفعال‌کردن مسیریابی کاربردی، آن را روی رشته خالی تنظیم کنید. وظایف کاربردی فراخوانی‌های مدل جداگانه‌ای هستند و ممکن است محتوای محدود وظیفه را به ارائه‌دهنده مدل انتخاب‌شده ارسال کنند.
- `agents.defaults.imageModel` فقط زمانی استفاده می‌شود که مدل اصلی نتواند تصویر بپذیرد.
- `agents.defaults.pdfModel` توسط ابزار `pdf` استفاده می‌شود. اگر تنظیم نشده باشد، ابزار ابتدا به `imageModel` و سپس به مدل حل‌شده نشست/پیش‌فرض برمی‌گردد.
- `agents.defaults.imageGenerationModel`، `musicGenerationModel` و `videoGenerationModel` پشتوانه ابزارهای مشترک تولید رسانه هستند. اگر تنظیم نشده باشند، هر ابزار یک پیش‌فرض ارائه‌دهنده دارای پشتوانه احراز هویت را استنتاج می‌کند: ابتدا ارائه‌دهنده پیش‌فرض فعلی، سپس سایر ارائه‌دهندگان ثبت‌شده برای آن قابلیت به‌ترتیب شناسه ارائه‌دهنده. برای غیرفعال‌کردن این استنتاج میان‌ارائه‌دهنده‌ای در حالی که جایگزین‌های صریح حفظ می‌شوند، `agents.defaults.mediaGenerationAutoProviderFallback: false` را تنظیم کنید.
- `agents.list[].model` مختص هر عامل (به‌همراه اتصال‌ها) مقدار `agents.defaults.model` را بازنویسی می‌کند — [مسیریابی چندعاملی](/fa/concepts/multi-agent) را ببینید.

مرجع کامل کلیدها، پیش‌فرض‌ها و نمونه‌های JSON5: [مرجع پیکربندی](/fa/gateway/config-agents#agent-defaults).

## منبع انتخاب و سخت‌گیری مدل جایگزین

یک `provider/model` یکسان بسته به منشأ آن رفتار متفاوتی دارد:

| منبع                                                                  | رفتار                                                                                                                                                                                                                                                       |
| ----------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| پیش‌فرض پیکربندی‌شده (`agents.defaults.model.primary`، مدل اصلی مختص هر عامل) | نقطه شروع عادی؛ از `agents.defaults.model.fallbacks` استفاده می‌کند.                                                                                                                                                                                                 |
| جایگزین خودکار                                                           | وضعیت بازیابی موقت که به‌صورت `modelOverrideSource: "auto"` ذخیره می‌شود. OpenClaw به‌طور دوره‌ای مدل اصلی اولیه را دوباره می‌آزماید، پس از بازیابی انتخاب خودکار را پاک می‌کند و گذارهای جایگزینی/بازیابی را برای هر تغییر وضعیت یک‌بار اعلام می‌کند.                              |
| انتخاب نشست کاربر                                                  | دقیق و سخت‌گیرانه. `/model`، انتخابگر مدل، `session_status(model=...)` و `sessions.patch` مقدار `modelOverrideSource: "user"` را ذخیره می‌کنند. اگر آن ارائه‌دهنده/مدل دسترس‌ناپذیر شود، اجرا به‌شکلی مشهود شکست می‌خورد و به مدل پیکربندی‌شده دیگری منتقل نمی‌شود. |
| Cron `--model` / محموله `model`                                        | مدل اصلی مختص هر کار. همچنان از جایگزین‌های پیکربندی‌شده استفاده می‌کند، مگر اینکه کار مقدار `fallbacks` خود را در محموله ارائه کند (`fallbacks: []` اجرای سخت‌گیرانه را تحمیل می‌کند).                                                                                                                    |

سایر قواعد انتخاب:

- تغییر `agents.defaults.model.primary` پین‌های نشست موجود را بازنویسی نمی‌کند. اگر وضعیت `This session is pinned to X; config primary Y will apply to new/unpinned sessions.` را گزارش کرد، برای پاک‌کردن پین `/model default` را اجرا کنید.
- انتخابگرهای مدل پیش‌فرض و فهرست مجاز CLI با فهرست‌کردن فقط `models.providers.*.models` به‌جای کل کاتالوگ داخلی، به `models.mode: "replace"` احترام می‌گذارند.
- انتخابگر مدل در رابط کنترل، نمای مدل پیکربندی‌شده را از Gateway درخواست می‌کند: در صورت تنظیم‌بودن، `agents.defaults.models` (از جمله ورودی‌های عام `provider/*`)؛ در غیر این صورت، `models.providers.*.models` به‌همراه ارائه‌دهندگانی که احراز هویت قابل‌استفاده دارند. کاتالوگ کامل داخلی فقط برای نماهای مرور صریح رزرو شده است (`models.list` با `view: "all"` یا `openclaw models list --all`).
- رابط‌های موجودی ارائه‌دهندگان از `models.list` با `view: "provider-config"` استفاده می‌کنند تا ردیف‌های `models.providers.*.models` تعریف‌شده در منبع را بدون اعمال فهرست‌های مجاز انتخابگر نمایش دهند.

جزئیات کامل سازوکار: [جابه‌جایی اضطراری مدل](/fa/concepts/model-failover).

## سیاست سریع مدل

- مدل اصلی را روی قوی‌ترین مدل نسل جدیدی که در دسترس دارید تنظیم کنید.
- برای وظایف حساس به هزینه/تأخیر و گفت‌وگوهای کم‌اهمیت‌تر از مدل‌های جایگزین استفاده کنید.
- برای عامل‌های مجهز به ابزار یا ورودی‌های غیرقابل‌اعتماد، از رده‌های قدیمی‌تر/ضعیف‌تر مدل اجتناب کنید.

## راه‌اندازی اولیه

```bash
openclaw onboard
```

مدل و احراز هویت ارائه‌دهندگان رایج را بدون ویرایش دستی پیکربندی راه‌اندازی می‌کند، از جمله OAuth اشتراک OpenAI Codex و Anthropic (کلید API یا استفاده مجدد از Claude CLI).

وقتی هیچ مدل اصلی پیکربندی نشده باشد، راه‌اندازی جدید با کلید API در OpenAI
مقدار `openai/gpt-5.6` را انتخاب می‌کند؛ شناسه ساده API مستقیم به رده Sol حل می‌شود. راه‌اندازی جدید
OAuth در ChatGPT/Codex ارجاع دقیق کاتالوگ `openai/gpt-5.6-sol` را انتخاب می‌کند.
احراز هویت مجدد مدل اصلی صریح موجود، از جمله
`openai/gpt-5.5`، را حفظ می‌کند. اگر GPT-5.6 برای حساب در دسترس نیست،
`openai/gpt-5.5` را صریحاً انتخاب کنید؛ OpenClaw آن را بی‌سروصدا تنزل نمی‌دهد.

## «مدل مجاز نیست» (و دلیل توقف پاسخ‌ها)

اگر `agents.defaults.models` تنظیم شده باشد، به فهرست مجاز برای `/model` و بازنویسی‌های نشست تبدیل می‌شود. انتخاب مدلی خارج از آن فهرست مجاز، پیش از تولید هر پاسخ عادی، این نتیجه را برمی‌گرداند:

```text
مدل "provider/model" مجاز نیست. برای فهرست‌کردن ارائه‌دهندگان از /models، یا برای فهرست‌کردن مدل‌ها از /models <provider> استفاده کنید.
آن را با این دستور اضافه کنید: openclaw config set agents.defaults.models '{"provider/model":{}}' --strict-json --merge
```

برای رفع مشکل، مدل را به `agents.defaults.models` اضافه کنید، فهرست مجاز را کاملاً پاک کنید (کلید را حذف کنید) یا مدلی را از `/model list` انتخاب کنید. اگر دستور ردشده شامل بازنویسی زمان‌اجرا مانند `/model openai/gpt-5.5 --runtime codex` بود، ابتدا فهرست مجاز را اصلاح کنید و سپس همان دستور `/model ... --runtime ...` را دوباره امتحان کنید.

برای مدل‌های محلی/GGUF، فهرست مجاز به ارجاع کامل دارای پیشوند ارائه‌دهنده نیاز دارد؛ برای مثال `ollama/gemma4:26b` یا `lmstudio/Gemma4-26b-a4-it-gguf` — رشته دقیق را در `openclaw models list --provider <provider>` بررسی کنید. وقتی فهرست مجاز فعال است، نام فایل ساده یا نام نمایشی کافی نیست.

برای محدودکردن ارائه‌دهندگان بدون فهرست‌کردن همه مدل‌ها، از ورودی‌های عام `provider/*` استفاده کنید:

```json5
{
  agents: {
    defaults: {
      models: {
        "openai/*": {},
        "vllm/*": {},
      },
    },
  },
}
```

سپس `/model`، `/models` و انتخابگرهای مدل فقط کاتالوگ کشف‌شده همان ارائه‌دهندگان را نمایش می‌دهند و مدل‌های جدید می‌توانند بدون ویرایش فهرست مجاز ظاهر شوند. برای افزودن یک مدل خاص از ارائه‌دهنده‌ای دیگر، ورودی‌های دقیق `provider/model` را با ورودی‌های `provider/*` ترکیب کنید.

نمونه فهرست مجاز با نام‌های مستعار:

```json5
{
  agents: {
    defaults: {
      model: { primary: "anthropic/claude-sonnet-4-6" },
      models: {
        "anthropic/claude-sonnet-4-6": { alias: "Sonnet" },
        "anthropic/claude-opus-4-6": { alias: "Opus" },
      },
    },
  },
}
```

<Accordion title="ویرایش ایمن فهرست مجاز از CLI">
برای تغییرات افزایشی از `--merge` استفاده کنید:

```bash
openclaw config set agents.defaults.models '{"openai/gpt-5.4":{}}' --strict-json --merge
```

`openclaw config set` انتساب اشیای ساده به `agents.defaults.models`، `models.providers` یا `models.providers.<id>.models` را هنگامی که باعث حذف ورودی‌های موجود شوند، رد می‌کند؛ فقط زمانی از `--replace` استفاده کنید که مقدار جدید باید به مقدار کامل مقصد تبدیل شود. راه‌اندازی تعاملی ارائه‌دهنده و `openclaw configure --section model` از قبل انتخاب‌های مختص ارائه‌دهنده را در فهرست مجاز ادغام می‌کنند، بنابراین افزودن یک ارائه‌دهنده ورودی‌های نامرتبط را حذف نمی‌کند؛ پیکربندی مقدار موجود `agents.defaults.model.primary` را حفظ می‌کند. دستورهای صریحی مانند `openclaw models auth login --provider <id> --set-default` و `openclaw models set <model>` همچنان مدل اصلی را جایگزین می‌کنند.
</Accordion>

## `/model` در گفت‌وگو

```text
/model
/model list
/model 3
/model openai/gpt-5.4
/model default
/model status
```

- `/model` و `/model list` یک انتخاب‌گر شماره‌دار فشرده (خانواده مدل + ارائه‌دهندگان موجود) را نمایش می‌دهند؛ `/model <#>` از آن انتخاب می‌کند. در Discord، این کار فهرست‌های کشویی ارائه‌دهنده/مدل را با یک مرحله Submit باز می‌کند؛ در Telegram، انتخاب‌های انتخاب‌گر محدود به نشست هستند و هرگز پیش‌فرض پایدار عامل را در `openclaw.json` بازنویسی نمی‌کنند. `/models add` منسوخ شده است و به‌جای ثبت مدل‌ها از طریق چت، پیامی برمی‌گرداند.
- `/model` انتخاب جدید نشست را بلافاصله ذخیره می‌کند. اگر عامل بیکار باشد، اجرای بعدی فوراً از آن استفاده می‌کند؛ اگر اجرایی از قبل فعال باشد، تغییر برای نقطه بعدی تلاش مجدد پاک در صف قرار می‌گیرد (یا نقطه‌ای بعدتر، اگر فعالیت ابزار یا خروجی پاسخ از قبل آغاز شده باشد).
- `/model default` انتخاب نشست را پاک می‌کند تا دوباره پیش‌فرض اصلی پیکربندی‌شده را به ارث ببرد.
- ارجاع `/model` انتخاب‌شده توسط کاربر برای آن نشست سخت‌گیرانه است: اگر دسترس‌ناپذیر شود، پاسخ به‌طور مشهود شکست می‌خورد، نه اینکه بی‌سروصدا از طریق `agents.defaults.model.fallbacks` به گزینه‌های جایگزین برود. پیش‌فرض‌های پیکربندی‌شده و مدل‌های اصلی کارهای Cron همچنان از زنجیره‌های جایگزین استفاده می‌کنند.
- `/model status` نمای تفصیلی است: نامزدهای احراز هویت برای هر ارائه‌دهنده و (در صورت پیکربندی) نقطه پایانی ارائه‌دهنده، یعنی `baseUrl`، به‌همراه حالت `api`.
- ارجاع‌های مدل با جداسازی در نخستین `/` تجزیه می‌شوند؛ `provider/model` را وارد کنید. اگر شناسه مدل خودش شامل `/` است (به سبک OpenRouter)، پیشوند ارائه‌دهنده را وارد کنید؛ برای نمونه، `/model openrouter/moonshotai/kimi-k2`. اگر ارائه‌دهنده را حذف کنید، OpenClaw این موارد را امتحان می‌کند: (1) تطبیق نام مستعار، (2) تطبیق یکتای ارائه‌دهنده پیکربندی‌شده برای دقیقاً همان شناسه مدل بدون پیشوند، (3) ارائه‌دهنده پیش‌فرض پیکربندی‌شده (گزینه جایگزین منسوخ‌شده) — و اگر آن ارائه‌دهنده دیگر مدل پیش‌فرض پیکربندی‌شده را عرضه نکند، به‌جای آن نخستین ارائه‌دهنده/مدل پیکربندی‌شده را امتحان می‌کند تا پیش‌فرض منسوخ و مربوط به ارائه‌دهنده حذف‌شده نمایش داده نشود.
- ارجاع‌های مدل به حروف کوچک عادی‌سازی می‌شوند؛ شناسه‌های ارائه‌دهنده در سایر موارد دقیق هستند، بنابراین از شناسه اعلام‌شده توسط Plugin استفاده کنید.

رفتار کامل فرمان و پیکربندی: [فرمان‌های اسلش](/fa/tools/slash-commands).

## CLI

```bash
openclaw models status
openclaw models list
openclaw models set <provider/model>
openclaw models set-image <provider/model>
openclaw models scan
openclaw models aliases list|add|remove
openclaw models fallbacks list|add|remove|clear
openclaw models image-fallbacks list|add|remove|clear
openclaw models auth list|add|login|paste-api-key|paste-token|setup-token|order
```

`openclaw models` بدون زیرفرمان، میان‌بری برای `models status` است که انقضای OAuth را نیز برای پروفایل‌های مخزن احراز هویت نمایش می‌دهد (به‌طور پیش‌فرض در فاصله کمتر از 24h هشدار می‌دهد). پرچم‌های کامل، ساختارهای JSON و زیرفرمان‌های پروفایل احراز هویت: [مرجع CLI مدل‌ها](/fa/cli/models).

<AccordionGroup>
  <Accordion title="اسکن (مدل‌های رایگان OpenRouter)">
    `openclaw models scan` کاتالوگ عمومی مدل‌های رایگان OpenRouter را بررسی می‌کند و می‌تواند نامزدها را به‌صورت زنده از نظر پشتیبانی از ابزار و تصویر آزمایش کند. خود کاتالوگ عمومی است، بنابراین اسکن‌های صرفاً فراداده‌ای (`--no-probe`) به کلید نیاز ندارند؛ آزمایش زنده و `--set-default`/`--set-image` به یک کلید API مربوط به OpenRouter (پروفایل احراز هویت یا `OPENROUTER_API_KEY`) نیاز دارند و بدون آن، به‌صورت بسته و تنها با خروجی فراداده‌ای شکست می‌خورند.

    نتایج بر این اساس رتبه‌بندی می‌شوند: ابتدا پشتیبانی از تصویر، سپس تأخیر ابزار، سپس اندازه زمینه و در پایان تعداد پارامترها. در یک TTY، نتایج آزمایش‌شده یک انتخاب تعاملی گزینه جایگزین را درخواست می‌کنند؛ حالت غیرتعاملی برای پذیرش پیش‌فرض‌ها به `--yes` نیاز دارد.

## رجیستری مدل‌ها (`models.json`)

ارائه‌دهندگان سفارشی پیکربندی‌شده زیر `models.providers` در `models.json` واقع در دایرکتوری عامل (پیش‌فرض: `~/.openclaw/agents/<agentId>/agent/models.json`) نوشته می‌شوند. کاتالوگ‌های Plugin ارائه‌دهنده جداگانه و به‌صورت قطعه‌های کاتالوگ تولیدشده و متعلق به Plugin ذخیره می‌شوند و به‌طور خودکار بارگذاری می‌شوند. این فایل به‌طور پیش‌فرض با پیکربندی ادغام می‌شود؛ برای استفاده صرفاً از ارائه‌دهندگان پیکربندی‌شده خود، `models.mode: "replace"` را تنظیم کنید.

<AccordionGroup>
  <Accordion title="اولویت حالت ادغام">
    برای شناسه‌های ارائه‌دهنده منطبق:

    - یک `baseUrl` غیرخالی که از قبل در `models.json` عامل وجود دارد، اولویت دارد.
    - یک `apiKey` غیرخالی در `models.json` فقط زمانی اولویت دارد که آن ارائه‌دهنده در زمینه فعلی پیکربندی/پروفایل احراز هویت تحت مدیریت SecretRef نباشد.
    - مقادیر `apiKey` تحت مدیریت SecretRef، به‌جای ذخیره اسرار حل‌شده، از نشانگرهای منبع تازه‌سازی می‌شوند: نام متغیر محیطی برای ارجاعات محیطی و `secretref-managed` برای ارجاعات فایل/اجرا.
    - مقادیر سربرگ تحت مدیریت SecretRef نیز به همین روش و با استفاده از `secretref-env:ENV_VAR_NAME` برای ارجاعات محیطی تازه‌سازی می‌شوند.
    - `apiKey`/`baseUrl` خالی یا ناموجود در `models.json` به `models.providers` پیکربندی برمی‌گردند.
    - سایر فیلدهای ارائه‌دهنده از پیکربندی و داده‌های عادی‌سازی‌شده کاتالوگ تازه‌سازی می‌شوند.

  </Accordion>
</AccordionGroup>

ماندگاری نشانگرها بر مرجعیت منبع استوار است: هرگاه OpenClaw فایل `models.json` را دوباره تولید کند — از جمله در مسیرهای هدایت‌شده با فرمان مانند `openclaw agent` — نشانگرها را از تصویر لحظه‌ای پیکربندی منبع فعال (پیش از حل‌وفصل) می‌نویسد، نه از مقادیر اسرار حل‌شده در زمان اجرا.
