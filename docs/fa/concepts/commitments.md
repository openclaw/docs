تعهدات، حافظه‌های پیگیری کوتاه‌مدت هستند. وقتی فعال باشند، OpenClaw می‌تواند
متوجه شود که یک گفتگو فرصتی برای پیگیری در آینده ایجاد کرده است و به خاطر بسپارد
که بعدا آن را دوباره مطرح کند.

نمونه‌ها:

- شما به مصاحبه‌ای در فردا اشاره می‌کنید. OpenClaw ممکن است بعد از آن پیگیری کند.
- می‌گویید که خسته‌اید. OpenClaw ممکن است بعدا بپرسد آیا خوابیده‌اید یا نه.
- عامل می‌گوید پس از تغییر چیزی پیگیری خواهد کرد. OpenClaw ممکن است آن چرخه باز را
  دنبال کند.

تعهدات مانند `MEMORY.md` واقعیت‌های پایدار نیستند و یادآورهای دقیق هم نیستند.
آن‌ها بین حافظه و خودکارسازی قرار می‌گیرند: OpenClaw یک تعهد وابسته به گفتگو را
به خاطر می‌سپارد، سپس Heartbeat آن را وقتی موعدش برسد تحویل می‌دهد.

## فعال‌سازی تعهدات

تعهدات به‌طور پیش‌فرض خاموش هستند. آن‌ها را در پیکربندی فعال کنید:

```bash
openclaw config set commitments.enabled true
openclaw config set commitments.maxPerDay 3
```

معادل `openclaw.json`:

```json
{
  "commitments": {
    "enabled": true,
    "maxPerDay": 3
  }
}
```

`commitments.maxPerDay` تعداد پیگیری‌های استنباط‌شده‌ای را که می‌توان در هر جلسه
عامل در یک روز چرخشی تحویل داد محدود می‌کند. مقدار پیش‌فرض `3` است.

## سازوکار

پس از پاسخ عامل، OpenClaw ممکن است در یک زمینه جداگانه یک گذر استخراج پس‌زمینه
پنهان اجرا کند. این گذر فقط به دنبال تعهدات پیگیری استنباط‌شده می‌گردد. در گفتگوی
قابل مشاهده چیزی نمی‌نویسد و از عامل اصلی نمی‌خواهد درباره استخراج استدلال کند.

وقتی یک نامزد با اطمینان بالا پیدا کند، OpenClaw تعهدی را با موارد زیر ذخیره می‌کند:

- شناسه عامل
- کلید جلسه
- کانال اصلی و مقصد تحویل
- بازه موعد
- یک پیگیری کوتاه پیشنهادی
- زمینه منبع کافی تا Heartbeat تصمیم بگیرد آیا آن را ارسال کند یا نه

تحویل از طریق Heartbeat انجام می‌شود. وقتی موعد یک تعهد برسد، Heartbeat تعهد را
به نوبت Heartbeat برای همان عامل و محدوده کانال اضافه می‌کند. مدل می‌تواند یک
پیگیری طبیعی ارسال کند یا برای رد کردن آن `HEARTBEAT_OK` پاسخ دهد.

OpenClaw هرگز یک تعهد استنباط‌شده را بلافاصله پس از نوشتن آن تحویل نمی‌دهد.
زمان موعد حداقل به یک فاصله Heartbeat پس از ایجاد تعهد محدود می‌شود، بنابراین
پیگیری نمی‌تواند در همان لحظه‌ای که استنباط شده است بازتاب داده شود.

## محدوده

تعهدات به زمینه دقیق عامل و کانالی محدود می‌شوند که در آن ایجاد شده‌اند. پیگیری‌ای
که هنگام گفتگو با یک عامل در Discord استنباط شده است، توسط عامل دیگر، کانال دیگر،
یا جلسه‌ای نامرتبط تحویل داده نمی‌شود.

این محدوده بخشی از قابلیت است. پیگیری‌های طبیعی باید مانند ادامه همان گفتگو به نظر
برسند، نه مانند یک سامانه یادآور سراسری.

به‌وضوح یک پیگیری مفید در آینده ایجاد کرده است.

## مدیریت تعهدات

برای بررسی و پاک کردن تعهدات ذخیره‌شده از CLI استفاده کنید:

```bash
openclaw commitments
openclaw commitments --all
openclaw commitments --agent main
openclaw commitments --status snoozed
openclaw commitments dismiss cm_abc123
```

برای مرجع فرمان، [`openclaw commitments`](/fa/cli/commitments) را ببینید.

## حریم خصوصی و هزینه

استخراج تعهد از یک گذر LLM استفاده می‌کند، بنابراین فعال‌سازی آن پس از نوبت‌های
واجد شرایط، مصرف مدل در پس‌زمینه را اضافه می‌کند. این گذر از گفتگوی قابل مشاهده
برای کاربر پنهان است، اما می‌تواند تبادل اخیر لازم را بخواند تا تصمیم بگیرد آیا
پیگیری‌ای وجود دارد یا نه.

تعهدات ذخیره‌شده وضعیت محلی OpenClaw هستند. آن‌ها حافظه عملیاتی‌اند، نه حافظه
بلندمدت. این قابلیت را با فرمان زیر غیرفعال کنید:

```bash
openclaw config set commitments.enabled false
```

## عیب‌یابی

اگر پیگیری‌های مورد انتظار ظاهر نمی‌شوند:

- تأیید کنید که `commitments.enabled` برابر `true` است.
- برای رکوردهای در انتظار، ردشده، به تعویق‌افتاده یا منقضی‌شده، `openclaw commitments --all` را بررسی کنید.
- مطمئن شوید Heartbeat برای عامل در حال اجرا است.
- بررسی کنید آیا `commitments.maxPerDay` برای آن جلسه عامل از قبل به حد خود رسیده است یا نه.
- به یاد داشته باشید که یادآورهای دقیق توسط استخراج تعهد نادیده گرفته می‌شوند و
  باید در عوض زیر [وظایف زمان‌بندی‌شده](/fa/automation/cron-jobs) ظاهر شوند.
