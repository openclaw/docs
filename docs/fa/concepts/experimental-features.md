ویژگی‌های آزمایشی، قابلیت‌های پیش‌نمایشی اختیاری هستند که پشت پرچم‌های صریح قرار دارند. پیش از آن‌که به یک پیش‌فرض پایدار یا قراردادی بلندمدت تبدیل شوند، باید بیشتر در شرایط واقعی آزموده شوند.

- به‌طور پیش‌فرض خاموش هستند، مگر آن‌که مستندی فعال‌کردن یکی از آن‌ها را توصیه کند.
- ساختار و رفتار آن‌ها می‌تواند سریع‌تر از پیکربندی پایدار تغییر کند.
- اگر از قبل مسیر پایداری وجود دارد، آن را ترجیح دهید.
- گسترش وسیع را تنها پس از آزمایش اولیه در محیطی کوچک‌تر انجام دهید.

## حالت سبک مدل محلی

تنظیم `agents.defaults.experimental.localModelLean: true` در هر نوبت، ابزارهای اختیاری سنگین را از سطح مستقیم عامل حذف می‌کند: `browser`، `cron`، `message`، `image_generate`، `music_generate`، `video_generate`، `tts` و `pdf`. ابزارهایی که صراحتاً مجاز شده‌اند یا برای تحویل ضروری‌اند همچنان در دسترس می‌مانند، هرچند «جست‌وجوی ابزار» ممکن است به‌جای نمایش مستقیم، آن‌ها را فهرست کند. همچنین اگر `tools.toolSearch` از قبل تنظیم نشده باشد، حالت سبک به‌طور پیش‌فرض فهرست‌های Plugin/MCP/کلاینت را روی «جست‌وجوی ابزار» ساختاریافته (`tool_search`، `tool_describe`، `tool_call`) تنظیم می‌کند. برای محدودکردن این تنظیم به یک عامل، از `agents.list[].experimental.localModelLean` استفاده کنید.

اگر «جست‌وجوی ابزار» را از قبل به‌صورت سراسری تنظیم کرده‌اید، OpenClaw آن پیکربندی را بدون تغییر نگه می‌دارد. برای انصراف از پیش‌فرض «جست‌وجوی ابزار» در حالت سبک، `tools.toolSearch: false` را تنظیم کنید.

در حالت ساختاریافته `tools`، اجراهای سبک، `exec` را در کنار کنترل‌های «جست‌وجوی ابزار» مستقیماً قابل مشاهده نگه می‌دارند تا مدل‌های محلی تنظیم‌شده برای کدنویسی همچنان بتوانند مسیر آشنای پوسته را انتخاب کنند. این فقط میزان دیده‌شدن شِما را تغییر می‌دهد: خط‌مشی عادی ابزار، محیط ایزوله و تأییدهای اجرای `exec` همچنان اعمال می‌شوند. حالت‌های صریح `code` و `directory` رفتار عادی Compaction خود را حفظ می‌کنند.

### چرا این ابزارها

این ابزارها طولانی‌ترین توضیحات، گسترده‌ترین ساختار پارامترها یا بیشترین احتمال منحرف‌کردن یک مدل کوچک از مسیر عادی کدنویسی و گفت‌وگو را دارند. در یک بک‌اند سازگار با OpenAI که زمینه کوچکی دارد یا سخت‌گیرتر است، این تفاوت میان موارد زیر است:

- جاگرفتن شِماهای ابزار در پرامپت، در برابر بیرون‌راندن تاریخچه گفت‌وگو.
- انتخاب ابزار درست توسط مدل، در برابر تولید فراخوانی‌های ناقص ابزار به‌دلیل وجود شِماهای مشابه بیش‌ازحد.
- باقی‌ماندن آداپتور Chat Completions در محدودیت‌های خروجی ساختاریافته، در برابر دریافت خطای 400 به‌علت اندازه محموله فراخوانی ابزار.

حذف آن‌ها فقط فهرست مستقیم ابزارها را کوتاه می‌کند. مدل همچنان به `read`، `write`، `edit`، `exec`، `apply_patch`، درک تصویر، جست‌وجو/دریافت از وب (در صورت پیکربندی)، حافظه و ابزارهای نشست/عامل دسترسی دارد. فهرست‌های اضافی از طریق «جست‌وجوی ابزار» قابل دسترسی می‌مانند، مگر آن‌که `tools.toolSearch: false` را تنظیم کنید؛ مجوزهای صریح ابزار می‌توانند عامل سبک را دوباره وارد یک گردش‌کار محدودشده کنند.

### چه زمانی آن را روشن کنید

حالت سبک را زمانی فعال کنید که ثابت کرده‌اید مدل می‌تواند با Gateway ارتباط برقرار کند، اما نوبت‌های کامل عامل درست عمل نمی‌کنند:

1. اجرای `openclaw infer model run --gateway --model <ref> --prompt "Reply with exactly: pong"` موفق است.
2. یک نوبت عادی عامل به‌دلیل فراخوانی‌های ناقص ابزار، پرامپت‌های بیش‌ازحد بزرگ یا نادیده‌گرفتن ابزارها توسط مدل شکست می‌خورد.
3. تغییر `localModelLean: true` مشکل را برطرف می‌کند.

### چه زمانی آن را خاموش نگه دارید

اگر بک‌اند شما زمان‌اجرای کامل پیش‌فرض را بدون مشکل مدیریت می‌کند، این گزینه را خاموش نگه دارید. این یک راهکار موقت برای پشته‌های محلی است که به سطح ابزار کوچک‌تری نیاز دارند، نه یک پیش‌فرض برای مدل‌های میزبانی‌شده یا سامانه‌های محلی برخوردار از منابع کافی.

حالت سبک جایگزین `tools.profile`، `tools.allow`/`tools.deny` یا راه گریز `compat.supportsTools: false` مدل نمی‌شود. برای محدودسازی دائمی سطح ابزار یک عامل مشخص، این کنترل‌های پایدار را ترجیح دهید.

### فعال‌سازی

```json5
{
  agents: {
    defaults: {
      experimental: {
        localModelLean: true,
      },
    },
  },
}
```

فقط برای یک عامل:

```json5
{
  agents: {
    list: [
      {
        id: "local",
        model: "lmstudio/gemma-4-e4b-it",
        experimental: {
          localModelLean: true,
        },
      },
    ],
  },
}
```

پس از تغییر پرچم، Gateway را راه‌اندازی مجدد کنید. پالایش حالت سبک، `browser`، `cron`، `message`، `image_generate`، `music_generate`، `video_generate`، `tts` و `pdf` را حذف می‌کند، مگر آن‌که با `tools.allow` یا `tools.alsoAllow` صراحتاً آن‌ها را حفظ کنید؛ «جست‌وجوی ابزار» ممکن است همچنان ابزارهای حفظ‌شده را به‌جای نمایش مستقیم، فهرست کند.

## آزمایشی به‌معنای پنهان نیست

آزمایشی‌بودن یک ویژگی باید هم در مستندات و هم در خود مسیر پیکربندی به‌روشنی بیان شود، نه آن‌که پشت یک کنترل پیش‌فرض با ظاهری پایدار پنهان شود.
