ویژگی‌های آزمایشی در OpenClaw **سطح‌های پیش‌نمایش اختیاری** هستند. آن‌ها پشت پرچم‌های صریح قرار دارند چون هنوز پیش از آن‌که شایسته‌ی پیش‌فرض پایدار یا قرارداد عمومی بلندمدت باشند، به آزمون واقعی بیشتری نیاز دارند.

با آن‌ها متفاوت از پیکربندی عادی برخورد کنید:

- آن‌ها را **به‌صورت پیش‌فرض خاموش** نگه دارید مگر این‌که سند مرتبط از شما بخواهد یکی را امتحان کنید.
- انتظار داشته باشید **شکل و رفتار** سریع‌تر از پیکربندی پایدار تغییر کند.
- وقتی مسیر پایدار از قبل وجود دارد، ابتدا همان را ترجیح دهید.
- اگر OpenClaw را به‌صورت گسترده عرضه می‌کنید، پیش از وارد کردن پرچم‌های آزمایشی به یک مبنای مشترک، آن‌ها را در محیطی کوچک‌تر آزمایش کنید.

## حالت سبک مدل محلی

`agents.defaults.experimental.localModelLean: true` یک شیر کاهش فشار برای راه‌اندازی‌های مدل محلی ضعیف‌تر است. وقتی روشن باشد، OpenClaw سه ابزار پیش‌فرض — `browser`، `cron`، و `message` — را در هر نوبت از سطح ابزار عامل حذف می‌کند. همچنین وقتی `tools.toolSearch` صراحتاً پیکربندی نشده باشد، آن اجرا را به‌صورت پیش‌فرض روی کنترل‌های ساختاریافته‌ی Tool Search می‌گذارد، تا کاتالوگ‌های بزرگ‌تر Plugin، MCP، یا ابزارهای کلاینت به‌جای ریخته شدن در پرامپت، پشت `tool_search`، `tool_describe`، و `tool_call` باقی بمانند. اجراهایی که به تحویل مستقیم `message` نیاز دارند، به‌جای فعال کردن پیش‌فرض Tool Search در حالت سبک، همان ابزار را مستقیم نگه می‌دارند. برای فعال یا غیرفعال کردن همین رفتار برای یک عامل پیکربندی‌شده، از `agents.list[].experimental.localModelLean` استفاده کنید.

### چرا این سه ابزار

این سه ابزار بزرگ‌ترین توضیحات و بیشترین شکل‌های پارامتر را در زمان‌اجرای پیش‌فرض OpenClaw دارند. در یک بک‌اند OpenAI-سازگار با زمینه‌ی کوچک یا سخت‌گیرتر، همین تفاوت میان این‌هاست:

- جا شدن تمیز شِماهای ابزار در پرامپت در برابر بیرون راندن تاریخچه‌ی گفت‌وگو.
- انتخاب ابزار درست توسط مدل در برابر تولید فراخوانی‌های ابزار بدشکل چون شِماهای شبیه به هم بیش از حد زیاد هستند.
- باقی ماندن آداپتور Chat Completions در محدوده‌های خروجی ساختاریافته‌ی سرور در برابر برخورد با خطای 400 به‌دلیل اندازه‌ی بار فراخوانی ابزار.

حذف آن‌ها OpenClaw را بی‌صدا دوباره سیم‌کشی نمی‌کند — فقط فهرست ابزار مستقیم را کوتاه‌تر می‌کند. مدل همچنان `read`، `write`، `edit`، `exec`، `apply_patch`، جست‌وجو/واکشی وب (وقتی پیکربندی شده باشد)، حافظه، و ابزارهای نشست/عامل را در دسترس دارد. کاتالوگ‌های اضافی همچنان از طریق Tool Search قابل فراخوانی می‌مانند مگر این‌که صراحتاً `tools.toolSearch: false` را تنظیم کنید.

### چه زمانی آن را روشن کنید

وقتی از قبل ثابت کرده‌اید مدل می‌تواند با Gateway صحبت کند اما نوبت‌های کامل عامل درست رفتار نمی‌کنند، حالت سبک را فعال کنید. زنجیره‌ی نشانه‌ی معمول این است:

1. `openclaw infer model run --gateway --model <ref> --prompt "Reply with exactly: pong"` موفق می‌شود.
2. یک نوبت عادی عامل با فراخوانی‌های ابزار بدشکل، پرامپت‌های بیش‌ازحد بزرگ، یا نادیده گرفتن ابزارها توسط مدل شکست می‌خورد.
3. تغییر `localModelLean: true` شکست را برطرف می‌کند.

### چه زمانی آن را خاموش بگذارید

اگر بک‌اند شما زمان‌اجرای پیش‌فرض کامل را تمیز مدیریت می‌کند، این گزینه را خاموش بگذارید. حالت سبک یک راه‌حل موقت است، نه یک پیش‌فرض. این حالت وجود دارد چون برخی پشته‌های محلی برای رفتار درست به سطح ابزار کوچک‌تری نیاز دارند؛ مدل‌های میزبانی‌شده و سیستم‌های محلی با منابع کافی نیازی به آن ندارند.

حالت سبک همچنین جایگزین `tools.profile`، `tools.allow`/`tools.deny`، یا راه فرار `compat.supportsTools: false` مدل نمی‌شود. اگر برای یک عامل مشخص به سطح ابزار باریک‌تر دائمی نیاز دارید، آن پیچ‌های پایدار را به پرچم آزمایشی ترجیح دهید.

اگر از قبل Tool Search را به‌صورت سراسری تنظیم می‌کنید، OpenClaw آن پیکربندی اپراتور را دست‌نخورده می‌گذارد. برای خروج از پیش‌فرض Tool Search در حالت سبک، `tools.toolSearch: false` را تنظیم کنید.

### فعال‌سازی

```json5
{
  agents: {
    defaults: {
      experimental: {
        localModelLean: true,
      },
    },
  },
}
```

فقط برای یک عامل:

```json5
{
  agents: {
    list: [
      {
        id: "local",
        model: "lmstudio/gemma-4-e4b-it",
        experimental: {
          localModelLean: true,
        },
      },
    ],
  },
}
```

پس از تغییر پرچم، Gateway را بازراه‌اندازی کنید، سپس فهرست ابزار کوتاه‌شده را با این دستور تأیید کنید:

```bash
openclaw status --deep
```

خروجی وضعیت عمیق ابزارهای فعال عامل را فهرست می‌کند؛ وقتی حالت سبک روشن است، `browser`، `cron`، و `message` نباید حضور داشته باشند مگر این‌که حالت تحویل فعلی پاسخ‌های مستقیم `message` را الزامی کند.

## آزمایشی به‌معنای پنهان نیست

اگر یک ویژگی آزمایشی است، OpenClaw باید این را در سندها و در خود مسیر پیکربندی به‌روشنی بگوید. کاری که **نباید** بکند این است که رفتار پیش‌نمایش را در یک پیچ پیش‌فرض با ظاهر پایدار پنهان کند و وانمود کند این عادی است. سطح‌های پیکربندی این‌گونه آشفته می‌شوند.
