موتور داخلی، بک‌اند پیش‌فرض حافظه است. این موتور نمایه حافظه شما را در
یک پایگاه داده SQLite مختص هر عامل ذخیره می‌کند و برای شروع به وابستگی اضافه‌ای نیاز ندارد.

## چه امکاناتی فراهم می‌کند

- **جست‌وجوی کلیدواژه‌ای** از طریق نمایه‌سازی تمام‌متن FTS5 (امتیازدهی BM25).
- **جست‌وجوی برداری** از طریق embeddingها از هر ارائه‌دهنده پشتیبانی‌شده.
- **جست‌وجوی ترکیبی** که هر دو را برای بهترین نتایج ترکیب می‌کند.
- **پشتیبانی از CJK** از طریق توکن‌سازی سه‌تایی برای چینی، ژاپنی و کره‌ای.
- **شتاب‌دهی sqlite-vec** برای پرس‌وجوهای برداری درون پایگاه داده (اختیاری).

## شروع به کار

اگر برای OpenAI، Gemini، Voyage، Mistral، یا DeepInfra کلید API دارید، موتور داخلی
آن را به‌صورت خودکار شناسایی می‌کند و جست‌وجوی برداری را فعال می‌سازد. نیازی به پیکربندی نیست.

برای تنظیم صریح یک ارائه‌دهنده:

```json5
{
  agents: {
    defaults: {
      memorySearch: {
        provider: "openai",
      },
    },
  },
}
```

بدون ارائه‌دهنده embedding، فقط جست‌وجوی کلیدواژه‌ای در دسترس است.

برای اجبار به استفاده از ارائه‌دهنده embedding محلی داخلی، بسته runtime اختیاری
`node-llama-cpp` را کنار OpenClaw نصب کنید، سپس `local.modelPath`
را به یک فایل GGUF اشاره دهید:

```json5
{
  agents: {
    defaults: {
      memorySearch: {
        provider: "local",
        fallback: "none",
        local: {
          modelPath: "~/.node-llama-cpp/models/embeddinggemma-300m-qat-Q8_0.gguf",
        },
      },
    },
  },
}
```

## ارائه‌دهندگان embedding پشتیبانی‌شده

| ارائه‌دهنده | ID          | شناسایی خودکار | یادداشت‌ها                              |
| --------- | ----------- | ------------- | ----------------------------------- |
| OpenAI    | `openai`    | بله           | پیش‌فرض: `text-embedding-3-small`   |
| Gemini    | `gemini`    | بله           | پشتیبانی از چندرسانه‌ای (تصویر + صدا) |
| Voyage    | `voyage`    | بله           |                                     |
| Mistral   | `mistral`   | بله           |                                     |
| DeepInfra | `deepinfra` | بله           | پیش‌فرض: `BAAI/bge-m3`              |
| Ollama    | `ollama`    | خیر           | محلی، صریح تنظیم شود                |
| Local     | `local`     | بله (اول)     | runtime اختیاری `node-llama-cpp`    |

شناسایی خودکار، نخستین ارائه‌دهنده‌ای را انتخاب می‌کند که کلید API آن قابل resolve باشد، به
ترتیب نشان‌داده‌شده. برای بازنویسی، `memorySearch.provider` را تنظیم کنید.

## نمایه‌سازی چگونه کار می‌کند

OpenClaw فایل‌های `MEMORY.md` و `memory/*.md` را به قطعه‌ها (~400 توکن با
هم‌پوشانی 80 توکنی) نمایه‌سازی می‌کند و آن‌ها را در یک پایگاه داده SQLite مختص هر عامل ذخیره می‌کند.

- **محل نمایه:** `~/.openclaw/memory/<agentId>.sqlite`
- **نگه‌داری فضای ذخیره‌سازی:** فایل‌های جانبی SQLite WAL با checkpointهای دوره‌ای و
  هنگام خاموشی محدود می‌شوند.
- **پایش فایل:** تغییرات در فایل‌های حافظه، بازنمایه‌سازی debounce‌شده (1.5s) را فعال می‌کند.
- **بازنمایه‌سازی خودکار:** وقتی ارائه‌دهنده embedding، مدل، یا پیکربندی قطعه‌بندی
  تغییر کند، کل نمایه به‌صورت خودکار از نو ساخته می‌شود.
- **بازنمایه‌سازی بنا به درخواست:** `openclaw memory index --force`

## چه زمانی استفاده شود

موتور داخلی برای بیشتر کاربران انتخاب مناسبی است:

- بدون وابستگی اضافه، آماده به کار است.
- جست‌وجوی کلیدواژه‌ای و برداری را به‌خوبی مدیریت می‌کند.
- از همه ارائه‌دهندگان embedding پشتیبانی می‌کند.
- جست‌وجوی ترکیبی، بهترین بخش‌های هر دو رویکرد بازیابی را ترکیب می‌کند.

اگر به بازرتبه‌بندی، گسترش پرس‌وجو، یا نمایه‌سازی دایرکتوری‌های خارج از فضای کاری نیاز دارید،
استفاده از [QMD](/fa/concepts/memory-qmd) را در نظر بگیرید.

اگر حافظه بین‌نشستی با مدل‌سازی خودکار کاربر می‌خواهید،
[Honcho](/fa/concepts/memory-honcho) را در نظر بگیرید.

## عیب‌یابی

**جست‌وجوی حافظه غیرفعال است؟** `openclaw memory status` را بررسی کنید. اگر هیچ ارائه‌دهنده‌ای
شناسایی نشد، یکی را صریح تنظیم کنید یا یک کلید API اضافه کنید.

**ارائه‌دهنده محلی شناسایی نمی‌شود؟** مطمئن شوید مسیر محلی وجود دارد و اجرا کنید:

```bash
openclaw memory status --deep --agent main
openclaw memory index --force --agent main
```

هم فرمان‌های CLI مستقل و هم Gateway از همان شناسه ارائه‌دهنده `local` استفاده می‌کنند.
اگر ارائه‌دهنده روی `auto` تنظیم شده باشد، embeddingهای محلی فقط زمانی ابتدا در نظر گرفته می‌شوند
که `memorySearch.local.modelPath` به یک فایل محلی موجود اشاره کند.

**نتایج کهنه هستند؟** برای ساخت دوباره، `openclaw memory index --force` را اجرا کنید. watcher
ممکن است در موارد نادر تغییرات را از دست بدهد.

**sqlite-vec بارگذاری نمی‌شود؟** OpenClaw به‌صورت خودکار به شباهت کسینوسی درون‌فرایندی
برمی‌گردد. برای خطای بارگذاری مشخص، گزارش‌ها را بررسی کنید.

## پیکربندی

برای راه‌اندازی ارائه‌دهنده embedding، تنظیم جست‌وجوی ترکیبی (وزن‌ها، MMR، افت زمانی)،
نمایه‌سازی دسته‌ای، حافظه چندرسانه‌ای، sqlite-vec، مسیرهای اضافه، و همه
گزینه‌های پیکربندی دیگر، به
[مرجع پیکربندی حافظه](/fa/reference/memory-config) مراجعه کنید.
