[Honcho](https://honcho.dev) از طریق یک Plugin خارجی، حافظهٔ بومی هوش مصنوعی را به OpenClaw اضافه می‌کند. این سامانه مکالمات را در یک سرویس اختصاصی ماندگار می‌کند و به‌مرور زمان مدل‌هایی از کاربر و عامل می‌سازد؛ در نتیجه، عامل شما به زمینه‌ای بین‌نشستی دست می‌یابد که فراتر از فایل‌های Markdown فضای کاری است.

## امکانات ارائه‌شده

- **حافظهٔ بین‌نشستی** - مکالمات پس از هر نوبت ماندگار می‌شوند؛ بنابراین
  زمینه در بازنشانی نشست‌ها، Compaction و جابه‌جایی بین کانال‌ها حفظ می‌شود.
- **مدل‌سازی کاربر** - Honcho برای هر کاربر نمایه‌ای شامل ترجیحات،
  واقعیت‌ها و سبک ارتباطی و برای عامل نیز نمایه‌ای شامل شخصیت و رفتارهای
  آموخته‌شده نگه می‌دارد.
- **جست‌وجوی معنایی** - امکان جست‌وجو در مشاهدات حاصل از مکالمات گذشته،
  نه فقط نشست جاری.
- **آگاهی چندعاملی** - عامل‌های والد به‌طور خودکار زیرعامل‌های ایجادشده را
  پیگیری می‌کنند و والدها در نشست‌های فرزند به‌عنوان ناظر افزوده می‌شوند.

## ابزارهای موجود

Honcho ابزارهایی ثبت می‌کند که عامل می‌تواند هنگام مکالمه از آن‌ها استفاده کند:

**بازیابی داده‌ها (سریع، بدون فراخوانی LLM):**

| ابزار                       | کاری که انجام می‌دهد                                      |
| --------------------------- | --------------------------------------------------------- |
| `honcho_context`            | بازنمایی کامل کاربر در تمام نشست‌ها                       |
| `honcho_search_conclusions` | جست‌وجوی معنایی در نتیجه‌گیری‌های ذخیره‌شده               |
| `honcho_search_messages`    | یافتن پیام‌ها در نشست‌ها (فیلتر بر اساس فرستنده و تاریخ) |
| `honcho_session`            | تاریخچه و خلاصهٔ نشست جاری                                |

**پرسش و پاسخ (با پشتیبانی LLM):**

| ابزار        | کاری که انجام می‌دهد                                                                           |
| ------------ | ---------------------------------------------------------------------------------------------- |
| `honcho_ask` | پرسش دربارهٔ کاربر؛ `depth='quick'` برای واقعیت‌ها و `'thorough'` برای ترکیب و تحلیل اطلاعات |

## شروع به کار

Plugin را نصب و راه‌اندازی را اجرا کنید:

```bash
openclaw plugins install @honcho-ai/openclaw-honcho
openclaw honcho setup
openclaw gateway --force
```

فرمان راه‌اندازی، اطلاعات اعتبارسنجی API شما را درخواست می‌کند، پیکربندی را
می‌نویسد و در صورت تمایل، فایل‌های حافظهٔ موجود در فضای کاری را منتقل می‌کند.

<Info>
Honcho می‌تواند کاملاً به‌صورت محلی (خودمیزبان) یا از طریق API مدیریت‌شده در
`api.honcho.dev` اجرا شود. برای گزینهٔ خودمیزبان هیچ وابستگی خارجی لازم نیست.
</Info>

## پیکربندی

تنظیمات در `plugins.entries["openclaw-honcho"].config` قرار دارند:

```json5
{
  plugins: {
    entries: {
      "openclaw-honcho": {
        config: {
          apiKey: "your-api-key", // omit for self-hosted
          workspaceId: "openclaw", // memory isolation
          baseUrl: "https://api.honcho.dev",
        },
      },
    },
  },
}
```

برای نمونه‌های خودمیزبان، `baseUrl` را به سرور محلی خود، برای مثال
`http://localhost:8000`، تنظیم و کلید API را حذف کنید.

## انتقال حافظهٔ موجود

اگر فایل‌های حافظهٔ موجود در فضای کاری (`USER.md`، `MEMORY.md`،
`IDENTITY.md`، `memory/`، `canvas/`) را داشته باشید، `openclaw honcho setup`
آن‌ها را شناسایی می‌کند و پیشنهاد انتقالشان را می‌دهد.

<Info>
انتقال غیرمخرب است؛ فایل‌ها در Honcho بارگذاری می‌شوند. فایل‌های اصلی
هرگز حذف یا جابه‌جا نمی‌شوند.
</Info>

## نحوهٔ کار

پس از هر نوبت هوش مصنوعی، مکالمه در Honcho ماندگار می‌شود. پیام‌های کاربر و
عامل هر دو مشاهده می‌شوند تا Honcho بتواند مدل‌های خود را به‌مرور زمان بسازد
و بهبود دهد.

هنگام مکالمه، ابزارهای Honcho در جریان قلاب Plugin با نام
`before_prompt_build` در OpenClaw، سرویس را پرس‌وجو می‌کنند و پیش از آنکه
مدل درخواست را ببیند، زمینهٔ مرتبط را تزریق می‌کنند.

## مقایسهٔ Honcho با حافظهٔ داخلی

|                   | داخلی / QMD                       | Honcho                                |
| ----------------- | --------------------------------- | ------------------------------------- |
| **فضای ذخیره‌سازی** | فایل‌های Markdown فضای کاری       | سرویس اختصاصی (محلی یا میزبانی‌شده)   |
| **بین‌نشستی**     | از طریق فایل‌های حافظه            | خودکار و داخلی                        |
| **مدل‌سازی کاربر** | دستی (نوشتن در MEMORY.md)         | نمایه‌های خودکار                      |
| **جست‌وجو**       | برداری + کلیدواژه‌ای (ترکیبی)     | معنایی بر مبنای مشاهدات              |
| **چندعاملی**      | پیگیری نمی‌شود                    | آگاهی از رابطهٔ والد/فرزند            |
| **وابستگی‌ها**    | هیچ‌کدام (داخلی) یا فایل اجرایی QMD | نصب Plugin                            |

Honcho و سامانهٔ حافظهٔ داخلی می‌توانند با یکدیگر کار کنند. وقتی QMD
پیکربندی شده باشد، ابزارهای دیگری برای جست‌وجوی فایل‌های Markdown محلی در
کنار حافظهٔ بین‌نشستی Honcho در دسترس قرار می‌گیرند.

## فرمان‌های CLI

```bash
openclaw honcho setup                        # Configure API key and migrate files
openclaw honcho status                       # Check connection status
openclaw honcho ask <question>               # Query Honcho about the user
openclaw honcho search <query> [-k N] [-d D] # Semantic search over memory
```
