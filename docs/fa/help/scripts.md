دایرکتوری `scripts/` شامل اسکریپت‌های کمکی برای گردش‌کارهای محلی و کارهای عملیاتی است.
وقتی یک کار به‌وضوح به یک اسکریپت مرتبط است از این‌ها استفاده کنید؛ در غیر این صورت CLI را ترجیح دهید.

## قراردادها

- اسکریپت‌ها **اختیاری** هستند مگر اینکه در مستندات یا چک‌لیست‌های انتشار به آن‌ها اشاره شده باشد.
- وقتی سطح‌های CLI وجود دارند، آن‌ها را ترجیح دهید (مثال: پایش احراز هویت از `openclaw models status --check` استفاده می‌کند).
- فرض کنید اسکریپت‌ها مختص میزبان هستند؛ پیش از اجرا روی یک دستگاه جدید آن‌ها را بخوانید.

## اسکریپت‌های پایش احراز هویت

پایش احراز هویت در [احراز هویت](/fa/gateway/authentication) پوشش داده شده است. اسکریپت‌های زیر `scripts/` گزینه‌های اضافی اختیاری برای گردش‌کارهای تلفن با systemd/Termux هستند.

## کمک‌کنندهٔ خواندن GitHub

وقتی می‌خواهید `gh` برای فراخوانی‌های خواندنی محدود به مخزن از توکن نصب GitHub App استفاده کند، در حالی که `gh` معمولی برای اقدام‌های نوشتنی روی ورود شخصی شما باقی بماند، از `scripts/gh-read` استفاده کنید.

env لازم:

- `OPENCLAW_GH_READ_APP_ID`
- `OPENCLAW_GH_READ_PRIVATE_KEY_FILE`

env اختیاری:

- `OPENCLAW_GH_READ_INSTALLATION_ID` وقتی می‌خواهید جست‌وجوی نصب مبتنی بر مخزن را رد کنید
- `OPENCLAW_GH_READ_PERMISSIONS` به‌عنوان یک جایگزین جداشده با ویرگول برای زیرمجموعهٔ مجوزهای خواندنی که باید درخواست شوند

ترتیب حل مخزن:

- `gh ... -R owner/repo`
- `GH_REPO`
- `git remote origin`

مثال‌ها:

- `scripts/gh-read pr view 123`
- `scripts/gh-read run list -R openclaw/openclaw`
- `scripts/gh-read api repos/openclaw/openclaw/pulls/123`

## هنگام افزودن اسکریپت‌ها

- اسکریپت‌ها را متمرکز و مستند نگه دارید.
- یک ورودی کوتاه در سند مرتبط اضافه کنید (یا اگر وجود ندارد یکی ایجاد کنید).
