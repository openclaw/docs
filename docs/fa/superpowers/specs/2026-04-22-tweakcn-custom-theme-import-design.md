# طراحی وارد کردن تم سفارشی Tweakcn

وضعیت: در ترمینال در 2026-04-22 تأیید شد

## خلاصه

دقیقاً یک جایگاه تم سفارشی محلیِ مرورگر برای رابط کاربری کنترل اضافه کنید که بتوان آن را از یک پیوند اشتراک‌گذاری tweakcn وارد کرد. خانواده‌های تم داخلی موجود همچنان `claw`، `knot` و `dash` باقی می‌مانند. خانواده جدید `custom` مانند یک خانواده تم عادی OpenClaw رفتار می‌کند و وقتی بار tweakcn واردشده شامل هر دو مجموعه توکن روشن و تاریک باشد، از حالت‌های `light`، `dark` و `system` پشتیبانی می‌کند.

تم واردشده فقط در پروفایل مرورگر فعلی همراه با بقیه تنظیمات رابط کاربری کنترل ذخیره می‌شود. در پیکربندی Gateway نوشته نمی‌شود و بین دستگاه‌ها یا مرورگرها همگام‌سازی نمی‌شود.

## مسئله

سیستم تم رابط کاربری کنترل در حال حاضر به سه خانواده تم کدنویسی‌شده محدود است:

- `ui/src/ui/theme.ts`
- `ui/src/ui/views/config.ts`
- `ui/src/styles/base.css`

کاربران می‌توانند بین خانواده‌های داخلی و گونه‌های حالت جابه‌جا شوند، اما نمی‌توانند بدون ویرایش CSS مخزن، تمی از tweakcn وارد کنند. خروجی درخواستی از یک سیستم تم‌سازی عمومی کوچک‌تر است: سه تم داخلی را نگه دارید و یک جایگاه واردشده تحت کنترل کاربر اضافه کنید که بتوان آن را از یک پیوند tweakcn جایگزین کرد.

## نااهداف

- بدون بازآرایی تم‌سازی در سراسر مخزن فراتر از مرزهای لازم برای رابط کاربری کنترل.

- بازنویسی جایگاه سفارشی با واردکردن بعدی، رفتار مورد نظر است.

## رویکرد پیشنهادی

یک شناسه خانواده تم چهارم، `custom`، به مدل تم رابط کاربری کنترل اضافه کنید. خانواده `custom` فقط وقتی یک واردکردن معتبر tweakcn وجود داشته باشد قابل انتخاب می‌شود. بار واردشده به یک رکورد تم سفارشی مخصوص OpenClaw نرمال‌سازی می‌شود و همراه با بقیه تنظیمات رابط کاربری در local storage مرورگر ذخیره می‌شود.

در زمان اجرا، OpenClaw یک برچسب `<style>` مدیریت‌شده رندر می‌کند که بلوک‌های متغیر CSS سفارشی حل‌شده را تعریف می‌کند:

```css
:root[data-theme="custom"] { ... }
:root[data-theme="custom-light"] { ... }
```

این کار متغیرهای تم سفارشی را به خانواده `custom` محدود می‌کند و از نشت متغیرهای CSS درون‌خطی به خانواده‌های داخلی جلوگیری می‌کند.

## معماری

### مدل تم

`ui/src/ui/theme.ts` را به‌روزرسانی کنید:

- `ThemeName` را گسترش دهید تا شامل `custom` شود.
- `ResolvedTheme` را گسترش دهید تا شامل `custom` و `custom-light` شود.
- `VALID_THEME_NAMES` را گسترش دهید.
- `resolveTheme()` را به‌روزرسانی کنید تا `custom` رفتار خانواده موجود را بازتاب دهد:
  - `custom + dark` -> `custom`
  - `custom + light` -> `custom-light`
  - `custom + system` -> `custom` یا `custom-light` بر اساس ترجیح سیستم‌عامل

هیچ نام مستعار قدیمی برای `custom` اضافه نمی‌شود.

### مدل پایداری

پایداری `UiSettings` را در `ui/src/ui/storage.ts` با یک بار اختیاری تم سفارشی گسترش دهید:

- `customTheme?: ImportedCustomTheme`

شکل ذخیره‌شده پیشنهادی:

```ts
type ImportedCustomTheme = {
  sourceUrl: string;
  themeId: string;
  label: string;
  importedAt: string;
  light: Record<string, string>;
  dark: Record<string, string>;
};
```

نکات:

- `sourceUrl` ورودی اصلی کاربر را پس از نرمال‌سازی ذخیره می‌کند.
- `themeId` شناسه تم tweakcn استخراج‌شده از URL است.
- `label` وقتی وجود داشته باشد فیلد `name` در tweakcn است، وگرنه `Custom`.
- `light` و `dark` نقشه‌های توکن OpenClaw از پیش نرمال‌شده هستند، نه بارهای خام tweakcn.
- بار واردشده کنار دیگر تنظیمات محلی مرورگر قرار می‌گیرد و در همان سند local-storage سریال‌سازی می‌شود.
- اگر داده تم سفارشی ذخیره‌شده هنگام بارگذاری مفقود یا نامعتبر باشد، بار را نادیده بگیرید و وقتی خانواده پایدارشده `custom` بوده است به `theme: "claw"` برگردید.

### اعمال در زمان اجرا

یک مدیر stylesheet باریک برای تم سفارشی در زمان اجرای رابط کاربری کنترل اضافه کنید، با مالکیت نزدیک به `ui/src/ui/app-settings.ts` و `ui/src/ui/theme.ts`.

مسئولیت‌ها:

- ایجاد یا به‌روزرسانی یک برچسب پایدار `<style id="openclaw-custom-theme">` در `document.head`.
- تولید CSS فقط وقتی یک بار معتبر تم سفارشی وجود دارد.
- حذف محتوای برچسب style وقتی بار پاک می‌شود.
- نگه داشتن CSS خانواده داخلی در `ui/src/styles/base.css`؛ توکن‌های واردشده را به stylesheet ثبت‌شده در مخزن وصله نکنید.

این مدیر هر بار که تنظیمات بارگذاری، ذخیره، وارد یا پاک می‌شوند اجرا می‌شود.

### انتخابگرهای حالت روشن

پیاده‌سازی باید برای سبک‌دهی روشن میان‌خانواده‌ای، به‌جای حالت‌گذاری ویژه برای `custom-light`، `data-theme-mode="light"` را ترجیح دهد. اگر انتخابگر موجودی به `data-theme="light"` چسبیده و لازم است به هر خانواده روشن اعمال شود، آن را در چارچوب این کار گسترش دهید.

## تجربه کاربری واردکردن

`ui/src/ui/views/config.ts` را در بخش `Appearance` به‌روزرسانی کنید:

- یک کارت تم `Custom` کنار `Claw`، `Knot` و `Dash` اضافه کنید.
- وقتی هیچ تم سفارشی واردشده‌ای وجود ندارد، کارت را غیرفعال نشان دهید.
- یک پنل واردکردن زیر شبکه تم اضافه کنید با:
  - یک ورودی متنی برای پیوند اشتراک‌گذاری tweakcn یا URL `/r/themes/{id}`
  - یک دکمه `Import`
  - یک مسیر `Replace` وقتی یک بار سفارشی از قبل وجود دارد
  - یک کنش `Clear` وقتی یک بار سفارشی از قبل وجود دارد
- وقتی یک بار وجود دارد، برچسب تم واردشده و میزبان منبع را نشان دهید.
- اگر تم فعال `custom` باشد، واردکردن جایگزین بلافاصله اعمال می‌شود.
- اگر تم فعال `custom` نباشد، واردکردن فقط بار جدید را ذخیره می‌کند تا کاربر کارت `Custom` را انتخاب کند.

انتخابگر سریع تم در `ui/src/ui/views/config-quick.ts` نیز باید `Custom` را فقط وقتی یک بار وجود دارد نشان دهد.

## تجزیه URL و دریافت راه‌دور

مسیر واردکردن مرورگر این موارد را می‌پذیرد:

- `https://tweakcn.com/themes/{id}`
- `https://tweakcn.com/r/themes/{id}`

پیاده‌سازی باید هر دو شکل را به این مقدار نرمال کند:

- `https://tweakcn.com/r/themes/{id}`

سپس مرورگر endpoint نرمال‌شده `/r/themes/{id}` را مستقیماً دریافت می‌کند.

برای بار خارجی از یک اعتبارسنج schema محدود استفاده کنید. چون این یک مرز خارجی غیرقابل اعتماد است، schema مبتنی بر zod ترجیح داده می‌شود.

فیلدهای راه‌دور لازم:

- `name` سطح بالا به‌عنوان رشته اختیاری
- `cssVars.theme` به‌عنوان شیء اختیاری
- `cssVars.light` به‌عنوان شیء
- `cssVars.dark` به‌عنوان شیء

اگر `cssVars.light` یا `cssVars.dark` مفقود باشد، واردکردن را رد کنید. این عمدی است: رفتار محصول تأییدشده پشتیبانی کامل از حالت‌ها است، نه ساخت بهترین تلاش برای سمت مفقود.

## رفتار شکست

واردکردن‌های نامعتبر باید بسته و ایمن شکست بخورند.

- قالب URL نامعتبر: خطای اعتبارسنجی درون‌خطی نشان دهید، دریافت نکنید.
- میزبان یا شکل مسیر پشتیبانی‌نشده: خطای اعتبارسنجی درون‌خطی نشان دهید، دریافت نکنید.
- شکست شبکه، پاسخ غیر OK، یا JSON بدشکل: خطای درون‌خطی نشان دهید، بار ذخیره‌شده فعلی را دست‌نخورده نگه دارید.
- شکست schema یا مفقود بودن بلوک‌های روشن/تاریک: خطای درون‌خطی نشان دهید، بار ذخیره‌شده فعلی را دست‌نخورده نگه دارید.
- کنش پاک‌سازی:
  - بار سفارشی ذخیره‌شده را حذف می‌کند
  - محتوای برچسب style سفارشی مدیریت‌شده را حذف می‌کند
  - اگر `custom` فعال باشد، خانواده تم را به `claw` برمی‌گرداند
- بار سفارشی ذخیره‌شده نامعتبر در نخستین بارگذاری:
  - بار ذخیره‌شده را نادیده بگیرید
  - CSS سفارشی تولید نکنید
  - اگر خانواده تم پایدارشده `custom` بوده، به `claw` برگردید

در هیچ نقطه‌ای نباید واردکردن ناموفق سند فعال را با متغیرهای CSS سفارشی ناقص اعمال‌شده باقی بگذارد.
