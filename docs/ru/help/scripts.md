`scripts/` содержит вспомогательные скрипты для локальных рабочих процессов и операционных задач. Используйте их, когда задача явно связана со скриптом; в остальных случаях отдавайте предпочтение CLI.

## Соглашения

- Скрипты **необязательны**, если на них нет ссылок в документации или контрольных списках выпуска.
- Отдавайте предпочтение возможностям CLI, если они доступны (например, `openclaw models status --check`).
- Считайте, что скрипты зависят от конкретной хост-системы; изучите их перед запуском на новом компьютере.

## Скрипты мониторинга аутентификации

Общая аутентификация моделей описана в разделе [Аутентификация](/ru/gateway/authentication). Приведённые ниже скрипты представляют собой отдельную необязательную систему для мониторинга **токена подписки Claude Code CLI** на удалённом хосте без графического интерфейса и повторной аутентификации с телефона:

- `scripts/setup-auth-system.sh` — однократная настройка: проверяет текущую аутентификацию, помогает создать долгоживущий `claude setup-token` и выводит инструкции по установке для systemd/Termux.
- `scripts/claude-auth-status.sh [full|json|simple]` — проверяет состояние аутентификации Claude Code и OpenClaw.
- `scripts/auth-monitor.sh` — периодически проверяет состояние и отправляет уведомление (через отправку OpenClaw и/или ntfy.sh), когда срок действия токена подходит к концу. Переменные окружения: `WARN_HOURS` (по умолчанию `2`), `NOTIFY_PHONE`, `NOTIFY_NTFY`. Запускайте по расписанию с помощью включённых файлов `scripts/systemd/openclaw-auth-monitor.{service,timer}` (каждые 30 минут).
- `scripts/mobile-reauth.sh` — повторно запускает `claude setup-token` и выводит URL-адреса, которые нужно открыть на телефоне; предназначен для использования через SSH из Termux.
- `scripts/termux-quick-auth.sh`, `scripts/termux-auth-widget.sh`, `scripts/termux-sync-widget.sh` — скрипты Termux:Widget, которые подключаются к хосту по SSH, показывают всплывающее уведомление о состоянии и открывают консоль или инструкции для повторной аутентификации, когда срок действия аутентификации истёк.

## Вспомогательный скрипт чтения GitHub

Используйте `scripts/gh-read`, когда требуется, чтобы `gh` применял токен установки GitHub App для вызовов чтения в пределах репозитория, а обычный `gh` сохранял вход в вашу личную учётную запись для операций записи.

Обязательные переменные окружения:

- `OPENCLAW_GH_READ_APP_ID`
- `OPENCLAW_GH_READ_PRIVATE_KEY_FILE`

Необязательные переменные окружения:

- `OPENCLAW_GH_READ_INSTALLATION_ID`, если требуется пропустить поиск установки по репозиторию
- `OPENCLAW_GH_READ_PERMISSIONS` — переопределение запрашиваемого подмножества разрешений на чтение в виде списка, разделённого запятыми

Порядок определения репозитория:

- `gh ... -R owner/repo`
- `GH_REPO`
- `git remote origin`

Примеры:

- `scripts/gh-read pr view 123`
- `scripts/gh-read run list -R openclaw/openclaw`
- `scripts/gh-read api repos/openclaw/openclaw/pulls/123`
