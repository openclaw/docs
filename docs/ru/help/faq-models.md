  Вопросы и ответы о моделях и профилях аутентификации. Сведения о настройке, сеансах, Gateway, каналах и
  устранении неполадок см. в основном разделе [часто задаваемых вопросов](/ru/help/faq).

  ## Модели: значения по умолчанию, выбор, псевдонимы и переключение

  <AccordionGroup>
  <Accordion title='Что такое «модель по умолчанию»?'>
    Задаётся с помощью:

    ```text
    agents.defaults.model.primary
    ```

    Ссылки на модели имеют формат `provider/model` (например, `openai/gpt-5.5`,
    `anthropic/claude-sonnet-4-6`). Всегда явно указывайте `provider/model`. Если
    поставщик не указан, OpenClaw сначала пытается найти соответствующий псевдоним,
    затем — уникальное совпадение идентификатора модели среди настроенных поставщиков,
    после чего использует настроенного поставщика по умолчанию (устаревший путь
    совместимости). Если у этого поставщика больше нет настроенной модели по умолчанию,
    OpenClaw вместо устаревшего значения по умолчанию использует первую настроенную
    пару поставщик/модель.

  </Accordion>

  <Accordion title="Какую модель вы рекомендуете?">
    Используйте самую мощную модель последнего поколения, доступную в вашем наборе
    поставщиков, особенно для агентов с доступом к инструментам или обрабатывающих
    недоверенные входные данные: более слабые или чрезмерно квантованные модели
    уязвимее для внедрения инструкций и небезопасного поведения (см.
    [Безопасность](/ru/gateway/security)). Направляйте более дешёвые модели на
    повседневные диалоги с низким уровнем риска в соответствии с ролью агента.

    Назначайте модели каждому агенту отдельно и используйте субагентов для
    параллельного выполнения длительных задач (каждый субагент расходует собственные
    токены). См. [Модели](/ru/concepts/models),
    [Субагенты](/ru/tools/subagents), [MiniMax](/ru/providers/minimax) и
    [Локальные модели](/ru/gateway/local-models).

  </Accordion>

  <Accordion title="Как переключить модель, не стирая конфигурацию?">
    Изменяйте только поля модели — не заменяйте всю конфигурацию.

    - `/model` в чате (для отдельного сеанса, см. [Команды с косой чертой](/ru/tools/slash-commands))
    - `openclaw models set ...` (обновляет только конфигурацию модели)
    - `openclaw configure --section model` (интерактивно)
    - напрямую измените `agents.defaults.model` в `~/.openclaw/openclaw.json`

    При изменении через RPC сначала проверьте данные с помощью
    `config.schema.lookup` (нормализованный путь, краткая документация схемы,
    сводки дочерних элементов), затем предпочитайте `config.patch` с частичным
    объектом вместо `config.apply`. Если конфигурация всё же была перезаписана,
    восстановите её из резервной копии или запустите `openclaw doctor` для
    исправления.

    Документация: [Модели](/ru/concepts/models), [Настройка](/ru/cli/configure),
    [Конфигурация](/ru/cli/config), [Диагностика](/ru/gateway/doctor).

  </Accordion>

  <Accordion title="Можно ли использовать модели на собственном сервере (llama.cpp, vLLM, Ollama)?">
    Да — проще всего использовать Ollama. Быстрая настройка:

    1. Установите Ollama с `https://ollama.com/download`
    2. Загрузите локальную модель, например `ollama pull gemma4`
    3. Чтобы также использовать облачные модели, выполните `ollama signin`
    4. Запустите `openclaw onboard`, выберите `Ollama`, затем `Local` или `Cloud + Local`

    `Cloud + Local` предоставляет облачные модели вместе с локальными моделями
    Ollama; облачные модели, такие как `kimi-k2.5:cloud`, не требуют локальной
    загрузки. Для ручного переключения выполните `openclaw models list`, а затем
    `openclaw models set ollama/<model>`.

    Небольшие или сильно квантованные модели уязвимее для внедрения инструкций.
    Используйте крупные модели для любых ботов с доступом к инструментам; если вы
    всё же используете небольшие модели, включите изоляцию и строгие списки
    разрешённых инструментов.

    Документация: [Ollama](/ru/providers/ollama), [Локальные модели](/ru/gateway/local-models),
    [Поставщики моделей](/ru/concepts/model-providers), [Безопасность](/ru/gateway/security),
    [Изоляция](/ru/gateway/sandboxing).

  </Accordion>

  <Accordion title="Как переключать модели на лету (без перезапуска)?">
    Отправьте `/model <name>` отдельным сообщением. Полный список команд, включая
    нумерованный выбор (`/model`, `/model list`, `/model 3`), команду
    `/model default` для удаления переопределения сеанса и `/model status` для
    просмотра сведений о конечной точке и режиме API, см. в разделе
    [Команды с косой чертой](/ru/tools/slash-commands).

    Чтобы принудительно назначить определённый профиль аутентификации для сеанса,
    используйте `@profile`:

    ```text
    /model opus@anthropic:default
    /model opus@anthropic:work
    ```

    Чтобы открепить профиль, заданный через `@profile`, повторно выполните `/model`
    без суффикса (например, `/model anthropic/claude-opus-4-6`) или выберите
    значение по умолчанию через `/model`. Используйте `/model status`, чтобы
    проверить активный профиль аутентификации.

  </Accordion>

  <Accordion title="Если два поставщика предоставляют модель с одинаковым идентификатором, какого из них использует /model?">
    `/model provider/model` выбирает маршрут именно этого поставщика. Например,
    `qianfan/deepseek-v4-flash` и `deepseek/deepseek-v4-flash` — разные ссылки,
    несмотря на совпадение идентификатора модели: OpenClaw не переключает
    поставщиков неявно только из-за совпадения идентификатора.

    Выбранная пользователем через `/model` ссылка обрабатывается строго при
    резервном переключении: если эта пара поставщик/модель становится недоступной,
    ответ завершается явной ошибкой вместо перехода к
    `agents.defaults.model.fallbacks`. Настроенные цепочки резервного переключения
    по-прежнему применяются к настроенным значениям по умолчанию, основным моделям
    заданий Cron и автоматически выбранному резервному состоянию. Если запуск без
    переопределения сеанса может использовать резервное переключение, OpenClaw
    сначала пробует запрошенную пару поставщик/модель, затем настроенные резервные
    варианты и после этого настроенную основную модель. Поэтому совпадающие
    идентификаторы моделей без поставщика никогда не приводят к немедленному
    возврату к поставщику по умолчанию.

    См. [Модели](/ru/concepts/models) и [Резервное переключение моделей](/ru/concepts/model-failover).

  </Accordion>

  <Accordion title="Можно ли использовать GPT 5.5 для повседневных задач, а Codex 5.5 — для программирования?">
    Да — выбор модели и среды выполнения не зависят друг от друга:

    - **Нативный агент Codex для программирования:** задайте для
      `agents.defaults.model.primary` значение `openai/gpt-5.5`. Выполните вход
      с помощью `openclaw models auth login --provider openai`, чтобы использовать
      аутентификацию подписки ChatGPT/Codex.
    - **Прямые задачи OpenAI API вне цикла агента:** настройте
      `OPENAI_API_KEY` для изображений, эмбеддингов, речи, взаимодействия в
      реальном времени и других интерфейсов OpenAI API, не относящихся к агенту.
    - **Аутентификация агента OpenAI с помощью ключа API:** используйте
      `/model openai/gpt-5.5` с упорядоченным профилем ключа API `openai`.
    - **Субагенты:** направляйте задачи программирования агенту, ориентированному
      на Codex и использующему собственную модель `openai/gpt-5.5`.

    См. [Модели](/ru/concepts/models) и [Команды с косой чертой](/ru/tools/slash-commands).

  </Accordion>

  <Accordion title="Как настроить быстрый режим для GPT 5.5?">
    - **Для отдельного сеанса:** отправьте `/fast on` при использовании `openai/gpt-5.5`.
    - **По умолчанию для модели:** задайте для
      `agents.defaults.models["openai/gpt-5.5"].params.fastMode` значение `true`.
    - **Автоматическое ограничение:** `/fast auto` или `params.fastMode: "auto"`
      выполняет новые вызовы модели в быстром режиме до достижения ограничения,
      а последующие повторные, резервные вызовы, вызовы после результата инструмента
      или продолжения — без быстрого режима. По умолчанию ограничение составляет
      60 секунд; его можно переопределить с помощью `params.fastAutoOnSeconds`
      для модели.

    ```json5
    {
      agents: {
        defaults: {
          models: {
            "openai/gpt-5.5": {
              params: {
                fastMode: "auto",
                fastAutoOnSeconds: 30,
              },
            },
          },
        },
      },
    }
    ```

    Для нативных запросов OpenAI Responses быстрый режим соответствует
    `service_tier = "priority"`; существующие значения `service_tier` сохраняются,
    а быстрый режим не изменяет `reasoning` или `text.verbosity`. Переопределения
    `/fast` для сеанса имеют приоритет над значениями конфигурации по умолчанию.

    См. [Режим рассуждения и быстрый режим](/ru/tools/thinking), а также раздел
    «Быстрый режим» в дополнительных настройках на странице поставщика
    [OpenAI](/ru/providers/openai).

  </Accordion>

  <Accordion title='Почему отображается сообщение «Model ... is not allowed», после чего ответ не приходит?'>
    Если задано `agents.defaults.models`, оно становится **списком разрешённых
    моделей** для `/model` и переопределений сеанса. При выборе модели вне этого
    списка вместо обычного ответа возвращается следующее:

    ```text
    Model "provider/model" is not allowed. Use /models to list providers, or /models <provider> to list models.
    Add it with: openclaw config set agents.defaults.models '{"provider/model":{}}' --strict-json --merge
    ```

    Исправление: добавьте точную модель в `agents.defaults.models`, добавьте
    шаблон поставщика, например `"provider/*": {}`, для динамических каталогов,
    удалите список разрешённых моделей или выберите модель из `/model list`.
    Если команда также содержала `--runtime codex`, сначала обновите список
    разрешённых моделей, а затем повторите ту же команду
    `/model provider/model --runtime codex`.

  </Accordion>

  <Accordion title='Почему отображается сообщение «Unknown model: minimax/MiniMax-M3»?'>
    Если вы используете старую версию OpenClaw, сначала обновите её (либо запустите
    исходный код из `main`) и перезапустите Gateway — модель `MiniMax-M3` может
    ещё отсутствовать в каталоге установленной версии. В противном случае поставщик
    MiniMax не настроен (не найдена запись поставщика или профиль аутентификации),
    поэтому модель невозможно разрешить. Полный список действий для исправления,
    таблицу идентификаторов поставщиков и моделей, а также пример блока конфигурации
    см. в разделе устранения неполадок на странице поставщика
    [MiniMax](/ru/providers/minimax).

  </Accordion>

  <Accordion title="Можно ли использовать MiniMax по умолчанию, а OpenAI — для сложных задач?">
    Да. Используйте MiniMax по умолчанию и переключайте модели для отдельных
    сеансов: резервные варианты предназначены для ошибок, а не для «сложных задач»,
    поэтому используйте `/model` или отдельного агента.

    **Вариант A: переключение для отдельного сеанса**

    ```json5
    {
      env: { MINIMAX_API_KEY: "sk-...", OPENAI_API_KEY: "sk-..." },
      agents: {
        defaults: {
          model: { primary: "minimax/MiniMax-M3" },
          models: {
            "minimax/MiniMax-M3": { alias: "minimax" },
            "openai/gpt-5.5": { alias: "gpt" },
          },
        },
      },
    }
    ```

    Затем выполните `/model gpt`.

    **Вариант B: отдельные агенты** — агент A по умолчанию использует MiniMax,
    агент B — OpenAI; направляйте запросы по агентам или используйте `/agent`
    для переключения.

    Документация: [Модели](/ru/concepts/models), [Маршрутизация между несколькими агентами](/ru/concepts/multi-agent),
    [MiniMax](/ru/providers/minimax), [OpenAI](/ru/providers/openai).

  </Accordion>

  <Accordion title="Являются ли opus / sonnet / gpt встроенными сокращениями?">
    Да — это встроенные сокращения, применяемые только в том случае, если целевая
    модель присутствует в `agents.defaults.models`:

    | Псевдоним | Разрешается в |
    | --- | --- |
    | `opus` | `anthropic/claude-opus-4-8` |
    | `sonnet` | `anthropic/claude-sonnet-4-6` |
    | `gpt` | `openai/gpt-5.4` |
    | `gpt-mini` | `openai/gpt-5.4-mini` |
    | `gpt-nano` | `openai/gpt-5.4-nano` |
    | `gemini` | `google/gemini-3.1-pro-preview` |
    | `gemini-flash` | `google/gemini-3-flash-preview` |
    | `gemini-flash-lite` | `google/gemini-3.1-flash-lite` |

    Ваш собственный псевдоним с таким же именем переопределяет встроенный.

  </Accordion>

  <Accordion title="Как определить или переопределить сокращения (псевдонимы) моделей?">
    Псевдонимы задаются в `agents.defaults.models.<modelId>.alias`:

    ```json5
    {
      agents: {
        defaults: {
          model: { primary: "anthropic/claude-opus-4-6" },
          models: {
            "anthropic/claude-opus-4-6": { alias: "opus" },
            "anthropic/claude-sonnet-4-6": { alias: "sonnet" },
          },
        },
      },
    }
    ```

    После этого `/model sonnet` (или `/<alias>`, если поддерживается)
    разрешается в идентификатор этой модели.

  </Accordion>

  <Accordion title="Как добавить модели других поставщиков, например OpenRouter или Z.AI?">
    OpenRouter (оплата за токен; множество моделей):

    ```json5
    {
      agents: {
        defaults: {
          model: { primary: "openrouter/anthropic/claude-sonnet-4-6" },
          models: { "openrouter/anthropic/claude-sonnet-4-6": {} },
        },
      },
      env: { OPENROUTER_API_KEY: "sk-or-..." },
    }
    ```

    Z.AI (модели GLM):

    ```json5
    {
      agents: {
        defaults: {
          model: { primary: "zai/glm-5.1" },
          models: { "zai/glm-5.1": {} },
        },
      },
      env: { ZAI_API_KEY: "..." },
    }
    ```

    Отсутствие ключа поставщика для указанной пары поставщик/модель вызывает
    ошибку аутентификации во время выполнения (например,
    `No API key found for provider "zai"`).

    **После добавления нового агента не найден ключ API поставщика**

    У нового агента хранилище аутентификации пусто — данные аутентификации
    хранятся отдельно для каждого агента по адресу:

    ```text
    ~/.openclaw/agents/<agentId>/agent/auth-profiles.json
    ```

    Исправление: выполните `openclaw agents add <id>` и настройте аутентификацию в мастере либо
    скопируйте из хранилища основного агента только переносимые статические
    профили `api_key`/`token`. При использовании OAuth войдите в систему от имени
    нового агента, когда ему потребуется собственная учетная запись. Полные правила
    повторного использования `agentDir` и совместного доступа к учетным данным см. в разделе
    [Маршрутизация между несколькими агентами](/ru/concepts/multi-agent) — никогда не используйте
    один `agentDir` для нескольких агентов.

  </Accordion>
</AccordionGroup>

## Переключение между моделями при сбое и ошибка «Сбой всех моделей»

<AccordionGroup>
  <Accordion title="Как работает переключение при сбое?">
    Оно состоит из двух этапов:

    1. **Ротация профилей аутентификации** в рамках одного поставщика.
    2. **Переход на резервную модель** — следующую модель из `agents.defaults.model.fallbacks`.

    Для профилей, в которых возникают ошибки, применяются периоды ожидания
    (экспоненциальная задержка), поэтому OpenClaw продолжает отвечать, когда
    поставщик ограничивает частоту запросов или временно недоступен.

    Категория ограничений частоты охватывает не только обычную ошибку `429`:
    `Too many concurrent requests`, `ThrottlingException`, `concurrency limit reached`,
    `workers_ai ... quota limit exceeded`, `resource exhausted` и периодические
    ограничения по окнам использования (`weekly/monthly limit reached`) также считаются
    ограничениями частоты, при которых следует выполнить переключение.

    Ответы, связанные с оплатой, не всегда имеют код `402`, а некоторые ответы `402`
    остаются в категории временных ошибок или ограничений частоты и не относятся к
    категории оплаты. Явный текст об оплате в ответах `401`/`403` по-прежнему может
    направить ошибку в категорию оплаты; средства сопоставления текста для конкретных
    поставщиков (например, OpenRouter `Key limit exceeded`) применяются только к
    соответствующему поставщику. Ответ `402`, похожий на повторяемое ограничение окна
    использования или расходов организации/рабочего пространства (`daily limit reached, resets tomorrow`,
    `organization spending limit exceeded`), рассматривается как `rate_limit`, а не как
    длительная блокировка из-за оплаты.

    Ошибки переполнения контекста вообще не приводят к переходу на резервную модель —
    сигнатуры вроде `request_too_large`, `input exceeds the maximum number of tokens`,
    `input token count exceeds the maximum number of input tokens`, `input is too long for the model`
    или `ollama error: context length exceeded` запускают Compaction и повторную попытку,
    а не переход к следующей резервной модели.

    Обобщенное распознавание текста серверных ошибок уже, чем «всё, что содержит
    unknown/error». К временным ошибкам конкретных поставщиков, которые считаются
    сигналами для переключения, относятся: отдельное сообщение Anthropic
    `An unknown error occurred`, отдельное сообщение OpenRouter `Provider returned error`,
    ошибки причины остановки вроде `Unhandled stop reason: error`, полезная нагрузка JSON
    `api_error` с текстом временной серверной ошибки (`internal server error`,
    `unknown error, 520`, `upstream error`, `backend error`) и ошибки занятости
    поставщика вроде `ModelNotReadyException`, если совпадает контекст поставщика.
    Общий внутренний текст ошибки вроде `LLM request failed with an unknown error.`
    обрабатывается консервативно и сам по себе не запускает переключение.

  </Accordion>

  <Accordion title='Что означает «No credentials found for profile anthropic:default»?'>
    У профиля аутентификации с идентификатором `anthropic:default` отсутствуют
    учетные данные в ожидаемом хранилище аутентификации.

    **Порядок устранения проблемы:**

    - Проверьте расположение профилей — текущее:
      `~/.openclaw/agents/<agentId>/agent/auth-profiles.json`; устаревшее:
      `~/.openclaw/agent/*` (переносится командой `openclaw doctor`).
    - Убедитесь, что Gateway загружает вашу переменную среды. Переменная
      `ANTHROPIC_API_KEY`, заданная только в оболочке, не будет доступна Gateway,
      запущенному через systemd/launchd, — добавьте ее в `~/.openclaw/.env` или
      включите `env.shellEnv`.
    - Убедитесь, что редактируете нужного агента: в конфигурациях с несколькими
      агентами имеется несколько файлов `auth-profiles.json`.
    - Выполните `openclaw models status`, чтобы просмотреть настроенные модели и
      состояние аутентификации поставщиков.

    **Для ошибки «No credentials found for profile anthropic» (без суффикса электронной почты):**

    Выполнение закреплено за профилем Anthropic, который Gateway не может найти.

    - Используйте Claude CLI: выполните `openclaw models auth login --provider anthropic
      --method cli --set-default` на узле Gateway.
    - Если предпочтителен ключ API: добавьте `ANTHROPIC_API_KEY` в
      `~/.openclaw/.env` на узле Gateway, затем удалите закрепленный порядок,
      принудительно использующий отсутствующий профиль:

      ```bash
      openclaw models auth order clear --provider anthropic
      ```

    - В удаленном режиме профили аутентификации хранятся на машине Gateway, а не
      на вашем ноутбуке, — убедитесь, что выполняете команды именно там.

  </Accordion>

  <Accordion title="Почему система также попыталась использовать Google Gemini и завершилась ошибкой?">
    Если конфигурация моделей включает Google Gemini как резервный вариант
    (или вы перешли на сокращенное обозначение Gemini), OpenClaw попытается
    использовать его при переключении. При отсутствии настроенных учетных данных
    Google возникает ошибка `No API key found for provider "google"`. Исправление:
    добавьте аутентификацию Google или удалите модели Google из
    `agents.defaults.model.fallbacks`/псевдонимов.

    **Запрос LLM отклонен: требуется подпись рассуждений (Google Antigravity)**

    Причина: история сеанса содержит блоки рассуждений без подписей (часто
    из-за прерванного или неполного потока); Google Antigravity требует наличия
    подписей у блоков рассуждений. OpenClaw удаляет неподписанные блоки рассуждений
    для Google Antigravity Claude; если ошибка сохраняется, начните новый сеанс
    или задайте `/thinking off` для этого агента.

  </Accordion>
</AccordionGroup>

## Профили аутентификации: что это такое и как ими управлять

См. также: [/concepts/oauth](/ru/concepts/oauth) (процессы OAuth, хранение токенов и схемы работы с несколькими учетными записями)

<AccordionGroup>
  <Accordion title="Что такое профиль аутентификации?">
    Именованная запись учетных данных (OAuth или ключ API), связанная с
    поставщиком и хранящаяся по адресу:

    ```text
    ~/.openclaw/agents/<agentId>/agent/auth-profiles.json
    ```

    Просмотреть сохраненные профили без вывода секретов можно командой
    `openclaw models auth list` (при необходимости с `--provider <id>` или
    `--json`). См. раздел [CLI моделей](/ru/cli/models#auth-profiles).

  </Accordion>

  <Accordion title="Какие идентификаторы профилей используются обычно?">
    С префиксом поставщика: `anthropic:default` (обычно при отсутствии
    идентификатора электронной почты), `anthropic:<email>` для идентификаторов
    OAuth или выбранный вами пользовательский идентификатор (например,
    `anthropic:work`).

  </Accordion>

  <Accordion title="Можно ли управлять тем, какой профиль аутентификации используется первым?">
    Да. Параметр конфигурации `auth.order.<provider>` задает порядок ротации для
    каждого поставщика (только метаданные — секреты не сохраняются).

    OpenClaw может пропустить профиль, находящийся в кратковременном состоянии
    **ожидания** (ограничения частоты, тайм-ауты, ошибки аутентификации), или в
    более длительном состоянии **отключения** (проблемы с оплатой или недостаточно
    средств). Состояние можно проверить командой `openclaw models status --json`
    в поле `auth.unusableProfiles`. Настройка выполняется с помощью
    `auth.cooldowns.billingBackoffHours*`. Периоды ожидания из-за ограничений
    частоты могут применяться к отдельной модели: профиль, находящийся в состоянии
    ожидания для одной модели, по-прежнему может обслуживать другую модель того же
    поставщика; периоды блокировки из-за оплаты или отключения блокируют профиль
    целиком.

    Задайте переопределение порядка для отдельного агента (оно хранится в файле
    `auth-state.json` этого агента):

    ```bash
    # Defaults to the configured default agent (omit --agent)
    openclaw models auth order get --provider anthropic

    # Lock rotation to a single profile
    openclaw models auth order set --provider anthropic anthropic:default

    # Or set an explicit order (fallback within provider)
    openclaw models auth order set --provider anthropic anthropic:work anthropic:default

    # Clear override (fall back to config auth.order / round-robin)
    openclaw models auth order clear --provider anthropic

    # Target a specific agent
    openclaw models auth order set --provider anthropic --agent main anthropic:default
    ```

    Проверьте, какие профили действительно будут опробованы: `openclaw models status --probe`.
    Для сохраненного профиля, исключенного из явно заданного порядка, вместо
    незаметной попытки использования отображается `excluded_by_auth_order`.

  </Accordion>

  <Accordion title="В чем разница между OAuth и ключом API?">
    - **Вход через OAuth / CLI** часто использует доступ по подписке, если
      поставщик его поддерживает. Для Anthropic серверная часть Claude CLI в
      OpenClaw использует Claude Code `claude -p`, который Anthropic в настоящее
      время рассматривает как использование Agent SDK/программный доступ,
      расходующий лимиты подписки. Текущее состояние приостановки оплаты и ссылки
      на источники см. в разделе [Anthropic](/ru/providers/anthropic).
    - **Ключи API** используют оплату за каждый токен.

    Мастер поддерживает Anthropic Claude CLI, OpenAI Codex OAuth и ключи API.

  </Accordion>
</AccordionGroup>
