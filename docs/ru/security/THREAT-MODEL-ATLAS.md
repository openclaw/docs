**Версия:** 1.0-черновик | **Фреймворк:** [MITRE ATLAS](https://atlas.mitre.org/) (ландшафт состязательных угроз для систем ИИ) + диаграммы потоков данных

Эта модель угроз описывает состязательные угрозы для платформы ИИ-агентов OpenClaw и каталога Skills ClawHub. Это постоянно обновляемый документ, который поддерживается сообществом OpenClaw. Сведения о том, как сообщать о новых угрозах, предлагать цепочки атак или меры защиты, см. в разделе [Участие в разработке модели угроз](/ru/security/CONTRIBUTING-THREAT-MODEL).

## 1. Область охвата

| Компонент                    | Включён     | Примечания                                               |
| ---------------------------- | ----------- | -------------------------------------------------------- |
| Среда выполнения агента OpenClaw | Да      | Выполнение ядра агента, вызовы инструментов, сеансы      |
| Gateway                      | Да          | Аутентификация, маршрутизация, интеграция с каналами      |
| Интеграции с каналами        | Да          | WhatsApp, Telegram, Discord, Signal, Slack и т. д.        |
| Каталог ClawHub              | Да          | Публикация, модерация и распространение Skills            |
| Серверы MCP                  | Да          | Внешние поставщики инструментов                           |
| Устройства пользователей     | Частично    | Мобильные приложения, клиенты для настольных компьютеров |

Не входящие в область охвата отчёты и шаблоны ложных срабатываний (доступность из общедоступного интернета, цепочки только с инъекцией промптов без обхода границы, совместное использование одного хоста Gateway взаимно недоверяющими операторами и другие) перечислены в [`SECURITY.md`](https://github.com/openclaw/openclaw/blob/main/SECURITY.md); актуальным источником истины об области охвата отчётов об уязвимостях является этот файл, а не данная страница.

## 4. Анализ цепочки поставок ClawHub

### 4.1 Текущие средства контроля безопасности

| Мера контроля                  | Реализация                                                                            | Эффективность                                                              |
| ------------------------------ | ------------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| Возраст учётной записи GitHub  | `requireGitHubAccountAge()` (минимум 14 дней)                                         | Средняя — повышает порог входа для новых злоумышленников                    |
| Санитизация пути               | `sanitizePath()`                                                                      | Высокая — предотвращает обход каталогов                                    |
| Проверка типа файла            | `isTextFile()`                                                                        | Средняя — сканируются только текстовые файлы, но уязвимость всё ещё возможна |
| Ограничения размера            | Общий размер пакета — 50 МБ (`MAX_PUBLISH_TOTAL_BYTES`)                               | Высокая — предотвращает истощение ресурсов                                 |
| Обязательный SKILL.md          | Обязательный файл readme при публикации                                               | Низкая ценность для безопасности — только информационная мера              |
| Статическое и близкое к AST сканирование | Механизм шаблонов, охватывающий выполнение команд, эксфильтрацию, сбор учётных данных, обфускацию и многое другое | Средне-высокая — охватывает многие известные схемы злоупотреблений, но по-прежнему основана на шаблонах |
| Агентная оценка рисков на основе LLM | Вердикт при публикации, формируемый на основе запроса безопасности                    | Средне-высокая — выявляет поведение, пропускаемое статическими шаблонами    |
| Сканирование VirusTotal        | Интегрировано в процессы публикации и повторного сканирования Skills и выпусков пакетов; требует API-ключа оператора | Высокая при включении — обнаружение статическим механизмом                  |
| Статус модерации               | Поле `moderationStatus`                                                               | Средняя — возможна ручная проверка                                         |

### 4.2 Ограничения модерации

Статическое сканирование ClawHub напрямую проверяет содержимое кода Skills (а не только идентификатор, метаданные или frontmatter), охватывая опасные вызовы выполнения команд, динамическое выполнение кода, сбор учётных данных, шаблоны эксфильтрации, обфусцированные полезные нагрузки и многое другое. Известные ограничения:

- Обнаружение на основе шаблонов всё ещё можно обойти с помощью достаточно новой обфускации.
- Проверка на основе LLM и сканирование VirusTotal зависят от включения API-ключей и конфигурации на стороне оператора.
- После установки отсутствует изолирующая среда выполнения, которая отделяла бы Skills от собственных привилегий агента.

### 4.3 Значки

Skills и пакеты получают назначаемые модераторами значки: `highlighted`, `official`, `deprecated`, `redactionApproved` (только для Skills). Сообщения сообщества (`skillReports`) и журналы аудита (`auditLogs`) обеспечивают рабочие процессы модерации.
