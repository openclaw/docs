OpenClaw подключается к Zalo ClawBot через указанный в каталоге внешний плагин `@zalo-platforms/openclaw-zaloclawbot`. Для входа используется QR-код Zalo Mini App; идентификатор плагина в конфигурации — `openclaw-zaloclawbot`.

## Совместимость

| Версия плагина | Версия OpenClaw | dist-tag npm | Статус        |
| -------------- | ---------------- | ------------ | ------------- |
| 0.1.4          | >=2026.4.10      | `latest`     | Активен / Бета |

## Предварительные требования

- Node.js >= 22
- Установлен [OpenClaw](https://docs.openclaw.ai/install) (доступен CLI `openclaw`)
- Учетная запись Zalo на мобильном устройстве для сканирования QR-кода входа

## Установка с помощью onboard (рекомендуется)

```bash
openclaw onboard
```

Выберите **Zalo ClawBot** в меню каналов. Мастер установит плагин из официального каталога с проверкой целостности, отобразит QR-код входа в терминале и завершит настройку канала после сканирования кода в приложении Zalo.

## Установка вручную

Чтобы добавить канал в уже настроенный Gateway:

### 1. Установите плагин

```bash
openclaw plugins install "@zalo-platforms/openclaw-zaloclawbot@0.1.4"
```

Используйте точно указанную закрепленную версию, чтобы во время установки OpenClaw проверил пакет по хешу целостности из каталога.

### 2. Включите плагин в конфигурации

```bash
openclaw config set plugins.entries.openclaw-zaloclawbot.enabled true
```

### 3. Создайте QR-код и войдите

```bash
openclaw channels login --channel openclaw-zaloclawbot
```

Отсканируйте отображаемый в терминале QR-код с помощью мобильного приложения Zalo, примите условия использования в Zalo Mini App и авторизуйте сеанс.

### 4. Перезапустите Gateway

```bash
openclaw gateway restart
```

## Как это работает

В отличие от стандартного канала Zalo, для которого требуется зарегистрировать собственную официальную учетную запись Zalo (OA) и настроить статические учетные данные разработчика, Zalo ClawBot представляет собой **привязанного к владельцу персонального помощника** в общей официальной инфраструктуре:

1. **Первоначальная настройка:** QR-код ведет в Zalo Mini App, которое привязывает недавно созданного приватного бота в общей официальной OA непосредственно к вашему идентификатору пользователя Zalo.
2. **Конфиденциальность с привязкой к владельцу:** бот взаимодействует только со своим владельцем. Сообщения от других пользователей отбрасываются на уровне платформы.
3. **Официальный путь API:** плагин использует API Zalo Bot Platform, а не автоматизацию браузера или веб-сеанса.

## Внутреннее устройство

Плагин взаимодействует с Zalo через постоянный цикл длительного опроса (`getUpdates`). Webhook по умолчанию отключены при локальном запуске Gateway в настольной среде или терминале. Сообщения обрабатываются на стороне клиента и передаются в локальную среду выполнения агента.

Плагин хранит учетные данные бота в каталоге состояния OpenClaw. Считайте этот каталог конфиденциальным и применяйте к нему ту же политику управления доступом и резервного копирования, что и к остальным данным состояния OpenClaw.

Среда выполнения этого плагина полностью находится во внешнем пакете `@zalo-platforms/openclaw-zaloclawbot`; приведенные ниже сведения о поведении, выходящие за рамки установки и конфигурации, предоставлены сопровождающими плагина и не проверены по исходному коду ядра OpenClaw.

## Устранение неполадок

- **Истечение времени ожидания входа по QR-коду:** в целях безопасности срок действия токена входа (`zbsk`) истекает через 5 минут. Если срок действия QR-кода истек до сканирования, повторно выполните команду входа, чтобы создать новый код.
- **Gateway не загружается:** убедитесь, что версия узла OpenClaw — `2026.4.10` или выше. Более старые версии не поддерживают реестр установки внешних npm-плагинов, необходимый для этого идентификатора.
