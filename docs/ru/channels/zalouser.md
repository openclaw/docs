Статус: экспериментальная интеграция. Она автоматизирует **личную учетную запись Zalo** через нативную библиотеку `zca-js`, работающую внутри процесса, без внешнего исполняемого файла CLI.

<Warning>
Это неофициальная интеграция, использование которой может привести к приостановке или блокировке учетной записи. Используйте на свой страх и риск.
</Warning>

## Установка

Zalo Personal — официальный внешний плагин, не входящий в состав ядра. Установите его перед использованием:

```bash
openclaw plugins install @openclaw/zalouser
```

- Зафиксировать версию: `openclaw plugins install @openclaw/zalouser@<version>`
- Из рабочей копии исходного кода: `openclaw plugins install ./path/to/local/zalouser-plugin`
- Подробнее: [Плагины](/ru/tools/plugin)

## Быстрая настройка

1. Установите плагин (см. выше).
2. Войдите в систему (по QR-коду на машине с Gateway):
   - `openclaw channels login --channel zalouser`
   - Отсканируйте QR-код в мобильном приложении Zalo.
3. Включите канал:

```json5
{
  channels: {
    zalouser: {
      enabled: true,
      dmPolicy: "pairing",
    },
  },
}
```

4. Перезапустите Gateway (или завершите настройку).
5. По умолчанию доступ к личным сообщениям предоставляется через сопряжение; при первом обращении подтвердите код сопряжения.

## Что это такое

- Полностью работает внутри процесса через библиотеку `zca-js` (без внешнего исполняемого файла `zca`/`openzca`).
- Использует нативные обработчики событий (`message`, `error`) для получения входящих сообщений.
- Отправляет ответы напрямую через API JavaScript (текст, медиафайлы и ссылки).
- Предназначена для сценариев с личной учетной записью, когда Zalo Bot API недоступен.

## Именование

Идентификатор канала — `zalouser`, чтобы явно указать, что автоматизируется **личная учетная запись пользователя Zalo** (неофициально). Идентификатор `zalo` зарезервирован для возможной будущей официальной интеграции с API Zalo.

## Поиск идентификаторов (каталог)

```bash
openclaw directory self --channel zalouser
openclaw directory peers list --channel zalouser --query "name"
openclaw directory groups list --channel zalouser --query "work"
```

## Ограничения

- Исходящий текст разбивается на фрагменты по 2000 символов (ограничение клиента Zalo).
- Потоковая передача не поддерживается.

## Управление доступом (личные сообщения)

`channels.zalouser.dmPolicy`: `pairing | allowlist | open | disabled` (по умолчанию: `pairing`).

В `channels.zalouser.allowFrom` следует указывать стабильные идентификаторы пользователей Zalo. Также можно ссылаться на статические группы доступа отправителей (`accessGroup:<name>`). Во время интерактивной настройки введенные имена можно преобразовать в идентификаторы с помощью встроенного в процесс поиска контактов плагина.

Если в конфигурации остается необработанное имя, при запуске оно преобразуется только при включенном параметре `channels.zalouser.dangerouslyAllowNameMatching: true`. Без этого явного разрешения проверки отправителей во время выполнения выполняются только по идентификаторам, а необработанные имена не учитываются при авторизации.

Подтверждение выполняется следующими командами:

- `openclaw pairing list zalouser`
- `openclaw pairing approve zalouser <code>`

## Доступ к группам (необязательно)

- По умолчанию: `channels.zalouser.groupPolicy = "allowlist"` (для групп требуется явная запись в списке разрешенных).
- Открыть все группы: `channels.zalouser.groupPolicy = "open"`.
- Заблокировать все группы: `channels.zalouser.groupPolicy = "disabled"`.
- При `groupPolicy = "allowlist"`:
  - Ключами `channels.zalouser.groups` должны быть стабильные идентификаторы групп; имена преобразуются в идентификаторы при запуске только при включенном параметре `channels.zalouser.dangerouslyAllowNameMatching: true`.
  - `channels.zalouser.groupAllowFrom` определяет, какие отправители в разрешенных группах могут активировать бота; на статические группы доступа отправителей можно ссылаться с помощью `accessGroup:<name>`.
- Мастер настройки может запросить списки разрешенных групп.
- По умолчанию сопоставление со списком разрешенных групп выполняется только по идентификаторам. Неразрешенные имена не учитываются при авторизации, если не включен параметр `channels.zalouser.dangerouslyAllowNameMatching: true`.
- `channels.zalouser.dangerouslyAllowNameMatching: true` — аварийный режим совместимости, который повторно включает преобразование изменяемых имен при запуске и сопоставление имен групп во время выполнения.
- Для обычных групповых сообщений `groupAllowFrom` **не** использует `allowFrom` в качестве резервного значения: если оставить его пустым для группы из списка разрешенных, любой отправитель в этой группе сможет активировать бота. Исключение составляют авторизованные управляющие команды (например, `/new`): если `groupAllowFrom` пуст, отправитель команды проверяется по `allowFrom`.

Пример:

```json5
{
  channels: {
    zalouser: {
      groupPolicy: "allowlist",
      groupAllowFrom: ["1471383327500481391"],
      groups: {
        "123456789": { enabled: true },
        "Work Chat": { enabled: true },
      },
    },
  },
}
```

<Note>
`channels.zalouser.groups.<id>.allow` — устаревшее имя поля; в текущей конфигурации используется `enabled`. Команда `openclaw doctor --fix` автоматически переносит значение `allow` в `enabled`.
</Note>

### Ограничение по упоминаниям в группах

- `channels.zalouser.groups.<group>.requireMention` определяет, требуется ли упоминание для ответа в группе.
- Порядок разрешения: идентификатор группы -> псевдоним `group:<id>` -> имя/короткое имя группы (варианты на основе имени применяются только при `dangerouslyAllowNameMatching: true`) -> `*` -> значение по умолчанию (`true`).
- Применяется как к группам из списка разрешенных, так и в режиме открытых групп.
- Цитирование сообщения бота считается неявным упоминанием для активации в группе.
- Авторизованные управляющие команды (например, `/new`) могут обходить требование упоминания.
- Если групповое сообщение пропускается из-за обязательного упоминания, OpenClaw сохраняет его в отложенной истории группы и включает в следующее обработанное групповое сообщение.
- Ограничение истории группы: `channels.zalouser.historyLimit`, затем `messages.groupChat.historyLimit`, затем резервное значение `50`.

Пример:

```json5
{
  channels: {
    zalouser: {
      groupPolicy: "allowlist",
      groups: {
        "*": { enabled: true, requireMention: true },
        "Work Chat": { enabled: true, requireMention: false },
      },
    },
  },
}
```

## Несколько учетных записей

Учетные записи сопоставляются с профилями `zalouser` в состоянии OpenClaw. Пример:

```json5
{
  channels: {
    zalouser: {
      enabled: true,
      defaultAccount: "default",
      accounts: {
        work: { enabled: true, profile: "work" },
      },
    },
  },
}
```

## Переменные окружения

Профиль также можно выбирать с помощью переменных окружения:

| Переменная         | Назначение                                                                                                    |
| ------------------ | ------------------------------------------------------------------------------------------------------------- |
| `ZALOUSER_PROFILE` | Имя используемого профиля, если `profile` не указан в конфигурации канала или учетной записи.                  |
| `ZCA_PROFILE`      | Устаревшее резервное значение, используемое только в том случае, если переменная `ZALOUSER_PROFILE` не задана. |

Имена профилей выбирают сохраненные учетные данные для входа в Zalo из состояния OpenClaw. Порядок разрешения:

1. Явно заданный `profile` в конфигурации.
2. `ZALOUSER_PROFILE`.
3. `ZCA_PROFILE`.
4. Идентификатор учетной записи для учетных записей не по умолчанию или `default` для учетной записи по умолчанию.

При настройке нескольких учетных записей рекомендуется задавать `profile` для каждой учетной записи в конфигурации, чтобы одна переменная окружения не приводила к совместному использованию одного сеанса входа несколькими учетными записями.

## Индикатор набора текста, реакции и подтверждения доставки

- Перед отправкой ответа OpenClaw по возможности отправляет событие набора текста.
- Действие реакции на сообщение `react` поддерживается для `zalouser` в действиях канала.
  - Используйте `remove: true`, чтобы удалить определенную реакцию-эмодзи из сообщения.
  - Семантика реакций: [Реакции](/ru/tools/reactions)
- Для входящих сообщений, содержащих метаданные событий, OpenClaw по возможности отправляет подтверждения доставки и просмотра.

## Устранение неполадок

**Вход в систему не сохраняется:**

- `openclaw channels status --probe`
- Повторный вход: `openclaw channels logout --channel zalouser && openclaw channels login --channel zalouser`

**Имя в списке разрешенных или имя группы не преобразовалось:**

- Используйте числовые идентификаторы в `allowFrom`/`groupAllowFrom` и стабильные идентификаторы групп в `groups`. Если вам намеренно требуется использовать точные имена друзей или групп, включите `channels.zalouser.dangerouslyAllowNameMatching: true`.

**Обновление со старой внешней конфигурации на основе `zca`/CLI:**

- Удалите все предположения о наличии внешнего процесса `zca`; теперь канал полностью работает внутри процесса через `zca-js`, без внешнего исполняемого файла CLI.
