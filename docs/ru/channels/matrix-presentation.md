OpenClaw прикрепляет нормализованные метаданные `MessagePresentation` к исходящим событиям Matrix `m.room.message` в ключе содержимого `com.openclaw.presentation`.

Стандартные клиенты Matrix продолжают отображать обычный текст из `body`. Клиенты с поддержкой OpenClaw могут считывать структурированные метаданные и отображать нативные элементы интерфейса, такие как кнопки, списки выбора, контекстные строки и разделители.

## Содержимое события

```json
{
  "msgtype": "m.text",
  "body": "Select model\n\nChoose model:\n- DeepSeek",
  "com.openclaw.presentation": {
    "version": 1,
    "type": "message.presentation",
    "title": "Select model",
    "tone": "info",
    "blocks": [
      {
        "type": "select",
        "placeholder": "Choose model",
        "options": [
          {
            "label": "DeepSeek",
            "value": "/model deepseek/deepseek-chat"
          }
        ]
      }
    ]
  }
}
```

- `version` — версия схемы метаданных; текущая версия — `1`. `type` — стабильный дискриминатор, всегда равный `"message.presentation"`. Адаптер Matrix отправляет только полезные нагрузки с точно такой версией и типом; клиентам также следует игнорировать неизвестные версии, которые они не могут безопасно интерпретировать, неизвестные значения `type` и неизвестные типы блоков.
- `title` и `tone` (`info`, `success`, `warning`, `danger`, `neutral`) — необязательные подсказки.
- Кнопки и варианты выбора могут содержать типизированное действие `action` (`{ "type": "command", "command": "/..." }` или `{ "type": "callback", "value": "..." }`) наряду с устаревшим строковым значением `value`. Если присутствуют оба поля, отдавайте предпочтение `action`.

## Резервное поведение

OpenClaw всегда формирует в `body` удобочитаемый резервный вариант в виде обычного текста. Структурированные метаданные являются дополнением и не должны требоваться для базовой совместимости с Matrix.

Правила резервного отображения:

- Содержимое `title`, `text` и `context` отображается обычными строками.
- Кнопки с действием `command` отображаются как ``метка: `/команда` ``, чтобы команду можно было скопировать. Кнопки с действием `callback` или только с устаревшим значением `value` отображаются лишь в виде метки, чтобы непрозрачные значения обратного вызова оставались скрытыми; отключённые кнопки всегда отображаются только в виде метки. Кнопки URL и веб-приложений отображаются как `метка: URL`.
- Блоки выбора отображают замещающий текст (или `Варианты:`) в качестве заголовка, а затем строки вариантов, содержащие только метки.
- Если ничего не отображается, например в представлении только с разделителем, содержимое `body` заменяется на `---`.

Неподдерживающие клиенты продолжают показывать резервный текст. Клиенты с поддержкой OpenClaw могут отдавать предпочтение структурированным метаданным при отображении, сохраняя резервный вариант для копирования, поиска, уведомлений и специальных возможностей.

## Поддерживаемые блоки

Исходящий адаптер Matrix заявляет нативную поддержку следующих блоков:

- `buttons`
- `select`
- `context`
- `divider`

Блоки `text` всегда поддерживаются через резервное содержимое `body`. Рассматривайте все блоки как необязательные подсказки для отображения; игнорируйте неизвестные поля и типы блоков вместо того, чтобы отклонять всё сообщение.

## Взаимодействия

Эти метаданные не добавляют в Matrix семантику обратных вызовов. Значения кнопок и вариантов выбора являются резервными полезными нагрузками взаимодействия — обычно это команды с косой чертой или текстовые команды. Клиент Matrix, желающий поддерживать взаимодействие, определяет значение элемента управления (`action.command`, затем `action.value`, затем `value`) и отправляет его обратно в комнату как обычное сообщение.

Например, кнопку со значением `/model deepseek/deepseek-chat` можно обработать, отправив это значение как зашифрованное текстовое сообщение Matrix в ту же комнату.

## Связь с метаданными подтверждения

`com.openclaw.presentation` предназначены для универсального расширенного представления сообщений.

Запросы подтверждения используют специальные метаданные `com.openclaw.approval`, поскольку подтверждения содержат важные для безопасности состояния, решения и сведения о выполнении команд или Plugin. Если в одном событии присутствуют оба ключа метаданных, клиентам следует отдавать предпочтение специальному средству отображения подтверждений.

## Медиасообщения

Если ответ содержит несколько URL-адресов медиафайлов, OpenClaw отправляет отдельное событие Matrix для каждого URL-адреса. Текст подписи и метаданные представления прикрепляются только к первому событию, чтобы клиенты получали одну стабильную структурированную полезную нагрузку без дублирования средств отображения. То же правило применяется при разбиении длинного текста между событиями: метаданные передаются только в первом событии.

Сохраняйте метаданные представления компактными. Большой видимый пользователю текст должен оставаться в `body` и передаваться через обычный механизм разбиения текста Matrix.
