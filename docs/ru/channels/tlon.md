Tlon — это децентрализованный мессенджер, построенный на Urbit. OpenClaw подключается к вашему кораблю Urbit и
отвечает на личные сообщения и сообщения в групповых чатах. По умолчанию для ответов в группах требуется упоминание через @,
а поверх этого действуют правила авторизации и процесс одобрения владельцем.

Статус: встроенный плагин. Поддерживаются личные сообщения, упоминания в группах, ветки обсуждений, форматированный текст, отправка и скачивание изображений, а также
система одобрения владельцем. Реакции и опросы не поддерживаются.

## Встроенный плагин

Tlon входит в состав текущих выпусков OpenClaw; для пакетных сборок отдельная установка не требуется.

В более старой или пользовательской сборке, из которой он исключён, установите пакет из npm:

```bash
openclaw plugins install @openclaw/tlon
```

Используйте имя пакета без версии, чтобы отслеживать тег текущего выпуска. Закрепляйте версию (`@openclaw/tlon@x.y.z`)
только для воспроизводимых установок.

Из локальной рабочей копии:

```bash
openclaw plugins install ./path/to/local/tlon-plugin
```

Подробнее: [Плагины](/ru/tools/plugin)

## Настройка

```bash
openclaw channels add --channel tlon --ship ~sampel-palnet --url https://your-ship-host --code lidlut-tabwed-pillex-ridrup
```

Или измените конфигурацию напрямую:

```json5
{
  channels: {
    tlon: {
      enabled: true,
      ship: "~sampel-palnet",
      url: "https://your-ship-host",
      code: "lidlut-tabwed-pillex-ridrup",
      ownerShip: "~your-main-ship", // рекомендуется: ваш корабль, всегда авторизован
    },
  },
}
```

После прямого изменения конфигурации перезапустите Gateway. Затем отправьте боту личное сообщение или упомяните его через @ в групповом
канале.

## Приватные корабли и корабли в локальной сети

По умолчанию OpenClaw блокирует приватные и внутренние имена хостов и диапазоны IP-адресов для защиты от SSRF. Если ваш
корабль работает в частной сети (localhost, IP-адрес локальной сети, внутреннее имя хоста), явно разрешите доступ:

```json5
{
  channels: {
    tlon: {
      url: "http://localhost:8080",
      network: {
        dangerouslyAllowPrivateNetwork: true,
      },
    },
  },
}
```

Это относится к адресам вроде `http://localhost:8080`, `http://192.168.x.x:8080` и
`http://my-ship.local:8080`. Включайте этот параметр только для доверенного URL корабля; он отключает защиту от SSRF
для HTTP-запросов этой учётной записи.

<Note>
`channels.tlon.allowPrivateNetwork` (ключ верхнего уровня) больше не используется. `openclaw doctor --fix` автоматически переносит его в
`channels.tlon.network.dangerouslyAllowPrivateNetwork`.
</Note>

## Групповые каналы

Закрепите каналы вручную или включите автоматическое обнаружение:

```json5
{
  channels: {
    tlon: {
      groupChannels: ["chat/~host-ship/general", "chat/~host-ship/support"],
      autoDiscoverChannels: true,
    },
  },
}
```

Если параметр `autoDiscoverChannels` не указан в конфигурации, по умолчанию используется значение `false`; мастер настройки по умолчанию предлагает
ответить утвердительно и явно записывает `true`. Когда параметр включён, OpenClaw при запуске выполняет scry-запросы для групп, к которым присоединился,
начинает отслеживать новые каналы после принятия приглашений в группы и повторяет проверку каждые 2 минуты.

## Управление доступом

Список разрешённых отправителей личных сообщений (пустой список = личные сообщения запрещены всем, кроме `ownerShip`):

```json5
{
  channels: {
    tlon: {
      dmAllowlist: ["~zod", "~nec"],
    },
  },
}
```

По умолчанию для каждого канала используется режим групповой авторизации `restricted`. Задайте `defaultAuthorizedShips` в качестве
базового списка и переопределите его для отдельных гнёзд каналов:

```json5
{
  channels: {
    tlon: {
      defaultAuthorizedShips: ["~zod"],
      authorization: {
        channelRules: {
          "chat/~host-ship/general": {
            mode: "restricted",
            allowedShips: ["~zod", "~nec"],
          },
          "chat/~host-ship/announcements": {
            mode: "open",
          },
        },
      },
    },
  },
}
```

После того как бот ответил внутри ветки обсуждения, он продолжает отвечать на последующие сообщения в этой ветке
без нового упоминания.

## Владелец и система одобрения

```json5
{
  channels: {
    tlon: {
      ownerShip: "~your-main-ship",
    },
  },
}
```

Корабль владельца авторизован везде: приглашения в личные сообщения всегда принимаются автоматически, приглашения в группы
всегда принимаются автоматически, а сообщения в каналах всегда проходят авторизацию. Корабль владельца не требуется
добавлять в `dmAllowlist`, `defaultAuthorizedShips` или `groupInviteAllowlist`.

Когда задан параметр `ownerShip`, неавторизованные запросы не просто отклоняются — они помещаются в очередь ожидающих
одобрения, а владельцу отправляется личное сообщение:

- Запросы личных сообщений от кораблей, отсутствующих в `dmAllowlist`
- Упоминания в каналах, где отправитель не проходит авторизацию
- Приглашения в группы от кораблей, отсутствующих в `groupInviteAllowlist` (когда автоматическое принятие выключено либо включено, но
  пригласивший корабль отсутствует в списке разрешённых)

Чтобы обработать запрос, владелец отвечает в личных сообщениях:

| Ответ владельца              | Результат                                                    |
| ---------------------------- | ------------------------------------------------------------ |
| `approve` / `deny` / `block` | Обрабатывает последний ожидающий одобрения запрос             |
| `approve <id>` / `deny <id>` | Обрабатывает конкретный запрос на одобрение по идентификатору |
| `block`                      | Также блокирует корабль штатными средствами, не позволяя ему подключиться повторно |
| `unblock ~ship`              | Снимает штатную блокировку                                   |
| `blocked`                    | Показывает список заблокированных в данный момент кораблей   |
| `pending`                    | Показывает ожидающие запросы на одобрение                     |

Если параметр `ownerShip` не настроен, неавторизованные личные сообщения и упоминания в каналах просто отклоняются и записываются в журнал;
запрос на одобрение не создаётся.

## Настройки автоматического принятия

Автоматически принимать приглашения в личные сообщения от кораблей, уже добавленных в `dmAllowlist` (приглашения от владельца всегда принимаются
автоматически независимо от этого параметра):

```json5
{
  channels: {
    tlon: {
      autoAcceptDmInvites: true,
    },
  },
}
```

Автоматически принимать приглашения в группы от кораблей из списка разрешённых (безопасное отклонение по умолчанию: при `autoAcceptGroupInvites: true` и
пустом `groupInviteAllowlist` не принимается ни одно приглашение не от владельца):

```json5
{
  channels: {
    tlon: {
      autoAcceptGroupInvites: true,
      groupInviteAllowlist: ["~zod"],
    },
  },
}
```

## Горячая перезагрузка через хранилище настроек Urbit

Большинство перечисленных выше настроек (`dmAllowlist`, `groupInviteAllowlist`, `groupChannels`,
`defaultAuthorizedShips`, `autoDiscoverChannels`, `autoAcceptDmInvites`,
`autoAcceptGroupInvites`, `ownerShip`, `showModelSignature`) при первом запуске копируются в агент
`%settings` корабля (деск `moltbot`, корзина `tlon`), а затем считываются оттуда в реальном времени,
поэтому изменения, внесённые через клиент Landscape или команды настроек встроенного навыка, применяются без
перезапуска Gateway. `channelRules` и ожидающие запросы на одобрение также сохраняются там в формате JSON. Файловая
конфигурация остаётся источником истины для значений, которые никогда не записывались в хранилище настроек.

## Цели доставки (CLI/Cron)

Используйте с `openclaw message send` или доставкой через Cron:

- Личное сообщение: `~sampel-palnet` или `dm/~sampel-palnet`
- Группа: `chat/~host-ship/channel` или `group:~host-ship/channel`

## Встроенный навык

Плагин включает [`@tloncorp/tlon-skill`](https://github.com/tloncorp/tlon-skill) — CLI для
прямых операций с Urbit, который становится доступен автоматически после установки плагина:

- **Активность**: упоминания, ответы, непрочитанные сообщения
- **Каналы**: просмотр списка, создание, переименование
- **Контакты**: просмотр списка, получение и обновление профилей
- **Группы**: создание, присоединение, процессы приглашения и отправки запросов, роли
- **Перехватчики**: управление перехватчиками каналов
- **Сообщения**: история, поиск
- **Личные сообщения**: отправка, реакции, принятие и отклонение
- **Публикации**: реакции, удаление
- **Блокнот**: публикация в каналах дневника
- **Настройки**: горячая перезагрузка конфигурации плагина через описанное выше хранилище настроек

## Возможности

| Функция          | Статус                                                        |
| ---------------- | ------------------------------------------------------------- |
| Личные сообщения | Поддерживаются                                                |
| Группы/каналы    | Поддерживаются (по умолчанию требуется упоминание)            |
| Ветки обсуждений | Поддерживаются (после присоединения бот продолжает отвечать)  |
| Форматированный текст | Markdown преобразуется в собственный формат Tlon         |
| Изображения      | Входящие скачиваются, исходящие загружаются                    |
| Реакции          | Только через [встроенный навык](#bundled-skill)                |
| Опросы           | Не поддерживаются                                             |
| Встроенные команды | По умолчанию доступны только владельцу                      |

## Устранение неполадок

```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
```

Распространённые проблемы:

- **Личные сообщения игнорируются**: отправитель отсутствует в `dmAllowlist`, а `ownerShip` не настроен для процесса одобрения.
- **Сообщения в группах игнорируются**: канал не обнаружен и не закреплён либо отправитель не проходит авторизацию, а
  `ownerShip` не задан для постановки запроса на одобрение в очередь.
- **Ошибки подключения**: убедитесь, что URL корабля доступен; для локальных кораблей задайте
  `network.dangerouslyAllowPrivateNetwork`.
- **Ошибки аутентификации**: коды входа периодически меняются — скопируйте текущий код со своего корабля.

## Справочник по конфигурации

Полная конфигурация: [Конфигурация](/ru/gateway/configuration)

| Ключ                                                   | Значение                                                              |
| ------------------------------------------------------ | --------------------------------------------------------------------- |
| `channels.tlon.enabled`                                | Включает или отключает запуск канала.                                 |
| `channels.tlon.ship`                                   | Имя корабля Urbit, используемого ботом (например, `~sampel-palnet`).   |
| `channels.tlon.url`                                    | URL корабля (например, `https://sampel-palnet.tlon.network`).          |
| `channels.tlon.code`                                   | Код входа на корабль.                                                 |
| `channels.tlon.network.dangerouslyAllowPrivateNetwork` | Разрешает URL кораблей на localhost или в локальной сети (явное разрешение SSRF). |
| `channels.tlon.ownerShip`                              | Корабль владельца: всегда авторизован и получает запросы на одобрение. |
| `channels.tlon.dmAllowlist`                            | Корабли, которым разрешены личные сообщения (пустой список = никто, кроме владельца). |
| `channels.tlon.autoAcceptDmInvites`                    | Автоматически принимает личные сообщения от кораблей из `dmAllowlist`. |
| `channels.tlon.autoAcceptGroupInvites`                 | Автоматически принимает приглашения в группы от кораблей из `groupInviteAllowlist`. |
| `channels.tlon.groupInviteAllowlist`                   | Корабли, приглашения в группы от которых принимаются автоматически.   |
| `channels.tlon.autoDiscoverChannels`                   | Автоматически обнаруживает групповые каналы, к которым выполнено присоединение (по умолчанию: `false`). |
| `channels.tlon.groupChannels`                          | Закреплённые вручную гнёзда каналов.                                  |
| `channels.tlon.defaultAuthorizedShips`                 | Корабли, авторизованные для всех каналов (используются, если ни одно правило не совпало). |
| `channels.tlon.authorization.channelRules`             | Режим авторизации и список разрешённых для каждого гнезда канала.      |
| `channels.tlon.showModelSignature`                     | Добавляет `_[Сгенерировано моделью <model>]_` к ответам.               |
| `channels.tlon.responsePrefix`                         | Статический префикс, добавляемый перед исходящими ответами.            |
| `channels.tlon.accounts.<id>`                          | Дополнительные именованные учётные записи (конфигурации с несколькими кораблями). |

## Примечания

- Для ответов в группах требуется упоминание через @ (например, `~your-bot-ship`), если бот ещё не присоединился к этой ветке обсуждения.
- Ответы на сообщения в ветках публикуются в той же ветке; бот также получает последние 10 сообщений из контекста ветки,
  добавленные перед запросом агента.
- Форматированный текст (полужирный, курсив, код, заголовки, списки) преобразуется в собственный формат Tlon.
- Входящее сообщение с запросом сводки по каналу (например, «кратко изложи содержимое этого
  канала») запускает встроенное создание сводки по истории вместо обычного процесса ответа.
