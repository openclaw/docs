Фоновые события комнаты позволяют OpenClaw обрабатывать сообщения группового чата или канала без упоминания как ненавязчивый контекст. Агент может обновлять память и состояние сеанса, но в комнате сохраняется тишина, если агент явно не вызывает инструмент `message`.

Для постоянно активных групповых чатов сочетайте `messages.groupChat.unmentionedInbound: "room_event"` с `messages.groupChat.visibleReplies: "message_tool"`. Агент слушает, решает, когда ответ будет полезен, и больше не нуждается в прежнем шаблоне промпта с ответом `NO_REPLY`.

В настоящее время поддерживаются каналы серверов Discord, публичные и приватные каналы Slack, групповые личные сообщения Slack, а также группы и супергруппы Telegram. Другие групповые каналы сохраняют существующее групповое поведение, если на странице соответствующего канала не указано, что он поддерживает фоновые события комнаты.

## Рекомендуемая настройка

Задайте глобальное поведение групповых чатов:

```json5
{
  messages: {
    groupChat: {
      unmentionedInbound: "room_event",
      visibleReplies: "message_tool",
      historyLimit: 50,
    },
  },
}
```

Затем сделайте комнату постоянно активной, отключив для неё требование упоминания. Комната по-прежнему должна соответствовать обычной политике `groupPolicy`, списку разрешённых комнат и списку разрешённых отправителей.

После сохранения конфигурации Gateway применяет настройки `messages` без перезапуска. Перезапуск требуется только тогда, когда наблюдение за файлами или перезагрузка конфигурации отключены (`gateway.reload.mode: "off"`).

## Что изменяется

При `messages.groupChat.unmentionedInbound: "room_event"`:

- разрешённые сообщения групп или каналов без упоминания становятся ненавязчивыми событиями комнаты
- сообщения с упоминанием остаются пользовательскими запросами
- текстовые управляющие команды и встроенные команды остаются пользовательскими запросами
- запросы на прерывание или остановку остаются пользовательскими запросами
- личные сообщения остаются пользовательскими запросами

Для событий комнаты применяется строгий режим видимой доставки. Итоговый текст ассистента остаётся приватным. Чтобы опубликовать сообщение в комнате, агент должен вызвать `message(action=send)`.

Для событий комнаты индикатор набора текста и реакции на состояния жизненного цикла остаются отключёнными. Единственное явное исключение для подтверждения получения — `messages.ackReactionScope: "all"`, которое отправляет настроенную реакцию подтверждения; если в комнате должна сохраняться полная тишина, используйте более узкую область или `"off"`.

## Пример для Discord

```json5
{
  messages: {
    groupChat: {
      unmentionedInbound: "room_event",
      visibleReplies: "message_tool",
      historyLimit: 50,
    },
  },
  channels: {
    discord: {
      groupPolicy: "allowlist",
      guilds: {
        "<DISCORD_SERVER_ID>": {
          requireMention: false,
          users: ["<YOUR_DISCORD_USER_ID>"],
        },
      },
    },
  },
}
```

Используйте конфигурацию Discord для отдельного канала, если фоновый режим нужен только в одном канале. При `groupPolicy: "allowlist"` канал разрешается его добавлением в список (`enabled: false` отключает запись):

```json5
{
  channels: {
    discord: {
      groupPolicy: "allowlist",
      guilds: {
        "<DISCORD_SERVER_ID>": {
          channels: {
            "<DISCORD_CHANNEL_ID_OR_NAME>": {
              requireMention: false,
            },
          },
        },
      },
    },
  },
}
```

## Пример для Slack

В списках разрешённых каналов Slack приоритет имеют идентификаторы. Используйте идентификаторы каналов, например `C12345678`, а не `#channel-name`. Канал разрешается его добавлением в `channels.slack.channels` (`enabled: false` отключает запись):

```json5
{
  messages: {
    groupChat: {
      unmentionedInbound: "room_event",
      visibleReplies: "message_tool",
      historyLimit: 50,
    },
  },
  channels: {
    slack: {
      groupPolicy: "allowlist",
      channels: {
        "<SLACK_CHANNEL_ID>": {
          requireMention: false,
        },
      },
    },
  },
}
```

## Пример для Telegram

Для групп Telegram бот должен иметь возможность видеть обычные групповые сообщения. Если задано `requireMention: false`, отключите режим конфиденциальности BotFather или используйте другую настройку Telegram, которая передаёт боту весь поток групповых сообщений.

```json5
{
  messages: {
    groupChat: {
      unmentionedInbound: "room_event",
      visibleReplies: "message_tool",
      historyLimit: 50,
    },
  },
  channels: {
    telegram: {
      groups: {
        "<TELEGRAM_GROUP_CHAT_ID>": {
          groupPolicy: "open",
          requireMention: false,
        },
      },
    },
  },
}
```

Идентификаторы групп Telegram обычно являются отрицательными числами, например `-1001234567890`. Получите `chat.id` из вывода `openclaw logs --follow`, перешлите групповое сообщение боту для определения идентификатора или проверьте `getUpdates` в Bot API.

## Политика для отдельного агента

Используйте переопределение агента, если одну комнату совместно используют несколько агентов, но только один из них должен воспринимать сообщения без упоминания как фоновый контекст:

```json5
{
  messages: {
    groupChat: {
      visibleReplies: "message_tool",
    },
  },
  agents: {
    list: [
      {
        id: "main",
        groupChat: {
          unmentionedInbound: "room_event",
          mentionPatterns: ["@openclaw", "openclaw"],
        },
      },
    ],
  },
}
```

Значение `agents.list[].groupChat.unmentionedInbound` конкретного агента переопределяет для него `messages.groupChat.unmentionedInbound`.

## Режимы видимых ответов

Для обычных пользовательских запросов в группах и каналах значение `messages.groupChat.visibleReplies` по умолчанию равно `"automatic"`. Сохраняйте это значение по умолчанию, если итоговый текст ассистента должен публиковаться в открытом виде без явного вызова инструмента отправки сообщений.

Для постоянно активных комнат с фоновым режимом по-прежнему рекомендуется `messages.groupChat.visibleReplies: "message_tool"`, особенно с моделями последнего поколения, надёжно работающими с инструментами, например GPT-5.6 Sol. Этот режим позволяет агенту самостоятельно решать, когда следует ответить, вызывая инструмент отправки сообщений. Если модель возвращает итоговый текст без вызова инструмента, OpenClaw сохраняет его приватным и регистрирует метаданные о подавленной доставке.

Для событий комнаты строгий режим сохраняется, даже если для других групповых запросов используются автоматические ответы. Для видимого вывода фоновых событий комнаты без упоминания всегда требуется `message(action=send)`.

## История

`messages.groupChat.historyLimit` задаёт глобальный предел истории групп по умолчанию (50, если значение не задано; значение должно быть положительным целым числом). Каналы могут переопределять его с помощью `channels.<channel>.historyLimit`, а некоторые каналы также поддерживают ограничения истории для отдельных учётных записей. Чтобы отключить контекст истории групп для канала, задайте на уровне канала `historyLimit: 0`.

Каналы, поддерживающие события комнаты, сохраняют недавние фоновые сообщения комнаты в качестве контекста. Telegram хранит постоянно обновляемое скользящее окно для каждой группы, ограниченное значением `historyLimit`; для пользовательских запросов выбираются записи после последнего зарегистрированного ответа бота, а события комнаты получают всё недавнее окно, чтобы модель могла видеть собственные недавние публикации. Устаревший ключ режима Telegram `includeGroupHistoryContext` удаляется командой `openclaw doctor --fix`.

## Устранение неполадок

Если в комнате отображается набор текста или расход токенов, но видимое сообщение отсутствует:

1. Убедитесь, что комната разрешена списком разрешённых каналов и списком разрешённых отправителей.
2. Убедитесь, что `requireMention: false` задано на нужном уровне комнаты.
3. Проверьте, имеет ли `messages.groupChat.unmentionedInbound` или переопределение агента значение `"room_event"`.
4. Проверьте журналы на наличие метаданных подавленного итогового содержимого или `didSendViaMessagingTool: false`.
5. Для обычных групповых запросов сохраните или восстановите `messages.groupChat.visibleReplies: "automatic"`, если хотите, чтобы итоговые ответы публиковались автоматически. Для фоновых комнат, использующих `message_tool`, используйте модель и среду выполнения, которые надёжно вызывают инструменты.

Если фоновые комнаты Telegram вообще не срабатывают, проверьте режим конфиденциальности BotFather и убедитесь, что Gateway получает обычные групповые сообщения.

Если фоновые комнаты Slack не срабатывают, убедитесь, что ключом канала является идентификатор канала Slack и у приложения есть область доступа к истории для соответствующего типа комнаты: `channels:history` для публичных каналов, `groups:history` для приватных или `mpim:history` для групповых личных сообщений.
