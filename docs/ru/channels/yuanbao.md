Tencent Yuanbao — платформа ИИ-ассистента от Tencent. Поддерживаемый сообществом Plugin `openclaw-plugin-yuanbao` подключает ботов Yuanbao к OpenClaw через WebSocket для личных сообщений и групповых чатов.

**Статус:** готово к использованию в рабочей среде для личных сообщений боту и групповых чатов. WebSocket — единственный поддерживаемый режим подключения. Этот Plugin поддерживается командой Tencent Yuanbao как внешняя запись каталога, а не основной командой OpenClaw; приведённые ниже сведения о конфигурации и поведении (кроме установки и общей поверхности CLI) взяты из собственной документации Plugin и не сверялись с исходным кодом ядра OpenClaw.

## Быстрый старт

Требуется OpenClaw версии 2026.4.10 или выше. Проверьте версию с помощью `openclaw --version`; обновите с помощью `openclaw update`.

<Steps>
  <Step title="Добавьте канал Yuanbao с учётными данными">
  ```bash
  openclaw channels add --channel yuanbao --token "appKey:appSecret"
  ```
  В `--token` используется разделённая двоеточием пара `appKey:appSecret`. Получите эти значения в приложении Yuanbao, создав бота в настройках приложения.
  </Step>

  <Step title="Перезапустите Gateway, чтобы применить изменение">
  ```bash
  openclaw gateway restart
  ```
  </Step>
</Steps>

### Интерактивная настройка (альтернативный способ)

```bash
openclaw channels login --channel yuanbao
```

Следуйте подсказкам, чтобы ввести App ID и App Secret.

## Управление доступом

### Личные сообщения

`channels.yuanbao.dm.policy`:

| Значение         | Поведение                                                          |
| ---------------- | ------------------------------------------------------------------ |
| `open` (по умолчанию) | Разрешить всех пользователей                                  |
| `pairing`        | Неизвестные пользователи получают код сопряжения; подтвердите через CLI |
| `allowlist`      | Общаться могут только пользователи из `allowFrom`                  |
| `disabled`       | Отключить все личные сообщения                                     |

Подтвердите запрос на сопряжение:

```bash
openclaw pairing list yuanbao
openclaw pairing approve yuanbao <CODE>
```

### Групповые чаты

`channels.yuanbao.requireMention` (по умолчанию `true`): требует @упоминания, прежде чем бот ответит в группе. Ответ на собственное сообщение бота считается неявным упоминанием.

## Примеры конфигурации

Базовая настройка с открытой политикой личных сообщений:

```json5
{
  channels: {
    yuanbao: {
      appKey: "your_app_key",
      appSecret: "your_app_secret",
      dm: {
        policy: "open",
      },
    },
  },
}
```

Ограничение личных сообщений определёнными пользователями:

```json5
{
  channels: {
    yuanbao: {
      appKey: "your_app_key",
      appSecret: "your_app_secret",
      dm: {
        policy: "allowlist",
        allowFrom: ["user_id_1", "user_id_2"],
      },
    },
  },
}
```

Отключение требования @упоминания в группах:

```json5
{
  channels: {
    yuanbao: {
      requireMention: false,
    },
  },
}
```

Настройка исходящей доставки:

```json5
{
  channels: {
    yuanbao: {
      outboundQueueStrategy: "merge-text",
      minChars: 2800, // накапливать до указанного количества символов
      maxChars: 3000, // принудительно разделять при превышении этого ограничения
      idleMs: 5000, // автоматически отправлять после периода бездействия (мс)
    },
  },
}
```

Установите `outboundQueueStrategy: "immediate"`, чтобы отправлять каждый фрагмент без буферизации.

## Основные команды

| Команда    | Описание                            |
| ---------- | ----------------------------------- |
| `/help`    | Показать доступные команды          |
| `/status`  | Показать состояние бота             |
| `/new`     | Начать новый сеанс                  |
| `/stop`    | Остановить текущий запуск           |
| `/restart` | Перезапустить OpenClaw              |
| `/compact` | Сжать контекст сеанса               |

Yuanbao поддерживает встроенные меню команд с косой чертой; команды автоматически синхронизируются с платформой при запуске Gateway.

## Устранение неполадок

**Бот не отвечает в групповых чатах:**

1. Убедитесь, что бот добавлен в группу
2. Убедитесь, что вы @упомянули бота (по умолчанию это обязательно)
3. Проверьте журналы: `openclaw logs --follow`

**Бот не получает сообщения:**

1. Убедитесь, что бот создан и одобрен в приложении Yuanbao
2. Убедитесь, что `appKey` и `appSecret` настроены правильно
3. Убедитесь, что Gateway запущен: `openclaw gateway status`
4. Проверьте журналы: `openclaw logs --follow`

**Бот отправляет пустые или резервные ответы:**

1. Проверьте, возвращает ли модель ИИ допустимое содержимое
2. Резервный ответ по умолчанию: "暂时无法解答，你可以换个问题问问我哦"
3. Настройте его с помощью `channels.yuanbao.fallbackReply`

**App Secret раскрыт:**

1. Сбросьте App Secret в приложении Yuanbao
2. Обновите значение в конфигурации
3. Перезапустите Gateway: `openclaw gateway restart`

## Расширенная конфигурация

### Несколько учётных записей

```json5
{
  channels: {
    yuanbao: {
      defaultAccount: "main",
      accounts: {
        main: {
          appKey: "key_xxx",
          appSecret: "secret_xxx",
          name: "Primary bot",
        },
        backup: {
          appKey: "key_yyy",
          appSecret: "secret_yyy",
          name: "Backup bot",
          enabled: false,
        },
      },
    },
  },
}
```

`defaultAccount` определяет, какая учётная запись используется, когда исходящие API не указывают `accountId`.

### Ограничения сообщений

- `maxChars`: максимальное количество символов в одном сообщении (по умолчанию `3000`)
- `mediaMaxMb`: ограничение на отправку и скачивание медиафайлов (по умолчанию `20` МБ)
- `overflowPolicy`: поведение при превышении сообщением ограничения — `"split"` (по умолчанию) или `"stop"`

### Потоковая передача

Yuanbao поддерживает потоковый вывод на уровне блоков; бот отправляет текст фрагментами по мере его создания.

```json5
{
  channels: {
    yuanbao: {
      disableBlockStreaming: false, // потоковая передача блоков включена (по умолчанию)
    },
  },
}
```

Установите `disableBlockStreaming: true`, чтобы отправлять полный ответ одним сообщением.

### Контекст истории группового чата

```json5
{
  channels: {
    yuanbao: {
      historyLimit: 100, // по умолчанию: 100; установите 0 для отключения
    },
  },
}
```

Определяет, сколько предыдущих сообщений включается в контекст ИИ для групповых чатов.

### Режим ответа с цитированием

```json5
{
  channels: {
    yuanbao: {
      replyToMode: "first", // "off" | "first" | "all" (по умолчанию: "first")
    },
  },
}
```

| Значение | Поведение                                                                  |
| -------- | ------------------------------------------------------------------------- |
| `off`    | Не использовать ответы с цитированием                                     |
| `first`  | Цитировать только первый ответ на каждое входящее сообщение (по умолчанию) |
| `all`    | Цитировать каждый ответ                                                    |

### Внедрение подсказки Markdown

По умолчанию бот добавляет в системный запрос инструкцию, запрещающую модели заключать весь ответ в блок кода Markdown.

```json5
{
  channels: {
    yuanbao: {
      markdownHintEnabled: true, // по умолчанию: true
    },
  },
}
```

### Режим отладки

```json5
{
  channels: {
    yuanbao: {
      debugBotIds: ["bot_user_id_1", "bot_user_id_2"],
    },
  },
}
```

Включает вывод журналов без удаления конфиденциальных данных для ботов с указанными идентификаторами.

### Маршрутизация между несколькими агентами

Используйте `bindings`, чтобы направлять личные сообщения или группы Yuanbao разным агентам:

```json5
{
  agents: {
    list: [
      { id: "main" },
      { id: "agent-a", workspace: "/home/user/agent-a" },
      { id: "agent-b", workspace: "/home/user/agent-b" },
    ],
  },
  bindings: [
    {
      agentId: "agent-a",
      match: {
        channel: "yuanbao",
        peer: { kind: "direct", id: "user_xxx" },
      },
    },
    {
      agentId: "agent-b",
      match: {
        channel: "yuanbao",
        peer: { kind: "group", id: "group_zzz" },
      },
    },
  ],
}
```

- `match.channel`: `"yuanbao"`
- `match.peer.kind`: `"direct"` (личное сообщение) или `"group"` (групповой чат)
- `match.peer.id`: идентификатор пользователя или код группы

## Поддерживаемые типы сообщений

**Получение:** текст, изображения, файлы, аудио и голосовые сообщения, видео, стикеры и пользовательские эмодзи, пользовательские элементы (карточки ссылок).

**Отправка:** текст (Markdown), изображения, файлы, аудио, видео, стикеры.

**Ветки и ответы:** ответы с цитированием (настраиваются через `replyToMode`); платформа не поддерживает ответы в ветках.
