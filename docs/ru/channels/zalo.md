Статус: экспериментальный. Реализованы как личные сообщения, так и групповые чаты; приведённая ниже таблица [Возможности](#capabilities) отражает поведение, подтверждённое для ботов Zalo Bot Creator / Marketplace.

## Встроенный Plugin

В текущих выпусках OpenClaw Zalo поставляется как встроенный Plugin, поэтому для сборок из пакетов отдельная установка не требуется.

В более старой сборке или пользовательской установке, исключающей Zalo, установите пакет npm напрямую:

- Установка: `openclaw plugins install @openclaw/zalo`
- Закреплённая версия: `openclaw plugins install @openclaw/zalo@2026.6.11`
- Из локальной рабочей копии: `openclaw plugins install ./path/to/local/zalo-plugin`
- Подробнее: [Плагины](/ru/tools/plugin)

## Быстрая настройка

1. Создайте токен бота на странице [https://bot.zaloplatforms.com](https://bot.zaloplatforms.com) (войдите, создайте бота и настройте параметры). Токен имеет вид `numeric_id:secret`; для ботов Marketplace пригодный для работы токен может находиться в приветственном сообщении бота.
2. Задайте токен либо через переменную окружения `ZALO_BOT_TOKEN=...` (только для учётной записи по умолчанию), либо в конфигурации.
3. Перезапустите Gateway.
4. При первом обращении через личные сообщения подтвердите код сопряжения (по умолчанию для личных сообщений действует политика сопряжения).

Минимальная конфигурация:

```json5
{
  channels: {
    zalo: {
      enabled: true,
      accounts: {
        default: {
          botToken: "12345689:abc-xyz",
          dmPolicy: "pairing",
        },
      },
    },
  },
}
```

Несколько учётных записей: добавьте дополнительные записи в `channels.zalo.accounts.<id>`, указав для каждой собственные `botToken`/`name`. `channels.zalo.botToken` (плоская структура без `accounts`) — устаревшая сокращённая форма для одной учётной записи; в новых конфигурациях предпочитайте `accounts.<id>.*`.

## Что это такое

Zalo — ориентированное на Вьетнам приложение для обмена сообщениями. Его Bot API позволяет Gateway запускать бота как для личных бесед, так и для групповых чатов с детерминированной маршрутизацией ответов обратно в Zalo (модель никогда не выбирает каналы).

На этой странице рассматриваются **боты Zalo Bot Creator / Marketplace**. **Боты Zalo Official Account (OA)** относятся к другой части продукта и могут вести себя иначе; на этой странице они не рассматриваются.

## Принцип работы

- Входящие сообщения нормализуются в общий конверт канала с заполнителями для медиафайлов.
- Ответы всегда направляются обратно в тот же чат Zalo; ответ с цитированием не используется (`replyToMode` всегда отключён).
- По умолчанию используется длительный опрос (`getUpdates`); режим Webhook доступен через `channels.zalo.webhookUrl`.
- В группах для активации бота требуется @упоминание; настройка этого поведения для отдельного канала не поддерживается.

## Ограничения

| Ограничение                     | Значение                                                                                                           |
| ------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| Размер фрагмента исходящего текста | 2000 символов (ограничение API Zalo)                                                                            |
| Размер медиафайла (входящего/исходящего) | `channels.zalo.mediaMaxMb`, по умолчанию `5` МБ                                                           |
| Тело запроса Webhook            | 1 МБ, тайм-аут чтения 30 с                                                                                         |
| Ограничение частоты Webhook     | 120 запросов за 60 с для каждой комбинации пути и IP-адреса клиента, затем HTTP 429                               |
| Окно обнаружения повторных событий Webhook | 5 минут (ключ формируется из пути, учётной записи, имени события, чата, отправителя и идентификатора сообщения) |

## Управление доступом

### Личные сообщения

- `channels.zalo.dmPolicy`: `pairing` (по умолчанию) | `allowlist` | `open` | `disabled`.
- Сопряжение: неизвестные отправители получают код сопряжения; сообщения игнорируются до его подтверждения. Срок действия кодов — 1 час.
  - `openclaw pairing list zalo`
  - `openclaw pairing approve zalo <CODE>`
  - Подробнее: [Сопряжение](/ru/channels/pairing)
- `channels.zalo.allowFrom` принимает числовые идентификаторы пользователей Zalo (поиск по имени пользователя не поддерживается). Для `open` требуется `"*"`.

### Группы

Групповые чаты поддерживаются Plugin (`chatTypes: ["direct", "group"]`), а доступ определяется упоминанием и групповой политикой:

- `channels.zalo.groupPolicy`: `open` | `allowlist` | `disabled`.
- `channels.zalo.groupAllowFrom` ограничивает идентификаторы отправителей, которые могут активировать бота в группах; если параметр не задан, используется `allowFrom`.
- Разрешение значений по умолчанию: если `channels.zalo` настроен, незаданный `groupPolicy` принимает значение `open`. Если раздел `channels.zalo` полностью отсутствует, во время работы применяется безопасное значение `allowlist`.
- Известное практическое ограничение: в некоторых конфигурациях ботов Marketplace добавить бота в группу вообще не удавалось. Если вы столкнулись с этим, проверьте параметры бота в Zalo Bot Platform; это ограничение платформы, а не политика OpenClaw.

## Длительный опрос и Webhook

- По умолчанию: длительный опрос (общедоступный URL не требуется).
- Режим Webhook: задайте `channels.zalo.webhookUrl` и `channels.zalo.webhookSecret`.
  - URL Webhook должен использовать HTTPS.
  - Секрет Webhook должен содержать от 8 до 256 символов.
  - Zalo отправляет события с заголовком `X-Bot-Api-Secret-Token`, который проверяется сравнением с постоянным временем выполнения.
  - HTTP-сервер Gateway обрабатывает запросы Webhook по пути `channels.zalo.webhookPath` (по умолчанию используется путь из URL Webhook).
  - Запросы должны использовать `Content-Type: application/json` (либо тип медиа с суффиксом `+json`).
  - Согласно документации API Zalo, опрос `getUpdates` и Webhook взаимоисключающие.

## Поддерживаемые типы сообщений

- Текст: полная поддержка с разбиением на фрагменты по 2000 символов.
- Медиафайлы: входящие и исходящие, с ограничением `mediaMaxMb`.
- Реакции, ветки обсуждений, опросы и встроенные команды: не поддерживаются Plugin.
- Потоковая передача: Plugin объявляет поддержку потоковой передачи блоками, однако в Zalo нет специальных параметров настройки очереди исходящих сообщений или объединения текста (в отличие от некоторых других региональных каналов); если это важно для вашего сценария, проверьте текущее поведение в своей среде.

## Устранение неполадок

**Бот не отвечает:**

- Проверьте токен: `openclaw channels status --probe`
- Убедитесь, что отправитель одобрен (через сопряжение или `allowFrom`)
- Проверьте журналы Gateway: `openclaw logs --follow`

**Webhook не получает события:**

- Убедитесь, что URL Webhook использует HTTPS
- Убедитесь, что секрет содержит от 8 до 256 символов
- Убедитесь, что конечная точка HTTP Gateway доступна по настроенному пути
- Убедитесь, что опрос `getUpdates` не выполняется одновременно (эти режимы взаимоисключающие)
- Всплеск запросов может привести к ответу HTTP 429 (120 запросов за 60 с для каждой комбинации пути и IP-адреса); увеличьте интервал и повторите попытку

## Справочник по конфигурации

Полная конфигурация: [Конфигурация](/ru/gateway/configuration)

| Параметр                                     | Описание                                                       | Значение по умолчанию   |
| -------------------------------------------- | -------------------------------------------------------------- | ----------------------- |
| `channels.zalo.enabled`                      | Включить или отключить запуск канала                           | `true`                  |
| `channels.zalo.accounts.<id>.botToken`       | Токен бота из Zalo Bot Platform                                | -                       |
| `channels.zalo.accounts.<id>.tokenFile`      | Прочитать токен из файла (символические ссылки отклоняются)    | -                       |
| `channels.zalo.accounts.<id>.name`           | Отображаемое имя                                               | -                       |
| `channels.zalo.accounts.<id>.enabled`        | Включить или отключить эту учётную запись                      | `true`                  |
| `channels.zalo.accounts.<id>.dmPolicy`       | Политика личных сообщений для учётной записи                   | `pairing`               |
| `channels.zalo.accounts.<id>.allowFrom`      | Список разрешённых отправителей личных сообщений (идентификаторы пользователей) | -        |
| `channels.zalo.accounts.<id>.groupPolicy`    | Групповая политика для учётной записи                          | см. [Группы](#groups)   |
| `channels.zalo.accounts.<id>.groupAllowFrom` | Список разрешённых отправителей в группах; при отсутствии используется `allowFrom` | -       |
| `channels.zalo.accounts.<id>.mediaMaxMb`     | Ограничение размера входящих и исходящих медиафайлов (МБ)      | `5`                     |
| `channels.zalo.accounts.<id>.webhookUrl`     | Включить режим Webhook (требуется HTTPS)                        | -                       |
| `channels.zalo.accounts.<id>.webhookSecret`  | Секрет Webhook (8–256 символов)                                | -                       |
| `channels.zalo.accounts.<id>.webhookPath`    | Путь Webhook на HTTP-сервере Gateway                           | путь из URL Webhook     |
| `channels.zalo.accounts.<id>.proxy`          | URL прокси-сервера для запросов к API                          | -                       |
| `channels.zalo.accounts.<id>.responsePrefix` | Переопределение префикса исходящих ответов                     | -                       |
| `channels.zalo.defaultAccount`               | Учётная запись по умолчанию при настройке нескольких записей  | `default`               |

`channels.zalo.botToken`, `channels.zalo.dmPolicy` и другие плоские ключи верхнего уровня — устаревшая сокращённая форма для одной учётной записи, соответствующая указанным выше полям; поддерживаются обе формы.

Параметр окружения `ZALO_BOT_TOKEN=...` задаёт токен только для учётной записи по умолчанию.
