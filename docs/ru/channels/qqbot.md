QQ Bot подключается к OpenClaw через официальный API QQ Bot (шлюз WebSocket).
Основные типы чатов — личные чаты C2C и упоминания через `@` в группах; поддерживаются мультимедийные
вложения (изображения, голосовые сообщения, видео, файлы). Для сообщений в каналах гильдий поддерживаются
только текст и изображения по удалённым URL; голосовые сообщения, видео, загрузка файлов и локальные изображения
или изображения в формате Base64 в каналах гильдий недоступны. Реакции и ветки обсуждений
не поддерживаются нигде.

Статус: официальный загружаемый plugin.

## Установка

```bash
openclaw plugins install @openclaw/qqbot
```

## Настройка

1. Перейдите на [открытую платформу QQ](https://q.qq.com/) и отсканируйте QR-код с помощью
   приложения QQ на телефоне, чтобы зарегистрироваться или войти.
2. Нажмите **Create Bot**, чтобы создать нового бота QQ.
3. Найдите **AppID** и **AppSecret** на странице настроек бота и скопируйте их.

<Note>
AppSecret не хранится в открытом виде. Если вы покинете страницу, не сохранив его, придётся создать новый.
</Note>

4. Добавьте канал:

```bash
openclaw channels add --channel qqbot --token "AppID:AppSecret"
```

5. Перезапустите Gateway.

Интерактивная настройка:

```bash
openclaw channels add
```

Мастер также предлагает привязку по QR-коду как альтернативу ручному вводу AppID/AppSecret:
отсканируйте код с помощью мобильного приложения, связанного с целевым QQ Bot, чтобы завершить
привязку. OpenClaw сохраняет полученные учётные данные в области конфигурации
учётной записи.

## Конфигурация

Минимальная конфигурация:

```json5
{
  channels: {
    qqbot: {
      enabled: true,
      appId: "YOUR_APP_ID",
      clientSecret: "YOUR_APP_SECRET",
    },
  },
}
```

Переменные окружения учётной записи по умолчанию (только учётная запись верхнего уровня):

- `QQBOT_APP_ID`
- `QQBOT_CLIENT_SECRET`

AppSecret из файла:

```json5
{
  channels: {
    qqbot: {
      enabled: true,
      appId: "YOUR_APP_ID",
      clientSecretFile: "/path/to/qqbot-secret.txt",
    },
  },
}
```

AppSecret как SecretRef из переменной окружения:

```json5
{
  channels: {
    qqbot: {
      enabled: true,
      appId: "YOUR_APP_ID",
      clientSecret: { source: "env", provider: "default", id: "QQBOT_CLIENT_SECRET" },
    },
  },
}
```

Примечания:

- `openclaw channels add --channel qqbot --token-file ...` задаёт только AppSecret;
  `appId` уже должен быть указан в конфигурации или в `QQBOT_APP_ID`.
- `clientSecret` принимает строку в открытом виде, путь к файлу (`clientSecretFile`)
  или структурированный объект SecretRef.
- Устаревшие строки-маркеры `secretref:...` / `secretref-env:...` для
  `clientSecret` отклоняются; вместо них используйте структурированный объект SecretRef.

### Политика доступа

- `allowFrom` / `groupAllowFrom` определяют, кто может общаться с ботом в контексте C2C /
  групп. `dmPolicy` / `groupPolicy` (`open` | `allowlist` | `disabled`)
  управляют режимом применения ограничений. По умолчанию `dmPolicy` принимает значение `allowlist`, если
  `allowFrom` содержит конкретную запись (не подстановочный символ), иначе — `open`.
  По умолчанию `groupPolicy` принимает значение `allowlist`, если `groupAllowFrom` или
  `allowFrom` содержит конкретную запись, иначе — `open`.
- Для slash-команд с авторизацией по списку разрешённых требуется явная запись без подстановочного символа в
  `allowFrom` (или в `groupAllowFrom` для вызовов из групп) независимо от
  `dmPolicy` / `groupPolicy` — см. [Slash-команды](#slash-commands).

### Настройка нескольких учётных записей

Запускайте несколько ботов QQ в одном экземпляре OpenClaw:

```json5
{
  channels: {
    qqbot: {
      enabled: true,
      appId: "111111111",
      clientSecret: "secret-of-bot-1",
      accounts: {
        bot2: {
          enabled: true,
          appId: "222222222",
          clientSecret: "secret-of-bot-2",
        },
      },
    },
  },
}
```

Каждая учётная запись имеет изолированные подключение WebSocket, клиент API и кеш токенов,
привязанные к `appId`. Строки журнала помечаются идентификатором соответствующей учётной записи, чтобы
диагностические данные оставались разделёнными при запуске нескольких ботов через один Gateway.

Добавление второго бота через CLI:

```bash
openclaw channels add --channel qqbot --account bot2 --token "222222222:secret-of-bot-2"
```

### Групповые чаты

Для поддержки групп используются OpenID групп QQ, а не отображаемые имена. Добавьте бота в
группу, затем упомяните его или настройте группу для работы без упоминания.

```json5
{
  channels: {
    qqbot: {
      groupPolicy: "allowlist",
      groupAllowFrom: ["member_openid"],
      groups: {
        "*": {
          requireMention: true,
          commandLevel: "all",
          historyLimit: 50,
          tools: { deny: ["exec", "read", "write"] },
        },
        GROUP_OPENID: {
          name: "Release room",
          requireMention: false,
          ignoreOtherMentions: true,
          commandLevel: "safety",
          historyLimit: 20,
          prompt: "Keep replies short and operational.",
        },
      },
    },
  },
}
```

`groups["*"]` задаёт значения по умолчанию для всех групп; конкретная запись `groups.GROUP_OPENID`
переопределяет эти значения для одной группы. Настройки групп:

| Поле                  | По умолчанию     | Описание                                                                                              |
| --------------------- | ---------------- | ----------------------------------------------------------------------------------------------------- |
| `requireMention`      | `true`           | Требовать упоминание через `@`, прежде чем бот ответит.                                               |
| `commandLevel`        | `all`            | Какие встроенные slash-команды можно выполнять в группе (см. ниже).                                   |
| `ignoreOtherMentions` | `false`          | Отбрасывать сообщения, в которых упомянут кто-то другой, но не бот.                                   |
| `historyLimit`        | `50`             | Недавние сообщения без упоминания, сохраняемые как контекст для следующего обращения. `0` отключает историю. |
| `tools`               | —                | Разрешить или запретить инструменты для всей группы.                                                  |
| `toolsBySender`       | —                | Переопределения инструментов для отдельных отправителей; см. [Группы](/ru/channels/groups#groupchannel-tool-restrictions-optional). |
| `name`                | префикс openid   | Понятная метка, используемая в журналах и контексте группы.                                           |
| `prompt`              | встроенное значение по умолчанию | Запрос поведения для конкретной группы, добавляемый к контексту агента.                |

`commandLevel` принимает следующие значения:

| Уровень  | Поведение                                                                                                                                     |
| -------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| `all`    | Существующие встроенные команды остаются доступными. Некоторые из них скрыты в меню, но авторизованные пользователи всё равно могут выполнять их в группе. |
| `safety` | `/help`, `/btw`, `/stop` остаются видимыми в группе; чувствительные команды (`/config`, `/tools`, `/bash` и т. д.) необходимо выполнять в личном чате. |
| `strict` | Разрешены только элементы управления групповой сессией, необходимые для строгого режима. `/stop` продолжает работать, чтобы авторизованный отправитель мог прервать активный запуск. |

Старые записи QQBot `toolPolicy` больше не используются. Выполните `openclaw doctor --fix`, чтобы перенести их в `tools`.

Режимы активации: `mention` и `always`. `requireMention: true` соответствует
`mention`, а `requireMention: false` — `always`. Переопределение активации на уровне сессии,
если оно задано, имеет приоритет над конфигурацией.

Входящая очередь создаётся отдельно для каждого собеседника. Для групповых собеседников действует увеличенный предел очереди (50 вместо 20
для личных собеседников); при заполнении сообщения бота удаляются раньше сообщений пользователей,
а серии обычных групповых сообщений объединяются в один ход с указанием авторов. Slash-команды
выполняются по одной независимо от объединяемого пакета.

### Голос (STT / TTS)

STT и TTS поддерживают двухуровневую конфигурацию с резервным переходом по приоритету:

| Настройка | Для plugin                                             | Резервная настройка фреймворка |
| --------- | ------------------------------------------------------ | ------------------------------ |
| STT       | `channels.qqbot.stt`                                   | `tools.media.audio.models[0]`  |
| TTS       | `channels.qqbot.tts`, `channels.qqbot.accounts.<id>.tts` | `messages.tts`               |

```json5
{
  channels: {
    qqbot: {
      stt: {
        provider: "your-provider",
        model: "your-stt-model",
      },
      tts: {
        provider: "your-provider",
        model: "your-tts-model",
        voice: "your-voice",
      },
      accounts: {
        "qq-main": {
          tts: {
            providers: {
              openai: { voice: "shimmer" },
            },
          },
        },
      },
    },
  },
}
```

Установите `enabled: false` для любого из них, чтобы отключить его. Переопределения TTS на уровне учётной записи используют
ту же структуру, что и `messages.tts`, и глубоко объединяются с конфигурацией TTS канала или глобальной конфигурацией.

По умолчанию время ожидания запросов STT истекает через 60 секунд. STT для plugin использует
переопределение `models.providers.<id>.timeoutSeconds` выбранного поставщика. STT аудио на уровне фреймворка
сначала использует `tools.media.audio.models[0].timeoutSeconds`, затем
`tools.media.audio.timeoutSeconds`, а затем переопределение выбранного поставщика.

Входящие голосовые вложения QQ предоставляются агентам как метаданные аудио,
при этом исходные голосовые файлы не добавляются в общий список `MediaPaths`. Маркер `[[audio_as_voice]]`
в текстовом ответе синтезирует TTS и отправляет нативное голосовое сообщение QQ, если
TTS настроен.

Поведение загрузки и перекодирования исходящего аудио также можно настроить через
`channels.qqbot.audioFormatPolicy`:

- `sttDirectFormats`
- `uploadDirectFormats`
- `transcodeEnabled`

## Форматы назначения

| Формат                     | Описание            |
| -------------------------- | ------------------- |
| `qqbot:c2c:OPENID`         | Личный чат (C2C)    |
| `qqbot:group:GROUP_OPENID` | Групповой чат       |
| `qqbot:channel:CHANNEL_ID` | Канал гильдии       |

<Note>
У каждого бота собственный набор пользовательских OpenID. OpenID, полученный ботом A, **нельзя** использовать для отправки сообщений через бота B.
</Note>

## Slash-команды

Встроенные команды, перехватываемые до очереди ИИ:

| Команда              | Авторизация       | Область действия  | Описание                                                                                     |
| -------------------- | ----------------- | ----------------- | -------------------------------------------------------------------------------------------- |
| `/bot-ping`          | —                 | любая             | Проверка задержки                                                                            |
| `/bot-help`          | —                 | любая             | Показать все команды                                                                         |
| `/bot-me`            | —                 | только личный чат | Показать идентификатор пользователя QQ (openid) отправителя для настройки `allowFrom` / `groupAllowFrom` |
| `/bot-version`       | —                 | только личный чат | Показать версию фреймворка OpenClaw и версию plugin                                          |
| `/bot-upgrade`       | —                 | только личный чат | Показать ссылку на руководство по обновлению QQBot                                            |
| `/bot-approve`       | список разрешённых | только личный чат | Управлять конфигурацией подтверждения выполнения команд (вкл. / выкл. / всегда / сброс / состояние) |
| `/bot-logs`          | список разрешённых | только личный чат | Экспортировать последние журналы Gateway в виде файла                                        |
| `/bot-clear-storage` | список разрешённых | только личный чат | Удалить кешированные загрузки из каталога мультимедиа QQBot                                   |
| `/bot-streaming`     | список разрешённых | только личный чат | Переключить потоковые ответы C2C                                                              |
| `/bot-group-allways` | список разрешённых | только личный чат | Переключить режим активации групп по умолчанию (требуется упоминание или всегда активен)       |

Добавьте `?` к любой команде, чтобы получить справку по использованию (например, `/bot-upgrade ?`).

Команды с авторизацией по списку разрешённых дополнительно требуют наличия openid отправителя в
явном списке `allowFrom` без подстановочных символов (`groupAllowFrom` имеет приоритет для
команд, вызванных из группы, с резервным переходом к `allowFrom`). Подстановочный символ
`allowFrom: ["*"]` разрешает общение, но не выполнение этих команд. При запуске такой команды
вне личного чата или без авторизации возвращается подсказка, а сообщение не отбрасывается
без уведомления.

`/bot-me`, `/bot-version` и `/bot-upgrade` доступны только в личных чатах, но не
требуют списка разрешённых отправителей — их может выполнить любой отправитель C2C.

Когда подтверждения выполнения QQ Bot используют стандартный резервный вариант в том
же чате, нажатия встроенных кнопок подтверждения подчиняются тому же явному списку
разрешённых команд без подстановочных знаков. Чтобы предоставить доступ только к
подтверждениям без расширения доступа к командам, настройте
`channels.qqbot.execApprovals.approvers`. Встроенные подтверждения выполнения включены
по умолчанию.

## Медиафайлы и хранилище

- Входящие, исходящие и передаваемые через мост Gateway медиафайлы используют единый
  корневой каталог данных в `~/.openclaw/media/qqbot` (с учётом `OPENCLAW_HOME`, если
  эта переменная задана), поэтому отправленные и загруженные файлы, а также кеши
  перекодирования находятся в одном защищённом каталоге.
- Доставка мультимедийного содержимого для адресатов C2C и групп выполняется через
  единый путь `sendMedia`. Для локальных файлов и буферов в памяти размером
  5&nbsp;МиБ и более используются конечные точки QQ для загрузки по частям; для
  данных меньшего размера и источников в виде удалённых URL/Base64 используется API
  однократной загрузки.
- Если горячее обновление прерывает работу Gateway до завершения записи
  `openclaw.json`, при следующем запуске Plugin восстанавливает последние известные
  значения `appId` / `clientSecret` для этой учётной записи из внутреннего снимка
  (никогда не перезаписывая намеренное изменение конфигурации), поэтому повторное
  сканирование QR-кода не требуется.

## Устранение неполадок

- **Gateway не запускается / нет входящих сообщений:** убедитесь, что `appId` и
  `clientSecret` указаны правильно, а бот включён на платформе QQ Open Platform.
  При отсутствии учётных данных отображается сообщение «QQBot не настроен (отсутствует
  appId или clientSecret)».
- **После настройки с `--token-file` по-прежнему отображается отсутствие
  конфигурации:** `--token-file` задаёт только AppSecret. `appId` всё равно необходимо
  указать в конфигурации или в `QQBOT_APP_ID`.
- **Пакетные ответы в группе конфликтуют:** когда очередь участника заполняется,
  входящая очередь удаляет сообщения от ботов раньше сообщений от людей и объединяет
  серии обычных (не являющихся командами) групповых сообщений в один ход с указанием
  авторов, поэтому поток сообщений от ботов не должен вытеснять сообщения от людей.
- **Проактивные сообщения не поступают:** QQ может блокировать сообщения,
  инициированные ботом, если пользователь давно с ним не взаимодействовал.
- **Голосовые сообщения не расшифровываются:** убедитесь, что STT настроено и
  провайдер доступен.
