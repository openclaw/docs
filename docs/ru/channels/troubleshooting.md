## Лестница команд

Сначала выполните эти команды по порядку:

```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
openclaw channels status --probe
```

Исправная базовая картина:

- `Runtime: running`
- `Connectivity probe: ok`
- `Capability: read-only`, `write-capable` или `admin-capable`
- Проверка канала показывает, что транспорт подключен и, где поддерживается, `works` или `audit ok`

## После обновления

Используйте это, когда Telegram, iMessage, конфигурации эпохи BlueBubbles или другой канал Plugin
исчезает после обновления.

```bash
openclaw status --all
openclaw doctor --fix
openclaw gateway restart
openclaw status --all
```

Ищите `plugin load failed: dependency tree corrupted; run openclaw doctor
--fix` в `openclaw status --all`. Это означает, что канал настроен, но
путь настройки/загрузки Plugin наткнулся на поврежденное дерево зависимостей вместо регистрации
канала. `openclaw doctor --fix` удаляет устаревшие промежуточные каталоги зависимостей Plugin
и устаревшие теневые данные авторизации, затем `openclaw gateway restart` перезагружает
чистое состояние.

## WhatsApp

### Сигнатуры сбоев WhatsApp

| Симптом                             | Самая быстрая проверка                                | Исправление                                                                                                                       |
| ----------------------------------- | ----------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| Подключен, но нет ответов в ЛС      | `openclaw pairing list whatsapp`                      | Одобрите отправителя или измените политику/allowlist для ЛС.                                                                      |
| Групповые сообщения игнорируются    | Проверьте `requireMention` + шаблоны упоминаний в конфигурации | Упомяните бота или ослабьте политику упоминаний для этой группы.                                                                  |
| Вход по QR истекает с 408           | Проверьте env Gateway `HTTPS_PROXY` / `HTTP_PROXY`    | Задайте доступный прокси; используйте `NO_PROXY` только для обходов.                                                              |
| Случайные циклы отключения/повторного входа | `openclaw channels status --probe` + журналы          | Недавние повторные подключения помечаются, даже если сейчас подключение есть; следите за журналами, перезапустите Gateway, затем перепривяжите, если нестабильность продолжается. |
| Цикл `status=408 Request Time-out`  | Проверка, журналы, doctor, затем статус Gateway       | Сначала исправьте связность/тайминги хоста; сделайте резервную копию авторизации и заново привяжите аккаунт, если цикл сохраняется. |
| Ответы приходят на секунды/минуты позже | `openclaw doctor --fix`                               | Doctor останавливает проверенные устаревшие локальные TUI-клиенты, когда они ухудшают цикл событий Gateway.                       |

Полное устранение неполадок: [устранение неполадок WhatsApp](/ru/channels/whatsapp#troubleshooting)

## Telegram

### Сигнатуры сбоев Telegram

| Симптом                              | Самая быстрая проверка                             | Исправление                                                                                                                       |
| ------------------------------------ | -------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| `/start`, но нет пригодного потока ответов | `openclaw pairing list telegram`                   | Одобрите привязку или измените политику ЛС.                                                                                       |
| Бот онлайн, но группа молчит         | Проверьте требование упоминания и режим приватности бота | Отключите режим приватности для видимости в группе или упомяните бота.                                                            |
| Сбои отправки с сетевыми ошибками    | Проверьте журналы на сбои вызовов Telegram API     | Исправьте маршрутизацию DNS/IPv6/прокси к `api.telegram.org`.                                                                      |
| При запуске сообщается `getMe returned 401` | Проверьте настроенный источник токена              | Скопируйте заново или сгенерируйте заново токен BotFather и обновите `botToken`, `tokenFile` или `TELEGRAM_BOT_TOKEN` аккаунта по умолчанию. |
| Polling зависает или медленно переподключается | `openclaw logs --follow` для диагностики polling   | Обновитесь; если перезапуски являются ложными срабатываниями, настройте `pollingStallThresholdMs`. Постоянные зависания все еще указывают на прокси/DNS/IPv6. |
| `setMyCommands` отклоняется при запуске | Проверьте журналы на `BOT_COMMANDS_TOO_MUCH`       | Сократите команды Telegram из Plugin/skill/пользовательские команды или отключите нативные меню.                                  |
| После обновления allowlist блокирует вас | `openclaw security audit` и allowlist в конфигурации | Запустите `openclaw doctor --fix` или замените `@username` числовыми ID отправителей.                                             |

Полное устранение неполадок: [устранение неполадок Telegram](/ru/channels/telegram#troubleshooting)

## Matrix

### Сигнатуры сбоев Matrix

| Симптом                             | Самая быстрая проверка                 | Исправление                                                               |
| ----------------------------------- | -------------------------------------- | ------------------------------------------------------------------------- |
| Вход выполнен, но сообщения в комнате игнорируются | `openclaw channels status --probe`     | Проверьте `groupPolicy`, список разрешенных комнат и фильтрацию по упоминанию. |

| Восстановление резервной копии ожидает выполнения или сломано | `openclaw matrix verify backup status` | Выполните `openclaw matrix verify backup restore` или запустите повторно с ключом восстановления. |
| Кросс-подпись/bootstrap выглядит неверно | `openclaw matrix verify bootstrap`     | Исправьте хранилище секретов, кросс-подпись и состояние резервной копии за один проход. |
