## Последовательность команд

Сначала выполните эти команды по порядку:

```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
openclaw channels status --probe
```

Признаки нормальной работы:

- `Runtime: running`
- `Connectivity probe: ok`
- `Capability: read-only`, `write-capable` или `admin-capable`
- Проверка канала показывает, что транспорт подключён и, если поддерживается, `works` или `audit ok`

## После обновления

Используйте эти действия, если после обновления исчез канал Telegram, iMessage, канал с конфигурацией времён BlueBubbles или другой канал плагина.

```bash
openclaw status --all
openclaw doctor --fix
openclaw gateway restart
openclaw status --all
```

Найдите `plugin load failed: dependency tree corrupted; run openclaw doctor --fix` в `openclaw
status --all`. Это означает, что канал настроен, но при настройке или загрузке плагина вместо регистрации канала было обнаружено повреждённое дерево зависимостей. `openclaw doctor --fix` удаляет устаревшие символические ссылки зависимостей среды выполнения плагина и устаревшие теневые данные аутентификации, после чего `openclaw gateway restart` повторно загружает чистое состояние.

## WhatsApp

### Признаки сбоев WhatsApp

| Симптом                             | Самая быстрая проверка                                       | Исправление                                                                                                                              |
| ----------------------------------- | --------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| Подключение установлено, но ответов в личных сообщениях нет         | `openclaw pairing list whatsapp`                    | Одобрите отправителя либо измените политику личных сообщений или список разрешённых отправителей.                                                                                    |
| Сообщения в группе игнорируются              | Проверьте `requireMention` и шаблоны упоминаний в конфигурации | Упомяните бота или смягчите политику упоминаний для этой группы.                                                                          |
| Вход по QR-коду завершается по тайм-ауту с кодом 408         | Проверьте переменные среды `HTTPS_PROXY` / `HTTP_PROXY` для Gateway      | Настройте доступный прокси-сервер; используйте `NO_PROXY` только для обхода прокси.                                                                         |
| Случайные циклы отключения и повторного входа     | `openclaw channels status --probe` и журналы           | Недавние переподключения отмечаются, даже если сейчас соединение установлено; наблюдайте за журналами, перезапустите Gateway, а если нестабильность сохранится — повторно привяжите учётную запись. |
| Цикл `status=408 Request Time-out`  | Проверка, журналы, doctor, затем состояние Gateway            | Сначала устраните проблемы с подключением или синхронизацией времени на хосте; если цикл сохраняется, создайте резервную копию данных аутентификации и повторно привяжите учётную запись.                                   |
| Ответы поступают с задержкой в несколько секунд или минут | `openclaw doctor --fix`                             | Doctor останавливает проверенные устаревшие локальные клиенты TUI, если они ухудшают работу цикла событий Gateway.                                    |

Полное руководство: [Устранение неполадок WhatsApp](/ru/channels/whatsapp#troubleshooting)

## Telegram

### Признаки сбоев Telegram

| Симптом                              | Самая быстрая проверка                                    | Исправление                                                                                                                        |
| ------------------------------------ | ------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------- |
| `/start`, но рабочий обмен ответами отсутствует    | `openclaw pairing list telegram`                 | Одобрите сопряжение или измените политику личных сообщений.                                                                                       |
| Бот в сети, но группа не получает ответов    | Проверьте требование упоминания и режим конфиденциальности бота  | Отключите режим конфиденциальности, чтобы бот видел сообщения группы, или упомяните бота.                                                                  |
| Ошибки отправки из-за сетевых ошибок    | Проверьте журналы на наличие ошибок вызовов API Telegram      | Исправьте маршрутизацию DNS, IPv6 или прокси к `api.telegram.org`.                                                                          |
| При запуске сообщается `getMe returned 401` | Проверьте настроенный источник токена                    | Повторно скопируйте или создайте токен BotFather и обновите `botToken`, `tokenFile` или `TELEGRAM_BOT_TOKEN` учётной записи по умолчанию.     |
| Опрос зависает или медленно переподключается  | `openclaw logs --follow` для диагностики опроса | Выполните обновление; если перезапуски оказываются ложными срабатываниями, настройте `pollingStallThresholdMs`. Сохраняющиеся зависания по-прежнему указывают на проблемы с прокси, DNS или IPv6. |
| `setMyCommands` отклоняется при запуске  | Проверьте журналы на наличие `BOT_COMMANDS_TOO_MUCH`         | Сократите количество команд Telegram от плагинов, Skills и пользовательских настроек либо отключите встроенные меню.                                                      |
| После обновления список разрешённых отправителей блокирует вас    | `openclaw security audit` и списки разрешений в конфигурации  | Выполните `openclaw doctor --fix` или замените `@username` числовыми идентификаторами отправителей.                                                |

Полное руководство: [Устранение неполадок Telegram](/ru/channels/telegram#troubleshooting)
