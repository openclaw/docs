OpenClaw может общаться с вами в любом уже используемом вами приложении для чатов. Каждый канал подключается через Gateway.
Текст поддерживается везде; поддержка медиафайлов и реакций зависит от канала.

iMessage, Telegram и интерфейс WebChat входят в базовую установку. Каналы с пометкой
«официальный плагин» устанавливаются одной командой (`openclaw plugins install @openclaw/<id>`)
или по запросу во время `openclaw onboard` / `openclaw channels add`, после чего требуется
перезапуск Gateway. Каналы с пометкой «внешний плагин» поддерживаются вне репозитория OpenClaw.

## Поддерживаемые каналы

- [Discord](/ru/channels/discord) — Discord Bot API + Gateway; поддерживает серверы, каналы и личные сообщения (официальный плагин).
- [Feishu](/ru/channels/feishu) — бот Feishu/Lark через WebSocket (официальный плагин).
- [Google Chat](/ru/channels/googlechat) — приложение Google Chat API через HTTP-Webhook (официальный плагин).
- [iMessage](/ru/channels/imessage) — входит в базовую установку. Нативная интеграция с macOS через мост `imsg` на Mac с выполненным входом в систему (или через SSH-обёртку, если Gateway работает в другом месте), включая действия через закрытый API для ответов, реакций Tapback, эффектов, вложений и управления группами.
- [IRC](/ru/channels/irc) — классические серверы IRC; каналы и личные сообщения с управлением через сопряжение и списки разрешённых пользователей (официальный плагин).
- [LINE](/ru/channels/line) — бот LINE Messaging API (официальный плагин).
- [Matrix](/ru/channels/matrix) — протокол Matrix (официальный плагин).
- [Mattermost](/ru/channels/mattermost) — Bot API + WebSocket; каналы, группы и личные сообщения (официальный плагин).
- [Microsoft Teams](/ru/channels/msteams) — Bot Framework; поддержка корпоративных сред (официальный плагин).
- [Nextcloud Talk](/ru/channels/nextcloud-talk) — самостоятельно размещаемый чат через Nextcloud Talk (официальный плагин).
- [Nostr](/ru/channels/nostr) — децентрализованные личные сообщения через NIP-04 (официальный плагин).
- [QQ Bot](/ru/channels/qqbot) — QQ Bot API; личные и групповые чаты, а также мультимедийные материалы (официальный плагин).
- [Reef](/channels/reef) — защищённый сквозным шифрованием обмен сообщениями между агентами OpenClaw разных пользователей (встроенный плагин).
- [Raft](/ru/channels/raft) — мост пробуждения Raft CLI для совместной работы людей и агентов (официальный плагин).
- [Signal](/ru/channels/signal) — signal-cli; ориентирован на конфиденциальность (официальный плагин).
- [Slack](/ru/channels/slack) — Bolt SDK; приложения для рабочих пространств (официальный плагин).
- [SMS](/ru/channels/sms) — SMS через Twilio и Webhook Gateway (официальный плагин).
- [Synology Chat](/ru/channels/synology-chat) — Synology NAS Chat через исходящие и входящие Webhook (официальный плагин).
- [Telegram](/ru/channels/telegram) — входит в базовую установку. Bot API через grammY; поддерживает группы.
- [Tlon](/ru/channels/tlon) — мессенджер на базе Urbit (официальный плагин).
- [Twitch](/ru/channels/twitch) — чат Twitch через подключение IRC (официальный плагин).
- [Голосовые вызовы](/ru/plugins/voice-call) — телефония через Plivo, Telnyx или Twilio (официальный плагин).
- [WebChat](/ru/web/webchat) — входит в базовую установку. Интерфейс WebChat Gateway через WebSocket.
- [WeChat](/ru/channels/wechat) — бот Tencent iLink со входом по QR-коду; только личные чаты (внешний плагин).
- [WhatsApp](/ru/channels/whatsapp) — самый популярный вариант; использует Baileys и требует сопряжения по QR-коду (официальный плагин).
- [Yuanbao](/ru/channels/yuanbao) — бот Tencent Yuanbao (внешний плагин).
- [Zalo](/ru/channels/zalo) — Zalo Bot API; популярный мессенджер во Вьетнаме (официальный плагин).
- [Zalo ClawBot](/ru/channels/zaloclawbot) — персональный помощник Zalo со входом по QR-коду; привязан к владельцу (внешний плагин).
- [Zalo Personal](/ru/channels/zalouser) — личная учётная запись Zalo со входом по QR-коду (официальный плагин).

## Примечания о доставке

- Ответы Telegram, содержащие синтаксис изображений Markdown, например `![alt](url)`,
  по возможности преобразуются в ответы с медиафайлами на завершающем этапе исходящей отправки.
- Личные сообщения Slack с несколькими участниками обрабатываются как групповые чаты, поэтому к беседам MPIM
  применяются групповая политика, правила упоминаний и групповых сеансов.
- Настройка WhatsApp выполняется с установкой по запросу: при первоначальной настройке процесс конфигурации может отображаться до
  установки пакета плагина, а Gateway загружает внешний
  плагин ClawHub/npm, только когда канал фактически активен.
- Каналы, принимающие входящие сообщения от ботов, могут использовать общую
  [защиту от циклических ответов ботов](/ru/channels/bot-loop-protection), чтобы пары ботов
  не отвечали друг другу бесконечно.
- Поддерживаемые постоянно активные комнаты могут использовать [фоновые события комнаты](/ru/channels/ambient-room-events),
  чтобы сообщения в комнате без упоминания становились ненавязчивым контекстом, если только агент не отправляет сообщение
  с помощью инструмента `message`.

## Примечания

- Каналы могут работать одновременно; настройте несколько каналов, и OpenClaw будет выполнять маршрутизацию для каждого чата.
- Быстрее всего обычно настраивается **Telegram** (простой токен бота, установка плагина не требуется). WhatsApp
  требует сопряжения по QR-коду и хранит больше состояния на диске.
- Поведение в группах зависит от канала; см. [Группы](/ru/channels/groups).
- Для безопасности применяются сопряжение личных сообщений и списки разрешённых пользователей; см. [Безопасность](/ru/gateway/security).
- Устранение неполадок: [Устранение неполадок каналов](/ru/channels/troubleshooting).
- Поставщики моделей описаны отдельно; см. [Поставщики моделей](/ru/providers/models).
