## Быстрый старт

### Режим Gateway

1. Запустите Gateway.

```bash
openclaw gateway
```

2. Откройте TUI.

```bash
openclaw tui
```

3. Введите сообщение и нажмите Enter.

Удалённый Gateway:

```bash
openclaw tui --url ws://<host>:<port> --token <gateway-token>
```

Используйте `--password`, если ваш Gateway использует аутентификацию по паролю.

### Локальный режим

Запустите TUI без Gateway:

```bash
openclaw chat
# или
openclaw tui --local
```

- `openclaw chat` и `openclaw terminal` — псевдонимы для `openclaw tui --local`.
- `--local` нельзя использовать вместе с `--url`, `--token` или `--password`.
- Локальный режим напрямую использует встроенную среду выполнения агента. Большинство локальных инструментов работают, но функции, доступные только через Gateway, недоступны.
- Команда `openclaw` без подкоманды выбирает цель автоматически: при ненастроенной установке запускается первоначальная настройка инференса; при недопустимой конфигурации открываются классические рекомендации Doctor; если настроенный Gateway доступен, эта оболочка TUI открывается в режиме Gateway; в противном случае при наличии настроенной локальной модели она открывается в локальном режиме.

## Что отображается

- Заголовок: URL подключения, текущий агент, текущий сеанс.
- Журнал чата: сообщения пользователя, ответы ассистента, системные уведомления, карточки инструментов.
- Строка состояния: состояние подключения или выполнения (подключение, выполнение, потоковая передача, ожидание, ошибка).
- Нижняя строка: агент + сеанс + модель + состояние цели + think/fast/verbose/trace/reasoning + количество токенов + доставка. Когда включён `tui.footer.showRemoteHost`, для удалённых подключений к Gateway также отображается хост подключения.
- Поле ввода: текстовый редактор с автодополнением.

## Ментальная модель: агенты и сеансы

- Агенты имеют уникальные идентификаторы-слаги (например, `main`, `research`). Gateway предоставляет их список.
- Сеансы принадлежат текущему агенту.
- Ключи сеансов хранятся в виде `agent:<agentId>:<sessionKey>`.
  - Если ввести `/session main`, TUI разворачивает его в `agent:<currentAgent>:main`.
  - Если ввести `/session agent:other:main`, произойдёт явное переключение на этот сеанс агента.
- Область сеансов:
  - `per-sender` (по умолчанию): у каждого агента может быть несколько сеансов.
  - `global`: TUI всегда использует сеанс `global` (список выбора может быть пустым).
- Текущие агент и сеанс всегда отображаются в нижней строке.
- Чтобы отображать хост Gateway для нелокальных подключений по URL, включите эту возможность:

  ```bash
  openclaw config set tui.footer.showRemoteHost true
  ```

  Значение по умолчанию — `false`. Для подключений через loopback и встроенных локальных подключений метка хоста никогда не отображается.

- Если у сеанса есть [цель](/ru/tools/goal), в нижней строке отображается её компактное состояние:
  `Pursuing goal`, `Goal paused (/goal resume)`, `Goal blocked (/goal resume)` или `Goal achieved`.
- При запуске без `--session` TUI в режиме Gateway возобновляет последний выбранный сеанс для того же Gateway, агента и области сеансов, если этот сеанс всё ещё существует. Передача `--session`, `/session`, `/new` или `/reset` по-прежнему считается явным выбором.

## Отправка и доставка

- Сообщения всегда отправляются в Gateway (или во встроенную среду выполнения в локальном режиме); доставка ответа ассистента обратно провайдеру чата — отдельный этап, по умолчанию отключённый.
- TUI — это внутренняя поверхность источника, аналогичная WebChat, а не универсальный исходящий канал. Среды выполнения, которым для видимых ответов требуется `tools.message`, могут завершить активный ход TUI с помощью `message.send` без цели; явная доставка провайдеру по-прежнему использует обычные настроенные каналы и никогда не переключается на `lastChannel`.
- Настройка доставки фиксируется на весь сеанс TUI при запуске: чтобы включить её, запустите TUI с `openclaw tui --deliver`. Команды с косой чертой `/deliver` и переключателя в Settings для изменения этой настройки во время сеанса нет; чтобы изменить её, перезапустите TUI.

## Списки выбора и накладываемые панели

- Выбор модели: просмотр доступных моделей и задание переопределения для сеанса.
- Выбор агента: выбор другого агента.
- Выбор сеанса: отображает до 50 сеансов текущего агента, обновлённых за последние 7 дней. Используйте `/session <key>`, чтобы перейти к более старому известному сеансу.
- Settings (`/settings`): переключение развёрнутого вывода инструментов и видимости процесса рассуждения. Эта панель не управляет доставкой.

## Сочетания клавиш

- Enter: отправить сообщение
- Esc: прервать активное выполнение
- Ctrl+C: очистить поле ввода (нажмите дважды для выхода)
- Ctrl+D: выйти
- Ctrl+L: выбор модели
- Ctrl+G: выбор агента
- Ctrl+P: выбор сеанса
- Ctrl+O: переключить развёрнутый вывод инструментов
- Ctrl+T: переключить видимость процесса рассуждения (история перезагружается)

## Команды с косой чертой

Основные:

- `/help`
- `/status` (перенаправляется в Gateway; показывает сводку по сеансу и модели)
- `/gateway-status` (псевдоним `/gwstatus`; напрямую показывает состояние подключения к Gateway)
- `/agent <id>` (или `/agents`)
- `/session <key>` (или `/sessions`)
- `/model <provider/model>` (или `/models`)

Управление сеансом:

- `/think <off|minimal|low|medium|high>` (на более высоких уровнях в зависимости от модели могут добавляться уровни, такие как `xhigh`/`max`)
- `/fast <status|auto|on|off>`
- `/verbose <on|full|off>`
- `/trace <on|off>`
- `/reasoning <on|off|stream>`
- `/usage <off|tokens|full|reset>` (`reset`/`inherit`/`clear`/`default` сбрасывает переопределение сеанса)
- `/goal [status] | /goal start <objective> | /goal edit <objective> | /goal pause|resume|complete|block|clear`
- `/elevated <on|off|ask|full>` (псевдоним: `/elev`)
- `/activation <mention|always>`

Жизненный цикл сеанса:

- `/new` (создаёт новый изолированный сеанс с новым ключом; не влияет на другие клиенты TUI в старом сеансе)
- `/reset` (сбрасывает текущий ключ сеанса на месте)
- `/abort` (прерывает активное выполнение)
- `/settings`
- `/exit` (или `/quit`)

Только в локальном режиме:

- `/auth [provider]` открывает в TUI процесс аутентификации или входа у провайдера.

OpenClaw:

- `/openclaw [request]` возвращает из обычного TUI агента в чат настройки и восстановления [OpenClaw](#openclaw-setup-and-repair-helper), при необходимости пересылая один запрос.

Другие команды Gateway с косой чертой (например, `/context`) перенаправляются в Gateway и отображаются как системный вывод. См. [Команды с косой чертой](/ru/tools/slash-commands).

## Команды локальной оболочки

- Добавьте префикс `!` к строке, чтобы выполнить команду локальной оболочки на хосте TUI.
- TUI один раз за сеанс запрашивает разрешение на локальное выполнение; при отказе `!` остаётся отключённым на протяжении сеанса.
- Команды выполняются в новой неинтерактивной оболочке в рабочем каталоге TUI (без сохранения `cd`/окружения).
- Локальные команды оболочки получают `OPENCLAW_SHELL=tui-local` в своём окружении.
- Одиночный `!` отправляется как обычное сообщение; начальные пробелы не запускают локальное выполнение.

## Помощник OpenClaw по настройке и восстановлению

OpenClaw — это помощник нулевого уровня по настройке и восстановлению, доступный как `openclaw setup` после успешной проверки настроенной модели по умолчанию с реальным инференсом. Если инференс недоступен, интерактивный запуск возвращает к первоначальной настройке инференса, а автоматизация завершается ошибкой с рекомендациями по восстановлению. Он работает в той же локальной оболочке TUI, что и `openclaw tui --local`, и использует ИИ-агента, ограниченного типизированными операциями OpenClaw, требующими подтверждения:

```bash
openclaw setup                       # интерактивный запуск
openclaw setup -m "status"           # выполнить один запрос и выйти
openclaw setup -m "set default model openai/gpt-5.2" --yes   # применить запись конфигурации
```

- Постоянные изменения конфигурации требуют подтверждения: подтвердите их интерактивно или передайте `--yes`.
- `--json` выводит начальный обзор в формате JSON вместо запуска чата.
- Внутри OpenClaw запрос `open-tui` (например, просьба перейти к обычному агенту) завершает OpenClaw и открывает обычный TUI агента; чтобы вернуться, используйте там `/openclaw`.

Используйте локальный режим, когда текущая конфигурация уже проходит проверку и требуется, чтобы встроенный агент проверил её на той же машине, сравнил с документацией и помог устранить расхождения без зависимости от работающего Gateway.

Если `openclaw config validate` уже завершается с ошибкой, сначала запустите `openclaw configure` или `openclaw doctor --fix`; для запуска `openclaw chat` всё равно требуется загружаемая конфигурация.

Типичный цикл:

1. Запустите локальный режим:

```bash
openclaw chat
```

2. Сообщите агенту, что требуется проверить, например:

```text
Сравни конфигурацию аутентификации моего Gateway с документацией и предложи минимальное исправление.
```

3. Используйте команды локальной оболочки для получения точных данных и проверки:

```text
!openclaw config file
!openclaw docs gateway auth token secretref
!openclaw config validate
!openclaw doctor
```

4. Внесите точечные изменения с помощью `openclaw config set` или `openclaw configure`, затем снова запустите `!openclaw config validate`.
5. Если Doctor рекомендует автоматическую миграцию или восстановление, проверьте изменения и запустите `!openclaw doctor --fix`.

Советы:

- Предпочитайте `openclaw config set` или `openclaw configure` ручному редактированию `openclaw.json`.
- `openclaw docs "<query>"` выполняет поиск по актуальному индексу документации с той же машины.
- `openclaw config validate --json` полезна, когда требуется структурированная схема и ошибки SecretRef или разрешимости.

## Вывод инструментов

- Вызовы инструментов отображаются в виде карточек с аргументами и результатами.
- Ctrl+O переключает между свёрнутым и развёрнутым представлениями.
- Во время работы инструментов частичные обновления передаются в ту же карточку в потоковом режиме.

## Цвета терминала

- TUI отображает основной текст ответов ассистента цветом переднего плана терминала по умолчанию, поэтому текст остаётся читаемым как в тёмных, так и в светлых терминалах.
- Если в терминале используется светлый фон и автоопределение работает неверно, задайте `OPENCLAW_THEME=light` перед запуском `openclaw tui`.
- Чтобы вместо этого принудительно использовать исходную тёмную палитру, задайте `OPENCLAW_THEME=dark`.

## История и потоковая передача

- При подключении TUI загружает последнюю историю (по умолчанию 200 сообщений).
- Потоковые ответы обновляются на месте до завершения.
- TUI также отслеживает события инструментов агента для отображения более информативных карточек инструментов.

## Сведения о подключении

- TUI подключается с идентификатором клиента `openclaw-tui` в общем режиме клиента `ui` (тот же режим используют Control UI и WebChat для политики Gateway).
- При повторном подключении отображается системное сообщение; пропуски событий отражаются в журнале.

## Параметры

- `--local`: Запустить с локальной встроенной средой выполнения агента
- `--url <url>`: URL WebSocket Gateway (по умолчанию — `gateway.remote.url` из конфигурации или `ws://127.0.0.1:<port>` для loopback)
- `--token <token>`: Токен Gateway (если требуется)
- `--password <password>`: Пароль Gateway (если требуется)
- `--tls-fingerprint <sha256>`: Ожидаемый отпечаток TLS-сертификата для Gateway с закреплённым `wss://`
- `--session <key>`: Ключ сеанса (по умолчанию: `main` или `global`, если область действия глобальная)
- `--deliver`: Доставлять ответы ассистента провайдеру (по умолчанию отключено)
- `--thinking <level>`: Переопределить уровень рассуждений для отправок
- `--message <text>`: Отправить начальное сообщение после подключения
- `--timeout-ms <ms>`: Время ожидания агента в мс (по умолчанию — `agents.defaults.timeoutSeconds`)
- `--history-limit <n>`: Количество загружаемых записей истории (по умолчанию — `200`)

<Warning>
Если задано `--url`, TUI не использует резервные учётные данные из конфигурации или переменных среды. Явно передайте `--token` или `--password`, а также `--tls-fingerprint`, если целевой узел использует закреплённый сертификат. Отсутствие явно указанных учётных данных является ошибкой. В локальном режиме не передавайте `--url`, `--token`, `--password` или `--tls-fingerprint`.
</Warning>

## Устранение неполадок

После отправки сообщения нет вывода:

- Выполните `/status` в TUI, чтобы убедиться, что Gateway подключён и находится в состоянии ожидания или занят.
- Проверьте журналы Gateway: `openclaw logs --follow`.
- Убедитесь, что агент может запускаться: `openclaw status` и `openclaw models status`.
- Если сообщения должны появляться в канале чата, убедитесь, что TUI был запущен с `--deliver` (включить этот параметр позднее без перезапуска невозможно).

## Устранение неполадок подключения

- `disconnected`: убедитесь, что Gateway запущен, а ваши `--url/--token/--password` указаны верно.
- В средстве выбора нет агентов: проверьте `openclaw agents list` и конфигурацию маршрутизации.
- Средство выбора сеанса пусто: возможно, используется глобальная область действия или сеансов пока нет.
