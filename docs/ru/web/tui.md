## Быстрый старт

### Режим Gateway

1. Запустите Gateway.

```bash
openclaw gateway
```

2. Откройте TUI.

```bash
openclaw tui
```

3. Введите сообщение и нажмите Enter.

Удалённый Gateway:

```bash
openclaw tui --url ws://<host>:<port> --token <gateway-token>
```

Используйте `--password`, если в Gateway применяется аутентификация по паролю.

### Локальный режим

Запустите TUI без Gateway:

```bash
openclaw chat
# или
openclaw tui --local
```

- `openclaw chat` и `openclaw terminal` — псевдонимы для `openclaw tui --local`.
- `--local` нельзя использовать вместе с `--url`, `--token` или `--password`.
- Локальный режим напрямую использует встроенную среду выполнения агента. Большинство локальных инструментов работает, но функции, доступные только через Gateway, недоступны.
- Команда `openclaw` без подкоманды выбирает цель автоматически: при ненастроенной установке запускается первоначальная настройка инференса; при недопустимой конфигурации открываются классические рекомендации Doctor; если настроенный Gateway доступен, эта оболочка TUI открывается в режиме Gateway; в противном случае при наличии настроенной локальной модели она открывается в локальном режиме.

## Что отображается

- Заголовок: URL подключения, текущий агент, текущий сеанс.
- Журнал чата: сообщения пользователя, ответы ассистента, системные уведомления, карточки инструментов.
- Строка состояния: состояние подключения или выполнения (подключение, выполнение, потоковая передача, ожидание, ошибка).
- Нижняя панель: агент + сеанс + модель + состояние цели + think/fast/verbose/trace/reasoning + количество токенов + доставка. Если включён параметр `tui.footer.showRemoteHost`, для удалённых подключений к Gateway также отображается хост подключения.
- Поле ввода: текстовый редактор с автодополнением.

## Ментальная модель: агенты и сеансы

- Агенты имеют уникальные идентификаторы-слаги (например, `main`, `research`). Gateway предоставляет их список.
- Сеансы принадлежат текущему агенту.
- Ключи сеансов хранятся в формате `agent:<agentId>:<sessionKey>`.
  - Если ввести `/session main`, TUI преобразует его в `agent:<currentAgent>:main`.
  - Если ввести `/session agent:other:main`, вы явно переключитесь на этот сеанс агента.
- Область сеансов:
  - `per-sender` (по умолчанию): у каждого агента может быть несколько сеансов.
  - `global`: TUI всегда использует сеанс `global` (список выбора может быть пуст).
- Текущие агент и сеанс всегда отображаются в нижней панели.
- Чтобы показывать хост Gateway для нелокальных подключений по URL, включите эту возможность:

  ```bash
  openclaw config set tui.footer.showRemoteHost true
  ```

  Значение по умолчанию — `false`. Для подключений через интерфейс обратной петли и встроенных локальных подключений метка хоста никогда не отображается.

- Если у сеанса есть [цель](/ru/tools/goal), в нижней панели отображается её компактное состояние:
  `Pursuing goal`, `Goal paused (/goal resume)`, `Goal blocked (/goal resume)` или `Goal achieved`.
- Если TUI в режиме Gateway запущен без `--session`, он возобновляет последний выбранный сеанс для тех же Gateway, агента и области сеансов, если этот сеанс ещё существует. Передача `--session`, `/session`, `/new` или `/reset` по-прежнему задаёт параметры явно.

## Отправка и доставка

- Сообщения всегда отправляются в Gateway (или во встроенную среду выполнения в локальном режиме); доставка ответа ассистента обратно в провайдер чата — отдельный шаг, по умолчанию отключённый.
- TUI — это внутренняя исходная поверхность, подобная WebChat, а не универсальный исходящий канал. Среды выполнения, которым для видимых ответов требуется `tools.message`, могут завершить активный ход TUI с помощью `message.send` без указания цели; явная доставка через провайдера по-прежнему использует обычные настроенные каналы и никогда не переключается на `lastChannel`.
- Режим доставки фиксируется при запуске на весь сеанс TUI: чтобы включить его, запустите TUI с `openclaw tui --deliver`. Команды с косой чертой `/deliver` или переключателя в настройках для изменения этого режима во время сеанса нет; чтобы изменить его, перезапустите TUI.

## Списки выбора и наложения

- Выбор модели: просмотр доступных моделей и установка переопределения для сеанса.
- Выбор агента: выбор другого агента.
- Выбор сеанса: отображает до 50 сеансов текущего агента, обновлённых за последние 7 дней. Используйте `/session <key>`, чтобы перейти к известному более старому сеансу.
- Настройки (`/settings`): переключение развёрнутого вывода инструментов и видимости процесса рассуждения. Эта панель не управляет доставкой.

## Сочетания клавиш

- Enter: отправить сообщение
- Esc: прервать активное выполнение
- Ctrl+C: очистить поле ввода (для выхода нажмите дважды)
- Ctrl+D: выйти
- Ctrl+L: открыть выбор модели
- Ctrl+G: открыть выбор агента
- Ctrl+P: открыть выбор сеанса
- Ctrl+O: переключить развёрнутый вывод инструментов
- Ctrl+T: переключить видимость процесса рассуждения (история перезагружается)

## Команды с косой чертой

Основные:

- `/help`
- `/status` (перенаправляется в Gateway; показывает сводку по сеансу и модели)
- `/gateway-status` (псевдоним: `/gwstatus`; напрямую показывает состояние подключения к Gateway)
- `/agent <id>` (или `/agents`)
- `/session <key>` (или `/sessions`)
- `/model <provider/model>` (или `/models`)

Управление сеансом:

- `/think <off|minimal|low|medium|high>` (на более высоких уровнях в зависимости от модели могут быть доступны дополнительные значения, например `xhigh`/`max`)
- `/fast <status|auto|on|off>`
- `/verbose <on|full|off>`
- `/trace <on|off>`
- `/reasoning <on|off|stream>`
- `/usage <off|tokens|full|reset>` (`reset`/`inherit`/`clear`/`default` сбрасывает переопределение для сеанса)
- `/goal [status] | /goal start <objective> | /goal edit <objective> | /goal pause|resume|complete|block|clear`
- `/elevated <on|off|ask|full>` (псевдоним: `/elev`)
- `/activation <mention|always>`

Жизненный цикл сеанса:

- `/new` (создаёт новый изолированный сеанс с новым ключом; не влияет на другие клиенты TUI в старом сеансе)
- `/reset` (сбрасывает текущий ключ сеанса на месте)
- `/abort` (прерывает активное выполнение)
- `/settings`
- `/exit` (или `/quit`)

Только для локального режима:

- `/auth [provider]` открывает в TUI процесс аутентификации или входа у провайдера.

Crestodian:

- `/crestodian [request]` возвращает из обычного TUI агента в чат помощника [Crestodian](#crestodian-setup-and-repair-helper) по настройке и восстановлению, при необходимости перенаправляя один запрос.

Другие команды Gateway с косой чертой (например, `/context`) перенаправляются в Gateway и отображаются как системный вывод. См. [Команды с косой чертой](/ru/tools/slash-commands).

## Локальные команды оболочки

- Добавьте `!` в начало строки, чтобы выполнить локальную команду оболочки на хосте TUI.
- TUI один раз за сеанс запрашивает разрешение на локальное выполнение; при отказе `!` остаётся отключённым до конца сеанса.
- Команды выполняются в новой неинтерактивной оболочке в рабочем каталоге TUI (без сохранения `cd`/переменных среды).
- Локальные команды оболочки получают `OPENCLAW_SHELL=tui-local` в своём окружении.
- Одиночный `!` отправляется как обычное сообщение; начальные пробелы не запускают локальное выполнение.

## Помощник Crestodian по настройке и восстановлению

Crestodian — это помощник нулевого уровня по настройке и восстановлению, доступный как `openclaw crestodian` после успешной интерактивной проверки инференса настроенной модели по умолчанию. Если инференс недоступен, при интерактивном запуске происходит возврат к первоначальной настройке инференса, а автоматический запуск завершается ошибкой с рекомендациями по восстановлению. Он работает в той же локальной оболочке TUI, что и `openclaw tui --local`, и использует ИИ-агента, ограниченного типизированными операциями Crestodian, требующими подтверждения:

```bash
openclaw crestodian                       # запустить в интерактивном режиме
openclaw crestodian -m "status"           # выполнить один запрос и завершить работу
openclaw crestodian -m "set default model openai/gpt-5.2" --yes   # применить изменение конфигурации
```

- Для постоянных изменений конфигурации требуется подтверждение: подтвердите их интерактивно или передайте `--yes`.
- `--json` выводит обзор запуска в формате JSON вместо запуска чата.
- В Crestodian запрос `open-tui` (например, просьба перейти к обычному агенту) завершает Crestodian и открывает обычный TUI агента; чтобы вернуться, используйте там `/crestodian`.

Используйте локальный режим, если текущая конфигурация уже проходит проверку и вы хотите, чтобы встроенный агент проверил её на том же компьютере, сравнил с документацией и помог устранить расхождения без зависимости от запущенного Gateway.

Если `openclaw config validate` уже завершается с ошибкой, сначала запустите `openclaw configure` или `openclaw doctor --fix`; для запуска `openclaw chat` по-прежнему требуется загружаемая конфигурация.

Типичный цикл:

1. Запустите локальный режим:

```bash
openclaw chat
```

2. Попросите агента выполнить нужную проверку, например:

```text
Сравни мою конфигурацию аутентификации Gateway с документацией и предложи минимальное исправление.
```

3. Используйте локальные команды оболочки для получения точных данных и проверки:

```text
!openclaw config file
!openclaw docs gateway auth token secretref
!openclaw config validate
!openclaw doctor
```

4. Внесите точечные изменения с помощью `openclaw config set` или `openclaw configure`, затем снова запустите `!openclaw config validate`.
5. Если Doctor рекомендует автоматическую миграцию или восстановление, проверьте предложенные изменения и запустите `!openclaw doctor --fix`.

Советы:

- Вместо ручного редактирования `openclaw.json` предпочитайте `openclaw config set` или `openclaw configure`.
- `openclaw docs "<query>"` выполняет поиск по актуальному индексу документации с того же компьютера.
- `openclaw config validate --json` полезен, когда нужны структурированная схема и ошибки SecretRef или разрешимости.

## Вывод инструментов

- Вызовы инструментов отображаются в виде карточек с аргументами и результатами.
- Ctrl+O переключает свёрнутое и развёрнутое представления.
- Во время выполнения инструментов частичные обновления поступают в ту же карточку в потоковом режиме.

## Цвета терминала

- TUI отображает основной текст ответов ассистента цветом переднего плана терминала по умолчанию, чтобы он оставался читаемым как в тёмных, так и в светлых терминалах.
- Если в терминале используется светлый фон, а автоматическое определение срабатывает неверно, перед запуском `openclaw tui` задайте `OPENCLAW_THEME=light`.
- Чтобы принудительно использовать исходную тёмную палитру, задайте `OPENCLAW_THEME=dark`.

## История и потоковая передача

- При подключении TUI загружает последние сообщения истории (по умолчанию 200 сообщений).
- Потоковые ответы обновляются на месте до завершения.
- TUI также отслеживает события инструментов агента для отображения более информативных карточек.

## Сведения о подключении

- TUI подключается с идентификатором клиента `openclaw-tui` в общем режиме клиента `ui` (этот же режим используют Control UI и WebChat для политик Gateway).
- При повторном подключении отображается системное сообщение; пропуски событий отмечаются в журнале.

## Параметры

- `--local`: Запуск с использованием локальной встроенной среды выполнения агента
- `--url <url>`: URL WebSocket для Gateway (по умолчанию — `gateway.remote.url` из конфигурации или `ws://127.0.0.1:<port>` для loopback-интерфейса)
- `--token <token>`: Токен Gateway (если требуется)
- `--password <password>`: Пароль Gateway (если требуется)
- `--tls-fingerprint <sha256>`: Ожидаемый отпечаток TLS-сертификата для Gateway с закреплённым `wss://`
- `--session <key>`: Ключ сеанса (по умолчанию: `main` или `global`, если область действия глобальная)
- `--deliver`: Доставлять ответы ассистента провайдеру (по умолчанию отключено)
- `--thinking <level>`: Переопределить уровень рассуждений для отправляемых сообщений
- `--message <text>`: Отправить начальное сообщение после подключения
- `--timeout-ms <ms>`: Тайм-аут агента в мс (по умолчанию — `agents.defaults.timeoutSeconds`)
- `--history-limit <n>`: Количество загружаемых записей истории (по умолчанию — `200`)

<Warning>
Если задан параметр `--url`, TUI не использует учётные данные из конфигурации или переменных окружения в качестве резервного варианта. Явно передайте `--token` или `--password`, а если целевой узел использует закреплённый сертификат — также `--tls-fingerprint`. Отсутствие явно указанных учётных данных считается ошибкой. В локальном режиме не передавайте `--url`, `--token`, `--password` или `--tls-fingerprint`.
</Warning>

## Устранение неполадок

После отправки сообщения нет вывода:

- Выполните `/status` в TUI, чтобы убедиться, что Gateway подключён и находится в состоянии ожидания или занят.
- Проверьте журналы Gateway: `openclaw logs --follow`.
- Убедитесь, что агент может запускаться: `openclaw status` и `openclaw models status`.
- Если сообщения должны поступать в канал чата, убедитесь, что TUI был запущен с параметром `--deliver` (его нельзя включить позднее без перезапуска).

## Устранение неполадок подключения

- `disconnected`: убедитесь, что Gateway запущен, а ваши `--url/--token/--password` указаны верно.
- В средстве выбора нет агентов: проверьте `openclaw agents list` и конфигурацию маршрутизации.
- Средство выбора сеансов пусто: возможно, используется глобальная область действия или сеансов пока нет.
