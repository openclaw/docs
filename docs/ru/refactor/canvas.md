# Рефакторинг плагина Canvas

Canvas используется редко и имеет экспериментальный статус. Рассматривайте его как встроенный плагин, а не как функцию ядра. В ядре могут оставаться универсальные механизмы Gateway, Node, HTTP, аутентификации, конфигурации и интеграции с нативными клиентами, но поведение, специфичное для Canvas, должно находиться в `extensions/canvas`.

## Цель

Перенести владение Canvas в `extensions/canvas`, сохранив текущее поведение сопряжённого узла:

- инструмент `canvas`, доступный агенту, регистрируется плагином Canvas
- команды узла Canvas разрешены только тогда, когда их регистрирует плагин Canvas
- файлы хоста и исходного кода A2UI находятся в плагине Canvas
- материализация документов Canvas выполняется в плагине Canvas
- реализация команды CLI находится в плагине Canvas или делегирует выполнение через принадлежащий плагину экспортируемый модуль среды выполнения
- документация и реестр плагинов описывают Canvas как экспериментальную функцию, реализованную через плагин

## Что не входит в задачу

- Не перерабатывайте интерфейс Canvas в нативном приложении в рамках этого рефакторинга.
- Не удаляйте поддержку протокола и клиентов Canvas из iOS, Android или macOS, если отдельное продуктовое решение не предписывает удалить Canvas.
- Не создавайте обширную платформу сервисов плагинов только для Canvas, если такой же механизм не требуется как минимум ещё одному встроенному плагину.

## Целевая структура

`extensions/canvas` должен владеть:

- манифестом плагина и метаданными пакета
- регистрацией инструмента агента
- политикой вызова команд узла
- хостом Canvas и средой выполнения A2UI
- исходным кодом пакета Canvas A2UI и скриптами сборки и копирования ресурсов
- созданием документов Canvas и разрешением ресурсов
- реализацией CLI Canvas
- страницей документации Canvas и записью в реестре плагинов

Ядро должно владеть только универсальными механизмами:

- обнаружением и регистрацией плагинов
- универсальным реестром инструментов агента
- универсальным реестром политик вызова узлов
- универсальными механизмами HTTP и аутентификации Gateway, а также диспетчеризацией обновления соединений WebSocket
- универсальным разрешением URL размещаемых поверхностей плагинов
- универсальной регистрацией обработчиков размещаемых медиаресурсов
- универсальным транспортом возможностей узлов
- универсальной интеграцией конфигурации
- универсальным обнаружением обработчиков ресурсов встроенных плагинов

Нативные приложения могут сохранять обработчики команд Canvas в качестве клиентов протокола. Они не владеют средой выполнения плагина.

## Этапы миграции

1. Рассматривать `plugins.entries.canvas.config.host` как принадлежащую плагину поверхность конфигурации.
2. Обновить документацию, чтобы Canvas описывался как экспериментальный встроенный плагин.
3. Запустить целевые тесты Canvas, проверки реестра плагинов, проверки API SDK плагинов, а также проверки сборки и типов, затронутые границами среды выполнения.

## Контрольный список аудита

Перед объявлением рефакторинга завершённым:

- `rg "src/canvas-host|../canvas-host"` не возвращает действующих импортов исходного кода.
- `rg "canvas-tool|createCanvasTool" src` не находит принадлежащей ядру реализации инструмента Canvas.
- `rg "canvas.present|canvas.snapshot|canvas.a2ui" src/gateway` не находит жёстко заданных списков разрешений по умолчанию за пределами тестов универсальной политики плагинов.
- Результат `rg "extensions/canvas/runtime-api" src --glob '!**/*.test.ts'` пуст.
- Результат `rg "canvas-documents" src` пуст.
- Результат `rg "registerNodesCanvasCommands|nodes-canvas" src` пуст; плагин Canvas регистрирует `openclaw nodes canvas` через вложенные метаданные CLI плагина.
- `rg "createCanvasHostHandler|handleA2uiHttpRequest" src/gateway` не обнаруживает владения средой выполнения Gateway.
- `rg "apps/shared/OpenClawKit/Tools/CanvasA2UI|canvas-a2ui-copy|extensions/canvas/src/host/a2ui" scripts .github package.json` находит только оболочки совместимости или пути, принадлежащие плагину.
- `pnpm plugins:inventory:check` завершается успешно.
- `pnpm plugin-sdk:api:check` завершается успешно либо сгенерированные эталонные данные API намеренно обновлены и проверены.
- Целевые тесты Canvas проходят.
- Тесты затронутых направлений проходят для путей хоста Canvas и A2UI.
- В описании PR явно указано, что Canvas имеет экспериментальный статус и реализован через плагин.
