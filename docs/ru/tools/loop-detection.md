OpenClaw использует два взаимодополняющих защитных механизма от повторяющихся шаблонов вызова инструментов,
оба настраиваются в разделе `tools.loopDetection`:

1. **Обнаружение циклов** (`enabled`) — по умолчанию отключено. Отслеживает скользящую
   историю вызовов инструментов на предмет повторяющихся шаблонов и повторных попыток вызова неизвестных инструментов.
2. **Защита после Compaction** (`postCompactionGuard`) — включена, если
   для `enabled` явно не задано значение `false`. Активируется после каждой повторной попытки с Compaction и
   прерывает выполнение, если агент повторяет одну и ту же тройку `(tool, args, result)`
   в пределах окна.

Чтобы отключить оба защитных механизма, задайте `tools.loopDetection.enabled: false`.

## Зачем это нужно

- Обнаруживать повторяющиеся последовательности, которые не приводят к прогрессу.
- Обнаруживать высокочастотные циклы без результата (тот же инструмент, те же входные данные, повторяющиеся
  ошибки).
- Обнаруживать определённые шаблоны повторных вызовов известных инструментов опроса.
- Прерывать циклы «переполнение контекста -> Compaction -> тот же цикл», а не позволять
  им выполняться бесконечно.

## Блок конфигурации

Глобальные значения по умолчанию со всеми документированными полями:

```json5
{
  tools: {
    loopDetection: {
      enabled: false, // master switch for the rolling-history detectors
      historySize: 30,
      warningThreshold: 10,
      criticalThreshold: 20,
      unknownToolThreshold: 10,
      globalCircuitBreakerThreshold: 30,
      detectors: {
        genericRepeat: true,
        knownPollNoProgress: true,
        pingPong: true,
      },
      postCompactionGuard: {
        windowSize: 3, // armed after compaction-retry; runs unless enabled is explicitly false
      },
    },
  },
}
```

Переопределение для отдельного агента (необязательно, в `agents.list[].tools.loopDetection`):

```json5
{
  agents: {
    list: [
      {
        id: "safe-runner",
        tools: {
          loopDetection: {
            enabled: true,
            warningThreshold: 8,
            criticalThreshold: 16,
          },
        },
      },
    ],
  },
}
```

Настройки агента накладываются на глобальный блок поле за полем (включая вложенные
`detectors` и `postCompactionGuard`), поэтому агенту достаточно задать только те
поля, которые требуется изменить.

### Поведение полей

| Поле                             | Значение по умолчанию | Эффект                                                                                                                                                                                                 |
| -------------------------------- | --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `enabled`                        | `false`               | Главный переключатель детекторов скользящей истории. Значение `false` также отключает защиту после Compaction.                                                                                         |
| `historySize`                    | `30`                  | Количество последних вызовов инструментов, сохраняемых для анализа.                                                                                                                                    |
| `warningThreshold`               | `10`                  | Количество повторений, после которого шаблон классифицируется как требующий только предупреждения.                                                                                                     |
| `criticalThreshold`              | `20`                  | Количество повторений, после которого блокируется циклический шаблон без прогресса. При неверной настройке среда выполнения ограничивает это значение так, чтобы оно было выше `warningThreshold`.       |
| `unknownToolThreshold`           | `10`                  | Блокирует повторные вызовы одного и того же недоступного инструмента после указанного количества неудач. Не зависит от `detectors`.                                                                      |
| `globalCircuitBreakerThreshold`  | `30`                  | Глобальный прерыватель отсутствия прогресса для всех детекторов. При неверной настройке среда выполнения ограничивает это значение так, чтобы оно было выше `criticalThreshold`. Не зависит от `detectors`. |
| `detectors.genericRepeat`        | `true`                | Предупреждает о повторных вызовах того же инструмента с теми же аргументами; блокирует, когда эти вызовы также возвращают одинаковые результаты.                                                        |
| `detectors.knownPollNoProgress`  | `true`                | Обнаруживает известные шаблоны опроса без прогресса (`process` с `action: "poll"`/`"log"`, `command_status`).                                                                                           |
| `detectors.pingPong`             | `true`                | Обнаруживает чередующиеся шаблоны «пинг-понг» без прогресса между двумя вызовами.                                                                                                                       |
| `postCompactionGuard.windowSize` | `3`                   | Количество попыток, в течение которых защита остаётся активной после Compaction, а также количество одинаковых троек, после которого выполнение прерывается.                                            |

Для `exec` хеширование отсутствия прогресса сравнивает стабильные результаты команд (состояние,
код выхода, признак истечения времени ожидания, вывод) и игнорирует изменчивые метаданные среды выполнения,
такие как длительность, PID, идентификатор сеанса и рабочий каталог. Результаты исходящей отправки сообщений
хешируются без изменчивых идентификаторов отдельных вызовов (идентификатора сообщения, идентификатора файла, временной метки),
поэтому один результат «отправлено» не выглядит идентичным другому результату «отправлено».
Если доступен идентификатор выполнения, история оценивается только в рамках этого выполнения,
поэтому запланированные циклы Heartbeat и новые выполнения не наследуют устаревшие счётчики циклов
из предыдущих выполнений.

## Рекомендуемая настройка

- Для небольших моделей задайте `enabled: true` и оставьте пороговые значения
  по умолчанию. Передовым моделям редко требуется обнаружение по скользящей истории, поэтому
  главный переключатель можно оставить в значении `false`, сохранив при этом преимущества
  защиты после Compaction.
- Сохраняйте порядок пороговых значений `warningThreshold < criticalThreshold <
globalCircuitBreakerThreshold`; среда выполнения увеличивает `criticalThreshold` и
  `globalCircuitBreakerThreshold`, если их значения равны или ниже
  порогового значения, которое они должны превышать.
- При ложных срабатываниях:
  - Увеличьте `warningThreshold` и/или `criticalThreshold`.
  - При необходимости увеличьте `globalCircuitBreakerThreshold`.
  - Отключите только конкретный детектор, вызывающий проблемы (`detectors.<name>: false`).
  - Уменьшите `historySize`, чтобы сократить историческое окно.
- Чтобы отключить всё, включая защиту после Compaction, явно задайте
  `tools.loopDetection.enabled: false`.

## Защита после Compaction

После повторной попытки с Compaction, вызванной переполнением контекста, исполнитель активирует
защиту с коротким окном для нескольких следующих вызовов инструментов. Если агент выдаёт одну и ту же
тройку `(toolName, argsHash, resultHash)` `postCompactionGuard.windowSize`
раз в пределах этого окна, защита заключает, что Compaction не прервала
цикл, и завершает выполнение с ошибкой `compaction_loop_persisted`.

Защита управляется главным флагом `tools.loopDetection.enabled`, но с одной
особенностью: она остаётся **включённой, если флаг не задан или имеет значение `true`**,
и отключается только тогда, когда флагу явно присвоено значение `false`. Это сделано намеренно:
защита позволяет выйти из циклов Compaction, которые иначе расходовали бы неограниченное количество токенов,
поэтому пользователь без настроенной конфигурации всё равно получает эту защиту.

```json5
{
  tools: {
    loopDetection: {
      // master switch; set false to disable the guard along with the rolling detectors
      enabled: true,
      postCompactionGuard: {
        windowSize: 3, // default
      },
    },
  },
}
```

- Меньшее значение `windowSize` означает более строгую защиту (меньше попыток до прерывания).
- Большее значение `windowSize` даёт агенту больше попыток на восстановление.
- Защита никогда не прерывает выполнение, пока результаты меняются; срабатывание вызывают только
  побайтово идентичные результаты во всём окне.
- Она активируется только непосредственно после повторной попытки с Compaction, а не в других
  точках выполнения.

<Note>
  Защита после Compaction работает всякий раз, когда главный флаг не имеет явно заданного значения `false`, даже если вы никогда не добавляли блок `tools.loopDetection`. Для проверки найдите запись `post-compaction guard armed for N attempts` в журнале Gateway непосредственно после события Compaction.
</Note>

## Журналы и ожидаемое поведение

При обнаружении цикла OpenClaw записывает событие цикла в журнал и либо выдаёт предупреждение, либо блокирует
следующий цикл работы с инструментом в зависимости от серьёзности, предотвращая неконтролируемый
расход токенов и зависания и при этом сохраняя обычный доступ к инструментам.

- Сначала выдаются предупреждения.
- Блокировка происходит, когда шаблон сохраняется после превышения порога предупреждений.
- Критические пороговые значения блокируют следующий цикл работы с инструментом и добавляют понятную
  причину обнаружения цикла в запись выполнения.
- Защита после Compaction выдаёт ошибки `compaction_loop_persisted` с указанием
  проблемного инструмента и количества идентичных вызовов.
