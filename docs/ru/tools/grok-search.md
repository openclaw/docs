OpenClaw поддерживает Grok в качестве провайдера `web_search`, используя ответы xAI с привязкой к веб-источникам для создания синтезированных ИИ ответов на основе актуальных результатов поиска с цитатами.

Для веб-поиска Grok предпочтительно используется существующая авторизация xAI через OAuth, если она доступна. Если профиль OAuth отсутствует, тот же API-ключ xAI также используется встроенным инструментом `x_search` для поиска публикаций в X (ранее Twitter) и инструментом `code_execution`. Хранение ключа в `plugins.entries.xai.config.webSearch.apiKey` также позволяет OpenClaw повторно использовать его в качестве резервного варианта для встроенного провайдера моделей xAI.

Для получения метрик отдельных публикаций X (репостов, ответов, закладок, просмотров) используйте [`x_search`](/ru/tools/web#x_search) с точным URL публикации или идентификатором статуса вместо общего поискового запроса.

## Первоначальная настройка и конфигурирование

Выбор **Grok** во время выполнения `openclaw onboard` или `openclaw configure --section
web` позволяет OpenClaw повторно использовать существующий профиль OAuth xAI, не запрашивая отдельный ключ для веб-поиска. При отсутствии OAuth используется настройка API-ключа xAI.

Затем OpenClaw предлагает дополнительный шаг для включения `x_search` с теми же учетными данными xAI. Этот дополнительный шаг:

- появляется только после выбора Grok для `web_search`
- не является отдельным вариантом провайдера веб-поиска верхнего уровня
- позволяет при необходимости задать модель `x_search` в рамках того же процесса

Пропустите его, чтобы включить или изменить `x_search` позже в конфигурации.

## Вход или получение API-ключа

<Steps>
  <Step title="Использование OAuth xAI">
    Если вы уже вошли в xAI во время первоначальной настройки или аутентификации модели, выберите Grok в качестве провайдера `web_search`. Отдельный API-ключ не требуется:

    ```bash
    openclaw onboard --auth-choice xai-oauth
    openclaw config set tools.web.search.provider grok
    ```

  </Step>
  <Step title="Использование резервного API-ключа">
    Получите API-ключ в [xAI](https://console.x.ai/), если OAuth недоступен или вы намеренно хотите использовать для веб-поиска конфигурацию на основе ключа.
  </Step>
  <Step title="Сохранение ключа">
    Задайте `XAI_API_KEY` в окружении Gateway или выполните настройку с помощью:

    ```bash
    openclaw configure --section web
    ```

  </Step>
</Steps>

## Конфигурация

```json5
{
  plugins: {
    entries: {
      xai: {
        config: {
          webSearch: {
            apiKey: "xai-...", // необязательно, если доступны OAuth xAI или XAI_API_KEY
            baseUrl: "https://api.x.ai/v1", // необязательное переопределение прокси/базового URL Responses API
          },
        },
      },
    },
  },
  tools: {
    web: {
      search: {
        provider: "grok",
      },
    },
  },
}
```

**Варианты учетных данных:** `openclaw models auth login --provider xai
--method oauth`, `XAI_API_KEY` в окружении Gateway или
`plugins.entries.xai.config.webSearch.apiKey`. При установке Gateway поместите переменные окружения в `~/.openclaw/.env`.

## Принцип работы

Grok использует ответы xAI с привязкой к веб-источникам для синтеза ответов со встроенными цитатами, аналогично подходу Gemini с привязкой к Google Поиску.

## Поддерживаемые параметры

Поиск Grok поддерживает `query`. Параметр `count` принимается для совместимости с общим интерфейсом `web_search`, но Grok всегда возвращает один синтезированный ответ с цитатами, а не список из N результатов. Фильтры, специфичные для провайдера, не поддерживаются.

По умолчанию для Grok установлено время ожидания 60 секунд, поскольку поиск xAI Responses с привязкой к веб-источникам может выполняться дольше стандартного времени ожидания общего `web_search`. Переопределите его с помощью `tools.web.search.timeoutSeconds`.

## Переопределение базового URL

Задайте `plugins.entries.xai.config.webSearch.baseUrl`, чтобы направлять веб-поиск Grok через прокси оператора или совместимую с xAI конечную точку Responses. После удаления завершающих косых черт OpenClaw отправляет POST-запросы на `<baseUrl>/responses`. `x_search` использует тот же `webSearch.baseUrl` в качестве резервного варианта, если не задан `plugins.entries.xai.config.xSearch.baseUrl`.
