Самообучение позволяет OpenClaw преобразовывать полезные сведения из разговоров в ожидающие рассмотрения предложения
[Мастерской Skills](/ru/tools/skill-workshop). Оно не обучает веса модели,
не редактирует активные навыки и не изменяет поведение агента без уведомления. Каждая изученная
процедура остается в состоянии ожидания, пока оператор не рассмотрит и не применит ее.

Самообучение **по умолчанию отключено**. Включайте его только тогда, когда дополнительный
фоновый запуск модели и проверка расшифровки разговора допустимы для вашего рабочего пространства.

## Включение самообучения

Используйте CLI:

```bash
openclaw config set skills.workshop.autonomous.enabled true --strict-json
```

Или отредактируйте `~/.openclaw/openclaw.json`:

```json5
{
  skills: {
    workshop: {
      autonomous: {
        enabled: true,
      },
    },
  },
}
```

Чтобы снова отключить его, выполните:

```bash
openclaw config set skills.workshop.autonomous.enabled false --strict-json
```

Создание навыков по запросу пользователя, `/learn` и выполняемые вручную операции Мастерской Skills
продолжают работать, пока самообучение отключено.

## Чему может научиться OpenClaw

Самообучение использует два консервативных механизма:

1. **Прямые инструкции и исправления.** OpenClaw обнаруживает формулировки долгосрочного действия,
   такие как «впредь», «в следующий раз», а также исправления неудачного подхода.
   Когда самообучение включено, он может преобразовывать такие сигналы в ожидающие рассмотрения предложения,
   не дожидаясь следующего запроса. Этот детерминированный механизм может объединить связанные
   инструкции не более чем в три предложения, выбрать доступный для записи навык рабочего пространства
   или изменить собственное связанное предложение, ожидающее рассмотрения. Он также запускается после неудачных
   обращений, поскольку фиксирует инструкции пользователя, а не оценивает завершенность.
2. **Анализ опыта.** После успешного и содержательного основного обращения
   OpenClaw может проанализировать выполненную работу в поисках многократно применимого способа восстановления или
   устойчивой процедуры, которая позволила бы избежать как минимум двух будущих циклов
   взаимодействия с моделью или инструментами.

Подходящими кандидатами могут быть:

- надежный способ восстановления после повторяющихся сбоев инструмента или модели;
- неочевидное ограничение порядка действий, предотвратившее повторяющуюся ошибку;
- устойчивый многоэтапный рабочий процесс, для которого требовалось неоднократно искать нужные сведения; или
- многократно применимая предварительная проверка, позволяющая избежать нескольких будущих вызовов.

Проверяющий процесс должен воздерживаться от создания предложений для обычной успешно выполненной работы, разовых запросов,
личных фактов, простых предпочтений, временных сбоев среды, общих
советов, неподтвержденных отрицательных утверждений и секретов.

## Когда запускается анализ опыта

Анализ опыта намеренно выполняется с задержкой и в установленных пределах:

- Основное обращение должно успешно завершиться.
- Текущее обращение должно содержать не менее десяти итераций модели.
- Сеансы Cron, Heartbeat, памяти, переполнения, хуков, субагентов и проверки
  исключаются.
- Во время основного запуска должны быть определены поставщик и модель, а также должен фактически
  быть доступен `skill_workshop`.
- OpenClaw ожидает 30 секунд после завершения. Если в том же сеансе позднее
  завершается другое основное обращение, этот период ожидания начинается заново.
- Если какой-либо запуск агента или ответа все еще активен, проверка ожидает еще 30 секунд.
- Одновременно выполняется только один анализ опыта.
- Отложенная проверка является локальной для процесса задачей Gateway. Gateway должен продолжать работать
  в течение всего периода бездействия; одноразовые локальные среды выполнения и среды на основе CLI не сохраняют
  достаточно контекста траектории и доступности инструментов для ее планирования.

Основной ответ никогда не задерживается ради обучения. Неудачное или не соответствующее условиям
обращение не запускает анализ опыта, хотя прямые исправления пользователя
по-прежнему могут предлагаться как рекомендация, когда автономный режим отключен.

## Что получает проверяющий процесс

Фоновый проверяющий процесс получает только текущее обращение, начиная с последнего
сообщения пользователя. Отображаемая траектория ограничена 60 000 символами;
при необходимости OpenClaw сохраняет первое сообщение и самые новые сведения,
помечая пропущенную среднюю часть.

Проверяющий процесс повторно использует определенные поставщика и модель. Он повторно использует профиль
аутентификации основного обращения, когда эта идентичность доступна, и отключает резервные модели. Таким
образом, проверка запускает дополнительное обращение к модели у настроенного поставщика.
Этот запуск может выполнить более одного запроса к поставщику при проверке или подготовке
предложения. Условия поставщика в отношении цен и обработки данных применяются так же, как и к
основному обращению.

Перед запуском OpenClaw повторно загружает текущую конфигурацию среды выполнения и еще раз проверяет
действующие политики песочницы и инструментов для исходного разговора. Если запуск выполняется
в песочнице, политика больше не разрешает `skill_workshop` или отсутствуют необходимые сведения
о среде выполнения, проверка завершается с запретом по умолчанию и ничего не создает.

<Warning>
  Включение самообучения разрешает отправлять выбранному поставщику модели подходящее содержимое разговора,
  включая входные данные и результаты инструментов текущего обращения,
  для одной дополнительной проверки. Не включайте его в рабочем пространстве, где
  такая проверка нарушила бы требования к обработке данных.
</Warning>

## Безопасность предложений

Проверяющий процесс выполняется в изолированном сеансе с намеренно ограниченным набором
инструментов:

- Он может только перечислять или просматривать предложения Мастерской, а также создавать или изменять одно
  ожидающее рассмотрения предложение.
- Он не может обновить активный навык, применить, отклонить или поместить в карантин
  предложение, отправить сообщение либо использовать инструменты агента общего назначения.
- Все повторные попытки модели используют единый лимит изменений, поэтому одна проверка может создать или
  изменить не более одного предложения.
- Проверяемая траектория считается недоверенными сведениями, а не инструкциями
  для фонового агента.
- Мастерская Skills сканирует содержимое предложения и отклоняет распознанные буквальные
  учетные данные до записи состояния предложения.

Обычные ограничения Мастерской продолжают действовать, включая `maxPending`, `maxSkillBytes`,
ограничения вспомогательных файлов, проверки сканера и запись только в рабочее пространство. Параметр
`approvalPolicy: "auto"` не предоставляет фоновому проверяющему процессу доступ
к действиям жизненного цикла.

## Проверка изученных предложений

Самообучение создает такие же ожидающие рассмотрения предложения, как и при ручном использовании Мастерской.
Проверьте их перед применением:

```bash
openclaw skills workshop list
openclaw skills workshop inspect <proposal-id>
openclaw skills workshop apply <proposal-id>
```

Изменяйте, отклоняйте или помещайте в карантин полезные, но еще не готовые предложения:

```bash
openclaw skills workshop revise <proposal-id> --proposal ./PROPOSAL.md
openclaw skills workshop reject <proposal-id> --reason "Слишком специфично"
openclaw skills workshop quarantine <proposal-id> --reason "Требуется проверка безопасности"
```

Применение — единственная операция, записывающая активный `SKILL.md`. Полное описание
жизненного цикла и модели хранения см. в разделе
[Мастерская Skills](/ru/tools/skill-workshop).

## Конфигурация

| Параметр                                   | Значение по умолчанию | Влияние на самообучение                                                                                                           |
| ------------------------------------------ | ----------- | --------------------------------------------------------------------------------------------------------------------------------- |
| `skills.workshop.autonomous.enabled`       | `false`     | Включает фиксацию прямых исправлений и отложенный анализ опыта.                                                                    |
| `skills.workshop.approvalPolicy`           | `"pending"` | Управляет запросами подтверждения для обычных действий жизненного цикла, инициированных агентом; не расширяет разрешения фонового проверяющего процесса. |
| `skills.workshop.maxPending`               | `50`        | Ограничивает количество ожидающих рассмотрения и помещенных в карантин предложений для каждого рабочего пространства.              |
| `skills.workshop.maxSkillBytes`            | `40000`     | Ограничивает размер тела предложения в байтах.                                                                                     |
| `skills.workshop.allowSymlinkTargetWrites` | `false`     | Влияет только на применение; самообучение записывает состояние предложения, а не активные целевые навыки.                           |

Полную схему, диапазоны и связанные параметры навыков см. в разделе
[Конфигурация Skills](/ru/tools/skills-config#workshop-skills-workshop).

## Устранение неполадок

### После длительного обращения предложение не появилось

Проверьте все перечисленные условия:

1. `skills.workshop.autonomous.enabled` имеет значение `true` в активной конфигурации Gateway.
2. Обращение завершилось успешно и содержало не менее десяти итераций модели после последнего
   сообщения пользователя.
3. Разговор был обычным основным запуском, а не запланированным запуском, запуском памяти,
   хука или субагента.
4. Исходному запуску был доступен `skill_workshop`, и он выполнялся не в песочнице.
5. Система оставалась бездействующей достаточно долго для выполнения отложенной проверки.
6. Долгоживущий процесс Gateway оставался активным в течение периода бездействия; одноразовая
   локальная команда не ожидает отложенную проверку.

Даже соответствующая условиям проверка может не создать предложение. Отказ от создания является ожидаемым
результатом, когда сведения не соответствуют требованиям к многократно применимой процедуре.

### Doctor сообщает, что инструмент Мастерской скрыт

Когда самообучение включено, `openclaw doctor` проверяет, разрешает ли действующая
политика инструментов агента по умолчанию использовать `skill_workshop`. Внесите указанное
изменение `tools.allow` или `tools.alsoAllow` либо отключите самообучение.

### Появляется слишком много малоценных предложений

Отключите самообучение и продолжайте использовать `/learn` или явные запросы к Мастерской:

```bash
openclaw config set skills.workshop.autonomous.enabled false --strict-json
```

Ожидающие рассмотрения предложения остаются доступными для проверки после отключения функции. Отключение
самообучения не применяет, не отклоняет и не удаляет их.
