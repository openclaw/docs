Самообучение позволяет OpenClaw преобразовывать полезные сведения из разговоров в ожидающие рассмотрения
предложения [Skill Workshop](/ru/tools/skill-workshop). Оно не обучает веса модели,
не изменяет активные навыки и не меняет поведение агента без уведомления. Каждая изученная
процедура остаётся в состоянии ожидания, пока оператор не проверит и не применит её.

Самообучение **по умолчанию отключено**. Включайте его только в том случае, если дополнительный
фоновый запуск модели и проверка расшифровки разговора допустимы для вашего рабочего пространства.

## Включение самообучения

В Control UI откройте **Plugins → Workshop** и включите **Self-learning**. Изменение
вступает в силу немедленно; если другой процесс записи конфигурации обновил
файл, Control UI обновит снимок конфигурации и повторит попытку переключения без
перезагрузки страницы или Gateway.

Используйте CLI:

```bash
openclaw config set skills.workshop.autonomous.enabled true --strict-json
```

Или измените `~/.openclaw/openclaw.json`:

```json5
{
  skills: {
    workshop: {
      autonomous: {
        enabled: true,
      },
    },
  },
}
```

Чтобы снова отключить эту функцию, выполните:

```bash
openclaw config set skills.workshop.autonomous.enabled false --strict-json
```

Создание навыков по запросу пользователя, `/learn` и ручные операции Skill Workshop
продолжают работать, пока самообучение отключено.

## Ручная проверка прошлых сеансов

Ручная проверка истории — консервативная альтернатива автономному сбору.
Откройте **Plugins → Workshop** в Control UI и выберите **Find skill ideas**.
Это не изменяет `skills.workshop.autonomous.enabled`.

При каждом сканировании:

- процесс начинается с самых новых непроверенных сеансов и движется назад;
- проверяется до 20 содержательных сеансов, включающих не менее шести обращений к модели;
- пропускаются сеансы Cron, Heartbeat, хуков, субагентов, ACP, принадлежащие плагинам и внутренние
  сеансы проверки;
- распознанные секреты маскируются, а размер пакета расшифровок ограничивается перед его отправкой
  настроенной модели выбранного агента;
- применяется тот же строгий порог, что и при автономной проверке опыта; и
- может быть создано или изменено не более трёх ожидающих предложений, но никогда не активных навыков.

Workshop сообщает совокупное количество сеансов, охваченный период и число найденных идей.
Выберите **Scan earlier work**, чтобы перейти к следующему более старому диапазону. Когда курсор достигает
начала подходящей истории, действие меняется на **Scan new work**.
OpenClaw сохраняет в общей базе данных состояния только курсор и метаданные охвата;
второй архив расшифровок не создаётся.

Сеансы сканируются только тогда, когда OpenClaw может подтвердить их принадлежность и исключить
содержимое внешних хуков. После обновления текущую расшифровку, созданную до обновления, можно
классифицировать локально, но ротированные расшифровки, созданные до обновления и не содержащие сведений
о происхождении каждого запуска, пропускаются. Новые расшифровки сохраняют эти сведения при ротации.

Ручное сканирование также влечёт расходы на провайдера модели и отправляет подходящее содержимое
разговоров настроенному провайдеру. Используйте его только в том случае, если такая проверка соответствует
требованиям рабочего пространства к конфиденциальности и обработке данных.

## Чему может научиться OpenClaw

У самообучения есть два консервативных пути:

1. **Прямые инструкции и исправления.** OpenClaw обнаруживает долговременные формулировки,
   такие как «впредь», «в следующий раз», а также исправления неудачного подхода.
   Когда самообучение включено, он может преобразовывать эти сигналы в ожидающие предложения,
   не дожидаясь следующего запроса. Этот детерминированный путь может объединять связанные
   инструкции максимум в три предложения, нацеливаться на доступный для записи навык рабочего пространства
   или изменять собственное связанное ожидающее предложение. Он также запускается после неудачных обращений,
   поскольку фиксирует инструкции пользователя, а не оценивает завершённость.
2. **Проверка опыта.** После успешного содержательного обращения переднего плана
   OpenClaw может проверить выполненную работу на наличие повторно используемого способа восстановления или
   устойчивой процедуры, которая позволила бы избежать как минимум двух будущих циклов
   обращения к модели или инструменту.

Подходящие примеры:

- надёжное восстановление после повторяющихся сбоев инструмента или модели;
- неочевидное ограничение порядка действий, предотвратившее повторяющуюся ошибку;
- устойчивый многоэтапный рабочий процесс, который требовал повторного исследования; или
- повторно используемая предварительная проверка, позволяющая избежать нескольких будущих вызовов.

Проверяющий должен воздержаться от создания предложения при обычной успешной работе, разовых запросах,
личных фактах, простых предпочтениях, временных сбоях среды, общих
рекомендациях, неподтверждённых отрицательных утверждениях и наличии секретов.

## Когда запускается проверка опыта

Проверка опыта намеренно выполняется с задержкой и в ограниченном объёме:

- Обращение переднего плана должно завершиться успешно.
- Текущее обращение должно содержать не менее десяти итераций модели.
- Сеансы Cron, Heartbeat, памяти, переполнения, хуков, субагентов и проверки
  исключаются.
- В ходе запуска переднего плана должны быть определены провайдер и модель, а также фактически
  должен быть доступен `skill_workshop`.
- OpenClaw ожидает 30 секунд после завершения. Более позднее завершение обращения переднего плана в
  том же сеансе перезапускает этот период бездействия.
- Если какой-либо запуск агента или ответа всё ещё активен, проверка ожидает ещё 30 секунд.
- Одновременно выполняется только одна проверка опыта.
- Отложенная проверка — это локальная для процесса работа Gateway. Gateway должен продолжать работать
  в течение периода бездействия; одноразовые локальные среды выполнения и среды на базе CLI не сохраняют
  достаточно контекста траектории и доступности инструментов для её планирования.

Ответ переднего плана никогда не задерживается ради обучения. Неудачное или неподходящее
обращение не запускает проверку опыта, хотя прямые исправления пользователя
по-прежнему могут предлагаться как рекомендация, когда автономность отключена.

## Что получает проверяющий

Фоновый проверяющий получает только текущее обращение, начиная с самого
последнего сообщения пользователя. Отображаемая траектория ограничена 60,000 символами;
при необходимости OpenClaw сохраняет первое сообщение и самые новые сведения,
отмечая пропущенную середину.

Проверяющий повторно использует определённые провайдер и модель. Он повторно использует профиль
аутентификации переднего плана, когда эта идентификация доступна, и отключает резервные модели. Поэтому
проверка запускает дополнительное обращение к модели у настроенного провайдера.
Этот запуск может выполнить более одного запроса к провайдеру при проверке или составлении
предложения. Тарифы провайдера и условия обработки данных применяются так же, как и к
обращению переднего плана.

Перед запуском OpenClaw повторно загружает текущую конфигурацию среды выполнения и заново проверяет
действующие политики песочницы и инструментов для исходного разговора. Если запуск выполняется
в песочнице, политика больше не разрешает `skill_workshop` или отсутствуют необходимые сведения
о среде выполнения, проверка завершается безопасным отказом и ничего не создаёт.

<Warning>
  Включение самообучения разрешает отправлять подходящее содержимое разговора, включая входные
  данные инструментов и результаты текущего обращения, выбранному провайдеру
  модели для одной дополнительной проверки. Не включайте его в рабочем пространстве, где
  такая проверка нарушит требования к обработке данных.
</Warning>

## Безопасность предложений

Проверяющий работает в изолированном сеансе с намеренно ограниченным набором
инструментов:

- Он может только перечислять или просматривать предложения Workshop и создавать или изменять одно
  ожидающее предложение.
- Он не может обновлять активный навык, применять, отклонять или помещать предложение в карантин,
  отправлять сообщения или использовать инструменты агента общего назначения.
- Один бюджет изменений распределяется между повторными попытками модели, поэтому проверка может создать или
  изменить не более одного предложения.
- Проверяемая траектория рассматривается как недоверенные сведения, а не как инструкции
  для фонового агента.
- Skill Workshop сканирует содержимое предложения и отклоняет распознанные буквальные
  учётные данные до записи состояния предложения.

Обычные ограничения Workshop продолжают действовать, включая `maxPending`, `maxSkillBytes`,
ограничения вспомогательных файлов, проверки сканера и запись только в рабочее пространство. Параметр
`approvalPolicy: "auto"` не предоставляет фоновому проверяющему доступ
к действиям жизненного цикла.

## Проверка изученных предложений

Самообучение создаёт такие же ожидающие предложения, как и ручное использование Workshop.
Проверьте их перед применением:

```bash
openclaw skills workshop list
openclaw skills workshop inspect <proposal-id>
openclaw skills workshop apply <proposal-id>
```

Изменяйте, отклоняйте или помещайте в карантин полезные, но ещё не готовые предложения:

```bash
openclaw skills workshop revise <proposal-id> --proposal ./PROPOSAL.md
openclaw skills workshop reject <proposal-id> --reason "Too specific"
openclaw skills workshop quarantine <proposal-id> --reason "Needs security review"
```

Применение — единственная операция, записывающая активный `SKILL.md`. Полная модель
жизненного цикла и хранения описана в разделе
[Skill Workshop](/ru/tools/skill-workshop).

## Конфигурация

| Параметр                                   | По умолчанию | Влияние на самообучение                                                                                                           |
| ------------------------------------------ | ------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| `skills.workshop.autonomous.enabled`       | `false`  | Включает прямой сбор исправлений и отложенную проверку опыта.                                                                      |
| `skills.workshop.approvalPolicy`           | `"auto"` | Управляет запросами подтверждения для обычных действий жизненного цикла, инициированных агентом; не расширяет разрешения фонового проверяющего. |
| `skills.workshop.maxPending`               | `50`     | Ограничивает количество ожидающих и помещённых в карантин предложений в одном рабочем пространстве.                                |
| `skills.workshop.maxSkillBytes`            | `40000`  | Ограничивает размер содержимого предложения в байтах.                                                                              |
| `skills.workshop.allowSymlinkTargetWrites` | `false`  | Влияет только на применение; самообучение записывает состояние предложения, а не целевые активные навыки.                           |

Полную схему, диапазоны и связанные параметры навыков см. в разделе
[Конфигурация Skills](/ru/tools/skills-config#workshop-skills-workshop).

## Устранение неполадок

### После длительного обращения предложение не появляется

Проверьте всё перечисленное ниже:

1. `skills.workshop.autonomous.enabled` имеет значение `true` в активной конфигурации Gateway.
2. Обращение завершилось успешно и включало не менее десяти итераций модели после самого
   последнего сообщения пользователя.
3. Разговор был обычным запуском переднего плана, а не запланированным запуском, запуском памяти,
   хука или субагента.
4. Исходный запуск имел доступ к `skill_workshop` и выполнялся не в песочнице.
5. Система оставалась неактивной достаточно долго для выполнения отложенной проверки.
6. Долгоживущий процесс Gateway оставался активным в течение периода бездействия; одноразовая
   локальная команда не ожидает отложенной проверки.

Даже подходящая проверка может не создать предложение. Воздержание — ожидаемый
результат, если сведения не соответствуют порогу повторно используемой процедуры.

### Doctor сообщает, что инструмент Workshop скрыт

Когда самообучение включено, `openclaw doctor` проверяет, разрешает ли действующая
политика инструментов агента по умолчанию использовать `skill_workshop`. Выполните указанное
изменение `tools.allow` или `tools.alsoAllow` либо отключите самообучение.

### Появляется слишком много малоценных предложений

Отключите самообучение и продолжайте использовать `/learn` или явные запросы Workshop:

```bash
openclaw config set skills.workshop.autonomous.enabled false --strict-json
```

После отключения функции ожидающие предложения остаются доступными для проверки. Отключение
самообучения не применяет, не отклоняет и не удаляет их.
