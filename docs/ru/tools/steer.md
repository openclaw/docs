`/steer` сначала пытается отправить указания уже активному запуску. Эта команда предназначена для
ситуаций, когда нужно «скорректировать этот запуск, пока он ещё выполняется». Если текущая среда выполнения
не может принять корректирующие указания, OpenClaw отправляет сообщение как обычный запрос,
а не отбрасывает его.

## Текущий сеанс

Используйте команду верхнего уровня `/steer`, чтобы обратиться к активному запуску текущего сеанса:

```text
/steer prefer the smaller patch and keep the tests focused
/tell summarize before making the next tool call
```

Поведение:

- Обращается только к активному запуску текущего сеанса.
- Работает независимо от режима `/queue` этого сеанса.
- Запускает обычный ход с тем же сообщением, если сеанс бездействует или
  активный запуск не может принять корректирующие указания.
- Использует механизм корректировки активной среды выполнения, поэтому модель получает указания
  на следующей поддерживаемой границе среды выполнения.

## Корректировка и очередь

В режиме `/queue steer` обычные входящие сообщения при поступлении пытаются скорректировать
активный запуск, если он выполняется. `/steer <message>` — это явная команда,
которая пытается внедрить своё сообщение в активный запуск на следующей
поддерживаемой границе среды выполнения независимо от сохранённой настройки `/queue`. Если
такое внедрение недоступно, префикс команды удаляется, а `<message>`
обрабатывается как обычный запрос.

Использование:

- `/steer <message>` — если нужно скорректировать активный запуск прямо сейчас.
- `/queue steer` — если нужно, чтобы будущие обычные сообщения по умолчанию
  корректировали активные запуски.
- `/queue collect` или `/queue followup` — если будущие обычные сообщения должны ожидать
  следующего хода, а не корректировать активный запуск.
- `/queue interrupt` — если новейшее сообщение должно заменить активный запуск,
  а не корректировать его.

Описание режимов очереди и границ корректировки см. в разделах [Очередь команд](/ru/concepts/queue) и
[Очередь корректировки](/ru/concepts/queue-steering).

## Субагенты

Команда верхнего уровня `/steer` обращается к активному запуску текущего сеанса. Субагенты отправляют
результаты обратно в родительский сеанс или сеанс-инициатор; `/subagents` служит только для просмотра.

## Сеансы ACP

Используйте `/acp steer`, если целевым является сеанс среды ACP:

```text
/acp steer --session agent:main:acp:codex tighten the repro
```

Сведения о выборе сеанса ACP и поведении среды выполнения см. в разделе
[Агенты ACP](/ru/tools/acp-agents).
