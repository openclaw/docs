`/steer` сначала пытается отправить указания уже активному запуску. Эта команда предназначена для моментов вида «скорректировать этот запуск, пока он еще работает». Если текущая среда выполнения не может принять steering, OpenClaw вместо отбрасывания сообщения отправляет его как обычный prompt.

## Текущий сеанс

Используйте верхнеуровневую команду `/steer`, чтобы нацелиться на активный запуск текущего сеанса:

```text
/steer prefer the smaller patch and keep the tests focused
/tell summarize before making the next tool call
```

Поведение:

- Нацеливается только на активный запуск текущего сеанса.
- Работает независимо от режима `/queue` сеанса.
- Запускает обычный ход с тем же сообщением, когда сеанс бездействует или активный запуск не может принять steering.
- Использует путь steering активной среды выполнения, поэтому модель видит указания на следующей поддерживаемой границе среды выполнения.

## Steer и очередь

`/queue steer` заставляет обычные входящие сообщения пытаться направлять активный запуск, когда они поступают во время активного запуска. `/steer <message>` — это явная команда, которая пытается внедрить сообщение этой команды в активный запуск на следующей поддерживаемой границе среды выполнения, независимо от сохраненной настройки `/queue`. Когда такое внедрение недоступно, префикс команды удаляется, а `<message>` продолжается как обычный prompt.

Используйте:

- `/steer <message>`, когда хотите направить активный запуск прямо сейчас.
- `/queue steer`, когда хотите, чтобы будущие обычные сообщения по умолчанию направляли активные запуски.
- `/queue collect` или `/queue followup`, когда будущие обычные сообщения должны ждать более позднего хода вместо steering активного запуска.
- `/queue interrupt`, когда новейшее сообщение должно заменить активный запуск вместо steering.

О режимах очереди и границах steering см. [Очередь команд](/ru/concepts/queue) и [Очередь steering](/ru/concepts/queue-steering).

## Субагенты

Верхнеуровневая команда `/steer` нацеливается на активный запуск текущего сеанса. Субагенты отчитываются обратно в свой родительский сеанс или сеанс запрашивающего; `/subagents` предназначена только для видимости.

## Сеансы ACP

Используйте `/acp steer`, когда целью является сеанс ACP harness:

```text
/acp steer --session agent:main:acp:codex tighten the repro
```

См. [Агенты ACP](/ru/tools/acp-agents) для выбора сеанса ACP и поведения среды выполнения.
