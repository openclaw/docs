Plugin Parallel предоставляет два провайдера `web_search` от [Parallel](https://parallel.ai/), каждый из которых возвращает ранжированные, оптимизированные для LLM фрагменты из веб-индекса, созданного для ИИ-агентов:

| Провайдер                 | id              | Аутентификация                                                                                       |
| ------------------------- | --------------- | ---------------------------------------------------------------------------------------------------- |
| Parallel Search (бесплатный) | `parallel-free` | Не требуется — бесплатный [Search MCP](https://docs.parallel.ai/integrations/mcp/search-mcp) от Parallel |
| Parallel Search           | `parallel`      | `PARALLEL_API_KEY` — платный Search API, более высокие лимиты запросов и настройка цели               |

Задайте для `tools.web.search.provider` значение `parallel-free` или `parallel`, чтобы явно выбрать один из них; ни один из них не определяется автоматически.

<Note>
  Модели прямого доступа OpenAI Responses (`api: "openai-responses"`, провайдер
  `openai`, официальный базовый URL API) автоматически используют встроенный
  веб-поиск OpenAI, когда `tools.web.search.provider` не задан, пуст, имеет
  значение `"auto"` или `"openai"`, поэтому по умолчанию они обходят Parallel.
  Задайте для `tools.web.search.provider` значение `parallel-free` или
  `parallel`, чтобы вместо этого направлять запросы через Parallel. См.
  [обзор веб-поиска](/ru/tools/web).
</Note>

## Установка плагина

```bash
openclaw plugins install @openclaw/parallel-plugin
openclaw gateway restart
```

## Ключ API (платный провайдер)

Для `parallel-free` ключ не требуется, но этот провайдер всё равно необходимо выбрать явно. Для платного провайдера `parallel` нужен ключ API:

<Steps>
  <Step title="Создание учётной записи">
    Зарегистрируйтесь на [platform.parallel.ai](https://platform.parallel.ai) и
    создайте ключ API в панели управления.
  </Step>
  <Step title="Сохранение ключа">
    Задайте `PARALLEL_API_KEY` в окружении Gateway или выполните настройку с помощью:

    ```bash
    openclaw configure --section web
    ```

  </Step>
</Steps>

## Конфигурация

```json5
{
  plugins: {
    entries: {
      parallel: {
        config: {
          webSearch: {
            apiKey: "par-...", // необязательно, если задан PARALLEL_API_KEY
            baseUrl: "https://api.parallel.ai", // необязательно; OpenClaw добавляет /v1/search
          },
        },
      },
    },
  },
  tools: {
    web: {
      search: {
        // "parallel-free" для бесплатного Search MCP или "parallel" для
        // показанного здесь платного провайдера на основе API.
        provider: "parallel",
      },
    },
  },
}
```

**Альтернатива через переменную окружения:** задайте `PARALLEL_API_KEY` в окружении Gateway. При установке Gateway поместите его в `~/.openclaw/.env`.

## Переопределение базового URL

Применяется только к платному провайдеру `parallel`; `parallel-free` всегда использует `https://search.parallel.ai/mcp` и игнорирует эту настройку.

Задайте `plugins.entries.parallel.config.webSearch.baseUrl`, чтобы направлять платные запросы через совместимый прокси-сервер или альтернативную конечную точку (например, Cloudflare AI Gateway). OpenClaw нормализует имена хостов без схемы, добавляя `https://`, и добавляет `/v1/search`, если путь ещё не заканчивается этой строкой. Разрешённая конечная точка входит в ключ кеша поиска, поэтому результаты от разных конечных точек никогда не используются совместно.

## Параметры инструмента

Оба провайдера предоставляют нативную структуру поиска Parallel, в которой модель указывает цель на естественном языке и несколько коротких поисковых запросов по ключевым словам — сочетание, которое Parallel [рекомендует](https://docs.parallel.ai/search/best-practices) для получения наилучших результатов.

<ParamField path="objective" type="string" required>
Описание основного вопроса или цели на естественном языке (не более 5000 символов). Должно быть самодостаточным.
</ParamField>

<ParamField path="search_queries" type="string[]" required>
Краткие поисковые запросы по ключевым словам, по 3–6 слов каждый (1–5 элементов, не более 200 символов каждый). Для получения наилучших результатов укажите 2–3 разнообразных запроса.
</ParamField>

<ParamField path="count" type="number">
Количество возвращаемых результатов (1–40).
</ParamField>

<ParamField path="session_id" type="string">
Необязательный идентификатор сеанса Parallel из поля `sessionId` предыдущего результата. Передавайте его при последующих поисках в рамках той же задачи, чтобы Parallel группировал связанные вызовы и улучшал последующие результаты. Максимальная длина для `parallel` — 1000 символов, а бесплатный Search MCP `parallel-free` ограничивает её 100 символами. Идентификатор, превышающий ограничение, отбрасывается в платном варианте или заменяется новым в бесплатном.
</ParamField>

<ParamField path="client_model" type="string">
Необязательный идентификатор модели, выполняющей вызов (например, `claude-opus-4-7` или `gpt-5.6-sol`), не более 100 символов. Позволяет Parallel адаптировать настройки по умолчанию к возможностям вашей модели. Передавайте точный идентификатор активной модели; не сокращайте его до псевдонима семейства.
</ParamField>

## Примечания

- Parallel ранжирует и сжимает результаты для удобства рассуждений LLM, а не для переходов пользователей по ссылкам; ожидайте насыщенные фрагменты для каждого результата, а не полное содержимое страниц.
- Фрагменты результатов возвращаются в массиве `excerpts`, а также объединяются в поле `description` для совместимости с универсальным контрактом `web_search`.
- Оба провайдера возвращают `session_id`; OpenClaw предоставляет его как `sessionId` в полезной нагрузке инструмента, чтобы вызывающие стороны могли группировать последующие поиски. Идентификатор сеанса, созданный Parallel (то есть не переданный вызывающей стороной), исключается из записи кеша, поскольку несвязанные задачи с одинаковыми запросами не должны наследовать его.
- Поля `searchId`, `warnings` и `usage` от Parallel передаются без изменений при их наличии.
- OpenClaw всегда передаёт Parallel вычисленное количество результатов как `advanced_settings.max_results` (`parallel`) или применяет `count` на стороне клиента после ответа Parallel фиксированного размера (`parallel-free`). Приоритет имеет аргумент `count` вызывающей стороны, затем `tools.web.search.maxResults`, а иначе используется стандартное значение универсального `web_search` в OpenClaw (5); собственное значение API Parallel по умолчанию — 10.
- По умолчанию результаты кешируются на 15 минут (`cacheTtlMinutes`).
- Если вызывающая сторона не передаёт `session_id`, `parallel-free` создаёт новый идентификатор для каждого вызова посредством согласования MCP, а `parallel` в этом случае оставляет его незаданным.
