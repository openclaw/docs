`/btw` (псевдоним `/side`) позволяет задать короткий побочный вопрос о **текущем
сеансе**, не добавляя его в историю диалога. Эта функция создана по образцу
`/btw` из Claude Code и адаптирована для Gateway и многоканальной
архитектуры OpenClaw.

```text
/btw что изменилось?
/side что означает эта ошибка?
```

## Что делает эта функция

1. Создает снимок текущего сеанса для использования в качестве фонового контекста (включая
   запрос выполняющегося основного запуска).
2. Выполняет отдельный однократный побочный запрос, предписывая модели ответить только на
   побочный вопрос, не возобновляя и не направляя основную задачу.
3. Доставляет ответ как оперативный побочный результат, а не как обычное сообщение ассистента.
4. Никогда не записывает вопрос или ответ в историю сеанса или `chat.history`.

Основной запуск, если он активен, остается без изменений.

Для сеансов среды Codex BTW создает ответвление активного потока app-server Codex во
временный дочерний поток вместо выполнения отдельного вызова провайдера. Это
сохраняет OAuth Codex и штатное поведение инструментов и потоков, а ответвленный
поток сохраняет текущую политику подтверждений родительского потока, песочницу и штатный
набор инструментов. Ответвленный поток получает граничный запрос, сообщающий модели, что
всё предшествующее содержимое является унаследованным справочным контекстом, а не активными инструкциями,
и что активными являются только сообщения после границы. Для `/btw` требуется
существующий поток Codex; сначала отправьте обычное сообщение.

Для псевдонимов среды выполнения CLI BTW вызывает соответствующий сервер CLI в режиме
однократного побочного вопроса: он передает очищенный контекст диалога в новый вызов CLI
с отключенными объединением инструментов и повторно используемым состоянием сеанса, а также добавляет
поддерживаемые сервером флаги запрета возобновления и использования инструментов. Прямые среды выполнения
(не CLI) вместо этого используют прямой однократный вызов провайдера.

## Чего эта функция не делает

`/btw` не создает постоянный сеанс, не продолжает незавершенную основную задачу,
не сохраняет данные вопроса и ответа в истории транскрипта и не переживает перезагрузку.

## Модель доставки

Обычный чат с ассистентом использует событие Gateway `chat`. BTW использует отдельное
событие `chat.side_result`, поэтому клиенты не могут принять его за обычную
историю диалога. Поскольку оно не воспроизводится из `chat.history`, оно
исчезает после перезагрузки.

## Поведение в интерфейсах

| Интерфейс         | Поведение                                                                                                                                                                                                                                                                           |
| ----------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| TUI               | Отображается непосредственно в журнале чата, заметно отличается от обычного ответа; его можно закрыть с помощью `Enter` или `Esc`.                                                                                                                                                 |
| Внешние каналы    | Доставляется как однократный ответ с понятной меткой (в Telegram, WhatsApp и Discord нет локального временного оверлея).                                                                                                                                                              |
| Control UI / веб  | Отображается как плавающая панель «Побочный чат», закрепленная за потоком. Ответы накапливаются в виде реплик, а поле «Уточнить» позволяет задать следующий побочный вопрос. Закрытие (`Esc` или кнопкой X) сохраняет диалог, и панель снова открывается при следующем ответе; кнопка корзины удаляет его и останавливает ожидающий выполнения запуск. |

## Всплывающее меню выделения (Control UI)

При выделении текста внутри сообщения чата в Control UI открывается небольшое
всплывающее меню с двумя действиями:

- **Подробнее** немедленно отправляет неявный вопрос `/btw` с просьбой
  к модели объяснить выделенный текст в контексте текущего
  сеанса. Ответ появляется в плавающей панели побочного чата.
- **Спросить в побочном чате** предварительно заполняет поле ввода черновиком `/btw` с цитатой
  выделенного текста, чтобы вы могли ввести собственный вопрос о нем.

Оба действия следуют обычной семантике `/btw`: вопрос и ответ не попадают
в историю сеанса, а основной запуск остается без изменений.

## Когда использовать

Используйте `/btw` для быстрого уточнения, получения фактического побочного ответа во время
продолжающегося длительного запуска или временного ответа, который не должен попадать в будущий
контекст сеанса.

```text
/btw какой файл мы редактируем?
/btw резюмируй текущую задачу одним предложением
/btw сколько будет 17 * 19?
```

Если информация должна стать частью будущего рабочего
контекста сеанса, задайте вопрос обычным способом в основном сеансе.
