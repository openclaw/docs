`code_execution` выполняет изолированный удалённый анализ на Python через Responses API xAI
(`https://api.x.ai/v1/responses`, ту же конечную точку использует `x_search`). Он
регистрируется встроенным плагином `xai` в рамках контракта `tools`.

<Warning>
  `code_execution` выполняется на серверах xAI. xAI взимает 5 долларов за 1 000 вызовов инструмента,
  а также плату за входные и выходные токены модели.
</Warning>

| Свойство             | Значение                                                                          |
| -------------------- | --------------------------------------------------------------------------------- |
| Название инструмента | `code_execution`                                                                  |
| Плагин провайдера    | `xai` (встроенный, `enabledByDefault: true`)                                      |
| Аутентификация       | профиль аутентификации xAI, `XAI_API_KEY` или `plugins.entries.xai.config.webSearch.apiKey` |
| Модель по умолчанию  | `grok-4.3`                                                                        |
| Тайм-аут по умолчанию | 30 секунд                                                                        |
| `maxTurns` по умолчанию | не задано (xAI применяет собственное внутреннее ограничение)                   |

Используйте его для вычислений, составления таблиц, быстрой статистики и
анализа с построением диаграмм, в том числе для данных, возвращённых `x_search`
или `web_search`. У него нет доступа к локальным файлам, вашей оболочке,
репозиторию или сопряжённым устройствам, и он не сохраняет состояние между
вызовами, поэтому рассматривайте каждый вызов как одноразовый анализ, а не как
сеанс блокнота. Для получения актуальных данных X сначала запустите
[`x_search`](/ru/tools/web#x_search) и передайте ему результат.

Для локального выполнения вместо этого используйте [`exec`](/ru/tools/exec).

## Настройка

<Steps>
  <Step title="Укажите учётные данные xAI">
    Для OAuth требуется подходящая подписка SuperGrok или X Premium
    (проверка по коду устройства, поэтому способ работает на удалённых узлах
    без обратного вызова localhost):

    ```bash
    openclaw models auth login --provider xai --method oauth
    ```

    При новой установке тот же вариант доступен во время первоначальной настройки:

    ```bash
    openclaw onboard --install-daemon --auth-choice xai-oauth
    ```

    Или используйте ключ API:

    ```bash
    openclaw models auth login --provider xai --method api-key
    export XAI_API_KEY=xai-...
    ```

    Или укажите его в конфигурации:

    ```json5
    {
      plugins: {
        entries: {
          xai: {
            config: {
              webSearch: {
                apiKey: "xai-...",
              },
            },
          },
        },
      },
    }
    ```

    Любой из этих трёх вариантов также обеспечивает работу `x_search` и
    `web_search` в Grok.

  </Step>

  <Step title="Включите и настройте code_execution">
    Если параметр `enabled` отсутствует, `code_execution` доступен только тогда,
    когда провайдер активной модели — `xai` и учётные данные xAI успешно
    определены. Для активной модели с известным провайдером, отличным от xAI,
    задайте для `plugins.entries.xai.config.codeExecution.enabled` значение
    `true`, чтобы разрешить использование с другим провайдером. Если провайдер
    активной модели отсутствует или не определён, инструмент остаётся скрытым.
    Задайте для `enabled` значение `false`, чтобы отключить его для всех
    провайдеров. Учётные данные xAI требуются всегда.

    В этом же блоке можно переопределить модель, ограничение количества
    итераций или тайм-аут:

    ```json5
    {
      plugins: {
        entries: {
          xai: {
            config: {
              codeExecution: {
                enabled: true, // требуется для известного провайдера модели, отличного от xAI
                model: "grok-4.3", // переопределяет модель xAI для выполнения кода по умолчанию
                maxTurns: 2,            // необязательное ограничение внутренних итераций инструмента
                timeoutSeconds: 30,     // тайм-аут запроса (по умолчанию: 30)
              },
            },
          },
        },
      },
    }
    ```

  </Step>

  <Step title="Перезапустите Gateway">
    ```bash
    openclaw gateway restart
    ```

    `code_execution` появится в списке инструментов агента после повторной
    регистрации плагина xAI и успешного прохождения описанных выше проверок
    провайдера, включения и аутентификации.

  </Step>
</Steps>

## Использование

Явно укажите цель анализа. Инструмент принимает единственный параметр `task`,
поэтому передавайте полный запрос и все встроенные данные в одной подсказке:

```text
Используй code_execution, чтобы вычислить 7-дневное скользящее среднее для этих чисел: ...
```

```text
Используй x_search, чтобы найти публикации с упоминанием OpenClaw за эту неделю, а затем используй code_execution, чтобы подсчитать их по дням.
```

```text
Используй web_search, чтобы собрать последние результаты тестов производительности ИИ, а затем используй code_execution для сравнения процентных изменений.
```

## Ошибки

При отсутствии аутентификации инструмент возвращает структурированную ошибку
JSON (а не выбрасывает исключение), поэтому агент может самостоятельно
исправить проблему:

```json
{
  "error": "missing_xai_api_key",
  "message": "Для code_execution требуются учётные данные xAI. Выполните `openclaw onboard --auth-choice xai-oauth`, чтобы войти через Grok, выполните `openclaw onboard --auth-choice xai-api-key`, задайте `XAI_API_KEY` в окружении Gateway или настройте `plugins.entries.xai.config.webSearch.apiKey`.",
  "docs": "https://docs.openclaw.ai/tools/code-execution"
}
```
