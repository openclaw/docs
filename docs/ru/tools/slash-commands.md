Gateway обрабатывает команды, отправленные как отдельные сообщения, начинающиеся с `/`.
Команды bash, выполняемые только на хосте, используют `! <cmd>` (с `/bash <cmd>` в качестве псевдонима).

Когда беседа привязана к сеансу ACP, обычный текст направляется в исполнительную среду ACP.
Команды управления Gateway остаются локальными: `/acp ...` всегда поступает
обработчику команд OpenClaw, а `/status` и `/unfocus` остаются локальными, когда
для данной поверхности включена обработка команд.

## Три типа команд

<CardGroup cols={3}>
  <Card title="Команды" icon="terminal">
    Отдельные сообщения `/...`, обрабатываемые Gateway. Они должны быть
    единственным содержимым сообщения.
  </Card>
  <Card title="Директивы" icon="sliders">
    `/think`, `/fast`, `/verbose`, `/trace`, `/reasoning`, `/elevated`,
    `/exec`, `/model`, `/queue` — удаляются из сообщения до того, как его
    увидит модель. При отправке отдельно сохраняют настройки сеанса; при отправке
    с другим текстом действуют как встроенные подсказки.
  </Card>
  <Card title="Встроенные сокращения" icon="bolt">
    `/help`, `/commands`, `/status`, `/whoami` — выполняются немедленно и
    удаляются до того, как модель увидит оставшийся текст. Только для авторизованных отправителей.
  </Card>
</CardGroup>

<AccordionGroup>
  <Accordion title="Подробности поведения директив">
    - Директивы удаляются из сообщения до того, как его увидит модель.
    - В сообщениях, содержащих **только директивы** (в сообщении нет ничего, кроме директив), они
      сохраняются в сеансе, а в ответ отправляется подтверждение.
    - В сообщениях **обычного чата** с другим текстом они действуют как встроенные подсказки и
      **не** сохраняют настройки сеанса.
    - Директивы применяются только для **авторизованных отправителей**. Если задан `commands.allowFrom`,
      используется исключительно этот список разрешений; в противном случае авторизация определяется
      списками разрешений канала или сопряжением, а также `commands.useAccessGroups`. Для неавторизованных
      отправителей директивы считаются обычным текстом.
  </Accordion>
</AccordionGroup>

## Конфигурация

```json5
{
  commands: {
    native: "auto",
    nativeSkills: "auto",
    text: true,
    bash: false,
    bashForegroundMs: 2000,
    config: false,
    mcp: false,
    plugins: false,
    debug: false,
    restart: true,
    ownerAllowFrom: ["discord:123456789012345678"],
    ownerDisplay: "raw",
    ownerDisplaySecret: "${OWNER_ID_HASH_SECRET}",
    allowFrom: {
      "*": ["user1"],
      discord: ["user:123"],
    },
    useAccessGroups: true,
  },
}
```

<ParamField path="commands.text" type="boolean" default="true">
  Включает разбор `/...` в сообщениях чата. На поверхностях без нативных команд
  (WhatsApp, WebChat, Signal, iMessage, Google Chat, Microsoft Teams) текстовые
  команды работают, даже если задано `false`.
</ParamField>

<ParamField path="commands.native" type='boolean | "auto"' default='"auto"'>
  Регистрирует нативные команды. Автоматический режим: включено для Discord/Telegram; выключено для Slack;
  игнорируется для провайдеров без нативной поддержки. Переопределяется для каждого канала с помощью
  `channels.<provider>.commands.native`. В Discord `false` пропускает регистрацию
  слеш-команд; ранее зарегистрированные команды могут оставаться видимыми до удаления.
</ParamField>

<ParamField path="commands.nativeSkills" type='boolean | "auto"' default='"auto"'>
  Регистрирует команды навыков как нативные, если это поддерживается. Автоматический режим: включено для
  Discord/Telegram; выключено для Slack. Переопределяется с помощью
  `channels.<provider>.commands.nativeSkills`.
</ParamField>

<ParamField path="commands.bash" type="boolean" default="false">
  Разрешает `! <cmd>` выполнять команды оболочки хоста (псевдоним `/bash <cmd>`). Требует
  списков разрешений `tools.elevated`.
</ParamField>

<ParamField path="commands.bashForegroundMs" type="number" default="2000">
  Время ожидания bash перед переходом в фоновый режим (`0` немедленно
  переводит выполнение в фоновый режим).
</ParamField>

<ParamField path="commands.config" type="boolean" default="false">
  Включает `/config` (чтение и запись `openclaw.json`). Только для владельца.
</ParamField>

<ParamField path="commands.mcp" type="boolean" default="false">
  Включает `/mcp` (чтение и запись управляемой OpenClaw конфигурации MCP в `mcp.servers`). Только для владельца.
</ParamField>

<ParamField path="commands.plugins" type="boolean" default="false">
  Включает `/plugins` (обнаружение и состояние плагинов, а также установка, включение и отключение). Запись доступна только владельцу.
</ParamField>

<ParamField path="commands.debug" type="boolean" default="false">
  Включает `/debug` (переопределения конфигурации только на время выполнения). Только для владельца.
</ParamField>

<ParamField path="commands.restart" type="boolean" default="true">
  Включает `/restart` и внешние запросы на перезапуск `SIGUSR1`.
</ParamField>

<ParamField path="commands.ownerAllowFrom" type="string[]">
  Явный список разрешений владельца для поверхностей команд, доступных только владельцу. Не зависит от
  `commands.allowFrom` и доступа через сопряжение в личных сообщениях.
</ParamField>

<ParamField path="channels.<channel>.commands.enforceOwnerForCommands" type="boolean" default="false">
  Для каждого канала: требует идентификации владельца для команд, доступных только владельцу. Когда `true`,
  отправитель должен соответствовать `commands.ownerAllowFrom` или иметь внутреннюю область доступа `operator.admin`.
  Записи с подстановочным знаком `allowFrom` **недостаточно**.
</ParamField>

<ParamField path="commands.ownerDisplay" type='"raw" | "hash"'>
  Определяет, как идентификаторы владельца отображаются в системном промпте.
</ParamField>

<ParamField path="commands.ownerDisplaySecret" type="string">
  Секрет HMAC, используемый при `commands.ownerDisplay: "hash"`.
</ParamField>

<ParamField path="commands.allowFrom" type="object">
  Список разрешений для авторизации команд, задаваемый отдельно для каждого провайдера. Если он настроен, то становится
  **единственным** источником авторизации команд и директив. Используйте `"*"` как
  глобальное значение по умолчанию; ключи конкретных провайдеров переопределяют его.
</ParamField>

<ParamField path="commands.useAccessGroups" type="boolean" default="true">
  Применяет списки разрешений и политики к командам, когда `commands.allowFrom` не задан.
</ParamField>

## Список команд

Команды поступают из трёх источников:

- **Встроенные команды ядра:** `src/auto-reply/commands-registry.shared.ts`
- **Созданные команды дока:** `src/auto-reply/commands-registry.data.ts`
- **Команды плагинов:** вызовы `registerCommand()` плагином

Доступность зависит от флагов конфигурации, поверхности канала и установленных и включённых
плагинов.

### Команды закрепления

Команды закрепления переключают маршрут ответов активного сеанса на другой связанный канал.
Сведения о настройке и устранении неполадок см. в разделе [Закрепление каналов](/ru/concepts/channel-docking).

Создаются плагинами каналов с поддержкой собственных команд:

- `/dock-discord` (псевдоним: `/dock_discord`)
- `/dock-mattermost` (псевдоним: `/dock_mattermost`)
- `/dock-slack` (псевдоним: `/dock_slack`)
- `/dock-telegram` (псевдоним: `/dock_telegram`)

Для команд закрепления требуется `session.identityLinks`. Исходный отправитель и целевой собеседник
должны входить в одну группу идентификации.

| `/voice status\|list\|set <voiceId>`                    | Управлять голосовой конфигурацией Talk. Собственное имя в Discord: `/talkvoice`                                                                                                                                    |
| `/card ...`                                             | Отправлять предустановленные интерактивные карточки LINE. См. [LINE](/ru/channels/line)                                                                                                                                        |
| `/codex <action> ...`                                   | Привязывать, направлять и проверять оснастку сервера приложений Codex (состояние, ветки, возобновление, модель, быстрый режим, разрешения, сжатие, проверка, MCP, навыки и другое). См. [Оснастка Codex](/ru/plugins/codex-harness) |

Только для QQBot: `/bot-ping`, `/bot-version`, `/bot-help`, `/bot-upgrade`, `/bot-logs`

### Команды навыков

Доступные пользователю навыки предоставляются в виде команд с косой чертой:

- `/skill <name> [input]` всегда работает как универсальная точка входа.
- Навыки могут регистрироваться как непосредственные команды (например, `/prose` для OpenProse).
- Регистрация собственных команд навыков управляется параметрами `commands.nativeSkills` и
  `channels.<provider>.commands.nativeSkills`.
- Имена очищаются до формата `a-z0-9_` (не более 32 символов); при конфликтах добавляются числовые суффиксы.

<AccordionGroup>
  <Accordion title="Диспетчеризация команд навыков">
    По умолчанию команды навыков направляются модели как обычные запросы.

    Навыки могут объявить `command-dispatch: tool`, чтобы направлять запрос непосредственно инструменту
    (детерминированно, без участия модели). Пример: `/prose` (плагин OpenProse)
    — см. [OpenProse](/ru/prose).

  </Accordion>
  <Accordion title="Аргументы собственных команд">
    Discord использует автодополнение для динамических параметров и меню кнопок, когда обязательные
    аргументы не указаны. Telegram и Slack отображают меню кнопок для команд с
    вариантами выбора. Динамические варианты определяются на основе модели целевого сеанса, поэтому зависящие от
    модели параметры, например уровни `/think`, учитывают переопределение `/model` сеанса.
  </Accordion>
</AccordionGroup>

## `/tools`: что агент может использовать сейчас

`/tools` отвечает на вопрос о среде выполнения: **что этот агент может использовать прямо сейчас в этом
разговоре**, а не отображает статический каталог конфигурации.

```text
/tools         # компактное представление
/tools verbose # с краткими описаниями
```

Результаты относятся к конкретному сеансу. Смена агента, канала, ветки, авторизации
отправителя или модели может изменить вывод. Для редактирования профиля и переопределений
используйте панель Tools в Control UI или интерфейсы конфигурации.

## `/model`: выбор модели

```text
/model             # показать средство выбора модели
/model list        # то же самое
/model 3           # выбрать по номеру из списка
/model openai/gpt-5.4
/model opus@anthropic:default
/model default     # сбросить выбор модели для сеанса
/model status      # подробное представление с конечной точкой и режимом API
```

В Discord команды `/model` и `/models` открывают интерактивное средство выбора с раскрывающимися списками
провайдеров и моделей. Средство выбора учитывает `agents.defaults.models`, включая
записи `provider/*`.

## `/config`: запись конфигурации на диск

<Note>
  Только для владельца. По умолчанию отключено — включите с помощью `commands.config: true`.
</Note>

```text
/config show
/config show messages.responsePrefix
/config get messages.responsePrefix
/config set messages.responsePrefix="[openclaw]"
/config unset messages.responsePrefix
```

Перед записью конфигурация проверяется. Недопустимые изменения отклоняются. Обновления `/config`
сохраняются после перезапуска.

## `/mcp`: конфигурация сервера MCP

<Note>
  Только для владельца. По умолчанию отключено — включите с помощью `commands.mcp: true`.
</Note>

```text
/mcp show
/mcp show context7
/mcp set context7={"command":"uvx","args":["context7-mcp"]}
/mcp unset context7
```

`/mcp` сохраняет конфигурацию в конфигурации OpenClaw, а не в настройках проекта встроенного агента.
`/mcp show` скрывает поля с учетными данными, распознанные значения флагов учетных данных
и известные аргументы, похожие на секреты. При запуске из группы
конфигурация отправляется владельцу в личном сообщении; если приватный маршрут к владельцу
недоступен, команда безопасно завершается с отказом и предлагает владельцу повторить попытку в личном
чате.

## `/debug`: переопределения только для среды выполнения

<Note>
  Только для владельца. По умолчанию отключено — включите с помощью `commands.debug: true`.
  Переопределения немедленно применяются при новых чтениях конфигурации, но **не** записываются на диск.
</Note>

```text
/debug show
/debug set messages.responsePrefix="[openclaw]"
/debug set channels.whatsapp.allowFrom=["+1555","+4477"]
/debug unset messages.responsePrefix
/debug reset
```

## `/plugins`: управление плагинами

<Note>
  Запись доступна только владельцу. По умолчанию отключено — включите с помощью `commands.plugins: true`.
</Note>

```text
/plugins
/plugins list
/plugin show context7
/plugins enable context7
/plugins disable context7
/plugins install clawhub:<package>
/plugins install npm:@openclaw/<official-package>
/plugins install npm:<package> --force
/plugins install git:<repository>@<ref> --force
```

`/plugins enable|disable` обновляет конфигурацию плагина и выполняет горячую перезагрузку
среды выполнения плагинов Gateway для новых обращений агента. `/plugins install` автоматически перезапускает управляемые
Gateway, поскольку исходные модули плагина изменились. Для установки из доверенного ClawHub
и официального каталога дополнительное подтверждение не требуется. Для произвольных источников npm,
git, архивов, `npm-pack:` и локальных путей отображается предупреждение о происхождении, и
после проверки исходного кода требуется добавить в конце `--force`. Этот флаг подтверждает
источник и разрешает замену существующей установки; он не позволяет обойти
`security.installPolicy` или проверки безопасности установщика. Для выпусков ClawHub с
предупреждениями о рисках по-прежнему требуется отдельный доступный только в оболочке
флаг `--acknowledge-clawhub-risk`. Установки из магазина, по ссылке и с закрепленной версией также
по-прежнему доступны только из оболочки.

## `/trace`: вывод трассировки плагинов

```text
/trace          # показать текущее состояние трассировки
/trace on
/trace off
```

`/trace` отображает относящиеся к сеансу строки трассировки и отладки плагинов без полного подробного
режима. Эта команда не заменяет `/debug` (переопределения среды выполнения) или `/verbose` (обычный
вывод инструментов).

## `/btw`: побочные вопросы

`/btw` — быстрый побочный вопрос о контексте текущего сеанса. Псевдоним: `/side`.

```text
/btw что мы сейчас делаем?
/side что изменилось, пока продолжался основной запуск?
```

В отличие от обычного сообщения:

- Использует текущий сеанс как фоновый контекст.
- В сеансах оснастки Codex выполняется как временная побочная ветка Codex.
- **Не** изменяет дальнейший контекст сеанса.
- Не записывается в историю расшифровки.

Полное описание поведения см. в разделе [Побочные вопросы BTW](/ru/tools/btw).

## Примечания об интерфейсах

<AccordionGroup>
  <Accordion title="Область действия сеанса для каждого интерфейса">
    - **Текстовые команды:** выполняются в обычном сеансе чата (личные сообщения используют общий `main`, у групп отдельные сеансы).
    - **Собственные команды Discord:** `agent:<agentId>:discord:slash:<userId>`
    - **Собственные команды Slack:** `agent:<agentId>:slack:slash:<userId>` (префикс настраивается через `channels.slack.slashCommand.sessionPrefix`)
    - **Собственные команды Telegram:** `telegram:slash:<userId>` (нацелены на сеанс чата через `CommandTargetSessionKey`)
    - **`/login codex`** отправляет коды сопряжения устройств только через личный чат или пути ответа Web UI. При вызове в группе или теме Telegram владельцу предлагается вместо этого написать боту личное сообщение.
    - **`/stop`** нацелена на активный сеанс чата, чтобы прервать текущий запуск.

  </Accordion>
  <Accordion title="Особенности Slack">
    `channels.slack.slashCommand` поддерживает одну команду в стиле `/openclaw`.
    При использовании `commands.native: true` создайте отдельную slash-команду Slack для каждой встроенной
    команды. Зарегистрируйте `/agentstatus` (не `/status`), поскольку Slack резервирует
    `/status`. Текст `/status` по-прежнему работает в сообщениях Slack.
  </Accordion>
  <Accordion title="Быстрый путь и встроенные сокращения">
    - Сообщения, содержащие только команды, от отправителей из списка разрешённых обрабатываются немедленно (в обход очереди и модели).
    - Встроенные сокращения (`/help`, `/commands`, `/status`, `/whoami`) также работают внутри обычных сообщений и удаляются до того, как модель увидит оставшийся текст.
    - Неавторизованные сообщения, содержащие только команды, молча игнорируются; встроенные токены `/...` обрабатываются как обычный текст.

  </Accordion>
  <Accordion title="Примечания об аргументах">
    - Команды принимают необязательный `:` между командой и аргументами (`/think: high`, `/send: on`).
    - `/new <model>` принимает псевдоним модели, `provider/model` или имя провайдера (с нечётким сопоставлением); если совпадение не найдено, текст обрабатывается как тело сообщения.
    - `/allowlist add|remove` требует `commands.config: true` и учитывает настройку канала `configWrites`.

  </Accordion>
</AccordionGroup>

## Использование провайдера и состояние

- **Использование/квота провайдера** (например, «Claude: осталось 80%») отображается в `/status` для провайдера текущей модели, когда включено отслеживание использования.
- **Строки токенов/кэша** в `/status` могут использовать последнюю запись об использовании из транскрипта, если снимок активного сеанса содержит мало данных.
- **Среда выполнения и исполнитель:** `/status` сообщает `Execution` для фактического пути песочницы и `Runtime` для того, кто выполняет сеанс: `OpenClaw Default`, `OpenAI Codex`, серверная часть CLI или серверная часть ACP.
- **Токены/стоимость для каждого ответа:** управляется параметром `/usage off|tokens|full`.
- `/model status` относится к моделям, аутентификации и конечным точкам, а не к использованию.
