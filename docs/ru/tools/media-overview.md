OpenClaw генерирует изображения, видео и музыку, распознаёт входящие медиафайлы
(изображения, аудио и видео) и озвучивает ответы с помощью преобразования текста в речь. Все
мультимедийные возможности реализованы через инструменты: агент решает, когда использовать их,
исходя из хода беседы, а каждый инструмент появляется только в том случае, если настроен хотя бы
один поддерживающий его провайдер.

Для живой речи используется контракт сеанса Talk, а не путь одноразового мультимедийного
инструмента. Talk поддерживает три режима: нативный для провайдера `realtime`, локальный или потоковый
`stt-tts` и `transcription` для захвата речи только в режиме наблюдения. Эти режимы
используют общие каталоги провайдеров, конверты событий и семантику отмены с
телефонией, совещаниями, браузерным взаимодействием в реальном времени и нативными клиентами с функцией «нажми и говори».

## Асинхронный и синхронный режимы

| Возможность                 | Режим         | Причина                                                                                                                     |
| --------------------------- | ------------- | --------------------------------------------------------------------------------------------------------------------------- |
| Изображения                 | Асинхронный   | Обработка провайдером может продолжаться дольше одного хода чата; сгенерированные вложения используют общий путь завершения. |
| Преобразование текста в речь | Синхронный    | Ответы провайдера возвращаются за несколько секунд и прикрепляются к аудиоответу.                                            |
| Видео                       | Асинхронный   | Обработка провайдером занимает от 30 с до нескольких минут; медленные очереди могут работать до настроенного тайм-аута.      |
| Музыка                      | Асинхронный   | Характеристики обработки провайдером такие же, как для видео.                                                                |

Для асинхронных инструментов OpenClaw отправляет запрос провайдеру, немедленно возвращает
идентификатор задачи и отслеживает задание в журнале задач. Пока задание выполняется, агент продолжает
отвечать на другие сообщения. Когда провайдер завершает работу,
OpenClaw активирует агент, передавая ему пути к сгенерированным медиа, чтобы он мог сообщить об этом
пользователю через обычный для сеанса режим видимого ответа: автоматическую отправку итогового ответа,
если она настроена, или `message(action="send")`, если сеанс требует
использования инструмента сообщений. Если сеанс инициатора неактивен или его активное пробуждение
завершается неудачей, а часть сгенерированных медиа всё ещё отсутствует в ответе о завершении,
OpenClaw отправляет идемпотентный прямой резервный ответ, содержащий только отсутствующие медиа. Медиа,
уже доставленные в ответе о завершении, повторно не публикуются.

## Преобразование речи в текст и Voice Call

Deepgram, DeepInfra, ElevenLabs, Google, Groq, Mistral, OpenAI, OpenRouter,
SenseAudio и xAI могут расшифровывать входящее аудио через пакетный путь
`tools.media.audio`, если они настроены. Плагины каналов, которые предварительно обрабатывают
голосовое сообщение для фильтрации упоминаний или разбора команд, помечают расшифрованное
вложение во входящем контексте, чтобы общий этап распознавания медиа
повторно использовал эту расшифровку вместо второго вызова STT для того же
аудио.

Deepgram, ElevenLabs, Mistral, OpenAI и xAI также регистрируют провайдеров
потокового STT для Voice Call, поэтому живое аудио телефонного звонка можно передавать выбранному
поставщику, не дожидаясь завершения записи.

Для живых бесед с пользователем предпочтителен [режим Talk](/ru/nodes/talk). Пакетные аудиовложения
остаются в пути обработки медиа; браузерное взаимодействие в реальном времени, нативные клиенты с функцией «нажми и говори»,
телефония и аудио совещаний должны использовать события Talk и каталоги в области сеанса,
возвращаемые Gateway.
