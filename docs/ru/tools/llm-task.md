`llm-task` — это встроенный **необязательный инструмент плагина**, который выполняет один вызов LLM с ответом только в формате JSON и возвращает структурированный результат, при необходимости проверенный по JSON Schema. Он позволяет движкам рабочих процессов, таким как Lobster, использовать шаг LLM без написания специального кода OpenClaw для каждого рабочего процесса.

## Включение

1. Включите плагин:

```json
{
  "plugins": {
    "entries": {
      "llm-task": { "enabled": true }
    }
  }
}
```

2. Разрешите инструмент:

```json
{
  "tools": {
    "alsoAllow": ["llm-task"]
  }
}
```

`alsoAllow` добавляет `llm-task` к активному профилю инструментов, не ограничивая другие основные инструменты. Используйте `tools.allow` только в том случае, если вместо этого вам нужен режим с ограничительным списком разрешений.

## Конфигурация (необязательно)

```json
{
  "plugins": {
    "entries": {
      "llm-task": {
        "enabled": true,
        "config": {
          "defaultProvider": "openai",
          "defaultModel": "gpt-5.6-sol",
          "defaultAuthProfileId": "main",
          "allowedModels": ["openai/gpt-5.6-sol"],
          "maxTokens": 800,
          "timeoutMs": 30000
        }
      }
    }
  }
}
```

`allowedModels` — это список разрешённых строк вида `provider/model`; запрос любой другой модели отклоняется. Все остальные ключи задают резервные значения для отдельных вызовов и используются, когда соответствующий параметр отсутствует в вызове инструмента.

## Параметры инструмента

| Параметр        | Тип    | Примечания                                                                                                                                                                                        |
| --------------- | ------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `prompt`        | string | Обязательный. Инструкция для LLM.                                                                                                                                                                  |
| `input`         | any    | Необязательные входные данные; сериализуются в JSON и добавляются к запросу.                                                                                                                       |
| `schema`        | object | Необязательная JSON Schema, которой должен соответствовать разобранный результат.                                                                                                                 |
| `provider`      | string | Переопределяет `defaultProvider` / провайдера агента по умолчанию.                                                                                                                                 |
| `model`         | string | Переопределяет `defaultModel`; принимает идентификаторы моделей без префикса, псевдонимы или ссылку `provider/model` (повторяющийся префикс провайдера удаляется автоматически).                    |
| `thinking`      | string | Уровень рассуждений (например, `low`, `medium`); должен поддерживаться выбранной моделью.                                                                                                          |
| `authProfileId` | string | Переопределяет `defaultAuthProfileId`.                                                                                                                                                             |
| `temperature`   | number | Применяется по возможности; поддерживается не всеми провайдерами.                                                                                                                                 |
| `maxTokens`     | number | Ограничение количества токенов в ответе, применяемое по возможности.                                                                                                                              |
| `timeoutMs`     | number | Тайм-аут выполнения; значение по умолчанию — `30000`.                                                                                                                                             |

## Вывод

Возвращает `details.json` (разобранный и проверенный по схеме JSON), а также `details.provider` и `details.model`, указывающие фактически использованные провайдер и модель.

## Пример: шаг рабочего процесса Lobster

### Важное ограничение

В приведённом ниже примере предполагается, что **автономный Lobster CLI** работает в среде, где `openclaw.invoke` уже имеет правильный URL Gateway и контекст аутентификации.

Для встроенного в OpenClaw **интегрированного** средства запуска Lobster этот шаблон с вложенным вызовом CLI **пока работает ненадёжно**:

```lobster
openclaw.invoke --tool llm-task --action json --args-json '{ ... }'
```

Пока в интегрированном Lobster не появится поддерживаемый мост для этого сценария, рекомендуется использовать:

- прямые вызовы инструмента `llm-task` вне Lobster или
- шаги Lobster, не зависящие от вложенных вызовов `openclaw.invoke`.

Пример для автономного Lobster CLI:

```lobster
openclaw.invoke --tool llm-task --action json --args-json '{
  "prompt": "Given the input email, return intent and draft.",
  "thinking": "low",
  "input": {
    "subject": "Hello",
    "body": "Can you help?"
  },
  "schema": {
    "type": "object",
    "properties": {
      "intent": { "type": "string" },
      "draft": { "type": "string" }
    },
    "required": ["intent", "draft"],
    "additionalProperties": false
  }
}'
```

## Примечания по безопасности

- **Только JSON**: модели предписывается возвращать только значение JSON, без блоков кода и комментариев.
- **Без инструментов**: инструменты отключены для базового выполнения, поэтому модель не может вызывать их во время выполнения задачи.
- Считайте результат недоверенным, если не проверяете его с помощью `schema`.
- Добавляйте подтверждения перед любым шагом с побочными эффектами (отправкой, публикацией, выполнением), использующим этот результат.
