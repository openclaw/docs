Сбор траектории — это бортовой регистратор OpenClaw для каждого сеанса. Он записывает
структурированную временную шкалу каждого запуска агента, после чего `/export-trajectory` упаковывает
текущий сеанс в отредактированный пакет для службы поддержки, включающий:

- Промпт, системный промпт и инструменты, отправленные модели
- Какие сообщения транскрипта и вызовы инструментов привели к ответу
- Завершился ли запуск по тайм-ауту, был ли прерван, подвергнут Compaction или столкнулся с ошибкой провайдера
- Какие модель, плагины, Skills и настройки среды выполнения были активны
- Метаданные использования и кэша промпта, возвращённые провайдером

Для общего отчёта службы поддержки по Gateway сначала используйте
[`/diagnostics`](/ru/gateway/diagnostics#chat-command); эта команда собирает
санитизированный пакет Gateway, а для сеансов среды OpenAI Codex может после
подтверждения отправить отзыв о Codex в OpenAI. Используйте `/export-trajectory`, когда вам нужна
подробная временная шкала промптов, инструментов и транскрипта для конкретного сеанса.

## Быстрый старт

Отправьте в активном сеансе (псевдоним `/trajectory`):

```text
/export-trajectory
```

OpenClaw записывает пакет в рабочую область:

```text
.openclaw/trajectory-exports/openclaw-trajectory-<session>-<timestamp>/
```

Чтобы переопределить его, передайте относительное имя выходного каталога:

```text
/export-trajectory bug-1234
```

Имя разрешается внутри `.openclaw/trajectory-exports/`. Абсолютные пути и
пути `~` отклоняются.

Пакеты траекторий могут содержать промпты, сообщения модели, схемы инструментов, результаты
инструментов, события среды выполнения и локальные пути, поэтому команда чата всегда выполняется
с подтверждением exec. Подтвердите экспорт один раз, когда действительно хотите создать
пакет; не используйте разрешение для всех операций. В групповых чатах OpenClaw отправляет запрос
на подтверждение и результат экспорта владельцу в личном сообщении, а не публикует сведения о траектории
в общей комнате.

Для локальной проверки или рабочих процессов службы поддержки выполните базовую команду CLI
напрямую:

```bash
openclaw sessions export-trajectory --session-key "agent:main:telegram:direct:123" --workspace .
```

Другие флаги: `--output <path>` (имя каталога внутри
`.openclaw/trajectory-exports`), `--store <path>` (переопределение хранилища сеансов),
`--agent <id>` (идентификатор агента для разрешения хранилища), `--json` (структурированный вывод).

## Доступ

Экспорт траектории — команда владельца. Отправитель должен пройти стандартные проверки
авторизации команд, а также проверку владельца для канала.

## Что записывается

Сбор траектории по умолчанию включён для запусков агентов OpenClaw.

События среды выполнения включают:

- `session.started`
- `trace.metadata`
- `context.compiled`
- `prompt.submitted`
- `model.fallback_step`, включая исходную модель, следующую модель, причину и сведения о сбое, позицию в цепочке и то, перешла ли цепочка дальше, завершилась успешно или была исчерпана
- `model.completed`
- `trace.artifacts`
- `session.ended`

События транскрипта восстанавливаются из активной ветви сеанса: сообщения
пользователя, сообщения ассистента, вызовы инструментов, результаты инструментов, операции
Compaction, смены моделей, метки и пользовательские записи сеанса.

События записываются в формате JSON Lines с таким маркером схемы:

```json
{
  "traceSchema": "openclaw-trajectory",
  "schemaVersion": 1
}
```

## Файлы пакета

| Файл                  | Содержимое                                                                                       |
| --------------------- | ---------------------------------------------------------------------------------------------- |
| `manifest.json`       | Схема пакета, исходные файлы, количество событий и список созданных файлов                             |
| `events.jsonl`        | Упорядоченная временная шкала среды выполнения и транскрипта                                                        |
| `session-branch.json` | Отредактированная активная ветвь транскрипта и заголовок сеанса                                           |
| `metadata.json`       | Версия OpenClaw, ОС и среда выполнения, модель, снимок конфигурации, плагины, Skills и метаданные промпта     |
| `artifacts.json`      | Итоговый статус, ошибки, использование, кэш промпта, количество операций Compaction, текст ассистента и метаданные инструментов |
| `prompts.json`        | Отправленные промпты и выбранные сведения о формировании промптов                                         |
| `system-prompt.txt`   | Последний скомпилированный системный промпт, если он был записан                                                   |
| `tools.json`          | Определения инструментов, отправленные модели, если они были записаны                                              |

`manifest.json` перечисляет файлы, присутствующие в конкретном пакете; некоторые файлы
не включаются, если в сеансе не были записаны соответствующие данные среды выполнения.

## Хранилище записей

События траектории среды выполнения хранятся вместе с сеансом в базе данных SQLite
соответствующего агента. При экспорте траектории создаётся отредактированный пакет поддержки JSONL;
активная запись среды выполнения не является дополнительным JSONL-файлом рядом с сеансом.

Устаревшие файлы `.trajectory.jsonl` и `.trajectory-path.json` могут по-прежнему появляться
из-за старых выпусков или явного экспорта в устаревшие файлы. Средства обслуживания сеансов считают
такие файлы целями для очистки; активный сбор записывает строки в базу данных.

## Отключение сбора

```bash
export OPENCLAW_TRAJECTORY=0
```

Это отключает сбор траектории среды выполнения перед запуском OpenClaw.
`/export-trajectory` всё ещё может экспортировать ветвь транскрипта, но данные, доступные только
среде выполнения, например скомпилированный контекст, артефакты провайдера и метаданные промпта, могут
отсутствовать.

## Настройка тайм-аута сброса

OpenClaw сбрасывает строки траектории среды выполнения при очистке после работы агента. Тайм-аут
очистки по умолчанию составляет 10,000 мс. При медленных дисках или больших хранилищах задайте
`OPENCLAW_TRAJECTORY_FLUSH_TIMEOUT_MS` перед запуском OpenClaw:

```bash
export OPENCLAW_TRAJECTORY_FLUSH_TIMEOUT_MS=30000
```

Этот параметр определяет, когда OpenClaw регистрирует тайм-аут `openclaw-trajectory-flush` и
продолжает работу; он не изменяет ограничения размера траектории. Чтобы настроить все этапы
очистки агента, для которых явно не задан тайм-аут, укажите
`OPENCLAW_AGENT_CLEANUP_TIMEOUT_MS`.

## Конфиденциальность и ограничения

Пакеты траекторий предназначены для поддержки и отладки, а не для публичного размещения. OpenClaw
редактирует конфиденциальные значения перед записью файлов экспорта:

- учётные данные и известные поля полезной нагрузки, похожие на секреты
- данные изображений
- пути локального состояния
- пути рабочей области, заменённые на `$WORKSPACE_DIR`
- пути к домашнему каталогу, если они обнаружены

Экспортёр также ограничивает размер входных данных:

- сбор среды выполнения: активная запись представляет собой скользящее окно размером не более 10 MiB; самые старые события удаляются, чтобы освободить место для новых; при экспорте принимаются существующие устаревшие дополнительные файлы среды выполнения размером до 50 MiB
- файлы сеансов: 50 MiB
- события среды выполнения на один экспорт: 200,000
- общее количество экспортируемых событий: 250,000
- отдельные строки событий среды выполнения обрезаются при превышении 256 KiB

Проверяйте пакеты перед передачей за пределы своей команды. Редактирование выполняется по мере
возможности и не может распознать все секреты, специфичные для конкретного приложения.

## Устранение неполадок

Если экспорт не содержит событий среды выполнения:

- убедитесь, что OpenClaw был запущен без `OPENCLAW_TRAJECTORY=0`
- отправьте ещё одно сообщение в сеансе, затем повторите экспорт
- проверьте `manifest.json` на наличие `runtimeEventCount`

Если команда отклоняет выходной путь:

- используйте относительное имя, например `bug-1234`
- не передавайте `/tmp/...` или `~/...`
- сохраняйте экспорт внутри `.openclaw/trajectory-exports/`

Если экспорт завершается ошибкой размера, значит сеанс или дополнительный файл превысил
указанные выше ограничения безопасности экспорта. Начните новый сеанс или экспортируйте более компактный
пример воспроизведения.
