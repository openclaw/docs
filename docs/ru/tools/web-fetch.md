`web_fetch` выполняет обычный HTTP GET-запрос и извлекает содержимое, удобное для чтения (преобразуя HTML в
Markdown или текст). Он **не** выполняет JavaScript. Для сайтов, активно использующих JS, или
страниц, защищённых авторизацией, используйте вместо него [веб-браузер](/ru/tools/browser).

## Быстрый старт

Включён по умолчанию, настройка не требуется:

```javascript
await web_fetch({ url: "https://example.com/article" });
```

## Параметры инструмента

<ParamField path="url" type="string" required>
URL для получения данных. Поддерживается только `http(s)`.
</ParamField>

<ParamField path="extractMode" type="'markdown' | 'text'" default="markdown">
Формат вывода после извлечения основного содержимого.
</ParamField>

<ParamField path="maxChars" type="number">
Обрезать вывод до указанного количества символов. Значение ограничивается параметром `tools.web.fetch.maxCharsCap`.
</ParamField>

## Принцип работы

<Steps>
  <Step title="Получение">
    Отправляет HTTP GET-запрос с похожим на Chrome значением User-Agent и заголовком
    `Accept-Language`. Блокирует частные и внутренние имена хостов и повторно проверяет перенаправления.
  </Step>
  <Step title="Извлечение">
    Запускает Readability (извлечение основного содержимого) для HTML-ответа.
  </Step>
  <Step title="Резервный вариант (необязательно)">
    Если Readability завершается с ошибкой и доступен поставщик получения данных, повторяет попытку
    через него (например, в режиме обхода защиты от ботов Firecrawl).
  </Step>
  <Step title="Кэширование">
    Результаты кэшируются на 15 минут (значение можно изменить), чтобы сократить количество повторных
    запросов к одному URL.
  </Step>
</Steps>

## Обновления хода выполнения

`web_fetch` выводит общедоступную строку хода выполнения, только если получение данных всё ещё продолжается
через пять секунд:

```text
Получение содержимого страницы...
```

Быстрые попадания в кэш и быстрые сетевые ответы завершаются до срабатывания таймера, поэтому
строка хода выполнения для них не отображается. При отмене вызова таймер сбрасывается. Эта
строка отражает только состояние интерфейса канала и никогда не содержит полученное содержимое страницы.

## Конфигурация

```json5
{
  tools: {
    web: {
      fetch: {
        enabled: true, // по умолчанию: true
        provider: "firecrawl", // необязательно; не указывайте для автоматического определения
        maxChars: 20000, // количество символов в выводе по умолчанию; ограничено maxCharsCap
        maxCharsCap: 20000, // жёсткое ограничение параметра maxChars
        maxResponseBytes: 750000, // максимальный размер загрузки до обрезки (32000-10000000)
        timeoutSeconds: 30,
        cacheTtlMinutes: 15,
        maxRedirects: 3,
        useTrustedEnvProxy: false, // разрешить доверенному HTTP(S)-прокси из переменной среды выполнять разрешение DNS
        readability: true, // использовать извлечение через Readability
        userAgent: "Mozilla/5.0 ...", // переопределить User-Agent
        ssrfPolicy: {
          allowRfc2544BenchmarkRange: true, // явное разрешение для доверенных прокси с поддельными IP-адресами, использующих 198.18.0.0/15
          allowIpv6UniqueLocalRange: true, // явное разрешение для доверенных прокси с поддельными IP-адресами, использующих fc00::/7
        },
      },
    },
  },
}
```

## Резервный вариант Firecrawl

Если извлечение через Readability завершается с ошибкой, `web_fetch` может использовать
[Firecrawl](/ru/tools/firecrawl) в качестве резервного варианта для обхода защиты от ботов и более качественного извлечения:

```json5
{
  tools: {
    web: {
      fetch: {
        provider: "firecrawl", // необязательно; не указывайте для автоматического определения по доступным учётным данным
      },
    },
  },
  plugins: {
    entries: {
      firecrawl: {
        enabled: true,
        config: {
          webFetch: {
            // apiKey: "fc-...", // необязательно; не указывайте для начального доступа без ключа
            baseUrl: "https://api.firecrawl.dev",
            onlyMainContent: true,
            maxAgeMs: 172800000, // срок хранения в кэше (2 дня)
            timeoutSeconds: 60,
          },
        },
      },
    },
  },
}
```

`plugins.entries.firecrawl.config.webFetch.apiKey` является необязательным и поддерживает объекты SecretRef.
Устаревшая конфигурация `tools.web.fetch.firecrawl.*` автоматически переносится в
`plugins.entries.firecrawl.config.webFetch` командой `openclaw doctor --fix`.

<Note>
  Если настроен SecretRef для API-ключа Firecrawl, но он не разрешается и резервная переменная среды
  `FIRECRAWL_API_KEY` отсутствует, запуск Gateway немедленно завершается с ошибкой.
</Note>

<Note>
  Переопределения `baseUrl` для Firecrawl строго ограничены: размещённый сервис использует
  `https://api.firecrawl.dev`; переопределения для самостоятельно размещённого сервиса должны указывать на частные или
  внутренние конечные точки, а `http://` допускается только для таких частных адресатов.
</Note>

Текущее поведение среды выполнения:

- `tools.web.fetch.provider` явно выбирает резервного поставщика получения данных.
- Если `provider` не указан, OpenClaw автоматически определяет первого готового поставщика
  веб-загрузки по настроенным учётным данным. `web_fetch` вне песочницы может использовать
  установленные плагины, которые объявляют `contracts.webFetchProviders` и регистрируют
  соответствующего поставщика во время выполнения. Сейчас этот резервный вариант предоставляет
  официальный плагин Firecrawl.
- Вызовы `web_fetch` в песочнице разрешают встроенных поставщиков, а также установленных поставщиков
  с подтверждённым официальным происхождением из npm или ClawHub. Сейчас это позволяет использовать
  официальный плагин Firecrawl; сторонние внешние плагины получения данных по-прежнему исключены.
- Если Readability отключён, `web_fetch` сразу переходит к выбранному резервному
  поставщику. Если доступный поставщик отсутствует, вызов завершается с ошибкой без небезопасного обходного пути.

## Доверенный прокси из переменной среды

Если в вашей среде развёртывания `web_fetch` должен работать через доверенный исходящий
HTTP(S)-прокси, задайте `tools.web.fetch.useTrustedEnvProxy: true`.

В этом режиме OpenClaw по-прежнему выполняет проверки имён хостов для защиты от SSRF перед отправкой
запроса, но позволяет прокси разрешать DNS вместо локальной фиксации DNS.
Включайте этот режим только для прокси, контролируемого оператором и применяющего
политику исходящих соединений после разрешения DNS.

<Note>
  Если переменная среды HTTP(S)-прокси не настроена или целевой хост исключён через
  `NO_PROXY`, `web_fetch` возвращается к обычному строгому пути с локальной
  фиксацией DNS.
</Note>

## Ограничения и безопасность

- Значение `maxChars` ограничивается параметром `tools.web.fetch.maxCharsCap` (по умолчанию `20000`)
- Размер тела ответа ограничивается `maxResponseBytes` (по умолчанию `750000`, допустимый диапазон —
  32000-10000000) до разбора; слишком большие ответы обрезаются с предупреждением
- Частные и внутренние имена хостов блокируются
- `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange` и
  `tools.web.fetch.ssrfPolicy.allowIpv6UniqueLocalRange` — это узкие явные разрешения
  для доверенных стеков прокси с поддельными IP-адресами; не задавайте их, если ваш прокси не контролирует
  эти синтетические диапазоны и не применяет собственную политику назначения
- Перенаправления проверяются и ограничиваются параметром `maxRedirects` (по умолчанию `3`)
- `useTrustedEnvProxy` требует явного включения и должен использоваться только для
  прокси, контролируемых оператором и продолжающих применять политику исходящих соединений после разрешения
  DNS
- `web_fetch` работает по принципу максимальных усилий — для некоторых сайтов требуется [веб-браузер](/ru/tools/browser)

## Профили инструментов

Если вы используете профили инструментов или списки разрешений, добавьте `web_fetch` или `group:web`:

```json5
{
  tools: {
    allow: ["web_fetch"],
    // или: allow: ["group:web"]  (включает web_fetch, web_search и x_search)
  },
}
```
