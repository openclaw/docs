Инструмент `music_generate` создаёт музыку или аудио с помощью общей
возможности генерации музыки, работающей на базе ComfyUI, fal, Google, MiniMax и
OpenRouter.

<Note>
`music_generate` отображается, только если доступен хотя бы один провайдер
генерации музыки: явно задана конфигурация `agents.defaults.musicGenerationModel`
или настроена аутентификация провайдера (например, задан API-ключ).
</Note>

При запусках агента с поддержкой сессий `music_generate` запускается как фоновая
задача, отслеживает ход выполнения в реестре задач, а после готовности композиции
активирует агента, чтобы тот мог уведомить пользователя и прикрепить готовое
аудио. Агент завершения следует контракту видимого ответа сессии: автоматически
отправляет итоговый ответ, если это настроено, либо использует
`message(action="send")`, если сессия требует применения инструмента сообщений.
Если сессия инициатора неактивна или её активация завершается неудачно, а
созданное аудио по-прежнему отсутствует в ответе, OpenClaw отправляет
идемпотентный прямой резервный ответ, содержащий только отсутствующее аудио.

Примеры запросов:

```text
Generate a cinematic piano track with soft strings and no vocals.
```

```text
Generate an energetic chiptune loop about launching a rocket at sunrise.
```

Используйте `action: "list"` для просмотра доступных провайдеров и моделей, а
`action: "status"` — для просмотра активной музыкальной задачи с поддержкой
сессии:

```text
/tool music_generate action=list
/tool music_generate action=status
```

Пример прямой генерации:

```text
/tool music_generate prompt="Dreamy lo-fi hip hop with vinyl texture and gentle rain" instrumental=true
```

## Поддерживаемые провайдеры

| Провайдер  | Модель по умолчанию           | Входные референсы       | Поддерживаемые параметры                              | Аутентификация                         |
| ---------- | ----------------------------- | ----------------------- | ----------------------------------------------------- | -------------------------------------- |
| ComfyUI    | `workflow`                    | До 1 изображения        | Музыка или аудио, определяемые рабочим процессом      | `COMFY_API_KEY`, `COMFY_CLOUD_API_KEY` |
| fal        | `fal-ai/minimax-music/v2.6`   | Нет                     | `lyrics`, `instrumental`, `durationSeconds`, `format` | `FAL_KEY` или `FAL_API_KEY`            |
| Google     | `lyria-3-clip-preview`        | До 10 изображений       | `lyrics`, `instrumental`, `format`                    | `GEMINI_API_KEY`, `GOOGLE_API_KEY`     |
| MiniMax    | `music-2.6`                   | Нет                     | `lyrics`, `instrumental`, `format` (только mp3)       | `MINIMAX_API_KEY` или OAuth MiniMax    |
| OpenRouter | `google/lyria-3-pro-preview`  | До 1 изображения        | `lyrics`, `instrumental`, `durationSeconds`, `format` | `OPENROUTER_API_KEY`                   |

MiniMax регистрирует два идентификатора провайдера, использующих одни и те же
модели: `minimax` для аутентификации по API-ключу и `minimax-portal` для OAuth.
Ссылки на модели соответствуют способу аутентификации
(`minimax/music-2.6` и `minimax-portal/music-2.6`); см.
[MiniMax](/ru/providers/minimax#music-generation).

Помимо модели по умолчанию на базе MiniMax, fal также предоставляет
`fal-ai/ace-step/prompt-to-audio` (wav, без текста песни и без переключателя
инструментального режима) и `fal-ai/stable-audio-25/text-to-audio` (wav, только
запрос). Модель Google по умолчанию `lyria-3-clip-preview` выводит только mp3;
`lyria-3-pro-preview` также поддерживает wav. MiniMax также предоставляет
`music-2.6-free`, `music-cover` и `music-cover-free`. OpenRouter также
предоставляет `google/lyria-3-clip-preview`.

### Матрица возможностей

Явный контракт режимов, используемый `music_generate`, контрактными тестами и
общей проверкой в реальной среде:

| Провайдер  | `generate` | `edit` | Ограничение редактирования | Общие проверки в реальной среде                                         |
| ---------- | :--------: | :----: | -------------------------- | ----------------------------------------------------------------------- |
| ComfyUI    |     ✓      |   ✓    | 1 изображение              | Не входит в общую проверку; покрывается `extensions/comfy/comfy.live.test.ts` |
| fal        |     ✓      |   —    | Нет                        | `generate`                                                              |
| Google     |     ✓      |   ✓    | 10 изображений             | `generate`, `edit`                                                      |
| MiniMax    |     ✓      |   —    | Нет                        | `generate`                                                              |
| OpenRouter |     ✓      |   ✓    | 1 изображение              | `generate`, `edit`                                                      |

## Параметры инструмента

<ParamField path="prompt" type="string" required>
  Запрос на генерацию музыки. Обязателен для `action: "generate"`.
</ParamField>
<ParamField path="action" type='"generate" | "status" | "list"' default="generate">
  `"status"` возвращает текущую задачу сессии; `"list"` показывает провайдеров.
</ParamField>
<ParamField path="model" type="string">
  Переопределение провайдера или модели (например,
  `google/lyria-3-pro-preview`, `comfy/workflow`).
</ParamField>
<ParamField path="lyrics" type="string">
  Необязательный текст песни, если провайдер поддерживает его явную передачу.
</ParamField>
<ParamField path="instrumental" type="boolean">
  Запрашивает только инструментальный результат, если провайдер это поддерживает.
</ParamField>
<ParamField path="image" type="string">
  Путь или URL одного референсного изображения.
</ParamField>
<ParamField path="images" type="string[]">
  Несколько референсных изображений (до 10 у поддерживающих провайдеров).
</ParamField>
<ParamField path="durationSeconds" type="number">
  Целевая длительность в секундах, если провайдер поддерживает указание
  длительности.
</ParamField>
<ParamField path="format" type='"mp3" | "wav"'>
  Желаемый формат вывода, если провайдер его поддерживает.
</ParamField>
<ParamField path="filename" type="string">Желаемое имя выходного файла.</ParamField>

<Note>
Не все провайдеры поддерживают все параметры. OpenClaw всё равно проверяет
жёсткие ограничения, например количество входных данных, до отправки запроса.
Если провайдер поддерживает длительность, но его максимальное значение меньше
запрошенного, OpenClaw ограничивает длительность ближайшим поддерживаемым
значением. Действительно неподдерживаемые необязательные параметры игнорируются
с предупреждением, если выбранный провайдер или модель не может их применить.
Результаты инструмента содержат применённые настройки;
`details.normalization` фиксирует все преобразования запрошенных значений в
применённые.
</Note>

Тайм-ауты запросов к провайдерам настраиваются только оператором. Если задано
`agents.defaults.musicGenerationModel.timeoutMs`, OpenClaw использует это
значение, повышает значения меньше 120000ms до 120000ms, а при отсутствии
настройки устанавливает для запросов к провайдерам тайм-аут 300000ms.

## Асинхронное поведение

Генерация музыки с поддержкой сессии выполняется как фоновая задача:

- **Фоновая задача:** `music_generate` создаёт фоновую задачу, сразу возвращает
  ответ о запуске и задаче, а затем публикует готовую композицию в последующем
  сообщении агента.
- **Предотвращение дубликатов:** пока задача находится в состоянии `queued` или
  `running`, последующие вызовы `music_generate` в той же сессии возвращают
  состояние задачи вместо запуска новой генерации. Для явной проверки
  используйте `action: "status"`. Недавно завершённый совпадающий запрос также
  исключается как дубликат в течение 2 минут.
- **Просмотр состояния:** `openclaw tasks list` или
  `openclaw tasks show <taskId>` показывает состояние задач в очереди,
  выполняющихся и завершённых задач.
- **Активация после завершения:** OpenClaw внедряет внутреннее событие завершения
  обратно в ту же сессию, чтобы модель могла самостоятельно написать
  последующий ответ пользователю.
- **Подсказка в запросе:** последующие пользовательские или ручные ходы в той же
  сессии получают небольшую подсказку среды выполнения, если музыкальная задача
  уже выполняется, чтобы модель не вызывала `music_generate` повторно вслепую.
- **Резервный режим без сессии:** в прямом или локальном контексте без реальной
  сессии агента инструмент выполняется синхронно и возвращает итоговый
  аудиорезультат в том же ходе.

### Жизненный цикл задачи

Музыкальная задача использует те же состояния, что и общий реестр задач (полную
машину состояний, включая `timed_out`, `cancelled` и `lost`, см. в разделе
[Фоновые задачи](/ru/automation/tasks#task-lifecycle)). Большинство задач генерации
музыки проходят через следующие состояния:

| Состояние   | Значение                                                                                           |
| ----------- | -------------------------------------------------------------------------------------------------- |
| `queued`    | Задача создана и ожидает, пока провайдер её примет.                                                |
| `running`   | Провайдер обрабатывает задачу (обычно от 30 секунд до 3 минут в зависимости от провайдера и длительности). |
| `succeeded` | Композиция готова; агент активируется и публикует её в беседе.                                     |
| `failed`    | Ошибка провайдера или тайм-аут; агент активируется с подробностями ошибки.                         |

Проверка состояния из CLI:

```bash
openclaw tasks list
openclaw tasks show <taskId>
openclaw tasks cancel <taskId>
```

## Конфигурация

### Выбор модели

```json5
{
  agents: {
    defaults: {
      musicGenerationModel: {
        primary: "google/lyria-3-clip-preview",
        fallbacks: ["fal/fal-ai/minimax-music/v2.6", "minimax/music-2.6"],
      },
    },
  },
}
```

### Порядок выбора провайдера

OpenClaw перебирает провайдеров в следующем порядке:

1. Параметр `model` из вызова инструмента (если агент его указал).
2. `musicGenerationModel.primary` из конфигурации.
3. Элементы `musicGenerationModel.fallbacks` по порядку.
4. Автоматическое обнаружение только по значениям по умолчанию провайдеров с
   настроенной аутентификацией:
   - сначала текущий провайдер текстовой модели по умолчанию, если он также
     предоставляет генерацию музыки;
   - затем остальные зарегистрированные провайдеры генерации музыки в алфавитном
     порядке по идентификатору провайдера.

При сбое провайдера автоматически выполняется попытка со следующим кандидатом.
Если завершаются сбоем все попытки, ошибка содержит подробности каждой из них.

Установите `agents.defaults.mediaGenerationAutoProviderFallback: false`, чтобы
использовать только явно заданные записи `model`, `primary` и `fallbacks`.

- **Общий механизм на основе провайдеров** — если нужны выбор модели, переключение

- **Путь через Plugin (ComfyUI)** — если нужен пользовательский граф рабочего процесса или

## Режимы возможностей провайдеров

Общий контракт генерации музыки поддерживает явное объявление режимов:

- `generate` — для генерации только по промпту.
- `edit` — если запрос содержит одно или несколько эталонных изображений.

В новых реализациях провайдеров следует отдавать предпочтение явным блокам режимов:

```typescript
capabilities: {
  generate: {
    maxTracks: 1,
    supportsLyrics: true,
    supportsFormat: true,
  },
  edit: {
    enabled: true,
    maxTracks: 1,
    maxInputImages: 1,
    supportsFormat: true,
  },
}
```

Устаревших плоских полей, таких как `maxInputImages`, `supportsLyrics` и
`supportsFormat`, **недостаточно** для объявления поддержки редактирования. Провайдеры
должны явно объявлять `generate` и `edit`, чтобы тесты в реальной среде, контрактные
тесты и общий инструмент `music_generate` могли детерминированно проверять
поддержку режимов.

## Тесты в реальной среде

Включаемое по желанию тестовое покрытие в реальной среде для общих встроенных провайдеров (fal, Google, MiniMax,
OpenRouter):

```bash
OPENCLAW_LIVE_TEST=1 pnpm test:live -- extensions/music-generation-providers.live.test.ts
```

Эквивалентная обёртка репозитория, запускающая тот же тестовый файл:

```bash
pnpm test:live:media:music
```

По умолчанию этот файл тестов в реальной среде использует уже экспортированные переменные окружения провайдеров
вместо сохранённых профилей аутентификации и проверяет как `generate`, так и объявленный режим `edit`,
если провайдер включает режим редактирования. Текущее покрытие:

- `google`: `generate` и `edit`
- `fal`: только `generate`
- `minimax`: только `generate`
- `openrouter`: `generate` и `edit`
- `comfy`: отдельное покрытие Comfy в реальной среде, не входящее в общий набор проверок провайдеров

Включаемое по желанию тестовое покрытие в реальной среде для встроенного пути генерации музыки ComfyUI:

```bash
OPENCLAW_LIVE_TEST=1 COMFY_LIVE_TEST=1 pnpm test:live -- extensions/comfy/comfy.live.test.ts
```

Файл тестов Comfy в реальной среде также охватывает рабочие процессы с изображениями и видео,
если соответствующие разделы настроены.
