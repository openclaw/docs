## Ручной вход (рекомендуется)

Если сайт требует входа, выполните его вручную в профиле `openclaw`
браузера хоста. Не передавайте модели свои учетные данные: автоматический вход часто
активирует защиту от ботов и может привести к блокировке учетной записи.

Используйте браузер хоста (с ручным входом) как для чтения (поиска/обсуждений), так и для
публикации в X/Twitter и на других сайтах, чувствительных к ботам. Сеансы браузера в песочнице
с большей вероятностью активируют обнаружение ботов.

Вернуться к основной документации по браузеру: [Браузер](/ru/tools/browser).

## Какой профиль Chrome используется?

OpenClaw управляет выделенным профилем Chrome с именем `openclaw` (интерфейс
с оранжевым оттенком), отдельным от вашего повседневного профиля браузера.

Для вызовов инструмента браузера агентом:

- Выбор по умолчанию: агент использует свой изолированный браузер `openclaw`.
- Используйте `profile="user"` только тогда, когда важны существующие сеансы с выполненным входом и вы
  находитесь за компьютером, чтобы нажать или подтвердить любой запрос на подключение.
- Если у вас несколько пользовательских профилей браузера, укажите профиль явно,
  а не пытайтесь угадать.

Два способа получить доступ к профилю `openclaw`:

1. Попросите агента открыть браузер, затем выполните вход самостоятельно.
2. Откройте его через CLI:

```bash
openclaw browser start
openclaw browser open https://x.com
```

Для профиля, отличного от используемого по умолчанию, укажите `--browser-profile <name>` перед
подкомандой (по умолчанию используется `openclaw`):

```bash
openclaw browser --browser-profile <name> open https://x.com
```

## Песочница: разрешение доступа к браузеру хоста

Если агент работает в песочнице, его вызовы инструмента `browser` по умолчанию направляются в браузер
песочницы, а не хоста. Чтобы агент мог вместо этого обращаться к браузеру хоста:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

Вызовы CLI всегда направляются в браузер хоста, а не в песочницу, поэтому вы можете
самостоятельно открыть браузер хоста независимо от этой настройки:

```bash
openclaw browser --browser-profile openclaw open https://x.com
```

После установки `sandbox.browser.allowHostControl: true` вызовы инструмента `browser`
агентом также смогут обращаться к хосту. В качестве альтернативы отключите песочницу для
агента, который публикует обновления.
