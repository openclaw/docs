OpenClaw поддерживает [SearXNG](https://docs.searxng.org/) как **самостоятельно размещаемый,
не требующий ключа** провайдер `web_search`. SearXNG — это метапоисковая система
с открытым исходным кодом, которая объединяет результаты из Google, Bing, DuckDuckGo и других источников.

Преимущества:

- **Бесплатно и без ограничений** — ключ API или коммерческая подписка не требуются
- **Конфиденциальность / изолированная сеть** — запросы никогда не покидают вашу сеть
- **Работает везде** — без региональных ограничений коммерческих поисковых API

## Настройка

<Steps>
  <Step title="Установите Plugin">
    ```bash
    openclaw plugins install @openclaw/searxng-plugin
    ```
  </Step>
  <Step title="Запустите экземпляр SearXNG">
    ```bash
    docker run -d -p 8888:8080 searxng/searxng
    ```

    Либо используйте любое доступное вам существующее развертывание SearXNG. Инструкции
    по настройке для эксплуатации см. в [документации SearXNG](https://docs.searxng.org/).

  </Step>
  <Step title="Настройте">
    ```bash
    openclaw configure --section web
    # Выберите "searxng" в качестве провайдера
    ```

    Либо задайте переменную окружения, чтобы она была обнаружена автоматически:

    ```bash
    export SEARXNG_BASE_URL="http://localhost:8888"
    ```

  </Step>
</Steps>

## Конфигурация

```json5
{
  tools: {
    web: {
      search: {
        provider: "searxng",
      },
    },
  },
}
```

Настройки экземпляра SearXNG на уровне Plugin:

```json5
{
  plugins: {
    entries: {
      searxng: {
        config: {
          webSearch: {
            baseUrl: "http://localhost:8888",
            categories: "general,news", // необязательно
            language: "en", // необязательно
          },
        },
      },
    },
  },
}
```

`baseUrl` также принимает объект SecretRef (например, `{ source: "env", id: "SEARXNG_BASE_URL" }`).

## Переменная окружения

В качестве альтернативы конфигурации задайте `SEARXNG_BASE_URL`:

```bash
export SEARXNG_BASE_URL="http://localhost:8888"
```

Порядок разрешения: настроенная строка `baseUrl`, затем встроенная ссылка SecretRef на переменную окружения
в `baseUrl`, затем `SEARXNG_BASE_URL`. Если ни один из путей конфигурации не задан,
переменная `SEARXNG_BASE_URL` присутствует и провайдер не выбран явно, система
автоматического обнаружения выбирает SearXNG.

## Справочник по конфигурации Plugin

| Поле         | Описание                                                                   |
| ------------ | -------------------------------------------------------------------------- |
| `baseUrl`    | Базовый URL вашего экземпляра SearXNG (обязательно)                        |
| `categories` | Разделенные запятыми категории, например `general`, `news` или `science`   |
| `language`   | Код языка результатов, например `en`, `de` или `fr`                        |

Вызов инструмента `web_search` также принимает `count` (от 1 до 10 результатов),
`categories` и `language` как переопределения для отдельного вызова.

## Примечания

- **API JSON** — использует собственную конечную точку SearXNG `format=json`, а не извлечение данных из HTML
- **URL результатов с изображениями** — результаты в категории изображений содержат `img_src`, когда SearXNG
  возвращает прямой URL изображения
- **Без ключа API** — работает с любым экземпляром SearXNG без дополнительной настройки
- **Проверка базового URL** — `baseUrl` должен быть допустимым URL с `http://` или `https://`
- **Сетевая защита** — базовые URL с `http://` должны указывать на доверенный частный узел или
  узел local loopback (публичные узлы должны использовать `https://`); для базовых URL с `https://`,
  которые разрешаются в частный или внутренний адрес, действует такое же разрешение для самостоятельного
  размещения, а для базовых URL с `https://`, которые разрешаются в публичный адрес, сохраняется строгая защита от SSRF
- **Порядок автоматического обнаружения** — для SearXNG требуется настроенный `baseUrl` (порядок
  200 среди провайдеров, у которых уже есть необходимые учетные данные). Провайдеры без ключей,
  такие как DuckDuckGo или Ollama Web Search, никогда не выбираются автоматически;
  они активируются только при явном выборе `provider`
- **Самостоятельное размещение** — вы контролируете экземпляр, запросы и вышестоящие поисковые системы
- **Категории** по умолчанию имеют значение `general`, если не настроены
- **Резервная категория** — если запрос категории, отличной от `general`, выполняется успешно,
  но возвращает ноль результатов, OpenClaw один раз повторяет тот же запрос с `general`,
  прежде чем вернуть пустой набор результатов
- **Кэширование результатов** — идентичные запросы (с одинаковыми запросом, количеством, категориями,
  языком и базовым URL) кэшируются внутри процесса на короткий срок действия
- **Требование к версии** — Plugin объявляет `minHostVersion: >=2026.6.9`

<Tip>
  Чтобы API JSON SearXNG работал, убедитесь, что в вашем экземпляре SearXNG включен формат `json`
  в файле `settings.yml` в разделе `search.formats`.
</Tip>
