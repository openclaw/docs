`tokenjuice` — это необязательный внешний плагин, который сжимает зашумленные результаты инструментов `exec` и `bash`
после того, как команда уже выполнена.

Он изменяет возвращаемый `tool_result`, а не саму команду. Tokenjuice
не переписывает ввод оболочки, не запускает команды повторно и не изменяет коды завершения.

Сейчас это применимо к встроенным запускам OpenClaw и динамическим инструментам OpenClaw в
среде app-server Codex. Tokenjuice подключается к промежуточному обработчику результатов инструментов OpenClaw и
сокращает вывод перед его возвратом в активный сеанс среды выполнения.

## Включение плагина

Установите один раз:

```bash
openclaw plugins install clawhub:@openclaw/tokenjuice
```

Затем включите его:

```bash
openclaw config set plugins.entries.tokenjuice.enabled true
```

Эквивалентная команда:

```bash
openclaw plugins enable tokenjuice
```

Если вы предпочитаете редактировать конфигурацию напрямую:

```json5
{
  plugins: {
    entries: {
      tokenjuice: {
        enabled: true,
      },
    },
  },
}
```

## Что изменяет tokenjuice

- Сжимает зашумленные результаты `exec` и `bash` перед их возвратом в сеанс.
- Не изменяет исходное выполнение команды.
- Применяет политику безопасной инвентаризации: точное чтение содержимого файлов остается без изменений, отдельные команды инвентаризации репозитория могут сжиматься, а небезопасные смешанные последовательности команд остаются без изменений.
- Остается опциональным: отключите плагин, если вам везде нужен дословный вывод.

## Проверка работы

1. Включите плагин.
2. Запустите сеанс, который может вызывать `exec`.
3. Выполните команду с большим объемом вывода, например `git status`.
4. Убедитесь, что возвращенный результат инструмента короче и лучше структурирован, чем необработанный вывод оболочки.

## Отключение плагина

```bash
openclaw config set plugins.entries.tokenjuice.enabled false
```

Или:

```bash
openclaw plugins disable tokenjuice
```
