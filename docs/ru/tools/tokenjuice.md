`tokenjuice` — это необязательный внешний Plugin, который сжимает зашумлённые результаты инструментов `exec` и `bash` после выполнения команды.

Он изменяет возвращаемый `tool_result`, а не саму команду. Tokenjuice не переписывает ввод оболочки, не запускает команды повторно и не изменяет коды завершения.

Сейчас это применяется к встроенным запускам OpenClaw и динамическим инструментам OpenClaw в среде app-server Codex. Tokenjuice подключается к промежуточному обработчику результатов инструментов OpenClaw и сокращает вывод перед его возвратом в активный сеанс среды выполнения.

## Включение Plugin

Установите один раз:

```bash
openclaw plugins install clawhub:@openclaw/tokenjuice
```

Затем включите его:

```bash
openclaw config set plugins.entries.tokenjuice.enabled true
```

Эквивалентная команда:

```bash
openclaw plugins enable tokenjuice
```

Если вы предпочитаете редактировать конфигурацию напрямую:

```json5
{
  plugins: {
    entries: {
      tokenjuice: {
        enabled: true,
      },
    },
  },
}
```

## Что изменяет tokenjuice

- Сжимает зашумлённые результаты `exec` и `bash` перед их возвратом в сеанс.
- Не изменяет исходное выполнение команды.
- Применяет безопасную политику инвентаризации: точное чтение содержимого файлов остаётся без изменений, отдельные команды инвентаризации репозитория могут сжиматься, а небезопасные смешанные последовательности команд остаются без изменений.
- Включается только по явному запросу: отключите Plugin, если вам везде нужен дословный вывод.

## Проверка работы

1. Включите Plugin.
2. Запустите сеанс, который может вызывать `exec`.
3. Выполните команду с объёмным выводом, например `git status`.
4. Убедитесь, что возвращённый результат инструмента короче и лучше структурирован, чем исходный вывод оболочки.

## Отключение Plugin

```bash
openclaw config set plugins.entries.tokenjuice.enabled false
```

Или:

```bash
openclaw plugins disable tokenjuice
```
