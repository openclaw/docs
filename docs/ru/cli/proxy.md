# `openclaw proxy`

Проверяйте маршрутизацию через прокси, управляемый оператором, или запускайте локальный явный отладочный прокси и анализируйте перехваченный трафик.

```bash
openclaw proxy validate [--json] [--proxy-url <url>] [--proxy-ca-file <path>] [--allowed-url <url>] [--denied-url <url>] [--apns-reachable] [--apns-authority <url>] [--timeout-ms <ms>]
openclaw proxy start [--host <host>] [--port <port>]
openclaw proxy run [--host <host>] [--port <port>] -- <cmd...>
openclaw proxy coverage
openclaw proxy sessions [--limit <count>]
openclaw proxy query --preset <name> [--session <id>]
openclaw proxy blob --id <blobId>
openclaw proxy purge
```

`validate` выполняет предварительную проверку прокси переадресации, управляемого оператором. Остальные команды — отладочные инструменты для исследования на транспортном уровне: запуск локального прокси с перехватом, выполнение дочерней команды через него, просмотр списка сеансов перехвата, анализ шаблонов трафика, чтение перехваченных двоичных объектов и удаление локальных данных перехвата.

## Проверка

Проверяет фактический URL прокси, управляемого оператором, полученный из `--proxy-url`, конфигурации (`proxy.proxyUrl`) или `OPENCLAW_PROXY_URL` в указанном порядке приоритета. Если прокси не включён и не настроен, сообщает о проблеме конфигурации; для разовой предварительной проверки без изменения конфигурации передайте `--proxy-url`.

URL управляемых прокси используют `http://` для обычного прослушивателя прокси переадресации или `https://`, когда OpenClaw должен сначала установить TLS-соединение с самой конечной точкой прокси, а затем отправлять прокси-запросы. Используйте `--proxy-ca-file`, чтобы доверять частному центру сертификации для этого TLS-соединения.

По умолчанию выполняются:

- одна проверка **разрешённого** адреса `https://example.com/` (переопределите или добавьте адрес с помощью повторяемого параметра `--allowed-url`)
- одна проверка **запрещённого** адреса с использованием временного контрольного адреса local loopback (переопределите с помощью повторяемого параметра `--denied-url`)

Для пользовательских целей `--denied-url` действует принцип запрета при сбое: как HTTP-ответы, так и неоднозначные транспортные ошибки считаются сбоями, если вы не можете независимо подтвердить характерный для развёртывания сигнал запрета. Встроенный контрольный адрес local loopback — единственная цель, для которой транспортная ошибка считается доказательством блокировки.

Добавьте `--apns-reachable`, чтобы также открыть через прокси HTTP/2-туннель CONNECT к APNs и подтвердить ответ тестовой среды APNs. Проверка отправляет заведомо недействительный токен поставщика, поэтому ответ APNs `403 InvalidProviderToken` считается успешным сигналом доступности, а не сбоем.

### Параметры

| Флаг                     | Действие                                                                                                                           |
| ------------------------ | ---------------------------------------------------------------------------------------------------------------------------------- |
| `--json`                 | вывести машиночитаемый JSON                                                                                                       |
| `--proxy-url <url>`      | проверить этот URL прокси `http://`/`https://` вместо значения из конфигурации или переменной окружения                            |
| `--proxy-ca-file <path>` | доверять этому PEM-файлу центра сертификации при TLS-проверке конечной точки HTTPS-прокси                                          |
| `--allowed-url <url>`    | адрес назначения, обращение к которому через прокси должно быть успешным (можно указывать несколько раз)                           |
| `--denied-url <url>`     | адрес назначения, который прокси должен блокировать (можно указывать несколько раз)                                                |
| `--apns-reachable`       | также проверить доступность HTTP/2 тестовой среды APNs через прокси                                                                |
| `--apns-authority <url>` | проверяемый адрес APNs (по умолчанию `https://api.sandbox.push.apple.com`; рабочая среда — `https://api.push.apple.com`)            |
| `--timeout-ms <ms>`      | тайм-аут каждого запроса                                                                                                          |

Завершается с кодом 1, если проверка конфигурации прокси или адресов назначения завершается сбоем.

Рекомендации по развёртыванию и семантику запрета см. в разделе [Сетевой прокси](/ru/security/network-proxy).

## Отладочный прокси

`start` запускает локальный прокси с перехватом и выводит его URL, путь к сертификату центра сертификации и путь к базе данных перехвата; для остановки нажмите Ctrl+C. По умолчанию привязывается к `127.0.0.1`, если не указан параметр `--host`.

`run` запускает локальный отладочный прокси, а затем выполняет `<cmd...>` (после `--`) с применёнными переменными окружения прокси в отдельном сеансе перехвата.

Прямая переадресация отладочного прокси открывает сокеты к вышестоящим узлам для диагностики. Когда активен режим управляемого прокси OpenClaw, прямая переадресация прокси-запросов и туннелей CONNECT по умолчанию отключена; устанавливайте `OPENCLAW_DEBUG_PROXY_ALLOW_DIRECT_CONNECT_WITH_MANAGED_PROXY=1` только для одобренной локальной диагностики.

`coverage` выводит отчёт JSON (`summary` и `entries` для каждого транспорта) о том, какие транспорты перехватываются, работают только через прокси или не охвачены.

`sessions` выводит список недавних сеансов перехвата (`--limit`, по умолчанию 20).

`query --preset <name>` выполняет встроенный запрос к перехваченному трафику, при необходимости ограничивая его сеансом `--session <id>`. Предустановки:

- `double-sends`
- `retry-storms`
- `cache-busting`
- `ws-duplicate-frames`
- `missing-ack`
- `error-bursts`

`blob --id <blobId>` выводит необработанное содержимое перехваченного двоичного объекта полезной нагрузки.

`purge` удаляет все метаданные и двоичные объекты перехваченного трафика. Перехваты представляют собой локальные отладочные данные; удалите их после завершения работы.
