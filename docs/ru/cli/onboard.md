# `openclaw onboard`

Пошаговая настройка, в которой сначала настраивается инференс: она обнаруживает существующий доступ к ИИ, требует успешного выполнения реального запроса, сохраняет только рабочий маршрут, а затем запускает Crestodian для настройки остального. `openclaw setup` — та же точка входа; `openclaw setup --baseline` только записывает базовую конфигурацию и рабочее пространство.

## Примеры

```bash
openclaw onboard
openclaw onboard --classic
openclaw onboard --modern
openclaw onboard --flow quickstart
openclaw onboard --flow manual
openclaw onboard --flow import
openclaw onboard --import-from hermes --import-source ~/.hermes
openclaw onboard --skip-bootstrap
openclaw onboard --mode remote --remote-url wss://gateway-host:18789
```

- `--classic`: открывает полный пошаговый мастер. Его нельзя использовать вместе с `--non-interactive`; для автоматической настройки не указывайте `--classic`.
- `--flow quickstart`: открывает классический мастер с минимальным числом запросов и автоматически создает токен Gateway.
- `--flow manual` (псевдоним `advanced`): открывает классический мастер с полным набором запросов для порта, привязки и аутентификации.
- `--flow import`: запускает обнаруженный поставщик миграции (например, Hermes через `--import-from hermes`), показывает предварительный план и применяет его после подтверждения. Импорт выполняется только в новой установке OpenClaw — если уже существуют конфигурация, учетные данные, сеансы или состояние рабочего пространства, сначала сбросьте их. Используйте [`openclaw migrate`](/ru/cli/migrate) для предварительных планов без внесения изменений, режима перезаписи, отчетов и точных сопоставлений.
- `--modern` — псевдоним для совместимости, запускающий диалогового помощника настройки Crestodian. Он использует ту же проверку реального инференса, что и `openclaw crestodian`, и принимает только `--workspace`, `--accept-risk`, `--non-interactive` и `--json`. Другие флаги настройки отклоняются, а не молча игнорируются.

## Пошаговый процесс

Обычная команда `openclaw onboard` запускает пошаговый процесс. Она показывает уведомление о безопасности, обнаруживает уже доступный ИИ через настроенные модели, переменные окружения с ключами API и поддерживаемые локальные CLI, а затем проверяет рекомендуемый вариант с помощью реального запроса. Если этот вариант не срабатывает, первоначальная настройка показывает причину и автоматически пробует следующий пригодный вариант.

Если автоматическое обнаружение исчерпало все варианты, выберите другой обнаруженный вариант или введите ключ API поставщика в поле с маскированным вводом. Введенный вручную ключ проверяется тем же способом с реальным запросом. Пошаговая первоначальная настройка не предлагает Crestodian и не позволяет выйти с пропуском ИИ, пока один из вариантов не пройдет проверку. OpenClaw сохраняет только проверенный маршрут модели и связанные с ним учетные данные после успешного теста; не прошедший проверку вариант не заменяет настроенную модель, а введенные для него учетные данные не сохраняются. Настройки рабочего пространства и Gateway остаются без изменений до запуска Crestodian.

В пошаговом режиме `--workspace <dir>` задает предлагаемое Crestodian рабочее пространство и изолированный контекст инференса. Они не сохраняются, пока вы не одобрите предложение настройки Crestodian. Классическая и неинтерактивная первоначальная настройка сохраняют рабочее пространство в рамках своего обычного процесса.

После успешной проверки инференса пошаговая первоначальная настройка немедленно запускает Crestodian с проверенной моделью. Затем Crestodian может настроить рабочее пространство, Gateway, каналы, агентов, плагины и другие необязательные функции. В Crestodian используйте `open channel wizard for <channel>`, чтобы передать сбор учетных данных канала терминальному мастеру с маскированным вводом. Чтобы изменить поставщика модели или способ его аутентификации, выйдите из Crestodian и выполните `openclaw onboard`; Crestodian не открывает пошаговые или классические процессы настройки поставщиков.

Если установка уже настроена, повторный запуск `openclaw onboard` сначала проверяет текущую модель по умолчанию, поэтому этот же процесс служит для проверки и восстановления. Если проверка завершается неудачно, настроенная модель никогда не заменяется автоматически — первоначальная настройка останавливается и спрашивает, как продолжить. Проверка выполняется вне рабочего пространства, поэтому предоставленная плагином рабочего пространства модель может не пройти ее, продолжая при этом работать в агенте.
Используйте `openclaw onboard --classic` для аутентификации конкретного поставщика, настройки каналов, Skills, удаленного Gateway, импорта или полного управления Gateway. Для диалоговой настройки и восстановления, не связанных с инференсом, выполните `openclaw crestodian`; `openclaw onboard --modern` — псевдоним для совместимости, использующий ту же проверку инференса. Классический мастер может дополнительно проверить модель по умолчанию с помощью реального запроса, но Crestodian не запустится, пока не пройдет собственную проверку реального инференса.

В интерактивном терминале команда `openclaw` без подкоманды выбирает действие в зависимости от состояния конфигурации:

- Если активный файл конфигурации отсутствует или не содержит заданных пользователем параметров (пуст либо содержит только метаданные), запускается пошаговая первоначальная настройка.
- Если файл конфигурации существует, но не проходит проверку, запускается классическая первоначальная настройка с рекомендацией использовать `openclaw doctor`. Crestodian требует работающего инференса и не используется для восстановления этого состояния до настройки инференса.
- Если файл конфигурации корректен, открывается обычный TUI агента. Доступный настроенный Gateway с агентом и моделью сразу открывает этот интерфейс без первоначальной настройки и Crestodian. В настроенной установке открыть Crestodian можно командой `/crestodian` в TUI или командой `openclaw crestodian`.

Незашифрованный `ws://` допускается для local loopback, литералов частных IP-адресов, `.local` и URL-адресов Gateway в Tailnet `*.ts.net`. Для других доверенных имен в частной DNS установите `OPENCLAW_ALLOW_INSECURE_PRIVATE_WS=1` в окружении процесса первоначальной настройки.

## Сброс

```bash
openclaw onboard --reset
openclaw onboard --reset --reset-scope full
```

`--reset` удаляет состояние перед запуском настройки. `--reset-scope` определяет объем сброса: `config` (только конфигурация), `config+creds+sessions` (по умолчанию, если `--reset` передан без области) или `full` (также сбрасывает рабочее пространство). Рабочее пространство сбрасывается только при `--reset-scope full`.

## Локаль

Интерактивная первоначальная настройка использует локаль мастера CLI для стандартных текстов настройки. Порядок определения:

1. `OPENCLAW_LOCALE`
2. `LC_ALL`
3. `LC_MESSAGES`
4. `LANG`
5. Английский язык по умолчанию

Мастер поддерживает локали `en`, `zh-CN` и `zh-TW`. Значения локали могут использовать форму с подчеркиванием или суффиксом POSIX, например `zh_CN.UTF-8`. Названия продуктов, имена команд, ключи конфигурации, URL-адреса, идентификаторы поставщиков, идентификаторы моделей и названия плагинов и каналов сохраняются без изменений.

```bash
OPENCLAW_LOCALE=zh-CN openclaw onboard
```

## Неинтерактивная настройка

Для `--non-interactive` требуется `--accept-risk` (подтверждение того, что агенты обладают широкими возможностями и полный доступ к системе сопряжен с риском). По умолчанию `--mode` имеет значение `local`.

```bash
openclaw onboard --non-interactive \
  --auth-choice custom-api-key \
  --custom-base-url "https://llm.example.com/v1" \
  --custom-model-id "foo-large" \
  --custom-api-key "$CUSTOM_API_KEY" \
  --secret-input-mode plaintext \
  --custom-compatibility openai \
  --custom-image-input
```

`--custom-api-key` необязателен; если он не указан, первоначальная настройка проверяет переменную окружения `CUSTOM_API_KEY`. OpenClaw автоматически помечает распространенные идентификаторы моделей компьютерного зрения (GPT-4o/4.1/5.x, Claude 3/4, Gemini, Qwen-VL, LLaVA, Pixtral и аналогичные) как поддерживающие изображения. Передайте `--custom-image-input` для неизвестных пользовательских идентификаторов моделей компьютерного зрения или `--custom-text-input`, чтобы принудительно указать в метаданных поддержку только текста. Используйте `--custom-compatibility openai-responses` для совместимых с OpenAI конечных точек, которые поддерживают `/v1/responses`, но не `/v1/chat/completions`; допустимые значения: `openai` (по умолчанию), `openai-responses`, `anthropic`.

Для LM Studio также предусмотрен отдельный флаг ключа поставщика:

```bash
openclaw onboard --non-interactive \
  --auth-choice lmstudio \
  --custom-base-url "http://localhost:1234/v1" \
  --custom-model-id "qwen/qwen3.5-9b" \
  --lmstudio-api-key "$LM_API_TOKEN" \
  --accept-risk
```

Неинтерактивная настройка Ollama:

```bash
openclaw onboard --non-interactive \
  --auth-choice ollama \
  --custom-base-url "http://ollama-host:11434" \
  --custom-model-id "qwen3.5:27b" \
  --accept-risk
```

По умолчанию `--custom-base-url` имеет значение `http://127.0.0.1:11434`. `--custom-model-id` необязателен; если он не указан, первоначальная настройка использует рекомендуемые Ollama значения по умолчанию. Здесь также поддерживаются идентификаторы облачных моделей, например `kimi-k2.5:cloud`.

Хранение ключей поставщиков в виде ссылок вместо открытого текста:

```bash
openclaw onboard --non-interactive \
  --auth-choice openai-api-key \
  --secret-input-mode ref \
  --accept-risk
```

При использовании `--secret-input-mode ref` первоначальная настройка вместо значений ключей в открытом виде записывает ссылки на переменные окружения: для поставщиков с профилями аутентификации записывается `keyRef: { source: "env", provider: "default", id: <envVar> }`; для пользовательских поставщиков таким же образом записывается `models.providers.<id>.apiKey` (например, `{ source: "env", provider: "default", id: "CUSTOM_API_KEY" }`). Условие: задайте переменную окружения поставщика в окружении процесса первоначальной настройки (например, `OPENAI_API_KEY`) и не передавайте одновременно встроенный флаг ключа, если эта переменная окружения не задана — значение флага без соответствующей переменной окружения приводит к немедленной ошибке с рекомендациями по исправлению.

### Аутентификация Gateway (неинтерактивный режим)

- `--gateway-auth token --gateway-token <token>` сохраняет токен в открытом виде. `token` — режим аутентификации по умолчанию.
- `--gateway-auth token --gateway-token-ref-env <name>` сохраняет `gateway.auth.token` как SecretRef на переменную окружения. В окружении процесса первоначальной настройки должна существовать непустая переменная окружения с таким именем.
- `--gateway-token` и `--gateway-token-ref-env` являются взаимоисключающими.
- При использовании `--install-daemon`: управляемый через SecretRef `gateway.auth.token` проверяется, но его разрешенное значение в открытом виде не сохраняется в метаданных окружения службы супервизора; если ссылку невозможно разрешить, установка прекращается с ошибкой и рекомендациями по исправлению. Если настроены и `gateway.auth.token`, и `gateway.auth.password`, а `gateway.auth.mode` не задан, установка блокируется, пока режим не будет указан явно.
- Локальная первоначальная настройка записывает `gateway.mode="local"` в конфигурацию. Если позднее в файле конфигурации отсутствует `gateway.mode`, это указывает на повреждение конфигурации или незавершенное ручное редактирование, а не на допустимый сокращенный способ включения локального режима.
- Локальная первоначальная настройка устанавливает загружаемые плагины, необходимые выбранному процессу настройки (например, Plugin среды выполнения Codex или Copilot для соответствующих вариантов аутентификации). Удаленная первоначальная настройка записывает только сведения о подключении к удаленному Gateway и никогда не устанавливает локальные пакеты плагинов.
- `--allow-unconfigured` — отдельный аварийный вариант команды `openclaw gateway run`; он не позволяет первоначальной настройке пропустить `gateway.mode`.

```bash
export OPENAI_API_KEY="your-provider-key"
export OPENCLAW_GATEWAY_TOKEN="your-token"
openclaw onboard --non-interactive \
  --mode local \
  --auth-choice openai-api-key \
  --secret-input-mode ref \
  --gateway-auth token \
  --gateway-token-ref-env OPENCLAW_GATEWAY_TOKEN \
  --accept-risk
```

### Состояние локального Gateway

- Если не указан `--skip-health`, первоначальная настройка перед успешным завершением ожидает, пока локальный Gateway станет доступен.
- `--install-daemon` сначала запускает процесс управляемой установки Gateway. Без этого флага локальный Gateway должен быть уже запущен (например, с помощью `openclaw gateway run`).
- `--skip-health` пропускает ожидание, если в автоматизации требуется только записать конфигурацию, рабочее пространство и файлы начальной загрузки.
- `--skip-bootstrap` устанавливает `agents.defaults.skipBootstrap: true` и пропускает создание `AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md` и `BOOTSTRAP.md`.
- В нативной Windows флаг `--install-daemon` сначала пытается использовать планировщик заданий, а при отказе в создании задания переключается на элемент входа в папке автозагрузки текущего пользователя.

### Интерактивный режим ссылок

- При появлении запроса выберите **Использовать ссылку на секрет**, а затем **Переменная окружения** или настроенного поставщика секретов (`file` либо `exec`).
- Перед сохранением ссылки первоначальная настройка выполняет быструю предварительную проверку и при ошибке позволяет повторить попытку.

### Варианты конечных точек Z.AI

<Note>
`--auth-choice zai-api-key` автоматически определяет оптимальную конечную точку Z.AI и модель для вашего ключа: конечные точки Coding Plan предпочитают `zai/glm-5.2` (с переходом на `glm-5.1`, если она недоступна); для конечных точек общего API по умолчанию используется `zai/glm-5.1`. Чтобы принудительно использовать конечную точку Coding Plan, укажите непосредственно `zai-coding-global` или `zai-coding-cn`.
</Note>

```bash
# Выбор конечной точки без запросов
openclaw onboard --non-interactive \
  --auth-choice zai-coding-global \
  --zai-api-key "$ZAI_API_KEY"

# Другие варианты конечных точек Z.AI: zai-coding-cn, zai-global, zai-cn
```

Mistral:

```bash
openclaw onboard --non-interactive \
  --auth-choice mistral-api-key \
  --mistral-api-key "$MISTRAL_API_KEY"
```

## Дополнительные флаги неинтерактивного режима

Аутентификация модели на основе токена (используется с `--auth-choice token`):

| Флаг                            | Описание                                                                                                                               |
| ------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| `--token-provider <id>`         | Идентификатор поставщика токена, выпустившего токен                                                                                    |
| `--token <token>`               | Значение токена для аутентификации модели                                                                                              |
| `--token-profile-id <id>`       | Идентификатор профиля аутентификации (по умолчанию `<provider>:manual`; некоторые процессы поставщиков используют собственное значение по умолчанию, например `anthropic:default`) |
| `--token-expires-in <duration>` | Необязательный срок действия токена (например, `365d`, `12h`)                                                                          |

Cloudflare AI Gateway: `--cloudflare-ai-gateway-account-id <id>`, `--cloudflare-ai-gateway-gateway-id <id>`.

Управление установкой фоновой службы: `--no-install-daemon` / `--skip-daemon` (псевдонимы; пропускают установку службы Gateway), `--daemon-runtime <node|bun>`.

Skills: `--node-manager <npm|pnpm|bun>` (по умолчанию `npm`), `--skip-skills`.

Настройка интерфейса и хуков: `--skip-ui` (пропустить запросы Control UI/TUI), `--skip-hooks` (пропустить настройку Webhook/хуков), `--skip-channels`, `--skip-search`.

Вывод: `--suppress-gateway-token-output` скрывает содержащий токены вывод Gateway/интерфейса (подсказки о токенах, URL автоматического входа со встроенным токеном и автоматический запуск Control UI) — это полезно в общих терминалах и CI.

<Note>
`--json` не включает неинтерактивный режим при пошаговой или классической первоначальной настройке.
При использовании `--modern` JSON однократно выводит обзор Crestodian, после чего программа завершается.
Для других скриптов используйте `--non-interactive`.
</Note>

## Предварительная фильтрация поставщиков

Если выбранный способ аутентификации предполагает предпочтительного поставщика, при первоначальной настройке списки выбора модели по умолчанию и разрешённых моделей предварительно фильтруются по моделям этого поставщика. Фильтр также сопоставляет других поставщиков, принадлежащих тому же Plugin, что охватывает варианты планов для программирования, такие как `volcengine`/`volcengine-plan` и `byteplus`/`byteplus-plan`. Если фильтр предпочтительного поставщика не находит загруженных моделей, первоначальная настройка использует нефильтрованный каталог, а не оставляет список выбора пустым.

## Дополнительные запросы для веб-поиска

Некоторые поставщики веб-поиска вызывают дополнительные запросы, относящиеся к конкретному поставщику, во время первоначальной настройки:

- **Grok** может предложить необязательную настройку `x_search` с той же аутентификацией xAI и выбором модели `x_search`.
- **Kimi** может запросить регион Moonshot API (`api.moonshot.ai` или `api.moonshot.cn`) и модель веб-поиска Kimi по умолчанию.

## Другие особенности поведения

- Поведение области личных сообщений при локальной первоначальной настройке: [справочник по настройке через CLI](/ru/start/wizard-cli-reference#outputs-and-internals).
- Самый быстрый способ начать первый чат: `openclaw dashboard` (Control UI, без настройки канала).
- Пользовательский поставщик: подключите любую конечную точку, совместимую с OpenAI или Anthropic, включая поставщиков хостинга, отсутствующих в списке. Используйте совместимость **Unknown** для автоматического определения посредством проверки в реальном времени.
- Если обнаружено состояние Hermes, первоначальная настройка предлагает процесс миграции (см. `--flow import` выше).

## Распространённые последующие команды

Используйте `openclaw configure` позднее для целевых изменений, не связанных с логическим выводом, а `openclaw
channels add` — только для настройки канала. Для изменения поставщика модели или маршрута аутентификации
вместо этого запустите `openclaw onboard`.

```bash
openclaw channels add
openclaw configure
openclaw agents add <name>
```
