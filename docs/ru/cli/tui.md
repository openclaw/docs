# `openclaw tui`

Откройте терминальный интерфейс, подключённый к Gateway, или запустите его в локальном встроенном
режиме.

Связанное руководство: [TUI](/ru/web/tui)

## Параметры

| Флаг                         | Значение по умолчанию                     | Описание                                                                           |
| ---------------------------- | ----------------------------------------- | ---------------------------------------------------------------------------------- |
| `--local`                    | `false`                                   | Использовать локальную встроенную среду выполнения агента вместо Gateway.          |
| `--url <url>`                | `gateway.remote.url` из конфигурации      | URL WebSocket для Gateway.                                                         |
| `--token <token>`            | (нет)                                     | Токен Gateway, если требуется.                                                     |
| `--password <pass>`          | (нет)                                     | Пароль Gateway, если требуется.                                                    |
| `--tls-fingerprint <sha256>` | `gateway.remote.tlsFingerprint`           | Ожидаемый отпечаток TLS-сертификата для закреплённого Gateway `wss://`.             |
| `--session <key>`            | `main` (или `global`, если область действия глобальная) | Ключ сеанса. В рабочем пространстве агента он выбирается автоматически, если не указан префикс. |
| `--deliver`                  | `false`                                   | Доставлять ответы ассистента через настроенные каналы.                             |
| `--thinking <level>`         | (значение модели по умолчанию)             | Переопределение уровня рассуждений.                                                |
| `--message <text>`           | (нет)                                     | Отправить начальное сообщение после подключения.                                  |
| `--timeout-ms <ms>`          | `agents.defaults.timeoutSeconds`          | Тайм-аут агента. При недопустимых значениях записывается предупреждение, а сами значения игнорируются. |
| `--history-limit <n>`        | `200`                                     | Количество записей истории, загружаемых при подключении.                           |

Псевдонимы `openclaw chat` и `openclaw terminal` вызывают эту команду с подразумеваемым
`--local`.

## Примечания

- `--local` нельзя использовать вместе с `--url`, `--token`, `--password` или `--tls-fingerprint`.
- `tui` по возможности разрешает настроенные SecretRef для аутентификации Gateway
  по токену или паролю (провайдеры `env`/`file`/`exec`).
- Если URL или порт не указаны явно, `tui` использует активный локальный порт Gateway,
  зарегистрированный запущенным Gateway. Явно заданные `--url`, `OPENCLAW_GATEWAY_URL`,
  `OPENCLAW_GATEWAY_PORT` и конфигурация удалённого Gateway имеют приоритет.
- При запуске из каталога настроенного рабочего пространства агента TUI автоматически выбирает
  этого агента для ключа сеанса по умолчанию (если только `--session` явно не задан как
  `agent:<id>:...`).
- Чтобы отображать имя хоста Gateway в нижней строке для нелокальных подключений
  по URL, запустите `openclaw config set tui.footer.showRemoteHost true`. По умолчанию отключено;
  никогда не отображается для подключений к loopback-адресу или локальных встроенных подключений.
- Локальный режим напрямую использует встроенную среду выполнения агента. Большинство локальных инструментов работает,
  но функции, доступные только через Gateway, недоступны.
- Локальный режим добавляет `/auth [provider]` в набор команд TUI.
- Проверки одобрения плагинов действуют и в локальном режиме: инструменты, которым требуется одобрение,
  запрашивают решение в терминале; автоматического неявного одобрения не происходит.
- [Цели](/ru/tools/goal) сеанса отображаются в нижней строке, и ими можно управлять с помощью
  `/goal`.

## Примеры

```bash
openclaw chat
openclaw tui --local
openclaw tui
openclaw tui --url ws://127.0.0.1:18789 --token <token>
openclaw tui --session main --deliver
openclaw chat --message "Сравни мою конфигурацию с документацией и расскажи, что нужно исправить"
# при запуске в рабочем пространстве агента этот агент определяется автоматически
openclaw tui --session bugfix
```

## Цикл исправления конфигурации

Используйте локальный режим, чтобы встроенный агент проверил текущую конфигурацию, сравнил
её с документацией и помог исправить её в том же терминале.

Если `openclaw config validate` уже завершается с ошибкой, сначала запустите `openclaw configure` или
`openclaw doctor --fix`; `openclaw chat` не обходит
защиту от недопустимой конфигурации.

```bash
openclaw chat
```

Затем в TUI:

```text
!openclaw config file
!openclaw docs gateway auth token secretref
!openclaw config validate
!openclaw doctor
```

Примените точечные исправления с помощью `openclaw config set` или `openclaw configure`, затем
снова запустите `openclaw config validate`. См. [TUI](/ru/web/tui) и
[Конфигурация](/ru/cli/config).
