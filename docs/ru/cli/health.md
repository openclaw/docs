# `openclaw health`

Получает снимок состояния работающего Gateway через RPC по WebSocket (CLI не подключается напрямую к сокетам каналов).

## Параметры

| Флаг             | По умолчанию | Описание                                                                       |
| ---------------- | ------------ | ------------------------------------------------------------------------------ |
| `--json`         | `false` | Выводит машиночитаемый JSON вместо текста.                                      |
| `--timeout <ms>` | `10000` | Время ожидания подключения в миллисекундах.                                     |
| `--verbose`      | `false` | Принудительно выполняет проверку в реальном времени и разворачивает вывод для всех настроенных учётных записей и агентов. |
| `--debug`        | `false` | Псевдоним для `--verbose`.                                               |

Примеры:

```bash
openclaw health
openclaw health --json
openclaw health --timeout 2500
openclaw health --verbose
openclaw health --debug
```

## Поведение

- Без `--verbose` Gateway может вернуть кэшированный снимок (актуальный до 60 секунд и не отличающийся от текущего состояния среды выполнения каналов), а затем обновить его в фоновом режиме для следующего вызывающего клиента.
- `--verbose` принудительно выполняет проверку в реальном времени (проверки учётных записей для каждого канала), выводит сведения о подключении к Gateway и разворачивает удобочитаемый вывод для всех настроенных учётных записей и агентов, а не только для агента по умолчанию.
- `--json` всегда возвращает полный снимок: каналы, проверки отдельных учётных записей, состояние загрузки плагинов, состояние карантина движка контекста, состояние кэша цен моделей, состояние цикла событий и хранилища сеансов отдельных агентов.
