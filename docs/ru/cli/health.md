# `openclaw health`

Получает снимок состояния работающего Gateway через WebSocket RPC (без прямых подключений CLI к сокетам каналов).

## Параметры

| Флаг             | По умолчанию | Описание                                                                                                      |
| ---------------- | ------------ | ------------------------------------------------------------------------------------------------------------- |
| `--json`         | `false`      | Выводит машиночитаемый JSON вместо текста.                                                                    |
| `--timeout <ms>` | `10000`      | Тайм-аут подключения в миллисекундах.                                                                         |
| `--verbose`      | `false`      | Принудительно выполняет проверку в реальном времени и расширяет вывод для всех настроенных аккаунтов и агентов. |
| `--debug`        | `false`      | Псевдоним для `--verbose`.                                                                                    |

Примеры:

```bash
openclaw health
openclaw health --json
openclaw health --timeout 2500
openclaw health --verbose
openclaw health --debug
```

## Поведение

- Без `--verbose` Gateway может вернуть кешированный снимок (актуальный до 60 секунд и соответствующий текущему состоянию среды выполнения каналов), а затем обновить его в фоновом режиме для следующего вызывающего клиента.
- `--verbose` принудительно выполняет проверку в реальном времени (проверки аккаунтов для каждого канала), выводит сведения о подключении к Gateway и расширяет человекочитаемый вывод для всех настроенных аккаунтов и агентов, а не только для агента по умолчанию.
- `--json` всегда возвращает полный снимок: каналы, результаты проверок каждого аккаунта, состояние загрузки плагинов, состояние карантина движка контекста, состояние кеша цен моделей, состояние цикла событий и хранилища сеансов каждого агента.
