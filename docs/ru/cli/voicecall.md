# `openclaw voicecall`

`voicecall` — команда, предоставляемая плагином. Она отображается, только когда плагин голосовых вызовов установлен и включён.

Когда Gateway запущен, операционные команды (`call`, `start`, `continue`, `speak`, `dtmf`, `end`, `status`) направляются в среду выполнения голосовых вызовов этого Gateway. Если ни один Gateway недоступен, они переключаются на автономную среду выполнения CLI.

## Подкоманды

```bash
openclaw voicecall setup    [--json]
openclaw voicecall smoke    [-t <phone>] [--message <text>] [--mode <m>] [--yes] [--json]
openclaw voicecall call     -m <text> [-t <phone>] [--mode <m>]
openclaw voicecall start    --to <phone> [--message <text>] [--mode <m>]
openclaw voicecall continue --call-id <id> --message <text>
openclaw voicecall speak    --call-id <id> --message <text>
openclaw voicecall dtmf     --call-id <id> --digits <digits>
openclaw voicecall end      --call-id <id>
openclaw voicecall status   [--call-id <id>] [--json]
openclaw voicecall tail     [--file <path>] [--since <n>] [--poll <ms>]
openclaw voicecall latency  [--file <path>] [--last <n>]
openclaw voicecall expose   [--mode <m>] [--path <p>] [--port <port>] [--serve-path <p>]
```

| Подкоманда | Описание                                                        |
| ---------- | --------------------------------------------------------------- |
| `setup`    | Показать результаты проверок готовности провайдера и Webhook.                     |
| `smoke`    | Выполнить проверки готовности; совершить реальный тестовый вызов только с `--yes`. |
| `call`     | Инициировать исходящий голосовой вызов.                                |
| `start`    | Псевдоним для `call`: `--to` обязателен, а `--message` необязателен. |
| `continue` | Произнести сообщение и дождаться следующего ответа.                 |
| `speak`    | Произнести сообщение, не дожидаясь ответа.                 |
| `dtmf`     | Отправить цифры DTMF в активный вызов.                             |
| `end`      | Завершить активный вызов.                                         |
| `status`   | Просмотреть активные вызовы (или один по `--call-id`).                   |
| `tail`     | Отслеживать `calls.jsonl` (полезно при тестировании провайдера).              |
| `latency`  | Сформировать сводку метрик задержки реплик из `calls.jsonl`.              |
| `expose`   | Переключить Tailscale Serve/Funnel для конечной точки Webhook.         |

## Настройка и быстрая проверка

### `setup`

По умолчанию выводит удобочитаемые результаты проверок готовности. Для сценариев передайте `--json`.

```bash
openclaw voicecall setup
openclaw voicecall setup --json
```

### `smoke`

Выполняет те же проверки готовности. Совершает реальный телефонный вызов, только когда указаны и `--to`, и `--yes`.

| Флаг               | Значение по умолчанию              | Описание                                    |
| ------------------ | ---------------------------------- | ------------------------------------------- |
| `-t, --to <phone>` | (нет)                              | Номер телефона для реальной быстрой проверки. |
| `--message <text>` | `OpenClaw voice call smoke test.` | Сообщение для воспроизведения во время проверочного вызова. |
| `--mode <mode>`    | `notify`                          | Режим вызова: `notify` или `conversation`.  |
| `--yes`            | `false`                           | Фактически совершить реальный исходящий вызов.  |
| `--json`           | `false`                           | Вывести машиночитаемый JSON.            |

```bash
openclaw voicecall smoke
openclaw voicecall smoke --to "+15555550123"        # пробный запуск
openclaw voicecall smoke --to "+15555550123" --yes  # реальный уведомительный вызов
```

<Note>
Для внешних провайдеров (`plivo`, `telnyx`, `twilio`) командам `setup` и `smoke` требуется общедоступный URL Webhook из `publicUrl`, туннель или публикация через Tailscale. Резервный адрес loopback или частного Serve отклоняется, поскольку операторы связи не могут получить к нему доступ.
</Note>

## Жизненный цикл вызова

### `call`

Инициирует исходящий голосовой вызов.

| Флаг                   | Обязателен | Значение по умолчанию | Описание                                                                |
| ---------------------- | ---------- | --------------------- | ----------------------------------------------------------------------- |
| `-m, --message <text>` | да      | (нет)            | Сообщение для воспроизведения после соединения вызова.                                   |
| `-t, --to <phone>`     | нет       | конфигурация `toNumber` | Номер телефона для вызова в формате E.164.                                                |
| `--mode <mode>`        | нет       | `conversation`    | Режим вызова: `notify` (завершить вызов после сообщения) или `conversation` (оставить соединение открытым). |

```bash
openclaw voicecall call --to "+15555550123" --message "Hello"
openclaw voicecall call -m "Heads up" --mode notify
```

### `start`

Псевдоним для `call` с другим набором флагов по умолчанию.

| Флаг               | Обязателен | Значение по умолчанию | Описание                              |
| ------------------ | ---------- | --------------------- | ------------------------------------- |
| `--to <phone>`     | да      | (нет)         | Номер телефона для вызова.                    |
| `--message <text>` | нет       | (нет)         | Сообщение для воспроизведения после соединения вызова. |
| `--mode <mode>`    | нет       | `conversation` | Режим вызова: `notify` или `conversation`.   |

### `continue`

Произносит сообщение и ожидает ответа.

| Флаг               | Обязателен | Описание                |
| ------------------ | ---------- | ----------------------- |
| `--call-id <id>`   | да      | Идентификатор вызова.   |
| `--message <text>` | да      | Сообщение для воспроизведения. |

### `speak`

Произносит сообщение, не дожидаясь ответа.

| Флаг               | Обязателен | Описание                |
| ------------------ | ---------- | ----------------------- |
| `--call-id <id>`   | да      | Идентификатор вызова.   |
| `--message <text>` | да      | Сообщение для воспроизведения. |

### `dtmf`

Отправляет цифры DTMF в активный вызов.

| Флаг                | Обязателен | Описание                                             |
| ------------------- | ---------- | ---------------------------------------------------- |
| `--call-id <id>`    | да      | Идентификатор вызова.                                |
| `--digits <digits>` | да      | Цифры DTMF (например, `ww123456#` для пауз). |

### `end`

Завершает активный вызов.

| Флаг             | Обязателен | Описание              |
| ---------------- | ---------- | --------------------- |
| `--call-id <id>` | да      | Идентификатор вызова. |

### `status`

Показывает активные вызовы.

| Флаг             | Значение по умолчанию | Описание                             |
| ---------------- | --------------------- | ------------------------------------ |
| `--call-id <id>` | (нет)  | Ограничить вывод одним вызовом. |
| `--json`         | `false` | Вывести машиночитаемый JSON. |

```bash
openclaw voicecall status
openclaw voicecall status --json
openclaw voicecall status --call-id <id>
```

## Журналы и метрики

### `tail`

Отслеживает журнал голосовых вызовов в формате JSONL. При запуске выводит последние `--since` строк, а затем передаёт новые строки по мере их записи.

| Флаг            | Значение по умолчанию         | Описание                              |
| --------------- | ----------------------------- | ------------------------------------- |
| `--file <path>` | определяется из хранилища плагина | Путь к `calls.jsonl`.         |
| `--since <n>`   | `25`                       | Количество строк для вывода перед началом отслеживания. |
| `--poll <ms>`   | `250` (минимум 50)         | Интервал опроса в миллисекундах. |

### `latency`

Формирует сводку метрик задержки реплик и ожидания прослушивания из `calls.jsonl`. Результат представляет собой JSON со сводками `recordsScanned`, `turnLatency` и `listenWait`.

| Флаг            | Значение по умолчанию         | Описание                                  |
| --------------- | ----------------------------- | ----------------------------------------- |
| `--file <path>` | определяется из хранилища плагина | Путь к `calls.jsonl`.               |
| `--last <n>`    | `200` (минимум 1)          | Количество последних записей для анализа. |

## Публикация Webhook

### `expose`

Включает, отключает или изменяет конфигурацию Tailscale Serve/Funnel для голосового Webhook.

| Флаг                  | Значение по умолчанию                     | Описание                                             |
| --------------------- | ----------------------------------------- | ---------------------------------------------------- |
| `--mode <mode>`       | `funnel`                                  | `off`, `serve` (tailnet) или `funnel` (общедоступный). |
| `--path <path>`       | конфигурация `tailscale.path` или `--serve-path` | Путь Tailscale для публикации.                       |
| `--port <port>`       | конфигурация `serve.port` или `3334`             | Порт локального Webhook.                             |
| `--serve-path <path>` | конфигурация `serve.path` или `/voice/webhook`   | Путь локального Webhook.                             |

```bash
openclaw voicecall expose --mode serve
openclaw voicecall expose --mode funnel
openclaw voicecall expose --mode off
```

<Warning>
Открывайте конечную точку Webhook только для сетей, которым доверяете. По возможности предпочитайте Tailscale Serve вместо Funnel.
</Warning>
