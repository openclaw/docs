# `openclaw transcripts`

Инспектор с доступом только для чтения к расшифровкам, созданным инструментом агента `transcripts`.
Запись, импорт и суммаризация выполняются этим инструментом, а не данным CLI.

Артефакты хранятся в каталоге состояния:

```text
$OPENCLAW_STATE_DIR/transcripts/YYYY-MM-DD/<session>/
  metadata.json
  transcript.jsonl
  summary.json
  summary.md
```

Каталог состояния по умолчанию — `~/.openclaw`; его можно переопределить с помощью `OPENCLAW_STATE_DIR`.
Каталог даты определяется временем начала сеанса; каталог сеанса представляет собой
безопасный для файловой системы идентификатор, сформированный из идентификатора сеанса.

## Команды

```bash
openclaw transcripts list
openclaw transcripts show <session>
openclaw transcripts show YYYY-MM-DD/<session>
openclaw transcripts path <session>
openclaw transcripts path YYYY-MM-DD/<session>
openclaw transcripts path <session> --dir
openclaw transcripts path <session> --metadata
openclaw transcripts path <session> --transcript
openclaw transcripts list --json
openclaw transcripts show <session> --json
openclaw transcripts path <session> --json
```

| Команда                       | Описание                                     |
| ----------------------------- | ----------------------------------------------- |
| `list`                        | Вывести список сохранённых сеансов.                           |
| `show <session>`              | Вывести сохранённый `summary.md`.                  |
| `path <session>`              | Вывести путь к `summary.md`.                    |
| `path <session> --dir`        | Вывести каталог сеанса.                    |
| `path <session> --metadata`   | Вывести `metadata.json`.                          |
| `path <session> --transcript` | Вывести `transcript.jsonl`.                       |
| `--json`                      | Вывести машиночитаемые данные (для любой подкоманды). |

`<session>` принимает либо простой идентификатор сеанса, либо селектор с датой
(`YYYY-MM-DD/<session>`). Используйте форму с датой, если один и тот же идентификатор сеанса
встречается более чем в одном дне, например `openclaw transcripts show
2026-05-22/standup`. Идентификаторы сеансов по умолчанию содержат метку времени и случайный
суффикс; задавайте сеансу фиксированный идентификатор, только если он уникален в пределах дня.

## Вывод

`list` выводит для каждого сеанса одну строку с полями, разделёнными табуляцией: селектор, время начала, заголовок,
путь к сводке.

```text
2026-05-22/standup  2026-05-22T09:00:00.000Z  Еженедельная планёрка  /Users/user/.openclaw/transcripts/2026-05-22/standup/summary.md
```

Селектор — наиболее безопасное значение для последующей передачи в `show` или `path`.

`list --json` возвращает объекты с `sessionId`, `selector`, `date`, `title`,
`startedAt`, `stoppedAt`, `source`, `path`, `summaryPath`, `hasSummary`.

`show --json` возвращает сохранённые метаданные сеанса, селектор, каталог сеанса,
путь к сводке и текст сводки в формате Markdown.

`path --json` возвращает выбранный путь и признак существования этого файла.

## Несколько сеансов в день

Сеансы группируются сначала по дате, затем по идентификатору сеанса. Десять встреч за один день образуют
десять соседних каталогов:

```text
~/.openclaw/transcripts/2026-05-22/
  transcript-2026-05-22T09-00-00-000Z-a1b2c3d4/
  transcript-2026-05-22T10-30-00-000Z-b2c3d4e5/
  standup/
```

Для автоматизации используйте автоматически создаваемые идентификаторы. Используйте фиксированный идентификатор, например `standup`, только
если он не будет повторяться в ту же дату.

## Отсутствующие сводки

Для активных сеансов `summary.md` записывается при остановке сеанса; для импортированных расшифровок
он записывается сразу после импорта. Сеанс может отображаться в `list` без
сводки, пока запись ещё активна, если при остановке произошёл сбой провайдера или если
метаданные были записаны до поступления реплик.

Используйте `path <session> --transcript` для просмотра необработанной расшифровки, доступной только для добавления,
или выполните действие `summarize` инструмента `transcripts`, чтобы заново создать сводку
в формате Markdown.

## Конфигурация

Запись включается явно (активные источники могут подключаться и записывать звук встречи). Включите её
следующим образом:

```json
{
  "transcripts": {
    "enabled": true,
    "maxUtterances": 2000
  }
}
```

- `enabled` (по умолчанию `false`): включить инструмент.
- `maxUtterances` (по умолчанию `2000`, ограничивается диапазоном 1-10000): размер буфера реплик для каждого
  сеанса.

Настройте источники автоматического запуска с помощью `transcripts.autoStart`. Каждая запись
включается самим фактом её наличия; чтобы отключить источник, удалите соответствующую запись. `discord-voice`
— встроенный источник с поддержкой автоматического запуска, которому требуются `guildId` и
`channelId`:

```json
{
  "transcripts": {
    "enabled": true,
    "autoStart": [
      {
        "providerId": "discord-voice",
        "guildId": "1234567890",
        "channelId": "2345678901"
      }
    ]
  }
}
```
