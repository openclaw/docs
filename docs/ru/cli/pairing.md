# `openclaw pairing`

Подтверждайте или проверяйте запросы на сопряжение в личных сообщениях для каналов, поддерживающих сопряжение (только личные сообщения в чатах — для сопряжения узлов и устройств используется `openclaw devices`).

См. также: [Процесс сопряжения](/ru/channels/pairing)

## Команды

```bash
openclaw pairing list telegram
openclaw pairing list --channel telegram --account work
openclaw pairing list telegram --json

openclaw pairing approve <code>
openclaw pairing approve telegram <code>
openclaw pairing approve --channel telegram --account work <code> --notify
```

## `pairing list`

Вывод ожидающих запросов на сопряжение для одного канала.

| Параметр                | Описание                                        |
| ----------------------- | ----------------------------------------------- |
| `[channel]`      | позиционный идентификатор канала                |
| `--channel <channel>`      | явно указанный идентификатор канала             |
| `--account <accountId>`      | идентификатор учётной записи для каналов с несколькими учётными записями |
| `--json`      | машиночитаемый вывод                            |

Если настроено несколько каналов с поддержкой сопряжения, передайте канал как позиционный аргумент или с помощью `--channel`. Каналы расширений работают при условии, что идентификатор канала действителен.

## `pairing approve`

Подтвердите ожидающий код сопряжения и разрешите доступ этому отправителю.

Использование:

- `openclaw pairing approve <channel> <code>`
- `openclaw pairing approve --channel <channel> <code>`
- `openclaw pairing approve <code>`, если настроен ровно один канал с поддержкой сопряжения

Параметры: `--channel <channel>`, `--account <accountId>`, `--notify` (отправить запрашивающему подтверждение в том же канале).

### Первоначальная настройка владельца

Если `commands.ownerAllowFrom` пуст при подтверждении кода сопряжения, OpenClaw также регистрирует подтверждённого отправителя как владельца команд, используя запись, привязанную к каналу, например `telegram:123456789`. Так настраивается только первый владелец — последующие подтверждения сопряжения никогда не заменяют и не расширяют `commands.ownerAllowFrom`.

Владелец команд — это учётная запись оператора-человека, которой разрешено выполнять команды только для владельца и подтверждать опасные действия, такие как `/diagnostics`, `/export-trajectory`, `/config`, а также выполнение команд. Сопряжение лишь позволяет отправителю общаться с агентом; само по себе оно не предоставляет привилегий владельца, кроме этой однократной первоначальной настройки.

Если вы подтвердили отправителя до появления этой первоначальной настройки, выполните `openclaw doctor`; команда предупредит, если владелец команд не настроен, и покажет точную команду `openclaw config set commands.ownerAllowFrom ...` для исправления.
