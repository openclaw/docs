# `openclaw pairing`

Подтверждайте или проверяйте запросы на сопряжение в личных сообщениях для каналов, поддерживающих сопряжение (только личные сообщения в чатах — для сопряжения узлов и устройств используется `openclaw devices`).

Связанный раздел: [Процесс сопряжения](/ru/channels/pairing)

## Команды

```bash
openclaw pairing list telegram
openclaw pairing list --channel telegram --account work
openclaw pairing list telegram --json

openclaw pairing approve <code>
openclaw pairing approve telegram <code>
openclaw pairing approve --channel telegram --account work <code> --notify
```

## `pairing list`

Выводит ожидающие запросы на сопряжение для одного канала.

| Параметр                | Описание                                      |
| ----------------------- | --------------------------------------------- |
| `[channel]`      | позиционный идентификатор канала              |
| `--channel <channel>`      | явно указанный идентификатор канала           |
| `--account <accountId>`      | идентификатор учётной записи для каналов с несколькими учётными записями |
| `--json`      | машиночитаемый вывод                          |

Если настроено несколько каналов с поддержкой сопряжения, укажите канал позиционно или с помощью `--channel`. Каналы расширений работают при условии, что идентификатор канала действителен.

## `pairing approve`

Подтверждает ожидающий код сопряжения и разрешает доступ соответствующему отправителю.

Использование:

- `openclaw pairing approve <channel> <code>`
- `openclaw pairing approve --channel <channel> <code>`
- `openclaw pairing approve <code>`, если настроен ровно один канал с поддержкой сопряжения

Параметры: `--channel <channel>`, `--account <accountId>`, `--notify` (отправляет подтверждение запросившему пользователю по тому же каналу).

### Первоначальная настройка владельца

Если `commands.ownerAllowFrom` пуст при подтверждении кода сопряжения, OpenClaw также регистрирует подтверждённого отправителя как владельца команд, используя запись в области канала, например `telegram:123456789`. Так назначается только первый владелец — последующие подтверждения сопряжения никогда не заменяют и не расширяют `commands.ownerAllowFrom`.

Владелец команд — это учётная запись оператора-человека, которой разрешено выполнять команды только для владельца и подтверждать опасные действия, такие как `/diagnostics`, `/export-session`, `/export-trajectory`, `/config` и подтверждения выполнения команд. Сопряжение лишь позволяет отправителю общаться с агентом; само по себе оно не предоставляет привилегий владельца, кроме этого однократного первоначального назначения.

Если отправитель был подтверждён до появления этого механизма первоначальной настройки, выполните `openclaw doctor`; команда предупредит, если владелец команд не настроен, и покажет точную команду `openclaw config set commands.ownerAllowFrom ...` для исправления.
