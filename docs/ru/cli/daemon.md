# `openclaw daemon`

Устаревший псевдоним для команд управления службой Gateway.

`openclaw daemon ...` соответствует той же поверхности управления службой, что и служебные команды `openclaw gateway ...`.

## Использование

```bash
openclaw daemon status
openclaw daemon install
openclaw daemon start
openclaw daemon stop
openclaw daemon restart
openclaw daemon uninstall
```

## Подкоманды

- `status`: показать состояние установки службы и проверить работоспособность Gateway
- `install`: установить службу (`launchd`/`systemd`/`schtasks`)
- `uninstall`: удалить службу
- `start`: запустить службу
- `stop`: остановить службу
- `restart`: перезапустить службу

## Общие параметры

- `status`: `--url`, `--token`, `--password`, `--timeout`, `--no-probe`, `--require-rpc`, `--deep`, `--json`
- `install`: `--port`, `--runtime <node|bun>`, `--token`, `--force`, `--json`
- `restart`: `--safe`, `--skip-deferral`, `--force`, `--wait <duration>`, `--json`
- жизненный цикл (`uninstall|start|stop`): `--json`

Примечания:

- `status` по возможности разрешает настроенные SecretRefs для аутентификации проверки.
- Если обязательный SecretRef аутентификации не разрешен в этом пути команды, `daemon status --json` сообщает `rpc.authWarning`, когда подключение или аутентификация проверки завершается ошибкой; передайте `--token`/`--password` явно или сначала разрешите источник секрета.
- Если проверка проходит успешно, предупреждения о неразрешенных auth-ref подавляются, чтобы избежать ложных срабатываний.
- `status --deep` добавляет проверку службы на системном уровне по мере возможности. Когда она находит другие Gateway-подобные службы, человекочитаемый вывод печатает подсказки по очистке и предупреждает, что один Gateway на машину по-прежнему является обычной рекомендацией.
- `status --deep` также запускает проверку конфигурации в режиме с учетом Plugin и показывает предупреждения настроенных манифестов Plugin (например, отсутствующие метаданные конфигурации канала), чтобы проверки установки и обновления их обнаруживали. Стандартный `status` сохраняет быстрый путь только для чтения, который пропускает проверку Plugin.
- В установках Linux systemd проверки расхождения токенов `status` включают оба источника unit: `Environment=` и `EnvironmentFile=`.
- Проверки расхождения разрешают SecretRefs `gateway.auth.token` с использованием объединенного env времени выполнения (сначала env команды службы, затем резервно env процесса).
- Если аутентификация по токену фактически не активна (явный `gateway.auth.mode` со значением `password`/`none`/`trusted-proxy` или режим не задан, пароль может победить, а кандидат токена не может победить), проверки расхождения токенов пропускают разрешение токена конфигурации.
- Когда аутентификация по токену требует токен и `gateway.auth.token` управляется через SecretRef, `install` проверяет, что SecretRef можно разрешить, но не сохраняет разрешенный токен в метаданных окружения службы.
- Если аутентификация по токену требует токен, а настроенный SecretRef токена не разрешен, установка завершается закрыто.
- Если настроены и `gateway.auth.token`, и `gateway.auth.password`, а `gateway.auth.mode` не задан, установка блокируется, пока режим не будет задан явно.
- На macOS `install` сохраняет plists LaunchAgent доступными только владельцу и загружает управляемые значения окружения службы через файл и обертку, доступные только владельцу, вместо сериализации API-ключей или ссылок env профиля аутентификации в `EnvironmentVariables`.
- Если вы намеренно запускаете несколько Gateway на одном хосте, изолируйте порты, конфигурацию/состояние и рабочие области; см. [/gateway#multiple-gateways-same-host](/ru/gateway#multiple-gateways-same-host).
- `restart --safe` просит работающий Gateway предварительно проверить активную работу и запланировать один объединенный перезапуск после завершения активной работы. Стандартный безопасный перезапуск ожидает активную работу до настроенного `gateway.reload.deferralTimeoutMs` (по умолчанию 5 минут); когда этот бюджет истекает, перезапуск выполняется принудительно. Установите `gateway.reload.deferralTimeoutMs` в `0` для бессрочного безопасного ожидания, которое никогда не принуждает перезапуск. Обычный `restart` сохраняет существующее поведение менеджера служб; `--force` остается путем немедленного переопределения.
- `restart --safe --skip-deferral` выполняет безопасный перезапуск с учетом OpenClaw, но обходит шлюз отсрочки активной работы, поэтому Gateway инициирует перезапуск немедленно, даже когда сообщается о блокировщиках. Это аварийный выход для оператора, когда зависший запуск задачи удерживает безопасный перезапуск; требует `--safe`.
