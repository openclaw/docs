# `openclaw daemon`

Устаревший псевдоним для управления службой Gateway. `openclaw daemon ...` соответствует тем же командам управления службой, что и `openclaw gateway ...`. В актуальной документации и примерах рекомендуется использовать [`openclaw gateway`](/ru/cli/gateway).

## Использование

```bash
openclaw daemon status
openclaw daemon install
openclaw daemon start
openclaw daemon stop
openclaw daemon restart
openclaw daemon uninstall
```

## Подкоманды и параметры

| Подкоманда  | Параметры                                                                                        |
| ----------- | ------------------------------------------------------------------------------------------------ |
| `status`    | `--url`, `--token`, `--password`, `--timeout`, `--no-probe`, `--require-rpc`, `--deep`, `--json` |
| `install`   | `--port`, `--runtime <node\|bun>`, `--token`, `--wrapper <path>`, `--force`, `--json`            |
| `uninstall` | `--json`                                                                                         |
| `start`     | `--json`                                                                                         |
| `stop`      | `--json`, `--disable` (только launchd: постоянно отключает KeepAlive/RunAtLoad до следующего запуска) |
| `restart`   | `--force`, `--safe`, `--skip-deferral`, `--wait <duration>`, `--json`                            |

- `status`: показывает состояние установки службы (launchd/systemd/schtasks) и проверяет работоспособность Gateway.
- `install`: устанавливает службу; `--force` повторно устанавливает или перезаписывает существующую установку.
- `restart --safe`: запрашивает у работающего Gateway предварительную проверку активных задач и планирует один объединённый перезапуск после их завершения с ограничением по времени `gateway.reload.deferralTimeoutMs` (по умолчанию 300000 мс/5 минут; задайте `0`, чтобы ожидать неограниченно долго). По истечении этого времени перезапуск всё равно выполняется принудительно. Обычная команда `restart` обращается непосредственно к диспетчеру служб; `--force` немедленно принудительно запускает перезапуск.
- `restart --safe --skip-deferral`: обходит механизм отсрочки из-за активных задач, поэтому Gateway перезапускается немедленно, даже если сообщается о блокирующих факторах. Требует `--safe`.

## Примечания

- `status` по возможности разрешает настроенные ссылки SecretRef для аутентификации при проверке. Если обязательную ссылку SecretRef разрешить не удаётся, `status --json` сообщает об этом в `rpc.authWarning`; явно передайте `--token`/`--password` или сначала устраните проблему с источником секрета. Предупреждения о неразрешённой аутентификации скрываются, если проверка во всём остальном завершается успешно.
- `status --deep` добавляет выполняемое по мере возможности сканирование на уровне системы для поиска других служб, похожих на Gateway (выводит рекомендации по очистке; по-прежнему рекомендуется использовать один Gateway на машину), и запускает проверку конфигурации с учётом плагинов, отображая предупреждения манифестов плагинов, которые пропускаются при быстрой проверке по умолчанию.
- При установке через systemd в Linux проверки расхождения токенов анализируют источники модуля как `Environment=`, так и `EnvironmentFile=`.
- Проверки расхождения токенов разрешают ссылки SecretRef из `gateway.auth.token`, используя объединённые переменные среды выполнения (сначала среду команды службы, затем среду процесса). Если аутентификация по токену фактически не активна (`gateway.auth.mode` имеет значение `password`/`none`/`trusted-proxy` либо не задан, но приоритет может получить пароль), разрешение токена из конфигурации пропускается.
- `install` проверяет, что управляемое через SecretRef значение `gateway.auth.token` можно разрешить, но никогда не сохраняет разрешённое значение в метаданных среды службы; если разрешить его невозможно, установка завершается с ошибкой без перехода в небезопасный режим.
- Если настроены и `gateway.auth.token`, и `gateway.auth.password`, а `gateway.auth.mode` не задан, `install` блокирует установку, пока режим не будет указан явно.
- В macOS команда `install` сохраняет файлы plist LaunchAgent и созданный файл среды или оболочку доступными только владельцу (режим `0600`/`0700`), а не встраивает секреты в `EnvironmentVariables`.
- При запуске нескольких экземпляров Gateway на одном хосте изолируйте порты, конфигурацию, состояние и рабочие пространства. См. раздел [Несколько экземпляров Gateway](/ru/gateway#multiple-gateways-same-host).
