# `openclaw completion`

Создаёт сценарии автодополнения для командной оболочки, кэширует их в каталоге состояния OpenClaw и при необходимости устанавливает в профиль командной оболочки.

## Использование

```bash
openclaw completion                          # вывести сценарий zsh в стандартный вывод
openclaw completion --shell fish             # вывести сценарий fish
openclaw completion --write-state            # кэшировать сценарии для всех оболочек
openclaw completion --write-state --install  # кэшировать и установить за один шаг
openclaw completion --shell bash --write-state
```

## Параметры

- `-s, --shell <shell>`: целевая командная оболочка (`zsh`, `bash`, `powershell`, `fish`; по умолчанию: `zsh`)
- `-i, --install`: установить автодополнение, добавив в профиль командной оболочки строку подключения кэшированного сценария
- `--write-state`: записать сценарий или сценарии автодополнения в `$OPENCLAW_STATE_DIR/completions` (по умолчанию `~/.openclaw/completions`) без вывода в стандартный вывод; с `--shell` записывает сценарий только для указанной оболочки, иначе — для всех четырёх
- `-y, --yes`: пропустить запросы подтверждения установки (неинтерактивный режим)

## Процесс установки

Параметр `--install` настраивает профиль для использования кэшированного сценария, поэтому кэш должен уже существовать: если его нет, команда завершится с ошибкой и предложит выполнить `openclaw completion --write-state`. Объедините `--write-state --install`, чтобы выполнить обе операции за один шаг. Если параметр `--shell` не указан, `--install` определяет оболочку по переменной `$SHELL` (если определить её не удаётся, используется zsh).

При установке в профиль командной оболочки записывается небольшой блок `# OpenClaw Completion`, а все прежние медленные строки `source <(openclaw completion ...)` заменяются строкой подключения кэшированного сценария:

| Оболочка   | Профиль                                                                                                                                                                                    |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| bash       | `~/.bashrc` (если `~/.bashrc` отсутствует, используется `~/.bash_profile`)                                                                                                                 |
| fish       | `~/.config/fish/config.fish`                                                                                                                                                               |
| powershell | `~/.config/powershell/Microsoft.PowerShell_profile.ps1` (в Windows: `Documents/PowerShell/Microsoft.PowerShell_profile.ps1` или `Documents/WindowsPowerShell/...` для Windows PowerShell) |
| zsh        | `~/.zshrc`                                                                                                                                                                                 |

## Примечания

- Без `--install` или `--write-state` команда выводит сценарий в стандартный вывод.
- При создании автодополнения заранее загружается полное дерево команд, включая CLI-команды плагинов, поэтому вложенные подкоманды также включаются.
- `openclaw update` автоматически обновляет кэш автодополнения после успешного обновления; `openclaw doctor` может исправить отсутствующую или устаревшую настройку автодополнения.
