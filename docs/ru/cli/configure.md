# `openclaw configure`

Интерактивный запрос для целевых изменений существующей настройки: учетные данные, устройства, значения по умолчанию для агента, Gateway, каналы, плагины, Skills и проверки работоспособности.

Используйте `openclaw onboard` или `openclaw setup` для полного пошагового первого запуска, `openclaw setup --baseline` только для базовой конфигурации/рабочей области, и `openclaw channels add`, когда нужно настроить только учетную запись канала.

<Note>
Раздел **Модель** включает множественный выбор для списка разрешенных `agents.defaults.models` (то, что отображается в `/model` и средстве выбора модели). Выборы настройки в области провайдера объединяют выбранные модели с существующим списком разрешенных, а не заменяют несвязанных провайдеров, уже присутствующих в конфигурации.

Повторный запуск аутентификации провайдера из configure сохраняет существующее значение `agents.defaults.model.primary`, даже когда шаг аутентификации провайдера возвращает патч конфигурации с собственной рекомендуемой моделью по умолчанию. Это означает, что добавление или повторная аутентификация xAI, OpenRouter или другого провайдера должна сделать новую модель доступной, не заменяя вашу текущую основную модель. Используйте `openclaw models auth login --provider <id> --set-default` или `openclaw models set <model>`, когда вы намеренно хотите изменить модель по умолчанию.
</Note>

Когда configure запускается с выбора аутентификации провайдера, средства выбора модели по умолчанию и списка разрешенных автоматически предпочитают этого провайдера. Для парных провайдеров, таких как Volcengine и BytePlus, то же предпочтение также сопоставляется с их вариантами плана для кодирования (`volcengine-plan/*`, `byteplus-plan/*`). Если фильтр предпочтительного провайдера дал бы пустой список, configure откатывается к нефильтрованному каталогу вместо показа пустого средства выбора.

<Tip>
`openclaw config` без подкоманды открывает тот же мастер. Используйте `openclaw config get|set|unset` для неинтерактивных правок.
</Tip>

Для веб-поиска `openclaw configure --section web` позволяет выбрать провайдера
и настроить его учетные данные. Некоторые провайдеры также показывают
последующие запросы, специфичные для провайдера:

- **Grok** может предложить необязательную настройку `x_search` с тем же OAuth-профилем xAI
  или API-ключом и позволить выбрать модель `x_search`.
- **Kimi** может запросить регион Moonshot API (`api.moonshot.ai` или
  `api.moonshot.cn`) и модель веб-поиска Kimi по умолчанию.

## Параметры

- `--section <section>`: повторяемый фильтр разделов

Доступные разделы:

- `workspace`
- `model`
- `web`
- `gateway`
- `daemon`
- `channels`
- `plugins`
- `skills`
- `health`

Примечания:

- Полный мастер и разделы, связанные с Gateway, спрашивают, где запускается Gateway, и обновляют `gateway.mode`. Фильтры разделов, которые не включают `gateway`, `daemon` или `health`, переходят напрямую к запрошенной настройке.
- После записи локальной конфигурации configure устанавливает выбранные загружаемые плагины, когда выбранный путь настройки требует их. Удаленная конфигурация Gateway не устанавливает локальные пакеты плагинов.
- Сервисы, ориентированные на каналы (Slack/Discord/Matrix/Microsoft Teams), запрашивают списки разрешенных каналов/комнат во время настройки. Можно вводить имена или идентификаторы; мастер по возможности разрешает имена в идентификаторы.
- Если вы запускаете шаг установки демона, аутентификация по токену требует токен, и `gateway.auth.token` управляется SecretRef, configure проверяет SecretRef, но не сохраняет разрешенные открытые значения токена в метаданные окружения службы supervisor.
- Если аутентификация по токену требует токен, а настроенный SecretRef токена не разрешен, configure блокирует установку демона с практическими указаниями по исправлению.
- Если настроены и `gateway.auth.token`, и `gateway.auth.password`, а `gateway.auth.mode` не задан, configure блокирует установку демона, пока режим не будет задан явно.

## Примеры

```bash
openclaw configure
openclaw configure --section web
openclaw configure --section model --section channels
openclaw configure --section gateway --section daemon
```
