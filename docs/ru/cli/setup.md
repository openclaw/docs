# `openclaw setup`

`openclaw setup` запускает тот же пошаговый процесс первоначальной настройки, что и `openclaw onboard`:
сначала он проверяет и сохраняет конфигурацию инференса, а затем запускает Crestodian для настройки
рабочего пространства, Gateway, каналов, навыков и состояния системы. Используйте `--baseline`, если
требуется только инициализировать каталоги конфигурации и рабочего пространства без мастера.

В пошаговом режиме `--workspace <dir>` — это рабочее пространство, предлагаемое Crestodian;
оно сохраняется только после вашего одобрения предложения. Базовая, классическая и
неинтерактивная настройка сохраняют указанное рабочее пространство в рамках своего обычного процесса.

`setup` принимает те же флаги первоначальной настройки, что и `openclaw onboard`, включая
аутентификацию (`--auth-choice`, `--token`, флаги ключей провайдера), Gateway
(`--gateway-port`, `--gateway-bind`, `--gateway-auth`, `--install-daemon`),
Tailscale (`--tailscale`), сброс (`--reset`, `--reset-scope`), процесс
(`--flow quickstart|advanced|manual|import`) и флаги пропуска
(`--skip-channels`, `--skip-skills`, `--skip-bootstrap`, `--skip-search`,
`--skip-health`, `--skip-ui`, `--skip-hooks`). Полный справочник флагов и
примеры неинтерактивного использования см. в разделах [Первоначальная настройка](/ru/cli/onboard) и
[Автоматизация CLI](/ru/start/wizard-cli-automation). `openclaw onboard --modern` — это псевдоним
совместимости для помощника Crestodian с обязательной проверкой инференса; эквивалента `setup` у него нет.

<Note>
`openclaw setup` предназначен для установок с изменяемой конфигурацией. В режиме Nix (`OPENCLAW_NIX_MODE=1`) OpenClaw запрещает запись при настройке, поскольку файл конфигурации управляется Nix. Используйте официальный [краткий старт nix-openclaw](https://github.com/openclaw/nix-openclaw#quick-start) или эквивалентную конфигурацию из исходного кода для другого пакета Nix.
</Note>

## Параметры

| Флаг                       | Описание                                                                                           |
| -------------------------- | ----------------------------------------------------------------------------------------------------- |
| `--workspace <dir>`        | Предлагаемое рабочее пространство в пошаговом режиме; при базовой, классической и неинтерактивной настройке сохраняется напрямую. |
| `--baseline`               | Создать каталоги базовой конфигурации, рабочего пространства и сеансов без первоначальной настройки.                                  |
| `--wizard`                 | Принимается для совместимости; по умолчанию команда настройки запускает первоначальную настройку.                                         |
| `--non-interactive`        | Выполнить первоначальную настройку без запросов.                                                                       |
| `--accept-risk`            | Подтвердить риск предоставления агенту доступа ко всей системе; обязательно при использовании `--non-interactive`.                         |
| `--mode <mode>`            | Режим первоначальной настройки: `local` или `remote`.                                                                 |
| `--flow <flow>`            | Процесс первоначальной настройки: `quickstart`, `advanced`, `manual` или `import`.                                        |
| `--reset`                  | Сбросить конфигурацию, учётные данные и сеансы перед первоначальной настройкой (рабочее пространство — только с `--reset-scope full`).   |
| `--reset-scope <scope>`    | Область сброса: `config`, `config+creds+sessions` или `full`.                                            |
| `--import-from <provider>` | Провайдер миграции, запускаемый во время первоначальной настройки.                                                          |
| `--import-source <path>`   | Исходный домашний каталог агента для `--import-from`.                                                                |
| `--import-secrets`         | Импортировать поддерживаемые секреты во время миграции при первоначальной настройке.                                                 |
| `--remote-url <url>`       | URL WebSocket удалённого Gateway.                                                                         |
| `--remote-token <token>`   | Токен удалённого Gateway (необязательно).                                                                      |
| `--json`                   | Вывести сводку в формате JSON.                                                                                |

`--classic` и `--non-interactive` взаимоисключающие: классический режим открывает
интерактивный мастер, а неинтерактивная настройка использует процесс автоматизации.

### Базовый режим

`openclaw setup --baseline` сохраняет прежнее поведение только с базовой настройкой: он
создаёт каталоги конфигурации, рабочего пространства и сеансов, а затем завершает работу без
запуска первоначальной настройки.

## Примеры

```bash
openclaw setup
openclaw setup --baseline
openclaw setup --workspace ~/.openclaw/workspace
openclaw setup --import-from hermes --import-source ~/.hermes
openclaw setup --non-interactive --accept-risk --mode remote --remote-url wss://gateway-host:18789 --remote-token <token>
```

## Примечания

- После базовой настройки запустите `openclaw setup` или `openclaw onboard`, чтобы пройти полный пошаговый процесс, `openclaw configure` — для целевых изменений или `openclaw channels add` — для добавления учётных записей каналов.
- Если обнаружено состояние Hermes, интерактивная первоначальная настройка может автоматически предложить миграцию. Для импорта при первоначальной настройке требуется чистая установка; используйте раздел [Миграция](/ru/cli/migrate) для планов пробного запуска, резервных копий и режима перезаписи вне первоначальной настройки.
