# `openclaw setup`

Запустите полный процесс первичной настройки CLI. `openclaw setup` — это псевдоним для `openclaw onboard`; используйте `--baseline`, когда нужно только инициализировать папки конфигурации/рабочей области без мастера.

<Note>
`openclaw setup` предназначена для изменяемых установок конфигурации. В режиме Nix (`OPENCLAW_NIX_MODE=1`) OpenClaw отказывается выполнять запись при настройке, потому что файл конфигурации управляется Nix. Используйте официальный [краткий старт nix-openclaw](https://github.com/openclaw/nix-openclaw#quick-start) или эквивалентную исходную конфигурацию для другого пакета Nix.
</Note>

## Параметры

| Флаг                       | Описание                                                                                         |
| -------------------------- | --------------------------------------------------------------------------------------------------- |
| `--workspace <dir>`        | Каталог рабочей области агента (по умолчанию `~/.openclaw/workspace`; сохраняется как `agents.defaults.workspace`). |
| `--baseline`               | Создать базовые папки конфигурации/рабочей области/сеанса без первичной настройки.                                |
| `--wizard`                 | Принимается для совместимости; настройка по умолчанию запускает первичную настройку.                                       |
| `--non-interactive`        | Запустить первичную настройку без запросов.                                                                     |
| `--accept-risk`            | Подтвердить риск доступа агента ко всей системе; требуется с `--non-interactive`.                       |
| `--mode <mode>`            | Режим первичной настройки: `local` или `remote`.                                                               |
| `--import-from <provider>` | Провайдер миграции, который нужно запустить во время первичной настройки.                                                        |
| `--import-source <path>`   | Домашний каталог исходного агента для `--import-from`.                                                              |
| `--import-secrets`         | Импортировать поддерживаемые секреты во время миграции при первичной настройке.                                               |
| `--remote-url <url>`       | URL WebSocket удаленного Gateway.                                                                       |
| `--remote-token <token>`   | Токен удаленного Gateway (необязательно).                                                                    |

### Базовый режим

`openclaw setup --baseline` сохраняет прежнее поведение только с базовой инициализацией: она создает каталоги конфигурации, рабочей области и сеансов, затем завершает работу без запуска первичной настройки.

## Примеры

```bash
openclaw setup
openclaw setup --baseline
openclaw setup --workspace ~/.openclaw/workspace
openclaw setup --import-from hermes --import-source ~/.hermes
openclaw setup --non-interactive --accept-risk --mode remote --remote-url wss://gateway-host:18789 --remote-token <token>
```

## Примечания

- Простая команда `openclaw setup` запускает тот же управляемый путь, что и `openclaw onboard`.
- После базовой настройки запустите `openclaw setup` или `openclaw onboard` для полного управляемого пути, `openclaw configure` для точечных изменений или `openclaw channels add`, чтобы добавить учетные записи каналов.
- Если обнаружено состояние Hermes, интерактивная первичная настройка может автоматически предложить миграцию. Первичная настройка с импортом требует свежей настройки; используйте [Миграция](/ru/cli/migrate) для планов пробного запуска, резервных копий и режима перезаписи вне первичной настройки.
