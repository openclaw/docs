# `openclaw setup`

`openclaw setup` — точка входа системного агента. В настроенной системе команда
`openclaw setup` без аргументов открывает интерактивный чат OpenClaw. В новой системе она
переходит к пошаговой первоначальной настройке. Используйте `-m`/`--message` для одного запроса или
`--baseline`, чтобы инициализировать каталоги конфигурации и рабочей области без мастера.

Порядок маршрутизации:

1. Любой параметр первоначальной настройки (`--wizard`, `--baseline`, рабочая область, сброс,
   неинтерактивный режим, сценарий, режим, Gateway, демон, пропуск, импорт, удалённый доступ или параметры
   аутентификации) запускает первоначальную настройку точно так же, как `openclaw onboard`.
2. `-m`/`--message` или `--yes` запускает системного агента.
3. Если параметры маршрутизации отсутствуют, настроенная интерактивная система открывает OpenClaw. В
   новой системе запускается первоначальная настройка. В настроенной системе `--json` выводит
   обзор системы даже без TTY; при наличии параметра первоначальной настройки сохраняется
   сводка первоначальной настройки в формате JSON.

В пошаговом режиме `--workspace <dir>` — рабочая область, предлагаемая OpenClaw;
она сохраняется только после одобрения предложения. Базовая, классическая и
неинтерактивная настройка сохраняют указанную рабочую область в рамках обычного процесса.

Пошаговое обнаружение средств инференса выполняется на хосте Gateway под управлением macOS или Linux. CLI
и приложение macOS вызывают один и тот же детектор, принадлежащий Gateway, который проверяет настроенные
модели, поддерживаемые входы через CLI, переменные среды с ключами API и уже
установленные модели Ollama или LM Studio. В ходе этой автоматической проверки локальные модели никогда не
загружаются; выбранный кандидат должен ответить на реальный запрос завершения, прежде чем
будет сохранена конфигурация его провайдера и модели.

`setup` принимает те же флаги первоначальной настройки, что и `openclaw onboard`, включая
аутентификацию (`--auth-choice`, `--token`, флаги ключей провайдера), Gateway
(`--gateway-port`, `--gateway-bind`, `--gateway-auth`, `--install-daemon`),
Tailscale (`--tailscale`), сброс (`--reset`, `--reset-scope`), сценарий
(`--flow quickstart|advanced|manual|import`) и флаги пропуска
(`--skip-channels`, `--skip-skills`, `--skip-bootstrap`, `--skip-search`,
`--skip-health`, `--skip-ui`, `--skip-hooks`). Полный справочник флагов и
примеры неинтерактивного использования приведены в разделах [Первоначальная настройка](/ru/cli/onboard) и
[Автоматизация CLI](/ru/start/wizard-cli-automation). `openclaw onboard --modern` остаётся совместимой
точкой входа для того же помощника OpenClaw, доступного после проверки средств инференса.

<Note>
`openclaw setup` предназначен для установок с изменяемой конфигурацией. В режиме Nix (`OPENCLAW_NIX_MODE=1`) OpenClaw запрещает запись при настройке, поскольку файл конфигурации управляется Nix. Используйте официальное [краткое руководство по nix-openclaw](https://github.com/openclaw/nix-openclaw#quick-start) или эквивалентную конфигурацию из исходного кода для другого пакета Nix.
</Note>

## Параметры

| Флаг                       | Описание                                                                                           |
| -------------------------- | ----------------------------------------------------------------------------------------------------- |
| `-m, --message <text>`     | Выполнить один запрос OpenClaw.                                                                             |
| `--yes`                    | Разрешить постоянную запись конфигурации для одного запроса `--message`.                                         |
| `--workspace <dir>`        | Предлагаемая рабочая область в пошаговом режиме; сохраняется напрямую при базовой, классической и неинтерактивной настройке. |
| `--baseline`               | Создать базовые каталоги конфигурации, рабочей области и сеансов без первоначальной настройки.                                  |
| `--wizard`                 | Принудительно запустить интерактивную первоначальную настройку.                                                                         |
| `--non-interactive`        | Запустить первоначальную настройку без запросов.                                                                       |
| `--accept-risk`            | Подтвердить риск доступа агента ко всей системе; обязательно при использовании `--non-interactive`.                         |
| `--mode <mode>`            | Режим первоначальной настройки: `local` или `remote`.                                                                 |
| `--flow <flow>`            | Сценарий первоначальной настройки: `quickstart`, `advanced`, `manual` или `import`.                                        |
| `--reset`                  | Сбросить конфигурацию, учётные данные и сеансы перед первоначальной настройкой (рабочую область — только с `--reset-scope full`).   |
| `--reset-scope <scope>`    | Область сброса: `config`, `config+creds+sessions` или `full`.                                            |
| `--import-from <provider>` | Провайдер миграции, запускаемый во время первоначальной настройки.                                                          |
| `--import-source <path>`   | Исходный домашний каталог агента для `--import-from`.                                                                |
| `--import-secrets`         | Импортировать поддерживаемые секреты во время миграции при первоначальной настройке.                                                 |
| `--remote-url <url>`       | URL WebSocket удалённого Gateway.                                                                         |
| `--remote-token <token>`   | Токен удалённого Gateway (необязательно).                                                                      |
| `--json`                   | Настроенная система: обзор OpenClaw. Маршрут первоначальной настройки: сводка первоначальной настройки.                           |

`--classic` и `--non-interactive` взаимоисключающие: классический режим открывает
интерактивный мастер, а неинтерактивная настройка использует путь автоматизации.

### Базовый режим

`openclaw setup --baseline` сохраняет прежнее поведение только базовой настройки: он
создаёт каталоги конфигурации, рабочей области и сеансов, а затем завершает работу без
запуска первоначальной настройки.

## Примеры

```bash
openclaw setup
openclaw setup -m "status"
openclaw setup -m "restart gateway" --yes
openclaw setup --json
openclaw setup --wizard
openclaw setup --baseline
openclaw setup --workspace ~/.openclaw/workspace
openclaw setup --import-from hermes --import-source ~/.hermes
openclaw setup --non-interactive --accept-risk --mode remote --remote-url wss://gateway-host:18789 --remote-token <token>
```

## Примечания

- После базовой настройки запустите `openclaw onboard`, чтобы пройти весь пошаговый процесс, `openclaw configure` — для целевых изменений или `openclaw channels add` — чтобы добавить учётные записи каналов.
- Если обнаружено состояние Hermes, интерактивная первоначальная настройка может автоматически предложить миграцию. Для первоначальной настройки с импортом требуется новая установка; используйте раздел [Миграция](/ru/cli/migrate) для планов пробного запуска, резервных копий и режима перезаписи вне первоначальной настройки.
