# `openclaw system`

Вспомогательные средства системного уровня для Gateway: постановка системных событий в очередь, управление Heartbeat и просмотр присутствия.

Все подкоманды `system` используют RPC Gateway и принимают общие клиентские флаги:

| Флаг              | Значение по умолчанию                              | Описание                                                                                                                                                                                            |
| ----------------- | ------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `--url <url>`     | `gateway.remote.url`, если настроено | URL WebSocket для Gateway.                                                                                                                                                                                 |
| `--token <token>` | отсутствует                                 | Токен Gateway (если требуется).                                                                                                                                                                           |
| `--timeout <ms>`  | `30000`                              | Тайм-аут RPC в миллисекундах.                                                                                                                                                                           |
| `--expect-final`  | выключено                                  | Ожидать окончательного ответа (агента).                                                                                                                                                                       |
| `--json`          | выключено                                  | Выводить JSON. `heartbeat last/enable/disable` и `system presence` всегда выводят необработанные данные RPC в формате JSON независимо от этого флага; `system event` использует его для переключения между JSON и обычной строкой `ok`. |

## Основные команды

```bash
openclaw system event --text "Check for urgent follow-ups" --mode now
openclaw system event --text "Check for urgent follow-ups" --url ws://127.0.0.1:18789 --token "$OPENCLAW_GATEWAY_TOKEN"
openclaw system heartbeat enable
openclaw system heartbeat last
openclaw system presence
```

## `system event`

По умолчанию ставит системное событие в очередь **основного** сеанса. Следующий Heartbeat внедряет его в запрос как строку `System:`. Используйте `--mode now`, чтобы запустить Heartbeat немедленно; `next-heartbeat` (по умолчанию) ожидает следующего запланированного такта.

Передайте `--session-key`, чтобы выбрать конкретный сеанс, например для передачи уведомления о завершении асинхронной задачи обратно в канал, который её запустил.

<Note>
**Исключение для времени выполнения с `--session-key`:** если указан `--session-key`, режим `--mode next-heartbeat` вместо ожидания следующего запланированного такта сводится к немедленному целевому пробуждению. Целевые пробуждения используют намерение Heartbeat `immediate`, поэтому обходят проверку исполнителя, что срок ещё не наступил, которая иначе отложила бы (и фактически отбросила) пробуждение с намерением `event`. Если требуется отложенная доставка, не указывайте `--session-key`, чтобы событие попало в основной сеанс и было доставлено со следующим обычным Heartbeat.
</Note>

Флаги:

- `--text <text>`: обязательный текст системного события.
- `--mode <mode>`: `now` или `next-heartbeat` (по умолчанию).
- `--session-key <sessionKey>`: необязательно; выбирает конкретный сеанс агента вместо основного сеанса агента. Для ключей, не принадлежащих выбранному агенту, используется основной сеанс агента.

## `system heartbeat last|enable|disable`

- `last`: показать последнее событие Heartbeat.
- `enable`: снова включить Heartbeat (используйте, если он был отключён).
- `disable`: приостановить Heartbeat.

## `system presence`

Выводит текущие известные Gateway записи о присутствии в системе (узлы, экземпляры и аналогичные строки состояния).

## Примечания

- Требуется запущенный Gateway, доступный с текущей конфигурацией (локально или удалённо).
- Системные события являются временными и не сохраняются после перезапуска.
