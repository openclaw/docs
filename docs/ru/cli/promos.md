# `openclaw promos`

Находите и активируйте промоакции на модели, опубликованные в ClawHub. Активация промоакции настраивает провайдера (аутентификацию и Plugin, когда это необходимо) и регистрирует модели промоакции — без повторного запуска первоначальной настройки и без изменения модели по умолчанию, если вы явно этого не укажете.

См. также:

- Модель по умолчанию и резервные варианты: [Модели](/ru/cli/models)
- Настройка аутентификации провайдера: [Начало работы](/ru/start/getting-started)

## Команды

```bash
openclaw promos list
openclaw promos claim <slug>
openclaw promos claim <slug> --api-key <key> --set-default
```

## `openclaw promos list`

Выводит список активных в данный момент промоакций с их моделями, рекомендуемой моделью по умолчанию, оставшимся временем действия и точной командой активации. `--json` выводит необработанные данные.

## `openclaw promos claim <slug>`

Активирует действующую промоакцию:

1. Получает промоакцию из ClawHub и проверяет, что срок её действия ещё не истёк.
2. Проверяет провайдера промоакции, способ аутентификации и заявленные пакеты Plugin на совместимость с установленной версией OpenClaw. Неизвестные идентификаторы или несоответствия пакетов отклоняются — промоакция ни при каких обстоятельствах не может заставить CLI выполнить что-либо, что он уже не умеет выполнять.
3. Использует существующие учётные данные провайдера, если они у вас есть. В противном случае запускает обычную процедуру аутентификации провайдера (сначала выводя URL регистрации из промоакции для получения бесплатного ключа). `--api-key <key>` завершает аутентификацию по ключу API без запросов, аналогично флагам неинтерактивного режима `openclaw onboard`. Чтобы не указывать ключ в командной строке, вместо этого экспортируйте переменную окружения провайдера (например, `OPENROUTER_API_KEY`) — существующие учётные данные из окружения обнаруживаются автоматически, и флаг не требуется.
4. Регистрирует модели промоакции и их псевдонимы. Существующие псевдонимы никогда не перезаписываются.
5. Предлагает сделать рекомендуемую модель промоакции моделью по умолчанию — `--set-default` пропускает этот вопрос; в противном случае настройки по умолчанию не изменяются.

После окончания срока действия промоакции провайдер прекращает предоставлять бесплатные модели; конфигурация и учётные данные остаются без изменений. В любой момент можно переключиться обратно с помощью `openclaw models set <model>`.

## Фоновое обнаружение в `models list`

`openclaw models list` также показывает промоакции без прямого обращения к ClawHub:

- Действующие предложения, модели которых вы ещё не настроили, отображаются под таблицей в группе «Доступно по промоакции», каждое со своей командой активации.
- Модели, зарегистрированные с помощью `promos claim`, получают тег `promo`, который меняется на `promo ended` после окончания срока действия предложения.
- При первом обнаружении нового предложения однократное уведомление указывает на `openclaw promos list`. Предложения, которые вы уже просматривали или активировали, больше не объявляются.

Для этого используется локально кешированная копия размещённого в ClawHub канала промоакций (обычно она обновляется раз в день с помощью условного запроса или раньше, если срок действия кешированного снимка истёк; ошибки обновления игнорируются без уведомлений). Обновление устаревших данных занимает не более 2,5 секунды и никогда не нарушает вывод списка. Вывод `--json` и `--plain` остаётся пригодным для машинной обработки: без разделов промоакций и уведомлений. При активации данные всегда повторно проверяются через действующий API ClawHub, поэтому досрочно отозванное предложение будет отклонено, даже если оно всё ещё отображается в кешированной копии.
