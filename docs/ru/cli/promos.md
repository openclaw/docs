# `openclaw promos`

Находите и активируйте промоакции на модели, опубликованные в ClawHub. Активация
промоакции настраивает провайдера (аутентификацию и плагин, когда это необходимо) и регистрирует
модели промоакции — без повторного прохождения первоначальной настройки и без изменения
модели по умолчанию, если вы этого не укажете.

См. также:

- Модель по умолчанию и резервные варианты: [Модели](/ru/cli/models)
- Настройка аутентификации провайдера: [Начало работы](/ru/start/getting-started)

## Команды

```bash
openclaw promos list
openclaw promos claim <slug>
openclaw promos claim <slug> --api-key <key> --set-default
```

## `openclaw promos list`

Выводит список действующих промоакций с их моделями, рекомендуемой
моделью по умолчанию, оставшимся временем и точной командой активации. `--json` выводит необработанные
данные.

## `openclaw promos claim <slug>`

Активирует действующую промоакцию:

1. Получает промоакцию из ClawHub и проверяет, что срок её действия не истёк.
2. Проверяет провайдера промоакции, способ аутентификации и заявленные пакеты плагинов
   на совместимость с установленной версией OpenClaw. Неизвестные идентификаторы или несоответствия пакетов
   отклоняются — промоакция никогда не может заставить CLI выполнить что-либо, чего он уже
   не умеет делать.
3. Повторно использует имеющиеся учётные данные провайдера, если они у вас есть. В противном случае
   запускает обычный процесс аутентификации провайдера (сначала выводя URL регистрации из промоакции
   для получения бесплатного ключа). `--api-key <key>` выполняет аутентификацию по ключу API без
   запросов, аналогично неинтерактивным флагам `openclaw onboard`; чтобы не указывать
   ключ в командной строке, вместо этого экспортируйте переменную среды провайдера
   (например, `OPENROUTER_API_KEY`) — существующие учётные данные из среды
   обнаруживаются автоматически, и флаг не требуется.
4. Регистрирует модели промоакции с их псевдонимами. Существующие псевдонимы
   никогда не перезаписываются.
5. Предлагает установить рекомендуемую модель промоакции в качестве модели по умолчанию —
   `--set-default` пропускает этот вопрос; в противном случае настройки моделей по умолчанию
   не изменяются.

После окончания срока действия промоакции провайдер прекращает предоставлять бесплатные модели;
ваши настройки и учётные данные остаются без изменений. В любой момент можно переключиться обратно с помощью
`openclaw models set <model>`.

## Пассивное обнаружение в `models list`

`openclaw models list` также показывает промоакции без прямого запроса
к ClawHub:

- Действующие предложения, модели которых вы ещё не настроили, отображаются в группе
  «Доступно по промоакции» под таблицей, каждое со своей командой
  активации.
- Модели, зарегистрированные через `promos claim`, получают тег `promo`, который
  меняется на `promo ended` после окончания срока действия предложения.
- При первом обнаружении нового предложения однократное уведомление указывает на
  `openclaw promos list`. Предложения, список которых вы уже просматривали или которые уже активировали, больше
  не объявляются.

Для этого используется локально кэшированная копия размещённого в ClawHub канала промоакций
(обычно она обновляется раз в день с помощью условного запроса либо раньше, когда срок действия
кэшированного снимка истекает; ошибки обновления незаметно игнорируются). Обновление устаревших данных
занимает не более 2,5 секунды и никогда не нарушает вывод списка. Вывод `--json` и
`--plain` остаётся пригодным для машинной обработки: без разделов промоакций и уведомлений.
При активации всегда выполняется повторная проверка через действующий API ClawHub, поэтому досрочно отозванное
предложение отклоняется, даже если оно всё ещё присутствует в кэшированной копии.
