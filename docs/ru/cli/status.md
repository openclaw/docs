Диагностика каналов и сеансов.

```bash
openclaw status
openclaw status --all
openclaw status --deep
openclaw status --usage
```

| Флаг                    | Описание                                                                                                     |
| ----------------------- | --------------------------------------------------------------------------------------------------------------- |
| `--all`                 | Полная диагностика (только чтение, результат можно вставить в сообщение). Включает аудит безопасности, проверку совместимости плагинов и проверки векторной памяти. |
| `--deep`                | Выполняет проверки в реальном времени (WhatsApp Web + Telegram + Discord + Slack + Signal). Также включает аудит безопасности.         |
| `--usage`               | Выводит нормализованные окна использования провайдера в формате `X% left`.                                                          |
| `--json`                | Машиночитаемый вывод.                                                                                        |
| `--verbose` / `--debug` | Также выводит необработанный результат определения целевого адреса Gateway перед отчётом.                                                 |

Обычная команда `openclaw status` использует быстрый путь только для чтения и помечает память как
`not checked`, а не как недоступную, если проверка памяти пропущена. Ресурсоёмкий
аудит безопасности, проверка совместимости плагинов и проверки векторной памяти выполняются командами
`openclaw status --all`, `openclaw status --deep`, `openclaw security audit`
и `openclaw memory status --deep`.

## Определение сеанса и модели

- В выводе состояния сеанса `Execution:` отделяется от `Runtime:`. `Execution`
  — это путь песочницы (`direct`, `docker/*`), а `Runtime` сообщает,
  использует ли сеанс `OpenClaw Default`, `OpenAI Codex`, серверную часть
  CLI или серверную часть ACP, например `codex (acp/acpx)`. Различия между
  провайдером, моделью и средой выполнения описаны в разделе
  [Среды выполнения агентов](/ru/concepts/agent-runtimes).
- Если текущий снимок сеанса содержит мало данных, `/status` может восстановить
  счётчики токенов и кеша из последнего журнала использования в транскрипте. Существующие
  ненулевые актуальные значения по-прежнему имеют приоритет над резервными значениями из транскрипта.
- Резервное восстановление из транскрипта также может определить метку активной модели среды выполнения,
  если она отсутствует в записи текущего сеанса. Если эта модель из транскрипта отличается
  от выбранной модели, для определения окна контекста состояние использует
  восстановленную модель среды выполнения, а не выбранную.
- При подсчёте размера запроса резервное восстановление из транскрипта предпочитает большее
  итоговое значение, ориентированное на запрос, если метаданные сеанса отсутствуют или содержат меньшее значение, чтобы
  в сеансах пользовательских провайдеров отображение числа токенов не сводилось к `0`.
- Если сеанс закреплён за моделью, отличной от настроенной
  основной модели, состояние выводит оба значения, причину (`session override`) и
  подсказку `/model default`. Настроенная основная модель применяется к новым или
  незакреплённым сеансам; существующие закреплённые сеансы сохраняют выбранную для них модель,
  пока закрепление не будет снято.
- Если настроено несколько агентов, вывод включает хранилища сеансов каждого агента.

## Использование и квота

- `--usage` выводит нормализованные окна использования провайдера в формате `X% left`.
- Необработанные поля MiniMax `usage_percent` / `usagePercent` содержат оставшуюся квоту,
  поэтому OpenClaw инвертирует их перед отображением; при наличии полей со счётчиками
  они имеют приоритет. В ответах `model_remains` предпочтение отдаётся записи модели чата,
  при необходимости метка окна вычисляется по временным меткам, а название модели
  включается в метку плана.
- Ошибки обновления цен моделей отображаются как необязательные предупреждения о ценах.
  Они не означают, что Gateway или каналы неисправны.

## Обзор и состояние обновления

- Обзор включает состояние установки и работы службы Gateway и службы узла на хосте,
  если оно доступно, а также краткое время работы процесса Gateway и время работы системы хоста.
- Обзор включает канал обновлений и SHA Git (для рабочих копий исходного кода).
- Сведения об обновлении отображаются в обзоре; если обновление доступно, состояние
  выводит подсказку запустить `openclaw update` (см. раздел [Обновление](/ru/install/updating)).

## Секреты

- Поверхности состояния только для чтения (`status`, `status --json`, `status --all`)
  по возможности разрешают поддерживаемые ссылки SecretRef для соответствующих путей конфигурации.
- Если для поддерживаемого канала настроена ссылка SecretRef, но она недоступна
  в текущем пути выполнения команды, состояние остаётся доступным только для чтения и сообщает о неполном выводе,
  а не завершается с ошибкой. В человекочитаемом выводе отображаются предупреждения, например «настроенный токен
  недоступен в этом пути выполнения команды», а вывод JSON содержит
  `secretDiagnostics`.
- Если локальное для команды разрешение SecretRef завершается успешно, состояние предпочитает
  разрешённый снимок и удаляет временные маркеры канала «секрет недоступен»
  из окончательного вывода.
- `status --all` включает строку обзора секретов и раздел диагностики,
  содержащий сводку диагностических данных о секретах (сокращённую для удобства чтения), при этом
  создание отчёта не прерывается.

## Память

`status --json --all` сообщает сведения о памяти из среды выполнения активного плагина памяти,
выбранной параметром `plugins.slots.memory`. Пользовательские плагины памяти могут оставлять
встроенный компонент `agents.defaults.memorySearch.enabled` отключённым и при этом сообщать
о состоянии собственных файлов, фрагментов, векторов и FTS.
