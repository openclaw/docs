Диагностика каналов и сеансов.

```bash
openclaw status
openclaw status --all
openclaw status --deep
openclaw status --usage
```

Примечания:

- `--deep` запускает живые пробы (WhatsApp Web + Telegram + Discord + Slack + Signal).
- Обычный `openclaw status` остается на быстром пути только для чтения и помечает память как `not checked`, а не как недоступную, когда пропускает проверку памяти. Тяжелый аудит безопасности, совместимость plugin и пробы векторов памяти оставлены для `openclaw status --all`, `openclaw status --deep`, `openclaw security audit` и `openclaw memory status --deep`.
- `status --json --all` сообщает сведения о памяти из активной среды выполнения plugin памяти, выбранной через `plugins.slots.memory`. Пользовательские plugin памяти могут оставить встроенный `agents.defaults.memorySearch.enabled` отключенным и при этом сообщать собственное состояние файлов, фрагментов, векторов и FTS.
- `--usage` выводит нормализованные окна использования провайдера в виде `X% left`.
- Вывод состояния сеанса разделяет `Execution:` и `Runtime:`. `Execution` — это путь песочницы (`direct`, `docker/*`), а `Runtime` сообщает, использует ли сеанс `OpenClaw Default`, `OpenAI Codex`, CLI-бэкенд или ACP-бэкенд, например `codex (acp/acpx)`. См. [среды выполнения агентов](/ru/concepts/agent-runtimes), где объясняется различие между провайдером, моделью и средой выполнения.
- Необработанные поля MiniMax `usage_percent` / `usagePercent` означают оставшуюся квоту, поэтому OpenClaw инвертирует их перед отображением; поля на основе количества имеют приоритет, когда присутствуют. Ответы `model_remains` предпочитают запись модели чата, при необходимости выводят метку окна из временных меток и включают имя модели в метку плана.
- Когда текущий снимок сеанса разрежен, `/status` может дозаполнить счетчики токенов и кеша из самого свежего журнала использования транскрипта. Существующие ненулевые живые значения все равно имеют приоритет над резервными значениями из транскрипта.
- `/status` включает краткое время работы процесса Gateway и время работы хост-системы.
- Резервные данные транскрипта также могут восстановить метку активной модели среды выполнения, если она отсутствует в живой записи сеанса. Если эта модель из транскрипта отличается от выбранной модели, status определяет контекстное окно по восстановленной модели среды выполнения, а не по выбранной.
- Когда сеанс закреплен за моделью, отличной от настроенной основной, status выводит оба значения, причину (`session override`) и понятную подсказку (`/model default`). Настроенная основная модель применяется к новым или незакрепленным сеансам; существующие закрепленные сеансы сохраняют свой выбор сеанса, пока он не будет очищен.
- Для учета размера промпта резервные данные транскрипта предпочитают больший общий показатель, ориентированный на промпт, когда метаданные сеанса отсутствуют или меньше, поэтому сеансы пользовательских провайдеров не сворачиваются до отображения `0` токенов.
- Вывод включает хранилища сеансов по агентам, когда настроено несколько агентов.
- Обзор включает состояние установки и выполнения хост-сервиса Gateway + узла, когда оно доступно.
- Обзор включает канал обновления + SHA git (для исходных рабочих копий).
- Информация об обновлениях отображается в обзоре; если обновление доступно, status выводит подсказку запустить `openclaw update` (см. [обновление](/ru/install/updating)).
- Сбои обновления цен моделей отображаются как необязательные предупреждения о ценах. Они не
  означают, что Gateway или каналы неработоспособны.
- Поверхности состояния только для чтения (`status`, `status --json`, `status --all`) по возможности разрешают поддерживаемые SecretRef для целевых путей конфигурации.
- Если поддерживаемый SecretRef канала настроен, но недоступен в текущем пути команды, status остается только для чтения и сообщает ухудшенный вывод вместо сбоя. Вывод для человека показывает предупреждения, например «настроенный токен недоступен в этом пути команды», а JSON-вывод включает `secretDiagnostics`.
- Когда локальное для команды разрешение SecretRef успешно, status предпочитает разрешенный снимок и очищает временные маркеры канала о недоступности секрета из итогового вывода.
- `status --all` включает строку обзора секретов и раздел диагностики, который суммирует диагностику секретов (усеченную для удобства чтения), не останавливая создание отчета.
