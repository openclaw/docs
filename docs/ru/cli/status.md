Диагностика каналов и сеансов.

```bash
openclaw status
openclaw status --all
openclaw status --deep
openclaw status --usage
```

| Флаг                    | Описание                                                                                                                     |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| `--all`                 | Полная диагностика (только для чтения, результат можно вставить в сообщение). Включает аудит безопасности, проверку совместимости плагинов и диагностику векторной памяти. |
| `--deep`                | Запускает проверки в реальном времени (WhatsApp Web + Telegram + Discord + Slack + Signal). Также включает аудит безопасности. |
| `--usage`               | Выводит нормализованные интервалы использования провайдера в виде `X% осталось`.                                              |
| `--json`                | Машиночитаемый вывод.                                                                                                        |
| `--verbose` / `--debug` | Также выводит необработанный результат определения целевого Gateway перед отчётом.                                            |

Обычная команда `openclaw status` использует быстрый режим только для чтения и помечает память как
`не проверено`, а не как недоступную, если проверка памяти пропущена. Ресурсоёмкий
аудит безопасности, проверка совместимости плагинов и диагностика векторной памяти выполняются командами
`openclaw status --all`, `openclaw status --deep`, `openclaw security audit`
и `openclaw memory status --deep`.

## Определение сеанса и модели

- В выводе состояния сеанса `Выполнение:` отделено от `Среда выполнения:`. `Выполнение`
  обозначает путь песочницы (`direct`, `docker/*`), а `Среда выполнения` показывает,
  использует ли сеанс `OpenClaw Default`, `OpenAI Codex`, серверную часть CLI
  или серверную часть ACP, например `codex (acp/acpx)`. Различия между
  провайдером, моделью и средой выполнения описаны в разделе
  [Среды выполнения агента](/ru/concepts/agent-runtimes).
- Если текущий снимок сеанса содержит мало данных, `/status` может дополнить счётчики
  токенов и кэша данными из последнего журнала использования транскрипта. При этом
  существующие ненулевые актуальные значения имеют приоритет над резервными значениями из транскрипта.
- Резервные данные транскрипта также позволяют восстановить метку активной модели среды выполнения,
  если она отсутствует в актуальной записи сеанса. Если модель из транскрипта отличается
  от выбранной модели, состояние определяет размер контекстного окна по
  восстановленной модели среды выполнения, а не по выбранной.
- При учёте размера промпта резервные данные транскрипта отдают предпочтение большему
  общему значению, ориентированному на промпт, если метаданные сеанса отсутствуют или содержат
  меньшее значение, чтобы для сеансов пользовательских провайдеров не отображалось `0` токенов.
- Если за сеансом закреплена модель, отличающаяся от настроенной
  основной модели, состояние выводит оба значения, причину (`переопределение сеанса`) и
  подсказку `/model default`. Настроенная основная модель применяется к новым или
  незакреплённым сеансам; существующие закреплённые сеансы сохраняют выбранную для них модель,
  пока закрепление не будет снято.
- Если настроено несколько агентов, вывод содержит хранилища сеансов каждого агента.

## Использование и квота

- `--usage` выводит нормализованные интервалы использования провайдера в виде `X% осталось`.
- Необработанные поля MiniMax `usage_percent` / `usagePercent` обозначают оставшуюся квоту,
  поэтому OpenClaw инвертирует их перед отображением; поля на основе количества имеют
  приоритет, если они присутствуют. Для ответов `model_remains` предпочтение отдаётся записи модели чата;
  при необходимости метка интервала вычисляется по временным меткам, а название модели
  включается в метку тарифного плана.
- Ошибки обновления цен моделей отображаются как необязательные предупреждения о ценах.
  Они не означают, что Gateway или каналы неработоспособны.

## Обзор и состояние обновления

- Обзор содержит состояние установки и среды выполнения службы хоста Gateway + Node,
  если оно доступно, а также краткое время работы процесса Gateway и время работы системы хоста.
- Обзор содержит канал обновлений и SHA Git (для рабочих копий исходного кода).
- Сведения об обновлениях отображаются в обзоре; если обновление доступно, состояние
  выводит подсказку запустить `openclaw update` (см. раздел [Обновление](/ru/install/updating)).

## Секреты

- Поверхности состояния только для чтения (`status`, `status --json`, `status --all`)
  по возможности разрешают поддерживаемые ссылки SecretRef для соответствующих путей конфигурации.
- Если для поддерживаемого канала настроена ссылка SecretRef, но она недоступна в
  текущем пути выполнения команды, состояние остаётся доступным только для чтения и сообщает о
  неполном выводе вместо аварийного завершения. В человекочитаемом выводе отображаются предупреждения,
  например «настроенный токен недоступен в этом пути выполнения команды», а вывод JSON содержит
  `secretDiagnostics`.
- Если локальное для команды разрешение SecretRef выполнено успешно, состояние использует
  разрешённый снимок и удаляет из итогового вывода временные отметки канала
  «секрет недоступен».
- `status --all` содержит строку обзора секретов и раздел диагностики,
  в котором сводятся диагностические данные секретов (с сокращением для удобства чтения), не
  останавливая формирование отчёта.

## Память

`status --json --all` сообщает сведения о памяти из среды выполнения активного плагина памяти,
выбранного параметром `plugins.slots.memory`. Пользовательские плагины памяти могут оставлять
встроенный параметр `agents.defaults.memorySearch.enabled` отключённым и при этом сообщать
собственное состояние файлов, фрагментов, векторов и FTS.
