```bash
openclaw onboard
```

Первоначальная настройка через CLI — рекомендуемый способ настройки в терминале на macOS, Linux и
Windows (нативно или в WSL2). По умолчанию она обнаруживает уже доступный на
компьютере доступ к ИИ, проверяет его реальным ответом модели и запускает OpenClaw для
настройки рабочего пространства, Gateway и дополнительных функций. `openclaw setup` запускает тот же процесс ([Настройка](/ru/cli/setup) описывает
вариант `--baseline`, настраивающий только конфигурацию). Пользователи настольной версии Windows также могут начать
с [Windows Hub](/ru/platforms/windows).

Пошаговая первоначальная настройка сначала настраивает инференс. Она обнаруживает доступный доступ к ИИ,
требует реального ответа модели и только после этого запускает [OpenClaw](/cli/openclaw)
для настройки остальных компонентов OpenClaw. Выбор **Пропустить пока** завершает первоначальную настройку
без запуска OpenClaw.

Классический мастер по-прежнему доступен для пользовательских провайдеров, настройки удалённого Gateway,
сопряжения каналов, управления демоном, навыков и импорта. Запустите его явно
с помощью `openclaw onboard --classic`; пошаговый выбор инференса не перенаправляет
в него. После успешной проверки инференса OpenClaw может использовать `open channel wizard for
<channel>`, чтобы передать настройку каналов, требующую секретов, терминальному мастеру с маскированием данных.
Чтобы изменить провайдера модели или способ его аутентификации, завершите работу OpenClaw и запустите
`openclaw onboard`; OpenClaw не открывает пошаговые или классические процессы настройки провайдера.

<Info>
Самый быстрый способ начать первый чат: завершите пошаговую настройку, запустите `openclaw dashboard` и общайтесь
в браузере через интерфейс управления. Документация: [Панель управления](/ru/web/dashboard).
</Info>

## Локаль

Мастер локализует фиксированный текст первоначальной настройки. Порядок определения: `OPENCLAW_LOCALE`,
`LC_ALL`, `LC_MESSAGES`, `LANG`, затем английский. Поддерживаемые локали: `en`,
`zh-CN`, `zh-TW`.

```bash
OPENCLAW_LOCALE=zh-CN openclaw onboard
```

Названия продуктов, команды, ключи конфигурации, URL, идентификаторы провайдеров, идентификаторы моделей и
метки плагинов и каналов остаются на английском независимо от локали.

Чтобы позднее перенастроить параметры, не связанные с инференсом:

```bash
openclaw configure
openclaw agents add <name>
```

<Note>
`--json` не включает неинтерактивный режим автоматически. Для скриптов используйте `--non-interactive` (см. [Автоматизация CLI](/ru/start/wizard-cli-automation)).
</Note>

<Tip>
Классический мастер включает этап настройки веб-поиска, на котором можно выбрать провайдера: Brave,
DuckDuckGo, Exa, Firecrawl, Gemini, Grok, Kimi, MiniMax Search, Ollama Web
Search, Perplexity, SearXNG или Tavily. Некоторым требуется ключ API, другие
работают без ключа. Позднее это можно настроить с помощью `openclaw configure --section web`. Документация:
[Веб-инструменты](/ru/tools/web).
</Tip>

## Пошаговый процесс по умолчанию

Обычный `openclaw onboard` выполняет следующие действия:

1. Примите уведомление о безопасности.
2. Обнаруживает настроенные модели, переменные окружения с ключами API, поддерживаемые локальные
   CLI для ИИ и уже установленные модели с поддержкой инструментов на доступных серверах Ollama или LM
   Studio на хосте Gateway. Эта проверка только для чтения никогда не загружает
   модель. Установки Gemini CLI и Antigravity отображаются, но не тестируются автоматически,
   поскольку для них невозможно принудительно выполнить проверку без инструментов.
3. Тестирует первого обнаруженного кандидата реальным ответом модели. При сбое отображает
   причину и переходит к следующему пригодному кандидату.
4. Если обнаруженные варианты исчерпаны, выберите OpenAI, Anthropic, xAI (Grok), Google или
   OpenRouter либо выберите **Ещё…** для просмотра остальных провайдеров. Регионы,
   тарифные планы и поддерживаемые методы через браузер, устройство, ключ API или токен для каждого провайдера
   отображаются во втором меню и проверяются тем же реальным ответом модели.
   Выберите **Пропустить пока**, чтобы выйти без запуска OpenClaw.
5. Сохраняет только проверенный маршрут модели и необходимые для него данные
   учётных данных и плагинов. Настройки рабочего пространства и Gateway остаются без изменений.
6. Запускает OpenClaw с проверенной моделью, чтобы он мог настроить рабочее пространство,
   Gateway, каналы, агентов, плагины и остальные дополнительные компоненты.

При повторном запуске команды в настроенной установке сначала тестируется текущая модель
по умолчанию, поэтому пошаговый процесс служит для проверки и исправления. Неудачная
проверка никогда не заменяет настроенную модель автоматически: первоначальная настройка останавливается и
предлагает выбрать дальнейшее действие. Запустите `openclaw channels add` или `openclaw configure` для
последующего добавления компонентов, не связанных с инференсом; для изменения маршрута провайдера или аутентификации
используйте `openclaw onboard`.

## Классический мастер: QuickStart и Advanced

Запустите `openclaw onboard --classic`, чтобы открыть полный мастер. Сначала он предлагает
выбрать между **QuickStart** (значения по умолчанию) и **Advanced** (полный контроль). Передайте
`--flow quickstart` или `--flow advanced` (псевдоним `manual`), чтобы выбрать классический
процесс и пропустить этот запрос.

<Tabs>
  <Tab title="QuickStart (значения по умолчанию)">
    - Локальный Gateway, привязка к loopback-интерфейсу
    - Рабочее пространство по умолчанию (или существующее рабочее пространство)
    - Порт Gateway: **18789**
    - Аутентификация Gateway: **Токен** (создаётся автоматически даже при привязке к loopback-интерфейсу)
    - Политика инструментов: `tools.profile: "coding"` для новых установок (существующий явно заданный профиль сохраняется)
    - Изоляция личных сообщений: `session.dmScope: "per-channel-peer"` для новых установок. Подробнее: [Справочник по настройке CLI](/ru/start/wizard-cli-reference#outputs-and-internals)
    - Доступ через Tailscale: **Выкл.**
    - Для личных сообщений Telegram и WhatsApp по умолчанию используется **список разрешённых**: Telegram запрашивает числовой идентификатор пользователя Telegram, а WhatsApp — номер телефона

  </Tab>
  <Tab title="Advanced (полный контроль)">
    - Предоставляет доступ ко всем этапам: режим, рабочее пространство, Gateway, каналы, демон, навыки

  </Tab>
</Tabs>

Удалённый режим (`--mode remote`) всегда использует расширенный процесс; он только
настраивает этот компьютер для подключения к Gateway в другом месте и никогда ничего не устанавливает
и не изменяет на удалённом хосте.

## Что настраивает классическая первоначальная настройка

В локальном режиме (по умолчанию) выполняются следующие этапы:

1. **Модель и аутентификация** — выберите процесс аутентификации провайдера (ключ API, OAuth или
   специфическую для провайдера ручную аутентификацию), включая пользовательского провайдера
   (совместимого с OpenAI, OpenAI Responses или Anthropic либо
   с автоматическим определением неизвестного типа). Выберите модель по умолчанию.
   При новой настройке ключа API OpenAI по умолчанию используется `openai/gpt-5.6` (идентификатор
   прямого API без уточнений разрешается в Sol); при новой настройке ChatGPT/Codex по умолчанию используется
   `openai/gpt-5.6-sol`. При повторном запуске настройки сохраняется существующая явно заданная модель,
   включая `openai/gpt-5.5`. Явно выберите `openai/gpt-5.5`, если
   учётная запись не предоставляет доступ к GPT-5.6.
   Примечание по безопасности: если этот агент будет запускать инструменты или обрабатывать содержимое
   вебхуков или хуков, предпочтительно использовать самую мощную доступную модель последнего поколения и сохранять
   строгую политику инструментов — более слабые или старые уровни легче подвергаются инъекциям в промпт.
   При неинтерактивном запуске `--secret-input-mode ref` сохраняет ссылки на переменные окружения
   вместо значений ключей API в открытом виде; указанная переменная окружения уже должна
   быть задана, иначе первоначальная настройка немедленно завершится с ошибкой. В интерактивном режиме ссылок на секреты
   можно указать переменную окружения или настроенную ссылку провайдера (`file` или
   `exec`); перед сохранением выполняется быстрая предварительная проверка. После настройки модели и аутентификации
   мастер предлагает необязательную проверку реальным ответом модели; при сбое можно один раз вернуться к
   настройке модели и аутентификации или проигнорировать ошибку, не блокируя остальные этапы
   классического мастера. Игнорирование не разблокирует OpenClaw; для настройки в диалоге
   по-прежнему требуется успешная проверка инференса.
2. **Рабочее пространство** — каталог для файлов агента (по умолчанию `~/.openclaw/workspace`). В нём создаются начальные файлы.
3. **Gateway** — порт, адрес привязки, режим аутентификации, доступ через Tailscale. В
   интерактивном режиме токена выберите хранение токена в открытом виде (по умолчанию) или
   использование SecretRef. Путь SecretRef для неинтерактивного режима: `--gateway-token-ref-env <ENV_VAR>`.
4. **Каналы** — встроенные каналы чатов и каналы официальных плагинов, включая
   Discord, Feishu, Google Chat, iMessage, Mattermost, Microsoft Teams,
   QQ Bot, Signal, Slack, Telegram, WhatsApp и другие.
5. **Демон** — устанавливает LaunchAgent (macOS), пользовательский модуль systemd
   (Linux/WSL2) или нативную запланированную задачу Windows с резервным
   вариантом в пользовательской папке автозагрузки.
   Если требуется аутентификация по токену и `gateway.auth.token` управляется через SecretRef,
   при установке демона ссылка проверяется, но разрешённый токен не сохраняется в
   метаданных окружения службы супервизора; неразрешённый SecretRef блокирует
   установку и выводит инструкции. Если одновременно заданы `gateway.auth.token` и
   `gateway.auth.password`, а `gateway.auth.mode` не задан, установка
   блокируется, пока режим не будет указан явно.
6. **Проверка работоспособности** — запускает Gateway и проверяет его доступность.
7. **Skills** — устанавливает рекомендуемые навыки и их необязательные зависимости.

<Note>
Повторный запуск первоначальной настройки **ничего не удаляет**, если явно не выбрать
**Сбросить** (или не передать `--reset`). В CLI `--reset` по умолчанию сбрасывает конфигурацию, учётные данные
и сеансы; используйте `--reset-scope full`, чтобы также удалить рабочее пространство. Если
конфигурация недействительна или содержит устаревшие ключи, первоначальная настройка предложит сначала запустить
`openclaw doctor`.
</Note>

`--flow import` запускает обнаруженный процесс миграции (например, Hermes) в
классическом мастере вместо новой настройки; см. [Миграция](/ru/cli/migrate) и руководства по миграции в разделе
[Установка](/ru/install/migrating-hermes). `openclaw onboard --modern` — это
псевдоним совместимости для [OpenClaw](/cli/openclaw). Он использует тот же
шлюз проверки инференса, что и `openclaw setup`: проверенный инференс запускает
ассистента, а сбой в интерактивном режиме возвращает к пошаговой настройке инференса.

## Добавление ещё одного агента

Используйте `openclaw agents add <name>`, чтобы создать отдельного агента с собственными
рабочим пространством, сеансами и профилями аутентификации. Запуск без `--workspace` начинает
интерактивный процесс настройки имени, рабочего пространства, аутентификации, каналов и привязок — это
не полный мастер `openclaw onboard`.

Настраиваемые параметры:

- `agents.list[].name`
- `agents.list[].workspace`
- `agents.list[].agentDir`

Примечания:

- Рабочее пространство по умолчанию: `~/.openclaw/workspace-<agentId>` (или внутри
  `agents.defaults.workspace`, если эта переменная задана).
- Добавьте `bindings`, чтобы направлять входящие сообщения этому агенту (первоначальная настройка может сделать это автоматически).
- Флаги неинтерактивного режима: `--model`, `--agent-dir`, `--bind`, `--non-interactive`.
