```bash
openclaw onboard
```

Онбординг через CLI — рекомендуемый способ настройки из терминала в macOS, Linux и
Windows (нативно или через WSL2). По умолчанию он обнаруживает уже доступный на
компьютере доступ к ИИ, проверяет его реальным запросом на генерацию и запускает Crestodian для
настройки рабочего пространства, Gateway и дополнительных функций. `openclaw setup` запускает тот же процесс ([Настройка](/ru/cli/setup) описывает
вариант `--baseline`, изменяющий только конфигурацию). Пользователи настольной версии Windows также могут начать
с [Windows Hub](/ru/platforms/windows).

Пошаговый онбординг сначала настраивает инференс. Он обнаруживает доступные способы доступа к ИИ,
требует успешного выполнения реального запроса на генерацию и только затем запускает [Crestodian](/ru/cli/crestodian)
для настройки остальных компонентов OpenClaw. В пошаговом процессе нельзя запустить Crestodian до
настройки инференса или пропустить настройку ИИ.

Классический мастер по-прежнему доступен для входа у провайдера, настройки удалённого Gateway,
подключения каналов, управления демоном, Skills и импорта. Запустите его явно
с помощью `openclaw onboard --classic`; экран выбора кандидата для инференса в пошаговом процессе
не перенаправляет в него. После успешной проверки инференса Crestodian может использовать `open channel
wizard for <channel>`, чтобы передать настройку каналов, требующую секретов, мастеру
в терминале с маскированным вводом. Чтобы изменить провайдера модели или способ аутентификации,
закройте Crestodian и запустите `openclaw onboard`; Crestodian не открывает пошаговые или
классические процессы настройки провайдера.

<Info>
Самый быстрый способ начать первый чат: завершите пошаговую настройку, запустите `openclaw dashboard` и общайтесь
в браузере через интерфейс управления. Документация: [Панель управления](/ru/web/dashboard).
</Info>

## Локаль

Мастер локализует неизменяемый текст онбординга. Порядок определения: `OPENCLAW_LOCALE`,
`LC_ALL`, `LC_MESSAGES`, `LANG`, затем английский язык. Поддерживаемые локали: `en`,
`zh-CN`, `zh-TW`.

```bash
OPENCLAW_LOCALE=zh-CN openclaw onboard
```

Названия продуктов, команды, ключи конфигурации, URL-адреса, идентификаторы провайдеров, идентификаторы моделей и
метки плагинов и каналов остаются на английском языке независимо от локали.

Чтобы позднее изменить настройки, не связанные с инференсом:

```bash
openclaw configure
openclaw agents add <name>
```

<Note>
`--json` не включает неинтерактивный режим автоматически. Для скриптов используйте `--non-interactive` (см. [Автоматизация CLI](/ru/start/wizard-cli-automation)).
</Note>

<Tip>
Классический мастер включает этап настройки веб-поиска, где можно выбрать провайдера: Brave,
DuckDuckGo, Exa, Firecrawl, Gemini, Grok, Kimi, MiniMax Search, Ollama Web
Search, Perplexity, SearXNG или Tavily. Для некоторых требуется ключ API, другие
работают без ключа. Чтобы настроить это позднее, используйте `openclaw configure --section web`. Документация:
[Веб-инструменты](/ru/tools/web).
</Tip>

## Пошаговый процесс по умолчанию

Обычная команда `openclaw onboard` выполняет следующие действия:

1. Принять уведомление о безопасности.
2. Обнаружить настроенные модели, переменные окружения с ключами API и поддерживаемые локальные
   CLI для ИИ.
3. Проверить первого обнаруженного кандидата реальным запросом на генерацию. При ошибке показать
   причину и перейти к следующему подходящему кандидату.
4. Если все обнаруженные варианты исчерпаны, повторить попытку с обнаруженным кандидатом или ввести
   ключ API провайдера в поле с маскированным вводом. Пошаговый онбординг
   не позволяет запустить Crestodian или завершить настройку без ИИ до успешной настройки инференса.
5. Сохранить только проверенный маршрут модели и необходимые ему данные учётных данных и плагинов.
   Настройки рабочего пространства и Gateway остаются без изменений.
6. Запустить Crestodian с проверенной моделью, чтобы он мог настроить рабочее пространство,
   Gateway, каналы, агентов, плагины и остальные дополнительные параметры.

При повторном запуске команды в настроенной установке сначала проверяется текущая модель
по умолчанию, поэтому пошаговый процесс также служит для проверки и исправления. При неудачной
проверке настроенная модель никогда не заменяется автоматически: онбординг останавливается и
предлагает выбрать дальнейшее действие. Для последующего добавления параметров, не связанных с инференсом,
запустите `openclaw channels add` или `openclaw configure`; для изменения провайдера или маршрута
аутентификации используйте `openclaw onboard`.

## Классический мастер: быстрый запуск и расширенная настройка

Запустите `openclaw onboard --classic`, чтобы открыть полный мастер. Сначала он предлагает
выбрать между **Быстрым запуском** (параметры по умолчанию) и **Расширенной настройкой** (полный контроль). Передайте
`--flow quickstart` или `--flow advanced` (псевдоним `manual`), чтобы выбрать классический
процесс и пропустить этот запрос.

<Tabs>
  <Tab title="Быстрый запуск (параметры по умолчанию)">
    - Локальный Gateway, привязка к loopback-интерфейсу
    - Рабочее пространство по умолчанию (или существующее рабочее пространство)
    - Порт Gateway: **18789**
    - Аутентификация Gateway: **Токен** (создаётся автоматически даже при привязке к loopback-интерфейсу)
    - Политика инструментов: `tools.profile: "coding"` для новых установок (существующий явно заданный профиль сохраняется)
    - Изоляция личных сообщений: `session.dmScope: "per-channel-peer"` для новых установок. Подробнее: [Справочник по настройке CLI](/ru/start/wizard-cli-reference#outputs-and-internals)
    - Доступ через Tailscale: **Отключён**
    - Для личных сообщений Telegram и WhatsApp по умолчанию используется **список разрешённых**: Telegram запрашивает числовой идентификатор пользователя Telegram, а WhatsApp — номер телефона

  </Tab>
  <Tab title="Расширенная настройка (полный контроль)">
    - Предоставляет доступ ко всем этапам: режим, рабочее пространство, Gateway, каналы, демон, Skills

  </Tab>
</Tabs>

Удалённый режим (`--mode remote`) всегда использует расширенный процесс; он только
настраивает этот компьютер для подключения к Gateway в другом месте и никогда ничего не устанавливает
и не изменяет на удалённом узле.

## Что настраивает классический онбординг

Локальный режим (по умолчанию) последовательно выполняет следующие этапы:

1. **Модель и аутентификация** — выбор способа аутентификации у провайдера (ключ API, OAuth или
   ручная аутентификация, зависящая от провайдера), включая собственного провайдера
   (совместимого с OpenAI, OpenAI Responses или Anthropic либо
   с автоматическим определением неизвестного типа). Выбор модели по умолчанию.
   При новой настройке с ключом API OpenAI по умолчанию используется `openai/gpt-5.6` (базовый идентификатор
   прямого API разрешается в Sol); при новой настройке ChatGPT/Codex по умолчанию используется
   `openai/gpt-5.6-sol`. При повторном запуске настройки существующая явно выбранная модель сохраняется,
   включая `openai/gpt-5.5`. Явно выберите `openai/gpt-5.5`, если
   учётная запись не предоставляет доступ к GPT-5.6.
   Примечание по безопасности: если этот агент будет запускать инструменты или обрабатывать содержимое
   Webhook/хуков, отдавайте предпочтение самой мощной доступной модели последнего поколения и сохраняйте
   строгую политику инструментов — более слабые или старые уровни легче подвергаются инъекциям в промпт.
   При неинтерактивном запуске `--secret-input-mode ref` сохраняет ссылки на переменные окружения
   вместо значений ключей API в открытом виде; соответствующая переменная окружения должна быть уже
   задана, иначе онбординг немедленно завершится с ошибкой. В интерактивном режиме ссылок на секреты
   можно указать переменную окружения или настроенную ссылку провайдера (`file` или
   `exec`); перед сохранением выполняется быстрая предварительная проверка. После настройки модели и аутентификации
   мастер предлагает необязательную проверку реальным запросом на генерацию; в случае ошибки можно один раз вернуться
   к настройке модели и аутентификации или проигнорировать ошибку, не блокируя остальные этапы
   классического мастера. Игнорирование ошибки не разблокирует Crestodian; для настройки
   в диалоговом режиме по-прежнему требуется успешная проверка инференса.
2. **Рабочее пространство** — каталог для файлов агента (по умолчанию `~/.openclaw/workspace`). Создаёт начальные файлы.
3. **Gateway** — порт, адрес привязки, режим аутентификации, доступ через Tailscale. В
   интерактивном режиме токена выберите хранение токена в открытом виде (по умолчанию) или
   использование SecretRef. Путь SecretRef для неинтерактивного режима: `--gateway-token-ref-env <ENV_VAR>`.
4. **Каналы** — встроенные каналы чата и каналы официальных плагинов, включая
   Discord, Feishu, Google Chat, iMessage, Mattermost, Microsoft Teams,
   QQ Bot, Signal, Slack, Telegram, WhatsApp и другие.
5. **Демон** — устанавливает LaunchAgent (macOS), пользовательский модуль systemd
   (Linux/WSL2) или нативную запланированную задачу Windows с резервным вариантом
   в пользовательской папке автозагрузки.
   Если требуется аутентификация по токену и `gateway.auth.token` управляется через SecretRef,
   установка демона проверяет его, но не сохраняет разрешённое значение токена в
   метаданных окружения службы супервизора; неразрешённый SecretRef блокирует
   установку и выводит рекомендации. Если одновременно заданы `gateway.auth.token` и
   `gateway.auth.password`, а `gateway.auth.mode` не задан, установка
   блокируется, пока режим не будет указан явно.
6. **Проверка работоспособности** — запускает Gateway и проверяет его доступность.
7. **Skills** — устанавливает рекомендуемые навыки и их необязательные зависимости.

<Note>
Повторный запуск онбординга **не** удаляет данные, если только вы явно не выберете
**Сброс** (или не передадите `--reset`). Команда CLI `--reset` по умолчанию удаляет конфигурацию, учётные данные
и сеансы; используйте `--reset-scope full`, чтобы также удалить рабочее пространство. Если
конфигурация недействительна или содержит устаревшие ключи, онбординг предложит сначала запустить
`openclaw doctor`.
</Note>

`--flow import` запускает обнаруженный процесс миграции (например, Hermes) в
классическом мастере вместо новой настройки; см. [Миграция](/ru/cli/migrate) и руководства по миграции в разделе
[Установка](/ru/install/migrating-hermes). `openclaw onboard --modern` — это
псевдоним для совместимости с [Crestodian](/ru/cli/crestodian). Он использует тот же
шлюз проверки инференса, что и `openclaw crestodian`: успешная проверка инференса запускает
помощника, а ошибка в интерактивном режиме возвращает к пошаговой настройке инференса.

## Добавление другого агента

Используйте `openclaw agents add <name>`, чтобы создать отдельного агента с собственными
рабочим пространством, сеансами и профилями аутентификации. Запуск без `--workspace` открывает
интерактивный процесс настройки имени, рабочего пространства, аутентификации, каналов и привязок — это
не полный мастер `openclaw onboard`.

Что он настраивает:

- `agents.list[].name`
- `agents.list[].workspace`
- `agents.list[].agentDir`

Примечания:

- Рабочее пространство по умолчанию: `~/.openclaw/workspace-<agentId>` (или внутри
  `agents.defaults.workspace`, если эта переменная задана).
- Добавьте `bindings`, чтобы направлять входящие сообщения этому агенту (онбординг может сделать это автоматически).
- Флаги неинтерактивного режима: `--model`, `--agent-dir`, `--bind`, `--non-interactive`.
