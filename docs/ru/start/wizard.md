CLI-онбординг — **рекомендуемый** путь настройки OpenClaw из терминала на
macOS, Linux или Windows. Пользователи рабочего стола Windows также могут начать с
[Windows Hub](/ru/platforms/windows).
Он настраивает локальный Gateway или подключение к удаленному Gateway, а также каналы, Skills
и значения рабочей области по умолчанию в одном пошаговом сценарии.

```bash
openclaw onboard
```

QuickStart обычно занимает всего несколько минут, но полный онбординг может занять больше времени,
если вход к провайдеру, сопряжение канала, установка демона, сетевые загрузки,
Skills или необязательные plugins требуют дополнительной настройки. Мастер заранее показывает этот график,
а необязательные шаги можно пропустить и вернуться к ним позже с помощью
`openclaw configure`.

## Локаль

Мастер CLI локализует фиксированный текст онбординга. Он определяет локаль из
`OPENCLAW_LOCALE`, затем `LC_ALL`, затем `LC_MESSAGES`, затем `LANG`, и
возвращается к английскому. Поддерживаемые локали мастера: `en`, `zh-CN` и `zh-TW`.

```bash
OPENCLAW_LOCALE=zh-CN openclaw onboard
```

Имена и стабильные идентификаторы остаются буквальными: `OpenClaw`, `Gateway`, `Tailscale`,
команды, ключи конфигурации, URL, идентификаторы провайдеров, идентификаторы моделей и метки plugin/каналов
не переводятся.

<Info>
Самый быстрый первый чат: откройте Control UI (настройка канала не нужна). Выполните
`openclaw dashboard` и общайтесь в браузере. Документация: [Dashboard](/ru/web/dashboard).
</Info>

Чтобы перенастроить позже:

```bash
openclaw configure
openclaw agents add <name>
```

<Note>
`--json` не подразумевает неинтерактивный режим. Для скриптов используйте `--non-interactive`.
</Note>

<Tip>
CLI-онбординг включает шаг веб-поиска, где можно выбрать провайдера,
например Brave, DuckDuckGo, Exa, Firecrawl, Gemini, Grok, Kimi, MiniMax Search,
Ollama Web Search, Perplexity, SearXNG или Tavily. Некоторым провайдерам требуется
ключ API, а другие работают без ключа. Это также можно настроить позже с помощью
`openclaw configure --section web`. Документация: [веб-инструменты](/ru/tools/web).
</Tip>

## QuickStart и расширенный режим

Онбординг начинается с выбора **QuickStart** (значения по умолчанию) или **расширенного режима** (полный контроль).

<Tabs>
  <Tab title="QuickStart (значения по умолчанию)">
    - Локальный Gateway (local loopback)
    - Рабочая область по умолчанию (или существующая рабочая область)
    - Порт Gateway **18789**
    - Аутентификация Gateway **Token** (автоматически генерируется даже на local loopback)
    - Политика инструментов по умолчанию для новых локальных настроек: `tools.profile: "coding"` (существующий явно заданный профиль сохраняется)
    - Значение по умолчанию для изоляции личных сообщений: локальный онбординг записывает `session.dmScope: "per-channel-peer"`, если значение не задано. Подробнее: [справочник по настройке CLI](/ru/start/wizard-cli-reference#outputs-and-internals)
    - Доступ через Tailscale **выключен**
    - Личные сообщения Telegram + WhatsApp по умолчанию используют **allowlist** (вам будет предложено ввести номер телефона)

  </Tab>
  <Tab title="Расширенный режим (полный контроль)">
    - Открывает каждый шаг (режим, рабочая область, Gateway, каналы, демон, Skills).

  </Tab>
</Tabs>

## Что настраивает онбординг

**Локальный режим (по умолчанию)** проводит вас через следующие шаги:

1. **Модель/аутентификация** — выберите любой поддерживаемый провайдер/сценарий аутентификации (ключ API, OAuth или ручная аутентификация, специфичная для провайдера), включая Custom Provider
   (совместимый с OpenAI, совместимый с Anthropic или Unknown с автоопределением). Выберите модель по умолчанию.
   Примечание по безопасности: если этот агент будет запускать инструменты или обрабатывать содержимое webhook/hooks, предпочитайте самую сильную доступную модель последнего поколения и держите политику инструментов строгой. Более слабые/старые уровни легче подвергаются prompt injection.
   Для неинтерактивных запусков `--secret-input-mode ref` сохраняет ссылки на значения из env в профилях аутентификации вместо открытых значений ключей API.
   В неинтерактивном режиме `ref` переменная окружения провайдера должна быть задана; передача флагов с ключом inline без этой переменной окружения быстро завершается ошибкой.
   В интерактивных запусках выбор режима ссылки на секрет позволяет указать либо переменную окружения, либо настроенную ссылку провайдера (`file` или `exec`) с быстрой предварительной проверкой перед сохранением.
   Для Anthropic интерактивный онбординг/настройка предлагает **Anthropic Claude CLI** как предпочтительный локальный путь и **ключ Anthropic API** как рекомендуемый путь для production. Anthropic setup-token также остается доступным как поддерживаемый путь аутентификации по токену.
2. **Рабочая область** — расположение файлов агента (по умолчанию `~/.openclaw/workspace`). Создает начальные bootstrap-файлы.
3. **Gateway** — порт, адрес привязки, режим аутентификации, доступ через Tailscale.
   В интерактивном режиме токена выберите хранение токена открытым текстом по умолчанию или включите SecretRef.
   Неинтерактивный путь SecretRef для токена: `--gateway-token-ref-env <ENV_VAR>`.
4. **Каналы** — встроенные и официальные plugin-каналы чата, такие как iMessage, Discord, Feishu, Google Chat, Mattermost, Microsoft Teams, QQ Bot, Signal, Slack, Telegram, WhatsApp и другие.
5. **Демон** — устанавливает LaunchAgent (macOS), пользовательский systemd unit (Linux/WSL2) или нативную Windows Scheduled Task с резервным вариантом через папку Startup для пользователя.
   Если аутентификация по токену требует токен и `gateway.auth.token` управляется SecretRef, установка демона проверяет его, но не сохраняет разрешенный токен в метаданных окружения сервиса supervisor.
   Если аутентификация по токену требует токен и настроенный SecretRef токена не разрешается, установка демона блокируется с практическими рекомендациями.
   Если настроены и `gateway.auth.token`, и `gateway.auth.password`, а `gateway.auth.mode` не задан, установка демона блокируется до явной настройки режима.
6. **Проверка работоспособности** — запускает Gateway и проверяет, что он работает.
7. **Skills** — устанавливает рекомендуемые Skills и необязательные зависимости.

<Note>
Повторный запуск онбординга **не** стирает ничего, если вы явно не выберете **Сброс** (или не передадите `--reset`).
CLI `--reset` по умолчанию сбрасывает конфигурацию, учетные данные и сессии; используйте `--reset-scope full`, чтобы включить рабочую область.
Если конфигурация недействительна или содержит устаревшие ключи, онбординг попросит сначала выполнить `openclaw doctor`.
</Note>

**Удаленный режим** только настраивает локальный клиент для подключения к Gateway в другом месте.
Он **не** устанавливает и не изменяет ничего на удаленном хосте.

## Добавить другого агента

Используйте `openclaw agents add <name>`, чтобы создать отдельного агента с собственной рабочей областью,
сессиями и профилями аутентификации. Запуск без `--workspace` запускает онбординг.

Что он задает:

- `agents.list[].name`
- `agents.list[].workspace`
- `agents.list[].agentDir`

Примечания:

- Рабочие области по умолчанию следуют шаблону `~/.openclaw/workspace-<agentId>`.
- Добавьте `bindings`, чтобы маршрутизировать входящие сообщения (онбординг может сделать это).
- Неинтерактивные флаги: `--model`, `--agent-dir`, `--bind`, `--non-interactive`.
