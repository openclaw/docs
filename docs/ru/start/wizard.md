```bash
openclaw onboard
```

Начальная настройка через CLI — рекомендуемый способ настройки из терминала в macOS, Linux и
Windows (нативно или через WSL2). По умолчанию она обнаруживает уже доступный на
компьютере доступ к ИИ, проверяет его реальным выполнением запроса и запускает Crestodian для
настройки рабочего пространства, Gateway и дополнительных функций. `openclaw setup` запускает тот же процесс ([Настройка](/ru/cli/setup) описывает
вариант только с конфигурацией `--baseline`). Пользователи настольной версии Windows также могут начать
с [Windows Hub](/ru/platforms/windows).

Пошаговая начальная настройка прежде всего настраивает инференс. Она обнаруживает доступные способы доступа к ИИ,
требует реального выполнения запроса и только после этого запускает [Crestodian](/ru/cli/crestodian)
для настройки остальной части OpenClaw. В пошаговом процессе нельзя запустить Crestodian до
настройки инференса или пропустить настройку ИИ.

Классический мастер по-прежнему доступен для входа у провайдера, настройки удалённого Gateway,
сопряжения каналов, управления демоном, Skills и импорта. Запустите его явно
командой `openclaw onboard --classic`; экран кандидатов для пошаговой настройки инференса
не перенаправляет в него. После успешной проверки инференса Crestodian может использовать `open channel
wizard for <channel>`, чтобы передать настройку канала, требующую секретов, терминальному
мастеру с маскировкой ввода. Чтобы изменить провайдера модели или способ аутентификации,
выйдите из Crestodian и выполните `openclaw onboard`; Crestodian не открывает пошаговые или
классические процессы настройки провайдера.

<Info>
Самый быстрый способ начать первый чат: завершите пошаговую настройку, выполните `openclaw dashboard` и общайтесь в
браузере через Control UI. Документация: [Панель управления](/ru/web/dashboard).
</Info>

## Локаль

Мастер локализует фиксированный текст начальной настройки. Порядок определения: `OPENCLAW_LOCALE`,
`LC_ALL`, `LC_MESSAGES`, `LANG`, затем английский язык. Поддерживаемые локали: `en`,
`zh-CN`, `zh-TW`.

```bash
OPENCLAW_LOCALE=zh-CN openclaw onboard
```

Названия продуктов, команды, ключи конфигурации, URL-адреса, идентификаторы провайдеров, идентификаторы моделей и
метки плагинов/каналов остаются на английском языке независимо от локали.

Чтобы позднее изменить настройки, не связанные с инференсом:

```bash
openclaw configure
openclaw agents add <name>
```

<Note>
`--json` не включает неинтерактивный режим. Для скриптов используйте `--non-interactive` (см. [Автоматизация CLI](/ru/start/wizard-cli-automation)).
</Note>

<Tip>
Классический мастер включает этап настройки веб-поиска, на котором можно выбрать провайдера: Brave,
DuckDuckGo, Exa, Firecrawl, Gemini, Grok, Kimi, MiniMax Search, Ollama Web
Search, Perplexity, SearXNG или Tavily. Некоторым требуется ключ API, другие
работают без ключа. Позднее это можно настроить командой `openclaw configure --section web`. Документация:
[Веб-инструменты](/ru/tools/web).
</Tip>

## Пошаговый процесс по умолчанию

Обычная команда `openclaw onboard` выполняет следующие действия:

1. Принять уведомление о безопасности.
2. Обнаружить настроенные модели, переменные окружения с ключами API и поддерживаемые локальные
   CLI для ИИ.
3. Проверить первого обнаруженного кандидата реальным выполнением запроса. При сбое показать
   причину и перейти к следующему доступному кандидату.
4. Если все обнаруженные варианты исчерпаны, повторно проверить обнаруженного кандидата или ввести ключ
   API провайдера в поле с маскировкой ввода. Пошаговая начальная настройка
   не предлагает запустить Crestodian или выйти с пропуском настройки ИИ, пока инференс не заработает.
5. Сохранить только проверенный маршрут модели и необходимое ему состояние учётных данных/плагина.
   Настройки рабочего пространства и Gateway остаются без изменений.
6. Запустить Crestodian с проверенной моделью, чтобы он мог настроить рабочее пространство,
   Gateway, каналы, агентов, плагины и остальные дополнительные параметры.

При повторном запуске команды в настроенной установке сначала проверяется текущая модель
по умолчанию, поэтому пошаговый процесс служит для проверки и исправления конфигурации. Неудачная
проверка никогда не заменяет настроенную модель автоматически; начальная настройка останавливается и
спрашивает, как продолжить. Для последующего добавления компонентов, не связанных с инференсом, выполните `openclaw channels add` или `openclaw configure`;
для изменения провайдера или маршрута аутентификации используйте `openclaw onboard`.

## Классический мастер: быстрый запуск и расширенный режим

Выполните `openclaw onboard --classic`, чтобы открыть полный мастер. Сначала он предлагает
выбрать между **быстрым запуском** (настройки по умолчанию) и **расширенным режимом** (полный контроль). Передайте
`--flow quickstart` или `--flow advanced` (псевдоним `manual`), чтобы выбрать классический
процесс и пропустить этот запрос.

<Tabs>
  <Tab title="Быстрый запуск (настройки по умолчанию)">
    - Локальный Gateway, привязка к local loopback
    - Рабочее пространство по умолчанию (или существующее рабочее пространство)
    - Порт Gateway **18789**
    - Аутентификация Gateway по **токену** (генерируется автоматически даже при local loopback)
    - Политика инструментов: `tools.profile: "coding"` для новых установок (существующий явно заданный профиль сохраняется)
    - Изоляция личных сообщений: `session.dmScope: "per-channel-peer"` для новых установок. Подробнее: [Справочник по настройке CLI](/ru/start/wizard-cli-reference#outputs-and-internals)
    - Доступ через Tailscale **выключен**
    - Для личных сообщений Telegram и WhatsApp по умолчанию используется **список разрешённых пользователей**: Telegram запрашивает числовой идентификатор пользователя Telegram, WhatsApp — номер телефона

  </Tab>
  <Tab title="Расширенный режим (полный контроль)">
    - Предоставляет доступ ко всем этапам: режиму, рабочему пространству, Gateway, каналам, демону и Skills

  </Tab>
</Tabs>

Удалённый режим (`--mode remote`) всегда использует расширенный процесс; он только
настраивает этот компьютер для подключения к Gateway на другом узле и никогда ничего
не устанавливает и не изменяет на удалённом узле.

## Что настраивает классическая начальная настройка

В локальном режиме (по умолчанию) выполняются следующие этапы:

1. **Модель/аутентификация** — выберите способ аутентификации у провайдера (ключ API, OAuth или
   ручная аутентификация, специфичная для провайдера), включая пользовательского провайдера
   (совместимого с OpenAI, OpenAI Responses или Anthropic либо с автоматическим определением
   неизвестного типа). Выберите модель по умолчанию.
   При новой настройке с ключом API OpenAI по умолчанию используется `openai/gpt-5.6` (прямой идентификатор
   API без дополнительных компонентов разрешается в Sol); при новой настройке ChatGPT/Codex по умолчанию используется
   `openai/gpt-5.6-sol`. При повторном запуске настройки существующая явно заданная модель сохраняется,
   включая `openai/gpt-5.5`. Явно выберите `openai/gpt-5.5`, если
   учётная запись не предоставляет доступ к GPT-5.6.
   Примечание по безопасности: если этот агент будет запускать инструменты или обрабатывать содержимое
   Webhook/хуков, предпочитайте наиболее мощную доступную модель последнего поколения и применяйте
   строгую политику инструментов — более слабые или старые уровни легче поддаются инъекциям в промпт.
   При неинтерактивном запуске `--secret-input-mode ref` сохраняет ссылки на переменные окружения
   вместо значений ключа API в открытом виде; указанная переменная окружения должна быть уже
   задана, иначе начальная настройка немедленно завершится ошибкой. Интерактивный режим ссылок на секреты позволяет
   указать переменную окружения или настроенную ссылку провайдера (`file` или
   `exec`) с быстрой предварительной проверкой перед сохранением. После настройки модели/аутентификации
   мастер предлагает необязательную проверку реальным выполнением запроса; при сбое можно один раз вернуться к
   настройке модели/аутентификации или проигнорировать ошибку, не блокируя остальные этапы
   классического мастера. Игнорирование не разблокирует Crestodian; для диалоговой настройки
   по-прежнему требуется успешная проверка инференса.
2. **Рабочее пространство** — каталог для файлов агента (по умолчанию `~/.openclaw/workspace`). Создаёт начальные файлы.
3. **Gateway** — порт, адрес привязки, режим аутентификации, доступ через Tailscale. В
   интерактивном режиме токена выберите хранение токена в открытом виде (по умолчанию) или
   использование SecretRef. Неинтерактивный путь SecretRef: `--gateway-token-ref-env <ENV_VAR>`.
4. **Каналы** — встроенные каналы чатов и каналы официальных плагинов, включая
   Discord, Feishu, Google Chat, iMessage, Mattermost, Microsoft Teams,
   QQ Bot, Signal, Slack, Telegram, WhatsApp и другие.
5. **Демон** — устанавливает LaunchAgent (macOS), пользовательский модуль systemd
   (Linux/WSL2) или нативную запланированную задачу Windows с резервным вариантом
   в папке автозагрузки пользователя.
   Если требуется аутентификация по токену и `gateway.auth.token` управляется через SecretRef,
   установка демона проверяет его, но не сохраняет разрешённый токен в
   метаданных окружения службы супервизора; неразрешённый SecretRef блокирует
   установку и выводит рекомендации. Если одновременно заданы `gateway.auth.token` и
   `gateway.auth.password`, а `gateway.auth.mode` не задан, установка
   блокируется, пока режим не будет указан явно.
6. **Проверка работоспособности** — запускает Gateway и проверяет его доступность.
7. **Skills** — устанавливает рекомендуемые Skills и их необязательные зависимости.

<Note>
Повторный запуск начальной настройки **не** удаляет данные, если только вы явно не выберете
**сброс** (или не передадите `--reset`). По умолчанию CLI `--reset` сбрасывает конфигурацию, учётные данные
и сеансы; используйте `--reset-scope full`, чтобы также удалить рабочее пространство. Если
конфигурация недействительна или содержит устаревшие ключи, начальная настройка предложит сначала выполнить
`openclaw doctor`.
</Note>

`--flow import` запускает обнаруженный процесс миграции (например, Hermes) в
классическом мастере вместо новой настройки; см. [Миграция](/ru/cli/migrate) и руководства по миграции в разделе
[Установка](/ru/install/migrating-hermes). `openclaw onboard --modern` — это
псевдоним для обратной совместимости с [Crestodian](/ru/cli/crestodian). Он использует тот же
барьер инференса, что и `openclaw crestodian`: при проверенном инференсе запускается
помощник, а при интерактивном сбое выполняется возврат к пошаговой настройке инференса.

## Добавление другого агента

Используйте `openclaw agents add <name>`, чтобы создать отдельного агента с собственным
рабочим пространством, сеансами и профилями аутентификации. Запуск без `--workspace` начинает
интерактивный процесс настройки имени, рабочего пространства, аутентификации, каналов и привязок — это
не полный мастер `openclaw onboard`.

Настраиваемые параметры:

- `agents.list[].name`
- `agents.list[].workspace`
- `agents.list[].agentDir`

Примечания:

- Рабочее пространство по умолчанию: `~/.openclaw/workspace-<agentId>` (или внутри
  `agents.defaults.workspace`, если этот параметр задан).
- Добавьте `bindings`, чтобы направлять входящие сообщения этому агенту (начальная настройка может сделать это за вас).
- Флаги неинтерактивного режима: `--model`, `--agent-dir`, `--bind`, `--non-interactive`.
