OpenClaw поддерживает первоначальную настройку через терминал и приложение macOS. Оба варианта сначала настраивают инференс:
они обнаруживают существующий доступ к ИИ, требуют успешного выполнения реального запроса и только затем запускают
Crestodian для завершения оставшейся настройки. Если настроенный Gateway доступен,
а для его агента по умолчанию уже настроена модель, первоначальная настройка пропускается
и открывается обычный интерфейс агента. Терминальный процесс также предлагает полный классический мастер
для подробной настройки.

## Какой вариант выбрать?

|                | Первоначальная настройка через CLI                 | Первоначальная настройка в приложении macOS |
| -------------- | -------------------------------------------------- | ------------------------------------------- |
| **Платформы**  | macOS, Linux, Windows (нативно или через WSL2)     | Только macOS                                |
| **Интерфейс**  | Настройка инференса, затем Crestodian              | Настройка инференса, затем Crestodian       |
| **Лучше всего подходит для** | Серверов, систем без графического интерфейса, полного контроля | Настольных компьютеров Mac, визуальной настройки |
| **Автоматизация** | `--non-interactive` для сценариев                | Только вручную                              |
| **Команда**    | `openclaw onboard`                                 | Запустить приложение                        |

Большинству пользователей следует начать с **первоначальной настройки через CLI** — она работает везде и предоставляет
максимальный контроль.

## Что настраивается при первоначальной настройке

Управляемый этап настройки инференса задаёт только следующее:

1. **Поставщик модели и аутентификация** — обнаруженный доступ или проверенный ключ API
2. **Проверенный инференс** — реальное выполнение запроса с фактически используемой
   моделью агента по умолчанию

После успешного выполнения запроса Crestodian может настроить рабочее пространство, Gateway,
службу Gateway, каналы, агентов, плагины и другие необязательные функции.

Классический мастер CLI дополнительно позволяет настроить:

1. **Каналы** (необязательно) — встроенные и поставляемые в комплекте каналы чатов, такие как
   Discord, Feishu, Google Chat, iMessage, Mattermost, Microsoft Teams,
   Telegram, WhatsApp и другие
2. **Расширенное управление Gateway** — удалённый режим, сетевые параметры и варианты демона

## Первоначальная настройка через CLI

Выполните в любом терминале:

```bash
openclaw onboard
```

Управляемый процесс обнаруживает существующий доступ к ИИ, последовательно проверяет варианты реальными запросами,
при сбое переходит к следующему и предлагает вручную ввести ключ с маскированием. Модель
и учётные данные сохраняются только после успешного выполнения запроса, после чего запускается Crestodian
для настройки рабочего пространства, Gateway, каналов, агентов, плагинов и других
необязательных функций. До настройки инференса Crestodian не запускается; также отсутствуют возможность пропустить настройку ИИ
и переход к классическому мастеру в рамках текущего процесса. Если вместо этого требуется
классический мастер, завершите процесс и выполните `openclaw onboard --classic`.

После успешной проверки инференса Crestodian может передать настройку каналов терминальному
мастеру с маскированием ввода. Он не открывает управляемую или классическую настройку поставщика; чтобы изменить
поставщика модели или способ аутентификации, завершите Crestodian и выполните
`openclaw onboard`.

Используйте `openclaw onboard --classic` для подробной настройки модели и аутентификации, каналов, навыков,
удалённого Gateway или импорта. Добавление `--install-daemon` также выбирает
классический процесс и устанавливает фоновую службу за один шаг. Используйте `openclaw
crestodian` для диалоговой настройки и восстановления, не связанных с инференсом. `openclaw
onboard --modern` — это псевдоним для совместимости, использующий ту же
проверку реального инференса.

Полная документация: [Первоначальная настройка (CLI)](/ru/start/wizard)
Документация по команде CLI: [`openclaw onboard`](/ru/cli/onboard)

## Первоначальная настройка в приложении macOS

Откройте приложение OpenClaw. Если настроенный локальный или удалённый Gateway доступен,
а для агента по умолчанию уже настроена модель, приложение пропускает первоначальную настройку
и Crestodian и немедленно открывает обычный интерфейс агента.

Для нового или не полностью настроенного Gateway процесс первого запуска обнаруживает существующий
доступ к ИИ (Claude Code, Codex или ключи API), проверяет лучший
вариант реальным запросом и сохраняет его только после получения настоящего ответа, автоматически переходя к резервным вариантам
и предлагая проверяемый этап ручного ввода ключа API, если ничего не найдено. Конфиденциальные
учётные данные вводятся с маскированием. После успешной проверки инференса запускается Crestodian
и помогает завершить настройку.

Gemini CLI остаётся доступным обычным агентам после настройки, но не
предлагается для этой проверки инференса, поскольку не позволяет гарантировать выполнение пробы без инструментов.

Полная документация: [Первоначальная настройка (приложение macOS)](/ru/start/onboarding)

## Пользовательские или отсутствующие в списке поставщики

Если вашего поставщика нет в списке, выполните `openclaw onboard --classic`, выберите
**Пользовательский поставщик** и укажите:

- Совместимость конечной точки: с OpenAI (`/chat/completions`), с OpenAI Responses (`/responses`), с Anthropic (`/messages`) или неизвестная (проверяются все три варианта с автоматическим определением)
- Базовый URL и ключ API (ключ API необязателен, если конечная точка его не требует)
- Идентификатор модели и необязательный псевдоним модели

Можно одновременно использовать несколько пользовательских конечных точек — каждая получает собственный идентификатор конечной точки.
