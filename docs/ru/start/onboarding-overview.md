OpenClaw поддерживает первоначальную настройку через терминал и приложение macOS. В обоих случаях сначала настраивается инференс:
обнаруживается существующий доступ к ИИ, требуется успешное выполнение реального запроса, и только после этого запускается
OpenClaw для настройки остальных компонентов. Если настроенный Gateway доступен
и для его агента по умолчанию уже настроена модель, первоначальная настройка пропускается и открывается
обычный интерфейс агента. Терминальный процесс также предлагает полный классический мастер для
детальной настройки.

## Какой вариант выбрать?

|                | Первоначальная настройка через CLI                         | Первоначальная настройка через приложение macOS           |
| -------------- | -------------------------------------- | ------------------------------ |
| **Платформы**  | macOS, Linux, Windows (нативно или через WSL2) | Только macOS                     |
| **Интерфейс**  | Настройка инференса, затем OpenClaw         | Настройка инференса, затем OpenClaw |
| **Лучше всего подходит для**   | Серверов, работы без графического интерфейса, полного контроля        | Настольных компьютеров Mac, визуальной настройки      |
| **Автоматизация** | `--non-interactive` для скриптов        | Только вручную                    |
| **Команда**    | `openclaw onboard`                     | Запустите приложение                 |

Большинству пользователей следует начать с **первоначальной настройки через CLI** — она работает везде и обеспечивает
максимальный контроль.

## Что настраивается при первоначальной настройке

На этапе управляемой настройки инференса задаются только:

1. **Провайдер модели и аутентификация** — обнаруженный доступ или проверенный вход в систему провайдера,
   API-ключ или токен
2. **Проверенный инференс** — реальное выполнение запроса на фактически используемой
   модели агента по умолчанию

После успешного выполнения запроса OpenClaw может настроить рабочее пространство, Gateway,
службу Gateway, каналы, агентов, плагины и другие необязательные функции.

Классический мастер CLI также позволяет настроить:

1. **Каналы** (необязательно) — встроенные и поставляемые в комплекте каналы чата, такие как
   Discord, Feishu, Google Chat, iMessage, Mattermost, Microsoft Teams,
   Telegram, WhatsApp и другие
2. **Расширенные параметры Gateway** — удалённый режим, сетевые настройки и варианты фоновой службы

## Первоначальная настройка через CLI

Выполните в любом терминале:

```bash
openclaw onboard
```

Управляемый процесс обнаруживает существующий доступ к ИИ, поочерёдно проверяет варианты реальными запросами
и при неудаче переходит к следующему. Если обнаружить доступ не удалось, сначала отображаются OpenAI,
Anthropic, xAI (Grok), Google и OpenRouter. В разделе **Ещё…** находятся
остальные провайдеры, объединённые в группы; во втором меню указаны регионы, тарифные планы и поддерживаемые
методы через браузер, устройство, API-ключ или токен. Модель
и учётные данные сохраняются только после успешного выполнения запроса, затем запускается OpenClaw для
настройки рабочего пространства, Gateway, каналов, агентов, плагинов и других необязательных
функций. **Пропустить пока** завершает процесс без запуска OpenClaw. Перейти к классическому мастеру
непосредственно из этого процесса нельзя; завершите его и выполните `openclaw onboard --classic`, если вместо него нужен
классический мастер.

После успешной проверки инференса OpenClaw может передать настройку каналов терминальному
мастеру с маскированным вводом. Он не открывает управляемую или классическую настройку провайдера; завершите работу OpenClaw и
выполните `openclaw onboard`, чтобы изменить провайдера модели или способ аутентификации.

Используйте `openclaw onboard --classic` для детальной настройки модели и аутентификации, каналов, Skills,
удалённого Gateway или импорта. Добавление `--install-daemon` также выбирает
классический процесс и устанавливает фоновую службу за один шаг. Используйте `openclaw
openclaw` для диалоговой настройки и исправления компонентов, не связанных с инференсом. `openclaw
onboard --modern` — это псевдоним для совместимости, использующий тот же этап
проверки инференса реальным запросом.

Полное описание: [Первоначальная настройка (CLI)](/ru/start/wizard)
Документация по команде CLI: [`openclaw onboard`](/ru/cli/onboard)

## Первоначальная настройка через приложение macOS

Откройте приложение OpenClaw. Если настроенный локальный или удалённый Gateway доступен
и для агента по умолчанию уже настроена модель, приложение пропускает первоначальную настройку
и OpenClaw и сразу открывает обычный интерфейс агента.

Для нового или не полностью настроенного Gateway процесс первого запуска обнаруживает существующий
доступ к ИИ (Claude Code, Codex или API-ключи), проверяет лучший
вариант реальным запросом и сохраняет его только после получения фактического ответа, автоматически переходя к резервным вариантам и
предлагая проверенный этап ручного ввода API-ключа, если ничего не найдено. Конфиденциальные
учётные данные вводятся в маскированном поле. После успешной проверки инференса запускается OpenClaw и
помогает настроить остальные компоненты.

После настройки Gemini CLI остаётся доступным для обычных агентов, но не
предлагается для этой проверки инференса, поскольку не позволяет гарантировать выполнение пробного запроса без инструментов.

Полное описание: [Первоначальная настройка (приложение macOS)](/ru/start/onboarding)

## Пользовательские провайдеры и провайдеры, отсутствующие в списке

Если вашего провайдера нет в списке, выполните `openclaw onboard --classic`, выберите
**Пользовательский провайдер** и укажите:

- Совместимость конечной точки: совместимая с OpenAI (`/chat/completions`), совместимая с OpenAI Responses (`/responses`), совместимая с Anthropic (`/messages`) или неизвестная (проверяются все три варианта и совместимость определяется автоматически)
- Базовый URL и API-ключ (API-ключ необязателен, если конечная точка его не требует)
- Идентификатор модели и необязательный псевдоним модели

Можно одновременно использовать несколько пользовательских конечных точек — каждой назначается собственный идентификатор конечной точки.
