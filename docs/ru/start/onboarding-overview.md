OpenClaw поддерживает первоначальную настройку через терминал и приложение macOS. Оба варианта сначала настраивают инференс:
они обнаруживают имеющийся доступ к ИИ, требуют успешного выполнения реального запроса и только после этого запускают
Crestodian для завершения оставшейся настройки. Если доступен настроенный Gateway,
у агента по умолчанию которого уже настроена модель, первоначальная настройка пропускается
и открывается обычный интерфейс агента. В терминале также доступен полный классический мастер
для детальной настройки.

## Какой вариант выбрать?

|                  | Первоначальная настройка через CLI                  | Первоначальная настройка в приложении macOS |
| ---------------- | --------------------------------------------------- | -------------------------------------------- |
| **Платформы**    | macOS, Linux, Windows (нативно или через WSL2)      | Только macOS                                 |
| **Интерфейс**    | Настройка инференса, затем Crestodian                | Настройка инференса, затем Crestodian         |
| **Лучше всего для** | Серверов, систем без графического интерфейса, полного контроля | Настольных компьютеров Mac, визуальной настройки |
| **Автоматизация** | `--non-interactive` для скриптов                    | Только вручную                               |
| **Команда**      | `openclaw onboard`                                  | Запуск приложения                            |

Большинству пользователей рекомендуется начать с **первоначальной настройки через CLI** — она работает
везде и предоставляет максимум контроля.

## Что настраивается при первоначальной настройке

Управляемый этап настройки инференса выполняет только следующее:

1. **Провайдер модели и аутентификация** — обнаруженный доступ или проверенный ключ API
2. **Проверенный инференс** — реальное выполнение запроса на фактически используемой
   модели агента по умолчанию

После успешного выполнения запроса Crestodian может настроить рабочее пространство, Gateway,
службу Gateway, каналы, агентов, плагины и другие необязательные функции.

Классический мастер CLI может дополнительно настроить:

1. **Каналы** (необязательно) — встроенные и поставляемые в комплекте каналы чатов, такие как
   Discord, Feishu, Google Chat, iMessage, Mattermost, Microsoft Teams,
   Telegram, WhatsApp и другие
2. **Расширенные параметры управления Gateway** — удалённый режим, сетевые настройки и варианты демона

## Первоначальная настройка через CLI

Запустите в любом терминале:

```bash
openclaw onboard
```

Управляемый процесс обнаруживает имеющийся доступ к ИИ, поочерёдно проверяет варианты реальными запросами,
при неудаче переходит к следующему и предлагает вручную ввести ключ в замаскированном поле. Модель
и учётные данные сохраняются только после успешного выполнения запроса, затем запускается Crestodian
для настройки рабочего пространства, Gateway, каналов, агентов, плагинов и других
необязательных функций. До настройки инференса Crestodian не запускается; также отсутствуют вариант
с пропуском ИИ и переход к классическому мастеру в рамках текущего процесса. Если вам нужен
классический мастер, завершите процесс и выполните `openclaw onboard --classic`.

После успешной проверки инференса Crestodian может передать настройку каналов терминальному мастеру
с маскированным вводом. Он не открывает управляемую или классическую настройку провайдера; чтобы изменить
провайдера модели или его параметры аутентификации, завершите Crestodian и выполните
`openclaw onboard`.

Используйте `openclaw onboard --classic` для детальной настройки модели и аутентификации, каналов, Skills,
удалённого Gateway или импорта. Добавление `--install-daemon` также выбирает
классический процесс и устанавливает фоновую службу за один шаг. Используйте `openclaw
crestodian` для диалоговой настройки и исправления параметров, не связанных с инференсом. `openclaw
onboard --modern` — это псевдоним для совместимости, использующий тот же
этап проверки реального инференса.

Полное описание: [Первоначальная настройка (CLI)](/ru/start/wizard)
Документация по команде CLI: [`openclaw onboard`](/ru/cli/onboard)

## Первоначальная настройка в приложении macOS

Откройте приложение OpenClaw. Если настроенный в нём локальный или удалённый Gateway доступен,
а у агента по умолчанию уже настроена модель, приложение пропускает первоначальную настройку
и Crestodian и сразу открывает обычный интерфейс агента.

Для нового или не полностью настроенного Gateway процесс первого запуска обнаруживает имеющийся доступ
к ИИ (Claude Code, Codex или ключи API), проверяет лучший
вариант реальным запросом и сохраняет его только после получения настоящего ответа — при неудаче
автоматически переходя к следующему варианту и предлагая проверенный ручной ввод ключа API,
если ничего не найдено. Конфиденциальные учётные данные вводятся в замаскированные поля.
После успешной проверки инференса запускается Crestodian, который помогает завершить настройку.

Gemini CLI остаётся доступным обычным агентам после настройки, но не предлагается
на этом этапе проверки инференса, поскольку не позволяет гарантировать выполнение проверки без инструментов.

Полное описание: [Первоначальная настройка (приложение macOS)](/ru/start/onboarding)

## Пользовательские или отсутствующие в списке провайдеры

Если вашего провайдера нет в списке, выполните `openclaw onboard --classic`, выберите
**Пользовательский провайдер** и укажите:

- Совместимость конечной точки: с OpenAI (`/chat/completions`), с OpenAI Responses (`/responses`), с Anthropic (`/messages`) или неизвестная (проверяются все три варианта и выполняется автоматическое определение)
- Базовый URL и ключ API (ключ API необязателен, если конечная точка его не требует)
- Идентификатор модели и необязательный псевдоним модели

Несколько пользовательских конечных точек могут использоваться одновременно — каждая получает собственный идентификатор конечной точки.
