Приложение для macOS — это **помощник OpenClaw в строке меню**. Используйте его, когда вам нужен
нативный интерфейс в трее, запросы разрешений macOS, уведомления, WebChat, голосовой ввод,
Canvas или размещенные на Mac инструменты узла, такие как `system.run`.

Если вам нужны только CLI и Gateway, начните с раздела [Начало работы](/ru/start/getting-started).

## Скачать

Скачивайте сборки приложения для macOS из
[релизов OpenClaw на GitHub](https://github.com/openclaw/openclaw/releases).
Когда релиз включает ресурсы приложения для macOS, ищите:

- `OpenClaw-<version>.dmg` (предпочтительно)
- `OpenClaw-<version>.zip`

Некоторые релизы включают только CLI, доказательства или ресурсы Windows. Если в новейшем
релизе нет ресурса приложения для macOS, используйте новейший релиз, в котором он есть, или соберите
приложение из исходного кода с помощью [настройки среды разработки для macOS](/ru/platforms/mac/dev-setup).

## Первый запуск

1. Установите и запустите **OpenClaw.app**.
2. Выберите **Этот Mac** для локального Gateway или подключитесь к удаленному Gateway.
3. В локальном режиме подождите, пока приложение установит свою пользовательскую среду выполнения и Gateway.
4. Завершите настройку провайдера и контрольный список разрешений macOS.
5. Отправьте тестовое сообщение для первичной настройки.

Для пути настройки CLI/Gateway используйте [Начало работы](/ru/start/getting-started).
Для восстановления разрешений используйте [Разрешения macOS](/ru/platforms/mac/permissions).

## Выберите режим Gateway

| Режим     | Когда использовать                                                                                       | Страница с подробностями                              |
| --------- | --------------------------------------------------------------------------------------------------------- | ----------------------------------------------------- |
| Локальный | Этот Mac должен запускать Gateway и поддерживать его работу через launchd.                                | [Gateway в macOS](/ru/platforms/mac/bundled-gateway)     |
| Удаленный | Другой хост запускает Gateway, а этот Mac должен управлять им через SSH, LAN или Tailnet.                 | [Удаленное управление](/ru/platforms/mac/remote)         |

Для локального режима требуется установленный CLI `openclaw`. На новом Mac приложение устанавливает
соответствующий CLI и среду выполнения автоматически перед запуском мастера Gateway.
См. [Gateway в macOS](/ru/platforms/mac/bundled-gateway) для ручного восстановления.

## За что отвечает приложение

- Статус в строке меню, уведомления, состояние работоспособности и WebChat.
- Запросы разрешений macOS для экрана, микрофона, речи, автоматизации и универсального доступа.
- Локальные инструменты узла, такие как Canvas, захват камеры/экрана, уведомления и `system.run`.
- Запросы подтверждения Exec для команд, размещенных на Mac.
- SSH-туннели в удаленном режиме или прямые подключения к Gateway.

Приложение **не** заменяет OpenClaw Gateway или общую документацию CLI. Основная
конфигурация Gateway, провайдеры, plugins, каналы, инструменты и безопасность описаны
в собственных разделах документации.
