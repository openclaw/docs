Приложение macOS — это **компаньон OpenClaw в строке меню**: нативный интерфейс в области уведомлений, запросы разрешений macOS, уведомления, WebChat, голосовой ввод, Canvas и
инструменты Node, размещённые на Mac, например `system.run`.

Нужны только CLI и Gateway? Начните с раздела [Начало работы](/ru/start/getting-started).

## Загрузка

Сборки приложения macOS доступны в [релизах OpenClaw на GitHub](https://github.com/openclaw/openclaw/releases).
Если релиз содержит ресурсы приложения macOS, ищите:

- `OpenClaw-<version>.dmg` (предпочтительно)
- `OpenClaw-<version>.zip`

Некоторые релизы содержат только CLI, материалы подтверждения или ресурсы для Windows. Если в новейшем релизе
нет ресурса приложения macOS, используйте самый новый релиз, в котором он есть, либо соберите приложение из исходного кода с помощью
[настройки среды разработки для macOS](/ru/platforms/mac/dev-setup).

## Первый запуск

1. Установите и запустите **OpenClaw.app**.
2. Выберите **Этот Mac** для локального Gateway или подключитесь к удалённому Gateway.
3. Подождите, пока приложение установит соответствующую среду выполнения CLI. В локальном режиме оно также
   устанавливает и запускает Gateway.
4. Установите подключение к модели посредством проверки активной модели. После успешной проверки Crestodian
   выполнит оставшуюся настройку.
5. Выполните контрольный список разрешений macOS и отправьте тестовое сообщение в рамках первоначальной настройки.

Если приложение подключается к существующему Gateway, у агента по умолчанию которого настроена
модель, оно считает этот Gateway уже настроенным, пропускает первоначальную настройку провайдера и
Crestodian и открывает панель управления. Если подключиться к Gateway не удаётся или у его
агента по умолчанию нет модели, первоначальная настройка подключения к модели остаётся доступной для
восстановления.

Для настройки через CLI/Gateway используйте раздел [Начало работы](/ru/start/getting-started).
Для восстановления разрешений используйте раздел [Разрешения macOS](/ru/platforms/mac/permissions).

## Обновления

Карточка обновления на панели управления сначала обновляет подписанное приложение macOS через Sparkle.
После перезапуска приложения оно автоматически обновляет и перезапускает соответствующий
управляемый приложением локальный Gateway. Установки CLI через Homebrew и другие установки, управляемые пользователем, сохраняют
обычный процесс обновления Gateway (карточка запускает обновление Gateway напрямую),
а автоматическое восстановление никогда не понижает версию более нового Gateway и не переопределяет
закрепление канала `extended-stable`.

Sparkle использует настройку `update.channel` Gateway. Значения `beta` и `dev` включают
бета-сборки приложения; значения `stable`, `extended-stable`, а также отсутствующие или неизвестные значения
оставляют приложение на стабильных сборках.

## Открытие ссылок панели управления

На встроенной панели управления приложения macOS щелчок по внешней веб-ссылке открывает её в боковой панели браузера с изменяемым размером. Каждая ссылка открывается в отдельной вкладке; повторный щелчок по той же ссылке использует уже существующую вкладку. Перетаскивайте вкладки, чтобы изменить их порядок, закрывайте их кнопкой закрытия вкладки или щелчком средней кнопки мыши, а щелчком правой кнопки мыши по вкладке открывайте команды **Открыть в браузере по умолчанию**, **Копировать ссылку**, **Перезагрузить**, **Закрыть вкладку** и **Закрыть другие вкладки**. Элементы управления переходом назад и вперёд в строке заголовка окна и жесты трекпада позволяют перемещаться по истории панели управления; собственные элементы управления переходом назад и вперёд на боковой панели позволяют перемещаться по истории активной вкладки. На боковой панели также есть элементы управления перезагрузкой, открытием в браузере по умолчанию и закрытием; её ширина запоминается.

Элементы управления строки заголовка следуют за боковой панелью приложения: когда она развёрнута, кнопки перехода назад и вперёд находятся у её правого края рядом с переключателем боковой панели; когда она свёрнута, их место занимают кнопка поиска (открывает палитру команд) и кнопка нового сеанса.

Щёлкните внешнюю ссылку правой кнопкой мыши, чтобы выбрать **Открыть на боковой панели**, **Открыть в браузере по умолчанию** или **Копировать ссылку**. Щелчки с клавишами-модификаторами и инициированные пользователем ссылки, открывающие новое окно с панели управления, по-прежнему открываются в браузере по умолчанию; ссылки, открывающие новое окно внутри боковой панели, открываются как новые вкладки боковой панели. Обычные страницы Control UI, размещённые в браузере, сохраняют стандартное поведение браузера для ссылок и контекстных меню.

## Импорт входов в браузер

Когда боковая панель браузера впервые открывается при работе приложения с локальным Gateway, панель управления показывает закрываемый баннер, если на Mac существует профиль браузера семейства Chrome с файлами cookie. Баннер предлагает скопировать эти файлы cookie в изолированный управляемый профиль, который агенты используют для просмотра веб-страниц. Выберите профиль с помощью элемента **Импорт** (может потребоваться Touch ID); ход выполнения и количество импортированных файлов cookie отображаются непосредственно на странице, при этом копируются только файлы cookie — пароли никогда не покидают исходный браузер. Закрытие баннера сохраняет этот выбор; пункт **Настройки → Основные → Вход в браузер → Импортировать…** позволяет повторно открыть его в любое время. Базовый процесс импорта и ограничение `browser.allowSystemProfileImport` описаны в разделе [Браузер](/ru/cli/browser).

## Выбор режима Gateway

| Режим     | Когда использовать                                                              | Страница с подробностями                            |
| --------- | ------------------------------------------------------------------------------- | -------------------------------------------------- |
| Локальный | Этот Mac должен запускать Gateway и поддерживать его работу с помощью launchd.  | [Gateway в macOS](/ru/platforms/mac/bundled-gateway)  |
| Удалённый | Gateway работает на другом узле; этот Mac управляет им через SSH, LAN или Tailnet. | [Удалённое управление](/ru/platforms/mac/remote)     |

Для обоих режимов требуется установленный CLI `openclaw`, поскольку приложение повторно использует его среду выполнения
узла-хоста. На новом Mac приложение автоматически устанавливает соответствующий CLI; затем локальный
режим запускает мастер Gateway, а удалённый режим подключается к выбранному
Gateway, не запуская второй локальный Gateway.
Инструкции по восстановлению вручную см. в разделе [Gateway в macOS](/ru/platforms/mac/bundled-gateway).

## За что отвечает приложение

- Состояние в строке меню, уведомления, работоспособность и WebChat.
- Запросы разрешений macOS для экрана, микрофона, распознавания речи, автоматизации и универсального доступа.
- Один узел Mac, объединяющий нативный Canvas, захват изображения с камеры и экрана, уведомления,
  местоположение и управление компьютером с командами системы, браузера,
  плагинов, Skills и MCP из узла-хоста CLI.
- Запросы подтверждения выполнения команд, размещённых на Mac.
- Выполнение одобренных команд оболочки в контексте приложения с сохранением привязки разрешений macOS
  к приложению, при этом общая политика узла остаётся под управлением среды выполнения CLI.
- SSH-туннели в удалённом режиме или прямые подключения к Gateway.

Приложение **не** заменяет документацию по Gateway или общую документацию CLI. Конфигурация
Gateway, провайдеры, плагины, каналы, инструменты и безопасность описаны в отдельных
разделах документации.
