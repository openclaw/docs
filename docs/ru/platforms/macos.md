Приложение для macOS — это **помощник OpenClaw в строке меню**: нативный интерфейс в области уведомлений, запросы разрешений macOS, уведомления, WebChat, голосовой ввод, Canvas и
инструменты узла на Mac, такие как `system.run`.

Нужны только CLI и Gateway? Начните с раздела [Начало работы](/ru/start/getting-started).

## Загрузка

Сборки приложения для macOS доступны в [выпусках OpenClaw на GitHub](https://github.com/openclaw/openclaw/releases).
Если выпуск содержит ресурсы приложения для macOS, ищите:

- `OpenClaw-<version>.dmg` (предпочтительно)
- `OpenClaw-<version>.zip`

Некоторые выпуски содержат только CLI, материалы подтверждения или ресурсы для Windows. Если в новейшем выпуске
нет ресурса приложения для macOS, используйте самый новый выпуск, в котором он есть, либо соберите приложение из исходного кода, следуя
[инструкциям по настройке среды разработки для macOS](/ru/platforms/mac/dev-setup).

## Первый запуск

1. Установите и запустите **OpenClaw.app**.
2. Выберите **This Mac** для локального Gateway или подключитесь к удалённому Gateway.
3. Дождитесь, пока приложение установит соответствующую среду выполнения CLI. В локальном режиме оно также
   устанавливает и запускает Gateway.
4. Настройте инференс с помощью проверки работающей модели. После успешной проверки OpenClaw
   выполнит оставшуюся настройку.
5. Завершите проверку разрешений macOS и отправьте тестовое сообщение начальной настройки.

Если приложение подключается к существующему Gateway, у агента по умолчанию которого настроена
модель, оно считает этот Gateway уже настроенным, пропускает начальную настройку провайдера и
OpenClaw и открывает панель управления. Если подключиться к Gateway не удаётся или у его
агента по умолчанию нет модели, начальная настройка инференса остаётся доступной для
восстановления.

Для настройки через CLI/Gateway используйте раздел [Начало работы](/ru/start/getting-started).
Для восстановления разрешений используйте раздел [Разрешения macOS](/ru/platforms/mac/permissions).

## Обновления

Карточка обновления на панели управления указывает, что именно обновит приложение:

- **Update Mac app + Gateway** означает, что подписанное приложение управляет локальным Gateway через launchd.
  Sparkle сначала обновляет приложение; после повторного запуска приложение автоматически
  обновляет и перезапускает свой Gateway до соответствующей версии, а затем проверяет
  подключение.
- **Update Gateway** означает, что приложение подключено к удалённому Gateway, локальному Gateway
  с ручным управлением или другой установке, которой приложение не управляет. Кнопка
  запускает обычный процесс обновления этого Gateway, не изменяя приложение для Mac.

При сбое согласованного обновления приложение остаётся в окне, похожем на окно настройки, где доступны повторная попытка,
[руководство по обновлению](/ru/install/updating) и действия Discord. Автоматическое восстановление никогда не
понижает версию более нового Gateway и не переопределяет закрепление канала `extended-stable`.

После успешного обновления приложение находит последний использованный человеком
прямой сеанс верхнего уровня и однократно отправляет этому агенту событие обновления. Активность Heartbeat
и Cron не влияет на этот выбор. Затем агент может поприветствовать вас
в беседе, которой вы, вероятнее всего, пользовались. В удалённом режиме приложение
обновляет только локальную среду выполнения узла Mac и не отправляет уведомление, если
удалённый Gateway старше приложения.

Sparkle следует настройке `update.channel` Gateway. `beta` и `dev` включают
бета-сборки приложения; `stable`, `extended-stable`, а также отсутствующие или неизвестные значения
оставляют приложение на стабильных сборках.

## Открытие ссылок панели управления

Если во встроенной панели управления приложения для macOS нажать внешнюю веб-ссылку, она откроется в боковой панели браузера с изменяемым размером, занимающей половину ширины окна, при этом навигация панели управления останется видимой. Перетащите разделитель, чтобы выбрать другую ширину; приложение запомнит её. Каждая ссылка открывается в отдельной вкладке, панель вкладок появляется при открытии нескольких страниц, а повторное нажатие той же ссылки использует уже существующую вкладку. Перетаскивайте вкладки, чтобы менять их порядок, закрывайте их кнопкой закрытия вкладки или нажатием средней кнопки мыши, а правой кнопкой мыши открывайте меню вкладки с пунктами **Open in Default Browser**, **Copy Link**, **Reload**, **Close Tab** и **Close Other Tabs**. Кнопки «назад» и «вперёд» в строке заголовка окна и жесты трекпада позволяют перемещаться по истории панели управления; собственные кнопки «назад» и «вперёд» боковой панели позволяют перемещаться по истории активной вкладки. В боковой панели также есть элементы управления перезагрузкой, открытием в браузере по умолчанию и закрытием.

Элементы строки заголовка следуют за боковой панелью приложения: когда она развёрнута, кнопки «назад» и «вперёд» располагаются у её правого края рядом с переключателем боковой панели; когда она свёрнута, их место занимают кнопка поиска (открывает палитру команд) и кнопка нового сеанса.

Нажмите внешнюю ссылку правой кнопкой мыши, чтобы выбрать **Open in Sidebar**, **Open in Default Browser** или **Copy Link**. Щелчки с клавишами-модификаторами и активированные пользователем ссылки для открытия нового окна с панели управления по-прежнему открываются в браузере по умолчанию; ссылки для открытия нового окна внутри боковой панели открываются в новых вкладках боковой панели. Обычные страницы Control UI, открытые в браузере, сохраняют стандартное поведение браузера для ссылок и контекстного меню.

## Импорт данных входа из браузера

При первом открытии боковой панели браузера, когда приложение работает с локальным Gateway, панель управления показывает закрываемый баннер, если на Mac существует профиль браузера семейства Chrome с файлами cookie. Баннер предлагает скопировать эти файлы cookie в изолированный управляемый профиль, который агенты используют для работы в браузере. Выберите профиль с помощью элемента **Import** (может потребоваться Touch ID); ход выполнения и количество импортированных файлов cookie отображаются непосредственно в баннере, при этом копируются только файлы cookie — пароли никогда не покидают исходный браузер. Закрытие баннера сохраняет этот выбор; пункт **Settings → General → Browser login → Import…** позволяет повторно выполнить импорт в любое время. Описание базового процесса импорта и ограничения `browser.allowSystemProfileImport` см. в разделе [Браузер](/ru/cli/browser).

## Выбор режима Gateway

| Режим   | Когда использовать                                                             | Подробная страница                                  |
| ------- | ------------------------------------------------------------------------------ | -------------------------------------------------- |
| Локальный  | Этот Mac должен запускать Gateway и поддерживать его работу с помощью launchd. | [Gateway в macOS](/ru/platforms/mac/bundled-gateway) |
| Удалённый | Gateway работает на другом хосте; этот Mac управляет им через SSH, LAN или Tailnet. | [Удалённое управление](/ru/platforms/mac/remote)            |

Для обоих режимов требуется установленный CLI `openclaw`, поскольку приложение повторно использует его среду выполнения
хоста узла. На новом Mac приложение автоматически устанавливает соответствующий CLI; затем в локальном
режиме запускается мастер Gateway, а в удалённом режиме устанавливается подключение к выбранному
Gateway без запуска второго локального Gateway.
Инструкции по ручному восстановлению см. в разделе [Gateway в macOS](/ru/platforms/mac/bundled-gateway).

## Чем управляет приложение

- Состояние в строке меню, уведомления, работоспособность и WebChat.
- Запросы разрешений macOS для экрана, микрофона, распознавания речи, автоматизации и универсального доступа.
- Один узел Mac, объединяющий нативный Canvas, захват изображения с камеры и экрана, уведомления,
  геолокацию и управление компьютером с командами системы, браузера,
  плагинов, Skills и MCP от хоста узла CLI.
- Запросы подтверждения выполнения команд, размещённых на Mac.
- Выполнение одобренных команд оболочки в контексте приложения, сохраняющее привязку
  разрешений macOS к приложению, в то время как среда выполнения CLI управляет общей политикой узла.
- SSH-туннели удалённого режима или прямые подключения к Gateway.

Приложение **не** заменяет документацию Gateway или общую документацию CLI. Настройка Gateway,
провайдеры, плагины, каналы, инструменты и безопасность описаны в отдельных
разделах документации.
