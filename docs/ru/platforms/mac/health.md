# Проверка работоспособности в macOS

Как просматривать состояние работоспособности подключённого канала в приложении в строке меню.

## Строка меню

Индикатор состояния:

- Зелёный: канал подключён, проверка пройдена успешно.
- Оранжевый: канал подключён, но проверка сообщает об ухудшении состояния или отсутствии соединения.
- Красный: канал ещё не подключён.

Во второй строке отображается «подключено · авторизация 12 мин назад» или причина сбоя.
Пункт меню «Запустить проверку работоспособности» запускает проверку по запросу.

## Настройки

- На вкладке «Общие» отображается карточка состояния: индикатор, строка сводки (состояние подключения +
  возраст авторизации) и необязательная строка с подробностями сбоя, а также кнопки **Повторить сейчас** и
  **Открыть журналы**.
- На **вкладке «Каналы»** отображаются состояние и элементы управления для каждого канала (QR-код для входа,
  выход, проверка, последнее отключение или ошибка) для WhatsApp и Telegram.

## Как работает проверка

Приложение вызывает RPC Gateway `health` через существующее соединение WebSocket
(без запуска CLI в оболочке) примерно каждые 60 с и по запросу. RPC загружает
учётные данные и сообщает состояние, не отправляя сообщений. Приложение отдельно кэширует последний
успешный снимок состояния и последнюю ошибку, чтобы интерфейс загружался мгновенно и
не мерцал при отсутствии подключения.

## Если есть сомнения

Используйте последовательность действий CLI из раздела [Работоспособность Gateway](/ru/gateway/health) (`openclaw status`,
`openclaw status --deep`, `openclaw health --json`) и отслеживайте
`/tmp/openclaw/openclaw-*.log`, фильтруя по `web-heartbeat` / `web-reconnect`.
