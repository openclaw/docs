# Проверки состояния на macOS

Как просматривать состояние связанных каналов в приложении в строке меню.

## Строка меню

Индикатор состояния:

- Зелёный: канал связан, проверка прошла успешно.
- Оранжевый: канал связан, но проверка канала сообщает об ухудшении состояния или отсутствии подключения.
- Красный: канал ещё не связан.

Во второй строке отображается «связан · аутентификация 12 мин назад» или причина сбоя.
Пункт меню «Запустить проверку состояния» запускает проверку по запросу.

## Настройки

- На вкладке «Основные» отображается карточка состояния: индикатор состояния, строка сводки (состояние связи и
  время с момента аутентификации) и необязательная строка с подробностями сбоя, а также кнопки **Повторить сейчас** и
  **Открыть журналы**.
- На вкладке **Каналы** отображаются состояние и элементы управления для каждого канала (QR-код для входа,
  выход, проверка, последнее отключение или ошибка) для WhatsApp и Telegram.

## Как работает проверка

Приложение вызывает RPC `health` в Gateway через существующее соединение WebSocket
(без запуска CLI через оболочку) примерно каждые 60 секунд и по запросу. RPC загружает
учётные данные и сообщает состояние, не отправляя сообщений. Приложение отдельно кэширует последний
успешный снимок состояния и последнюю ошибку, поэтому интерфейс загружается мгновенно и
не мерцает при отсутствии подключения.

## Если есть сомнения

Используйте последовательность команд CLI из раздела [Состояние Gateway](/ru/gateway/health) (`openclaw status`,
`openclaw status --deep`, `openclaw health --json`) и просматривайте
`/tmp/openclaw/openclaw-*.log`, фильтруя строки по `web-heartbeat` / `web-reconnect`.
