## Что отображается

- Текущее рабочее состояние агента отображается в значке строки меню и в первой строке состояния меню.
- Состояние работоспособности скрывается во время активной работы и снова появляется, когда все сеансы переходят в режим ожидания.
- Корневой пункт «Контекст» открывает подменю с недавними сеансами вместо их разворачивания в корневом меню.
- Блок «Узлы» в корневом меню содержит только сопряжённые **устройства** (из `node.list`), а не записи клиентов или присутствия.
- Когда доступны снимки использования провайдера, под разделом «Контекст» появляется корневой раздел «Использование», после которого, при наличии данных, отображается информация о расходах.

## Модель состояния

- Источник: `WorkActivityStore` (`apps/macos/Sources/OpenClaw/WorkActivityStore.swift`).
- События поступают как `ControlAgentEvent` с `runId`; обработчик (`ControlChannel.routeWorkActivity`) считывает `sessionKey` из полезной нагрузки события и по умолчанию использует `"main"`, если он отсутствует.
- Приоритет: основной сеанс (по умолчанию `sessionKey == "main"`) всегда имеет преимущество. Если основной сеанс активен, его состояние отображается немедленно. Если он находится в режиме ожидания, вместо него отображается последний активный неосновной сеанс. Хранилище не переключается во время активности; переключение происходит только тогда, когда текущий сеанс переходит в режим ожидания или основной сеанс становится активным.
- Виды активности:
  - `job`: выполнение высокоуровневой команды (`state: started|streaming|done|error|...`).
  - `tool`: `phase: start|result` с `name` и необязательными `meta`/`args`.

## Перечисление IconState (Swift)

- `idle`
- `workingMain(ActivityKind)`
- `workingOther(ActivityKind)`
- `overridden(ActivityKind)` (отладочное переопределение)

### ActivityKind -> символ индикатора

`ActivityKind` содержит `ToolKind` (`bash`, `read`, `write`, `edit`, `attach`, `other`) или отдельный `job`. Каждому из них соответствует символ SF Symbols, который отображается в виде индикатора поверх значка существа (`IconState.badgeSymbolName`):

| Вид             | Символ                             |
| --------------- | ---------------------------------- |
| `bash`          | `chevron.left.slash.chevron.right` |
| `read`          | `doc`                              |
| `write`         | `pencil`                           |
| `edit`          | `pencil.tip`                       |
| `attach`        | `paperclip`                        |
| `other` / `job` | `gearshape.fill`                   |

### Визуальное представление

- `idle`: обычное существо без индикатора.
- `workingMain`: индикатор с символом, насыщенный оттенок (выделение `.primary`), анимация «работающих» лапок.
- `workingOther`: индикатор с символом, приглушённый оттенок (выделение `.secondary`), без суетливого движения.
- `overridden`: выбранные символ и оттенок используются независимо от реальной активности.

## Подменю контекста

- В корневом меню отображается одна строка «Контекст» с количеством и состоянием сессий; она открывает подменю (`MenuSessionsInjector`).
- В заголовке подменю отображается количество активных сессий за последние 24 часа.
- В каждой строке сессии сохраняются индикатор токенов, возраст, предварительный просмотр, переключатель режима рассуждений и подробного вывода, а также действия сброса, сжатия и удаления.
- Сообщения о загрузке, отключении и ошибке загрузки сессий отображаются внутри подменю «Контекст».
- Разделы использования и стоимости остаются на корневом уровне под пунктом «Контекст», чтобы их можно было быстро просмотреть, не открывая подменю.

## Текст строки состояния (меню)

- Во время активной работы: `<Session role> · <activity label>` (`"\(roleLabel) · \(activity.label)"` в `MenuContentView`), где метка роли — `Main` или `Other`.
- В режиме ожидания: возвращается к сводке о состоянии.

## Приём событий

- Источник: события `agent` канала управления, маршрутизируемые методом `ControlChannel.routeWorkActivity(from:)`.
- Разбираемые поля:
  - `stream: "job"` с `data.state` для запуска и остановки.
  - `stream: "tool"` с `data.phase`, `data.name` и необязательными `data.meta`/`data.args`.
- Метки инструментов поступают из `ToolDisplayRegistry.resolve(name:args:meta:)`; для нераспознанных имён используется исходное имя инструмента.

## Переопределение для отладки

- Средство выбора Settings > Debug > "Icon override":
  - `System (auto)` (по умолчанию)
  - `Working: main` / `Working: other` (для каждого типа инструмента: bash, чтение, запись, редактирование, другое)
  - `Idle`
- Хранится в ключе `UserDefaults` `openclaw.iconOverride`; сопоставляется с `IconState.overridden`.

## Контрольный список тестирования

- Запустите задачу основного сеанса: значок переключается немедленно, а в строке состояния отображается метка основного сеанса.
- Запустите задачу неосновного сеанса, когда основной сеанс бездействует: значок и строка состояния отображают неосновной сеанс; состояние остаётся стабильным до завершения задачи.
- Запустите основной сеанс, пока активен другой сеанс: значок мгновенно переключается на основной сеанс.
- Быстрые серии вызовов инструментов: индикатор не мерцает (окно задержки 2 с перед удалением завершённого инструмента, `WorkActivityStore.toolResultGrace`).
- Строка состояния работоспособности снова появляется, когда все сеансы бездействуют.
