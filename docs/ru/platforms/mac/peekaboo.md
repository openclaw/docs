OpenClaw может размещать **PeekabooBridge** в качестве локального посредника автоматизации пользовательского интерфейса с учётом разрешений (`PeekabooBridgeHostCoordinator`, использующий пакет Swift `steipete/Peekaboo`). Это позволяет CLI `peekaboo` выполнять автоматизацию пользовательского интерфейса, используя разрешения TCC приложения для macOS.

## Что это такое (и чем не является)

- **Хост**: OpenClaw.app может выступать хостом PeekabooBridge.
- **Клиент**: CLI `peekaboo` (отдельного интерфейса `openclaw ui ...` нет).
- **Пользовательский интерфейс**: визуальные наложения остаются в Peekaboo.app; OpenClaw выступает лишь в роли хоста-посредника.

## Связь с другими способами управления рабочим столом

В OpenClaw предусмотрено четыре намеренно разделённых способа управления рабочим столом:

- **Хост PeekabooBridge**: OpenClaw.app размещает локальный сокет PeekabooBridge. CLI `peekaboo` выступает клиентом и использует разрешения OpenClaw.app в macOS для создания снимков экрана, выполнения щелчков, работы с меню и диалоговыми окнами, действий с Dock и управления окнами.
- **Управление компьютером агентом (`computer.act`)**: встроенный инструмент `computer` агента Gateway создаёт снимки экрана через `screen.snapshot` и управляет указателем и клавиатурой с помощью опасной команды Node `computer.act`. Node macOS выполняет `computer.act` внутри процесса, используя встроенные службы автоматизации Peekaboo, предоставляемые этим мостом, а также ограниченный набор примитивов CoreGraphics, не обращаясь к сокету PeekabooBridge или CLI `peekaboo`. См. [Управление компьютером](/nodes/computer-use).
- **Codex Computer Use**: встроенный Plugin `codex` проверяет наличие MCP-плагина Codex `computer-use` и может установить его (`extensions/codex/src/app-server/computer-use.ts`), после чего Codex самостоятельно выполняет вызовы инструментов нативного управления рабочим столом во время сеансов в режиме Codex. OpenClaw не проксирует эти действия через PeekabooBridge.
- **Прямой MCP `cua-driver`**: OpenClaw может зарегистрировать сервер `cua-driver mcp` от TryCua в качестве обычного MCP-сервера, предоставляя агентам собственные схемы драйвера CUA и рабочий процесс с PID, окнами и индексами элементов без маршрутизации через каталог Codex или сокет PeekabooBridge.

Используйте Peekaboo для широких возможностей автоматизации macOS через хост-мост OpenClaw.app, учитывающий разрешения. Используйте управление компьютером агентом, когда агент Gateway должен видеть рабочий стол и управлять им через унифицированную команду Node `computer.act`, доступную любой модели с поддержкой компьютерного зрения. Используйте Codex Computer Use, когда агент в режиме Codex должен полагаться на нативный Plugin Codex. Используйте прямой `cua-driver mcp`, чтобы предоставить драйвер CUA любой среде выполнения под управлением OpenClaw в качестве обычного MCP-сервера.

## Включение моста

В приложении для macOS: **Settings -> Enable Peekaboo Bridge**.

После включения OpenClaw запускает локальный сервер UNIX-сокета по адресу `~/Library/Application Support/OpenClaw/<socket-name>`. Если мост отключён, хост останавливается, а `peekaboo` переключается на другие доступные хосты. Координатор также поддерживает символические ссылки на устаревшие сокеты (`clawdbot`, `clawdis`, `moltbot` в каталоге Application Support), указывающие на текущий сокет для старых установок `peekaboo`.

## Порядок обнаружения клиентов

Клиенты Peekaboo обычно пытаются подключиться к хостам в следующем порядке:

1. Peekaboo.app (полноценный пользовательский интерфейс)
2. Claude.app (если установлено)
3. OpenClaw.app (минимальный посредник)

Чтобы узнать, какой хост активен и какой путь к сокету используется, выполните `peekaboo bridge status --verbose`. Чтобы переопределить его, используйте:

```bash
export PEEKABOO_BRIDGE_SOCKET=/path/to/bridge.sock
```

## Безопасность и разрешения

- Мост проверяет **подписи кода вызывающих клиентов**; применяется список разрешённых TeamID (TeamID хоста Peekaboo и собственный TeamID запущенного приложения).
- Для доступа к специальным возможностям предпочитайте подписанную идентификацию моста или приложения универсальной среде выполнения `node`. Предоставление специальных возможностей среде `node` позволяет любому пакету, запущенному этим исполняемым файлом Node, наследовать доступ к автоматизации графического интерфейса; см. [Разрешения macOS](/ru/platforms/mac/permissions#accessibility-grants-for-node-and-cli-runtimes).
- Время ожидания запросов составляет 10 секунд (`requestTimeoutSec: 10`).
- Если необходимые разрешения отсутствуют, мост возвращает понятное сообщение об ошибке вместо запуска System Settings.

## Поведение снимков (автоматизация)

Снимки хранятся в памяти в течение 10 минут, не более 50 снимков (`InMemorySnapshotManager`); при очистке артефакты не удаляются. Если требуется более длительное хранение, повторно создайте снимок на стороне клиента.

## Устранение неполадок

- Если `peekaboo` сообщает «клиент моста не авторизован», убедитесь, что клиент правильно подписан, либо запустите хост с `PEEKABOO_ALLOW_UNSIGNED_SOCKET_CLIENTS=1` исключительно в режиме **отладки**.
- Если хосты не найдены, откройте одно из приложений-хостов (Peekaboo.app или OpenClaw.app) и убедитесь, что разрешения предоставлены.
