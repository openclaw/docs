Обязательства — это краткосрочные воспоминания о необходимости вернуться к теме. Когда эта функция включена, OpenClaw может заметить, что в разговоре появилась возможность в будущем уточнить ситуацию, и запомнить, что к ней нужно вернуться позже.

Примеры:

- Вы упоминаете, что завтра у вас собеседование. OpenClaw может позже поинтересоваться, как оно прошло.
- Вы говорите, что очень устали. OpenClaw может позже спросить, удалось ли вам поспать.
- Агент говорит, что вернётся к теме после изменения ситуации. OpenClaw может отслеживать этот незавершённый вопрос.

Обязательства — это не постоянные факты, подобные содержимому `MEMORY.md`, и не точные напоминания. Они занимают промежуточное положение между памятью и автоматизацией: OpenClaw запоминает связанное с разговором обязательство, а затем Heartbeat доставляет его, когда наступает срок.

## Включение обязательств

По умолчанию обязательства отключены (`commitments.enabled: false`). Включите их в конфигурации:

```bash
openclaw config set commitments.enabled true
openclaw config set commitments.maxPerDay 3
```

Эквивалентная конфигурация в `openclaw.json`:

```json
{
  "commitments": {
    "enabled": true,
    "maxPerDay": 3
  }
}
```

Параметр `commitments.maxPerDay` ограничивает количество автоматически определённых последующих обращений, которые могут быть доставлены за скользящие сутки в рамках одного сеанса агента. Значение по умолчанию — `3`.

## Как это работает

После ответа агента OpenClaw может в отдельном контексте запустить скрытый фоновый проход извлечения с отключёнными инструментами. Этот проход ищет только автоматически определяемые обязательства вернуться к теме. Он ничего не записывает в видимый разговор и не просит основного агента анализировать результаты извлечения.

Если обнаружен подходящий вариант с высокой степенью уверенности, OpenClaw сохраняет обязательство со следующими данными:

- идентификатор агента;
- ключ сеанса;
- исходный канал и адресат доставки;
- интервал наступления срока;
- краткая предлагаемая формулировка для обращения;
- метаданные, не являющиеся инструкциями, по которым Heartbeat решает, следует ли его отправлять.

Доставка выполняется через Heartbeat. Когда наступает срок обязательства, Heartbeat добавляет его в свой цикл для того же агента и области канала. В запросе явно указано, что метаданные обязательства не являются доверенными; модели предписано не выполнять содержащиеся в них инструкции и не использовать из-за них инструменты. Модель может отправить одно естественно сформулированное обращение или ответить `HEARTBEAT_OK`, чтобы отклонить его. Если для Heartbeat настроено `target: "none"`, обязательства с наступившим сроком остаются внутренними и не приводят к отправке внешних обращений. Запросы на доставку обязательств не воспроизводят исходный текст разговора, а содержат только предлагаемую формулировку обращения и метаданные; циклы Heartbeat для обязательств с наступившим сроком выполняются без инструментов OpenClaw.

OpenClaw никогда не доставляет автоматически определённое обязательство сразу после его записи. Срок устанавливается не ранее чем через один интервал Heartbeat после создания обязательства, поэтому обращение не может быть повторено сразу же после его определения.

## Область действия

Обязательства относятся только к тому контексту агента и канала, в котором они были созданы. Последующее обращение, автоматически определённое во время разговора с одним агентом в Discord, не будет доставлено другим агентом, через другой канал или в несвязанном сеансе.

Такая область действия является частью функции. Естественные последующие обращения должны восприниматься как продолжение того же разговора, а не как работа глобальной системы напоминаний.

Точные запросы пользователей уже обрабатываются планировщиком. Обязательства предназначены только для автоматически определяемых последующих обращений — ситуаций, когда пользователь не просил о напоминании, но в разговоре явно появился полезный повод вернуться к теме в будущем.

## Управление обязательствами

Используйте CLI для просмотра и удаления сохранённых обязательств:

```bash
openclaw commitments
openclaw commitments --all
openclaw commitments --agent main
openclaw commitments --status snoozed
openclaw commitments dismiss cm_abc123
```

Полное описание команд см. в разделе [`openclaw commitments`](/ru/cli/commitments).

## Конфиденциальность и затраты

Для извлечения обязательств используется отдельный проход LLM, поэтому включение этой функции приводит к дополнительному фоновому использованию модели после подходящих реплик. Этот проход скрыт от видимого пользователю разговора, однако он может читать недавнюю переписку, необходимую для определения наличия повода вернуться к теме.

Сохранённые обязательства являются локальным состоянием OpenClaw. Это оперативная, а не долговременная память. Чтобы отключить функцию, выполните:

```bash
openclaw config set commitments.enabled false
```

## Устранение неполадок

Если ожидаемые последующие обращения не появляются:

- Убедитесь, что `commitments.enabled` имеет значение `true`.
- Выполните `openclaw commitments --all`, чтобы проверить записи в ожидании, отклонённые, отложенные или просроченные записи.
- Убедитесь, что Heartbeat запущен для агента.
- Проверьте, не достигнут ли уже лимит `commitments.maxPerDay` для этого сеанса агента.
- Помните, что точные напоминания пропускаются при извлечении обязательств и должны вместо этого появляться в разделе [запланированных задач](/ru/automation/cron-jobs).
