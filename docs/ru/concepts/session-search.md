# Поиск по сеансам

`sessions_search` выполняет поиск по тексту сообщений пользователя и ассистента в ваших прошлых сеансах. Каждый результат
содержит `sessionKey`, временную метку, роль и короткий совпавший фрагмент. Передайте возвращённый
`sessionKey` в `sessions_history`, когда потребуется окружающий контекст беседы.

## Видимость и вывод

При поиске применяются те же правила видимости сеансов, что и для `sessions_history`. Результаты за пределами
видимого вызывающей стороне дерева сеансов удаляются до применения ограничений на количество результатов. Агенты в песочнице по-прежнему имеют доступ
только к созданным ими сеансам, если включена видимость созданных сеансов.

Перед возвратом модели конфиденциальные данные во фрагментах скрываются. Результаты также ограничиваются по количеству, длине
фрагмента и общему размеру ответа.

## Жизненный цикл индекса

OpenClaw хранит полнотекстовый индекс рядом со строками расшифровки в базе данных SQLite каждого агента.
Новые сообщения пользователя и ассистента индексируются в той же транзакции, в которой они сохраняются, поэтому
индекс никогда не отстаёт от текущих бесед; результаты инструментов, блоки рассуждений и изображения исключаются.
Поиск выполняется только по активной ветви расшифровки.

Расшифровки, созданные до появления индекса (например, сеансы, импортированные с помощью `openclaw doctor`), и
сеансы, активная ветвь которых была перемотана назад, переиндексируются в ходе фонового согласования, запускаемого
при следующем поиске. Поэтому ответ с `indexing: true` может быть неполным; повторите попытку после
завершения индексирования. При удалении сеанса его записи индекса удаляются в той же транзакции.

В настоящее время поиск использует токенизатор слов Unicode в SQLite с удалением диакритических знаков. Токенизация
триграммами для поиска подстрок в текстах на языках CJK планируется в будущем.

## Поиск по сеансам и поиск по памяти

Используйте `sessions_search` для поиска точных слов или фраз в необработанных расшифровках сеансов. Используйте
[`memory_search`](/ru/concepts/memory-search) для файлов долговременной памяти и семантического поиска. Экспериментальный
корпус памяти сеансов служит семантическим дополнением к этому точному поиску по расшифровкам.
