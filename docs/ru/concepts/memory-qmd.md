[QMD](https://github.com/tobi/qmd) — это локальный вспомогательный поисковый сервис, который работает
вместе с OpenClaw. Он объединяет BM25, векторный поиск и переранжирование в одном
исполняемом файле и может индексировать содержимое за пределами файлов памяти рабочего пространства.

## Преимущества по сравнению со встроенным механизмом

- **Переранжирование и расширение запросов** для повышения полноты поиска.
- **Индексирование дополнительных каталогов** — документации проектов, заметок команды и любых файлов на диске.
- **Индексирование расшифровок сеансов** — для обращения к предыдущим беседам.
- **Полностью локальная работа** — используется официальный Plugin провайдера llama.cpp,
  а модели GGUF загружаются автоматически.
- **Автоматический резервный вариант** — если QMD недоступен, OpenClaw незаметно
  переключается на встроенный механизм.

## Начало работы

### Предварительные требования

- Установите QMD: `npm install -g @tobilu/qmd` или `bun install -g @tobilu/qmd`
- Сборка SQLite с поддержкой расширений (`brew install sqlite` в macOS).
- QMD должен находиться в `PATH` Gateway.
- macOS и Linux работают без дополнительной настройки. В Windows рекомендуется использовать WSL2.

### Включение

```json5
{
  memory: {
    backend: "qmd",
  },
}
```

OpenClaw создаёт автономный домашний каталог QMD по пути
`~/.openclaw/agents/<agentId>/qmd/` и автоматически управляет жизненным циклом
вспомогательного сервиса — коллекциями, обновлениями и созданием эмбеддингов.
Предпочтение отдаётся текущим форматам коллекций QMD и запросов MCP, но при
необходимости используются альтернативные флаги шаблонов коллекций и старые
имена инструментов MCP. Сверка при запуске также пересоздаёт устаревшие
управляемые коллекции с каноническими шаблонами, если всё ещё существует более
старая коллекция QMD с тем же именем.

## Как работает вспомогательный сервис

- OpenClaw создаёт коллекции из файлов памяти рабочего пространства и всех
  настроенных путей `memory.qmd.paths`, а затем запускает `qmd update` при
  открытии диспетчера QMD и периодически после этого (`memory.qmd.update.interval`,
  значение по умолчанию — `5m`). Обновление выполняется через подпроцессы QMD,
  а не посредством обхода файловой системы внутри процесса. В режимах
  семантического поиска также запускается `qmd embed`
  (`memory.qmd.update.embedInterval`, значение по умолчанию — `60m`).
- Стандартная коллекция рабочего пространства отслеживает `MEMORY.md` и дерево
  `memory/`. Файл `memory.md` в нижнем регистре не индексируется как корневой
  файл памяти.
- Собственный сканер QMD игнорирует скрытые пути и распространённые каталоги
  зависимостей и сборки, такие как `.git`, `.cache`, `node_modules`, `vendor`,
  `dist` и `build`. По умолчанию при запуске Gateway QMD не инициализируется
  (`memory.qmd.update.startup` по умолчанию имеет значение `off`), поэтому при
  холодном запуске среда памяти не импортируется и долгоживущий наблюдатель не
  создаётся до первого обращения к памяти.
- Задайте для `memory.qmd.update.startup` значение `idle` или `immediate`, чтобы
  всё равно инициализировать QMD при запуске Gateway.
  `memory.qmd.update.onBoot` по умолчанию имеет значение `true` и запускает
  первоначальное обновление при запуске; задайте `false`, чтобы пропустить это
  немедленное обновление (долгоживущий диспетчер всё равно открывается, если
  настроены интервалы обновления или создания эмбеддингов, поэтому QMD
  продолжает управлять штатным наблюдателем и таймерами).
- Для поиска используется настроенный `searchMode` (по умолчанию — `search`;
  также поддерживаются `vsearch` и `query`). Режим `search` использует только
  BM25, поэтому OpenClaw в этом режиме пропускает проверки готовности
  семантического векторного поиска и обслуживание эмбеддингов. При сбое режима
  OpenClaw повторяет попытку с помощью `qmd query`.
- Если `searchMode` имеет значение `query`, задайте
  `memory.qmd.rerank` значение `false`, чтобы использовать гибридный путь
  запросов QMD без переранжировщика (требуется QMD 2.1 или новее).
  OpenClaw передаёт `--no-rerank` при прямом вызове CLI QMD и
  `rerank: false` инструменту запросов MCP в QMD.
- В выпусках QMD, заявляющих поддержку фильтров по нескольким коллекциям,
  OpenClaw группирует коллекции из одного источника в один вызов поиска QMD.
  Для старых выпусков QMD сохраняется совместимый резервный вариант с отдельным
  поиском по каждой коллекции.
- Если QMD полностью выходит из строя, OpenClaw переключается на встроенный
  механизм SQLite. После ошибки открытия повторные попытки в последовательных
  ходах чата ненадолго приостанавливаются, чтобы отсутствующий исполняемый файл
  или неисправная зависимость вспомогательного сервиса не вызывали лавину
  повторных попыток; `openclaw memory status` и одноразовые проверки CLI
  по-прежнему проверяют QMD напрямую.

<Info>
Первый поиск может выполняться медленно: при первом запуске `qmd query` QMD
автоматически загружает модели GGUF (около 2 ГБ) для переранжирования и
расширения запросов.
</Info>

## Производительность и совместимость поиска

OpenClaw обеспечивает совместимость пути поиска QMD как с текущими, так и со
старыми установками QMD.

При запуске OpenClaw один раз для каждого диспетчера проверяет текст справки
установленного QMD. Если исполняемый файл заявляет поддержку нескольких
фильтров коллекций, OpenClaw выполняет поиск по всем коллекциям из одного
источника одной командой:

```bash
qmd search "router notes" --json -n 10 -c memory-root-main -c memory-dir-main
```

Это позволяет не запускать отдельный подпроцесс QMD для каждой коллекции
постоянной памяти. Коллекции расшифровок сеансов остаются в собственной группе
источников, поэтому смешанный поиск по `memory` и `sessions` по-прежнему
предоставляет средству диверсификации результаты из обоих источников.

Старые сборки QMD принимают только один фильтр коллекции. Обнаружив такую
сборку, OpenClaw использует совместимый путь и выполняет поиск по каждой
коллекции отдельно, а затем объединяет результаты и удаляет дубликаты.

Чтобы вручную проверить контракт установленной версии, выполните:

```bash
qmd --help | grep -i collection
```

В текущей справке QMD упоминается поиск по одной или нескольким коллекциям.
В старой справке обычно описывается одна коллекция.

## Переопределение моделей

Переменные окружения моделей QMD без изменений передаются из процесса Gateway,
поэтому QMD можно настроить глобально без добавления новых параметров
конфигурации OpenClaw:

```bash
export QMD_EMBED_MODEL="hf:Qwen/Qwen3-Embedding-0.6B-GGUF/Qwen3-Embedding-0.6B-Q8_0.gguf"
export QMD_RERANK_MODEL="/absolute/path/to/reranker.gguf"
export QMD_GENERATE_MODEL="/absolute/path/to/generator.gguf"
```

После изменения модели эмбеддингов повторно создайте эмбеддинги, чтобы индекс
соответствовал новому векторному пространству.

## Индексирование дополнительных путей

Укажите QMD дополнительные каталоги, чтобы сделать их доступными для поиска:

```json5
{
  memory: {
    backend: "qmd",
    qmd: {
      paths: [{ name: "docs", path: "~/notes", pattern: "**/*.md" }],
    },
  },
}
```

Фрагменты из дополнительных путей отображаются в результатах поиска в виде
`qmd/<collection>/<relative-path>`. `memory_get` распознаёт этот префикс и
читает данные из правильного корня коллекции.

## Индексирование расшифровок сеансов

Включите индексирование сеансов, чтобы обращаться к предыдущим беседам. QMD
требуются как общий источник сеансов `memorySearch`, так и экспортёр
расшифровок QMD:

```json5
{
  agents: {
    defaults: {
      memorySearch: {
        experimental: { sessionMemory: true },
        sources: ["memory", "sessions"],
      },
    },
  },
  memory: {
    backend: "qmd",
    qmd: {
      sessions: { enabled: true },
    },
  },
}
```

Расшифровки экспортируются в виде очищенных ходов пользователя и ассистента в
отдельную коллекцию QMD по пути
`~/.openclaw/agents/<id>/qmd/sessions/`. Установка только
`memorySearch.experimental.sessionMemory` не экспортирует расшифровки в QMD.

Результаты из сеансов по-прежнему фильтруются параметром
[`tools.sessions.visibility`](/ru/gateway/config-tools#toolssessions). Видимость
`tree`, используемая по умолчанию, не предоставляет доступ к несвязанным
сеансам того же агента. Если сеанс, запущенный через Gateway, должен быть
доступен для поиска из отдельного сеанса личных сообщений, намеренно задайте
`tools.sessions.visibility: "agent"`.

## Область поиска

По умолчанию результаты поиска QMD отображаются только в прямых сеансах, но не
в групповых чатах или чатах каналов. Чтобы изменить это поведение, настройте
`memory.qmd.scope`:

```json5
{
  memory: {
    qmd: {
      scope: {
        default: "deny",
        rules: [{ action: "allow", match: { chatType: "direct" } }],
      },
    },
  },
}
```

Приведённый выше фрагмент представляет фактическое правило по умолчанию. Когда
область запрещает поиск, OpenClaw записывает предупреждение с определёнными
каналом и типом чата, чтобы упростить диагностику пустых результатов.

## Ссылки на источники

Если `memory.citations` имеет значение `auto` или `on`, к фрагментам результатов
поиска добавляется нижний колонтитул `Source: <path>#L<line>` (или
`#L<start>-L<end>`). В режиме `auto` нижний колонтитул добавляется только для
сеансов прямого чата. Задайте `memory.citations = "off"`, чтобы не отображать
нижний колонтитул, продолжая при этом передавать путь агенту внутри системы.

## Когда использовать

Выбирайте QMD, если вам нужны:

- Переранжирование для повышения качества результатов.
- Поиск в документации проекта или заметках за пределами рабочего пространства.
- Доступ к предыдущим беседам в сеансах.
- Полностью локальный поиск без ключей API.

Для более простых конфигураций хорошо подходит
[встроенный механизм](/ru/concepts/memory-builtin), не требующий дополнительных
зависимостей.

## Устранение неполадок

**QMD не найден?** Убедитесь, что исполняемый файл находится в `PATH` Gateway.
Если OpenClaw работает как служба, создайте символическую ссылку:
`sudo ln -s ~/.bun/bin/qmd /usr/local/bin/qmd`.

Если `qmd --version` работает в вашей оболочке, но OpenClaw по-прежнему
сообщает `spawn qmd ENOENT`, вероятно, процесс Gateway использует другой
`PATH`, чем интерактивная оболочка. Явно укажите исполняемый файл:

```json5
{
  memory: {
    backend: "qmd",
    qmd: {
      command: "/absolute/path/to/qmd",
    },
  },
}
```

Выполните `command -v qmd` в среде, где установлен QMD, а затем повторно
проверьте состояние командой `openclaw memory status --deep`.

**Первый поиск выполняется очень медленно?** При первом использовании QMD
загружает модели GGUF. Выполните предварительный прогрев командой
`qmd query "test"`, используя те же каталоги XDG, которые применяет OpenClaw.

**Во время поиска запускается много подпроцессов QMD?** По возможности обновите
QMD. OpenClaw использует один процесс для поиска по нескольким коллекциям из
одного источника, только если установленная версия QMD заявляет поддержку
нескольких фильтров `-c`; в противном случае для обеспечения корректности
сохраняется старый резервный вариант с отдельным поиском по каждой коллекции.

**QMD в режиме только BM25 всё равно пытается собрать llama.cpp?** Задайте
`memory.qmd.searchMode = "search"`. OpenClaw считает этот режим исключительно
лексическим, пропускает проверки состояния векторного поиска QMD и обслуживание
эмбеддингов, а проверки готовности семантического поиска оставляет конфигурациям
`vsearch` или `query`.

**Истекает время ожидания поиска?** Увеличьте
`memory.qmd.limits.timeoutMs` (значение по умолчанию: 4000 мс). Для более
медленного оборудования задайте большее значение, например `120000`.

**Пустые результаты в групповых чатах или чатах каналов?** Это ожидаемое
поведение при стандартной области `memory.qmd.scope`, которая разрешает только
прямые сеансы. Добавьте правило `allow` для типов чата `group` или `channel`,
если хотите получать там результаты QMD.

**Поиск в корневой памяти внезапно стал слишком широким?** Перезапустите Gateway
или дождитесь следующей сверки при запуске. Обнаружив конфликт с коллекцией
того же имени, OpenClaw пересоздаёт устаревшие управляемые коллекции с
каноническими шаблонами `MEMORY.md` и `memory/`.

**Временные репозитории, видимые из рабочего пространства, вызывают
`ENAMETOOLONG` или нарушают индексирование?** При обходе QMD использует
собственный базовый сканер, а не встроенные правила OpenClaw для символических
ссылок. Пока QMD не предоставит безопасный при наличии циклов обход или явные
средства исключения, храните временные копии монорепозитория в скрытых
каталогах, например `.tmp/`, или за пределами индексируемых корней QMD.
