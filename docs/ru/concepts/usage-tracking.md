## Что это такое

- Получает данные об использовании и квотах провайдера непосредственно из конечной точки использования каждого провайдера. Никаких расчётных данных о расходах у провайдера — только сообщаемые провайдером названия тарифов, окна квот, балансы, расходы, бюджеты, ежедневная история затрат, распределение по токенам и моделям или сводки о состоянии учётной записи.
- Удобочитаемый вывод окна квоты нормализуется до формата `X% осталось`, даже если провайдер сообщает израсходованную квоту, оставшуюся квоту или только необработанные числовые значения. Для провайдеров без сбрасываемых окон квот вместо этого отображается сводный текст провайдера, например баланс.
- Команда `/status` на уровне сеанса и инструмент `session_status` используют журнал расшифровки сеанса как резервный источник, если в актуальном снимке сеанса отсутствуют данные о токенах или модели. Этот резервный механизм заполняет отсутствующие счётчики токенов и кеша, может восстановить метку активной модели среды выполнения и предпочитает большее итоговое значение, ориентированное на запрос, если метаданные сеанса отсутствуют или содержат меньшее значение (`totalTokensFresh !== true`, ноль или значение ниже полученного из расшифровки). Ненулевые актуальные значения всегда имеют приоритет над резервными.

## Где это отображается

- `/status` в чатах: карточка состояния с токенами сеанса и расчётной стоимостью (только для моделей с ключом API). Когда доступны данные об использовании провайдера **текущей модели**, они отображаются как нормализованное окно `X% осталось` или сводный текст провайдера.
- `/usage off|tokens|full` в чатах: нижний колонтитул с данными об использовании для каждого ответа.
- `/usage cost` в чатах: локальная сводка затрат, агрегированная из журналов сеансов OpenClaw.
- CLI: `openclaw status --usage` выводит полную разбивку использования и квот по провайдерам.
- CLI: `openclaw models status` перечисляет профили аутентификации OAuth и токенов и показывает сводку окна использования рядом с каждым провайдером, для которого она доступна.
- Интерфейс управления: раздел **Использование** показывает карточки тарифа и оплаты провайдера над анализом токенов и расчётной стоимости, полученным из сеансов OpenClaw. Учётные данные Anthropic и OpenAI Admin API дополнительно показывают сообщаемые провайдером расходы за сегодня, 7 и 30 дней, ежедневную динамику, итоговые значения токенов, наиболее используемые модели и категории затрат.
- Интерфейс управления: всплывающее окно кольца контекста в редакторе сообщений чата показывает **использование тарифа** для провайдеров с подпиской — индикаторы для каждого окна (5 часов, неделя, отдельная модель) со временем сброса, тариф провайдера, если он известен (например, `Max (20x)`), и кредиты на дополнительное использование. В сеансах, оплачиваемых по тарифу, расчётные суммы в долларах за токены скрыты; в сеансах с оплатой через API сохраняются `Расч. стоимость` и разбивка затрат по типам. Конфигурации Claude Code CLI (`claude-cli`) используют те же данные о подписке Anthropic.
- Строка меню macOS: корневой раздел «Использование» отображается под разделом «Контекст», когда доступны снимки использования провайдера. См. [Строка меню](/ru/platforms/mac/menu-bar).

`openclaw channels list` больше не выводит данные об использовании провайдера; вместо этого команда направляет пользователей к `openclaw status` или `openclaw models list`.

## История затрат Anthropic и OpenAI

Квоты подписки и оплата API относятся к разным интерфейсам провайдера:

- Учётные данные подписки или конфигурации Anthropic продолжают отображать окна квот Claude и необязательные бюджеты дополнительного использования. Задайте `ANTHROPIC_ADMIN_KEY` или `ANTHROPIC_ADMIN_API_KEY`, чтобы вместо этого отображалась история из API использования и затрат организации. Учётные данные провайдера Anthropic, начинающиеся с `sk-ant-admin`, обнаруживаются автоматически.
- OAuth OpenAI ChatGPT/Codex продолжает отображать тариф, окна квот и кредитный баланс. Задайте `OPENAI_ADMIN_KEY`, чтобы вместо этого отображалась история затрат организации и использования завершений; при необходимости задайте `OPENAI_PROJECT_ID`, чтобы ограничить данные одним проектом. OpenClaw никогда не отправляет учётные данные для инференса из `OPENAI_API_KEY`, конфигурации провайдера или профилей аутентификации в API организации, поскольку эти ключи могут относиться к пользовательским конечным точкам.

Учётные данные администратора имеют приоритет, поскольку предоставляют фактические данные об оплате организации. OpenClaw не объединяет эти сообщаемые провайдером итоговые значения со своими локальными оценками по сеансам: эти два раздела намеренно отвечают на разные вопросы.

## Режим нижнего колонтитула использования по умолчанию

`/usage off|tokens|full` задаёт нижний колонтитул для сеанса, и этот выбор
запоминается для данного сеанса. `messages.responseUsage` задаёт начальный режим
для сеансов, в которых он ещё не выбран, поэтому нижний колонтитул можно включить
по умолчанию без необходимости каждый раз вводить `/usage`.

Задайте один режим для всех каналов или карту режимов по каналам с резервным
значением `default`:

```jsonc
{
  "messages": {
    "responseUsage": "tokens",
    // или: { "default": "off", "discord": "full" }
  },
}
```

Допустимые значения: `"off"`, `"tokens"`, `"full"` и устаревший псевдоним `"on"` (обрабатывается как `"tokens"`).

### Три различных состояния сеанса

Поле `responseUsage` сеанса имеет три представимых состояния, каждое со своей
семантикой:

| Состояние                    | Сохранённое значение             | Действующий режим                                                               |
| ---------------------------- | -------------------------------- | ------------------------------------------------------------------------------- |
| **Не задано / наследование** | `undefined` (отсутствует)        | Используется значение `messages.responseUsage` из конфигурации, затем `off`.    |
| **Явно выключено**           | `"off"` (сохранено)              | Всегда выключено; конфигурация со значением не `off` не может включить колонтитул. |
| **Явно включено**            | `"tokens"` или `"full"` (сохранено) | Этот режим независимо от значения конфигурации по умолчанию.                 |

### Приоритет

Действующий режим = переопределение сеанса → запись конфигурации канала → `default` → `off`.

Явное значение `/usage off` **сохраняется** в сеансе как буквальное значение
`"off"` и не равнозначно состоянию «не задано». Значение
`messages.responseUsage` по умолчанию, отличное от `off`, не может снова включить
нижний колонтитул после того, как пользователь явно его отключил.

### Сброс и выключение

- `/usage off` принудительно отключает нижний колонтитул и сохраняет этот выбор.
  Настроенное значение по умолчанию, отличное от `off`, не может его переопределить.
- `/usage reset` (псевдонимы: `default`, `inherit`, `inherited`, `clear`, `unpin`) очищает
  переопределение сеанса. После этого сеанс **наследует** действующее значение
  конфигурации по умолчанию (`messages.responseUsage`). Если значение по умолчанию
  не настроено, нижний колонтитул остаётся выключенным.
- Полный сброс сеанса (`/reset` или `/new`) или переход к новому сеансу
  **сохраняет** явно выбранный режим использования, чтобы выбранный пользователем
  вариант отображения сохранялся при смене сеансов. Переопределение очищает только
  `/usage reset` и его псевдонимы.

### Поведение переключателя

Команда `/usage` без аргументов циклически переключает режимы: off → tokens → full → off.
Начальной точкой цикла служит **действующий** текущий режим: переопределение сеанса
или значение конфигурации по умолчанию, если переопределение не задано. Поэтому
цикл всегда соответствует тому, что пользователь сейчас видит в нижнем колонтитуле.

### Конфигурация

Без конфигурации сохраняется прежнее поведение: нижний колонтитул выключен до
выполнения `/usage`. Используйте `/usage reset`, чтобы очистить переопределение
сеанса и снова наследовать настроенное значение по умолчанию.

## Пользовательский нижний колонтитул `/usage full`

`/usage tokens` всегда отображает простую строку `Использование: X входящих / Y исходящих`
с суффиксами кеша и расчётной стоимости, когда они доступны. Только `/usage full`
отображает более подробный нижний колонтитул, описанный ниже.

`/usage full` показывает встроенный компактный нижний колонтитул с моделью, режимом
рассуждений, быстрым или медленным режимом, окном контекста и стоимостью, когда
соответствующие поля доступны. Для встроенного нижнего колонтитула файл шаблона
не требуется.

`messages.usageTemplate` предназначен только для расширенных пользовательских
макетов. Значением служит путь к файлу JSON (поддерживается `~`) или встроенный
объект; при корректном значении он заменяет встроенный нижний колонтитул.
Изменения файла по указанному пути отслеживаются и загружаются без перезапуска.

```json
{
  "messages": {
    "usageTemplate": "~/.openclaw/usage-footer.json"
  }
}
```

При отсутствующем или пустом шаблоне незаметно используется встроенный нижний
колонтитул. Если настроенный шаблон невозможно прочитать или он некорректен
(неверный JSON или структура без отображаемых элементов), также используется
встроенный нижний колонтитул и выводится предупреждение для оператора.

Возьмите встроенную структуру за основу пользовательского шаблона, а затем
измените нужные части:

### Структура

```jsonc
{
  "schema": "openclaw.usageBar.v1",
  "scales": { "<name>": "глифы от низкого к высокому" }, // строка (1 глиф/символ) или массив
  "aliases": { "<table>": { "<value>": "<label>" } },
  "output": {
    "sep": "", // объединяет оставшиеся элементы
    "default": [/* элементы */], // резервный вариант для любой поверхности
    "surfaces": {
      "discord": [/* элементы */],
      "telegram": [/* элементы */],
    },
  },
}
```

Каждая поверхность представляет собой упорядоченный список **элементов**; движок
отрисовывает каждый из них, удаляет пустые и объединяет оставшиеся с помощью `sep`.
Для поверхности без отдельной записи используется `output.default`.

### Пути контракта

Элемент считывает значения из контракта текущего шага по пути с точками.
Отсутствующие значения считаются пустыми, поэтому условие `when` или
резервное значение `|fallback` сохраняет элемент корректным.

| Путь                                                                                | Значение                                                                                              |
| ----------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------- |
| `surface`                                                                           | идентификатор канала (`discord`/`telegram`/и т. д.)                                                   |
| `agentId` / `chat_type`                                                             | идентификатор агента-владельца / тип поверхности чата                                                |
| `model.id` / `model.display_name` / `model.provider`                                | идентификатор модели / отображаемое имя / идентификатор провайдера                                   |
| `model.actual`, `model.resolved_ref`                                                | ссылка на провайдера/модель, фактически использованную для хода                                       |
| `model.requested`                                                                   | запрошенная ссылка на провайдера/модель (до резервного переключения)                                  |
| `model.reasoning`                                                                   | интенсивность (`off`—`xhigh`)                                                                         |
| `model.is_fallback` / `model.is_override`                                           | логическое значение: использовано резервное переключение / модель закреплена                         |
| `model.override_source` / `model.auth_mode`                                         | метка источника переопределения / режим учётных данных (`oauth`, `api-key`, `token`, `mixed`, `aws-sdk`, `unknown`) |
| `state.fast_mode`                                                                   | логическое значение: быстрый или медленный режим                                                     |
| `state.compactions`                                                                 | количество Compaction за сеанс                                                                        |
| `context.max_tokens` / `context.used_tokens` / `context.pct_used`                   | бюджет окна / занятые токены / использовано 0–100                                                    |
| `usage.input_tokens` / `usage.output_tokens` / `usage.total_tokens`                 | совокупные показатели хода                                                                           |
| `usage.cache_read_tokens` / `usage.cache_write_tokens`                              | токены чтения из кеша и записи в кеш за ход                                                          |
| `usage.has_tokens` / `usage.has_split_tokens` / `usage.has_total_only_tokens`       | условия отображения токенов                                                                          |
| `usage.cache_hit_pct`                                                               | доля чтения из кеша от общего количества токенов запроса                                             |
| `usage.last.input_tokens` / `usage.last.output_tokens` / `usage.last.cache_hit_pct` | только последний вызов модели (также содержит `cache_read_tokens`, `cache_write_tokens`, `total_tokens`) |
| `cost.turn_usd` / `cost.available`                                                  | оценочная стоимость хода / удалось ли определить таблицу стоимости                                   |
| `timing.duration_ms`                                                                | фактическая продолжительность хода                                                                    |
| `identity.name` / `identity.emoji` / `identity.avatar`                              | имя профиля агента / эмодзи / аватар                                                                  |
| `session.id`                                                                        | идентификатор сеанса                                                                                  |

(Окна ограничений частоты запросов провайдера **не** входят в этот контракт; сейчас нет ни одного пути со значением-массивом, поэтому элементу `each` нечего перебирать.)

### Операции

Передавайте значение через операции слева направо; сегмент, не являющийся операцией, используется как резервное значение.

| Операция        | Эффект                                        | Пример                            |
| --------------- | --------------------------------------------- | --------------------------------- |
| `num`           | компактное представление числа                | `272000 -> 272k`                  |
| `fixed:N`       | N знаков после запятой (по умолчанию 2)       | `0.0377`                          |
| `dur`           | преобразование секунд в длительность          | `14820 -> 4h07m`                  |
| `pct`           | добавление `%`                                | `96 -> 96%`                       |
| `inv`           | `100 - x`                                     | из использованного в оставшееся   |
| `alias:TABLE`   | поиск в `aliases`, исходное значение при отсутствии | `medium -> 🌗`              |
| `meter:W:SCALE` | полоса из W ячеек для значения от 0 до 100    | `[⣿⣿⠐⠐⠐]` (`meter:1` = один символ) |

### Формы элементов

- `{ "text": "📚 {context.max_tokens|num}" }`: литерал + интерполяция.
- `{ "when": "<path>", "text": "..." }`: отображать, только если значение по пути истинно.
- `{ "map": "<path>", "cases": { "true": "⚡", "false": "🐌" } }`: преобразование значения в символ (вариант `_default` охватывает несовпавшие значения).
- `{ "each": "<array-path>", "item": "{label}" }`: перебор пути со значением-массивом (ни один текущий путь контракта не является массивом).

### Пример

```jsonc
{
  "schema": "openclaw.usageBar.v1",
  "scales": { "braille": "⠐⡀⡄⡆⡇⣇⣧⣷⣿" },
  "aliases": { "reasoning": { "medium": "🌗", "high": "🌕" } },
  "output": {
    "surfaces": {
      "discord": [
        { "text": "{model.display_name}" },
        { "when": "model.reasoning", "text": " {model.reasoning|alias:reasoning}" },
        { "map": "state.fast_mode", "cases": { "true": " ⚡", "false": " 🐌" } },
        {
          "when": "context.max_tokens",
          "text": " | 📚 [{context.pct_used|meter:5:braille}]{context.max_tokens|num}",
        },
      ],
    },
  },
}
```

отображается, например, как `claude-sonnet-4-6 🌗 🐌 | 📚 [⣿⣿⣿⣿⣧]272k`.

## Провайдеры и учётные данные

Данные об использовании скрываются, если не удаётся определить пригодные учётные данные для доступа к статистике провайдера. OpenClaw автоматически обнаруживает включённые плагины провайдеров, которые объявляют `contracts.usageProviders` и реализуют как `resolveUsageAuth`, так и `fetchUsageSnapshot`; отдельного списка разрешённых провайдеров в ядре нет. Статический контракт ограничивает область обнаружения без импорта каждого плагина провайдера. Каждый плагин отвечает за свою внешнюю конечную точку и преобразование ответа. Общий снимок сохраняет нейтральное к провайдерам представление названий тарифов, окон квот, балансов, расходов и бюджетов для потребителей CLI, приложения и интерфейса управления.

- **Anthropic (Claude)**: токены OAuth в профилях аутентификации. Если у токена OAuth отсутствует область действия `user:profile`, выполняется резервное переключение на веб-сеанс `claude.ai` (`CLAUDE_AI_SESSION_KEY`, `CLAUDE_WEB_SESSION_KEY` или файл cookie `sessionKey=` в `CLAUDE_WEB_COOKIE`), если он задан. Ограничения для конкретных моделей и включённые ежемесячные расходы/бюджеты дополнительного использования учитываются, когда Anthropic их сообщает. Явный ключ Anthropic Admin API либо автоматически обнаруженный профиль провайдера `sk-ant-admin...` вместо этого показывает расходы организации за 30 дней и историю Messages API.
- **ClawRouter**: ключ API (`CLAWROUTER_API_KEY`). Если настроено, показывает месячное окно бюджета и типизированный бюджет в долларах США; иначе показывает совокупные расходы и сводку по запросам, токенам и стоимости.
- **DeepSeek**: ключ API из переменной среды, конфигурации или хранилища аутентификации (`DEEPSEEK_API_KEY`). Показывает каждый баланс в валюте, сообщённый провайдером.
- **GitHub Copilot**: токены OAuth в профилях аутентификации.
- **Gemini CLI**: токены OAuth в профилях аутентификации.
- **MiniMax**: ключ API или профиль аутентификации MiniMax OAuth. OpenClaw рассматривает `minimax`, `minimax-cn` и `minimax-portal` как одну поверхность квот MiniMax, отдаёт предпочтение сохранённому MiniMax OAuth при его наличии, а иначе выполняет резервное переключение на `MINIMAX_CODE_PLAN_KEY`, `MINIMAX_CODING_API_KEY` или `MINIMAX_API_KEY`.
  При настроенном параметре опрос использования получает адрес Coding Plan из `models.providers.minimax-portal.baseUrl` или `models.providers.minimax.baseUrl`, а иначе использует адрес MiniMax CN.
  Необработанные поля MiniMax `usage_percent` / `usagePercent` означают **оставшуюся** квоту, поэтому OpenClaw инвертирует их перед отображением; при наличии полей со счётчиками они имеют приоритет.
  - Метки окон берутся из переданных провайдером полей часов/минут, если они присутствуют, а иначе — из интервала `start_time` / `end_time`.
  - Если конечная точка тарифного плана для программирования возвращает `model_remains`, OpenClaw отдаёт предпочтение записи модели чата, вычисляет метку окна по временным меткам при отсутствии явных полей `window_hours` / `window_minutes` и включает название модели в метку тарифа.
- **OpenAI (тариф Codex/ChatGPT)**: токены OAuth в профилях аутентификации (заголовок `ChatGPT-Account-Id` отправляется при наличии идентификатора учётной записи). Показывает тариф ChatGPT, сбрасываемые окна Codex и кредитный баланс, если провайдер его сообщает. Кредиты остаются кредитами провайдера; OpenClaw не обозначает их как доллары. `OPENAI_ADMIN_KEY` добавляет расходы организации за 30 дней и историю использования завершений, если ключ имеет доступ к Usage Dashboard. Учётные данные для инференса никогда не передаются API организации.
- **OpenRouter**: ключ API или ключ API с поддержкой OAuth (`OPENROUTER_API_KEY` либо профиль аутентификации). Объединяет конечную точку кредитов учётной записи с конечной точкой квоты ключа, поэтому при наличии у учётных данных соответствующего доступа отображаются баланс и расходы учётной записи, бюджет ключа, а также дневное, недельное и месячное использование. Каждая конечная точка может независимо дополнять снимок.
- **Venice**: ключ API из переменной среды, конфигурации или хранилища аутентификации (`VENICE_API_KEY`). Показывает балансы в долларах США и DIEM, а также использование распределения DIEM за эпоху, если провайдер сообщает эти данные.
- **Xiaomi MiMo**: две отдельные поверхности использования. Оплата по мере использования применяет ключ API (`XIAOMI_API_KEY`); Token Plan использует отдельный ключ (`XIAOMI_TOKEN_PLAN_API_KEY`). Сейчас ни одна из них не сообщает окна квот.
- **z.ai**: ключ API из переменной среды, конфигурации или хранилища аутентификации (`ZAI_API_KEY` или `Z_AI_API_KEY`).
