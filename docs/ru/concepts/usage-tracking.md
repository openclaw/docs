## Что это такое

- Получает использование и квоты провайдера напрямую из его endpoint'ов использования.
- Без расчетных затрат; только окна квот или сводки состояния аккаунта, которые сообщает провайдер.
- Удобочитаемый вывод статуса окна квоты нормализуется к `X% left`, даже когда upstream API сообщает израсходованную квоту, оставшуюся квоту или только необработанные счетчики. Провайдеры без сбрасываемых окон квот могут вместо этого показывать текст сводки провайдера, например баланс.
- `/status` на уровне сессии и `session_status` могут откатываться к последней записи использования из транскрипта, когда live-снимок сессии неполный. Этот fallback заполняет отсутствующие счетчики токенов/кэша, может восстановить метку активной runtime-модели и предпочитает больший prompt-ориентированный итог, когда метаданные сессии отсутствуют или меньше. Существующие ненулевые live-значения все равно имеют приоритет.

## Где это отображается

- `/status` в чатах: статусная карточка с эмодзи, токенами сессии и расчетной стоимостью (только API-ключ). Использование провайдера отображается для **провайдера текущей модели**, когда доступно, как нормализованное окно `X% left` или текст сводки провайдера.
- `/usage off|tokens|full` в чатах: футер использования для каждого ответа.
- `/usage cost` в чатах: локальная сводка затрат, агрегированная из логов сессий OpenClaw.
- CLI: `openclaw status --usage` печатает полный разбор по каждому провайдеру.
- CLI: `openclaw channels list` печатает тот же снимок использования рядом с конфигурацией провайдера (используйте `--no-usage`, чтобы пропустить).
- Строка меню macOS: раздел "Usage" в Context (только если доступно).

## Режим футера использования по умолчанию

`/usage off|tokens|full` задает футер для сессии и запоминается для этой сессии. `messages.responseUsage` задает начальное значение этого режима для сессий, которые еще не выбрали режим, поэтому футер может быть включен по умолчанию без ввода `/usage` каждый раз.

Задайте один режим для каждого канала или карту по каналам с fallback `default`:

```jsonc
{
  "messages": {
    "responseUsage": "tokens",
    // or: { "default": "off", "discord": "full" }
  },
}
```

### Три различных состояния сессии

Поле `responseUsage` у сессии имеет три представимых состояния, каждое с разной семантикой:

| Состояние                 | Сохраненное значение             | Эффективный режим                                                      |
| ------------------------- | -------------------------------- | ---------------------------------------------------------------------- |
| **Не задано / наследовать** | `undefined` (отсутствует)        | Переходит к значению по умолчанию из конфигурации `messages.responseUsage`, затем к `off`. |
| **Явно выключено**        | `"off"` (сохранено)              | Всегда выключено — значение конфигурации по умолчанию не `off` не может снова включить футер. |
| **Явно включено**         | `"tokens"` или `"full"` (сохранено) | Этот режим, независимо от значения конфигурации по умолчанию.          |

### Приоритет

Эффективный режим = переопределение сессии → запись конфигурации канала → `default` → `off`.

Явный `/usage off` **сохраняется** как буквальное значение `"off"` в сессии, а не как «не задано». Это означает, что значение `messages.responseUsage` по умолчанию, отличное от `off`, не может снова включить футер после того, как пользователь явно его отключил.

### Сброс и выключение

- `/usage off` — принудительно выключает футер и сохраняет этот выбор. Настроенное значение по умолчанию, отличное от `off`, не может это переопределить.
- `/usage reset` (псевдонимы: `inherit`, `clear`, `default`) — очищает переопределение сессии. После этого сессия **наследует** эффективное значение конфигурации по умолчанию (`messages.responseUsage`). Если значение по умолчанию не настроено, футер выключен (без изменений относительно прежнего поведения). Используйте это, чтобы «вернуться к значению по умолчанию» без явного включения футера.
- Полный сброс сессии (`/reset` или `/new`) или rollover сессии **сохраняет** явное предпочтение режима использования, чтобы выбранный пользователем вариант отображения переживал rollover'ы сессии. Только `/usage reset` (и его псевдонимы) фактически очищает переопределение.

### Поведение переключателя

`/usage` без аргументов циклически переключает: off → tokens → full → off. Начальная точка цикла — **эффективный** текущий режим (переопределение сессии с переходом к значению конфигурации по умолчанию, если оно не задано), поэтому цикл всегда соответствует тому, что пользователь видит в футере.

### Конфигурация

Без конфигурации сохраняется прежнее поведение (футер выключен до `/usage`). Используйте `/usage reset`, чтобы очистить переопределение сессии и снова наследовать настроенное значение по умолчанию.

## Пользовательский футер `/usage full`

`/usage full` показывает встроенный компактный футер с моделью, reasoning, быстрым/медленным режимом, окном контекста и стоимостью, когда эти поля доступны. Поля токенов и кэша остаются доступными для пользовательских шаблонов. Файл шаблона не требуется.

`messages.usageTemplate` предназначен только для продвинутых пользовательских компоновок. Значение — путь к JSON-файлу (поддерживает `~`) или inline-объект, и при валидности оно заменяет встроенный футер:

```json
{
  "messages": {
    "usageTemplate": "~/.openclaw/usage-footer.json"
  }
}
```

Отсутствующие или пустые шаблоны тихо откатываются к встроенному футеру. Нечитаемые или невалидные настроенные шаблоны также откатываются к встроенному футеру и выводят предупреждение оператору.

Начните пользовательские шаблоны со встроенной формы, затем измените нужные части:

    "moon": "🌑🌘🌗🌖🌕",
    "level": "▁▂▃▄▅▆▇█",
    "weather": ["🥶", "☁️", "🌥", "⛅️", "🌤", "☀️"],
    "plants": ["🪾", "🍂", "🌱", "☘️", "🍀", "🌿"],
    "moons6": ["🌑", "🌚", "🌘", "🌗", "🌖", "🌝"],
  },
  "aliases": {
    "models": {
      "claude-opus-4-6": "opus46",
      "claude-opus-4-8": "opus48",
      "claude-sonnet-4-6": "sonnet46",

      {
        "when": "context.max_tokens",

        { "text": "-# -\n" },

### Форма

```jsonc
{
  "schema": "openclaw.usageBar.v1",
  "scales": { "<name>": "low-to-high glyphs" }, // string (1 glyph/char) or array
  "aliases": { "<table>": { "<value>": "<label>" } },
  "output": {
    "sep": "", // joins surviving pieces
    "default": [
      /* pieces */
    ], // fallback for any surface
    "surfaces": {
      "discord": [
        /* pieces */
      ],
      "telegram": [
        /* pieces */
      ],
    },
  },
}
```

Каждая поверхность — это упорядоченный список **фрагментов**; движок рендерит каждый из них, отбрасывает пустые и соединяет оставшиеся через `sep`. Поверхность без записи использует `output.default`.

### Пути контракта

Фрагмент читает значения из контракта текущего turn по dot-path. Отсутствующие значения считаются пустыми (поэтому guard `when` или `|fallback` сохраняет фрагмент чистым).

| Путь                                                                                | Значение                               |
| ----------------------------------------------------------------------------------- | -------------------------------------- |
| `surface`                                                                           | id канала (`discord`/`telegram`/и т. д.) |
| `model.provider` / `model.display_name`                                             | id провайдера / id модели              |
| `model.reasoning`                                                                   | effort (от `off` до `xhigh`)           |
| `model.is_fallback` / `model.is_override`                                           | bool: использован fallback / модель закреплена |
| `state.fast_mode`                                                                   | bool: быстрый или медленный            |
| `context.max_tokens` / `context.pct_used`                                           | бюджет окна / использовано 0-100       |
| `usage.input_tokens` / `usage.output_tokens` / `usage.total_tokens`                 | агрегат turn                           |
| `usage.has_split_tokens` / `usage.has_total_only_tokens` / `usage.cache_hit_pct`    | guards отображения токенов и процент попаданий в кэш |
| `usage.last.input_tokens` / `usage.last.output_tokens` / `usage.last.cache_hit_pct` | только последний вызов модели          |
| `cost.turn_usd`                                                                     | расчетная стоимость turn               |
| `identity.name` / `identity.emoji`                                                  | имя агента / выбранный эмодзи          |

(Окна rate limit провайдера **не** входят в этот контракт.)

### Операции

Передавайте значение через операции слева направо; сегмент, который не является операцией, используется как fallback.

| Операция        | Эффект                                | Пример                            |
| --------------- | ------------------------------------- | --------------------------------- |
| `num`           | компактный счетчик                    | `272000 -> 272k`                  |
| `fixed:N`       | N знаков после запятой (по умолчанию 2) | `0.0377`                          |
| `dur`           | секунды в длительность                | `14820 -> 4h07m`                  |
| `pct`           | добавить `%`                          | `96 -> 96%`                       |
| `inv`           | `100 - x`                             | из использованного в оставшееся   |
| `alias:TABLE`   | поиск в `aliases`, вывод как есть, если нет в списке | `medium -> 🌗`                    |
| `meter:W:SCALE` | глифовая шкала W ячеек для значения 0-100 | `[⣿⣿⠐⠐⠐]` (`meter:1` = один глиф) |

### Формы фрагментов

- `{ "text": "📚 {context.max_tokens|num}" }`: литерал + интерполяция.
- `{ "when": "<path>", "text": "..." }`: рендерить только если путь truthy.
- `{ "map": "<path>", "cases": { "true": "⚡", "false": "🐌" } }`: значение в глиф.
- `{ "each": "limits.windows", "item": "{label}" }`: итерация по массиву.

### Пример

```jsonc
{
  "schema": "openclaw.usageBar.v1",
  "scales": { "braille": "⠐⡀⡄⡆⡇⣇⣧⣷⣿" },
  "aliases": { "reasoning": { "medium": "🌗", "high": "🌕" } },
  "output": {
    "surfaces": {
      "discord": [
        { "text": "{model.display_name}" },
        { "when": "model.reasoning", "text": " {model.reasoning|alias:reasoning}" },
        { "map": "state.fast_mode", "cases": { "true": " ⚡", "false": " 🐌" } },
        {
          "when": "context.max_tokens",
          "text": " | 📚 [{context.pct_used|meter:5:braille}]{context.max_tokens|num}",
        },
      ],
    },
  },
}
```

рендерит, например, `claude-sonnet-4-6 🌗 🐌 | 📚 [⣿⣿⣿⣿⣧]272k`.

## Провайдеры и учетные данные

- **Anthropic (Claude)**: токены OAuth в профилях аутентификации.
- **GitHub Copilot**: токены OAuth в профилях аутентификации.
- **Gemini CLI**: токены OAuth в профилях аутентификации.
  - Использование JSON откатывается к `stats`; `stats.cached` нормализуется в
    `cacheRead`.
- **OpenAI Codex**: токены OAuth в профилях аутентификации (`accountId` используется, если присутствует).
- **MiniMax**: API-ключ или профиль аутентификации MiniMax OAuth. OpenClaw рассматривает
  `minimax`, `minimax-cn` и `minimax-portal` как одну и ту же поверхность квоты MiniMax,
  предпочитает сохраненный MiniMax OAuth, если он присутствует, а иначе откатывается
  к `MINIMAX_CODE_PLAN_KEY`, `MINIMAX_CODING_API_KEY` или `MINIMAX_API_KEY`.
  Опрос использования выводит хост Coding Plan из `models.providers.minimax-portal.baseUrl`
  или `models.providers.minimax.baseUrl`, если он настроен, а иначе использует
  хост MiniMax CN.
  Необработанные поля MiniMax `usage_percent` / `usagePercent` означают **оставшуюся**
  квоту, поэтому OpenClaw инвертирует их перед отображением; поля на основе количества
  имеют приоритет, если присутствуют.
  - Метки окон coding-plan берутся из полей часов/минут провайдера, если
    присутствуют, затем откатываются к промежутку `start_time` / `end_time`.
  - Если эндпоинт coding-plan возвращает `model_remains`, OpenClaw предпочитает
    запись модели чата, выводит метку окна из временных меток, когда явные
    поля `window_hours` / `window_minutes` отсутствуют, и включает имя модели
    в метку плана.
- **Xiaomi MiMo**: API-ключ через env/config/auth store (`XIAOMI_API_KEY`).
- **z.ai**: API-ключ через env/config/auth store.
- **DeepSeek**: API-ключ через env/config/auth store (`DEEPSEEK_API_KEY`).
  OpenClaw вызывает эндпоинт баланса DeepSeek и показывает баланс, сообщенный провайдером,
  как текст вместо окна квоты с оставшимся процентом.

Использование скрывается, когда невозможно определить пригодную аутентификацию использования провайдера. Провайдеры
могут предоставлять логику аутентификации использования, специфичную для Plugin; иначе OpenClaw откатывается к
соответствующим учетным данным OAuth/API-ключа из профилей аутентификации, переменных окружения
или конфигурации.
