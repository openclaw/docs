OpenClaw стандартизирует временные метки, чтобы модель видела **единое эталонное время**, а не смесь локальных часов разных провайдеров. Часовые пояса отображаются в трёх местах, каждое из которых имеет своё назначение:

## Три места отображения часового пояса

| Место                 | Что отображается                                                                                                              | Значение по умолчанию                                      | Способ настройки                                       |
| --------------------- | ----------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------- | ------------------------------------------------------ |
| Оболочки сообщений    | Обрамляют входящие сообщения каналов: `[Signal +1555 Sun 2026-01-18 00:19:42 PST] hello`                                      | Локальный часовой пояс хоста                               | `agents.defaults.envelopeTimezone`                     |
| Данные инструментов   | Инструменты каналов типа `readMessages` возвращают исходное время провайдера и нормализованные `timestampMs` / `timestampUtc` | Поля UTC присутствуют всегда                               | Не настраивается; сохраняет исходные временные метки провайдера |
| Системный промпт      | Небольшой блок `Текущие дата и время` **только с часовым поясом** (без значения времени для стабильности кэша)                 | Часовой пояс хоста, если `userTimezone` не задан           | `agents.defaults.userTimezone`                         |

Текущее время намеренно не включается в системный промпт, чтобы кэширование промпта оставалось стабильным между запросами. Когда агенту требуется текущее время, он вызывает `session_status`.

## Настройка часового пояса пользователя

```json5
{
  agents: {
    defaults: {
      userTimezone: "America/Chicago",
    },
  },
}
```

Если `userTimezone` не задан, OpenClaw определяет часовой пояс хоста во время выполнения с помощью `Intl.DateTimeFormat().resolvedOptions().timeZone` (без записи в конфигурацию). `agents.defaults.timeFormat` (`auto` | `12` | `24`) управляет отображением времени в 12- или 24-часовом формате в оболочках и последующих местах вывода, но не в разделе системного промпта.

## Значения часового пояса оболочки

`agents.defaults.envelopeTimezone` принимает следующие значения:

- `"local"` (по умолчанию) или `"host"` — часовой пояс хост-машины.
- `"utc"` или `"gmt"` — UTC.
- `"user"` — определённый часовой пояс `agents.defaults.userTimezone` (если он не задан, используется часовой пояс хоста).
- Любая явно указанная строка часового пояса IANA, например `"Europe/Vienna"`.

## Когда следует переопределять

- **Используйте `"utc"`** для единообразных временных меток на хостах в разных регионах или для соответствия диагностике и журналам, использующим UTC.
- **Используйте `"user"`**, чтобы часовой пояс оболочек соответствовал настроенному часовому поясу пользователя независимо от часового пояса хоста Gateway.
- **Используйте фиксированный часовой пояс IANA**, если хост Gateway находится в одном часовом поясе, а оболочка должна всегда отображать время в другом независимо от миграции хоста.
- **Установите `envelopeTimestamp: "off"`**, если временной контекст не нужен для диалога. Это удаляет абсолютные временные метки из оболочек, префиксов прямых промптов агента и встроенных префиксов входных данных модели.
