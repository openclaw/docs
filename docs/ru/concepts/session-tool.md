OpenClaw предоставляет агентам инструменты для работы между сессиями, проверки статуса и
оркестрации субагентов.

## Доступные инструменты

| Инструмент         | Что он делает                                                              |
| ------------------ | --------------------------------------------------------------------------- |
| `sessions_list`    | Выводит список сессий с необязательными фильтрами (тип, метка, агент, архив, предпросмотр) |
| `sessions_history` | Читает расшифровку конкретной сессии                                       |
| `sessions_send`    | Отправляет сообщение в другую сессию и при необходимости ожидает ответа    |
| `sessions_spawn`   | Запускает изолированную сессию субагента для фоновой работы                |
| `sessions_yield`   | Завершает текущий ход и ожидает последующие результаты от субагента        |
| `subagents`        | Выводит статус запущенных субагентов для этой сессии                       |
| `session_status`   | Показывает карточку в стиле `/status` и при необходимости задает переопределение модели для сессии |

На эти инструменты по-прежнему распространяются активный профиль инструментов и
политика разрешений/запретов. `tools.profile: "coding"` включает полный набор
оркестрации сессий, включая `sessions_spawn`, `sessions_yield` и `subagents`.
`tools.profile: "messaging"` включает инструменты обмена сообщениями между сессиями
(`sessions_list`, `sessions_history`, `sessions_send`, `session_status`), но
не включает запуск субагентов. Чтобы сохранить профиль обмена сообщениями и при этом
разрешить нативное делегирование, добавьте:

```json5
{
  tools: {
    profile: "messaging",
    alsoAllow: ["sessions_spawn", "sessions_yield", "subagents"],
  },
}
```

Политики групп, провайдеров, песочницы и отдельных агентов все еще могут удалить эти инструменты
после этапа профиля. Используйте `/tools` из затронутой сессии, чтобы проверить
эффективный список инструментов.

## Просмотр списка и чтение сессий

`sessions_list` возвращает сессии с их ключом, agentId, типом, каналом, моделью,
количеством токенов и временными метками. Фильтруйте по типу (`main`, `group`, `cron`, `hook`,
`node`), точной `label`, точному `agentId`, тексту поиска или давности
(`activeMinutes`). Активные сессии возвращаются по умолчанию; передайте `archived: true`,
чтобы просмотреть архивные сессии. Строки включают их закрепленное и архивное состояние. Когда
нужна сортировка в стиле почтового ящика, инструмент также может запросить
производный заголовок с учетом области видимости, фрагмент предпросмотра последнего сообщения или ограниченные недавние
сообщения в каждой строке. Производные заголовки и предпросмотры создаются только для сессий,
которые вызывающий уже может видеть согласно настроенной политике видимости инструментов сессий, поэтому
несвязанные сессии остаются скрытыми. Когда видимость ограничена, `sessions_list`
возвращает необязательные метаданные `visibility`, показывающие эффективный режим и предупреждение о том, что
результаты могут быть ограничены областью.

`sessions_history` извлекает расшифровку разговора для конкретной сессии.
По умолчанию результаты инструментов исключены -- передайте `includeTools: true`, чтобы увидеть их.
Используйте `limit` для новейшего ограниченного хвоста. Передайте `offset: 0`, когда нужны
метаданные пагинации, затем передавайте возвращенные значения `nextOffset`, чтобы постранично двигаться назад
по более старым окнам расшифровки OpenClaw без чтения сырых файлов расшифровки.
Явные страницы смещения не объединяют внешние резервные импорты CLI; используйте
представление новейшего хвоста по умолчанию, когда нужна эта объединенная история отображения.
Возвращаемое представление намеренно ограничено и фильтруется для безопасности:

- текст assistant нормализуется перед извлечением:
  - теги размышлений удаляются
  - служебные блоки `<relevant-memories>` / `<relevant_memories>` удаляются
  - XML-блоки полезной нагрузки вызовов инструментов в обычном тексте, такие как `<tool_call>...</tool_call>`,
    `<function_call>...</function_call>`, `<tool_calls>...</tool_calls>` и
    `<function_calls>...</function_calls>`, удаляются, включая усеченные
    полезные нагрузки, которые не закрылись корректно
  - пониженные служебные конструкции вызовов/результатов инструментов, такие как `[Tool Call: ...]`,
    `[Tool Result ...]` и `[Historical context ...]`, удаляются
  - утекшие управляющие токены модели, такие как `<|assistant|>`, другие ASCII
    токены `<|...|>` и полноширинные варианты `<｜...｜>` удаляются
  - некорректный XML вызова инструмента MiniMax, такой как `<invoke ...>` /
    `</minimax:tool_call>`, удаляется
- текст, похожий на учетные данные или токены, редактируется перед возвратом
- длинные текстовые блоки усекаются
- очень большие истории могут отбрасывать старые строки или заменять слишком большую строку на
  `[sessions_history omitted: message too large]`
- инструмент сообщает сводные флаги, такие как `truncated`, `droppedMessages`,
  `contentTruncated`, `contentRedacted`, `bytes`, а также метаданные пагинации

Оба инструмента принимают либо **ключ сессии** (например, `"main"`), либо **ID сессии**
из предыдущего вызова списка.

Если нужна точная побайтовая расшифровка, проверьте файл расшифровки на
диске, а не рассматривайте `sessions_history` как сырой дамп.

## Отправка сообщений между сессиями

`sessions_send` доставляет сообщение в другую сессию и при необходимости ожидает
ответ:

- **Отправить и не ждать:** задайте `timeoutSeconds: 0`, чтобы поставить сообщение в очередь и вернуться
  немедленно.
- **Ждать ответа:** задайте тайм-аут и получите ответ встроенно.

Чат-сессии, привязанные к треду, например ключи Slack или Discord, заканчивающиеся на
`:thread:<id>`, не являются допустимыми целями `sessions_send`. Используйте ключ сессии родительского канала
для координации между агентами, чтобы сообщения, маршрутизируемые инструментами, не появлялись
в активном треде, обращенном к человеку.

Сообщения и последующие ответы A2A помечаются как межсессионные данные во
входящем промпте (`[Inter-session message ... isUser=false]`) и в происхождении
расшифровки. Принимающий агент должен рассматривать их как данные, маршрутизированные инструментом, а не как
прямую инструкцию, написанную конечным пользователем.

После ответа целевой сессии OpenClaw может запустить **цикл ответов назад**, в котором
агенты поочередно обмениваются сообщениями (до `session.agentToAgent.maxPingPongTurns`, диапазон
0-20, по умолчанию 5). Целевой агент может ответить
`REPLY_SKIP`, чтобы остановиться раньше.

## Помощники статуса и оркестрации

`session_status` — легковесный инструмент, эквивалентный `/status`, для текущей
или другой видимой сессии. Он сообщает использование, время, состояние модели/среды выполнения и
связанный контекст фоновой задачи, если он есть. Как и `/status`, он может дозаполнять
разреженные счетчики токенов/кэша из последней записи использования в расшифровке, а
`model=default` очищает переопределение для сессии. Используйте `sessionKey="current"` для
текущей сессии вызывающего; видимые клиентские метки, такие как `openclaw-tui`, не являются
ключами сессий.

Когда доступны метаданные маршрута, `session_status` также включает видимый
JSON-блок `Route context` и соответствующие структурированные поля `details`. Эти
поля различают ключ сессии и маршрут, который сейчас обрабатывает
живой запуск:

- `origin` — место, где была создана сессия, или провайдер, выведенный из
  префикса доставляемого ключа сессии, когда в старом состоянии нет сохраненных метаданных origin.
- `active` — текущий маршрут живого запуска. Он сообщается только для живой или
  текущей сессии, обрабатываемой сейчас.
- `deliveryContext` — сохраненный маршрут доставки, хранящийся в сессии,
  который OpenClaw может повторно использовать для последующей доставки, даже когда активная поверхность
  отличается.

`sessions_yield` намеренно завершает текущий ход, чтобы следующим сообщением могло быть
последующее событие, которого вы ждете. Используйте его после запуска субагентов, когда
хотите, чтобы результаты завершения пришли следующим сообщением, а не строить
циклы опроса.

`subagents` — помощник видимости для уже запущенных субагентов OpenClaw.
Он поддерживает `action: "list"` для проверки активных/недавних запусков.

## Запуск субагентов

`sessions_spawn` по умолчанию создает изолированную сессию для фоновой задачи.
Он всегда неблокирующий -- он немедленно возвращает `runId` и
`childSessionKey`. Нативные запуски субагентов получают делегированную задачу в
первом видимом сообщении `[Subagent Task]` дочерней сессии, а системный
промпт содержит только правила среды выполнения субагента и контекст маршрутизации.

Ключевые параметры:

- `runtime: "subagent"` (по умолчанию) или `"acp"` для внешних агентов обвязки.
- Переопределения `model` и `thinking` для дочерней сессии.
- `thread: true` для привязки запуска к чат-треду (Discord, Slack и т. д.).
- `sandbox: "require"` для принудительного применения песочницы к дочерней сессии.
- `context: "fork"` для нативных субагентов, когда дочерней сессии нужна текущая
  расшифровка запрашивающего; опустите его или используйте `context: "isolated"` для чистой дочерней сессии.
  Нативные субагенты, привязанные к треду, по умолчанию используют `context: "fork"`, если
  `threadBindings.defaultSpawnContext` не задает иное.

Листовые субагенты по умолчанию не получают инструменты сессий. Когда
`maxSpawnDepth >= 2`, субагенты-оркестраторы глубины 1 дополнительно получают
`sessions_spawn`, `subagents`, `sessions_list` и `sessions_history`, чтобы они
могли управлять собственными дочерними сессиями. Листовые запуски по-прежнему не получают рекурсивные
инструменты оркестрации.

После завершения этап объявления публикует результат в канал запрашивающего.
Доставка завершения сохраняет привязанную маршрутизацию треда/темы, когда она доступна, а если
источник завершения указывает только канал, OpenClaw все еще может повторно использовать
сохраненный маршрут сессии запрашивающего (`lastChannel` / `lastTo`) для прямой
доставки.

Поведение, специфичное для ACP, см. в разделе [Агенты ACP](/ru/tools/acp-agents).

## Видимость

Инструменты сессий ограничены областью, чтобы ограничить то, что агент может видеть:

| Уровень | Область                                  |
| ------- | ---------------------------------------- |
| `self`  | Только текущая сессия                    |
| `tree`  | Текущая сессия + запущенные субагенты    |
| `agent` | Все сессии для этого агента              |
| `all`   | Все сессии (между агентами, если настроено) |

По умолчанию используется `tree`. Сессии в песочнице принудительно ограничены до `tree` независимо от
конфигурации.
