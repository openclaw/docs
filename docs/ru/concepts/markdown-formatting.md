OpenClaw преобразует исходящий Markdown в общее промежуточное представление
(IR) перед формированием вывода для конкретного канала. IR хранит обычный текст,
а также диапазоны стилей и ссылок, поэтому результат одного этапа разбора используется
для всех каналов, а разбиение на фрагменты никогда не разделяет форматирование внутри диапазона.

## Конвейер

1. **Разбор Markdown в IR** (`markdownToIR`) — обычный текст и диапазоны стилей
   (полужирный, курсив, зачёркивание, код, блок кода, спойлер, цитата,
   заголовки уровней 1–6), а также диапазоны ссылок. Смещения задаются в кодовых единицах UTF-16,
   поэтому диапазоны стилей Signal непосредственно соответствуют его API. Таблицы разбираются,
   только если для канала включён режим таблиц.
2. **Разбиение IR на фрагменты** (`chunkMarkdownIR` / `renderMarkdownIRChunksWithinLimit`)
   — разделение выполняется по тексту IR до формирования вывода, поэтому встроенные стили и
   ссылки разделяются по фрагментам, а не разрываются на границе.
3. **Формирование вывода для каждого канала** (`renderMarkdownWithMarkers`) — карта маркеров
   стилей преобразует диапазоны в нативную разметку канала.

| Канал                                                            | Средство формирования                                                               | Примечания                                                                                         |
| ---------------------------------------------------------------- | ----------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------- |
| Slack                                                            | токены mrkdwn (`*bold*`, `_italic_`, `` `code` ``, блоки кода)                      | Ссылки преобразуются в `<url\|label>`; автоматические ссылки отключены при разборе во избежание дублирования |
| Telegram                                                         | HTML-теги (`<b>`, `<i>`, `<s>`, `<code>`, `<pre><code>`, `<a href>`, `<tg-spoiler>`) | Также поддерживает таблицы и заголовки (`<h1>`–`<h6>`) в форматированных сообщениях, когда включён `richMessages` |
| Signal                                                           | обычный текст и диапазоны `text-style`                                               | Ссылки отображаются как `label (url)`, если подпись отличается от URL                              |
| Discord, WhatsApp, iMessage, Microsoft Teams и другие каналы     | обычный текст                                                                        | Стилизация на основе IR отсутствует; преобразование таблиц Markdown по-прежнему выполняется через `convertMarkdownTables` |

## Обработка таблиц

`markdown.tables` определяет способ преобразования таблиц Markdown для каждого
канала и, при необходимости, для каждой учётной записи:

| Режим    | Поведение                                                                                       |
| -------- | ------------------------------------------------------------------------------------------------ |
| `code`   | Отображать как выровненную ASCII-таблицу внутри блока кода (резервное значение по умолчанию)     |
| `bullets` | Преобразовывать каждую строку в пункты списка вида `label: value`                                |
| `block`  | Сохранять нативные таблицы там, где транспорт их поддерживает; иначе использовать режим `code`   |
| `off`    | Отключить разбор таблиц; исходный текст таблицы передаётся без изменений                          |

Значения по умолчанию для Plugin каждого канала: Signal, WhatsApp и Matrix по умолчанию
используют `bullets`; Mattermost — `off`; Telegram — `block` (который
преобразуется в `code`, если для учётной записи не включён `richMessages`). Для любого
канала без явно заданного значения Plugin используется `code`.
__OC_I18N_900002__
## Правила разбиения на фрагменты

- Ограничения размера фрагментов задаются адаптерами или конфигурацией каналов и применяются
  к тексту IR, а не к сформированному выводу.
- Ограждённые блоки кода сохраняются как единый блок с завершающим переводом строки, чтобы
  каналы правильно отображали закрывающее ограждение.
- Префиксы списков и цитат являются частью текста IR, поэтому разбиение никогда
  не происходит внутри префикса.
- Встроенные стили никогда не разделяются между фрагментами; средство формирования повторно открывает
  незакрытый стиль в начале следующего фрагмента.

Сведения о границах фрагментов и поведении доставки в разных каналах см. в разделе
[Потоковая передача и разбиение на фрагменты](/concepts/streaming).

## Политика обработки ссылок

- **Slack:** `[label](url)` -> `<url|label>`; URL без подписи остаются без изменений.
- **Telegram:** `[label](url)` -> `<a href="url">label</a>` (режим разбора HTML).
- **Signal:** `[label](url)` -> `label (url)`, если подпись не
  совпадает с URL.

## Спойлеры

Маркеры спойлеров (`||spoiler||`) разбираются для Signal (преобразуются в диапазоны
стиля `SPOILER`) и Telegram (преобразуются в `<tg-spoiler>`). Другие каналы обрабатывают
`||...||` как обычный текст.

## Добавление или обновление средства форматирования канала

1. **Выполните разбор один раз** с помощью `markdownToIR(...)`, передав подходящие для канала
   параметры (`autolink`, `headingStyle`, `blockquotePrefix`, `tableMode`).
2. **Сформируйте вывод** с помощью `renderMarkdownWithMarkers(...)` и карты маркеров стилей (или
   пользовательской логики диапазонов стилей для таких транспортов, как Signal).
3. **Разбейте на фрагменты** с помощью `chunkMarkdownIR(...)` или
   `renderMarkdownIRChunksWithinLimit(...)` перед формированием каждого фрагмента.
4. **Подключите адаптер**, чтобы новый механизм разбиения и средство формирования вызывались
   из пути исходящей отправки.
5. **Протестируйте** форматирование и, если канал разбивает сообщения на фрагменты,
   исходящую доставку.

## Распространённые проблемы

- Токены Slack в угловых скобках (`<@U123>`, `<#C123>`, `<https://...>`) должны
  сохраняться при экранировании; необработанный HTML всё равно необходимо безопасно экранировать.
- Для HTML в Telegram необходимо экранировать текст вне тегов, чтобы избежать повреждения разметки.
- Диапазоны стилей Signal используют смещения UTF-16, а не смещения по кодовым точкам.
- Сохраняйте завершающие переводы строк в ограждённых блоках кода, чтобы закрывающий маркер
  находился на отдельной строке.
