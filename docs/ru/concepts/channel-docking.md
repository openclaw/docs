Стыковка каналов — это переадресация вызовов для одного сеанса OpenClaw. Она сохраняет тот же
контекст разговора, но меняет место доставки будущих ответов для этого сеанса.
Стыковка работает только из личного чата; из группового чата она не выполняется.

## Пример

Алиса может отправлять сообщения OpenClaw в Telegram и Discord:

```json5
{
  session: {
    identityLinks: {
      alice: ["telegram:123", "discord:456"],
    },
  },
}
```

Если Алиса отправит следующую команду из личного чата Telegram:

```text
/dock_discord
```

OpenClaw сохранит контекст текущего сеанса и изменит маршрут ответов:

| До стыковки                         | После `/dock_discord`              |
| ----------------------------------- | ---------------------------------- |
| Ответы поступают в Telegram `123`   | Ответы поступают в Discord `456`   |

Сеанс не создаётся заново. История переписки остаётся привязанной к тому же
сеансу.

## Зачем это использовать

Используйте стыковку, если задача начинается в одном приложении для общения, а
следующие ответы должны поступать в другое.

Типичный порядок действий:

1. Запустите задачу агента из Telegram.
2. Перейдите в Discord, где вы координируете работу.
3. Отправьте `/dock_discord` из личного чата Telegram.
4. Продолжайте использовать тот же сеанс OpenClaw, но получайте последующие ответы в Discord.

## Обязательная конфигурация

Для стыковки требуется `session.identityLinks`. Исходный отправитель и целевой
собеседник должны находиться в одной группе идентичности:

```json5
{
  session: {
    identityLinks: {
      alice: ["telegram:123", "discord:456", "slack:U123"],
    },
  },
}
```

Значения представляют собой идентификаторы собеседников с префиксом канала:

| Значение       | Смысл                                      |
| -------------- | ------------------------------------------ |
| `telegram:123` | Идентификатор отправителя Telegram `123`   |
| `discord:456`  | Идентификатор собеседника Discord `456`    |
| `slack:U123`   | Идентификатор пользователя Slack `U123`    |

Канонический ключ (`alice` выше) — это только общее имя группы идентичности.
Команды стыковки используют значения с префиксом канала, чтобы подтвердить, что
исходный отправитель и целевой собеседник — один и тот же человек.

## Команды

OpenClaw создаёт одну команду `/dock-<channel>` для каждого загруженного Plugin
канала, поддерживающего нативные команды, поэтому список расширяется по мере
добавления Plugin. Встроенные Plugin, которые поддерживают эту функцию в
настоящее время:

| Целевой канал | Команда            | Псевдоним          |
| ------------- | ------------------ | ------------------ |
| Discord       | `/dock-discord`    | `/dock_discord`    |
| Mattermost    | `/dock-mattermost` | `/dock_mattermost` |
| Slack         | `/dock-slack`      | `/dock_slack`      |
| Telegram      | `/dock-telegram`   | `/dock_telegram`   |

Форма с подчёркиванием также используется в качестве имени нативной команды на
платформах вроде Telegram, которые предоставляют прямой доступ к командам с
косой чертой.

## Что изменяется

Стыковка обновляет поля доставки активного сеанса:

| Поле сеанса    | Пример после `/dock_discord`                  |
| -------------- | --------------------------------------------- |
| `lastChannel`  | `discord`                                     |
| `lastTo`       | `456`                                         |
| `lastAccountId`| учётная запись целевого канала или `default`  |

Эти поля сохраняются в хранилище сеансов и используются для доставки последующих
ответов в рамках этого сеанса.

## Что не изменяется

Стыковка не выполняет следующие действия:

- не создаёт учётные записи каналов
- не подключает нового бота Discord, Telegram, Slack или Mattermost
- не предоставляет пользователю доступ
- не обходит списки разрешённых пользователей каналов или политики личных сообщений
- не переносит историю переписки в другой сеанс
- не объединяет сеансы не связанных между собой пользователей

Она изменяет только маршрут доставки для текущего сеанса.

## Устранение неполадок

**Команда сообщает, что отправитель не связан.**

Добавьте текущего отправителя и целевого собеседника в одну группу
`session.identityLinks`. Например, если отправителя Telegram `123` нужно
состыковать с собеседником Discord `456`, добавьте и `telegram:123`, и
`discord:456`.

**Команда сообщает, что стыковка доступна только из личных чатов.**

Отправьте команду стыковки из личного чата с OpenClaw, а не из группового чата.

**Команда сообщает, что активного сеанса нет.**

Выполняйте стыковку из существующего сеанса личного чата. Команде нужна запись
активного сеанса, чтобы сохранить новый маршрут.

**Ответы по-прежнему поступают в старый канал.**

Убедитесь, что команда вернула сообщение об успешном выполнении, и проверьте,
что идентификатор целевого собеседника совпадает с идентификатором, используемым
этим каналом. Стыковка изменяет только маршрут активного сеанса; другой сеанс
может по-прежнему использовать иной маршрут.

**Мне нужно переключиться обратно.**

Отправьте соответствующую команду исходного канала, например `/dock_telegram`
или `/dock-telegram`, от связанного отправителя.
