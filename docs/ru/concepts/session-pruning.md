Обрезка сеанса удаляет **старые результаты инструментов** из контекста перед каждым вызовом LLM. Она уменьшает разрастание контекста из-за накопленных результатов инструментов (результатов выполнения команд, чтения файлов, результатов поиска), не изменяя обычный текст беседы.

<Info>
Обрезка выполняется только в памяти — она не изменяет хранящуюся на диске расшифровку сеанса. Полная история всегда сохраняется.
</Info>

## Почему это важно

В длительных сеансах накапливаются результаты инструментов, увеличивающие окно контекста. Это повышает стоимость и может потребовать [Compaction](/ru/concepts/compaction) раньше, чем необходимо.

Обрезка особенно полезна для **кэширования промптов Anthropic**. После истечения TTL кэша следующий запрос повторно кэширует весь промпт. Обрезка уменьшает объём записи в кэш, напрямую снижая стоимость.

## Как это работает

Обрезка выполняется в режиме `cache-ttl` при одновременном выполнении условий проверки времени и размера контекста:

1. Дождаться истечения TTL кэша (по умолчанию 5 минут при ручной настройке; автоматическое значение по умолчанию для Anthropic см. в разделе [Интеллектуальные значения по умолчанию](#smart-defaults)). До истечения TTL обрезка полностью пропускается, чтобы сохранить повторное использование кэша промпта для близких по времени ходов.
2. После истечения TTL оценить общий размер контекста относительно окна контекста модели. Если отношение ниже `softTrimRatio` (по умолчанию 0,3), пропустить обрезку и не сбрасывать отсчёт TTL.
3. Выполнить **мягкую обрезку** слишком больших результатов инструментов, превышающих порог отношения: сохранить начало и конец (по умолчанию по 1500 символов, но не более 4000 символов в сумме), вставив между ними `...`.
4. Если отношение всё ещё не ниже `hardClearRatio` (по умолчанию 0,5) и остаётся не менее `minPrunableToolChars` (по умолчанию 50 000) символов содержимого инструментов, доступного для обрезки, выполнить **полную очистку** этих результатов: заменить их содержимое заполнителем (по умолчанию `[Содержимое старого результата инструмента очищено]`).
5. Сбрасывать отсчёт TTL только тогда, когда обрезка фактически изменила контекст, чтобы последующие запросы повторно использовали новый кэш.

Независимо от пороговых значений действуют два правила безопасности: последние `keepLastAssistants` ходов ассистента (по умолчанию 3) никогда не обрезаются, а содержимое до первого сообщения пользователя в сеансе никогда не обрезается (это защищает начальные чтения таких файлов, как `SOUL.md`/`USER.md`).

Обрезке подлежат только сообщения `toolResult`; обычный текст беседы остаётся без изменений. Используйте `agents.defaults.contextPruning.tools.{allow,deny}`, чтобы определить, результаты каких инструментов можно обрезать.

## Очистка устаревших изображений

OpenClaw также создаёт отдельное идемпотентное представление для повторного воспроизведения сеансов, в истории которых сохраняются необработанные блоки изображений или маркеры медиафайлов для гидратации промпта.

- Оно побайтово сохраняет **3 последних завершённых хода**, чтобы префиксы кэша промпта для недавних последующих запросов оставались стабильными. Учитываются все завершённые ходы, а не только содержащие изображения, поэтому ходы только с текстом также занимают это окно.
- В представлении повторного воспроизведения старые уже обработанные блоки изображений из истории `user` или `toolResult` заменяются на `[данные изображения удалены — уже обработаны моделью]`.
- Старые текстовые ссылки на медиафайлы, такие как `[media attached: ...]`, `[Image: source: ...]` и `media://inbound/...`, заменяются на `[ссылка на медиафайл удалена — уже обработана моделью]`. Маркеры вложений текущего хода остаются без изменений, чтобы модели с поддержкой компьютерного зрения могли по-прежнему гидратировать новые изображения.
- Исходная расшифровка сеанса не перезаписывается, поэтому средства просмотра истории по-прежнему могут отображать исходные записи сообщений и их изображения.
- Этот механизм отделён от обычной обрезки по TTL кэша, описанной выше. Он предотвращает нарушение кэшей промптов в последующих ходах из-за повторяющихся данных изображений или устаревших ссылок на медиафайлы.

## Интеллектуальные значения по умолчанию

Встроенный Plugin Anthropic автоматически настраивает обрезку и периодичность Heartbeat при первом разрешении профиля аутентификации Anthropic (или Claude CLI), но только для полей, которые вы ещё не задали явно:

| Режим аутентификации                              | `contextPruning.mode` | `contextPruning.ttl` | `heartbeat.every` |
| ------------------------------------------------- | --------------------- | -------------------- | ----------------- |
| OAuth/токен (включая повторное использование Claude CLI) | `cache-ttl`           | `1h`                 | `1h`              |
| Ключ API                                          | `cache-ttl`           | `1h`                 | `30m`             |

Если вы самостоятельно зададите `agents.defaults.contextPruning.mode` или `agents.defaults.heartbeat.every`, OpenClaw не переопределит их. Это автоматическое значение по умолчанию применяется только к аутентификации семейства Anthropic; для других поставщиков обрезка остаётся `off`, пока вы её не настроите.

## Включение и отключение

Для поставщиков, отличных от Anthropic, обрезка по умолчанию отключена. Чтобы включить её:

```json5
{
  agents: {
    defaults: {
      contextPruning: { mode: "cache-ttl", ttl: "5m" },
    },
  },
}
```

Чтобы отключить её, задайте `mode: "off"`.

## Обрезка и Compaction

|                | Обрезка                              | Compaction                |
| -------------- | ------------------------------------ | ------------------------- |
| **Что делает** | Обрезает результаты инструментов     | Резюмирует беседу         |
| **Сохраняется?** | Нет (для каждого запроса отдельно) | Да (в расшифровке)        |
| **Область действия** | Только результаты инструментов | Вся беседа                |

Эти механизмы дополняют друг друга: обрезка уменьшает объём результатов инструментов между циклами Compaction.
