Экспериментальные функции — это предварительные возможности, включаемые явно с помощью специальных флагов. Им требуется более длительная проверка в реальных условиях, прежде чем они получат стабильные настройки по умолчанию или долгосрочные гарантии совместимости.

- По умолчанию отключены, если в документации прямо не указано включить конкретную функцию.
- Структура и поведение могут меняться быстрее, чем у стабильной конфигурации.
- Если уже существует стабильный способ, отдавайте предпочтение ему.
- Широко внедряйте функцию только после предварительного тестирования в меньшей среде.

## Облегчённый режим локальных моделей

`agents.defaults.experimental.localModelLean: true` при каждом ходе исключает ресурсоёмкие необязательные инструменты из непосредственно доступного агенту набора: `browser`, `cron`, `message`, `image_generate`, `music_generate`, `video_generate`, `tts` и `pdf`. Явно разрешённые или необходимые для доставки инструменты остаются доступными, хотя Tool Search может включать их в каталог, а не предоставлять напрямую. Кроме того, в облегчённом режиме для каталогов плагинов, MCP и клиентов по умолчанию используется структурированный Tool Search (`tool_search`, `tool_describe`, `tool_call`), если параметр `tools.toolSearch` ещё не задан. Используйте `agents.list[].experimental.localModelLean`, чтобы ограничить эту настройку одним агентом.

Если Tool Search уже настроен глобально, OpenClaw не изменяет эту конфигурацию. Задайте `tools.toolSearch: false`, чтобы отказаться от включаемой облегчённым режимом настройки Tool Search по умолчанию.

В структурированном режиме `tools` при облегчённых запусках `exec` остаётся непосредственно видимым рядом с элементами управления Tool Search, чтобы локальные модели, оптимизированные для программирования, по-прежнему могли выбирать привычный им способ работы с оболочкой. Это изменяет только видимость схемы: обычная политика инструментов, изоляция в песочнице и подтверждения выполнения продолжают действовать. Явные режимы `code` и `directory` сохраняют обычное поведение Compaction.

### Почему выбраны эти инструменты

У этих инструментов самые объёмные описания, наиболее широкие наборы параметров или наибольшая вероятность отвлечь небольшую модель от обычной работы с кодом и диалогом. При использовании серверной части с небольшим контекстом или более строгой совместимостью с OpenAI это определяет разницу между следующими результатами:

- Схемы инструментов помещаются в запрос или вытесняют историю диалога.
- Модель выбирает правильный инструмент или создаёт некорректные вызовы инструментов из-за слишком большого количества похожих схем.
- Адаптер Chat Completions укладывается в ограничения структурированного вывода или получает ошибку 400 из-за размера данных вызова инструмента.

Их удаление лишь сокращает непосредственно доступный список инструментов. Модели по-прежнему доступны `read`, `write`, `edit`, `exec`, `apply_patch`, распознавание изображений, поиск и получение данных из интернета (если настроено), память, а также инструменты сеансов и агентов. Дополнительные каталоги остаются доступными через Tool Search, если не задан параметр `tools.toolSearch: false`; явное разрешение инструментов позволяет облегчённому агенту снова использовать исключённый рабочий процесс.

### Когда следует включать

Включайте облегчённый режим, когда уже подтверждено, что модель может взаимодействовать с Gateway, но полные ходы агента выполняются неправильно:

1. `openclaw infer model run --gateway --model <ref> --prompt "Reply with exactly: pong"` завершается успешно.
2. Обычный ход агента завершается ошибкой из-за некорректных вызовов инструментов, слишком объёмных запросов или игнорирования инструментов моделью.
3. Переключение `localModelLean: true` устраняет ошибку.

### Когда следует оставить выключенным

Если серверная часть корректно обрабатывает полную среду выполнения по умолчанию, не включайте этот режим. Это обходное решение для локальных стеков, которым требуется меньший набор инструментов, а не настройка по умолчанию для размещённых моделей или локальных систем с достаточными ресурсами.

Облегчённый режим не заменяет `tools.profile`, `tools.allow`/`tools.deny` или резервный механизм модели `compat.supportsTools: false`. Чтобы на постоянной основе ограничить набор инструментов конкретного агента, используйте эти стабильные параметры.

### Включение

```json5
{
  agents: {
    defaults: {
      experimental: {
        localModelLean: true,
      },
    },
  },
}
```

Только для одного агента:

```json5
{
  agents: {
    list: [
      {
        id: "local",
        model: "lmstudio/gemma-4-e4b-it",
        experimental: {
          localModelLean: true,
        },
      },
    ],
  },
}
```

После изменения флага перезапустите Gateway. Фильтрация облегчённого режима удаляет `browser`, `cron`, `message`, `image_generate`, `music_generate`, `video_generate`, `tts` и `pdf`, если явно не сохранить их с помощью `tools.allow` или `tools.alsoAllow`; Tool Search при этом может включать сохранённые инструменты в каталог, а не предоставлять их напрямую.

## Экспериментальный не означает скрытый

Экспериментальный статус функции должен быть явно указан в документации и непосредственно в пути конфигурации, а не скрыт за параметром по умолчанию, выглядящим стабильным.
