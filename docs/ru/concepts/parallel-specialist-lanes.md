Параллельные специализированные линии позволяют одному Gateway направлять разные чаты или комнаты разным агентам, сохраняя высокую скорость взаимодействия с пользователем. Рассматривайте параллелизм как задачу проектирования с учетом дефицитных ресурсов, а не просто как «больше агентов».

## Основные принципы

Специализированная линия повышает пропускную способность, только если снижает конкуренцию за реальные узкие места:

- **Блокировки сеансов**: одновременно изменять определенный сеанс должен только один запуск.
- **Общая емкость модели**: все видимые запуски чатов по-прежнему совместно используют лимиты поставщика.
- **Емкость инструментов**: работа с оболочкой, браузером, сетью и репозиторием может занимать больше времени, чем сам ход модели.
- **Бюджет контекста**: длинные стенограммы замедляют каждый последующий ход и снижают его сфокусированность.
- **Неоднозначность ответственности**: агенты, дублирующие одну и ту же работу, впустую расходуют ресурсы.

OpenClaw уже выполняет запуски последовательно в пределах каждого сеанса и ограничивает общий параллелизм с помощью [очереди команд](/ru/concepts/queue). Специализированные линии добавляют к этому правила: какой агент отвечает за какую работу, что остается в чате, а что становится фоновой задачей.

## Рекомендуемый порядок внедрения

### Этап 1: контракты линий и ресурсоемкая работа в фоне

Задайте для каждой линии письменный контракт в ее рабочем пространстве и системной инструкции:

- **Назначение**: работа, за которую отвечает эта линия.
- **Что не входит в задачи**: работа, которую следует передать, а не выполнять самостоятельно.
- **Бюджет чата**: на быстрые вопросы отвечайте в чате; для длительных задач сначала отправьте краткое подтверждение, а затем запустите фонового субагента или фоновую задачу.
- **Правило передачи**: если за работу отвечает другая линия, укажите, куда ее следует направить, и предоставьте краткое резюме для передачи.
- **Правило риска инструментов**: отдавайте предпочтение минимальному набору инструментов, достаточному для выполнения задачи.

Это самый дешевый этап, устраняющий большинство заторов: одна задача по программированию больше не превращает исследовательскую линию в патоку, а каждый чат сохраняет чистоту собственного контекста.

### Этап 2: управление приоритетами и параллелизмом

Настройте очередь и емкость модели в соответствии с ценностью каждой линии для бизнеса:

```json5
{
  agents: {
    defaults: {
      maxConcurrent: 4,
      subagents: { maxConcurrent: 8, delegationMode: "prefer" },
    },
  },
  messages: {
    queue: {
      mode: "collect",
      debounceMs: 1000,
      cap: 20,
      drop: "summarize",
    },
  },
}
```

Используйте личные чаты и агентов производственной эксплуатации для высокоприоритетной работы. Когда система занята, переносите исследования, подготовку текстов и пакетное программирование в фоновые задачи.

### Этап 3: координатор / диспетчер трафика

Когда активно несколько линий, добавьте небольшой шаблон координатора:

- Отслеживайте активные задачи линий и ответственных за них.
- Выявляйте дублирующиеся запросы в разных группах.
- Передавайте между линиями краткие резюме.
- Выводите только сведения о препятствиях, завершенные результаты и решения, которые должен принять человек.

Не начинайте с этого этапа. Координатор без контрактов линий лишь координирует хаос.

## Минимальный шаблон контракта линии

```md
# Контракт линии

## Отвечает за

- <задача, за которую отвечает эта линия>

## Не отвечает за

- <работа, которую следует передать>

## Бюджет чата

- Отвечайте на простые вопросы напрямую.
- Для многоэтапной, медленной или требующей активного использования инструментов работы: кратко подтвердите запрос, запустите работу через субагента или в фоновом режиме, а после завершения верните результат.

## Передача

Если за запрос отвечает другая линия, укажите:

- целевую линию
- цель
- релевантный контекст
- точное следующее действие

## Подход к инструментам

Используйте минимальный набор инструментов, достаточный для выполнения задачи. Избегайте обширной работы с оболочкой или сетью, если эта линия не отвечает за нее явно.
```
