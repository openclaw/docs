Линия QA для Matrix запускает встроенный плагин `@openclaw/matrix` с одноразовым домашним сервером Tuwunel в Docker, используя временные учётные записи драйвера, тестируемой системы (SUT) и наблюдателя, а также предварительно созданные комнаты. Она обеспечивает проверку Matrix в реальном транспортном окружении.

Инструменты только для сопровождающих. В пакетных выпусках OpenClaw отсутствует `qa-lab`, поэтому `openclaw qa` запускается только из рабочей копии исходного кода, которая загружает встроенный исполнитель напрямую, без этапа установки плагина.

Общий контекст инфраструктуры QA см. в разделе [Обзор QA](/ru/concepts/qa-e2e-automation).

## Быстрый старт

```bash
pnpm openclaw qa matrix --profile fast --fail-fast
```

Обычный запуск `pnpm openclaw qa matrix` выполняет `--profile all` и не останавливается после первого сбоя. Распределяйте полный набор между параллельными заданиями с помощью `--profile transport|media|e2ee-smoke|e2ee-deep|e2ee-cli`.

## Что делает линия

1. Развёртывает одноразовый домашний сервер Tuwunel в Docker (образ по умолчанию — `ghcr.io/matrix-construct/tuwunel:v1.5.1`, имя сервера — `matrix-qa.test`, порт — `28008`) за ограниченным регистратором запросов и ответов, скрывающим конфиденциальные данные.
2. Регистрирует трёх временных пользователей: `driver` (отправляет входящий трафик), `sut` (тестируемая учётная запись OpenClaw в Matrix), `observer` (перехватывает сторонний трафик).
3. Предварительно создаёт комнаты, необходимые выбранным сценариям (основную, для веток обсуждения, медиафайлов, перезапуска, дополнительную, для списка разрешений, E2EE, личных сообщений проверки и т. д.).
4. Запускает не зависящую от среды выполнения проверку протокола `matrix-qa-v1` на записанной границе Tuwunel. Модульные тесты подтверждают контракт проверки с фикстурой протокола Matrix; канонический узел транспортного адаптера QA в [#99707](https://github.com/openclaw/openclaw/pull/99707) отвечает за реальное подключение целевых сред Crabline.
5. Запускает дочерний Gateway OpenClaw с реальным плагином Matrix, ограниченным учётной записью SUT.
6. Последовательно запускает сценарии, наблюдая события через клиенты Matrix драйвера и наблюдателя и определяя ожидаемые маршруты и состояния по записанному трафику.
7. Завершает работу домашнего сервера, записывает отчёт и артефакты подтверждения, затем завершает процесс.

## CLI

```text
pnpm openclaw qa matrix [options]
```

### Общие флаги

| Флаг                  | По умолчанию                                       | Описание                                                                                                                                   |
| --------------------- | --------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| `--profile <profile>` | `all`                                         | Профиль сценариев. См. [Профили](#profiles).                                                                                                  |
| `--fail-fast`         | выключено                                           | Остановиться после первой неуспешной проверки или первого неуспешного сценария.                                                                                                |
| `--scenario <id>`     | -                                             | Запустить только этот сценарий. Можно указывать многократно. См. [Сценарии](#scenarios).                                                                              |
| `--output-dir <path>` | `<repo>/.artifacts/qa-e2e/matrix-<timestamp>` | Каталог для записи отчётов, сводки, перечня маршрутов и состояний, наблюдаемых событий и журнала вывода. Относительные пути разрешаются относительно `--repo-root`. |
| `--repo-root <path>`  | `process.cwd()`                               | Корень репозитория при запуске из нейтрального рабочего каталога.                                                                               |
| `--sut-account <id>`  | `sut`                                         | Идентификатор учётной записи Matrix в конфигурации Gateway для QA.                                                                                               |

### Флаги провайдера

Линия использует реальный транспорт Matrix, но провайдер модели можно настроить:

| Флаг                     | По умолчанию          | Описание                                                                                                                               |
| ------------------------ | ---------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| `--provider-mode <mode>` | `live-frontier`  | `mock-openai` для детерминированной имитации диспетчеризации или `live-frontier` для реальных передовых провайдеров. Устаревший псевдоним `live-openai` по-прежнему работает. |
| `--model <ref>`          | значение провайдера по умолчанию | Ссылка на основную `provider/model`.                                                                                                             |
| `--alt-model <ref>`      | значение провайдера по умолчанию | Ссылка на альтернативную `provider/model`, на которую сценарии переключаются в ходе выполнения.                                                                            |
| `--fast`                 | выключено              | Включить быстрый режим провайдера там, где он поддерживается.                                                                                                |

QA для Matrix не принимает `--credential-source` или `--credential-role`. Линия создаёт одноразовых пользователей локально; общего пула учётных данных для аренды нет.

## Профили

| Профиль         | Назначение                                                                                                                                                                                                            |
| --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `all` (по умолчанию) | Полный каталог. Медленно, но исчерпывающе.                                                                                                                                                                                    |
| `fast`          | Подмножество для контроля выпуска, проверяющее императивный контракт реального транспорта: фильтрацию по упоминаниям, блокировку по списку разрешений, форму ответа, возобновление после перезапуска, наблюдение за реакциями, доставку метаданных одобрения выполнения и базовый ответ E2EE. |
| `transport`     | Сценарии транспорта для веток обсуждения, личных сообщений, комнат, автоматического присоединения, упоминаний и списков разрешений, одобрений и реакций.                                                                                                                   |
| `media`         | Проверка вложений: изображений, аудио, видео, PDF и EPUB.                                                                                                                                                                   |
| `e2ee-smoke`    | Минимальная проверка E2EE: базовый зашифрованный ответ, продолжение ветки обсуждения, успешная начальная настройка.                                                                                                                                    |
| `e2ee-deep`     | Исчерпывающие сценарии E2EE для потери состояния, резервного копирования, ключей и восстановления.                                                                                                                                                      |
| `e2ee-cli`      | Сценарии CLI `openclaw matrix encryption setup` и `verify *`, запускаемые через тестовую инфраструктуру QA.                                                                                                                        |

Точное сопоставление находится в `extensions/qa-matrix/src/runners/contract/scenario-catalog.ts`.

## Сценарии

Общий адаптер Matrix предоставляет следующие канонические сценарии YAML через `openclaw qa suite --channel-driver live --channel matrix`:

- `channel-chat-baseline`
- `thread-follow-up`
- `thread-isolation`
- `thread-reply-override`
- `dm-shared-session`
- `dm-per-room-session`

`subagent-thread-spawn` остаётся доступным при явном выборе `--scenario subagent-thread-spawn`,
но не входит в общий набор Matrix по умолчанию, пока подтверждение завершения дочернего процесса в реальной среде не станет стабильным.

Оставшийся список идентификаторов императивных сценариев представляет собой объединение `MatrixQaScenarioId` в `extensions/qa-matrix/src/runners/contract/scenario-catalog.ts`. Категории:

- ветки обсуждения: `matrix-thread-root-preservation`, `matrix-thread-nested-reply-shape`
- верхний уровень / личные сообщения / комнаты: `matrix-top-level-reply-shape`, `matrix-room-*`, `matrix-dm-*`
- потоковая передача и ход выполнения инструментов: `matrix-room-partial-streaming-preview`, `matrix-room-quiet-streaming-preview`, `matrix-room-tool-progress-*`, `matrix-room-block-streaming`
- медиафайлы: `matrix-media-type-coverage`, `matrix-room-image-understanding-attachment`, `matrix-attachment-only-ignored`, `matrix-unsupported-media-safe`
- маршрутизация: `matrix-room-autojoin-invite`, `matrix-secondary-room-*`
- реакции: `matrix-reaction-*`
- одобрения: `matrix-approval-*` (метаданные выполнения/плагина, резервная передача частями, реакции на отказ, ветки обсуждения и маршрутизация `target: "both"`)
- перезапуск и повторное воспроизведение: `matrix-restart-*`, `matrix-stale-sync-replay-dedupe`, `matrix-room-membership-loss`, `matrix-homeserver-restart-resume`, `matrix-initial-catchup-then-incremental`
- фильтрация по упоминаниям, взаимодействие между ботами и списки разрешений: `matrix-mention-*`, `matrix-allowbots-*`, `matrix-allowlist-*`, `matrix-multi-actor-ordering`, `matrix-inbound-edit-*`, `matrix-mxid-prefixed-command-block`, `matrix-observer-allowlist-override`
- E2EE: `matrix-e2ee-*` (базовый ответ, продолжение ветки обсуждения, начальная настройка, жизненный цикл ключа восстановления, варианты потери состояния, поведение серверного резервного копирования, гигиена устройств, проверка через SAS / QR / личные сообщения, перезапуск, сокрытие конфиденциальных данных в артефактах)
- CLI E2EE: `matrix-e2ee-cli-*` (настройка шифрования, идемпотентная настройка, сбой начальной настройки, жизненный цикл ключа восстановления, несколько учётных записей, полный цикл ответа Gateway, самопроверка)

Передайте `--scenario <id>` (можно указывать многократно), чтобы запустить выбранный вручную набор; сочетайте с `--profile all`, чтобы игнорировать ограничения профиля.

## Переменные окружения

| Переменная                              | Значение по умолчанию                     | Действие                                                                                                                                                                                       |
| --------------------------------------- | ----------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `OPENCLAW_QA_MATRIX_TIMEOUT_MS`         | `1800000` (30 мин)                        | Жёсткий верхний предел времени выполнения всего запуска.                                                                                                                                      |
| `OPENCLAW_QA_MATRIX_CANARY_TIMEOUT_MS`  | `45000`                                   | Ограничение времени для первоначального ответа канарейки. CI выпуска увеличивает его на общих раннерах, чтобы медленный первый цикл Gateway не завершился с ошибкой до начала выполнения сценариев. |
| `OPENCLAW_QA_MATRIX_NO_REPLY_WINDOW_MS` | `8000`                                    | Окно ожидания для отрицательных проверок отсутствия ответа. Ограничивается значением `<=` — тайм-аутом запуска.                                                                  |
| `OPENCLAW_QA_MATRIX_CLEANUP_TIMEOUT_MS` | `90000`                                   | Ограничение времени для остановки Docker. При сбое выводится в том числе команда восстановления `docker compose ... down --remove-orphans`.                                                                           |
| `OPENCLAW_QA_MATRIX_TUWUNEL_IMAGE`      | `ghcr.io/matrix-construct/tuwunel:v1.5.1` | Переопределяет образ домашнего сервера при проверке с другой версией Tuwunel.                                                                                                                   |
| `OPENCLAW_QA_MATRIX_PROGRESS`           | вкл.                                      | `0` отключает вывод строк хода выполнения `[matrix-qa] ...` в stderr. `1` принудительно включает их.                                                           |
| `OPENCLAW_QA_MATRIX_CAPTURE_CONTENT`    | отредактировано                           | `1` сохраняет тело сообщения и `formatted_body` в `matrix-qa-observed-events.json`. По умолчанию данные редактируются для безопасности артефактов CI.                                      |
| `OPENCLAW_QA_MATRIX_DISABLE_FORCE_EXIT` | выкл.                                     | `1` пропускает детерминированный `process.exit` после записи артефактов. По умолчанию выполняется принудительный выход, поскольку нативные криптографические дескрипторы matrix-js-sdk могут поддерживать цикл событий после завершения записи артефактов. |
| `OPENCLAW_RUN_NODE_OUTPUT_LOG`          | не задано                                 | Если значение задано внешней программой запуска (например, `scripts/run-node.mjs`), Matrix QA повторно использует этот путь к журналу вместо запуска собственного tee.                            |

## Выходные артефакты

Записываются в `--output-dir` (по умолчанию `<repo>/.artifacts/qa-e2e/matrix-<timestamp>`, чтобы последовательные запуски не перезаписывали друг друга):

- `matrix-qa-report.md`: отчёт о протоколе в формате Markdown (что прошло, завершилось с ошибкой, было пропущено и почему).
- `matrix-qa-summary.json`: структурированная сводка, подходящая для анализа в CI и панелей мониторинга.
- `matrix-qa-route-state-manifest.json`: динамический реестр `matrix-qa-v1` с ключами по идентификаторам сценариев. В нём записываются отредактированные структуры маршрутов и тел, порядок запросов, наблюдаемые повторные попытки, ошибки, непрерывность токенов синхронизации, а также семейства состояний устройств, ключей, медиафайлов и резервных копий, наблюдавшиеся во время этого запуска. Это исполняемое доказательство, а не зафиксированная в репозитории эталонная версия.
- `matrix-qa-observed-events.json`: наблюдаемые события Matrix от клиентов-драйверов и клиентов-наблюдателей. Тела редактируются, если не задано `OPENCLAW_QA_MATRIX_CAPTURE_CONTENT=1`; метаданные одобрения сводятся к выбранным безопасным полям и усечённому предварительному просмотру команды.
- `matrix-qa-output.log`: объединённые stdout/stderr запуска. Если задано `OPENCLAW_RUN_NODE_OUTPUT_LOG`, вместо этого повторно используется журнал внешней программы запуска.

## Советы по диагностике

- **Запуск зависает ближе к завершению:** нативные криптографические дескрипторы `matrix-js-sdk` могут существовать дольше тестового стенда. По умолчанию после записи артефактов принудительно выполняется корректный `process.exit`; если задать `OPENCLAW_QA_MATRIX_DISABLE_FORCE_EXIT=1`, процесс может продолжать работать.
- **Ошибка очистки:** найдите выведенную команду восстановления (вызов `docker compose ... down --remove-orphans`) и выполните её вручную, чтобы освободить порт домашнего сервера.
- **Нестабильные окна отрицательных проверок в CI:** уменьшите `OPENCLAW_QA_MATRIX_NO_REPLY_WINDOW_MS` (по умолчанию 8 с), если CI работает быстро; увеличьте его на медленных общих раннерах.
- **Для отчёта об ошибке нужны отредактированные тела:** перезапустите с `OPENCLAW_QA_MATRIX_CAPTURE_CONTENT=1` и приложите `matrix-qa-observed-events.json`. Считайте полученный артефакт конфиденциальным.
- **Другая версия Tuwunel:** укажите в `OPENCLAW_QA_MATRIX_TUWUNEL_IMAGE` тестируемую версию. В репозитории для этого потока зафиксирован только образ по умолчанию.

## Контракт реального транспорта

Matrix — один из трёх потоков реального транспорта (Matrix, Telegram, Discord), использующих единый контрольный список контракта, определённый в разделе [Обзор QA: Покрытие реального транспорта](/ru/concepts/qa-e2e-automation#live-transport-coverage). `qa-channel` остаётся широким набором синтетических тестов и намеренно не входит в эту матрицу.
