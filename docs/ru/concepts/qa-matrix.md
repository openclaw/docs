Линия QA для Matrix запускает встроенный плагин `@openclaw/matrix` с одноразовым домашним сервером Tuwunel в Docker, временными учетными записями драйвера, тестируемой системы и наблюдателя, а также предварительно созданными комнатами. Она обеспечивает проверку Matrix с реальным транспортом в рабочем режиме.

Инструментарий только для сопровождающих. В пакетные выпуски OpenClaw не входит `qa-lab`, поэтому `openclaw qa` запускается только из рабочей копии исходного кода, которая загружает встроенный исполнитель напрямую, без установки плагина.

Более широкий контекст платформы QA см. в разделе [Обзор QA](/ru/concepts/qa-e2e-automation).

## Быстрый старт

```bash
pnpm openclaw qa matrix --profile fast --fail-fast
```

Обычная команда `pnpm openclaw qa matrix` запускается с `--profile all` и не останавливается после первого сбоя. Распределяйте полный набор по параллельным заданиям с помощью `--profile transport|media|e2ee-smoke|e2ee-deep|e2ee-cli`.

## Что делает линия

1. Развертывает одноразовый домашний сервер Tuwunel в Docker (образ по умолчанию — `ghcr.io/matrix-construct/tuwunel:v1.5.1`, имя сервера — `matrix-qa.test`, порт — `28008`) за ограниченным регистратором запросов и ответов, скрывающим конфиденциальные данные.
2. Регистрирует трех временных пользователей: `driver` (отправляет входящий трафик), `sut` (тестируемая учетная запись OpenClaw в Matrix), `observer` (перехватывает сторонний трафик).
3. Создает комнаты, необходимые для выбранных сценариев (основную, для веток обсуждений, медиафайлов, перезапуска, вторичную, со списком разрешений, E2EE, личную переписку для проверки и т. д.).
4. Запускает независимую от базовой среды проверку протокола `matrix-qa-v1` на записанной границе Tuwunel. Модульные тесты подтверждают контракт проверки с помощью фикстуры протокола Matrix; канонический хост адаптера транспорта QA в [#99707](https://github.com/openclaw/openclaw/pull/99707) отвечает за подключение реальной цели Crabline.
5. Запускает дочерний Gateway OpenClaw с реальным плагином Matrix, ограниченным учетной записью тестируемой системы.
6. Последовательно выполняет сценарии, наблюдая за событиями через клиенты Matrix драйвера и наблюдателя и формируя ожидаемые маршруты и состояния на основе записанного трафика.
7. Завершает работу домашнего сервера, записывает отчет и артефакты подтверждения, затем завершает процесс.

## CLI

```text
pnpm openclaw qa matrix [options]
```

### Общие флаги

| Флаг                  | Значение по умолчанию                         | Описание                                                                                                                                                                                        |
| --------------------- | --------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--profile <profile>` | `all`                                         | Профиль сценариев. См. [Профили](#profiles).                                                                                                                                                    |
| `--fail-fast`         | выключено                                     | Остановиться после первой неудачной проверки или первого сбойного сценария.                                                                                                                     |
| `--scenario <id>`     | -                                             | Запустить только этот сценарий. Можно указывать несколько раз. См. [Сценарии](#scenarios).                                                                                                      |
| `--output-dir <path>` | `<repo>/.artifacts/qa-e2e/matrix-<timestamp>` | Каталог для записи отчетов, сводки, перечня маршрутов и состояний, наблюдаемых событий и журнала вывода. Относительные пути разрешаются относительно `--repo-root`.                              |
| `--repo-root <path>`  | `process.cwd()`                               | Корень репозитория при запуске из нейтрального рабочего каталога.                                                                                                                               |
| `--sut-account <id>`  | `sut`                                         | Идентификатор учетной записи Matrix в конфигурации Gateway для QA.                                                                                                                              |

### Флаги поставщика

Линия использует реальный транспорт Matrix, но поставщика модели можно настраивать:

| Флаг                     | Значение по умолчанию               | Описание                                                                                                                                                                                                           |
| ------------------------ | ----------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `--provider-mode <mode>` | `live-frontier`                     | `mock-openai` — для детерминированной имитации диспетчеризации, `live-frontier` — для реальных передовых поставщиков. Устаревший псевдоним `live-openai` по-прежнему работает.                                       |
| `--model <ref>`          | значение поставщика по умолчанию    | Основная ссылка `provider/model`.                                                                                                                                                                                   |
| `--alt-model <ref>`      | значение поставщика по умолчанию    | Альтернативная ссылка `provider/model` для сценариев, переключающих модель в процессе выполнения.                                                                                                                  |
| `--fast`                 | выключено                           | Включить быстрый режим поставщика, если он поддерживается.                                                                                                                                                         |

QA для Matrix не принимает `--credential-source` и `--credential-role`. Линия локально создает одноразовых пользователей; общий пул учетных данных для аренды отсутствует.

## Профили

| Профиль          | Назначение                                                                                                                                                                                                                                                             |
| ---------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `all` (по умолчанию) | Полный каталог. Медленный, но исчерпывающий.                                                                                                                                                                                                                       |
| `fast`           | Подмножество для проверки выпуска, охватывающее императивный контракт реального транспорта: фильтрацию упоминаний, блокировку списком разрешений, форму ответа, возобновление после перезапуска, наблюдение за реакциями, доставку метаданных одобрения выполнения и базовый ответ E2EE. |
| `transport`      | Сценарии веток обсуждений, личных сообщений, комнат, автоматического присоединения, упоминаний и списков разрешений, одобрений и реакций на транспортном уровне.                                                                                                       |
| `media`          | Проверка вложений: изображений, аудио, видео, PDF и EPUB.                                                                                                                                                                                                              |
| `e2ee-smoke`     | Минимальная проверка E2EE: базовый зашифрованный ответ, продолжение ветки обсуждения и успешная начальная настройка.                                                                                                                                                     |
| `e2ee-deep`      | Исчерпывающие сценарии E2EE для потери состояния, резервного копирования, ключей и восстановления.                                                                                                                                                                      |
| `e2ee-cli`       | Сценарии CLI `openclaw matrix encryption setup` и `verify *`, выполняемые через среду QA.                                                                                                                                                                              |

Точное сопоставление находится в `extensions/qa-matrix/src/runners/contract/scenario-catalog.ts`.

## Сценарии

Общий адаптер Matrix предоставляет следующие канонические сценарии YAML через `openclaw qa suite --channel-driver live --channel matrix`:

- `channel-chat-baseline`
- `thread-follow-up`
- `thread-isolation`
- `thread-reply-override`
- `dm-shared-session`
- `dm-per-room-session`

`subagent-thread-spawn` остается доступным при явном выборе `--scenario subagent-thread-spawn`,
но не входит в общий набор Matrix по умолчанию, пока подтверждение завершения дочернего процесса в рабочем режиме не станет стабильным.

Остальной список идентификаторов императивных сценариев представляет собой объединение `MatrixQaScenarioId` в `extensions/qa-matrix/src/runners/contract/scenario-catalog.ts`. Категории:

- ветки обсуждений: `matrix-thread-root-preservation`, `matrix-thread-nested-reply-shape`
- верхний уровень / личные сообщения / комнаты: `matrix-top-level-reply-shape`, `matrix-room-*`, `matrix-dm-*`
- потоковая передача и ход выполнения инструментов: `matrix-room-partial-streaming-preview`, `matrix-room-quiet-streaming-preview`, `matrix-room-tool-progress-*`, `matrix-room-block-streaming`
- медиафайлы: `matrix-media-type-coverage`, `matrix-room-image-understanding-attachment`, `matrix-attachment-only-ignored`, `matrix-unsupported-media-safe`
- маршрутизация: `matrix-room-autojoin-invite`, `matrix-secondary-room-*`
- реакции: `matrix-reaction-*`
- одобрения: `matrix-approval-*` (метаданные выполнения и плагина, резервная отправка частями, реакции отклонения, ветки обсуждений и маршрутизация `target: "both"`)
- перезапуск и повторное воспроизведение: `matrix-restart-*`, `matrix-stale-sync-replay-dedupe`, `matrix-room-membership-loss`, `matrix-homeserver-restart-resume`, `matrix-initial-catchup-then-incremental`
- фильтрация упоминаний, взаимодействие между ботами и списки разрешений: `matrix-mention-*`, `matrix-allowbots-*`, `matrix-allowlist-*`, `matrix-multi-actor-ordering`, `matrix-inbound-edit-*`, `matrix-mxid-prefixed-command-block`, `matrix-observer-allowlist-override`
- E2EE: `matrix-e2ee-*` (базовый ответ, продолжение ветки обсуждения, начальная настройка, жизненный цикл ключа восстановления, варианты потери состояния, поведение серверной резервной копии, гигиена устройств, проверка через SAS / QR / личные сообщения, перезапуск, скрытие конфиденциальных данных в артефактах)
- CLI E2EE: `matrix-e2ee-cli-*` (настройка шифрования, идемпотентная настройка, сбой начальной настройки, жизненный цикл ключа восстановления, несколько учетных записей, полный цикл ответа Gateway, самопроверка)

Передайте `--scenario <id>` (можно указывать несколько раз), чтобы запустить выбранный вручную набор; сочетайте с `--profile all`, чтобы игнорировать фильтрацию по профилю.

## Переменные среды

| Переменная                              | Значение по умолчанию                     | Действие                                                                                                                                                                                                                  |
| --------------------------------------- | ----------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `OPENCLAW_QA_MATRIX_TIMEOUT_MS`         | `1800000` (30 мин)                        | Жёсткий верхний предел продолжительности всего запуска.                                                                                                                                                                   |
| `OPENCLAW_QA_MATRIX_CANARY_TIMEOUT_MS`  | `45000`                                   | Ограничение времени ожидания первоначального контрольного ответа. В CI выпуска оно увеличивается на общих исполнителях, чтобы медленный первый цикл Gateway не приводил к сбою до начала выполнения сценариев.             |
| `OPENCLAW_QA_MATRIX_NO_REPLY_WINDOW_MS` | `8000`                                    | Период ожидания отсутствия событий для отрицательных проверок отсутствия ответа. Ограничивается значением, не превышающим (`<=`) тайм-аут запуска.                                                                          |
| `OPENCLAW_QA_MATRIX_CLEANUP_TIMEOUT_MS` | `90000`                                   | Ограничение времени завершения работы Docker. При сбое отображается команда восстановления `docker compose ... down --remove-orphans`.                                                                                     |
| `OPENCLAW_QA_MATRIX_TUWUNEL_IMAGE`      | `ghcr.io/matrix-construct/tuwunel:v1.5.1` | Переопределяет образ домашнего сервера при проверке с другой версией Tuwunel.                                                                                                                                              |
| `OPENCLAW_QA_MATRIX_PROGRESS`           | включено                                  | `0` отключает вывод строк хода выполнения `[matrix-qa] ...` в stderr. `1` принудительно включает их.                                                                                                                       |
| `OPENCLAW_QA_MATRIX_CAPTURE_CONTENT`    | отредактировано                           | `1` сохраняет тело сообщения и `formatted_body` в `matrix-qa-observed-events.json`. По умолчанию содержимое редактируется для безопасности артефактов CI.                                                                  |
| `OPENCLAW_QA_MATRIX_DISABLE_FORCE_EXIT` | выключено                                 | `1` пропускает детерминированный вызов `process.exit` после записи артефактов. По умолчанию выход выполняется принудительно, поскольку нативные криптографические дескрипторы matrix-js-sdk могут поддерживать цикл событий после завершения записи артефактов. |
| `OPENCLAW_RUN_NODE_OUTPUT_LOG`          | не задано                                 | Если значение задано внешним средством запуска (например, `scripts/run-node.mjs`), Matrix QA повторно использует этот путь к журналу вместо запуска собственного перенаправления через tee.                                  |

## Выходные артефакты

Записываются в `--output-dir` (по умолчанию `<repo>/.artifacts/qa-e2e/matrix-<timestamp>`, чтобы последовательные запуски не перезаписывали друг друга):

- `matrix-qa-report.md`: отчёт о протоколе в формате Markdown (что прошло проверку, завершилось сбоем, было пропущено и почему).
- `matrix-qa-summary.json`: структурированная сводка, подходящая для анализа в CI и информационных панелей.
- `matrix-qa-route-state-manifest.json`: динамический перечень `matrix-qa-v1`, индексированный по идентификатору сценария. Он содержит отредактированные структуры маршрутов и тел запросов, порядок запросов, наблюдавшиеся повторные попытки, ошибки, непрерывность токенов синхронизации, а также семейства состояний устройств, ключей, медиафайлов и резервных копий, наблюдавшиеся во время этого запуска. Это исполняемое доказательство, а не зафиксированная в репозитории базовая версия.
- `matrix-qa-observed-events.json`: события Matrix, наблюдавшиеся клиентами-драйверами и клиентами-наблюдателями. Тела редактируются, если не задано `OPENCLAW_QA_MATRIX_CAPTURE_CONTENT=1`; метаданные подтверждений представлены в виде сводки с выбранными безопасными полями и сокращённым предварительным просмотром команды.
- `matrix-qa-output.log`: объединённый вывод stdout/stderr запуска. Если задано `OPENCLAW_RUN_NODE_OUTPUT_LOG`, вместо него повторно используется журнал внешнего средства запуска.

## Советы по диагностике

- **Запуск зависает ближе к завершению:** нативные криптографические дескрипторы `matrix-js-sdk` могут продолжать существовать после завершения работы среды тестирования. По умолчанию после записи артефактов принудительно выполняется корректный вызов `process.exit`; если задано `OPENCLAW_QA_MATRIX_DISABLE_FORCE_EXIT=1`, процесс может продолжать работу.
- **Ошибка очистки:** найдите выведенную команду восстановления (вызов `docker compose ... down --remove-orphans`) и запустите её вручную, чтобы освободить порт домашнего сервера.
- **Нестабильные интервалы отрицательных проверок в CI:** уменьшите `OPENCLAW_QA_MATRIX_NO_REPLY_WINDOW_MS` (по умолчанию 8 с), если CI работает быстро; увеличьте его на медленных общих исполнителях.
- **Для отчёта об ошибке нужны отредактированные тела:** повторите запуск с `OPENCLAW_QA_MATRIX_CAPTURE_CONTENT=1` и приложите `matrix-qa-observed-events.json`. Считайте полученный артефакт конфиденциальным.
- **Другая версия Tuwunel:** укажите в `OPENCLAW_QA_MATRIX_TUWUNEL_IMAGE` тестируемую версию. В репозитории для этого контура зафиксирован только образ по умолчанию.

## Контракт реального транспорта

Matrix — один из трёх контуров реального транспорта (Matrix, Telegram, Discord), использующих единый контрольный список контракта, определённый в разделе [Обзор QA: покрытие реального транспорта](/ru/concepts/qa-e2e-automation#live-transport-coverage). `qa-channel` остаётся комплексным набором синтетических тестов и намеренно не входит в эту матрицу.
