Канал Matrix QA запускает встроенный Plugin `@openclaw/matrix` с одноразовым домашним сервером Tuwunel в Docker, временными учетными записями драйвера, SUT и наблюдателя, а также предварительно заполненными комнатами. Это покрытие Matrix с реальным live-транспортом.

Это инструменты только для сопровождающих. Упакованные релизы OpenClaw намеренно не включают `qa-lab`, поэтому `openclaw qa` доступен только из исходного checkout. Исходные checkout загружают встроенный runner напрямую - шаг установки Plugin не нужен.

Более широкий контекст QA-фреймворка см. в [обзоре QA](/ru/concepts/qa-e2e-automation).

## Быстрый старт

```bash
pnpm openclaw qa matrix --profile fast --fail-fast
```

Обычная команда `pnpm openclaw qa matrix` запускает `--profile all` и не останавливается при первом сбое. Используйте `--profile fast --fail-fast` как релизный gate; разбивайте каталог на шарды с `--profile transport|media|e2ee-smoke|e2ee-deep|e2ee-cli`, когда запускаете полный набор параллельно.

## Что делает канал

1. Поднимает одноразовый домашний сервер Tuwunel в Docker (образ по умолчанию `ghcr.io/matrix-construct/tuwunel:v1.5.1`, имя сервера `matrix-qa.test`, порт `28008`) за ограниченным рекордером запросов/ответов с редактированием чувствительных данных.
2. Регистрирует трех временных пользователей - `driver` (отправляет входящий трафик), `sut` (тестируемая учетная запись OpenClaw Matrix), `observer` (захват стороннего трафика).
3. Заполняет комнаты, требуемые выбранными сценариями (основная, трединг, медиа, перезапуск, вторичная, allowlist, E2EE, DM для верификации и т. д.).
4. Запускает транспортно-нейтральный протокольный probe `matrix-qa-v1` на записанной границе Tuwunel. Модульные тесты подтверждают контракт probe с фикстурой протокола Matrix; канонический хост транспортного адаптера QA в [#99707](https://github.com/openclaw/openclaw/pull/99707) владеет подключением реальной цели Crabline.
5. Запускает дочерний OpenClaw gateway с реальным Matrix Plugin, ограниченным учетной записью SUT; `qa-channel` не загружается в дочерний процесс.
6. Выполняет сценарии последовательно, наблюдая события через клиенты Matrix драйвера/наблюдателя и выводя ожидания маршрутов/состояния из записанного трафика.
7. Завершает работу домашнего сервера, записывает отчет и артефакты доказательств, затем выходит.

## CLI

```text
pnpm openclaw qa matrix [options]
```

### Общие флаги

| Флаг                  | По умолчанию                                | Описание                                                                                                                                                       |
| --------------------- | ------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--profile <profile>` | `all`                                       | Профиль сценариев. См. [Профили](#profiles).                                                                                                                   |
| `--fail-fast`         | выкл.                                       | Остановиться после первой неуспешной проверки или сценария.                                                                                                    |
| `--scenario <id>`     | -                                           | Запустить только этот сценарий. Можно повторять. См. [Сценарии](#scenarios).                                                                                   |
| `--output-dir <path>` | `<repo>/.artifacts/qa-e2e/matrix-<timestamp>` | Куда записываются отчеты, сводка, инвентарь маршрутов/состояния, наблюдаемые события и выходной лог. Относительные пути разрешаются относительно `--repo-root`. |
| `--repo-root <path>`  | `process.cwd()`                             | Корень репозитория при запуске из нейтрального рабочего каталога.                                                                                               |
| `--sut-account <id>`  | `sut`                                       | ID учетной записи Matrix внутри конфигурации QA gateway.                                                                                                       |

### Флаги провайдера

Канал использует реальный транспорт Matrix, но провайдер модели настраивается:

| Флаг                     | По умолчанию     | Описание                                                                                                                                                         |
| ------------------------ | ---------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--provider-mode <mode>` | `live-frontier`  | `mock-openai` для детерминированной mock-диспетчеризации или `live-frontier` для live frontier-провайдеров. Устаревший alias `live-openai` все еще работает.     |
| `--model <ref>`          | значение провайдера по умолчанию | Основной ref `provider/model`.                                                                                                                   |
| `--alt-model <ref>`      | значение провайдера по умолчанию | Альтернативный ref `provider/model`, когда сценарии переключаются в середине выполнения.                                                         |
| `--fast`                 | выкл.            | Включить быстрый режим провайдера там, где он поддерживается.                                                                                                    |

Matrix QA не принимает `--credential-source` или `--credential-role`. Канал локально создает одноразовых пользователей; общего пула учетных данных для аренды нет.

## Профили

Выбранный профиль определяет, какие сценарии запускаются.

| Профиль         | Для чего использовать                                                                                                                                                                                                                          |
| --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `all` (по умолчанию) | Полный каталог. Медленно, но исчерпывающе.                                                                                                                                                                                             |
| `fast`          | Подмножество для релизного gate, проверяющее контракт live-транспорта: canary, mention gating, блок allowlist, форму ответа, возобновление после перезапуска, follow-up в треде, изоляцию тредов, наблюдение реакций и доставку метаданных exec approval. |
| `transport`     | Сценарии трединга, DM, комнат, autojoin, mention/allowlist, approval и реакций на уровне транспорта.                                                                                                                                           |
| `media`         | Покрытие вложений изображений, аудио, видео, PDF, EPUB.                                                                                                                                                                                       |
| `e2ee-smoke`    | Минимальное покрытие E2EE - базовый зашифрованный ответ, follow-up в треде, успешный bootstrap.                                                                                                                                               |
| `e2ee-deep`     | Исчерпывающие сценарии потери состояния E2EE, backup, ключей и восстановления.                                                                                                                                                                |
| `e2ee-cli`      | Сценарии CLI `openclaw matrix encryption setup` и `verify *`, выполняемые через QA harness.                                                                                                                                                    |

Точное сопоставление находится в `extensions/qa-matrix/src/runners/contract/scenario-catalog.ts`.

## Сценарии

Полный список ID сценариев - это union `MatrixQaScenarioId` в `extensions/qa-matrix/src/runners/contract/scenario-catalog.ts:15`. Категории включают:

- трединг - `matrix-thread-*`, `matrix-subagent-thread-spawn`
- верхний уровень / DM / комната - `matrix-top-level-reply-shape`, `matrix-room-*`, `matrix-dm-*`
- streaming и прогресс инструментов - `matrix-room-partial-streaming-preview`, `matrix-room-quiet-streaming-preview`, `matrix-room-tool-progress-*`, `matrix-room-block-streaming`
- медиа - `matrix-media-type-coverage`, `matrix-room-image-understanding-attachment`, `matrix-attachment-only-ignored`, `matrix-unsupported-media-safe`
- маршрутизация - `matrix-room-autojoin-invite`, `matrix-secondary-room-*`
- реакции - `matrix-reaction-*`
- approvals - `matrix-approval-*` (метаданные exec/Plugin, chunked fallback, реакции deny, треды и маршрутизация `target: "both"`)
- перезапуск и replay - `matrix-restart-*`, `matrix-stale-sync-replay-dedupe`, `matrix-room-membership-loss`, `matrix-homeserver-restart-resume`, `matrix-initial-catchup-then-incremental`
- mention gating, bot-to-bot и allowlists - `matrix-mention-*`, `matrix-allowbots-*`, `matrix-allowlist-*`, `matrix-multi-actor-ordering`, `matrix-inbound-edit-*`, `matrix-mxid-prefixed-command-block`, `matrix-observer-allowlist-override`
- E2EE - `matrix-e2ee-*` (базовый ответ, follow-up в треде, bootstrap, жизненный цикл recovery key, варианты потери состояния, поведение server backup, гигиена устройств, верификация SAS / QR / DM, перезапуск, редактирование артефактов)
- E2EE CLI - `matrix-e2ee-cli-*` (настройка шифрования, идемпотентная настройка, сбой bootstrap, жизненный цикл recovery-key, несколько учетных записей, gateway-reply round-trip, самоверификация)

Передайте `--scenario <id>` (можно повторять), чтобы запустить выбранный вручную набор; объедините с `--profile all`, чтобы игнорировать фильтрацию профиля.

## Переменные среды

| Переменная                             | По умолчанию                             | Эффект                                                                                                                                                                                        |
| -------------------------------------- | ---------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `OPENCLAW_QA_MATRIX_TIMEOUT_MS`        | `1800000` (30 мин)                       | Жесткая верхняя граница для всего запуска.                                                                                                                                                    |
| `OPENCLAW_QA_MATRIX_CANARY_TIMEOUT_MS` | `45000`                                  | Ограничение для начального canary-ответа. Release CI увеличивает его на общих раннерах, чтобы медленный первый ход Gateway не приводил к сбою до начала покрытия сценариев.                  |
| `OPENCLAW_QA_MATRIX_NO_REPLY_WINDOW_MS` | `8000`                                  | Тихое окно для отрицательных проверок отсутствия ответа. Ограничивается значением `≤` тайм-аута запуска.                                                                                     |
| `OPENCLAW_QA_MATRIX_CLEANUP_TIMEOUT_MS` | `90000`                                 | Ограничение для остановки Docker. Поверхности сбоя включают команду восстановления `docker compose ... down --remove-orphans`.                                                               |
| `OPENCLAW_QA_MATRIX_TUWUNEL_IMAGE`     | `ghcr.io/matrix-construct/tuwunel:v1.5.1` | Переопределяет образ домашнего сервера при проверке с другой версией Tuwunel.                                                                                                                 |
| `OPENCLAW_QA_MATRIX_PROGRESS`          | включено                                 | `0` отключает строки прогресса `[matrix-qa] ...` в stderr. `1` принудительно включает их.                                                                                                    |
| `OPENCLAW_QA_MATRIX_CAPTURE_CONTENT`   | отредактировано                          | `1` сохраняет тело сообщения и `formatted_body` в `matrix-qa-observed-events.json`. По умолчанию данные редактируются, чтобы артефакты CI оставались безопасными.                            |
| `OPENCLAW_QA_MATRIX_DISABLE_FORCE_EXIT` | выключено                               | `1` пропускает детерминированный `process.exit` после записи артефакта. По умолчанию выход выполняется принудительно, потому что нативные crypto-дескрипторы matrix-js-sdk могут удерживать цикл событий активным после завершения артефакта. |
| `OPENCLAW_RUN_NODE_OUTPUT_LOG`         | не задано                                | Когда задано внешним запускателем (например, `scripts/run-node.mjs`), Matrix QA переиспользует этот путь журнала вместо запуска собственного tee.                                             |

## Выходные артефакты

Записываются в `--output-dir`:

- `matrix-qa-report.md` - протокольный отчет Markdown (что прошло, завершилось с ошибкой, было пропущено и почему).
- `matrix-qa-summary.json` - структурированная сводка, подходящая для разбора в CI и панелей мониторинга.
- `matrix-qa-route-state-manifest.json` - динамический инвентарь `matrix-qa-v1`, индексированный по id сценария. Он фиксирует отредактированные формы маршрутов/тел, порядок запросов, наблюдаемые повторы, ошибки, непрерывность sync-токенов и семейства состояний устройств/ключей/медиа/резервных копий, наблюдавшиеся во время этого запуска. Это исполняемое доказательство, а не зафиксированный в репозитории базовый файл.
- `matrix-qa-observed-events.json` - наблюдаемые события Matrix от клиентов драйвера и наблюдателя. Тела редактируются, если не задано `OPENCLAW_QA_MATRIX_CAPTURE_CONTENT=1`; метаданные подтверждения суммируются с выбранными безопасными полями и усеченным предварительным просмотром команды.
- `matrix-qa-output.log` - объединенные stdout/stderr запуска. Если задан `OPENCLAW_RUN_NODE_OUTPUT_LOG`, вместо этого переиспользуется журнал внешнего запускателя.

Каталог вывода по умолчанию: `<repo>/.artifacts/qa-e2e/matrix-<timestamp>`, поэтому последовательные запуски не перезаписывают друг друга.

## Советы по диагностике

- **Запуск зависает ближе к концу:** нативные crypto-дескрипторы `matrix-js-sdk` могут жить дольше harness. По умолчанию после записи артефакта принудительно выполняется чистый `process.exit`; если вы сняли `OPENCLAW_QA_MATRIX_DISABLE_FORCE_EXIT=1`, ожидайте, что процесс задержится.
- **Ошибка очистки:** найдите напечатанную команду восстановления (вызов `docker compose ... down --remove-orphans`) и выполните ее вручную, чтобы освободить порт домашнего сервера.
- **Нестабильные окна отрицательных проверок в CI:** уменьшите `OPENCLAW_QA_MATRIX_NO_REPLY_WINDOW_MS` (по умолчанию 8 с), когда CI быстрый; увеличьте его на медленных общих раннерах.
- **Нужны отредактированные тела для отчета об ошибке:** перезапустите с `OPENCLAW_QA_MATRIX_CAPTURE_CONTENT=1` и приложите `matrix-qa-observed-events.json`. Считайте полученный артефакт чувствительным.
- **Другая версия Tuwunel:** укажите в `OPENCLAW_QA_MATRIX_TUWUNEL_IMAGE` проверяемую версию. Линия проверяет только закрепленный образ по умолчанию.

## Контракт live-транспорта

Matrix — одна из трех линий live-транспорта (Matrix, Telegram, Discord), которые используют единый контрольный список контракта, определенный в [обзоре QA → Покрытие live-транспорта](/ru/concepts/qa-e2e-automation#live-transport-coverage). `qa-channel` остается широким синтетическим набором и намеренно не входит в эту матрицу.
