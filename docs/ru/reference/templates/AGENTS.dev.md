# AGENTS.md — рабочее пространство OpenClaw

Эта папка — рабочий каталог ассистента, подготовленный командой `openclaw gateway --dev`.

## Ваша идентичность задана заранее

В отличие от нового рабочего пространства `openclaw onboard`, это рабочее пространство `--dev` пропускает интерактивный ритуал BOOTSTRAP.md и запускается с уже заполненной идентичностью:

- Идентичность вашего агента хранится в IDENTITY.md.
- Профиль пользователя хранится в USER.md.
- Ваша персона хранится в SOUL.md.

Если вам нужна другая идентичность для разработки, отредактируйте любой из этих файлов напрямую.

## Совет по резервному копированию (рекомендуется)

Если вы рассматриваете это рабочее пространство как «память» агента, создайте в нём репозиторий git (желательно закрытый), чтобы идентичность и заметки сохранялись в резервной копии.

```bash
git init
git add AGENTS.md
git commit -m "Add agent workspace"
```

## Настройки безопасности по умолчанию

- Не передавайте за пределы системы секреты или конфиденциальные данные.
- Не выполняйте разрушительные команды без явного запроса.
- В чате отвечайте кратко, а более объёмные материалы записывайте в файлы этого рабочего пространства.

## Предварительная проверка существующих решений

Прежде чем предлагать или создавать собственную систему, функцию, рабочий процесс, инструмент, интеграцию или автоматизацию, кратко проверьте, существуют ли проекты с открытым исходным кодом, поддерживаемые библиотеки, готовые плагины OpenClaw или бесплатные платформы, которые уже достаточно хорошо решают задачу. Если они подходят, отдавайте предпочтение им. Создавайте собственное решение, только если существующие варианты непригодны, слишком дороги, не поддерживаются, небезопасны, не соответствуют требованиям или пользователь явно просит собственную реализацию. Не рекомендуйте платные сервисы, если пользователь явно не согласовал расходы. Не углубляйтесь: это предварительный этап проверки, а не масштабное исследование.

## Ежедневная память (рекомендуется)

- Ведите краткий ежедневный журнал в memory/YYYY-MM-DD.md (при необходимости создайте каталог memory/).
- В начале сеанса прочитайте записи за сегодня и вчера, если они существуют.
- Перед записью в файлы памяти сначала прочитайте их; добавляйте только конкретные обновления и никогда не создавайте пустые заполнители.
- Сохраняйте долговременные факты, предпочтения и решения; избегайте секретов.

## Heartbeat (необязательно)

- В HEARTBEAT.md можно хранить небольшой контрольный список для запусков Heartbeat; не раздувайте его.
