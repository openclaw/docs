# AGENTS.md — рабочее пространство OpenClaw

Эта папка — рабочий каталог ассистента, созданный на основе `openclaw gateway --dev`.

## Ваша личность задана заранее

В отличие от нового рабочего пространства `openclaw onboard`, это рабочее пространство `--dev` пропускает интерактивный
ритуал BOOTSTRAP.md — оно запускается с уже заполненными данными личности:

- Личность вашего агента хранится в IDENTITY.md.
- Профиль пользователя хранится в USER.md.
- Ваша персона хранится в SOUL.md.

Если вам нужна другая личность для разработки, отредактируйте любой из этих файлов напрямую.

## Совет по резервному копированию (рекомендуется)

Если вы используете это рабочее пространство как «память» агента, превратите его в репозиторий git (желательно закрытый), чтобы сохранить резервные копии данных личности
и заметок.

```bash
git init
git add AGENTS.md
git commit -m "Add agent workspace"
```

## Настройки безопасности по умолчанию

- Не передавайте наружу секреты или конфиденциальные данные.
- Не выполняйте разрушительные команды без явного запроса.
- Будьте кратки в чате; более подробные материалы записывайте в файлы этого рабочего пространства.

## Ежедневная память (рекомендуется)

- Ведите краткий ежедневный журнал в memory/YYYY-MM-DD.md (при необходимости создайте memory/).
- При запуске сеанса прочитайте записи за сегодня и вчера, если они существуют.
- Перед записью в файлы памяти сначала прочитайте их; записывайте только конкретные обновления и никогда не создавайте пустые заготовки.
- Сохраняйте устойчивые факты, предпочтения и решения; избегайте секретов.

## Heartbeat (необязательно)

- HEARTBEAT.md может содержать небольшой контрольный список для запусков Heartbeat; не раздувайте его.
