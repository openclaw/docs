# AGENTS.md - рабочая область OpenClaw

Эта папка является рабочим каталогом ассистента.

## Первый запуск (однократно)

- Если BOOTSTRAP.md существует, выполните описанный в нем ритуал и удалите его после завершения.
- Идентичность вашего агента хранится в IDENTITY.md.
- Ваш профиль хранится в USER.md.

## Совет по резервному копированию (рекомендуется)

Если вы рассматриваете эту рабочую область как «память» агента, сделайте ее git-репозиторием (желательно приватным), чтобы идентичность
и заметки сохранялись в резервной копии.

```bash
git init
git add AGENTS.md
git commit -m "Add agent workspace"
```

## Настройки безопасности по умолчанию

- Не извлекайте секреты или приватные данные.
- Не выполняйте разрушительные команды, если об этом явно не попросили.
- Будьте кратки в чате; более длинный вывод записывайте в файлы в этой рабочей области.

## Предварительная проверка существующих решений

Перед тем как предлагать или создавать пользовательскую систему, функцию, рабочий процесс, инструмент, интеграцию или автоматизацию, выполните краткую проверку проектов с открытым исходным кодом, поддерживаемых библиотек, существующих Plugin OpenClaw или бесплатных платформ, которые уже решают задачу достаточно хорошо. Предпочитайте их, когда они подходят. Создавайте пользовательское решение только тогда, когда существующие варианты непригодны, слишком дороги, не поддерживаются, небезопасны, не соответствуют требованиям или пользователь явно просит пользовательское решение. Избегайте рекомендаций платных сервисов, если пользователь явно не одобряет расходы. Сохраняйте легковесный подход: это предварительный контрольный этап, а не широкое исследовательское задание.

## Ежедневная память (рекомендуется)

- Ведите короткий ежедневный журнал в memory/YYYY-MM-DD.md (создайте memory/, если нужно).
- При начале сеанса прочитайте сегодняшний и вчерашний журналы, если они есть.
- Перед записью файлов памяти сначала прочитайте их; записывайте только конкретные обновления, никогда пустые заполнители.
- Фиксируйте долговременные факты, предпочтения и решения; избегайте секретов.

## Heartbeat (необязательно)

- HEARTBEAT.md может содержать небольшой контрольный список для запусков Heartbeat; держите его кратким.
