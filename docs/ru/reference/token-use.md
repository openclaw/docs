OpenClaw отслеживает **токены**, а не символы. Токены зависят от модели, но для большинства
моделей в стиле OpenAI среднее значение для английского текста составляет около 4 символов на токен.

## Как формируется системный промпт

OpenClaw формирует собственный системный промпт при каждом запуске. Он включает:

- Список инструментов и краткие описания
- Список Skills (только метаданные; инструкции загружаются по требованию с помощью `read`). Нативные
  ходы Codex получают компактный блок Skills в виде ограниченных текущим ходом инструкций
  разработчика по совместной работе; в других средах выполнения он передаётся в обычной части промпта.
  Размер ограничивается параметром `skills.limits.maxSkillsPromptChars` с необязательным переопределением
  для отдельного агента в `agents.list[].skillsLimits.maxSkillsPromptChars`.
- Инструкции по самообновлению
- Файлы рабочей области и начальной загрузки (`AGENTS.md`, `SOUL.md`, `TOOLS.md`,
  `IDENTITY.md`, `USER.md`, `HEARTBEAT.md`, `BOOTSTRAP.md` для нового агента, а также
  `MEMORY.md`, если он существует). Большие внедряемые файлы усекаются согласно
  `agents.defaults.bootstrapMaxChars` (по умолчанию: `20000`); общий объём внедряемых
  данных начальной загрузки ограничен параметром `agents.defaults.bootstrapTotalMaxChars`
  (по умолчанию: `60000`).
  - Нативные ходы Codex не вставляют необработанное содержимое `MEMORY.md`, если для этой
    рабочей области доступны инструменты памяти; вместо этого они получают небольшой указатель
    на память в ограниченных текущим ходом инструкциях разработчика по совместной работе и используют
    инструменты памяти по требованию. Если инструменты отключены, поиск по памяти недоступен или
    активная рабочая область отличается от рабочей области памяти агента, `MEMORY.md`
    передаётся обычным ограниченным способом в контексте хода.
  - Корневой файл `memory.md` в нижнем регистре никогда не внедряется. Это устаревший входной файл
    для исправления командой `openclaw doctor --fix`, которая переносит его содержимое в `MEMORY.md`.
  - Ежедневные файлы `memory/*.md` не входят в обычный промпт начальной загрузки;
    в стандартных ходах они остаются доступными по требованию через инструменты памяти. Запуски
    модели при сбросе или запуске могут добавить одноразовый блок контекста запуска с недавними
    ежедневными записями памяти для первого хода; это управляется параметром
    `agents.defaults.startupContext`. Команды обычного чата `/new` и `/reset`
    подтверждаются без вызова модели.
  - Фрагменты `AGENTS.md` после Compaction передаются отдельно и требуют явного включения
    через `agents.defaults.compaction.postCompactionSections`.
- Время (UTC и часовой пояс пользователя)
- Теги ответа и поведение Heartbeat
- Метаданные среды выполнения (хост/ОС/модель/режим рассуждений)

Полное описание см. в разделе [Системный промпт](/ru/concepts/system-prompt).

При документировании учётных данных или фрагментов аутентификации используйте
[Соглашения о заполнителях секретов](/ru/reference/secret-placeholder-conventions), чтобы
избежать ложных срабатываний сканера секретов в изменениях, затрагивающих только документацию.

## Что учитывается в окне контекста

Всё, что получает модель, учитывается в ограничении контекста:

- Системный промпт (все перечисленные выше разделы)
- История разговора (сообщения пользователя и ассистента)
- Вызовы инструментов и их результаты
- Вложения и расшифровки (изображения, аудио, файлы)
- Сводки Compaction и артефакты очистки
- Обёртки провайдера или заголовки безопасности (не видны, но всё равно учитываются)

Для поверхностей с интенсивным использованием среды выполнения предусмотрены собственные
явные ограничения в `agents.defaults.contextLimits` (переопределения для отдельных агентов —
в `agents.list[].contextLimits`):

| Ключ                     | Назначение                                                               |
| ------------------------ | ------------------------------------------------------------------------ |
| `memoryGetMaxChars`      | Максимальное число символов, возвращаемых `memory_get` до усечения.       |
| `memoryGetDefaultLines`  | Размер окна строк `memory_get` по умолчанию, если в запросе нет `lines`.  |
| `toolResultMaxChars`     | Расширенный предел для одного текущего результата инструмента (до `1000000` символов). |
| `postCompactionMaxChars` | Максимальное число символов из `AGENTS.md`, сохраняемых при обновлении после Compaction. |

Это ограниченные фрагменты среды выполнения и внедряемые блоки, принадлежащие среде выполнения,
отделённые от ограничений начальной загрузки, контекста запуска и промпта Skills.

По умолчанию `toolResultMaxChars` не задан, поэтому OpenClaw определяет ограничение текущего
результата инструмента по фактическому размеру окна контекста модели: `16000` символов при
объёме менее 100 тыс. токенов, `32000` символов при 100 тыс. и более токенов, `64000` символов
при 200 тыс. и более токенов. Защитное ограничение доли контекста в среде выполнения по-прежнему
ограничивает один результат инструмента 30% окна контекста, даже если явно настроен более высокий предел.

Для изображений OpenClaw уменьшает разрешение изображений из расшифровок и результатов инструментов
перед вызовами провайдера. Настройка выполняется с помощью `agents.defaults.imageMaxDimensionPx`
(по умолчанию: `1200`):

- Меньшие значения сокращают расход токенов распознавания изображений и размер полезной нагрузки.
- Большие значения сохраняют больше визуальных деталей для снимков экрана с большим объёмом OCR/UI.

Для практической детализации размеров отдельных внедряемых файлов, инструментов, Skills и системного
промпта используйте `/context list` или `/context detail`. См. раздел
[Контекст](/ru/concepts/context).

## Как посмотреть текущее использование токенов

В чате:

- `/status` -> насыщенная эмодзи карточка состояния с моделью сеанса, использованием контекста,
  числом входных и выходных токенов последнего ответа и оценочной стоимостью, если для активной
  модели настроены локальные цены.
- `/usage off|tokens|full` -> добавляет к каждому ответу нижний колонтитул с использованием ресурсов.
  Настройка сохраняется для каждого сеанса отдельно (в поле `responseUsage`).
  - `/usage reset` (псевдонимы: `inherit`, `clear`, `default`) удаляет переопределение
    сеанса, чтобы снова наследовалось настроенное значение по умолчанию.
  - `/usage tokens` показывает сведения о токенах и кэше хода.
  - `/usage full` показывает компактные сведения о модели, контексте и стоимости; оценочная стоимость
    отображается только при наличии у OpenClaw метаданных использования и локальных цен для
    активной модели. Пользовательские макеты `messages.usageTemplate` могут включать
    поля токенов и кэша.
- `/usage cost` -> локальная сводка стоимости из журналов сеансов OpenClaw.

Другие поверхности:

- **TUI/веб-TUI:** поддерживаются `/status` и `/usage`.
- **CLI:** `openclaw status --usage` и `openclaw channels list` показывают
  нормализованные окна квот провайдеров (`X% left`, а не стоимость каждого ответа).
  В настоящее время окна использования поддерживают следующие провайдеры: Claude (Anthropic), ClawRouter, Copilot
  (GitHub), DeepSeek, Gemini (Google Gemini CLI), MiniMax, OpenAI, Xiaomi,
  Xiaomi Token Plan и z.ai.

Перед отображением поверхности использования нормализуют распространённые собственные псевдонимы
полей провайдеров. Для трафика Responses семейства OpenAI это включает как
`input_tokens`/`output_tokens`, так и `prompt_tokens`/`completion_tokens`, поэтому
названия полей, зависящие от транспорта, не влияют на `/status`, `/usage` или сводки
сеансов. Использование Gemini CLI также нормализуется: стандартный анализатор `stream-json`
считывает события `message` ассистента, а `stats.cached` преобразуется в
`cacheRead`; если CLI не предоставляет явное поле `stats.input`, используется
`stats.input_tokens - stats.cached`. Устаревшие переопределения JSON по-прежнему считывают текст
ответа из `response`.

Для нативного трафика Responses семейства OpenAI псевдонимы использования WebSocket/SSE
нормализуются таким же образом, а при отсутствии `total_tokens` или его значении `0`
итоговое значение вычисляется как сумма нормализованных входных и выходных токенов.

Если текущий снимок сеанса содержит мало данных, `/status` и `session_status`
могут восстановить счётчики токенов и кэша, а также метку активной модели среды выполнения из
последней записи об использовании в расшифровке. Существующие ненулевые текущие значения по-прежнему
имеют приоритет над резервными значениями из расшифровки, а более крупные ориентированные на промпт
итоговые значения расшифровки могут иметь приоритет, если сохранённые итоги отсутствуют или меньше.

Данные аутентификации для окон квот провайдеров сначала берутся из специализированных перехватчиков
провайдера; если у провайдера нет такого перехватчика или он не возвращает токен,
OpenClaw выполняет резервный поиск подходящих учётных данных OAuth/API-ключа в профилях
аутентификации, переменных окружения или конфигурации.

Записи ассистента в расшифровке сохраняют такую же нормализованную структуру использования,
включая `usage.cost`, если для активной модели настроены цены и провайдер возвращает
метаданные использования. Благодаря этому `/usage cost` и состояние сеанса на основе
расшифровки имеют стабильный источник данных даже после исчезновения текущего состояния
среды выполнения.

OpenClaw хранит учёт использования провайдера отдельно от текущего снимка контекста.
Поле провайдера `usage.total` может включать кэшированный ввод, вывод и несколько
вызовов модели в цикле инструментов, поэтому оно полезно для расчёта стоимости и телеметрии,
но может завышать фактическое заполнение текущего окна контекста. Отображение контекста и
диагностика используют последний снимок промпта (`promptTokens` или последний вызов модели,
если снимок промпта недоступен) для значения `context.used`.

## Оценка стоимости (если отображается)

Стоимость оценивается на основе конфигурации цен модели:

```text
models.providers.<provider>.models[].cost
```

Значения задаются в **долларах США за 1 млн токенов** для `input`, `output`, `cacheRead` и
`cacheWrite`. Если цены отсутствуют, `/usage full` не показывает стоимость; используйте
`/usage tokens` или пользовательский `messages.usageTemplate`, если сведения о токенах
и кэше нужны в каждом ответе. Отображение стоимости не ограничивается аутентификацией по API-ключу:
провайдеры без API-ключей, такие как `aws-sdk`, могут показывать оценочную стоимость, если
соответствующая запись настроенной модели содержит локальные цены и провайдер возвращает
метаданные использования.

После того как вспомогательные процессы и каналы достигают состояния готовности Gateway, OpenClaw
запускает необязательную фоновую начальную загрузку цен для настроенных ссылок на модели, у которых
ещё нет локальных цен. Во время этой загрузки извлекаются удалённые каталоги цен OpenRouter и
LiteLLM. Установите `models.pricing.enabled: false`, чтобы пропустить загрузку этих
каталогов в автономных или ограниченных сетях; явные записи
`models.providers.*.models[].cost` по-прежнему используются для локальной оценки стоимости.

## Влияние TTL кэша и очистки

Кэширование промпта у провайдера применяется только в пределах окна TTL кэша. OpenClaw
может дополнительно выполнять **очистку по TTL кэша**: после истечения TTL кэша он очищает
сеанс, а затем сбрасывает окно кэша, чтобы последующие запросы повторно использовали недавно
закэшированный контекст вместо повторного кэширования всей истории.
Это снижает стоимость записи в кэш, если сеанс простаивает дольше TTL.

Настройте это в разделе [Конфигурация Gateway](/ru/gateway/configuration), а подробности
поведения см. в разделе [Очистка сеанса](/ru/concepts/session-pruning).

Heartbeat может поддерживать кэш **прогретым** во время периодов бездействия. Если TTL кэша
модели равен `1h`, установка интервала Heartbeat немного меньше этого значения (например, `55m`)
позволяет избежать повторного кэширования всего промпта и снизить стоимость записи в кэш.

В конфигурациях с несколькими агентами можно использовать одну общую конфигурацию модели и
настраивать поведение кэша отдельно для каждого агента через `agents.list[].params.cacheRetention`.

Полное описание всех параметров см. в разделе [Кэширование промпта](/ru/reference/prompt-caching).

При использовании API Anthropic чтение из кэша значительно дешевле входных токенов, тогда как
запись в кэш тарифицируется с более высоким коэффициентом. Актуальные тарифы и коэффициенты TTL
см. на странице Anthropic о стоимости кэширования промпта:
[https://docs.anthropic.com/docs/build-with-claude/prompt-caching](https://docs.anthropic.com/docs/build-with-claude/prompt-caching)

### Пример: поддержание прогретого кэша на 1 час с помощью Heartbeat

```yaml
agents:
  defaults:
    model:
      primary: "anthropic/claude-opus-4-6"
    models:
      "anthropic/claude-opus-4-6":
        params:
          cacheRetention: "long"
    heartbeat:
      every: "55m"
```

### Пример: смешанный трафик со стратегией кэша для каждого агента

```yaml
agents:
  defaults:
    model:
      primary: "anthropic/claude-opus-4-6"
    models:
      "anthropic/claude-opus-4-6":
        params:
          cacheRetention: "long" # базовое значение по умолчанию для большинства агентов
  list:
    - id: "research"
      default: true
      heartbeat:
        every: "55m" # поддерживать длительный кэш прогретым для глубоких сеансов
    - id: "alerts"
      params:
        cacheRetention: "none" # избегать записи в кэш для пакетных уведомлений
```

`agents.list[].params` объединяется с `params` выбранной модели, поэтому можно
переопределить только `cacheRetention`, сохранив остальные значения модели по умолчанию
без изменений.

### Контекст Anthropic объёмом 1 млн токенов

OpenClaw назначает моделям Claude 4.x с общедоступной поддержкой, таким как Opus 4.8, Opus 4.7, Opus
4.6 и Sonnet 4.6, окно контекста Anthropic объёмом 1 млн токенов. Для этих моделей
не требуется указывать `params.context1m: true`.

```yaml
agents:
  defaults:
    models:
      "anthropic/claude-opus-4-6":
        alias: opus
```

В старых конфигурациях можно сохранить `context1m: true`, но OpenClaw больше не отправляет
устаревший бета-заголовок Anthropic `context-1m-2025-08-07` для этой настройки и
не расширяет окно неподдерживаемых старых моделей Claude до 1 млн токенов.

Требование: учетные данные должны поддерживать использование длинного контекста. В противном случае Anthropic возвращает для этого запроса ошибку ограничения частоты на стороне провайдера.

Если для аутентификации в Anthropic используются токены OAuth/подписки
(`sk-ant-oat-*`), OpenClaw сохраняет обязательные для OAuth бета-заголовки
Anthropic, удаляя при этом устаревший бета-заголовок `context-1m-*`, если он
остался в старой конфигурации.

## Советы по снижению нагрузки на лимит токенов

- Используйте `/compact` для суммаризации длинных сеансов.
- Сокращайте объемные результаты работы инструментов в своих рабочих процессах.
- Уменьшите `agents.defaults.imageMaxDimensionPx` для сеансов с большим количеством снимков экрана.
- Делайте описания Skills краткими (список Skills добавляется в промпт).
- Для многословной исследовательской работы предпочитайте модели меньшего размера.

Точная формула накладных расходов на список Skills приведена в разделе [Skills](/ru/tools/skills).
