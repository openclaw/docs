Карта функций OpenClaw, которые могут обращаться к платным API провайдеров, с указанием того, откуда каждая из них получает учетные данные и где отображаются соответствующие расходы.

## Где отображаются расходы

**`/status`** (снимок по сеансу)

- Показывает текущую модель сеанса, использование контекста и количество токенов последнего ответа.
- Добавляет **расчетную стоимость** последнего ответа, когда у OpenClaw есть метаданные об использовании и локальные сведения о ценах для активной модели, включая провайдеров без API-ключей с явно заданной ценой, например модели Bedrock `aws-sdk`.
- Если оперативный снимок сеанса содержит мало данных, `/status` восстанавливает счетчики токенов и кеша, а также метку активной модели из последней записи об использовании в журнале сеанса. Существующие ненулевые оперативные значения имеют приоритет над данными журнала; итоговое значение из журнала, соответствующее размеру запроса, все равно может иметь приоритет, если сохраненное итоговое значение отсутствует или меньше.

**`/usage`** (нижний колонтитул каждого сообщения)

- `/usage full` добавляет к каждому ответу нижний колонтитул с данными об использовании, включая **расчетную стоимость**, когда настроены локальные цены и доступны метаданные об использовании.
- `/usage tokens` показывает только токены. Среды выполнения с OAuth/токенами по подписке и CLI показывают только токены, если не предоставляют совместимые метаданные об использовании и явно заданную локальную цену.
- `/usage cost` выводит локальную сводку расходов; `/usage off` отключает нижний колонтитул.
- Примечание о Gemini CLI: и `stream-json`, и устаревший формат вывода `json` передают данные об использовании в `stats`. OpenClaw преобразует `stats.cached` в `cacheRead` и при необходимости вычисляет количество входных токенов как `stats.input_tokens - stats.cached`.

**Интерфейс управления → Использование** (анализ по нескольким сеансам)

- Показывает полученные из журналов сеансов итоговые значения токенов и расчетной стоимости за выбранный диапазон дат с разбивкой по провайдерам, моделям, агентам, каналам и типам токенов.
- Сравнивает более короткие календарные интервалы, заканчивающиеся в дату окончания выбранного диапазона. Пропущенные даты учитываются как календарные дни с нулевым использованием; они не пропускаются для создания более плотного интервала.
- Непосредственно обозначает масштаб дневного графика. Значок `√` означает, что сжатие по квадратному корню сохраняет видимость дней с низким использованием.
- Эти итоговые значения описывают доступную локальную историю сеансов, а не счет провайдера или реестр расходов за все время. Интерфейс предупреждает, если для некоторых записей отсутствуют сведения о ценах.

**Интервалы использования в CLI** (квоты провайдеров, а не стоимость отдельных сообщений)

- `openclaw status --usage` и `openclaw channels list` показывают **интервалы использования** провайдеров в виде `X% left`.
- Провайдеры, для которых сейчас поддерживаются интервалы использования: Anthropic, ClawRouter, DeepSeek, GitHub Copilot, Gemini CLI, MiniMax, OpenAI (включая аутентификацию ChatGPT/Codex через OAuth или токен), Xiaomi и z.ai. Полный список провайдеров и флагов см. в разделах [CLI моделей](/ru/cli/models) и [CLI каналов](/ru/cli/channels).
- Необработанные поля MiniMax `usage_percent` / `usagePercent` сообщают об оставшейся квоте, поэтому OpenClaw инвертирует их; поля со счетчиками имеют приоритет, если присутствуют. Если ответ содержит массив `model_remains`, OpenClaw выбирает запись модели чата, при необходимости определяет метку интервала по временным меткам и включает имя модели в метку плана.
- Учетные данные для получения данных об использовании поступают из специализированных обработчиков провайдера, если они доступны; в противном случае OpenClaw использует подходящие учетные данные OAuth или API-ключи из профилей аутентификации, переменных среды или конфигурации.

Подробные примеры см. в разделе [Использование токенов и расходы](/ru/reference/token-use).

<Note>
Anthropic подтвердила, что повторное использование Claude CLI (включая `claude -p`) является разрешенным способом интеграции, пока компания не опубликует новую политику. Anthropic не предоставляет оценку стоимости в долларах для отдельных сообщений, поэтому `/usage full` не может показывать стоимость использования Claude CLI.
</Note>

## Как обнаруживаются ключи

- **Профили аутентификации**: отдельные для каждого агента, хранятся в `auth-profiles.json`.
- **Переменные среды**: например, `OPENAI_API_KEY`, `BRAVE_API_KEY`, `FIRECRAWL_API_KEY`.
- **Конфигурация**: `models.providers.*.apiKey`, `plugins.entries.*.config.webSearch.apiKey`, `plugins.entries.firecrawl.config.webFetch.apiKey`, `agents.defaults.memorySearch.*`, `talk.providers.*.apiKey`.
- **Skills**: `skills.entries.<name>.apiKey`, откуда ключ может экспортироваться в среду процесса Skills.

## Функции, которые могут расходовать средства с ключей

### Основные ответы модели (чат и инструменты)

Каждый ответ или вызов инструмента выполняется через текущего провайдера модели. Это основной источник использования и расходов, включая размещенные планы по подписке, плата за которые взимается вне локального интерфейса OpenClaw: OpenAI Codex, Alibaba Cloud Model Studio Coding Plan, MiniMax Coding Plan, Z.AI/GLM Coding Plan и вход в Claude от Anthropic с включенной функцией Extra Usage.

Настройку цен см. в разделе [Модели](/ru/providers/models), а отображение — в разделе [Использование токенов и расходы](/ru/reference/token-use).

### Анализ медиаданных (аудио, изображения и видео)

Входящие медиаданные могут быть сведены в краткое описание или расшифрованы через API провайдера до запуска конвейера формирования ответа. Поддержка провайдеров регистрируется отдельно для каждого плагина и меняется по мере добавления плагинов; текущий список и конфигурацию см. в разделе [Анализ медиаданных](/ru/nodes/media-understanding).

### Создание изображений и видео

`image_generate` и `video_generate` направляют запрос доступному настроенному провайдеру. Для создания изображений можно автоматически определить провайдера по умолчанию с доступными учетными данными, если параметр `agents.defaults.imageGenerationModel` не задан; для создания видео необходимо явно задать `agents.defaults.videoGenerationModel` (например, `qwen/wan2.6-t2v`).

Текущий список провайдеров см. в разделах [Создание изображений](/ru/tools/image-generation) и [Создание видео](/ru/tools/video-generation).

### Векторные представления памяти и семантический поиск

Семантический поиск по памяти использует API векторных представлений, когда `agents.defaults.memorySearch.provider` указывает удаленный адаптер (например, `openai`, `gemini`, `voyage`, `mistral`, `deepinfra`, `github-copilot`, `amazon-bedrock`). При `memorySearch.provider = "lmstudio"` или `"ollama"` используется локальный или самостоятельно размещенный сервер, что обычно не влечет расходов за размещенную службу. При `memorySearch.provider = "local"` все выполняется на устройстве без использования API. Необязательный провайдер `memorySearch.fallback` может использоваться при сбоях локального создания векторных представлений.

См. раздел [Память](/ru/concepts/memory).

### Инструмент веб-поиска

Использование `web_search` может тарифицироваться в зависимости от выбранного провайдера. Каждый провайдер сначала считывает ключ из переменной среды, а затем из `plugins.entries.<id>.config.webSearch.apiKey`:

| Провайдер              | Переменные среды                                                                                                                                                                                     |
| ---------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Brave Search           | `BRAVE_API_KEY`                                                                                                                                                                                      |
| DuckDuckGo             | ключ не требуется; неофициальный поиск на основе HTML, без оплаты                                                                                                                                    |
| Exa                    | `EXA_API_KEY`                                                                                                                                                                                        |
| Firecrawl              | `FIRECRAWL_API_KEY`                                                                                                                                                                                  |
| Gemini (Google Search) | `GEMINI_API_KEY`                                                                                                                                                                                     |
| Grok (xAI)             | профиль OAuth xAI или `XAI_API_KEY`                                                                                                                                                                  |
| Kimi (Moonshot)        | `KIMI_API_KEY` или `MOONSHOT_API_KEY`                                                                                                                                                                |
| MiniMax Search         | `MINIMAX_CODE_PLAN_KEY`, `MINIMAX_CODING_API_KEY`, `MINIMAX_OAUTH_TOKEN` или `MINIMAX_API_KEY`                                                                                                       |
| Ollama Web Search      | ключ не требуется для доступного локального узла с выполненным входом; прямой поиск через `https://ollama.com` использует `OLLAMA_API_KEY`; узлы с защитой аутентификацией повторно используют обычную аутентификацию провайдера Ollama с токеном предъявителя |
| Parallel               | `PARALLEL_API_KEY`                                                                                                                                                                                   |
| Perplexity Search API  | `PERPLEXITY_API_KEY` или `OPENROUTER_API_KEY`                                                                                                                                                        |
| SearXNG                | `SEARXNG_BASE_URL`; ключ не требуется, самостоятельное размещение без оплаты размещенной службы                                                                                                      |
| Tavily                 | `TAVILY_API_KEY`                                                                                                                                                                                     |

Устаревшие пути конфигурации `tools.web.search.*` по-прежнему загружаются через слой совместимости, но больше не рекомендуются.

**Бесплатный кредит Brave Search**: каждый план включает ежемесячно возобновляемый бесплатный кредит в размере 5 долларов. План Search стоит 5 долларов за 1000 запросов, поэтому кредит покрывает 1000 запросов в месяц без оплаты. Установите ограничение использования в панели управления Brave, чтобы избежать непредвиденных расходов.

См. раздел [Веб-инструменты](/ru/tools/web).

### Инструмент веб-загрузки (Firecrawl)

`web_fetch` может обращаться к Firecrawl с начальным доступом без ключа; добавьте `FIRECRAWL_API_KEY` (или `plugins.entries.firecrawl.config.webFetch.apiKey`), чтобы повысить лимиты. Если Firecrawl не настроен, инструмент переключается на прямую загрузку и встроенный плагин `web-readability` без платного API. Отключите `plugins.entries.web-readability.enabled`, чтобы пропустить локальное извлечение с помощью Readability.

См. раздел [Веб-инструменты](/ru/tools/web).

### Снимки использования провайдера (состояние и работоспособность)

`openclaw status --usage` и `openclaw models status --json` обращаются к конечным точкам данных об использовании провайдеров, чтобы показать интервалы квот или состояние аутентификации. Такие вызовы выполняются редко, но все же обращаются к API провайдеров.

См. раздел [CLI моделей](/ru/cli/models).

### Защитное суммирование при Compaction

Защитный механизм Compaction может суммировать историю сеанса с помощью текущей модели, обращаясь при этом к API провайдера.

См. раздел [Управление сеансами и Compaction](/ru/reference/session-management-compaction).

### Сканирование и проверка моделей

`openclaw models scan` может проверять модели OpenRouter и использует `OPENROUTER_API_KEY`, когда проверка включена.

См. раздел [CLI моделей](/ru/cli/models).

### Речь

Режим речи может обращаться к ElevenLabs, если он настроен: `ELEVENLABS_API_KEY` или `talk.providers.elevenlabs.apiKey`.

См. раздел [Режим речи](/ru/nodes/talk).

### Skills (сторонние API)

Skills могут хранить `apiKey` в `skills.entries.<name>.apiKey`. Если Skills использует этот ключ для обращения к внешнему API, расходы определяются провайдером Skills.

См. раздел [Skills](/ru/tools/skills).
