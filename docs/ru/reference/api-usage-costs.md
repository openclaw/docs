Карта функций OpenClaw, которые могут обращаться к платным API провайдеров, с указанием того, откуда каждая из них получает учётные данные и где отображаются итоговые расходы.

## Где отображаются расходы

**`/status`** (снимок состояния отдельного сеанса)

- Показывает текущую модель сеанса, использование контекста и количество токенов последнего ответа.
- Добавляет **расчётную стоимость** последнего ответа, если у OpenClaw есть метаданные об использовании и локальные данные о ценах для активной модели, включая провайдеров без API-ключей с явно заданной ценой, например модели Bedrock `aws-sdk`.
- Если оперативный снимок состояния сеанса содержит мало данных, `/status` восстанавливает счётчики токенов и кэша, а также метку активной модели из последней записи об использовании в транскрипте. Существующие ненулевые оперативные значения имеют приоритет над данными транскрипта; итоговое значение из транскрипта, соответствующее размеру промпта, всё равно может иметь приоритет, если сохранённое итоговое значение отсутствует или меньше.

**`/usage`** (нижний колонтитул каждого сообщения)

- `/usage full` добавляет к каждому ответу нижний колонтитул с данными об использовании, включая **расчётную стоимость**, если настроены локальные цены и доступны метаданные об использовании.
- `/usage tokens` показывает только токены. Среды выполнения с OAuth/токенами по подписке и CLI показывают только токены, если не предоставляют совместимые метаданные об использовании и явно заданную локальную цену.
- `/usage cost` выводит локальную сводку расходов; `/usage off` отключает нижний колонтитул.
- Примечание о Gemini CLI: выходные данные как `stream-json`, так и устаревшего `json` содержат сведения об использовании в `stats`. OpenClaw нормализует `stats.cached` в `cacheRead` и при необходимости вычисляет количество входных токенов из `stats.input_tokens - stats.cached`.

**Control UI → Usage** (анализ по нескольким сеансам)

- Показывает полученные из транскриптов итоговые значения токенов и расчётной стоимости за выбранный диапазон дат с разбивкой по провайдерам, моделям, агентам, каналам и типам токенов.
- Сравнивает более короткие календарные периоды, заканчивающиеся в дату окончания выбранного диапазона. Отсутствующие даты считаются календарными днями с нулевым использованием; они не пропускаются для создания более плотного периода.
- Отображает шкалу дневной диаграммы непосредственно на ней. Значок `√` означает, что для сохранения видимости дней с низким использованием применяется сжатие по квадратному корню.
- Эти итоговые значения описывают доступную локальную историю сеансов, а не счёт провайдера или журнал расходов за всё время. Интерфейс предупреждает, если для некоторых записей отсутствуют цены.

**Окна использования CLI** (квоты провайдера, а не стоимость отдельных сообщений)

- `openclaw status --usage` и `openclaw channels list` показывают **окна использования** провайдера как `X% left`.
- Текущие провайдеры окон использования: Anthropic, ClawRouter, DeepSeek, GitHub Copilot, Gemini CLI, MiniMax, OpenAI (включая аутентификацию ChatGPT/Codex через OAuth или токен), Xiaomi и z.ai. Полный список провайдеров и флагов см. в разделах [CLI моделей](/ru/cli/models) и [CLI каналов](/ru/cli/channels).
- Необработанные поля MiniMax `usage_percent` / `usagePercent` сообщают оставшуюся квоту, поэтому OpenClaw инвертирует их; при наличии полей на основе количества они имеют приоритет. Если ответ содержит массив `model_remains`, OpenClaw выбирает запись модели чата, при необходимости формирует метку окна из временных меток и включает название модели в метку плана.
- Данные аутентификации для получения сведений об использовании поступают из специализированных перехватчиков провайдера, если они доступны; в противном случае OpenClaw подбирает соответствующие учётные данные OAuth/API-ключей из профилей аутентификации, переменных среды или конфигурации.

Подробные примеры см. в разделе [Использование токенов и расходы](/ru/reference/token-use).

<Note>
Anthropic подтвердила, что повторное использование Claude CLI (включая `claude -p`) является разрешённым шаблоном интеграции, если компания не опубликует новую политику. Anthropic не предоставляет оценку стоимости отдельных сообщений в долларах, поэтому `/usage full` не может показывать стоимость использования Claude CLI.
</Note>

## Как обнаруживаются ключи

- **Профили аутентификации**: отдельные для каждого агента, хранятся в `auth-profiles.json`.
- **Переменные среды**: например, `OPENAI_API_KEY`, `BRAVE_API_KEY`, `FIRECRAWL_API_KEY`.
- **Конфигурация**: `models.providers.*.apiKey`, `plugins.entries.*.config.webSearch.apiKey`, `plugins.entries.firecrawl.config.webFetch.apiKey`, `agents.defaults.memorySearch.*`, `talk.providers.*.apiKey`.
- **Skills**: `skills.entries.<name>.apiKey`, который может экспортировать ключ в среду процесса навыка.

## Функции, которые могут расходовать средства с ключей

### Основные ответы модели (чат и инструменты)

Каждый ответ или вызов инструмента выполняется через текущего провайдера модели. Это основной источник использования и расходов, включая размещённые планы по подписке, счета за которые выставляются вне локального интерфейса OpenClaw: OpenAI Codex, Alibaba Cloud Model Studio Coding Plan, MiniMax Coding Plan, Z.AI/GLM Coding Plan и путь входа Claude от Anthropic с включённой функцией Extra Usage.

Настройку цен см. в разделе [Модели](/ru/providers/models), а сведения об отображении — в разделе [Использование токенов и расходы](/ru/reference/token-use).

### Анализ мультимедиа (аудио, изображения и видео)

Входящие мультимедийные данные могут быть обобщены или транскрибированы через API провайдера до запуска конвейера формирования ответа. Поддержка провайдеров регистрируется отдельно для каждого плагина и меняется по мере добавления плагинов; актуальный список и конфигурацию см. в разделе [Анализ мультимедиа](/ru/nodes/media-understanding).

### Создание изображений и видео

`image_generate` и `video_generate` направляют запрос любому доступному настроенному провайдеру. Для создания изображений может автоматически выбираться провайдер по умолчанию на основе доступной аутентификации, если `agents.defaults.imageGenerationModel` не задан; для создания видео требуется явно указать `agents.defaults.videoGenerationModel` (например, `qwen/wan2.6-t2v`).

Актуальный список провайдеров см. в разделах [Создание изображений](/ru/tools/image-generation) и [Создание видео](/ru/tools/video-generation).

### Векторные представления памяти и семантический поиск

Семантический поиск по памяти использует API векторных представлений, если `agents.defaults.memorySearch.provider` указывает удалённый адаптер (например, `openai`, `gemini`, `voyage`, `mistral`, `deepinfra`, `github-copilot`, `amazon-bedrock`). `memorySearch.provider = "lmstudio"` или `"ollama"` работает с локальным или самостоятельно размещённым сервером и обычно не предполагает оплату размещённого сервиса. `memorySearch.provider = "local"` выполняет всю обработку на устройстве без использования API. Необязательный провайдер `memorySearch.fallback` может использоваться при сбоях локального формирования векторных представлений.

См. раздел [Память](/ru/concepts/memory).

### Инструмент веб-поиска

`web_search` может повлечь расходы в зависимости от выбранного провайдера. Каждый провайдер сначала получает ключ из переменной среды, а затем из `plugins.entries.<id>.config.webSearch.apiKey`:

| Провайдер              | Переменные среды                                                                                                                                                       |
| ---------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Brave Search           | `BRAVE_API_KEY`                                                                                                                                                     |
| DuckDuckGo             | без ключа; неофициальный поиск на основе HTML, без оплаты                                                                                                              |
| Exa                    | `EXA_API_KEY`                                                                                                                                                     |
| Firecrawl              | `FIRECRAWL_API_KEY`                                                                                                                                                     |
| Gemini (Google Search) | `GEMINI_API_KEY`                                                                                                                                                     |
| Grok (xAI)             | профиль OAuth xAI или `XAI_API_KEY`                                                                                                                               |
| Kimi (Moonshot)        | `KIMI_API_KEY` или `MOONSHOT_API_KEY`                                                                                                                             |
| MiniMax Search         | `MINIMAX_CODE_PLAN_KEY`, `MINIMAX_CODING_API_KEY`, `MINIMAX_OAUTH_TOKEN` или `MINIMAX_API_KEY`                                                                                     |
| Ollama Web Search      | без ключа для доступного локального узла с выполненным входом; прямой поиск `https://ollama.com` использует `OLLAMA_API_KEY`; узлы с защитой аутентификацией повторно используют обычную аутентификацию провайдера Ollama с токеном предъявителя |
| Parallel               | `PARALLEL_API_KEY`                                                                                                                                                     |
| Perplexity Search API  | `PERPLEXITY_API_KEY` или `OPENROUTER_API_KEY`                                                                                                                             |
| SearXNG                | `SEARXNG_BASE_URL`; без ключа, самостоятельное размещение, без оплаты размещённого сервиса                                                                              |
| Tavily                 | `TAVILY_API_KEY`                                                                                                                                                     |

Устаревшие пути конфигурации `tools.web.search.*` по-прежнему загружаются через слой совместимости, но больше не рекомендуются к использованию.

**Бесплатный кредит Brave Search**: каждый план включает возобновляемый бесплатный кредит в размере $5 в месяц. План Search стоит $5 за 1,000 запросов, поэтому кредит бесплатно покрывает 1,000 запросов в месяц. Установите лимит использования в панели управления Brave, чтобы избежать непредвиденных расходов.

См. раздел [Веб-инструменты](/ru/tools/web).

### Инструмент получения веб-страниц (Firecrawl)

`web_fetch` может обращаться к Firecrawl, используя начальный доступ без ключа; добавьте `FIRECRAWL_API_KEY` (или `plugins.entries.firecrawl.config.webFetch.apiKey`) для повышения лимитов. Если Firecrawl не настроен, инструмент переключается на прямое получение данных и встроенный плагин `web-readability` (без платного API). Отключите `plugins.entries.web-readability.enabled`, чтобы пропустить локальное извлечение с помощью Readability.

См. раздел [Веб-инструменты](/ru/tools/web).

### Снимки использования провайдера (состояние и работоспособность)

`openclaw status --usage` и `openclaw models status --json` обращаются к конечным точкам использования провайдера, чтобы показывать окна квот или состояние аутентификации. Такие вызовы выполняются редко, но всё же обращаются к API провайдера.

См. раздел [CLI моделей](/ru/cli/models).

### Защитное обобщение при Compaction

Защитный механизм Compaction может обобщать историю сеанса с помощью текущей модели, что при его срабатывании приводит к обращению к API провайдера.

См. раздел [Управление сеансами и Compaction](/ru/reference/session-management-compaction).

### Сканирование и проверка моделей

`openclaw models scan` может проверять модели OpenRouter и использует `OPENROUTER_API_KEY`, когда проверка включена.

См. раздел [CLI моделей](/ru/cli/models).

### Разговорный режим (речь)

При соответствующей настройке разговорный режим может обращаться к ElevenLabs: `ELEVENLABS_API_KEY` или `talk.providers.elevenlabs.apiKey`.

См. раздел [Разговорный режим](/ru/nodes/talk).

### Skills (сторонние API)

Skills могут хранить `apiKey` в `skills.entries.<name>.apiKey`. Если навык использует этот ключ для обращения к внешнему API, расходы определяются провайдером навыка.

См. раздел [Skills](/ru/tools/skills).
