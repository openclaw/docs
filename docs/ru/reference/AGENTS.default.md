## Первый запуск (рекомендуется)

Агенты OpenClaw используют каталог рабочего пространства. По умолчанию: `~/.openclaw/workspace` (настраивается через `agents.defaults.workspace`, поддерживает `~`).

1. Создайте рабочее пространство:

```bash
mkdir -p ~/.openclaw/workspace
```

2. Скопируйте в него стандартные шаблоны рабочего пространства:

```bash
cp docs/reference/templates/AGENTS.md ~/.openclaw/workspace/AGENTS.md
cp docs/reference/templates/SOUL.md ~/.openclaw/workspace/SOUL.md
cp docs/reference/templates/TOOLS.md ~/.openclaw/workspace/TOOLS.md
```

3. Необязательно: используйте перечень навыков персонального ассистента из этого файла вместо универсального шаблона:

```bash
cp docs/reference/AGENTS.default.md ~/.openclaw/workspace/AGENTS.md
```

4. Необязательно: укажите другое рабочее пространство:

```json5
{
  agents: { defaults: { workspace: "~/.openclaw/workspace" } },
}
```

## Настройки безопасности по умолчанию

- Не отправляйте содержимое каталогов или секреты в чат.
- Не выполняйте деструктивные команды без явного запроса.
- Перед изменением конфигурации или планировщиков (crontab, модулей systemd, конфигураций nginx, файлов rc оболочки) сначала проверьте текущее состояние; по умолчанию сохраняйте или объединяйте существующие настройки.
- Не отправляйте частичные или потоковые ответы во внешние каналы обмена сообщениями (только окончательные ответы).

## Предварительная проверка существующих решений

Прежде чем предлагать или создавать собственную систему, функцию, рабочий процесс, инструмент, интеграцию или автоматизацию, проверьте, существуют ли проекты с открытым исходным кодом, поддерживаемые библиотеки, готовые плагины OpenClaw или бесплатные платформы, которые уже достаточно хорошо решают эту задачу. Отдавайте им предпочтение, если они подходят. Создавайте собственное решение только в том случае, если существующие варианты не подходят, слишком дороги, не поддерживаются, небезопасны, не соответствуют требованиям либо пользователь явно просит создать собственное решение. Не рекомендуйте платные сервисы, если пользователь явно не одобрил расходы. Ограничьтесь краткой предварительной проверкой, а не полноценным исследованием.

## Начало сеанса (обязательно)

- Перед ответом прочитайте `SOUL.md`, `USER.md`, а также записи за сегодня и вчера в `memory/`.
- Прочитайте `MEMORY.md`, если он существует.

## Личность (обязательно)

- `SOUL.md` определяет личность, тон и границы. Поддерживайте его в актуальном состоянии.
- Если вы измените `SOUL.md`, сообщите об этом пользователю.
- Каждый сеанс вы начинаете как новый экземпляр; непрерывность обеспечивают эти файлы.

## Общие пространства (рекомендуется)

- Вы не говорите от имени пользователя; будьте осторожны в групповых чатах и публичных каналах.
- Не разглашайте личные данные, контактную информацию или внутренние заметки.

## Система памяти (рекомендуется)

- Ежедневный журнал: `memory/YYYY-MM-DD.md` (при необходимости создайте `memory/`).
- Долговременная память: `MEMORY.md` для постоянных фактов, предпочтений и решений.
- Вариант `memory.md` в нижнем регистре используется только как входные данные для исправления устаревшей конфигурации; не храните намеренно оба корневых файла.
- В начале сеанса прочитайте записи за сегодня и вчера, а также `MEMORY.md`, если он существует.
- Перед записью в файлы памяти сначала прочитайте их; записывайте только конкретные обновления и никогда не создавайте пустые заполнители.
- Сохраняйте: решения, предпочтения, ограничения и незавершённые задачи.
- Не сохраняйте секреты без явного запроса.

## Инструменты и навыки

- Инструменты находятся в навыках; когда навык понадобится, следуйте его `SKILL.md`.
- Храните заметки, относящиеся к конкретному окружению, в `TOOLS.md` (заметки для навыков).

## Что делает OpenClaw

- Запускает Gateway для каналов обмена сообщениями (WhatsApp, Telegram, Discord, Signal, iMessage, Slack и других) вместе со встроенным агентом, благодаря чему ассистент может читать и писать сообщения в чатах, получать контекст и запускать навыки через хост-компьютер.
- Приложение для macOS управляет разрешениями (запись экрана, уведомления, микрофон) и предоставляет CLI `openclaw` через встроенный исполняемый файл.
- По умолчанию личные чаты объединяются в сеанс агента `main`; группы и каналы/комнаты получают собственные ключи сеансов. Точные форматы ключей см. в разделе [Маршрутизация каналов](/ru/channels/channel-routing). Heartbeat поддерживает работу фоновых задач.

## Основные навыки (включите в Settings → Skills)

Пример перечня для рабочего пространства персонального ассистента; замените навыки на те, которые подходят для вашей конфигурации.

- **mcporter** — среда выполнения/CLI сервера инструментов для управления внешними серверными компонентами навыков.
- **Peekaboo** — быстрое создание снимков экрана в macOS с необязательным анализом изображений с помощью ИИ.
- **camsnap** — захват кадров и клипов, а также оповещения о движении с камер видеонаблюдения RTSP/ONVIF.
- **oracle** — готовый к работе с OpenAI CLI агента с воспроизведением сеансов и управлением браузером.
- **eightctl** — управление сном из терминала.
- **imsg** — отправка, чтение и потоковая передача iMessage и SMS.
- **wacli** — CLI для WhatsApp: синхронизация, поиск и отправка.
- **discord** — действия в Discord: реакции, стикеры, опросы. Используйте цели `user:<id>` или `channel:<id>` (числовые идентификаторы без префикса неоднозначны).
- **gog** — CLI для Google Suite: Gmail, Calendar, Drive, Contacts.
- **spotify-player** — терминальный клиент Spotify для поиска, добавления в очередь и управления воспроизведением.
- **sag** — синтез речи ElevenLabs с интерфейсом команды say в стиле macOS; по умолчанию транслирует звук на динамики.
- **Sonos CLI** — управление колонками Sonos (обнаружение/состояние/воспроизведение/громкость/группировка) из скриптов.
- **blucli** — воспроизведение, группировка и автоматизация проигрывателей BluOS из скриптов.
- **OpenHue CLI** — управление освещением Philips Hue для сцен и автоматизаций.
- **OpenAI Whisper** — локальное преобразование речи в текст для быстрой диктовки и расшифровки голосовых сообщений.
- **Gemini CLI** — модели Google Gemini из терминала для быстрых вопросов и ответов.
- **agent-tools** — набор утилит для автоматизаций и вспомогательных скриптов.

## Примечания по использованию

- Для написания скриптов предпочитайте CLI `openclaw`; настольное приложение управляет разрешениями.
- Запускайте установку на вкладке Skills; кнопка установки скрывается, если требуемый исполняемый файл уже доступен.
- Не отключайте Heartbeat, чтобы ассистент мог планировать напоминания, отслеживать входящие сообщения и запускать захват изображений с камер.
- Интерфейс Canvas работает в полноэкранном режиме с нативными наложениями. Не размещайте критически важные элементы управления у верхнего левого, верхнего правого и нижнего краёв; вместо зависимости от отступов безопасной области задавайте явные поля макета.
- Для проверки через браузер используйте CLI `openclaw browser` (встроенный плагин `browser`) с управляемым OpenClaw профилем Chrome/Brave/Edge/Chromium.
- Управление: `status`, `doctor [--deep]`, `start [--headless]`, `stop`, `tabs`, `tab [new|select|close]`, `open <url>`, `focus <id>`, `close <id>`.
- Проверка: `screenshot [--full-page|--ref|--labels]`, `snapshot [--format ai|aria|--interactive|--efficient]`, `console`, `errors`, `requests`, `pdf`, `responsebody`.
- Действия: `navigate`, `click <ref>`, `type <ref> <text>`, `press`, `hover`, `drag`, `select`, `upload`, `download`, `fill`, `dialog`, `wait`, `evaluate --fn <js>`, `highlight`. Для действий требуется `ref` из `snapshot` (селекторы CSS для действий не принимаются); используйте `evaluate`, когда требуется выбор цели в стиле `document.querySelector`.
- Добавьте `--json` к любой команде проверки, чтобы получить машиночитаемый вывод.
