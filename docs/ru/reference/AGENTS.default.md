## Первый запуск (рекомендуется)

OpenClaw использует отдельный каталог рабочей области для агента. По умолчанию: `~/.openclaw/workspace` (настраивается через `agents.defaults.workspace`).

1. Создайте рабочую область (если она еще не существует):

```bash
mkdir -p ~/.openclaw/workspace
```

2. Скопируйте шаблоны рабочей области по умолчанию в рабочую область:

```bash
cp docs/reference/templates/AGENTS.md ~/.openclaw/workspace/AGENTS.md
cp docs/reference/templates/SOUL.md ~/.openclaw/workspace/SOUL.md
cp docs/reference/templates/TOOLS.md ~/.openclaw/workspace/TOOLS.md
```

3. Необязательно: если вам нужен список навыков персонального ассистента, замените AGENTS.md этим файлом:

```bash
cp docs/reference/AGENTS.default.md ~/.openclaw/workspace/AGENTS.md
```

4. Необязательно: выберите другую рабочую область, задав `agents.defaults.workspace` (поддерживает `~`):

```json5
{
  agents: { defaults: { workspace: "~/.openclaw/workspace" } },
}
```

## Настройки безопасности по умолчанию

- Не выгружайте каталоги или секреты в чат.
- Не запускайте разрушительные команды, если об этом явно не попросили.
- Перед изменением конфигурации или планировщиков (например, crontab, модулей systemd, конфигураций nginx или shell rc-файлов) сначала проверьте текущее состояние и по умолчанию сохраняйте/объединяйте.
- Не отправляйте частичные/потоковые ответы во внешние поверхности обмена сообщениями (только финальные ответы).

## Начало сеанса (обязательно)

- Прочитайте `SOUL.md`, `USER.md` и сегодня+вчера в `memory/`.
- Прочитайте `MEMORY.md`, если он есть.
- Сделайте это перед ответом.

## Soul (обязательно)

- `SOUL.md` определяет идентичность, тон и границы. Поддерживайте его в актуальном состоянии.
- Если вы изменяете `SOUL.md`, сообщите пользователю.
- Каждый сеанс вы являетесь новым экземпляром; непрерывность хранится в этих файлах.

## Общие пространства (рекомендуется)

- Вы не являетесь голосом пользователя; будьте осторожны в групповых чатах или публичных каналах.
- Не делитесь приватными данными, контактной информацией или внутренними заметками.

## Система памяти (рекомендуется)

- Ежедневный журнал: `memory/YYYY-MM-DD.md` (создайте `memory/`, если нужно).
- Долгосрочная память: `MEMORY.md` для долговечных фактов, предпочтений и решений.
- Нижнерегистровый `memory.md` является только входом для восстановления устаревших данных; не храните оба корневых файла намеренно.
- При запуске сеанса прочитайте сегодня + вчера + `MEMORY.md`, если он есть.
- Перед записью файлов памяти сначала прочитайте их; записывайте только конкретные обновления, никогда пустые заполнители.
- Фиксируйте: решения, предпочтения, ограничения, открытые циклы.
- Избегайте секретов, если об этом явно не попросили.

## Что делает OpenClaw

- Запускает WhatsApp gateway + встроенного агента OpenClaw, чтобы ассистент мог читать/писать чаты, получать контекст и запускать Skills через хост Mac.
- Приложение macOS управляет разрешениями (запись экрана, уведомления, микрофон) и предоставляет CLI `openclaw` через свой встроенный бинарный файл.
- Личные чаты по умолчанию сворачиваются в сеанс агента `main`; группы остаются изолированными как `agent:<agentId>:<channel>:group:<id>` (комнаты/каналы: `agent:<agentId>:<channel>:channel:<id>`); heartbeats поддерживают фоновые задачи активными.

## Основные Skills (включите в Settings → Skills)

- **mcporter** - Среда выполнения/CLI сервера инструментов для управления внешними бэкендами навыков.
- **Peekaboo** - Быстрые снимки экрана macOS с необязательным AI-анализом изображений.
- **camsnap** - Захватывайте кадры, клипы или оповещения о движении с камер безопасности RTSP/ONVIF.
- **oracle** - Готовый к OpenAI CLI агента с повтором сеанса и управлением браузером.
- **eightctl** - Управляйте своим сном из терминала.
- **imsg** - Отправляйте, читайте, транслируйте iMessage и SMS.
- **wacli** - CLI WhatsApp: синхронизация, поиск, отправка.
- **discord** - Действия Discord: реакции, стикеры, опросы. Используйте цели `user:<id>` или `channel:<id>` (простые числовые id неоднозначны).
- **gog** - CLI Google Suite: Gmail, Календарь, Drive, Контакты.
- **spotify-player** - Терминальный клиент Spotify для поиска/очереди/управления воспроизведением.
- **sag** - Речь ElevenLabs с mac-стилем UX say; по умолчанию потоково выводит на динамики.
- **Sonos CLI** - Управляйте колонками Sonos (обнаружение/статус/воспроизведение/громкость/группировка) из скриптов.
- **blucli** - Воспроизводите, группируйте и автоматизируйте плееры BluOS из скриптов.
- **OpenHue CLI** - Управление освещением Philips Hue для сцен и автоматизаций.
- **OpenAI Whisper** - Локальное преобразование речи в текст для быстрой диктовки и расшифровки голосовой почты.
- **Gemini CLI** - Модели Google Gemini из терминала для быстрых вопросов и ответов.
- **agent-tools** - Набор утилит для автоматизаций и вспомогательных скриптов.

## Примечания по использованию

- Для скриптинга предпочитайте CLI `openclaw`; приложение Mac обрабатывает разрешения.
- Запускайте установки из вкладки Skills; она скрывает кнопку, если бинарный файл уже присутствует.
- Держите heartbeats включенными, чтобы ассистент мог планировать напоминания, отслеживать входящие и запускать захват с камер.
- UI Canvas работает в полноэкранном режиме с нативными оверлеями. Избегайте размещения критически важных элементов управления в верхнем левом/верхнем правом/нижних краях; добавляйте явные отступы в макете и не полагайтесь на safe-area insets.
- Для проверки через браузер используйте `openclaw browser` (tabs/status/screenshot) с управляемым OpenClaw профилем Chrome.
- Для проверки DOM используйте `openclaw browser eval|query|dom|snapshot` (и `--json`/`--out`, когда нужен машинный вывод).
- Для взаимодействий используйте `openclaw browser click|type|hover|drag|select|upload|press|wait|navigate|back|evaluate|run` (click/type требуют ссылок snapshot; используйте `evaluate` для CSS-селекторов).
