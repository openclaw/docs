OpenClaw выполняет команды оболочки с помощью инструмента `exec` и хранит длительно выполняющиеся задачи в памяти. Инструмент `process` управляет этими фоновыми сеансами.

## Инструмент exec

Параметры:

| Параметр     | Описание                                                                                                                                                                                                 |
| ------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `command`    | Обязательный. Команда оболочки для выполнения.                                                                                                                                                           |
| `workdir`    | Рабочий каталог; не указывайте, чтобы использовать текущий рабочий каталог по умолчанию.                                                                                                                 |
| `env`        | Дополнительные переменные окружения для команды.                                                                                                                                                         |
| `yieldMs`    | Время ожидания в миллисекундах перед переводом в фоновый режим (по умолчанию 10000).                                                                                                                      |
| `background` | Немедленно запустить в фоновом режиме.                                                                                                                                                                   |
| `timeout`    | Время ожидания в секундах (по умолчанию `tools.exec.timeoutSec`); по истечении процесс завершается. Установите `timeout: 0`, чтобы отключить ограничение времени процесса exec для этого вызова.            |
| `pty`        | Запустить в псевдотерминале, если он доступен (для CLI, требующих TTY, и агентов программирования).                                                                                                       |
| `elevated`   | Запустить вне песочницы, если режим повышенных привилегий включён или разрешён (по умолчанию `gateway`, либо `node`, если целевой узел exec — `node`).                                                     |
| `host`       | Цель выполнения exec: `auto`, `sandbox`, `gateway` или `node`.                                                                                                                                           |
| `node`       | Идентификатор или имя Node, используется с `host: "node"`.                                                                                                                                               |

Поведение:

- При выполнении на переднем плане вывод возвращается напрямую.
- При переводе в фоновый режим (явно или по истечении `yieldMs`) инструмент возвращает `status: "running"` + `sessionId` и короткий фрагмент последних строк вывода.
- Запуски в фоновом режиме и с `yieldMs` наследуют `tools.exec.timeoutSec`, если при вызове явно не передан `timeout`.
- Вывод остаётся в памяти, пока состояние сеанса не будет опрошено или очищено.
- Если инструмент `process` запрещён, `exec` выполняется синхронно и игнорирует `yieldMs`/`background`.
- Запущенные команды exec получают `OPENCLAW_SHELL=exec` для контекстно-зависимых правил оболочки и профиля.
- Для длительной работы, начинающейся сейчас: запустите её один раз и дождитесь автоматического уведомления о завершении (если оно включено), когда команда выведет данные или завершится с ошибкой.
- Если автоматическое уведомление о завершении недоступно либо требуется подтвердить успешное завершение команды без вывода, опрашивайте состояние с помощью `process`.
- Не имитируйте напоминания или отложенные действия циклами `sleep` либо повторными опросами — для будущих задач используйте Cron.

### Переопределения через переменные окружения

| Переменная                               | Действие                                                                                                                                  |
| ---------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| `OPENCLAW_BASH_YIELD_MS`                 | Время ожидания по умолчанию перед переводом в фоновый режим (мс). По умолчанию 10000, ограничивается диапазоном 10–120000.                 |
| `OPENCLAW_BASH_MAX_OUTPUT_CHARS`         | Ограничение объёма вывода в памяти (в символах).                                                                                          |
| `OPENCLAW_BASH_PENDING_MAX_OUTPUT_CHARS` | Ограничение ожидающего обработки вывода stdout/stderr для каждого потока (в символах).                                                     |
| `OPENCLAW_BASH_JOB_TTL_MS`               | TTL завершённых сеансов (мс), ограниченный диапазоном от 1 мин до 3 ч.                                                                    |
| `OPENCLAW_PROCESS_INPUT_WAIT_IDLE_MS`    | Порог отсутствия вывода, после которого доступные для записи фоновые сеансы помечаются как, вероятно, ожидающие ввода. По умолчанию 15000. |

### Конфигурация (предпочтительнее переопределений через переменные окружения)

| Ключ                                  | Значение по умолчанию | Действие                                                                                                         |
| ------------------------------------- | --------------------- | ---------------------------------------------------------------------------------------------------------------- |
| `tools.exec.backgroundMs`             | 10000                 | То же, что `OPENCLAW_BASH_YIELD_MS`.                                                                              |
| `tools.exec.timeoutSec`               | 1800                  | Время ожидания по умолчанию для каждого вызова.                                                                   |
| `tools.exec.cleanupMs`                | 1800000               | То же, что `OPENCLAW_BASH_JOB_TTL_MS`.                                                                            |
| `tools.exec.notifyOnExit`             | true                  | Добавлять системное событие в очередь и запрашивать Heartbeat при завершении фонового процесса exec.             |
| `tools.exec.notifyOnExitEmptySuccess` | false                 | Также добавлять события завершения для успешно завершившихся фоновых запусков без вывода.                         |

## Связывание дочерних процессов

При запуске длительно работающих дочерних процессов вне инструментов exec/process (перезапуски CLI, вспомогательные процессы Gateway) подключайте вспомогательный мост дочерних процессов, чтобы сигналы завершения перенаправлялись, а обработчики отключались при выходе или ошибке. Это предотвращает появление осиротевших процессов в systemd и обеспечивает единообразное завершение работы на разных платформах.

## Инструмент process

Действия:

| Действие    | Результат                                                                                                           |
| ----------- | ------------------------------------------------------------------------------------------------------------------- |
| `list`      | Выполняющиеся и завершённые сеансы.                                                                                 |
| `poll`      | Получить новый вывод сеанса (также сообщает статус завершения).                                                     |
| `log`       | Прочитать объединённый вывод и подсказки по восстановлению ввода. Поддерживает `offset` + `limit`.                   |
| `write`     | Отправить данные в stdin (`data`, необязательный `eof`).                                                            |
| `send-keys` | Отправить явные клавиши или байты в сеанс на основе PTY.                                                            |
| `submit`    | Отправить Enter/возврат каретки в сеанс на основе PTY.                                                              |
| `paste`     | Отправить текст без изменений, при необходимости обернув его в режим вставки со скобками.                           |
| `kill`      | Завершить фоновый сеанс.                                                                                            |
| `clear`     | Удалить завершённый сеанс из памяти.                                                                                |
| `remove`    | Завершить выполняющийся сеанс либо очистить завершённый.                                                            |

Примечания:

- В список попадают и сохраняются только фоновые сеансы — исключительно в памяти, не на диске. При перезапуске процесса сеансы теряются.
- Активный фоновый сеанс блокирует кооперативную приостановку хоста и безопасный перезапуск Gateway, пока владелец процесса не подтвердит его фактическое завершение.
- `process remove` может скрыть выполняющийся сеанс сразу после запроса на завершение; приостановка и перезапуск останутся заблокированными до подтверждения выхода.
- Журналы сеансов сохраняются в истории чата, только если выполнить `process poll`/`log` и результат инструмента будет записан.
- Область действия `process` ограничена отдельным агентом; он видит только сеансы, запущенные этим агентом.
- Используйте `poll`/`log` для проверки состояния, просмотра журналов или подтверждения завершения, когда автоматическое уведомление о завершении недоступно.
- Перед восстановлением интерактивного CLI используйте `log`, чтобы одновременно видеть текущую расшифровку сеанса, состояние stdin и подсказку об ожидании ввода.
- Используйте `write`/`send-keys`/`submit`/`paste`/`kill`, когда требуется ввод или вмешательство.
- `process list` содержит вычисляемое поле `name` (командный глагол + цель) для быстрого просмотра.
- `process list`, `poll` и `log` сообщают `waitingForInput`, только если stdin сеанса по-прежнему доступен для записи и сеанс бездействует дольше порога ожидания ввода (по умолчанию 15000 мс, `OPENCLAW_PROCESS_INPUT_WAIT_IDLE_MS`).
- `process log` использует построчные `offset`/`limit`. Если оба параметра не указаны, возвращаются последние 200 строк с подсказкой по постраничному просмотру. Если задан `offset`, а `limit` не задан, возвращаются все строки от `offset` до конца (без ограничения в 200 строк).
- Параметр `timeout` действия `poll` задаёт максимальное время ожидания в миллисекундах перед возвратом; значения свыше 30000 ограничиваются до 30000.
- Опрос предназначен для проверки состояния по запросу, а не для планирования циклов ожидания. Если работу нужно выполнить позже, используйте Cron.

## Примеры

Запустить длительную задачу и опросить её позже:

```json
{ "tool": "exec", "command": "sleep 5 && echo done", "yieldMs": 1000 }
```

```json
{ "tool": "process", "action": "poll", "sessionId": "<id>" }
```

Проверить интерактивный сеанс перед отправкой ввода:

```json
{ "tool": "process", "action": "log", "sessionId": "<id>" }
```

Немедленно запустить в фоновом режиме:

```json
{ "tool": "exec", "command": "npm run build", "background": true }
```

Отправить данные в stdin:

```json
{ "tool": "process", "action": "write", "sessionId": "<id>", "data": "y\n" }
```

Отправить клавиши PTY:

```json
{ "tool": "process", "action": "send-keys", "sessionId": "<id>", "keys": ["C-c"] }
```

Отправить текущую строку:

```json
{ "tool": "process", "action": "submit", "sessionId": "<id>" }
```

Вставить текст без изменений:

```json
{ "tool": "process", "action": "paste", "sessionId": "<id>", "text": "line1\nline2\n" }
```
