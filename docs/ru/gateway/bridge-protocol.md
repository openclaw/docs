<Warning>
TCP-мост **удалён**. Текущие сборки OpenClaw не содержат слушатель моста, а ключи конфигурации `bridge.*` больше не входят в схему. Эта страница предназначена только для исторической справки. Для всех клиентов узлов и операторов используйте [протокол Gateway](/ru/gateway/protocol).
</Warning>

## Зачем он существовал

- **Граница безопасности**: вместо полной поверхности API Gateway предоставлялся небольшой список разрешённых операций.
- **Сопряжение и идентификация узла**: допуск узла контролировался Gateway и был привязан к отдельному токену каждого узла.
- **Удобство обнаружения**: узлы могли обнаруживать Gateway через Bonjour в локальной сети или подключаться напрямую через tailnet.
- **Локальный WS**: полная плоскость управления WS оставалась локальной, если только для неё не создавался туннель через SSH.

## Транспорт

- TCP, по одному объекту JSON в строке (JSONL).
- Необязательный TLS (`bridge.tls.enabled: true`).
- Порт слушателя по умолчанию — `18790`.

Когда TLS был включён, TXT-записи обнаружения содержали `bridgeTls=1` и `bridgeTlsSha256` в качестве несекретной подсказки. TXT-записи Bonjour/mDNS не проходят проверку подлинности; клиенты не могли считать объявленный отпечаток авторитетной привязкой без дополнительной проверки по независимому каналу.

## Установление соединения и сопряжение

1. Клиент отправляет `hello` с метаданными узла и токеном (если узел уже сопряжён).
2. Если сопряжение отсутствует, Gateway отвечает `error` (`NOT_PAIRED` / `UNAUTHORIZED`).
3. Клиент отправляет `pair-request`.
4. Gateway ожидает подтверждения, а затем отправляет `pair-ok` и `hello-ok`.

Ранее `hello-ok` возвращал `serverName`; теперь поверхности размещённых плагинов объявляются через `pluginSurfaceUrls` в текущем протоколе Gateway (Canvas/A2UI использует `pluginSurfaceUrls.canvas`).

## Фреймы

От клиента к Gateway:

- `req` / `res`: RPC Gateway с ограниченной областью действия (чат, сеансы, конфигурация, состояние, активация голосом, skills.bins).
- `event`: сигналы узла (расшифровка речи, запрос агента, подписка на чат, жизненный цикл выполнения).

От Gateway к клиенту:

- `invoke` / `invoke-res`: команды узла (`canvas.*`, `camera.*`, `screen.record`, `location.get`, `sms.send`).
- `event`: обновления чата для сеансов с активной подпиской.
- `ping` / `pong`: поддержание соединения.

Проверка списка разрешённых операций выполнялась в `src/gateway/server-bridge.ts` (удалён).

## События жизненного цикла выполнения

Узлы отправляли `exec.finished`, чтобы сообщить о завершённой активности `system.run`, которая преобразовывалась Gateway в системные события (устаревшие узлы также могли отправлять `exec.started`). `exec.denied` отмечал отклонённую попытку `system.run` как окончательный отказ без постановки системного события в очередь и без пробуждения работы агента.

Поля полезной нагрузки (все необязательны, если не указано иное):

| Поле                             | Примечания                                                                                                       |
| -------------------------------- | ---------------------------------------------------------------------------------------------------------------- |
| `sessionKey`                     | Обязательно. Сеанс агента для сопоставления события, а для `exec.finished` — также для доставки системного события. |
| `runId`                          | Уникальный идентификатор выполнения для группировки.                                                             |
| `command`                        | Необработанная или отформатированная строка команды.                                                             |
| `exitCode`, `timedOut`, `output` | Сведения о завершении (только для завершённого выполнения).                                                       |
| `reason`                         | Причина отказа (только для отклонённого выполнения).                                                             |

## Историческое использование tailnet

- Привязка моста к IP-адресу tailnet: `bridge.bind: "tailnet"` в `~/.openclaw/openclaw.json` (только историческая справка; `bridge.*` больше не является допустимой конфигурацией).
- Клиенты подключались по имени MagicDNS или IP-адресу tailnet.
- Bonjour не работает между разными сетями; в противном случае требовался глобальный DNS-SD либо вручную указанные хост и порт.

## Управление версиями

Мост неявно использовал версию v1 без согласования минимальной и максимальной версий. Текущие клиенты узлов и операторов используют WebSocket-[протокол Gateway](/ru/gateway/protocol), который согласовывает диапазон версий протокола.
