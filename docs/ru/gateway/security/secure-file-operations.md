OpenClaw использует [`@openclaw/fs-safe`](https://github.com/openclaw/fs-safe) для локальных файловых операций, критичных для безопасности: чтения и записи в пределах корневого каталога, атомарной замены, извлечения архивов, временных рабочих пространств, состояния в формате JSON и обработки файлов с секретами.

Это **защитный механизм библиотеки** для доверенного кода OpenClaw, который получает недоверенные имена путей, а не песочница. Реальную область возможного воздействия по-прежнему определяют разрешения файловой системы хоста, пользователи ОС, контейнеры и политика агента и инструментов.

## По умолчанию: без вспомогательного процесса Python

OpenClaw по умолчанию **отключает** вспомогательный процесс Python для POSIX в fs-safe:

- Gateway не должен запускать постоянный вспомогательный процесс Python, если оператор явно не включил его;
- большинству установок не требуется дополнительная защита операций изменения родительских каталогов;
- отключение Python обеспечивает предсказуемое поведение среды выполнения в настольных приложениях, Docker, CI и средах комплектных приложений.

OpenClaw изменяет только _значение по умолчанию_. Явная настройка всегда имеет приоритет:

```bash
# Поведение OpenClaw по умолчанию: резервные механизмы fs-safe только на Node.
OPENCLAW_FS_SAFE_PYTHON_MODE=off

# Использовать вспомогательный процесс при его наличии, иначе перейти к резервному механизму.
OPENCLAW_FS_SAFE_PYTHON_MODE=auto

# Безопасно завершить работу с ошибкой, если вспомогательный процесс не удаётся запустить.
OPENCLAW_FS_SAFE_PYTHON_MODE=require

# Необязательный явный путь к интерпретатору.
OPENCLAW_FS_SAFE_PYTHON=/usr/bin/python3
```

Также поддерживаются общие имена переменных окружения fs-safe: `FS_SAFE_PYTHON_MODE` и `FS_SAFE_PYTHON`.

Используйте `require` (а не `auto`), если вспомогательный процесс является частью вашей стратегии безопасности; при `auto` система незаметно переходит к поведению только на Node, если вспомогательный процесс не удаётся запустить.

## Что остаётся защищённым без Python

Когда вспомогательный процесс отключён, OpenClaw по-прежнему использует защитные механизмы fs-safe только на Node:

- отклоняет выходы за пределы каталога через относительные пути (`..`), абсолютные пути и разделители путей там, где разрешены только простые имена;
- выполняет операции через доверенный дескриптор корневого каталога вместо разрозненных проверок `path.resolve(...).startsWith(...)`;
- отклоняет шаблоны с символическими и жёсткими ссылками в API, где этого требует политика;
- открывает файлы с проверкой идентичности, когда API возвращает или принимает содержимое файлов;
- записывает файлы состояния и конфигурации через временный файл в том же каталоге с последующим атомарным переименованием;
- применяет ограничения объёма данных при чтении и извлечении архивов;
- устанавливает закрытые режимы доступа для секретов и файлов состояния там, где этого требует API.

Это соответствует обычной модели угроз OpenClaw: доверенный код Gateway обрабатывает недоверенные пути, полученные от модели, Plugin или канала, в пределах единой доверенной зоны оператора.

## Что добавляет Python

В POSIX необязательный вспомогательный процесс поддерживает один постоянный процесс Python и использует файловые операции относительно файлового дескриптора для изменений родительских каталогов: переименования, удаления, создания каталогов, получения метаданных и списков, а также некоторых операций записи.

Это сокращает окна состязаний между процессами с одинаковым UID, в которых другой процесс может подменить родительский каталог между проверкой и изменением, — дополнительный уровень защиты на хостах, где недоверенные локальные процессы могут изменять те же каталоги, с которыми работает OpenClaw.

Если в вашей среде развёртывания существует такой риск и наличие Python гарантировано, задайте:

```bash
OPENCLAW_FS_SAFE_PYTHON_MODE=require
```

## Рекомендации для Plugin и ядра

- Доступ Plugin к файлам должен выполняться через вспомогательные функции `openclaw/plugin-sdk/*`, а не напрямую через `fs`, если путь поступает из сообщения, вывода модели, конфигурации или входных данных Plugin.
- Код ядра должен использовать обёртки fs-safe из `src/infra/*`, чтобы политика процесса OpenClaw применялась единообразно.
- Для извлечения архивов следует использовать вспомогательные функции fs-safe для архивов с явными ограничениями размера, количества элементов, ссылок и места назначения.
- Для секретов следует использовать вспомогательные функции OpenClaw для секретов или функции fs-safe для секретов и закрытого состояния; не реализуйте собственные проверки режимов доступа вокруг `fs.writeFile`.
- Для изоляции от недоверенных локальных пользователей не полагайтесь только на fs-safe. Запускайте отдельные экземпляры Gateway от имени разных пользователей ОС или на разных хостах либо используйте песочницу.
