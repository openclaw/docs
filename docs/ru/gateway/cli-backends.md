OpenClaw может запускать локальный CLI ИИ в качестве резервного варианта только для текста, когда поставщики API недоступны, ограничивают частоту запросов или работают некорректно. Этот механизм намеренно консервативен:

- Инструменты OpenClaw не внедряются напрямую, но бэкенд с `bundleMcp: true` может получать инструменты Gateway через MCP-мост по local loopback.
- Потоковая передача JSONL для поддерживающих её CLI.
- Поддерживаются сеансы, поэтому последующие запросы сохраняют связность контекста.
- Изображения передаются, если CLI принимает пути к изображениям.

Используйте этот механизм как страховку для текстовых ответов, которые «всегда работают», а не как основной способ взаимодействия. Для полноценной среды выполнения с управлением сеансами ACP, фоновыми задачами, привязкой веток и разговоров, а также постоянными внешними сеансами программирования используйте вместо этого [агентов ACP](/ru/tools/acp-agents); бэкенды CLI не являются ACP.

<Tip>
  Создаёте новый Plugin бэкенда? См. [плагины бэкендов CLI](/ru/plugins/cli-backend-plugins). На этой странице описаны настройка и эксплуатация уже зарегистрированного бэкенда.
</Tip>

## Быстрый старт

Встроенный Plugin Anthropic регистрирует бэкенд `claude-cli` по умолчанию, поэтому достаточно установить Claude Code и войти в учётную запись — дополнительная конфигурация не требуется:

```bash
openclaw agent --agent main --message "hi" --model claude-cli/claude-sonnet-4-6
```

Если явный список агентов не настроен, `main` используется как идентификатор агента по умолчанию; в противном случае замените его идентификатором своего агента.

Если Gateway работает под управлением launchd/systemd с минимальным `PATH`, укажите путь к исполняемому файлу явно:

```json5
{
  agents: {
    defaults: {
      cliBackends: {
        "claude-cli": {
          command: "/opt/homebrew/bin/claude",
        },
      },
    },
  },
}
```

Если вы используете встроенный бэкенд CLI как основного поставщика сообщений на узле Gateway, OpenClaw автоматически загружает владеющий им встроенный Plugin, когда конфигурация ссылается на этот бэкенд в ссылке на модель или в `agents.defaults.cliBackends`.

## Использование в качестве резервного варианта

Добавьте бэкенд CLI в список резервных вариантов, чтобы он запускался только при сбое основных моделей:

```json5
{
  agents: {
    defaults: {
      model: {
        primary: "anthropic/claude-opus-4-6",
        fallbacks: ["claude-cli/claude-sonnet-4-6"],
      },
      models: {
        "anthropic/claude-opus-4-6": { alias: "Opus" },
        "claude-cli/claude-sonnet-4-6": {},
      },
    },
  },
}
```

Если вы используете `agents.defaults.models` как список разрешённых моделей, добавьте туда и модели бэкенда CLI. При сбое основного поставщика из-за аутентификации, ограничений частоты запросов или тайм-аутов OpenClaw следующим пробует бэкенд CLI.

## Конфигурация

Все бэкенды CLI находятся в `agents.defaults.cliBackends` и индексируются по идентификатору поставщика, например `claude-cli` или `my-cli`. Идентификатор поставщика становится левой частью ссылки на модель: `<provider>/<model>`.

```json5
{
  agents: {
    defaults: {
      cliBackends: {
        "my-cli": {
          command: "my-cli",
          args: ["--json"],
          output: "json",
          input: "arg",
          modelArg: "--model",
          modelAliases: {
            "claude-opus-4-6": "opus",
            "claude-sonnet-4-6": "sonnet",
          },
          sessionArg: "--session",
          sessionMode: "existing",
          sessionIdFields: ["session_id", "conversation_id"],
          systemPromptArg: "--system",
          // Отдельный флаг файла промпта:
          // systemPromptFileArg: "--system-file",
          // Или флаг переопределения конфигурации в стиле Codex:
          // systemPromptFileConfigArg: "-c",
          // systemPromptFileConfigKey: "model_instructions_file",
          systemPromptWhen: "first",
          imageArg: "--image",
          imageMode: "repeat",
          // Включайте только в том случае, если этот бэкенд может заново заполнять
          // недействительные сеансы из ограниченной необработанной истории
          // транскрипта OpenClaw до Compaction.
          reseedFromRawTranscriptWhenUncompacted: true,
          serialize: true,
        },
      },
    },
  },
}
```

## Принцип работы

1. Выбирает бэкенд по префиксу поставщика (`claude-cli/...`).
2. Формирует системный промпт, используя тот же промпт OpenClaw и контекст рабочего пространства.
3. Запускает CLI с идентификатором сеанса, если он поддерживается, чтобы история оставалась согласованной. Встроенный бэкенд `claude-cli` поддерживает отдельный процесс Claude stdio для каждого сеанса OpenClaw и отправляет последующие запросы через stdin в формате stream-json.
4. Разбирает вывод в формате JSON или обычного текста и возвращает итоговый текст.
5. Сохраняет идентификаторы сеансов отдельно для каждого бэкенда, чтобы последующие запросы повторно использовали тот же сеанс CLI.

### Особенности Claude CLI

Встроенный бэкенд `claude-cli` предпочитает собственный механизм разрешения Skills в Claude Code. Если текущий снимок Skills содержит хотя бы один выбранный навык с материализованным путём, OpenClaw передаёт временный Plugin Claude Code через `--plugin-dir` и не включает дублирующий каталог Skills OpenClaw в добавляемый системный промпт. Если материализованный Plugin навыка отсутствует, OpenClaw сохраняет каталог в промпте как резервный вариант. Переопределения переменных среды и ключей API для навыка по-прежнему применяются к среде дочернего процесса на время запуска.

Claude CLI имеет собственный неинтерактивный режим разрешений; OpenClaw сопоставляет его с существующей политикой выполнения, а не добавляет конфигурацию, специфичную для Claude. Для управляемых OpenClaw активных сеансов Claude фактическая политика выполнения является определяющей: режим YOLO (`tools.exec.security: "full"` и `tools.exec.ask: "off"`) запускает Claude с `--permission-mode bypassPermissions`, а ограничительная политика — с `--permission-mode default`. Настройки `agents.list[].tools.exec` для отдельного агента переопределяют глобальные `tools.exec` для этого агента. Необработанные аргументы бэкенда всё ещё могут содержать `--permission-mode`, но при запуске активных сеансов Claude значение этого флага нормализуется в соответствии с фактической политикой.

Бэкенд также сопоставляет уровни `/think` OpenClaw с собственным флагом `--effort` Claude Code: `minimal`/`low` -> `low`, `medium` -> `medium`, а `high`/`xhigh`/`max` передаются без изменений. Значение `adaptive` удаляет настроенные флаги `--effort` и не предоставляет замену, поэтому Claude Code определяет фактический уровень усилий по собственной среде, настройкам и значениям модели по умолчанию. Чтобы `/think` влиял на запускаемый CLI в других бэкендах CLI, владеющий ими Plugin должен объявить аналогичный преобразователь argv.

Прежде чем OpenClaw сможет использовать `claude-cli`, необходимо войти в Claude Code на том же узле:

```bash
claude auth login
claude auth status --text
openclaw models auth login --provider anthropic --method cli --set-default
```

При установке в Docker Claude Code должен быть установлен, а вход в учётную запись должен быть выполнен внутри сохраняемого домашнего каталога контейнера, а не только на узле; см. [бэкенд Claude CLI в Docker](/ru/install/docker#claude-cli-backend-in-docker).

Задавайте `agents.defaults.cliBackends.claude-cli.command` только в том случае, если исполняемый файл `claude` ещё не доступен через `PATH`.

## Сеансы

- Если CLI поддерживает сеансы, задайте `sessionArg`, например `--session-id`, или `sessionArgs` с заполнителем `{sessionId}`, если идентификатор требуется передать в нескольких флагах.
- Если CLI использует подкоманду возобновления с другими флагами, задайте `resumeArgs`, который заменяет `args` при возобновлении, и при необходимости `resumeOutput` для возобновлений не в формате JSON.
- `sessionMode`:
  - `always`: всегда передавать идентификатор сеанса; если сохранённого идентификатора нет, создаётся новый UUID.
  - `existing`: передавать идентификатор сеанса, только если он был сохранён ранее.
  - `none`: никогда не передавать идентификатор сеанса.
- Для `claude-cli` по умолчанию используются `liveSession: "claude-stdio"`, `output: "jsonl"` и `input: "stdin"`, поэтому последующие запросы повторно используют активный процесс Claude, в том числе при пользовательских конфигурациях без полей транспорта. Если Gateway перезапускается или бездействующий процесс завершается, OpenClaw возобновляет работу по сохранённому идентификатору сеанса Claude. Перед возобновлением сохранённые идентификаторы сеансов сверяются с доступным для чтения транскриптом проекта; при отсутствии транскрипта привязка очищается с записью `reason=transcript-missing` в журнал вместо неявного запуска нового сеанса с `--resume`.
- Для активных сеансов Claude действуют ограничители вывода JSONL: по умолчанию 8 МиБ и 20 000 необработанных строк JSONL на запрос. Эти значения можно увеличить для отдельного бэкенда с помощью `agents.defaults.cliBackends.claude-cli.reliability.outputLimits.maxTurnRawChars` и `maxTurnLines`; OpenClaw ограничивает их сверху значениями 64 МиБ и 100 000 строк.
- Сохранённые сеансы CLI обеспечивают непрерывность, которой управляет поставщик. Неявный ежедневный сброс сеанса их не прерывает; `/reset` и явные политики `session.reset` по-прежнему прерывают.
- Новые сеансы CLI обычно заново заполняются только из сводки Compaction OpenClaw и хвоста после Compaction. Для восстановления коротких сеансов, ставших недействительными до Compaction, бэкенд может включить `reseedFromRawTranscriptWhenUncompacted: true`. Повторное заполнение из необработанного транскрипта остаётся ограниченным и применяется только при безопасных причинах аннулирования, таких как отсутствие транскрипта CLI, потерянный хвост вызова инструмента, изменение политики сообщений, системного промпта, cwd или MCP, а также повторная попытка после истечения срока действия сеанса; при изменении профиля аутентификации или эпохи учётных данных история из необработанного транскрипта не используется никогда.

Сериализация: `serialize: true` сохраняет порядок запусков в одной линии выполнения; большинство CLI сериализуют запросы в одной линии поставщика. OpenClaw также прекращает повторное использование сохранённого сеанса CLI при изменении выбранной аутентификационной идентичности, включая изменение идентификатора профиля аутентификации, статического ключа API, статического токена или идентичности учётной записи OAuth, если CLI её предоставляет; сама по себе ротация токенов доступа или обновления OAuth сеанс не прерывает. Если CLI не предоставляет стабильный идентификатор учётной записи OAuth, OpenClaw позволяет этому CLI самостоятельно применять разрешения на возобновление.

## Вводный контекст резервного варианта из сеансов claude-cli

Когда попытка через `claude-cli` переключается после сбоя на кандидата, не являющегося CLI, из [`agents.defaults.model.fallbacks`](/ru/concepts/model-failover), OpenClaw снабжает следующую попытку вводным контекстом, извлечённым из локального транскрипта JSONL Claude Code в `~/.claude/projects/`, с отдельным ключом для каждого рабочего пространства. Без этого начального контекста резервный поставщик начинает без истории, поскольку собственный транскрипт сеанса OpenClaw для запусков `claude-cli` пуст.

- Для вводного контекста предпочтительно используется последняя сводка `/compact` или маркер `compact_boundary`, после чего добавляются последние запросы после границы в пределах ограничения по количеству символов. Запросы до границы отбрасываются, поскольку они уже представлены в сводке.
- Блоки инструментов объединяются в компактные подсказки `(tool call: name)` и `(tool result: …)`, чтобы честно соблюдать бюджет промпта; слишком большая сводка обрезается и помечается как `(truncated)`.
- Резервные переключения между `claude-cli` одного поставщика полагаются на собственный механизм `--resume` Claude и не используют вводный контекст.
- Для начального контекста повторно используется существующая проверка пути к файлу сеанса Claude, поэтому чтение произвольных путей невозможно.

## Изображения

Если ваш CLI принимает пути к изображениям, задайте `imageArg`:

```json5
imageArg: "--image",
imageMode: "repeat"
```

OpenClaw записывает изображения в кодировке base64 во временные файлы. Если задан `imageArg`, эти пути передаются как аргументы CLI; в противном случае OpenClaw добавляет пути к файлам в промпт посредством внедрения путей, что работает для CLI, которые автоматически загружают локальные файлы по обычным путям.

## Ввод и вывод

- `output: "text"` по умолчанию рассматривает stdout как итоговый ответ.
- `output: "json"` пытается разобрать JSON и извлечь текст вместе с идентификатором сеанса.
- `output: "jsonl"` разбирает поток JSONL и извлекает итоговое сообщение агента вместе с идентификаторами сеанса, если они присутствуют.
- Для вывода Gemini CLI в формате JSON OpenClaw считывает текст ответа из `response`, а сведения об использовании — из `stats`, если `usage` отсутствует или пуст. Встроенная конфигурация Gemini CLI по умолчанию использует `stream-json`; старые переопределения `--output-format json` по-прежнему используют анализатор JSON.

Режимы ввода:

- `input: "arg"` по умолчанию передаёт промпт как последний аргумент CLI.
- `input: "stdin"` отправляет промпт через stdin.
- Если промпт очень длинный и задано `maxPromptArgChars`, вместо аргумента используется stdin.

## Значения по умолчанию, принадлежащие Plugin

Значения бэкенда CLI по умолчанию являются частью поверхности Plugin:

- Плагины регистрируют их с помощью `api.registerCliBackend(...)`.
- Поле `id` бэкенда становится префиксом поставщика в ссылках на модели.
- Пользовательская конфигурация в `agents.defaults.cliBackends.<id>` по-прежнему переопределяет значение Plugin по умолчанию.
- Очистка конфигурации, специфичной для бэкенда, остаётся в ведении Plugin и выполняется через необязательный обработчик `normalizeConfig`.

Anthropic владеет `claude-cli`, а Google — `google-gemini-cli`. Запуски агента OpenAI Codex используют среду app-server Codex через `openai/*`; OpenClaw больше не регистрирует встроенный бэкенд `codex-cli`.

Встроенный Plugin Anthropic регистрирует для `claude-cli`:

| `output`              | `jsonl`                                                                                                                                                                                                       |
| `input`               | `stdin`                                                                                                                                                                                                       |

| `sessionArg`          | `--session-id`                                                                                                                                                                                                |

| `imageArg`            | `@`                                                                                                                                                                                                           |

| `systemPromptFileArg` | `--append-system-prompt-file`                                                                                                                                                                                 |
| `systemPromptMode`    | `append`                                                                                                                                                                                                      |

Встроенный Plugin Google регистрируется для `google-gemini-cli`:

| Ключ                      | Значение                                                                               |
| ------------------------- | -------------------------------------------------------------------------------------- |
| `command`                 | `gemini`                                                                               |
| `args`                    | `--skip-trust --approval-mode auto_edit --output-format stream-json --prompt {prompt}` |
| `resumeArgs`              | то же самое, но с `--resume {sessionId}`                                                |

| `imageArg`                | `@`                                                                                    |
| `imagePathScope`          | `workspace`                                                                            |
| `modelArg`                | `--model`                                                                              |
| `sessionMode`             | `existing`                                                                             |
| `sessionIdFields`         | `["session_id", "sessionId"]`                                                          |

Предварительное условие: локальный Gemini CLI должен быть установлен и доступен в `PATH` как `gemini` (`brew install gemini-cli` или `npm install -g @google/gemini-cli`).

Примечания о выводе Gemini CLI:

- Стандартный анализатор `stream-json` считывает события `message` ассистента, события инструментов, сведения об использовании из итогового `result` и события критических ошибок Gemini.
- Если переопределить аргументы Gemini на `--output-format json`, OpenClaw нормализует этот бэкенд обратно к `output: "json"` и считывает текст ответа из поля JSON `response`.
- Если `usage` отсутствует или пусто, используются данные из `stats`; `stats.cached` нормализуется в OpenClaw `cacheRead`, а при отсутствии `stats.input` количество входных токенов вычисляется как `stats.input_tokens - stats.cached`.

Переопределяйте значения по умолчанию только при необходимости (чаще всего — для указания абсолютного пути в `command`).

## Наложения преобразований текста

Plugins, которым требуются небольшие слои совместимости для запросов или сообщений, могут объявлять двунаправленные преобразования текста без замены провайдера или бэкенда CLI:

```typescript
api.registerTextTransforms({
  input: [{ from: /red basket/g, to: "blue basket" }],
  output: [{ from: /blue basket/g, to: "red basket" }],
});
```

`input` преобразует системный и пользовательский запросы, передаваемые CLI. `output` преобразует потоковый текст ассистента и разобранный итоговый текст до того, как OpenClaw обработает собственные управляющие маркеры и доставку в канал; для вызовов моделей через провайдера оно также восстанавливает строковые значения внутри структурированных аргументов вызова инструментов после восстановления потока и до выполнения инструмента. Необработанные фрагменты JSON провайдера остаются неизменными; потребителям следует использовать структурированную частичную, завершающую или итоговую полезную нагрузку.

Для CLI, которые выдают специфичные для провайдера события JSONL, задайте `jsonlDialect` в конфигурации соответствующего бэкенда: `claude-stream-json` для потоков, совместимых с Claude Code, и `gemini-stream-json` для событий Gemini CLI `stream-json`.

## Владение встроенной Compaction

Некоторые бэкенды CLI запускают агента, который самостоятельно выполняет Compaction своей истории, поэтому OpenClaw не должен применять к ним защитное суммирование — иначе оно конфликтует с собственной Compaction бэкенда и может привести к критическому сбою хода.

У `claude-cli` нет конечной точки управляющей среды (Claude Code выполняет Compaction внутренне), поэтому он объявляет `ownsNativeCompaction: true`, и путь Compaction в OpenClaw возвращает запись сеанса без изменений. Сеансы с нативной управляющей средой, такие как Codex, вместо этого продолжают направляться к конечной точке Compaction своей управляющей среды.

```typescript
api.registerCliBackend({ id: "my-cli", ownsNativeCompaction: true /* ... */ });
```

Объявляйте `ownsNativeCompaction` только для бэкенда, который действительно управляет Compaction: он должен надёжно ограничивать собственную историю вблизи окна контекста и сохранять возобновляемый сеанс (например, с помощью `--resume` / `--session-id`), иначе отложенный сеанс может остаться за пределами бюджета.

## Наложения пакетного MCP

Бэкенды CLI не получают вызовы инструментов OpenClaw напрямую, но бэкенд может включить создаваемое наложение конфигурации MCP с помощью `bundleMcp: true`. Текущее встроенное поведение:

- `claude-cli`: создаваемый строгий файл конфигурации MCP.
- `google-gemini-cli`: создаваемый файл системных настроек Gemini.

Когда пакетный MCP включён, OpenClaw:

- запускает HTTP-сервер MCP на local loopback, который предоставляет процессу CLI инструменты Gateway и проходит аутентификацию посредством контекстного разрешения для отдельного запуска (`OPENCLAW_MCP_TOKEN`), действующего только в рамках текущей попытки выполнения;
- привязывает доступ к инструментам к выбранным Gateway контекстам сеанса, учётной записи и канала вместо доверия заголовкам дочернего процесса;
- загружает включённые серверы пакетного MCP для текущего рабочего пространства и объединяет их с существующей конфигурацией или структурой настроек MCP бэкенда;
- изменяет конфигурацию запуска с использованием режима интеграции, принадлежащего Plugin-владельцу бэкенда.

Если ни один сервер MCP не включён, OpenClaw всё равно внедряет строгую конфигурацию, когда бэкенд использует пакетный MCP, чтобы фоновые запуски оставались изолированными.

Привязанные к сеансу встроенные среды выполнения MCP кэшируются для повторного использования в рамках сеанса, а затем удаляются после `mcp.sessionIdleTtlMs` миллисекунд бездействия (по умолчанию 10 минут; задайте `0`, чтобы отключить). Одноразовые встроенные запуски, такие как проверки аутентификации, создание коротких идентификаторов и извлечение из Active Memory, запрашивают очистку по завершении запуска, чтобы дочерние процессы stdio и потоки Streamable HTTP/SSE не продолжали работу после его окончания.

## Ограничение истории при повторном заполнении

Когда новый сеанс CLI заполняется из предыдущей истории OpenClaw (например, после повторной попытки из-за `session_expired`), размер сформированного блока `<conversation_history>` ограничивается, чтобы запросы повторного заполнения не разрастались чрезмерно. Значение по умолчанию — 12 288 символов (около 3 000 токенов).

Для бэкендов Claude CLI это ограничение масштабируется в соответствии с определённым размером окна контекста Claude: при больших окнах контекста используется больший фрагмент предыдущей истории вплоть до фиксированного максимума; другие бэкенды CLI сохраняют консервативное значение по умолчанию. Это ограничение распространяется только на блок предыдущей истории в запросе повторного заполнения — ограничения вывода активного сеанса настраиваются отдельно в `reliability.outputLimits` (см. [Сеансы](#sessions)).

## Ограничения

- Нет прямых вызовов инструментов OpenClaw: OpenClaw не внедряет вызовы инструментов в протокол бэкенда CLI. Бэкенды видят инструменты Gateway только при включении `bundleMcp: true`.
- Потоковая передача зависит от бэкенда: некоторые бэкенды передают JSONL потоком, а другие буферизуют данные до завершения процесса.
- Структурированный вывод зависит от собственного формата JSON CLI.

## Устранение неполадок

| Симптом                     | Решение                                                                                  |
| --------------------------- | ---------------------------------------------------------------------------------------- |
| CLI не найден               | Задайте в `command` полный путь.                                                         |
| Неверное имя модели         | Используйте `modelAliases`, чтобы сопоставить `provider/model` с идентификатором модели CLI. |
| Нет непрерывности сеанса    | Убедитесь, что задан `sessionArg`, а значение `sessionMode` отлично от `none`.            |
| Изображения игнорируются    | Задайте `imageArg` и убедитесь, что CLI поддерживает пути к файлам.                       |
