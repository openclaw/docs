OpenClaw имеет три связанных, но разных механизма управления:

1. **Песочница** (`agents.defaults.sandbox.*` / `agents.list[].sandbox.*`) определяет, **где выполняются инструменты** (в бэкенде песочницы или на хосте).
2. **Политика инструментов** (`tools.*`, `tools.sandbox.tools.*`, `agents.list[].tools.*`) определяет, **какие инструменты доступны и разрешены**.
3. **Повышенный режим** (`tools.elevated.*`, `agents.list[].tools.elevated.*`) — это **аварийный механизм только для `exec`**, позволяющий выполнять команды вне песочницы, когда сеанс изолирован (`gateway` по умолчанию или `node`, если целью выполнения `exec` настроен `node`).

## Быстрая диагностика

Используйте инспектор, чтобы увидеть, что OpenClaw делает _на самом деле_:

```bash
openclaw sandbox explain
openclaw sandbox explain --session agent:main:main
openclaw sandbox explain --agent work
openclaw sandbox explain --json
```

Он выводит:

- фактический режим, область действия и доступ песочницы к рабочей области;
- находится ли текущий сеанс в песочнице (основной или неосновной);
- фактические разрешения и запреты инструментов песочницы, а также источник настройки: агент, глобальная конфигурация или значение по умолчанию;
- условия доступа к повышенному режиму и пути ключей для исправления конфигурации.

## Песочница: где выполняются инструменты

Работа песочницы определяется параметром `agents.defaults.sandbox.mode`:

- `"off"`: всё выполняется на хосте.
- `"non-main"`: в песочнице выполняются только неосновные сеансы (частая причина «неожиданностей» для групп и каналов).
- `"all"`: всё выполняется в песочнице.

Параметр `agents.defaults.sandbox.workspaceAccess` определяет, что доступно песочнице: `"none"`, `"ro"` или `"rw"`.

Полную таблицу параметров (область действия, монтирование рабочей области, образы) см. в разделе [«Песочница»](/ru/gateway/sandboxing).

### Привязанные монтирования: быстрая проверка безопасности

- `docker.binds` _нарушает изоляцию_ файловой системы песочницы: всё смонтированное доступно внутри контейнера в заданном режиме (`:ro` или `:rw`).
- Если режим не указан, по умолчанию используется чтение и запись; для исходного кода и секретов предпочитайте `:ro`.
- При `scope: "shared"` привязанные монтирования отдельных агентов игнорируются — применяются только глобальные.
- OpenClaw дважды проверяет исходные пути привязанных монтирований: сначала нормализованный исходный путь, затем путь после разрешения через самого глубокого существующего предка. Выход через родительскую символьную ссылку не позволяет обойти проверки запрещённых путей или разрешённых корневых каталогов.
- Несуществующие конечные пути также проверяются безопасно. Если `/workspace/alias-out/new-file` через родительскую символьную ссылку разрешается в запрещённый путь или за пределы настроенных разрешённых корневых каталогов, монтирование отклоняется.
- Привязка `/var/run/docker.sock` фактически предоставляет песочнице управление хостом; делайте это только намеренно.
- Доступ к рабочей области (`workspaceAccess`) не зависит от режимов привязанных монтирований.

## Политика инструментов: какие инструменты существуют и могут вызываться

Важны следующие уровни:

- **Профиль инструментов**: `tools.profile` и `agents.list[].tools.profile` (базовый список разрешений)
- **Профиль инструментов провайдера**: `tools.byProvider[provider].profile` и `agents.list[].tools.byProvider[provider].profile`
- **Глобальная и агентская политика инструментов**: `tools.allow`/`tools.deny` и `agents.list[].tools.allow`/`agents.list[].tools.deny`
- **Политика инструментов провайдера**: `tools.byProvider[provider].allow/deny` и `agents.list[].tools.byProvider[provider].allow/deny`
- **Политика инструментов песочницы** (применяется только в песочнице): `tools.sandbox.tools.allow`/`tools.sandbox.tools.deny` и `agents.list[].tools.sandbox.tools.*`

Основные правила:

- `deny` всегда имеет приоритет.
- Если `allow` не пуст, всё остальное считается заблокированным.
- Политика инструментов — окончательное ограничение: `/exec` не может обойти запрет инструмента `exec`.
- Политика инструментов фильтрует доступность инструментов по имени, но не анализирует побочные эффекты внутри `exec`. Если `exec` разрешён, запрет `write`, `edit` или `apply_patch` не делает команды оболочки доступными только для чтения.
- `/exec` изменяет только настройки сеанса по умолчанию для авторизованных отправителей; он не предоставляет доступ к инструментам.
- Ключи инструментов провайдера принимают либо `provider` (например, `google-antigravity`), либо `provider/model` (например, `openai/gpt-5.4`).
- Журналы Gateway содержат записи аудита `agents/tool-policy`, когда этап политики инструментов удаляет инструменты или политика песочницы блокирует вызов. Используйте `openclaw logs`, чтобы увидеть метку правила, ключ конфигурации и имена затронутых инструментов.

### Группы инструментов (сокращения)

Политики инструментов (глобальные, агентские и песочницы) поддерживают записи `group:*`, которые разворачиваются в несколько инструментов:

```json5
{
  tools: {
    sandbox: {
      tools: {
        allow: ["group:runtime", "group:fs", "group:sessions", "group:memory"],
      },
    },
  },
}
```

Доступные группы:

| Группа             | Инструменты                                                                                                                                                      |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `group:runtime`    | `exec`, `process`, `code_execution` (`bash` принимается как псевдоним для `exec`)                                                                            |
| `group:fs`         | `read`, `write`, `edit`, `apply_patch`                                                                                                                     |
| `group:sessions`   | `sessions_list`, `sessions_history`, `sessions_send`, `sessions_spawn`, `sessions_yield`, `subagents`, `session_status`                                    |
| `group:memory`     | `memory_search`, `memory_get`                                                                                                                              |
| `group:web`        | `web_search`, `x_search`, `web_fetch`                                                                                                                      |
| `group:ui`         | `browser`, `canvas`                                                                                                                                        |
| `group:automation` | `heartbeat_respond`, `cron`, `gateway`                                                                                                                     |
| `group:messaging`  | `message`                                                                                                                                                  |
| `group:nodes`      | `nodes`, `computer`                                                                                                                                        |
| `group:agents`     | `agents_list`, `get_goal`, `create_goal`, `update_goal`, `update_plan`, `skill_workshop`                                                                   |
| `group:media`      | `image`, `image_generate`, `music_generate`, `video_generate`, `tts`                                                                                       |
| `group:openclaw`   | большинство встроенных инструментов OpenClaw (кроме примитивов файловой системы и среды выполнения `read`/`write`/`edit`/`apply_patch`/`exec`/`process`, а также `canvas` и плагинов провайдеров) |
| `group:plugins`    | все загруженные инструменты, принадлежащие плагинам, включая настроенные серверы MCP, предоставляемые через `bundle-mcp`                                                               |

Для агентов с доступом только для чтения запрещайте `group:runtime` вместе с инструментами, изменяющими файловую систему, если режим только для чтения не обеспечивается политикой файловой системы песочницы или отдельной границей хоста.

Для серверов MCP в песочнице политика инструментов песочницы служит вторым уровнем разрешений. Если `mcp.servers` настроен, но в изолированных сеансах отображаются только встроенные инструменты, добавьте `bundle-mcp`, `group:plugins` или имя либо шаблон имени инструмента MCP с префиксом сервера, например `outlook__send_mail` или `outlook__*`, в `tools.sandbox.tools.alsoAllow`, затем перезапустите или перезагрузите Gateway и повторно получите список инструментов. В шаблонах серверов используется безопасный для провайдера префикс сервера MCP: символы, не входящие в `[A-Za-z0-9_-]`, заменяются на `-`; имена, не начинающиеся с буквы, получают префикс `mcp-`; длинные или повторяющиеся префиксы могут быть усечены или дополнены суффиксом.

Сейчас `openclaw doctor` проверяет эту конфигурацию для серверов в `mcp.servers`, управляемых OpenClaw. Серверы MCP, загруженные из манифестов встроенных плагинов или файла Claude `.mcp.json`, используют тот же уровень разрешений песочницы, но эта диагностика пока не перечисляет такие источники; если их инструменты исчезают в изолированных сеансах, используйте те же записи списка разрешений.

## Повышенный режим: выполнение `exec` на хосте

Повышенный режим **не** предоставляет дополнительные инструменты; он влияет только на `exec`.

- Если сеанс находится в песочнице, `/elevated on` или `exec` с `elevated: true` выполняется вне песочницы, хотя подтверждение всё ещё может требоваться.
- Используйте `/elevated full`, чтобы отключить подтверждения `exec` для сеанса.
- Если выполнение уже происходит напрямую, повышенный режим фактически ничего не меняет, но условия доступа всё равно применяются.
- Повышенный режим **не** ограничен областью Skills и **не** переопределяет разрешения и запреты инструментов.
- Повышенный режим не предоставляет произвольного переопределения целевого хоста при `host=auto`; он следует обычным правилам выбора цели `exec` и сохраняет `node` только тогда, когда настроенной или сеансовой целью уже является `node`.
- `/exec` не связан с повышенным режимом. Он лишь изменяет настройки `exec` для текущего сеанса у авторизованных отправителей.

Условия доступа:

- Включение: `tools.elevated.enabled` и, при необходимости, `agents.list[].tools.elevated.enabled`
- Списки разрешённых отправителей: `tools.elevated.allowFrom.<provider>` и, при необходимости, `agents.list[].tools.elevated.allowFrom.<provider>`

См. раздел [«Повышенный режим»](/ru/tools/elevated).

## Распространённые способы выхода из «тюрьмы песочницы»

### «Инструмент X заблокирован политикой инструментов песочницы»

Ключи для исправления — выберите один вариант:

- Отключите песочницу: `agents.defaults.sandbox.mode=off` или для отдельного агента `agents.list[].sandbox.mode=off`.
- Разрешите инструмент внутри песочницы:
  - удалите его из `tools.sandbox.tools.deny` или из агентского `agents.list[].tools.sandbox.tools.deny`;
  - либо добавьте его в `tools.sandbox.tools.allow` или в агентский список разрешений.
- Найдите запись `agents/tool-policy` в выводе `openclaw logs`. В ней указаны режим песочницы и правило `allow` или `deny`, заблокировавшее инструмент.

### «Я думал, что это основной сеанс. Почему он находится в песочнице?»

В режиме `"non-main"` ключи групп и каналов _не_ считаются основными. Используйте ключ основного сеанса, показанный командой `sandbox explain`, или переключите режим на `"off"`.
