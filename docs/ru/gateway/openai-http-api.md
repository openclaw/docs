Gateway может предоставлять небольшой OpenAI-совместимый интерфейс Chat Completions. По умолчанию он **отключён**.

После включения все перечисленные ниже конечные точки становятся доступны на том же порту, что и Gateway (мультиплексирование WS + HTTP):

| Метод | Путь                   |
| ----- | ---------------------- |
| POST  | `/v1/chat/completions` |
| GET   | `/v1/models`           |
| GET   | `/v1/models/{id}`      |
| POST  | `/v1/embeddings`       |
| POST  | `/v1/responses`        |

Запросы выполняются как обычный запуск агента Gateway (по тому же пути выполнения кода, что и `openclaw agent`), поэтому маршрутизация, разрешения и конфигурация соответствуют вашему Gateway.

## Включение конечной точки

```json5
{
  gateway: {
    http: {
      endpoints: {
        chatCompletions: { enabled: true },
      },
    },
  },
}
```

Чтобы отключить её, задайте `enabled: false` (или не указывайте этот параметр).

## Граница безопасности (важно)

Считайте эту конечную точку предоставляющей **полный операторский доступ** к экземпляру Gateway:

- Действительный токен или пароль Gateway для этой конечной точки эквивалентен учётным данным владельца или оператора, а не узкой области доступа отдельного пользователя.
- Запросы проходят по тому же пути агента плоскости управления, что и доверенные действия оператора, поэтому, если политика целевого агента разрешает инструменты для работы с конфиденциальными данными, эта конечная точка может их использовать.
- Разрешайте доступ только через local loopback, tailnet или частную точку входа. Не публикуйте её в интернете.

Матрица аутентификации:

| Способ аутентификации                                                                                | Поведение                                                                                                                                                                                                                                                                                                                                |
| ---------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `gateway.auth.mode="token"` или `"password"` + `Authorization: Bearer ...`                            | Подтверждает владение общим секретом Gateway. Игнорирует заголовок `x-openclaw-scopes` и восстанавливает полный набор операторских областей доступа по умолчанию: `operator.admin`, `operator.approvals`, `operator.pairing`, `operator.read`, `operator.talk.secrets`, `operator.write`. Обрабатывает реплики чата как реплики владельца-отправителя. |
| Доверенный HTTP с удостоверением личности (аутентификация через доверенный прокси или `gateway.auth.mode="none"` на частной точке входа) | Учитывает `x-openclaw-scopes`, если он присутствует; при его отсутствии использует набор операторских областей доступа по умолчанию. Семантика владельца утрачивается только тогда, когда вызывающая сторона явно сужает области доступа и исключает `operator.admin`. Для элементов управления уровня владельца, таких как `x-openclaw-model`, требуется `operator.admin`. |

См. [Операторские области доступа](/ru/gateway/operator-scopes), [Безопасность](/ru/gateway/security) и [Удалённый доступ](/ru/gateway/remote).

## Аутентификация

Используется конфигурация аутентификации Gateway (подробности этого режима см. в разделе [Аутентификация через доверенный прокси](/ru/gateway/trusted-proxy-auth)):

| Режим                              | Способ аутентификации                                                                                                                                                                                  |
| ---------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `gateway.auth.mode="token"`         | `Authorization: Bearer <token>`. Задаётся через `gateway.auth.token` или `OPENCLAW_GATEWAY_TOKEN`.                                                                                                      |
| `gateway.auth.mode="password"`      | `Authorization: Bearer <password>`. Задаётся через `gateway.auth.password` или `OPENCLAW_GATEWAY_PASSWORD`.                                                                                             |
| `gateway.auth.mode="trusted-proxy"` | Направьте запрос через настроенный прокси, учитывающий удостоверение личности; он добавит необходимые заголовки идентификации. Для прокси local loopback на том же хосте требуется явно задать `gateway.auth.trustedProxy.allowLoopback = true`. |
| `gateway.auth.mode="none"`          | Заголовок аутентификации не требуется (только для частной точки входа).                                                                                                                                |

Примечания:

- Вызывающие стороны на том же хосте, которые обходят прокси Gateway в режиме `trusted-proxy`, могут напрямую использовать резервную аутентификацию через `gateway.auth.password` / `OPENCLAW_GATEWAY_PASSWORD`. При наличии любых признаков в заголовках `Forwarded`, `X-Forwarded-*` или `X-Real-IP` запрос вместо этого продолжает обрабатываться по пути доверенного прокси.
- Если настроен `gateway.auth.rateLimit` и происходит слишком много неудачных попыток аутентификации, конечная точка возвращает код `429` с заголовком `Retry-After`.

## Когда использовать эту конечную точку

- Предпочитайте её добавлению нового встроенного канала, если ваша интеграция представляет собой ещё один операторский или клиентский интерфейс к тому же Gateway.
- Для нативных мобильных клиентов, подключающихся непосредственно к удалённому Gateway, предпочитайте [WebChat](/ru/web/webchat) или [протокол Gateway](/ru/gateway/protocol) с процессом начальной настройки сопряжённого устройства и токеном устройства, чтобы устройству не требовался общий HTTP-токен или пароль.
- Если вы интегрируете внешнюю сеть обмена сообщениями с собственными пользователями, комнатами, доставкой через Webhook или исходящим транспортом, вместо этого создайте Plugin канала. См. [Создание плагинов](/ru/plugins/building-plugins).

## Контракт модели, ориентированный на агента

OpenClaw рассматривает поле OpenAI `model` как **целевого агента**, а не как необработанный идентификатор модели провайдера.

| Значение `model`                              | Маршрутизируется к                                                                                                                    |
| --------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| `openclaw`                                    | Настроенному агенту по умолчанию                                                                                                      |
| `openclaw/default`                            | Настроенному агенту по умолчанию (стабильный псевдоним; его можно безопасно жёстко задать, даже если фактический идентификатор агента по умолчанию различается между средами) |
| `openclaw/<agentId>` или `openclaw:<agentId>` | Конкретному агенту                                                                                                                    |
| `agent:<agentId>`                             | Конкретному агенту (псевдоним для совместимости)                                                                                      |

Необязательные заголовки запроса:

| Заголовок                                       | Эффект                                                                                                                                                                                                                                                                                                                              |
| ----------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `x-openclaw-model: <provider/model-or-bare-id>` | Переопределяет серверную модель для выбранного агента. Вызывающие стороны с общим секретом в bearer-токене могут использовать его напрямую; вызывающим сторонам с удостоверением личности (`trusted-proxy` или частная точка входа без аутентификации с `x-openclaw-scopes`) требуется `operator.admin`, иначе возвращается `403 missing scope: operator.admin`. |
| `x-openclaw-agent-id: <agentId>`                | Переопределение выбора агента для совместимости.                                                                                                                                                                                                                                                                                     |
| `x-openclaw-session-key: <sessionKey>`          | Явная маршрутизация сеанса. Отклоняется с ошибкой `400 invalid_request_error`, если используется зарезервированное внутреннее пространство имён (`subagent:`, `cron:`, `acp:`).                                                                                                                                                        |
| `x-openclaw-message-channel: <channel>`         | Задаёт синтетический контекст входящего канала для учитывающих канал запросов и политик.                                                                                                                                                                                                                                             |

`/v1/models` перечисляет цели агентов верхнего уровня (`openclaw`, `openclaw/default`, `openclaw/<agentId>`), а не серверные модели провайдеров и не субагентов; субагенты остаются частью внутренней топологии выполнения. Если не указывать `x-openclaw-model`, выбранный агент будет работать со своей обычной настроенной моделью.

`/v1/embeddings` использует те же идентификаторы `model`, обозначающие целевых агентов. Передайте `x-openclaw-model` (от вызывающей стороны с общим секретом или с удостоверением личности и областью доступа `operator.admin`), чтобы выбрать конкретную модель эмбеддингов; в противном случае запрос использует обычную конфигурацию эмбеддингов выбранного агента.

## Поведение сеансов

По умолчанию конечная точка **не сохраняет состояние между запросами** (при каждом вызове создаётся новый ключ сеанса).

Если запрос содержит строку OpenAI `user`, Gateway создаёт из неё стабильный ключ сеанса, благодаря чему повторные вызовы могут использовать общий сеанс агента. В пользовательских приложениях повторно используйте одно и то же значение `user` для каждой ветви диалога; избегайте идентификаторов уровня учётной записи, если не хотите, чтобы несколько диалогов или устройств совместно использовали один сеанс OpenClaw. Используйте `x-openclaw-session-key` только тогда, когда требуется явное управление маршрутизацией между несколькими клиентами или ветвями, и применяйте принадлежащие приложению ключи, которые не используют перечисленные выше зарезервированные пространства имён.

## Ограничения запросов (конфигурация)

Значения по умолчанию можно настроить в `gateway.http.endpoints.chatCompletions`:

```json5
{
  gateway: {
    http: {
      endpoints: {
        chatCompletions: {
          enabled: true,
          maxBodyBytes: 20000000,
          maxImageParts: 8,
          maxTotalImageBytes: 20000000,
          images: {
            allowUrl: false,
            urlAllowlist: ["cdn.example.com", "*.assets.example.com"],
            allowedMimes: [
              "image/jpeg",
              "image/png",
              "image/gif",
              "image/webp",
              "image/heic",
              "image/heif",
            ],
            maxBytes: 10485760,
            maxRedirects: 3,
            timeoutMs: 10000,
          },
        },
      },
    },
  },
}
```

Значения по умолчанию, если параметры не указаны:

| Ключ                  | Значение по умолчанию                                                                                          |
| --------------------- | -------------------------------------------------------------------------------------------------------------- |
| `maxBodyBytes`        | 20 МБ                                                                                                          |
| `maxImageParts`       | 8 (максимальное количество частей `image_url`, считываемых из последнего сообщения пользователя)               |
| `maxTotalImageBytes`  | 20 МБ (суммарное количество декодированных байтов во всех частях `image_url` одного запроса)                    |
| `images.allowUrl`     | `false` (части `image_url`, загружаемые по URL, отклоняются, если параметр не включён)                          |
| `images.maxBytes`     | 10 МБ на изображение                                                                                           |
| `images.maxRedirects` | 3                                                                                                              |
| `images.timeoutMs`    | 10 с                                                                                                           |

Источники `image_url` в форматах HEIC/HEIF принимаются и перед отправкой провайдеру преобразуются в JPEG общим обработчиком изображений OpenClaw (Rastermill), который для форматов, требующих поддержки внешнего кодека, использует резервный системный конвертер (`sips`, ImageMagick, GraphicsMagick или ffmpeg).

Примечание по безопасности: добавление имени хоста в список разрешённых не отменяет блокировку частных/внутренних IP-адресов. Для шлюзов, доступных из интернета, применяйте средства контроля исходящего сетевого трафика в дополнение к защитным механизмам на уровне приложения. См. раздел [Безопасность](/ru/gateway/security).

## Контракт инструментов чата

`/v1/chat/completions` поддерживает подмножество функциональных инструментов, совместимое с распространёнными клиентами OpenAI Chat.

### Поддерживаемые поля запроса

| Поле                       | Примечания                                                                                                                                               |
| -------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `tools`                    | Массив объектов `{ "type": "function", "function": { ... } }`                                                                                            |
| `tool_choice`              | `"auto"`, `"none"`, `"required"` или `{ "type": "function", "function": { "name": "..." } }`                                                              |
| `messages[*].role: "tool"` | Последующие ходы                                                                                                                                        |
| `messages[*].tool_call_id` | Связывает результат инструмента с предшествующим вызовом инструмента                                                                                     |
| `max_completion_tokens`    | Число; ограничение общего количества токенов завершения на вызов, включая токены рассуждения. Текущее имя поля; используется, если переданы оно и `max_tokens`. |
| `max_tokens`               | Число; устаревший псевдоним, игнорируется, если также присутствует `max_completion_tokens`.                                                              |
| `temperature`              | Число от 0 до 2; передаётся вышестоящему провайдеру по мере возможности. При выходе за диапазон возвращается `400 invalid_request_error`.                 |
| `top_p`                    | Число от 0 до 1; применяется по мере возможности. При выходе за диапазон возвращается `400 invalid_request_error`.                                       |
| `frequency_penalty`        | Число от -2.0 до 2.0; применяется по мере возможности. При выходе за диапазон возвращается `400 invalid_request_error`.                                   |
| `presence_penalty`         | Число от -2.0 до 2.0; применяется по мере возможности. При выходе за диапазон возвращается `400 invalid_request_error`.                                   |
| `seed`                     | Целое число; применяется по мере возможности. Для нецелых значений возвращается `400 invalid_request_error`.                                             |
| `stop`                     | Строка или массив не более чем из 4 строк; применяется по мере возможности. При более чем 4 последовательностях, а также строковых значениях неверного типа или пустых строках возвращается `400 invalid_request_error`. |

Все поля выборки и ограничения токенов передаются через один канал параметров потока агента и перенаправляются по мере возможности:

- Ограничение токенов: имя поля в протоколе выбирается транспортом провайдера: `max_completion_tokens` для конечных точек семейства OpenAI, `max_tokens` для провайдеров, принимающих только устаревшее имя (Mistral, Chutes).
- `stop` сопоставляется с полем остановки транспорта: `stop` для серверных реализаций Chat Completions, `stop_sequences` для Anthropic. В OpenAI Responses API параметр остановки отсутствует, поэтому `stop` не применяется к моделям на базе Responses.
- Серверная реализация Codex Responses на основе ChatGPT использует фиксированные серверные параметры выборки и удаляет `temperature`/`top_p` (вместе с `max_output_tokens`, `metadata`, `prompt_cache_retention`, `service_tier`) до поступления запроса в эту серверную реализацию.

### Неподдерживаемые варианты

Возвращается `400 invalid_request_error` в следующих случаях:

- `tools` не является массивом, содержит элементы, не являющиеся функциональными инструментами, либо отсутствует `tool.function.name`
- варианты `tool_choice`, такие как `allowed_tools` и `custom`
- значения `tool_choice.function.name`, не соответствующие ни одному предоставленному инструменту

Для `tool_choice: "required"` и `tool_choice` с явно закреплённой функцией конечная точка сужает доступный набор клиентских функциональных инструментов, предписывает среде выполнения вызвать клиентский инструмент перед ответом и возвращает ошибку, если ответ агента не содержит соответствующего структурированного вызова клиентского инструмента. Это относится к переданному вызывающей стороной HTTP-списку `tools`, а не ко всем внутренним инструментам агента OpenClaw.

### Формат непотокового ответа инструмента

Когда агент вызывает инструменты, ответ содержит:

- `choices[0].finish_reason = "tool_calls"`
- элементы `choices[0].message.tool_calls[]` с `id`, `type: "function"`, `function.name`, `function.arguments` (строка JSON)
- комментарий ассистента перед вызовом инструмента в `choices[0].message.content` (возможно, пустой)

### Формат потокового ответа инструмента

При `stream: true` вызовы инструментов поступают в виде последовательных фрагментов SSE: начального изменения роли ассистента, необязательных изменений комментария ассистента, одного или нескольких фрагментов `delta.tool_calls` с идентификатором инструмента и частями аргументов, а затем заключительного фрагмента с `finish_reason: "tool_calls"` и `data: [DONE]`.

Если задано `stream_options.include_usage=true`, перед `[DONE]` отправляется заключительный фрагмент со статистикой использования.

### Цикл продолжения после вызова инструмента

Получив `tool_calls`, выполните запрошенные функции и отправьте последующий запрос, включающий предыдущее сообщение ассистента с вызовами инструментов, а также одно или несколько сообщений с `role: "tool"` и соответствующими значениями `tool_call_id`. Это продолжает тот же цикл рассуждений агента для формирования окончательного ответа.

## Потоковая передача (SSE)

Установите `stream: true`, чтобы получать события Server-Sent Events:

- `Content-Type: text/event-stream`
- Каждая строка события имеет вид `data: <json>`
- Поток завершается строкой `data: [DONE]`

## Быстрая настройка Open WebUI

- Базовый URL: `http://127.0.0.1:18789/v1`
- Базовый URL Docker в macOS: `http://host.docker.internal:18789/v1`
- Ключ API: ваш токен-носитель Gateway
- Модель: `openclaw/default`

Ожидаемое поведение: `GET /v1/models` выводит `openclaw/default`, а Open WebUI использует его как идентификатор модели чата. Чтобы указать конкретного серверного провайдера или модель, задайте обычную модель агента по умолчанию либо отправьте `x-openclaw-model` (вызывающая сторона с общим секретом или вызывающая сторона с подтверждённой идентичностью и разрешением `operator.admin`).

Быстрая проверка работоспособности:

```bash
curl -sS http://127.0.0.1:18789/v1/models \
  -H 'Authorization: Bearer YOUR_TOKEN'
```

Если ответ содержит `openclaw/default`, большинство конфигураций Open WebUI смогут подключиться с теми же базовым URL и токеном.

## Примеры

Стабильный сеанс для одной беседы приложения:

```bash
curl -sS http://127.0.0.1:18789/v1/chat/completions \
  -H 'Authorization: Bearer YOUR_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{
    "model": "openclaw/default",
    "user": "conv:YOUR_CONVERSATION_ID",
    "messages": [{"role":"user","content":"Summarize my tasks for today"}]
  }'
```

Повторно используйте то же значение `user` в последующих вызовах для этой беседы, чтобы продолжить тот же сеанс агента.

Без потоковой передачи:

```bash
curl -sS http://127.0.0.1:18789/v1/chat/completions \
  -H 'Authorization: Bearer YOUR_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{
    "model": "openclaw/default",
    "messages": [{"role":"user","content":"hi"}]
  }'
```

С потоковой передачей:

```bash
curl -N http://127.0.0.1:18789/v1/chat/completions \
  -H 'Authorization: Bearer YOUR_TOKEN' \
  -H 'Content-Type: application/json' \
  -H 'x-openclaw-model: openai/gpt-5.4' \
  -d '{
    "model": "openclaw/research",
    "stream": true,
    "messages": [{"role":"user","content":"hi"}]
  }'
```

Получить список моделей:

```bash
curl -sS http://127.0.0.1:18789/v1/models \
  -H 'Authorization: Bearer YOUR_TOKEN'
```

Получить одну модель:

```bash
curl -sS http://127.0.0.1:18789/v1/models/openclaw%2Fdefault \
  -H 'Authorization: Bearer YOUR_TOKEN'
```

Создать эмбеддинги:

```bash
curl -sS http://127.0.0.1:18789/v1/embeddings \
  -H 'Authorization: Bearer YOUR_TOKEN' \
  -H 'Content-Type: application/json' \
  -H 'x-openclaw-model: openai/text-embedding-3-small' \
  -d '{
    "model": "openclaw/default",
    "input": ["alpha", "beta"]
  }'
```

`/v1/embeddings` поддерживает `input` в виде строки или массива строк.
