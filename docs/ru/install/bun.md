<Warning>
Bun не может запускать CLI OpenClaw или Gateway, поскольку не предоставляет необходимый API `node:sqlite`. Установите поддерживаемую версию Node для выполнения всех команд среды выполнения OpenClaw.
</Warning>

Bun по-прежнему можно использовать как необязательный установщик зависимостей и средство запуска скриптов пакетов. Менеджером пакетов по умолчанию остаётся `pnpm`, который полностью поддерживается и используется инструментами документации. Bun не может использовать `pnpm-lock.yaml` и игнорирует его.

## Установка

<Steps>
  <Step title="Установите зависимости">
    ```sh
    bun install
    ```

    `bun.lock` / `bun.lockb` добавлены в gitignore, поэтому лишних изменений в репозитории не возникает. Чтобы полностью отключить запись файла блокировки:

    ```sh
    bun install --no-save
    ```

  </Step>
  <Step title="Соберите и протестируйте">
    ```sh
    bun run build
    bun run vitest run
    ```

    Команды, запускающие сам OpenClaw, по-прежнему необходимо выполнять через Node.

  </Step>
</Steps>

## Скрипты жизненного цикла

Bun блокирует скрипты жизненного цикла зависимостей, если они явно не признаны доверенными. Для этого репозитория обычно блокируемые скрипты не требуются:

- `baileys` `preinstall`: проверяет, что основная версия Node >= 20 (для OpenClaw требуется Node 22.22.3+, 24.15+ или 25.9+; рекомендуется Node 24)
- `protobufjs` `postinstall`: выводит предупреждения о несовместимых схемах версий (без артефактов сборки)

Если возникнет проблема во время выполнения, для устранения которой нужны эти скрипты, явно добавьте их в доверенные:

```sh
bun pm trust baileys protobufjs
```

## Ограничения

В некоторых скриптах пакетов внутри жёстко задан `pnpm` (например, `check:docs`, `ui:*`, `protocol:check`). При их запуске через `bun run` всё равно выполняется `pnpm`, поэтому запускайте их напрямую через `pnpm`.
