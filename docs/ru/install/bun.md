<Warning>
Bun не рекомендуется использовать в качестве среды выполнения Gateway (известны проблемы с WhatsApp и Telegram). Для рабочей среды используйте Node.
</Warning>

Bun — это необязательная локальная среда выполнения для непосредственного запуска TypeScript (`bun run ...`, `bun --watch ...`). Пакетным менеджером по умолчанию остаётся `pnpm`: он полностью поддерживается и используется инструментами документации. Bun не может использовать `pnpm-lock.yaml` и игнорирует этот файл.

## Установка

<Steps>
  <Step title="Установка зависимостей">
    ```sh
    bun install
    ```

    Файлы `bun.lock` / `bun.lockb` добавлены в gitignore, поэтому лишних изменений в репозитории не возникает. Чтобы полностью отключить запись файла блокировки:

    ```sh
    bun install --no-save
    ```

  </Step>
  <Step title="Сборка и тестирование">
    ```sh
    bun run build
    bun run vitest run
    ```
  </Step>
</Steps>

## Скрипты жизненного цикла

Bun блокирует скрипты жизненного цикла зависимостей, если они явно не отмечены как доверенные. Для этого репозитория обычно блокируемые скрипты не требуются:

- `baileys` `preinstall`: проверяет, что основная версия Node не ниже 20 (для OpenClaw требуется Node 22.19+ или 23.11+; рекомендуется Node 24)
- `protobufjs` `postinstall`: выводит предупреждения о несовместимых схемах версий (артефакты сборки не создаются)

Если возникнет проблема во время выполнения, для устранения которой требуются эти скрипты, явно отметьте их как доверенные:

```sh
bun pm trust baileys protobufjs
```

## Ограничения

В некоторых пакетных скриптах внутри жёстко задан `pnpm` (например, `check:docs`, `ui:*`, `protocol:check`). При запуске через `bun run` они всё равно вызывают `pnpm` в командной оболочке, поэтому запускайте их напрямую через `pnpm`.
