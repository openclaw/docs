Встроенный провайдер миграции Hermes обнаруживает состояние в `~/.hermes`, показывает предварительный просмотр каждого изменения перед применением, скрывает секреты в планах и отчётах и создаёт проверенную резервную копию OpenClaw, прежде чем что-либо изменить.

<Note>
Для импорта требуется чистая установка OpenClaw. Если у вас уже есть локальное состояние OpenClaw, сначала сбросьте конфигурацию, учётные данные, сеансы и рабочее пространство либо используйте `openclaw migrate apply hermes` напрямую с `--overwrite` после проверки плана.
</Note>

## Два способа импорта

<Tabs>
  <Tab title="Мастер первоначальной настройки">
    Обнаруживает Hermes в `~/.hermes` и показывает предварительный просмотр перед применением.

    ```bash
    openclaw onboard --flow import
    ```

    Или укажите конкретный источник:

    ```bash
    openclaw onboard --import-from hermes --import-source ~/.hermes
    ```

  </Tab>
  <Tab title="CLI">
    Используйте `openclaw migrate` для автоматизированных или повторяемых запусков. Полное справочное руководство см. в разделе [`openclaw migrate`](/ru/cli/migrate).

    ```bash
    openclaw migrate hermes --dry-run    # только предварительный просмотр
    openclaw migrate apply hermes --yes  # применить без запроса подтверждения
    ```

    Добавьте `--from <path>`, если Hermes находится не в `~/.hermes`.

  </Tab>
</Tabs>

## Что импортируется

<AccordionGroup>
  <Accordion title="Конфигурация модели">
    - Выбор модели по умолчанию из файла Hermes `config.yaml`.
    - Настроенные провайдеры моделей и пользовательские конечные точки, совместимые с OpenAI, из `providers` и `custom_providers`.

  </Accordion>
  <Accordion title="Серверы MCP">
    Определения серверов MCP из `mcp_servers` или `mcp.servers`.
  </Accordion>
  <Accordion title="Файлы рабочего пространства">
    - `SOUL.md` и `AGENTS.md` копируются в рабочее пространство агента OpenClaw.
    - `memories/MEMORY.md` и `memories/USER.md` **добавляются** в соответствующие файлы памяти OpenClaw вместо их перезаписи.

  </Accordion>
  <Accordion title="Конфигурация памяти">
    Значения конфигурации по умолчанию для файловой памяти OpenClaw. Внешние провайдеры памяти, такие как Honcho, записываются как элементы архива или элементы для ручной проверки, чтобы вы могли перенести их осознанно.
  </Accordion>
  <Accordion title="Skills">
    Skills с файлом `SKILL.md` в каталоге `skills/<name>/` копируются вместе со значениями конфигурации каждого Skills из `skills.config`.
  </Accordion>
  <Accordion title="Учётные данные аутентификации">
    Интерактивный запуск `openclaw migrate` запрашивает подтверждение перед импортом учётных данных аутентификации, при этом по умолчанию выбран ответ «да». При согласии импортируются записи OpenCode OpenAI OAuth и GitHub Copilot из файла OpenCode `auth.json`, а также [поддерживаемые ключи `.env` Hermes](/ru/cli/migrate#supported-env-keys). Собственные записи OAuth из файла Hermes `auth.json` являются устаревшим состоянием: вместо импорта в действующую систему аутентификации они отображаются как элементы, требующие повторной аутентификации вручную или запуска doctor. Используйте `--include-secrets` для импорта учётных данных при неинтерактивном запуске, `--no-auth-credentials` для полного пропуска импорта учётных данных или флаг мастера первоначальной настройки `--import-secrets`.
  </Accordion>
</AccordionGroup>

## Что остаётся только в архиве

Провайдер копирует перечисленные ниже элементы в каталог отчёта о миграции для ручной проверки, но **не** загружает их в действующую конфигурацию или учётные данные OpenClaw:

- `plugins/`
- `sessions/`
- `logs/`
- `cron/`
- `mcp-tokens/`
- `state.db`

OpenClaw отказывается автоматически выполнять это состояние или считать его доверенным, поскольку форматы и предположения о доверии могут различаться между системами. После проверки архива перенесите нужные элементы вручную.

## Рекомендуемый порядок действий

<Steps>
  <Step title="Предварительный просмотр плана">
    ```bash
    openclaw migrate hermes --dry-run
    ```

    В плане перечислены все предстоящие изменения, включая конфликты, пропущенные и конфиденциальные элементы. Вложенные ключи, похожие на секреты, скрываются в выводе.

  </Step>
  <Step title="Применение с резервным копированием">
    ```bash
    openclaw migrate apply hermes --yes
    ```

    Перед применением OpenClaw создаёт и проверяет резервную копию. Этот неинтерактивный пример импортирует только несекретное состояние. Запустите команду без `--yes`, чтобы ответить на запрос об учётных данных в интерактивном режиме, или добавьте `--include-secrets`, чтобы включить поддерживаемые учётные данные в автоматический запуск.

  </Step>
  <Step title="Запуск doctor">
    ```bash
    openclaw doctor
    ```

    [Doctor](/ru/gateway/doctor) повторно применяет все ожидающие миграции конфигурации и проверяет наличие проблем, возникших во время импорта.

  </Step>
  <Step title="Перезапуск и проверка">
    ```bash
    openclaw gateway restart
    openclaw status
    ```

    Убедитесь, что Gateway исправен, а импортированные модель, память и Skills загружены.

  </Step>
</Steps>

## Обработка конфликтов

Применение прерывается, если план сообщает о конфликтах — когда файл или значение конфигурации уже существует в целевом расположении.

<Warning>
Повторно запускайте команду с `--overwrite` только в том случае, если существующий целевой элемент необходимо заменить намеренно. Провайдеры всё равно могут создавать резервные копии отдельных перезаписываемых файлов в каталоге отчёта о миграции.
</Warning>

При чистой установке конфликты возникают редко. Обычно они появляются при повторном импорте в установку, уже содержащую пользовательские изменения.

Если конфликт возникает в процессе применения, например из-за неожиданной гонки при работе с файлом конфигурации, Hermes помечает оставшиеся зависимые элементы конфигурации как `skipped` с причиной `blocked by earlier apply conflict`, не записывая их частично. В отчёте о миграции фиксируется каждый заблокированный элемент, чтобы вы могли устранить исходный конфликт и повторить импорт.

## Секреты

Интерактивный запуск `openclaw migrate` спрашивает, следует ли импортировать обнаруженные учётные данные аутентификации, при этом по умолчанию выбран ответ «да».

- При согласии импортируются записи OpenCode OpenAI OAuth и GitHub Copilot из файла OpenCode `auth.json`, а также [поддерживаемые ключи `.env`](/ru/cli/migrate#supported-env-keys). Собственные записи OAuth из файла Hermes `auth.json` вместо этого включаются в отчёт как требующие ручной повторной аутентификации OpenAI или исправления с помощью doctor.
- Используйте `--no-auth-credentials` или ответьте «нет» на запрос, чтобы импортировать только несекретное состояние.
- Используйте `--include-secrets`, чтобы импортировать учётные данные при автоматическом запуске с `--yes`.
- Используйте флаг мастера первоначальной настройки `--import-secrets`, чтобы импортировать учётные данные через мастер.

## Вывод JSON для автоматизации

```bash
openclaw migrate hermes --dry-run --json
openclaw migrate apply hermes --json --yes
```

При использовании `--json` без `--yes` команда применения выводит план и не изменяет состояние — это самый безопасный режим для CI и общих скриптов.

## Устранение неполадок

<AccordionGroup>
  <Accordion title="Применение прерывается из-за конфликтов">
    Изучите вывод плана. Для каждого конфликта указаны исходный путь и существующий целевой элемент. Для каждого элемента решите, следует ли его пропустить, изменить целевой элемент или повторить запуск с `--overwrite`.
  </Accordion>
  <Accordion title="Hermes находится не в ~/.hermes">
    Передайте `--from /actual/path` для CLI или `--import-source /actual/path` для первоначальной настройки.
  </Accordion>
  <Accordion title="Первоначальная настройка отказывается импортировать в существующую установку">
    Для импорта через первоначальную настройку требуется чистая установка. Сбросьте состояние и повторите первоначальную настройку либо используйте `openclaw migrate apply hermes` напрямую: эта команда поддерживает `--overwrite` и явное управление резервным копированием.
  </Accordion>
  <Accordion title="Ключи API не импортировались">
    Интерактивный запуск `openclaw migrate` импортирует ключи API, только если вы согласитесь на импорт учётных данных. Для неинтерактивных запусков с `--yes` требуется `--include-secrets`, а для импорта через первоначальную настройку — `--import-secrets`. Распознаются только [поддерживаемые ключи `.env`](/ru/cli/migrate#supported-env-keys), остальные переменные `.env` игнорируются.
  </Accordion>
</AccordionGroup>
