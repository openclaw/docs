## Рекомендуемый вариант по умолчанию (для большинства пользователей)

- **Небольшой VPS с Linux** для постоянно работающего Gateway и низких затрат. См. [Хостинг на VPS](/ru/vps).
- **Выделенное оборудование** (Mac mini или компьютер с Linux), если вам нужен полный контроль и **резидентский IP-адрес** для автоматизации браузера. Многие сайты блокируют IP-адреса центров обработки данных, поэтому локальный браузер часто работает лучше.
- **Гибридный вариант**: разместите Gateway на недорогом VPS и подключайте Mac как **узел**, когда требуется автоматизация браузера или пользовательского интерфейса. См. [Узлы](/ru/nodes) и [Удалённый Gateway](/ru/gateway/remote).

Используйте виртуальную машину macOS только тогда, когда вам нужны возможности исключительно для macOS, например iMessage, или строгая изоляция от вашего основного Mac.

## Варианты виртуальных машин macOS

### Локальная виртуальная машина на Mac с Apple Silicon (Lume)

Запустите OpenClaw в изолированной виртуальной машине macOS на имеющемся Mac с Apple Silicon с помощью [Lume](https://cua.ai/docs/lume). Это обеспечивает:

- Полностью изолированную среду macOS (основная система остаётся чистой)
- Поддержку iMessage через `imsg`; стандартный локальный вариант невозможен в Linux/Windows
- Мгновенный сброс путём клонирования виртуальных машин
- Отсутствие дополнительных расходов на оборудование или облачные ресурсы

### Поставщики размещённых Mac (облако)

Если вам нужна macOS в облаке, также подойдут поставщики размещённых Mac:

- [MacStadium](https://www.macstadium.com/) (размещённые Mac)
- Подойдут и другие поставщики размещённых Mac; следуйте их документации по виртуальным машинам и SSH

Получив доступ по SSH к виртуальной машине macOS, перейдите к разделу [Установка OpenClaw](#6-install-openclaw) ниже.

## Быстрый способ (Lume, для опытных пользователей)

1. Установите Lume.
2. `lume create openclaw --os macos --ipsw latest`
3. Завершите работу Ассистента настройки и включите Remote Login (SSH).
4. `lume run openclaw --no-display`
5. Подключитесь по SSH, установите OpenClaw и настройте каналы.
6. Готово.

## Что потребуется (Lume)

- Mac с Apple Silicon (M1/M2/M3/M4)
- macOS Sequoia или более поздней версии на основной системе
- Около 60 ГБ свободного места на диске для каждой виртуальной машины
- Около 20 минут

## 1) Установите Lume

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/trycua/cua/main/libs/lume/scripts/install.sh)"
```

Если `~/.local/bin` отсутствует в PATH:

```bash
echo 'export PATH="$PATH:$HOME/.local/bin"' >> ~/.zshrc && source ~/.zshrc
```

Проверьте:

```bash
lume --version
```

Документация: [Установка Lume](https://cua.ai/docs/lume/guide/getting-started/installation)

## 2) Создайте виртуальную машину macOS

```bash
lume create openclaw --os macos --ipsw latest
```

Эта команда загрузит macOS и создаст виртуальную машину. Окно VNC откроется автоматически.

<Note>
Загрузка может занять некоторое время в зависимости от скорости подключения.
</Note>

## 3) Завершите работу Ассистента настройки

В окне VNC:

1. Выберите язык и регион.
2. Пропустите вход в Apple ID (или войдите, если позже хотите использовать iMessage).
3. Создайте учётную запись пользователя (запомните имя пользователя и пароль).
4. Пропустите все необязательные функции.

После завершения настройки:

1. Включите SSH: System Settings -> General -> Sharing, затем включите "Remote Login".
2. Для использования виртуальной машины без графического интерфейса включите автоматический вход: System Settings -> Users & Groups, выберите "Automatically log in as:" и укажите пользователя виртуальной машины.

## 4) Узнайте IP-адрес виртуальной машины

```bash
lume get openclaw
```

Найдите IP-адрес (обычно `192.168.64.x`).

## 5) Подключитесь к виртуальной машине по SSH

```bash
ssh youruser@192.168.64.X
```

Замените `youruser` именем созданной учётной записи, а IP-адрес — IP-адресом виртуальной машины.

## 6) Установите OpenClaw

Внутри виртуальной машины:

```bash
npm install -g openclaw@latest
openclaw onboard --install-daemon
```

Следуйте подсказкам первоначальной настройки, чтобы настроить поставщика модели (Anthropic, OpenAI и т. д.).

## 7) Настройте каналы

Откройте файл конфигурации:

```bash
nano ~/.openclaw/openclaw.json
```

Добавьте каналы:

```json5
{
  channels: {
    telegram: {
      botToken: "YOUR_BOT_TOKEN",
    },
    whatsapp: {
      dmPolicy: "allowlist",
      allowFrom: ["+15551234567"],
    },
  },
}
```

Затем войдите в WhatsApp (отсканируйте QR-код):

```bash
openclaw channels login
```

## 8) Запустите виртуальную машину без графического интерфейса

Остановите виртуальную машину и перезапустите её без дисплея:

```bash
lume stop openclaw
lume run openclaw --no-display
```

Виртуальная машина работает в фоновом режиме, а демон OpenClaw поддерживает работу Gateway. Чтобы проверить состояние:

```bash
ssh youruser@192.168.64.X "openclaw status"
```

## Дополнительно: интеграция с iMessage

Это главное преимущество работы в macOS. Используйте [iMessage](/ru/channels/imessage) с `imsg`, чтобы добавить «Сообщения» в OpenClaw.

Внутри виртуальной машины:

1. Войдите в «Сообщения».
2. Установите `imsg`.
3. Предоставьте полный доступ к диску и разрешение на автоматизацию процессу, запускающему OpenClaw/`imsg`.
4. Проверьте поддержку RPC с помощью `imsg rpc --help`.

Добавьте в конфигурацию OpenClaw:

```json5
{
  channels: {
    imessage: {
      enabled: true,
      cliPath: "imsg",
      dbPath: "~/Library/Messages/chat.db",
    },
  },
}
```

Перезапустите Gateway. Теперь ваш агент может отправлять и получать сообщения iMessage. Полные сведения о настройке: [Канал iMessage](/ru/channels/imessage).

## Сохраните эталонный образ

Перед дальнейшей настройкой создайте снимок чистого состояния:

```bash
lume stop openclaw
lume clone openclaw openclaw-golden
```

Чтобы выполнить сброс в любое время:

```bash
lume stop openclaw && lume delete openclaw
lume clone openclaw-golden openclaw
lume run openclaw --no-display
```

## Круглосуточная работа

Чтобы виртуальная машина продолжала работать:

- Держите Mac подключённым к электросети
- Отключите переход в режим сна в System Settings -> Energy Saver
- При необходимости используйте `caffeinate`

Для действительно непрерывной работы рассмотрите выделенный Mac mini или небольшой VPS. См. [Хостинг на VPS](/ru/vps).

## Устранение неполадок

| Проблема                                 | Решение                                                                                                  |
| ---------------------------------------- | -------------------------------------------------------------------------------------------------------- |
| Не удаётся подключиться к ВМ по SSH      | Убедитесь, что "Remote Login" включён в System Settings виртуальной машины                               |
| IP-адрес виртуальной машины не отображается | Дождитесь полной загрузки виртуальной машины и снова выполните `lume get openclaw`                      |
| Команда Lume не найдена                  | Добавьте `~/.local/bin` в PATH                                                                       |
| QR-код WhatsApp не сканируется           | При выполнении `openclaw channels login` убедитесь, что вы вошли в виртуальную, а не в основную систему         |
