Разверните OpenClaw на [Render](https://render.com), используя Blueprint из репозитория: `render.yaml`. Он объявляет сервис, диск и переменные окружения в одном файле.

## Предварительные требования

- Учетная запись [Render](https://render.com) (доступен бесплатный тариф)
- Ключ API выбранного [провайдера моделей](/ru/providers)

## Развертывание

[Развернуть в Render](https://render.com/deploy?repo=https://github.com/openclaw/openclaw)

При этом из `render.yaml` создается сервис Render, собирается образ Docker и выполняется его развертывание. URL вашего сервиса соответствует шаблону `https://<service-name>.onrender.com`.

## Blueprint

```yaml
services:
  - type: web
    name: openclaw
    runtime: docker
    plan: starter
    healthCheckPath: /health
    envVars:
      - key: OPENCLAW_GATEWAY_PORT
        value: "8080"
      - key: OPENCLAW_STATE_DIR
        value: /data/.openclaw
      - key: OPENCLAW_WORKSPACE_DIR
        value: /data/workspace
      - key: OPENCLAW_GATEWAY_TOKEN
        generateValue: true # автоматически создает защищенный токен
    disk:
      name: openclaw-data
      mountPath: /data
      sizeGB: 1
```

| Возможность               | Назначение                                                    |
| --------------------- | ---------------------------------------------------------- |
| `runtime: docker`     | Сборка из Dockerfile репозитория                          |
| `healthCheckPath`     | Render отслеживает `/health` и перезапускает неработоспособные экземпляры |
| `generateValue: true` | Автоматически создает криптографически стойкое значение            |
| `disk`                | Постоянное хранилище, сохраняющееся при повторных развертываниях                 |

## Выбор тарифа

| Тариф      | Остановка         | Диск          | Лучше всего подходит для                      |
| --------- | ----------------- | ------------- | ----------------------------- |
| Free      | После 15 мин бездействия | Недоступен | Тестирования, демонстраций                |
| Starter   | Никогда             | 1GB+          | Личного использования, небольших команд     |
| Standard+ | Никогда             | 1GB+          | Рабочей среды, нескольких каналов |

По умолчанию Blueprint использует `starter`. Чтобы использовать бесплатный тариф, измените `plan: free` в файле `render.yaml` своего форка. Учтите, что без постоянного диска состояние OpenClaw сбрасывается при каждом развертывании.

## После развертывания

### Доступ к интерфейсу управления

Веб-панель доступна по адресу `https://<your-service>.onrender.com/`. Подключитесь с помощью общего секрета — автоматически созданного `OPENCLAW_GATEWAY_TOKEN` (его можно найти в **Dashboard → your service → Environment**) — или с помощью пароля, если вы переключились на аутентификацию по паролю.

### Журналы

В разделе **Dashboard → your service → Logs** отображаются журналы сборки (создание образа Docker), журналы развертывания (запуск сервиса) и журналы среды выполнения (вывод приложения).

### Доступ к оболочке

Раздел **Dashboard → your service → Shell** открывает сеанс оболочки. Постоянный диск смонтирован в `/data`.

### Переменные окружения

Изменяйте переменные в разделе **Dashboard → your service → Environment**. Изменения автоматически запускают повторное развертывание.

### Автоматическое развертывание

Render автоматически выполняет повторное развертывание, когда в ветке подключенного репозитория появляется новый коммит. Если вы выполнили развертывание непосредственно из `openclaw/openclaw`, а не из собственного форка, у вас нет доступа на отправку изменений для его запуска. Поэтому обновляйте сервис вручную, запуская синхронизацию Blueprint из Dashboard, либо подключите сервис к собственному форку.

## Пользовательский домен

1. **Dashboard → your service → Settings → Custom Domains**
2. Добавьте свой домен
3. Настройте DNS согласно инструкциям (CNAME на `*.onrender.com`)
4. Render автоматически выпускает сертификат TLS

## Масштабирование

- **Вертикальное**: измените тариф, чтобы получить больше ресурсов CPU и RAM. Обычно этого достаточно для OpenClaw.
- **Горизонтальное**: увеличьте количество экземпляров (тариф Standard и выше). Поскольку OpenClaw хранит состояние среды выполнения на локальном диске, потребуются закрепленные сеансы или внешнее управление состоянием.

## Резервное копирование и миграция

Из оболочки Render Dashboard можно в любое время экспортировать состояние, конфигурацию, профили аутентификации и рабочее пространство:

```bash
openclaw backup create
```

Команда создает переносимый архив резервной копии. См. раздел [Резервное копирование](/ru/cli/backup).

## Устранение неполадок

### Сервис не запускается

Проверьте журналы развертывания в Render Dashboard. Распространенные проблемы:

- Отсутствует `OPENCLAW_GATEWAY_TOKEN` — убедитесь, что значение задано в разделе **Dashboard → Environment**
- Несоответствие порта — убедитесь, что `OPENCLAW_GATEWAY_PORT=8080`, чтобы Gateway привязывался к порту, ожидаемому Render

### Медленный холодный запуск (бесплатный тариф)

Сервисы бесплатного тарифа останавливаются после 15 минут бездействия. Первый запрос после остановки занимает несколько секунд, пока запускается контейнер. Перейдите на тариф Starter, чтобы сервис работал постоянно.

### Потеря данных после повторного развертывания

Происходит на бесплатном тарифе из-за отсутствия постоянного диска. Перейдите на платный тариф или регулярно экспортируйте резервную копию с помощью `openclaw backup create` из оболочки Render.

### Сбои проверки работоспособности

Если сборка завершается успешно, но развертывание заканчивается сбоем, возможно, сервис запускается слишком долго или `/health` недоступен. Проверьте:

- Журналы сборки на наличие ошибок
- Запускается ли контейнер локально с помощью `docker build && docker run`
