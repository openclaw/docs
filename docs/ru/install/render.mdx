Разверните OpenClaw на [Render](https://render.com), используя Blueprint-файл `render.yaml` из репозитория. В нём в одном файле объявлены сервис, диск и переменные окружения.

## Предварительные требования

- [Учётная запись Render](https://render.com) (доступен бесплатный тариф)
- Ключ API выбранного [поставщика моделей](/ru/providers)

## Развёртывание

[Развернуть в Render](https://render.com/deploy?repo=https://github.com/openclaw/openclaw)

Будет создан сервис Render на основе `render.yaml`, собран образ Docker и выполнено развёртывание. URL вашего сервиса будет иметь вид `https://<service-name>.onrender.com`.

## Blueprint

```yaml
services:
  - type: web
    name: openclaw
    runtime: docker
    plan: starter
    healthCheckPath: /health
    envVars:
      - key: OPENCLAW_GATEWAY_PORT
        value: "8080"
      - key: OPENCLAW_STATE_DIR
        value: /data/.openclaw
      - key: OPENCLAW_WORKSPACE_DIR
        value: /data/workspace
      - key: OPENCLAW_GATEWAY_TOKEN
        generateValue: true # автоматически создаёт защищённый токен
    disk:
      name: openclaw-data
      mountPath: /data
      sizeGB: 1
```

| Возможность           | Назначение                                                        |
| --------------------- | ----------------------------------------------------------------- |
| `runtime: docker`     | Сборка из Dockerfile репозитория                                  |
| `healthCheckPath`     | Render проверяет `/health` и перезапускает неработоспособные экземпляры |
| `generateValue: true` | Автоматическое создание криптографически стойкого значения        |
| `disk`                | Постоянное хранилище, сохраняющее данные при повторных развёртываниях |

## Выбор тарифа

| Тариф     | Остановка при простое      | Диск          | Лучше всего подходит для             |
| --------- | -------------------------- | ------------- | ------------------------------------ |
| Free      | После 15 минут бездействия | Недоступен    | Тестирования, демонстраций           |
| Starter   | Никогда                    | 1 ГБ и более  | Личного использования, небольших команд |
| Standard+ | Никогда                    | 1 ГБ и более  | Рабочей среды, нескольких каналов    |

По умолчанию Blueprint использует тариф `starter`. Чтобы использовать бесплатный тариф, измените `plan: free` в файле `render.yaml` вашего форка. Обратите внимание: без постоянного диска состояние OpenClaw будет сбрасываться при каждом развёртывании.

## После развёртывания

### Доступ к интерфейсу управления

Веб-панель доступна по адресу `https://<your-service>.onrender.com/`. Для подключения используйте общий секрет: автоматически созданный `OPENCLAW_GATEWAY_TOKEN` (его можно найти в разделе **Dashboard → your service → Environment**) или пароль, если вы перешли на аутентификацию по паролю.

### Журналы

В разделе **Dashboard → your service → Logs** отображаются журналы сборки (создание образа Docker), развёртывания (запуск сервиса) и выполнения (вывод приложения).

### Доступ к командной оболочке

Раздел **Dashboard → your service → Shell** открывает сеанс командной оболочки. Постоянный диск подключён по пути `/data`.

### Переменные окружения

Изменяйте переменные в разделе **Dashboard → your service → Environment**. Изменения запускают автоматическое повторное развёртывание.

### Автоматическое развёртывание

Render автоматически выполняет повторное развёртывание, когда в подключённой ветке репозитория появляется новый коммит. Если вы выполняли развёртывание непосредственно из `openclaw/openclaw`, а не из собственного форка, у вас нет прав на отправку изменений для запуска этого процесса. В таком случае выполните ручную синхронизацию Blueprint из Dashboard или подключите к сервису собственный форк.

## Пользовательский домен

1. **Dashboard → your service → Settings → Custom Domains**
2. Добавьте свой домен
3. Настройте DNS в соответствии с инструкциями (CNAME на `*.onrender.com`)
4. Render автоматически выпустит сертификат TLS

## Масштабирование

- **Вертикальное**: смените тариф, чтобы получить больше ресурсов ЦП и оперативной памяти. Обычно этого достаточно для OpenClaw.
- **Горизонтальное**: увеличьте количество экземпляров (доступно на тарифе Standard и выше). Потребуются привязка сеансов к экземплярам или внешнее управление состоянием, поскольку OpenClaw хранит состояние выполнения на локальном диске.

## Резервное копирование и миграция

Из командной оболочки Render Dashboard можно в любое время экспортировать состояние, конфигурацию, профили аутентификации и рабочую область:

```bash
openclaw backup create
```

Эта команда создаёт переносимый архив резервной копии. См. раздел [Резервное копирование](/ru/cli/backup).

## Устранение неполадок

### Сервис не запускается

Проверьте журналы развёртывания в Render Dashboard. Распространённые проблемы:

- Отсутствует `OPENCLAW_GATEWAY_TOKEN` — убедитесь, что он задан в разделе **Dashboard → Environment**
- Несоответствие порта — убедитесь, что задано значение `OPENCLAW_GATEWAY_PORT=8080`, чтобы Gateway прослушивал порт, ожидаемый Render

### Медленный холодный запуск на бесплатном тарифе

Сервисы бесплатного тарифа останавливаются после 15 минут бездействия. Первый запрос после остановки занимает несколько секунд, пока запускается контейнер. Перейдите на тариф Starter, чтобы сервис работал непрерывно.

### Потеря данных после повторного развёртывания

Это происходит на бесплатном тарифе, где нет постоянного диска. Перейдите на платный тариф или регулярно экспортируйте резервную копию из командной оболочки Render с помощью команды `openclaw backup create`.

### Ошибки проверки работоспособности

Если сборка завершается успешно, но развёртывание не удаётся, возможно, сервис запускается слишком долго или путь `/health` недоступен. Проверьте:

- Журналы сборки на наличие ошибок
- Запускается ли контейнер локально с помощью `docker build && docker run`
