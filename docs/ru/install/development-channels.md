OpenClaw поставляется с четырьмя каналами обновлений:

- **stable**: dist-tag npm `latest`. Рекомендуется большинству пользователей.
- **extended-stable**: dist-tag npm `extended-stable`. Совершенно новый канал пакетов
  за предыдущий поддерживаемый месяц. Он предназначен только для пакетов, а установка
  выполняется только в интерактивном режиме на переднем плане. При сохранённом выборе отображаются доступные только для чтения подсказки об обновлениях, если
  включён `update.checkOnStart`, но обновления никогда не применяются автоматически.
- **beta**: dist-tag npm `beta`. Использует `latest`, если `beta` отсутствует
  или старше текущего стабильного выпуска.
- **dev**: постоянно обновляемая вершина `main` (git). При публикации используется dist-tag npm `dev`. `main`
  предназначен для экспериментов и активной разработки; он может содержать незавершённые
  функции или несовместимые изменения. Не используйте его для рабочих Gateway.

Стабильные сборки обычно сначала выпускаются в канале **beta**, проходят там проверку, а затем
переводятся в **latest** без повышения версии. Сопровождающие также могут публиковать
непосредственно в `latest`. Dist-tag являются источником истины для установок npm.

## Переключение каналов

```bash
openclaw update --channel stable
openclaw update --channel extended-stable
openclaw update --channel beta
openclaw update --channel dev
```

`--channel` сохраняет выбор в параметре `update.channel` конфигурации и управляет обоими
способами установки:

| Канал             | Установки npm/пакетов                                                                                                                                                                  | Установки git                                                                                                                                                      |
| ----------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `stable`          | dist-tag `latest`                                                                                                                                                                      | последний стабильный тег git (исключая `-alpha.N`, `-beta.N`, `-rc.N`, `-dev.N`, `-next.N`, `-preview.N`, `-canary.N`, `-nightly.N` и другие именованные суффиксы предварительных выпусков) |
| `extended-stable` | разрешает общедоступный селектор npm `extended-stable`, проверяет точно выбранный пакет и устанавливает именно эту версию. Завершается отказом без перехода к `latest`, `beta` или `dev`. | не поддерживается: OpenClaw оставляет рабочую копию без изменений и предлагает использовать установку пакета                                                       |
| `beta`            | dist-tag `beta` с переходом к `latest`, если `beta` отсутствует или старше                                                                                                              | последний бета-тег git с переходом к последнему стабильному тегу git, если бета-версия отсутствует или старше                                                     |
| `dev`             | dist-tag `dev` (используется редко; большинство пользователей dev используют установки git)                                                                                                                        | получает изменения, перебазирует рабочую копию на вышестоящую ветку `main`, выполняет сборку и переустанавливает глобальный CLI                         |

Для установок git `dev` рабочая копия по умолчанию находится в `~/openclaw` (или
`$OPENCLAW_HOME/openclaw`, если задан `OPENCLAW_HOME`); переопределите это с помощью
`OPENCLAW_GIT_DIR`.

<Tip>
Чтобы использовать stable и dev параллельно, создайте две отдельные рабочие копии и направьте каждый Gateway на собственную.
</Tip>

## Однократный выбор версии или тега

Используйте `--tag`, чтобы выбрать конкретный dist-tag, версию или спецификацию пакета для
одного обновления **без** изменения сохранённого канала:

```bash
# Установить конкретную версию
openclaw update --tag 2026.4.1-beta.1

# Установить из dist-tag beta (однократно, без сохранения)
openclaw update --tag beta

# Переключиться на постоянно обновляемую рабочую копию GitHub main (с сохранением)
openclaw update --channel dev

# Установить конкретную спецификацию пакета npm
openclaw update --tag openclaw@2026.4.1-beta.1

# Однократно установить из GitHub main без сохранения канала
openclaw update --tag main
```

Примечания:

- `--tag` применяется **только к установкам пакетов (npm)**; установки git его игнорируют.
- Тег не сохраняется; следующий запуск `openclaw update` использует настроенный
  канал.
- `--tag main` сопоставляется с совместимой с npm спецификацией `github:openclaw/openclaw#main`
  для этого одного запуска. Для постоянной установки из обновляемой ветки `main` используйте
  `openclaw update --channel dev` (установки пакетов переключатся на рабочую копию git)
  или переустановите с помощью метода git установщика:
  `curl -fsSL https://openclaw.ai/install.sh | bash -s -- --install-method git --version main`.
  Способ установки через npm сразу отклоняет цели, указывающие на источники GitHub/git, и
  предлагает вместо этого использовать метод git.
- Защита от понижения версии: если целевая версия старше текущей,
  OpenClaw запрашивает подтверждение (пропустите его с помощью `--yes`).
- Extended-stable всегда использует точную проверенную целевую версию пакета. Это не
  однократный псевдоним для `--tag extended-stable`, а `--tag` нельзя сочетать
  с фактически выбранным каналом extended-stable.
- `--channel beta` отличается от `--tag beta`: при выборе канала может использоваться
  stable/latest, если beta отсутствует или старше, тогда как `--tag beta` всегда
  выбирает исходный dist-tag `beta` для этого одного запуска.

## Пробный запуск

Просмотрите, что сделает `openclaw update`, без внесения изменений:

```bash
openclaw update --dry-run
openclaw update --channel beta --dry-run
openclaw update --tag 2026.4.1-beta.1 --dry-run
openclaw update --dry-run --json
```

Пробный запуск сообщает фактический канал, целевую версию, запланированные действия,
а также необходимость подтверждения понижения версии.

## Плагины и каналы

Переключение каналов с помощью `openclaw update` также синхронизирует источники плагинов:

- `dev` переключает установленные плагины, имеющие встроенные аналоги, обратно на
  встроенный источник (рабочую копию git).
- `stable` и `beta` восстанавливают пакеты плагинов,
  установленные через npm или ClawHub.
- `extended-stable` разрешает подходящие официальные плагины npm с пустым намерением, намерением по умолчанию
  или намерением `latest` в точную установленную версию ядра. Во время выполнения он не запрашивает
  теги `@extended-stable` плагинов.
- Плагины, установленные через npm, обновляются после завершения обновления ядра.

## Проверка текущего состояния

```bash
openclaw update status
```

Показывает активный канал с указанием определившего его источника (конфигурация, тег git,
ветка git, установленная версия или значение по умолчанию), тип установки (git или пакет),
текущую версию и наличие обновления.

## Рекомендации по тегам

- Помечайте тегами выпуски, на которых должны останавливаться рабочие копии git: `vYYYY.M.PATCH` для stable,
  `vYYYY.M.PATCH-beta.N` для beta. Именованные суффиксы предварительных выпусков, такие как
  `-alpha.N`, `-rc.N` и `-next.N`, не являются целевыми версиями stable или beta.
- Устаревшие числовые стабильные теги, такие как `vYYYY.M.PATCH-1` и `v1.0.1-1`, по-прежнему
  распознаются как стабильные теги git для совместимости.
- `vYYYY.M.PATCH.beta.N` (с точками в качестве разделителей) также распознаётся для совместимости;
  предпочитайте `-beta.N`.
- Сохраняйте теги неизменными: никогда не перемещайте и не используйте тег повторно.
- Dist-tag npm остаются источником истины для установок npm:
  - `latest` -> stable
  - `extended-stable` -> выпуск пакета за предыдущий поддерживаемый месяц
  - `beta` -> сборка-кандидат или стабильная сборка, сначала выпущенная как beta
  - `dev` -> снимок main (необязательно)

## Доступность приложения macOS

Сборки beta и dev могут **не** включать выпуск приложения macOS. Это допустимо:

- Тег git и dist-tag npm всё равно можно публиковать независимо.
- Укажите «нет сборки macOS для этой бета-версии» в примечаниях к выпуску или журнале изменений.
