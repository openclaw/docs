OpenClaw выполняет работу в фоновом режиме с помощью задач, запланированных заданий, предполагаемых
обязательств, обработчиков событий и постоянных инструкций. Используйте эту страницу, чтобы выбрать
подходящий механизм.

### Запланированные задачи (Cron) и Heartbeat

| Характеристика   | Запланированные задачи (Cron)       | Heartbeat                                  |
| ---------------- | ----------------------------------- | ------------------------------------------ |
| Время запуска    | Точное (выражения cron, однократный запуск) | Приблизительное (по умолчанию каждые 30 минут) |
| Контекст сеанса  | Новый (изолированный) или общий     | Полный контекст основного сеанса           |
| Записи задач     | Создаются всегда                    | Никогда не создаются                       |
| Доставка         | Канал, вебхук или без доставки      | В основном сеансе                          |
| Лучше всего для  | Отчётов, напоминаний, фоновых заданий | Проверки входящих, календаря и уведомлений |

Используйте запланированные задачи (Cron), когда требуется точное время или изолированное выполнение. Используйте Heartbeat, когда для работы полезен полный контекст сеанса и приблизительное время запуска допустимо.

## Основные понятия

### Запланированные задачи (cron)

Cron — встроенный в Gateway планировщик для точного запуска по времени. Он сохраняет задания, активирует агента в нужный момент и может доставлять результат в канал чата или конечную точку вебхука. Поддерживает однократные напоминания, повторяющиеся выражения и входящие триггеры вебхуков.

См. [Запланированные задачи](/ru/automation/cron-jobs).

### Задачи

Журнал фоновых задач отслеживает всю отделённую работу: выполнения ACP, запуск субагентов, изолированные выполнения cron и операции CLI. Задачи — это записи, а не планировщики. Для их просмотра используйте `openclaw tasks list` и `openclaw tasks audit`.

См. [Фоновые задачи](/ru/automation/tasks).

### Предполагаемые обязательства

Обязательства — это включаемые пользователем кратковременные воспоминания о необходимых последующих действиях. OpenClaw определяет их
из обычных разговоров, ограничивает тем же агентом и каналом и
доставляет своевременные проверки через Heartbeat. Точные напоминания, явно запрошенные пользователем, по-прежнему
должны обрабатываться через cron.

См. [Предполагаемые обязательства](/ru/concepts/commitments).

### Поток задач

Поток задач — это уровень оркестрации потоков поверх фоновых задач. Он управляет надёжными многоэтапными потоками с управляемым и зеркальным режимами синхронизации, отслеживанием редакций и командами `openclaw tasks flow list|show|cancel` для просмотра.

См. [Поток задач](/ru/automation/taskflow).

### Постоянные инструкции

Постоянные инструкции предоставляют агенту бессрочные полномочия на выполнение определённых программ. Они хранятся в файлах рабочего пространства (обычно `AGENTS.md`) и внедряются в каждый сеанс. Для исполнения по расписанию сочетайте их с cron.

См. [Постоянные инструкции](/ru/automation/standing-orders).

### Обработчики

Внутренние обработчики — это событийные скрипты, запускаемые событиями жизненного цикла агента
(`/new`, `/reset`, `/stop`), Compaction сеанса, запуском Gateway и потоком
сообщений. Они обнаруживаются в каталогах обработчиков и управляются с помощью
`openclaw hooks`. Для внутрипроцессного перехвата вызовов инструментов используйте
[обработчики Plugin](/ru/plugins/hooks).

См. [Обработчики](/ru/automation/hooks).

### Heartbeat

Heartbeat — это периодический ход в основном сеансе (по умолчанию каждые 30 минут). Он объединяет несколько проверок (входящие, календарь, уведомления) в один ход агента с полным контекстом сеанса. Ходы Heartbeat не создают записи задач и не продлевают период актуальности до ежедневного сброса сеанса или сброса из-за бездействия. Используйте `HEARTBEAT.md` для небольшого списка проверок или блок `tasks:`, если нужны только наступившие периодические проверки непосредственно внутри Heartbeat. Пустые файлы Heartbeat пропускаются с причиной `empty-heartbeat-file`; режим задач только с наступившим сроком пропускается с причиной `no-tasks-due`. Heartbeat откладывается, пока работа cron активна или находится в очереди, а `heartbeat.skipWhenBusy` также может отложить работу агента, пока заняты привязанные к ключу сеанса субагентские или вложенные линии того же агента.

См. [Heartbeat](/ru/gateway/heartbeat).

## Совместная работа механизмов

- **Cron** обрабатывает точные расписания (ежедневные отчёты, еженедельные обзоры) и однократные напоминания. Все выполнения cron создают записи задач.
- **Heartbeat** выполняет регулярный мониторинг (входящие, календарь, уведомления) одним объединённым ходом каждые 30 минут.
- **Обработчики** реагируют на определённые события (сбросы сеансов, Compaction, поток сообщений) с помощью пользовательских скриптов. Обработчики Plugin охватывают вызовы инструментов.
- **Постоянные инструкции** предоставляют агенту постоянный контекст и границы полномочий.
- **Поток задач** координирует многоэтапные потоки поверх отдельных задач.
- **Задачи** автоматически отслеживают всю отделённую работу, чтобы её можно было просматривать и проверять.
