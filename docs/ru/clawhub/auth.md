# Аутентификация

ClawHub использует GitHub для входа в веб-интерфейс. CLI использует API-токены ClawHub, созданные
через эту учетную запись после входа.

## Вход через веб

Используйте GitHub, чтобы войти на [clawhub.ai](https://clawhub.ai).

Удаленные, заблокированные или отключенные учетные записи не могут пройти обычный вход в ClawHub.
Если после входа вы возвращаетесь в состояние без авторизации, ваша учетная запись может быть не в хорошем
состоянии. Если ваша учетная запись была заблокирована или отключена, используйте
[форму апелляции ClawHub](https://appeals.openclaw.ai/), если считаете, что это
ошибка.

## Вход в CLI

Стандартный процесс входа в CLI открывает браузер:

```bash
clawhub login
clawhub whoami
```

Что происходит:

1. CLI запускает временный сервер обратного вызова на `127.0.0.1`.
2. Ваш браузер открывает страницу входа ClawHub.
3. После входа через GitHub ClawHub создает API-токен.
4. Браузер перенаправляет обратно на локальный обратный вызов.
5. CLI сохраняет токен в вашем конфигурационном файле ClawHub.

Если ваш браузер не может обратиться к локальному обратному вызову из-за правил файрвола, VPN или
прокси, используйте поток токена без браузера.

## Вход без браузера

Создайте токен в веб-интерфейсе ClawHub, затем передайте его в CLI:

```bash
clawhub login --token clh_...
```

Используйте этот поток для серверов, заданий CI или сред только с терминалом.

Для удаленных оболочек, где браузер можно открыть в другом месте, выполните:

```bash
clawhub login --device
```

CLI выводит одноразовый код и ожидает, пока вы авторизуете его на
`https://clawhub.ai/cli/device`.

## Хранение токена

Пути конфигурации по умолчанию:

- macOS: `~/Library/Application Support/clawhub/config.json`
- Linux/XDG: `$XDG_CONFIG_HOME/clawhub/config.json` или `~/.config/clawhub/config.json`
- Windows: `%APPDATA%\\clawhub\\config.json`

Переопределите путь с помощью:

```bash
export CLAWHUB_CONFIG_PATH=/path/to/config.json
```

Выведите сохраненный токен для настройки CI с помощью:

```bash
clawhub token
```

## Отзыв

Вы можете отзывать API-токены в веб-интерфейсе ClawHub.

Отозванные, недействительные или отсутствующие токены возвращают `401 Unauthorized`. Войдите снова
с помощью `clawhub login` или предоставьте новый токен с помощью `clawhub login --token`.

Удаленные, заблокированные или отключенные учетные записи не могут продолжать использовать существующие API-токены.
Если ваша учетная запись была заблокирована или отключена, используйте
[форму апелляции ClawHub](https://appeals.openclaw.ai/), если считаете, что это
ошибка.
