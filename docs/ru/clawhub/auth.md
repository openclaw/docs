# Аутентификация

ClawHub использует GitHub для входа через веб-интерфейс. CLI использует API-токены ClawHub, созданные через эту учётную запись после входа.

## Вход через веб-интерфейс

Используйте GitHub для входа на [clawhub.ai](https://clawhub.ai).

Удалённые, заблокированные или отключённые учётные записи не могут выполнить обычный вход в ClawHub. Если после входа вы снова оказываетесь в состоянии без авторизации, возможно, у вашей учётной записи есть ограничения. Если ваша учётная запись была заблокирована или отключена и вы считаете, что это произошло по ошибке, воспользуйтесь [формой апелляции ClawHub](https://appeals.openclaw.ai/).

## Вход через CLI

При входе через CLI по умолчанию открывается браузер:

```bash
clawhub login
clawhub whoami
```

Что происходит:

1. CLI запускает временный сервер обратного вызова на `127.0.0.1`.
2. В браузере открывается страница входа в ClawHub.
3. После входа через GitHub ClawHub создаёт API-токен.
4. Браузер перенаправляет обратно на локальный адрес обратного вызова.
5. CLI сохраняет токен в файле конфигурации ClawHub.

Если браузер не может обратиться к локальному адресу обратного вызова из-за правил межсетевого экрана, VPN или прокси-сервера, используйте безбраузерный вход с помощью токена.

## Безбраузерный вход

Создайте токен в веб-интерфейсе ClawHub, а затем передайте его в CLI:

```bash
clawhub login --token clh_...
```

Используйте этот способ для серверов, заданий CI или сред, доступных только через терминал.

Для удалённых оболочек, когда браузер можно открыть на другом устройстве, выполните:

```bash
clawhub login --device
```

CLI выведет одноразовый код и будет ожидать, пока вы авторизуете его на странице `https://clawhub.ai/cli/device`.

## Хранение токена

Пути к конфигурации по умолчанию:

- macOS: `~/Library/Application Support/clawhub/config.json`
- Linux/XDG: `$XDG_CONFIG_HOME/clawhub/config.json` или `~/.config/clawhub/config.json`
- Windows: `%APPDATA%\\clawhub\\config.json`

Чтобы переопределить путь, используйте:

```bash
export CLAWHUB_CONFIG_PATH=/path/to/config.json
```

Чтобы вывести сохранённый токен для настройки CI, используйте:

```bash
clawhub token
```

## Отзыв токена

API-токены можно отозвать в веб-интерфейсе ClawHub.

При использовании отозванного, недействительного или отсутствующего токена возвращается ошибка `401 Unauthorized`. Войдите снова с помощью `clawhub login` или укажите новый токен с помощью `clawhub login --token`.

Удалённые, заблокированные или отключённые учётные записи не могут продолжать использовать существующие API-токены. Если ваша учётная запись была заблокирована или отключена и вы считаете, что это произошло по ошибке, воспользуйтесь [формой апелляции ClawHub](https://appeals.openclaw.ai/).
