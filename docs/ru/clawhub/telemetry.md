# Телеметрия

ClawHub использует минимальную телеметрию CLI для подсчёта совокупного количества установок.

## Когда собирается телеметрия

Телеметрия отправляется только в следующих случаях:

- Выполнен вход в CLI.
- Выполняется `clawhub install <slug>`.
- Телеметрия **не отключена** (см. раздел «Как отключить» ниже).

Если вход не выполнен, никакие данные не передаются.

## Какие данные мы собираем

При каждом передаваемом событии `clawhub install` CLI предпринимает одну попытку отправить событие установки.

Событие содержит:

- `slug`: идентификатор установленного навыка.
- `version`: установленная версия, если она известна.

### Какие данные мы _не_ собираем

- Пути к папкам и идентификаторы, полученные из путей к папкам.
- Содержимое файлов.
- Журналы отдельных запусков, запросы и прочие выходные данные CLI.

## Количество установок

ClawHub ведёт совокупные счётчики для каждого навыка:

- `installsAllTime`: уникальные пользователи, сообщившие хотя бы об одной установке навыка через CLI.
- `installsCurrent`: уникальные пользователи, сообщившие об установке и не удалившие свои
  телеметрические данные.

## Прозрачность и пользовательские настройки

Всем доступны только **совокупные счётчики установок**.

При удалении учётной записи также удаляются данные телеметрии.

## Как отключить телеметрию

Задайте переменную окружения:

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

После её задания CLI не будет отправлять телеметрию установок.
