# Безопасность

О проблемах безопасности ClawHub можно сообщать через GitHub Security Advisories для
`openclaw/clawhub`.

Используйте GitHub Security Advisories для уязвимостей в самом ClawHub. Хорошие
отчеты advisories ClawHub включают ошибки в:

- веб-сайте, API или CLI ClawHub
- публикации в реестре, загрузках, установках или целостности артефактов
- аутентификации, авторизации или токенах API
- сканировании, модерации или обработке отчетов

Не используйте advisories ClawHub для уязвимостей в собственном исходном коде
стороннего навыка или Plugin. Сообщайте о них напрямую издателю или в исходный
репозиторий, указанный в листинге ClawHub.

## Раскрытие уязвимостей

Поскольку ClawHub является размещенным облачным приложением, уязвимости сервиса
ClawHub по умолчанию не раскрываются публично. Они раскрываются публично, когда
есть доказательства реального влияния на пользователей или когда пользователям
нужно принять меры.

Примеры реального влияния на пользователей включают подтвержденную эксплуатацию,
раскрытие пользовательских данных или секретов, попадание вредоносного контента
к пользователям из-за сбоя платформы или любую проблему, которая требует от
пользователей ротировать учетные данные, обновить локальное ПО или принять
другие защитные меры.

Уязвимости в ПО, установленном пользователем, раскрываются публично, например
в пакетах CLI ClawHub, бинарных файлах, библиотеках или других артефактах
релиза, которые пользователям нужно обновить локально.
