# Безопасность

О проблемах безопасности ClawHub можно сообщать через GitHub Security Advisories для
`openclaw/clawhub`.

Используйте GitHub Security Advisories для уязвимостей в самом ClawHub. Хорошие
отчеты об advisory для ClawHub включают ошибки в:

- веб-сайте, API или CLI ClawHub
- публикации в реестре, загрузках, установках или целостности артефактов
- аутентификации, авторизации или API-токенах
- сканировании, модерации или обработке отчетов

Не используйте advisory ClawHub для уязвимостей в исходном коде стороннего навыка или
Plugin. Сообщайте о них напрямую издателю или в исходный
репозиторий, указанный в карточке ClawHub.

## Раскрытие уязвимостей

Поскольку ClawHub является размещенным облачным приложением, уязвимости сервиса ClawHub
по умолчанию публично не раскрываются. Они раскрываются публично, когда есть
доказательства реального влияния на пользователей или когда пользователям нужно предпринять действия.

Примеры реального влияния на пользователей включают подтвержденную эксплуатацию, раскрытие пользовательских
данных или секретов, попадание вредоносного содержимого к пользователям из-за сбоя платформы
или любую проблему, из-за которой пользователям требуется заменить учетные данные, обновить локальное ПО или
предпринять другие защитные действия.

Уязвимости в установленном пользователями ПО раскрываются публично, например
пакеты CLI ClawHub, бинарные файлы, библиотеки или другие артефакты релиза, которые пользователям
нужно обновить локально.
