# Безопасность

О проблемах безопасности ClawHub можно сообщать через рекомендации по безопасности GitHub для
`openclaw/clawhub`.

Используйте рекомендации по безопасности GitHub для уязвимостей непосредственно в ClawHub. К обоснованным
сообщениям об уязвимостях ClawHub относятся ошибки в:

- веб-сайте, API или CLI ClawHub
- публикации в реестре, скачивании, установке или обеспечении целостности артефактов
- аутентификации, авторизации или токенах API
- сканировании, модерации или обработке жалоб

Не используйте рекомендации по безопасности ClawHub для уязвимостей в исходном коде стороннего Skill или
Plugin. Сообщайте о них непосредственно издателю или в репозиторий исходного кода,
ссылка на который приведена в публикации ClawHub.

## Раскрытие информации об уязвимостях

Поскольку ClawHub — это размещаемое облачное приложение, уязвимости сервиса ClawHub
по умолчанию публично не раскрываются. Они раскрываются публично, если имеются
свидетельства реального воздействия на пользователей или пользователям необходимо принять меры.

К примерам реального воздействия на пользователей относятся подтверждённая эксплуатация, раскрытие пользовательских
данных или секретов, попадание вредоносного содержимого к пользователям из-за сбоя платформы,
а также любые проблемы, требующие от пользователей сменить учётные данные, обновить локальное программное обеспечение или
принять другие защитные меры.

Уязвимости в устанавливаемом пользователями программном обеспечении раскрываются публично, например
в пакетах CLI ClawHub, исполняемых файлах, библиотеках или других артефактах выпуска, которые пользователям
необходимо обновить локально.
