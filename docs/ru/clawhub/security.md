# Безопасность

О проблемах безопасности ClawHub можно сообщать через GitHub Security Advisories для
`openclaw/clawhub`.

Используйте GitHub Security Advisories для уязвимостей в самом ClawHub. Хорошие
отчеты для advisory ClawHub включают ошибки в:

- сайте, API или CLI ClawHub
- публикации в реестре, скачиваниях, установках или целостности артефактов
- аутентификации, авторизации или токенах API
- сканировании, модерации или обработке отчетов

Не используйте advisory ClawHub для уязвимостей в собственном исходном коде
стороннего навыка или Plugin. Сообщайте о них напрямую издателю или в исходный
репозиторий, указанный в листинге ClawHub.

## Раскрытие уязвимостей

Поскольку ClawHub — это размещенное облачное приложение, уязвимости сервиса
ClawHub по умолчанию публично не раскрываются. Они раскрываются публично, когда
есть доказательства реального влияния на пользователей или когда пользователям
нужно предпринять действия.

Примеры реального влияния на пользователей включают подтвержденную эксплуатацию,
раскрытие пользовательских данных или секретов, попадание вредоносного контента
к пользователям из-за сбоя платформы или любую проблему, из-за которой
пользователям необходимо сменить учетные данные, обновить локальное ПО или
предпринять другие защитные действия.

Уязвимости в ПО, устанавливаемом пользователями, раскрываются публично, например
в пакетах CLI ClawHub, бинарных файлах, библиотеках или других артефактах
выпуска, которые пользователям нужно обновить локально.
