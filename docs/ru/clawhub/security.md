# Безопасность

О проблемах безопасности ClawHub можно сообщать через GitHub Security Advisories для
`openclaw/clawhub`.

Используйте GitHub Security Advisories для уязвимостей в самом ClawHub. Хорошие
отчеты ClawHub advisory включают ошибки в:

- сайте ClawHub, API или CLI
- публикации в реестре, скачиваниях, установках или целостности артефактов
- аутентификации, авторизации или API-токенах
- сканировании, модерации или обработке отчетов

Не используйте ClawHub advisories для уязвимостей в собственном исходном коде стороннего навыка или
Plugin. Сообщайте о них напрямую издателю или в исходный
репозиторий, указанный в листинге ClawHub.

## Раскрытие уязвимостей

Поскольку ClawHub является размещенным облачным приложением, уязвимости сервиса ClawHub
по умолчанию не раскрываются публично. Они раскрываются публично, когда есть
доказательства реального влияния на пользователей или когда пользователям нужно принять меры.

Примеры реального влияния на пользователей включают подтвержденную эксплуатацию, раскрытие пользовательских
данных или секретов, попадание вредоносного контента к пользователям из-за сбоя платформы
или любую проблему, требующую от пользователей ротации учетных данных, обновления локального ПО или
принятия других защитных мер.

Уязвимости в установленном пользователем ПО раскрываются публично, например
пакеты ClawHub CLI, бинарные файлы, библиотеки или другие релизные артефакты, которые пользователям
нужно обновить локально.
