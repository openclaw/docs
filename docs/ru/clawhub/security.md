# Безопасность

О проблемах безопасности ClawHub можно сообщать через GitHub Security Advisories для
`openclaw/clawhub`.

Используйте GitHub Security Advisories для уязвимостей в самом ClawHub. Хорошие
отчеты advisory ClawHub включают ошибки в:

- веб-сайте, API или CLI ClawHub
- публикации в реестре, загрузках, установках или целостности артефактов
- аутентификации, авторизации или API-токенах
- сканировании, модерации или обработке отчетов

Не используйте advisory ClawHub для уязвимостей в собственном исходном коде стороннего skill или
plugin. Сообщайте о них напрямую издателю или в исходный
репозиторий, указанный в листинге ClawHub.

## Раскрытие уязвимостей

Поскольку ClawHub — размещенное облачное приложение, уязвимости сервиса ClawHub
по умолчанию не раскрываются публично. Они раскрываются публично, когда есть
доказательства реального воздействия на пользователей или когда пользователям нужно принять меры.

Примеры реального воздействия на пользователей включают подтвержденную эксплуатацию, раскрытие пользовательских
данных или секретов, попадание вредоносного контента к пользователям из-за сбоя платформы
или любую проблему, из-за которой пользователям требуется ротировать учетные данные, обновить локальное ПО или
принять другие защитные меры.

Уязвимости в установленном пользователями ПО раскрываются публично, например в
пакетах ClawHub CLI, бинарных файлах, библиотеках или других релизных артефактах, которые пользователям
нужно обновить локально.
