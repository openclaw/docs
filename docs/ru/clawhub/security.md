# Безопасность

О проблемах безопасности ClawHub можно сообщать через рекомендации GitHub по безопасности для
`openclaw/clawhub`.

Используйте рекомендации GitHub по безопасности для уязвимостей непосредственно в ClawHub. Качественные
отчёты о рекомендациях для ClawHub включают ошибки в:

- веб-сайте, API или CLI ClawHub
- публикации в реестре, скачивании, установке или целостности артефактов
- аутентификации, авторизации или токенах API
- сканировании, модерации или обработке жалоб

Не используйте рекомендации ClawHub для сообщения об уязвимостях в исходном коде стороннего Skills или
плагина. Сообщайте о них непосредственно издателю или в репозиторий исходного
кода, указанный на странице ClawHub.

## Раскрытие информации об уязвимостях

Поскольку ClawHub представляет собой размещённое в облаке приложение, уязвимости сервиса ClawHub
по умолчанию не раскрываются публично. Они раскрываются публично при наличии
свидетельств реального воздействия на пользователей или необходимости действий со стороны пользователей.

Примеры реального воздействия на пользователей включают подтверждённую эксплуатацию, раскрытие пользовательских
данных или секретов, попадание вредоносного содержимого к пользователям из-за сбоя платформы
или любую проблему, требующую от пользователей заменить учётные данные, обновить локальное программное обеспечение либо
принять другие защитные меры.

Уязвимости в программном обеспечении, устанавливаемом пользователями, раскрываются публично, например
в пакетах CLI ClawHub, исполняемых файлах, библиотеках или других выпускаемых артефактах, которые пользователям
необходимо обновлять локально.
