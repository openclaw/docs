# Безопасность

О проблемах безопасности ClawHub можно сообщать через GitHub Security Advisories для
`openclaw/clawhub`.

Используйте GitHub Security Advisories для уязвимостей в самом ClawHub. Хорошие
отчеты advisories ClawHub включают ошибки в:

- сайте, API или CLI ClawHub
- публикации в реестре, загрузках, установках или целостности артефактов
- аутентификации, авторизации или токенах API
- сканировании, модерации или обработке отчетов

Не используйте advisories ClawHub для уязвимостей в собственном исходном коде стороннего Skill или
Plugin. Сообщайте о них напрямую издателю или в исходный
репозиторий, указанный в листинге ClawHub.

## Раскрытие уязвимостей

Поскольку ClawHub — размещенное облачное приложение, уязвимости сервиса ClawHub
по умолчанию не раскрываются публично. Они раскрываются публично, когда есть
доказательства реального влияния на пользователей или когда пользователям нужно предпринять действия.

Примеры реального влияния на пользователей включают подтвержденную эксплуатацию, раскрытие пользовательских
данных или секретов, попадание вредоносного содержимого к пользователям из-за сбоя платформы
или любую проблему, из-за которой пользователям нужно сменить учетные данные, обновить локальное программное обеспечение или
предпринять другие защитные действия.

Уязвимости в установленном пользователями программном обеспечении раскрываются публично, например в
пакетах CLI ClawHub, бинарных файлах, библиотеках или других релизных артефактах, которые пользователям
нужно обновлять локально.
