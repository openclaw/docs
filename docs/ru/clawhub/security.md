# Безопасность

О проблемах безопасности ClawHub можно сообщать через GitHub Security Advisories для
`openclaw/clawhub`.

Используйте GitHub Security Advisories для уязвимостей в самом ClawHub. Хорошие
отчеты advisory по ClawHub включают ошибки в:

- сайте, API или CLI ClawHub
- публикации в реестре, скачиваниях, установках или целостности артефактов
- аутентификации, авторизации или токенах API
- сканировании, модерации или обработке отчетов

Не используйте advisory ClawHub для уязвимостей в исходном коде стороннего навыка или
Plugin. Сообщайте о них напрямую издателю или в исходный репозиторий,
указанный в листинге ClawHub.

## Раскрытие уязвимостей

Поскольку ClawHub является размещенным облачным приложением, уязвимости сервиса ClawHub
по умолчанию не раскрываются публично. Они раскрываются публично, когда есть
доказательства реального воздействия на пользователей или когда пользователям необходимо
предпринять действия.

Примеры реального воздействия на пользователей включают подтвержденную эксплуатацию,
раскрытие пользовательских данных или секретов, попадание вредоносного содержимого к
пользователям из-за сбоя платформы или любую проблему, из-за которой пользователям
необходимо ротировать учетные данные, обновлять локальное программное обеспечение или
предпринимать другие защитные меры.

Уязвимости в установленном пользователями программном обеспечении раскрываются публично,
например в пакетах CLI ClawHub, бинарных файлах, библиотеках или других артефактах релиза,
которые пользователям необходимо обновлять локально.
