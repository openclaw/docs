# Безопасность

О проблемах безопасности ClawHub можно сообщать через GitHub Security Advisories для
`openclaw/clawhub`.

Используйте GitHub Security Advisories для уязвимостей в самом ClawHub. Качественные
отчёты об уязвимостях ClawHub включают ошибки в следующих компонентах:

- веб-сайт, API или CLI ClawHub
- публикация в реестре, скачивание, установка или целостность артефактов
- аутентификация, авторизация или токены API
- сканирование, модерация или обработка жалоб

Не используйте отчёты ClawHub для сообщения об уязвимостях в исходном коде стороннего навыка или
плагина. Сообщайте о них непосредственно издателю или в репозиторий исходного
кода, указанный на странице соответствующего объекта в ClawHub.

## Раскрытие информации об уязвимостях

Поскольку ClawHub является облачным приложением, уязвимости сервиса ClawHub
по умолчанию публично не раскрываются. Они раскрываются публично при наличии
доказательств реального воздействия на пользователей или необходимости действий со стороны пользователей.

Примеры реального воздействия на пользователей: подтверждённая эксплуатация, раскрытие пользовательских
данных или секретов, попадание вредоносного содержимого к пользователям из-за сбоя платформы
или любая проблема, требующая от пользователей сменить учётные данные, обновить локальное программное обеспечение или
принять другие защитные меры.

Уязвимости в устанавливаемом пользователями программном обеспечении раскрываются публично — например, в
пакетах CLI ClawHub, двоичных файлах, библиотеках или других артефактах выпуска, которые пользователям
необходимо обновлять локально.
