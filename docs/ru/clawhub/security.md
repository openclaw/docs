# Безопасность

О проблемах безопасности ClawHub можно сообщать через GitHub Security Advisories для
`openclaw/clawhub`.

Используйте GitHub Security Advisories для уязвимостей в самом ClawHub. Хорошие
отчеты advisory для ClawHub включают ошибки в:

- веб-сайте, API или CLI ClawHub
- публикации в реестре, загрузках, установках или целостности артефактов
- аутентификации, авторизации или API-токенах
- сканировании, модерации или обработке отчетов

Не используйте advisory ClawHub для уязвимостей в собственном исходном коде стороннего навыка или
плагина. Сообщайте о них напрямую издателю или в исходный репозиторий,
указанный в карточке ClawHub.

## Раскрытие уязвимостей

Поскольку ClawHub является размещенным облачным приложением, уязвимости сервиса ClawHub
по умолчанию не раскрываются публично. Они раскрываются публично, когда есть
доказательства реального воздействия на пользователей или когда пользователям нужно выполнить действие.

Примеры реального воздействия на пользователей включают подтвержденную эксплуатацию, раскрытие пользовательских
данных или секретов, попадание вредоносного содержимого к пользователям из-за сбоя платформы
или любую проблему, требующую от пользователей сменить учетные данные, обновить локальное ПО или
предпринять другие защитные действия.

Уязвимости в установленном пользователем ПО раскрываются публично, например в
пакетах CLI ClawHub, бинарных файлах, библиотеках или других релизных артефактах, которые пользователям
нужно обновлять локально.
