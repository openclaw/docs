# Формат Skills

## На диске

Skill — это папка.

Обязательно:

- `SKILL.md` (или `skill.md`; также принимается устаревший `skills.md`)

Необязательно:

- любые вспомогательные _текстовые_ файлы (см. «Разрешённые файлы»)
- `.clawhubignore` (шаблоны игнорирования при публикации, устаревший `.clawdhubignore`)
- `.gitignore` (также учитывается)

## Импорт из GitHub

Веб-импортёр GitHub строже локальной публикации и синхронизации. Он обнаруживает только
файлы `SKILL.md` или устаревшие `skills.md` в публичных репозиториях, которые не являются форками и принадлежат
выполнившей вход учётной записи GitHub. Он не импортирует приватные репозитории, форки,
архивированные или отключённые репозитории, а также сторонние публичные репозитории.

Метаданные локальной установки (записываются CLI):

- `<skill>/.clawhub/origin.json` (устаревший `.clawdhub`)

Состояние установки в рабочем каталоге (записывается CLI):

- `<workdir>/.clawhub/lock.json` (устаревший `.clawdhub`)

## `SKILL.md`

- Markdown с необязательным YAML frontmatter.
- При публикации сервер извлекает метаданные из frontmatter.
- `description` используется как краткое описание Skill в интерфейсе и поиске.

Для переносимых Agent Skills значение `name` должно совпадать с именем родительского каталога и содержать
от 1 до 64 строчных букв, цифр или дефисов. ClawHub хранит маршрутизируемый слаг отдельно от
отображаемого имени в каталоге, поэтому существующие имена из других клиентов можно
публиковать без неявного переписывания. В списках каталога длинные имена могут визуально
сокращаться без изменения сохранённого имени.

## Метаданные frontmatter

Метаданные Skill объявляются в YAML frontmatter в начале файла `SKILL.md`. Они сообщают реестру (и системе анализа безопасности), что требуется вашему Skill для работы.

### Базовый frontmatter

```yaml
---
name: my-skill
description: Краткое описание назначения этого Skill.
version: 1.0.0
---
```

### Метаданные среды выполнения (`metadata.openclaw`)

Объявите требования Skill к среде выполнения в `metadata.openclaw` (псевдонимы: `metadata.clawdbot`, `metadata.clawdis`).

```yaml
---
name: my-skill
description: Управление задачами через API Todoist.
metadata:
  openclaw:
    requires:
      env:
        - TODOIST_API_KEY
      bins:
        - curl
    primaryEnv: TODOIST_API_KEY
---
```

Используйте `requires.env` для переменных окружения, которые должны присутствовать перед запуском Skill. Используйте `envVars`, если нужны метаданные для отдельных переменных, включая необязательные переменные с `required: false`.

### Полный справочник полей

| Поле               | Тип        | Описание                                                                                                                                     |
| ------------------ | ---------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| `requires.env`     | `string[]` | Обязательные переменные окружения, необходимые вашему Skill.                                                                                  |
| `requires.bins`    | `string[]` | Исполняемые файлы CLI, каждый из которых должен быть установлен.                                                                              |
| `requires.anyBins` | `string[]` | Исполняемые файлы CLI, из которых должен присутствовать хотя бы один.                                                                         |
| `requires.config`  | `string[]` | Пути к файлам конфигурации, которые читает ваш Skill.                                                                                         |
| `primaryEnv`       | `string`   | Основная переменная окружения с учётными данными для вашего Skill.                                                                            |
| `envVars`          | `array`    | Объявления переменных окружения с `name`, необязательным `required` и необязательным `description`. Для необязательных переменных окружения задайте `required: false`. |
| `always`           | `boolean`  | Если `true`, Skill всегда активен (явная установка не требуется).                                                                            |
| `skillKey`         | `string`   | Переопределяет ключ вызова Skill.                                                                                                             |
| `emoji`            | `string`   | Эмодзи, отображаемый для Skill.                                                                                                               |
| `homepage`         | `string`   | URL домашней страницы или документации Skill.                                                                                                |
| `os`               | `string[]` | Ограничения по ОС (например, `["macos"]`, `["linux"]`).                                                                                       |
| `install`          | `array`    | Спецификации установки зависимостей (см. ниже).                                                                                               |
| `nix`              | `object`   | Спецификация плагина Nix (см. README).                                                                                                        |
| `config`           | `object`   | Спецификация конфигурации Clawdbot (см. README).                                                                                              |

### Спецификации установки

Если для вашего Skill необходимо установить зависимости, объявите их в массиве `install`:

```yaml
metadata:
  openclaw:
    install:
      - kind: brew
        formula: jq
        bins: [jq]
      - kind: node
        package: typescript
        bins: [tsc]
```

Поддерживаемые виды установки: `brew`, `node`, `go`, `uv`.

### Необязательные переменные окружения

Объявите необязательные переменные окружения в `metadata.openclaw.envVars` и задайте `required: false`. Не добавляйте необязательные записи в `requires.env`, поскольку `requires.env` означает, что без них Skill не может работать.

```yaml
metadata:
  openclaw:
    primaryEnv: TODOIST_API_KEY
    envVars:
      - name: TODOIST_API_KEY
        required: true
        description: Токен API Todoist, используемый для аутентифицированных запросов.
      - name: TODOIST_PROJECT_ID
        required: false
        description: Необязательный идентификатор проекта по умолчанию, если пользователь не указал другой.
```

### Почему это важно

Система анализа безопасности ClawHub проверяет, соответствует ли заявленное вашим Skill его фактическому поведению. Если код обращается к `TODOIST_API_KEY`, но frontmatter не объявляет это в `requires.env`, `primaryEnv` или `envVars`, анализ отметит несоответствие метаданных. Точные объявления помогают вашему Skill пройти проверку, а пользователям — понять, что они устанавливают.

### Пример: полный frontmatter

```yaml
---
name: todoist-cli
description: Управление задачами, проектами и метками Todoist из командной строки.
version: 1.2.0
metadata:
  openclaw:
    requires:
      env:
        - TODOIST_API_KEY
      bins:
        - curl
    primaryEnv: TODOIST_API_KEY
    envVars:
      - name: TODOIST_API_KEY
        required: true
        description: Токен API Todoist.
      - name: TODOIST_PROJECT_ID
        required: false
        description: Необязательный идентификатор проекта по умолчанию.
    emoji: "\u2705"
    homepage: https://github.com/example/todoist-cli
---
```

## Разрешённые файлы

При публикации принимаются только «текстовые» файлы.

- Список разрешённых расширений находится в `packages/schema/src/textFiles.ts` (`TEXT_FILE_EXTENSIONS`).
- Файлы скриптов по-прежнему сканируются после загрузки; файлы PowerShell `.ps1`, `.psm1` и `.psd1` принимаются как текстовые.
- Типы содержимого, начинающиеся с `text/`, считаются текстовыми; также действует небольшой список разрешённых форматов (JSON/YAML/TOML/JS/TS/Markdown/SVG).

Ограничения (на стороне сервера):

- Общий размер пакета: 50MB.
- Текст для эмбеддинга включает `SKILL.md` и до ~40 файлов, отличных от `.md` (ограничение применяется по возможности).

## Слаги

- По умолчанию формируются из имени папки.
- Области видимости пакетов должны в точности совпадать с идентификатором издателя ClawHub. Идентификаторы издателей могут содержать строчные буквы, цифры, дефисы, точки и символы подчёркивания; они должны начинаться и заканчиваться строчной буквой или цифрой.
- Слаги пакетов должны быть записаны строчными буквами и быть безопасными для npm, например `@example.tools/demo-plugin` или `demo-plugin`.

## Управление версиями и теги

- При каждой публикации создаётся новая версия (semver).
- Теги — это строковые указатели на версию; обычно используется `latest`.

## Лицензия

- Все Skills, опубликованные в ClawHub, распространяются по лицензии `MIT-0`.
- Любой может использовать, изменять и распространять опубликованные Skills, в том числе в коммерческих целях.
- Указание авторства не требуется.
- Не добавляйте противоречащие условия лицензирования в `SKILL.md`; ClawHub не поддерживает переопределение лицензии для отдельных Skills.

## Платные Skills

- ClawHub не поддерживает платные Skills, отдельные цены для Skills, платный доступ или распределение доходов.
- Не добавляйте метаданные о цене в `SKILL.md`; они не входят в формат Skill и не сделают опубликованный Skill платным.
- Если ваш Skill интегрируется с платным сторонним сервисом, ясно укажите внешнюю стоимость и требования к учётной записи в инструкциях Skill и объявлениях переменных окружения (`requires.env` для обязательных переменных или `envVars` с `required: false` для необязательных переменных).
