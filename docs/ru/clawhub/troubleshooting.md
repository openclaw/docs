# Устранение неполадок

## `clawhub login` открывает браузер, но не завершается

CLI запускает кратковременный локальный callback-сервер во время входа через браузер.

- Убедитесь, что ваш браузер может открыть `http://127.0.0.1:<port>/callback`.
- Проверьте локальный файрвол, VPN и правила прокси, если callback так и не приходит.
- В headless-окружениях создайте API-токен в веб-интерфейсе ClawHub и выполните:

```bash
clawhub login --token clh_...
```

## `whoami` или `publish` возвращает `Unauthorized` (401)

- Войдите снова с помощью `clawhub login`.
- Если вы используете пользовательский путь конфигурации, убедитесь, что `CLAWHUB_CONFIG_PATH` указывает на
  файл, содержащий ваш текущий токен.
- Если вы используете API-токен, убедитесь, что он не был отозван в веб-интерфейсе.

## Поиск или установка возвращает `Rate limit exceeded` (429)

Прочитайте информацию о повторной попытке в ответе:

- `Retry-After`: сколько секунд ждать перед повторной попыткой.
- `RateLimit-Limit`: лимит, примененный к этому запросу.
- `RateLimit-Remaining`: ваш точный оставшийся бюджет, когда заголовок присутствует. При `429` он равен `0`.
- `RateLimit-Reset` или `X-RateLimit-Reset`: время сброса.

Если много пользователей используют один исходящий IP-адрес, анонимные IP-лимиты могут быть достигнуты, даже если каждый
человек отправляет всего несколько запросов. Войдите в систему, где это возможно, и повторите попытку после
указанной задержки.

## Поиск или установка не работает за прокси

CLI учитывает стандартные переменные прокси:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

Поддерживаемые имена включают `HTTPS_PROXY`, `HTTP_PROXY`, `https_proxy` и
`http_proxy`.

## Skill не появляется в поиске

- Проверьте точный slug или страницу владельца, если вы их знаете.
- Убедитесь, что релиз публичный и не удерживается проверкой или модерацией.
- Если вы владеете этим Skill, войдите и проверьте его:

```bash
clawhub inspect @openclaw/demo
```

Диагностика, видимая владельцу, может объяснить состояние проверки, upload-gate или модерации.

## Публикация не удается из-за отсутствия обязательных метаданных

Для Skills проверьте frontmatter в `SKILL.md`. Обязательные переменные окружения и
инструменты должны быть объявлены, чтобы пользователи и сканеры могли понять пакет.

Для Plugins проверьте метаданные совместимости в `package.json`. Публикации code-plugin
требуют поля совместимости OpenClaw, такие как `openclaw.compat.pluginApi` и
`openclaw.build.openclawVersion`.

Сначала просмотрите payload публикации:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## Публикация не удается из-за ошибки владельца GitHub или источника

ClawHub использует идентичность GitHub и атрибуцию источника, чтобы связывать пакеты с их
издателями.

- Убедитесь, что вы вошли с учетной записью GitHub, которая владеет пакетом или может его публиковать.
- Проверьте, что URL источника публичен или доступен ClawHub.
- Для источников GitHub используйте `owner/repo`, `owner/repo@ref` или полный URL GitHub.

## Публикация не удается, потому что namespace занят или зарезервирован

Если публикация не удается, потому что handle владельца, namespace организации, scope пакета, slug Skill
или имя пакета уже заняты или зарезервированы, сначала убедитесь, что вы
публикуете от имени владельца, соответствующего namespace. Для пакетов Plugin
scoped-имена, такие как `@example-org/example-plugin`, должны публиковаться от имени
соответствующего владельца `example-org`.

Если вы считаете, что ваша организация, проект или бренд является законным владельцем namespace, но
вы не можете управлять текущим владельцем ClawHub, откройте
[issue о заявке на организацию / namespace](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
с публичным, нечувствительным доказательством. См.
[заявки на организации и namespace](/clawhub/namespace-claims), чтобы узнать, какие доказательства нужны и что
не следует включать в публичные issue.

## `sync` сообщает, что Skills не найдены

`sync` ищет папки, содержащие `SKILL.md` или `skill.md`.

Укажите корневые папки, которые нужно просканировать:

```bash
clawhub sync --root /path/to/skills
```

Сначала выполните предварительный просмотр, если вы не уверены, что будет опубликовано:

```bash
clawhub sync --all --dry-run --no-input
```

## `update` отказывается работать из-за локальных изменений

Локальные файлы не соответствуют ни одной версии, известной ClawHub. Выберите один вариант:

- Сохранить локальные правки и пропустить обновление.
- Перезаписать опубликованной версией:

```bash
clawhub update @openclaw/demo --force
```

- Опубликовать отредактированную копию как новый slug или fork.

## Установка Plugin завершается ошибкой в OpenClaw

- Используйте явный источник ClawHub:

```bash
openclaw plugins install clawhub:<package>
```

- Проверьте страницу сведений о пакете для статуса проверки и метаданных совместимости.
- Убедитесь, что ваша версия OpenClaw удовлетворяет заявленному диапазону
  совместимости пакета.
- Если пакет скрыт, удерживается или заблокирован, его может быть невозможно установить, пока
  владелец не решит проблему.

## Публичные API-запросы завершаются ошибкой

- Учитывайте заголовки повторной попытки `429` и кэшируйте публичные ответы списков/поиска.
- Перенаправляйте пользователей к канонической записи ClawHub.
- Не зеркалируйте скрытый, приватный, удерживаемый или заблокированный модерацией контент за пределами
  публичной поверхности API.

См. [HTTP API](/clawhub/http-api) для сведений об эндпоинтах.
