# Устранение неполадок

## `clawhub login` открывает браузер, но вход не завершается

CLI запускает кратковременный локальный сервер обратного вызова во время входа через браузер.

- Убедитесь, что ваш браузер может открыть `http://127.0.0.1:<port>/callback`.
- Проверьте правила локального файрвола, VPN и прокси, если обратный вызов не приходит.
- В средах без графического интерфейса создайте API-токен в веб-интерфейсе ClawHub и выполните:

```bash
clawhub login --token clh_...
```

## `whoami` или `publish` возвращает `Unauthorized` (401)

- Войдите снова с помощью `clawhub login`.
- Если вы используете пользовательский путь к конфигурации, убедитесь, что `CLAWHUB_CONFIG_PATH` указывает на
  файл с вашим текущим токеном.
- Если вы используете API-токен, убедитесь, что он не был отозван в веб-интерфейсе.

## Поиск или установка возвращает `Rate limit exceeded` (429)

Прочитайте информацию о повторной попытке в ответе:

- `Retry-After`: количество секунд ожидания перед повторной попыткой.
- `RateLimit-Limit`: лимит, примененный к этому запросу.
- `RateLimit-Remaining`: ваш точный оставшийся лимит, если заголовок присутствует. При `429` он равен `0`.
- `RateLimit-Reset` или `X-RateLimit-Reset`: время сброса.

Если много пользователей используют один исходящий IP-адрес, анонимные IP-лимиты могут быть исчерпаны, даже когда каждый
человек отправляет всего несколько запросов. Войдите в систему, где это возможно, и повторите попытку после
указанной задержки.

## Поиск или установка завершается ошибкой за прокси

CLI учитывает стандартные переменные прокси:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

Поддерживаемые имена включают `HTTPS_PROXY`, `HTTP_PROXY`, `https_proxy` и
`http_proxy`.

## Skill не появляется в поиске

- Проверьте точный слаг или страницу владельца, если они вам известны.
- Убедитесь, что релиз публичный и не удерживается сканированием или модерацией.
- Если Skill принадлежит вам, войдите и проверьте его:

```bash
clawhub inspect @openclaw/demo
```

Диагностика, видимая владельцу, может объяснить состояние сканирования, блокировки загрузки или модерации.

## Публикация завершается ошибкой из-за отсутствующих обязательных метаданных

Для Skills проверьте frontmatter в `SKILL.md`. Необходимые переменные окружения и
инструменты должны быть объявлены, чтобы пользователи и сканеры могли понять пакет.

Для плагинов проверьте метаданные совместимости в `package.json`. Публикации code-plugin
требуют поля совместимости OpenClaw, такие как `openclaw.compat.pluginApi` и
`openclaw.build.openclawVersion`.

Сначала предварительно просмотрите полезную нагрузку публикации:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## Публикация завершается ошибкой владельца GitHub или источника

ClawHub использует идентичность GitHub и атрибуцию источника, чтобы связать пакеты с их
издателями.

- Убедитесь, что вы вошли с учетной записью GitHub, которая владеет пакетом или может его публиковать.
- Проверьте, что URL источника публичный или доступен ClawHub.
- Для источников GitHub используйте `owner/repo`, `owner/repo@ref` или полный URL GitHub.

## Публикация завершается ошибкой, потому что namespace занят или зарезервирован

Если публикация завершается ошибкой из-за того, что идентификатор владельца, namespace организации, область пакета, слаг Skill
или имя пакета уже заняты или зарезервированы, сначала убедитесь, что вы
публикуете от имени владельца, соответствующего namespace. Для пакетов плагинов
имена с областью, такие как `@example-org/example-plugin`, должны публиковаться от имени
соответствующего владельца `example-org`.

Если вы считаете, что ваша организация, проект или бренд является законным владельцем namespace, но
вы не можете управлять текущим владельцем ClawHub, откройте
[заявку на организацию / namespace](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
с публичными, неконфиденциальными доказательствами. См.
[Заявки на организации и namespace](/clawhub/namespace-claims), чтобы узнать требования к доказательствам и что
не следует указывать в публичных issue.

## `sync` сообщает, что Skills не найдены

`sync` ищет папки, содержащие `SKILL.md` или `skill.md`.

Укажите корни, которые нужно просканировать:

```bash
clawhub sync --root /path/to/skills
```

Если вы не уверены, что будет опубликовано, сначала выполните предварительный просмотр:

```bash
clawhub sync --all --dry-run --no-input
```

## `update` отказывается выполняться из-за локальных изменений

Локальные файлы не соответствуют ни одной версии, известной ClawHub. Выберите один вариант:

- Сохранить локальные правки и пропустить обновление.
- Перезаписать опубликованной версией:

```bash
clawhub update @openclaw/demo --force
```

- Опубликовать отредактированную копию с новым слагом или как форк.

## Установка плагина в OpenClaw завершается ошибкой

- Используйте явный источник ClawHub:

```bash
openclaw plugins install clawhub:<package>
```

- Проверьте страницу сведений о пакете на наличие статуса сканирования и метаданных совместимости.
- Убедитесь, что ваша версия OpenClaw удовлетворяет заявленному пакетом
  диапазону совместимости.
- Если пакет скрыт, удерживается или заблокирован, он может быть недоступен для установки, пока
  владелец не устранит проблему.

## Публичные запросы API завершаются ошибкой

- Соблюдайте заголовки повторной попытки `429` и кэшируйте публичные ответы списков и поиска.
- Направляйте пользователей обратно к канонической странице ClawHub.
- Не зеркалируйте скрытый, приватный, удерживаемый или заблокированный модерацией контент за пределами
  публичной поверхности API.

См. [HTTP API](/clawhub/http-api) для подробностей об endpoint.
