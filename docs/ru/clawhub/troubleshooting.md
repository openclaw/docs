# Устранение неполадок

## `clawhub login` открывает браузер, но не завершается

CLI запускает кратковременный локальный сервер обратного вызова во время входа через браузер.

- Убедитесь, что браузер может открыть `http://127.0.0.1:<port>/callback`.
- Проверьте локальный firewall, VPN и правила proxy, если обратный вызов не приходит.
- В headless-окружениях создайте API-токен в веб-интерфейсе ClawHub и выполните:

```bash
clawhub login --token clh_...
```

## `whoami` или `publish` возвращает `Unauthorized` (401)

- Войдите снова с помощью `clawhub login`.
- Если вы используете собственный путь к конфигурации, убедитесь, что `CLAWHUB_CONFIG_PATH` указывает на
  файл, содержащий ваш текущий токен.
- Если вы используете API-токен, убедитесь, что он не был отозван в веб-интерфейсе.

## Поиск или установка возвращает `Rate limit exceeded` (429)

Прочитайте информацию о повторной попытке в ответе:

- `Retry-After`: сколько секунд ждать перед повторной попыткой.
- `RateLimit-Limit`: лимит, примененный к этому запросу.
- `RateLimit-Remaining`: ваш точный оставшийся бюджет, если заголовок присутствует. При `429` он равен `0`.
- `RateLimit-Reset` или `X-RateLimit-Reset`: время сброса.

Если много пользователей используют один исходящий IP-адрес, анонимные IP-лимиты могут быть исчерпаны, даже когда каждый
человек отправляет всего несколько запросов. По возможности войдите в систему и повторите попытку после
указанной задержки.

## Поиск или установка не работает за proxy

CLI учитывает стандартные переменные proxy:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

Поддерживаемые имена включают `HTTPS_PROXY`, `HTTP_PROXY`, `https_proxy` и
`http_proxy`.

## Skill не появляется в поиске

- Проверьте точный slug или страницу владельца, если она вам известна.
- Убедитесь, что релиз публичный и не задержан проверкой или модерацией.
- Если Skill принадлежит вам, войдите в систему и проверьте его:

```bash
clawhub inspect @openclaw/demo
```

Диагностика, видимая владельцу, может объяснить состояние проверки, блокировки загрузки или модерации.

## Публикация завершается ошибкой, потому что отсутствуют обязательные метаданные

Для Skills проверьте frontmatter в `SKILL.md`. Обязательные переменные окружения и
инструменты должны быть объявлены, чтобы пользователи и сканеры могли понять пакет.

Для plugins проверьте метаданные совместимости в `package.json`. Для публикаций
code-plugin нужны поля совместимости OpenClaw, такие как `openclaw.compat.pluginApi` и
`openclaw.build.openclawVersion`.

Сначала предварительно просмотрите payload публикации:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## Публикация завершается ошибкой владельца GitHub или источника

ClawHub использует идентичность GitHub и атрибуцию источника, чтобы связывать пакеты с их
издателями.

- Убедитесь, что вы вошли под учетной записью GitHub, которая владеет пакетом или может его публиковать.
- Проверьте, что URL источника публичный или доступен для ClawHub.
- Для источников GitHub используйте `owner/repo`, `owner/repo@ref` или полный URL GitHub.

## Публикация завершается ошибкой, потому что namespace уже занят или зарезервирован

Если публикация завершается ошибкой, потому что handle владельца, namespace организации, scope пакета, slug Skill
или имя пакета уже заняты или зарезервированы, сначала убедитесь, что вы
публикуете от владельца, соответствующего namespace. Для пакетов plugin
scoped-имена, такие как `@example-org/example-plugin`, должны публиковаться от
соответствующего владельца `example-org`.

Если вы считаете, что ваша организация, проект или бренд является законным владельцем namespace, но
вы не можете управлять текущим владельцем ClawHub, откройте
[issue Org / Namespace Claim](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
с публичными, нечувствительными доказательствами. См.
[Заявки на организации и namespace](/ru/clawhub/namespace-claims) для рекомендаций по доказательствам и информации о том,
что не следует включать в публичные issues.

## `sync` сообщает, что Skills не найдены

`sync` ищет папки, содержащие `SKILL.md` или `skill.md`.

Укажите корни, которые нужно просканировать:

```bash
clawhub sync --root /path/to/skills
```

Если вы не уверены, что будет опубликовано, сначала выполните предварительный просмотр:

```bash
clawhub sync --all --dry-run --no-input
```

## `update` отказывается выполняться из-за локальных изменений

Локальные файлы не соответствуют ни одной версии, известной ClawHub. Выберите один вариант:

- Сохранить локальные правки и пропустить обновление.
- Перезаписать опубликованной версией:

```bash
clawhub update @openclaw/demo --force
```

- Опубликовать вашу отредактированную копию с новым slug или как fork.

## Установка plugin завершается ошибкой в OpenClaw

- Используйте явный источник ClawHub:

```bash
openclaw plugins install clawhub:<package>
```

- Проверьте страницу сведений о пакете на наличие статуса проверки и метаданных совместимости.
- Убедитесь, что ваша версия OpenClaw удовлетворяет объявленному диапазону
  совместимости пакета.
- Если пакет скрыт, задержан или заблокирован, он может быть недоступен для установки, пока
  владелец не решит проблему.

## Запросы к публичному API завершаются ошибкой

- Учитывайте заголовки повторной попытки `429` и кэшируйте ответы публичных списков и поиска.
- Направляйте пользователей обратно к канонической странице ClawHub.
- Не зеркалируйте скрытый, приватный, задержанный или заблокированный модерацией контент за пределами
  поверхности публичного API.

См. [HTTP API](/ru/clawhub/http-api) для сведений об endpoint.
