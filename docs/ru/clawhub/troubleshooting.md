# Устранение неполадок

## `clawhub login` открывает браузер, но не завершается

CLI запускает кратковременный локальный сервер обратного вызова во время входа через браузер.

- Убедитесь, что браузер может открыть `http://127.0.0.1:<port>/callback`.
- Проверьте локальный брандмауэр, VPN и правила прокси, если обратный вызов не приходит.
- В средах без графического интерфейса создайте API-токен в веб-интерфейсе ClawHub и выполните:

```bash
clawhub login --token clh_...
```

## `whoami` или `publish` возвращает `Unauthorized` (401)

- Войдите снова с помощью `clawhub login`.
- Если вы используете пользовательский путь к конфигурации, убедитесь, что `CLAWHUB_CONFIG_PATH` указывает на
  файл, содержащий ваш текущий токен.
- Если вы используете API-токен, убедитесь, что он не был отозван в веб-интерфейсе.

## Поиск или установка возвращает `Rate limit exceeded` (429)

Прочитайте информацию о повторной попытке в ответе:

- `Retry-After`: сколько секунд подождать перед повторной попыткой.
- `RateLimit-Limit`: лимит, примененный к этому запросу.
- `RateLimit-Remaining`: ваш точный оставшийся лимит, если заголовок присутствует. При `429` он равен `0`.
- `RateLimit-Reset` или `X-RateLimit-Reset`: время сброса.

Если много пользователей используют один исходящий IP-адрес, анонимные IP-лимиты могут быть достигнуты, даже если каждый
человек отправляет всего несколько запросов. По возможности войдите в систему и повторите попытку после
указанной задержки.

## Поиск или установка завершается ошибкой за прокси

CLI учитывает стандартные переменные прокси:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

Поддерживаемые имена включают `HTTPS_PROXY`, `HTTP_PROXY`, `https_proxy` и
`http_proxy`.

## Навык не появляется в поиске

- Проверьте точный slug или страницу владельца, если вы их знаете.
- Убедитесь, что релиз является публичным и не задержан сканированием или модерацией.
- Если вы владеете навыком, войдите и проверьте его:

```bash
clawhub inspect @openclaw/demo
```

Диагностика, видимая владельцу, может объяснить состояние сканирования, шлюза загрузки или модерации.

## Публикация завершается ошибкой из-за отсутствия обязательных метаданных

Для Skills проверьте frontmatter в `SKILL.md`. Обязательные переменные окружения и
инструменты должны быть объявлены, чтобы пользователи и сканеры могли понять пакет.

Для plugins проверьте метаданные совместимости в `package.json`. Для публикации code-plugin
нужны поля совместимости OpenClaw, такие как `openclaw.compat.pluginApi` и
`openclaw.build.openclawVersion`.

Сначала просмотрите полезную нагрузку публикации:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## Публикация завершается ошибкой владельца GitHub или источника

ClawHub использует идентичность GitHub и атрибуцию источника, чтобы связывать пакеты с их
издателями.

- Убедитесь, что вы вошли с учетной записью GitHub, которая владеет пакетом или может его публиковать.
- Проверьте, что URL источника является публичным или доступен ClawHub.
- Для источников GitHub используйте `owner/repo`, `owner/repo@ref` или полный URL GitHub.

## Публикация завершается ошибкой, потому что пространство имен занято или зарезервировано

Если публикация завершается ошибкой, потому что дескриптор владельца, пространство имен организации, область пакета, slug навыка
или имя пакета уже заняты или зарезервированы, сначала убедитесь, что вы
публикуете от владельца, соответствующего пространству имен. Для пакетов plugin
имена с областью, такие как `@example-org/example-plugin`, должны публиковаться от имени
соответствующего владельца `example-org`.

Если вы считаете, что ваша организация, проект или бренд является законным владельцем пространства имен, но
вы не можете управлять текущим владельцем ClawHub, откройте
[issue с заявкой на организацию / пространство имен](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
с публичными, неконфиденциальными доказательствами. См.
[Заявки на организации и пространства имен](/clawhub/namespace-claims) для рекомендаций по доказательствам и о том, что
не следует включать в публичные issue.

## `sync` сообщает, что навыки не найдены

`sync` ищет папки, содержащие `SKILL.md` или `skill.md`.

Укажите корневые каталоги, которые хотите просканировать:

```bash
clawhub sync --root /path/to/skills
```

Если вы не уверены, что будет опубликовано, сначала выполните предварительный просмотр:

```bash
clawhub sync --all --dry-run --no-input
```

## `update` отказывается выполняться из-за локальных изменений

Локальные файлы не соответствуют ни одной версии, известной ClawHub. Выберите один вариант:

- Сохранить локальные правки и пропустить обновление.
- Перезаписать опубликованной версией:

```bash
clawhub update @openclaw/demo --force
```

- Опубликовать отредактированную копию как новый slug или fork.

## Установка plugin завершается ошибкой в OpenClaw

- Используйте явный источник ClawHub:

```bash
openclaw plugins install clawhub:<package>
```

- Проверьте страницу сведений о пакете на статус сканирования и метаданные совместимости.
- Убедитесь, что ваша версия OpenClaw соответствует объявленному пакетом
  диапазону совместимости.
- Если пакет скрыт, задержан или заблокирован, его установка может быть невозможна, пока
  владелец не решит проблему.

## Запросы Public API завершаются ошибкой

- Учитывайте заголовки повторной попытки `429` и кэшируйте ответы публичного списка/поиска.
- Направляйте пользователей обратно к каноническому листингу ClawHub.
- Не зеркалируйте скрытый, приватный, задержанный или заблокированный модерацией контент за пределами
  поверхности Public API.

См. [HTTP API](/clawhub/http-api) для сведений об endpoint.
