# Устранение неполадок

## `clawhub login` открывает браузер, но не завершается

Во время входа через браузер CLI запускает кратковременно работающий локальный сервер обратного вызова.

- Убедитесь, что браузер может подключиться к `http://127.0.0.1:<port>/callback`.
- Если обратный вызов не поступает, проверьте правила локального брандмауэра, VPN и прокси-сервера.
- В средах без графического интерфейса создайте API-токен в веб-интерфейсе ClawHub и выполните:

```bash
clawhub login --token clh_...
```

## `whoami` или `publish` возвращает `Unauthorized` (401)

- Снова войдите с помощью `clawhub login`.
- Если используется нестандартный путь к конфигурации, убедитесь, что `CLAWHUB_CONFIG_PATH` указывает на
  файл с текущим токеном.
- Если используется API-токен, убедитесь, что он не был отозван в веб-интерфейсе.

## Поиск или установка возвращает `Rate limit exceeded` (429)

Ознакомьтесь с информацией о повторной попытке в ответе:

- `Retry-After`: количество секунд до повторной попытки.
- `RateLimit-Limit`: ограничение, применённое к этому запросу.
- `RateLimit-Remaining`: точный оставшийся лимит, если заголовок присутствует. Для `429` он равен `0`.
- `RateLimit-Reset` или `X-RateLimit-Reset`: время сброса.

Если много пользователей используют один исходящий IP-адрес, анонимные ограничения по IP могут быть достигнуты, даже когда каждый
пользователь отправляет лишь несколько запросов. По возможности войдите в систему и повторите попытку после
указанной задержки.

## Поиск или установка не работает через прокси-сервер

CLI учитывает стандартные переменные прокси-сервера:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

Поддерживаются, в частности, имена `HTTPS_PROXY`, `HTTP_PROXY`, `https_proxy` и
`http_proxy`.

## Skill не отображается в результатах поиска

- Если известен точный слаг или страница владельца, проверьте их.
- Убедитесь, что релиз общедоступен и не задержан проверкой или модерацией.
- Если Skill принадлежит вам, войдите в систему и проверьте его:

```bash
clawhub inspect @openclaw/demo
```

Диагностические сведения, доступные владельцу, могут объяснить состояние проверки, ограничения загрузки или модерации.

## Публикация завершается ошибкой из-за отсутствия обязательных метаданных

Для Skills проверьте frontmatter в `SKILL.md`. Необходимо объявить требуемые переменные среды и
инструменты, чтобы пользователи и сканеры могли понять содержимое пакета.

Для плагинов проверьте метаданные совместимости в `package.json`. При публикации плагинов с кодом
необходимы поля совместимости с OpenClaw, такие как `openclaw.compat.pluginApi` и
`openclaw.build.openclawVersion`.

Сначала просмотрите данные публикации:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## Публикация завершается ошибкой владельца или источника GitHub

ClawHub использует идентификационные данные GitHub и сведения об источнике, чтобы связывать пакеты с их
издателями.

- Убедитесь, что выполнен вход в учётную запись GitHub, которая владеет пакетом или может его опубликовать.
- Убедитесь, что URL-адрес источника общедоступен или доступен для ClawHub.
- Для источников GitHub используйте `owner/repo`, `owner/repo@ref` или полный URL-адрес GitHub.

## Публикация завершается ошибкой, поскольку пространство имён занято или зарезервировано

Если публикация завершается ошибкой из-за того, что идентификатор владельца, пространство имён организации, область пакета, слаг Skill
или имя пакета уже заняты или зарезервированы, сначала убедитесь, что публикация выполняется
от имени владельца, соответствующего пространству имён. Для пакетов плагинов
имена с областью, такие как `@example-org/example-plugin`, должны публиковаться от имени
соответствующего владельца `example-org`.

Если вы считаете свою организацию, проект или бренд законным владельцем пространства имён, но
не можете управлять текущим владельцем в ClawHub, создайте
[обращение о правах на организацию или пространство имён](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
с общедоступными неконфиденциальными доказательствами. Рекомендации по доказательствам и сведениям,
которые не следует публиковать в общедоступных обращениях, приведены в разделе
[Права на организации и пространства имён](/clawhub/namespace-claims).

## `sync` сообщает, что Skills не найдены

`sync` ищет папки, содержащие `SKILL.md` или `skill.md`.

Укажите корневые каталоги, которые необходимо просканировать:

```bash
clawhub sync --root /path/to/skills
```

Если неизвестно, что будет опубликовано, сначала выполните предварительный просмотр:

```bash
clawhub sync --all --dry-run --no-input
```

## `update` отказывается продолжать из-за локальных изменений

Локальные файлы не соответствуют ни одной версии, известной ClawHub. Выберите один из вариантов:

- Сохранить локальные изменения и пропустить обновление.
- Перезаписать файлы опубликованной версией:

```bash
clawhub update @openclaw/demo --force
```

- Опубликовать изменённую копию с новым слагом или как форк.

## Не удаётся установить плагин в OpenClaw

- Используйте явно указанный источник ClawHub:

```bash
openclaw plugins install clawhub:<package>
```

- Проверьте на странице сведений о пакете состояние проверки и метаданные совместимости.
- Убедитесь, что версия OpenClaw соответствует заявленному пакетом
  диапазону совместимости.
- Если пакет скрыт, задержан или заблокирован, его установка может быть невозможна, пока
  владелец не устранит проблему.

## Запросы к общедоступному API завершаются ошибкой

- Учитывайте заголовки повторной попытки `429` и кэшируйте ответы общедоступных запросов списка и поиска.
- Направляйте пользователей на каноническую страницу пакета в ClawHub.
- Не зеркалируйте скрытый, закрытый, задержанный или заблокированный модерацией контент за пределами
  общедоступной поверхности API.

Подробные сведения о конечных точках приведены в разделе [HTTP API](/clawhub/http-api).
