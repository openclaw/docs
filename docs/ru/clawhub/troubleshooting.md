# Устранение неполадок

## `clawhub login` открывает браузер, но не завершается

Во время входа через браузер CLI запускает локальный сервер обратного вызова с коротким временем жизни.

- Убедитесь, что браузер может обратиться к `http://127.0.0.1:<port>/callback`.
- Если обратный вызов не поступает, проверьте правила локального брандмауэра, VPN и прокси-сервера.
- В средах без графического интерфейса создайте API-токен в веб-интерфейсе ClawHub и выполните:

```bash
clawhub login --token clh_...
```

## `whoami` или `publish` возвращает `Unauthorized` (401)

- Снова войдите с помощью `clawhub login`.
- Если вы используете пользовательский путь к конфигурации, убедитесь, что `CLAWHUB_CONFIG_PATH` указывает на файл с вашим текущим токеном.
- Если вы используете API-токен, убедитесь, что он не был отозван в веб-интерфейсе.

## Поиск или установка возвращает `Rate limit exceeded` (429)

Ознакомьтесь с информацией о повторной попытке в ответе:

- `Retry-After`: количество секунд до повторной попытки.
- `RateLimit-Limit`: лимит, применённый к этому запросу.
- `RateLimit-Remaining`: точный остаток лимита, если заголовок присутствует. При `429` он равен `0`.
- `RateLimit-Reset` или `X-RateLimit-Reset`: время сброса лимита.

Если много пользователей используют один исходящий IP-адрес, анонимные лимиты для IP могут быть исчерпаны, даже если каждый пользователь отправляет лишь несколько запросов. По возможности войдите в систему и повторите попытку после указанной задержки.

## Поиск или установка не работает через прокси-сервер

CLI учитывает стандартные переменные прокси-сервера:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

Поддерживаются имена `HTTPS_PROXY`, `HTTP_PROXY`, `https_proxy` и `http_proxy`.

## Навык не отображается в результатах поиска

- Проверьте точный идентификатор или страницу владельца, если они вам известны.
- Убедитесь, что выпуск общедоступен и не задержан проверкой или модерацией.
- Если навык принадлежит вам, войдите в систему и проверьте его:

```bash
clawhub inspect @openclaw/demo
```

Диагностические сведения, доступные владельцу, могут объяснить состояние проверки, блокировки загрузки или модерации.

## Публикация не выполняется из-за отсутствия обязательных метаданных

Для навыков проверьте метаданные frontmatter в `SKILL.md`. Необходимые переменные среды и инструменты следует объявить, чтобы пользователи и сканеры могли понять содержимое пакета.

Для плагинов проверьте метаданные совместимости в `package.json`. Для публикации плагинов с кодом требуются поля совместимости с OpenClaw, например `openclaw.compat.pluginApi` и `openclaw.build.openclawVersion`.

Сначала просмотрите данные публикации:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## Публикация не выполняется из-за ошибки владельца или источника GitHub

ClawHub использует идентификационные данные GitHub и сведения об источнике, чтобы связать пакеты с их издателями.

- Убедитесь, что вы вошли с учётной записью GitHub, которая владеет пакетом или может его публиковать.
- Убедитесь, что URL-адрес источника общедоступен или доступен для ClawHub.
- Для источников GitHub используйте `owner/repo`, `owner/repo@ref` или полный URL-адрес GitHub.

## Публикация не выполняется, поскольку пространство имён занято или зарезервировано

Если публикация не выполняется, поскольку идентификатор владельца, пространство имён организации, область пакета, идентификатор навыка или имя пакета уже заняты либо зарезервированы, сначала убедитесь, что публикация выполняется от имени владельца, соответствующего пространству имён. Для пакетов плагинов имена с областью, например `@example-org/example-plugin`, должны публиковаться от имени соответствующего владельца `example-org`.

Если вы считаете, что ваша организация, проект или бренд являются законными владельцами пространства имён, но не можете управлять текущим владельцем в ClawHub, создайте [запрос на получение пространства имён организации](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml), приложив общедоступные доказательства, не содержащие конфиденциальных данных. Рекомендации по доказательствам и сведения о том, что не следует публиковать в общедоступных запросах, см. в разделе [Получение пространств имён организаций](/clawhub/namespace-claims).

## `sync` сообщает, что навыки не найдены

`sync` ищет папки, содержащие `SKILL.md` или `skill.md`.

Укажите корневые каталоги, которые требуется сканировать:

```bash
clawhub sync --root /path/to/skills
```

Если вы не уверены, что будет опубликовано, сначала выполните предварительный просмотр:

```bash
clawhub sync --all --dry-run --no-input
```

## `update` отказывается выполняться из-за локальных изменений

Локальные файлы не соответствуют ни одной версии, известной ClawHub. Выберите один из вариантов:

- Сохранить локальные изменения и пропустить обновление.
- Перезаписать файлы опубликованной версией:

```bash
clawhub update @openclaw/demo --force
```

- Опубликовать изменённую копию под новым идентификатором или как ответвление.

## Установка плагина в OpenClaw завершается ошибкой

- Явно укажите источник ClawHub:

```bash
openclaw plugins install clawhub:<package>
```

- Проверьте на странице сведений о пакете состояние проверки и метаданные совместимости.
- Убедитесь, что ваша версия OpenClaw соответствует заявленному пакетом диапазону совместимости.
- Если пакет скрыт, задержан или заблокирован, его установка может быть невозможна, пока владелец не устранит проблему.

## Запросы к общедоступному API завершаются ошибкой

- Учитывайте заголовки повторной попытки при `429` и кэшируйте ответы общедоступных списков и поиска.
- Направляйте пользователей на каноническую страницу пакета в ClawHub.
- Не копируйте скрытое, приватное, задержанное или заблокированное модерацией содержимое за пределы общедоступного API.

Подробные сведения о конечных точках см. в разделе [HTTP API](/clawhub/http-api).
