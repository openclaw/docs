# HTTP API

Базовый URL: `https://clawhub.ai` (по умолчанию).

Все пути v1 находятся в `/api/v1/...`.
Устаревшие `/api/...` и `/api/cli/...` сохранены для совместимости (см. `DEPRECATIONS.md`).
OpenAPI: `/api/v1/openapi.json`.

## Повторное использование общедоступного каталога

Сторонние каталоги могут использовать общедоступные конечные точки чтения для вывода списка или поиска Skills ClawHub. Кэшируйте результаты, соблюдайте `429`/`Retry-After`, направляйте пользователей на каноническую страницу ClawHub (`https://clawhub.ai/<owner>/skills/<slug>`) и не создавайте впечатление, что ClawHub одобряет сторонний сайт. Не пытайтесь копировать скрытое, приватное или заблокированное модерацией содержимое за пределами общедоступной поверхности API.

Веб-сокращения на основе слагов разрешаются во всех семействах реестра, но клиенты API должны использовать
канонические URL-адреса, возвращаемые конечными точками чтения, вместо самостоятельного восстановления
приоритета маршрутов.

## Ограничения частоты запросов

Модель применения:

- Анонимные запросы: ограничения применяются для каждого IP-адреса.
- Аутентифицированные запросы (действительный Bearer-токен): ограничения применяются для пользовательского лимита.
- Если токен отсутствует или недействителен, применяются ограничения по IP-адресу.
- Аутентифицированные конечные точки записи не должны возвращать только `Unauthorized`, когда
  серверу известна причина. Для отсутствующих, недействительных или отозванных токенов, а также
  удалённых, заблокированных или отключённых учётных записей следует возвращать пригодный для действий текст, чтобы клиенты
  CLI могли сообщить пользователям, что именно заблокировало запрос.

- Чтение: 3000/мин на IP-адрес, 12000/мин на ключ
- Запись: 300/мин на IP-адрес, 3000/мин на ключ
- Скачивание: 1200/мин на IP-адрес, 6000/мин на ключ (конечные точки скачивания)

Заголовки:

- Совместимость с устаревшими клиентами: `X-RateLimit-Limit`, `X-RateLimit-Reset`
- Стандартизированные: `RateLimit-Limit`, `RateLimit-Reset`
- При `429`: `X-RateLimit-Remaining: 0` и `RateLimit-Remaining: 0`
- При `429`: `Retry-After`

Семантика заголовков:

- `X-RateLimit-Reset`: абсолютное время в секундах эпохи Unix
- `RateLimit-Reset`: количество секунд до сброса (задержка)
- `X-RateLimit-Remaining` / `RateLimit-Remaining`: точный оставшийся лимит, если указан.
  При успешных сегментированных запросах этот заголовок опускается вместо возврата приблизительного глобального значения.
- `Retry-After`: количество секунд ожидания перед повторной попыткой (задержка) при `429`

Пример ответа `429`:

```http
HTTP/2 429
content-type: text/plain; charset=utf-8
x-ratelimit-limit: 20
x-ratelimit-remaining: 0
x-ratelimit-reset: 1771404540
ratelimit-limit: 20
ratelimit-remaining: 0
ratelimit-reset: 34
retry-after: 34

Превышено ограничение частоты запросов
```

Рекомендации для клиентов:

- Если присутствует `Retry-After`, перед повторной попыткой подождите указанное количество секунд.
- Используйте задержку с джиттером, чтобы избежать синхронизированных повторных попыток.
- Если `Retry-After` отсутствует, используйте `RateLimit-Reset` (или вычислите значение из `X-RateLimit-Reset`).

Источник IP-адреса:

- Доверенные заголовки IP-адреса клиента, включая `cf-connecting-ip`, используются, только если
  в развёртывании явно включены доверенные перенаправленные заголовки.
- ClawHub использует доверенные заголовки перенаправления для определения IP-адресов клиентов на периферии.
- Если доверенный IP-адрес клиента недоступен, для анонимных запросов используются резервные лимиты,
  область действия которых определяется только типом ограничения частоты запросов. Эти резервные лимиты не включают
  предоставленные вызывающей стороной пути, слаги, имена пакетов, версии, строки запросов и другие
  параметры артефактов.

## Ответы с ошибками

Общедоступные ответы v1 с ошибками представляют собой обычный текст с `content-type: text/plain; charset=utf-8`.
К ним относятся ошибки проверки (`400`), отсутствие общедоступных ресурсов (`404`), ошибки аутентификации и
разрешений (`401`/`403`), ограничения частоты запросов (`429`) и заблокированные скачивания. Клиенты
должны считывать тело ответа как понятную человеку строку. Неизвестные параметры запроса
игнорируются для совместимости, но известные параметры запроса с недействительными значениями возвращают
`400`.

## Общедоступные конечные точки (без аутентификации)

### `GET /api/v1/search`

Параметры запроса:

- `q` (обязательный): строка запроса
- `limit` (необязательный): целое число
- `highlightedOnly` (необязательный): `true` для фильтрации только выделенных Skills
- `nonSuspiciousOnly` (необязательный): `true`, чтобы скрыть подозрительные (`flagged.suspicious`) Skills
- `nonSuspicious` (необязательный): устаревший псевдоним для `nonSuspiciousOnly`

Ответ:

```json
{
  "results": [
    {
      "score": 0.123,
      "slug": "gifgrep",
      "displayName": "GifGrep",
      "summary": "…",
      "version": "1.2.3",
      "updatedAt": 1730000000000,
      "ownerHandle": "openclaw",
      "owner": {
        "handle": "openclaw",
        "displayName": "OpenClaw",
        "image": "https://example.com/avatar.png"
      }
    }
  ]
}
```

Примечания:

- Результаты возвращаются в порядке релевантности (сходство эмбеддингов + усиление точных совпадений токенов слага или имени + небольшой коэффициент популярности).
- Релевантность имеет больший вес, чем популярность. Точное совпадение токена слага или отображаемого имени может ранжироваться выше менее точного совпадения со значительно большей вовлечённостью.
- Текст ASCII разбивается на токены по границам слов и знаков препинания. Например, `personal-map` содержит отдельный токен `map`, а `amap-jsapi-skill` содержит `amap`, `jsapi` и `skill`; поэтому при поиске `map` у `personal-map` будет более сильное лексическое совпадение, чем у `amap-jsapi-skill`.
- Популярность масштабируется логарифмически и ограничивается сверху. Skills с высокой вовлечённостью могут ранжироваться ниже, если текст запроса совпадает хуже.
- Подозрительный или скрытый статус модерации может исключить Skill из общедоступного поиска в зависимости от фильтров вызывающей стороны и текущего статуса модерации.

Рекомендации издателям по доступности в поиске:

- Добавьте термины, которые пользователи будут буквально вводить при поиске, в отображаемое имя, описание и теги. Используйте отдельный токен слага, только если он также является стабильным идентификатором, который вы хотите сохранить.
- Не переименовывайте слаг только ради одного запроса, если новый слаг не является более подходящим долгосрочным каноническим именем. Старые слаги становятся псевдонимами перенаправления, но в каноническом URL-адресе, отображаемом слаге и будущих поисковых сводках используется новый слаг.
- Псевдонимы переименования сохраняют разрешение старых URL-адресов и установок, которые разрешаются через реестр, но после индексации переименования поисковое ранжирование основывается на канонических метаданных Skill. Существующая статистика сохраняется за Skill.
- Если Skill неожиданно не отображается, прежде чем изменять метаданные, влияющие на ранжирование, сначала проверьте статус модерации с помощью `clawhub inspect @owner/slug`, выполнив вход.

### `GET /api/v1/skills`

Параметры запроса:

- `limit` (необязательный): целое число (1–200)
- `cursor` (необязательный): курсор пагинации для любой сортировки, кроме `trending`
- `sort` (необязательный): `updated` (по умолчанию), `recommended` (псевдоним: `default`), `createdAt` (псевдоним: `newest`), `downloads`, `stars` (псевдоним: `rating`), устаревшие псевдонимы установок `installsCurrent`/`installs`/`installsAllTime` соответствуют `downloads`, `trending`
- `nonSuspiciousOnly` (необязательный): `true`, чтобы скрыть подозрительные (`flagged.suspicious`) Skills
- `nonSuspicious` (необязательный): устаревший псевдоним для `nonSuspiciousOnly`

Недействительные значения `sort` возвращают `400`.

Примечания:

- `recommended` использует сигналы вовлечённости и давности.
- `trending` ранжирует по количеству установок за последние 7 дней (на основе телеметрии).
- `createdAt` стабилен для обхода новых Skills; `updated` изменяется при повторной публикации существующих Skills.
- При `nonSuspiciousOnly=true` сортировки на основе курсора могут возвращать на странице меньше `limit` элементов, поскольку подозрительные Skills отфильтровываются после получения страницы.
- Если присутствует `nextCursor`, используйте его для продолжения пагинации. Неполная страница сама по себе не означает конец результатов.

Ответ:

```json
{
  "items": [
    {
      "slug": "gifgrep",
      "displayName": "GifGrep",
      "summary": "…",
      "topics": ["Productivity"],
      "tags": { "latest": "1.2.3" },
      "stats": {},
      "createdAt": 0,
      "updatedAt": 0,
      "latestVersion": { "version": "1.2.3", "createdAt": 0, "changelog": "…" },
      "metadata": { "os": ["macos"], "systems": ["aarch64-darwin"] }
    }
  ],
  "nextCursor": null
}
```

Примечания:

- `metadata.os`: ограничения ОС, объявленные во frontmatter Skill (например, `["macos"]`, `["linux"]`). `null`, если не объявлены.

- `metadata` имеет значение `null`, если у Skill отсутствуют метаданные платформы.
- `moderation` включается, только если Skill помечен или его просматривает владелец.

### `POST /api/v1/skills/-/reports/{reportId}/triage`

Эндпоинт для модераторов и администраторов, предназначенный для закрытия или повторного открытия жалоб на Skills.

Запрос:

```json
{ "status": "confirmed", "note": "Проверено; затронутая версия скрыта.", "finalAction": "hide" }
```

`note` обязателен для `confirmed` и `dismissed`; его можно не указывать при
возврате `status` в состояние `open`. Передайте `finalAction: "hide"` вместе с прошедшей рассмотрение
жалобой, чтобы скрыть Skills в рамках того же аудируемого рабочего процесса.

### `GET /api/v1/skills/{slug}/versions`

Параметры запроса:

- `limit` (необязательно): целое число
- `cursor` (необязательно): курсор пагинации

### `GET /api/v1/skills/{slug}/versions/{version}`

Возвращает метаданные версии и список файлов.

- `version.security` включает нормализованный статус проверки сканированием и сведения о сканерах
  (VirusTotal + LLM), если они доступны.

### `GET /api/v1/skills/{slug}/scan`

Возвращает сведения о проверке безопасности для версии Skills.

Параметры запроса:

- `version` (необязательно): строка конкретной версии.
- `tag` (необязательно): разрешить версию по тегу (например, `latest`).

Примечания:

- Если не указан ни `version`, ни `tag`, используется последняя версия.
- Включает нормализованный статус проверки и сведения по каждому сканеру.
- `security.hasScanResult` имеет значение `true` только в том случае, если сканер выдал однозначный вердикт (`clean`, `suspicious` или `malicious`).
- `moderation` — текущий снимок модерации на уровне Skills, полученный из последней версии.
- При запросе исторической версии проверьте `moderation.matchesRequestedVersion` и `moderation.sourceVersion`, прежде чем считать `moderation` и `security` контекстом одной и той же версии.

### `POST /api/v1/skills/-/scan`

Аутентифицированный эндпоинт отправки новых заданий ClawScan.

Сканирование локальных загрузок больше не поддерживается. Запросы с использованием
`multipart/form-data` или `{ "source": { "kind": "upload" } }` возвращают `410`.

Для опубликованных версий сканирование использует JSON:

```json
{
  "source": { "kind": "published", "slug": "gifgrep", "version": "1.2.3" },
  "update": false
}
```

Примечания:

- Полезная нагрузка запросов на сканирование и загружаемые отчёты удаляются из хранилища запросов на сканирование по истечении срока хранения.
- Для сканирования опубликованных версий требуется доступ владельца или издателя к управлению либо полномочия модератора или администратора платформы.
- Результаты сканирования опубликованных версий записываются обратно только при `update: true` и успешном завершении сканирования.
- Ответ — `202` с `{ "ok": true, "scanId": "...", "jobId": "...", "status": "queued", "sourceKind": "published", "update": false, "queue": { "queuedAhead": 0, "queuedAheadIsEstimate": false, "position": 1, "running": 0, "runningIsEstimate": false, "note": "Scans are asynchronous and may take time to complete." } }`.
- Задания сканирования выполняются асинхронно. Ручные запросы на сканирование имеют приоритет над обычными заданиями публикации и обратного заполнения, но завершение всё равно зависит от доступности рабочих процессов.

### `GET /api/v1/skills/-/scan/{scanId}`

Аутентифицированный эндпоинт опроса состояния отправленного сканирования.

- Возвращает состояние: в очереди, выполняется, успешно завершено или завершено с ошибкой.
- Пока запрос находится в очереди, возвращает `queue.queuedAhead` и `queue.position`, чтобы клиенты могли показать количество приоритетных ручных сканирований перед этим запросом. Очень большие очереди ограничиваются, а значение возвращается с `queuedAheadIsEstimate: true`.
- Если данные доступны, `report` содержит разделы `clawscan`, `skillspector`, `staticAnalysis` и `virustotal`.
- Задания сканирования, завершившиеся с ошибкой, возвращают `status: "failed"` с `lastError`.

### `GET /api/v1/skills/-/scan/{scanId}/download`

Аутентифицированный эндпоинт архива отчёта.

- Требует успешно завершённого сканирования; незавершённые сканирования возвращают `409`.
- Возвращает ZIP-архив с `manifest.json`, `clawscan.json`, `skillspector.json`, `static-analysis.json`, `virustotal.json` и `README.md`.

### `GET /api/v1/skills/-/scan/download/{name}?version=<version>&kind=skill|plugin`

Аутентифицированный эндпоинт хранящегося архива отчёта для отправленных версий.

- Требует доступа владельца или издателя к управлению Skills или плагином либо полномочий модератора или администратора платформы.
- Возвращает сохранённые результаты сканирования для конкретной отправленной версии, включая заблокированные или скрытые версии.
- По умолчанию `kind` имеет значение `skill`; для сканирования плагинов и пакетов используйте `kind=plugin`.
- Возвращает ZIP-архив той же структуры, что и при загрузке результатов запросов на сканирование.

### `POST /api/v1/skills/-/scan/batch`

Канонический маршрут пакетного повторного сканирования, доступный только администраторам. Он принимает полезную нагрузку той же структуры, что и устаревший `POST /api/v1/skills/-/rescan-batch`.

### `POST /api/v1/skills/-/scan/batch/status`

Канонический маршрут состояния пакетной обработки, доступный только администраторам. Он принимает `{ "jobIds": ["..."] }` и возвращает те же агрегированные счётчики, что и устаревший `POST /api/v1/skills/-/rescan-batch/status`.

### `GET /api/v1/skills/{slug}/verify`

Возвращает конверт проверки карточки Skills, используемый `clawhub skill verify`.

Параметры запроса:

- `version` (необязательно): строка конкретной версии.
- `tag` (необязательно): разрешить версию по тегу (например, `latest`).

Примечания:

- `ok` имеет значение `true` только в том случае, если для выбранной версии создана карточка Skills, версия не заблокирована модерацией как вредоносная и проверка ClawScan не выявила угроз.
- Идентификатор Skills, идентификатор издателя и метаданные выбранной версии находятся в полях верхнего уровня конверта (`slug`, `displayName`, `publisherHandle`, `version`, `resolvedFrom`, `tag`, `createdAt`), чтобы автоматизация оболочки могла читать их без распаковки вложенных обёрток.
- `security` — вердикт ClawScan и проверки безопасности верхнего уровня. Автоматизация должна ориентироваться на `ok`, `decision`, `reasons` и `security.status`.
- `security.signals` содержит подтверждающие данные сканеров, например `staticScan`, `virusTotal` и `skillSpector`.
- `security.signals.dependencyRegistry` сохранён для совместимости с ответами v1, однако сканер наличия зависимостей в реестре выведен из эксплуатации, поэтому этот ключ всегда имеет значение `null`.
- `provenance` имеет значение `server-resolved-github-import` только в том случае, если при публикации или импорте ClawHub разрешил и сохранил репозиторий, ссылку, коммит и путь GitHub; в противном случае значение равно `unavailable`.

### `POST /api/v1/skills/-/security-verdicts`

Возвращает текущие краткие вердикты безопасности для конкретных версий Skills. Этот
эндпоинт коллекции предназначен для клиентов, которым уже известно, какие установленные
версии Skills из ClawHub необходимо отображать, например OpenClaw Control UI.

Запрос:

```json
{
  "items": [{ "slug": "gifgrep", "version": "1.2.3" }]
}
```

Примечания:

- `items` должен содержать от 1 до 100 уникальных пар `{ slug, version }`.
- Результаты возвращаются отдельно для каждого элемента; отсутствие одного Skills или версии не приводит к ошибке всего ответа.
- Ответ содержит только сведения о безопасности. Он не включает данные карточки Skills, состояние создания карточки, списки файлов артефактов или подробные данные сканеров.
- `security.signals` содержит только подтверждающие данные на уровне состояния; полные сведения о сканерах доступны в `/scan` или на странице аудита безопасности ClawHub.
- `security.signals.dependencyRegistry` сохранён для совместимости с ответами v1, однако сканер наличия зависимостей в реестре выведен из эксплуатации, поэтому этот ключ всегда имеет значение `null`.
- Отсутствие карточки Skills не влияет на `ok`, `decision` или `reasons` этого эндпоинта; когда клиентам требуется содержимое карточки, они должны локально прочитать установленный `skill-card.md`.
- Используйте `/verify`, когда требуется конверт проверки карточки для одного Skills, `/card`, когда требуется Markdown созданной карточки, и `/scan`, когда требуются подробные данные сканеров.

### `GET /api/v1/packages`

Единый эндпоинт каталога для:

- Skills
- плагинов с кодом
- плагинов-пакетов

Параметры запроса:

- `limit` (необязательно): целое число (1–100)
- `cursor` (необязательно): курсор пагинации
- `family` (необязательно): `skill`, `code-plugin` или `bundle-plugin`
- `channel` (необязательно): `official`, `community` или `private`
- `isOfficial` (необязательно): `true` или `false`
- `sort` (необязательно): `updated` (по умолчанию), `recommended`, `trending`, `downloads`, устаревший псевдоним `installs`
- `category` (необязательно): фильтр по категории плагина. Поддерживается только тогда, когда
  запрос ограничен пакетами плагинов (`/api/v1/plugins`,
  `/api/v1/code-plugins`, `/api/v1/bundle-plugins` или эндпоинтами пакетов с
  `family=code-plugin`/`family=bundle-plugin`). Управляемые категории и
  устаревшие псевдонимы фильтров v1 описаны в разделе `GET /api/v1/plugins`.

Примечания:

- Недопустимые значения `family`, `channel`, `isOfficial`, `featured`,
  `highlightedOnly` или `sort` возвращают `400`. Неизвестные параметры запроса игнорируются.
- `GET /api/v1/code-plugins` и `GET /api/v1/bundle-plugins` остаются псевдонимами фиксированных семейств.
- Записи Skills по-прежнему используют реестр Skills и могут публиковаться только через `POST /api/v1/skills`.
- `POST /api/v1/packages` по-прежнему предназначен только для выпусков плагинов с кодом и плагинов-пакетов.
- Неаутентифицированные клиенты видят только общедоступные каналы пакетов.
- Аутентифицированные клиенты могут видеть в результатах просмотра и поиска закрытые пакеты издателей, к которым они относятся.
- `channel=private` возвращает только пакеты, доступные для чтения аутентифицированному клиенту.

### `GET /api/v1/packages/search`

Единый поиск по каталогу Skills и пакетов плагинов.

Параметры запроса:

- `q` (обязательный): строка запроса
- `limit` (необязательный): целое число (1–100)
- `family` (необязательный): `skill`, `code-plugin` или `bundle-plugin`
- `channel` (необязательный): `official`, `community` или `private`
- `isOfficial` (необязательный): `true` или `false`
- `category` (необязательный): фильтр по категории плагина. Поддерживается, только если
  запрос ограничен пакетами плагинов. Управляемые категории и устаревшие псевдонимы
  фильтров v1 описаны в разделе `GET /api/v1/plugins`.

Примечания:

- Недопустимые значения для `family`, `channel`, `isOfficial`, `featured` или
  `highlightedOnly` возвращают `400`. Неизвестные параметры запроса игнорируются.
- Анонимные вызывающие стороны видят только общедоступные каналы пакетов.
- Аутентифицированные вызывающие стороны могут искать закрытые пакеты издателей, к которым они относятся.
- `channel=private` возвращает только пакеты, которые может читать аутентифицированная вызывающая сторона.

### `GET /api/v1/plugins`

Просмотр каталога, содержащего только плагины, среди пакетов плагинов кода и пакетов-плагинов.

Параметры запроса:

- `limit` (необязательный): целое число (1-100)
- `cursor` (необязательный): курсор пагинации
- `isOfficial` (необязательный): `true` или `false`
- `sort` (необязательный): `recommended` (по умолчанию), `trending`, `downloads`, `updated`, устаревший псевдоним `installs`
- `category` (необязательный): фильтр по категории плагина. Текущие значения:
  `channels`, `models`, `memory`, `context`, `voice`, `media`, `web`,
  `tools`, `runtime`, `gateway`, `security`, `other`.

Устаревшие псевдонимы фильтров v1 по-прежнему принимаются конечными точками чтения:

- `mcp-tooling`, `data` и `automation` разрешаются в `tools`.
- `observability` и `deployment` разрешаются в `gateway`.
- `dev-tools` разрешается в `runtime`.

`trending` — рейтинг установок и загрузок за семь дней, не использующий итоговые показатели за всё время.
В объединённой конечной точке `/api/v1/packages` он применяется только к плагинам; для каталога навыков
используйте `/api/v1/skills?sort=trending`.

Устаревшие псевдонимы не принимаются в качестве сохранённых или объявленных автором значений категорий.

### `GET /api/v1/skills/export`

Массовый экспорт последних общедоступных навыков для автономного анализа.

Аутентификация:

- Требуется токен API.

Параметры запроса:

- `startDate` (обязательный): нижняя граница в миллисекундах Unix для `updatedAt` навыка.
- `endDate` (обязательный): верхняя граница в миллисекундах Unix для `updatedAt` навыка.
- `limit` (необязательный): целое число (1-250), по умолчанию `250`.
- `cursor` (необязательный): курсор пагинации из предыдущего ответа.

Ответ:

- Тело: ZIP-архив.
- Корневой каталог каждого экспортированного навыка — `{publisher}/{slug}/`.
- Размещённые навыки содержат файлы последней сохранённой версии и перечислены в
  `_manifest.json` с `sourceRef: "public-clawhub"`.
- Текущие навыки на основе GitHub со сканированием `clean` или `suspicious` содержат
  `_source_handoff.json` с `sourceRef: "public-github"`, репозиторием, коммитом, путём,
  хешем содержимого и URL архива. Они не содержат исходные файлы, размещённые в ClawHub.
- Каждый навык содержит `_export_skill_meta.json`.
- `_manifest.json` всегда включается в корень ZIP-архива.
- `_errors.json` включается, если отдельные навыки или файлы не удалось
  экспортировать.

Заголовки:

- `X-Next-Cursor`
- `X-Has-More`
- `X-Total-Returned`
- `X-Date-Range`
- `X-Export-Errors`

### `GET /api/v1/packages/{name}/versions/{version}/security`

Возвращает точную сводку безопасности и доверия для выпуска пакета,
предназначенную для клиентов установки. Это общедоступная поверхность потребления OpenClaw,
позволяющая определить, можно ли установить разрешённый выпуск.

Аутентификация:

- Общедоступная конечная точка чтения. Токен владельца, издателя, модератора или администратора
  не требуется.

Ответ:

```json
{
  "package": {
    "name": "@openclaw/example-plugin",
    "displayName": "Пример плагина",
    "family": "code-plugin"
  },
  "release": {
    "releaseId": "packageReleases:...",
    "version": "1.2.3",
    "artifactKind": "npm-pack",
    "artifactSha256": "0123456789abcdef...",
    "npmIntegrity": "sha512-...",
    "npmShasum": "0123456789abcdef0123456789abcdef01234567",
    "npmTarballName": "example-plugin-1.2.3.tgz",
    "createdAt": 1730000000000
  },
  "trust": {
    "scanStatus": "malicious",
    "moderationState": "quarantined",
    "blockedFromDownload": true,
    "reasons": ["manual:quarantined", "scan:malicious"],
    "pending": false,
    "stale": false
  }
}
```

Поля ответа:

- `package.name`, `package.displayName` и `package.family` идентифицируют
  разрешённый пакет реестра.
- `release.releaseId`, `release.version` и `release.createdAt` идентифицируют
  конкретный оценённый выпуск.
- `release.artifactKind`, `release.artifactSha256`, `release.npmIntegrity`,
  `release.npmShasum` и `release.npmTarballName` присутствуют, если они известны
  для артефакта выпуска.
- `trust.scanStatus` — фактический статус доверия, полученный на основе входных данных сканера
  и ручной модерации выпуска.
- `trust.moderationState` допускает значение null. Он имеет значение `null`, если ручная модерация
  выпуска отсутствует.
- `trust.blockedFromDownload` — сигнал блокировки установки. OpenClaw и другие
  клиенты установки должны блокировать установку, когда это значение равно `true`, вместо
  повторного вычисления правил блокировки на основе полей сканера или модерации.
- `trust.reasons` — список пояснений для пользователя и аудита. Коды причин —
  стабильные компактные строки, например `manual:quarantined`, `scan:malicious`
  и `package:malicious`.
- `trust.pending` означает, что один или несколько входных данных для оценки доверия всё ещё ожидают обработки.
- `trust.stale` означает, что сводка доверия была вычислена на основе устаревших входных данных и
  перед принятием решения о разрешении с высокой степенью уверенности её следует обновить.

Примечания:

- Эта конечная точка привязана к конкретной версии. Клиентам следует вызывать её после разрешения
  версии пакета, которую они намерены установить, а не только после чтения метаданных
  последней версии пакета.
- Закрытые пакеты возвращают `404`, если вызывающая сторона не может читать данные издателя-владельца.
- Эта конечная точка намеренно имеет более узкую область, чем конечные точки модерации
  для владельцев и модераторов. Она предоставляет решение об установке и общедоступное пояснение, но не
  личности отправителей жалоб, тексты жалоб, закрытые доказательства или внутренние сроки
  рассмотрения.

Скачивает устаревший детерминированный ZIP-архив выпуска пакета.

Параметры запроса:

- `version` (необязательно)
- `tag` (необязательно)

Примечания:

- По умолчанию используется последний выпуск.

## Конечные точки аутентификации (токен Bearer)

Для всех конечных точек требуется:

```
Authorization: Bearer clh_...
```

### `GET /api/v1/whoami`

Проверяет токен и возвращает идентификатор пользователя.

### `POST /api/v1/skills`

Публикует новую версию.

- Предпочтительный вариант: `multipart/form-data` с JSON `payload` и двоичными объектами `files[]`.
- Также принимается тело JSON с `files` (на основе storageId).
- Необязательное поле полезной нагрузки: `ownerHandle`. Если оно указано, API определяет
  этого издателя на стороне сервера и требует, чтобы субъект имел доступ издателя.
- Необязательное поле полезной нагрузки: `migrateOwner`. При `true` вместе с `ownerHandle`
  существующий Skill можно передать этому владельцу, если субъект является администратором/владельцем
  как у текущего, так и у целевого издателя. Без этого явного согласия изменения владельца
  отклоняются.

### `POST /api/v1/packages`

Публикует выпуск плагина кода или пакетного плагина.

- Требуется аутентификация с помощью токена Bearer.
- Требуется `multipart/form-data`.
- Допустимые поля формы: `payload`, повторяющиеся двоичные объекты `files` или одна ссылка
  на tarball `clawpack`. `clawpack` может быть двоичным объектом `.tgz` или идентификатором хранилища,
  возвращённым процессом получения URL загрузки. Публикации с подготовленным идентификатором хранилища
  также должны содержать `clawpackUploadTicket`, возвращённый вместе с этим URL загрузки.
- Используйте либо `files`, либо `clawpack`, но никогда не оба в одном запросе.
- Тела JSON и предоставленные вызывающей стороной метаданные `payload.files` / `payload.artifact`
  отклоняются.
- Размер прямых multipart-запросов публикации ограничен 18MB. Для tarball ClawPack
  можно использовать процесс получения URL загрузки с ограничением размера tarball в 120MB.
- Необязательное поле полезной нагрузки: `ownerHandle`. Если оно указано, публиковать от имени этого владельца могут только администраторы.

Основные правила проверки:

- `family` должен иметь значение `code-plugin` или `bundle-plugin`.
- Для пакетов плагинов требуется `openclaw.plugin.json`. Загрузки ClawPack `.tgz`
  должны содержать его по пути `package/openclaw.plugin.json`.
- Для плагинов кода требуются `package.json`, метаданные репозитория исходного кода, метаданные
  коммита исходного кода, метаданные схемы конфигурации, `openclaw.compat.pluginApi` и
  `openclaw.build.openclawVersion`.
- `openclaw.hostTargets` и `openclaw.environment` — необязательные метаданные.
- Публиковать в канал `official` могут только издатель организации `openclaw`
  и личные издатели текущих участников организации `openclaw`.
- При публикации от чужого имени право на официальный канал всё равно проверяется для учётной записи целевого владельца.

### `DELETE /api/v1/skills/{slug}` / `POST /api/v1/skills/{slug}/undelete`

Программно удалить / восстановить Skill (владелец, модератор или администратор).

Необязательное тело JSON:

```json
{ "reason": "Удерживается для модерации до завершения юридической проверки." }
```

Если поле `reason` указано, оно сохраняется как примечание модерации Skill и копируется в журнал аудита.
При программном удалении владельцем слаг резервируется на 30 дней, после чего его может занять
другой издатель. Когда применяется этот срок, ответ на удаление содержит `slugReservedUntil`.
Скрытие модератором/администратором и удаление по соображениям безопасности таким образом не истекают.

Ответ на удаление:

```json
{ "ok": true, "slugReservedUntil": 1730000000000 }
```

Коды состояния:

- `200`: успешно
- `401`: аутентификация не выполнена
- `403`: доступ запрещён
- `404`: Skill/пользователь не найден
- `500`: внутренняя ошибка сервера

### `POST /api/v1/users/publisher`

Только для администраторов. Гарантирует наличие издателя организации для указанного идентификатора. Если идентификатор всё ещё указывает на
устаревшего общего пользователя/личного издателя, конечная точка сначала переносит его в издателя организации.
Для вновь созданной организации укажите `memberHandle`; действующий администратор не добавляется в качестве участника.
По умолчанию `memberRole` имеет значение `owner`.

- Тело: `{ "handle": "openclaw", "displayName": "OpenClaw", "memberHandle": "alice", "memberRole": "owner", "trusted": true }`
- Ответ: `{ "ok": true, "publisherId": "...", "handle": "openclaw", "created": true, "migrated": false, "trusted": true, "member": { "userId": "...", "handle": "alice", "role": "owner" } }`

### `POST /api/v1/publishers`

Самостоятельное создание издателя организации с аутентификацией. Создаёт нового издателя организации и добавляет
вызывающую сторону в качестве владельца. Эта конечная точка не переносит существующие пользовательские/личные идентификаторы и
не помечает издателя как доверенного/официального.

- Тело: `{ "handle": "opik", "displayName": "Opik" }`
- Ответ: `{ "ok": true, "publisherId": "...", "handle": "opik", "created": true, "trusted": false }`
- Возвращает `409`, если идентификатор уже используется издателем, пользователем или личным издателем.

### `POST /api/v1/users/reserve`

Только для администраторов. Резервирует корневые слаги и имена пакетов для законного владельца без публикации
выпуска. Имена пакетов становятся приватными пакетами-заполнителями без строк выпусков, поэтому тот же
владелец впоследствии может опубликовать настоящий выпуск плагина кода или пакетного плагина с этим именем.

- Тело: `{ "handle": "openclaw", "slugs": ["diffs"], "packageNames": ["@openclaw/diffs"], "reason": "reserved for official OpenClaw plugin" }`
- Ответ: `{ "ok": true, "succeeded": 2, "failed": 0, "results": [{ "kind": "slug", "name": "diffs", "ok": true, "action": "reserved" }] }`

### `POST /api/v1/users/publisher-recovery`

Только для администраторов. Восстанавливает личного издателя для проверенного замещающего субъекта GitHub OAuth
без редактирования строк учётной записи Convex Auth. В запросе необходимо указать оба неизменяемых идентификатора
учётных записей провайдера GitHub; изменяемые идентификаторы используются только как проверочное условие для оператора.

По умолчанию конечная точка выполняет пробный запуск. Для применения восстановления требуются `dryRun: false` и
`confirmIdentityVerified: true` после того, как сотрудники независимо подтвердят преемственность между обоими
субъектами GitHub. Восстановление завершается отказом, если у текущего личного издателя целевого пользователя
есть Skills, пакеты или источники Skills GitHub.
Восстановление также переносит устаревшие поля `ownerUserId` для Skills восстановленного издателя,
псевдонимы слагов Skills, пакеты, предупреждения инспектора пакетов и производные строки дайджеста поиска, чтобы
пути прямого владельца соответствовали полномочиям нового издателя. Активное резервирование защищённого
идентификатора для восстановленного идентификатора также переназначается замещающему пользователю, чтобы последующая
синхронизация профиля не могла восстановить конкурирующие полномочия прежнего пользователя. Каждая основная таблица ограничена
100 строками на транзакцию применения; для более крупных восстановлений сначала необходимо использовать возобновляемый перенос владельца.
Источники Skills GitHub привязаны к издателю и отмечаются в отчёте как проверенные, а не перезаписанные.

- Тело: `{ "handle": "gingiris", "nextUserHandle": "gingiris-1031", "previousGitHubProviderAccountId": "123", "nextGitHubProviderAccountId": "456", "reason": "Verified account continuity for issue #2555", "confirmIdentityVerified": true, "dryRun": false }`
- Ответ: `{ "ok": true, "dryRun": false, "recovered": true, "publisherId": "...", "handle": "gingiris", "previousUser": { "userId": "...", "handle": "gingiris", "nextHandle": "gingiris-recovered", "githubProviderAccountId": "123", "authAccountCount": 1 }, "nextUser": { "userId": "...", "handle": "gingiris-1031", "nextHandle": "gingiris", "githubProviderAccountId": "456", "authAccountCount": 1 }, "retiredPersonalPublisher": null, "resourceOwnerMigration": { "limitPerTable": 100, "skills": 1, "skillSlugAliases": 1, "packages": 0, "packageInspectorWarnings": 0, "githubSourcesChecked": 1, "handleReservations": 1 }, "identityVerified": true, "reason": "Verified account continuity for issue #2555" }`

### Конечные точки управления слагами владельца

- `POST /api/v1/skills/{slug}/rename`
  - Тело: `{ "newSlug": "new-canonical-slug" }`
  - Ответ: `{ "ok": true, "slug": "new-canonical-slug", "previousSlug": "old-slug" }`
- `POST /api/v1/skills/{slug}/merge`
  - Тело: `{ "targetSlug": "canonical-target-slug" }`
  - Ответ: `{ "ok": true, "sourceSlug": "old-slug", "targetSlug": "canonical-target-slug" }`

Примечания:

- Обе конечные точки требуют аутентификации с помощью токена API и работают только для владельца Skill.
- `rename` сохраняет предыдущий слаг как псевдоним перенаправления.
- `merge` скрывает исходную публикацию и перенаправляет исходный слаг на целевую публикацию.

- `POST /api/v1/skills/{slug}/transfer`

### `GET /api/v1/users`

Получить список пользователей или выполнить поиск по ним (только для администраторов).

Параметры запроса:

- `q` (необязательно): поисковый запрос
- `query` (необязательно): псевдоним для `q`
- `limit` (необязательно): максимальное число результатов (по умолчанию 20, максимум 200)

Ответ:

```json
{
  "items": [
    {
      "userId": "users_...",
      "handle": "user_handle",
      "displayName": "Пользователь",
      "name": "Пользователь",
      "role": "moderator"
    }
  ],
  "total": 1
}
```

## Обнаружение реестра (`/.well-known/clawhub.json`)

CLI может получать настройки реестра/аутентификации с сайта:

- `/.well-known/clawhub.json` (JSON, предпочтительно)
- `/.well-known/clawdhub.json` (устаревший вариант)

Схема:

```json
{ "apiBase": "https://clawhub.ai", "authBase": "https://clawhub.ai", "minCliVersion": "0.0.5" }
```

При самостоятельном размещении предоставьте этот файл (или явно задайте `CLAWHUB_REGISTRY`; устаревший вариант — `CLAWDHUB_REGISTRY`).
