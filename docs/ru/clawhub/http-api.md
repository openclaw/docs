# HTTP API

Базовый URL: `https://clawhub.ai` (по умолчанию).

Все пути v1 находятся в `/api/v1/...`.
Устаревшие `/api/...` и `/api/cli/...` сохранены для совместимости (см. `DEPRECATIONS.md`).
OpenAPI: `/api/v1/openapi.json`.

## Повторное использование публичного каталога

Сторонние каталоги могут использовать публичные конечные точки чтения для отображения списка или поиска Skills ClawHub. Кэшируйте результаты, соблюдайте `429`/`Retry-After`, направляйте пользователей на каноническую страницу ClawHub (`https://clawhub.ai/<owner>/skills/<slug>`) и не создавайте впечатление, что ClawHub рекомендует сторонний сайт. Не пытайтесь зеркалировать скрытый, приватный или заблокированный модерацией контент за пределами публичной поверхности API.

Сокращённые веб-идентификаторы разрешаются между семействами реестра, но клиенты API должны использовать
канонические URL, возвращаемые конечными точками чтения, вместо самостоятельного восстановления
приоритета маршрутов.

## Ограничения частоты запросов

Модель применения ограничений:

- Анонимные запросы: ограничение применяется для каждого IP-адреса.
- Аутентифицированные запросы (действительный Bearer-токен): ограничение применяется для пользовательской квоты.
- Если токен отсутствует или недействителен, применяется ограничение по IP-адресу.
- Аутентифицированные конечные точки записи не должны возвращать только `Unauthorized`, если
  серверу известна причина. Для отсутствующих, недействительных или отозванных токенов, а также
  удалённых, заблокированных или отключённых учётных записей должен возвращаться информативный текст, чтобы клиенты
  CLI могли сообщить пользователям, что именно заблокировало запрос.

- Чтение: 3000/мин на IP-адрес, 12000/мин на ключ
- Запись: 300/мин на IP-адрес, 3000/мин на ключ
- Скачивание: 1200/мин на IP-адрес, 6000/мин на ключ (конечные точки скачивания)

Заголовки:

- Совместимость с устаревшими клиентами: `X-RateLimit-Limit`, `X-RateLimit-Reset`
- Стандартизированные: `RateLimit-Limit`, `RateLimit-Reset`
- При `429`: `X-RateLimit-Remaining: 0` и `RateLimit-Remaining: 0`
- При `429`: `Retry-After`

Семантика заголовков:

- `X-RateLimit-Reset`: абсолютное время в секундах эпохи Unix
- `RateLimit-Reset`: количество секунд до сброса (задержка)
- `X-RateLimit-Remaining` / `RateLimit-Remaining`: точный остаток квоты, если указан.
  Для успешно выполненных сегментированных запросов этот заголовок не возвращается вместо указания приблизительного глобального значения.
- `Retry-After`: количество секунд ожидания перед повторной попыткой (задержка) при `429`

Пример ответа `429`:

```http
HTTP/2 429
content-type: text/plain; charset=utf-8
x-ratelimit-limit: 20
x-ratelimit-remaining: 0
x-ratelimit-reset: 1771404540
ratelimit-limit: 20
ratelimit-remaining: 0
ratelimit-reset: 34
retry-after: 34

Превышено ограничение частоты запросов
```

Рекомендации для клиентов:

- Если присутствует `Retry-After`, перед повторной попыткой подождите указанное количество секунд.
- Используйте экспоненциальную задержку со случайным разбросом, чтобы избежать синхронных повторных попыток.
- Если `Retry-After` отсутствует, используйте `RateLimit-Reset` (или вычислите значение по `X-RateLimit-Reset`).

Источник IP-адреса:

- Доверенные заголовки с IP-адресом клиента, включая `cf-connecting-ip`, используются только тогда, когда
  в развёртывании явно включена поддержка доверенных перенаправленных заголовков.
- ClawHub использует доверенные заголовки перенаправления для определения IP-адресов клиентов на периферии сети.
- Если доверенный IP-адрес клиента недоступен, анонимные запросы используют резервные квоты,
  область действия которых определяется только типом ограничения частоты запросов. Эти резервные квоты не включают
  предоставленные вызывающей стороной пути, сокращённые идентификаторы, имена пакетов, версии, строки запросов или другие
  параметры артефактов.

## Ответы с ошибками

Публичные ответы v1 с ошибками представляют собой обычный текст с `content-type: text/plain; charset=utf-8`.
Это относится к ошибкам проверки (`400`), отсутствующим публичным ресурсам (`404`), ошибкам аутентификации и
разрешений (`401`/`403`), ограничениям частоты запросов (`429`) и заблокированным скачиваниям. Клиенты
должны читать тело ответа как удобочитаемую строку. Неизвестные параметры запроса
игнорируются для совместимости, но для распознанных параметров запроса с недопустимыми значениями возвращается
`400`.

## Публичные конечные точки (без аутентификации)

### `GET /api/v1/search`

Параметры запроса:

- `q` (обязательный): строка запроса
- `limit` (необязательный): целое число
- `highlightedOnly` (необязательный): `true` для фильтрации только избранных Skills
- `nonSuspiciousOnly` (необязательный): `true`, чтобы скрыть подозрительные (`flagged.suspicious`) Skills
- `nonSuspicious` (необязательный): устаревший псевдоним для `nonSuspiciousOnly`

Ответ:

```json
{
  "results": [
    {
      "score": 0.123,
      "slug": "gifgrep",
      "displayName": "GifGrep",
      "summary": "…",
      "version": "1.2.3",
      "updatedAt": 1730000000000,
      "ownerHandle": "openclaw",
      "owner": {
        "handle": "openclaw",
        "displayName": "OpenClaw",
        "image": "https://example.com/avatar.png"
      }
    }
  ]
}
```

Примечания:

- Результаты возвращаются в порядке релевантности (сходство векторных представлений + повышение рейтинга за точное совпадение токена сокращённого идентификатора или имени + небольшой приоритет популярности).
- Релевантность важнее популярности. Точное совпадение токена сокращённого идентификатора или отображаемого имени может получить более высокий рейтинг, чем менее точное совпадение с гораздо более высокими показателями вовлечённости.
- Текст ASCII разбивается на токены по границам слов и знаков препинания. Например, `personal-map` содержит отдельный токен `map`, а `amap-jsapi-skill` содержит `amap`, `jsapi` и `skill`; поэтому при поиске `map` значение `personal-map` получает более сильное лексическое совпадение, чем `amap-jsapi-skill`.
- Популярность масштабируется логарифмически и ограничивается сверху. Skills с высокой вовлечённостью могут занимать более низкие позиции, если текст запроса соответствует им хуже.
- Подозрительный или скрытый статус модерации может исключить навык из публичного поиска в зависимости от фильтров вызывающей стороны и текущего статуса модерации.

Рекомендации издателям по доступности для поиска:

- Добавляйте термины, которые пользователи будут буквально вводить при поиске, в отображаемое имя, сводку и теги. Используйте отдельный токен сокращённого идентификатора только в том случае, если это также стабильный идентификатор, который требуется сохранить.
- Не переименовывайте сокращённый идентификатор только ради одного запроса, если новое значение не является более подходящим долгосрочным каноническим именем. Старые сокращённые идентификаторы становятся псевдонимами перенаправления, но канонический URL, отображаемый сокращённый идентификатор и будущие поисковые дайджесты используют новый сокращённый идентификатор.
- Псевдонимы переименования сохраняют разрешение старых URL и установок, разрешаемых через реестр, но после индексации переименования поисковый рейтинг основывается на канонических метаданных Skills. Существующая статистика сохраняется за навыком.
- Если навык неожиданно не отображается, перед изменением метаданных, связанных с ранжированием, сначала проверьте статус модерации с помощью `clawhub inspect @owner/slug`, войдя в систему.

### `GET /api/v1/skills`

Параметры запроса:

- `limit` (необязательный): целое число (1–200)
- `cursor` (необязательный): курсор пагинации для любой сортировки, кроме `trending`
- `sort` (необязательный): `updated` (по умолчанию), `recommended` (псевдоним: `default`), `createdAt` (псевдоним: `newest`), `downloads`, `stars` (псевдоним: `rating`), устаревшие псевдонимы установок `installsCurrent`/`installs`/`installsAllTime` сопоставляются с `downloads`, `trending`
- `nonSuspiciousOnly` (необязательный): `true`, чтобы скрыть подозрительные (`flagged.suspicious`) Skills
- `nonSuspicious` (необязательный): устаревший псевдоним для `nonSuspiciousOnly`

Для недопустимых значений `sort` возвращается `400`.

Примечания:

- `recommended` использует сигналы вовлечённости и актуальности.
- `trending` ранжирует по количеству установок за последние 7 дней (на основе телеметрии).
- `createdAt` стабилен для сканирования новых Skills; `updated` изменяется при повторной публикации существующих Skills.
- При `nonSuspiciousOnly=true` сортировки на основе курсора могут возвращать на странице меньше `limit` элементов, поскольку подозрительные Skills фильтруются после получения страницы.
- Используйте `nextCursor` для продолжения пагинации, если он присутствует. Короткая страница сама по себе не означает конец результатов.

Ответ:

```json
{
  "items": [
    {
      "slug": "gifgrep",
      "displayName": "GifGrep",
      "summary": "…",
      "topics": ["Продуктивность"],
      "tags": { "latest": "1.2.3" },
      "stats": {},
      "createdAt": 0,
      "updatedAt": 0,
      "latestVersion": { "version": "1.2.3", "createdAt": 0, "changelog": "…" },
      "metadata": { "os": ["macos"], "systems": ["aarch64-darwin"] }
    }
  ],
  "nextCursor": null
}
```

- `moderation` включается только тогда, когда навык помечен или его просматривает владелец.

### `POST /api/v1/skills/-/reports/{reportId}/triage`

Конечная точка для модераторов и администраторов, позволяющая закрывать или повторно открывать жалобы на Skills.

Запрос:

```json
{ "status": "confirmed", "note": "Проверено, затронутая версия скрыта.", "finalAction": "hide" }
```

`note` обязателен для `confirmed` и `dismissed`; его можно не указывать при
возврате `status` в состояние `open`. Передайте `finalAction: "hide"` вместе с прошедшей разбор
жалобой, чтобы скрыть Skill в рамках того же аудируемого рабочего процесса.

### `GET /api/v1/skills/{slug}/versions`

Параметры запроса:

- `limit` (необязательно): целое число
- `cursor` (необязательно): курсор пагинации

### `GET /api/v1/skills/{slug}/versions/{version}`

Возвращает метаданные версии и список файлов.

- `version.security` содержит нормализованный статус проверки сканированием и сведения о сканерах
  (VirusTotal + LLM), если они доступны.

### `GET /api/v1/skills/{slug}/scan`

Возвращает сведения о проверке безопасности для версии Skill.

Параметры запроса:

- `version` (необязательно): конкретная строка версии.
- `tag` (необязательно): разрешить версию по тегу (например, `latest`).

Примечания:

- Если не указаны ни `version`, ни `tag`, используется последняя версия.
- Содержит нормализованный статус проверки и сведения для каждого сканера.
- `security.hasScanResult` имеет значение `true`, только когда сканер выдал однозначный вердикт (`clean`, `suspicious` или `malicious`).
- `moderation` — это текущий снимок состояния модерации на уровне Skill, полученный из последней версии.
- При запросе исторической версии проверьте `moderation.matchesRequestedVersion` и `moderation.sourceVersion`, прежде чем считать, что `moderation` и `security` относятся к одной и той же версии.

### `POST /api/v1/skills/-/scan`

Аутентифицированная конечная точка отправки новых заданий ClawScan.

Сканирование локально загруженных файлов больше не поддерживается. Запросы с использованием
`multipart/form-data` или `{ "source": { "kind": "upload" } }` возвращают `410`.

Для сканирования опубликованных материалов используется JSON:

```json
{
  "source": { "kind": "published", "slug": "gifgrep", "version": "1.2.3" },
  "update": false
}
```

Примечания:

- Данные запросов на сканирование и загружаемые отчёты удаляются из хранилища запросов на сканирование по истечении срока хранения.
- Для сканирования опубликованных материалов требуется доступ владельца или издателя к управлению либо полномочия модератора или администратора платформы.
- Результаты сканирования опубликованных материалов записываются обратно, только если задано `update: true` и сканирование завершилось успешно.
- Ответ — `202` с `{ "ok": true, "scanId": "...", "jobId": "...", "status": "queued", "sourceKind": "published", "update": false, "queue": { "queuedAhead": 0, "queuedAheadIsEstimate": false, "position": 1, "running": 0, "runningIsEstimate": false, "note": "Scans are asynchronous and may take time to complete." } }`.
- Задания сканирования выполняются асинхронно. Запросы на ручное сканирование имеют приоритет перед обычными заданиями публикации и дозаполнения, но завершение всё равно зависит от доступности обработчиков.

### `GET /api/v1/skills/-/scan/{scanId}`

Аутентифицированная конечная точка опроса состояния отправленного сканирования.

- Возвращает состояние: в очереди, выполняется, успешно завершено или завершено с ошибкой.
- Пока задание находится в очереди, возвращает `queue.queuedAhead` и `queue.position`, чтобы клиенты могли показать, сколько приоритетных ручных сканирований предшествует запросу. Размер очень больших очередей ограничивается и сообщается с помощью `queuedAheadIsEstimate: true`.
- Если доступно, `report` содержит разделы `clawscan`, `skillspector`, `staticAnalysis` и `virustotal`.
- Неудачно завершившиеся задания сканирования возвращают `status: "failed"` с `lastError`.

### `GET /api/v1/skills/-/scan/{scanId}/download`

Аутентифицированная конечная точка архива отчёта.

- Требует успешно завершённого сканирования; сканирования, не достигшие конечного состояния, возвращают `409`.
- Возвращает ZIP-архив с `manifest.json`, `clawscan.json`, `skillspector.json`, `static-analysis.json`, `virustotal.json` и `README.md`.

### `GET /api/v1/skills/-/scan/download/{name}?version=<version>&kind=skill|plugin`

Аутентифицированная конечная точка архива сохранённых отчётов для отправленных версий.

- Требует доступа владельца или издателя к управлению Skill или плагином либо полномочий модератора или администратора платформы.
- Возвращает сохранённые результаты сканирования для точной отправленной версии, включая заблокированные или скрытые версии.
- `kind` по умолчанию имеет значение `skill`; для сканирований плагинов или пакетов используйте `kind=plugin`.
- Возвращает ZIP-архив той же структуры, что и при загрузке результатов запросов на сканирование.

Канонический маршрут пакетного повторного сканирования, доступный только администраторам. Он принимает данные той же структуры, что и устаревший `POST /api/v1/skills/-/rescan-batch`.

Канонический маршрут состояния пакетной операции, доступный только администраторам. Он принимает `{ "jobIds": ["..."] }` и возвращает те же агрегированные счётчики, что и устаревший `POST /api/v1/skills/-/rescan-batch/status`.

### `GET /api/v1/skills/{slug}/verify`

Возвращает оболочку проверки карточки Skill, используемую `clawhub skill verify`.

Параметры запроса:

- `version` (необязательно): конкретная строка версии.
- `tag` (необязательно): разрешить версию по тегу (например, `latest`).

Примечания:

- `ok` имеет значение `true`, только если для выбранной версии создана карточка Skill, версия не заблокирована модерацией как вредоносная и проверка ClawScan не обнаружила проблем.
- Идентификатор Skill, идентификатор издателя и метаданные выбранной версии находятся в полях верхнего уровня оболочки (`slug`, `displayName`, `publisherHandle`, `version`, `resolvedFrom`, `tag`, `createdAt`), чтобы сценарии оболочки могли читать их без распаковки вложенных оболочек.
- `security` — это вердикт ClawScan и проверки безопасности верхнего уровня. Автоматизация должна ориентироваться на `ok`, `decision`, `reasons` и `security.status`.
- `security.signals` содержит подтверждающие данные сканеров, такие как `staticScan`, `virusTotal` и `skillSpector`.
- `security.signals.dependencyRegistry` сохранён для совместимости ответов v1, но сканер наличия зависимостей в реестре выведен из эксплуатации, поэтому этот ключ всегда имеет значение `null`.
- `provenance` имеет значение `server-resolved-github-import`, только если ClawHub определил и сохранил репозиторий, ссылку, коммит и путь GitHub во время публикации или импорта; в противном случае его значение — `unavailable`.

### `GET /api/v1/plugins`

Просмотр каталога только плагинов среди пакетов плагинов кода и пакетов плагинов-комплектов.

Параметры запроса:

- `limit` (необязательный): целое число (1-100)
- `cursor` (необязательный): курсор пагинации
- `isOfficial` (необязательный): `true` или `false`
- `sort` (необязательный): `recommended` (по умолчанию), `trending`, `downloads`, `updated`, устаревший псевдоним `installs`
- `category` (необязательный): фильтр по категории плагинов. Текущие значения:

Устаревшие псевдонимы фильтров v1 по-прежнему принимаются конечными точками чтения:

### `GET /api/v1/skills/export`

Массовый экспорт последних публичных Skills для автономного анализа.

Аутентификация:

- Требуется токен API.

Параметры запроса:

- `startDate` (обязательный): нижняя граница в миллисекундах Unix для `updatedAt` Skills.
- `endDate` (обязательный): верхняя граница в миллисекундах Unix для `updatedAt` Skills.
- `limit` (необязательный): целое число (1-250), по умолчанию `250`.
- `cursor` (необязательный): курсор пагинации из предыдущего ответа.

Ответ:

- Тело: ZIP-архив.
- Корневой каталог каждого экспортированного Skills находится по адресу `{publisher}/{slug}/`.
- Размещённые Skills содержат файлы последней сохранённой версии и перечислены в
  `_manifest.json` с `sourceRef: "public-clawhub"`.
- Текущие Skills на базе GitHub со сканированием `clean` или `suspicious` содержат
  `_source_handoff.json` с `sourceRef: "public-github"`, репозиторием, коммитом, путём,
  хешем содержимого и URL-адресом архива. Они не содержат исходные файлы, размещённые в ClawHub.
- Каждый Skills содержит `_export_skill_meta.json`.
- `_manifest.json` всегда включается в корень ZIP-архива.
- `_errors.json` включается, если отдельные Skills или файлы не удалось
  экспортировать.

Заголовки:

- `X-Next-Cursor`
- `X-Has-More`
- `X-Total-Returned`
- `X-Date-Range`
- `X-Export-Errors`

### `GET /api/v1/packages/{name}/versions/{version}/security`

Возвращает точную сводку безопасности и доверия для артефакта выпуска пакета,
предназначенную для клиентов установки. Это публичная поверхность OpenClaw для принятия решения о том, можно ли
установить разрешённый выпуск.

Аутентификация:

- Публичная конечная точка чтения. Токен владельца, издателя, модератора или администратора
  не требуется.

Ответ:

```json
{
  "package": {
    "name": "@openclaw/example-plugin",
    "displayName": "Пример плагина",
    "family": "code-plugin"
  },
  "release": {
    "releaseId": "packageReleases:...",
    "version": "1.2.3",
    "artifactKind": "npm-pack",
    "artifactSha256": "0123456789abcdef...",
    "npmIntegrity": "sha512-...",
    "npmShasum": "0123456789abcdef0123456789abcdef01234567",
    "npmTarballName": "example-plugin-1.2.3.tgz",
    "createdAt": 1730000000000
  },
  "trust": {
    "scanStatus": "malicious",
    "moderationState": "quarantined",
    "blockedFromDownload": true,
    "reasons": ["manual:quarantined", "scan:malicious"],
    "pending": false,
    "stale": false
  }
}
```

Поля ответа:

- `package.name`, `package.displayName` и `package.family` идентифицируют
  разрешённый пакет реестра.
- `release.releaseId`, `release.version` и `release.createdAt` идентифицируют
  конкретный выпуск, который был оценён.
- `release.artifactKind`, `release.artifactSha256`, `release.npmIntegrity`,
  `release.npmShasum` и `release.npmTarballName` присутствуют, если они известны
  для артефакта выпуска.
- `trust.scanStatus` — эффективный статус доверия, вычисляемый на основе данных сканера
  и ручной модерации выпуска.
- `trust.moderationState` допускает значение null. Оно равно `null`, если ручная модерация
  выпуска отсутствует.
- `trust.blockedFromDownload` — сигнал блокировки установки. OpenClaw и другие
  клиенты установки должны блокировать установку, когда это значение равно `true`, вместо
  повторного вычисления правил блокировки по полям сканера или модерации.
- `trust.reasons` — список объяснений для пользователя и аудита. Коды причин
  представляют собой стабильные компактные строки, например `manual:quarantined`, `scan:malicious`
  и `package:malicious`.
- `trust.pending` означает, что один или несколько источников данных о доверии всё ещё ожидают завершения.
- `trust.stale` означает, что сводка доверия была вычислена на основе устаревших данных и
  перед принятием решения о разрешении с высокой степенью уверенности требует обновления.

Примечания:

- Эта конечная точка привязана к конкретной версии. Клиентам следует вызывать её после разрешения
  версии пакета, которую они намерены установить, а не только после чтения последних
  метаданных пакета.
- Для приватных пакетов возвращается `404`, если вызывающая сторона не может читать данные владеющего ими издателя.
- Эта конечная точка намеренно имеет более узкую область, чем конечные точки модерации
  для владельцев и модераторов. Она предоставляет решение об установке и публичное объяснение, но не
  личности отправителей жалоб, тексты жалоб, закрытые доказательства или внутренние временные шкалы
  проверок.

## Конечные точки аутентификации (токен Bearer)

Все конечные точки требуют:

```
Authorization: Bearer clh_...
```

### `GET /api/v1/whoami`

Проверяет токен и возвращает идентификатор пользователя.

### `POST /api/v1/skills`

Публикует новую версию.

- Предпочтительный вариант: `multipart/form-data` с JSON `payload` и двоичными объектами `files[]`.
- Также принимается тело JSON с `files` (на основе storageId).
- Необязательное поле полезной нагрузки: `ownerHandle`. Если оно присутствует, API определяет
  этого издателя на стороне сервера и требует, чтобы субъект имел доступ издателя.
- Необязательное поле полезной нагрузки: `migrateOwner`. При `true` вместе с `ownerHandle`
  существующий Skill можно передать этому владельцу, если субъект является администратором/владельцем как у текущего,
  так и у целевого издателя. Без этого явного согласия изменения владельца
  отклоняются.

### `POST /api/v1/packages`

Публикует выпуск плагина с кодом или плагина-пакета.

- Требует аутентификации с токеном Bearer.
- Требует `multipart/form-data`.
- Допустимые поля формы: `payload`, повторяющиеся двоичные объекты `files` или одна ссылка на tar-архив `clawpack`.
  `clawpack` может быть двоичным объектом `.tgz` или идентификатором хранилища, возвращённым
  процедурой получения URL загрузки. Публикации с промежуточным идентификатором хранилища также должны включать
  `clawpackUploadTicket`, возвращённый вместе с этим URL загрузки.
- Используйте либо `files`, либо `clawpack`, но никогда не оба в одном запросе.
- Тела JSON и предоставленные вызывающей стороной метаданные `payload.files` / `payload.artifact`
  отклоняются.
- Размер прямых многочастных запросов публикации ограничен 18MB. Для tar-архивов ClawPack
  можно использовать процедуру получения URL загрузки с ограничением размера tar-архива до 120MB.
- Необязательное поле полезной нагрузки: `ownerHandle`. Если оно присутствует, публиковать от имени этого владельца могут только администраторы.

Основные моменты проверки:

- `family` должно иметь значение `code-plugin` или `bundle-plugin`.
- Пакеты плагинов требуют `openclaw.plugin.json`. Загружаемые объекты ClawPack `.tgz` должны
  содержать его по адресу `package/openclaw.plugin.json`.
- Плагины с кодом требуют `package.json`, метаданные исходного репозитория, метаданные
  исходного коммита, метаданные схемы конфигурации, `openclaw.compat.pluginApi` и
  `openclaw.build.openclawVersion`.
- `openclaw.hostTargets` и `openclaw.environment` являются необязательными метаданными.
- Публиковать в канале `official` могут только издатель организации `openclaw`
  и личные издатели текущих участников организации `openclaw`.
- При публикации от чужого имени право на официальный канал всё равно проверяется относительно учётной записи целевого владельца.

### `DELETE /api/v1/skills/{slug}` / `POST /api/v1/skills/{slug}/undelete`

Программное удаление / восстановление Skill (владелец, модератор или администратор).

Необязательное тело JSON:

```json
{ "reason": "Удержано для модерации до завершения юридической проверки." }
```

Если `reason` указано, оно сохраняется как примечание о модерации Skill и копируется в журнал аудита.
При программном удалении владельцем короткое имя резервируется на 30 дней, после чего его может получить
другой издатель. Если применяется этот срок, ответ на удаление включает `slugReservedUntil`.
Скрытие модератором/администратором и удаление по соображениям безопасности таким образом не истекают.

Ответ на удаление:

```json
{ "ok": true, "slugReservedUntil": 1730000000000 }
```

Коды состояния:

- `200`: успешно
- `401`: не авторизован
- `403`: запрещено
- `404`: Skill/пользователь не найден
- `500`: внутренняя ошибка сервера

### `POST /api/v1/users/publisher`

Только для администраторов. Гарантирует существование издателя организации для идентификатора. Если идентификатор всё ещё указывает на
устаревшего общего пользователя/личного издателя, конечная точка сначала преобразует его в издателя организации.
Для вновь созданной организации укажите `memberHandle`; действующий администратор не добавляется в качестве участника.
По умолчанию `memberRole` имеет значение `owner`.

- Тело: `{ "handle": "openclaw", "displayName": "OpenClaw", "memberHandle": "alice", "memberRole": "owner", "trusted": true }`
- Ответ: `{ "ok": true, "publisherId": "...", "handle": "openclaw", "created": true, "migrated": false, "trusted": true, "member": { "userId": "...", "handle": "alice", "role": "owner" } }`

### `POST /api/v1/publishers`

Самостоятельное создание издателя организации с аутентификацией. Создаёт нового издателя организации и добавляет
вызывающую сторону в качестве владельца. Эта конечная точка не преобразует существующие пользовательские/личные идентификаторы и
не помечает издателя как доверенного/официального.

- Тело: `{ "handle": "opik", "displayName": "Opik" }`
- Ответ: `{ "ok": true, "publisherId": "...", "handle": "opik", "created": true, "trusted": false }`
- Возвращает `409`, если идентификатор уже используется издателем, пользователем или личным издателем.

### `POST /api/v1/users/reserve`

Только для администраторов. Резервирует корневые короткие имена и имена пакетов для законного владельца без публикации
выпуска. Имена пакетов становятся закрытыми пакетами-заполнителями без строк выпусков, поэтому тот же
владелец позднее сможет опубликовать настоящий выпуск плагина с кодом или плагина-пакета под этим именем.

- Тело: `{ "handle": "openclaw", "slugs": ["diffs"], "packageNames": ["@openclaw/diffs"], "reason": "reserved for official OpenClaw plugin" }`
- Ответ: `{ "ok": true, "succeeded": 2, "failed": 0, "results": [{ "kind": "slug", "name": "diffs", "ok": true, "action": "reserved" }] }`

### `POST /api/v1/users/publisher-recovery`

Только для администраторов. Восстанавливает личного издателя для проверенного замещающего субъекта GitHub OAuth
без изменения строк учётных записей Convex Auth. В запросе необходимо указать оба неизменяемых идентификатора
учётных записей провайдера GitHub; изменяемые идентификаторы используются только как проверочное ограничение для оператора.

По умолчанию конечная точка выполняет пробный запуск. Для применения восстановления требуются `dryRun: false` и
`confirmIdentityVerified: true` после того, как сотрудники независимо подтвердят преемственность между обоими
субъектами GitHub. Восстановление завершается отказом, если у текущего личного издателя целевого пользователя есть Skills,
пакеты или источники Skills в GitHub.
Восстановление также переносит устаревшие поля `ownerUserId` для Skills восстановленного издателя,
псевдонимов коротких имён Skills, пакетов, предупреждений инспектора пакетов и производных строк поискового дайджеста, чтобы
пути прямого владения соответствовали новым полномочиям издателя. Активное резервирование защищённого идентификатора
для восстановленного идентификатора также переназначается замещающему пользователю, чтобы последующая
синхронизация профиля не могла восстановить конкурирующие полномочия прежнего пользователя. Каждая основная таблица ограничена
100 строками на транзакцию применения; для более крупных восстановлений сначала необходимо использовать возобновляемый перенос владельца.
Источники Skills в GitHub привязаны к издателю и отмечаются в отчёте как проверенные, а не перезаписанные.

- Тело: `{ "handle": "gingiris", "nextUserHandle": "gingiris-1031", "previousGitHubProviderAccountId": "123", "nextGitHubProviderAccountId": "456", "reason": "Verified account continuity for issue #2555", "confirmIdentityVerified": true, "dryRun": false }`
- Ответ: `{ "ok": true, "dryRun": false, "recovered": true, "publisherId": "...", "handle": "gingiris", "previousUser": { "userId": "...", "handle": "gingiris", "nextHandle": "gingiris-recovered", "githubProviderAccountId": "123", "authAccountCount": 1 }, "nextUser": { "userId": "...", "handle": "gingiris-1031", "nextHandle": "gingiris", "githubProviderAccountId": "456", "authAccountCount": 1 }, "retiredPersonalPublisher": null, "resourceOwnerMigration": { "limitPerTable": 100, "skills": 1, "skillSlugAliases": 1, "packages": 0, "packageInspectorWarnings": 0, "githubSourcesChecked": 1, "handleReservations": 1 }, "identityVerified": true, "reason": "Verified account continuity for issue #2555" }`

### Конечные точки управления короткими именами владельца

- `POST /api/v1/skills/{slug}/rename`
  - Тело: `{ "newSlug": "new-canonical-slug" }`
  - Ответ: `{ "ok": true, "slug": "new-canonical-slug", "previousSlug": "old-slug" }`
- `POST /api/v1/skills/{slug}/merge`
  - Тело: `{ "targetSlug": "canonical-target-slug" }`
  - Ответ: `{ "ok": true, "sourceSlug": "old-slug", "targetSlug": "canonical-target-slug" }`

Примечания:

- Обе конечные точки требуют аутентификации с токеном API и работают только для владельца Skill.
- `rename` сохраняет предыдущее короткое имя в качестве псевдонима перенаправления.
- `merge` скрывает исходную публикацию и перенаправляет исходное короткое имя на целевую публикацию.

### `GET /api/v1/users`

Вывести список пользователей или найти их (только для администраторов).

Параметры запроса:

- `q` (необязательно): поисковый запрос
- `query` (необязательно): псевдоним для `q`
- `limit` (необязательно): максимальное количество результатов (по умолчанию 20, максимум 200)

Ответ:

```json
{
  "items": [
    {
      "userId": "users_...",
      "handle": "user_handle",
      "displayName": "Пользователь",
      "name": "Пользователь",
      "role": "moderator"
    }
  ],
  "total": 1
}
```

## Устаревшие конечные точки CLI (не рекомендуются)

По-прежнему поддерживаются для старых версий CLI:

- `GET /api/cli/whoami`
- `POST /api/cli/upload-url`
- `POST /api/cli/publish`
- `POST /api/cli/telemetry/install`
- `POST /api/cli/skill/delete`
- `POST /api/cli/skill/undelete`

План удаления см. в `DEPRECATIONS.md`.

`POST /api/cli/upload-url` возвращает `uploadUrl` и `uploadTicket`. Публикации
пакетов, которые помещают tar-архив ClawPack в промежуточное хранилище, должны отправить полученный идентификатор хранилища как
`clawpack`, а возвращённый билет — как `clawpackUploadTicket`.

## Обнаружение реестра (`/.well-known/clawhub.json`)

CLI может получать параметры реестра/аутентификации с сайта:

- `/.well-known/clawhub.json` (JSON, предпочтительно)
- `/.well-known/clawdhub.json` (устаревший вариант)

Схема:

```json
{ "apiBase": "https://clawhub.ai", "authBase": "https://clawhub.ai", "minCliVersion": "0.0.5" }
```

При самостоятельном размещении предоставляйте этот файл (либо явно задайте `CLAWHUB_REGISTRY`; устаревший вариант — `CLAWDHUB_REGISTRY`).
