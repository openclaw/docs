# Как работает ClawHub

ClawHub — это слой реестра для Skills и плагинов OpenClaw. Он дает пользователям
место для обнаружения пакетов, издателям — место для выпуска версий, а
OpenClaw — достаточно метаданных для безопасной установки и обновления этих пакетов.

## Записи реестра

Каждая публичная карточка является записью реестра с:

- владельцем и slug или именем пакета
- одной или несколькими опубликованными версиями
- метаданными, кратким описанием, файлами и указанием источника
- журналом изменений и информацией о тегах, например `latest`
- сигналами скачивания, установки и избранного
- статусом проверки безопасности и модерации

Страница карточки — каноническое место, где пользователи могут проверить, что, по
заявлению навыка или плагина, он делает, перед установкой.

## Skills

Навык — это версионируемый текстовый пакет, сосредоточенный вокруг `SKILL.md`. Он может включать
вспомогательные файлы, примеры, шаблоны и скрипты.

ClawHub читает frontmatter `SKILL.md`, чтобы понять имя навыка,
описание, требования, переменные окружения и метаданные. Точные
метаданные важны, потому что они помогают пользователям решить, стоит ли устанавливать навык, и
помогают автоматическим проверкам обнаруживать несоответствия между заявленным и наблюдаемым поведением.

См. [Формат навыка](/ru/clawhub/skill-format).

## Плагины

Плагины — это упакованные расширения OpenClaw. ClawHub хранит метаданные пакета,
информацию о совместимости, ссылки на источник, артефакты и записи версий.

Когда OpenClaw устанавливает плагин из ClawHub, он проверяет заявленные метаданные
совместимости перед установкой. Записи пакетов могут включать совместимость API,
минимальную версию Gateway, целевые хосты, требования к окружению и дайджесты
артефактов.

Используйте явный источник установки ClawHub, когда хотите, чтобы реестр был
источником истины:

```bash
openclaw plugins install clawhub:<package>
```

## Публикация

Публикация создает новую неизменяемую запись версии. Издатели используют CLI `clawhub`
для аутентифицированных рабочих процессов реестра:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Используйте пробные запуски, чтобы предварительно посмотреть разрешенную полезную нагрузку перед загрузкой. Затем публичные страницы
показывают опубликованные метаданные, файлы, указание источника и статус проверки.

## Установки и обновления

Команды установки OpenClaw используют ClawHub как источник пакетов:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw записывает метаданные источника установки, чтобы обновления позже могли разрешить тот же
пакет реестра. CLI ClawHub также поддерживает прямые рабочие процессы установки и
обновления навыков для пользователей, которым нужны управляемые реестром папки навыков вне
полноценного рабочего пространства OpenClaw.

## Состояние безопасности

ClawHub открыт для публикации, но выпуски все равно проходят через ограничения загрузки,
автоматические проверки, отчеты пользователей и действия модераторов.

Публичные страницы показывают сводки проверок, когда они доступны. Контент, который удержан, скрыт
или заблокирован, может исчезнуть из публичного поиска и потоков установки, оставаясь
видимым владельцу для диагностики.

См. [Безопасность](/clawhub/security), [Аудиты безопасности](/clawhub/security-audits),
[Модерация и безопасность аккаунта](/ru/clawhub/moderation) и
[Допустимое использование](/clawhub/acceptable-usage).

## Доступ к API

ClawHub предоставляет публичные API чтения для обнаружения, поиска, сведений о пакетах и
скачиваний. Сторонние каталоги могут использовать эти API, если они ссылаются на
каноническую карточку ClawHub, соблюдают лимиты частоты запросов и не создают впечатление одобрения.

См. [Публичный API](/ru/clawhub/api) и [HTTP API](/clawhub/http-api).
