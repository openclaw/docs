# Как работает ClawHub

ClawHub — это уровень реестра для OpenClaw Skills и плагинов. Он дает пользователям
место для поиска пакетов, издателям — место для выпуска версий, а
OpenClaw — достаточно метаданных, чтобы безопасно устанавливать и обновлять эти пакеты.

## Записи реестра

Каждая публичная публикация — это запись реестра, содержащая:

- владельца и slug или имя пакета
- одну или несколько опубликованных версий
- метаданные, сводку, файлы и указание источника
- changelog и информацию о тегах, например `latest`
- сигналы скачиваний, установок и звезд
- статус сканирования безопасности и модерации

Страница публикации — каноническое место, где пользователи могут проверить, что, по
заявлению навыка или плагина, он делает, прежде чем устанавливать его.

## Skills

Навык — это версионируемый текстовый набор, сосредоточенный вокруг `SKILL.md`. Он может включать
вспомогательные файлы, примеры, шаблоны и скрипты.

ClawHub читает frontmatter в `SKILL.md`, чтобы понять имя навыка,
описание, требования, переменные окружения и метаданные. Точные
метаданные важны, потому что они помогают пользователям решить, стоит ли устанавливать навык, и
помогают автоматическим проверкам обнаруживать несоответствия между заявленным и наблюдаемым поведением.

См. [Формат навыка](/ru/clawhub/skill-format).

## Плагины

Плагины — это упакованные расширения OpenClaw. ClawHub хранит метаданные пакетов,
информацию о совместимости, ссылки на источники, артефакты и записи версий.

Когда OpenClaw устанавливает плагин из ClawHub, он проверяет заявленные метаданные
совместимости перед установкой. Записи пакетов могут включать совместимость API,
минимальную версию gateway, целевые хосты, требования к окружению и дайджесты
артефактов.

Используйте явный источник установки ClawHub, когда хотите, чтобы реестр был
источником истины:

```bash
openclaw plugins install clawhub:<package>
```

## Публикация

Публикация создает новую неизменяемую запись версии. Издатели используют CLI `clawhub`
для аутентифицированных рабочих процессов реестра:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Используйте пробные запуски, чтобы предварительно просмотреть разрешенную полезную нагрузку перед загрузкой. Затем публичные страницы
показывают опубликованные метаданные, файлы, указание источника и статус сканирования.

## Установки и обновления

Команды установки OpenClaw используют ClawHub как источник пакетов:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw записывает метаданные источника установки, чтобы обновления могли позже разрешать тот же
пакет реестра. CLI ClawHub также поддерживает прямые рабочие процессы установки и
обновления навыков для пользователей, которым нужны управляемые реестром папки навыков вне
полного рабочего пространства OpenClaw.

## Состояние безопасности

ClawHub открыт для публикации, но выпуски по-прежнему проходят через шлюзы загрузки,
автоматические проверки, пользовательские отчеты и действия модераторов.

Публичные страницы показывают сводки сканирования, когда они доступны. Контент, который удержан, скрыт
или заблокирован, может исчезнуть из публичного поиска и потоков установки, оставаясь
видимым владельцу для диагностики.

См. [Безопасность](/clawhub/security), [Аудиты безопасности](/clawhub/security-audits),
[Модерация и безопасность аккаунта](/ru/clawhub/moderation) и
[Допустимое использование](/clawhub/acceptable-usage).

## Доступ к API

ClawHub предоставляет публичные API чтения для обнаружения, поиска, сведений о пакетах и
скачиваний. Сторонние каталоги могут использовать эти API, если они ссылаются обратно на
каноническую публикацию ClawHub, соблюдают ограничения частоты запросов и не создают впечатление одобрения.

См. [Публичный API](/ru/clawhub/api) и [HTTP API](/clawhub/http-api).
