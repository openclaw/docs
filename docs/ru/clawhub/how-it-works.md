# Как работает ClawHub

ClawHub — это слой реестра для Skills и плагинов OpenClaw. Он дает пользователям
место для поиска пакетов, издателям — место для выпуска версий, а OpenClaw —
достаточно метаданных для безопасной установки и обновления этих пакетов.

## Записи реестра

Каждая публичная карточка — это запись реестра, содержащая:

- владельца и слаг или имя пакета
- одну или несколько опубликованных версий
- метаданные, краткое описание, файлы и указание источника
- журнал изменений и информацию о тегах, например `latest`
- сигналы скачиваний, установок и звезд
- статус проверки безопасности и модерации

Страница карточки — каноническое место, где пользователи могут проверить, что
заявляет о себе навык или плагин перед установкой.

## Skills

Навык — это версионируемый текстовый пакет, построенный вокруг `SKILL.md`. Он
может включать вспомогательные файлы, примеры, шаблоны и скрипты.

ClawHub читает frontmatter в `SKILL.md`, чтобы понять имя навыка, описание,
требования, переменные окружения и метаданные. Точные метаданные важны, потому
что они помогают пользователям решить, стоит ли устанавливать навык, и помогают
автоматическим проверкам находить расхождения между заявленным и наблюдаемым
поведением.

См. [Формат навыка](/ru/clawhub/skill-format).

## Плагины

Плагины — это упакованные расширения OpenClaw. ClawHub хранит метаданные пакета,
информацию о совместимости, ссылки на исходный код, артефакты и записи версий.

Когда OpenClaw устанавливает плагин из ClawHub, он проверяет заявленные
метаданные совместимости перед установкой. Записи пакетов могут включать
совместимость API, минимальную версию Gateway, целевые хосты, требования к
окружению и дайджесты артефактов.

Используйте явный источник установки ClawHub, когда нужно, чтобы реестр был
источником истины:

```bash
openclaw plugins install clawhub:<package>
```

## Публикация

Публикация создает новую неизменяемую запись версии. Издатели используют CLI
`clawhub` для аутентифицированных рабочих процессов реестра:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Используйте пробные запуски, чтобы предварительно просмотреть разрешенную
полезную нагрузку перед загрузкой. Затем публичные страницы показывают
опубликованные метаданные, файлы, указание источника и статус проверки.

## Установки и обновления

Команды установки OpenClaw используют ClawHub как источник пакетов:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw записывает метаданные источника установки, чтобы обновления позже могли
разрешить тот же пакет реестра. CLI ClawHub также поддерживает рабочие процессы
прямой установки и обновления навыков для пользователей, которым нужны папки
навыков, управляемые реестром, вне полноценного рабочего пространства OpenClaw.

## Состояние безопасности

ClawHub открыт для публикации, но выпуски все равно проходят через шлюзы
загрузки, автоматические проверки, пользовательские жалобы и действия
модераторов.

Публичные страницы показывают сводки проверок, когда они доступны. Контент,
который удержан, скрыт или заблокирован, может исчезнуть из публичного поиска и
процессов установки, оставаясь видимым владельцу для диагностики.

См. [Безопасность](/clawhub/security), [Аудиты безопасности](/clawhub/security-audits),
[Модерация и безопасность аккаунта](/ru/clawhub/moderation) и
[Допустимое использование](/clawhub/acceptable-usage).

## Доступ к API

ClawHub предоставляет публичные API чтения для обнаружения, поиска, сведений о
пакетах и скачиваний. Сторонние каталоги могут использовать эти API, если они
ссылаются на каноническую карточку ClawHub, соблюдают ограничения частоты
запросов и не создают впечатления одобрения.

См. [Публичный API](/clawhub/api) и [HTTP API](/clawhub/http-api).
