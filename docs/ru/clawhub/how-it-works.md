# Как работает ClawHub

ClawHub — это реестр Skills и плагинов OpenClaw. Он предоставляет пользователям
место для поиска пакетов, издателям — место для выпуска версий, а OpenClaw —
достаточный объём метаданных для безопасной установки и обновления этих пакетов.

## Записи реестра

Каждая общедоступная публикация представляет собой запись реестра, содержащую:

- владельца и идентификатор или имя пакета
- одну или несколько опубликованных версий
- метаданные, краткое описание, файлы и сведения об источнике
- журнал изменений и сведения о тегах, таких как `latest`
- показатели скачиваний, установок и добавлений в избранное
- результаты проверки безопасности и статус модерации

Страница публикации — основное место, где пользователи могут ознакомиться с
заявленными возможностями Skills или плагина перед установкой.

## Skills

Skills представляет собой версионируемый текстовый пакет, основой которого
является `SKILL.md`. Он может включать вспомогательные файлы, примеры, шаблоны и
скрипты.

ClawHub считывает метаданные frontmatter из `SKILL.md`, чтобы определить название,
описание, требования, переменные окружения и другие метаданные Skills. Точность
метаданных важна, поскольку она помогает пользователям решить, стоит ли
устанавливать Skills, а автоматизированным проверкам — выявлять расхождения между
заявленным и наблюдаемым поведением.

См. [Формат Skills](/clawhub/skill-format).

## Плагины

Плагины представляют собой упакованные расширения OpenClaw. ClawHub хранит
метаданные пакетов, сведения о совместимости, ссылки на исходный код, артефакты и
записи о версиях.

Когда OpenClaw устанавливает плагин из ClawHub, перед установкой он проверяет
заявленные метаданные совместимости. Записи пакетов могут содержать сведения о
совместимости API, минимальной версии Gateway, целевых хостах, требованиях к
окружению и контрольных суммах артефактов.

Если реестр должен быть источником достоверных данных, явно укажите ClawHub в
качестве источника установки:

```bash
openclaw plugins install clawhub:<package>
```

## Публикация

При публикации создаётся новая неизменяемая запись версии. Для
аутентифицированных операций с реестром издатели используют CLI `clawhub`:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Используйте пробные запуски, чтобы предварительно просмотреть сформированные
данные перед отправкой. После этого на общедоступных страницах отображаются
опубликованные метаданные, файлы, сведения об источнике и статус проверки.

## Установка и обновление

Команды установки OpenClaw используют ClawHub как источник пакетов:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw сохраняет метаданные источника установки, чтобы при последующих
обновлениях можно было найти тот же пакет в реестре. CLI ClawHub также
поддерживает непосредственную установку и обновление Skills для пользователей,
которым нужны управляемые реестром каталоги Skills за пределами полноценного
рабочего пространства OpenClaw.

## Состояние безопасности

ClawHub открыт для публикаций, но выпуски по-прежнему проходят проверки при
загрузке, автоматизированные проверки и могут становиться предметом жалоб
пользователей и действий модераторов.

Если доступны сводные результаты проверки, они отображаются на общедоступных
страницах. Материалы, публикация которых приостановлена, скрыта или заблокирована,
могут исчезнуть из общедоступного поиска и процессов установки, оставаясь при
этом видимыми владельцу для диагностики.

См. [Безопасность](/ru/clawhub/security), [Аудиты безопасности](/clawhub/security-audits),
[Модерация и безопасность учётной записи](/ru/clawhub/moderation) и
[Допустимое использование](/clawhub/acceptable-usage).

## Доступ к API

ClawHub предоставляет общедоступные API чтения для поиска, просмотра сведений о
пакетах и скачивания. Сторонние каталоги могут использовать эти API при условии,
что они ссылаются на каноническую публикацию ClawHub, соблюдают ограничения
частоты запросов и не создают впечатления официального одобрения.

См. [Общедоступный API](/clawhub/api) и [HTTP API](/clawhub/http-api).
