# Как работает ClawHub

ClawHub — это слой реестра для Skills и plugins OpenClaw. Он дает пользователям
место для поиска пакетов, издателям — место для выпуска версий, а OpenClaw —
достаточно метаданных для безопасной установки и обновления этих пакетов.

## Записи реестра

Каждая публичная карточка — это запись реестра с:

- владельцем и slug или именем пакета
- одной или несколькими опубликованными версиями
- метаданными, кратким описанием, файлами и указанием источника
- журналом изменений и сведениями о тегах, такими как `latest`
- сигналами скачиваний, установок и звезд
- статусом проверки безопасности и модерации

Страница карточки — каноническое место, где пользователи могут проверить, что, по
заявлению skill или plugin, он делает, перед установкой.

## Skills

Skill — это версионируемый текстовый комплект, сосредоточенный вокруг `SKILL.md`. Он может включать
вспомогательные файлы, примеры, шаблоны и скрипты.

ClawHub читает frontmatter `SKILL.md`, чтобы понять имя skill,
описание, требования, переменные окружения и метаданные. Точные
метаданные важны, потому что они помогают пользователям решить, стоит ли устанавливать skill, и
помогают автоматическим проверкам находить несоответствия между заявленным и наблюдаемым поведением.

См. [Формат skill](/ru/clawhub/skill-format).

## Plugins

Plugins — это упакованные расширения OpenClaw. ClawHub хранит метаданные пакета,
сведения о совместимости, ссылки на исходный код, артефакты и записи версий.

Когда OpenClaw устанавливает plugin из ClawHub, он проверяет заявленные метаданные
совместимости перед установкой. Записи пакета могут включать совместимость API,
минимальную версию Gateway, целевые хосты, требования окружения и дайджесты
артефактов.

Используйте явный источник установки ClawHub, если хотите, чтобы реестр был
источником истины:

```bash
openclaw plugins install clawhub:<package>
```

## Публикация

Публикация создает новую неизменяемую запись версии. Издатели используют CLI `clawhub`
для аутентифицированных рабочих процессов реестра:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Используйте пробные запуски, чтобы предварительно просмотреть разрешенную полезную нагрузку перед загрузкой. Затем публичные страницы
показывают опубликованные метаданные, файлы, указание источника и статус проверки.

## Установки и обновления

Команды установки OpenClaw используют ClawHub как источник пакетов:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw записывает метаданные источника установки, чтобы обновления могли позже разрешать тот же
пакет реестра. CLI ClawHub также поддерживает прямые рабочие процессы установки и
обновления skill для пользователей, которым нужны управляемые реестром папки skill вне
полного рабочего пространства OpenClaw.

## Состояние безопасности

ClawHub открыт для публикации, но выпуски все равно проходят шлюзы загрузки,
автоматические проверки, пользовательские жалобы и действия модераторов.

Публичные страницы показывают сводки сканирования, когда они доступны. Контент, который удержан, скрыт
или заблокирован, может исчезнуть из публичного поиска и потоков установки, оставаясь
видимым владельцу для диагностики.

См. [Безопасность](/clawhub/security), [Аудиты безопасности](/clawhub/security-audits),
[Модерация и безопасность учетной записи](/ru/clawhub/moderation) и
[Допустимое использование](/clawhub/acceptable-usage).

## Доступ к API

ClawHub предоставляет публичные API чтения для обнаружения, поиска, сведений о пакетах и
скачиваний. Сторонние каталоги могут использовать эти API, когда они ссылаются на
каноническую карточку ClawHub, соблюдают ограничения частоты запросов и не создают впечатления одобрения.

См. [Публичный API](/clawhub/api) и [HTTP API](/clawhub/http-api).
