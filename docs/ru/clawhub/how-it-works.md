# Как работает ClawHub

ClawHub — это уровень реестра для Skills и плагинов OpenClaw. Он предоставляет пользователям
место для поиска пакетов, издателям — место для выпуска версий, а
OpenClaw — достаточно метаданных для безопасной установки и обновления этих пакетов.

## Записи реестра

Каждая публичная публикация представляет собой запись реестра, содержащую:

- владельца и слаг или имя пакета
- одну или несколько опубликованных версий
- метаданные, краткое описание, файлы и указание источника
- журнал изменений и информацию о тегах, например `latest`
- показатели скачиваний, установок и отметок звёздами
- результаты проверки безопасности и статус модерации

Страница публикации — это основное место, где пользователи могут проверить заявленное назначение навыка или
плагина перед его установкой.

## Skills

Навык — это версионируемый текстовый пакет, основой которого служит `SKILL.md`. Он может включать
вспомогательные файлы, примеры, шаблоны и скрипты.

ClawHub считывает frontmatter из `SKILL.md`, чтобы определить название навыка,
описание, требования, переменные окружения и метаданные. Точные
метаданные важны, поскольку помогают пользователям решить, следует ли устанавливать навык, а
автоматизированным проверкам — выявлять расхождения между заявленным и наблюдаемым поведением.

См. [Формат навыка](/ru/clawhub/skill-format).

## Плагины

Плагины — это упакованные расширения OpenClaw. ClawHub хранит метаданные пакетов,
информацию о совместимости, ссылки на исходный код, артефакты и записи версий.

Когда OpenClaw устанавливает плагин из ClawHub, он перед установкой проверяет
заявленные метаданные совместимости. Записи пакетов могут содержать сведения о совместимости API,
минимальной версии Gateway, целевых хостах, требованиях к окружению и дайджестах
артефактов.

Указывайте источник установки ClawHub явно, если реестр должен быть
источником достоверных данных:

```bash
openclaw plugins install clawhub:<package>
```

## Публикация

При публикации создаётся новая неизменяемая запись версии. Для аутентифицированных операций с реестром издатели используют
CLI `clawhub`:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Используйте пробные запуски, чтобы перед загрузкой просмотреть сформированные данные. После этого на публичных страницах
отображаются опубликованные метаданные, файлы, указание источника и статус проверки.

## Установка и обновления

Команды установки OpenClaw используют ClawHub как источник пакетов:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw сохраняет метаданные источника установки, чтобы при последующих обновлениях можно было найти тот же
пакет реестра. CLI ClawHub также поддерживает непосредственную установку и
обновление навыков для пользователей, которым нужны управляемые реестром папки навыков вне
полного рабочего пространства OpenClaw.

## Состояние безопасности

ClawHub открыт для публикации, однако выпуски всё равно проходят проверки при загрузке,
автоматизированные проверки, рассматриваются на основании жалоб пользователей и могут подвергаться действиям модераторов.

На публичных страницах отображаются сводки проверок, когда они доступны. Материалы, публикация которых приостановлена, которые скрыты
или заблокированы, могут исчезнуть из публичного поиска и процессов установки, оставаясь при этом
доступными владельцу для диагностики.

См. [Безопасность](/clawhub/security), [Аудиты безопасности](/clawhub/security-audits),
[Модерация и безопасность учётной записи](/ru/clawhub/moderation) и
[Допустимое использование](/clawhub/acceptable-usage).

## Доступ к API

ClawHub предоставляет публичные API чтения для обнаружения, поиска, получения сведений о пакетах и
скачивания. Сторонние каталоги могут использовать эти API при условии, что они ссылаются на
каноническую публикацию ClawHub, соблюдают ограничения частоты запросов и не создают впечатления официального одобрения.

См. [Публичный API](/clawhub/api) и [HTTP API](/clawhub/http-api).
