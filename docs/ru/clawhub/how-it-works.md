# Как работает ClawHub

ClawHub — это уровень реестра для навыков и плагинов OpenClaw. Он предоставляет пользователям
место для поиска пакетов, издателям — место для выпуска версий, а
OpenClaw — достаточно метаданных для безопасной установки и обновления этих пакетов.

## Записи реестра

Каждая общедоступная публикация представляет собой запись реестра, содержащую:

- владельца и короткое имя или имя пакета
- одну или несколько опубликованных версий
- метаданные, краткое описание, файлы и сведения об источнике
- журнал изменений и сведения о тегах, например `latest`
- показатели скачиваний, установок и добавлений в избранное
- результаты проверки безопасности и статус модерации

Страница публикации — это основное место, где пользователи могут проверить заявленные
возможности навыка или плагина перед его установкой.

## Навыки

Навык — это версионируемый текстовый набор, основой которого служит `SKILL.md`. Он может включать
вспомогательные файлы, примеры, шаблоны и скрипты.

ClawHub считывает метаданные frontmatter из `SKILL.md`, чтобы определить название навыка,
описание, требования, переменные окружения и метаданные. Точность
метаданных важна, поскольку помогает пользователям решить, стоит ли устанавливать навык, а
автоматизированным проверкам — обнаруживать расхождения между заявленным и наблюдаемым поведением.

См. [Формат навыка](/clawhub/skill-format).

## Плагины

Плагины — это упакованные расширения OpenClaw. ClawHub хранит метаданные пакетов,
сведения о совместимости, ссылки на исходный код, артефакты и записи о версиях.

Когда OpenClaw устанавливает плагин из ClawHub, перед установкой он проверяет
заявленные метаданные совместимости. Записи пакетов могут содержать сведения о совместимости API,
минимальной версии Gateway, целевых хостах, требованиях к окружению и
дайджестах артефактов.

Указывайте ClawHub как источник установки явно, если реестр должен быть
источником истины:

```bash
openclaw plugins install clawhub:<package>
```

## Публикация

При публикации создаётся новая неизменяемая запись версии. Для аутентифицированных операций
с реестром издатели используют CLI `clawhub`:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Используйте пробные запуски, чтобы просмотреть сформированные данные перед отправкой. После этого на общедоступных страницах
отображаются опубликованные метаданные, файлы, сведения об источнике и статус проверки.

## Установка и обновление

Команды установки OpenClaw используют ClawHub как источник пакетов:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw сохраняет метаданные источника установки, чтобы при последующих обновлениях можно было найти тот же
пакет реестра. CLI ClawHub также поддерживает прямую установку и
обновление навыков для пользователей, которым нужны управляемые реестром папки навыков вне
полноценного рабочего пространства OpenClaw.

## Состояние безопасности

ClawHub открыт для публикации, однако выпуски по-прежнему проходят проверки при загрузке,
автоматизированные проверки, рассматриваются по сообщениям пользователей и могут подвергаться действиям модераторов.

На общедоступных страницах отображаются сводки проверок, когда они доступны. Материалы, которые были задержаны, скрыты
или заблокированы, могут исчезнуть из общедоступного поиска и процессов установки, оставаясь
доступными владельцу для диагностики.

См. [Безопасность](/ru/clawhub/security), [Аудиты безопасности](/clawhub/security-audits),
[Модерация и безопасность учётной записи](/ru/clawhub/moderation) и
[Правила допустимого использования](/clawhub/acceptable-usage).

## Доступ к API

ClawHub предоставляет общедоступные API чтения для поиска, обзора, получения сведений о пакетах и
скачивания. Сторонние каталоги могут использовать эти API при условии, что они ссылаются на
каноническую публикацию в ClawHub, соблюдают ограничения частоты запросов и не создают впечатления официального одобрения.

См. [Общедоступный API](/clawhub/api) и [HTTP API](/clawhub/http-api).
