## Что это делает

Когда понимание аудио включено (или обнаружено автоматически), OpenClaw:

1. Находит первое аудиовложение (локальный путь или URL) и при необходимости загружает его.
2. Проверяет ограничение `maxBytes` перед отправкой каждой записи модели.
3. По порядку запускает первую подходящую запись модели (провайдер или CLI); если запись завершается с ошибкой или пропускается (из-за размера или тайм-аута), пробуется следующая.
4. При успешном выполнении заменяет `Body` блоком `[Audio]` и задаёт `{{Transcript}}`.

При успешной транскрипции `CommandBody`/`RawBody` также получают текст транскрипции, поэтому команды с косой чертой продолжают работать. При использовании `--verbose` в журналах отображается, когда выполняется транскрипция и когда она заменяет тело сообщения.

## Автообнаружение (по умолчанию)

Если модели не настроены и `tools.media.audio.enabled` не имеет значения `false`, OpenClaw выполняет автообнаружение в следующем порядке и останавливается на первом рабочем варианте:

1. **Активная модель ответа**, если её провайдер поддерживает понимание аудио.
2. **Настроенная аутентификация провайдера** — любая запись `models.providers.*` с доступной аутентификацией для провайдера, поддерживающего транскрипцию аудио. Эта проверка выполняется до проверки локальных CLI, поэтому настроенный ключ API всегда имеет приоритет над локальным исполняемым файлом в `PATH`.
   Приоритет при наличии нескольких настроенных провайдеров: Groq, OpenAI, xAI, Deepgram, Google, SenseAudio, ElevenLabs, Mistral.
3. **Локальные CLI** (только если не удалось определить аутентификацию провайдера). OpenClaw формирует упорядоченный список резервных вариантов:
   - `whisper-cli` — перед стандартными вариантами для CPU, только если более ранний вызов модели в текущем процессе обнаружил Metal или CUDA
   - `sherpa-onnx-offline` со стандартным провайдером CPU (требуется `SHERPA_ONNX_MODEL_DIR` с файлами `tokens.txt`, `encoder.onnx`, `decoder.onnx` и `joiner.onnx`)
   - `whisper-cli`, если сборка лишь поддерживает Metal/CUDA или выбранный бэкенд ещё не был фактически обнаружен
   - `parakeet-mlx` на Apple Silicon (поддерживает MLX; использование устройства остаётся неподтверждённым)
   - `whisper` (CLI на Python; автоматически загружает модели)

Сведения о происхождении установки или ссылки подтверждают наличие возможности, но не факт выполнения. Сами по себе они никогда не перемещают кандидата выше sherpa на CPU. OpenClaw не загружает модель во время настройки или проверок состояния только ради определения бэкенда.
Автоматически обнаруженный whisper.cpp оставляет включёнными обычные журналы запуска модели, чтобы OpenClaw мог зафиксировать строку вышестоящего компонента `using … backend`. Явно заданные записи CLI сохраняют настроенные для них параметры вывода.

Автообнаружение Gemini CLI для понимания мультимедиа было заменено резервным вариантом с изолированным CLI Antigravity (`agy`) для изображений и видео; для аудио резервные CLI-варианты, помимо перечисленных выше локальных исполняемых файлов, не используются.

Чтобы отключить автообнаружение, задайте `tools.media.audio.enabled: false`. Для индивидуальной настройки задайте `tools.media.audio.models`.

<Note>
Обнаружение исполняемых файлов в macOS/Linux/Windows выполняется по мере возможности. Убедитесь, что CLI находится в `PATH` (`~` разворачивается), либо явно задайте модель CLI с полным путём к команде.
</Note>

Проверьте локальный выбор без транскрипции аудио:

```bash
openclaw capability audio providers
openclaw doctor --lint --only core/doctor/local-audio-acceleration --severity-min info
```

В списке провайдеров победитель среди локальных резервных вариантов указывается отдельно от глобального выбора провайдера, вместе с полями доступного, запрошенного и обнаруженного бэкенда. После выполнения транскрипции `/status` показывает запрошенный или обнаруженный бэкенд в строке мультимедиа. Явные записи CLI в `tools.media.audio.models` по-прежнему обходят автоматический выбор; используйте параметры конкретного бэкенда, например `--provider=cuda` для sherpa или `--no-gpu`/`--device` для whisper.cpp.

## Примеры конфигурации

### Провайдер и резервный CLI (OpenAI + Whisper CLI)

```json5
{
  tools: {
    media: {
      audio: {
        enabled: true,
        maxBytes: 20971520,
        models: [
          { provider: "openai", model: "gpt-4o-transcribe" },
          {
            type: "cli",
            command: "whisper",
            args: ["--model", "base", "{{MediaPath}}"],
            timeoutSeconds: 45,
          },
        ],
      },
    },
  },
}
```

### Только провайдер с ограничением области действия

```json5
{
  tools: {
    media: {
      audio: {
        enabled: true,
        scope: {
          default: "allow",
          rules: [{ action: "deny", match: { chatType: "group" } }],
        },
        models: [{ provider: "openai", model: "gpt-4o-transcribe" }],
      },
    },
  },
}
```

### Только провайдер (Deepgram)

```json5
{
  tools: {
    media: {
      audio: {
        enabled: true,
        models: [{ provider: "deepgram", model: "nova-3" }],
      },
    },
  },
}
```

### Только провайдер (Mistral Voxtral)

```json5
{
  tools: {
    media: {
      audio: {
        enabled: true,
        models: [{ provider: "mistral", model: "voxtral-mini-latest" }],
      },
    },
  },
}
```

### Только провайдер (SenseAudio)

```json5
{
  tools: {
    media: {
      audio: {
        enabled: true,
        models: [{ provider: "senseaudio", model: "senseaudio-asr-pro-1.5-260319" }],
      },
    },
  },

```

### Отправка транскрипции в чат (по явному включению)

```json5
{
  tools: {
    media: {
      audio: {
        enabled: true,
        echoTranscript: true, // по умолчанию false
        echoFormat: '📝 "{transcript}"', // необязательно, поддерживает {transcript}
        models: [{ provider: "openai", model: "gpt-4o-transcribe" }],
      },
    },
  },
}
```

## Примечания и ограничения

- Аутентификация провайдера выполняется в стандартном порядке аутентификации моделей (профили аутентификации, переменные среды, `models.providers.*.apiKey`).
- Сведения о настройке Groq: [Groq](/ru/providers/groq).
- Deepgram использует `DEEPGRAM_API_KEY`, когда задано `provider: "deepgram"`. Сведения о настройке: [Deepgram](/ru/providers/deepgram).
- Сведения о настройке Mistral: [Mistral](/ru/providers/mistral).
- SenseAudio использует `SENSEAUDIO_API_KEY`, когда задано `provider: "senseaudio"`. Сведения о настройке: [SenseAudio](/ru/providers/senseaudio).
- Провайдеры аудио могут переопределять `baseUrl`, `headers` и `providerOptions` через `tools.media.audio`.
- Стандартное ограничение размера составляет 20 МБ (`tools.media.audio.maxBytes`). Аудио, превышающее это ограничение, пропускается для данной модели, после чего пробуется следующая запись.
- Аудиофайлы размером менее 1024 байт пропускаются до транскрипции провайдером или CLI.
- Стандартное значение `maxChars` для аудио **не задано** (полная транскрипция). Чтобы обрезать вывод, задайте `tools.media.audio.maxChars` или `maxChars` для отдельной записи.
- Стандартная модель при автообнаружении OpenAI — `gpt-4o-transcribe`; для более дешёвого и быстрого варианта задайте `model: "gpt-4o-mini-transcribe"`.
- Используйте `tools.media.audio.attachments` для обработки нескольких голосовых сообщений (`mode: "all"` вместе с `maxAttachments`, по умолчанию 1).
- Транскрипция доступна в шаблонах как `{{Transcript}}`.
- `tools.media.audio.echoTranscript` по умолчанию отключён; включите его, чтобы отправлять подтверждение с транскрипцией обратно в исходный чат до обработки агентом.
- `tools.media.audio.echoFormat` настраивает текст подтверждения (заполнитель: `{transcript}`; по умолчанию `📝 "{transcript}"`).
- Стандартный вывод CLI ограничен 5 МБ; вывод CLI должен быть кратким.
- В `args` CLI следует использовать `{{MediaPath}}` для локального пути к аудиофайлу. Запустите `openclaw doctor --fix`, чтобы перенести устаревшие заполнители `{input}` из старых конфигураций `audio.transcription.command` (выведенный из эксплуатации ключ: `audio.transcription`, заменён на `tools.media.audio.models`).
- `tools.media.concurrency` ограничивает количество задач обработки мультимедиа; это не планировщик GPU.

### Постоянно работающая локальная система распознавания речи

Автоматически обнаруженная локальная система распознавания речи по-прежнему запускает отдельный процесс для каждого запроса. Сейчас OpenClaw не управляет постоянно работающим сервером whisper.cpp, поскольку стандартный пакет Homebrew `whisper-cpp` отключает этот сервер, а пример из вышестоящего проекта не имеет настроенной ограниченной очереди допуска. Для безопасного включения постоянного жизненного цикла под управлением Plugin необходим поддерживаемый пакетированный рабочий процесс с проверками работоспособности и запуска, постоянным хранением модели в памяти, ограниченной очередью, отменой и тайм-аутами, работой без аутентификации только через local loopback и без резервного перехода в облако.

### Поддержка переменных среды прокси

Транскрипция аудио через провайдера учитывает стандартные переменные среды исходящего прокси в соответствии с семантикой `EnvHttpProxyAgent` из undici:

- `HTTPS_PROXY` / `https_proxy`
- `HTTP_PROXY` / `http_proxy`
- `ALL_PROXY` / `all_proxy`

Переменные в нижнем регистре имеют приоритет над переменными в верхнем; записи `NO_PROXY`/`no_proxy` (имена узлов, `*.suffix` или `host:port`) обходят прокси. Если переменные среды прокси не заданы, используется прямое исходящее соединение. Если настройка прокси завершается ошибкой (некорректный URL), OpenClaw записывает предупреждение в журнал и возвращается к прямому получению данных.

## Обнаружение упоминаний в группах

В каналах, поддерживающих предварительную обработку аудио, OpenClaw транскрибирует аудио **до** проверки упоминаний, если для группового чата задано `requireMention: true`. Благодаря этому голосовое сообщение без подписи может пройти проверку упоминания, если его транскрипция содержит настроенный шаблон упоминания. В документации конкретных каналов описаны транспорты, для которых требуется текстовое упоминание.

**Как это работает:**

1. Если голосовое сообщение не содержит текстового тела, а в группе требуются упоминания, OpenClaw выполняет предварительную транскрипцию первого аудиовложения.
2. Транскрипция проверяется на наличие шаблонов упоминания (например, `@BotName` или триггеров-эмодзи).
3. Если упоминание найдено, сообщение проходит через полный конвейер формирования ответа.

**Резервное поведение:** если предварительная транскрипция завершается ошибкой (тайм-аут, ошибка API и т. д.), для сообщения выполняется проверка упоминаний только в тексте, поэтому смешанные сообщения (текст и аудио) никогда не отбрасываются.

**Отключение для отдельной группы или темы Telegram:**

- Задайте `channels.telegram.groups.<chatId>.disableAudioPreflight: true`, чтобы пропустить предварительную проверку упоминаний в транскрипции для этой группы.
- Задайте `channels.telegram.groups.<chatId>.topics.<threadId>.disableAudioPreflight`, чтобы переопределить настройку для отдельной темы (`true` — пропустить, `false` — принудительно включить).
- Значение по умолчанию — `false` (предварительная обработка включена, когда выполняются условия проверки упоминания).

**Пример:** пользователь отправляет в группе Telegram с `requireMention: true` голосовое сообщение со словами «Привет, @Claude, какая погода?». Голосовое сообщение транскрибируется, упоминание обнаруживается, и агент отвечает.

## Подводные камни

- Для правил области действия применяется первое совпавшее правило; `chatType` нормализуется до `direct`, `group` или `channel`.
- Убедитесь, что CLI завершается с кодом 0 и выводит обычный текст; вывод JSON необходимо преобразовать с помощью `jq -r .text`.
- Известные режимы вывода в файл имеют приоритет: пустой или отсутствующий предполагаемый файл транскрипции означает отсутствие транскрипции вместо использования вывода хода выполнения CLI.
- Для `parakeet-mlx` используйте `--output-format txt` (или `all`) вместе с `--output-dir` и стандартным шаблоном вывода `{filename}`. Также учитываются переменные среды вышестоящего проекта `PARAKEET_OUTPUT_FORMAT` и `PARAKEET_OUTPUT_TEMPLATE`. OpenClaw читает `<output-dir>/<media-basename>.txt`; при стандартном формате `srt`, других форматах и пользовательских шаблонах вывода по-прежнему используется стандартный вывод.
- Устанавливайте разумные тайм-ауты (`timeoutSeconds`, по умолчанию 60 с), чтобы не блокировать очередь ответов.
- Для обнаружения упоминаний предварительная транскрипция обрабатывает только **первое** аудиовложение. Дополнительные аудиовложения обрабатываются на основном этапе понимания мультимедиа.
