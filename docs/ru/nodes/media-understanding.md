OpenClaw может создавать краткое описание входящих медиафайлов (изображений, аудио и видео) до запуска конвейера ответа, чтобы разбор команд и маршрутизация работали с коротким текстом, а не с необработанными байтами. Механизм распознавания автоматически обнаруживает локальные инструменты или ключи провайдеров; также можно явно настроить модели. Исходные медиафайлы всегда передаются модели обычным образом; если распознавание завершается с ошибкой или отключено, поток ответа продолжается без изменений.

Плагины поставщиков регистрируют метаданные возможностей: какой провайдер поддерживает тот или иной тип медиа, модель по умолчанию и приоритет. Ядро OpenClaw управляет общей конфигурацией `tools.media`, порядком резервных вариантов и интеграцией с конвейером ответа.

## Принцип работы

<Steps>
  <Step title="Сбор вложений">
    Собираются входящие вложения (`MediaPaths`, `MediaUrls`, `MediaTypes`).
  </Step>
  <Step title="Выбор для каждой возможности">
    Для каждой включённой возможности (изображение, аудио или видео) вложения выбираются согласно политике `attachments` (по умолчанию — только первое вложение).
  </Step>
  <Step title="Выбор модели">
    Выбирается первая подходящая запись модели с учётом размера, возможностей и доступности аутентификации.
  </Step>
  <Step title="Переход к резервному варианту при сбое">
    Если модель возвращает ошибку, превышает время ожидания или размер медиа превышает `maxBytes`, используется следующая запись.
  </Step>
  <Step title="Применение при успешном выполнении">
    `Body` преобразуется в блок `[Image]`, `[Audio]` или `[Video]`. Для аудио также задаётся `{{Transcript}}`; при разборе команд используется текст подписи, если он есть, иначе — транскрипция. Подписи сохраняются внутри блока как `User text:`.
  </Step>
</Steps>

## Конфигурация

`tools.media` содержит общий список моделей и переопределения для отдельных возможностей:

```json5
{
  tools: {
    media: {
      concurrency: 2, // max concurrent capability runs (default)
      models: [/* shared list, gate with capabilities */],
      image: {/* optional overrides */},
      audio: {
        /* optional overrides */
        echoTranscript: true,
        echoFormat: '📝 "{transcript}"',
      },
      video: {/* optional overrides */},
    },
  },
}
```

Ключи для отдельных возможностей (`image`/`audio`/`video`):

| Ключ                                            | Тип       | Значение по умолчанию                                | Примечания                                                                                       |
| ----------------------------------------------- | --------- | ---------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| `enabled`                                       | `boolean` | автоматически (`false` отключает)                    | Установите `false`, чтобы отключить автоматическое обнаружение для этой возможности              |
| `models`                                        | массив    | отсутствует                                          | Имеет приоритет над общим списком `tools.media.models`                                           |
| `prompt`                                        | `string`  | `"Describe the {media}."` (+ указание `maxChars`)    | По умолчанию только для изображений и видео                                                      |
| `maxChars`                                      | `number`  | `500` (изображение/видео), не задано (аудио)         | Вывод обрезается, если модель возвращает больше символов                                         |
| `maxBytes`                                      | `number`  | изображение `10485760`, аудио `20971520`, видео `52428800` | При слишком большом размере медиа выполняется переход к следующей модели                  |
| `timeoutSeconds`                                | `number`  | `60` (изображение/аудио), `120` (видео)              |                                                                                                  |
| `language`                                      | `string`  | не задано                                            | Подсказка о языке для транскрибирования аудио                                                    |
| `baseUrl`/`headers`/`providerOptions`/`request` | -         | -                                                    | Переопределения запроса провайдера; см. [Инструменты и пользовательские провайдеры](/ru/gateway/config-tools) |
| `attachments`                                   | объект    | `{ mode: "first", maxAttachments: 1 }`               | См. [Политика вложений](#attachment-policy)                                                      |
| `scope`                                         | объект    | не задано                                            | Ограничение по каналу, `chatType` или `keyPrefix`                                                |
| `echoTranscript`                                | `boolean` | `false`                                              | Только для аудио: отправить транскрипцию обратно в чат до обработки агентом                      |
| `echoFormat`                                    | `string`  | `'📝 "{transcript}"'`                                | Только для аудио: заполнитель `{transcript}`                                                     |

Параметры Deepgram задаются в `providerOptions.deepgram` (поле верхнего уровня `deepgram: { detectLanguage, punctuate, smartFormat }` устарело, но по-прежнему считывается).

### Записи моделей

Каждая запись `models[]` является записью **провайдера** (по умолчанию) или записью **CLI**:

<Tabs>
  <Tab title="Запись провайдера">
    ```json5
    {
      type: "provider", // default if omitted
      provider: "openai",
      model: "gpt-5.6-sol",
      prompt: "Describe the image in <= 500 chars.",
      maxChars: 500,
      maxBytes: 10485760,
      timeoutSeconds: 60,
      capabilities: ["image"], // optional, for multi-modal shared entries
      profile: "vision-profile",
      preferredProfile: "vision-fallback",
    }
    ```
  </Tab>
  <Tab title="Запись CLI">
    ```json5
    {
      type: "cli",
      command: "gemini",
      args: [
        "-m",
        "gemini-3-flash",
        "--allowed-tools",
        "read_file",
        "Read the media at {{MediaPath}} and describe it in <= {{MaxChars}} characters.",
      ],
      maxChars: 500,
      maxBytes: 52428800,
      timeoutSeconds: 120,
      capabilities: ["video", "image"],
    }
    ```

    В шаблонах CLI также можно использовать `{{MediaDir}}` (каталог, содержащий медиафайл), `{{OutputDir}}` (временный каталог, созданный для этого запуска) и `{{OutputBase}}` (базовый путь временного файла без расширения).

  </Tab>
</Tabs>

### Учётные данные провайдера

Для распознавания медиа через провайдера используется тот же порядок разрешения аутентификации, что и для обычных вызовов моделей: профили аутентификации, переменные окружения, затем `models.providers.<providerId>.apiKey`. Записи `tools.media.*.models[]` не принимают встроенное поле `apiKey`.

```json5
{
  models: {
    providers: {
      openai: { apiKey: "<OPENAI_API_KEY>" },
      moonshot: { apiKey: "<MOONSHOT_API_KEY>" },
    },
  },
}
```

Сведения о профилях, переменных окружения и пользовательских базовых URL см. в разделе [Инструменты и пользовательские провайдеры](/ru/gateway/config-tools).

## Правила и поведение

- Если размер медиа превышает `maxBytes`, эта модель пропускается и используется следующая.
- Аудиофайлы размером менее 1024 байт считаются пустыми или повреждёнными и пропускаются до транскрибирования; вместо этого агент получает детерминированную транскрипцию-заполнитель.
- Если активная основная модель для изображений уже имеет встроенную поддержку компьютерного зрения, OpenClaw пропускает блок сводки `[Image]` и передаёт исходное изображение непосредственно модели. MiniMax является исключением: `minimax`, `minimax-cn`, `minimax-portal` и `minimax-portal-cn` всегда направляют распознавание изображений через принадлежащий плагину медиапровайдер `MiniMax-VL-01`, даже если устаревшие метаданные чата MiniMax M2.x заявляют поддержку изображений на входе (встроенная поддержка компьютерного зрения считается доступной только у `MiniMax-M3` и более поздних версий).
- Если основная модель Gateway/WebChat поддерживает только текст, вложения-изображения сохраняются как вынесенные ссылки `media://inbound/*`, чтобы инструменты для изображений и PDF или настроенная модель изображений всё ещё могли их проанализировать, а вложение не было потеряно.
- Явная команда `openclaw infer image describe --file <path> --model <provider/model>` (псевдоним: `openclaw capability image describe`) запускает указанные провайдер и модель с поддержкой изображений напрямую, включая ссылки Ollama, например `ollama/qwen2.5vl:7b`, если соответствующая модель с поддержкой изображений настроена в `models.providers.ollama.models[]`.
- Если `<capability>.enabled` не равно `false`, но модели не настроены, OpenClaw пытается использовать активную модель ответа, когда её провайдер поддерживает эту возможность.

### Автоматическое обнаружение (по умолчанию)

Если `tools.media.<capability>.enabled` не равно `false` и модели не настроены, OpenClaw последовательно пробует следующие варианты и останавливается на первом работающем:

<Steps>
  <Step title="Настроенная модель изображений (только для изображений)">
    Основная и резервные ссылки `agents.defaults.imageModel`, если активная модель ответа ещё не имеет встроенной поддержки компьютерного зрения. Предпочтительны ссылки `provider/model`; ссылки без указания провайдера дополняются данными из настроенных записей моделей провайдеров с поддержкой изображений, только если совпадение однозначно.
  </Step>
  <Step title="Активная модель ответа">
    Активная модель ответа, если её провайдер поддерживает соответствующую возможность.
  </Step>
  <Step title="Аутентификация провайдера (только для аудио, перед локальными CLI)">
    Настроенные записи `models.providers.*`, поддерживающие аудио, проверяются перед локальными CLI. Порядок приоритета встроенных провайдеров (при равном приоритете используется алфавитный порядок идентификаторов провайдеров): Groq/OpenAI &rarr; xAI &rarr; Deepgram &rarr; OpenRouter &rarr; Google/SenseAudio &rarr; Deepinfra/ElevenLabs &rarr; Mistral.
  </Step>
  <Step title="Локальные CLI (только для аудио)">
    Готовые локальные исполняемые файлы образуют упорядоченный список резервных вариантов:
    - `whisper-cli` используется первым, только если при предыдущем вызове модели в текущем процессе был обнаружен Metal или CUDA
    - ориентированный на CPU `sherpa-onnx-offline` (требуется `SHERPA_ONNX_MODEL_DIR` с файлами `tokens.txt`/`encoder.onnx`/`decoder.onnx`/`joiner.onnx`)
    - `whisper-cli`, если ускорение лишь поддерживается сборкой или ещё не было обнаружено
    - `parakeet-mlx` на Apple Silicon (поддерживает MLX, использование устройства не наблюдалось)
    - `whisper` (CLI на Python; по умолчанию использует модель `turbo`, загружаемую автоматически)

    Результат проверки возможностей серверной части кэшируется, при этом модель не загружается. Поддержка при сборке, запрошенные флаги серверной части и серверная часть, фактически обнаруженная при реальном вызове, учитываются отдельно. Автоматически обнаруженный whisper.cpp оставляет журналы запуска модели включёнными, чтобы можно было записать строку выбранной серверной части от вышестоящего компонента. Явные записи CLI сохраняют настроенный порядок, флаги серверной части и флаги вывода.

  </Step>
  <Step title="Аутентификация провайдера (изображение/видео)">
    Настроенные записи `models.providers.*`, поддерживающие соответствующую возможность, проверяются перед встроенным порядком резервных вариантов. Провайдеры, настроенные только для изображений и содержащие модель с поддержкой изображений, автоматически регистрируются для распознавания медиа, даже если они не являются встроенными плагинами поставщиков.

    Порядок приоритета встроенных провайдеров (при равном приоритете используется алфавитный порядок идентификаторов провайдеров):
    - Изображение: Anthropic/OpenAI &rarr; Google &rarr; MiniMax &rarr; Deepinfra &rarr; MiniMax Portal &rarr; Z.AI
    - Видео: Google &rarr; Qwen &rarr; Moonshot

  </Step>
  <Step title="CLI Antigravity (только для изображений и видео)">
    Первый установленный исполняемый файл `agy` или `antigravity` (можно переопределить с помощью `OPENCLAW_ANTIGRAVITY_CLI`), изолированный в каталоге медиафайла.
  </Step>
</Steps>

Чтобы отключить автоматическое обнаружение для определённой возможности:

```json5
{
  tools: {
    media: {
      audio: {
        enabled: false,
      },
    },
  },
}
```

<Note>
Обнаружение исполняемых файлов в macOS/Linux/Windows выполняется по возможности; убедитесь, что CLI доступен через `PATH` (`~` раскрывается), либо задайте явную запись модели CLI с полным путём к команде.
</Note>

### Поддержка прокси-сервера (вызовы провайдера для аудио и видео)

Распознавание **аудио** и **видео** через провайдера учитывает стандартные переменные окружения исходящего прокси-сервера, включая правила обхода `NO_PROXY`/`no_proxy`: `HTTPS_PROXY`, `HTTP_PROXY`, `ALL_PROXY`, `https_proxy`, `http_proxy`, `all_proxy`. Переменные в нижнем регистре имеют приоритет над переменными в верхнем регистре. Если ни одна из них не задана, распознавание медиа использует прямое исходящее подключение; если значение прокси-сервера имеет неверный формат, OpenClaw записывает предупреждение в журнал и выполняет прямую загрузку. Распознавание изображений не использует этот путь прокси-сервера.

## Возможности

Задайте `capabilities` в записи `models[]`, чтобы ограничить её определёнными типами медиа. Для общих списков OpenClaw определяет значения по умолчанию для каждого встроенного провайдера:

| Поставщик                                                                | Возможности           |
| ------------------------------------------------------------------------ | --------------------- |
| `openai`, `anthropic`, `minimax`                                         | изображения           |
| `minimax-portal`                                                         | изображения           |
| `moonshot`                                                               | изображения + видео   |
| `openrouter`                                                             | изображения + аудио   |
| `google` (Gemini API)                                                    | изображения + аудио + видео |
| `qwen`                                                                   | изображения + видео   |
| `deepinfra`                                                              | изображения + аудио   |
| `mistral`                                                                | аудио                 |
| `zai`                                                                    | изображения           |
| `groq`, `xai`, `deepgram`, `senseaudio`                                  | аудио                 |
| Любой каталог `models.providers.<id>.models[]` с моделью, поддерживающей изображения | изображения           |

Для записей CLI явно задавайте `capabilities`, чтобы избежать неожиданных совпадений; если параметр не указан, запись считается подходящей для каждого списка возможностей, в котором она присутствует.

</Note>

## Рекомендации по выбору модели

- Если важны качество и безопасность, выбирайте для каждой мультимедийной возможности наиболее мощную модель текущего поколения.
- Для агентов с поддержкой инструментов, обрабатывающих недоверенные входные данные, избегайте старых или менее мощных мультимедийных моделей.
- Для обеспечения доступности сохраняйте как минимум один резервный вариант для каждой возможности: качественную модель и более быструю или дешёвую модель.
- Резервные варианты CLI (`whisper-cli`, `whisper`, `gemini`) помогают, когда API поставщиков недоступны.
- Известные режимы вывода в файл имеют приоритет: если предполагаемый файл транскрипции пуст или отсутствует, транскрипция не создаётся, а вывод о ходе выполнения CLI не используется как резервный источник.
- `parakeet-mlx`: используйте `--output-format txt` (или `all`) вместе с `--output-dir` и стандартным шаблоном вывода `{filename}`. Также учитываются переменные окружения вышестоящего проекта `PARAKEET_OUTPUT_FORMAT` и `PARAKEET_OUTPUT_TEMPLATE`. OpenClaw читает `<output-dir>/<media-basename>.txt`; при стандартном формате `srt`, других форматах и пользовательских шаблонах вывода по-прежнему используется stdout.

## Политика обработки вложений

Параметр `attachments` для каждой возможности определяет, какие вложения обрабатываются:

<ParamField path="mode" type='"first" | "all"' default="first">
  Обрабатывать только первое выбранное вложение или все вложения.
</ParamField>
<ParamField path="maxAttachments" type="number" default="1">
  Ограничить количество обрабатываемых вложений.
</ParamField>
<ParamField path="prefer" type='"first" | "last" | "path" | "url"'>
  Предпочтительный способ выбора среди подходящих вложений.
</ParamField>

При `mode: "all"` результаты помечаются как `[Изображение 1/2]`, `[Аудио 2/2]` и т. д.

### Извлечение содержимого файловых вложений

- Извлечённый из файла текст перед добавлением в мультимедийный промпт оборачивается как недоверенное внешнее содержимое с помощью граничных маркеров вида `<<<EXTERNAL_UNTRUSTED_CONTENT id="...">>>` / `<<<END_EXTERNAL_UNTRUSTED_CONTENT id="...">>>` и строки метаданных `Source: External`.
- В этом пути намеренно отсутствует длинный баннер `SECURITY NOTICE:`, чтобы мультимедийный промпт оставался коротким; граничные маркеры и метаданные по-прежнему применяются.
- Для файла, из которого невозможно извлечь текст, используется `[Нет текста для извлечения]`.
- Если для PDF используется резервный вариант с визуализированными изображениями страниц, OpenClaw передаёт эти изображения моделям ответа с поддержкой компьютерного зрения и сохраняет заполнитель `[Содержимое PDF преобразовано в изображения]` в блоке файла.

## Вывод состояния

Когда выполняется анализ мультимедиа, `/status` содержит строку сводки для каждой возможности:

```
📎 Media: image ok (openai/gpt-5.6-sol) · audio ok (whisper-cli observed=metal)
```

Для предварительной инвентаризации выполните `openclaw capability audio providers`. В локальных строках победитель среди локальных резервных вариантов отображается отдельно от глобального выбора поставщика, готовности и отдельных полей поддерживаемого, запрошенного и фактически используемого бэкенда. Тот же локальный выбор доступен как информационный результат диагностики:

```bash
openclaw doctor --lint --only core/doctor/local-audio-acceleration --severity-min info
```

## Примечания

- Анализ выполняется по мере возможности. Ошибки не блокируют ответы.
- Вложения всё равно передаются моделям, даже если анализ отключён.
- Используйте `scope`, чтобы ограничить области выполнения анализа, например только личными сообщениями.
