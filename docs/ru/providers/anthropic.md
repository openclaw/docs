Anthropic разрабатывает семейство моделей **Claude**. OpenClaw поддерживает два способа аутентификации:

- **Ключ API** — прямой доступ к API Anthropic с оплатой по факту использования (модели `anthropic/*`)
- **Claude CLI** — повторное использование существующей авторизации Claude Code на том же хосте

## Отслеживание использования и затрат

OpenClaw обнаруживает доступные учетные данные Anthropic и выбирает соответствующий способ отображения использования:

- Для учетных данных подписки или настройки Claude отображаются периоды квот и необязательный бюджет дополнительного использования.
- `ANTHROPIC_ADMIN_KEY` или `ANTHROPIC_ADMIN_API_KEY` позволяет просматривать в разделе **Usage** интерфейса Control UI сообщаемые провайдером затраты организации и использование Messages API за 30 дней, включая ежедневные расходы, общие объемы токенов и кэша, наиболее используемые модели и категории затрат.
- Учетные данные `sk-ant-admin...`, сохраненные в профиле провайдера Anthropic, автоматически распознаются как ключ Admin API.

История затрат Admin API поступает из [API использования и затрат](https://platform.claude.com/docs/en/manage-claude/usage-cost-api) Anthropic. Это фактические счета провайдера, отдельные от расчетной стоимости OpenClaw, вычисляемой на основе сеансов.

<Warning>
Бэкенд Claude CLI в OpenClaw запускает установленный Claude Code CLI в
неинтерактивном режиме вывода (`claude -p`). В текущей документации Anthropic по Claude Code
этот режим описывается как использование Agent SDK или программный доступ. В обновлении поддержки от 15 июня 2026 года
Anthropic приостановила объявленное изменение раздельной тарификации Agent SDK: использование Claude
Agent SDK, `claude -p` и сторонних приложений по-прежнему учитывается
в лимитах использования активной подписки, а ранее объявленный ежемесячный кредит Agent SDK
недоступен, пока Anthropic пересматривает этот план.

Интерактивный Claude Code по-прежнему использует лимиты активного плана Claude.
Аутентификация по ключу API оплачивается напрямую по факту использования и не зависит от этого плана.
Для долгосрочно работающих хостов Gateway, общей автоматизации и предсказуемых
производственных расходов используйте ключ API Anthropic.

Текущие статьи поддержки Anthropic могут изменить это поведение без
выпуска новой версии OpenClaw:

- [Справочник по Claude Code CLI](https://code.claude.com/docs/en/cli-usage)
- [Использование Claude Agent SDK с планом Claude](https://support.claude.com/en/articles/15036540-use-the-claude-agent-sdk-with-your-claude-plan)
- [Использование Claude Code с планом Pro или Max](https://support.claude.com/en/articles/11145838-use-claude-code-with-your-pro-or-max-plan)
- [Использование Claude Code с планом Team или Enterprise](https://support.claude.com/en/articles/11845131-using-claude-code-with-your-team-or-enterprise-plan)
- [Управление затратами Claude Code](https://code.claude.com/docs/en/costs)

</Warning>

    ### Получение токена настройки

    Запустите `claude setup-token` на любом компьютере с установленным Claude Code. Команда выведет
    долгоживущий токен, начинающийся с `sk-ant-oat01-`.

    Во время первоначальной настройки вставьте токен в приложении macOS, выбрав
    **Anthropic setup-token** в разделе **Connect with an API key or token**, либо выполните:

    ```bash
    openclaw models auth login --provider anthropic --method setup-token
    ```

    ### Пример конфигурации

    Предпочтительно использовать каноническую ссылку на модель Anthropic вместе с переопределением среды выполнения CLI:

    ```json5
    {
      agents: {
        defaults: {
          model: { primary: "anthropic/claude-opus-4-8" },
          models: {
            "anthropic/claude-opus-4-8": {
              agentRuntime: { id: "claude-cli" },
            },
          },
        },
      },
    }
    ```

    Устаревшие ссылки на модели `claude-cli/claude-opus-4-7` по-прежнему работают
    для совместимости, но в новой конфигурации выбор провайдера и модели следует задавать как
    `anthropic/*`, а бэкенд выполнения — в политике среды выполнения провайдера или модели.

    ### Тарификация и `claude -p`

    Для запусков Claude CLI OpenClaw использует неинтерактивный путь `claude -p` Claude Code.
    В настоящее время Anthropic считает этот путь использованием Agent SDK или программным доступом:

    - В обновлении поддержки от 15 июня 2026 года Anthropic приостановила ранее объявленный
      отдельный план кредитов Agent SDK.
    - Использование Claude Agent SDK, `claude -p` и сторонних приложений
      в рамках подписки по-прежнему учитывается в лимитах активной подписки.
    - Ранее объявленный ежемесячный кредит Agent SDK недоступен, пока
      Anthropic пересматривает этот план.
    - Для входа через Console или ключ API применяется оплата API по факту использования, и такой способ
      не предоставляет кредит Agent SDK подписки.

    Уведомление о приостановке см. в [статье Anthropic о плане Agent SDK](https://support.claude.com/en/articles/15036540-use-the-claude-agent-sdk-with-your-claude-plan),
    а сведения о поведении подписок — в статьях о планах Claude Code:
    [Pro/Max](https://support.claude.com/en/articles/11145838-use-claude-code-with-your-pro-or-max-plan)
    и
    [Team/Enterprise](https://support.claude.com/en/articles/11845131-use-claude-code-with-your-team-or-enterprise-plan).

    Anthropic может изменять тарификацию Claude Code и поведение ограничений частоты запросов без
    выпуска новой версии OpenClaw. Если важна предсказуемость затрат, проверяйте `claude auth status`, `/status` и
    документацию Anthropic по приведенным ссылкам.

## Сеансы Claude на разных компьютерах

Встроенный плагин Anthropic добавляет группу **Claude Code** на обычную боковую панель
сеансов. Строки открываются в обычной панели чата. Плагин обнаруживает неархивированные сеансы Claude
Code на Gateway и подключенных хостах Node:

- Сеансы Claude CLI извлекаются из действительных записей индекса проектов и текущих файлов JSONL,
  в ограниченном префиксе метаданных которых указан не являющийся боковой цепочкой сеанс `sdk-cli`
  в каталоге `~/.claude/projects/`.
- Для сеансов Claude Desktop используются заголовок Desktop, время активности и
  состояние архива, если его метаданные указывают на тот же идентификатор сеанса Claude Code.
- Сеанс, существующий только в CLI, не имеет флага архива, поэтому остается видимым, пока
  доступна его расшифровка.

Для обнаружения дополнительная конфигурация OpenClaw не требуется. Плагин Anthropic
встроен и включен по умолчанию; нативный узел macOS объявляет доступные только для чтения
команды сеансов Claude, если существует локальный каталог `~/.claude/projects/`.
Подтвердите обновление сопряжения узла при первом появлении этих команд.

На боковой панели строки группируются по хостам Gateway или сопряженных узлов; сначала отображается
самая новая ограниченная страница с каждого хоста, а обновление выполняется с обычным
интервалом 30 секунд. Используйте **Загрузить больше сеансов** под группой каталога, чтобы добавить следующую страницу
для каждого хоста с дополнительной историей; добавленные строки остаются видимыми и
при обновлении повторно загружаются до той же глубины. Клиенты каталога используют
`sessions.catalog.list`; при открытии строки используется `sessions.catalog.read`.

При передаче управления терминалу `claude` определяется по PATH оболочки входа
пользователя соответствующего хоста до PATH службы или демона. Благодаря этому запущенные приложением сеансы используют
тот же Claude CLI, который доступен оператору в обычном терминале.

При выборе строки сначала загружается самая новая страница расшифровки. **Загрузить более старые
элементы расшифровки** использует непрозрачный байтовый курсор и читает следующий ограниченный раздел из
файла JSONL вместо загрузки всей истории. Обычное содержимое пользователя, ассистента,
рассуждений, вызовов инструментов и результатов инструментов сохраняется. Отдельный элемент,
превышающий безопасный предел узла или Gateway, явно помечается как обрезанный.

Если строка `claude-cli` относится к локальному Gateway, ввод текста в обычном редакторе вызывает
`sessions.catalog.continue`. OpenClaw повторно разрешает локальную запись каталога,
создает или повторно использует нативный сеанс с зафиксированной моделью, импортирует не более 200 видимых
элементов или 512 КиБ и инициализирует привязку Claude CLI. Первый ход возобновляется с
`--fork-session`; Claude назначает ответвлению новый идентификатор сеанса, поэтому последующие ходы используют
ответвление, а исходный сеанс остается неизменным.

Безголовый хост узла также может разрешить продолжение своих строк Claude CLI, включив
следующий локальный параметр узла и перезапустив хост узла:

```json5
{
  nodeHost: {
    agentRuns: {
      claude: { enabled: true },
    },
  },
}
```

Узел объявляет `agent.cli.claude.run.v1`, только если параметр включен
и локальный исполняемый файл `claude` успешно разрешается. OpenClaw повторно разрешает запись каталога
на этом узле, импортирует ту же ограниченную историю и привязывает принятый
сеанс к узлу и рабочему каталогу, указанному каталогом. Каждый ход запускает
реальный процесс `claude -p` узла, используя файлы Claude и учетную запись этого узла. Политика
подтверждения выполнения команд узла продолжает действовать; Gateway не может принудительно включить эту возможность.

Продолжение на узле версии v1 допускает только один запуск. Оно не включает конфигурацию MCP обратной связи с Gateway и
аргументы плагина Skills Gateway, не выполняет повторную инициализацию из расшифровки Gateway и
отклоняет вложения и изображения. Строки Claude Desktop остаются доступными только для просмотра. Узлы
нативного приложения macOS также остаются доступными только для просмотра, пока приложение не объявит команду запуска.

<Note>
Сеансы Claude на сопряженных узлах остаются доступными только для чтения, если безголовый узел явно
не объявляет `agent.cli.claude.run.v1`. OpenClaw никогда не изменяет метаданные Claude Desktop
и не архивирует сеансы Claude. Для страницы требуется подключение оператора
с областью действия записи, поскольку она использует аутентифицированный `node.invoke`; операции перечисления и чтения
остаются доступными только для чтения даже на узле с включенным продолжением.
</Note>

Сведения о команде узла и границе безопасности см. в разделе [Узлы: сеансы и расшифровки Claude](/ru/nodes#claude-sessions-and-transcripts).

## Значения рассуждений по умолчанию (Claude Sonnet 5, Mythos 5, Fable 5, 4.8 и 4.6)

`anthropic/claude-sonnet-5` по умолчанию использует адаптивное мышление с уровнем усилий `high`.
Используйте `/think off`, чтобы отключить мышление, или `/think xhigh|max` для более
высоких встроенных уровней усилий модели. OpenClaw не указывает задаваемые вручную бюджеты
мышления, пользовательские параметры сэмплирования, предварительное заполнение ответов
ассистента и Priority Tier для Sonnet 5, поскольку Anthropic не поддерживает эти возможности
запросов для данной модели.
В каталоге до 31 августа 2026 года включительно используются вводные тарифы Anthropic
`$2/$10` на ввод/вывод; стандартные тарифы `$3/$15` начинают действовать
1 сентября 2026 года.

`anthropic/claude-fable-5` всегда использует адаптивное мышление с уровнем усилий
`high` по умолчанию. Anthropic не позволяет отключать мышление для этой модели,
поэтому `/think off` и `/think minimal` вместо этого сопоставляются с уровнем усилий
`low`. OpenClaw также не указывает пользовательские значения температуры
для запросов Fable 5, поскольку Anthropic отклоняет переопределение температуры в любых
запросах с включённым мышлением.

`anthropic/claude-mythos-5` — модель с ограниченным доступом и таким же контрактом
постоянно включённого адаптивного мышления. OpenClaw по умолчанию использует
`high`, сопоставляет `/think off` и `/think minimal` с
`low` и не указывает выбранные вызывающей стороной параметры сэмплирования.
В каталоге указаны её контекстное окно на 1 000 000 токенов, ограничение вывода
в 128 000 токенов, поддержка изображений на входе и тарифы `$10/$50`
на ввод/вывод.

Для Claude Opus 4.8 мышление в OpenClaw по умолчанию отключено. Когда адаптивное
мышление явно включается с помощью `/think high|xhigh|max`, OpenClaw отправляет
значения уровней усилий Anthropic для Opus 4.8; модели Claude 4.6 (Opus 4.6 и Sonnet 4.6)
по умолчанию используют `adaptive`.

Переопределите параметр для отдельного сообщения с помощью `/think:<level>` или в параметрах модели:

```json5
{
  agents: {
    defaults: {
      models: {
        "anthropic/claude-opus-4-8": {
          params: { thinking: "high" },
        },
      },
    },
  },
}
```

- [Расширенное мышление](https://platform.claude.com/docs/en/build-with-claude/extended-thinking)

## Резервный вариант при отказе по соображениям безопасности (Claude Fable 5)

<Warning>
Использование Claude Fable 5 также означает использование Claude Opus 4.8. Fable 5
поставляется с классификаторами безопасности, которые могут отклонить запрос, а
предписанный Anthropic способ восстановления заключается в том, чтобы этот запрос
обработала `claude-opus-4-8`. OpenClaw автоматически активирует эту возможность
для прямых запросов с API-ключом, поэтому некоторые запросы к Fable обрабатываются
и тарифицируются как Claude Opus 4.8. Если ваша политика или бюджет не допускают
обработку запросов моделью Opus, не выбирайте `anthropic/claude-fable-5`.
</Warning>

### Зачем это нужно

Классификаторы Fable 5 возвращают `stop_reason: "refusal"` для запросов в ограниченных
областях, а также дают ложноположительные срабатывания для смежных безобидных задач
(инструменты безопасности, науки о жизни или даже просьба к модели воспроизвести
свои необработанные рассуждения). Без резервного варианта запрос завершается ошибкой,
хотя другая модель Claude могла бы его успешно обработать. В собственном сообщении
Anthropic об отказе интеграторам API предлагается настроить резервную модель.

### Как это работает

1. Для каждого прямого запроса с API-ключом к `anthropic/claude-fable-5` OpenClaw
   отправляет согласие на использование серверного резервного варианта Anthropic:
   бета-заголовок `server-side-fallback-2026-06-01` и
   `fallbacks: [{"model": "claude-opus-4-8"}]`. Claude Opus 4.8 — единственная
   резервная модель, которую Anthropic разрешает использовать для Fable 5.
2. Резервный вариант запускается только при отказе классификатора
   безопасности. Ограничения частоты запросов, перегрузки и ошибки сервера обрабатываются
   точно так же, как и раньше, через обычный механизм [переключения между моделями](/ru/concepts/model-failover)
   в OpenClaw.
3. Восстановление происходит в рамках того же вызова. Отказ до начала вывода
   незаметен, за исключением задержки; весь ответ поступает от Opus 4.8. При отказе
   в середине потоковой передачи частичный текст сохраняется как префикс, с которого
   продолжает резервная модель, а рассуждения и вызовы инструментов отклонившей запрос
   модели отбрасываются в соответствии с правилами повторного воспроизведения Anthropic
   (их нельзя возвращать в запросе или выполнять).
4. Если Claude Opus 4.8 также отклоняет запрос, отказ возвращается как
   ошибка — точно так же, как до появления этой возможности.

Резервное переключение происходит на уровне API Anthropic, поэтому `claude-opus-4-8`
не требуется добавлять в настроенный список моделей или цепочку резервных моделей:
API-ключ с доступом к Fable всегда может использовать Opus.

### Наблюдаемость и тарификация

- Для запроса, обработанного резервной моделью, в сообщении ассистента
  записывается диагностическое поле `provider_fallback`, содержащее `fromModel`
  и `toModel`, а поле сообщения `responseModel` сообщает
  `claude-opus-4-8`.
- Anthropic тарифицирует каждую попытку: отказ до начала вывода бесплатен,
  а восстановительный запрос оплачивается по тарифам Claude Opus 4.8 (в настоящее
  время они вдвое ниже тарифов Fable 5). Оценка стоимости запроса в OpenClaw
  рассчитывает запросы, обработанные резервной моделью, по тарифам Opus.
- При отказе в середине потоковой передачи Anthropic дополнительно
  тарифицирует уже переданную часть ответа Fable; эта часть указывается в данных
  API об использовании для каждой попытки, но не включается в оценку стоимости
  запроса в OpenClaw.

### Область применения

Применяется к `anthropic/claude-fable-5` при аутентификации с помощью API-ключа через
`api.anthropic.com`. Запросы через OAuth (повторное использование подписки Claude CLI),
прокси-базовые URL-адреса, Bedrock, Vertex и Foundry не изменяются и по-прежнему
возвращают отказы как ошибки.

Проверено в рабочей среде: безобидный запрос к Fable 5 с просьбой воспроизвести
необработанную цепочку рассуждений отклоняется с `category: "reasoning_extraction"`, если отправить
его без резервных моделей, а тот же запрос через OpenClaw возвращает обычный ответ,
обработанный Opus, с прикреплённым диагностическим полем `provider_fallback`.

Описание базового поведения см. в [руководстве Anthropic по отказам и резервным
моделям](https://platform.claude.com/docs/en/build-with-claude/refusals-and-fallback).

## Кэширование промптов

OpenClaw поддерживает функцию кэширования промптов Anthropic при аутентификации
с помощью API-ключа.

| Значение               | Срок хранения кэша | Описание                                      |
| ---------------------- | ------------------ | --------------------------------------------- |
| `"short"` (по умолчанию) | 5 минут            | Применяется автоматически при аутентификации с помощью API-ключа |
| `"long"`            | 1 час              | Расширенный срок хранения кэша                |
| `"none"`            | Без кэширования    | Отключает кэширование промптов                 |

```json5
{
  agents: {
    defaults: {
      models: {
        "anthropic/claude-opus-4-6": {
          params: { cacheRetention: "long" },
        },
      },
    },
  },
}
```

<AccordionGroup>
  <Accordion title="Переопределение кэша для отдельных агентов">
    Используйте параметры уровня модели как базовые, а затем переопределяйте их для отдельных агентов через `agents.list[].params`:

    ```json5
    {
      agents: {
        defaults: {
          model: { primary: "anthropic/claude-opus-4-6" },
          models: {
            "anthropic/claude-opus-4-6": {
              params: { cacheRetention: "long" },
            },
          },
        },
        list: [
          { id: "research", default: true },
          { id: "alerts", params: { cacheRetention: "none" } },
        ],
      },
    }
    ```

    Порядок объединения конфигурации:

    1. `agents.defaults.models["provider/model"].params`
    2. `agents.list[].params` (соответствующий `id`, переопределяет по ключу)

    Это позволяет одному агенту сохранять долгоживущий кеш, а другому агенту на той же модели отключать кеширование для всплескового трафика или трафика с низкой степенью повторного использования.

  </Accordion>

  <Accordion title="Примечания о Claude в Bedrock">
    - Модели Anthropic Claude в Bedrock (`amazon-bedrock/*anthropic.claude*`) принимают сквозную передачу `cacheRetention`, если она настроена.
    - Для моделей Bedrock, не относящихся к Anthropic, во время выполнения принудительно устанавливается `cacheRetention: "none"`.
    - Интеллектуальные значения по умолчанию для API-ключей также задают `cacheRetention: "short"` для ссылок Claude в Bedrock, если явное значение не установлено.

  </Accordion>
</AccordionGroup>

## Расширенная конфигурация

<AccordionGroup>
  <Accordion title="Быстрый режим">
    Общий переключатель OpenClaw `/fast` устанавливает для прямого трафика с API-ключом поле Anthropic `service_tier` в значение `api.anthropic.com`.

    | Команда | Соответствует |
    |---------|---------|
    | `/fast on` | `service_tier: "auto"` |
    | `/fast off` | `service_tier: "standard_only"` |

    ```json5
    {
      agents: {
        defaults: {
          models: {
            "anthropic/claude-sonnet-4-6": {
              params: { fastMode: true },
            },
          },
        },
      },
    }
    ```

    <Note>
    - Применяется только к прямым запросам `api.anthropic.com`, выполненным с API-ключом. Запросы с OAuth/токеном подписки и маршруты через прокси никогда не получают поле `service_tier`.
    - Явные параметры `serviceTier` или `service_tier` переопределяют `/fast`, если заданы оба.
    - В учетных записях без доступной емкости Priority Tier значение `service_tier: "auto"` может разрешиться в `standard`.

    </Note>

  </Accordion>

  <Accordion title="Распознавание мультимедиа (изображения и PDF)">
    Встроенный плагин Anthropic регистрирует распознавание изображений и PDF. OpenClaw
    автоматически определяет возможности работы с мультимедиа на основе настроенной аутентификации Anthropic;
    дополнительная конфигурация не требуется.

    | Свойство             | Значение                 |
    | -------------------- | ------------------------ |
    | Модель по умолчанию  | `claude-opus-4-8`       |
    | Поддерживаемый ввод  | Изображения, документы PDF |

    Когда к разговору прикрепляется изображение или PDF, OpenClaw автоматически
    направляет его через провайдер распознавания мультимедиа Anthropic.

  </Accordion>

  <Accordion title="Контекстное окно 1M">
    Claude Sonnet 5, Mythos 5 и Fable 5 имеют входное окно размером ровно 1 000 000 токенов
    и поддерживают до 128 000 выходных токенов. Контекстное окно Anthropic размером 1M
    также общедоступно для моделей Claude 4.x с адаптивным мышлением: Opus 4.8,
    Opus 4.7, Opus 4.6 и Sonnet 4.6. OpenClaw автоматически определяет размеры этих моделей,
    поэтому `params.context1m` не требуется:

    ```json5
    {
      agents: {
        defaults: {
          models: {
            "anthropic/claude-sonnet-5": {},
            "anthropic/claude-mythos-5": {},
            "anthropic/claude-opus-4-6": {},
          },
        },
      },
    }
    ```

    В старых конфигурациях можно сохранить `params.context1m: true`; для
    этих моделей это безвредная пустая операция, и OpenClaw больше не отправляет устаревший
    бета-заголовок `context-1m-2025-08-07` в любом случае. Старые записи конфигурации `anthropicBeta`
    с этим значением отбрасываются при разрешении заголовков запроса, а
    неподдерживаемые старые модели Claude продолжают использовать свое обычное контекстное окно.

    `params.context1m: true` аналогичным образом работает для бэкенда Claude CLI
    (`claude-cli/*`): подходящие модели Opus и Sonnet с общедоступной поддержкой уже автоматически
    получают окно 1M, поэтому и там этот параметр необязателен.

    <Warning>
    Требуется доступ к длинному контексту для ваших учетных данных Anthropic. Аутентификация через OAuth/токен подписки сохраняет необходимые бета-заголовки Anthropic, но OpenClaw удаляет устаревший бета-заголовок 1M, если он остался в старой конфигурации.
    </Warning>

  </Accordion>

  <Accordion title="Контекст Claude Opus 4.8 размером 1M">
    `anthropic/claude-opus-4-8` и его вариант `claude-cli` по умолчанию имеют контекстное
    окно размером 1M; `params.context1m: true` не требуется.
  </Accordion>
</AccordionGroup>

## Устранение неполадок

<AccordionGroup>
  <Accordion title="Ошибки 401 / токен внезапно стал недействительным">
    Срок действия аутентификации по токену Anthropic истекает, и токен может быть отозван. Для новых настроек вместо него используйте API-ключ Anthropic.
  </Accordion>

  <Accordion title='Не найден API-ключ для провайдера "anthropic"'>
    Аутентификация Anthropic настраивается **отдельно для каждого агента**; новые агенты не наследуют ключи основного агента. Повторно запустите первоначальную настройку для этого агента (или настройте API-ключ на хосте Gateway), затем выполните проверку с помощью `openclaw models status`.
  </Accordion>

  <Accordion title='Не найдены учетные данные для профиля "anthropic:default"'>
    Выполните `openclaw models status`, чтобы узнать, какой профиль аутентификации активен. Повторно запустите первоначальную настройку или настройте API-ключ для пути этого профиля.
  </Accordion>

  <Accordion title="Нет доступного профиля аутентификации (все находятся в периоде ожидания)">
    Проверьте `openclaw models status --json` на наличие `auth.unusableProfiles`. Периоды ожидания после превышения лимита запросов Anthropic могут применяться к отдельным моделям, поэтому другая модель Anthropic всё ещё может быть доступна. Добавьте другой профиль Anthropic или дождитесь окончания периода ожидания.
  </Accordion>
</AccordionGroup>

<Note>
Дополнительная помощь: [Устранение неполадок](/ru/help/troubleshooting) и [Часто задаваемые вопросы](/ru/help/faq).
</Note>
