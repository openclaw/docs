Anthropic разрабатывает семейство моделей **Claude**. OpenClaw поддерживает два способа аутентификации:

- **Ключ API** — прямой доступ к API Anthropic с оплатой по фактическому использованию (модели `anthropic/*`)
- **Claude CLI** — повторное использование существующего входа Claude Code на том же хосте

## Отслеживание использования и затрат

OpenClaw обнаруживает доступные учётные данные Anthropic и выбирает соответствующий способ отображения использования:

- Учётные данные подписки или настройки Claude отображают окна квот и необязательный бюджет дополнительного использования.
- `ANTHROPIC_ADMIN_KEY` или `ANTHROPIC_ADMIN_API_KEY` отображает в разделе **Использование** интерфейса Control UI сообщаемые провайдером затраты организации и использование Messages API за 30 дней, включая ежедневные расходы, итоговые показатели токенов и кеша, наиболее используемые модели и категории затрат.
- Учётные данные `sk-ant-admin...`, сохранённые в профиле провайдера Anthropic, автоматически распознаются как ключ Admin API.

История затрат Admin API поступает из [API использования и затрат](https://platform.claude.com/docs/en/manage-claude/usage-cost-api) Anthropic. Это фактические счета провайдера, отдельные от расчётной стоимости OpenClaw, определяемой на основе сеансов.

<Warning>
Бэкенд Claude CLI в OpenClaw запускает установленный Claude Code CLI в
неинтерактивном режиме вывода (`claude -p`). В текущей документации Anthropic по Claude Code
этот режим описывается как использование Agent SDK или программное использование. В обновлении поддержки
от 15 июня 2026 года Anthropic приостановила объявленное отдельное изменение оплаты Agent SDK:
Claude Agent SDK, `claude -p` и использование сторонних приложений по-прежнему
расходуют лимиты вошедшей в систему подписки, а ранее объявленный ежемесячный
кредит Agent SDK недоступен, пока Anthropic пересматривает этот план.

Интерактивный Claude Code по-прежнему расходует лимиты плана Claude, в который выполнен вход.
Аутентификация с помощью ключа API оплачивается напрямую по мере использования и не зависит от этого плана.
Для долгосрочно работающих хостов Gateway, общей автоматизации и предсказуемых
производственных затрат используйте ключ API Anthropic.

Текущие статьи поддержки Anthropic могут изменить это поведение без
выпуска новой версии OpenClaw:

- [Справочник по Claude Code CLI](https://code.claude.com/docs/en/cli-usage)
- [Использование Claude Agent SDK с планом Claude](https://support.claude.com/en/articles/15036540-use-the-claude-agent-sdk-with-your-claude-plan)
- [Использование Claude Code с планом Pro или Max](https://support.claude.com/en/articles/11145838-use-claude-code-with-your-pro-or-max-plan)
- [Использование Claude Code с планом Team или Enterprise](https://support.claude.com/en/articles/11845131-using-claude-code-with-your-team-or-enterprise-plan)
- [Управление затратами Claude Code](https://code.claude.com/docs/en/costs)

</Warning>

    провайдера с управляемым OpenClaw OAuth, например
    [OpenAI Codex](/ru/providers/openai).
    </Warning>

    ### Пример конфигурации

    Предпочтительно использовать каноническую ссылку на модель Anthropic с переопределением среды выполнения CLI:

    ```json5
    {
      agents: {
        defaults: {
          model: { primary: "anthropic/claude-opus-4-8" },
          models: {
            "anthropic/claude-opus-4-8": {
              agentRuntime: { id: "claude-cli" },
            },
          },
        },
      },
    }
    ```

    Устаревшие ссылки на модели `claude-cli/claude-opus-4-7` по-прежнему работают для
    совместимости, однако в новой конфигурации выбор провайдера и модели должен оставаться
    `anthropic/*`, а бэкенд выполнения следует указывать в политике среды выполнения провайдера или модели.

    ### Оплата и `claude -p`

    Для запусков Claude CLI OpenClaw использует неинтерактивный путь `claude -p`
    Claude Code. Сейчас Anthropic рассматривает этот путь как использование Agent SDK или программное использование:

    - В обновлении поддержки от 15 июня 2026 года Anthropic приостановила ранее объявленный
      отдельный план кредитов Agent SDK.
    - Claude Agent SDK в рамках подписки, `claude -p` и использование сторонних приложений
      по-прежнему расходуют лимиты подписки, в которую выполнен вход.
    - Ранее объявленный ежемесячный кредит Agent SDK недоступен, пока
      Anthropic пересматривает этот план.
    - Входы через Console или ключ API используют оплату API по фактическому использованию и не получают
      кредит Agent SDK по подписке.

Уведомление о приостановке см. в [статье Anthropic о плане Agent SDK](https://support.claude.com/en/articles/15036540-use-the-claude-agent-sdk-with-your-claude-plan),
а поведение подписок Claude Code — в статьях о планах
[Pro/Max](https://support.claude.com/en/articles/11145838-use-claude-code-with-your-pro-or-max-plan)
и
[Team/Enterprise](https://support.claude.com/en/articles/11845131-use-claude-code-with-your-team-or-enterprise-plan).

    Anthropic может изменить оплату Claude Code и поведение ограничений частоты запросов без
    выпуска новой версии OpenClaw. Когда важна предсказуемость оплаты, проверяйте `claude auth status`,
    `/status` и документацию Anthropic по приведённым ссылкам.

## Сеансы Claude на разных компьютерах

Встроенный плагин Anthropic добавляет группу **Claude Code** в обычную боковую панель
сеансов. Строки открываются в обычной панели чата. Плагин обнаруживает неархивированные сеансы
Claude Code на Gateway и подключённых хостах Node:

- Сеансы Claude CLI поступают из действительных записей индекса проектов и текущих файлов JSONL,
  ограниченный префикс метаданных которых определяет сеанс `sdk-cli`,
  не являющийся боковой цепочкой, в каталоге `~/.claude/projects/`.
- Сеансы Claude Desktop используют заголовок Desktop, время активности и
  состояние архива, когда его метаданные указывают на тот же идентификатор сеанса Claude Code.
- Сеанс, существующий только в CLI, не имеет признака архива, поэтому он остаётся видимым,
  пока присутствует его расшифровка.

Для обнаружения не требуется дополнительная конфигурация OpenClaw. Плагин Anthropic
встроен и включён по умолчанию; собственный узел macOS объявляет команды чтения
сеансов Claude, когда существует локальный каталог `~/.claude/projects/`.
Подтвердите обновление сопряжения узла при первом появлении этих команд.

Сначала боковая панель отображает новейшую ограниченную страницу с каждого хоста и обновляется
с обычным интервалом в 30 секунд. Используйте **Загрузить дополнительные сеансы** под группой каталога,
чтобы добавить следующую страницу для каждого хоста, у которого есть дополнительная история; добавленные строки остаются
видимыми и при обновлениях повторно загружаются до той же глубины. Клиенты каталога
используют `sessions.catalog.list`; при открытии строки используется `sessions.catalog.read`.

При выборе строки сначала загружается новейшая страница расшифровки. **Загрузить более старые
элементы расшифровки** следует непрозрачному байтовому курсору и считывает ещё один ограниченный фрагмент
файла JSONL вместо загрузки всей истории. Обычное содержимое пользователя, ассистента,
рассуждений, вызовов инструментов и результатов инструментов сохраняется. Отдельный элемент,
превышающий предел безопасности узла или Gateway, явно помечается как усечённый.

Если строка `claude-cli` находится локально на Gateway, ввод текста в обычном поле создания сообщения вызывает
`sessions.catalog.continue`. OpenClaw повторно разрешает локальную запись каталога,
создаёт или повторно использует собственный сеанс, закреплённый за моделью, импортирует не более 200 видимых
элементов или 512 КиБ и инициализирует привязку Claude CLI. Первый ход продолжается с
`--fork-session`; Claude назначает ответвлению новый идентификатор сеанса, поэтому последующие ходы используют
ответвление, а исходный сеанс остаётся неизменным.

Хост безголового узла также может разрешить продолжение своих строк Claude CLI, включив
указанный ниже локальный параметр узла и перезапустив хост узла:

```json5
{
  nodeHost: {
    agentRuns: {
      claude: { enabled: true },
    },
  },
}
```

Узел объявляет `agent.cli.claude.run.v1` только тогда, когда параметр включён
и удаётся разрешить его локальный исполняемый файл `claude`. OpenClaw повторно разрешает запись каталога
на этом узле, импортирует ту же ограниченную историю и привязывает принятый
сеанс к узлу и рабочему каталогу, указанному каталогом. Каждый ход запускает
реальный процесс `claude -p` узла, используя файлы Claude и вход этого узла. Политика
подтверждения выполнения узла по-прежнему применяется; Gateway не может принудительно включить эту возможность.

Продолжение на узле версии 1 является только одноразовым. Оно не включает конфигурацию MCP обратной связи Gateway
и аргументы плагина Skills Gateway, не выполняет повторную инициализацию из расшифровки Gateway и
отклоняет вложения и изображения. Строки Claude Desktop остаются доступными только для просмотра. Собственные
узлы приложения macOS также остаются доступными только для просмотра, пока приложение не объявит команду запуска.

<Note>
Сеансы Claude на сопряжённых узлах остаются доступными только для чтения, если безголовый узел явно
не объявляет `agent.cli.claude.run.v1`. OpenClaw никогда не изменяет метаданные Claude Desktop
и не архивирует сеансы Claude. Страница требует подключения оператора
с областью доступа на запись, поскольку использует аутентифицированный `node.invoke`; операции перечисления и чтения
остаются доступными только для чтения даже на узле с включённым продолжением.
</Note>

Описание команды узла и границы безопасности см. в разделе
[Узлы: сеансы и расшифровки Claude](/ru/nodes#claude-sessions-and-transcripts).

## Параметры размышления по умолчанию (Claude Sonnet 5, Mythos 5, Fable 5, 4.8 и 4.6)

`anthropic/claude-sonnet-5` по умолчанию использует адаптивное размышление с уровнем усилий `high`.
Используйте `/think off`, чтобы отключить размышление, или `/think xhigh|max` для более
высоких встроенных уровней усилий модели. OpenClaw не передаёт ручные бюджеты размышления, пользовательские
параметры семплирования, предварительное заполнение ответов ассистента и Priority Tier для Sonnet 5, поскольку
Anthropic не поддерживает эти возможности запросов для данной модели.
До 31 августа 2026 года каталог использует вводные цены Anthropic `$2/$10` на ввод и вывод;
стандартные цены `$3/$15` начинают действовать 1 сентября 2026 года.

`anthropic/claude-fable-5` всегда использует адаптивное мышление, а уровень усилий по умолчанию — `high`.
Anthropic не позволяет отключать мышление для этой модели, поэтому
`/think off` и `/think minimal` вместо этого сопоставляются с уровнем усилий `low`. OpenClaw также
не передаёт пользовательские значения температуры в запросах Fable 5, поскольку Anthropic отклоняет
переопределение температуры в любых запросах с включённым мышлением.

`anthropic/claude-mythos-5` — модель с ограниченным доступом и тем же контрактом
постоянно включённого адаптивного мышления. По умолчанию OpenClaw использует `high`, сопоставляет `/think off` и
`/think minimal` с `low` и не передаёт выбранные вызывающей стороной параметры сэмплирования.
В каталоге указаны окно контекста в 1,000,000 токенов, ограничение вывода
в 128,000 токенов, поддержка изображений на входе и стоимость ввода/вывода `$10/$50`.

Для Claude Opus 4.8 в OpenClaw мышление по умолчанию отключено. Если вы явно
включаете адаптивное мышление с помощью `/think high|xhigh|max`, OpenClaw передаёт
значения усилий Anthropic для Opus 4.8; модели Claude 4.6 (Opus 4.6 и Sonnet 4.6)
по умолчанию используют `adaptive`.

Переопределите для отдельного сообщения с помощью `/think:<level>` или в параметрах модели:

```json5
{
  agents: {
    defaults: {
      models: {
        "anthropic/claude-opus-4-8": {
          params: { thinking: "high" },
        },
      },
    },
  },
}
```

Связанная документация Anthropic:
- [Адаптивное мышление](https://platform.claude.com/docs/en/build-with-claude/adaptive-thinking)
- [Расширенное мышление](https://platform.claude.com/docs/en/build-with-claude/extended-thinking)

## Резервный переход при отказе по соображениям безопасности (Claude Fable 5)

<Warning>
Использование Claude Fable 5 также подразумевает использование Claude Opus 4.8. Fable 5 поставляется с
классификаторами безопасности, которые могут отклонить запрос, а утверждённый Anthropic
способ восстановления — поручить обработку этого запроса `claude-opus-4-8`. OpenClaw включает эту
возможность автоматически для прямых запросов с API-ключом, поэтому некоторые запросы к Fable
обрабатываются и тарифицируются как Claude Opus 4.8. Если ваша политика или бюджет не допускают
обработку запросов моделью Opus, не выбирайте `anthropic/claude-fable-5`.
</Warning>

### Зачем это нужно

Классификаторы Fable 5 возвращают `stop_reason: "refusal"` для запросов в ограниченных
областях, а также дают ложные срабатывания для смежных безобидных задач (инструменты
безопасности, науки о жизни и даже просьбы к модели воспроизвести свои необработанные
рассуждения). Без резервного перехода запрос завершается с ошибкой, хотя
другая модель Claude могла бы успешно его обработать — собственное сообщение Anthropic
об отказе рекомендует интеграторам API настроить резервную модель.

### Как это работает

1. Для каждого прямого запроса с API-ключом к `anthropic/claude-fable-5` OpenClaw
   передаёт Anthropic согласие на серверный резервный переход: бета-заголовок
   `server-side-fallback-2026-06-01` плюс
   `fallbacks: [{"model": "claude-opus-4-8"}]`. Claude Opus 4.8 — единственная
   резервная модель, которую Anthropic разрешает использовать для Fable 5.
2. Резервный переход запускается только при отказе классификатора безопасности. Ограничения частоты,
   перегрузки и серверные ошибки обрабатываются точно так же, как раньше, через
   обычный [механизм переключения моделей](/ru/concepts/model-failover) OpenClaw.
3. Восстановление происходит в рамках того же вызова. Отказ до начала вывода
   незаметен, кроме увеличения задержки; весь ответ поступает от Opus 4.8. При
   отказе в процессе потоковой передачи частичный текст сохраняется как префикс, с которого
   продолжает резервная модель, а рассуждения и вызовы инструментов отклонившей запрос модели
   отбрасываются в соответствии с правилами повторного воспроизведения Anthropic (их нельзя возвращать
   в запросе или выполнять).
4. Если Claude Opus 4.8 также отклоняет запрос, отказ возвращается как
   ошибка — точно так же, как до появления этой функции.

Резервный переход происходит на уровне API Anthropic, поэтому `claude-opus-4-8` не
обязательно добавлять в настроенный список моделей или цепочку резервных моделей — API-ключ
с доступом к Fable всегда может использовать Opus.

### Наблюдаемость и тарификация

- Для запроса, обработанного резервной моделью, в сообщении ассистента записывается диагностическая запись `provider_fallback`,
  указывающая `fromModel` и `toModel`, а поле сообщения
  `responseModel` сообщает `claude-opus-4-8`.
- Anthropic тарифицирует каждую попытку: отказ до начала вывода бесплатен, а восстановление
  оплачивается по тарифам Claude Opus 4.8 (в настоящее время они вдвое ниже тарифов Fable 5). Оценка
  стоимости запроса в OpenClaw рассчитывает запросы, обработанные резервной моделью, по тарифам Opus.
- При отказе в процессе потоковой передачи Anthropic дополнительно тарифицирует уже переданную часть ответа Fable;
  эта часть отражается в данных API об использовании для каждой попытки,
  но не включается в оценку стоимости запроса OpenClaw.

### Область действия

Применяется к `anthropic/claude-fable-5` с аутентификацией по API-ключу при обращении к
`api.anthropic.com`. Запросы через OAuth (повторное использование подписки Claude CLI), базовые URL прокси,
Bedrock, Vertex и Foundry не изменяются и по-прежнему возвращают
отказы как ошибки.

Проверено в реальной среде: безобидный запрос к Fable 5 с просьбой воспроизвести необработанную цепочку
рассуждений отклоняется с `category: "reasoning_extraction"`, если отправить его без
резервных моделей, а тот же запрос через OpenClaw возвращает обычный ответ от Opus
с прикреплённой диагностической записью `provider_fallback`.

Подробнее об этом поведении см. в [руководстве Anthropic по отказам и резервным
моделям](https://platform.claude.com/docs/en/build-with-claude/refusals-and-fallback).

## Кэширование промптов

OpenClaw поддерживает функцию кэширования промптов Anthropic при аутентификации по API-ключу.

| Значение               | Срок хранения кэша | Описание                            |
| ------------------- | -------------- | -------------------------------------- |
| `"short"` (по умолчанию) | 5 минут      | Применяется автоматически при аутентификации по API-ключу |
| `"long"`            | 1 час         | Расширенный кэш                         |
| `"none"`            | Без кэширования     | Отключает кэширование промптов                 |

```json5
{
  agents: {
    defaults: {
      models: {
        "anthropic/claude-opus-4-6": {
          params: { cacheRetention: "long" },
        },
      },
    },
  },
}
```

<AccordionGroup>
  <Accordion title="Переопределение кэша для отдельных агентов">
    Используйте параметры уровня модели как базовые, а затем переопределяйте их для отдельных агентов через `agents.list[].params`:

    ```json5
    {
      agents: {
        defaults: {
          model: { primary: "anthropic/claude-opus-4-6" },
          models: {
            "anthropic/claude-opus-4-6": {
              params: { cacheRetention: "long" },
            },
          },
        },
        list: [
          { id: "research", default: true },
          { id: "alerts", params: { cacheRetention: "none" } },
        ],
      },
    }
    ```

    Порядок объединения конфигурации:

    1. `agents.defaults.models["provider/model"].params`
    2. `agents.list[].params` (соответствующий `id`, переопределяет по ключу)

    Это позволяет одному агенту сохранять долговременный кэш, а другому агенту на той же модели отключить кэширование для всплескового трафика с низким повторным использованием.

  </Accordion>

  <Accordion title="Примечания о Claude в Bedrock">
    - Модели Anthropic Claude в Bedrock (`amazon-bedrock/*anthropic.claude*`) принимают сквозной параметр `cacheRetention`, если он настроен.
    - Для моделей Bedrock, не принадлежащих Anthropic, во время выполнения принудительно устанавливается `cacheRetention: "none"`.
    - Интеллектуальные значения по умолчанию для API-ключа также задают `cacheRetention: "short"` для ссылок Claude в Bedrock, если явное значение не указано.

  </Accordion>
</AccordionGroup>

## Расширенная конфигурация

<AccordionGroup>
  <Accordion title="Быстрый режим">
    Общий переключатель OpenClaw `/fast` устанавливает поле Anthropic `service_tier` для прямого трафика с API-ключом к `api.anthropic.com`.

    | Команда | Сопоставляется с |
    |---------|---------|
    | `/fast on` | `service_tier: "auto"` |
    | `/fast off` | `service_tier: "standard_only"` |

    ```json5
    {
      agents: {
        defaults: {
          models: {
            "anthropic/claude-sonnet-4-6": {
              params: { fastMode: true },
            },
          },
        },
      },
    }
    ```

    <Note>
    - Применяется только к прямым запросам `api.anthropic.com` с API-ключом. В запросы с OAuth/токеном подписки и через прокси поле `service_tier` никогда не добавляется.
    - Явные параметры `serviceTier` или `service_tier` переопределяют `/fast`, если заданы одновременно.
    - Для учётных записей без ресурсов Priority Tier значение `service_tier: "auto"` может быть преобразовано в `standard`.

    </Note>

  </Accordion>

  <Accordion title="Понимание медиаданных (изображения и PDF)">
    Встроенный плагин Anthropic регистрирует поддержку понимания изображений и PDF. OpenClaw
    автоматически определяет возможности обработки медиаданных на основе настроенной аутентификации Anthropic;
    дополнительная конфигурация не требуется.

    | Свойство        | Значение                 |
    | --------------- | --------------------- |
    | Модель по умолчанию   | `claude-opus-4-8`     |
    | Поддерживаемый ввод | Изображения, документы PDF |

    Когда к разговору прикрепляется изображение или PDF, OpenClaw автоматически
    направляет его через провайдера понимания медиаданных Anthropic.

  </Accordion>

  <Accordion title="Окно контекста 1M">
    Claude Sonnet 5, Mythos 5 и Fable 5 имеют точное окно ввода
    в 1,000,000 токенов и поддерживают до 128,000 токенов вывода. Окно контекста Anthropic
    размером 1M также общедоступно для моделей Claude 4.x с адаптивным мышлением: Opus 4.8,
    Opus 4.7, Opus 4.6 и Sonnet 4.6. OpenClaw автоматически задаёт размеры для этих моделей,
    параметр `params.context1m` не требуется:

    ```json5
    {
      agents: {
        defaults: {
          models: {
            "anthropic/claude-sonnet-5": {},
            "anthropic/claude-mythos-5": {},
            "anthropic/claude-opus-4-6": {},
          },
        },
      },
    }
    ```

    В старых конфигурациях можно оставить `params.context1m: true`; для
    этих моделей он ничего не делает и не причиняет вреда, а OpenClaw больше не передаёт устаревший
    бета-заголовок `context-1m-2025-08-07` в любом случае. Старые записи конфигурации `anthropicBeta`
    с этим значением удаляются при разрешении заголовков запроса, а
    неподдерживаемые старые модели Claude сохраняют обычное окно контекста.

    `params.context1m: true` работает так же для бэкенда Claude CLI
    (`claude-cli/*`): подходящие модели Opus и Sonnet с общедоступной поддержкой уже автоматически получают
    окно 1M, поэтому этот параметр там также необязателен.

    <Warning>
    Требуется доступ к длинному контексту для ваших учётных данных Anthropic. При аутентификации через OAuth/токен подписки сохраняются необходимые бета-заголовки Anthropic, но OpenClaw удаляет устаревший бета-заголовок 1M, если он остался в старой конфигурации.
    </Warning>

  </Accordion>

  <Accordion title="Контекст 1M в Claude Opus 4.8">
    `anthropic/claude-opus-4-8` и его вариант `claude-cli` по умолчанию имеют окно контекста
    1M; параметр `params.context1m: true` не требуется.
  </Accordion>
</AccordionGroup>

## Устранение неполадок

<AccordionGroup>
  <Accordion title="Ошибки 401 / токен внезапно стал недействительным">
    Срок действия аутентификации по токену Anthropic истекает, и токен может быть отозван. Для новых настроек вместо него используйте API-ключ Anthropic.
  </Accordion>

  <Accordion title='API-ключ для провайдера "anthropic" не найден'>
    Аутентификация Anthropic настраивается **для каждого агента**; новые агенты не наследуют ключи основного агента. Повторно выполните первоначальную настройку для этого агента (или настройте API-ключ на хосте Gateway), затем проверьте с помощью `openclaw models status`.
  </Accordion>

  <Accordion title='Учётные данные для профиля "anthropic:default" не найдены'>
    Выполните `openclaw models status`, чтобы узнать, какой профиль аутентификации активен. Повторно выполните первоначальную настройку или настройте API-ключ для пути этого профиля.
  </Accordion>

  <Accordion title="Нет доступного профиля аутентификации (все на паузе)">
    Проверьте `openclaw models status --json` на наличие `auth.unusableProfiles`. Периоды ожидания после ограничения частоты Anthropic могут действовать для конкретной модели, поэтому другая модель Anthropic может оставаться доступной. Добавьте другой профиль Anthropic или дождитесь окончания периода ожидания.
  </Accordion>
</AccordionGroup>

<Note>
Дополнительная помощь: [Устранение неполадок](/ru/help/troubleshooting) и [Часто задаваемые вопросы](/ru/help/faq).
</Note>
