`qwen-oauth` — это идентификатор провайдера Qwen Portal, зарегистрированный плагином Qwen
(`@openclaw/qwen-provider`). Он использует конечную точку Qwen Portal по адресу
`https://portal.qwen.ai/v1` и сохраняет доступность старых конфигураций Qwen OAuth / Portal
через отдельный идентификатор провайдера, отличный от канонического провайдера `qwen`.

Выберите `qwen-oauth`, если у вас уже есть действующий токен Qwen Portal, вы
переносите устаревший рабочий процесс Qwen OAuth или Qwen CLI либо вам необходимо
протестировать именно конечную точку Qwen Portal. Для новых конфигураций
предпочтительно использовать [Qwen](/ru/providers/qwen) со стандартной конечной точкой
ModelStudio: она поддерживает новые конфигурации с API-ключами, более широкий выбор
конечных точек, стандартную оплату по мере использования, Coding Plan
и полный каталог плагина Qwen.

## Настройка

Установите плагин Qwen, если он ещё не установлен:

```bash
openclaw plugins install @openclaw/qwen-provider
openclaw gateway restart
```

Укажите токен Portal при первоначальной настройке:

```bash
openclaw onboard --auth-choice qwen-oauth
```

При неинтерактивном запуске токен считывается из `--qwen-oauth-token <token>`, либо задайте:

```bash
export QWEN_API_KEY="<your-qwen-portal-token>" # pragma: allowlist secret
```

При первоначальной настройке токен сохраняется в профиле аутентификации `qwen-oauth`,
добавляется каталог моделей Portal, а `qwen-oauth/qwen3.5-plus` назначается моделью по умолчанию,
если другая модель не настроена.

## Значения по умолчанию

- Провайдер: `qwen-oauth`
- Псевдонимы: `qwen-portal`, `qwen-cli`
- Базовый URL: `https://portal.qwen.ai/v1`
- Переменная окружения: `QWEN_API_KEY`
- Стиль API: совместимый с OpenAI
- Модель по умолчанию: `qwen-oauth/qwen3.5-plus`

## Отличия от Qwen

В OpenClaw предусмотрено два идентификатора провайдеров для работы с Qwen:

| Провайдер     | Семейство конечных точек                                 | Лучше всего подходит для                                                                |
| ------------ | -------------------------------------------------------- | -------------------------------------------------------------------------------------- |
| `qwen`       | Конечные точки Qwen Cloud / Alibaba DashScope и Coding Plan | Новых конфигураций с API-ключами, стандартной оплаты по мере использования, Coding Plan и мультимодальных возможностей DashScope |
| `qwen-oauth` | Конечная точка Qwen Portal по адресу `portal.qwen.ai/v1`              | Существующих токенов Qwen Portal и устаревших конфигураций Qwen OAuth / CLI                         |

Оба провайдера используют форматы запросов, совместимые с OpenAI, однако у них
раздельные механизмы аутентификации. Токен, сохранённый для `qwen-oauth`, не следует
считать ключом DashScope или ModelStudio, а новый ключ DashScope необходимо использовать
с каноническим провайдером `qwen`.

## Модели

Плагин Qwen добавляет этот статический каталог для конечной точки Qwen Portal. Для всех
записей максимальный объём вывода составляет 65 536 токенов; доступность зависит от
текущей учётной записи и токена Qwen Portal.

Если ваша учётная запись использует API-ключи ModelStudio / DashScope, настройте
канонический провайдер `qwen`:

```bash
openclaw onboard --auth-choice qwen-standard-api-key
openclaw models set qwen/qwen3-coder-plus
```

## Миграция

Устаревшие профили OAuth Qwen Portal не поддерживают обновление токенов; `openclaw doctor`
помечает их. Если профиль Portal перестал работать, повторно выполните первоначальную
настройку с актуальным токеном или перейдите на стандартный провайдер Qwen:

```bash
openclaw onboard --auth-choice qwen-standard-api-key
```

Стандартный глобальный ModelStudio использует:

```text
https://dashscope-intl.aliyuncs.com/compatible-mode/v1
```

## Устранение неполадок

- Ошибки обновления OAuth Portal: устаревшие профили OAuth Qwen Portal не
  поддерживают обновление токенов. Повторно выполните первоначальную настройку с актуальным токеном.
- Ошибки из-за неверной конечной точки: при использовании токена Portal убедитесь,
  что ссылка на модель начинается с `qwen-oauth/`. Ссылки `qwen/` используйте
  только для канонического провайдера Qwen.
- Путаница с `QWEN_API_KEY`: на обеих страницах Qwen упоминается эта переменная
  окружения, однако при первоначальной настройке учётные данные сохраняются для выбранного
  идентификатора провайдера. Если на одном компьютере доступны и `qwen`, и
  `qwen-oauth`, предпочтительно использовать первоначальную настройку.
