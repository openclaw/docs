`qwen-oauth` — это идентификатор провайдера Qwen Portal, зарегистрированный плагином Qwen
(`@openclaw/qwen-provider`). Он использует конечную точку Qwen Portal
`https://portal.qwen.ai/v1` и позволяет обращаться к более старым конфигурациям
Qwen OAuth и портала через отдельный идентификатор провайдера, отличный от
канонического провайдера `qwen`.

Выберите `qwen-oauth`, если у вас уже есть рабочий токен Qwen Portal, вы
переносите устаревший рабочий процесс Qwen OAuth или Qwen CLI либо вам нужно
протестировать именно конечную точку Qwen Portal. Для новых конфигураций
предпочтительно использовать [Qwen](/ru/providers/qwen) со стандартной конечной
точкой ModelStudio: она поддерживает новые конфигурации с ключом API, более
широкий выбор конечных точек, стандартную оплату по мере использования,
Coding Plan и полный каталог плагина Qwen.

## Настройка

Установите плагин Qwen, если он ещё не установлен:

```bash
openclaw plugins install @openclaw/qwen-provider
openclaw gateway restart
```

Укажите токен портала во время первоначальной настройки:

```bash
openclaw onboard --auth-choice qwen-oauth
```

При неинтерактивном запуске токен считывается из параметра `--qwen-oauth-token <token>`; также можно задать:

```bash
export QWEN_API_KEY="<your-qwen-portal-token>" # pragma: allowlist secret
```

Первоначальная настройка сохраняет токен в профиле аутентификации `qwen-oauth`,
заполняет каталог моделей портала и назначает `qwen-oauth/qwen3.5-plus` моделью
по умолчанию, если другая модель не настроена.

## Значения по умолчанию

- Провайдер: `qwen-oauth`
- Псевдонимы: `qwen-portal`, `qwen-cli`
- Базовый URL: `https://portal.qwen.ai/v1`
- Переменная окружения: `QWEN_API_KEY`
- Стиль API: совместимый с OpenAI
- Модель по умолчанию: `qwen-oauth/qwen3.5-plus`

## Отличия от Qwen

В OpenClaw предусмотрено два идентификатора провайдеров для работы с Qwen:

| Провайдер    | Семейство конечных точек                                  | Оптимальное применение                                                                                       |
| ------------ | --------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| `qwen`       | Конечные точки Qwen Cloud / Alibaba DashScope и Coding Plan | Новые конфигурации с ключом API, стандартная оплата по мере использования, Coding Plan, мультимодальные возможности DashScope |
| `qwen-oauth` | Конечная точка Qwen Portal на `portal.qwen.ai/v1`         | Существующие токены Qwen Portal и устаревшие конфигурации Qwen OAuth / CLI                                   |

Оба провайдера используют совместимые с OpenAI форматы запросов, но их
механизмы аутентификации разделены. Токен, сохранённый для `qwen-oauth`, не
следует использовать как ключ DashScope или ModelStudio, а для нового ключа
DashScope следует использовать канонический провайдер `qwen`.

Если ваша учётная запись использует ключи API ModelStudio / DashScope,
настройте канонический провайдер `qwen`:

```bash
openclaw onboard --auth-choice qwen-standard-api-key
openclaw models set qwen/qwen3-coder-plus
```

## Миграция

Устаревшие профили Qwen Portal OAuth невозможно обновить; `openclaw doctor`
помечает их. Если профиль портала перестал работать, повторно выполните
первоначальную настройку с актуальным токеном или перейдите на стандартный
провайдер Qwen:

```bash
openclaw onboard --auth-choice qwen-standard-api-key
```

Стандартная глобальная версия ModelStudio использует:

```text
https://dashscope-intl.aliyuncs.com/compatible-mode/v1
```

## Устранение неполадок

- Сбои обновления Portal OAuth: устаревшие профили Qwen Portal OAuth невозможно
  обновить. Повторно выполните первоначальную настройку с актуальным токеном.
- Ошибки из-за неверной конечной точки: при использовании токена портала
  убедитесь, что ссылка на модель начинается с `qwen-oauth/`. Ссылки с `qwen/`
  используйте только для канонического провайдера Qwen.
- Путаница с `QWEN_API_KEY`: эта переменная окружения упоминается на обеих
  страницах Qwen, но первоначальная настройка сохраняет учётные данные под
  идентификатором выбранного провайдера. Если на одном компьютере доступны и
  `qwen`, и `qwen-oauth`, предпочтительно использовать первоначальную настройку.
