OpenClaw поставляется со встроенным провайдером `runway` для облачной генерации видео, который включён по умолчанию и зарегистрирован в соответствии с контрактом `videoGenerationProviders`.

| Свойство                    | Значение                                                          |
| --------------------------- | ----------------------------------------------------------------- |
| Идентификатор провайдера    | `runway`                                                |
| Плагин                      | встроенный, `enabledByDefault: true`                                    |
| Переменные среды авторизации | `RUNWAYML_API_SECRET` (каноническая) или `RUNWAY_API_KEY`          |
| Флаг первоначальной настройки | `--auth-choice runway-api-key`                                              |
| Прямой флаг CLI             | `--runway-api-key <key>`                                                |
| API                         | Генерация видео Runway на основе задач (опрос `GET /v1/tasks/{id}`) |
| Модель по умолчанию         | `runway/gen4.5`                                                |

## Начало работы

<Steps>
  <Step title="Укажите ключ API">
    ```bash
    openclaw onboard --auth-choice runway-api-key
    ```
  </Step>
  <Step title="Установите Runway как провайдер видео по умолчанию">
    ```bash
    openclaw config set agents.defaults.videoGenerationModel.primary "runway/gen4.5"
    ```
  </Step>
  <Step title="Создайте видео">
    Попросите агента создать видео. Runway будет использован автоматически.
  </Step>
</Steps>

## Поддерживаемые режимы и модели

Провайдер предоставляет семь моделей Runway, распределённых по трём режимам. Один и тот же идентификатор модели может поддерживать несколько режимов (например, `gen4.5` работает как для преобразования текста в видео, так и для преобразования изображения в видео).

| Режим                       | Модели                                                                 | Входной референс                   |
| --------------------------- | ---------------------------------------------------------------------- | ---------------------------------- |
| Преобразование текста в видео | `gen4.5` (по умолчанию), `veo3.1`, `veo3.1_fast`, `veo3` | Нет                                |
| Преобразование изображения в видео | `gen4.5`, `gen4_turbo`, `gen3a_turbo`, `veo3.1`, `veo3.1_fast`, `veo3` | 1 локальное или удалённое изображение |
| Преобразование видео в видео | `gen4_aleph`                                                    | 1 локальное или удалённое видео    |

Локальные референсные изображения и видео поддерживаются через URI данных.

| Соотношения сторон                    | Допустимые значения                         |
| ------------------------------------- | ------------------------------------------- |
| Преобразование текста в видео         | `16:9`, `9:16`      |
| Редактирование изображений и видео    | `1:1`, `16:9`, `9:16`, `3:4`, `4:3`, `21:9` |

<Warning>
  Для преобразования видео в видео сейчас требуется `runway/gen4_aleph`. Другие идентификаторы моделей Runway отклоняют входные референсные видео.
</Warning>

<Note>
  Выбор идентификатора модели Runway не из того столбца приводит к явной ошибке ещё до отправки API-запроса из OpenClaw. Провайдер проверяет `model` по списку разрешённых значений режима (`TEXT_ONLY_MODELS`, `IMAGE_MODELS`, `VIDEO_MODELS`) в `extensions/runway/video-generation-provider.ts`.
</Note>

## Конфигурация

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "runway/gen4.5",
      },
    },
  },
}
```

## Расширенная конфигурация

<AccordionGroup>
  <Accordion title="Псевдонимы переменных среды">
    OpenClaw распознаёт как `RUNWAYML_API_SECRET` (каноническую), так и `RUNWAY_API_KEY`.
    Любая из этих переменных обеспечивает авторизацию у провайдера Runway.
  </Accordion>

  <Accordion title="Опрос состояния задачи">
    Runway использует API на основе задач. После отправки запроса на генерацию OpenClaw
    опрашивает `GET /v1/tasks/{id}`, пока видео не будет готово. Для поведения
    опроса дополнительная конфигурация не требуется.
  </Accordion>
</AccordionGroup>
