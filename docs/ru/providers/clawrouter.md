ClawRouter предоставляет OpenClaw один ключ с заданной областью действия политики для нескольких вышестоящих поставщиков моделей. Встроенный Plugin `clawrouter` обнаруживает только модели, разрешённые для этого ключа, направляет запросы к каждой модели по заявленному для неё протоколу и отображает бюджет ключа и совокупное использование в интерфейсах статистики использования OpenClaw.

Учётные данные вышестоящих сервисов и перенаправление запросов с учётом особенностей поставщиков остаются в ClawRouter, поэтому вам не нужно устанавливать или аутентифицировать Plugin каждого вышестоящего поставщика на узле OpenClaw. Plugin поставляется в составе OpenClaw (`enabledByDefault: true`); вам нужны только выданные учётные данные ClawRouter.

| Свойство          | Значение                                          |
| ----------------- | ------------------------------------------------- |
| Поставщик         | `clawrouter`                                      |
| Plugin            | встроенный (входит в состав OpenClaw)             |
| Аутентификация    | `CLAWROUTER_API_KEY`                              |
| URL по умолчанию  | `https://clawrouter.openclaw.ai`                  |
| Каталог моделей   | Ограничен областью учётных данных через `/v1/catalog` |
| Квоты             | Месячный бюджет и использование через `/v1/usage` |

## Начало работы

<Steps>
  <Step title="Получите учётные данные с заданной областью действия">
    Запросите у администратора ClawRouter учётные данные, политика которых включает поставщиков, модели и месячный бюджет, которыми вы должны пользоваться. Учётные данные показываются только один раз при выдаче.
  </Step>
  <Step title="Настройте OpenClaw">
    ```bash
    export CLAWROUTER_API_KEY="..."
    openclaw onboard --auth-choice clawrouter-api-key
    openclaw plugins enable clawrouter
    ```

    `clawrouter` входит в комплект поставки и включён по умолчанию. Если в вашей конфигурации задан `plugins.allow`, добавьте `clawrouter` в этот список перед включением. Для пользовательского развёртывания задайте в `models.providers.clawrouter.baseUrl` адрес источника ClawRouter; по умолчанию используется `https://clawrouter.openclaw.ai`.

  </Step>
  <Step title="Выведите список предоставленных моделей">
    ```bash
    openclaw models list --all --provider clawrouter
    ```

    Используйте возвращённые ссылки на модели в точности как показано. В них сохраняется пространство имён вышестоящего поставщика, например `clawrouter/openai/gpt-5.5`, `clawrouter/anthropic/claude-sonnet-4-6` или `clawrouter/google/gemini-3.5-flash`. Если `agents.defaults.models` используется в вашей конфигурации как список разрешённых значений, добавьте в него каждую выбранную ссылку ClawRouter.

  </Step>
  <Step title="Выберите модель">
    ```bash
    openclaw models set clawrouter/<provider>/<model>
    ```

    Также можно выбрать возвращённую модель для одного запуска с помощью `openclaw agent --model clawrouter/<provider>/<model> --message "..."`.

  </Step>
</Steps>

## Управляемое неинтерактивное развёртывание

Храните ключ прокси в системе внедрения секретов рабочей нагрузки, а в `openclaw.json` сохраняйте только SecretRef. Канонические поля управляемой конфигурации:

| Назначение                 | Поле конфигурации или окружения                                             |
| -------------------------- | --------------------------------------------------------------------------- |
| Адрес маршрутизатора       | `models.providers.clawrouter.baseUrl`                                       |
| Учётные данные             | `models.providers.clawrouter.apiKey` -> SecretRef окружения                  |
| Значение секрета           | `CLAWROUTER_API_KEY` в окружении процесса Gateway                           |
| Модель по умолчанию        | `agents.defaults.model.primary` -> `clawrouter/<provider>/<model>`           |
| Метка рабочей нагрузки     | `models.providers.clawrouter.headers.X-ClawRouter-Project-Id` (необязательно) |

Например, контроллер развёртывания может управлять следующим исправлением JSON5:

```json5
{
  plugins: {
    entries: { clawrouter: { enabled: true } },
  },
  models: {
    providers: {
      clawrouter: {
        baseUrl: "https://clawrouter.internal.example",
        apiKey: {
          source: "env",
          provider: "default",
          id: "CLAWROUTER_API_KEY",
        },
        headers: {
          "X-ClawRouter-Project-Id": "fakeco",
        },
      },
    },
  },
  agents: {
    defaults: {
      model: { primary: "clawrouter/openai/gpt-5.5" },
    },
  },
}
```

Если в развёртывании задан `plugins.allow`, сохраните существующие элементы и добавьте `clawrouter`. Проверьте и примените изменения без интерактивного мастера:

```bash
openclaw config patch --file ./clawrouter.patch.json5 --dry-run --json
openclaw config patch --file ./clawrouter.patch.json5
```

Пробный запуск разрешает SecretRef, но никогда не выводит его значение. Чтобы сменить учётные данные, обновите внешний секрет, предоставляющий `CLAWROUTER_API_KEY`, и перезапустите рабочую нагрузку Gateway, чтобы загрузилось новое окружение процесса. Файл конфигурации и ссылка на модель не изменяются.

Для автономного Docker Gateway, собранного из исходного кода, ClawRouter уже включён в корневую среду выполнения. Выберите только Plugin канала, которому нужна отдельная упаковка, например `OPENCLAW_EXTENSIONS=clickclack`, `slack` или `msteams`; см. [образы, собранные из исходного кода с выбранными плагинами](/ru/install/docker#source-built-images-with-selected-plugins).
В архивных и аппаратных развёртываниях тот же принятый исходный код должен упаковываться через собственный конвейер артефактов, а не использоваться из образа OCI.

## Готовность и проверка в рабочей среде

Эти проверки подтверждают разные границы; не заменяйте одну другой:

```bash
# Проверяется только работоспособность процесса ClawRouter; учётные данные и вышестоящая модель не используются.
curl -fsS https://clawrouter.internal.example/v1/health

# Проверяется только готовность OpenClaw Gateway после запуска; вызов модели не выполняется.
curl -fsS http://127.0.0.1:18789/readyz

# Обнаружение каталога в области действия учётных данных.
openclaw models list --all --provider clawrouter --json

# Минимальная проверка реального вывода через настроенного поставщика ClawRouter.
openclaw models status --probe --probe-provider clawrouter --probe-max-tokens 8 --json

# Контрольная проверка рабочей нагрузки с точной ссылкой на предоставленную модель.
openclaw agent --agent main \
  --model clawrouter/openai/gpt-5.5 \
  --message "Reply exactly: CLAWROUTER_CANARY_OK" \
  --json
```

Используйте модель, возвращённую каталогом с заданной областью действия, вместо слепого копирования модели из примера. Успешный ответ `/readyz` означает, что Gateway может обслуживать запросы; он не подтверждает готовность ClawRouter, его учётных данных или вышестоящего поставщика. Проверка модели и контрольный запрос агента подтверждают выполнение вывода.

Для диагностики в рабочей среде выполните контрольный запрос и проверьте стандартные журналы Gateway. Существующая диагностика транспорта модели, содержащая только метаданные, выводит строки следующего вида:

```text
[model-fetch] start provider=clawrouter api=openai-responses model=openai/gpt-5.5 method=POST url=https://clawrouter.internal.example/v1/responses
[model-fetch] response provider=clawrouter api=openai-responses model=openai/gpt-5.5 status=200
```

Plugin отправляет ограниченные по длине заголовки `X-ClawRouter-Client`, `X-ClawRouter-Agent-Id` и `X-ClawRouter-Session-Id`, когда соответствующие идентификаторы доступны. Он также сопоставляет диагностический `callId` вызова модели (`<run-id>:model:<n>`) с `X-Request-ID`, благодаря чему событие вызова модели OpenClaw можно связать с журналом аудита ClawRouter, содержащим только метаданные. Значения, укладывающиеся в ограничение идентификатора запроса в 128 символов, остаются идентичными. Более длинные значения сохраняют суффикс `:model:<n>` и детерминированный хеш, чтобы различные вызовы оставались ограниченными по длине и пригодными для сопоставления. Статические метаданные развёртывания, такие как `X-ClawRouter-Project-Id`, можно задать в карте `headers` поставщика. Для заголовков атрибуции агента и сеанса сохраняется отдельное ограничение в 256 символов. Автоматические идентификаторы запросов, содержащие символы вне набора ASCII-идентификаторов ClawRouter, преобразуются в ту же детерминированную форму с ограниченной длиной.
Явно настроенные заголовки, включая любой вариант регистра `X-Request-ID`, имеют приоритет над автоматическими значениями. Диагностика транспорта записывает метаданные маршрутизации и ответа; она не регистрирует учётные данные, идентификаторы запросов, запросы к модели или ответы модели. Собственное событие аудита ClawRouter содержит выбранного вышестоящего поставщика и состояние хранения содержимого.

## Обнаружение моделей

`GET /v1/catalog` возвращает `{ providers: [...] }`, где каждая запись поставщика содержит собственный список `models[]` (с вышестоящим идентификатором, возможностями и ценами) и поддерживаемые маршруты запросов. OpenClaw не поставляет второй фиксированный список моделей ClawRouter. Модель каталога публикуется как модель OpenClaw, если:

- политика учётных данных разрешает её поставщика;
- модель каталога заявляет поддерживаемую возможность LLM (`llm.responses`, `llm.chat`, `llm.messages` или `llm.stream` с соответствующим потоковым маршрутом); и
- поставщик предоставляет соответствующий маршрут для одного из перечисленных ниже транспортов.

Добавление модели к поддерживаемому поставщику ClawRouter не требует выпуска новой версии OpenClaw: она обнаруживается при следующем обновлении каталога, который кэшируется на 60 секунд для каждой области действия учётных данных. Для модели, которой нужен новый сетевой протокол, сначала требуется поддержка в Plugin.

## Протокол и плагины поставщиков

ClawRouter управляет учётными данными вышестоящих сервисов; его каталог сообщает OpenClaw, какой транспорт использовать, поэтому вам не нужно устанавливать Plugin аутентификации каждой вышестоящей компании.

| Возможность или маршрут каталога                         | Транспорт OpenClaw     |
| ------------------------------------------------------- | ---------------------- |
| `llm.responses` (OpenAI-совместимый поставщик)          | `openai-responses`     |
| `llm.chat` (OpenAI-совместимый поставщик)               | `openai-completions`   |
| `llm.messages` + маршрут `anthropic.messages`           | `anthropic-messages`   |
| `llm.stream` + потоковый маршрут `google.generate_content` | `google-generative-ai` |

Plugin также применяет соответствующие политики повторного воспроизведения и схем инструментов для этих семейств (совместимость схем инструментов OpenAI/DeepSeek/Gemini; собственные политики повторного воспроизведения Anthropic и Google Gemini). Поставщик каталога, предоставляющий только неподдерживаемый формат запроса, намеренно не публикуется как текстовая модель OpenClaw. Нормализуйте таких поставщиков в ClawRouter в соответствии с одним из поддерживаемых контрактов вместо отправки несовместимой полезной нагрузки.

## Квоты и использование

Ответ ClawRouter `/v1/usage` передаётся в стандартные интерфейсы OpenClaw для статистики использования поставщика: итоговые значения запросов, токенов и расходов, а также окно месячного бюджета, если для ключа установлен лимит. Для ключей без лимита по-прежнему отображается совокупное использование, но без процентного окна.

Для запроса квоты используется тот же ключ с заданной областью действия, что и для обнаружения моделей. Ошибка запроса квоты не блокирует выполнение модели.

Проверьте актуальный снимок следующими командами:

```bash
openclaw status --usage
openclaw models status
```

Тот же снимок состояния поставщика доступен для `/status` в чате и в интерфейсе использования OpenClaw. Бюджет действует на всю политику, поэтому запросы другого клиента, использующего ту же политику ClawRouter, могут изменить оставшийся процент.

## Устранение неполадок

| Признак                                      | Проверка                                                                                                                                                                      |
| -------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Нет моделей ClawRouter                       | Убедитесь, что Plugin включён и разрешён в `plugins.allow`, затем проверьте, что учётные данные активны и предоставляют доступ хотя бы к одному готовому поставщику.             |
| Настроенная модель ClawRouter отсутствует    | Проверьте её возможность и поддержку маршрута в `/v1/catalog`. Неподдерживаемые транспортные контракты намеренно отфильтровываются.                                             |
| `Unknown model: clawrouter/...`              | Добавьте точную ссылку из каталога в `agents.defaults.models`, если эта карта конфигурации используется как список разрешённых значений.                                       |
| `401` или `403` от каталога или статистики   | Перевыпустите учётные данные ClawRouter или измените их область действия; OpenClaw не переключается на ключи вышестоящих поставщиков.                                           |
| Вызов модели завершается ошибкой после обнаружения | Проверьте подключение поставщика и состояние вышестоящего сервиса в ClawRouter, затем повторите попытку после восстановления его готовности.                              |
| В статистике есть итоговые значения, но нет процентов | Для политики не задан лимит; добавьте месячный бюджет в ClawRouter, чтобы отображалось процентное окно.                                                               |

## Поведение безопасности

- Обнаружение каталога ограничено областью настроенного ключа прокси и кэшируется отдельно для каждой области учётных данных (каталог агента, каталог рабочего пространства, идентификатор профиля аутентификации и базовый URL).
- Ключ прокси добавляется только при отправке запроса; он не сохраняется в метаданных модели.
- Значения автоматической атрибуции и корреляции запросов перед отправкой обрезаются по краям и отклоняются при наличии управляющих символов. Значения атрибуции ограничены 256 символами, а идентификаторы запросов — 128.
- Диагностические данные транспорта модели содержат только метаданные и никогда не включают ключ прокси или содержимое модели.
- Идентификаторы нативных моделей Anthropic и Gemini преобразуются в их исходные идентификаторы только при отправке.
- Неподдерживаемые или недоступные строки каталога обрабатываются с отказом по умолчанию и недоступны для выбора.
