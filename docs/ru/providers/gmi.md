GMI Cloud — это размещённая платформа инференса для передовых моделей и моделей с открытыми весами
с OpenAI-совместимым API. В OpenClaw она представлена официальным внешним
плагином провайдера: установите его один раз, сохраните учётные данные через стандартную аутентификацию моделей и используйте
ссылки на модели вида `gmi/google/gemini-3.1-flash-lite`.

Используйте GMI, если вам нужен один ключ API для нескольких семейств размещённых моделей, включая
маршруты Anthropic, DeepSeek, Google, Moonshot, OpenAI и Z.AI, доступные в
каталоге GMI. Он подходит в качестве резервного провайдера при переключении между моделями, для сравнения
размещённых маршрутов разных поставщиков или когда модель появляется в GMI раньше,
чем у вашего основного провайдера. OpenClaw отвечает за идентификатор провайдера, профиль аутентификации, псевдонимы,
начальное содержимое каталога моделей и базовый URL; GMI отвечает за текущую доступность моделей, биллинг,
ограничения частоты запросов и любые правила маршрутизации на стороне провайдера.

| Свойство      | Значение                                    |
| ------------- | ---------------------------------------- |
| Идентификатор провайдера   | `gmi` (псевдонимы: `gmi-cloud`, `gmicloud`) |
| Пакет       | `@openclaw/gmi-provider`                 |
| Переменная среды для аутентификации  | `GMI_API_KEY`                            |
| API           | OpenAI-совместимый (`openai-completions`) |
| Базовый URL      | `https://api.gmi-serving.com/v1`         |
| Модель по умолчанию | `gmi/google/gemini-3.1-flash-lite`       |

## Настройка

Установите плагин, перезапустите Gateway, затем создайте ключ API в GMI Cloud
(`https://www.gmicloud.ai/`):

```bash
openclaw plugins install @openclaw/gmi-provider
openclaw gateway restart
```

Затем выполните:

```bash
openclaw onboard --auth-choice gmi-api-key
```

При неинтерактивной настройке можно передать `--gmi-api-key <key>` или задать:

```bash
export GMI_API_KEY="<your-gmi-api-key>" # pragma: allowlist secret
```

## Когда стоит выбрать GMI

- Вам нужна размещённая OpenAI-совместимая конечная точка вместо локального сервера моделей.
- Вы хотите использовать несколько семейств коммерческих моделей и моделей с открытыми весами через одну
  учётную запись провайдера.
- Вам нужен резервный провайдер с маршрутизацией к вышестоящим сервисам, отличной от DeepInfra,
  OpenRouter, Together или прямых API поставщиков.
- Вам нужны специфичные для GMI идентификаторы моделей, цены или средства управления учётной записью.

Выберите прямого провайдера поставщика, если вам нужны нативные возможности поставщика,
которые GMI не предоставляет через свой OpenAI-совместимый маршрут. Выберите локального
провайдера, такого как LM Studio, Ollama, SGLang или vLLM, если локальное размещение данных или управление локальным
GPU важнее удобства размещённого сервиса.

## Модели

Каталог плагина содержит начальный набор часто доступных идентификаторов маршрутов GMI Cloud:

| Ссылка на модель                          | Входные данные        | Контекст   | Максимальный вывод |
| ---------------------------------- | ------------ | --------- | ---------- |
| `gmi/anthropic/claude-sonnet-4.6`  | текст + изображение | 200,000   | 64,000     |
| `gmi/deepseek-ai/DeepSeek-V3.2`    | текст         | 163,840   | 65,536     |
| `gmi/google/gemini-3.1-flash-lite` | текст + изображение | 1,048,576 | 65,536     |
| `gmi/moonshotai/Kimi-K2.5`         | текст + изображение | 262,144   | 65,536     |
| `gmi/openai/gpt-5.4`               | текст + изображение | 400,000   | 128,000    |
| `gmi/zai-org/GLM-5.1-FP8`          | текст         | 202,752   | 65,536     |

Каталог представляет собой начальный набор, а не гарантию того, что любая учётная запись может вызывать любую модель в
любой момент. Выведите список моделей, о которых сообщает настроенный провайдер в вашей среде:

```bash
openclaw models list --provider gmi
```

## Устранение неполадок

- `401` или `403`: убедитесь, что `GMI_API_KEY` задана для процесса, в котором работает
  OpenClaw, или повторно выполните первоначальную настройку, чтобы сохранить ключ в профиле аутентификации провайдера.
- Ошибки неизвестной модели: убедитесь, что модель существует в вашей учётной записи GMI, и используйте
  полную ссылку `gmi/<route-id>`, показанную командой `openclaw models list --provider gmi`.
- Периодические ошибки провайдера: попробуйте другой маршрут GMI или настройте GMI как
  резервный, а не единственный основной провайдер моделей.
