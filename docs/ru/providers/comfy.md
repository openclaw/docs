OpenClaw поставляется со встроенным плагином `comfy` для запуска ComfyUI на основе рабочих процессов. Плагин
полностью управляется рабочим процессом: OpenClaw не сопоставляет универсальные элементы управления `size`,
`aspectRatio`, `resolution`, `durationSeconds` или элементы управления в стиле TTS
с вашим графом.

| Свойство         | Сведения                                                                         |
| ---------------- | -------------------------------------------------------------------------------- |
| Провайдер        | `comfy`                                                               |
| Модель           | `comfy/workflow`                                                               |
| Общие инструменты | `image_generate`, `video_generate`, `music_generate`                      |
| Аутентификация   | Не требуется для локального ComfyUI; `COMFY_API_KEY` или `COMFY_CLOUD_API_KEY` для Comfy Cloud |
| API              | ComfyUI `/prompt` / `/history` / `/view`; Comfy Cloud `/api/*` |

## Поддерживаемые возможности

- Генерация и редактирование изображений на основе JSON рабочего процесса (для редактирования требуется 1 загруженное эталонное изображение)
- Генерация видео на основе JSON рабочего процесса: из текста или изображения (1 эталонное изображение)
- Генерация музыки и аудио через общий инструмент `music_generate`, необязательно с 1 эталонным изображением
- Скачивание результата из настроенного узла или из всех соответствующих выходных узлов, если узел не настроен

## Начало работы

Выберите между запуском ComfyUI на собственном компьютере и использованием Comfy Cloud.

<Tabs>
  <Tab title="Локально">
    **Лучше всего подходит для:** запуска собственного экземпляра ComfyUI на компьютере или в локальной сети.

    <Steps>
      <Step title="Запустите ComfyUI локально">
        Убедитесь, что локальный экземпляр ComfyUI запущен (по умолчанию используется `http://127.0.0.1:8188`).
      </Step>
      <Step title="Подготовьте JSON рабочего процесса">
        Экспортируйте или создайте JSON-файл рабочего процесса ComfyUI. Запишите идентификаторы узла ввода запроса и выходного узла, из которого OpenClaw должен считывать результат.
      </Step>
      <Step title="Настройте провайдер">
        Задайте `mode: "local"` и укажите файл рабочего процесса. Минимальный пример для изображений:

        ```json5
        {
          plugins: {
            entries: {
              comfy: {
                config: {
                  mode: "local",
                  baseUrl: "http://127.0.0.1:8188",
                  image: {
                    workflowPath: "./workflows/flux-api.json",
                    promptNodeId: "6",
                    outputNodeId: "9",
                  },
                },
              },
            },
          },
        }
        ```
      </Step>
      <Step title="Задайте модель по умолчанию">
        Настройте OpenClaw на использование модели `comfy/workflow` для заданной возможности:

        ```json5
        {
          agents: {
            defaults: {
              imageGenerationModel: {
                primary: "comfy/workflow",
              },
            },
          },
        }
        ```
      </Step>
      <Step title="Проверьте настройку">
        ```bash
        openclaw models list --provider comfy
        ```
      </Step>
    </Steps>

  </Tab>

  <Tab title="Comfy Cloud">
    **Лучше всего подходит для:** запуска рабочих процессов в Comfy Cloud без управления локальными ресурсами GPU.

    <Steps>
      <Step title="Получите ключ API">
        Зарегистрируйтесь на [comfy.org](https://comfy.org) и создайте ключ API на панели управления своей учетной записи.
      </Step>
      <Step title="Задайте ключ API">
        Передайте ключ любым из следующих способов:

        ```bash
        # Флаг первоначальной настройки
        openclaw onboard --comfy-api-key "your-key"

        # Переменная окружения (предпочтительно для фоновых служб)
        export COMFY_API_KEY="your-key"

        # Альтернативная переменная окружения
        export COMFY_CLOUD_API_KEY="your-key"

        # Или непосредственно в конфигурации
        openclaw config set plugins.entries.comfy.config.apiKey "your-key"
        ```
      </Step>
      <Step title="Подготовьте JSON рабочего процесса">
        Экспортируйте или создайте JSON-файл рабочего процесса ComfyUI. Запишите идентификаторы узла ввода запроса и выходного узла.
      </Step>
      <Step title="Настройте провайдер">
        Задайте `mode: "cloud"` и укажите файл рабочего процесса:

        ```json5
        {
          plugins: {
            entries: {
              comfy: {
                config: {
                  mode: "cloud",
                  image: {
                    workflowPath: "./workflows/flux-api.json",
                    promptNodeId: "6",
                    outputNodeId: "9",
                  },
                },
              },
            },
          },
        }
        ```

        <Tip>
        В облачном режиме для `baseUrl` по умолчанию используется `https://cloud.comfy.org`. Задавайте `baseUrl` только для пользовательской облачной конечной точки.
        </Tip>
      </Step>
      <Step title="Задайте модель по умолчанию">
        ```json5
        {
          agents: {
            defaults: {
              imageGenerationModel: {
                primary: "comfy/workflow",
              },
            },
          },
        }
        ```
      </Step>
      <Step title="Проверьте настройку">
        ```bash
        openclaw models list --provider comfy
        ```
      </Step>
    </Steps>

## Конфигурация

Comfy поддерживает общие параметры подключения верхнего уровня и разделы рабочих процессов для каждой возможности (`image`, `video`, `music`):

```json5
{
  plugins: {
    entries: {
      comfy: {
        config: {
          mode: "local",
          baseUrl: "http://127.0.0.1:8188",
          image: {
            workflowPath: "./workflows/flux-api.json",
            promptNodeId: "6",
            outputNodeId: "9",
          },
          video: {
            workflowPath: "./workflows/video-api.json",
            promptNodeId: "12",
            outputNodeId: "21",
          },
          music: {
            workflowPath: "./workflows/music-api.json",
            promptNodeId: "3",
            outputNodeId: "18",
          },
        },
      },
    },
  },
}
```

### Общие ключи

| Ключ                  | Тип                    | Описание                                                                              |
| --------------------- | ---------------------- | ------------------------------------------------------------------------------------- |
| `mode`    | `"local"` или `"cloud"` | Режим подключения. По умолчанию — `"local"`.                    |
| `baseUrl`    | строка                 | По умолчанию — `http://127.0.0.1:8188` для локального режима или `https://cloud.comfy.org` для облачного. |
| `apiKey`    | строка                 | Необязательный встроенный ключ, альтернатива переменным окружения `COMFY_API_KEY` / `COMFY_CLOUD_API_KEY`. |
| `allowPrivateNetwork`    | логическое значение    | Разрешает частный адрес или адрес локальной сети `baseUrl` в облачном режиме либо локальное полное доменное имя из частной DNS. |

<Note>
В режиме `local` литералы кольцевых и частных IP-адресов, а также одноуровневые имена служб, например `http://comfyui:8188`, работают без `allowPrivateNetwork`. Полные доменные имена частной DNS, выглядящие как публичные, например `https://comfy.local.example.com`, требуют `allowPrivateNetwork: true`. Доверие к частному источнику ограничивается настроенными схемой, именем хоста и портом; локальные перенаправления не могут покидать настроенное имя хоста, а облачные перенаправления на публичные CDN проверяются стандартной политикой SSRF.
</Note>

### Ключи отдельных возможностей

Эти ключи применяются внутри разделов `image`, `video` или `music`:

| Ключ                         | Обязателен | По умолчанию | Описание                                                                    |
| ---------------------------- | ---------- | ------------ | --------------------------------------------------------------------------- |
| `workflow` или `workflowPath` | Да | -- | Встроенный JSON рабочего процесса или путь к JSON-файлу рабочего процесса ComfyUI. |
| `promptNodeId`           | Да         | --           | Идентификатор узла, который получает текстовый запрос.                      |
| `promptInputName`           | Нет        | `"text"` | Имя входа в узле запроса.                                              |
| `outputNodeId`           | Нет        | --           | Идентификатор узла для чтения результата. Если не указан, используются все соответствующие выходные узлы. |
| `pollIntervalMs`           | Нет        | `1500` | Интервал опроса завершения задания в миллисекундах.                    |
| `timeoutMs`           | Нет        | `300000` | Время ожидания выполнения рабочего процесса в миллисекундах.          |

Разделы `image` и `video` также поддерживают узел ввода эталонного изображения:

| Ключ                  | Обязателен                                  | По умолчанию | Описание                                                  |
| --------------------- | ------------------------------------------- | ------------ | --------------------------------------------------------- |
| `inputImageNodeId`    | Да (при передаче эталонного изображения)    | --           | Идентификатор узла, который получает загруженное эталонное изображение. |
| `inputImageInputName`    | Нет                                         | `"image"` | Имя входа в узле изображения.                       |

`apiKey` принимает либо строковый литерал, либо объект [ссылки на секрет](/ru/gateway/configuration-reference#secrets).

## Сведения о рабочих процессах

<AccordionGroup>
  <Accordion title="Рабочие процессы изображений">
    Задайте для модели изображений по умолчанию значение `comfy/workflow`:

    ```json5
    {
      agents: {
        defaults: {
          imageGenerationModel: {
            primary: "comfy/workflow",
          },
        },
      },
    }
    ```

    **Пример редактирования с эталонным изображением:**

    Чтобы включить редактирование изображений с загруженным эталонным изображением, добавьте `inputImageNodeId` в конфигурацию изображений:

    ```json5
    {
      plugins: {
        entries: {
          comfy: {
            config: {
              image: {
                workflowPath: "./workflows/edit-api.json",
                promptNodeId: "6",
                inputImageNodeId: "7",
                inputImageInputName: "image",
                outputNodeId: "9",
              },
            },
          },
        },
      },
    }
    ```

  </Accordion>

  <Accordion title="Рабочие процессы видео">
    Задайте для модели видео по умолчанию значение `comfy/workflow`:

    ```json5
    {
      agents: {
        defaults: {
          videoGenerationModel: {
            primary: "comfy/workflow",
          },
        },
      },
    }
    ```

    Видеопроцессы Comfy поддерживают генерацию видео из текста и изображения с помощью настроенного графа.

    <Note>
    OpenClaw не передает входные видео в рабочие процессы Comfy. В качестве входных данных поддерживаются только текстовые запросы и одно эталонное изображение.
    </Note>

  </Accordion>

  <Accordion title="Рабочие процессы музыки">
    Встроенный плагин регистрирует провайдер генерации музыки для аудио- или музыкальных результатов, определенных рабочим процессом и доступных через общий инструмент `music_generate`. Он принимает необязательное эталонное изображение (до 1):

    ```text
    /tool music_generate prompt="Теплый эмбиентный синтезаторный луп с мягкой пленочной текстурой"
    ```

    Используйте раздел конфигурации `music`, чтобы указать JSON аудиопроцесса и выходной узел.

  </Accordion>

  <Accordion title="Обратная совместимость">
    Существующая конфигурация изображений верхнего уровня (без вложенного раздела `image`) продолжает работать:

    ```json5
    {
      plugins: {
        entries: {
          comfy: {
            config: {
              workflowPath: "./workflows/flux-api.json",
              promptNodeId: "6",
              outputNodeId: "9",
            },
          },
        },
      },
    }
    ```

    OpenClaw интерпретирует эту устаревшую структуру как конфигурацию рабочего процесса генерации изображений. Немедленная миграция не требуется, но для новых конфигураций рекомендуется использовать вложенные разделы `image` / `video` / `music`. Если вы используете только генерацию изображений, устаревшая плоская конфигурация и новый вложенный раздел `image` функционально эквивалентны.

  </Accordion>

  <Accordion title="Интерактивные тесты">
    Для встроенного плагина доступно подключаемое интерактивное тестовое покрытие:

    ```bash
    OPENCLAW_LIVE_TEST=1 COMFY_LIVE_TEST=1 pnpm test:live -- extensions/comfy/comfy.live.test.ts
    ```

    Интерактивный тест пропускает отдельные сценарии для изображений, видео или музыки, если не настроен соответствующий раздел рабочего процесса Comfy.

  </Accordion>
</AccordionGroup>
