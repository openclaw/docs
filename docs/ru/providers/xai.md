OpenClaw поставляется со встроенным плагином провайдера `xai` для моделей Grok.
Рекомендуемый способ — Grok OAuth с подходящей подпиской SuperGrok или X Premium.
Gateway, конфигурация, маршрутизация и инструменты остаются локальными; только запросы
Grok отправляются в API xAI.

Для OAuth не требуется API-ключ xAI или приложение Grok Build. xAI всё равно может
показывать Grok Build на экране согласия, поскольку OpenClaw использует общий
OAuth-клиент xAI.

## Настройка

<Steps>
  <Step title="Новая установка">
    Запустите первоначальную настройку с установкой демона, затем на этапе
    выбора модели и способа аутентификации выберите xAI/Grok OAuth:

    ```bash
    openclaw onboard --install-daemon
    ```

    На VPS или при подключении по SSH выберите xAI OAuth напрямую; этот способ
    использует проверку по коду устройства и не требует обратного вызова localhost:

    ```bash
    openclaw onboard --install-daemon --auth-choice xai-oauth
    ```

  </Step>
  <Step title="Существующая установка">
    Войдите только в xAI; не запускайте полную первоначальную настройку повторно
    лишь для подключения Grok:

    ```bash
    openclaw models auth login --provider xai --method oauth
    ```

    Отдельно назначьте Grok моделью по умолчанию:

    ```bash
    openclaw models set xai/grok-4.3
    ```

    Повторно запускайте полную первоначальную настройку, только если намеренно
    хотите изменить Gateway, демон, канал, рабочую область или другие параметры настройки.

  </Step>
  <Step title="Вариант с API-ключом">
    Настройка с API-ключом по-прежнему работает для ключей xAI Console и
    мультимедийных интерфейсов, которым требуется конфигурация провайдера с ключом:

    ```bash
    openclaw models auth login --provider xai --method api-key
    export XAI_API_KEY=xai-...
    ```

  </Step>
  <Step title="Выбор модели">
    ```json5
    {
      agents: { defaults: { model: { primary: "xai/grok-4.3" } } },
    }
    ```
  </Step>
</Steps>

<Note>
OpenClaw использует xAI Responses API как встроенный транспорт xAI. Те же
учётные данные из `openclaw models auth login --provider xai --method oauth` или
`--method api-key` также обеспечивают работу `web_search` (идентификатор провайдера `grok`), `x_search`,
`code_execution`, синтеза и транскрибирования речи, а также генерации изображений и видео xAI. Если
сохранить ключ xAI в `plugins.entries.xai.config.webSearch.apiKey`,
встроенный провайдер моделей xAI также использует его как резервный вариант.
</Note>

## Устранение неполадок OAuth

- Для SSH, Docker, VPS и других удалённых сред используйте
  `openclaw models auth login --provider xai --method oauth`; этот способ использует
  проверку по коду устройства, а не обратный вызов localhost.
- Если вход выполнен успешно, но Grok не является моделью по умолчанию, выполните
  `openclaw models set xai/grok-4.3`.
- Просмотрите сохранённые профили аутентификации xAI:

  ```bash
  openclaw models auth list --provider xai
  openclaw models status
  ```

- xAI определяет, какие учётные записи могут получать API-токены OAuth. Если учётная запись
  не соответствует требованиям, используйте вариант с API-ключом или проверьте подписку на стороне xAI.

<Tip>
Используйте `xai-oauth` при входе через SSH, Docker или VPS. OpenClaw выводит
URL и короткий код; завершите вход в любом локальном браузере, пока удалённый
процесс опрашивает xAI в ожидании завершения обмена токенами.
</Tip>

## Встроенный каталог

Идентификаторы, доступные в средствах выбора моделей. Плагин по-прежнему распознаёт старые
идентификаторы Grok 3, Grok 4, Grok 4 Fast, Grok 4.1 Fast и Grok Code для существующих конфигураций;
см. [совместимость с устаревшими версиями и изменяемые псевдонимы](#legacy-compatibility-and-moving-aliases).

| Семейство      | Идентификаторы моделей                                        |
| -------------- | ------------------------------------------------------------ |
| Grok 4.5       | `grok-4.5` (псевдонимы: `grok-4.5-latest`, `grok-build-latest`) |
| Grok Build 0.1 | `grok-build-0.1`                                             |
| Grok 4.3       | `grok-4.3` (псевдонимы: `grok-4.3-latest`, `grok-latest`)       |
| Grok 4.20      | `grok-4.20-0309-reasoning`, `grok-4.20-0309-non-reasoning`   |

<Tip>
Используйте `grok-4.5` для обычного общения, программирования и агентных задач, где эта модель доступна.
Grok 4.3 остаётся безопасной для разных регионов моделью настройки по умолчанию; `grok-build-0.1` и оба
датированных варианта Grok 4.20 по-прежнему доступны для выбора.
</Tip>

## Поддерживаемые возможности

Встроенный плагин сопоставляет поддерживаемые API xAI с общими контрактами провайдеров и
инструментов OpenClaw. Возможности, которые не соответствуют общему контракту, перечислены
ниже или в разделе известных ограничений.

| Возможность xAI             | Интерфейс OpenClaw                       | Статус                                               |
| -------------------------- | --------------------------------------- | ---------------------------------------------------- |
| Чат / Responses            | Провайдер моделей `xai/<model>`         | Да                                                   |
| Серверный веб-поиск        | Провайдер `web_search` `grok`        | Да                                                   |
| Серверный поиск в X        | Инструмент `x_search`                  | Да                                                   |
| Серверное выполнение кода  | Инструмент `code_execution`            | Да                                                   |
| Изображения                | `image_generate`                        | Да                                                   |
| Видео                      | `video_generate`                        | Да                                                   |
| Пакетный синтез речи       | `messages.tts.provider: "xai"` / `tts`  | Да                                                   |
| Потоковый TTS              | `textToSpeechStream`                    | Да, через `wss://api.x.ai/v1/tts` (не голос в реальном времени) |
| Пакетное распознавание речи | Анализ мультимедиа `tools.media.audio` | Да                                                   |
| Потоковое распознавание речи | Voice Call `streaming.provider: "xai"` | Да                                                   |
| Голос в реальном времени  | Talk `talk.realtime.provider: "xai"`    | Да; ретрансляция через Gateway для нативных узлов Talk |
| Файлы / пакеты             | Только совместимость с универсальным API моделей | Не является полноценным инструментом OpenClaw       |

<Note>
OpenClaw использует REST API xAI для изображений, видео, TTS и STT при генерации мультимедиа и
пакетном транскрибировании, потоковый WebSocket STT xAI для транскрибирования
голосовых вызовов в реальном времени, WebSocket Grok Voice Agent от xAI для сеансов Talk
в реальном времени и Responses API для чата, поиска и инструментов выполнения кода.
</Note>

### Совместимость с устаревшим быстрым режимом

`/fast on` или `agents.defaults.models["xai/<model>"].params.fastMode: true`
по-прежнему преобразует старые конфигурации xAI следующим образом. Эти целевые идентификаторы
сохраняются только для совместимости; для новых конфигураций используйте текущие
доступные для выбора модели.

| Исходная модель | Цель быстрого режима |
| --------------- | -------------------- |
| `grok-3`      | `grok-3-fast`      |
| `grok-3-mini` | `grok-3-mini-fast` |
| `grok-4`      | `grok-4-fast`      |
| `grok-4-0709` | `grok-4-fast`      |

### Совместимость с устаревшими версиями и изменяемые псевдонимы

Старые псевдонимы нормализуются следующим образом:

| Устаревший псевдоним                                        | Нормализованный идентификатор |
| ------------------------------------------------------------- | ---------------- |
| `grok-code-fast-1`, `grok-code-fast`, `grok-code-fast-1-0825` | `grok-build-0.1` |

Датированные идентификаторы 0309 являются доступными для выбора записями каталога. OpenClaw передаёт все остальные
текущие псевдонимы Grok 4.20 без изменений, чтобы xAI сохраняла контроль над семантикой стабильных,
новейших, бета-, экспериментальных и датированных псевдонимов. Глобальный псевдоним `grok-latest`
также сохраняется без изменений.

xAI прекратила поддержку следующих точных идентификаторов. OpenClaw сохраняет их как скрытые строки
совместимости для выпущенных конфигураций с ограничениями и ценами их текущих
целей перенаправления:

| Устаревшие идентификаторы                                        | Текущее поведение                |
| -------------------------------------------------------------------- | -------------------------------- |
| `grok-4-1-fast-reasoning`, `grok-4-fast-reasoning`, `grok-4-0709`    | Grok 4.3 с рассуждением `low` |
| `grok-4-1-fast-non-reasoning`, `grok-4-fast-non-reasoning`, `grok-3` | Grok 4.3 с отключённым рассуждением |
| `grok-code-fast-1`                                                   | Grok Build 0.1                   |
| `grok-imagine-image-pro`                                             | Качество изображений Grok Imagine |

`openclaw doctor --fix` обновляет сохранённые значения по умолчанию для серверных инструментов xAI и
устаревший идентификатор качества изображений, удаляет неактуальные сгенерированные строки каталога и исправляет
устаревшие метаданные контекста в активных строках 4.20. Она не закрепляет активные псевдонимы
4.20 `beta-latest` за датированным снимком.

## Возможности

<Warning>
  `x_search` и `code_execution` выполняются на серверах xAI. xAI взимает $5 за 1,000
  вызовов инструментов в дополнение к входным и выходным токенам модели. Если настройка
  `enabled` каждого инструмента не указана, OpenClaw предоставляет его только для активной модели xAI.
  Для известного провайдера моделей, отличного от xAI, необходимо явно указать `enabled: true` для каждого инструмента;
  при отсутствии или невозможности определить провайдера доступ блокируется. Аутентификация xAI требуется всегда,
  а `enabled: false` отключает инструмент для всех провайдеров.
</Warning>

  <Accordion title="Генерация видео">
    Встроенный плагин `xai` регистрирует генерацию видео через общий
    инструмент `video_generate`.

    - Модель по умолчанию: `xai/grok-imagine-video`
    - Дополнительная модель: `xai/grok-imagine-video-1.5`
    - Классические режимы: преобразование текста в видео, изображения в видео, генерация
      по эталонным изображениям, удалённое редактирование видео и удалённое продолжение видео
    - Режим Video 1.5: только преобразование изображения в видео с ровно одним изображением
      первого кадра
    - Соотношения сторон: `1:1`, `16:9`, `9:16`, `4:3`, `3:4`, `3:2`, `2:3`;
      если значение не указано, классический режим и режим Video 1.5 преобразования изображения в видео
      наследуют соотношение сторон исходного изображения
    - Разрешения: классические `480P`/`720P`; Video 1.5 также поддерживает `1080P`; во всех
      режимах генерации по умолчанию используется `480P`
    - Длительность: 1-15 секунд для генерации и преобразования изображения в видео, 1-10 секунд при
      использовании классических ролей `reference_image`, 2-10 секунд для классического продолжения
    - Генерация по эталонным изображениям: задайте `imageRoles` равным `reference_image` для
      каждого предоставленного изображения; xAI принимает до 7 таких изображений
    - При редактировании и продолжении видео наследуются соотношение сторон и разрешение входного видео;
      эти операции не допускают переопределения геометрии
    - Тайм-аут операции по умолчанию: 600 секунд, если не задано `video_generate.timeoutMs`
      или `agents.defaults.videoGenerationModel.timeoutMs`

    <Warning>
    Локальные буферы видео не принимаются. Для входных данных редактирования и продолжения видео
    используйте удалённые URL-адреса `http(s)`. Преобразование изображения в видео принимает локальные
    буферы изображений, поскольку OpenClaw кодирует их для xAI как URL-адреса данных.
    </Warning>

    Video 1.5 также распознаёт идентификаторы xAI `grok-imagine-video-1.5-preview` и
    `grok-imagine-video-1.5-2026-05-30`. OpenClaw передаёт выбранный
    идентификатор без изменений, но применяет ту же проверку, допускающую только изображения.

    Чтобы использовать xAI как провайдера видео по умолчанию:

    ```json5
    {
      agents: {
        defaults: {
          videoGenerationModel: {
            primary: "xai/grok-imagine-video",
          },
        },
      },
    }
    ```

    <Note>
    Общие параметры инструмента, выбор провайдера и поведение при переключении
    на резервный вариант описаны в разделе [Генерация видео](/ru/tools/video-generation).
    </Note>

  </Accordion>

  <Accordion title="Генерация изображений">
    Встроенный плагин `xai` регистрирует генерацию изображений через общий
    инструмент `image_generate`.

    - Модель изображений по умолчанию: `xai/grok-imagine-image`
    - Дополнительная модель: `xai/grok-imagine-image-quality`
    - Режимы: преобразование текста в изображение и редактирование по референсному изображению
    - Входные референсы: один `image` или до трёх `images`
    - Соотношения сторон: `1:1`, `16:9`, `9:16`, `4:3`, `3:4`, `3:2`, `2:3`, `2:1`,
      `1:2`, `19.5:9`, `9:19.5`, `20:9`, `9:20`
    - Разрешения: `1K`, `2K`
    - Количество: до 4 изображений
    - Тайм-аут операции по умолчанию: 600 секунд, если не задан
      `image_generate.timeoutMs` или `agents.defaults.imageGenerationModel.timeoutMs`

    OpenClaw запрашивает у xAI ответы с изображениями в формате `b64_json`, чтобы созданные медиафайлы можно было
    сохранять и доставлять через обычный путь вложений канала. Локальные
    референсные изображения преобразуются в URL-адреса данных; удалённые ссылки `http(s)`
    передаются без изменений.

    Чтобы использовать xAI как поставщика изображений по умолчанию:

    ```json5
    {
      agents: {
        defaults: {
          imageGenerationModel: {
            primary: "xai/grok-imagine-image",
          },
        },
      },
    }
    ```

    <Note>
    В документации xAI также указаны `quality`, `mask`, `user` и соотношение сторон `auto`.
    Сейчас OpenClaw передаёт только общие для разных поставщиков параметры управления изображениями;
    эти специфичные для xAI параметры недоступны через `image_generate`.
    </Note>

  </Accordion>

  <Accordion title="Преобразование текста в речь">
    Встроенный плагин `xai` регистрирует преобразование текста в речь через общий
    интерфейс поставщика `tts`.

    - Голоса: аутентифицированный актуальный каталог xAI; выведите его с помощью
      `openclaw infer tts voices --provider xai`
    - Резервные голоса для автономного режима: `ara`, `eve`, `leo`, `rex`, `sal`
    - Голос по умолчанию: `eve`
    - Идентификаторы пользовательских голосов учётной записи передаются, даже если они отсутствуют
      в ответе встроенного каталога
    - Форматы: `mp3`, `wav`, `pcm`, `mulaw`, `alaw`
    - Язык: код BCP-47 или `auto`
    - Скорость: собственное переопределение скорости поставщика
    - Собственный формат голосовых сообщений Opus не поддерживается

    Чтобы использовать xAI как поставщика TTS по умолчанию:

    ```json5
    {
      messages: {
        tts: {
          provider: "xai",
          providers: {
            xai: {
              voiceId: "eve",
            },
          },
        },
      },
    }
    ```

    <Note>
    OpenClaw использует пакетную конечную точку xAI `/v1/tts` для буферизованного синтеза,
    аутентифицированное обнаружение каталога `/v1/tts/voices` и собственный
    `wss://api.x.ai/v1/tts` для потокового синтеза. Потоковая передача ограничена
    собственным узлом `api.x.ai`, поэтому пользовательские значения `baseUrl` в этом
    пути отклоняются. Используются существующие параметры языка, голоса, кодека и скорости; для
    частоты дискретизации и битрейта применяются значения xAI по умолчанию. При синтезе аудиофайлов учитываются все
    настроенные кодеки. Для целевых голосовых сообщений при потоковой передаче и буферизованном
    резервном режиме используется MP3, поскольку необработанные кодеки xAI не содержат метаданных кодека и частоты. Поток
    отправляет `text.delta`, затем
    `text.done`, получает `audio.delta`, `audio.done` или `error` и применяет
    тайм-аут простоя `timeoutMs`, который обновляется для каждого фрагмента аудио. Этот механизм не связан
    с сеансами голосовой связи в реальном времени. См. контракт xAI [API потокового TTS](https://docs.x.ai/developers/rest-api-reference/inference/voice).
    </Note>

  </Accordion>

  <Accordion title="Преобразование речи в текст">
    Встроенный плагин `xai` регистрирует пакетное преобразование речи в текст через
    интерфейс транскрибирования для анализа медиафайлов OpenClaw.

    - Конечная точка: xAI REST `/v1/stt`
    - Путь ввода: загрузка аудиофайла как составных данных
    - Выбор модели: xAI выбирает модель транскрибирования автоматически; в этой
      конечной точке нет параметра выбора модели
    - Используется везде, где при транскрибировании входящего аудио считывается `tools.media.audio`,
      включая сегменты голосовых каналов Discord и аудиовложения каналов

    Чтобы принудительно использовать xAI для транскрибирования входящего аудио:

    ```json5
    {
      tools: {
        media: {
          audio: {
            models: [
              {
                type: "provider",
                provider: "xai",
              },
            ],
          },
        },
      },
    }
    ```

    Язык можно указать в общей конфигурации аудиомедиа или в отдельном
    запросе транскрибирования. Общий интерфейс OpenClaw принимает подсказки,
    но интеграция xAI REST STT передаёт только файл и язык,
    поскольку только они соответствуют текущей общедоступной конечной точке xAI.

  </Accordion>

  <Accordion title="Потоковое преобразование речи в текст">
    Встроенный плагин `xai` также регистрирует поставщика транскрибирования в реальном времени
    для аудио голосовых вызовов.

    - Конечная точка: xAI WebSocket `wss://api.x.ai/v1/stt`
    - Кодирование по умолчанию: `mulaw`
    - Частота дискретизации по умолчанию: `8000`
    - Определение конца речи по умолчанию: `800ms`
    - Промежуточные транскрипции: включены по умолчанию

    Медиапоток Twilio для голосовых вызовов отправляет аудиокадры G.711 mu-law, поэтому
    поставщик xAI передаёт эти кадры напрямую без перекодирования:

    ```json5
    {
      plugins: {
        entries: {
          "voice-call": {
            config: {
              streaming: {
                enabled: true,
                provider: "xai",
                providers: {
                  xai: {
                    apiKey: "${XAI_API_KEY}",
                    endpointingMs: 800,
                    language: "en",
                  },
                },
              },
            },
          },
        },
      },
    }
    ```

    Конфигурация поставщика находится в
    `plugins.entries.voice-call.config.streaming.providers.xai`. Поддерживаются
    ключи `apiKey`, `baseUrl`, `sampleRate`, `encoding` (`pcm`, `mulaw` или
    `alaw`), `interimResults`, `endpointingMs` и `language`.

    <Note>
    Этот поставщик потоковой передачи предназначен для пути транскрибирования голосовых вызовов в реальном времени.
    Discord записывает короткие сегменты и вместо него использует путь пакетного
    транскрибирования `tools.media.audio`.
    </Note>

  </Accordion>

  <Accordion title="Голосовая связь в реальном времени (Talk)">
    Встроенный плагин `xai` регистрирует сеансы Grok Voice Agent в реальном времени для
    режима Talk через общий контракт `registerRealtimeVoiceProvider`.

    - Конечная точка: `wss://api.x.ai/v1/realtime?model=<voice-model>`
    - Модель по умолчанию: `grok-voice-latest`
    - Голос по умолчанию: `eve`
    - Транспорт: `gateway-relay` (пути ретрансляции iOS, Android и Control UI)
    - Аудио: PCM16 24 кГц или G.711 µ-law 8 кГц
    - Перебивание: серверный VAD xAI прерывает ответ; OpenClaw очищает очередь воспроизведения
      и обрезает невоспроизведённую историю поставщика

    Настройте Talk на Gateway:

    ```json5
    {
      talk: {
        realtime: {
          provider: "xai",
          mode: "realtime",
          transport: "gateway-relay",
          brain: "agent-consult",
          providers: {
            xai: {
              model: "grok-voice-latest",
              voice: "eve",
              // Включайте только в том случае, если допустимо воспроизведение сеанса на стороне поставщика.
              sessionResumption: false,
            },
          },
        },
      },
      env: { XAI_API_KEY: "xai-..." },
    }
    ```

    Конфигурация поставщика также разрешается из
    `plugins.entries.voice-call.config.realtime.providers.xai`, когда Voice Call
    или общие селекторы реального времени повторно используют ту же карту поставщиков. Поддерживаются ключи
    `apiKey`, `baseUrl`, `model`, `voice`, `vadThreshold`, `silenceDurationMs`,
    `prefixPaddingMs`, `reasoningEffort` и `sessionResumption`.
    `reasoningEffort` принимает только `high` или `none`, согласно API xAI Voice Agent.

    Серверный VAD xAI всегда создаёт ответы и обрабатывает прерывание аудио.
    Используйте `consultRouting: "provider-direct"`; принудительная маршрутизация транскрипций и отключение
    прерывания входящего аудио не поддерживаются протоколом xAI Voice Agent.

    <Note>
    Для аутентификации голосовой связи в реальном времени можно использовать xAI OAuth или `XAI_API_KEY`.
    WebRTC под управлением браузера пока не входит в интерфейс этого поставщика; используйте Talk с ретрансляцией через Gateway
    на нативных узлах или путь ретрансляции Control UI.
    </Note>

    <Note>
    По умолчанию `sessionResumption` имеет значение `false`. Если задано значение `true`, OpenClaw просит
    xAI сохранить достаточно состояния сеанса, чтобы после
    повторного подключения продолжить тот же разговор, а затем подключается повторно с возвращённым идентификатором разговора. Оставьте этот параметр
    отключённым, если воспроизведение или хранение данных на стороне поставщика неприемлемо; в таком случае при обрыве
    соединения происходит безопасный отказ вместо незаметного запуска нового разговора.
    </Note>

  </Accordion>

  <Accordion title="Настройка x_search">
    Встроенный плагин xAI предоставляет `x_search` как инструмент OpenClaw для
    поиска содержимого X (ранее Twitter) с помощью Grok.

    Путь конфигурации: `plugins.entries.xai.config.xSearch`

    | Ключ              | Тип     | По умолчанию              | Описание                                                     |
    | ----------------- | ------- | ------------------------- | ------------------------------------------------------------ |
    | `enabled`         | логический | Автоматически для моделей xAI | Отключить или включить для известного поставщика, отличного от xAI |
    | `model`           | строка  | `grok-4.3`                | Модель, используемая для запросов x_search                   |
    | `baseUrl`         | строка  | -                         | Переопределение базового URL-адреса xAI Responses            |
    | `inlineCitations` | логический | -                         | Включать встроенные ссылки на источники в результаты         |
    | `maxTurns`        | число   | -                         | Максимальное количество ходов диалога                       |
    | `timeoutSeconds`  | число   | `30`                      | Тайм-аут запроса в секундах                                  |
    | `cacheTtlMinutes` | число   | `15`                      | Время жизни кэша в минутах                                   |

    ```json5
    {
      plugins: {
        entries: {
          xai: {
            config: {
              xSearch: {
                enabled: true,
                model: "grok-4.3",
                baseUrl: "https://api.x.ai/v1",
                inlineCitations: true,
              },
            },
          },
        },
      },
    }
    ```

  </Accordion>

  <Accordion title="Настройка выполнения кода">
    Встроенный плагин xAI предоставляет `code_execution` как инструмент OpenClaw для
    удалённого выполнения кода в изолированной среде xAI.

    Путь конфигурации: `plugins.entries.xai.config.codeExecution`

    | Ключ             | Тип     | По умолчанию             | Описание                                                     |
    | ---------------- | ------- | ------------------------ | ------------------------------------------------------------ |
    | `enabled`        | логический | Автоматически для моделей xAI | Отключить или включить для известного поставщика, отличного от xAI |
    | `model`          | строка  | `grok-4.3`               | Модель, используемая для запросов выполнения кода            |
    | `maxTurns`       | число   | -                        | Максимальное количество ходов диалога                       |
    | `timeoutSeconds` | число   | `30`                     | Тайм-аут запроса в секундах                                  |

    <Note>
    Это удалённое выполнение в изолированной среде xAI, а не локальный [`exec`](/ru/tools/exec).
    </Note>

    ```json5
    {
      plugins: {
        entries: {
          xai: {
            config: {
              codeExecution: {
                enabled: true,
                model: "grok-4.3",
              },
            },
          },
        },
      },
    }
    ```

  </Accordion>

  <Accordion title="Известные ограничения">
    - Для аутентификации xAI можно использовать ключ API, переменную окружения, резервную конфигурацию плагина или OAuth с подходящей учётной записью xAI. OAuth использует проверку по коду устройства без обратного вызова через localhost. xAI определяет, какие учётные записи могут получать токены API OAuth, а на странице согласия может отображаться Grok Build, хотя OpenClaw не требует приложения Grok Build.
    - В настоящее время OpenClaw не предоставляет доступ к семейству мультиагентных моделей xAI. xAI обслуживает эти модели через Responses API, но они не принимают клиентские или пользовательские инструменты, используемые общим циклом агента OpenClaw. См.
      [ограничения мультиагентного режима xAI](https://docs.x.ai/developers/model-capabilities/text/multi-agent#limitations).
    - В настоящее время голосовой режим xAI Realtime предоставляет только транспорт Talk через ретрансляцию Gateway. Сеансы WebSocket провайдера, управляемые браузером, пока не подключены в интерфейсе управления.
    - Изображение xAI `quality`, изображение `mask` и дополнительные соотношения сторон, доступные только в нативном режиме, не предоставляются, пока в общем инструменте `image_generate` не появятся соответствующие межпровайдерные элементы управления.

  </Accordion>

  <Accordion title="Дополнительные сведения">
    - OpenClaw автоматически применяет исправления совместимости схем инструментов и вызовов инструментов, специфичные для xAI, в общем пути выполнения.
    - Для нативных запросов xAI по умолчанию используется `tool_stream: true`. Чтобы отключить это, задайте для `agents.defaults.models["xai/<model>"].params.tool_stream` значение `false`.
    - Встроенная обёртка xAI удаляет неподдерживаемые ограничения количества вхождений в схемах и неподдерживаемые ключи *усилия* рассуждения из полезной нагрузки перед отправкой нативных запросов xAI. Grok 4.5 поддерживает низкое, среднее и высокое усилие (по умолчанию — высокое). Grok 4.3 поддерживает отсутствие усилия, низкое, среднее и высокое усилие (по умолчанию — низкое). Другие модели xAI с поддержкой рассуждений не предоставляют настраиваемого управления усилием, но по-прежнему запрашивают `include: ["reasoning.encrypted_content"]`, чтобы ранее зашифрованные рассуждения можно было повторно использовать в последующих ходах.
    - `web_search`, `x_search` и `code_execution` предоставляются как инструменты OpenClaw. OpenClaw добавляет к запросу каждого инструмента только конкретный встроенный инструмент xAI, который ему необходим, вместо добавления всех нативных инструментов к каждому ходу чата.
    - Grok `web_search` считывает `plugins.entries.xai.config.webSearch.baseUrl`. `x_search` считывает `plugins.entries.xai.config.xSearch.baseUrl`, а затем использует базовый URL веб-поиска Grok в качестве резервного варианта.
    - `x_search` и `code_execution` принадлежат встроенному плагину xAI, а не жёстко заданы в основном исполняющем модуле моделей.
    - `code_execution` — это удалённое выполнение в песочнице xAI, а не локальный [`exec`](/ru/tools/exec).

  </Accordion>
</AccordionGroup>

## Тестирование в реальном окружении

Медиапути xAI покрыты модульными тестами и подключаемыми наборами проверок в реальной среде. Экспортируйте
`XAI_API_KEY` в окружение процесса перед запуском проверок в реальной среде.

```bash
pnpm test extensions/xai
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_TEST_QUIET=1 pnpm test:live -- extensions/xai/xai.live.test.ts
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_XAI_VIDEO=1 pnpm test:live -- extensions/xai/xai.live.test.ts -t "classic Grok Imagine"
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_XAI_VIDEO=1 pnpm test:live -- extensions/xai/xai.live.test.ts -t "Grok Imagine Video 1.5"
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_TEST_QUIET=1 pnpm test:live -- extensions/xai/x-search.live.test.ts
OPENCLAW_LIVE_GATEWAY_MODELS="xai/grok-4.5,xai/grok-build-0.1,xai/grok-4.3,xai/grok-4.20-0309-reasoning,xai/grok-4.20-0309-non-reasoning" OPENCLAW_LIVE_GATEWAY_MAX_MODELS=0 OPENCLAW_LIVE_GATEWAY_SMOKE=0 pnpm test:live -- src/gateway/gateway-models.profiles.live.test.ts
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_TEST_QUIET=1 OPENCLAW_LIVE_IMAGE_GENERATION_PROVIDERS=xai pnpm test:live -- test/image-generation.runtime.live.test.ts
```

Файл проверок в реальной среде для конкретного провайдера синтезирует обычную речь TTS и пригодную для телефонии речь TTS в формате PCM, транскрибирует аудио через пакетный STT xAI, передаёт тот же PCM в потоковом режиме через STT xAI в реальном времени, генерирует изображение из текста и редактирует эталонное изображение.
Общий файл проверок изображений в реальной среде проверяет того же провайдера xAI через пути выбора среды выполнения OpenClaw, резервного переключения, нормализации и прикрепления медиафайлов. Подключаемый сценарий Video 1.5 отправляет одно сгенерированное изображение первого кадра в разрешении 1080P и проверяет загрузку готового видео.
