Z.AI — это API-платформа для моделей **GLM**. Она предоставляет REST API для GLM и
использует API-ключи для аутентификации. Создайте API-ключ в консоли Z.AI.
OpenClaw использует провайдер `zai` с API-ключом Z.AI.

| Свойство | Значение                                     |
| -------- | -------------------------------------------- |
| Провайдер | `zai`                                        |
| Пакет    | `@openclaw/zai-provider`                     |
| Аутентификация | `ZAI_API_KEY` (устаревший псевдоним: `Z_AI_API_KEY`) |
| API      | Z.AI Chat Completions (аутентификация Bearer) |

## Модели GLM

GLM — это семейство моделей, а не отдельный провайдер. В OpenClaw модели GLM используют
ссылки вида `zai/glm-5.2`: провайдер `zai`, идентификатор модели `glm-5.2`.

## Начало работы

Сначала установите плагин провайдера:

```bash
openclaw plugins install @openclaw/zai-provider
```

<Tabs>

### Конечные точки

| Вариант первоначальной настройки | Базовый URL                                   | Модель по умолчанию |
| ------------------- | --------------------------------------------- | ------------- |
| `zai-global`        | `https://api.z.ai/api/paas/v4`                | `glm-5.1`     |
| `zai-cn`            | `https://open.bigmodel.cn/api/paas/v4`        | `glm-5.1`     |
| `zai-coding-global` | `https://api.z.ai/api/coding/paas/v4`         | `glm-5.2`     |
| `zai-coding-cn`     | `https://open.bigmodel.cn/api/coding/paas/v4` | `glm-5.2`     |

`zai-api-key` автоматически определяет одну из этих четырёх конечных точек, проверяя ваш ключ через API
завершения чата каждой конечной точки: сначала проверяются общие конечные точки (`zai-global`,
затем `zai-cn`), а после них — конечные точки Coding Plan (`zai-coding-global`, затем
`zai-coding-cn`). Проверка останавливается на первой конечной точке, которая принимает запрос.
Используйте явно заданный `--auth-choice`, чтобы принудительно выбрать конечную точку Coding Plan, если ваш ключ
работает с обоими типами.

## Пример конфигурации

<Tip>
`zai-api-key` позволяет OpenClaw определить по ключу подходящую конечную точку Z.AI и
автоматически применить правильный базовый URL. Используйте явно заданные региональные варианты,
если требуется принудительно выбрать определённый Coding Plan или общий API.
</Tip>

```json5
{
  env: { ZAI_API_KEY: "sk-..." },
  models: {
    providers: {
      zai: {
        // GLM-5.2 использует конечную точку Coding Plan.
        baseUrl: "https://api.z.ai/api/coding/paas/v4",
      },
    },
  },
  agents: { defaults: { model: { primary: "zai/glm-5.2" } } },
}
```

## Встроенный каталог

Плагин провайдера `zai` поставляет свой каталог в манифесте плагина, поэтому при выводе
списка в режиме только для чтения можно показывать известные строки GLM без загрузки среды выполнения провайдера:

```bash
openclaw models list --all --provider zai
```

В каталог на основе манифеста сейчас входят:

| Ссылка на модель     | Примечания                      |

| `zai/glm-5.1`        | Модель общего API по умолчанию  |
| `zai/glm-5`          |                                 |
| `zai/glm-5-turbo`    |                                 |
| `zai/glm-5v-turbo`   |                                 |

| `zai/glm-4.7-flashx` |                                 |
| `zai/glm-4.6`        |                                 |
| `zai/glm-4.6v`       |                                 |
| `zai/glm-4.5`        |                                 |
| `zai/glm-4.5-air`    |                                 |
| `zai/glm-4.5-flash`  |                                 |

<Tip>
Модели GLM доступны как `zai/<model>` (пример: `zai/glm-5`).
</Tip>

<Note>
При настройке Coding Plan по умолчанию используется `zai/glm-5.2`; при настройке общего API сохраняется
`zai/glm-5.1`. На конечных точках Coding Plan автоматическое определение переключается на
`glm-5.1`, а затем на `glm-4.7`, если ключ или план не предоставляет GLM-5.2. Версии и
доступность GLM могут меняться; выполните `openclaw models list --all --provider zai`,
чтобы просмотреть каталог, известный установленной версии.
</Note>

## Уровни рассуждений

<Tabs>
  <Tab title="GLM-5.2">
    Полный диапазон: `off`, `low`, `high`, `max` (по умолчанию `off`). OpenClaw сопоставляет
    `low` и `high` с уровнем интенсивности рассуждений Z.AI `high`, а `max` — с
    уровнем `max` Z.AI через `reasoning_effort` в полезной нагрузке запроса.
  </Tab>
  <Tab title="Другие модели GLM">
    Только двоичный переключатель: `off` и `low` (в средствах выбора отображаются как `on`), по умолчанию
    `off`. Если для рассуждений задано значение `off`, отправляется `thinking: { type: "disabled" }`;
    при любом другом уровне полезная нагрузка запроса остаётся без изменений (применяется собственное
    поведение рассуждений Z.AI по умолчанию).
  </Tab>
</Tabs>

Установка уровня рассуждений в `off` предотвращает ответы, которые расходуют бюджет вывода на
`reasoning_content` до появления видимого текста.

## Расширенная конфигурация

<AccordionGroup>
  <Accordion title="Предварительное разрешение неизвестных моделей GLM-5">
    Неизвестные идентификаторы `glm-5*` по-прежнему предварительно разрешаются на пути провайдера путём
    создания принадлежащих провайдеру метаданных из шаблона `glm-4.7`, если идентификатор
    соответствует текущему формату семейства GLM-5.
  </Accordion>

  <Accordion title="Потоковая передача вызовов инструментов">
    `tool_stream` по умолчанию включён для потоковой передачи вызовов инструментов Z.AI. Чтобы отключить его:

    ```json5
    {
      agents: {
        defaults: {
          models: {
            "zai/<model>": {
              params: { tool_stream: false },
            },
          },
        },
      },
    }
    ```

  </Accordion>

  <Accordion title="Сохранённые рассуждения">
    Сохранение рассуждений включается явно, поскольку Z.AI требует повторной передачи полной исторической
    последовательности `reasoning_content`, что увеличивает число токенов запроса. Включите эту функцию
    отдельно для каждой модели:

    ```json5
    {
      agents: {
        defaults: {
          models: {
            "zai/glm-5.2": {
              params: { preserveThinking: true },
            },
          },
        },
      },
    }
    ```

    Когда функция включена и рассуждения активны, OpenClaw отправляет
    `thinking: { type: "enabled", clear_thinking: false }` и повторно передаёт предыдущие
    `reasoning_content` для того же OpenAI-совместимого транскрипта. Ключ параметра в формате snake_case
    `preserve_thinking` работает как псевдоним.

    Опытные пользователи по-прежнему могут переопределить точную полезную нагрузку провайдера с помощью
    `params.extra_body.thinking`.

  </Accordion>

  <Accordion title="Распознавание изображений">
    Плагин Z.AI регистрирует функцию распознавания изображений.

    | Свойство      | Значение    |
    | ------------- | ----------- |
    | Модель        | `glm-4.6v`  |

    Распознавание изображений автоматически определяется на основе настроенной аутентификации Z.AI —
    дополнительная конфигурация не требуется.

  </Accordion>

  <Accordion title="Сведения об аутентификации">
    - Z.AI использует аутентификацию Bearer с вашим API-ключом.
    - Вариант первоначальной настройки `zai-api-key` автоматически определяет подходящую конечную точку Z.AI, проверяя поддерживаемые конечные точки с вашим ключом.
    - Используйте явно заданные региональные варианты (`zai-coding-global`, `zai-coding-cn`, `zai-global`, `zai-cn`), если требуется принудительно выбрать определённый API.
    - Устаревшая переменная среды `Z_AI_API_KEY` по-прежнему поддерживается; OpenClaw копирует её в `ZAI_API_KEY` при запуске, если `ZAI_API_KEY` не задана.

  </Accordion>
</AccordionGroup>
