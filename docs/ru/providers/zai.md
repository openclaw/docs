Z.AI — это платформа API для моделей **GLM**. Она предоставляет REST API для GLM и
использует ключи API для аутентификации. Создайте ключ API в консоли Z.AI.
OpenClaw использует провайдер `zai` с ключом API Z.AI.

| Свойство    | Значение                                            |
| ----------- | --------------------------------------------------- |
| Провайдер   | `zai`                                               |
| Пакет       | `@openclaw/zai-provider`                            |
| Аутентификация | `ZAI_API_KEY` (устаревший псевдоним: `Z_AI_API_KEY`) |
| API         | Z.AI Chat Completions (аутентификация Bearer)       |

## Модели GLM

GLM — это семейство моделей, а не отдельный провайдер. В OpenClaw модели GLM используют
ссылки вида `zai/glm-5.2`: провайдер `zai`, идентификатор модели `glm-5.2`.

## Начало работы

Сначала установите Plugin провайдера:

```bash
openclaw plugins install @openclaw/zai-provider
```

### Конечные точки

| Вариант первоначальной настройки | Базовый URL                                   | Модель по умолчанию |
| -------------------------------- | --------------------------------------------- | ------------------- |
| `zai-global`                     | `https://api.z.ai/api/paas/v4`                | `glm-5.1`           |
| `zai-cn`                         | `https://open.bigmodel.cn/api/paas/v4`        | `glm-5.1`           |
| `zai-coding-global`              | `https://api.z.ai/api/coding/paas/v4`         | `glm-5.2`           |
| `zai-coding-cn`                  | `https://open.bigmodel.cn/api/coding/paas/v4` | `glm-5.2`           |

`zai-api-key` автоматически определяет одну из этих четырёх конечных точек, проверяя ваш ключ
через API завершений чата каждой из них: сначала проверяются общие конечные точки (`zai-global`,
затем `zai-cn`), после них — конечные точки Coding Plan (`zai-coding-global`, затем
`zai-coding-cn`). Проверка прекращается на первой конечной точке, которая принимает запрос.
Используйте явный параметр `--auth-choice`, чтобы принудительно выбрать конечную точку Coding Plan, если ваш ключ
работает с обоими типами.

## Пример конфигурации

<Tip>
`zai-api-key` позволяет OpenClaw определить по ключу соответствующую конечную точку Z.AI и
автоматически применить правильный базовый URL. Используйте явные региональные варианты, если
хотите принудительно выбрать конкретный Coding Plan или общий API.
</Tip>

```json5
{
  env: { ZAI_API_KEY: "sk-..." },
  models: {
    providers: {
      zai: {
        // GLM-5.2 использует конечную точку Coding Plan.
        baseUrl: "https://api.z.ai/api/coding/paas/v4",
      },
    },
  },
  agents: { defaults: { model: { primary: "zai/glm-5.2" } } },
}
```

списка только для чтения известные строки GLM могут отображаться без загрузки среды выполнения провайдера:

```bash
openclaw models list --all --provider zai
```

<Tip>
Модели GLM доступны в формате `zai/<model>` (пример: `zai/glm-5`).
</Tip>

<Note>
При настройке Coding Plan по умолчанию используется `zai/glm-5.2`, а при настройке общего API сохраняется
`zai/glm-5.1`. Для конечных точек Coding Plan автоматическое определение переключается на
`glm-5.1`, а затем на `glm-4.7`, если ключ или тарифный план не предоставляет доступ к GLM-5.2. Версии
и доступность GLM могут меняться; выполните `openclaw models list --all --provider zai`,
чтобы просмотреть каталог, известный установленной версии.
</Note>

## Уровни рассуждения

<Tabs>
  <Tab title="GLM-5.2">
    Полный диапазон: `off`, `low`, `high`, `max` (по умолчанию `off`). OpenClaw сопоставляет
    `low` и `high` с уровнем рассуждения `high` в Z.AI, а `max` — с уровнем
    `max` в Z.AI через `reasoning_effort` в полезной нагрузке запроса.
  </Tab>
  <Tab title="Другие модели GLM">
    Только двоичный переключатель: `off` и `low` (в средствах выбора отображается как `on`), по умолчанию
    `off`. При установке уровня рассуждения `off` отправляется `thinking: { type: "disabled" }`;
    при любом другом уровне полезная нагрузка запроса остаётся без изменений (применяется собственное
    поведение рассуждения Z.AI по умолчанию).
  </Tab>
</Tabs>

Установка уровня рассуждения `off` предотвращает ответы, в которых бюджет вывода расходуется на
`reasoning_content` до появления видимого текста.

## Расширенная конфигурация

<AccordionGroup>
  <Accordion title="Предварительное разрешение неизвестных моделей GLM-5">
    Неизвестные идентификаторы `glm-5*` по-прежнему предварительно разрешаются в пути провайдера путём
    синтеза принадлежащих провайдеру метаданных из шаблона `glm-4.7`, если идентификатор
    соответствует текущему формату семейства GLM-5.
  </Accordion>

  <Accordion title="Потоковая передача вызовов инструментов">
    `tool_stream` по умолчанию включён для потоковой передачи вызовов инструментов Z.AI. Чтобы отключить его:

    ```json5
    {
      agents: {
        defaults: {
          models: {
            "zai/<model>": {
              params: { tool_stream: false },
            },
          },
        },
      },
    }
    ```

  </Accordion>

  <Accordion title="Сохранённые рассуждения">
    Сохранение рассуждений включается явно, поскольку Z.AI требует повторно передавать полное историческое
    содержимое `reasoning_content`, что увеличивает число токенов запроса. Включите его
    для отдельной модели:

    ```json5
    {
      agents: {
        defaults: {
          models: {
            "zai/glm-5.2": {
              params: { preserveThinking: true },
            },
          },
        },
      },
    }
    ```

    Когда эта возможность включена и рассуждение активно, OpenClaw отправляет
    `thinking: { type: "enabled", clear_thinking: false }` и повторно передаёт предыдущее
    содержимое `reasoning_content` для той же совместимой с OpenAI стенограммы. Ключ параметра
    `preserve_thinking` в формате snake_case также работает как псевдоним.

    Опытные пользователи по-прежнему могут переопределить точную полезную нагрузку провайдера с помощью
    `params.extra_body.thinking`.

  </Accordion>

  <Accordion title="Распознавание изображений">
    Plugin Z.AI регистрирует функцию распознавания изображений.

    | Свойство | Значение    |
    | -------- | ----------- |
    | Модель   | `glm-4.6v`  |

    Распознавание изображений автоматически определяется на основе настроенной аутентификации Z.AI —
    дополнительная конфигурация не требуется.

  </Accordion>

  <Accordion title="Сведения об аутентификации">
    - Z.AI использует аутентификацию Bearer с вашим ключом API.
    - Вариант первоначальной настройки `zai-api-key` автоматически определяет соответствующую конечную точку Z.AI, проверяя поддерживаемые конечные точки с помощью вашего ключа.
    - Используйте явные региональные варианты (`zai-coding-global`, `zai-coding-cn`, `zai-global`, `zai-cn`), если хотите принудительно выбрать конкретный API.
    - Устаревшая переменная среды `Z_AI_API_KEY` по-прежнему поддерживается; OpenClaw копирует её в `ZAI_API_KEY` при запуске, если `ZAI_API_KEY` не задана.

  </Accordion>
</AccordionGroup>
