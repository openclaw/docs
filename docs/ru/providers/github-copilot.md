GitHub Copilot — ИИ-помощник GitHub для написания кода. Он предоставляет доступ к моделям Copilot
для вашей учетной записи и тарифного плана GitHub. OpenClaw может использовать Copilot как
поставщика моделей или среду выполнения агентов тремя различными способами.

## Три способа использования Copilot в OpenClaw

<Tabs>
  <Tab title="Встроенный поставщик (github-copilot)">
    Используйте встроенный процесс входа с устройства, чтобы получить токен GitHub, который затем
    будет обмениваться на токены API Copilot во время работы OpenClaw. Это **стандартный** и самый простой способ,
    поскольку для него не требуется VS Code.

    <Steps>
      <Step title="Выполните команду входа">
        ```bash
        openclaw models auth login-github-copilot
        ```

        Вам будет предложено перейти по URL-адресу и ввести одноразовый код. Не закрывайте
        терминал до завершения процесса.
      </Step>
      <Step title="Задайте модель по умолчанию">
        ```bash
        openclaw models set github-copilot/claude-opus-4.7
        ```

        Или в конфигурации:

        ```json5
        {
          agents: {
            defaults: { model: { primary: "github-copilot/claude-opus-4.7" } },
          },
        }
        ```
      </Step>
    </Steps>

  </Tab>

  <Tab title="Плагин среды Copilot SDK (copilot)">
    Установите внешний плагин `@openclaw/copilot`, если хотите, чтобы
    Copilot CLI и SDK от GitHub управляли низкоуровневым циклом агента для выбранных
    моделей `github-copilot/*`.

    ```bash
    openclaw plugins install @openclaw/copilot
    ```

    Затем подключите модель или поставщика к среде выполнения:

    ```json5
    {
      agents: {
        defaults: {
          model: "github-copilot/gpt-5.5",
          models: {
            "github-copilot/gpt-5.5": {
              agentRuntime: { id: "copilot" },
            },
          },
        },
      },
    }
    ```

    Выберите этот вариант, если для таких обращений к агенту вам нужны нативные сеансы Copilot CLI,
    состояние потоков под управлением SDK и управляемая Copilot компактизация. Без
    явного согласия через `agentRuntime` модели `github-copilot/*` продолжат использовать
    встроенного поставщика. Полный контракт среды выполнения см. в разделе [Среда Copilot SDK](/ru/plugins/copilot).

  </Tab>

  <Tab title="Плагин Copilot Proxy (copilot-proxy)">
    Используйте расширение **Copilot Proxy** для VS Code как локальный мост. OpenClaw обращается
    к конечной точке `/v1` прокси-сервера (по умолчанию `http://localhost:3000/v1`) и использует
    настроенный вами список моделей.

    Плагин `copilot-proxy` поставляется с OpenClaw и включен по умолчанию.
    Настройте базовый URL-адрес и идентификаторы моделей с помощью команды:

    ```bash
    openclaw models auth login --provider copilot-proxy --set-default
    ```

    <Note>
    Выберите этот вариант, если вы уже используете Copilot Proxy в VS Code или вам требуется
    маршрутизация через него. Расширение VS Code должно оставаться запущенным.
    </Note>

  </Tab>
</Tabs>

## GitHub Enterprise (резидентность данных)

Если ваша организация использует клиент GitHub Enterprise с резидентностью данных
(узел `*.ghe.com`, например `your-org.ghe.com`), Copilot работает через локальные
конечные точки клиента, а не через общедоступный `github.com`. OpenClaw предоставляет для этого
отдельный вариант аутентификации, поэтому редактировать URL-адреса вручную не требуется.

<Steps>
  <Step title="Выберите вариант аутентификации Enterprise">
    Во время первоначальной настройки или в `openclaw models auth` выберите
    **GitHub Copilot (Enterprise / data residency)**. Вам будет предложено указать
    домен Enterprise (например, `your-org.ghe.com`), после чего вход
    с устройства будет выполнен через этот клиент.

    Указывайте только корневой адрес клиента (`your-org.ghe.com`). Производные узлы служб, такие
    как `api.your-org.ghe.com` или `copilot-api.your-org.ghe.com`, не принимаются:
    OpenClaw автоматически формирует эти конечные точки из корневого адреса клиента.

    ```bash
    openclaw models auth login --provider github-copilot --method device-enterprise
    ```

  </Step>
  <Step title="Домен сохраняется в конфигурации">
    Выбранный узел сохраняется в параметрах поставщика, чтобы последующие обновления токена
    и запросы на дополнение автоматически направлялись клиенту:

    ```json5
    {
      models: {
        providers: {
          "github-copilot": { params: { githubDomain: "your-org.ghe.com" } },
        },
      },
    }
    ```

  </Step>
</Steps>

Процесс входа с устройства, обмен токена и запросы на дополнение направляются соответственно на
`https://your-org.ghe.com/login/device/code`,
`https://api.your-org.ghe.com/copilot_internal/v2/token` и
`https://copilot-api.your-org.ghe.com`. Токены с резидентностью данных содержат
метку клиента и не содержат подсказки прокси, поэтому базовый URL-адрес дополнений переключается
на узел Copilot клиента вместо общедоступной конечной точки.

<Note>
При смене домена всегда повторно запускается вход с устройства. Если у вас уже сохранен
токен Copilot и вы выбираете другой домен (общедоступный `github.com` ↔ клиент `*.ghe.com`,
один клиент вместо другого), OpenClaw не будет повторно использовать существующий токен —
будет принудительно выполнен новый вход, чтобы область действия токена соответствовала домену,
записываемому в конфигурацию. При повторном входе для *того же* домена по-прежнему предлагается
использовать текущий токен. При возврате к общедоступному `github.com` сохраненный
`githubDomain` удаляется, и конфигурация возвращается к значениям по умолчанию.
</Note>

<Note>
Переменная окружения `COPILOT_GITHUB_DOMAIN` переопределяет определенный домен
для всех путей Copilot, которые его определяют: вход с устройства Enterprise
(`--method device-enterprise`), отдельную
сокращенную команду `openclaw models auth login-github-copilot`, обновление токена, векторные представления
и дополнения. Укажите в ней свой узел `*.ghe.com` для полностью автономной настройки или CI.
Не задавайте ее (и не добавляйте параметр в конфигурацию), чтобы использовать общедоступный `github.com`.
При входе сохраняется домен, для которого был выпущен токен (а при входе через
общедоступный `github.com` он удаляется), поэтому маршрутизация остается корректной
даже после удаления переменной окружения.
</Note>

## Необязательные флаги

| Команда                                                                | Флаг            | Описание                                             |
| ---------------------------------------------------------------------- | --------------- | ---------------------------------------------------- |
| `openclaw models auth login-github-copilot`                            | `--yes`         | Перезаписать существующий профиль аутентификации без запроса |
| `openclaw models auth login --provider github-copilot --method device` | `--set-default` | Также применить рекомендованную поставщиком модель по умолчанию |

```bash
# Пропустить подтверждение повторного входа
openclaw models auth login-github-copilot --yes

# Выполнить вход и задать модель по умолчанию за один шаг
openclaw models auth login --provider github-copilot --method device --set-default
```

## Неинтерактивная первоначальная настройка

Для процесса входа с устройства требуется интерактивный TTY. Для настройки без графического интерфейса импортируйте
существующий токен доступа GitHub OAuth с помощью `openclaw onboard --non-interactive`:

```bash
openclaw onboard --non-interactive --accept-risk \
  --auth-choice github-copilot \
  --github-copilot-token "$COPILOT_GITHUB_TOKEN" \
  --skip-channels --skip-health
```

Можно также опустить `--auth-choice`: передача `--github-copilot-token` подразумевает
выбор аутентификации поставщика GitHub Copilot. Если флаг не указан, первоначальная настройка
последовательно использует `COPILOT_GITHUB_TOKEN`, `GH_TOKEN`, затем `GITHUB_TOKEN`. Используйте
`--secret-input-mode ref` с заданной `COPILOT_GITHUB_TOKEN`, чтобы сохранить
`tokenRef`, ссылающийся на переменную окружения, вместо открытого текста в `auth-profiles.json`.

<AccordionGroup>
  <Accordion title="Требуется интерактивный TTY">
    Для процесса входа с устройства требуется интерактивный TTY. Запускайте его непосредственно
    в терминале, а не в неинтерактивном скрипте или конвейере CI.
  </Accordion>

  <Accordion title="Доступность моделей зависит от вашего тарифного плана">
    Доступность моделей Copilot зависит от вашего тарифного плана GitHub. Если модель
    отклоняется, попробуйте другой идентификатор (например, `github-copilot/gpt-5.5`). Актуальный список
    моделей см. в документации GitHub по [поддерживаемым моделям для каждого тарифного плана Copilot](https://docs.github.com/en/copilot/reference/ai-models/supported-models#supported-ai-models-per-copilot-plan).
  </Accordion>

  <Accordion title="Динамическое обновление каталога из API Copilot">
    После получения токена GitHub через вход с устройства (или переменную окружения)
    OpenClaw по запросу обновляет каталог моделей из `${baseUrl}/models`
    (той же конечной точки, которую использует Copilot в VS Code), чтобы среда выполнения учитывала
    права конкретной учетной записи и точные размеры контекстного окна без
    изменений манифеста. Недавно опубликованные модели Copilot становятся доступны без обновления
    OpenClaw, а размеры контекстного окна отражают реальные ограничения каждой модели
    (например, 400k для серии gpt-5.x и 1M для внутренних
    вариантов `claude-opus-*-1m`).

    Встроенный статический каталог остается видимым резервным вариантом, если обнаружение
    отключено, у пользователя нет профиля аутентификации GitHub, произошел сбой обмена
    токена или HTTPS-вызов `/models` завершился ошибкой. Чтобы отказаться от этого механизма и полностью
    полагаться на статический каталог манифеста (в автономных или изолированных от сети средах):

    ```json5
    {
      plugins: {
        entries: {
          "github-copilot": {
            config: { discovery: { enabled: false } },
          },
        },
      },
    }
    ```

  </Accordion>

  <Accordion title="Выбор транспорта">
    Для идентификаторов моделей Claude автоматически используется транспорт Anthropic Messages.
    Модели Gemini используют транспорт OpenAI Chat Completions, а модели GPT и серии o
    продолжают использовать транспорт OpenAI Responses. OpenClaw выбирает правильный
    транспорт на основе ссылки на модель.
  </Accordion>

  <Accordion title="Совместимость запросов">
    OpenClaw отправляет заголовки запросов в стиле Copilot IDE через транспорты Copilot
    (версии редактора/плагина VS Code и идентификатор интеграции `vscode-chat`),
    помечает последующие обращения с результатами инструментов как инициированные агентом
    и устанавливает заголовок поддержки изображений Copilot, если обращение содержит изображение.
  </Accordion>

  <Accordion title="Порядок разрешения переменных окружения">
    OpenClaw определяет данные аутентификации Copilot из переменных окружения в следующем
    порядке приоритета:

    | Приоритет | Переменная            | Примечания                                  |
    | --------- | --------------------- | ------------------------------------------- |
    | 1         | `COPILOT_GITHUB_TOKEN` | Наивысший приоритет, специально для Copilot |
    | 2         | `GH_TOKEN`            | Токен GitHub CLI (резервный вариант)        |
    | 3         | `GITHUB_TOKEN`        | Стандартный токен GitHub (низший приоритет) |

    Если задано несколько переменных, OpenClaw использует переменную с наивысшим приоритетом.
    Процесс входа с устройства (`openclaw models auth login-github-copilot`) сохраняет
    свой токен в хранилище профилей аутентификации и имеет приоритет над всеми переменными
    окружения.

  </Accordion>

  <Accordion title="Хранение токенов">
    При входе токен GitHub сохраняется в хранилище профилей аутентификации (идентификатор профиля
    `github-copilot:github`) и обменивается на краткосрочный токен API Copilot
    во время работы OpenClaw. Управлять токеном вручную не требуется.
  </Accordion>
</AccordionGroup>

## Векторные представления для поиска по памяти

GitHub Copilot также может выступать поставщиком векторных представлений для
[поиска по памяти](/ru/concepts/memory-search). Если у вас есть подписка Copilot и
вы выполнили вход, OpenClaw может использовать ее для векторных представлений без отдельного ключа API.

### Конфигурация

Явно задайте `memorySearch.provider`, чтобы использовать векторные представления GitHub Copilot. Если
доступен токен GitHub, OpenClaw обнаружит доступные модели векторных представлений через
API Copilot и автоматически выберет лучшую.

```json5
{
  agents: {
    defaults: {
      memorySearch: {
        provider: "github-copilot",
        // Необязательно: переопределить автоматически обнаруженную модель
        model: "text-embedding-3-small",
      },
    },
  },
}
```

### Как это работает

1. OpenClaw определяет ваш токен GitHub (из переменных окружения или профиля аутентификации).
2. Обменивает его на краткосрочный токен API Copilot.
3. Обращается к конечной точке Copilot `/models`, чтобы обнаружить доступные модели векторных представлений.
4. Выбирает лучшую модель (порядок предпочтения: `text-embedding-3-small`,
   `text-embedding-3-large`, `text-embedding-ada-002`).
5. Отправляет запросы векторных представлений в конечную точку Copilot `/embeddings`.

Доступность моделей зависит от вашего тарифного плана GitHub. Если модели векторных представлений
недоступны, OpenClaw пропускает Copilot и пробует следующего поставщика.
