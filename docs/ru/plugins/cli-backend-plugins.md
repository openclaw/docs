Плагины бэкендов CLI позволяют OpenClaw вызывать локальный CLI ИИ в качестве
бэкенда текстового инференса. Бэкенд отображается как префикс поставщика в ссылках на модели:

```text
acme-cli/acme-large
```

Используйте бэкенд CLI, когда вышестоящая интеграция уже доступна в виде локальной
команды, когда CLI управляет локальным состоянием входа или как резервный вариант,
если поставщики API недоступны.

<Info>
  Если вышестоящий сервис предоставляет обычный HTTP API моделей, вместо этого
  создайте [плагин поставщика](/ru/plugins/sdk-provider-plugins). Если вышестоящая
  среда выполнения управляет полными сеансами агента, событиями инструментов,
  Compaction или состоянием фоновых задач, используйте
  [среду выполнения агента](/ru/plugins/sdk-agent-harness).
</Info>

## За что отвечает плагин

У плагина бэкенда CLI есть три контракта:

| Контракт                 | Файл                   | Назначение                                                        |
| ------------------------ | ---------------------- | ----------------------------------------------------------------- |
| Точка входа пакета       | `package.json`         | Указывает OpenClaw модуль среды выполнения плагина                |
| Владение в манифесте     | `openclaw.plugin.json` | Объявляет идентификатор бэкенда до загрузки среды выполнения      |
| Регистрация при запуске  | `index.ts`             | Вызывает `api.registerCliBackend(...)` с настройками команды по умолчанию |

Манифест содержит метаданные обнаружения: он не запускает CLI и не регистрирует
поведение среды выполнения. Поведение среды выполнения начинает действовать, когда
точка входа плагина вызывает `api.registerCliBackend(...)`.

## Минимальный плагин бэкенда

<Steps>
  <Step title="Создайте метаданные пакета">
    ```json package.json
    {
      "name": "@acme/openclaw-acme-cli",
      "version": "1.0.0",
      "type": "module",
      "openclaw": {
        "extensions": ["./index.ts"],
        "compat": {
          "pluginApi": ">=2026.3.24-beta.2",
          "minGatewayVersion": "2026.3.24-beta.2"
        },
        "build": {
          "openclawVersion": "2026.3.24-beta.2",
          "pluginSdkVersion": "2026.3.24-beta.2"
        }
      },
      "dependencies": {
        "openclaw": "^2026.3.24"
      },
      "devDependencies": {
        "typescript": "^5.9.0"
      }
    }
    ```

    Опубликованные пакеты должны содержать собранные JavaScript-файлы среды
    выполнения. Если исходной точкой входа служит `./src/index.ts`, добавьте
    `openclaw.runtimeExtensions`, указывающий на соответствующий собранный
    JavaScript-файл. См. [Точки входа](/ru/plugins/sdk-entrypoints).

  </Step>

  <Step title="Объявите владение бэкендом">
    ```json openclaw.plugin.json
    {
      "id": "acme-cli",
      "name": "Acme CLI",
      "description": "Run Acme's local AI CLI through OpenClaw",
      "cliBackends": ["acme-cli"],
      "setup": {
        "cliBackends": ["acme-cli"],
        "requiresRuntime": false
      },
      "activation": {
        "onStartup": false
      },
      "configSchema": {
        "type": "object",
        "additionalProperties": false
      }
    }
    ```

    `cliBackends` — это список владения средой выполнения; он позволяет OpenClaw
    автоматически загружать плагин, когда конфигурация или выбор модели упоминает
    `acme-cli/...`.

    `setup.cliBackends` — это поверхность настройки на основе дескрипторов. Добавьте
    её, если обнаружение моделей, первоначальная настройка или состояние должны
    распознавать бэкенд без загрузки среды выполнения плагина. Используйте
    `requiresRuntime: false`, только если этих статических дескрипторов достаточно
    для настройки.

  </Step>

  <Step title="Зарегистрируйте бэкенд">
    ```typescript index.ts
    import { definePluginEntry } from "openclaw/plugin-sdk/plugin-entry";
    import {
      CLI_FRESH_WATCHDOG_DEFAULTS,
      CLI_RESUME_WATCHDOG_DEFAULTS,
      type CliBackendPlugin,
    } from "openclaw/plugin-sdk/cli-backend";

    function buildAcmeCliBackend(): CliBackendPlugin {
      return {
        id: "acme-cli",
        liveTest: {
          defaultModelRef: "acme-cli/acme-large",
          defaultImageProbe: false,
          defaultMcpProbe: false,
          docker: {
            npmPackage: "@acme/acme-cli",
            binaryName: "acme",
          },
        },
        config: {
          command: "acme",
          args: ["chat", "--json"],
          output: "json",
          input: "stdin",
          modelArg: "--model",
          sessionArg: "--session",
          sessionMode: "existing",
          sessionIdFields: ["session_id", "conversation_id"],
          systemPromptFileArg: "--system-file",
          systemPromptWhen: "first",
          imageArg: "--image",
          imageMode: "repeat",
          reliability: {
            watchdog: {
              fresh: { ...CLI_FRESH_WATCHDOG_DEFAULTS },
              resume: { ...CLI_RESUME_WATCHDOG_DEFAULTS },
            },
          },
          serialize: true,
        },
      };
    }

    export default definePluginEntry({
      id: "acme-cli",
      name: "Acme CLI",
      description: "Run Acme's local AI CLI through OpenClaw",
      register(api) {
        api.registerCliBackend(buildAcmeCliBackend());
      },
    });
    ```

    Идентификатор бэкенда должен соответствовать записи `cliBackends` в манифесте.
    Зарегистрированная `config` задаёт только значения по умолчанию; пользовательская
    конфигурация из `agents.defaults.cliBackends.acme-cli` объединяется с ней во
    время выполнения и имеет приоритет.

  </Step>
</Steps>

## Структура конфигурации

`CliBackendConfig` описывает, как OpenClaw должен запускать CLI и разбирать его вывод:

| Поле                                                      | Назначение                                                                          |
| --------------------------------------------------------- | ----------------------------------------------------------------------------------- |
| `command`                                                 | Имя исполняемого файла или абсолютный путь к команде                                |
| `args`                                                    | Базовый список argv для новых запусков                                              |
| `resumeArgs`                                              | Альтернативный список argv для возобновлённых сеансов; поддерживает `{sessionId}`    |
| `output` / `resumeOutput`                                 | Парсер: `json`, `jsonl` или `text`                                                  |
| `jsonlDialect`                                            | Диалект событий JSONL: `claude-stream-json` или `gemini-stream-json`                |
| `liveSession`                                             | Режим долгоживущего процесса CLI (`claude-stdio`)                                   |
| `input`                                                   | Передача запроса: `arg` или `stdin`                                                 |
| `maxPromptArgChars`                                       | Максимальная длина запроса в режиме `arg` до перехода на stdin                      |
| `env` / `clearEnv`                                        | Дополнительные переменные окружения для внедрения или имена переменных для удаления перед запуском |
| `modelArg`                                                | Флаг, используемый перед идентификатором модели                                     |
| `modelAliases`                                            | Сопоставление идентификаторов моделей OpenClaw с нативными идентификаторами CLI     |
| `sessionArg` / `sessionArgs`                              | Способ передачи идентификатора сеанса                                               |
| `sessionMode`                                             | `always`, `existing` или `none`                                                     |
| `sessionIdFields`                                         | Поля JSON, из которых OpenClaw читает данные в выводе CLI                           |
| `systemPromptArg` / `systemPromptFileArg`                 | Передача системного запроса                                                         |
| `systemPromptFileConfigArg` / `systemPromptFileConfigKey` | Передача переопределения конфигурации для файла системного запроса, например `-c`   |
| `systemPromptMode`                                        | `append` или `replace`                                                              |
| `systemPromptWhen`                                        | `first`, `always` или `never`                                                       |
| `imageArg` / `imageMode`                                  | Флаг пути к изображению и способ передачи нескольких изображений (`repeat` или `list`) |
| `imagePathScope`                                          | Расположение подготовленных файлов изображений перед передачей: `temp` или `workspace` |
| `serialize`                                               | Сохранять порядок запусков одного бэкенда                                           |
| `reseedFromRawTranscriptWhenUncompacted`                  | Включить ограниченное повторное заполнение из необработанной расшифровки до Compaction для безопасного сброса сеансов |
| `reliability.outputLimits`                                | Максимальное количество символов или строк необработанного JSONL, сохраняемых для одного обращения к активному CLI (для бэкендов с активным сеансом) |
| `reliability.watchdog`                                    | Настройка тайм-аута отсутствия вывода отдельно для новых и возобновлённых запусков  |

Предпочитайте минимальную статическую конфигурацию, соответствующую CLI. Добавляйте
обратные вызовы плагина только для поведения, которое действительно относится к
бэкенду.

## Расширенные хуки бэкенда

`CliBackendPlugin` также может определять:

| Хук                                | Назначение                                                                    |
| ---------------------------------- | ----------------------------------------------------------------------------- |
| `normalizeConfig(config, context)` | Преобразовать устаревшую пользовательскую конфигурацию после объединения      |
| `resolveExecutionArgs(ctx)`        | Добавить флаги уровня запроса, например интенсивность рассуждений или изоляцию побочных вопросов |
| `prepareExecution(ctx)`            | Создать временные мосты аутентификации или конфигурации перед запуском         |
| `transformSystemPrompt(ctx)`       | Применить итоговое преобразование системного запроса для конкретного CLI      |
| `textTransforms`                   | Двунаправленные замены в запросах и выводе                                    |
| `defaultAuthProfileId`             | Предпочитать определённый профиль аутентификации OpenClaw                     |
| `authEpochMode`                    | Определить, как изменения аутентификации делают сохранённые сеансы CLI недействительными |
| `nativeToolMode`                   | Объявить, отсутствуют ли нативные инструменты, всегда ли они включены или могут выбираться хостом |
| `sideQuestionToolMode`             | Объявить отключённые нативные инструменты для побочных вопросов `/btw`        |
| `bundleMcp` / `bundleMcpMode`      | Включить мост инструментов MCP OpenClaw через local loopback                  |
| `ownsNativeCompaction`             | Бэкенд управляет собственной Compaction — OpenClaw передаёт ему управление    |
| `runtimeArtifact`                  | Привязать средство запуска скрипта к полному дереву его комплектного пакета   |

Оставляйте эти хуки в ведении поставщика. Не добавляйте в ядро ветви для конкретного
CLI, если поведение можно выразить хуком бэкенда.

`runtimeArtifact` принадлежит плагину и не может быть переопределён пользователем.
Он используется только тогда, когда активное обращение к инференсу создаёт или
повторно проверяет подтверждённое полномочие настройки; обычным запускам CLI он не
требуется. Бэкенд без этого объявления не может создавать подтверждённое полномочие
настройки CLI. Объявление `bundled-package-tree` указывает точного владельца
`package.json` и требует, чтобы точкой входа пакета была команда. OpenClaw вычисляет
хеш ограниченного полного дерева установленного пакета, включая вложенные
зависимости, и блокирует выполнение при перенаправляющих символических ссылках,
средствах запуска за пределами объявленного пакета, объявлениях обязательных
внешних зависимостей, слишком больших деревьях и неизвестных скриптах. Объявляйте
это только тогда, когда дерево содержит полную реализацию инференса; необязательные
интеграции инструментов не делают внешний граф реализации безопасным.

Если тот же бэкенд также поставляет автономный нативный исполняемый файл, перечислите
его канонические базовые имена в `nativeExecutableNames`. Другие нативные команды
остаются неподтверждёнными, даже если пользователь переопределяет команду бэкенда.

`ctx.executionMode` имеет значение `"agent"` для обычных ходов и `"side-question"` для
эфемерных вызовов `/btw`. Используйте его, когда CLI требуются другие одноразовые флаги,
например для отключения нативных инструментов, сохранения сеанса или возобновления работы
для BTW. Если у бэкенда обычно задано `nativeToolMode: "always-on"`, но его аргументы
командной строки для побочного вопроса надёжно отключают эти инструменты, также задайте
`sideQuestionToolMode: "disabled"`; иначе OpenClaw завершает работу с отказом, когда для BTW
требуется запуск CLI без инструментов.

Задавайте `nativeToolMode: "selectable"` только тогда, когда `resolveExecutionArgs` может
отключить все нативные инструменты бэкенда для отдельного запуска. Для таких ограниченных
запусков `ctx.toolAvailability.native` представляет собой пустой кортеж, а
`ctx.toolAvailability.mcp` — точный изолированный хостом список разрешённых MCP. Хук
должен заменить конфликтующие флаги инструментов и вернуть аргументы командной строки,
обеспечивающие соблюдение обоих значений; OpenClaw вызывает его один раз с окончательными
аргументами нового запуска или возобновления и завершает работу с отказом, если бэкенд
не может обеспечить ограничение. Имена MCP в этом контексте можно безопасно одобрять
автоматически только потому, что хост уже ограничил создаваемую конфигурацию MCP этими
серверами и инструментами.

### `ownsNativeCompaction`: отказ от Compaction в OpenClaw

Если ваш бэкенд запускает агента, который выполняет Compaction **собственной** расшифровки,
задайте `ownsNativeCompaction: true`, чтобы защитный суммаризатор OpenClaw никогда не
запускался для его сеансов: жизненный цикл Compaction в CLI не выполняет никаких действий,
и ход продолжается. `claude-cli` объявляет этот параметр, поскольку Claude Code выполняет
Compaction внутренними средствами без конечной точки среды выполнения. Сеансы нативной
среды выполнения, такие как Codex, вместо этого продолжают направляться в конечную точку
Compaction своей среды выполнения.

**Объявляйте его, только если выполняются все следующие условия**, иначе отложенный
сеанс с превышенным бюджетом может остаться с превышенным бюджетом или устареть
(OpenClaw больше не сможет его восстановить):

- бэкенд надёжно выполняет Compaction собственной расшифровки или ограничивает её
  по мере приближения к пределу окна;
- он сохраняет возобновляемый сеанс, чтобы состояние после Compaction сохранялось
  между ходами (например, `--resume` / `--session-id`);
- это не сеанс Compaction нативной среды выполнения: сеансы с соответствующим
  `agentHarnessId` вместо этого направляются в конечную точку среды выполнения.

## Мост инструментов MCP

Бэкенды CLI по умолчанию не получают инструменты OpenClaw. Если CLI может использовать
конфигурацию MCP, явно включите эту возможность:

```typescript
return {
  id: "acme-cli",
  bundleMcp: true,
  bundleMcpMode: "codex-config-overrides",
  config: {
    command: "acme",
    args: ["chat", "--json"],
    output: "json",
  },
};
```

Поддерживаемые режимы моста:

| Режим                    | Применение                                                        |
| ------------------------ | ----------------------------------------------------------------- |
| `claude-config-file`     | CLI, принимающие файл конфигурации MCP                            |
| `codex-config-overrides` | CLI, принимающие переопределения конфигурации в аргументах запуска |
| `gemini-system-settings` | CLI, читающие настройки MCP из каталога системных настроек        |

Включайте мост, только если CLI действительно может его использовать. Если CLI имеет
собственный встроенный слой инструментов, который невозможно отключить, задайте
`nativeToolMode: "always-on"`, чтобы OpenClaw мог завершить работу с отказом, когда
вызывающей стороне требуется отсутствие нативных инструментов. Если CLI может отключить
все нативные инструменты для каждого отдельного запуска, используйте `"selectable"` с
описанным выше контрактом `resolveExecutionArgs`.

## Пользовательская конфигурация

Пользователи могут переопределить любое значение бэкенда по умолчанию:

```json5
{
  agents: {
    defaults: {
      cliBackends: {
        "acme-cli": {
          command: "/opt/acme/bin/acme",
          args: ["chat", "--json", "--profile", "work"],
          modelAliases: {
            large: "acme-large-2026",
          },
        },
      },
      model: {
        primary: "openai/gpt-5.6-sol",
        fallbacks: ["acme-cli/large"],
      },
    },
  },
}
```

Документируйте минимальное переопределение, которое, скорее всего, потребуется
пользователям, — обычно это только `command`, если исполняемый файл находится вне `PATH`.

## Проверка

Для встроенных плагинов добавьте целевой тест построителя и регистрации настройки,
затем запустите целевой набор тестов плагина:

```bash
pnpm test extensions/acme-cli
```

Для локальных или установленных плагинов проверьте обнаружение и один реальный запуск модели:

```bash
openclaw plugins inspect acme-cli --runtime --json
openclaw agent --message "reply exactly: backend ok" --model acme-cli/acme-large
```

Если бэкенд поддерживает изображения или MCP, добавьте проверочный запуск с реальным CLI,
подтверждающий работу этих путей. Не полагайтесь на статическую проверку поведения
системной подсказки, изображений, MCP или возобновления сеанса.
