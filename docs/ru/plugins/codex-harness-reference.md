Этот справочник содержит подробное описание конфигурации официального плагина `codex`.
Сведения о настройке и выборе маршрутизации см. прежде всего в разделе
[Среда выполнения Codex](/ru/plugins/codex-harness).

## Параметры конфигурации плагина

Все настройки среды выполнения Codex находятся в `plugins.entries.codex.config`.

```json5
{
  plugins: {
    entries: {
      codex: {
        enabled: true,
        config: {
          discovery: {
            enabled: true,
            timeoutMs: 2500,
          },
          appServer: {
            mode: "guardian",
          },
        },
      },
    },
  },
}
```

Поля верхнего уровня:

| Поле                       | Значение по умолчанию                   | Значение                                                                                                                                                                                             |
| -------------------------- | --------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `discovery`                | включено                                | Настройки обнаружения моделей для `model/list` сервера приложений Codex.                                                                                                                             |
| `appServer`                | управляемый stdio-сервер приложений     | Настройки транспорта, команды, аутентификации, подтверждений, песочницы и тайм-аутов. Обычная среда выполнения по умолчанию использует состояние, относящееся к агенту.                                |
| `codexDynamicToolsLoading` | `"searchable"`                          | Используйте `"direct"`, чтобы добавить динамические инструменты OpenClaw непосредственно в начальный контекст инструментов Codex.                                                                     |
| `codexDynamicToolsExclude` | `[]`                                    | Дополнительные имена динамических инструментов OpenClaw, которые следует исключить из ходов сервера приложений Codex.                                                                                 |
| `codexPlugins`             | отключено                               | Поддержка собственных плагинов и приложений Codex, включая доступ по явному согласию к приложениям подключённых учётных записей. См. [Собственные плагины Codex](/ru/plugins/codex-native-plugins).        |
| `computerUse`              | отключено                               | Настройка управления компьютером в Codex. См. [Управление компьютером в Codex](/ru/plugins/codex-computer-use).                                                                                          |
| `supervision`              | отключено                               | Каталог неархивированных собственных сеансов, продолжение локальных ветвей и политика инструментов агента. См. [Надзор за Codex](/plugins/codex-supervision).                                         |

## Надзор

Надзор выводит список неархивированных сеансов Codex с компьютера Gateway и
подключённых узлов, для которых эта функция явно включена. Включайте его независимо от среды выполнения агента:

```json5
{
  plugins: {
    entries: {
      codex: {
        enabled: true,
        config: {
          supervision: {
            enabled: true,
          },
        },
      },
    },
  },
}
```

Поля `supervision`:

| Поле                  | Значение по умолчанию          | Значение                                                                                                                                                                                                                                                                                  |
| --------------------- | ------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `enabled`             | `false`                        | Публикует локальный каталог сеансов, а на Gateway также объединяет каталоги подключённых узлов, для которых эта функция явно включена, для страницы сеансов Codex.                                                                                                                         |
| `endpoints`           | встроенная локальная конечная точка | Целевые конечные точки для совместимости и расширенной настройки сохранённого агента надзора Codex и автономных инструментов MCP. Каталог для пользователей и поток ветвления игнорируют эти цели и используют сервер приложений надзора, разрешённый из `appServer`.                          |
| `allowRawTranscripts` | `false`                        | При включённом надзоре разрешает автономному агенту или автономному MCP читать расшифровки и поля списков, сформированные на их основе. Чтение только метаданных `codex_threads` остаётся доступным. Не управляет продолжением работы через аутентифицированный интерфейс управления.             |
| `allowWriteControls`  | `false`                        | При включённом надзоре разрешает автономные операции `codex_threads` по ответвлению, переименованию, архивированию и разархивированию, а также автономные операции MCP по отправке, корректировке и прерыванию. Не обходит другие проверки привязки, узла, состояния или подтверждения.          |

Записи конечных точек принимают следующие поля:

| Поле           | Применимо к   | Значение                                                                    |
| -------------- | ------------- | --------------------------------------------------------------------------- |
| `id`           | всем          | Стабильный идентификатор конечной точки.                                    |
| `label`        | всем          | Необязательная отображаемая метка.                                          |
| `transport`    | всем          | `"stdio-proxy"` или `"websocket"`.                                          |
| `command`      | `stdio-proxy` | Необязательная команда сервера приложений.                                  |
| `args`         | `stdio-proxy` | Необязательные аргументы команды.                                           |
| `cwd`          | `stdio-proxy` | Необязательный рабочий каталог дочернего процесса.                          |
| `url`          | `websocket`   | Обязательный URL WebSocket или поддерживаемого локального сокета.           |
| `authTokenEnv` | `websocket`   | Необязательная переменная окружения, значение которой аутентифицирует конечную точку. |

Страница **сеансов Codex** использует сервер приложений надзора плагина и
показывает только неархивированные сеансы. Без явных настроек подключения
`appServer` это управляемое stdio-подключение с домашним каталогом пользователя. Сохранённые или бездействующие локальные строки могут создавать
привязанный к модели чат с ограниченной историей пользователя и ассистента вплоть до последнего
сохранённого исходного завершающего хода. Его закрытая привязка сохраняет ответвление снимка,
каноническую ветвь из источника `appServer`, внедрение истории и последующие ходы в рамках этого
подключения. При первом каноническом запуске используется пара, возвращённая ответвлением. При последующих
возобновлениях переопределения модели и провайдера OpenClaw не передаются, поэтому Codex восстанавливает
сохранённую пару канонического потока; отдельное собственное изменение может обновить эту
пару, но внешняя модель и цепочка резервных вариантов никогда её не заменяют. Сохранённые и бездействующие
строки можно архивировать после подтверждения отсутствия других исполнителей, если только точной целью или одним из её неархивированных порождённых
потомков не владеет другая активная привязка OpenClaw.
OpenClaw следует правилам постраничного перечисления потомков Codex и безопасно прекращает операцию при
ошибках перечисления, циклах или исчерпании предела безопасности. Подтверждение по-прежнему
учитывает неизвестные собственные клиенты и состояние гонки между проверкой состояния и архивированием. Контролируемый
привязанный к модели чат нельзя удалить, пока он защищает собственную привязку.
Активные источники не могут создавать ветви или архивироваться, но существующий контролируемый
чат по-прежнему можно открыть. Каждая строка подключённого узла остаётся доступной только для чтения; транспорт узла
пока не предоставляет потоковый жизненный цикл, необходимый среде выполнения.

Сам по себе параметр `appServer.homeScope: "user"` изменяет только домашний каталог Codex, используемый управляемым процессом
среды выполнения; он не публикует каталог парка. Включение надзора не
изменяет значение среды выполнения по умолчанию. Вместо этого отдельное подключение надзора
по умолчанию использует управляемый stdio с домашним каталогом пользователя, если отсутствуют явные настройки подключения
`appServer`. Для этого подключения применяются явные настройки.
Ожидающие и подтверждённые контролируемые привязки сохраняют это подключение для каждого хода;
при отключённом надзоре или рассинхронизации подключения либо жизненного цикла операция безопасно завершается ошибкой вместо
перехода к среде выполнения с домашним каталогом агента. Подключение по умолчанию использует общие сохранённые
сеансы с собственными клиентами Codex, но не их локальное для процесса состояние активности.

Устаревшие настройки `plugins.entries.codex-supervisor` выведены из эксплуатации. Выполните
`openclaw doctor --fix`, чтобы перенести старую запись, определения конечных точек, флаги политик
и ссылки списков разрешённых и запрещённых плагинов в этот блок. При конфликтах явные канонические
значения `codex.config.supervision` имеют приоритет.

## Транспорт сервера приложений

Для обычных ходов среды выполнения OpenClaw запускает управляемый бинарный файл Codex, поставляемый
с официальным плагином (в настоящее время `@openai/codex` `0.144.1`):

```bash
codex app-server --listen stdio://
```

Это привязывает версию сервера приложений к официальному плагину `codex`, а не к
отдельному CLI Codex, который может быть установлен локально. Задавайте
`appServer.command` только тогда, когда намеренно хотите использовать другой исполняемый файл.
Обычные управляемые ходы с изолированным домашним каталогом агента по умолчанию предпочитают этот закреплённый
пакет, даже если установлено настольное приложение macOS. Когда включено
[управление компьютером](/ru/plugins/codex-computer-use) или когда `homeScope` имеет значение
`"user"` и может загружать собственное состояние управления компьютером, управляемый запуск вместо этого предпочитает
бинарный файл настольного приложения, которому принадлежат необходимые разрешения macOS. То же
правило приоритета настольного приложения применяется, когда действующая конфигурация Codex изолированного домашнего каталога агента
включает собственное управление компьютером. Если комплект настольного приложения не установлен, OpenClaw
возвращается к бинарному файлу закреплённого пакета.

Передача исполняемого файла и изоляция собственной конфигурации координируют работу клиентов внутри одного
запущенного процесса Gateway. Перезапустите Gateway после того, как другой процесс изменит
конфигурацию собственного плагина Codex.

Для надзора разрешается отдельное подключение. При отсутствии явных настроек подключения
`appServer` используется управляемый stdio с `homeScope: "user"`;
обычная среда выполнения остаётся на управляемом stdio с `homeScope: "agent"`. Явные
настройки подключения применяются обоими путями. Явно задайте `homeScope: "user"`,
если обычная среда выполнения должна совместно использовать `$CODEX_HOME` (или `~/.codex`)
с собственными клиентами. Закрытая контролируемая привязка использует подключение
надзора независимо от значения по умолчанию для обычной среды выполнения. Независимые процессы сервера приложений
сохраняют раздельные актуальные состояния и состояния подтверждений.

Для уже запущенного сервера приложений используйте транспорт WebSocket:

```json5
{
  plugins: {
    entries: {
      codex: {
        enabled: true,
        config: {
          appServer: {
            transport: "websocket",
            url: "ws://gateway-host:39175",
            authToken: "${CODEX_APP_SERVER_TOKEN}",
            requestTimeoutMs: 60000,
          },
        },
      },
    },
  },
}
```

Поля `appServer`:

| Поле                                          | Значение по умолчанию                                  | Назначение                                                                                                                                                                                                                                                                                                                                                                                           |
| --------------------------------------------- | ------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `transport`                                   | `"stdio"`                                              | `"stdio"` запускает Codex; явно заданное значение `"unix"` подключается к локальному управляющему сокету; `"websocket"` подключается к `url`.                                                                                                                                                                                                                                                         |
| `homeScope`                                   | `"agent"`                                              | `"agent"` изолирует обычное состояние окружения для каждого агента OpenClaw. `"user"` — явно включаемый режим, который предоставляет общий доступ к нативному `$CODEX_HOME` или `~/.codex`, использует нативную аутентификацию и включает управление потоками только для владельца. Пользовательская область поддерживает локальный транспорт stdio или Unix. Для отдельного управляющего подключения незаданное значение преобразуется в `"user"` для stdio или Unix и в `"agent"` для WebSocket. |
| `command`                                     | управляемый исполняемый файл Codex                     | Исполняемый файл для транспорта stdio. Не задавайте значение, чтобы использовать управляемый исполняемый файл.                                                                                                                                                                                                                                                                                       |
| `args`                                        | `["app-server", "--listen", "stdio://"]`               | Аргументы для транспорта stdio.                                                                                                                                                                                                                                                                                                                                                                       |
| `url`                                         | не задано                                              | URL WebSocket-сервера приложений или URL `unix://`. Явно заданный пустой путь Unix выбирает канонический управляющий сокет в домашнем каталоге пользователя.                                                                                                                                                                                                                                            |
| `authToken`                                   | не задано                                              | Bearer-токен для транспорта WebSocket. Принимает строковый литерал или SecretInput, например `${CODEX_APP_SERVER_TOKEN}`.                                                                                                                                                                                                                                                                              |
| `headers`                                     | `{}`                                                   | Дополнительные заголовки WebSocket. Значения заголовков принимают строковые литералы или значения SecretInput, например `x-codex-client-session-token: "${CODEX_CLIENT_SESSION_TOKEN}"`.                                                                                                                                                                                                                |
| `clearEnv`                                    | `[]`                                                   | Имена дополнительных переменных окружения, удаляемых из порождённого процесса app-server с транспортом stdio после того, как OpenClaw сформирует наследуемое им окружение.                                                                                                                                                                                                                            |
| `remoteWorkspaceRoot`                         | не задано                                              | Корень удалённого рабочего пространства app-server Codex. Если он задан, OpenClaw определяет корень локального рабочего пространства по разрешённому рабочему пространству OpenClaw, сохраняет текущий суффикс cwd относительно этого удалённого корня и отправляет Codex только итоговый cwd app-server. Если cwd находится вне разрешённого корня рабочего пространства OpenClaw, OpenClaw безопасно завершает операцию с ошибкой вместо отправки локального для Gateway пути удалённому app-server. |
| `requestTimeoutMs`                            | `60000`                                                | Время ожидания вызовов плоскости управления app-server.                                                                                                                                                                                                                                                                                                                                               |
| `turnCompletionIdleTimeoutMs`                 | `60000`                                                | Период бездействия после того, как Codex принимает ход, или после относящегося к ходу запроса app-server, пока OpenClaw ожидает `turn/completed`.                                                                                                                                                                                                                                                       |
| `postToolRawAssistantCompletionIdleTimeoutMs` | `300000`                                               | Ограничитель бездействия до завершения и прогресса, используемый после передачи управления инструменту, завершения нативного инструмента, необработанного прогресса ассистента после инструмента, завершения необработанного рассуждения или прогресса рассуждения, пока OpenClaw ожидает `turn/completed`. Используйте его для доверенных или ресурсоёмких задач, в которых синтез после инструмента может обоснованно оставаться неактивным дольше, чем допускает бюджет финального ответа ассистента. |
| `mode`                                        | `"yolo"`, если локальные требования Codex не запрещают YOLO | Предустановка для выполнения в режиме YOLO или с проверкой контролёром.                                                                                                                                                                                                                                                                                                                               |
| `approvalPolicy`                              | `"never"` или разрешённая политика одобрения контролёра | Нативная политика одобрения Codex, отправляемая при запуске и возобновлении потока, а также при выполнении хода.                                                                                                                                                                                                                                                                                        |
| `sandbox`                                     | `"danger-full-access"` или разрешённая песочница контролёра | Нативный режим песочницы Codex, отправляемый при запуске и возобновлении потока. Активные песочницы OpenClaw ограничивают ходы с `danger-full-access` до `workspace-write` в Codex; сетевой флаг хода соответствует исходящему доступу песочницы OpenClaw.                                                                                                                                                    |
| `approvalsReviewer`                           | `"user"` или разрешённый проверяющий контролёр          | Используйте `"auto_review"`, чтобы Codex мог проверять нативные запросы одобрения, когда это разрешено.                                                                                                                                                                                                                                                                                                |
| `defaultWorkspaceDir`                         | текущий каталог процесса                               | Рабочее пространство, используемое командой `/codex bind`, если параметр `--cwd` не указан.                                                                                                                                                                                                                                                                                                            |
| `serviceTier`                                 | не задано                                              | Необязательный уровень обслуживания app-server Codex. `"priority"` включает маршрутизацию в быстром режиме, `"flex"` запрашивает гибкую обработку, а `null` удаляет переопределение. Устаревшее значение `"fast"` принимается как `"priority"`.                                                                                                                                                          |
| `networkProxy`                                | отключено                                              | Включает сетевой доступ профиля разрешений Codex для команд app-server. OpenClaw определяет выбранную конфигурацию `permissions.<profile>.network` и выбирает её с помощью `default_permissions` вместо отправки `sandbox`.                                                                                                                                                                             |
| `experimental.sandboxExecServer`              | `false`                                                | Предварительно доступная настройка, регистрирующая в поддерживаемом app-server Codex среду Codex на основе песочницы OpenClaw, чтобы нативное выполнение Codex могло происходить внутри активной песочницы OpenClaw.                                                                                                                                                                                     |

`appServer.networkProxy` задаётся явно, поскольку изменяет контракт песочницы
Codex. Когда эта настройка включена, OpenClaw также задаёт
`features.network_proxy.enabled` и `default_permissions` в конфигурации потока
Codex, чтобы сгенерированный профиль разрешений мог запустить управляемый Codex
сетевой доступ. По умолчанию OpenClaw генерирует устойчивое к коллизиям имя
профиля `openclaw-network-<fingerprint>` на основе содержимого профиля;
используйте `profileName`, только если требуется стабильное локальное имя.

```js
export default {
  plugins: {
    entries: {
      codex: {
        config: {
          appServer: {
            sandbox: "workspace-write",
            networkProxy: {
              enabled: true,
              domains: {
                "api.openai.com": "allow",
                "blocked.example.com": "deny",
              },
              allowUpstreamProxy: true,
              proxyUrl: "http://127.0.0.1:3128",
            },
          },
        },
      },
    },
  },
};
```

Если обычная среда выполнения app-server использовала бы `danger-full-access`, то при включении
`networkProxy` для создаваемого профиля разрешений вместо этого применяется доступ
к файловой системе в стиле рабочей области. Управляемое Codex ограничение сети
представляет собой изолированную сетевую среду, поэтому профиль полного доступа не защищал бы
исходящий трафик.

Plugin блокирует устаревшие или неверсионированные рукопожатия app-server: Codex app-server
должен сообщать стабильную версию `0.143.0` или новее.

OpenClaw считает WebSocket URL-адреса app-server, не относящиеся к local loopback,
удалёнными и требует WebSocket-аутентификацию с идентификацией через
`appServer.authToken` или заголовок `Authorization`. `appServer.authToken` и каждое
значение `appServer.headers.*` могут быть SecretInput; среда выполнения секретов разрешает
SecretRef и сокращённые ссылки на переменные окружения до того, как OpenClaw формирует
параметры запуска app-server, а неразрешённые структурированные SecretRef приводят к ошибке
до отправки любого токена или заголовка. Если настроены нативные плагины Codex, OpenClaw
использует плоскость управления плагинами подключённого app-server, чтобы установить или
обновить эти плагины, а затем обновляет перечень приложений, чтобы приложения, принадлежащие
плагинам, были видны потоку Codex. `app/list` по-прежнему является авторитетным источником
перечня и метаданных, однако политика OpenClaw определяет, будет ли `thread/start`
отправлять `config.apps[appId].enabled = true` для указанного в перечне доступного
приложения, даже если Codex в данный момент помечает его как отключённое. Неизвестные или
отсутствующие идентификаторы приложений по-прежнему приводят к безопасному отказу; этот путь
только активирует плагины из каталога через `plugin/install` и обновляет перечень.
Подключайте OpenClaw только к тем удалённым app-server, которым можно доверять установку
управляемых OpenClaw плагинов и обновление перечня приложений.

## Режимы подтверждений и песочницы

Локальные сеансы app-server через stdio по умолчанию используют режим YOLO:
`approvalPolicy: "never"`, `approvalsReviewer: "user"` и
`sandbox: "danger-full-access"`. Такая модель доверенного локального оператора позволяет
автоматическим запросам OpenClaw и Heartbeat продолжать работу без нативных запросов
подтверждения, на которые некому ответить.

Если локальный файл системных требований Codex запрещает неявные значения YOLO для
подтверждений, проверяющего или песочницы, OpenClaw вместо этого рассматривает неявное
значение по умолчанию как guardian и выбирает разрешённые права guardian.
`tools.exec.mode: "auto"` также принудительно включает проверяемые guardian подтверждения
Codex и не сохраняет небезопасные устаревшие переопределения
`approvalPolicy: "never"` или `sandbox: "danger-full-access"`; для намеренного режима
без подтверждений задайте `tools.exec.mode: "full"`. Записи
`[[remote_sandbox_config]]` из того же файла требований, соответствующие имени хоста,
учитываются при выборе режима песочницы по умолчанию.

Задайте `appServer.mode: "guardian"` для подтверждений, проверяемых guardian Codex:

```json5
{
  plugins: {
    entries: {
      codex: {
        enabled: true,
        config: {
          appServer: {
            mode: "guardian",
            serviceTier: "priority",
          },
        },
      },
    },
  },
}
```

Предустановка `guardian` разворачивается в `approvalPolicy: "on-request"`,
`approvalsReviewer: "auto_review"` и `sandbox: "workspace-write"`, если эти значения
разрешены. Отдельные поля политики переопределяют `mode`. Более старое значение
проверяющего `guardian_subagent` по-прежнему принимается как псевдоним для совместимости,
но в новых конфигурациях следует использовать `auto_review`.

Когда активна песочница OpenClaw, локальный процесс Codex app-server всё равно выполняется
на хосте Gateway. Поэтому OpenClaw отключает для этого запроса нативный режим кода Codex,
пользовательские MCP-серверы и выполнение плагинов на базе приложений, вместо того чтобы
считать изоляцию на стороне хоста Codex эквивалентной серверной части песочницы OpenClaw.
Доступ к оболочке предоставляется через динамические инструменты на базе песочницы OpenClaw,
такие как `sandbox_exec` и `sandbox_process`, когда доступны обычные инструменты
выполнения и управления процессами.

<Note>
На хостах песочницы OpenClaw на базе Docker (`agents.defaults.sandbox.mode` настроен
на серверную часть Docker) `openclaw doctor` проверяет, разрешает ли хост пространства
имён непривилегированного пользователя, а когда исходящий сетевой трафик песочницы Docker
отключён — также пространства имён сети, необходимые вложенному Codex `bwrap` для
выполнения команд оболочки с `workspace-write` внутри контейнера песочницы. Неудачная
проверка обычно проявляется ошибкой `bwrap: setting up uid map: Permission denied` или
`bwrap: loopback: Failed RTM_NEWADDR: Operation not permitted` на хостах Ubuntu/AppArmor.
Исправьте указанную политику пространств имён хоста для пользователя службы OpenClaw и
перезапустите Gateway; отдавайте предпочтение ограниченному профилю AppArmor для процесса
службы, а не общесистемному запасному варианту
`kernel.apparmor_restrict_unprivileged_userns=0`, и не предоставляйте контейнеру Docker
более широкие привилегии только ради работы вложенного `bwrap`.
</Note>

## Нативное выполнение в песочнице

Стабильное поведение по умолчанию — безопасный отказ: активная песочница OpenClaw отключает
нативные поверхности выполнения Codex, которые иначе запускались бы с хоста Codex
app-server. Используйте `appServer.experimental.sandboxExecServer: true`, только если
хотите опробовать поддержку удалённой среды Codex с серверной частью песочницы OpenClaw.
Этот предварительный вариант работает со всеми поддерживаемыми версиями Codex app-server.

```json5
{
  plugins: {
    entries: {
      codex: {
        enabled: true,
        config: {
          appServer: {
            experimental: {
              sandboxExecServer: true,
            },
          },
        },
      },
    },
  },
}
```

Когда флаг включён и текущий сеанс OpenClaw работает в песочнице, OpenClaw запускает
локальный exec-server через local loopback на базе активной песочницы, регистрирует его
в Codex app-server и запускает поток и запрос Codex с этой средой, принадлежащей OpenClaw.
Если app-server не может зарегистрировать среду, выполнение завершается безопасным отказом,
а не незаметным переходом к выполнению на хосте.

Этот предварительный вариант работает только локально. Удалённый WebSocket app-server
не может обратиться к exec-server через local loopback, если не работает на том же хосте,
поэтому OpenClaw отклоняет такое сочетание.

## Изоляция аутентификации и окружения

В домашнем каталоге для каждого агента по умолчанию аутентификация выбирается в следующем
порядке:

1. Явный профиль аутентификации OpenClaw Codex для агента.
2. Существующая учётная запись app-server в домашнем каталоге Codex этого агента.
3. Только для локальных запусков app-server через stdio: `CODEX_API_KEY`, затем
   `OPENAI_API_KEY`, если учётная запись app-server отсутствует, а аутентификация OpenAI
   всё ещё требуется.

Когда OpenClaw обнаруживает профиль аутентификации Codex в стиле подписки ChatGPT
(тип учётных данных OAuth или токена), он удаляет `CODEX_API_KEY` и `OPENAI_API_KEY`
из среды дочернего процесса Codex. Благодаря этому API-ключи уровня Gateway остаются
доступны для векторных представлений или прямых моделей OpenAI, но нативные запросы
Codex app-server не начинают случайно оплачиваться через API.

Явные профили Codex с API-ключом и резервное использование ключей из окружения для
локального stdio применяют вход через app-server вместо наследования окружения дочернего
процесса. Подключения к WebSocket app-server не получают резервные API-ключи из окружения
Gateway; используйте явный профиль аутентификации или собственную учётную запись
удалённого app-server.

Запуски app-server через stdio по умолчанию наследуют окружение процесса OpenClaw.
OpenClaw управляет связующим механизмом учётной записи Codex app-server и задаёт
`CODEX_HOME` как отдельный каталог для агента в состоянии OpenClaw этого агента.
Благодаря этому конфигурация Codex, учётные записи, кэш и данные плагинов, а также состояние
потоков ограничиваются агентом OpenClaw и не проникают из личного домашнего каталога
оператора `~/.codex`.

Задайте `appServer.homeScope: "user"`, чтобы совместно использовать нативное состояние
Codex с Codex Desktop и CLI. Этот режим локального домашнего каталога пользователя
поддерживает управляемый stdio и явный транспорт Unix. Он использует `$CODEX_HOME`,
если эта переменная задана, и `~/.codex` в противном случае, включая нативную
аутентификацию, конфигурацию, плагины и потоки. OpenClaw не использует для app-server
свой связующий механизм профиля аутентификации. Подтверждённые запросы владельца могут
использовать `codex_threads`, чтобы выводить список этих потоков (с необязательным фильтром
`search`), читать, создавать ответвления, переименовывать, архивировать и разархивировать
их. Создайте ответвление потока, прежде чем продолжать его в OpenClaw; независимые
процессы Codex не координируют одновременную запись в один поток.

Такое явное включение `homeScope` применяется к обычным сеансам среды выполнения. Чат,
созданный через Codex Sessions, вместо этого использует собственное контролируемое
подключение, которое сохраняет конфигурацию аутентификации и поставщика нативного
подключения для канонической ветви и будущих возобновлений.

В контролируемом чате с зафиксированной моделью `codex_threads` не может подключить другое
ответвление или архивировать привязанный к чату нативный поток. Вывод списка и чтение
только метаданных остаются доступными. Для чтения необработанных расшифровок требуется
`allowRawTranscripts`; когда этот параметр отключён, поиск по списку также отклоняется,
поскольку нативный поиск может находить совпадения в предварительном просмотре
расшифровок. Для переименования, разархивирования, отдельного ответвления и архивирования
постороннего потока, не принадлежащего другому чату OpenClaw, требуется
`allowWriteControls`. Ни один из этих параметров не обходит зафиксированную привязку.

OpenClaw не переопределяет `HOME` для обычных локальных запусков app-server.
Подпроцессы, запускаемые Codex, такие как `openclaw`, `gh`, `git`, облачные CLI и команды
оболочки, видят обычный домашний каталог процесса и могут находить пользовательскую
конфигурацию и токены. Codex также может обнаруживать `$HOME/.agents/skills` и
`$HOME/.agents/plugins/marketplace.json`; обнаружение `.agents` намеренно использует
общий домашний каталог оператора и не связано с изолированным состоянием `~/.codex`.

В области агента по умолчанию плагины OpenClaw и снимки Skills OpenClaw по-прежнему
поступают через собственный реестр плагинов и загрузчик Skills OpenClaw; личные ресурсы
Codex из `~/.codex` — нет. Если в домашнем каталоге Codex есть полезные Skills или плагины
Codex CLI, которые должны стать частью изолированного агента OpenClaw, явно
проинвентаризируйте их:

```bash
openclaw migrate codex --dry-run
openclaw migrate apply codex --yes
```

Если развёртыванию требуется дополнительная изоляция окружения, добавьте соответствующие
переменные в `appServer.clearEnv`:

```json5
{
  plugins: {
    entries: {
      codex: {
        enabled: true,
        config: {
          appServer: {
            clearEnv: ["CODEX_API_KEY", "OPENAI_API_KEY"],
          },
        },
      },
    },
  },
}
```

`appServer.clearEnv` влияет только на запускаемый дочерний процесс Codex app-server.
OpenClaw удаляет `CODEX_HOME` и `HOME` из этого списка во время нормализации локального
запуска: `CODEX_HOME` продолжает указывать на выбранную область агента или пользователя,
а `HOME` по-прежнему наследуется, чтобы подпроцессы могли использовать обычное состояние
домашнего каталога пользователя.

## Динамические инструменты

Для динамических инструментов Codex по умолчанию используется загрузка `searchable`;
они предоставляются в пространстве имён `openclaw` с `deferLoading: true`. OpenClaw
не предоставляет динамические инструменты, дублирующие нативные операции Codex с рабочей
областью или собственную поверхность поиска инструментов Codex:

- `read`
- `write`
- `edit`
- `apply_patch`
- `exec`
- `process`
- `update_plan`
- `tool_call`
- `tool_describe`
- `tool_search`
- `tool_search_code`

Большинство остальных интеграционных инструментов OpenClaw, например обмен сообщениями,
работа с мультимедиа, Cron, браузером, узлами и Gateway, а также `heartbeat_respond`
и `web_search`, доступны через поиск инструментов Codex в этом пространстве имён.
Это уменьшает начальный контекст модели. Небольшой набор инструментов остаётся доступным
для прямого вызова независимо от `codexDynamicToolsLoading`, поскольку поиск инструментов
Codex может быть недоступен или возвращать только пространство коннекторов:
`agents_list`, `sessions_spawn` и `sessions_yield`. Инструкции разработчика по-прежнему
направляют обычных субагентов Codex к нативному `spawn_agent` для нативной работы
субагентов Codex, а `sessions_spawn` остаётся доступным для явно заданного делегирования
через OpenClaw или ACP. Ответы источника, использующие только инструмент сообщений,
также остаются прямыми, поскольку это контракт управления запросом.

Инструменты с пометкой `catalogMode: "direct-only"`, включая инструмент OpenClaw
`computer`, объединяются в пространство имён `openclaw_direct`. OpenClaw добавляет это
пространство имён в список Codex `code_mode.direct_only_tool_namespaces`, не заменяя
записи, указанные оператором. Поэтому Codex предоставляет эти инструменты как
`DirectModelOnly` в обычных потоках и потоках только для режима кода, а не направляет
их через вложенные вызовы режима кода `tools.*`. Эта граница необходима для результатов
с изображениями: вложенная сериализация режима кода преобразует вывод изображения
в текст, из-за чего снимок экрана, необходимый для следующего действия с компьютером,
будет утрачен.

Задавайте `codexDynamicToolsLoading: "direct"` только при подключении к пользовательскому
Codex app-server, который не поддерживает поиск отложенно загружаемых динамических
инструментов, или при отладке полной полезной нагрузки инструментов.

## Тайм-ауты

Вызовы динамических инструментов, принадлежащих OpenClaw, ограничиваются независимо от
`appServer.requestTimeoutMs`. Для каждого запроса Codex `item/tool/call` используется
первый доступный тайм-аут в следующем порядке:

- Положительное значение аргумента `timeoutMs` для отдельного вызова.
- Для `image_generate` — `agents.defaults.imageGenerationModel.timeoutMs`.
- Для `image_generate` без настроенного тайм-аута — стандартное значение
  генерации изображений 120 секунд.
- Для инструмента анализа медиаданных `image` — значение
  `tools.media.image.timeoutSeconds`, преобразованное в миллисекунды, либо
  стандартное значение для медиаданных 60 секунд. При анализе изображений
  оно применяется к самому запросу и не уменьшается на время, затраченное
  на предшествующую подготовку.
- Для инструмента `message` — фиксированное стандартное значение 120 секунд.
- Стандартное значение для динамических инструментов — 90 секунд.

Этот сторожевой таймер задаёт внешний бюджет динамического вызова
`item/tool/call`. Тайм-ауты запросов, зависящие от конкретного провайдера,
действуют внутри этого вызова и сохраняют собственную семантику. Бюджеты
динамических инструментов ограничены 600000 мс. При истечении тайм-аута
OpenClaw, если это поддерживается, прерывает сигнал инструмента и возвращает
Codex ответ о сбое динамического инструмента, чтобы ход мог продолжиться, а
сеанс не оставался в состоянии `processing`.

После того как Codex принимает ход, а также после ответа OpenClaw на запрос
app-server в рамках хода, среда выполнения ожидает, что Codex продолжит
обработку текущего хода и в конечном итоге завершит нативный ход событием
`turn/completed`. Если app-server не проявляет активности в течение
`appServer.turnCompletionIdleTimeoutMs`, OpenClaw пытается прервать ход Codex,
регистрирует диагностический тайм-аут и освобождает полосу сеанса OpenClaw,
чтобы последующие сообщения чата не ожидали завершения устаревшего нативного
хода.

Большинство нетерминальных уведомлений того же хода отключают этот короткий
сторожевой таймер, поскольку Codex подтвердил, что ход всё ещё активен. Для
передачи управления инструментам используется более длительный бюджет
бездействия после инструмента: после возврата OpenClaw ответа
`item/tool/call`, после завершения нативных элементов инструментов, таких как
`commandExecution`, после завершений необработанного
`custom_tool_call_output`, а также после необработанного прогресса ассистента,
завершения рассуждений или прогресса рассуждений после инструмента. Если
настроено значение
`appServer.postToolRawAssistantCompletionIdleTimeoutMs`, защитный механизм
использует его; в противном случае по умолчанию применяется пять минут. Этот
же бюджет после инструмента продлевает сторожевой таймер прогресса на период
безмолвного формирования ответа до того, как Codex выдаст следующее событие
текущего хода. После завершения рассуждений, завершения комментариев
`agentMessage`, а также необработанного прогресса рассуждений или ассистента
до вызова инструмента может автоматически последовать окончательный ответ,
поэтому для них используется защитный таймер ответа после прогресса, а полоса
сеанса не освобождается немедленно. Освобождение после вывода ассистента
активируют только завершённые элементы `agentMessage`, являющиеся
окончательными или не являющиеся комментариями, а также необработанные
завершения ассистента до вызова инструмента: если после этого Codex перестаёт
проявлять активность, не отправив `turn/completed`, OpenClaw пытается прервать
нативный ход и освобождает полосу сеанса. Безопасные для повторного
воспроизведения сбои app-server через stdio, включая тайм-ауты ожидания
завершения хода без признаков ответа ассистента, инструментов, активных
элементов или побочных эффектов, один раз повторяются с новым экземпляром
app-server. При небезопасных тайм-аутах зависший клиент app-server всё равно
выводится из эксплуатации, а полоса сеанса OpenClaw освобождается. При этом
устаревшая привязка нативного потока также очищается, а автоматическое
повторное воспроизведение не выполняется. Тайм-ауты ожидания завершения
отображают сообщения, относящиеся к Codex: в безопасных для повторного
воспроизведения случаях сообщается, что ответ может быть неполным, а в
небезопасных случаях пользователю предлагается проверить текущее состояние
перед повторной попыткой. Общедоступная диагностика тайм-аутов включает
структурные поля, такие как метод последнего уведомления app-server,
идентификатор, тип и роль необработанного элемента ответа ассистента,
количество активных запросов и элементов, а также состояние активированного
сторожевого таймера. Если последним уведомлением является необработанный
элемент ответа ассистента, диагностика также включает ограниченный по длине
предварительный фрагмент текста ассистента. Необработанное содержимое
приглашений и инструментов в неё не включается.

## Обнаружение моделей

По умолчанию Plugin Codex запрашивает у app-server список доступных моделей.
Доступностью моделей управляет app-server Codex, поэтому список может
измениться при обновлении встроенной версии `@openai/codex` в OpenClaw или
если развёртывание направляет `appServer.command` на другой исполняемый файл
Codex. Доступность также может зависеть от учётной записи. Чтобы просмотреть
актуальный каталог для этой среды выполнения и учётной записи, выполните
`/codex models` на работающем Gateway.

Если обнаружение завершается ошибкой или превышает тайм-аут, OpenClaw
использует встроенный резервный каталог:

| Идентификатор модели | Отображаемое имя | Уровни рассуждения     |
| -------------------- | --------------- | ---------------------- |
| `gpt-5.5`            | gpt-5.5         | low, medium, high, xhigh |
| `gpt-5.4-mini`       | GPT-5.4-Mini    | low, medium, high, xhigh |

<Note>
Текущая встроенная среда выполнения использует `@openai/codex` версии
`0.144.1`. Проверка `model/list` на этом встроенном app-server вернула
следующие общедоступные строки средства выбора:

| Идентификатор модели | Входные модальности | Уровни рассуждения                      |
| -------------------- | ------------------- | --------------------------------------- |
| `gpt-5.6-sol`        | текст, изображение  | low, medium, high, xhigh, max, ultra    |
| `gpt-5.6-terra`      | текст, изображение  | low, medium, high, xhigh, max, ultra    |
| `gpt-5.6-luna`       | текст, изображение  | low, medium, high, xhigh, max           |
| `gpt-5.5`            | текст, изображение  | low, medium, high, xhigh                |
| `gpt-5.4`            | текст, изображение  | low, medium, high, xhigh                |
| `gpt-5.4-mini`       | текст, изображение  | low, medium, high, xhigh                |
| `gpt-5.2`            | текст, изображение  | low, medium, high, xhigh                |

Каталог app-server может сообщать уровень `ultra`; элементы управления
рассуждениями OpenClaw в настоящее время предоставляют уровни до `max`.

Актуальные строки средства выбора зависят от учётной записи и могут меняться
в зависимости от учётной записи, каталога Codex или встроенной версии;
запускайте `/codex models`, чтобы получить текущий список, вместо использования
таблицы, зафиксированной на определённый момент времени. В каталоге app-server
также могут присутствовать скрытые модели для внутренних или
специализированных сценариев, которые не являются обычными вариантами в
средстве выбора моделей.
</Note>

Настройте обнаружение в `plugins.entries.codex.config.discovery`:

```json5
{
  plugins: {
    entries: {
      codex: {
        enabled: true,
        config: {
          discovery: {
            enabled: true,
            timeoutMs: 2500,
          },
        },
      },
    },
  },
}
```

Отключите обнаружение, если при запуске не требуется опрашивать Codex и
следует использовать только резервный каталог:

```json5
{
  plugins: {
    entries: {
      codex: {
        enabled: true,
        config: {
          discovery: {
            enabled: false,
          },
        },
      },
    },
  },
}
```

## Файлы начальной настройки рабочей области

Codex самостоятельно обрабатывает `AGENTS.md` с помощью нативного обнаружения
документации проекта. OpenClaw не записывает синтетические файлы документации
проекта Codex и не использует резервные имена файлов Codex для файлов
персоны, поскольку резервные варианты Codex применяются только при отсутствии
`AGENTS.md`.

Для согласованности с рабочей областью OpenClaw среда выполнения Codex
передаёт остальные файлы начальной настройки как инструкции разработчика, но
не одинаковым образом:

- `TOOLS.md` передаётся как **унаследованные** инструкции разработчика Codex,
  поэтому нативные субагенты Codex, созданные во время хода, также их видят.
- `SOUL.md`, `IDENTITY.md` и `USER.md` передаются как инструкции по
  взаимодействию **в рамках хода**. Нативные субагенты Codex их не наследуют,
  благодаря чему ходы субагентов не получают персону родительского агента и
  профиль пользователя.
- Компактный загруженный список Skills OpenClaw также передаётся как
  инструкции разработчика по взаимодействию в рамках хода, поэтому нативные
  субагенты Codex его тоже не наследуют.
- Содержимое `HEARTBEAT.md` не внедряется; ходы Heartbeat получают в режиме
  взаимодействия указание прочитать файл, если он существует и не пуст.
- Содержимое `MEMORY.md` из настроенной рабочей области агента не вставляется
  во входные данные нативного хода Codex, когда для этой рабочей области
  доступны инструменты памяти; если файл существует, среда выполнения
  добавляет в инструкции разработчика по взаимодействию в рамках хода
  краткое указание на память рабочей области, а Codex должен использовать
  `memory_search` или `memory_get`, когда требуется долговременная память.
  Если инструменты отключены, поиск по памяти недоступен или активная рабочая
  область отличается от рабочей области памяти агента, для `MEMORY.md`
  используется обычный ограниченный контекст хода.
- `BOOTSTRAP.md`, если он присутствует, передаётся как справочный контекст
  входных данных хода OpenClaw.

## Переопределения через переменные окружения

Переопределения через переменные окружения остаются доступными для локального
тестирования:

- `OPENCLAW_CODEX_APP_SERVER_BIN`
- `OPENCLAW_CODEX_APP_SERVER_ARGS`
- `OPENCLAW_CODEX_APP_SERVER_MODE=yolo|guardian`
- `OPENCLAW_CODEX_APP_SERVER_APPROVAL_POLICY`
- `OPENCLAW_CODEX_APP_SERVER_SANDBOX`

`OPENCLAW_CODEX_APP_SERVER_BIN` позволяет обойти управляемый исполняемый файл,
если `appServer.command` не задан.

`OPENCLAW_CODEX_APP_SERVER_GUARDIAN=1` удалена. Вместо неё используйте
`plugins.entries.codex.config.appServer.mode: "guardian"` либо
`OPENCLAW_CODEX_APP_SERVER_MODE=guardian` для разового локального
тестирования. Для воспроизводимых развёртываний предпочтительна конфигурация,
поскольку она сохраняет поведение Plugin в том же проверяемом файле, что и
остальные настройки среды выполнения Codex.
