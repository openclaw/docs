Плагины сообщества — это сторонние пакеты, которые расширяют возможности OpenClaw,
добавляя каналы, инструменты, провайдеров, хуки и другие функции. Используйте
[ClawHub](/clawhub) как основной ресурс для поиска общедоступных плагинов
сообщества.

## Поиск плагинов

Выполните поиск в ClawHub через CLI:

```bash
openclaw plugins search "calendar"
```

Установите плагин ClawHub, явно указав префикс источника:

```bash
openclaw plugins install clawhub:<package-name>
```

В период перехода после запуска npm по-прежнему поддерживается как способ прямой установки:

```bash
openclaw plugins install npm:<package-name>
```

Распространённые примеры установки, обновления, проверки
и удаления приведены в разделе [Управление плагинами](/ru/plugins/manage-plugins). Полный
справочник команд и правила выбора источника см. в разделе [`openclaw plugins`](/ru/cli/plugins).

## Публикация плагинов

Публикуйте общедоступные плагины сообщества в ClawHub, чтобы пользователи OpenClaw могли
находить и устанавливать их. ClawHub содержит актуальный список пакетов, историю выпусков,
статус сканирования и подсказки по установке; документация не ведёт статический
каталог сторонних плагинов.

```bash
clawhub package publish your-org/your-plugin --dry-run
clawhub package publish your-org/your-plugin
```

Перед публикацией убедитесь, что у плагина есть метаданные пакета, манифест
плагина, документация по настройке и чётко определённый ответственный за сопровождение. Перед
созданием выпуска ClawHub проверяет область владельца, имя пакета, версию,
ограничения файлов и метаданные исходного кода, а затем скрывает новые выпуски на обычных страницах
установки и скачивания до завершения проверки и подтверждения.

Контрольный список перед публикацией:

| Требование               | Зачем                                                |
| ------------------------ | ---------------------------------------------------- |
| Опубликован в ClawHub    | Для работы подсказок `openclaw plugins install` |
| Общедоступный репозиторий GitHub | Проверка исходного кода, отслеживание проблем, прозрачность |
| Документация по настройке и использованию | Пользователям необходимо знать, как настроить плагин |
| Активное сопровождение   | Недавние обновления или оперативная обработка проблем |

Полные условия публикации:

- [Публикация в ClawHub](/ru/clawhub/publishing) — владельцы, области, выпуски,
  проверка, валидация пакетов и передача пакетов
- [Создание плагинов](/ru/plugins/building-plugins) — структура пакета плагина
  и процесс первой публикации
- [Манифест плагина](/ru/plugins/manifest) — поля нативного манифеста плагина
