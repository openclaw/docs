Community plugins — это сторонние пакеты, которые расширяют OpenClaw каналами,
инструментами, провайдерами, хуками или другими возможностями. Используйте [ClawHub](/ru/clawhub) как
основную поверхность обнаружения публичных Community plugins.

## Найти Plugin

Ищите в ClawHub из CLI:

```bash
openclaw plugins search "calendar"
```

Установите Plugin из ClawHub с явным префиксом источника:

```bash
openclaw plugins install clawhub:<package-name>
```

npm остается поддерживаемым путем прямой установки на время перехода запуска:

```bash
openclaw plugins install npm:<package-name>
```

Используйте [Управление Plugin](/ru/plugins/manage-plugins) для распространенных примеров установки, обновления,
проверки и удаления. Используйте [`openclaw plugins`](/ru/cli/plugins) для
полного справочника команд и правил выбора источника.

## Публикация Plugin

Публикуйте публичные Community plugins в ClawHub, если хотите, чтобы пользователи OpenClaw могли
находить и устанавливать их. ClawHub владеет актуальным списком пакетов, историей
релизов, статусом сканирования и подсказками по установке; документация не ведет статический
каталог сторонних Plugin.

```bash
clawhub package publish your-org/your-plugin --dry-run
clawhub package publish your-org/your-plugin
```

Перед публикацией убедитесь, что у Plugin есть метаданные пакета, манифест Plugin,
документация по настройке и ясно обозначенный ответственный за сопровождение. ClawHub проверяет область владельца,
имя пакета, версию, ограничения файлов и метаданные источника, прежде чем создать
релиз, а затем скрывает новые релизы от обычных поверхностей установки и скачивания
до завершения проверки и верификации.

Используйте этот контрольный список перед публикацией:

| Требование          | Зачем                                                 |
| -------------------- | --------------------------------------------------- |
| Опубликован в ClawHub | Пользователям нужны рабочие подсказки `openclaw plugins install` |
| Публичный репозиторий GitHub   | Проверка исходного кода, отслеживание задач, прозрачность         |
| Документация по настройке и использованию | Пользователям нужно знать, как его настроить              |
| Активное сопровождение   | Недавние обновления или оперативная обработка задач         |

Используйте эти страницы для полного контракта публикации:

- [Публикация ClawHub](/ru/clawhub/publishing) объясняет владельцев, области, релизы,
  ревью, проверку пакетов и передачу пакетов.
- [Создание Plugin](/ru/plugins/building-plugins) показывает форму пакета Plugin
  и рабочий процесс первой публикации.
- [Манифест Plugin](/ru/plugins/manifest) определяет поля нативного манифеста Plugin.
