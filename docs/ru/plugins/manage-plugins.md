В Control UI реализован типовой рабочий процесс поиска, установки, включения и отключения
плагинов. CLI также поддерживает обновление, удаление, расширенную настройку и явное
управление источником установки. Полное описание команд, флагов, правил выбора
источника и особых случаев см. в разделе [`openclaw plugins`](/ru/cli/plugins).

Типовой рабочий процесс в CLI: найти пакет, установить его из ClawHub, npm, git или по
локальному пути, дождаться автоматического перезапуска управляемого Gateway (либо
перезапустить его вручную), а затем проверить регистрации плагина в среде выполнения.

## Использование Control UI

Откройте **Плагины** в Control UI или используйте `/settings/plugins` относительно
настроенного базового пути Control UI. Например, для базового пути `/openclaw` используется
`/openclaw/settings/plugins`. На странице есть две вкладки:

- **Установленные** — полный список локальных объектов, сгруппированных по категориям (каналы,
  поставщики моделей, память, инструменты). Каждая строка открывает подробное представление; меню
  дополнительных действий (`…`) позволяет включить или отключить плагин, а для установленных
  извне плагинов также предлагает действие **Удалить**. На вкладке также перечислены настроенные
  [серверы MCP](/ru/cli/mcp) с такими же доступными через меню действиями включения, отключения и удаления,
  изменяющими `mcp.servers` в конфигурации Gateway.
- **Обзор** — магазин с избранными плагинами, входящими в OpenClaw, официальными
  внешними плагинами и подобранной коллекцией коннекторов. Карточки коннекторов либо добавляют
  размещённый сервер MCP одним нажатием (GitHub, Notion, Linear, Sentry,
  Home Assistant), либо открывают поиск в ClawHub с предварительно заполненным запросом. При вводе в поле поиска
  выполняется встроенный запрос к [ClawHub](https://clawhub.ai/plugins), а также добавляется раздел **Из
  ClawHub** с количеством скачиваний и значками проверки источника.

Входящие в комплект плагины не требуют установки пакета. В их меню доступно действие **Включить**
или **Отключить**. Например, Workboard входит в OpenClaw и по умолчанию отключён,
поэтому для его активации выберите **Включить**. Встроенные плагины нельзя
удалить — их можно только отключить.

Для доступа к каталогу и поиску требуется `operator.read`. Для установки, включения, отключения,
удаления и изменения серверов MCP требуется `operator.admin`. Установка из ClawHub
выполняется Gateway с сохранением проверок доверия, целостности и политики установки
плагинов. Включение установленного плагина администратором также фиксирует
явное доверие: выбранный плагин добавляется в существующий ограничивающий список
`plugins.allow`. Явная запись `plugins.deny` остаётся приоритетной, и
перед включением плагина её необходимо удалить.

После установки или удаления кода плагина требуется перезапуск Gateway. Изменения
состояния включения можно применить без перезапуска, если установленный плагин и текущая
среда выполнения Gateway это поддерживают; в противном случае UI сообщает о необходимости перезапуска.
Для коннекторов MCP на основе OAuth после добавления всё равно требуется однократно выполнить
`openclaw mcp login <name>` в CLI.

Control UI не устанавливает плагины из произвольных источников npm, git или локальных путей,
не обновляет плагины и не предоставляет расширенную настройку плагинов. Для этих операций
используйте описанные ниже рабочие процессы CLI.

## Просмотр и поиск плагинов

```bash
openclaw plugins list
openclaw plugins list --enabled
openclaw plugins list --verbose
openclaw plugins list --json
openclaw plugins search "calendar"
```

`--json` для скриптов:

```bash
openclaw plugins list --json \
  | jq '.plugins[] | {id, enabled, format, source, dependencyStatus}'
```

`plugins list` выполняет холодную проверку списка: она показывает, что OpenClaw может обнаружить
по конфигурации, манифестам и сохранённому реестру плагинов. Эта проверка не подтверждает,
что уже запущенный Gateway импортировал среду выполнения плагина. Вывод JSON содержит
диагностику реестра и значение `dependencyStatus` каждого плагина (разрешаются ли объявленные
`dependencies`/`optionalDependencies` на диске).

`plugins search` запрашивает в ClawHub доступные для установки пакеты плагинов и выводит
для каждого результата подсказку по установке (`openclaw plugins install clawhub:<package>`).

## Включение и отключение плагинов

```bash
openclaw plugins enable <plugin-id>
openclaw plugins disable <plugin-id>
```

Переключает запись плагина в конфигурации, не затрагивая установленные файлы. Некоторые
встроенные плагины (встроенные поставщики моделей и синтеза речи, встроенный плагин браузера)
включены по умолчанию; для остальных после установки требуется `enable`.

## Установка плагинов

```bash
# Поиск пакетов плагинов в ClawHub.
openclaw plugins search "calendar"

# Установка из ClawHub.
openclaw plugins install clawhub:<package>
openclaw plugins install clawhub:<package>@1.2.3
openclaw plugins install clawhub:<package>@beta

# Установка из npm.
openclaw plugins install npm:<package>
openclaw plugins install npm:@scope/openclaw-plugin@1.2.3
openclaw plugins install npm:@openclaw/codex

# Установка из локального артефакта npm-pack.
openclaw plugins install npm-pack:<path.tgz>

# Установка из git или локальной рабочей копии для разработки.
openclaw plugins install git:github.com/acme/openclaw-plugin@v1.0.0
openclaw plugins install ./my-plugin
openclaw plugins install --link ./my-plugin
```

Во время переходного периода запуска спецификации пакетов без префикса устанавливаются из npm, если
имя не совпадает с идентификатором встроенного или официального плагина — в таком случае OpenClaw вместо этого использует
соответствующую локальную или официальную копию. Для детерминированного выбора источника используйте `clawhub:`, `npm:`, `git:` или
`npm-pack:`. Пакеты из встроенного и официального каталогов OpenClaw считаются доверенными
наряду с пакетами ClawHub. Для новых произвольных источников npm,
git, локальных путей или архивов, `npm-pack:` либо маркетплейсов при неинтерактивной установке требуется
`--force` после проверки
источника и подтверждения доверия к нему.

`--force` подтверждает источник, отличный от ClawHub, без запроса и при необходимости перезаписывает
существующий объект установки. Для обычных обновлений отслеживаемой установки из npm,
ClawHub или hook-pack используйте вместо этого `openclaw plugins update`. При
`--link` параметр `--force` только подтверждает источник; связанный каталог не
копируется и не перезаписывается.

## Перезапуск и проверка

Запущенный управляемый Gateway с включённой перезагрузкой конфигурации автоматически перезапускается
после установки, обновления или удаления кода плагина. Если Gateway
неуправляемый или перезагрузка отключена, перезапустите его вручную перед проверкой действующих
поверхностей среды выполнения:

```bash
openclaw gateway restart
openclaw plugins inspect <plugin-id> --runtime --json
```

`inspect --runtime` загружает модуль плагина и подтверждает регистрацию
поверхностей среды выполнения (инструментов, перехватчиков, служб, методов Gateway, HTTP-маршрутов и принадлежащих плагину
команд CLI). Обычные `inspect` и `list` выполняют только холодные проверки манифеста, конфигурации и реестра.

## Обновление плагинов

```bash
openclaw plugins update <plugin-id>
openclaw plugins update <npm-package-or-spec>
openclaw plugins update --all
openclaw plugins update <plugin-id> --dry-run
```

При передаче идентификатора плагина повторно используется его отслеживаемая спецификация установки: сохранённые dist-теги
(`@beta`) и точно закреплённые версии переносятся в последующие запуски `update <plugin-id>`.

`openclaw plugins update --all` — способ массового обслуживания. Он по-прежнему
учитывает обычные отслеживаемые спецификации установки, но записи доверенных официальных плагинов OpenClaw
синхронизируются с текущей целевой версией официального каталога вместо
сохранения привязки к устаревшему точному официальному пакету; когда `update.channel` имеет значение
`beta`, при синхронизации предпочтение отдаётся бета-ветке выпусков. Чтобы не изменять точную или помеченную тегом
официальную спецификацию, используйте адресную команду `update <plugin-id>`.

Для установок из npm передайте явную спецификацию пакета, чтобы изменить отслеживаемую
запись:

```bash
openclaw plugins update @scope/openclaw-plugin@beta
openclaw plugins update @scope/openclaw-plugin
```

Вторая команда возвращает плагин к стандартной ветке выпусков реестра,
если ранее он был привязан к точной версии или тегу.

Точные правила резервного выбора и закрепления версий см. в разделе
[`openclaw plugins`](/ru/cli/plugins#update).

## Удаление плагинов

```bash
openclaw plugins uninstall <plugin-id> --dry-run
openclaw plugins uninstall <plugin-id>
openclaw plugins uninstall <plugin-id> --keep-files
```

При удалении удаляются запись плагина в конфигурации, сохранённая запись индекса плагинов,
записи списков разрешений и запретов, а также связанные записи `plugins.load.paths`, если
применимо. Управляемый каталог установки удаляется, если не передан параметр
`--keep-files`. Запущенный управляемый Gateway автоматически перезапускается, если
удаление изменяет источник плагина.

В режиме Nix (`OPENCLAW_NIX_MODE=1`) установка, обновление, удаление,
включение и отключение плагинов недоступны; управляйте этими параметрами в исходном коде Nix,
используемом для установки.

## Выбор источника

| Источник    | Когда использовать                                                          | Пример                                                         |
| ----------- | --------------------------------------------------------------------------- | -------------------------------------------------------------- |
| ClawHub     | Нужны встроенные в OpenClaw поиск, сводки сканирования, версии и подсказки   | `openclaw plugins install clawhub:<package>`                   |
| git         | Нужна ветка, тег или коммит из репозитория                                   | `openclaw plugins install git:github.com/<owner>/<repo>@<ref>` |
| локальный путь | Плагин разрабатывается или тестируется на том же компьютере               | `openclaw plugins install --link ./my-plugin`                  |
| маркетплейс | Устанавливается совместимый с Claude плагин из маркетплейса                  | `openclaw plugins install <plugin> --marketplace <source>`     |
| npm pack    | Локальный артефакт пакета проверяется с использованием семантики установки npm | `openclaw plugins install npm-pack:<path.tgz>`                 |
| npmjs.com   | Пакеты JavaScript уже публикуются или требуются dist-теги npm либо частный реестр | `openclaw plugins install npm:@acme/openclaw-plugin`           |

Управляемые установки по локальному пути должны указывать на каталоги или архивы плагинов. Помещайте
отдельные файлы плагинов в `plugins.load.paths`, а не устанавливайте их
с помощью `plugins install`.

## Публикация плагинов

ClawHub — основная общедоступная площадка поиска плагинов OpenClaw. Публикуйте
их там, если хотите, чтобы пользователи могли найти метаданные плагина, историю версий, результаты
сканирования реестра и подсказки по установке до начала установки.

```bash
npm i -g clawhub
clawhub login
clawhub package publish your-org/your-plugin --dry-run
clawhub package publish your-org/your-plugin
clawhub package publish your-org/your-plugin@v1.0.0
```

Перед публикацией нативные плагины npm должны содержать манифест плагина (`openclaw.plugin.json`) и
метаданные `package.json`:

```json package.json
{
  "name": "@acme/openclaw-plugin",
  "version": "1.0.0",
  "type": "module",
  "openclaw": {
    "extensions": ["./dist/index.js"]
  }
}
```

```bash
npm publish --access public
openclaw plugins install npm:@acme/openclaw-plugin
openclaw plugins install npm:@acme/openclaw-plugin@beta
openclaw plugins install npm:@acme/openclaw-plugin@1.0.0
```

Для ознакомления с полным контрактом публикации используйте следующие страницы, а не эту
страницу в качестве справочника по публикации:

- [Публикация в ClawHub](/ru/clawhub/publishing) описывает владельцев, области,
  выпуски, проверку, валидацию пакетов и передачу пакетов.
- [Создание плагинов](/ru/plugins/building-plugins) показывает полную структуру пакета плагина
  (включая `openclaw.plugin.json`) и рабочий процесс первой публикации.
- [Манифест плагина](/ru/plugins/manifest) определяет поля нативного манифеста плагина.

Если один и тот же пакет доступен и в ClawHub, и в npm, используйте явный префикс
`clawhub:` или `npm:`, чтобы принудительно выбрать один источник.
