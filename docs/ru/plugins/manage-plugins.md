Интерфейс управления охватывает типовой рабочий процесс поиска, установки, включения и отключения. CLI дополнительно поддерживает обновление, удаление, расширенную настройку и явное управление источником установки. Полное описание команд, флагов, правил выбора источника и особых случаев см. в разделе [`openclaw plugins`](/ru/cli/plugins).

Типовой рабочий процесс CLI: найдите пакет, установите его из ClawHub, npm, git или локального пути, дождитесь автоматического перезапуска управляемого Gateway (либо перезапустите его вручную), а затем проверьте регистрации среды выполнения плагина.

## Использование интерфейса управления

Откройте **Plugins** в интерфейсе управления или перейдите по адресу `/settings/plugins` относительно настроенного базового пути интерфейса управления. Например, при базовом пути `/openclaw` используется `/openclaw/settings/plugins`. На странице есть две вкладки:

- **Installed** показывает полный локальный перечень, сгруппированный по категориям (каналы, поставщики моделей, память, инструменты). Каждая строка открывает подробное представление; через меню дополнительных действий (`…`) можно включить или отключить плагин, а для установленных извне плагинов также доступно действие **Remove**. На вкладке также перечислены настроенные [серверы MCP](/ru/cli/mcp) с такими же доступными через меню действиями включения, отключения и удаления, которые изменяют `mcp.servers` в конфигурации Gateway.
- **Discover** — это магазин: рекомендуемые плагины, включённые в OpenClaw, официальные внешние плагины и отобранная подборка коннекторов. Карточки коннекторов либо добавляют размещённый сервер MCP одним щелчком (GitHub, Notion, Linear, Sentry, Home Assistant), либо открывают поиск ClawHub с предварительно заполненным запросом. При вводе текста в поле поиска выполняется встроенный запрос к [ClawHub](https://clawhub.ai/plugins), а также добавляется раздел **From ClawHub** с количеством загрузок и значками проверки источника.

Включённые в поставку плагины не требуют установки пакета. В их меню доступно действие **Enable** или **Disable**. Например, Workboard включён в OpenClaw, но по умолчанию отключён, поэтому для его включения выберите **Enable**. Встроенные плагины нельзя удалить — их можно только отключить.

Для доступа к каталогу и поиску требуется `operator.read`. Для установки, включения, отключения, удаления и изменения серверов MCP требуется `operator.admin`. Установка из ClawHub выполняется Gateway с сохранением проверок доверия, целостности и политики установки плагинов. Когда администратор включает установленный плагин, это явное доверие также фиксируется путём добавления выбранного плагина в существующий ограничивающий список `plugins.allow`. Явная запись `plugins.deny` остаётся приоритетной, и перед включением плагина её необходимо удалить.

Для установки или удаления кода плагина требуется перезапуск Gateway. Изменения состояния включения можно применить без перезапуска, если установленный плагин и текущая среда выполнения Gateway это поддерживают; в противном случае интерфейс сообщает о необходимости перезапуска. Для коннекторов MCP с OAuth после добавления по-прежнему требуется однократно выполнить `openclaw mcp login <name>` через CLI.

Интерфейс управления не устанавливает плагины из произвольных источников npm, git или локальных путей, не обновляет их и не предоставляет расширенную настройку плагинов. Для этих операций используйте описанные ниже рабочие процессы CLI.

## Просмотр списка и поиск плагинов

```bash
openclaw plugins list
openclaw plugins list --enabled
openclaw plugins list --verbose
openclaw plugins list --json
openclaw plugins search "calendar"
```

`--json` для скриптов:

```bash
openclaw plugins list --json \
  | jq '.plugins[] | {id, enabled, format, source, dependencyStatus}'
```

`plugins list` выполняет холодную проверку перечня: она показывает, что OpenClaw может обнаружить по конфигурации, манифестам и сохранённому реестру плагинов. Она не подтверждает, что уже запущенный Gateway импортировал среду выполнения плагина. Вывод JSON содержит диагностику реестра и значение `dependencyStatus` каждого плагина (разрешаются ли объявленные `dependencies`/`optionalDependencies` на диске).

`plugins search` запрашивает в ClawHub доступные для установки пакеты плагинов и выводит для каждого результата подсказку по установке (`openclaw plugins install clawhub:<package>`).

## Включение и отключение плагинов

```bash
openclaw plugins enable <plugin-id>
openclaw plugins disable <plugin-id>
```

Переключает запись плагина в конфигурации, не затрагивая установленные файлы. Некоторые встроенные плагины (встроенные поставщики моделей и синтеза речи, встроенный плагин браузера) включены по умолчанию; для остальных после установки требуется `enable`.

## Установка плагинов

```bash
# Поиск пакетов плагинов в ClawHub.
openclaw plugins search "calendar"

# Установка из ClawHub.
openclaw plugins install clawhub:<package>
openclaw plugins install clawhub:<package>@1.2.3
openclaw plugins install clawhub:<package>@beta

# Установка из npm.
openclaw plugins install npm:<package>
openclaw plugins install npm:@scope/openclaw-plugin@1.2.3
openclaw plugins install npm:@openclaw/codex

# Установка из локального артефакта npm-pack.
openclaw plugins install npm-pack:<path.tgz>

# Установка из git или локальной рабочей копии для разработки.
openclaw plugins install git:github.com/acme/openclaw-plugin@v1.0.0
openclaw plugins install ./my-plugin
openclaw plugins install --link ./my-plugin
```

Во время перехода при запуске спецификации пакетов без префикса устанавливаются из npm, если только имя не совпадает с идентификатором встроенного или официального плагина — в таком случае OpenClaw использует соответствующую локальную или официальную копию. Для детерминированного выбора источника используйте `clawhub:`, `npm:`, `git:` или `npm-pack:`.

Используйте `--force` только для перезаписи существующей цели установки из другого источника. Для обычного обновления отслеживаемой установки из npm, ClawHub или hook-pack используйте `openclaw plugins update`; `--force` не поддерживается вместе с `--link`.

## Перезапуск и проверка

Запущенный управляемый Gateway с включённой перезагрузкой конфигурации автоматически перезапускается после установки, обновления или удаления кода плагина. Если Gateway не управляется или перезагрузка отключена, перезапустите его вручную перед проверкой активных поверхностей среды выполнения:

```bash
openclaw gateway restart
openclaw plugins inspect <plugin-id> --runtime --json
```

`inspect --runtime` загружает модуль плагина и подтверждает, что он зарегистрировал поверхности среды выполнения (инструменты, хуки, службы, методы Gateway, маршруты HTTP, принадлежащие плагину команды CLI). Обычные `inspect` и `list` выполняют только холодные проверки манифеста, конфигурации и реестра.

## Обновление плагинов

```bash
openclaw plugins update <plugin-id>
openclaw plugins update <npm-package-or-spec>
openclaw plugins update --all
openclaw plugins update <plugin-id> --dry-run
```

При передаче идентификатора плагина повторно используется его отслеживаемая спецификация установки: сохранённые dist-теги (`@beta`) и точно закреплённые версии переносятся в последующие запуски `update <plugin-id>`.

`openclaw plugins update --all` — способ массового обслуживания. Он по-прежнему учитывает обычные отслеживаемые спецификации установки, однако доверенные записи официальных плагинов OpenClaw синхронизируются с текущей целью официального каталога, а не остаются закреплёнными за устаревшим точным официальным пакетом; когда `update.channel` имеет значение `beta`, при синхронизации предпочтение отдаётся ветке бета-версий. Чтобы сохранить точную или помеченную тегом официальную спецификацию без изменений, используйте адресную команду `update <plugin-id>`.

Для установок из npm передайте явную спецификацию пакета, чтобы изменить отслеживаемую запись:

```bash
openclaw plugins update @scope/openclaw-plugin@beta
openclaw plugins update @scope/openclaw-plugin
```

Вторая команда возвращает плагин к стандартной ветке выпусков реестра, если ранее он был закреплён за точной версией или тегом.

Точные правила резервного выбора и закрепления см. в разделе [`openclaw plugins`](/ru/cli/plugins#update).

## Удаление плагинов

```bash
openclaw plugins uninstall <plugin-id> --dry-run
openclaw plugins uninstall <plugin-id>
openclaw plugins uninstall <plugin-id> --keep-files
```

При удалении удаляются запись плагина в конфигурации, сохранённая запись в индексе плагинов, записи в списках разрешений и запретов, а также связанные записи `plugins.load.paths`, если применимо. Управляемый каталог установки удаляется, если не передан параметр `--keep-files`. Запущенный управляемый Gateway автоматически перезапускается, если удаление изменяет источник плагина.

В режиме Nix (`OPENCLAW_NIX_MODE=1`) установка, обновление, удаление, включение и отключение плагинов недоступны; управляйте этими параметрами в исходном коде Nix для установки.

## Выбор источника

| Источник    | Когда использовать                                                           | Пример                                                         |
| ----------- | ----------------------------------------------------------------------------- | -------------------------------------------------------------- |
| ClawHub     | Нужны встроенный в OpenClaw поиск, сводки проверок, версии и подсказки         | `openclaw plugins install clawhub:<package>`                   |
| git         | Нужна ветка, тег или коммит из репозитория                                    | `openclaw plugins install git:github.com/<owner>/<repo>@<ref>` |
| локальный путь | Вы разрабатываете или тестируете плагин на том же компьютере               | `openclaw plugins install --link ./my-plugin`                  |
| marketplace | Вы устанавливаете совместимый с Claude плагин из marketplace                  | `openclaw plugins install <plugin> --marketplace <source>`     |
| npm pack    | Вы проверяете локальный артефакт пакета с использованием семантики установки npm | `openclaw plugins install npm-pack:<path.tgz>`                 |
| npmjs.com   | Вы уже распространяете пакеты JavaScript или вам нужны dist-теги npm либо частный реестр | `openclaw plugins install npm:@acme/openclaw-plugin`           |

Управляемые установки из локального пути должны указывать на каталоги или архивы плагинов. Отдельные файлы плагинов следует помещать в `plugins.load.paths`, а не устанавливать с помощью `plugins install`.

## Публикация плагинов

ClawHub — основная общедоступная платформа поиска плагинов OpenClaw. Публикуйте их там, если хотите, чтобы перед установкой пользователи могли найти метаданные плагина, историю версий, результаты проверки реестром и подсказки по установке.

```bash
npm i -g clawhub
clawhub login
clawhub package publish your-org/your-plugin --dry-run
clawhub package publish your-org/your-plugin
clawhub package publish your-org/your-plugin@v1.0.0
```

Перед публикацией нативные плагины npm должны содержать манифест плагина (`openclaw.plugin.json`) и метаданные `package.json`:

```json package.json
{
  "name": "@acme/openclaw-plugin",
  "version": "1.0.0",
  "type": "module",
  "openclaw": {
    "extensions": ["./dist/index.js"]
  }
}
```

```bash
npm publish --access public
openclaw plugins install npm:@acme/openclaw-plugin
openclaw plugins install npm:@acme/openclaw-plugin@beta
openclaw plugins install npm:@acme/openclaw-plugin@1.0.0
```

Для полного описания контракта публикации используйте следующие страницы, а не рассматривайте эту страницу как справочник по публикации:

- [Публикация в ClawHub](/ru/clawhub/publishing) описывает владельцев, области видимости, выпуски, проверку, валидацию пакетов и передачу пакетов.
- [Создание плагинов](/ru/plugins/building-plugins) показывает полную структуру пакета плагина (включая `openclaw.plugin.json`) и рабочий процесс первой публикации.
- [Манифест плагина](/ru/plugins/manifest) определяет поля манифеста нативного плагина.

Если один и тот же пакет доступен и в ClawHub, и в npm, используйте явный префикс `clawhub:` или `npm:`, чтобы принудительно выбрать источник.
