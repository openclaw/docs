OpenClaw может устанавливать плагины из трёх внешних экосистем: **Codex**, **Claude**
и **Cursor**. Они называются **пакетами** — наборами содержимого и метаданных,
которые OpenClaw преобразует в нативные функции, такие как Skills, хуки и инструменты MCP.

<Info>
  Пакеты — **не** то же самое, что нативные плагины OpenClaw. Нативные плагины
  выполняются внутри процесса и могут регистрировать любые возможности. Пакеты —
  это наборы содержимого с выборочным преобразованием функций и более узкой
  границей доверия.
</Info>

## Зачем нужны пакеты

Многие полезные плагины публикуются в формате Codex, Claude или Cursor. Вместо
того чтобы требовать от авторов переписывать их как нативные плагины OpenClaw,
OpenClaw обнаруживает эти форматы и преобразует поддерживаемое содержимое в набор
нативных функций. Можно установить пакет команд Claude или пакет Skills Codex
и сразу использовать его.

## Установка пакета

<Steps>
  <Step title="Установка из каталога, архива или маркетплейса">
    ```bash
    # Локальный каталог
    openclaw plugins install ./my-bundle

    # Архив
    openclaw plugins install ./my-bundle.tgz

    # Маркетплейс Claude
    openclaw plugins marketplace list <source>
    openclaw plugins install <plugin> --marketplace <source>
    ```

    `<source>` — путь к локальному маркетплейсу или репозиторию либо источник git/GitHub.

  </Step>

  <Step title="Проверка обнаружения">
    ```bash
    openclaw plugins list
    openclaw plugins inspect <id>
    ```

    Для пакетов отображается `Format: bundle`, а также значение `Bundle format:`:
    `codex`, `claude` или `cursor`.

  </Step>

  <Step title="Перезапуск и использование">
    ```bash
    openclaw gateway restart
    ```

    Преобразованные функции (Skills, хуки, инструменты MCP, настройки LSP по умолчанию) будут доступны в следующем сеансе.

  </Step>
</Steps>

## Что OpenClaw преобразует из пакетов

В настоящее время в OpenClaw выполняются не все функции пакетов. Ниже указано,
что работает, а что обнаруживается, но пока не подключено.

### Поддерживается сейчас

| Функция             | Как преобразуется                                                                                         | Применимо к         |
| ------------------- | --------------------------------------------------------------------------------------------------------- | ------------------- |
| Содержимое Skills   | Корневые каталоги Skills пакета загружаются как обычные Skills OpenClaw                                   | Все форматы         |
| Команды             | `commands/` и `.cursor/commands/` считаются корневыми каталогами Skills                                   | Claude, Cursor      |
| Пакеты хуков        | Структуры в стиле OpenClaw с `HOOK.md` и `handler.ts`                                                     | Codex               |
| Инструменты MCP     | Конфигурация MCP пакета объединяется со встроенными настройками OpenClaw; загружаются поддерживаемые серверы stdio и HTTP | Все форматы |
| Серверы LSP         | `.lsp.json` Claude и объявленные в манифесте `lspServers` объединяются со встроенными настройками LSP OpenClaw по умолчанию | Claude |
| Настройки           | `settings.json` Claude импортируется как встроенные настройки OpenClaw по умолчанию                       | Claude              |

#### Содержимое Skills

- Корневые каталоги Skills пакета загружаются как обычные корневые каталоги Skills OpenClaw.
- Корневые каталоги Claude `commands/` считаются дополнительными корневыми каталогами Skills.
- Корневые каталоги Cursor `.cursor/commands/` считаются дополнительными корневыми каталогами Skills.

Файлы команд Claude в формате Markdown и команды Cursor в формате Markdown
работают через обычный загрузчик Skills OpenClaw.

#### Пакеты хуков

Корневые каталоги хуков пакета работают **только** при использовании обычной
структуры пакета хуков OpenClaw: `HOOK.md` вместе с `handler.ts` или `handler.js`.
В настоящее время это прежде всего относится к совместимому с Codex варианту.

#### MCP для встроенного OpenClaw

- Включённые пакеты могут предоставлять конфигурацию сервера MCP.
- OpenClaw объединяет конфигурацию MCP пакета с действующими настройками
  встроенного OpenClaw в разделе `mcpServers`.
- OpenClaw предоставляет поддерживаемые инструменты MCP пакета во время
  обращений встроенного агента OpenClaw, запуская серверы stdio или подключаясь
  к серверам HTTP.
- Профили инструментов `coding` и `messaging` по умолчанию включают инструменты
  MCP пакетов; чтобы отключить их для агента или Gateway, используйте
  `tools.deny: ["bundle-mcp"]`.
- Локальные настройки встроенного агента для проекта применяются после настроек
  пакета по умолчанию, поэтому настройки рабочего пространства при необходимости
  могут переопределять записи MCP пакета.
- Каталоги инструментов MCP пакетов перед регистрацией сортируются
  детерминированно, поэтому изменения порядка в вышестоящем `listTools()` не
  приводят к постоянным изменениям блоков инструментов в кэше промптов.

##### Транспорты

Серверы MCP могут использовать транспорт stdio или HTTP.

**Stdio** запускает дочерний процесс:

```json
{
  "mcp": {
    "servers": {
      "my-server": {
        "command": "node",
        "args": ["server.js"],
        "env": { "PORT": "3000" }
      }
    }
  }
}
```

**HTTP** подключается к запущенному серверу MCP, по умолчанию используя `sse`,
если не запрошен `streamable-http`:

```json
{
  "mcp": {
    "servers": {
      "my-server": {
        "url": "http://localhost:3100/mcp",
        "transport": "streamable-http",
        "headers": {
          "Authorization": "Bearer ${MY_SECRET_TOKEN}"
        },
        "connectionTimeoutMs": 30000
      }
    }
  }
}
```

- `transport` принимает `"streamable-http"` или `"sse"`; если значение не указано, по умолчанию используется `sse`.
- `type: "http"` — нативная для CLI нисходящая структура; в конфигурации OpenClaw используйте `transport: "streamable-http"`. Команды `openclaw mcp set` и `openclaw doctor --fix` нормализуют распространённый псевдоним.
- Разрешены только схемы URL `http:` и `https:`.
- Значения `headers` поддерживают подстановку `${ENV_VAR}`.
- Запись сервера, содержащая одновременно `command` и `url`, отклоняется.
- Учётные данные в URL (информация о пользователе и параметры запроса)
  скрываются в описаниях инструментов и журналах.
- `connectionTimeoutMs` переопределяет стандартный 30-секундный тайм-аут
  подключения для транспортов stdio и HTTP. Тайм-аут запроса по умолчанию
  составляет 60 секунд и может быть переопределён параметром `requestTimeoutMs`.

##### Именование инструментов

OpenClaw регистрирует инструменты MCP пакетов с безопасными для провайдера именами
в формате `serverName__toolName`. Например, сервер с ключом `"vigil-harbor"`,
предоставляющий инструмент `memory_search`, регистрируется как
`vigil-harbor__memory_search`.

- Символы, не входящие в `A-Za-z0-9_-`, заменяются на `-`.
- Фрагменты, которые начинались бы не с буквы, получают буквенный префикс,
  поэтому числовые ключи серверов, такие как `12306`, становятся безопасными
  для провайдера префиксами инструментов.
- Длина префиксов серверов ограничена 30 символами.
- Полные имена инструментов ограничены 64 символами.
- Для пустых имён серверов используется резервное значение `mcp`.
- Совпадающие нормализованные имена различаются числовыми суффиксами.
- Итоговый порядок предоставляемых инструментов детерминирован по безопасному
  имени, что обеспечивает стабильность кэша при повторных обращениях
  встроенного агента.
- При фильтрации по профилю каждый инструмент одного сервера MCP пакета
  считается принадлежащим плагину `bundle-mcp`, поэтому списки разрешений и
  запретов профиля могут ссылаться как на отдельные предоставляемые имена
  инструментов, так и на ключ плагина `bundle-mcp`.

#### Настройки встроенного OpenClaw

Когда пакет включён, `settings.json` Claude импортируется как настройки
встроенного OpenClaw по умолчанию. Перед их применением OpenClaw удаляет ключи
переопределения оболочки:

- `shellPath`
- `shellCommandPrefix`

#### Встроенный LSP OpenClaw

- Включённые пакеты Claude могут предоставлять конфигурацию сервера LSP.
- OpenClaw загружает `.lsp.json`, а также все пути `lspServers`, объявленные
  в манифесте.
- Конфигурация LSP пакета объединяется с действующими настройками LSP
  встроенного OpenClaw по умолчанию.
- В настоящее время можно запускать только поддерживаемые серверы LSP на базе
  stdio; неподдерживаемые транспорты всё равно отображаются в
  `openclaw plugins inspect <id>`.

### Обнаруживается, но не выполняется

Следующие элементы распознаются и отображаются в диагностике, но OpenClaw
их не выполняет:

- Автоматизация Claude `agents`, `hooks/hooks.json`, `outputStyles`
- Cursor `.cursor/agents`, `.cursor/hooks.json`, `.cursor/rules`
- Метаданные Codex `.app.json`, кроме сведений о возможностях

## Форматы пакетов

<AccordionGroup>
  <Accordion title="Пакеты Codex">
    Маркеры: `.codex-plugin/plugin.json`

    Необязательное содержимое: `skills/`, `hooks/`, `.mcp.json`, `.app.json`

    Пакеты Codex лучше всего подходят для OpenClaw, когда используют корневые
    каталоги Skills и каталоги пакетов хуков в стиле OpenClaw
    (`HOOK.md` + `handler.ts`).

  </Accordion>

  <Accordion title="Пакеты Claude">
    Два режима обнаружения:

    - **На основе манифеста:** `.claude-plugin/plugin.json`
    - **Без манифеста:** стандартная структура Claude (`skills/`, `commands/`, `agents/`, `hooks/`, `.mcp.json`, `.lsp.json`, `settings.json`)

    Особенности Claude:

    - `commands/` считается содержимым Skills
    - `settings.json` импортируется в настройки встроенного OpenClaw (ключи переопределения оболочки удаляются)
    - `.mcp.json` предоставляет поддерживаемые инструменты stdio встроенному OpenClaw
    - `.lsp.json` вместе с путями `lspServers`, объявленными в манифесте, загружается в настройки LSP встроенного OpenClaw по умолчанию
    - `hooks/hooks.json` обнаруживается, но не выполняется
    - Пользовательские пути компонентов в манифесте дополняют стандартные пути, а не заменяют их

  </Accordion>

  <Accordion title="Пакеты Cursor">
    Маркеры: `.cursor-plugin/plugin.json`

    Необязательное содержимое: `skills/`, `.cursor/commands/`, `.cursor/agents/`, `.cursor/rules/`, `.cursor/hooks.json`, `.mcp.json`

    - `.cursor/commands/` считается содержимым Skills
    - `.cursor/rules/`, `.cursor/agents/` и `.cursor/hooks.json` только обнаруживаются

  </Accordion>
</AccordionGroup>

## Приоритет обнаружения

Сначала OpenClaw проверяет формат нативного плагина:

1. `openclaw.plugin.json` или действительный `package.json` с `openclaw.extensions` — считается **нативным плагином**
2. Маркеры пакетов (`.codex-plugin/`, `.claude-plugin/` или стандартная структура Claude/Cursor) — считается **пакетом**

Если каталог содержит оба варианта, OpenClaw использует нативный путь. Это
предотвращает частичную установку пакетов с двумя форматами в качестве пакетов.

## Зависимости среды выполнения и очистка

- Для сторонних совместимых пакетов при запуске не выполняется восстановление
  через `npm install`. Их следует устанавливать с помощью
  `openclaw plugins install`; всё необходимое должно поставляться в каталоге
  установленного плагина.
- Поставляемые плагины OpenClaw либо входят в ядро в облегчённом виде, либо
  загружаются через установщик плагинов. При запуске Gateway менеджер пакетов
  для них никогда не запускается.
- `openclaw doctor --fix` удаляет устаревшие записи об установленных локальных
  поставляемых плагинах и может восстанавливать загружаемые плагины,
  отсутствующие в локальном индексе плагинов, если конфигурация всё ещё
  ссылается на них.

## Безопасность

Граница доверия пакетов уже, чем у нативных плагинов:

- OpenClaw **не** загружает произвольные модули среды выполнения пакетов внутри процесса.
- Пути Skills и пакетов хуков должны оставаться внутри корневого каталога плагина
  с проверкой границ.
- Файлы настроек читаются с такими же проверками границ.
- Поддерживаемые серверы MCP stdio могут запускаться как подпроцессы.

Благодаря этому пакеты по умолчанию безопаснее, но сторонние пакеты всё равно
следует считать доверенным содержимым в отношении предоставляемых ими функций.

## Устранение неполадок

<AccordionGroup>
  <Accordion title="Пакет обнаружен, но возможности не работают">
    Выполните `openclaw plugins inspect <id>`. Если возможность указана, но
    помечена как неподключённая, это ограничение продукта, а не ошибка установки.
  </Accordion>

  <Accordion title="Файлы команд Claude не отображаются">
    Убедитесь, что пакет включён, а файлы Markdown находятся в обнаруженном
    корневом каталоге `commands/` или `skills/`.
  </Accordion>

  <Accordion title="Настройки Claude не применяются">
    Поддерживаются только настройки встроенного OpenClaw из `settings.json`.
    OpenClaw не рассматривает настройки пакета как необработанные исправления
    конфигурации.
  </Accordion>

  <Accordion title="Хуки Claude не выполняются">
    `hooks/hooks.json` только обнаруживается. Если нужны выполняемые хуки,
    используйте структуру пакета хуков OpenClaw или поставляйте нативный плагин.
  </Accordion>
</AccordionGroup>
