Computer Use — нативный для Codex MCP-плагин для локального управления рабочим столом. OpenClaw
не включает приложение для рабочего стола в свой состав, не выполняет действия на рабочем столе самостоятельно и не обходит
разрешения Codex. Встроенный плагин `codex` лишь подготавливает app-server Codex:
включает поддержку плагинов Codex, находит или устанавливает настроенный плагин Computer Use,
проверяет доступность MCP-сервера `computer-use`, а затем позволяет
Codex управлять нативными вызовами инструментов MCP во время ходов в режиме Codex.

Используйте эту страницу, если OpenClaw уже использует нативную среду Codex. О настройке
самой среды см. в разделе [Среда Codex](/ru/plugins/codex-harness).

Это отличается от встроенного в OpenClaw [инструмента управления компьютером на базе Node](/ru/nodes/computer-use). Используйте встроенный инструмент, когда один и тот же контракт агента должен управлять сопряжённым Mac независимо от того, работает ли агент на Gateway или другом узле. Используйте Codex Computer Use, когда app-server Codex должен управлять локальной установкой MCP, разрешениями и нативными вызовами инструментов.

## OpenClaw.app и Peekaboo

Интеграция Peekaboo в OpenClaw.app отделена от Codex Computer Use. Приложение
macOS может предоставлять сокет PeekabooBridge, чтобы CLI `peekaboo` мог повторно использовать
локальные разрешения приложения на Accessibility и Screen Recording для собственных
инструментов автоматизации Peekaboo. Этот мост не устанавливает и не проксирует Codex Computer Use, а
Codex Computer Use не выполняет вызовы через сокет PeekabooBridge.

Используйте [мост Peekaboo](/ru/platforms/mac/peekaboo), если хотите, чтобы OpenClaw.app
выступало хостом автоматизации Peekaboo CLI с учётом разрешений. Используйте эту страницу, если
агент OpenClaw в режиме Codex должен иметь доступ к нативному MCP-плагину Codex
`computer-use` до начала хода.

## Приложение iOS

Приложение iOS отделено от Codex Computer Use. Оно не устанавливает и не проксирует
MCP-сервер Codex `computer-use` и не является бэкендом управления рабочим столом.
Вместо этого приложение iOS подключается как узел OpenClaw и предоставляет мобильные
возможности через команды узла, такие как `canvas.*`, `camera.*`, `screen.*`,
`location.*` и `talk.*`.

Используйте [iOS](/ru/platforms/ios), если хотите, чтобы агент управлял узлом iPhone
через Gateway. Используйте эту страницу, если агент в режиме Codex должен управлять
локальным рабочим столом macOS через нативный плагин Computer Use для Codex.

## Прямое подключение MCP cua-driver

Codex Computer Use — не единственный способ предоставить управление рабочим столом. Если вы хотите,
чтобы среды выполнения под управлением OpenClaw вызывали драйвер TryCua напрямую, используйте вышестоящий
сервер `cua-driver mcp` через реестр MCP OpenClaw вместо
специального потока маркетплейса Codex.

После установки `cua-driver` запросите у него команду OpenClaw:

```bash
cua-driver mcp-config --client openclaw
```

либо зарегистрируйте stdio-сервер напрямую:

```bash
openclaw mcp set cua-driver '{"command":"cua-driver","args":["mcp"]}'
```

Этот путь сохраняет поверхность инструментов вышестоящего MCP без изменений, включая схемы драйвера
и структурированные ответы MCP. Используйте его, если хотите, чтобы драйвер CUA
был доступен как обычный MCP-сервер OpenClaw. Используйте настройку Codex Computer Use на
этой странице, если app-server Codex должен управлять установкой плагина, перезагрузкой MCP-серверов
и нативными вызовами инструментов во время ходов в режиме Codex.

Драйвер CUA предназначен только для macOS и по-прежнему требует локальных разрешений macOS,
которые запрашивает его приложение, например Accessibility и Screen Recording. OpenClaw не
устанавливает `cua-driver`, не предоставляет эти разрешения и не обходит модель безопасности
вышестоящего драйвера.

## Быстрая настройка

Задайте `plugins.entries.codex.config.computerUse`, если Computer Use должен быть
доступен для ходов в режиме Codex до запуска потока. `autoInstall: true` включает
Computer Use и позволяет OpenClaw установить или повторно включить его перед ходом:

```json5
{
  plugins: {
    entries: {
      codex: {
        enabled: true,
        config: {
          computerUse: {
            autoInstall: true,
          },
        },
      },
    },
  },
  agents: {
    defaults: {
      model: "openai/gpt-5.6-sol",
    },
  },
}
```

С этой конфигурацией OpenClaw проверяет app-server Codex перед каждым ходом в режиме
Codex. Если Computer Use отсутствует, но app-server Codex уже обнаружил
маркетплейс, из которого возможна установка, OpenClaw запрашивает у app-server Codex установку или
повторное включение плагина и перезагрузку MCP-серверов. В macOS, если ни один подходящий
маркетплейс не зарегистрирован и существует стандартный пакет приложения для рабочего стола, OpenClaw
также пытается зарегистрировать встроенный маркетплейс Codex из
`/Applications/ChatGPT.app/Contents/Resources/plugins/openai-bundled`, сохраняя
`/Applications/Codex.app/Contents/Resources/plugins/openai-bundled`
как резервный вариант для устаревших автономных установок. Если после этого MCP-сервер
всё ещё недоступен, ход завершается ошибкой до запуска потока.

После изменения конфигурации Computer Use используйте `/new` или `/reset` в соответствующем
чате перед тестированием, если существующий поток Codex уже запущен.

В macOS управляемый запуск Computer Use предпочитает бинарный файл приложения для рабочего стола по пути
`/Applications/ChatGPT.app/Contents/Resources/codex`, а затем
переходит к `/Applications/Codex.app/Contents/Resources/codex` для устаревших
автономных установок. Это также относится к разовым командам состояния и установки Computer Use,
которые запускают собственный клиент. Таким образом управление рабочим столом остаётся в пакете приложения,
владеющем локальными разрешениями macOS. Если приложение для рабочего стола не установлено,
OpenClaw переходит к управляемому бинарному файлу Codex, установленному рядом с
плагином. Обычные управляемые ходы Codex с изолированным домашним каталогом агента по умолчанию предпочитают
сначала этот закреплённый пакет, чтобы более старое приложение для рабочего стола не могло перекрыть актуальную поддержку
моделей. Домашние каталоги пользователя по-прежнему отдают приоритет приложению для рабочего стола, поскольку могут загружать нативное
состояние Computer Use. Изолированный домашний каталог агента, в действующей конфигурации Codex которого включён
Computer Use, также продолжает отдавать приоритет приложению для рабочего стола. Явная конфигурация
`appServer.command` или `OPENCLAW_CODEX_APP_SERVER_BIN` по-прежнему переопределяет
этот управляемый выбор.

OpenClaw последовательно выполняет чтение нативной конфигурации Codex и установку Computer Use
в рамках одного запущенного Gateway. Отдельный процесс Codex или другой Gateway
не входит в эту область синхронизации. После изменения нативной конфигурации плагина Codex вне
Gateway перезапустите Gateway и начните новый чат, прежде чем полагаться на новый
выбор.

## Команды

Используйте команды `/codex computer-use` на любой поверхности чата, где
доступна поверхность команд плагина `codex`. Это команды чата/среды выполнения
OpenClaw, а не подкоманды CLI `openclaw codex ...`:

```text
/codex computer-use status
/codex computer-use install
/codex computer-use install --source <marketplace-source>
/codex computer-use install --marketplace-path <path>
/codex computer-use install --marketplace <name>
```

`status` — действие по умолчанию, доступное только для чтения: оно не добавляет источники
маркетплейсов, не устанавливает плагины и не включает поддержку плагинов Codex. Если ни одна конфигурация не включает
Computer Use, `status` может сообщать, что он отключён, даже после разовой команды
установки.

`install` включает поддержку плагинов app-server Codex, при необходимости добавляет
настроенный источник маркетплейса, устанавливает или повторно включает настроенный плагин
через app-server Codex, перезагружает MCP-серверы и проверяет, что MCP-сервер
предоставляет инструменты. Поскольку установка изменяет доверенные ресурсы хоста,
запускать `install` может только владелец или клиент Gateway `operator.admin`. Другие
авторизованные отправители могут продолжать использовать команду `status`, доступную только для чтения,
в том числе с переопределениями.

Более старые выпуски принимали разовые переопределения идентификации `--plugin`, `--server` и `--mcp-server`.
Вместо этого настройте `computerUse.pluginName` и
`computerUse.mcpServerName` на постоянной основе. При использовании устаревшего флага идентификации
команда указывает точный параметр, который нужно сохранить, и повторяет
запрошенное действие вместе со всеми поддерживаемыми флагами маркетплейса в рекомендациях по миграции.

## Варианты маркетплейса

OpenClaw использует тот же API app-server, который предоставляет сам Codex.
Поля маркетплейса определяют, где Codex должен искать `computer-use`.

| Поле                 | Когда использовать                                               | Поддержка установки                                       |
| -------------------- | --------------------------------------------------------------- | -------------------------------------------------------- |
| Без поля маркетплейса | Вы хотите, чтобы app-server Codex использовал уже известные ему маркетплейсы. | Да, когда app-server возвращает локальный маркетплейс.    |
| `marketplaceSource`  | У вас есть источник маркетплейса Codex, который app-server может добавить. | Да, для явного `/codex computer-use install`.         |
| `marketplacePath`    | Вы уже знаете локальный путь к файлу маркетплейса на хосте.      | Да, для явной установки и автоматической установки при запуске хода. |
| `marketplaceName`    | Вы хотите выбрать по имени один уже зарегистрированный маркетплейс. | Да, только если у выбранного маркетплейса есть локальный путь. |

Свежим домашним каталогам Codex может потребоваться немного времени для первоначального заполнения официальных
маркетплейсов. Во время установки OpenClaw опрашивает `plugin/list` в течение не более
`marketplaceDiscoveryTimeoutMs` миллисекунд (по умолчанию 60 секунд).

Если Computer Use присутствует в нескольких известных маркетплейсах, OpenClaw предпочитает
`openai-bundled`, затем `openai-curated`, а затем `local`. Неизвестные неоднозначные
совпадения приводят к безопасному отказу с запросом задать `marketplaceName` или
`marketplacePath`.

## Встроенный маркетплейс macOS

Текущие сборки приложения ChatGPT для рабочего стола включают Computer Use по следующему пути; устаревшие автономные
сборки Codex для рабочего стола используют ту же структуру в `Codex.app`:

```text
/Applications/ChatGPT.app/Contents/Resources/plugins/openai-bundled/plugins/computer-use
/Applications/Codex.app/Contents/Resources/plugins/openai-bundled/plugins/computer-use
```

Если `computerUse.autoInstall` имеет значение true и маркетплейс, содержащий
`computer-use`, не зарегистрирован, OpenClaw пытается добавить первый существующий стандартный
корень встроенного маркетплейса:

```text
/Applications/ChatGPT.app/Contents/Resources/plugins/openai-bundled
/Applications/Codex.app/Contents/Resources/plugins/openai-bundled
```

Его также можно явно зарегистрировать из оболочки с помощью Codex:

```bash
codex plugin marketplace add /Applications/ChatGPT.app/Contents/Resources/plugins/openai-bundled
```

Если вы используете нестандартный путь к приложению Codex, один раз выполните `/codex computer-use install
--source <marketplace-root>` либо задайте для `computerUse.marketplacePath` локальный
путь к файлу маркетплейса. Используйте `--marketplace-path`, только если у вас есть
путь к JSON-файлу маркетплейса, а не корень встроенного маркетплейса.

### Общий кеш плагинов

Значение `pluginCacheMode: "independent"` по умолчанию оставляет каждый домашний каталог Codex и его
кеш плагинов неуправляемыми. Задайте `pluginCacheMode: "shared"`, чтобы скопировать встроенный
плагин Computer Use в доступный для обнаружения кеш плагинов активного домашнего каталога Codex
до запуска app-server. Общий режим сохраняет более старые кешированные версии, поскольку
работающие клиенты Codex всё ещё могут ссылаться на каталоги плагинов своих версий; при
неудачной замене копии активный кеш также сохраняется. Явная конфигурация
`marketplaceName` или `marketplacePath` отключает эту
сверку, чтобы OpenClaw не переопределял данный выбор.

## Ограничение удалённого каталога

App-server Codex может выводить список и читать записи, доступные только в удалённом каталоге, но в настоящее время
не поддерживает удалённый `plugin/install`. Это означает, что `marketplaceName`
может выбрать маркетплейс, доступный только удалённо, для проверки состояния, но для установки и
повторного включения по-прежнему требуется локальный маркетплейс через `marketplaceSource` или
`marketplacePath`.

Если в состоянии указано, что плагин доступен в удалённом маркетплейсе Codex, но
удалённая установка не поддерживается, выполните установку с локальным источником или путём:

```text
/codex computer-use install --source <marketplace-source>
/codex computer-use install --marketplace-path <path>
```

## Справочник по конфигурации

| Поле                           | Значение по умолчанию | Назначение                                                                        |
| ------------------------------- | -------------- | ------------------------------------------------------------------------------ |
| `enabled`                       | inferred       | Требовать управление компьютером. По умолчанию принимает значение true, если задано другое поле управления компьютером. |
| `autoInstall`                   | false          | Устанавливать или повторно включать из уже обнаруженных маркетплейсов в начале хода.       |
| `marketplaceDiscoveryTimeoutMs` | 60000          | Время ожидания обнаружения маркетплейса сервером приложений Codex при установке.             |
| `liveTestTimeoutMs`             | 60000          | Время ожидания временного потока проверки готовности и запросов на его очистку.           |
| `toolCallTimeoutMs`             | 60000          | Время ожидания вызова инструмента готовности управления компьютером `list_apps`.                  |
| `healthCheckEnabled`            | false          | Периодически проверять готовность, пока активен владеющий клиент сервера приложений.    |
| `healthCheckIntervalMinutes`    | 60             | Периодичность проверок; допустимые значения: 30, 60, 120 или 240 минут.                |
| `pluginCacheMode`               | `independent`  | Использовать `shared` для обновления кеша домашнего каталога Codex из встроенного настольного плагина.  |
| `strictReadiness`               | false          | Останавливать запуск при неудачной активной проверке вместо продолжения с предупреждением.      |
| `autoRepair`                    | false          | Завершать устаревшие дочерние процессы MCP управления компьютером в соответствующей области и один раз повторять неудачную проверку.     |
| `marketplaceSource`             | не задано          | Строка источника, передаваемая в `marketplace/add` сервера приложений Codex.                    |
| `marketplacePath`               | не задано          | Путь к локальному файлу маркетплейса Codex, содержащему плагин.                       |
| `marketplaceName`               | не задано          | Имя зарегистрированного маркетплейса Codex для выбора.                                   |
| `pluginName`                    | `computer-use` | Имя плагина в маркетплейсе Codex.                                                 |
| `mcpServerName`                 | `computer-use` | Имя сервера MCP, предоставляемого установленным плагином.                               |

Автоматическая установка в начале хода намеренно отклоняет настроенные значения `marketplaceSource`.
Добавление нового источника — это явная операция настройки, поэтому один раз используйте
`/codex computer-use install --source <marketplace-source>`, а затем разрешите
`autoInstall` выполнять последующие повторные включения из обнаруженных локальных маркетплейсов.
Автоматическая установка в начале хода может использовать настроенный `marketplacePath`, поскольку это
уже локальный путь на хосте.

Для каждого поля также можно задать переопределение переменной окружения, которое проверяется, если
соответствующий ключ конфигурации не задан:

| Поле                           | Переменная окружения                                                        |
| ------------------------------- | -------------------------------------------------------------- |
| `enabled`                       | `OPENCLAW_CODEX_COMPUTER_USE`                                  |
| `autoInstall`                   | `OPENCLAW_CODEX_COMPUTER_USE_AUTO_INSTALL`                     |
| `marketplaceDiscoveryTimeoutMs` | `OPENCLAW_CODEX_COMPUTER_USE_MARKETPLACE_DISCOVERY_TIMEOUT_MS` |
| `liveTestTimeoutMs`             | `OPENCLAW_CODEX_COMPUTER_USE_LIVE_TEST_TIMEOUT_MS`             |
| `toolCallTimeoutMs`             | `OPENCLAW_CODEX_COMPUTER_USE_TOOL_CALL_TIMEOUT_MS`             |
| `healthCheckEnabled`            | `OPENCLAW_CODEX_COMPUTER_USE_HEALTH_CHECK_ENABLED`             |
| `healthCheckIntervalMinutes`    | `OPENCLAW_CODEX_COMPUTER_USE_HEALTH_CHECK_INTERVAL_MINUTES`    |
| `pluginCacheMode`               | `OPENCLAW_CODEX_COMPUTER_USE_PLUGIN_CACHE_MODE`                |
| `strictReadiness`               | `OPENCLAW_CODEX_COMPUTER_USE_STRICT_READINESS`                 |
| `autoRepair`                    | `OPENCLAW_CODEX_COMPUTER_USE_AUTO_REPAIR`                      |
| `marketplaceSource`             | `OPENCLAW_CODEX_COMPUTER_USE_MARKETPLACE_SOURCE`               |
| `marketplacePath`               | `OPENCLAW_CODEX_COMPUTER_USE_MARKETPLACE_PATH`                 |
| `marketplaceName`               | `OPENCLAW_CODEX_COMPUTER_USE_MARKETPLACE_NAME`                 |
| `pluginName`                    | `OPENCLAW_CODEX_COMPUTER_USE_PLUGIN_NAME`                      |
| `mcpServerName`                 | `OPENCLAW_CODEX_COMPUTER_USE_MCP_SERVER_NAME`                  |

## Что проверяет OpenClaw

OpenClaw регистрирует внутри системы стабильную причину состояния настройки и форматирует
пользовательский статус для чата:

| Причина                       | Значение                                                | Следующий шаг                                     |
| ---------------------------- | ------------------------------------------------------ | --------------------------------------------- |
| `disabled`                   | `computerUse.enabled` разрешилось в false.               | Задайте `enabled` или другое поле управления компьютером.  |
| `marketplace_missing`        | Подходящий маркетплейс недоступен.                 | Настройте источник, путь или имя маркетплейса.  |
| `plugin_not_installed`       | Маркетплейс существует, но плагин не установлен.   | Запустите установку или включите `autoInstall`.          |
| `plugin_disabled`            | Плагин установлен, но отключен в конфигурации Codex.      | Запустите установку, чтобы повторно включить его.                  |
| `remote_install_unsupported` | Выбранный маркетплейс доступен только удалённо.                   | Используйте `marketplaceSource` или `marketplacePath`. |
| `mcp_missing`                | Плагин включён, но сервер MCP недоступен.  | Проверьте управление компьютером в Codex и разрешения ОС.  |
| `ready`                      | Плагин и инструменты MCP доступны.                    | Начните ход в режиме Codex.                    |
| `check_failed`               | Во время проверки статуса произошёл сбой запроса к серверу приложений Codex. | Проверьте подключение к серверу приложений и журналы.       |
| `auto_install_blocked`       | Для настройки в начале хода потребуется добавить новый источник.       | Сначала выполните явную установку.                   |

Вывод в чате содержит состояние плагина, состояние сервера MCP, маркетплейс,
инструменты, если они доступны, и конкретное сообщение о неудачном этапе настройки.

## Разрешения macOS

Управление компьютером поддерживается только в macOS. Серверу MCP, принадлежащему Codex, могут потребоваться локальные
разрешения ОС, прежде чем он сможет проверять приложения или управлять ими. Если OpenClaw сообщает, что управление
компьютером установлено, но сервер MCP недоступен, сначала проверьте настройку
управления компьютером на стороне Codex:

- Сервер приложений Codex работает на том же хосте, где должно осуществляться управление рабочим столом.
- Плагин управления компьютером включён в конфигурации Codex.
- Сервер MCP `computer-use` отображается в статусе MCP сервера приложений Codex.
- macOS предоставила необходимые разрешения приложению управления рабочим столом.
- Текущий сеанс хоста имеет доступ к управляемому рабочему столу.

OpenClaw намеренно прекращает работу при ошибке, когда `computerUse.enabled` имеет значение true. Ход
в режиме Codex не должен незаметно продолжаться без нативных инструментов рабочего стола,
которые требуются конфигурацией.

## Устранение неполадок

**Статус сообщает, что плагин не установлен.** Запустите `/codex computer-use install`. Если
маркетплейс не обнаружен, передайте `--source` или `--marketplace-path`.

**Статус сообщает, что плагин установлен, но отключён.** Снова запустите `/codex computer-use install`.
При установке сервер приложений Codex записывает конфигурацию плагина обратно с включённым состоянием.

**Статус сообщает, что удалённая установка не поддерживается.** Используйте локальный источник
или путь маркетплейса. Записи каталога, доступные только удалённо, можно просматривать, но нельзя
устанавливать через текущий API сервера приложений.

**Статус сообщает, что сервер MCP недоступен.** Повторно запустите установку один раз, чтобы серверы MCP
перезагрузились. Если сервер остаётся недоступным, исправьте приложение управления компьютером Codex,
статус MCP сервера приложений Codex или разрешения macOS.

**Проверка статуса или готовности завершается по времени ожидания на `computer-use.list_apps`.** Плагин и
сервер MCP присутствуют, но локальный мост управления компьютером не ответил.
Завершите или перезапустите управление компьютером Codex, при необходимости перезапустите Codex Desktop, затем
повторите попытку в новом сеансе OpenClaw. Если ранее на хосте управление компьютером
выполнялось через старый управляемый сервер приложений Codex, обновите установленный плагин из
встроенного маркетплейса настольного приложения (для автономных установок
настольного приложения Codex используйте путь `Codex.app`):

```text
/codex computer-use install --source /Applications/ChatGPT.app/Contents/Resources/plugins/openai-bundled
```

**Инструмент управления компьютером сообщает `Native hook relay unavailable`.**
Нативному перехватчику инструментов Codex не удалось связаться с активным ретранслятором OpenClaw через
локальный мост или резервный Gateway. Запустите новый сеанс OpenClaw с `/new`
или `/reset`. Если это сработает один раз, а затем снова завершится ошибкой при последующем вызове инструмента,
`/new` очищает только текущую попытку; перезапустите сервер приложений Codex или
Gateway OpenClaw, чтобы удалить старые потоки и регистрации перехватчиков, затем
повторите попытку в новом сеансе.

**Автоматическая установка в начале хода отклоняет источник.** Это намеренное поведение. Сначала добавьте
источник с помощью явного `/codex computer-use install --source
<marketplace-source>`, после чего последующая автоматическая установка в начале хода сможет использовать
обнаруженный локальный маркетплейс.
