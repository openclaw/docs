يوفّر OpenClaw أربع قنوات تحديث:

- **المستقرة**: وسم توزيع npm‏ `latest`. موصى بها لمعظم المستخدمين.
- **المستقرة الممتدة**: وسم توزيع npm‏ `extended-stable`. قناة حزم جديدة كليًا
  لشهر دعم سابق. وهي مخصّصة للحزم فقط، ولا يتم التثبيت فيها إلا في الواجهة
  الأمامية. يتلقى الاختيار المحفوظ تلميحات تحديث للقراءة فقط عند تمكين
  `update.checkOnStart`، لكنه لا يطبّقها تلقائيًا مطلقًا.
- **التجريبية**: وسم توزيع npm‏ `beta`. تعود إلى `latest` عندما يكون `beta`
  مفقودًا أو أقدم من الإصدار المستقر الحالي.
- **التطويرية**: الرأس المتحرك للفرع `main` ‏(git). وسم توزيع npm‏ `dev` عند
  نشره. الفرع `main` مخصّص للتجارب والتطوير النشط؛ وقد يحتوي على ميزات غير
  مكتملة أو تغييرات كاسرة. لا تشغّله لبوابات Gateway الإنتاجية.

عادةً ما تُطرح البُنى المستقرة أولًا في قناة **التجريبية**، وتخضع للتحقق
هناك، ثم تُرقّى إلى **`latest`** من دون زيادة رقم الإصدار. ويمكن للمشرفين
أيضًا النشر مباشرةً إلى `latest`. تُعد أوسمة التوزيع مصدر الحقيقة لعمليات
تثبيت npm.

## التبديل بين القنوات

```bash
openclaw update --channel stable
openclaw update --channel extended-stable
openclaw update --channel beta
openclaw update --channel dev
```

يحفظ `--channel` الاختيار في `update.channel` ضمن الإعدادات ويتحكم في مساري
التثبيت كليهما:

| القناة            | عمليات تثبيت npm/الحزم                                                                                                                                                                             | عمليات تثبيت git                                                                                                                                                                         |
| ----------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `stable`          | وسم التوزيع `latest`                                                                                                                                                                               | أحدث وسم git مستقر (يستبعد `-alpha.N` و`-beta.N` و`-rc.N` و`-dev.N` و`-next.N` و`-preview.N` و`-canary.N` و`-nightly.N` وغيرها من لواحق الإصدارات التمهيدية المسماة)                    |
| `extended-stable` | يحلّ محدد npm العام `extended-stable`، ويتحقق من الحزمة المحددة بدقة، ويثبّت ذلك الإصدار المحدد بعينه. يفشل بإغلاق آمن من دون عودة إلى `latest` أو `beta` أو `dev`.                              | غير مدعوم: يترك OpenClaw نسخة العمل دون تغيير ويطلب منك استخدام تثبيت عبر الحزم                                                                                                          |
| `beta`            | وسم التوزيع `beta`، مع العودة إلى `latest` عندما يكون `beta` مفقودًا أو أقدم                                                                                                                       | أحدث وسم git تجريبي، مع العودة إلى أحدث وسم git مستقر عندما تكون النسخة التجريبية مفقودة أو أقدم                                                                                         |
| `dev`             | وسم التوزيع `dev` (نادر؛ يستخدم معظم مستخدمي التطوير عمليات تثبيت git)                                                                                                                             | يجلب التغييرات، ويعيد تأسيس نسخة العمل على فرع `main` المنبع، ويبني، ويعيد تثبيت CLI العام                                                                                                |

في عمليات تثبيت git لقناة `dev`، تكون نسخة العمل الافتراضية في `~/openclaw`
(أو `$OPENCLAW_HOME/openclaw` عند تعيين `OPENCLAW_HOME`)؛ ويمكن تجاوز ذلك
باستخدام `OPENCLAW_GIT_DIR`.

<Tip>
للإبقاء على القناتين المستقرة والتطويرية بالتوازي، استخدم نسختي عمل منفصلتين ووجّه كل Gateway إلى نسخته الخاصة.
</Tip>

## استهداف إصدار أو وسم لمرة واحدة

استخدم `--tag` لاستهداف وسم توزيع أو إصدار أو مواصفة حزمة محددة في تحديث
واحد **من دون** تغيير القناة المحفوظة:

```bash
# تثبيت إصدار محدد
openclaw update --tag 2026.4.1-beta.1

# التثبيت من وسم التوزيع التجريبي (لمرة واحدة، لا يُحفظ)
openclaw update --tag beta

# التبديل إلى نسخة عمل GitHub main المتحركة (دائم)
openclaw update --channel dev

# تثبيت مواصفة حزمة npm محددة
openclaw update --tag openclaw@2026.4.1-beta.1

# التثبيت من GitHub main مرة واحدة من دون حفظ القناة
openclaw update --tag main
```

ملاحظات:

- ينطبق `--tag` على **عمليات تثبيت الحزم (npm) فقط**؛ وتتجاهله عمليات تثبيت git.
- لا يُحفظ الوسم؛ إذ يستخدم التحديث التالي عبر `openclaw update` القناة
  المضبوطة.
- يربط `--tag main` بالمواصفة المتوافقة مع npm‏
  `github:openclaw/openclaw#main` لذلك التشغيل الواحد. ولتثبيت متحرك دائم
  للفرع `main`، استخدم `openclaw update --channel dev` (تتحول عمليات تثبيت
  الحزم إلى نسخة عمل git) أو أعد التثبيت باستخدام طريقة git الخاصة بأداة
  التثبيت:
  `curl -fsSL https://openclaw.ai/install.sh | bash -s -- --install-method git --version main`.
  يرفض مسار تثبيت npm استهداف مصادر GitHub/git رفضًا صريحًا ويوجهك بدلًا من
  ذلك إلى طريقة git.
- الحماية من الرجوع إلى إصدار أقدم: إذا كان الإصدار المستهدف أقدم من الإصدار
  الحالي، يطلب OpenClaw التأكيد (يمكن تخطيه باستخدام `--yes`).
- تستخدم القناة المستقرة الممتدة دائمًا هدف الحزمة الدقيق الذي تم التحقق منه.
  وهي ليست اسمًا بديلًا لمرة واحدة عن `--tag extended-stable`، ولا يمكن دمج
  `--tag` مع قناة مستقرة ممتدة فعّالة.
- يختلف `--channel beta` عن `--tag beta`: يمكن لمسار القناة العودة إلى
  المستقرة/`latest` عندما تكون النسخة التجريبية مفقودة أو أقدم، بينما يستهدف
  `--tag beta` دائمًا وسم التوزيع `beta` الخام لذلك التشغيل الواحد.

## التشغيل التجريبي

عاين ما سيفعله `openclaw update` من دون إجراء تغييرات:

```bash
openclaw update --dry-run
openclaw update --channel beta --dry-run
openclaw update --tag 2026.4.1-beta.1 --dry-run
openclaw update --dry-run --json
```

يعرض التشغيل التجريبي القناة الفعّالة، والإصدار المستهدف، والإجراءات المخططة،
وما إذا كان تأكيد الرجوع إلى إصدار أقدم مطلوبًا.

## Plugins والقنوات

تؤدي أيضًا عملية التبديل بين القنوات باستخدام `openclaw update` إلى مزامنة
مصادر Plugins:

- تعيد `dev` تحويل Plugins المثبتة التي لها نظير مضمّن إلى مصدرها المضمّن
  (نسخة عمل git).
- تستعيد `stable` و`beta` حزم Plugins المثبتة عبر npm أو ClawHub.
- تحلّ `extended-stable`‏ Plugins الرسمية المؤهلة من npm ذات النية
  المجردة/الافتراضية أو `latest` إلى إصدار النواة المثبت بدقة. ولا تستعلم عن
  أوسمة Plugins‏ `@extended-stable` في وقت التشغيل.
- تُحدّث Plugins المثبتة عبر npm بعد اكتمال تحديث النواة.

## التحقق من الحالة الحالية

```bash
openclaw update status
```

يعرض القناة النشطة (مع المصدر الذي حددها: الإعدادات أو وسم git أو فرع git أو
الإصدار المثبت أو القيمة الافتراضية)، ونوع التثبيت (git أو حزمة)، والإصدار
الحالي، ومدى توفر تحديث.

## أفضل ممارسات وضع الوسوم

- ضع وسومًا على الإصدارات التي تريد أن تستقر عليها نسخ عمل git:
  `vYYYY.M.PATCH` للمستقرة، و`vYYYY.M.PATCH-beta.N` للتجريبية. لا تُعد لواحق
  الإصدارات التمهيدية المسماة مثل `-alpha.N` و`-rc.N` و`-next.N` أهدافًا
  مستقرة أو تجريبية.
- ما تزال وسوم الإصدارات المستقرة الرقمية القديمة مثل `vYYYY.M.PATCH-1`
  و`v1.0.1-1` معروفة كوسوم git مستقرة لأغراض التوافق.
- يُتعرف أيضًا على `vYYYY.M.PATCH.beta.N` (المفصول بالنقاط) لأغراض التوافق؛
  ويُفضّل استخدام `-beta.N`.
- حافظ على ثبات الوسوم: لا تنقل وسمًا أو تعِد استخدامه مطلقًا.
- تظل أوسمة توزيع npm مصدر الحقيقة لعمليات تثبيت npm:
  - `latest` -> المستقرة
  - `extended-stable` -> إصدار حزمة شهر الدعم السابق
  - `beta` -> بنية مرشحة أو بنية مستقرة تُطرح أولًا في القناة التجريبية
  - `dev` -> لقطة من `main` (اختيارية)

## توفر تطبيق macOS

قد **لا** تتضمن البُنى التجريبية والتطويرية إصدارًا من تطبيق macOS. لا بأس
بذلك:

- لا يزال من الممكن نشر وسم git ووسم توزيع npm كلٌّ على حدة.
- اذكر "لا توجد بنية macOS لهذه النسخة التجريبية" في ملاحظات الإصدار أو سجل
  التغييرات.
