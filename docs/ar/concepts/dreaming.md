# Dreaming

يُعد Dreaming نظام دمج ذاكرة يعمل في الخلفية داخل `memory-core`.
ويساعد OpenClaw على نقل الإشارات القوية قصيرة الأمد إلى ذاكرة دائمة مع
إبقاء العملية قابلة للتفسير والمراجعة.

ميزة Dreaming **اختيارية** ومعطّلة افتراضيًا.

## ما الذي يكتبه Dreaming

يحتفظ Dreaming بنوعين من المخرجات:

- **حالة الآلة** في `memory/.dreams/` (مخزن الاستدعاء، وإشارات المراحل، ونقاط تحقق الإدخال، والأقفال).
- **مخرجات قابلة للقراءة البشرية** في `DREAMS.md` (أو `dreams.md` الموجود) وملفات تقارير المراحل الاختيارية تحت `memory/dreaming/<phase>/YYYY-MM-DD.md`.

تظل الترقية طويلة الأمد تكتب فقط إلى `MEMORY.md`.

## نموذج المراحل

يستخدم Dreaming ثلاث مراحل متعاونة:

| المرحلة | الغرض                                    | الكتابة الدائمة     |
| ------- | ---------------------------------------- | ------------------- |
| Light   | فرز وتجهيز المواد الحديثة قصيرة الأمد    | لا                  |
| Deep    | تقييم المرشحين الدائمين وترقيتهم         | نعم (`MEMORY.md`)   |
| REM     | التأمل في السمات والأفكار المتكررة       | لا                  |

هذه المراحل تفاصيل تنفيذ داخلية وليست "أوضاعًا" منفصلة يضبطها المستخدم.

### مرحلة Light

تستوعب مرحلة Light إشارات الذاكرة اليومية الحديثة وآثار الاستدعاء، وتزيل
التكرارات منها، وتجهّز الأسطر المرشحة.

- تقرأ من حالة الاستدعاء قصيرة الأمد، وملفات الذاكرة اليومية الحديثة، ونصوص الجلسات المنقّحة عند توفرها.
- تكتب كتلة `## Light Sleep` مُدارة عندما يتضمن التخزين مخرجات مضمّنة.
- تسجل إشارات تعزيز لاستخدامها لاحقًا في ترتيب Deep.
- لا تكتب أبدًا إلى `MEMORY.md`.

### مرحلة Deep

تحدد مرحلة Deep ما الذي يصبح ذاكرة طويلة الأمد.

- ترتب المرشحين باستخدام التقييم الموزون وبوابات العتبات.
- تتطلب اجتياز `minScore` و`minRecallCount` و`minUniqueQueries`.
- تعيد تحميل المقاطع من الملفات اليومية الحية قبل الكتابة، لذلك تُتخطى المقاطع القديمة أو المحذوفة.
- تضيف الإدخالات المُرقّاة إلى `MEMORY.md`.
- تكتب ملخص `## Deep Sleep` في `DREAMS.md`، ويمكنها اختياريًا كتابة `memory/dreaming/deep/YYYY-MM-DD.md`.

### مرحلة REM

تستخرج مرحلة REM الأنماط والإشارات التأملية.

- تبني ملخصات للسمات والتأملات من الآثار قصيرة الأمد الحديثة.
- تكتب كتلة `## REM Sleep` مُدارة عندما يتضمن التخزين مخرجات مضمّنة.
- تسجل إشارات تعزيز REM المستخدمة في ترتيب Deep.
- لا تكتب أبدًا إلى `MEMORY.md`.

## إدخال نصوص الجلسات

يمكن لـ Dreaming إدخال نصوص الجلسات المنقّحة إلى مجموعة بيانات Dreaming. وعندما
تكون النصوص متاحة، تُغذّى إلى مرحلة Light إلى جانب إشارات
الذاكرة اليومية وآثار الاستدعاء. ويُحذف المحتوى الشخصي والحساس
قبل الإدخال.

## Dream Diary

يحتفظ Dreaming أيضًا بـ **Dream Diary** سردية في `DREAMS.md`.
بعد أن تمتلك كل مرحلة ما يكفي من المواد، يشغّل `memory-core` بدوره أفضل جهد لوكيل فرعي
في الخلفية (باستخدام نموذج وقت التشغيل الافتراضي) ويضيف إدخالًا يوميًا قصيرًا.

هذه اليومية مخصصة للقراءة البشرية في واجهة Dreams، وليست مصدرًا للترقية.
وتُستبعد عناصر اليومية/التقارير التي ينشئها Dreaming من
الترقية قصيرة الأمد. فقط مقاطع الذاكرة المرتبطة بمرجع واضح تكون مؤهلة للترقية إلى
`MEMORY.md`.

يوجد أيضًا مسار تعبئة تاريخية لاحقة مرتبط بمرجع واضح لأعمال المراجعة والاستعادة:

- `memory rem-harness --path ... --grounded` يعرض مسبقًا مخرجات يومية مرتبطة بمرجع واضح من ملاحظات `YYYY-MM-DD.md` التاريخية.
- `memory rem-backfill --path ...` يكتب إدخالات يومية مرتبطة بمرجع واضح وقابلة للعكس في `DREAMS.md`.
- `memory rem-backfill --path ... --stage-short-term` يجهّز مرشحين دائمين مرتبطين بمرجع واضح في نفس مخزن الأدلة قصيرة الأمد الذي تستخدمه مرحلة Deep العادية بالفعل.
- `memory rem-backfill --rollback` و`--rollback-short-term` يزيلان عناصر التعبئة اللاحقة المجهزة تلك دون المساس بإدخالات اليومية العادية أو الاستدعاء الحي قصير الأمد.

تعرض واجهة Control UI التدفق نفسه للتعبئة اللاحقة/إعادة تعيين اليومية لكي تتمكن من فحص
النتائج في مشهد Dreams قبل أن تقرر ما إذا كان المرشحون المرتبطون بمرجع واضح
يستحقون الترقية. كما يعرض المشهد أيضًا مسارًا منفصلًا للعناصر المرتبطة بمرجع واضح حتى تتمكن من رؤية
أي الإدخالات قصيرة الأمد المجهزة جاءت من إعادة تشغيل تاريخية، وأي العناصر المُرقّاة كانت بقيادة عناصر مرتبطة بمرجع واضح،
ومسح الإدخالات المجهزة المرتبطة بمرجع واضح فقط دون
المساس بحالة العناصر الحية قصيرة الأمد العادية.

## إشارات ترتيب Deep

يستخدم ترتيب Deep ست إشارات أساسية موزونة بالإضافة إلى تعزيز المراحل:

| الإشارة            | الوزن | الوصف                                            |
| ------------------ | ----- | ------------------------------------------------ |
| التكرار            | 0.24  | عدد الإشارات قصيرة الأمد التي تراكمت للإدخال     |
| الصلة              | 0.30  | متوسط جودة الاسترجاع للإدخال                     |
| تنوع الاستعلامات   | 0.15  | سياقات الاستعلام/اليوم المميزة التي أظهرته      |
| الحداثة            | 0.15  | درجة حداثة متناقصة مع الزمن                      |
| الدمج              | 0.10  | قوة التكرار عبر عدة أيام                         |
| الثراء المفاهيمي   | 0.06  | كثافة وسوم المفاهيم من المقطع/المسار             |

تضيف ضربات مرحلتي Light وREM دفعة صغيرة متناقصة مع الزمن إلى
`memory/.dreams/phase-signals.json`.

## الجدولة

عند التمكين، يدير `memory-core` تلقائيًا مهمة Cron واحدة لاجتياح Dreaming كامل.
يشغّل كل اجتياح المراحل بالترتيب التالي: light -> REM -> deep.

سلوك الوتيرة الافتراضية:

| الإعداد              | الافتراضي |
| -------------------- | --------- |
| `dreaming.frequency` | `0 3 * * *` |

## البدء السريع

فعّل Dreaming:

```json
{
  "plugins": {
    "entries": {
      "memory-core": {
        "config": {
          "dreaming": {
            "enabled": true
          }
        }
      }
    }
  }
}
```

فعّل Dreaming مع وتيرة اجتياح مخصصة:

```json
{
  "plugins": {
    "entries": {
      "memory-core": {
        "config": {
          "dreaming": {
            "enabled": true,
            "timezone": "America/Los_Angeles",
            "frequency": "0 */6 * * *"
          }
        }
      }
    }
  }
}
```

## أمر الشرطة المائلة

```
/dreaming status
/dreaming on
/dreaming off
/dreaming help
```

## سير عمل CLI

استخدم الترقية عبر CLI للمعاينة أو التطبيق اليدوي:

```bash
openclaw memory promote
openclaw memory promote --apply
openclaw memory promote --limit 5
openclaw memory status --deep
```

تستخدم عملية `memory promote` اليدوية عتبات مرحلة Deep افتراضيًا ما لم يتم تجاوزها
بأعلام CLI.

اشرح لماذا سيُرقّى مرشح معيّن أو لن يُرقّى:

```bash
openclaw memory promote-explain "router vlan"
openclaw memory promote-explain "router vlan" --json
```

عاين تأملات REM، وحقائق المرشحين، ومخرجات ترقية Deep دون
كتابة أي شيء:

```bash
openclaw memory rem-harness
openclaw memory rem-harness --json
```

## القيم الافتراضية الأساسية

توجد جميع الإعدادات تحت `plugins.entries.memory-core.config.dreaming`.

| المفتاح     | الافتراضي |
| ----------- | --------- |
| `enabled`   | `false`   |
| `frequency` | `0 3 * * *` |

سياسة المراحل، والعتبات، وسلوك التخزين هي تفاصيل تنفيذ داخلية
(وليست إعدادات موجهة للمستخدم).

راجع [مرجع إعدادات الذاكرة](/ar/reference/memory-config#dreaming)
للاطلاع على القائمة الكاملة للمفاتيح.

## واجهة Dreams

عند التمكين، تعرض علامة تبويب **Dreams** في Gateway ما يلي:

- حالة تمكين Dreaming الحالية
- حالة على مستوى المرحلة ووجود اجتياح مُدار
- أعداد قصيرة الأمد، والمرتبطة بمرجع واضح، والإشارات، والمُرقّاة اليوم
- توقيت التشغيل المجدول التالي
- مسار مشهد منفصل للعناصر المرتبطة بمرجع واضح لإدخالات إعادة التشغيل التاريخية المجهزة
- قارئ Dream Diary قابل للتوسيع ومدعوم بواسطة `doctor.memory.dreamDiary`
