# Dreaming (تجريبي)

Dreaming هو نظام توحيد الذاكرة في الخلفية داخل `memory-core`.
يساعد OpenClaw على نقل الإشارات القوية قصيرة الأجل إلى ذاكرة دائمة مع
الحفاظ على العملية قابلة للتفسير والمراجعة.

Dreaming هو **اختياري** ومُعطَّل افتراضيًا.

## ما الذي يكتبه Dreaming

يحتفظ Dreaming بنوعين من المخرجات:

- **حالة الآلة** في `memory/.dreams/` (مخزن الاسترجاع، إشارات المراحل، نقاط تحقق الاستيعاب، الأقفال).
- **مخرجات قابلة للقراءة البشرية** في `DREAMS.md` (أو `dreams.md` الموجود) وملفات تقارير المراحل الاختيارية ضمن `memory/dreaming/<phase>/YYYY-MM-DD.md`.

تستمر الترقية طويلة الأجل في الكتابة إلى `MEMORY.md` فقط.

## نموذج المراحل

يستخدم Dreaming ثلاث مراحل تعاونية:

| المرحلة | الغرض                                   | الكتابة الدائمة     |
| ------- | --------------------------------------- | ------------------- |
| Light   | فرز وتجهيز المواد الحديثة قصيرة الأجل   | لا                  |
| Deep    | تقييم المرشحين الدائمين وترقيتهم        | نعم (`MEMORY.md`)   |
| REM     | التأمل في السمات والأفكار المتكررة      | لا                  |

هذه المراحل هي تفاصيل تنفيذ داخلية، وليست "أوضاعًا" منفصلة
يمكن للمستخدم إعدادها.

### مرحلة Light

تستوعب مرحلة Light إشارات الذاكرة اليومية الحديثة وآثار الاسترجاع، وتزيل التكرار منها،
وتجهز سطورًا مرشحة.

- تقرأ من حالة الاسترجاع قصيرة الأجل، وملفات الذاكرة اليومية الحديثة، ونصوص الجلسات المنقحة عند توفرها.
- تكتب كتلة `## Light Sleep` مُدارة عندما يتضمن التخزين مخرجات مضمّنة.
- تسجل إشارات تعزيز لاستخدامها لاحقًا في ترتيب Deep.
- لا تكتب أبدًا إلى `MEMORY.md`.

### مرحلة Deep

تقرر مرحلة Deep ما الذي يصبح ذاكرة طويلة الأجل.

- ترتب المرشحين باستخدام التقييم الموزون وبوابات العتبة.
- تتطلب اجتياز `minScore` و`minRecallCount` و`minUniqueQueries`.
- تعيد ترطيب المقتطفات من الملفات اليومية الحية قبل الكتابة، لذلك يتم تخطي المقتطفات القديمة أو المحذوفة.
- تُلحق الإدخالات المُرقّاة إلى `MEMORY.md`.
- تكتب ملخص `## Deep Sleep` داخل `DREAMS.md`، ويمكنها اختياريًا الكتابة إلى `memory/dreaming/deep/YYYY-MM-DD.md`.

### مرحلة REM

تستخرج مرحلة REM الأنماط والإشارات التأملية.

- تُنشئ ملخصات للسمات والتأملات من آثار قصيرة الأجل حديثة.
- تكتب كتلة `## REM Sleep` مُدارة عندما يتضمن التخزين مخرجات مضمّنة.
- تسجل إشارات تعزيز REM المستخدمة في ترتيب Deep.
- لا تكتب أبدًا إلى `MEMORY.md`.

## استيعاب نصوص الجلسات

يمكن لـ Dreaming استيعاب نصوص الجلسات المنقحة ضمن corpus الخاص بـ Dreaming. عند
توفر النصوص، يتم تمريرها إلى مرحلة Light جنبًا إلى جنب مع إشارات
الذاكرة اليومية وآثار الاسترجاع. يتم تنقيح المحتوى الشخصي والحساس
قبل الاستيعاب.

## Dream Diary

يحتفظ Dreaming أيضًا بـ **Dream Diary** سردية داخل `DREAMS.md`.
بعد أن تتوفر لكل مرحلة مادة كافية، يشغّل `memory-core`
دورة فرعية في الخلفية بأفضل جهد ممكن (باستخدام نموذج وقت التشغيل الافتراضي)
ويُلحق إدخال يوميات قصيرًا.

هذه اليوميات مخصصة للقراءة البشرية في واجهة Dreams، وليست مصدرًا للترقية.
تُستبعد ملفات اليوميات/التقارير التي يُنشئها Dreaming من
الترقية قصيرة الأجل. فقط مقتطفات الذاكرة المرتكزة على مصادر حقيقية تكون مؤهلة للترقية إلى
`MEMORY.md`.

يوجد أيضًا مسار تعبئة تاريخية مرتكز على مصادر حقيقية لأعمال المراجعة والاستعادة:

- يعاين `memory rem-harness --path ... --grounded` مخرجات يوميات مرتكزة على مصادر حقيقية من ملاحظات `YYYY-MM-DD.md` التاريخية.
- يكتب `memory rem-backfill --path ...` إدخالات يوميات مرتكزة قابلة للعكس داخل `DREAMS.md`.
- يُجهّز `memory rem-backfill --path ... --stage-short-term` مرشحين دائمين مرتكزين على مصادر حقيقية في نفس مخزن الأدلة قصيرة الأجل الذي تستخدمه مرحلة Deep العادية بالفعل.
- يزيل `memory rem-backfill --rollback` و`--rollback-short-term` ملفات التعبئة المرحلية هذه دون المساس بإدخالات اليوميات العادية أو الاسترجاع الحي قصير الأجل.

تعرض واجهة Control أيضًا نفس تدفق تعبئة/إعادة تعيين اليوميات بحيث يمكنك فحص
النتائج في مشهد Dreams قبل أن تقرر ما إذا كانت المرشحات المرتكزة على مصادر حقيقية
تستحق الترقية. ويعرض المشهد أيضًا مسارًا مرتكزًا مميزًا حتى تتمكن من رؤية
إدخالات المرحلة قصيرة الأجل التي جاءت من إعادة تشغيل تاريخية، والعناصر المُرقّاة
التي كانت مدفوعة بالمحتوى المرتكز، ومسح الإدخالات المرحلية المرتكزة فقط دون
المساس بحالة الإدخالات الحية قصيرة الأجل العادية.

## إشارات ترتيب Deep

يستخدم ترتيب Deep ست إشارات أساسية موزونة بالإضافة إلى تعزيز المراحل:

| الإشارة             | الوزن | الوصف                                            |
| ------------------- | ----- | ------------------------------------------------ |
| التكرار             | 0.24  | عدد الإشارات قصيرة الأجل التي تراكمت للإدخال     |
| الصلة               | 0.30  | متوسط جودة الاسترجاع للإدخال                     |
| تنوع الاستعلامات    | 0.15  | سياقات الاستعلام/اليوم المميزة التي أظهرته      |
| الحداثة             | 0.15  | درجة حداثة متضائلة مع الزمن                      |
| التوحيد             | 0.10  | قوة التكرار عبر أيام متعددة                      |
| الثراء المفاهيمي    | 0.06  | كثافة وسوم المفاهيم من المقتطف/المسار            |

تضيف إصابات مرحلتي Light وREM دفعة صغيرة متضائلة مع الزمن من
`memory/.dreams/phase-signals.json`.

## الجدولة

عند تفعيله، يدير `memory-core` تلقائيًا مهمة Cron واحدة لاجتياز Dreaming كامل.
يشغّل كل اجتياز المراحل بالترتيب: light -> REM -> deep.

سلوك الوتيرة الافتراضية:

| الإعداد              | الافتراضي  |
| -------------------- | ---------- |
| `dreaming.frequency` | `0 3 * * *` |

## البدء السريع

فعّل Dreaming:

```json
{
  "plugins": {
    "entries": {
      "memory-core": {
        "config": {
          "dreaming": {
            "enabled": true
          }
        }
      }
    }
  }
}
```

فعّل Dreaming مع وتيرة اجتياز مخصصة:

```json
{
  "plugins": {
    "entries": {
      "memory-core": {
        "config": {
          "dreaming": {
            "enabled": true,
            "timezone": "America/Los_Angeles",
            "frequency": "0 */6 * * *"
          }
        }
      }
    }
  }
}
```

## أمر الشرطة المائلة

```
/dreaming status
/dreaming on
/dreaming off
/dreaming help
```

## سير عمل CLI

استخدم ترقية CLI للمعاينة أو التطبيق اليدوي:

```bash
openclaw memory promote
openclaw memory promote --apply
openclaw memory promote --limit 5
openclaw memory status --deep
```

يستخدم `memory promote` اليدوي عتبات مرحلة Deep افتراضيًا ما لم يتم تجاوزها
باستخدام أعلام CLI.

اشرح لماذا سيُرقّى مرشح معيّن أو لن يُرقّى:

```bash
openclaw memory promote-explain "router vlan"
openclaw memory promote-explain "router vlan" --json
```

عاين تأملات REM، والحقائق المرشحة، ومخرجات ترقية Deep دون
كتابة أي شيء:

```bash
openclaw memory rem-harness
openclaw memory rem-harness --json
```

## القيم الافتراضية الأساسية

توجد جميع الإعدادات تحت `plugins.entries.memory-core.config.dreaming`.

| المفتاح    | الافتراضي  |
| ---------- | ---------- |
| `enabled`  | `false`    |
| `frequency` | `0 3 * * *` |

سياسة المراحل، والعتبات، وسلوك التخزين هي تفاصيل تنفيذ داخلية
(وليست إعدادات موجهة للمستخدم).

راجع [مرجع إعدادات الذاكرة](/ar/reference/memory-config#dreaming-experimental)
للحصول على القائمة الكاملة للمفاتيح.

## واجهة Dreams

عند التفعيل، تعرض علامة التبويب **Dreams** في Gateway ما يلي:

- حالة تفعيل Dreaming الحالية
- حالة على مستوى المراحل ووجود اجتياز مُدار
- أعداد العناصر قصيرة الأجل، والمرتكزة، والإشارات، والمُرقّاة اليوم
- توقيت التشغيل المجدول التالي
- مسار Scene مرتكز مميز لإدخالات إعادة التشغيل التاريخية المرحلية
- قارئ Dream Diary قابل للتوسيع ومدعوم بواسطة `doctor.memory.dreamDiary`
