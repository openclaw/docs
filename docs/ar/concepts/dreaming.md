Dreaming هو نظام دمج الذاكرة في الخلفية داخل `memory-core`.
ويساعد OpenClaw على نقل الإشارات القوية قصيرة الأمد إلى ذاكرة دائمة مع
الحفاظ على العملية قابلة للتفسير والمراجعة.

يكون Dreaming **اختياريًا** ومعطلًا افتراضيًا.

## ما الذي يكتبه Dreaming

يحتفظ Dreaming بنوعين من المخرجات:

- **حالة آلية** في `memory/.dreams/` (مخزن الاستدعاء، وإشارات المراحل، ونقاط تحقق الإدخال، والأقفال).
- **مخرجات قابلة للقراءة البشرية** في `DREAMS.md` (أو `dreams.md` الموجود) وملفات تقارير المراحل الاختيارية ضمن `memory/dreaming/<phase>/YYYY-MM-DD.md`.

تستمر الترقية طويلة الأمد في الكتابة إلى `MEMORY.md` فقط.

## نموذج المراحل

يستخدم Dreaming ثلاث مراحل تعاونية:

| المرحلة | الغرض                                   | الكتابة الدائمة      |
| ------- | --------------------------------------- | -------------------- |
| Light   | فرز وتجهيز المواد قصيرة الأمد الحديثة   | لا                   |
| Deep    | تقييم المرشحين الدائمين وترقيتهم        | نعم (`MEMORY.md`)    |
| REM     | التأمل في السمات والأفكار المتكررة      | لا                   |

هذه المراحل هي تفاصيل تنفيذ داخلية، وليست "أوضاعًا" منفصلة يضبطها المستخدم.

### مرحلة Light

تستوعب مرحلة Light إشارات الذاكرة اليومية الحديثة وآثار الاستدعاء، وتزيل التكرار،
وتجهّز أسطر المرشحين.

- تقرأ من حالة الاستدعاء القصير الأمد، وملفات الذاكرة اليومية الحديثة، ونصوص الجلسات المنقحة عند توفرها.
- تكتب كتلة `## Light Sleep` مُدارة عندما يتضمن التخزين مخرجات مضمنة.
- تسجّل إشارات تعزيز للترتيب العميق لاحقًا.
- لا تكتب أبدًا إلى `MEMORY.md`.

### مرحلة Deep

تقرر مرحلة Deep ما الذي يصبح ذاكرة طويلة الأمد.

- ترتب المرشحين باستخدام تقييم موزون وبوابات حدودية.
- تتطلب اجتياز `minScore` و`minRecallCount` و`minUniqueQueries`.
- تعيد ترطيب المقتطفات من الملفات اليومية الحية قبل الكتابة، لذلك يتم تخطي المقتطفات القديمة/المحذوفة.
- تلحق الإدخالات المرقّاة إلى `MEMORY.md`.
- تكتب ملخص `## Deep Sleep` داخل `DREAMS.md` وتكتب اختياريًا إلى `memory/dreaming/deep/YYYY-MM-DD.md`.

### مرحلة REM

تستخرج مرحلة REM الأنماط والإشارات التأملية.

- تبني ملخصات للسمات والتأملات من آثار الذاكرة قصيرة الأمد الحديثة.
- تكتب كتلة `## REM Sleep` مُدارة عندما يتضمن التخزين مخرجات مضمنة.
- تسجّل إشارات تعزيز REM المستخدمة في الترتيب العميق.
- لا تكتب أبدًا إلى `MEMORY.md`.

## استيعاب نصوص الجلسات

يمكن لـ Dreaming استيعاب نصوص الجلسات المنقحة ضمن corpus الخاص بـ Dreaming. وعندما
تكون النصوص متاحة، يتم إدخالها إلى مرحلة Light إلى جانب إشارات الذاكرة اليومية
وآثار الاستدعاء. ويتم تنقيح المحتوى الشخصي والحساس قبل الاستيعاب.

## Dream Diary

يحتفظ Dreaming أيضًا بـ **Dream Diary** سردي في `DREAMS.md`.
بعد أن تمتلك كل مرحلة مادة كافية، يقوم `memory-core` بتشغيل دور وكيل فرعي
في الخلفية بأفضل جهد (باستخدام نموذج وقت التشغيل الافتراضي) ويلحق إدخالًا يوميًا قصيرًا.

هذا اليوميات مخصص للقراءة البشرية في واجهة Dreams، وليس مصدرًا للترقية.
ويتم استبعاد العناصر الناتجة عن Dreaming في اليوميات/التقارير من
الترقية قصيرة الأمد. ولا تكون إلا مقتطفات الذاكرة المؤسَّسة مؤهلة للترقية إلى
`MEMORY.md`.

يوجد أيضًا مسار backfill تاريخي مؤسَّس لأعمال المراجعة والاسترداد:

- يعاين `memory rem-harness --path ... --grounded` مخرجات اليوميات المؤسَّسة من ملاحظات `YYYY-MM-DD.md` التاريخية.
- يكتب `memory rem-backfill --path ...` إدخالات يوميات مؤسَّسة قابلة للعكس إلى `DREAMS.md`.
- يقوم `memory rem-backfill --path ... --stage-short-term` بتجهيز مرشحين دائمين مؤسَّسين في مخزن الأدلة القصيرة الأمد نفسه الذي تستخدمه مرحلة Deep العادية أصلًا.
- يزيل `memory rem-backfill --rollback` و`--rollback-short-term` عناصر backfill المجهزة تلك من دون المساس بإدخالات اليوميات العادية أو الاستدعاء القصير الأمد الحي.

تكشف واجهة Control التدفق نفسه الخاص بـ backfill/إعادة التعيين لليوميات بحيث يمكنك
فحص النتائج في مشهد Dreams قبل أن تقرر ما إذا كانت المرشحات المؤسَّسة
تستحق الترقية. ويعرض المشهد أيضًا مسارًا مؤسَّسًا مميزًا بحيث يمكنك رؤية
أي الإدخالات القصيرة الأمد المجهزة جاءت من إعادة تشغيل تاريخية، وأي العناصر
المرقّاة كانت مدفوعة بالتأسيس، ومسح الإدخالات المجهزة المؤسَّسة فقط من دون
المساس بحالة الذاكرة القصيرة الأمد الحية العادية.

## إشارات الترتيب العميق

يستخدم الترتيب العميق ست إشارات أساسية موزونة بالإضافة إلى تعزيز المراحل:

| الإشارة              | الوزن | الوصف                                              |
| -------------------- | ------ | -------------------------------------------------- |
| Frequency            | 0.24   | عدد إشارات الذاكرة قصيرة الأمد التي راكمها الإدخال |
| Relevance            | 0.30   | متوسط جودة الاسترجاع للإدخال                       |
| Query diversity      | 0.15   | سياقات الاستعلام/اليوم المختلفة التي أظهرته       |
| Recency              | 0.15   | درجة حداثة متناقصة زمنيًا                          |
| Consolidation        | 0.10   | قوة التكرار عبر عدة أيام                           |
| Conceptual richness  | 0.06   | كثافة وسوم المفاهيم من المقتطف/المسار              |

تضيف إصابات مرحلتي Light وREM دفعة صغيرة متناقصة زمنيًا من
`memory/.dreams/phase-signals.json`.

## الجدولة

عند التمكين، يدير `memory-core` تلقائيًا مهمة Cron واحدة من أجل
اجتياح Dreaming كامل. ويشغّل كل اجتياح المراحل بالترتيب: light -> REM -> deep.

السلوك الافتراضي للإيقاع:

| الإعداد              | الافتراضي |
| -------------------- | --------- |
| `dreaming.frequency` | `0 3 * * *` |

## البدء السريع

تمكين Dreaming:

```json
{
  "plugins": {
    "entries": {
      "memory-core": {
        "config": {
          "dreaming": {
            "enabled": true
          }
        }
      }
    }
  }
}
```

تمكين Dreaming بإيقاع اجتياح مخصص:

```json
{
  "plugins": {
    "entries": {
      "memory-core": {
        "config": {
          "dreaming": {
            "enabled": true,
            "timezone": "America/Los_Angeles",
            "frequency": "0 */6 * * *"
          }
        }
      }
    }
  }
}
```

## تدفق CLI

استخدم ترقية CLI للمعاينة أو التطبيق اليدوي:

```bash
openclaw memory promote
openclaw memory promote --apply
openclaw memory promote --limit 5
openclaw memory status --deep
```

يستخدم `memory promote` اليدوي حدود مرحلة Deep افتراضيًا ما لم يتم تجاوزها
بعلامات CLI.

اشرح لماذا سيتم أو لن يتم ترقية مرشح محدد:

```bash
openclaw memory promote-explain "router vlan"
openclaw memory promote-explain "router vlan" --json
```

عاين تأملات REM، والحقائق المرشحة، ومخرجات ترقية Deep من دون
كتابة أي شيء:

```bash
openclaw memory rem-harness
openclaw memory rem-harness --json
```

## القيم الافتراضية الأساسية

توجد جميع الإعدادات ضمن `plugins.entries.memory-core.config.dreaming`.

| المفتاح    | الافتراضي |
| ---------- | --------- |
| `enabled`  | `false`   |
| `frequency` | `0 3 * * *` |

إن سياسة المراحل، والحدود، وسلوك التخزين هي تفاصيل تنفيذ
داخلية (وليست إعدادات موجهة للمستخدم).

راجع [مرجع إعدادات الذاكرة](/ar/reference/memory-config#dreaming)
للحصول على قائمة المفاتيح الكاملة.

## واجهة Dreams

عند التمكين، تعرض علامة تبويب **Dreams** في Gateway ما يلي:

- حالة تمكين Dreaming الحالية
- حالة على مستوى المرحلة ووجود الاجتياح المُدار
- أعداد العناصر قصيرة الأمد، والمؤسَّسة، والإشارات، والعناصر المرقّاة اليوم
- توقيت التشغيل المجدول التالي
- مسار Scene مؤسَّس مميز لإدخالات إعادة التشغيل التاريخية المجهزة
- قارئ Dream Diary قابل للتوسيع مدعوم بواسطة `doctor.memory.dreamDiary`
