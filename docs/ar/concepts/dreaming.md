Dreaming هو نظام دمج الذاكرة في الخلفية داخل `memory-core`. وهو يساعد OpenClaw على نقل الإشارات القوية قصيرة الأجل إلى ذاكرة دائمة مع الحفاظ على إمكانية شرح العملية ومراجعتها.

<Note>
Dreaming **اختياري** وهو معطّل افتراضيًا.
</Note>

## ما الذي يكتبه Dreaming

يحافظ Dreaming على نوعين من المخرجات:

- **حالة الآلة** في `memory/.dreams/` (مخزن الاستدعاء، وإشارات المراحل، ونقاط تحقق الإدخال، والأقفال).
- **مخرجات قابلة للقراءة البشرية** في `DREAMS.md` (أو `dreams.md` الموجود) وملفات تقارير المراحل الاختيارية تحت `memory/dreaming/<phase>/YYYY-MM-DD.md`.

لا تزال الترقية طويلة الأجل تكتب فقط إلى `MEMORY.md`.

## نموذج المراحل

يستخدم Dreaming ثلاث مراحل متعاونة:

| المرحلة | الغرض                                   | الكتابة الدائمة     |
| ------- | --------------------------------------- | ------------------- |
| Light   | فرز وتجهيز المواد الحديثة قصيرة الأجل   | لا                  |
| Deep    | تقييم المرشحين الدائمين وترقيتهم        | نعم (`MEMORY.md`)   |
| REM     | التأمل في الموضوعات والأفكار المتكررة   | لا                  |

هذه المراحل تفاصيل تنفيذ داخلية وليست "أوضاعًا" منفصلة يهيئها المستخدم.

<AccordionGroup>
  <Accordion title="مرحلة Light">
    تستوعب مرحلة Light إشارات الذاكرة اليومية الحديثة وآثار الاستدعاء، وتزيل التكرار منها، وتجهّز أسطرًا مرشحة.

    - تقرأ من حالة الاستدعاء قصيرة الأجل، وملفات الذاكرة اليومية الحديثة، ونصوص الجلسات المنقحة عند توفرها.
    - تكتب كتلة مُدارة `## Light Sleep` عندما يتضمن التخزين مخرجات مضمّنة.
    - تسجل إشارات التعزيز من أجل الترتيب اللاحق في مرحلة Deep.
    - لا تكتب أبدًا إلى `MEMORY.md`.

  </Accordion>
  <Accordion title="مرحلة Deep">
    تحدد مرحلة Deep ما الذي يصبح ذاكرة طويلة الأجل.

    - ترتب المرشحين باستخدام تقييم موزون وبوابات عتبات.
    - تتطلب اجتياز `minScore` و`minRecallCount` و`minUniqueQueries`.
    - تعيد ترطيب المقتطفات من الملفات اليومية الحية قبل الكتابة، لذلك يتم تجاوز المقتطفات القديمة/المحذوفة.
    - تُلحق الإدخالات المُرقّاة إلى `MEMORY.md`.
    - تكتب ملخص `## Deep Sleep` في `DREAMS.md`، ويمكنها اختياريًا الكتابة إلى `memory/dreaming/deep/YYYY-MM-DD.md`.

  </Accordion>
  <Accordion title="مرحلة REM">
    تستخرج مرحلة REM الأنماط والإشارات التأملية.

    - تبني ملخصات للموضوعات والتأملات من الآثار الحديثة قصيرة الأجل.
    - تكتب كتلة مُدارة `## REM Sleep` عندما يتضمن التخزين مخرجات مضمّنة.
    - تسجل إشارات تعزيز REM المستخدمة في ترتيب مرحلة Deep.
    - لا تكتب أبدًا إلى `MEMORY.md`.

  </Accordion>
</AccordionGroup>

## إدخال نصوص الجلسات

يمكن لـ Dreaming استيعاب نصوص الجلسات المنقحة ضمن corpus الخاص بـ Dreaming. وعند توفر النصوص، يتم تمريرها إلى مرحلة Light إلى جانب إشارات الذاكرة اليومية وآثار الاستدعاء. ويتم تنقيح المحتوى الشخصي والحساس قبل الإدخال.

## يوميات الأحلام

يحتفظ Dreaming أيضًا بـ **يوميات أحلام** سردية في `DREAMS.md`. وبعد أن تمتلك كل مرحلة مادة كافية، يشغّل `memory-core` دورة subagent في الخلفية على أساس أفضل جهد (باستخدام نموذج runtime الافتراضي) ويُلحق إدخال يوميات قصيرًا.

<Note>
هذه اليوميات مخصصة للقراءة البشرية في واجهة Dreams UI، وليست مصدرًا للترقية. وتُستبعد عناصر اليوميات/التقارير التي ينشئها Dreaming من الترقية قصيرة الأجل. ولا تكون مؤهلة للترقية إلى `MEMORY.md` إلا مقتطفات الذاكرة المستندة إلى أساس واقعي.
</Note>

يوجد أيضًا مسار backfill تاريخي مستند إلى أساس واقعي لأعمال المراجعة والاستعادة:

<AccordionGroup>
  <Accordion title="أوامر backfill">
    - يعرض `memory rem-harness --path ... --grounded` مخرجات يوميات مستندة إلى أساس واقعي من ملاحظات `YYYY-MM-DD.md` التاريخية.
    - يكتب `memory rem-backfill --path ...` إدخالات يوميات مستندة إلى أساس واقعي وقابلة للعكس في `DREAMS.md`.
    - يجهّز `memory rem-backfill --path ... --stage-short-term` مرشحين دائمين مستندين إلى أساس واقعي في مخزن الأدلة قصيرة الأجل نفسه الذي تستخدمه مرحلة Deep العادية أصلًا.
    - يزيل `memory rem-backfill --rollback` و`--rollback-short-term` عناصر backfill المجهزة هذه من دون المساس بإدخالات اليوميات العادية أو الاستدعاء الحي قصير الأجل.

  </Accordion>
</AccordionGroup>

تعرض Control UI أيضًا تدفق backfill/reset نفسه الخاص باليوميات حتى تتمكن من فحص النتائج في مشهد Dreams قبل تحديد ما إذا كان المرشحون المستندون إلى أساس واقعي يستحقون الترقية. كما يعرض المشهد مسارًا مستندًا إلى أساس واقعي بشكل منفصل بحيث يمكنك رؤية أي إدخالات قصيرة الأجل مجهزة جاءت من إعادة تشغيل تاريخية، وأي عناصر مُرقّاة كانت بقيادة grounded، ومسح الإدخالات المجهزة grounded-only فقط من دون المساس بالحالة العادية الحية قصيرة الأجل.

## إشارات الترتيب في مرحلة Deep

يستخدم ترتيب Deep ست إشارات أساسية موزونة بالإضافة إلى تعزيز المرحلة:

| الإشارة            | الوزن | الوصف                                              |
| ------------------ | ----- | -------------------------------------------------- |
| Frequency          | 0.24  | عدد إشارات المدى القصير التي راكمها الإدخال        |
| Relevance          | 0.30  | متوسط جودة الاسترجاع للإدخال                      |
| Query diversity    | 0.15  | سياقات الاستعلام/اليوم المختلفة التي أظهرته       |
| Recency            | 0.15  | درجة الحداثة المتناقصة بمرور الوقت                 |
| Consolidation      | 0.10  | قوة التكرار عبر عدة أيام                           |
| Conceptual richness| 0.06  | كثافة وسوم المفاهيم من المقتطف/المسار              |

تضيف إصابات مرحلتَي Light وREM تعزيزًا صغيرًا متناقصًا بمرور الوقت من `memory/.dreams/phase-signals.json`.

## الجدولة

عند التمكين، يدير `memory-core` تلقائيًا مهمة Cron واحدة لاجتياح Dreaming كامل. ويشغّل كل اجتياح المراحل بالترتيب: light ← REM ← deep.

سلوك الإيقاع الافتراضي:

| الإعداد              | الافتراضي  |
| -------------------- | ---------- |
| `dreaming.frequency` | `0 3 * * *` |

## سير عمل CLI

<Tabs>
  <Tab title="معاينة الترقية / التطبيق">
    ```bash
    openclaw memory promote
    openclaw memory promote --apply
    openclaw memory promote --limit 5
    openclaw memory status --deep
    ```

    يستخدم `memory promote` اليدوي عتبات مرحلة Deep افتراضيًا ما لم يتم تجاوزها بعلامات CLI.

  </Tab>
  <Tab title="شرح الترقية">
    اشرح سبب ترقية مرشح معيّن أو عدم ترقيته:

    ```bash
    openclaw memory promote-explain "router vlan"
    openclaw memory promote-explain "router vlan" --json
    ```

  </Tab>
  <Tab title="معاينة REM harness">
    عاين تأملات REM وحقائق المرشحين ومخرجات ترقية Deep من دون كتابة أي شيء:

    ```bash
    openclaw memory rem-harness
    openclaw memory rem-harness --json
    ```

  </Tab>
</Tabs>

## الإعدادات الافتراضية الأساسية

تقع جميع الإعدادات تحت `plugins.entries.memory-core.config.dreaming`.

<ParamField path="enabled" type="boolean" default="false">
  تمكين أو تعطيل اجتياح Dreaming.
</ParamField>
<ParamField path="frequency" type="string" default="0 3 * * *">
  إيقاع Cron لاجتياح Dreaming الكامل.
</ParamField>

<Note>
تُعد سياسة المراحل والعتبات وسلوك التخزين تفاصيل تنفيذ داخلية (وليست إعدادات موجهة للمستخدم). راجع [مرجع إعدادات الذاكرة](/ar/reference/memory-config#dreaming) للحصول على القائمة الكاملة للمفاتيح.
</Note>

## واجهة Dreams UI

عند التمكين، تعرض علامة تبويب **Dreams** في Gateway ما يلي:

- حالة تمكين Dreaming الحالية
- الحالة على مستوى المراحل ووجود الاجتياح المُدار
- أعداد العناصر قصيرة الأجل والمستندة إلى أساس واقعي والإشارات والعناصر المُرقّاة اليوم
- توقيت التشغيل المجدول التالي
- مسار Scene مستند إلى أساس واقعي بشكل منفصل لإدخالات إعادة التشغيل التاريخية المجهزة
- قارئ يوميات أحلام قابل للتوسيع ومدعوم من `doctor.memory.dreamDiary`
