يبني OpenClaw system prompt مخصصًا لكل تشغيل للوكيل. ويكون الـ prompt **مملوكًا لـ OpenClaw** ولا يستخدم prompt الافتراضي الخاص بـ pi-coding-agent.

يتم تجميع الـ prompt بواسطة OpenClaw وحقنه في كل تشغيل للوكيل.

يمكن لـ Plugins الخاصة بالموفرين المساهمة بإرشادات prompts تراعي التخزين المؤقت من دون استبدال
الـ prompt الكامل المملوك لـ OpenClaw. ويمكن لوقت تشغيل الموفر أن:

- يستبدل مجموعة صغيرة من الأقسام الأساسية المسماة (`interaction_style`،
  `tool_call_style`، `execution_bias`)
- يحقن **بادئة مستقرة** فوق حد التخزين المؤقت للـ prompt
- يحقن **لاحقة ديناميكية** تحت حد التخزين المؤقت للـ prompt

استخدم المساهمات المملوكة للموفر للضبط الخاص بعائلة النماذج. واحتفظ بتعديل الـ prompt القديم
`before_prompt_build` من أجل التوافق أو التغييرات العامة حقًا على الـ prompt، وليس
لسلوك الموفر العادي.

يحافظ التراكب الخاص بعائلة OpenAI GPT-5 على صغر قاعدة التنفيذ الأساسية ويضيف
إرشادات خاصة بالنموذج تتعلق بالالتقاط الشخصي، والإخراج الموجز، وانضباط الأدوات،
والبحث المتوازي، وتغطية الناتج، والتحقق، والسياق الناقص، ونظافة أداة الطرفية.

## البنية

الـ prompt مضغوط عمدًا ويستخدم أقسامًا ثابتة:

- **الأدوات**: تذكير بالمصدر المرجعي للأدوات المنظمة بالإضافة إلى إرشادات استخدام الأدوات في وقت التشغيل.
- **انحياز التنفيذ**: إرشادات متابعة مضغوطة: التصرّف خلال الدور نفسه
  عند الطلبات القابلة للتنفيذ، والاستمرار حتى الاكتمال أو وجود مانع، والتعافي من نتائج الأدوات
  الضعيفة، والتحقق من الحالة القابلة للتغيير مباشرة، والتحقق قبل الإنهاء.
- **الأمان**: تذكير مختصر بالحواجز الوقائية لتجنب السلوك الساعي إلى النفوذ أو تجاوز الإشراف.
- **Skills** (عند توفرها): يخبر النموذج بكيفية تحميل تعليمات المهارة عند الطلب.
- **التحديث الذاتي لـ OpenClaw**: كيفية فحص الإعداد بأمان باستخدام
  `config.schema.lookup`، وترقيع الإعداد باستخدام `config.patch`، واستبدال
  الإعداد بالكامل باستخدام `config.apply`، وتشغيل `update.run` فقط عند وجود
  طلب صريح من المستخدم. كما ترفض أداة `gateway` المخصصة للمالك فقط أيضًا إعادة كتابة
  `tools.exec.ask` / `tools.exec.security`، بما في ذلك الأسماء المستعارة القديمة
  `tools.bash.*` التي تُطبَّع إلى مسارات exec المحمية تلك.
- **مساحة العمل**: دليل العمل (`agents.defaults.workspace`).
- **التوثيق**: المسار المحلي إلى مستندات OpenClaw (مستودع أو حزمة npm) ومتى يجب قراءتها.
- **ملفات مساحة العمل (محقونة)**: يشير إلى أن ملفات التهيئة الأولية مضمّنة أدناه.
- **Sandbox** (عند تفعيله): يشير إلى بيئة تشغيل sandbox، ومسارات sandbox، وما إذا كان exec المرتفع الصلاحية متاحًا.
- **التاريخ والوقت الحاليان**: الوقت المحلي للمستخدم، والمنطقة الزمنية، وتنسيق الوقت.
- **وسوم الرد**: صياغة وسوم الرد الاختيارية للموفرين المدعومين.
- **Heartbeats**: prompt الخاصة بـ Heartbeat وسلوك الإقرار عندما تكون Heartbeats مفعلة للوكيل الافتراضي.
- **وقت التشغيل**: المضيف، ونظام التشغيل، وnode، والنموذج، وجذر المستودع (عند اكتشافه)، ومستوى التفكير (سطر واحد).
- **الاستدلال**: مستوى الظهور الحالي + تلميح تبديل `/reasoning`.

يتضمن قسم الأدوات أيضًا إرشادات وقت التشغيل للأعمال طويلة التشغيل:

- استخدم Cron للمتابعة المستقبلية (`check back later`، والتذكيرات، والعمل المتكرر)
  بدلًا من حلقات sleep في `exec`، أو حيل التأخير عبر `yieldMs`، أو الاستطلاع
  المتكرر لـ `process`
- استخدم `exec` / `process` فقط للأوامر التي تبدأ الآن وتستمر في التشغيل
  في الخلفية
- عند تفعيل تنبيه الإكمال التلقائي، ابدأ الأمر مرة واحدة ثم اعتمد على
  مسار التنبيه المعتمد على الدفع عندما يُصدر مخرجات أو يفشل
- استخدم `process` للسجلات، أو الحالة، أو الإدخال، أو التدخل عندما تحتاج إلى
  فحص أمر قيد التشغيل
- إذا كانت المهمة أكبر، ففضّل `sessions_spawn`; إذ إن إكمال الوكيل الفرعي يعتمد على
  الدفع ويُعلن تلقائيًا مرة أخرى إلى الطالب
- لا تستطلع `subagents list` / `sessions_list` في حلقة لمجرد انتظار
  الاكتمال

عند تفعيل الأداة التجريبية `update_plan`، يخبر قسم الأدوات أيضًا
النموذج باستخدامها فقط للعمل غير التافه متعدد الخطوات، والحفاظ على خطوة واحدة فقط
بحالة `in_progress`، وتجنب تكرار الخطة كاملة بعد كل تحديث.

إن حواجز الأمان الوقائية في system prompt استشارية. فهي توجّه سلوك النموذج لكنها لا تفرض سياسة. استخدم سياسة الأدوات، وموافقات exec، وsandboxing، وقوائم السماح للقنوات من أجل فرض صارم؛ ويمكن للمشغلين تعطيل هذه العناصر عمدًا.

في القنوات التي تحتوي على بطاقات/أزرار موافقة أصلية، يخبر prompt وقت التشغيل الآن
الوكيل بالاعتماد أولًا على واجهة الموافقة الأصلية تلك. ولا ينبغي له تضمين أمر
`/approve` يدوي إلا عندما تقول نتيجة الأداة إن موافقات الدردشة غير متاحة أو
إن الموافقة اليدوية هي المسار الوحيد.

## أوضاع الـ prompt

يمكن لـ OpenClaw تصيير system prompts أصغر للوكلاء الفرعيين. ويضبط وقت التشغيل
`promptMode` لكل تشغيل (وليس إعدادًا ظاهرًا للمستخدم):

- `full` (الافتراضي): يتضمن جميع الأقسام أعلاه.
- `minimal`: يُستخدم للوكلاء الفرعيين؛ ويحذف **Skills**، و**استدعاء الذاكرة**، و**التحديث الذاتي لـ OpenClaw**، و**الأسماء المستعارة للنموذج**، و**هوية المستخدم**، و**وسوم الرد**،
  و**المراسلة**، و**الردود الصامتة**، و**Heartbeats**. وتظل **الأدوات**، و**الأمان**،
  ومساحة العمل، وSandbox، والتاريخ والوقت الحاليان (عند معرفتهما)، ووقت التشغيل، والسياق
  المحقون متاحة.
- `none`: يعيد سطر الهوية الأساسي فقط.

عندما تكون `promptMode=minimal`، تتم تسمية prompts المحقونة الإضافية باسم **سياق الوكيل الفرعي**
بدلًا من **سياق محادثة المجموعة**.

## حقن التهيئة الأولية لمساحة العمل

تُقتطع ملفات التهيئة الأولية وتُلحق تحت **سياق المشروع** بحيث يرى النموذج سياق الهوية والملف الشخصي دون الحاجة إلى قراءات صريحة:

- `AGENTS.md`
- `SOUL.md`
- `TOOLS.md`
- `IDENTITY.md`
- `USER.md`
- `HEARTBEAT.md`
- `BOOTSTRAP.md` (في مساحات العمل الجديدة تمامًا فقط)
- `MEMORY.md` عند وجوده

تُحقن جميع هذه الملفات **في نافذة السياق** في كل دور ما لم
تنطبق بوابة خاصة بالملف. ويتم حذف `HEARTBEAT.md` في التشغيلات العادية عندما
تكون Heartbeats معطلة للوكيل الافتراضي أو عندما تكون
`agents.defaults.heartbeat.includeSystemPromptSection` مساوية لـ false. حافظ على إيجاز
الملفات المحقونة — وخاصة `MEMORY.md` الذي قد ينمو بمرور الوقت ويؤدي إلى
استخدام مرتفع للسياق بشكل غير متوقع وإلى Compaction أكثر تكرارًا.

> **ملاحظة:** لا تُعد ملفات `memory/*.md` اليومية **جزءًا** من التهيئة الأولية العادية
> ضمن سياق المشروع. ففي الأدوار العادية يتم الوصول إليها عند الطلب عبر
> الأداتين `memory_search` و`memory_get`، لذلك لا تُحتسب ضمن
> نافذة السياق ما لم يقرأها النموذج صراحة. أما أدوار `/new` و
> `/reset` المجردة فهي الاستثناء: إذ يمكن لوقت التشغيل أن يسبق بذاكرة يومية حديثة
> ككتلة سياق بدء تشغيل لمرة واحدة لذلك الدور الأول.

تُقتطع الملفات الكبيرة مع إضافة علامة. ويتم التحكم في الحد الأقصى لحجم كل ملف بواسطة
`agents.defaults.bootstrapMaxChars` (الافتراضي: 12000). كما يتم تقييد إجمالي محتوى
التهيئة الأولية المحقون عبر جميع الملفات بواسطة `agents.defaults.bootstrapTotalMaxChars`
(الافتراضي: 60000). وتحقن الملفات المفقودة علامة قصيرة تشير إلى غياب الملف. وعند حدوث الاقتطاع،
يمكن لـ OpenClaw حقن كتلة تحذير في سياق المشروع؛ ويتم التحكم في ذلك بواسطة
`agents.defaults.bootstrapPromptTruncationWarning` (`off` أو `once` أو `always`؛
والافتراضي: `once`).

تحقن جلسات الوكلاء الفرعيين فقط `AGENTS.md` و`TOOLS.md` (أما ملفات التهيئة الأولية الأخرى
فتتم تصفيتها للإبقاء على صغر سياق الوكيل الفرعي).

يمكن للخطافات الداخلية اعتراض هذه الخطوة عبر `agent:bootstrap` لتعديل أو استبدال
ملفات التهيئة الأولية المحقونة (على سبيل المثال استبدال `SOUL.md` بشخصية بديلة).

إذا كنت تريد جعل الوكيل يبدو أقل عمومية، فابدأ بـ
[دليل شخصية SOUL.md](/ar/concepts/soul).

ولفحص مقدار مساهمة كل ملف محقون (الخام مقابل المحقون، والاقتطاع، بالإضافة إلى كلفة مخطط الأداة)، استخدم `/context list` أو `/context detail`. راجع [السياق](/ar/concepts/context).

## التعامل مع الوقت

يتضمن system prompt قسمًا مخصصًا باسم **التاريخ والوقت الحاليان** عندما تكون
المنطقة الزمنية للمستخدم معروفة. وللحفاظ على استقرار التخزين المؤقت للـ prompt، فإنه يتضمن الآن فقط
**المنطقة الزمنية** (من دون ساعة ديناميكية أو تنسيق وقت).

استخدم `session_status` عندما يحتاج الوكيل إلى الوقت الحالي؛ إذ تتضمن
بطاقة الحالة سطر طابع زمني. ويمكن للأداة نفسها أيضًا تعيين تجاوز نموذج لكل جلسة
(`model=default` يمسحه).

اضبط ذلك باستخدام:

- `agents.defaults.userTimezone`
- `agents.defaults.timeFormat` (`auto` | `12` | `24`)

راجع [التاريخ والوقت](/ar/date-time) لمعرفة تفاصيل السلوك كاملة.

## Skills

عند وجود Skills مؤهلة، يحقن OpenClaw **قائمة Skills متاحة** مضغوطة
(`formatSkillsForPrompt`) تتضمن **مسار الملف** لكل Skill. ويوجّه
الـ prompt النموذج إلى استخدام `read` لتحميل ملف SKILL.md من الموقع المدرج
(مساحة عمل، أو مُدار، أو مضمّن). وإذا لم توجد Skills مؤهلة، يتم حذف
قسم Skills.

تشمل الأهلية بوابات البيانات الوصفية للـ Skill، وفحوصات بيئة/إعداد وقت التشغيل،
وقائمة السماح الفعلية لـ Skills الخاصة بالوكيل عندما تكون `agents.defaults.skills` أو
`agents.list[].skills` مضبوطة.

تكون Skills المضمّنة في Plugins مؤهلة فقط عندما يكون Plugin المالكة لها مفعّلة.
وهذا يسمح لـ Plugins الأدوات بكشف أدلة تشغيل أعمق من دون تضمين جميع
تلك الإرشادات مباشرة داخل كل وصف أداة.

```
<available_skills>
  <skill>
    <name>...</name>
    <description>...</description>
    <location>...</location>
  </skill>
</available_skills>
```

وهذا يُبقي الـ prompt الأساسي صغيرًا مع الاستمرار في تمكين الاستخدام المستهدف للـ Skills.

تكون ميزانية قائمة Skills مملوكة للنظام الفرعي للـ Skills:

- الافتراضي العام: `skills.limits.maxSkillsPromptChars`
- تجاوز لكل وكيل: `agents.list[].skillsLimits.maxSkillsPromptChars`

أما المقتطفات العامة المحدودة في وقت التشغيل فتستخدم سطحًا مختلفًا:

- `agents.defaults.contextLimits.*`
- `agents.list[].contextLimits.*`

يحافظ هذا الفصل على بقاء تحجيم Skills منفصلًا عن تحجيم القراءة/الحقن في وقت التشغيل مثل
`memory_get`، ونتائج الأدوات الحية، وتحديثات AGENTS.md بعد Compaction.

## التوثيق

يتضمن system prompt قسم **التوثيق**. وعندما تكون المستندات المحلية متاحة، فإنه
يشير إلى دليل مستندات OpenClaw المحلي (`docs/` في نسخة Git checkout أو مستندات
حزمة npm المضمنة). وإذا لم تكن المستندات المحلية متاحة، فإنه يعود إلى
[https://docs.openclaw.ai](https://docs.openclaw.ai).

ويتضمن القسم نفسه أيضًا موقع مصدر OpenClaw. تعرض نُسخ Git checkout جذر
المصدر المحلي حتى يستطيع الوكيل فحص الشيفرة مباشرة. أما عمليات تثبيت الحزم فتتضمن
عنوان URL لمصدر GitHub وتطلب من الوكيل مراجعة المصدر هناك كلما كانت المستندات ناقصة أو
قديمة. كما يشير الـ prompt أيضًا إلى مرآة المستندات العامة، وDiscord الخاص بالمجتمع، وClawHub
([https://clawhub.ai](https://clawhub.ai)) لاكتشاف Skills. ويوجّه النموذج إلى
الرجوع إلى المستندات أولًا فيما يتعلق بسلوك OpenClaw، أو الأوامر، أو الإعداد، أو البنية، وإلى
تشغيل `openclaw status` بنفسه عندما يكون ذلك ممكنًا (وسؤال المستخدم فقط عندما يفتقر إلى الوصول).
وبالنسبة إلى الإعداد تحديدًا، فإنه يوجّه الوكلاء إلى إجراء أداة `gateway`
`config.schema.lookup` للحصول على مستندات دقيقة على مستوى الحقول والقيود، ثم إلى
`docs/gateway/configuration.md` و`docs/gateway/configuration-reference.md`
للحصول على إرشادات أوسع.
