يضيف [Honcho](https://honcho.dev) ذاكرة مصممة أصلًا للذكاء الاصطناعي إلى OpenClaw عبر
Plugin خارجي. ويحفظ المحادثات في خدمة مخصصة ويبني
نماذج للمستخدم والوكيل بمرور الوقت، ما يمنح وكيلك سياقًا عابرًا للجلسات
يتجاوز ملفات Markdown في مساحة العمل.

## ما الذي يقدمه

- **ذاكرة عابرة للجلسات** - تستمر المحادثات بعد كل دور، وبذلك
  ينتقل السياق عبر عمليات إعادة تعيين الجلسة، وCompaction، والتبديل بين القنوات.
- **نمذجة المستخدم** - يحتفظ Honcho بملف تعريف لكل مستخدم (التفضيلات،
  والحقائق، وأسلوب التواصل) وللوكيل (الشخصية، والسلوكيات
  المكتسبة).
- **البحث الدلالي** - البحث ضمن الملاحظات المستخلصة من المحادثات السابقة، وليس
  في الجلسة الحالية فحسب.
- **الوعي بتعدد الوكلاء** - تتعقب الوكلاء الأم تلقائيًا
  الوكلاء الفرعيين الذين أنشأتهم، مع إضافة الوكلاء الأم كمراقبين في جلسات الوكلاء الفرعيين.

## الأدوات المتاحة

يسجّل Honcho أدوات يمكن للوكيل استخدامها أثناء المحادثة:

**استرجاع البيانات (سريع، من دون استدعاء نموذج لغوي كبير):**

| الأداة                      | ما الذي تفعله                                           |
| --------------------------- | ------------------------------------------------------ |
| `honcho_context`            | تمثيل كامل للمستخدم عبر الجلسات               |
| `honcho_search_conclusions` | بحث دلالي ضمن الاستنتاجات المخزنة                |
| `honcho_search_messages`    | العثور على الرسائل عبر الجلسات (التصفية حسب المرسل والتاريخ) |
| `honcho_session`            | سجل الجلسة الحالية وملخصها                    |

**الأسئلة والأجوبة (مدعومة بنموذج لغوي كبير):**

| الأداة       | ما الذي تفعله                                                              |
| ------------ | ------------------------------------------------------------------------- |
| `honcho_ask` | طرح سؤال عن المستخدم. استخدم `depth='quick'` للحقائق و`'thorough'` للاستنتاج المركب |

## بدء الاستخدام

ثبّت Plugin وشغّل الإعداد:

```bash
openclaw plugins install @honcho-ai/openclaw-honcho
openclaw honcho setup
openclaw gateway --force
```

يطلب أمر الإعداد بيانات اعتماد واجهة API، ويكتب الإعدادات، كما
يرحّل اختياريًا ملفات الذاكرة الموجودة في مساحة العمل.

<Info>
يمكن تشغيل Honcho محليًا بالكامل (باستضافة ذاتية) أو عبر واجهة API المُدارة على
`api.honcho.dev`. لا يتطلب خيار الاستضافة الذاتية أي
تبعيات خارجية.
</Info>

## الإعدادات

توجد الإعدادات ضمن `plugins.entries["openclaw-honcho"].config`:

```json5
{
  plugins: {
    entries: {
      "openclaw-honcho": {
        config: {
          apiKey: "your-api-key", // omit for self-hosted
          workspaceId: "openclaw", // memory isolation
          baseUrl: "https://api.honcho.dev",
        },
      },
    },
  },
}
```

بالنسبة إلى النُسخ ذاتية الاستضافة، وجّه `baseUrl` إلى خادمك المحلي (على سبيل المثال
`http://localhost:8000`) واحذف مفتاح API.

## ترحيل الذاكرة الموجودة

إذا كانت لديك ملفات ذاكرة موجودة في مساحة العمل (`USER.md` و`MEMORY.md`
و`IDENTITY.md` و`memory/` و`canvas/`)، فإن `openclaw honcho setup` يكتشفها
ويعرض ترحيلها.

<Info>
الترحيل غير إتلافي؛ إذ تُرفع الملفات إلى Honcho. ولا
تُحذف الملفات الأصلية أو تُنقل مطلقًا.
</Info>

## آلية العمل

بعد كل دور للذكاء الاصطناعي، تُحفظ المحادثة في Honcho. وتُرصد رسائل
المستخدم والوكيل معًا، ما يتيح لـ Honcho بناء نماذجه وتحسينها
بمرور الوقت.

أثناء المحادثة، تستعلم أدوات Honcho من الخدمة خلال نقطة ربط Plugin
`before_prompt_build` في OpenClaw، وتحقن السياق ذي الصلة قبل أن
يرى النموذج الموجّه.

## مقارنة Honcho بالذاكرة المضمنة

|                   | المضمنة / QMD                | Honcho                              |
| ----------------- | ---------------------------- | ----------------------------------- |
| **التخزين**       | ملفات Markdown في مساحة العمل     | خدمة مخصصة (محلية أو مستضافة) |
| **عبر الجلسات** | عبر ملفات الذاكرة             | تلقائي ومضمن                 |
| **نمذجة المستخدم** | يدوي (الكتابة في MEMORY.md)  | ملفات تعريف تلقائية                  |
| **البحث**        | متجهي + كلمات مفتاحية (هجين)    | دلالي ضمن الملاحظات          |
| **تعدد الوكلاء**   | غير متعقب                  | وعي بعلاقة الوكيل الأم والفرعي              |
| **التبعيات**  | لا شيء (مضمن) أو ملف QMD تنفيذي | تثبيت Plugin                      |

يمكن أن يعمل Honcho ونظام الذاكرة المضمن معًا. عند
إعداد QMD، تتوفر أدوات إضافية للبحث في ملفات Markdown المحلية
إلى جانب ذاكرة Honcho العابرة للجلسات.

## أوامر CLI

```bash
openclaw honcho setup                        # Configure API key and migrate files
openclaw honcho status                       # Check connection status
openclaw honcho ask <question>               # Query Honcho about the user
openclaw honcho search <query> [-k N] [-d D] # Semantic search over memory
```
