# قائمة انتظار الأوامر (2026-01-16)

نقوم بتسلسل تشغيلات الرد التلقائي الواردة (على جميع القنوات) عبر قائمة انتظار صغيرة داخل العملية لمنع تصادم عدة تشغيلات لوكيل واحد، مع الاستمرار في السماح بالتوازي الآمن عبر الجلسات.

## لماذا

- يمكن أن تكون تشغيلات الرد التلقائي مكلفة (استدعاءات LLM) وقد تتصادم عندما تصل عدة رسائل واردة في وقت متقارب.
- يقلل التسلسل من التنافس على الموارد المشتركة (ملفات الجلسات، والسجلات، وCLI stdin) ويخفف احتمال حدود المعدل من الجهات العليا.

## كيف يعمل

- تقوم قائمة انتظار FIFO مدركة للمسارات بتصريف كل مسار مع حد تزامن قابل للإعداد (الافتراضي 1 للمسارات غير المضبوطة؛ ويفترض `main` القيمة 4 و`subagent` القيمة 8).
- يضع `runEmbeddedPiAgent` التشغيل في قائمة الانتظار حسب **مفتاح الجلسة** (المسار `session:<key>`) لضمان وجود تشغيل نشط واحد فقط لكل جلسة.
- بعد ذلك يتم وضع تشغيل كل جلسة في **مسار عام** (`main` افتراضيًا) بحيث يُحدَّد التوازي الإجمالي بواسطة `agents.defaults.maxConcurrent`.
- عند تفعيل السجلات المفصلة، تصدر التشغيلات الموجودة في قائمة الانتظار إشعارًا قصيرًا إذا انتظرت أكثر من حوالي ثانيتين قبل البدء.
- تظل مؤشرات الكتابة تُطلق فور الإدراج في قائمة الانتظار (عند دعم القناة لذلك) بحيث تبقى تجربة المستخدم كما هي أثناء انتظار الدور.

## أوضاع قائمة الانتظار (لكل قناة)

يمكن للرسائل الواردة أن توجه التشغيل الحالي، أو تنتظر دور متابعة، أو تفعل الأمرين معًا:

- `steer`: حقن فوري في التشغيل الحالي (يلغي استدعاءات الأدوات المعلقة بعد حد الأداة التالي). وإذا لم يكن هناك بث، يعود إلى followup.
- `followup`: الإدراج في قائمة الانتظار لدور الوكيل التالي بعد انتهاء التشغيل الحالي.
- `collect`: دمج جميع الرسائل الموجودة في قائمة الانتظار في **دور متابعة واحد** (الافتراضي). وإذا كانت الرسائل تستهدف قنوات/خيوطًا مختلفة، فسيتم تصريفها بشكل منفصل للحفاظ على التوجيه.
- `steer-backlog` (ويعرف أيضًا باسم `steer+backlog`): توجيه الآن **مع** الاحتفاظ بالرسالة لدور متابعة.
- `interrupt` (قديم): إيقاف التشغيل النشط لتلك الجلسة، ثم تشغيل أحدث رسالة.
- `queue` (اسم بديل قديم): مماثل لـ `steer`.

يعني steer-backlog أنه قد تحصل على رد متابعة بعد التشغيل الموجّه، لذلك
قد تبدو أسطح البث وكأنها ترسل ردودًا مكررة. فضّل `collect`/`steer` إذا كنت تريد
ردًا واحدًا لكل رسالة واردة.
أرسل `/queue collect` كأمر مستقل (لكل جلسة) أو اضبط `messages.queue.byChannel.discord: "collect"`.

القيم الافتراضية (عند عدم ضبطها في الإعداد):

- جميع الأسطح → `collect`

اضبط ذلك عالميًا أو لكل قناة عبر `messages.queue`:

```json5
{
  messages: {
    queue: {
      mode: "collect",
      debounceMs: 1000,
      cap: 20,
      drop: "summarize",
      byChannel: { discord: "collect" },
    },
  },
}
```

## خيارات قائمة الانتظار

تنطبق الخيارات على `followup` و`collect` و`steer-backlog` (وكذلك على `steer` عندما يعود إلى followup):

- `debounceMs`: الانتظار حتى الهدوء قبل بدء دور متابعة (لمنع “continue, continue”).
- `cap`: الحد الأقصى للرسائل الموجودة في قائمة الانتظار لكل جلسة.
- `drop`: سياسة الفائض (`old` أو `new` أو `summarize`).

يحافظ summarize على قائمة نقطية قصيرة للرسائل المُسقطة ويحقنها كمطالبة متابعة اصطناعية.
القيم الافتراضية: `debounceMs: 1000` و`cap: 20` و`drop: summarize`.

## تجاوزات لكل جلسة

- أرسل `/queue <mode>` كأمر مستقل لتخزين الوضع للجلسة الحالية.
- يمكن دمج الخيارات: `/queue collect debounce:2s cap:25 drop:summarize`
- يمسح `/queue default` أو `/queue reset` التجاوز الخاص بالجلسة.

## النطاق والضمانات

- ينطبق على تشغيلات الوكيل الخاصة بالرد التلقائي عبر جميع القنوات الواردة التي تستخدم خط أنابيب الرد الخاص بـ gateway (WhatsApp web، وTelegram، وSlack، وDiscord، وSignal، وiMessage، وwebchat، وما إلى ذلك).
- المسار الافتراضي (`main`) يشمل العملية كلها للرسائل الواردة + Heartbeat الرئيسية؛ اضبط `agents.defaults.maxConcurrent` للسماح بعدة جلسات بالتوازي.
- قد توجد مسارات إضافية (مثل `cron` و`subagent`) بحيث يمكن للوظائف الخلفية أن تعمل بالتوازي من دون حجب الردود الواردة. ويتم تتبع هذه التشغيلات المنفصلة على أنها [مهام خلفية](/ar/automation/tasks).
- تضمن مسارات كل جلسة أن تشغيل وكيل واحد فقط يلمس جلسة معينة في الوقت نفسه.
- لا توجد تبعيات خارجية أو خيوط عامل في الخلفية؛ TypeScript + promises فقط.

## استكشاف الأخطاء وإصلاحها

- إذا بدت الأوامر عالقة، فعّل السجلات المفصلة وابحث عن أسطر “queued for …ms” للتأكد من أن قائمة الانتظار تُصرَّف.
- إذا كنت تحتاج إلى عمق قائمة الانتظار، فعّل السجلات المفصلة وراقب أسطر توقيت قائمة الانتظار.
