ينظّم OpenClaw المحادثات في **جلسات**. تُوجَّه كل رسالة إلى
جلسة بناءً على مصدرها -- الرسائل المباشرة، الدردشات الجماعية، مهام cron، وما إلى ذلك.

## كيفية توجيه الرسائل

| المصدر          | السلوك                  |
| --------------- | ------------------------- |
| الرسائل المباشرة | جلسة مشتركة افتراضيًا |
| الدردشات الجماعية     | معزولة لكل مجموعة        |
| الغرف/القنوات  | معزولة لكل غرفة         |
| مهام Cron       | جلسة جديدة لكل تشغيل     |
| Webhooks        | معزولة لكل hook         |

## عزل الرسائل المباشرة

افتراضيًا، تشترك جميع الرسائل المباشرة في جلسة واحدة للحفاظ على الاستمرارية. هذا مناسب
لإعدادات المستخدم الواحد.

<Warning>
إذا كان بإمكان عدة أشخاص مراسلة وكيلك، ففعّل عزل الرسائل المباشرة. بدونه، سيشارك جميع
المستخدمين سياق المحادثة نفسه -- ما يعني أن رسائل Alice الخاصة ستكون
مرئية لـ Bob.
</Warning>

**الإصلاح:**

```json5
{
  session: {
    dmScope: "per-channel-peer", // isolate by channel + sender
  },
}
```

خيارات أخرى:

- `main` (الافتراضي) -- تشترك جميع الرسائل المباشرة في جلسة واحدة.
- `per-peer` -- العزل حسب المرسل (عبر القنوات).
- `per-channel-peer` -- العزل حسب القناة + المرسل (موصى به).
- `per-account-channel-peer` -- العزل حسب الحساب + القناة + المرسل.

<Tip>
إذا تواصل معك الشخص نفسه من عدة قنوات، فاستخدم
`session.identityLinks` لربط هوياته بحيث يشارك جلسة واحدة.
</Tip>

### إرساء القنوات المرتبطة

تتيح أوامر الإرساء للمستخدم نقل مسار الرد الخاص بجلسة الدردشة المباشرة الحالية إلى
قناة مرتبطة أخرى دون بدء جلسة جديدة. راجع
[إرساء القنوات](/ar/concepts/channel-docking) للحصول على أمثلة وإعدادات
واستكشاف الأخطاء وإصلاحها.

تحقق من إعدادك باستخدام `openclaw security audit`.

## دورة حياة الجلسة

تُعاد استخدام الجلسات حتى تنتهي صلاحيتها:

- **إعادة تعيين يومية** (الافتراضي) -- جلسة جديدة عند الساعة 4:00 صباحًا بالتوقيت المحلي على مضيف
  Gateway. تستند الحداثة اليومية إلى وقت بدء `sessionId` الحالي، وليس
  إلى كتابات البيانات الوصفية اللاحقة.
- **إعادة تعيين عند الخمول** (اختياري) -- جلسة جديدة بعد فترة من عدم النشاط. عيّن
  `session.reset.idleMinutes`. تستند حداثة الخمول إلى آخر تفاعل حقيقي
  للمستخدم/القناة، لذلك لا تُبقي أحداث النظام الخاصة بـ Heartbeat وCron وexec
  الجلسة نشطة.
- **إعادة تعيين يدوية** -- اكتب `/new` أو `/reset` في الدردشة. يؤدي `/new <model>` أيضًا إلى
  تبديل النموذج.

عند إعداد كل من إعادة التعيين اليومية وإعادة التعيين عند الخمول، يُطبّق الخيار الذي تنتهي صلاحيته أولًا.
قد تكتب Heartbeat وCron وexec وغيرها من دورات أحداث النظام بيانات وصفية للجلسة،
لكن هذه الكتابات لا تمدد حداثة إعادة التعيين اليومية أو عند الخمول. عندما تؤدي إعادة التعيين إلى
تدوير الجلسة، تُتجاهل إشعارات أحداث النظام المصفوفة للجلسة القديمة
حتى لا تُسبق تحديثات الخلفية القديمة بأول موجه في
الجلسة الجديدة.

لا تقطع الإعدادات اليومية الافتراضية الضمنية الجلسات التي تحتوي على جلسة CLI نشطة مملوكة للمزوّد.
استخدم `/reset` أو اضبط `session.reset` صراحةً عندما يجب أن تنتهي صلاحية تلك
الجلسات وفق مؤقت.

## مكان حفظ الحالة

تعود ملكية كل حالة الجلسة إلى **Gateway**. تستعلم عملاء واجهة المستخدم من Gateway عن
بيانات الجلسة.

- **المخزن:** `~/.openclaw/agents/<agentId>/sessions/sessions.json`
- **النصوص:** `~/.openclaw/agents/<agentId>/sessions/<sessionId>.jsonl`

يحتفظ `sessions.json` بطوابع زمنية منفصلة لدورة الحياة:

- `sessionStartedAt`: وقت بدء `sessionId` الحالي؛ تستخدم إعادة التعيين اليومية هذا.
- `lastInteractionAt`: آخر تفاعل مستخدم/قناة يمدد مدة الخمول.
- `updatedAt`: آخر تعديل لصف المخزن؛ مفيد للعرض والتنظيف، لكنه ليس
  مرجعًا موثوقًا لحداثة إعادة التعيين اليومية/عند الخمول.

تُحل الصفوف الأقدم التي لا تحتوي على `sessionStartedAt` من ترويسة جلسة JSONL للنص
عند توفرها. إذا كان صف أقدم يفتقر أيضًا إلى `lastInteractionAt`،
تعود حداثة الخمول إلى وقت بدء تلك الجلسة، وليس إلى كتابات مسك السجلات
اللاحقة.

## صيانة الجلسات

يضع OpenClaw حدودًا تلقائية لتخزين الجلسات بمرور الوقت. افتراضيًا، يعمل
في وضع `warn` (يُبلغ عمّا كان سيُنظّف). عيّن `session.maintenance.mode`
إلى `"enforce"` للتنظيف التلقائي:

```json5
{
  session: {
    maintenance: {
      mode: "enforce",
      pruneAfter: "30d",
      maxEntries: 500,
    },
  },
}
```

بالنسبة إلى حدود `maxEntries` بحجم الإنتاج، تستخدم كتابات وقت تشغيل Gateway مخزنًا مؤقتًا صغيرًا ذا حد علوي عالٍ وتنظّف وصولًا إلى الحد الأقصى المضبوط على دفعات. يتجنب هذا تشغيل تنظيف كامل للمخزن عند كل جلسة cron معزولة. يطبّق `openclaw sessions cleanup --enforce` الحد الأقصى فورًا.

عاين باستخدام `openclaw sessions cleanup --dry-run`.

## فحص الجلسات

- `openclaw status` -- مسار مخزن الجلسات والنشاط الأخير.
- `openclaw sessions --json` -- جميع الجلسات (رشّح باستخدام `--active <minutes>`).
- `/status` في الدردشة -- استخدام السياق، والنموذج، والمفاتيح التبديلية.
- `/context list` -- ما الموجود في موجه النظام.
