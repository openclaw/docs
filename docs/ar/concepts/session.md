ينظم OpenClaw المحادثات ضمن **جلسات**. وتُوجَّه كل رسالة إلى
جلسة بحسب مصدرها -- الرسائل المباشرة، والدردشات الجماعية، ووظائف Cron، وما إلى ذلك.

## كيف تُوجَّه الرسائل

| المصدر          | السلوك                      |
| --------------- | --------------------------- |
| الرسائل المباشرة | جلسة مشتركة افتراضيًا       |
| الدردشات الجماعية | معزولة لكل مجموعة           |
| الغرف/القنوات    | معزولة لكل غرفة             |
| وظائف Cron       | جلسة جديدة لكل تشغيل        |
| Webhooks         | معزولة لكل hook             |

## عزل الرسائل المباشرة

افتراضيًا، تشترك كل الرسائل المباشرة في جلسة واحدة من أجل الاستمرارية. وهذا مناسب
للإعدادات أحادية المستخدم.

<Warning>
إذا كان بإمكان عدة أشخاص مراسلة وكيلك، ففعّل عزل الرسائل المباشرة. من دون ذلك، سيشترك
جميع المستخدمين في سياق المحادثة نفسه -- وستكون الرسائل الخاصة بـ Alice مرئية لـ Bob.
</Warning>

**الحل:**

```json5
{
  session: {
    dmScope: "per-channel-peer", // العزل حسب القناة + المرسل
  },
}
```

خيارات أخرى:

- `main` (الافتراضي) -- تشترك كل الرسائل المباشرة في جلسة واحدة.
- `per-peer` -- العزل حسب المرسل (عبر القنوات).
- `per-channel-peer` -- العزل حسب القناة + المرسل (موصى به).
- `per-account-channel-peer` -- العزل حسب الحساب + القناة + المرسل.

<Tip>
إذا كان الشخص نفسه يتواصل معك من قنوات متعددة، فاستخدم
`session.identityLinks` لربط هوياته بحيث يشترك في جلسة واحدة.
</Tip>

تحقق من إعدادك باستخدام `openclaw security audit`.

## دورة حياة الجلسة

تُعاد استخدام الجلسات إلى أن تنتهي صلاحيتها:

- **إعادة تعيين يومية** (افتراضيًا) -- جلسة جديدة عند الساعة 4:00 صباحًا بالتوقيت المحلي على
  مضيف Gateway.
- **إعادة تعيين بعد الخمول** (اختياري) -- جلسة جديدة بعد فترة من عدم النشاط. اضبط
  `session.reset.idleMinutes`.
- **إعادة تعيين يدوية** -- اكتب `/new` أو `/reset` في الدردشة. كما أن `/new <model>`
  يبدّل النموذج أيضًا.

عند تهيئة كل من إعادة التعيين اليومية وإعادة التعيين بعد الخمول، فإن أول ما تنتهي صلاحيته هو الذي يفوز.

لا تُقطع الجلسات التي تحتوي على جلسة CLI نشطة مملوكة للمزوّد بسبب القيمة الافتراضية
اليومية الضمنية. استخدم `/reset` أو هيّئ `session.reset` صراحةً عندما ينبغي أن تنتهي
تلك الجلسات بمؤقت.

## مكان وجود الحالة

تكون كل حالة الجلسة مملوكة لـ **Gateway**. وتستعلم عملاء واجهة المستخدم من
Gateway عن بيانات الجلسة.

- **المخزن:** `~/.openclaw/agents/<agentId>/sessions/sessions.json`
- **النصوص المنسوخة:** `~/.openclaw/agents/<agentId>/sessions/<sessionId>.jsonl`

## صيانة الجلسة

يحد OpenClaw تلقائيًا من تخزين الجلسات بمرور الوقت. افتراضيًا، يعمل
في وضع `warn` ‏(يبلغ عما سيتم تنظيفه). اضبط `session.maintenance.mode`
على `"enforce"` من أجل التنظيف التلقائي:

```json5
{
  session: {
    maintenance: {
      mode: "enforce",
      pruneAfter: "30d",
      maxEntries: 500,
    },
  },
}
```

عاين ذلك باستخدام `openclaw sessions cleanup --dry-run`.

## فحص الجلسات

- `openclaw status` -- مسار مخزن الجلسات والنشاط الأخير.
- `openclaw sessions --json` -- جميع الجلسات (صفِّها باستخدام `--active <minutes>`).
- `/status` في الدردشة -- استخدام السياق، والنموذج، وأوضاع التبديل.
- `/context list` -- ما الموجود في توجيه النظام.
