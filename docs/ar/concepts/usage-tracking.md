## ما هو

- يسحب الاستخدام/الحصة الخاصة بالمزوّد مباشرة من نقاط نهاية الاستخدام الخاصة به.
- لا توجد تكاليف تقديرية؛ فقط النوافذ التي يبلغ عنها المزوّد.
- يتم توحيد خرج الحالة المقروء بشريًا إلى `X% left`، حتى عندما
  تعرض واجهة API upstream الحصة المستهلكة، أو الحصة المتبقية، أو الأعداد الخام فقط.
- يمكن أن يعود `/status` و`session_status` على مستوى الجلسة إلى أحدث
  إدخال استخدام في transcript عندما تكون لقطة الجلسة الحية قليلة البيانات. هذا
  البديل يملأ عدادات tokens/cache المفقودة، ويمكنه استعادة
  تسمية النموذج النشط وقت التشغيل، ويفضّل الإجمالي الأكبر الموجّه للمطالبة عندما
  تكون بيانات الجلسة الوصفية مفقودة أو أصغر. وتظل القيم الحية غير الصفرية
  الموجودة هي الفائزة.

## أين يظهر

- `/status` في الدردشات: بطاقة حالة غنية بالرموز التعبيرية تعرض tokens الجلسة + التكلفة التقديرية (لمفتاح API فقط). يظهر استخدام المزوّد لـ **مزوّد النموذج الحالي** عندما يكون متاحًا بوصفه نافذة موحدة من نوع `X% left`.
- `/usage off|tokens|full` في الدردشات: تذييل استخدام لكل رد (يعرض OAuth tokens فقط).
- `/usage cost` في الدردشات: ملخص تكلفة محلي مجمع من سجلات جلسات OpenClaw.
- CLI: يطبع `openclaw status --usage` تفصيلًا كاملًا لكل مزوّد.
- CLI: يطبع `openclaw channels list` لقطة الاستخدام نفسها إلى جانب إعداد المزوّد (استخدم `--no-usage` للتخطي).
- شريط قوائم macOS: قسم “Usage” تحت Context (فقط إذا كان متاحًا).

## المزوّدون + بيانات الاعتماد

- **Anthropic (Claude)**: رموز OAuth في ملفات تعريف المصادقة.
- **GitHub Copilot**: رموز OAuth في ملفات تعريف المصادقة.
- **Gemini CLI**: رموز OAuth في ملفات تعريف المصادقة.
  - يعود استخدام JSON احتياطيًا إلى `stats`؛ ويتم توحيد `stats.cached` إلى
    `cacheRead`.
- **OpenAI Codex**: رموز OAuth في ملفات تعريف المصادقة (يُستخدم accountId عند وجوده).
- **MiniMax**: مفتاح API أو ملف تعريف مصادقة MiniMax OAuth. يتعامل OpenClaw مع
  `minimax` و`minimax-cn` و`minimax-portal` على أنها سطح الحصة نفسه الخاص بـ MiniMax،
  ويفضّل MiniMax OAuth المخزن عند وجوده، وإلا يعود احتياطيًا إلى
  `MINIMAX_CODE_PLAN_KEY` أو `MINIMAX_CODING_API_KEY` أو `MINIMAX_API_KEY`.
  تعني الحقول الخام `usage_percent` / `usagePercent` في MiniMax الحصة
  **المتبقية**، لذلك يعكسها OpenClaw قبل العرض؛ أما الحقول القائمة على العد فتفوز عند
  وجودها.
  - تأتي تسميات نافذة coding-plan من حقول الساعات/الدقائق الخاصة بالمزوّد عند
    وجودها، ثم تعود احتياطيًا إلى المدى `start_time` / `end_time`.
  - إذا أعادت نقطة نهاية coding-plan القيمة `model_remains`، فإن OpenClaw يفضّل
    إدخال chat-model، ويشتق تسمية النافذة من الطوابع الزمنية عندما تغيب
    الحقول الصريحة `window_hours` / `window_minutes`، ويضمّن اسم النموذج
    في تسمية الخطة.
- **Xiaomi MiMo**: مفتاح API عبر env/config/auth store (`XIAOMI_API_KEY`).
- **z.ai**: مفتاح API عبر env/config/auth store.

يُخفى الاستخدام عندما لا يمكن حل مصادقة صالحة لاستخدام المزوّد. ويمكن للمزوّدين
توفير منطق مصادقة خاص بالاستخدام عبر Plugin؛ وإلا يعود OpenClaw احتياطيًا إلى
مطابقة بيانات اعتماد OAuth/API-key من ملفات تعريف المصادقة،
أو متغيرات البيئة، أو الإعداد.
