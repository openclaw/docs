تُشغّل قناة ضمان الجودة لـ Matrix إضافة `@openclaw/matrix` المضمّنة مقابل خادم منزلي مؤقت من Tuwunel داخل Docker، مع حسابات مؤقتة للمُشغّل والنظام قيد الاختبار والمراقب، بالإضافة إلى غرف مُهيّأة مسبقًا. وهي توفّر تغطية حية وواقعية لطبقة النقل في Matrix.

أدوات مخصّصة للمشرفين فقط. لا تتضمن إصدارات OpenClaw المجمّعة `qa-lab`، لذا لا يعمل `openclaw qa` إلا من نسخة مصدر مستخرجة، حيث يحمّل المشغّل المضمّن مباشرةً دون خطوة تثبيت Plugin.

للاطلاع على سياق أوسع لإطار ضمان الجودة، راجع [نظرة عامة على ضمان الجودة](/ar/concepts/qa-e2e-automation).

## البدء السريع

```bash
pnpm openclaw qa matrix --profile fast --fail-fast
```

يشغّل الأمر العادي `pnpm openclaw qa matrix` الخيار `--profile all` ولا يتوقف عند أول فشل. قسّم القائمة الكاملة على مهام متوازية باستخدام `--profile transport|media|e2ee-smoke|e2ee-deep|e2ee-cli`.

## ما تفعله القناة

1. توفّر خادمًا منزليًا مؤقتًا من Tuwunel داخل Docker (الصورة الافتراضية `ghcr.io/matrix-construct/tuwunel:v1.5.1`، واسم الخادم `matrix-qa.test`، والمنفذ `28008`) خلف مسجّل محدود للطلبات والاستجابات يحجب البيانات الحساسة.
2. تسجّل ثلاثة مستخدمين مؤقتين: `driver` (يرسل حركة المرور الواردة)، و`sut` (حساب OpenClaw على Matrix قيد الاختبار)، و`observer` (يلتقط حركة مرور الجهات الخارجية).
3. تهيّئ الغرف التي تتطلبها السيناريوهات المحددة (الرئيسية، وسلاسل المحادثات، والوسائط، وإعادة التشغيل، والثانوية، وقائمة السماح، وE2EE، ورسائل التحقق المباشرة، وما إلى ذلك).
4. تشغّل مسبار البروتوكول `matrix-qa-v1` المحايد تجاه البنية التحتية مقابل حدود Tuwunel المسجّلة. تثبت اختبارات الوحدات عقد المسبار باستخدام تجهيز بروتوكول Matrix؛ ويتولى المضيف القياسي لمهايئ نقل ضمان الجودة في [#99707](https://github.com/openclaw/openclaw/pull/99707) توصيل أهداف Crabline الحقيقية.
5. تبدأ Gateway فرعيًا لـ OpenClaw مع Plugin الحقيقي لـ Matrix والمقيّد بحساب النظام قيد الاختبار.
6. تشغّل السيناريوهات بالتتابع، وتراقب الأحداث عبر عملاء Matrix الخاصين بالمُشغّل والمراقب، وتستنتج توقعات التوجيه والحالة من حركة المرور المسجّلة.
7. توقف الخادم المنزلي، وتكتب التقرير وأدلة الإثبات، ثم تنهي التشغيل.

## CLI

```text
pnpm openclaw qa matrix [options]
```

### العلامات الشائعة

| العلامة               | القيمة الافتراضية                            | الوصف                                                                                                                                                             |
| --------------------- | --------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--profile <profile>` | `all`                                         | ملف السيناريوهات. راجع [الملفات](#profiles).                                                                                                                      |
| `--fail-fast`         | معطّل                                         | التوقف بعد أول عملية تحقق أو سيناريو فاشل.                                                                                                                        |
| `--scenario <id>`     | -                                             | تشغيل هذا السيناريو فقط. يمكن تكراره. راجع [السيناريوهات](#scenarios).                                                                                            |
| `--output-dir <path>` | `<repo>/.artifacts/qa-e2e/matrix-<timestamp>` | موضع كتابة التقارير والملخص وقائمة التوجيه والحالة والأحداث المرصودة وسجل المخرجات. تُحل المسارات النسبية بالاستناد إلى `--repo-root`.                            |
| `--repo-root <path>`  | `process.cwd()`                               | جذر المستودع عند الاستدعاء من دليل عمل محايد.                                                                                                                     |
| `--sut-account <id>`  | `sut`                                         | معرّف حساب Matrix داخل إعدادات Gateway لضمان الجودة.                                                                                                              |

### علامات المزوّد

تستخدم القناة طبقة نقل حقيقية من Matrix، لكن يمكن ضبط مزوّد النموذج:

| العلامة                  | القيمة الافتراضية  | الوصف                                                                                                                                                                    |
| ------------------------ | ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `--provider-mode <mode>` | `live-frontier`    | استخدم `mock-openai` للإرسال الوهمي الحتمي أو `live-frontier` لمزوّدي النماذج المتقدمة المباشرين. لا يزال الاسم البديل القديم `live-openai` يعمل.                       |
| `--model <ref>`          | افتراضي المزوّد    | مرجع `provider/model` الأساسي.                                                                                                                                           |
| `--alt-model <ref>`      | افتراضي المزوّد    | مرجع `provider/model` البديل عندما تبدّل السيناريوهات النموذج أثناء التشغيل.                                                                                             |
| `--fast`                 | معطّل              | تفعيل الوضع السريع للمزوّد حيثما يكون مدعومًا.                                                                                                                           |

لا يقبل ضمان الجودة لـ Matrix الخيارين `--credential-source` أو `--credential-role`. توفّر القناة مستخدمين مؤقتين محليًا؛ ولا توجد مجموعة مشتركة من بيانات الاعتماد للاستئجار منها.

## الملفات

| الملف           | استخدامه                                                                                                                                                                                                                       |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `all` (افتراضي) | القائمة الكاملة. بطيء لكنه شامل.                                                                                                                                                                                               |
| `fast`          | مجموعة فرعية لبوابة الإصدار تختبر عقد النقل الحي الإجرائي: اشتراط الإشارة، والحظر بقائمة السماح، وشكل الرد، واستئناف التشغيل بعد إعادة البدء، ورصد التفاعلات، وتسليم بيانات اعتماد تنفيذ الأوامر، والرد الأساسي المشفّر من طرف إلى طرف. |
| `transport`     | سيناريوهات سلاسل المحادثات على مستوى النقل، والرسائل المباشرة، والغرف، والانضمام التلقائي، والإشارات وقوائم السماح، والموافقات، والتفاعلات.                                                                                     |
| `media`         | تغطية مرفقات الصور والصوت والفيديو وPDF وEPUB.                                                                                                                                                                                  |
| `e2ee-smoke`    | الحد الأدنى من تغطية التشفير من طرف إلى طرف: الرد المشفّر الأساسي، ومتابعة سلسلة المحادثة، ونجاح التهيئة الأولية.                                                                                                               |
| `e2ee-deep`     | سيناريوهات شاملة لفقدان حالة التشفير من طرف إلى طرف، والنسخ الاحتياطي، والمفاتيح، والاسترداد.                                                                                                                                   |
| `e2ee-cli`      | سيناريوهات CLI للأمر `openclaw matrix encryption setup` والأمر `verify *` التي تُدار من خلال بيئة ضمان الجودة.                                                                                                                  |

يوجد التعيين الدقيق في `extensions/qa-matrix/src/runners/contract/scenario-catalog.ts`.

## السيناريوهات

يُظهر مهايئ Matrix المشترك سيناريوهات YAML القياسية التالية من خلال `openclaw qa suite --channel-driver live --channel matrix`:

- `channel-chat-baseline`
- `thread-follow-up`
- `thread-isolation`
- `thread-reply-override`
- `dm-shared-session`
- `dm-per-room-session`

يظل `subagent-thread-spawn` متاحًا عبر التحديد الصريح `--scenario subagent-thread-spawn`، لكنه لا يشكّل جزءًا من مجموعة Matrix المشتركة الافتراضية حتى يستقر إثبات الإكمال الحي للعملية الفرعية.

قائمة معرّفات السيناريوهات الإجرائية المتبقية هي اتحاد `MatrixQaScenarioId` في `extensions/qa-matrix/src/runners/contract/scenario-catalog.ts`. الفئات:

- سلاسل المحادثات: `matrix-thread-root-preservation`، و`matrix-thread-nested-reply-shape`
- المستوى الأعلى / الرسائل المباشرة / الغرف: `matrix-top-level-reply-shape`، و`matrix-room-*`، و`matrix-dm-*`
- البث وتقدم الأدوات: `matrix-room-partial-streaming-preview`، و`matrix-room-quiet-streaming-preview`، و`matrix-room-tool-progress-*`، و`matrix-room-block-streaming`
- الوسائط: `matrix-media-type-coverage`، و`matrix-room-image-understanding-attachment`، و`matrix-attachment-only-ignored`، و`matrix-unsupported-media-safe`
- التوجيه: `matrix-room-autojoin-invite`، و`matrix-secondary-room-*`
- التفاعلات: `matrix-reaction-*`
- الموافقات: `matrix-approval-*` (البيانات الوصفية للتنفيذ وPlugin، والرجوع المجزأ، وتفاعلات الرفض، وسلاسل المحادثات، وتوجيه `target: "both"`)
- إعادة التشغيل وإعادة العرض: `matrix-restart-*`، و`matrix-stale-sync-replay-dedupe`، و`matrix-room-membership-loss`، و`matrix-homeserver-restart-resume`، و`matrix-initial-catchup-then-incremental`
- اشتراط الإشارة، والتواصل بين الروبوتات، وقوائم السماح: `matrix-mention-*`، و`matrix-allowbots-*`، و`matrix-allowlist-*`، و`matrix-multi-actor-ordering`، و`matrix-inbound-edit-*`، و`matrix-mxid-prefixed-command-block`، و`matrix-observer-allowlist-override`
- التشفير من طرف إلى طرف: `matrix-e2ee-*` (الرد الأساسي، ومتابعة سلسلة المحادثة، والتهيئة الأولية، ودورة حياة مفتاح الاسترداد، ومتغيرات فقدان الحالة، وسلوك النسخ الاحتياطي على الخادم، وسلامة الأجهزة، والتحقق عبر SAS وQR والرسائل المباشرة، وإعادة التشغيل، وحجب البيانات الحساسة من الأدلة)
- CLI للتشفير من طرف إلى طرف: `matrix-e2ee-cli-*` (إعداد التشفير، والإعداد القابل للتكرار بأمان، وفشل التهيئة الأولية، ودورة حياة مفتاح الاسترداد، وتعدد الحسابات، ودورة الرد الكاملة عبر Gateway، والتحقق الذاتي)

مرّر `--scenario <id>` (يمكن تكراره) لتشغيل مجموعة مختارة يدويًا؛ وادمجه مع `--profile all` لتجاهل تقييد الملفات.

## متغيرات البيئة

| المتغير                                | القيمة الافتراضية                                   | التأثير                                                                                                                                                                                         |
| --------------------------------------- | ----------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `OPENCLAW_QA_MATRIX_TIMEOUT_MS`         | `1800000` (30 دقيقة)                        | حد أقصى صارم لمدة التشغيل بالكامل.                                                                                                                                                            |
| `OPENCLAW_QA_MATRIX_CANARY_TIMEOUT_MS`  | `45000`                                   | الحد الزمني لرد الاختبار الأولي. ترفع عملية CI للإصدار هذه القيمة على المشغّلات المشتركة كي لا يؤدي بطء أول دورة في Gateway إلى الفشل قبل بدء تغطية السيناريوهات.                                       |
| `OPENCLAW_QA_MATRIX_NO_REPLY_WINDOW_MS` | `8000`                                    | نافذة السكون لتأكيدات عدم الرد السلبية. تُقيَّد بحيث تكون `<=` مهلة التشغيل.                                                                                                                |
| `OPENCLAW_QA_MATRIX_CLEANUP_TIMEOUT_MS` | `90000`                                   | الحد الزمني لإيقاف Docker. تتضمن تفاصيل الفشل أمر الاسترداد `docker compose ... down --remove-orphans`.                                                                           |
| `OPENCLAW_QA_MATRIX_TUWUNEL_IMAGE`      | `ghcr.io/matrix-construct/tuwunel:v1.5.1` | تجاوز صورة الخادم المنزلي عند التحقق باستخدام إصدار مختلف من Tuwunel.                                                                                                             |
| `OPENCLAW_QA_MATRIX_PROGRESS`           | مفعّل                                        | تؤدي `0` إلى كتم أسطر التقدم `[matrix-qa] ...` في stderr. وتفرض `1` إظهارها.                                                                                                                   |
| `OPENCLAW_QA_MATRIX_CAPTURE_CONTENT`    | منقّح                                  | تحتفظ `1` بنص الرسالة و`formatted_body` في `matrix-qa-observed-events.json`. ينقّح الإعداد الافتراضي المحتوى للحفاظ على أمان عناصر CI.                                                                    |
| `OPENCLAW_QA_MATRIX_DISABLE_FORCE_EXIT` | معطّل                                       | تتخطى `1` استدعاء `process.exit` الحتمي بعد كتابة العناصر. يفرض الإعداد الافتراضي الخروج لأن مقابض التشفير الأصلية في matrix-js-sdk قد تُبقي حلقة الأحداث قيد التشغيل بعد اكتمال كتابة العناصر. |
| `OPENCLAW_RUN_NODE_OUTPUT_LOG`          | غير معيّن                                     | عند تعيينه بواسطة مشغّل خارجي (مثل `scripts/run-node.mjs`)، تعيد عملية QA لـ Matrix استخدام مسار السجل هذا بدلًا من بدء عملية tee خاصة بها.                                                                   |

## عناصر الإخراج

تُكتب في `--output-dir` (القيمة الافتراضية `<repo>/.artifacts/qa-e2e/matrix-<timestamp>` كي لا تستبدل عمليات التشغيل المتتالية بعضها ببعض):

- `matrix-qa-report.md`: تقرير البروتوكول بتنسيق Markdown (ما نجح، وما فشل، وما تم تخطيه، وسبب ذلك).
- `matrix-qa-summary.json`: ملخص منظّم مناسب للتحليل بواسطة CI ولوحات المعلومات.
- `matrix-qa-route-state-manifest.json`: مخزون ديناميكي لـ `matrix-qa-v1` مفهرس بمعرّف السيناريو. يسجّل أشكال المسارات والنصوص المنقّحة، وترتيب الطلبات، وإعادات المحاولة المرصودة، والأخطاء، واستمرارية رمز المزامنة، ومجموعات حالات الأجهزة والمفاتيح والوسائط والنسخ الاحتياطي المرصودة أثناء ذلك التشغيل. هذا دليل قابل للتنفيذ، وليس خط أساس محفوظًا في المستودع.
- `matrix-qa-observed-events.json`: أحداث Matrix المرصودة من عملاء برنامج التشغيل والمراقبة. تُنقّح النصوص ما لم تكن `OPENCLAW_QA_MATRIX_CAPTURE_CONTENT=1`؛ وتُلخّص بيانات الموافقة الوصفية باستخدام حقول آمنة محددة ومعاينة مقتطعة للأمر.
- `matrix-qa-output.log`: دمج stdout وstderr من التشغيل. إذا عُيّن `OPENCLAW_RUN_NODE_OUTPUT_LOG`، فيُعاد استخدام سجل المشغّل الخارجي بدلًا منه.

## نصائح استكشاف الأخطاء

- **توقّف التشغيل قرب النهاية:** قد تستمر مقابض التشفير الأصلية في `matrix-js-sdk` بعد انتهاء عمل إطار الاختبار. يفرض الإعداد الافتراضي استدعاءً نظيفًا لـ `process.exit` بعد كتابة العناصر؛ وإذا عيّنت `OPENCLAW_QA_MATRIX_DISABLE_FORCE_EXIT=1`، فتوقّع بقاء العملية قيد التشغيل.
- **خطأ في التنظيف:** ابحث عن أمر الاسترداد المطبوع (استدعاء `docker compose ... down --remove-orphans`) وشغّله يدويًا لتحرير منفذ الخادم المنزلي.
- **عدم استقرار نوافذ التأكيد السلبي في CI:** اخفض `OPENCLAW_QA_MATRIX_NO_REPLY_WINDOW_MS` (القيمة الافتراضية 8 ثوانٍ) عندما تكون CI سريعة؛ وارفعها على المشغّلات المشتركة البطيئة.
- **الحاجة إلى نصوص منقّحة لتقرير خطأ:** أعِد التشغيل باستخدام `OPENCLAW_QA_MATRIX_CAPTURE_CONTENT=1` وأرفق `matrix-qa-observed-events.json`. تعامل مع العنصر الناتج على أنه حساس.
- **إصدار مختلف من Tuwunel:** وجّه `OPENCLAW_QA_MATRIX_TUWUNEL_IMAGE` إلى الإصدار قيد الاختبار. لا يضمّن مسار الاختبار في المستودع سوى الصورة الافتراضية المثبّتة.

## عقد النقل المباشر

Matrix هو أحد مسارات النقل المباشر الثلاثة (Matrix وTelegram وDiscord) التي تشترك في قائمة تحقق واحدة للعقد، والمحددة في [نظرة عامة على QA: تغطية النقل المباشر](/ar/concepts/qa-e2e-automation#live-transport-coverage). تظل `qa-channel` مجموعة الاختبارات الاصطناعية الشاملة، ولا تُعد جزءًا من تلك المصفوفة عن قصد.
