تمنح OpenClaw الوكلاء أدوات للعمل عبر الجلسات، وفحص الحالة، وتنسيق
الوكلاء الفرعيين.

## الأدوات المتاحة

| الأداة              | ماذا تفعل                                                                    |
| ------------------ | --------------------------------------------------------------------------- |
| `sessions_list`    | تسرد الجلسات مع مرشحات اختيارية (النوع، التسمية، الوكيل، الحداثة، المعاينة) |
| `sessions_history` | تقرأ نص جلسة محددة                                                           |
| `sessions_send`    | ترسل رسالة إلى جلسة أخرى وتنتظر اختياريًا                                   |
| `sessions_spawn`   | تنشئ جلسة وكيل فرعي معزولة للعمل في الخلفية                                 |
| `sessions_yield`   | تنهي الدور الحالي وتنتظر نتائج متابعة من الوكيل الفرعي                      |
| `subagents`        | تسرد حالة الوكلاء الفرعيين المنشئين لهذه الجلسة                             |
| `session_status`   | تعرض بطاقة بأسلوب `/status` وتضبط اختياريًا تجاوز نموذج لكل جلسة           |

تظل هذه الأدوات خاضعة لملف تعريف الأدوات النشط وسياسة السماح/الرفض.
يتضمن `tools.profile: "coding"` مجموعة تنسيق الجلسات كاملة، بما في ذلك
`sessions_spawn` و`sessions_yield` و`subagents`.
يتضمن `tools.profile: "messaging"` أدوات المراسلة عبر الجلسات
(`sessions_list` و`sessions_history` و`sessions_send` و`session_status`) لكنه
لا يتضمن إنشاء الوكلاء الفرعيين. للاحتفاظ بملف تعريف مراسلة مع السماح
بالتفويض الأصلي أيضًا، أضف:

```json5
{
  tools: {
    profile: "messaging",
    alsoAllow: ["sessions_spawn", "sessions_yield", "subagents"],
  },
}
```

لا يزال بإمكان سياسات المجموعة والمزود وبيئة العزل وكل وكيل إزالة تلك الأدوات
بعد مرحلة ملف التعريف. استخدم `/tools` من الجلسة المتأثرة لفحص قائمة الأدوات
الفعالة.

## سرد الجلسات وقراءتها

يعيد `sessions_list` الجلسات مع مفتاحها وagentId والنوع والقناة والنموذج
وأعداد الرموز والطوابع الزمنية. رشح حسب النوع (`main` أو `group` أو `cron` أو `hook`
أو `node`) أو `label` المطابق تمامًا أو `agentId` المطابق تمامًا أو نص البحث
أو الحداثة (`activeMinutes`). عندما تحتاج إلى فرز بأسلوب صندوق البريد، يمكنه
أيضًا طلب عنوان مشتق محدود الرؤية، أو مقتطف معاينة لآخر رسالة، أو رسائل حديثة
محدودة لكل صف. لا تُنتج العناوين والمُعاينات المشتقة إلا للجلسات التي يستطيع
المستدعي رؤيتها مسبقًا بموجب سياسة رؤية أداة الجلسة المكوّنة، لذلك تبقى
الجلسات غير ذات الصلة مخفية. عندما تكون الرؤية مقيدة، يعيد `sessions_list`
بيانات تعريف `visibility` اختيارية تعرض الوضع الفعال وتحذيرًا بأن النتائج قد
تكون محدودة النطاق.

يجلب `sessions_history` نص المحادثة لجلسة محددة. افتراضيًا، تُستبعد نتائج
الأدوات -- مرر `includeTools: true` لرؤيتها. العرض المُعاد محدود عمدًا ومفلتر
من ناحية السلامة:

- يُطبّع نص المساعد قبل الاسترجاع:
  - تُزال وسوم التفكير
  - تُزال كتل السقالات `<relevant-memories>` / `<relevant_memories>`
  - تُزال كتل حمولات XML لاستدعاءات الأدوات بالنص الصريح مثل `<tool_call>...</tool_call>`،
    و`<function_call>...</function_call>`، و`<tool_calls>...</tool_calls>`، و
    `<function_calls>...</function_calls>`، بما في ذلك الحمولات المقطوعة
    التي لا تُغلق بشكل نظيف
  - تُزال سقالات استدعاء/نتيجة الأداة المخفضة مثل `[Tool Call: ...]`،
    و`[Tool Result ...]`، و`[Historical context ...]`
  - تُزال رموز التحكم بالنموذج المسرّبة مثل `<|assistant|>`، ورموز ASCII
    الأخرى على شكل `<|...|>`، ومتغيرات العرض الكامل `<｜...｜>`
  - تُزال XML استدعاء الأدوات المشوهة الخاصة بـ MiniMax مثل `<invoke ...>` /
    `</minimax:tool_call>`
- يُنقح النص الشبيه ببيانات الاعتماد/الرموز قبل إعادته
- تُقتطع كتل النص الطويلة
- يمكن للتواريخ الكبيرة جدًا إسقاط صفوف أقدم أو استبدال صف كبير جدًا بـ
  `[sessions_history omitted: message too large]`
- تبلغ الأداة عن أعلام ملخص مثل `truncated` و`droppedMessages` و
  `contentTruncated` و`contentRedacted` و`bytes`

تقبل كلتا الأداتين إما **مفتاح جلسة** (مثل `"main"`) أو **معرف جلسة**
من استدعاء قائمة سابق.

إذا احتجت إلى نص مطابق بايتًا ببايت، فافحص ملف النص على القرص بدلًا من
التعامل مع `sessions_history` كتفريغ خام.

## إرسال رسائل عبر الجلسات

يوصل `sessions_send` رسالة إلى جلسة أخرى وينتظر الرد اختياريًا:

- **الإرسال دون انتظار:** اضبط `timeoutSeconds: 0` لإدراجها في الطابور
  والعودة فورًا.
- **انتظار الرد:** اضبط مهلة واحصل على الرد ضمنيًا.

جلسات الدردشة محدودة النطاق بالخيط، مثل مفاتيح Slack أو Discord المنتهية بـ
`:thread:<id>`، ليست أهدافًا صالحة لـ `sessions_send`. استخدم مفتاح جلسة
القناة الأصلية لتنسيق ما بين الوكلاء حتى لا تظهر الرسائل الموجهة عبر الأدوات
داخل خيط نشط موجه للبشر.

تُعلّم الرسائل وردود المتابعة A2A كبيانات بين الجلسات في مطالبة المستلم
(`[Inter-session message ... isUser=false]`) وفي مصدرية النص. يجب أن يتعامل
الوكيل المستلم معها كبيانات موجهة عبر الأدوات، لا كتعليمات مباشرة كتبها
المستخدم النهائي.

بعد أن يرد الهدف، يمكن لـ OpenClaw تشغيل **حلقة رد عكسي** حيث يتبادل
الوكلاء الرسائل (حتى `session.agentToAgent.maxPingPongTurns`، بالنطاق
0-20، والافتراضي 5). يمكن للوكيل الهدف الرد بـ
`REPLY_SKIP` للتوقف مبكرًا.

## مساعدات الحالة والتنسيق

`session_status` هي الأداة الخفيفة المكافئة لـ `/status` للجلسة الحالية أو
لجلسة مرئية أخرى. تبلغ عن الاستخدام والوقت وحالة النموذج/وقت التشغيل وسياق
المهمة الخلفية المرتبط عند وجوده. مثل `/status`، يمكنها ملء عدادات الرموز/
الذاكرة المخبأة المتفرقة من أحدث إدخال استخدام في النص، ويمسح
`model=default` تجاوزًا لكل جلسة. استخدم `sessionKey="current"` لجلسة
المستدعي الحالية؛ تسميات العميل المرئية مثل `openclaw-tui` ليست مفاتيح
جلسات.

عندما تكون بيانات تعريف المسار متاحة، يتضمن `session_status` أيضًا كتلة JSON
مرئية باسم `Route context` وحقول `details` منظمة مطابقة. تزيل هذه الحقول
الالتباس بين مفتاح الجلسة والمسار الذي يتولى حاليًا تشغيل التنفيذ الحي:

- `origin` هو المكان الذي أُنشئت فيه الجلسة، أو المزود المستنتج من بادئة
  مفتاح جلسة قابلة للتسليم عندما تفتقر الحالة الأقدم إلى بيانات تعريف أصل
  مخزنة.
- `active` هو مسار التشغيل الحي الحالي. لا يُبلغ عنه إلا للجلسة الحية أو
  الحالية التي تُعالج الآن.
- `deliveryContext` هو مسار التسليم المستمر المخزن على الجلسة، والذي يمكن
  لـ OpenClaw إعادة استخدامه للتسليم لاحقًا حتى عندما يختلف السطح النشط.

ينهي `sessions_yield` الدور الحالي عمدًا حتى تكون الرسالة التالية هي حدث
المتابعة الذي تنتظره. استخدمه بعد إنشاء وكلاء فرعيين عندما تريد أن تصل
نتائج الإكمال كرسالة تالية بدلًا من بناء حلقات استطلاع.

`subagents` هو مساعد الرؤية للوكلاء الفرعيين في OpenClaw الذين أُنشئوا
بالفعل. يدعم `action: "list"` لفحص عمليات التشغيل النشطة/الحديثة.

## إنشاء وكلاء فرعيين

ينشئ `sessions_spawn` افتراضيًا جلسة معزولة لمهمة خلفية. وهو دائمًا غير
حاجب -- إذ يعود فورًا مع `runId` و`childSessionKey`. تتلقى عمليات تشغيل
الوكلاء الفرعيين الأصلية المهمة المفوضة في أول رسالة مرئية `[Subagent Task]`
في جلسة الطفل، بينما تحمل مطالبة النظام قواعد وقت تشغيل الوكيل الفرعي وسياق
التوجيه فقط.

الخيارات الأساسية:

- `runtime: "subagent"` (الافتراضي) أو `"acp"` لوكلاء الحاضنات الخارجية.
- تجاوزات `model` و`thinking` لجلسة الطفل.
- `thread: true` لربط الإنشاء بخيط دردشة (Discord وSlack وما إلى ذلك).
- `sandbox: "require"` لفرض العزل على الطفل.
- `context: "fork"` للوكلاء الفرعيين الأصليين عندما يحتاج الطفل إلى نص
  الطالب الحالي؛ احذفه أو استخدم `context: "isolated"` لطفل نظيف.
  يعتمد الوكلاء الفرعيون الأصليون المرتبطون بالخيط افتراضيًا
  `context: "fork"` ما لم يحدد `threadBindings.defaultSpawnContext` خلاف ذلك.

لا تحصل الوكلاء الفرعيون الطرفيون الافتراضيون على أدوات الجلسة. عندما يكون
`maxSpawnDepth >= 2`، يتلقى الوكلاء الفرعيون المنسقون عند العمق 1 أيضًا
`sessions_spawn` و`subagents` و`sessions_list` و`sessions_history` حتى
يتمكنوا من إدارة أطفالهم. لا تزال عمليات التشغيل الطرفية لا تحصل على أدوات
تنسيق تكرارية.

بعد الإكمال، تنشر خطوة إعلان النتيجة إلى قناة الطالب. يحافظ تسليم الإكمال
على توجيه الخيط/الموضوع المرتبط عند توفره، وإذا كان أصل الإكمال يحدد قناة
فقط فلا يزال بإمكان OpenClaw إعادة استخدام المسار المخزن لجلسة الطالب
(`lastChannel` / `lastTo`) للتسليم المباشر.

للسلوك الخاص بـ ACP، راجع [وكلاء ACP](/ar/tools/acp-agents).

## الرؤية

تُحدد أدوات الجلسة النطاق لتقييد ما يستطيع الوكيل رؤيته:

| المستوى | النطاق                                  |
| ------- | ---------------------------------------- |
| `self`  | الجلسة الحالية فقط                      |
| `tree`  | الجلسة الحالية + الوكلاء الفرعيون المنشؤون |
| `agent` | كل الجلسات لهذا الوكيل                  |
| `all`   | كل الجلسات (عبر الوكلاء إذا كان ذلك مكوّنًا) |

الافتراضي هو `tree`. تُقيّد الجلسات المعزولة إلى `tree` بغض النظر عن
التكوين.
