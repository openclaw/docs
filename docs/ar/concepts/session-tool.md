يمنح OpenClaw الوكلاء أدوات للعمل عبر الجلسات، وفحص الحالة، و
تنسيق الوكلاء الفرعيين.

## الأدوات المتاحة

| الأداة              | ما تفعله                                                                    |
| ------------------ | --------------------------------------------------------------------------- |
| `sessions_list`    | يسرد الجلسات مع مرشحات اختيارية (النوع، التسمية، الوكيل، الحداثة، المعاينة) |
| `sessions_history` | يقرأ نص جلسة محددة                                                          |
| `sessions_send`    | يرسل رسالة إلى جلسة أخرى وينتظر اختياريا                                    |
| `sessions_spawn`   | ينشئ جلسة وكيل فرعي معزولة للعمل في الخلفية                                |
| `sessions_yield`   | ينهي الدور الحالي وينتظر نتائج المتابعة من الوكيل الفرعي                    |
| `subagents`        | يسرد الوكلاء الفرعيين المنشئين لهذه الجلسة أو يوجههم أو يوقفهم             |
| `session_status`   | يعرض بطاقة بنمط `/status` ويعين اختياريا تجاوزا للنموذج لكل جلسة           |

تظل هذه الأدوات خاضعة لملف تعريف الأدوات النشط وسياسة السماح/المنع.
يتضمن `tools.profile: "coding"` مجموعة تنسيق الجلسات الكاملة،
بما في ذلك `sessions_spawn` و`sessions_yield` و`subagents`.
يتضمن `tools.profile: "messaging"` أدوات المراسلة عبر الجلسات
(`sessions_list` و`sessions_history` و`sessions_send` و`session_status`) لكنه
لا يتضمن إنشاء الوكلاء الفرعيين. للاحتفاظ بملف تعريف مراسلة مع السماح
بالتفويض الأصلي، أضف:

```json5
{
  tools: {
    profile: "messaging",
    alsoAllow: ["sessions_spawn", "sessions_yield", "subagents"],
  },
}
```

لا يزال بإمكان سياسات المجموعة والمزود وبيئة العزل ولكل وكيل إزالة تلك الأدوات
بعد مرحلة ملف التعريف. استخدم `/tools` من الجلسة المتأثرة لفحص قائمة الأدوات
الفعالة.

## سرد الجلسات وقراءتها

يعيد `sessions_list` الجلسات مع مفتاحها وagentId والنوع والقناة والنموذج
وأعداد الرموز والطوابع الزمنية. رشح حسب النوع (`main` و`group` و`cron` و`hook`
و`node`)، أو `label` دقيق، أو `agentId` دقيق، أو نص بحث، أو الحداثة
(`activeMinutes`). عندما تحتاج إلى فرز شبيه بصندوق البريد، يمكنه أيضا طلب
عنوان مشتق ضمن نطاق الرؤية، أو مقتطف معاينة لآخر رسالة، أو رسائل حديثة محدودة
في كل صف. لا تنتج العناوين المشتقة والمعاينات إلا للجلسات التي يستطيع المستدعي
رؤيتها بالفعل بموجب سياسة رؤية أداة الجلسة المكونة، لذا تبقى الجلسات غير ذات
الصلة مخفية.

يجلب `sessions_history` نص المحادثة لجلسة محددة. افتراضيا، تستبعد نتائج
الأدوات -- مرر `includeTools: true` لرؤيتها. العرض المعاد محدود ومفلتر للسلامة
عمدا:

- يطبع نص المساعد قبل الاستدعاء:
  - تزال وسوم التفكير
  - تزال كتل السقالات `<relevant-memories>` / `<relevant_memories>`
  - تزال كتل حمولات XML لاستدعاء الأدوات بنص عادي مثل `<tool_call>...</tool_call>`،
    و`<function_call>...</function_call>`، و`<tool_calls>...</tool_calls>`، و
    `<function_calls>...</function_calls>`، بما في ذلك الحمولات المقتطعة
    التي لا تغلق بشكل سليم أبدا
  - تزال سقالات استدعاء/نتيجة الأداة المخفضة مثل `[Tool Call: ...]`،
    و`[Tool Result ...]`، و`[Historical context ...]`
  - تزال رموز تحكم النموذج المسربة مثل `<|assistant|>`، ورموز ASCII الأخرى
    من نوع `<|...|>`، ومتغيرات العرض الكامل `<｜...｜>`
  - تزال XML استدعاء أدوات MiniMax المشوهة مثل `<invoke ...>` /
    `</minimax:tool_call>`
- ينقح النص الشبيه ببيانات الاعتماد/الرموز قبل إرجاعه
- تقتطع كتل النص الطويلة
- يمكن للسجلات الكبيرة جدا إسقاط الصفوف الأقدم أو استبدال صف كبير جدا بـ
  `[sessions_history omitted: message too large]`
- تبلغ الأداة عن أعلام ملخصة مثل `truncated` و`droppedMessages` و
  `contentTruncated` و`contentRedacted` و`bytes`

تقبل كلتا الأداتين إما **مفتاح جلسة** (مثل `"main"`) أو **معرف جلسة**
من استدعاء سرد سابق.

إذا كنت تحتاج إلى النص الحرفي مطابقا بايتا ببايت، فافحص ملف النص على القرص
بدلا من التعامل مع `sessions_history` كنسخة أولية خام.

## إرسال رسائل عبر الجلسات

يسلم `sessions_send` رسالة إلى جلسة أخرى وينتظر الرد اختياريا:

- **إرسال دون انتظار:** عين `timeoutSeconds: 0` لوضع الرسالة في الصف والعودة
  فورا.
- **انتظار الرد:** عين مهلة واحصل على الرد مضمنًا.

تعلم الرسائل وردود متابعة A2A كبيانات بين الجلسات في موجه الاستقبال
(`[Inter-session message ... isUser=false]`) وفي مصدرية النص. يجب على الوكيل
المستقبل التعامل معها كبيانات موجهة عبر الأداة، لا كتعليمة مباشرة كتبها
المستخدم النهائي.

بعد استجابة الهدف، يمكن لـ OpenClaw تشغيل **حلقة رد عكسي** يتبادل فيها
الوكلاء الرسائل (حتى 5 أدوار). يمكن للوكيل الهدف الرد بـ `REPLY_SKIP`
للتوقف مبكرا.

## الحالة ومساعدات التنسيق

`session_status` هو الأداة الخفيفة المكافئة لـ `/status` للجلسة الحالية
أو جلسة أخرى مرئية. تبلغ عن الاستخدام والوقت وحالة النموذج/وقت التشغيل وسياق
مهمة الخلفية المرتبطة عند وجوده. مثل `/status`، يمكنها ملء عدادات الرموز/
ذاكرة التخزين المؤقت المتناثرة من أحدث إدخال استخدام في النص، ويمسح
`model=default` تجاوزا لكل جلسة. استخدم `sessionKey="current"` لجلسة
المستدعي الحالية؛ التسميات المرئية للعميل مثل `openclaw-tui` ليست مفاتيح
جلسات.

ينهي `sessions_yield` الدور الحالي عمدا كي تكون الرسالة التالية هي حدث
المتابعة الذي تنتظره. استخدمه بعد إنشاء وكلاء فرعيين عندما تريد أن تصل نتائج
الإكمال كرسالة تالية بدلا من بناء حلقات استطلاع.

`subagents` هو مساعد مستوى التحكم للوكلاء الفرعيين من OpenClaw الذين سبق
إنشاؤهم. يدعم:

- `action: "list"` لفحص التشغيلات النشطة/الحديثة
- `action: "steer"` لإرسال إرشادات متابعة إلى تابع قيد التشغيل
- `action: "kill"` لإيقاف تابع واحد أو `all`

## إنشاء وكلاء فرعيين

ينشئ `sessions_spawn` افتراضيا جلسة معزولة لمهمة خلفية. وهو دائما غير حاجب --
يعيد مباشرة `runId` و`childSessionKey`.

الخيارات الرئيسية:

- `runtime: "subagent"` (الافتراضي) أو `"acp"` لوكلاء الحاضنات الخارجية.
- تجاوزات `model` و`thinking` للجلسة التابعة.
- `thread: true` لربط الإنشاء بمحادثة متفرعة (Discord وSlack وغيرهما).
- `sandbox: "require"` لفرض العزل على التابع.
- `context: "fork"` للوكلاء الفرعيين الأصليين عندما يحتاج التابع إلى نص
  المستدعي الحالي؛ احذفه أو استخدم `context: "isolated"` لتابع نظيف.

لا تحصل الوكلاء الفرعيون الورقيون الافتراضيون على أدوات الجلسة. عندما يكون
`maxSpawnDepth >= 2`، يتلقى وكلاء التنسيق الفرعيون في العمق 1 أيضا
`sessions_spawn` و`subagents` و`sessions_list` و`sessions_history` كي يتمكنوا
من إدارة توابعهم. لا تزال التشغيلات الورقية لا تحصل على أدوات تنسيق تكرارية.

بعد الإكمال، تنشر خطوة إعلان النتيجة في قناة المستدعي. يحافظ تسليم الإكمال على
توجيه المحادثة المتفرعة/الموضوع المرتبط عند توفره، وإذا كان أصل الإكمال لا
يحدد إلا قناة فيمكن لـ OpenClaw مع ذلك إعادة استخدام المسار المخزن لجلسة
المستدعي (`lastChannel` / `lastTo`) للتسليم المباشر.

للسلوك الخاص بـ ACP، راجع [وكلاء ACP](/ar/tools/acp-agents).

## الرؤية

تحدد أدوات الجلسة نطاق ما يستطيع الوكيل رؤيته:

| المستوى | النطاق                                   |
| ------- | ---------------------------------------- |
| `self`  | الجلسة الحالية فقط                       |
| `tree`  | الجلسة الحالية + الوكلاء الفرعيون المنشؤون |
| `agent` | كل الجلسات لهذا الوكيل                   |
| `all`   | كل الجلسات (عبر الوكلاء إذا تم تكوين ذلك) |

الافتراضي هو `tree`. تقيد الجلسات المعزولة إلى `tree` بغض النظر عن
التكوين.
