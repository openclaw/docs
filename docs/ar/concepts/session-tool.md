يمنح OpenClaw الوكلاء أدوات للعمل عبر الجلسات، وفحص الحالة،
وتنسيق الوكلاء الفرعيين.

## الأدوات المتاحة

| الأداة | ما الذي تفعله |
| ------ | ------------- |
| `sessions_list` | سرد الجلسات مع عوامل تصفية اختيارية (النوع، والملصق، والوكيل، والحداثة، والمعاينة) |
| `sessions_history` | قراءة سجل جلسة محددة |
| `sessions_send` | إرسال رسالة إلى جلسة أخرى وانتظارها اختياريًا |
| `sessions_spawn` | إنشاء جلسة وكيل فرعي معزولة للعمل في الخلفية |
| `sessions_yield` | إنهاء الدور الحالي وانتظار نتائج متابعة من وكيل فرعي |
| `subagents` | سرد الوكلاء الفرعيين المنشأين لهذه الجلسة أو توجيههم أو إنهاؤهم |
| `session_status` | إظهار بطاقة على نمط `/status` وضبط تجاوز نموذج لكل جلسة اختياريًا |

## سرد الجلسات وقراءتها

تعيد `sessions_list` الجلسات مع مفتاحها، و`agentId`، ونوعها، وقناتها، ونموذجها،
وعدد الرموز، والطوابع الزمنية. يمكنك التصفية حسب النوع (`main` أو `group` أو `cron` أو `hook` أو
`node`)، أو `label` المطابق تمامًا، أو `agentId` المطابق تمامًا، أو نص البحث، أو الحداثة
(`activeMinutes`). وعندما تحتاج إلى فرز على نمط صندوق البريد، يمكنها أيضًا
طلب عنوان مشتق ضمن نطاق الرؤية، أو مقتطف معاينة من آخر رسالة، أو
رسائل حديثة محدودة في كل صف. ولا تُنتَج العناوين المشتقة والمعاينات إلا
للجلسات التي يمكن للطالب رؤيتها أصلًا وفقًا لسياسة
رؤية أدوات الجلسة المضبوطة، بحيث تبقى الجلسات غير ذات الصلة مخفية.

تجلب `sessions_history` سجل المحادثة لجلسة محددة.
افتراضيًا، تُستبعد نتائج الأدوات -- مرّر `includeTools: true` لرؤيتها.
ويكون العرض المُعاد محدودًا ومفلترًا أمنيًا عمدًا:

- يُطبَّع نص المساعد قبل الاسترجاع:
  - تُزال وسوم التفكير
  - تُزال كتل scaffolding من `<relevant-memories>` / `<relevant_memories>`
  - تُزال كتل حمولة XML النصية الصريحة لاستدعاءات الأدوات مثل `<tool_call>...</tool_call>`،
    و`<function_call>...</function_call>`، و`<tool_calls>...</tool_calls>`، و
    `<function_calls>...</function_calls>`، بما في ذلك الحمولات المبتورة
    التي لا تُغلق بشكل نظيف
  - تُزال بنى scaffolding المُخفَّضة لاستدعاءات/نتائج الأدوات مثل `[Tool Call: ...]`،
    و`[Tool Result ...]`، و`[Historical context ...]`
  - تُزال رموز التحكم بالنموذج المتسربة مثل `<|assistant|>`، وغيرها من
    رموز ASCII بالشكل `<|...|>`، ومتغيرات العرض الكامل `<｜...｜>`
  - تُزال XML استدعاءات أدوات MiniMax المشوهة مثل `<invoke ...>` /
    `</minimax:tool_call>`
- يُحجب أي نص يشبه بيانات الاعتماد/الرموز قبل إرجاعه
- تُقتطع الكتل النصية الطويلة
- قد تؤدي السجلات الكبيرة جدًا إلى إسقاط الصفوف الأقدم أو استبدال صف
  كبير الحجم بـ `[sessions_history omitted: message too large]`
- تُبلغ الأداة عن علامات تلخيصية مثل `truncated` و`droppedMessages` و
  `contentTruncated` و`contentRedacted` و`bytes`

تقبل كلتا الأداتين إما **مفتاح جلسة** (مثل `"main"`) أو **معرّف جلسة**
من استدعاء سرد سابق.

إذا كنت تحتاج إلى السجل الدقيق بايتًا مقابل بايت، فافحص
ملف السجل على القرص بدلًا من اعتبار `sessions_history` تفريغًا خامًا.

## إرسال رسائل عبر الجلسات

تسلّم `sessions_send` رسالة إلى جلسة أخرى وتنتظر الرد اختياريًا:

- **إرسال من دون انتظار:** اضبط `timeoutSeconds: 0` لوضع الرسالة في الطابور والعودة
  فورًا.
- **انتظار الرد:** اضبط مهلة واحصل على الرد مضمنًا.

بعد أن تستجيب الجلسة المستهدفة، يمكن لـ OpenClaw تشغيل **حلقة رد عكسي**
حيث يتناوب الوكلاء في تبادل الرسائل (حتى 5 أدوار). ويمكن للوكيل المستهدف أن يرد
`REPLY_SKIP` للتوقف مبكرًا.

## مساعدات الحالة والتنسيق

تُعد `session_status` أداة مكافئة خفيفة لـ `/status` للجلسة الحالية
أو لأي جلسة مرئية أخرى. وهي تعرض الاستخدام، والوقت، وحالة النموذج/وقت التشغيل،
وسياق مهام الخلفية المرتبط عند توفره. ومثل `/status`، يمكنها إكمال
عدادات الرموز/الذاكرة المؤقتة الشحيحة من أحدث إدخال استخدام في السجل، ويؤدي
`model=default` إلى مسح أي تجاوز لكل جلسة.

تنهي `sessions_yield` الدور الحالي عمدًا بحيث تكون الرسالة التالية
هي حدث المتابعة الذي تنتظره. استخدمها بعد إنشاء وكلاء فرعيين عندما
تريد وصول نتائج الإكمال كرسالة تالية بدلًا من بناء حلقات استطلاع.

تُعد `subagents` مساعد طبقة التحكم للوكلاء الفرعيين الموجودين أصلًا في OpenClaw.
وهي تدعم:

- `action: "list"` لفحص التشغيلات النشطة/الحديثة
- `action: "steer"` لإرسال توجيهات متابعة إلى child قيد التشغيل
- `action: "kill"` لإيقاف child واحد أو `all`

## إنشاء وكلاء فرعيين

تنشئ `sessions_spawn` جلسة معزولة لمهمة خلفية افتراضيًا.
وهي دائمًا غير حاجبة -- إذ تعود فورًا مع `runId` و
`childSessionKey`.

الخيارات الأساسية:

- `runtime: "subagent"` (الافتراضي) أو `"acp"` لوكلاء harness الخارجيين.
- تجاوزات `model` و`thinking` لجلسة child.
- `thread: true` لربط الإنشاء بسلسلة دردشة (Discord وSlack وغيرهما).
- `sandbox: "require"` لفرض sandbox على child.
- `context: "fork"` للوكلاء الفرعيين الأصليين عندما يحتاج child إلى
  سجل طالب الخدمة الحالي؛ احذفها أو استخدم `context: "isolated"` للحصول على child نظيف.

لا تحصل الوكلاء الفرعيون الورقيون الافتراضيون على أدوات الجلسة. وعندما
تكون `maxSpawnDepth >= 2`، تحصل الوكلاء الفرعيون المنسّقون في العمق 1 بالإضافة إلى
`sessions_spawn` و`subagents` و`sessions_list` و`sessions_history` حتى
يتمكنوا من إدارة children الخاصة بهم. أما التشغيلات الورقية فلا تزال لا تحصل على
أدوات تنسيق متكررة.

بعد الاكتمال، تنشر خطوة إعلان النتيجة في قناة الطالب.
ويحافظ تسليم الإكمال على التوجيه المرتبط بالسلسلة/الموضوع عند توفره، وإذا كان
أصل الإكمال يحدد قناة فقط، فلا يزال بإمكان OpenClaw إعادة استخدام
المسار المخزَّن في جلسة الطالب (`lastChannel` / `lastTo`) من أجل
التسليم المباشر.

للسلوك الخاص بـ ACP، راجع [وكلاء ACP](/ar/tools/acp-agents).

## الرؤية

تكون أدوات الجلسة محددة النطاق للحد مما يمكن للوكيل رؤيته:

| المستوى | النطاق |
| ------- | ------ |
| `self` | الجلسة الحالية فقط |
| `tree` | الجلسة الحالية + الوكلاء الفرعيون المنشأون |
| `agent` | جميع جلسات هذا الوكيل |
| `all` | جميع الجلسات (عبر الوكلاء إذا تم ضبط ذلك) |

القيمة الافتراضية هي `tree`. وتُقيَّد الجلسات المعزولة إلى `tree` بغض النظر عن
الإعدادات.
