تمنح OpenClaw الوكلاء أدوات للعمل عبر الجلسات، وفحص الحالة، و
تنسيق الوكلاء الفرعيين.

## الأدوات المتاحة

| الأداة              | ما الذي تفعله                                                               |
| ------------------ | --------------------------------------------------------------------------- |
| `sessions_list`    | تسرد الجلسات مع عوامل تصفية اختيارية (النوع، التصنيف، الوكيل، الحداثة، المعاينة) |
| `sessions_history` | تقرأ نص جلسة محددة                                                          |
| `sessions_send`    | ترسل رسالة إلى جلسة أخرى وتنتظر اختيارياً                                   |
| `sessions_spawn`   | تنشئ جلسة وكيل فرعي معزولة للعمل في الخلفية                                |
| `sessions_yield`   | تنهي الدور الحالي وتنتظر نتائج متابعة من الوكيل الفرعي                      |
| `subagents`        | تسرد حالة الوكلاء الفرعيين المنشأين لهذه الجلسة                             |
| `session_status`   | تعرض بطاقة بأسلوب `/status` وتعيّن اختيارياً تجاوزاً للنموذج لكل جلسة      |

تظل هذه الأدوات خاضعة لملف الأدوات النشط وسياسة السماح/المنع.
يتضمن `tools.profile: "coding"` مجموعة تنسيق الجلسات كاملة،
بما في ذلك `sessions_spawn` و`sessions_yield` و`subagents`.
ويتضمن `tools.profile: "messaging"` أدوات المراسلة عبر الجلسات
(`sessions_list` و`sessions_history` و`sessions_send` و`session_status`) لكنه
لا يتضمن إنشاء الوكلاء الفرعيين. للاحتفاظ بملف مراسلة مع السماح أيضاً
بالتفويض الأصلي، أضف:

```json5
{
  tools: {
    profile: "messaging",
    alsoAllow: ["sessions_spawn", "sessions_yield", "subagents"],
  },
}
```

لا يزال بإمكان سياسات المجموعة، والمزوّد، وبيئة العزل، وكل وكيل إزالة تلك الأدوات
بعد مرحلة الملف. استخدم `/tools` من الجلسة المتأثرة لفحص
قائمة الأدوات الفعلية.

## سرد الجلسات وقراءتها

يعيد `sessions_list` الجلسات مع مفتاحها، وagentId، والنوع، والقناة، والنموذج،
وعدادات الرموز، والطوابع الزمنية. صفّ حسب النوع (`main` و`group` و`cron` و`hook` و
`node`)، أو `label` مطابق، أو `agentId` مطابق، أو نص بحث، أو الحداثة
(`activeMinutes`). عندما تحتاج إلى فرز بأسلوب صندوق البريد، يمكنه أيضاً طلب
عنوان مشتق ضمن نطاق الرؤية، أو مقتطف معاينة لآخر رسالة، أو رسائل حديثة محدودة
في كل صف. لا تُنتج العناوين والمعاينات المشتقة إلا للجلسات التي يستطيع المستدعي
رؤيتها بالفعل ضمن سياسة رؤية أدوات الجلسة المكوّنة، لذلك تظل الجلسات غير ذات الصلة
مخفية. عندما تكون الرؤية مقيّدة، يعيد `sessions_list`
بيانات تعريف اختيارية `visibility` تعرض الوضع الفعلي وتحذيراً بأن
النتائج قد تكون محدودة النطاق.

يجلب `sessions_history` نص المحادثة لجلسة محددة.
افتراضياً، تُستثنى نتائج الأدوات -- مرّر `includeTools: true` لرؤيتها.
استخدم `limit` لأحدث ذيل محدود. مرّر `offset: 0` عندما تحتاج إلى
بيانات تعريف للترقيم، ثم مرّر قيم `nextOffset` المعادة للتنقل للخلف
عبر نوافذ نص OpenClaw الأقدم دون قراءة ملفات النص الخام.
لا تدمج صفحات الإزاحة الصريحة استيرادات الرجوع الاحتياطي الخارجية من CLI؛ استخدم
عرض أحدث ذيل الافتراضي عندما تحتاج إلى سجل العرض المدمج ذلك.
العرض المعاد محدود ومفلتر للسلامة عمداً:

- يُطبّع نص المساعد قبل الاستدعاء:
  - تزال وسوم التفكير
  - تزال كتل السقالات `<relevant-memories>` / `<relevant_memories>`
  - تزال كتل حمولات XML لاستدعاءات الأدوات كنص عادي مثل `<tool_call>...</tool_call>`،
    و`<function_call>...</function_call>`، و`<tool_calls>...</tool_calls>`، و
    `<function_calls>...</function_calls>`، بما في ذلك الحمولات المبتورة
    التي لا تُغلق بشكل سليم أبداً
  - تزال سقالات استدعاء/نتيجة الأداة المخفّضة مثل `[Tool Call: ...]`،
    و`[Tool Result ...]`، و`[Historical context ...]`
  - تزال رموز تحكم النموذج المسرّبة مثل `<|assistant|>`، ورموز ASCII الأخرى
    `<|...|>`، ومتغيرات العرض الكامل `<｜...｜>`
  - يزال XML غير السليم لاستدعاء أدوات MiniMax مثل `<invoke ...>` /
    `</minimax:tool_call>`
- يُنقّح النص الشبيه ببيانات الاعتماد/الرموز قبل إعادته
- تُقتطع كتل النص الطويلة
- يمكن للسجلات الكبيرة جداً إسقاط صفوف أقدم أو استبدال صف كبير جداً بـ
  `[sessions_history omitted: message too large]`
- تبلغ الأداة عن أعلام موجزة مثل `truncated` و`droppedMessages` و
  `contentTruncated` و`contentRedacted` و`bytes` وبيانات تعريف الترقيم

تقبل كلتا الأداتين إما **مفتاح جلسة** (مثل `"main"`) أو **معرّف جلسة**
من استدعاء قائمة سابق.

إذا كنت تحتاج إلى النص الدقيق بايتاً مقابل بايت، فافحص ملف النص على
القرص بدلاً من التعامل مع `sessions_history` كتفريغ خام.

## إرسال رسائل عبر الجلسات

يوصل `sessions_send` رسالة إلى جلسة أخرى وينتظر اختيارياً
الاستجابة:

- **أرسل وانسَ:** عيّن `timeoutSeconds: 0` للإدراج في الطابور والعودة
  فوراً.
- **انتظر الرد:** عيّن مهلة واحصل على الاستجابة ضمنياً.

جلسات الدردشة محددة النطاق بالخيط، مثل مفاتيح Slack أو Discord التي تنتهي بـ
`:thread:<id>`، ليست أهدافاً صالحة لـ `sessions_send`. استخدم مفتاح جلسة القناة
الأم لتنسيق ما بين الوكلاء حتى لا تظهر الرسائل الموجّهة بالأدوات
داخل خيط نشط مواجه للبشر.

تُعلّم الرسائل وردود متابعة A2A كبيانات بين الجلسات في
موجّه المستلم (`[Inter-session message ... isUser=false]`) وفي منشأ النص.
يجب أن يعاملها الوكيل المستلم كبيانات موجّهة بالأدوات، لا كتعليمة
مباشرة كتبها المستخدم النهائي.

بعد أن يستجيب الهدف، يمكن لـ OpenClaw تشغيل **حلقة رد عكسي** حيث
يتبادل الوكلاء الرسائل (حتى `session.agentToAgent.maxPingPongTurns`، بنطاق
0-20، والافتراضي 5). يمكن للوكيل الهدف الرد بـ
`REPLY_SKIP` للإيقاف مبكراً.

## مساعدات الحالة والتنسيق

`session_status` هي أداة خفيفة مكافئة لـ `/status` للجلسة الحالية
أو جلسة مرئية أخرى. تبلغ عن الاستخدام، والوقت، وحالة النموذج/وقت التشغيل، و
سياق مهمة الخلفية المرتبط عند وجوده. مثل `/status`، يمكنها ملء
عدادات الرموز/التخزين المؤقت المتناثرة من أحدث إدخال استخدام في النص، و
`model=default` يمسح تجاوز النموذج لكل جلسة. استخدم `sessionKey="current"` من أجل
جلسة المستدعي الحالية؛ التصنيفات المرئية للعميل مثل `openclaw-tui` ليست
مفاتيح جلسات.

عندما تكون بيانات تعريف التوجيه متاحة، يتضمن `session_status` أيضاً كتلة JSON مرئية
`Route context` وحقول `details` منظمة مطابقة. تزيل هذه
الحقول الالتباس بين مفتاح الجلسة والمسار الذي يتعامل حالياً مع
التشغيل المباشر:

- `origin` هو المكان الذي أُنشئت فيه الجلسة، أو المزوّد المستنتج من
  بادئة مفتاح جلسة قابلة للتسليم عندما تفتقر الحالة الأقدم إلى بيانات تعريف origin مخزنة.
- `active` هو مسار التشغيل المباشر الحالي. لا يُبلّغ عنه إلا للجلسة المباشرة أو
  الحالية التي يجري التعامل معها الآن.
- `deliveryContext` هو مسار التسليم المستمر المخزن في الجلسة،
  والذي يمكن لـ OpenClaw إعادة استخدامه للتسليم لاحقاً حتى عندما يختلف السطح
  النشط.

ينهي `sessions_yield` الدور الحالي عمداً حتى تكون الرسالة التالية
حدث المتابعة الذي تنتظره. استخدمه بعد إنشاء وكلاء فرعيين عندما
تريد وصول نتائج الإكمال كرسالة تالية بدلاً من بناء
حلقات استطلاع.

`subagents` هو مساعد الرؤية للوكلاء الفرعيين في OpenClaw الذين تم إنشاؤهم
بالفعل. يدعم `action: "list"` لفحص التشغيلات النشطة/الحديثة.

## إنشاء الوكلاء الفرعيين

ينشئ `sessions_spawn` جلسة معزولة لمهمة خلفية افتراضياً.
وهو دائماً غير حاجب -- يعود فوراً مع `runId` و
`childSessionKey`. تتلقى تشغيلات الوكيل الفرعي الأصلية المهمة المفوضة في
أول رسالة مرئية `[Subagent Task]` في جلسة الطفل، بينما يحمل موجّه النظام
قواعد وقت تشغيل الوكيل الفرعي وسياق التوجيه فقط.

خيارات رئيسية:

- `runtime: "subagent"` (افتراضي) أو `"acp"` لوكلاء أدوات التشغيل الخارجية.
- تجاوزات `model` و`thinking` لجلسة الطفل.
- `thread: true` لربط الإنشاء بخيط دردشة (Discord وSlack وما إلى ذلك).
- `sandbox: "require"` لفرض العزل على الطفل.
- `context: "fork"` للوكلاء الفرعيين الأصليين عندما يحتاج الطفل إلى نص
  الطالب الحالي؛ احذفه أو استخدم `context: "isolated"` لطفل نظيف.
  الوكلاء الفرعيون الأصليون المرتبطون بالخيوط يستخدمون افتراضياً `context: "fork"` ما لم
  يقل `threadBindings.defaultSpawnContext` خلاف ذلك.

لا تحصل الوكلاء الفرعيون الطرفيون الافتراضيون على أدوات الجلسة. عندما يكون
`maxSpawnDepth >= 2`، يتلقى وكلاء التنسيق الفرعيون في العمق 1 أيضاً
`sessions_spawn` و`subagents` و`sessions_list` و`sessions_history` حتى
يتمكنوا من إدارة أطفالهم. لا تزال التشغيلات الطرفية لا تحصل على أدوات
تنسيق تكرارية.

بعد الإكمال، تنشر خطوة إعلان النتيجة إلى قناة الطالب.
يحافظ تسليم الإكمال على توجيه الخيط/الموضوع المرتبط عند توفره، وإذا
كان origin الإكمال يحدد قناة فقط، فلا يزال بإمكان OpenClaw إعادة استخدام
المسار المخزن لجلسة الطالب (`lastChannel` / `lastTo`) للتسليم
المباشر.

للسلوك الخاص بـ ACP، راجع [وكلاء ACP](/ar/tools/acp-agents).

## الرؤية

تُقيّد أدوات الجلسة للحد مما يستطيع الوكيل رؤيته:

| المستوى | النطاق                                  |
| ------- | ---------------------------------------- |
| `self`  | الجلسة الحالية فقط                      |
| `tree`  | الجلسة الحالية + الوكلاء الفرعيون المنشؤون |
| `agent` | كل الجلسات لهذا الوكيل                  |
| `all`   | كل الجلسات (عبر الوكلاء إذا كانت مكوّنة) |

الافتراضي هو `tree`. تُثبّت الجلسات المعزولة على `tree` بغض النظر عن
الإعداد.
