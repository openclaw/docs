OpenClaw يشغّل **بيئة تشغيل وكيل مضمنة واحدة** — عملية وكيل واحدة لكل
Gateway، مع مساحة العمل الخاصة بها وملفات التمهيد ومخزن الجلسات. تغطي هذه الصفحة
عقد بيئة التشغيل هذا: ما الذي يجب أن تحتويه مساحة العمل، وأي الملفات تُحقن،
وكيف تمهّد الجلسات بالاستناد إليها.

## مساحة العمل (مطلوبة)

يستخدم OpenClaw دليل مساحة عمل وكيل واحدًا (`agents.defaults.workspace`) بوصفه دليل العمل **الوحيد** (`cwd`) للوكيل من أجل الأدوات والسياق.

موصى به: استخدم `openclaw setup` لإنشاء `~/.openclaw/openclaw.json` إذا كان مفقودًا وتهيئة ملفات مساحة العمل.

تخطيط مساحة العمل الكامل + دليل النسخ الاحتياطي: [مساحة عمل الوكيل](/ar/concepts/agent-workspace)

إذا كان `agents.defaults.sandbox` مفعّلًا، يمكن للجلسات غير الرئيسية تجاوز ذلك
بمساحات عمل لكل جلسة ضمن `agents.defaults.sandbox.workspaceRoot` (راجع
[تكوين Gateway](/ar/gateway/configuration)).

## ملفات التمهيد (محقونة)

داخل `agents.defaults.workspace`، يتوقع OpenClaw هذه الملفات القابلة للتحرير من المستخدم:

- `AGENTS.md` — تعليمات التشغيل + "الذاكرة"
- `SOUL.md` — الشخصية، والحدود، والنبرة
- `TOOLS.md` — ملاحظات أدوات يصونها المستخدم (مثل `imsg`، و`sag`، والاصطلاحات)
- `BOOTSTRAP.md` — إجراء تشغيل أولي لمرة واحدة (يُحذف بعد الاكتمال)
- `IDENTITY.md` — اسم الوكيل/الطابع/الرمز التعبيري
- `USER.md` — ملف المستخدم + طريقة المخاطبة المفضلة

في أول دور من جلسة جديدة، يحقن OpenClaw محتويات هذه الملفات في Project Context لموجّه النظام.

يتم تخطي الملفات الفارغة. تُقصّر الملفات الكبيرة وتُبتر مع علامة كي تبقى الموجّهات خفيفة (اقرأ الملف للاطلاع على المحتوى الكامل).

إذا كان ملف مفقودًا، يحقن OpenClaw سطر علامة "ملف مفقود" واحدًا (وسينشئ `openclaw setup` قالبًا افتراضيًا آمنًا).

لا يُنشأ `BOOTSTRAP.md` إلا من أجل **مساحة عمل جديدة تمامًا** (لا توجد ملفات تمهيد أخرى). أثناء انتظاره، يُبقيه OpenClaw في Project Context ويضيف إرشادات تمهيد في موجّه النظام للإجراء الأولي بدلًا من نسخه إلى رسالة المستخدم. إذا حذفته بعد إكمال الإجراء، فلا ينبغي أن يُعاد إنشاؤه عند عمليات إعادة التشغيل اللاحقة.

لتعطيل إنشاء ملفات التمهيد بالكامل (لمساحات العمل المجهزة مسبقًا)، اضبط:

```json5
{ agents: { defaults: { skipBootstrap: true } } }
```

## الأدوات المضمنة

الأدوات الأساسية (القراءة/التنفيذ/التحرير/الكتابة وأدوات النظام ذات الصلة) متاحة دائمًا،
وفقًا لسياسة الأدوات. `apply_patch` اختياري ومحكوم بواسطة
`tools.exec.applyPatch`. لا يتحكم `TOOLS.md` في الأدوات الموجودة؛ بل هو
إرشاد لكيفية رغبتك _أنت_ في استخدامها.

## Skills

يحمّل OpenClaw Skills من هذه المواقع (الأولوية الأعلى أولًا):

- مساحة العمل: `<workspace>/skills`
- Skills وكيل المشروع: `<workspace>/.agents/skills`
- Skills الوكيل الشخصية: `~/.agents/skills`
- المُدارة/المحلية: `~/.openclaw/skills`
- المضمّنة (المشحونة مع التثبيت)
- مجلدات Skills إضافية: `skills.load.extraDirs`

يمكن تقييد Skills بواسطة التكوين/البيئة (راجع `skills` في [تكوين Gateway](/ar/gateway/configuration)).

## حدود بيئة التشغيل

بيئة تشغيل الوكيل المضمنة مبنية على نواة وكيل Pi (النماذج، والأدوات، ومسار
الموجّهات). إدارة الجلسات، والاكتشاف، وربط الأدوات، وتسليم القنوات
هي طبقات يملكها OpenClaw فوق تلك النواة.

## الجلسات

تُخزّن نصوص الجلسات بصيغة JSONL في:

- `~/.openclaw/agents/<agentId>/sessions/<SessionId>.jsonl`

معرّف الجلسة ثابت ويختاره OpenClaw.
لا تُقرأ مجلدات الجلسات القديمة من أدوات أخرى.

## التوجيه أثناء البث

عندما يكون وضع الطابور `steer`، تُحقن الرسائل الواردة في التشغيل الحالي.
يُسلّم التوجيه في الطابور **بعد أن ينتهي دور المساعد الحالي من
تنفيذ استدعاءات أدواته**، وقبل استدعاء LLM التالي. يفرّغ Pi كل رسائل
التوجيه المعلّقة معًا من أجل `steer`؛ أما `queue` القديم فيفرّغ رسالة واحدة لكل
حدّ نموذج. لم يعد التوجيه يتخطى استدعاءات الأدوات المتبقية من رسالة
المساعد الحالية.

عندما يكون وضع الطابور `followup` أو `collect`، تُحتفظ بالرسائل الواردة حتى
ينتهي الدور الحالي، ثم يبدأ دور وكيل جديد مع الحمولات الموضوعة في الطابور. راجع
[الطابور](/ar/concepts/queue) و[طابور التوجيه](/ar/concepts/queue-steering) لسلوك الأوضاع
والحدود.

يرسل بث الكتل كتل المساعد المكتملة فور انتهائها؛ وهو
**متوقف افتراضيًا** (`agents.defaults.blockStreamingDefault: "off"`).
اضبط الحد عبر `agents.defaults.blockStreamingBreak` (`text_end` مقابل `message_end`؛ القيمة الافتراضية text_end).
تحكم في تجزئة الكتل اللينة باستخدام `agents.defaults.blockStreamingChunk` (القيمة الافتراضية
800–1200 حرف؛ يفضّل فواصل الفقرات، ثم الأسطر الجديدة؛ والجمل أخيرًا).
ادمج المقاطع المبثوثة باستخدام `agents.defaults.blockStreamingCoalesce` لتقليل
الرسائل أحادية السطر المتكررة (دمج قائم على الخمول قبل الإرسال). تتطلب القنوات غير Telegram
إعداد `*.blockStreaming: true` صراحةً لتفعيل ردود الكتل.
تُصدر ملخصات الأدوات المطوّلة عند بدء الأداة (بدون debounce)؛ وتبث Control UI
مخرجات الأدوات عبر أحداث الوكيل عند توفرها.
مزيد من التفاصيل: [البث + التجزئة](/ar/concepts/streaming).

## مراجع النماذج

تُحلّل مراجع النماذج في التكوين (مثل `agents.defaults.model` و`agents.defaults.models`) بالتقسيم عند أول `/`.

- استخدم `provider/model` عند تكوين النماذج.
- إذا كان معرّف النموذج نفسه يحتوي على `/` (بأسلوب OpenRouter)، فأدرج بادئة المزوّد (مثال: `openrouter/moonshotai/kimi-k2`).
- إذا حذفت المزوّد، يجرّب OpenClaw اسمًا مستعارًا أولًا، ثم مطابقة مزوّد مكوّن فريدة
  لمعرّف النموذج الدقيق ذاك، وبعدها فقط يرجع
  إلى المزوّد الافتراضي المكوّن. إذا لم يعد ذلك المزوّد يعرض
  النموذج الافتراضي المكوّن، يرجع OpenClaw إلى أول
  مزوّد/نموذج مكوّن بدلًا من إظهار افتراضي قديم لمزوّد مُزال.

## التكوين (الحد الأدنى)

كحد أدنى، اضبط:

- `agents.defaults.workspace`
- `channels.whatsapp.allowFrom` (موصى به بشدة)
