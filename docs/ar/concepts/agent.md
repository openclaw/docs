يشغّل OpenClaw **بيئة تشغيل وكيل مضمّنة واحدة** — عملية وكيل واحدة لكل
Gateway، مع مساحة العمل الخاصة بها، وملفات bootstrap الخاصة بها، ومخزن الجلسات الخاص بها. تغطي هذه الصفحة
عقد بيئة التشغيل هذا: ما الذي يجب أن تحتويه مساحة العمل، وما الملفات التي يتم حقنها، وكيف تُهيَّأ الجلسات الأولية اعتمادًا عليها.

## مساحة العمل (مطلوبة)

يستخدم OpenClaw دليل مساحة عمل وكيل واحدة (`agents.defaults.workspace`) بوصفه **الدليل العامل الوحيد** (`cwd`) للوكيل للأدوات والسياق.

الموصى به: استخدم `openclaw setup` لإنشاء `~/.openclaw/openclaw.json` إذا كان مفقودًا وتهيئة ملفات مساحة العمل.

دليل تخطيط مساحة العمل الكامل + النسخ الاحتياطي: [مساحة عمل الوكيل](/ar/concepts/agent-workspace)

إذا كان `agents.defaults.sandbox` مفعّلًا، فيمكن للجلسات غير الرئيسية تجاوز هذا باستخدام
مساحات عمل لكل جلسة تحت `agents.defaults.sandbox.workspaceRoot` (راجع
[إعداد Gateway](/ar/gateway/configuration)).

## ملفات Bootstrap (محقونة)

داخل `agents.defaults.workspace`، يتوقع OpenClaw هذه الملفات القابلة للتحرير من المستخدم:

- `AGENTS.md` — تعليمات التشغيل + “الذاكرة”
- `SOUL.md` — الشخصية، والحدود، والنبرة
- `TOOLS.md` — ملاحظات الأدوات التي يحتفظ بها المستخدم (مثل `imsg` و`sag` والاصطلاحات)
- `BOOTSTRAP.md` — طقس التشغيل الأول لمرة واحدة (يُحذف بعد الاكتمال)
- `IDENTITY.md` — اسم الوكيل/طابعه/رمزه التعبيري
- `USER.md` — ملف تعريف المستخدم + أسلوب المخاطبة المفضل

في أول دور من جلسة جديدة، يحقن OpenClaw محتويات هذه الملفات مباشرة في سياق الوكيل.

يتم تخطي الملفات الفارغة. تُقص الملفات الكبيرة وتُقتطع مع واسم حتى تبقى المطالبات خفيفة (اقرأ الملف للحصول على المحتوى الكامل).

إذا كان الملف مفقودًا، يحقن OpenClaw سطر واسم واحدًا يشير إلى “missing file” (كما أن `openclaw setup` سينشئ قالبًا افتراضيًا آمنًا).

يُنشأ `BOOTSTRAP.md` فقط من أجل **مساحة عمل جديدة تمامًا** (عند عدم وجود أي ملفات bootstrap أخرى). إذا حذفته بعد إكمال الطقس، فلا ينبغي إعادة إنشائه في عمليات إعادة التشغيل اللاحقة.

لتعطيل إنشاء ملفات bootstrap بالكامل (لمساحات العمل المزودة مسبقًا)، اضبط:

```json5
{ agent: { skipBootstrap: true } }
```

## الأدوات المضمّنة

تكون الأدوات الأساسية (read/exec/edit/write وأدوات النظام ذات الصلة) متاحة دائمًا،
رهناً بسياسة الأدوات. تكون `apply_patch` اختيارية وتخضع للتحكم عبر
`tools.exec.applyPatch`. لا يتحكم `TOOLS.md` في الأدوات الموجودة؛ بل هو
إرشاد لكيفية _رغبتك_ في استخدامها.

## Skills

يحمّل OpenClaw Skills من هذه المواقع (الأعلى أسبقية أولًا):

- مساحة العمل: `<workspace>/skills`
- Skills وكيل المشروع: `<workspace>/.agents/skills`
- Skills الوكيل الشخصية: `~/.agents/skills`
- المُدارة/المحلية: `~/.openclaw/skills`
- المضمّنة (التي تُشحن مع التثبيت)
- مجلدات Skills إضافية: `skills.load.extraDirs`

يمكن تقييد Skills عبر config/env (راجع `skills` في [إعداد Gateway](/ar/gateway/configuration)).

## حدود بيئة التشغيل

تُبنى بيئة تشغيل الوكيل المضمّنة على نواة وكيل Pi (النماذج، والأدوات، وخط أنابيب
المطالبات). أما إدارة الجلسات، والاكتشاف، وتوصيل الأدوات، وتسليم القنوات، فهي
طبقات يملكها OpenClaw فوق تلك النواة.

## الجلسات

تُخزَّن نصوص الجلسات بصيغة JSONL في:

- `~/.openclaw/agents/<agentId>/sessions/<SessionId>.jsonl`

يكون معرّف الجلسة ثابتًا ويختاره OpenClaw.
لا تتم قراءة مجلدات الجلسات القديمة من أدوات أخرى.

## التوجيه أثناء البث

عندما يكون وضع قائمة الانتظار هو `steer`، تُحقن الرسائل الواردة في التشغيل الحالي.
ويتم تسليم التوجيهات الموضوعة في قائمة الانتظار **بعد أن ينتهي دور المساعد الحالي
من تنفيذ استدعاءات أدواته**، وقبل استدعاء LLM التالي. لم يعد التوجيه يتخطى
استدعاءات الأدوات المتبقية من رسالة المساعد الحالية؛ بل يحقن الرسالة الموجودة في
قائمة الانتظار عند حد النموذج التالي.

عندما يكون وضع قائمة الانتظار هو `followup` أو `collect`، تُحتجز الرسائل الواردة حتى
ينتهي الدور الحالي، ثم يبدأ دور وكيل جديد مع الحمولات الموضوعة في قائمة الانتظار. راجع
[Queue](/ar/concepts/queue) لمعرفة سلوك الوضع + debounce/cap.

يرسل block streaming كتل المساعد المكتملة بمجرد انتهائها؛ وهو
**معطّل افتراضيًا** (`agents.defaults.blockStreamingDefault: "off"`).
اضبط الحد عبر `agents.defaults.blockStreamingBreak` (`text_end` مقابل `message_end`؛ والافتراضي text_end).
وتحكم في تقسيم block المرن عبر `agents.defaults.blockStreamingChunk` (الافتراضي
800–1200 حرفًا؛ ويفضل فواصل الفقرات، ثم الأسطر الجديدة، ثم الجمل أخيرًا).
واجمع القطع المتدفقة عبر `agents.defaults.blockStreamingCoalesce` لتقليل
الرسائل القصيرة أحادية السطر المزعجة (دمج قائم على الخمول قبل الإرسال). تتطلب القنوات غير
Telegram إعداد `*.blockStreaming: true` صراحةً لتمكين ردود block.
تُرسل الملخصات المفصلة للأدوات عند بدء الأداة (من دون debounce)؛ وتقوم Control UI
ببث خرج الأداة عبر أحداث الوكيل عند توفر ذلك.
مزيد من التفاصيل: [البث + التقسيم](/ar/concepts/streaming).

## مراجع النماذج

تُحلَّل مراجع النماذج في الإعداد (مثل `agents.defaults.model` و`agents.defaults.models`) بتقسيمها عند **أول** `/`.

- استخدم `provider/model` عند إعداد النماذج.
- إذا كان معرّف النموذج نفسه يحتوي على `/` (بنمط OpenRouter)، فأدرج بادئة المزوّد (مثال: `openrouter/moonshotai/kimi-k2`).
- إذا حذفت المزوّد، يحاول OpenClaw أولًا الاسم المستعار، ثم
  مطابقة مزوّد مُعدّ بشكل فريد لذلك المعرّف النموذجي الدقيق، ثم يعود فقط
  إلى المزوّد الافتراضي المُعدّ. وإذا لم يعد هذا المزوّد يوفّر
  النموذج الافتراضي المُعدّ، فإن OpenClaw يعود إلى أول
  `provider/model` مُعدّ بدلًا من إظهار افتراضي قديم لمزوّد أُزيل.
