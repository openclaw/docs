الميزات التجريبية في OpenClaw هي **أسطح معاينة اختيارية التفعيل**. وهي
خلف علامات صريحة لأنها ما زالت تحتاج إلى استخدام واقعي كاف قبل أن
تستحق إعدادًا افتراضيًا مستقرًا أو عقدًا عامًا طويل الأمد.

تعامل معها بطريقة مختلفة عن الإعدادات العادية:

- أبقها **معطلة افتراضيًا** ما لم يخبرك المستند المرتبط بتجربة إحداها.
- توقّع أن **يتغير الشكل والسلوك** أسرع من الإعدادات المستقرة.
- فضّل المسار المستقر أولًا عندما يكون موجودًا بالفعل.
- إذا كنت تنشر OpenClaw على نطاق واسع، فاختبر العلامات التجريبية في بيئة
  أصغر قبل تثبيتها ضمن خط أساس مشترك.

## وضع النماذج المحلية الخفيف

`agents.defaults.experimental.localModelLean: true` هو صمام لتخفيف الضغط في إعدادات النماذج المحلية الأضعف. عند تفعيله، يزيل OpenClaw ثلاث أدوات افتراضية — `browser` و`cron` و`message` — من سطح أدوات الوكيل في كل دورة. كما يجعل ذلك التشغيل يستخدم افتراضيًا عناصر تحكم البحث عن الأدوات المهيكلة عندما لا يكون `tools.toolSearch` مضبوطًا صراحة، بحيث تبقى كتالوجات أدوات Plugin أو MCP أو العميل الأكبر خلف `tool_search` و`tool_describe` و`tool_call` بدلًا من ضخها في الموجّه. التشغيلات التي تتطلب تسليم `message` المباشر تبقي تلك الأداة مباشرة بدلًا من تفعيل إعداد البحث عن الأدوات الافتراضي الخاص بالوضع الخفيف. استخدم `agents.list[].experimental.localModelLean` لتفعيل السلوك نفسه أو تعطيله لوكيل واحد مضبوط.

### لماذا هذه الأدوات الثلاث

لهذه الأدوات الثلاث أكبر الأوصاف وأكثر أشكال المعاملات في تشغيل OpenClaw الافتراضي. وعلى خلفية صغيرة السياق أو خلفية متوافقة مع OpenAI وأكثر صرامة، يكون ذلك هو الفارق بين:

- ملاءمة مخططات الأدوات داخل الموجّه بوضوح مقابل مزاحمة سجل المحادثة.
- اختيار النموذج للأداة الصحيحة مقابل إصدار استدعاءات أدوات مشوهة بسبب وجود عدد كبير جدًا من المخططات المتشابهة.
- بقاء محوّل Chat Completions داخل حدود الخادم للمخرجات المهيكلة مقابل التسبب بخطأ 400 بسبب حجم حمولة استدعاء الأداة.

إزالتها لا تعيد توصيل OpenClaw بصمت — بل تجعل قائمة الأدوات المباشرة أقصر فقط. ما يزال لدى النموذج أدوات `read` و`write` و`edit` و`exec` و`apply_patch`، والبحث/الجلب عبر الويب (عند ضبطه)، والذاكرة، وأدوات الجلسة/الوكيل متاحة. تبقى الكتالوجات الإضافية قابلة للاستدعاء عبر البحث عن الأدوات ما لم تضبط `tools.toolSearch: false` صراحة.

### متى تفعّله

فعّل الوضع الخفيف عندما تكون قد أثبتّ بالفعل أن النموذج يستطيع التحدث إلى Gateway لكن دورات الوكيل الكاملة تسيء التصرف. سلسلة الإشارات المعتادة هي:

1. ينجح `openclaw infer model run --gateway --model <ref> --prompt "Reply with exactly: pong"`.
2. تفشل دورة وكيل عادية بسبب استدعاءات أدوات مشوهة، أو موجّهات زائدة الحجم، أو تجاهل النموذج لأدواته.
3. يؤدي تبديل `localModelLean: true` إلى إزالة الفشل.

### متى تتركه معطلًا

إذا كانت الخلفية لديك تتعامل مع التشغيل الافتراضي الكامل بوضوح، فاترك هذا الخيار معطلًا. الوضع الخفيف حل التفافي، وليس إعدادًا افتراضيًا. وهو موجود لأن بعض الحِزم المحلية تحتاج إلى سطح أدوات أصغر كي تتصرف كما ينبغي؛ أما النماذج المستضافة والتجهيزات المحلية جيدة الموارد فلا تحتاج إليه.

كذلك لا يحل الوضع الخفيف محل `tools.profile` أو `tools.allow`/`tools.deny` أو مخرج الطوارئ `compat.supportsTools: false` الخاص بالنموذج. إذا كنت تحتاج إلى سطح أدوات أضيق دائمًا لوكيل محدد، ففضّل هذه المفاتيح المستقرة على العلامة التجريبية.

إذا كنت تضبط البحث عن الأدوات عالميًا بالفعل، فإن OpenClaw يترك إعدادات المشغّل تلك كما هي. اضبط `tools.toolSearch: false` لإلغاء إعداد البحث عن الأدوات الافتراضي الخاص بالوضع الخفيف.

### التفعيل

```json5
{
  agents: {
    defaults: {
      experimental: {
        localModelLean: true,
      },
    },
  },
}
```

لوكيل واحد فقط:

```json5
{
  agents: {
    list: [
      {
        id: "local",
        model: "lmstudio/gemma-4-e4b-it",
        experimental: {
          localModelLean: true,
        },
      },
    ],
  },
}
```

أعد تشغيل Gateway بعد تغيير العلامة، ثم أكّد قائمة الأدوات المختصرة باستخدام:

```bash
openclaw status --deep
```

يسرد خرج الحالة العميقة أدوات الوكيل النشطة؛ يجب أن تكون `browser` و`cron` و`message` غائبة عند تفعيل الوضع الخفيف ما لم يفرض وضع التسليم الحالي ردود `message` المباشرة.

## التجريبي لا يعني مخفيًا

إذا كانت ميزة ما تجريبية، فينبغي أن يقول OpenClaw ذلك بوضوح في المستندات وفي
مسار الإعداد نفسه. وما لا ينبغي فعله هو تهريب سلوك المعاينة إلى مفتاح افتراضي
يبدو مستقرًا والتظاهر بأن ذلك طبيعي. بهذه الطريقة تصبح أسطح الإعدادات فوضوية.
