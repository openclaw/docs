الميزات التجريبية هي واجهات معاينة اختيارية تُفعَّل عبر علامات صريحة. وتحتاج إلى مزيد من الاستخدام في ظروف واقعية قبل أن تحصل على إعداد افتراضي مستقر أو عقد طويل الأمد.

- تكون معطّلة افتراضيًا ما لم تطلب منك وثيقة تفعيل إحداها.
- يمكن أن يتغير شكلها وسلوكها بوتيرة أسرع من الإعدادات المستقرة.
- فضّل مسارًا مستقرًا عندما يكون متاحًا بالفعل.
- لا تطبّقها على نطاق واسع إلا بعد اختبارها أولًا في بيئة أصغر.

## الوضع الخفيف للنموذج المحلي

يؤدي ضبط `agents.defaults.experimental.localModelLean: true` إلى إزالة الأدوات الاختيارية الثقيلة من الواجهة المباشرة للوكيل في كل دورة: `browser` و`cron` و`message` و`image_generate` و`music_generate` و`video_generate` و`tts` و`pdf`. وتظل الأدوات المسموح بها صراحةً أو المطلوبة للتسليم متاحة، مع أن البحث عن الأدوات قد يفهرسها بدلًا من عرضها مباشرةً. كما يضبط الوضع الخفيف افتراضيًا أدلة Plugin وMCP والعميل لاستخدام البحث المنظّم عن الأدوات (`tool_search` و`tool_describe` و`tool_call`) عندما لا تكون قيمة `tools.toolSearch` مضبوطة بالفعل. استخدم `agents.list[].experimental.localModelLean` لقصر ذلك على وكيل واحد.

إذا كنت تضبط البحث عن الأدوات على مستوى عام بالفعل، فلن يغيّر OpenClaw ذلك الإعداد. اضبط `tools.toolSearch: false` لإلغاء الاشتراك في إعداد البحث عن الأدوات الافتراضي للوضع الخفيف.

في وضع `tools` المنظّم، تُبقي عمليات التشغيل الخفيفة `exec` ظاهرًا مباشرةً بجانب عناصر التحكم في البحث عن الأدوات، كي تظل النماذج المحلية المضبوطة للبرمجة قادرة على اختيار مسار الصدفة المألوف لديها. يغيّر هذا إمكانية رؤية المخطط فقط؛ إذ تظل سياسة الأدوات العادية، والعزل في صندوق الحماية، وموافقات التنفيذ سارية. وتحافظ أوضاع `code` و`directory` الصريحة على سلوك Compaction المعتاد.

### لماذا هذه الأدوات

تمتلك هذه الأدوات أطول الأوصاف، أو أوسع أشكال المعاملات، أو أعلى احتمال لتشتيت نموذج صغير عن المسار المعتاد للبرمجة والمحادثة. وفي واجهة خلفية ذات سياق صغير أو أكثر صرامة ومتوافقة مع OpenAI، يمثّل ذلك الفرق بين:

- احتواء موجّه الإدخال على مخططات الأدوات مقابل مزاحمتها لسجل المحادثة.
- اختيار النموذج للأداة الصحيحة مقابل إصدار استدعاءات أدوات غير صحيحة بسبب كثرة المخططات المتشابهة.
- بقاء محوّل Chat Completions ضمن حدود المخرجات المنظّمة مقابل ظهور خطأ 400 بسبب حجم حمولة استدعاء الأداة.

لا تؤدي إزالتها إلا إلى تقصير قائمة الأدوات المباشرة. يظل لدى النموذج `read` و`write` و`edit` و`exec` و`apply_patch` وفهم الصور والبحث/الجلب من الويب (عند إعدادهما) والذاكرة وأدوات الجلسة/الوكيل. وتظل الأدلة الإضافية قابلة للوصول عبر البحث عن الأدوات ما لم تضبط `tools.toolSearch: false`؛ ويمكن لسماحات الأدوات الصريحة إعادة إدخال وكيل خفيف في سير عمل مختصر.

### متى ينبغي تفعيله

فعّل الوضع الخفيف بعد أن تثبت قدرة النموذج على الاتصال بـ Gateway، مع استمرار سوء عمل دورات الوكيل الكاملة:

1. ينجح الأمر `openclaw infer model run --gateway --model <ref> --prompt "Reply with exactly: pong"`.
2. تفشل دورة وكيل عادية بسبب استدعاءات أدوات غير صحيحة، أو موجّهات إدخال مفرطة الحجم، أو تجاهل النموذج لأدواته.
3. يؤدي تبديل `localModelLean: true` إلى إزالة الفشل.

### متى ينبغي تركه معطّلًا

إذا كانت واجهتك الخلفية تتعامل بسلاسة مع وقت التشغيل الافتراضي الكامل، فاترك هذا الخيار معطّلًا. إنه حل بديل للحزم المحلية التي تحتاج إلى واجهة أدوات أصغر، وليس إعدادًا افتراضيًا للنماذج المستضافة أو التجهيزات المحلية ذات الموارد الوفيرة.

لا يحل الوضع الخفيف محل `tools.profile` أو `tools.allow`/`tools.deny` أو مخرج الطوارئ `compat.supportsTools: false` الخاص بالنموذج. للحصول على واجهة أدوات أضيق بصورة دائمة لوكيل محدد، فضّل عناصر التحكم المستقرة هذه.

### التفعيل

```json5
{
  agents: {
    defaults: {
      experimental: {
        localModelLean: true,
      },
    },
  },
}
```

لوكيل واحد فقط:

```json5
{
  agents: {
    list: [
      {
        id: "local",
        model: "lmstudio/gemma-4-e4b-it",
        experimental: {
          localModelLean: true,
        },
      },
    ],
  },
}
```

أعد تشغيل Gateway بعد تغيير العلامة. تزيل التصفية الخفيفة `browser` و`cron` و`message` و`image_generate` و`music_generate` و`video_generate` و`tts` و`pdf` ما لم تحتفظ بها صراحةً باستخدام `tools.allow` أو `tools.alsoAllow`؛ وقد يظل البحث عن الأدوات يفهرس الأدوات المحتفظ بها بدلًا من عرضها مباشرةً.

## تجريبي لا يعني مخفيًا

ينبغي توضيح أن الميزة تجريبية بشكل صريح في الوثائق وفي مسار الإعداد نفسه، لا إخفاؤها خلف عنصر تحكم افتراضي يبدو مستقرًا.
