تُرسل مؤشرات الكتابة إلى قناة الدردشة أثناء نشاط عملية التشغيل. استخدم `agents.defaults.typingMode` للتحكم في **وقت** بدء الكتابة و`typingIntervalSeconds` للتحكم في **معدل** تحديثها (وتيرة إبقاء الاتصال نشطًا، والقيمة الافتراضية 6 ثوانٍ).

## الإعدادات الافتراضية

عندما يكون `agents.defaults.typingMode` **غير معيّن**:

- **الدردشات المباشرة**: تبدأ الكتابة فور بدء حلقة النموذج.
- **الدردشات الجماعية التي تتضمن إشارة**: تبدأ الكتابة فورًا.
- **الدردشات الجماعية التي لا تتضمن إشارة**: تبدأ الكتابة عندما يظهر في عملية التشغيل المقبولة نشاط مرئي للمستخدم، مثل نشاط تنفيذ إطار التشغيل أو نص الرسالة.
- **عمليات تشغيل Heartbeat**: تبدأ الكتابة عند بدء عملية تشغيل Heartbeat، إذا كان هدف Heartbeat الذي جرى تحديده دردشة تدعم الكتابة ولم تكن الكتابة معطّلة.

## الأوضاع

عيّن `agents.defaults.typingMode` إلى أحد الخيارات التالية:

- `never` - لا يظهر مؤشر كتابة مطلقًا.
- `instant` - تبدأ الكتابة **بمجرد بدء حلقة النموذج**، حتى إذا لم تُرجع عملية التشغيل لاحقًا سوى رمز الرد الصامت.
- `thinking` - تبدأ الكتابة عند **أول تحديث جزئي للاستدلال**، أو عند تنفيذ إطار التشغيل النشط بعد قبول الدور.
- `message` - تبدأ الكتابة عند **أول نشاط رد مرئي للمستخدم**، مثل تنفيذ إطار التشغيل النشط أو تحديث نصي جزئي غير صامت. لا تُعد رموز الرد الصامت مثل `NO_REPLY` نشاطًا نصيًا.

الترتيب بحسب «مدى التبكير في التشغيل»: `never` -> `message`/`thinking` -> `instant`.

## الإعداد

عيّن القيمة الافتراضية على مستوى الوكيل:

```json5
{
  agents: {
    defaults: {
      typingMode: "thinking",
      typingIntervalSeconds: 6,
    },
  },
}
```

تجاوز الوضع أو الوتيرة لكل جلسة:

```json5
{
  session: {
    typingMode: "message",
    typingIntervalSeconds: 4,
  },
}
```

## ملاحظات

- لا يبدأ وضع `message` استجابةً لرموز الرد الصامت، لكن يمكن أن يُظهر التنفيذ النشط مؤشر الكتابة قبل توفر أي نص من المساعد.
- يظل `thinking` يستجيب للاستدلال المتدفق (`reasoningLevel: "stream"`)، ويمكنه أيضًا البدء نتيجة التنفيذ النشط قبل وصول التحديثات الجزئية للاستدلال.
- تمثل الكتابة أثناء Heartbeat إشارة إلى أن هدف التسليم المحدد يعمل. وهي تبدأ عند بدء عملية تشغيل Heartbeat بدلًا من اتباع توقيت تدفق `message` أو `thinking`. عيّن `typingMode: "never"` لتعطيلها.
- لا تعرض عمليات Heartbeat مؤشر الكتابة عندما يكون هدف Heartbeat هو `"none"`، أو عندما يتعذر تحديد الهدف، أو عندما يكون تسليم الدردشة معطّلًا لعملية Heartbeat، أو عندما لا تدعم القناة الكتابة.
- يتحكم `typingIntervalSeconds` في **وتيرة التحديث**، وليس وقت البدء. القيمة الافتراضية: 6 ثوانٍ.
