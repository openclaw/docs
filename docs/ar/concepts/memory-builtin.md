المحرك المضمّن هو الواجهة الخلفية الافتراضية للذاكرة. وهو يخزّن فهرس الذاكرة لديك في
قاعدة بيانات SQLite لكل وكيل ولا يحتاج إلى أي تبعيات إضافية للبدء.

## ما الذي يوفّره

- **البحث بالكلمات المفتاحية** عبر فهرسة النص الكامل FTS5 ‏(تسجيل BM25).
- **البحث المتجهي** عبر التضمينات من أي مزوّد مدعوم.
- **البحث الهجين** الذي يجمع بين الاثنين للحصول على أفضل النتائج.
- **دعم CJK** عبر تقسيم trigram للغات الصينية واليابانية والكورية.
- **تسريع sqlite-vec** لاستعلامات المتجهات داخل قاعدة البيانات (اختياري).

## البدء

إذا كان لديك مفتاح API لـ OpenAI أو Gemini أو Voyage أو Mistral، فسيكتشفه
المحرك المضمّن تلقائيًا ويُفعّل البحث المتجهي. ولا حاجة إلى أي تهيئة.

لتعيين مزوّد صراحةً:

```json5
{
  agents: {
    defaults: {
      memorySearch: {
        provider: "openai",
      },
    },
  },
}
```

من دون مزوّد تضمين، يتوفر فقط البحث بالكلمات المفتاحية.

لفرض مزوّد التضمين المحلي المضمّن، وجّه `local.modelPath` إلى
ملف GGUF:

```json5
{
  agents: {
    defaults: {
      memorySearch: {
        provider: "local",
        fallback: "none",
        local: {
          modelPath: "~/.node-llama-cpp/models/embeddinggemma-300m-qat-Q8_0.gguf",
        },
      },
    },
  },
}
```

## مزوّدو التضمين المدعومون

| المزوّد | المعرّف    | يُكتشف تلقائيًا | ملاحظات                              |
| ------- | ---------- | --------------- | ------------------------------------ |
| OpenAI  | `openai`   | نعم             | الافتراضي: `text-embedding-3-small` |
| Gemini  | `gemini`   | نعم             | يدعم الوسائط المتعددة (صورة + صوت)   |
| Voyage  | `voyage`   | نعم             |                                      |
| Mistral | `mistral`  | نعم             |                                      |
| Ollama  | `ollama`   | لا              | محلي، اضبطه صراحةً                  |
| Local   | `local`    | نعم (أولًا)     | نموذج GGUF، تنزيل بحجم ~0.6 GB       |

يختار الاكتشاف التلقائي أول مزوّد يمكن حل مفتاح API الخاص به، وفق
الترتيب المعروض. اضبط `memorySearch.provider` للتجاوز.

## كيف تعمل الفهرسة

يفهرس OpenClaw الملفين `MEMORY.md` و`memory/*.md` إلى أجزاء (~400 رمز مع
تداخل 80 رمزًا) ويخزّنها في قاعدة بيانات SQLite لكل وكيل.

- **موقع الفهرس:** `~/.openclaw/memory/<agentId>.sqlite`
- **مراقبة الملفات:** تؤدي التغييرات في ملفات الذاكرة إلى إعادة فهرسة مؤجلة (1.5 ثانية).
- **إعادة الفهرسة التلقائية:** عندما تتغير تهيئة مزوّد التضمين أو النموذج أو
  التقسيم إلى أجزاء، يُعاد بناء الفهرس بالكامل تلقائيًا.
- **إعادة الفهرسة عند الطلب:** `openclaw memory index --force`

<Info>
يمكنك أيضًا فهرسة ملفات Markdown خارج workspace باستخدام
`memorySearch.extraPaths`. راجع
[مرجع التهيئة](/ar/reference/memory-config#additional-memory-paths).
</Info>

## متى تستخدمه

المحرك المضمّن هو الاختيار المناسب لمعظم المستخدمين:

- يعمل مباشرةً دون أي تبعيات إضافية.
- يتعامل جيدًا مع البحث بالكلمات المفتاحية والبحث المتجهي.
- يدعم جميع مزوّدي التضمين.
- يجمع البحث الهجين بين أفضل ما في نهجي الاسترجاع.

فكّر في الانتقال إلى [QMD](/ar/concepts/memory-qmd) إذا كنت بحاجة إلى إعادة ترتيب النتائج أو
توسيع الاستعلام، أو إذا كنت تريد فهرسة دلائل خارج workspace.

وفكّر في [Honcho](/ar/concepts/memory-honcho) إذا كنت تريد ذاكرة عبر الجلسات مع
نمذجة تلقائية للمستخدم.

## استكشاف الأخطاء وإصلاحها

**هل تم تعطيل البحث في الذاكرة؟** تحقق من `openclaw memory status`. وإذا لم يُكتشف أي مزوّد،
فاضبط واحدًا صراحةً أو أضف مفتاح API.

**لم يُكتشف المزوّد المحلي؟** تأكد من أن المسار المحلي موجود وشغّل:

```bash
openclaw memory status --deep --agent main
openclaw memory index --force --agent main
```

تستخدم كل من أوامر CLI المستقلة وGateway معرّف المزوّد `local` نفسه.
إذا ضُبط المزوّد على `auto`، فستؤخذ التضمينات المحلية في الحسبان أولًا فقط
عندما يشير `memorySearch.local.modelPath` إلى ملف محلي موجود.

**هل النتائج قديمة؟** شغّل `openclaw memory index --force` لإعادة البناء. فقد
تفوت أداة المراقبة التغييرات في حالات نادرة.

**ألا يتم تحميل sqlite-vec؟** يعود OpenClaw تلقائيًا إلى تشابه جيب التمام داخل العملية.
تحقق من السجلات لمعرفة خطأ التحميل المحدد.

## التهيئة

للاطلاع على إعداد مزوّد التضمين، وضبط البحث الهجين (الأوزان، وMMR، والاضمحلال
الزمني)، وفهرسة الدُفعات، والذاكرة متعددة الوسائط، وsqlite-vec، والمسارات الإضافية،
وجميع مفاتيح التهيئة الأخرى، راجع
[مرجع تهيئة الذاكرة](/ar/reference/memory-config).
