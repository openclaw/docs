حلقة الوكيل هي التشغيل «الحقيقي» الكامل للوكيل: الاستقبال → تجميع السياق → استدلال النموذج →
تنفيذ الأدوات → الردود المتدفقة → الاستمرارية. إنها المسار المعتمد الذي يحول الرسالة
إلى إجراءات ورد نهائي، مع الحفاظ على اتساق حالة الجلسة.

في OpenClaw، الحلقة هي تشغيل واحد متسلسل لكل جلسة يصدر أحداث دورة الحياة والتدفق
بينما يفكر النموذج، ويستدعي الأدوات، ويدفق المخرجات. يشرح هذا المستند كيف تُربط هذه الحلقة الأصيلة
من البداية إلى النهاية.

## نقاط الدخول

- Gateway RPC: `agent` و`agent.wait`.
- CLI: أمر `agent`.

## كيف تعمل (على مستوى عال)

1. يتحقق `agent` RPC من المعلمات، ويحل الجلسة (sessionKey/sessionId)، ويحفظ بيانات تعريف الجلسة، ثم يعيد `{ runId, acceptedAt }` فورًا.
2. يشغل `agentCommand` الوكيل:
   - يحل إعدادات النموذج + التفكير/الإسهاب/التتبع الافتراضية
   - يحمل لقطة Skills
   - يستدعي `runEmbeddedPiAgent` (وقت تشغيل pi-agent-core)
   - يصدر **نهاية/خطأ دورة الحياة** إذا لم تصدر الحلقة المضمنة واحدًا
3. `runEmbeddedPiAgent`:
   - يسلسل عمليات التشغيل عبر قوائم انتظار لكل جلسة + عامة
   - يحل النموذج + ملف تعريف المصادقة ويبني جلسة pi
   - يشترك في أحداث pi ويدفق فروقات المساعد/الأداة
   - يفرض المهلة -> يجهض التشغيل إذا تم تجاوزها
   - بالنسبة إلى أدوار خادم تطبيق Codex، يجهض دورًا مقبولًا يتوقف عن إنتاج تقدم خادم التطبيق قبل حدث نهائي
   - يعيد الحمولات + بيانات تعريف الاستخدام
4. يربط `subscribeEmbeddedPiSession` أحداث pi-agent-core بتدفق `agent` في OpenClaw:
   - أحداث الأداة => `stream: "tool"`
   - فروقات المساعد => `stream: "assistant"`
   - أحداث دورة الحياة => `stream: "lifecycle"` (`phase: "start" | "end" | "error"`)
5. يستخدم `agent.wait` الدالة `waitForAgentRun`:
   - ينتظر **نهاية/خطأ دورة الحياة** لـ `runId`
   - يعيد `{ status: ok|error|timeout, startedAt, endedAt, error? }`

## قوائم الانتظار + التزامن

- تُسلسل عمليات التشغيل لكل مفتاح جلسة (مسار الجلسة) واختياريًا عبر مسار عام.
- يمنع هذا سباقات الأداة/الجلسة ويحافظ على اتساق سجل الجلسة.
- يمكن لقنوات المراسلة اختيار أوضاع قوائم الانتظار (جمع/توجيه/متابعة) التي تغذي نظام المسارات هذا.
  انظر [قائمة انتظار الأوامر](/ar/concepts/queue).
- كما تُحمى عمليات كتابة النصوص بسجل قفل كتابة للجلسة على ملف الجلسة. القفل
  واعٍ بالعمليات وقائم على الملفات، لذلك يلتقط الكتّاب الذين يتجاوزون قائمة الانتظار داخل العملية أو يأتون من
  عملية أخرى. ينتظر كتّاب نص الجلسة حتى `session.writeLock.acquireTimeoutMs`
  قبل الإبلاغ عن أن الجلسة مشغولة؛ الافتراضي هو `60000` ms.
- أقفال كتابة الجلسة غير قابلة لإعادة الدخول افتراضيًا. إذا كان مساعد يقصد تداخل الحصول على
  القفل نفسه مع الحفاظ على كاتب منطقي واحد، فيجب أن يختار ذلك صراحةً باستخدام
  `allowReentrant: true`.

## إعداد الجلسة + مساحة العمل

- تُحل مساحة العمل وتُنشأ؛ قد تعيد عمليات التشغيل المعزولة توجيهها إلى جذر مساحة عمل معزولة.
- تُحمّل Skills (أو يُعاد استخدامها من لقطة) وتُحقن في البيئة والموجه.
- تُحل ملفات التمهيد/السياق وتُحقن في تقرير موجه النظام.
- يُكتسب قفل كتابة الجلسة؛ يُفتح `SessionManager` ويُجهز قبل التدفق. أي
  مسار لاحق لإعادة كتابة النص، أو Compaction، أو الاقتطاع يجب أن يأخذ القفل نفسه قبل فتح
  ملف النص أو تعديله.

## تجميع الموجه + موجه النظام

- يُبنى موجه النظام من موجه OpenClaw الأساسي، وموجه Skills، وسياق التمهيد، والتجاوزات لكل تشغيل.
- تُفرض حدود النموذج المحددة واحتياطي رموز Compaction.
- انظر [موجه النظام](/ar/concepts/system-prompt) لمعرفة ما يراه النموذج.

## نقاط الخطاف (حيث يمكنك الاعتراض)

لدى OpenClaw نظاما خطافات:

- **الخطافات الداخلية** (خطافات Gateway): سكربتات مدفوعة بالأحداث للأوامر وأحداث دورة الحياة.
- **خطافات Plugin**: نقاط امتداد داخل دورة حياة الوكيل/الأداة وخط أنابيب Gateway.

### الخطافات الداخلية (خطافات Gateway)

- **`agent:bootstrap`**: يعمل أثناء بناء ملفات التمهيد قبل إنهاء موجه النظام.
  استخدم هذا لإضافة/إزالة ملفات سياق التمهيد.
- **خطافات الأوامر**: `/new` و`/reset` و`/stop` وأحداث أوامر أخرى (انظر مستند الخطافات).

انظر [الخطافات](/ar/automation/hooks) للإعداد والأمثلة.

### خطافات Plugin (دورة حياة الوكيل + Gateway)

تعمل هذه داخل حلقة الوكيل أو خط أنابيب Gateway:

- **`before_model_resolve`**: يعمل قبل الجلسة (من دون `messages`) لتجاوز المزوّد/النموذج بشكل حتمي قبل حل النموذج.
- **`before_prompt_build`**: يعمل بعد تحميل الجلسة (مع `messages`) لحقن `prependContext` أو `systemPrompt` أو `prependSystemContext` أو `appendSystemContext` قبل إرسال الموجه. استخدم `prependContext` للنص الديناميكي لكل دور وحقول سياق النظام للإرشادات الثابتة التي يجب أن تكون ضمن مساحة موجه النظام.
- **`before_agent_start`**: خطاف توافق قديم قد يعمل في أي من المرحلتين؛ فضّل الخطافات الصريحة أعلاه.
- **`before_agent_reply`**: يعمل بعد الإجراءات المضمنة وقبل استدعاء LLM، مما يتيح لـ Plugin المطالبة بالدور وإرجاع رد اصطناعي أو إسكات الدور بالكامل.
- **`agent_end`**: افحص قائمة الرسائل النهائية وبيانات تعريف التشغيل بعد الاكتمال.
- **`before_compaction` / `after_compaction`**: راقب أو علّق دورات Compaction.
- **`before_tool_call` / `after_tool_call`**: اعترض معلمات/نتائج الأداة.
- **`before_install`**: افحص نتائج الفحص المضمنة واحظر اختياريًا تثبيت Skills أو Plugin.
- **`tool_result_persist`**: حوّل نتائج الأدوات بشكل متزامن قبل كتابتها إلى نص جلسة مملوك لـ OpenClaw.
- **`message_received` / `message_sending` / `message_sent`**: خطافات الرسائل الواردة + الصادرة.
- **`session_start` / `session_end`**: حدود دورة حياة الجلسة.
- **`gateway_start` / `gateway_stop`**: أحداث دورة حياة Gateway.

قواعد قرار الخطاف لحمايات الصادر/الأداة:

- `before_tool_call`: `{ block: true }` نهائي ويوقف المعالجات ذات الأولوية الأقل.
- `before_tool_call`: `{ block: false }` بلا تأثير ولا يمحو حظرًا سابقًا.
- `before_install`: `{ block: true }` نهائي ويوقف المعالجات ذات الأولوية الأقل.
- `before_install`: `{ block: false }` بلا تأثير ولا يمحو حظرًا سابقًا.
- `message_sending`: `{ cancel: true }` نهائي ويوقف المعالجات ذات الأولوية الأقل.
- `message_sending`: `{ cancel: false }` بلا تأثير ولا يمحو إلغاءً سابقًا.

انظر [خطافات Plugin](/ar/plugins/hooks) لمعرفة API الخطاف وتفاصيل التسجيل.

قد تكيف الحاضنات هذه الخطافات بشكل مختلف. تحافظ حاضنة خادم تطبيق Codex على
خطافات Plugin في OpenClaw بوصفها عقد التوافق للأسطح المعكوسة الموثقة،
بينما تبقى خطافات Codex الأصلية آلية Codex منفصلة ذات مستوى أدنى.

## التدفق + الردود الجزئية

- تُدفق فروقات المساعد من pi-agent-core وتُصدر كأحداث `assistant`.
- يمكن لتدفق الكتل إصدار ردود جزئية إما عند `text_end` أو `message_end`.
- يمكن إصدار تدفق الاستدلال كتدفق منفصل أو كردود كتل.
- انظر [التدفق](/ar/concepts/streaming) لسلوك التقسيم وردود الكتل.

## تنفيذ الأدوات + أدوات المراسلة

- تُصدر أحداث بدء/تحديث/نهاية الأداة على تدفق `tool`.
- تُنظف نتائج الأدوات من حيث الحجم وحمولات الصور قبل التسجيل/الإصدار.
- تُتبع عمليات إرسال أداة المراسلة لقمع تأكيدات المساعد المكررة.

## تشكيل الرد + القمع

- تُجمع الحمولات النهائية من:
  - نص المساعد (والاستدلال الاختياري)
  - ملخصات الأدوات المضمنة (عند الإسهاب + السماح)
  - نص خطأ المساعد عندما يخطئ النموذج
- يُرشح الرمز الصامت الدقيق `NO_REPLY` / `no_reply` من الحمولات
  الصادرة.
- تُزال تكرارات أداة المراسلة من قائمة الحمولات النهائية.
- إذا لم تبق أي حمولات قابلة للعرض وحدث خطأ في أداة، يُصدر رد احتياطي لخطأ الأداة
  (ما لم تكن أداة مراسلة قد أرسلت بالفعل ردًا مرئيًا للمستخدم).

## Compaction + إعادة المحاولة

- يصدر Compaction التلقائي أحداث تدفق `compaction` ويمكن أن يؤدي إلى إعادة محاولة.
- عند إعادة المحاولة، تُعاد تهيئة المخازن المؤقتة داخل الذاكرة وملخصات الأدوات لتجنب المخرجات المكررة.
- انظر [Compaction](/ar/concepts/compaction) لمعرفة خط أنابيب Compaction.

## تدفقات الأحداث (اليوم)

- `lifecycle`: يصدره `subscribeEmbeddedPiSession` (وكاحتياط بواسطة `agentCommand`)
- `assistant`: فروقات متدفقة من pi-agent-core
- `tool`: أحداث أدوات متدفقة من pi-agent-core

## معالجة قناة الدردشة

- تُخزن فروقات المساعد مؤقتًا في رسائل دردشة `delta`.
- تُصدر رسالة دردشة `final` عند **نهاية/خطأ دورة الحياة**.

## المهلات

- افتراضي `agent.wait`: 30 ث (الانتظار فقط). تتجاوز المعلمة `timeoutMs` ذلك.
- وقت تشغيل الوكيل: افتراضي `agents.defaults.timeoutSeconds` هو 172800 ث (48 ساعة)؛ يُفرض في مؤقت الإجهاض الخاص بـ `runEmbeddedPiAgent`.
- وقت تشغيل Cron: يملك cron قيمة `timeoutSeconds` لدور الوكيل المعزول. يبدأ المجدول ذلك المؤقت عند بدء التنفيذ، ويجهض التشغيل الأساسي عند الموعد النهائي المكوّن، ثم يشغل تنظيفًا محدودًا قبل تسجيل المهلة حتى لا تبقي جلسة فرعية قديمة المسار عالقًا.
- تشخيصات حيوية الجلسة: مع تمكين التشخيصات، يصنف `diagnostics.stuckSessionWarnMs` جلسات `processing` الطويلة التي لا تملك ردًا أو أداة أو حالة أو كتلة أو تقدم ACP مرصودًا. تُبلغ عمليات التشغيل المضمنة النشطة، واستدعاءات النموذج، واستدعاءات الأدوات كـ `session.long_running`؛ ويُبلغ العمل النشط من دون تقدم حديث كـ `session.stalled`؛ أما `session.stuck` فمحجوز لمسك دفاتر الجلسة القديمة من دون عمل نشط، وهذا المسار وحده يحرر مسار الجلسة المتأثر حتى يتم تصريف عمل البدء الموجود في قائمة الانتظار. تتراجع تشخيصات `session.stuck` المتكررة بينما تبقى الجلسة من دون تغيير.
- مهلة خمول النموذج: يجهض OpenClaw طلب النموذج عندما لا تصل أي أجزاء استجابة قبل نافذة الخمول. توسع `models.providers.<id>.timeoutSeconds` مراقب الخمول هذا للمزوّدين المحليين/ذاتيّي الاستضافة البطيئين؛ وإلا يستخدم OpenClaw `agents.defaults.timeoutSeconds` عند تكوينه، مع سقف افتراضي قدره 120 ث. عمليات التشغيل المشغلة بواسطة Cron من دون مهلة نموذج أو وكيل صريحة تعطل مراقب الخمول وتعتمد على مهلة Cron الخارجية.
- مهلة طلب HTTP للمزوّد: تنطبق `models.providers.<id>.timeoutSeconds` على جلب HTTP للنموذج الخاص بذلك المزوّد، بما في ذلك الاتصال، والرؤوس، والجسم، ومهلة طلب SDK، ومعالجة إجهاض الجلب المحروس الكلية، ومراقب خمول تدفق النموذج. استخدم هذا للمزوّدين المحليين/ذاتيّي الاستضافة البطيئين مثل Ollama قبل رفع مهلة وقت تشغيل الوكيل بالكامل.

## أين يمكن أن تنتهي الأمور مبكرًا

- مهلة الوكيل (إجهاض)
- AbortSignal (إلغاء)
- انقطاع Gateway أو مهلة RPC
- مهلة `agent.wait` (انتظار فقط، لا توقف الوكيل)
