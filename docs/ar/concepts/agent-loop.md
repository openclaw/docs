الحلقة الوكيلية هي التشغيل "الحقيقي" الكامل للوكيل: الإدخال → تجميع السياق → استدلال النموذج →
تنفيذ الأدوات → بث الردود → الاستمرار. إنها المسار الموثوق الذي يحول الرسالة
إلى إجراءات ورد نهائي، مع الحفاظ على اتساق حالة الجلسة.

في OpenClaw، الحلقة هي تشغيل واحد ومتسلسل لكل جلسة يصدر أحداث دورة الحياة والبث
أثناء تفكير النموذج واستدعائه للأدوات وبثه للمخرجات. يشرح هذا المستند كيف تُوصّل تلك الحلقة الأصيلة
من البداية إلى النهاية.

## نقاط الدخول

- Gateway RPC: `agent` و`agent.wait`.
- CLI: أمر `agent`.

## كيف يعمل (على مستوى عال)

1. يتحقق `agent` RPC من المعاملات، ويحل الجلسة (`sessionKey`/`sessionId`)، ويحفظ بيانات تعريف الجلسة، ثم يعيد `{ runId, acceptedAt }` فورًا.
2. يشغّل `agentCommand` الوكيل:
   - يحل النموذج + الإعدادات الافتراضية للتفكير/الإسهاب/التتبع
   - يحمّل لقطة Skills
   - يستدعي `runEmbeddedPiAgent` (وقت تشغيل `pi-agent-core`)
   - يصدر **نهاية/خطأ دورة الحياة** إذا لم تصدر الحلقة المضمّنة واحدًا
3. `runEmbeddedPiAgent`:
   - يسلسل عمليات التشغيل عبر قوائم انتظار لكل جلسة + عامة
   - يحل النموذج + ملف تعريف المصادقة ويبني جلسة Pi
   - يشترك في أحداث Pi ويبث فروقات المساعد/الأداة
   - يفرض المهلة -> يجهض التشغيل إذا تم تجاوزها
   - في أدوار خادم تطبيق Codex، يجهض الدور المقبول الذي يتوقف عن إنتاج تقدم لخادم التطبيق قبل حدث نهائي
   - يعيد الحمولات + بيانات تعريف الاستخدام
4. يربط `subscribeEmbeddedPiSession` أحداث `pi-agent-core` ببث `agent` في OpenClaw:
   - أحداث الأداة => `stream: "tool"`
   - فروقات المساعد => `stream: "assistant"`
   - أحداث دورة الحياة => `stream: "lifecycle"` (`phase: "start" | "end" | "error"`)
5. يستخدم `agent.wait` الدالة `waitForAgentRun`:
   - ينتظر **نهاية/خطأ دورة الحياة** لـ `runId`
   - يعيد `{ status: ok|error|timeout, startedAt, endedAt, error? }`

## قوائم الانتظار + التزامن

- تُسلسل عمليات التشغيل لكل مفتاح جلسة (مسار الجلسة) واختياريًا عبر مسار عام.
- يمنع هذا تسابق الأدوات/الجلسات ويحافظ على اتساق سجل الجلسة.
- يمكن لقنوات المراسلة اختيار أوضاع قائمة الانتظار (جمع/توجيه/متابعة) التي تغذي نظام المسارات هذا.
  راجع [قائمة انتظار الأوامر](/ar/concepts/queue).
- تُحمى كتابات السجل النصي أيضًا بقفل كتابة الجلسة على ملف الجلسة. القفل
  واعٍ بالعملية ومبني على الملفات، لذلك يلتقط الكتّاب الذين يتجاوزون قائمة الانتظار داخل العملية أو يأتون من
  عملية أخرى. ينتظر كتّاب سجل الجلسة النصي حتى `session.writeLock.acquireTimeoutMs`
  قبل الإبلاغ عن أن الجلسة مشغولة؛ القيمة الافتراضية هي `60000` مللي ثانية.
- أقفال كتابة الجلسة غير قابلة لإعادة الدخول افتراضيًا. إذا تعمد مساعد تداخل الحصول على
  القفل نفسه مع الحفاظ على كاتب منطقي واحد، فيجب أن يفعّل ذلك صراحةً باستخدام
  `allowReentrant: true`.

## إعداد الجلسة + مساحة العمل

- تُحل مساحة العمل وتُنشأ؛ قد تعيد عمليات التشغيل المعزولة توجيهها إلى جذر مساحة عمل معزولة.
- تُحمّل Skills (أو يُعاد استخدامها من لقطة) وتُحقن في البيئة والموجه.
- تُحل ملفات التمهيد/السياق وتُحقن في تقرير موجه النظام.
- يُكتسب قفل كتابة الجلسة؛ ويُفتح `SessionManager` ويُحضّر قبل البث. يجب على أي
  مسار لاحق لإعادة كتابة السجل النصي، أو Compaction، أو الاقتطاع أن يأخذ القفل نفسه قبل فتح
  ملف السجل النصي أو تعديله.

## تجميع الموجه + موجه النظام

- يُبنى موجه النظام من موجه OpenClaw الأساسي، وموجه Skills، وسياق التمهيد، والتجاوزات لكل تشغيل.
- تُفرض حدود النموذج المحددة واحتياطي رموز Compaction.
- راجع [موجه النظام](/ar/concepts/system-prompt) لمعرفة ما يراه النموذج.

## نقاط الخطاف (حيث يمكنك الاعتراض)

لدى OpenClaw نظاما خطافات:

- **الخطافات الداخلية** (خطافات Gateway): سكربتات مدفوعة بالأحداث للأوامر وأحداث دورة الحياة.
- **خطافات Plugin**: نقاط امتداد داخل دورة حياة الوكيل/الأداة ومسار Gateway.

### الخطافات الداخلية (خطافات Gateway)

- **`agent:bootstrap`**: يعمل أثناء بناء ملفات التمهيد قبل إنهاء موجه النظام.
  استخدمه لإضافة/إزالة ملفات سياق التمهيد.
- **خطافات الأوامر**: `/new` و`/reset` و`/stop` وأحداث الأوامر الأخرى (راجع مستند الخطافات).

راجع [الخطافات](/ar/automation/hooks) للإعداد والأمثلة.

### خطافات Plugin (دورة حياة الوكيل + Gateway)

تعمل هذه داخل حلقة الوكيل أو مسار Gateway:

- **`before_model_resolve`**: يعمل قبل الجلسة (بدون `messages`) لتجاوز المزوّد/النموذج بشكل حتمي قبل حل النموذج.
- **`before_prompt_build`**: يعمل بعد تحميل الجلسة (مع `messages`) لحقن `prependContext` أو `systemPrompt` أو `prependSystemContext` أو `appendSystemContext` قبل إرسال الموجه. استخدم `prependContext` للنص الديناميكي لكل دور وحقول سياق النظام للإرشاد المستقر الذي ينبغي أن يكون في مساحة موجه النظام.
- **`before_agent_start`**: خطاف توافق قديم قد يعمل في أي من المرحلتين؛ فضّل الخطافات الصريحة أعلاه.
- **`before_agent_reply`**: يعمل بعد الإجراءات المضمّنة وقبل استدعاء LLM، مما يتيح لـ Plugin المطالبة بالدور وإرجاع رد اصطناعي أو إسكات الدور بالكامل.
- **`agent_end`**: افحص قائمة الرسائل النهائية وبيانات تعريف التشغيل بعد الاكتمال.
- **`before_compaction` / `after_compaction`**: راقب دورات Compaction أو علّق عليها.
- **`before_tool_call` / `after_tool_call`**: اعترض معاملات/نتائج الأدوات.
- **`before_install`**: افحص نتائج الفحص المدمجة واحظر اختياريًا تثبيت Skills أو Plugin.
- **`tool_result_persist`**: حوّل نتائج الأدوات بشكل متزامن قبل كتابتها إلى سجل نصي لجلسة مملوك لـ OpenClaw.
- **`message_received` / `message_sending` / `message_sent`**: خطافات الرسائل الواردة + الصادرة.
- **`session_start` / `session_end`**: حدود دورة حياة الجلسة.
- **`gateway_start` / `gateway_stop`**: أحداث دورة حياة Gateway.

قواعد قرارات الخطافات لحواجز الصادر/الأدوات:

- `before_tool_call`: يكون `{ block: true }` نهائيًا ويوقف المعالجات ذات الأولوية الأدنى.
- `before_tool_call`: يكون `{ block: false }` بلا أثر ولا يمحو حظرًا سابقًا.
- `before_install`: يكون `{ block: true }` نهائيًا ويوقف المعالجات ذات الأولوية الأدنى.
- `before_install`: يكون `{ block: false }` بلا أثر ولا يمحو حظرًا سابقًا.
- `message_sending`: يكون `{ cancel: true }` نهائيًا ويوقف المعالجات ذات الأولوية الأدنى.
- `message_sending`: يكون `{ cancel: false }` بلا أثر ولا يمحو إلغاءً سابقًا.

راجع [خطافات Plugin](/ar/plugins/hooks) لتفاصيل واجهة برمجة تطبيقات الخطاف والتسجيل.

قد تكيّف أدوات الاختبار هذه الخطافات بطرق مختلفة. يحافظ إطار خادم تطبيق Codex على
خطافات Plugin في OpenClaw باعتبارها عقد التوافق للأسطح المعكوسة الموثقة،
بينما تبقى خطافات Codex الأصلية آلية Codex منفصلة ذات مستوى أدنى.

## البث + الردود الجزئية

- تُبث فروقات المساعد من `pi-agent-core` وتصدر كأحداث `assistant`.
- يمكن لبث الكتل إصدار ردود جزئية إما عند `text_end` أو `message_end`.
- يمكن إصدار بث الاستدلال كبث منفصل أو كردود كتل.
- راجع [البث](/ar/concepts/streaming) لسلوك التقسيم إلى أجزاء وردود الكتل.

## تنفيذ الأدوات + أدوات المراسلة

- تصدر أحداث بدء/تحديث/نهاية الأداة على بث `tool`.
- تُنقّى نتائج الأدوات من حيث الحجم وحمولات الصور قبل التسجيل/الإصدار.
- تُتبع عمليات إرسال أدوات المراسلة لمنع تأكيدات المساعد المكررة.

## تشكيل الرد + الكبت

- تُجمّع الحمولات النهائية من:
  - نص المساعد (والاستدلال الاختياري)
  - ملخصات الأدوات المضمّنة (عند السماح بالإسهاب + تمكينه)
  - نص خطأ المساعد عندما يخطئ النموذج
- تُرشح الرمز الصامت الدقيق `NO_REPLY` / `no_reply` من الحمولات
  الصادرة.
- تُزال تكرارات أدوات المراسلة من قائمة الحمولات النهائية.
- إذا لم تبق أي حمولات قابلة للعرض وحدث خطأ في أداة، يصدر رد احتياطي لخطأ الأداة
  (ما لم تكن أداة مراسلة قد أرسلت بالفعل ردًا مرئيًا للمستخدم).

## Compaction + إعادة المحاولة

- يصدر Compaction التلقائي أحداث بث `compaction` ويمكنه تشغيل إعادة محاولة.
- عند إعادة المحاولة، تُعاد تهيئة المخازن المؤقتة في الذاكرة وملخصات الأدوات لتجنب تكرار الإخراج.
- راجع [Compaction](/ar/concepts/compaction) لمعرفة مسار Compaction.

## بثوث الأحداث (اليوم)

- `lifecycle`: يصدره `subscribeEmbeddedPiSession` (وكاحتياط بواسطة `agentCommand`)
- `assistant`: فروقات مبثوثة من `pi-agent-core`
- `tool`: أحداث أدوات مبثوثة من `pi-agent-core`

## معالجة قناة الدردشة

- تُخزن فروقات المساعد مؤقتًا في رسائل دردشة `delta`.
- يصدر `final` للدردشة عند **نهاية/خطأ دورة الحياة**.

## المهل

- القيمة الافتراضية لـ `agent.wait`: 30 ثانية (للانتظار فقط). يتجاوز معامل `timeoutMs` ذلك.
- وقت تشغيل الوكيل: القيمة الافتراضية لـ `agents.defaults.timeoutSeconds` هي 172800 ثانية (48 ساعة)؛ تُفرض في مؤقت إجهاض `runEmbeddedPiAgent`.
- وقت تشغيل Cron: تمتلك Cron قيمة `timeoutSeconds` لدور الوكيل المعزول. يبدأ المجدول ذلك المؤقت عند بدء التنفيذ، ويجهض التشغيل الأساسي عند الموعد النهائي المضبوط، ثم يشغّل تنظيفًا محدودًا قبل تسجيل انتهاء المهلة بحيث لا تبقي جلسة فرعية قديمة المسار عالقًا.
- تشخيصات حيوية الجلسة: مع تمكين التشخيصات، يصنف `diagnostics.stuckSessionWarnMs` جلسات `processing` الطويلة التي ليس لديها أي رد أو أداة أو حالة أو كتلة أو تقدم ACP مرصود. تُبلغ عمليات التشغيل المضمّنة النشطة، واستدعاءات النموذج، واستدعاءات الأدوات كـ `session.long_running`؛ ويُبلغ العمل النشط بلا تقدم حديث كـ `session.stalled`؛ أما `session.stuck` فمحجوز لمسك دفاتر الجلسات القديمة بلا عمل نشط. يحرر مسك دفاتر الجلسات القديمة مسار الجلسة المتأثر فورًا؛ ولا تُجهض عمليات التشغيل المضمّنة المتوقفة وتُفرّغ إلا بعد نافذة مطولة بلا تقدم (10 دقائق على الأقل و5 أضعاف عتبة التحذير) كي يُستأنف العمل المصطف دون قطع عمليات تشغيل بطيئة فحسب. تتراجع تشخيصات `session.stuck` المتكررة بينما تبقى الجلسة بلا تغيير.
- مهلة خمول النموذج: يجهض OpenClaw طلب النموذج عندما لا تصل أي أجزاء استجابة قبل نافذة الخمول. يمدد `models.providers.<id>.timeoutSeconds` مراقب الخمول هذا للمزوّدين المحليين/المستضافين ذاتيًا البطيئين؛ وإلا يستخدم OpenClaw `agents.defaults.timeoutSeconds` عندما يكون مضبوطًا، مع حد افتراضي أقصى قدره 120 ثانية. تعطّل عمليات التشغيل المشغلة بواسطة Cron من دون مهلة صريحة للنموذج أو الوكيل مراقب الخمول وتعتمد على مهلة Cron الخارجية.
- مهلة طلب HTTP للمزوّد: ينطبق `models.providers.<id>.timeoutSeconds` على جلب HTTP لنموذج ذلك المزوّد، بما يشمل الاتصال، والترويسات، والجسم، ومهلة طلب SDK، ومعالجة إجهاض الجلب المحروس الكلية، ومراقب خمول بث النموذج. استخدم هذا للمزوّدين المحليين/المستضافين ذاتيًا البطيئين مثل Ollama قبل رفع مهلة وقت تشغيل الوكيل بالكامل.

## أين يمكن أن تنتهي الأمور مبكرًا

- مهلة الوكيل (إجهاض)
- AbortSignal (إلغاء)
- انقطاع Gateway أو مهلة RPC
- مهلة `agent.wait` (انتظار فقط، لا توقف الوكيل)
