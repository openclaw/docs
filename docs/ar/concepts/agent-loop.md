الحلقة الوكيلية هي التشغيل الكامل "الحقيقي" للوكيل: الإدخال → تجميع السياق → استدلال النموذج →
تنفيذ الأدوات → بث الردود → الاستمرارية. إنها المسار المعتمد الذي يحوّل الرسالة
إلى إجراءات ورد نهائي، مع الحفاظ على اتساق حالة الجلسة.

في OpenClaw، الحلقة هي تشغيل واحد ومتسلسل لكل جلسة، يصدر أحداث دورة الحياة والبث
بينما يفكر النموذج، ويستدعي الأدوات، ويبث المخرجات. يشرح هذا المستند كيف تُوصل تلك الحلقة الأصلية
من البداية إلى النهاية.

## نقاط الدخول

- Gateway RPC: `agent` و`agent.wait`.
- CLI: أمر `agent`.

## كيف يعمل ذلك (على مستوى عال)

1. يتحقق `agent` RPC من المعاملات، ويحل الجلسة (sessionKey/sessionId)، ويحفظ بيانات تعريف الجلسة، ثم يعيد `{ runId, acceptedAt }` فورًا.
2. يشغل `agentCommand` الوكيل:
   - يحل النموذج + الإعدادات الافتراضية للتفكير/الإسهاب/التتبع
   - يحمّل لقطة Skills
   - يستدعي `runEmbeddedPiAgent` (وقت تشغيل pi-agent-core)
   - يصدر **نهاية/خطأ دورة الحياة** إذا لم تصدر الحلقة المضمنة ذلك
3. `runEmbeddedPiAgent`:
   - يسلسل التشغيلات عبر طوابير لكل جلسة + طوابير عامة
   - يحل النموذج + ملف تعريف المصادقة ويبني جلسة Pi
   - يشترك في أحداث Pi ويبث دلتا المساعد/الأداة
   - يفرض المهلة -> يلغي التشغيل إذا تم تجاوزها
   - بالنسبة لدورات خادم تطبيق Codex، يلغي دورة مقبولة تتوقف عن إنتاج تقدم خادم التطبيق قبل حدث نهائي
   - يعيد الحمولات + بيانات تعريف الاستخدام
4. يربط `subscribeEmbeddedPiSession` أحداث pi-agent-core ببث `agent` في OpenClaw:
   - أحداث الأدوات => `stream: "tool"`
   - دلتا المساعد => `stream: "assistant"`
   - أحداث دورة الحياة => `stream: "lifecycle"` (`phase: "start" | "end" | "error"`)
5. يستخدم `agent.wait` الدالة `waitForAgentRun`:
   - ينتظر **نهاية/خطأ دورة الحياة** لـ `runId`
   - يعيد `{ status: ok|error|timeout, startedAt, endedAt, error? }`

## وضع الطوابير + التزامن

- تُسلسل التشغيلات لكل مفتاح جلسة (مسار الجلسة) واختياريًا عبر مسار عام.
- يمنع ذلك سباقات الأدوات/الجلسات ويحافظ على اتساق سجل الجلسة.
- يمكن لقنوات المراسلة اختيار أوضاع الطابور (جمع/توجيه/متابعة) التي تغذي نظام المسارات هذا.
  راجع [طابور الأوامر](/ar/concepts/queue).
- تتم أيضًا حماية عمليات كتابة السجل عبر قفل كتابة للجلسة على ملف الجلسة. القفل
  واعٍ بالعملية وقائم على الملفات، لذلك يلتقط الكتّاب الذين يتجاوزون الطابور داخل العملية أو يأتون من
  عملية أخرى. ينتظر كتّاب سجل الجلسة حتى `session.writeLock.acquireTimeoutMs`
  قبل الإبلاغ عن أن الجلسة مشغولة؛ القيمة الافتراضية هي `60000` مللي ثانية.
- أقفال كتابة الجلسة غير قابلة لإعادة الدخول افتراضيًا. إذا كان مساعد ما يتداخل عمدًا في الحصول على
  القفل نفسه مع الحفاظ على كاتب منطقي واحد، فيجب أن يفعّل ذلك صراحةً باستخدام
  `allowReentrant: true`.

## إعداد الجلسة + مساحة العمل

- تُحل مساحة العمل وتُنشأ؛ وقد تعيد التشغيلات المعزولة توجيهها إلى جذر مساحة عمل معزولة.
- تُحمّل Skills (أو يُعاد استخدامها من لقطة) وتُحقن في البيئة والموجه.
- تُحل ملفات التمهيد/السياق وتُحقن في تقرير موجه النظام.
- يُكتسب قفل كتابة للجلسة؛ ويُفتح `SessionManager` ويُجهز قبل البث. أي
  مسار لاحق لإعادة كتابة السجل أو Compaction أو الاقتطاع يجب أن يحصل على القفل نفسه قبل فتح
  ملف السجل أو تعديله.

## تجميع الموجه + موجه النظام

- يُبنى موجه النظام من موجه OpenClaw الأساسي، وموجه Skills، وسياق التمهيد، وتجاوزات كل تشغيل.
- تُفرض حدود النموذج المحددة واحتياطي رموز Compaction.
- راجع [موجه النظام](/ar/concepts/system-prompt) لمعرفة ما يراه النموذج.

## نقاط الخطاف (حيث يمكنك الاعتراض)

لدى OpenClaw نظاما خطافات:

- **الخطافات الداخلية** (خطافات Gateway): سكربتات مدفوعة بالأحداث للأوامر وأحداث دورة الحياة.
- **خطافات Plugin**: نقاط توسيع داخل دورة حياة الوكيل/الأداة ومسار Gateway.

### الخطافات الداخلية (خطافات Gateway)

- **`agent:bootstrap`**: يعمل أثناء بناء ملفات التمهيد قبل إنهاء موجه النظام.
  استخدمه لإضافة/إزالة ملفات سياق التمهيد.
- **خطافات الأوامر**: `/new`، و`/reset`، و`/stop`، وأحداث أوامر أخرى (راجع مستند الخطافات).

راجع [الخطافات](/ar/automation/hooks) للإعداد والأمثلة.

### خطافات Plugin (دورة حياة الوكيل + Gateway)

تعمل هذه داخل حلقة الوكيل أو مسار Gateway:

- **`before_model_resolve`**: يعمل قبل الجلسة (بلا `messages`) لتجاوز المزوّد/النموذج بشكل حتمي قبل حل النموذج.
- **`before_prompt_build`**: يعمل بعد تحميل الجلسة (مع `messages`) لحقن `prependContext`، أو `systemPrompt`، أو `prependSystemContext`، أو `appendSystemContext` قبل إرسال الموجه. استخدم `prependContext` للنص الديناميكي لكل دورة، وحقول سياق النظام للإرشادات الثابتة التي يجب أن تكون في مساحة موجه النظام.
- **`before_agent_start`**: خطاف توافق قديم قد يعمل في أي من المرحلتين؛ فضّل الخطافات الصريحة أعلاه.
- **`before_agent_reply`**: يعمل بعد الإجراءات المضمنة وقبل استدعاء LLM، مما يتيح لـ Plugin المطالبة بالدورة وإرجاع رد اصطناعي أو إسكات الدورة بالكامل.
- **`agent_end`**: يفحص قائمة الرسائل النهائية وبيانات تعريف التشغيل بعد الاكتمال.
- **`before_compaction` / `after_compaction`**: يراقب دورات Compaction أو يعلّق عليها.
- **`before_tool_call` / `after_tool_call`**: يعترض معاملات/نتائج الأدوات.
- **`before_install`**: يفحص نتائج الفحص المضمنة ويمكنه اختياريًا حظر تثبيت Skills أو Plugin.
- **`tool_result_persist`**: يحوّل نتائج الأدوات بشكل متزامن قبل كتابتها في سجل جلسة مملوك لـ OpenClaw.
- **`message_received` / `message_sending` / `message_sent`**: خطافات الرسائل الواردة + الصادرة.
- **`session_start` / `session_end`**: حدود دورة حياة الجلسة.
- **`gateway_start` / `gateway_stop`**: أحداث دورة حياة Gateway.

قواعد قرار الخطافات لحراس الصادر/الأداة:

- `before_tool_call`: `{ block: true }` نهائي ويوقف المعالجات ذات الأولوية الأدنى.
- `before_tool_call`: `{ block: false }` لا يفعل شيئًا ولا يمحو حظرًا سابقًا.
- `before_install`: `{ block: true }` نهائي ويوقف المعالجات ذات الأولوية الأدنى.
- `before_install`: `{ block: false }` لا يفعل شيئًا ولا يمحو حظرًا سابقًا.
- `message_sending`: `{ cancel: true }` نهائي ويوقف المعالجات ذات الأولوية الأدنى.
- `message_sending`: `{ cancel: false }` لا يفعل شيئًا ولا يمحو إلغاءً سابقًا.

راجع [خطافات Plugin](/ar/plugins/hooks) للاطلاع على API الخطافات وتفاصيل التسجيل.

قد تكيف أحزمة الاختبار هذه الخطافات بطرق مختلفة. يحافظ حزام خادم تطبيق Codex على
خطافات Plugin في OpenClaw باعتبارها عقد التوافق للأسطح المعكوسة الموثقة،
بينما تبقى خطافات Codex الأصلية آلية Codex منفصلة وأدنى مستوى.

## البث + الردود الجزئية

- تُبث دلتا المساعد من pi-agent-core وتُصدر كأحداث `assistant`.
- يمكن لبث الكتل إصدار ردود جزئية إما عند `text_end` أو `message_end`.
- يمكن إصدار بث الاستدلال كبث منفصل أو كردود كتل.
- راجع [البث](/ar/concepts/streaming) لسلوك التقسيم إلى أجزاء وردود الكتل.

## تنفيذ الأدوات + أدوات المراسلة

- تُصدر أحداث بدء/تحديث/انتهاء الأداة على بث `tool`.
- تُنظف نتائج الأدوات من حيث الحجم وحمولات الصور قبل التسجيل/الإصدار.
- تُتتبع عمليات إرسال أدوات المراسلة لمنع تأكيدات المساعد المكررة.

## تشكيل الرد + الكبت

- تُجمع الحمولات النهائية من:
  - نص المساعد (والاستدلال الاختياري)
  - ملخصات الأدوات المضمنة (عند السماح بالإسهاب)
  - نص خطأ المساعد عندما يخطئ النموذج
- تتم تصفية الرمز الصامت المطابق تمامًا `NO_REPLY` / `no_reply` من
  الحمولات الصادرة.
- تُزال تكرارات أدوات المراسلة من قائمة الحمولات النهائية.
- إذا لم تبق أي حمولات قابلة للعرض وحدث خطأ في أداة، يُصدر رد احتياطي لخطأ الأداة
  (ما لم تكن أداة مراسلة قد أرسلت بالفعل ردًا مرئيًا للمستخدم).

## Compaction + إعادة المحاولة

- يصدر Compaction التلقائي أحداث بث `compaction` ويمكنه تشغيل إعادة محاولة.
- عند إعادة المحاولة، تُعاد تهيئة المخازن المؤقتة داخل الذاكرة وملخصات الأدوات لتجنب تكرار المخرجات.
- راجع [Compaction](/ar/concepts/compaction) لمسار Compaction.

## تدفقات الأحداث (اليوم)

- `lifecycle`: يصدره `subscribeEmbeddedPiSession` (وكحل احتياطي بواسطة `agentCommand`)
- `assistant`: دلتا مبثوثة من pi-agent-core
- `tool`: أحداث أدوات مبثوثة من pi-agent-core

## التعامل مع قناة الدردشة

- تُخزن دلتا المساعد مؤقتًا في رسائل دردشة `delta`.
- تُصدر رسالة دردشة `final` عند **نهاية/خطأ دورة الحياة**.

## المهلات

- القيمة الافتراضية لـ `agent.wait`: 30 ثانية (الانتظار فقط). يتجاوز معامل `timeoutMs` ذلك.
- وقت تشغيل الوكيل: القيمة الافتراضية لـ `agents.defaults.timeoutSeconds` هي 172800 ثانية (48 ساعة)؛ وتُفرض في مؤقت إلغاء `runEmbeddedPiAgent`.
- وقت تشغيل Cron: `timeoutSeconds` لدورة الوكيل المعزولة مملوك لـ Cron. يبدأ المجدول ذلك المؤقت عند بدء التنفيذ، ويلغي التشغيل الأساسي عند الموعد النهائي المكوّن، ثم يشغل تنظيفًا محدودًا قبل تسجيل المهلة حتى لا تتمكن جلسة فرعية قديمة من إبقاء المسار عالقًا.
- تشخيصات حيوية الجلسة: عند تفعيل التشخيصات، يصنف `diagnostics.stuckSessionWarnMs` الجلسات الطويلة ذات حالة `processing` التي لا تحتوي على رد أو أداة أو حالة أو كتلة أو تقدم ACP مرصود. تُبلغ التشغيلات المضمنة النشطة، واستدعاءات النموذج، واستدعاءات الأدوات على أنها `session.long_running`؛ ويُبلغ العمل النشط بلا تقدم حديث على أنه `session.stalled`؛ وتُحجز `session.stuck` لحفظ سجلات الجلسة القديمة بلا عمل نشط. يحرر حفظ سجلات الجلسة القديمة مسار الجلسة المتأثرة فورًا؛ أما التشغيلات المضمنة المتوقفة فتُلغى وتُصفى فقط بعد `diagnostics.stuckSessionAbortMs` (افتراضيًا: 10 دقائق على الأقل و5 أضعاف حد التحذير) حتى يمكن استئناف العمل المصطف دون قطع التشغيلات البطيئة فقط. تصدر الاستعادة نتائج منظمة مطلوبة/مكتملة، وتُعلّم حالة التشخيص على أنها خاملة فقط إذا كان جيل المعالجة نفسه لا يزال حاليًا. تتراجع تشخيصات `session.stuck` المتكررة بينما تبقى الجلسة بلا تغيير.
- مهلة خمول النموذج: يلغي OpenClaw طلب النموذج عندما لا تصل أجزاء استجابة قبل نافذة الخمول. يمدد `models.providers.<id>.timeoutSeconds` مراقب الخمول هذا للمزوّدين المحليين/المستضافين ذاتيًا البطيئين؛ وإلا يستخدم OpenClaw `agents.defaults.timeoutSeconds` عند تكوينه، وبحد أقصى 120 ثانية افتراضيًا. التشغيلات المحفزة بواسطة Cron بلا مهلة نموذج أو وكيل صريحة تعطل مراقب الخمول وتعتمد على مهلة Cron الخارجية.
- مهلة طلب HTTP للمزوّد: ينطبق `models.providers.<id>.timeoutSeconds` على عمليات جلب HTTP لنموذج ذلك المزوّد، بما في ذلك الاتصال، والرؤوس، والجسم، ومهلة طلب SDK، ومعالجة الإلغاء الكلية لجلب محروس، ومراقب خمول بث النموذج. استخدم هذا للمزوّدين المحليين/المستضافين ذاتيًا البطيئين مثل Ollama قبل رفع مهلة وقت تشغيل الوكيل بالكامل.

## أين يمكن أن تنتهي الأمور مبكرًا

- مهلة الوكيل (إلغاء)
- AbortSignal (إلغاء)
- انقطاع Gateway أو مهلة RPC
- مهلة `agent.wait` (انتظار فقط، لا توقف الوكيل)
