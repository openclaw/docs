الالتزامات هي ذكريات متابعة قصيرة الأجل. عند تمكينها، يمكن لـ OpenClaw
ملاحظة أن محادثة ما أنشأت فرصة للتواصل لاحقًا، وتذكّر
العودة إليها في وقت لاحق.

أمثلة:

- تذكر أن لديك مقابلة غدًا. قد يتواصل معك OpenClaw بعدها.
- تقول إنك مرهق. قد يسألك OpenClaw لاحقًا عما إذا كنت قد نمت.
- يقول الوكيل إنه سيتابع الأمر بعد تغيّر شيء ما. قد يتتبع OpenClaw
  تلك المسألة المفتوحة.

الالتزامات ليست حقائق دائمة مثل `MEMORY.md`، وليست تذكيرات
دقيقة. فهي تقع بين الذاكرة والأتمتة: يتذكّر OpenClaw التزامًا
مرتبطًا بمحادثة، ثم يوصله Heartbeat عند حلول موعده.

## تمكين الالتزامات

تكون الالتزامات معطلة افتراضيًا (`commitments.enabled: false`). مكّنها في الإعدادات:

```bash
openclaw config set commitments.enabled true
openclaw config set commitments.maxPerDay 3
```

ملف `openclaw.json` المكافئ:

```json
{
  "commitments": {
    "enabled": true,
    "maxPerDay": 3
  }
}
```

يحد `commitments.maxPerDay` من عدد المتابعات المستنتجة التي يمكن إيصالها
لكل جلسة وكيل خلال يوم متجدد. القيمة الافتراضية هي `3`.

## آلية العمل

بعد رد الوكيل، قد يشغّل OpenClaw عملية استخراج مخفية في الخلفية ضمن
سياق منفصل، مع تعطيل الأدوات. تبحث تلك العملية فقط عن التزامات المتابعة المستنتجة. وهي
لا تكتب في المحادثة المرئية، ولا تطلب من الوكيل الرئيسي
الاستدلال بشأن عملية الاستخراج.

عندما يعثر OpenClaw على حالة مرشحة بدرجة ثقة عالية، فإنه يخزّن التزامًا يتضمن:

- معرّف الوكيل
- مفتاح الجلسة
- القناة الأصلية ووجهة الإيصال
- نافذة زمنية للاستحقاق
- اقتراحًا موجزًا للتواصل
- بيانات وصفية غير إرشادية تساعد Heartbeat على تحديد ما إذا كان ينبغي إرسالها

يتم الإيصال عبر Heartbeat. عندما يحين موعد التزام، يضيف Heartbeat
الالتزام إلى دورة Heartbeat ضمن نطاق الوكيل والقناة نفسيهما.
تحذّر المطالبة صراحةً من أن البيانات الوصفية للالتزام غير موثوقة، وتوجّه
النموذج إلى عدم اتباع أي تعليمات فيها أو استخدام الأدوات بسببها. يمكن
للنموذج إرسال رسالة تواصل طبيعية واحدة أو الرد بـ `HEARTBEAT_OK` لتجاهلها.
إذا ضُبط Heartbeat على `target: "none"`، تظل الالتزامات المستحقة
داخلية ولا ترسل رسائل تواصل خارجية. لا تعيد مطالبات إيصال الالتزامات
عرض نص المحادثة الأصلية، بل تعرض فقط اقتراح التواصل والبيانات
الوصفية، كما تُشغّل دورات Heartbeat الخاصة بالالتزامات المستحقة دون أدوات OpenClaw.

لا يوصّل OpenClaw التزامًا مستنتجًا فور كتابته مطلقًا.
يُضبط وقت الاستحقاق ليكون بعد فترة Heartbeat واحدة على الأقل من إنشاء الالتزام،
كي لا تتردد المتابعة في اللحظة نفسها التي استُنتجت فيها.

## النطاق

تقتصر الالتزامات على السياق المحدد للوكيل والقناة اللذين
أُنشئت ضمنهما. لا يوصّل وكيل آخر أو قناة أخرى أو جلسة غير مرتبطة
متابعةً استُنتجت أثناء التحدث إلى وكيل معين في Discord.

هذا النطاق جزء من الميزة. ينبغي أن تبدو رسائل التواصل الطبيعية كأنها استمرار
للمحادثة نفسها، لا كنظام تذكير عام.

لكن المحادثة أنشأت بوضوح فرصة مفيدة للتواصل مستقبلًا.

## إدارة الالتزامات

استخدم CLI لفحص الالتزامات المخزنة ومسحها:

```bash
openclaw commitments
openclaw commitments --all
openclaw commitments --agent main
openclaw commitments --status snoozed
openclaw commitments dismiss cm_abc123
```

راجع [`openclaw commitments`](/ar/cli/commitments) للاطلاع على مرجع الأوامر الكامل.

## الخصوصية والتكلفة

يستخدم استخراج الالتزامات عملية تمرير عبر نموذج لغوي كبير، لذا فإن تمكينه يضيف استخدامًا
للنموذج في الخلفية بعد الدورات المؤهلة. تكون العملية مخفية عن
المحادثة المرئية للمستخدم، لكنها تستطيع قراءة التبادل الأخير اللازم لتحديد ما إذا كانت
هناك متابعة.

تُعد الالتزامات المخزنة حالة محلية لـ OpenClaw. وهي ذاكرة تشغيلية، وليست
ذاكرة طويلة الأجل. عطّل الميزة باستخدام:

```bash
openclaw config set commitments.enabled false
```

## استكشاف الأخطاء وإصلاحها

إذا لم تظهر المتابعات المتوقعة:

- تأكد من أن `commitments.enabled` مضبوط على `true`.
- تحقق من `openclaw commitments --all` بحثًا عن السجلات المعلقة أو المتجاهلة أو المؤجلة أو المنتهية
  الصلاحية.
- تأكد من تشغيل Heartbeat للوكيل.
- تحقق مما إذا كان حد `commitments.maxPerDay` قد تم بلوغه بالفعل لجلسة
  الوكيل تلك.
- تذكّر أن استخراج الالتزامات يتخطى التذكيرات الدقيقة، وينبغي أن
  تظهر ضمن [المهام المجدولة](/ar/automation/cron-jobs) بدلًا من ذلك.
