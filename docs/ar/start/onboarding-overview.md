يتضمن OpenClaw إعدادًا أوليًا عبر الطرفية وتطبيق macOS. ويبدأ كلاهما بإعداد الاستدلال:
إذ يكتشفان الوصول الحالي إلى الذكاء الاصطناعي، ويشترطان إكمالًا فعليًا، وبعد ذلك فقط يشغّلان
OpenClaw لتهيئة بقية الإعداد. وإذا كان Gateway المُهيأ متاحًا
وكان النموذج مضبوطًا بالفعل للوكيل الافتراضي، فسيُتخطى الإعداد الأولي وتُفتح
واجهة الوكيل العادية. كما يتيح مسار الطرفية المعالج التقليدي الكامل
للإعداد التفصيلي.

## أي مسار ينبغي استخدامه؟

|                | الإعداد الأولي عبر CLI                         | الإعداد الأولي عبر تطبيق macOS           |
| -------------- | -------------------------------------- | ------------------------------ |
| **المنصات**  | macOS وLinux وWindows (أصليًا أو عبر WSL2) | macOS فقط                     |
| **الواجهة**  | إعداد الاستدلال، ثم OpenClaw         | إعداد الاستدلال، ثم OpenClaw |
| **الأنسب لـ**   | الخوادم، والأنظمة بلا واجهة رسومية، والتحكم الكامل        | أجهزة Mac المكتبية، والإعداد المرئي      |
| **الأتمتة** | `--non-interactive` للبرامج النصية        | يدوي فقط                    |
| **الأمر**    | `openclaw onboard`                     | تشغيل التطبيق                 |

ينبغي لمعظم المستخدمين البدء بـ **الإعداد الأولي عبر CLI** — فهو يعمل في كل مكان ويمنح
أكبر قدر من التحكم.

## ما الذي يهيئه الإعداد الأولي؟

لا تُنشئ مرحلة الاستدلال الموجّه سوى:

1. **موفّر النموذج والمصادقة** — الوصول المكتشف أو تسجيل دخول متحقق منه إلى الموفّر،
   أو مفتاح API، أو رمز مميز
2. **الاستدلال المتحقق منه** — إكمال فعلي باستخدام النموذج الفعّال
   للوكيل الافتراضي

بعد نجاح ذلك الإكمال، يستطيع OpenClaw تهيئة مساحة العمل وGateway
وخدمة Gateway والقنوات والوكلاء والـ plugins والميزات الاختيارية الأخرى.

يمكن لمعالج CLI التقليدي أيضًا تهيئة:

1. **القنوات** (اختياري) — قنوات الدردشة المدمجة والمرفقة مثل
   Discord وFeishu وGoogle Chat وiMessage وMattermost وMicrosoft Teams
   وTelegram وWhatsApp وغيرها
2. **عناصر التحكم المتقدمة في Gateway** — الوضع البعيد، وإعدادات الشبكة، وخيارات الخدمة الخفية

## الإعداد الأولي عبر CLI

شغّل الأمر في أي طرفية:

```bash
openclaw onboard
```

يكتشف المسار الموجّه الوصول الحالي إلى الذكاء الاصطناعي، ويختبر الخيارات المرشحة فعليًا بالترتيب،
وينتقل إلى الخيار التالي عند الفشل. وإذا استُنفدت عملية الاكتشاف، فإنه يعرض OpenAI
وAnthropic وxAI (Grok) وGoogle وOpenRouter أولًا. ويحتوي **المزيد…** على
الموفّرين المتبقين ضمن مجموعات الموفّرين، مع المناطق والخطط وطرق
المتصفح أو الجهاز أو مفتاح API أو الرمز المميز المدعومة في قائمة ثانية. ولا يحفظ النموذج
وبيانات الاعتماد إلا بعد إكمال ناجح، ثم يشغّل OpenClaw
لتهيئة مساحة العمل وGateway والقنوات والوكلاء والـ plugins والميزات الاختيارية
الأخرى. يؤدي خيار **التخطي الآن** إلى الخروج دون تشغيل OpenClaw. ولا يوجد
انتقال إلى المسار التقليدي من داخل التدفق؛ اخرج وشغّل `openclaw onboard --classic` عندما تريد
استخدام المعالج التقليدي بدلًا منه.

بعد نجاح الاستدلال، يمكن لـ OpenClaw نقل إعداد القنوات إلى معالج
طرفية ذي إدخال محجوب. ولا يفتح إعداد الموفّر الموجّه أو التقليدي؛ اخرج من OpenClaw
وشغّل `openclaw onboard` لتغيير موفّر النموذج أو مصادقته.

استخدم `openclaw onboard --classic` لإعداد مفصل للنموذج/المصادقة أو القنوات أو Skills
أو Gateway البعيد أو الاستيراد. كما تؤدي إضافة `--install-daemon` إلى تحديد
المسار التقليدي وتثبيت خدمة الخلفية في خطوة واحدة. استخدم `openclaw
openclaw` للإعداد والإصلاح الحواريين غير المتعلقين بالاستدلال. ويُعد `openclaw
onboard --modern` اسمًا مستعارًا للتوافق يستخدم بوابة
الاستدلال الفعلي نفسها.

توثيق أمر CLI: [`openclaw onboard`](/ar/cli/onboard)

## الإعداد الأولي عبر تطبيق macOS

افتح تطبيق OpenClaw. إذا كان Gateway المحلي أو البعيد المُهيأ متاحًا
وكان النموذج مضبوطًا بالفعل للوكيل الافتراضي، فسيتخطى التطبيق الإعداد الأولي
وOpenClaw ويفتح واجهة الوكيل العادية فورًا.

بالنسبة إلى Gateway جديد أو غير مكتمل الإعداد، يكتشف مسار التشغيل الأول الوصول الحالي إلى الذكاء
الاصطناعي (Claude Code أو Codex أو مفاتيح API)، ويختبر أفضل
خيار فعليًا، ولا يحفظه إلا بعد تلقي رد حقيقي — مع الرجوع تلقائيًا
وتقديم خطوة يدوية متحقق منها لإدخال مفتاح API عند عدم العثور على أي خيار. تستخدم بيانات
الاعتماد الحساسة إدخالًا محجوبًا. وبعد نجاح الاستدلال، يبدأ OpenClaw
ويساعد على تهيئة البقية.

يظل Gemini CLI متاحًا للوكلاء العاديين بعد الإعداد، لكنه لا يُعرض
لبوابة الاستدلال هذه لأنه لا يستطيع فرض الاختبار الخالي من الأدوات.

المرجع الكامل: [الإعداد الأولي (تطبيق macOS)](/ar/start/onboarding)

## الموفّرون المخصصون أو غير المدرجين

إذا لم يكن موفّرك مدرجًا، فشغّل `openclaw onboard --classic`، واختر
**موفّر مخصص**، وأدخل:

- توافق نقطة النهاية: متوافقة مع OpenAI ‏(`/chat/completions`)، أو متوافقة مع OpenAI Responses ‏(`/responses`)، أو متوافقة مع Anthropic ‏(`/messages`)، أو غير معروفة (تختبر الأنواع الثلاثة جميعًا وتكتشف النوع تلقائيًا)
- عنوان URL الأساسي ومفتاح API (مفتاح API اختياري إذا كانت نقطة النهاية لا تتطلبه)
- معرّف النموذج والاسم المستعار الاختياري للنموذج

يمكن أن تتعايش عدة نقاط نهاية مخصصة — إذ تحصل كل واحدة منها على معرّف نقطة نهاية خاص بها.
