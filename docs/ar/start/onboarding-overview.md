لدى OpenClaw إعداد أولي عبر الطرفية وتطبيق macOS. يبدأ كلاهما بتهيئة الاستدلال:
يكتشفان الوصول الحالي إلى الذكاء الاصطناعي، ويتطلبان إكمالًا فعليًا، وعندها فقط يشغّلان
Crestodian لتهيئة بقية الإعداد. إذا كان هناك Gateway مُهيّأ ويمكن الوصول إليه
وكان الوكيل الافتراضي فيه يملك نموذجًا مُهيّأ بالفعل، فسيتم تجاوز الإعداد الأولي وفتح
واجهة الوكيل العادية. يوفّر مسار الطرفية أيضًا المعالج التقليدي الكامل من أجل
الإعداد التفصيلي.

## أي مسار ينبغي أن أستخدم؟

|                | الإعداد الأولي عبر CLI                         | الإعداد الأولي عبر تطبيق macOS             |
| -------------- | -------------------------------------- | -------------------------------- |
| **المنصات**  | macOS وLinux وWindows (أصليًا أو عبر WSL2) | macOS فقط                       |
| **الواجهة**  | إعداد الاستدلال، ثم Crestodian       | إعداد الاستدلال، ثم Crestodian |
| **الأنسب لـ**   | الخوادم، والأنظمة بلا واجهة رسومية، والتحكم الكامل        | أجهزة Mac المكتبية، والإعداد المرئي        |
| **الأتمتة** | `--non-interactive` للنصوص البرمجية        | يدوي فقط                      |
| **الأمر**    | `openclaw onboard`                     | شغّل التطبيق                   |

ينبغي لمعظم المستخدمين البدء بـ **الإعداد الأولي عبر CLI** — فهو يعمل في كل مكان ويمنحك
أكبر قدر من التحكم.

## ما الذي يهيّئه الإعداد الأولي

لا تهيّئ مرحلة الاستدلال الإرشادية سوى ما يلي:

1. **موفّر النموذج والمصادقة** — وصول مكتشف أو مفتاح API تم التحقق منه
2. **استدلال تم التحقق منه** — إكمال فعلي باستخدام النموذج الفعلي
   للوكيل الافتراضي

بعد نجاح هذا الإكمال، يمكن لـ Crestodian تهيئة مساحة العمل وGateway
وخدمة Gateway والقنوات والوكلاء والـ plugins والميزات الاختيارية الأخرى.

يمكن لمُعالج CLI التقليدي أيضًا تهيئة ما يلي:

1. **القنوات** (اختياري) — قنوات الدردشة المدمجة والمضمّنة مثل
   Discord وFeishu وGoogle Chat وiMessage وMattermost وMicrosoft Teams
   وTelegram وWhatsApp وغيرها
2. **عناصر التحكم المتقدمة في Gateway** — الوضع البعيد وإعدادات الشبكة وخيارات البرنامج الخفي

## الإعداد الأولي عبر CLI

شغّل في أي طرفية:

```bash
openclaw onboard
```

يكتشف المسار الإرشادي الوصول الحالي إلى الذكاء الاصطناعي، ويختبر المرشحين فعليًا بالترتيب،
وينتقل إلى التالي عند الفشل، ويتيح إدخال المفتاح يدويًا مع إخفائه. ولا يحفظ
النموذج وبيانات الاعتماد إلا بعد نجاح الإكمال، ثم يشغّل Crestodian
لتهيئة مساحة العمل وGateway والقنوات والوكلاء والـ plugins وغيرها من
الميزات الاختيارية. لا توجد مرحلة Crestodian قبل الاستدلال، ولا مسار لتخطي الذكاء الاصطناعي، ولا
انتقال إلى المسار التقليدي داخل التدفق. اخرج وشغّل `openclaw onboard --classic` عندما
تريد استخدام المعالج التقليدي بدلًا من ذلك.

بعد نجاح الاستدلال، يمكن لـ Crestodian تسليم إعداد القناة إلى معالج طرفية
يخفي المدخلات الحساسة. وهو لا يفتح إعداد الموفّر الإرشادي أو التقليدي؛ اخرج من Crestodian وشغّل
`openclaw onboard` لتغيير موفّر النموذج أو مصادقته.

استخدم `openclaw onboard --classic` لإعداد النموذج/المصادقة والقناة والمهارة
وGateway البعيد أو الاستيراد بالتفصيل. تؤدي إضافة `--install-daemon` أيضًا إلى تحديد
المسار التقليدي وتثبيت خدمة الخلفية في خطوة واحدة. استخدم `openclaw
crestodian` للإعداد والإصلاح الحواريين غير المرتبطين بالاستدلال. يمثّل `openclaw
onboard --modern` اسمًا مستعارًا للتوافق يستخدم بوابة الاستدلال الفعلي
نفسها.

وثائق أمر CLI: [`openclaw onboard`](/ar/cli/onboard)

## الإعداد الأولي عبر تطبيق macOS

افتح تطبيق OpenClaw. إذا أمكن الوصول إلى Gateway المحلي أو البعيد المُهيّأ
وكان الوكيل الافتراضي يملك نموذجًا مُهيّأ بالفعل، فسيتجاوز التطبيق الإعداد الأولي
وCrestodian ويفتح واجهة الوكيل العادية فورًا.

بالنسبة إلى Gateway جديد أو غير مكتمل، يكتشف تدفق التشغيل الأول الوصول الحالي إلى الذكاء
الاصطناعي (Claude Code أو Codex أو مفاتيح API)، ويختبر أفضل
خيار فعليًا، ولا يحفظه إلا بعد تلقي رد حقيقي — مع الرجوع تلقائيًا إلى البدائل
وتقديم خطوة يدوية تم التحقق منها لإدخال مفتاح API عند عدم العثور على أي خيار. تستخدم
بيانات الاعتماد الحساسة إدخالًا مخفيًا. وبعد نجاح الاستدلال، يبدأ Crestodian
ويساعد في تهيئة البقية.

يظل Gemini CLI متاحًا للوكلاء العاديين بعد الإعداد، لكنه لا يُعرض
لبوابة الاستدلال هذه لأنه لا يستطيع فرض اختبار بلا أدوات.

المرجع الكامل: [الإعداد الأولي (تطبيق macOS)](/ar/start/onboarding)

## الموفّرون المخصصون أو غير المدرجين

إذا لم يكن موفّرك مدرجًا، فشغّل `openclaw onboard --classic`، واختر
**Custom Provider**، وأدخل:

- توافق نقطة النهاية: متوافقة مع OpenAI ‏(`/chat/completions`)، أو متوافقة مع OpenAI Responses ‏(`/responses`)، أو متوافقة مع Anthropic ‏(`/messages`)، أو غير معروفة (تختبر الأنواع الثلاثة وتكتشف النوع تلقائيًا)
- عنوان URL الأساسي ومفتاح API (مفتاح API اختياري إذا كانت نقطة النهاية لا تتطلبه)
- معرّف النموذج واسم مستعار اختياري للنموذج

يمكن لعدة نقاط نهاية مخصصة أن تتعايش — وتحصل كل واحدة منها على معرّف نقطة نهاية خاص بها.
