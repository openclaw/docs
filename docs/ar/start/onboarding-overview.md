لدى OpenClaw مساران لتهيئة البداية. يضبط كلاهما المصادقة، وGateway، وقنوات
الدردشة الاختيارية — لكنهما يختلفان فقط في طريقة تفاعلك مع الإعداد.

## أي مسار ينبغي أن أستخدم؟

|                | تهيئة البداية عبر CLI                  | تهيئة البداية عبر تطبيق macOS |
| -------------- | -------------------------------------- | ------------------------- |
| **المنصات**  | macOS، وLinux، وWindows (أصلي أو WSL2) | macOS فقط                |
| **الواجهة**  | معالج في الطرفية                        | واجهة مستخدم موجهة في التطبيق |
| **الأفضل لـ**   | الخوادم، والبيئات بلا واجهة، والتحكم الكامل | Mac مكتبي، إعداد مرئي |
| **الأتمتة** | `--non-interactive` للسكربتات        | يدوي فقط               |
| **الأمر**    | `openclaw onboard`                     | شغّل التطبيق            |

ينبغي لمعظم المستخدمين البدء بـ **تهيئة البداية عبر CLI** — فهي تعمل في كل مكان وتمنحك
أكبر قدر من التحكم.

## ما الذي تضبطه تهيئة البداية

بغض النظر عن المسار الذي تختاره، تضبط تهيئة البداية ما يلي:

1. **موفر النموذج والمصادقة** — مفتاح API أو OAuth أو رمز إعداد للموفر الذي اخترته
2. **مساحة العمل** — دليل لملفات الوكيل، وقوالب التمهيد، والذاكرة
3. **Gateway** — المنفذ، وعنوان الربط، ووضع المصادقة
4. **القنوات** (اختياري) — قنوات الدردشة المدمجة والمضمنة مثل
   iMessage، وDiscord، وFeishu، وGoogle Chat، وMattermost، وMicrosoft Teams،
   وTelegram، وWhatsApp، والمزيد
5. **الخدمة الخفية** (اختياري) — خدمة تعمل في الخلفية بحيث يبدأ Gateway تلقائيًا

## تهيئة البداية عبر CLI

شغّل في أي طرفية:

```bash
openclaw onboard
```

أضف `--install-daemon` لتثبيت الخدمة الخلفية أيضًا في خطوة واحدة.

المرجع الكامل: [تهيئة البداية (CLI)](/ar/start/wizard)
وثائق أمر CLI: [`openclaw onboard`](/ar/cli/onboard)

## تهيئة البداية عبر تطبيق macOS

افتح تطبيق OpenClaw. يرشدك معالج التشغيل الأول عبر الخطوات نفسها
باستخدام واجهة مرئية.

المرجع الكامل: [تهيئة البداية (تطبيق macOS)](/ar/start/onboarding)

## موفرو الخدمات المخصصون أو غير المدرجين

إذا لم يكن موفرك مدرجًا في تهيئة البداية، فاختر **الموفر المخصص** وأدخل:

- وضع توافق API (متوافق مع OpenAI، أو متوافق مع Anthropic، أو اكتشاف تلقائي)
- عنوان URL الأساسي ومفتاح API
- معرّف النموذج والاسم المستعار الاختياري

يمكن أن تتعايش عدة نقاط نهاية مخصصة — إذ تحصل كل واحدة على معرّف نقطة نهاية خاص بها.
