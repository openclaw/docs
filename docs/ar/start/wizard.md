إعداد CLI التمهيدي هو مسار إعداد الطرفية **الموصى به** لـ OpenClaw على
macOS أو Linux أو Windows. يمكن لمستخدمي سطح مكتب Windows البدء أيضًا من
[مركز Windows](/ar/platforms/windows).
يضبط Gateway محليًا أو اتصال Gateway بعيدًا، إضافة إلى القنوات وSkills
وافتراضات مساحة العمل ضمن تدفق إرشادي واحد.

```bash
openclaw onboard
```

## اللغة المحلية

يعرض معالج CLI نصوص الإعداد التمهيدي الثابتة باللغة المحلية. يحدد اللغة من
`OPENCLAW_LOCALE`، ثم `LC_ALL`، ثم `LC_MESSAGES`، ثم `LANG`، ويعود
إلى الإنجليزية. لغات المعالج المدعومة هي `en` و`zh-CN` و`zh-TW`.

```bash
OPENCLAW_LOCALE=zh-CN openclaw onboard
```

تبقى الأسماء والمعرّفات الثابتة حرفية: `OpenClaw` و`Gateway` و`Tailscale`
والأوامر ومفاتيح الإعدادات وعناوين URL ومعرّفات المزوّدين ومعرّفات النماذج وتسميات Plugin/القنوات
لا تُترجم.

<Info>
أسرع محادثة أولى: افتح واجهة التحكم (لا حاجة إلى إعداد قناة). شغّل
`openclaw dashboard` وتحادث في المتصفح. الوثائق: [لوحة المعلومات](/ar/web/dashboard).
</Info>

لإعادة الضبط لاحقًا:

```bash
openclaw configure
openclaw agents add <name>
```

<Note>
لا يعني `--json` الوضع غير التفاعلي. للبرامج النصية، استخدم `--non-interactive`.
</Note>

<Tip>
يتضمن إعداد CLI التمهيدي خطوة بحث ويب يمكنك فيها اختيار مزوّد
مثل Brave أو DuckDuckGo أو Exa أو Firecrawl أو Gemini أو Grok أو Kimi أو MiniMax Search
أو Ollama Web Search أو Perplexity أو SearXNG أو Tavily. تتطلب بعض المزوّدات
مفتاح API، بينما لا يتطلب بعضها مفتاحًا. يمكنك أيضًا ضبط ذلك لاحقًا باستخدام
`openclaw configure --section web`. الوثائق: [أدوات الويب](/ar/tools/web).
</Tip>

## البدء السريع مقابل المتقدم

يبدأ الإعداد التمهيدي بـ **البدء السريع** (الافتراضات) مقابل **المتقدم** (تحكم كامل).

<Tabs>
  <Tab title="البدء السريع (الافتراضات)">
    - Gateway محلي (local loopback)
    - افتراض مساحة العمل (أو مساحة عمل موجودة)
    - منفذ Gateway **18789**
    - مصادقة Gateway **رمز مميز** (يُنشأ تلقائيًا، حتى على local loopback)
    - افتراضي سياسة الأدوات للإعدادات المحلية الجديدة: `tools.profile: "coding"` (يُحافظ على الملف الشخصي الصريح الموجود)
    - افتراضي عزل الرسائل المباشرة: يكتب الإعداد التمهيدي المحلي `session.dmScope: "per-channel-peer"` عند عدم ضبطه. التفاصيل: [مرجع إعداد CLI](/ar/start/wizard-cli-reference#outputs-and-internals)
    - إتاحة Tailscale **متوقفة**
    - الرسائل المباشرة في Telegram وWhatsApp تستخدم **قائمة السماح** افتراضيًا (سيُطلب منك رقم هاتفك)

  </Tab>
  <Tab title="المتقدم (تحكم كامل)">
    - يعرض كل خطوة (الوضع، مساحة العمل، Gateway، القنوات، الخفيّة، Skills).

  </Tab>
</Tabs>

## ما يضبطه الإعداد التمهيدي

يرشدك **الوضع المحلي (افتراضي)** خلال هذه الخطوات:

1. **النموذج/المصادقة** — اختر أي مزوّد/تدفق مصادقة مدعوم (مفتاح API أو OAuth أو مصادقة يدوية خاصة بالمزوّد)، بما في ذلك المزوّد المخصص
   (متوافق مع OpenAI أو متوافق مع Anthropic أو كشف تلقائي غير معروف). اختر نموذجًا افتراضيًا.
   ملاحظة أمنية: إذا كان هذا الوكيل سيشغّل أدوات أو يعالج محتوى Webhook/الخطافات، ففضّل أقوى نموذج من أحدث جيل متاح واجعل سياسة الأدوات صارمة. المستويات الأضعف/الأقدم أسهل في حقن المطالبات.
   للتشغيلات غير التفاعلية، يخزّن `--secret-input-mode ref` مراجع مدعومة بمتغيرات البيئة في ملفات تعريف المصادقة بدلًا من قيم مفاتيح API بنص صريح.
   في وضع `ref` غير التفاعلي، يجب ضبط متغير بيئة المزوّد؛ ويفشل تمرير أعلام المفاتيح المضمنة دون ذلك المتغير بسرعة.
   في التشغيلات التفاعلية، يتيح لك اختيار وضع مرجع السر الإشارة إلى متغير بيئة أو مرجع مزوّد مضبوط (`file` أو `exec`)، مع تحقق تمهيدي سريع قبل الحفظ.
   بالنسبة إلى Anthropic، يقدّم الإعداد التمهيدي/الضبط التفاعلي **Anthropic Claude CLI** كالمسار المحلي المفضل و**مفتاح Anthropic API** كمسار الإنتاج الموصى به. يظل رمز إعداد Anthropic متاحًا أيضًا كمسار مصادقة رمز مدعوم.
2. **مساحة العمل** — موقع ملفات الوكيل (الافتراضي `~/.openclaw/workspace`). يضيف ملفات تمهيدية أولية.
3. **Gateway** — المنفذ، عنوان الربط، وضع المصادقة، إتاحة Tailscale.
   في وضع الرمز التفاعلي، اختر تخزين الرمز بنص صريح افتراضيًا أو اشترك في SecretRef.
   مسار SecretRef للرمز غير التفاعلي: `--gateway-token-ref-env <ENV_VAR>`.
4. **القنوات** — قنوات المحادثة المضمنة والرسمية عبر Plugin مثل iMessage وDiscord وFeishu وGoogle Chat وMattermost وMicrosoft Teams وQQ Bot وSignal وSlack وTelegram وWhatsApp والمزيد.
5. **الخفيّة** — يثبّت LaunchAgent (macOS)، أو وحدة مستخدم systemd (Linux/WSL2)، أو Windows Scheduled Task أصلية مع احتياطي مجلد بدء التشغيل لكل مستخدم.
   إذا كانت مصادقة الرمز تتطلب رمزًا وكان `gateway.auth.token` مُدارًا بواسطة SecretRef، يتحقق تثبيت الخفيّة منه لكنه لا يحفظ الرمز المحلول في بيانات بيئة خدمة المشرف.
   إذا كانت مصادقة الرمز تتطلب رمزًا وكان SecretRef للرمز المضبوط غير محلول، يُحظر تثبيت الخفيّة مع إرشادات قابلة للتنفيذ.
   إذا كان كل من `gateway.auth.token` و`gateway.auth.password` مضبوطين وكان `gateway.auth.mode` غير مضبوط، يُحظر تثبيت الخفيّة حتى يُضبط الوضع صراحة.
6. **فحص الصحة** — يبدأ Gateway ويتحقق من أنه يعمل.
7. **Skills** — يثبّت Skills الموصى بها والتبعيات الاختيارية.

<Note>
لا تؤدي إعادة تشغيل الإعداد التمهيدي إلى مسح أي شيء إلا إذا اخترت **إعادة تعيين** صراحة (أو مررت `--reset`).
يضبط `--reset` في CLI افتراضيًا الإعدادات وبيانات الاعتماد والجلسات؛ استخدم `--reset-scope full` لتضمين مساحة العمل.
إذا كان ملف الإعداد غير صالح أو يحتوي على مفاتيح قديمة، يطلب منك الإعداد التمهيدي تشغيل `openclaw doctor` أولًا.
</Note>

يضبط **الوضع البعيد** العميل المحلي فقط للاتصال بـ Gateway في مكان آخر.
ولا يثبّت أو يغيّر أي شيء على المضيف البعيد.

## إضافة وكيل آخر

استخدم `openclaw agents add <name>` لإنشاء وكيل منفصل له مساحة العمل
والجلسات وملفات تعريف المصادقة الخاصة به. التشغيل دون `--workspace` يطلق الإعداد التمهيدي.

ما يضبطه:

- `agents.list[].name`
- `agents.list[].workspace`
- `agents.list[].agentDir`

ملاحظات:

- تتبع مساحات العمل الافتراضية `~/.openclaw/workspace-<agentId>`.
- أضف `bindings` لتوجيه الرسائل الواردة (يمكن للإعداد التمهيدي فعل ذلك).
- الأعلام غير التفاعلية: `--model` و`--agent-dir` و`--bind` و`--non-interactive`.
