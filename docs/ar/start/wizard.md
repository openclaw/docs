مسار إعداد CLI هو المسار الطرفي **الموصى به** لإعداد OpenClaw على
macOS أو Linux أو Windows. يمكن لمستخدمي سطح مكتب Windows البدء أيضًا من
[مركز Windows](/ar/platforms/windows).
يضبط Gateway محليًا أو اتصالًا بـ Gateway بعيد، إضافة إلى القنوات وSkills
وافتراضات مساحة العمل ضمن تدفق إرشادي واحد.

```bash
openclaw onboard
```

عادة لا يستغرق البدء السريع سوى بضع دقائق، لكن الإعداد الكامل قد يستغرق وقتًا أطول
عندما يتطلب تسجيل دخول المزوّد، أو إقران القنوات، أو تثبيت الخادم الخفي، أو تنزيلات الشبكة،
أو Skills، أو Plugins الاختيارية إعدادًا إضافيًا. يعرض المعالج هذا الجدول الزمني
مسبقًا، ويمكن تخطي الخطوات الاختيارية والعودة إليها لاحقًا باستخدام
`openclaw configure`.

## اللغة المحلية

يترجم معالج CLI نصوص الإعداد الثابتة. يحدد اللغة المحلية من
`OPENCLAW_LOCALE`، ثم `LC_ALL`، ثم `LC_MESSAGES`، ثم `LANG`، ويعود
إلى الإنجليزية افتراضيًا. اللغات المحلية المدعومة للمعالج هي `en` و`zh-CN` و`zh-TW`.

```bash
OPENCLAW_LOCALE=zh-CN openclaw onboard
```

تبقى الأسماء والمعرّفات الثابتة حرفية: `OpenClaw` و`Gateway` و`Tailscale`
والأوامر ومفاتيح الإعداد وعناوين URL ومعرّفات المزوّدين ومعرّفات النماذج وتسميات Plugin/القنوات
لا تُترجم.

<Info>
أسرع محادثة أولى: افتح واجهة التحكم (لا حاجة إلى إعداد قناة). شغّل
`openclaw dashboard` وتحدث في المتصفح. المستندات: [لوحة التحكم](/ar/web/dashboard).
</Info>

لإعادة الضبط لاحقًا:

```bash
openclaw configure
openclaw agents add <name>
```

<Note>
`--json` لا يعني الوضع غير التفاعلي. للبرامج النصية، استخدم `--non-interactive`.
</Note>

<Tip>
يتضمن إعداد CLI خطوة بحث ويب يمكنك فيها اختيار مزوّد
مثل Brave أو DuckDuckGo أو Exa أو Firecrawl أو Gemini أو Grok أو Kimi أو MiniMax Search
أو Ollama Web Search أو Perplexity أو SearXNG أو Tavily. يتطلب بعض المزوّدين
مفتاح API، بينما لا يحتاج آخرون إلى مفتاح. يمكنك أيضًا ضبط ذلك لاحقًا باستخدام
`openclaw configure --section web`. المستندات: [أدوات الويب](/ar/tools/web).
</Tip>

## البدء السريع مقابل المتقدم

يبدأ الإعداد بخيار **البدء السريع** (الافتراضات) مقابل **المتقدم** (تحكم كامل).

<Tabs>
  <Tab title="البدء السريع (الافتراضات)">
    - Gateway محلي (loopback)
    - افتراضي مساحة العمل (أو مساحة عمل موجودة)
    - منفذ Gateway هو **18789**
    - مصادقة Gateway هي **Token** (تُنشأ تلقائيًا، حتى على loopback)
    - افتراضي سياسة الأدوات للإعدادات المحلية الجديدة: `tools.profile: "coding"` (يُحافَظ على الملف الشخصي الصريح الموجود)
    - افتراضي عزل DM: يكتب الإعداد المحلي `session.dmScope: "per-channel-peer"` عندما يكون غير معيّن. التفاصيل: [مرجع إعداد CLI](/ar/start/wizard-cli-reference#outputs-and-internals)
    - تعريض Tailscale **متوقف**
    - رسائل Telegram + WhatsApp المباشرة تكون افتراضيًا على **قائمة السماح** (سيُطلب منك إدخال رقم هاتفك)

  </Tab>
  <Tab title="المتقدم (تحكم كامل)">
    - يعرض كل خطوة (الوضع، مساحة العمل، Gateway، القنوات، الخادم الخفي، Skills).

  </Tab>
</Tabs>

## ما الذي يضبطه الإعداد

يرشدك **الوضع المحلي (الافتراضي)** عبر هذه الخطوات:

1. **النموذج/المصادقة** — اختر أي مزوّد/تدفق مصادقة مدعوم (مفتاح API أو OAuth أو مصادقة يدوية خاصة بالمزوّد)، بما في ذلك المزوّد المخصص
   (متوافق مع OpenAI، أو متوافق مع Anthropic، أو اكتشاف تلقائي غير معروف). اختر نموذجًا افتراضيًا.
   ملاحظة أمنية: إذا كان هذا الوكيل سيشغّل أدوات أو يعالج محتوى Webhook/hooks، ففضّل أقوى نموذج متاح من أحدث جيل وحافظ على سياسة أدوات صارمة. الشرائح الأضعف/الأقدم أسهل في حقن التعليمات إليها.
   للتشغيل غير التفاعلي، يخزّن `--secret-input-mode ref` مراجع مدعومة بمتغيرات البيئة في ملفات تعريف المصادقة بدلًا من قيم مفاتيح API بالنص الصريح.
   في وضع `ref` غير التفاعلي، يجب تعيين متغير بيئة المزوّد؛ تمرير أعلام مفاتيح مضمنة دون متغير البيئة هذا يفشل سريعًا.
   في التشغيل التفاعلي، يتيح لك اختيار وضع مرجع السر الإشارة إما إلى متغير بيئة أو إلى مرجع مزوّد مضبوط (`file` أو `exec`)، مع تحقق مسبق سريع قبل الحفظ.
   بالنسبة إلى Anthropic، يعرض الإعداد/الضبط التفاعلي **Anthropic Claude CLI** باعتباره المسار المحلي المفضل و**مفتاح Anthropic API** باعتباره مسار الإنتاج الموصى به. كما يظل setup-token الخاص بـ Anthropic متاحًا كمسار مصادقة رمزية مدعوم.
2. **مساحة العمل** — موقع ملفات الوكيل (الافتراضي `~/.openclaw/workspace`). يزرع ملفات التمهيد.
3. **Gateway** — المنفذ، عنوان الربط، وضع المصادقة، تعريض Tailscale.
   في وضع الرمز التفاعلي، اختر تخزين الرمز الافتراضي بالنص الصريح أو انتقل إلى SecretRef.
   مسار SecretRef للرمز غير التفاعلي: `--gateway-token-ref-env <ENV_VAR>`.
4. **القنوات** — قنوات الدردشة المدمجة وPlugin الرسمية مثل iMessage وDiscord وFeishu وGoogle Chat وMattermost وMicrosoft Teams وQQ Bot وSignal وSlack وTelegram وWhatsApp والمزيد.
5. **الخادم الخفي** — يثبّت LaunchAgent (macOS)، أو وحدة مستخدم systemd (Linux/WSL2)، أو مهمة Windows Scheduled Task أصلية مع مسار احتياطي لمجلد Startup لكل مستخدم.
   إذا كانت مصادقة الرمز تتطلب رمزًا وكان `gateway.auth.token` مُدارًا عبر SecretRef، فإن تثبيت الخادم الخفي يتحقق منه لكنه لا يحفظ الرمز المحلول في بيانات تعريف بيئة خدمة المشرف.
   إذا كانت مصادقة الرمز تتطلب رمزًا وكان SecretRef للرمز المضبوط غير قابل للحل، يُحظر تثبيت الخادم الخفي مع إرشادات قابلة للتنفيذ.
   إذا كان كل من `gateway.auth.token` و`gateway.auth.password` مضبوطين وكان `gateway.auth.mode` غير معيّن، يُحظر تثبيت الخادم الخفي حتى يُعيّن الوضع صراحةً.
6. **فحص الصحة** — يبدأ Gateway ويتحقق من أنه يعمل.
7. **Skills** — يثبّت Skills الموصى بها والاعتماديات الاختيارية.

<Note>
لا تؤدي إعادة تشغيل الإعداد إلى مسح أي شيء ما لم تختر **إعادة تعيين** صراحةً (أو تمرر `--reset`).
يعيد `--reset` في CLI افتراضيًا تعيين الإعداد وبيانات الاعتماد والجلسات؛ استخدم `--reset-scope full` لتضمين مساحة العمل.
إذا كان الإعداد غير صالح أو يحتوي على مفاتيح قديمة، يطلب منك الإعداد تشغيل `openclaw doctor` أولًا.
</Note>

يضبط **الوضع البعيد** العميل المحلي فقط للاتصال بـ Gateway في مكان آخر.
ولا يثبّت أو يغيّر أي شيء على المضيف البعيد.

## إضافة وكيل آخر

استخدم `openclaw agents add <name>` لإنشاء وكيل منفصل بمساحة عمل خاصة به،
وجلساته، وملفات تعريف المصادقة الخاصة به. التشغيل دون `--workspace` يطلق الإعداد.

ما يعيّنه:

- `agents.list[].name`
- `agents.list[].workspace`
- `agents.list[].agentDir`

ملاحظات:

- تتبع مساحات العمل الافتراضية `~/.openclaw/workspace-<agentId>`.
- أضف `bindings` لتوجيه الرسائل الواردة (يمكن للإعداد فعل ذلك).
- أعلام الوضع غير التفاعلي: `--model` و`--agent-dir` و`--bind` و`--non-interactive`.
