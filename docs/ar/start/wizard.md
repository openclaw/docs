```bash
openclaw onboard
```

يُعد الإعداد الأولي عبر CLI مسار إعداد الطرفية الموصى به على macOS وLinux وWindows (أصليًا أو عبر WSL2). يكتشف افتراضيًا إمكانية الوصول المتاحة بالفعل إلى الذكاء الاصطناعي على الجهاز، ويتحقق منها بإكمال فعلي، ثم يشغّل OpenClaw لتهيئة مساحة العمل وGateway والميزات الاختيارية. يشغّل `openclaw setup` التدفق نفسه (تتناول [التهيئة](/ar/cli/setup) صيغة الإعداد الخاصة بالتهيئة فقط عبر `--baseline`). ويمكن لمستخدمي سطح مكتب Windows أيضًا البدء من [مركز Windows](/ar/platforms/windows).

يبدأ الإعداد الأولي الموجّه بإنشاء الاستدلال أولًا. فهو يكتشف إمكانية الوصول المتاحة إلى الذكاء الاصطناعي، ويتطلب إكمالًا فعليًا، وبعد ذلك فقط يشغّل [OpenClaw](/cli/openclaw) لتهيئة بقية OpenClaw. يؤدي اختيار **التخطي الآن** إلى الخروج من الإعداد الأولي من دون تشغيل OpenClaw.

يظل المعالج التقليدي متاحًا للمزوّدين المخصصين، وإعداد Gateway البعيد، وإقران القنوات، وعناصر التحكم في الخدمة الخفية، وSkills، وعمليات الاستيراد. شغّله صراحةً باستخدام `openclaw onboard --classic`؛ إذ لا يفوّض منتقي الاستدلال الموجّه إليه. بعد نجاح الاستدلال، يمكن لـ OpenClaw استخدام `open channel wizard for
<channel>` لتسليم إعداد القنوات الذي يحتاج إلى أسرار إلى معالج طرفية يحجبها.
لتغيير مزوّد النموذج أو مصادقته، اخرج من OpenClaw وشغّل `openclaw onboard`؛ لا يفتح OpenClaw تدفقات المزوّد الموجّهة أو التقليدية.

<Info>
لأسرع محادثة أولى: أكمل الإعداد الموجّه، وشغّل `openclaw dashboard`، ثم ابدأ المحادثة في المتصفح عبر واجهة التحكم. الوثائق: [لوحة المعلومات](/ar/web/dashboard).
</Info>

## الإعدادات المحلية

يترجم المعالج نصوص الإعداد الأولي الثابتة. ترتيب الحل: `OPENCLAW_LOCALE`، ثم `LC_ALL`، ثم `LC_MESSAGES`، ثم `LANG`، ثم الإنجليزية. الإعدادات المحلية المدعومة: `en`، و`zh-CN`، و`zh-TW`.

```bash
OPENCLAW_LOCALE=zh-CN openclaw onboard
```

تبقى أسماء المنتجات، والأوامر، ومفاتيح التهيئة، وعناوين URL، ومعرّفات المزوّدين، ومعرّفات النماذج، وتسميات الإضافات/القنوات باللغة الإنجليزية بصرف النظر عن الإعدادات المحلية.

لإعادة تهيئة الإعدادات غير المتعلقة بالاستدلال لاحقًا:

```bash
openclaw configure
openclaw agents add <name>
```

<Note>
لا يعني `--json` الوضع غير التفاعلي. بالنسبة إلى البرامج النصية، استخدم `--non-interactive` (راجع [أتمتة CLI](/ar/start/wizard-cli-automation)).
</Note>

<Tip>
يتضمن المعالج التقليدي خطوة للبحث على الويب يمكنك فيها اختيار مزوّد: Brave أو DuckDuckGo أو Exa أو Firecrawl أو Gemini أو Grok أو Kimi أو MiniMax Search أو Ollama Web Search أو Perplexity أو SearXNG أو Tavily. يحتاج بعضها إلى مفتاح API، بينما لا يحتاج بعضها الآخر إلى مفتاح. هيّئ ذلك لاحقًا باستخدام `openclaw configure --section web`. الوثائق:
[أدوات الويب](/ar/tools/web).
</Tip>

## الإعداد الموجّه الافتراضي

يتبع `openclaw onboard` العادي هذا المسار:

1. اقبل إشعار الأمان.
2. اكتشف النماذج المهيأة، ومتغيرات بيئة مفاتيح API، وواجهات CLI المحلية المدعومة للذكاء الاصطناعي، والنماذج القادرة على استخدام الأدوات والمثبّتة بالفعل من خوادم Ollama أو LM Studio التي يمكن الوصول إليها على مضيف Gateway. لا تنزّل عملية الفحص للقراءة فقط هذه أي نموذج مطلقًا. يُبلّغ عن عمليات تثبيت Gemini CLI وAntigravity، لكنها لا تُختبر تلقائيًا لأنها لا تستطيع فرض اختبار من دون أدوات.
3. اختبر أول مرشح مكتشف بإكمال فعلي. عند الفشل، اعرض السبب وتابع إلى المرشح التالي القابل للاستخدام.
4. إذا استُنفدت نتائج الاكتشاف، فاختر OpenAI أو Anthropic أو xAI (Grok) أو Google أو OpenRouter، أو اختر **المزيد…** لبقية المزوّدين. تظهر مناطق كل مزوّد وخططه وطرقه المدعومة عبر المتصفح أو الجهاز أو مفتاح API أو الرمز المميز في قائمة ثانية، وتُختبر باستخدام الإكمال الفعلي نفسه. اختر **التخطي الآن** للخروج من دون تشغيل OpenClaw.
5. احفظ فقط مسار النموذج الذي تم التحقق منه وأي حالة لبيانات اعتماد/Plugin يتطلبها. تظل إعدادات مساحة العمل وGateway من دون تغيير.
6. شغّل OpenClaw باستخدام النموذج الذي تم التحقق منه كي يتمكن من تهيئة مساحة العمل وGateway والقنوات والوكلاء والإضافات وبقية الإعداد الاختياري.

تؤدي إعادة تشغيل الأمر على تثبيت مهيأ إلى اختبار النموذج الافتراضي الحالي أولًا، ما يجعل التدفق الموجّه عملية تحقق وإصلاح. لا يستبدل الاختبار الفاشل النموذج المهيأ تلقائيًا مطلقًا؛ بل يتوقف الإعداد الأولي ويسأل عن كيفية المتابعة. شغّل `openclaw channels add` أو `openclaw configure` للإضافات اللاحقة غير المتعلقة بالاستدلال؛ واستخدم `openclaw onboard` لتغيير مسار المزوّد أو المصادقة.

## المعالج التقليدي: البدء السريع مقابل الوضع المتقدم

شغّل `openclaw onboard --classic` لفتح المعالج الكامل. يبدأ بخيار بين **البدء السريع** (الإعدادات الافتراضية) و**الوضع المتقدم** (تحكم كامل). مرّر `--flow quickstart` أو `--flow advanced` (الاسم البديل `manual`) لتحديد التدفق التقليدي وتخطي ذلك السؤال.

<Tabs>
  <Tab title="البدء السريع (الإعدادات الافتراضية)">
    - Gateway محلي، وربط استرجاعي
    - مساحة العمل الافتراضية (أو مساحة العمل الحالية)
    - منفذ Gateway ‏**18789**
    - مصادقة Gateway باستخدام **رمز مميز** (يُنشأ تلقائيًا، حتى عند استخدام الربط الاسترجاعي)
    - سياسة الأدوات: `tools.profile: "coding"` للإعدادات الجديدة (يُحتفظ بالملف التعريفي الصريح الحالي)
    - عزل الرسائل الخاصة: `session.dmScope: "per-channel-peer"` للإعدادات الجديدة. التفاصيل: [مرجع إعداد CLI](/ar/start/wizard-cli-reference#outputs-and-internals)
    - إتاحة Tailscale ‏**متوقفة**
    - يكون الإعداد الافتراضي للرسائل الخاصة في Telegram وWhatsApp هو **قائمة السماح**: يطلب Telegram معرّف مستخدم Telegram رقميًا، ويطلب WhatsApp رقم هاتف

  </Tab>
  <Tab title="الوضع المتقدم (تحكم كامل)">
    - يعرض كل خطوة: الوضع، ومساحة العمل، وGateway، والقنوات، والخدمة الخفية، وSkills

  </Tab>
</Tabs>

يستخدم الوضع البعيد (`--mode remote`) دائمًا التدفق المتقدم؛ فهو لا يفعل سوى تهيئة هذا الجهاز للاتصال بـ Gateway في مكان آخر، ولا يثبّت أو يغيّر أي شيء على المضيف البعيد مطلقًا.

## ما الذي يهيئه الإعداد الأولي التقليدي

يمر الوضع المحلي (الافتراضي) بهذه الخطوات:

1. **النموذج/المصادقة** - اختر تدفق مصادقة للمزوّد (مفتاح API أو OAuth أو مصادقة يدوية خاصة بالمزوّد)، بما في ذلك المزوّد المخصص (متوافق مع OpenAI، أو متوافق مع OpenAI Responses، أو متوافق مع Anthropic، أو اكتشاف تلقائي غير معروف). اختر نموذجًا افتراضيًا.
   يستخدم إعداد مفتاح API جديد لـ OpenAI الإعداد الافتراضي `openai/gpt-5.6` (يُحل معرّف API المباشر المجرد إلى Sol)؛ ويستخدم إعداد ChatGPT/Codex الجديد الإعداد الافتراضي `openai/gpt-5.6-sol`. تحافظ إعادة تشغيل الإعداد على نموذج صريح حالي، بما في ذلك `openai/gpt-5.5`. حدّد `openai/gpt-5.5` صراحةً إذا كان الحساب لا يتيح GPT-5.6.
   ملاحظة أمنية: إذا كان هذا الوكيل سيشغّل أدوات أو يعالج محتوى Webhook/الخطافات، ففضّل أقوى نموذج متاح من أحدث جيل وأبقِ سياسة الأدوات صارمة؛ إذ يسهل تعريض الفئات الأضعف أو الأقدم لحقن الأوامر.
   بالنسبة إلى عمليات التشغيل غير التفاعلية، يخزّن `--secret-input-mode ref` مراجع مدعومة بالبيئة بدلًا من قيم مفاتيح API بنص صريح؛ ويجب أن يكون متغير البيئة المشار إليه معيّنًا بالفعل، وإلا يفشل الإعداد الأولي فورًا. يمكن لوضع مرجع السر التفاعلي الإشارة إلى متغير بيئة أو مرجع مزوّد مهيأ (`file` أو `exec`) مع إجراء فحص تمهيدي سريع قبل الحفظ. بعد إعداد النموذج/المصادقة، يقدّم المعالج اختبار إكمال مباشرًا اختياريًا؛ ويمكن عند الفشل العودة إلى إعداد النموذج/المصادقة مرة واحدة أو تجاهله من دون حظر بقية المعالج التقليدي. لا يؤدي تجاهله إلى فتح OpenClaw؛ إذ لا يزال إعداد المحادثة يتطلب نجاح فحص الاستدلال.
2. **مساحة العمل** - دليل ملفات الوكيل (الافتراضي `~/.openclaw/workspace`). يضيف ملفات التمهيد الأولية.
3. **Gateway** - المنفذ، وعنوان الربط، ووضع المصادقة، وإتاحة Tailscale. في وضع الرمز المميز التفاعلي، اختر تخزين الرمز المميز بنص صريح (الافتراضي) أو اشترك في SecretRef. مسار SecretRef غير التفاعلي: `--gateway-token-ref-env <ENV_VAR>`.
4. **القنوات** - قنوات المحادثة المضمّنة وقنوات الإضافات الرسمية، بما في ذلك Discord وFeishu وGoogle Chat وiMessage وMattermost وMicrosoft Teams وQQ Bot وSignal وSlack وTelegram وWhatsApp وغيرها.
5. **الخدمة الخفية** - يثبّت LaunchAgent ‏(macOS)، أو وحدة مستخدم systemd ‏(Linux/WSL2)، أو مهمة Windows مجدولة أصلية مع خيار احتياطي لكل مستخدم في مجلد Startup.
   إذا كانت مصادقة الرمز المميز مطلوبة وكان `gateway.auth.token` مُدارًا بواسطة SecretRef، فإن تثبيت الخدمة الخفية يتحقق منه، لكنه لا يحفظ رمزًا مميزًا محلولًا في بيانات تعريف بيئة خدمة المشرف؛ ويحظر SecretRef غير المحلول التثبيت مع تقديم إرشادات. إذا كان كل من `gateway.auth.token` و`gateway.auth.password` معيّنًا بينما يظل `gateway.auth.mode` غير معيّن، فسيُحظر التثبيت حتى تعيّن الوضع صراحةً.
6. **فحص السلامة** - يشغّل Gateway ويتحقق من إمكانية الوصول إليه.
7. **Skills** - يثبّت المهارات الموصى بها وتبعياتها الاختيارية.

<Note>
لا تمحو إعادة تشغيل الإعداد الأولي أي شيء **مطلقًا** ما لم تختر صراحةً **إعادة التعيين** (أو تمرّر `--reset`). يستخدم `--reset` في CLI افتراضيًا التهيئة وبيانات الاعتماد والجلسات؛ استخدم `--reset-scope full` لإزالة مساحة العمل أيضًا. إذا كانت التهيئة غير صالحة أو تحتوي على مفاتيح قديمة، يطلب منك الإعداد الأولي تشغيل `openclaw doctor` أولًا.
</Note>

يشغّل `--flow import` تدفق ترحيل مكتشفًا (مثل Hermes) في المعالج التقليدي بدلًا من الإعداد الجديد؛ راجع [الترحيل](/ar/cli/migrate) وأدلة الترحيل ضمن [التثبيت](/ar/install/migrating-hermes). يُعد `openclaw onboard --modern` اسمًا بديلًا للتوافق مع [OpenClaw](/cli/openclaw). وهو يستخدم بوابة الاستدلال نفسها المستخدمة في `openclaw setup`: يبدأ الاستدلال الذي تم التحقق منه تشغيل المساعد، بينما يعيد الفشل التفاعلي إلى إعداد الاستدلال الموجّه.

## إضافة وكيل آخر

استخدم `openclaw agents add <name>` لإنشاء وكيل منفصل له مساحة عمل وجلسات وملفات تعريف مصادقة خاصة به. يؤدي التشغيل من دون `--workspace` إلى بدء تدفق تفاعلي للاسم، ومساحة العمل، والمصادقة، والقنوات، وعمليات الربط؛ وليس هذا معالج `openclaw onboard` الكامل.

ما يعيّنه:

- `agents.list[].name`
- `agents.list[].workspace`
- `agents.list[].agentDir`

ملاحظات:

- مساحة العمل الافتراضية: `~/.openclaw/workspace-<agentId>` (أو ضمن `agents.defaults.workspace` إذا كان معيّنًا).
- أضف `bindings` لتوجيه الرسائل الواردة إلى هذا الوكيل (يمكن للإعداد الأولي تنفيذ ذلك نيابةً عنك).
- علامات الوضع غير التفاعلي: `--model`، و`--agent-dir`، و`--bind`، و`--non-interactive`.
