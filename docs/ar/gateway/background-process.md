# أداة التنفيذ في الخلفية + العملية

يشغّل OpenClaw أوامر الصدفة عبر أداة `exec` ويحتفظ بالمهام طويلة التشغيل في الذاكرة. تدير أداة `process` تلك الجلسات في الخلفية.

## أداة `exec`

المعاملات الأساسية:

- `command` (مطلوب)
- `yieldMs` (الافتراضي 10000): النقل التلقائي إلى الخلفية بعد هذا التأخير
- `background` (قيمة منطقية): النقل إلى الخلفية فورًا
- `timeout` (بالثواني، الافتراضي `tools.exec.timeoutSec`): اقتل العملية بعد انتهاء هذه المهلة؛ اضبط `timeout: 0` فقط لتعطيل مهلة عملية exec لتلك الاستدعاء
- `elevated` (قيمة منطقية): التشغيل خارج صندوق العزل إذا كان الوضع المرتفع مفعّلًا/مسموحًا (`gateway` افتراضيًا، أو `node` عندما يكون هدف exec هو `node`)
- هل تحتاج إلى TTY حقيقي؟ اضبط `pty: true`.
- `workdir`, `env`

السلوك:

- عمليات التشغيل في المقدمة تعيد المخرجات مباشرة.
- عند النقل إلى الخلفية (صراحةً أو بسبب المهلة)، تعيد الأداة `status: "running"` + `sessionId` وذيلًا قصيرًا.
- عمليات الخلفية و`yieldMs` ترث `tools.exec.timeoutSec` ما لم يوفّر الاستدعاء `timeout` صريحًا.
- تُحفظ المخرجات في الذاكرة إلى أن يتم استطلاع الجلسة أو مسحها.
- إذا كانت أداة `process` غير مسموح بها، فإن `exec` يعمل تزامنيًا ويتجاهل `yieldMs`/`background`.
- تتلقى أوامر exec المُنشأة `OPENCLAW_SHELL=exec` لقواعد الصدفة/الملف الشخصي الواعية بالسياق.
- للعمل طويل التشغيل الذي يبدأ الآن، ابدأه مرة واحدة واعتمد على إيقاظ الإكمال التلقائي عند تمكينه وعندما يصدر الأمر مخرجات أو يفشل.
- إذا كان إيقاظ الإكمال التلقائي غير متاح، أو كنت تحتاج إلى تأكيد نجاح هادئ لأمر خرج بنجاح دون مخرجات، فاستخدم `process` لتأكيد الإكمال.
- لا تحاكِ التذكيرات أو المتابعات المؤجلة بحلقات `sleep` أو الاستطلاع المتكرر؛ استخدم cron للأعمال المستقبلية.

## ربط العمليات الفرعية

عند إنشاء عمليات فرعية طويلة التشغيل خارج أدوات exec/process (على سبيل المثال، عمليات إعادة تشغيل CLI أو مساعدو gateway)، أرفق مساعد جسر العملية الفرعية بحيث تُمرَّر إشارات الإنهاء وتُفصل المستمعات عند الخروج/الخطأ. يتجنب هذا العمليات اليتيمة على systemd ويحافظ على اتساق سلوك إيقاف التشغيل عبر المنصات.

تجاوزات البيئة:

- `PI_BASH_YIELD_MS`: مدة الانتظار الافتراضية (مللي ثانية)
- `PI_BASH_MAX_OUTPUT_CHARS`: حد المخرجات في الذاكرة (أحرف)
- `OPENCLAW_BASH_PENDING_MAX_OUTPUT_CHARS`: حد stdout/stderr المعلّق لكل تدفق (أحرف)
- `PI_BASH_JOB_TTL_MS`: مدة TTL للجلسات المنتهية (مللي ثانية، محدودة بين 1د و3س)

الإعدادات (مفضّلة):

- `tools.exec.backgroundMs` (الافتراضي 10000)
- `tools.exec.timeoutSec` (الافتراضي 1800)
- `tools.exec.cleanupMs` (الافتراضي 1800000)
- `tools.exec.notifyOnExit` (الافتراضي true): يضع حدث نظام في الطابور + يطلب Heartbeat عند خروج exec يعمل في الخلفية.
- `tools.exec.notifyOnExitEmptySuccess` (الافتراضي false): عند true، يضع أيضًا أحداث الإكمال في الطابور لعمليات الخلفية الناجحة التي لم تُنتج أي مخرجات.

## أداة `process`

الإجراءات:

- `list`: الجلسات الجارية + المنتهية
- `poll`: تصريف المخرجات الجديدة لجلسة (ويبلغ أيضًا عن حالة الخروج)
- `log`: قراءة المخرجات المجمّعة (يدعم `offset` + `limit`)
- `write`: إرسال stdin (`data`، و`eof` اختياري)
- `send-keys`: إرسال رموز مفاتيح صريحة أو بايتات إلى جلسة مدعومة بـ PTY
- `submit`: إرسال Enter / carriage return إلى جلسة مدعومة بـ PTY
- `paste`: إرسال نص حرفي، ملفوف اختياريًا في وضع اللصق بين أقواس
- `kill`: إنهاء جلسة خلفية
- `clear`: إزالة جلسة منتهية من الذاكرة
- `remove`: القتل إذا كانت جارية، وإلا المسح إذا كانت منتهية

ملاحظات:

- تُدرج/تُستبقى في الذاكرة الجلسات المنقولة إلى الخلفية فقط.
- تُفقد الجلسات عند إعادة تشغيل العملية (لا يوجد حفظ على القرص).
- لا تُحفظ سجلات الجلسات في سجل الدردشة إلا إذا شغّلت `process poll/log` وسُجّلت نتيجة الأداة.
- نطاق `process` لكل وكيل؛ فهي لا ترى إلا الجلسات التي بدأها ذلك الوكيل.
- استخدم `poll` / `log` للحالة، أو السجلات، أو تأكيد النجاح الهادئ، أو تأكيد الإكمال عندما يكون إيقاظ الإكمال التلقائي غير متاح.
- استخدم `write` / `send-keys` / `submit` / `paste` / `kill` عندما تحتاج إلى إدخال أو تدخل.
- يتضمن `process list` اسمًا مشتقًا `name` (فعل الأمر + الهدف) للفحص السريع.
- يستخدم `process log` قيم `offset`/`limit` المعتمدة على الأسطر.
- عند حذف كل من `offset` و`limit`، يعيد آخر 200 سطر ويتضمن تلميحًا للترقيم.
- عند توفير `offset` وحذف `limit`، يعيد من `offset` إلى النهاية (غير محدود بـ 200).
- الاستطلاع مخصص للحالة عند الطلب، وليس لجدولة حلقات الانتظار. إذا كان العمل يجب أن يحدث لاحقًا، فاستخدم cron بدلًا من ذلك.

## أمثلة

تشغيل مهمة طويلة واستطلاعها لاحقًا:

```json
{ "tool": "exec", "command": "sleep 5 && echo done", "yieldMs": 1000 }
```

```json
{ "tool": "process", "action": "poll", "sessionId": "<id>" }
```

البدء فورًا في الخلفية:

```json
{ "tool": "exec", "command": "npm run build", "background": true }
```

إرسال stdin:

```json
{ "tool": "process", "action": "write", "sessionId": "<id>", "data": "y\n" }
```

إرسال مفاتيح PTY:

```json
{ "tool": "process", "action": "send-keys", "sessionId": "<id>", "keys": ["C-c"] }
```

إرسال السطر الحالي:

```json
{ "tool": "process", "action": "submit", "sessionId": "<id>" }
```

لصق نص حرفي:

```json
{ "tool": "process", "action": "paste", "sessionId": "<id>", "text": "line1\nline2\n" }
```
