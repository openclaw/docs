يمكن لـ OpenClaw إنشاء ملف diagnostics zip محلي لتقارير الأخطاء. فهو يجمع
حالة Gateway المنقّحة، والسلامة، والسجلات، وشكل الإعدادات، وأحداث الاستقرار الحديثة
الخالية من الحمولات.

تعامل مع حزم التشخيص كما تتعامل مع الأسرار إلى أن تراجعها. فهي
مصممة لحذف الحمولات وبيانات الاعتماد أو تنقيحها، لكنها لا تزال تلخص
سجلات Gateway المحلية وحالة التشغيل على مستوى المضيف.

## البدء السريع

```bash
openclaw gateway diagnostics export
```

يطبع الأمر مسار zip المكتوب. لاختيار مسار:

```bash
openclaw gateway diagnostics export --output openclaw-diagnostics.zip
```

للأتمتة:

```bash
openclaw gateway diagnostics export --json
```

## أمر الدردشة

يمكن للمالكين استخدام `/diagnostics [note]` في الدردشة لطلب تصدير Gateway محلي.
استخدم هذا عندما يحدث الخطأ في محادثة حقيقية وتريد تقريرًا واحدًا
قابلًا للنسخ واللصق للدعم:

1. أرسل `/diagnostics` في المحادثة التي لاحظت فيها المشكلة. أضف
   ملاحظة قصيرة إذا كان ذلك مفيدًا، مثل `/diagnostics bad tool choice`.
2. يرسل OpenClaw تمهيد التشخيص ويطلب موافقة تنفيذ صريحة واحدة.
   تشغّل الموافقة `openclaw gateway diagnostics export --json`.
   لا توافق على التشخيص عبر قاعدة تسمح بكل شيء.
3. بعد الموافقة، يرد OpenClaw بتقرير قابل للصق يحتوي على مسار الحزمة المحلي،
   وملخص البيان، وملاحظات الخصوصية، ومعرّفات الجلسات ذات الصلة.

في الدردشات الجماعية، لا يزال بإمكان المالك تشغيل `/diagnostics`، لكن OpenClaw لا
ينشر تفاصيل التشخيص مرة أخرى في الدردشة المشتركة. يرسل التمهيد،
ومطالبات الموافقة، ونتيجة تصدير Gateway، وتفصيل جلسة/سلسلة Codex إلى
المالك عبر مسار الموافقة الخاص. لا تتلقى المجموعة إلا إشعارًا قصيرًا
بأن مسار التشخيص أُرسل بشكل خاص. إذا تعذر على OpenClaw العثور على مسار مالك
خاص، يفشل الأمر بصورة مغلقة ويطلب من المالك تشغيله من رسالة مباشرة.

عندما تستخدم جلسة OpenClaw النشطة أداة OpenAI Codex الأصلية،
تغطي موافقة التنفيذ نفسها أيضًا رفع ملاحظات OpenAI لسلاسل تشغيل Codex
التي يعرفها OpenClaw. هذا الرفع منفصل عن ملف Gateway zip المحلي
ولا يظهر إلا لجلسات أداة Codex. قبل الموافقة، يوضح
المطلب أن الموافقة على التشخيص سترسل أيضًا ملاحظات Codex، لكنه
لا يسرد معرّفات جلسات أو سلاسل Codex. بعد الموافقة، يسرد رد الدردشة
القنوات، ومعرّفات جلسات OpenClaw، ومعرّفات سلاسل Codex، وأوامر الاستئناف المحلية
للسلاسل التي أُرسلت إلى خوادم OpenAI. إذا رفضت أو تجاهلت
الموافقة، لا يشغّل OpenClaw التصدير، ولا يرسل ملاحظات Codex، ولا
يطبع معرّفات Codex.

هذا يجعل حلقة تصحيح Codex الشائعة قصيرة: لاحظ السلوك السيئ في
Telegram، أو Discord، أو قناة أخرى، وشغّل `/diagnostics`، ووافق مرة واحدة، وشارك
التقرير مع الدعم، ثم شغّل أمر `codex resume <thread-id>` المطبوع
محليًا إذا أردت فحص سلسلة Codex الأصلية بنفسك. راجع
[أداة Codex](/ar/plugins/codex-harness#inspect-a-codex-thread-from-the-cli) لمعرفة
سير عمل الفحص هذا.

## ما يحتويه التصدير

يتضمن ملف zip ما يلي:

- `summary.md`: نظرة عامة قابلة للقراءة البشرية للدعم.
- `diagnostics.json`: ملخص قابل للقراءة آليًا للإعدادات، والسجلات، والحالة، والسلامة،
  وبيانات الاستقرار.
- `manifest.json`: بيانات تعريف التصدير وقائمة الملفات.
- شكل إعدادات منقّح وتفاصيل إعدادات غير سرية.
- ملخصات سجلات منقّحة وأسطر سجلات حديثة منقّحة.
- لقطات حالة وسلامة Gateway بأفضل جهد.
- `stability/latest.json`: أحدث حزمة استقرار محفوظة، عند توفرها.

يكون التصدير مفيدًا حتى عندما يكون Gateway غير سليم. إذا تعذر على Gateway
الرد على طلبات الحالة أو السلامة، فستظل السجلات المحلية، وشكل الإعدادات، وأحدث
حزمة استقرار تُجمع عند توفرها.

## نموذج الخصوصية

صُممت التشخيصات لتكون قابلة للمشاركة. يحتفظ التصدير بالبيانات التشغيلية
التي تساعد على تصحيح الأخطاء، مثل:

- أسماء الأنظمة الفرعية، ومعرّفات plugins، ومعرّفات المزوّدين، ومعرّفات القنوات، والأوضاع المكوّنة
- رموز الحالة، والمدد، وأعداد البايت، وحالة الصف، وقراءات الذاكرة
- بيانات تعريف سجلات منقّحة ورسائل تشغيلية منقّحة
- شكل الإعدادات وإعدادات الميزات غير السرية

يحذف التصدير أو ينقّح:

- نص الدردشة، والمطالبات، والتعليمات، وأجسام webhook، ومخرجات الأدوات
- بيانات الاعتماد، ومفاتيح API، والرموز، وملفات تعريف الارتباط، والقيم السرية
- أجسام الطلبات أو الاستجابات الخام
- معرّفات الحسابات، ومعرّفات الرسائل، ومعرّفات الجلسات الخام، وأسماء المضيفين، وأسماء المستخدمين المحلية

عندما تبدو رسالة سجل كنص مستخدم، أو دردشة، أو مطالبة، أو حمولة أداة، فإن
التصدير يحتفظ فقط بأن رسالة حُذفت وعدد البايتات.

## مسجل الاستقرار

يسجل Gateway بشكل افتراضي تدفق استقرار محدودًا وخاليًا من الحمولات عندما
تكون التشخيصات مفعّلة. وهو مخصص للحقائق التشغيلية، وليس للمحتوى.

يسجل Heartbeat التشخيصي نفسه عينات الحيوية عندما يستمر Gateway في
العمل لكن حلقة أحداث Node.js أو CPU تبدو مشبعة. تتضمن أحداث
`diagnostic.liveness.warning` هذه تأخر حلقة الأحداث، واستخدام حلقة الأحداث،
ونسبة أنوية CPU، وأعداد الجلسات النشطة/المنتظرة/المصطفة. تبقى
العينات الخاملة في القياسات عن بُعد عند مستوى `info`. تصبح عينات الحيوية
تحذيرات Gateway فقط عندما يكون العمل منتظرًا أو مصطفًا، أو عندما يتداخل العمل النشط مع
تأخر مستمر في حلقة الأحداث. تبقى قفزات التأخير القصوى العابرة أثناء
عمل خلفية سليم خلاف ذلك في سجلات التصحيح. وهي لا تعيد تشغيل Gateway
بمفردها.

افحص المسجل الحي:

```bash
openclaw gateway stability
openclaw gateway stability --type payload.large
openclaw gateway stability --json
```

افحص أحدث حزمة استقرار محفوظة بعد خروج قاتل، أو مهلة إيقاف تشغيل،
أو فشل بدء تشغيل إعادة التشغيل:

```bash
openclaw gateway stability --bundle latest
```

أنشئ ملف diagnostics zip من أحدث حزمة محفوظة:

```bash
openclaw gateway stability --bundle latest --export
```

توجد الحزم المحفوظة ضمن `~/.openclaw/logs/stability/` عند وجود أحداث.

## خيارات مفيدة

```bash
openclaw gateway diagnostics export \
  --output openclaw-diagnostics.zip \
  --log-lines 5000 \
  --log-bytes 1000000
```

- `--output <path>`: اكتب إلى مسار zip محدد.
- `--log-lines <count>`: الحد الأقصى لأسطر السجل المنقّحة المطلوب تضمينها.
- `--log-bytes <bytes>`: الحد الأقصى لبايتات السجل المطلوب فحصها.
- `--url <url>`: عنوان URL لـ Gateway WebSocket للقطات الحالة والسلامة.
- `--token <token>`: رمز Gateway للقطات الحالة والسلامة.
- `--password <password>`: كلمة مرور Gateway للقطات الحالة والسلامة.
- `--timeout <ms>`: مهلة لقطات الحالة والسلامة.
- `--no-stability-bundle`: تخطَّ البحث عن حزمة الاستقرار المحفوظة.
- `--json`: اطبع بيانات تعريف التصدير القابلة للقراءة آليًا.

## تعطيل التشخيصات

التشخيصات مفعّلة افتراضيًا. لتعطيل مسجل الاستقرار
وجمع أحداث التشخيص:

```json5
{
  diagnostics: {
    enabled: false,
  },
}
```

يقلل تعطيل التشخيصات تفاصيل تقارير الأخطاء. ولا يؤثر في تسجيل
Gateway العادي.
