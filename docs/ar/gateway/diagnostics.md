يمكن لـ OpenClaw إنشاء ملف zip تشخيصي محلي آمن لإرفاقه بتقارير
الأخطاء. وهو يجمع بين حالة Gateway المنقحة، والسلامة، والسجلات، وشكل الإعداد،
وأحداث الاستقرار الحديثة الخالية من الحمولات.

## البدء السريع

```bash
openclaw gateway diagnostics export
```

يطبع الأمر مسار ملف zip الذي تمت كتابته. ولاختيار مسار:

```bash
openclaw gateway diagnostics export --output openclaw-diagnostics.zip
```

وللأتمتة:

```bash
openclaw gateway diagnostics export --json
```

## ما الذي يحتويه التصدير

يتضمن ملف zip ما يلي:

- `summary.md`: نظرة عامة قابلة للقراءة البشرية للدعم.
- `diagnostics.json`: ملخص قابل للقراءة آليًا للإعداد، والسجلات، والحالة، والسلامة،
  وبيانات الاستقرار.
- `manifest.json`: بيانات التصدير الوصفية وقائمة الملفات.
- شكل إعداد منقح وتفاصيل إعداد غير سرية.
- ملخصات سجلات منقحة وأسطر سجلات حديثة منقحة.
- لقطات حالة وسلامة Gateway بأفضل جهد.
- `stability/latest.json`: أحدث حزمة استقرار ثابتة، عند توفرها.

يكون التصدير مفيدًا حتى عندما تكون Gateway غير سليمة. وإذا تعذر على Gateway
الرد على طلبات الحالة أو السلامة، فلا يزال يتم جمع السجلات المحلية، وشكل الإعداد، وأحدث
حزمة استقرار عند توفرها.

## نموذج الخصوصية

صُممت التشخيصات بحيث تكون قابلة للمشاركة. ويحتفظ التصدير بالبيانات التشغيلية
التي تساعد على التصحيح، مثل:

- أسماء الأنظمة الفرعية، ومعرّفات Plugins، ومعرّفات الموفّرين، ومعرّفات القنوات، والأوضاع المضبوطة
- رموز الحالة، والمدد، وأعداد البايتات، وحالة الطابور، وقراءات الذاكرة
- بيانات تعريف السجل المنقحة والرسائل التشغيلية المنقحة
- شكل الإعداد وإعدادات الميزات غير السرية

ويحذف التصدير أو ينقح:

- نص الدردشة، والمطالبات، والتعليمات، وأجسام Webhook، ومخرجات الأدوات
- بيانات الاعتماد، ومفاتيح API، والرموز المميزة، وملفات تعريف الارتباط، والقيم السرية
- أجسام الطلبات أو الاستجابات الخام
- معرّفات الحسابات، ومعرّفات الرسائل، ومعرّفات الجلسات الخام، وأسماء المضيفين، وأسماء المستخدمين المحليين

عندما تبدو رسالة السجل كنص حمولة خاص بالمستخدم أو الدردشة أو المطالبة أو الأداة،
فإن التصدير يحتفظ فقط بوجود رسالة محذوفة وعدد البايتات.

## مسجل الاستقرار

تسجل Gateway افتراضيًا تدفق استقرار محدودًا وخاليًا من الحمولات عندما
تكون التشخيصات مفعلة. وهو مخصص للحقائق التشغيلية، لا للمحتوى.

افحص المسجل الحي:

```bash
openclaw gateway stability
openclaw gateway stability --type payload.large
openclaw gateway stability --json
```

افحص أحدث حزمة استقرار ثابتة بعد خروج قاتل، أو مهلة إيقاف تشغيل،
أو فشل بدء تشغيل بعد إعادة تشغيل:

```bash
openclaw gateway stability --bundle latest
```

أنشئ ملف zip تشخيصيًا من أحدث حزمة ثابتة:

```bash
openclaw gateway stability --bundle latest --export
```

توجد الحزم الثابتة ضمن `~/.openclaw/logs/stability/` عند وجود أحداث.

## خيارات مفيدة

```bash
openclaw gateway diagnostics export \
  --output openclaw-diagnostics.zip \
  --log-lines 5000 \
  --log-bytes 1000000
```

- `--output <path>`: الكتابة إلى مسار zip محدد.
- `--log-lines <count>`: الحد الأقصى لأسطر السجل المنقحة المراد تضمينها.
- `--log-bytes <bytes>`: الحد الأقصى لبايتات السجل التي سيتم فحصها.
- `--url <url>`: عنوان URL لـ Gateway WebSocket لالتقاط لقطات الحالة والسلامة.
- `--token <token>`: رمز Gateway لالتقاط لقطات الحالة والسلامة.
- `--password <password>`: كلمة مرور Gateway لالتقاط لقطات الحالة والسلامة.
- `--timeout <ms>`: مهلة لقطات الحالة والسلامة.
- `--no-stability-bundle`: تخطي البحث عن حزمة الاستقرار الثابتة.
- `--json`: طباعة بيانات التصدير الوصفية بصيغة قابلة للقراءة آليًا.

## تعطيل التشخيصات

تكون التشخيصات مفعلة افتراضيًا. ولتعطيل مسجل الاستقرار
وجمع أحداث التشخيص:

```json5
{
  diagnostics: {
    enabled: false,
  },
}
```

يؤدي تعطيل التشخيصات إلى تقليل تفاصيل تقارير الأخطاء. ولا يؤثر ذلك في
تسجيل Gateway العادي.
