# استخدام الرموز وتكاليفها

يتتبع OpenClaw **الرموز**، لا الأحرف. الرموز خاصة بكل نموذج، لكن معظم
النماذج ذات أسلوب OpenAI يبلغ متوسطها نحو 4 أحرف لكل رمز في النص الإنجليزي.

## كيف يُبنى موجه النظام

يجمع OpenClaw موجه النظام الخاص به في كل تشغيل. ويشمل:

- قائمة الأدوات + أوصافًا قصيرة
- قائمة Skills (البيانات الوصفية فقط؛ تُحمَّل التعليمات عند الطلب باستخدام `read`).
  كتلة Skills الموجزة مقيّدة بواسطة `skills.limits.maxSkillsPromptChars`،
  مع إمكانية تجاوز اختيارية لكل وكيل عند
  `agents.list[].skillsLimits.maxSkillsPromptChars`.
- تعليمات التحديث الذاتي
- مساحة العمل + ملفات التمهيد (`AGENTS.md`، و`SOUL.md`، و`TOOLS.md`، و`IDENTITY.md`، و`USER.md`، و`HEARTBEAT.md`، و`BOOTSTRAP.md` عندما يكون جديدًا، إضافة إلى `MEMORY.md` عند وجوده). لا يُحقن ملف الجذر `memory.md` بالحروف الصغيرة؛ فهو إدخال إصلاح قديم لـ `openclaw doctor --fix` عند اقترانه بـ `MEMORY.md`. تُقتطع الملفات الكبيرة بواسطة `agents.defaults.bootstrapMaxChars` (القيمة الافتراضية: 12000)، ويُقيَّد إجمالي حقن التمهيد بواسطة `agents.defaults.bootstrapTotalMaxChars` (القيمة الافتراضية: 60000). ملفات `memory/*.md` اليومية ليست جزءًا من موجه التمهيد العادي؛ تبقى متاحة عند الطلب عبر أدوات الذاكرة في الأدوار العادية، لكن تشغيلات النموذج عند إعادة الضبط/بدء التشغيل يمكن أن تسبق الدور الأول بكتلة سياق بدء تشغيل لمرة واحدة تحتوي على ذاكرة يومية حديثة. أوامر الدردشة المجردة `/new` و`/reset` يُقرّ بها دون استدعاء النموذج. تتحكم `agents.defaults.startupContext` في تمهيد بدء التشغيل.
- الوقت (UTC + المنطقة الزمنية للمستخدم)
- وسوم الرد + سلوك Heartbeat
- بيانات وصفية لوقت التشغيل (المضيف/نظام التشغيل/النموذج/التفكير)

راجع التفصيل الكامل في [موجه النظام](/ar/concepts/system-prompt).

## ما الذي يُحتسب ضمن نافذة السياق

كل ما يتلقاه النموذج يُحتسب ضمن حد السياق:

- موجه النظام (كل الأقسام المذكورة أعلاه)
- سجل المحادثة (رسائل المستخدم + المساعد)
- استدعاءات الأدوات ونتائج الأدوات
- المرفقات/النصوص المفرغة (الصور، والصوت، والملفات)
- ملخصات Compaction ومخرجات التقليم
- أغلفة المزوّد أو ترويسات السلامة (غير مرئية، لكنها لا تزال محتسبة)

لبعض الأسطح كثيفة وقت التشغيل حدود صريحة خاصة بها:

- `agents.defaults.contextLimits.memoryGetMaxChars`
- `agents.defaults.contextLimits.memoryGetDefaultLines`
- `agents.defaults.contextLimits.toolResultMaxChars`
- `agents.defaults.contextLimits.postCompactionMaxChars`

توجد التجاوزات لكل وكيل تحت `agents.list[].contextLimits`. هذه المفاتيح
مخصصة للمقتطفات المحدودة في وقت التشغيل والكتل المحقونة التي يملكها وقت التشغيل. وهي
منفصلة عن حدود التمهيد، وحدود سياق بدء التشغيل، وحدود موجه Skills.

بالنسبة إلى الصور، يقلّص OpenClaw حمولات صور النصوص المفرغة/الأدوات قبل استدعاءات المزوّد.
استخدم `agents.defaults.imageMaxDimensionPx` (القيمة الافتراضية: `1200`) لضبط ذلك:

- القيم الأقل تقلل عادةً استخدام رموز الرؤية وحجم الحمولة.
- القيم الأعلى تحافظ على مزيد من التفاصيل المرئية للقطات الشاشة الكثيفة بـ OCR/واجهة المستخدم.

للحصول على تفصيل عملي (لكل ملف محقون، والأدوات، وSkills، وحجم موجه النظام)، استخدم `/context list` أو `/context detail`. راجع [السياق](/ar/concepts/context).

## كيف ترى استخدام الرموز الحالي

استخدم هذه في الدردشة:

- `/status` → **بطاقة حالة غنية بالرموز التعبيرية** تتضمن نموذج الجلسة، واستخدام السياق،
  ورموز الإدخال/الإخراج لآخر استجابة، و**التكلفة المقدّرة** (مفتاح API فقط).
- `/usage off|tokens|full` → يضيف **تذييل استخدام لكل استجابة** إلى كل رد.
  - يستمر لكل جلسة (مخزّن باسم `responseUsage`).
  - مصادقة OAuth **تخفي التكلفة** (الرموز فقط).
- `/usage cost` → يعرض ملخص تكلفة محليًا من سجلات جلسة OpenClaw.

أسطح أخرى:

- **TUI/Web TUI:** يتم دعم `/status` + `/usage`.
- **CLI:** يعرض `openclaw status --usage` و`openclaw channels list`
  نوافذ حصص المزوّدين المعيارية (`X% left`، لا تكاليف لكل استجابة).
  مزوّدو نوافذ الاستخدام الحاليون: Anthropic، وGitHub Copilot، وGemini CLI،
  وOpenAI Codex، وMiniMax، وXiaomi، وz.ai.

توحّد أسطح الاستخدام الأسماء البديلة الشائعة للحقول الأصلية لدى المزوّدين قبل العرض.
بالنسبة إلى حركة Responses لعائلة OpenAI، يشمل ذلك كلًا من `input_tokens` /
`output_tokens` و`prompt_tokens` / `completion_tokens`، لذلك لا تغيّر أسماء الحقول
الخاصة بالنقل `/status` أو `/usage` أو ملخصات الجلسة.
كما يُوحّد استخدام Gemini CLI بصيغة JSON: يأتي نص الرد من `response`، ويتم ربط
`stats.cached` بـ `cacheRead` مع استخدام `stats.input_tokens - stats.cached`
عندما يحذف CLI حقل `stats.input` الصريح.
بالنسبة إلى حركة Responses الأصلية لعائلة OpenAI، تُوحّد أسماء استخدام WebSocket/SSE
بالطريقة نفسها، وتعود المجاميع إلى الإدخال + الإخراج الموحّدين عندما يكون
`total_tokens` مفقودًا أو `0`.
عندما تكون لقطة الجلسة الحالية متفرقة، يمكن لـ `/status` و`session_status`
أيضًا استرداد عدادات الرموز/الذاكرة المؤقتة وتسمية نموذج وقت التشغيل النشط من
أحدث سجل استخدام في النص المفرغ. تظل القيم الحية غير الصفرية الحالية لها
الأسبقية على قيم الرجوع إلى النص المفرغ، ويمكن أن تفوز مجاميع النصوص المفرغة
الأكبر والموجهة للموجهات عندما تكون المجاميع المخزّنة مفقودة أو أصغر.
تأتي مصادقة الاستخدام لنوافذ حصص المزوّدين من خطافات خاصة بالمزوّد عندما
تكون متاحة؛ وإلا يعود OpenClaw إلى مطابقة بيانات اعتماد OAuth/مفتاح API
من ملفات تعريف المصادقة أو env أو التكوين.
تحتفظ إدخالات نص المساعد المفرغ بشكل الاستخدام الموحّد نفسه، بما في ذلك
`usage.cost` عندما يكون للنموذج النشط تسعير مهيأ ويعيد المزوّد بيانات وصفية للاستخدام.
يوفر ذلك لـ `/usage cost` وحالة الجلسة المستندة إلى النصوص المفرغة مصدرًا ثابتًا
حتى بعد زوال حالة وقت التشغيل الحية.

يبقي OpenClaw محاسبة استخدام المزوّدين منفصلة عن لقطة السياق الحالية.
يمكن أن يتضمن `usage.total` لدى المزوّد إدخالًا مخزنًا مؤقتًا، وإخراجًا، واستدعاءات
نموذج متعددة ضمن حلقة أدوات، لذلك فهو مفيد للتكلفة والقياس عن بُعد لكنه قد يبالغ
في نافذة السياق الحية. تستخدم عروض السياق والتشخيصات أحدث لقطة موجه
(`promptTokens`، أو آخر استدعاء نموذج عندما لا تكون لقطة الموجه متاحة)
لـ `context.used`.

## تقدير التكلفة (عند العرض)

تُقدَّر التكاليف من تكوين تسعير نموذجك:

```
models.providers.<provider>.models[].cost
```

هذه هي **الدولارات الأمريكية لكل مليون رمز** لـ `input`، و`output`، و`cacheRead`، و
`cacheWrite`. إذا كان التسعير مفقودًا، يعرض OpenClaw الرموز فقط. رموز OAuth
لا تعرض تكلفة بالدولار مطلقًا.

ينفّذ بدء تشغيل Gateway أيضًا تمهيد تسعير اختياريًا في الخلفية لمراجع النماذج
المهيأة التي لا تملك تسعيرًا محليًا بالفعل. يجلب هذا التمهيد كتالوجات تسعير
OpenRouter وLiteLLM البعيدة. عيّن
`models.pricing.enabled: false` لتخطي جلب كتالوجات بدء التشغيل هذه على الشبكات غير المتصلة
أو المقيّدة؛ وتستمر إدخالات `models.providers.*.models[].cost` الصريحة
في قيادة تقديرات التكلفة المحلية.

## مدة صلاحية الذاكرة المؤقتة وتأثير التقليم

لا ينطبق التخزين المؤقت لموجهات المزوّد إلا ضمن نافذة مدة صلاحية الذاكرة المؤقتة. يمكن لـ OpenClaw
تشغيل **تقليص مدة صلاحية الذاكرة المؤقتة** اختياريًا: يقلّم الجلسة بمجرد انتهاء
مدة صلاحية الذاكرة المؤقتة، ثم يعيد ضبط نافذة الذاكرة المؤقتة بحيث تستطيع الطلبات اللاحقة
إعادة استخدام السياق المخزن مؤقتًا حديثًا بدلًا من إعادة تخزين السجل الكامل مؤقتًا. يحافظ هذا على
انخفاض تكاليف كتابة الذاكرة المؤقتة عندما تصبح الجلسة خاملة بعد مدة الصلاحية.

اضبطه في [تكوين Gateway](/ar/gateway/configuration) وراجع
تفاصيل السلوك في [تقليم الجلسة](/ar/concepts/session-pruning).

يمكن لـ Heartbeat إبقاء الذاكرة المؤقتة **دافئة** عبر فترات الخمول. إذا كانت مدة صلاحية ذاكرة نموذجك المؤقتة
`1h`، فإن ضبط فاصل Heartbeat على أقل من ذلك بقليل (مثلًا `55m`) يمكن أن يتجنب
إعادة تخزين الموجه الكامل مؤقتًا، مما يقلل تكاليف كتابة الذاكرة المؤقتة.

في إعدادات متعددة الوكلاء، يمكنك الاحتفاظ بتكوين نموذج مشترك واحد وضبط سلوك الذاكرة المؤقتة
لكل وكيل باستخدام `agents.list[].params.cacheRetention`.

للحصول على دليل كامل لكل مفتاح على حدة، راجع [التخزين المؤقت للموجهات](/ar/reference/prompt-caching).

بالنسبة إلى تسعير API من Anthropic، تكون قراءات الذاكرة المؤقتة أرخص بكثير من رموز الإدخال،
بينما تُفوتر كتابات الذاكرة المؤقتة بمضاعِف أعلى. راجع تسعير التخزين المؤقت للموجهات لدى Anthropic
للحصول على أحدث الأسعار ومضاعفات مدة الصلاحية:
[https://docs.anthropic.com/docs/build-with-claude/prompt-caching](https://docs.anthropic.com/docs/build-with-claude/prompt-caching)

### مثال: إبقاء ذاكرة مؤقتة مدتها 1h دافئة باستخدام Heartbeat

```yaml
agents:
  defaults:
    model:
      primary: "anthropic/claude-opus-4-6"
    models:
      "anthropic/claude-opus-4-6":
        params:
          cacheRetention: "long"
    heartbeat:
      every: "55m"
```

### مثال: حركة مختلطة مع استراتيجية ذاكرة مؤقتة لكل وكيل

```yaml
agents:
  defaults:
    model:
      primary: "anthropic/claude-opus-4-6"
    models:
      "anthropic/claude-opus-4-6":
        params:
          cacheRetention: "long" # خط أساس افتراضي لمعظم الوكلاء
  list:
    - id: "research"
      default: true
      heartbeat:
        every: "55m" # أبقِ الذاكرة المؤقتة الطويلة دافئة للجلسات العميقة
    - id: "alerts"
      params:
        cacheRetention: "none" # تجنب كتابات الذاكرة المؤقتة للإشعارات المتقطعة
```

تُدمج `agents.list[].params` فوق `params` الخاصة بالنموذج المحدد، بحيث يمكنك
تجاوز `cacheRetention` فقط ووراثة افتراضات النموذج الأخرى دون تغيير.

### مثال: تمكين ترويسة بيتا لسياق Anthropic بحجم 1M

نافذة سياق Anthropic بحجم 1M مقيّدة حاليًا بإصدار بيتا. يستطيع OpenClaw حقن
قيمة `anthropic-beta` المطلوبة عندما تمكّن `context1m` على نماذج Opus
أو Sonnet المدعومة.

```yaml
agents:
  defaults:
    models:
      "anthropic/claude-opus-4-6":
        params:
          context1m: true
```

يرتبط هذا بترويسة بيتا `context-1m-2025-08-07` الخاصة بـ Anthropic.

ينطبق هذا فقط عندما يتم تعيين `context1m: true` على إدخال ذلك النموذج.

المتطلب: يجب أن تكون بيانات الاعتماد مؤهلة لاستخدام السياق الطويل. إذا لم تكن كذلك،
تستجيب Anthropic بخطأ حد معدل من جانب المزوّد لذلك الطلب.

إذا صادقت مع Anthropic باستخدام رموز OAuth/الاشتراك (`sk-ant-oat-*`)،
يتخطى OpenClaw ترويسة بيتا `context-1m-*` لأن Anthropic ترفض حاليًا
هذا الجمع مع HTTP 401.

## نصائح لتقليل ضغط الرموز

- استخدم `/compact` لتلخيص الجلسات الطويلة.
- قلّص مخرجات الأدوات الكبيرة في سير عملك.
- خفّض `agents.defaults.imageMaxDimensionPx` للجلسات الكثيفة بلقطات الشاشة.
- اجعل أوصاف Skills قصيرة (تُحقن قائمة Skills في الموجه).
- فضّل النماذج الأصغر للعمل الاستكشافي كثير الكلام.

راجع [Skills](/ar/tools/skills) لمعرفة صيغة الحمل الزائد الدقيقة لقائمة Skills.
