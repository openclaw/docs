# استخدام الرموز والتكاليف

يتتبع OpenClaw **الرموز**، وليس الأحرف. تختلف الرموز حسب النموذج، لكن معظم النماذج بأسلوب OpenAI يكون متوسطها نحو 4 أحرف لكل رمز في النص الإنجليزي.

## كيف يُبنى system prompt

يقوم OpenClaw بتجميع system prompt الخاص به في كل تشغيل. ويتضمن ما يلي:

- قائمة الأدوات + أوصاف قصيرة
- قائمة Skills (بيانات وصفية فقط؛ تُحمَّل التعليمات عند الطلب باستخدام `read`).
  تكون كتلة Skills المضغوطة محدودة بواسطة `skills.limits.maxSkillsPromptChars`،
  مع إمكانية التجاوز لكل وكيل بشكل اختياري عند
  `agents.list[].skillsLimits.maxSkillsPromptChars`.
- تعليمات التحديث الذاتي
- ملفات مساحة العمل + ملفات bootstrap (`AGENTS.md` و`SOUL.md` و`TOOLS.md` و`IDENTITY.md` و`USER.md` و`HEARTBEAT.md` و`BOOTSTRAP.md` عند كونها جديدة، بالإضافة إلى `MEMORY.md` عند وجوده أو `memory.md` كبديل بأحرف صغيرة). تُقتطع الملفات الكبيرة بواسطة `agents.defaults.bootstrapMaxChars` (الافتراضي: 12000)، ويكون الحد الأقصى الإجمالي لحقن bootstrap هو `agents.defaults.bootstrapTotalMaxChars` (الافتراضي: 60000). لا تُعد ملفات `memory/*.md` اليومية جزءًا من prompt bootstrap العادي؛ بل تبقى عند الطلب عبر أدوات الذاكرة في الأدوار العادية، لكن يمكن لأوامر `/new` و`/reset` المجردة إلحاق كتلة سياق بدء تشغيل لمرة واحدة تحتوي على الذاكرة اليومية الحديثة لذلك الدور الأول. ويتم التحكم في هذه المقدمة الابتدائية بواسطة `agents.defaults.startupContext`.
- الوقت (UTC + المنطقة الزمنية للمستخدم)
- وسوم الرد + سلوك Heartbeat
- بيانات Runtime الوصفية (المضيف/نظام التشغيل/النموذج/التفكير)

راجع التفصيل الكامل في [System Prompt](/ar/concepts/system-prompt).

## ما الذي يُحتسب ضمن نافذة السياق

كل ما يتلقاه النموذج يُحتسب ضمن حد السياق:

- system prompt (جميع الأقسام المذكورة أعلاه)
- محفوظات المحادثة (رسائل المستخدم + المساعد)
- استدعاءات الأدوات ونتائجها
- المرفقات/التفريغات (الصور والصوت والملفات)
- ملخصات Compaction وعناصر التقليم
- أغلفة المزوّد أو ترويسات الأمان (غير مرئية، لكنها ما تزال محتسبة)

لبعض الأسطح الثقيلة وقت التشغيل حدود صريحة خاصة بها:

- `agents.defaults.contextLimits.memoryGetMaxChars`
- `agents.defaults.contextLimits.memoryGetDefaultLines`
- `agents.defaults.contextLimits.toolResultMaxChars`
- `agents.defaults.contextLimits.postCompactionMaxChars`

توجد التجاوزات الخاصة بكل وكيل تحت `agents.list[].contextLimits`. هذه المفاتيح
مخصصة للمقتطفات المحدودة وقت التشغيل والكتل المحقونة المملوكة لوقت التشغيل. وهي
منفصلة عن حدود bootstrap، وحدود سياق بدء التشغيل، وحدود prompt الخاصة بـ Skills.

بالنسبة للصور، يقوم OpenClaw بتصغير أحجام حمولات صور التفريغ/الأدوات قبل استدعاءات المزوّد.
استخدم `agents.defaults.imageMaxDimensionPx` (الافتراضي: `1200`) لضبط ذلك:

- القيم الأقل تقلل عادةً استخدام رموز الرؤية وحجم الحمولة.
- القيم الأعلى تحافظ على مزيد من التفاصيل البصرية لعمليات OCR ولقطات الشاشة الثقيلة بواجهات المستخدم.

للحصول على تفصيل عملي (لكل ملف محقون، والأدوات، وSkills، وحجم system prompt)، استخدم `/context list` أو `/context detail`. راجع [Context](/ar/concepts/context).

## كيفية رؤية استخدام الرموز الحالي

استخدم ما يلي في الدردشة:

- `/status` ← **بطاقة حالة غنية بالرموز التعبيرية** تعرض نموذج الجلسة، واستخدام السياق،
  ورموز الإدخال/الإخراج لآخر استجابة، و**التكلفة التقديرية** (عند استخدام مفتاح API فقط).
- `/usage off|tokens|full` ← يُلحق **تذييل استخدام لكل استجابة** بكل رد.
  - يُحفَظ لكل جلسة (ويُخزَّن باعتباره `responseUsage`).
  - تخفي مصادقة OAuth **التكلفة** (الرموز فقط).
- `/usage cost` ← يعرض ملخص تكلفة محليًا من سجلات جلسة OpenClaw.

أسطح أخرى:

- **TUI/Web TUI:** يتم دعم `/status` و`/usage`.
- **CLI:** يعرض `openclaw status --usage` و`openclaw channels list`
  نوافذ حصص المزوّد الموحّدة (`X% left`، وليس التكاليف لكل استجابة).
  مزوّدو نوافذ الاستخدام الحاليون: Anthropic وGitHub Copilot وGemini CLI
  وOpenAI Codex وMiniMax وXiaomi وz.ai.

تقوم أسطح الاستخدام بتوحيد الأسماء البديلة الشائعة الأصلية للمزوّد قبل العرض.
وبالنسبة لحركة Responses من عائلة OpenAI، يشمل ذلك كلاً من `input_tokens` /
`output_tokens` و`prompt_tokens` / `completion_tokens`، بحيث لا تؤدي
أسماء الحقول الخاصة بالنقل إلى تغيير `/status` أو `/usage` أو ملخصات الجلسات.
كما يتم أيضًا توحيد استخدام JSON الخاص بـ Gemini CLI: يأتي نص الرد من `response`،
ويُطابِق `stats.cached` إلى `cacheRead` مع استخدام `stats.input_tokens - stats.cached`
عندما تحذف CLI الحقل الصريح `stats.input`.
وبالنسبة لحركة Responses الأصلية من عائلة OpenAI، يتم توحيد الأسماء البديلة للاستخدام في WebSocket/SSE بالطريقة نفسها، وتعود المجاميع إلى الإدخال + الإخراج الموحّدين عندما يكون
`total_tokens` مفقودًا أو يساوي `0`.
وعندما تكون لقطة الجلسة الحالية متناثرة، يمكن أيضًا لـ `/status` و`session_status`
استعادة عدادات الرموز/الذاكرة المؤقتة ووسم نموذج Runtime النشط من أحدث سجل استخدام في التفريغ.
وتبقى القيم الحية غير الصفرية الموجودة مسبقًا ذات أولوية على قيم الرجوع إلى التفريغ، ويمكن أن تفوز المجاميع الأكبر المرتبطة بـ prompt في التفريغ عندما تكون المجاميع المخزنة مفقودة أو أصغر.
تأتي مصادقة الاستخدام الخاصة بنوافذ حصص المزوّد من الخطافات الخاصة بالمزوّد عند توفرها؛ وإلا يعود OpenClaw إلى مطابقة بيانات اعتماد OAuth/مفتاح API من ملفات تعريف المصادقة أو البيئة أو الإعدادات.

## تقدير التكلفة (عند العرض)

تُقدَّر التكاليف من إعدادات تسعير النموذج لديك:

```
models.providers.<provider>.models[].cost
```

وهذه القيم هي **بالدولار الأمريكي لكل مليون رمز** لكل من `input` و`output` و`cacheRead` و`cacheWrite`. وإذا كانت التسعيرة مفقودة، فسيعرض OpenClaw الرموز فقط. لا تعرض رموز OAuth تكلفة بالدولار مطلقًا.

## تأثير TTL الخاص بالذاكرة المؤقتة والتقليم

لا ينطبق التخزين المؤقت لـ prompt الخاص بالمزوّد إلا ضمن نافذة TTL الخاصة بالذاكرة المؤقتة. يمكن لـ OpenClaw اختياريًا تشغيل **تقليم cache-ttl**: حيث يقوم بتقليم الجلسة بمجرد انتهاء TTL الخاص بالذاكرة المؤقتة، ثم يعيد ضبط نافذة الذاكرة المؤقتة حتى تتمكن الطلبات اللاحقة من إعادة استخدام السياق المُخزَّن حديثًا بدلًا من إعادة تخزين المحفوظات الكاملة مؤقتًا. وهذا يُبقي تكاليف كتابة الذاكرة المؤقتة أقل عندما تبقى الجلسة خاملة بعد TTL.

قم بإعداده في [إعداد Gateway](/ar/gateway/configuration) وراجع تفاصيل السلوك في [تقليم الجلسة](/ar/concepts/session-pruning).

يمكن لـ Heartbeat إبقاء الذاكرة المؤقتة **دافئة** خلال فترات الخمول. إذا كان TTL لذاكرة النموذج المؤقتة لديك هو `1h`، فإن ضبط فاصل Heartbeat إلى ما دون ذلك بقليل (مثل `55m`) يمكن أن يتجنب إعادة تخزين prompt الكامل مؤقتًا، مما يقلل تكاليف كتابة الذاكرة المؤقتة.

في إعدادات الوكلاء المتعددين، يمكنك الاحتفاظ بإعداد نموذج مشترك واحد وضبط سلوك الذاكرة المؤقتة
لكل وكيل عبر `agents.list[].params.cacheRetention`.

للحصول على دليل كامل لكل مفتاح إعداد على حدة، راجع [Prompt Caching](/ar/reference/prompt-caching).

بالنسبة إلى تسعير Anthropic API، تكون قراءات الذاكرة المؤقتة أرخص بكثير من
رموز الإدخال، بينما تُفوتر كتابات الذاكرة المؤقتة بمعامل أعلى. راجع تسعير التخزين المؤقت لـ prompt الخاص بـ Anthropic لأحدث الأسعار ومعاملات TTL:
[https://docs.anthropic.com/docs/build-with-claude/prompt-caching](https://docs.anthropic.com/docs/build-with-claude/prompt-caching)

### مثال: إبقاء ذاكرة مؤقتة لمدة 1h دافئة باستخدام Heartbeat

```yaml
agents:
  defaults:
    model:
      primary: "anthropic/claude-opus-4-6"
    models:
      "anthropic/claude-opus-4-6":
        params:
          cacheRetention: "long"
    heartbeat:
      every: "55m"
```

### مثال: حركة مختلطة مع استراتيجية ذاكرة مؤقتة لكل وكيل

```yaml
agents:
  defaults:
    model:
      primary: "anthropic/claude-opus-4-6"
    models:
      "anthropic/claude-opus-4-6":
        params:
          cacheRetention: "long" # baseline الافتراضي لمعظم الوكلاء
  list:
    - id: "research"
      default: true
      heartbeat:
        every: "55m" # إبقاء الذاكرة المؤقتة الطويلة دافئة للجلسات العميقة
    - id: "alerts"
      params:
        cacheRetention: "none" # تجنب كتابات الذاكرة المؤقتة للإشعارات المتدفقة
```

يُدمَج `agents.list[].params` فوق `params` الخاص بالنموذج المحدد، بحيث يمكنك
تجاوز `cacheRetention` فقط مع إبقاء بقية إعدادات النموذج الافتراضية كما هي.

### مثال: تفعيل ترويسة Anthropic التجريبية لسياق 1M

تكون نافذة السياق 1M من Anthropic حاليًا خلف بوابة تجريبية. يمكن لـ OpenClaw حقن قيمة `anthropic-beta` المطلوبة عند تفعيل `context1m` على نماذج Opus
أو Sonnet المدعومة.

```yaml
agents:
  defaults:
    models:
      "anthropic/claude-opus-4-6":
        params:
          context1m: true
```

يُطابِق هذا ترويسة Anthropic التجريبية `context-1m-2025-08-07`.

ولا ينطبق هذا إلا عندما تكون `context1m: true` مضبوطة على إدخال ذلك النموذج.

المتطلب: يجب أن تكون بيانات الاعتماد مؤهلة لاستخدام السياق الطويل. وإذا لم تكن كذلك،
فإن Anthropic تستجيب بخطأ تحديد معدل من جهة المزوّد لذلك الطلب.

إذا كنت تصادق Anthropic باستخدام رموز OAuth/الاشتراك (`sk-ant-oat-*`)،
فإن OpenClaw يتخطى الترويسة التجريبية `context-1m-*` لأن Anthropic ترفض حاليًا
هذا الجمع باستخدام HTTP 401.

## نصائح لتقليل ضغط الرموز

- استخدم `/compact` لتلخيص الجلسات الطويلة.
- قلّص مخرجات الأدوات الكبيرة في تدفقات العمل لديك.
- اخفض `agents.defaults.imageMaxDimensionPx` للجلسات التي تعتمد بكثرة على لقطات الشاشة.
- اجعل أوصاف Skills قصيرة (يتم حقن قائمة Skills في prompt).
- فضّل النماذج الأصغر للأعمال الاستكشافية المطوّلة.

راجع [Skills](/ar/tools/skills) للاطلاع على الصيغة الدقيقة لحساب الحمل الإضافي لقائمة Skills.
