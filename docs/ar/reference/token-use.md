# استخدام الرموز والتكاليف

يتتبع OpenClaw **الرموز**، وليس الأحرف. والرموز خاصة بكل نموذج، لكن معظم
النماذج على نمط OpenAI تحتسب في المتوسط حوالي 4 أحرف لكل رمز في النص الإنجليزي.

## كيف يتم بناء system prompt

يقوم OpenClaw بتجميع system prompt الخاصة به في كل تشغيل. وهي تتضمن:

- قائمة الأدوات + أوصاف قصيرة
- قائمة Skills ‏(بيانات وصفية فقط؛ أما التعليمات فتُحمّل عند الطلب باستخدام `read`).
  وتكون كتلة Skills المضغوطة محدودة بواسطة `skills.limits.maxSkillsPromptChars`,
  مع تجاوز اختياري لكل وكيل في
  `agents.list[].skillsLimits.maxSkillsPromptChars`.
- تعليمات التحديث الذاتي
- ملفات مساحة العمل وbootstrap ‏(`AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md`, و`BOOTSTRAP.md` عندما يكون جديدًا، بالإضافة إلى `MEMORY.md` عند وجوده). لا يتم حقن `memory.md` الصغير في الجذر؛ إذ إنه مدخل إصلاح قديم لـ `openclaw doctor --fix` عندما يقترن مع `MEMORY.md`. وتُقتطع الملفات الكبيرة بواسطة `agents.defaults.bootstrapMaxChars` ‏(الافتراضي: 12000)، كما يُحد حقن bootstrap الكلي بواسطة `agents.defaults.bootstrapTotalMaxChars` ‏(الافتراضي: 60000). لا تُعد ملفات `memory/*.md` اليومية جزءًا من مطالبة bootstrap العادية؛ إذ تبقى عند الطلب عبر أدوات الذاكرة في الأدوار العادية، لكن `/new` و`/reset` المجردتين يمكنهما إلحاق كتلة سياق بدء لمرة واحدة تحتوي على ذاكرة يومية حديثة لذلك الدور الأول. ويتم التحكم في هذه المقدمة عبر `agents.defaults.startupContext`.
- الوقت ‏(UTC + المنطقة الزمنية للمستخدم)
- وسوم الرد + سلوك Heartbeat
- بيانات وقت التشغيل الوصفية ‏(المضيف/نظام التشغيل/النموذج/التفكير)

راجع التفصيل الكامل في [System Prompt](/ar/concepts/system-prompt).

## ما الذي يُحتسب ضمن نافذة السياق

كل ما يتلقاه النموذج يُحتسب ضمن حد السياق:

- System prompt ‏(كل الأقسام المدرجة أعلاه)
- سجل المحادثة ‏(رسائل المستخدم + المساعد)
- استدعاءات الأدوات ونتائج الأدوات
- المرفقات/النصوص المفرغة ‏(الصور، والصوت، والملفات)
- ملخصات Compaction وآثار الاقتطاع
- أغلفة المزوّد أو رؤوس الأمان ‏(غير مرئية، لكنها ما تزال محتسبة)

لبعض الأسطح الثقيلة وقت التشغيل حدود صريحة خاصة بها:

- `agents.defaults.contextLimits.memoryGetMaxChars`
- `agents.defaults.contextLimits.memoryGetDefaultLines`
- `agents.defaults.contextLimits.toolResultMaxChars`
- `agents.defaults.contextLimits.postCompactionMaxChars`

توجد التجاوزات لكل وكيل تحت `agents.list[].contextLimits`. هذه المفاتيح
مخصصة للمقتطفات المحدودة وقت التشغيل والكتل المحقونة المملوكة لوقت التشغيل. وهي
منفصلة عن حدود bootstrap، وحدود startup-context، وحدود مطالبة Skills.

بالنسبة إلى الصور، يقوم OpenClaw بتقليل أبعاد حمولات الصور في النص المفرغ/الأدوات قبل استدعاءات المزوّد.
استخدم `agents.defaults.imageMaxDimensionPx` ‏(الافتراضي: `1200`) لضبط ذلك:

- تؤدي القيم الأقل عادةً إلى تقليل استخدام رموز الرؤية وحجم الحمولة.
- تحافظ القيم الأعلى على مزيد من التفاصيل المرئية في لقطات الشاشة الثقيلة على OCR/واجهة المستخدم.

وللحصول على تفصيل عملي (لكل ملف محقون، والأدوات، وSkills، وحجم system prompt)، استخدم `/context list` أو `/context detail`. راجع [السياق](/ar/concepts/context).

## كيف ترى استخدام الرموز الحالي

استخدم هذه الأوامر في الدردشة:

- `/status` ← **بطاقة حالة غنية بـ emoji** تتضمن نموذج الجلسة، واستخدام السياق،
  ورموز الإدخال/الإخراج في آخر استجابة، و**التكلفة التقديرية** (لمفتاح API فقط).
- `/usage off|tokens|full` ← يضيف **تذييل استخدام لكل استجابة** إلى كل رد.
  - يستمر لكل جلسة (ويُخزَّن باعتباره `responseUsage`).
  - تقوم مصادقة OAuth **بإخفاء التكلفة** (الرموز فقط).
- `/usage cost` ← يعرض ملخص تكلفة محليًا من سجلات جلسات OpenClaw.

أسطح أخرى:

- **TUI/Web TUI:** الأمران `/status` و`/usage` مدعومان.
- **CLI:** يعرض `openclaw status --usage` و`openclaw channels list`
  نوافذ حصة المزوّد المطبّعة (`X% left`، وليست تكاليف لكل استجابة).
  مزوّدو نافذة الاستخدام الحاليون: Anthropic، وGitHub Copilot، وGemini CLI،
  وOpenAI Codex، وMiniMax، وXiaomi، وz.ai.

تقوم أسطح الاستخدام بتطبيع الأسماء البديلة الشائعة للحقول الأصلية في المزوّد قبل العرض.
وبالنسبة إلى حركة Responses الخاصة بعائلة OpenAI، يشمل ذلك كلًا من `input_tokens` /
`output_tokens` و`prompt_tokens` / `completion_tokens`، بحيث لا تغيّر
أسماء الحقول الخاصة بالنقل أوامر `/status` أو `/usage` أو ملخصات الجلسات.
ويتم تطبيع استخدام Gemini CLI ‏JSON أيضًا: يأتي نص الرد من `response`، و
تُعيَّن `stats.cached` إلى `cacheRead` مع استخدام `stats.input_tokens - stats.cached`
عندما يحذف CLI الحقل الصريح `stats.input`.
وبالنسبة إلى حركة Responses الأصلية لعائلة OpenAI، تُطبّع أيضًا الأسماء البديلة للاستخدام في WebSocket/SSE
بالطريقة نفسها، وتعود الإجماليات إلى الإدخال + الإخراج المطبّعين عندما تكون
`total_tokens` مفقودة أو تساوي `0`.
وعندما تكون لقطة الجلسة الحالية قليلة، يمكن أيضًا لـ `/status` و`session_status`
استعادة عدادات الرموز/التخزين المؤقت ووسم نموذج وقت التشغيل النشط من أحدث سجل استخدام
في النص المفرغ. وما تزال القيم الحية غير الصفرية الموجودة مسبقًا تحظى بالأولوية على قيم الرجوع من النص المفرغ، ويمكن أن تفوز إجماليات النص المفرغ الأكبر والموجهة نحو المطالبة عندما تكون الإجماليات المخزنة مفقودة أو أصغر.
تأتي مصادقة الاستخدام الخاصة بنوافذ حصة المزوّد من خطافات خاصة بالمزوّد
عند توفرها؛ وإلا يعود OpenClaw إلى مطابقة بيانات اعتماد OAuth/API-key
من ملفات تعريف المصادقة، أو البيئة، أو الإعدادات.
تحتفظ إدخالات النص المفرغ الخاصة بالمساعد بالشكل المطبّع نفسه للاستخدام، بما في ذلك
`usage.cost` عندما يكون لدى النموذج النشط تسعير مضبوط ويعيد المزوّد بيانات استخدام
وصفية. ويوفر هذا لكل من `/usage cost` وحالة الجلسة المدعومة بالنص المفرغ
مصدرًا ثابتًا حتى بعد اختفاء حالة وقت التشغيل الحية.

## تقدير التكلفة (عند عرضه)

تُقدَّر التكاليف من إعدادات تسعير النموذج لديك:

```
models.providers.<provider>.models[].cost
```

وهذه هي **دولارات أمريكية لكل 1M رمز** بالنسبة إلى `input`, و`output`, و`cacheRead`, و
`cacheWrite`. وإذا كان التسعير مفقودًا، فإن OpenClaw يعرض الرموز فقط. ولا تعرض رموز OAuth
أبدًا تكلفة بالدولار.

## أثر TTL الخاص بالتخزين المؤقت والاقتطاع

لا ينطبق التخزين المؤقت لمطالبة المزوّد إلا ضمن نافذة TTL الخاصة بالتخزين المؤقت. ويمكن لـ OpenClaw
اختياريًا تشغيل **اقتطاع cache-ttl**: إذ يقتطع الجلسة بمجرد انتهاء TTL الخاص بالتخزين المؤقت،
ثم يعيد تعيين نافذة التخزين المؤقت بحيث يمكن للطلبات اللاحقة إعادة استخدام
السياق المخزن حديثًا بدلًا من إعادة تخزين السجل الكامل مؤقتًا. ويحافظ هذا على انخفاض
تكاليف كتابة التخزين المؤقت عندما تدخل الجلسة في خمول بعد انتهاء TTL.

قم بإعداده في [إعدادات Gateway](/ar/gateway/configuration) وراجع
تفاصيل السلوك في [اقتطاع الجلسات](/ar/concepts/session-pruning).

يمكن لـ Heartbeat إبقاء التخزين المؤقت **دافئًا** عبر فجوات الخمول. فإذا كان TTL الخاص بالتخزين المؤقت في نموذجك
هو `1h`, فإن ضبط فترة Heartbeat أقل بقليل من ذلك (مثل `55m`) يمكن أن يتجنب
إعادة تخزين المطالبة الكاملة، مما يقلل تكاليف كتابة التخزين المؤقت.

في الإعدادات متعددة الوكلاء، يمكنك الاحتفاظ بإعداد نموذج مشترك واحد وضبط سلوك التخزين المؤقت
لكل وكيل عبر `agents.list[].params.cacheRetention`.

وللحصول على دليل كامل لكل مفتاح على حدة، راجع [Prompt Caching](/ar/reference/prompt-caching).

وبالنسبة إلى تسعير Anthropic API، تكون قراءات التخزين المؤقت أرخص بكثير من رموز الإدخال،
بينما تُفوتر كتابات التخزين المؤقت بمضاعف أعلى. راجع تسعير التخزين المؤقت للمطالبة لدى Anthropic للحصول على أحدث الأسعار ومضاعفات TTL:
[https://docs.anthropic.com/docs/build-with-claude/prompt-caching](https://docs.anthropic.com/docs/build-with-claude/prompt-caching)

### مثال: أبقِ التخزين المؤقت لمدة 1h دافئًا باستخدام Heartbeat

```yaml
agents:
  defaults:
    model:
      primary: "anthropic/claude-opus-4-6"
    models:
      "anthropic/claude-opus-4-6":
        params:
          cacheRetention: "long"
    heartbeat:
      every: "55m"
```

### مثال: حركة مرور مختلطة مع استراتيجية تخزين مؤقت لكل وكيل

```yaml
agents:
  defaults:
    model:
      primary: "anthropic/claude-opus-4-6"
    models:
      "anthropic/claude-opus-4-6":
        params:
          cacheRetention: "long" # خط أساس افتراضي لمعظم الوكلاء
  list:
    - id: "research"
      default: true
      heartbeat:
        every: "55m" # أبقِ التخزين المؤقت الطويل دافئًا للجلسات العميقة
    - id: "alerts"
      params:
        cacheRetention: "none" # تجنب كتابات التخزين المؤقت للإشعارات المتقطعة
```

يُدمج `agents.list[].params` فوق `params` الخاصة بالنموذج المحدد، بحيث يمكنك
تجاوز `cacheRetention` فقط ووراثة بقية افتراضيات النموذج دون تغيير.

### مثال: فعّل رأس Anthropic 1M context beta

تخضع نافذة السياق 1M الخاصة بـ Anthropic حاليًا لحالة beta. ويمكن لـ OpenClaw حقن
قيمة `anthropic-beta` المطلوبة عندما تفعّل `context1m` على نماذج Opus
أو Sonnet المدعومة.

```yaml
agents:
  defaults:
    models:
      "anthropic/claude-opus-4-6":
        params:
          context1m: true
```

يتم تعيين هذا إلى رأس Anthropic beta ‏`context-1m-2025-08-07`.

ولا ينطبق هذا إلا عندما تكون `context1m: true` مضبوطة على إدخال ذلك النموذج.

المتطلب: يجب أن تكون بيانات الاعتماد مؤهلة لاستخدام السياق الطويل. وإذا لم تكن كذلك،
فإن Anthropic تستجيب بخطأ حد معدل من جانب المزوّد لذلك الطلب.

إذا قمت بمصادقة Anthropic باستخدام رموز OAuth/الاشتراك (`sk-ant-oat-*`)،
فإن OpenClaw يتخطى رأس beta ‏`context-1m-*` لأن Anthropic ترفض حاليًا
هذا الجمع مع HTTP 401.

## نصائح لتقليل ضغط الرموز

- استخدم `/compact` لتلخيص الجلسات الطويلة.
- اقتطع المخرجات الكبيرة للأدوات في مسارات العمل الخاصة بك.
- خفّض `agents.defaults.imageMaxDimensionPx` في الجلسات الثقيلة على لقطات الشاشة.
- أبقِ أوصاف Skills قصيرة (إذ تُحقن قائمة Skills في المطالبة).
- فضّل النماذج الأصغر للأعمال الاستكشافية المطولة.

راجع [Skills](/ar/tools/skills) للحصول على الصيغة الدقيقة لعبء قائمة Skills.
