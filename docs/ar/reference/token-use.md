# استخدام الرموز المميزة والتكاليف

يتتبع OpenClaw **الرموز المميزة**، وليس الأحرف. تعتمد الرموز المميزة على النموذج، لكن معظم
النماذج ذات أسلوب OpenAI يبلغ متوسطها نحو 4 أحرف لكل رمز مميز في النص الإنجليزي.

## كيف تُبنى مطالبة النظام

يجمع OpenClaw مطالبة النظام الخاصة به في كل تشغيل. وهي تتضمن:

- قائمة الأدوات + أوصاف قصيرة
- قائمة Skills (البيانات الوصفية فقط؛ تُحمَّل التعليمات عند الطلب باستخدام `read`).
  تُقيَّد كتلة Skills المضغوطة بواسطة `skills.limits.maxSkillsPromptChars`،
  مع إمكانية تجاوز اختيارية لكل وكيل عند
  `agents.list[].skillsLimits.maxSkillsPromptChars`.
- تعليمات التحديث الذاتي
- ملفات مساحة العمل + التمهيد (`AGENTS.md`، و`SOUL.md`، و`TOOLS.md`، و`IDENTITY.md`، و`USER.md`، و`HEARTBEAT.md`، و`BOOTSTRAP.md` عند كونها جديدة، بالإضافة إلى `MEMORY.md` عند وجوده). لا يُحقن جذر `memory.md` المكتوب بأحرف صغيرة؛ فهو إدخال إصلاح قديم لـ `openclaw doctor --fix` عند إقرانه بـ `MEMORY.md`. تُقتطع الملفات الكبيرة بواسطة `agents.defaults.bootstrapMaxChars` (الافتراضي: 12000)، ويُحدّ إجمالي حقن التمهيد بواسطة `agents.defaults.bootstrapTotalMaxChars` (الافتراضي: 60000). ملفات `memory/*.md` اليومية ليست جزءًا من مطالبة التمهيد العادية؛ فهي تبقى متاحة عند الطلب عبر أدوات الذاكرة في الأدوار العادية، لكن تشغيلات النموذج عند إعادة الضبط/بدء التشغيل يمكن أن تسبقها كتلة سياق بدء تشغيل لمرة واحدة مع ذاكرة يومية حديثة لذلك الدور الأول. تُقرّ أوامر الدردشة الصرفة `/new` و`/reset` دون استدعاء النموذج. تتحكم `agents.defaults.startupContext` في تمهيد بدء التشغيل.
- الوقت (UTC + المنطقة الزمنية للمستخدم)
- وسوم الرد + سلوك Heartbeat
- بيانات تعريف وقت التشغيل (المضيف/نظام التشغيل/النموذج/التفكير)

راجع التفصيل الكامل في [مطالبة النظام](/ar/concepts/system-prompt).

## ما الذي يُحتسب في نافذة السياق

كل ما يتلقاه النموذج يُحتسب ضمن حد السياق:

- مطالبة النظام (كل الأقسام المذكورة أعلاه)
- سجل المحادثة (رسائل المستخدم + المساعد)
- استدعاءات الأدوات ونتائج الأدوات
- المرفقات/النصوص المنسوخة (الصور، الصوت، الملفات)
- ملخصات Compaction وآثار التقليم
- أغلفة المزوّد أو ترويسات الأمان (غير مرئية، لكنها ما تزال محتسبة)

لبعض الأسطح الثقيلة في وقت التشغيل حدود صريحة خاصة بها:

- `agents.defaults.contextLimits.memoryGetMaxChars`
- `agents.defaults.contextLimits.memoryGetDefaultLines`
- `agents.defaults.contextLimits.toolResultMaxChars`
- `agents.defaults.contextLimits.postCompactionMaxChars`

توجد التجاوزات لكل وكيل ضمن `agents.list[].contextLimits`. هذه المفاتيح مخصصة
للمقتطفات المحدودة في وقت التشغيل والكتل المحقونة المملوكة لوقت التشغيل. وهي
منفصلة عن حدود التمهيد، وحدود سياق بدء التشغيل، وحدود مطالبة Skills.

بالنسبة إلى الصور، يصغّر OpenClaw أحمال صور النصوص/الأدوات قبل استدعاءات المزوّد.
استخدم `agents.defaults.imageMaxDimensionPx` (الافتراضي: `1200`) لضبط ذلك:

- القيم الأقل تقلل عادةً استخدام رموز الرؤية المميزة وحجم الحمولة.
- القيم الأعلى تحفظ تفاصيل بصرية أكثر للقطات الشاشة الثقيلة في OCR/واجهة المستخدم.

للحصول على تفصيل عملي (لكل ملف محقون، والأدوات، وSkills، وحجم مطالبة النظام)، استخدم `/context list` أو `/context detail`. راجع [السياق](/ar/concepts/context).

## كيفية رؤية الاستخدام الحالي للرموز المميزة

استخدم هذه في الدردشة:

- `/status` → **بطاقة حالة غنية بالرموز التعبيرية** تعرض نموذج الجلسة، واستخدام السياق،
  ورموز الإدخال/الإخراج المميزة لآخر رد، و**التكلفة المقدّرة** (مفتاح API فقط).
- `/usage off|tokens|full` → يضيف **تذييل استخدام لكل رد** إلى كل رد.
  - يستمر لكل جلسة (مخزّن باسم `responseUsage`).
  - مصادقة OAuth **تخفي التكلفة** (الرموز المميزة فقط).
- `/usage cost` → يعرض ملخص تكلفة محليًا من سجلات جلسات OpenClaw.

أسطح أخرى:

- **TUI/واجهة TUI على الويب:** يدعمان `/status` + `/usage`.
- **CLI:** يعرض `openclaw status --usage` و`openclaw channels list`
  نوافذ حصص المزوّد الموحّدة (`X% left`، وليس تكاليف لكل رد).
  مزوّدو نافذة الاستخدام الحاليون: Anthropic، وGitHub Copilot، وGemini CLI،
  وOpenAI Codex، وMiniMax، وXiaomi، وz.ai.

تُوحّد أسطح الاستخدام الأسماء البديلة الشائعة للحقول الأصلية لدى المزوّد قبل العرض.
بالنسبة إلى حركة Responses من عائلة OpenAI، يشمل ذلك كلًا من `input_tokens` /
`output_tokens` و`prompt_tokens` / `completion_tokens`، لذا لا تغيّر أسماء الحقول
الخاصة بالنقل `/status` أو `/usage` أو ملخصات الجلسات.
يُوحّد استخدام JSON في Gemini CLI أيضًا: يأتي نص الرد من `response`، ويُعيَّن
`stats.cached` إلى `cacheRead` مع استخدام `stats.input_tokens - stats.cached`
عندما يحذف CLI حقل `stats.input` الصريح.
بالنسبة إلى حركة Responses الأصلية من عائلة OpenAI، تُوحّد أسماء استخدام WebSocket/SSE البديلة
بالطريقة نفسها، وتعود المجاميع إلى الإدخال + الإخراج الموحّدين عندما يكون
`total_tokens` مفقودًا أو `0`.
عندما تكون لقطة الجلسة الحالية قليلة البيانات، يمكن لـ `/status` و`session_status`
أيضًا استرداد عدادات الرموز/التخزين المؤقت وتسمية نموذج وقت التشغيل النشط من
أحدث سجل استخدام للنص المنسوخ. تظل القيم الحية غير الصفرية الحالية لها
الأسبقية على قيم الرجوع من النص المنسوخ، ويمكن للمجاميع الأكبر الموجهة إلى المطالبة
في النص المنسوخ أن تفوز عندما تكون المجاميع المخزنة مفقودة أو أصغر.
تأتي مصادقة الاستخدام لنوافذ حصص المزوّد من خطافات خاصة بالمزوّد عند
توفرها؛ وإلا يرجع OpenClaw إلى مطابقة بيانات اعتماد OAuth/API-key
من ملفات تعريف المصادقة أو البيئة أو الإعدادات.
تحتفظ إدخالات نصوص المساعد بالشكل الموحّد نفسه للاستخدام، بما في ذلك
`usage.cost` عندما تكون للنموذج النشط أسعار مكوّنة ويعيد المزوّد
بيانات تعريف الاستخدام. يمنح هذا `/usage cost` وحالة الجلسة المدعومة بالنصوص
مصدرًا مستقرًا حتى بعد زوال حالة وقت التشغيل الحية.

يبقي OpenClaw محاسبة استخدام المزوّد منفصلة عن لقطة السياق الحالية.
يمكن أن يتضمن `usage.total` الخاص بالمزوّد إدخالًا مخزنًا مؤقتًا، وإخراجًا، واستدعاءات
نموذج متعددة في حلقة أدوات، لذلك فهو مفيد للتكلفة والقياسات، لكنه قد يبالغ
في تقدير نافذة السياق الحية. تستخدم عروض السياق والتشخيصات أحدث لقطة مطالبة
(`promptTokens`، أو آخر استدعاء نموذج عند عدم توفر لقطة مطالبة)
لـ `context.used`.

## تقدير التكلفة (عند عرضها)

تُقدَّر التكاليف من إعدادات تسعير نموذجك:

```
models.providers.<provider>.models[].cost
```

هذه **دولارات أمريكية لكل مليون رمز مميز** لـ `input` و`output` و`cacheRead` و
`cacheWrite`. إذا كان التسعير مفقودًا، يعرض OpenClaw الرموز المميزة فقط. رموز OAuth
لا تعرض تكلفة بالدولار أبدًا.

بعد أن تصل السيارات الجانبية والقنوات إلى مسار جاهزية Gateway، يبدأ OpenClaw
تمهيد تسعير اختياريًا في الخلفية لمراجع النماذج المكوّنة التي لا
تملك تسعيرًا محليًا بالفعل. يجلب ذلك التمهيد كتالوجات تسعير OpenRouter وLiteLLM
البعيدة. اضبط `models.pricing.enabled: false` لتخطي جلب تلك الكتالوجات
على الشبكات غير المتصلة أو المقيّدة؛ تستمر إدخالات
`models.providers.*.models[].cost` الصريحة في قيادة تقديرات التكلفة المحلية.

## مدة صلاحية التخزين المؤقت وتأثير التقليم

لا ينطبق التخزين المؤقت للمطالبة لدى المزوّد إلا ضمن نافذة مدة صلاحية التخزين المؤقت. يمكن لـ OpenClaw
اختياريًا تشغيل **تقليم مدة صلاحية التخزين المؤقت**: فهو يقلّم الجلسة بمجرد انتهاء مدة صلاحية التخزين المؤقت،
ثم يعيد ضبط نافذة التخزين المؤقت بحيث يمكن للطلبات اللاحقة إعادة استخدام
السياق المخزن مؤقتًا حديثًا بدلًا من إعادة تخزين السجل الكامل مؤقتًا. يبقي هذا
تكاليف كتابة التخزين المؤقت أقل عندما تصبح الجلسة خاملة بعد مدة الصلاحية.

كوّنه في [إعدادات Gateway](/ar/gateway/configuration) وراجع
تفاصيل السلوك في [تقليم الجلسة](/ar/concepts/session-pruning).

يمكن أن يبقي Heartbeat التخزين المؤقت **دافئًا** عبر فجوات الخمول. إذا كانت مدة صلاحية التخزين المؤقت لنموذجك
`1h`، فإن ضبط فاصل Heartbeat ليكون أقل قليلًا من ذلك (مثل `55m`) يمكن أن يتجنب
إعادة تخزين المطالبة الكاملة مؤقتًا، مما يقلل تكاليف كتابة التخزين المؤقت.

في إعدادات متعددة الوكلاء، يمكنك إبقاء إعداد نموذج مشترك واحد وضبط سلوك التخزين المؤقت
لكل وكيل باستخدام `agents.list[].params.cacheRetention`.

للحصول على دليل كامل لكل مفتاح على حدة، راجع [تخزين المطالبات مؤقتًا](/ar/reference/prompt-caching).

بالنسبة إلى تسعير Anthropic API، تكون قراءات التخزين المؤقت أرخص بكثير من رموز الإدخال
المميزة، بينما تُفوتر كتابات التخزين المؤقت بمضاعف أعلى. راجع تسعير تخزين المطالبات مؤقتًا لدى Anthropic
لأحدث الأسعار ومضاعفات مدة الصلاحية:
[https://docs.anthropic.com/docs/build-with-claude/prompt-caching](https://docs.anthropic.com/docs/build-with-claude/prompt-caching)

### مثال: إبقاء تخزين مؤقت لمدة ساعة واحدة دافئًا باستخدام Heartbeat

```yaml
agents:
  defaults:
    model:
      primary: "anthropic/claude-opus-4-6"
    models:
      "anthropic/claude-opus-4-6":
        params:
          cacheRetention: "long"
    heartbeat:
      every: "55m"
```

### مثال: حركة مختلطة مع استراتيجية تخزين مؤقت لكل وكيل

```yaml
agents:
  defaults:
    model:
      primary: "anthropic/claude-opus-4-6"
    models:
      "anthropic/claude-opus-4-6":
        params:
          cacheRetention: "long" # default baseline for most agents
  list:
    - id: "research"
      default: true
      heartbeat:
        every: "55m" # keep long cache warm for deep sessions
    - id: "alerts"
      params:
        cacheRetention: "none" # avoid cache writes for bursty notifications
```

يدمج `agents.list[].params` فوق `params` الخاصة بالنموذج المحدد، لذا يمكنك
تجاوز `cacheRetention` فقط ووراثة افتراضات النموذج الأخرى دون تغيير.

### مثال: تفعيل ترويسة الإصدار التجريبي لسياق Anthropic 1M

نافذة سياق Anthropic بحجم 1M مقيّدة حاليًا بإصدار تجريبي. يستطيع OpenClaw حقن
قيمة `anthropic-beta` المطلوبة عندما تفعّل `context1m` على نماذج Opus
أو Sonnet المدعومة.

```yaml
agents:
  defaults:
    models:
      "anthropic/claude-opus-4-6":
        params:
          context1m: true
```

يُعيَّن هذا إلى ترويسة الإصدار التجريبي `context-1m-2025-08-07` لدى Anthropic.

ينطبق هذا فقط عندما يُضبط `context1m: true` على إدخال ذلك النموذج.

المتطلب: يجب أن تكون بيانات الاعتماد مؤهلة لاستخدام السياق الطويل. إذا لم تكن كذلك،
يرد Anthropic بخطأ حد معدل من جانب المزوّد لذلك الطلب.

إذا صادقت Anthropic باستخدام رموز OAuth/الاشتراك (`sk-ant-oat-*`)،
يتخطى OpenClaw ترويسة الإصدار التجريبي `context-1m-*` لأن Anthropic يرفض حاليًا
ذلك المزيج مع HTTP 401.

## نصائح لتقليل ضغط الرموز المميزة

- استخدم `/compact` لتلخيص الجلسات الطويلة.
- قلّم مخرجات الأدوات الكبيرة في سير عملك.
- خفّض `agents.defaults.imageMaxDimensionPx` للجلسات الثقيلة في لقطات الشاشة.
- اجعل أوصاف Skills قصيرة (تُحقن قائمة Skills في المطالبة).
- فضّل النماذج الأصغر للعمل الاستكشافي المطوّل.

راجع [Skills](/ar/tools/skills) لمعرفة صيغة العبء الدقيقة لقائمة Skills.
