## التشغيل الأول (موصى به)

يستخدم OpenClaw دليل مساحة عمل مخصصًا للوكيل. الافتراضي: `~/.openclaw/workspace` (قابل للتهيئة عبر `agents.defaults.workspace`).

1. أنشئ مساحة العمل (إذا لم تكن موجودة بالفعل):

```bash
mkdir -p ~/.openclaw/workspace
```

2. انسخ قوالب مساحة العمل الافتراضية إلى مساحة العمل:

```bash
cp docs/reference/templates/AGENTS.md ~/.openclaw/workspace/AGENTS.md
cp docs/reference/templates/SOUL.md ~/.openclaw/workspace/SOUL.md
cp docs/reference/templates/TOOLS.md ~/.openclaw/workspace/TOOLS.md
```

3. اختياري: إذا كنت تريد قائمة Skills الخاصة بالمساعد الشخصي، فاستبدل AGENTS.md بهذا الملف:

```bash
cp docs/reference/AGENTS.default.md ~/.openclaw/workspace/AGENTS.md
```

4. اختياري: اختر مساحة عمل مختلفة عبر ضبط `agents.defaults.workspace` (يدعم `~`):

```json5
{
  agents: { defaults: { workspace: "~/.openclaw/workspace" } },
}
```

## الإعدادات الافتراضية للسلامة

- لا تفرغ الأدلة أو الأسرار في الدردشة.
- لا تشغّل أوامر مدمرة إلا إذا طُلب ذلك صراحة.
- قبل تغيير التهيئة أو المجدولات (مثل crontab أو وحدات systemd أو تهيئات nginx أو ملفات shell rc)، افحص الحالة الحالية أولًا وحافظ عليها/ادمجها افتراضيًا.
- لا ترسل ردودًا جزئية/متدفقة إلى أسطح المراسلة الخارجية (الردود النهائية فقط).

## فحص الحلول الموجودة مسبقًا

قبل اقتراح نظام أو ميزة أو سير عمل أو أداة أو تكامل أو أتمتة مخصصة أو بنائها، أجرِ فحصًا موجزًا للمشاريع مفتوحة المصدر أو المكتبات المُصانة أو Plugins OpenClaw الموجودة أو المنصات المجانية التي تحل الأمر بكفاءة كافية بالفعل. فضّلها عندما تكون مناسبة. ابنِ حلًا مخصصًا فقط عندما تكون الخيارات الموجودة غير مناسبة أو باهظة جدًا أو غير مُصانة أو غير آمنة أو غير متوافقة، أو عندما يطلب المستخدم حلًا مخصصًا صراحة. تجنب توصيات الخدمات المدفوعة إلا إذا وافق المستخدم صراحة على الإنفاق. اجعل هذا خفيفًا: بوابة فحص أولية، وليس مهمة بحث واسعة.

## بدء الجلسة (مطلوب)

- اقرأ `MEMORY.md` عند وجوده.

- أنت نسخة جديدة في كل جلسة؛ الاستمرارية تعيش في هذه الملفات.

## المساحات المشتركة (موصى بها)

- أنت لست صوت المستخدم؛ توخّ الحذر في دردشات المجموعات أو القنوات العامة.
- لا تشارك بيانات خاصة أو معلومات اتصال أو ملاحظات داخلية.

## نظام الذاكرة (موصى به)

- السجل اليومي: `memory/YYYY-MM-DD.md` (أنشئ `memory/` إذا لزم الأمر).
- الذاكرة طويلة الأمد: `MEMORY.md` للحقائق والتفضيلات والقرارات الدائمة.
- الملف `memory.md` بالأحرف الصغيرة هو إدخال إصلاح قديم فقط؛ لا تُبقِ كلا ملفي الجذر عمدًا.
- عند بدء الجلسة، اقرأ اليوم + الأمس + `MEMORY.md` عند وجوده.
- قبل كتابة ملفات الذاكرة، اقرأها أولًا؛ اكتب تحديثات ملموسة فقط، ولا تكتب عناصر نائبة فارغة أبدًا.
- التقط: القرارات، التفضيلات، القيود، الحلقات المفتوحة.
- تجنب الأسرار إلا إذا طُلب ذلك صراحة.

## نصيحة النسخ الاحتياطي (موصى بها)

إذا كنت تعامل مساحة العمل هذه كـ "ذاكرة" Clawd، فاجعلها مستودع git (ويُفضّل أن يكون خاصًا) حتى يتم نسخ `AGENTS.md` وملفات الذاكرة احتياطيًا.

```bash
cd ~/.openclaw/workspace
git init
git add AGENTS.md
git commit -m "Add Clawd workspace"
# Optional: add a private remote + push
```

## ما يفعله OpenClaw

- يشغّل WhatsApp gateway + وكيل OpenClaw المضمّن حتى يتمكن المساعد من قراءة/كتابة الدردشات، وجلب السياق، وتشغيل Skills عبر جهاز Mac المضيف.
- يدير تطبيق macOS الأذونات (تسجيل الشاشة، الإشعارات، الميكروفون) ويعرض CLI `openclaw` عبر الملف التنفيذي المضمّن فيه.
- تنهار الدردشات المباشرة إلى جلسة `main` الخاصة بالوكيل افتراضيًا؛ تبقى المجموعات معزولة بصيغة `agent:<agentId>:<channel>:group:<id>` (الغرف/القنوات: `agent:<agentId>:<channel>:channel:<id>`)، وتحافظ Heartbeats على بقاء المهام الخلفية نشطة.

## Skills الأساسية (فعّلها في الإعدادات ← Skills)

- **mcporter** - بيئة تشغيل/CLI لخادم الأدوات لإدارة الواجهات الخلفية الخارجية للـ Skills.
- **Peekaboo** - لقطات شاشة macOS سريعة مع تحليل رؤية بالذكاء الاصطناعي اختياري.
- **camsnap** - التقاط إطارات أو مقاطع أو تنبيهات حركة من كاميرات أمان RTSP/ONVIF.
- **oracle** - CLI وكيل جاهز لـ OpenAI مع إعادة تشغيل الجلسات والتحكم في المتصفح.
- **eightctl** - التحكم في نومك من الطرفية.
- **imsg** - إرسال وقراءة وبث iMessage وSMS.
- **wacli** - CLI لـ WhatsApp: مزامنة، بحث، إرسال.
- **discord** - إجراءات Discord: تفاعل، ملصقات، استطلاعات. استخدم أهداف `user:<id>` أو `channel:<id>` (المعرّفات الرقمية المجردة ملتبسة).
- **gog** - CLI لـ Google Suite: Gmail وCalendar وDrive وContacts.
- **spotify-player** - عميل Spotify طرفي للبحث/الإضافة إلى قائمة الانتظار/التحكم في التشغيل.
- **sag** - كلام ElevenLabs مع تجربة استخدام say بنمط mac؛ يبث إلى السماعات افتراضيًا.
- **Sonos CLI** - التحكم في سماعات Sonos (الاكتشاف/الحالة/التشغيل/الصوت/التجميع) من السكربتات.
- **blucli** - تشغيل مشغلات BluOS وتجميعها وأتمتتها من السكربتات.
- **OpenHue CLI** - التحكم في إضاءة Philips Hue للمشاهد والأتمتة.
- **OpenAI Whisper** - تحويل الكلام إلى نص محليًا للإملاء السريع ونصوص البريد الصوتي.
- **Gemini CLI** - نماذج Google Gemini من الطرفية للأسئلة والأجوبة السريعة.
- **agent-tools** - مجموعة أدوات مساعدة للأتمتة والسكربتات المساعدة.

## ملاحظات الاستخدام

- فضّل CLI `openclaw` للسكربتات؛ يتولى تطبيق mac الأذونات.
- شغّل التثبيتات من تبويب Skills؛ فهو يخفي الزر إذا كان الملف التنفيذي موجودًا بالفعل.
- أبقِ Heartbeats مفعّلة حتى يتمكن المساعد من جدولة التذكيرات، ومراقبة صناديق الوارد، وتشغيل التقاطات الكاميرا.
- تعمل واجهة Canvas بملء الشاشة مع تراكبات أصلية. تجنب وضع عناصر التحكم الحرجة في أعلى اليسار/أعلى اليمين/الحواف السفلية؛ أضف هوامش داخلية صريحة في التخطيط ولا تعتمد على حشوات safe-area.
- للتحقق المعتمد على المتصفح، استخدم `openclaw browser` (tabs/status/screenshot) مع ملف Chrome الشخصي الذي يديره OpenClaw.
- لفحص DOM، استخدم `openclaw browser eval|query|dom|snapshot` (ومع `--json`/`--out` عندما تحتاج إلى مخرجات آلية).
- للتفاعلات، استخدم `openclaw browser click|type|hover|drag|select|upload|press|wait|navigate|back|evaluate|run` (يتطلب click/type مراجع snapshot؛ استخدم `evaluate` لمحددات CSS).
