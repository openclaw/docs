## التشغيل الأول (موصى به)

تستخدم وكلاء OpenClaw دليل مساحة عمل. الافتراضي: `~/.openclaw/workspace` (قابل للتهيئة عبر `agents.defaults.workspace`، ويدعم `~`).

1. أنشئ مساحة العمل:

```bash
mkdir -p ~/.openclaw/workspace
```

2. انسخ قوالب مساحة العمل الافتراضية إليها:

```bash
cp docs/reference/templates/AGENTS.md ~/.openclaw/workspace/AGENTS.md
cp docs/reference/templates/SOUL.md ~/.openclaw/workspace/SOUL.md
cp docs/reference/templates/TOOLS.md ~/.openclaw/workspace/TOOLS.md
```

3. اختياري: استخدم قائمة Skills الخاصة بالمساعد الشخصي في هذا الملف بدلًا من القالب العام:

```bash
cp docs/reference/AGENTS.default.md ~/.openclaw/workspace/AGENTS.md
```

4. اختياري: حدّد مساحة عمل مختلفة:

```json5
{
  agents: { defaults: { workspace: "~/.openclaw/workspace" } },
}
```

## إعدادات الأمان الافتراضية

- لا تعرض محتويات الأدلة أو الأسرار في الدردشة.
- لا تشغّل أوامر مدمّرة ما لم يُطلب ذلك صراحةً.
- قبل تغيير التهيئة أو أدوات الجدولة (crontab، ووحدات systemd، وتهيئات nginx، وملفات rc الخاصة بالصدفة)، افحص الحالة الحالية أولًا وحافظ عليها أو ادمج التغييرات معها افتراضيًا.
- لا ترسل ردودًا جزئية أو متدفقة إلى واجهات المراسلة الخارجية (أرسل الردود النهائية فقط).

## الفحص المسبق للحلول الموجودة

قبل اقتراح أو بناء نظام مخصص، أو ميزة، أو سير عمل، أو أداة، أو تكامل، أو أتمتة، تحقّق من وجود مشاريع مفتوحة المصدر، أو مكتبات تتم صيانتها، أو plugins موجودة لـ OpenClaw، أو منصات مجانية تحلّ المشكلة على نحو كافٍ بالفعل. فضّلها عندما تكون ملائمة. لا تبنِ حلًا مخصصًا إلا عندما تكون الخيارات الموجودة غير مناسبة، أو باهظة التكلفة، أو غير مُصانة، أو غير آمنة، أو غير متوافقة، أو عندما يطلب المستخدم حلًا مخصصًا صراحةً. تجنّب التوصية بالخدمات المدفوعة ما لم يوافق المستخدم صراحةً على الإنفاق. اجعل هذا فحصًا خفيفًا وبوابةً مسبقة، لا مهمة بحثية.

## بدء الجلسة (مطلوب)

- اقرأ `SOUL.md` و`USER.md` وملفَي اليوم والأمس في `memory/` قبل الرد.
- اقرأ `MEMORY.md` عندما يكون موجودًا.

## الهوية (مطلوب)

- يحدّد `SOUL.md` الهوية والنبرة والحدود. حافظ على تحديثه.
- إذا غيّرت `SOUL.md`، فأخبر المستخدم.
- أنت نسخة جديدة في كل جلسة؛ ويُحفظ الاستمرار في هذه الملفات.

## المساحات المشتركة (موصى بها)

- أنت لا تتحدث باسم المستخدم؛ توخَّ الحذر في الدردشات الجماعية أو القنوات العامة.
- لا تشارك البيانات الخاصة أو معلومات الاتصال أو الملاحظات الداخلية.

## نظام الذاكرة (موصى به)

- السجل اليومي: `memory/YYYY-MM-DD.md` (أنشئ `memory/` عند الحاجة).
- الذاكرة طويلة الأمد: `MEMORY.md` للحقائق والتفضيلات والقرارات الدائمة.
- ملف `memory.md` ذي الأحرف الصغيرة هو مُدخل لإصلاح البيانات القديمة فقط؛ لا تحتفظ عمدًا بالملفَين في الجذر.
- عند بدء الجلسة، اقرأ ملفَي اليوم والأمس و`MEMORY.md` عندما يكون موجودًا.
- قبل الكتابة في ملفات الذاكرة، اقرأها أولًا؛ اكتب تحديثات ملموسة فقط، ولا تكتب عناصر نائبة فارغة مطلقًا.
- سجّل: القرارات، والتفضيلات، والقيود، والمهام غير المكتملة.
- تجنّب الأسرار ما لم تُطلب صراحةً.

## الأدوات وSkills

- توجد الأدوات داخل Skills؛ اتبع ملف `SKILL.md` الخاص بكل Skill عند الحاجة إليها.
- احتفظ بالملاحظات الخاصة بالبيئة في `TOOLS.md` (ملاحظات لـ Skills).

## نصيحة للنسخ الاحتياطي (موصى بها)

تعامل مع مساحة العمل هذه بوصفها ذاكرة المساعد: اجعلها مستودع git (ويُفضّل أن يكون خاصًا) حتى تُنسخ ملفات `AGENTS.md` والذاكرة احتياطيًا.

```bash
cd ~/.openclaw/workspace
git init
git add AGENTS.md
git commit -m "Add workspace"
# اختياري: أضف مستودعًا بعيدًا خاصًا ثم ادفع التغييرات
```

## ما الذي يفعله OpenClaw

- يشغّل Gateway لقنوات المراسلة (WhatsApp وTelegram وDiscord وSignal وiMessage وSlack وغيرها) بالإضافة إلى وكيل مضمّن، بحيث يستطيع المساعد قراءة الدردشات والكتابة فيها، وجلب السياق، وتشغيل Skills عبر الجهاز المضيف.
- يدير تطبيق macOS الأذونات (تسجيل الشاشة والإشعارات والميكروفون)، ويتيح CLI ‏`openclaw` عبر الملف التنفيذي المضمّن معه.
- تُدمج الدردشات المباشرة افتراضيًا في جلسة `main` الخاصة بالوكيل؛ بينما تحصل المجموعات والقنوات/الغرف على مفاتيح جلسات خاصة بها. راجع [توجيه القنوات](/ar/channels/channel-routing) لمعرفة صيغ المفاتيح الدقيقة. تُبقي Heartbeat المهام التي تعمل في الخلفية نشطة.

## Skills الأساسية (فعّلها في Settings → Skills)

هذه قائمة نموذجية لمساحة عمل مساعد شخصي؛ استبدلها بأي Skills تناسب إعدادك.

- **mcporter** - بيئة تشغيل/CLI لخادم الأدوات لإدارة الواجهات الخلفية الخارجية لـ Skills.
- **Peekaboo** - لقطات شاشة سريعة على macOS مع تحليل اختياري للرؤية بالذكاء الاصطناعي.
- **camsnap** - التقاط إطارات أو مقاطع أو تنبيهات حركة من كاميرات المراقبة RTSP/ONVIF.
- **oracle** - CLI لوكيل جاهز لـ OpenAI مع إعادة تشغيل الجلسات والتحكم في المتصفح.
- **eightctl** - التحكّم في نومك من الطرفية.
- **imsg** - إرسال iMessage وSMS وقراءتهما وبثهما.
- **wacli** - CLI لـ WhatsApp: المزامنة والبحث والإرسال.
- **discord** - إجراءات Discord: التفاعلات والملصقات والاستطلاعات. استخدم الوجهات `user:<id>` أو `channel:<id>` (المعرّفات الرقمية المجرّدة ملتبسة).
- **gog** - CLI لحزمة Google:‏ Gmail والتقويم وDrive وجهات الاتصال.
- **spotify-player** - عميل Spotify للطرفية للبحث والإضافة إلى قائمة الانتظار والتحكم في التشغيل.
- **sag** - تحويل النص إلى كلام من ElevenLabs بتجربة استخدام شبيهة بأمر say في Mac؛ ويبث إلى مكبرات الصوت افتراضيًا.
- **Sonos CLI** - التحكّم في مكبرات صوت Sonos (الاكتشاف/الحالة/التشغيل/مستوى الصوت/التجميع) من النصوص البرمجية.
- **blucli** - تشغيل مشغلات BluOS وتجميعها وأتمتتها من النصوص البرمجية.
- **OpenHue CLI** - التحكّم في إضاءة Philips Hue للمشاهد وعمليات الأتمتة.
- **OpenAI Whisper** - تحويل الكلام إلى نص محليًا للإملاء السريع ونسخ رسائل البريد الصوتي.
- **Gemini CLI** - نماذج Google Gemini من الطرفية للحصول على إجابات سريعة عن الأسئلة.
- **agent-tools** - مجموعة أدوات مساعدة لعمليات الأتمتة والنصوص البرمجية المساعدة.

## ملاحظات الاستخدام

- فضّل CLI ‏`openclaw` لكتابة النصوص البرمجية؛ يتولى تطبيق سطح المكتب معالجة الأذونات.
- نفّذ عمليات التثبيت من علامة التبويب Skills؛ يختفي زر التثبيت عندما يكون الملف التنفيذي المطلوب موجودًا بالفعل.
- أبقِ Heartbeat مفعّلة حتى يتمكن المساعد من جدولة التذكيرات، ومراقبة صناديق الوارد، وتشغيل التقاط الصور من الكاميرات.
- تعمل واجهة Canvas بملء الشاشة مع طبقات أصلية متراكبة. تجنّب وضع عناصر التحكم المهمة عند الحواف العلوية اليسرى أو العلوية اليمنى أو السفلية؛ وأضف هوامش تخطيط صريحة بدلًا من الاعتماد على حشوات المنطقة الآمنة.
- للتحقق المعتمد على المتصفح، استخدم CLI ‏`openclaw browser` (plugin ‏`browser` المضمّن) مع ملف تعريف Chrome/Brave/Edge/Chromium الذي يديره OpenClaw.
- الإدارة: `status`، و`doctor [--deep]`، و`start [--headless]`، و`stop`، و`tabs`، و`tab [new|select|close]`، و`open <url>`، و`focus <id>`، و`close <id>`.
- الفحص: `screenshot [--full-page|--ref|--labels]`، و`snapshot [--format ai|aria|--interactive|--efficient]`، و`console`، و`errors`، و`requests`، و`pdf`، و`responsebody`.
- التنفيذ: `navigate`، و`click <ref>`، و`type <ref> <text>`، و`press`، و`hover`، و`drag`، و`select`، و`upload`، و`download`، و`fill`، و`dialog`، و`wait`، و`evaluate --fn <js>`، و`highlight`. تتطلب الإجراءات قيمة `ref` من `snapshot` (لا تُقبل محددات CSS للإجراءات)؛ استخدم `evaluate` عندما تحتاج إلى استهداف بأسلوب `document.querySelector`.
- أضف `--json` إلى أي أمر فحص للحصول على مخرجات قابلة للقراءة آليًا.
