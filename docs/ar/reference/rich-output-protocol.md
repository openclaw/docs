يمكن أن يحمل خرج المساعد مجموعة صغيرة من توجيهات التسليم/العرض:

- حقول `mediaUrl` / `mediaUrls` المهيكلة لتسليم المرفقات
- `[[audio_as_voice]]` لتلميحات عرض الصوت
- `[[reply_to_current]]` / `[[reply_to:<id>]]` لبيانات الرد الوصفية
- `[embed ...]` للعرض الغني في واجهة Control UI

يجب أن تكون مرفقات الوسائط البعيدة عناوين URL عامة من نوع `https:`. يتم تجاهل عناوين `http:` العادية،
وloopback، وlink-local، وأسماء المضيف الخاصة والداخلية كتوجيهات مرفقات؛ وتظل جالبات الوسائط من جهة الخادم تفرض حواجز الشبكة الخاصة بها.

يمكن أن تستخدم مرفقات الوسائط المحلية مسارات مطلقة، أو مسارات نسبية إلى مساحة العمل، أو
مسارات نسبية إلى المنزل `~/`. وتظل تمر عبر سياسة قراءة الملفات الخاصة بالوكيل
وفحوصات نوع الوسائط قبل التسليم.

<Warning>
لا تُصدر أوامر نصية للمرفقات من الأدوات، أو plugins، أو كتل البث،
أو خرج المتصفح، أو إجراءات الرسائل. استخدم حقول الوسائط المهيكلة بدلاً من ذلك.

حمولة أداة رسائل صالحة:

```json
{ "message": "Here is your image.", "mediaUrl": "/workspace/image.png" }
```

قد يظل نص رد المساعد النهائي القديم مُطبّعاً للتوافق، لكنه
ليس بروتوكولاً عاماً للـ plugin/الأداة.
</Warning>

تبقى صيغة صور Markdown العادية نصاً افتراضياً. القنوات التي تعمد إلى
مطابقة ردود صور Markdown مع مرفقات وسائط تختار ذلك في
المحوّل الصادر الخاص بها؛ يفعل Telegram ذلك بحيث يمكن أن يتحول `![alt](url)` إلى رد وسائط.

هذه التوجيهات منفصلة. حقول الوسائط المهيكلة ووسوم الرد/الصوت هي
بيانات وصفية للتسليم؛ أما `[embed ...]` فهو مسار العرض الغني المخصص للويب فقط.

عند تفعيل بث الكتل، يجب حمل الوسائط على حقول الحمولة
المهيكلة. إذا أُرسل عنوان URL للوسائط نفسه في كتلة مبثوثة وتكرر في
حمولة المساعد النهائية، يسلّم OpenClaw المرفق مرة واحدة ويزيل
التكرار من الحمولة النهائية.

## `[embed ...]`

`[embed ...]` هي صيغة العرض الغني الوحيدة الموجهة للوكيل في Control UI.

مثال ذاتي الإغلاق:

```text
[embed ref="cv_123" title="Status" /]
```

القواعد:

- لم تعد `[view ...]` صالحة للخرج الجديد.
- تُعرض اختصارات التضمين القصيرة في مساحة رسالة المساعد فقط.
- لا تُعرض إلا التضمينات المدعومة بعنوان URL. استخدم `ref="..."` أو `url="..."`.
- لا تُعرض اختصارات تضمين HTML المضمّنة بصيغة الكتل.
- تزيل واجهة الويب الاختصار القصير من النص المرئي وتعرض التضمين ضمن السطر.
- الوسائط المهيكلة ليست اسماً مستعاراً للتضمين ولا ينبغي استخدامها لعرض التضمين الغني.

## شكل العرض المخزن

كتلة محتوى المساعد المُطبّعة/المخزنة هي عنصر `canvas` مهيكل:

```json
{
  "type": "canvas",
  "preview": {
    "kind": "canvas",
    "surface": "assistant_message",
    "render": "url",
    "viewId": "cv_123",
    "url": "/__openclaw__/canvas/documents/cv_123/index.html",
    "title": "Status",
    "preferredHeight": 320
  }
}
```

تستخدم الكتل الغنية المخزنة/المعروضة شكل `canvas` هذا مباشرةً. لا يتم التعرف على `present_view`.
