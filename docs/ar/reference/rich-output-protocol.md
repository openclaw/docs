يمكن أن تحمل مخرجات المساعد مجموعة صغيرة من توجيهات التسليم/العرض:

- `MEDIA:` لتسليم المرفقات
- `[[audio_as_voice]]` لتلميحات عرض الصوت
- `[[reply_to_current]]` / `[[reply_to:<id>]]` لبيانات الرد الوصفية
- `[embed ...]` للعرض الغني في Control UI

هذه التوجيهات منفصلة. تظل وسوم `MEDIA:` ووسوم الرد/الصوت بيانات وصفية للتسليم؛ أما `[embed ...]` فهو مسار العرض الغني المخصص للويب فقط.
تستخدم وسائط نتائج الأدوات الموثوقة المحلل نفسه لـ `MEDIA:` / `[[audio_as_voice]]` قبل التسليم، لذلك لا تزال مخرجات الأدوات النصية قادرة على تمييز مرفق صوتي كملاحظة صوتية.

عند تمكين البث الكتلي، تظل `MEDIA:` بيانات وصفية لتسليم واحد
لكل دور. إذا أُرسل عنوان URL للوسائط نفسه في كتلة متدفقة وتكرر في
حمولة المساعد النهائية، يسلّم OpenClaw المرفق مرة واحدة ويزيل التكرار
من الحمولة النهائية.

## `[embed ...]`

`[embed ...]` هو صيغة العرض الغني الوحيدة الموجّهة إلى الوكيل في Control UI.

مثال ذاتي الإغلاق:

```text
[embed ref="cv_123" title="Status" /]
```

القواعد:

- لم يعد `[view ...]` صالحًا للمخرجات الجديدة.
- تُعرض الرموز القصيرة للتضمين في سطح رسالة المساعد فقط.
- لا يتم عرض إلا التضمينات المدعومة بعناوين URL. استخدم `ref="..."` أو `url="..."`.
- لا يتم عرض الرموز القصيرة للتضمين بصيغة HTML المضمنة على شكل كتلة.
- تزيل واجهة الويب الرمز القصير من النص المرئي وتعرض التضمين بشكل مضمّن.
- لا يُعد `MEDIA:` اسمًا بديلًا لـ embed ولا ينبغي استخدامه للعرض الغني للتضمين.

## شكل العرض المخزَّن

كتلة محتوى المساعد المعيارية/المخزنة هي عنصر `canvas` منظّم:

```json
{
  "type": "canvas",
  "preview": {
    "kind": "canvas",
    "surface": "assistant_message",
    "render": "url",
    "viewId": "cv_123",
    "url": "/__openclaw__/canvas/documents/cv_123/index.html",
    "title": "Status",
    "preferredHeight": 320
  }
}
```

تستخدم الكتل الغنية المخزنة/المعروضة شكل `canvas` هذا مباشرةً. ولا يتم التعرف على `present_view`.
