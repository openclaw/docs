يحمل إخراج المساعد توجيهات التسليم/العرض عبر عدد قليل من القنوات المخصصة:

- حقلا `mediaUrl` / `mediaUrls` المنظمان لتسليم المرفقات.
- `[[audio_as_voice]]` لتلميحات تقديم الصوت.
- `[[reply_to_current]]` / `[[reply_to:<id>]]` للبيانات الوصفية للرد.
- `[embed ...]` للعرض الغني في واجهة التحكم.

حقول الوسائط المنظمة ووسوم `[[...]]` هي بيانات وصفية للتسليم. أما `[embed ...]` فهو مسار منفصل للعرض الغني خاص بالويب فقط؛ وليس اسمًا بديلًا للوسائط.

## مرفقات الوسائط

يجب أن تكون المرفقات البعيدة عناوين URL عامة تستخدم `https:`. تُرفض أسماء المضيفين التي تستخدم `http:` أو local loopback أو عناوين الارتباط المحلي أو الأسماء الخاصة أو الداخلية باعتبارها توجيهات للمرفقات؛ وتطبّق أدوات جلب الوسائط من جانب الخادم ضوابط الشبكة الخاصة بها أيضًا.

تقبل المرفقات المحلية المسارات المطلقة، أو المسارات النسبية إلى مساحة العمل، أو المسارات النسبية إلى المجلد الرئيسي التي تبدأ بـ `~/`. وتظل خاضعة لسياسة قراءة الملفات الخاصة بالوكيل وفحوصات نوع الوسائط قبل التسليم.

<Warning>
لا تُخرج أوامر نصية للمرفقات من الأدوات أو الإضافات أو كتل البث أو مخرجات المتصفح أو إجراءات الرسائل. استخدم حقول الوسائط المنظمة بدلًا من ذلك:

```json
{ "message": "Here is your image.", "mediaUrl": "/workspace/image.png" }
```

قد تستمر تسوية نص الرد النهائي القديم لأغراض التوافق، لكن هذا ليس بروتوكولًا عامًا للإضافات/الأدوات.
</Warning>

تظل صيغة صور Markdown العادية (`![alt](url)`) نصًا افتراضيًا. ويمكن للقنوات التي تريد معاملة صور Markdown كاستجابات وسائط تفعيل ذلك في محوّل الإرسال الخاص بها؛ ويفعل Telegram ذلك بحيث تصبح `![alt](url)` مرفق وسائط.

عند تمكين بث الكتل، يجب تمرير الوسائط عبر حقول الحمولة المنظمة. إذا ظهر عنوان URL نفسه للوسائط في كتلة مبثوثة ثم ظهر مجددًا في الحمولة النهائية للمساعد، فسيقوم OpenClaw بتسليمه مرة واحدة وإزالة النسخة المكررة من الحمولة النهائية.

## `[embed ...]`

`[embed ...]` هي صيغة العرض الغني الوحيدة المتاحة للوكيل في واجهة التحكم. مثال ذاتي الإغلاق:

```text
[embed ref="cv_123" title="Status" /]
```

القواعد:

- لم تعد `[view ...]` صالحة للمخرجات الجديدة.
- لا تُعرض الأكواد المختصرة للتضمين إلا في سطح رسالة المساعد.
- لا تُعرض إلا التضمينات المدعومة بعنوان URL؛ استخدم `ref="..."` أو `url="..."`.
- لا تُعرض أكواد تضمين HTML المضمنة ذات صيغة الكتلة.
- تزيل واجهة الويب الكود المختصر من النص المرئي وتعرض التضمين ضمن السطر.

## بنية العرض المخزنة

كتلة محتوى المساعد بعد التسوية/التخزين هي عنصر `canvas` منظم:

```json
{
  "type": "canvas",
  "preview": {
    "kind": "canvas",
    "surface": "assistant_message",
    "render": "url",
    "viewId": "cv_123",
    "url": "/__openclaw__/canvas/documents/cv_123/index.html",
    "title": "Status",
    "preferredHeight": 320
  }
}
```

لا يتم التعرف على `present_view`؛ إذ تستخدم الكتل الغنية المخزنة/المعروضة دائمًا بنية `canvas` هذه.
