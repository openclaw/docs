تسحب OpenClaw متغيرات البيئة من مصادر متعددة. القاعدة هي **عدم تجاوز القيم الموجودة مطلقًا**.

## الأسبقية (الأعلى → الأدنى)

1. **بيئة العملية** (ما تكون عملية Gateway قد ورثته بالفعل من الصدفة/الخدمة الأم).
2. **`.env` في دليل العمل الحالي** (افتراضي dotenv؛ لا يتجاوز).
3. **`.env` العام** في `~/.openclaw/.env` (ويُعرف أيضًا باسم `$OPENCLAW_STATE_DIR/.env`؛ لا يتجاوز).
4. **كتلة `env` في الإعدادات** داخل `~/.openclaw/openclaw.json` (تُطبَّق فقط إذا كانت مفقودة).
5. **استيراد اختياري من صدفة تسجيل الدخول** (`env.shellEnv.enabled` أو `OPENCLAW_LOAD_SHELL_ENV=1`)، ويُطبَّق فقط للمفاتيح المتوقعة المفقودة.

في تثبيتات Ubuntu الجديدة التي تستخدم دليل الحالة الافتراضي، تتعامل OpenClaw أيضًا مع `~/.config/openclaw/gateway.env` كمسار احتياطي للتوافق بعد `.env` العام. إذا كان الملفان موجودين ويختلفان، تُبقي OpenClaw على `~/.openclaw/.env` وتطبع تحذيرًا.

إذا كان ملف الإعدادات مفقودًا بالكامل، تُتخطى الخطوة 4؛ ولا يزال استيراد الصدفة يعمل إذا كان مفعّلًا.

## كتلة `env` في الإعدادات

طريقتان متكافئتان لتعيين متغيرات البيئة المضمنة (كلتاهما لا تتجاوزان القيم الموجودة):

```json5
{
  env: {
    OPENROUTER_API_KEY: "sk-or-...",
    vars: {
      GROQ_API_KEY: "gsk-...",
    },
  },
}
```

## استيراد بيئة الصدفة

يشغّل `env.shellEnv` صدفة تسجيل الدخول لديك ويستورد فقط المفاتيح المتوقعة **المفقودة**:

```json5
{
  env: {
    shellEnv: {
      enabled: true,
      timeoutMs: 15000,
    },
  },
}
```

مكافئات متغيرات البيئة:

- `OPENCLAW_LOAD_SHELL_ENV=1`
- `OPENCLAW_SHELL_ENV_TIMEOUT_MS=15000`

## متغيرات البيئة المحقونة وقت التشغيل

تحقن OpenClaw أيضًا علامات السياق في العمليات الفرعية المُنشأة:

- `OPENCLAW_SHELL=exec`: يُعيَّن للأوامر التي تُشغَّل عبر أداة `exec`.
- `OPENCLAW_SHELL=acp`: يُعيَّن عند إنشاء عمليات خلفية تشغيل ACP (على سبيل المثال `acpx`).
- `OPENCLAW_SHELL=acp-client`: يُعيَّن لـ `openclaw acp client` عندما ينشئ عملية جسر ACP.
- `OPENCLAW_SHELL=tui-local`: يُعيَّن لأوامر صدفة TUI المحلية `!`.

هذه علامات وقت تشغيل (وليست إعدادات مستخدم مطلوبة). يمكن استخدامها في منطق الصدفة/الملف الشخصي
لتطبيق قواعد خاصة بالسياق.

## متغيرات بيئة الواجهة

- `OPENCLAW_THEME=light`: يفرض لوحة TUI الفاتحة عندما تكون خلفية الطرفية فاتحة.
- `OPENCLAW_THEME=dark`: يفرض لوحة TUI الداكنة.
- `COLORFGBG`: إذا صدّرته الطرفية لديك، تستخدم OpenClaw تلميح لون الخلفية لاختيار لوحة TUI تلقائيًا.

## استبدال متغيرات البيئة في الإعدادات

يمكنك الإشارة إلى متغيرات البيئة مباشرةً في قيم السلاسل النصية داخل الإعدادات باستخدام صيغة `${VAR_NAME}`:

```json5
{
  models: {
    providers: {
      "vercel-gateway": {
        apiKey: "${VERCEL_GATEWAY_API_KEY}",
      },
    },
  },
}
```

راجع [الإعدادات: استبدال متغيرات البيئة](/ar/gateway/configuration-reference#env-var-substitution) للتفاصيل الكاملة.

## مراجع الأسرار مقابل سلاسل `${ENV}`

تدعم OpenClaw نمطين مدفوعين بمتغيرات البيئة:

- استبدال السلاسل النصية `${VAR}` في قيم الإعدادات.
- كائنات SecretRef (`{ source: "env", provider: "default", id: "VAR" }`) للحقول التي تدعم مراجع الأسرار.

كلاهما يُحلّ من بيئة العملية عند وقت التفعيل. وُثّقت تفاصيل SecretRef في [إدارة الأسرار](/ar/gateway/secrets).

## متغيرات البيئة المتعلقة بالمسارات

| المتغير               | الغرض                                                                                                                                                                          |
| ---------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `OPENCLAW_HOME`        | يتجاوز دليل المنزل المستخدم لكل عمليات حل المسارات الداخلية (`~/.openclaw/`، أدلة الوكلاء، الجلسات، بيانات الاعتماد). مفيد عند تشغيل OpenClaw كمستخدم خدمة مخصص. |
| `OPENCLAW_STATE_DIR`   | يتجاوز دليل الحالة (الافتراضي `~/.openclaw`).                                                                                                                            |
| `OPENCLAW_CONFIG_PATH` | يتجاوز مسار ملف الإعدادات (الافتراضي `~/.openclaw/openclaw.json`).                                                                                                             |

## التسجيل

| المتغير             | الغرض                                                                                                                                                                                      |
| -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `OPENCLAW_LOG_LEVEL` | يتجاوز مستوى السجل لكل من الملف ووحدة التحكم (مثل `debug` و`trace`). تكون له الأسبقية على `logging.level` و`logging.consoleLevel` في الإعدادات. تُتجاهل القيم غير الصالحة مع تحذير. |

### `OPENCLAW_HOME`

عند تعيينه، يستبدل `OPENCLAW_HOME` دليل المنزل في النظام (`$HOME` / `os.homedir()`) لكل عمليات حل المسارات الداخلية. يتيح ذلك عزلًا كاملًا لنظام الملفات لحسابات الخدمة عديمة الواجهة.

**الأسبقية:** `OPENCLAW_HOME` > `$HOME` > `USERPROFILE` > `os.homedir()`

**مثال** (macOS LaunchDaemon):

```xml
<key>EnvironmentVariables</key>
<dict>
  <key>OPENCLAW_HOME</key>
  <string>/Users/user</string>
</dict>
```

يمكن أيضًا تعيين `OPENCLAW_HOME` إلى مسار يبدأ بالتلدة (مثل `~/svc`)، والذي يُوسَّع باستخدام `$HOME` قبل الاستخدام.

## مستخدمو nvm: حالات فشل TLS في web_fetch

إذا ثُبّت Node.js عبر **nvm** (وليس عبر مدير حزم النظام)، فإن `fetch()` المدمجة تستخدم
مخزن شهادات CA المضمّن في nvm، والذي قد يفتقد مراجع CA الجذرية الحديثة (ISRG Root X1/X2 الخاصة بـ Let's Encrypt،
وDigiCert Global Root G2، وغيرها). يؤدي ذلك إلى فشل `web_fetch` برسالة `"fetch failed"` في معظم مواقع HTTPS.

على Linux، تكتشف OpenClaw تلقائيًا nvm وتطبّق الإصلاح في بيئة بدء التشغيل الفعلية:

- يكتب `openclaw gateway install` المتغير `NODE_EXTRA_CA_CERTS` في بيئة خدمة systemd
- تعيد نقطة دخول CLI الخاصة بـ `openclaw` تنفيذ نفسها مع تعيين `NODE_EXTRA_CA_CERTS` قبل بدء Node

**الإصلاح اليدوي (للإصدارات الأقدم أو عمليات التشغيل المباشرة بـ `node ...`):**

صدّر المتغير قبل بدء OpenClaw:

```bash
export NODE_EXTRA_CA_CERTS=/etc/ssl/certs/ca-certificates.crt
openclaw gateway run
```

لا تعتمد على الكتابة إلى `~/.openclaw/.env` فقط لهذا المتغير؛ إذ يقرأ Node
`NODE_EXTRA_CA_CERTS` عند بدء العملية.

## متغيرات البيئة القديمة

تقرأ OpenClaw فقط متغيرات البيئة `OPENCLAW_*`. أما البادئات القديمة
`CLAWDBOT_*` و`MOLTBOT_*` من الإصدارات السابقة فتُتجاهل بصمت.

إذا كان أي منها لا يزال معيّنًا على عملية Gateway عند بدء التشغيل، تصدر OpenClaw
تحذير إهمال واحدًا من Node (`OPENCLAW_LEGACY_ENV_VARS`) يسرد
البادئات المكتشفة والعدد الإجمالي. أعد تسمية كل قيمة باستبدال
البادئة القديمة بـ `OPENCLAW_` (على سبيل المثال `CLAWDBOT_GATEWAY_TOKEN` →
`OPENCLAW_GATEWAY_TOKEN`)؛ ولا يكون للأسماء القديمة أي تأثير.
