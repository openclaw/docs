# المساهمة في نموذج التهديدات الخاص بـ OpenClaw

شكرًا لمساعدتك في جعل OpenClaw أكثر أمانًا. نموذج التهديدات هذا وثيقة حية ونرحب بالمساهمات من أي شخص — ولا تحتاج إلى أن تكون خبيرًا أمنيًا.

## طرق المساهمة

### أضف تهديدًا

هل لاحظت متجه هجوم أو خطرًا لم نغطيه؟ افتح issue على [openclaw/trust](https://github.com/openclaw/trust/issues) ووصفه بكلماتك الخاصة. لا تحتاج إلى معرفة أي أطر عمل أو ملء كل حقل — فقط صف السيناريو.

**من المفيد تضمين ما يلي (ولكنه غير مطلوب):**

- سيناريو الهجوم وكيف يمكن استغلاله
- ما الأجزاء المتأثرة من OpenClaw ‏(CLI، وgateway، والقنوات، وClawHub، وخوادم MCP، وغير ذلك)
- مدى الخطورة التي تعتقد أنها تمثلها ‏(منخفض / متوسط / مرتفع / حرج)
- أي روابط إلى أبحاث ذات صلة، أو CVEs، أو أمثلة من العالم الحقيقي

سنتولى نحن ربطها بـ ATLAS، وتعيين معرّفات التهديدات، وتقييم المخاطر أثناء المراجعة. وإذا أردت تضمين هذه التفاصيل، فهذا رائع — لكنه غير متوقع.

> **هذا مخصص للإضافة إلى نموذج التهديدات، وليس للإبلاغ عن ثغرات حية.** إذا كنت قد وجدت ثغرة قابلة للاستغلال، فراجع [صفحة Trust](https://trust.openclaw.ai) الخاصة بنا للحصول على تعليمات الإفصاح المسؤول.

### اقترح إجراء تخفيف

هل لديك فكرة لمعالجة تهديد قائم؟ افتح issue أو PR يشير إلى التهديد. تكون إجراءات التخفيف المفيدة محددة وقابلة للتنفيذ — فعلى سبيل المثال، "تحديد معدل 10 رسائل/دقيقة لكل مرسل عند gateway" أفضل من "تنفيذ تحديد معدل."

### اقترح سلسلة هجوم

تُظهر سلاسل الهجوم كيف تتحد عدة تهديدات لتكوين سيناريو هجوم واقعي. وإذا رأيت تركيبة خطيرة، فصف الخطوات وكيف سيربط المهاجم بينها. ويكون السرد القصير لكيفية تطور الهجوم عمليًا أكثر قيمة من قالب رسمي.

### أصلح أو حسّن المحتوى الحالي

الأخطاء المطبعية، أو التوضيحات، أو المعلومات القديمة، أو الأمثلة الأفضل — نرحب بطلبات PR، ولا حاجة إلى issue.

## ما الذي نستخدمه

### MITRE ATLAS

يستند نموذج التهديدات هذا إلى [MITRE ATLAS](https://atlas.mitre.org/) ‏(مشهد التهديدات العدائية لأنظمة الذكاء الاصطناعي)، وهو إطار عمل مصمم خصيصًا لتهديدات الذكاء الاصطناعي/التعلم الآلي مثل حقن prompt، وإساءة استخدام الأدوات، واستغلال الوكلاء. ولا تحتاج إلى معرفة ATLAS للمساهمة — فنحن نربط المساهمات بالإطار أثناء المراجعة.

### معرّفات التهديدات

يحصل كل تهديد على معرّف مثل `T-EXEC-003`. والفئات هي:

| الرمز | الفئة |
| ------ | ------ |
| RECON | الاستطلاع - جمع المعلومات |
| ACCESS | الوصول الأولي - اكتساب المدخل |
| EXEC | التنفيذ - تشغيل إجراءات خبيثة |
| PERSIST | الاستمرارية - الحفاظ على الوصول |
| EVADE | التهرب الدفاعي - تجنب الاكتشاف |
| DISC | الاكتشاف - التعرف على البيئة |
| EXFIL | الاستخراج - سرقة البيانات |
| IMPACT | الأثر - الضرر أو التعطيل |

يتم تعيين المعرّفات بواسطة المشرفين أثناء المراجعة. ولا تحتاج إلى اختيار واحد.

### مستويات المخاطر

| المستوى | المعنى |
| ------- | ------ |
| **حرج** | اختراق كامل للنظام، أو احتمال مرتفع + أثر حرج |
| **مرتفع** | ضرر كبير محتمل، أو احتمال متوسط + أثر حرج |
| **متوسط** | خطر متوسط، أو احتمال منخفض + أثر مرتفع |
| **منخفض** | غير مرجح وأثره محدود |

إذا لم تكن متأكدًا من مستوى الخطر، فصف الأثر فقط وسنقوم نحن بتقييمه.

## عملية المراجعة

1. **الفرز** — نراجع المساهمات الجديدة خلال 48 ساعة
2. **التقييم** — نتحقق من القابلية للتنفيذ، ونعين ربط ATLAS ومعرّف التهديد، ونتحقق من مستوى الخطر
3. **التوثيق** — نتأكد من أن كل شيء منسق ومكتمل
4. **الدمج** — يُضاف إلى نموذج التهديدات وإلى العرض البصري
