# المساهمة في نموذج تهديدات OpenClaw

شكرًا لمساعدتك في جعل OpenClaw أكثر أمانًا. نموذج التهديدات هذا مستند حي، ونرحب بالمساهمات من أي شخص - لست بحاجة إلى أن تكون خبيرًا أمنيًا.

## طرق المساهمة

### إضافة تهديد

رصدت متجه هجوم أو خطرًا لم نغطه؟ افتح مشكلة في [openclaw/trust](https://github.com/openclaw/trust/issues) وصفه بكلماتك. لست بحاجة إلى معرفة أي أطر عمل أو ملء كل حقل - فقط صف السيناريو.

**من المفيد تضمين ما يلي (لكن ليس مطلوبًا):**

- سيناريو الهجوم وكيف يمكن استغلاله
- أي أجزاء من OpenClaw متأثرة (CLI، Gateway، القنوات، ClawHub، خوادم MCP، وما إلى ذلك)
- مدى شدته في رأيك (منخفض / متوسط / عالٍ / حرج)
- أي روابط لأبحاث ذات صلة أو CVEs أو أمثلة واقعية

سنتولى ربط ATLAS ومعرفات التهديدات وتقييم المخاطر أثناء المراجعة. إذا أردت تضمين هذه التفاصيل، فهذا رائع - لكنه غير متوقع.

> **هذا مخصص للإضافة إلى نموذج التهديدات، وليس للإبلاغ عن الثغرات النشطة.** إذا وجدت ثغرة قابلة للاستغلال، فراجع [صفحة الثقة](https://trust.openclaw.ai) الخاصة بنا للحصول على تعليمات الإفصاح المسؤول.

### اقتراح تخفيف

لديك فكرة حول كيفية معالجة تهديد قائم؟ افتح مشكلة أو PR يشير إلى التهديد. التخفيفات المفيدة محددة وقابلة للتنفيذ - على سبيل المثال، "تقييد المعدل لكل مرسل بـ 10 رسائل/دقيقة عند Gateway" أفضل من "تنفيذ تقييد المعدل."

### اقتراح سلسلة هجوم

توضح سلاسل الهجوم كيف تتحد تهديدات متعددة في سيناريو هجوم واقعي. إذا رأيت مجموعة خطيرة، فصف الخطوات وكيف يمكن للمهاجم ربطها معًا. السرد القصير لكيفية حدوث الهجوم عمليًا أكثر قيمة من قالب رسمي.

### إصلاح المحتوى الحالي أو تحسينه

الأخطاء المطبعية، التوضيحات، المعلومات القديمة، الأمثلة الأفضل - نرحب بـ PRs، ولا حاجة إلى مشكلة.

## ما نستخدمه

### MITRE ATLAS

بُني نموذج التهديدات هذا على [MITRE ATLAS](https://atlas.mitre.org/) (مشهد التهديدات العدائية لأنظمة الذكاء الاصطناعي)، وهو إطار عمل مصمم خصيصًا لتهديدات AI/ML مثل حقن الموجهات، وإساءة استخدام الأدوات، واستغلال الوكلاء. لست بحاجة إلى معرفة ATLAS للمساهمة - فنحن نربط الإرسالات بالإطار أثناء المراجعة.

### معرفات التهديدات

يحصل كل تهديد على معرف مثل `T-EXEC-003`. الفئات هي:

| الرمز   | الفئة                                      |
| ------- | ------------------------------------------ |
| RECON   | الاستطلاع - جمع المعلومات                  |
| ACCESS  | الوصول الأولي - الحصول على الدخول          |
| EXEC    | التنفيذ - تشغيل إجراءات ضارة               |
| PERSIST | الاستمرارية - الحفاظ على الوصول            |
| EVADE   | تفادي الدفاعات - تجنب الاكتشاف             |
| DISC    | الاكتشاف - التعرف على البيئة               |
| EXFIL   | الاستخراج - سرقة البيانات                  |
| IMPACT  | التأثير - الضرر أو التعطيل                 |

يعين المشرفون المعرفات أثناء المراجعة. لست بحاجة إلى اختيار واحد.

### مستويات المخاطر

| المستوى      | المعنى                                                        |
| ------------ | ------------------------------------------------------------- |
| **حرج**      | اختراق كامل للنظام، أو احتمال عالٍ + تأثير حرج               |
| **عالٍ**     | ضرر كبير محتمل، أو احتمال متوسط + تأثير حرج                  |
| **متوسط**    | خطر متوسط، أو احتمال منخفض + تأثير عالٍ                      |
| **منخفض**    | غير محتمل وتأثيره محدود                                      |

إذا لم تكن متأكدًا من مستوى الخطر، فما عليك سوى وصف التأثير وسنقيّمه.

## عملية المراجعة

1. **الفرز** - نراجع الإرسالات الجديدة خلال 48 ساعة
2. **التقييم** - نتحقق من الجدوى، ونعيّن ربط ATLAS ومعرف التهديد، ونتحقق من مستوى الخطر
3. **التوثيق** - نضمن أن كل شيء منسق ومكتمل
4. **الدمج** - يُضاف إلى نموذج التهديدات والتصور المرئي
