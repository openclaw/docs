### المهام المجدولة (Cron) مقابل Heartbeat

| البعد       | المهام المجدولة (Cron)              | Heartbeat                             |
| --------------- | ----------------------------------- | ------------------------------------- |
| التوقيت          | دقيق (تعبيرات cron، لمرة واحدة)  | تقريبي (الافتراضي كل 30 دقيقة)    |
| سياق الجلسة | جديد (معزول) أو مشترك          | سياق الجلسة الرئيسية الكامل             |
| سجلات المهام    | تُنشأ دائمًا                      | لا تُنشأ أبدًا                         |
| التسليم        | قناة، أو webhook، أو صامت         | ضمن الجلسة الرئيسية                |
| الأنسب لـ        | التقارير، والتذكيرات، ومهام الخلفية | فحوصات البريد الوارد، والتقويم، والإشعارات |

استخدم المهام المجدولة (Cron) عندما تحتاج إلى توقيت دقيق أو تنفيذ معزول. استخدم Heartbeat عندما يستفيد العمل من سياق الجلسة الكامل ويكون التوقيت التقريبي مناسبًا.

## المفاهيم الأساسية

### المهام المجدولة (cron)

Cron هو المجدول المدمج في Gateway للتوقيت الدقيق. يحتفظ بالمهام، ويوقظ الوكيل في الوقت المناسب، ويمكنه تسليم المخرجات إلى قناة دردشة أو نقطة نهاية webhook. يدعم التذكيرات لمرة واحدة، والتعبيرات المتكررة، ومحفزات webhook الواردة.

راجع [المهام المجدولة](/ar/automation/cron-jobs).

### المهام

يتتبع دفتر مهام الخلفية كل الأعمال المنفصلة: تشغيلات ACP، وإنشاء الوكلاء الفرعيين، وتنفيذات cron المعزولة، وعمليات CLI. المهام سجلات وليست مجدولات. استخدم `openclaw tasks list` و`openclaw tasks audit` لفحصها.

راجع [مهام الخلفية](/ar/automation/tasks).

### الالتزامات المستنتجة

الالتزامات هي ذكريات متابعة قصيرة الأجل وقابلة للتفعيل اختياريًا. يستنتجها OpenClaw من المحادثات العادية، ويحصر نطاقها في الوكيل والقناة نفسيهما، ويسلم تسجيلات المتابعة المستحقة عبر Heartbeat. تظل التذكيرات الدقيقة التي يطلبها المستخدم تابعة لـ cron.

راجع [الالتزامات المستنتجة](/ar/concepts/commitments).

### تدفق المهام

تدفق المهام هو طبقة تنسيق التدفقات فوق مهام الخلفية. يدير تدفقات دائمة متعددة الخطوات مع أوضاع مزامنة مُدارة ومعكوسة، وتتبع المراجعات، و`openclaw tasks flow list|show|cancel` للفحص.

راجع [تدفق المهام](/ar/automation/taskflow).

### التعليمات الدائمة

تمنح التعليمات الدائمة الوكيل صلاحية تشغيلية دائمة للبرامج المحددة. تعيش في ملفات مساحة العمل (عادةً `AGENTS.md`) وتُحقن في كل جلسة. ادمجها مع cron للإنفاذ المستند إلى الوقت.

راجع [التعليمات الدائمة](/ar/automation/standing-orders).

### الخطافات

الخطافات الداخلية هي برامج نصية مدفوعة بالأحداث تُشغلها أحداث دورة حياة الوكيل (`/new`، `/reset`، `/stop`)، وCompaction الجلسة، وبدء تشغيل Gateway، وتدفق الرسائل. تُكتشف تلقائيًا من الأدلة ويمكن إدارتها باستخدام `openclaw hooks`. لاعتراض استدعاءات الأدوات داخل العملية، استخدم [خطافات Plugin](/ar/plugins/hooks).

راجع [الخطافات](/ar/automation/hooks).

### Heartbeat

Heartbeat هو دور دوري في الجلسة الرئيسية (الافتراضي كل 30 دقيقة). يجمع عدة فحوصات (البريد الوارد، التقويم، الإشعارات) في دور وكيل واحد مع سياق الجلسة الكامل. لا تُنشئ أدوار Heartbeat سجلات مهام ولا تمدد حداثة إعادة تعيين الجلسة اليومية/الخاملة. استخدم `HEARTBEAT.md` لقائمة تحقق صغيرة، أو كتلة `tasks:` عندما تريد فحوصات دورية للمستحق فقط داخل Heartbeat نفسها. تتخطى ملفات Heartbeat الفارغة باسم `empty-heartbeat-file`؛ ويتخطى وضع مهام المستحق فقط باسم `no-tasks-due`. تؤجل Heartbeat عندما يكون عمل cron نشطًا أو في قائمة الانتظار، ويمكن لـ`heartbeat.skipWhenBusy` أيضًا تأجيلها عندما تكون مسارات الوكيل الفرعي أو المسارات المتداخلة مشغولة.

راجع [Heartbeat](/ar/gateway/heartbeat).

## كيف تعمل معًا

- **Cron** يعالج الجداول الدقيقة (التقارير اليومية، المراجعات الأسبوعية) والتذكيرات لمرة واحدة. كل تنفيذات cron تُنشئ سجلات مهام.
- **Heartbeat** يعالج المراقبة الروتينية (البريد الوارد، التقويم، الإشعارات) في دور مجمع واحد كل 30 دقيقة.
- **الخطافات** تستجيب لأحداث محددة (إعادة تعيين الجلسات، وCompaction، وتدفق الرسائل) باستخدام برامج نصية مخصصة. تغطي خطافات Plugin استدعاءات الأدوات.
- **التعليمات الدائمة** تمنح الوكيل سياقًا دائمًا وحدود صلاحية.
- **تدفق المهام** ينسق التدفقات متعددة الخطوات فوق المهام الفردية.
- **المهام** تتتبع تلقائيًا كل الأعمال المنفصلة لكي تتمكن من فحصها وتدقيقها.
