يرتبط OpenClaw بـ Zalo ClawBot عبر Plugin الخارجي
`@zalo-platforms/openclaw-zaloclawbot` المدرج في الكتالوج. يستخدم تسجيل الدخول رمز QR
لتطبيق Zalo المصغّر.

## التوافق

| إصدار Plugin | إصدار OpenClaw | وسم توزيع npm | الحالة        |
| -------------- | ---------------- | ------------ | ------------- |
| 0.1.x          | >=2026.4.10      | `latest`     | نشط / تجريبي |

## المتطلبات الأساسية

- Node.js **>= 22**
- يجب تثبيت [OpenClaw](https://docs.openclaw.ai/install) (توفّر CLI باسم `openclaw`).
- حساب Zalo على جهاز محمول لمسح رمز QR الخاص بتسجيل الدخول.

## التثبيت باستخدام الإعداد الأولي (موصى به)

شغّل معالج الإعداد الأولي في OpenClaw واختر **Zalo ClawBot** من قائمة القنوات:

```bash
openclaw onboard
```

يثبّت المعالج Plugin من الكتالوج الرسمي (مع التحقق من السلامة)، ويعرض رمز QR لتسجيل الدخول مباشرة في الطرفية، وينهي إعداد القناة بعد أن تمسحه باستخدام تطبيق Zalo. لا حاجة إلى أوامر إضافية.

## التثبيت اليدوي

لإضافة القناة إلى Gateway تم إعداده مسبقًا، اتبع هذه الخطوات:

### 1. ثبّت Plugin

```bash
openclaw plugins install "@zalo-platforms/openclaw-zaloclawbot@0.1.4"
```

استخدم الإصدار المثبّت أعلاه بالضبط (فهو يطابق إدخال الكتالوج الرسمي)، بحيث يتحقق OpenClaw من الحزمة مقابل تجزئة سلامة الكتالوج أثناء التثبيت.

### 2. فعّل Plugin في الإعدادات

```bash
openclaw config set plugins.entries.openclaw-zaloclawbot.enabled true
```

### 3. أنشئ رمز QR وسجّل الدخول

```bash
openclaw channels login --channel openclaw-zaloclawbot
```

امسح رمز QR المعروض في الطرفية باستخدام تطبيق Zalo للجوال، واقبل شروط الاستخدام داخل تطبيق Zalo المصغّر، ثم خوّل الجلسة.

### 4. أعد تشغيل Gateway

```bash
openclaw gateway restart
```

## كيف يعمل

على عكس قناة Zalo القياسية للمطورين التي تتطلب منك تسجيل حساب Zalo رسمي (OA) خاص بك ولصق بيانات اعتماد مطور ثابتة، يعمل Zalo ClawBot بصفته **مساعدًا شخصيًا مرتبطًا بالمالك** باستخدام بنية تحتية رسمية مشتركة:

1. **إعداد أولي آمن:** يوجّه رمز QR إلى تطبيق Zalo مصغّر آمن يربط بوتًا خاصًا تم توفيره حديثًا، ضمن حساب OA رسمي مشترك، مباشرة بمعرّف مستخدم Zalo الخاص بك.
2. **خصوصية مرتبطة بالمالك:** بحكم التصميم، يقتصر البوت على التواصل _فقط_ مع مالكه. تُسقَط الرسائل الواردة من مستخدمين آخرين على مستوى المنصة، مما يجعل الاتصال خاصًا وآمنًا.
3. **مسار API رسمي:** يستخدم Plugin واجهات API لمنصة Zalo Bot بدلًا من
   أتمتة المتصفح أو جلسة الويب.

## التفاصيل الداخلية

يتواصل Plugin الخاص بـ Zalo ClawBot مع واجهات API في Zalo عبر حلقة رسائل مستمرة تعتمد على الاستقصاء الطويل. للحفاظ على وقت تشغيل نظيف وخفيف:

- تستخدم اتصالات الاستقصاء الطويل نقطة النهاية `getUpdates`.
- تكون Webhooks معطّلة افتراضيًا لتشغيلات Gateway المحلية على سطح المكتب/الطرفية.
- تُعالَج الرسائل من جهة العميل وتُربَط مباشرة بوقت تشغيل الوكيل المحلي لديك.

يدير Plugin الخارجي بيانات اعتماد البوت ضمن دليل حالة OpenClaw.
تعامل مع ذلك الدليل على أنه حساس، وأدرجه في سياسة التحكم في الوصول
والنسخ الاحتياطي نفسها المطبّقة على بقية حالة OpenClaw.

## استكشاف الأخطاء وإصلاحها

- **انتهاء مهلة تسجيل الدخول عبر QR:** تنتهي صلاحية رمز تسجيل الدخول (`zbsk`) بعد 5 دقائق لأسباب أمنية. إذا انتهت صلاحية رمز QR قبل أن تمسحه، فأعد ببساطة تشغيل أمر تسجيل الدخول لإنشاء رمز جديد.
- **فشل تحميل Gateway:** تأكد من أن إصدار مضيف OpenClaw هو `2026.4.10` أو أعلى. لا تدعم الإصدارات الأقدم سجل تثبيت Plugin الخارجي من npm.
