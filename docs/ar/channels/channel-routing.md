# القنوات والتوجيه

يوجّه OpenClaw الردود **مرة أخرى إلى القناة التي جاءت منها الرسالة**. لا
يختار النموذج قناة؛ فالتوجيه حتمي وتتحكم فيه تهيئة المضيف.

## المصطلحات الأساسية

- **القناة**: `telegram`، `whatsapp`، `discord`، `irc`، `googlechat`، `slack`، `signal`، `imessage`، `line`، بالإضافة إلى قنوات Plugin. `webchat` هي قناة واجهة WebChat الداخلية وليست قناة صادرة قابلة للتهيئة.
- **AccountId**: نسخة الحساب لكل قناة (عندما يكون ذلك مدعوما).
- الحساب الافتراضي الاختياري للقناة: يختار `channels.<channel>.defaultAccount`
  الحساب الذي يُستخدم عندما لا يحدد مسار صادر `accountId`.
  - في إعدادات الحسابات المتعددة، عيّن افتراضيا صريحا (`defaultAccount` أو `accounts.default`) عند تهيئة حسابين أو أكثر. من دونه، قد يختار التوجيه الاحتياطي أول معرّف حساب مُطبّع.
- **AgentId**: مساحة عمل معزولة + مخزن جلسات ("دماغ").
- **SessionKey**: مفتاح الحاوية المستخدم لتخزين السياق والتحكم في التزامن.

## أشكال مفتاح الجلسة (أمثلة)

تُطوى الرسائل المباشرة إلى الجلسة **الرئيسية** للوكيل افتراضيا:

- `agent:<agentId>:<mainKey>` (الافتراضي: `agent:main:main`)

حتى عندما يكون سجل محادثة الرسائل المباشرة مشتركا مع الرئيسية، تستخدم سياسة صندوق الحماية
والأدوات مفتاح تشغيل دردشة مباشرة مشتقا لكل حساب للرسائل المباشرة الخارجية
حتى لا تُعامَل الرسائل القادمة من القنوات مثل تشغيلات الجلسة الرئيسية المحلية.

تبقى المجموعات والقنوات معزولة لكل قناة:

- المجموعات: `agent:<agentId>:<channel>:group:<id>`
- القنوات/الغرف: `agent:<agentId>:<channel>:channel:<id>`

المحادثات الفرعية:

- تُلحق محادثات Slack/Discord الفرعية `:thread:<threadId>` بالمفتاح الأساسي.
- تُضمّن موضوعات منتدى Telegram ‏`:topic:<topicId>` في مفتاح المجموعة.

أمثلة:

- `agent:main:telegram:group:-1001234567890:topic:42`
- `agent:main:discord:channel:123456:thread:987654`

## تثبيت مسار الرسائل المباشرة الرئيسية

عندما تكون `session.dmScope` هي `main`، قد تشترك الرسائل المباشرة في جلسة رئيسية واحدة.
لمنع استبدال `lastRoute` الخاصة بالجلسة برسائل مباشرة من غير المالك،
يستنتج OpenClaw مالكا مثبتا من `allowFrom` عندما تتحقق كل الشروط الآتية:

- يحتوي `allowFrom` على إدخال واحد فقط غير شامل.
- يمكن تطبيع الإدخال إلى معرّف مرسل ملموس لتلك القناة.
- لا يطابق مرسل الرسالة المباشرة الواردة ذلك المالك المثبت.

في حالة عدم التطابق هذه، لا يزال OpenClaw يسجل بيانات تعريف الجلسة الواردة، لكنه
يتخطى تحديث `lastRoute` للجلسة الرئيسية.

## التسجيل الوارد المحمي

يمكن لقنوات Plugin وسم سجل جلسة وارد على أنه `createIfMissing: false`
عندما يجب ألا ينشئ مسار محمي جلسة OpenClaw جديدة. في هذا الوضع،
قد يحدّث OpenClaw بيانات التعريف و`lastRoute` لجلسة موجودة، لكنه
لا ينشئ إدخال جلسة للتوجيه فقط لمجرد رصد رسالة.

## قواعد التوجيه (كيفية اختيار وكيل)

يختار التوجيه **وكيلا واحدا** لكل رسالة واردة:

1. **تطابق النظير الدقيق** (`bindings` مع `peer.kind` + `peer.id`).
2. **تطابق النظير الأصل** (وراثة المحادثة الفرعية).
3. **تطابق الخادم + الأدوار** (Discord) عبر `guildId` + `roles`.
4. **تطابق الخادم** (Discord) عبر `guildId`.
5. **تطابق الفريق** (Slack) عبر `teamId`.
6. **تطابق الحساب** (`accountId` على القناة).
7. **تطابق القناة** (أي حساب على تلك القناة، `accountId: "*"`).
8. **الوكيل الافتراضي** (`agents.list[].default`، وإلا أول إدخال في القائمة، مع الرجوع احتياطيا إلى `main`).

عندما يتضمن ارتباط عدة حقول مطابقة (`peer`، `guildId`، `teamId`، `roles`)، **يجب أن تتطابق كل الحقول المقدمة** لكي يُطبق ذلك الارتباط.

يحدد الوكيل المطابق مساحة العمل ومخزن الجلسات المستخدمين.

## مجموعات البث (تشغيل عدة وكلاء)

تتيح لك مجموعات البث تشغيل **عدة وكلاء** للنظير نفسه **عندما يرد OpenClaw عادة** (على سبيل المثال: في مجموعات WhatsApp، بعد بوابات الإشارة/التنشيط).

التهيئة:

```json5
{
  broadcast: {
    strategy: "parallel",
    "120363403215116621@g.us": ["alfred", "baerbel"],
    "+15555550123": ["support", "logger"],
  },
}
```

انظر: [مجموعات البث](/ar/channels/broadcast-groups).

## نظرة عامة على التهيئة

- `agents.list`: تعريفات الوكلاء المسماة (مساحة العمل، النموذج، إلخ).
- `bindings`: ربط القنوات/الحسابات/النظراء الواردين بالوكلاء.

مثال:

```json5
{
  agents: {
    list: [{ id: "support", name: "Support", workspace: "~/.openclaw/workspace-support" }],
  },
  bindings: [
    { match: { channel: "slack", teamId: "T123" }, agentId: "support" },
    { match: { channel: "telegram", peer: { kind: "group", id: "-100123" } }, agentId: "support" },
  ],
}
```

## تخزين الجلسات

توجد مخازن الجلسات ضمن دليل الحالة (الافتراضي `~/.openclaw`):

- `~/.openclaw/agents/<agentId>/sessions/sessions.json`
- توجد نصوص JSONL بجانب المخزن

يمكنك تجاوز مسار المخزن عبر `session.store` وقوالب `{agentId}`.

يفحص اكتشاف جلسات Gateway وACP أيضا مخازن الوكلاء المدعومة بالقرص ضمن جذر
`agents/` الافتراضي وضمن جذور `session.store` ذات القوالب. يجب أن تبقى
المخازن المكتشفة داخل جذر الوكيل المحلول ذاك وأن تستخدم ملف
`sessions.json` عاديا. تُتجاهل الروابط الرمزية والمسارات الخارجة عن الجذر.

## سلوك WebChat

تتصل WebChat **بالوكيل المحدد** وتنتقل افتراضيا إلى الجلسة الرئيسية للوكيل.
ولهذا السبب، تتيح لك WebChat رؤية السياق العابر للقنوات لذلك
الوكيل في مكان واحد.

## سياق الرد

تتضمن الردود الواردة:

- `ReplyToId` و`ReplyToBody` و`ReplyToSender` عند توفرها.
- يُلحق السياق المقتبس بـ `Body` ككتلة `[Replying to ...]`.

هذا متسق عبر القنوات.
