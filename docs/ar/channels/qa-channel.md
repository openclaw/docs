`qa-channel` هو نقل رسائل اصطناعي مضمّن لاختبارات ضمان جودة OpenClaw الآلية. ليس قناة إنتاج — فهو موجود لاختبار حدود Plugin القناة نفسها التي تستخدمها وسائل النقل الحقيقية مع إبقاء الحالة حتمية وقابلة للفحص بالكامل.

## ما يفعله

- قواعد أهداف من فئة Slack:
  - `dm:<user>`
  - `channel:<room>`
  - `thread:<room>/<thread>`
- ناقل اصطناعي مدعوم بـ HTTP لحقن الرسائل الواردة، والتقاط النصوص الصادرة، وإنشاء السلاسل، والتفاعلات، والتعديلات، والحذف، وإجراءات البحث/القراءة.
- مشغّل فحص ذاتي من جهة المضيف يكتب تقرير Markdown إلى `.artifacts/qa-e2e/`.

## الإعدادات

```json
{
  "channels": {
    "qa-channel": {
      "baseUrl": "http://127.0.0.1:43123",
      "botUserId": "openclaw",
      "botDisplayName": "OpenClaw QA",
      "allowFrom": ["*"],
      "pollTimeoutMs": 1000
    }
  }
}
```

مفاتيح الحساب:

- `enabled` — مفتاح التفعيل الرئيسي لهذا الحساب.
- `name` — تسمية عرض اختيارية.
- `baseUrl` — عنوان URL للناقل الاصطناعي.
- `botUserId` — معرّف مستخدم البوت بأسلوب Matrix المستخدم في قواعد الأهداف.
- `botDisplayName` — اسم العرض للرسائل الصادرة.
- `pollTimeoutMs` — نافذة انتظار الاستطلاع الطويل. عدد صحيح بين 100 و30000.
- `allowFrom` — قائمة السماح للمرسلين (معرّفات المستخدمين أو `"*"`).
- `defaultTo` — الهدف الاحتياطي عند عدم تقديم أي هدف.
- `actions.messages` / `actions.reactions` / `actions.search` / `actions.threads` — تقييد الأدوات لكل إجراء.

مفاتيح الحسابات المتعددة في المستوى الأعلى:

- `accounts` — سجل لتجاوزات كل حساب مسمّى، مفهرس بمعرّف الحساب.
- `defaultAccount` — معرّف الحساب المفضّل عند تكوين عدة حسابات.

## المشغّلات

الفحص الذاتي من جهة المضيف (يكتب تقرير Markdown ضمن `.artifacts/qa-e2e/`):

```bash
pnpm qa:e2e
```

يمر هذا عبر `qa-lab`، ويشغّل ناقل QA داخل المستودع، ويقلع شريحة وقت تشغيل `qa-channel` المضمّنة، ثم يشغّل فحصًا ذاتيًا حتميًا.

مجموعة السيناريوهات الكاملة المدعومة بالمستودع:

```bash
pnpm openclaw qa suite
```

يشغّل السيناريوهات بالتوازي مقابل مسار QA الخاص بـ Gateway. راجع [نظرة عامة على QA](/ar/concepts/qa-e2e-automation) لمعرفة السيناريوهات والملفات التعريفية وأوضاع المزوّد.

موقع QA المدعوم بـ Docker (Gateway + واجهة مصحح QA Lab في حزمة واحدة):

```bash
pnpm qa:lab:up
```

يبني موقع QA، ويشغّل حزمة Gateway + QA Lab المدعومة بـ Docker، ويطبع عنوان URL الخاص بـ QA Lab. من هناك يمكنك اختيار السيناريوهات، واختيار مسار النموذج، وتشغيل عمليات فردية، ومشاهدة النتائج مباشرة. مصحح QA Lab منفصل عن حزمة Control UI المشحونة.
