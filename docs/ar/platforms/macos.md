تطبيق macOS هو **رفيق شريط القوائم** لـ OpenClaw: واجهة أصلية في شريط النظام، ومطالبات أذونات macOS، والإشعارات، وWebChat، والإدخال الصوتي، وCanvas، وأدوات Node المستضافة على Mac مثل `system.run`.

هل تحتاج فقط إلى CLI وGateway؟ ابدأ بـ [بدء الاستخدام](/ar/start/getting-started).

## التنزيل

احصل على إصدارات تطبيق macOS من [إصدارات OpenClaw على GitHub](https://github.com/openclaw/openclaw/releases).
عندما يتضمن إصدارٌ أصول تطبيق macOS، ابحث عن:

- `OpenClaw-<version>.dmg` (المفضّل)
- `OpenClaw-<version>.zip`

تتضمن بعض الإصدارات أصول CLI أو الأدلة أو Windows فقط. إذا لم يتضمن أحدث إصدار
أصلًا لتطبيق macOS، فاستخدم أحدث إصدار يتضمنه، أو ابنِ التطبيق من المصدر باستخدام
[إعداد تطوير macOS](/ar/platforms/mac/dev-setup).

## التشغيل الأول

1. ثبّت **OpenClaw.app** وشغّله.
2. اختر **This Mac** لاستخدام Gateway محلي، أو اتصل بـ Gateway بعيد.
3. انتظر بينما يثبّت التطبيق بيئة تشغيل CLI المطابقة. وفي الوضع المحلي،
   يثبّت Gateway ويشغّله أيضًا.
4. أنشئ اتصال الاستدلال من خلال فحص مباشر للنموذج. بعد نجاحه، يتولى OpenClaw
   بقية الإعداد.
5. أكمل قائمة التحقق من أذونات macOS وأرسل رسالة اختبار الإعداد الأولي.

إذا وصل التطبيق إلى Gateway موجود أُعدّ نموذج للوكيل الافتراضي فيه،
فإنه يعامل Gateway هذا على أنه مُعدّ بالفعل، ويتخطى الإعداد الأولي لموفّر الخدمة وOpenClaw،
ويفتح لوحة المعلومات. وإذا تعذّر الاتصال بـ Gateway أو لم يكن لدى
وكيله الافتراضي نموذج، يظل الإعداد الأولي للاستدلال متاحًا
للاسترداد.

لمسار إعداد CLI/Gateway، استخدم [بدء الاستخدام](/ar/start/getting-started).
ولاسترداد الأذونات، استخدم [أذونات macOS](/ar/platforms/mac/permissions).

## التحديثات

توضح بطاقة التحديث في لوحة المعلومات ما سيحدّثه التطبيق:

- يعني **تحديث تطبيق Mac وGateway** أن التطبيق الموقّع يملك Gateway المحلي الذي يديره launchd.
  يحدّث Sparkle التطبيق أولًا؛ وبعد إعادة التشغيل، يحدّث التطبيق
  Gateway الخاص به تلقائيًا ويعيد تشغيله بالإصدار المطابق، ثم يتحقق من
  الاتصال.
- يعني **تحديث Gateway** أن التطبيق متصل بـ Gateway بعيد، أو Gateway محلي
  مُدار يدويًا، أو تثبيت آخر لا يملكه التطبيق. يشغّل الزر
  مسار التحديث المعتاد لذلك الـ Gateway بدلًا من تغيير تطبيق Mac.

يبقى التحديث المنسّق الفاشل في نافذته الشبيهة بنافذة الإعداد، مع إجراءات لإعادة المحاولة
و[دليل التحديث](/ar/install/updating) وDiscord. ولا يخفض الإصلاح التلقائي
إصدار Gateway أحدث مطلقًا، ولا يتجاوز تثبيت قناة `extended-stable`.

بعد نجاح التحديث، يعثر التطبيق على أحدث جلسة مباشرة من المستوى الأعلى
استخدمها شخص، ويمنح ذلك الوكيل حدث تحديث لمرة واحدة. لا يؤثر نشاط Heartbeat
وCron في هذا الاختيار. ويمكن للوكيل بعد ذلك الترحيب بعودتك
من المحادثة التي يُرجح أنك كنت تستخدمها. وفي الوضع البعيد، لا يحدّث التطبيق
إلا بيئة تشغيل Node المحلية على Mac، ويتخطى الإشعار عندما يكون
Gateway البعيد أقدم من التطبيق.

يتبع Sparkle إعداد `update.channel` الخاص بـ Gateway. يفعّل `beta` و`dev`
إصدارات التطبيق التجريبية؛ بينما يُبقي `stable` و`extended-stable` والقيم المفقودة أو غير المعروفة
التطبيق على الإصدارات المستقرة.

## فتح الروابط في لوحة المعلومات

في لوحة المعلومات المضمّنة في تطبيق macOS، يؤدي النقر على رابط ويب خارجي إلى فتحه في شريط جانبي قابل لتغيير الحجم في المتصفح، بعرض نصف النافذة، مع إبقاء تنقل لوحة المعلومات ظاهرًا. اسحب الفاصل لاختيار عرض آخر؛ وسيتذكره التطبيق. يُفتح كل رابط في علامة تبويب مستقلة، ويظهر شريط علامات التبويب عند فتح عدة صفحات، وتؤدي إعادة النقر على الرابط نفسه إلى إعادة استخدام علامة تبويبه الموجودة. اسحب علامات التبويب لإعادة ترتيبها، وأغلقها بزر إغلاق علامة التبويب أو بالنقر الأوسط، وانقر بزر الماوس الأيمن على علامة تبويب للوصول إلى **Open in Default Browser** و**Copy Link** و**Reload** و**Close Tab** و**Close Other Tabs**. تتنقل عناصر تحكم الرجوع/التقدم في شريط عنوان النافذة وإيماءات لوحة اللمس عبر سجل لوحة المعلومات؛ بينما تتنقل عناصر تحكم الرجوع/التقدم الخاصة بالشريط الجانبي عبر سجل علامة التبويب النشطة. ويتضمن الشريط الجانبي أيضًا عناصر تحكم لإعادة التحميل، والفتح في المتصفح الافتراضي، والإغلاق.

تتبع عناصر تحكم شريط العنوان الشريط الجانبي للتطبيق: فعندما يكون موسّعًا، يظهر زرا الرجوع/التقدم عند حافته اليمنى بجانب زر تبديل الشريط الجانبي؛ وعندما يكون مطويًا، يفسحان المجال لزر بحث (يفتح لوحة الأوامر) وزر جلسة جديدة.

انقر بزر الماوس الأيمن على رابط خارجي لاختيار **Open in Sidebar** أو **Open in Default Browser** أو **Copy Link**. تستمر النقرات المعدّلة وروابط النوافذ الجديدة التي يفعّلها المستخدم من لوحة المعلومات في الفتح داخل المتصفح الافتراضي؛ أما روابط النوافذ الجديدة داخل الشريط الجانبي فتُفتح كعلامات تبويب جديدة فيه. وتحافظ صفحات واجهة التحكم المعتادة المستضافة في المتصفح على السلوك الطبيعي للروابط وقائمة السياق في المتصفح.

## استيراد جلسات تسجيل الدخول من المتصفح

عند فتح الشريط الجانبي للمتصفح لأول مرة بينما يعمل التطبيق مع Gateway محلي، تعرض لوحة المعلومات لافتة قابلة للإغلاق عند وجود ملف تعريف من عائلة Chrome يحتوي على ملفات تعريف ارتباط على Mac. تعرض اللافتة نسخ ملفات تعريف الارتباط هذه إلى ملف تعريف مُدار ومعزول تستخدمه الوكلاء للتصفح. اختر ملف تعريف من عنصر التحكم **Import** (قد يكون Touch ID مطلوبًا)؛ ويظهر التقدم وعدد ملفات تعريف الارتباط المستوردة في الموضع نفسه، ولا تُنسخ سوى ملفات تعريف الارتباط — ولا تغادر كلمات المرور المتصفح المصدر مطلقًا. يسجّل إغلاق اللافتة هذا الاختيار؛ ويتيح المسار **Settings → General → Browser login → Import…** عرضها مجددًا في أي وقت. راجع [المتصفح](/ar/cli/browser) للاطلاع على مسار الاستيراد الأساسي وبوابة `browser.allowSystemProfileImport`.

## اختيار وضع Gateway

| الوضع   | استخدمه عندما                                                                    | صفحة التفاصيل                                        |
| ------ | ------------------------------------------------------------------------------ | -------------------------------------------------- |
| محلي  | يجب أن يشغّل Mac هذا Gateway ويُبقيه قيد التشغيل باستخدام launchd.                | [Gateway على macOS](/ar/platforms/mac/bundled-gateway) |
| بعيد | يشغّل مضيف آخر Gateway؛ ويتحكم فيه Mac هذا عبر SSH أو LAN أو Tailnet. | [التحكم عن بُعد](/ar/platforms/mac/remote)            |

يتطلب كلا الوضعين تثبيت CLI ‏`openclaw` لأن التطبيق يعيد استخدام بيئة تشغيل
مضيف Node الخاصة به. على جهاز Mac جديد، يثبّت التطبيق CLI المطابق تلقائيًا؛ ثم
يبدأ الوضع المحلي معالج Gateway، بينما يتصل الوضع البعيد بـ Gateway المحدد
من دون تشغيل Gateway محلي ثانٍ.
راجع [Gateway على macOS](/ar/platforms/mac/bundled-gateway) للاسترداد اليدوي.

## ما يديره التطبيق

- حالة شريط القوائم، والإشعارات، والصحة، وWebChat.
- مطالبات أذونات macOS للشاشة والميكروفون والكلام والأتمتة وإمكانية الوصول.
- Node واحد على Mac يجمع بين Canvas الأصلي، والتقاط الكاميرا/الشاشة، والإشعارات،
  والموقع، والتحكم في الحاسوب، وبين أوامر النظام والمتصفح
  وPlugin وSkills وMCP الخاصة بمضيف Node في CLI.
- مطالبات الموافقة على التنفيذ للأوامر المستضافة على Mac.
- التنفيذ في سياق التطبيق لأوامر الصدفة المعتمدة، مع الحفاظ على إسناد
  أذونات macOS إلى التطبيق بينما تملك بيئة تشغيل CLI سياسة Node المشتركة.
- أنفاق SSH في الوضع البعيد أو اتصالات Gateway المباشرة.

لا يستبدل التطبيق وثائق Gateway أو CLI العامة. توجد إعدادات Gateway
وموفّرو الخدمة وplugins والقنوات والأدوات والأمان في
وثائقها الخاصة.
