## بدء سريع

### وضع Gateway

1. ابدأ Gateway.

```bash
openclaw gateway
```

2. افتح TUI.

```bash
openclaw tui
```

3. اكتب رسالة واضغط Enter.

Gateway بعيدة:

```bash
openclaw tui --url ws://<host>:<port> --token <gateway-token>
```

استخدم `--password` إذا كانت Gateway لديك تستخدم مصادقة كلمة المرور.

### الوضع المحلي

شغّل TUI من دون Gateway:

```bash
openclaw chat
# or
openclaw tui --local
```

ملاحظات:

- إن `openclaw chat` و`openclaw terminal` هما اسمان مستعاران لـ `openclaw tui --local`.
- لا يمكن الجمع بين `--local` و`--url` أو `--token` أو `--password`.
- يستخدم الوضع المحلي وقت تشغيل الوكيل المضمّن مباشرة. وتعمل معظم الأدوات المحلية، لكن الميزات الخاصة بـ Gateway غير متاحة.

## ما الذي تراه

- الرأس: عنوان URL الخاص بالاتصال، والوكيل الحالي، والجلسة الحالية.
- سجل الدردشة: رسائل المستخدم، وردود المساعد، وإشعارات النظام، وبطاقات الأدوات.
- سطر الحالة: حالة الاتصال/التشغيل (connecting، running، streaming، idle، error).
- التذييل: حالة الاتصال + الوكيل + الجلسة + النموذج + think/fast/verbose/trace/reasoning + عدد الرموز + deliver.
- الإدخال: محرر نص مع إكمال تلقائي.

## النموذج الذهني: الوكلاء + الجلسات

- الوكلاء هم slugs فريدة (مثل `main` و`research`). وتكشف Gateway عن القائمة.
- تنتمي الجلسات إلى الوكيل الحالي.
- تُخزَّن مفاتيح الجلسات بالشكل `agent:<agentId>:<sessionKey>`.
  - إذا كتبت `/session main`، فإن TUI توسّعه إلى `agent:<currentAgent>:main`.
  - إذا كتبت `/session agent:other:main`، فإنك تنتقل إلى جلسة ذلك الوكيل صراحةً.
- نطاق الجلسة:
  - `per-sender` ‏(الافتراضي): يمتلك كل وكيل عدة جلسات.
  - `global`: تستخدم TUI دائمًا الجلسة `global` ‏(وقد يكون المحدِّد فارغًا).
- يكون الوكيل الحالي + الجلسة الحالية مرئيين دائمًا في التذييل.

## الإرسال + التسليم

- تُرسل الرسائل إلى Gateway؛ ويكون التسليم إلى providers معطلًا افتراضيًا.
- فعّل تسليم الأدوار:
  - `/deliver on`
  - أو لوحة Settings
  - أو ابدأ باستخدام `openclaw tui --deliver`

## المحددات + الطبقات العلوية

- محدِّد النموذج: يسرد النماذج المتاحة ويضبط تجاوز الجلسة.
- محدِّد الوكيل: اختر وكيلًا مختلفًا.
- محدِّد الجلسة: يعرض فقط جلسات الوكيل الحالي.
- Settings: بدّل deliver، وتوسيع مخرجات الأدوات، ورؤية التفكير.

## اختصارات لوحة المفاتيح

- Enter: إرسال الرسالة
- Esc: إجهاض التشغيل النشط
- Ctrl+C: مسح الإدخال (اضغط مرتين للخروج)
- Ctrl+D: خروج
- Ctrl+L: محدِّد النموذج
- Ctrl+G: محدِّد الوكيل
- Ctrl+P: محدِّد الجلسة
- Ctrl+O: تبديل توسيع مخرجات الأدوات
- Ctrl+T: تبديل رؤية التفكير (يعيد تحميل السجل)

## أوامر الشرطة المائلة

الأساسية:

- `/help`
- `/status`
- `/agent <id>` ‏(أو `/agents`)
- `/session <key>` ‏(أو `/sessions`)
- `/model <provider/model>` ‏(أو `/models`)

عناصر التحكم في الجلسة:

- `/think <off|minimal|low|medium|high>`
- `/fast <status|on|off>`
- `/verbose <on|full|off>`
- `/trace <on|off>`
- `/reasoning <on|off|stream>`
- `/usage <off|tokens|full>`
- `/elevated <on|off|ask|full>` ‏(الاسم المستعار: `/elev`)
- `/activation <mention|always>`
- `/deliver <on|off>`

دورة حياة الجلسة:

- `/new` أو `/reset` ‏(إعادة تعيين الجلسة)
- `/abort` ‏(إجهاض التشغيل النشط)
- `/settings`
- `/exit`

الوضع المحلي فقط:

- يفتح `/auth [provider]` تدفق المصادقة/تسجيل الدخول الخاص بالـ provider داخل TUI.

يتم تمرير أوامر الشرطة المائلة الأخرى في Gateway ‏(مثل `/context`) إلى Gateway وتُعرض كمخرجات نظام. راجع [أوامر الشرطة المائلة](/ar/tools/slash-commands).

## أوامر shell المحلية

- ابدأ السطر بالبادئة `!` لتشغيل أمر shell محلي على مضيف TUI.
- تطلب TUI مرة واحدة في كل جلسة السماح بالتنفيذ المحلي؛ ويؤدي الرفض إلى إبقاء `!` معطلًا للجلسة.
- تعمل الأوامر في shell جديدة وغير تفاعلية داخل دليل العمل الخاص بـ TUI ‏(من دون `cd`/بيئة مستمرة).
- تستقبل أوامر shell المحلية المتغير `OPENCLAW_SHELL=tui-local` في بيئتها.
- تُرسل `!` المفردة كرسالة عادية؛ كما أن المسافات البادئة لا تفعّل التنفيذ المحلي.

## إصلاح الإعدادات من TUI المحلي

استخدم الوضع المحلي عندما تكون الإعدادات الحالية صحيحة أصلًا وتريد أن يقوم
الوكيل المضمّن بفحصها على الجهاز نفسه، ومقارنتها بالوثائق،
والمساعدة في إصلاح الانحراف من دون الاعتماد على Gateway عاملة.

إذا كان `openclaw config validate` يفشل بالفعل، فابدأ بـ `openclaw configure`
أو `openclaw doctor --fix` أولًا. فالأمر `openclaw chat` لا يتجاوز حاجز
الإعدادات غير الصالحة.

حلقة نموذجية:

1. ابدأ الوضع المحلي:

```bash
openclaw chat
```

2. اطلب من الوكيل ما الذي تريد التحقق منه، على سبيل المثال:

```text
Compare my gateway auth config with the docs and suggest the smallest fix.
```

3. استخدم أوامر shell المحلية للحصول على دليل دقيق والتحقق:

```text
!openclaw config file
!openclaw docs gateway auth token secretref
!openclaw config validate
!openclaw doctor
```

4. طبّق تغييرات ضيقة باستخدام `openclaw config set` أو `openclaw configure`، ثم أعد تشغيل `!openclaw config validate`.
5. إذا أوصى Doctor بترحيل أو إصلاح تلقائي، فراجعه وشغّل `!openclaw doctor --fix`.

نصائح:

- فضّل `openclaw config set` أو `openclaw configure` على التحرير اليدوي لـ `openclaw.json`.
- يقوم `openclaw docs "<query>"` بالبحث في فهرس الوثائق الحي من الجهاز نفسه.
- يكون `openclaw config validate --json` مفيدًا عندما تريد أخطاء schema وSecretRef/resolvability بشكل منظم.

## مخرجات الأدوات

- تظهر استدعاءات الأدوات كبطاقات تحتوي على الوسيطات + النتائج.
- يبدّل Ctrl+O بين العرض المطوي/الموسع.
- أثناء تشغيل الأدوات، تتدفق التحديثات الجزئية إلى البطاقة نفسها.

## ألوان الطرفية

- تحتفظ TUI بنص جسم المساعد بلون المقدمة الافتراضي لطرفيتك بحيث تبقى الطرفيات الداكنة والفاتحة قابلة للقراءة.
- إذا كانت طرفيتك تستخدم خلفية فاتحة وكان الاكتشاف التلقائي خاطئًا، فاضبط `OPENCLAW_THEME=light` قبل تشغيل `openclaw tui`.
- لفرض اللوحة الداكنة الأصلية بدلًا من ذلك، اضبط `OPENCLAW_THEME=dark`.

## السجل + البث

- عند الاتصال، تقوم TUI بتحميل أحدث سجل (الافتراضي 200 رسالة).
- يتم تحديث الاستجابات المتدفقة في مكانها حتى تُنهى.
- كما تستمع TUI أيضًا إلى أحداث أدوات الوكيل للحصول على بطاقات أدوات أغنى.

## تفاصيل الاتصال

- تسجّل TUI نفسها لدى Gateway على أنها `mode: "tui"`.
- تظهر عمليات إعادة الاتصال كرسالة نظام؛ وتُعرض فجوات الأحداث في السجل.

## الخيارات

- `--local`: التشغيل مقابل وقت تشغيل الوكيل المحلي المضمّن
- `--url <url>`: عنوان URL لـ WebSocket الخاص بـ Gateway ‏(الافتراضي من الإعدادات أو `ws://127.0.0.1:<port>`)
- `--token <token>`: رمز Gateway ‏(إذا كان مطلوبًا)
- `--password <password>`: كلمة مرور Gateway ‏(إذا كانت مطلوبة)
- `--session <key>`: مفتاح الجلسة ‏(الافتراضي: `main`، أو `global` عندما يكون النطاق عامًا)
- `--deliver`: تسليم ردود المساعد إلى provider ‏(معطل افتراضيًا)
- `--thinking <level>`: تجاوز مستوى التفكير للإرسال
- `--message <text>`: إرسال رسالة أولية بعد الاتصال
- `--timeout-ms <ms>`: مهلة الوكيل بالمللي ثانية ‏(الافتراضي من `agents.defaults.timeoutSeconds`)
- `--history-limit <n>`: عدد إدخالات السجل المطلوب تحميلها ‏(الافتراضي `200`)

ملاحظة: عندما تضبط `--url`، لا تعود TUI إلى بيانات الاعتماد الموجودة في الإعدادات أو البيئة.
مرّر `--token` أو `--password` صراحةً. ويُعد غياب بيانات الاعتماد الصريحة خطأ.
وفي الوضع المحلي، لا تمرر `--url` أو `--token` أو `--password`.

## استكشاف الأخطاء وإصلاحها

لا توجد مخرجات بعد إرسال رسالة:

- شغّل `/status` داخل TUI للتأكد من أن Gateway متصلة وفي حالة idle/busy.
- تحقّق من سجلات Gateway: ‏`openclaw logs --follow`.
- تأكد من أن الوكيل يمكنه التشغيل: ‏`openclaw status` و`openclaw models status`.
- إذا كنت تتوقع رسائل في قناة دردشة، فعّل التسليم (`/deliver on` أو `--deliver`).

## استكشاف أخطاء الاتصال وإصلاحها

- `disconnected`: تأكد من أن Gateway تعمل وأن قيم `--url/--token/--password` صحيحة.
- لا يوجد وكلاء في المحدِّد: تحقق من `openclaw agents list` ومن إعدادات التوجيه لديك.
- محدِّد الجلسات فارغ: قد تكون في النطاق العام أو لا توجد لديك جلسات بعد.
