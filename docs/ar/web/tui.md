## البدء السريع

### وضع Gateway

1. شغّل Gateway.

```bash
openclaw gateway
```

2. افتح TUI.

```bash
openclaw tui
```

3. اكتب رسالة واضغط Enter.

Gateway بعيد:

```bash
openclaw tui --url ws://<host>:<port> --token <gateway-token>
```

استخدم `--password` إذا كان Gateway يستخدم مصادقة كلمة المرور.

### الوضع المحلي

شغّل TUI من دون Gateway:

```bash
openclaw chat
# أو
openclaw tui --local
```

- `openclaw chat` و`openclaw terminal` اسمان مستعاران لـ `openclaw tui --local`.
- لا يمكن الجمع بين `--local` و`--url` أو `--token` أو `--password`.
- يستخدم الوضع المحلي وقت تشغيل الوكيل المضمّن مباشرةً. تعمل معظم الأدوات المحلية، لكن الميزات المتاحة حصريًا عبر Gateway غير متوفرة.
- يختار `openclaw` المجرّد (من دون أمر فرعي) هدفًا تلقائيًا: يشغّل التثبيت غير المضبوط إعداد الاستدلال الأولي؛ وتفتح التهيئة غير الصالحة إرشادات Doctor التقليدية؛ ويفتح Gateway مضبوط ويمكن الوصول إليه واجهة TUI هذه في وضع Gateway؛ وإلا فيفتحها نموذج محلي مضبوط في الوضع المحلي.

## ما يظهر لك

- الرأس: عنوان URL للاتصال، والوكيل الحالي، والجلسة الحالية.
- سجل المحادثة: رسائل المستخدم، وردود المساعد، وإشعارات النظام، وبطاقات الأدوات.
- سطر الحالة: حالة الاتصال/التشغيل (جارٍ الاتصال، قيد التشغيل، قيد البث، خامل، خطأ).
- التذييل: الوكيل + الجلسة + النموذج + حالة الهدف + التفكير/السرعة/الإسهاب/التتبّع/الاستدلال + أعداد الرموز + التسليم. عند تمكين `tui.footer.showRemoteHost`، تعرض اتصالات Gateway البعيدة أيضًا مضيف الاتصال.
- الإدخال: محرر نصوص مزوّد بالإكمال التلقائي.

## النموذج الذهني: الوكلاء + الجلسات

- الوكلاء عبارة عن أسماء مختصرة فريدة (مثل `main` و`research`). يوفّر Gateway القائمة.
- تنتمي الجلسات إلى الوكيل الحالي.
- تُخزَّن مفاتيح الجلسات بصيغة `agent:<agentId>:<sessionKey>`.
  - إذا كتبت `/session main`، فستوسّعه TUI إلى `agent:<currentAgent>:main`.
  - إذا كتبت `/session agent:other:main`، فستنتقل صراحةً إلى جلسة ذلك الوكيل.
- نطاق الجلسة:
  - `per-sender` (الافتراضي): لكل وكيل جلسات متعددة.
  - `global`: تستخدم TUI دائمًا جلسة `global` (قد يكون المنتقي فارغًا).
- يظهر الوكيل الحالي + الجلسة دائمًا في التذييل.
- لإظهار مضيف Gateway للاتصالات غير المحلية المستندة إلى عنوان URL، فعّل ذلك باستخدام:

  ```bash
  openclaw config set tui.footer.showRemoteHost true
  ```

  القيمة الافتراضية هي `false`. لا تعرض اتصالات الاسترجاع المحلي والاتصالات المحلية المضمّنة تسمية مضيف مطلقًا.

- إذا كانت للجلسة [غاية](/ar/tools/goal)، فسيعرض التذييل حالتها الموجزة:
  `Pursuing goal` أو `Goal paused (/goal resume)` أو `Goal blocked (/goal resume)` أو `Goal achieved`.
- عند التشغيل من دون `--session`، تستأنف TUI في وضع Gateway آخر جلسة محددة لنفس Gateway والوكيل ونطاق الجلسة إذا كانت تلك الجلسة لا تزال موجودة. يظل تمرير `--session` أو `/session` أو `/new` أو `/reset` إجراءً صريحًا.

## الإرسال + التسليم

- تذهب الرسائل دائمًا إلى Gateway (أو وقت التشغيل المضمّن في الوضع المحلي)؛ أما تسليم رد المساعد مجددًا إلى مزوّد محادثة فهو خطوة منفصلة معطّلة افتراضيًا.
- تمثّل TUI سطح مصدر داخليًا مثل WebChat، وليست قناة صادرة عامة. يمكن لأُطر التشغيل التي تتطلب `tools.message` للردود المرئية استيفاء دورة TUI النشطة باستخدام `message.send` بلا هدف؛ ويظل التسليم الصريح عبر المزوّد يستخدم القنوات المضبوطة المعتادة ولا يرجع مطلقًا إلى `lastChannel`.
- يُثبَّت إعداد التسليم لجلسة TUI بأكملها عند التشغيل: ابدأ باستخدام `openclaw tui --deliver` لتفعيله. لا يوجد أمر شرطة مائلة `/deliver` أو مفتاح تبديل في الإعدادات لتغييره في منتصف الجلسة؛ أعد تشغيل TUI لتغييره.

## المنتقيات + الطبقات المتراكبة

- منتقي النموذج: يسرد النماذج المتاحة ويضبط تجاوز الجلسة.
- منتقي الوكيل: اختر وكيلاً مختلفًا.
- منتقي الجلسة: يعرض ما يصل إلى 50 جلسة للوكيل الحالي تم تحديثها خلال آخر 7 أيام. استخدم `/session <key>` للانتقال إلى جلسة أقدم معروفة.
- الإعدادات (`/settings`): بدّل توسيع مخرجات الأدوات وإظهار التفكير. لا تتحكم هذه اللوحة في التسليم.

## اختصارات لوحة المفاتيح

- Enter: إرسال الرسالة
- Esc: إيقاف التشغيل النشط
- Ctrl+C: مسح الإدخال (اضغط مرتين للخروج)
- Ctrl+D: الخروج
- Ctrl+L: منتقي النموذج
- Ctrl+G: منتقي الوكيل
- Ctrl+P: منتقي الجلسة
- Ctrl+O: تبديل توسيع مخرجات الأدوات
- Ctrl+T: تبديل إظهار التفكير (يعيد تحميل السجل)

## أوامر الشرطة المائلة

الأساسية:

- `/help`
- `/status` (يُمرَّر إلى Gateway؛ يعرض ملخص الجلسة/النموذج)
- `/gateway-status` (الاسم المستعار `/gwstatus`؛ يعرض حالة اتصال Gateway مباشرةً)
- `/agent <id>` (أو `/agents`)
- `/session <key>` (أو `/sessions`)
- `/model <provider/model>` (أو `/models`)

عناصر التحكم في الجلسة:

- `/think <off|minimal|low|medium|high>` (قد تضيف المستويات الأعلى درجات مثل `xhigh`/`max` بحسب النموذج)
- `/fast <status|auto|on|off>`
- `/verbose <on|full|off>`
- `/trace <on|off>`
- `/reasoning <on|off|stream>`
- `/usage <off|tokens|full|reset>` (تؤدي `reset`/`inherit`/`clear`/`default` إلى مسح تجاوز الجلسة)
- `/goal [status] | /goal start <objective> | /goal edit <objective> | /goal pause|resume|complete|block|clear`
- `/elevated <on|off|ask|full>` (الاسم المستعار: `/elev`)
- `/activation <mention|always>`

دورة حياة الجلسة:

- `/new` (إنشاء جلسة جديدة ومعزولة تحت مفتاح جديد؛ لا يؤثر في عملاء TUI الآخرين الموجودين في الجلسة القديمة)
- `/reset` (إعادة ضبط مفتاح الجلسة الحالي في موضعه)
- `/abort` (إيقاف التشغيل النشط)
- `/settings`
- `/exit` (أو `/quit`)

الوضع المحلي فقط:

- يفتح `/auth [provider]` مسار مصادقة/تسجيل دخول المزوّد داخل TUI.

OpenClaw:

- يعيدك `/openclaw [request]` من واجهة TUI العادية للوكيل إلى محادثة الإعداد/الإصلاح [OpenClaw](#openclaw-setup-and-repair-helper)، مع إمكانية تمرير طلب واحد.

تُمرَّر أوامر الشرطة المائلة الأخرى الخاصة بـ Gateway (مثل `/context`) إلى Gateway وتظهر كمخرجات للنظام. راجع [أوامر الشرطة المائلة](/ar/tools/slash-commands).

## أوامر الصدفة المحلية

- ابدأ السطر بـ `!` لتشغيل أمر صدفة محلي على مضيف TUI.
- تطلب TUI الإذن مرة واحدة في كل جلسة للسماح بالتنفيذ المحلي؛ ويؤدي الرفض إلى إبقاء `!` معطّلاً طوال الجلسة.
- تعمل الأوامر في صدفة جديدة غير تفاعلية ضمن دليل عمل TUI (من دون `cd`/بيئة مستديمة).
- تتلقى أوامر الصدفة المحلية `OPENCLAW_SHELL=tui-local` ضمن بيئتها.
- يُرسل `!` المنفرد كرسالة عادية؛ ولا تؤدي المسافات البادئة إلى تشغيل التنفيذ المحلي.

## مساعد إعداد OpenClaw وإصلاحه

OpenClaw هو مساعد الإعداد/الإصلاح من المستوى الصفري، ويُتاح باسم `openclaw setup` بعد اجتياز النموذج الافتراضي المضبوط فحص استدلال مباشر. إذا لم يكن الاستدلال متاحًا، تعود عملية التشغيل التفاعلية إلى إعداد الاستدلال الأولي، وتفشل الأتمتة مع إرشادات الإصلاح. يعمل داخل صدفة TUI المحلية نفسها مثل `openclaw tui --local`، ومدعوم بوكيل ذكاء اصطناعي مقيّد بعمليات OpenClaw محددة الأنواع والخاضعة للموافقة:

```bash
openclaw setup                       # البدء تفاعليًا
openclaw setup -m "status"           # تشغيل طلب واحد ثم الخروج
openclaw setup -m "set default model openai/gpt-5.2" --yes   # تطبيق كتابة على التهيئة
```

- تحتاج عمليات الكتابة المستديمة على التهيئة إلى موافقة: إما التأكيد تفاعليًا أو تمرير `--yes`.
- يطبع `--json` النظرة العامة عند بدء التشغيل بصيغة JSON بدلاً من بدء المحادثة.
- من داخل OpenClaw، يؤدي طلب `open-tui` (مثل طلب التحدث إلى وكيل عادي) إلى الخروج من OpenClaw وفتح واجهة TUI العادية للوكيل؛ استخدم `/openclaw` هناك للعودة.

استخدم الوضع المحلي عندما تكون التهيئة الحالية صالحة بالفعل وتريد من الوكيل المضمّن فحصها على الجهاز نفسه، ومقارنتها بالوثائق، والمساعدة في إصلاح الانحراف من دون الاعتماد على Gateway قيد التشغيل.

إذا كان `openclaw config validate` يفشل بالفعل، فابدأ أولاً باستخدام `openclaw configure` أو `openclaw doctor --fix`؛ إذ لا يزال `openclaw chat` يحتاج إلى تهيئة قابلة للتحميل كي يبدأ.

المسار المعتاد:

1. ابدأ الوضع المحلي:

```bash
openclaw chat
```

2. اطلب من الوكيل ما تريد التحقق منه، على سبيل المثال:

```text
قارن تهيئة مصادقة Gateway لديّ بالوثائق واقترح أصغر إصلاح ممكن.
```

3. استخدم أوامر الصدفة المحلية للحصول على أدلة دقيقة وإجراء التحقق:

```text
!openclaw config file
!openclaw docs gateway auth token secretref
!openclaw config validate
!openclaw doctor
```

4. طبّق تغييرات محدودة باستخدام `openclaw config set` أو `openclaw configure`، ثم أعد تشغيل `!openclaw config validate`.
5. إذا أوصى Doctor بترحيل أو إصلاح تلقائي، فراجعه وشغّل `!openclaw doctor --fix`.

نصائح:

- فضّل `openclaw config set` أو `openclaw configure` على تحرير `openclaw.json` يدويًا.
- يبحث `openclaw docs "<query>"` في فهرس الوثائق المباشر من الجهاز نفسه.
- يكون `openclaw config validate --json` مفيدًا عندما تريد مخططًا منظمًا وأخطاء SecretRef/قابلية الحل.

## مخرجات الأدوات

- تظهر استدعاءات الأدوات كبطاقات تتضمن الوسائط + النتائج.
- يبدّل Ctrl+O بين العرضين المطوي والموسّع.
- أثناء تشغيل الأدوات، تتدفق التحديثات الجزئية إلى البطاقة نفسها.

## ألوان الطرفية

- تحافظ TUI على نص متن المساعد بلون المقدمة الافتراضي للطرفية، بحيث تظل الطرفيات الداكنة والفاتحة قابلة للقراءة.
- إذا كانت خلفية الطرفية فاتحة وكان الاكتشاف التلقائي خاطئًا، فاضبط `OPENCLAW_THEME=light` قبل تشغيل `openclaw tui`.
- لفرض لوحة الألوان الداكنة الأصلية بدلاً من ذلك، اضبط `OPENCLAW_THEME=dark`.

## السجل + البث

- عند الاتصال، تحمّل TUI أحدث سجل (الافتراضي 200 رسالة).
- تُحدَّث الردود المتدفقة في موضعها حتى اكتمالها.
- تستمع TUI أيضًا إلى أحداث أدوات الوكيل لعرض بطاقات أدوات أكثر ثراءً.

## تفاصيل الاتصال

- تتصل TUI باستخدام معرّف العميل `openclaw-tui` ضمن وضع العميل العام `ui` (وهو الوضع نفسه الذي تستخدمه Control UI وWebChat لسياسة Gateway).
- تُظهر عمليات إعادة الاتصال رسالة نظام؛ وتظهر فجوات الأحداث في السجل.

## الخيارات

- `--local`: التشغيل باستخدام وقت تشغيل الوكيل المضمّن المحلي
- `--url <url>`: عنوان URL لـ WebSocket الخاص بـ Gateway (الإعداد الافتراضي هو `gateway.remote.url` من الإعدادات، أو `ws://127.0.0.1:<port>` على عنوان الاسترجاع)
- `--token <token>`: رمز Gateway المميز (إذا كان مطلوبًا)
- `--password <password>`: كلمة مرور Gateway (إذا كانت مطلوبة)
- `--tls-fingerprint <sha256>`: بصمة شهادة TLS المتوقعة لـ Gateway من النوع `wss://` ذي الشهادة المثبّتة
- `--session <key>`: مفتاح الجلسة (الإعداد الافتراضي: `main`، أو `global` عندما يكون النطاق عامًا)
- `--deliver`: تسليم ردود المساعد إلى المزوّد (معطّل افتراضيًا)
- `--thinking <level>`: تجاوز مستوى التفكير لعمليات الإرسال
- `--message <text>`: إرسال رسالة أولية بعد الاتصال
- `--timeout-ms <ms>`: مهلة الوكيل بالمللي ثانية (الإعداد الافتراضي هو `agents.defaults.timeoutSeconds`)
- `--history-limit <n>`: إدخالات السجل المطلوب تحميلها (الإعداد الافتراضي `200`)

<Warning>
عند تعيين `--url`، لا يعود TUI إلى بيانات الاعتماد الموجودة في الإعدادات أو متغيرات البيئة. مرّر `--token` أو `--password` صراحةً، بالإضافة إلى `--tls-fingerprint` عندما تستخدم الوجهة شهادة مثبّتة. يُعد غياب بيانات الاعتماد الصريحة خطأً. في الوضع المحلي، لا تمرّر `--url` أو `--token` أو `--password` أو `--tls-fingerprint`.
</Warning>

## استكشاف الأخطاء وإصلاحها

لا يظهر أي إخراج بعد إرسال رسالة:

- شغّل `/status` في TUI للتأكد من أن Gateway متصل وفي حالة خمول/انشغال.
- تحقق من سجلات Gateway: `openclaw logs --follow`.
- تأكد من إمكانية تشغيل الوكيل: `openclaw status` و`openclaw models status`.
- إذا كنت تتوقع ظهور الرسائل في قناة دردشة، فتأكد من بدء TUI باستخدام `--deliver` (لا يمكن تفعيل ذلك لاحقًا من دون إعادة التشغيل).

## استكشاف مشكلات الاتصال وإصلاحها

- `disconnected`: تأكد من تشغيل Gateway ومن صحة `--url/--token/--password`.
- لا توجد وكلاء في أداة الاختيار: تحقق من `openclaw agents list` وإعدادات التوجيه.
- أداة اختيار الجلسات فارغة: قد تكون في النطاق العام أو لم تُنشأ أي جلسات بعد.
