## البدء السريع

### وضع Gateway

1. شغّل Gateway.

```bash
openclaw gateway
```

2. افتح TUI.

```bash
openclaw tui
```

3. اكتب رسالة واضغط Enter.

Gateway بعيد:

```bash
openclaw tui --url ws://<host>:<port> --token <gateway-token>
```

استخدم `--password` إذا كان Gateway لديك يستخدم مصادقة بكلمة مرور.

### الوضع المحلي

شغّل TUI من دون Gateway:

```bash
openclaw chat
# أو
openclaw tui --local
```

- يُعدّ `openclaw chat` و`openclaw terminal` اسمين مستعارين لـ `openclaw tui --local`.
- لا يمكن دمج `--local` مع `--url` أو `--token` أو `--password`.
- يستخدم الوضع المحلي بيئة تشغيل الوكيل المضمّنة مباشرةً. تعمل معظم الأدوات المحلية، لكن الميزات التي تتطلب Gateway غير متاحة.
- يختار الأمر `openclaw` وحده (من دون أمر فرعي) هدفًا تلقائيًا: يشغّل التثبيت غير المُعَدّ عملية إعداد الاستدلال؛ ويفتح الإعداد غير الصالح إرشادات Doctor التقليدية؛ ويفتح Gateway المُعَدّ والقابل للوصول غلاف TUI هذا في وضع Gateway؛ وإلا، فيفتح النموذج المحلي المُعَدّ في الوضع المحلي.

## ما تراه

- الرأس: عنوان URL للاتصال، والوكيل الحالي، والجلسة الحالية.
- سجل المحادثة: رسائل المستخدم، وردود المساعد، وإشعارات النظام، وبطاقات الأدوات.
- سطر الحالة: حالة الاتصال/التشغيل (جارٍ الاتصال، قيد التشغيل، جارٍ البث، خامل، خطأ).
- التذييل: الوكيل + الجلسة + النموذج + حالة الهدف + التفكير/السرعة/الإسهاب/التتبّع/الاستدلال + أعداد الرموز + التسليم. عند تمكين `tui.footer.showRemoteHost`، تعرض اتصالات Gateway البعيدة أيضًا مضيف الاتصال.
- الإدخال: محرر نصوص مزوّد بالإكمال التلقائي.

## النموذج الذهني: الوكلاء + الجلسات

- الوكلاء معرّفات نصية فريدة (مثل `main` و`research`). يعرض Gateway القائمة.
- تنتمي الجلسات إلى الوكيل الحالي.
- تُخزَّن مفاتيح الجلسات بالصيغة `agent:<agentId>:<sessionKey>`.
  - إذا كتبت `/session main`، يوسّعها TUI إلى `agent:<currentAgent>:main`.
  - إذا كتبت `/session agent:other:main`، تنتقل صراحةً إلى جلسة ذلك الوكيل.
- نطاق الجلسة:
  - `per-sender` (الافتراضي): لكل وكيل جلسات متعددة.
  - `global`: يستخدم TUI دائمًا جلسة `global` (وقد تكون أداة الاختيار فارغة).
- يظهر الوكيل الحالي + الجلسة دائمًا في التذييل.
- لإظهار مضيف Gateway للاتصالات غير المحلية المستندة إلى عنوان URL، فعّل ذلك باستخدام:

  ```bash
  openclaw config set tui.footer.showRemoteHost true
  ```

  القيمة الافتراضية هي `false`. لا تعرض اتصالات local loopback والاتصالات المحلية المضمّنة تسمية للمضيف مطلقًا.

- إذا كانت للجلسة [غاية](/ar/tools/goal)، يعرض التذييل حالتها المختصرة:
  `قيد السعي إلى الغاية` أو `الغاية متوقفة مؤقتًا (/goal resume)` أو `الغاية محظورة (/goal resume)` أو `تحققت الغاية`.
- عند البدء من دون `--session`، يستأنف TUI في وضع Gateway آخر جلسة محددة لنفس Gateway والوكيل ونطاق الجلسة إذا كانت تلك الجلسة لا تزال موجودة. يظل تمرير `--session` أو استخدام `/session` أو `/new` أو `/reset` إجراءً صريحًا.

## الإرسال + التسليم

- تذهب الرسائل دائمًا إلى Gateway (أو إلى بيئة التشغيل المضمّنة في الوضع المحلي)؛ أما تسليم رد المساعد مجددًا إلى موفّر محادثة فهو خطوة منفصلة ومعطّلة افتراضيًا.
- يُعدّ TUI سطح مصدر داخليًا مثل WebChat، وليس قناة صادرة عامة. يمكن لأُطر التشغيل التي تتطلب `tools.message` للردود المرئية تلبية دورة TUI النشطة باستخدام `message.send` من دون هدف؛ أما التسليم الصريح إلى الموفّر فيستمر باستخدام القنوات المُعَدّة المعتادة ولا يعود مطلقًا إلى `lastChannel`.
- يُثبَّت إعداد التسليم طوال جلسة TUI عند التشغيل: ابدأ باستخدام `openclaw tui --deliver` لتفعيله. لا يوجد أمر شرطة مائلة `/deliver` أو مفتاح تبديل في الإعدادات لتغييره في منتصف الجلسة؛ أعد تشغيل TUI لتغييره.

## أدوات الاختيار + الطبقات المتراكبة

- أداة اختيار النموذج: تسرد النماذج المتاحة وتعيّن التجاوز الخاص بالجلسة.
- أداة اختيار الوكيل: اختر وكيلًا مختلفًا.
- أداة اختيار الجلسة: تعرض ما يصل إلى 50 جلسة للوكيل الحالي جرى تحديثها خلال آخر 7 أيام. استخدم `/session <key>` للانتقال إلى جلسة أقدم معروفة.
- الإعدادات (`/settings`): بدّل توسيع مخرجات الأدوات وإظهار التفكير. لا تتحكم هذه اللوحة في التسليم.

## اختصارات لوحة المفاتيح

- Enter: إرسال الرسالة
- Esc: إيقاف التشغيل النشط
- Ctrl+C: مسح الإدخال (اضغط مرتين للخروج)
- Ctrl+D: خروج
- Ctrl+L: أداة اختيار النموذج
- Ctrl+G: أداة اختيار الوكيل
- Ctrl+P: أداة اختيار الجلسة
- Ctrl+O: تبديل توسيع مخرجات الأدوات
- Ctrl+T: تبديل إظهار التفكير (يعيد تحميل السجل)

## أوامر الشرطة المائلة

الأساسية:

- `/help`
- `/status` (يُمرَّر إلى Gateway؛ يعرض ملخص الجلسة/النموذج)
- `/gateway-status` (الاسم المستعار `/gwstatus`؛ يعرض حالة اتصال Gateway مباشرةً)
- `/agent <id>` (أو `/agents`)
- `/session <key>` (أو `/sessions`)
- `/model <provider/model>` (أو `/models`)

عناصر التحكم في الجلسة:

- `/think <off|minimal|low|medium|high>` (قد تضيف المستويات الأعلى درجات مثل `xhigh`/`max` وفقًا للنموذج)
- `/fast <status|auto|on|off>`
- `/verbose <on|full|off>`
- `/trace <on|off>`
- `/reasoning <on|off|stream>`
- `/usage <off|tokens|full|reset>` (تمسح `reset`/`inherit`/`clear`/`default` تجاوز الجلسة)
- `/goal [status] | /goal start <objective> | /goal edit <objective> | /goal pause|resume|complete|block|clear`
- `/elevated <on|off|ask|full>` (الاسم المستعار: `/elev`)
- `/activation <mention|always>`

دورة حياة الجلسة:

- `/new` (إنشاء جلسة جديدة ومعزولة باستخدام مفتاح جديد؛ لا يؤثر في عملاء TUI الآخرين في الجلسة القديمة)
- `/reset` (إعادة ضبط مفتاح الجلسة الحالية في موضعه)
- `/abort` (إيقاف التشغيل النشط)
- `/settings`
- `/exit` (أو `/quit`)

الوضع المحلي فقط:

- يفتح `/auth [provider]` تدفق مصادقة/تسجيل دخول الموفّر داخل TUI.

Crestodian:

- يعيدك `/crestodian [request]` من TUI العادي للوكيل إلى محادثة [Crestodian](#crestodian-setup-and-repair-helper) للإعداد/الإصلاح، مع إعادة توجيه طلب واحد اختياريًا.

تُمرَّر أوامر الشرطة المائلة الأخرى الخاصة بـ Gateway (مثل `/context`) إلى Gateway وتظهر كمخرجات للنظام. راجع [أوامر الشرطة المائلة](/ar/tools/slash-commands).

## أوامر الصدفة المحلية

- ابدأ السطر بالرمز `!` لتشغيل أمر صدفة محلي على مضيف TUI.
- يطلب TUI الإذن مرة واحدة لكل جلسة للسماح بالتنفيذ المحلي؛ ويؤدي الرفض إلى إبقاء `!` معطّلًا طوال الجلسة.
- تعمل الأوامر في صدفة جديدة وغير تفاعلية داخل دليل عمل TUI (من دون استمرار `cd`/البيئة).
- تتلقى أوامر الصدفة المحلية `OPENCLAW_SHELL=tui-local` في بيئتها.
- يُرسل الرمز `!` منفردًا كرسالة عادية؛ ولا تؤدي المسافات البادئة إلى تشغيل التنفيذ المحلي.

## مساعد Crestodian للإعداد والإصلاح

Crestodian هو مساعد الإعداد/الإصلاح من الحلقة صفر، ويُتاح باسم `openclaw crestodian` بعد اجتياز النموذج الافتراضي المُعَدّ فحص استدلال مباشر. إذا لم يكن الاستدلال متاحًا، تعود عملية التشغيل التفاعلية إلى إعداد الاستدلال وتفشل الأتمتة مع إرشادات الإصلاح. يعمل داخل غلاف TUI المحلي نفسه الذي يستخدمه `openclaw tui --local`، ويدعمه وكيل ذكاء اصطناعي مقيّد بعمليات Crestodian ذات الأنواع المحددة والخاضعة للموافقة:

```bash
openclaw crestodian                       # البدء تفاعليًا
openclaw crestodian -m "status"           # تشغيل طلب واحد ثم الخروج
openclaw crestodian -m "set default model openai/gpt-5.2" --yes   # تطبيق كتابة في الإعداد
```

- تتطلب عمليات الكتابة الدائمة في الإعداد موافقة: أكّد تفاعليًا أو مرّر `--yes`.
- يطبع `--json` نظرة عامة على بدء التشغيل بتنسيق JSON بدلًا من بدء المحادثة.
- من داخل Crestodian، يؤدي طلب `open-tui` (مثل طلب التحدث إلى وكيل عادي) إلى الخروج من Crestodian وفتح TUI العادي للوكيل؛ استخدم `/crestodian` هناك للعودة.

استخدم الوضع المحلي عندما يكون الإعداد الحالي صالحًا بالفعل وتريد أن يفحصه الوكيل المضمّن على الجهاز نفسه، ويقارنه بالوثائق، ويساعد في إصلاح الانحراف من دون الاعتماد على Gateway قيد التشغيل.

إذا كان `openclaw config validate` يفشل بالفعل، فابدأ أولًا باستخدام `openclaw configure` أو `openclaw doctor --fix`؛ إذ لا يزال `openclaw chat` يحتاج إلى إعداد قابل للتحميل كي يبدأ.

الدورة المعتادة:

1. ابدأ الوضع المحلي:

```bash
openclaw chat
```

2. اطلب من الوكيل ما تريد فحصه، مثلًا:

```text
قارن إعداد مصادقة Gateway لدي بالوثائق واقترح أصغر إصلاح.
```

3. استخدم أوامر الصدفة المحلية للحصول على أدلة دقيقة والتحقق:

```text
!openclaw config file
!openclaw docs gateway auth token secretref
!openclaw config validate
!openclaw doctor
```

4. طبّق تغييرات محدودة باستخدام `openclaw config set` أو `openclaw configure`، ثم أعد تشغيل `!openclaw config validate`.
5. إذا أوصى Doctor بترحيل أو إصلاح تلقائي، فراجعه وشغّل `!openclaw doctor --fix`.

نصائح:

- فضّل `openclaw config set` أو `openclaw configure` على تحرير `openclaw.json` يدويًا.
- يبحث `openclaw docs "<query>"` في فهرس الوثائق المباشر من الجهاز نفسه.
- يفيد `openclaw config validate --json` عندما تريد مخططًا منظمًا وأخطاء SecretRef/قابلية الحل.

## مخرجات الأدوات

- تظهر استدعاءات الأدوات كبطاقات تتضمن الوسائط + النتائج.
- يبدّل Ctrl+O بين العرضين المطوي والموسّع.
- أثناء تشغيل الأدوات، تتدفق التحديثات الجزئية إلى البطاقة نفسها.

## ألوان الطرفية

- يُبقي TUI نص جسم المساعد بلون المقدمة الافتراضي في طرفيتك لكي تظل الطرفيات الداكنة والفاتحة قابلة للقراءة.
- إذا كانت طرفيتك تستخدم خلفية فاتحة وكان الاكتشاف التلقائي خاطئًا، فعيّن `OPENCLAW_THEME=light` قبل تشغيل `openclaw tui`.
- لفرض لوحة الألوان الداكنة الأصلية بدلًا من ذلك، عيّن `OPENCLAW_THEME=dark`.

## السجل + البث

- عند الاتصال، يحمّل TUI أحدث سجل (الافتراضي 200 رسالة).
- تُحدَّث الردود المتدفقة في موضعها حتى اكتمالها.
- يستمع TUI أيضًا إلى أحداث أدوات الوكيل لعرض بطاقات أدوات أكثر ثراءً.

## تفاصيل الاتصال

- يتصل TUI باستخدام معرّف العميل `openclaw-tui` ضمن وضع العميل العام `ui` (وهو الوضع نفسه الذي تستخدمه Control UI وWebChat لسياسة Gateway).
- تعرض عمليات إعادة الاتصال رسالة نظام؛ وتظهر فجوات الأحداث في السجل.

## الخيارات

- `--local`: التشغيل باستخدام بيئة تشغيل الوكيل المحلي المضمّن
- `--url <url>`: عنوان URL لـ WebSocket الخاص بـ Gateway (الافتراضي هو `gateway.remote.url` من الإعداد، أو `ws://127.0.0.1:<port>` على local loopback)
- `--token <token>`: رمز Gateway (إذا كان مطلوبًا)
- `--password <password>`: كلمة مرور Gateway (إذا كانت مطلوبة)
- `--tls-fingerprint <sha256>`: بصمة شهادة TLS المتوقعة لـ Gateway يستخدم `wss://` مع تثبيت الشهادة
- `--session <key>`: مفتاح الجلسة (الافتراضي: `main`، أو `global` عندما يكون النطاق عامًا)
- `--deliver`: تسليم ردود المساعد إلى الموفّر (معطّل افتراضيًا)
- `--thinking <level>`: تجاوز مستوى التفكير للإرسالات
- `--message <text>`: إرسال رسالة أولية بعد الاتصال
- `--timeout-ms <ms>`: مهلة الوكيل بالمللي ثانية (الافتراضي هو `agents.defaults.timeoutSeconds`)
- `--history-limit <n>`: عدد إدخالات السجل المطلوب تحميلها (الافتراضي `200`)

<Warning>
عند تعيين `--url`، لا يعود TUI إلى بيانات الاعتماد الموجودة في الإعداد أو البيئة. مرّر `--token` أو `--password` صراحةً، بالإضافة إلى `--tls-fingerprint` عندما يستخدم الهدف شهادة مثبّتة. يُعد غياب بيانات الاعتماد الصريحة خطأً. في الوضع المحلي، لا تمرّر `--url` أو `--token` أو `--password` أو `--tls-fingerprint`.
</Warning>

## استكشاف الأخطاء وإصلاحها

لا توجد مخرجات بعد إرسال رسالة:

- شغّل `/status` في TUI للتأكد من أن Gateway متصل وخامل/مشغول.
- افحص سجلات Gateway: `openclaw logs --follow`.
- تأكد من قدرة الوكيل على العمل: `openclaw status` و`openclaw models status`.
- إذا كنت تتوقع رسائل في قناة محادثة، فتأكد من أن TUI بدأ باستخدام `--deliver` (لا يمكن تفعيل ذلك لاحقًا من دون إعادة التشغيل).

## استكشاف أخطاء الاتصال وإصلاحها

- `disconnected`: تأكد من أن Gateway قيد التشغيل وأن `--url/--token/--password` صحيحة.
- لا توجد وكلاء في أداة الاختيار: افحص `openclaw agents list` وإعداد التوجيه لديك.
- أداة اختيار الجلسة فارغة: قد تكون في النطاق العام أو لا تكون لديك جلسات بعد.
