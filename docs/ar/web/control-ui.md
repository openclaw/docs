واجهة التحكم هي تطبيق صفحة واحدة صغير مبني بـ **Vite + Lit** يقدمه الـ Gateway:

- الافتراضي: `http://<host>:18789/`
- بادئة اختيارية: عيّن `gateway.controlUi.basePath` (مثل `/openclaw`)

تتواصل **مباشرة مع WebSocket الخاص بالـ Gateway** على المنفذ نفسه.

## فتح سريع (محلي)

إذا كان الـ Gateway يعمل على الكمبيوتر نفسه، فافتح:

- [http://127.0.0.1:18789/](http://127.0.0.1:18789/) (أو [http://localhost:18789/](http://localhost:18789/))

إذا فشل تحميل الصفحة، فابدأ الـ Gateway أولاً: `openclaw gateway`.

تُقدَّم المصادقة أثناء مصافحة WebSocket عبر:

- `connect.params.auth.token`
- `connect.params.auth.password`
- ترويسات هوية Tailscale Serve عندما تكون `gateway.auth.allowTailscale: true`
- ترويسات هوية الوكيل الموثوق عندما تكون `gateway.auth.mode: "trusted-proxy"`

تحتفظ لوحة إعدادات لوحة المعلومات برمز مميز لجلسة تبويب المتصفح الحالية وعنوان gateway المحدد؛ ولا تُحفَظ كلمات المرور. عادةً ما ينشئ الإعداد الأولي رمز gateway للمصادقة عبر السر المشترك عند أول اتصال، لكن مصادقة كلمة المرور تعمل أيضاً عندما تكون `gateway.auth.mode` هي `"password"`.

## إقران الجهاز (الاتصال الأول)

عند الاتصال بواجهة التحكم من متصفح أو جهاز جديد، يتطلب الـ Gateway عادةً **موافقة إقران لمرة واحدة**. هذا إجراء أمني لمنع الوصول غير المصرح به.

**ما ستراه:** "disconnected (1008): pairing required"

<Steps>
  <Step title="عرض الطلبات المعلّقة">
    ```bash
    openclaw devices list
    ```
  </Step>
  <Step title="الموافقة باستخدام معرّف الطلب">
    ```bash
    openclaw devices approve <requestId>
    ```
  </Step>
</Steps>

إذا أعاد المتصفح محاولة الإقران بتفاصيل مصادقة متغيرة (الدور/النطاقات/المفتاح العام)، فيُستبدل الطلب المعلّق السابق ويُنشأ `requestId` جديد. أعد تشغيل `openclaw devices list` قبل الموافقة.

إذا كان المتصفح مقترناً بالفعل وغيّرته من وصول قراءة إلى وصول كتابة/مسؤول، فيُعامل ذلك كترقية موافقة، وليس إعادة اتصال صامتة. يبقي OpenClaw الموافقة القديمة نشطة، ويحظر إعادة الاتصال ذات النطاق الأوسع، ويطلب منك الموافقة صراحةً على مجموعة النطاقات الجديدة.

بعد الموافقة، يُتذكَّر الجهاز ولن يتطلب إعادة موافقة إلا إذا ألغيت تفويضه باستخدام `openclaw devices revoke --device <id> --role <role>`. راجع [CLI الأجهزة](/ar/cli/devices) لتدوير الرموز وإلغاء التفويض.

<Note>
- تتم الموافقة تلقائياً على اتصالات متصفح local loopback المباشرة (`127.0.0.1` / `localhost`).
- يمكن لـ Tailscale Serve تخطي جولة الإقران لجلسات مشغّل واجهة التحكم عندما تكون `gateway.auth.allowTailscale: true`، وتتحقق هوية Tailscale، ويقدم المتصفح هوية جهازه.
- لا تزال ارتباطات Tailnet المباشرة، واتصالات متصفح LAN، وملفات تعريف المتصفح التي لا تحتوي على هوية جهاز تتطلب موافقة صريحة.
- ينشئ كل ملف تعريف متصفح معرّف جهاز فريداً، لذا سيتطلب تبديل المتصفحات أو مسح بيانات المتصفح إعادة الإقران.

</Note>

## الهوية الشخصية (محلية للمتصفح)

تدعم واجهة التحكم هوية شخصية لكل متصفح (اسم عرض وصورة رمزية) تُرفق بالرسائل الصادرة للإسناد في الجلسات المشتركة. تعيش هذه الهوية في تخزين المتصفح، وتكون محصورة في ملف تعريف المتصفح الحالي، ولا تتم مزامنتها إلى أجهزة أخرى أو حفظها على جانب الخادم خارج بيانات تعريف مؤلف النص المعتادة للرسائل التي ترسلها فعلياً. يؤدي مسح بيانات الموقع أو تبديل المتصفحات إلى إعادة ضبطها لتصبح فارغة.

ينطبق النمط المحلي للمتصفح نفسه على تجاوز صورة المساعد الرمزية. تُركّب صور المساعد الرمزية المرفوعة فوق الهوية التي يحلها الـ gateway في المتصفح المحلي فقط ولا تمر أبداً ذهاباً وإياباً عبر `config.patch`. لا يزال حقل الإعداد المشترك `ui.assistant.avatar` متاحاً للعملاء غير واجهة المستخدم الذين يكتبون الحقل مباشرةً (مثل gateways المبرمجة أو لوحات المعلومات المخصصة).

## نقطة نهاية إعدادات وقت التشغيل

تجلب واجهة التحكم إعدادات وقت التشغيل الخاصة بها من `/__openclaw/control-ui-config.json`. هذه النقطة محمية بمصادقة gateway نفسها مثل بقية سطح HTTP: لا تستطيع المتصفحات غير المصادق عليها جلبها، ويتطلب الجلب الناجح إما رمز/كلمة مرور gateway صالحاً بالفعل، أو هوية Tailscale Serve، أو هوية وكيل موثوق.

## دعم اللغة

يمكن لواجهة التحكم توطين نفسها عند أول تحميل بناءً على لغة متصفحك. لتجاوز ذلك لاحقاً، افتح **نظرة عامة -> وصول Gateway -> اللغة**. يوجد منتقي اللغة في بطاقة وصول Gateway، وليس ضمن المظهر.

- اللغات المدعومة: `en`, `zh-CN`, `zh-TW`, `pt-BR`, `de`, `es`, `ja-JP`, `ko`, `fr`, `ar`, `it`, `tr`, `uk`, `id`, `pl`, `th`, `vi`, `nl`, `fa`
- تُحمّل الترجمات غير الإنجليزية كسولياً في المتصفح.
- تُحفظ اللغة المحددة في تخزين المتصفح وتُعاد استخدامها في الزيارات المستقبلية.
- تعود مفاتيح الترجمة المفقودة إلى الإنجليزية.

تُولّد ترجمات المستندات لمجموعة اللغات غير الإنجليزية نفسها، لكن منتقي اللغة المدمج في موقع المستندات من Mintlify محدود برموز اللغات التي يقبلها Mintlify. لا تزال مستندات التايلاندية (`th`) والفارسية (`fa`) تُولّد في مستودع النشر؛ وقد لا تظهر في ذلك المنتقي حتى يدعم Mintlify هذه الرموز.

## سمات المظهر

تحتفظ لوحة المظهر بسمات Claw وKnot وDash المدمجة، بالإضافة إلى خانة استيراد tweakcn واحدة محلية للمتصفح. لاستيراد سمة، افتح [سمات tweakcn](https://tweakcn.com/themes)، واختر سمة أو أنشئها، وانقر **مشاركة**، ثم الصق رابط السمة المنسوخ في المظهر. يقبل المستورد أيضاً عناوين URL لسجل `https://tweakcn.com/r/themes/<id>`، وعناوين URL للمحرر مثل `https://tweakcn.com/editor/theme?theme=amethyst-haze`، ومسارات `/themes/<id>` النسبية، ومعرّفات السمات الخام، وأسماء السمات الافتراضية مثل `amethyst-haze`.

تُخزّن السمات المستوردة في ملف تعريف المتصفح الحالي فقط. لا تُكتب إلى إعدادات gateway ولا تتم مزامنتها عبر الأجهزة. يؤدي استبدال السمة المستوردة إلى تحديث الخانة المحلية الواحدة؛ ويؤدي مسحها إلى إعادة السمة النشطة إلى Claw إذا كانت السمة المستوردة محددة.

## سلوك الدردشة

<AccordionGroup>
  <Accordion title="دلالات الإرسال والسجل">
    - `chat.send` **غير حاجب**: يقر فورا بـ `{ runId, status: "started" }` ويتدفق الرد عبر أحداث `chat`.
    - تقبل تحميلات الدردشة الصور إضافة إلى الملفات غير المرئية. تحتفظ الصور بمسار الصورة الأصلي؛ وتخزن الملفات الأخرى كوسائط مدارة وتظهر في السجل كروابط مرفقات.
    - إعادة الإرسال باستخدام `idempotencyKey` نفسه ترجع `{ status: "in_flight" }` أثناء التشغيل، و`{ status: "ok" }` بعد الاكتمال.
    - تكون ردود `chat.history` محدودة الحجم لحماية واجهة المستخدم. عندما تكون إدخالات النص المنسوخ كبيرة جدا، قد يقتطع Gateway حقول النص الطويلة، ويحذف كتل البيانات الوصفية الثقيلة، ويستبدل الرسائل كبيرة الحجم بعنصر نائب (`[chat.history omitted: message too large]`).
    - تستمر الصور التي ينتجها المساعد/النظام كإشارات وسائط مدارة وتقدم مرة أخرى عبر عناوين URL لوسائط Gateway المصادق عليها، لذلك لا تعتمد عمليات إعادة التحميل على بقاء حمولات الصور الخام بصيغة base64 في رد سجل الدردشة.
    - يزيل `chat.history` أيضا وسوم التوجيه المضمنة المخصصة للعرض فقط من نص المساعد المرئي (مثل `[[reply_to_*]]` و`[[audio_as_voice]]`)، وحمولات XML لاستدعاءات الأدوات بنص عادي (بما في ذلك `<tool_call>...</tool_call>`، و`<function_call>...</function_call>`، و`<tool_calls>...</tool_calls>`، و`<function_calls>...</function_calls>`، وكتل استدعاء الأدوات المقتطعة)، ورموز تحكم النموذج المسربة بصيغة ASCII/العرض الكامل، ويحذف إدخالات المساعد التي يكون نصها المرئي بالكامل هو الرمز الصامت الدقيق فقط `NO_REPLY` / `no_reply`.
    - أثناء إرسال نشط وتحديث السجل النهائي، يحافظ عرض الدردشة على ظهور رسائل المستخدم/المساعد المحلية التفاؤلية إذا أرجع `chat.history` لقطة أقدم لفترة وجيزة؛ ويستبدل النص المنسوخ القانوني تلك الرسائل المحلية بمجرد أن يلحق سجل Gateway.
    - يضيف `chat.inject` ملاحظة مساعد إلى نص الجلسة المنسوخ ويبث حدث `chat` لتحديثات واجهة المستخدم فقط (بلا تشغيل وكيل، ولا تسليم قناة).
    - تقوم منتقيات نموذج رأس الدردشة والتفكير بتعديل الجلسة النشطة فورا عبر `sessions.patch`؛ وهي تجاوزات جلسة مستمرة، وليست خيارات إرسال لدورة واحدة فقط.
    - تؤدي كتابة `/new` في واجهة التحكم إلى إنشاء جلسة لوحة معلومات جديدة مماثلة لـ New Chat والتبديل إليها. تحافظ كتابة `/reset` على إعادة الضبط الصريحة داخل المكان في Gateway للجلسة الحالية.
    - يطلب منتقي نموذج الدردشة عرض النموذج المكون في Gateway. إذا كان `agents.defaults.models` موجودا، فإن قائمة السماح هذه تقود المنتقي. وإلا يعرض المنتقي إدخالات `models.providers.*.models` الصريحة إضافة إلى المزودين ذوي المصادقة القابلة للاستخدام. يبقى الفهرس الكامل متاحا عبر RPC التصحيحي `models.list` مع `view: "all"`.
    - عندما تعرض تقارير استخدام جلسة Gateway الحديثة ضغط سياق مرتفعا، تعرض منطقة مؤلف الدردشة إشعار سياق، وعند مستويات Compaction الموصى بها، زرا مدمجا يشغل مسار Compaction العادي للجلسة. تخفى لقطات الرموز القديمة حتى يبلغ Gateway عن استخدام حديث مرة أخرى.

  </Accordion>
  <Accordion title="وضع التحدث (زمن حقيقي في المتصفح)">
    يستخدم وضع التحدث مزود صوت زمن حقيقي مسجلا. اضبط OpenAI باستخدام `talk.provider: "openai"` مع `talk.providers.openai.apiKey`، أو اضبط Google باستخدام `talk.provider: "google"` مع `talk.providers.google.apiKey`؛ ولا يزال من الممكن إعادة استخدام إعداد مزود الزمن الحقيقي Voice Call كاحتياطي. لا يتلقى المتصفح أبدا مفتاح API قياسيا للمزود. يتلقى OpenAI سرا مؤقتا لعميل Realtime من أجل WebRTC. يتلقى Google Live رمز مصادقة Live API مقيدا صالحا لاستخدام واحد لجلسة WebSocket في المتصفح، مع تعليمات وتصريحات أدوات مقفلة داخل الرمز بواسطة Gateway. المزودون الذين لا يعرضون إلا جسرا خلفيا للزمن الحقيقي يعملون عبر نقل ترحيل Gateway، بحيث تبقى بيانات الاعتماد ومآخذ المورد على جانب الخادم بينما ينتقل صوت المتصفح عبر RPCs مصادق عليها في Gateway. تجمع مطالبة جلسة Realtime بواسطة Gateway؛ ولا يقبل `talk.realtime.session` تجاوزات تعليمات مقدمة من المستدعي.

    في مؤلف الدردشة، يكون عنصر تحكم التحدث هو زر الموجات بجوار زر إملاء الميكروفون. عند بدء التحدث، يعرض صف حالة المؤلف `Connecting Talk...`، ثم `Talk live` أثناء اتصال الصوت، أو `Asking OpenClaw...` أثناء استشارة استدعاء أداة زمن حقيقي للنموذج الأكبر المكون عبر `chat.send`.

    فحص دخان مباشر للمشرف: يتحقق `OPENAI_API_KEY=... GEMINI_API_KEY=... node --import tsx scripts/dev/realtime-talk-live-smoke.ts` من تبادل SDP لـ OpenAI WebRTC في المتصفح، وإعداد WebSocket في المتصفح برمز Google Live مقيد، ومحول متصفح ترحيل Gateway مع وسائط ميكروفون مزيفة. يطبع الأمر حالة المزود فقط ولا يسجل الأسرار.

  </Accordion>
  <Accordion title="الإيقاف والإجهاض">
    - انقر **إيقاف** (يستدعي `chat.abort`).
    - أثناء نشاط تشغيل، تصطف المتابعات العادية في الطابور. انقر **توجيه** على رسالة مصطفة لحقن تلك المتابعة في الدورة الجارية.
    - اكتب `/stop` (أو عبارات إجهاض مستقلة مثل `stop`، و`stop action`، و`stop run`، و`stop openclaw`، و`please stop`) للإجهاض خارج النطاق.
    - يدعم `chat.abort` الصيغة `{ sessionKey }` (من دون `runId`) لإجهاض كل عمليات التشغيل النشطة لتلك الجلسة.

  </Accordion>
  <Accordion title="الاحتفاظ الجزئي عند الإجهاض">
    - عند إجهاض تشغيل، لا يزال من الممكن عرض نص مساعد جزئي في واجهة المستخدم.
    - يستمر Gateway في حفظ نص المساعد الجزئي المجهض في سجل النص المنسوخ عند وجود مخرجات مخزنة مؤقتا.
    - تتضمن الإدخالات المستمرة بيانات وصفية للإجهاض حتى يتمكن مستهلكو النص المنسوخ من تمييز الأجزاء المجهضة عن مخرجات الاكتمال العادية.

  </Accordion>
</AccordionGroup>

## تثبيت PWA والدفع عبر الويب

تتضمن واجهة التحكم `manifest.webmanifest` وعامل خدمة، لذلك تستطيع المتصفحات الحديثة تثبيتها كتطبيق PWA مستقل. يتيح Web Push لـ Gateway إيقاظ PWA المثبت بالإشعارات حتى عندما لا يكون اللسان أو نافذة المتصفح مفتوحا.

| السطح                                                | ما يفعله                                                           |
| ----------------------------------------------------- | ------------------------------------------------------------------ |
| `ui/public/manifest.webmanifest`                      | بيان PWA. تعرض المتصفحات "Install app" بمجرد أن يصبح قابلا للوصول. |
| `ui/public/sw.js`                                     | عامل خدمة يتعامل مع أحداث `push` ونقرات الإشعارات.                 |
| `push/vapid-keys.json` (ضمن دليل حالة OpenClaw)       | زوج مفاتيح VAPID مولد تلقائيا يستخدم لتوقيع حمولات Web Push.       |
| `push/web-push-subscriptions.json`                    | نقاط نهاية اشتراكات المتصفح المستمرة.                              |

تجاوز زوج مفاتيح VAPID عبر متغيرات البيئة في عملية Gateway عندما تريد تثبيت المفاتيح (لعمليات النشر متعددة المضيفين، أو تدوير الأسرار، أو الاختبارات):

- `OPENCLAW_VAPID_PUBLIC_KEY`
- `OPENCLAW_VAPID_PRIVATE_KEY`
- `OPENCLAW_VAPID_SUBJECT` (القيمة الافتراضية `mailto:openclaw@localhost`)

تستخدم واجهة التحكم طرق Gateway المقيدة بالنطاق هذه لتسجيل اشتراكات المتصفح واختبارها:

- `push.web.vapidPublicKey` — يجلب مفتاح VAPID العام النشط.
- `push.web.subscribe` — يسجل `endpoint` مع `keys.p256dh`/`keys.auth`.
- `push.web.unsubscribe` — يزيل نقطة نهاية مسجلة.
- `push.web.test` — يرسل إشعار اختبار إلى اشتراك المستدعي.

<Note>
Web Push مستقل عن مسار ترحيل iOS APNS (راجع [الإعداد](/ar/gateway/configuration) للدفع المدعوم بالترحيل) وعن طريقة `push.test` الحالية، التي تستهدف إقران الهاتف المحمول الأصلي.
</Note>

## التضمينات المستضافة

يمكن لرسائل المساعد عرض محتوى ويب مستضاف مضمن باستخدام الرمز المختصر `[embed ...]`. تتحكم `gateway.controlUi.embedSandbox` في سياسة صندوق حماية iframe:

<Tabs>
  <Tab title="صارم">
    يعطل تنفيذ البرامج النصية داخل التضمينات المستضافة.
  </Tab>
  <Tab title="البرامج النصية (افتراضي)">
    يسمح بالتضمينات التفاعلية مع الحفاظ على عزل الأصل؛ وهذا هو الوضع الافتراضي وعادة ما يكون كافيا لألعاب/ودجات المتصفح المستقلة.
  </Tab>
  <Tab title="موثوق">
    يضيف `allow-same-origin` فوق `allow-scripts` للمستندات من الموقع نفسه التي تحتاج عمدا إلى صلاحيات أقوى.
  </Tab>
</Tabs>

مثال:

```json5
{
  gateway: {
    controlUi: {
      embedSandbox: "scripts",
    },
  },
}
```

<Warning>
استخدم `trusted` فقط عندما يحتاج المستند المضمن فعلا إلى سلوك نفس الأصل. بالنسبة إلى معظم الألعاب واللوحات التفاعلية التي ينتجها الوكيل، يكون `scripts` هو الخيار الأكثر أمانا.
</Warning>

تبقى عناوين URL الخارجية المطلقة لتضمين `http(s)` محظورة افتراضيا. إذا كنت تريد عمدا تحميل صفحات تابعة لجهات خارجية باستخدام `[embed url="https://..."]`، فاضبط `gateway.controlUi.allowExternalEmbedUrls: true`.

## عرض رسالة الدردشة

تستخدم رسائل الدردشة المجمعة حدا أقصى افتراضيا للعرض يسهل قراءته. يمكن لعمليات النشر على الشاشات العريضة تجاوزه من دون تعديل CSS المضمنة عن طريق ضبط `gateway.controlUi.chatMessageMaxWidth`:

```json5
{
  gateway: {
    controlUi: {
      chatMessageMaxWidth: "min(1280px, 82%)",
    },
  },
}
```

تتحقق القيمة قبل أن تصل إلى المتصفح. تشمل القيم المدعومة الأطوال والنسب المئوية البسيطة مثل `960px` أو `82%`، إضافة إلى تعبيرات عرض مقيدة مثل `min(...)`، و`max(...)`، و`clamp(...)`، و`calc(...)`، و`fit-content(...)`.

## الوصول عبر Tailnet (موصى به)

<Tabs>
  <Tab title="Tailscale Serve مدمج (مفضل)">
    أبق Gateway على local loopback ودع Tailscale Serve ينوب عنه عبر HTTPS:

    ```bash
    openclaw gateway --tailscale serve
    ```

    افتح:

    - `https://<magicdns>/` (أو `gateway.controlUi.basePath` الذي ضبطته)

    افتراضيا، يمكن لطلبات Control UI/WebSocket Serve المصادقة عبر ترويسات هوية Tailscale (`tailscale-user-login`) عندما تكون `gateway.auth.allowTailscale` هي `true`. يتحقق OpenClaw من الهوية بحل عنوان `x-forwarded-for` باستخدام `tailscale whois` ومطابقته مع الترويسة، ولا يقبل ذلك إلا عندما يصل الطلب إلى local loopback مع ترويسات `x-forwarded-*` الخاصة بـ Tailscale. بالنسبة إلى جلسات مشغل واجهة التحكم ذات هوية جهاز المتصفح، يتجاوز مسار Serve المتحقق هذا أيضا جولة إقران الجهاز؛ أما المتصفحات بلا جهاز واتصالات دور العقدة فتظل تتبع فحوصات الجهاز العادية. اضبط `gateway.auth.allowTailscale: false` إذا كنت تريد طلب بيانات اعتماد سر مشترك صريحة حتى لحركة مرور Serve. ثم استخدم `gateway.auth.mode: "token"` أو `"password"`.

    بالنسبة إلى مسار هوية Serve غير المتزامن هذا، تسلسل محاولات المصادقة الفاشلة للعنوان IP نفسه ونطاق المصادقة نفسه قبل كتابة حدود المعدل. لذلك قد تعرض عمليات إعادة المحاولة السيئة المتزامنة من المتصفح نفسه `retry later` في الطلب الثاني بدلا من عدم تطابقين عاديين يتسابقان بالتوازي.

    <Warning>
    تفترض مصادقة Serve بلا رمز أن مضيف Gateway موثوق. إذا كان من الممكن تشغيل كود محلي غير موثوق على ذلك المضيف، فاطلب مصادقة برمز/كلمة مرور.
    </Warning>

  </Tab>
  <Tab title="الربط بـ tailnet + رمز">
    ```bash
    openclaw gateway --bind tailnet --token "$(openssl rand -hex 32)"
    ```

    ثم افتح:

    - `http://<tailscale-ip>:18789/` (أو `gateway.controlUi.basePath` الذي ضبطته)

    الصق السر المشترك المطابق في إعدادات واجهة المستخدم (يرسل كـ `connect.params.auth.token` أو `connect.params.auth.password`).

  </Tab>
</Tabs>

## HTTP غير آمن

إذا فتحت لوحة المعلومات عبر HTTP عادي (`http://<lan-ip>` أو `http://<tailscale-ip>`)، يعمل المتصفح في **سياق غير آمن** ويحظر WebCrypto. افتراضيا، **يحظر** OpenClaw اتصالات واجهة التحكم من دون هوية جهاز.

الاستثناءات الموثقة:

- توافق HTTP غير آمن على localhost فقط باستخدام `gateway.controlUi.allowInsecureAuth=true`
- مصادقة ناجحة لمشغل واجهة التحكم عبر `gateway.auth.mode: "trusted-proxy"`
- خيار طوارئ `gateway.controlUi.dangerouslyDisableDeviceAuth=true`

**الإصلاح الموصى به:** استخدم HTTPS (Tailscale Serve) أو افتح واجهة المستخدم محليا:

- `https://<magicdns>/` (Serve)
- `http://127.0.0.1:18789/` (على مضيف Gateway)

<AccordionGroup>
  <Accordion title="سلوك مفتاح المصادقة غير الآمنة">
    ```json5
    {
      gateway: {
        controlUi: { allowInsecureAuth: true },
        bind: "tailnet",
        auth: { mode: "token", token: "replace-me" },
      },
    }
    ```

    `allowInsecureAuth` هو مفتاح توافق محلي فقط:

    - يسمح لجلسات Control UI على localhost بالمتابعة من دون هوية جهاز في سياقات HTTP غير الآمنة.
    - لا يتجاوز فحوصات الاقتران.
    - لا يخفف متطلبات هوية الجهاز البعيد (غير localhost).

  </Accordion>
  <Accordion title="للطوارئ فقط">
    ```json5
    {
      gateway: {
        controlUi: { dangerouslyDisableDeviceAuth: true },
        bind: "tailnet",
        auth: { mode: "token", token: "replace-me" },
      },
    }
    ```

    <Warning>
    يعطل `dangerouslyDisableDeviceAuth` فحوصات هوية جهاز Control UI ويعد خفضا شديدا لمستوى الأمان. تراجع عنه بسرعة بعد الاستخدام الطارئ.
    </Warning>

  </Accordion>
  <Accordion title="ملاحظة الوكيل الموثوق">
    - يمكن لمصادقة الوكيل الموثوق الناجحة قبول جلسات Control UI الخاصة بـ **المشغل** من دون هوية جهاز.
    - لا يمتد هذا إلى جلسات Control UI ذات دور العقدة.
    - لا تزال وكلاء reverse proxy عبر local loopback على المضيف نفسه لا تستوفي مصادقة الوكيل الموثوق؛ راجع [مصادقة الوكيل الموثوق](/ar/gateway/trusted-proxy-auth).

  </Accordion>
</AccordionGroup>

راجع [Tailscale](/ar/gateway/tailscale) للحصول على إرشادات إعداد HTTPS.

## سياسة أمان المحتوى

يأتي Control UI مع سياسة `img-src` صارمة: لا يسمح إلا بالأصول من **نفس المصدر**، وعناوين URL من نوع `data:`، وعناوين URL من نوع `blob:` المنشأة محليا. يرفض المتصفح عناوين صور `http(s)` البعيدة والعناوين النسبية للبروتوكول ولا يجري طلبات شبكة لها.

ما يعنيه هذا عمليا:

- لا تزال الصور الرمزية والصور المقدمة ضمن مسارات نسبية (على سبيل المثال `/avatars/<id>`) تظهر، بما في ذلك مسارات الصور الرمزية المصادق عليها التي يجلبها UI ويحولها إلى عناوين URL محلية من نوع `blob:`.
- لا تزال عناوين URL المضمنة من نوع `data:image/...` تظهر (مفيدة للحمولات داخل البروتوكول).
- لا تزال عناوين URL المحلية من نوع `blob:` التي ينشئها Control UI تظهر.
- تزال عناوين URL البعيدة للصور الرمزية الصادرة من بيانات تعريف القنوات عند مساعدات الصور الرمزية في Control UI وتستبدل بالشعار/الشارة المدمجين، لذلك لا تستطيع قناة مخترقة أو ضارة إجبار متصفح المشغل على جلب صور بعيدة عشوائية.

لا تحتاج إلى تغيير أي شيء للحصول على هذا السلوك — فهو مفعل دائما وغير قابل للتهيئة.

## مصادقة مسار الصورة الرمزية

عند تهيئة مصادقة Gateway، يتطلب طرفية الصورة الرمزية في Control UI رمز Gateway نفسه مثل بقية API:

- يعيد `GET /avatar/<agentId>` صورة الصورة الرمزية للمتصلين المصادق عليهم فقط. يعيد `GET /avatar/<agentId>?meta=1` بيانات تعريف الصورة الرمزية وفق القاعدة نفسها.
- ترفض الطلبات غير المصادق عليها إلى أي من المسارين (بما يطابق مسار وسائط المساعد الشقيق). يمنع هذا مسار الصورة الرمزية من تسريب هوية الوكيل على المضيفات المحمية بطريقة أخرى.
- يمرر Control UI نفسه رمز Gateway كترويسة bearer عند جلب الصور الرمزية، ويستخدم عناوين URL مصادق عليها من نوع blob بحيث تظل الصورة تظهر في لوحات المعلومات.

إذا عطلت مصادقة Gateway (غير موصى به على المضيفات المشتركة)، يصبح مسار الصورة الرمزية أيضا غير مصادق عليه، بما يتماشى مع بقية Gateway.

## بناء UI

يقدم Gateway الملفات الثابتة من `dist/control-ui`. ابنها باستخدام:

```bash
pnpm ui:build
```

قاعدة مطلقة اختيارية (عندما تريد عناوين URL ثابتة للأصول):

```bash
OPENCLAW_CONTROL_UI_BASE_PATH=/openclaw/ pnpm ui:build
```

للتطوير المحلي (خادم تطوير منفصل):

```bash
pnpm ui:dev
```

ثم وجه UI إلى عنوان URL الخاص بـ Gateway WS لديك (مثلا `ws://127.0.0.1:18789`).

## التصحيح/الاختبار: خادم التطوير + Gateway بعيد

Control UI عبارة عن ملفات ثابتة؛ هدف WebSocket قابل للتهيئة ويمكن أن يكون مختلفا عن أصل HTTP. يكون هذا مفيدا عندما تريد خادم تطوير Vite محليا بينما يعمل Gateway في مكان آخر.

<Steps>
  <Step title="ابدأ خادم تطوير UI">
    ```bash
    pnpm ui:dev
    ```
  </Step>
  <Step title="افتح باستخدام gatewayUrl">
    ```text
    http://localhost:5173/?gatewayUrl=ws%3A%2F%2F<gateway-host>%3A18789
    ```

    مصادقة اختيارية لمرة واحدة (إذا لزم الأمر):

    ```text
    http://localhost:5173/?gatewayUrl=wss%3A%2F%2F<gateway-host>%3A18789#token=<gateway-token>
    ```

  </Step>
</Steps>

<AccordionGroup>
  <Accordion title="ملاحظات">
    - يخزن `gatewayUrl` في localStorage بعد التحميل ويزال من عنوان URL.
    - إذا مررت نقطة نهاية كاملة `ws://` أو `wss://` عبر `gatewayUrl`، فقم بترميز قيمة `gatewayUrl` في URL حتى يفسر المتصفح سلسلة الاستعلام بشكل صحيح.
    - ينبغي تمرير `token` عبر جزء عنوان URL (`#token=...`) كلما أمكن. لا ترسل الأجزاء إلى الخادم، ما يتجنب تسربها في سجلات الطلبات وReferer. لا تزال معلمات الاستعلام القديمة `?token=` تستورد مرة واحدة للتوافق، لكن كخيار احتياطي فقط، وتزال فورا بعد bootstrap.
    - يبقى `password` في الذاكرة فقط.
    - عند تعيين `gatewayUrl`، لا يعود UI إلى بيانات اعتماد التهيئة أو البيئة. وفر `token` (أو `password`) صراحة. عدم وجود بيانات اعتماد صريحة يعد خطأ.
    - استخدم `wss://` عندما يكون Gateway خلف TLS (Tailscale Serve، وكيل HTTPS، إلخ).
    - لا يقبل `gatewayUrl` إلا في نافذة بالمستوى الأعلى (غير مضمّنة) لمنع clickjacking.
    - يجب على عمليات نشر Control UI خارج local loopback تعيين `gateway.controlUi.allowedOrigins` صراحة (أصول كاملة). يشمل هذا إعدادات التطوير البعيدة.
    - قد يزرع بدء تشغيل Gateway أصولا محلية مثل `http://localhost:<port>` و`http://127.0.0.1:<port>` من الربط والمنفذ الفعليين في وقت التشغيل، لكن أصول المتصفح البعيدة لا تزال تحتاج إلى إدخالات صريحة.
    - لا تستخدم `gateway.controlUi.allowedOrigins: ["*"]` إلا للاختبار المحلي المحكوم بإحكام. يعني ذلك السماح لأي أصل متصفح، وليس "مطابقة أي مضيف أستخدمه".
    - يفعّل `gateway.controlUi.dangerouslyAllowHostHeaderOriginFallback=true` وضع الرجوع إلى أصل ترويسة Host، لكنه وضع أمان خطير.

  </Accordion>
</AccordionGroup>

مثال:

```json5
{
  gateway: {
    controlUi: {
      allowedOrigins: ["http://localhost:5173"],
    },
  },
}
```

تفاصيل إعداد الوصول البعيد: [الوصول البعيد](/ar/gateway/remote).
