Control UI هو تطبيق صغير أحادي الصفحة مبني بـ **Vite + Lit** ويخدمه Gateway:

- الافتراضي: `http://<host>:18789/`
- بادئة اختيارية: اضبط `gateway.controlUi.basePath` (مثلًا `/openclaw`)

يتواصل **مباشرةً مع WebSocket الخاص بـ Gateway** على المنفذ نفسه.

## فتح سريع (محلي)

إذا كان Gateway يعمل على الحاسوب نفسه، فافتح:

- [http://127.0.0.1:18789/](http://127.0.0.1:18789/) (أو [http://localhost:18789/](http://localhost:18789/))

إذا فشل تحميل الصفحة، فابدأ Gateway أولًا: `openclaw gateway`.

<Note>
في ارتباطات LAN الأصلية على Windows، قد يظل Windows Firewall أو Group Policy المُدار من المؤسسة يحظر عنوان URL المعلن على LAN حتى عندما يعمل `127.0.0.1` على مضيف Gateway. شغّل `openclaw gateway status --deep` على مضيف Windows؛ فهو يبلغ عن المنافذ المحتمل حظرها، وعدم تطابق ملفات التعريف، وقواعد الجدار الناري المحلي التي قد تتجاهلها السياسة.
</Note>

تُوفَّر المصادقة أثناء مصافحة WebSocket عبر:

- `connect.params.auth.token`
- `connect.params.auth.password`
- ترويسات هوية Tailscale Serve عندما يكون `gateway.auth.allowTailscale: true`
- ترويسات هوية الوكيل الموثوق عندما يكون `gateway.auth.mode: "trusted-proxy"`

تحتفظ لوحة إعدادات لوحة المعلومات برمز مميز لجلسة تبويب المتصفح الحالية وعنوان URL المحدد لـ gateway؛ ولا تُحفظ كلمات المرور بشكل دائم. عادةً ما ينشئ الإعداد الأولي رمز gateway لمصادقة السر المشترك عند أول اتصال، لكن مصادقة كلمة المرور تعمل أيضًا عندما يكون `gateway.auth.mode` هو `"password"`.

## إقران الجهاز (الاتصال الأول)

عند الاتصال بـ Control UI من متصفح أو جهاز جديد، يتطلب Gateway عادةً **موافقة إقران لمرة واحدة**. هذا إجراء أمني لمنع الوصول غير المصرح به.

**ما ستراه:** "disconnected (1008): pairing required"

<Steps>
  <Step title="اسرد الطلبات المعلقة">
    ```bash
    openclaw devices list
    ```
  </Step>
  <Step title="الموافقة بواسطة معرّف الطلب">
    ```bash
    openclaw devices approve <requestId>
    ```
  </Step>
</Steps>

إذا أعاد المتصفح محاولة الإقران مع تفاصيل مصادقة متغيرة (الدور/النطاقات/المفتاح العام)، فسيُستبدل الطلب المعلق السابق ويُنشأ `requestId` جديد. أعد تشغيل `openclaw devices list` قبل الموافقة.

إذا كان المتصفح مقترنًا بالفعل وغيّرته من وصول قراءة إلى وصول كتابة/مسؤول، فيُعامل ذلك كترقية موافقة، وليس إعادة اتصال صامتة. يُبقي OpenClaw الموافقة القديمة نشطة، ويحظر إعادة الاتصال ذات النطاق الأوسع، ويطلب منك الموافقة صراحةً على مجموعة النطاق الجديدة.

بمجرد الموافقة، يُتذكر الجهاز ولن يتطلب إعادة موافقة إلا إذا ألغيت تفويضه باستخدام `openclaw devices revoke --device <id> --role <role>`. راجع [CLI الأجهزة](/ar/cli/devices) لتدوير الرموز المميزة والإلغاء.

تستخدم وكلاء Paperclip التي تتصل عبر مهايئ `openclaw_gateway` تدفق الموافقة نفسه عند التشغيل الأول. بعد محاولة الاتصال الأولية، شغّل `openclaw devices approve --latest` لمعاينة الطلب المعلق، ثم أعد تشغيل أمر `openclaw devices approve <requestId>` المطبوع للموافقة عليه. مرّر قيم `--url` و`--token` صريحة لـ gateway بعيد. للحفاظ على ثبات الموافقات عبر عمليات إعادة التشغيل، اضبط `adapterConfig.devicePrivateKeyPem` دائم في Paperclip بدلًا من السماح له بإنشاء هوية جهاز مؤقتة جديدة في كل تشغيل.

<Note>
- تتم الموافقة تلقائيًا على اتصالات المتصفح المباشرة عبر local loopback (`127.0.0.1` / `localhost`).
- يمكن لـ Tailscale Serve تجاوز رحلة الإقران ذهابًا وإيابًا لجلسات مشغل Control UI عندما يكون `gateway.auth.allowTailscale: true`، وتتحقق هوية Tailscale، ويقدّم المتصفح هوية جهازه.
- لا تزال ارتباطات Tailnet المباشرة، واتصالات متصفح LAN، وملفات تعريف المتصفح التي لا تحتوي على هوية جهاز تتطلب موافقة صريحة.
- ينشئ كل ملف تعريف متصفح معرّف جهاز فريدًا، لذلك سيتطلب تبديل المتصفحات أو مسح بيانات المتصفح إعادة الإقران.

</Note>

## الهوية الشخصية (محلية في المتصفح)

يدعم Control UI هوية شخصية لكل متصفح (اسم عرض وصورة رمزية) تُرفق بالرسائل الصادرة للإسناد في الجلسات المشتركة. تعيش في تخزين المتصفح، ونطاقها ملف تعريف المتصفح الحالي، ولا تُزامَن إلى أجهزة أخرى أو تُحفظ من جهة الخادم بما يتجاوز بيانات تعريف تأليف النصوص المعتادة للرسائل التي ترسلها فعليًا. يؤدي مسح بيانات الموقع أو تبديل المتصفحات إلى إعادة تعيينها لتصبح فارغة.

ينطبق نمط المتصفح المحلي نفسه على تجاوز الصورة الرمزية للمساعد. تُراكب الصور الرمزية المرفوعة للمساعد الهوية التي يحلها gateway على المتصفح المحلي فقط ولا تمر أبدًا ذهابًا وإيابًا عبر `config.patch`. لا يزال حقل إعدادات `ui.assistant.avatar` المشترك متاحًا للعملاء غير واجهة المستخدم الذين يكتبون الحقل مباشرةً (مثل gateways المبرمجة أو لوحات معلومات مخصصة).

## نقطة نهاية إعدادات وقت التشغيل

يجلب Control UI إعدادات وقت التشغيل الخاصة به من `/control-ui-config.json`، محسوبةً نسبيًا إلى المسار الأساسي لـ Control UI في gateway (على سبيل المثال `/__openclaw__/control-ui-config.json` عندما تُخدم واجهة المستخدم تحت `/__openclaw__/`). تخضع نقطة النهاية هذه لمصادقة gateway نفسها مثل بقية سطح HTTP: لا يمكن للمتصفحات غير المصادقة جلبها، ويتطلب الجلب الناجح إما رمز gateway/كلمة مرور صالحين بالفعل، أو هوية Tailscale Serve، أو هوية وكيل موثوق.

## دعم اللغة

يمكن لـ Control UI توطين نفسه عند التحميل الأول بناءً على لغة متصفحك. لتجاوز ذلك لاحقًا، افتح **نظرة عامة -> وصول Gateway -> اللغة**. يوجد منتقي اللغة في بطاقة وصول Gateway، وليس ضمن المظهر.

- اللغات المدعومة: `en`, `zh-CN`, `zh-TW`, `pt-BR`, `de`, `es`, `ja-JP`, `ko`, `fr`, `ar`, `it`, `tr`, `uk`, `id`, `pl`, `th`, `vi`, `nl`, `fa`
- تُحمَّل الترجمات غير الإنجليزية كسولًا في المتصفح.
- تُحفظ اللغة المحددة في تخزين المتصفح ويُعاد استخدامها في الزيارات المستقبلية.
- تعود مفاتيح الترجمة المفقودة إلى الإنجليزية.

تُولَّد ترجمات المستندات لمجموعة اللغات غير الإنجليزية نفسها، لكن منتقي اللغة المضمّن في موقع المستندات من Mintlify يقتصر على رموز اللغات التي يقبلها Mintlify. لا تزال مستندات التايلاندية (`th`) والفارسية (`fa`) تُولَّد في مستودع النشر؛ وقد لا تظهر في ذلك المنتقي حتى يدعم Mintlify تلك الرموز.

## سمات المظهر

تحتفظ لوحة المظهر بسمات Claw وKnot وDash المضمّنة، بالإضافة إلى خانة استيراد tweakcn محلية في المتصفح. لاستيراد سمة، افتح [محرر tweakcn](https://tweakcn.com/editor/theme)، واختر سمة أو أنشئها، وانقر **مشاركة**، ثم الصق رابط السمة المنسوخ في المظهر. يقبل المستورد أيضًا عناوين URL لسجل `https://tweakcn.com/r/themes/<id>`، وعناوين URL للمحرر مثل `https://tweakcn.com/editor/theme?theme=amethyst-haze`، ومسارات `/themes/<id>` النسبية، ومعرّفات السمات الخام، وأسماء السمات الافتراضية مثل `amethyst-haze`.

يتضمن المظهر أيضًا إعداد حجم نص محليًا في المتصفح. يُخزَّن الإعداد مع بقية تفضيلات Control UI، وينطبق على نص الدردشة، ونص المؤلف، وبطاقات الأدوات، والأشرطة الجانبية للدردشة، ويحافظ على مدخلات النص بحجم 16px على الأقل حتى لا يقوم Safari على الجوال بالتكبير تلقائيًا عند التركيز.

تُخزَّن السمات المستوردة فقط في ملف تعريف المتصفح الحالي. ولا تُكتب في إعدادات gateway ولا تُزامَن عبر الأجهزة. يؤدي استبدال السمة المستوردة إلى تحديث الخانة المحلية الواحدة؛ ويؤدي مسحها إلى إعادة السمة النشطة إلى Claw إذا كانت السمة المستوردة محددة.

## صفحة MCP

صفحة MCP المخصصة هي عرض للمشغل لخوادم MCP التي يديرها OpenClaw ضمن `mcp.servers`. لا تبدأ نقل MCP بنفسها؛ استخدمها لفحص التكوين المحفوظ وتحريره، ثم استخدم `openclaw mcp doctor --probe` عندما تحتاج إلى إثبات مباشر للخادم.

سير العمل المعتاد:

1. افتح **MCP** من الشريط الجانبي.
2. تحقق من بطاقات الملخص لإجمالي الخوادم وعدد الخوادم المفعلة وخوادم OAuth والخوادم المفلترة.
3. راجع كل صف خادم للاطلاع على النقل، والتفعيل، والمصادقة، والفلاتر، والمهلات، وتلميحات الأوامر.
4. بدّل التفعيل عندما يجب أن يبقى الخادم مكوّنا لكن خارج اكتشاف وقت التشغيل.
5. حرر قسم تكوين `mcp` محدود النطاق لتعريفات الخوادم، والترويسات، ومسارات TLS/mTLS، وبيانات OAuth الوصفية، وفلاتر الأدوات، وبيانات إسقاط Codex الوصفية.
6. استخدم **حفظ** لكتابة التكوين، أو **حفظ ونشر** عندما يجب أن يطبق Gateway قيد التشغيل التكوين المتغير.
7. شغّل `openclaw mcp status --verbose` أو `openclaw mcp doctor --probe` أو `openclaw mcp reload` من طرفية عندما تحتاج العملية المعدلة إلى تشخيصات ثابتة أو إثبات مباشر أو التخلص من وقت التشغيل المخزن مؤقتا.

تحجب الصفحة القيم الشبيهة بعناوين URL التي تحمل بيانات اعتماد قبل عرضها، وتضع أسماء الخوادم بين علامات اقتباس في مقتطفات الأوامر حتى تظل الأوامر المنسوخة تعمل مع المسافات أو المحارف الخاصة بالصدفة. يوجد مرجع CLI والتكوين الكامل في [MCP](/ar/cli/mcp).

## تبويب النشاط

تبويب النشاط هو مراقب مؤقت محلي للمتصفح لنشاط الأدوات المباشر. وهو مشتق من تدفق أحداث Gateway نفسه `session.tool` / tool الذي يشغل بطاقات أدوات الدردشة؛ ولا يضيف عائلة أحداث Gateway أخرى، أو نقطة نهاية، أو مخزن نشاط دائم، أو موجز مقاييس، أو تدفق مراقب خارجي.

تحتفظ إدخالات النشاط بملخصات منقحة ومعاينات خرج منقحة ومقتطعة فقط. لا تُخزن قيم وسيطات الأدوات في حالة النشاط؛ وتعرض الواجهة أن الوسيطات مخفية وتسجل عدد حقول الوسيطات فقط. تتبع القائمة الموجودة في الذاكرة تبويب المتصفح الحالي، وتبقى بعد التنقل داخل واجهة التحكم، وتُعاد ضبطها عند إعادة تحميل الصفحة، أو تبديل الجلسة، أو **مسح**.

## سلوك الدردشة

<AccordionGroup>
  <Accordion title="دلالات الإرسال والسجل">
    - `chat.send` **غير حاظر**: يؤكد فورا بـ `{ runId, status: "started" }` وتتدفق الاستجابة عبر أحداث `chat`. قد تتلقى عملاء واجهة التحكم الموثوقون أيضا بيانات وصفية اختيارية لتوقيت ACK للتشخيص المحلي.
    - تقبل تحميلات الدردشة الصور والملفات غير الفيديوية. تحتفظ الصور بمسار الصورة الأصلي؛ وتُخزن الملفات الأخرى كوسائط مُدارة وتظهر في السجل كروابط مرفقات.
    - إعادة الإرسال باستخدام `idempotencyKey` نفسه تعيد `{ status: "in_flight" }` أثناء التشغيل، و`{ status: "ok" }` بعد الإكمال.
    - تكون استجابات `chat.history` محدودة الحجم من أجل سلامة الواجهة. عندما تكون إدخالات النص كبيرة جدا، قد يقتطع Gateway حقول النص الطويلة، ويحذف كتل البيانات الوصفية الثقيلة، ويستبدل الرسائل كبيرة الحجم بعنصر نائب (`[chat.history omitted: message too large]`).
    - عندما تُقتطع رسالة مساعد مرئية في `chat.history`، يمكن للقارئ الجانبي جلب إدخال النص الكامل المطبع للعرض عند الطلب عبر `chat.message.get` باستخدام `sessionKey`، و`agentId` النشط عند الحاجة، و`messageId` للنص. إذا ظل Gateway غير قادر على إرجاع المزيد، يعرض القارئ حالة عدم توفر صريحة بدلا من تكرار المعاينة المقتطعة بصمت.
    - تُحفظ الصور التي ينشئها المساعد كإشارات وسائط مُدارة وتُقدم مرة أخرى عبر عناوين URL لوسائط Gateway مصادق عليها، لذلك لا تعتمد عمليات إعادة التحميل على بقاء حمولات صور base64 الخام في استجابة سجل الدردشة.
    - عند عرض `chat.history`، تجرد واجهة التحكم وسوم التوجيه المضمنة الخاصة بالعرض فقط من نص المساعد المرئي (مثل `[[reply_to_*]]` و`[[audio_as_voice]]`)، وحمولات XML لاستدعاءات الأدوات ذات النص العادي (بما في ذلك `<tool_call>...</tool_call>`، و`<function_call>...</function_call>`، و`<tool_calls>...</tool_calls>`، و`<function_calls>...</function_calls>`، وكتل استدعاء الأدوات المقتطعة)، ورموز تحكم النموذج المسربة بنمط ASCII/العرض الكامل، وتحذف إدخالات المساعد التي يكون نصها المرئي كله الرمز الصامت الدقيق `NO_REPLY` / `no_reply` أو رمز إقرار Heartbeat `HEARTBEAT_OK`.
    - أثناء إرسال نشط وتحديث السجل النهائي، يبقي عرض الدردشة رسائل المستخدم/المساعد التفاؤلية المحلية مرئية إذا أعاد `chat.history` لقطة أقدم لفترة وجيزة؛ ويستبدل النص القانوني تلك الرسائل المحلية بمجرد أن يلحق سجل Gateway.
    - أحداث `chat` المباشرة هي حالة التسليم، بينما يُعاد بناء `chat.history` من نص الجلسة الدائم. بعد أحداث tool-final تعيد واجهة التحكم تحميل السجل وتدمج ذيلا تفاؤليا صغيرا فقط؛ وموثق حد النص في [WebChat](/ar/web/webchat).
    - يضيف `chat.inject` ملاحظة مساعد إلى نص الجلسة ويبث حدث `chat` للتحديثات الخاصة بالواجهة فقط (لا تشغيل وكيل، ولا تسليم قناة).
    - يعرض رأس الدردشة فلتر الوكيل قبل منتقي الجلسة، ويكون منتقي الجلسة محدودا بالوكيل المحدد. يعرض تبديل الوكلاء الجلسات المرتبطة بذلك الوكيل فقط، ويرجع إلى الجلسة الرئيسية لذلك الوكيل عندما لا تكون لديه جلسات لوحة معلومات محفوظة بعد.
    - في عروض سطح المكتب، تبقى عناصر التحكم في الدردشة في صف مدمج واحد وتنهار أثناء التمرير إلى أسفل النص؛ ويؤدي التمرير إلى أعلى، أو الرجوع إلى القمة، أو الوصول إلى الأسفل إلى استعادة عناصر التحكم.
    - تُعرض الرسائل النصية المتكررة المتتالية كفقاعة واحدة مع شارة عدد. لا تُطوى الرسائل التي تحمل صورا، أو مرفقات، أو خرج أدوات، أو معاينات لوحة.
    - يحدّث منتقيا النموذج والتفكير في رأس الدردشة الجلسة النشطة فورا عبر `sessions.patch`؛ وهما تجاوزات جلسة دائمة، وليسا خيارات إرسال لدورة واحدة فقط.
    - إذا أرسلت رسالة بينما لا يزال تغيير منتقي النموذج للجلسة نفسها قيد الحفظ، ينتظر المُنشئ تصحيح تلك الجلسة قبل استدعاء `chat.send` حتى يستخدم الإرسال النموذج المحدد.
    - تؤدي كتابة `/new` في واجهة التحكم إلى إنشاء جلسة لوحة معلومات جديدة نفسها والتبديل إليها مثل دردشة جديدة، إلا عندما يكون `session.dmScope: "main"` مكوّنا ويكون الأصل الحالي هو الجلسة الرئيسية للوكيل؛ في هذه الحالة تعيد ضبط الجلسة الرئيسية في مكانها. وتحافظ كتابة `/reset` على إعادة الضبط الصريحة في المكان نفسه من Gateway للجلسة الحالية.
    - يطلب منتقي نموذج الدردشة عرض النموذج المكوّن في Gateway. إذا كانت `agents.defaults.models` موجودة، تقود قائمة السماح تلك المنتقي، بما في ذلك إدخالات `provider/*` التي تبقي فهارس الموفرين محدودة النطاق بالموفر ديناميكية. وإلا يعرض المنتقي إدخالات `models.providers.*.models` الصريحة إضافة إلى الموفرين ذوي المصادقة القابلة للاستخدام. يبقى الفهرس الكامل متاحا عبر RPC التصحيح `models.list` مع `view: "all"`.
    - عندما تتضمن تقارير استخدام جلسة Gateway الحديثة رموز السياق الحالية، تعرض منطقة منشئ الدردشة مؤشرا مدمجا لاستخدام السياق. يتحول إلى نمط تحذير عند ضغط سياق عال، ويعرض عند مستويات Compaction الموصى بها زرا مدمجا يشغّل مسار Compaction العادي للجلسة. تُخفى لقطات الرموز القديمة حتى يبلغ Gateway عن استخدام حديث مرة أخرى.

  </Accordion>
  <Accordion title="وضع Talk (الوقت الحقيقي في المتصفح)">
    يستخدم وضع Talk موفر صوت وقت حقيقي مسجلا. كوّن OpenAI باستخدام `talk.realtime.provider: "openai"` إضافة إلى ملف مصادقة مفتاح API لـ `openai`، أو `talk.realtime.providers.openai.apiKey`، أو `OPENAI_API_KEY`؛ لا تكوّن ملفات OpenAI OAuth صوت Realtime. كوّن Google باستخدام `talk.realtime.provider: "google"` إضافة إلى `talk.realtime.providers.google.apiKey`. لا يتلقى المتصفح أبدا مفتاح API قياسيا للموفر. يتلقى OpenAI سر عميل Realtime مؤقتا لـ WebRTC. يتلقى Google Live رمز مصادقة Live API مقيدا لمرة واحدة لجلسة WebSocket في المتصفح، مع تعليمات وتصريحات أدوات مقفلة داخل الرمز بواسطة Gateway. تعمل الموفرات التي تعرض جسرا خلفيا للوقت الحقيقي فقط عبر نقل مرحل Gateway، لذلك تبقى بيانات الاعتماد ومقابس البائع من جهة الخادم بينما ينتقل صوت المتصفح عبر RPCs Gateway مصادق عليها. يُجمّع موجه جلسة Realtime بواسطة Gateway؛ ولا يقبل `talk.client.create` تجاوزات تعليمات يقدمها المستدعي.

    يتضمن منشئ الدردشة زر خيارات Talk بجوار زر بدء/إيقاف Talk. تنطبق الخيارات على جلسة Talk التالية ويمكنها تجاوز الموفر، والنقل، والنموذج، والصوت، وجهد الاستدلال، وعتبة VAD، ومدة الصمت، وحشو البادئة. عندما يكون خيار فارغا، يستخدم Gateway الإعدادات الافتراضية المكوّنة حيثما توفرت أو الإعداد الافتراضي للموفر. يجبر اختيار مرحل Gateway مسار المرحل الخلفي؛ ويحافظ اختيار WebRTC على ملكية العميل للجلسة ويفشل بدلا من الرجوع بصمت إلى المرحل إذا تعذر على الموفر إنشاء جلسة متصفح.

    في منشئ الدردشة، يكون عنصر تحكم Talk هو زر الموجات بجوار زر إملاء الميكروفون. عند بدء Talk، يعرض صف حالة المنشئ `Connecting Talk...`، ثم `Talk live` أثناء اتصال الصوت، أو `Asking OpenClaw...` بينما يستشير استدعاء أداة في الوقت الحقيقي النموذج الأكبر المكوّن عبر `talk.client.toolCall`.

    اختبار دخاني مباشر للمشرف: يتحقق `OPENAI_API_KEY=... GEMINI_API_KEY=... node --import tsx scripts/dev/realtime-talk-live-smoke.ts` من جسر WebSocket الخلفي لـ OpenAI، وتبادل SDP لـ WebRTC في متصفح OpenAI، وإعداد WebSocket في متصفح Google Live برمز مقيد، ومحوّل متصفح مرحل Gateway مع وسائط ميكروفون وهمية. يطبع الأمر حالة الموفر فقط ولا يسجل الأسرار.

  </Accordion>

    - يدعم `chat.abort` الصيغة `{ sessionKey }` (دون `runId`) لإلغاء كل التشغيلات النشطة لتلك الجلسة.

  </Accordion>

  </Accordion>

## تثبيت PWA وWeb Push

تشحن واجهة التحكم `manifest.webmanifest` وعامل خدمة، لذلك يمكن للمتصفحات الحديثة تثبيتها كتطبيق PWA مستقل. يتيح Web Push لـ Gateway إيقاظ PWA المثبت بالإشعارات حتى عندما لا يكون التبويب أو نافذة المتصفح مفتوحا.

إذا عرضت الصفحة **عدم تطابق البروتوكول** مباشرة بعد تحديث OpenClaw، فأعد أولا فتح لوحة المعلومات باستخدام `openclaw dashboard` وأجر تحديثا قسريا للصفحة. إذا استمر الفشل، فامسح بيانات الموقع لأصل لوحة المعلومات أو اختبر في نافذة متصفح خاصة؛ إذ يمكن لتبويب قديم أو ذاكرة عامل خدمة المتصفح المؤقتة أن تبقي حزمة واجهة التحكم السابقة للتحديث قيد التشغيل مقابل Gateway الأحدث.

| السطح                                                | ما يفعله                                                         |
| ----------------------------------------------------- | ------------------------------------------------------------------ |
| `ui/public/manifest.webmanifest`                      | بيان PWA. تعرض المتصفحات خيار "تثبيت التطبيق" بمجرد أن يصبح متاحًا.   |
| `ui/public/sw.js`                                     | عامل خدمة يتعامل مع أحداث `push` ونقرات الإشعارات. |
| `push/vapid-keys.json` (تحت دليل حالة OpenClaw) | زوج مفاتيح VAPID مولّد تلقائيًا يُستخدم لتوقيع حمولات Web Push.       |
| `push/web-push-subscriptions.json`                    | نقاط نهاية اشتراكات المتصفح المستمرة.                          |

تجاوز زوج مفاتيح VAPID عبر متغيرات البيئة في عملية Gateway عندما تريد تثبيت المفاتيح (للنشر متعدد المضيفين، أو تدوير الأسرار، أو الاختبارات):

- `OPENCLAW_VAPID_PUBLIC_KEY`
- `OPENCLAW_VAPID_PRIVATE_KEY`
- `OPENCLAW_VAPID_SUBJECT` (القيمة الافتراضية `https://openclaw.ai`)

تستخدم واجهة التحكم طرق Gateway هذه، المقيّدة بالنطاق، لتسجيل اشتراكات المتصفح واختبارها:

- `push.web.vapidPublicKey` — تجلب مفتاح VAPID العام النشط.
- `push.web.subscribe` — تسجل `endpoint` مع `keys.p256dh`/`keys.auth`.
- `push.web.unsubscribe` — تزيل نقطة نهاية مسجلة.
- `push.web.test` — ترسل إشعار اختبار إلى اشتراك المستدعي.

<Note>
Web Push مستقل عن مسار ترحيل iOS APNS (راجع [الإعداد](/ar/gateway/configuration) للدفع المدعوم بالترحيل) وعن طريقة `push.test` الحالية، اللذين يستهدفان اقتران الجوال الأصلي.
</Note>

## التضمينات المستضافة

يمكن لرسائل المساعد عرض محتوى ويب مستضافًا داخل السطر باستخدام الاختصار `[embed ...]`. تتحكم `gateway.controlUi.embedSandbox` في سياسة صندوق رمل iframe:

<Tabs>
  <Tab title="صارم">
    يعطل تنفيذ السكربتات داخل التضمينات المستضافة.
  </Tab>
  <Tab title="سكربتات (افتراضي)">
    يسمح بالتضمينات التفاعلية مع الحفاظ على عزل الأصل؛ هذا هو الافتراضي ويكفي عادة للألعاب/الأدوات المصغّرة المستقلة في المتصفح.
  </Tab>
  <Tab title="موثوق">
    يضيف `allow-same-origin` فوق `allow-scripts` للمستندات من الموقع نفسه التي تحتاج عمدًا إلى صلاحيات أقوى.
  </Tab>
</Tabs>

مثال:

```json5
{
  gateway: {
    controlUi: {
      embedSandbox: "scripts",
    },
  },
}
```

<Warning>
استخدم `trusted` فقط عندما يحتاج المستند المضمّن فعلًا إلى سلوك الأصل نفسه. لمعظم الألعاب ولوحات الرسم التفاعلية التي يولدها الوكلاء، يكون `scripts` الخيار الأكثر أمانًا.
</Warning>

تبقى عناوين URL الخارجية المطلقة لتضمينات `http(s)` محظورة افتراضيًا. إذا كنت تريد عمدًا أن يحمل `[embed url="https://..."]` صفحات تابعة لجهات خارجية، فاضبط `gateway.controlUi.allowExternalEmbedUrls: true`.

## عرض رسالة المحادثة

تستخدم رسائل المحادثة المجمّعة حدًا أقصى افتراضيًا للعرض يسهّل القراءة. يمكن لعمليات النشر على الشاشات العريضة تجاوزه دون تعديل CSS المضمّن عبر ضبط `gateway.controlUi.chatMessageMaxWidth`:

```json5
{
  gateway: {
    controlUi: {
      chatMessageMaxWidth: "min(1280px, 82%)",
    },
  },
}
```

تُتحقق القيمة قبل وصولها إلى المتصفح. تشمل القيم المدعومة الأطوال والنسب المئوية البسيطة مثل `960px` أو `82%`، بالإضافة إلى تعبيرات عرض مقيّدة مثل `min(...)` و`max(...)` و`clamp(...)` و`calc(...)` و`fit-content(...)`.

    افتح:

    - `https://<magicdns>/` (أو `gateway.controlUi.basePath` الذي ضبطته)

    افتراضيًا، يمكن لطلبات واجهة التحكم/WebSocket Serve المصادقة عبر ترويسات هوية Tailscale (`tailscale-user-login`) عندما تكون `gateway.auth.allowTailscale` مضبوطة على `true`. يتحقق OpenClaw من الهوية بحل عنوان `x-forwarded-for` باستخدام `tailscale whois` ومطابقته مع الترويسة، ولا يقبل هذه الطلبات إلا عندما يصل الطلب إلى loopback مع ترويسات `x-forwarded-*` الخاصة بـ Tailscale. بالنسبة إلى جلسات مشغل واجهة التحكم ذات هوية جهاز المتصفح، يتجاوز مسار Serve المتحقق منه هذا أيضًا رحلة اقتران الجهاز؛ أما المتصفحات بلا جهاز واتصالات دور العقدة فتظل تتبع فحوصات الجهاز المعتادة. اضبط `gateway.auth.allowTailscale: false` إذا كنت تريد طلب بيانات اعتماد سر مشترك صريحة حتى لحركة Serve. ثم استخدم `gateway.auth.mode: "token"` أو `"password"`.

    في مسار هوية Serve غير المتزامن هذا، تُسلسل محاولات المصادقة الفاشلة لعنوان IP العميل نفسه ونطاق المصادقة نفسه قبل كتابات حد المعدل. لذلك يمكن للمحاولات السيئة المتزامنة من المتصفح نفسه أن تعرض `retry later` في الطلب الثاني بدل عدم تطابقين عاديين يتسابقان بالتوازي.

    <Warning>
    تفترض مصادقة Serve بلا رمز أن مضيف Gateway موثوق. إذا كان يمكن تشغيل كود محلي غير موثوق على ذلك المضيف، فاطلب مصادقة رمز/كلمة مرور.
    </Warning>

  </Tab>
  <Tab title="الربط بـ tailnet + رمز">
    ```bash
    openclaw gateway --bind tailnet --token "$(openssl rand -hex 32)"
    ```

    ثم افتح:

    - `http://<tailscale-ip>:18789/` (أو `gateway.controlUi.basePath` الذي ضبطته)

    الصق السر المشترك المطابق في إعدادات الواجهة (يُرسل كـ `connect.params.auth.token` أو `connect.params.auth.password`).

  </Tab>
</Tabs>

## HTTP غير آمن

إذا فتحت لوحة المعلومات عبر HTTP عادي (`http://<lan-ip>` أو `http://<tailscale-ip>`)، يعمل المتصفح في **سياق غير آمن** ويحظر WebCrypto. افتراضيًا، **يحظر** OpenClaw اتصالات واجهة التحكم بلا هوية جهاز.

الاستثناءات الموثقة:

- توافق HTTP غير الآمن للـ localhost فقط مع `gateway.controlUi.allowInsecureAuth=true`
- مصادقة ناجحة لمشغل واجهة التحكم عبر `gateway.auth.mode: "trusted-proxy"`
- خيار الطوارئ `gateway.controlUi.dangerouslyDisableDeviceAuth=true`

**الإصلاح الموصى به:** استخدم HTTPS (Tailscale Serve) أو افتح الواجهة محليًا:

- `https://<magicdns>/` (Serve)
- `http://127.0.0.1:18789/` (على مضيف Gateway)

<AccordionGroup>
  <Accordion title="سلوك مفتاح المصادقة غير الآمنة">
    ```json5
    {
      gateway: {
        controlUi: { allowInsecureAuth: true },
        bind: "tailnet",
        auth: { mode: "token", token: "replace-me" },
      },
    }
    ```

    `allowInsecureAuth` هو مفتاح توافق محلي فقط:

    - يسمح لجلسات واجهة التحكم على localhost بالمتابعة دون هوية جهاز في سياقات HTTP غير الآمنة.
    - لا يتجاوز فحوصات الاقتران.
    - لا يخفف متطلبات هوية الجهاز البعيد (غير localhost).

  </Accordion>
  <Accordion title="للطوارئ فقط">
    ```json5
    {
      gateway: {
        controlUi: { dangerouslyDisableDeviceAuth: true },
        bind: "tailnet",
        auth: { mode: "token", token: "replace-me" },
      },
    }
    ```

    <Warning>
    يعطل `dangerouslyDisableDeviceAuth` فحوصات هوية جهاز واجهة التحكم ويمثل تخفيضًا أمنيًا شديدًا. أعده سريعًا بعد استخدامه في الطوارئ.
    </Warning>

  </Accordion>
  <Accordion title="ملاحظة الوكيل الموثوق">
    - يمكن لمصادقة الوكيل الموثوق الناجحة إدخال جلسات واجهة التحكم الخاصة بـ **المشغل** دون هوية جهاز.
    - هذا **لا** يمتد إلى جلسات واجهة التحكم بدور العقدة.
    - لا تزال الوكلاء العكسيون عبر loopback على المضيف نفسه لا يستوفون مصادقة الوكيل الموثوق؛ راجع [مصادقة الوكيل الموثوق](/ar/gateway/trusted-proxy-auth).

  </Accordion>
</AccordionGroup>

راجع [Tailscale](/ar/gateway/tailscale) لإرشادات إعداد HTTPS.

## سياسة أمان المحتوى

تأتي واجهة التحكم بسياسة `img-src` صارمة: لا يُسمح إلا بالأصول من **الأصل نفسه**، وعناوين URL من نوع `data:`، وعناوين URL من نوع `blob:` المنشأة محليًا. يرفض المتصفح عناوين URL البعيدة من نوع `http(s)` والعناوين النسبية للبروتوكول ولا يصدر طلبات شبكة لها.

ما يعنيه ذلك عمليًا:

- تظل الصور الرمزية والصور المقدمة تحت مسارات نسبية (مثل `/avatars/<id>`) تُعرض، بما في ذلك مسارات الصور الرمزية المصادق عليها التي تجلبها الواجهة وتحولها إلى عناوين URL محلية من نوع `blob:`.
- تظل عناوين URL المضمنة `data:image/...` تُعرض (مفيدة للحمولات داخل البروتوكول).
- تظل عناوين URL المحلية من نوع `blob:` التي تنشئها واجهة التحكم تُعرض.
- تُزال عناوين URL البعيدة للصور الرمزية الصادرة عن بيانات تعريف القناة عند مساعدات الصور الرمزية في واجهة التحكم وتُستبدل بالشعار/الشارة المدمجة، بحيث لا تستطيع قناة مخترقة أو خبيثة إجبار متصفح المشغل على جلب صور بعيدة عشوائية.

لا تحتاج إلى تغيير أي شيء للحصول على هذا السلوك — فهو مفعّل دائمًا وغير قابل للتهيئة.

## مصادقة مسار الصورة الرمزية

عند تهيئة مصادقة Gateway، تتطلب نقطة نهاية الصورة الرمزية في واجهة التحكم رمز Gateway نفسه كبقية API:

- يعيد `GET /avatar/<agentId>` صورة الصورة الرمزية فقط للمستدعين المصادق عليهم. يعيد `GET /avatar/<agentId>?meta=1` بيانات تعريف الصورة الرمزية وفق القاعدة نفسها.
- تُرفض الطلبات غير المصادق عليها إلى أي من المسارين (مطابقة لمسار وسائط المساعد الشقيق). يمنع هذا مسار الصورة الرمزية من تسريب هوية الوكيل على المضيفين المحميين خلاف ذلك.
- تمرر واجهة التحكم نفسها رمز Gateway كترويسة bearer عند جلب الصور الرمزية، وتستخدم عناوين URL مصادقًا عليها من نوع blob بحيث تظل الصورة تُعرض في لوحات المعلومات.

إذا عطلت مصادقة Gateway (غير موصى به على المضيفين المشتركة)، يصبح مسار الصورة الرمزية أيضًا غير مصادق عليه، بما يتوافق مع بقية Gateway.

## مصادقة مسار وسائط المساعد

عند تهيئة مصادقة Gateway، تستخدم معاينات الوسائط المحلية للمساعد مسارًا من خطوتين:

- يتطلب `GET /__openclaw__/assistant-media?meta=1&source=<path>` مصادقة مشغل واجهة التحكم العادية. يرسل المتصفح رمز Gateway كترويسة bearer عند التحقق من التوفر.
- تتضمن استجابات البيانات التعريفية الناجحة `mediaTicket` قصيرة العمر ومحددة لنطاق مسار المصدر الدقيق هذا.
- تستخدم عناوين URL للصور والصوت والفيديو والمستندات المعروضة في المتصفح `mediaTicket=<ticket>` بدل رمز Gateway النشط أو كلمة المرور. تنتهي صلاحية التذكرة سريعًا ولا يمكنها تفويض مصدر مختلف.

يحافظ هذا على توافق عرض الوسائط العادي مع عناصر الوسائط الأصلية في المتصفح دون وضع بيانات اعتماد Gateway القابلة لإعادة الاستخدام في عناوين URL المرئية للوسائط.

## بناء الواجهة

يقدم Gateway الملفات الثابتة من `dist/control-ui`. ابنها باستخدام:

```bash
pnpm ui:build
```

قاعدة مطلقة اختيارية (عندما تريد عناوين URL ثابتة للأصول):

```bash
OPENCLAW_CONTROL_UI_BASE_PATH=/openclaw/ pnpm ui:build
```

للتطوير المحلي (خادم تطوير منفصل):

```bash
pnpm ui:dev
```

ثم وجّه الواجهة إلى عنوان URL الخاص بـ Gateway WS لديك (مثل `ws://127.0.0.1:18789`).

## صفحة واجهة تحكم فارغة

إذا حمّل المتصفح لوحة معلومات فارغة ولم تعرض DevTools أي خطأ مفيد، فربما منع امتداد أو سكربت محتوى مبكر تطبيق وحدة JavaScript من التقييم. تتضمن الصفحة الثابتة لوحة استرداد HTML بسيطة تظهر عندما لا يكون `<openclaw-app>` مسجلًا بعد بدء التشغيل.

استخدم إجراء **المحاولة مجددًا** في اللوحة بعد تغيير بيئة المتصفح، أو أعد التحميل يدويًا بعد هذه الفحوصات:

- عطّل الامتدادات التي تحقن في كل الصفحات، وخاصة الامتدادات ذات سكربتات محتوى `<all_urls>`.
- جرّب نافذة خاصة، أو ملف تعريف متصفح نظيفًا، أو متصفحًا آخر.
- أبقِ Gateway قيد التشغيل وتحقق من عنوان URL نفسه للوحة المعلومات بعد تغيير المتصفح.

## التصحيح/الاختبار: خادم التطوير + Gateway بعيد

واجهة التحكم هي ملفات ثابتة؛ هدف WebSocket قابل للتهيئة ويمكن أن يختلف عن أصل HTTP. هذا مفيد عندما تريد خادم تطوير Vite محليًا بينما يعمل Gateway في مكان آخر.

<Steps>
  <Step title="ابدأ خادم تطوير الواجهة">
    ```bash
    pnpm ui:dev
    ```
  </Step>
  <Step title="افتح باستخدام gatewayUrl">
    ```text
    http://localhost:5173/?gatewayUrl=ws%3A%2F%2F<gateway-host>%3A18789
    ```

    مصادقة اختيارية لمرة واحدة (إذا لزم الأمر):

    ```text
    http://localhost:5173/?gatewayUrl=wss%3A%2F%2F<gateway-host>%3A18789#token=<gateway-token>
    ```

  </Step>
</Steps>

<AccordionGroup>
  <Accordion title="ملاحظات">
    - يتم تخزين `gatewayUrl` في localStorage بعد التحميل وإزالته من عنوان URL.
    - إذا مررت نقطة نهاية كاملة بصيغة `ws://` أو `wss://` عبر `gatewayUrl`، فقم بترميز قيمة `gatewayUrl` لعنوان URL حتى يفسر المتصفح سلسلة الاستعلام بشكل صحيح.
    - يجب تمرير `token` عبر جزء عنوان URL (`#token=...`) كلما أمكن. لا تُرسل الأجزاء إلى الخادم، مما يتجنب تسرب سجلات الطلبات وReferer. لا تزال معلمات الاستعلام القديمة `?token=` تُستورد مرة واحدة للتوافق، ولكن كخيار احتياطي فقط، وتُزال فورًا بعد bootstrap.
    - يُحتفظ بـ `password` في الذاكرة فقط.
    - عند تعيين `gatewayUrl`، لا تعود واجهة المستخدم إلى بيانات اعتماد الإعدادات أو البيئة. قدّم `token` (أو `password`) صراحةً. غياب بيانات الاعتماد الصريحة خطأ.
    - استخدم `wss://` عندما يكون Gateway خلف TLS (Tailscale Serve، أو وكيل HTTPS، وما إلى ذلك).
    - لا يُقبل `gatewayUrl` إلا في نافذة من المستوى الأعلى (وليس مضمنًا) لمنع clickjacking.
    - يجب أن تضبط عمليات نشر واجهة مستخدم التحكم العامة غير المعتمدة على loopback `gateway.controlUi.allowedOrigins` صراحةً (أصول كاملة). تُقبل التحميلات الخاصة من LAN/Tailnet ذات الأصل نفسه من loopback أو RFC1918/link-local أو `.local` أو `.ts.net` أو مضيفي Tailscale CGNAT دون تمكين الرجوع إلى ترويسة Host.
    - قد يزرع بدء تشغيل Gateway أصولًا محلية مثل `http://localhost:<port>` و`http://127.0.0.1:<port>` من ربط ومنفذ وقت التشغيل الفعليين، لكن أصول المتصفح البعيدة لا تزال تحتاج إلى إدخالات صريحة.
    - لا تستخدم `gateway.controlUi.allowedOrigins: ["*"]` إلا للاختبار المحلي المحكوم بإحكام. فهذا يعني السماح لأي أصل متصفح، وليس "مطابقة أي مضيف أستخدمه."
    - يفعّل `gateway.controlUi.dangerouslyAllowHostHeaderOriginFallback=true` وضع الرجوع إلى أصل ترويسة Host، لكنه وضع أمني خطير.

  </Accordion>
</AccordionGroup>

مثال:

```json5
{
  gateway: {
    controlUi: {
      allowedOrigins: ["http://localhost:5173"],
    },
  },
}
```
