واجهة التحكم هي تطبيق صغير أحادي الصفحة مبني بـ **Vite + Lit** ويخدمه Gateway:

- الافتراضي: `http://<host>:18789/`
- بادئة اختيارية: عيّن `gateway.controlUi.basePath` (مثل `/openclaw`)

وهي تتواصل **مباشرة مع Gateway WebSocket** على المنفذ نفسه.

## فتح سريع (محلي)

إذا كان Gateway يعمل على الكمبيوتر نفسه، فافتح:

- [http://127.0.0.1:18789/](http://127.0.0.1:18789/) (أو [http://localhost:18789/](http://localhost:18789/))

إذا فشل تحميل الصفحة، فابدأ Gateway أولاً: `openclaw gateway`.

تُقدَّم المصادقة أثناء مصافحة WebSocket عبر:

- `connect.params.auth.token`
- `connect.params.auth.password`
- ترويسات هوية Tailscale Serve عند ضبط `gateway.auth.allowTailscale: true`
- ترويسات هوية الوكيل الموثوق عند ضبط `gateway.auth.mode: "trusted-proxy"`

تحتفظ لوحة إعدادات لوحة المعلومات برمز للجلسة الحالية لعلامة تبويب المتصفح وعنوان URL المحدد للـ gateway؛ ولا تُحفَظ كلمات المرور. عادةً ما تنشئ عملية الإعداد الأولي رمز gateway لمصادقة السر المشترك عند أول اتصال، لكن مصادقة كلمة المرور تعمل أيضاً عندما يكون `gateway.auth.mode` هو `"password"`.

## إقران الجهاز (الاتصال الأول)

عند الاتصال بواجهة التحكم من متصفح أو جهاز جديد، يتطلب Gateway عادةً **موافقة إقران لمرة واحدة**. هذا إجراء أمني لمنع الوصول غير المصرح به.

**ما ستراه:** "disconnected (1008): pairing required"

<Steps>
  <Step title="سرد الطلبات المعلقة">
    ```bash
    openclaw devices list
    ```
  </Step>
  <Step title="الموافقة بواسطة معرّف الطلب">
    ```bash
    openclaw devices approve <requestId>
    ```
  </Step>
</Steps>

إذا أعاد المتصفح محاولة الإقران مع تفاصيل مصادقة متغيرة (الدور/النطاقات/المفتاح العام)، فيتم استبدال الطلب المعلق السابق وإنشاء `requestId` جديد. شغّل `openclaw devices list` مرة أخرى قبل الموافقة.

إذا كان المتصفح مقترناً بالفعل وغيّرته من وصول قراءة إلى وصول كتابة/مسؤول، فسيُعامل ذلك كترقية موافقة، وليس كإعادة اتصال صامتة. يحافظ OpenClaw على الموافقة القديمة نشطة، ويحظر إعادة الاتصال ذات النطاق الأوسع، ويطلب منك الموافقة صراحةً على مجموعة النطاقات الجديدة.

بعد الموافقة، يتم تذكّر الجهاز ولن يتطلب إعادة الموافقة إلا إذا ألغيتَه باستخدام `openclaw devices revoke --device <id> --role <role>`. راجع [CLI الأجهزة](/ar/cli/devices) لتدوير الرموز وإلغائها.

<Note>
- تتم الموافقة تلقائياً على اتصالات متصفح local loopback المباشرة (`127.0.0.1` / `localhost`).
- يمكن لـ Tailscale Serve تخطي جولة الإقران لجلسات مشغّل واجهة التحكم عندما يكون `gateway.auth.allowTailscale: true`، وتتحقق هوية Tailscale، ويعرض المتصفح هوية جهازه.
- لا تزال ارتباطات Tailnet المباشرة، واتصالات متصفح LAN، وملفات تعريف المتصفح بدون هوية جهاز تتطلب موافقة صريحة.
- ينشئ كل ملف تعريف متصفح معرّف جهاز فريداً، لذا فإن تبديل المتصفحات أو مسح بيانات المتصفح سيتطلب إعادة الإقران.

</Note>

## الهوية الشخصية (محلية في المتصفح)

تدعم واجهة التحكم هوية شخصية لكل متصفح (اسم عرض وصورة رمزية) تُرفق بالرسائل الصادرة لإسنادها في الجلسات المشتركة. تعيش هذه الهوية في تخزين المتصفح، وتكون مقصورة على ملف تعريف المتصفح الحالي، ولا تتم مزامنتها مع أجهزة أخرى أو حفظها من جهة الخادم خارج بيانات تعريف مؤلفي النصوص المعتادة للرسائل التي ترسلها فعلياً. يؤدي مسح بيانات الموقع أو تبديل المتصفحات إلى إعادة تعيينها إلى فارغة.

ينطبق نمط التخزين المحلي نفسه في المتصفح على تجاوز صورة المساعد الرمزية. تُطبّق صور المساعد الرمزية المرفوعة فوق الهوية التي حلّها gateway في المتصفح المحلي فقط ولا تُرسل ذهاباً وإياباً عبر `config.patch`. يظل حقل الإعداد المشترك `ui.assistant.avatar` متاحاً للعملاء غير التابعين للواجهة الذين يكتبون الحقل مباشرةً (مثل gateways المبرمجة أو لوحات المعلومات المخصصة).

## نقطة نهاية إعدادات وقت التشغيل

تجلب واجهة التحكم إعدادات وقت التشغيل الخاصة بها من `/__openclaw/control-ui-config.json`. تخضع نقطة النهاية هذه لمصادقة gateway نفسها كبقية سطح HTTP: لا يمكن للمتصفحات غير المصادق عليها جلبها، ويتطلب الجلب الناجح إما رمز/كلمة مرور gateway صالحة بالفعل، أو هوية Tailscale Serve، أو هوية وكيل موثوق.

## دعم اللغة

يمكن لواجهة التحكم توطين نفسها عند أول تحميل بناءً على لغة متصفحك. لتجاوز ذلك لاحقاً، افتح **نظرة عامة -> الوصول إلى Gateway -> اللغة**. يوجد منتقي اللغة في بطاقة الوصول إلى Gateway، وليس ضمن المظهر.

- اللغات المدعومة: `en`, `zh-CN`, `zh-TW`, `pt-BR`, `de`, `es`, `ja-JP`, `ko`, `fr`, `ar`, `it`, `tr`, `uk`, `id`, `pl`, `th`, `vi`, `nl`, `fa`
- تُحمَّل الترجمات غير الإنجليزية كسولياً في المتصفح.
- تُحفَظ اللغة المحددة في تخزين المتصفح وتُعاد استخدامها في الزيارات المستقبلية.
- تعود مفاتيح الترجمة المفقودة إلى الإنجليزية.

يتم إنشاء ترجمات المستندات لمجموعة اللغات غير الإنجليزية نفسها، لكن منتقي اللغة المدمج في موقع المستندات من Mintlify يقتصر على رموز اللغات التي يقبلها Mintlify. لا تزال مستندات التايلاندية (`th`) والفارسية (`fa`) تُنشأ في مستودع النشر؛ وقد لا تظهر في ذلك المنتقي حتى يدعم Mintlify هذه الرموز.

## سمات المظهر

تحتفظ لوحة المظهر بسمات Claw وKnot وDash المدمجة، إضافةً إلى خانة استيراد tweakcn محلية واحدة في المتصفح. لاستيراد سمة، افتح [سمات tweakcn](https://tweakcn.com/themes)، واختر سمة أو أنشئ واحدة، وانقر **مشاركة**، ثم الصق رابط السمة المنسوخ في المظهر. يقبل المستورِد أيضاً عناوين URL للسجل من نمط `https://tweakcn.com/r/themes/<id>`، وعناوين URL للمحرر مثل `https://tweakcn.com/editor/theme?theme=amethyst-haze`، والمسارات النسبية `/themes/<id>`، ومعرّفات السمات الخام، وأسماء السمات الافتراضية مثل `amethyst-haze`.

تُخزَّن السمات المستوردة في ملف تعريف المتصفح الحالي فقط. ولا تُكتب إلى إعدادات gateway ولا تتم مزامنتها عبر الأجهزة. يؤدي استبدال السمة المستوردة إلى تحديث الخانة المحلية الواحدة؛ أما مسحها فيعيد السمة النشطة إلى Claw إذا كانت السمة المستوردة محددة.

## سلوك الدردشة

<AccordionGroup>
  <Accordion title="دلالات الإرسال والسجل">
    - `chat.send` **غير حاظر**: يقر فورا بـ `{ runId, status: "started" }` ويتدفق الرد عبر أحداث `chat`.
    - تقبل عمليات رفع الدردشة الصور إضافة إلى الملفات غير الفيديوية. تحتفظ الصور بمسار الصورة الأصلي؛ أما الملفات الأخرى فتخزن كوسائط مدارة وتظهر في السجل كروابط مرفقات.
    - تؤدي إعادة الإرسال باستخدام `idempotencyKey` نفسه إلى إرجاع `{ status: "in_flight" }` أثناء التشغيل، و`{ status: "ok" }` بعد الاكتمال.
    - تكون ردود `chat.history` محدودة الحجم حفاظا على سلامة واجهة المستخدم. عندما تكون إدخالات النص كبيرة جدا، قد يقتطع Gateway حقول النص الطويلة، ويحذف كتل البيانات الوصفية الثقيلة، ويستبدل الرسائل المفرطة الحجم بعنصر نائب (`[chat.history omitted: message too large]`).
    - تحفظ الصور التي ينشئها المساعد كمرجعيات وسائط مدارة وتقدم مرة أخرى عبر عناوين URL لوسائط Gateway المصادق عليها، بحيث لا تعتمد عمليات إعادة التحميل على بقاء حمولات صور base64 الخام في رد سجل الدردشة.
    - يزيل `chat.history` أيضا وسوم التوجيه المضمنة الخاصة بالعرض فقط من نص المساعد المرئي (مثل `[[reply_to_*]]` و`[[audio_as_voice]]`)، وحمولات XML لاستدعاءات الأدوات بالنص العادي (بما في ذلك `<tool_call>...</tool_call>`، و`<function_call>...</function_call>`، و`<tool_calls>...</tool_calls>`، و`<function_calls>...</function_calls>`، وكتل استدعاء الأدوات المقتطعة)، ورموز تحكم النموذج المسرّبة بنمط ASCII/العرض الكامل، ويحذف إدخالات المساعد التي يكون نصها المرئي بالكامل هو رمز الصمت الدقيق فقط `NO_REPLY` / `no_reply`.
    - أثناء إرسال نشط وتحديث السجل النهائي، تبقي طريقة عرض الدردشة رسائل المستخدم/المساعد المحلية المتفائلة مرئية إذا أعاد `chat.history` لقطة أقدم لفترة وجيزة؛ ويستبدل النص canonical تلك الرسائل المحلية بمجرد أن يلحق سجل Gateway.
    - يضيف `chat.inject` ملاحظة مساعد إلى نص الجلسة ويبث حدث `chat` لتحديثات واجهة المستخدم فقط (لا تشغيل وكيل، ولا تسليم قناة).
    - يحدّث منتقيا النموذج والتفكير في رأس الدردشة الجلسة النشطة فورا عبر `sessions.patch`؛ وهما تجاوزات جلسة دائمة، وليسا خيارات إرسال خاصة بدور واحد فقط.
    - يطلب منتقي نموذج الدردشة طريقة عرض النماذج المكونة في Gateway. إذا كان `agents.defaults.models` موجودا، فإن قائمة السماح تلك تقود المنتقي. وإلا يعرض المنتقي إدخالات `models.providers.*.models` الصريحة إضافة إلى المزودين ذوي المصادقة القابلة للاستخدام. يبقى الفهرس الكامل متاحا عبر RPC التصحيح `models.list` مع `view: "all"`.
    - عندما تعرض تقارير استخدام جلسة Gateway الحديثة ضغط سياق مرتفعا، تعرض منطقة مؤلف الدردشة إشعار سياق، وعند مستويات Compaction الموصى بها، زرا مضغوطا يشغل مسار Compaction العادي للجلسة. تخفى لقطات الرموز القديمة إلى أن يبلغ Gateway عن استخدام حديث مرة أخرى.

  </Accordion>
  <Accordion title="وضع التحدث (الوقت الحقيقي في المتصفح)">
    يستخدم وضع التحدث مزود صوت وقت حقيقي مسجلا. كوّن OpenAI باستخدام `talk.provider: "openai"` إضافة إلى `talk.providers.openai.apiKey`، أو كوّن Google باستخدام `talk.provider: "google"` إضافة إلى `talk.providers.google.apiKey`؛ ولا يزال من الممكن إعادة استخدام إعداد مزود الوقت الحقيقي لمكالمة الصوت كبديل احتياطي. لا يتلقى المتصفح أبدا مفتاح API قياسيا للمزود. يتلقى OpenAI سر عميل Realtime مؤقتا لـ WebRTC. يتلقى Google Live رمز مصادقة Live API مقيدا ومخصصا لاستخدام واحد لجلسة WebSocket في المتصفح، مع تثبيت التعليمات وتعريفات الأدوات داخل الرمز بواسطة Gateway. يعمل المزودون الذين لا يعرضون إلا جسرا خلفيا للوقت الحقيقي عبر نقل ترحيل Gateway، لذلك تبقى بيانات الاعتماد ومقابس المورّد على جانب الخادم بينما ينتقل صوت المتصفح عبر RPCs Gateway المصادق عليها. يجمّع Gateway مطالبة جلسة Realtime؛ ولا يقبل `talk.realtime.session` تجاوزات تعليمات مقدمة من المستدعي.

    في مؤلف الدردشة، عنصر تحكم التحدث هو زر الموجات بجانب زر إملاء الميكروفون. عند بدء التحدث، يعرض صف حالة المؤلف `Connecting Talk...`، ثم `Talk live` أثناء اتصال الصوت، أو `Asking OpenClaw...` أثناء استدعاء أداة في الوقت الحقيقي يستشير النموذج الأكبر المكوّن عبر `chat.send`.

    فحص حي للمشرفين: يتحقق `OPENAI_API_KEY=... GEMINI_API_KEY=... node --import tsx scripts/dev/realtime-talk-live-smoke.ts` من تبادل SDP لـ OpenAI WebRTC في المتصفح، وإعداد WebSocket للمتصفح برمز Google Live المقيد، ومحول متصفح ترحيل Gateway باستخدام وسائط ميكروفون مزيفة. يطبع الأمر حالة المزود فقط ولا يسجل الأسرار.

  </Accordion>
  <Accordion title="الإيقاف والإجهاض">
    - انقر **إيقاف** (يستدعي `chat.abort`).
    - أثناء نشاط تشغيل ما، تصطف المتابعات العادية في الطابور. انقر **توجيه** على رسالة في الطابور لحقن تلك المتابعة في الدور الجاري.
    - اكتب `/stop` (أو عبارات إجهاض مستقلة مثل `stop`، و`stop action`، و`stop run`، و`stop openclaw`، و`please stop`) للإجهاض خارج النطاق.
    - يدعم `chat.abort` الصيغة `{ sessionKey }` (بلا `runId`) لإجهاض كل عمليات التشغيل النشطة لتلك الجلسة.

  </Accordion>
  <Accordion title="الاحتفاظ الجزئي عند الإجهاض">
    - عند إجهاض تشغيل ما، يمكن أن يظل نص المساعد الجزئي ظاهرا في واجهة المستخدم.
    - يحفظ Gateway نص المساعد الجزئي المجهض في سجل النص عندما توجد مخرجات مخزنة مؤقتا.
    - تتضمن الإدخالات المحفوظة بيانات وصفية للإجهاض حتى يتمكن مستهلكو النص من تمييز الجزئيات المجهضة عن مخرجات الاكتمال العادية.

  </Accordion>
</AccordionGroup>

## تثبيت PWA ودفع الويب

تشحن واجهة التحكم `manifest.webmanifest` وعامل خدمة، لذلك يمكن للمتصفحات الحديثة تثبيتها كتطبيق PWA مستقل. يتيح دفع الويب لـ Gateway تنبيه PWA المثبتة بالإشعارات حتى عندما لا تكون علامة التبويب أو نافذة المتصفح مفتوحة.

| السطح                                                | ما يفعله                                                           |
| ----------------------------------------------------- | ------------------------------------------------------------------ |
| `ui/public/manifest.webmanifest`                      | بيان PWA. تعرض المتصفحات "تثبيت التطبيق" بمجرد أن يصبح قابلا للوصول. |
| `ui/public/sw.js`                                     | عامل خدمة يتعامل مع أحداث `push` ونقرات الإشعارات. |
| `push/vapid-keys.json` (ضمن دليل حالة OpenClaw) | زوج مفاتيح VAPID مولد تلقائيا يستخدم لتوقيع حمولات دفع الويب.       |
| `push/web-push-subscriptions.json`                    | نقاط نهاية اشتراك المتصفح المحفوظة.                          |

تجاوز زوج مفاتيح VAPID عبر متغيرات البيئة في عملية Gateway عندما تريد تثبيت المفاتيح (لعمليات نشر متعددة المضيفين، أو تدوير الأسرار، أو الاختبارات):

- `OPENCLAW_VAPID_PUBLIC_KEY`
- `OPENCLAW_VAPID_PRIVATE_KEY`
- `OPENCLAW_VAPID_SUBJECT` (الإعداد الافتراضي هو `mailto:openclaw@localhost`)

تستخدم واجهة التحكم طرق Gateway المقيدة بالنطاق هذه لتسجيل اشتراكات المتصفح واختبارها:

- `push.web.vapidPublicKey` — يجلب مفتاح VAPID العام النشط.
- `push.web.subscribe` — يسجل `endpoint` إضافة إلى `keys.p256dh`/`keys.auth`.
- `push.web.unsubscribe` — يزيل نقطة نهاية مسجلة.
- `push.web.test` — يرسل إشعار اختبار إلى اشتراك المستدعي.

<Note>
دفع الويب مستقل عن مسار ترحيل iOS APNS (راجع [التكوين](/ar/gateway/configuration) للدفع المدعوم بالترحيل) وعن طريقة `push.test` الحالية، التي تستهدف إقران الأجهزة المحمولة الأصلية.
</Note>

## التضمينات المستضافة

يمكن لرسائل المساعد عرض محتوى ويب مستضاف مضمن باستخدام الرمز القصير `[embed ...]`. تتحكم `gateway.controlUi.embedSandbox` في سياسة صندوق حماية iframe:

<Tabs>
  <Tab title="صارم">
    يعطل تنفيذ النصوص البرمجية داخل التضمينات المستضافة.
  </Tab>
  <Tab title="النصوص البرمجية (افتراضي)">
    يسمح بالتضمينات التفاعلية مع إبقاء عزل الأصل؛ وهذا هو الافتراضي وعادة ما يكون كافيا لألعاب/ودجات المتصفح المستقلة بذاتها.
  </Tab>
  <Tab title="موثوق">
    يضيف `allow-same-origin` فوق `allow-scripts` للمستندات من الموقع نفسه التي تحتاج عمدا إلى امتيازات أقوى.
  </Tab>
</Tabs>

مثال:

```json5
{
  gateway: {
    controlUi: {
      embedSandbox: "scripts",
    },
  },
}
```

<Warning>
استخدم `trusted` فقط عندما يحتاج المستند المضمن فعلا إلى سلوك نفس الأصل. لمعظم الألعاب واللوحات التفاعلية التي ينشئها الوكيل، يكون `scripts` هو الخيار الأكثر أمانا.
</Warning>

تبقى عناوين URL الخارجية المطلقة للتضمين من نوع `http(s)` محظورة افتراضيا. إذا كنت تريد عمدا تحميل صفحات طرف ثالث عبر `[embed url="https://..."]`، فعيّن `gateway.controlUi.allowExternalEmbedUrls: true`.

## الوصول عبر tailnet (موصى به)

<Tabs>
  <Tab title="Tailscale Serve متكامل (مفضل)">
    أبق Gateway على local loopback ودع Tailscale Serve يوكله عبر HTTPS:

    ```bash
    openclaw gateway --tailscale serve
    ```

    افتح:

    - `https://<magicdns>/` (أو `gateway.controlUi.basePath` الذي كوّنته)

    افتراضيا، يمكن لمطالبات Control UI/WebSocket Serve المصادقة عبر ترويسات هوية Tailscale (`tailscale-user-login`) عندما تكون `gateway.auth.allowTailscale` هي `true`. يتحقق OpenClaw من الهوية عبر حل عنوان `x-forwarded-for` باستخدام `tailscale whois` ومطابقته مع الترويسة، ولا يقبل هذه إلا عندما تصل المطالبة إلى local loopback مع ترويسات `x-forwarded-*` الخاصة بـ Tailscale. بالنسبة إلى جلسات مشغل Control UI ذات هوية جهاز المتصفح، يتخطى مسار Serve المتحقق منه هذا أيضا رحلة إقران الجهاز؛ ولا تزال المتصفحات بلا جهاز واتصالات دور العقدة تتبع فحوص الجهاز العادية. عيّن `gateway.auth.allowTailscale: false` إذا كنت تريد اشتراط بيانات اعتماد سر مشترك صريحة حتى لحركة Serve. ثم استخدم `gateway.auth.mode: "token"` أو `"password"`.

    بالنسبة إلى مسار هوية Serve غير المتزامن ذاك، تسلسل محاولات المصادقة الفاشلة لعنوان IP العميل نفسه ونطاق المصادقة نفسه قبل عمليات كتابة حد المعدل. لذلك يمكن أن تعرض المحاولات السيئة المتزامنة من المتصفح نفسه `retry later` في الطلب الثاني بدلا من حدوث عدم تطابقين عاديين يتسابقان بالتوازي.

    <Warning>
    تفترض مصادقة Serve بلا رمز أن مضيف Gateway موثوق. إذا كان من الممكن تشغيل كود محلي غير موثوق على ذلك المضيف، فاشترط مصادقة الرمز/كلمة المرور.
    </Warning>

  </Tab>
  <Tab title="الربط بـ tailnet + الرمز">
    ```bash
    openclaw gateway --bind tailnet --token "$(openssl rand -hex 32)"
    ```

    ثم افتح:

    - `http://<tailscale-ip>:18789/` (أو `gateway.controlUi.basePath` الذي كوّنته)

    الصق السر المشترك المطابق في إعدادات واجهة المستخدم (يرسل كـ `connect.params.auth.token` أو `connect.params.auth.password`).

  </Tab>
</Tabs>

## HTTP غير آمن

إذا فتحت لوحة المعلومات عبر HTTP عادي (`http://<lan-ip>` أو `http://<tailscale-ip>`)، يعمل المتصفح في **سياق غير آمن** ويحظر WebCrypto. افتراضيا، **يحظر** OpenClaw اتصالات Control UI من دون هوية جهاز.

الاستثناءات الموثقة:

- توافق HTTP غير الآمن الخاص بـ localhost فقط مع `gateway.controlUi.allowInsecureAuth=true`
- مصادقة Control UI ناجحة للمشغل عبر `gateway.auth.mode: "trusted-proxy"`
- خيار الطوارئ `gateway.controlUi.dangerouslyDisableDeviceAuth=true`

**الإصلاح الموصى به:** استخدم HTTPS (Tailscale Serve) أو افتح واجهة المستخدم محليا:

- `https://<magicdns>/` (Serve)
- `http://127.0.0.1:18789/` (على مضيف Gateway)

<AccordionGroup>
  <Accordion title="سلوك مفتاح المصادقة غير الآمنة">
    ```json5
    {
      gateway: {
        controlUi: { allowInsecureAuth: true },
        bind: "tailnet",
        auth: { mode: "token", token: "replace-me" },
      },
    }
    ```

    `allowInsecureAuth` هو مفتاح توافق محلي فقط:

    - يسمح لجلسات Control UI على localhost بالمتابعة من دون هوية جهاز في سياقات HTTP غير الآمنة.
    - لا يتجاوز فحوص الإقران.
    - لا يخفف متطلبات هوية الجهاز البعيد (غير localhost).

  </Accordion>
  <Accordion title="للطوارئ فقط">
    ```json5
    {
      gateway: {
        controlUi: { dangerouslyDisableDeviceAuth: true },
        bind: "tailnet",
        auth: { mode: "token", token: "replace-me" },
      },
    }
    ```

    <Warning>
    يعطل `dangerouslyDisableDeviceAuth` عمليات تحقق هوية جهاز Control UI، وهو خفض أمني خطير. تراجع عنه بسرعة بعد الاستخدام الطارئ.
    </Warning>

  </Accordion>
  <Accordion title="ملاحظة الوكيل الموثوق">
    - يمكن أن تسمح مصادقة الوكيل الموثوق الناجحة بجلسات Control UI بصلاحية **operator** دون هوية جهاز.
    - لا يمتد هذا إلى جلسات Control UI بدور العقدة.
    - لا تزال وكلاء reverse proxies عبر loopback على المضيف نفسه لا تستوفي مصادقة الوكيل الموثوق؛ راجع [مصادقة الوكيل الموثوق](/ar/gateway/trusted-proxy-auth).

  </Accordion>
</AccordionGroup>

راجع [Tailscale](/ar/gateway/tailscale) للحصول على إرشادات إعداد HTTPS.

## سياسة أمان المحتوى

يأتي Control UI بسياسة `img-src` صارمة: لا يُسمح إلا بالأصول من **نفس المصدر**، وعناوين URL من نوع `data:`، وعناوين URL من نوع `blob:` المنشأة محليًا. يرفض المتصفح عناوين URL البعيدة للصور من نوع `http(s)` والعناوين النسبية إلى البروتوكول، ولا تُرسل طلبات جلب عبر الشبكة.

ما يعنيه ذلك عمليًا:

- تظل الصور الرمزية والصور المقدمة عبر مسارات نسبية (مثل `/avatars/<id>`) معروضة، بما في ذلك مسارات الصور الرمزية المصادق عليها التي يجلبها UI ويحولها إلى عناوين URL محلية من نوع `blob:`.
- تظل عناوين URL المضمنة من نوع `data:image/...` معروضة (مفيدة للحمولات داخل البروتوكول).
- تظل عناوين URL المحلية من نوع `blob:` التي ينشئها Control UI معروضة.
- تُزال عناوين URL البعيدة للصور الرمزية الصادرة من بيانات تعريف القناة في مساعدات الصور الرمزية الخاصة بـ Control UI وتُستبدل بالشعار/الشارة المدمجة، لذلك لا يمكن لقناة مخترقة أو خبيثة فرض عمليات جلب صور بعيدة عشوائية من متصفح المشغل.

لا تحتاج إلى تغيير أي شيء للحصول على هذا السلوك — فهو مفعّل دائمًا وغير قابل للتهيئة.

## مصادقة مسار الصورة الرمزية

عند تهيئة مصادقة Gateway، تتطلب نقطة نهاية الصورة الرمزية في Control UI رمز Gateway نفسه المستخدم في بقية API:

- يعيد `GET /avatar/<agentId>` صورة الصورة الرمزية للمتصلين المصادق عليهم فقط. يعيد `GET /avatar/<agentId>?meta=1` بيانات تعريف الصورة الرمزية وفق القاعدة نفسها.
- تُرفض الطلبات غير المصادق عليها إلى أي من المسارين (مطابقةً لمسار assistant-media الشقيق). يمنع هذا مسار الصورة الرمزية من تسريب هوية الوكيل على المضيفين المحميين بخلاف ذلك.
- يمرر Control UI نفسه رمز Gateway كترويسة bearer عند جلب الصور الرمزية، ويستخدم عناوين URL مصادقًا عليها من نوع blob كي تظل الصورة معروضة في لوحات المعلومات.

إذا عطلت مصادقة Gateway (غير موصى به على المضيفين المشتركة)، يصبح مسار الصورة الرمزية أيضًا غير مصادق عليه، بما يتماشى مع بقية Gateway.

## بناء UI

يقدم Gateway الملفات الثابتة من `dist/control-ui`. ابنها باستخدام:

```bash
pnpm ui:build
```

قاعدة مطلقة اختيارية (عندما تريد عناوين URL ثابتة للأصول):

```bash
OPENCLAW_CONTROL_UI_BASE_PATH=/openclaw/ pnpm ui:build
```

للتطوير المحلي (خادم تطوير منفصل):

```bash
pnpm ui:dev
```

ثم وجّه UI إلى عنوان URL الخاص بـ Gateway WS لديك (مثل `ws://127.0.0.1:18789`).

## التصحيح/الاختبار: خادم التطوير + Gateway بعيد

Control UI هو ملفات ثابتة؛ هدف WebSocket قابل للتهيئة ويمكن أن يكون مختلفًا عن مصدر HTTP. يكون هذا مفيدًا عندما تريد خادم تطوير Vite محليًا بينما يعمل Gateway في مكان آخر.

<Steps>
  <Step title="ابدأ خادم تطوير UI">
    ```bash
    pnpm ui:dev
    ```
  </Step>
  <Step title="افتح باستخدام gatewayUrl">
    ```text
    http://localhost:5173/?gatewayUrl=ws%3A%2F%2F<gateway-host>%3A18789
    ```

    مصادقة اختيارية لمرة واحدة (إذا لزم الأمر):

    ```text
    http://localhost:5173/?gatewayUrl=wss%3A%2F%2F<gateway-host>%3A18789#token=<gateway-token>
    ```

  </Step>
</Steps>

<AccordionGroup>
  <Accordion title="ملاحظات">
    - يُخزن `gatewayUrl` في localStorage بعد التحميل ويُزال من عنوان URL.
    - إذا مررت نقطة نهاية كاملة من نوع `ws://` أو `wss://` عبر `gatewayUrl`، فرمز قيمة `gatewayUrl` على هيئة URL كي يفسر المتصفح سلسلة الاستعلام بشكل صحيح.
    - يجب تمرير `token` عبر جزء URL (`#token=...`) كلما أمكن. لا تُرسل الأجزاء إلى الخادم، ما يتجنب تسربها في سجلات الطلب وReferer. لا تزال معاملات الاستعلام القديمة `?token=` تُستورد مرة واحدة للتوافق، ولكن كخيار احتياطي فقط، وتُزال فورًا بعد bootstrap.
    - يُحتفظ بـ `password` في الذاكرة فقط.
    - عند ضبط `gatewayUrl`، لا يعود UI إلى بيانات اعتماد التهيئة أو البيئة. قدم `token` (أو `password`) صراحةً. غياب بيانات الاعتماد الصريحة خطأ.
    - استخدم `wss://` عندما يكون Gateway خلف TLS (Tailscale Serve، وكيل HTTPS، إلخ).
    - لا يُقبل `gatewayUrl` إلا في نافذة عليا (غير مضمنة) لمنع clickjacking.
    - يجب على عمليات نشر Control UI غير loopback ضبط `gateway.controlUi.allowedOrigins` صراحةً (المصادر الكاملة). يشمل ذلك إعدادات التطوير البعيدة.
    - قد يزرع بدء تشغيل Gateway مصادر محلية مثل `http://localhost:<port>` و`http://127.0.0.1:<port>` من ربط وport وقت التشغيل الفعليين، لكن مصادر المتصفح البعيدة لا تزال تحتاج إلى إدخالات صريحة.
    - لا تستخدم `gateway.controlUi.allowedOrigins: ["*"]` إلا للاختبار المحلي الخاضع لرقابة صارمة. فهذا يعني السماح لأي مصدر متصفح، وليس "مطابقة أي مضيف أستخدمه".
    - يفعّل `gateway.controlUi.dangerouslyAllowHostHeaderOriginFallback=true` وضع الرجوع إلى مصدر ترويسة Host، لكنه وضع أمني خطير.

  </Accordion>
</AccordionGroup>

مثال:

```json5
{
  gateway: {
    controlUi: {
      allowedOrigins: ["http://localhost:5173"],
    },
  },
}
```

تفاصيل إعداد الوصول عن بُعد: [الوصول عن بُعد](/ar/gateway/remote).
