# بيان المكوّن الإضافي (`openclaw.plugin.json`)

هذه الصفحة مخصّصة فقط لـ **بيان المكوّن الإضافي الأصلي في OpenClaw**.

للتعرّف على تخطيطات الحِزم المتوافقة، راجع [حِزم المكوّنات الإضافية](/ar/plugins/bundles).

تستخدم تنسيقات الحِزم المتوافقة ملفات بيان مختلفة:

- حزمة Codex: `.codex-plugin/plugin.json`
- حزمة Claude: `.claude-plugin/plugin.json` أو تخطيط مكوّن Claude الافتراضي
  بدون بيان
- حزمة Cursor: `.cursor-plugin/plugin.json`

يكتشف OpenClaw تخطيطات الحِزم هذه تلقائيًا أيضًا، لكنها لا تُتحقَّق
مطابقتها مع مخطط `openclaw.plugin.json` الموضّح هنا.

بالنسبة إلى الحِزم المتوافقة، يقرأ OpenClaw حاليًا بيانات الحزمة الوصفية بالإضافة إلى
جذور Skills المعلنة، وجذور أوامر Claude، وقيم Claude bundle `settings.json` الافتراضية،
وقيم Claude bundle LSP الافتراضية، وحِزم hooks المدعومة عندما يطابق التخطيط
توقّعات وقت تشغيل OpenClaw.

يجب أن يوفّر كل مكوّن إضافي أصلي في OpenClaw ملف `openclaw.plugin.json` داخل
**جذر المكوّن الإضافي**. يستخدم OpenClaw هذا البيان للتحقق من صحة الإعدادات
**من دون تنفيذ شيفرة المكوّن الإضافي**. وتُعامَل البيانات المفقودة أو غير الصالحة
بوصفها أخطاء في المكوّن الإضافي وتمنع التحقق من صحة الإعدادات.

راجع دليل نظام المكوّنات الإضافية الكامل: [المكوّنات الإضافية](/ar/tools/plugin).
وللاطلاع على نموذج القدرات الأصلي وإرشادات التوافق الخارجي الحالية:
[نموذج القدرات](/ar/plugins/architecture#public-capability-model).

## ما الذي يفعله هذا الملف

`openclaw.plugin.json` هي البيانات الوصفية التي يقرأها OpenClaw قبل تحميل
شيفرة المكوّن الإضافي لديك.

استخدمه من أجل:

- هوية المكوّن الإضافي
- التحقق من صحة الإعدادات
- بيانات auth وonboarding الوصفية التي يجب أن تكون متاحة من دون تشغيل
  وقت تشغيل المكوّن الإضافي
- تلميحات تفعيل منخفضة الكلفة يمكن لواجهات مستوى التحكّم فحصها قبل تحميل
  وقت التشغيل
- واصفات إعداد منخفضة الكلفة يمكن لواجهات الإعداد/التهيئة الأولية فحصها قبل
  تحميل وقت التشغيل
- بيانات وصفية للأسماء البديلة والتفعيل التلقائي يجب حلّها قبل تحميل وقت تشغيل
  المكوّن الإضافي
- بيانات وصفية مختصرة لملكية عائلات النماذج يجب أن تفعّل المكوّن الإضافي
  تلقائيًا قبل تحميل وقت التشغيل
- لقطات ثابتة لملكية القدرات تُستخدم في توصيلات التوافق المضمّنة وتغطية
  العقود
- بيانات وصفية خاصة بإعدادات القنوات يجب دمجها في أسطح الكتالوج والتحقق
  من دون تحميل وقت التشغيل
- تلميحات واجهة المستخدم للإعدادات

لا تستخدمه من أجل:

- تسجيل سلوك وقت التشغيل
- تعريف نقاط إدخال الشيفرة
- بيانات تثبيت npm الوصفية

فهذه تخص شيفرة المكوّن الإضافي لديك و`package.json`.

## مثال بسيط

```json
{
  "id": "voice-call",
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {}
  }
}
```

## مثال غني

```json
{
  "id": "openrouter",
  "name": "OpenRouter",
  "description": "OpenRouter provider plugin",
  "version": "1.0.0",
  "providers": ["openrouter"],
  "modelSupport": {
    "modelPrefixes": ["router-"]
  },
  "cliBackends": ["openrouter-cli"],
  "providerAuthEnvVars": {
    "openrouter": ["OPENROUTER_API_KEY"]
  },
  "providerAuthAliases": {
    "openrouter-coding": "openrouter"
  },
  "channelEnvVars": {
    "openrouter-chatops": ["OPENROUTER_CHATOPS_TOKEN"]
  },
  "providerAuthChoices": [
    {
      "provider": "openrouter",
      "method": "api-key",
      "choiceId": "openrouter-api-key",
      "choiceLabel": "OpenRouter API key",
      "groupId": "openrouter",
      "groupLabel": "OpenRouter",
      "optionKey": "openrouterApiKey",
      "cliFlag": "--openrouter-api-key",
      "cliOption": "--openrouter-api-key <key>",
      "cliDescription": "OpenRouter API key",
      "onboardingScopes": ["text-inference"]
    }
  ],
  "uiHints": {
    "apiKey": {
      "label": "API key",
      "placeholder": "sk-or-v1-...",
      "sensitive": true
    }
  },
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {
      "apiKey": {
        "type": "string"
      }
    }
  }
}
```

| `legacyPluginIds`                   | لا     | `string[]`                       | معرّفات قديمة تُطبَّع إلى هذا المعرّف القياسي للمكوّن الإضافي.                                                                                                                                               |

| `kind`                              | لا     | `"memory"` \| `"context-engine"` | يعرّف نوعًا حصريًا للمكوّن الإضافي يُستخدم بواسطة `plugins.slots.*`.                                                                                                                                        |
| `channels`                          | لا     | `string[]`                       | معرّفات القنوات التي يملكها هذا المكوّن الإضافي. تُستخدم للاكتشاف والتحقق من صحة الإعدادات.                                                                                                                 |

| `modelSupport`                      | لا     | `object`                         | بيانات وصفية مختصرة لعائلات النماذج مملوكة للبيان وتُستخدم لتحميل المكوّن الإضافي تلقائيًا قبل وقت التشغيل.                                                                                                 |
| `cliBackends`                       | لا     | `string[]`                       | معرّفات واجهات CLI الخلفية للاستدلال التي يملكها هذا المكوّن الإضافي. تُستخدم للتفعيل التلقائي عند بدء التشغيل من مراجع الإعدادات الصريحة.                                                                  |
| `commandAliases`                    | لا     | `object[]`                       | أسماء الأوامر التي يملكها هذا المكوّن الإضافي والتي يجب أن تنتج إعدادات وتشخيصات CLI مدركة للمكوّن الإضافي قبل تحميل وقت التشغيل.                                                                            |
| `providerAuthEnvVars`               | لا     | `Record<string, string[]>`       | بيانات وصفية منخفضة الكلفة لمتغيرات بيئة auth الخاصة بالمزوّد يمكن لـ OpenClaw فحصها من دون تحميل شيفرة المكوّن الإضافي.                                                                                   |
| `providerAuthAliases`               | لا     | `Record<string, string>`         | معرّفات مزوّدين يجب أن تعيد استخدام معرّف مزوّد آخر لبحث auth، مثل مزوّد برمجي يشارك مفتاح API الأساسي وملفات تعريف auth الخاصة بالمزوّد الأساسي.                                                          |

| `skills`                            | لا     | `string[]`                       | أدلة Skills المطلوب تحميلها، نسبةً إلى جذر المكوّن الإضافي.                                                                                                                                                 |
| `name`                              | لا     | `string`                         | اسم مقروء للمكوّن الإضافي.                                                                                                                                                                                   |
| `description`                       | لا     | `string`                         | ملخص قصير يُعرض في أسطح المكوّنات الإضافية.                                                                                                                                                                  |

| `uiHints`                           | لا     | `Record<string, object>`         | تسميات واجهة المستخدم، والعناصر النائبة، وتلميحات الحساسية لحقول الإعدادات.                                                                                                                                  |

## مرجع `providerAuthChoices`

يصف كل إدخال في `providerAuthChoices` خيارًا واحدًا من خيارات onboarding أو auth.
يقرأ OpenClaw هذا قبل تحميل وقت تشغيل المزوّد.

| الحقل                 | مطلوب | النوع                                           | المعنى                                                                                           |
| --------------------- | ------ | ----------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| `provider`            | نعم    | `string`                                        | معرّف المزوّد الذي ينتمي إليه هذا الخيار.                                                         |
| `method`              | نعم    | `string`                                        | معرّف طريقة auth التي سيتم التوجيه إليها.                                                         |
| `choiceId`            | نعم    | `string`                                        | معرّف ثابت لخيار auth يُستخدم بواسطة تدفقات onboarding وCLI.                                      |
| `choiceLabel`         | لا     | `string`                                        | تسمية موجّهة للمستخدم. إذا حُذفت، يستخدم OpenClaw القيمة `choiceId` كبديل.                      |
| `choiceHint`          | لا     | `string`                                        | نص مساعد قصير لمنتقي الخيارات.                                                                   |
| `assistantPriority`   | لا     | `number`                                        | القيم الأقل تظهر أولًا في منتقيات الخيارات التفاعلية التي يقودها المساعد.                       |
| `assistantVisibility` | لا     | `"visible"` \| `"manual-only"`                  | يُخفي الخيار من منتقيات المساعد مع الاستمرار في السماح باختياره يدويًا عبر CLI.                 |
| `deprecatedChoiceIds` | لا     | `string[]`                                      | معرّفات قديمة لخيارات يجب أن تعيد توجيه المستخدمين إلى هذا الخيار البديل.                        |
| `groupId`             | لا     | `string`                                        | معرّف مجموعة اختياري لتجميع الخيارات المرتبطة.                                                   |
| `groupLabel`          | لا     | `string`                                        | تسمية موجّهة للمستخدم لتلك المجموعة.                                                             |
| `groupHint`           | لا     | `string`                                        | نص مساعد قصير للمجموعة.                                                                          |
| `optionKey`           | لا     | `string`                                        | مفتاح خيار داخلي لتدفقات auth البسيطة التي تستخدم علمًا واحدًا.                                 |
| `cliFlag`             | لا     | `string`                                        | اسم علم CLI، مثل `--openrouter-api-key`.                                                         |
| `cliOption`           | لا     | `string`                                        | الشكل الكامل لخيار CLI، مثل `--openrouter-api-key <key>`.                                        |
| `cliDescription`      | لا     | `string`                                        | الوصف المستخدم في تعليمات CLI.                                                                   |
| `onboardingScopes`    | لا     | `Array<"text-inference" \| "image-generation">` | أسطح onboarding التي يجب أن يظهر فيها هذا الخيار. إذا حُذفت، تكون القيمة الافتراضية `["text-inference"]`. |

## مرجع `commandAliases`

استخدم `commandAliases` عندما يملك المكوّن الإضافي اسم أمر وقت تشغيل قد يضعه
المستخدمون بالخطأ في `plugins.allow` أو يحاولون تشغيله كأمر CLI جذري. يستخدم OpenClaw
هذه البيانات الوصفية للتشخيصات من دون استيراد شيفرة وقت تشغيل المكوّن الإضافي.

```json
{
  "commandAliases": [
    {
      "name": "dreaming",
      "kind": "runtime-slash",
      "cliCommand": "memory"
    }
  ]
}
```

| الحقل        | مطلوب | النوع             | المعنى                                                                |
| ------------ | ------ | ----------------- | --------------------------------------------------------------------- |
| `name`       | نعم    | `string`          | اسم الأمر الذي ينتمي إلى هذا المكوّن الإضافي.                         |
| `kind`       | لا     | `"runtime-slash"` | يحدّد أن الاسم البديل هو أمر slash في الدردشة وليس أمر CLI جذريًا.    |
| `cliCommand` | لا     | `string`          | أمر CLI جذري ذو صلة يُقترح لعمليات CLI، إذا كان موجودًا.             |

## مرجع `activation`

استخدم `activation` عندما يكون بإمكان المكوّن الإضافي أن يحدّد بتكلفة منخفضة
أحداث مستوى التحكّم التي ينبغي أن تفعّله لاحقًا.

هذه الكتلة هي بيانات وصفية فقط. فهي لا تسجّل سلوك وقت التشغيل، ولا
تستبدل `register(...)` أو `setupEntry` أو نقاط إدخال وقت التشغيل/المكوّن الإضافي الأخرى.

```json
{
  "activation": {
    "onProviders": ["openai"],
    "onCommands": ["models"],
    "onChannels": ["web"],
    "onRoutes": ["gateway-webhook"],
    "onCapabilities": ["provider", "tool"]
  }
}
```

| الحقل            | مطلوب | النوع                                                | المعنى                                                       |
| ---------------- | ------ | ---------------------------------------------------- | ------------------------------------------------------------ |
| `onProviders`    | لا     | `string[]`                                           | معرّفات المزوّدين التي يجب أن تفعّل هذا المكوّن الإضافي عند طلبها. |
| `onCommands`     | لا     | `string[]`                                           | معرّفات الأوامر التي يجب أن تفعّل هذا المكوّن الإضافي.            |
| `onChannels`     | لا     | `string[]`                                           | معرّفات القنوات التي يجب أن تفعّل هذا المكوّن الإضافي.            |
| `onRoutes`       | لا     | `string[]`                                           | أنواع المسارات التي يجب أن تفعّل هذا المكوّن الإضافي.            |
| `onCapabilities` | لا     | `Array<"provider" \| "channel" \| "tool" \| "hook">` | تلميحات قدرات عامة تُستخدم في تخطيط التفعيل على مستوى التحكّم.   |

## مرجع `setup`

استخدم `setup` عندما تحتاج أسطح الإعداد وonboarding إلى بيانات وصفية
منخفضة الكلفة يملكها المكوّن الإضافي قبل تحميل وقت التشغيل.

```json
{
  "setup": {
    "providers": [
      {
        "id": "openai",
        "authMethods": ["api-key"],
        "envVars": ["OPENAI_API_KEY"]
      }
    ],
    "cliBackends": ["openai-cli"],
    "configMigrations": ["legacy-openai-auth"],
    "requiresRuntime": false
  }
}
```

يبقى `cliBackends` من المستوى الأعلى صالحًا، ويستمر في وصف
واجهات CLI الخلفية للاستدلال. أمّا `setup.cliBackends` فهو سطح الواصفات الخاص بالإعداد
لتدفقات مستوى التحكّم/الإعداد التي يجب أن تظل بيانات وصفية فقط.

عند وجودها، تكون `setup.providers` و`setup.cliBackends` هما سطح البحث المفضّل
القائم على الواصفات أولًا لاكتشاف الإعداد. وإذا كانت الواصفة لا تقوم إلا
بتضييق نطاق المكوّن الإضافي المرشّح وكان الإعداد ما يزال يحتاج إلى hooks
أغنى في وقت الإعداد، فاضبط `requiresRuntime: true` وأبقِ `setup-api` في مكانه
بوصفه مسار التنفيذ الاحتياطي.

ولأن البحث عن الإعداد يمكنه تنفيذ شيفرة `setup-api` التي يملكها المكوّن الإضافي،
فيجب أن تبقى القيم المطبّعة في `setup.providers[].id` و`setup.cliBackends[]`
فريدة عبر المكوّنات الإضافية المكتشفة. وتفشل الملكية الملتبسة بشكل مغلق بدلًا من
اختيار فائز استنادًا إلى ترتيب الاكتشاف.

### مرجع `setup.providers`

| الحقل         | مطلوب | النوع      | المعنى                                                                          |
| ------------- | ------ | ---------- | ------------------------------------------------------------------------------- |
| `id`          | نعم    | `string`   | معرّف المزوّد الذي يُكشف أثناء الإعداد أو onboarding. أبقِ المعرّفات المطبّعة فريدة عالميًا. |
| `authMethods` | لا     | `string[]` | معرّفات طرق الإعداد/‏auth التي يدعمها هذا المزوّد من دون تحميل وقت التشغيل الكامل.       |
| `envVars`     | لا     | `string[]` | متغيرات البيئة التي يمكن لأسطح الإعداد/الحالة العامة فحصها قبل تحميل وقت تشغيل المكوّن الإضافي. |

### حقول `setup`

| الحقل              | مطلوب | النوع      | المعنى                                                                                          |
| ------------------ | ------ | ---------- | ----------------------------------------------------------------------------------------------- |
| `providers`        | لا     | `object[]` | واصفات إعداد المزوّدين التي تُكشف أثناء الإعداد وonboarding.                                   |
| `cliBackends`      | لا     | `string[]` | معرّفات الواجهات الخلفية في وقت الإعداد والمستخدمة للبحث القائم على الواصفات أولًا. أبقِ المعرّفات المطبّعة فريدة عالميًا. |
| `configMigrations` | لا     | `string[]` | معرّفات ترحيل الإعدادات التي يملكها سطح الإعداد الخاص بهذا المكوّن الإضافي.                   |
| `requiresRuntime`  | لا     | `boolean`  | ما إذا كان الإعداد ما يزال يحتاج إلى تنفيذ `setup-api` بعد البحث بالاعتماد على الواصفات.      |

## مرجع `uiHints`

`uiHints` هي خريطة من أسماء حقول الإعدادات إلى تلميحات عرض صغيرة.

```json
{
  "uiHints": {
    "apiKey": {
      "label": "API key",
      "help": "Used for OpenRouter requests",
      "placeholder": "sk-or-v1-...",
      "sensitive": true
    }
  }
}
```

يمكن أن يتضمّن تلميح كل حقل ما يلي:

| الحقل         | النوع      | المعنى                                |
| ------------- | ---------- | ------------------------------------- |
| `label`       | `string`   | تسمية الحقل الموجّهة للمستخدم.         |
| `help`        | `string`   | نص مساعد قصير.                        |
| `tags`        | `string[]` | وسوم واجهة مستخدم اختيارية.            |
| `advanced`    | `boolean`  | يحدّد أن الحقل متقدّم.                 |
| `sensitive`   | `boolean`  | يحدّد أن الحقل سرّي أو حساس.           |
| `placeholder` | `string`   | نص العنصر النائب لمدخلات النماذج.      |

## مرجع `contracts`

استخدم `contracts` فقط لبيانات وصفية ثابتة لملكية القدرات يمكن لـ OpenClaw
قراءتها من دون استيراد وقت تشغيل المكوّن الإضافي.

```json
{
  "contracts": {
    "speechProviders": ["openai"],
    "realtimeTranscriptionProviders": ["openai"],
    "realtimeVoiceProviders": ["openai"],
    "mediaUnderstandingProviders": ["openai", "openai-codex"],
    "imageGenerationProviders": ["openai"],
    "videoGenerationProviders": ["qwen"],
    "webFetchProviders": ["firecrawl"],
    "webSearchProviders": ["gemini"],
    "tools": ["firecrawl_search", "firecrawl_scrape"]
  }
}
```

كل قائمة اختيارية:

| الحقل                            | النوع      | المعنى                                                     |
| -------------------------------- | ---------- | ---------------------------------------------------------- |
| `speechProviders`                | `string[]` | معرّفات مزوّدي الكلام التي يملكها هذا المكوّن الإضافي.     |
| `realtimeTranscriptionProviders` | `string[]` | معرّفات مزوّدي النسخ الفوري التي يملكها هذا المكوّن الإضافي. |
| `realtimeVoiceProviders`         | `string[]` | معرّفات مزوّدي الصوت الفوري التي يملكها هذا المكوّن الإضافي. |
| `mediaUnderstandingProviders`    | `string[]` | معرّفات مزوّدي فهم الوسائط التي يملكها هذا المكوّن الإضافي. |
| `imageGenerationProviders`       | `string[]` | معرّفات مزوّدي توليد الصور التي يملكها هذا المكوّن الإضافي. |
| `videoGenerationProviders`       | `string[]` | معرّفات مزوّدي توليد الفيديو التي يملكها هذا المكوّن الإضافي. |
| `webFetchProviders`              | `string[]` | معرّفات مزوّدي جلب الويب التي يملكها هذا المكوّن الإضافي.  |
| `webSearchProviders`             | `string[]` | معرّفات مزوّدي البحث في الويب التي يملكها هذا المكوّن الإضافي. |
| `tools`                          | `string[]` | أسماء أدوات الوكيل التي يملكها هذا المكوّن الإضافي لفحوصات العقود المضمّنة. |

## مرجع `channelConfigs`

استخدم `channelConfigs` عندما يحتاج مكوّن إضافي خاص بالقنوات إلى بيانات وصفية منخفضة الكلفة للإعدادات قبل
تحميل وقت التشغيل.

```json
{
  "channelConfigs": {
    "matrix": {
      "schema": {
        "type": "object",
        "additionalProperties": false,
        "properties": {
          "homeserverUrl": { "type": "string" }
        }
      },
      "uiHints": {
        "homeserverUrl": {
          "label": "Homeserver URL",
          "placeholder": "https://matrix.example.com"
        }
      },
      "label": "Matrix",
      "description": "Matrix homeserver connection",
      "preferOver": ["matrix-legacy"]
    }
  }
}
```

يمكن أن يتضمّن كل إدخال قناة ما يلي:

| الحقل         | النوع                    | المعنى                                                                                   |
| ------------- | ------------------------ | ---------------------------------------------------------------------------------------- |
| `schema`      | `object`                 | مخطط JSON لـ `channels.<id>`. وهو مطلوب لكل إدخال مُعلَن لإعدادات القنوات.              |
| `uiHints`     | `Record<string, object>` | تسميات واجهة مستخدم اختيارية/عناصر نائبة/تلميحات حساسية لقسم إعدادات تلك القناة.         |
| `label`       | `string`                 | تسمية القناة التي تُدمج في أسطح المنتقي والفحص عندما لا تكون بيانات وقت التشغيل الوصفية جاهزة. |
| `description` | `string`                 | وصف قصير للقناة لأسطح الفحص والكتالوج.                                                   |
| `preferOver`  | `string[]`               | معرّفات مكوّنات إضافية قديمة أو أقل أولوية ينبغي أن تتفوّق عليها هذه القناة في أسطح الاختيار. |

## مرجع `modelSupport`

استخدم `modelSupport` عندما ينبغي أن يستنتج OpenClaw مكوّن المزوّد الإضافي الخاص بك من
معرّفات نماذج مختصرة مثل `gpt-5.4` أو `claude-sonnet-4.6` قبل أن يُحمَّل وقت تشغيل المكوّن الإضافي.

```json
{
  "modelSupport": {
    "modelPrefixes": ["gpt-", "o1", "o3", "o4"],
    "modelPatterns": ["^computer-use-preview"]
  }
}
```

يطبّق OpenClaw ترتيب الأولوية هذا:

- تستخدم مراجع `provider/model` الصريحة بيانات البيان الوصفية `providers` التابعة للمالك
- تتقدّم `modelPatterns` على `modelPrefixes`
- إذا طابق مكوّن إضافي واحد غير مضمّن ومكوّن إضافي واحد مضمّن معًا، يفوز المكوّن الإضافي غير المضمّن
- يُتجاهَل أي غموض متبقٍ حتى يحدّد المستخدم أو الإعداد مزوّدًا

الحقول:

| الحقل           | النوع      | المعنى                                                                     |
| --------------- | ---------- | -------------------------------------------------------------------------- |
| `modelPrefixes` | `string[]` | بادئات تُطابَق باستخدام `startsWith` مع معرّفات النماذج المختصرة.         |
| `modelPatterns` | `string[]` | مصادر regex تُطابَق مع معرّفات النماذج المختصرة بعد إزالة لاحقة الملف الشخصي. |

مفاتيح القدرات القديمة من المستوى الأعلى أصبحت مهجورة. استخدم `openclaw doctor --fix` من أجل
نقل `speechProviders` و`realtimeTranscriptionProviders` و`realtimeVoiceProviders` و`mediaUnderstandingProviders`
و`imageGenerationProviders` و`videoGenerationProviders` و`webFetchProviders` و`webSearchProviders`
إلى `contracts`؛ إذ لم يعد تحميل البيان العادي يتعامل مع تلك الحقول من المستوى الأعلى
على أنها ملكية للقدرات.

## البيان مقابل package.json

يؤدي الملفان وظيفتين مختلفتين:

| الملف                  | الاستخدام                                                                                                                         |
| ---------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| `openclaw.plugin.json` | الاكتشاف، والتحقق من صحة الإعدادات، والبيانات الوصفية لخيارات auth، وتلميحات واجهة المستخدم التي يجب أن تكون موجودة قبل تشغيل شيفرة المكوّن الإضافي |
| `package.json`         | بيانات npm الوصفية، وتثبيت التبعيات، وكتلة `openclaw` المستخدمة لنقاط الإدخال، وتقييد التثبيت، والإعداد، أو بيانات الكتالوج الوصفية |

إذا لم تكن متأكدًا من موضع قطعة من البيانات الوصفية، فاستخدم هذه القاعدة:

- إذا كان يجب على OpenClaw معرفتها قبل تحميل شيفرة المكوّن الإضافي، فضعها في `openclaw.plugin.json`
- إذا كانت تتعلق بالتغليف، أو ملفات الإدخال، أو سلوك تثبيت npm، فضعها في `package.json`

### حقول `package.json` التي تؤثر في الاكتشاف

توجد بعض البيانات الوصفية للمكوّن الإضافي قبل وقت التشغيل عمدًا في `package.json` ضمن
كتلة `openclaw` بدلًا من `openclaw.plugin.json`.

أمثلة مهمة:

| الحقل                                                             | المعنى                                                                                                                                      |
| ----------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| `openclaw.extensions`                                             | يعرّف نقاط إدخال المكوّنات الإضافية الأصلية.                                                                                                |
| `openclaw.setupEntry`                                             | نقطة إدخال خفيفة للإعداد فقط تُستخدم أثناء onboarding وبدء تشغيل القنوات المؤجّل.                                                           |
| `openclaw.channel`                                                | بيانات وصفية خفيفة لكتالوج القنوات مثل التسميات، ومسارات التوثيق، والأسماء البديلة، ونصوص الاختيار.                                       |
| `openclaw.channel.configuredState`                                | بيانات وصفية خفيفة لفاحص حالة الإعداد يمكنها الإجابة عن سؤال "هل يوجد إعداد قائم على البيئة فقط بالفعل؟" من دون تحميل وقت تشغيل القناة الكامل. |
| `openclaw.channel.persistedAuthState`                             | بيانات وصفية خفيفة لفاحص auth المخزَّن يمكنها الإجابة عن سؤال "هل توجد أي حالة تسجيل دخول بالفعل؟" من دون تحميل وقت تشغيل القناة الكامل.     |
| `openclaw.install.npmSpec` / `openclaw.install.localPath`         | تلميحات التثبيت/التحديث للمكوّنات الإضافية المضمّنة والمنشورة خارجيًا.                                                                      |
| `openclaw.install.defaultChoice`                                  | مسار التثبيت المفضّل عند توفّر عدة مصادر تثبيت.                                                                                            |
| `openclaw.install.minHostVersion`                                 | الحد الأدنى لإصدار مضيف OpenClaw المدعوم، باستخدام حد أدنى semver مثل `>=2026.3.22`.                                                      |
| `openclaw.install.allowInvalidConfigRecovery`                     | يسمح بمسار استرداد ضيّق لإعادة تثبيت المكوّنات الإضافية المضمّنة عندما تكون الإعدادات غير صالحة.                                           |
| `openclaw.startup.deferConfiguredChannelFullLoadUntilAfterListen` | يسمح بتحميل أسطح القنوات الخاصة بالإعداد فقط قبل المكوّن الإضافي الكامل للقناة أثناء بدء التشغيل.                                           |

يُفرَض `openclaw.install.minHostVersion` أثناء التثبيت وتحميل سجل البيانات.
وتُرفَض القيم غير الصالحة؛ أما القيم الأحدث ولكن الصالحة فتتجاوز
المكوّن الإضافي على المضيفين الأقدم.

إن `openclaw.install.allowInvalidConfigRecovery` ضيّق عمدًا. فهو
لا يجعل الإعدادات المعطوبة العشوائية قابلة للتثبيت. حاليًا لا يسمح إلا
لتدفقات التثبيت بالتعافي من أعطال ترقية محددة قديمة في المكوّنات الإضافية المضمّنة،
مثل مسار مكوّن إضافي مضمّن مفقود أو إدخال `channels.<id>` قديم لذلك
المكوّن الإضافي المضمّن نفسه. أما أخطاء الإعدادات غير المرتبطة فما تزال تمنع التثبيت
وتوجّه المشغّلين إلى `openclaw doctor --fix`.

إن `openclaw.channel.persistedAuthState` هي بيانات حزمة وصفية لوحدة فحص صغيرة:

```json
{
  "openclaw": {
    "channel": {
      "id": "whatsapp",
      "persistedAuthState": {
        "specifier": "./auth-presence",
        "exportName": "hasAnyWhatsAppAuth"
      }
    }
  }
}
```

استخدمها عندما تحتاج تدفقات الإعداد، أو Doctor، أو الحالة المُعدّة إلى
فحص auth رخيص من نوع نعم/لا قبل تحميل المكوّن الإضافي الكامل للقناة. يجب أن يكون التصدير الهدف
دالة صغيرة تقرأ الحالة المخزَّنة فقط؛ لا توجّهها عبر الشريط الكامل لوقت تشغيل
القناة.

يتبع `openclaw.channel.configuredState` الشكل نفسه لفحوصات الإعداد الخفيفة
المعتمدة على البيئة فقط:

```json
{
  "openclaw": {
    "channel": {
      "id": "telegram",
      "configuredState": {
        "specifier": "./configured-state",
        "exportName": "hasTelegramConfiguredState"
      }
    }
  }
}
```

استخدمه عندما تتمكن قناة من الإجابة عن حالة الإعداد من البيئة أو من
مدخلات صغيرة أخرى غير تشغيلية. وإذا كان الفحص يحتاج إلى حلّ كامل للإعدادات أو إلى
وقت تشغيل القناة الفعلي، فأبقِ ذلك المنطق داخل hook
`config.hasConfiguredState` الخاص بالمكوّن الإضافي بدلًا من ذلك.

## متطلبات مخطط JSON

- **يجب على كل مكوّن إضافي توفير مخطط JSON**، حتى لو كان لا يقبل أي إعدادات.
- يُقبل مخطط فارغ (على سبيل المثال، `{ "type": "object", "additionalProperties": false }`).
- يجري التحقق من المخططات عند وقت قراءة/كتابة الإعدادات، وليس عند وقت التشغيل.

## سلوك التحقق

- مفاتيح `channels.*` غير المعروفة تُعد **أخطاء**، ما لم يكن معرّف القناة مُعلَنًا من
  خلال بيان مكوّن إضافي.
- يجب أن تشير `plugins.entries.<id>` و`plugins.allow` و`plugins.deny` و`plugins.slots.*`
  إلى معرّفات مكوّنات إضافية **قابلة للاكتشاف**. والمعرّفات غير المعروفة تُعد **أخطاء**.
- إذا كان المكوّن الإضافي مثبّتًا لكن بيانه أو مخططه معطوبًا أو مفقودًا،
  يفشل التحقق ويُبلِغ Doctor عن خطأ المكوّن الإضافي.
- إذا كانت إعدادات المكوّن الإضافي موجودة لكن المكوّن الإضافي **معطّل**،
  فسيُحتفَظ بالإعدادات وتظهر **تحذير** في Doctor + السجلات.

راجع [مرجع الإعدادات](/ar/gateway/configuration) للاطلاع على مخطط `plugins.*` الكامل.

## ملاحظات

- البيان **مطلوب للمكوّنات الإضافية الأصلية في OpenClaw**، بما في ذلك الأحمال المحلية من نظام الملفات.
- ما يزال وقت التشغيل يحمّل وحدة المكوّن الإضافي بشكل منفصل؛ البيان مخصّص فقط
  للاكتشاف + التحقق.
- تُحلَّل البيانات الأصلية باستخدام JSON5، لذا تُقبل التعليقات، والفواصل اللاحقة،
  والمفاتيح غير الموضوعة بين علامتَي اقتباس طالما أن القيمة النهائية ما تزال كائنًا.
- لا يقرأ محمّل البيان إلا حقول البيان الموثّقة. تجنّب إضافة
  مفاتيح مخصّصة من المستوى الأعلى هنا.
- `providerAuthEnvVars` هو مسار البيانات الوصفية الخفيف لفحوصات auth، والتحقق من
  علامات البيئة، والأسطح المماثلة لـ auth الخاصة بالمزوّد التي يجب ألا تشغّل
  وقت تشغيل المكوّن الإضافي لمجرد فحص أسماء البيئة.
- يتيح `providerAuthAliases` لمتغيرات المزوّد إعادة استخدام auth الخاصة بمزوّد آخر
  env vars وملفات تعريف auth وauth المعتمدة على الإعدادات وخيار onboarding لمفتاح API
  من دون ترميز هذه العلاقة بشكل ثابت في النواة.
- `channelEnvVars` هو مسار البيانات الوصفية الخفيف للاحتياط المعتمد على shell-env، ورسائل
  الإعداد، والأسطح المشابهة للقنوات التي يجب ألا تشغّل وقت تشغيل المكوّن الإضافي
  لمجرد فحص أسماء البيئة.
- `providerAuthChoices` هو مسار البيانات الوصفية الخفيف لمنتقيات خيارات auth،
  وحلّ `--auth-choice`، وربط المزوّد المفضّل، وتسجيل أعلام CLI البسيطة الخاصة بـ onboarding
  قبل تحميل وقت تشغيل المزوّد. أما بيانات معالج وقت التشغيل الوصفية
  التي تتطلب شيفرة مزوّد، فراجع
  [hooks وقت تشغيل المزوّد](/ar/plugins/architecture#provider-runtime-hooks).
- تُحدَّد أنواع المكوّنات الإضافية الحصرية من خلال `plugins.slots.*`.
  - يُحدَّد `kind: "memory"` بواسطة `plugins.slots.memory`.
  - يُحدَّد `kind: "context-engine"` بواسطة `plugins.slots.contextEngine`
    (الافتراضي: `legacy` المضمّن).
- يمكن حذف `channels` و`providers` و`cliBackends` و`skills` عندما لا يحتاج
  المكوّن الإضافي إليها.
- إذا كان المكوّن الإضافي لديك يعتمد على وحدات أصلية، فوثّق خطوات البناء وأي
  متطلبات قائمة سماح لمدير الحزم (على سبيل المثال، pnpm `allow-build-scripts`
  - `pnpm rebuild <package>`).
