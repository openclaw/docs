يسمح Plugin `codex` المضمّن لـ OpenClaw بتشغيل دورات وكيل OpenAI مضمنة
عبر خادم تطبيق Codex بدلا من حزمة OpenClaw المدمجة.

استخدم حزمة Codex عندما تريد أن يتولى Codex جلسة الوكيل منخفضة المستوى:
استئناف السلسلة الأصلي، ومتابعة الأدوات الأصلية، وCompaction الأصلي، وتنفيذ
خادم التطبيق. يظل OpenClaw مسؤولا عن قنوات الدردشة، وملفات الجلسات، واختيار
النموذج، وأدوات OpenClaw الديناميكية، والموافقات، وتسليم الوسائط، ونسخة السجل
المرئية.

يستخدم الإعداد العادي مراجع نماذج OpenAI القياسية مثل `openai/gpt-5.5`.
لا تضبط مراجع GPT القديمة الخاصة بـ Codex. ضع ترتيب مصادقة وكيل OpenAI
تحت `auth.order.openai`؛ أما معرّفات ملفات تعريف مصادقة Codex القديمة وإدخالات
ترتيب مصادقة Codex القديمة فهي حالة قديمة يصلحها
`openclaw doctor --fix`.

عندما لا يكون صندوق رمل OpenClaw نشطا، يبدأ OpenClaw سلاسل خادم تطبيق Codex
مع تفعيل وضع الكود الأصلي في Codex مع إبقاء وضع الكود فقط معطلا افتراضيا.
يحافظ ذلك على إتاحة مساحة عمل Codex الأصلية وقدرات الكود، بينما تستمر أدوات
OpenClaw الديناميكية عبر جسر خادم التطبيق `item/tool/call`.
تعطل سياسات وضع صندوق رمل OpenClaw النشطة وسياسات الأدوات المقيدة وضع الكود
الأصلي بالكامل إلا إذا اخترت مسار خادم تنفيذ صندوق الرمل التجريبي.

هذه الميزة الأصلية في Codex منفصلة عن
[وضع كود OpenClaw](/ar/reference/code-mode)، وهو وقت تشغيل QuickJS-WASI اختياري
لتشغيلات OpenClaw العامة مع شكل إدخال `exec` مختلف.

للاطلاع على التقسيم الأوسع بين النموذج/المزوّد/وقت التشغيل، ابدأ من
[أوقات تشغيل الوكلاء](/ar/concepts/agent-runtimes). الخلاصة المختصرة هي:
`openai/gpt-5.5` هو مرجع النموذج، و`codex` هو وقت التشغيل، وتظل Telegram
أو Discord أو Slack أو قناة أخرى سطح الاتصال.

## المتطلبات

- OpenClaw مع توفر Plugin `codex` المضمّن.
- إذا كان إعدادك يستخدم `plugins.allow`، فأدرج `codex`.
- خادم تطبيق Codex `0.125.0` أو أحدث. يدير Plugin المضمّن ملفا تنفيذيا متوافقا
  لخادم تطبيق Codex افتراضيا، لذلك لا تؤثر أوامر `codex` المحلية على `PATH`
  في بدء تشغيل الحزمة العادي.
- توفر مصادقة Codex عبر `openclaw models auth login --provider openai`،
  أو حساب خادم تطبيق في منزل Codex الخاص بالوكيل، أو ملف تعريف مصادقة Codex
  صريح بمفتاح API.

لأسبقية المصادقة، وعزل البيئة، وأوامر خادم التطبيق المخصصة، واكتشاف النماذج،
وجميع حقول الإعداد، راجع
[مرجع حزمة Codex](/ar/plugins/codex-harness-reference).

## البدء السريع

يريد معظم المستخدمين الذين يريدون Codex في OpenClaw هذا المسار: سجّل الدخول
باشتراك ChatGPT/Codex، وفعّل Plugin `codex` المضمّن، واستخدم مرجع نموذج
`openai/gpt-*` قياسيا.

سجّل الدخول باستخدام OAuth الخاص بـ Codex:

```bash
openclaw models auth login --provider openai
```

فعّل Plugin `codex` المضمّن واختر نموذج وكيل OpenAI:

```json5
{
  plugins: {
    entries: {
      codex: {
        enabled: true,
      },
    },
  },
  agents: {
    defaults: {
      model: "openai/gpt-5.5",
    },
  },
}
```

إذا كان إعدادك يستخدم `plugins.allow`، فأضف `codex` هناك أيضا:

```json5
{
  plugins: {
    allow: ["codex"],
    entries: {
      codex: {
        enabled: true,
      },
    },
  },
}
```

أعد تشغيل Gateway بعد تغيير إعدادات Plugin. إذا كانت دردشة موجودة لديها جلسة
بالفعل، فاستخدم `/new` أو `/reset` قبل اختبار تغييرات وقت التشغيل كي تحل
الدورة التالية الحزمة من الإعداد الحالي.

## الإعداد

إعداد البدء السريع هو الحد الأدنى القابل للاستخدام لإعداد حزمة Codex. عيّن
خيارات حزمة Codex في إعداد OpenClaw، واستخدم CLI لمصادقة Codex فقط:

| الحاجة                                   | عيّن                                                                             | المكان                              |
| ---------------------------------------- | -------------------------------------------------------------------------------- | ----------------------------------- |
| تفعيل الحزمة                             | `plugins.entries.codex.enabled: true`                                            | إعداد OpenClaw                      |
| الاحتفاظ بتثبيت Plugin ضمن قائمة السماح | أدرج `codex` في `plugins.allow`                                                  | إعداد OpenClaw                      |
| توجيه دورات وكيل OpenAI عبر Codex        | `agents.defaults.model` أو `agents.list[].model` كـ `openai/gpt-*`               | إعداد وكيل OpenClaw                 |
| تسجيل الدخول باستخدام OAuth لـ ChatGPT/Codex | `openclaw models auth login --provider openai`                                   | ملف تعريف مصادقة CLI                |
| إضافة احتياطي مفتاح API لتشغيلات Codex   | ملف تعريف مفتاح API `openai:*` مدرج بعد مصادقة الاشتراك في `auth.order.openai` | ملف تعريف مصادقة CLI + إعداد OpenClaw |
| الفشل بإغلاق آمن عندما لا يتوفر Codex    | `agentRuntime.id: "codex"` على مستوى المزوّد أو النموذج                         | إعداد نموذج/مزوّد OpenClaw          |
| استخدام حركة OpenAI API المباشرة         | `agentRuntime.id: "openclaw"` على مستوى المزوّد أو النموذج مع مصادقة OpenAI العادية | إعداد نموذج/مزوّد OpenClaw          |
| ضبط سلوك خادم التطبيق                    | `plugins.entries.codex.config.appServer.*`                                       | إعداد Plugin Codex                  |
| تفعيل تطبيقات Plugin الأصلية في Codex    | `plugins.entries.codex.config.codexPlugins.*`                                    | إعداد Plugin Codex                  |
| تفعيل Codex Computer Use                 | `plugins.entries.codex.config.computerUse.*`                                     | إعداد Plugin Codex                  |

استخدم مراجع نماذج `openai/gpt-*` لدورات وكيل OpenAI المدعومة بـ Codex. فضّل
`auth.order.openai` لترتيب أولوية الاشتراك ثم احتياطي مفتاح API. معرّفات ملفات
تعريف مصادقة Codex القديمة وترتيب مصادقة Codex القديم هي حالة قديمة خاصة
بالطبيب فقط؛ لا تكتب مراجع GPT قديمة جديدة لـ Codex.

لا تعيّن `compaction.model` أو `compaction.provider` على الوكلاء المدعومين
بـ Codex. يجري Codex الـ Compaction عبر حالة سلسلة خادم التطبيق الأصلية الخاصة
به، لذلك يتجاهل OpenClaw تجاوزات الملخّص المحلية هذه في وقت التشغيل ويزيلها
`openclaw doctor --fix` عندما يستخدم الوكيل Codex.

يبقى Lossless مدعوما كمحرك سياق للتجميع، والإدخال، والصيانة حول دورات Codex.
اضبطه عبر `plugins.slots.contextEngine: "lossless-claw"` و
`plugins.entries.lossless-claw.config.summaryModel`، وليس عبر
`agents.defaults.compaction.provider`. يرحّل `openclaw doctor --fix` الشكل القديم
`compaction.provider: "lossless-claw"` إلى خانة محرك سياق Lossless عندما يكون
Codex هو وقت التشغيل النشط، لكن Codex الأصلي يظل مسؤولا عن Compaction.

تدعم حزمة خادم تطبيق Codex الأصلية محركات السياق التي تتطلب تجميعا قبل الموجّه.
لا توفر خلفيات CLI العامة، بما في ذلك `codex-cli`، قدرة المضيف هذه.

بالنسبة إلى الوكلاء المدعومين بـ Codex، يبدأ `/compact` تشغيل Compaction
الأصلي لخادم تطبيق Codex على السلسلة المرتبطة. لا ينتظر OpenClaw الاكتمال،
ولا يفرض مهلة OpenClaw، ولا يعيد تشغيل خادم التطبيق المشترك، ولا يرجع إلى
محرك سياق أو ملخّص OpenAI عام. إذا كان ربط سلسلة Codex الأصلية مفقودا أو
قديما، يفشل الأمر بإغلاق آمن حتى يرى المشغّل حد وقت التشغيل الحقيقي بدلا من
التبديل الصامت إلى خلفيات Compaction أخرى.

```json5
{
  auth: {
    order: {
      openai: ["openai:user@example.com", "openai:api-key-backup"],
    },
  },
}
```

في ذلك الشكل، يظل كلا الملفين الشخصيين يعملان عبر Codex لدورات وكيل
`openai/gpt-*`. مفتاح API هو احتياطي مصادقة فقط، وليس طلبا للتبديل إلى OpenClaw
أو OpenAI Responses العادية.

يغطي باقي هذه الصفحة المتغيرات الشائعة التي يجب على المستخدمين الاختيار بينها:
شكل النشر، والتوجيه مع الفشل بإغلاق آمن، وسياسة موافقة الحارس، وPlugins Codex
الأصلية، وComputer Use. للاطلاع على قوائم الخيارات الكاملة، والقيم الافتراضية،
والتعدادات، والاكتشاف، وعزل البيئة، والمهلات، وحقول نقل خادم التطبيق، راجع
[مرجع حزمة Codex](/ar/plugins/codex-harness-reference).

## التحقق من وقت تشغيل Codex

استخدم `/status` في الدردشة التي تتوقع فيها Codex. تظهر دورة وكيل OpenAI
مدعومة بـ Codex:

```text
Runtime: OpenAI Codex
```

ثم تحقق من حالة خادم تطبيق Codex:

```text
/codex status
/codex models
```

يعرض `/codex status` اتصال خادم التطبيق، والحساب، وحدود المعدلات، وخوادم MCP،
وSkills. يسرد `/codex models` كتالوج خادم تطبيق Codex المباشر للحزمة والحساب.
إذا كانت نتيجة `/status` غير متوقعة، فراجع
[استكشاف الأخطاء وإصلاحها](#troubleshooting).

## التوجيه واختيار النموذج

أبقِ مراجع المزوّد وسياسة وقت التشغيل منفصلين:

- استخدم `openai/gpt-*` لدورات وكيل OpenAI عبر Codex.
- لا تستخدم مراجع GPT القديمة الخاصة بـ Codex في الإعداد. شغّل `openclaw doctor --fix` لإصلاح المراجع القديمة وتثبيتات مسارات الجلسات القديمة.
- `agentRuntime.id: "codex"` اختياري للوضع التلقائي العادي في OpenAI، لكنه مفيد عندما يجب أن يفشل النشر بإغلاق آمن إذا لم يتوفر Codex.
- `agentRuntime.id: "openclaw"` يوجّه مزوّدا أو نموذجا إلى وقت تشغيل OpenClaw المضمّن عندما يكون ذلك مقصودا.
- يتحكم `/codex ...` في محادثات خادم تطبيق Codex الأصلية من الدردشة.
- ACP/acpx هو مسار حزمة خارجي منفصل. استخدمه فقط عندما يطلب المستخدم ACP/acpx أو محوّل حزمة خارجي.

توجيه الأوامر الشائع:

| نية المستخدم                                           | استخدم                                                                                               |
| ------------------------------------------------------ | ---------------------------------------------------------------------------------------------------- |
| إرفاق الدردشة الحالية                                  | `/codex bind [--cwd <path>]`                                                                         |
| استئناف سلسلة Codex موجودة                             | `/codex resume <thread-id>`                                                                          |
| سرد سلاسل Codex أو تصفيتها                             | `/codex threads [filter]`                                                                            |
| سرد Plugins Codex الأصلية                              | `/codex plugins list`                                                                                |
| تفعيل Plugin Codex أصلي مضبوط أو تعطيله                | `/codex plugins enable <name>`, `/codex plugins disable <name>`                                      |
| إرفاق جلسة Codex CLI موجودة على عقدة مقترنة            | `/codex sessions --host <node> [filter]`، ثم `/codex resume <session-id> --host <node> --bind here` |
| إرسال ملاحظات Codex فقط                               | `/codex diagnostics [note]`                                                                          |
| بدء مهمة ACP/acpx                                      | أوامر جلسة ACP/acpx، وليس `/codex`                                                                  |

| حالة الاستخدام                                      | التكوين                                                               | التحقق                                  | ملاحظات                              |
| ---------------------------------------------------- | ---------------------------------------------------------------------- | --------------------------------------- | ------------------------------------- |
| اشتراك ChatGPT/Codex مع وقت تشغيل Codex الأصلي       | `openai/gpt-*` مع تفعيل Plugin `codex`                                 | يعرض `/status` القيمة `Runtime: OpenAI Codex` | المسار الموصى به                    |
| الفشل المغلق إذا لم يكن Codex متاحا                  | الموفر أو النموذج `agentRuntime.id: "codex"`                           | تفشل الجولة بدلا من الرجوع المضمن      | استخدمه لنشر Codex فقط              |
| تمرير حركة مرور مفتاح OpenAI API مباشرة عبر OpenClaw | الموفر أو النموذج `agentRuntime.id: "openclaw"` ومصادقة OpenAI العادية | يعرض `/status` وقت تشغيل OpenClaw      | استخدمه فقط عندما يكون OpenClaw مقصودا |
| التكوين القديم                                      | مراجع Codex GPT القديمة                                                | يعيد `openclaw doctor --fix` كتابته    | لا تكتب تكوينا جديدا بهذه الطريقة   |
| محول ACP/acpx لـ Codex                               | ACP `sessions_spawn({ runtime: "acp" })`                               | حالة مهمة/جلسة ACP                     | منفصل عن حزمة Codex الأصلية          |

يتبع `agents.defaults.imageModel` تقسيم البادئة نفسه. استخدم `openai/gpt-*`
للمسار العادي في OpenAI، واستخدم `codex/gpt-*` فقط عندما ينبغي أن يعمل فهم الصور
عبر جولة محدودة في خادم تطبيق Codex. لا تستخدم
مراجع Codex GPT القديمة؛ يعيد الطبيب كتابة تلك البادئة القديمة إلى `openai/gpt-*`.

## أنماط النشر

### نشر Codex الأساسي

استخدم تكوين البدء السريع عندما ينبغي أن تستخدم كل جولات وكلاء OpenAI نظام Codex
افتراضيا.

```json5
{
  plugins: {
    entries: {
      codex: {
        enabled: true,
      },
    },
  },
  agents: {
    defaults: {
      model: "openai/gpt-5.5",
    },
  },
}
```

### نشر موفرين مختلط

يبقي هذا الشكل Claude وكيلا افتراضيا ويضيف وكيلا مسمى لـ Codex:

```json5
{
  plugins: {
    entries: {
      codex: {
        enabled: true,
      },
    },
  },
  agents: {
    defaults: {
      model: "anthropic/claude-opus-4-6",
    },
    list: [
      {
        id: "main",
        default: true,
        model: "anthropic/claude-opus-4-6",
      },
      {
        id: "codex",
        name: "Codex",
        model: "openai/gpt-5.5",
      },
    ],
  },
}
```

مع هذا التكوين، يستخدم الوكيل `main` مسار الموفر العادي الخاص به، ويستخدم الوكيل
`codex` خادم تطبيق Codex.

### نشر Codex بفشل مغلق

بالنسبة إلى جولات وكيل OpenAI، يتحول `openai/gpt-*` بالفعل إلى Codex عندما تكون
Plugin المضمنة متاحة. أضف سياسة وقت تشغيل صريحة عندما تريد قاعدة فشل مغلق مكتوبة:

```json5
{
  models: {
    providers: {
      openai: {
        agentRuntime: {
          id: "codex",
        },
      },
    },
  },
  agents: {
    defaults: {
      model: "openai/gpt-5.5",
    },
  },
  plugins: {
    entries: {
      codex: {
        enabled: true,
      },
    },
  },
}
```

عند فرض Codex، يفشل OpenClaw مبكرا إذا كانت Plugin الخاصة بـ Codex معطلة، أو كان
خادم التطبيق قديما جدا، أو تعذر بدء خادم التطبيق.

## سياسة خادم التطبيق

افتراضيا، تبدأ Plugin ثنائي Codex المدار من OpenClaw محليا باستخدام نقل stdio.
عيّن `appServer.command` فقط عندما تريد عمدا تشغيل ملف تنفيذي مختلف. استخدم نقل WebSocket فقط عندما يكون خادم تطبيق قيد التشغيل بالفعل في مكان آخر:

```json5
{
  plugins: {
    entries: {
      codex: {
        enabled: true,
        config: {
          appServer: {
            transport: "websocket",
            url: "ws://gateway-host:39175",
            authToken: "${CODEX_APP_SERVER_TOKEN}",
          },
        },
      },
    },
  },
}
```

تستخدم جلسات خادم تطبيق stdio المحلية افتراضيا وضعية المشغل المحلي الموثوق:
`approvalPolicy: "never"` و`approvalsReviewer: "user"` و
`sandbox: "danger-full-access"`. إذا كانت متطلبات Codex المحلية لا تسمح بهذه
الوضعية الضمنية بلا قيود، يختار OpenClaw أذونات الحارس المسموح بها بدلا من ذلك.
عندما يكون صندوق رمل OpenClaw نشطا للجلسة، يعطل OpenClaw نمط Code Mode الأصلي في Codex، وخوادم MCP الخاصة بالمستخدم، وتنفيذ Plugin المدعوم بالتطبيقات لتلك
الجولة بدلا من الاعتماد على وضع الصندوق الرملي من جانب مضيف Codex. يتاح وصول Shell
من خلال أدوات OpenClaw الديناميكية المدعومة بالصندوق الرملي مثل `sandbox_exec` و
`sandbox_process` عندما تكون أدوات exec/process العادية متاحة.

استخدم نمط تنفيذ OpenClaw الموحد عندما تريد مراجعة Codex الأصلية التلقائية قبل
الخروج من الصندوق الرملي أو منح أذونات إضافية:

```json5
{
  tools: {
    exec: {
      mode: "auto",
    },
  },
  plugins: {
    entries: {
      codex: {
        enabled: true,
      },
    },
  },
}
```

بالنسبة إلى جلسات خادم تطبيق Codex، يطابق OpenClaw `tools.exec.mode: "auto"` مع موافقات Codex
التي يراجعها Guardian، عادة
`approvalPolicy: "on-request"` و`approvalsReviewer: "auto_review"` و
`sandbox: "workspace-write"` عندما تسمح المتطلبات المحلية بهذه القيم.
في `tools.exec.mode: "auto"`، لا يحافظ OpenClaw على تجاوزات Codex القديمة غير الآمنة
`approvalPolicy: "never"` أو `sandbox: "danger-full-access"`؛ استخدم
`tools.exec.mode: "full"` لوضعية Codex مقصودة بلا موافقة. لا يزال الإعداد المسبق القديم
`plugins.entries.codex.config.appServer.mode: "guardian"` يعمل، لكن
`tools.exec.mode: "auto"` هو سطح OpenClaw الموحد.

للمقارنة على مستوى النمط مع موافقات تنفيذ المضيف وأذونات ACPX،
راجع [أنماط الأذونات](/ar/tools/permission-modes).

لكل حقل في خادم التطبيق، وترتيب المصادقة، وعزل البيئة، والاكتشاف،
وسلوك المهلة، راجع [مرجع حزمة Codex](/ar/plugins/codex-harness-reference).

## الأوامر والتشخيصات

تسجل Plugin المضمنة الأمر `/codex` كأمر شرطة مائلة على أي قناة
تدعم أوامر OpenClaw النصية.

يتطلب التنفيذ والتحكم الأصليان مالكا أو عميل Gateway بصلاحية `operator.admin`.
يشمل ذلك ربط السلاسل أو استئنافها، وإرسال الجولات أو إيقافها،
وتغيير النموذج أو النمط السريع أو حالة الأذونات، والضغط أو المراجعة،
وفصل ربط. يحتفظ المرسلون المخولون الآخرون بأوامر القراءة فقط الخاصة بالحالة، والمساعدة،
والحساب، والنموذج، والسلسلة، وخادم MCP، وSkill، وفحص الربط.

الأشكال الشائعة:

- يتحقق `/codex status` من اتصال خادم التطبيق، والنماذج، والحساب، وحدود المعدل،
  وخوادم MCP، وSkills.
- يعرض `/codex models` نماذج خادم تطبيق Codex الحية.
- يعرض `/codex threads [filter]` سلاسل خادم تطبيق Codex الحديثة.
- يربط `/codex resume <thread-id>` جلسة OpenClaw الحالية بسلسلة Codex
  موجودة.
- يطلب `/codex compact` من خادم تطبيق Codex ضغط السلسلة المرتبطة.
- يبدأ `/codex review` مراجعة Codex الأصلية للسلسلة المرتبطة.
- يطلب `/codex diagnostics [note]` الإذن قبل إرسال ملاحظات Codex للسلسلة
  المرتبطة.
- يعرض `/codex account` حالة الحساب وحدود المعدل.
- يعرض `/codex mcp` حالة خوادم MCP في خادم تطبيق Codex.
- يعرض `/codex skills` Skills خادم تطبيق Codex.

في معظم تقارير الدعم، ابدأ بـ `/diagnostics [note]` في المحادثة
التي حدث فيها الخلل. ينشئ هذا تقرير تشخيص Gateway واحدا، وبالنسبة إلى جلسات
حزمة Codex، يطلب الموافقة على إرسال حزمة ملاحظات Codex ذات الصلة.
راجع [تصدير التشخيصات](/ar/gateway/diagnostics) للاطلاع على نموذج الخصوصية وسلوك
الدردشة الجماعية.

استخدم `/codex diagnostics [note]` فقط عندما تريد تحديدا رفع ملاحظات Codex
للسلسلة المرتبطة حاليا من دون حزمة تشخيصات Gateway الكاملة.

### فحص سلاسل Codex محليا

غالبا ما تكون أسرع طريقة لفحص تشغيل Codex سيئ هي فتح سلسلة Codex الأصلية
مباشرة:

```bash
codex resume <thread-id>
```

احصل على معرف السلسلة من رد `/diagnostics` المكتمل، أو `/codex binding`، أو
`/codex threads [filter]`.

لآليات الرفع وحدود التشخيصات على مستوى وقت التشغيل، راجع
[وقت تشغيل حزمة Codex](/ar/plugins/codex-harness-runtime#codex-feedback-upload).

تحدد المصادقة بهذا الترتيب:

1. ملفات تعريف مصادقة OpenAI المرتبة للوكيل، ويفضل أن تكون تحت
   `auth.order.openai`. شغّل `openclaw doctor --fix` لترحيل
   معرفات ملفات تعريف مصادقة Codex القديمة وترتيب مصادقة Codex القديم.
2. الحساب الموجود لخادم التطبيق في موطن Codex لذلك الوكيل.
3. لإطلاقات خادم تطبيق stdio المحلية فقط، `CODEX_API_KEY` ثم
   `OPENAI_API_KEY`، عندما لا يكون أي حساب خادم تطبيق موجودا وتظل مصادقة OpenAI
   مطلوبة.

عندما يرى OpenClaw ملف تعريف مصادقة Codex بنمط اشتراك ChatGPT، فإنه يزيل
`CODEX_API_KEY` و`OPENAI_API_KEY` من عملية Codex الفرعية المنشأة. يحافظ ذلك
على إتاحة مفاتيح API على مستوى Gateway للتضمينات أو نماذج OpenAI المباشرة
من دون جعل جولات خادم تطبيق Codex الأصلية تحاسب عبر API بالخطأ.
تستخدم ملفات تعريف مفتاح Codex API الصريحة والرجوع المحلي إلى مفتاح بيئة stdio
تسجيل دخول خادم التطبيق بدلا من بيئة العملية الفرعية الموروثة. لا تتلقى اتصالات
خادم تطبيق WebSocket رجوع مفتاح API من بيئة Gateway؛ استخدم ملف تعريف مصادقة صريحا أو حساب خادم التطبيق البعيد نفسه.
عند تكوين Plugins Codex الأصلية، يثبت OpenClaw تلك Plugins أو يحدّثها
عبر خادم التطبيق المتصل قبل إتاحة التطبيقات المملوكة لـ Plugin لسلسلة
Codex. يظل `app/list` مصدر الحقيقة لمعرفات التطبيقات،
والإتاحة، والبيانات الوصفية، لكن OpenClaw يملك قرار التمكين لكل سلسلة:
إذا سمحت السياسة بتطبيق مدرج ومتاح، يرسل OpenClaw
`thread/start.config.apps[appId].enabled = true` حتى عندما يبلغ `app/list` حاليا
أن ذلك التطبيق معطل. لا يخترع هذا المسار تثبيت تطبيق لمعرفات
غير معروفة؛ يفعّل OpenClaw فقط Plugins السوق باستخدام `plugin/install`
ثم يحدّث المخزون.

إذا بلغ ملف تعريف اشتراك حد استخدام Codex، يسجل OpenClaw وقت إعادة الضبط
عندما يبلغ Codex عنه، ويحاول ملف تعريف المصادقة المرتب التالي لجولة Codex نفسها.
عندما يمر وقت إعادة الضبط، يصبح ملف تعريف الاشتراك مؤهلا
مرة أخرى من دون تغيير نموذج `openai/gpt-*` المحدد أو وقت تشغيل Codex.

بالنسبة إلى إطلاقات خادم تطبيق stdio المحلية، يضبط OpenClaw `CODEX_HOME` على دليل لكل وكيل
بحيث لا تقرأ أو تكتب إعدادات Codex، وملفات المصادقة/الحساب، وذاكرة التخزين المؤقت/بيانات Plugin، وحالة
السلاسل الأصلية في `~/.codex` الشخصي للمشغل
افتراضيا. يحافظ OpenClaw على قيمة العملية العادية `HOME`؛ ولا تزال العمليات الفرعية التي يشغلها Codex
قادرة على العثور على إعدادات ورموز موطن المستخدم، وقد يكتشف Codex إدخالات
`$HOME/.agents/skills` و`$HOME/.agents/plugins/marketplace.json` المشتركة.

إذا احتاج النشر إلى عزل بيئة إضافي، فأضف تلك المتغيرات إلى
`appServer.clearEnv`:

```json5
{
  plugins: {
    entries: {
      codex: {
        enabled: true,
        config: {
          appServer: {
            clearEnv: ["CODEX_API_KEY", "OPENAI_API_KEY"],
          },
        },
      },
    },
  },
}
```

لا يؤثر `appServer.clearEnv` إلا في عملية خادم تطبيق Codex الفرعية المنشأة.
يزيل OpenClaw `CODEX_HOME` و`HOME` من هذه القائمة أثناء تسوية الإطلاق المحلي:
يبقى `CODEX_HOME` لكل وكيل، وتبقى `HOME` موروثة حتى تتمكن
العمليات الفرعية من استخدام حالة موطن المستخدم العادية.

تُحمَّل أدوات Codex الديناميكية افتراضيًا بوضع `searchable`. لا يعرِض OpenClaw
أدوات ديناميكية تكرر عمليات مساحة العمل الأصلية في Codex: `read` و`write` و
`edit` و`apply_patch` و`exec` و`process` و`update_plan`. تتوفر معظم أدوات
تكامل OpenClaw المتبقية، مثل المراسلة والوسائط وCron والمتصفح والعُقد و
Gateway و`heartbeat_respond`، عبر بحث أدوات Codex ضمن مساحة الاسم
`openclaw`، مما يبقي سياق النموذج الأولي أصغر. يستخدم بحث الويب أداة
`web_search` المستضافة من Codex افتراضيًا عند تمكين البحث وعدم تحديد
مزود مُدار. البحث المستضاف الأصلي وأداة OpenClaw الديناميكية المُدارة
`web_search` متنافيان، بحيث لا يمكن للبحث المُدار تجاوز قيود النطاقات
الأصلية. يستخدم OpenClaw الأداة المُدارة عندما يكون البحث المستضاف غير
متاح، أو معطّلًا صراحة، أو مستبدلًا بمزود مُدار محدد. يبقي OpenClaw إضافة
Codex المستقلة `web.run` معطلة لأن حركة خادم التطبيق الإنتاجية ترفض مساحة
الاسم `web` المعرّفة من المستخدم. يعطّل `tools.web.search.enabled: false`
كلا المسارين، وكذلك عمليات التشغيل المخصصة لـ LLM فقط مع تعطيل الأدوات.
يتعامل Codex مع `"cached"` كتفضيل ويحلّه إلى وصول خارجي مباشر للمنعطفات غير
المقيدة في خادم التطبيق. يفشل الرجوع التلقائي المُدار بإغلاق آمن عند ضبط
`allowedDomains` الأصلية حتى لا يمكن تجاوز قائمة السماح. تؤدي تغييرات سياسة
البحث الفعالة والمستمرة إلى تدوير سلسلة Codex المرتبطة قبل المنعطف التالي.
تستخدم القيود المؤقتة لكل منعطف سلسلة مؤقتة مقيدة وتحافظ على الارتباط
الحالي للاستئناف لاحقًا. تبقى ردود المصدر الخاصة بـ `sessions_yield` وأداة
الرسائل فقط مباشرة لأن هذه عقود تحكم في المنعطف. يبقى `sessions_spawn`
قابلًا للبحث بحيث يظل `spawn_agent` الأصلي في Codex هو سطح الوكيل الفرعي
الأساسي في Codex، بينما يظل تفويض OpenClaw أو ACP الصريح متاحًا عبر مساحة
اسم الأدوات الديناميكية `openclaw`. تخبر تعليمات التعاون في Heartbeat
Codex بالبحث عن `heartbeat_respond` قبل إنهاء منعطف Heartbeat عندما لا تكون
الأداة محمّلة بالفعل.

اضبط `codexDynamicToolsLoading: "direct"` فقط عند الاتصال بخادم تطبيق Codex
مخصص لا يمكنه البحث في الأدوات الديناميكية المؤجلة، أو عند تصحيح حمولة
الأدوات الكاملة.

`appServer.networkProxy` صريح لأنه يغيّر عقد صندوق رمل Codex.
عند تفعيله، يعيّن OpenClaw أيضا `features.network_proxy.enabled` و
`default_permissions` في تهيئة خيط Codex لكي يتمكن ملف تعريف الأذونات
المُنشأ من بدء الشبكة المُدارة بواسطة Codex. افتراضيا، ينشئ OpenClaw اسم
ملف تعريف مقاوما للتصادم `openclaw-network-<fingerprint>` من نص ملف
التعريف؛ استخدم `profileName` فقط عندما يكون اسم محلي ثابت مطلوبا.

```js
export default {
  plugins: {
    entries: {
      codex: {
        config: {
          appServer: {
            sandbox: "workspace-write",
            networkProxy: {
              enabled: true,
              domains: {
                "api.openai.com": "allow",
                "blocked.example.com": "deny",
              },
              unixSockets: {
                "/tmp/proxy.sock": "allow",
                "/tmp/blocked.sock": "none",
              },
              allowUpstreamProxy: true,
              proxyUrl: "http://127.0.0.1:3128",
            },
          },
        },
      },
    },
  },
};
```

إذا كان وقت تشغيل خادم التطبيق العادي سيكون `danger-full-access`، فإن تفعيل
`networkProxy` يستخدم وصولا إلى نظام الملفات على نمط مساحة العمل لملف
تعريف الأذونات المُنشأ. إن إنفاذ الشبكة المُدار بواسطة Codex هو شبكة ضمن
صندوق رمل، لذلك لن يحمي ملف تعريف الوصول الكامل حركة المرور الصادرة.
تستخدم إدخالات النطاق `allow` أو `deny`؛ وتستخدم إدخالات مقبس Unix قيم
Codex ‏`allow` أو `none`.

مكالمات الأدوات الديناميكية المملوكة من OpenClaw محدودة بشكل مستقل عن
`appServer.requestTimeoutMs`: تستخدم طلبات Codex `item/tool/call` مهلة مراقبة
من OpenClaw قدرها 90 ثانية افتراضيا. وسيطة `timeoutMs` الموجبة لكل مكالمة تمدد
أو تقصر ميزانية تلك الأداة المحددة. تستخدم أداة `image_generate`
`agents.defaults.imageGenerationModel.timeoutMs` عندما لا توفر مكالمة الأداة
مهلتها الخاصة، أو تستخدم افتراضي توليد الصور البالغ 120 ثانية بخلاف ذلك.
تستخدم أداة فهم الوسائط `image`
`tools.media.image.timeoutSeconds` أو افتراضي الوسائط البالغ 60 ثانية. بالنسبة
إلى فهم الصور، تنطبق تلك المهلة على الطلب نفسه ولا يتم تقليلها بسبب أعمال
التحضير السابقة. تحدد ميزانيات الأدوات الديناميكية بحد أقصى قدره 600000 ms.
عند انتهاء المهلة، يوقف OpenClaw إشارة الأداة حيث يكون ذلك مدعوما، ويعيد
استجابة أداة ديناميكية فاشلة إلى Codex كي تتمكن الدورة من المتابعة بدلا من ترك
الجلسة في `processing`. هذه المهلة الحارسة هي ميزانية `item/tool/call`
الديناميكية الخارجية؛ أما مهل طلبات المزوّد الخاصة فتعمل داخل تلك المكالمة
وتحتفظ بدلالات المهلة الخاصة بها.

بعد أن يقبل Codex دورة، وبعد أن يرد OpenClaw على طلب خادم تطبيق محدد بنطاق
الدورة، يتوقع الحاضن أن يحرز Codex تقدما في الدورة الحالية وأن ينهي في النهاية
الدورة الأصلية عبر `turn/completed`. إذا ظل خادم التطبيق صامتا لمدة
`appServer.turnCompletionIdleTimeoutMs`، يحاول OpenClaw بأفضل جهد مقاطعة دورة
Codex، ويسجل مهلة تشخيصية، ويحرر مسار جلسة OpenClaw حتى لا تصطف رسائل الدردشة
اللاحقة خلف دورة أصلية قديمة. معظم الإشعارات غير النهائية للدورة نفسها تعطل
هذه المهلة الحارسة القصيرة لأن Codex أثبت أن الدورة ما زالت حية. تستخدم عمليات
تسليم الأدوات ميزانية خمول أطول بعد الأداة: بعد أن يعيد OpenClaw استجابة
`item/tool/call`، وبعد اكتمال عناصر الأدوات الأصلية مثل `commandExecution`،
وبعد اكتمالات `custom_tool_call_output` الخام، وبعد تقدم المساعد الخام بعد
الأداة، أو اكتمالات الاستدلال، أو تقدم الاستدلال. يستخدم الحارس
`appServer.postToolRawAssistantCompletionIdleTimeoutMs` عند تكوينه، ويفترض خمس
دقائق بخلاف ذلك. تمدد ميزانية ما بعد الأداة نفسها أيضا مهلة مراقبة التقدم لنافذة
التوليف الصامتة قبل أن يصدر Codex حدث الدورة الحالية التالي. الإشعارات العامة
لخادم التطبيق، مثل تحديثات حدود المعدل، لا تعيد ضبط تقدم خمول الدورة. يمكن أن
تتبع اكتمالات الاستدلال، واكتمالات `agentMessage` التعليقية، وتقدم الاستدلال
الخام أو المساعد قبل الأداة رسالة نهائية تلقائية، لذلك تستخدم حارس الرد بعد
التقدم بدلا من تحرير مسار الجلسة فورا. وحدها عناصر `agentMessage` النهائية/غير
التعليقية المكتملة واكتمالات المساعد الخام قبل الأداة تسلح تحرير مخرجات
المساعد: إذا أصبح Codex صامتا بعد ذلك من دون `turn/completed`، يحاول OpenClaw
بأفضل جهد مقاطعة الدورة الأصلية وتحرير مسار الجلسة. إذا فازت مراقبة دورة أخرى
بسباق التحرير هذا، يظل OpenClaw يقبل عنصر المساعد النهائي المكتمل عندما لا يبقى
أي طلب أصلي أو عنصر أو اكتمال أداة ديناميكية نشطا، ويظل تحرير مخرجات المساعد
ينتمي إلى آخر عنصر مكتمل، من دون اكتمال عنصر لاحق. يمكن أن يحافظ هذا على
الإجابة النهائية بعد عمل أداة مكتمل من دون إعادة تشغيل الدورة. لا تؤهل دلتا
المساعد الجزئية، والردود السابقة القديمة، والاكتمالات اللاحقة الفارغة. يعاد
محاولة إخفاقات خادم التطبيق عبر stdio الآمنة لإعادة التشغيل مرة واحدة في محاولة
خادم تطبيق جديدة، بما في ذلك مهل خمول إكمال الدورة من دون دليل على مساعد أو
أداة أو عنصر نشط أو أثر جانبي. لا تزال المهل غير الآمنة تنهي عميل خادم التطبيق
العالق وتحرر مسار جلسة OpenClaw. كما أنها تمسح ارتباط الخيط الأصلي القديم بدلا
من إعادة تشغيله تلقائيا. تعرض مهل مراقبة الإكمال نص مهلة خاصا بـ Codex: تقول
الحالات الآمنة لإعادة التشغيل إن الاستجابة قد تكون غير مكتملة، بينما تطلب
الحالات غير الآمنة من المستخدم التحقق من الحالة الحالية قبل إعادة المحاولة.
تتضمن تشخيصات المهلة العامة حقولا بنيوية مثل آخر طريقة إشعار من خادم التطبيق،
ومعرّف/نوع/دور عنصر استجابة المساعد الخام، وأعداد الطلبات/العناصر النشطة، وحالة
المراقبة المسلحة. عندما يكون آخر إشعار عنصرا خاما لاستجابة المساعد، فإنها تتضمن
أيضا معاينة محدودة لنص المساعد. ولا تتضمن محتوى المطالبة الخام أو الأداة.

تظل تجاوزات البيئة متاحة للاختبار المحلي:

- `OPENCLAW_CODEX_APP_SERVER_BIN`
- `OPENCLAW_CODEX_APP_SERVER_ARGS`
- `OPENCLAW_CODEX_APP_SERVER_MODE=yolo|guardian`
- `OPENCLAW_CODEX_APP_SERVER_APPROVAL_POLICY`
- `OPENCLAW_CODEX_APP_SERVER_SANDBOX`

يتجاوز `OPENCLAW_CODEX_APP_SERVER_BIN` الملف الثنائي المدار عندما لا يكون
`appServer.command` مضبوطا.

تمت إزالة `OPENCLAW_CODEX_APP_SERVER_GUARDIAN=1`. استخدم
`plugins.entries.codex.config.appServer.mode: "guardian"` بدلا من ذلك، أو
`OPENCLAW_CODEX_APP_SERVER_MODE=guardian` للاختبار المحلي لمرة واحدة. يفضل
التكوين للنشر القابل للتكرار لأنه يبقي سلوك Plugin في الملف المراجع نفسه مثل
بقية إعداد حاضن Codex.

## Plugins Codex الأصلية

يستخدم دعم Plugins Codex الأصلية قدرات التطبيق وPlugin الخاصة بخادم تطبيق Codex
ضمن خيط Codex نفسه مثل دورة حاضن OpenClaw. لا يترجم OpenClaw Plugins Codex إلى
أدوات OpenClaw ديناميكية اصطناعية `codex_plugin_*`.

يؤثر `codexPlugins` فقط في الجلسات التي تختار حاضن Codex الأصلي. ولا يؤثر في
تشغيلات الحاضن المدمج، أو تشغيلات مزود OpenAI العادية، أو ارتباطات محادثة ACP،
أو الحاضنات الأخرى.

تكوين مرحل بالحد الأدنى:

```json5
{
  plugins: {
    entries: {
      codex: {
        enabled: true,
        config: {
          codexPlugins: {
            enabled: true,
            allow_destructive_actions: true,
            plugins: {
              "google-calendar": {
                enabled: true,
                marketplaceName: "openai-curated",
                pluginName: "google-calendar",
              },
            },
          },
        },
      },
    },
  },
}
```

يتم حساب تكوين تطبيق الخيط عندما ينشئ OpenClaw جلسة حاضن Codex أو يستبدل ارتباط
خيط Codex قديما. ولا يعاد حسابه في كل دورة. بعد تغيير `codexPlugins`، استخدم
`/new`، أو `/reset`، أو أعد تشغيل Gateway حتى تبدأ جلسات حاضن Codex المستقبلية
بمجموعة التطبيقات المحدثة.

لأهلية الترحيل، ومخزون التطبيقات، وسياسة الإجراءات التدميرية، والاستثارات،
وتشخيصات Plugin الأصلية، راجع
[Plugins Codex الأصلية](/ar/plugins/codex-native-plugins).

يتحكم حساب Codex الذي سجل الدخول في الوصول إلى التطبيقات وPlugin من جهة OpenAI،
وكذلك تتحكم عناصر تحكم تطبيقات مساحة العمل بالنسبة إلى مساحات عمل Business و
Enterprise/Edu. راجع
[استخدام Codex مع خطة ChatGPT الخاصة بك](https://help.openai.com/en/articles/11369540-using-codex-with-your-chatgpt-plan)
للاطلاع على نظرة OpenAI العامة إلى الحساب وعناصر تحكم مساحة العمل.

## استخدام الكمبيوتر

استخدام الكمبيوتر مغطى في دليل الإعداد الخاص به:
[استخدام الكمبيوتر في Codex](/ar/plugins/codex-computer-use).

النسخة المختصرة: لا يضمن OpenClaw تطبيق التحكم بسطح المكتب ولا ينفذ إجراءات سطح
المكتب بنفسه. إنه يحضر خادم تطبيق Codex، ويتحقق من توفر خادم MCP
`computer-use`، ثم يترك Codex يملك مكالمات أداة MCP الأصلية أثناء دورات وضع
Codex.

## حدود وقت التشغيل

يغير حاضن Codex منفذ الوكيل المضمن منخفض المستوى فقط.

- أدوات OpenClaw الديناميكية مدعومة. يطلب Codex من OpenClaw تنفيذ تلك الأدوات،
  لذلك يبقى OpenClaw ضمن مسار التنفيذ.
- أدوات الصدفة والتصحيح وMCP والتطبيقات الأصلية الخاصة بـ Codex مملوكة لـ Codex.
  يمكن لـ OpenClaw مراقبة أحداث أصلية محددة أو حظرها عبر المرحل المدعوم، لكنه
  لا يعيد كتابة وسيطات الأدوات الأصلية.
- يملك Codex Compaction الأصلي. يحتفظ OpenClaw بمرآة للنصوص لمحفوظات القناة،
  والبحث، و`/new`، و`/reset`، والتبديل المستقبلي للنموذج أو الحاضن، لكنه لا
  يستبدل Compaction الخاص بـ Codex بملخص OpenClaw أو محرك سياق.
- يستمر توليد الوسائط، وفهم الوسائط، وTTS، والموافقات، ومخرجات أداة المراسلة عبر
  إعدادات مزود/نموذج OpenClaw المطابقة.
- ينطبق `tool_result_persist` على نتائج أدوات النصوص المملوكة من OpenClaw، لا
  على سجلات نتائج أدوات Codex الأصلية.

لطبقات الخطافات، والأسطح المدعومة من V1، ومعالجة الأذونات الأصلية، وتوجيه
الطابور، وآليات رفع ملاحظات Codex، وتفاصيل Compaction، راجع
[وقت تشغيل حاضن Codex](/ar/plugins/codex-harness-runtime).

## استكشاف الأخطاء وإصلاحها

**لا يظهر Codex كمزود `/model` عادي:** هذا متوقع للتكوينات الجديدة. اختر نموذج
`openai/gpt-*`، وفعّل `plugins.entries.codex.enabled`، وتحقق مما إذا كان
`plugins.allow` يستبعد `codex`.

**يستخدم OpenClaw الحاضن المدمج بدلا من Codex:** تأكد من أن مرجع النموذج هو
`openai/gpt-*` على مزود OpenAI الرسمي وأن Plugin Codex مثبت ومفعّل. إذا احتجت
إلى إثبات صارم أثناء الاختبار، فاضبط `agentRuntime.id: "codex"` على مستوى
المزود أو النموذج. يفشل وقت تشغيل Codex الإجباري بدلا من الرجوع إلى OpenClaw.

**يرجع وقت تشغيل OpenAI Codex إلى مسار مفتاح API:** اجمع مقتطف Gateway منقحا
يعرض النموذج، ووقت التشغيل، والمزود المحدد، والإخفاق. اطلب من المتعاونين
المتأثرين تشغيل هذا الأمر للقراءة فقط على مضيف OpenClaw لديهم:

```bash
(
  pattern='openai/gpt-5\.[45]|openai[-]codex|agentRuntime(\.id)?|harnessRuntime|Runtime: OpenAI Codex|legacy OpenAI Codex prefix|resolveSelectedOpenAIRuntimeProvider|candidateProvider[": ]+openai|status[": ]+401|Incorrect API key|No API key|api-key path|API-key path|OAuth'

  if ls /tmp/openclaw/openclaw-*.log >/dev/null 2>&1; then
    grep -E -i -n "$pattern" /tmp/openclaw/openclaw-*.log 2>/dev/null || true
  else
    journalctl --user -u openclaw-gateway --since today --no-pager 2>/dev/null \
      | grep -E -i "$pattern" || true
  fi
) | sed -E \
    -e 's/(Authorization: Bearer )[A-Za-z0-9._~+\/-]+/\1[REDACTED]/Ig' \
    -e 's/(Bearer )[A-Za-z0-9._~+\/-]+/\1[REDACTED]/Ig' \
    -e 's/(api[_ -]?key[=: ]+)[^ ,}"]+/\1[REDACTED]/Ig' \
    -e 's/(OPENAI_API_KEY[=: ]+)[^ ,}"]+/\1[REDACTED]/Ig' \
    -e 's/sk-[A-Za-z0-9_-]{12,}/sk-[REDACTED]/g' \
    -e 's/[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}/[EMAIL-REDACTED]/g' \
  | tail -200
```

تتضمن المقتطفات المفيدة عادة `openai/gpt-5.5` أو `openai/gpt-5.4`،
و`Runtime: OpenAI Codex`، و`agentRuntime.id` أو `harnessRuntime`،
و`candidateProvider: "openai"`، ونتيجة `401`، أو `Incorrect API key`، أو
`No API key`. ينبغي أن يعرض التشغيل المصحح مسار OpenAI OAuth بدلا من إخفاق
مفتاح API عادي لـ OpenAI.

**لا يزال تكوين مراجع نماذج Codex القديمة موجودا:** شغّل `openclaw doctor --fix`.
يعيد Doctor كتابة مراجع النماذج القديمة إلى `openai/*`، ويزيل تثبيتات وقت
التشغيل القديمة للجلسة والوكيل بالكامل، ويحافظ على تجاوزات ملف تعريف المصادقة
الموجودة.

**تم رفض خادم التطبيق:** استخدم خادم تطبيق Codex `0.125.0` أو أحدث. يتم رفض
الإصدارات التمهيدية من الإصدار نفسه أو الإصدارات ذات لاحقات البناء مثل
`0.125.0-alpha.2` أو `0.125.0+custom` لأن OpenClaw يختبر حد بروتوكول
`0.125.0` المستقر الأدنى.

**لا يستطيع `/codex status` الاتصال:** تحقق من أن Plugin `codex` المضمن مفعّل،
وأن `plugins.allow` يتضمنه عندما تكون قائمة السماح مكونة، وأن أي
`appServer.command` أو `url` أو `authToken` أو رؤوس مخصصة صالحة.

**اكتشاف النماذج بطيء:** خفض
`plugins.entries.codex.config.discovery.timeoutMs` أو عطل الاكتشاف. راجع
[مرجع حاضن Codex](/ar/plugins/codex-harness-reference#model-discovery).

**يفشل نقل WebSocket فورا:** تحقق من `appServer.url`، و`authToken`، والرؤوس،
ومن أن خادم التطبيق البعيد يتحدث إصدار بروتوكول خادم تطبيق Codex نفسه.

**أدوات الصدفة الأصلية أو أدوات الترقيع محظورة مع `Native hook relay unavailable`:**
لا تزال سلسلة Codex تحاول استخدام معرّف ترحيل خطاف أصلي لم يعد OpenClaw
يسجّله. هذه مشكلة نقل خطاف Codex أصلية، وليست فشلًا في ACP
أو الخلفية، أو المزوّد، أو GitHub، أو أمر الصدفة. ابدأ جلسة جديدة في
المحادثة المتأثرة باستخدام `/new` أو `/reset`، ثم أعد محاولة أمر غير ضار. إذا
نجح ذلك مرة واحدة لكن استدعاء الأداة الأصلية التالي فشل مرة أخرى، فتعامل مع `/new` كحل مؤقت
فقط: انسخ المطالبة إلى جلسة جديدة بعد إعادة تشغيل خادم تطبيق Codex
أو OpenClaw Gateway بحيث تُسقَط السلاسل القديمة وتُعاد إنشاء
تسجيلات الخطافات الأصلية.

**يستخدم نموذج غير Codex الحزمة المضمنة:** هذا متوقع ما لم
توجّهه سياسة تشغيل المزوّد أو النموذج إلى حزمة أخرى. تبقى مراجع المزوّدين العادية غير OpenAI
على مسار المزوّد المعتاد في وضع `auto`.

**Computer Use مثبتة لكن الأدوات لا تعمل:** تحقق من
`/codex computer-use status` من جلسة جديدة. إذا أبلغت أداة عن
`Native hook relay unavailable`، فاستخدم استرداد ترحيل الخطاف الأصلي أعلاه. راجع
[Codex Computer Use](/ar/plugins/codex-computer-use#troubleshooting).
