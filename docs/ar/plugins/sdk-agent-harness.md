**حزام الوكيل** هو المنفّذ منخفض المستوى لدورة OpenClaw واحدة مُحضّرة
للوكيل. ليس مزوّد نموذج، ولا قناة، ولا سجل أدوات.
للنموذج الذهني المواجه للمستخدم، راجع [أوقات تشغيل الوكلاء](/ar/concepts/agent-runtimes).

استخدم هذا السطح فقط مع Pluginات أصلية مضمّنة أو موثوقة. لا يزال العقد
تجريبياً لأن أنواع المعاملات تعكس عمداً المشغّل المضمّن الحالي.

## متى تستخدم حزاماً

سجّل حزام وكيل عندما تكون لعائلة نماذج بيئة جلسة أصلية خاصة بها
ويكون نقل مزوّد OpenClaw المعتاد هو التجريد غير المناسب.

أمثلة:

- خادم وكيل برمجة أصلي يملك الخيوط وCompaction
- CLI محلي أو عفريت يجب أن يبث أحداث الخطة/الاستدلال/الأدوات الأصلية
- بيئة تشغيل نموذج تحتاج إلى معرّف استئناف خاص بها إضافةً إلى نسخة جلسة
  OpenClaw

لا تسجّل حزاماً **فقط** لإضافة API جديد لنموذج LLM. بالنسبة إلى APIs النماذج
العادية عبر HTTP أو WebSocket، ابنِ [Plugin مزوّد](/ar/plugins/sdk-provider-plugins).

## ما الذي لا يزال اللب يملكه

قبل اختيار حزام، يكون OpenClaw قد حسم بالفعل:

- المزوّد والنموذج
- حالة مصادقة وقت التشغيل
- مستوى التفكير وميزانية السياق
- ملف نسخة/جلسة OpenClaw
- مساحة العمل، والصندوق الرملي، وسياسة الأدوات
- استدعاءات رد القناة واستدعاءات البث
- سياسة الرجوع الاحتياطي للنموذج والتبديل الحي للنموذج

هذا الفصل مقصود. يشغّل الحزام محاولة مُحضّرة؛ ولا يختار المزوّدين،
ولا يستبدل تسليم القناة، ولا يبدّل النماذج بصمت.

تتضمن المحاولة المُحضّرة أيضاً `params.runtimePlan`، وهي حزمة سياسات يملكها
OpenClaw لقرارات وقت التشغيل التي يجب أن تبقى مشتركة بين PI والأحزمة الأصلية:

- `runtimePlan.tools.normalize(...)` و
  `runtimePlan.tools.logDiagnostics(...)` لسياسة مخطط الأدوات الواعية بالمزوّد
- `runtimePlan.transcript.resolvePolicy(...)` لسياسة تنقية النسخة وإصلاح
  استدعاءات الأدوات
- `runtimePlan.delivery.isSilentPayload(...)` لكبت التسليم المشترك لـ `NO_REPLY`
  والوسائط
- `runtimePlan.outcome.classifyRunResult(...)` لتصنيف الرجوع الاحتياطي للنموذج
- `runtimePlan.observability` لبيانات وصف المزوّد/النموذج/الحزام المحسومة

يمكن للأحزمة استخدام الخطة للقرارات التي تحتاج إلى مطابقة سلوك PI، لكن
ينبغي أن تظل تعاملها كحالة محاولة يملكها المضيف. لا تعدّلها أو تستخدمها
لتبديل المزوّدين/النماذج داخل دورة.

## تسجيل حزام

**الاستيراد:** `openclaw/plugin-sdk/agent-harness`

```typescript
import type { AgentHarness } from "openclaw/plugin-sdk/agent-harness";
import { definePluginEntry } from "openclaw/plugin-sdk/plugin-entry";

const myHarness: AgentHarness = {
  id: "my-harness",
  label: "My native agent harness",

  supports(ctx) {
    return ctx.provider === "my-provider"
      ? { supported: true, priority: 100 }
      : { supported: false };
  },

  async runAttempt(params) {
    // Start or resume your native thread.
    // Use params.prompt, params.tools, params.images, params.onPartialReply,
    // params.onAgentEvent, and the other prepared attempt fields.
    return await runMyNativeTurn(params);
  },
};

export default definePluginEntry({
  id: "my-native-agent",
  name: "My Native Agent",
  description: "Runs selected models through a native agent daemon.",
  register(api) {
    api.registerAgentHarness(myHarness);
  },
});
```

## سياسة الاختيار

يختار OpenClaw حزاماً بعد حسم المزوّد/النموذج:

1. يفوز معرّف الحزام المسجّل في جلسة موجودة، بحيث لا تؤدي تغييرات الإعداد/البيئة
   إلى تبديل تلك النسخة فورياً إلى وقت تشغيل آخر.
2. يفرض `OPENCLAW_AGENT_RUNTIME=<id>` حزاماً مسجلاً بذلك المعرّف للجلسات
   غير المثبّتة مسبقاً.
3. يفرض `OPENCLAW_AGENT_RUNTIME=pi` حزام PI المضمّن.
4. يطلب `OPENCLAW_AGENT_RUNTIME=auto` من الأحزمة المسجّلة ما إذا كانت تدعم
   المزوّد/النموذج المحسوم.
5. إذا لم يطابق أي حزام مسجّل، يستخدم OpenClaw PI ما لم يكن الرجوع الاحتياطي
   إلى PI معطلاً.

تظهر إخفاقات أحزمة Plugin كإخفاقات تشغيل. في وضع `auto`، لا يُستخدم الرجوع
الاحتياطي إلى PI إلا عندما لا يدعم أي حزام Plugin مسجّل المزوّد/النموذج
المحسوم. بمجرد أن يطالب حزام Plugin بتشغيل، لا يعيد OpenClaw تشغيل الدورة
نفسها عبر PI لأن ذلك قد يغيّر دلالات المصادقة/وقت التشغيل أو يكرر الآثار الجانبية.

يُحفظ معرّف الحزام المختار مع معرّف الجلسة بعد تشغيل مضمّن.
تُعامل الجلسات القديمة المنشأة قبل تثبيتات الأحزمة على أنها مثبتة على PI بمجرد
أن يكون لديها سجل نسخة. استخدم جلسة جديدة/مُعاد ضبطها عند التغيير بين PI وحزام
Plugin أصلي. يعرض `/status` معرّفات الأحزمة غير الافتراضية مثل `codex`
بجوار `Fast`؛ يظل PI مخفياً لأنه مسار التوافق الافتراضي.
إذا كان الحزام المختار مفاجئاً، فعّل تسجيل تصحيح `agents/harness` وافحص سجل
Gateway المنظم `agent harness selected`. يتضمن معرّف الحزام المختار، وسبب
الاختيار، وسياسة وقت التشغيل/الرجوع الاحتياطي، وفي وضع `auto`، نتيجة دعم كل
مرشح Plugin.

يسجّل Plugin Codex المضمّن `codex` كمعرّف حزامه. يتعامل اللب مع ذلك كمعرّف
حزام Plugin عادي؛ الأسماء البديلة الخاصة بـ Codex مكانها في Plugin أو إعدادات
المشغّل، لا في محدد وقت التشغيل المشترك.

## اقتران المزوّد والحزام

ينبغي أن تسجّل معظم الأحزمة مزوّداً أيضاً. يجعل المزوّد مراجع النماذج، وحالة
المصادقة، وبيانات وصف النموذج، واختيار `/model` مرئية لبقية OpenClaw. ثم يطالب
الحزام بذلك المزوّد في `supports(...)`.

يتبع Plugin Codex المضمّن هذا النمط:

- مراجع نموذج المستخدم المفضلة: `openai/gpt-5.5` إضافةً إلى
  `agentRuntime.id: "codex"`
- مراجع التوافق: تظل مراجع `codex/gpt-*` القديمة مقبولة، لكن لا ينبغي أن
  تستخدمها الإعدادات الجديدة كمراجع مزوّد/نموذج عادية
- معرّف الحزام: `codex`
- المصادقة: إتاحة مزوّد اصطناعية، لأن حزام Codex يملك تسجيل دخول/جلسة Codex
  الأصلية
- طلب خادم التطبيق: يرسل OpenClaw معرّف النموذج المجرد إلى Codex ويترك الحزام
  يتحدث إلى بروتوكول خادم التطبيق الأصلي

Plugin Codex إضافي. تواصل مراجع `openai/gpt-*` العادية استخدام مسار مزوّد
OpenClaw المعتاد ما لم تفرض حزام Codex عبر `agentRuntime.id: "codex"`. لا تزال
مراجع `codex/gpt-*` الأقدم تختار مزوّد Codex وحزامه للتوافق.

لإعدادات المشغّل، وأمثلة بادئات النماذج، وإعدادات Codex فقط، راجع
[حزام Codex](/ar/plugins/codex-harness).

يتطلب OpenClaw خادم تطبيق Codex بإصدار `0.125.0` أو أحدث. يفحص Plugin Codex
مصافحة تهيئة خادم التطبيق ويحظر الخوادم الأقدم أو غير المرقّمة حتى لا يعمل
OpenClaw إلا على سطح البروتوكول الذي تم اختباره معه. يتضمن الحد الأدنى
`0.125.0` دعم حمولة خطاف MCP الأصلي الذي وصل في Codex `0.124.0`، مع تثبيت
OpenClaw على السلسلة المستقرة الأحدث المختبرة.

### برمجية وسيطة لنتائج الأدوات

يمكن للـ Pluginات المضمّنة إرفاق برمجية وسيطة محايدة لوقت التشغيل لنتائج
الأدوات عبر `api.registerAgentToolResultMiddleware(...)` عندما يصرّح بيانها
بمعرّفات وقت التشغيل المستهدفة في `contracts.agentToolResultMiddleware`. هذا
السيم الموثوق مخصص لتحويلات نتائج الأدوات غير المتزامنة التي يجب أن تعمل قبل
أن يعيد PI أو Codex تغذية خرج الأداة إلى النموذج.

لا تزال Pluginات المضمّنة القديمة قادرة على استخدام
`api.registerCodexAppServerExtensionFactory(...)` لبرمجيات وسيطة خاصة بخادم
تطبيق Codex فقط، لكن ينبغي أن تستخدم تحويلات النتائج الجديدة API المحايد لوقت
التشغيل. أزيل خطاف `api.registerEmbeddedExtensionFactory(...)` الخاص بـ Pi فقط؛
يجب أن تستخدم تحويلات نتائج أدوات Pi البرمجية الوسيطة المحايدة لوقت التشغيل.

### تصنيف النتيجة الطرفية

يمكن للأحزمة الأصلية التي تملك إسقاط البروتوكول الخاص بها استخدام
`classifyAgentHarnessTerminalOutcome(...)` من
`openclaw/plugin-sdk/agent-harness-runtime` عندما لا تنتج دورة مكتملة أي نص
مساعد مرئي. يعيد المساعد `empty` أو `reasoning-only` أو `planning-only` حتى
تقرر سياسة الرجوع الاحتياطي في OpenClaw ما إذا كانت ستعيد المحاولة على نموذج
مختلف. وهو يتعمد ترك أخطاء المطالبات، والدورات قيد التنفيذ، والردود الصامتة
المقصودة مثل `NO_REPLY` غير مصنّفة.

### وضع حزام Codex الأصلي

الحزام المضمّن `codex` هو وضع Codex الأصلي لدورات وكيل OpenClaw المضمّنة.
فعّل Plugin `codex` المضمّن أولاً، وأدرج `codex` في `plugins.allow` إذا كانت
إعداداتك تستخدم قائمة سماح مقيّدة. ينبغي أن تستخدم إعدادات خادم التطبيق الأصلية
`openai/gpt-*` مع `agentRuntime.id: "codex"`.
استخدم `openai-codex/*` لـ OAuth الخاص بـ Codex عبر PI بدلاً من ذلك. تظل مراجع
النماذج القديمة `codex/*` أسماء بديلة للتوافق مع الحزام الأصلي.

عند تشغيل هذا الوضع، يملك Codex معرّف الخيط الأصلي، وسلوك الاستئناف،
وCompaction، وتنفيذ خادم التطبيق. لا يزال OpenClaw يملك قناة المحادثة، ومرآة
النسخة المرئية، وسياسة الأدوات، والموافقات، وتسليم الوسائط، واختيار الجلسة.
استخدم `agentRuntime.id: "codex"` بدون تجاوز `fallback` عندما تحتاج إلى إثبات
أن مسار خادم تطبيق Codex فقط يمكنه المطالبة بالتشغيل. أوقات تشغيل Plugin
الصريحة تفشل مغلقة افتراضياً بالفعل. عيّن `fallback: "pi"` فقط عندما تريد
عمداً أن يتعامل PI مع غياب اختيار الحزام. إخفاقات خادم تطبيق Codex تفشل مباشرة
بالفعل بدلاً من إعادة المحاولة عبر PI.

## تعطيل الرجوع الاحتياطي إلى PI

افتراضياً، يشغّل OpenClaw الوكلاء المضمّنين مع ضبط
`agents.defaults.agentRuntime` على `{ id: "auto", fallback: "pi" }`. في وضع
`auto`، يمكن لأحزمة Plugin المسجّلة المطالبة بزوج مزوّد/نموذج. إذا لم يطابق
أي منها، يرجع OpenClaw احتياطياً إلى PI.

في وضع `auto`، عيّن `fallback: "none"` عندما تحتاج إلى أن يفشل غياب اختيار
حزام Plugin بدلاً من استخدام PI. أوقات تشغيل Plugin الصريحة مثل
`agentRuntime.id: "codex"` تفشل مغلقة افتراضياً بالفعل، ما لم يُعيّن
`fallback: "pi"` في الإعداد نفسه أو في نطاق تجاوز البيئة نفسه. إخفاقات حزام
Plugin المختار تفشل دائماً بشكل حاسم. لا يمنع هذا `agentRuntime.id: "pi"` صريحاً
أو `OPENCLAW_AGENT_RUNTIME=pi`.

لتشغيلات Codex المضمّنة فقط:

```json
{
  "agents": {
    "defaults": {
      "model": "openai/gpt-5.5",
      "agentRuntime": {
        "id": "codex"
      }
    }
  }
}
```

إذا أردت أن يطالب أي حزام Plugin مسجّل بالنماذج المطابقة لكنك لا تريد أبداً
أن يرجع OpenClaw بصمت إلى PI، أبقِ `runtime: "auto"` وعطّل الرجوع الاحتياطي:

```json
{
  "agents": {
    "defaults": {
      "agentRuntime": {
        "id": "auto",
        "fallback": "none"
      }
    }
  }
}
```

تستخدم تجاوزات كل وكيل الشكل نفسه:

```json
{
  "agents": {
    "defaults": {
      "agentRuntime": {
        "id": "auto",
        "fallback": "pi"
      }
    },
    "list": [
      {
        "id": "codex-only",
        "model": "openai/gpt-5.5",
        "agentRuntime": {
          "id": "codex",
          "fallback": "none"
        }
      }
    ]
  }
}
```

لا يزال `OPENCLAW_AGENT_RUNTIME` يتجاوز وقت التشغيل المكوّن. استخدم
`OPENCLAW_AGENT_HARNESS_FALLBACK=none` لتعطيل الرجوع الاحتياطي إلى PI من البيئة.

```bash
OPENCLAW_AGENT_RUNTIME=codex \
OPENCLAW_AGENT_HARNESS_FALLBACK=none \
openclaw gateway run
```

مع تعطيل الرجوع الاحتياطي، تفشل الجلسة مبكراً عندما لا يكون الحزام المطلوب
مسجلاً، أو لا يدعم المزوّد/النموذج المحسوم، أو يفشل قبل إنتاج آثار جانبية
للدورة. هذا مقصود لنشرات Codex فقط وللاختبارات الحية التي يجب أن تثبت أن مسار
خادم تطبيق Codex قيد الاستخدام فعلاً.

يتحكم هذا الإعداد فقط في حزام الوكيل المضمّن. ولا يعطّل توجيه نماذج الصور، أو
الفيديو، أو الموسيقى، أو TTS، أو PDF، أو غيرها من التوجيهات الخاصة بالمزوّدين.

## الجلسات الأصلية ومرآة النسخة

قد يحتفظ الحزام بمعرّف جلسة أصلي، أو معرّف خيط، أو رمز استئناف من جهة العفريت.
أبقِ ذلك الربط مرتبطاً صراحةً بجلسة OpenClaw، وواصل عكس خرج المساعد/الأداة
المرئي للمستخدم في نسخة OpenClaw.

تظل نسخة OpenClaw طبقة التوافق من أجل:

- سجل الجلسة المرئي في القناة
- بحث النسخ وفهرستها
- التبديل مرة أخرى إلى حزام PI المضمّن في دورة لاحقة
- سلوك `/new` و`/reset` العام وحذف الجلسات

إذا كان حزامك يخزّن ربطاً جانبياً، فنفّذ `reset(...)` حتى يتمكن OpenClaw من
مسحه عند إعادة ضبط جلسة OpenClaw المالكة.

## نتائج الأدوات والوسائط

ينشئ Core قائمة أدوات OpenClaw ويمرّرها إلى المحاولة المُحضّرة.
عندما ينفّذ harness استدعاء أداة ديناميكيًا، أعد نتيجة الأداة عبر
شكل نتيجة harness بدلًا من إرسال وسائط القناة بنفسك.

يبقي هذا مخرجات النصوص والصور والفيديو والموسيقى وTTS والموافقة وأدوات المراسلة
على مسار التسليم نفسه مثل عمليات التشغيل المدعومة بـ Pi.

## القيود الحالية

- مسار الاستيراد العام عام، لكن بعض الأسماء البديلة لأنواع المحاولات/النتائج لا تزال
  تحمل أسماء `Pi` للتوافق.
- تثبيت harness تابع لجهة خارجية تجريبي. فضّل Plugins المزوّدين
  حتى تحتاج إلى وقت تشغيل جلسة أصلي.
- تبديل harness مدعوم عبر الأدوار. لا تبدّل harness في
  منتصف دور بعد بدء الأدوات الأصلية أو الموافقات أو نص المساعد أو عمليات إرسال الرسائل.
