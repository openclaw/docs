يحافظ OpenClaw على عقود Plugin الأقدم موصولة عبر محوّلات توافق مسماة
قبل إزالتها. يحمي هذا plugins المضمّنة والخارجية الحالية بينما تتطور
عقود SDK والبيان والإعداد والتكوين ووقت تشغيل الوكيل.

## سجل التوافق

تُتتبّع عقود توافق Plugin في السجل الأساسي في
`src/plugins/compat/registry.ts`.

يحتوي كل سجل على:

- رمز توافق مستقر
- الحالة: `active` أو `deprecated` أو `removal-pending` أو `removed`
- المالك: SDK، أو التكوين، أو الإعداد، أو القناة، أو المزوّد، أو تنفيذ Plugin، أو وقت تشغيل الوكيل،
  أو النواة
- تواريخ الإدخال والإهمال عند الاقتضاء
- إرشادات الاستبدال
- الوثائق والتشخيصات والاختبارات التي تغطي السلوك القديم والجديد

السجل هو المصدر لتخطيط المشرفين وفحوصات مفتش Plugin المستقبلية. إذا تغيّر
سلوك مواجه لـ Plugin، فأضف سجل التوافق أو حدّثه في التغيير نفسه الذي يضيف
المحوّل.

يُتتبّع توافق إصلاح الطبيب والترحيل بشكل منفصل في
`src/commands/doctor/shared/deprecation-compat.ts`. تغطي هذه السجلات أشكال
التكوين القديمة، وتخطيطات سجل التثبيت، وشِيمات الإصلاح التي قد تحتاج إلى
البقاء متاحة بعد إزالة مسار توافق وقت التشغيل.

يجب أن تتحقق مسوحات الإصدار من كلا السجلين. لا تحذف ترحيل الطبيب لمجرد أن
سجل توافق وقت التشغيل أو التكوين المطابق قد انتهت صلاحيته؛ تحقّق أولاً من
عدم وجود مسار ترقية مدعوم لا يزال يحتاج إلى الإصلاح. وأعد أيضاً التحقق من
كل تعليمة استبدال أثناء تخطيط الإصدار، لأن ملكية Plugin وبصمة التكوين قد
تتغيران مع انتقال المزوّدين والقنوات إلى خارج النواة.

## حزمة مفتش Plugin

يجب أن يعيش مفتش Plugin خارج مستودع OpenClaw الأساسي كحزمة/مستودع منفصل
مدعوم بعقود التوافق والبيان ذات الإصدارات.

يجب أن تكون CLI لليوم الأول:

```sh
openclaw-plugin-inspector ./my-plugin
```

يجب أن تُخرج:

- تحقق البيان/المخطط
- إصدار توافق العقد الذي يجري فحصه
- فحوصات بيانات تعريف التثبيت/المصدر
- فحوصات استيراد المسار البارد
- تحذيرات الإهمال والتوافق

استخدم `--json` للحصول على مخرجات مستقرة قابلة للقراءة آلياً في تعليقات CI. يجب
أن تكشف نواة OpenClaw العقود والتجهيزات التي يمكن للمفتش استهلاكها، لكن يجب
ألا تنشر ملف المفتش التنفيذي من حزمة `openclaw` الرئيسية.

### مسار قبول المشرفين

استخدم Blacksmith Testbox لمسار قبول الحزمة القابلة للتثبيت عند التحقق من
المفتش الخارجي مقابل حزم Plugin الخاصة بـ OpenClaw. شغّله من نسخة OpenClaw
نظيفة بعد بناء الحزمة:

```sh
blacksmith testbox warmup ci-check-testbox.yml --ref main --idle-timeout 90
blacksmith testbox run --id <tbx_id> "pnpm install && pnpm build && npm exec --yes @openclaw/plugin-inspector@0.1.0 -- ./extensions/telegram --json"
blacksmith testbox run --id <tbx_id> "npm exec --yes @openclaw/plugin-inspector@0.1.0 -- ./extensions/discord --json"
blacksmith testbox run --id <tbx_id> "npm exec --yes @openclaw/plugin-inspector@0.1.0 -- <clawhub-plugin-dir> --json"
blacksmith testbox stop <tbx_id>
```

أبقِ هذا المسار اختيارياً للمشرفين لأنه يثبّت حزمة npm خارجية وقد يفحص حزم
Plugin مستنسخة خارج المستودع. تغطي حراس المستودع المحلي خريطة تصدير SDK،
وبيانات تعريف سجل التوافق، وخفض استيرادات SDK المهملة، وحدود استيراد
الإضافات المضمّنة؛ بينما يغطّي دليل مفتش Testbox الحزمة كما يستهلكها مؤلفو
Plugin الخارجيون.

## سياسة الإهمال

يجب ألا يزيل OpenClaw عقد Plugin موثقاً في الإصدار نفسه الذي يقدّم بديله.

تسلسل الترحيل هو:

1. أضف العقد الجديد.
2. أبقِ السلوك القديم موصولاً عبر محوّل توافق مسمى.
3. أصدِر تشخيصات أو تحذيرات عندما يستطيع مؤلفو Plugin اتخاذ إجراء.
4. وثّق البديل والجدول الزمني.
5. اختبر المسارين القديم والجديد.
6. انتظر خلال نافذة الترحيل المعلنة.
7. أزل فقط بموافقة صريحة على إصدار كاسر للتوافق.

يجب أن تتضمن السجلات المهملة تاريخ بدء التحذير، والبديل، ورابط الوثائق،
وتاريخ الإزالة النهائي خلال مدة لا تتجاوز ثلاثة أشهر بعد بدء التحذير. لا
تضف مسار توافق مهملاً مع نافذة إزالة مفتوحة ما لم يقرر المشرفون صراحة أنه
توافق دائم ويضعوا عليه علامة `active` بدلاً من ذلك.

## مناطق التوافق الحالية

تشمل سجلات التوافق الحالية:

- استيرادات SDK الواسعة القديمة مثل `openclaw/plugin-sdk/compat`
- أشكال Plugin القديمة المعتمدة على الخطافات فقط و`before_agent_start`
- نقاط دخول Plugin القديمة `activate(api)` بينما تنتقل plugins إلى
  `register(api)`
- أسماء SDK المستعارة القديمة مثل `openclaw/extension-api`،
  و`openclaw/plugin-sdk/channel-runtime`، وبُناة حالة `openclaw/plugin-sdk/command-auth`،
  و`openclaw/plugin-sdk/test-utils` (استُبدلت بمسارات اختبار فرعية مركّزة
  `openclaw/plugin-sdk/*`)، والاسمين المستعارين للنوع `ClawdbotConfig` /
  `OpenClawSchemaType`
- قائمة سماح Plugin المضمّنة وسلوك التمكين
- بيانات تعريف بيان متغيرات بيئة المزوّد/القناة القديمة
- خطافات Plugin الخاصة بالمزوّد والأسماء المستعارة للأنواع القديمة بينما ينتقل المزوّدون إلى
  خطافات صريحة للفهرس، والمصادقة، والتفكير، وإعادة التشغيل، والنقل
- أسماء وقت التشغيل المستعارة القديمة مثل `api.runtime.taskFlow`،
  و`api.runtime.subagent.getSession`، و`api.runtime.stt`، و
  `api.runtime.config.loadConfig()` / `api.runtime.config.writeConfigFile(...)` المهملة
- تسجيل تقسيم Plugin الذاكرة القديم بينما تنتقل plugins الذاكرة إلى
  `registerMemoryCapability`
- مساعدات SDK للقنوات القديمة لمخططات الرسائل الأصلية، وبوابة الإشارات،
  وتنسيق مغلف الوارد، وتداخل قدرة الموافقة
- مفتاح مسار القناة القديم وأسماء مساعد الهدف القابل للمقارنة المستعارة بينما تنتقل plugins
  إلى `openclaw/plugin-sdk/channel-route`
- تلميحات التنشيط التي يستبدلها امتلاك مساهمات البيان
- تحميل sidecar بدء التشغيل الضمني المهمل لـ plugins التي لم تعلن
  `activation.onStartup`؛ يمكن للمشرفين اختبار السلوك المستقبلي الأكثر صرامة باستخدام
  `OPENCLAW_DISABLE_LEGACY_IMPLICIT_STARTUP_SIDECARS=1`
- احتياطي وقت التشغيل `setup-api` بينما تنتقل واصفات الإعداد إلى بيانات تعريف باردة
  `setup.requiresRuntime: false`
- خطافات `discovery` للمزوّد بينما تنتقل خطافات فهرس المزوّد إلى
  `catalog.run(...)`
- بيانات تعريف القناة `showConfigured` / `showInSetup` بينما تنتقل حزم القنوات
  إلى `openclaw.channel.exposure`
- مفاتيح تكوين سياسة وقت التشغيل القديمة بينما يرحّل الطبيب المشغلين إلى
  `agentRuntime`
- احتياطي بيانات تعريف تكوين القناة المضمّنة المولّدة بينما تصل بيانات تعريف
  `channelConfigs` المعتمدة على السجل أولاً
- علامات بيئة تعطيل سجل Plugin المستمر وترحيل التثبيت بينما ترحّل تدفقات
  الإصلاح المشغلين إلى `openclaw plugins registry --refresh` و
  `openclaw doctor --fix`
- مسارات تكوين البحث في الويب، وجلب الويب، وx_search القديمة المملوكة لـ Plugin بينما
  يرحّلها الطبيب إلى `plugins.entries.<plugin>.config`
- تكوين `plugins.installs` المؤلَّف القديم وأسماء مسار تحميل Plugin المضمّنة المستعارة
  بينما تنتقل بيانات تعريف التثبيت إلى سجل Plugin المُدار بالحالة

يجب أن تفضّل شفرة Plugin الجديدة البديل المدرج في السجل وفي دليل الترحيل
المحدد. يمكن لـ plugins الحالية أن تواصل استخدام مسار توافق إلى أن تعلن
الوثائق والتشخيصات وملاحظات الإصدار نافذة إزالة.

## ملاحظات الإصدار

يجب أن تتضمن ملاحظات الإصدار إهمالات Plugin القادمة مع التواريخ المستهدفة
وروابط إلى وثائق الترحيل. يجب أن يحدث ذلك التحذير قبل أن ينتقل مسار توافق
إلى `removal-pending` أو `removed`.
