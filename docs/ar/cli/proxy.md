# `openclaw proxy`

تحقق من صحة توجيه الوكيل المُدار من المشغّل، أو شغّل وكيل التصحيح المحلي الصريح
وافحص حركة المرور الملتقطة.

استخدم `validate` لإجراء فحص مسبق لوكيل إعادة توجيه مُدار من المشغّل قبل تفعيل
توجيه وكيل OpenClaw. الأوامر الأخرى هي أدوات تصحيح لتحقيقات
مستوى النقل: يمكنها بدء وكيل محلي، وتشغيل أمر فرعي
مع تفعيل الالتقاط، وسرد جلسات الالتقاط، والاستعلام عن أنماط حركة المرور الشائعة، وقراءة
البيانات الثنائية الملتقطة، وحذف بيانات الالتقاط المحلية.

## الأوامر

```bash
openclaw proxy start [--host <host>] [--port <port>]
openclaw proxy run [--host <host>] [--port <port>] -- <cmd...>
openclaw proxy validate [--json] [--proxy-url <url>] [--allowed-url <url>] [--denied-url <url>] [--timeout-ms <ms>]
openclaw proxy coverage
openclaw proxy sessions [--limit <count>]
openclaw proxy query --preset <name> [--session <id>]
openclaw proxy blob --id <blobId>
openclaw proxy purge
```

## التحقق

يتحقق `openclaw proxy validate` من عنوان URL الفعّال للوكيل المُدار من المشغّل من
`--proxy-url` أو الإعدادات أو `OPENCLAW_PROXY_URL`. ويبلّغ عن مشكلة في الإعدادات عندما
لا يكون أي وكيل مفعّلًا ومُعدًا؛ استخدم `--proxy-url` لإجراء فحص مسبق لمرة واحدة
قبل تغيير الإعدادات. افتراضيًا، يتحقق من نجاح الوصول إلى وجهة عامة
عبر الوكيل ومن أن الوكيل لا يمكنه الوصول إلى مؤشر canary مؤقت لـ loopback.
الوجهات المرفوضة المخصصة تكون مغلقة عند الفشل: استجابات HTTP وحالات فشل
النقل الغامضة كلاهما يفشل ما لم تتمكن من التحقق من إشارة رفض خاصة بالنشر
بشكل منفصل.

الخيارات:

- `--json`: اطبع JSON قابلًا للقراءة آليًا.
- `--proxy-url <url>`: تحقق من عنوان URL هذا للوكيل بدلًا من الإعدادات أو البيئة.
- `--allowed-url <url>`: أضف وجهة يُتوقع أن تنجح عبر الوكيل. كرر ذلك للتحقق من وجهات متعددة.
- `--denied-url <url>`: أضف وجهة يُتوقع أن يحظرها الوكيل. كرر ذلك للتحقق من وجهات متعددة.
- `--timeout-ms <ms>`: مهلة كل طلب بالمللي ثانية.

راجع [وكيل الشبكة](/ar/security/network-proxy) للحصول على إرشادات النشر ودلالات
الرفض.

## الإعدادات المسبقة للاستعلام

يقبل `openclaw proxy query --preset <name>` ما يلي:

- `double-sends`
- `retry-storms`
- `cache-busting`
- `ws-duplicate-frames`
- `missing-ack`
- `error-bursts`

## ملاحظات

- يستخدم `start` القيمة الافتراضية `127.0.0.1` ما لم يتم تعيين `--host`.
- يبدأ `run` وكيل تصحيح محليًا ثم يشغّل الأمر بعد `--`.
- يخرج `validate` بالرمز 1 عندما تفشل إعدادات الوكيل أو فحوصات الوجهة.
- الالتقاطات هي بيانات تصحيح محلية؛ استخدم `openclaw proxy purge` عند الانتهاء.
