# `openclaw backup`

أنشئ أرشيف نسخ احتياطي محليًا لحالة OpenClaw، والإعدادات، وملفات تعريف المصادقة، وبيانات اعتماد القنوات/المزوّدين، والجلسات، ومساحات العمل اختياريًا.

```bash
openclaw backup create
openclaw backup create --output ~/Backups
openclaw backup create --dry-run --json
openclaw backup create --verify
openclaw backup create --no-include-workspace
openclaw backup create --only-config
openclaw backup verify ./2026-03-09T00-00-00.000Z-openclaw-backup.tar.gz
```

## ملاحظات

- يتضمن الأرشيف ملف `manifest.json` يحتوي على مسارات المصدر المحلولة وتخطيط الأرشيف.
- يكون الإخراج الافتراضي أرشيف `.tar.gz` مؤرشفًا زمنيًا في دليل العمل الحالي.
- إذا كان دليل العمل الحالي داخل شجرة مصدر يتم نسخها احتياطيًا، فسيعود OpenClaw إلى دليلك المنزلي كموقع افتراضي للأرشيف.
- لا يتم أبدًا الكتابة فوق ملفات الأرشيف الموجودة.
- تُرفض مسارات الإخراج الموجودة داخل أشجار المصدر للحالة/مساحة العمل لتجنب تضمين الأرشيف لنفسه.
- يتحقق `openclaw backup verify <archive>` من أن الأرشيف يحتوي على manifest جذري واحد فقط، ويرفض مسارات الأرشيف بأسلوب الاجتياز، ويتحقق من وجود كل حمولة معلنة في manifest داخل tarball.
- يشغّل `openclaw backup create --verify` هذا التحقق مباشرة بعد كتابة الأرشيف.
- يقوم `openclaw backup create --only-config` بنسخ ملف إعداد JSON النشط فقط احتياطيًا.

## ما الذي يتم نسخه احتياطيًا

يخطط `openclaw backup create` لمصادر النسخ الاحتياطي من تثبيت OpenClaw المحلي لديك:

- دليل الحالة الذي يعيده محلل الحالة المحلية في OpenClaw، وعادةً `~/.openclaw`
- مسار ملف الإعداد النشط
- دليل `credentials/` المحلول عندما يكون موجودًا خارج دليل الحالة
- أدلة مساحة العمل المكتشفة من الإعداد الحالي، ما لم تمرر `--no-include-workspace`

تكون ملفات تعريف مصادقة النموذج جزءًا بالفعل من دليل الحالة ضمن
`agents/<agentId>/agent/auth-profiles.json`، لذا فهي عادةً مشمولة ضمن
إدخال النسخ الاحتياطي للحالة.

إذا استخدمت `--only-config`، فسيتجاوز OpenClaw اكتشاف الحالة، ودليل بيانات الاعتماد، ومساحة العمل، وسيؤرشف مسار ملف الإعداد النشط فقط.

يقوم OpenClaw بتوحيد المسارات قبل بناء الأرشيف. إذا كانت الإعدادات أو
دليل بيانات الاعتماد أو مساحة العمل موجودة بالفعل داخل دليل الحالة،
فلن يتم تكرارها كمصادر نسخ احتياطي مستقلة من المستوى الأعلى. ويتم تخطي
المسارات المفقودة.

تخزّن حمولة الأرشيف محتويات الملفات من أشجار المصدر هذه، ويسجّل `manifest.json` المضمّن مسارات المصدر المطلقة المحلولة بالإضافة إلى تخطيط الأرشيف المستخدم لكل أصل.

## سلوك الإعدادات غير الصالحة

يتجاوز `openclaw backup` عمدًا الفحص التمهيدي العادي للإعدادات حتى يظل مفيدًا أثناء الاستعادة. ولأن اكتشاف مساحة العمل يعتمد على إعداد صالح، فإن `openclaw backup create` يفشل سريعًا الآن عندما يكون ملف الإعداد موجودًا لكنه غير صالح بينما لا يزال النسخ الاحتياطي لمساحة العمل مفعّلًا.

إذا كنت لا تزال تريد نسخة احتياطية جزئية في هذه الحالة، فأعد التشغيل باستخدام:

```bash
openclaw backup create --no-include-workspace
```

يُبقي هذا الحالة، والإعدادات، ودليل بيانات الاعتماد الخارجي ضمن النطاق مع
تخطي اكتشاف مساحة العمل بالكامل.

إذا كنت تحتاج فقط إلى نسخة من ملف الإعداد نفسه، فإن `--only-config` يعمل أيضًا عندما تكون الإعدادات معطوبة لأنه لا يعتمد على تحليل الإعدادات لاكتشاف مساحة العمل.

## الحجم والأداء

لا يفرض OpenClaw حدًا أقصى مدمجًا لحجم النسخ الاحتياطي أو حدًا لحجم كل ملف.

تأتي الحدود العملية من الجهاز المحلي ونظام الملفات الوجهة:

- المساحة المتاحة لكتابة الأرشيف المؤقت بالإضافة إلى الأرشيف النهائي
- الوقت اللازم لاجتياز أشجار مساحة العمل الكبيرة وضغطها إلى `.tar.gz`
- الوقت اللازم لإعادة فحص الأرشيف إذا استخدمت `openclaw backup create --verify` أو شغّلت `openclaw backup verify`
- سلوك نظام الملفات في مسار الوجهة. يفضّل OpenClaw خطوة نشر hard-link بدون كتابة فوقية، ويعود إلى النسخ الحصري عندما لا تكون hard links مدعومة

تكون مساحات العمل الكبيرة عادةً العامل الرئيسي في حجم الأرشيف. إذا كنت تريد نسخة احتياطية أصغر أو أسرع، فاستخدم `--no-include-workspace`.

ولأصغر أرشيف ممكن، استخدم `--only-config`.
