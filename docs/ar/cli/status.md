تشخيص القنوات + الجلسات.

```bash
openclaw status
openclaw status --all
openclaw status --deep
openclaw status --usage
```

ملاحظات:

- يشغّل `--deep` مجسات مباشرة (WhatsApp Web + Telegram + Discord + Slack + Signal).
- يبقى `openclaw status` العادي على المسار السريع للقراءة فقط، ويضع علامة على الذاكرة بأنها `not checked` بدلا من غير متاحة عندما يتخطى فحص الذاكرة. تُترك عملية تدقيق الأمان الثقيلة، وتوافق Plugin، ومجسات متجهات الذاكرة إلى `openclaw status --all`، و`openclaw status --deep`، و`openclaw security audit`، و`openclaw memory status --deep`.
- يبلّغ `status --json --all` عن تفاصيل الذاكرة من بيئة تشغيل Plugin الذاكرة النشطة التي يحددها `plugins.slots.memory`. يمكن لإضافات الذاكرة المخصصة ترك `agents.defaults.memorySearch.enabled` المضمّن معطلا ومع ذلك الإبلاغ عن ملفاتها وأجزائها ومتجهاتها وحالة FTS الخاصة بها.
- يطبع `--usage` نوافذ استخدام المزوّد المعيارية بصيغة `X% left`.
- يفصل خرج حالة الجلسة بين `Execution:` و`Runtime:`. يمثل `Execution` مسار صندوق العزل (`direct`، و`docker/*`)، بينما يخبرك `Runtime` بما إذا كانت الجلسة تستخدم `OpenClaw Default`، أو `OpenAI Codex`، أو خلفية CLI، أو خلفية ACP مثل `codex (acp/acpx)`. راجع [بيئات تشغيل الوكيل](/ar/concepts/agent-runtimes) لمعرفة الفرق بين المزوّد/النموذج/بيئة التشغيل.
- حقول MiniMax الخام `usage_percent` / `usagePercent` هي الحصة المتبقية، لذلك يعكسها OpenClaw قبل العرض؛ وتكون الغلبة للحقول المعتمدة على العد عند وجودها. تفضّل استجابات `model_remains` إدخال نموذج المحادثة، وتشتق تسمية النافذة من الطوابع الزمنية عند الحاجة، وتضمّن اسم النموذج في تسمية الخطة.
- عندما تكون لقطة الجلسة الحالية محدودة، يمكن لـ `/status` تعبئة عدادات الرموز المميزة وذاكرة التخزين المؤقت من أحدث سجل استخدام في النص المنسوخ. تظل القيم المباشرة غير الصفرية الموجودة لها الأولوية على قيم الرجوع إلى النص المنسوخ.
- يتضمن `/status` مدة تشغيل عملية Gateway الموجزة ومدة تشغيل نظام المضيف.
- يمكن لآلية الرجوع إلى النص المنسوخ أيضا استرداد تسمية نموذج بيئة التشغيل النشطة عندما يفتقدها إدخال الجلسة المباشر. إذا اختلف نموذج النص المنسوخ هذا عن النموذج المحدد، فستحل الحالة نافذة السياق وفق نموذج بيئة التشغيل المسترد بدلا من النموذج المحدد.
- عندما تكون جلسة مثبتة على نموذج يختلف عن الأساسي المكوّن، تطبع الحالة القيمتين معا، والسبب (`session override`)، والتلميح الواضح (`/model default`). ينطبق الأساسي المكوّن على الجلسات الجديدة أو غير المثبتة؛ أما الجلسات المثبتة الحالية فتحتفظ باختيار الجلسة إلى أن يتم مسحه.
- لمحاسبة حجم الموجّه، تفضّل آلية الرجوع إلى النص المنسوخ الإجمالي الأكبر الموجه إلى الموجّه عندما تكون بيانات تعريف الجلسة مفقودة أو أصغر، لذلك لا تنهار جلسات المزوّد المخصص إلى عروض رموز مميزة بقيمة `0`.
- يتضمن الخرج مخازن جلسات لكل وكيل عند تكوين عدة وكلاء.
- تتضمن النظرة العامة حالة تثبيت/تشغيل خدمة مضيف Gateway + Node عند توفرها.
- تتضمن النظرة العامة قناة التحديث + SHA الخاص بـ git (لنسخ المصدر المحلية).
- تظهر معلومات التحديث في النظرة العامة؛ إذا كان هناك تحديث متاح، تطبع الحالة تلميحا لتشغيل `openclaw update` (راجع [التحديث](/ar/install/updating)).
- تُعرض إخفاقات تحديث تسعير النماذج كتحذيرات تسعير اختيارية. وهي لا
  تعني أن Gateway أو القنوات غير سليمة.
- تقوم أسطح الحالة للقراءة فقط (`status`، و`status --json`، و`status --all`) بحل SecretRefs المدعومة لمسارات التكوين المستهدفة الخاصة بها عندما يكون ذلك ممكنا.
- إذا كان SecretRef لقناة مدعومة مكوّنا لكنه غير متاح في مسار الأمر الحالي، تظل الحالة للقراءة فقط وتبلّغ عن خرج متدهور بدلا من التعطل. يعرض الخرج البشري تحذيرات مثل "الرمز المميز المكوّن غير متاح في مسار الأمر هذا"، ويتضمن خرج JSON المفتاح `secretDiagnostics`.
- عند نجاح حل SecretRef المحلي للأمر، تفضّل الحالة اللقطة المحلولة وتمسح علامات القناة العابرة التي تفيد بأن "السر غير متاح" من الخرج النهائي.
- يتضمن `status --all` صف نظرة عامة على الأسرار وقسم تشخيص يلخص تشخيصات الأسرار (مقتطعة لسهولة القراءة) دون إيقاف إنشاء التقرير.
