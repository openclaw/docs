تشخيص للقنوات + الجلسات.

```bash
openclaw status
openclaw status --all
openclaw status --deep
openclaw status --usage
```

ملاحظات:

- يشغّل `--deep` مجسّات مباشرة (WhatsApp Web + Telegram + Discord + Slack + Signal).
- يبقى `openclaw status` العادي على المسار السريع للقراءة فقط ويضع علامة `not checked` على الذاكرة بدلًا من غير متاحة عندما يتخطّى فحص الذاكرة. تُترك مراجعة الأمان الثقيلة، وتوافق Plugin، ومجسّات متجهات الذاكرة إلى `openclaw status --all` و`openclaw status --deep` و`openclaw security audit` و`openclaw memory status --deep`.
- يبلّغ `status --json --all` عن تفاصيل الذاكرة من وقت تشغيل Plugin الذاكرة النشط المحدد بواسطة `plugins.slots.memory`. يمكن لـ Plugins الذاكرة المخصّصة أن تترك `agents.defaults.memorySearch.enabled` المدمجة معطّلة ومع ذلك تبلّغ عن ملفاتها، ومقاطعها، ومتجهاتها، وحالة FTS الخاصة بها.
- يطبع `--usage` نوافذ استخدام المزوّد الموحّدة بصيغة `X% left`.
- يفصل خرج حالة الجلسة بين `Execution:` و`Runtime:`. `Execution` هو مسار صندوق العزل (`direct`، `docker/*`)، بينما يخبرك `Runtime` بما إذا كانت الجلسة تستخدم `OpenClaw Pi Default` أو `OpenAI Codex` أو خلفية CLI أو خلفية ACP مثل `codex (acp/acpx)`. راجع [أوقات تشغيل الوكلاء](/ar/concepts/agent-runtimes) لمعرفة الفرق بين المزوّد/النموذج/وقت التشغيل.
- حقلا MiniMax الخامّان `usage_percent` / `usagePercent` يمثّلان الحصة المتبقية، لذلك يعكسهما OpenClaw قبل العرض؛ وتكون الأولوية للحقول المعتمدة على العدّ عند وجودها. تفضّل استجابات `model_remains` إدخال نموذج المحادثة، وتستنتج تسمية النافذة من الطوابع الزمنية عند الحاجة، وتضمّن اسم النموذج في تسمية الخطة.
- عندما تكون لقطة الجلسة الحالية متفرقة، يمكن لـ`/status` إكمال عدّادات الرموز والتخزين المؤقت من أحدث سجل استخدام للنص المنسوخ. تظل القيم الحية غير الصفرية الحالية متقدمة على قيم الرجوع إلى النص المنسوخ.
- يتضمن `/status` مدة تشغيل عملية Gateway الموجزة ومدة تشغيل نظام المضيف.
- يمكن للرجوع إلى النص المنسوخ أيضًا استعادة تسمية نموذج وقت التشغيل النشط عندما يفتقدها إدخال الجلسة الحية. إذا كان نموذج النص المنسوخ هذا مختلفًا عن النموذج المحدد، فستحل الحالة نافذة السياق مقابل نموذج وقت التشغيل المستعاد بدلًا من النموذج المحدد.
- بالنسبة إلى احتساب حجم الموجّه، يفضّل الرجوع إلى النص المنسوخ الإجمالي الأكبر الموجّه نحو الموجّه عندما تكون بيانات تعريف الجلسة مفقودة أو أصغر، بحيث لا تنهار جلسات المزوّد المخصّص إلى عروض رموز `0`.
- يتضمن الخرج مخازن جلسات لكل وكيل عند تكوين عدة وكلاء.
- تتضمن النظرة العامة حالة تثبيت/تشغيل خدمة مضيف Gateway + node عند توفرها.
- تتضمن النظرة العامة قناة التحديث + SHA الخاص بـgit (لنسخ المصدر).
- تظهر معلومات التحديث في النظرة العامة؛ إذا كان هناك تحديث متاح، تطبع الحالة تلميحًا لتشغيل `openclaw update` (راجع [التحديث](/ar/install/updating)).
- تحل أسطح الحالة للقراءة فقط (`status`، `status --json`، `status --all`) SecretRefs المدعومة لمسارات التكوين المستهدفة الخاصة بها عند الإمكان.
- إذا تم تكوين SecretRef لقناة مدعومة ولكنه غير متاح في مسار الأمر الحالي، تبقى الحالة للقراءة فقط وتبلّغ عن خرج متدهور بدلًا من التعطل. يعرض الخرج البشري تحذيرات مثل "الرمز المميز المكوّن غير متاح في مسار الأمر هذا"، ويتضمن خرج JSON `secretDiagnostics`.
- عندما ينجح حل SecretRef المحلي للأمر، تفضّل الحالة اللقطة المحلولة وتمسح علامات القناة المؤقتة "السر غير متاح" من الخرج النهائي.
- يتضمن `status --all` صف نظرة عامة على الأسرار وقسم تشخيص يلخص تشخيصات الأسرار (مختصرة لسهولة القراءة) دون إيقاف إنشاء التقرير.
