# `openclaw status`

تشخيصات للقنوات + الجلسات.

```bash
openclaw status
openclaw status --all
openclaw status --deep
openclaw status --usage
```

ملاحظات:

- يشغّل `--deep` عمليات فحص مباشرة (WhatsApp Web + Telegram + Discord + Slack + Signal).
- يبقى `openclaw status` العادي على المسار السريع للقراءة فقط، ويضع علامة على الذاكرة بأنها `not checked` بدلاً من غير متاحة عندما يتخطى فحص الذاكرة. تُترك عملية التدقيق الأمني الثقيلة، وتوافق Plugin، وفحوصات متجهات الذاكرة إلى `openclaw status --all`، و`openclaw status --deep`، و`openclaw security audit`، و`openclaw memory status --deep`.
- يعرض `status --json --all` تفاصيل الذاكرة من وقت تشغيل Plugin الذاكرة النشط المحدد بواسطة `plugins.slots.memory`. يمكن لـ Plugins الذاكرة المخصصة ترك `agents.defaults.memorySearch.enabled` المدمج معطلاً ومع ذلك الإبلاغ عن ملفاتها وقطعها ومتجهاتها وحالة FTS الخاصة بها.
- يطبع `--usage` نوافذ استخدام المزوّد بعد تطبيعها بصيغة `X% left`.
- يفصل إخراج حالة الجلسة بين `Execution:` و`Runtime:`. يشير `Execution` إلى مسار sandbox (`direct`، `docker/*`)، بينما يخبرك `Runtime` بما إذا كانت الجلسة تستخدم `OpenClaw Pi Default`، أو `OpenAI Codex`، أو واجهة خلفية CLI، أو واجهة خلفية ACP مثل `codex (acp/acpx)`. راجع [أوقات تشغيل الوكيل](/ar/concepts/agent-runtimes) لمعرفة الفرق بين المزوّد/النموذج/وقت التشغيل.
- حقول MiniMax الخام `usage_percent` / `usagePercent` تمثل الحصة المتبقية، لذلك يعكسها OpenClaw قبل العرض؛ تكون الأولوية للحقول المستندة إلى العدّ عند وجودها. تفضّل استجابات `model_remains` إدخال نموذج المحادثة، وتستنتج تسمية النافذة من الطوابع الزمنية عند الحاجة، وتضمّن اسم النموذج في تسمية الخطة.
- عندما تكون لقطة الجلسة الحالية محدودة، يمكن لـ `/status` استكمال عدادات الرموز والتخزين المؤقت من أحدث سجل استخدام للنصّ المنسوخ. تظل القيم المباشرة غير الصفرية الموجودة مقدّمة على قيم الرجوع إلى النصّ المنسوخ.
- يتضمن `/status` مدة تشغيل عملية Gateway ومدة تشغيل نظام المضيف بإيجاز.
- يمكن للرجوع إلى النصّ المنسوخ أيضاً استرداد تسمية نموذج وقت التشغيل النشط عندما يفتقدها إدخال الجلسة المباشر. إذا اختلف نموذج النصّ المنسوخ هذا عن النموذج المحدد، تحل الحالة نافذة السياق مقابل نموذج وقت التشغيل المسترد بدلاً من النموذج المحدد.
- بالنسبة إلى حساب حجم الموجّه، يفضّل الرجوع إلى النصّ المنسوخ الإجمالي الأكبر الموجّه نحو الموجّه عندما تكون بيانات تعريف الجلسة مفقودة أو أصغر، حتى لا تنهار جلسات المزوّد المخصص إلى عروض رموز `0`.
- يتضمن الإخراج مخازن جلسات لكل وكيل عند تكوين عدة وكلاء.
- تتضمن النظرة العامة حالة تثبيت/تشغيل خدمة مضيف Gateway + node عند توفرها.
- تتضمن النظرة العامة قناة التحديث + SHA الخاص بـ git (لنسخ المصدر).
- تظهر معلومات التحديث في النظرة العامة؛ إذا كان هناك تحديث متاح، تطبع الحالة تلميحاً لتشغيل `openclaw update` (راجع [التحديث](/ar/install/updating)).
- تحل أسطح الحالة للقراءة فقط (`status`، `status --json`، `status --all`) SecretRefs المدعومة لمسارات التكوين المستهدفة عند الإمكان.
- إذا تم تكوين SecretRef لقناة مدعومة لكنه غير متاح في مسار الأمر الحالي، تبقى الحالة للقراءة فقط وتعرض إخراجاً متدهوراً بدلاً من التعطل. يعرض الإخراج البشري تحذيرات مثل “الرمز المميز المكوّن غير متاح في مسار الأمر هذا”، ويتضمن إخراج JSON الحقل `secretDiagnostics`.
- عندما ينجح حل SecretRef المحلي للأمر، تفضّل الحالة اللقطة المحلولة وتمسح علامات القناة العابرة “السر غير متاح” من الإخراج النهائي.
- يتضمن `status --all` صف نظرة عامة على الأسرار وقسم تشخيص يلخص تشخيصات الأسرار (مع اقتطاعها لسهولة القراءة) من دون إيقاف توليد التقرير.
