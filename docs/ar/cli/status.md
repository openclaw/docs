تشخيص القنوات + الجلسات.

```bash
openclaw status
openclaw status --all
openclaw status --deep
openclaw status --usage
```

| العلامة                  | الوصف                                                                                                                        |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| `--all`                 | تشخيص كامل (للقراءة فقط وقابل للصق). يتضمن تدقيق الأمان، وتوافق المكونات الإضافية، وفحوصات متجهات الذاكرة.                   |
| `--deep`                | يُجري فحوصات مباشرة (WhatsApp Web + Telegram + Discord + Slack + Signal). كما يفعّل تدقيق الأمان.                            |
| `--usage`               | يطبع نوافذ استخدام موفّر الخدمة بعد توحيدها بالصيغة `X% left`.                                                               |
| `--json`                | مخرجات قابلة للقراءة آليًا.                                                                                                  |
| `--verbose` / `--debug` | يطبع أيضًا تحليل هدف Gateway الخام قبل التقرير.                                                                              |

يبقى الأمر العادي `openclaw status` ضمن المسار السريع المخصّص للقراءة فقط، ويضع علامة
`not checked` على الذاكرة بدلًا من الإشارة إلى عدم توفرها عندما يتخطى فحصها. وتُترك
فحوصات تدقيق الأمان المكثفة، وتوافق المكونات الإضافية، ومتجهات الذاكرة للأوامر
`openclaw status --all` و`openclaw status --deep` و`openclaw security audit`
و`openclaw memory status --deep`.

## تحليل الجلسة والنموذج

- تفصل مخرجات حالة الجلسة بين `Execution:` و`Runtime:`. يشير `Execution`
  إلى مسار بيئة العزل (`direct` أو `docker/*`)، بينما يوضح `Runtime`
  ما إذا كانت الجلسة تستخدم `OpenClaw Default` أو `OpenAI Codex` أو واجهة
  خلفية لـ CLI أو واجهة خلفية لـ ACP مثل `codex (acp/acpx)`. راجع
  [بيئات تشغيل الوكيل](/ar/concepts/agent-runtimes) لمعرفة الفرق بين موفّر الخدمة والنموذج وبيئة التشغيل.
- عندما تكون اللقطة الحالية للجلسة شحيحة البيانات، يمكن للأمر `/status` استكمال عدادات
  الرموز وذاكرة التخزين المؤقت من أحدث سجل لاستخدام النص المنسوخ. وتظل
  القيم المباشرة الحالية غير الصفرية مقدّمة على القيم الاحتياطية للنص المنسوخ.
- يمكن للاحتياط من النص المنسوخ أيضًا استعادة تسمية نموذج بيئة التشغيل النشط عندما
  لا يتضمنها إدخال الجلسة المباشر. وإذا اختلف نموذج النص المنسوخ عن
  النموذج المحدد، فتحلل الحالة نافذة السياق وفق نموذج بيئة التشغيل
  المستعاد بدلًا من النموذج المحدد.
- عند احتساب حجم الموجّه، يفضّل الاحتياط من النص المنسوخ الإجمالي الأكبر
  الموجّه نحو المطالبات عندما تكون بيانات الجلسة الوصفية مفقودة أو أصغر، حتى
  لا تتقلص جلسات موفّري الخدمة المخصصين إلى عرض `0` من الرموز.
- عندما تُثبّت جلسة على نموذج يختلف عن النموذج الأساسي المضبوط،
  تطبع الحالة القيمتين والسبب (`session override`) والتلميح
  `/model default`. ينطبق النموذج الأساسي المضبوط على الجلسات الجديدة أو
  غير المثبتة، بينما تحتفظ الجلسات المثبتة الحالية باختيار الجلسة
  إلى أن يُمسح.
- تتضمن المخرجات مخازن جلسات خاصة بكل وكيل عند ضبط عدة وكلاء.

## الاستخدام والحصة

- يطبع `--usage` نوافذ استخدام موفّر الخدمة بعد توحيدها بالصيغة `X% left`.
- تمثل حقول MiniMax الخام `usage_percent` و`usagePercent` الحصة المتبقية،
  لذا يعكسها OpenClaw قبل العرض؛ وتكون الأولوية للحقول القائمة على العدد عند
  توفرها. تفضّل استجابات `model_remains` إدخال نموذج المحادثة، وتستنتج
  تسمية النافذة من الطوابع الزمنية عند الحاجة، وتدرج اسم النموذج ضمن
  تسمية الخطة.
- تظهر حالات فشل تحديث تسعير النماذج كتحذيرات اختيارية بشأن التسعير.
  ولا تعني أن Gateway أو القنوات ليست سليمة.

## النظرة العامة وحالة التحديث

- تتضمن النظرة العامة حالة تثبيت وتشغيل خدمة مضيف Gateway + Node عند
  توفرها، بالإضافة إلى مدة تشغيل موجزة لعملية Gateway ومدة تشغيل نظام المضيف.
- تتضمن النظرة العامة قناة التحديث + بصمة git SHA (لنسخ الشيفرة المصدرية المستخرجة).
- تظهر معلومات التحديث في النظرة العامة؛ وإذا توفر تحديث، تطبع الحالة
  تلميحًا لتشغيل `openclaw update` (راجع [التحديث](/ar/install/updating)).

## الأسرار

- تحاول أسطح الحالة المخصّصة للقراءة فقط (`status` و`status --json` و`status --all`)
  تحليل مراجع الأسرار SecretRefs المدعومة لمسارات الضبط المستهدفة متى
  أمكن.
- إذا ضُبط مرجع سر SecretRef لقناة مدعومة لكنه غير متوفر ضمن
  مسار الأمر الحالي، تظل الحالة مخصّصة للقراءة فقط وتبلّغ عن مخرجات
  متدهورة بدلًا من التعطل. تعرض المخرجات المقروءة تحذيرات مثل "الرمز
  المضبوط غير متوفر ضمن مسار الأمر هذا"، وتتضمن مخرجات JSON الحقل
  `secretDiagnostics`.
- عند نجاح تحليل SecretRef المحلي للأمر، تفضّل الحالة اللقطة
  المحللة وتمسح علامات القناة المؤقتة "السر غير متوفر" من
  المخرجات النهائية.
- يتضمن `status --all` صفًا عامًا للأسرار وقسم تشخيص
  يلخّص تشخيصات الأسرار (مختصرة لتسهيل القراءة) من دون
  إيقاف إنشاء التقرير.

## الذاكرة

يُبلغ `status --json --all` عن تفاصيل الذاكرة من بيئة تشغيل المكوّن الإضافي
للذاكرة النشطة التي يحددها `plugins.slots.memory`. يمكن لمكونات الذاكرة الإضافية
المخصصة إبقاء `agents.defaults.memorySearch.enabled` المضمّن معطّلًا مع الاستمرار
في الإبلاغ عن ملفاتها ومقاطعها وحالة المتجهات والبحث النصي الكامل.
