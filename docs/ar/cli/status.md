# `openclaw status`

تشخيصات للقنوات + الجلسات.

```bash
openclaw status
openclaw status --all
openclaw status --deep
openclaw status --usage
```

ملاحظات:

- يشغّل `--deep` فحوصات حية (WhatsApp Web + Telegram + Discord + Slack + Signal).
- يبقى `openclaw status` العادي على مسار سريع للقراءة فقط ويضع علامة على الذاكرة كـ `not checked` بدلا من غير متاحة عندما يتجاوز فحص الذاكرة. تُترك عملية التدقيق الأمني الثقيلة، وتوافق Plugin، وفحوصات متجهات الذاكرة إلى `openclaw status --all` و`openclaw status --deep` و`openclaw security audit` و`openclaw memory status --deep`.
- يبلّغ `status --json --all` عن تفاصيل الذاكرة من وقت تشغيل Plugin الذاكرة النشط المحدد بواسطة `plugins.slots.memory`. يمكن لـ Plugins الذاكرة المخصصة إبقاء `agents.defaults.memorySearch.enabled` المضمّن معطلا مع الاستمرار في الإبلاغ عن ملفاتها ومقاطعها وحالة المتجه وFTS الخاصة بها.
- يطبع `--usage` نوافذ استخدام المزوّد المعيارية بصيغة `X% left`.
- يفصل إخراج حالة الجلسة بين `Execution:` و`Runtime:`. `Execution` هو مسار sandbox (`direct`، `docker/*`)، بينما يخبرك `Runtime` بما إذا كانت الجلسة تستخدم `OpenClaw Pi Default` أو `OpenAI Codex` أو واجهة خلفية CLI أو واجهة خلفية ACP مثل `codex (acp/acpx)`. راجع [أوقات تشغيل الوكلاء](/ar/concepts/agent-runtimes) لفهم التمييز بين المزوّد/النموذج/وقت التشغيل.
- حقلا MiniMax الخامان `usage_percent` / `usagePercent` يمثلان الحصة المتبقية، لذلك يعكسهما OpenClaw قبل العرض؛ وتفوز الحقول القائمة على العدّ عند وجودها. تفضّل استجابات `model_remains` إدخال نموذج المحادثة، وتستنتج تسمية النافذة من الطوابع الزمنية عند الحاجة، وتضمّن اسم النموذج في تسمية الخطة.
- عندما تكون لقطة الجلسة الحالية متفرقة، يمكن لـ `/status` استكمال عدادات الرموز وذاكرة التخزين المؤقت من أحدث سجل لاستخدام النص المنقول. تظل القيم الحية غير الصفرية الموجودة مقدّمة على قيم الاسترجاع من النص المنقول.
- يمكن لاسترجاع النص المنقول أيضا استعادة تسمية نموذج وقت التشغيل النشط عندما تكون مفقودة من إدخال الجلسة الحية. إذا اختلف نموذج النص المنقول هذا عن النموذج المحدد، فإن status يحل نافذة السياق مقابل نموذج وقت التشغيل المستعاد بدلا من النموذج المحدد.
- لحساب حجم الموجه، يفضّل استرجاع النص المنقول الإجمالي الأكبر الموجّه للموجه عندما تكون بيانات تعريف الجلسة مفقودة أو أصغر، بحيث لا تنهار جلسات المزوّدين المخصصين إلى عرض `0` رموز.
- يتضمن الإخراج مخازن جلسات لكل وكيل عندما تكون عدة وكلاء مهيأة.
- يتضمن العرض العام حالة تثبيت/تشغيل خدمة Gateway + مضيف Node عند توفرها.
- يتضمن العرض العام قناة التحديث + SHA الخاص بـ git (لعمليات checkout من المصدر).
- تظهر معلومات التحديث في العرض العام؛ إذا كان هناك تحديث متاح، يطبع status تلميحا لتشغيل `openclaw update` (راجع [التحديث](/ar/install/updating)).
- تقوم أسطح الحالة للقراءة فقط (`status`، و`status --json`، و`status --all`) بحل SecretRefs المدعومة لمسارات الإعداد المستهدفة عندما يكون ذلك ممكنا.
- إذا تم تكوين SecretRef لقناة مدعومة لكنه غير متاح في مسار الأمر الحالي، تبقى status للقراءة فقط وتبلغ عن إخراج متدهور بدلا من التعطل. يعرض الإخراج البشري تحذيرات مثل “الرمز المميز المكوّن غير متاح في مسار الأمر هذا”، ويتضمن إخراج JSON الحقل `secretDiagnostics`.
- عندما ينجح حل SecretRef المحلي للأمر، تفضّل status اللقطة المحلولة وتمسح علامات القناة المؤقتة “السر غير متاح” من الإخراج النهائي.
- يتضمن `status --all` صفا عاما للأسرار وقسما للتشخيص يلخص تشخيصات الأسرار (مختصرة لتحسين قابلية القراءة) دون إيقاف إنشاء التقرير.
