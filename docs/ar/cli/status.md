# `openclaw status`

تشخيصات القنوات + الجلسات.

```bash
openclaw status
openclaw status --all
openclaw status --deep
openclaw status --usage
```

ملاحظات:

- يقوم `--deep` بتشغيل عمليات فحص حية (WhatsApp Web + Telegram + Discord + Slack + Signal).
- يطبع `--usage` نوافذ استخدام المزوّد الموحّدة بصيغة `X% left`.
- يفصل خرج حالة الجلسة الآن بين `Runtime:` و`Runner:`. يشير `Runtime` إلى مسار التنفيذ وحالة sandbox (`direct` و`docker/*`)، بينما يوضح `Runner` ما إذا كانت الجلسة تستخدم Pi المضمّن، أو مزوّدًا مدعومًا عبر CLI، أو واجهة خلفية لحزمة ACP مثل `codex (acp/acpx)`.
- تمثل الحقول الخام `usage_percent` / `usagePercent` في MiniMax الحصة المتبقية، لذلك يعكسها OpenClaw قبل العرض؛ وتُفضّل الحقول المعتمدة على العد عند وجودها. وتُفضّل استجابات `model_remains` إدخال نموذج الدردشة، وتشتق تسمية النافذة من الطوابع الزمنية عند الحاجة، وتضمّن اسم النموذج في تسمية الخطة.
- عندما تكون لقطة الجلسة الحالية متناثرة، يمكن لـ `/status` أن يملأ عدادات الرموز المميزة وذاكرة التخزين المؤقت من أحدث سجل استخدام في النص التفريغي. وتظل القيم الحية غير الصفرية الموجودة هي الأعلى أولوية مقارنةً بقيم النص التفريغي الاحتياطية.
- يمكن لآلية الاحتياط المستندة إلى النص التفريغي أيضًا استعادة تسمية نموذج Runtime النشط عندما يكون إدخال الجلسة الحي مفقودًا لها. وإذا اختلف نموذج النص التفريغي عن النموذج المحدد، يحلل status نافذة السياق اعتمادًا على نموذج Runtime المستعاد بدلًا من النموذج المحدد.
- بالنسبة إلى احتساب حجم المطالبة، تفضّل آلية الاحتياط المستندة إلى النص التفريغي الإجمالي الأكبر المرتبط بالمطالبة عندما تكون بيانات الجلسة الوصفية مفقودة أو أصغر، حتى لا تنهار جلسات المزوّد المخصص إلى عرض `0` للرموز المميزة.
- يتضمن الخرج مخازن جلسات لكل وكيل عند تكوين عدة وكلاء.
- تتضمن النظرة العامة حالة تثبيت/تشغيل خدمة Gateway + مضيف Node عند توفرها.
- تتضمن النظرة العامة قناة التحديث + Git SHA (لنسخ المصدر).
- تظهر معلومات التحديث في النظرة العامة؛ وإذا كان هناك تحديث متاح، يطبع status تلميحًا لتشغيل `openclaw update` (راجع [التحديث](/ar/install/updating)).
- تقوم أسطح status للقراءة فقط (`status` و`status --json` و`status --all`) بتحليل SecretRefs المدعومة لمسارات التكوين المستهدفة عندما يكون ذلك ممكنًا.
- إذا كان هناك SecretRef مكوّن لقناة مدعومة لكنه غير متاح في مسار الأمر الحالي، يظل status للقراءة فقط ويبلغ عن خرج متدهور بدلًا من التعطل. يعرض الخرج البشري تحذيرات مثل “configured token unavailable in this command path”، ويتضمن خرج JSON الحقل `secretDiagnostics`.
- عندما ينجح تحليل SecretRef المحلي للأمر، يفضّل status اللقطة المحللة ويمسح العلامات العابرة “secret unavailable” الخاصة بالقنوات من الخرج النهائي.
- يتضمن `status --all` صفًا لنظرة عامة على Secrets وقسمًا للتشخيص يلخص تشخيصات الأسرار (مقتطعًا لتحسين قابلية القراءة) من دون إيقاف إنشاء التقرير.
