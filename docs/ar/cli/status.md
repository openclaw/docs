تشخيصات للقنوات + الجلسات.

```bash
openclaw status
openclaw status --all
openclaw status --deep
openclaw status --usage
```

ملاحظات:

- يشغّل `--deep` فحوصات مباشرة (WhatsApp Web + Telegram + Discord + Slack + Signal).
- يبقى `openclaw status` العادي على مسار القراءة السريع فقط ويضع علامة `not checked` على الذاكرة بدلا من اعتبارها غير متاحة عندما يتخطى فحص الذاكرة. تُترك عمليات تدقيق الأمان الثقيلة، وتوافق plugin، وفحوصات متجهات الذاكرة إلى `openclaw status --all`، و`openclaw status --deep`، و`openclaw security audit`، و`openclaw memory status --deep`.
- يبلّغ `status --json --all` عن تفاصيل الذاكرة من وقت تشغيل plugin الذاكرة النشط المحدد بواسطة `plugins.slots.memory`. يمكن لplugins الذاكرة المخصصة ترك `agents.defaults.memorySearch.enabled` المضمّن معطلا ومع ذلك الإبلاغ عن ملفاتها ومقاطعها ومتجهها وحالة FTS الخاصة بها.
- يطبع `--usage` نوافذ استخدام المزوّد بعد التطبيع بصيغة `X% left`.
- يفصل خرج حالة الجلسة بين `Execution:` و`Runtime:`. يمثّل `Execution` مسار صندوق العزل (`direct`، `docker/*`)، بينما يوضح `Runtime` ما إذا كانت الجلسة تستخدم `OpenClaw Pi Default` أو `OpenAI Codex` أو واجهة خلفية CLI أو واجهة خلفية ACP مثل `codex (acp/acpx)`. راجع [أزمنة تشغيل الوكيل](/ar/concepts/agent-runtimes) لمعرفة الفرق بين المزوّد/النموذج/وقت التشغيل.
- حقلا MiniMax الخامان `usage_percent` / `usagePercent` يمثلان الحصة المتبقية، لذلك يعكسهما OpenClaw قبل العرض؛ وتتقدم الحقول المعتمدة على العد عند وجودها. تفضّل استجابات `model_remains` إدخال نموذج الدردشة، وتستنتج تسمية النافذة من الطوابع الزمنية عند الحاجة، وتضمّن اسم النموذج في تسمية الخطة.
- عندما تكون لقطة الجلسة الحالية قليلة البيانات، يمكن لـ`/status` استكمال عدادات الرموز والتخزين المؤقت من أحدث سجل استخدام للنص الكامل للجلسة. تظل القيم المباشرة غير الصفرية الموجودة متقدمة على قيم الرجوع إلى النص الكامل للجلسة.
- يتضمن `/status` مدة تشغيل مختصرة لعملية Gateway ومدة تشغيل نظام المضيف.
- يمكن للرجوع إلى النص الكامل للجلسة أيضا استرداد تسمية نموذج وقت التشغيل النشط عندما يكون إدخال الجلسة المباشر يفتقدها. إذا اختلف نموذج النص الكامل للجلسة هذا عن النموذج المحدد، تحل الحالة نافذة السياق اعتمادا على نموذج وقت التشغيل المسترد بدلا من النموذج المحدد.
- بالنسبة لحساب حجم الموجّه، يفضّل الرجوع إلى النص الكامل للجلسة الإجمالي الأكبر الموجّه للموجّهات عندما تكون بيانات تعريف الجلسة مفقودة أو أصغر، لذلك لا تنهار جلسات المزوّد المخصص إلى عروض رموز بقيمة `0`.
- يتضمن الخرج مخازن جلسات لكل وكيل عند تكوين وكلاء متعددين.
- تتضمن النظرة العامة حالة تثبيت/تشغيل Gateway + خدمة مضيف Node عند توفرها.
- تتضمن النظرة العامة قناة التحديث + SHA الخاص بـ git (لاستخراجات المصدر).
- تظهر معلومات التحديث في النظرة العامة؛ إذا كان هناك تحديث متاح، تطبع الحالة تلميحا لتشغيل `openclaw update` (راجع [التحديث](/ar/install/updating)).
- تظهر إخفاقات تحديث أسعار النماذج كتحذيرات أسعار اختيارية. وهي لا
  تعني أن Gateway أو القنوات غير سليمة.
- تحاول أسطح الحالة للقراءة فقط (`status`، و`status --json`، و`status --all`) حل SecretRefs المدعومة لمسارات التكوين المستهدفة عندما يكون ذلك ممكنا.
- إذا تم تكوين SecretRef لقناة مدعومة لكنه غير متاح في مسار الأمر الحالي، تبقى الحالة للقراءة فقط وتبلّغ عن خرج متدهور بدلا من التعطل. يعرض الخرج البشري تحذيرات مثل "الرمز المميز المكوّن غير متاح في مسار الأمر هذا"، ويتضمن خرج JSON الحقل `secretDiagnostics`.
- عند نجاح حل SecretRef المحلي للأمر، تفضّل الحالة اللقطة المحلولة وتمسح علامات القنوات العابرة "secret unavailable" من الخرج النهائي.
- يتضمن `status --all` صف نظرة عامة على الأسرار وقسم تشخيص يلخص تشخيصات الأسرار (مقتطعة لتحسين قابلية القراءة) من دون إيقاف توليد التقرير.
