# `openclaw dashboard`

افتح واجهة التحكم باستخدام مصادقتك الحالية.

```bash
openclaw dashboard
openclaw dashboard --no-open
openclaw dashboard --yes
```

- `--no-open`: اطبع عنوان URL من دون تشغيل متصفح.
- `--yes`: شغّل/ثبّت Gateway من دون مطالبة عند الحاجة.

ملاحظات:

- يحل مراجع الأسرار `gateway.auth.token` من نوع SecretRef عند الإمكان.
- يتبع إعداد `gateway.tls.enabled`: تطبع بوابات Gateway المفعّل فيها TLS عناوين URL لواجهة التحكم التي تبدأ بـ `https://` وتفتحها، وتتصل عبر `wss://`.
- عند استخدام ربط `lan` أو ربط `custom` بحرف بدل، تستخدم عمليات التشغيل على المضيف نفسه دائمًا local loopback لأن حرف البدل ليس وجهة صالحة للمتصفح. كما تستخدم ارتباطات `tailnet` و`custom` ذات النص الصريح العنوان `127.0.0.1` كي يتوفر للمتصفح سياق آمن؛ أما المضيفون المحددون الذين فُعّل لديهم TLS فيحتفظون بالعنوان المضبوط كي تتطابق أسماء الشهادات.
- قبل تسليم عنوان URL مصادق عليه عبر local loopback لربط بواجهة محددة، يفحص الأمر الواجهة المضبوطة ويتحقق من أنها والعنوان `127.0.0.1` مملوكان لعملية Gateway نفسها. يؤدي غموض ملكية المستمع إلى الإخفاق الآمن مع إرشادات حول الحالة.
- بالنسبة إلى الرموز المميزة المُدارة عبر SecretRef، سواء حُلّت أم لم تُحل، لا يتضمن عنوان URL المطبوع أو المنسوخ أو المفتوح الرمز المميز مطلقًا، كي لا تتسرب الأسرار الخارجية إلى مخرجات الطرفية أو سجل الحافظة أو وسيطات تشغيل المتصفح.
- إذا كان `gateway.auth.token` مُدارًا عبر SecretRef لكنه غير محلول، يطبع الأمر عنوان URL بلا رمز مميز وإرشادات للمعالجة بدلًا من عنصر نائب غير صالح للرمز المميز.
- إذا فشل التسليم عبر الحافظة أو المتصفح لعنوان URL مصادق عليه برمز مميز، يسجّل الأمر تلميحًا آمنًا للمصادقة اليدوية يذكر `OPENCLAW_GATEWAY_TOKEN` و`gateway.auth.token` ومفتاح جزء عنوان URL المسمى `token`، من دون طباعة قيمة الرمز المميز.
