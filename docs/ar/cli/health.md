# `openclaw health`

اجلب لقطة للحالة الصحية من Gateway قيد التشغيل عبر WebSocket RPC (من دون مقابس قنوات مباشرة من CLI).

## الخيارات

| العلامة           | القيمة الافتراضية | الوصف                                                                                       |
| ---------------- | ------------------ | ------------------------------------------------------------------------------------------- |
| `--json`         | `false`            | اطبع JSON قابلًا للقراءة آليًا بدلًا من النص.                                               |
| `--timeout <ms>` | `10000`            | مهلة الاتصال بالمللي ثانية.                                                                 |
| `--verbose`      | `false`            | يفرض فحصًا مباشرًا ويوسّع المخرجات لتشمل جميع الحسابات والوكلاء الذين جرى إعدادهم.           |
| `--debug`        | `false`            | اسم مستعار للخيار `--verbose`.                                                              |

أمثلة:

```bash
openclaw health
openclaw health --json
openclaw health --timeout 2500
openclaw health --verbose
openclaw health --debug
```

## السلوك

- من دون `--verbose`، يمكن لـ Gateway إرجاع لقطة مخزنة مؤقتًا (تظل حديثة لمدة تصل إلى 60 ثانية ولا تختلف عن حالة وقت تشغيل القناة المباشرة) وتحديثها في الخلفية للجهة التالية التي تطلبها.
- يفرض `--verbose` فحصًا مباشرًا (فحوصات الحسابات لكل قناة)، ويطبع تفاصيل اتصال Gateway، ويوسّع المخرجات القابلة للقراءة لتشمل جميع الحسابات والوكلاء الذين جرى إعدادهم بدلًا من الوكيل الافتراضي فقط.
- يعيد `--json` دائمًا اللقطة الكاملة: القنوات، وفحوصات كل حساب، وحالة تحميل Plugin، وحالة عزل محرك السياق، وحالة ذاكرة التخزين المؤقت لتسعير النماذج، وصحة حلقة الأحداث، ومخازن الجلسات لكل وكيل.
