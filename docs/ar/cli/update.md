# `openclaw update`

حدّث OpenClaw بأمان وبدّل بين قنوات stable/beta/dev.

إذا ثبّتّه عبر **npm/pnpm/bun** (تثبيت عام، بلا بيانات git وصفية)،
تتم التحديثات عبر مسار مدير الحزم في [التحديث](/ar/install/updating).

## الاستخدام

```bash
openclaw update
openclaw update status
openclaw update wizard
openclaw update --channel beta
openclaw update --channel dev
openclaw update --tag beta
openclaw update --tag main
openclaw update --dry-run
openclaw update --no-restart
openclaw update --yes
openclaw update --json
openclaw --update
```

## الخيارات

- `--no-restart`: تخطَّ إعادة تشغيل خدمة Gateway بعد تحديث ناجح. تحديثات مدير الحزم التي تعيد تشغيل Gateway تتحقق من أن الخدمة المعاد تشغيلها تبلّغ عن الإصدار المحدّث المتوقع قبل نجاح الأمر.
- `--channel <stable|beta|dev>`: عيّن قناة التحديث (git + npm؛ تُحفَظ في الإعدادات).
- `--tag <dist-tag|version|spec>`: تجاوز هدف الحزمة لهذا التحديث فقط. بالنسبة إلى تثبيتات الحزم، يشير `main` إلى `github:openclaw/openclaw#main`.
- `--dry-run`: اعرض إجراءات التحديث المخططة (القناة/الوسم/الهدف/مسار إعادة التشغيل) دون كتابة الإعدادات أو التثبيت أو مزامنة Plugins أو إعادة التشغيل.
- `--json`: اطبع JSON قابلًا للقراءة آليًا من نوع `UpdateRunResult`، بما في ذلك
  `postUpdate.plugins.integrityDrifts` عند اكتشاف انحراف في أثر Plugin الخاص بـ npm
  أثناء مزامنة Plugins بعد التحديث.
- `--timeout <seconds>`: مهلة لكل خطوة (القيمة الافتراضية 1800s).
- `--yes`: تخطَّ مطالبات التأكيد (مثل تأكيد الرجوع إلى إصدار أقدم).

<Warning>
يتطلب الرجوع إلى إصدار أقدم تأكيدًا لأن الإصدارات الأقدم قد تكسر الإعدادات.
</Warning>

## `update status`

اعرض قناة التحديث النشطة + وسم/فرع/SHA الخاص بـ git (لنسخ المصدر)، إضافةً إلى توفر التحديث.

```bash
openclaw update status
openclaw update status --json
openclaw update status --timeout 10
```

الخيارات:

- `--json`: اطبع JSON للحالة قابلًا للقراءة آليًا.
- `--timeout <seconds>`: مهلة عمليات الفحص (القيمة الافتراضية 3s).

## `update wizard`

مسار تفاعلي لاختيار قناة تحديث وتأكيد ما إذا كان ينبغي إعادة تشغيل Gateway
بعد التحديث (الافتراضي هو إعادة التشغيل). إذا اخترت `dev` دون نسخة git، فإنه
يعرض إنشاء واحدة.

الخيارات:

- `--timeout <seconds>`: مهلة كل خطوة تحديث (الافتراضي `1800`)

## ما الذي يفعله

عند تبديل القنوات صراحةً (`--channel ...`)، يحافظ OpenClaw أيضًا على توافق
طريقة التثبيت:

- `dev` → يضمن وجود نسخة git (الافتراضي: `~/openclaw`، ويمكن تجاوزه عبر `OPENCLAW_GIT_DIR`)،
  ويحدّثها، ويثبّت CLI العام من تلك النسخة.
- `stable` → يثبّت من npm باستخدام `latest`.
- `beta` → يفضّل وسم توزيع npm وهو `beta`، لكنه يرجع إلى `latest` عندما يكون beta
  مفقودًا أو أقدم من إصدار stable الحالي.

يشغّل المحدّث التلقائي لنواة Gateway (عند تفعيله عبر الإعدادات) مسار تحديث CLI
خارج معالج طلبات Gateway الحي. تحديثات مدير الحزم في مستوى التحكم `update.run`
تفرض إعادة تشغيل تحديث غير مؤجلة وبلا فترة تهدئة بعد تبديل الحزمة،
لأن عملية Gateway القديمة قد لا تزال تحتوي في الذاكرة على أجزاء تشير إلى
ملفات أزالتها الحزمة الجديدة.

بالنسبة إلى تثبيتات مدير الحزم، يحل `openclaw update` إصدار الحزمة الهدف
قبل استدعاء مدير الحزم. تستخدم تثبيتات npm العامة تثبيتًا مرحليًا:
يثبّت OpenClaw الحزمة الجديدة في بادئة npm مؤقتة، ويتحقق من مخزون `dist`
المعبأ هناك، ثم يبدّل شجرة الحزمة النظيفة تلك إلى البادئة العامة الحقيقية.
إذا فشل التحقق، فلا تُشغَّل أعمال doctor بعد التحديث، أو مزامنة Plugin، أو
إعادة التشغيل من الشجرة المشكوك فيها. حتى عندما يطابق الإصدار المثبّت
الهدف بالفعل، يحدّث الأمر تثبيت الحزمة العامة، ثم يشغّل مزامنة Plugin،
وتحديث إكمال الأوامر الأساسية، وأعمال إعادة التشغيل. هذا يبقي المكونات
الجانبية المعبأة وسجلات Plugin المملوكة للقناة متوافقة مع بناء OpenClaw
المثبّت، مع ترك إعادة بناء إكمال أوامر Plugin الكاملة لتشغيلات
`openclaw completion --write-state` الصريحة.

عند تثبيت خدمة Gateway محلية مُدارة وتفعيل إعادة التشغيل،
توقف تحديثات مدير الحزم الخدمة العاملة قبل استبدال شجرة الحزمة،
ثم تحدّث بيانات الخدمة الوصفية من التثبيت المحدّث، وتعيد تشغيل
الخدمة، وتتحقق من أن Gateway المعاد تشغيله يبلّغ عن الإصدار المتوقع. مع
`--no-restart`، لا يزال استبدال الحزمة يعمل، لكن الخدمة المُدارة لا
تُوقَف ولا تُعاد تشغيلها، لذلك قد يظل Gateway العامل يستخدم الكود القديم
حتى تعيد تشغيله يدويًا.

## مسار نسخة git

### اختيار القناة

- `stable`: اجلب أحدث وسم غير beta، ثم ابنِ وشغّل doctor.
- `beta`: فضّل أحدث وسم `-beta`، لكن ارجع إلى أحدث وسم stable عندما يكون beta مفقودًا أو أقدم.
- `dev`: اجلب `main`، ثم نفّذ fetch وrebase.

### خطوات التحديث

<Steps>
  <Step title="التحقق من نظافة شجرة العمل">
    يتطلب عدم وجود تغييرات غير ملتزم بها.
  </Step>
  <Step title="تبديل القناة">
    يبدّل إلى القناة المحددة (وسم أو فرع).
  </Step>
  <Step title="جلب المنبع">
    لـ Dev فقط.
  </Step>
  <Step title="بناء تمهيدي (لـ dev فقط)">
    يشغّل lint وبناء TypeScript في شجرة عمل مؤقتة. إذا فشل الطرف، يرجع للخلف حتى 10 التزامات للعثور على أحدث بناء نظيف.
  </Step>
  <Step title="Rebase">
    يعيد التأسيس فوق الالتزام المحدد (لـ dev فقط).
  </Step>
  <Step title="تثبيت الاعتماديات">
    يستخدم مدير الحزم الخاص بالمستودع. بالنسبة إلى نسخ pnpm، يمهّد المحدّث `pnpm` عند الطلب (عبر `corepack` أولًا، ثم بديل مؤقت `npm install pnpm@10`) بدلًا من تشغيل `npm run build` داخل مساحة عمل pnpm.
  </Step>
  <Step title="بناء Control UI">
    يبني Gateway وControl UI.
  </Step>
  <Step title="تشغيل doctor">
    يعمل `openclaw doctor` كفحص التحديث الآمن النهائي.
  </Step>
  <Step title="مزامنة Plugins">
    يزامن Plugins مع القناة النشطة. يستخدم Dev الـ Plugins المضمّنة؛ وتستخدم stable وbeta npm. يحدّث Plugins المثبّتة عبر npm.
  </Step>
</Steps>

<Warning>
إذا حُل تحديث npm Plugin مثبت بدقة إلى أثر تختلف سلامته عن سجل التثبيت المخزّن، فإن `openclaw update` يلغي تحديث أثر Plugin ذلك بدلًا من تثبيته. أعد تثبيت Plugin أو حدّثه صراحةً فقط بعد التحقق من أنك تثق بالأثر الجديد.
</Warning>

<Note>
تؤدي إخفاقات مزامنة Plugin بعد التحديث إلى فشل نتيجة التحديث وإيقاف أعمال متابعة إعادة التشغيل. أصلح خطأ تثبيت Plugin أو تحديثه، ثم أعد تشغيل `openclaw update`.

عند بدء Gateway المحدّث، يكون تحميل Plugin للتحقق فقط: لا يشغّل بدء التشغيل مديري حزم ولا يغيّر أشجار الاعتماديات. تتجاوز عمليات إعادة التشغيل الخاصة بـ `update.run` عبر مدير الحزم التأجيل الخامل العادي وفترة تهدئة إعادة التشغيل بعد تبديل شجرة الحزمة، بحيث لا تستطيع العملية القديمة الاستمرار في التحميل الكسول لأجزاء تمت إزالتها.

إذا ظل تمهيد pnpm يفشل، يتوقف المحدّث مبكرًا برسالة خطأ خاصة بمدير الحزم بدلًا من محاولة `npm run build` داخل النسخة.
</Note>

## الاختصار `--update`

يعيد `openclaw --update` الكتابة إلى `openclaw update` (مفيد للصدفات وسكربتات التشغيل).
