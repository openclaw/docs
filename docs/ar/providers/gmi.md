GMI Cloud هي منصة استدلال مستضافة للنماذج المتقدمة والنماذج مفتوحة الأوزان
خلف واجهة API متوافقة مع OpenAI. في OpenClaw، تُعد Plugin رسمية خارجية لمزوّد:
ثبّتها مرة واحدة، وخزّن بيانات الاعتماد عبر مصادقة النماذج المعتادة، واستخدم
مراجع نماذج مثل `gmi/google/gemini-3.1-flash-lite`.

استخدم GMI عندما تريد مفتاح API واحدًا لعدة عائلات من النماذج المستضافة، بما في ذلك
مسارات Anthropic وDeepSeek وGoogle وMoonshot وOpenAI وZ.AI التي يتيحها
كتالوج GMI. تعمل كمزوّد ثانوي للرجوع الاحتياطي بين النماذج، أو لمقارنة
المسارات المستضافة بين المورّدين، أو عندما يتوفر لدى GMI نموذج قبل أن يتوفر
لدى مزوّدك الأساسي. يدير OpenClaw معرّف المزوّد وملف المصادقة والأسماء المستعارة
وبذرة كتالوج النماذج وعنوان URL الأساسي؛ بينما تدير GMI التوفر الفعلي للنماذج والفوترة
وحدود المعدّل وأي سياسة توجيه من جانب المزوّد.

| الخاصية      | القيمة                                    |
| ------------- | ---------------------------------------- |
| معرّف المزوّد   | `gmi` (الأسماء المستعارة: `gmi-cloud`، `gmicloud`) |
| الحزمة       | `@openclaw/gmi-provider`                 |
| متغير بيئة المصادقة  | `GMI_API_KEY`                            |
| واجهة API           | متوافقة مع OpenAI (`openai-completions`) |
| عنوان URL الأساسي      | `https://api.gmi-serving.com/v1`         |
| النموذج الافتراضي | `gmi/google/gemini-3.1-flash-lite`       |

## الإعداد

ثبّت Plugin، وأعد تشغيل Gateway، ثم أنشئ مفتاح API في GMI Cloud
(`https://www.gmicloud.ai/`):

```bash
openclaw plugins install @openclaw/gmi-provider
openclaw gateway restart
```

ثم شغّل:

```bash
openclaw onboard --auth-choice gmi-api-key
```

يمكن لعمليات الإعداد غير التفاعلية تمرير `--gmi-api-key <key>`، أو تعيين:

```bash
export GMI_API_KEY="<your-gmi-api-key>" # pragma: allowlist secret
```

## متى تختار GMI

- تريد نقطة نهاية مستضافة متوافقة مع OpenAI بدلًا من خادم نماذج محلي.
- تريد تجربة عدة عائلات من النماذج التجارية ومفتوحة الأوزان عبر حساب
  مزوّد واحد.
- تريد مزوّد رجوع احتياطي بتوجيه مختلف إلى الجهات المنبع عن DeepInfra
  أو OpenRouter أو Together أو واجهات API المباشرة للمورّدين.
- تحتاج إلى معرّفات نماذج أو أسعار أو عناصر تحكم في الحساب خاصة بـ GMI.

اختر مزوّد المورّد المباشر بدلًا من ذلك عندما تحتاج إلى ميزات أصلية للمورّد
لا تتيحها GMI عبر مسارها المتوافق مع OpenAI. اختر مزوّدًا محليًا
مثل LM Studio أو Ollama أو SGLang أو vLLM عندما تكون محلية البيانات أو التحكم المحلي
في وحدة معالجة الرسومات أهم من سهولة الاستضافة.

## النماذج

تتضمن بذرة كتالوج Plugin معرّفات مسارات GMI Cloud المتاحة عادةً:

| مرجع النموذج                          | الإدخال        | السياق   | الحد الأقصى للإخراج |
| ---------------------------------- | ------------ | --------- | ---------- |
| `gmi/anthropic/claude-sonnet-4.6`  | نص + صورة | 200,000   | 64,000     |
| `gmi/deepseek-ai/DeepSeek-V3.2`    | نص         | 163,840   | 65,536     |
| `gmi/google/gemini-3.1-flash-lite` | نص + صورة | 1,048,576 | 65,536     |
| `gmi/moonshotai/Kimi-K2.5`         | نص + صورة | 262,144   | 65,536     |
| `gmi/openai/gpt-5.4`               | نص + صورة | 400,000   | 128,000    |
| `gmi/zai-org/GLM-5.1-FP8`          | نص         | 202,752   | 65,536     |

الكتالوج مجرد بذرة، وليس ضمانًا بأن كل حساب يمكنه استدعاء كل نموذج في
جميع الأوقات. اعرض ما يبلّغ عنه المزوّد المُهيأ في بيئتك:

```bash
openclaw models list --provider gmi
```

## استكشاف الأخطاء وإصلاحها

- `401` أو `403`: تحقق من تعيين `GMI_API_KEY` للعملية التي تشغّل
  OpenClaw، أو أعد تشغيل الإعداد الأولي لتخزين المفتاح في ملف مصادقة المزوّد.
- أخطاء النموذج غير المعروف: تأكد من وجود النموذج في حساب GMI الخاص بك، واستخدم
  مرجع `gmi/<route-id>` الكامل الذي يعرضه `openclaw models list --provider gmi`.
- أخطاء المزوّد المتقطعة: جرّب مسار GMI مختلفًا أو هيّئ GMI بوصفها
  مزوّد رجوع احتياطي بدلًا من كونها مزوّد النماذج الأساسي الوحيد.
