# الهدف

**الهدف** هو غاية واحدة دائمة مرتبطة بجلسة OpenClaw الحالية.
وهو يمنح الوكيل والمشغّل غاية مشتركة للعمل طويل الأمد،
من دون تحويلها إلى مهمة في الخلفية أو تذكير أو مهمة Cron أو
أمر دائم.

الأهداف جزء من حالة الجلسة: تنتقل مع مفتاح الجلسة، وتستمر بعد إعادة تشغيل
العملية، وتظهر في `/goal`، وأدوات الأهداف الموجّهة إلى النموذج، وتذييل TUI.

## البدء السريع

```text
/goal start اجعل CI ناجحًا لطلب PR 87469 وادفع الإصلاح
/goal
/goal edit اجعل CI ناجحًا لطلب PR 87469، وادفع الإصلاح، وحدّث التوثيق
/goal pause في انتظار CI
/goal resume
/goal complete تم الدفع والتحقق
/goal clear
```

استخدام `start` اختياري: ينشئ `/goal get CI green for PR 87469` هدفًا أيضًا،
لأن أي نص بعد `/goal` لا يكون كلمة إجراء معروفة يُعامل بوصفه
غاية جديدة.

## استخدامات الأهداف

استخدم هدفًا عندما تكون للجلسة نتيجة ملموسة ينبغي أن تظل ظاهرة
عبر العديد من التفاعلات:

- إنهاء طلب PR: الإصلاح، والتحقق، والمراجعة التلقائية، والدفع، وفتح طلب PR أو تحديثه.
- جولة تصحيح أخطاء: إعادة إنتاج الخطأ، وتحديد السطح المسؤول عنه، وإجراء التصحيح،
  وإثبات نجاح الإصلاح.
- جولة توثيق: قراءة التوثيق ذي الصلة، وكتابة الصفحة الجديدة، وإضافة الروابط المتبادلة، و
  التحقق من بناء التوثيق.
- مهمة صيانة: فحص الحالة الحالية، وإجراء تغييرات محدودة، وتشغيل
  عمليات التحقق المناسبة، والإبلاغ عمّا تغيّر.

الهدف ليس قائمة انتظار للمهام. استخدم [تدفق المهام](/ar/automation/taskflow)،
أو [المهام](/ar/automation/tasks)، أو [مهام Cron](/ar/automation/cron-jobs)، أو
[الأوامر الدائمة](/ar/automation/standing-orders) عندما ينبغي للعمل أن يعمل بشكل منفصل،
أو يتكرر وفق جدول زمني، أو يتفرع إلى أعمال فرعية مُدارة، أو يستمر كسياسة.

## مرجع الأوامر

يعرض `/goal` من دون وسائط ملخص الهدف الحالي:

```text
الهدف
الحالة: نشط
الغاية: اجعل CI ناجحًا لطلب PR 87469 وادفع الإصلاح
الرموز المستخدمة: 12k
ميزانية الرموز: 12k/50k

الأوامر: /goal edit <الغاية>، /goal pause، /goal complete، /goal clear
```

| الأمر                                               | التأثير                                                                  |
| --------------------------------------------------- | ------------------------------------------------------------------------ |
| `/goal` أو `/goal status`                           | عرض الهدف الحالي.                                                        |
| `/goal start <objective>`                           | إنشاء هدف جديد للجلسة الحالية.                                           |
| `/goal set <objective>`، `/goal create <objective>` | اسمان بديلان لـ `start`.                                                 |
| `/goal <objective>`                                 | ينشئ هدفًا جديدًا أيضًا (أي نص ليس كلمة إجراء معروفة).                    |
| `/goal edit <objective>`                            | إعادة صياغة الغاية الحالية؛ تظل الحالة واحتساب الرموز كما هما.            |
| `/goal pause [note]`                                | إيقاف هدف نشط مؤقتًا.                                                     |
| `/goal resume [note]`                               | استئناف هدف متوقف مؤقتًا أو محظور أو مقيّد بالاستخدام أو مقيّد بالميزانية. |
| `/goal complete [note]`                             | وضع علامة على الهدف بأنه تحقق.                                           |
| `/goal done [note]`                                 | اسم بديل لـ `complete`.                                                   |
| `/goal block [note]`                                | وضع علامة على الهدف بأنه محظور.                                          |
| `/goal blocked [note]`                              | اسم بديل لـ `block`.                                                      |
| `/goal clear`                                       | إزالة الهدف من الجلسة.                                                    |

لا يمكن أن يوجد سوى هدف واحد في الجلسة في الوقت نفسه. تفشل محاولة بدء هدف ثانٍ
بالرسالة `Goal error: goal already exists` حتى يُمسح الهدف الحالي.

لا يقبل `/goal start` خيارًا لميزانية الرموز؛ لا يمكن تعيين الميزانية إلا
من خلال أداة `create_goal` الموجّهة إلى النموذج.

## الحالات

- `active`: تسعى الجلسة إلى تحقيق الهدف.
- `paused`: أوقف المشغّل الهدف مؤقتًا؛ يعيده `/goal resume` إلى الحالة النشطة.
- `blocked`: أبلغ الوكيل أو المشغّل عن عائق فعلي؛ يعيده `/goal resume`
  إلى الحالة النشطة عند توفر معلومات أو حالة جديدة.
- `budget_limited`: بُلغت ميزانية الرموز المضبوطة؛ يعيد `/goal resume`
  بدء السعي من الغاية نفسها بنافذة ميزانية جديدة.
- `usage_limited`: محجوز لحالة إيقاف مستقبلية بسبب حد الاستخدام؛ يعيد `/goal
resume` بدء السعي بالطريقة نفسها.
- `complete`: تحقق الهدف. الأهداف المكتملة نهائية؛ استخدم `/goal
clear` قبل بدء هدف آخر.

يمسح `/new` و`/reset` هدف الجلسة الحالي، لأنهما يبدآن عمدًا
بسياق جلسة جديد.

## ميزانيات الرموز

يمكن أن تكون للأهداف ميزانية رموز موجبة اختيارية، تُعيَّن عبر
المعامل `token_budget` في أداة `create_goal`. تُقاس الميزانية بدءًا من
عدد الرموز الحديث للجلسة وقت إنشاء الهدف. إذا لم يكن لدى الجلسة عند بدء الهدف سوى
لقطة رموز قديمة أو مجهولة، ينتظر OpenClaw اللقطة الحديثة
التالية ويستخدمها خط أساس، وبذلك لا تُحتسب عليه الرموز المستهلكة قبل
وجود الهدف.

عندما يبلغ الاستخدام الميزانية، ينتقل الهدف إلى `budget_limited`. هذا لا
يحذف الهدف أو يمحو الغاية؛ بل يُعلم المشغّل و
الوكيل بأن السعي النشط إلى الهدف قد توقف حتى يُستأنف أو
يُمسح. يبدأ الاستئناف نافذة ميزانية جديدة عند عدد الرموز الحديث
الحالي.

ميزانيات الرموز حاجز حماية لأهداف الجلسة، وليست سقفًا للفوترة. تظل حصة
المزوّد، وتقارير التكلفة، وسلوك نافذة السياق خاضعة لعناصر التحكم المعتادة
في الاستخدام والنموذج لدى OpenClaw.

## أدوات النموذج

يوفّر OpenClaw ثلاث أدوات للأهداف لأطر تشغيل الوكلاء:

| الأداة         | الغرض                                                                                                                     |
| ------------- | ------------------------------------------------------------------------------------------------------------------------ |
| `get_goal`    | قراءة هدف الجلسة الحالي: الحالة، والغاية، واستخدام الرموز، وميزانية الرموز.                                                |
| `create_goal` | إنشاء هدف فقط عندما تطلب تعليمات المستخدم أو النظام ذلك صراحةً. تفشل إذا كان للجلسة هدف بالفعل.                           |
| `update_goal` | وضع علامة على الهدف بأنه `complete` أو `blocked`.                                                                         |

لا يستطيع النموذج إيقاف هدف مؤقتًا أو استئنافه أو مسحه أو استبداله بصمت. تظل هذه
عناصر تحكم للمشغّل والجلسة عبر `/goal` وأوامر إعادة الضبط، لكي يتمكن الوكيل
من الإبلاغ عن الإنجاز أو عن عائق حقيقي من دون تغيير
الغاية بهدوء.

ينبغي لـ `update_goal` وضع علامة `complete` على الهدف فقط عندما
تتحقق الغاية فعليًا. وينبغي له وضع علامة `blocked` على الهدف فقط بعد تكرار
حالة العرقلة نفسها في ثلاثة تفاعلات متتالية للهدف على الأقل، وليس بسبب
صعوبة عادية أو نقص في الصقل.

## سياق الهدف في كل تفاعل

يتضمن كل تفاعل للمستخدم أو الدردشة مع وجود هدف نشط سطر السياق التالي بدور المستخدم:

```text
الهدف النشط: <الغاية> — أحرز تقدمًا فيه أو حدّث حالته (get_goal/update_goal).
```

يبقي OpenClaw السطر موجزًا باقتطاع الغايات الطويلة. لا تُحقن الأهداف
المتوقفة مؤقتًا أو المحظورة أو المقيّدة بالميزانية أو المقيّدة بالاستخدام أو المكتملة،
لذلك يظل إيقاف المشغّل ساريًا حتى استئناف الهدف.

## واجهة التحكم

تعرض واجهة التحكم على الويب الهدف في شارة مدمجة أعلى محرر الدردشة:
أيقونة للحالة، وتسمية الحالة (مثل `متابعة الهدف`)، والغاية المقتطعة،
ومؤقت مباشر للوقت المنقضي.

تتضمن الشارة عناصر تحكم مضمّنة:

- **قلم الرصاص** يملأ المحرر مسبقًا بـ `/goal edit <objective>` حتى يمكن
  إعادة صياغة الغاية وإرسالها.
- **إيقاف مؤقت / استئناف** يبدّل بين `/goal pause` و`/goal resume` استنادًا
  إلى الحالة الحالية.
- **سلة المهملات** يرسل `/goal clear`.
- **سهم الرتبة** يوسّع الشارة لإظهار الغاية الكاملة، وأحدث ملاحظة للحالة،
  واستخدام الرموز، والوقت المنقضي.

تُخفى أزرار الإجراءات عندما يتعذر على المحرر الإرسال (على سبيل المثال
عند انقطاع اتصال Gateway)؛ ويظل سهم التوسيع يعمل.

## TUI

يبقي تذييل TUI هدف الجلسة النشطة ظاهرًا بجوار حقول الوكيل،
والجلسة، والنموذج، وقبل مؤشرات الرموز والوضع.

أمثلة للتذييل:

- `متابعة الهدف (12k/50k)` لهدف نشط بميزانية رموز.
- `الهدف متوقف مؤقتًا (/goal resume)` لهدف متوقف مؤقتًا.
- `الهدف محظور (/goal resume)` لهدف محظور.
- `بلغ الهدف حدود الاستخدام (/goal resume)` لهدف مقيّد بالاستخدام.
- `لم يتحقق الهدف (50k/50k)` لهدف مقيّد بالميزانية.
- `تحقق الهدف (42k)` لهدف مكتمل.

التذييل موجز عمدًا. استخدم `/goal` للاطلاع على الغاية الكاملة،
والملاحظة، وميزانية الرموز، والأوامر المتاحة.

## سلوك القنوات

يعمل `/goal` في جلسات OpenClaw التي تدعم الأوامر، بما في ذلك TUI و
واجهات الدردشة التي تسمح بالأوامر النصية. ترتبط حالة الهدف
بمفتاح الجلسة، لا بوسيلة النقل، لذا ترى واجهتان تشتركان في مفتاح جلسة
الهدف نفسه.

حالة الهدف ليست توجيهًا للتسليم: فهي لا تفرض الردود عبر
قناة، ولا تغيّر سلوك قائمة الانتظار، ولا توافق على الأدوات، ولا تجدول العمل.

## استكشاف الأخطاء وإصلاحها

| الرسالة                                 | المعنى                                                                                                                                       |
| --------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| `Goal error: goal already exists`       | للجلسة هدف بالفعل. استخدم `/goal` لفحصه، أو `/goal complete` إذا اكتمل، أو `/goal clear` قبل بدء غاية مختلفة.                                |
| `Goal error: goal not found`            | ليس للجلسة هدف بعد. ابدأ هدفًا باستخدام `/goal start <objective>`.                                                                            |
| `Goal error: goal is already complete`  | الهدف نهائي. امسحه قبل بدء غاية أخرى أو استئنافها.                                                                                            |

إذا أظهر استخدام الرموز `0` أو بدا قديمًا، فقد لا تكون لدى الجلسة النشطة
لقطة رموز حديثة بعد. يتجدد الاستخدام عندما يسجل OpenClaw استخدام الجلسة
والإجماليات المستمدة من سجل المحادثة.
