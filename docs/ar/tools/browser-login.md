## تسجيل الدخول اليدوي (موصى به)

عندما يتطلب أحد المواقع تسجيل الدخول، **سجّل الدخول يدويًا** في ملف تعريف متصفح **المضيف** (متصفح openclaw).

لا تعطِ النموذج بيانات اعتمادك. غالبًا ما تؤدي عمليات تسجيل الدخول الآلية إلى تفعيل وسائل الحماية من الروبوتات، وقد تؤدي إلى قفل الحساب.

العودة إلى مستندات المتصفح الرئيسية: [المتصفح](/ar/tools/browser).

## أي ملف تعريف Chrome يُستخدم؟

يتحكم OpenClaw في **ملف تعريف Chrome مخصص** (اسمه `openclaw`، وواجهة مستخدم بلون برتقالي خفيف). هذا منفصل عن ملف تعريف متصفحك اليومي.

بالنسبة إلى استدعاءات أداة متصفح الوكيل:

- الخيار الافتراضي: ينبغي أن يستخدم الوكيل متصفح `openclaw` المعزول الخاص به.
- استخدم `profile="user"` فقط عندما تكون جلسات تسجيل الدخول الحالية مهمة ويكون المستخدم أمام الكمبيوتر للنقر/الموافقة على أي مطالبة إرفاق.
- إذا كانت لديك عدة ملفات تعريف لمتصفح المستخدم، فحدد ملف التعريف صراحة بدلًا من التخمين.

طريقتان سهلتان للوصول إليه:

1. **اطلب من الوكيل فتح المتصفح** ثم سجّل الدخول بنفسك.
2. **افتحه عبر CLI**:

```bash
openclaw browser start
openclaw browser open https://x.com
```

إذا كانت لديك عدة ملفات تعريف، فمرّر `--browser-profile <name>` (الافتراضي هو `openclaw`).

## X/Twitter: التدفق الموصى به

- **القراءة/البحث/السلاسل:** استخدم متصفح **المضيف** (تسجيل دخول يدوي).
- **نشر التحديثات:** استخدم متصفح **المضيف** (تسجيل دخول يدوي).

## العزل في بيئة رملية + الوصول إلى متصفح المضيف

جلسات المتصفح المعزولة في بيئة رملية **أكثر احتمالًا** لتفعيل اكتشاف الروبوتات. بالنسبة إلى X/Twitter (والمواقع الصارمة الأخرى)، يُفضّل استخدام متصفح **المضيف**.

إذا كان الوكيل معزولًا في بيئة رملية، فستستخدم أداة المتصفح البيئة الرملية افتراضيًا. للسماح بالتحكم في المضيف:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

ثم استهدف متصفح المضيف:

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

أو عطّل العزل في بيئة رملية للوكيل الذي ينشر التحديثات.
