## تسجيل الدخول اليدوي (موصى به)

عندما يتطلب أحد المواقع تسجيل الدخول، سجّل الدخول يدويًا في ملف تعريف `openclaw`
لمتصفح المضيف. لا تعطِ النموذج بيانات اعتمادك: فغالبًا ما تؤدي عمليات تسجيل الدخول
المؤتمتة إلى تشغيل آليات الحماية من الروبوتات وقد تتسبب في قفل الحساب.

استخدم متصفح المضيف (بتسجيل دخول يدوي) لكل من القراءة (البحث/سلاسل النقاش)
والنشر على X/Twitter والمواقع الأخرى الحساسة للروبوتات. من المرجح أن تؤدي جلسات
المتصفح المعزولة إلى تشغيل آليات اكتشاف الروبوتات.

العودة إلى وثائق المتصفح الرئيسية: [المتصفح](/ar/tools/browser).

## أي ملف تعريف Chrome يُستخدم؟

يتحكم OpenClaw في ملف تعريف مخصص في Chrome باسم `openclaw` (بواجهة ذات مسحة
برتقالية)، وهو منفصل عن ملف تعريف متصفحك اليومي.

بالنسبة إلى استدعاءات أداة المتصفح الخاصة بالوكيل:

- الخيار الافتراضي: يستخدم الوكيل متصفح `openclaw` المعزول الخاص به.
- استخدم `profile="user"` فقط عندما تكون جلسات تسجيل الدخول الحالية مهمة وتكون
  موجودًا أمام الحاسوب للنقر على أي مطالبة بالإرفاق أو الموافقة عليها.
- إذا كانت لديك عدة ملفات تعريف لمتصفح المستخدم، فحدّد ملف التعريف صراحةً
  بدلًا من التخمين.

توجد طريقتان للوصول إلى ملف تعريف `openclaw`:

1. اطلب من الوكيل فتح المتصفح، ثم سجّل الدخول بنفسك.
2. افتحه عبر CLI:

```bash
openclaw browser start
openclaw browser open https://x.com
```

بالنسبة إلى ملف تعريف غير افتراضي، ضع `--browser-profile <name>` قبل
الأمر الفرعي (الافتراضي هو `openclaw`):

```bash
openclaw browser --browser-profile <name> open https://x.com
```

## العزل: السماح بالوصول إلى متصفح المضيف

إذا كان الوكيل معزولًا، فستستهدف استدعاءات أداة `browser` الخاصة به افتراضيًا
متصفح البيئة المعزولة، وليس متصفح المضيف. للسماح للوكيل باستهداف متصفح المضيف بدلًا من ذلك:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

تستهدف استدعاءات CLI دائمًا متصفح المضيف، ولا تستهدف البيئة المعزولة مطلقًا، لذا يمكنك
فتح متصفح المضيف بنفسك بصرف النظر عن هذا الإعداد:

```bash
openclaw browser --browser-profile openclaw open https://x.com
```

بعد ضبط `sandbox.browser.allowHostControl: true`، يمكن لاستدعاءات أداة `browser`
الخاصة بالوكيل استهداف المضيف أيضًا. وبدلًا من ذلك، عطّل العزل للوكيل
الذي ينشر التحديثات.
