يمكن لـ OpenClaw منع الوكلاء من التعثر في أنماط متكررة من استدعاءات الأدوات.
تكون الحماية **معطلة افتراضيا**.

فعّلها فقط عند الحاجة، لأنها قد تمنع استدعاءات متكررة مشروعة عند استخدام إعدادات صارمة.

## سبب وجود هذا

- اكتشاف التسلسلات المتكررة التي لا تحرز تقدما.
- اكتشاف حلقات عدم وجود نتائج عالية التكرار (الأداة نفسها، المدخلات نفسها، الأخطاء المتكررة).
- اكتشاف أنماط محددة من الاستدعاءات المتكررة لأدوات polling المعروفة.

## كتلة الإعدادات

الإعدادات الافتراضية العامة:

```json5
{
  tools: {
    loopDetection: {
      enabled: false,
      historySize: 30,
      warningThreshold: 10,
      criticalThreshold: 20,
      globalCircuitBreakerThreshold: 30,
      detectors: {
        genericRepeat: true,
        knownPollNoProgress: true,
        pingPong: true,
      },
    },
  },
}
```

تجاوز لكل وكيل (اختياري):

```json5
{
  agents: {
    list: [
      {
        id: "safe-runner",
        tools: {
          loopDetection: {
            enabled: true,
            warningThreshold: 8,
            criticalThreshold: 16,
          },
        },
      },
    ],
  },
}
```

### سلوك الحقول

- `enabled`: المفتاح الرئيسي. تعني `false` عدم تنفيذ أي اكتشاف للحلقات.
- `historySize`: عدد استدعاءات الأدوات الحديثة المحفوظة للتحليل.
- `warningThreshold`: العتبة قبل تصنيف نمط ما على أنه تحذيري فقط.
- `criticalThreshold`: عتبة حظر أنماط الحلقات المتكررة.
- `globalCircuitBreakerThreshold`: عتبة قاطع عدم التقدم العامة.
- `detectors.genericRepeat`: يكتشف أنماط تكرار الأداة نفسها + المعلمات نفسها.
- `detectors.knownPollNoProgress`: يكتشف الأنماط المعروفة الشبيهة بـ polling من دون تغير في الحالة.
- `detectors.pingPong`: يكتشف أنماط ping-pong المتناوبة.

بالنسبة إلى `exec`، تقارن فحوصات عدم التقدم نتائج الأوامر المستقرة وتتجاهل بيانات تعريف وقت التشغيل المتقلبة مثل المدة، وPID، ومعرف الجلسة، ودليل العمل.
عند توفر معرف تشغيل، لا يُقيّم سجل استدعاءات الأدوات الحديثة إلا ضمن ذلك التشغيل، كي لا ترث دورات Heartbeat المجدولة وعمليات التشغيل الجديدة عدادات حلقات قديمة من عمليات تشغيل سابقة.

## الإعداد الموصى به

- للنماذج الأصغر، ابدأ بـ `enabled: true` مع إبقاء الإعدادات الافتراضية من دون تغيير. نادرا ما تحتاج النماذج الرائدة إلى اكتشاف الحلقات ويمكنها تركه معطلا.
- أبق العتبات مرتبة على النحو `warningThreshold < criticalThreshold < globalCircuitBreakerThreshold`.
- إذا حدثت نتائج موجبة كاذبة:
  - ارفع `warningThreshold` و/أو `criticalThreshold`
  - ارفع `globalCircuitBreakerThreshold` (اختياريا)
  - عطّل الكاشف الذي يسبب المشكلات فقط
  - قلل `historySize` لسياق تاريخي أقل صرامة

## السجلات والسلوك المتوقع

عند اكتشاف حلقة، يبلغ OpenClaw عن حدث حلقة ويحظر دورة الأداة التالية أو يخففها حسب الشدة.
يحمي هذا المستخدمين من الإنفاق المنفلت للرموز وحالات التوقف، مع الحفاظ على الوصول الطبيعي إلى الأدوات.

- فضّل التحذير والكبت المؤقت أولا.
- لا تصعّد إلا عند تراكم أدلة متكررة.

## ملاحظات

- يُدمج `tools.loopDetection` مع التجاوزات على مستوى الوكيل.
- إعدادات كل وكيل تتجاوز القيم العامة أو توسعها بالكامل.
- إذا لم توجد إعدادات، تبقى guardrails متوقفة.
