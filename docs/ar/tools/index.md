استخدم هذه الصفحة لاختيار سطح Capabilities المناسب. **Tools** هي إجراءات قابلة للاستدعاء، و**Skills** تعلّم الوكلاء كيفية العمل، و**plugins** تضيف قدرات وقت التشغيل مثل الأدوات، والمزوّدين، والقنوات، والخطافات، وSkills المعبّأة.

هذه صفحة نظرة عامة وتوجيه. للحصول على سياسة الأدوات الكاملة، والقيم الافتراضية، وعضوية المجموعات، وقيود المزوّدين، وحقول الإعداد، استخدم
[الأدوات والمزوّدون المخصصون](/ar/gateway/config-tools).

## اختر الأدوات، أو Skills، أو plugins

<Steps>
  <Step title="استخدم أداة عندما يحتاج الوكيل إلى التصرف">
    الأداة هي دالة ذات أنواع يمكن للوكيل استدعاؤها، مثل `exec`، أو `browser`،
    أو `web_search`، أو `message`، أو `image_generate`. استخدم الأدوات عندما
    يحتاج الوكيل إلى قراءة بيانات، أو تغيير ملفات، أو إرسال رسائل، أو استدعاء
    مزوّد، أو تشغيل نظام آخر. تُرسَل الأدوات المرئية إلى النموذج كتعريفات
    دوال مهيكلة.

    لا يرى النموذج إلا الأدوات التي تتجاوز الملف التعريفي النشط، وسياسة السماح/الرفض،
    وقيود المزوّد، وحالة صندوق الرمل، وأذونات القناة، وتوافر plugin.

  </Step>

  <Step title="استخدم Skill عندما يحتاج الوكيل إلى تعليمات">
    Skill هي حزمة تعليمات `SKILL.md` تُحمَّل في مطالبة الوكيل. استخدم
    Skill عندما تكون لدى الوكيل الأدوات التي يحتاج إليها بالفعل، لكنه يحتاج إلى
    سير عمل قابل للتكرار، أو معيار مراجعة، أو تسلسل أوامر، أو قيد تشغيلي.

    يمكن أن تكون Skills في مساحة عمل، أو دليل Skills مشترك، أو جذر Skills
    مُدار من OpenClaw، أو حزمة plugin.

    [Skills](/ar/tools/skills) | [إنشاء Skills](/ar/tools/creating-skills) | [إعدادات Skills](/ar/tools/skills-config)

  </Step>

  <Step title="استخدم Plugin عندما يحتاج OpenClaw إلى قدرة جديدة">
    يمكن لـ Plugin إضافة أدوات، وSkills، وقنوات، ومزوّدي نماذج، وكلام، وصوت
    فوري، وتوليد وسائط، وبحث ويب، وجلب ويب، وخطافات، وقدرات وقت تشغيل أخرى.
    استخدم Plugin عندما تحتوي القدرة على كود، أو بيانات اعتماد، أو خطافات دورة
    حياة، أو بيانات وصفية للبيان، أو تغليف قابل للتثبيت. يمكن تثبيت plugins
    الحالية من ClawHub، أو npm، أو git، أو الأدلة المحلية، أو الأرشيفات.

    [تثبيت plugins وتهيئتها](/ar/tools/plugin) | [بناء plugins](/ar/plugins/building-plugins) | [Plugin SDK](/ar/plugins/sdk-overview)

  </Step>
</Steps>

## فئات الأدوات المدمجة

يعرض الجدول أدوات تمثيلية حتى تتمكن من التعرّف إلى السطح. وهو ليس مرجع السياسة
الكامل. للمجموعات الدقيقة، والقيم الافتراضية، ودلالات السماح/الرفض، استخدم
[الأدوات والمزوّدون المخصصون](/ar/gateway/config-tools).

| الفئة                  | استخدمها عندما يحتاج الوكيل إلى...                                           | أدوات تمثيلية                                                         | اقرأ التالي                                                           |
| ---------------------- | ----------------------------------------------------------------------------- | -------------------------------------------------------------------- | ---------------------------------------------------------------------- |
| وقت التشغيل            | تشغيل أوامر، أو إدارة عمليات، أو استخدام تحليل Python مدعوم من مزوّد         | `exec`, `process`, `code_execution`                                  | [Exec](/ar/tools/exec), [تنفيذ الكود](/ar/tools/code-execution)              |
| الملفات                | قراءة ملفات مساحة العمل وتغييرها                                             | `read`, `write`, `edit`, `apply_patch`                               | [Apply patch](/ar/tools/apply-patch)                                      |
| الويب                  | البحث في الويب، أو البحث في منشورات X، أو جلب محتوى صفحة قابل للقراءة       | `web_search`, `x_search`, `web_fetch`                                | [أدوات الويب](/ar/tools/web), [جلب الويب](/ar/tools/web-fetch)               |
| المتصفح                | تشغيل جلسة متصفح                                                             | `browser`                                                            | [المتصفح](/ar/tools/browser)                                              |
| المراسلة والقنوات      | إرسال ردود أو إجراءات قناة                                                   | `message`                                                            | [إرسال الوكيل](/ar/tools/agent-send)                                      |
| الجلسات والوكلاء       | فحص الجلسات، أو تفويض العمل، أو توجيه تشغيل آخر، أو الإبلاغ عن الحالة        | `sessions_*`, `subagents`, `agents_list`, `session_status`           | [الوكلاء الفرعيون](/ar/tools/subagents), [أداة الجلسة](/ar/concepts/session-tool) |
| الأتمتة                | جدولة العمل أو الاستجابة لأحداث الخلفية                                      | `cron`, `heartbeat_respond`                                          | [الأتمتة](/ar/automation)                                                 |
| Gateway والعُقد        | فحص حالة Gateway أو الأجهزة المستهدفة المقترنة                                | `gateway`, `nodes`                                                   | [تكوين Gateway](/ar/gateway/configuration), [العُقد](/ar/nodes)              |
| الوسائط                | تحليل الوسائط أو توليدها أو نطقها                                            | `image`, `image_generate`, `music_generate`, `video_generate`, `tts` | [نظرة عامة على الوسائط](/ar/tools/media-overview)                        |
| كتالوجات PI الكبيرة    | البحث واستدعاء كثير من الأدوات المؤهلة دون إرسال كل مخطط إلى النموذج         | `tool_search_code`, `tool_search`, `tool_describe`                   | [Tool Search](/ar/tools/tool-search)                                      |

<Note>
Tool Search هو سطح تجريبي لوكيل PI. تستخدم تشغيلات مسخّر Codex
وضع الكود الأصلي في Codex، والبحث الأصلي عن الأدوات، والأدوات الديناميكية المؤجلة،
واستدعاءات الأدوات المتداخلة بدلاً من `tools.toolSearch`.
</Note>

## الأدوات التي توفرها plugins

يمكن لـ plugins تسجيل أدوات إضافية. يربط مؤلفو Plugin الأدوات عبر
`api.registerTool(...)` و`contracts.tools` في البيان؛ استخدم
[Plugin SDK](/ar/plugins/sdk-overview) و[بيان Plugin](/ar/plugins/manifest)
لتفاصيل العقد.

تشمل الأدوات الشائعة التي توفرها plugins:

- [الفروقات](/ar/tools/diffs) لعرض فروقات الملفات وMarkdown
- [مهمة LLM](/ar/tools/llm-task) لخطوات سير العمل الخاصة بـ JSON فقط
- [Lobster](/ar/tools/lobster) لسير عمل ذي أنواع مع موافقات قابلة للاستئناف
- [Tokenjuice](/ar/tools/tokenjuice) لضغط مخرجات أدوات `exec` و`bash` المليئة بالضجيج
- [Tool Search](/ar/tools/tool-search) لاكتشاف كتالوجات أدوات كبيرة واستدعائها
  دون وضع كل مخطط في المطالبة
- [Canvas](/ar/plugins/reference/canvas) للتحكم في Canvas الخاص بالعُقد وعرض A2UI

## تكوين الوصول والموافقات

تُفرَض سياسة الأدوات قبل استدعاء النموذج. إذا أزالت السياسة أداة، فلن يتلقى
النموذج مخطط تلك الأداة للدورة الحالية. يمكن أن يفقد التشغيل أدوات بسبب الإعداد
العام، أو إعداد كل وكيل، أو سياسة القناة، أو قيود المزوّد، أو قواعد صندوق الرمل،
أو الحجب الخاص بالمالك فقط، أو توافر plugin.

- توثّق [الأدوات والمزوّدون المخصصون](/ar/gateway/config-tools) ملفات تعريف الأدوات،
  وقوائم السماح/الرفض، والقيود الخاصة بالمزوّدين، واكتشاف الحلقات، وإعدادات
  الأدوات المدعومة من المزوّدين.
- توثّق [موافقات Exec](/ar/tools/exec-approvals) سياسة الموافقة على أوامر المضيف.
- يوثّق [Exec المرتفع](/ar/tools/elevated) التنفيذ المتحكم فيه خارج صندوق الرمل.
- يشرح [صندوق الرمل مقابل سياسة الأدوات مقابل الرفع](/ar/gateway/sandbox-vs-tool-policy-vs-elevated) أي طبقة تتحكم في الوصول إلى الملفات والعمليات.
- توثّق [قيود صندوق الرمل والأدوات لكل وكيل](/ar/tools/multi-agent-sandbox-tools)
  القيود الخاصة بالوكيل في التشغيلات المفوضة.

## توسيع القدرات

اختر مسار التوسيع حسب المهمة التي تحتاج إلى أن ينفذها OpenClaw:

- ثبّت Plugin حالياً أو أدِره باستخدام [Plugins](/ar/tools/plugin).
- ابنِ تكاملاً جديداً، أو مزوّداً، أو قناة، أو أداة، أو خطافاً باستخدام
  [بناء plugins](/ar/plugins/building-plugins).
- أضف تعليمات وكيل قابلة لإعادة الاستخدام أو اضبطها باستخدام [Skills](/ar/tools/skills) و
  [إنشاء Skills](/ar/tools/creating-skills).
- غلّف مواد سير عمل قابلة لإعادة الاستخدام باستخدام
  [ورشة Skills](/ar/plugins/skill-workshop) عندما ينتمي سير العمل إلى حزمة Skills
  موزعة عبر plugin.
- استخدم [Plugin SDK](/ar/plugins/sdk-overview) و[بيان Plugin](/ar/plugins/manifest) عندما تحتاج إلى عقود تنفيذ.

## استكشاف أخطاء الأدوات المفقودة وإصلاحها

إذا لم يستطع النموذج رؤية أداة أو استدعاءها، فابدأ بالسياسة الفعالة للدورة
الحالية:

1. تحقق من الملف التعريفي النشط، و`tools.allow`، و`tools.deny` في
   [الأدوات والمزوّدون المخصصون](/ar/gateway/config-tools).
2. تحقق من القيود الخاصة بالمزوّدين في
   [الأدوات والمزوّدون المخصصون](/ar/gateway/config-tools) وتأكّد من أن
   [مزوّد النموذج](/ar/concepts/model-providers) المحدد يدعم شكل الأداة.
3. تحقق من أذونات القناة، وحالة صندوق الرمل، والوصول المرتفع باستخدام
   [صندوق الرمل مقابل سياسة الأدوات مقابل الرفع](/ar/gateway/sandbox-vs-tool-policy-vs-elevated) و[Exec المرتفع](/ar/tools/elevated).
4. تحقق مما إذا كان plugin المالك مثبتاً ومفعلاً في
   [Plugins](/ar/tools/plugin).
5. بالنسبة للتشغيلات المفوضة، تحقق من القيود لكل وكيل في
   [قيود صندوق الرمل والأدوات لكل وكيل](/ar/tools/multi-agent-sandbox-tools).
6. بالنسبة إلى كتالوجات PI الكبيرة، تأكّد مما إذا كان التشغيل يستخدم تعريض الأدوات مباشرة أو
   [Tool Search](/ar/tools/tool-search).
