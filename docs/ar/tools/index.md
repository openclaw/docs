استخدم هذه الصفحة لاختيار واجهة الإمكانات المناسبة. **الأدوات** هي
إجراءات قابلة للاستدعاء، وتعلّم **Skills** الوكلاء كيفية العمل، بينما تضيف **Plugins**
إمكانات وقت التشغيل مثل الأدوات والموفّرين والقنوات والخطافات وحزم
Skills.

هذه صفحة نظرة عامة وتوجيه. للاطلاع على سياسة الأدوات والإعدادات الافتراضية
وعضوية المجموعات وقيود الموفّرين وحقول التكوين بصورة شاملة، استخدم
[الأدوات والموفّرين المخصصين](/ar/gateway/config-tools).

فقط عندما ينبغي أن يرى الوكيل أدوات أقل أو يحتاج إلى وصول صريح إلى المضيف.

## اختر الأدوات أو Skills أو Plugins

<Steps>
  <Step title="استخدم أداة عندما يحتاج الوكيل إلى تنفيذ إجراء">
    الأداة هي دالة محددة الأنواع يمكن للوكيل استدعاؤها، مثل `exec` أو `browser`
    أو `web_search` أو `message` أو `image_generate`. استخدم الأدوات عندما يحتاج الوكيل
    إلى قراءة البيانات أو تغيير الملفات أو إرسال الرسائل أو استدعاء موفّر أو
    تشغيل نظام آخر. تُرسل الأدوات المرئية إلى النموذج على هيئة
    تعريفات دوال منظّمة.

    لا يرى النموذج إلا الأدوات التي تجتاز ملف التعريف النشط وسياسة
    السماح/المنع وقيود الموفّر وحالة بيئة العزل وأذونات القناة
    وتوفّر Plugin.

  </Step>

  <Step title="استخدم Skill عندما يحتاج الوكيل إلى تعليمات">
    Skill هي حزمة تعليمات `SKILL.md` تُحمّل في مطالبة الوكيل. استخدم
    Skill عندما يمتلك الوكيل الأدوات التي يحتاج إليها بالفعل، لكنه يحتاج إلى
    سير عمل قابل للتكرار أو معايير مراجعة أو تسلسل أوامر أو قيد
    تشغيلي.

    يمكن أن توجد Skills في مساحة عمل أو دليل Skills مشترك أو جذر Skills
    مُدار من OpenClaw أو حزمة Plugin.

    [Skills](/ar/tools/skills) | [ورشة Skills](/ar/tools/skill-workshop) | [إنشاء Skills](/ar/tools/creating-skills) | [تكوين Skills](/ar/tools/skills-config)

  </Step>

  <Step title="استخدم Plugin عندما يحتاج OpenClaw إلى إمكانية جديدة">
    يمكن أن يضيف Plugin أدوات وSkills وقنوات وموفّري نماذج وتحويل النص إلى كلام
    وصوتًا آنيًا وإنشاء الوسائط والبحث في الويب وجلب محتوى الويب والخطافات وإمكانات
    أخرى لوقت التشغيل. استخدم Plugin عندما تتضمن الإمكانية شيفرة
    أو بيانات اعتماد أو خطافات دورة حياة أو بيانات وصفية لبيان أو حزمة
    قابلة للتثبيت. يمكن تثبيت Plugins الموجودة من ClawHub أو npm أو git
    أو الأدلة المحلية أو الأرشيفات.

    [تثبيت Plugins وتكوينها](/ar/tools/plugin) | [بناء Plugins](/ar/plugins/building-plugins) | [حزمة تطوير Plugin](/ar/plugins/sdk-overview)

  </Step>
</Steps>

## فئات الأدوات المضمّنة

يسرد الجدول أدوات تمثيلية لمساعدتك على التعرّف على الواجهة. وهو
ليس مرجع السياسة الكامل. لمعرفة المجموعات الدقيقة والإعدادات الافتراضية ودلالات
السماح/المنع، استخدم [الأدوات والموفّرين المخصصين](/ar/gateway/config-tools).

| الفئة                | تُستخدم عندما يحتاج الوكيل إلى...                                                | أدوات تمثيلية                                                                                 | اقرأ بعد ذلك                                                                                   |
| ----------------------- | ----------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------- |
| وقت التشغيل                 | تشغيل الأوامر أو إدارة العمليات أو استخدام تحليل Python المدعوم من موفّر        | `exec`, `process`, `code_execution`                                                                  | [التنفيذ](/ar/tools/exec)، [تنفيذ الشيفرة](/ar/tools/code-execution)                                |
| الملفات                   | قراءة ملفات مساحة العمل وتغييرها                                               | `read`, `write`, `edit`, `apply_patch`                                                               | [تطبيق رقعة](/ar/tools/apply-patch)                                                           |
| الويب                     | البحث في الويب أو البحث في منشورات X أو جلب محتوى صفحات قابل للقراءة                | `web_search`, `x_search`, `web_fetch`                                                                | [أدوات الويب](/ar/tools/web)، [جلب محتوى الويب](/ar/tools/web-fetch)                                      |
| المتصفح                 | تشغيل جلسة متصفح                                                     | `browser`                                                                                            | [المتصفح](/ar/tools/browser)                                                                   |
| المراسلة والقنوات  | إرسال ردود أو إجراءات إلى القنوات                                               | `message`                                                                                            | [إرسال الوكيل](/ar/tools/agent-send)                                                             |
| الجلسات والوكلاء     | فحص الجلسات أو تفويض العمل أو توجيه تشغيل آخر أو الإبلاغ عن الحالة          | `sessions_*`, `subagents`, `agents_list`, `session_status`, `get_goal`, `create_goal`, `update_goal` | [الهدف](/ar/tools/goal)، [الوكلاء الفرعيون](/ar/tools/subagents)، [أداة الجلسة](/ar/concepts/session-tool) |
| الأتمتة              | جدولة العمل أو الاستجابة لأحداث الخلفية                                 | `cron`, `heartbeat_respond`                                                                          | [الأتمتة](/ar/automation)                                                                   |
| Gateway والعُقد       | فحص حالة Gateway أو الأجهزة الهدف المقترنة                                | `gateway`, `nodes`                                                                                   | [تكوين Gateway](/ar/gateway/configuration)، [العُقد](/ar/nodes)                            |
| الوسائط                   | تحليل الوسائط أو إنشاؤها أو نطقها                                             | `image`, `image_generate`, `music_generate`, `video_generate`, `tts`                                 | [نظرة عامة على الوسائط](/ar/tools/media-overview)                                                     |
| فهارس OpenClaw الكبيرة | البحث عن عدد كبير من الأدوات المؤهلة واستدعاؤها دون إرسال كل مخطط إلى النموذج | `tool_search_code`, `tool_search`, `tool_describe`                                                   | [البحث عن الأدوات](/ar/tools/tool-search)                                                           |

<Note>
البحث عن الأدوات هو واجهة تجريبية لوكلاء OpenClaw. تستخدم عمليات بيئة تشغيل Codex
وضع الشيفرة الأصلي في Codex والبحث الأصلي عن الأدوات والأدوات الديناميكية المؤجلة
واستدعاءات الأدوات المتداخلة بدلًا من `tools.toolSearch`.
</Note>

## الأدوات التي توفّرها Plugins

يمكن لـ Plugins تسجيل أدوات إضافية. يربط مؤلفو Plugins الأدوات عبر
`api.registerTool(...)` و`contracts.tools` في البيان؛ استخدم
[حزمة تطوير Plugin](/ar/plugins/sdk-overview) و[بيان Plugin](/ar/plugins/manifest)
للحصول على تفاصيل العقود.

تشمل الأدوات الشائعة التي توفّرها Plugins ما يلي:

- [الفروقات](/ar/tools/diffs) لعرض فروقات الملفات وMarkdown
- [عرض عنصر واجهة](/tools/show-widget) لعرض SVG وHTML مستقلين مضمنين في محادثة الويب
- [مهمة LLM](/ar/tools/llm-task) لخطوات سير العمل التي تقتصر على JSON
- [Lobster](/ar/tools/lobster) لعمليات سير عمل محددة الأنواع ذات موافقات قابلة للاستئناف
- [Tokenjuice](/ar/tools/tokenjuice) لضغط مخرجات أداتي `exec` و`bash`
  كثيرة الضوضاء
- [البحث عن الأدوات](/ar/tools/tool-search) لاكتشاف فهارس الأدوات الكبيرة واستدعائها
  دون وضع كل مخطط في المطالبة
- [Canvas](/ar/plugins/reference/canvas) للتحكم في Canvas على العُقد وعرض A2UI

## تكوين الوصول والموافقات

تُفرض سياسة الأدوات قبل استدعاء النموذج. إذا أزالت السياسة أداة، فلن
يتلقى النموذج مخطط تلك الأداة في هذه الدورة. قد يفقد التشغيل أدوات
بسبب التكوين العام أو تكوين كل وكيل أو سياسة القناة أو قيود
الموفّر أو قواعد بيئة العزل أو سياسة القناة/وقت التشغيل أو توفّر Plugin.

- توثّق [الأدوات والموفّرون المخصصون](/ar/gateway/config-tools) ملفات تعريف الأدوات
  وقوائم السماح/المنع والقيود الخاصة بالموفّرين واكتشاف الحلقات
  وإعدادات الأدوات المدعومة من الموفّرين.
- توثّق [موافقات التنفيذ](/ar/tools/exec-approvals) سياسة الموافقة على أوامر
  المضيف.
- يوثّق [التنفيذ بصلاحيات مرتفعة](/ar/tools/elevated) التنفيذ المتحكم فيه خارج
  بيئة العزل.
- تشرح [بيئة العزل مقابل سياسة الأدوات مقابل الصلاحيات المرتفعة](/ar/gateway/sandbox-vs-tool-policy-vs-elevated)
  الطبقة التي تتحكم في الوصول إلى الملفات والعمليات.
- توثّق [قيود بيئة العزل والأدوات لكل وكيل](/ar/tools/multi-agent-sandbox-tools)
  القيود الخاصة بالوكلاء لعمليات التشغيل المفوّضة.

## توسيع الإمكانات

اختر مسار التوسعة وفقًا للمهمة التي تريد من OpenClaw تنفيذها:

- ثبّت Plugin موجودًا أو أدره باستخدام [Plugins](/ar/tools/plugin).
- أنشئ تكاملًا أو موفّرًا أو قناة أو أداة أو خطافًا جديدًا باستخدام
  [بناء Plugins](/ar/plugins/building-plugins).
- أضف تعليمات وكيل قابلة لإعادة الاستخدام أو اضبطها باستخدام [Skills](/ar/tools/skills) و
  [إنشاء Skills](/ar/tools/creating-skills).
- استخدم [حزمة تطوير Plugin](/ar/plugins/sdk-overview) و
  [بيان Plugin](/ar/plugins/manifest) عندما تحتاج إلى عقود
  التنفيذ.

## استكشاف أخطاء الأدوات المفقودة وإصلاحها

إذا تعذّر على النموذج رؤية أداة أو استدعاؤها، فابدأ بالسياسة الفعلية
للدورة الحالية:

1. تحقّق من ملف التعريف النشط و`tools.allow` و`tools.deny` في
   [الأدوات والموفّرين المخصصين](/ar/gateway/config-tools).
2. تحقّق من القيود الخاصة بالموفّر في
   [الأدوات والموفّرين المخصصين](/ar/gateway/config-tools)، وتأكّد من أن
   [موفّر النموذج](/ar/concepts/model-providers) المحدد يدعم بنية
   الأداة.
3. تحقّق من أذونات القناة وحالة بيئة العزل والوصول بصلاحيات مرتفعة باستخدام
   [بيئة العزل مقابل سياسة الأدوات مقابل الصلاحيات المرتفعة](/ar/gateway/sandbox-vs-tool-policy-vs-elevated)
   و[التنفيذ بصلاحيات مرتفعة](/ar/tools/elevated).
4. تحقّق مما إذا كان Plugin المالك مثبّتًا ومفعّلًا في
   [Plugins](/ar/tools/plugin).
5. بالنسبة إلى عمليات التشغيل المفوّضة، تحقّق من القيود الخاصة بكل وكيل في
   [قيود بيئة العزل والأدوات لكل وكيل](/ar/tools/multi-agent-sandbox-tools).
6. بالنسبة إلى فهارس OpenClaw الكبيرة، تأكّد مما إذا كان التشغيل يستخدم عرض الأدوات
   مباشرةً أو [البحث عن الأدوات](/ar/tools/tool-search).
