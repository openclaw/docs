يُعد التقاط المسار مسجل الرحلة لكل جلسة في OpenClaw. فهو يسجل
خطًا زمنيًا منظمًا لكل تشغيل وكيل، ثم يقوم `/export-trajectory` بحزم
الجلسة الحالية في حزمة دعم منقحة.

استخدمه عندما تحتاج إلى الإجابة عن أسئلة مثل:

- ما المطالبة، ومطالبة النظام، والأدوات التي أُرسلت إلى النموذج؟
- ما رسائل النص واستدعاءات الأدوات التي أدت إلى هذه الإجابة؟
- هل انتهى التشغيل بالمهلة، أو أُلغي، أو ضُغط، أو واجه خطأ من الموفّر؟
- ما النموذج، وPlugins، وSkills، وإعدادات وقت التشغيل التي كانت نشطة؟
- ما بيانات الاستخدام وبيانات prompt-cache الوصفية التي أعادها الموفّر؟

## البدء السريع

أرسل هذا في الجلسة النشطة:

```text
/export-trajectory
```

الاسم البديل:

```text
/trajectory
```

يكتب OpenClaw الحزمة ضمن مساحة العمل:

```text
.openclaw/trajectory-exports/openclaw-trajectory-<session>-<timestamp>/
```

يمكنك اختيار اسم دليل إخراج نسبي:

```text
/export-trajectory bug-1234
```

يُحل المسار المخصص داخل `.openclaw/trajectory-exports/`. وتُرفض
المسارات المطلقة ومسارات `~`.

## الوصول

يمثل تصدير المسار أمر مالك. ويجب أن يجتاز المُرسل فحوصات تفويض الأوامر
العادية وفحوصات المالك الخاصة بالقناة.

## ما الذي يُسجَّل

يكون التقاط المسار مفعّلًا افتراضيًا لعمليات تشغيل وكلاء OpenClaw.

تتضمن أحداث وقت التشغيل:

- `session.started`
- `trace.metadata`
- `context.compiled`
- `prompt.submitted`
- `model.completed`
- `trace.artifacts`
- `session.ended`

كما يُعاد أيضًا بناء أحداث النص من الفرع النشط للجلسة:

- رسائل المستخدم
- رسائل المساعد
- استدعاءات الأدوات
- نتائج الأدوات
- عمليات Compaction
- تغييرات النموذج
- التسميات وإدخالات الجلسة المخصصة

تُكتب الأحداث بصيغة JSON Lines مع علامة المخطط هذه:

```json
{
  "traceSchema": "openclaw-trajectory",
  "schemaVersion": 1
}
```

## ملفات الحزمة

يمكن أن تحتوي الحزمة المصدّرة على:

| الملف                | المحتويات                                                                                  |
| -------------------- | ------------------------------------------------------------------------------------------ |
| `manifest.json`      | مخطط الحزمة، وملفات المصدر، وعدّادات الأحداث، وقائمة الملفات المولدة                       |
| `events.jsonl`       | الخط الزمني المرتب لوقت التشغيل والنص                                                       |
| `session-branch.json` | الفرع النشط المنقح من النص ورأس الجلسة                                                    |
| `metadata.json`      | إصدار OpenClaw، ونظام التشغيل/وقت التشغيل، والنموذج، ولقطة التهيئة، وPlugins، وSkills، وبيانات المطالبة الوصفية |
| `artifacts.json`     | الحالة النهائية، والأخطاء، والاستخدام، وprompt cache، وعدد مرات Compaction، ونص المساعد، وبيانات الأدوات الوصفية |
| `prompts.json`       | المطالبات المرسلة وتفاصيل مختارة لبناء المطالبات                                           |
| `system-prompt.txt`  | أحدث مطالبة نظام مجمعة، عند التقاطها                                                      |
| `tools.json`         | تعريفات الأدوات المرسلة إلى النموذج، عند التقاطها                                          |

يسرد `manifest.json` الملفات الموجودة في تلك الحزمة. وتُحذف بعض الملفات
عندما لا تكون الجلسة قد التقطت بيانات وقت التشغيل المقابلة.

## موقع الالتقاط

افتراضيًا، تُكتب أحداث مسار وقت التشغيل بجوار ملف الجلسة:

```text
<session>.trajectory.jsonl
```

ويكتب OpenClaw أيضًا ملف مؤشر بأفضل جهد بجوار الجلسة:

```text
<session>.trajectory-path.json
```

اضبط `OPENCLAW_TRAJECTORY_DIR` لتخزين ملفات sidecar الخاصة بمسار وقت التشغيل في
دليل مخصص:

```bash
export OPENCLAW_TRAJECTORY_DIR=/var/lib/openclaw/trajectories
```

عند ضبط هذا المتغير، يكتب OpenClaw ملف JSONL واحدًا لكل معرّف جلسة في ذلك
الدليل.

## تعطيل الالتقاط

اضبط `OPENCLAW_TRAJECTORY=0` قبل بدء OpenClaw:

```bash
export OPENCLAW_TRAJECTORY=0
```

يؤدي هذا إلى تعطيل التقاط مسار وقت التشغيل. ولا يزال بإمكان `/export-trajectory`
تصدير فرع النص، لكن قد تكون الملفات الخاصة بوقت التشغيل فقط مثل
السياق المجمّع، وآثار الموفّر، وبيانات المطالبة الوصفية مفقودة.

## الخصوصية والحدود

صُممت حزم المسار لأغراض الدعم وتصحيح الأخطاء، وليس للنشر العام.
يقوم OpenClaw بتنقيح القيم الحساسة قبل كتابة ملفات التصدير:

- بيانات الاعتماد والحقول الشبيهة بالأسرار المعروفة في الحمولة
- بيانات الصور
- مسارات الحالة المحلية
- مسارات مساحة العمل، مع استبدالها بـ `$WORKSPACE_DIR`
- مسارات الدليل الرئيسي، عند اكتشافها

ويضع المصدّر أيضًا حدودًا على حجم الإدخال:

- ملفات sidecar لوقت التشغيل: 50 MiB
- ملفات الجلسات: 50 MiB
- أحداث وقت التشغيل: 200,000
- إجمالي الأحداث المصدّرة: 250,000
- تُقتطع أسطر أحداث وقت التشغيل الفردية فوق 256 KiB

راجع الحزم قبل مشاركتها خارج فريقك. فالتنقيح يتم بأفضل جهد
ولا يمكنه معرفة كل الأسرار الخاصة بكل تطبيق.

## استكشاف الأخطاء وإصلاحها

إذا لم يحتوي التصدير على أي أحداث لوقت التشغيل:

- تأكد من أن OpenClaw بدأ من دون `OPENCLAW_TRAJECTORY=0`
- تحقق مما إذا كان `OPENCLAW_TRAJECTORY_DIR` يشير إلى دليل قابل للكتابة
- شغّل رسالة أخرى في الجلسة، ثم أعد التصدير
- افحص `manifest.json` بحثًا عن `runtimeEventCount`

إذا رفض الأمر مسار الإخراج:

- استخدم اسمًا نسبيًا مثل `bug-1234`
- لا تمرر `/tmp/...` أو `~/...`
- أبقِ التصدير داخل `.openclaw/trajectory-exports/`

إذا فشل التصدير بخطأ حجم، فقد تجاوزت الجلسة أو ملف sidecar
حدود الأمان الخاصة بالتصدير. ابدأ جلسة جديدة أو صدّر إعادة إنتاج أصغر.
