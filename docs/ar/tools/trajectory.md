التقاط المسار هو مسجل الرحلة لكل جلسة في OpenClaw. ويسجل
خطًا زمنيًا منظمًا لكل تشغيل للوكيل، ثم تحزم `/export-trajectory`
الجلسة الحالية في حزمة دعم منقحة تشمل:

- الموجّه وموجّه النظام والأدوات المرسلة إلى النموذج
- رسائل النص المنقول واستدعاءات الأدوات التي أدت إلى إجابة
- ما إذا انتهت مهلة التشغيل أو أُلغي أو خضع لـ Compaction أو واجه خطأً من المزوّد
- النموذج وplugins وskills وإعدادات وقت التشغيل التي كانت نشطة
- بيانات الاستخدام وبيانات التعريف لذاكرة التخزين المؤقت للموجّه التي أعادها المزوّد

للحصول على تقرير دعم شامل لـ Gateway، ابدأ باستخدام
[`/diagnostics`](/ar/gateway/diagnostics#chat-command) بدلًا من ذلك؛ فهو يجمع
حزمة Gateway المنقحة، ويمكنه، في جلسات بيئة OpenAI Codex، إرسال ملاحظات Codex
إلى OpenAI بعد الموافقة. استخدم `/export-trajectory` عندما تحتاج إلى
الخط الزمني التفصيلي للموجّه والأدوات والنص المنقول لكل جلسة.

## البدء السريع

أرسل في الجلسة النشطة (الاسم المستعار `/trajectory`):

```text
/export-trajectory
```

يكتب OpenClaw الحزمة ضمن مساحة العمل:

```text
.openclaw/trajectory-exports/openclaw-trajectory-<session>-<timestamp>/
```

مرّر اسم دليل إخراج نسبيًا لتجاوزه:

```text
/export-trajectory bug-1234
```

يُحدَّد الاسم داخل `.openclaw/trajectory-exports/`. تُرفض المسارات المطلقة ومسارات
`~`.

قد تحتوي حزم المسار على موجّهات ورسائل النموذج ومخططات الأدوات ونتائج
الأدوات وأحداث وقت التشغيل والمسارات المحلية، لذلك يُنفَّذ أمر الدردشة دائمًا
عبر موافقة exec. وافق على التصدير مرة واحدة عندما تنوي إنشاء
الحزمة؛ ولا تستخدم السماح للكل. في الدردشات الجماعية، يرسل OpenClaw مطالبة
الموافقة ونتيجة التصدير إلى المالك على نحو خاص بدلًا من نشر تفاصيل المسار
في الغرفة المشتركة.

للفحص المحلي أو سير عمل الدعم، شغّل أمر CLI الأساسي
مباشرةً:

```bash
openclaw sessions export-trajectory --session-key "agent:main:telegram:direct:123" --workspace .
```

العلامات الأخرى: `--output <path>` (اسم الدليل داخل
`.openclaw/trajectory-exports`)، و`--store <path>` (تجاوز مخزن الجلسة)،
و`--agent <id>` (معرّف الوكيل لتحديد المخزن)، و`--json` (إخراج منظم).

## الوصول

تصدير المسار هو أمر خاص بالمالك. يجب أن يجتاز المرسل فحوصات تخويل الأمر
العادية، بالإضافة إلى فحص المالك للقناة.

## ما يُسجَّل

يكون التقاط المسار مفعّلًا افتراضيًا لتشغيلات وكلاء OpenClaw.

تشمل أحداث وقت التشغيل:

- `session.started`
- `trace.metadata`
- `context.compiled`
- `prompt.submitted`
- `model.fallback_step`، بما في ذلك النموذج المصدر والنموذج التالي وسبب الفشل/تفاصيله وموضع السلسلة، وما إذا كانت السلسلة قد تقدمت أو نجحت أو استُنفدت
- `model.completed`
- `trace.artifacts`
- `session.ended`

تُعاد صياغة أحداث النص المنقول من الفرع النشط للجلسة: رسائل المستخدم
ورسائل المساعد واستدعاءات الأدوات ونتائج الأدوات وعمليات Compaction وتغييرات
النموذج والتسميات وإدخالات الجلسة المخصصة.

تُكتب الأحداث بصيغة JSON Lines مع علامة المخطط التالية:

```json
{
  "traceSchema": "openclaw-trajectory",
  "schemaVersion": 1
}
```

## ملفات الحزمة

| الملف                  | المحتويات                                                                                       |
| --------------------- | ---------------------------------------------------------------------------------------------- |
| `manifest.json`       | مخطط الحزمة والملفات المصدر وأعداد الأحداث وقائمة الملفات المنشأة                             |
| `events.jsonl`        | الخط الزمني المرتب لوقت التشغيل والنص المنقول                                                        |
| `session-branch.json` | فرع النص المنقول النشط المنقح ورأس الجلسة                                           |
| `metadata.json`       | إصدار OpenClaw ونظام التشغيل/وقت التشغيل والنموذج ولقطة الإعدادات وplugins وskills وبيانات تعريف الموجّه     |
| `artifacts.json`      | الحالة النهائية والأخطاء والاستخدام وذاكرة الموجّه المؤقتة وعدد عمليات Compaction ونص المساعد وبيانات تعريف الأدوات |
| `prompts.json`        | الموجّهات المقدمة وتفاصيل مختارة لبناء الموجّه                                         |
| `system-prompt.txt`   | أحدث موجّه نظام مُجمَّع، عند التقاطه                                                   |
| `tools.json`          | تعريفات الأدوات المرسلة إلى النموذج، عند التقاطها                                              |

يسرد `manifest.json` الملفات الموجودة في حزمة معينة؛ وتُحذف بعض الملفات
عندما لا تلتقط الجلسة بيانات وقت التشغيل المقابلة.

## تخزين الالتقاط

تُخزّن أحداث مسار وقت التشغيل مع الجلسة في قاعدة بيانات SQLite
الخاصة بكل وكيل. يؤدي تصدير المسار إلى إنشاء حزمة دعم JSONL منقحة؛
ولا يكون التقاط وقت التشغيل المباشر ملف JSONL جانبيًا مجاورًا للجلسة.

قد تظل ملفات `.trajectory.jsonl` و`.trajectory-path.json` تظهر
من الإصدارات الأقدم أو عمليات تصدير الملفات القديمة الصريحة. تتعامل صيانة الجلسة مع
هذه الملفات بوصفها أهدافًا للتنظيف؛ بينما يكتب الالتقاط النشط صفوفًا في قاعدة البيانات.

## تعطيل الالتقاط

```bash
export OPENCLAW_TRAJECTORY=0
```

يعطّل هذا التقاط مسار وقت التشغيل قبل بدء OpenClaw.
لا يزال بإمكان `/export-trajectory` تصدير فرع النص المنقول، لكن قد تكون
بيانات وقت التشغيل فقط، مثل السياق المُجمَّع وعناصر المزوّد وبيانات تعريف الموجّه،
مفقودة.

## ضبط مهلة التفريغ

يفرّغ OpenClaw صفوف مسار وقت التشغيل أثناء تنظيف الوكيل. مهلة
التنظيف الافتراضية هي 10,000 ms. على الأقراص البطيئة أو المخازن الكبيرة، عيّن
`OPENCLAW_TRAJECTORY_FLUSH_TIMEOUT_MS` قبل بدء OpenClaw:

```bash
export OPENCLAW_TRAJECTORY_FLUSH_TIMEOUT_MS=30000
```

يتحكم هذا في توقيت تسجيل OpenClaw لانتهاء مهلة `openclaw-trajectory-flush`
ومتابعته؛ ولا يغيّر حدود حجم المسار. لضبط جميع خطوات تنظيف الوكيل
التي لا تمرر مهلة صريحة، عيّن
`OPENCLAW_AGENT_CLEANUP_TIMEOUT_MS`.

## الخصوصية والحدود

حزم المسار مخصصة للدعم وتصحيح الأخطاء، لا للنشر العام. ينقّح OpenClaw
القيم الحساسة قبل كتابة ملفات التصدير:

- بيانات الاعتماد وحقول الحمولات المعروفة بأنها شبيهة بالأسرار
- بيانات الصور
- مسارات الحالة المحلية
- مسارات مساحة العمل، وتُستبدل بـ `$WORKSPACE_DIR`
- مسارات الدليل الرئيسي، حيثما تُكتشف

يضع المُصدّر أيضًا حدودًا لحجم الإدخال:

- التقاط وقت التشغيل: الالتقاط المباشر نافذة متجددة بحد أقصى 10 MiB، وتُسقط أقدم الأحداث لإفساح المجال للجديدة؛ يقبل التصدير ملفات وقت التشغيل الجانبية القديمة الموجودة حتى 50 MiB
- ملفات الجلسة: 50 MiB
- أحداث وقت التشغيل لكل تصدير: 200,000
- إجمالي الأحداث المصدرة: 250,000
- تُقتطع أسطر أحداث وقت التشغيل الفردية عند تجاوز 256 KiB

راجع الحزم قبل مشاركتها خارج فريقك. يُجرى التنقيح بأفضل جهد
ولا يمكنه معرفة كل سر خاص بالتطبيق.

## استكشاف الأخطاء وإصلاحها

إذا لم يتضمن التصدير أي أحداث لوقت التشغيل:

- تأكد من بدء OpenClaw من دون `OPENCLAW_TRAJECTORY=0`
- شغّل رسالة أخرى في الجلسة، ثم صدّر مرة أخرى
- افحص `manifest.json` بحثًا عن `runtimeEventCount`

إذا رفض الأمر مسار الإخراج:

- استخدم اسمًا نسبيًا مثل `bug-1234`
- لا تمرّر `/tmp/...` أو `~/...`
- احتفظ بالتصدير داخل `.openclaw/trajectory-exports/`

إذا فشل التصدير بسبب خطأ في الحجم، فقد تجاوزت الجلسة أو الملف الجانبي
حدود أمان التصدير المذكورة أعلاه. ابدأ جلسة جديدة أو صدّر
إعادة إنتاج أصغر.
