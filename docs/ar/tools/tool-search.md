البحث عن الأدوات هو ميزة تجريبية في وقت تشغيل وكيل OpenClaw. يمنح الوكلاء طريقة
موجزة واحدة لاكتشاف كتالوجات أدوات كبيرة واستدعائها. يكون مفيدا عندما يتضمن التشغيل
العديد من الأدوات المتاحة لكن من المرجح أن يحتاج النموذج إلى عدد قليل منها فقط.

توثق هذه الصفحة البحث عن الأدوات في OpenClaw. وهي ليست سطح البحث عن الأدوات أو
الأدوات الديناميكية الأصلي في Codex. إن وضع الكود الأصلي في Codex، والبحث عن
الأدوات، والأدوات الديناميكية المؤجلة، واستدعاءات الأدوات المتداخلة هي أسطح مستقرة
في حاضنة Codex ولا تعتمد على `tools.toolSearch`.

عند تفعيلها لتشغيلات OpenClaw، يتلقى النموذج أداة `tool_search_code` واحدة
افتراضيا. تشغل تلك الأداة جسما قصيرا من JavaScript في عملية Node فرعية معزولة
باستخدام جسر `openclaw.tools`:

```js
const hits = await openclaw.tools.search("create a GitHub issue");
const tool = await openclaw.tools.describe(hits[0].id);
return await openclaw.tools.call(tool.id, {
  title: "Crash on startup",
  body: "Steps to reproduce...",
});
```

يمكن أن يتضمن الكتالوج أدوات OpenClaw، وأدوات Plugin، وأدوات MCP، وأدوات مقدمة
من العميل. لا يرى النموذج كل مخطط كامل مسبقا. بدلا من ذلك، يبحث في واصفات موجزة،
ويصف أداة واحدة محددة عندما يحتاج إلى المخطط الدقيق، ثم يستدعي تلك الأداة عبر
OpenClaw.

لا تتلقى تشغيلات حاضنة Codex عناصر التحكم التجريبية هذه للبحث عن الأدوات في
OpenClaw. يمرر OpenClaw إمكانات المنتج إلى Codex كأدوات ديناميكية، ويمتلك
Codex وضع الكود الأصلي المستقر، والبحث الأصلي عن الأدوات، والأدوات الديناميكية
المؤجلة، واستدعاءات الأدوات المتداخلة.

## كيفية تشغيل الدور

في وقت التخطيط، يبني مشغل OpenClaw المضمن الكتالوج الفعال للتشغيل:

1. حل سياسة الأدوات النشطة للوكيل، والملف الشخصي، وصندوق الحماية، والجلسة.
2. سرد أدوات OpenClaw وPlugin المؤهلة.
3. سرد أدوات MCP المؤهلة عبر وقت تشغيل MCP للجلسة.
4. إضافة أدوات العميل المؤهلة المقدمة للتشغيل الحالي.
5. فهرسة الواصفات الموجزة للبحث.
6. إتاحة جسر كود OpenClaw، أو أدوات الرجوع المنظمة، أو سطح الدليل الموجز
   للنموذج.

في وقت التنفيذ، يعود كل استدعاء أداة حقيقي إلى OpenClaw. لا يحتفظ وقت تشغيل
Node المعزول بتنفيذات Plugin، أو كائنات عميل MCP، أو الأسرار.
يعبر `openclaw.tools.call(...)` الجسر عائدا إلى Gateway، حيث لا تزال السياسة
العادية، والموافقة، والخطافات، والتسجيل، ومعالجة النتائج مطبقة.

## الأوضاع

يحتوي `tools.toolSearch` على ثلاثة أوضاع موجهة للنموذج:

- `code`: يتيح `tool_search_code`، وهو جسر JavaScript الموجز الافتراضي.
- `tools`: يتيح `tool_search` و`tool_describe` و`tool_call` كأدوات منظمة عادية
  للموفرين الذين لا ينبغي أن يتلقوا كودا.
- `directory`: يتيح `tool_search` و`tool_describe` و`tool_call` بالإضافة إلى
  دليل مطالبة محدود بأسماء الأدوات المتاحة وأوصافها للموفرين الذين ينبغي أن يروا
  أسماء الأدوات دون كل مخطط كامل. يمكن لـ OpenClaw أيضا إتاحة مجموعة صغيرة
  ومحدودة من مخططات الأدوات المرجحة أو المطلوبة مباشرة للدور الحالي.

تستخدم كل الأوضاع الكتالوج نفسه المرشح بالسياسة ومسار تنفيذ OpenClaw العادي.
إذا لم يتمكن وقت التشغيل الحالي من تشغيل عملية Node الفرعية المعزولة لوضع الكود،
يرجع وضع `code` الافتراضي إلى `tools` قبل Compaction الكتالوج. في وضع
`directory`، تبقى الأدوات المقدمة من العميل مرئية مباشرة للتشغيل الحالي، بينما
يمكن ضغط أدوات OpenClaw وأدوات Plugin وأدوات MCP خلف كتالوج الدليل. يتم ترطيب
الاستدعاء المباشر لاسم دليل مخفي دقيق من ذلك الكتالوج المصرح نفسه قبل التنفيذ.

كل الأوضاع تجريبية. فضل إتاحة الأدوات مباشرة لكتالوجات أدوات OpenClaw الصغيرة،
وافضل الأسطح الأصلية المستقرة في Codex لتشغيلات حاضنة Codex.

لا يوجد إعداد منفصل لاختيار المصدر. عند تفعيل البحث عن الأدوات، يتضمن الكتالوج
أدوات OpenClaw وMCP والعميل المؤهلة بعد الترشيح العادي بالسياسة.

## سبب وجود هذا

الكتالوجات الكبيرة مفيدة لكنها مكلفة. إرسال كل مخطط أداة إلى النموذج يجعل الطلب
أكبر، ويبطئ التخطيط، ويزيد اختيار الأدوات بالخطأ.

يغير البحث عن الأدوات الشكل:

- الأدوات المباشرة: يرى النموذج كل مخطط محدد قبل أول رمز
- وضع الكود في البحث عن الأدوات: يرى النموذج أداة كود موجزة واحدة وعقد API قصيرا
- وضع الأدوات في البحث عن الأدوات: يرى النموذج ثلاث أدوات رجوع منظمة موجزة
- وضع الدليل في البحث عن الأدوات: يرى النموذج دليلا محدودا بالإضافة إلى عناصر
  تحكم search/describe/call ومجموعة صغيرة محدودة من المخططات المرجحة أو المطلوبة
- أثناء الدور: يمكن للنموذج تحميل المخططات المتبقية عند الحاجة

لا تزال إتاحة الأدوات المباشرة هي الافتراض الصحيح للكتالوجات الصغيرة. يكون البحث
عن الأدوات أفضل عندما يستطيع تشغيل واحد رؤية العديد من الأدوات، خصوصا من خوادم
MCP أو أدوات تطبيق مقدمة من العميل.

## API

`openclaw.tools.search(query, options?)`

يبحث في الكتالوج الفعال للتشغيل الحالي. النتائج موجزة وآمنة لإعادتها إلى سياق
المطالبة.

```js
const hits = await openclaw.tools.search("calendar event", { limit: 5 });
```

`openclaw.tools.describe(id)`

يحمل البيانات الوصفية الكاملة لنتيجة بحث واحدة، بما في ذلك مخطط الإدخال الدقيق.

```js
const calendarCreate = await openclaw.tools.describe("mcp:calendar:create_event");
```

`openclaw.tools.call(id, args)`

يستدعي أداة محددة عبر OpenClaw.

```js
await openclaw.tools.call(calendarCreate.id, {
  summary: "Planning",
  start: "2026-05-09T14:00:00Z",
});
```

يعرض وضع الرجوع المنظم العمليات نفسها كأدوات:

- `tool_search`
- `tool_describe`
- `tool_call`

يعرض وضع الدليل:

- `tool_search`
- `tool_describe`
- `tool_call`

كما يبقي الأدوات المقدمة من العميل مرئية مباشرة وقد يعرض مجموعة صغيرة محدودة من
مخططات أدوات الكتالوج المرجحة أو المطلوبة مباشرة للدور الحالي. إذا حذف الدليل
المحدود إدخالات، فاستخدم `tool_search` للعثور عليها. إذا طلب النموذج اسم أداة
دليل مخفي دقيقا مباشرة، يرطبه OpenClaw من الكتالوج المصرح قبل التنفيذ العادي.
يجب ألا تتعارض أسماء أدوات العميل في وضع الدليل مع أسماء أدوات OpenClaw أو
Plugin أو MCP لأن الإرسال المؤجل الدقيق يستخدم تلك الأسماء.

## حد وقت التشغيل

يعمل جسر الكود في عملية Node فرعية قصيرة العمر. تبدأ العملية الفرعية مع تفعيل وضع
أذونات Node، وبيئة فارغة، ومن دون منح لنظام الملفات أو الشبكة، ومن دون منح
للعمليات الفرعية أو العمال. يفرض OpenClaw مهلة زمنية جدارية في العملية الأصلية
ويقتل العملية الفرعية عند انتهاء المهلة، بما في ذلك بعد الاستمرارات غير المتزامنة.

يعرض وقت التشغيل فقط:

- `console.log` و`console.warn` و`console.error`
- `openclaw.tools.search`
- `openclaw.tools.describe`
- `openclaw.tools.call`

لا يزال سلوك OpenClaw العادي مطبقا على الاستدعاءات النهائية:

- سياسات السماح بالأدوات ورفضها
- قيود الأدوات لكل وكيل ولكل صندوق حماية
- سياسة أدوات القناة/وقت التشغيل
- خطافات الموافقة
- خطافات Plugin `before_tool_call`
- هوية الجلسة، والسجلات، والقياسات عن بعد

## الإعداد

فعل البحث عن الأدوات لتشغيلات OpenClaw باستخدام جسر الكود الافتراضي:

```bash
openclaw config set tools.toolSearch true
```

JSON المكافئ:

```json5
{
  tools: {
    toolSearch: true,
  },
}
```

استخدم أدوات الرجوع المنظمة بدلا من ذلك لتشغيلات OpenClaw:

```json5
{
  tools: {
    toolSearch: {
      mode: "tools",
    },
  },
}
```

استخدم سطح الدليل الموجز بدلا من ذلك لتشغيلات OpenClaw:

```json5
{
  tools: {
    toolSearch: {
      mode: "directory",
    },
  },
}
```

اضبط مهلة وضع الكود وحدود نتائج البحث:

```json5
{
  tools: {
    toolSearch: {
      mode: "code",
      codeTimeoutMs: 10000,
      searchDefaultLimit: 8,
      maxSearchLimit: 20,
    },
  },
}
```

عطله:

```json5
{
  tools: {
    toolSearch: false,
  },
}
```

## المطالبة والقياسات عن بعد

يسجل البحث عن الأدوات قياسات عن بعد كافية لمقارنته بإتاحة الأدوات مباشرة:

- إجمالي بايتات الأدوات والمطالبات المتسلسلة المرسلة إلى الحاضنة
- حجم الكتالوج وتفصيل المصادر
- أعداد البحث، والوصف، والاستدعاء
- استدعاءات الأدوات النهائية المنفذة عبر OpenClaw
- معرفات الأدوات المحددة ومصادرها

ينبغي أن تجعل سجلات الجلسة من الممكن الإجابة عن:

- عدد مخططات الأدوات التي رآها النموذج مسبقا
- عدد عمليات البحث والوصف التي أجراها
- الأداة النهائية التي تم استدعاؤها
- ما إذا كانت النتيجة جاءت من OpenClaw أو MCP أو أداة عميل

## تحقق E2E

يثبت مشغل E2E الخاص بـ Gateway كلا المسارين باستخدام وقت تشغيل OpenClaw:

```bash
node --import tsx scripts/tool-search-gateway-e2e.ts
```

ينشئ Plugin وهميا مؤقتا بكتالوج أدوات كبير، ويبدأ موفر OpenAI الوهمي، ويبدأ
Gateway مرة في الوضع المباشر ومرة مع تفعيل البحث عن الأدوات، ثم يقارن حمولات
طلبات الموفر وسجلات الجلسة.

يثبت الاختبار الارتدادي:

1. يمكن للوضع المباشر استدعاء أداة Plugin الوهمية.
2. يمكن للبحث عن الأدوات استدعاء أداة Plugin الوهمية نفسها.
3. يعرض الوضع المباشر مخططات أداة Plugin الوهمية مباشرة للموفر.
4. يعرض البحث عن الأدوات الجسر الموجز فقط.
5. تكون حمولة طلب البحث عن الأدوات أصغر للكتالوج الوهمي الكبير.
6. تظهر سجلات الجلسة أعداد استدعاءات الأدوات المتوقعة وقياسات الاستدعاء الموصولة
   عبر الجسر.

## سلوك الفشل

ينبغي أن يفشل البحث عن الأدوات بإغلاق آمن:

- إذا لم تكن أداة ضمن السياسة الفعالة، فينبغي ألا يعيدها البحث
- إذا أصبحت أداة محددة غير متاحة، فينبغي أن يفشل `tool_call`
- إذا منعت السياسة أو الموافقة التنفيذ، فينبغي أن تبلغ نتيجة الاستدعاء عن ذلك
  الحظر بدلا من تجاوزه
- إذا لم يتمكن جسر الكود من إنشاء وقت تشغيل معزول، فاستخدم `mode: "tools"` أو
  عطل البحث عن الأدوات لذلك النشر
