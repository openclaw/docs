يمكن لوكلاء OpenClaw إنشاء مقاطع فيديو من مطالبات نصية، أو صور مرجعية، أو مقاطع فيديو موجودة. هناك أربع عشرة واجهة مزود خلفية مدعومة، ولكل منها خيارات نماذج، وأوضاع إدخال، ومجموعات ميزات مختلفة. ويختار الوكيل المزوّد المناسب تلقائيًا بناءً على إعداداتك ومفاتيح API المتاحة لديك.

<Note>
لا تظهر أداة `video_generate` إلا عندما يكون هناك مزود واحد على الأقل لإنشاء الفيديو متاحًا. إذا لم ترها ضمن أدوات الوكيل لديك، فاضبط مفتاح API لمزوّد ما أو هيّئ `agents.defaults.videoGenerationModel`.
</Note>

يتعامل OpenClaw مع إنشاء الفيديو باعتباره ثلاثة أوضاع وقت تشغيل:

- `generate` لطلبات النص إلى فيديو من دون أي وسائط مرجعية
- `imageToVideo` عندما يتضمن الطلب صورة مرجعية واحدة أو أكثر
- `videoToVideo` عندما يتضمن الطلب مقطع فيديو مرجعيًا واحدًا أو أكثر

يمكن للمزوّدين دعم أي مجموعة فرعية من هذه الأوضاع. وتتحقق الأداة من صحة
الوضع النشط قبل الإرسال وتعرض الأوضاع المدعومة في `action=list`.

## بداية سريعة

1. اضبط مفتاح API لأي مزوّد مدعوم:

```bash
export GEMINI_API_KEY="your-key"
```

2. يمكنك اختياريًا تثبيت نموذج افتراضي:

```bash
openclaw config set agents.defaults.videoGenerationModel.primary "google/veo-3.1-fast-generate-preview"
```

3. اطلب من الوكيل:

> أنشئ فيديو سينمائيًا مدته 5 ثوانٍ لجراد بحر ودود يركب الأمواج عند الغروب.

سيستدعي الوكيل `video_generate` تلقائيًا. لا حاجة إلى قائمة سماح للأداة.

## ماذا يحدث عندما تُنشئ فيديو

إنشاء الفيديو غير متزامن. عندما يستدعي الوكيل `video_generate` داخل جلسة:

1. يرسل OpenClaw الطلب إلى المزوّد ويعيد فورًا معرّف مهمة.
2. يعالج المزوّد المهمة في الخلفية (عادةً من 30 ثانية إلى 5 دقائق بحسب المزوّد والدقة).
3. عندما يصبح الفيديو جاهزًا، يوقظ OpenClaw الجلسة نفسها بحدث إكمال داخلي.
4. ينشر الوكيل الفيديو النهائي مرة أخرى في المحادثة الأصلية.

أثناء وجود مهمة قيد التنفيذ، تعيد استدعاءات `video_generate` المكررة في الجلسة نفسها حالة المهمة الحالية بدلًا من بدء عملية إنشاء أخرى. استخدم `openclaw tasks list` أو `openclaw tasks show <taskId>` للتحقق من التقدم من CLI.

خارج تشغيلات الوكيل المدعومة بالجلسة (مثل استدعاءات الأداة المباشرة)، تعود الأداة إلى الإنشاء المضمّن وتعيد مسار الوسائط النهائي في الدور نفسه.

### دورة حياة المهمة

ينتقل كل طلب `video_generate` عبر أربع حالات:

1. **queued** -- تم إنشاء المهمة، وهي بانتظار أن يقبلها المزوّد.
2. **running** -- يعالج المزوّد المهمة (عادةً من 30 ثانية إلى 5 دقائق بحسب المزوّد والدقة).
3. **succeeded** -- أصبح الفيديو جاهزًا؛ يستيقظ الوكيل وينشره في المحادثة.
4. **failed** -- خطأ من المزوّد أو انتهاء مهلة؛ يستيقظ الوكيل مع تفاصيل الخطأ.

تحقق من الحالة من CLI:

```bash
openclaw tasks list
openclaw tasks show <taskId>
openclaw tasks cancel <taskId>
```

منع التكرار: إذا كانت هناك بالفعل مهمة فيديو بالحالة `queued` أو `running` للجلسة الحالية، فستعيد `video_generate` حالة المهمة الموجودة بدلًا من بدء مهمة جديدة. استخدم `action: "status"` للتحقق صراحةً من دون تشغيل إنشاء جديد.

## معلمات الأداة

### مطلوبة

| المعلمة | النوع | الوصف |
| --------- | ------ | ----------------------------------------------------------------------------- |
| `prompt` | string | الوصف النصي للفيديو المراد إنشاؤه (مطلوب عند `action: "generate"`) |

### مدخلات المحتوى

| المعلمة | النوع | الوصف |
| ------------ | -------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| `image` | string | صورة مرجعية واحدة (مسار أو URL) |
| `images` | string[] | عدة صور مرجعية (حتى 9) |
| `imageRoles` | string[] | تلميحات أدوار اختيارية لكل موضع بالتوازي مع قائمة الصور المجمعة. القيم القياسية: `first_frame` و`last_frame` و`reference_image` |
| `video` | string | فيديو مرجعي واحد (مسار أو URL) |
| `videos` | string[] | عدة فيديوهات مرجعية (حتى 4) |
| `videoRoles` | string[] | تلميحات أدوار اختيارية لكل موضع بالتوازي مع قائمة الفيديوهات المجمعة. القيمة القياسية: `reference_video` |
| `audioRef` | string | مرجع صوتي واحد (مسار أو URL). يُستخدم مثلًا للموسيقى الخلفية أو مرجع الصوت عندما يدعم المزوّد مدخلات الصوت |
| `audioRefs` | string[] | عدة مراجع صوتية (حتى 3) |
| `audioRoles` | string[] | تلميحات أدوار اختيارية لكل موضع بالتوازي مع قائمة الصوت المجمعة. القيمة القياسية: `reference_audio` |

تُمرر تلميحات الأدوار إلى المزوّد كما هي. وتأتي القيم القياسية من
اتحاد `VideoGenerationAssetRole`، لكن قد تقبل المزوّدات
سلاسل أدوار إضافية. ويجب ألا تحتوي مصفوفات `*Roles` على عدد إدخالات أكبر من
قائمة المراجع المقابلة؛ وتفشل أخطاء الزيادة أو النقصان بمقدار واحد مع رسالة خطأ واضحة.
استخدم سلسلة فارغة لترك موضع غير مضبوط.

### عناصر التحكم في النمط

| المعلمة | النوع | الوصف |
| ----------------- | ------- | --------------------------------------------------------------------------------------- |
| `aspectRatio` | string | `1:1`، أو `2:3`، أو `3:2`، أو `3:4`، أو `4:3`، أو `4:5`، أو `5:4`، أو `9:16`، أو `16:9`، أو `21:9`، أو `adaptive` |
| `resolution` | string | `480P`، أو `720P`، أو `768P`، أو `1080P` |
| `durationSeconds` | number | المدة المستهدفة بالثواني (تُقرّب إلى أقرب قيمة يدعمها المزوّد) |
| `size` | string | تلميح للحجم عندما يدعمه المزوّد |
| `audio` | boolean | تفعيل الصوت المُولّد في المخرجات عندما يكون مدعومًا. وهو مختلف عن `audioRef*` (المدخلات) |
| `watermark` | boolean | تبديل العلامة المائية الخاصة بالمزوّد عندما تكون مدعومة |

تُعد `adaptive` قيمة sentinel خاصة بالمزوّد: إذ تُمرر كما هي إلى
المزوّدين الذين يعلنون `adaptive` ضمن قدراتهم (مثل BytePlus
Seedance التي تستخدمها لاكتشاف النسبة تلقائيًا من أبعاد
صورة الإدخال). أما المزوّدون الذين لا يعلنونها فيعرضون القيمة عبر
`details.ignoredOverrides` في نتيجة الأداة بحيث يكون إسقاطها مرئيًا.

### متقدمة

| المعلمة | النوع | الوصف |
| ----------------- | ------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `action` | string | `"generate"` (الافتراضي)، أو `"status"`، أو `"list"` |
| `model` | string | تجاوز للمزوّد/النموذج (مثل `runway/gen4.5`) |
| `filename` | string | تلميح لاسم ملف الإخراج |
| `timeoutMs` | number | مهلة اختيارية لطلب المزوّد بالمللي ثانية |
| `providerOptions` | object | خيارات خاصة بالمزوّد على هيئة كائن JSON ‏(مثل `{"seed": 42, "draft": true}`). تتحقق المزوّدات التي تعلن مخططًا Typed من المفاتيح والأنواع؛ وتؤدي المفاتيح غير المعروفة أو عدم التطابق إلى تخطي المرشح أثناء fallback. أما المزوّدات التي لا تحتوي على مخطط معلن فتتلقى الخيارات كما هي. شغّل `video_generate action=list` لرؤية ما يقبله كل مزوّد |

لا تدعم جميع المزوّدات جميع المعلمات. يقوم OpenClaw بالفعل بتطبيع المدة إلى أقرب قيمة يدعمها المزوّد، كما يعيد أيضًا تعيين تلميحات الهندسة المترجمة مثل التحويل من الحجم إلى نسبة الأبعاد عندما يوفّر مزوّد fallback سطح تحكم مختلفًا. أما التجاوزات غير المدعومة فعلًا فيتم تجاهلها على أساس أفضل جهد ويجري الإبلاغ عنها كتحذيرات في نتيجة الأداة. وتفشل الحدود الصارمة للإمكانات (مثل وجود عدد كبير جدًا من المدخلات المرجعية) قبل الإرسال.

تبلغ نتائج الأداة عن الإعدادات المطبقة. وعندما يعيد OpenClaw تعيين المدة أو الهندسة أثناء fallback بين المزوّدين، فإن القيم المعادة `durationSeconds` و`size` و`aspectRatio` و`resolution` تعكس ما تم إرساله، وتلتقط `details.normalization` التحويل من المطلوب إلى المطبق.

تحدد المدخلات المرجعية أيضًا وضع وقت التشغيل:

- بدون وسائط مرجعية: `generate`
- أي مرجع صورة: `imageToVideo`
- أي مرجع فيديو: `videoToVideo`
- لا تغيّر مدخلات الصوت المرجعية الوضع المحلول؛ بل تُطبّق فوق أي وضع تحدده مراجع الصور/الفيديو، ولا تعمل إلا مع المزوّدين الذين يعلنون `maxInputAudios`

ليست مراجع الصور والفيديو المختلطة سطح قدرات مشتركة مستقرة.
ومن الأفضل استخدام نوع مرجعي واحد لكل طلب.

#### Fallback والخيارات Typed

تُطبّق بعض فحوصات الإمكانات في طبقة fallback بدلًا من
حدود الأداة، حتى يتمكن الطلب الذي يتجاوز حدود المزوّد الأساسي
من العمل مع fallback قادرة:

- إذا لم يعلن المرشح النشط عن `maxInputAudios` (أو أعلنها على أنها
  `0`)، فسيتم تخطيه عندما يحتوي الطلب على مراجع صوتية، ثم
  تُجرَّب المرشح التالية.
- إذا كانت قيمة `maxDurationSeconds` للمرشح النشط أقل من
  `durationSeconds` المطلوبة ولم يعلن المرشح عن قائمة
  `supportedDurationSeconds`، فسيتم تخطيه.
- إذا احتوى الطلب على `providerOptions` وكان المرشح النشط
  يعلن صراحةً عن مخطط Typed لـ `providerOptions`، فسيتم
  تخطي المرشح عندما لا تكون المفاتيح الموردة موجودة في المخطط أو عندما لا
  تتطابق أنواع القيم. أما المزوّدات التي لم تعلن بعد عن مخطط فتتلقى
  الخيارات كما هي (تمرير متوافق مع الإصدارات السابقة). ويمكن للمزوّد
  إلغاء الاشتراك صراحةً في جميع provider options عبر إعلان مخطط فارغ
  (`capabilities.providerOptions: {}`)، مما يسبب التخطي نفسه كما في
  عدم تطابق النوع.

يتم تسجيل أول سبب للتخطي في الطلب عند مستوى `warn` حتى يرى المشغلون
متى تم تجاوز مزوّدهم الأساسي؛ أما التخطيّات اللاحقة فتُسجَّل عند
`debug` للحفاظ على هدوء سلاسل fallback الطويلة. وإذا تم تخطي كل المرشحين،
فسيتضمن الخطأ المجمّع سبب التخطي لكل واحد منهم.

## الإجراءات

- **generate** (الافتراضي) -- إنشاء فيديو من المطالبة المعطاة والمدخلات المرجعية الاختيارية.
- **status** -- التحقق من حالة مهمة الفيديو قيد التنفيذ للجلسة الحالية من دون بدء إنشاء جديد.
- **list** -- عرض المزوّدين المتاحين، والنماذج، وقدراتها.

## اختيار النموذج

عند إنشاء فيديو، يحل OpenClaw النموذج بهذا الترتيب:

1. **معلمة الأداة `model`** -- إذا حددها الوكيل في الاستدعاء.
2. **`videoGenerationModel.primary`** -- من الإعداد.
3. **`videoGenerationModel.fallbacks`** -- يتم تجربتها بالترتيب.
4. **الاكتشاف التلقائي** -- يستخدم المزوّدين الذين لديهم مصادقة صالحة، بدءًا من المزوّد الافتراضي الحالي، ثم بقية المزوّدين بترتيب أبجدي.

إذا فشل أحد المزوّدين، تتم تجربة المرشح التالي تلقائيًا. وإذا فشل كل المرشحين، فسيتضمن الخطأ تفاصيل كل محاولة.

اضبط `agents.defaults.mediaGenerationAutoProviderFallback: false` إذا كنت تريد
أن يستخدم إنشاء الفيديو فقط الإدخالات الصريحة `model`، و`primary`، و`fallbacks`.

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "google/veo-3.1-fast-generate-preview",
        fallbacks: ["runway/gen4.5", "qwen/wan2.6-t2v"],
      },
    },
  },
}
```

## أوضاع قدرات المزوّد

يسمح عقد إنشاء الفيديو المشترك الآن للمزوّدين بالإعلان عن قدرات خاصة بكل وضع
بدلًا من حدود إجمالية مسطحة فقط. ويجب أن تفضّل
تنفيذات المزوّدين الجديدة كتل الأوضاع الصريحة:

```typescript
capabilities: {
  generate: {
    maxVideos: 1,
    maxDurationSeconds: 10,
    supportsResolution: true,
  },
  imageToVideo: {
    enabled: true,
    maxVideos: 1,
    maxInputImages: 1,
    maxDurationSeconds: 5,
  },
  videoToVideo: {
    enabled: true,
    maxVideos: 1,
    maxInputVideos: 1,
    maxDurationSeconds: 5,
  },
}
```

لا تكفي الحقول الإجمالية المسطحة مثل `maxInputImages` و`maxInputVideos`
للإعلان عن دعم أوضاع التحويل. يجب على المزوّدين الإعلان عن
`generate` و`imageToVideo` و`videoToVideo` صراحةً حتى تتمكن الاختبارات الحية،
واختبارات العقد، وأداة `video_generate` المشتركة من التحقق من دعم الأوضاع
بشكل حتمي.

## الاختبارات الحية

تغطية حية اختيارية للمزوّدين المجمّعين المشتركين:

```bash
OPENCLAW_LIVE_TEST=1 pnpm test:live -- extensions/video-generation-providers.live.test.ts
```

غلاف المستودع:

```bash
pnpm test:live:media video
```

يحمّل هذا الملف الحي متغيرات البيئة المفقودة للمزوّد من `~/.profile`، ويفضّل
مفاتيح API الحية/من env على ملفات تعريف المصادقة المخزنة افتراضيًا، ويشغّل
اختبارًا دخانيًا آمنًا للإصدار افتراضيًا:

- `generate` لكل مزوّد غير FAL في المسح
- مطالبة جراد بحر مدتها ثانية واحدة
- حد العمليات لكل مزوّد من `OPENCLAW_LIVE_VIDEO_GENERATION_TIMEOUT_MS`
  (الافتراضي `180000`)

يكون FAL اختياريًا لأن زمن الانتظار في الطابور من جهة المزوّد قد يهيمن على وقت الإصدار:

```bash
pnpm test:live:media video --video-providers fal
```

اضبط `OPENCLAW_LIVE_VIDEO_GENERATION_FULL_MODES=1` لتشغيل أوضاع التحويل المعلنة أيضًا
التي يمكن للمسح المشترك اختبارها بأمان باستخدام وسائط محلية:

- `imageToVideo` عندما تكون `capabilities.imageToVideo.enabled`
- `videoToVideo` عندما تكون `capabilities.videoToVideo.enabled` وكان المزوّد/النموذج
  يقبل إدخال فيديو محليًا مدعومًا بـ buffer في المسح المشترك

يغطي مسار `videoToVideo` الحي المشترك اليوم:

- `runway` فقط عندما تختار `runway/gen4_aleph`

## الإعداد

اضبط نموذج إنشاء الفيديو الافتراضي في إعداد OpenClaw لديك:

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "qwen/wan2.6-t2v",
        fallbacks: ["qwen/wan2.6-r2v-flash"],
      },
    },
  },
}
```

أو عبر CLI:

```bash
openclaw config set agents.defaults.videoGenerationModel.primary "qwen/wan2.6-t2v"
```
