# توليد الفيديو

يمكن لوكلاء OpenClaw إنشاء مقاطع فيديو من مطالبات نصية، أو صور مرجعية، أو مقاطع فيديو موجودة. هناك اثنتا عشرة واجهة خلفية للموفّرين مدعومة، ولكل منها خيارات نماذج مختلفة، وأوضاع إدخال، ومجموعات ميزات. يختار الوكيل الموفّر المناسب تلقائيًا بناءً على إعدادك ومفاتيح API المتاحة.

<Note>
لا تظهر أداة `video_generate` إلا عندما يكون موفّر واحد على الأقل لتوليد الفيديو متاحًا. إذا لم ترها ضمن أدوات الوكيل لديك، فاضبط مفتاح API لموفّر أو قم بإعداد `agents.defaults.videoGenerationModel`.
</Note>

يتعامل OpenClaw مع توليد الفيديو باعتباره ثلاثة أوضاع لبيئة التشغيل:

- `generate` لطلبات تحويل النص إلى فيديو من دون وسائط مرجعية
- `imageToVideo` عندما يتضمن الطلب صورة مرجعية واحدة أو أكثر
- `videoToVideo` عندما يتضمن الطلب مقطع فيديو مرجعيًا واحدًا أو أكثر

يمكن للموفّرين دعم أي مجموعة فرعية من هذه الأوضاع. تتحقق الأداة من صحة
الوضع النشط قبل الإرسال وتبلّغ عن الأوضاع المدعومة في `action=list`.

## بدء سريع

1. اضبط مفتاح API لأي موفّر مدعوم:

```bash
export GEMINI_API_KEY="your-key"
```

2. اختياريًا، ثبّت نموذجًا افتراضيًا:

```bash
openclaw config set agents.defaults.videoGenerationModel.primary "google/veo-3.1-fast-generate-preview"
```

3. اطلب من الوكيل:

> أنشئ مقطع فيديو سينمائيًا مدته 5 ثوانٍ لكركند ودود يركب الأمواج عند الغروب.

سيستدعي الوكيل `video_generate` تلقائيًا. لا حاجة إلى قائمة سماح للأدوات.

## ماذا يحدث عندما تنشئ مقطع فيديو

توليد الفيديو غير متزامن. عندما يستدعي الوكيل `video_generate` في جلسة ما:

1. يرسل OpenClaw الطلب إلى الموفّر ويعيد فورًا معرّف مهمة.
2. يعالج الموفّر المهمة في الخلفية (عادةً من 30 ثانية إلى 5 دقائق بحسب الموفّر والدقة).
3. عندما يصبح الفيديو جاهزًا، يوقظ OpenClaw الجلسة نفسها بحدث إكمال داخلي.
4. ينشر الوكيل الفيديو النهائي مرة أخرى في المحادثة الأصلية.

أثناء وجود مهمة قيد التنفيذ، تعيد استدعاءات `video_generate` المكررة في الجلسة نفسها حالة المهمة الحالية بدلًا من بدء عملية توليد أخرى. استخدم `openclaw tasks list` أو `openclaw tasks show <taskId>` للتحقق من التقدم من CLI.

خارج تشغيلات الوكيل المعتمدة على الجلسة (على سبيل المثال، الاستدعاءات المباشرة للأداة)، تعود الأداة إلى التوليد المضمّن وتعيد مسار الوسائط النهائي في الجولة نفسها.

### دورة حياة المهمة

ينتقل كل طلب `video_generate` عبر أربع حالات:

1. **queued** -- تم إنشاء المهمة، وهي بانتظار أن يقبلها الموفّر.
2. **running** -- يعالج الموفّر المهمة (عادةً من 30 ثانية إلى 5 دقائق بحسب الموفّر والدقة).
3. **succeeded** -- أصبح الفيديو جاهزًا؛ يستيقظ الوكيل وينشره في المحادثة.
4. **failed** -- خطأ من الموفّر أو مهلة؛ يستيقظ الوكيل مع تفاصيل الخطأ.

تحقق من الحالة من CLI:

```bash
openclaw tasks list
openclaw tasks show <taskId>
openclaw tasks cancel <taskId>
```

منع التكرار: إذا كانت هناك مهمة فيديو بالفعل بالحالة `queued` أو `running` للجلسة الحالية، فإن `video_generate` يعيد حالة المهمة الموجودة بدلًا من بدء مهمة جديدة. استخدم `action: "status"` للتحقق صراحةً من دون تشغيل توليد جديد.

## معلمات الأداة

### مطلوبة

| المعلمة | النوع  | الوصف                                                                        |
| ------- | ------ | ---------------------------------------------------------------------------- |
| `prompt` | string | وصف نصي للفيديو المراد إنشاؤه (مطلوب من أجل `action: "generate"`) |

### مدخلات المحتوى

| المعلمة | النوع    | الوصف                                 |
| ------- | -------- | ------------------------------------- |
| `image` | string   | صورة مرجعية واحدة (مسار أو URL)       |
| `images` | string[] | عدة صور مرجعية (حتى 5)               |
| `video` | string   | فيديو مرجعي واحد (مسار أو URL)        |
| `videos` | string[] | عدة مقاطع فيديو مرجعية (حتى 4)       |

### عناصر التحكم في النمط

| المعلمة          | النوع   | الوصف                                                                    |
| ---------------- | ------- | ------------------------------------------------------------------------ |
| `aspectRatio`    | string  | `1:1` أو `2:3` أو `3:2` أو `3:4` أو `4:3` أو `4:5` أو `5:4` أو `9:16` أو `16:9` أو `21:9` |
| `resolution`     | string  | `480P` أو `720P` أو `768P` أو `1080P`                                    |
| `durationSeconds` | number | المدة المستهدفة بالثواني (تُقرَّب إلى أقرب قيمة يدعمها الموفّر)         |
| `size`           | string  | تلميح للحجم عندما يدعمه الموفّر                                           |
| `audio`          | boolean | تفعيل الصوت المُنشأ عند دعمه                                              |
| `watermark`      | boolean | تبديل العلامة المائية الخاصة بالموفّر عند دعمها                           |

### متقدمة

| المعلمة   | النوع  | الوصف                                                |
| --------- | ------ | ---------------------------------------------------- |
| `action`  | string | `"generate"` (الافتراضي) أو `"status"` أو `"list"`   |
| `model`   | string | تجاوز الموفّر/النموذج (مثل `runway/gen4.5`)          |
| `filename` | string | تلميح لاسم ملف الإخراج                              |

لا تدعم جميع الموفّرين جميع المعلمات. يقوم OpenClaw بالفعل بتطبيع المدة إلى أقرب قيمة يدعمها الموفّر، كما يعيد تعيين تلميحات الهندسة المترجمة مثل تحويل الحجم إلى نسبة أبعاد عندما يكشف موفّر احتياطي عن سطح تحكم مختلف. أما التجاوزات غير المدعومة فعلًا فيتم تجاهلها على أساس أفضل جهد ممكن وتُبلّغ كتحذيرات في نتيجة الأداة. وتفشل حدود القدرات الصارمة (مثل وجود عدد كبير جدًا من المدخلات المرجعية) قبل الإرسال.

تُبلّغ نتائج الأداة عن الإعدادات المطبّقة. وعندما يعيد OpenClaw تعيين المدة أو الهندسة أثناء الرجوع إلى موفّر احتياطي، فإن القيم المعادة `durationSeconds` و `size` و `aspectRatio` و `resolution` تعكس ما تم إرساله، ويلتقط `details.normalization` التحويل من المطلوب إلى المطبّق.

كما تحدد المدخلات المرجعية وضع بيئة التشغيل:

- لا توجد وسائط مرجعية: `generate`
- أي مرجع صورة: `imageToVideo`
- أي مرجع فيديو: `videoToVideo`

لا يُعد خلط مراجع الصور والفيديو سطح قدرات مشتركًا ومستقرًا.
يُفضّل استخدام نوع مرجعي واحد لكل طلب.

## الإجراءات

- **generate** (الافتراضي) -- أنشئ فيديو من المطالبة المعطاة والمدخلات المرجعية الاختيارية.
- **status** -- تحقق من حالة مهمة الفيديو الجاري تنفيذها للجلسة الحالية من دون بدء عملية توليد أخرى.
- **list** -- اعرض الموفّرين والنماذج وقدراتهم المتاحة.

## اختيار النموذج

عند إنشاء فيديو، يحل OpenClaw النموذج بهذا الترتيب:

1. **معلمة الأداة `model`** -- إذا حدّدها الوكيل في الاستدعاء.
2. **`videoGenerationModel.primary`** -- من الإعداد.
3. **`videoGenerationModel.fallbacks`** -- تتم تجربتها بالترتيب.
4. **الاكتشاف التلقائي** -- يستخدم الموفّرين الذين لديهم مصادقة صالحة، بدءًا من الموفّر الافتراضي الحالي، ثم بقية الموفّرين بترتيب أبجدي.

إذا فشل أحد الموفّرين، تتم تجربة المرشح التالي تلقائيًا. وإذا فشل جميع المرشحين، فسيتضمن الخطأ تفاصيل من كل محاولة.

اضبط `agents.defaults.mediaGenerationAutoProviderFallback: false` إذا كنت تريد
أن يستخدم توليد الفيديو فقط إدخالات `model` و `primary` و `fallbacks`
الصريحة.

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "google/veo-3.1-fast-generate-preview",
        fallbacks: ["runway/gen4.5", "qwen/wan2.6-t2v"],
      },
    },
  },
}
```

يمكن تثبيت وكيل الفيديو HeyGen على fal باستخدام:

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "fal/fal-ai/heygen/v2/video-agent",
      },
    },
  },
}
```

يمكن تثبيت Seedance 2.0 على fal باستخدام:

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "fal/bytedance/seedance-2.0/fast/text-to-video",
      },
    },
  },
}
```

## أوضاع قدرات الموفّر

يتيح عقد توليد الفيديو المشترك الآن للموفّرين إعلان قدرات خاصة بكل وضع
بدلًا من الاكتفاء بالحدود المجمّعة المسطحة فقط. ويجب أن تفضّل
تنفيذات الموفّرين الجديدة كتل الأوضاع الصريحة:

```typescript
capabilities: {
  generate: {
    maxVideos: 1,
    maxDurationSeconds: 10,
    supportsResolution: true,
  },
  imageToVideo: {
    enabled: true,
    maxVideos: 1,
    maxInputImages: 1,
    maxDurationSeconds: 5,
  },
  videoToVideo: {
    enabled: true,
    maxVideos: 1,
    maxInputVideos: 1,
    maxDurationSeconds: 5,
  },
}
```

لا تكفي الحقول المجمّعة المسطحة مثل `maxInputImages` و `maxInputVideos`
للإعلان عن دعم أوضاع التحويل. يجب على الموفّرين أن يعلنوا
`generate` و `imageToVideo` و `videoToVideo` صراحةً حتى تتمكن الاختبارات الحية،
واختبارات العقود، وأداة `video_generate` المشتركة من التحقق من دعم الأوضاع
بشكل حتمي.

## الاختبارات الحية

تغطية حية اختيارية للموفّرين المضمّنين المشتركين:

```bash
OPENCLAW_LIVE_TEST=1 pnpm test:live -- extensions/video-generation-providers.live.test.ts
```

مغلف المستودع:

```bash
pnpm test:live:media video
```

يحمّل هذا الملف الحي متغيرات env الناقصة الخاصة بالموفّر من `~/.profile`، ويفضّل
مفاتيح API الحية/من env على ملفات المصادقة المخزنة افتراضيًا، ويشغّل
الأوضاع المعلنة التي يمكنه اختبارها بأمان باستخدام الوسائط المحلية:

- `generate` لكل موفّر في المسح
- `imageToVideo` عندما تكون `capabilities.imageToVideo.enabled`
- `videoToVideo` عندما تكون `capabilities.videoToVideo.enabled` ويقبل الموفّر/النموذج
  إدخال فيديو محليًا مدعومًا بالمخزن المؤقت في المسح المشترك

يغطي مسار `videoToVideo` الحي المشترك اليوم:

- `runway` فقط عندما تختار `runway/gen4_aleph`

## الإعداد

اضبط نموذج توليد الفيديو الافتراضي في إعداد OpenClaw الخاص بك:

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "qwen/wan2.6-t2v",
        fallbacks: ["qwen/wan2.6-r2v-flash"],
      },
    },
  },
}
```

أو عبر CLI:

```bash
openclaw config set agents.defaults.videoGenerationModel.primary "qwen/wan2.6-t2v"
```
