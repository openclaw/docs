يمكن لوكلاء OpenClaw إنشاء مقاطع فيديو من موجّهات نصية أو صور مرجعية أو مقاطع فيديو موجودة. هناك أربع عشرة خلفية مزوّدين مدعومة، ولكل منها خيارات نماذج وأنماط إدخال ومجموعات ميزات مختلفة. ويختار الوكيل المزوّد المناسب تلقائيًا استنادًا إلى إعداداتك ومفاتيح API المتاحة.

<Note>
لا تظهر الأداة `video_generate` إلا عند توفر مزوّد واحد على الأقل لتوليد الفيديو. وإذا لم ترها ضمن أدوات وكيلك، فاضبط مفتاح API للمزوّد أو هيّئ `agents.defaults.videoGenerationModel`.
</Note>

يتعامل OpenClaw مع توليد الفيديو بوصفه ثلاثة أوضاع وقت تشغيل:

- `generate` لطلبات النص إلى فيديو التي لا تحتوي على وسائط مرجعية
- `imageToVideo` عندما يتضمن الطلب صورة مرجعية واحدة أو أكثر
- `videoToVideo` عندما يتضمن الطلب مقطع فيديو مرجعيًا واحدًا أو أكثر

يمكن أن يدعم المزوّد أي مجموعة فرعية من هذه الأوضاع. وتتحقق الأداة من الوضع النشط
قبل الإرسال وتبلّغ عن الأوضاع المدعومة في `action=list`.

## بداية سريعة

1. اضبط مفتاح API لأي مزوّد مدعوم:

```bash
export GEMINI_API_KEY="your-key"
```

2. ويمكنك اختياريًا تثبيت نموذج افتراضي:

```bash
openclaw config set agents.defaults.videoGenerationModel.primary "google/veo-3.1-fast-generate-preview"
```

3. اطلب من الوكيل:

> أنشئ فيديو سينمائيًا مدته 5 ثوانٍ لكركند ودود يتزلج على الأمواج عند الغروب.

يستدعي الوكيل `video_generate` تلقائيًا. لا حاجة إلى قائمة سماح للأدوات.

## ما الذي يحدث عند إنشاء فيديو

توليد الفيديو غير متزامن. عندما يستدعي الوكيل `video_generate` داخل جلسة:

1. يرسل OpenClaw الطلب إلى المزوّد ويعيد فورًا معرّف مهمة.
2. يعالج المزوّد المهمة في الخلفية (عادةً من 30 ثانية إلى 5 دقائق بحسب المزوّد والدقة).
3. عندما يصبح الفيديو جاهزًا، يوقظ OpenClaw الجلسة نفسها بحدث إكمال داخلي.
4. ينشر الوكيل الفيديو النهائي مرة أخرى في المحادثة الأصلية.

أثناء تنفيذ مهمة ما، تعيد استدعاءات `video_generate` المكررة في الجلسة نفسها حالة المهمة الحالية بدلًا من بدء عملية توليد أخرى. استخدم `openclaw tasks list` أو `openclaw tasks show <taskId>` للتحقق من التقدّم من CLI.

خارج عمليات تشغيل الوكيل المدعومة بالجلسات (على سبيل المثال، الاستدعاءات المباشرة للأدوات)، تعود الأداة إلى التوليد المضمّن وتعيد مسار الوسائط النهائي في الدور نفسه.

تُحفَظ ملفات الفيديو المُولَّدة ضمن مساحة تخزين الوسائط المُدارة من OpenClaw عندما
يعيد المزوّد بايتات. ويتبع الحد الأقصى الافتراضي لحفظ الفيديو المُولَّد حد وسائط
الفيديو، كما يرفع `agents.defaults.mediaMaxMb` هذا الحد للعمليات الأكبر.
وعندما يعيد المزوّد أيضًا عنوان URL مستضافًا للمخرجات، يمكن لـ OpenClaw تسليم ذلك العنوان
بدلًا من فشل المهمة إذا رفض التخزين المحلي ملفًا كبيرًا جدًا.

### دورة حياة المهمة

يمر كل طلب `video_generate` عبر أربع حالات:

1. **queued** -- تم إنشاء المهمة وتنتظر قبول المزوّد لها.
2. **running** -- يعالج المزوّد المهمة (عادةً من 30 ثانية إلى 5 دقائق بحسب المزوّد والدقة).
3. **succeeded** -- أصبح الفيديو جاهزًا؛ فيستيقظ الوكيل وينشره في المحادثة.
4. **failed** -- خطأ من المزوّد أو انتهاء مهلة؛ فيستيقظ الوكيل مع تفاصيل الخطأ.

تحقق من الحالة من CLI:

```bash
openclaw tasks list
openclaw tasks show <taskId>
openclaw tasks cancel <taskId>
```

منع التكرار: إذا كانت هناك بالفعل مهمة فيديو بالحالة `queued` أو `running` للجلسة الحالية، فإن `video_generate` يعيد حالة المهمة الحالية بدلًا من بدء مهمة جديدة. استخدم `action: "status"` للتحقق صراحةً دون تشغيل عملية توليد جديدة.

| ------------ | -------- | -------------------------------------------------------------------------------------------------------------------------------------- |

| `video`      | string   | مقطع فيديو مرجعي واحد (مسار أو URL)                                                                                                   |
| `videos`     | string[] | مقاطع فيديو مرجعية متعددة (حتى 4)                                                                                                    |
| `videoRoles` | string[] | تلميحات أدوار اختيارية لكل موضع بالتوازي مع قائمة الفيديو المجمّعة. القيمة الأساسية: `reference_video`                               |
| `audioRef`   | string   | مرجع صوتي واحد (مسار أو URL). يُستخدم مثلًا للموسيقى الخلفية أو مرجع الصوت عندما يدعم المزوّد مدخلات الصوت        |
| `audioRefs`  | string[] | مراجع صوتية متعددة (حتى 3)                                                                                                    |
| `audioRoles` | string[] | تلميحات أدوار اختيارية لكل موضع بالتوازي مع قائمة الصوت المجمّعة. القيمة الأساسية: `reference_audio`                               |

تُمرَّر تلميحات الأدوار إلى المزوّد كما هي. وتأتي القيم الأساسية من
اتحاد `VideoGenerationAssetRole` لكن قد تقبل المزوّدات سلاسل أدوار
إضافية. ويجب ألا تحتوي مصفوفات `*Roles` على إدخالات أكثر من
قائمة المراجع المقابلة؛ وتفشل أخطاء الزيادة أو النقصان بمقدار عنصر واحد مع خطأ واضح.
استخدم سلسلة فارغة لترك خانة من دون تعيين. وبالنسبة إلى xAI، اضبط كل دور صورة على
`reference_image` لاستخدام وضع التوليد `reference_images` الخاص به؛ واحذف الدور
أو استخدم `first_frame` لصورة واحدة في وضع الصورة إلى فيديو.

### عناصر التحكّم في النمط

| المعلمة         | النوع    | الوصف                                                                             |
| ----------------- | ------- | --------------------------------------------------------------------------------------- |
| `aspectRatio`     | string  | `1:1` أو `2:3` أو `3:2` أو `3:4` أو `4:3` أو `4:5` أو `5:4` أو `9:16` أو `16:9` أو `21:9` أو `adaptive`  |
| `resolution`      | string  | `480P` أو `720P` أو `768P` أو `1080P`                                                      |
| `durationSeconds` | number  | المدة المستهدفة بالثواني (تُقرَّب إلى أقرب قيمة يدعمها المزوّد)                |
| `size`            | string  | تلميح الحجم عندما يدعمه المزوّد                                                 |
| `audio`           | boolean | تفعيل الصوت المُولَّد في المخرجات عند دعمه. وهو يختلف عن `audioRef*` (المدخلات) |
| `watermark`       | boolean | تبديل العلامة المائية الخاصة بالمزوّد عند دعمها                                             |

تُعد `adaptive` قيمة حارسة خاصة بالمزوّد: وتُمرَّر كما هي إلى
المزوّدين الذين يعلنون `adaptive` ضمن قدراتهم (على سبيل المثال، يستخدمها BytePlus
Seedance لاكتشاف النسبة تلقائيًا من أبعاد
الصورة المدخلة). أما المزوّدون الذين لا يعلنون عنها فيُظهرون القيمة عبر
`details.ignoredOverrides` في نتيجة الأداة بحيث يكون إسقاطها مرئيًا.

| `providerOptions` | object | خيارات خاصة بالمزوّد على هيئة كائن JSON (مثل `{"seed": 42, "draft": true}`). تتحقق المزوّدات التي تعلن مخططًا مطبّعًا من المفاتيح والأنواع؛ وتؤدي المفاتيح غير المعروفة أو حالات عدم التطابق إلى تخطي المرشح أثناء الرجوع الاحتياطي. أما المزوّدات التي لا تعلن مخططًا فتستقبل الخيارات كما هي. شغّل `video_generate action=list` لمعرفة ما الذي يقبله كل مزوّد |

لا تدعم جميع المزوّدات كل المعلمات. ويقوم OpenClaw بالفعل بتطبيع المدة إلى أقرب قيمة يدعمها المزوّد، كما يعيد أيضًا تعيين تلميحات الهندسة المترجمة مثل تحويل الحجم إلى نسبة أبعاد عندما يكشف مزوّد احتياطي سطح تحكّم مختلفًا. أما التجاوزات غير المدعومة فعلًا فتُتجاهل على أساس أفضل جهد ويُبلَّغ عنها كتحذيرات في نتيجة الأداة. بينما تؤدي حدود القدرات الصارمة (مثل كثرة المدخلات المرجعية) إلى الفشل قبل الإرسال.

تبلّغ نتائج الأداة عن الإعدادات المطبقة. وعندما يعيد OpenClaw تعيين المدة أو الهندسة أثناء الرجوع الاحتياطي بين المزوّدات، فإن القيم المعادة لـ `durationSeconds` و`size` و`aspectRatio` و`resolution` تعكس ما تم إرساله، وتلتقط `details.normalization` التحويل من المطلوب إلى المطبق.

تحدد المدخلات المرجعية أيضًا وضع وقت التشغيل:

- بلا وسائط مرجعية: `generate`
- أي مرجع صورة: `imageToVideo`
- أي مرجع فيديو: `videoToVideo`
- لا تغيّر المدخلات الصوتية المرجعية الوضع المحلول؛ بل تُطبَّق فوق أي وضع تختاره المراجع الصورية/الفيديو، ولا تعمل إلا مع المزوّدات التي تعلن `maxInputAudios`

لا تمثّل المراجع المختلطة من الصور والفيديو سطح قدرات مشتركة مستقرًا.
ومن الأفضل استخدام نوع مرجعي واحد لكل طلب.

#### الرجوع الاحتياطي والخيارات المطبّعة

تُطبَّق بعض فحوصات القدرات على طبقة الرجوع الاحتياطي بدلًا من
حدود الأداة بحيث يمكن للطلب الذي يتجاوز حدود المزوّد الأساسي
أن يعمل مع مزوّد احتياطي قادر:

- إذا لم يعلن المرشح النشط عن `maxInputAudios` (أو أعلنها على أنها
  `0`)، فيُتخطى عندما يحتوي الطلب على مراجع صوتية، وتُجرَّب
  المرشح التالي.
- إذا كانت قيمة `maxDurationSeconds` لدى المرشح النشط أقل من
  `durationSeconds` المطلوبة ولم يعلن المرشح عن قائمة
  `supportedDurationSeconds`، فيُتخطى.
- إذا احتوى الطلب على `providerOptions` وكان المرشح النشط
  يعلن صراحةً عن مخطط `providerOptions` مطبّع، فإن المرشح
  يُتخطى عندما لا تكون المفاتيح المزوّدة ضمن المخطط أو عندما لا
  تتطابق أنواع القيم. أما المزوّدات التي لم تعلن مخططًا بعد فتستقبل
  الخيارات كما هي (تمرير متوافق مع الإصدارات السابقة). ويمكن للمزوّد
  إلغاء الاشتراك صراحةً في جميع خيارات المزوّد بإعلان مخطط فارغ
  (`capabilities.providerOptions: {}`)، ما يؤدي إلى التخطي نفسه كما في
  عدم تطابق النوع.

يُسجَّل أول سبب للتخطي في الطلب عند مستوى `warn` حتى يرى المشغّلون
متى تم تجاوز مزوّدهم الأساسي؛ أما حالات التخطي اللاحقة فتُسجَّل عند
`debug` لإبقاء سلاسل الرجوع الاحتياطي الطويلة هادئة. وإذا تم تخطي كل المرشحين،
فيتضمن الخطأ المجمّع سبب التخطي لكل واحد منهم.

## الإجراءات

- **generate** (الافتراضي) -- أنشئ فيديو من الموجّه المحدد والمدخلات المرجعية الاختيارية.
- **status** -- تحقّق من حالة مهمة الفيديو قيد التنفيذ للجلسة الحالية من دون بدء عملية توليد أخرى.
- **list** -- اعرض المزوّدين، والنماذج، وقدراتهم المتاحة.

## اختيار النموذج

عند توليد فيديو، يحل OpenClaw النموذج بهذا الترتيب:

1. **معلمة الأداة `model`** -- إذا حدّدها الوكيل في الاستدعاء.
2. **`videoGenerationModel.primary`** -- من الإعدادات.
3. **`videoGenerationModel.fallbacks`** -- تُجرَّب بالترتيب.
4. **الاكتشاف التلقائي** -- يستخدم المزوّدات التي لديها مصادقة صالحة، بدءًا من المزوّد الافتراضي الحالي، ثم بقية المزوّدات بترتيب أبجدي.

إذا فشل أحد المزوّدين، فيُجرَّب المرشح التالي تلقائيًا. وإذا فشل جميع المرشحين، يتضمن الخطأ تفاصيل من كل محاولة.

اضبط `agents.defaults.mediaGenerationAutoProviderFallback: false` إذا كنت تريد
أن يستخدم توليد الفيديو فقط إدخالات `model` و`primary` و`fallbacks`
الصريحة.

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "google/veo-3.1-fast-generate-preview",
        fallbacks: ["runway/gen4.5", "qwen/wan2.6-t2v"],
      },
    },
  },
}
```

## أوضاع قدرات المزوّد

يتيح عقد توليد الفيديو المشترك الآن للمزوّدين إعلان
قدرات خاصة بكل وضع بدلًا من حدود كلية مسطحة فقط. ويجب أن تفضّل
تنفيذات المزوّدين الجديدة كتل الأوضاع الصريحة:

```typescript
capabilities: {
  generate: {
    maxVideos: 1,
    maxDurationSeconds: 10,
    supportsResolution: true,
  },
  imageToVideo: {
    enabled: true,
    maxVideos: 1,
    maxInputImages: 1,
    maxDurationSeconds: 5,
  },
  videoToVideo: {
    enabled: true,
    maxVideos: 1,
    maxInputVideos: 1,
    maxDurationSeconds: 5,
  },
}
```

إن الحقول الكلية المسطحة مثل `maxInputImages` و`maxInputVideos` ليست
كافية للإعلان عن دعم أوضاع التحويل. وينبغي للمزوّدين إعلان
`generate`، و`imageToVideo`، و`videoToVideo` صراحةً حتى تتمكن الاختبارات الحية،
واختبارات العقد، والأداة المشتركة `video_generate` من التحقق من دعم الأوضاع
بصورة حتمية.

## الاختبارات الحية

تغطية حية اختيارية للمزوّدين المضمّنين المشتركين:

```bash
OPENCLAW_LIVE_TEST=1 pnpm test:live -- extensions/video-generation-providers.live.test.ts
```

مغلّف المستودع:

```bash
pnpm test:live:media video
```

يحمّل هذا الملف الحي متغيرات بيئة المزوّدين المفقودة من `~/.profile`، ويفضّل
مفاتيح API الحية/البيئية على ملفات تعريف المصادقة المخزنة افتراضيًا، ويشغّل
فحصًا آمنًا للإصدار افتراضيًا:

- `generate` لكل مزوّد غير FAL ضمن الفحص
- موجّه كركند مدته ثانية واحدة
- حد عمليات لكل مزوّد من `OPENCLAW_LIVE_VIDEO_GENERATION_TIMEOUT_MS`
  (الافتراضي `180000`)

يكون FAL اختياريًا لأن كمون الطابور من جهة المزوّد قد يهيمن على زمن الإصدار:

```bash
pnpm test:live:media video --video-providers fal
```

اضبط `OPENCLAW_LIVE_VIDEO_GENERATION_FULL_MODES=1` لتشغيل أوضاع التحويل المُعلنة أيضًا
التي يستطيع الفحص المشترك ممارستها بأمان باستخدام وسائط محلية:

- `imageToVideo` عندما تكون `capabilities.imageToVideo.enabled`
- `videoToVideo` عندما تكون `capabilities.videoToVideo.enabled` وعندما يقبل المزوّد/النموذج
  إدخال فيديو محليًا مدعومًا بالذاكرة المؤقتة في الفحص المشترك

اليوم يغطي المسار الحي المشترك لـ `videoToVideo` ما يلي:

- `runway` فقط عندما تختار `runway/gen4_aleph`

## الإعدادات

اضبط نموذج توليد الفيديو الافتراضي في إعدادات OpenClaw الخاصة بك:

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "qwen/wan2.6-t2v",
        fallbacks: ["qwen/wan2.6-r2v-flash"],
      },
    },
  },
}
```

أو عبر CLI:

```bash
openclaw config set agents.defaults.videoGenerationModel.primary "qwen/wan2.6-t2v"
```
