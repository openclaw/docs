تتيح أداة `music_generate` للوكيل إنشاء موسيقى أو صوت عبر
قدرة توليد الموسيقى المشتركة باستخدام موفّرين مضبوطة مثل Google،
وMiniMax، وComfyUI المضبوطة عبر سير العمل.

بالنسبة إلى جلسات الوكيل المعتمدة على موفّرين مشتركة، يبدأ OpenClaw توليد الموسيقى
كمهمة في الخلفية، ويتتبعها في سجل المهام، ثم يوقظ الوكيل مرة أخرى عندما يصبح
المسار جاهزًا حتى يتمكن الوكيل من نشر الصوت النهائي مرة أخرى في
القناة الأصلية.

<Note>
لا تظهر الأداة المشتركة المضمنة إلا عندما يكون موفر واحد على الأقل لتوليد الموسيقى متاحًا. وإذا لم ترَ `music_generate` ضمن أدوات الوكيل لديك، فاضبط `agents.defaults.musicGenerationModel` أو قم بإعداد مفتاح API لموفر.
</Note>

## بدء سريع

### توليد معتمد على موفّرين مشتركة

1. اضبط مفتاح API لموفر واحد على الأقل، مثل `GEMINI_API_KEY` أو
   `MINIMAX_API_KEY`.
2. اضبط نموذجك المفضل اختياريًا:

```json5
{
  agents: {
    defaults: {
      musicGenerationModel: {
        primary: "google/lyria-3-clip-preview",
      },
    },
  },
}
```

3. اطلب من الوكيل: _"Generate an upbeat synthpop track about a night drive
   through a neon city."_

سيستدعي الوكيل `music_generate` تلقائيًا. ولا حاجة إلى قائمة سماح للأداة.

أما في السياقات المتزامنة المباشرة من دون تشغيل وكيل مدعوم بجلسة، فإن الأداة
المضمنة ما تزال ترجع إلى التوليد المضمن وتعيد مسار الوسائط النهائي في
نتيجة الأداة.

أمثلة على المطالبات:

```text
Generate a cinematic piano track with soft strings and no vocals.
```

```text
Generate an energetic chiptune loop about launching a rocket at sunrise.
```

### توليد Comfy المعتمد على سير العمل

تتكامل Plugin ‏`comfy` المضمنة مع أداة `music_generate` المشتركة عبر
سجل موفري توليد الموسيقى.

1. اضبط `models.providers.comfy.music` باستخدام JSON لسير العمل و
   عقد prompt/output.
2. إذا كنت تستخدم Comfy Cloud، فاضبط `COMFY_API_KEY` أو `COMFY_CLOUD_API_KEY`.
3. اطلب من الوكيل موسيقى أو استدعِ الأداة مباشرة.

مثال:

```text
/tool music_generate prompt="Warm ambient synth loop with soft tape texture"
```

## دعم الموفّرين المضمنين المشتركة

| الموفّر  | النموذج الافتراضي       | مدخلات مرجعية   | عناصر التحكم المدعومة                                 | مفتاح API                                |
| -------- | ----------------------- | --------------- | ----------------------------------------------------- | ---------------------------------------- |
| ComfyUI  | `workflow`              | حتى صورة واحدة  | موسيقى أو صوت يحددهما سير العمل                       | `COMFY_API_KEY`, `COMFY_CLOUD_API_KEY`   |
| Google   | `lyria-3-clip-preview`  | حتى 10 صور      | `lyrics`, `instrumental`, `format`                    | `GEMINI_API_KEY`, `GOOGLE_API_KEY`       |
| MiniMax  | `music-2.5+`            | لا شيء          | `lyrics`, `instrumental`, `durationSeconds`, `format=mp3` | `MINIMAX_API_KEY`                    |

### مصفوفة القدرات المعلنة

هذا هو عقد الأوضاع الصريح المستخدم بواسطة `music_generate`، واختبارات العقود،
والفحص المباشر المشترك.

| الموفّر  | `generate` | `edit` | حد التعديل | المسارات المباشرة المشتركة                                                     |
| -------- | ---------- | ------ | ---------- | ------------------------------------------------------------------------------ |
| ComfyUI  | نعم        | نعم    | صورة واحدة | ليست ضمن الفحص المشترك؛ وتُغطى بواسطة `extensions/comfy/comfy.live.test.ts`   |
| Google   | نعم        | نعم    | 10 صور     | `generate`, `edit`                                                             |
| MiniMax  | نعم        | لا     | لا شيء     | `generate`                                                                     |

استخدم `action: "list"` لفحص الموفّرين والنماذج المشتركة المتاحة في
وقت التشغيل:

```text
/tool music_generate action=list
```

استخدم `action: "status"` لفحص مهمة الموسيقى النشطة المعتمدة على الجلسة:

```text
/tool music_generate action=status
```

مثال على التوليد المباشر:

```text
/tool music_generate prompt="Dreamy lo-fi hip hop with vinyl texture and gentle rain" instrumental=true
```

## معلمات الأداة المضمنة

| المعلمة          | النوع    | الوصف                                                                                           |
| ---------------- | -------- | ------------------------------------------------------------------------------------------------ |
| `prompt`         | string   | مطالبة توليد الموسيقى (مطلوبة لـ `action: "generate"`)                                          |
| `action`         | string   | `"generate"` ‏(افتراضي)، أو `"status"` لمهمة الجلسة الحالية، أو `"list"` لفحص الموفّرين        |
| `model`          | string   | تجاوز provider/model، مثل `google/lyria-3-pro-preview` أو `comfy/workflow`                      |
| `lyrics`         | string   | كلمات اختيارية عندما يدعم الموفّر إدخال كلمات صريح                                             |
| `instrumental`   | boolean  | طلب مخرجات آلية فقط عندما يدعمها الموفّر                                                         |
| `image`          | string   | مسار صورة مرجعية واحد أو عنوان URL                                                              |
| `images`         | string[] | عدة صور مرجعية (حتى 10)                                                                          |
| `durationSeconds`| number   | المدة المستهدفة بالثواني عندما يدعم الموفّر تلميحات المدة                                        |
| `timeoutMs`      | number   | مهلة اختيارية لطلب الموفّر بالمللي ثانية                                                        |
| `format`         | string   | تلميح تنسيق الإخراج (`mp3` أو `wav`) عندما يدعمه الموفّر                                        |
| `filename`       | string   | تلميح اسم ملف الإخراج                                                                            |

لا يدعم جميع الموفّرين جميع المعلمات. ومع ذلك، يتحقق OpenClaw من الحدود الصلبة
مثل عدد المدخلات قبل الإرسال. وعندما يدعم الموفّر المدة لكنه
يستخدم حدًا أقصى أقصر من القيمة المطلوبة، فإن OpenClaw تقيدها تلقائيًا
إلى أقرب مدة مدعومة. أما التلميحات الاختيارية غير المدعومة حقًا فيتم تجاهلها
مع تحذير عندما لا يستطيع الموفّر أو النموذج المحدد احترامها.

تبلغ نتائج الأداة عن الإعدادات المطبقة. وعندما تقيد OpenClaw المدة أثناء الرجوع إلى موفر بديل، تعكس `durationSeconds` المعادة القيمة المرسلة بينما تُظهر `details.normalization.durationSeconds` الربط بين القيمة المطلوبة والمطبقة.

## السلوك غير المتزامن للمسار المعتمد على موفّرين مشتركة

- عمليات تشغيل الوكيل المدعومة بجلسة: تقوم `music_generate` بإنشاء مهمة في الخلفية، وتعيد استجابة started/task فورًا، وتنشر المسار النهائي لاحقًا في رسالة وكيل متابعة.
- منع التكرار: ما دامت مهمة الخلفية تلك لا تزال `queued` أو `running`، فإن استدعاءات `music_generate` اللاحقة في الجلسة نفسها تعيد حالة المهمة بدلًا من بدء توليد آخر.
- فحص الحالة: استخدم `action: "status"` لفحص مهمة الموسيقى النشطة المعتمدة على الجلسة من دون بدء توليد جديد.
- تتبع المهام: استخدم `openclaw tasks list` أو `openclaw tasks show <taskId>` لفحص الحالات queued وrunning والحالات النهائية الخاصة بعملية التوليد.
- إيقاظ الإكمال: تقوم OpenClaw بحقن حدث إكمال داخلي مرة أخرى في الجلسة نفسها حتى يتمكن النموذج من كتابة رسالة المتابعة الموجهة للمستخدم بنفسه.
- تلميح المطالبة: تحصل الأدوار اللاحقة الخاصة بالمستخدم/اليدوية في الجلسة نفسها على تلميح صغير أثناء وقت التشغيل عندما تكون مهمة موسيقى جارية بالفعل حتى لا يستدعي النموذج `music_generate` مرة أخرى بشكل أعمى.
- الرجوع من دون جلسة: لا تزال السياقات المباشرة/المحلية من دون جلسة وكيل حقيقية تعمل بشكل مضمن وتعيد نتيجة الصوت النهائية في الدور نفسه.

### دورة حياة المهمة

ينتقل كل طلب `music_generate` عبر أربع حالات:

1. **queued** -- تم إنشاء المهمة، وهي تنتظر أن يقبلها الموفّر.
2. **running** -- يعالجها الموفّر (عادةً من 30 ثانية إلى 3 دقائق بحسب الموفّر والمدة).
3. **succeeded** -- أصبح المسار جاهزًا؛ يستيقظ الوكيل وينشره في المحادثة.
4. **failed** -- خطأ من الموفّر أو انتهاء مهلة؛ يستيقظ الوكيل مع تفاصيل الخطأ.

تحقق من الحالة من CLI:

```bash
openclaw tasks list
openclaw tasks show <taskId>
openclaw tasks cancel <taskId>
```

منع التكرار: إذا كانت هناك بالفعل مهمة موسيقى في الحالة `queued` أو `running` للجلسة الحالية، فإن `music_generate` تعيد حالة المهمة الموجودة بدلًا من بدء مهمة جديدة. استخدم `action: "status"` للتحقق صراحةً من دون تشغيل توليد جديد.

## الإعداد

### اختيار النموذج

```json5
{
  agents: {
    defaults: {
      musicGenerationModel: {
        primary: "google/lyria-3-clip-preview",
        fallbacks: ["minimax/music-2.5+"],
      },
    },
  },
}
```

### ترتيب اختيار الموفّر

عند توليد الموسيقى، تحاول OpenClaw الموفّرين بهذا الترتيب:

1. معلمة `model` من استدعاء الأداة، إذا حدد الوكيل واحدة
2. `musicGenerationModel.primary` من الإعدادات
3. `musicGenerationModel.fallbacks` بالترتيب
4. الاكتشاف التلقائي باستخدام قيم الموفّرين الافتراضية المدعومة بالمصادقة فقط:
   - الموفّر الافتراضي الحالي أولًا
   - بقية موفّري توليد الموسيقى المسجلين بترتيب معرّف الموفّر

إذا فشل موفّر، تتم تجربة المرشح التالي تلقائيًا. وإذا فشل الجميع، يتضمن
الخطأ تفاصيل كل محاولة.

اضبط `agents.defaults.mediaGenerationAutoProviderFallback: false` إذا كنت تريد أن تستخدم
عملية توليد الموسيقى فقط الإدخالات الصريحة `model` و`primary` و`fallbacks`.

## ملاحظات خاصة بالموفّرين

- تستخدم Google التوليد الدفعي Lyria 3. ويدعم التدفق المضمن الحالي
  المطالبة، ونص كلمات اختياري، وصورًا مرجعية اختيارية.
- تستخدم MiniMax نقطة النهاية الدفعية `music_generation`. ويدعم التدفق المضمن الحالي
  المطالبة، وكلمات اختيارية، ووضعًا آليًا، وتوجيهًا للمدة، وخرج
  mp3.
- دعم ComfyUI مدفوع بسير العمل ويعتمد على الرسم البياني المضبوط بالإضافة إلى
  ربط العقد لحقول prompt/output.

## أوضاع قدرات الموفّر

يدعم عقد توليد الموسيقى المشترك الآن تعريفات أوضاع صريحة:

- `generate` للتوليد اعتمادًا على المطالبة فقط
- `edit` عندما يتضمن الطلب صورة مرجعية واحدة أو أكثر

ينبغي لتطبيقات الموفّرين الجديدة أن تفضّل كتل الأوضاع الصريحة:

```typescript
capabilities: {
  generate: {
    maxTracks: 1,
    supportsLyrics: true,
    supportsFormat: true,
  },
  edit: {
    enabled: true,
    maxTracks: 1,
    maxInputImages: 1,
    supportsFormat: true,
  },
}
```

لا تكفي الحقول المسطحة القديمة مثل `maxInputImages` و`supportsLyrics` و
`supportsFormat` للإعلان عن دعم التعديل. ويجب على الموفّرين
التصريح بـ `generate` و`edit` صراحةً حتى تتمكن الاختبارات المباشرة واختبارات العقود
والأداة المشتركة `music_generate` من التحقق من دعم الأوضاع بشكل حتمي.

## اختيار المسار الصحيح

- استخدم المسار المعتمد على موفّرين مشتركة عندما تريد اختيار النماذج، وfailover الموفّرين، وتدفق task/status غير المتزامن المضمن.
- استخدم مسار Plugin مثل ComfyUI عندما تحتاج إلى رسم بياني مخصص لسير العمل أو إلى موفّر ليس جزءًا من قدرة الموسيقى المضمنة المشتركة.
- إذا كنت تصحح سلوكًا خاصًا بـ ComfyUI، فراجع [ComfyUI](/ar/providers/comfy). وإذا كنت تصحح سلوكًا خاصًا بموفّرين مشتركة، فابدأ من [Google ‏(Gemini)](/ar/providers/google) أو [MiniMax](/ar/providers/minimax).

## الاختبارات المباشرة

التغطية المباشرة الاختيارية للموفّرين المضمنين المشتركة:

```bash
OPENCLAW_LIVE_TEST=1 pnpm test:live -- extensions/music-generation-providers.live.test.ts
```

غلاف المستودع:

```bash
pnpm test:live:media music
```

يقوم هذا الملف المباشر بتحميل متغيرات بيئة الموفّرين المفقودة من `~/.profile`، ويفضل
مفاتيح API الحية/الموجودة في env على ملفات المصادقة المخزنة افتراضيًا، ويشغّل كلا من
تغطية `generate` وتغطية `edit` المعلنة عندما يفعّل الموفّر وضع التعديل.

وهذا يعني اليوم:

- `google`: ‏`generate` بالإضافة إلى `edit`
- `minimax`: ‏`generate` فقط
- `comfy`: تغطية مباشرة منفصلة لـ Comfy، وليست ضمن فحص الموفّرين المشترك

التغطية المباشرة الاختيارية لمسار ComfyUI الموسيقي المضمن:

```bash
OPENCLAW_LIVE_TEST=1 COMFY_LIVE_TEST=1 pnpm test:live -- extensions/comfy/comfy.live.test.ts
```

يغطي ملف Comfy المباشر أيضًا مسارات الصور والفيديو الخاصة بـ comfy عندما تكون تلك
الأقسام مضبوطة.
