عندما يعمل وكيل داخل sandbox، تكون أوامر `exec` الخاصة به محصورة داخل
بيئة sandbox. يتيح **وضع Elevated** للوكيل الخروج من العزل وتشغيل الأوامر
خارجه بدلًا من ذلك، مع بوابات موافقة قابلة للتهيئة.

<Info>
  لا يغيّر وضع Elevated السلوك إلا عندما يكون الوكيل **معزولًا داخل sandbox**. أما
  بالنسبة إلى الوكلاء غير المعزولين، فإن exec يعمل بالفعل على المضيف.
</Info>

## التوجيهات

تحكم في وضع Elevated لكل جلسة باستخدام أوامر الشرطة المائلة:

| التوجيه | ما الذي يفعله |
| ------- | ------------- |
| `/elevated on` | التشغيل خارج sandbox على مسار المضيف المضبوط، مع الإبقاء على الموافقات |
| `/elevated ask` | مثل `on` ‏(اسم مستعار) |
| `/elevated full` | التشغيل خارج sandbox على مسار المضيف المضبوط وتجاوز الموافقات |
| `/elevated off` | العودة إلى التنفيذ المحصور داخل sandbox |

ومتاح أيضًا بالشكل `/elev on|off|ask|full`.

أرسل `/elevated` من دون وسيطة لرؤية المستوى الحالي.

## كيف يعمل

<Steps>
  <Step title="تحقق من الإتاحة">
    يجب أن يكون Elevated مفعّلًا في الإعدادات، ويجب أن يكون المرسل ضمن قائمة السماح:

    ```json5
    {
      tools: {
        elevated: {
          enabled: true,
          allowFrom: {
            discord: ["user-id-123"],
            whatsapp: ["+15555550123"],
          },
        },
      },
    }
    ```

  </Step>

  <Step title="اضبط المستوى">
    أرسل رسالة تحتوي على التوجيه فقط لضبط الإعداد الافتراضي للجلسة:

    ```
    /elevated full
    ```

    أو استخدمه ضمن الرسالة نفسها (ويُطبَّق على تلك الرسالة فقط):

    ```
    /elevated on run the deployment script
    ```

  </Step>

  <Step title="تُشغَّل الأوامر خارج sandbox">
    عند تفعيل elevated، تغادر استدعاءات `exec` الـ sandbox. ويكون المضيف الفعّال
    هو `gateway` افتراضيًا، أو `node` عندما يكون هدف exec المضبوط/الخاص بالجلسة هو
    `node`. وفي وضع `full`، يتم تجاوز موافقات exec. أما في وضعي `on`/`ask`،
    فلا تزال قواعد الموافقة المضبوطة تُطبّق.
  </Step>
</Steps>

## ترتيب الحلّ

1. **توجيه ضمني** داخل الرسالة (يُطبَّق على تلك الرسالة فقط)
2. **تجاوز الجلسة** ‏(يُضبط عبر إرسال رسالة تحتوي على التوجيه فقط)
3. **الافتراضي العام** ‏(`agents.defaults.elevatedDefault` في الإعدادات)

## الإتاحة وقوائم السماح

- **البوابة العامة**: ‏`tools.elevated.enabled` ‏(يجب أن تكون `true`)
- **قائمة سماح المرسِل**: ‏`tools.elevated.allowFrom` مع قوائم لكل قناة
- **بوابة لكل وكيل**: ‏`agents.list[].tools.elevated.enabled` ‏(يمكنها فقط فرض مزيد من التقييد)
- **قائمة سماح لكل وكيل**: ‏`agents.list[].tools.elevated.allowFrom` ‏(يجب أن يطابق المرسل كلًا من القائمة العامة + الخاصة بالوكيل)
- **رجوع احتياطي لـ Discord**: إذا تم حذف `tools.elevated.allowFrom.discord`، تُستخدم `channels.discord.allowFrom` كرجوع احتياطي
- **يجب أن تنجح جميع البوابات**؛ وإلا يُعامل elevated على أنه غير متاح

تنسيقات إدخالات قائمة السماح:

| السابقة | ما الذي تطابقه |
| ------- | --------------- |
| (لا شيء) | معرّف المرسل، أو E.164، أو حقل From |
| `name:` | اسم العرض الخاص بالمرسل |
| `username:` | اسم المستخدم الخاص بالمرسل |
| `tag:` | الوسم الخاص بالمرسل |
| `id:`, `from:`, `e164:` | استهداف هوية صريح |

## ما الذي لا يتحكم فيه Elevated

- **سياسة الأدوات**: إذا كان `exec` محظورًا بواسطة سياسة الأدوات، فلا يمكن لـ elevated تجاوزه
- **سياسة اختيار المضيف**: لا يحوّل elevated القيمة `auto` إلى تجاوز حر عبر المضيفين. بل يستخدم قواعد هدف exec المضبوطة/الخاصة بالجلسة، ويختار `node` فقط عندما يكون الهدف أصلًا `node`.
- **منفصل عن `/exec`**: يضبط التوجيه `/exec` افتراضيات exec لكل جلسة للمرسلين المخولين ولا يتطلب وضع elevated
