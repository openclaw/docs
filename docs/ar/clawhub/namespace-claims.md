# مطالبات المؤسسات ومساحات الأسماء

يستخدم ClawHub معرّفات المالكين، ومعرّفات المؤسسات، ومعرّفات المهارات المختصرة، وأسماء حزم Plugin،
ونطاقات الحزم كمساحات أسماء عامة. إذا بدت مساحة اسم وكأنها تنتمي إلى
مشروع واقعي، أو علامة تجارية، أو منظومة حزم، أو مؤسسة، لكنها
مطالب بها مسبقًا أو محجوزة أو مضللة أو محل نزاع على ClawHub، فاطلب من الفريق مراجعتها
باستخدام
[نموذج مشكلة مطالبة بمؤسسة / مساحة اسم](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة غير الحساسة. لا تستخدم البلاغات
داخل المنتج أو نموذج استئناف الحساب لمطالبات مساحات الأسماء.

## متى تفتح مطالبة

افتح مطالبة بمساحة اسم عندما تعتقد أن فريق ClawHub يجب أن يراجع ما إذا كان ينبغي
حجز مساحة اسم، أو نقلها، أو إعادة تسميتها، أو إخفاؤها، أو عزلها، أو إضافة اسم بديل لها،
أو تغييرها بطريقة أخرى بسبب ملكية واقعية.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسسة GitHub الخاصة بك، أو مشروعك، أو شركتك، أو مجتمعك
- نطاق حزمة مثل `@example-org/*` ينبغي أن ينشر فقط تحت مالك ClawHub
  المطابق
- معرّف مهارة مختصر أو اسم حزمة Plugin يبدو أنه ينتحل هوية مشروع
- نزاع حول علامة تجارية، أو علامة مسجلة، أو إعادة تسمية مشروع، أو تاريخ حزمة
- مالك محذوف أو غير نشط أو غير قابل للوصول يحجب مالك مساحة الاسم
  الشرعي

إذا كانت القائمة غير آمنة أو ضارة أو مضللة بما يتجاوز نزاع الملكية،
فاتبع أيضًا إرشادات الإشراف أو الأمان ذات الصلة. نموذج مطالبة مساحة الاسم
مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات.

## قبل أن تقدم

تأكد أولًا من أنك تنشر باستخدام المالك الذي يطابق مساحة الاسم.
بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin`
باسم مالك `example-org` المطابق.

إذا كنت تستطيع إدارة المالك الحالي، فأصلح مساحة الاسم مباشرة عبر النشر،
أو إعادة التسمية، أو النقل، أو الإخفاء، أو حذف المورد المتأثر. استخدم مطالبة
عندما لا تستطيع إدارة المالك الحالي أو عندما يحتاج الفريق إلى حل
نزاع.

## الأدلة المطلوب تضمينها

استخدم أدلة عامة غير حساسة. تشمل الأدلة المفيدة:

- تاريخ مؤسسة GitHub أو المستودع أو الإصدار أو المشرفين
- وثائق المشروع الرسمية التي تذكر مساحة الاسم
- إثبات النطاق أو نطاق البريد الإلكتروني الرسمي
- التحكم في نطاق npm أو PyPI أو crates.io أو سجل حزم آخر
- أدلة ملكية علامة مسجلة أو علامة تجارية أو مشروع يمكن مناقشتها
  علنًا بأمان
- تاريخ مستودع المصدر، أو تاريخ الحزمة، أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub أو المهارة أو Plugin أو الحزمة أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. يجب أن يتمكن الفريق من فهم
العلاقة دون الحاجة إلى بيانات اعتماد خاصة أو أسرار.

## ما يجب عدم تضمينه

لا تضع أسرارًا أو أدلة خاصة في مشكلة GitHub عامة. لا تضمّن:

- رموز API، أو مفاتيح التوقيع، أو بيانات الاعتماد
- رموز تحدي DNS
- ملفات أو عقود قانونية خاصة
- وثائق هوية شخصية
- رسائل بريد إلكتروني خاصة، أو تقارير أمان خاصة، أو بيانات عملاء سرية

يسأل نموذج المطالبة عما إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع الفريق.
استخدم هذا الخيار بدلًا من نشر المواد الحساسة علنًا.

## النتائج المحتملة

اعتمادًا على الأدلة والمخاطر، قد يحجز فريق ClawHub مساحة اسم،
أو ينقل الملكية، أو يعيد تسمية مورد، أو يخفي أو يعزل قائمة موجودة،
أو يضيف اسمًا بديلًا أو إعادة توجيه، أو يطلب مزيدًا من الأدلة، أو يرفض الطلب.

لا تضمن مراجعة مساحة الاسم نقل كل اسم مطابق.
يزن الفريق الأدلة العامة، والاستخدام الحالي، ومخاطر الأمان، وتأثير ذلك على المستخدمين.
