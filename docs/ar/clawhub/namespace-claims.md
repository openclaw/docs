# مطالبات المؤسسة ومساحة الأسماء

يستخدم ClawHub معرّفات المالكين، ومعرّفات المؤسسات، ومعرّفات المهارات، وأسماء حزم Plugin، ونطاقات
الحزم كمساحات أسماء عامة. إذا بدت مساحة أسماء وكأنها تعود إلى
مشروع واقعي أو علامة تجارية أو منظومة حزم أو مؤسسة، لكنها كانت
مطالبًا بها أو محجوزة أو مضللة أو محل نزاع بالفعل على ClawHub، فاطلب من فريق العمل مراجعتها
عبر
[نموذج مشكلة مطالبة مؤسسة / مساحة أسماء](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة غير الحساسة. لا تستخدم بلاغات
داخل المنتج أو نموذج استئناف الحساب لمطالبات مساحة الأسماء.

## متى تفتح مطالبة

افتح مطالبة مساحة أسماء عندما تعتقد أن على فريق ClawHub مراجعة ما إذا كان ينبغي
حجز مساحة أسماء أو نقلها أو إعادة تسميتها أو إخفاؤها أو عزلها أو إنشاء اسم مستعار لها
أو تغييرها بأي طريقة أخرى بسبب ملكية واقعية.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسسة GitHub أو مشروعك أو شركتك أو مجتمعك
- نطاق حزمة مثل `@example-org/*` ينبغي أن ينشر فقط تحت مالك
  ClawHub المطابق
- معرّف مهارة أو اسم حزمة Plugin يبدو أنه ينتحل صفة مشروع
- علامة تجارية أو علامة مسجلة أو إعادة تسمية مشروع أو نزاع حول تاريخ حزمة
- مالك محذوف أو غير نشط أو لا يمكن الوصول إليه يمنع مالك مساحة الأسماء
  المستحق

إذا كانت القائمة غير آمنة أو ضارة أو مضللة بما يتجاوز نزاع الملكية،
فاتبع أيضًا إرشادات الإشراف أو الأمان ذات الصلة. نموذج مطالبة مساحة الأسماء
مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات.

## قبل أن تقدّم

تأكد أولًا من أنك تنشر باستخدام المالك المطابق لمساحة الأسماء.
بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin`
باسم المالك المطابق `example-org`.

إذا كان بإمكانك إدارة المالك الحالي، فأصلح مساحة الأسماء مباشرة عبر النشر
أو إعادة التسمية أو النقل أو الإخفاء أو حذف المورد المتأثر. استخدم مطالبة
عندما لا يمكنك إدارة المالك الحالي أو عندما يحتاج فريق العمل إلى حل
نزاع.

## الأدلة المطلوب تضمينها

استخدم أدلة عامة غير حساسة. تشمل الأدلة المفيدة:

- تاريخ مؤسسة GitHub أو المستودع أو الإصدار أو المشرف
- وثائق المشروع الرسمية التي تسمي مساحة الأسماء
- دليل على النطاق أو نطاق البريد الإلكتروني الرسمي
- التحكم في نطاق npm أو PyPI أو crates.io أو سجل حزم آخر
- أدلة ملكية العلامة المسجلة أو العلامة التجارية أو المشروع التي يمكن مناقشتها
  علنًا بأمان
- تاريخ مستودع المصدر أو تاريخ الحزمة أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub أو المهارة أو Plugin أو الحزمة أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. ينبغي أن يكون فريق العمل قادرًا على فهم
العلاقة دون الحاجة إلى بيانات اعتماد خاصة أو أسرار.

## ما لا ينبغي تضمينه

لا تضع أسرارًا أو أدلة خاصة في مشكلة GitHub عامة. لا تضمّن:

- رموز API أو مفاتيح التوقيع أو بيانات الاعتماد
- رموز تحدي DNS
- ملفات قانونية أو عقود خاصة
- وثائق هوية شخصية
- رسائل بريد إلكتروني خاصة أو تقارير أمان خاصة أو بيانات عملاء سرية

يسأل نموذج المطالبة عمّا إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع فريق العمل.
استخدم ذلك الخيار بدلًا من نشر مواد حساسة علنًا.

## النتائج المحتملة

بناءً على الأدلة والمخاطر، قد يحجز فريق ClawHub مساحة أسماء،
أو ينقل الملكية، أو يعيد تسمية مورد، أو يخفي أو يعزل قائمة موجودة،
أو يضيف اسمًا مستعارًا أو إعادة توجيه، أو يطلب مزيدًا من الأدلة، أو يرفض الطلب.

لا تضمن مراجعة مساحة الأسماء نقل كل اسم مطابق.
يراعي فريق العمل الأدلة العامة والاستخدام الحالي ومخاطر الأمان وتأثير ذلك على المستخدمين.
