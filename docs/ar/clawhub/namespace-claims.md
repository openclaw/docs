# مطالبات المؤسسة ومساحة الاسم

يستخدم ClawHub معرّفات المالكين، ومعرّفات المؤسسات، والأسماء المختصرة للمهارات، وأسماء حزم Plugin، ونطاقات الحزم كمساحات أسماء عامة. إذا بدت مساحة اسم وكأنها تعود إلى مشروع واقعي، أو علامة تجارية، أو منظومة حزم، أو مؤسسة، لكنها مُطالب بها بالفعل، أو محجوزة، أو مضللة، أو محل نزاع على ClawHub، فاطلب من فريق العمل مراجعتها باستخدام
[نموذج مشكلة مطالبة المؤسسة / مساحة الاسم](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة غير الحساسة. لا تستخدم البلاغات داخل المنتج أو نموذج استئناف الحساب لمطالبات مساحة الاسم.

## متى تفتح مطالبة

افتح مطالبة مساحة اسم عندما تعتقد أن فريق ClawHub يجب أن يراجع ما إذا كان ينبغي حجز مساحة اسم أو نقلها أو إعادة تسميتها أو إخفاؤها أو عزلها أو إضافة اسم مستعار لها أو تغييرها بطريقة أخرى بسبب ملكية واقعية.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسسة GitHub أو مشروعك أو شركتك أو مجتمعك
- نطاق حزمة مثل `@example-org/*` ينبغي ألا ينشر إلا تحت مالك ClawHub المطابق
- اسم مختصر لمهارة أو اسم حزمة Plugin يبدو وكأنه ينتحل صفة مشروع
- نزاع حول علامة تجارية أو علامة مسجلة أو إعادة تسمية مشروع أو سجل حزمة
- مالك محذوف أو غير نشط أو لا يمكن الوصول إليه يمنع مالك مساحة الاسم الشرعي

إذا كانت القائمة غير آمنة أو ضارة أو مضللة بما يتجاوز نزاع الملكية، فاتبع أيضًا إرشادات الإشراف أو الأمان ذات الصلة. نموذج مطالبة مساحة الاسم مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات.

## قبل أن تقدم الطلب

تأكد أولًا من أنك تنشر باستخدام المالك الذي يطابق مساحة الاسم. بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin` بصفتها المالك المطابق `example-org`.

إذا كان بإمكانك إدارة المالك الحالي، فأصلح مساحة الاسم مباشرةً عن طريق نشر المورد المتأثر أو إعادة تسميته أو نقله أو إخفائه أو حذفه. استخدم مطالبة عندما لا يمكنك إدارة المالك الحالي أو عندما يحتاج فريق العمل إلى حل نزاع.

## الأدلة التي يجب تضمينها

استخدم أدلة عامة وغير حساسة. تشمل الإثباتات المفيدة:

- سجل مؤسسة GitHub أو المستودع أو الإصدار أو المشرفين
- وثائق المشروع الرسمية التي تذكر مساحة الاسم
- إثبات النطاق أو نطاق البريد الإلكتروني الرسمي
- التحكم في نطاق npm أو PyPI أو crates.io أو غيرها من سجلات الحزم
- دليل ملكية علامة مسجلة أو علامة تجارية أو مشروع يمكن مناقشته علنًا بأمان
- سجل مستودع المصدر أو سجل الحزمة أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub أو المهارة أو Plugin أو الحزمة أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. ينبغي أن يتمكن فريق العمل من فهم العلاقة دون الحاجة إلى بيانات اعتماد خاصة أو أسرار.

## ما يجب عدم تضمينه

لا تضع أسرارًا أو إثباتات خاصة في مشكلة GitHub عامة. لا تضمّن:

- رموز API أو مفاتيح التوقيع أو بيانات الاعتماد
- رموز تحدي DNS
- ملفات أو عقود قانونية خاصة
- مستندات هوية شخصية
- رسائل بريد إلكتروني خاصة أو تقارير أمنية خاصة أو بيانات عملاء سرية

يسأل نموذج المطالبة عمّا إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع فريق العمل. استخدم هذا الخيار بدلًا من نشر مواد حساسة علنًا.

## النتائج المحتملة

اعتمادًا على الأدلة والمخاطر، قد يحجز فريق ClawHub مساحة اسم، أو ينقل الملكية، أو يعيد تسمية مورد، أو يخفي قائمة موجودة أو يعزلها، أو يضيف اسمًا مستعارًا أو إعادة توجيه، أو يطلب مزيدًا من الإثباتات، أو يرفض الطلب.

لا تضمن مراجعة مساحة الاسم نقل كل اسم مطابق. يوازن فريق العمل بين الأدلة العامة والاستخدام الحالي ومخاطر الأمان وتأثير المستخدمين.
