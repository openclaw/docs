# مطالبات المؤسسة ومساحة الأسماء

يستخدم ClawHub معرّفات المالكين، ومعرّفات المؤسسات، ومعرّفات Skills، وأسماء حزم Plugin، ونطاقات الحزم بوصفها مساحات أسماء عامة. إذا بدت مساحة أسماء ما وكأنها تنتمي إلى مشروع حقيقي، أو علامة تجارية، أو منظومة حزم، أو مؤسسة، لكنها مطالَب بها بالفعل، أو محجوزة، أو مضللة، أو محل نزاع على ClawHub، فاطلب من فريق العمل مراجعتها عبر
[نموذج مشكلة مطالبة المؤسسة / مساحة الأسماء](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة غير الحساسة. لا تستخدم البلاغات داخل المنتج أو نموذج استئناف الحساب لمطالبات مساحات الأسماء.

## متى تفتح مطالبة

افتح مطالبة مساحة أسماء عندما تعتقد أن على فريق ClawHub مراجعة ما إذا كان ينبغي حجز مساحة أسماء، أو نقلها، أو إعادة تسميتها، أو إخفاؤها، أو عزلها، أو إضافة اسم بديل لها، أو تغييرها بطريقة أخرى بسبب ملكية واقعية.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسسة GitHub الخاصة بك، أو مشروعك، أو شركتك، أو مجتمعك
- نطاق حزمة مثل `@example-org/*` ينبغي أن يُنشر فقط تحت مالك ClawHub المطابق
- معرّف Skill أو اسم حزمة Plugin يبدو أنه ينتحل صفة مشروع
- نزاع حول علامة تجارية، أو علامة مسجلة، أو إعادة تسمية مشروع، أو تاريخ حزمة
- مالك محذوف، أو غير نشط، أو غير قابل للوصول يمنع مالك مساحة الأسماء الشرعي

إذا كانت القائمة غير آمنة، أو خبيثة، أو مضللة بما يتجاوز نزاع الملكية، فاتبع أيضًا إرشادات الإشراف أو الأمان ذات الصلة. نموذج مطالبة مساحة الأسماء مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات.

## قبل أن تقدم الطلب

تحقق أولًا من أنك تنشر باستخدام المالك المطابق لمساحة الأسماء. بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin` باسم المالك المطابق `example-org`.

إذا كان بإمكانك إدارة المالك الحالي، فأصلح مساحة الأسماء مباشرة بالنشر، أو إعادة التسمية، أو النقل، أو الإخفاء، أو حذف المورد المتأثر. استخدم المطالبة عندما لا يمكنك إدارة المالك الحالي أو عندما يحتاج فريق العمل إلى حل نزاع.

## الأدلة المطلوب تضمينها

استخدم أدلة عامة وغير حساسة. تشمل الأدلة المفيدة:

- تاريخ مؤسسة GitHub، أو المستودع، أو الإصدار، أو المشرفين
- وثائق المشروع الرسمية التي تذكر مساحة الأسماء
- دليل النطاق أو نطاق البريد الإلكتروني الرسمي
- التحكم في نطاق npm، أو PyPI، أو crates.io، أو سجل حزم آخر
- أدلة ملكية العلامة المسجلة، أو العلامة التجارية، أو المشروع التي يمكن مناقشتها علنًا بأمان
- تاريخ مستودع المصدر، أو تاريخ الحزمة، أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub، أو Skill، أو Plugin، أو الحزمة، أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. ينبغي أن يتمكن فريق العمل من فهم العلاقة دون الحاجة إلى بيانات اعتماد خاصة أو أسرار.

## ما يجب عدم تضمينه

لا تضع أسرارًا أو أدلة خاصة في مشكلة GitHub عامة. لا تضمّن:

- رموز API، أو مفاتيح التوقيع، أو بيانات الاعتماد
- رموز تحدي DNS
- ملفات أو عقود قانونية خاصة
- وثائق هوية شخصية
- رسائل بريد إلكتروني خاصة، أو تقارير أمان خاصة، أو بيانات عملاء سرية

يسأل نموذج المطالبة عما إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع فريق العمل. استخدم هذا الخيار بدلًا من نشر المواد الحساسة علنًا.

## النتائج المحتملة

اعتمادًا على الأدلة والمخاطر، قد يحجز فريق ClawHub مساحة أسماء، أو ينقل الملكية، أو يعيد تسمية مورد، أو يخفي قائمة موجودة أو يعزلها، أو يضيف اسمًا بديلًا أو إعادة توجيه، أو يطلب مزيدًا من الأدلة، أو يرفض الطلب.

لا تضمن مراجعة مساحة الأسماء نقل كل اسم مطابق. يوازن فريق العمل بين الأدلة العامة، والاستخدام الحالي، ومخاطر الأمان، وتأثير ذلك على المستخدمين.
