# مطالبات المؤسسات ومساحات الأسماء

يستخدم ClawHub مقابض المالكين، ومقابض المؤسسات، والمعرّفات المختصرة للـ Skills، وأسماء حزم Plugin، و
نطاقات الحزم كمساحات أسماء عامة. إذا بدا أن مساحة اسم تنتمي إلى
مشروع واقعي أو علامة تجارية أو منظومة حزم أو مؤسسة، لكنها
مطالب بها بالفعل أو محجوزة أو مضللة أو محل نزاع على ClawHub، فاطلب من فريق العمل مراجعتها
باستخدام
[نموذج مشكلة مطالبة مؤسسة / مساحة اسم](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعات الملكية العامة غير الحساسة. لا تستخدم البلاغات داخل المنتج
أو نموذج استئناف الحساب لمطالبات مساحات الأسماء.

## متى تفتح مطالبة

افتح مطالبة مساحة اسم عندما تعتقد أن على فريق ClawHub مراجعة ما إذا كان ينبغي
حجز مساحة اسم أو نقلها أو إعادة تسميتها أو إخفاؤها أو عزلها أو إضافة اسم مستعار لها
أو تغييرها بطريقة أخرى بسبب ملكية واقعية.

تشمل الأمثلة:

- مقبض مؤسسة يطابق مؤسسة GitHub أو مشروعك أو شركتك أو مجتمعك
- نطاق حزمة مثل `@example-org/*` ينبغي أن ينشر فقط تحت مالك ClawHub
  المطابق
- معرّف مختصر لـ Skills أو اسم حزمة Plugin يبدو أنه ينتحل مشروعًا
- نزاع بشأن علامة تجارية أو علامة مسجلة أو إعادة تسمية مشروع أو تاريخ حزمة
- مالك محذوف أو غير نشط أو لا يمكن الوصول إليه يمنع مالك مساحة الاسم الشرعي

إذا كانت القائمة غير آمنة أو ضارة أو مضللة بما يتجاوز نزاع الملكية،
فاتبع أيضًا إرشادات الإشراف أو الأمان ذات الصلة. نموذج مطالبة مساحة الاسم
مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات.

## قبل أن تقدم

تأكد أولًا من أنك تنشر باستخدام المالك الذي يطابق مساحة الاسم.
بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin`
باسم مالك `example-org` المطابق.

إذا كان بإمكانك إدارة المالك الحالي، فأصلح مساحة الاسم مباشرة عن طريق النشر
أو إعادة التسمية أو النقل أو الإخفاء أو حذف المورد المتأثر. استخدم مطالبة
عندما لا يمكنك إدارة المالك الحالي أو عندما يحتاج فريق العمل إلى حل
نزاع.

## الأدلة المطلوب تضمينها

استخدم أدلة عامة غير حساسة. تشمل الأدلة المفيدة:

- تاريخ مؤسسة GitHub أو المستودع أو الإصدار أو المشرف
- وثائق المشروع الرسمية التي تذكر مساحة الاسم
- دليل النطاق أو نطاق البريد الإلكتروني الرسمي
- التحكم في نطاق سجل حزم npm أو PyPI أو crates.io أو غيرها من سجلات الحزم
- دليل ملكية العلامة المسجلة أو العلامة التجارية أو المشروع الذي يمكن مناقشته
  علنًا بأمان
- تاريخ مستودع المصدر أو تاريخ الحزمة أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub أو Skills أو Plugin أو الحزمة أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. ينبغي أن يتمكن فريق العمل من فهم
العلاقة من دون الحاجة إلى بيانات اعتماد خاصة أو أسرار.

## ما لا يجب تضمينه

لا تضع أسرارًا أو أدلة خاصة في مشكلة GitHub عامة. لا تضمّن:

- رموز API أو مفاتيح التوقيع أو بيانات الاعتماد
- رموز تحدي DNS
- ملفات قانونية أو عقود خاصة
- وثائق هوية شخصية
- رسائل بريد إلكتروني خاصة أو تقارير أمنية خاصة أو بيانات عملاء سرية

يسأل نموذج المطالبة ما إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع فريق العمل.
استخدم هذا الخيار بدلًا من نشر مواد حساسة علنًا.

## النتائج المحتملة

اعتمادًا على الأدلة والمخاطر، قد يحجز فريق ClawHub مساحة اسم،
أو ينقل الملكية، أو يعيد تسمية مورد، أو يخفي قائمة موجودة أو يعزلها،
أو يضيف اسمًا مستعارًا أو إعادة توجيه، أو يطلب مزيدًا من الأدلة، أو يرفض الطلب.

لا تضمن مراجعة مساحة الاسم نقل كل اسم مطابق.
يزن فريق العمل الأدلة العامة، والاستخدام الحالي، ومخاطر الأمان، وتأثير المستخدمين.
