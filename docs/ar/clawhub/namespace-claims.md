# مطالبات المؤسسات ومساحات الأسماء

يستخدم ClawHub معرّفات المالكين، ومعرّفات المؤسسات، ومعرّفات المهارات، وأسماء حزم Plugin،
ونطاقات الحزم كمساحات أسماء عامة. إذا بدت مساحة أسماء وكأنها تخص
مشروعًا واقعيًا، أو علامة تجارية، أو منظومة حزم، أو مؤسسة، لكنها
مطالب بها بالفعل أو محجوزة أو مضللة أو محل نزاع على ClawHub، فاطلب من فريق العمل مراجعتها
باستخدام
[نموذج مشكلة مطالبة مؤسسة / مساحة أسماء](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة غير الحساسة. لا تستخدم البلاغات داخل المنتج
أو نموذج استئناف الحساب لمطالبات مساحات الأسماء.

## متى تفتح مطالبة

افتح مطالبة مساحة أسماء عندما تعتقد أن على فريق ClawHub مراجعة ما إذا كان ينبغي
حجز مساحة أسماء، أو نقلها، أو إعادة تسميتها، أو إخفاؤها، أو عزلها، أو إنشاء اسم بديل لها،
أو تغييرها بطريقة أخرى بسبب ملكية واقعية.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسسة GitHub الخاصة بك، أو مشروعك، أو شركتك، أو مجتمعك
- نطاق حزمة مثل `@example-org/*` ينبغي ألا ينشر إلا تحت مالك ClawHub
  المطابق
- معرّف مهارة أو اسم حزمة Plugin يبدو أنه ينتحل هوية مشروع
- نزاع حول علامة تجارية، أو علامة مسجلة، أو إعادة تسمية مشروع، أو سجل حزمة
- مالك محذوف أو غير نشط أو لا يمكن الوصول إليه يمنع مالك مساحة الأسماء الشرعي

إذا كانت القائمة غير آمنة أو ضارة أو مضللة بما يتجاوز نزاع الملكية،
فاتبع أيضًا إرشادات الإشراف أو الأمان ذات الصلة. نموذج مطالبة مساحة الأسماء
مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات.

## قبل أن تقدم

تأكد أولًا من أنك تنشر باستخدام المالك المطابق لمساحة الأسماء.
بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin`
باسم مالك `example-org` المطابق.

إذا كان بإمكانك إدارة المالك الحالي، فأصلح مساحة الأسماء مباشرةً عن طريق النشر،
أو إعادة التسمية، أو النقل، أو الإخفاء، أو حذف المورد المتأثر. استخدم المطالبة
عندما لا يمكنك إدارة المالك الحالي أو عندما يحتاج فريق العمل إلى حل
نزاع.

## الأدلة المطلوب تضمينها

استخدم أدلة عامة غير حساسة. تشمل الأدلة المفيدة:

- سجل مؤسسة GitHub أو المستودع أو الإصدار أو المشرفين
- مستندات المشروع الرسمية التي تذكر مساحة الأسماء
- إثبات النطاق أو نطاق البريد الإلكتروني الرسمي
- التحكم في نطاق سجل حزم مثل npm أو PyPI أو crates.io أو غيرها
- أدلة ملكية العلامة المسجلة أو العلامة التجارية أو المشروع التي يمكن مناقشتها
  علنًا بأمان
- سجل مستودع المصدر، أو سجل الحزمة، أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub أو المهارة أو Plugin أو الحزمة أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. ينبغي أن يتمكن فريق العمل من فهم
العلاقة من دون الحاجة إلى بيانات اعتماد خاصة أو أسرار.

## ما لا ينبغي تضمينه

لا تضع أسرارًا أو أدلة خاصة في مشكلة GitHub عامة. لا تدرج:

- رموز API، أو مفاتيح التوقيع، أو بيانات الاعتماد
- رموز تحدي DNS
- ملفات قانونية أو عقودًا خاصة
- وثائق هوية شخصية
- رسائل بريد إلكتروني خاصة، أو تقارير أمان خاصة، أو بيانات عملاء سرية

يسأل نموذج المطالبة عما إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع فريق العمل.
استخدم هذا الخيار بدلًا من نشر مواد حساسة علنًا.

## النتائج المحتملة

اعتمادًا على الأدلة والمخاطر، قد يحجز فريق ClawHub مساحة أسماء،
أو ينقل الملكية، أو يعيد تسمية مورد، أو يخفي قائمة موجودة أو يعزلها،
أو يضيف اسمًا بديلًا أو إعادة توجيه، أو يطلب مزيدًا من الأدلة، أو يرفض الطلب.

لا تضمن مراجعة مساحة الأسماء نقل كل اسم مطابق.
يزن فريق العمل الأدلة العامة، والاستخدام الحالي، ومخاطر الأمان، وتأثير ذلك في المستخدمين.
