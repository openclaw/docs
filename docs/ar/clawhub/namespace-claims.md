# مطالبات المؤسسات ومساحات الأسماء

يستخدم ClawHub معرّفات المالكين، ومعرّفات المؤسسات، ومعرّفات Skills، وأسماء حزم Plugin، ونطاقات الحزم بوصفها مساحات أسماء عامة. إذا بدت مساحة اسم كأنها تنتمي إلى مشروع واقعي، أو علامة تجارية، أو منظومة حزم، أو مؤسسة، لكنها مطالَب بها مسبقًا، أو محجوزة، أو مضللة، أو متنازع عليها في ClawHub، فاطلب من الفريق مراجعتها باستخدام
[نموذج مشكلة مطالبة المؤسسة / مساحة الاسم](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة غير الحساسة. لا تستخدم البلاغات داخل المنتج أو نموذج استئناف الحساب لمطالبات مساحات الأسماء.

## متى تفتح مطالبة

افتح مطالبة مساحة اسم عندما تعتقد أن على فريق ClawHub مراجعة ما إذا كان ينبغي حجز مساحة اسم، أو نقلها، أو إعادة تسميتها، أو إخفاؤها، أو عزلها، أو ربطها باسم مستعار، أو تغييرها بطريقة أخرى بسبب الملكية الواقعية.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسسة GitHub أو مشروعك أو شركتك أو مجتمعك
- نطاق حزمة مثل `@example-org/*` ينبغي ألا ينشر إلا تحت مالك ClawHub المطابق
- معرّف Skills أو اسم حزمة Plugin يبدو أنه ينتحل صفة مشروع
- نزاع حول علامة تجارية، أو علامة مسجلة، أو إعادة تسمية مشروع، أو تاريخ حزمة
- مالك محذوف أو غير نشط أو لا يمكن الوصول إليه يمنع مالك مساحة الاسم الشرعي

إذا كانت القائمة غير آمنة، أو ضارة، أو مضللة بما يتجاوز نزاع الملكية، فاتبع أيضًا إرشادات الإشراف أو الأمان ذات الصلة. نموذج مطالبة مساحة الاسم مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات.

## قبل التقديم

تأكد أولًا من أنك تنشر باستخدام المالك الذي يطابق مساحة الاسم. بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin` باسم مالك `example-org` المطابق.

إذا كان بإمكانك إدارة المالك الحالي، فأصلح مساحة الاسم مباشرةً عن طريق نشر المورد المتأثر، أو إعادة تسميته، أو نقله، أو إخفائه، أو حذفه. استخدم مطالبة عندما لا يمكنك إدارة المالك الحالي أو عندما يحتاج الفريق إلى حل نزاع.

## الأدلة التي يجب تضمينها

استخدم أدلة عامة غير حساسة. تشمل الأدلة المفيدة:

- تاريخ مؤسسة GitHub أو المستودع أو الإصدار أو المشرفين
- وثائق المشروع الرسمية التي تذكر مساحة الاسم
- إثبات النطاق أو نطاق البريد الإلكتروني الرسمي
- التحكم في نطاق سجل الحزم في npm أو PyPI أو crates.io أو سجل حزم آخر
- أدلة ملكية العلامة المسجلة أو العلامة التجارية أو المشروع التي يمكن مناقشتها علنًا بأمان
- تاريخ مستودع المصدر، أو تاريخ الحزمة، أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub أو Skills أو Plugin أو الحزمة أو المشكلة المتنازع عليها

اشرح ما يثبته كل رابط. ينبغي أن يتمكن الفريق من فهم العلاقة دون الحاجة إلى بيانات اعتماد خاصة أو أسرار.

## ما لا ينبغي تضمينه

لا تضع أسرارًا أو أدلة خاصة في مشكلة GitHub عامة. لا تضمّن:

- رموز API، أو مفاتيح التوقيع، أو بيانات الاعتماد
- رموز تحدي DNS
- ملفات أو عقود قانونية خاصة
- وثائق هوية شخصية
- رسائل بريد إلكتروني خاصة، أو تقارير أمان خاصة، أو بيانات عملاء سرية

يسأل نموذج المطالبة عما إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع الفريق. استخدم ذلك الخيار بدلًا من نشر مواد حساسة علنًا.

## النتائج المحتملة

اعتمادًا على الأدلة والمخاطر، قد يحجز فريق ClawHub مساحة اسم، أو ينقل الملكية، أو يعيد تسمية مورد، أو يخفي قائمة موجودة أو يعزلها، أو يضيف اسمًا مستعارًا أو إعادة توجيه، أو يطلب مزيدًا من الأدلة، أو يرفض الطلب.

لا تضمن مراجعة مساحة الاسم نقل كل اسم مطابق. يوازن الفريق بين الأدلة العامة، والاستخدام الحالي، ومخاطر الأمان، وتأثير ذلك على المستخدمين.
