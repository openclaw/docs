# مطالبات المؤسسات ومساحات الأسماء

يستخدم ClawHub معرّفات المالكين، ومعرّفات المؤسسات، ومعرّفات المهارات، وأسماء حزم Plugin،
ونطاقات الحزم كمساحات أسماء عامة. إذا بدت مساحة اسم وكأنها تنتمي إلى
مشروع حقيقي، أو علامة تجارية، أو منظومة حزم، أو مؤسسة، لكنها سبق أن
طُولب بها، أو حُجزت، أو كانت مضللة، أو محل نزاع على ClawHub، فاطلب من الفريق مراجعتها
باستخدام
[نموذج مشكلة مطالبة مؤسسة / مساحة اسم](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة غير الحساسة. لا تستخدم البلاغات داخل المنتج
أو نموذج استئناف الحساب لمطالبات مساحات الأسماء.

## متى تفتح مطالبة

افتح مطالبة مساحة اسم عندما تعتقد أن فريق ClawHub يجب أن يراجع ما إذا كان ينبغي
حجز مساحة اسم، أو نقلها، أو إعادة تسميتها، أو إخفاؤها، أو عزلها، أو إنشاء اسم مستعار لها،
أو تغييرها بأي طريقة أخرى بسبب ملكية حقيقية.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسسة GitHub الخاصة بك، أو مشروعك، أو شركتك، أو مجتمعك
- نطاق حزمة مثل `@example-org/*` ينبغي أن ينشر فقط تحت مالك ClawHub
  المطابق
- معرّف مهارة أو اسم حزمة Plugin يبدو أنه ينتحل مشروعًا
- نزاع حول علامة تجارية، أو علامة مسجلة، أو إعادة تسمية مشروع، أو تاريخ حزمة
- مالك محذوف، أو غير نشط، أو لا يمكن الوصول إليه، يمنع مالك مساحة الاسم الشرعي

إذا كانت القائمة غير آمنة، أو خبيثة، أو مضللة بما يتجاوز نزاع الملكية،
فاتبع أيضًا إرشادات الإشراف أو الأمان ذات الصلة. نموذج مطالبة مساحة الاسم
مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات.

## قبل أن تقدم

تأكد أولًا من أنك تنشر بالمالك الذي يطابق مساحة الاسم.
بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin`
بصفتها مالك `example-org` المطابق.

إذا كان بإمكانك إدارة المالك الحالي، فأصلح مساحة الاسم مباشرة عبر نشر المورد المتأثر،
أو إعادة تسميته، أو نقله، أو إخفائه، أو حذفه. استخدم مطالبة
عندما لا يمكنك إدارة المالك الحالي أو عندما يحتاج الفريق إلى حل
نزاع.

## الأدلة التي يجب تضمينها

استخدم أدلة عامة غير حساسة. تشمل الأدلة المفيدة:

- تاريخ مؤسسة GitHub، أو المستودع، أو الإصدار، أو المشرفين
- وثائق المشروع الرسمية التي تذكر مساحة الاسم
- دليل النطاق أو نطاق البريد الإلكتروني الرسمي
- التحكم في نطاق npm، أو PyPI، أو crates.io، أو سجل حزم آخر
- دليل ملكية علامة مسجلة، أو علامة تجارية، أو مشروع يكون آمنًا للنقاش
  علنًا
- تاريخ مستودع المصدر، أو تاريخ الحزمة، أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub، أو المهارة، أو Plugin، أو الحزمة، أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. ينبغي أن يكون الفريق قادرًا على فهم
العلاقة دون الحاجة إلى بيانات اعتماد خاصة أو أسرار.

## ما يجب عدم تضمينه

لا تضع أسرارًا أو أدلة خاصة في مشكلة GitHub عامة. لا تضمّن:

- رموز API، أو مفاتيح التوقيع، أو بيانات الاعتماد
- رموز تحدي DNS
- ملفات قانونية أو عقود خاصة
- مستندات هوية شخصية
- رسائل بريد إلكتروني خاصة، أو تقارير أمنية خاصة، أو بيانات عملاء سرية

يسأل نموذج المطالبة ما إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع الفريق.
استخدم هذا الخيار بدلًا من نشر مواد حساسة علنًا.

## النتائج المحتملة

اعتمادًا على الأدلة والمخاطر، قد يحجز فريق ClawHub مساحة اسم،
أو ينقل الملكية، أو يعيد تسمية مورد، أو يخفي أو يعزل قائمة قائمة،
أو يضيف اسمًا مستعارًا أو إعادة توجيه، أو يطلب المزيد من الأدلة، أو يرفض الطلب.

لا تضمن مراجعة مساحة الاسم نقل كل اسم مطابق.
يراعي الفريق الأدلة العامة، والاستخدام الحالي، ومخاطر الأمان، وتأثير ذلك على المستخدمين.
