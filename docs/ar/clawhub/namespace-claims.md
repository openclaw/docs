# مطالبات المؤسسة ومساحة الاسم

تستخدم ClawHub معرّفات المالكين، ومعرّفات المؤسسات، ومعرّفات Skills المختصرة، وأسماء حزم Plugin، و
نطاقات الحزم كمساحات أسماء عامة. إذا بدت مساحة اسم كأنها تنتمي إلى
مشروع حقيقي، أو علامة تجارية، أو منظومة حزم، أو مؤسسة ولكنها
مطالب بها بالفعل، أو محجوزة، أو مضللة، أو محل نزاع على ClawHub، فاطلب من الموظفين مراجعتها
باستخدام
[نموذج مشكلة مطالبة مؤسسة / مساحة اسم](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة غير الحساسة. لا تستخدم البلاغات داخل المنتج
أو نموذج استئناف الحساب لمطالبات مساحات الأسماء.

## متى تفتح مطالبة

افتح مطالبة مساحة اسم عندما تعتقد أن موظفي ClawHub يجب أن يراجعوا ما إذا كان ينبغي
حجز مساحة اسم، أو نقلها، أو إعادة تسميتها، أو إخفاؤها، أو عزلها، أو جعلها اسمًا مستعارًا،
أو تغييرها بطريقة أخرى بسبب ملكية حقيقية.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسسة GitHub الخاصة بك، أو مشروعك، أو شركتك، أو مجتمعك
- نطاق حزمة مثل `@example-org/*` يجب أن ينشر فقط تحت مالك
  ClawHub المطابق
- معرّف Skills مختصر أو اسم حزمة Plugin يبدو أنه ينتحل صفة مشروع
- نزاع حول علامة تجارية، أو علامة مسجلة، أو إعادة تسمية مشروع، أو تاريخ حزمة
- مالك محذوف، أو غير نشط، أو لا يمكن الوصول إليه يمنع مالك مساحة الاسم الشرعي

إذا كانت القائمة غير آمنة، أو خبيثة، أو مضللة بما يتجاوز نزاع الملكية،
فاتبع أيضًا إرشادات الإشراف أو الأمان ذات الصلة. نموذج مطالبة مساحة الاسم
مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات.

## قبل أن تقدم

تأكد أولًا أنك تنشر باستخدام المالك المطابق لمساحة الاسم.
بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin`
باسم مالك `example-org` المطابق.

إذا كان بإمكانك إدارة المالك الحالي، فأصلح مساحة الاسم مباشرة بالنشر،
أو إعادة التسمية، أو النقل، أو الإخفاء، أو حذف المورد المتأثر. استخدم مطالبة
عندما لا يمكنك إدارة المالك الحالي أو عندما يحتاج الموظفون إلى حل
نزاع.

## الأدلة التي يجب تضمينها

استخدم أدلة عامة غير حساسة. تشمل الأدلة المفيدة:

- تاريخ مؤسسة GitHub، أو المستودع، أو الإصدار، أو المشرفين
- وثائق المشروع الرسمية التي تسمي مساحة الاسم
- دليل النطاق أو نطاق البريد الإلكتروني الرسمي
- التحكم في نطاق npm، أو PyPI، أو crates.io، أو سجل حزم آخر
- دليل ملكية العلامة المسجلة، أو العلامة التجارية، أو المشروع مما يمكن مناقشته
  علنًا بأمان
- تاريخ مستودع المصدر، أو تاريخ الحزمة، أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub، أو Skills، أو Plugin، أو الحزمة، أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. يجب أن يتمكن الموظفون من فهم
العلاقة دون الحاجة إلى بيانات اعتماد خاصة أو أسرار.

## ما لا يجب تضمينه

لا تضع أسرارًا أو أدلة خاصة في مشكلة GitHub عامة. لا تضمّن:

- رموز API، أو مفاتيح التوقيع، أو بيانات الاعتماد
- رموز تحدي DNS
- ملفات أو عقود قانونية خاصة
- مستندات هوية شخصية
- رسائل بريد إلكتروني خاصة، أو تقارير أمنية خاصة، أو بيانات عملاء سرية

يسألك نموذج المطالبة عما إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع الموظفين.
استخدم هذا الخيار بدلًا من نشر مواد حساسة علنًا.

## النتائج المحتملة

اعتمادًا على الأدلة والمخاطر، قد يحجز موظفو ClawHub مساحة اسم،
أو ينقلوا الملكية، أو يعيدوا تسمية مورد، أو يخفوا أو يعزلوا قائمة موجودة،
أو يضيفوا اسمًا مستعارًا أو إعادة توجيه، أو يطلبوا مزيدًا من الأدلة، أو يرفضوا الطلب.

لا تضمن مراجعة مساحة الاسم نقل كل اسم مطابق.
يزن الموظفون الأدلة العامة، والاستخدام الحالي، والمخاطر الأمنية، وتأثير المستخدم.
