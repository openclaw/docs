# مطالبات المؤسسات والنطاقات الاسمية

يستخدم ClawHub معرّفات المالكين، ومعرّفات المؤسسات، ومعرّفات المهارات، وأسماء حزم Plugin،
ونطاقات الحزم بوصفها نطاقات اسمية عامة. إذا بدا أن نطاقًا اسميًا ينتمي إلى
مشروع واقعي، أو علامة تجارية، أو منظومة حزم، أو مؤسسة لكنه مُطالب به مسبقًا،
أو محجوز، أو مضلّل، أو محل نزاع على ClawHub، فاطلب من فريق العمل مراجعته
باستخدام
[نموذج مشكلة مطالبة مؤسسة / نطاق اسمي](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة غير الحساسة. لا تستخدم البلاغات داخل المنتج
أو نموذج استئناف الحساب لمطالبات النطاقات الاسمية.

## متى تفتح مطالبة

افتح مطالبة نطاق اسمي عندما تعتقد أن على فريق ClawHub مراجعة ما إذا كان ينبغي
حجز نطاق اسمي، أو نقل ملكيته، أو إعادة تسميته، أو إخفاؤه، أو عزله، أو جعله اسمًا بديلًا،
أو تغييره بطريقة أخرى بسبب ملكية واقعية.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسسة GitHub أو مشروعك أو شركتك أو مجتمعك
- نطاق حزمة مثل `@example-org/*` ينبغي ألا ينشر إلا تحت مالك ClawHub
  المطابق
- معرّف مهارة أو اسم حزمة Plugin يبدو أنه ينتحل مشروعًا
- نزاع حول علامة تجارية، أو علامة مسجلة، أو إعادة تسمية مشروع، أو سجل حزمة
- مالك محذوف أو غير نشط أو يتعذر الوصول إليه يمنع مالك النطاق الاسمي الشرعي

إذا كانت القائمة غير آمنة، أو خبيثة، أو مضللة بما يتجاوز نزاع الملكية،
فاتبع أيضًا إرشادات الإشراف أو الأمان ذات الصلة. نموذج مطالبة النطاق الاسمي
مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات.

## قبل التقديم

تأكد أولًا من أنك تنشر باستخدام المالك الذي يطابق النطاق الاسمي.
بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin`
بصفتها مالك `example-org` المطابق.

إذا كان بإمكانك إدارة المالك الحالي، فأصلح النطاق الاسمي مباشرة عبر النشر،
أو إعادة التسمية، أو النقل، أو الإخفاء، أو حذف المورد المتأثر. استخدم مطالبة
عندما لا يمكنك إدارة المالك الحالي أو عندما يحتاج فريق العمل إلى حل
نزاع.

## الأدلة التي يجب تضمينها

استخدم أدلة عامة وغير حساسة. تشمل الأدلة المفيدة:

- سجل مؤسسة GitHub أو المستودع أو الإصدار أو المشرفين
- وثائق المشروع الرسمية التي تذكر النطاق الاسمي
- دليل النطاق أو نطاق البريد الإلكتروني الرسمي
- التحكم في نطاق npm أو PyPI أو crates.io أو سجل حزم آخر
- أدلة ملكية علامة مسجلة أو علامة تجارية أو مشروع يمكن مناقشتها بأمان
  علنًا
- سجل مستودع المصدر، أو سجل الحزمة، أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub أو المهارة أو Plugin أو الحزمة أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. يجب أن يتمكن فريق العمل من فهم
العلاقة دون الحاجة إلى بيانات اعتماد أو أسرار خاصة.

## ما لا يجب تضمينه

لا تضع أسرارًا أو أدلة خاصة في مشكلة GitHub عامة. لا تُضمّن:

- رموز API، أو مفاتيح التوقيع، أو بيانات الاعتماد
- رموز تحدي DNS
- ملفات أو عقود قانونية خاصة
- وثائق هوية شخصية
- رسائل بريد إلكتروني خاصة، أو تقارير أمان خاصة، أو بيانات عملاء سرية

يسأل نموذج المطالبة عما إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع فريق العمل.
استخدم ذلك الخيار بدلًا من نشر المواد الحساسة علنًا.

## النتائج المحتملة

بناءً على الأدلة والمخاطر، قد يحجز فريق ClawHub نطاقًا اسميًا،
أو ينقل الملكية، أو يعيد تسمية مورد، أو يخفي أو يعزل قائمة موجودة،
أو يضيف اسمًا بديلًا أو إعادة توجيه، أو يطلب مزيدًا من الأدلة، أو يرفض الطلب.

لا تضمن مراجعة النطاق الاسمي نقل كل اسم مطابق.
يراعي فريق العمل الأدلة العامة، والاستخدام الحالي، ومخاطر الأمان، وتأثير ذلك على المستخدمين.
