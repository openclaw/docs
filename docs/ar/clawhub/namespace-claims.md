# مطالبات المؤسسات ومساحات الأسماء

تستخدم ClawHub معرّفات المالكين ومعرّفات المؤسسات والأسماء المختصرة للـ Skills وأسماء حزم الـ Plugin ونطاقات الحزم بوصفها مساحات أسماء عامة. إذا بدت مساحة أسماء وكأنها تخص مشروعًا فعليًا أو علامة تجارية أو منظومة حزم أو مؤسسة، لكنها مطالَب بها بالفعل أو محجوزة أو مضللة أو محل نزاع على ClawHub، فاطلب من فريق العمل مراجعتها باستخدام
[نموذج مشكلة المطالبة بمؤسسة / مساحة أسماء](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة غير الحساسة. لا تستخدم البلاغات داخل المنتج أو نموذج الاعتراض على الحساب لمطالبات مساحات الأسماء.

## متى تفتح مطالبة

افتح مطالبة بمساحة أسماء عندما تعتقد أن على فريق ClawHub مراجعة ما إذا كان ينبغي حجز مساحة أسماء أو نقلها أو إعادة تسميتها أو إخفاؤها أو عزلها أو إنشاء اسم بديل لها أو تغييرها بطريقة أخرى بسبب ملكية فعلية.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسستك أو مشروعك أو شركتك أو مجتمعك على GitHub
- نطاق حزمة مثل `@example-org/*` ينبغي ألا ينشر تحته سوى مالك ClawHub المطابق
- اسم مختصر لـ Skill أو اسم حزمة Plugin يبدو وكأنه ينتحل صفة مشروع
- نزاع بشأن علامة تجارية أو مشروع أو إعادة تسمية مشروع أو سجل حزمة
- مالك محذوف أو غير نشط أو يتعذر الوصول إليه يمنع المالك الشرعي لمساحة الأسماء

إذا كان الإدراج غير آمن أو ضارًا أو مضللًا بما يتجاوز نزاع الملكية، فاتبع أيضًا إرشادات الإشراف أو الأمان ذات الصلة. نموذج المطالبة بمساحة الأسماء مخصص لمراجعة الملكية، وليس للإفصاح العاجل عن الثغرات الأمنية.

## قبل تقديم المطالبة

تأكد أولًا من أنك تنشر باستخدام المالك المطابق لمساحة الأسماء. بالنسبة إلى حزم الـ Plugin، يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin` بواسطة المالك المطابق `example-org`.

إذا كان بإمكانك إدارة المالك الحالي، فأصلح مساحة الأسماء مباشرةً من خلال نشر المورد المتأثر أو إعادة تسميته أو نقله أو إخفائه أو حذفه. استخدم المطالبة عندما يتعذر عليك إدارة المالك الحالي أو عندما يحتاج فريق العمل إلى حل نزاع.

## الأدلة المطلوب تضمينها

استخدم أدلة عامة وغير حساسة. تشمل الأدلة المفيدة:

- سجل مؤسسة GitHub أو المستودع أو الإصدارات أو المشرفين
- وثائق المشروع الرسمية التي تذكر مساحة الأسماء
- إثبات النطاق أو نطاق البريد الإلكتروني الرسمي
- التحكم في نطاق على npm أو PyPI أو crates.io أو سجل حزم آخر
- أدلة ملكية العلامة التجارية أو المشروع التي يمكن مناقشتها علنًا بأمان
- سجل مستودع المصدر أو سجل الحزمة أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub أو Skill أو Plugin أو الحزمة أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. ينبغي أن يتمكن فريق العمل من فهم العلاقة دون الحاجة إلى بيانات اعتماد خاصة أو أسرار.

## ما يجب عدم تضمينه

لا تضع أسرارًا أو أدلة خاصة في مشكلة عامة على GitHub. لا تُضمّن:

- رموز API المميزة أو مفاتيح التوقيع أو بيانات الاعتماد
- رموز تحدي DNS المميزة
- ملفات أو عقود قانونية خاصة
- مستندات الهوية الشخصية
- رسائل بريد إلكتروني خاصة أو تقارير أمان خاصة أو بيانات عملاء سرية

يسأل نموذج المطالبة عما إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع فريق العمل. استخدم هذا الخيار بدلًا من نشر المواد الحساسة علنًا.

## النتائج المحتملة

استنادًا إلى الأدلة والمخاطر، قد يحجز فريق ClawHub مساحة أسماء أو ينقل ملكيتها أو يعيد تسمية مورد أو يخفي إدراجًا قائمًا أو يعزله أو يضيف اسمًا بديلًا أو إعادة توجيه أو يطلب مزيدًا من الأدلة أو يرفض الطلب.

لا تضمن مراجعة مساحة الأسماء نقل كل اسم مطابق. يوازن فريق العمل بين الأدلة العامة والاستخدام الحالي والمخاطر الأمنية والتأثير في المستخدمين.
