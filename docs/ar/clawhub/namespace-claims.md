# مطالبات المؤسسات ومساحات الأسماء

يستخدم ClawHub معرّفات المالكين، ومعرّفات المؤسسات، وأسماء Skills المختصرة، وأسماء حزم Plugin،
ونطاقات الحزم بوصفها مساحات أسماء عامة. إذا بدا أن مساحة اسم تخص
مشروعًا فعليًا أو علامة تجارية أو منظومة حزم أو مؤسسة، لكنها
مطالب بها أو محجوزة أو مضللة أو محل نزاع بالفعل على ClawHub، فاطلب من فريق العمل مراجعتها
باستخدام
[نموذج مشكلة المطالبة بمؤسسة / مساحة اسم](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة وغير الحساسة. لا تستخدم البلاغات
داخل المنتج أو نموذج استئناف الحساب لمطالبات مساحات الأسماء.

## متى تُفتح مطالبة

افتح مطالبة بمساحة اسم عندما ترى أن على فريق ClawHub مراجعة ما إذا كان ينبغي
حجز مساحة اسم أو نقلها أو إعادة تسميتها أو إخفاؤها أو عزلها أو إنشاء اسم مستعار لها
أو تغييرها بطريقة أخرى بسبب ملكيتها الفعلية.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسستك أو مشروعك أو شركتك أو مجتمعك على GitHub
- نطاق حزمة مثل `@example-org/*` ينبغي ألا ينشر إلا تحت
  مالك ClawHub المطابق
- اسم مختصر لـ Skills أو اسم حزمة Plugin يبدو أنه ينتحل صفة مشروع
- نزاع بشأن علامة تجارية أو علامة مسجلة أو إعادة تسمية مشروع أو سجل حزمة
- مالك محذوف أو غير نشط أو يتعذر الوصول إليه يمنع المالك الشرعي لمساحة
  الاسم

إذا كان الإدراج غير آمن أو ضارًا أو مضللًا بما يتجاوز نزاع الملكية،
فاتبع أيضًا إرشادات الإشراف أو الأمان ذات الصلة. نموذج المطالبة
بمساحة الاسم مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات الأمنية.

## قبل تقديم المطالبة

تأكد أولًا من أنك تنشر باستخدام المالك المطابق لمساحة الاسم.
بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin`
باسم المالك المطابق `example-org`.

إذا كان بإمكانك إدارة المالك الحالي، فأصلح مساحة الاسم مباشرة بالنشر
أو إعادة التسمية أو النقل أو الإخفاء أو حذف المورد المتأثر. استخدم المطالبة
عندما لا يمكنك إدارة المالك الحالي أو عندما يحتاج فريق العمل إلى حل
نزاع.

## الأدلة المطلوب تضمينها

استخدم أدلة عامة وغير حساسة. تشمل الإثباتات المفيدة:

- سجل مؤسسة GitHub أو المستودع أو الإصدارات أو المشرفين
- وثائق المشروع الرسمية التي تذكر مساحة الاسم
- إثبات النطاق أو نطاق البريد الإلكتروني الرسمي
- التحكم في نطاق npm أو PyPI أو crates.io أو سجل حزم آخر
- دليل ملكية العلامة المسجلة أو العلامة التجارية أو المشروع الذي يمكن مناقشته بأمان
  علنًا
- سجل مستودع المصدر أو سجل الحزمة أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub أو Skills أو Plugin أو الحزمة أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. ينبغي أن يتمكن فريق العمل من فهم
العلاقة دون الحاجة إلى بيانات اعتماد خاصة أو أسرار.

## ما يجب عدم تضمينه

لا تضع أسرارًا أو أدلة خاصة في مشكلة عامة على GitHub. لا تُضمّن:

- رموز API المميزة أو مفاتيح التوقيع أو بيانات الاعتماد
- رموز تحدي DNS المميزة
- ملفات قانونية أو عقود خاصة
- وثائق هوية شخصية
- رسائل بريد إلكتروني خاصة أو تقارير أمنية خاصة أو بيانات عملاء سرية

يسأل نموذج المطالبة عما إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع فريق العمل.
استخدم هذا الخيار بدلًا من نشر المواد الحساسة علنًا.

## النتائج المحتملة

اعتمادًا على الأدلة والمخاطر، قد يحجز فريق ClawHub مساحة اسم،
أو ينقل الملكية، أو يعيد تسمية مورد، أو يخفي إدراجًا موجودًا أو يعزله،
أو يضيف اسمًا مستعارًا أو إعادة توجيه، أو يطلب مزيدًا من الأدلة، أو يرفض الطلب.

لا تضمن مراجعة مساحة الاسم نقل كل اسم مطابق.
يوازن فريق العمل بين الأدلة العامة والاستخدام الحالي والمخاطر الأمنية وتأثير ذلك في المستخدمين.
