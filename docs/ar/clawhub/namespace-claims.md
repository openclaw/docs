# مطالبات المؤسسات ومساحات الأسماء

يستخدم ClawHub معرّفات المالكين، ومعرّفات المؤسسات، ومعرّفات Skills، وأسماء حزم Plugin، و
نطاقات الحزم كمساحات أسماء عامة. إذا بدت مساحة اسم وكأنها تنتمي إلى
مشروع حقيقي، أو علامة تجارية، أو منظومة حزم، أو مؤسسة، لكنها مطالَب بها مسبقًا،
أو محجوزة، أو مضللة، أو محل نزاع على ClawHub، فاطلب من فريق العمل مراجعتها
باستخدام
[نموذج مشكلة مطالبة المؤسسة / مساحة الاسم](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة غير الحساسة. لا تستخدم البلاغات داخل المنتج
أو نموذج استئناف الحساب لمطالبات مساحات الأسماء.

## متى تفتح مطالبة

افتح مطالبة مساحة اسم عندما تعتقد أن على فريق ClawHub مراجعة ما إذا كان ينبغي
حجز مساحة اسم، أو نقلها، أو إعادة تسميتها، أو إخفاؤها، أو عزلها، أو إنشاء اسم مستعار لها،
أو تغييرها بطريقة أخرى بسبب ملكية حقيقية.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسسة GitHub أو مشروعك أو شركتك أو مجتمعك
- نطاق حزمة مثل `@example-org/*` ينبغي ألا ينشر إلا تحت
  مالك ClawHub المطابق
- معرّف Skill أو اسم حزمة Plugin يبدو أنه ينتحل مشروعًا
- نزاع حول علامة تجارية، أو علامة مسجلة، أو إعادة تسمية مشروع، أو سجل حزمة
- مالك محذوف، أو غير نشط، أو غير قابل للوصول يمنع مالك مساحة الاسم الشرعي

إذا كانت القائمة غير آمنة، أو خبيثة، أو مضللة بما يتجاوز نزاع الملكية،
فاتبع أيضًا إرشادات الإشراف أو الأمان ذات الصلة. نموذج مطالبة مساحة الاسم
مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات.

## قبل أن تقدّم

تأكد أولًا من أنك تنشر باستخدام المالك المطابق لمساحة الاسم.
بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاقات مثل `@example-org/example-plugin`
باسم المالك المطابق `example-org`.

إذا كان بإمكانك إدارة المالك الحالي، فأصلح مساحة الاسم مباشرة عبر النشر،
أو إعادة التسمية، أو النقل، أو الإخفاء، أو حذف المورد المتأثر. استخدم مطالبة
عندما لا يمكنك إدارة المالك الحالي أو عندما يحتاج فريق العمل إلى حل
نزاع.

## الأدلة التي يجب تضمينها

استخدم أدلة عامة وغير حساسة. تشمل الأدلة المفيدة:

- سجل مؤسسة GitHub أو المستودع أو الإصدار أو المشرفين
- وثائق المشروع الرسمية التي تذكر مساحة الاسم
- إثبات النطاق أو نطاق البريد الإلكتروني الرسمي
- التحكم في نطاقات npm أو PyPI أو crates.io أو سجلات الحزم الأخرى
- أدلة ملكية العلامة المسجلة أو العلامة التجارية أو المشروع التي يمكن مناقشتها
  علنًا بأمان
- سجل مستودع المصدر، أو سجل الحزمة، أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub أو Skill أو Plugin أو الحزمة أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. ينبغي أن يتمكن فريق العمل من فهم
العلاقة دون الحاجة إلى بيانات اعتماد خاصة أو أسرار.

## ما لا يجب تضمينه

لا تضع أسرارًا أو أدلة خاصة في مشكلة GitHub عامة. لا تضمّن:

- رموز API، أو مفاتيح التوقيع، أو بيانات الاعتماد
- رموز تحدي DNS
- ملفات أو عقود قانونية خاصة
- وثائق هوية شخصية
- رسائل بريد إلكتروني خاصة، أو تقارير أمان خاصة، أو بيانات عملاء سرية

يسأل نموذج المطالبة عما إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع فريق العمل.
استخدم ذلك الخيار بدلًا من نشر مواد حساسة علنًا.

## النتائج المحتملة

اعتمادًا على الأدلة والمخاطر، قد يحجز فريق ClawHub مساحة اسم،
أو ينقل الملكية، أو يعيد تسمية مورد، أو يخفي قائمة موجودة أو يعزلها،
أو يضيف اسمًا مستعارًا أو إعادة توجيه، أو يطلب المزيد من الأدلة، أو يرفض الطلب.

لا تضمن مراجعة مساحة الاسم نقل كل اسم مطابق.
يرجّح فريق العمل الأدلة العامة، والاستخدام الحالي، ومخاطر الأمان، وتأثير المستخدم.
