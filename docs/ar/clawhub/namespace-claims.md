# مطالبات المؤسسات والنطاقات الاسمية

يستخدم ClawHub معرّفات المالكين، ومعرّفات المؤسسات، ومعرّفات Skills، وأسماء حزم Plugin،
ونطاقات الحزم بوصفها نطاقات اسمية عامة. إذا بدا أن نطاقاً اسمياً ينتمي إلى
مشروع واقعي أو علامة تجارية أو منظومة حزم أو مؤسسة، لكنه مطالب به مسبقاً أو
محجوز أو مضلل أو محل نزاع على ClawHub، فاطلب من فريق العمل مراجعته باستخدام
[نموذج مشكلة مطالبة المؤسسة / النطاق الاسمي](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة غير الحساسة. لا تستخدم البلاغات داخل المنتج
أو نموذج استئناف الحساب لمطالبات النطاقات الاسمية.

## متى تفتح مطالبة

افتح مطالبة نطاق اسمي عندما تعتقد أن على فريق ClawHub مراجعة ما إذا كان يجب
حجز نطاق اسمي أو نقل ملكيته أو إعادة تسميته أو إخفاؤه أو عزله أو منحه اسماً
مستعاراً أو تغييره بطريقة أخرى بسبب ملكية واقعية.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسسة GitHub أو مشروعك أو شركتك أو مجتمعك
- نطاق حزمة مثل `@example-org/*` يجب أن ينشر فقط ضمن مالك ClawHub
  المطابق
- معرّف Skill أو اسم حزمة Plugin يبدو أنه ينتحل مشروعاً
- نزاع حول علامة تجارية أو علامة مسجلة أو إعادة تسمية مشروع أو تاريخ حزمة
- مالك محذوف أو غير نشط أو لا يمكن الوصول إليه يمنع مالك النطاق الاسمي
  الشرعي

إذا كان الإدراج غير آمن أو خبيثاً أو مضللاً بما يتجاوز نزاع الملكية،
فاتبع أيضاً إرشادات الإشراف أو الأمان ذات الصلة. نموذج مطالبة النطاق الاسمي
مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات.

## قبل التقديم

تأكد أولاً من أنك تنشر باستخدام المالك الذي يطابق النطاق الاسمي.
بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin`
باسم مالك `example-org` المطابق.

إذا كان بإمكانك إدارة المالك الحالي، فأصلح النطاق الاسمي مباشرة بالنشر أو
إعادة التسمية أو النقل أو الإخفاء أو حذف المورد المتأثر. استخدم مطالبة
عندما لا يمكنك إدارة المالك الحالي أو عندما يحتاج فريق العمل إلى حل
نزاع.

## الأدلة المطلوب تضمينها

استخدم أدلة عامة غير حساسة. تشمل الأدلة المفيدة:

- تاريخ مؤسسة GitHub أو المستودع أو الإصدار أو المشرفين
- وثائق المشروع الرسمية التي تذكر النطاق الاسمي
- دليل النطاق أو نطاق البريد الإلكتروني الرسمي
- التحكم في نطاق npm أو PyPI أو crates.io أو سجل حزم آخر
- دليل ملكية علامة مسجلة أو علامة تجارية أو مشروع يكون آمناً للمناقشة
  علناً
- تاريخ مستودع المصدر أو تاريخ الحزمة أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub أو Skill أو Plugin أو الحزمة أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. يجب أن يكون فريق العمل قادراً على فهم
العلاقة دون الحاجة إلى بيانات اعتماد خاصة أو أسرار.

## ما يجب عدم تضمينه

لا تضع أسراراً أو أدلة خاصة في مشكلة GitHub عامة. لا تضمّن:

- رموز API أو مفاتيح التوقيع أو بيانات الاعتماد
- رموز تحدي DNS
- ملفات قانونية أو عقوداً خاصة
- وثائق هوية شخصية
- رسائل بريد إلكتروني خاصة أو تقارير أمان خاصة أو بيانات عملاء سرية

يسأل نموذج المطالبة عمّا إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع فريق العمل.
استخدم ذلك الخيار بدلاً من نشر مواد حساسة علناً.

## النتائج المحتملة

اعتماداً على الأدلة والمخاطر، قد يحجز فريق ClawHub نطاقاً اسمياً،
أو ينقل الملكية، أو يعيد تسمية مورد، أو يخفي إدراجاً قائماً أو يعزله،
أو يضيف اسماً مستعاراً أو إعادة توجيه، أو يطلب مزيداً من الأدلة، أو يرفض الطلب.

لا تضمن مراجعة النطاق الاسمي نقل كل اسم مطابق.
يزن فريق العمل الأدلة العامة والاستخدام الحالي ومخاطر الأمان وتأثير المستخدمين.
