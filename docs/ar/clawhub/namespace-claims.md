# مطالبات المؤسسات ومساحات الأسماء

يستخدم ClawHub معرّفات المالكين، ومعرّفات المؤسسات، والأسماء المختصرة لـ Skills، وأسماء حزم Plugin، ونطاقات الحزم كمساحات أسماء عامة. إذا بدت مساحة اسم ما كأنها تنتمي إلى مشروع أو علامة تجارية أو منظومة حزم أو مؤسسة في العالم الحقيقي، لكنها مطالَب بها مسبقًا أو محجوزة أو مضللة أو محل نزاع على ClawHub، فاطلب من فريق العمل مراجعتها عبر
[نموذج مشكلة مطالبة المؤسسة / مساحة الاسم](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة غير الحساسة. لا تستخدم البلاغات داخل المنتج أو نموذج استئناف الحساب لمطالبات مساحات الأسماء.

## متى تفتح مطالبة

افتح مطالبة بمساحة اسم عندما تعتقد أن فريق ClawHub يجب أن يراجع ما إذا كان ينبغي حجز مساحة اسم أو نقلها أو إعادة تسميتها أو إخفاؤها أو عزلها أو إضافة اسم بديل لها أو تغييرها بطريقة أخرى بسبب ملكية في العالم الحقيقي.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسسة GitHub أو مشروعك أو شركتك أو مجتمعك
- نطاق حزمة مثل `@example-org/*` يجب أن ينشر فقط تحت مالك ClawHub المطابق
- اسم مختصر لـ Skills أو اسم حزمة Plugin يبدو كأنه ينتحل صفة مشروع
- نزاع حول علامة تجارية أو علامة مسجلة أو إعادة تسمية مشروع أو سجل حزمة
- مالك محذوف أو غير نشط أو غير قابل للوصول يمنع مالك مساحة الاسم الشرعي

إذا كانت القائمة غير آمنة أو خبيثة أو مضللة بما يتجاوز نزاع الملكية، فاتبع أيضًا إرشادات الإشراف أو الأمان ذات الصلة. نموذج مطالبة مساحة الاسم مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات.

## قبل أن تقدم الطلب

تأكد أولًا من أنك تنشر باستخدام المالك الذي يطابق مساحة الاسم. بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin` بصفتها المالك المطابق `example-org`.

إذا كان بإمكانك إدارة المالك الحالي، فأصلح مساحة الاسم مباشرة بالنشر أو إعادة التسمية أو النقل أو الإخفاء أو حذف المورد المتأثر. استخدم المطالبة عندما لا يمكنك إدارة المالك الحالي أو عندما يحتاج فريق العمل إلى حل نزاع.

## الأدلة المطلوب تضمينها

استخدم أدلة عامة غير حساسة. تشمل الأدلة المفيدة:

- سجل مؤسسة GitHub أو المستودع أو الإصدار أو المشرفين
- وثائق المشروع الرسمية التي تذكر مساحة الاسم
- إثبات النطاق أو نطاق البريد الإلكتروني الرسمي
- التحكم في نطاق سجل الحزم على npm أو PyPI أو crates.io أو غيرها
- دليل ملكية علامة مسجلة أو علامة تجارية أو مشروع آمن للمناقشة علنًا
- سجل مستودع المصدر أو سجل الحزمة أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub أو Skills أو Plugin أو الحزمة أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. يجب أن يتمكن فريق العمل من فهم العلاقة دون الحاجة إلى بيانات اعتماد خاصة أو أسرار.

## ما يجب عدم تضمينه

لا تضع أسرارًا أو أدلة خاصة في مشكلة GitHub عامة. لا تُدرج:

- رموز API أو مفاتيح التوقيع أو بيانات الاعتماد
- رموز تحدي DNS
- ملفات أو عقود قانونية خاصة
- وثائق هوية شخصية
- رسائل بريد إلكتروني خاصة أو تقارير أمان خاصة أو بيانات عملاء سرية

يسأل نموذج المطالبة عما إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع فريق العمل. استخدم ذلك الخيار بدلًا من نشر مواد حساسة علنًا.

## النتائج المحتملة

بحسب الأدلة والمخاطر، قد يحجز فريق ClawHub مساحة اسم، أو ينقل الملكية، أو يعيد تسمية مورد، أو يخفي أو يعزل قائمة موجودة، أو يضيف اسمًا بديلًا أو إعادة توجيه، أو يطلب المزيد من الأدلة، أو يرفض الطلب.

لا تضمن مراجعة مساحة الاسم نقل كل اسم مطابق. يوازن فريق العمل بين الأدلة العامة والاستخدام الحالي ومخاطر الأمان وتأثير ذلك على المستخدمين.
