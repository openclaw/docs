# مطالبات المؤسسة ومساحة الاسم

يستخدم ClawHub معرّفات المالكين، ومعرّفات المؤسسات، ومعرّفات المهارات، وأسماء حزم Plugin،
ونطاقات الحزم بوصفها مساحات أسماء عامة. إذا بدت مساحة اسم كأنها تنتمي إلى
مشروع واقعي، أو علامة تجارية، أو منظومة حزم، أو مؤسسة، لكنها مطالَب بها بالفعل،
أو محجوزة، أو مضللة، أو محل نزاع على ClawHub، فاطلب من فريق العمل مراجعتها
باستخدام
[نموذج مشكلة مطالبة المؤسسة / مساحة الاسم](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة غير الحساسة. لا تستخدم البلاغات داخل المنتج
أو نموذج استئناف الحساب لمطالبات مساحة الاسم.

## متى تفتح مطالبة

افتح مطالبة مساحة اسم عندما تعتقد أن فريق ClawHub يجب أن يراجع ما إذا كان ينبغي
حجز مساحة اسم، أو نقلها، أو إعادة تسميتها، أو إخفاؤها، أو عزلها، أو جعلها اسمًا
مستعارًا، أو تغييرها بطريقة أخرى بسبب ملكية واقعية.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسسة GitHub أو مشروعك أو شركتك أو مجتمعك
- نطاق حزمة مثل `@example-org/*` يجب أن ينشر فقط تحت مالك ClawHub المطابق
- معرّف مهارة أو اسم حزمة Plugin يبدو أنه ينتحل صفة مشروع
- نزاع بشأن علامة تجارية، أو علامة مسجلة، أو إعادة تسمية مشروع، أو تاريخ حزمة
- مالك محذوف، أو غير نشط، أو لا يمكن الوصول إليه يحظر مالك مساحة الاسم المستحق

إذا كانت القائمة غير آمنة، أو خبيثة، أو مضللة بما يتجاوز نزاع الملكية،
فاتبع أيضًا إرشادات الإشراف أو الأمان ذات الصلة. نموذج مطالبة مساحة الاسم
مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات.

## قبل أن تقدّم

تأكد أولًا من أنك تنشر باستخدام المالك الذي يطابق مساحة الاسم.
بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin`
باسم المالك المطابق `example-org`.

إذا كان بإمكانك إدارة المالك الحالي، فأصلح مساحة الاسم مباشرة عن طريق نشر المورد
المتأثر أو إعادة تسميته أو نقله أو إخفائه أو حذفه. استخدم مطالبة عندما لا تستطيع
إدارة المالك الحالي أو عندما يحتاج فريق العمل إلى حل نزاع.

## الأدلة المطلوب تضمينها

استخدم أدلة عامة وغير حساسة. تشمل الأدلة المفيدة:

- مؤسسة GitHub أو مستودعًا أو إصدارًا أو تاريخ صيانة
- وثائق المشروع الرسمية التي تذكر مساحة الاسم
- دليل نطاق أو نطاق بريد إلكتروني رسمي
- التحكم في نطاق سجل حزم مثل npm أو PyPI أو crates.io أو غيرها
- دليل ملكية علامة مسجلة، أو علامة تجارية، أو مشروع يكون آمنًا للنقاش علنًا
- تاريخ مستودع المصدر، أو تاريخ الحزمة، أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub أو المهارة أو Plugin أو الحزمة أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. يجب أن يتمكن فريق العمل من فهم العلاقة من دون الحاجة
إلى بيانات اعتماد أو أسرار خاصة.

## ما يجب عدم تضمينه

لا تضع أسرارًا أو أدلة خاصة في مشكلة GitHub عامة. لا تضمّن:

- رموز API، أو مفاتيح التوقيع، أو بيانات الاعتماد
- رموز تحدي DNS
- ملفات أو عقود قانونية خاصة
- وثائق هوية شخصية
- رسائل بريد إلكتروني خاصة، أو تقارير أمان خاصة، أو بيانات عملاء سرية

يسأل نموذج المطالبة عما إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع فريق العمل.
استخدم هذا الخيار بدلًا من نشر المواد الحساسة علنًا.

## النتائج المحتملة

اعتمادًا على الأدلة والمخاطر، قد يحجز فريق ClawHub مساحة اسم،
أو ينقل الملكية، أو يعيد تسمية مورد، أو يخفي قائمة حالية أو يعزلها،
أو يضيف اسمًا مستعارًا أو إعادة توجيه، أو يطلب مزيدًا من الأدلة، أو يرفض الطلب.

لا تضمن مراجعة مساحة الاسم نقل كل اسم مطابق.
يزن فريق العمل الأدلة العامة، والاستخدام الحالي، ومخاطر الأمان، وتأثير المستخدمين.
