# مطالبات المؤسسة ومساحة الاسم

يستخدم ClawHub معرّفات المالكين، ومعرّفات المؤسسات، ومعرّفات Skills، وأسماء حزم Plugin، ونطاقات الحزم كمساحات أسماء عامة. إذا بدت مساحة اسم كأنها تنتمي إلى مشروع واقعي، أو علامة تجارية، أو منظومة حزم، أو مؤسسة، لكنها مطالَب بها بالفعل، أو محجوزة، أو مضللة، أو محل نزاع على ClawHub، فاطلب من الفريق مراجعتها باستخدام
[نموذج مشكلة مطالبة المؤسسة / مساحة الاسم](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة غير الحساسة. لا تستخدم البلاغات داخل المنتج أو نموذج استئناف الحساب لمطالبات مساحات الأسماء.

## متى تفتح مطالبة

افتح مطالبة بمساحة اسم عندما تعتقد أن على فريق ClawHub مراجعة ما إذا كان ينبغي حجز مساحة اسم أو نقلها أو إعادة تسميتها أو إخفاؤها أو عزلها أو إنشاء اسم مستعار لها أو تغييرها بطريقة أخرى بسبب الملكية الواقعية.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسسة GitHub أو مشروعك أو شركتك أو مجتمعك
- نطاق حزمة مثل `@example-org/*` يجب ألا ينشر إلا تحت مالك ClawHub المطابق
- معرّف Skills أو اسم حزمة Plugin يبدو أنه ينتحل صفة مشروع
- نزاع حول علامة تجارية أو علامة مسجلة أو إعادة تسمية مشروع أو سجل حزمة
- مالك محذوف أو غير نشط أو غير قابل للوصول يحجب مالك مساحة الاسم الشرعي

إذا كانت القائمة غير آمنة أو خبيثة أو مضللة بما يتجاوز نزاع الملكية، فاتبع أيضا إرشادات الإشراف أو الأمان ذات الصلة. نموذج مطالبة مساحة الاسم مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات.

## قبل أن تقدم

تأكد أولا من أنك تنشر باستخدام المالك المطابق لمساحة الاسم. بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin` باسم المالك المطابق `example-org`.

إذا كان بإمكانك إدارة المالك الحالي، فأصلح مساحة الاسم مباشرة عبر نشر المورد المتأثر أو إعادة تسميته أو نقله أو إخفائه أو حذفه. استخدم مطالبة عندما لا يمكنك إدارة المالك الحالي أو عندما يحتاج الفريق إلى حل نزاع.

## الأدلة المطلوب تضمينها

استخدم أدلة عامة غير حساسة. تشمل الأدلة المفيدة:

- سجل مؤسسة GitHub أو المستودع أو الإصدار أو المشرفين
- وثائق المشروع الرسمية التي تذكر مساحة الاسم
- دليل النطاق أو نطاق البريد الإلكتروني الرسمي
- التحكم في نطاق سجل الحزم مثل npm أو PyPI أو crates.io أو غيرها
- دليل ملكية علامة تجارية أو علامة مسجلة أو مشروع يكون آمنا للنقاش علنا
- سجل مستودع المصدر أو سجل الحزمة أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub أو Skills أو Plugin أو الحزمة أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. يجب أن يكون الفريق قادرا على فهم العلاقة دون الحاجة إلى بيانات اعتماد خاصة أو أسرار.

## ما يجب عدم تضمينه

لا تضع أسرارا أو أدلة خاصة في مشكلة GitHub عامة. لا تضمّن:

- رموز API أو مفاتيح التوقيع أو بيانات الاعتماد
- رموز تحدي DNS
- ملفات قانونية أو عقود خاصة
- مستندات هوية شخصية
- رسائل بريد إلكتروني خاصة أو تقارير أمان خاصة أو بيانات عملاء سرية

يسأل نموذج المطالبة عما إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع الفريق. استخدم هذا الخيار بدلا من نشر مواد حساسة علنا.

## النتائج المحتملة

اعتمادا على الأدلة والمخاطر، قد يحجز فريق ClawHub مساحة اسم، أو ينقل الملكية، أو يعيد تسمية مورد، أو يخفي قائمة موجودة أو يعزلها، أو يضيف اسما مستعارا أو إعادة توجيه، أو يطلب مزيدا من الأدلة، أو يرفض الطلب.

لا تضمن مراجعة مساحة الاسم نقل كل اسم مطابق. يوازن الفريق بين الأدلة العامة والاستخدام الحالي ومخاطر الأمان وتأثير ذلك على المستخدمين.
