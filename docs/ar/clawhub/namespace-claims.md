# مطالبات المؤسسة ومساحة الأسماء

يستخدم ClawHub معرّفات المالكين، ومعرّفات المؤسسات، ومعرّفات Skills المختصرة، وأسماء حزم Plugin،
ونطاقات الحزم كمساحات أسماء عامة. إذا بدت مساحة أسماء ما وكأنها تنتمي إلى
مشروع واقعي، أو علامة تجارية، أو منظومة حزم، أو مؤسسة، لكنها مطالَب بها مسبقًا،
أو محجوزة، أو مضللة، أو محل نزاع على ClawHub، فاطلب من فريق العمل مراجعتها
باستخدام
[نموذج مشكلة مطالبة المؤسسة / مساحة الأسماء](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة غير الحساسة. لا تستخدم البلاغات داخل المنتج
أو نموذج استئناف الحساب لمطالبات مساحة الأسماء.

## متى تفتح مطالبة

افتح مطالبة مساحة أسماء عندما تعتقد أن فريق ClawHub يجب أن يراجع ما إذا كان ينبغي
حجز مساحة أسماء، أو نقلها، أو إعادة تسميتها، أو إخفاؤها، أو عزلها، أو إضافة اسم بديل لها،
أو تغييرها بطريقة أخرى بسبب ملكية واقعية.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسسة GitHub، أو مشروعك، أو شركتك، أو مجتمعك
- نطاق حزمة مثل `@example-org/*` ينبغي أن ينشر فقط تحت مالك ClawHub المطابق
- معرّف Skill مختصر أو اسم حزمة Plugin يبدو أنه ينتحل هوية مشروع
- نزاع حول علامة تجارية، أو علامة مسجلة، أو إعادة تسمية مشروع، أو سجل حزمة
- مالك محذوف، أو غير نشط، أو غير قابل للوصول يمنع مالك مساحة الأسماء الشرعي

إذا كانت القائمة غير آمنة، أو ضارة، أو مضللة بما يتجاوز نزاع الملكية،
فاتبع أيضًا إرشادات الإشراف أو الأمان ذات الصلة. نموذج مطالبة مساحة الأسماء
مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات.

## قبل أن تقدم الطلب

تأكد أولًا من أنك تنشر باستخدام المالك الذي يطابق مساحة الأسماء.
بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاقات مثل `@example-org/example-plugin`
بصفتها مالك `example-org` المطابق.

إذا كان بإمكانك إدارة المالك الحالي، فأصلح مساحة الأسماء مباشرة عبر نشر المورد المتأثر،
أو إعادة تسميته، أو نقله، أو إخفائه، أو حذفه. استخدم المطالبة
عندما لا يمكنك إدارة المالك الحالي أو عندما يحتاج فريق العمل إلى حل
نزاع.

## الأدلة التي يجب تضمينها

استخدم أدلة عامة وغير حساسة. تشمل الأدلة المفيدة:

- سجل مؤسسة GitHub، أو المستودع، أو الإصدار، أو المشرفين
- وثائق المشروع الرسمية التي تذكر مساحة الأسماء
- إثبات النطاق أو نطاق البريد الإلكتروني الرسمي
- التحكم في نطاق npm، أو PyPI، أو crates.io، أو سجل حزم آخر
- أدلة ملكية العلامة المسجلة، أو العلامة التجارية، أو المشروع التي يمكن مناقشتها
  علنًا بأمان
- سجل مستودع المصدر، أو سجل الحزمة، أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub أو Skill أو Plugin أو الحزمة أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. ينبغي أن يتمكن فريق العمل من فهم
العلاقة دون الحاجة إلى بيانات اعتماد خاصة أو أسرار.

## ما لا يجب تضمينه

لا تضع أسرارًا أو أدلة خاصة في مشكلة GitHub عامة. لا تضمّن:

- رموز API، أو مفاتيح التوقيع، أو بيانات الاعتماد
- رموز تحدي DNS
- ملفات قانونية أو عقود خاصة
- وثائق هوية شخصية
- رسائل بريد إلكتروني خاصة، أو تقارير أمنية خاصة، أو بيانات عملاء سرية

يسأل نموذج المطالبة عما إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع فريق العمل.
استخدم هذا الخيار بدلًا من نشر المواد الحساسة علنًا.

## النتائج المحتملة

استنادًا إلى الأدلة والمخاطر، قد يحجز فريق ClawHub مساحة أسماء،
أو ينقل الملكية، أو يعيد تسمية مورد، أو يخفي أو يعزل قائمة موجودة،
أو يضيف اسمًا بديلًا أو إعادة توجيه، أو يطلب مزيدًا من الأدلة، أو يرفض الطلب.

لا تضمن مراجعة مساحة الأسماء نقل كل اسم مطابق.
يزن فريق العمل الأدلة العامة، والاستخدام الحالي، ومخاطر الأمان، وتأثير المستخدمين.
