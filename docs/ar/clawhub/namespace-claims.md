# مطالبات المؤسسات ومساحات الأسماء

يستخدم ClawHub معرّفات المالكين، ومعرّفات المؤسسات، ومعرّفات Skills، وأسماء حزم Plugin، ونطاقات الحزم بوصفها مساحات أسماء عامة. إذا بدت مساحة اسم وكأنها تنتمي إلى مشروع واقعي، أو علامة تجارية، أو منظومة حزم، أو مؤسسة، لكنها مطالَب بها بالفعل، أو محجوزة، أو مضللة، أو محل نزاع على ClawHub، فاطلب من فريق العمل مراجعتها عبر [نموذج مشكلة مطالبة مؤسسة / مساحة اسم](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة غير الحساسة. لا تستخدم البلاغات داخل المنتج أو نموذج استئناف الحساب لمطالبات مساحات الأسماء.

## متى تفتح مطالبة

افتح مطالبة بمساحة اسم عندما تعتقد أن على فريق عمل ClawHub مراجعة ما إذا كان ينبغي حجز مساحة اسم، أو نقلها، أو إعادة تسميتها، أو إخفاؤها، أو عزلها، أو إضافة اسم مستعار لها، أو تغييرها بطريقة أخرى بسبب ملكية واقعية.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسسة GitHub الخاصة بك، أو مشروعك، أو شركتك، أو مجتمعك
- نطاق حزمة مثل `@example-org/*` ينبغي أن ينشر فقط تحت مالك ClawHub المطابق
- معرّف مهارة أو اسم حزمة Plugin يبدو أنه ينتحل مشروعًا
- نزاع حول علامة تجارية، أو علامة مسجلة، أو إعادة تسمية مشروع، أو سجل حزمة
- مالك محذوف، أو غير نشط، أو لا يمكن الوصول إليه، ويمنع مالك مساحة الاسم الشرعي

إذا كانت القائمة غير آمنة، أو ضارة، أو مضللة بما يتجاوز نزاع الملكية، فاتبع أيضًا إرشادات الإشراف أو الأمان ذات الصلة. نموذج مطالبة مساحة الاسم مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات.

## قبل أن تقدّم

تأكد أولًا من أنك تنشر باستخدام المالك المطابق لمساحة الاسم. بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin` بصفتها المالك المطابق `example-org`.

إذا كان بإمكانك إدارة المالك الحالي، فأصلح مساحة الاسم مباشرة عبر نشر المورد المتأثر أو إعادة تسميته أو نقله أو إخفائه أو حذفه. استخدم مطالبة عندما لا يمكنك إدارة المالك الحالي أو عندما يحتاج فريق العمل إلى حل نزاع.

## الأدلة التي يجب تضمينها

استخدم أدلة عامة وغير حساسة. تشمل الأدلة المفيدة:

- سجل مؤسسة GitHub أو المستودع أو الإصدار أو المشرفين
- مستندات المشروع الرسمية التي تذكر مساحة الاسم
- إثبات النطاق أو نطاق البريد الإلكتروني الرسمي
- التحكم في نطاق npm أو PyPI أو crates.io أو أي سجل حزم آخر
- دليل ملكية علامة مسجلة أو علامة تجارية أو مشروع يكون آمنًا للنقاش علنًا
- سجل مستودع المصدر، أو سجل الحزمة، أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub أو Skill أو Plugin أو الحزمة أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. ينبغي أن يتمكن فريق العمل من فهم العلاقة من دون الحاجة إلى بيانات اعتماد خاصة أو أسرار.

## ما لا يجب تضمينه

لا تضع أسرارًا أو أدلة خاصة في مشكلة GitHub عامة. لا تضمّن:

- رموز API أو مفاتيح التوقيع أو بيانات الاعتماد
- رموز تحدي DNS
- ملفات أو عقود قانونية خاصة
- وثائق هوية شخصية
- رسائل بريد إلكتروني خاصة، أو تقارير أمنية خاصة، أو بيانات عملاء سرية

يسأل نموذج المطالبة عما إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع فريق العمل. استخدم ذلك الخيار بدلًا من نشر مواد حساسة علنًا.

## النتائج المحتملة

اعتمادًا على الأدلة والمخاطر، قد يحجز فريق عمل ClawHub مساحة اسم، أو ينقل الملكية، أو يعيد تسمية مورد، أو يخفي أو يعزل قائمة موجودة، أو يضيف اسمًا مستعارًا أو إعادة توجيه، أو يطلب مزيدًا من الأدلة، أو يرفض الطلب.

لا تضمن مراجعة مساحة الاسم نقل كل اسم مطابق. يوازن فريق العمل بين الأدلة العامة، والاستخدام الحالي، ومخاطر الأمان، وتأثير ذلك على المستخدمين.
