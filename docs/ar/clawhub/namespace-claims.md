# مطالبات المؤسسات ومساحات الأسماء

يستخدم ClawHub معرّفات المالكين، ومعرّفات المؤسسات، ومعرّفات Skills المختصرة، وأسماء حزم Plugin، ونطاقات
الحزم كمساحات أسماء عامة. إذا بدت مساحة أسماء ما وكأنها تعود إلى
مشروع واقعي أو علامة تجارية أو منظومة حزم أو مؤسسة، لكنها
مطالب بها مسبقًا أو محجوزة أو مضللة أو محل نزاع على ClawHub، فاطلب من فريق العمل مراجعتها
باستخدام
[نموذج مشكلة مطالبة مؤسسة / مساحة أسماء](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة غير الحساسة. لا تستخدم البلاغات داخل المنتج
أو نموذج استئناف الحساب لمطالبات مساحات الأسماء.

## متى تفتح مطالبة

افتح مطالبة مساحة أسماء عندما تعتقد أن فريق ClawHub يجب أن يراجع ما إذا كان ينبغي
حجز مساحة أسماء أو نقلها أو إعادة تسميتها أو إخفاؤها أو عزلها أو إنشاء اسم بديل لها
أو تغييرها بطريقة أخرى بسبب ملكية واقعية.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسسة GitHub أو مشروعك أو شركتك أو مجتمعك
- نطاق حزمة مثل `@example-org/*` يجب أن ينشر فقط تحت
  مالك ClawHub المطابق
- معرّف Skills مختصر أو اسم حزمة Plugin يبدو أنه ينتحل صفة مشروع
- علامة تجارية أو علامة مسجلة أو إعادة تسمية مشروع أو نزاع حول تاريخ حزمة
- مالك محذوف أو غير نشط أو غير قابل للوصول يمنع مالك مساحة الأسماء الشرعي

إذا كانت القائمة غير آمنة أو ضارة أو مضللة بما يتجاوز نزاع الملكية،
فاتبع أيضًا إرشادات الإشراف أو الأمان ذات الصلة. نموذج مطالبة مساحة الأسماء
مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات.

## قبل أن تقدّم الطلب

تحقق أولًا من أنك تنشر بالمالك الذي يطابق مساحة الأسماء.
بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin`
بصفتها المالك المطابق `example-org`.

إذا كان بإمكانك إدارة المالك الحالي، فأصلح مساحة الأسماء مباشرة بالنشر
أو إعادة التسمية أو النقل أو الإخفاء أو حذف المورد المتأثر. استخدم المطالبة
عندما لا تستطيع إدارة المالك الحالي أو عندما يحتاج فريق العمل إلى حل
نزاع.

## الأدلة التي يجب تضمينها

استخدم أدلة عامة غير حساسة. تشمل الأدلة المفيدة:

- مؤسسة GitHub أو مستودع أو إصدار أو تاريخ المشرفين
- وثائق المشروع الرسمية التي تسمي مساحة الأسماء
- دليل النطاق أو نطاق البريد الإلكتروني الرسمي
- التحكم في نطاق سجل الحزم مثل npm أو PyPI أو crates.io أو غيرها
- دليل ملكية العلامة التجارية أو العلامة المسجلة أو المشروع مما يمكن مناقشته
  علنًا بأمان
- تاريخ مستودع المصدر أو تاريخ الحزمة أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub أو Skills أو Plugin أو الحزمة أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. يجب أن يكون فريق العمل قادرًا على فهم
العلاقة دون الحاجة إلى بيانات اعتماد خاصة أو أسرار.

## ما يجب عدم تضمينه

لا تضع أسرارًا أو أدلة خاصة في مشكلة GitHub عامة. لا تضمّن:

- رموز API أو مفاتيح التوقيع أو بيانات الاعتماد
- رموز تحدي DNS
- ملفات أو عقود قانونية خاصة
- وثائق هوية شخصية
- رسائل بريد إلكتروني خاصة أو تقارير أمان خاصة أو بيانات عملاء سرية

يسأل نموذج المطالبة عما إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع فريق العمل.
استخدم ذلك الخيار بدلًا من نشر مواد حساسة علنًا.

## النتائج المحتملة

بحسب الأدلة والمخاطر، قد يحجز فريق ClawHub مساحة أسماء،
أو ينقل الملكية، أو يعيد تسمية مورد، أو يخفي قائمة موجودة أو يعزلها،
أو يضيف اسمًا بديلًا أو إعادة توجيه، أو يطلب مزيدًا من الأدلة، أو يرفض الطلب.

لا تضمن مراجعة مساحة الأسماء نقل كل اسم مطابق.
يزن فريق العمل الأدلة العامة والاستخدام الحالي ومخاطر الأمان وتأثير المستخدمين.
