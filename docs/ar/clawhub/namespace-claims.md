# مطالبات المؤسسة ومساحة الاسم

يستخدم ClawHub معرّفات المالكين، ومعرّفات المؤسسات، ومعرّفات Skills المختصرة، وأسماء حزم Plugin، ونطاقات الحزم كمساحات أسماء عامة. إذا بدت مساحة اسم كأنها تنتمي إلى مشروع واقعي، أو علامة تجارية، أو منظومة حزم، أو مؤسسة، لكنها مُطالب بها مسبقًا، أو محجوزة، أو مضلِّلة، أو محل نزاع على ClawHub، فاطلب من فريق العمل مراجعتها باستخدام
[نموذج مشكلة مطالبة المؤسسة / مساحة الاسم](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة غير الحساسة. لا تستخدم بلاغات داخل المنتج أو نموذج استئناف الحساب لمطالبات مساحة الاسم.

## متى تفتح مطالبة

افتح مطالبة مساحة اسم عندما تعتقد أن فريق ClawHub يجب أن يراجع ما إذا كان ينبغي حجز مساحة اسم، أو نقلها، أو إعادة تسميتها، أو إخفاؤها، أو عزلها، أو جعلها اسمًا مستعارًا، أو تغييرها بطريقة أخرى بسبب ملكية واقعية.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسسة GitHub أو مشروعك أو شركتك أو مجتمعك
- نطاق حزمة مثل `@example-org/*` ينبغي ألا ينشر إلا تحت مالك ClawHub المطابق
- معرّف Skills مختصر أو اسم حزمة Plugin يبدو أنه ينتحل صفة مشروع
- نزاع حول علامة تجارية، أو علامة مسجلة، أو إعادة تسمية مشروع، أو تاريخ حزمة
- مالك محذوف، أو غير نشط، أو لا يمكن الوصول إليه، يمنع مالك مساحة الاسم الشرعي

إذا كانت القائمة غير آمنة، أو خبيثة، أو مضللة بما يتجاوز نزاع الملكية، فاتبع أيضًا إرشادات الإشراف أو الأمان ذات الصلة. نموذج مطالبة مساحة الاسم مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات.

## قبل أن تقدم الطلب

تأكد أولًا من أنك تنشر باستخدام المالك الذي يطابق مساحة الاسم. بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin` باسم مالك `example-org` المطابق.

إذا كنت تستطيع إدارة المالك الحالي، فأصلح مساحة الاسم مباشرةً عبر النشر، أو إعادة التسمية، أو النقل، أو الإخفاء، أو حذف المورد المتأثر. استخدم مطالبة عندما لا تستطيع إدارة المالك الحالي أو عندما يحتاج فريق العمل إلى حل نزاع.

## الأدلة المطلوب تضمينها

استخدم أدلة عامة غير حساسة. تشمل الأدلة المفيدة:

- تاريخ مؤسسة GitHub أو المستودع أو الإصدار أو المشرفين
- وثائق المشروع الرسمية التي تذكر مساحة الاسم
- دليل النطاق أو نطاق البريد الإلكتروني الرسمي
- التحكم في نطاق npm أو PyPI أو crates.io أو سجل حزم آخر
- دليل ملكية العلامة المسجلة أو العلامة التجارية أو المشروع الذي يمكن مناقشته علنًا بأمان
- تاريخ مستودع المصدر، أو تاريخ الحزمة، أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub أو Skill أو Plugin أو الحزمة أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. يجب أن يكون فريق العمل قادرًا على فهم العلاقة دون الحاجة إلى بيانات اعتماد خاصة أو أسرار.

## ما لا ينبغي تضمينه

لا تضع أسرارًا أو أدلة خاصة في مشكلة GitHub عامة. لا تضمّن:

- رموز API، أو مفاتيح التوقيع، أو بيانات الاعتماد
- رموز تحدي DNS
- ملفات قانونية أو عقود خاصة
- مستندات هوية شخصية
- رسائل بريد إلكتروني خاصة، أو تقارير أمان خاصة، أو بيانات عملاء سرية

يسأل نموذج المطالبة عما إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع فريق العمل. استخدم هذا الخيار بدلًا من نشر مواد حساسة علنًا.

## النتائج المحتملة

بناءً على الأدلة والمخاطر، قد يحجز فريق ClawHub مساحة اسم، أو ينقل الملكية، أو يعيد تسمية مورد، أو يخفي قائمة موجودة أو يعزلها، أو يضيف اسمًا مستعارًا أو إعادة توجيه، أو يطلب مزيدًا من الأدلة، أو يرفض الطلب.

لا تضمن مراجعة مساحة الاسم نقل كل اسم مطابق. يوازن فريق العمل بين الأدلة العامة، والاستخدام الحالي، ومخاطر الأمان، وتأثير ذلك على المستخدمين.
