# مطالبات المؤسسة ومساحة الاسم

يستخدم ClawHub معرّفات المالك، ومعرّفات المؤسسة، ومعرّفات Skills المختصرة، وأسماء حزم Plugin، ونطاقات
الحزم كمساحات أسماء عامة. إذا بدت مساحة اسم وكأنها تنتمي إلى مشروع أو علامة تجارية أو منظومة حزم أو مؤسسة
واقعية، لكنها مطالَب بها مسبقًا أو محجوزة أو مضللة أو محل نزاع على ClawHub، فاطلب من فريق العمل مراجعتها
باستخدام
[نموذج مشكلة مطالبة المؤسسة / مساحة الاسم](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة غير الحساسة. لا تستخدم البلاغات داخل المنتج
أو نموذج استئناف الحساب لمطالبات مساحات الأسماء.

## متى تفتح مطالبة

افتح مطالبة مساحة اسم عندما تعتقد أن فريق ClawHub يجب أن يراجع ما إذا كان ينبغي
حجز مساحة اسم أو نقلها أو إعادة تسميتها أو إخفاؤها أو عزلها أو إضافة اسم مستعار لها
أو تغييرها بطريقة أخرى بسبب ملكية واقعية.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسسة GitHub أو مشروعًا أو شركة أو مجتمعًا يخصك
- نطاق حزمة مثل `@example-org/*` يجب أن ينشر فقط تحت مالك ClawHub
  المطابق
- معرّف Skill مختصر أو اسم حزمة Plugin يبدو كأنه ينتحل هوية مشروع
- نزاع حول علامة تجارية أو علامة مسجلة أو إعادة تسمية مشروع أو سجل حزمة
- مالك محذوف أو غير نشط أو غير قابل للوصول يمنع مالك مساحة الاسم الشرعي

إذا كانت القائمة غير آمنة أو ضارة أو مضللة بما يتجاوز نزاع الملكية،
فاتبع أيضًا إرشادات الإشراف أو الأمان ذات الصلة. نموذج مطالبة مساحة الاسم
مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات.

## قبل أن تقدم الطلب

تأكد أولًا من أنك تنشر باستخدام المالك المطابق لمساحة الاسم.
بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin`
باسم المالك المطابق `example-org`.

إذا كان بإمكانك إدارة المالك الحالي، فأصلح مساحة الاسم مباشرة عن طريق النشر
أو إعادة التسمية أو النقل أو الإخفاء أو حذف المورد المتأثر. استخدم مطالبة
عندما لا تستطيع إدارة المالك الحالي أو عندما يحتاج فريق العمل إلى حل
نزاع.

## الأدلة التي ينبغي تضمينها

استخدم أدلة عامة غير حساسة. تشمل الأدلة المفيدة:

- سجل مؤسسة GitHub أو المستودع أو الإصدار أو المشرفين
- وثائق المشروع الرسمية التي تذكر مساحة الاسم
- دليل نطاق أو نطاق بريد إلكتروني رسمي
- التحكم في نطاق سجل حزم npm أو PyPI أو crates.io أو غيرها
- دليل ملكية علامة مسجلة أو علامة تجارية أو مشروع يكون آمنًا لمناقشته
  علنًا
- سجل مستودع المصدر أو سجل الحزمة أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub أو Skill أو Plugin أو الحزمة أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. يجب أن يتمكن فريق العمل من فهم
العلاقة دون الحاجة إلى بيانات اعتماد خاصة أو أسرار.

## ما يجب عدم تضمينه

لا تضع أسرارًا أو أدلة خاصة في مشكلة GitHub عامة. لا تدرج:

- رموز API أو مفاتيح التوقيع أو بيانات الاعتماد
- رموز تحدي DNS
- ملفات قانونية أو عقود خاصة
- مستندات هوية شخصية
- رسائل بريد إلكتروني خاصة أو تقارير أمنية خاصة أو بيانات عملاء سرية

يسأل نموذج المطالبة عما إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع فريق العمل.
استخدم ذلك الخيار بدلًا من نشر مواد حساسة علنًا.

## النتائج المحتملة

بناءً على الأدلة والمخاطر، قد يحجز فريق ClawHub مساحة اسم،
أو ينقل الملكية، أو يعيد تسمية مورد، أو يخفي قائمة حالية أو يعزلها،
أو يضيف اسمًا مستعارًا أو إعادة توجيه، أو يطلب مزيدًا من الأدلة، أو يرفض الطلب.

لا تضمن مراجعة مساحة الاسم نقل كل اسم مطابق.
يزن فريق العمل الأدلة العامة والاستخدام الحالي ومخاطر الأمان وتأثير المستخدمين.
