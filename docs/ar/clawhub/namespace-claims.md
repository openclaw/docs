# مطالبات المؤسسات ومساحات الأسماء

يستخدم ClawHub معرّفات المالكين، ومعرّفات المؤسسات، والشرائح المختصرة لـ Skills، وأسماء حزم Plugin،
ونطاقات الحزم كمساحات أسماء عامة. إذا بدت مساحة اسم وكأنها تنتمي إلى
مشروع أو علامة تجارية أو منظومة حزم أو مؤسسة في العالم الحقيقي، لكنها
مطالب بها أو محجوزة أو مضللة أو محل نزاع على ClawHub، فاطلب من الموظفين مراجعتها
باستخدام
[نموذج مشكلة مطالبة مؤسسة / مساحة اسم](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

استخدم هذا المسار لمراجعة الملكية العامة غير الحساسة. لا تستخدم البلاغات داخل المنتج
أو نموذج استئناف الحساب لمطالبات مساحات الأسماء.

## متى تفتح مطالبة

افتح مطالبة مساحة اسم عندما تعتقد أن على موظفي ClawHub مراجعة ما إذا كان ينبغي
حجز مساحة اسم أو نقلها أو إعادة تسميتها أو إخفاؤها أو عزلها أو إضافة اسم بديل لها
أو تغييرها بطريقة أخرى بسبب ملكية في العالم الحقيقي.

تشمل الأمثلة:

- معرّف مؤسسة يطابق مؤسسة GitHub أو مشروعك أو شركتك أو مجتمعك
- نطاق حزمة مثل `@example-org/*` ينبغي أن ينشر فقط تحت مالك ClawHub المطابق
- شريحة مختصرة لـ Skills أو اسم حزمة Plugin يبدو أنه ينتحل صفة مشروع
- نزاع حول علامة تجارية أو علامة مسجلة أو إعادة تسمية مشروع أو تاريخ حزمة
- مالك محذوف أو غير نشط أو لا يمكن الوصول إليه يحجب مالك مساحة الاسم المستحق

إذا كانت القائمة غير آمنة أو خبيثة أو مضللة بما يتجاوز نزاع الملكية،
فاتبع أيضًا إرشادات الإشراف أو الأمان ذات الصلة. نموذج مطالبة مساحة الاسم
مخصص لمراجعة الملكية، وليس للإفصاح الطارئ عن الثغرات.

## قبل أن تقدم الطلب

تأكد أولًا من أنك تنشر باستخدام المالك الذي يطابق مساحة الاسم.
بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin`
بصفتها المالك المطابق `example-org`.

إذا كان بإمكانك إدارة المالك الحالي، فأصلح مساحة الاسم مباشرة عن طريق النشر
أو إعادة التسمية أو النقل أو الإخفاء أو حذف المورد المتأثر. استخدم المطالبة
عندما لا تستطيع إدارة المالك الحالي أو عندما يحتاج الموظفون إلى حل
نزاع.

## الأدلة التي ينبغي تضمينها

استخدم أدلة عامة غير حساسة. تشمل الأدلة المفيدة:

- سجل مؤسسة GitHub أو المستودع أو الإصدار أو المشرفين
- وثائق المشروع الرسمية التي تذكر مساحة الاسم
- دليل النطاق أو نطاق البريد الإلكتروني الرسمي
- التحكم في نطاق سجل الحزم في npm أو PyPI أو crates.io أو غيرها
- دليل ملكية علامة مسجلة أو علامة تجارية أو مشروع يكون آمنًا للنقاش
  علنًا
- تاريخ مستودع المصدر أو تاريخ الحزمة أو إشعارات إعادة التسمية العامة
- روابط إلى مالك ClawHub أو Skills أو Plugin أو الحزمة أو المشكلة محل النزاع

اشرح ما يثبته كل رابط. ينبغي أن يتمكن الموظفون من فهم
العلاقة دون الحاجة إلى بيانات اعتماد خاصة أو أسرار.

## ما لا ينبغي تضمينه

لا تضع أسرارًا أو أدلة خاصة في مشكلة GitHub عامة. لا تضمّن:

- رموز API أو مفاتيح التوقيع أو بيانات الاعتماد
- رموز تحدي DNS
- ملفات أو عقود قانونية خاصة
- مستندات هوية شخصية
- رسائل بريد إلكتروني خاصة أو تقارير أمان خاصة أو بيانات عملاء سرية

يسأل نموذج المطالبة عما إذا كانت الأدلة الحساسة تحتاج إلى قناة خاصة مع الموظفين.
استخدم ذلك الخيار بدلًا من نشر مواد حساسة علنًا.

## النتائج المحتملة

اعتمادًا على الأدلة والمخاطر، قد يحجز موظفو ClawHub مساحة اسم،
أو ينقلوا الملكية، أو يعيدوا تسمية مورد، أو يخفوا قائمة حالية أو يعزلوها،
أو يضيفوا اسمًا بديلًا أو إعادة توجيه، أو يطلبوا مزيدًا من الأدلة، أو يرفضوا الطلب.

لا تضمن مراجعة مساحة الاسم نقل كل اسم مطابق.
يزن الموظفون الأدلة العامة والاستخدام الحالي ومخاطر الأمان وتأثير المستخدم.
