# استكشاف الأخطاء وإصلاحها

## يفتح `clawhub login` متصفحًا لكنه لا يكتمل أبدًا

تبدأ CLI خادم استدعاء محلي قصير العمر أثناء تسجيل الدخول عبر المتصفح.

- تأكد من أن متصفحك يستطيع الوصول إلى `http://127.0.0.1:<port>/callback`.
- تحقق من قواعد جدار الحماية المحلي وVPN والوكيل إذا لم يصل الاستدعاء أبدًا.
- في البيئات بلا واجهة رسومية، أنشئ رمز API في واجهة الويب الخاصة بـ ClawHub ثم شغّل:

```bash
clawhub login --token clh_...
```

## يُرجع `whoami` أو `publish` الخطأ `Unauthorized` (401)

- سجّل الدخول مرة أخرى باستخدام `clawhub login`.
- إذا كنت تستخدم مسار إعدادات مخصصًا، فتأكد من أن `CLAWHUB_CONFIG_PATH` يشير إلى
  الملف الذي يحتوي على الرمز الحالي لديك.
- إذا كنت تستخدم رمز API، فتأكد من أنه لم يُلغَ في واجهة الويب.

## يُرجع البحث أو التثبيت `Rate limit exceeded` (429)

اقرأ معلومات إعادة المحاولة في الاستجابة:

- `Retry-After`: عدد الثواني التي يجب انتظارها قبل إعادة المحاولة.
- `RateLimit-Limit`: الحد المطبّق على هذا الطلب.
- `RateLimit-Remaining`: ميزانيتك المتبقية بالضبط عند وجود هذا الترويس. عند `429`، تكون `0`.
- `RateLimit-Reset` أو `X-RateLimit-Reset`: توقيت إعادة الضبط.

إذا كان مستخدمون كثيرون يتشاركون عنوان IP خروج واحدًا، فقد تُستنفد حدود IP المجهولة حتى لو كان كل
شخص يرسل بضع طلبات فقط. سجّل الدخول حيثما أمكن، ثم أعد المحاولة بعد
التأخير المُبلّغ عنه.

## يفشل البحث أو التثبيت خلف وكيل

تحترم CLI متغيرات الوكيل القياسية:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

تشمل الأسماء المدعومة `HTTPS_PROXY` و`HTTP_PROXY` و`https_proxy` و
`http_proxy`.

## لا تظهر Skills في البحث

- تحقق من slug الدقيق أو صفحة المالك إذا كنت تعرفها.
- تأكد من أن الإصدار عام وليس محتجزًا بسبب الفحص أو الإشراف.
- إذا كنت تملك Skills، فسجّل الدخول وافحصها:

```bash
clawhub inspect @openclaw/demo
```

قد توضح التشخيصات المرئية للمالك حالة الفحص أو بوابة الرفع أو الإشراف.

## يفشل النشر بسبب نقص بيانات وصفية مطلوبة

بالنسبة إلى Skills، تحقق من frontmatter في `SKILL.md`. يجب التصريح بمتغيرات البيئة
والأدوات المطلوبة حتى يتمكن المستخدمون والماسحات من فهم الحزمة.

بالنسبة إلى Plugins، تحقق من بيانات توافق `package.json` الوصفية. تحتاج عمليات نشر Code-plugin
إلى حقول توافق OpenClaw مثل `openclaw.compat.pluginApi` و
`openclaw.build.openclawVersion`.

عاين حمولة النشر أولًا:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## يفشل النشر بسبب خطأ في مالك GitHub أو المصدر

يستخدم ClawHub هوية GitHub وإسناد المصدر لربط الحزم بناشريها.

- تأكد من أنك سجّلت الدخول بحساب GitHub الذي يملك الحزمة أو يستطيع نشرها.
- تحقق من أن عنوان URL للمصدر عام أو يمكن لـ ClawHub الوصول إليه.
- بالنسبة إلى مصادر GitHub، استخدم `owner/repo` أو `owner/repo@ref` أو عنوان URL كاملًا من GitHub.

## يفشل النشر لأن مساحة اسم محجوزة أو مُطالَب بها

إذا فشل النشر لأن مقبض المالك أو مساحة اسم المؤسسة أو نطاق الحزمة أو slug الخاص بـ Skills
أو اسم الحزمة مُطالَب به أو محجوز بالفعل، فتأكد أولًا من أنك
تنشر باستخدام المالك المطابق لمساحة الاسم. بالنسبة إلى حزم Plugin،
يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin` بصفتها المالك
المطابق `example-org`.

إذا كنت تعتقد أن مؤسستك أو مشروعك أو علامتك التجارية هي المالك الشرعي لمساحة الاسم لكنك
لا تستطيع إدارة مالك ClawHub الحالي، فافتح
[مشكلة مطالبة مؤسسة / مساحة اسم](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
مع إثبات عام وغير حساس. راجع
[مطالبات المؤسسات ومساحات الأسماء](/clawhub/namespace-claims) للحصول على إرشادات الأدلة وما
ينبغي إبقاؤه خارج المشكلات العامة.

## يقول `sync` إنه لم يتم العثور على Skills

يبحث `sync` عن مجلدات تحتوي على `SKILL.md` أو `skill.md`.

وجّهه إلى الجذور التي تريد فحصها:

```bash
clawhub sync --root /path/to/skills
```

عاين أولًا إذا لم تكن متأكدًا مما سيُنشر:

```bash
clawhub sync --all --dry-run --no-input
```

## يرفض `update` بسبب تغييرات محلية

لا تطابق الملفات المحلية أي إصدار يعرفه ClawHub. اختر واحدًا:

- احتفظ بالتعديلات المحلية وتجاوز التحديث.
- اكتب فوقها بالإصدار المنشور:

```bash
clawhub update @openclaw/demo --force
```

- انشر نسختك المعدّلة كـ slug جديد أو fork.

## يفشل تثبيت Plugin في OpenClaw

- استخدم مصدر ClawHub صريحًا:

```bash
openclaw plugins install clawhub:<package>
```

- تحقق من صفحة تفاصيل الحزمة لمعرفة حالة الفحص وبيانات التوافق الوصفية.
- تأكد من أن إصدار OpenClaw لديك يستوفي نطاق التوافق
  المُعلن للحزمة.
- إذا كانت الحزمة مخفية أو محتجزة أو محظورة، فقد لا تكون قابلة للتثبيت حتى
  يحل المالك المشكلة.

## تفشل طلبات API العامة

- احترم ترويسات إعادة المحاولة `429` وخزّن استجابات القوائم/البحث العامة مؤقتًا.
- أعد ربط المستخدمين بقائمة ClawHub القانونية.
- لا تعكس المحتوى المخفي أو الخاص أو المحتجز أو المحظور بالإشراف خارج
  سطح API العام.

راجع [HTTP API](/clawhub/http-api) لمعرفة تفاصيل نقاط النهاية.
