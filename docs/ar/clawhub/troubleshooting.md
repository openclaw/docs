# استكشاف الأخطاء وإصلاحها

## `clawhub login` يفتح متصفحًا لكنه لا يكتمل أبدًا

يبدأ CLI خادم رد اتصال محليًا قصير العمر أثناء تسجيل الدخول عبر المتصفح.

- تأكد من أن متصفحك يستطيع الوصول إلى `http://127.0.0.1:<port>/callback`.
- تحقق من قواعد جدار الحماية المحلي وVPN والوكيل إذا لم يصل رد الاتصال مطلقًا.
- في البيئات عديمة الواجهة، أنشئ رمز API مميزًا في واجهة الويب لـ ClawHub وشغّل:

```bash
clawhub login --token clh_...
```

## يعيد `whoami` أو `publish` القيمة `Unauthorized` (401)

- سجّل الدخول مجددًا باستخدام `clawhub login`.
- إذا كنت تستخدم مسار إعدادات مخصصًا، فتأكد من أن `CLAWHUB_CONFIG_PATH` يشير إلى
  الملف الذي يحتوي على رمزك المميز الحالي.
- إذا كنت تستخدم رمز API مميزًا، فتأكد من أنه لم يُلغَ في واجهة الويب.

## يعيد البحث أو التثبيت `Rate limit exceeded` (429)

اقرأ معلومات إعادة المحاولة في الاستجابة:

- `Retry-After`: عدد الثواني التي يجب انتظارها قبل إعادة المحاولة.
- `RateLimit-Limit`: الحد المطبّق على هذا الطلب.
- `RateLimit-Remaining`: ميزانيتك المتبقية بدقة عند وجود الترويسة. في `429`، تكون `0`.
- `RateLimit-Reset` أو `X-RateLimit-Reset`: توقيت إعادة التعيين.

إذا كان العديد من المستخدمين يتشاركون عنوان IP واحدًا للخروج، فقد تُستنفد حدود عناوين IP المجهولة حتى لو كان كل
شخص لا يرسل سوى بضعة طلبات. سجّل الدخول حيثما أمكن، وأعد المحاولة بعد
مدة التأخير المبلّغ عنها.

## يفشل البحث أو التثبيت خلف وكيل

يراعي CLI متغيرات الوكيل القياسية:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

تشمل الأسماء المدعومة `HTTPS_PROXY` و`HTTP_PROXY` و`https_proxy` و
`http_proxy`.

## لا تظهر مهارة في البحث

- تحقق من المعرّف المختصر الدقيق أو صفحة المالك إذا كنت تعرفه.
- تأكد من أن الإصدار عام وغير محتجز بسبب الفحص أو الإشراف.
- إذا كنت تملك المهارة، فسجّل الدخول وافحصها:

```bash
clawhub inspect @openclaw/demo
```

قد توضّح بيانات التشخيص المرئية للمالك حالة الفحص أو بوابة الرفع أو الإشراف.

## يفشل النشر بسبب فقدان البيانات الوصفية المطلوبة

بالنسبة إلى المهارات، تحقق من بيانات frontmatter في `SKILL.md`. يجب التصريح بمتغيرات البيئة
والأدوات المطلوبة حتى يتمكن المستخدمون وأدوات الفحص من فهم الحزمة.

بالنسبة إلى Plugins، تحقق من بيانات تعريف التوافق في `package.json`. تتطلب عمليات نشر
Plugin البرمجية حقول توافق OpenClaw مثل `openclaw.compat.pluginApi` و
`openclaw.build.openclawVersion`.

عاين حمولة النشر أولًا:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## يفشل النشر بسبب خطأ في مالك GitHub أو المصدر

يستخدم ClawHub هوية GitHub وإسناد المصدر لربط الحزم
بناشريها.

- تأكد من تسجيل دخولك باستخدام حساب GitHub الذي يملك الحزمة أو يمكنه نشرها.
- تحقق من أن عنوان URL للمصدر عام أو يمكن لـ ClawHub الوصول إليه.
- بالنسبة إلى مصادر GitHub، استخدم `owner/repo` أو `owner/repo@ref` أو عنوان URL كاملًا لـ GitHub.

## يفشل النشر لأن نطاق أسماء محجوز أو مُطالب به

إذا فشل النشر لأن معرّف المالك أو نطاق أسماء المؤسسة أو نطاق الحزمة أو المعرّف المختصر
للمهارة أو اسم الحزمة مُطالب به أو محجوز بالفعل، فتأكد أولًا من أنك
تنشر باستخدام المالك المطابق لنطاق الأسماء. بالنسبة إلى حزم Plugins،
يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin` باسم المالك
المطابق `example-org`.

إذا كنت تعتقد أن مؤسستك أو مشروعك أو علامتك التجارية هي المالك الشرعي لنطاق الأسماء، لكنك
لا تستطيع إدارة مالك ClawHub الحالي، فافتح
[بلاغ مطالبة بمؤسسة / نطاق أسماء](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
مع دليل عام وغير حساس. راجع
[مطالبات المؤسسات ونطاقات الأسماء](/clawhub/namespace-claims) للحصول على إرشادات الأدلة وما
يجب استبعاده من البلاغات العامة.

## يقول `sync` إنه لم يُعثر على مهارات

يبحث `sync` عن مجلدات تحتوي على `SKILL.md` أو `skill.md`.

وجّهه إلى الجذور التي تريد فحصها:

```bash
clawhub sync --root /path/to/skills
```

عاين أولًا إذا لم تكن متأكدًا مما سيُنشر:

```bash
clawhub sync --all --dry-run --no-input
```

## يرفض `update` بسبب تغييرات محلية

لا تطابق الملفات المحلية أي إصدار يعرفه ClawHub. اختر أحد الخيارات التالية:

- احتفظ بالتعديلات المحلية وتخطَّ التحديث.
- استبدلها بالإصدار المنشور:

```bash
clawhub update @openclaw/demo --force
```

- انشر نسختك المعدّلة بمعرّف مختصر جديد أو كتفرّع.

## يفشل تثبيت Plugin في OpenClaw

- استخدم مصدر ClawHub صريحًا:

```bash
openclaw plugins install clawhub:<package>
```

- تحقق من صفحة تفاصيل الحزمة لمعرفة حالة الفحص وبيانات تعريف التوافق.
- تأكد من أن إصدار OpenClaw لديك يستوفي نطاق التوافق
  المُعلن للحزمة.
- إذا كانت الحزمة مخفية أو محتجزة أو محظورة، فقد لا تكون قابلة للتثبيت حتى
  يحل المالك المشكلة.

## تفشل طلبات API العامة

- التزم بترويسات إعادة المحاولة `429` وخزّن استجابات القوائم العامة والبحث مؤقتًا.
- أعِد توجيه المستخدمين إلى قائمة ClawHub الأساسية.
- لا تنسخ المحتوى المخفي أو الخاص أو المحتجز أو المحظور بالإشراف خارج
  سطح API العام.

راجع [واجهة HTTP البرمجية](/clawhub/http-api) للاطلاع على تفاصيل نقاط النهاية.
