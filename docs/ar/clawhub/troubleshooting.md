# استكشاف الأخطاء وإصلاحها

## يفتح `clawhub login` متصفحًا لكنه لا يكتمل أبدًا

تبدأ CLI خادم استدعاء راجع محلي قصير العمر أثناء تسجيل الدخول عبر المتصفح.

- تأكد من أن متصفحك يستطيع الوصول إلى `http://127.0.0.1:<port>/callback`.
- تحقق من قواعد جدار الحماية المحلي وVPN والوكيل إذا لم يصل الاستدعاء الراجع أبدًا.
- في البيئات بلا واجهة، أنشئ رمز API في واجهة ويب ClawHub وشغّل:

```bash
clawhub login --token clh_...
```

## يعيد `whoami` أو `publish` القيمة `Unauthorized` (401)

- سجّل الدخول مرة أخرى باستخدام `clawhub login`.
- إذا كنت تستخدم مسار إعدادات مخصصًا، فتأكد من أن `CLAWHUB_CONFIG_PATH` يشير إلى
  الملف الذي يحتوي على رمزك الحالي.
- إذا كنت تستخدم رمز API، فتأكد من أنه لم يُلغَ في واجهة الويب.

## يعيد البحث أو التثبيت `Rate limit exceeded` (429)

اقرأ معلومات إعادة المحاولة في الاستجابة:

- `Retry-After`: عدد الثواني التي يجب انتظارها قبل إعادة المحاولة.
- `RateLimit-Limit`: الحد المطبّق على هذا الطلب.
- `RateLimit-Remaining`: ميزانيتك المتبقية بالضبط عند وجود الترويسة. عند `429`، تكون `0`.
- `RateLimit-Reset` أو `X-RateLimit-Reset`: توقيت إعادة الضبط.

إذا كان العديد من المستخدمين يشتركون في عنوان IP خروج واحد، فقد تُستنفد حدود IP
المجهول حتى عندما يرسل كل شخص بضعة طلبات فقط. سجّل الدخول حيثما أمكن وأعد المحاولة بعد
التأخير المبلّغ عنه.

## يفشل البحث أو التثبيت خلف وكيل

تحترم CLI متغيرات الوكيل القياسية:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

تشمل الأسماء المدعومة `HTTPS_PROXY` و`HTTP_PROXY` و`https_proxy` و
`http_proxy`.

## لا تظهر مهارة في البحث

- تحقق من المعرّف المختصر الدقيق أو صفحة المالك إذا كنت تعرفها.
- تأكد من أن الإصدار عام وليس محتجزًا بسبب الفحص أو الإشراف.
- إذا كنت تملك المهارة، فسجّل الدخول وافحصها:

```bash
clawhub inspect @openclaw/demo
```

قد تشرح التشخيصات المرئية للمالك حالة الفحص أو بوابة الرفع أو الإشراف.

## يفشل النشر بسبب نقص بيانات وصفية مطلوبة

بالنسبة إلى المهارات، تحقق من الواجهة الأمامية في `SKILL.md`. يجب التصريح بمتغيرات
البيئة المطلوبة والأدوات حتى يتمكن المستخدمون والماسحات من فهم الحزمة.

بالنسبة إلى Plugins، تحقق من بيانات وصف التوافق في `package.json`. تتطلب عمليات نشر
Code-plugin حقول توافق OpenClaw مثل `openclaw.compat.pluginApi` و
`openclaw.build.openclawVersion`.

عاين حمولة النشر أولًا:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## يفشل النشر بسبب خطأ في مالك GitHub أو المصدر

يستخدم ClawHub هوية GitHub وإسناد المصدر لربط الحزم بناشريها.

- تأكد من أنك مسجل الدخول بحساب GitHub الذي يملك الحزمة أو يمكنه نشرها.
- تحقق من أن عنوان URL للمصدر عام أو يمكن لـ ClawHub الوصول إليه.
- بالنسبة إلى مصادر GitHub، استخدم `owner/repo` أو `owner/repo@ref` أو عنوان URL كاملًا لـ GitHub.

## يفشل النشر بسبب المطالبة بمساحة اسم أو حجزها

إذا فشل النشر لأن معرّف المالك أو مساحة اسم المؤسسة أو نطاق الحزمة أو المعرّف المختصر
للمهارة أو اسم الحزمة مُطالب به بالفعل أو محجوز، فتأكد أولًا من أنك تنشر باستخدام
المالك الذي يطابق مساحة الاسم. بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق
مثل `@example-org/example-plugin` بصفتها مالك `example-org` المطابق.

إذا كنت تعتقد أن مؤسستك أو مشروعك أو علامتك التجارية هي مالك مساحة الاسم الشرعي لكنك
لا تستطيع إدارة مالك ClawHub الحالي، فافتح
[مشكلة مطالبة بمؤسسة / مساحة اسم](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
مع دليل عام غير حساس. راجع
[مطالبات المؤسسة ومساحات الأسماء](/ar/clawhub/namespace-claims) للحصول على إرشادات الأدلة وما
يجب إبقاؤه خارج المشكلات العامة.

## يقول `sync` إنه لم تُعثر على مهارات

يبحث `sync` عن مجلدات تحتوي على `SKILL.md` أو `skill.md`.

وجّهه إلى الجذور التي تريد فحصها:

```bash
clawhub sync --root /path/to/skills
```

عاين أولًا إذا لم تكن متأكدًا مما سيُنشر:

```bash
clawhub sync --all --dry-run --no-input
```

## يرفض `update` بسبب تغييرات محلية

لا تطابق الملفات المحلية أي إصدار يعرفه ClawHub. اختر واحدًا:

- احتفظ بالتعديلات المحلية وتجاوز التحديث.
- استبدلها بالإصدار المنشور:

```bash
clawhub update @openclaw/demo --force
```

- انشر نسختك المعدّلة كمعرّف مختصر جديد أو تفريع.

## يفشل تثبيت Plugin في OpenClaw

- استخدم مصدر ClawHub صريحًا:

```bash
openclaw plugins install clawhub:<package>
```

- تحقق من صفحة تفاصيل الحزمة لمعرفة حالة الفحص وبيانات وصف التوافق.
- تأكد من أن إصدار OpenClaw لديك يفي بنطاق التوافق المعلن للحزمة.
- إذا كانت الحزمة مخفية أو محتجزة أو محظورة، فقد لا تكون قابلة للتثبيت حتى
  يحل المالك المشكلة.

## تفشل طلبات API العامة

- احترم ترويسات إعادة المحاولة `429` وخزّن استجابات القوائم العامة/البحث مؤقتًا.
- أعد ربط المستخدمين بالقائمة الأساسية في ClawHub.
- لا تعكس المحتوى المخفي أو الخاص أو المحتجز أو المحظور بالإشراف خارج سطح
  API العام.

راجع [HTTP API](/ar/clawhub/http-api) لتفاصيل نقاط النهاية.
