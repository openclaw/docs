# استكشاف الأخطاء وإصلاحها

## يفتح `clawhub login` متصفحًا لكنه لا يكتمل أبدًا

تشغّل CLI خادم استدعاء عكسي محليًا قصير العمر أثناء تسجيل الدخول عبر المتصفح.

- تأكد من أن متصفحك يستطيع الوصول إلى `http://127.0.0.1:<port>/callback`.
- تحقق من قواعد جدار الحماية المحلي وVPN والوكيل إذا لم يصل الاستدعاء العكسي مطلقًا.
- في البيئات عديمة الواجهة، أنشئ رمز API مميزًا في واجهة الويب لـ ClawHub، ثم نفّذ:

```bash
clawhub login --token clh_...
```

## يعيد `whoami` أو `publish` الخطأ `Unauthorized` (401)

- سجّل الدخول مجددًا باستخدام `clawhub login`.
- إذا كنت تستخدم مسار إعدادات مخصصًا، فتأكد من أن `CLAWHUB_CONFIG_PATH` يشير إلى
  الملف الذي يحتوي على رمزك المميز الحالي.
- إذا كنت تستخدم رمز API مميزًا، فتأكد من أنه لم يُلغَ في واجهة الويب.

## يعيد البحث أو التثبيت الخطأ `Rate limit exceeded` (429)

اقرأ معلومات إعادة المحاولة في الاستجابة:

- `Retry-After`: عدد الثواني المطلوب انتظارها قبل إعادة المحاولة.
- `RateLimit-Limit`: الحد المطبّق على هذا الطلب.
- `RateLimit-Remaining`: الحصة المتبقية لديك بدقة عند وجود الترويسة. عند `429`، تكون `0`.
- `RateLimit-Reset` أو `X-RateLimit-Reset`: توقيت إعادة التعيين.

إذا كان العديد من المستخدمين يشتركون في عنوان IP واحد للخروج، فقد تُستنفد حدود عناوين IP المجهولة حتى لو كان كل
شخص يرسل بضعة طلبات فقط. سجّل الدخول حيثما أمكن، وأعد المحاولة بعد
فترة التأخير المُبلّغ عنها.

## يفشل البحث أو التثبيت خلف وكيل

تحترم CLI متغيرات الوكيل القياسية:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

تشمل الأسماء المدعومة `HTTPS_PROXY` و`HTTP_PROXY` و`https_proxy`
و`http_proxy`.

## لا تظهر Skill في البحث

- تحقق من المعرّف النصي الدقيق أو صفحة المالك إذا كنت تعرفه.
- تأكد من أن الإصدار عام وليس محتجزًا بسبب الفحص أو الإشراف.
- إذا كنت تملك Skill، فسجّل الدخول وافحصها:

```bash
clawhub inspect @openclaw/demo
```

قد توضّح معلومات التشخيص المرئية للمالك حالة الفحص أو بوابة الرفع أو الإشراف.

## يفشل النشر بسبب فقدان البيانات الوصفية المطلوبة

بالنسبة إلى Skills، تحقق من البيانات الأمامية في `SKILL.md`. ينبغي التصريح بمتغيرات البيئة
والأدوات المطلوبة حتى يتمكن المستخدمون وأدوات الفحص من فهم الحزمة.

بالنسبة إلى Plugins، تحقق من بيانات التوافق الوصفية في `package.json`. تتطلب عمليات نشر Plugins البرمجية
حقول توافق OpenClaw مثل `openclaw.compat.pluginApi`
و`openclaw.build.openclawVersion`.

عاين حمولة النشر أولًا:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## يفشل النشر بسبب خطأ في مالك GitHub أو المصدر

يستخدم ClawHub هوية GitHub ونَسب المصدر لربط الحزم
بناشريها.

- تأكد من تسجيل دخولك بحساب GitHub الذي يملك الحزمة أو يمكنه نشرها.
- تحقق من أن عنوان URL للمصدر عام أو متاح لـ ClawHub.
- بالنسبة إلى مصادر GitHub، استخدم `owner/repo` أو `owner/repo@ref` أو عنوان URL كاملًا لـ GitHub.

## يفشل النشر لأن مساحة اسم محجوزة أو مُطالب بها

إذا فشل النشر لأن معرّف المالك أو مساحة اسم المؤسسة أو نطاق الحزمة أو المعرّف النصي لـ Skill
أو اسم الحزمة مُطالب به أو محجوز بالفعل، فتأكد أولًا من أنك
تنشر باستخدام المالك المطابق لمساحة الاسم. بالنسبة إلى حزم Plugins،
يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin` باسم المالك
المطابق `example-org`.

إذا كنت تعتقد أن مؤسستك أو مشروعك أو علامتك التجارية هي المالك الشرعي لمساحة الاسم، لكنك
لا تستطيع إدارة مالك ClawHub الحالي، فافتح
[مشكلة مطالبة بمؤسسة / مساحة اسم](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
مع إثبات عام وغير حساس. راجع
[مطالبات المؤسسات ومساحات الأسماء](/clawhub/namespace-claims) للحصول على إرشادات بشأن الأدلة وما
يجب عدم تضمينه في المشكلات العامة.

## يفيد `sync` بعدم العثور على Skills

يبحث `sync` عن المجلدات التي تحتوي على `SKILL.md` أو `skill.md`.

وجّهه إلى الجذور التي تريد فحصها:

```bash
clawhub sync --root /path/to/skills
```

عاين أولًا إذا لم تكن متأكدًا مما سيُنشر:

```bash
clawhub sync --all --dry-run --no-input
```

## يرفض `update` بسبب تغييرات محلية

لا تطابق الملفات المحلية أي إصدار معروف لدى ClawHub. اختر أحد الخيارات التالية:

- احتفظ بالتعديلات المحلية وتخطَّ التحديث.
- استبدلها بالإصدار المنشور:

```bash
clawhub update @openclaw/demo --force
```

- انشر نسختك المعدّلة بمعرّف نصي جديد أو كتفرّع.

## يفشل تثبيت Plugin في OpenClaw

- استخدم مصدر ClawHub صريحًا:

```bash
openclaw plugins install clawhub:<package>
```

- تحقق من صفحة تفاصيل الحزمة لمعرفة حالة الفحص وبيانات التوافق الوصفية.
- تأكد من أن إصدار OpenClaw لديك يستوفي نطاق التوافق المُعلن للحزمة.
- إذا كانت الحزمة مخفية أو محتجزة أو محظورة، فقد يتعذر تثبيتها حتى
  يحل المالك المشكلة.

## تفشل طلبات API العامة

- احترم ترويسات إعادة المحاولة الخاصة بـ `429`، وخزّن استجابات القوائم العامة والبحث مؤقتًا.
- أَعِد توجيه المستخدمين إلى قائمة ClawHub المرجعية.
- لا تنسخ المحتوى المخفي أو الخاص أو المحتجز أو المحظور بالإشراف خارج
  السطح العام لـ API.

راجع [واجهة HTTP البرمجية](/clawhub/http-api) للحصول على تفاصيل نقاط النهاية.
