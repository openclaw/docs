# استكشاف الأخطاء وإصلاحها

## يفتح `clawhub login` متصفحًا لكنه لا يكتمل أبدًا

يبدأ CLI خادم رد نداء محليًا قصير العمر أثناء تسجيل الدخول عبر المتصفح.

- تأكد من أن متصفحك يمكنه الوصول إلى `http://127.0.0.1:<port>/callback`.
- تحقق من قواعد جدار الحماية المحلي وVPN والوكيل إذا لم يصل رد النداء أبدًا.
- في البيئات عديمة الواجهة، أنشئ رمز API مميزًا في واجهة ويب ClawHub وشغّل:

```bash
clawhub login --token clh_...
```

## يعيد `whoami` أو `publish` الخطأ `Unauthorized` (401)

- سجّل الدخول مرة أخرى باستخدام `clawhub login`.
- إذا كنت تستخدم مسار إعدادات مخصصًا، فتأكد من أن `CLAWHUB_CONFIG_PATH` يشير إلى
  الملف الذي يحتوي على رمزك المميز الحالي.
- إذا كنت تستخدم رمز API مميزًا، فتأكد من أنه لم يُلغَ في واجهة الويب.

## يعيد البحث أو التثبيت `Rate limit exceeded` (429)

اقرأ معلومات إعادة المحاولة في الاستجابة:

- `Retry-After`: عدد الثواني التي يجب انتظارها قبل إعادة المحاولة.
- `RateLimit-Limit`: الحد المطبّق على هذا الطلب.
- `RateLimit-Remaining`: الميزانية المتبقية الدقيقة لديك عند وجود الترويسة. عند `429`، تكون `0`.
- `RateLimit-Reset` أو `X-RateLimit-Reset`: توقيت إعادة التعيين.

إذا كان مستخدمون كثيرون يتشاركون عنوان IP خروج واحدًا، فقد تُستنفد حدود IP المجهول حتى عندما
يرسل كل شخص بضعة طلبات فقط. سجّل الدخول حيثما أمكن وأعد المحاولة بعد
التأخير المبلّغ عنه.

## يفشل البحث أو التثبيت خلف وكيل

يحترم CLI متغيرات الوكيل القياسية:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

تشمل الأسماء المدعومة `HTTPS_PROXY` و`HTTP_PROXY` و`https_proxy` و
`http_proxy`.

## لا تظهر مهارة في البحث

- تحقق من المعرّف النصي الدقيق أو صفحة المالك إذا كنت تعرفها.
- تأكد من أن الإصدار عام وليس محتجزًا بسبب الفحص أو الإشراف.
- إذا كنت تملك المهارة، فسجّل الدخول وافحصها:

```bash
clawhub inspect @openclaw/demo
```

قد تشرح التشخيصات المرئية للمالك حالة الفحص أو بوابة الرفع أو الإشراف.

## يفشل النشر لأن بيانات وصفية مطلوبة مفقودة

بالنسبة إلى المهارات، تحقق من frontmatter في `SKILL.md`. يجب التصريح عن متغيرات البيئة
والأدوات المطلوبة حتى يتمكن المستخدمون والماسحات من فهم الحزمة.

بالنسبة إلى Plugins، تحقق من بيانات وصف التوافق في `package.json`. تتطلب عمليات نشر Code-plugin
حقول توافق OpenClaw مثل `openclaw.compat.pluginApi` و
`openclaw.build.openclawVersion`.

عاين حمولة النشر أولًا:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## يفشل النشر بسبب خطأ في مالك GitHub أو المصدر

يستخدم ClawHub هوية GitHub ونَسب المصدر لربط الحزم بناشريها.

- تأكد من أنك مسجّل الدخول بحساب GitHub الذي يملك الحزمة أو يمكنه نشرها.
- تحقق من أن عنوان URL للمصدر عام أو يمكن لـ ClawHub الوصول إليه.
- بالنسبة إلى مصادر GitHub، استخدم `owner/repo` أو `owner/repo@ref` أو عنوان URL كاملًا من GitHub.

## يفشل النشر لأن نطاق اسم محجوز أو مُطالب به

إذا فشل النشر لأن معرّف المالك أو نطاق اسم المؤسسة أو نطاق الحزمة أو المعرّف النصي للمهارة
أو اسم الحزمة مُطالب به أو محجوز مسبقًا، فتأكد أولًا من أنك
تنشر باستخدام المالك المطابق لنطاق الاسم. بالنسبة إلى حزم Plugin،
يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin` بصفتها
المالك المطابق `example-org`.

إذا كنت تعتقد أن مؤسستك أو مشروعك أو علامتك التجارية هي المالك الشرعي لنطاق الاسم لكنك
لا تستطيع إدارة مالك ClawHub الحالي، فافتح
[مشكلة مطالبة مؤسسة / نطاق اسم](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
مع إثبات عام وغير حساس. راجع
[مطالبات المؤسسات ونطاقات الأسماء](/clawhub/namespace-claims) للحصول على إرشادات الأدلة وما
ينبغي إبقاؤه خارج المشكلات العامة.

## يقول `sync` إنه لم يعثر على أي مهارات

يبحث `sync` عن مجلدات تحتوي على `SKILL.md` أو `skill.md`.

وجّهه إلى الجذور التي تريد فحصها:

```bash
clawhub sync --root /path/to/skills
```

عاين أولًا إذا لم تكن متأكدًا مما سيُنشر:

```bash
clawhub sync --all --dry-run --no-input
```

## يرفض `update` بسبب تغييرات محلية

لا تطابق الملفات المحلية أي إصدار يعرفه ClawHub. اختر واحدًا:

- احتفظ بالتعديلات المحلية وتجاوز التحديث.
- اكتب فوقها بالإصدار المنشور:

```bash
clawhub update @openclaw/demo --force
```

- انشر نسختك المعدّلة كمعرّف نصي جديد أو تفرّع.

## يفشل تثبيت Plugin في OpenClaw

- استخدم مصدر ClawHub صريحًا:

```bash
openclaw plugins install clawhub:<package>
```

- تحقق من صفحة تفاصيل الحزمة لمعرفة حالة الفحص وبيانات وصف التوافق.
- تأكد من أن إصدار OpenClaw لديك يستوفي نطاق التوافق
  المعلن للحزمة.
- إذا كانت الحزمة مخفية أو محتجزة أو محظورة، فقد لا تكون قابلة للتثبيت حتى
  يحل المالك المشكلة.

## تفشل طلبات API العامة

- احترم ترويسات إعادة المحاولة `429` وخزّن استجابات القوائم/البحث العامة مؤقتًا.
- أعد توجيه المستخدمين إلى قائمة ClawHub المرجعية.
- لا تعكس المحتوى المخفي أو الخاص أو المحتجز أو المحظور بالإشراف خارج
  سطح API العام.

راجع [HTTP API](/clawhub/http-api) للحصول على تفاصيل نقاط النهاية.
