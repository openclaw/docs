# استكشاف الأخطاء وإصلاحها

## يفتح `clawhub login` متصفحًا لكنه لا يكتمل أبدًا

يبدأ CLI خادم استدعاء محلي قصير العمر أثناء تسجيل الدخول عبر المتصفح.

- تأكد من أن متصفحك يمكنه الوصول إلى `http://127.0.0.1:<port>/callback`.
- تحقق من قواعد جدار الحماية المحلي وVPN والوكيل إذا لم يصل الاستدعاء مطلقًا.
- في البيئات بلا واجهة رسومية، أنشئ رمز API في واجهة ClawHub على الويب وشغّل:

```bash
clawhub login --token clh_...
```

## يُرجع `whoami` أو `publish` الحالة `Unauthorized` (401)

- سجّل الدخول مرة أخرى باستخدام `clawhub login`.
- إذا كنت تستخدم مسار إعدادات مخصصًا، فتأكد من أن `CLAWHUB_CONFIG_PATH` يشير إلى
  الملف الذي يحتوي على رمزك الحالي.
- إذا كنت تستخدم رمز API، فتأكد من أنه لم يُلغَ في واجهة الويب.

## يُرجع البحث أو التثبيت `Rate limit exceeded` (429)

اقرأ معلومات إعادة المحاولة في الاستجابة:

- `Retry-After`: عدد الثواني التي يجب انتظارها قبل إعادة المحاولة.
- `RateLimit-Limit`: الحد المطبق على هذا الطلب.
- `RateLimit-Remaining`: حصتك المتبقية بدقة عندما يكون الترويس موجودًا. عند `429`، تكون `0`.
- `RateLimit-Reset` أو `X-RateLimit-Reset`: توقيت إعادة التعيين.

إذا كان العديد من المستخدمين يتشاركون عنوان IP خروج واحدًا، فقد تُستنفد حدود IP المجهول حتى عندما يرسل كل
شخص بضعة طلبات فقط. سجّل الدخول حيثما أمكن وأعد المحاولة بعد التأخير
المُبلّغ عنه.

## يفشل البحث أو التثبيت خلف وكيل

يحترم CLI متغيرات الوكيل القياسية:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

تشمل الأسماء المدعومة `HTTPS_PROXY` و`HTTP_PROXY` و`https_proxy` و
`http_proxy`.

## لا تظهر skill في البحث

- تحقق من slug الدقيق أو صفحة المالك إذا كنت تعرفها.
- تأكد من أن الإصدار عام وليس محتجزًا بسبب الفحص أو الإشراف.
- إذا كنت تملك skill، فسجّل الدخول وافحصها:

```bash
clawhub inspect @openclaw/demo
```

قد توضح التشخيصات المرئية للمالك حالة الفحص أو بوابة الرفع أو الإشراف.

## يفشل النشر لأن البيانات الوصفية المطلوبة مفقودة

بالنسبة إلى Skills، تحقق من frontmatter في `SKILL.md`. يجب التصريح بمتغيرات البيئة المطلوبة
والأدوات حتى يتمكن المستخدمون والماسحات من فهم الحزمة.

بالنسبة إلى plugins، تحقق من بيانات توافق `package.json` الوصفية. تتطلب عمليات نشر code-plugin
حقول توافق OpenClaw مثل `openclaw.compat.pluginApi` و
`openclaw.build.openclawVersion`.

عاين حمولة النشر أولًا:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## يفشل النشر بسبب خطأ في مالك GitHub أو المصدر

يستخدم ClawHub هوية GitHub وإسناد المصدر لربط الحزم
بناشريها.

- تأكد من أنك مسجّل الدخول بحساب GitHub الذي يملك الحزمة أو يمكنه نشرها.
- تحقق من أن عنوان URL للمصدر عام أو يمكن لـ ClawHub الوصول إليه.
- بالنسبة إلى مصادر GitHub، استخدم `owner/repo` أو `owner/repo@ref` أو عنوان URL كاملًا لـ GitHub.

## يفشل النشر لأن مساحة اسم مُطالب بها أو محجوزة

إذا فشل النشر لأن معرّف المالك أو مساحة اسم المؤسسة أو نطاق الحزمة أو slug الخاص بالـ skill
أو اسم الحزمة مُطالب به أو محجوز بالفعل، فتأكد أولًا من أنك
تنشر بالمالك الذي يطابق مساحة الاسم. بالنسبة إلى حزم plugins،
يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin` باسم
المالك المطابق `example-org`.

إذا كنت تعتقد أن مؤسستك أو مشروعك أو علامتك التجارية هي المالكة الشرعية لمساحة الاسم لكنك
لا تستطيع إدارة مالك ClawHub الحالي، فافتح
[قضية مطالبة بمؤسسة / مساحة اسم](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
مع دليل عام وغير حساس. راجع
[مطالبات المؤسسة ومساحة الاسم](/ar/clawhub/namespace-claims) للحصول على إرشادات الأدلة وما
يجب إبقاؤه خارج القضايا العامة.

## يقول `sync` إنه لم يتم العثور على Skills

يبحث `sync` عن مجلدات تحتوي على `SKILL.md` أو `skill.md`.

وجّهه إلى الجذور التي تريد فحصها:

```bash
clawhub sync --root /path/to/skills
```

عاين أولًا إذا لم تكن متأكدًا مما سيُنشر:

```bash
clawhub sync --all --dry-run --no-input
```

## يرفض `update` بسبب تغييرات محلية

لا تطابق الملفات المحلية أي إصدار يعرفه ClawHub. اختر أحد الخيارات:

- احتفظ بالتعديلات المحلية وتخطَّ التحديث.
- استبدلها بالإصدار المنشور:

```bash
clawhub update @openclaw/demo --force
```

- انشر نسختك المعدّلة باسم slug جديد أو fork.

## يفشل تثبيت Plugin في OpenClaw

- استخدم مصدر ClawHub صريحًا:

```bash
openclaw plugins install clawhub:<package>
```

- تحقق من صفحة تفاصيل الحزمة لمعرفة حالة الفحص وبيانات التوافق الوصفية.
- تأكد من أن إصدار OpenClaw لديك يفي بنطاق التوافق
  الذي تعلنه الحزمة.
- إذا كانت الحزمة مخفية أو محتجزة أو محظورة، فقد لا تكون قابلة للتثبيت إلى أن
  يحل المالك المشكلة.

## تفشل طلبات API العامة

- احترم ترويسات إعادة المحاولة `429` وخزّن استجابات القوائم/البحث العامة مؤقتًا.
- أعد ربط المستخدمين بقائمة ClawHub القانونية.
- لا تعكس المحتوى المخفي أو الخاص أو المحتجز أو المحظور بسبب الإشراف خارج
  سطح API العام.

راجع [HTTP API](/ar/clawhub/http-api) للحصول على تفاصيل نقاط النهاية.
