# استكشاف الأخطاء وإصلاحها

## يفتح `clawhub login` متصفحًا لكنه لا يكتمل أبدًا

تبدأ CLI خادم استدعاء محليًا قصير العمر أثناء تسجيل الدخول عبر المتصفح.

- تأكد من أن متصفحك يستطيع الوصول إلى `http://127.0.0.1:<port>/callback`.
- تحقق من قواعد جدار الحماية المحلي وVPN والوكيل إذا لم يصل الاستدعاء مطلقًا.
- في البيئات بلا واجهة رسومية، أنشئ رمز API في واجهة ويب ClawHub وشغّل:

```bash
clawhub login --token clh_...
```

## يعيد `whoami` أو `publish` القيمة `Unauthorized` (401)

- سجّل الدخول مرة أخرى باستخدام `clawhub login`.
- إذا كنت تستخدم مسار إعدادات مخصصًا، فتأكد من أن `CLAWHUB_CONFIG_PATH` يشير إلى
  الملف الذي يحتوي على رمزك الحالي.
- إذا كنت تستخدم رمز API، فتأكد من أنه لم يُلغَ في واجهة الويب.

## يعيد البحث أو التثبيت `Rate limit exceeded` (429)

اقرأ معلومات إعادة المحاولة في الاستجابة:

- `Retry-After`: عدد الثواني التي يجب انتظارها قبل إعادة المحاولة.
- `RateLimit-Remaining` و`RateLimit-Limit`: ميزانيتك الحالية.
- `RateLimit-Reset` أو `X-RateLimit-Reset`: توقيت إعادة الضبط.

إذا كان كثير من المستخدمين يتشاركون عنوان IP خروجًا واحدًا، فقد تُستنفد حدود عناوين IP المجهولة حتى عندما يرسل كل
شخص بضع طلبات فقط. سجّل الدخول حيثما أمكن وأعد المحاولة بعد
التأخير المُبلّغ عنه.

## يفشل البحث أو التثبيت خلف وكيل

تحترم CLI متغيرات الوكيل القياسية:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

تشمل الأسماء المدعومة `HTTPS_PROXY` و`HTTP_PROXY` و`https_proxy` و
`http_proxy`.

## لا تظهر مهارة في البحث

- تحقق من المعرّف النصي الدقيق أو صفحة المالك إذا كنت تعرفها.
- تأكد من أن الإصدار عام وغير محتجز بسبب الفحص أو الإشراف.
- إذا كنت تملك المهارة، فسجّل الدخول وافحصها:

```bash
clawhub inspect <skill-slug>
```

قد تشرح التشخيصات المرئية للمالك حالة الفحص أو بوابة الرفع أو الإشراف.

## يفشل النشر لأن البيانات الوصفية المطلوبة مفقودة

بالنسبة إلى المهارات، تحقق من frontmatter في `SKILL.md`. يجب التصريح بمتغيرات البيئة والأدوات
المطلوبة حتى يتمكن المستخدمون والماسحات من فهم الحزمة.

بالنسبة إلى plugins، تحقق من بيانات توافق `package.json` الوصفية. تحتاج عمليات نشر code-plugin
إلى حقول توافق OpenClaw مثل `openclaw.compat.pluginApi` و
`openclaw.build.openclawVersion`.

عاين حمولة النشر أولًا:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## يفشل النشر بسبب خطأ في مالك GitHub أو المصدر

يستخدم ClawHub هوية GitHub وإسناد المصدر لربط الحزم
بناشريها.

- تأكد من أنك مسجل الدخول بحساب GitHub الذي يملك الحزمة أو يستطيع
  نشرها.
- تحقق من أن URL المصدر عام أو يمكن لـ ClawHub الوصول إليه.
- بالنسبة إلى مصادر GitHub، استخدم `owner/repo` أو `owner/repo@ref` أو URL GitHub كاملًا.

## يقول `sync` إنه لم يتم العثور على أي مهارات

يبحث `sync` عن مجلدات تحتوي على `SKILL.md` أو `skill.md`.

وجّهه إلى الجذور التي تريد فحصها:

```bash
clawhub sync --root /path/to/skills
```

عاين أولًا إذا لم تكن متأكدًا مما سيُنشر:

```bash
clawhub sync --all --dry-run --no-input
```

## يرفض `update` بسبب تغييرات محلية

لا تطابق الملفات المحلية أي إصدار يعرفه ClawHub. اختر أحد الخيارات:

- احتفظ بالتعديلات المحلية وتخطَّ التحديث.
- اكتب فوقها بالإصدار المنشور:

```bash
clawhub update <slug> --force
```

- انشر نسختك المعدلة كمعرّف نصي جديد أو تفرّع.

## يفشل تثبيت Plugin في OpenClaw

- استخدم مصدر ClawHub صريحًا:

```bash
openclaw plugins install clawhub:<package>
```

- تحقق من صفحة تفاصيل الحزمة لمعرفة حالة الفحص وبيانات التوافق الوصفية.
- تأكد من أن إصدار OpenClaw لديك يستوفي نطاق التوافق
  المعلن للحزمة.
- إذا كانت الحزمة مخفية أو محتجزة أو محظورة، فقد لا تكون قابلة للتثبيت حتى
  يحل المالك المشكلة.

## تفشل طلبات API العامة

- احترم ترويسات إعادة المحاولة `429` وخزّن استجابات القوائم/البحث العامة مؤقتًا.
- أعد ربط المستخدمين بقائمة ClawHub الأساسية.
- لا تنسخ المحتوى المخفي أو الخاص أو المحتجز أو المحظور بالإشراف خارج
  سطح API العام.

راجع [HTTP API](/ar/clawhub/http-api) للحصول على تفاصيل نقاط النهاية.
