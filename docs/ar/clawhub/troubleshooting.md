# استكشاف الأخطاء وإصلاحها

## يفتح `clawhub login` المتصفح لكنه لا يكتمل أبدًا

يبدأ CLI خادم رد نداء محلي قصير العمر أثناء تسجيل الدخول عبر المتصفح.

- تأكد من أن متصفحك يمكنه الوصول إلى `http://127.0.0.1:<port>/callback`.
- تحقق من قواعد جدار الحماية المحلي وVPN والوكيل إذا لم يصل رد النداء أبدًا.
- في البيئات بلا واجهة رسومية، أنشئ رمز API مميزًا في واجهة ويب ClawHub ثم شغّل:

```bash
clawhub login --token clh_...
```

## يعيد `whoami` أو `publish` الخطأ `Unauthorized` (401)

- سجّل الدخول مرة أخرى باستخدام `clawhub login`.
- إذا كنت تستخدم مسار إعداد مخصصًا، فتأكد من أن `CLAWHUB_CONFIG_PATH` يشير إلى
  الملف الذي يحتوي على رمزك المميز الحالي.
- إذا كنت تستخدم رمز API مميزًا، فتأكد من أنه لم يُلغَ في واجهة الويب.

## يعيد البحث أو التثبيت `Rate limit exceeded` (429)

اقرأ معلومات إعادة المحاولة في الاستجابة:

- `Retry-After`: عدد الثواني الواجب انتظارها قبل إعادة المحاولة.
- `RateLimit-Limit`: الحد المطبق على هذا الطلب.
- `RateLimit-Remaining`: ميزانيتك المتبقية بدقة عند وجود هذا الترويس. عند `429`، تكون `0`.
- `RateLimit-Reset` أو `X-RateLimit-Reset`: توقيت إعادة الضبط.

إذا كان مستخدمون كثيرون يتشاركون عنوان IP خروج واحدًا، فقد تُستنفد حدود IP المجهولة حتى عندما يرسل كل
شخص بضعة طلبات فقط. سجّل الدخول حيثما أمكن، ثم أعد المحاولة بعد
المهلة المبلغ عنها.

## يفشل البحث أو التثبيت خلف وكيل

يحترم CLI متغيرات الوكيل القياسية:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

تشمل الأسماء المدعومة `HTTPS_PROXY` و`HTTP_PROXY` و`https_proxy` و
`http_proxy`.

## لا تظهر إحدى Skills في البحث

- تحقق من المعرّف النصي الدقيق أو صفحة المالك إذا كنت تعرفها.
- تأكد من أن الإصدار عام وليس محتجزًا بسبب الفحص أو الإشراف.
- إذا كنت تملك Skill، فسجّل الدخول وافحصها:

```bash
clawhub inspect @openclaw/demo
```

قد تفسر التشخيصات المرئية للمالك حالة الفحص أو بوابة الرفع أو الإشراف.

## يفشل النشر بسبب نقص البيانات الوصفية المطلوبة

بالنسبة إلى Skills، تحقق من frontmatter في `SKILL.md`. يجب التصريح عن متغيرات البيئة
والأدوات المطلوبة حتى يتمكن المستخدمون والماسحات من فهم الحزمة.

بالنسبة إلى plugins، تحقق من بيانات توافق التعريف في `package.json`. تحتاج عمليات نشر Plugin البرمجي
إلى حقول توافق OpenClaw مثل `openclaw.compat.pluginApi` و
`openclaw.build.openclawVersion`.

عاين حمولة النشر أولًا:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## يفشل النشر بسبب خطأ في مالك GitHub أو المصدر

يستخدم ClawHub هوية GitHub ونَسب المصدر لربط الحزم
بناشريها.

- تأكد من أنك مسجل الدخول بحساب GitHub الذي يملك الحزمة أو يستطيع نشرها.
- تحقق من أن عنوان URL للمصدر عام أو يمكن لـ ClawHub الوصول إليه.
- بالنسبة إلى مصادر GitHub، استخدم `owner/repo` أو `owner/repo@ref` أو عنوان URL كاملًا لـ GitHub.

## يفشل النشر لأن نطاقًا اسميًا مطالب به أو محجوز

إذا فشل النشر لأن معرّف المالك أو نطاق المؤسسة الاسمي أو نطاق الحزمة أو معرّف Skill
أو اسم الحزمة مطالب به أو محجوز بالفعل، فتأكد أولًا من أنك
تنشر بالمالك المطابق للنطاق الاسمي. بالنسبة إلى حزم plugin،
يجب نشر الأسماء ذات النطاق مثل `@example-org/example-plugin` باسم المالك
المطابق `example-org`.

إذا كنت تعتقد أن مؤسستك أو مشروعك أو علامتك التجارية هي مالك النطاق الاسمي الشرعي ولكنك
لا تستطيع إدارة مالك ClawHub الحالي، فافتح
[مشكلة مطالبة مؤسسة / نطاق اسمي](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
مع دليل عام غير حساس. راجع
[مطالبات المؤسسات والنطاقات الاسمية](/clawhub/namespace-claims) للحصول على إرشادات الأدلة وما
يجب إبعاده عن المشكلات العامة.

## يقول `sync` إنه لم يتم العثور على Skills

يبحث `sync` عن مجلدات تحتوي على `SKILL.md` أو `skill.md`.

وجّهه إلى الجذور التي تريد فحصها:

```bash
clawhub sync --root /path/to/skills
```

عاين أولًا إذا لم تكن متأكدًا مما سيُنشر:

```bash
clawhub sync --all --dry-run --no-input
```

## يرفض `update` بسبب تغييرات محلية

لا تطابق الملفات المحلية أي إصدار يعرفه ClawHub. اختر أحد الخيارات:

- احتفظ بالتعديلات المحلية وتجاوز التحديث.
- استبدلها بالإصدار المنشور:

```bash
clawhub update @openclaw/demo --force
```

- انشر نسختك المعدلة كمعرّف نصي جديد أو fork.

## يفشل تثبيت plugin في OpenClaw

- استخدم مصدر ClawHub صريحًا:

```bash
openclaw plugins install clawhub:<package>
```

- تحقق من صفحة تفاصيل الحزمة لمعرفة حالة الفحص وبيانات توافق التعريف.
- تأكد من أن إصدار OpenClaw لديك يفي بنطاق التوافق
  المعلن للحزمة.
- إذا كانت الحزمة مخفية أو محتجزة أو محظورة، فقد لا تكون قابلة للتثبيت حتى
  يحل المالك المشكلة.

## تفشل طلبات API العامة

- احترم ترويسات إعادة المحاولة `429` وخزّن استجابات القوائم/البحث العامة مؤقتًا.
- اربط المستخدمين مرة أخرى بقائمة ClawHub الأساسية.
- لا تعكس محتوى مخفيًا أو خاصًا أو محتجزًا أو محظورًا بالإشراف خارج
  سطح API العام.

راجع [HTTP API](/clawhub/http-api) لتفاصيل نقاط النهاية.
