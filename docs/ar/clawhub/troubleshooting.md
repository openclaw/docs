# استكشاف الأخطاء وإصلاحها

## يفتح `clawhub login` متصفحًا لكنه لا يكتمل أبدًا

تبدأ CLI خادم استدعاء محليًا قصير العمر أثناء تسجيل الدخول عبر المتصفح.

- تأكد من أن متصفحك يستطيع الوصول إلى `http://127.0.0.1:<port>/callback`.
- تحقق من قواعد جدار الحماية المحلي وVPN والوكيل إذا لم يصل الاستدعاء أبدًا.
- في البيئات عديمة الواجهة، أنشئ رمز API في واجهة ويب ClawHub وشغّل:

```bash
clawhub login --token clh_...
```

## يعيد `whoami` أو `publish` القيمة `Unauthorized` (401)

- سجّل الدخول مجددًا باستخدام `clawhub login`.
- إذا كنت تستخدم مسار إعدادات مخصصًا، فتأكد من أن `CLAWHUB_CONFIG_PATH` يشير إلى
  الملف الذي يحتوي على رمزك الحالي.
- إذا كنت تستخدم رمز API، فتأكد من أنه لم يُلغ في واجهة الويب.

## يعيد البحث أو التثبيت `Rate limit exceeded` (429)

اقرأ معلومات إعادة المحاولة في الاستجابة:

- `Retry-After`: عدد الثواني التي يجب انتظارها قبل إعادة المحاولة.
- `RateLimit-Limit`: الحد المطبق على هذا الطلب.
- `RateLimit-Remaining`: ميزانيتك المتبقية الدقيقة عند وجود هذا الترويس. عند `429`، تكون `0`.
- `RateLimit-Reset` أو `X-RateLimit-Reset`: توقيت إعادة الضبط.

إذا كان العديد من المستخدمين يشتركون في عنوان IP خروج واحد، فقد تُستنفد حدود IP
المجهول حتى عندما يرسل كل شخص بضع طلبات فقط. سجّل الدخول حيثما أمكن وأعد
المحاولة بعد التأخير المُبلّغ عنه.

## يفشل البحث أو التثبيت خلف وكيل

تحترم CLI متغيرات الوكيل القياسية:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

تشمل الأسماء المدعومة `HTTPS_PROXY` و`HTTP_PROXY` و`https_proxy` و
`http_proxy`.

## لا تظهر إحدى Skills في البحث

- تحقق من السبيكة الدقيقة أو صفحة المالك إذا كنت تعرفها.
- تأكد من أن الإصدار عام وليس موقوفًا بسبب الفحص أو الإشراف.
- إذا كنت تملك Skills، فسجّل الدخول وافحصها:

```bash
clawhub inspect @openclaw/demo
```

قد تشرح التشخيصات المرئية للمالك حالة الفحص أو بوابة الرفع أو الإشراف.

## يفشل النشر لأن البيانات الوصفية المطلوبة مفقودة

بالنسبة إلى Skills، تحقق من الواجهة الأمامية في `SKILL.md`. يجب الإعلان عن متغيرات
البيئة والأدوات المطلوبة حتى يستطيع المستخدمون والماسحات فهم الحزمة.

بالنسبة إلى Plugin، تحقق من بيانات التوافق الوصفية في `package.json`. تحتاج عمليات
نشر Plugin البرمجية إلى حقول توافق OpenClaw مثل `openclaw.compat.pluginApi` و
`openclaw.build.openclawVersion`.

عاين حمولة النشر أولًا:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## يفشل النشر بسبب خطأ في مالك GitHub أو المصدر

يستخدم ClawHub هوية GitHub ونسبة المصدر لربط الحزم بناشريها.

- تأكد من أنك مسجّل الدخول بحساب GitHub الذي يملك الحزمة أو يستطيع نشرها.
- تحقق من أن عنوان URL للمصدر عام أو يمكن لـ ClawHub الوصول إليه.
- لمصادر GitHub، استخدم `owner/repo` أو `owner/repo@ref` أو عنوان URL كاملًا لـ GitHub.

## يفشل النشر لأن مساحة أسماء محجوزة أو مُطالب بها

إذا فشل النشر لأن معرّف المالك أو مساحة أسماء المؤسسة أو نطاق الحزمة أو سبيكة Skills
أو اسم الحزمة مُطالب به أو محجوز بالفعل، فتأكد أولًا من أنك تنشر باستخدام المالك
الذي يطابق مساحة الأسماء. بالنسبة إلى حزم Plugin، يجب نشر الأسماء ذات النطاق مثل
`@example-org/example-plugin` باسم المالك المطابق `example-org`.

إذا كنت تعتقد أن مؤسستك أو مشروعك أو علامتك التجارية هي المالكة الشرعية لمساحة
الأسماء لكنك لا تستطيع إدارة مالك ClawHub الحالي، فافتح
[مشكلة مطالبة مؤسسة / مساحة أسماء](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
مع دليل عام وغير حساس. راجع
[مطالبات المؤسسات ومساحات الأسماء](/clawhub/namespace-claims) للحصول على إرشادات الأدلة وما
يجب إبقاؤه خارج المشكلات العامة.

## يقول `sync` إنه لم يتم العثور على Skills

يبحث `sync` عن مجلدات تحتوي على `SKILL.md` أو `skill.md`.

وجّهه إلى الجذور التي تريد فحصها:

```bash
clawhub sync --root /path/to/skills
```

عاين أولًا إذا لم تكن متأكدًا مما سيُنشر:

```bash
clawhub sync --all --dry-run --no-input
```

## يرفض `update` بسبب تغييرات محلية

لا تطابق الملفات المحلية أي إصدار يعرفه ClawHub. اختر واحدًا:

- احتفظ بالتعديلات المحلية وتجاوز التحديث.
- استبدلها بالإصدار المنشور:

```bash
clawhub update @openclaw/demo --force
```

- انشر نسختك المعدلة كسبيكة جديدة أو تفرع.

## يفشل تثبيت Plugin في OpenClaw

- استخدم مصدر ClawHub صريحًا:

```bash
openclaw plugins install clawhub:<package>
```

- تحقق من صفحة تفاصيل الحزمة لمعرفة حالة الفحص وبيانات التوافق الوصفية.
- تأكد من أن إصدار OpenClaw لديك يفي بنطاق التوافق المُعلن للحزمة.
- إذا كانت الحزمة مخفية أو موقوفة أو محظورة، فقد لا تكون قابلة للتثبيت حتى
  يحل المالك المشكلة.

## تفشل طلبات API العامة

- احترم ترويسات إعادة المحاولة `429` وخزّن استجابات القوائم/البحث العامة مؤقتًا.
- أعد ربط المستخدمين بالقائمة الأساسية في ClawHub.
- لا تعكس المحتوى المخفي أو الخاص أو الموقوف أو المحظور بالإشراف خارج
  سطح API العام.

راجع [HTTP API](/clawhub/http-api) للحصول على تفاصيل نقاط النهاية.
