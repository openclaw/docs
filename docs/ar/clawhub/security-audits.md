# عمليات التدقيق الأمني

تساعدك عمليات التدقيق الأمني في ClawHub على تحديد ما إذا كانت المهارة أو Plugin آمنة بما يكفي
للتثبيت. فهي تعرض ما تفعله الإصدارة، وما الصلاحيات التي تطلبها، وما إذا كان هناك
أي شيء يستحق انتباها إضافيا قبل أن تتمكن من الوصول إلى الملفات أو الحسابات أو
بيانات الاعتماد أو الكود أو الخدمات الخارجية.

تعد عمليات التدقيق إشارات أمان قوية، لكنها لا تضمن أن الإصدارة
خالية من المخاطر. استخدم حكمك دائما قبل منح وصول حساس.

راجع أيضا [الأمان](/clawhub/security)، و[الاستخدام المقبول](/ar/clawhub/acceptable-usage)،
و[الإشراف وسلامة الحساب](/clawhub/moderation).

## ما يجب التحقق منه قبل التثبيت

قبل التثبيت، راجع:

- حالة التدقيق الإجمالية
- مستوى المخاطر
- أي نتائج مدرجة
- بيانات الاعتماد أو الأذونات أو متغيرات البيئة المطلوبة
- المالك والمصدر والإصدار وسجل التغييرات والتنزيلات والنجوم وغيرها من إشارات الثقة

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالة التدقيق

تخبرك حالة التدقيق بكيفية التعامل مع نتيجة التدقيق:

| الحالة      | المعنى                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | لم يتم العثور على مشكلة ظاهرة أعلى من مستوى المخاطر المنخفضة.                                |
| `Review`    | اقرأ النتائج قبل التثبيت. قد تظل الإصدارة مشروعة. |
| `Warn`      | توخّ حذرا إضافيا. عثر ClawHub على مصدر قلق عالي التأثير أو إشارة تحذير. |
| `Malicious` | لا تثبّت.                                                           |
| `Pending`   | لم تكتمل عمليات التدقيق بعد.                                             |
| `Error`     | تعذر إكمال التدقيق.                                         |

تعد حالة `Pass` مطمئنة، لكنها لا تغني عن حكمك الشخصي. يكون هذا مهما
بشكل خاص للأدوات التي يمكنها نشر المحتوى أو تعديل البيانات أو تشغيل الأوامر أو قراءة الملفات أو
الوصول إلى أنظمة الإنتاج.

## مستوى المخاطر

يصف مستوى المخاطر نطاق التأثير: مقدار القوة التي يبدو أن الإصدارة تمتلكها إذا
استخدمتها كما هو مقصود.

| مستوى المخاطر | المعنى                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | تم العثور على قدر قليل من الصلاحيات الحساسة أو التأثير على المستخدم.                          |
| `Medium`   | تمتلك الإصدارة صلاحيات مؤثرة، مثل الوصول إلى الحسابات أو تغيير البيانات. |
| `High`     | تمتلك الإصدارة صلاحيات عالية التأثير أو نتائج خطيرة أو إشارات ضارة. |

يجيب مستوى المخاطر وحالة التدقيق عن سؤالين مختلفين:

- يسأل مستوى المخاطر: "ما مقدار القوة الموجودة هنا؟"
- تسأل حالة التدقيق: "ماذا يجب أن أفعل بهذه النتيجة؟"

على سبيل المثال، قد تعرض مهارة نشر الحالة `Review` مع مستوى مخاطر `Medium`. هذا
لا يعني أنها ضارة. بل يعني أن المهارة تبدو متوافقة مع الغرض، لكنها يمكنها
التصرف بصلاحية حساب مؤثرة.

## النتائج

توضح النتائج سبب عرض نتيجة التدقيق. تتضمن كل نتيجة عادة:

- ما تعنيه
- سبب الإشارة إليها
- محتوى المهارة أو Plugin ذي الصلة
- توصية

قد تحمل النتائج التصنيفات `Info` أو `Low` أو `Medium` أو `High` أو `Critical`. تسهم النتائج ذات
الخطورة الأعلى بقوة أكبر في مستوى المخاطر وحالة التدقيق.

تُخفى النتائج منخفضة الثقة من ملخص التدقيق العام كي تبقى الصفحة
مركزة على الأدلة المفيدة.

## ما الذي يتحقق منه ClawHub

يدقق ClawHub عناصر الإصدار المقدمة، بما في ذلك:

- تعليمات المهارة أو بيانات Plugin الوصفية
- متغيرات البيئة والأذونات المعلنة
- تعليمات التثبيت وبيانات الحزمة الوصفية
- الملفات المضمنة وبيانات ملفاتها
- بيانات التوافق والقدرات الوصفية

السؤال الرئيسي هو الاتساق: هل يتوافق الاسم والملخص والبيانات الوصفية والصلاحية
المطلوبة والمحتوى الفعلي مع ما يتوقعه المستخدمون بشكل معقول؟

السلوك القوي ليس سيئًا تلقائيًا. تحتاج كثير من الأدوات المفيدة إلى بيانات اعتماد
أو أوامر محلية أو واجهات API لمزودين أو تثبيت حزم. يتحقق التدقيق مما إذا كانت تلك
القوة متوقعة ومعلنة ومتناسبة.

ترتبط صفحات العناصر بالتدقيق الكامل على:

```text
/<owner>/skills/<slug>/security-audit
```

تجمع صفحة التدقيق بين:

1. SkillSpector
2. VirusTotal
3. تحليل المخاطر

## VirusTotal

يستخدم ClawHub VirusTotal كقياسات عن البرمجيات الخبيثة ضمن حزمة التدقيق. يُعد VirusTotal
معيارًا موثوقًا في الصناعة لسمعة الملفات وفحص البرمجيات الخبيثة، وتتيح شراكتنا
لـ ClawHub إضافة معلومات أمنية أوسع إلى مراجعة المهارات وPlugin.

يُعد VirusTotal مفيدًا بشكل خاص للعناصر الخبيثة المعروفة، ونتائج المحركات، وإشارات
السمعة التي تكمل مراجعة ClawHub الواعية بالوكلاء. عندما تتوفر أعداد محركات
الموردين، يلخصها التدقيق بلغة واضحة، مثل:

```text
62/62 vendors flagged this skill as clean.
```

أو:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

عندما لا تتوفر لدى ClawHub قياسات لأعداد الموردين يمكن تلخيصها، يقول التدقيق:

```text
No VirusTotal findings
```

يبقى VirusTotal قياسات فقط. ولا يحل محل تحليل المخاطر الخاص بـ ClawHub والواعي
بالعناصر.

## تحليل المخاطر

يُشغّل تحليل المخاطر داخليًا ClawScan، وهو نظام التدقيق الأمني الخاص بـ ClawHub.
يراجع كل إصدار بوصفه عنصرًا موجّهًا للوكلاء: التعليمات، والبيانات الوصفية،
والأذونات المعلنة، والملفات، وإشارات القدرات، وإشارات الفحص الثابت،
ونتائج SkillSpector، وقياسات VirusTotal، والسياق الذي يقدمه الناشر.
تُعد إشارات الفحص الثابت سياقًا داخليًا لهذه المراجعة؛ فهي ليست قسم تدقيق عامًا
مستقلًا ولا حكمًا يمنع التثبيت.

يستخدم تحليل المخاطر
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
كمنظور للمخاطر مثل حقن المطالبات، وإساءة استخدام الأدوات، وكشف بيانات الاعتماد،
والتنفيذ غير الآمن، وتسميم الذاكرة أو السياق، والوكالة المفرطة.

لا يتعامل ClawScan مع القدرة التي تبدو مخيفة على أنها خبيثة تلقائيًا.
بل يسأل ما إذا كانت القدرة معلنة، ومتوافقة مع الغرض، ومدعومة بحالة الاستخدام
المذكورة للإصدار.
