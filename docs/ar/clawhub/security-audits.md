# تدقيقات الأمان

تساعدك تدقيقات أمان ClawHub على تحديد ما إذا كانت المهارة أو Plugin آمنة بما يكفي
للتثبيت. فهي توضّح ما يفعله الإصدار، وما الصلاحيات التي يطلبها، وما إذا كان
هناك ما يستحق انتباهاً إضافياً قبل أن يتمكن من الوصول إلى الملفات أو الحسابات
أو بيانات الاعتماد أو الكود أو الخدمات الخارجية.

تُعد التدقيقات مؤشرات أمان قوية، لكنها ليست ضماناً بأن الإصدار
خالٍ من المخاطر. استخدم حكمك دائماً قبل منح وصول حساس.

راجع أيضاً [الأمان](/ar/clawhub/security)، و[الاستخدام المقبول](/ar/clawhub/acceptable-usage)،
و[الإشراف وسلامة الحساب](/ar/clawhub/moderation).

## ما يجب التحقق منه قبل التثبيت

قبل التثبيت، راجع:

- حالة التدقيق العامة
- مستوى المخاطر
- أي نتائج مدرجة
- بيانات الاعتماد أو الأذونات أو متغيرات البيئة المطلوبة
- المالك، والمصدر، والإصدار، وسجل التغييرات، والتنزيلات، والنجوم، وإشارات الثقة الأخرى

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالة التدقيق

تخبرك حالة التدقيق بكيفية التعامل مع نتيجة التدقيق:

| الحالة      | المعنى                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | لم يتم العثور على مشكلة ظاهرة أعلى من مستوى المخاطر المنخفضة.                                |
| `Review`    | اقرأ النتائج قبل التثبيت. قد يظل الإصدار مشروعاً. |
| `Warn`      | توخَّ حذراً إضافياً. وجد ClawHub مشكلة عالية الأثر أو إشارة تحذير. |
| `Malicious` | لا تثبّت.                                                           |
| `Pending`   | لم تكتمل التدقيقات بعد.                                             |
| `Error`     | تعذّر إكمال التدقيق.                                         |

تُعد حالة `Pass` مطمئنة، لكنها لا تغني عن حكمك الشخصي. ويكون هذا مهماً
بشكل خاص للأدوات التي يمكنها نشر المحتوى أو تعديل البيانات أو تشغيل الأوامر أو قراءة الملفات أو
الوصول إلى أنظمة الإنتاج.

## مستوى المخاطر

يصف مستوى المخاطر نطاق التأثير: مقدار القوة التي يبدو أن الإصدار يمتلكها إذا
استخدمته كما هو مقصود.

| مستوى المخاطر | المعنى                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | وُجدت صلاحيات حساسة أو آثار على المستخدم قليلة.                          |
| `Medium`   | يمتلك الإصدار صلاحيات ذات معنى، مثل الوصول إلى الحسابات أو تغيير البيانات. |
| `High`     | يمتلك الإصدار صلاحيات عالية الأثر، أو نتائج خطيرة، أو إشارات خبيثة. |

يجيب مستوى المخاطر وحالة التدقيق عن سؤالين مختلفين:

- مستوى المخاطر يسأل: "ما مقدار القوة الموجودة هنا؟"
- حالة التدقيق تسأل: "ماذا يجب أن أفعل بهذه النتيجة؟"

على سبيل المثال، قد تعرض مهارة نشر الحالة `Review` مع مستوى مخاطر `Medium`. هذا
لا يعني أنها خبيثة. بل يعني أن المهارة تبدو متوافقة مع الغرض، لكنها تستطيع
التصرف بصلاحيات حساب ذات معنى.

## النتائج

توضح النتائج سبب عرض نتيجة تدقيق معينة. تتضمن كل نتيجة عادةً:

- ما تعنيه
- سبب وضع علامة عليها
- محتوى المهارة أو Plugin ذي الصلة
- توصية

قد تُصنّف النتائج بعلامات `Info` أو `Low` أو `Medium` أو `High` أو `Critical`. وتسهم النتائج الأعلى
خطورةً بقوة أكبر في مستوى المخاطر وحالة التدقيق.

تُخفى النتائج منخفضة الثقة من ملخص التدقيق العام حتى تبقى الصفحة
مركزة على الأدلة المفيدة.

## ما الذي يفحصه ClawHub

يفحص ClawHub ملفات إصدارات النشر المقدمة، بما في ذلك:

- تعليمات المهارة أو بيانات Plugin الوصفية
- متغيرات البيئة والأذونات المعلنة
- تعليمات التثبيت وبيانات الحزمة الوصفية
- الملفات المضمنة وقوائم الملفات
- بيانات التوافق والقدرات الوصفية

السؤال الرئيسي هو الاتساق: هل يتوافق الاسم والملخص والبيانات الوصفية
والصلاحيات المطلوبة والمحتوى الفعلي مع ما يتوقعه المستخدمون على نحو معقول؟

السلوك القوي ليس سيئاً تلقائياً. تحتاج كثير من الأدوات المفيدة إلى بيانات اعتماد،
أو أوامر محلية، أو واجهات API لمزوّدين، أو تثبيت حزم. يتحقق التدقيق مما إذا كانت تلك
القوة متوقعة ومعلنة ومتناسبة.

ترتبط صفحات الملفات بالتدقيق الكامل على:

```text
/<owner>/skills/<slug>/security-audit
```

تجمع صفحة التدقيق بين:

1. SkillSpector
2. VirusTotal
3. تحليل المخاطر

## VirusTotal

يستخدم ClawHub خدمة VirusTotal كقياسات عن البرمجيات الخبيثة ضمن حزمة التدقيق. وتُعد VirusTotal
معياراً موثوقاً في القطاع لسمعة الملفات وفحص البرمجيات الخبيثة، وتتيح شراكتنا
لـ ClawHub إضافة معلومات أمنية أوسع إلى مراجعة المهارات وPlugin.

تكون VirusTotal مفيدة بشكل خاص للملفات الخبيثة المعروفة، وإصابات المحركات، و
إشارات السمعة التي تكمل مراجعة ClawHub الواعية بالعوامل. عندما تتوفر
أعداد محركات البائعين، يلخصها التدقيق بلغة واضحة، مثل:

```text
62/62 vendors flagged this skill as clean.
```

أو:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

عندما لا تتوفر لدى ClawHub قياسات عدد البائعين لتلخيصها، يقول التدقيق:

```text
No VirusTotal findings
```

تظل VirusTotal قياسات مساعدة. وهي لا تستبدل تحليل المخاطر الخاص بـ ClawHub
والمستند إلى فهم ملفات الإصدار.

## تحليل المخاطر

يعمل تحليل المخاطر داخلياً بواسطة ClawScan، وهو نظام تدقيق الأمان الخاص
بـ ClawHub. يراجع كل إصدار باعتباره ملفاً موجهاً للعوامل: التعليمات،
والبيانات الوصفية، والأذونات المعلنة، والملفات، وإشارات القدرات، وإشارات الفحص الثابت،
ونتائج SkillSpector، وقياسات VirusTotal، والسياق المقدم من الناشر.
تُعد إشارات الفحص الثابت سياقاً داخلياً لهذه المراجعة؛ وهي ليست
قسماً عاماً مستقلاً للتدقيق أو حكماً يمنع التثبيت.

يستخدم تحليل المخاطر
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
كمنظور للمخاطر مثل حقن المطالبات، وإساءة استخدام الأدوات، وانكشاف بيانات الاعتماد،
والتنفيذ غير الآمن، وتسميم الذاكرة أو السياق، والوكالة المفرطة.

لا يتعامل ClawScan مع القدرة التي تبدو مخيفة على أنها خبيثة تلقائياً.
بل يسأل ما إذا كانت القدرة معلنة، ومتوافقة مع الغرض، ومدعومة
بحالة الاستخدام المعلنة للإصدار.
