# عمليات التدقيق الأمني

تساعدك عمليات التدقيق الأمني في ClawHub على تحديد ما إذا كانت مهارة أو Plugin آمنة بما يكفي
لتثبيتها. فهي توضّح ما يفعله الإصدار، وما الصلاحيات التي يطلبها، وما إذا كان هناك ما يستحق
انتباهًا إضافيًا قبل أن يتمكن من الوصول إلى الملفات أو الحسابات أو بيانات الاعتماد أو التعليمات البرمجية أو الخدمات الخارجية.

تُعد عمليات التدقيق إشارات أمان قوية، لكنها ليست ضمانًا بأن الإصدار
خالٍ من المخاطر. استخدم حكمك دائمًا قبل منح وصول حساس.

راجع أيضًا [الأمان](/clawhub/security)، و[الاستخدام المقبول](/clawhub/acceptable-usage)،
و[الإشراف وسلامة الحساب](/clawhub/moderation).

## ما يجب التحقق منه قبل التثبيت

قبل التثبيت، راجع:

- حالة التدقيق الإجمالية
- مستوى المخاطر
- أي نتائج مدرجة
- بيانات الاعتماد أو الأذونات أو متغيرات البيئة المطلوبة
- المالك والمصدر والإصدار وسجل التغييرات والتنزيلات والنجوم وإشارات الثقة الأخرى

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالة التدقيق

تخبرك حالة التدقيق بكيفية التعامل مع نتيجة التدقيق:

| الحالة      | المعنى                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | لم تُعثر مشكلة ظاهرة أعلى من المخاطر المنخفضة.                                |
| `Review`    | اقرأ النتائج قبل التثبيت. قد يظل الإصدار مشروعًا. |
| `Warn`      | توخَّ حذرًا إضافيًا. وجد ClawHub مشكلة عالية التأثير أو إشارة تحذير. |
| `Malicious` | لا تثبّت.                                                           |
| `Pending`   | لم تكتمل عمليات التدقيق بعد.                                             |
| `Error`     | تعذّر إكمال التدقيق.                                         |

تُعد حالة `Pass` مطمئنة، لكنها لا تغني عن حكمك الشخصي. ويكون هذا مهمًا
خصوصًا للأدوات التي يمكنها نشر المحتوى أو تعديل البيانات أو تشغيل الأوامر أو قراءة الملفات أو
الوصول إلى أنظمة الإنتاج.

## مستوى المخاطر

يصف مستوى المخاطر نطاق التأثير: مقدار القوة التي يبدو أن الإصدار يمتلكها إذا
استخدمته كما هو مقصود.

| مستوى المخاطر | المعنى                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | لم تُعثر صلاحيات حساسة أو تأثير كبير على المستخدم.                          |
| `Medium`   | يمتلك الإصدار صلاحيات ذات معنى، مثل الوصول إلى الحساب أو تغيير البيانات. |
| `High`     | يمتلك الإصدار صلاحيات عالية التأثير أو نتائج خطيرة أو إشارات ضارة. |

يجيب مستوى المخاطر وحالة التدقيق عن سؤالين مختلفين:

- يسأل مستوى المخاطر: "ما مقدار القوة الموجودة هنا؟"
- تسأل حالة التدقيق: "ماذا ينبغي أن أفعل بهذه النتيجة؟"

على سبيل المثال، قد تعرض مهارة نشر حالة `Review` مع مخاطر `Medium`. هذا
لا يعني أنها ضارة. بل يعني أن المهارة تبدو متوافقة مع الغرض، لكنها تستطيع
التصرف بصلاحيات حساب ذات معنى.

## النتائج

تشرح النتائج سبب عرض نتيجة التدقيق. تتضمن كل نتيجة عادةً:

- ما تعنيه
- سبب وضع علامة عليها
- محتوى المهارة أو Plugin ذي الصلة
- توصية

قد تُصنّف النتائج باسم `Info` أو `Low` أو `Medium` أو `High` أو `Critical`. تساهم النتائج الأعلى
خطورةً بدرجة أكبر في مستوى المخاطر وحالة التدقيق.

تُخفى النتائج منخفضة الثقة من ملخص التدقيق العام حتى تظل الصفحة
مركزة على الأدلة المفيدة.

## ما الذي يتحقق منه ClawHub

يدقق ClawHub عناصر الإصدار المقدّمة، بما في ذلك:

- تعليمات المهارة أو بيانات Plugin الوصفية
- متغيرات البيئة والأذونات المعلنة
- تعليمات التثبيت وبيانات الحزمة الوصفية
- الملفات المضمّنة وبيانات الملفات
- بيانات التوافق والقدرات الوصفية

السؤال الرئيسي هو الاتساق: هل يتوافق الاسم والملخص والبيانات الوصفية والصلاحيات
المطلوبة والمحتوى الفعلي مع ما يتوقعه المستخدمون بشكل معقول؟

السلوك القوي ليس سيئًا تلقائيًا. تحتاج أدوات مفيدة كثيرة إلى بيانات اعتماد
أو أوامر محلية أو واجهات API لمزوّدي الخدمة أو تثبيت حزم. يتحقق التدقيق مما إذا كانت تلك
القوة متوقعة ومفصحًا عنها ومتناسبة.

ترتبط صفحات العناصر بالتدقيق الكامل على:

```text
/<owner>/skills/<slug>/security-audit
```

تجمع صفحة التدقيق بين:

1. SkillSpector
2. VirusTotal
3. تحليل المخاطر

## VirusTotal

يستخدم ClawHub ‏VirusTotal بوصفه قياسًا عن بُعد للبرمجيات الضارة ضمن حزمة التدقيق. VirusTotal هو
معيار صناعي موثوق لسمعة الملفات وفحص البرمجيات الضارة، وتتيح شراكتنا
لـ ClawHub إضافة معلومات أمان أوسع إلى مراجعة المهارات وPlugins.

يكون VirusTotal مفيدًا خصوصًا للعناصر الضارة المعروفة، وإصابات المحركات، و
إشارات السمعة التي تكمل مراجعة ClawHub الواعية بالوكلاء. عندما تتوفر
أعداد محركات المورّدين، يلخصها التدقيق بلغة واضحة، مثل:

```text
62/62 vendors flagged this skill as clean.
```

أو:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

عندما لا تتوفر لدى ClawHub بيانات قياس عن بُعد لأعداد المورّدين يمكن تلخيصها، يقول التدقيق:

```text
No VirusTotal findings
```

يبقى VirusTotal قياسًا عن بُعد. وهو لا يستبدل تحليل المخاطر الخاص بـ ClawHub
والواعي بالعناصر.

## تحليل المخاطر

يُشغَّل تحليل المخاطر داخليًا بواسطة ClawScan، وهو نظام التدقيق الأمني الخاص
بـ ClawHub. يراجع كل إصدار بوصفه عنصرًا موجّهًا للوكلاء: التعليمات،
والبيانات الوصفية، والأذونات المعلنة، والملفات، وإشارات القدرات، وإشارات الفحص الثابت،
ونتائج SkillSpector، وقياسات VirusTotal عن بُعد، والسياق المقدّم من الناشر.
تُعد إشارات الفحص الثابت سياقًا داخليًا لهذه المراجعة؛ وليست
قسم تدقيق عامًا مستقلًا أو حكمًا يمنع التثبيت.

يستخدم تحليل المخاطر
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
كمنظور للمخاطر مثل حقن المطالبات، وإساءة استخدام الأدوات، وكشف بيانات الاعتماد،
والتنفيذ غير الآمن، وتسميم الذاكرة أو السياق، والوكالة المفرطة.

لا يتعامل ClawScan مع قدرة تبدو مخيفة على أنها ضارة تلقائيًا.
بل يسأل ما إذا كانت القدرة مفصحًا عنها، ومتوافقة مع الغرض، ومدعومة
بحالة الاستخدام المعلنة للإصدار.
