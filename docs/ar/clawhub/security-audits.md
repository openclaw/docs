# تدقيقات الأمان

تساعدك تدقيقات أمان ClawHub على تحديد ما إذا كانت المهارة أو Plugin آمنة بما يكفي
لتثبيتها. فهي توضّح ما يفعله الإصدار، وما الصلاحيات التي يطلبها، وما إذا كان
هناك ما يستحق اهتمامًا إضافيًا قبل أن يتمكن من الوصول إلى الملفات أو الحسابات أو
بيانات الاعتماد أو التعليمات البرمجية أو الخدمات الخارجية.

تُعد التدقيقات مؤشرات أمان قوية، لكنها لا تضمن أن الإصدار خالٍ من المخاطر.
استخدم حكمك دائمًا قبل منح وصول حساس.

راجع أيضًا [الأمان](/ar/clawhub/security)، و[الاستخدام المقبول](/ar/clawhub/acceptable-usage)،
و[الإشراف وسلامة الحساب](/ar/clawhub/moderation).

## ما يجب التحقق منه قبل التثبيت

قبل التثبيت، راجع:

- حالة التدقيق العامة
- مستوى المخاطر
- أي نتائج مدرجة
- بيانات الاعتماد أو الأذونات أو متغيرات البيئة المطلوبة
- المالك، والمصدر، والإصدار، وسجل التغييرات، والتنزيلات، والنجوم، ومؤشرات الثقة الأخرى

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالة التدقيق

تخبرك حالة التدقيق بكيفية التعامل مع نتيجة التدقيق:

| الحالة      | المعنى                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | لم تُعثر على مشكلة ظاهرة أعلى من المخاطر المنخفضة.                                |
| `Review`    | اقرأ النتائج قبل التثبيت. قد يظل الإصدار مشروعًا. |
| `Warn`      | توخَّ حذرًا إضافيًا. عثر ClawHub على مصدر قلق عالي التأثير أو إشارة تحذيرية. |
| `Malicious` | لا تثبّت.                                                           |
| `Pending`   | لم تنتهِ التدقيقات بعد.                                             |
| `Error`     | تعذّر إكمال التدقيق.                                         |

تُعد حالة `Pass` مطمئنة، لكنها لا تغني عن حكمك الشخصي. وهذا مهم أكثر
للأدوات التي يمكنها نشر المحتوى أو تعديل البيانات أو تشغيل الأوامر أو قراءة الملفات أو
الوصول إلى أنظمة الإنتاج.

## مستوى المخاطر

يصف مستوى المخاطر نطاق التأثير: مقدار القوة التي يبدو أن الإصدار يمتلكها إذا
استخدمته كما هو مقصود.

| مستوى المخاطر | المعنى                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | لم تُعثر إلا على صلاحيات حساسة قليلة أو تأثير محدود على المستخدم.                          |
| `Medium`   | يمتلك الإصدار صلاحيات ذات معنى، مثل الوصول إلى الحساب أو تغييرات البيانات. |
| `High`     | يمتلك الإصدار صلاحيات عالية التأثير، أو نتائج خطيرة، أو إشارات خبيثة. |

يجيب مستوى المخاطر وحالة التدقيق عن سؤالين مختلفين:

- يسأل مستوى المخاطر: "ما مقدار القوة الموجودة هنا؟"
- تسأل حالة التدقيق: "ماذا يجب أن أفعل بهذه النتيجة؟"

على سبيل المثال، قد تعرض مهارة نشر حالة `Review` مع خطر `Medium`. وهذا
لا يعني أنها خبيثة. بل يعني أن المهارة تبدو متوافقة مع الغرض، لكنها تستطيع
التصرف بصلاحيات حساب ذات معنى.

## النتائج

تشرح النتائج سبب عرض نتيجة تدقيق معينة. تتضمن كل نتيجة عادةً:

- ما تعنيه
- سبب وضع علامة عليها
- محتوى المهارة أو Plugin ذي الصلة
- توصية

قد تُصنّف النتائج بعلامات `Info` أو `Low` أو `Medium` أو `High` أو `Critical`. وتساهم
النتائج الأعلى خطورة بقوة أكبر في مستوى المخاطر وحالة التدقيق.

تُخفى النتائج منخفضة الثقة من ملخص التدقيق العام حتى تظل الصفحة
مركّزة على الأدلة المفيدة.

## ما الذي يتحقق منه ClawHub

يدقق ClawHub مصنوعات الإصدار المقدمة، بما في ذلك:

- تعليمات المهارة أو بيانات Plugin الوصفية
- متغيرات البيئة والأذونات المعلنة
- تعليمات التثبيت وبيانات الحزمة الوصفية
- الملفات المضمّنة وبيانات الملفات
- بيانات التوافق والإمكانات الوصفية

السؤال الرئيسي هو الاتساق: هل يتوافق الاسم والملخص والبيانات الوصفية والصلاحيات المطلوبة
والمحتوى الفعلي مع ما يتوقعه المستخدمون على نحو معقول؟

السلوك القوي ليس سيئًا تلقائيًا. تحتاج كثير من الأدوات المفيدة إلى بيانات اعتماد،
أوامر محلية، أو واجهات API لمزوّدين، أو تثبيت حزم. يتحقق التدقيق مما إذا كانت هذه
القوة متوقعة ومعلنة ومتناسبة.

ترتبط صفحات المصنوعات بالتدقيق الكامل على:

```text
/<owner>/skills/<slug>/security-audit
```

تجمع صفحة التدقيق بين:

1. SkillSpector
2. VirusTotal
3. تحليل المخاطر

## VirusTotal

يستخدم ClawHub VirusTotal كقياسات عن البرمجيات الخبيثة ضمن حزمة التدقيق. يُعد VirusTotal
معيارًا موثوقًا في القطاع لسمعة الملفات وفحص البرمجيات الخبيثة، وتتيح شراكتنا
لـ ClawHub إضافة ذكاء أمني أوسع إلى مراجعة المهارات وPlugin.

يكون VirusTotal مفيدًا خصوصًا للمصنوعات الخبيثة المعروفة، ومطابقات المحركات، و
إشارات السمعة التي تكمل مراجعة ClawHub المدركة للوكلاء. عندما تكون أعداد محركات
المورّدين متاحة، يلخصها التدقيق بلغة واضحة، مثل:

```text
62/62 vendors flagged this skill as clean.
```

أو:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

عندما لا تتوفر لدى ClawHub قياسات أعداد المورّدين لتلخيصها، يقول التدقيق:

```text
No VirusTotal findings
```

يبقى VirusTotal قياسات مساعدة. ولا يحل محل تحليل المخاطر الخاص بـ ClawHub
والمدرك للمصنوعات.

## تحليل المخاطر

يعمل تحليل المخاطر داخليًا بواسطة ClawScan، وهو نظام تدقيق الأمان الخاص بـ ClawHub.
يراجع كل إصدار باعتباره مصنوعة موجهة إلى الوكيل: التعليمات،
والبيانات الوصفية، والأذونات المعلنة، والملفات، وإشارات الإمكانات، وإشارات الفحص الساكن،
ونتائج SkillSpector، وقياسات VirusTotal، والسياق المقدم من الناشر.
إشارات الفحص الساكن هي سياق داخلي لهذه المراجعة؛ وليست
قسم تدقيق عام مستقلًا أو حكمًا يمنع التثبيت.

يستخدم تحليل المخاطر
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
كعدسة للمخاطر مثل حقن الموجّهات، وإساءة استخدام الأدوات، وكشف بيانات الاعتماد،
والتنفيذ غير الآمن، وتسميم الذاكرة أو السياق، والوكالة المفرطة.

لا يتعامل ClawScan مع الإمكانية ذات المظهر المخيف على أنها خبيثة تلقائيًا.
بل يسأل ما إذا كانت الإمكانية معلنة، ومتوافقة مع الغرض، ومدعومة بحالة الاستخدام
المذكورة للإصدار.
