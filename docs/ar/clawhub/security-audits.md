# عمليات التدقيق الأمني

تساعدك عمليات التدقيق الأمني في ClawHub على تحديد ما إذا كانت المهارة أو الإضافة آمنة بما يكفي
لتثبيتها. فهي توضّح ما يفعله الإصدار، وما الصلاحيات التي يطلبها، وما إذا كان هناك
أي شيء يستحق انتباهاً إضافياً قبل أن يتمكن من الوصول إلى الملفات أو الحسابات أو
بيانات الاعتماد أو الكود أو الخدمات الخارجية.

تُعد عمليات التدقيق إشارات أمان قوية، لكنها لا تضمن أن الإصدار
خالٍ من المخاطر. استخدم حكمك دائماً قبل منح وصول حساس.

راجع أيضاً [الأمان](/clawhub/security)، و[الاستخدام المقبول](/ar/clawhub/acceptable-usage)،
و[الإشراف وسلامة الحساب](/clawhub/moderation).

## ما يجب التحقق منه قبل التثبيت

قبل التثبيت، راجع:

- حالة التدقيق العامة
- مستوى المخاطر
- أي نتائج مدرجة
- بيانات الاعتماد أو الأذونات أو متغيرات البيئة المطلوبة
- المالك، والمصدر، والإصدار، وسجل التغييرات، والتنزيلات، والنجوم، وإشارات الثقة الأخرى

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالة التدقيق

تخبرك حالة التدقيق بكيفية التعامل مع نتيجة التدقيق:

| الحالة      | المعنى                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | لم تُعثر مشكلة ظاهرة أعلى من مستوى المخاطر المنخفض.                                |
| `Review`    | اقرأ النتائج قبل التثبيت. قد يظل الإصدار مشروعاً. |
| `Warn`      | توخّ حذراً إضافياً. وجد ClawHub مصدر قلق عالي الأثر أو إشارة تحذير. |
| `Malicious` | لا تثبّت.                                                           |
| `Pending`   | لم تكتمل عمليات التدقيق بعد.                                             |
| `Error`     | تعذّر إكمال التدقيق.                                         |

حالة `Pass` مطمئنة، لكنها لا تغني عن حكمك الشخصي. ويكون هذا مهماً
بشكل خاص للأدوات التي يمكنها نشر المحتوى أو تعديل البيانات أو تشغيل الأوامر أو قراءة الملفات أو
الوصول إلى أنظمة الإنتاج.

## مستوى المخاطر

يصف مستوى المخاطر نطاق التأثير: مقدار السلطة التي يبدو أن الإصدار يمتلكها إذا
استخدمته كما هو مقصود.

| مستوى المخاطر | المعنى                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | وُجدت سلطة حساسة محدودة أو تأثير محدود على المستخدم.                          |
| `Medium`   | يمتلك الإصدار سلطة مؤثرة، مثل الوصول إلى الحسابات أو تغيير البيانات. |
| `High`     | يمتلك الإصدار سلطة عالية الأثر، أو نتائج خطيرة، أو إشارات ضارة. |

يجيب مستوى المخاطر وحالة التدقيق عن سؤالين مختلفين:

- يسأل مستوى المخاطر: "ما مقدار السلطة الموجودة هنا؟"
- تسأل حالة التدقيق: "ماذا ينبغي أن أفعل بهذه النتيجة؟"

على سبيل المثال، قد تعرض مهارة نشر حالة `Review` مع مخاطر `Medium`. وهذا
لا يعني أنها ضارة. بل يعني أن المهارة تبدو متوافقة مع الغرض، لكنها تستطيع
التصرف بسلطة حساب مؤثرة.

## النتائج

توضح النتائج سبب عرض نتيجة التدقيق. يتضمن كل اكتشاف عادةً:

- ما يعنيه
- سبب وضع علامة عليه
- محتوى المهارة أو الإضافة ذي الصلة
- توصية

قد تُوسم النتائج بـ `Info` أو `Low` أو `Medium` أو `High` أو `Critical`. تساهم النتائج الأعلى
خطورة بشكل أقوى في مستوى المخاطر وحالة التدقيق.

تُخفى النتائج منخفضة الثقة من الملخص العام للتدقيق حتى تظل الصفحة
مركزة على الأدلة المفيدة.

## ما الذي يتحقق منه ClawHub

يدقق ClawHub في عناصر الإصدار المقدمة، بما في ذلك:

- تعليمات المهارة أو بيانات تعريف الإضافة
- متغيرات البيئة والأذونات المعلنة
- تعليمات التثبيت وبيانات تعريف الحزمة
- الملفات المضمنة وبيانات الملفات
- بيانات تعريف التوافق والقدرات

السؤال الرئيسي هو الاتساق: هل يتوافق الاسم والملخص وبيانات التعريف والسلطة
المطلوبة والمحتوى الفعلي مع ما يتوقعه المستخدمون بشكل معقول؟

السلوك القوي ليس سيئاً تلقائياً. تحتاج كثير من الأدوات المفيدة إلى بيانات اعتماد
أو أوامر محلية أو واجهات API لمزوّدين أو تثبيت حزم. يتحقق التدقيق مما إذا كانت تلك
القوة متوقعة ومعلنة ومتناسبة.

ترتبط صفحات العناصر بالتدقيق الكامل على:

```text
/<owner>/skills/<slug>/security-audit
```

تجمع صفحة التدقيق بين:

1. SkillSpector
2. VirusTotal
3. تحليل المخاطر

## VirusTotal

يستخدم ClawHub VirusTotal كقياسات عن البرمجيات الخبيثة ضمن منظومة التدقيق. يُعد VirusTotal
معياراً موثوقاً في الصناعة لسمعة الملفات وفحص البرمجيات الخبيثة، وتتيح
شراكتنا لـ ClawHub إضافة معلومات أمنية أوسع إلى مراجعة المهارات والإضافات.

يكون VirusTotal مفيداً بشكل خاص للعناصر الضارة المعروفة، وإشارات محركات الفحص، و
إشارات السمعة التي تكمل مراجعة ClawHub المدركة للوكلاء. عندما تتوفر
أعداد محركات المورّدين، يلخصها التدقيق بلغة واضحة، مثل:

```text
62/62 vendors flagged this skill as clean.
```

أو:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

عندما لا تتوفر لدى ClawHub قياسات أعداد المورّدين لتلخيصها، يقول التدقيق:

```text
No VirusTotal findings
```

يبقى VirusTotal مصدراً للقياسات. ولا يحل محل تحليل المخاطر
الخاص بـ ClawHub والمدرك للعناصر.

## تحليل المخاطر

يعمل تحليل المخاطر داخلياً بواسطة ClawScan، وهو نظام التدقيق الأمني
الخاص بـ ClawHub. يراجع كل إصدار بوصفه عنصراً موجهاً للوكلاء: التعليمات،
وبيانات التعريف، والأذونات المعلنة، والملفات، وإشارات القدرات، وإشارات الفحص الثابت،
ونتائج SkillSpector، وقياسات VirusTotal، والسياق الذي يقدمه الناشر.
إشارات الفحص الثابت هي سياق داخلي لهذه المراجعة؛ وليست
قسماً عاماً مستقلاً للتدقيق أو حكماً يمنع التثبيت.

يستخدم تحليل المخاطر
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
كمنظور للمخاطر مثل حقن المطالبات، وإساءة استخدام الأدوات، وكشف بيانات الاعتماد،
والتنفيذ غير الآمن، وتسميم الذاكرة أو السياق، والوكالة المفرطة.

لا يتعامل ClawScan مع القدرة التي تبدو مخيفة على أنها ضارة تلقائياً.
بل يسأل ما إذا كانت القدرة معلنة ومتوافقة مع الغرض ومدعومة
بحالة الاستخدام المصرح بها للإصدار.
