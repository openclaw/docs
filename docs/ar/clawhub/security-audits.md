# تدقيقات الأمان

تساعدك تدقيقات أمان ClawHub على تحديد ما إذا كانت المهارة أو Plugin آمنة بما يكفي
للتثبيت. فهي تعرض ما يفعله الإصدار، وما الصلاحيات التي يطلبها، وما إذا كان أي شيء
يستحق انتباها إضافيا قبل أن يتمكن من الوصول إلى الملفات أو الحسابات أو
بيانات الاعتماد أو التعليمات البرمجية أو الخدمات الخارجية.

تعد التدقيقات إشارات أمان قوية، لكنها ليست ضمانا بأن الإصدار
خال من المخاطر. استخدم حكمك دائما قبل منح وصول حساس.

راجع أيضا [الأمان](/clawhub/security)، و[الاستخدام المقبول](/clawhub/acceptable-usage)،
و[الإشراف وسلامة الحساب](/clawhub/moderation).

## ما يجب التحقق منه قبل التثبيت

قبل التثبيت، راجع:

- حالة التدقيق الإجمالية
- مستوى المخاطر
- أي نتائج مدرجة
- بيانات الاعتماد أو الأذونات أو متغيرات البيئة المطلوبة
- المالك، والمصدر، والإصدار، وسجل التغييرات، والتنزيلات، والنجوم، وإشارات الثقة الأخرى

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالة التدقيق

تخبرك حالة التدقيق بكيفية التعامل مع نتيجة التدقيق:

| الحالة      | المعنى                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | لم يتم العثور على مشكلة ظاهرة تتجاوز المخاطر المنخفضة.                                |
| `Review`    | اقرأ النتائج قبل التثبيت. قد يظل الإصدار مشروعا. |
| `Warn`      | توخ حذرا إضافيا. وجد ClawHub مصدر قلق عالي التأثير أو إشارة تحذير. |
| `Malicious` | لا تثبّت.                                                           |
| `Pending`   | لم تنته التدقيقات بعد.                                             |
| `Error`     | تعذر إكمال التدقيق.                                         |

تعد حالة `Pass` مطمئنة، لكنها لا تحل محل حكمك الشخصي. ويكون هذا مهما
بشكل خاص للأدوات التي يمكنها نشر المحتوى، أو تعديل البيانات، أو تشغيل الأوامر، أو قراءة الملفات، أو
الوصول إلى أنظمة الإنتاج.

## مستوى المخاطر

يصف مستوى المخاطر نطاق التأثير: مقدار القوة التي يبدو أن الإصدار يمتلكها إذا
استخدمته كما هو مقصود.

| مستوى المخاطر | المعنى                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | تم العثور على قدر ضئيل من الصلاحيات الحساسة أو التأثير على المستخدم.                          |
| `Medium`   | يمتلك الإصدار صلاحيات ذات أهمية، مثل الوصول إلى الحسابات أو تغيير البيانات. |
| `High`     | يمتلك الإصدار صلاحيات عالية التأثير، أو نتائج شديدة، أو إشارات خبيثة. |

يجيب مستوى المخاطر وحالة التدقيق عن سؤالين مختلفين:

- يسأل مستوى المخاطر: "ما مقدار القوة هنا؟"
- تسأل حالة التدقيق: "ماذا ينبغي أن أفعل بهذه النتيجة؟"

على سبيل المثال، قد تعرض مهارة نشر حالة `Review` مع مخاطر `Medium`. هذا
لا يعني أنها خبيثة. بل يعني أن المهارة تبدو متوافقة مع الغرض، لكنها تستطيع
التصرف بصلاحيات حساب ذات أهمية.

## النتائج

تشرح النتائج سبب عرض نتيجة التدقيق. يتضمن كل استنتاج عادة:

- ما يعنيه
- سبب وضع علامة عليه
- محتوى المهارة أو Plugin ذي الصلة
- توصية

قد تحمل النتائج تسميات `Info` أو `Low` أو `Medium` أو `High` أو `Critical`. تساهم النتائج الأعلى
خطورة بقوة أكبر في مستوى المخاطر وحالة التدقيق.

تُخفى النتائج منخفضة الثقة من ملخص التدقيق العام حتى تظل الصفحة
مركزة على الأدلة المفيدة.

## ما الذي يتحقق منه ClawHub

يدقق ClawHub في عناصر الإصدار المقدمة، بما في ذلك:

- تعليمات المهارة أو بيانات Plugin الوصفية
- متغيرات البيئة والأذونات المعلنة
- تعليمات التثبيت وبيانات الحزمة الوصفية
- الملفات المضمنة وبيانات ملفات التعريف
- بيانات التوافق والقدرات الوصفية

السؤال الرئيسي هو الاتساق: هل يتوافق الاسم والملخص والبيانات الوصفية
والصلاحيات المطلوبة والمحتوى الفعلي مع ما يتوقعه المستخدمون بشكل معقول؟

السلوك القوي ليس سيئا تلقائيا. تحتاج كثير من الأدوات المفيدة إلى بيانات اعتماد،
أو أوامر محلية، أو واجهات API للمزودين، أو تثبيت حزم. يتحقق التدقيق مما إذا كانت تلك
القوة متوقعة ومفصحا عنها ومتناسبة.

ترتبط صفحات العناصر بالتدقيق الكامل على:

```text
/<owner>/skills/<slug>/security-audit
```

تجمع صفحة التدقيق بين:

1. SkillSpector
2. VirusTotal
3. تحليل المخاطر

## VirusTotal

يستخدم ClawHub VirusTotal كقياسات عن البرمجيات الضارة ضمن حزمة التدقيق. يعد VirusTotal
معيارا موثوقا في المجال لسمعة الملفات وفحص البرمجيات الضارة، وتتيح شراكتنا
لـ ClawHub إضافة معلومات أمان أوسع إلى مراجعة المهارات وPlugins.

يكون VirusTotal مفيدا بشكل خاص للعناصر الخبيثة المعروفة، وإصابات المحركات، و
إشارات السمعة التي تكمل مراجعة ClawHub الواعية بالوكلاء. عندما تكون أعداد
محركات المورّدين متاحة، يلخصها التدقيق بلغة واضحة، مثل:

```text
62/62 vendors flagged this skill as clean.
```

أو:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

عندما لا تتوفر لدى ClawHub قياسات أعداد المورّدين لتلخيصها، يقول التدقيق:

```text
No VirusTotal findings
```

يبقى VirusTotal قياسات فقط. ولا يحل محل تحليل المخاطر الخاص بـ ClawHub
والواعي بالعناصر.

## تحليل المخاطر

يعمل تحليل المخاطر داخليا بواسطة ClawScan، وهو نظام تدقيق الأمان الخاص بـ ClawHub.
يراجع كل إصدار كعنصر موجه إلى الوكيل: التعليمات،
والبيانات الوصفية، والأذونات المعلنة، والملفات، وإشارات القدرات، وإشارات الفحص الثابت،
ونتائج SkillSpector، وقياسات VirusTotal، والسياق الذي يقدمه الناشر.
إشارات الفحص الثابت هي سياق داخلي لهذه المراجعة؛ وهي ليست
قسما عاما مستقلا في التدقيق أو حكما يمنع التثبيت.

يستخدم تحليل المخاطر
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
كمنظور للمخاطر مثل حقن الموجهات، وإساءة استخدام الأدوات، وانكشاف بيانات الاعتماد،
والتنفيذ غير الآمن، وتسميم الذاكرة أو السياق، والوكالة المفرطة.

لا يتعامل ClawScan مع القدرة التي تبدو مخيفة على أنها خبيثة تلقائيا.
بل يسأل ما إذا كانت القدرة مفصحا عنها، ومتوافقة مع الغرض، ومدعومة
بحالة الاستخدام المعلنة للإصدار.
