# تدقيقات الأمان

تساعدك تدقيقات أمان ClawHub على تحديد ما إذا كانت المهارة أو الإضافة آمنة بما يكفي
لتثبيتها. فهي تعرض ما يفعله الإصدار، وما الصلاحيات التي يطلبها، وما إذا كان
أي شيء يستحق اهتماما إضافيا قبل أن يتمكن من الوصول إلى الملفات أو الحسابات أو
بيانات الاعتماد أو الشيفرة أو الخدمات الخارجية.

تدقيقات الأمان إشارات أمان قوية، لكنها ليست ضمانا بأن الإصدار
خال من المخاطر. استخدم حكمك دائما قبل منح صلاحيات وصول حساسة.

انظر أيضا [الأمان](/clawhub/security)، و[الاستخدام المقبول](/clawhub/acceptable-usage)،
و[الإشراف وسلامة الحساب](/clawhub/moderation).

## ما يجب التحقق منه قبل التثبيت

قبل التثبيت، راجع:

- حالة التدقيق الإجمالية
- مستوى المخاطر
- أي نتائج مدرجة
- بيانات الاعتماد أو الأذونات أو متغيرات البيئة المطلوبة
- المالك، والمصدر، والإصدار، وسجل التغييرات، والتنزيلات، والنجوم، وإشارات الثقة الأخرى

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالة التدقيق

تخبرك حالة التدقيق بكيفية التعامل مع نتيجة التدقيق:

| الحالة      | المعنى                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | لم يتم العثور على مشكلة ظاهرة تتجاوز المخاطر المنخفضة.                                |
| `Review`    | اقرأ النتائج قبل التثبيت. قد يظل الإصدار مشروعا. |
| `Warn`      | توخَّ حذرا إضافيا. وجد ClawHub مصدر قلق عالي التأثير أو إشارة تحذير. |
| `Malicious` | لا تثبّت.                                                           |
| `Pending`   | لم تنته التدقيقات بعد.                                             |
| `Error`     | تعذر إكمال التدقيق.                                         |

وجود `Pass` مطمئن، لكنه لا يغني عن حكمك الشخصي. يهم هذا
بشكل خاص للأدوات التي يمكنها نشر المحتوى، أو تعديل البيانات، أو تشغيل الأوامر، أو قراءة الملفات، أو
الوصول إلى أنظمة الإنتاج.

## مستوى المخاطر

يصف مستوى المخاطر نطاق التأثير: مقدار السلطة التي يبدو أن الإصدار يمتلكها إذا
استخدمته كما هو مقصود.

| مستوى المخاطر | المعنى                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | تم العثور على قدر ضئيل من الصلاحيات الحساسة أو تأثير المستخدم.                          |
| `Medium`   | يمتلك الإصدار صلاحيات ذات شأن، مثل الوصول إلى الحسابات أو تغيير البيانات. |
| `High`     | يمتلك الإصدار صلاحيات عالية التأثير، أو نتائج خطيرة، أو إشارات خبيثة. |

يجيب مستوى المخاطر وحالة التدقيق عن سؤالين مختلفين:

- يسأل مستوى المخاطر: "ما مقدار السلطة الموجودة هنا؟"
- تسأل حالة التدقيق: "ماذا ينبغي أن أفعل بهذه النتيجة؟"

على سبيل المثال، قد تعرض مهارة نشر حالة `Review` مع خطر `Medium`. هذا
لا يعني أنها خبيثة. بل يعني أن المهارة تبدو متوافقة مع غرضها، لكنها تستطيع
التصرف بصلاحيات حساب ذات شأن.

## النتائج

توضح النتائج سبب عرض نتيجة التدقيق. تتضمن كل نتيجة عادة:

- ما تعنيه
- سبب وضع علامة عليها
- محتوى المهارة أو الإضافة ذي الصلة
- توصية

قد تحمل النتائج تسميات `Info` أو `Low` أو `Medium` أو `High` أو `Critical`. تسهم النتائج
الأعلى شدة بقوة أكبر في مستوى المخاطر وحالة التدقيق.

تُخفى النتائج منخفضة الثقة من الملخص العام للتدقيق حتى تبقى الصفحة
مركزة على الأدلة المفيدة.

## ما الذي يتحقق منه ClawHub

يدقق ClawHub عناصر الإصدار المقدمة، بما في ذلك:

- تعليمات المهارة أو بيانات الإضافة الوصفية
- متغيرات البيئة والأذونات المعلنة
- تعليمات التثبيت وبيانات الحزمة الوصفية
- الملفات المضمنة وبيانات قوائم الملفات
- بيانات التوافق والقدرات الوصفية

السؤال الرئيسي هو الاتساق: هل يتوافق الاسم، والملخص، والبيانات الوصفية، والصلاحيات
المطلوبة، والمحتوى الفعلي مع ما يتوقعه المستخدمون على نحو معقول؟

السلوك القوي ليس سيئا تلقائيا. تحتاج أدوات مفيدة كثيرة إلى بيانات اعتماد،
أو أوامر محلية، أو واجهات API لمزوّدين، أو تثبيت حزم. يتحقق التدقيق مما إذا كانت تلك
السلطة متوقعة، ومعلنة، ومتناسبة.

ترتبط صفحات العناصر بالتدقيق الكامل على:

```text
/<owner>/skills/<slug>/security-audit
```

تجمع صفحة التدقيق بين:

1. SkillSpector
2. VirusTotal
3. تحليل المخاطر

## VirusTotal

يستخدم ClawHub خدمة VirusTotal كمؤشرات قياس للبرمجيات الخبيثة ضمن حزمة التدقيق. تعد VirusTotal
معيارا موثوقا في الصناعة لسمعة الملفات وفحص البرمجيات الخبيثة، وتتيح
شراكتنا لـ ClawHub إضافة معلومات أمان أوسع إلى مراجعة المهارات والإضافات.

تكون VirusTotal مفيدة خصوصا للعناصر الخبيثة المعروفة، وإصابات المحركات، وإشارات
السمعة التي تكمل مراجعة ClawHub المدركة للوكلاء. عند توفر
أعداد محركات المورّدين، يلخصها التدقيق بلغة واضحة، مثل:

```text
62/62 vendors flagged this skill as clean.
```

أو:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

عندما لا تتوفر لدى ClawHub مؤشرات قياس لأعداد المورّدين لتلخيصها، يقول التدقيق:

```text
No VirusTotal findings
```

تبقى VirusTotal مؤشرات قياس. وهي لا تحل محل تحليل المخاطر الخاص بـ ClawHub
والمراعي لطبيعة العناصر.

## تحليل المخاطر

يشغّل ClawScan، وهو نظام تدقيق الأمان الخاص بـ ClawHub، تحليل المخاطر داخليا.
يراجع كل إصدار باعتباره عنصرا موجها للوكلاء: التعليمات،
والبيانات الوصفية، والأذونات المعلنة، والملفات، وإشارات القدرات، وإشارات الفحص الثابت،
ونتائج SkillSpector، ومؤشرات VirusTotal، والسياق المقدم من الناشر.
تعد إشارات الفحص الثابت سياقا داخليا لهذه المراجعة؛ وليست
قسما عاما مستقلا للتدقيق أو حكما يمنع التثبيت.

يستخدم تحليل المخاطر
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
كعدسة للمخاطر مثل حقن المطالبات، وإساءة استخدام الأدوات، وكشف بيانات الاعتماد،
والتنفيذ غير الآمن، وتسميم الذاكرة أو السياق، والوكالة المفرطة.

لا يتعامل ClawScan مع القدرة التي تبدو مخيفة باعتبارها خبيثة تلقائيا.
بل يسأل عما إذا كانت القدرة معلنة، ومتوافقة مع الغرض، ومدعومة
بحالة الاستخدام المعلنة للإصدار.
