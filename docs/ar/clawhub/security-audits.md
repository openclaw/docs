# تدقيقات الأمان

تساعدك تدقيقات أمان ClawHub على تحديد ما إذا كانت skill أو Plugin آمنة بما يكفي
للتثبيت. فهي توضّح ما يفعله الإصدار، وما الصلاحيات التي يطلبها، وما إذا كان
هناك أي شيء يستحق انتباها إضافيا قبل أن يتمكن من الوصول إلى الملفات أو الحسابات
أو بيانات الاعتماد أو الشيفرة أو الخدمات الخارجية.

تُعد التدقيقات مؤشرات أمان قوية، لكنها ليست ضمانا بأن الإصدار خال تماما من
المخاطر. استخدم حكمك دائما قبل منح وصول حساس.

راجع أيضا [الأمان](/ar/clawhub/security)، و[الاستخدام المقبول](/ar/clawhub/acceptable-usage)،
و[الإشراف وسلامة الحساب](/ar/clawhub/moderation).

## ما يجب التحقق منه قبل التثبيت

قبل التثبيت، راجع:

- حالة التدقيق العامة
- مستوى المخاطر
- أي نتائج مدرجة
- بيانات الاعتماد أو الأذونات أو متغيرات البيئة المطلوبة
- المالك، والمصدر، والإصدار، وسجل التغييرات، والتنزيلات، والنجوم، وإشارات الثقة الأخرى

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالة التدقيق

تخبرك حالة التدقيق بكيفية التعامل مع نتيجة التدقيق:

| الحالة      | المعنى                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | لم تُعثر أي مشكلة ظاهرة تتجاوز المخاطر المنخفضة.                                |
| `Review`    | اقرأ النتائج قبل التثبيت. قد يظل الإصدار مشروعا. |
| `Warn`      | توخَّ حذرا إضافيا. وجد ClawHub مصدر قلق عالي الأثر أو إشارة تحذير. |
| `Malicious` | لا تثبّت.                                                           |
| `Pending`   | لم تكتمل التدقيقات بعد.                                             |
| `Error`     | تعذّر إكمال التدقيق.                                         |

تُعد حالة `Pass` مطمئنة، لكنها لا تغني عن حكمك الشخصي. وهذا مهم
خصوصا للأدوات التي يمكنها نشر محتوى، أو تعديل بيانات، أو تشغيل أوامر، أو قراءة ملفات، أو
الوصول إلى أنظمة الإنتاج.

## مستوى المخاطر

يصف مستوى المخاطر نطاق التأثير: مقدار السلطة التي يبدو أن الإصدار يمتلكها إذا
استخدمته على النحو المقصود.

| مستوى المخاطر | المعنى                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | لم تُعثر سلطة حساسة كبيرة أو تأثير كبير على المستخدم.                          |
| `Medium`   | يمتلك الإصدار سلطة ذات معنى، مثل الوصول إلى الحسابات أو تغيير البيانات. |
| `High`     | يمتلك الإصدار سلطة عالية الأثر، أو نتائج خطيرة، أو إشارات خبيثة. |

يجيب مستوى المخاطر وحالة التدقيق عن سؤالين مختلفين:

- يسأل مستوى المخاطر: "ما مقدار السلطة الموجودة هنا؟"
- تسأل حالة التدقيق: "ماذا ينبغي أن أفعل بهذه النتيجة؟"

على سبيل المثال، قد تعرض skill للنشر حالة `Review` مع مخاطر `Medium`. وهذا
لا يعني أنها خبيثة. بل يعني أن skill تبدو متوافقة مع الغرض، لكنها تستطيع
التصرف بسلطة حساب ذات معنى.

## النتائج

تشرح النتائج سبب عرض نتيجة التدقيق. تتضمن كل نتيجة عادة:

- ما الذي تعنيه
- سبب وضع علامة عليها
- محتوى skill أو Plugin ذي الصلة
- توصية

قد تحمل النتائج تسميات `Info` أو `Low` أو `Medium` أو `High` أو `Critical`. وتساهم
النتائج الأعلى شدة بدرجة أكبر في مستوى المخاطر وحالة التدقيق.

تُخفى النتائج منخفضة الثقة من ملخص التدقيق العام حتى تبقى الصفحة
مركزة على الأدلة المفيدة.

## ما الذي يتحقق منه ClawHub

يدقق ClawHub عناصر الإصدار المرسلة، بما في ذلك:

- تعليمات skill أو بيانات Plugin الوصفية
- متغيرات البيئة والأذونات المعلنة
- تعليمات التثبيت وبيانات الحزمة الوصفية
- الملفات المضمنة وبيانات الملفات
- بيانات التوافق والإمكانات الوصفية

السؤال الرئيسي هو الاتساق: هل يتماشى الاسم والملخص والبيانات الوصفية والسلطة
المطلوبة والمحتوى الفعلي مع ما يتوقعه المستخدمون على نحو معقول؟

السلوك القوي ليس سيئا تلقائيا. تحتاج كثير من الأدوات المفيدة إلى بيانات اعتماد،
أو أوامر محلية، أو واجهات API للمزوّدين، أو تثبيت حزم. يتحقق التدقيق مما إذا كانت تلك
السلطة متوقعة ومفصحا عنها ومتناسبة.

ترتبط صفحات العناصر بالتدقيق الكامل على:

```text
/<owner>/skills/<slug>/security-audit
```

تجمع صفحة التدقيق بين:

1. SkillSpector
2. VirusTotal
3. تحليل المخاطر

## VirusTotal

يستخدم ClawHub VirusTotal كبيانات قياس عن البرمجيات الخبيثة ضمن حزمة التدقيق. يُعد VirusTotal
معيارا موثوقا في الصناعة لسمعة الملفات وفحص البرمجيات الخبيثة، وتتيح شراكتنا
لـ ClawHub إضافة معلومات أمان أوسع إلى مراجعة Skills وPlugin.

يكون VirusTotal مفيدا خصوصا للعناصر الخبيثة المعروفة، وإشارات محركات الفحص، و
إشارات السمعة التي تكمل مراجعة ClawHub الواعية بالوكلاء. عندما تكون أعداد
محركات المورّدين متاحة، يلخصها التدقيق بلغة واضحة، مثل:

```text
62/62 vendors flagged this skill as clean.
```

أو:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

عندما لا تتوفر لدى ClawHub بيانات قياس عن أعداد المورّدين لتلخيصها، يقول التدقيق:

```text
No VirusTotal findings
```

يبقى VirusTotal بيانات قياس. ولا يحل محل تحليل المخاطر الخاص بـ ClawHub
والواعي بالعناصر.

## تحليل المخاطر

يعمل تحليل المخاطر داخليا بواسطة ClawScan، وهو نظام تدقيق الأمان الخاص بـ ClawHub.
يراجع كل إصدار بوصفه عنصرا موجها للوكلاء: التعليمات،
والبيانات الوصفية، والأذونات المعلنة، والملفات، وإشارات الإمكانات، وإشارات الفحص الثابت،
ونتائج SkillSpector، وبيانات VirusTotal، والسياق الذي يوفره الناشر.
إشارات الفحص الثابت هي سياق داخلي لهذه المراجعة؛ وليست
قسما عاما مستقلا في التدقيق أو حكما يمنع التثبيت.

يستخدم تحليل المخاطر
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
كعدسة للمخاطر مثل حقن المطالبات، وإساءة استخدام الأدوات، وكشف بيانات الاعتماد،
والتنفيذ غير الآمن، وتسميم الذاكرة أو السياق، والوكالة المفرطة.

لا يتعامل ClawScan مع الإمكانات التي تبدو مخيفة على أنها خبيثة تلقائيا.
بل يسأل ما إذا كانت الإمكانية مفصحا عنها، ومتوافقة مع الغرض، ومدعومة
بحالة الاستخدام المعلنة للإصدار.
