# تدقيقات الأمان

تساعدك تدقيقات أمان ClawHub على تحديد ما إذا كانت المهارة أو Plugin آمنين بما يكفي
للتثبيت. فهي توضّح ما يفعله الإصدار، وما الصلاحيات التي يطلبها، وما إذا كان
هناك ما يستحق اهتمامًا إضافيًا قبل أن يتمكن من الوصول إلى الملفات أو الحسابات
أو بيانات الاعتماد أو التعليمات البرمجية أو الخدمات الخارجية.

التدقيقات مؤشرات أمان قوية، لكنها لا تضمن أن الإصدار خالٍ من المخاطر.
استخدم حكمك دائمًا قبل منح وصول حساس.

راجع أيضًا [الأمان](/clawhub/security)، و[الاستخدام المقبول](/clawhub/acceptable-usage)،
و[الإشراف وسلامة الحساب](/clawhub/moderation).

## ما الذي يجب التحقق منه قبل التثبيت

قبل التثبيت، راجع:

- حالة التدقيق العامة
- مستوى المخاطر
- أي نتائج مدرجة
- بيانات الاعتماد أو الأذونات أو متغيرات البيئة المطلوبة
- المالك والمصدر والإصدار وسجل التغييرات والتنزيلات والنجوم وغيرها من مؤشرات الثقة

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالة التدقيق

تخبرك حالة التدقيق بكيفية التعامل مع نتيجة التدقيق:

| الحالة      | المعنى                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | لم يتم العثور على مشكلة ظاهرة تتجاوز المخاطر المنخفضة.                                |
| `Review`    | اقرأ النتائج قبل التثبيت. قد يظل الإصدار مشروعًا. |
| `Warn`      | توخَّ حذرًا إضافيًا. عثر ClawHub على مشكلة عالية الأثر أو إشارة تحذيرية. |
| `Malicious` | لا تثبّته.                                                           |
| `Pending`   | لم تنتهِ التدقيقات بعد.                                             |
| `Error`     | تعذّر إكمال التدقيق.                                         |

تُعد حالة `Pass` مطمئنة، لكنها لا تغني عن حكمك الشخصي. يهم هذا أكثر
للأدوات التي يمكنها نشر المحتوى أو تعديل البيانات أو تشغيل الأوامر أو قراءة الملفات أو
الوصول إلى أنظمة الإنتاج.

## مستوى المخاطر

يصف مستوى المخاطر نطاق التأثير: مقدار القوة التي يبدو أن الإصدار يمتلكها إذا
استخدمته كما هو مقصود.

| مستوى المخاطر | المعنى                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | لم يتم العثور على صلاحيات حساسة كبيرة أو تأثير كبير على المستخدم.                          |
| `Medium`   | يمتلك الإصدار صلاحيات مؤثرة، مثل الوصول إلى الحساب أو تغيير البيانات. |
| `High`     | يمتلك الإصدار صلاحيات عالية الأثر أو نتائج خطيرة أو إشارات ضارة. |

يجيب مستوى المخاطر وحالة التدقيق عن سؤالين مختلفين:

- يسأل مستوى المخاطر: "ما مقدار القوة هنا؟"
- تسأل حالة التدقيق: "ماذا يجب أن أفعل بهذه النتيجة؟"

على سبيل المثال، قد تعرض مهارة نشر حالة `Review` مع مخاطر `Medium`. هذا لا
يعني أنها ضارة. بل يعني أن المهارة تبدو متوافقة مع الغرض، لكنها تستطيع
التصرف بصلاحيات حساب مؤثرة.

## النتائج

تشرح النتائج سبب عرض نتيجة تدقيق معينة. يتضمن كلٌّ منها عادةً:

- ما الذي تعنيه
- سبب وضع العلامة عليها
- محتوى المهارة أو Plugin ذي الصلة
- توصية

قد تُوسم النتائج بـ `Info` أو `Low` أو `Medium` أو `High` أو `Critical`. تساهم
النتائج الأعلى خطورة بدرجة أقوى في مستوى المخاطر وحالة التدقيق.

تُخفى النتائج منخفضة الثقة من ملخص التدقيق العام حتى تظل الصفحة
مركّزة على الأدلة المفيدة.

## ما الذي يتحقق منه ClawHub

يدقق ClawHub عناصر الإصدار المقدمة، بما في ذلك:

- تعليمات المهارة أو بيانات Plugin الوصفية
- متغيرات البيئة والأذونات المعلنة
- تعليمات التثبيت وبيانات الحزمة الوصفية
- الملفات المضمنة وقوائم الملفات
- بيانات التوافق والقدرات الوصفية

السؤال الرئيسي هو الاتساق: هل يتماشى الاسم والملخص والبيانات الوصفية
والصلاحيات المطلوبة والمحتوى الفعلي مع ما يتوقعه المستخدمون على نحو معقول؟

السلوك القوي ليس سيئًا تلقائيًا. تحتاج كثير من الأدوات المفيدة إلى بيانات اعتماد
أو أوامر محلية أو واجهات API لمزوّدين أو تثبيت حزم. يتحقق التدقيق مما إذا كانت هذه
القوة متوقعة ومفصحًا عنها ومتناسبة.

تربط صفحات العناصر بالتدقيق الكامل على:

```text
/<owner>/skills/<slug>/security-audit
```

تجمع صفحة التدقيق بين:

1. SkillSpector
2. VirusTotal
3. تحليل المخاطر

## VirusTotal

يستخدم ClawHub VirusTotal كقياسات عن البرمجيات الخبيثة ضمن مجموعة التدقيق. يُعد VirusTotal
معيارًا موثوقًا في الصناعة لسمعة الملفات وفحص البرمجيات الخبيثة، وتتيح
شراكتنا لـ ClawHub إضافة معلومات أمنية أوسع إلى مراجعة المهارات وPlugin.

يكون VirusTotal مفيدًا خصوصًا للعناصر الضارة المعروفة، ونتائج محركات الفحص، و
إشارات السمعة التي تكمل مراجعة ClawHub الواعية بالوكلاء. عندما تتوفر
أعداد محركات الموردين، يلخصها التدقيق بلغة واضحة، مثل:

```text
62/62 vendors flagged this skill as clean.
```

أو:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

عندما لا تتوفر لدى ClawHub قياسات أعداد الموردين لتلخيصها، يقول التدقيق:

```text
No VirusTotal findings
```

يبقى VirusTotal قياسات مساندة. ولا يحل محل تحليل المخاطر الخاص بـ ClawHub
والواعي بطبيعة العناصر.

## تحليل المخاطر

يُشغَّل تحليل المخاطر داخليًا بواسطة ClawScan، وهو نظام تدقيق الأمان الخاص بـ ClawHub.
يراجع كل إصدار بوصفه عنصرًا موجهًا للوكلاء: التعليمات،
والبيانات الوصفية، والأذونات المعلنة، والملفات، وإشارات القدرات، وإشارات الفحص الثابت،
ونتائج SkillSpector، وقياسات VirusTotal، والسياق المقدم من الناشر.
تُعد إشارات الفحص الثابت سياقًا داخليًا لهذه المراجعة؛ وليست
قسم تدقيق عامًا مستقلًا أو حكمًا مانعًا للتثبيت.

يستخدم تحليل المخاطر
[أفضل 10 مخاطر لمهارات الوكلاء من OWASP](https://owasp.org/www-project-agentic-skills-top-10/)
كعدسة للمخاطر مثل حقن المطالبات، وإساءة استخدام الأدوات، وكشف بيانات الاعتماد،
والتنفيذ غير الآمن، وتسميم الذاكرة أو السياق، والقدرة الزائدة على التصرف.

لا يتعامل ClawScan مع القدرة التي تبدو مخيفة على أنها ضارة تلقائيًا.
بل يسأل ما إذا كانت القدرة مفصحًا عنها، ومتوافقة مع الغرض، ومدعومة
بحالة الاستخدام المعلنة للإصدار.
