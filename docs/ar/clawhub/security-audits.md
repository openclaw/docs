# تدقيقات الأمان

تساعدك تدقيقات أمان ClawHub على تحديد ما إذا كانت المهارة أو Plugin آمنين بما يكفي
للتثبيت. فهي توضّح ما يفعله الإصدار، وما الصلاحيات التي يطلبها، وما إذا كان هناك
أي شيء يستحق انتباهاً إضافياً قبل أن يتمكن من الوصول إلى الملفات أو الحسابات أو
بيانات الاعتماد أو التعليمات البرمجية أو الخدمات الخارجية.

تُعد التدقيقات مؤشرات أمان قوية، لكنها لا تضمن أن الإصدار خالٍ
من المخاطر. استخدم حكمك دائماً قبل منح وصول حساس.

راجع أيضاً [الأمان](/clawhub/security)، و[الاستخدام المقبول](/clawhub/acceptable-usage)،
و[الإشراف وسلامة الحساب](/clawhub/moderation).

## ما الذي يجب التحقق منه قبل التثبيت

قبل التثبيت، راجع:

- حالة التدقيق الإجمالية
- مستوى الخطر
- أي نتائج مدرجة
- بيانات الاعتماد أو الأذونات أو متغيرات البيئة المطلوبة
- المالك، والمصدر، والإصدار، وسجل التغييرات، والتنزيلات، والنجوم، وإشارات الثقة الأخرى

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالة التدقيق

تخبرك حالة التدقيق بكيفية التعامل مع نتيجة التدقيق:

| الحالة      | المعنى                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | لم يتم العثور على مشكلة مرئية أعلى من خطر منخفض.                                |
| `Review`    | اقرأ النتائج قبل التثبيت. قد يظل الإصدار مشروعاً. |
| `Warn`      | توخَّ حذراً إضافياً. وجد ClawHub مشكلة عالية التأثير أو إشارة تحذير. |
| `Malicious` | لا تثبّت.                                                           |
| `Pending`   | لم تنتهِ التدقيقات بعد.                                             |
| `Error`     | تعذّر إكمال التدقيق.                                         |

حالة `Pass` مطمئنة، لكنها لا تغني عن حكمك الشخصي. ويكون هذا مهماً
خصوصاً للأدوات التي يمكنها نشر المحتوى، أو تعديل البيانات، أو تشغيل الأوامر، أو قراءة الملفات، أو
الوصول إلى أنظمة الإنتاج.

## مستوى الخطر

يصف مستوى الخطر نطاق التأثير: مقدار القوة التي يبدو أن الإصدار يمتلكها إذا
استخدمته كما هو مقصود.

| مستوى الخطر | المعنى                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | تم العثور على قدر قليل من الصلاحيات الحساسة أو التأثير على المستخدم.                          |
| `Medium`   | يمتلك الإصدار صلاحيات ذات معنى، مثل الوصول إلى الحسابات أو تغيير البيانات. |
| `High`     | يمتلك الإصدار صلاحيات عالية التأثير، أو نتائج خطيرة، أو إشارات خبيثة. |

يجيب مستوى الخطر وحالة التدقيق عن سؤالين مختلفين:

- يسأل مستوى الخطر: "ما مقدار القوة الموجودة هنا؟"
- تسأل حالة التدقيق: "ماذا ينبغي أن أفعل بهذه النتيجة؟"

على سبيل المثال، قد تعرض مهارة نشر حالة `Review` مع خطر `Medium`. هذا
لا يعني أنها خبيثة. بل يعني أن المهارة تبدو متوافقة مع الغرض، لكنها يمكنها
التصرف بصلاحية حساب ذات معنى.

## النتائج

تشرح النتائج سبب عرض نتيجة التدقيق. يتضمن كل اكتشاف عادةً:

- ما يعنيه
- سبب وضع علامة عليه
- محتوى المهارة أو Plugin ذي الصلة
- توصية

قد تحمل النتائج تسميات `Info` أو `Low` أو `Medium` أو `High` أو `Critical`. تسهم النتائج ذات
الخطورة الأعلى بقوة أكبر في مستوى الخطر وحالة التدقيق.

تُخفى النتائج منخفضة الثقة من ملخص التدقيق العام حتى تظل الصفحة
مركزة على الأدلة المفيدة.

## ما الذي يتحقق منه ClawHub

يدقق ClawHub في عناصر الإصدار المقدمة، بما في ذلك:

- تعليمات المهارة أو بيانات تعريف Plugin
- متغيرات البيئة والأذونات المعلنة
- تعليمات التثبيت وبيانات تعريف الحزمة
- الملفات المضمّنة وبيانات الملفات
- بيانات تعريف التوافق والقدرات

السؤال الرئيسي هو الاتساق: هل يتوافق الاسم والملخص وبيانات التعريف
والصلاحيات المطلوبة والمحتوى الفعلي مع ما يتوقعه المستخدمون بشكل معقول؟

السلوك القوي ليس سيئاً تلقائياً. تحتاج كثير من الأدوات المفيدة إلى بيانات اعتماد،
أو أوامر محلية، أو واجهات برمجة تطبيقات للمزوّدين، أو تثبيت حزم. يتحقق التدقيق مما إذا كانت تلك
القوة متوقعة، ومعلنة، ومتناسبة.

ترتبط صفحات العناصر بالتدقيق الكامل على:

```text
/<owner>/skills/<slug>/security-audit
```

تجمع صفحة التدقيق بين:

1. SkillSpector
2. VirusTotal
3. تحليل المخاطر

## VirusTotal

يستخدم ClawHub خدمة VirusTotal كقياسات عن البرمجيات الخبيثة ضمن مجموعة التدقيق. VirusTotal
معيار موثوق في الصناعة لسمعة الملفات وفحص البرمجيات الخبيثة، وتتيح شراكتنا
لـ ClawHub إضافة معلومات أمنية أوسع إلى مراجعة المهارات وPlugins.

تكون VirusTotal مفيدة خصوصاً للعناصر الخبيثة المعروفة، ونتائج المحركات، و
إشارات السمعة التي تكمل مراجعة ClawHub الواعية بالوكلاء. عندما تتوفر
أعداد محركات المورّدين، يلخصها التدقيق بلغة واضحة، مثل:

```text
62/62 vendors flagged this skill as clean.
```

أو:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

عندما لا يملك ClawHub قياسات أعداد مورّدين لتلخيصها، يقول التدقيق:

```text
No VirusTotal findings
```

تظل VirusTotal قياسات فقط. ولا تستبدل تحليل المخاطر الواعي بالعناصر
الخاص بـ ClawHub.

## تحليل المخاطر

يعمل تحليل المخاطر داخلياً بواسطة ClawScan، وهو نظام تدقيق الأمان الخاص
بـ ClawHub. يراجع كل إصدار باعتباره عنصراً موجهاً إلى الوكلاء: التعليمات،
وبيانات التعريف، والأذونات المعلنة، والملفات، وإشارات القدرات، وإشارات الفحص الثابت،
ونتائج SkillSpector، وقياسات VirusTotal، والسياق الذي يقدمه الناشر.
تُعد إشارات الفحص الثابت سياقاً داخلياً لهذه المراجعة؛ وليست
قسماً عاماً مستقلاً للتدقيق أو حكماً يمنع التثبيت.

يستخدم تحليل المخاطر
[أفضل 10 مخاطر للمهارات الوكيلة من OWASP](https://owasp.org/www-project-agentic-skills-top-10/)
كعدسة للمخاطر مثل حقن المطالبات، وإساءة استخدام الأدوات، وكشف بيانات الاعتماد،
والتنفيذ غير الآمن، وتسميم الذاكرة أو السياق، والوكالة المفرطة.

لا يتعامل ClawScan مع القدرة التي تبدو مخيفة على أنها خبيثة تلقائياً.
بل يسأل ما إذا كانت القدرة معلنة، ومتوافقة مع الغرض، ومدعومة
بحالة الاستخدام المعلنة للإصدار.
