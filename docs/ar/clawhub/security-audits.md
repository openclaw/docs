# تدقيقات الأمان

تساعدك تدقيقات الأمان في ClawHub على تحديد ما إذا كانت Skills أو Plugin آمنة بما يكفي
لتثبيتها. فهي توضّح ما يفعله الإصدار، وما الصلاحيات التي يطلبها، وما إذا كان
هناك أي شيء يستحق انتباهاً إضافياً قبل أن يتمكن من الوصول إلى الملفات أو الحسابات
أو بيانات الاعتماد أو التعليمات البرمجية أو الخدمات الخارجية.

تُعد التدقيقات إشارات أمان قوية، لكنها ليست ضماناً بأن الإصدار
خالٍ من المخاطر. استخدم حكمك دائماً قبل منح وصول حساس.

راجع أيضاً [الأمان](/clawhub/security)، و[الاستخدام المقبول](/clawhub/acceptable-usage)،
و[الإشراف وسلامة الحساب](/clawhub/moderation).

## ما يجب التحقق منه قبل التثبيت

قبل التثبيت، راجع:

- حالة التدقيق الإجمالية
- مستوى المخاطر
- أي نتائج مدرجة
- بيانات الاعتماد أو الأذونات أو متغيرات البيئة المطلوبة
- المالك، والمصدر، والإصدار، وسجل التغييرات، والتنزيلات، والنجوم، وإشارات الثقة الأخرى

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالة التدقيق

تخبرك حالة التدقيق بكيفية التعامل مع نتيجة التدقيق:

| الحالة      | المعنى                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | لم تُعثر مشكلة ظاهرة أعلى من مستوى المخاطر المنخفضة.                                |
| `Review`    | اقرأ النتائج قبل التثبيت. قد يكون الإصدار مشروعاً رغم ذلك. |
| `Warn`      | توخَّ حذراً إضافياً. وجد ClawHub مشكلة عالية التأثير أو إشارة تحذير. |
| `Malicious` | لا تثبّت.                                                           |
| `Pending`   | لم تنتهِ التدقيقات بعد.                                             |
| `Error`     | تعذّر إكمال التدقيق.                                         |

تُعد حالة `Pass` مطمئنة، لكنها لا تغني عن حكمك الشخصي. وهذا مهم
بشكل خاص للأدوات التي يمكنها نشر المحتوى أو تحرير البيانات أو تشغيل الأوامر أو قراءة الملفات أو
الوصول إلى أنظمة الإنتاج.

## مستوى المخاطر

يصف مستوى المخاطر نطاق التأثير: مقدار القوة التي يبدو أن الإصدار يمتلكها إذا
استخدمته على النحو المقصود.

| مستوى المخاطر | المعنى                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | لم تُعثر صلاحيات حساسة أو آثار على المستخدم تُذكر.                          |
| `Medium`   | يمتلك الإصدار صلاحيات ذات معنى، مثل الوصول إلى الحسابات أو تغيير البيانات. |
| `High`     | يمتلك الإصدار صلاحيات عالية التأثير أو نتائج خطيرة أو إشارات خبيثة. |

يجيب مستوى المخاطر وحالة التدقيق عن سؤالين مختلفين:

- يسأل مستوى المخاطر: "ما مقدار القوة الموجودة هنا؟"
- تسأل حالة التدقيق: "ماذا يجب أن أفعل بهذه النتيجة؟"

على سبيل المثال، قد تعرض Skills للنشر حالة `Review` مع مخاطر `Medium`. هذا
لا يعني أنها خبيثة. بل يعني أن Skills تبدو متوافقة مع الغرض، لكنها قادرة
على التصرف بصلاحية حساب ذات معنى.

## النتائج

تشرح النتائج سبب عرض نتيجة تدقيق معينة. تتضمن كل نتيجة عادةً:

- ما تعنيه
- سبب الإبلاغ عنها
- محتوى Skills أو Plugin ذي الصلة
- توصية

قد تحمل النتائج تسميات `Info` أو `Low` أو `Medium` أو `High` أو `Critical`. تساهم النتائج ذات
الخطورة الأعلى بقوة أكبر في مستوى المخاطر وحالة التدقيق.

تُخفى النتائج منخفضة الثقة من ملخص التدقيق العام حتى تظل الصفحة
مركزة على الأدلة المفيدة.

## ما يتحقق منه ClawHub

يدقق ClawHub عناصر الإصدار المقدمة، بما في ذلك:

- تعليمات Skills أو بيانات Plugin الوصفية
- متغيرات البيئة والأذونات المعلنة
- تعليمات التثبيت وبيانات الحزمة الوصفية
- الملفات المضمنة وبيانات الملفات
- بيانات التوافق والقدرات الوصفية

السؤال الرئيسي هو الاتساق: هل يتوافق الاسم والملخص والبيانات الوصفية والصلاحيات المطلوبة
والمحتوى الفعلي مع ما يتوقعه المستخدمون على نحو معقول؟

السلوك القوي ليس سيئاً تلقائياً. تحتاج كثير من الأدوات المفيدة إلى بيانات اعتماد
أو أوامر محلية أو واجهات برمجة تطبيقات للمزوّدين أو تثبيت حزم. يتحقق التدقيق مما إذا كانت هذه
القوة متوقعة ومُفصحاً عنها ومتناسبة.

ترتبط صفحات العناصر بالتدقيق الكامل على:

```text
/<owner>/skills/<slug>/security-audit
```

تجمع صفحة التدقيق بين:

1. SkillSpector
2. VirusTotal
3. تحليل المخاطر

## VirusTotal

يستخدم ClawHub خدمة VirusTotal كقياسات عن بُعد للبرمجيات الخبيثة ضمن حزمة التدقيق. VirusTotal
معيار موثوق في القطاع لسمعة الملفات وفحص البرمجيات الخبيثة، وتتيح
شراكتنا لـ ClawHub إضافة معلومات أمان أوسع إلى مراجعة Skills وPlugin.

تكون VirusTotal مفيدة خصوصاً للعناصر الخبيثة المعروفة، ونتائج محركات الفحص، و
إشارات السمعة التي تكمل مراجعة ClawHub الواعية بالوكلاء. عندما تتوفر
أعداد محركات المورّدين، يلخصها التدقيق بلغة واضحة، مثل:

```text
62/62 vendors flagged this skill as clean.
```

أو:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

عندما لا تتوفر لدى ClawHub قياسات عن بُعد لأعداد المورّدين لتلخيصها، يقول التدقيق:

```text
No VirusTotal findings
```

تبقى VirusTotal قياسات عن بُعد. ولا تحل محل تحليل المخاطر الخاص بـ ClawHub
والواعي بالعناصر.

## تحليل المخاطر

يُشغَّل تحليل المخاطر داخلياً بواسطة ClawScan، وهو نظام تدقيق الأمان الخاص بـ ClawHub.
يراجع كل إصدار باعتباره عنصراً موجهاً إلى الوكيل: التعليمات،
والبيانات الوصفية، والأذونات المعلنة، والملفات، وإشارات القدرات، وإشارات الفحص الثابت،
ونتائج SkillSpector، وقياسات VirusTotal عن بُعد، والسياق المقدم من الناشر.
إشارات الفحص الثابت هي سياق داخلي لهذه المراجعة؛ وليست
قسماً عاماً مستقلاً للتدقيق أو حكماً يمنع التثبيت.

يستخدم تحليل المخاطر
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
كمنظور للمخاطر مثل حقن المطالبات، وإساءة استخدام الأدوات، وانكشاف بيانات الاعتماد،
والتنفيذ غير الآمن، وتسميم الذاكرة أو السياق، والوكالة المفرطة.

لا يتعامل ClawScan مع القدرة التي تبدو مخيفة على أنها خبيثة تلقائياً.
بل يسأل ما إذا كانت القدرة مُفصحاً عنها، ومتوافقة مع الغرض، ومدعومة
بحالة الاستخدام المعلنة للإصدار.
