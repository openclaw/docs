# عمليات تدقيق الأمان

تساعدك عمليات تدقيق الأمان في ClawHub على تحديد ما إذا كانت المهارة أو Plugin آمنة بما يكفي للتثبيت. فهي تعرض ما يفعله الإصدار، وما الصلاحيات التي يطلبها، وما إذا كان أي شيء يستحق اهتماما إضافيا قبل أن يتمكن من الوصول إلى الملفات أو الحسابات أو بيانات الاعتماد أو التعليمات البرمجية أو الخدمات الخارجية.

عمليات التدقيق مؤشرات أمان قوية، لكنها لا تضمن أن الإصدار خال من المخاطر. استخدم تقديرك دائما قبل منح وصول حساس.

راجع أيضا [الأمان](/clawhub/security)، و[الاستخدام المقبول](/ar/clawhub/acceptable-usage)، و[الإشراف وسلامة الحساب](/clawhub/moderation).

## ما يجب التحقق منه قبل التثبيت

قبل التثبيت، راجع:

- حالة التدقيق العامة
- مستوى المخاطر
- أي نتائج مدرجة
- بيانات الاعتماد أو الأذونات أو متغيرات البيئة المطلوبة
- المالك والمصدر والإصدار وسجل التغييرات والتنزيلات والنجوم وغيرها من مؤشرات الثقة

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالة التدقيق

تخبرك حالة التدقيق بكيفية التعامل مع نتيجة التدقيق:

| الحالة      | المعنى                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | لم تُعثر أي مشكلة مرئية أعلى من مستوى المخاطر المنخفضة.                                |
| `Review`    | اقرأ النتائج قبل التثبيت. قد يظل الإصدار مشروعا. |
| `Warn`      | استخدم حذرا إضافيا. عثر ClawHub على مصدر قلق عالي التأثير أو إشارة تحذير. |
| `Malicious` | لا تثبّت.                                                           |
| `Pending`   | لم تكتمل عمليات التدقيق بعد.                                             |
| `Error`     | تعذر إكمال التدقيق.                                         |

حالة `Pass` مطمئنة، لكنها لا تغني عن حكمك الشخصي. هذا مهم أكثر للأدوات التي يمكنها نشر المحتوى أو تعديل البيانات أو تشغيل الأوامر أو قراءة الملفات أو الوصول إلى أنظمة الإنتاج.

## مستوى المخاطر

يصف مستوى المخاطر نطاق التأثير: مقدار السلطة التي يبدو أن الإصدار يمتلكها إذا استخدمته كما هو مقصود.

| مستوى المخاطر | المعنى                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | وُجدت سلطة حساسة أو تأثير على المستخدم بدرجة قليلة.                          |
| `Medium`   | يمتلك الإصدار سلطة ذات معنى، مثل الوصول إلى الحسابات أو تغيير البيانات. |
| `High`     | يمتلك الإصدار سلطة عالية التأثير أو نتائج خطيرة أو إشارات خبيثة. |

يجيب مستوى المخاطر وحالة التدقيق عن سؤالين مختلفين:

- يسأل مستوى المخاطر: "كم مقدار السلطة هنا؟"
- تسأل حالة التدقيق: "ماذا ينبغي أن أفعل بهذه النتيجة؟"

على سبيل المثال، قد تعرض مهارة نشر حالة `Review` مع خطر `Medium`. هذا لا يعني أنها خبيثة. بل يعني أن المهارة تبدو متوافقة مع الغرض، لكنها تستطيع التصرف بسلطة ذات معنى على الحساب.

## النتائج

تشرح النتائج سبب عرض نتيجة التدقيق. يتضمن كل اكتشاف عادة:

- ما يعنيه
- سبب وضع علامة عليه
- محتوى المهارة أو Plugin ذي الصلة
- توصية

قد تُوسم النتائج بـ `Info` أو `Low` أو `Medium` أو `High` أو `Critical`. تسهم النتائج الأعلى شدة بقوة أكبر في مستوى المخاطر وحالة التدقيق.

تُخفى النتائج منخفضة الثقة من ملخص التدقيق العام كي تظل الصفحة مركزة على الأدلة المفيدة.

## ما الذي يتحقق منه ClawHub

يدقق ClawHub عناصر الإصدار المقدمة، بما في ذلك:

- تعليمات المهارات أو بيانات Plugin الوصفية
- متغيرات البيئة والأذونات المعلنة
- تعليمات التثبيت وبيانات الحزمة الوصفية
- الملفات المضمنة وبيانات الملفات
- بيانات التوافق والقدرات الوصفية

السؤال الرئيسي هو الاتساق: هل يتماشى الاسم والملخص والبيانات الوصفية والسلطة المطلوبة والمحتوى الفعلي مع ما يتوقعه المستخدمون بشكل معقول؟

السلوك القوي ليس سيئا تلقائيا. تحتاج العديد من الأدوات المفيدة إلى بيانات اعتماد أو أوامر محلية أو واجهات API للمزودين أو تثبيت حزم. يتحقق التدقيق مما إذا كانت تلك السلطة متوقعة ومعلنة ومتناسبة.

ترتبط صفحات العناصر بالتدقيق الكامل على:

```text
/<owner>/skills/<slug>/security-audit
```

تجمع صفحة التدقيق بين:

1. SkillSpector
2. VirusTotal
3. تحليل المخاطر

## VirusTotal

يستخدم ClawHub‏ VirusTotal باعتباره قياسات برمجيات خبيثة ضمن حزمة التدقيق. VirusTotal معيار موثوق في الصناعة لسمعة الملفات وفحص البرمجيات الخبيثة، وتتيح شراكتنا لـ ClawHub إضافة معلومات أمان أوسع إلى مراجعة المهارات وPlugin.

يكون VirusTotal مفيدا بشكل خاص للعناصر الخبيثة المعروفة، وإصابات المحركات، وإشارات السمعة التي تكمل مراجعة ClawHub المدركة للعوامل. عند توفر أعداد محركات المورّدين، يلخصها التدقيق بلغة بسيطة، مثل:

```text
62/62 vendors flagged this skill as clean.
```

أو:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

عندما لا يمتلك ClawHub قياسات عدد المورّدين لتلخيصها، يقول التدقيق:

```text
No VirusTotal findings
```

يبقى VirusTotal قياسات. ولا يحل محل تحليل المخاطر الخاص بـ ClawHub والمدرك للعناصر.

## تحليل المخاطر

يُشغّل تحليل المخاطر داخليا بواسطة ClawScan، نظام تدقيق الأمان الخاص بـ ClawHub. يراجع كل إصدار باعتباره عنصرا موجها إلى العامل: التعليمات، والبيانات الوصفية، والأذونات المعلنة، والملفات، وإشارات القدرات، وإشارات الفحص الثابت، ونتائج SkillSpector، وقياسات VirusTotal، والسياق الذي يقدمه الناشر. إشارات الفحص الثابت سياق داخلي لهذه المراجعة؛ وليست قسما عاما مستقلا للتدقيق أو حكما يمنع التثبيت.

يستخدم تحليل المخاطر
[أفضل 10 مخاطر للـ Skills الوكيلة من OWASP](https://owasp.org/www-project-agentic-skills-top-10/)
كعدسة للمخاطر مثل حقن المطالبات، وإساءة استخدام الأدوات، وكشف بيانات الاعتماد، والتنفيذ غير الآمن، وتسميم الذاكرة أو السياق، والوكالة المفرطة.

لا يتعامل ClawScan مع القدرة التي تبدو مخيفة على أنها خبيثة تلقائيا. فهو يسأل عما إذا كانت القدرة معلنة ومتوافقة مع الغرض ومدعومة بحالة الاستخدام المصرح بها للإصدار.
