# تدقيقات الأمان

تساعدك تدقيقات أمان ClawHub على تحديد ما إذا كانت المهارة أو Plugin آمنة بما يكفي
للتثبيت. فهي توضّح ما يفعله الإصدار، وما الصلاحيات التي يطلبها، وما إذا كان
هناك ما يستحق انتباهاً إضافياً قبل أن يتمكن من الوصول إلى الملفات أو الحسابات
أو بيانات الاعتماد أو الشيفرة أو الخدمات الخارجية.

التدقيقات مؤشرات أمان قوية، لكنها ليست ضماناً بأن الإصدار خالٍ من المخاطر.
استخدم حكمك دائماً قبل منح وصول حساس.

راجع أيضاً [الأمان](/clawhub/security)، و[الاستخدام المقبول](/clawhub/acceptable-usage)،
و[الإشراف وسلامة الحساب](/clawhub/moderation).

## ما يجب التحقق منه قبل التثبيت

قبل التثبيت، راجع:

- حالة التدقيق العامة
- مستوى المخاطر
- أي نتائج مدرجة
- بيانات الاعتماد أو الأذونات أو متغيرات البيئة المطلوبة
- المالك، والمصدر، والإصدار، وسجل التغييرات، والتنزيلات، والنجوم، وغيرها من مؤشرات الثقة

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالة التدقيق

تخبرك حالة التدقيق بكيفية التعامل مع نتيجة التدقيق:

| الحالة      | المعنى                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | لم تُعثر أي مشكلة ظاهرة أعلى من مستوى الخطر المنخفض.                                |
| `Review`    | اقرأ النتائج قبل التثبيت. قد يظل الإصدار مشروعاً. |
| `Warn`      | توخَّ حذراً إضافياً. وجد ClawHub مشكلة عالية التأثير أو إشارة تحذير. |
| `Malicious` | لا تثبّت.                                                           |
| `Pending`   | لم تكتمل التدقيقات بعد.                                             |
| `Error`     | تعذّر إكمال التدقيق.                                         |

تُعد حالة `Pass` مطمئنة، لكنها لا تغني عن حكمك الشخصي. ويكون ذلك أهم
للأدوات التي يمكنها نشر محتوى، أو تعديل بيانات، أو تشغيل أوامر، أو قراءة ملفات، أو
الوصول إلى أنظمة الإنتاج.

## مستوى المخاطر

يصف مستوى المخاطر نطاق التأثير: مقدار القوة التي يبدو أن الإصدار يمتلكها إذا
استخدمته كما هو مقصود.

| مستوى المخاطر | المعنى                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | لم تُعثر صلاحيات حساسة أو تأثير كبير على المستخدم.                          |
| `Medium`   | يمتلك الإصدار صلاحيات مؤثرة، مثل الوصول إلى الحسابات أو تغيير البيانات. |
| `High`     | يمتلك الإصدار صلاحيات عالية التأثير، أو نتائج خطيرة، أو إشارات خبيثة. |

يجيب مستوى المخاطر وحالة التدقيق عن سؤالين مختلفين:

- يسأل مستوى المخاطر: "ما مقدار القوة الموجودة هنا؟"
- تسأل حالة التدقيق: "ماذا ينبغي أن أفعل بهذه النتيجة؟"

على سبيل المثال، قد تعرض مهارة نشر الحالة `Review` مع خطر `Medium`. هذا
لا يعني أنها خبيثة. بل يعني أن المهارة تبدو متوافقة مع الغرض، لكنها تستطيع
التصرف بصلاحيات حساب مؤثرة.

## النتائج

تشرح النتائج سبب عرض نتيجة تدقيق معينة. عادةً ما تتضمن كل نتيجة:

- ما الذي تعنيه
- سبب وسمها
- محتوى المهارة أو Plugin ذي الصلة
- توصية

قد تُوسم النتائج بـ `Info` أو `Low` أو `Medium` أو `High` أو `Critical`. تسهم النتائج
الأعلى شدة بقوة أكبر في مستوى المخاطر وحالة التدقيق.

تُخفى النتائج منخفضة الثقة من ملخص التدقيق العام حتى تبقى الصفحة
مركزة على الأدلة المفيدة.

## ما الذي يتحقق منه ClawHub

يدقق ClawHub عناصر الإصدار المقدمة، بما في ذلك:

- تعليمات المهارة أو بيانات تعريف Plugin
- متغيرات البيئة والأذونات المعلنة
- تعليمات التثبيت وبيانات تعريف الحزمة
- الملفات المضمنة وبيانات الملفات
- بيانات تعريف التوافق والقدرات

السؤال الرئيسي هو الاتساق: هل يتوافق الاسم والملخص وبيانات التعريف والصلاحيات
المطلوبة والمحتوى الفعلي مع ما يتوقعه المستخدمون بشكل معقول؟

السلوك القوي ليس سيئاً تلقائياً. تحتاج أدوات مفيدة كثيرة إلى بيانات اعتماد،
أو أوامر محلية، أو واجهات API لمزودين، أو تثبيت حزم. يتحقق التدقيق مما إذا كانت هذه
القوة متوقعة ومعلنة ومتناسبة.

ترتبط صفحات العناصر بالتدقيق الكامل على:

```text
/<owner>/skills/<slug>/security-audit
```

تجمع صفحة التدقيق بين:

1. SkillSpector
2. VirusTotal
3. تحليل المخاطر

## VirusTotal

يستخدم ClawHub VirusTotal كقياسات عن البرمجيات الخبيثة ضمن حزمة التدقيق. يُعد VirusTotal
معياراً موثوقاً في القطاع لسمعة الملفات وفحص البرمجيات الخبيثة، وتتيح شراكتنا
لـ ClawHub إضافة ذكاء أمني أوسع إلى مراجعة المهارات وPlugins.

يكون VirusTotal مفيداً خصوصاً للعناصر الخبيثة المعروفة، ونتائج محركات الفحص،
وإشارات السمعة التي تكمل مراجعة ClawHub الواعية بالوكلاء. عندما تتوفر أعداد
محركات المورّدين، يلخصها التدقيق بلغة واضحة، مثل:

```text
62/62 vendors flagged this skill as clean.
```

أو:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

عندما لا تتوفر لدى ClawHub قياسات عددية من المورّدين لتلخيصها، يقول التدقيق:

```text
No VirusTotal findings
```

يبقى VirusTotal قياسات مساندة. ولا يحل محل تحليل المخاطر الخاص بـ ClawHub
والواعي بالعناصر.

## تحليل المخاطر

يُشغَّل تحليل المخاطر داخلياً بواسطة ClawScan، وهو نظام تدقيق الأمان الخاص
بـ ClawHub. يراجع كل إصدار بوصفه عنصراً موجهاً للوكلاء: التعليمات،
وبيانات التعريف، والأذونات المعلنة، والملفات، وإشارات القدرات، وإشارات الفحص الثابت،
ونتائج SkillSpector، وقياسات VirusTotal، والسياق الذي يقدمه الناشر.
تُعد إشارات الفحص الثابت سياقاً داخلياً لهذه المراجعة؛ وليست قسماً عاماً
مستقلاً في التدقيق أو حكماً يمنع التثبيت.

يستخدم تحليل المخاطر
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
كعدسة للمخاطر مثل حقن المطالبات، وإساءة استخدام الأدوات، وكشف بيانات الاعتماد،
والتنفيذ غير الآمن، وتسميم الذاكرة أو السياق، والوكالة المفرطة.

لا يتعامل ClawScan مع القدرة التي تبدو مخيفة على أنها خبيثة تلقائياً.
بل يسأل ما إذا كانت القدرة معلنة، ومتوافقة مع الغرض، ومدعومة بحالة الاستخدام
المذكورة للإصدار.
