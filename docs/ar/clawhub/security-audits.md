# عمليات التدقيق الأمني

تساعدك عمليات التدقيق الأمني في ClawHub على تحديد ما إذا كانت Skills أو Plugin آمنة بما يكفي
لتثبيتها. وهي توضّح ما يفعله الإصدار، وما الصلاحيات التي يطلبها، وما
إذا كان هناك ما يستدعي اهتمامًا إضافيًا قبل أن يتمكن من الوصول إلى الملفات أو الحسابات
أو بيانات الاعتماد أو الشيفرة أو الخدمات الخارجية.

تُعد عمليات التدقيق مؤشرات أمان قوية، لكنها لا تضمن خلو الإصدار
من المخاطر. استخدم تقديرك دائمًا قبل منح صلاحيات وصول حساسة.

راجع أيضًا [الأمان](/clawhub/security)، و[الاستخدام المقبول](/clawhub/acceptable-usage)،
و[الإشراف وسلامة الحساب](/clawhub/moderation).

## ما يجب التحقق منه قبل التثبيت

قبل التثبيت، راجع:

- حالة التدقيق العامة
- مستوى المخاطر
- أي نتائج مدرجة
- بيانات الاعتماد أو الأذونات أو متغيرات البيئة المطلوبة
- المالك والمصدر والإصدار وسجل التغييرات والتنزيلات والنجوم وغيرها من مؤشرات الثقة

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالة التدقيق

توضّح حالة التدقيق كيفية التعامل مع نتيجة التدقيق:

| الحالة      | المعنى                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | لم يُعثر على مشكلة ظاهرة تتجاوز مستوى المخاطر المنخفض.                                |
| `Review`    | اقرأ النتائج قبل التثبيت. قد يظل الإصدار مشروعًا. |
| `Warn`      | توخَّ مزيدًا من الحذر. وجد ClawHub مصدر قلق عالي التأثير أو إشارة تحذير. |
| `Malicious` | لا تثبّته.                                                           |
| `Pending`   | لم تكتمل عمليات التدقيق بعد.                                             |
| `Error`     | تعذّر إكمال التدقيق.                                         |

تُعد حالة `Pass` مطمئنة، لكنها لا تغني عن تقديرك الشخصي. ويكتسب هذا الأمر
أهمية خاصة بالنسبة إلى الأدوات التي يمكنها نشر المحتوى أو تعديل البيانات أو تشغيل الأوامر أو قراءة الملفات أو
الوصول إلى أنظمة الإنتاج.

## مستوى المخاطر

يصف مستوى المخاطر نطاق التأثير: مقدار القوة التي يبدو أن الإصدار يمتلكها إذا
استخدمته على النحو المقصود.

| مستوى المخاطر | المعنى                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | لم يُعثر على صلاحيات حساسة أو تأثير يُذكر على المستخدم.                          |
| `Medium`   | يمتلك الإصدار صلاحيات مؤثرة، مثل الوصول إلى الحساب أو تغيير البيانات. |
| `High`     | يمتلك الإصدار صلاحيات عالية التأثير، أو نتائج شديدة الخطورة، أو إشارات ضارة. |

يجيب مستوى المخاطر وحالة التدقيق عن سؤالين مختلفين:

- يسأل مستوى المخاطر: "ما مقدار القوة الموجودة هنا؟"
- تسأل حالة التدقيق: "ماذا ينبغي أن أفعل بهذه النتيجة؟"

على سبيل المثال، قد تعرض Skills للنشر حالة `Review` مع مخاطر `Medium`. وهذا
لا يعني أنها ضارة. بل يعني أن Skills تبدو متوافقة مع غرضها، لكنها تستطيع
التصرف بصلاحيات مؤثرة على الحساب.

## النتائج

توضّح النتائج سبب عرض نتيجة التدقيق. ويتضمن كل استنتاج عادةً:

- ما الذي يعنيه
- سبب الإبلاغ عنه
- محتوى Skills أو Plugin ذي الصلة
- توصية

قد تحمل النتائج التصنيفات `Info` أو `Low` أو `Medium` أو `High` أو `Critical`. وتسهم النتائج ذات
الخطورة الأعلى بدرجة أكبر في تحديد مستوى المخاطر وحالة التدقيق.

تُخفى النتائج منخفضة الثقة من الملخص العام للتدقيق كي تظل الصفحة
مركّزة على الأدلة المفيدة.

## ما الذي يتحقق منه ClawHub

يدقّق ClawHub في عناصر الإصدار المقدَّمة، بما في ذلك:

- تعليمات Skills أو البيانات الوصفية لـ Plugin
- متغيرات البيئة والأذونات المعلنة
- تعليمات التثبيت والبيانات الوصفية للحزمة
- الملفات المضمّنة وبيانات حصر الملفات
- البيانات الوصفية للتوافق والإمكانات

السؤال الأساسي هو مدى الاتساق: هل يتوافق الاسم والملخص والبيانات الوصفية والصلاحيات
المطلوبة والمحتوى الفعلي مع ما يتوقعه المستخدمون بشكل معقول؟

السلوك القوي ليس سيئًا تلقائيًا. تحتاج العديد من الأدوات المفيدة إلى بيانات اعتماد
أو أوامر محلية أو واجهات API لموفري الخدمات أو تثبيت حزم. يتحقق التدقيق مما إذا كانت هذه
القوة متوقعة ومُفصحًا عنها ومتناسبة.

ترتبط صفحات العناصر بالتدقيق الكامل على:

```text
/<owner>/skills/<slug>/security-audit
```

تجمع صفحة التدقيق بين:

1. SkillSpector
2. VirusTotal
3. تحليل المخاطر

## VirusTotal

يستخدم ClawHub خدمة VirusTotal كمصدر لبيانات الرصد الخاصة بالبرمجيات الضارة ضمن منظومة التدقيق. تُعد VirusTotal
معيارًا موثوقًا في القطاع لتقييم سمعة الملفات وفحص البرمجيات الضارة، وتتيح
شراكتنا لـ ClawHub إضافة معلومات أمنية أوسع إلى مراجعة Skills وPlugin.

تُعد VirusTotal مفيدة خصوصًا للعناصر الضارة المعروفة، واكتشافات محركات الفحص،
ومؤشرات السمعة التي تُكمّل مراجعة ClawHub الواعية بسياق الوكلاء. وعند توفر
أعداد نتائج محركات المورّدين، يلخصها التدقيق بلغة واضحة، مثل:

```text
62/62 vendors flagged this skill as clean.
```

أو:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

عندما لا تتوفر لدى ClawHub بيانات رصد لأعداد المورّدين يمكن تلخيصها، يعرض التدقيق:

```text
No VirusTotal findings
```

تظل VirusTotal مصدرًا لبيانات الرصد. ولا تحل محل تحليل المخاطر الخاص بـ ClawHub
والواعي بمحتوى العناصر.

## تحليل المخاطر

يعمل تحليل المخاطر داخليًا بواسطة ClawScan، وهو نظام التدقيق الأمني الخاص
بـ ClawHub. ويراجع كل إصدار بوصفه عنصرًا موجّهًا للوكلاء، بما يشمل: التعليمات،
والبيانات الوصفية، والأذونات المعلنة، والملفات، ومؤشرات الإمكانات، ومؤشرات الفحص الثابت،
ونتائج SkillSpector، وبيانات رصد VirusTotal، والسياق الذي يقدّمه الناشر.
تُعد مؤشرات الفحص الثابت سياقًا داخليًا لهذه المراجعة؛ وليست
قسمًا عامًا مستقلًا في التدقيق ولا حكمًا يحظر التثبيت.

يستخدم تحليل المخاطر
[أهم 10 مخاطر في Skills الوكيلة من OWASP](https://owasp.org/www-project-agentic-skills-top-10/)
كمنظور لتقييم مخاطر مثل حقن الموجّهات، وإساءة استخدام الأدوات، وكشف بيانات الاعتماد،
والتنفيذ غير الآمن، وتسميم الذاكرة أو السياق، والاستقلالية المفرطة.

لا يعتبر ClawScan الإمكانات التي تبدو مخيفة ضارة تلقائيًا.
بل يتحقق مما إذا كانت الإمكانية مُفصحًا عنها ومتوافقة مع الغرض ومدعومة
بحالة الاستخدام المعلنة للإصدار.
