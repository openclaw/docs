# عمليات التدقيق الأمني

تساعدك عمليات التدقيق الأمني في ClawHub على تحديد ما إذا كانت Skill أو Plugin آمنة بما يكفي
للتثبيت. فهي تعرض ما يفعله الإصدار، وما الصلاحيات التي يطلبها، وما إذا
كان هناك أي شيء يستحق انتباها إضافيا قبل أن يتمكن من الوصول إلى الملفات أو الحسابات
أو بيانات الاعتماد أو الكود أو الخدمات الخارجية.

تعد عمليات التدقيق إشارات أمان قوية، لكنها ليست ضمانا بأن الإصدار
خال تماما من المخاطر. استخدم تقديرك دائما قبل منح وصول حساس.

راجع أيضا [الأمان](/clawhub/security)، و[الاستخدام المقبول](/clawhub/acceptable-usage)،
و[الإشراف وسلامة الحساب](/clawhub/moderation).

## ما يجب التحقق منه قبل التثبيت

قبل التثبيت، راجع:

- حالة التدقيق العامة
- مستوى المخاطر
- أي نتائج مدرجة
- بيانات الاعتماد أو الأذونات أو متغيرات البيئة المطلوبة
- المالك، والمصدر، والإصدار، وسجل التغييرات، والتنزيلات، والنجوم، وإشارات الثقة الأخرى

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالة التدقيق

تخبرك حالة التدقيق بكيفية التعامل مع نتيجة التدقيق:

| الحالة      | المعنى                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | لم يتم العثور على مشكلة ظاهرة تتجاوز المخاطر المنخفضة.                                |
| `Review`    | اقرأ النتائج قبل التثبيت. قد يظل الإصدار مشروعا. |
| `Warn`      | توخ حذرا إضافيا. وجد ClawHub مصدر قلق عالي التأثير أو إشارة تحذير. |
| `Malicious` | لا تثبته.                                                           |
| `Pending`   | لم تنته عمليات التدقيق بعد.                                             |
| `Error`     | تعذر إكمال التدقيق.                                         |

تعد حالة `Pass` مطمئنة، لكنها لا تغني عن تقديرك الشخصي. ويكون هذا مهما
خصوصا للأدوات التي يمكنها نشر المحتوى أو تعديل البيانات أو تشغيل الأوامر أو قراءة الملفات أو
الوصول إلى أنظمة الإنتاج.

## مستوى المخاطر

يصف مستوى المخاطر نطاق التأثير: مقدار القوة التي يبدو أن الإصدار يمتلكها إذا
استخدمته كما هو مقصود.

| مستوى المخاطر | المعنى                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | لم يتم العثور على صلاحيات حساسة كبيرة أو أثر كبير على المستخدم.                          |
| `Medium`   | يمتلك الإصدار صلاحيات معتبرة، مثل الوصول إلى الحسابات أو إجراء تغييرات على البيانات. |
| `High`     | يمتلك الإصدار صلاحيات عالية التأثير أو نتائج خطيرة أو إشارات خبيثة. |

يجيب مستوى المخاطر وحالة التدقيق عن سؤالين مختلفين:

- يسأل مستوى المخاطر: "ما مقدار القوة الموجودة هنا؟"
- تسأل حالة التدقيق: "ماذا ينبغي أن أفعل بهذه النتيجة؟"

على سبيل المثال، قد تعرض Skill للنشر حالة `Review` مع مستوى مخاطر `Medium`. وهذا
لا يعني أنها خبيثة. بل يعني أن Skill تبدو متوافقة مع الغرض، لكنها يمكنها
التصرف بصلاحيات حساب معتبرة.

## النتائج

تشرح النتائج سبب عرض نتيجة تدقيق معينة. تتضمن كل نتيجة عادة:

- ما تعنيه
- سبب الإشارة إليها
- محتوى Skill أو Plugin ذي الصلة
- توصية

قد تحمل النتائج التصنيفات `Info` أو `Low` أو `Medium` أو `High` أو `Critical`. تسهم النتائج الأعلى
خطورة بدرجة أكبر في مستوى المخاطر وحالة التدقيق.

تُخفى النتائج منخفضة الثقة من ملخص التدقيق العام حتى تظل الصفحة
مركزة على الأدلة المفيدة.

## ما الذي يفحصه ClawHub

يدقق ClawHub في تحف الإصدارات المقدمة، بما في ذلك:

- تعليمات Skill أو بيانات Plugin الوصفية
- متغيرات البيئة والأذونات المعلنة
- تعليمات التثبيت وبيانات الحزمة الوصفية
- الملفات المضمنة وبيانات الملفات
- بيانات التوافق والقدرات الوصفية

السؤال الرئيسي هو الاتساق: هل يتوافق الاسم والملخص والبيانات الوصفية
والصلاحيات المطلوبة والمحتوى الفعلي مع ما يتوقعه المستخدمون على نحو معقول؟

السلوك القوي ليس سيئا تلقائيا. تحتاج كثير من الأدوات المفيدة إلى بيانات اعتماد
أو أوامر محلية أو واجهات API للمزودين أو عمليات تثبيت حزم. يتحقق التدقيق مما إذا كانت هذه
القوة متوقعة ومعلنة ومتناسبة.

ترتبط صفحات التحف بالتدقيق الكامل على:

```text
/<owner>/skills/<slug>/security-audit
```

تجمع صفحة التدقيق بين:

1. SkillSpector
2. VirusTotal
3. تحليل المخاطر

## VirusTotal

يستخدم ClawHub خدمة VirusTotal كقياسات عن البرمجيات الخبيثة ضمن حزمة التدقيق. VirusTotal
معيار موثوق في القطاع لسمعة الملفات وفحص البرمجيات الخبيثة، وتتيح
شراكتنا لـ ClawHub إضافة معلومات أمنية أوسع إلى مراجعة Skills والـ Plugins.

تكون VirusTotal مفيدة خصوصا للتحف الخبيثة المعروفة، ونتائج محركات الفحص، و
إشارات السمعة التي تكمل مراجعة ClawHub المدركة للوكلاء. عند توفر
أعداد محركات الموردين، يلخصها التدقيق بلغة واضحة، مثل:

```text
62/62 vendors flagged this skill as clean.
```

أو:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

عندما لا تتوفر لدى ClawHub قياسات أعداد الموردين لتلخيصها، يقول التدقيق:

```text
No VirusTotal findings
```

تظل VirusTotal قياسات مساعدة. فهي لا تحل محل تحليل المخاطر الخاص بـ ClawHub
والمدرك للتحف.

## تحليل المخاطر

يعمل تحليل المخاطر داخليا بواسطة ClawScan، نظام التدقيق الأمني الخاص بـ ClawHub.
يراجع كل إصدار باعتباره تحفة موجهة إلى الوكلاء: التعليمات،
والبيانات الوصفية، والأذونات المعلنة، والملفات، وإشارات القدرات، وإشارات الفحص الثابت،
ونتائج SkillSpector، وقياسات VirusTotal، والسياق المقدم من الناشر.
تعد إشارات الفحص الثابت سياقا داخليا لهذه المراجعة؛ وليست
قسما عاما مستقلا للتدقيق أو حكما مانعا للتثبيت.

يستخدم تحليل المخاطر
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
كعدسة للمخاطر مثل حقن المطالبات، وإساءة استخدام الأدوات، وانكشاف بيانات الاعتماد،
والتنفيذ غير الآمن، وتسميم الذاكرة أو السياق، والوكالة المفرطة.

لا يتعامل ClawScan مع قدرة تبدو مخيفة على أنها خبيثة تلقائيا.
بل يسأل ما إذا كانت القدرة معلنة، ومتوافقة مع الغرض، ومدعومة
بحالة الاستخدام المعلنة للإصدار.
