# عمليات تدقيق الأمان

تساعدك عمليات تدقيق أمان ClawHub على تحديد ما إذا كانت skill أو Plugin آمنة بما يكفي
للتثبيت. فهي تُظهر ما يفعله الإصدار، وما الصلاحية التي يطلبها، وما إذا كان
هناك أي شيء يستحق انتباهًا إضافيًا قبل أن يتمكن من الوصول إلى الملفات أو الحسابات
أو بيانات الاعتماد أو التعليمات البرمجية أو الخدمات الخارجية.

تُعد عمليات التدقيق إشارات أمان قوية، لكنها ليست ضمانًا بأن الإصدار
خالٍ من المخاطر. استخدم حكمك دائمًا قبل منح وصول حساس.

انظر أيضًا [الأمان](/clawhub/security)، و[الاستخدام المقبول](/clawhub/acceptable-usage)،
و[الإشراف وسلامة الحساب](/clawhub/moderation).

## ما يجب التحقق منه قبل التثبيت

قبل التثبيت، راجع:

- حالة التدقيق الإجمالية
- مستوى المخاطر
- أي نتائج مدرجة
- بيانات الاعتماد أو الأذونات أو متغيرات البيئة المطلوبة
- المالك والمصدر والإصدار وسجل التغييرات والتنزيلات والنجوم وإشارات الثقة الأخرى

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالة التدقيق

تخبرك حالة التدقيق بكيفية التعامل مع نتيجة التدقيق:

| الحالة | المعنى |
| ----------- | ------------------------------------------------------------------------- |
| `Pass` | لم يتم العثور على مشكلة مرئية فوق مستوى المخاطر المنخفضة. |
| `Review` | اقرأ النتائج قبل التثبيت. قد يظل الإصدار مشروعًا. |
| `Warn` | توخَّ حذرًا إضافيًا. عثر ClawHub على مشكلة عالية التأثير أو إشارة تحذير. |
| `Malicious` | لا تثبّت. |
| `Pending` | لم تنتهِ عمليات التدقيق بعد. |
| `Error` | تعذر إكمال التدقيق. |

تُعد نتيجة `Pass` مطمئنة، لكنها لا تغني عن حكمك الخاص. يهم هذا
أكثر للأدوات التي يمكنها نشر المحتوى أو تعديل البيانات أو تشغيل الأوامر أو قراءة الملفات أو
الوصول إلى أنظمة الإنتاج.

## مستوى المخاطر

يصف مستوى المخاطر نطاق التأثير: مقدار السلطة التي يبدو أن الإصدار يمتلكها إذا
استخدمته كما هو مقصود.

| مستوى المخاطر | المعنى |
| ---------- | ----------------------------------------------------------------------------- |
| `Low` | وُجد قدر قليل من الصلاحيات الحساسة أو التأثير على المستخدم. |
| `Medium` | يمتلك الإصدار صلاحيات ذات شأن، مثل الوصول إلى الحساب أو تغيير البيانات. |
| `High` | يمتلك الإصدار صلاحيات عالية التأثير أو نتائج شديدة أو إشارات خبيثة. |

يجيب مستوى المخاطر وحالة التدقيق عن سؤالين مختلفين:

- يسأل مستوى المخاطر: "ما مقدار الصلاحية الموجودة هنا؟"
- تسأل حالة التدقيق: "ماذا ينبغي أن أفعل بهذه النتيجة؟"

على سبيل المثال، قد تعرض skill مخصصة للنشر الحالة `Review` مع مخاطرة `Medium`. هذا
لا يعني أنها خبيثة. بل يعني أن skill تبدو متوافقة مع الغرض، لكنها تستطيع
التصرف بصلاحية حساب ذات شأن.

## النتائج

تشرح النتائج سبب عرض نتيجة تدقيق معينة. تتضمن كل نتيجة عادةً:

- ما تعنيه
- سبب الإشارة إليها
- محتوى skill أو Plugin ذي الصلة
- توصية

قد تُصنّف النتائج بعلامات `Info` أو `Low` أو `Medium` أو `High` أو `Critical`. تسهم النتائج الأعلى
شدةً بقوة أكبر في مستوى المخاطر وحالة التدقيق.

تُخفى النتائج منخفضة الثقة من ملخص التدقيق العام لكي تظل الصفحة
مركزة على الأدلة المفيدة.

## ما الذي يتحقق منه ClawHub

يدقق ClawHub عناصر الإصدار المقدمة، بما في ذلك:

- تعليمات skill أو بيانات Plugin الوصفية
- متغيرات البيئة والأذونات المصرح بها
- تعليمات التثبيت وبيانات الحزمة الوصفية
- الملفات المضمنة وبيانات الملفات
- بيانات التوافق والقدرات الوصفية

السؤال الرئيسي هو الاتساق: هل يتوافق الاسم والملخص والبيانات الوصفية والسلطة
المطلوبة والمحتوى الفعلي مع ما يتوقعه المستخدمون على نحو معقول؟

السلوك القوي ليس سيئًا تلقائيًا. تحتاج كثير من الأدوات المفيدة إلى بيانات اعتماد
أو أوامر محلية أو واجهات API لمزوّدين أو تثبيت حزم. يتحقق التدقيق مما إذا كانت تلك
القوة متوقعة ومعلنة ومتناسبة.

ترتبط صفحات العناصر بالتدقيق الكامل على:

```text
/<owner>/skills/<slug>/security-audit
```

تجمع صفحة التدقيق بين:

1. SkillSpector
2. VirusTotal
3. تحليل المخاطر

## VirusTotal

يستخدم ClawHub VirusTotal كقياسات عن بُعد للبرمجيات الخبيثة في منظومة التدقيق. يُعد VirusTotal
معيارًا موثوقًا في الصناعة لسمعة الملفات وفحص البرمجيات الخبيثة، وتتيح
شراكتنا لـ ClawHub إضافة معلومات أمان أوسع إلى مراجعة skills وPlugins.

يُعد VirusTotal مفيدًا على وجه الخصوص للعناصر الخبيثة المعروفة، ونتائج محركات الفحص، و
إشارات السمعة التي تكمل مراجعة ClawHub الواعية بالوكلاء. عندما تكون أعداد
محركات المورّدين متاحة، يلخصها التدقيق بلغة واضحة، مثل:

```text
62/62 vendors flagged this skill as clean.
```

أو:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

عندما لا تتوفر لدى ClawHub قياسات عن بُعد لأعداد المورّدين يمكن تلخيصها، يقول التدقيق:

```text
No VirusTotal findings
```

يبقى VirusTotal قياسات عن بُعد. ولا يحل محل تحليل المخاطر الخاص بـ ClawHub
والواعي بالعناصر.

## تحليل المخاطر

يعمل تحليل المخاطر داخليًا بواسطة ClawScan، وهو نظام تدقيق الأمان الخاص بـ ClawHub.
يراجع كل إصدار باعتباره عنصرًا موجهًا للوكلاء: التعليمات
والبيانات الوصفية والأذونات المصرح بها والملفات وإشارات القدرات وإشارات الفحص الثابتة
ونتائج SkillSpector وقياسات VirusTotal عن بُعد والسياق المقدم من الناشر.
تُعد إشارات الفحص الثابتة سياقًا داخليًا لهذه المراجعة؛ وليست
قسم تدقيق عامًا مستقلًا أو حكمًا يمنع التثبيت.

يستخدم تحليل المخاطر
[أهم 10 مخاطر للـ Agentic Skills من OWASP](https://owasp.org/www-project-agentic-skills-top-10/)
كعدسة لمخاطر مثل حقن الموجهات، وإساءة استخدام الأدوات، وانكشاف بيانات الاعتماد،
والتنفيذ غير الآمن، وتسميم الذاكرة أو السياق، والوكالة المفرطة.

لا يتعامل ClawScan مع القدرة التي تبدو مخيفة على أنها خبيثة تلقائيًا.
بل يسأل عما إذا كانت القدرة معلنة ومتوافقة مع الغرض ومدعومة
بحالة الاستخدام المعلنة للإصدار.
