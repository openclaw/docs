# تدقيقات الأمان

تساعدك تدقيقات الأمان في ClawHub على تحديد ما إذا كانت Skills أو Plugin آمنة بما يكفي
للتثبيت. فهي توضّح ما يفعله الإصدار، وما الصلاحيات التي يطلبها، وما إذا كان هناك
أي شيء يستحق انتباهاً إضافياً قبل أن يتمكن من الوصول إلى الملفات أو الحسابات أو
بيانات الاعتماد أو الكود أو الخدمات الخارجية.

تُعد التدقيقات إشارات أمان قوية، لكنها ليست ضماناً بأن الإصدار خالٍ من
المخاطر. استخدم حكمك دائماً قبل منح وصول حساس.

راجع أيضاً [الأمان](/ar/clawhub/security)، و[الاستخدام المقبول](/ar/clawhub/acceptable-usage)،
و[الإشراف وسلامة الحساب](/ar/clawhub/moderation).

## ما يجب التحقق منه قبل التثبيت

قبل التثبيت، راجع:

- حالة التدقيق العامة
- مستوى المخاطر
- أي نتائج مدرجة
- بيانات الاعتماد أو الأذونات أو متغيرات البيئة المطلوبة
- المالك، والمصدر، والإصدار، وسجل التغييرات، والتنزيلات، والنجوم، وإشارات الثقة الأخرى

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالة التدقيق

تخبرك حالة التدقيق بكيفية التعامل مع نتيجة التدقيق:

| الحالة      | المعنى                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | لم يتم العثور على مشكلة ظاهرة أعلى من المخاطر المنخفضة.                                |
| `Review`    | اقرأ النتائج قبل التثبيت. قد يظل الإصدار مشروعاً. |
| `Warn`      | توخَّ حذراً إضافياً. وجد ClawHub مشكلة ذات تأثير عالٍ أو إشارة تحذير. |
| `Malicious` | لا تثبّت.                                                           |
| `Pending`   | لم تكتمل التدقيقات بعد.                                             |
| `Error`     | تعذر إكمال التدقيق.                                         |

تُعد حالة `Pass` مطمئنة، لكنها لا تغني عن حكمك الشخصي. يهم هذا
أكثر للأدوات التي يمكنها نشر المحتوى، أو تعديل البيانات، أو تشغيل الأوامر، أو قراءة الملفات، أو
الوصول إلى أنظمة الإنتاج.

## مستوى المخاطر

يصف مستوى المخاطر نطاق التأثير: مقدار القوة التي يبدو أن الإصدار يمتلكها إذا
استخدمته على النحو المقصود.

| مستوى المخاطر | المعنى                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | لم يتم العثور على قدر كبير من الصلاحيات الحساسة أو التأثير على المستخدم.                          |
| `Medium`   | يمتلك الإصدار صلاحيات مهمة، مثل الوصول إلى الحسابات أو تغيير البيانات. |
| `High`     | يمتلك الإصدار صلاحيات عالية التأثير، أو نتائج خطيرة، أو إشارات خبيثة. |

يجيب مستوى المخاطر وحالة التدقيق عن سؤالين مختلفين:

- يسأل مستوى المخاطر: "ما مقدار القوة الموجودة هنا؟"
- تسأل حالة التدقيق: "ماذا ينبغي أن أفعل بهذه النتيجة؟"

على سبيل المثال، قد تعرض Skills للنشر حالة `Review` مع مخاطر `Medium`. هذا
لا يعني أنها خبيثة. بل يعني أن Skills تبدو متوافقة مع الغرض، لكنها تستطيع
التصرف بصلاحيات حساب ذات معنى.

## النتائج

تشرح النتائج سبب عرض نتيجة التدقيق. تتضمن كل نتيجة عادةً:

- ماذا تعني
- لماذا تم وسمها
- محتوى Skills أو Plugin ذي الصلة
- توصية

قد تُوسم النتائج بـ `Info` أو `Low` أو `Medium` أو `High` أو `Critical`. تسهم النتائج الأعلى
شدةً بقوة أكبر في مستوى المخاطر وحالة التدقيق.

تُخفى النتائج منخفضة الثقة من ملخص التدقيق العام بحيث تبقى الصفحة
مركزة على الأدلة المفيدة.

## ما الذي يفحصه ClawHub

يدقق ClawHub في آثار الإصدارات المقدمة، بما في ذلك:

- تعليمات Skills أو بيانات Plugin الوصفية
- متغيرات البيئة والأذونات المعلنة
- تعليمات التثبيت وبيانات الحزمة الوصفية
- الملفات المضمنة وبيانات الملفات
- بيانات التوافق والقدرات الوصفية

السؤال الرئيسي هو الاتساق: هل يتوافق الاسم، والملخص، والبيانات الوصفية، والصلاحيات المطلوبة،
والمحتوى الفعلي مع ما يتوقعه المستخدمون على نحو معقول؟

السلوك القوي ليس سيئاً تلقائياً. تحتاج كثير من الأدوات المفيدة إلى بيانات اعتماد،
أوامر محلية، أو واجهات API للمزوّدين، أو تثبيت حزم. يتحقق التدقيق مما إذا كانت هذه
القوة متوقعة، ومفصحاً عنها، ومتناسبة.

ترتبط صفحات الآثار بالتدقيق الكامل على:

```text
/<owner>/skills/<slug>/security-audit
```

تجمع صفحة التدقيق بين:

1. SkillSpector
2. VirusTotal
3. تحليل المخاطر

## VirusTotal

يستخدم ClawHub VirusTotal كقياسات عن البرمجيات الخبيثة ضمن حزمة التدقيق. يُعد VirusTotal
معياراً موثوقاً في القطاع لسمعة الملفات وفحص البرمجيات الخبيثة، وتتيح
شراكتنا لـ ClawHub إضافة معلومات أمان أوسع إلى مراجعة Skills وPlugin.

يفيد VirusTotal بشكل خاص في الآثار الخبيثة المعروفة، وإصابات المحركات، و
إشارات السمعة التي تكمل مراجعة ClawHub المدركة للوكلاء. عندما تكون أعداد
محركات المورّدين متاحة، يلخصها التدقيق بلغة واضحة، مثل:

```text
62/62 vendors flagged this skill as clean.
```

أو:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

عندما لا تتوفر لدى ClawHub قياسات لأعداد المورّدين لتلخيصها، يقول التدقيق:

```text
No VirusTotal findings
```

يبقى VirusTotal قياسات مساعدة. فهو لا يحل محل تحليل المخاطر المدرك للآثار
الخاص بـ ClawHub.

## تحليل المخاطر

يُشغّل تحليل المخاطر داخلياً بواسطة ClawScan، وهو نظام تدقيق الأمان الخاص بـ ClawHub.
يراجع كل إصدار بوصفه أثراً موجهاً للوكلاء: التعليمات،
والبيانات الوصفية، والأذونات المعلنة، والملفات، وإشارات القدرات، وإشارات الفحص الثابت،
ونتائج SkillSpector، وقياسات VirusTotal، والسياق الذي يقدمه الناشر.
إشارات الفحص الثابت هي سياق داخلي لهذه المراجعة؛ وليست
قسماً مستقلاً في التدقيق العام أو حكماً يمنع التثبيت.

يستخدم تحليل المخاطر
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
كعدسة للمخاطر مثل حقن الموجهات، وإساءة استخدام الأدوات، وانكشاف بيانات الاعتماد،
والتنفيذ غير الآمن، وتسميم الذاكرة أو السياق، والاستقلالية المفرطة.

لا يتعامل ClawScan مع القدرة التي تبدو مخيفة على أنها خبيثة تلقائياً.
بل يسأل ما إذا كانت القدرة مفصحاً عنها، ومتوافقة مع الغرض، ومدعومة
بحالة الاستخدام المعلنة للإصدار.
