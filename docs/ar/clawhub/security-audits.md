# تدقيقات الأمان

تساعدك تدقيقات أمان ClawHub على تحديد ما إذا كانت المهارة أو Plugin آمنة بما يكفي
للتثبيت. فهي توضّح ما يفعله الإصدار، وما الصلاحية التي يطلبها، وما إذا كان
هناك ما يستحق انتباهًا إضافيًا قبل أن يتمكن من الوصول إلى الملفات أو الحسابات
أو بيانات الاعتماد أو الشيفرة أو الخدمات الخارجية.

تُعد التدقيقات إشارات أمان قوية، لكنها ليست ضمانًا بأن الإصدار خالٍ من
المخاطر. استخدم حكمك دائمًا قبل منح وصول حساس.

راجع أيضًا [الأمان](/ar/clawhub/security)، و[الاستخدام المقبول](/ar/clawhub/acceptable-usage)،
و[الإشراف وسلامة الحساب](/ar/clawhub/moderation).

## ما يجب التحقق منه قبل التثبيت

قبل التثبيت، راجع:

- حالة التدقيق العامة
- مستوى المخاطر
- أي نتائج مدرجة
- بيانات الاعتماد أو الأذونات أو متغيرات البيئة المطلوبة
- المالك، والمصدر، والإصدار، وسجل التغييرات، والتنزيلات، والنجوم، وإشارات الثقة الأخرى

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالة التدقيق

تخبرك حالة التدقيق بكيفية التعامل مع نتيجة التدقيق:

| الحالة      | المعنى                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | لم يتم العثور على مشكلة ظاهرة أعلى من مستوى المخاطر المنخفضة.                                |
| `Review`    | اقرأ النتائج قبل التثبيت. قد يظل الإصدار مشروعًا. |
| `Warn`      | توخَّ حذرًا إضافيًا. عثر ClawHub على مصدر قلق عالي التأثير أو إشارة تحذير. |
| `Malicious` | لا تثبّت.                                                           |
| `Pending`   | لم تنتهِ التدقيقات بعد.                                             |
| `Error`     | تعذر إكمال التدقيق.                                         |

تُعد حالة `Pass` مطمئنة، لكنها لا تغني عن حكمك الشخصي. يهم ذلك
أكثر بالنسبة إلى الأدوات التي يمكنها نشر المحتوى، أو تعديل البيانات، أو تشغيل الأوامر، أو قراءة الملفات، أو
الوصول إلى أنظمة الإنتاج.

## مستوى المخاطر

يصف مستوى المخاطر نطاق التأثير: مقدار القوة التي يبدو أن الإصدار يمتلكها إذا
استخدمته على النحو المقصود.

| مستوى المخاطر | المعنى                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | تم العثور على صلاحية حساسة أو تأثير على المستخدم محدودين.                          |
| `Medium`   | يمتلك الإصدار صلاحية ذات معنى، مثل الوصول إلى الحساب أو تغييرات البيانات. |
| `High`     | يمتلك الإصدار صلاحية عالية التأثير، أو نتائج شديدة، أو إشارات خبيثة. |

يجيب مستوى المخاطر وحالة التدقيق عن سؤالين مختلفين:

- يسأل مستوى المخاطر: "ما مقدار القوة الموجودة هنا؟"
- تسأل حالة التدقيق: "ماذا ينبغي أن أفعل بهذه النتيجة؟"

على سبيل المثال، قد تعرض مهارة نشر حالة `Review` مع خطر `Medium`. هذا
لا يعني أنها خبيثة. بل يعني أن المهارة تبدو متوافقة مع الغرض، لكنها تستطيع
التصرف بصلاحية حساب ذات معنى.

## النتائج

توضح النتائج سبب عرض نتيجة التدقيق. يتضمن كل اكتشاف عادةً:

- ما يعنيه
- سبب وضع العلامة عليه
- محتوى المهارة أو Plugin ذي الصلة
- توصية

قد تُصنّف النتائج بعلامات `Info` أو `Low` أو `Medium` أو `High` أو `Critical`. تسهم النتائج الأعلى
شدة بشكل أقوى في مستوى المخاطر وحالة التدقيق.

تُخفى النتائج منخفضة الثقة من ملخص التدقيق العام حتى تبقى الصفحة
مركزة على الأدلة المفيدة.

## ما يتحقق منه ClawHub

يدقق ClawHub عناصر الإصدار المقدمة، بما في ذلك:

- تعليمات المهارة أو بيانات Plugin الوصفية
- متغيرات البيئة والأذونات المعلنة
- تعليمات التثبيت وبيانات الحزمة الوصفية
- الملفات المضمنة وبيانات قوائم الملفات
- بيانات التوافق والقدرات الوصفية

السؤال الرئيسي هو الاتساق: هل يتوافق الاسم والملخص والبيانات الوصفية والصلاحية
المطلوبة والمحتوى الفعلي مع ما يتوقعه المستخدمون بشكل معقول؟

السلوك القوي ليس سيئًا تلقائيًا. تحتاج أدوات مفيدة كثيرة إلى بيانات اعتماد،
أو أوامر محلية، أو واجهات API لمزوّدين، أو تثبيت حزم. يتحقق التدقيق مما إذا كانت تلك
القوة متوقعة ومفصحًا عنها ومتناسبة.

ترتبط صفحات العناصر بالتدقيق الكامل على:

```text
/<owner>/skills/<slug>/security-audit
```

تجمع صفحة التدقيق بين:

1. SkillSpector
2. VirusTotal
3. تحليل المخاطر

## VirusTotal

يستخدم ClawHub خدمة VirusTotal كقياس عن بُعد للبرمجيات الخبيثة ضمن مجموعة التدقيق. تُعد VirusTotal
معيارًا موثوقًا في القطاع لسمعة الملفات وفحص البرمجيات الخبيثة، وتتيح
شراكتنا لـ ClawHub إضافة معلومات أمان أوسع إلى مراجعة المهارات وPlugins.

تكون VirusTotal مفيدة خصوصًا للعناصر الخبيثة المعروفة، وإصابات المحركات، و
إشارات السمعة التي تكمل مراجعة ClawHub الواعية بالعوامل. عندما تكون أعداد
محركات المورّدين متاحة، يلخصها التدقيق بلغة واضحة، مثل:

```text
62/62 vendors flagged this skill as clean.
```

أو:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

عندما لا تتوفر لدى ClawHub قياسات عن بُعد لأعداد المورّدين لتلخيصها، يقول التدقيق:

```text
No VirusTotal findings
```

تظل VirusTotal قياسًا عن بُعد. ولا تحل محل تحليل المخاطر الخاص بـ ClawHub
والواعي بالعناصر.

## تحليل المخاطر

يُشغَّل تحليل المخاطر داخليًا بواسطة ClawScan، وهو نظام تدقيق الأمان الخاص بـ ClawHub.
يراجع كل إصدار باعتباره عنصرًا موجهًا إلى عامل: التعليمات،
والبيانات الوصفية، والأذونات المعلنة، والملفات، وإشارات القدرات، وإشارات الفحص الثابت،
ونتائج SkillSpector، وقياسات VirusTotal عن بُعد، والسياق المقدم من الناشر.
إشارات الفحص الثابت هي سياق داخلي لهذه المراجعة؛ وليست
قسم تدقيق عام مستقلًا أو حكمًا يمنع التثبيت.

يستخدم تحليل المخاطر
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
كعدسة للمخاطر مثل حقن المطالبات، وإساءة استخدام الأدوات، وانكشاف بيانات الاعتماد،
والتنفيذ غير الآمن، وتسميم الذاكرة أو السياق، والوكالة المفرطة.

لا يتعامل ClawScan مع القدرة التي تبدو مخيفة على أنها خبيثة تلقائيًا.
بل يسأل عما إذا كانت القدرة مفصحًا عنها، ومتوافقة مع الغرض، ومدعومة
بحالة الاستخدام المعلنة للإصدار.
