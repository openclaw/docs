# عمليات تدقيق الأمان

تساعدك عمليات تدقيق الأمان في ClawHub على تحديد ما إذا كانت المهارة أو Plugin آمنة بما يكفي
للتثبيت. فهي تعرض ما يفعله الإصدار، وما الصلاحيات التي يطلبها، وما إذا كان
هناك ما يستحق انتباهًا إضافيًا قبل أن يتمكن من الوصول إلى الملفات أو الحسابات
أو بيانات الاعتماد أو الشيفرة أو الخدمات الخارجية.

تُعد عمليات التدقيق إشارات أمان قوية، لكنها ليست ضمانًا بأن الإصدار
خالٍ من المخاطر. استخدم حكمك دائمًا قبل منح وصول حساس.

راجع أيضًا [الأمان](/clawhub/security)، و[الاستخدام المقبول](/clawhub/acceptable-usage)،
و[الإشراف وسلامة الحساب](/clawhub/moderation).

## ما يجب التحقق منه قبل التثبيت

قبل التثبيت، راجع:

- حالة التدقيق الإجمالية
- مستوى المخاطر
- أي نتائج مدرجة
- بيانات الاعتماد أو الأذونات أو متغيرات البيئة المطلوبة
- المالك، والمصدر، والإصدار، وسجل التغييرات، والتنزيلات، والنجوم، وإشارات الثقة الأخرى

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالة التدقيق

تخبرك حالة التدقيق بكيفية التعامل مع نتيجة التدقيق:

| الحالة      | المعنى                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | لم تُعثر أي مشكلة مرئية أعلى من مستوى المخاطر المنخفضة.                                |
| `Review`    | اقرأ النتائج قبل التثبيت. قد يظل الإصدار مشروعًا. |
| `Warn`      | توخَّ حذرًا إضافيًا. وجد ClawHub مشكلة عالية التأثير أو إشارة تحذير. |
| `Malicious` | لا تثبّت.                                                           |
| `Pending`   | لم تكتمل عمليات التدقيق بعد.                                             |
| `Error`     | تعذّر إكمال التدقيق.                                         |

حالة `Pass` مطمئنة، لكنها لا تغني عن حكمك الشخصي. يهم هذا
أكثر للأدوات التي يمكنها نشر المحتوى، أو تعديل البيانات، أو تشغيل الأوامر، أو قراءة الملفات، أو
الوصول إلى أنظمة الإنتاج.

## مستوى المخاطر

يصف مستوى المخاطر نطاق التأثير: مقدار القوة التي يبدو أن الإصدار يمتلكها إذا
استخدمته كما هو مقصود.

| مستوى المخاطر | المعنى                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | لم تُعثر صلاحيات حساسة كبيرة أو تأثير كبير على المستخدم.                          |
| `Medium`   | يمتلك الإصدار صلاحيات مؤثرة، مثل الوصول إلى الحسابات أو تغييرات البيانات. |
| `High`     | يمتلك الإصدار صلاحيات عالية التأثير، أو نتائج شديدة، أو إشارات خبيثة. |

يجيب مستوى المخاطر وحالة التدقيق عن سؤالين مختلفين:

- يسأل مستوى المخاطر: "ما مقدار القوة الموجودة هنا؟"
- تسأل حالة التدقيق: "ماذا يجب أن أفعل بهذه النتيجة؟"

على سبيل المثال، قد تعرض مهارة نشر حالة `Review` مع مخاطر `Medium`. هذا
لا يعني أنها خبيثة. بل يعني أن المهارة تبدو متوافقة مع الغرض، لكنها تستطيع
التصرف بصلاحيات حساب مؤثرة.

## النتائج

تشرح النتائج سبب عرض نتيجة التدقيق. يتضمن كل بند عادةً:

- ما يعنيه
- سبب وضع علامة عليه
- محتوى المهارة أو Plugin ذي الصلة
- توصية

قد تُوسم النتائج بـ `Info` أو `Low` أو `Medium` أو `High` أو `Critical`. تسهم النتائج الأعلى
خطورة بقوة أكبر في مستوى المخاطر وحالة التدقيق.

تُخفى النتائج منخفضة الثقة من ملخص التدقيق العام كي تظل الصفحة
مركزة على الأدلة المفيدة.

## ما الذي يتحقق منه ClawHub

يدقق ClawHub في ملفات الإصدار المقدمة، بما في ذلك:

- تعليمات المهارة أو بيانات تعريف Plugin
- متغيرات البيئة والأذونات المعلنة
- تعليمات التثبيت وبيانات تعريف الحزمة
- الملفات المضمنة وبيانات الملفات
- بيانات تعريف التوافق والقدرات

السؤال الرئيسي هو الاتساق: هل يتوافق الاسم والملخص وبيانات التعريف
والصلاحيات المطلوبة والمحتوى الفعلي مع ما يتوقعه المستخدمون بشكل معقول؟

السلوك القوي ليس سيئًا تلقائيًا. تحتاج أدوات مفيدة كثيرة إلى بيانات اعتماد،
أو أوامر محلية، أو واجهات API للموفرين، أو تثبيت حزم. يتحقق التدقيق مما إذا كانت تلك
القوة متوقعة ومعلنة ومتناسبة.

تربط صفحات الملفات بالتدقيق الكامل على:

```text
/<owner>/skills/<slug>/security-audit
```

تجمع صفحة التدقيق بين:

1. SkillSpector
2. VirusTotal
3. تحليل المخاطر

## VirusTotal

يستخدم ClawHub خدمة VirusTotal كمؤشرات برمجيات خبيثة ضمن حزمة التدقيق. تُعد VirusTotal
معيارًا موثوقًا في الصناعة لسمعة الملفات وفحص البرمجيات الخبيثة، وتتيح شراكتنا
لـ ClawHub إضافة معلومات أمان أوسع إلى مراجعة المهارات وPlugin.

تكون VirusTotal مفيدة خصوصًا للملفات الخبيثة المعروفة، ونتائج محركات الفحص، و
إشارات السمعة التي تكمل مراجعة ClawHub الواعية بالوكلاء. عندما تتوفر
أعداد محركات البائعين، يلخصها التدقيق بلغة واضحة، مثل:

```text
62/62 vendors flagged this skill as clean.
```

أو:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

عندما لا يملك ClawHub مؤشرات بعدد البائعين لتلخيصها، يقول التدقيق:

```text
No VirusTotal findings
```

تبقى VirusTotal مؤشرات فقط. ولا تحل محل تحليل المخاطر الواعي بالملفات
الخاص بـ ClawHub.

## تحليل المخاطر

يشغّل ClawScan، نظام تدقيق الأمان الخاص بـ ClawHub، تحليل المخاطر داخليًا.
يراجع كل إصدار باعتباره ملفًا موجهًا للوكلاء: التعليمات،
وبيانات التعريف، والأذونات المعلنة، والملفات، وإشارات القدرات، وإشارات الفحص الثابت،
ونتائج SkillSpector، ومؤشرات VirusTotal، والسياق المقدم من الناشر.
إشارات الفحص الثابت هي سياق داخلي لهذه المراجعة؛ وليست
قسم تدقيق عامًا مستقلًا أو حكمًا يمنع التثبيت.

يستخدم تحليل المخاطر
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
كمنظور للمخاطر مثل حقن المطالبات، وإساءة استخدام الأدوات، وكشف بيانات الاعتماد،
والتنفيذ غير الآمن، وتسميم الذاكرة أو السياق، والوكالة المفرطة.

لا يتعامل ClawScan مع قدرة تبدو مخيفة على أنها خبيثة تلقائيًا.
بل يسأل ما إذا كانت القدرة معلنة، ومتوافقة مع الغرض، ومدعومة
بحالة الاستخدام المعلنة للإصدار.
