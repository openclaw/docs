# تدقيقات الأمان

تساعدك تدقيقات أمان ClawHub على تحديد ما إذا كانت المهارة أو Plugin آمنين بما يكفي
للتثبيت. فهي تعرض ما يفعله الإصدار، وما الصلاحيات التي يطلبها، وما إذا كان هناك
أي شيء يستحق انتباها إضافيا قبل أن يتمكن من الوصول إلى الملفات أو الحسابات أو
بيانات الاعتماد أو الكود أو الخدمات الخارجية.

تعد التدقيقات إشارات أمان قوية، لكنها ليست ضمانا بأن الإصدار خال من المخاطر.
استخدم حكمك دائما قبل منح وصول حساس.

راجع أيضا [الأمان](/clawhub/security)، و[الاستخدام المقبول](/clawhub/acceptable-usage)،
و[الإشراف وسلامة الحساب](/clawhub/moderation).

## ما يجب التحقق منه قبل التثبيت

قبل التثبيت، راجع:

- حالة التدقيق الإجمالية
- مستوى المخاطر
- أي نتائج مدرجة
- بيانات الاعتماد أو الأذونات أو متغيرات البيئة المطلوبة
- المالك والمصدر والإصدار وسجل التغييرات والتنزيلات والنجوم وإشارات الثقة الأخرى

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالة التدقيق

توضح لك حالة التدقيق كيفية التعامل مع نتيجة التدقيق:

| الحالة      | المعنى                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | لم يتم العثور على مشكلة ظاهرة تتجاوز المخاطر المنخفضة.                                |
| `Review`    | اقرأ النتائج قبل التثبيت. قد يظل الإصدار مشروعا. |
| `Warn`      | استخدم حذرا إضافيا. وجد ClawHub مشكلة عالية التأثير أو إشارة تحذير. |
| `Malicious` | لا تثبّت.                                                           |
| `Pending`   | لم تكتمل التدقيقات بعد.                                             |
| `Error`     | تعذر إكمال التدقيق.                                         |

تعد حالة `Pass` مطمئنة، لكنها لا تغني عن حكمك الشخصي. ويكون هذا أكثر أهمية
للأدوات التي يمكنها نشر المحتوى أو تحرير البيانات أو تشغيل الأوامر أو قراءة الملفات أو
الوصول إلى أنظمة الإنتاج.

## مستوى المخاطر

يصف مستوى المخاطر نطاق التأثير: مقدار السلطة التي يبدو أن الإصدار يمتلكها إذا
استخدمته كما هو مقصود.

| مستوى المخاطر | المعنى                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | تم العثور على قدر محدود من السلطة الحساسة أو تأثير المستخدم.                          |
| `Medium`   | يمتلك الإصدار سلطة ذات معنى، مثل الوصول إلى الحساب أو تغيير البيانات. |
| `High`     | يمتلك الإصدار سلطة عالية التأثير أو نتائج خطيرة أو إشارات ضارة. |

يجيب مستوى المخاطر وحالة التدقيق عن سؤالين مختلفين:

- يسأل مستوى المخاطر: "ما مقدار السلطة الموجودة هنا؟"
- تسأل حالة التدقيق: "ماذا يجب أن أفعل بهذه النتيجة؟"

على سبيل المثال، قد تعرض مهارة نشر حالة `Review` مع خطر `Medium`. هذا
لا يعني أنها ضارة. بل يعني أن المهارة تبدو متوافقة مع الغرض، لكنها يمكنها
التصرف بسلطة حساب ذات معنى.

## النتائج

تشرح النتائج سبب عرض نتيجة التدقيق. تتضمن كل نتيجة عادة:

- معناها
- سبب تمييزها
- محتوى المهارة أو Plugin ذي الصلة
- توصية

قد تحمل النتائج تسميات `Info` أو `Low` أو `Medium` أو `High` أو `Critical`. تساهم النتائج الأعلى
خطورة بشكل أقوى في مستوى المخاطر وحالة التدقيق.

تُخفى النتائج منخفضة الثقة من ملخص التدقيق العام حتى تظل الصفحة
مركزة على الأدلة المفيدة.

## ما الذي يتحقق منه ClawHub

يدقق ClawHub عناصر الإصدار المقدمة، بما في ذلك:

- تعليمات المهارة أو بيانات تعريف Plugin
- متغيرات البيئة والأذونات المعلنة
- تعليمات التثبيت وبيانات تعريف الحزمة
- الملفات المضمنة وبيانات ملفاتها
- بيانات تعريف التوافق والقدرات

السؤال الرئيسي هو الاتساق: هل يتوافق الاسم والملخص وبيانات التعريف والسلطة
المطلوبة والمحتوى الفعلي مع ما يتوقعه المستخدمون بشكل معقول؟

السلوك القوي ليس سيئا تلقائيا. تحتاج كثير من الأدوات المفيدة إلى بيانات اعتماد
أو أوامر محلية أو واجهات API لمزودين أو عمليات تثبيت حزم. يتحقق التدقيق مما إذا كانت تلك
السلطة متوقعة ومفصحا عنها ومتناسبة.

ترتبط صفحات العناصر بالتدقيق الكامل على:

```text
/<owner>/skills/<slug>/security-audit
```

تجمع صفحة التدقيق بين:

1. SkillSpector
2. VirusTotal
3. تحليل المخاطر

## VirusTotal

يستخدم ClawHub VirusTotal كقياسات عن البرمجيات الخبيثة ضمن مجموعة التدقيق. يعد VirusTotal
معيارا موثوقا في الصناعة لسمعة الملفات وفحص البرمجيات الخبيثة، وتتيح شراكتنا
لـ ClawHub إضافة معلومات أمان أوسع إلى مراجعة المهارات وPlugin.

يعد VirusTotal مفيدا بشكل خاص للعناصر الخبيثة المعروفة، واكتشافات المحركات، و
إشارات السمعة التي تكمل مراجعة ClawHub الواعية بالوكلاء. عندما تتوفر
أعداد محركات البائعين، يلخصها التدقيق بلغة واضحة، مثل:

```text
62/62 vendors flagged this skill as clean.
```

أو:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

عندما لا يمتلك ClawHub قياسات أعداد البائعين لتلخيصها، يقول التدقيق:

```text
No VirusTotal findings
```

يبقى VirusTotal قياسات. ولا يستبدل تحليل المخاطر الخاص بـ ClawHub
والواعي بالعناصر.

## تحليل المخاطر

يعمل تحليل المخاطر داخليا بواسطة ClawScan، وهو نظام تدقيق الأمان الخاص بـ ClawHub.
يراجع كل إصدار باعتباره عنصرا موجها إلى الوكيل: التعليمات،
وبيانات التعريف، والأذونات المعلنة، والملفات، وإشارات القدرات، وإشارات الفحص الثابت،
ونتائج SkillSpector، وقياسات VirusTotal، والسياق المقدم من الناشر.
إشارات الفحص الثابت هي سياق داخلي لهذه المراجعة؛ وليست قسما عاما مستقلا
في التدقيق أو حكما يمنع التثبيت.

يستخدم تحليل المخاطر
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
كمنظور للمخاطر مثل حقن المطالبات، وإساءة استخدام الأدوات، وكشف بيانات الاعتماد،
والتنفيذ غير الآمن، وتسميم الذاكرة أو السياق، والوكالة المفرطة.

لا يتعامل ClawScan مع القدرة التي تبدو مخيفة على أنها ضارة تلقائيا.
بل يسأل ما إذا كانت القدرة مفصحا عنها، ومتوافقة مع الغرض، ومدعومة
بحالة الاستخدام المعلنة للإصدار.
