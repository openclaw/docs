# عمليات التدقيق الأمني

تساعدك عمليات التدقيق الأمني في ClawHub على تحديد ما إذا كانت المهارة أو Plugin آمنة بما يكفي
للتثبيت. فهي تعرض ما يفعله الإصدار، وما الصلاحيات التي يطلبها، وما إذا كان هناك
أي شيء يستحق انتباها إضافيا قبل أن يتمكن من الوصول إلى الملفات أو الحسابات أو
بيانات الاعتماد أو التعليمات البرمجية أو الخدمات الخارجية.

تعد عمليات التدقيق إشارات أمان قوية، لكنها ليست ضمانا بأن الإصدار
خال تماما من المخاطر. استخدم حكمك دائما قبل منح وصول حساس.

انظر أيضا [الأمان](/ar/clawhub/security)، و[الاستخدام المقبول](/ar/clawhub/acceptable-usage)،
و[الإشراف وسلامة الحساب](/ar/clawhub/moderation).

## ما يجب التحقق منه قبل التثبيت

قبل التثبيت، راجع:

- حالة التدقيق الإجمالية
- مستوى المخاطر
- أي نتائج مدرجة
- بيانات الاعتماد أو الأذونات أو متغيرات البيئة المطلوبة
- المالك، والمصدر، والإصدار، وسجل التغييرات، والتنزيلات، والنجوم، وإشارات الثقة الأخرى

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالة التدقيق

تخبرك حالة التدقيق بكيفية التعامل مع نتيجة التدقيق:

| الحالة      | المعنى                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | لم يتم العثور على مشكلة ظاهرة تتجاوز المخاطر المنخفضة.                                |
| `Review`    | اقرأ النتائج قبل التثبيت. قد يظل الإصدار مشروعا. |
| `Warn`      | توخ الحذر الإضافي. وجد ClawHub مشكلة ذات أثر كبير أو إشارة تحذير. |
| `Malicious` | لا تثبّت.                                                           |
| `Pending`   | لم تكتمل عمليات التدقيق بعد.                                             |
| `Error`     | تعذر إكمال التدقيق.                                         |

تعد حالة `Pass` مطمئنة، لكنها لا تغني عن حكمك الخاص. وهذا مهم
بشكل خاص للأدوات التي يمكنها نشر المحتوى أو تعديل البيانات أو تشغيل الأوامر أو قراءة الملفات أو
الوصول إلى أنظمة الإنتاج.

## مستوى المخاطر

يصف مستوى المخاطر نطاق التأثير: مقدار القوة التي يبدو أن الإصدار يمتلكها إذا
استخدمته على النحو المقصود.

| مستوى المخاطر | المعنى                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | وُجدت صلاحيات حساسة محدودة أو أثر محدود على المستخدم.                          |
| `Medium`   | يمتلك الإصدار صلاحيات ذات معنى، مثل الوصول إلى الحسابات أو تغيير البيانات. |
| `High`     | يمتلك الإصدار صلاحيات ذات أثر كبير، أو نتائج خطيرة، أو إشارات خبيثة. |

يجيب مستوى المخاطر وحالة التدقيق عن سؤالين مختلفين:

- يسأل مستوى المخاطر: "ما مقدار القوة هنا؟"
- تسأل حالة التدقيق: "ماذا ينبغي أن أفعل بهذه النتيجة؟"

على سبيل المثال، قد تعرض مهارة نشر حالة `Review` مع مخاطر `Medium`. وهذا
لا يعني أنها خبيثة. بل يعني أن المهارة تبدو متوافقة مع الغرض، لكنها تستطيع
التصرف بصلاحيات حساب ذات معنى.

## النتائج

توضح النتائج سبب عرض نتيجة تدقيق معينة. تتضمن كل نتيجة عادة:

- ما تعنيه
- سبب وضع علامة عليها
- محتوى المهارة أو Plugin ذي الصلة
- توصية

قد تُوسم النتائج بـ `Info` أو `Low` أو `Medium` أو `High` أو `Critical`. تسهم
النتائج الأعلى شدة بشكل أقوى في مستوى المخاطر وحالة التدقيق.

تُخفى النتائج منخفضة الثقة من ملخص التدقيق العام حتى تظل الصفحة
مركزة على الأدلة المفيدة.

## ما الذي يتحقق منه ClawHub

يدقق ClawHub عناصر الإصدار المقدمة، بما في ذلك:

- تعليمات المهارة أو بيانات Plugin الوصفية
- متغيرات البيئة والأذونات المعلنة
- تعليمات التثبيت وبيانات الحزمة الوصفية
- الملفات المضمنة وبيانات الملفات
- بيانات التوافق والقدرات الوصفية

السؤال الرئيسي هو الاتساق: هل يتوافق الاسم والملخص والبيانات الوصفية والصلاحيات
المطلوبة والمحتوى الفعلي مع ما يتوقعه المستخدمون بشكل معقول؟

السلوك القوي ليس سيئا تلقائيا. تحتاج كثير من الأدوات المفيدة إلى بيانات اعتماد،
أو أوامر محلية، أو واجهات API لموفرين، أو تثبيت حزم. يتحقق التدقيق مما إذا كانت تلك
القوة متوقعة ومفصحا عنها ومتناسبة.

تربط صفحات العناصر بالتدقيق الكامل على:

```text
/<owner>/skills/<slug>/security-audit
```

تجمع صفحة التدقيق بين:

1. SkillSpector
2. VirusTotal
3. تحليل المخاطر

## VirusTotal

يستخدم ClawHub VirusTotal كبيانات قياس عن البرمجيات الضارة ضمن مجموعة التدقيق. يعد VirusTotal
معيارا موثوقا في القطاع لسمعة الملفات وفحص البرمجيات الضارة، وتتيح شراكتنا
لـ ClawHub إضافة معلومات أمان أوسع إلى مراجعة المهارات وPlugin.

يفيد VirusTotal بشكل خاص في العناصر الخبيثة المعروفة، ومطابقات المحركات، وإشارات
السمعة التي تكمل مراجعة ClawHub المدركة للوكيل. عندما تتوفر أعداد محركات
الموردين، يلخصها التدقيق بلغة واضحة، مثل:

```text
62/62 vendors flagged this skill as clean.
```

أو:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

عندما لا يمتلك ClawHub بيانات قياس لأعداد الموردين يمكن تلخيصها، يقول التدقيق:

```text
No VirusTotal findings
```

يبقى VirusTotal بيانات قياس. ولا يحل محل تحليل المخاطر الخاص بـ ClawHub
والمدرك للعناصر.

## تحليل المخاطر

يعمل تحليل المخاطر داخليا بواسطة ClawScan، وهو نظام التدقيق الأمني الخاص
بـ ClawHub. يراجع كل إصدار بوصفه عنصرا موجها إلى وكيل: التعليمات،
والبيانات الوصفية، والأذونات المعلنة، والملفات، وإشارات القدرات، وإشارات الفحص الثابت،
ونتائج SkillSpector، وبيانات VirusTotal، والسياق الذي يوفره الناشر.
إشارات الفحص الثابت هي سياق داخلي لهذه المراجعة؛ وليست
قسما عاما مستقلا من التدقيق أو حكما يحظر التثبيت.

يستخدم تحليل المخاطر
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
كعدسة للمخاطر مثل حقن المطالبات، وإساءة استخدام الأدوات، وكشف بيانات الاعتماد،
والتنفيذ غير الآمن، وتسميم الذاكرة أو السياق، والوكالة المفرطة.

لا يتعامل ClawScan مع قدرة تبدو مخيفة على أنها خبيثة تلقائيا.
بل يسأل ما إذا كانت القدرة مفصحا عنها، ومتوافقة مع الغرض، ومدعومة
بحالة الاستخدام المعلنة للإصدار.
