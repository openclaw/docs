# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub عبر إرشادات GitHub الأمنية الخاصة بـ
`openclaw/clawhub`.

استخدم إرشادات GitHub الأمنية للثغرات في ClawHub نفسه. تتضمن تقارير
إرشادات ClawHub الجيدة أخطاءً في:

- موقع ClawHub أو API أو CLI
- نشر السجل، أو التنزيلات، أو عمليات التثبيت، أو سلامة العناصر الأثرية
- المصادقة، أو التفويض، أو رموز API
- الفحص، أو الإشراف، أو معالجة البلاغات

لا تستخدم إرشادات ClawHub للثغرات الموجودة في الشيفرة المصدرية الخاصة بمهارة تابعة لجهة خارجية أو
Plugin تابع لها. أبلغ عنها مباشرةً إلى الناشر أو مستودع المصدر
المرتبط من قائمة ClawHub.

## الإفصاح عن الثغرات

لأن ClawHub تطبيق سحابي مستضاف، لا يُفصح علنًا عن ثغرات خدمة ClawHub
افتراضيًا. يُفصح عنها علنًا عندما توجد أدلة على تأثير حقيقي في المستخدمين أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الحقيقي في المستخدمين الاستغلال المؤكد، أو كشف بيانات المستخدمين
أو أسرارهم، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد، أو تحديث البرمجيات المحلية، أو
اتخاذ إجراء وقائي آخر.

يُفصح علنًا عن الثغرات في البرمجيات التي يثبتها المستخدمون، مثل
حزم CLI الخاصة بـ ClawHub، أو الملفات الثنائية، أو المكتبات، أو عناصر الإصدار الأخرى التي يحتاج المستخدمون
إلى تحديثها محليًا.
