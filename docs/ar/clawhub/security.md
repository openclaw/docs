# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub عبر GitHub Security Advisories لـ
`openclaw/clawhub`.

استخدم GitHub Security Advisories للثغرات في ClawHub نفسه. تتضمن بلاغات
ClawHub الاستشارية الجيدة أخطاءً في:

- موقع ClawHub الإلكتروني أو API أو CLI
- النشر في السجل أو التنزيلات أو التثبيتات أو سلامة الأثر
- المصادقة أو التفويض أو رموز API
- الفحص أو الإشراف أو التعامل مع البلاغات

لا تستخدم بلاغات ClawHub الاستشارية للثغرات الموجودة في شفرة المصدر الخاصة بمهارة تابعة لجهة خارجية أو Plugin. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر المرتبط من قائمة ClawHub.

## الإفصاح عن الثغرات

نظرًا لأن ClawHub تطبيق سحابي مستضاف، لا يتم الإفصاح عن ثغرات خدمة ClawHub
علنًا افتراضيًا. يتم الإفصاح عنها علنًا عندما يوجد دليل على تأثير حقيقي في
المستخدمين أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الحقيقي في المستخدمين الاستغلال المؤكد، أو كشف بيانات
المستخدمين أو أسرارهم، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد، أو تحديث البرامج
المحلية، أو اتخاذ إجراء وقائي آخر.

يتم الإفصاح علنًا عن الثغرات في البرامج التي يثبتها المستخدمون، مثل حزم
ClawHub CLI أو الثنائيات أو المكتبات أو غيرها من آثار الإصدار التي يحتاج
المستخدمون إلى تحديثها محليًا.
