# الأمان + الإشراف

ClawHub مفتوح للنشر، لكن القوائم العامة لا تزال تمر عبر ضوابط الثقة
والفحص والإبلاغ والإشراف. الهدف عملي: مساعدة المستخدمين على
فحص ما يثبتونه، ومنح الناشرين مسار استرداد للحالات الإيجابية الكاذبة،
وإبقاء الحزم المسيئة خارج الاكتشاف العام.

راجع أيضًا [الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## ما يمكن للمستخدمين فحصه

قبل تثبيت Skills أو Plugin، تحقق من قائمته على ClawHub لمعرفة:

- المالك وإسناد المصدر
- أحدث إصدار وسجل التغييرات
- متغيرات البيئة أو الأذونات المطلوبة
- بيانات توافق تعريفية للـ Plugins
- حالة الفحص أو الإشراف
- البلاغات والتعليقات والنجوم والتنزيلات وإشارات التثبيت حيث تظهر

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالات الفحص

قد يعرض ClawHub نتائج الفحص أو الإشراف على الصفحات العامة وتشخيصات
مرئية للمالك.

تشمل النتائج الشائعة:

- `clean`: لم تُعثر أي مشكلة مانعة.
- `suspicious`: يتطلب الإصدار الحذر أو المراجعة.
- `malicious`: يُعد الإصدار غير آمن.
- `pending`: لم تكتمل الفحوصات بعد.
- `held` أو `quarantined` أو `revoked` أو `hidden`: الإصدار غير متاح بالكامل
  على أسطح التثبيت العامة.

قد تختلف الصياغة الدقيقة حسب السطح، لكن المعنى العملي واحد: إذا كان
الإصدار موقوفًا أو محظورًا، فينبغي ألا يثبته المستخدمون حتى يحل المالك
المشكلة أو يعيده الإشراف.

## Skills

تفحص عمليات فحص Skills حزمة Skills المنشورة، والبيانات التعريفية، والمتطلبات
المصرح بها، والتعليمات المشبوهة.

يولي ClawHub اهتمامًا خاصًا لعدم التطابق بين ما تصرح به Skills وما يبدو
أنها تفعله. على سبيل المثال، ينبغي أن تصرح Skills التي تشير إلى مفتاح API
مطلوب بهذا المتطلب في `SKILL.md` حتى يتمكن المستخدمون من رؤيته قبل
التثبيت.

تستند نتائج الفحص إلى الأثر. يُتعامل مع سلوك المزود المتوقع، مثل بيانات
اعتماد API المصرح بها، أو عمليات رد نداء OAuth على localhost، أو تنظيف
إلغاء التثبيت المحدد النطاق، أو ترميز Basic Auth، أو تحميلات الملفات التي
يحددها المستخدم إلى المزود المعلن، بشكل مختلف عن إعادة توجيه بيانات
الاعتماد المخفية، أو الوصول الواسع إلى الملفات الخاصة، أو وجهات الشبكة غير
ذات الصلة، أو إساءة استخدام المتصفح خلسة.

راجع [تنسيق Skills](/ar/clawhub/skill-format).

## Plugins

تتضمن إصدارات Plugin بيانات تعريفية للحزمة، وإسناد المصدر، وحقول التوافق،
ومعلومات سلامة الأثر.

يتحقق OpenClaw من التوافق قبل تثبيت Plugins المستضافة على ClawHub. وقد
تعرض سجلات الحزم أيضًا بيانات تعريفية للملخص حتى يتمكن OpenClaw من التحقق
من الآثار التي تم تنزيلها. يتضمن ClawScan بيانات `openclaw.environment`
الخاصة بالبيئة/الإعدادات المصرح بها للحزمة عند مراجعة إصدارات Plugin، بحيث
تُقارن متطلبات التشغيل المصرح بها بالسلوك المرصود.

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن Skills والحزم والتعليقات.

ينبغي أن تكون البلاغات محددة وقابلة للتنفيذ. وقد يؤدي إساءة استخدام
الإبلاغ بحد ذاته إلى إجراء بحق الحساب.

أمثلة على البلاغات:

- بيانات تعريفية مضللة
- متطلبات بيانات اعتماد أو أذونات غير مصرح بها
- تعليمات تثبيت مشبوهة
- تعليقات احتيالية أو انتحال شخصية
- تسجيلات سيئة النية أو إساءة استخدام العلامات التجارية
- محتوى يخالف [الاستخدام المقبول](/ar/clawhub/acceptable-usage)

## ملاحظات ClawScan للناشرين

يمكن للناشرين تقديم ملاحظة ClawScan اختيارية عند نشر Skills أو Plugin.
تمنح هذه الملاحظة ClawScan سياقًا للسلوك الذي قد يبدو غير معتاد بخلاف ذلك،
مثل الوصول إلى الشبكة، أو الوصول إلى مضيف أصلي، أو بيانات اعتماد خاصة
بمزود.

## إيقافات الإشراف

عندما يضع الفاحص الثابت علامة على Skills مرفوعة على أنها ضارة، يوضع الناشر
تلقائيًا تحت إيقاف إشرافي (`requiresModerationAt` مضبوطة على المستخدم).
يخفي هذا كل Skills الخاصة بالناشر، ويتسبب في بدء عمليات النشر المستقبلية
مخفية، وينشئ إدخال سجل تدقيق `user.moderation.auto`.

تُحتفظ بالنتائج الثابتة المشبوهة كأدلة ملف/سطر للمشرفين، لكنها لا تخفي
المحتوى ولا تحدد حكم الفحص العام بمفردها. تظل الرفعات الجديدة في حالة
المراجعة/الانتظار إلى أن تستقر مراجعة LLM. لا يحظر الفحص الثابت فورًا إلا
للتوقيعات الضارة. تظل نتائج محركات VirusTotal أدلة أمان مرئية، لكن أحكام
VirusTotal Code Insight/Palm إرشادية ولا تخفي Skills بمفردها. تحتفظ مراجعات
ClawScan LLM بالملاحظات المتوافقة مع الغرض كإرشادات. تظل نتائج المراجعة
المتوسطة مرئية على الأثر، بينما يُحجز مرشح الاشتباه لمخاوف LLM عالية
التأثير، أو النتائج الضارة، أو اكتشافات محركات مكافحة الفيروسات المؤكدة.

يمكن للمسؤولين رفع إيقاف إيجابي كاذب:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

يمسح هذا `requiresModerationAt` و`requiresModerationReason`، ويعيد Skills
التي أخفاها الإيقاف على مستوى المستخدم، ويكتب إدخال سجل تدقيق
`user.moderation.lift`. تظل Skills المخفية لأسباب أخرى، أو التي لا يزال
فحصها الثابت نفسه ضارًا، مخفية.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تخالف سياسة ClawHub صلاحية النشر. وقد تؤدي إساءة
الاستخدام الشديدة إلى حظر الحسابات، أو إبطال الرموز، أو إخفاء المحتوى، أو
إزالة القوائم.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز ClawHub API.
إذا بدأ توثيق CLI يفشل بعد إجراء على الحساب، فسجّل الدخول إلى واجهة الويب
لمراجعة حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محظورًا،
فاتصل بـ security@openclaw.ai لمراجعة الاسترداد.

## إرشادات الناشرين

لتقليل الإيجابيات الكاذبة وتحسين ثقة المستخدم:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغييرات
- صرّح بمتغيرات البيئة والأذونات المطلوبة
- أضف ملاحظة ClawScan للناشر عندما يتضمن الإصدار سلوكًا غير معتاد لكنه مقصود
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنًا
- استخدم التشغيلات الجافة قبل نشر Plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الحزمة
