# الأمان + الإشراف

ClawHub مفتوح للنشر، لكن القوائم العامة لا تزال تمر عبر ضوابط الثقة
والفحص والإبلاغ والإشراف. الهدف عملي: مساعدة المستخدمين على
فحص ما يثبتونه، ومنح الناشرين مسار استرداد عند النتائج الإيجابية الكاذبة،
وإبقاء الحزم المسيئة خارج الاكتشاف العام.

راجع أيضًا [الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## ما يمكن للمستخدمين فحصه

قبل تثبيت Skills أو Plugin، تحقق من قائمة ClawHub الخاصة به لمعرفة:

- المالك وإسناد المصدر
- أحدث إصدار وسجل التغييرات
- متغيرات البيئة أو الأذونات المطلوبة
- بيانات تعريف التوافق لـ Plugins
- حالة الفحص أو الإشراف
- البلاغات، والتعليقات، والنجوم، والتنزيلات، وإشارات التثبيت حيثما تظهر

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالات الفحص

قد يعرض ClawHub نتائج الفحص أو الإشراف على الصفحات العامة والتشخيصات
المرئية للمالك.

تشمل النتائج الشائعة:

- `clean`: لم يتم العثور على مشكلة حاجبة.
- `suspicious`: يحتاج الإصدار إلى الحذر أو المراجعة.
- `malicious`: يُعد الإصدار غير آمن.
- `pending`: لم تكتمل الفحوصات بعد.
- `held` أو `quarantined` أو `revoked` أو `hidden`: الإصدار غير متاح بالكامل
  على واجهات التثبيت العامة.

قد تختلف الصياغة الدقيقة حسب الواجهة، لكن المعنى العملي واحد: إذا كان
الإصدار معلّقًا أو محظورًا، فينبغي ألا يثبته المستخدمون حتى يحل المالك
المشكلة أو يعيده الإشراف.

## Skills

تفحص عمليات مسح Skills حزمة Skills المنشورة، وبيانات التعريف، والمتطلبات
المعلنة، والتعليمات المشبوهة.

يهتم ClawHub خصوصًا بعدم التطابق بين ما تعلنه Skills وما يبدو أنها تفعله.
على سبيل المثال، ينبغي لـ Skills التي تشير إلى مفتاح API مطلوب أن تعلن ذلك
المتطلب في `SKILL.md` حتى يتمكن المستخدمون من رؤيته قبل التثبيت.

تستند نتائج الفحص إلى الآثار. يُعامل سلوك المزوّد المتوقع، مثل بيانات اعتماد
API المعلنة، وعمليات رد نداء OAuth على المضيف المحلي، والتنظيف محدود النطاق
عند إلغاء التثبيت، وترميز Basic Auth، أو تحميلات الملفات التي يختارها
المستخدم إلى المزوّد المعلن، بشكل مختلف عن تمرير بيانات الاعتماد المخفي،
أو الوصول الواسع إلى الملفات الخاصة، أو وجهات الشبكة غير ذات الصلة، أو إساءة
استخدام المتصفح خلسة.

راجع [تنسيق Skills](/ar/clawhub/skill-format).

## Plugins

تتضمن إصدارات Plugin بيانات تعريف الحزمة، وإسناد المصدر، وحقول التوافق،
ومعلومات سلامة الآثار.

يتحقق OpenClaw من التوافق قبل تثبيت Plugins المستضافة على ClawHub. قد تعرض
سجلات الحزم أيضًا بيانات تعريف الملخص حتى يتمكن OpenClaw من التحقق من
الآثار المنزّلة. يتضمن ClawScan بيانات تعريف البيئة/الإعداد
`openclaw.environment` المعلنة للحزمة عند مراجعة إصدارات Plugin حتى تتم
مقارنة متطلبات وقت التشغيل المعلنة بالسلوك المرصود.

## البلاغات

يمكن للمستخدمين المسجّلين الإبلاغ عن Skills والحزم والتعليقات.

ينبغي أن تكون البلاغات محددة وقابلة للتنفيذ. ويمكن أن تؤدي إساءة استخدام
الإبلاغ نفسها إلى إجراء على الحساب.

أمثلة على البلاغات:

- بيانات تعريف مضللة
- متطلبات بيانات اعتماد أو أذونات غير معلنة
- تعليمات تثبيت مشبوهة
- تعليقات احتيالية أو انتحال شخصية
- تسجيلات بسوء نية أو إساءة استخدام علامة تجارية
- محتوى ينتهك [الاستخدام المقبول](/ar/clawhub/acceptable-usage)

## ملاحظات ClawScan للناشرين

يمكن للناشرين تقديم ملاحظة ClawScan اختيارية عند نشر Skills أو Plugin. تمنح
هذه الملاحظة ClawScan سياقًا لسلوك قد يبدو غير معتاد، مثل الوصول إلى الشبكة،
أو الوصول إلى المضيف الأصلي، أو بيانات الاعتماد الخاصة بمزوّد معين.

## حالات التعليق الإشرافي

عندما يعلّم الفاحص الثابت Skills مرفوعة على أنها خبيثة، يوضع الناشر تلقائيًا
تحت تعليق إشرافي (`requiresModerationAt` مضبوط على المستخدم). يؤدي ذلك إلى
إخفاء جميع Skills الخاصة بالناشر، وجعل عمليات النشر المستقبلية تبدأ مخفية،
وإنشاء إدخال سجل تدقيق `user.moderation.auto`.

تُحتفظ بالنتائج الثابتة المشبوهة كدليل ملف/سطر للمشرفين، لكنها لا تخفي
المحتوى ولا تحدد نتيجة الفحص العامة بمفردها. تبقى الرفعات الجديدة في حالة
المراجعة/الانتظار حتى تستقر مراجعة LLM. لا يحظر الفحص الثابت فورًا إلا
للتواقيع الخبيثة. تبقى إصابات محركات VirusTotal أدلة أمنية مرئية، لكن أحكام
VirusTotal Code Insight/Palm استشارية ولا تخفي Skills بمفردها. تحتفظ مراجعات
ClawScan LLM بالملاحظات المتوافقة مع الغرض كإرشاد. تبقى نتائج المراجعة
المتوسطة مرئية على الأثر، بينما يُحجز مرشح الاشتباه لمخاوف LLM عالية التأثير،
أو النتائج الخبيثة، أو اكتشافات محركات مكافحة الفيروسات المؤيدة بأدلة.

يمكن للمسؤولين رفع تعليق إيجابي كاذب:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

يمسح هذا `requiresModerationAt` و`requiresModerationReason`، ويعيد Skills
المخفية بسبب التعليق على مستوى المستخدم، ويكتب إدخال سجل تدقيق
`user.moderation.lift`. تبقى Skills المخفية لأسباب أخرى، أو التي يظل فحصها
الثابت نفسه خبيثًا، مخفية.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub حق الوصول إلى النشر. يمكن أن تؤدي
الإساءة الشديدة إلى حظر الحساب، أو إبطال الرموز، أو إخفاء المحتوى، أو إزالة
القوائم.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز ClawHub API.
إذا بدأ فشل مصادقة CLI بعد إجراء على الحساب، فسجّل الدخول إلى واجهة الويب
لمراجعة حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محظورًا،
فاتصل بـ security@openclaw.ai لمراجعة الاسترداد.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الكاذبة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء، والملخصات، والوسوم، وسجلات التغييرات
- أعلن متغيرات البيئة والأذونات المطلوبة
- أضف ملاحظة ClawScan للناشر عندما يحتوي الإصدار على سلوك غير معتاد لكنه مقصود
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنًا
- استخدم عمليات التشغيل الجاف قبل نشر Plugins
- رد بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الحزمة
