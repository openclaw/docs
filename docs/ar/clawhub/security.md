# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub عبر استشارات أمان GitHub الخاصة بـ
`openclaw/clawhub`.

استخدم استشارات أمان GitHub للثغرات في ClawHub نفسه. تتضمن تقارير استشارات
ClawHub الجيدة أخطاء في:

- موقع ClawHub أو API أو CLI
- نشر السجل أو التنزيلات أو عمليات التثبيت أو سلامة الأثر
- المصادقة أو التفويض أو رموز API
- الفحص أو الإشراف أو معالجة التقارير

لا تستخدم استشارات ClawHub للثغرات الموجودة في شيفرة المصدر الخاصة بمهارة تابعة لجهة خارجية أو
Plugin تابع لها. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من إدراج ClawHub.

## الإفصاح عن الثغرات

نظرًا لأن ClawHub تطبيق سحابي مستضاف، فإن ثغرات خدمة ClawHub
لا يُفصح عنها علنًا افتراضيًا. يُفصح عنها علنًا عند وجود
دليل على تأثير حقيقي في المستخدمين أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الحقيقي في المستخدمين الاستغلال المؤكد، أو كشف بيانات
المستخدمين أو أسرارهم، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد أو تحديث البرامج المحلية أو
اتخاذ إجراء وقائي آخر.

يُفصح علنًا عن الثغرات في البرامج المثبتة من قبل المستخدمين، مثل
حزم ClawHub CLI أو الملفات الثنائية أو المكتبات أو آثار الإصدار الأخرى التي يحتاج المستخدمون
إلى تحديثها محليًا.

## الصفحات ذات الصلة

للاطلاع على تسميات تدقيق وقت التثبيت، ومستويات المخاطر، والنتائج، والتفسير، راجع
