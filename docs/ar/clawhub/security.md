# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub عبر GitHub Security Advisories لـ
`openclaw/clawhub`.

استخدم GitHub Security Advisories للثغرات في ClawHub نفسه. تتضمن تقارير
استشارات ClawHub الجيدة الأخطاء في:

- موقع ClawHub أو API أو CLI
- نشر السجل أو التنزيلات أو عمليات التثبيت أو سلامة العناصر الأثرية
- المصادقة أو التفويض أو رموز API
- الفحص أو الإشراف أو التعامل مع البلاغات

لا تستخدم استشارات ClawHub للثغرات في الشفرة المصدرية الخاصة بمهارة أو
plugin تابع لطرف ثالث. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من قائمة ClawHub.

## الإفصاح عن الثغرات

لأن ClawHub تطبيق سحابي مستضاف، لا تُفصح ثغرات خدمة ClawHub علنًا افتراضيًا.
يُفصح عنها علنًا عندما توجد أدلة على تأثير حقيقي في المستخدمين أو عندما يحتاج
المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الحقيقي في المستخدمين الاستغلال المؤكد، أو كشف بيانات
المستخدمين أو أسرارهم، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد، أو تحديث البرامج المحلية،
أو اتخاذ إجراء وقائي آخر.

يُفصح علنًا عن الثغرات في البرامج التي يثبتها المستخدمون، مثل حزم ClawHub CLI
أو الثنائيات أو المكتبات أو غيرها من عناصر الإصدار التي يحتاج المستخدمون إلى
تحديثها محليًا.
