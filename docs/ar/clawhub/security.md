# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub من خلال GitHub Security Advisories لـ
`openclaw/clawhub`.

استخدم GitHub Security Advisories للثغرات في ClawHub نفسه. تتضمن تقارير
إرشادات ClawHub الجيدة أخطاءً في:

- موقع ClawHub أو API أو CLI
- نشر السجل، أو التنزيلات، أو عمليات التثبيت، أو سلامة المصنوعات
- المصادقة، أو التفويض، أو رموز API
- الفحص، أو الإشراف، أو معالجة التقارير

لا تستخدم إرشادات ClawHub للثغرات في الشيفرة المصدرية الخاصة بمهارة أو
plugin تابع لجهة خارجية. أبلغ عنها مباشرةً إلى الناشر أو مستودع المصدر
المرتبط من قائمة ClawHub.

## الإفصاح عن الثغرات

نظرًا لأن ClawHub تطبيق سحابي مستضاف، لا يتم الإفصاح عن ثغرات خدمة ClawHub
علنًا بشكل افتراضي. يتم الإفصاح عنها علنًا عند وجود دليل على تأثير حقيقي على
المستخدمين أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الحقيقي على المستخدمين الاستغلال المؤكد، أو كشف بيانات
المستخدمين أو الأسرار، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد، أو تحديث البرامج
المحلية، أو اتخاذ إجراء وقائي آخر.

يتم الإفصاح علنًا عن الثغرات في البرامج المثبتة من قبل المستخدم، مثل حزم
ClawHub CLI أو الثنائيات أو المكتبات أو غيرها من مصنوعات الإصدار التي يحتاج
المستخدمون إلى تحديثها محليًا.

## الصفحات ذات الصلة

للاطلاع على تسميات التدقيق وقت التثبيت، ومستويات المخاطر، والنتائج، والتفسير، راجع
[تدقيقات الأمان](/clawhub/security-audits).

للاطلاع على تقارير السوق، وتعليقات الإشراف، والقوائم المخفية، والحظر، وحالة الحساب،
راجع [الإشراف وسلامة الحساب](/clawhub/moderation).
