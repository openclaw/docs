# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub من خلال استشارات أمان GitHub الخاصة بـ
`openclaw/clawhub`.

استخدم استشارات أمان GitHub للإبلاغ عن الثغرات في ClawHub نفسه. تتضمن
تقارير استشارات ClawHub الجيدة الأخطاء في:

- موقع ClawHub الإلكتروني أو API أو CLI
- نشر السجل أو التنزيلات أو عمليات التثبيت أو سلامة العناصر البرمجية
- المصادقة أو التخويل أو رموز API المميزة
- الفحص أو الإشراف أو معالجة البلاغات

لا تستخدم استشارات ClawHub للإبلاغ عن الثغرات في الشيفرة المصدرية الخاصة بمهارة
أو Plugin تابع لجهة خارجية. أبلغ عنها مباشرةً إلى الناشر أو مستودع الشيفرة المصدرية
المرتبط من صفحة الإدراج في ClawHub.

## الإفصاح عن الثغرات

نظرًا إلى أن ClawHub تطبيق سحابي مستضاف، لا يُفصح علنًا افتراضيًا عن ثغرات
خدمة ClawHub. ويُفصح عنها علنًا عند وجود دليل على تأثير فعلي في المستخدمين أو
عندما يتعين على المستخدمين اتخاذ إجراء.

تشمل أمثلة التأثير الفعلي في المستخدمين الاستغلال المؤكد، أو كشف بيانات المستخدمين
أو أسرارهم، أو وصول محتوى ضار إلى المستخدمين بسبب إخفاق في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد، أو تحديث البرامج المحلية، أو
اتخاذ إجراء وقائي آخر.

يُفصح علنًا عن الثغرات في البرامج التي يثبتها المستخدمون، مثل
حزم CLI الخاصة بـ ClawHub أو الملفات الثنائية أو المكتبات أو عناصر الإصدار الأخرى التي يحتاج المستخدمون
إلى تحديثها محليًا.

## صفحات ذات صلة

للاطلاع على تسميات التدقيق في وقت التثبيت ومستويات المخاطر والنتائج وتفسيرها، راجع
[عمليات التدقيق الأمني](/clawhub/security-audits).

للاطلاع على بلاغات السوق وعمليات التعليق لأغراض الإشراف والقوائم المخفية وعمليات الحظر
وحالة الحساب، راجع [الإشراف وأمان الحساب](/clawhub/moderation).
