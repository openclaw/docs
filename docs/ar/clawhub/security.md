# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub عبر GitHub Security Advisories الخاصة بـ
`openclaw/clawhub`.

استخدم GitHub Security Advisories للثغرات الموجودة في ClawHub نفسه. تشمل
تقارير إرشادات ClawHub الجيدة الأخطاء في:

- موقع ClawHub أو API أو CLI
- نشر السجل، أو التنزيلات، أو عمليات التثبيت، أو سلامة القطع الأثرية
- المصادقة، أو التفويض، أو رموز API
- الفحص، أو الإشراف، أو معالجة التقارير

لا تستخدم إرشادات ClawHub للثغرات الموجودة في الشيفرة المصدرية الخاصة بـ Skill أو Plugin تابع لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من قائمة ClawHub.

## الإفصاح عن الثغرات

نظرًا لأن ClawHub تطبيق سحابي مستضاف، لا يتم الإفصاح عن ثغرات خدمة ClawHub
علنًا بشكل افتراضي. يتم الإفصاح عنها علنًا عندما توجد أدلة على تأثير حقيقي
في المستخدمين أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الحقيقي في المستخدمين الاستغلال المؤكد، أو كشف بيانات
المستخدمين أو أسرارهم، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد، أو تحديث البرامج المحلية، أو
اتخاذ إجراء وقائي آخر.

يتم الإفصاح علنًا عن الثغرات الموجودة في البرامج التي يثبتها المستخدمون، مثل
حزم ClawHub CLI، أو الملفات الثنائية، أو المكتبات، أو القطع الأثرية الأخرى للإصدارات التي يحتاج المستخدمون
إلى تحديثها محليًا.
