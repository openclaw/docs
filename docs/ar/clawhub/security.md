# الأمان + الإشراف

ClawHub مفتوح للنشر، لكن القوائم العامة تظل تمر عبر ضوابط الثقة،
والفحص، والإبلاغ، والإشراف. الهدف عملي: مساعدة المستخدمين على
فحص ما يثبتونه، ومنح الناشرين مسار استرداد للحالات الإيجابية الكاذبة،
وإبقاء الحزم المسيئة خارج الاكتشاف العام.

راجع أيضًا [الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## ما يمكن للمستخدمين فحصه

قبل تثبيت Skills أو Plugin، تحقق من قائمة ClawHub الخاصة به لمعرفة:

- المالك ونسبة المصدر
- أحدث إصدار وسجل التغييرات
- متغيرات البيئة أو الأذونات المطلوبة
- بيانات تعريف التوافق للـ Plugins
- حالة الفحص أو الإشراف
- البلاغات، والتعليقات، والنجوم، والتنزيلات، وإشارات التثبيت حيث تظهر

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالات الفحص

قد يعرض ClawHub نتائج الفحص أو الإشراف على الصفحات العامة وفي
التشخيصات المرئية للمالك.

تشمل النتائج الشائعة:

- `clean`: لم يتم العثور على مشكلة مانعة.
- `suspicious`: يحتاج الإصدار إلى الحذر أو المراجعة.
- `malicious`: يُعد الإصدار غير آمن.
- `pending`: لم تكتمل الفحوصات بعد.
- `held` أو `quarantined` أو `revoked` أو `hidden`: الإصدار غير متاح بالكامل
  على أسطح التثبيت العامة.

قد تختلف الصياغة الدقيقة حسب السطح، لكن المعنى العملي واحد: إذا كان
الإصدار محجوزًا أو محظورًا، فينبغي ألا يثبته المستخدمون حتى يحل المالك
المشكلة أو يعيده الإشراف.

## Skills

تفحص عمليات فحص Skills حزمة Skills المنشورة، وبياناتها التعريفية، والمتطلبات
المعلنة، والتعليمات المريبة.

يولي ClawHub اهتمامًا خاصًا لحالات عدم التطابق بين ما تعلنه Skills وما
يبدو أنها تفعله. على سبيل المثال، ينبغي للـ Skills التي تشير إلى مفتاح API
مطلوب أن تعلن ذلك المتطلب في `SKILL.md` حتى يتمكن المستخدمون من رؤيته قبل
التثبيت.

تستند نتائج الفحص إلى الأثر. ويُعامل سلوك المزوّد المتوقع، مثل بيانات اعتماد
API المعلنة، أو عمليات رد نداء OAuth على localhost، أو تنظيف إلغاء التثبيت
المحدود النطاق، أو ترميز Basic Auth، أو تحميلات الملفات التي يختارها المستخدم
إلى المزوّد المعلن، بطريقة مختلفة عن تمرير بيانات الاعتماد المخفي، أو الوصول
الواسع إلى الملفات الخاصة، أو وجهات الشبكة غير ذات الصلة، أو إساءة استخدام
المتصفح خفية.

راجع [تنسيق Skills](/ar/clawhub/skill-format).

## Plugins

تتضمن إصدارات Plugin بيانات تعريف الحزمة، ونسبة المصدر، وحقول التوافق،
ومعلومات سلامة الأثر.

يتحقق OpenClaw من التوافق قبل تثبيت Plugins المستضافة على ClawHub. وقد
تعرض سجلات الحزم أيضًا بيانات تعريف الملخص حتى يتمكن OpenClaw من التحقق
من الآثار التي تم تنزيلها. يتضمن ClawScan بيانات تعريف البيئة/الإعداد
`openclaw.environment` المعلنة للحزمة عند مراجعة إصدارات Plugin، بحيث
تُقارن متطلبات التشغيل المعلنة بالسلوك المرصود.

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن Skills، والحزم، والتعليقات.

ينبغي أن تكون البلاغات محددة وقابلة للتنفيذ. ويمكن أن تؤدي إساءة استخدام
الإبلاغ بحد ذاتها إلى إجراء على الحساب.

أمثلة على البلاغات:

- بيانات تعريف مضللة
- متطلبات بيانات اعتماد أو أذونات غير معلنة
- تعليمات تثبيت مريبة
- تعليقات احتيالية أو انتحال شخصية
- تسجيلات بسوء نية أو إساءة استخدام علامة تجارية
- محتوى ينتهك [الاستخدام المقبول](/ar/clawhub/acceptable-usage)

## ملاحظات ClawScan للناشرين

يمكن للناشرين تقديم ملاحظة ClawScan اختيارية عند نشر Skills أو Plugin. تمنح
هذه الملاحظة ClawScan سياقًا للسلوك الذي قد يبدو غير معتاد بخلاف ذلك،
مثل الوصول إلى الشبكة، أو الوصول إلى المضيف الأصلي، أو بيانات الاعتماد
الخاصة بالمزوّد.

## حجوزات الإشراف

عندما يضع الفاحص الثابت علامة على Skills مرفوعة باعتبارها خبيثة، يُوضع
الناشر تلقائيًا تحت حجز إشراف (`requiresModerationAt` مضبوط على المستخدم).
يؤدي ذلك إلى إخفاء جميع Skills الخاصة بالناشر، وجعل عمليات النشر المستقبلية
تبدأ مخفية، وإنشاء إدخال سجل تدقيق `user.moderation.auto`.

تُحتفظ بالنتائج الثابتة المريبة كدليل ملف/سطر للمشرفين، لكنها لا تخفي المحتوى
أو تقرر حكم الفحص العام بمفردها. تظل الرفعات الجديدة في حالة مراجعة/انتظار
حتى تستقر مراجعة LLM. لا يحظر الفحص الثابت فورًا إلا عند وجود توقيعات خبيثة.
تظل إصابات محركات VirusTotal مرئية كدليل أمني، لكن أحكام VirusTotal Code
Insight/Palm استشارية ولا تخفي Skills بمفردها. تحتفظ مراجعات ClawScan LLM
بالملاحظات المتوافقة مع الغرض كإرشاد. تظل نتائج المراجعة المتوسطة مرئية على
الأثر، بينما يُحجز مرشح الاشتباه لمخاوف LLM عالية التأثير، أو النتائج الخبيثة،
أو اكتشافات محركات مكافحة الفيروسات المدعومة بأدلة.

يمكن للمسؤولين رفع حجز إيجابي كاذب:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

يمسح هذا `requiresModerationAt` و`requiresModerationReason`، ويستعيد Skills
التي أخفاها الحجز على مستوى المستخدم، ويكتب إدخال سجل تدقيق
`user.moderation.lift`. تظل Skills المخفية لأسباب أخرى، أو التي يبقى فحصها
الثابت نفسه خبيثًا، مخفية.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub إمكانية النشر. ويمكن أن تؤدي
الإساءات الجسيمة إلى حظر الحساب، أو إبطال الرموز، أو إخفاء المحتوى، أو إزالة
القوائم.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز ClawHub API.
إذا بدأ فشل مصادقة CLI بعد إجراء على الحساب، فسجّل الدخول إلى واجهة الويب
لمراجعة حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محظورًا،
فاتصل بـ security@openclaw.ai لمراجعة الاسترداد.

## إرشادات الناشرين

لتقليل الإيجابيات الكاذبة وتحسين ثقة المستخدم:

- حافظ على دقة الأسماء، والملخصات، والوسوم، وسجلات التغييرات
- أعلن متغيرات البيئة والأذونات المطلوبة
- أضف ملاحظة ClawScan للناشر عندما يتضمن الإصدار سلوكًا غير معتاد لكنه مقصود
- تجنب أوامر التثبيت المموهة
- اربط بالمصدر عندما يكون ذلك ممكنًا
- استخدم التشغيل التجريبي قبل نشر Plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الحزمة
