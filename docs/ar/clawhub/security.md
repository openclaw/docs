# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub من خلال GitHub Security Advisories الخاصة بـ
`openclaw/clawhub`.

استخدم GitHub Security Advisories للثغرات في ClawHub نفسه. تتضمن تقارير
ClawHub الجيدة للثغرات أخطاءً في:

- موقع ClawHub الإلكتروني أو API أو CLI
- نشر السجل أو التنزيلات أو التثبيتات أو سلامة الأثر
- المصادقة أو التفويض أو رموز API
- الفحص أو الإشراف أو معالجة البلاغات

لا تستخدم إرشادات ClawHub للثغرات في الشيفرة المصدرية الخاصة بمهارة تابعة لجهة خارجية أو
Plugin. أبلغ عنها مباشرةً إلى الناشر أو مستودع المصدر
المرتبط من قائمة ClawHub.

## الإفصاح عن الثغرات

نظرًا لأن ClawHub تطبيق سحابي مستضاف، فإن ثغرات خدمة ClawHub
لا يُفصح عنها علنًا افتراضيًا. يُفصح عنها علنًا عندما توجد
أدلة على تأثير حقيقي في المستخدمين أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الحقيقي في المستخدمين الاستغلال المؤكد، أو كشف بيانات المستخدمين
أو الأسرار، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد أو تحديث البرامج المحلية أو
اتخاذ إجراء وقائي آخر.

يُفصح علنًا عن الثغرات في البرامج التي يثبتها المستخدمون، مثل
حزم ClawHub CLI أو الثنائيات أو المكتبات أو غيرها من آثار الإصدارات التي يحتاج المستخدمون
إلى تحديثها محليًا.

## الصفحات ذات الصلة

لتسميات التدقيق وقت التثبيت، ومستويات المخاطر، والنتائج، والتفسير، راجع
[تدقيقات الأمان](/clawhub/security-audits).

لتقارير السوق، وتعليقات الإشراف، والقوائم المخفية، والحظر، وحالة الحساب،
راجع [الإشراف وسلامة الحساب](/clawhub/moderation).
