# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub من خلال GitHub Security Advisories لـ
`openclaw/clawhub`.

استخدم GitHub Security Advisories للثغرات في ClawHub نفسه. تتضمن بلاغات
ClawHub الجيدة عن الاستشارات الأمنية أخطاء في:

- موقع ClawHub أو API أو CLI
- نشر السجل، أو التنزيلات، أو التثبيتات، أو سلامة العناصر الأثرية
- المصادقة، أو التخويل، أو رموز API
- الفحص، أو الإشراف، أو معالجة البلاغات

لا تستخدم استشارات ClawHub للثغرات في الشفرة المصدرية الخاصة بمهارة أو
Plugin تابع لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من قائمة ClawHub.

## الإفصاح عن الثغرات

لأن ClawHub تطبيق سحابي مستضاف، لا يُفصح عن ثغرات خدمة ClawHub علنًا
افتراضيًا. يُفصح عنها علنًا عند وجود دليل على تأثير حقيقي في المستخدمين أو
عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الحقيقي في المستخدمين الاستغلال المؤكد، أو كشف بيانات
المستخدمين أو الأسرار، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد، أو تحديث البرامج
المحلية، أو اتخاذ إجراء وقائي آخر.

يُفصح علنًا عن الثغرات في البرامج التي يثبتها المستخدمون، مثل حزم ClawHub CLI،
أو الثنائيات، أو المكتبات، أو عناصر الإصدار الأثرية الأخرى التي يحتاج
المستخدمون إلى تحديثها محليًا.

## الصفحات ذات الصلة

لتسميات التدقيق وقت التثبيت، ومستويات المخاطر، والنتائج، والتفسير، راجع
[تدقيقات الأمان](/clawhub/security-audits).

لبلاغات السوق، وعمليات التعليق بسبب الإشراف، والقوائم المخفية، والحظر، وحالة
الحساب، راجع [الإشراف وسلامة الحساب](/clawhub/moderation).
