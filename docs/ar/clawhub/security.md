# الأمان + الإشراف

ClawHub مفتوح للنشر، لكن القوائم العامة ما زالت تمر عبر ضوابط الثقة،
والفحص، والإبلاغ، والإشراف. الهدف عملي: مساعدة المستخدمين على
فحص ما يثبتونه، ومنح الناشرين مسار استرداد للحالات الإيجابية الكاذبة،
وإبعاد الحزم المسيئة عن الاكتشاف العام.

راجع أيضًا [الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## ما يمكن للمستخدمين فحصه

قبل تثبيت skill أو plugin، تحقق من قائمة ClawHub الخاصة به بحثًا عن:

- إسناد المالك والمصدر
- أحدث إصدار وسجل التغييرات
- متغيرات البيئة أو الأذونات المطلوبة
- بيانات تعريف التوافق للـ plugins
- حالة الفحص أو الإشراف
- البلاغات، والتعليقات، والنجوم، والتنزيلات، وإشارات التثبيت حيثما تظهر

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالات الفحص

قد يعرض ClawHub نتائج الفحص أو الإشراف على الصفحات العامة والتشخيصات
المرئية للمالك.

تشمل النتائج الشائعة:

- `clean`: لم يتم العثور على مشكلة مانعة.
- `suspicious`: يحتاج الإصدار إلى الحذر أو المراجعة.
- `malicious`: يُعد الإصدار غير آمن.
- `pending`: لم تنتهِ عمليات التحقق بعد.
- `held`، أو `quarantined`، أو `revoked`، أو `hidden`: الإصدار غير متاح بالكامل
  على واجهات التثبيت العامة.

قد تختلف الصياغة الدقيقة حسب الواجهة، لكن المعنى العملي هو نفسه: إذا كان
الإصدار موقوفًا أو محظورًا، فيجب ألا يثبته المستخدمون حتى يحل المالك
المشكلة أو يعيده الإشراف.

## Skills

تنظر فحوصات Skills في حزمة skill المنشورة، وبياناتها التعريفية، والمتطلبات
المعلنة، والتعليمات المشبوهة.

يولي ClawHub اهتمامًا خاصًا لحالات عدم التطابق بين ما يعلنه skill وما يبدو
أنه يفعله. على سبيل المثال، يجب أن يعلن skill الذي يشير إلى مفتاح API مطلوب
عن ذلك المتطلب في `SKILL.md` حتى يتمكن المستخدمون من رؤيته قبل التثبيت.

تستند نتائج الفحص إلى الأثر. يُعامل سلوك المزود المتوقع، مثل بيانات اعتماد
API المعلنة، واستدعاءات OAuth إلى localhost، وتنظيف إلغاء التثبيت المحدد
النطاق، وترميز Basic Auth، أو تحميلات الملفات التي يختارها المستخدم إلى
المزود المعلن، على نحو مختلف عن تمرير بيانات الاعتماد المخفية، أو الوصول
الواسع إلى الملفات الخاصة، أو وجهات الشبكة غير ذات الصلة، أو إساءة استخدام
المتصفح خفية.

راجع [تنسيق Skill](/ar/clawhub/skill-format).

## Plugins

تتضمن إصدارات Plugin بيانات تعريف الحزمة، وإسناد المصدر، وحقول التوافق،
ومعلومات سلامة الأثر.

يتحقق OpenClaw من التوافق قبل تثبيت plugins المستضافة على ClawHub. قد تعرض
سجلات الحزم أيضًا بيانات تعريف البصمة حتى يتمكن OpenClaw من التحقق من
الأثرات المنزلة. يتضمن ClawScan بيانات تعريف env/config المعلنة في
`openclaw.environment` للحزمة عند مراجعة إصدارات plugin بحيث تُقارن متطلبات
وقت التشغيل المعلنة بالسلوك المرصود.

## البلاغات

يمكن للمستخدمين الذين سجلوا الدخول الإبلاغ عن Skills، والحزم، والتعليقات.

يجب أن تكون البلاغات محددة وقابلة للتنفيذ. قد يؤدي إساءة استخدام الإبلاغ
بحد ذاتها إلى إجراء على الحساب.

أمثلة على البلاغات:

- بيانات تعريف مضللة
- متطلبات بيانات اعتماد أو أذونات غير معلنة
- تعليمات تثبيت مشبوهة
- تعليقات احتيالية أو انتحال شخصية
- تسجيلات بسوء نية أو إساءة استخدام علامات تجارية
- محتوى ينتهك [الاستخدام المقبول](/ar/clawhub/acceptable-usage)

## ملاحظات ClawScan للناشرين

يمكن للناشرين تقديم ملاحظة ClawScan اختيارية عند نشر skill أو
plugin. تمنح هذه الملاحظة ClawScan سياقًا للسلوك الذي قد يبدو غير معتاد
بخلاف ذلك، مثل الوصول إلى الشبكة، أو الوصول إلى مضيف أصلي، أو بيانات
اعتماد خاصة بمزود.

## حالات التعليق الإشرافي

عندما يعلّم الفاحص الثابت skill مرفوعًا على أنه ضار، يوضع الناشر تلقائيًا
تحت تعليق إشرافي (`requiresModerationAt` مضبوط على المستخدم). يؤدي ذلك إلى
إخفاء كل Skills الخاصة بالناشر، ويجعل عمليات النشر المستقبلية تبدأ مخفية،
وينشئ إدخال سجل تدقيق `user.moderation.auto`.

تُحفظ النتائج الثابتة المشبوهة كدليل ملف/سطر للمشرفين، لكنها لا تخفي
المحتوى ولا تقرر حكم الفحص العام بمفردها. تبقى التحميلات الجديدة في حالة
المراجعة/الانتظار حتى تستقر مراجعة LLM. لا يحظر الفحص الثابت فورًا إلا
للتواقيع الضارة. تبقى نتائج محركات VirusTotal أدلة أمنية مرئية، لكن أحكام
VirusTotal Code Insight/Palm استشارية ولا تخفي Skills بمفردها. تحتفظ
مراجعات ClawScan LLM بالملاحظات المتوافقة مع الغرض كإرشاد. تبقى نتائج
المراجعة المتوسطة مرئية على الأثر، بينما يُحجز مرشح الاشتباه لمخاوف LLM
عالية التأثير، أو النتائج الضارة، أو اكتشافات محركات AV المؤيدة.

يمكن للمسؤولين رفع تعليق إيجابي كاذب:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

يمسح هذا `requiresModerationAt` و`requiresModerationReason`، ويستعيد
Skills المخفية بسبب التعليق على مستوى المستخدم، ويكتب إدخال سجل تدقيق
`user.moderation.lift`. تبقى Skills المخفية لأسباب أخرى، أو التي يظل
فحصها الثابت الخاص بها ضارًا، مخفية.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub صلاحية النشر. وقد يؤدي الإساءة
الشديدة إلى حظر الحساب، أو إبطال الرموز، أو إخفاء المحتوى، أو إزالة
القوائم.

لا يمكن للحسابات المحذوفة، أو المحظورة، أو المعطلة استخدام رموز ClawHub
API. إذا بدأ فشل مصادقة CLI بعد إجراء على الحساب، فسجل الدخول إلى واجهة
الويب لمراجعة حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI
محظورًا، فاتصل بـ security@openclaw.ai لمراجعة الاسترداد.

## إرشادات الناشرين

لتقليل الإيجابيات الكاذبة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء، والملخصات، والوسوم، وسجلات التغييرات
- أعلن متغيرات البيئة والأذونات المطلوبة
- أضف ملاحظة ClawScan للناشر عندما يحتوي الإصدار على سلوك غير معتاد لكنه مقصود
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنًا
- استخدم التشغيلات الجافة قبل نشر plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الحزمة
