# الأمان + الإشراف

ClawHub مفتوح للنشر، لكن القوائم العامة لا تزال تمر عبر ضوابط الثقة،
والفحص، والإبلاغ، والإشراف. الهدف عملي: مساعدة المستخدمين على
فحص ما يثبتونه، ومنح الناشرين مسار استرداد للحالات الإيجابية الكاذبة،
وإبعاد الحزم المسيئة عن الاكتشاف العام.

انظر أيضًا [الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## ما يمكن للمستخدمين فحصه

قبل تثبيت مهارة أو Plugin، تحقق من قائمة ClawHub الخاصة بها للاطلاع على:

- المالك ونسبة المصدر
- أحدث إصدار وسجل التغييرات
- متغيرات البيئة أو الأذونات المطلوبة
- بيانات التوافق الوصفية للـ plugins
- حالة الفحص أو الإشراف
- البلاغات، والتعليقات، والنجوم، والتنزيلات، وإشارات التثبيت حيث تظهر

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالات الفحص

قد يعرض ClawHub نتائج الفحص أو الإشراف على الصفحات العامة وفي
التشخيصات المرئية للمالك.

تشمل النتائج الشائعة:

- `clean`: لم يتم العثور على مشكلة حاجبة.
- `suspicious`: يتطلب الإصدار الحذر أو المراجعة.
- `malicious`: يُعد الإصدار غير آمن.
- `pending`: لم تكتمل الفحوصات بعد.
- `held` أو `quarantined` أو `revoked` أو `hidden`: الإصدار غير متاح بالكامل
  على واجهات التثبيت العامة.

قد تختلف الصياغة الدقيقة حسب الواجهة، لكن المعنى العملي واحد: إذا كان
الإصدار محتجزًا أو محظورًا، فيجب على المستخدمين عدم تثبيته حتى يحل المالك
المشكلة أو يعيده الإشراف.

## Skills

تنظر فحوصات المهارات في حزمة المهارة المنشورة، والبيانات الوصفية، والمتطلبات
المعلنة، والتعليمات المشبوهة.

يولي ClawHub اهتمامًا خاصًا لعدم التطابق بين ما تعلنه المهارة وما يبدو أنها
تفعله. على سبيل المثال، يجب على المهارة التي تشير إلى مفتاح API مطلوب أن
تعلن هذا المتطلب في `SKILL.md` حتى يتمكن المستخدمون من رؤيته قبل التثبيت.

تعتمد نتائج الفحص على الأثر. يُعامل سلوك المزوّد المتوقع، مثل بيانات اعتماد
API المعلنة، أو استدعاءات OAuth المرتدة إلى localhost، أو تنظيف إلغاء التثبيت
المحدد النطاق، أو ترميز Basic Auth، أو عمليات رفع الملفات التي يختارها
المستخدم إلى المزوّد المعلن، بشكل مختلف عن تمرير بيانات الاعتماد المخفية،
أو الوصول الواسع إلى الملفات الخاصة، أو وجهات الشبكة غير المرتبطة، أو إساءة
استخدام المتصفح خلسة.

انظر [تنسيق المهارة](/ar/clawhub/skill-format).

## Plugins

تتضمن إصدارات Plugin بيانات وصفية للحزمة، ونسبة المصدر، وحقول التوافق،
ومعلومات سلامة الأثر.

يتحقق OpenClaw من التوافق قبل تثبيت plugins المستضافة على ClawHub. قد تعرض
سجلات الحزم أيضًا بيانات وصفية للملخص الرقمي حتى يتمكن OpenClaw من التحقق من
الأثر الذي تم تنزيله. يتضمن ClawScan بيانات env/config الوصفية المعلنة في
`openclaw.environment` الخاصة بالحزمة عند مراجعة إصدارات Plugin، بحيث تتم
مقارنة متطلبات التشغيل المعلنة بالسلوك المرصود.

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن المهارات، والحزم، والتعليقات.

يجب أن تكون البلاغات محددة وقابلة للتنفيذ. إساءة استخدام الإبلاغ قد تؤدي
بحد ذاتها إلى إجراء على الحساب.

أمثلة على البلاغات:

- بيانات وصفية مضللة
- متطلبات غير معلنة لبيانات الاعتماد أو الأذونات
- تعليمات تثبيت مشبوهة
- تعليقات احتيالية أو انتحال شخصية
- تسجيلات بسوء نية أو إساءة استخدام للعلامات التجارية
- محتوى ينتهك [الاستخدام المقبول](/ar/clawhub/acceptable-usage)

## ملاحظات ClawScan للناشرين

يمكن للناشرين تقديم ملاحظة اختيارية لـ ClawScan عند نشر مهارة أو Plugin. تمنح
هذه الملاحظة ClawScan سياقًا لسلوك قد يبدو غير معتاد بخلاف ذلك، مثل الوصول
إلى الشبكة، أو الوصول إلى المضيف الأصلي، أو بيانات اعتماد خاصة بالمزوّد.

## احتجازات الإشراف

عندما يضع الفاحص الثابت علامة على مهارة مرفوعة باعتبارها خبيثة، يتم وضع
الناشر تلقائيًا تحت احتجاز إشراف (`requiresModerationAt` مضبوط على المستخدم).
يؤدي ذلك إلى إخفاء جميع مهارات الناشر، وجعل عمليات النشر المستقبلية تبدأ
مخفية، وإنشاء إدخال سجل تدقيق `user.moderation.auto`.

تُحتفظ بالنتائج الثابتة المشبوهة كدليل ملف/سطر للمشرفين، لكنها لا تخفي
المحتوى ولا تحدد حكم الفحص العام بمفردها. تبقى الرفوعات الجديدة في حالة
مراجعة/انتظار حتى تستقر مراجعة LLM. لا يحظر الفحص الثابت فورًا إلا عند وجود
تواقيع خبيثة. تبقى إصابات محركات VirusTotal أدلة أمنية مرئية، لكن أحكام
VirusTotal Code Insight/Palm استشارية ولا تخفي المهارات بمفردها. تحتفظ
مراجعات ClawScan LLM بالملاحظات المتوافقة مع الغرض كإرشاد. تبقى نتائج
المراجعة المتوسطة مرئية على الأثر، بينما يُحجز مرشح الاشتباه لمخاوف LLM
عالية التأثير، أو النتائج الخبيثة، أو اكتشافات محركات مكافحة الفيروسات
المؤيدة بأدلة.

يمكن للمسؤولين رفع احتجاز إيجابي كاذب:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

يمسح هذا `requiresModerationAt` و`requiresModerationReason`، ويستعيد المهارات
المخفية بسبب الاحتجاز على مستوى المستخدم، ويكتب إدخال سجل تدقيق
`user.moderation.lift`. تبقى المهارات المخفية لأسباب أخرى، أو التي لا يزال
فحصها الثابت خبيثًا، مخفية.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub حق الوصول إلى النشر. يمكن أن تؤدي
الإساءات الشديدة إلى حظر الحسابات، أو إبطال الرموز، أو إخفاء المحتوى، أو
إزالة القوائم.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـ
ClawHub. إذا بدأ فشل مصادقة CLI بعد إجراء على الحساب، فسجّل الدخول إلى واجهة
الويب لمراجعة حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI
محظورًا، فاتصل بـ security@openclaw.ai لمراجعة الاسترداد.

## إرشادات الناشرين

لتقليل الحالات الإيجابية الكاذبة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء، والملخصات، والوسوم، وسجلات التغييرات
- أعلن متغيرات البيئة والأذونات المطلوبة
- أضف ملاحظة ClawScan للناشر عندما يتضمن الإصدار سلوكًا غير معتاد لكنه مقصود
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنًا
- استخدم التشغيل التجريبي الجاف قبل نشر plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الحزمة
