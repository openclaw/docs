# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub من خلال GitHub Security Advisories لـ
`openclaw/clawhub`.

استخدم GitHub Security Advisories للثغرات في ClawHub نفسه. تتضمن بلاغات
ClawHub الجيدة عن الاستشارات أخطاء في:

- موقع ClawHub أو API أو CLI
- نشر السجل أو التنزيلات أو التثبيتات أو سلامة الأثر البرمجي
- المصادقة أو التفويض أو رموز API
- الفحص أو الإشراف أو معالجة البلاغات

لا تستخدم استشارات ClawHub للثغرات في الشيفرة المصدرية الخاصة بمهارة خارجية أو
Plugin خارجي. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر المرتبط من إدراج
ClawHub.

## الإفصاح عن الثغرات

لأن ClawHub تطبيق سحابي مستضاف، لا يتم الإفصاح علنًا عن ثغرات خدمة ClawHub
افتراضيًا. يتم الإفصاح عنها علنًا عندما توجد أدلة على تأثير حقيقي في المستخدمين
أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الحقيقي في المستخدمين الاستغلال المؤكد، أو كشف بيانات
المستخدمين أو الأسرار، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة، أو
أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد، أو تحديث البرامج المحلية، أو
اتخاذ إجراء وقائي آخر.

يتم الإفصاح علنًا عن الثغرات في البرامج التي يثبتها المستخدم، مثل حزم ClawHub
CLI أو الثنائيات أو المكتبات أو غيرها من آثار الإصدار التي يحتاج المستخدمون إلى
تحديثها محليًا.
