# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub من خلال إرشادات GitHub الأمنية الخاصة بـ
`openclaw/clawhub`.

استخدم إرشادات GitHub الأمنية للثغرات الأمنية في ClawHub نفسه. تتضمن تقارير
إرشادات ClawHub الأمنية الجيدة أخطاء في:

- موقع ClawHub أو API أو CLI
- النشر في السجل، أو التنزيلات، أو التثبيتات، أو سلامة الأرتيفاكتات
- المصادقة، أو التفويض، أو رموز API المميزة
- الفحص، أو الإشراف، أو معالجة البلاغات

لا تستخدم إرشادات ClawHub الأمنية للثغرات الأمنية في الكود المصدري الخاص بـ skill
أو Plugin تابع لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من إدراج ClawHub.

## الإفصاح عن الثغرات الأمنية

بما أن ClawHub تطبيق سحابي مستضاف، فلا يتم الإفصاح علنًا عن ثغرات خدمة ClawHub
افتراضيًا. يتم الإفصاح عنها علنًا عندما توجد أدلة على تأثير حقيقي في المستخدمين
أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الحقيقي في المستخدمين الاستغلال المؤكد، أو كشف بيانات
المستخدمين أو أسرارهم، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد، أو تحديث البرامج المحلية،
أو اتخاذ إجراء وقائي آخر.

يتم الإفصاح علنًا عن الثغرات الأمنية في البرامج التي يثبتها المستخدمون، مثل
حزم ClawHub CLI، أو الثنائيات، أو المكتبات، أو أرتيفاكتات الإصدار الأخرى التي
يحتاج المستخدمون إلى تحديثها محليًا.

## صفحات ذات صلة

للاطلاع على تسميات التدقيق وقت التثبيت، ومستويات المخاطر، والنتائج، والتفسير، راجع
