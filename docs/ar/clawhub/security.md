# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub عبر GitHub Security Advisories لمستودع
`openclaw/clawhub`.

استخدم GitHub Security Advisories للثغرات في ClawHub نفسه. تتضمن تقارير
استشارات ClawHub الجيدة أخطاء في:

- موقع ClawHub أو API أو CLI
- نشر السجل أو التنزيلات أو عمليات التثبيت أو سلامة القطع الأثرية
- المصادقة أو التفويض أو رموز API
- الفحص أو الإشراف أو معالجة البلاغات

لا تستخدم استشارات ClawHub للثغرات الموجودة في الشيفرة المصدرية الخاصة بمهارة أو
Plugin من طرف ثالث. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر المرتبط من قائمة
ClawHub.

## الإفصاح عن الثغرات

نظرًا لأن ClawHub تطبيق سحابي مستضاف، لا يُفصح عن ثغرات خدمة ClawHub علنًا
افتراضيًا. يُفصح عنها علنًا عندما توجد أدلة على تأثير حقيقي في المستخدمين أو
عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الحقيقي في المستخدمين الاستغلال المؤكد، أو كشف بيانات
المستخدمين أو الأسرار، أو وصول محتوى خبيث إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد، أو تحديث البرامج المحلية،
أو اتخاذ إجراء وقائي آخر.

يُفصح علنًا عن الثغرات في البرامج التي يثبتها المستخدمون، مثل حزم ClawHub CLI أو
الثنائيات أو المكتبات أو غيرها من قطع الإصدار الأثرية التي يحتاج المستخدمون إلى
تحديثها محليًا.
