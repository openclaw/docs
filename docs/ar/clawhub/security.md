# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub عبر إرشادات GitHub الأمنية لـ
`openclaw/clawhub`.

استخدم إرشادات GitHub الأمنية للثغرات في ClawHub نفسه. تتضمن بلاغات
ClawHub الجيدة عن الإرشادات أخطاءً في:

- موقع ClawHub أو API أو CLI
- نشر السجل أو التنزيلات أو التثبيتات أو سلامة المصنوعات
- المصادقة أو التفويض أو رموز API
- الفحص أو الإشراف أو التعامل مع البلاغات

لا تستخدم إرشادات ClawHub للثغرات الموجودة في الشيفرة المصدرية الخاصة بمهارة
أو Plugin تابع لجهة خارجية. أبلغ عنها مباشرةً إلى الناشر أو مستودع المصدر
المرتبط من إدراج ClawHub.

## الإفصاح عن الثغرات

نظرًا لأن ClawHub تطبيق سحابي مستضاف، لا يتم الإفصاح علنًا عن ثغرات خدمة
ClawHub افتراضيًا. يتم الإفصاح عنها علنًا عندما يوجد دليل على تأثير فعلي في
المستخدمين أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الفعلي في المستخدمين الاستغلال المؤكد، أو كشف بيانات
المستخدمين أو أسرارهم، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد، أو تحديث البرامج المحلية،
أو اتخاذ إجراء وقائي آخر.

يتم الإفصاح علنًا عن الثغرات في البرامج المثبتة من قبل المستخدمين، مثل حزم
ClawHub CLI أو الملفات الثنائية أو المكتبات أو غيرها من مصنوعات الإصدار التي
يحتاج المستخدمون إلى تحديثها محليًا.

لملصقات التدقيق وقت التثبيت، ومستويات المخاطر، والنتائج، والتفسير، راجع
