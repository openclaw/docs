# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub عبر GitHub Security Advisories لـ
`openclaw/clawhub`.

استخدم GitHub Security Advisories للثغرات في ClawHub نفسه. تتضمن تقارير
استشارات ClawHub الجيدة أخطاء في:

- موقع ClawHub أو API أو CLI
- نشر السجل، أو التنزيلات، أو التثبيتات، أو سلامة المصنوعات
- المصادقة، أو التخويل، أو رموز API
- الفحص، أو الإشراف، أو معالجة التقارير

لا تستخدم استشارات ClawHub للثغرات في الشيفرة المصدرية الخاصة بـ skill أو
plugin تابع لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من قائمة ClawHub.

## الإفصاح عن الثغرات

نظرًا لأن ClawHub تطبيق سحابي مستضاف، لا يُفصح افتراضيًا عن ثغرات خدمة
ClawHub علنًا. يُفصح عنها علنًا عندما يوجد دليل على أثر حقيقي على المستخدمين
أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة الأثر الحقيقي على المستخدمين الاستغلال المؤكد، أو كشف بيانات
المستخدمين أو الأسرار، أو وصول محتوى خبيث إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد، أو تحديث البرامج
المحلية، أو اتخاذ إجراء وقائي آخر.

يُفصح علنًا عن الثغرات في البرامج التي يثبتها المستخدمون، مثل حزم ClawHub CLI
أو الثنائيات أو المكتبات أو مصنوعات الإصدار الأخرى التي يحتاج المستخدمون إلى
تحديثها محليًا.

## الصفحات ذات الصلة

لتسميات التدقيق وقت التثبيت، ومستويات المخاطر، والنتائج، والتفسير، راجع
[عمليات تدقيق الأمان](/clawhub/security-audits).

لتقارير السوق، وتعليقات الإشراف، والقوائم المخفية، والحظر، وحالة الحساب،
راجع [الإشراف وسلامة الحساب](/clawhub/moderation).
