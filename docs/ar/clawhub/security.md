# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub عبر إرشادات GitHub الأمنية الخاصة بـ
`openclaw/clawhub`.

استخدم إرشادات GitHub الأمنية للثغرات الموجودة في ClawHub نفسه. تتضمن
تقارير إرشادات ClawHub الجيدة أخطاء في:

- موقع ClawHub أو API أو CLI
- النشر في السجل أو التنزيلات أو عمليات التثبيت أو سلامة القطع البرمجية
- المصادقة أو التخويل أو رموز API
- الفحص أو الإشراف أو معالجة البلاغات

لا تستخدم إرشادات ClawHub للثغرات الموجودة في الشيفرة المصدرية الخاصة بمهارة
أو Plugin تابع لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من صفحة ClawHub.

## الإفصاح عن الثغرات

نظرًا لأن ClawHub تطبيق سحابي مستضاف، لا تُفصح ثغرات خدمة ClawHub علنًا
افتراضيًا. ويُفصح عنها علنًا عندما توجد أدلة على تأثير حقيقي في المستخدمين
أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الحقيقي في المستخدمين الاستغلال المؤكد، أو كشف بيانات
المستخدمين أو أسرارهم، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد أو تحديث البرامج
المحلية أو اتخاذ إجراء وقائي آخر.

يُفصح علنًا عن الثغرات في البرامج التي يثبتها المستخدمون، مثل حزم ClawHub
CLI أو الثنائيات أو المكتبات أو غيرها من عناصر الإصدار التي يحتاج المستخدمون
إلى تحديثها محليًا.
