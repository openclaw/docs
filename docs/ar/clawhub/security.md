# الأمان + الإشراف

ClawHub مفتوح للنشر، لكن القوائم العامة لا تزال تمر عبر ضوابط الثقة،
والفحص، والإبلاغ، والإشراف. الهدف عملي: مساعدة المستخدمين على
فحص ما يثبتونه، ومنح الناشرين مسار استرداد للحالات الإيجابية الكاذبة،
وإبعاد الحزم المسيئة عن الاكتشاف العام.

راجع أيضًا [الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## ما يمكن للمستخدمين فحصه

قبل تثبيت Skill أو Plugin، تحقق من قائمة ClawHub الخاصة به لمعرفة:

- المالك ونسبة المصدر
- أحدث إصدار وسجل التغييرات
- متغيرات البيئة أو الأذونات المطلوبة
- بيانات التوافق الوصفية للـ plugins
- حالة الفحص أو الإشراف
- البلاغات، والتعليقات، والنجوم، والتنزيلات، وإشارات التثبيت حيث تظهر

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالات الفحص

قد يعرض ClawHub نتائج الفحص أو الإشراف على الصفحات العامة وبيانات
التشخيص المرئية للمالك.

تشمل النتائج الشائعة:

- `clean`: لم يتم العثور على مشكلة حاظرة.
- `suspicious`: يحتاج الإصدار إلى الحذر أو المراجعة.
- `malicious`: يُعد الإصدار غير آمن.
- `pending`: لم تنتهِ الفحوصات بعد.
- `held` أو `quarantined` أو `revoked` أو `hidden`: الإصدار غير متاح بالكامل
  على أسطح التثبيت العامة.

قد تختلف الصياغة الدقيقة حسب السطح، لكن المعنى العملي واحد: إذا كان
الإصدار محتجزًا أو محظورًا، فيجب ألا يثبته المستخدمون حتى يحل المالك
المشكلة أو يعيده الإشراف.

## Skills

تنظر فحوصات Skill في حزمة Skill المنشورة، والبيانات الوصفية، والمتطلبات
المعلنة، والتعليمات المشبوهة.

يولي ClawHub اهتمامًا خاصًا لحالات عدم التطابق بين ما يعلنه Skill وما
يبدو أنه يفعله. على سبيل المثال، يجب أن يعلن Skill الذي يشير إلى مفتاح API
مطلوب ذلك المتطلب في `SKILL.md` حتى يتمكن المستخدمون من رؤيته قبل
التثبيت.

تستند نتائج الفحص إلى الأثر. يُعامل سلوك الموفر المتوقع، مثل بيانات اعتماد
API المعلنة، أو استدعاءات OAuth الراجعة إلى localhost، أو تنظيف إزالة
التثبيت المحدد النطاق، أو ترميز Basic Auth، أو تحميلات الملفات التي
يختارها المستخدم إلى الموفر المعلن، بشكل مختلف عن تمرير بيانات الاعتماد
المخفي، أو الوصول الواسع إلى الملفات الخاصة، أو وجهات الشبكة غير ذات
الصلة، أو إساءة استخدام المتصفح خفية.

راجع [تنسيق Skill](/ar/clawhub/skill-format).

## Plugins

تتضمن إصدارات Plugin بيانات الحزمة الوصفية، ونسبة المصدر، وحقول
التوافق، ومعلومات سلامة الأثر.

يتحقق OpenClaw من التوافق قبل تثبيت plugins المستضافة على ClawHub. قد
تعرض سجلات الحزم أيضًا بيانات وصفية للخلاصة حتى يتمكن OpenClaw من
التحقق من الآثار المنزلة. يتضمن ClawScan بيانات env/config الوصفية
المعلنة في `openclaw.environment` الخاصة بالحزمة عند مراجعة إصدارات
Plugin بحيث تتم مقارنة متطلبات وقت التشغيل المعلنة بالسلوك المرصود.

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن Skills، والحزم، والتعليقات.

يجب أن تكون البلاغات محددة وقابلة للتنفيذ. ويمكن أن يؤدي إساءة استخدام
الإبلاغ بحد ذاتها إلى إجراء بحق الحساب.

أمثلة على البلاغات:

- بيانات وصفية مضللة
- متطلبات بيانات اعتماد أو أذونات غير معلنة
- تعليمات تثبيت مشبوهة
- تعليقات احتيالية أو انتحال شخصية
- تسجيلات بسوء نية أو إساءة استخدام علامة تجارية
- محتوى ينتهك [الاستخدام المقبول](/ar/clawhub/acceptable-usage)

## ملاحظات ClawScan للناشرين

يمكن للناشرين تقديم ملاحظة ClawScan اختيارية عند نشر Skill أو
Plugin. تمنح هذه الملاحظة ClawScan سياقًا للسلوك الذي قد يبدو غير معتاد
بخلاف ذلك، مثل الوصول إلى الشبكة، أو الوصول إلى مضيف أصلي، أو بيانات
اعتماد خاصة بموفر.

## عمليات الاحتجاز للإشراف

عندما يضع الفاحص الثابت علامة على Skill مرفوع باعتباره ضارًا، يوضع
الناشر تلقائيًا تحت احتجاز إشرافي (يتم تعيين `requiresModerationAt` على
المستخدم). يؤدي هذا إلى إخفاء جميع Skills الخاصة بالناشر، ويجعل عمليات
النشر المستقبلية تبدأ مخفية، وينشئ إدخال سجل تدقيق `user.moderation.auto`.

تُحتفظ بالنتائج الثابتة المشبوهة كأدلة ملف/سطر للمشرفين، لكنها لا تخفي
المحتوى أو تحدد حكم الفحص العام وحدها. تبقى الرفعات الجديدة في حالة
مراجعة/انتظار إلى أن تستقر مراجعة LLM. لا يحظر الفحص الثابت فورًا إلا
للتواقيع الضارة. تظل نتائج محركات VirusTotal أدلة أمنية مرئية، لكن
أحكام VirusTotal Code Insight/Palm استشارية ولا تخفي Skills وحدها.
تحتفظ مراجعات ClawScan LLM بالملاحظات المتوافقة مع الغرض كإرشاد.
تظل نتائج المراجعة المتوسطة مرئية على الأثر، بينما يُحجز مرشح الاشتباه
لمخاوف LLM عالية التأثير، أو النتائج الضارة، أو اكتشافات محركات AV
المؤيدة بأدلة.

يمكن للمسؤولين رفع احتجاز إيجابي كاذب:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

يمسح هذا `requiresModerationAt` و`requiresModerationReason`، ويستعيد
Skills التي أخفاها الاحتجاز على مستوى المستخدم، ويكتب إدخال سجل تدقيق
`user.moderation.lift`. تبقى Skills المخفية لأسباب أخرى، أو التي لا يزال
فحصها الثابت الخاص بها ضارًا، مخفية.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub صلاحية النشر. ويمكن أن تؤدي
الإساءات الشديدة إلى حظر الحساب، أو إلغاء الرموز، أو إخفاء المحتوى، أو
إزالة القوائم.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز ClawHub
API. إذا بدأت مصادقة CLI بالفشل بعد إجراء على الحساب، فسجّل الدخول إلى
واجهة الويب لمراجعة حالة الحساب. إذا كان تسجيل الدخول أو وصول CLI العادي
محظورًا، فاتصل بـ security@openclaw.ai لمراجعة الاسترداد.

## إرشادات الناشرين

لتقليل الحالات الإيجابية الكاذبة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء، والملخصات، والوسوم، وسجلات التغييرات
- أعلن متغيرات البيئة والأذونات المطلوبة
- أضف ملاحظة ClawScan للناشر عندما يكون للإصدار سلوك غير معتاد لكنه مقصود
- تجنب أوامر التثبيت المعماة
- اربط بالمصدر عندما يكون ذلك ممكنًا
- استخدم التشغيلات التجريبية قبل نشر plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الحزمة
