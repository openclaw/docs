# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub عبر إرشادات GitHub الأمنية الخاصة بـ
`openclaw/clawhub`.

استخدم إرشادات GitHub الأمنية للثغرات في ClawHub نفسه. تتضمن تقارير
إرشادات ClawHub الجيدة أخطاء في:

- موقع ClawHub أو API أو CLI
- نشر السجل أو التنزيلات أو التثبيتات أو سلامة العناصر الأثرية
- المصادقة أو التفويض أو رموز API
- الفحص أو الإشراف أو معالجة التقارير

لا تستخدم إرشادات ClawHub للثغرات في الشفرة المصدرية الخاصة بمهارة تابعة
لجهة خارجية أو Plugin تابع لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو
مستودع المصدر المرتبط من قائمة ClawHub.

## الإفصاح عن الثغرات

نظرًا لأن ClawHub تطبيق سحابي مستضاف، لا يُفصح عن ثغرات خدمة ClawHub
علنًا افتراضيًا. يُفصح عنها علنًا عند وجود دليل على تأثير فعلي في المستخدمين
أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الفعلي في المستخدمين الاستغلال المؤكد، أو كشف بيانات
المستخدمين أو أسرارهم، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في
المنصة، أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد أو تحديث
البرامج المحلية أو اتخاذ إجراء وقائي آخر.

يُفصح علنًا عن الثغرات في البرامج التي يثبتها المستخدمون، مثل حزم ClawHub CLI
أو الملفات الثنائية أو المكتبات أو عناصر الإصدار الأثرية الأخرى التي يحتاج
المستخدمون إلى تحديثها محليًا.

## الصفحات ذات الصلة

للاطلاع على تسميات التدقيق وقت التثبيت، ومستويات المخاطر، والنتائج، والتفسير، راجع
[تدقيقات الأمان](/clawhub/security-audits).

لتقارير السوق، وعمليات تعليق الإشراف، والقوائم المخفية، والحظر، وحالة الحساب،
راجع [الإشراف وأمان الحساب](/clawhub/moderation).
