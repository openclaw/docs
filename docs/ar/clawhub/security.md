# الأمان + الإشراف

ClawHub مفتوح للنشر، لكن القوائم العامة لا تزال تمر عبر ضوابط الثقة
والفحص والإبلاغ والإشراف. الهدف عملي: مساعدة المستخدمين على
فحص ما يثبتونه، ومنح الناشرين مسار استرداد للحالات الإيجابية الكاذبة،
وإبقاء الحزم المسيئة خارج الاكتشاف العام.

راجع أيضًا [الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## ما يمكن للمستخدمين فحصه

قبل تثبيت Skill أو Plugin، تحقق من قائمة ClawHub الخاصة به لمعرفة:

- إسناد المالك والمصدر
- أحدث إصدار وسجل التغييرات
- متغيرات البيئة أو الأذونات المطلوبة
- بيانات تعريف التوافق للـ Plugins
- حالة الفحص أو الإشراف
- البلاغات والتعليقات والنجوم والتنزيلات وإشارات التثبيت حيث تظهر

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالات الفحص

قد يعرض ClawHub نتائج الفحص أو الإشراف على الصفحات العامة والتشخيصات
المرئية للمالك.

تشمل النتائج الشائعة:

- `clean`: لم تُعثر أي مشكلة حاظرة.
- `suspicious`: يحتاج الإصدار إلى الحذر أو المراجعة.
- `malicious`: يُعد الإصدار غير آمن.
- `pending`: لم تنتهِ الفحوصات بعد.
- `held` أو `quarantined` أو `revoked` أو `hidden`: الإصدار غير متاح بالكامل
  على أسطح التثبيت العامة.

قد تختلف الصياغة الدقيقة بحسب السطح، لكن المعنى العملي واحد: إذا كان
الإصدار محتجزًا أو محظورًا، فينبغي ألا يثبته المستخدمون حتى يحل المالك
المشكلة أو يعيده الإشراف.

## Skills

تنظر فحوصات Skill إلى حزمة Skill المنشورة، وبياناتها التعريفية، والمتطلبات
المعلنة، والتعليمات المريبة.

يولي ClawHub اهتمامًا خاصًا لحالات عدم التطابق بين ما تعلنه Skill وما يبدو
أنها تفعله. على سبيل المثال، ينبغي لـ Skill تشير إلى مفتاح API مطلوب أن تعلن
هذا المتطلب في `SKILL.md` حتى يتمكن المستخدمون من رؤيته قبل التثبيت.

تستند نتائج الفحص إلى الأثر. يُعامل سلوك المزوّد المتوقع، مثل بيانات اعتماد
API المعلنة، أو استدعاءات OAuth إلى المضيف المحلي، أو تنظيف إلغاء التثبيت
محدد النطاق، أو ترميز Basic Auth، أو تحميلات الملفات التي يختارها المستخدم
إلى المزوّد المعلن، بطريقة مختلفة عن تمرير بيانات الاعتماد المخفي، أو الوصول
الواسع إلى الملفات الخاصة، أو وجهات الشبكة غير ذات الصلة، أو إساءة استخدام
المتصفح خفية.

راجع [تنسيق Skill](/ar/clawhub/skill-format).

## Plugins

تتضمن إصدارات Plugin بيانات تعريف الحزمة، وإسناد المصدر، وحقول التوافق،
ومعلومات سلامة الأثر.

يتحقق OpenClaw من التوافق قبل تثبيت Plugins المستضافة على ClawHub. قد تعرض
سجلات الحزم أيضًا بيانات تعريف الملخص حتى يتمكن OpenClaw من التحقق من
الأثرات التي تم تنزيلها. يتضمن ClawScan بيانات تعريف env/config المعلنة في
الحزمة `openclaw.environment` عند مراجعة إصدارات Plugin بحيث تُقارن متطلبات
وقت التشغيل المعلنة بالسلوك المرصود.

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن Skills والحزم والتعليقات.

ينبغي أن تكون البلاغات محددة وقابلة للتنفيذ. وقد يؤدي إساءة استخدام الإبلاغ
بحد ذاتها إلى إجراء على الحساب.

أمثلة البلاغات:

- بيانات تعريف مضللة
- متطلبات غير معلنة لبيانات الاعتماد أو الأذونات
- تعليمات تثبيت مريبة
- تعليقات احتيالية أو انتحال
- تسجيلات بسوء نية أو إساءة استخدام للعلامات التجارية
- محتوى ينتهك [الاستخدام المقبول](/ar/clawhub/acceptable-usage)

## ملاحظات ClawScan للناشرين

يمكن للناشرين تقديم ملاحظة ClawScan اختيارية عند نشر Skill أو Plugin. تمنح هذه
الملاحظة ClawScan سياقًا لسلوك قد يبدو غير معتاد بخلاف ذلك، مثل الوصول إلى
الشبكة، أو الوصول إلى المضيف الأصلي، أو بيانات اعتماد خاصة بمزوّد.

## احتجازات الإشراف

عندما يضع الفاحص الثابت علامة على Skill مرفوعة باعتبارها خبيثة، يُوضع الناشر
تلقائيًا تحت احتجاز إشراف (`requiresModerationAt` مضبوط على المستخدم). يؤدي
ذلك إلى إخفاء جميع Skills الخاصة بالناشر، ويتسبب في بدء عمليات النشر
المستقبلية مخفية، وينشئ إدخال سجل تدقيق `user.moderation.auto`.

تُحتفظ بالنتائج الثابتة المريبة كأدلة ملف/سطر للمشرفين، لكنها لا تخفي
المحتوى أو تقرر حكم الفحص العام بمفردها. تبقى الرفعات الجديدة في حالة
مراجعة/انتظار حتى تستقر مراجعة LLM. لا يحظر الفحص الثابت فورًا إلا للتواقيع
الخبيثة. تبقى إصابات محركات VirusTotal مرئية كدليل أمني، لكن أحكام
VirusTotal Code Insight/Palm استشارية ولا تخفي Skills بمفردها. تحتفظ مراجعات
ClawScan LLM بالملاحظات المتوافقة مع الغرض كإرشاد. تبقى نتائج المراجعة
المتوسطة مرئية على الأثر، بينما يُحجز مرشح الاشتباه للمخاوف عالية الأثر من
LLM، أو النتائج الخبيثة، أو اكتشافات محركات AV المؤكدة.

يمكن للمشرفين رفع احتجاز إيجابي كاذب:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

يمسح هذا `requiresModerationAt` و`requiresModerationReason`، ويستعيد
Skills المخفية بسبب الاحتجاز على مستوى المستخدم، ويكتب إدخال سجل تدقيق
`user.moderation.lift`. تبقى Skills المخفية لأسباب أخرى، أو التي لا يزال
فحصها الثابت نفسه خبيثًا، مخفية.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub صلاحية النشر. ويمكن أن تؤدي
الإساءة الشديدة إلى حظر الحسابات، أو إبطال الرموز، أو إخفاء المحتوى، أو إزالة
القوائم.

لا تستطيع الحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة
بـ ClawHub. إذا بدأ توثيق CLI بالفشل بعد إجراء على الحساب، فسجّل الدخول إلى
واجهة الويب لمراجعة حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر
CLI محظورًا، فاتصل بـ security@openclaw.ai لمراجعة الاسترداد.

## إرشادات الناشرين

لتقليل الإيجابيات الكاذبة وتحسين ثقة المستخدم:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغييرات
- أعلن متغيرات البيئة والأذونات المطلوبة
- أضف ملاحظة ClawScan للناشر عندما يكون للإصدار سلوك غير معتاد لكنه مقصود
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنًا
- استخدم التشغيلات التجريبية قبل نشر Plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الحزمة
