# الأمان + الإشراف

ClawHub مفتوح للنشر، لكن القوائم العامة لا تزال تمر عبر ضوابط الثقة،
والفحص، والإبلاغ، والإشراف. الهدف عملي: مساعدة المستخدمين على
فحص ما يثبتونه، ومنح الناشرين مسار استرداد عند الإيجابيات الكاذبة،
وإبقاء الحزم المسيئة خارج الاكتشاف العام.

راجع أيضًا [الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## ما يمكن للمستخدمين فحصه

قبل تثبيت Skill أو Plugin، تحقق من قائمة ClawHub الخاصة به لمعرفة:

- إسناد المالك والمصدر
- أحدث إصدار وسجل التغييرات
- متغيرات البيئة أو الأذونات المطلوبة
- بيانات توافق التعريف لـ plugins
- حالة الفحص أو الإشراف
- البلاغات، والتعليقات، والنجوم، والتنزيلات، وإشارات التثبيت حيثما تظهر

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالات الفحص

قد يعرض ClawHub نتائج الفحص أو الإشراف على الصفحات العامة والتشخيصات
المرئية للمالك.

تشمل النتائج الشائعة:

- `clean`: لم يتم العثور على مشكلة مانعة.
- `suspicious`: يتطلب الإصدار الحذر أو المراجعة.
- `malicious`: يُعد الإصدار غير آمن.
- `pending`: لم تنتهِ الفحوصات بعد.
- `held` أو `quarantined` أو `revoked` أو `hidden`: لا يكون الإصدار متاحًا بالكامل
  على واجهات التثبيت العامة.

قد تختلف الصياغة الدقيقة حسب الواجهة، لكن المعنى العملي واحد: إذا كان
الإصدار محتجزًا أو محظورًا، فينبغي للمستخدمين عدم تثبيته حتى يحل المالك
المشكلة أو يعيده الإشراف.

## Skills

تفحص عمليات فحص Skills حزمة Skill المنشورة، وبيانات التعريف، والمتطلبات
المعلنة، والتعليمات المشبوهة.

يولي ClawHub اهتمامًا خاصًا لعدم التطابق بين ما يعلنه Skill وما يبدو أنه
يفعله. على سبيل المثال، ينبغي لـ Skill يشير إلى مفتاح API مطلوب أن يعلن
ذلك المتطلب في `SKILL.md` حتى يتمكن المستخدمون من رؤيته قبل التثبيت.

تستند نتائج الفحص إلى الأثر. يُعامل سلوك المزوّد المتوقع، مثل بيانات اعتماد
API المعلنة، واستدعاءات OAuth الراجعة إلى localhost، وتنظيف الإلغاء المحدد
النطاق، وترميز Basic Auth، أو تحميلات الملفات التي يختارها المستخدم إلى
المزوّد المعلن، بشكل مختلف عن تمرير بيانات الاعتماد المخفية، أو الوصول
الواسع إلى الملفات الخاصة، أو وجهات الشبكة غير ذات الصلة، أو إساءة استخدام
المتصفح خفية.

راجع [تنسيق Skill](/ar/clawhub/skill-format).

## Plugins

تتضمن إصدارات Plugin بيانات تعريف الحزمة، وإسناد المصدر، وحقول التوافق،
ومعلومات سلامة الأثر.

يتحقق OpenClaw من التوافق قبل تثبيت plugins المستضافة على ClawHub. قد تعرض
سجلات الحزم أيضًا بيانات تعريف الملخص حتى يتمكن OpenClaw من التحقق من
الأثرات المنزلة. يتضمن ClawScan بيانات تعريف env/config المعلنة في
`openclaw.environment` للحزمة عند مراجعة إصدارات Plugin، بحيث تُقارن
متطلبات وقت التشغيل المعلنة بالسلوك المرصود.

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن Skills، والحزم، والتعليقات.

ينبغي أن تكون البلاغات محددة وقابلة للإجراء. وقد يؤدي إساءة استخدام الإبلاغ
بحد ذاتها إلى إجراء على الحساب.

أمثلة على البلاغات:

- بيانات تعريف مضللة
- متطلبات بيانات اعتماد أو أذونات غير معلنة
- تعليمات تثبيت مشبوهة
- تعليقات احتيالية أو انتحال شخصية
- تسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- محتوى ينتهك [الاستخدام المقبول](/ar/clawhub/acceptable-usage)

## بلاغات سوء النية أو العلامات التجارية

يستخدم ClawHub مسار البلاغات وإشراف الموظفين نفسه للتسجيلات بسوء نية،
والانتحال، والنزاعات المتعلقة بالعلامات التجارية. تحتاج هذه البلاغات إلى
سياق كافٍ ليتمكن الموظفون من تحديد صاحب المطالبة، والقائمة المتنازع عليها،
والإجراء المطلوب.

ضمّن:

- عنوان URL الأساسي لـ Skill أو الحزمة على ClawHub ومعرّف المالك
- العلامة التجارية أو المشروع أو الشركة أو اسم المنتج محل النزاع
- دليلًا عامًا على ملكية صاحب المطالبة أو سلطته
- سبب عدم تفويض المالك الحالي بالنشر تحت ذلك الاسم
- الإجراء المطلوب، مثل الإخفاء بانتظار المراجعة، أو نقل الملكية، أو إعادة التسمية،
  أو الإزالة

لا تضع أسرارًا خاصة أو مستندات قانونية حساسة في البلاغات العامة. افتح
مشكلة على GitHub مع أدلة غير حساسة واطلب من المشرفين مسار تسليم خاصًا
عند الحاجة.

## الاستئنافات وإعادة الفحص

يمكن للمالكين طلب إعادة فحص عندما يعتقدون أن Skill أو الحزمة تم احتجازها
أو وسمها بشكل غير صحيح. يمكن لمشرفي المنصة والمديرين طلب إعادة الفحص لأي
Skill أو حزمة أثناء معالجة البلاغات أو طلبات الدعم:

```bash
clawhub skill rescan <slug>
clawhub package rescan <name>
```

بالنسبة إلى المحتوى الخاضع للإشراف، قد يتمكن المالكون من تقديم استئناف من
واجهات ClawHub المرئية للمالك. ينبغي أن تشرح الاستئنافات ما الذي تغير أو
لماذا يكون الوسم غير صحيح.

## احتجازات الإشراف

عندما يوسم الفاحص الثابت Skill محمّلًا بأنه ضار، يوضع الناشر تلقائيًا تحت
احتجاز إشراف (`requiresModerationAt` مضبوط على المستخدم). يخفي ذلك كل
Skills الخاصة بالناشر، ويجعل عمليات النشر المستقبلية تبدأ مخفية، وينشئ
إدخال سجل تدقيق `user.moderation.auto`.

تُحتفظ بنتائج الاشتباه الثابتة كدليل ملف/سطر للمشرفين، لكنها لا تخفي
المحتوى ولا تقرر حكم الفحص العام وحدها. تبقى التحميلات الجديدة في حالة
مراجعة/انتظار حتى تستقر مراجعة LLM. يحظر الفحص الثابت فورًا فقط في حالة
التواقيع الضارة. تظل إصابات محركات VirusTotal دليل أمان مرئيًا، لكن أحكام
VirusTotal Code Insight/Palm استشارية ولا تخفي Skills وحدها. تحتفظ مراجعات
ClawScan LLM بالملاحظات المتوافقة مع الغرض كإرشاد. تبقى نتائج المراجعة
المتوسطة مرئية على الأثر، بينما يُحجز مرشح الاشتباه لمخاوف LLM عالية
الأثر، أو النتائج الضارة، أو اكتشافات محركات AV المؤيدة بأدلة.

يمكن للمديرين رفع احتجاز إيجابي كاذب:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

يمسح هذا `requiresModerationAt` و`requiresModerationReason`، ويستعيد
Skills المخفية بسبب الاحتجاز على مستوى المستخدم، ويكتب إدخال سجل تدقيق
`user.moderation.lift`. تبقى Skills المخفية لأسباب أخرى، أو التي لا يزال
فحصها الثابت الخاص ضارًا، مخفية.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub حق الوصول إلى النشر. قد تؤدي
الإساءات الجسيمة إلى حظر الحسابات، أو إبطال الرموز، أو إخفاء المحتوى، أو
إزالة القوائم.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز ClawHub API.
إذا بدأ فشل مصادقة CLI بعد إجراء على الحساب، فسجّل الدخول إلى واجهة الويب
لمراجعة حالة الحساب أو تواصل مع المشرفين عبر قناة دعم المشروع المتوقعة.

## إرشادات الناشرين

لتقليل الإيجابيات الكاذبة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء، والملخصات، والوسوم، وسجلات التغييرات
- أعلن متغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر متى أمكن
- استخدم التشغيلات الجافة قبل نشر plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الحزمة
