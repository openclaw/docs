# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub من خلال GitHub Security Advisories لـ
`openclaw/clawhub`.

استخدم GitHub Security Advisories للثغرات في ClawHub نفسه. تتضمن تقارير
إرشادات ClawHub الجيدة أخطاء في:

- موقع ClawHub الإلكتروني أو API أو CLI
- نشر السجل، أو التنزيلات، أو عمليات التثبيت، أو سلامة القطع الأثرية
- المصادقة، أو التفويض، أو رموز API
- الفحص، أو الإشراف، أو معالجة التقارير

لا تستخدم إرشادات ClawHub للثغرات في الشيفرة المصدرية الخاصة بـ Skills أو
Plugin تابع لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من إدراج ClawHub.

## الإفصاح عن الثغرات

نظرًا لأن ClawHub تطبيق سحابي مستضاف، فإن ثغرات خدمة ClawHub لا يُفصح عنها
علنًا افتراضيًا. يُفصح عنها علنًا عندما توجد أدلة على تأثير حقيقي في
المستخدمين أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الحقيقي في المستخدمين الاستغلال المؤكد، أو كشف بيانات
المستخدمين أو أسرارهم، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد، أو تحديث البرامج
المحلية، أو اتخاذ إجراء وقائي آخر.

يُفصح علنًا عن الثغرات في البرامج المثبتة من قبل المستخدمين، مثل حزم
ClawHub CLI أو الثنائيات أو المكتبات أو قطع الإصدار الأثرية الأخرى التي
يحتاج المستخدمون إلى تحديثها محليًا.

## الصفحات ذات الصلة

للاطلاع على تسميات التدقيق وقت التثبيت، ومستويات المخاطر، والنتائج،
والتفسير، راجع [تدقيقات الأمان](/clawhub/security-audits).

بالنسبة إلى تقارير السوق، وتعليقات الإشراف، والإدراجات المخفية، والحظر،
وحالة الحساب، راجع [الإشراف وسلامة الحساب](/ar/clawhub/moderation).
