# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub عبر GitHub Security Advisories لـ
`openclaw/clawhub`.

استخدم GitHub Security Advisories للثغرات في ClawHub نفسه. تشمل تقارير
إرشادات ClawHub الجيدة الأخطاء في:

- موقع ClawHub الإلكتروني أو API أو CLI
- نشر السجل أو التنزيلات أو عمليات التثبيت أو سلامة القطع الأثرية
- المصادقة أو التفويض أو رموز API
- الفحص أو الإشراف أو معالجة التقارير

لا تستخدم إرشادات ClawHub للثغرات في الشيفرة المصدرية الخاصة بـ skill أو
plugin تابع لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من قائمة ClawHub.

## الإفصاح عن الثغرات

لأن ClawHub تطبيق سحابي مستضاف، لا يتم الإفصاح علنًا عن ثغرات خدمة ClawHub
افتراضيًا. يتم الإفصاح عنها علنًا عندما يكون هناك دليل على تأثير حقيقي في
المستخدمين أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الحقيقي في المستخدمين الاستغلال المؤكد، أو كشف بيانات
المستخدمين أو أسرارهم، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد أو تحديث البرامج
المحلية أو اتخاذ إجراء وقائي آخر.

يتم الإفصاح علنًا عن الثغرات في البرامج التي يثبّتها المستخدمون، مثل حزم
ClawHub CLI أو الثنائيات أو المكتبات أو غيرها من قطع الإصدار الأثرية التي
يحتاج المستخدمون إلى تحديثها محليًا.

## الصفحات ذات الصلة

للاطلاع على تسميات التدقيق وقت التثبيت ومستويات المخاطر والنتائج والتفسير، راجع
