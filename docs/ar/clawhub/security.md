# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub من خلال تنبيهات الأمان في GitHub الخاصة بـ
`openclaw/clawhub`.

استخدم تنبيهات الأمان في GitHub للثغرات الموجودة في ClawHub نفسه. تشمل تقارير
تنبيهات ClawHub الجيدة الأخطاء في:

- موقع ClawHub الإلكتروني أو واجهة API أو CLI
- نشر السجل أو التنزيلات أو عمليات التثبيت أو سلامة العناصر البرمجية
- المصادقة أو التخويل أو رموز API
- الفحص أو الإشراف أو معالجة البلاغات

لا تستخدم تنبيهات ClawHub للثغرات الموجودة في الشيفرة المصدرية الخاصة بـ Skill
أو Plugin تابع لجهة خارجية. أبلغ عنها مباشرةً إلى الناشر أو مستودع الشيفرة
المصدرية المرتبط من قائمة ClawHub.

## الإفصاح عن الثغرات

نظرًا إلى أن ClawHub تطبيق سحابي مستضاف، لا يُفصح علنًا افتراضيًا عن ثغرات خدمة
ClawHub. ويُفصح عنها علنًا عند وجود دليل على تأثير فعلي في المستخدمين أو عندما
يتعين على المستخدمين اتخاذ إجراء.

تشمل أمثلة التأثير الفعلي في المستخدمين الاستغلال المؤكد، أو كشف بيانات
المستخدمين أو أسرارهم، أو وصول محتوى ضار إلى المستخدمين بسبب إخفاق في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد، أو تحديث البرامج
المحلية، أو اتخاذ إجراء وقائي آخر.

يُفصح علنًا عن الثغرات في البرامج التي يثبتها المستخدمون، مثل حزم CLI الخاصة
بـ ClawHub أو الملفات الثنائية أو المكتبات أو عناصر الإصدار الأخرى التي يتعين
على المستخدمين تحديثها محليًا.

للاطلاع على تسميات التدقيق أثناء التثبيت ومستويات المخاطر والنتائج وتفسيرها،
