# الأمان + الإشراف

ClawHub مفتوح للنشر، لكن القوائم العامة ما تزال تمر عبر ضوابط الثقة
والفحص والإبلاغ والإشراف. الهدف عملي: مساعدة المستخدمين على فحص ما
يثبتونه، ومنح الناشرين مسار استرداد عند النتائج الإيجابية الكاذبة،
وإبقاء الحزم المسيئة خارج الاكتشاف العام.

راجع أيضًا [الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## ما يمكن للمستخدمين فحصه

قبل تثبيت Skill أو Plugin، تحقق من قائمة ClawHub الخاصة به لمعرفة:

- المالك ونسبة المصدر
- أحدث إصدار وسجل التغييرات
- متغيرات البيئة أو الأذونات المطلوبة
- بيانات التوافق الوصفية للـ Plugins
- حالة الفحص أو الإشراف
- البلاغات، والتعليقات، والنجوم، والتنزيلات، وإشارات التثبيت حيثما تظهر

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالات الفحص

قد يعرض ClawHub نتائج الفحص أو الإشراف على الصفحات العامة وبيانات التشخيص
المرئية للمالك.

تشمل النتائج الشائعة:

- `clean`: لم تُعثر أي مشكلة حاجبة.
- `suspicious`: يحتاج الإصدار إلى الحذر أو المراجعة.
- `malicious`: يُعد الإصدار غير آمن.
- `pending`: لم تكتمل الفحوصات بعد.
- `held` أو `quarantined` أو `revoked` أو `hidden`: الإصدار غير متاح بالكامل
  على أسطح التثبيت العامة.

قد تختلف الصياغة الدقيقة بحسب السطح، لكن المعنى العملي هو نفسه: إذا كان
الإصدار محتجزًا أو محجوبًا، فينبغي ألا يثبته المستخدمون إلى أن يحل المالك
المشكلة أو يعيده الإشراف.

## Skills

تفحص عمليات فحص Skills حزمة الـ Skill المنشورة، وبياناتها الوصفية، والمتطلبات
المعلنة، والتعليمات المشبوهة.

يركز ClawHub بشكل خاص على حالات عدم التطابق بين ما تعلنه الـ Skill وما يبدو
أنها تفعله. على سبيل المثال، ينبغي للـ Skill التي تشير إلى مفتاح API مطلوب
أن تعلن هذا المتطلب في `SKILL.md` حتى يتمكن المستخدمون من رؤيته قبل التثبيت.

نتائج الفحص مبنية على الأثر. يُعامل سلوك المزوّد المتوقع، مثل بيانات اعتماد
API المعلنة، أو استدعاءات OAuth الراجعة إلى المضيف المحلي، أو تنظيف إلغاء
التثبيت محدود النطاق، أو ترميز Basic Auth، أو عمليات رفع الملفات التي يختارها
المستخدم إلى المزوّد المعلن، بشكل مختلف عن تمرير بيانات الاعتماد المخفي، أو
الوصول الواسع إلى الملفات الخاصة، أو وجهات الشبكة غير ذات الصلة، أو إساءة
استخدام المتصفح خفية.

راجع [تنسيق Skill](/ar/clawhub/skill-format).

## Plugins

تتضمن إصدارات Plugin بيانات الحزمة الوصفية، ونسبة المصدر، وحقول التوافق،
ومعلومات سلامة الأثر.

يتحقق OpenClaw من التوافق قبل تثبيت Plugins المستضافة على ClawHub. وقد تعرض
سجلات الحزم أيضًا بيانات وصفية للملخص حتى يتمكن OpenClaw من التحقق من الآثار
المنزلة. يتضمن ClawScan بيانات env/config الوصفية المعلنة للحزمة
`openclaw.environment` عند مراجعة إصدارات Plugin حتى تُقارن متطلبات وقت
التشغيل المعلنة بالسلوك المرصود.

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن Skills والحزم والتعليقات.

ينبغي أن تكون البلاغات محددة وقابلة للتنفيذ. وقد يؤدي إساءة استخدام الإبلاغ
بحد ذاتها إلى إجراء على الحساب.

أمثلة على البلاغات:

- بيانات وصفية مضللة
- متطلبات غير معلنة لبيانات الاعتماد أو الأذونات
- تعليمات تثبيت مشبوهة
- تعليقات احتيالية أو انتحال
- تسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- محتوى ينتهك [الاستخدام المقبول](/ar/clawhub/acceptable-usage)

## ملاحظات ClawScan للناشرين

يمكن للناشرين تقديم ملاحظة ClawScan اختيارية عند نشر Skill أو Plugin. تمنح
هذه الملاحظة ClawScan سياقًا لسلوك قد يبدو غير معتاد بخلاف ذلك، مثل الوصول
إلى الشبكة، أو الوصول إلى المضيف الأصلي، أو بيانات اعتماد خاصة بالمزوّد.

## حالات الاحتجاز للإشراف

عندما يعلّم الفاحص الثابت Skill مرفوعة على أنها خبيثة، يوضع الناشر تلقائيًا
تحت احتجاز إشرافي (مع ضبط `requiresModerationAt` على المستخدم). يخفي ذلك كل
Skills الخاصة بالناشر، ويجعل عمليات النشر المستقبلية تبدأ مخفية، وينشئ إدخال
سجل تدقيق `user.moderation.auto`.

تُحتفظ بالنتائج الثابتة المشبوهة كدليل ملف/سطر للمشرفين، لكنها لا تخفي
المحتوى ولا تقرر حكم الفحص العام بمفردها. تبقى عمليات الرفع الجديدة في حالة
مراجعة/انتظار حتى تستقر مراجعة LLM. لا يحجب الفحص الثابت فورًا إلا عند وجود
تواقيع خبيثة. تظل إصابات محركات VirusTotal دليلًا أمنيًا مرئيًا، لكن أحكام
VirusTotal Code Insight/Palm إرشادية ولا تخفي Skills بمفردها. تحتفظ مراجعات
ClawScan LLM بالملاحظات المتوافقة مع الغرض كإرشاد. تبقى نتائج المراجعة
المتوسطة مرئية على الأثر، بينما يُحجز مرشح الاشتباه لمخاوف LLM عالية الأثر،
أو النتائج الخبيثة، أو اكتشافات محركات AV المؤيدة بأدلة.

يمكن للمسؤولين رفع احتجاز نتيجة إيجابية كاذبة:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

يمسح ذلك `requiresModerationAt` و`requiresModerationReason`، ويعيد Skills التي
أخفاها الاحتجاز على مستوى المستخدم، ويكتب إدخال سجل تدقيق
`user.moderation.lift`. تبقى Skills المخفية لأسباب أخرى، أو التي لا يزال فحصها
الثابت نفسه خبيثًا، مخفية.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub صلاحية النشر. ويمكن أن تؤدي
الإساءة الشديدة إلى حظر الحسابات، أو إبطال الرموز، أو إخفاء المحتوى، أو إزالة
القوائم.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز ClawHub API.
إذا بدأ فشل مصادقة CLI بعد إجراء على الحساب، فسجّل الدخول إلى واجهة الويب
لمراجعة حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محجوبًا،
فاتصل بـ security@openclaw.ai لمراجعة الاسترداد.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الكاذبة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغييرات
- أعلن متغيرات البيئة والأذونات المطلوبة
- أضف ملاحظة ClawScan للناشر عندما يتضمن الإصدار سلوكًا غير معتاد لكنه مقصود
- تجنب أوامر التثبيت المموهة
- اربط بالمصدر عندما يكون ذلك ممكنًا
- استخدم التشغيلات الجافة قبل نشر Plugins
- رد بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الحزمة
