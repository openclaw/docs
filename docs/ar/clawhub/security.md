# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub من خلال GitHub Security Advisories لـ
`openclaw/clawhub`.

استخدم GitHub Security Advisories للثغرات في ClawHub نفسه. تتضمن تقارير
استشارات ClawHub الجيدة أخطاء في:

- موقع ClawHub الإلكتروني أو API أو CLI
- نشر السجل أو التنزيلات أو التثبيتات أو سلامة الأثر
- المصادقة أو التخويل أو رموز API
- الفحص أو الإشراف أو معالجة البلاغات

لا تستخدم استشارات ClawHub للثغرات في الشيفرة المصدرية الخاصة بمهارة أو
Plugin تابع لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من قائمة ClawHub.

## الإفصاح عن الثغرات

نظرا إلى أن ClawHub تطبيق سحابي مستضاف، لا يتم الإفصاح علنا عن ثغرات خدمة
ClawHub افتراضيا. يتم الإفصاح عنها علنا عندما توجد أدلة على تأثير حقيقي في
المستخدمين أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الحقيقي في المستخدمين الاستغلال المؤكد، أو كشف بيانات
المستخدمين أو أسرارهم، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد، أو تحديث البرامج
المحلية، أو اتخاذ إجراء وقائي آخر.

يتم الإفصاح علنا عن الثغرات في البرامج التي يثبتها المستخدمون، مثل حزم
ClawHub CLI أو الثنائيات أو المكتبات أو آثار الإصدارات الأخرى التي يحتاج
المستخدمون إلى تحديثها محليا.

## الصفحات ذات الصلة

للاطلاع على تسميات التدقيق وقت التثبيت ومستويات المخاطر والنتائج والتفسير، راجع
[تدقيقات الأمان](/ar/clawhub/security-audits).

بالنسبة إلى بلاغات السوق، وعمليات تعليق الإشراف، والقوائم المخفية، والحظر،
وحالة الحساب، راجع [الإشراف وسلامة الحساب](/ar/clawhub/moderation).
