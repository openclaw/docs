# الأمان + الإشراف

ClawHub مفتوح للنشر، لكن الإدراجات العامة لا تزال تمر عبر ضوابط الثقة،
والفحص، والإبلاغ، والإشراف. الهدف عملي: مساعدة المستخدمين على
فحص ما يثبتونه، ومنح الناشرين مسار استرداد للحالات الإيجابية الكاذبة،
وإبقاء الحزم المسيئة خارج الاكتشاف العام.

راجع أيضًا [الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## ما يمكن للمستخدمين فحصه

قبل تثبيت Skill أو Plugin، تحقق من إدراجه في ClawHub للاطلاع على:

- إسناد المالك والمصدر
- أحدث إصدار وسجل التغييرات
- متغيرات البيئة أو الأذونات المطلوبة
- بيانات توافق التعريفات للـ Plugins
- حالة الفحص أو الإشراف
- البلاغات، والتعليقات، والنجوم، والتنزيلات، وإشارات التثبيت حيثما تظهر

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالات الفحص

قد يعرض ClawHub نتائج الفحص أو الإشراف على الصفحات العامة وفي
التشخيصات المرئية للمالك.

تشمل النتائج الشائعة:

- `clean`: لم تُعثر مشكلة مانعة.
- `suspicious`: يحتاج الإصدار إلى حذر أو مراجعة.
- `malicious`: يُعد الإصدار غير آمن.
- `pending`: لم تنتهِ الفحوصات بعد.
- `held`، أو `quarantined`، أو `revoked`، أو `hidden`: الإصدار غير متاح بالكامل
  على واجهات التثبيت العامة.

قد تختلف الصياغة الدقيقة حسب الواجهة، لكن المعنى العملي هو نفسه: إذا كان
الإصدار محتجزًا أو محظورًا، فيجب على المستخدمين عدم تثبيته حتى يحل المالك
المشكلة أو يعيده الإشراف.

## Skills

تفحص عمليات فحص Skill حزمة Skill المنشورة، وبيانات التعريف، والمتطلبات
المعلنة، والتعليمات المشبوهة.

يولي ClawHub اهتمامًا خاصًا لعدم التطابق بين ما تعلنه Skill وما يبدو أنها
تفعله. على سبيل المثال، يجب على Skill التي تشير إلى مفتاح API مطلوب أن تعلن
ذلك المتطلب في `SKILL.md` حتى يتمكن المستخدمون من رؤيته قبل التثبيت.

تعتمد نتائج الفحص على الأثر. ويُعامل سلوك المزوّد المتوقع، مثل بيانات اعتماد
API المعلنة، أو استدعاءات OAuth الراجعة إلى localhost، أو تنظيف إلغاء التثبيت
المقيّد النطاق، أو ترميز Basic Auth، أو تحميلات الملفات التي يختارها المستخدم
إلى المزوّد المعلن، بطريقة مختلفة عن تمرير بيانات الاعتماد المخفي، أو الوصول
الواسع إلى الملفات الخاصة، أو وجهات الشبكة غير المرتبطة، أو إساءة استخدام
المتصفح خفيةً.

راجع [تنسيق Skill](/ar/clawhub/skill-format).

## Plugins

تتضمن إصدارات Plugin بيانات تعريف الحزمة، وإسناد المصدر، وحقول التوافق،
ومعلومات سلامة الأثر.

يتحقق OpenClaw من التوافق قبل تثبيت Plugins المستضافة على ClawHub. قد تكشف
سجلات الحزم أيضًا عن بيانات تعريف البصمة حتى يتمكن OpenClaw من التحقق من
الأثار المنزلة. يتضمن ClawScan بيانات تعريف env/config الخاصة بـ
`openclaw.environment` المعلنة للحزمة عند مراجعة إصدارات Plugin، بحيث تُقارن
متطلبات وقت التشغيل المعلنة بالسلوك المرصود.

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن Skills، والحزم، والتعليقات.

يجب أن تكون البلاغات محددة وقابلة للتنفيذ. قد يؤدي إساءة استخدام الإبلاغ
بحد ذاتها إلى إجراء على الحساب.

أمثلة على البلاغات:

- بيانات تعريف مضللة
- متطلبات بيانات اعتماد أو أذونات غير معلنة
- تعليمات تثبيت مشبوهة
- تعليقات احتيالية أو انتحال شخصية
- تسجيلات بسوء نية أو إساءة استخدام علامة تجارية
- محتوى ينتهك [الاستخدام المقبول](/ar/clawhub/acceptable-usage)

## ملاحظات ClawScan للناشر

يمكن للناشرين تقديم ملاحظة ClawScan اختيارية عند نشر Skill أو Plugin. تمنح
هذه الملاحظة ClawScan سياقًا لسلوك قد يبدو غير معتاد بخلاف ذلك، مثل الوصول
إلى الشبكة، أو الوصول إلى المضيف الأصلي، أو بيانات الاعتماد الخاصة بمزوّد معين.

## احتجازات الإشراف

عندما يعلّم الفاحص الثابت Skill مرفوعة على أنها ضارة، يُوضع الناشر تلقائيًا
تحت احتجاز إشراف (`requiresModerationAt` معيّن على المستخدم). يؤدي ذلك إلى
إخفاء جميع Skills الخاصة بالناشر، ويجعل عمليات النشر المستقبلية تبدأ مخفية،
وينشئ إدخال سجل تدقيق `user.moderation.auto`.

تُحتفظ بنتائج الاشتباه الثابتة كأدلة ملف/سطر للمشرفين، لكنها لا تخفي المحتوى
ولا تقرر حكم الفحص العام بمفردها. تبقى الرفعات الجديدة في حالة مراجعة/انتظار
حتى تستقر مراجعة LLM. لا يحظر الفحص الثابت فورًا إلا للتواقيع الضارة. تبقى
إصابات محركات VirusTotal أدلة أمان مرئية، لكن أحكام VirusTotal Code Insight/Palm
استشارية ولا تخفي Skills بمفردها. تحتفظ مراجعات ClawScan LLM بالملاحظات
المتوافقة مع الغرض كإرشاد. تبقى نتائج المراجعة المتوسطة مرئية على الأثر،
بينما يُحجز مرشح الاشتباه للمخاوف عالية التأثير من LLM، أو النتائج الضارة،
أو اكتشافات محركات مكافحة الفيروسات المؤيدة بأدلة.

يمكن للمسؤولين رفع احتجاز إيجابي كاذب:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

يمسح هذا `requiresModerationAt` و`requiresModerationReason`، ويستعيد
Skills التي أخفاها الاحتجاز على مستوى المستخدم، ويكتب إدخال سجل تدقيق
`user.moderation.lift`. تبقى Skills المخفية لأسباب أخرى، أو التي يظل فحصها
الثابت ضارًا، مخفية.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub صلاحية النشر. قد تؤدي الإساءة
الجسيمة إلى حظر الحساب، أو إبطال الرموز، أو إخفاء المحتوى، أو إزالة الإدراجات.

لا يمكن للحسابات المحذوفة، أو المحظورة، أو المعطلة استخدام رموز ClawHub API.
إذا بدأ توثيق CLI بالفشل بعد إجراء على الحساب، فسجّل الدخول إلى واجهة الويب
لمراجعة حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محظورًا،
فاتصل بـ security@openclaw.ai لمراجعة الاسترداد.

## إرشادات الناشرين

لتقليل الحالات الإيجابية الكاذبة وتحسين ثقة المستخدمين:

- أبقِ الأسماء، والملخصات، والوسوم، وسجلات التغييرات دقيقة
- أعلن متغيرات البيئة والأذونات المطلوبة
- أضف ملاحظة ClawScan للناشر عندما يكون للإصدار سلوك غير معتاد لكنه مقصود
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنًا
- استخدم عمليات التشغيل التجريبي قبل نشر Plugins
- أجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الحزمة
