# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub عبر GitHub Security Advisories الخاصة بـ
`openclaw/clawhub`.

استخدم GitHub Security Advisories للثغرات في ClawHub نفسه. تتضمن تقارير
ClawHub الاستشارية الجيدة أخطاء في:

- موقع ClawHub الإلكتروني أو API أو CLI
- نشر السجل أو التنزيلات أو عمليات التثبيت أو سلامة القطع الأثرية
- المصادقة أو التخويل أو رموز API
- الفحص أو الإشراف أو معالجة البلاغات

لا تستخدم استشارات ClawHub للثغرات في شيفرة المصدر الخاصة بـ skill تابع لجهة خارجية أو
plugin. أبلغ عنها مباشرةً إلى الناشر أو مستودع المصدر
المرتبط من قائمة ClawHub.

## الإفصاح عن الثغرات

نظرًا لأن ClawHub تطبيق سحابي مستضاف، لا يتم الإفصاح علنًا عن ثغرات خدمة ClawHub
افتراضيًا. ويتم الإفصاح عنها علنًا عندما يكون هناك
دليل على تأثير حقيقي في المستخدمين أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الحقيقي في المستخدمين الاستغلال المؤكد، أو كشف بيانات المستخدمين
أو أسرارهم، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد أو تحديث البرامج المحلية أو
اتخاذ إجراء وقائي آخر.

يتم الإفصاح علنًا عن الثغرات في البرامج المثبتة من قِبل المستخدمين، مثل
حزم ClawHub CLI أو الثنائيات أو المكتبات أو غيرها من قطع الإصدار الأثرية التي يحتاج المستخدمون
إلى تحديثها محليًا.
