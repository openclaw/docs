# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub عبر استشارات GitHub الأمنية الخاصة بـ
`openclaw/clawhub`.

استخدم استشارات GitHub الأمنية للثغرات في ClawHub نفسه. تتضمن تقارير
استشارات ClawHub الجيدة أخطاءً في:

- موقع ClawHub أو API أو CLI
- النشر في السجل، أو التنزيلات، أو عمليات التثبيت، أو سلامة القطع الأثرية
- المصادقة، أو التفويض، أو رموز API
- الفحص، أو الإشراف، أو معالجة التقارير

لا تستخدم استشارات ClawHub للثغرات الموجودة في الشيفرة المصدرية الخاصة بمهارة
أو Plugin تابع لجهة خارجية. أبلغ عنها مباشرةً إلى الناشر أو مستودع المصدر
المرتبط من إدراج ClawHub.

## الإفصاح عن الثغرات

نظرًا لأن ClawHub تطبيق سحابي مستضاف، لا يُفصح عن ثغرات خدمة ClawHub علنًا
بشكل افتراضي. يُفصح عنها علنًا عندما توجد أدلة على تأثير فعلي في المستخدمين
أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تتضمن أمثلة التأثير الفعلي في المستخدمين الاستغلال المؤكد، أو كشف بيانات
المستخدمين أو أسرارهم، أو وصول محتوى خبيث إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد، أو تحديث البرامج
المحلية، أو اتخاذ إجراء وقائي آخر.

يُفصح علنًا عن الثغرات في البرامج المثبتة من قبل المستخدمين، مثل حزم
ClawHub CLI أو الثنائيات أو المكتبات أو غيرها من قطع الإصدار الأثرية التي
يحتاج المستخدمون إلى تحديثها محليًا.
