# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub من خلال GitHub Security Advisories لـ
`openclaw/clawhub`.

استخدم GitHub Security Advisories للثغرات في ClawHub نفسه. تتضمن تقارير
ClawHub الاستشارية الجيدة أخطاء في:

- موقع ClawHub أو API أو CLI
- نشر السجل أو التنزيلات أو عمليات التثبيت أو سلامة العناصر
- المصادقة أو التخويل أو رموز API
- الفحص أو الإشراف أو معالجة البلاغات

لا تستخدم الاستشارات الأمنية الخاصة بـ ClawHub للثغرات في الشيفرة المصدرية الخاصة بمهارة أو
plugin تابعة لطرف ثالث. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من قائمة ClawHub.

## الإفصاح عن الثغرات

نظرًا إلى أن ClawHub تطبيق سحابي مستضاف، لا يتم الإفصاح علنًا عن ثغرات خدمة ClawHub
افتراضيًا. يتم الإفصاح عنها علنًا عندما توجد
أدلة على أثر حقيقي على المستخدمين أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة الأثر الحقيقي على المستخدمين الاستغلال المؤكد، أو كشف بيانات المستخدمين
أو الأسرار، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد، أو تحديث البرامج المحلية، أو
اتخاذ إجراء وقائي آخر.

يتم الإفصاح علنًا عن الثغرات في البرامج المثبتة من قبل المستخدمين، مثل
حزم ClawHub CLI أو الملفات الثنائية أو المكتبات أو عناصر الإصدار الأخرى التي يحتاج المستخدمون
إلى تحديثها محليًا.

## الصفحات ذات الصلة

للاطلاع على تسميات التدقيق وقت التثبيت، ومستويات المخاطر، والنتائج، وتفسيرها، راجع
[تدقيقات الأمان](/ar/clawhub/security-audits).

للاطلاع على بلاغات السوق، وتعليقات الإشراف، والقوائم المخفية، والحظر، وحالة الحساب،
راجع [الإشراف وسلامة الحساب](/ar/clawhub/moderation).
