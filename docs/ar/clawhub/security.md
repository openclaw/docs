# الأمان + الإشراف

ClawHub مفتوح للنشر، لكن القوائم العامة ما زالت تمر عبر ضوابط الثقة
والفحص والإبلاغ والإشراف. الهدف عملي: مساعدة المستخدمين على
فحص ما يثبتونه، ومنح الناشرين مسار استرداد للنتائج الإيجابية الكاذبة،
وإبقاء الحزم المسيئة خارج الاكتشاف العام.

راجع أيضًا [الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## ما يمكن للمستخدمين فحصه

قبل تثبيت Skill أو plugin، تحقق من قائمة ClawHub الخاصة به لمعرفة:

- المالك ونسبة المصدر
- أحدث إصدار وسجل التغييرات
- متغيرات البيئة أو الأذونات المطلوبة
- بيانات التوافق الوصفية للـ plugins
- حالة الفحص أو الإشراف
- البلاغات والتعليقات والنجوم والتنزيلات وإشارات التثبيت حيثما تظهر

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالات الفحص

قد يعرض ClawHub نتائج الفحص أو الإشراف على الصفحات العامة والتشخيصات
المرئية للمالك.

تشمل النتائج الشائعة:

- `clean`: لم تُعثر مشكلة حاجبة.
- `suspicious`: يحتاج الإصدار إلى الحذر أو المراجعة.
- `malicious`: يُعد الإصدار غير آمن.
- `pending`: لم تكتمل الفحوصات بعد.
- `held` أو `quarantined` أو `revoked` أو `hidden`: الإصدار غير متاح بالكامل
  على أسطح التثبيت العامة.

قد تختلف الصياغة الدقيقة حسب السطح، لكن المعنى العملي واحد: إذا كان
الإصدار محتجزًا أو محظورًا، فينبغي ألا يثبته المستخدمون حتى يحل المالك
المشكلة أو يعيد الإشراف إتاحته.

## Skills

تنظر فحوصات Skills إلى حزمة Skill المنشورة، والبيانات الوصفية، والمتطلبات
المعلنة، والتعليمات المشبوهة.

يهتم ClawHub خصوصًا بحالات عدم التطابق بين ما يعلنه Skill وما يبدو أنه
يفعله. على سبيل المثال، ينبغي أن يعلن Skill يشير إلى مفتاح API مطلوب عن ذلك
المطلب في `SKILL.md` حتى يتمكن المستخدمون من رؤيته قبل التثبيت.

تستند نتائج الفحص إلى الأثر. يُعامل سلوك المزوّد المتوقع، مثل بيانات اعتماد
API المعلنة، أو استدعاءات OAuth الراجعة عبر localhost، أو تنظيف إلغاء
التثبيت محدود النطاق، أو ترميز Basic Auth، أو تحميلات الملفات التي يختارها
المستخدم إلى المزوّد المعلن، بشكل مختلف عن تمرير بيانات الاعتماد المخفي،
أو الوصول الواسع إلى الملفات الخاصة، أو الوجهات الشبكية غير ذات الصلة،
أو إساءة استخدام المتصفح سرًا.

راجع [تنسيق Skill](/ar/clawhub/skill-format).

## Plugins

تتضمن إصدارات Plugins بيانات الحزمة الوصفية، ونسبة المصدر، وحقول التوافق،
ومعلومات سلامة الأثر.

يفحص OpenClaw التوافق قبل تثبيت plugins المستضافة على ClawHub. قد تكشف
سجلات الحزم أيضًا بيانات digest وصفية حتى يتمكن OpenClaw من التحقق من
الآثار التي تم تنزيلها. يتضمن ClawScan بيانات env/config الوصفية المعلنة
في `openclaw.environment` للحزمة عند مراجعة إصدارات Plugin، بحيث تُقارن
متطلبات وقت التشغيل المعلنة بالسلوك المرصود.

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن Skills والحزم والتعليقات.

ينبغي أن تكون البلاغات محددة وقابلة للتنفيذ. إساءة استخدام الإبلاغ قد تؤدي
بحد ذاتها إلى إجراء على الحساب.

أمثلة البلاغات:

- بيانات وصفية مضللة
- متطلبات بيانات اعتماد أو أذونات غير معلنة
- تعليمات تثبيت مشبوهة
- تعليقات احتيالية أو انتحال
- تسجيلات بسوء نية أو إساءة استخدام علامات تجارية
- محتوى ينتهك [الاستخدام المقبول](/ar/clawhub/acceptable-usage)

## بلاغات سوء النية أو العلامات التجارية

يستخدم ClawHub مسار البلاغات وإشراف الموظفين نفسه للتسجيلات بسوء نية،
والانتحال، والنزاعات المتعلقة بالعلامات التجارية. تحتاج هذه البلاغات إلى
سياق كافٍ حتى يتمكن الموظفون من تحديد المدعي، والقائمة المتنازع عليها،
والإجراء المطلوب.

ضمّن:

- عنوان URL الرسمي لـ Skill أو الحزمة على ClawHub ومعرّف المالك
- العلامة التجارية أو المشروع أو الشركة أو اسم المنتج محل النزاع
- دليلًا عامًا على ملكية المدعي أو صلاحيته
- سبب عدم تخويل المالك الحالي للنشر تحت ذلك الاسم
- الإجراء المطلوب، مثل الإخفاء بانتظار المراجعة، أو نقل الملكية، أو إعادة التسمية،
  أو الإزالة

لا تضع أسرارًا خاصة أو مستندات قانونية حساسة في البلاغات العامة. افتح
مشكلة على GitHub مع أدلة غير حساسة واطلب من المشرفين مسار تسليم خاصًا عند
الحاجة.

## الاستئنافات وإعادة الفحص

يمكن للمالكين طلب إعادة فحص عندما يعتقدون أن Skill أو حزمة احتُجزت أو وُسمت
بشكل غير صحيح. يمكن لمشرفي المنصة ومديريها طلب إعادة الفحص لأي Skill أو
حزمة أثناء التعامل مع البلاغات أو طلبات الدعم:

```bash
clawhub skill rescan <slug>
clawhub package rescan <name>
```

بالنسبة إلى المحتوى الخاضع للإشراف، قد يتمكن المالكون من تقديم استئناف من
أسطح ClawHub المرئية للمالك. ينبغي أن تشرح الاستئنافات ما تغيّر أو لماذا
تكون العلامة غير صحيحة.

## احتجازات الإشراف

عندما يوسم الفاحص الثابت Skill مرفوعًا بأنه ضار، يوضع الناشر تلقائيًا تحت
احتجاز إشرافي (`requiresModerationAt` مضبوط على المستخدم). يؤدي ذلك إلى
إخفاء جميع Skills الخاصة بالناشر، ويجعل عمليات النشر المستقبلية تبدأ مخفية،
وينشئ إدخال سجل تدقيق `user.moderation.auto`.

تُحتفظ بنتائج الاشتباه الثابتة كأدلة ملف/سطر للمشرفين، لكنها لا تخفي
المحتوى ولا تحدد حكم الفحص العام بمفردها. تبقى التحميلات الجديدة في حالة
المراجعة/الانتظار حتى تستقر مراجعات VirusTotal وLLM؛ ولا يحظر الفحص الثابت
فورًا إلا للتوقيعات الضارة. تحتفظ مراجعات ClawScan LLM بملاحظات متوافقة مع
الغرض كإرشاد؛ ولا تعيد حكم Review/suspicious إلا عندما تتضمن المراجعة
المنظمة مخاوف جوهرية.

يمكن للمديرين رفع احتجاز إيجابي كاذب:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

يمسح هذا `requiresModerationAt` و`requiresModerationReason`، ويستعيد
Skills المخفية بسبب الاحتجاز على مستوى المستخدم، ويكتب إدخال سجل تدقيق
`user.moderation.lift`. تبقى Skills المخفية لأسباب أخرى، أو التي لا يزال
فحصها الثابت نفسه ضارًا، مخفية.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub إمكانية النشر. وقد يؤدي سوء
الاستخدام الشديد إلى حظر الحساب، أو إبطال الرموز، أو إخفاء المحتوى، أو إزالة
القوائم.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز ClawHub API.
إذا بدأت مصادقة CLI بالفشل بعد إجراء على الحساب، فسجّل الدخول إلى واجهة
الويب لمراجعة حالة الحساب أو تواصل مع المشرفين عبر قناة دعم المشروع المتوقعة.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الكاذبة وتحسين ثقة المستخدمين:

- أبقِ الأسماء والملخصات والوسوم وسجلات التغييرات دقيقة
- أعلن عن متغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنًا
- استخدم عمليات التشغيل الجاف قبل نشر plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الحزمة
