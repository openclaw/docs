# الأمان + الإشراف

ClawHub مفتوح للنشر، لكن القوائم العامة لا تزال تمر عبر ضوابط الثقة،
والفحص، والإبلاغ، والإشراف. الهدف عملي: مساعدة المستخدمين على
فحص ما يثبتونه، ومنح الناشرين مسار استرداد للحالات الإيجابية الكاذبة،
ومنع الحزم المسيئة من الظهور في الاكتشاف العام.

انظر أيضًا [الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## ما يمكن للمستخدمين فحصه

قبل تثبيت skill أو plugin، تحقق من قائمة ClawHub الخاصة بها لمعرفة:

- إسناد المالك والمصدر
- أحدث إصدار وسجل التغييرات
- متغيرات البيئة أو الأذونات المطلوبة
- بيانات تعريف التوافق للـ plugins
- حالة الفحص أو الإشراف
- البلاغات، والتعليقات، والنجوم، والتنزيلات، وإشارات التثبيت حيثما تظهر

ثبّت فقط المحتوى الذي تفهمه وتثق به.

## حالات الفحص

قد يعرض ClawHub نتائج الفحص أو الإشراف على الصفحات العامة وفي التشخيصات
المرئية للمالك.

تشمل النتائج الشائعة:

- `clean`: لم يتم العثور على مشكلة مانعة.
- `suspicious`: يحتاج الإصدار إلى الحذر أو المراجعة.
- `malicious`: يُعد الإصدار غير آمن.
- `pending`: لم تكتمل الفحوصات بعد.
- `held` أو `quarantined` أو `revoked` أو `hidden`: الإصدار غير متاح بالكامل
  على أسطح التثبيت العامة.

قد تختلف الصياغة الدقيقة حسب السطح، لكن المعنى العملي واحد: إذا كان الإصدار
محتجزًا أو محظورًا، فيجب ألا يثبته المستخدمون حتى يحل المالك المشكلة أو يعيده
الإشراف.

## Skills

تفحص عمليات فحص Skills حزمة skill المنشورة، وبياناتها التعريفية، والمتطلبات
المعلنة، والتعليمات المشبوهة.

يولي ClawHub اهتمامًا خاصًا لحالات عدم التطابق بين ما تصرح به skill وما يبدو
أنها تفعله. على سبيل المثال، يجب على skill تشير إلى مفتاح API مطلوب أن تصرح
بهذا المتطلب في `SKILL.md` حتى يتمكن المستخدمون من رؤيته قبل التثبيت.

تستند نتائج الفحص إلى الآثار. ويُعامل سلوك المزوّد المتوقع، مثل بيانات اعتماد
API المعلنة، أو استدعاءات OAuth إلى localhost، أو تنظيف إلغاء التثبيت محدود
النطاق، أو ترميز Basic Auth، أو تحميلات الملفات التي يختارها المستخدم إلى
المزوّد المعلن، بطريقة مختلفة عن تمرير بيانات الاعتماد المخفي، أو الوصول الواسع
إلى الملفات الخاصة، أو وجهات الشبكة غير ذات الصلة، أو إساءة استخدام المتصفح
بشكل خفي.

انظر [تنسيق Skill](/ar/clawhub/skill-format).

## Plugins

تتضمن إصدارات Plugin بيانات تعريف الحزمة، وإسناد المصدر، وحقول التوافق،
ومعلومات سلامة الآثار.

يتحقق OpenClaw من التوافق قبل تثبيت plugins المستضافة على ClawHub. وقد تعرض
سجلات الحزم أيضًا بيانات تعريف الملخص حتى يتمكن OpenClaw من التحقق من الآثار
المنزلة. يتضمن ClawScan بيانات تعريف env/config الخاصة بالحزمة المعلنة
`openclaw.environment` عند مراجعة إصدارات plugin، بحيث تُقارن متطلبات وقت
التشغيل المعلنة بالسلوك المرصود.

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن Skills، والحزم، والتعليقات.

يجب أن تكون البلاغات محددة وقابلة للتنفيذ. وقد يؤدي إساءة استخدام الإبلاغ بحد
ذاته إلى إجراء على الحساب.

أمثلة على البلاغات:

- بيانات تعريف مضللة
- متطلبات بيانات اعتماد أو أذونات غير معلنة
- تعليمات تثبيت مشبوهة
- تعليقات احتيالية أو انتحال شخصية
- تسجيلات بسوء نية أو إساءة استخدام علامات تجارية
- محتوى ينتهك [الاستخدام المقبول](/ar/clawhub/acceptable-usage)

## ملاحظات ClawScan للناشرين

يمكن للناشرين تقديم ملاحظة ClawScan اختيارية عند نشر skill أو plugin. تمنح هذه
الملاحظة ClawScan سياقًا لسلوك قد يبدو غير معتاد بخلاف ذلك، مثل الوصول إلى
الشبكة، أو الوصول إلى المضيف الأصلي، أو بيانات اعتماد خاصة بالمزوّد.

## حالات احتجاز الإشراف

عندما يعلّم الفاحص الثابت skill مرفوعة على أنها ضارة، يوضع الناشر تلقائيًا تحت
احتجاز إشراف (`requiresModerationAt` معيّن على المستخدم). يؤدي هذا إلى إخفاء
جميع Skills الخاصة بالناشر، ويجعل عمليات النشر المستقبلية تبدأ مخفية، وينشئ
إدخال سجل تدقيق `user.moderation.auto`.

تُحتفظ بالنتائج الثابتة المشبوهة كدليل ملف/سطر للمشرفين، لكنها لا تخفي المحتوى
ولا تقرر نتيجة الفحص العامة بمفردها. تبقى الرفعات الجديدة في حالة
المراجعة/الانتظار حتى تستقر مراجعة LLM. لا يحظر الفحص الثابت فورًا إلا عند
وجود توقيعات ضارة. تبقى إصابات محركات VirusTotal أدلة أمان مرئية، لكن أحكام
VirusTotal Code Insight/Palm استشارية ولا تخفي Skills بمفردها. تحتفظ مراجعات
ClawScan LLM بالملاحظات المتوافقة مع الغرض كإرشادات. تبقى نتائج المراجعة
المتوسطة مرئية على الأثر، بينما يُحجز مرشح الاشتباه للمخاوف عالية التأثير من
LLM، أو النتائج الضارة، أو اكتشافات محركات AV المؤيدة.

يمكن للمسؤولين رفع احتجاز إيجابي كاذب:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

يمسح هذا `requiresModerationAt` و`requiresModerationReason`، ويستعيد Skills
التي أخفاها الاحتجاز على مستوى المستخدم، ويكتب إدخال سجل تدقيق
`user.moderation.lift`. تبقى Skills المخفية لأسباب أخرى، أو التي لا يزال فحصها
الثابت الخاص ضارًا، مخفية.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub إمكانية النشر. وقد تؤدي الإساءة
الشديدة إلى حظر الحسابات، أو إبطال الرموز، أو إخفاء المحتوى، أو إزالة القوائم.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز ClawHub API. إذا
بدأت مصادقة CLI بالفشل بعد إجراء على الحساب، فسجّل الدخول إلى واجهة الويب
لمراجعة حالة الحساب. إذا كان تسجيل الدخول أو وصول CLI العادي محظورًا، فاتصل بـ
security@openclaw.ai لمراجعة الاسترداد.

## إرشادات الناشرين

لتقليل الحالات الإيجابية الكاذبة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء، والملخصات، والوسوم، وسجلات التغييرات
- صرّح بمتغيرات البيئة والأذونات المطلوبة
- أضف ملاحظة ClawScan للناشر عندما يتضمن الإصدار سلوكًا غير معتاد لكنه مقصود
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عند الإمكان
- استخدم التشغيلات الجافة قبل نشر plugins
- رد بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الحزمة
