# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub من خلال GitHub Security Advisories لـ
`openclaw/clawhub`.

استخدم GitHub Security Advisories للثغرات في ClawHub نفسه. تتضمن تقارير
إرشادات ClawHub الجيدة أخطاء في:

- موقع ClawHub أو API أو CLI
- نشر السجل، أو التنزيلات، أو عمليات التثبيت، أو سلامة العناصر الأثرية
- المصادقة، أو التفويض، أو رموز API
- الفحص، أو الإشراف، أو معالجة البلاغات

لا تستخدم إرشادات ClawHub للثغرات الموجودة في الشيفرة المصدرية الخاصة بـ Skills
أو Plugin تابعين لطرف ثالث. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من قائمة ClawHub.

## الإفصاح عن الثغرات

نظرًا لأن ClawHub تطبيق سحابي مستضاف، لا تُفصح ثغرات خدمة ClawHub علنًا
بشكل افتراضي. يُفصح عنها علنًا عندما توجد أدلة على تأثير حقيقي في المستخدمين
أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الحقيقي في المستخدمين الاستغلال المؤكد، أو كشف بيانات
المستخدمين أو الأسرار، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد، أو تحديث البرامج المحلية،
أو اتخاذ إجراء وقائي آخر.

يُفصح علنًا عن الثغرات في البرامج التي يثبتها المستخدمون، مثل حزم ClawHub CLI
أو الثنائيات أو المكتبات أو عناصر الإصدار الأثرية الأخرى التي يحتاج المستخدمون
إلى تحديثها محليًا.

## الصفحات ذات الصلة

للاطلاع على تسميات التدقيق وقت التثبيت، ومستويات المخاطر، والنتائج، والتفسير، راجع
[تدقيقات الأمان](/clawhub/security-audits).

لتقارير السوق، وتعليقات الإشراف، والقوائم المخفية، والحظر، وحالة الحساب،
راجع [الإشراف وسلامة الحساب](/clawhub/moderation).
