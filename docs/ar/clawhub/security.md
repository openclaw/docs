# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub من خلال GitHub Security Advisories الخاصة بـ
`openclaw/clawhub`.

استخدم GitHub Security Advisories للثغرات في ClawHub نفسه. تتضمن تقارير
ClawHub الاستشارية الجيدة أخطاءً في:

- موقع ClawHub أو API أو CLI
- النشر في السجل أو التنزيلات أو عمليات التثبيت أو سلامة القطع الأثرية
- المصادقة أو التفويض أو رموز API
- الفحص أو الإشراف أو التعامل مع التقارير

لا تستخدم تنبيهات ClawHub للثغرات الموجودة في الكود المصدري الخاص بـ skill أو
Plugin تابعين لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من قائمة ClawHub.

## الإفصاح عن الثغرات

لأن ClawHub تطبيق سحابي مستضاف، لا يتم الإفصاح عن ثغرات خدمة ClawHub علنًا
افتراضيًا. يتم الإفصاح عنها علنًا عندما توجد أدلة على تأثير حقيقي في المستخدمين
أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الحقيقي في المستخدمين الاستغلال المؤكد، أو انكشاف بيانات
المستخدمين أو أسرارهم، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد، أو تحديث البرامج المحلية،
أو اتخاذ إجراء وقائي آخر.

يتم الإفصاح علنًا عن الثغرات في البرامج التي يثبتها المستخدمون، مثل حزم
ClawHub CLI أو الثنائيات أو المكتبات أو غيرها من قطع الإصدار الأثرية التي يحتاج
المستخدمون إلى تحديثها محليًا.

## الصفحات ذات الصلة

للاطلاع على تسميات التدقيق وقت التثبيت، ومستويات المخاطر، والنتائج، والتفسير، راجع
[تدقيقات الأمان](/clawhub/security-audits).

للاطلاع على تقارير marketplace، وحجوزات الإشراف، والقوائم المخفية، والحظر، وحالة الحساب،
راجع [الإشراف وسلامة الحساب](/clawhub/moderation).
