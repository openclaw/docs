# الأمان

يمكن الإبلاغ عن مشكلات الأمان في ClawHub عبر إرشادات GitHub الأمنية الخاصة بـ
`openclaw/clawhub`.

استخدم إرشادات GitHub الأمنية للثغرات في ClawHub نفسه. تتضمن تقارير
إرشادات ClawHub الجيدة أخطاء في:

- موقع ClawHub الإلكتروني أو API أو CLI
- نشر السجل أو التنزيلات أو التثبيتات أو سلامة المصنوعات
- المصادقة أو التفويض أو رموز API
- الفحص أو الإشراف أو معالجة البلاغات

لا تستخدم إرشادات ClawHub للثغرات في الشيفرة المصدرية الخاصة بمهارة أو
Plugin تابعين لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من إدراج ClawHub.

## الإفصاح عن الثغرات الأمنية

لأن ClawHub تطبيق سحابي مستضاف، لا تُفصح ثغرات خدمة ClawHub علنًا
افتراضيًا. يُفصح عنها علنًا عند وجود دليل على تأثير حقيقي في المستخدمين أو
عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الحقيقي في المستخدمين الاستغلال المؤكد، أو كشف بيانات
المستخدمين أو أسرارهم، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد أو تحديث البرامج
المحلية أو اتخاذ إجراء وقائي آخر.

يُفصح علنًا عن الثغرات في البرامج التي يثبتها المستخدمون، مثل حزم ClawHub
CLI أو الثنائيات أو المكتبات أو مصنوعات الإصدار الأخرى التي يحتاج المستخدمون
إلى تحديثها محليًا.
