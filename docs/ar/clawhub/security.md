# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub عبر GitHub Security Advisories الخاصة بـ
`openclaw/clawhub`.

استخدم GitHub Security Advisories للثغرات في ClawHub نفسه. تتضمن تقارير
إرشادات ClawHub الجيدة أخطاء في:

- موقع ClawHub أو API أو CLI
- نشر السجل، أو التنزيلات، أو عمليات التثبيت، أو سلامة الآثار
- المصادقة، أو التخويل، أو رموز API
- الفحص، أو الإشراف، أو معالجة البلاغات

لا تستخدم إرشادات ClawHub للثغرات في التعليمات البرمجية المصدرية الخاصة بـ skill أو
plugin تابع لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من إدراج ClawHub.

## الإفصاح عن الثغرات

لأن ClawHub تطبيق سحابي مستضاف، لا يُفصح عن ثغرات خدمة ClawHub
علنا افتراضيا. يُفصح عنها علنا عند وجود دليل على أثر حقيقي على المستخدمين
أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة الأثر الحقيقي على المستخدمين الاستغلال المؤكد، أو انكشاف بيانات
المستخدمين أو أسرارهم، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد، أو تحديث البرامج المحلية، أو
اتخاذ إجراء وقائي آخر.

يُفصح علنا عن الثغرات في البرامج المثبتة من قبل المستخدم، مثل
حزم ClawHub CLI، أو الملفات الثنائية، أو المكتبات، أو آثار الإصدار الأخرى التي يحتاج المستخدمون
إلى تحديثها محليا.

## صفحات ذات صلة

للاطلاع على تسميات التدقيق وقت التثبيت، ومستويات المخاطر، والنتائج، والتفسير، راجع
[تدقيقات الأمان](/ar/clawhub/security-audits).

للاطلاع على بلاغات السوق، وتعليقات الإشراف، والإدراجات المخفية، والحظر، وحالة الحساب،
راجع [الإشراف وسلامة الحساب](/ar/clawhub/moderation).
