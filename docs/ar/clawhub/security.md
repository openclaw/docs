# الأمان

يمكن الإبلاغ عن مشكلات أمان ClawHub عبر إشعارات GitHub الأمنية الخاصة بـ
`openclaw/clawhub`.

استخدم إشعارات GitHub الأمنية للثغرات في ClawHub نفسه. تتضمن تقارير
إشعارات ClawHub الجيدة أخطاءً في:

- موقع ClawHub أو API أو CLI
- نشر السجل أو التنزيلات أو عمليات التثبيت أو سلامة الأثر
- المصادقة أو التفويض أو رموز API
- الفحص أو الإشراف أو معالجة البلاغات

لا تستخدم إشعارات ClawHub للثغرات في الشيفرة المصدرية الخاصة بمهارة تابعة
لجهة خارجية أو Plugin تابع لها. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من قائمة ClawHub.

## الإفصاح عن الثغرات

نظرًا إلى أن ClawHub تطبيق سحابي مستضاف، لا يُفصح عن ثغرات خدمة ClawHub
علناً بشكل افتراضي. يُفصح عنها علناً عندما توجد أدلة على تأثير حقيقي في
المستخدمين أو عندما يحتاج المستخدمون إلى اتخاذ إجراء.

تشمل أمثلة التأثير الحقيقي في المستخدمين الاستغلال المؤكد، أو كشف بيانات
المستخدمين أو أسرارهم، أو وصول محتوى ضار إلى المستخدمين بسبب فشل في المنصة،
أو أي مشكلة تتطلب من المستخدمين تدوير بيانات الاعتماد، أو تحديث البرامج
المحلية، أو اتخاذ إجراء وقائي آخر.

يُفصح علناً عن الثغرات في البرامج التي يثبتها المستخدمون، مثل حزم ClawHub CLI
أو الثنائيات أو المكتبات أو آثار الإصدارات الأخرى التي يحتاج المستخدمون إلى
تحديثها محلياً.

## صفحات ذات صلة

للاطلاع على تسميات التدقيق وقت التثبيت، ومستويات المخاطر، والنتائج، وكيفية
تفسيرها، راجع [تدقيقات الأمان](/clawhub/security-audits).

للاطلاع على بلاغات السوق، وعمليات التعليق بسبب الإشراف، والقوائم المخفية،
والحظر، وحالة الحساب، راجع [الإشراف وسلامة الحساب](/clawhub/moderation).
