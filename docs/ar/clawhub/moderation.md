# الإشراف وسلامة الحسابات

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف والتثبيت العامة لا تزال
تحتاج إلى ضوابط وقائية. تساعد البلاغات، وتعليقات الإشراف المؤقتة، والقوائم المخفية، وإجراءات الحسابات
على حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن أو مضللا أو خارجا
عن السياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. بالنسبة إلى تسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/clawhub/security-audits).

راجع أيضا [الأمان](/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). بالنسبة إلى مخاوف حقوق النشر أو غيرها من
مخاوف حقوق المحتوى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين الذين سجلوا الدخول الإبلاغ عن Skills وplugins والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- قوائم ضارة
- بيانات وصفية مضللة
- بيانات اعتماد أو متطلبات أذونات غير مصرح بها
- تعليمات تثبيت مريبة
- انتحال الشخصية
- تسجيلات بسوء نية أو إساءة استخدام للعلامات التجارية
- محتوى ينتهك [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن Skill** في صفحة Skill، أو أمر/واجهة API الإبلاغ عن الحزم
للحزم.

لا تستخدم بلاغات ClawHub للثغرات في التعليمات البرمجية المصدرية الخاصة بـ Skill أو
plugin تابع لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من القائمة. لا يصون ClawHub تعليمات Skill أو plugin البرمجية التابعة
لجهات خارجية ولا يرقعها.

تخص GitHub Security Advisories لـ `openclaw/clawhub` الثغرات في
ClawHub نفسه. تشمل الأمثلة الأخطاء في الموقع الإلكتروني أو API أو CLI أو السجل أو المصادقة أو
الفحص أو الإشراف أو حدود الثقة الخاصة بالتنزيل/التثبيت. لا تستخدم
استشارات ClawHub للثغرات في Skills أو plugins التابعة لجهات خارجية.

البلاغات الجيدة محددة وقابلة للتنفيذ. يمكن أن تؤدي إساءة استخدام الإبلاغ نفسها إلى
إجراء على الحساب.

## مطالبات المؤسسة والنطاق الاسمي

ينبغي لنزاعات ملكية المؤسسة أو العلامة التجارية أو نطاق الحزمة أو مقبض المالك أو النطاق الاسمي
استخدام عملية [مطالبات المؤسسة والنطاق الاسمي](/clawhub/namespace-claims)، لا
مسار الإبلاغ داخل المنتج ولا نموذج استئناف الحساب.

استخدم هذه العملية عندما تحتاج إلى أن يراجع فريق ClawHub دليلا غير حساس على أن
نطاقا اسميا ينبغي حجزه أو نقله أو إعادة تسميته أو إخفاؤه أو عزله أو تعيين اسم مستعار له
أو مراجعته بطريقة أخرى. لا تدرج الأسرار أو المستندات الخاصة أو الملفات القانونية الخاصة
أو مستندات الهوية الشخصية أو رموز API أو رموز تحدي DNS في
مسألة عامة.

## تعليقات الإشراف المؤقتة

يمكن لبعض النتائج الشديدة أو مشكلات السياسة أن تضع ناشرا أو قائمة تحت
تعليق إشراف مؤقت. عندما يحدث ذلك، قد يخفى المحتوى المتأثر من
الاكتشاف العام، أو قد تبدأ عمليات النشر المستقبلية مخفية حتى تتم مراجعة المشكلة.

تهدف تعليقات الإشراف المؤقتة إلى حماية المستخدمين بينما يحل ClawHub الحالات عالية المخاطر.
ويمكن أيضا رفعها عند تأكيد نتيجة إيجابية خاطئة.

## القوائم المخفية أو المحظورة

قد تكون القائمة معلقة أو مخفية أو معزولة أو ملغاة أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبت الإصدار إلا إذا حل المالك
المشكلة أو أعاده الإشراف.

قد يظل المالكون قادرين على رؤية التشخيصات لقوائمهم المعلقة أو المخفية. تساعد هذه
التشخيصات في شرح ما حدث وما الذي يجب تغييره قبل أن
تعود القائمة إلى الأسطح العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub إمكانية النشر. يمكن أن يؤدي سوء الاستخدام الشديد إلى
حظر الحسابات أو إبطال الرموز أو إخفاء المحتوى أو إزالة القوائم.
تُفحص إشارات ضغط إساءة استخدام الناشرين يوميا. يمكن للإشارات التي تصل إلى
عتبة الحظر المحتمل في ClawHub أن تطلق تحذيرا تلقائيا. إذا كان الفحص
المؤهل التالي بعد الموعد النهائي للتحذير لا يزال يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيا.
تبقى إشارات المراجعة الزمنية المحدودة ومنخفضة الثقة خارج
الإنفاذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـ ClawHub. إذا بدأت مصادقة CLI
بالفشل بعد إجراء على الحساب، فسجل الدخول إلى واجهة الويب لمراجعة
حالة الحساب. إذا كان تسجيل الدخول أو وصول CLI العادي محظورا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا سمى بريد إلكتروني شغله ماسح ضوئي إصدار Skill أو plugin على أنه ضار،
فنزل نتائج الفحص المخزنة للإصدار المقدم المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى plugins، أضف
`--kind plugin`. راجع مخرجات الفحص، وأصلح القائمة، وزد رقم الإصدار،
وارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الخاطئة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغيير
- صرح بمتغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنا
- استخدم التشغيل التجريبي قبل نشر plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
