# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف العام والتثبيت لا تزال
تحتاج إلى ضوابط حماية. تساعد البلاغات، وتعليقات الإشراف، والقوائم المخفية، وإجراءات الحساب
على حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن، أو مضللاً، أو خارج
السياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. لتسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى الخطر، راجع
[عمليات التدقيق الأمني](/clawhub/security-audits).

راجع أيضًا [الأمان](/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). لمخاوف حقوق النشر أو غيرها من حقوق
المحتوى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن Skills وplugins والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الخبيثة
- البيانات الوصفية المضللة
- بيانات الاعتماد أو متطلبات الأذونات غير المصرح بها
- تعليمات التثبيت المشبوهة
- انتحال الهوية
- التسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن Skill** في صفحة Skill، أو أمر/API الإبلاغ عن الحزم
للحزم.

لا تستخدم بلاغات ClawHub للثغرات في الشيفرة المصدرية الخاصة بSkill أو
plugin تابع لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من القائمة. لا يصون ClawHub شيفرة Skills أو plugins التابعة
لجهات خارجية ولا يرقعها.

GitHub Security Advisories الخاصة ب`openclaw/clawhub` مخصصة للثغرات في
ClawHub نفسه. تشمل الأمثلة الأخطاء في الموقع الإلكتروني، أو API، أو CLI، أو السجل، أو المصادقة،
أو الفحص، أو الإشراف، أو حدود الثقة للتنزيل/التثبيت. لا تستخدم
استشارات ClawHub للثغرات في Skills أو plugins تابعة لجهات خارجية.

البلاغات الجيدة محددة وقابلة للتنفيذ. وقد يؤدي إساءة استخدام الإبلاغ نفسها إلى
إجراء على الحساب.

## مطالبات المؤسسة والنطاق الاسمي

ينبغي استخدام عملية [مطالبات المؤسسة والنطاق الاسمي](/clawhub/namespace-claims) في
نزاعات ملكية المؤسسة، أو العلامة التجارية، أو نطاق الحزمة، أو معرّف المالك، أو النطاق الاسمي، وليس
تدفق البلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم هذه العملية عندما تحتاج إلى أن يراجع فريق ClawHub دليلاً غير حساس على أن
نطاقًا اسميًا ينبغي حجزه، أو نقله، أو إعادة تسميته، أو إخفاؤه، أو عزله، أو تسميته باسم مستعار،
أو مراجعته بطريقة أخرى. لا تُضمّن أسرارًا، أو مستندات خاصة، أو ملفات قانونية خاصة،
أو وثائق هوية شخصية، أو رموز API، أو رموز تحدي DNS في
مشكلة عامة.

## تعليقات الإشراف

يمكن لبعض النتائج الشديدة أو مشكلات السياسة أن تضع ناشرًا أو قائمة تحت
تعليق إشرافي. عند حدوث ذلك، قد يُخفى المحتوى المتأثر من
الاكتشاف العام أو قد تبدأ عمليات النشر المستقبلية مخفية حتى تتم مراجعة المشكلة.

تهدف تعليقات الإشراف إلى حماية المستخدمين بينما يحل ClawHub الحالات عالية الخطر.
ويمكن أيضًا رفعها عندما يتم تأكيد نتيجة إيجابية خاطئة.

## القوائم المخفية أو المحظورة

قد تكون القائمة معلقة، أو مخفية، أو معزولة، أو ملغاة، أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبّت الإصدار إلا إذا حل المالك
المشكلة أو أعاده الإشراف.

قد يظل المالكون يرون التشخيصات لقوائمهم المعلقة أو المخفية. تساعد هذه
التشخيصات في شرح ما حدث وما الذي يجب تغييره قبل أن تتمكن
القائمة من العودة إلى الأسطح العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub إمكانية النشر. وقد تؤدي الإساءة الشديدة إلى
حظر الحساب، أو إبطال الرموز، أو إخفاء المحتوى، أو إزالة القوائم.
تُفحص إشارات ضغط إساءة الناشر يوميًا. يمكن للإشارات التي تصل إلى
عتبة الحظر المحتمل لدى ClawHub أن تطلق تحذيرًا تلقائيًا. إذا كان الفحص
المؤهل التالي بعد موعد التحذير لا يزال يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيًا.
تبقى إشارات المراجعة الأقل ثقة والمحدودة زمنيًا خارج
التنفيذ التلقائي.

لا يمكن للحسابات المحذوفة، أو المحظورة، أو المعطلة استخدام رموز ClawHub API. إذا بدأت مصادقة CLI
تفشل بعد إجراء على الحساب، فسجّل الدخول إلى واجهة الويب لمراجعة حالة
الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محظورًا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا سمّت رسالة بريد إلكتروني أطلقها ماسح ضوئي إصدار Skill أو plugin على أنه خبيث،
فنزل نتائج الفحص المخزنة للإصدار المقدم المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى plugins، أضف
`--kind plugin`. راجع مخرجات الفحص، وأصلح القائمة، وزد رقم الإصدار،
وارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الخاطئة وتحسين ثقة المستخدمين:

- أبقِ الأسماء والملخصات والوسوم وسجلات التغيير دقيقة
- صرّح بمتغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنًا
- استخدم التشغيلات الجافة قبل نشر plugins
- رد بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
