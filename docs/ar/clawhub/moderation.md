# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف والتثبيت العامة لا تزال
تحتاج إلى ضوابط حماية. تساعد البلاغات، وعمليات تعليق الإشراف، والقوائم المخفية، وإجراءات الحساب
على حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن، أو مضللا، أو خارج
السياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. بالنسبة لتسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/clawhub/security-audits).

راجع أيضا [الأمان](/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). بالنسبة لمخاوف حقوق النشر أو حقوق المحتوى
الأخرى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن Skills وPlugins والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الخبيثة
- البيانات الوصفية المضللة
- بيانات الاعتماد أو متطلبات الأذونات غير المعلنة
- تعليمات التثبيت المشبوهة
- انتحال الهوية
- التسجيلات سيئة النية أو إساءة استخدام العلامات التجارية
- المحتوى الذي يخالف [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن Skill** في صفحة Skill، أو أمر/API الإبلاغ عن الحزم
بالنسبة للحزم.

لا تستخدم بلاغات ClawHub للثغرات الأمنية في الكود المصدري الخاص بجهة خارجية
لـ Skill أو Plugin. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من القائمة. لا يصون ClawHub كود Skills أو Plugins
التابعة لجهات خارجية ولا يرقعه.

تخص GitHub Security Advisories لـ `openclaw/clawhub` الثغرات الأمنية في
ClawHub نفسه. تشمل الأمثلة الأخطاء في الموقع الإلكتروني، أو API، أو CLI، أو السجل، أو المصادقة،
أو الفحص، أو الإشراف، أو حدود الثقة الخاصة بالتنزيل/التثبيت. لا تستخدم تنبيهات ClawHub
للثغرات الأمنية في Skills أو Plugins التابعة لجهات خارجية.

البلاغات الجيدة محددة وقابلة للتنفيذ. وقد يؤدي إساءة استخدام الإبلاغ بحد ذاته إلى
إجراء على الحساب.

## مطالبات المؤسسات والنطاقات الاسمية

ينبغي أن تستخدم نزاعات ملكية المؤسسة، أو العلامة التجارية، أو نطاق الحزمة، أو معرّف المالك، أو النطاق الاسمي
عملية [مطالبات المؤسسات والنطاقات الاسمية](/clawhub/namespace-claims)، وليس
تدفق الإبلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم تلك العملية عندما تحتاج إلى أن يراجع موظفو ClawHub دليلا غير حساس على أن
النطاق الاسمي ينبغي حجزه، أو نقله، أو إعادة تسميته، أو إخفاؤه، أو عزله، أو جعله اسما بديلا،
أو مراجعته بطريقة أخرى. لا تدرج أسرارا، أو مستندات خاصة، أو ملفات قانونية خاصة،
أو وثائق هوية شخصية، أو رموز API، أو رموز تحدي DNS في
مشكلة عامة.

## عمليات تعليق الإشراف

يمكن لبعض النتائج الشديدة أو مشكلات السياسة أن تضع الناشر أو القائمة تحت
تعليق إشراف. عند حدوث ذلك، قد يخفى المحتوى المتأثر من الاكتشاف العام
أو قد تبدأ عمليات النشر المستقبلية مخفية حتى تتم مراجعة المشكلة.

تهدف عمليات تعليق الإشراف إلى حماية المستخدمين بينما يحل ClawHub الحالات عالية المخاطر.
ويمكن رفعها أيضا عند تأكيد وجود نتيجة إيجابية كاذبة.

## القوائم المخفية أو المحظورة

قد تكون القائمة معلقة، أو مخفية، أو معزولة، أو ملغاة، أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبت الإصدار إلا إذا حل المالك
المشكلة أو أعاده الإشراف.

قد يظل المالكون قادرين على رؤية التشخيصات الخاصة بقوائمهم المعلقة أو المخفية. تساعد هذه
التشخيصات على شرح ما حدث وما الذي يجب تغييره قبل أن
تعود القائمة إلى الأسطح العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تخالف سياسة ClawHub صلاحية النشر. ويمكن أن تؤدي الإساءة الشديدة إلى
حظر الحساب، أو إلغاء الرموز، أو إخفاء المحتوى، أو إزالة القوائم.
تتحقق إشارات ضغط إساءة الناشرين يوميا. يمكن للإشارات التي تبلغ
عتبة الحظر المحتمل في ClawHub أن تطلق تحذيرا تلقائيا. إذا كان الفحص
المؤهل التالي بعد موعد التحذير لا يزال يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيا.
تبقى إشارات المراجعة ذات الثقة الأقل والمحددة زمنيا خارج
الإنفاذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـ ClawHub. إذا بدأت مصادقة CLI
بالفشل بعد إجراء على الحساب، فسجل الدخول إلى واجهة الويب لمراجعة
حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محظورا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا سمى بريد إلكتروني أطلقه الفاحص إصدار Skill أو Plugin بأنه خبيث،
فنزل نتائج الفحص المخزنة للإصدار المقدم المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى Plugins، أضف
`--kind plugin`. راجع مخرجات الفحص، وأصلح القائمة، وارفع رقم الإصدار،
وارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل الإيجابيات الكاذبة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغييرات
- أعلن متغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنا
- استخدم التشغيلات التجريبية قبل نشر Plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
