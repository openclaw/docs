# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن واجهات الاكتشاف والتثبيت العامة ما زالت
تحتاج إلى ضوابط حماية. تساعد البلاغات، وقيود الإشراف، والقوائم المخفية، وإجراءات الحساب
على حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن، أو مضللا، أو مخالفا
للسياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. لتسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/clawhub/security-audits).

راجع أيضا [الأمان](/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). لمخاوف حقوق النشر أو غيرها من
حقوق المحتوى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين دخولهم الإبلاغ عن Skills وplugins والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الخبيثة
- البيانات الوصفية المضللة
- بيانات الاعتماد أو متطلبات الأذونات غير المصرح بها
- تعليمات التثبيت المريبة
- انتحال الهوية
- التسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن Skill** في صفحة Skill، أو أمر/API
الإبلاغ عن الحزم للحزم.

لا تستخدم بلاغات ClawHub للثغرات الأمنية في كود المصدر الخاص بـ Skill أو
Plugin تابع لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من القائمة. لا يصون ClawHub كود Skills أو plugins التابعة لجهات خارجية
ولا يطبّق عليه تصحيحات.

إرشادات أمان GitHub الخاصة بـ `openclaw/clawhub` مخصصة للثغرات الأمنية في
ClawHub نفسه. تشمل الأمثلة الأخطاء في الموقع الإلكتروني، أو API، أو CLI، أو السجل، أو المصادقة،
أو الفحص، أو الإشراف، أو حدود الثقة للتنزيل/التثبيت. لا تستخدم إرشادات ClawHub
للثغرات الأمنية في Skills أو plugins التابعة لجهات خارجية.

البلاغات الجيدة محددة وقابلة للتنفيذ. وقد يؤدي إساءة استخدام الإبلاغ بحد ذاتها إلى
إجراء على الحساب.

## مطالبات المؤسسات ومساحات الأسماء

ينبغي استخدام عملية [مطالبات المؤسسات ومساحات الأسماء](/clawhub/namespace-claims) في
نزاعات ملكية المؤسسة، أو العلامة التجارية، أو نطاق الحزمة، أو معرّف المالك، أو مساحة الاسم، وليس
مسار الإبلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم تلك العملية عندما تحتاج إلى أن يراجع فريق ClawHub إثباتا غير حساس بأن
مساحة اسم ينبغي حجزها، أو نقلها، أو إعادة تسميتها، أو إخفاؤها، أو عزلها، أو ربطها باسم بديل،
أو مراجعتها بطريقة أخرى. لا تضمّن أسرارا، أو مستندات خاصة، أو ملفات قانونية خاصة،
أو مستندات هوية شخصية، أو رموز API، أو رموز تحدي DNS في
مشكلة عامة.

## قيود الإشراف

قد تؤدي بعض النتائج الخطيرة أو مشكلات السياسة إلى وضع ناشر أو قائمة تحت
قيد إشراف. عند حدوث ذلك، قد يُخفى المحتوى المتأثر من الاكتشاف العام
أو قد تبدأ عمليات النشر المستقبلية مخفية إلى أن تتم مراجعة المشكلة.

تهدف قيود الإشراف إلى حماية المستخدمين بينما يعالج ClawHub الحالات عالية المخاطر.
ويمكن رفعها أيضا عند تأكيد وجود نتيجة إيجابية كاذبة.

## القوائم المخفية أو المحظورة

قد تكون القائمة معلقة، أو مخفية، أو معزولة، أو ملغاة، أو غير متاحة بطريقة أخرى على
واجهات التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبّت الإصدار إلا إذا حلّ المالك
المشكلة أو أعاد الإشراف إتاحته.

قد يظل بإمكان المالكين رؤية التشخيصات لقوائمهم المعلّقة أو المخفية. تساعد هذه
التشخيصات في شرح ما حدث وما يجب تغييره قبل أن
تعود القائمة إلى الواجهات العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub صلاحية النشر. ويمكن أن تؤدي الإساءة الشديدة إلى
حظر الحسابات، أو إبطال الرموز، أو إخفاء المحتوى، أو إزالة القوائم.
تُفحص إشارات ضغط إساءة الناشرين يوميا. يمكن للإشارات التي تصل إلى
عتبة الحظر المحتمل في ClawHub أن تطلق تحذيرا تلقائيا. إذا ظل الفحص
المؤهل التالي بعد مهلة التحذير يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيا.
وتبقى إشارات المراجعة الزمنية منخفضة الثقة والمحدودة خارج
الإنفاذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـ ClawHub. إذا بدأت مصادقة CLI
بالفشل بعد إجراء على الحساب، فسجّل الدخول إلى واجهة الويب لمراجعة
حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محظورا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا سمّت رسالة بريد إلكتروني مفعّلة بواسطة ماسح إصدار Skill أو Plugin بأنه خبيث،
فنزل نتائج الفحص المخزنة للإصدار المرسل المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى plugins، أضف
`--kind plugin`. راجع مخرجات الفحص، وأصلح القائمة، وزد رقم الإصدار،
وارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الكاذبة وتحسين ثقة المستخدمين:

- أبقِ الأسماء والملخصات والوسوم وسجلات التغيير دقيقة
- صرّح بمتغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنا
- استخدم التشغيلات التجريبية قبل نشر plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
