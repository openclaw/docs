# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف العام والتثبيت لا تزال
تحتاج إلى ضوابط حماية. تساعد البلاغات، وقيود الإشراف، والقوائم المخفية، وإجراءات الحساب
في حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن أو مضللًا أو خارجًا
عن السياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. للاطلاع على تسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/clawhub/security-audits).

راجع أيضًا [الأمان](/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). بالنسبة إلى مخاوف حقوق النشر أو غيرها من
حقوق المحتوى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الدخول الإبلاغ عن المهارات والإضافات والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الخبيثة
- البيانات الوصفية المضللة
- بيانات الاعتماد غير المعلنة أو متطلبات الأذونات
- تعليمات التثبيت المشبوهة
- انتحال الهوية
- التسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن المهارة** في صفحة المهارة، أو أمر/API الإبلاغ عن الحزم
للحزم.

لا تستخدم بلاغات ClawHub للثغرات في الشيفرة المصدرية الخاصة بمهارة أو
إضافة تابعة لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من القائمة. لا يصون ClawHub شيفرة المهارات أو الإضافات التابعة
لجهات خارجية ولا يرقعها.

إرشادات GitHub الأمنية لـ `openclaw/clawhub` مخصصة للثغرات في
ClawHub نفسه. تشمل الأمثلة العلل في الموقع الإلكتروني أو API أو CLI أو السجل أو المصادقة أو
الفحص أو الإشراف أو حدود الثقة للتنزيل/التثبيت. لا تستخدم إرشادات ClawHub
للثغرات في المهارات أو الإضافات التابعة لجهات خارجية.

البلاغات الجيدة محددة وقابلة للتنفيذ. إساءة استخدام الإبلاغ قد تؤدي بحد ذاتها إلى
إجراء على الحساب.

## مطالبات المؤسسات والنطاقات الاسمية

ينبغي لنزاعات ملكية المؤسسة أو العلامة التجارية أو نطاق الحزم أو معرف المالك أو النطاق الاسمي
استخدام عملية [مطالبات المؤسسات والنطاقات الاسمية](/clawhub/namespace-claims)، وليس
تدفق الإبلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم هذه العملية عندما تحتاج إلى أن يراجع موظفو ClawHub دليلًا غير حساس على أن
النطاق الاسمي يجب حجزه أو نقله أو إعادة تسميته أو إخفاؤه أو عزله أو إنشاء اسم مستعار له
أو مراجعته بطريقة أخرى. لا تدرج أسرارًا أو مستندات خاصة أو ملفات قانونية خاصة
أو مستندات هوية شخصية أو رموز API أو رموز تحدي DNS في
مسألة عامة.

## قيود الإشراف

قد تؤدي بعض النتائج الخطيرة أو مشكلات السياسة إلى وضع الناشر أو القائمة تحت
قيد إشراف. عند حدوث ذلك، قد يخفى المحتوى المتأثر من
الاكتشاف العام، أو قد تبدأ عمليات النشر المستقبلية مخفية حتى تتم مراجعة المشكلة.

تهدف قيود الإشراف إلى حماية المستخدمين بينما يعالج ClawHub الحالات عالية المخاطر.
ويمكن رفعها أيضًا عند تأكيد وجود نتيجة إيجابية كاذبة.

## القوائم المخفية أو المحظورة

قد تكون القائمة معلقة أو مخفية أو معزولة أو ملغاة أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبت الإصدار إلا إذا حل المالك
المشكلة أو أعاده الإشراف.

قد يظل المالكون قادرين على رؤية التشخيصات لقوائمهم المعلقة أو المخفية. تساعد هذه
التشخيصات في شرح ما حدث وما الذي يجب تغييره قبل أن
تعود القائمة إلى الأسطح العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub إمكانية النشر. يمكن أن تؤدي الإساءة الشديدة إلى
حظر الحسابات أو إلغاء الرموز أو إخفاء المحتوى أو إزالة القوائم.
تُفحص إشارات ضغط إساءة استخدام الناشرين يوميًا. يمكن للإشارات التي تصل إلى
عتبة الحظر المحتمل في ClawHub أن تطلق تحذيرًا تلقائيًا. إذا ظل الفحص
المؤهل التالي بعد مهلة التحذير يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيًا.
تبقى إشارات المراجعة منخفضة الثقة والمحدودة زمنيًا خارج
الإنفاذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـ ClawHub. إذا بدأت مصادقة CLI
بالفشل بعد إجراء على الحساب، فسجل الدخول إلى واجهة الويب لمراجعة
حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محجوبًا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا سمّت رسالة بريد إلكتروني ناتجة عن الفاحص إصدار مهارة أو إضافة على أنه خبيث،
فنزل نتائج الفحص المخزنة للإصدار المرسل المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى الإضافات، أضف
`--kind plugin`. راجع مخرجات الفحص، وأصلح القائمة، وزد رقم الإصدار،
ثم ارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الكاذبة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغييرات
- أعلن متغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنًا
- استخدم التشغيلات التجريبية قبل نشر الإضافات
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
