# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف العام والتثبيت ما زالت
تحتاج إلى ضوابط حماية. تساعد البلاغات، وتعليقات الإشراف، والإدراجات المخفية، وإجراءات الحساب
في حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن، أو مضللا، أو خارجا
عن السياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. لتسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/ar/clawhub/security-audits).

راجع أيضا [الأمان](/ar/clawhub/security) و
[الاستخدام المقبول](/ar/clawhub/acceptable-usage). بالنسبة لمخاوف حقوق النشر أو حقوق المحتوى
الأخرى، استخدم [طلبات حقوق المحتوى](/ar/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن Skills وPlugins والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- إدراجات خبيثة
- بيانات وصفية مضللة
- بيانات اعتماد أو متطلبات أذونات غير معلنة
- تعليمات تثبيت مريبة
- انتحال الهوية
- تسجيلات بسوء نية أو إساءة استخدام للعلامات التجارية
- محتوى ينتهك [الاستخدام المقبول](/ar/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن Skill** في صفحة Skill، أو أمر/API الإبلاغ عن الحزم
للحزم.

لا تستخدم بلاغات ClawHub للثغرات في الكود المصدري الخاص بـ Skill أو
Plugin تابع لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من الإدراج. لا يصون ClawHub كود Skill أو Plugin
التابع لجهة خارجية ولا يصححه.

إرشادات أمان GitHub الخاصة بـ `openclaw/clawhub` مخصصة للثغرات في
ClawHub نفسه. تشمل الأمثلة أخطاء في الموقع الإلكتروني، أو API، أو CLI، أو السجل، أو المصادقة،
أو الفحص، أو الإشراف، أو حدود الثقة الخاصة بالتنزيل/التثبيت. لا تستخدم إرشادات ClawHub
للثغرات في Skills أو Plugins تابعة لجهات خارجية.

البلاغات الجيدة محددة وقابلة للتنفيذ. يمكن أن يؤدي إساءة استخدام الإبلاغ بحد ذاتها إلى
إجراء على الحساب.

## مطالبات المؤسسات ومساحات الأسماء

ينبغي أن تستخدم نزاعات ملكية المؤسسة أو العلامة التجارية أو نطاق الحزمة أو معرف المالك أو مساحة الأسماء
عملية [مطالبات المؤسسات ومساحات الأسماء](/ar/clawhub/namespace-claims)، وليس
مسار الإبلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم هذه العملية عندما تحتاج إلى أن يراجع فريق ClawHub دليلا غير حساس على أن
مساحة أسماء يجب حجزها، أو نقلها، أو إعادة تسميتها، أو إخفاؤها، أو عزلها، أو جعلها اسما بديلا،
أو مراجعتها بطريقة أخرى. لا تدرج أسرارا، أو مستندات خاصة، أو ملفات قانونية خاصة،
أو وثائق هوية شخصية، أو رموز API، أو رموز تحدي DNS في
مشكلة عامة.

## تعليقات الإشراف

يمكن لبعض النتائج الخطيرة أو مشكلات السياسة أن تضع ناشرا أو إدراجا تحت
تعليق إشراف. عند حدوث ذلك، قد يخفى المحتوى المتأثر من
الاكتشاف العام أو قد تبدأ المنشورات المستقبلية مخفية إلى أن تتم مراجعة المشكلة.

تهدف تعليقات الإشراف إلى حماية المستخدمين بينما يحل ClawHub الحالات عالية المخاطر.
كما يمكن رفعها عند تأكيد نتيجة إيجابية كاذبة.

## الإدراجات المخفية أو المحظورة

قد يكون الإدراج معلقا، أو مخفيا، أو معزولا، أو ملغى، أو غير متاح بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبت الإصدار ما لم يحل المالك
المشكلة أو يعيده الإشراف.

قد يظل بإمكان المالكين رؤية التشخيصات الخاصة بإدراجاتهم المعلقة أو المخفية. تساعد هذه
التشخيصات في شرح ما حدث وما الذي يجب تغييره قبل أن يتمكن
الإدراج من العودة إلى الأسطح العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub إمكانية النشر. يمكن أن يؤدي إساءة الاستخدام الشديدة إلى
حظر الحساب، أو إبطال الرموز، أو إخفاء المحتوى، أو إزالة الإدراجات.
تراجع إشارات ضغط إساءة استخدام الناشرين يوميا. يمكن للإشارات التي تصل إلى
عتبة الحظر المحتمل في ClawHub أن تؤدي إلى تحذير تلقائي. إذا كان الفحص
المؤهل التالي بعد الموعد النهائي للتحذير ما زال يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيا.
تبقى إشارات المراجعة الأقل ثقة والمحدودة زمنيا خارج
التنفيذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـ ClawHub. إذا بدأت مصادقة CLI
بالفشل بعد إجراء على الحساب، فسجل الدخول إلى واجهة الويب لمراجعة
حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محظورا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا سمى بريد إلكتروني ناتج عن ماسح ضوئي إصدار Skill أو Plugin على أنه خبيث،
فنزل نتائج الفحص المخزنة للإصدار المقدم المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى Plugins، أضف
`--kind plugin`. راجع ناتج الفحص، وأصلح الإدراج، وزد رقم الإصدار،
وارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الكاذبة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغيير
- أعلن متغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنا
- استخدم التشغيلات التجريبية قبل نشر Plugins
- أجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
