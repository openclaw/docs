# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف والتثبيت العامة لا تزال
تحتاج إلى ضوابط حماية. تساعد البلاغات، وتعليقات الإشراف، والقوائم المخفية،
وإجراءات الحساب على حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن أو
مضللًا أو خارجًا عن السياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. للاطلاع على تسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[عمليات التدقيق الأمني](/clawhub/security-audits).

راجع أيضًا [الأمان](/ar/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). ولمخاوف حقوق النشر أو حقوق
المحتوى الأخرى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الدخول الإبلاغ عن المهارات والمكونات الإضافية والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الخبيثة
- البيانات الوصفية المضللة
- بيانات الاعتماد أو متطلبات الأذونات غير المعلنة
- تعليمات التثبيت المشبوهة
- انتحال الهوية
- التسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- المحتوى الذي يخالف [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن المهارة** في صفحة المهارة، أو أمر/API الإبلاغ عن الحزم
للحزم.

لا تستخدم بلاغات ClawHub للثغرات في كود المصدر الخاص بمهارة أو مكون إضافي
تابع لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر المرتبط من
القائمة. لا يصون ClawHub كود المهارات أو المكونات الإضافية التابعة لجهات
خارجية ولا يرقّعه.

تُستخدم استشارات GitHub الأمنية الخاصة بـ `openclaw/clawhub` للثغرات في
ClawHub نفسه. تشمل الأمثلة العلل في الموقع الإلكتروني، أو API، أو CLI، أو
السجل، أو المصادقة، أو الفحص، أو الإشراف، أو حدود الثقة الخاصة بالتنزيل/التثبيت.
لا تستخدم استشارات ClawHub للثغرات في المهارات أو المكونات الإضافية التابعة
لجهات خارجية.

البلاغات الجيدة محددة وقابلة للتنفيذ. وقد يؤدي إساءة استخدام الإبلاغ بحد ذاته
إلى إجراء على الحساب.

## مطالبات المؤسسات والنطاقات

يجب أن تستخدم نزاعات ملكية المؤسسة أو العلامة التجارية أو نطاق الحزمة أو معرّف
المالك أو النطاق عملية [مطالبات المؤسسات والنطاقات](/clawhub/namespace-claims)،
وليس مسار البلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم هذه العملية عندما تحتاج إلى أن يراجع فريق ClawHub دليلًا غير حساس على
أن نطاقًا يجب حجزه أو نقله أو إعادة تسميته أو إخفاؤه أو عزله أو جعله اسمًا
مستعارًا أو مراجعته بطريقة أخرى. لا تضمّن أسرارًا أو مستندات خاصة أو ملفات
قانونية خاصة أو وثائق هوية شخصية أو رموز API أو رموز تحدي DNS في قضية عامة.

## تعليقات الإشراف

قد تؤدي بعض النتائج الشديدة أو مشكلات السياسة إلى وضع ناشر أو قائمة تحت تعليق
إشرافي. عند حدوث ذلك، قد يُخفى المحتوى المتأثر من الاكتشاف العام، أو قد تبدأ
عمليات النشر المستقبلية مخفية إلى أن تتم مراجعة المشكلة.

تهدف تعليقات الإشراف إلى حماية المستخدمين أثناء حل ClawHub للحالات عالية
المخاطر. ويمكن أيضًا رفعها عند تأكيد أنها نتيجة إيجابية كاذبة.

## القوائم المخفية أو المحظورة

قد تكون القائمة معلقة أو مخفية أو معزولة أو ملغاة أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبّت الإصدار إلا إذا حل المالك المشكلة أو أعاده
الإشراف.

قد يظل بإمكان المالكين رؤية التشخيصات لقوائمهم المعلقة أو المخفية. تساعد هذه
التشخيصات على شرح ما حدث وما الذي يجب تغييره قبل أن تتمكن القائمة من العودة
إلى الأسطح العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تخالف سياسة ClawHub صلاحية النشر. وقد تؤدي الإساءة
الشديدة إلى حظر الحسابات، أو إلغاء الرموز، أو إخفاء المحتوى، أو إزالة القوائم.
تُفحص إشارات ضغط إساءة الناشرين يوميًا. ويمكن للإشارات التي تبلغ عتبة الحظر
المحتمل في ClawHub أن تطلق تحذيرًا تلقائيًا. إذا كان الفحص المؤهل التالي بعد
موعد التحذير النهائي لا يزال يضع الناشر ضمن عتبة الحظر المحتمل، فقد يطبق
ClawHub إجراء الحساب تلقائيًا. تبقى إشارات المراجعة الزمنية محدودة النطاق
ومنخفضة الثقة خارج الإنفاذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـ
ClawHub. إذا بدأت مصادقة CLI بالفشل بعد إجراء على الحساب، فسجّل الدخول إلى
واجهة الويب لمراجعة حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI
محظورًا بسبب حظر أو حساب معطل، فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/)
لمراجعة الاسترداد.

إذا سمّى بريد إلكتروني ناتج عن ماسح إصدار مهارة أو مكون إضافي على أنه خبيث،
فنزل نتائج الفحص المخزنة للإصدار المقدم المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى المكونات
الإضافية، أضف `--kind plugin`. راجع مخرجات الفحص، وأصلح القائمة، وزد رقم
الإصدار، وارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الكاذبة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغييرات
- أعلن متغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المعماة
- اربط بالمصدر عندما يكون ذلك ممكنًا
- استخدم التشغيلات التجريبية قبل نشر المكونات الإضافية
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
