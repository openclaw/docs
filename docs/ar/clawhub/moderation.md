# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف العام والتثبيت ما زالت
تحتاج إلى ضوابط وقائية. تساعد البلاغات، وتعليقات الإشراف، والقوائم المخفية، وإجراءات الحساب
على حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن، أو مضللاً، أو خارج
السياسة.

تغطي هذه الصفحة الإشراف ووضع الحساب. لتسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/clawhub/security-audits).

راجع أيضًا [الأمان](/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). لمخاوف حقوق النشر أو غيرها من
حقوق المحتوى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن Skills وPlugins والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الضارة
- البيانات الوصفية المضللة
- بيانات الاعتماد غير المعلنة أو متطلبات الأذونات
- تعليمات التثبيت المشبوهة
- انتحال الهوية
- التسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن مهارة** في صفحة المهارة، أو أمر/API الإبلاغ عن الحزم
للحزم.

لا تستخدم بلاغات ClawHub للثغرات في الشيفرة المصدرية الخاصة بمهارة أو
Plugin تابعين لطرف ثالث. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من القائمة. لا يتولى ClawHub صيانة أو تصحيح شيفرة المهارات أو Plugins
التابعة لأطراف ثالثة.

تخص GitHub Security Advisories لـ `openclaw/clawhub` الثغرات الموجودة في
ClawHub نفسه. تشمل الأمثلة أخطاء في الموقع الإلكتروني أو API أو CLI أو السجل أو المصادقة
أو الفحص أو الإشراف أو حدود الثقة للتنزيل/التثبيت. لا تستخدم إرشادات ClawHub
للثغرات في مهارات أو Plugins تابعة لأطراف ثالثة.

البلاغات الجيدة محددة وقابلة للتنفيذ. وقد يؤدي إساءة استخدام الإبلاغ بحد ذاتها إلى
إجراء على الحساب.

## مطالبات المؤسسات والنطاقات الاسمية

ينبغي أن تستخدم نزاعات ملكية المؤسسات أو العلامات التجارية أو نطاقات الحزم أو معرّفات المالكين أو النطاقات الاسمية
عملية [مطالبات المؤسسات والنطاقات الاسمية](/clawhub/namespace-claims)، وليس
مسار البلاغات داخل المنتج أو نموذج استئناف الحساب.

استخدم هذه العملية عندما تحتاج إلى أن يراجع فريق ClawHub دليلاً غير حساس على أن
نطاقًا اسميًا يجب حجزه، أو نقله، أو تغيير اسمه، أو إخفاؤه، أو عزله، أو إضافة اسم بديل له،
أو مراجعته بطريقة أخرى. لا تُدرج أسرارًا أو مستندات خاصة أو ملفات قانونية خاصة
أو مستندات هوية شخصية أو رموز API أو رموز تحدي DNS في
مسألة عامة.

## تعليقات الإشراف

قد تؤدي بعض النتائج الخطيرة أو مشكلات السياسة إلى وضع ناشر أو قائمة تحت
تعليق إشراف. عند حدوث ذلك، قد يُخفى المحتوى المتأثر من الاكتشاف العام
أو قد تبدأ عمليات النشر المستقبلية مخفية إلى أن تتم مراجعة المشكلة.

تهدف تعليقات الإشراف إلى حماية المستخدمين بينما يحل ClawHub الحالات عالية المخاطر.
ويمكن أيضًا رفعها عند تأكيد وجود نتيجة إيجابية خاطئة.

## القوائم المخفية أو المحظورة

قد تكون القائمة معلقة، أو مخفية، أو معزولة، أو ملغاة، أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبت الإصدار إلا إذا حل المالك
المشكلة أو أعاده الإشراف.

قد يظل المالكون قادرين على رؤية التشخيصات لقوائمهم المعلقة أو المخفية. تساعد هذه
التشخيصات في شرح ما حدث وما الذي يجب تغييره قبل أن
تعود القائمة إلى الأسطح العامة.

## الحظر ووضع الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub صلاحية النشر. قد تؤدي الإساءة الشديدة إلى
حظر الحسابات، أو إلغاء الرموز، أو إخفاء المحتوى، أو إزالة القوائم.
تُفحص إشارات ضغط إساءة الناشرين يوميًا. يمكن للإشارات التي تصل إلى
عتبة الحظر المحتمل في ClawHub أن تطلق تحذيرًا تلقائيًا. إذا كان الفحص المؤهل التالي
بعد مهلة التحذير ما يزال يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيًا.
تبقى إشارات المراجعة الزمنية الأقل ثقة والمحدودة خارج
التنفيذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـ ClawHub. إذا بدأت مصادقة CLI
بالفشل بعد إجراء على الحساب، فسجّل الدخول إلى واجهة الويب لمراجعة
حالة الحساب. إذا كان تسجيل الدخول أو وصول CLI العادي محظورًا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا ذكرت رسالة بريد إلكتروني أطلقها الماسح أن إصدار مهارة أو Plugin ضار،
فنزّل نتائج الفحص المخزنة للإصدار المرسل المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى Plugins، أضف
`--kind plugin`. راجع مخرجات الفحص، وأصلح القائمة، وزد رقم الإصدار،
وارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الخاطئة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغييرات
- أعلن متغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المعتمة
- اربط بالمصدر عندما يكون ذلك ممكنًا
- استخدم التشغيل التجريبي قبل نشر Plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
