# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف العام والتثبيت ما زالت
تحتاج إلى ضوابط حماية. تساعد البلاغات، وتعليقات الإشراف، والقوائم المخفية، وإجراءات الحساب
في حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن، أو مضللا، أو مخالفا
للسياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. لتسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/clawhub/security-audits).

راجع أيضا [الأمان](/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). لمخاوف حقوق النشر أو حقوق المحتوى
الأخرى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن المهارات، وplugins، والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الخبيثة
- البيانات الوصفية المضللة
- بيانات الاعتماد أو متطلبات الأذونات غير المعلنة
- تعليمات التثبيت المشبوهة
- انتحال الهوية
- التسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن مهارة** في صفحة المهارة، أو أمر/API الإبلاغ عن الحزم
للحزم.

لا تستخدم بلاغات ClawHub للثغرات في الشفرة المصدرية الخاصة بمهارة أو
plugin تابع لجهة خارجية. أبلغ عنها مباشرة للناشر أو مستودع المصدر
المرتبط من القائمة. لا يصون ClawHub أو يرقع
شفرة المهارات أو plugins التابعة لجهات خارجية.

استشارات GitHub الأمنية الخاصة بـ `openclaw/clawhub` مخصصة للثغرات في
ClawHub نفسه. تشمل الأمثلة العلل في الموقع الإلكتروني، وAPI، وCLI، والسجل، والمصادقة،
والمسح، والإشراف، أو حدود الثقة للتنزيل/التثبيت. لا تستخدم استشارات ClawHub
للثغرات في المهارات أو plugins التابعة لجهات خارجية.

البلاغات الجيدة محددة وقابلة للتنفيذ. إساءة استخدام الإبلاغ يمكن أن تؤدي هي نفسها إلى
إجراء على الحساب.

## مطالبات المؤسسة والنطاق الاسمي

ينبغي أن تستخدم نزاعات ملكية المؤسسة أو العلامة التجارية أو نطاق الحزمة أو معرّف المالك أو النطاق الاسمي
عملية [مطالبات المؤسسة والنطاق الاسمي](/clawhub/namespace-claims)، وليس
تدفق الإبلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم تلك العملية عندما تحتاج إلى أن يراجع موظفو ClawHub دليلا غير حساس على أن
نطاقا اسميا يجب حجزه، أو نقله، أو إعادة تسميته، أو إخفاؤه، أو عزله، أو إضافة اسم بديل له،
أو مراجعته بطريقة أخرى. لا تدرج أسرارا، أو مستندات خاصة، أو ملفات قانونية خاصة،
أو وثائق هوية شخصية، أو رموز API، أو رموز تحدي DNS في
مسألة عامة.

## تعليقات الإشراف

يمكن لبعض النتائج الشديدة أو مسائل السياسة أن تضع ناشرا أو قائمة تحت
تعليق إشراف. عندما يحدث ذلك، قد يخفى المحتوى المتأثر من
الاكتشاف العام، أو قد تبدأ عمليات النشر المستقبلية مخفية حتى تتم مراجعة المسألة.

تهدف تعليقات الإشراف إلى حماية المستخدمين بينما يحل ClawHub الحالات عالية المخاطر.
ويمكن أيضا رفعها عند تأكيد نتيجة إيجابية كاذبة.

## القوائم المخفية أو المحظورة

قد تكون القائمة معلقة، أو مخفية، أو معزولة، أو ملغاة، أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبت الإصدار إلا إذا حل المالك
المسألة أو أعاد الإشراف إتاحته.

قد يظل المالكون قادرين على رؤية التشخيصات لقوائمهم المعلقة أو المخفية. تساعد هذه
التشخيصات في توضيح ما حدث وما يحتاج إلى تغيير قبل أن
تعود القائمة إلى الأسطح العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub إمكانية النشر. ويمكن أن تؤدي الإساءة الشديدة إلى
حظر الحساب، أو إبطال الرموز، أو إخفاء المحتوى، أو إزالة القوائم.
تراجع إشارات ضغط إساءة الناشرين يوميا. يمكن للإشارات التي تبلغ
عتبة الحظر المحتمل في ClawHub أن تطلق تحذيرا تلقائيا. إذا كان الفحص
المؤهل التالي بعد مهلة التحذير لا يزال يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيا.
تبقى إشارات المراجعة الزمنية ذات الثقة المنخفضة والمحدودة خارج
التنفيذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز ClawHub API. إذا بدأت مصادقة CLI
بالفشل بعد إجراء على الحساب، فسجل الدخول إلى واجهة الويب لمراجعة حالة
الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محظورا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا سمى بريد إلكتروني ناتج عن الماسح إصدار مهارة أو plugin على أنه خبيث،
فنزل نتائج المسح المخزنة للإصدار المقدم المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى plugins، أضف
`--kind plugin`. راجع مخرجات المسح، وأصلح القائمة، وزد رقم الإصدار،
وارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الكاذبة وتحسين ثقة المستخدم:

- أبق الأسماء والملخصات والوسوم وسجلات التغيير دقيقة
- أعلن متغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المعماة
- اربط بالمصدر عندما يكون ذلك ممكنا
- استخدم التشغيلات الجافة قبل نشر plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
