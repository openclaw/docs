# الإشراف وسلامة الحساب

يتيح ClawHub النشر للجميع، لكن أسطح الاكتشاف العام والتثبيت لا تزال بحاجة إلى
ضوابط وقائية. تساعد البلاغات، وتعليقات الإشراف، والقوائم المخفية، وإجراءات الحساب
على حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن، أو مضللًا، أو مخالفًا
للسياسة.

تتناول هذه الصفحة الإشراف وحالة الحساب. للاطلاع على تسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[عمليات التدقيق الأمني](/clawhub/security-audits).

راجع أيضًا [الأمان](/ar/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). للمخاوف المتعلقة بحقوق النشر أو حقوق المحتوى
الأخرى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الدخول الإبلاغ عن Skills وplugins والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الضارة
- البيانات الوصفية المضللة
- بيانات الاعتماد أو متطلبات الأذونات غير المعلنة
- تعليمات التثبيت المشبوهة
- انتحال الهوية
- التسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن Skill** في صفحة Skill، أو أمر/واجهة API للإبلاغ
عن الحزم.

لا تستخدم بلاغات ClawHub للإبلاغ عن الثغرات في الشيفرة المصدرية الخاصة بـSkill أو
Plugin تابع لجهة خارجية. أبلغ عنها مباشرةً إلى الناشر أو مستودع الشيفرة المصدرية
المرتبط من القائمة. لا يتولى ClawHub صيانة أو تصحيح شيفرة Skills أوplugins
التابعة لجهات خارجية.

تُستخدم استشارات GitHub الأمنية الخاصة بـ`openclaw/clawhub` للثغرات الموجودة في
ClawHub نفسه. تشمل الأمثلة الأخطاء في الموقع الإلكتروني أو API أو CLI أو السجل أو المصادقة
أو الفحص أو الإشراف أو حدود الثقة الخاصة بالتنزيل/التثبيت. لا تستخدم استشارات ClawHub
للثغرات الموجودة في Skills أوplugins التابعة لجهات خارجية.

تكون البلاغات الجيدة محددة وقابلة للتنفيذ. وقد تؤدي إساءة استخدام الإبلاغ بحد ذاتها إلى
اتخاذ إجراء بحق الحساب.

## مطالبات المؤسسات ومساحات الأسماء

يجب معالجة النزاعات حول ملكية المؤسسة أو العلامة التجارية أو نطاق الحزمة أو معرّف المالك أو مساحة الاسم
من خلال إجراء [مطالبات المؤسسات ومساحات الأسماء](/clawhub/namespace-claims)، وليس من خلال
مسار الإبلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم هذا الإجراء عندما تحتاج إلى أن يراجع موظفو ClawHub إثباتًا غير حساس يفيد بضرورة
حجز مساحة اسم أو نقلها أو إعادة تسميتها أو إخفائها أو عزلها أو إنشاء اسم مستعار لها
أو مراجعتها بطريقة أخرى. لا تُضمّن أسرارًا أو مستندات خاصة أو ملفات قانونية خاصة
أو وثائق هوية شخصية أو رموز API المميزة أو رموز تحدي DNS المميزة في
مسألة عامة.

## تعليقات الإشراف

قد تؤدي بعض النتائج الخطيرة أو مشكلات السياسة إلى وضع ناشر أو قائمة قيد
تعليق الإشراف. عند حدوث ذلك، قد يُخفى المحتوى المتأثر من
الاكتشاف العام، أو قد تبدأ عمليات النشر المستقبلية مخفية حتى تتم مراجعة المشكلة.

تهدف تعليقات الإشراف إلى حماية المستخدمين بينما يعالج ClawHub الحالات
عالية المخاطر. ويمكن أيضًا رفعها عند التأكد من وجود نتيجة إيجابية خاطئة.

## القوائم المخفية أو المحظورة

قد تكون القائمة معلّقة أو مخفية أو معزولة أو ملغاة أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبّت الإصدار ما لم يعالج المالك
المشكلة أو يعِد الإشراف إتاحته.

قد يظل بإمكان المالكين رؤية بيانات التشخيص لقوائمهم المعلّقة أو المخفية. تساعد هذه
البيانات في توضيح ما حدث وما يجب تغييره قبل أن تتمكن
القائمة من العودة إلى الأسطح العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub صلاحية النشر. ويمكن أن تؤدي
الإساءة الجسيمة إلى حظر الحساب أو إلغاء الرموز المميزة أو إخفاء المحتوى أو إزالة القوائم.
يتم التحقق يوميًا من إشارات الضغط الدالة على إساءة الناشر. ويمكن للإشارات التي تبلغ
عتبة الحظر المحتمل في ClawHub أن تطلق تحذيرًا تلقائيًا. وإذا استمر الفحص المؤهل التالي
بعد الموعد النهائي للتحذير في وضع الناشر ضمن عتبة الحظر المحتمل،
فقد يطبق ClawHub إجراء الحساب تلقائيًا.
وتظل إشارات المراجعة الزمنية محدودة النطاق والأقل موثوقية خارج نطاق
الإنفاذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API المميزة الخاصة بـClawHub. إذا بدأت مصادقة CLI
بالفشل بعد اتخاذ إجراء بحق الحساب، فسجّل الدخول إلى واجهة الويب لمراجعة
حالة الحساب. وإذا كان تسجيل الدخول أو الوصول العادي عبر CLI محظورًا بسبب حظر أو تعطيل الحساب،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لطلب مراجعة الاسترداد.

إذا ذكرت رسالة بريد إلكتروني ناتجة عن أداة الفحص أن إصدارًا من Skill أوPlugin ضار،
فنزّل نتائج الفحص المخزنة للإصدار المرسل والمحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلىplugins، أضف
`--kind plugin`. راجع مخرجات الفحص، وأصلح القائمة، وزِد رقم الإصدار،
ثم ارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الخاطئة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغييرات
- صرّح بمتغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المبهمة
- أضف رابطًا إلى المصدر متى أمكن
- استخدم عمليات التشغيل التجريبي قبل نشر plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
