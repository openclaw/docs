# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف العام والتثبيت ما زالت
تحتاج إلى ضوابط حماية. تساعد البلاغات، وحالات حجز الإشراف، والقوائم المخفية، وإجراءات الحساب
في حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن أو مضللا أو خارجا
عن السياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. للاطلاع على تسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/ar/clawhub/security-audits).

راجع أيضا [الأمان](/ar/clawhub/security) و
[الاستخدام المقبول](/ar/clawhub/acceptable-usage). بالنسبة إلى مخاوف حقوق النشر أو حقوق المحتوى
الأخرى، استخدم [طلبات حقوق المحتوى](/ar/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن Skills وplugins والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الخبيثة
- البيانات الوصفية المضللة
- متطلبات بيانات الاعتماد أو الأذونات غير المعلنة
- تعليمات التثبيت المريبة
- انتحال الهوية
- التسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/ar/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن Skill** في صفحة Skill، أو أمر/API الإبلاغ عن الحزم
للحزم.

لا تستخدم بلاغات ClawHub للثغرات في الشيفرة المصدرية الخاصة بـ Skill أو
plugin تابع لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من القائمة. لا يتولى ClawHub صيانة أو تصحيح شيفرة Skills أو plugins
التابعة لجهات خارجية.

تخص استشارات GitHub الأمنية لـ `openclaw/clawhub` الثغرات الموجودة في
ClawHub نفسه. تشمل الأمثلة الأخطاء في الموقع الإلكتروني، أو API، أو CLI، أو السجل، أو المصادقة،
أو الفحص، أو الإشراف، أو حدود الثقة الخاصة بالتنزيل/التثبيت. لا تستخدم استشارات ClawHub
للثغرات في Skills أو plugins التابعة لجهات خارجية.

البلاغات الجيدة محددة وقابلة للتنفيذ. ويمكن أن يؤدي إساءة استخدام الإبلاغ بحد ذاته إلى
إجراء على الحساب.

## مطالبات المؤسسات والنطاقات الاسمية

يجب أن تستخدم نزاعات ملكية المؤسسة أو العلامة التجارية أو نطاق الحزمة أو معرّف المالك أو النطاق الاسمي
عملية [مطالبات المؤسسات والنطاقات الاسمية](/ar/clawhub/namespace-claims)، وليس
تدفق الإبلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم تلك العملية عندما تحتاج إلى أن يراجع فريق ClawHub دليلا غير حساس على أن
نطاقا اسميا يجب حجزه أو نقله أو إعادة تسميته أو إخفاؤه أو عزله أو إنشاء اسم بديل له
أو مراجعته بطريقة أخرى. لا تضمّن أسرارا، أو مستندات خاصة، أو ملفات قانونية خاصة،
أو مستندات هوية شخصية، أو رموز API، أو رموز تحدي DNS في
قضية عامة.

## حالات حجز الإشراف

يمكن لبعض النتائج الشديدة أو مشكلات السياسة أن تضع ناشرا أو قائمة تحت
حجز إشراف. عند حدوث ذلك، قد يخفى المحتوى المتأثر من
الاكتشاف العام أو قد تبدأ عمليات النشر المستقبلية مخفية حتى تتم مراجعة المشكلة.

تهدف حالات حجز الإشراف إلى حماية المستخدمين بينما يحل ClawHub الحالات عالية المخاطر.
ويمكن رفعها أيضا عند تأكيد نتيجة إيجابية كاذبة.

## القوائم المخفية أو المحظورة

قد تكون القائمة محتجزة أو مخفية أو معزولة أو ملغاة أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبت الإصدار إلا إذا حل المالك
المشكلة أو أعاده الإشراف.

قد يظل المالكون قادرين على رؤية التشخيصات لقوائمهم المحتجزة أو المخفية. تساعد هذه
التشخيصات في شرح ما حدث وما الذي يجب تغييره قبل أن
تعود القائمة إلى الأسطح العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub صلاحية النشر. ويمكن أن يؤدي الإساءة الشديدة إلى
حظر الحسابات أو إلغاء الرموز أو إخفاء المحتوى أو إزالة القوائم.
تُفحص إشارات ضغط إساءة الناشرين يوميا. يمكن أن تؤدي الإشارات التي تصل إلى
عتبة الحظر المحتمل في ClawHub إلى تحذير تلقائي. إذا كان الفحص
المؤهل التالي بعد موعد التحذير ما زال يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيا.
تبقى إشارات المراجعة الأقل ثقة والمحدودة زمنيا خارج
التنفيذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـ ClawHub. إذا بدأت مصادقة CLI
بالفشل بعد إجراء على الحساب، فسجل الدخول إلى واجهة الويب لمراجعة
حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محظورا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا كانت رسالة بريد إلكتروني ناتجة عن ماسح تذكر إصدار Skill أو plugin على أنه خبيث،
فنزل نتائج الفحص المخزنة للإصدار المقدم المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى plugins، أضف
`--kind plugin`. راجع مخرجات الفحص، وأصلح القائمة، وزد رقم الإصدار،
وارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الكاذبة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغييرات
- أعلن متغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنا
- استخدم التشغيل التجريبي قبل نشر plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
