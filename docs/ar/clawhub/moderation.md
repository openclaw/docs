# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف العام والتثبيت لا تزال
تحتاج إلى ضوابط وقائية. تساعد البلاغات، وتعليقات الإشراف، والقوائم المخفية، وإجراءات الحساب
في حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن أو مضللا أو خارج
السياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. بالنسبة لتسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[عمليات تدقيق الأمان](/clawhub/security-audits).

راجع أيضا [الأمان](/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). بالنسبة لمخاوف حقوق النشر أو حقوق المحتوى
الأخرى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن Skills وPlugins والحزم.

استخدم بلاغات ClawHub فقط للمحتوى غير الآمن في السوق، مثل:

- القوائم الخبيثة
- البيانات الوصفية المضللة
- بيانات اعتماد أو متطلبات أذونات غير معلنة
- تعليمات تثبيت مشبوهة
- انتحال الهوية
- تسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن Skill** في صفحة Skill، أو أمر/API الإبلاغ عن الحزم
للحزم.

لا تستخدم بلاغات ClawHub للثغرات في كود المصدر الخاص بSkill أو
Plugin تابع لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من القائمة. لا يصون ClawHub أو يرقع كود Skill أو Plugin
تابع لجهة خارجية.

إرشادات أمان GitHub الخاصة ب`openclaw/clawhub` مخصصة للثغرات في
ClawHub نفسه. تشمل الأمثلة الأخطاء في الموقع الإلكتروني، وAPI، وCLI، والسجل، والمصادقة،
والمسح، والإشراف، أو حدود الثقة للتنزيل/التثبيت. لا تستخدم إرشادات ClawHub
للثغرات في Skills أو Plugins تابعة لجهات خارجية.

البلاغات الجيدة محددة وقابلة للتنفيذ. يمكن أن يؤدي إساءة استخدام الإبلاغ بحد ذاتها إلى
إجراء على الحساب.

## مطالبات المؤسسات والنطاقات الاسمية

ينبغي أن تستخدم نزاعات ملكية المؤسسة، أو العلامة التجارية، أو نطاق الحزمة، أو مقبض المالك، أو النطاق الاسمي
عملية [مطالبات المؤسسات والنطاقات الاسمية](/clawhub/namespace-claims)، وليس
تدفق الإبلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم هذه العملية عندما تحتاج إلى أن يراجع موظفو ClawHub دليلا غير حساس على أن
نطاقا اسميا ينبغي حجزه، أو نقله، أو إعادة تسميته، أو إخفاؤه، أو عزله، أو إضافة اسم مستعار له،
أو مراجعته بطريقة أخرى. لا تضمّن أسرارا، أو مستندات خاصة، أو ملفات قانونية خاصة،
أو مستندات هوية شخصية، أو رموز API، أو رموز تحدي DNS في
مشكلة عامة.

## تعليقات الإشراف

يمكن لبعض النتائج الخطيرة أو مشكلات السياسة أن تضع ناشرا أو قائمة تحت
تعليق إشراف. عندما يحدث ذلك، قد يتم إخفاء المحتوى المتأثر من الاكتشاف العام
أو قد تبدأ عمليات النشر المستقبلية مخفية إلى أن تتم مراجعة المشكلة.

تهدف تعليقات الإشراف إلى حماية المستخدمين بينما يحل ClawHub الحالات عالية المخاطر.
ويمكن أيضا رفعها عندما يتم تأكيد نتيجة إيجابية كاذبة.

## القوائم المخفية أو المحظورة

قد تكون القائمة معلقة، أو مخفية، أو معزولة، أو ملغاة، أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبت الإصدار إلا إذا حل المالك
المشكلة أو أعاده الإشراف.

قد يظل المالكون قادرين على رؤية التشخيصات لقوائمهم المعلقة أو المخفية. تساعد هذه
التشخيصات في شرح ما حدث وما الذي يجب تغييره قبل أن
تعود القائمة إلى الأسطح العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub حق الوصول إلى النشر. يمكن أن
تؤدي الإساءة الجسيمة إلى حظر الحساب، أو إبطال الرموز، أو إخفاء المحتوى، أو إزالة القوائم.
تُفحص إشارات ضغط إساءة الناشر يوميا. يمكن للإشارات التي تصل إلى
عتبة الحظر المحتمل في ClawHub أن تطلق تحذيرا تلقائيا. إذا كان الفحص
المؤهل التالي بعد مهلة التحذير لا يزال يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيا.
تبقى إشارات المراجعة الأقل ثقة والمحدودة زمنيا خارج
التنفيذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بClawHub. إذا بدأت مصادقة CLI
بالفشل بعد إجراء على الحساب، فسجل الدخول إلى واجهة الويب لمراجعة
حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محظورا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا سمى بريد إلكتروني ناتج عن الماسح إصدار Skill أو Plugin على أنه خبيث،
فنزل نتائج المسح المخزنة للإصدار المقدم المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى Plugins، أضف
`--kind plugin`. راجع مخرجات المسح، وأصلح القائمة، وزد رقم الإصدار،
وارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل الإيجابيات الكاذبة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغيير
- أعلن متغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنا
- استخدم التشغيلات الجافة قبل نشر Plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
