# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف العام والتثبيت لا تزال
تحتاج إلى ضوابط وقائية. تساعد البلاغات، وتعليقات الإشراف، والقوائم المخفية، وإجراءات الحساب
في حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن، أو مضللاً، أو خارجاً
عن السياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. للتسميات الخاصة بالتدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/clawhub/security-audits).

راجع أيضاً [الأمان](/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). للمخاوف المتعلقة بحقوق النشر أو غيرها من
حقوق المحتوى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين الذين سجلوا الدخول الإبلاغ عن Skills وPlugins والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الخبيثة
- البيانات الوصفية المضللة
- بيانات اعتماد أو متطلبات أذونات غير معلنة
- تعليمات تثبيت مريبة
- انتحال الهوية
- تسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- محتوى ينتهك [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن Skill** في صفحة Skill، أو أمر/API
الإبلاغ عن الحزم للحزم.

لا تستخدم بلاغات ClawHub للثغرات في الشيفرة المصدرية الخاصة بـ Skill أو
Plugin تابع لطرف ثالث. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من القائمة. لا يصون ClawHub ولا يرقّع شيفرة Skills أو Plugins
التابعة لأطراف ثالثة.

تنبيهات أمان GitHub الخاصة بـ `openclaw/clawhub` مخصصة للثغرات في
ClawHub نفسه. تشمل الأمثلة أخطاء في الموقع، أو API، أو CLI، أو السجل، أو المصادقة،
أو الفحص، أو الإشراف، أو حدود الثقة للتنزيل/التثبيت. لا تستخدم تنبيهات
ClawHub للثغرات في Skills أو Plugins التابعة لأطراف ثالثة.

البلاغات الجيدة محددة وقابلة للتنفيذ. وقد يؤدي إساءة استخدام البلاغات بحد ذاته إلى
إجراء على الحساب.

## مطالبات المؤسسة ونطاق الاسم

يجب أن تستخدم نزاعات ملكية المؤسسة، أو العلامة التجارية، أو نطاق الحزمة، أو معرّف المالك، أو نطاق الاسم
عملية [مطالبات المؤسسة ونطاق الاسم](/clawhub/namespace-claims)، وليس
تدفق البلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم هذه العملية عندما تحتاج إلى أن يراجع موظفو ClawHub إثباتاً غير حساس بأن
نطاق اسم ينبغي حجزه، أو نقله، أو إعادة تسميته، أو إخفاؤه، أو عزله، أو إضافة اسم مستعار له،
أو مراجعته بطريقة أخرى. لا تُدرج أسراراً، أو مستندات خاصة، أو ملفات قانونية خاصة،
أو وثائق هوية شخصية، أو رموز API، أو رموز تحدي DNS في
قضية عامة.

## تعليقات الإشراف

يمكن لبعض النتائج الشديدة أو مشكلات السياسة أن تضع ناشراً أو قائمة تحت
تعليق إشراف. عندما يحدث ذلك، قد يُخفى المحتوى المتأثر من
الاكتشاف العام أو قد تبدأ عمليات النشر المستقبلية مخفية حتى تتم مراجعة المشكلة.

تهدف تعليقات الإشراف إلى حماية المستخدمين بينما يحل ClawHub الحالات عالية المخاطر.
ويمكن رفعها أيضاً عند تأكيد أن الحالة إيجابية كاذبة.

## القوائم المخفية أو المحظورة

قد تكون القائمة معلّقة، أو مخفية، أو معزولة، أو ملغاة، أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبّت الإصدار ما لم يحل المالك
المشكلة أو يعيده الإشراف.

قد يظل المالكون قادرين على رؤية تشخيصات لقوائمهم المعلّقة أو المخفية. تساعد هذه
التشخيصات في شرح ما حدث وما الذي يجب تغييره قبل أن تتمكن
القائمة من العودة إلى الأسطح العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub صلاحية النشر. يمكن أن تؤدي الإساءة الشديدة إلى
حظر الحساب، أو إبطال الرمز، أو إخفاء المحتوى، أو إزالة القوائم.
تُفحص إشارات ضغط إساءة الناشرين يومياً. الإشارات التي تصل إلى
عتبة الحظر المحتمل في ClawHub يمكن أن تطلق تحذيراً تلقائياً. إذا ظل الفحص
المؤهل التالي بعد مهلة التحذير يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائياً.
تبقى إشارات المراجعة الزمنية الأقل ثقة والمحدودة خارج
الإنفاذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـ ClawHub. إذا بدأت مصادقة CLI
تفشل بعد إجراء على الحساب، فسجّل الدخول إلى واجهة الويب لمراجعة حالة
الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محظوراً بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا سمّت رسالة بريد إلكتروني ناتجة عن ماسح إصدار Skill أو Plugin على أنه خبيث،
فنزّل نتائج الفحص المخزنة للإصدار المرسل المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى Plugins، أضف
`--kind plugin`. راجع مخرجات الفحص، وأصلح القائمة، وزد رقم الإصدار،
ثم ارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل الإيجابيات الكاذبة وتحسين ثقة المستخدم:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغييرات
- أعلن متغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر متى أمكن
- استخدم التشغيل التجريبي قبل نشر Plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
