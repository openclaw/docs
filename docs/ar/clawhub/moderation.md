# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف العام والتثبيت لا تزال
تحتاج إلى ضوابط وقائية. تساعد البلاغات، وتعليقات الإشراف، والقوائم المخفية، وإجراءات الحساب
على حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن أو مضللا أو خارجا
عن السياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. للاطلاع على تسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/clawhub/security-audits).

راجع أيضا [الأمان](/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). لمخاوف حقوق النشر أو حقوق المحتوى
الأخرى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن Skills وplugins والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الخبيثة
- البيانات الوصفية المضللة
- بيانات الاعتماد أو متطلبات الأذونات غير المصرح بها
- تعليمات التثبيت المشبوهة
- انتحال الهوية
- التسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن Skill** في صفحة Skill، أو أمر/واجهة API
الإبلاغ عن الحزم للحزم.

لا تستخدم بلاغات ClawHub للثغرات في الكود المصدري الخاص بـ Skill أو
plugin تابع لطرف ثالث. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من القائمة. لا يصون ClawHub كود Skills أو plugins
التابعة لأطراف ثالثة ولا يرقعه.

تنبيهات GitHub Security Advisories الخاصة بـ `openclaw/clawhub` مخصصة للثغرات في
ClawHub نفسه. تشمل الأمثلة العلل في الموقع الإلكتروني، وAPI، وCLI، والسجل، والمصادقة،
والمسح، والإشراف، أو حدود الثقة للتنزيل/التثبيت. لا تستخدم تنبيهات ClawHub
للثغرات في Skills أو plugins التابعة لأطراف ثالثة.

البلاغات الجيدة محددة وقابلة للتنفيذ. يمكن أن تؤدي إساءة استخدام الإبلاغ بحد ذاتها إلى
إجراء على الحساب.

## مطالبات المؤسسات والنطاقات الاسمية

يجب أن تستخدم نزاعات ملكية المؤسسة أو العلامة التجارية أو نطاق الحزمة أو معرف المالك أو النطاق الاسمي
عملية [مطالبات المؤسسات والنطاقات الاسمية](/clawhub/namespace-claims)، وليس مسار
الإبلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم تلك العملية عندما تحتاج إلى أن يراجع موظفو ClawHub دليلا غير حساس على أن
نطاقا اسميا يجب حجزه أو نقله أو إعادة تسميته أو إخفاؤه أو عزله أو جعله اسما مستعارا
أو مراجعته بطريقة أخرى. لا تضمّن أسرارا أو مستندات خاصة أو ملفات قانونية خاصة
أو مستندات هوية شخصية أو رموز API أو رموز تحدي DNS في
تذكرة عامة.

## تعليقات الإشراف

يمكن لبعض النتائج الخطيرة أو مشكلات السياسة أن تضع ناشرا أو قائمة تحت
تعليق إشراف. عندما يحدث ذلك، قد يخفى المحتوى المتأثر من الاكتشاف العام
أو قد تبدأ عمليات النشر المستقبلية مخفية حتى تتم مراجعة المشكلة.

تهدف تعليقات الإشراف إلى حماية المستخدمين أثناء حل ClawHub للحالات عالية المخاطر.
يمكن أيضا رفعها عندما يتم تأكيد أن الحالة نتيجة إيجابية كاذبة.

## القوائم المخفية أو المحظورة

قد تكون القائمة معلقة أو مخفية أو معزولة أو ملغاة أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبت الإصدار إلا إذا حل المالك
المشكلة أو أعاده الإشراف.

قد يظل بإمكان المالكين رؤية التشخيصات لقوائمهم المعلقة أو المخفية. تساعد هذه
التشخيصات في توضيح ما حدث وما الذي يجب تغييره قبل أن
تعود القائمة إلى الأسطح العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub صلاحية النشر. يمكن أن تؤدي الإساءة الشديدة إلى
حظر الحساب، أو إلغاء الرموز، أو إخفاء المحتوى، أو إزالة القوائم.
تُفحص إشارات ضغط إساءة الناشرين يوميا. يمكن للإشارات التي تصل إلى
عتبة الحظر المحتمل في ClawHub أن تؤدي إلى تحذير تلقائي. إذا كان الفحص
المؤهل التالي بعد موعد التحذير لا يزال يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيا.
تبقى إشارات المراجعة الزمنية المحدودة ومنخفضة الثقة خارج
الإنفاذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـ ClawHub. إذا بدأت مصادقة CLI
بالفشل بعد إجراء على الحساب، فسجل الدخول إلى واجهة الويب لمراجعة
حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محظورا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا ذكر بريد إلكتروني أطلقه الماسح إصدار Skill أو plugin على أنه خبيث،
فنزل نتائج المسح المخزنة للإصدار المرسل المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى plugins، أضف
`--kind plugin`. راجع مخرجات المسح، وأصلح القائمة، وارفع رقم الإصدار،
وارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الكاذبة وتحسين ثقة المستخدم:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغيير
- صرح بمتغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنا
- استخدم التشغيلات التجريبية قبل نشر plugins
- رد بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
