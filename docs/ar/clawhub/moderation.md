# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف العام والتثبيت ما زالت
تحتاج إلى ضوابط حماية. تساعد البلاغات، وتعليقات الإشراف، والقوائم المخفية، وإجراءات الحساب
في حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن أو مضللا أو خارجا
عن السياسة.

تغطي هذه الصفحة الإشراف ووضع الحساب. للتسميات التدقيقية مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/clawhub/security-audits).

راجع أيضا [الأمان](/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). بالنسبة إلى حقوق النشر أو غيرها من
مخاوف حقوق المحتوى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن Skills وPlugins والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الخبيثة
- البيانات الوصفية المضللة
- بيانات الاعتماد أو متطلبات الأذونات غير المعلنة
- تعليمات التثبيت المريبة
- انتحال الهوية
- التسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن Skill** في صفحة Skill، أو أمر/API الإبلاغ عن الحزم
للحزم.

لا تستخدم بلاغات ClawHub للثغرات في الشيفرة المصدرية الخاصة بـ Skill أو
Plugin تابع لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المربوط من القائمة. لا يتولى ClawHub صيانة أو تصحيح شيفرة
Skills أو Plugins التابعة لجهات خارجية.

تخص GitHub Security Advisories لـ `openclaw/clawhub` الثغرات في
ClawHub نفسه. تشمل الأمثلة أخطاء في الموقع الإلكتروني أو API أو CLI أو السجل أو المصادقة أو
الفحص أو الإشراف أو حدود الثقة الخاصة بالتنزيل/التثبيت. لا تستخدم
استشارات ClawHub للثغرات في Skills أو Plugins التابعة لجهات خارجية.

البلاغات الجيدة محددة وقابلة للتنفيذ. قد يؤدي إساءة استخدام الإبلاغ نفسها إلى
إجراء على الحساب.

## مطالبات المؤسسة ونطاق الاسم

ينبغي أن تستخدم نزاعات ملكية المؤسسة أو العلامة التجارية أو نطاق الحزمة أو معرّف المالك أو نطاق الاسم
عملية [مطالبات المؤسسة ونطاق الاسم](/clawhub/namespace-claims)، وليس
مسار الإبلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم تلك العملية عندما تحتاج إلى أن يراجع فريق ClawHub دليلا غير حساس على أن
نطاق اسم ينبغي حجزه أو نقله أو إعادة تسميته أو إخفاؤه أو عزله أو إنشاء اسم بديل له
أو مراجعته بطريقة أخرى. لا تضمّن أسرارا أو مستندات خاصة أو ملفات قانونية خاصة
أو مستندات هوية شخصية أو رموز API أو رموز تحدي DNS في
قضية عامة.

## تعليقات الإشراف

يمكن لبعض النتائج الشديدة أو مشكلات السياسة أن تضع ناشرا أو قائمة تحت
تعليق إشراف. عند حدوث ذلك، قد يُخفى المحتوى المتأثر من
الاكتشاف العام أو قد تبدأ عمليات النشر المستقبلية مخفية حتى تُراجع المشكلة.

تهدف تعليقات الإشراف إلى حماية المستخدمين بينما يحل ClawHub الحالات عالية المخاطر.
ويمكن أيضا رفعها عند تأكيد وجود نتيجة إيجابية كاذبة.

## القوائم المخفية أو المحظورة

قد تكون القائمة معلقة أو مخفية أو معزولة أو ملغاة أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبت الإصدار ما لم يحل المالك
المشكلة أو يعيده الإشراف.

قد يظل المالكون قادرين على رؤية التشخيصات لقوائمهم المعلقة أو المخفية. تساعد هذه
التشخيصات في شرح ما حدث وما يجب تغييره قبل أن
تعود القائمة إلى الأسطح العامة.

## الحظر ووضع الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub حق الوصول إلى النشر. يمكن أن يؤدي سوء الاستخدام الشديد إلى
حظر الحساب أو إلغاء الرموز أو إخفاء المحتوى أو إزالة القوائم.
تُفحص إشارات ضغط إساءة استخدام الناشرين يوميا. يمكن للإشارات التي تصل إلى
عتبة الحظر المحتمل في ClawHub أن تطلق تحذيرا تلقائيا. إذا كان الفحص التالي
المؤهل بعد موعد التحذير ما زال يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيا.
تبقى إشارات المراجعة الزمنية محدودة النطاق والأقل ثقة خارج
التنفيذ التلقائي.

لا تستطيع الحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـ ClawHub. إذا بدأت مصادقة CLI
بالفشل بعد إجراء على الحساب، فسجّل الدخول إلى واجهة الويب لمراجعة
حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محظورا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا سمّى بريد إلكتروني أطلقه الماسح إصدار Skill أو Plugin بأنه خبيث،
فنزّل نتائج الفحص المخزنة للإصدار المرسل والمحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى Plugins، أضف
`--kind plugin`. راجع مخرجات الفحص، وأصلح القائمة، وزد رقم الإصدار،
وارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الكاذبة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغيير
- صرّح بمتغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنا
- استخدم التشغيلات التجريبية قبل نشر Plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
