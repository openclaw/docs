# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف العام والتثبيت ما تزال
تحتاج إلى ضوابط حماية. تساعد البلاغات، وتعليقات الإشراف، والقوائم المخفية، وإجراءات الحساب
على حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن، أو مضللًا، أو خارجًا
عن السياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. للاطلاع على تسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/clawhub/security-audits).

راجع أيضًا [الأمان](/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). للمخاوف المتعلقة بحقوق النشر أو حقوق المحتوى
الأخرى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن Skills وPlugin والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الخبيثة
- البيانات الوصفية المضللة
- بيانات الاعتماد أو متطلبات الأذونات غير المعلنة
- تعليمات التثبيت المشبوهة
- انتحال الهوية
- التسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن Skill** في صفحة Skill، أو أمر/واجهة API الخاصة بالإبلاغ عن الحزم
للحزم.

لا تستخدم بلاغات ClawHub للثغرات في الشيفرة المصدرية الخاصة بجهة خارجية
لـ Skill أو Plugin. أبلغ عنها مباشرةً إلى الناشر أو مستودع المصدر
المرتبط من القائمة. لا يصون ClawHub شيفرة Skills أو Plugin التابعة لجهات خارجية
ولا يرقعها.

إرشادات GitHub الأمنية الخاصة بـ `openclaw/clawhub` مخصصة للثغرات في
ClawHub نفسه. تشمل الأمثلة الأخطاء في الموقع الإلكتروني، أو API، أو CLI، أو السجل، أو المصادقة،
أو الفحص، أو الإشراف، أو حدود الثقة للتنزيل/التثبيت. لا تستخدم إرشادات ClawHub
للثغرات في Skills أو Plugin التابعة لجهات خارجية.

البلاغات الجيدة محددة وقابلة للتنفيذ. قد يؤدي إساءة استخدام البلاغات بحد ذاتها إلى
إجراء على الحساب.

## مطالبات المؤسسات ومساحات الأسماء

ينبغي أن تستخدم نزاعات ملكية المؤسسة، أو العلامة التجارية، أو نطاق الحزمة، أو مقبض المالك، أو مساحة الاسم
عملية [مطالبات المؤسسات ومساحات الأسماء](/clawhub/namespace-claims)، وليس
مسار البلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم تلك العملية عندما تحتاج إلى أن يراجع فريق ClawHub إثباتًا غير حساس بأن
مساحة اسم ينبغي حجزها، أو نقلها، أو إعادة تسميتها، أو إخفاؤها، أو عزلها، أو إنشاء اسم مستعار لها،
أو مراجعتها بطريقة أخرى. لا تضمّن أسرارًا، أو مستندات خاصة، أو ملفات قانونية خاصة،
أو مستندات هوية شخصية، أو رموز API، أو رموز تحدي DNS في
تذكرة عامة.

## تعليقات الإشراف

يمكن لبعض النتائج الشديدة أو مشكلات السياسة أن تضع ناشرًا أو قائمة تحت
تعليق إشراف. عند حدوث ذلك، قد يُخفى المحتوى المتأثر من الاكتشاف
العام، أو قد تبدأ عمليات النشر المستقبلية مخفية إلى أن تتم مراجعة المشكلة.

تهدف تعليقات الإشراف إلى حماية المستخدمين بينما يحل ClawHub الحالات عالية المخاطر.
ويمكن أيضًا رفعها عند تأكيد نتيجة إيجابية كاذبة.

## القوائم المخفية أو المحظورة

قد تكون قائمة ما معلقة، أو مخفية، أو معزولة، أو ملغاة، أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبت الإصدار ما لم يحل المالك
المشكلة أو يعيده الإشراف.

قد يظل بإمكان المالكين رؤية التشخيصات الخاصة بقوائمهم المعلقة أو المخفية. تساعد هذه
التشخيصات على شرح ما حدث وما يجب تغييره قبل أن تتمكن
القائمة من العودة إلى الأسطح العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub إمكانية النشر. يمكن أن يؤدي الإساءة الشديدة إلى
حظر الحسابات، أو إلغاء الرموز، أو إخفاء المحتوى، أو إزالة القوائم.
تُفحص إشارات ضغط إساءة الناشرين يوميًا. يمكن للإشارات التي تصل إلى
عتبة الحظر المحتمل في ClawHub أن تطلق تحذيرًا تلقائيًا. إذا كان الفحص
المؤهل التالي بعد مهلة التحذير ما يزال يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيًا.
تبقى إشارات المراجعة منخفضة الثقة والمحدودة زمنيًا خارج
التنفيذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـ ClawHub. إذا بدأت مصادقة CLI
بالفشل بعد إجراء على الحساب، فسجل الدخول إلى واجهة الويب لمراجعة حالة
الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محظورًا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا سمّت رسالة بريد إلكتروني ناتجة عن فاحص إصدار Skill أو Plugin بأنه خبيث،
فنزل نتائج الفحص المخزنة للإصدار المقدم المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى Plugin، أضف
`--kind plugin`. راجع مخرجات الفحص، وأصلح القائمة، وزد رقم الإصدار،
وارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الكاذبة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغييرات
- أعلن متغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المموهة
- اربط بالمصدر عندما يكون ذلك ممكنًا
- استخدم التشغيلات التجريبية قبل نشر Plugin
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
