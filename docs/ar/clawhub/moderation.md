# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف والتثبيت العامة ما زالت
تحتاج إلى ضوابط حماية. تساعد البلاغات، وتعليقات الإشراف، والقوائم المخفية، وإجراءات الحساب
في حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن أو مضللا أو خارج
السياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. لتسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/clawhub/security-audits).

راجع أيضا [الأمان](/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). للمخاوف المتعلقة بحقوق النشر أو حقوق المحتوى
الأخرى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن المهارات وPlugins والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الخبيثة
- البيانات الوصفية المضللة
- بيانات الاعتماد أو متطلبات الأذونات غير المعلنة
- تعليمات التثبيت المشبوهة
- انتحال الهوية
- التسجيلات سيئة النية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن المهارة** في صفحة المهارة، أو أمر/API الإبلاغ عن الحزم
للحزم.

لا تستخدم بلاغات ClawHub للثغرات في الكود المصدري الخاص بمهارة أو
Plugin تابع لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من القائمة. لا يصون ClawHub كود مهارات أو Plugins
الجهات الخارجية ولا يرقعه.

GitHub Security Advisories الخاصة بـ `openclaw/clawhub` مخصصة للثغرات في
ClawHub نفسه. تشمل الأمثلة الأخطاء في الموقع الإلكتروني، أو API، أو CLI، أو السجل، أو المصادقة،
أو الفحص، أو الإشراف، أو حدود الثقة الخاصة بالتنزيل/التثبيت. لا تستخدم
استشارات ClawHub للثغرات في مهارات أو Plugins الجهات الخارجية.

البلاغات الجيدة محددة وقابلة للتنفيذ. يمكن أن تؤدي إساءة استخدام الإبلاغ بحد ذاتها إلى
إجراء على الحساب.

## مطالبات المؤسسات والنطاقات الاسمية

يجب أن تستخدم نزاعات ملكية المؤسسة، أو العلامة التجارية، أو نطاق الحزمة، أو معرف المالك، أو النطاق الاسمي
عملية [مطالبات المؤسسات والنطاقات الاسمية](/clawhub/namespace-claims)، وليس
مسار البلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم تلك العملية عندما تحتاج إلى أن يراجع فريق ClawHub دليلا غير حساس على أن
النطاق الاسمي يجب حجزه، أو نقله، أو إعادة تسميته، أو إخفاؤه، أو عزله، أو إضافة اسم بديل له،
أو مراجعته بطريقة أخرى. لا تدرج أسرارا، أو مستندات خاصة، أو ملفات قانونية خاصة،
أو وثائق هوية شخصية، أو رموز API، أو رموز تحدي DNS في
قضية عامة.

## تعليقات الإشراف

قد تؤدي بعض النتائج الخطيرة أو مشكلات السياسة إلى وضع ناشر أو قائمة تحت
تعليق إشراف. عندما يحدث ذلك، قد يخفى المحتوى المتأثر من
الاكتشاف العام أو قد تبدأ عمليات النشر المستقبلية مخفية إلى أن تتم مراجعة المشكلة.

تهدف تعليقات الإشراف إلى حماية المستخدمين بينما يحل ClawHub الحالات عالية المخاطر.
ويمكن رفعها أيضا عند تأكيد وجود نتيجة إيجابية كاذبة.

## القوائم المخفية أو المحظورة

قد تكون قائمة ما معلقة، أو مخفية، أو معزولة، أو ملغاة، أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبت الإصدار إلا إذا حل المالك
المشكلة أو أعادته جهة الإشراف.

قد يظل المالكون قادرين على رؤية تشخيصات قوائمهم المعلقة أو المخفية. تساعد هذه
التشخيصات في شرح ما حدث وما الذي يجب تغييره قبل أن تتمكن
القائمة من العودة إلى الأسطح العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub صلاحية النشر. يمكن أن يؤدي سوء الاستخدام الشديد إلى
حظر الحساب، أو إلغاء الرموز، أو إخفاء المحتوى، أو إزالة القوائم.
يتم فحص إشارات ضغط إساءة استخدام الناشر يوميا. يمكن للإشارات التي تبلغ
حد الحظر المحتمل في ClawHub أن تشغل تحذيرا تلقائيا. إذا ظل الفحص
المؤهل التالي بعد موعد التحذير يضع الناشر ضمن
حد الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيا.
تبقى إشارات المراجعة الزمنية محدودة النطاق والأقل ثقة خارج
التنفيذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـ ClawHub. إذا بدأت مصادقة CLI
بالفشل بعد إجراء على الحساب، فسجل الدخول إلى واجهة الويب لمراجعة
حالة الحساب. إذا كان تسجيل الدخول أو وصول CLI العادي محظورا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا ذكرت رسالة بريد إلكتروني مشغلة بواسطة ماسح إصدار مهارة أو Plugin على أنه خبيث،
فنزل نتائج الفحص المخزنة للإصدار المقدم المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى Plugins، أضف
`--kind plugin`. راجع مخرجات الفحص، وأصلح القائمة، وزد رقم الإصدار،
وارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل الإيجابيات الكاذبة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغيير
- صرح بمتغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنا
- استخدم التشغيلات الجافة قبل نشر Plugins
- رد بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
