# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن واجهات الاكتشاف والتثبيت العامة لا تزال
تحتاج إلى ضوابط حماية. تساعد البلاغات، وحالات التعليق للمراجعة، والقوائم
المخفية، وإجراءات الحساب على حماية المستخدمين عندما يبدو إصدار أو حساب غير
آمن أو مضللًا أو مخالفًا للسياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. لتسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/ar/clawhub/security-audits).

راجع أيضًا [الأمان](/ar/clawhub/security) و
[الاستخدام المقبول](/ar/clawhub/acceptable-usage). للمخاوف المتعلقة بحقوق النشر أو
حقوق المحتوى الأخرى، استخدم [طلبات حقوق المحتوى](/ar/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن المهارات، وplugins، والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الضارة
- البيانات الوصفية المضللة
- بيانات الاعتماد أو متطلبات الأذونات غير المعلنة
- تعليمات التثبيت المشبوهة
- انتحال الهوية
- التسجيلات سيئة النية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/ar/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن مهارة** في صفحة المهارة، أو أمر/API الإبلاغ عن الحزم
للحزم.

لا تستخدم بلاغات ClawHub للثغرات في الشيفرة المصدرية الخاصة بمهارة أو
plugin تابع لطرف ثالث. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من القائمة. لا يصون ClawHub ولا يرقع شيفرة المهارات أو plugins
التابعة لأطراف ثالثة.

إرشادات GitHub الأمنية الخاصة بـ `openclaw/clawhub` مخصصة للثغرات في
ClawHub نفسه. تشمل الأمثلة الأخطاء في موقع الويب، أو API، أو CLI، أو السجل، أو المصادقة،
أو الفحص، أو الإشراف، أو حدود الثقة الخاصة بالتنزيل/التثبيت. لا تستخدم إرشادات ClawHub
للثغرات في المهارات أو plugins التابعة لأطراف ثالثة.

البلاغات الجيدة محددة وقابلة للتنفيذ. قد يؤدي إساءة استخدام الإبلاغ بحد ذاته إلى
إجراء على الحساب.

## مطالبات المؤسسة والنطاق الاسمي

يجب أن تستخدم نزاعات ملكية المؤسسات، أو العلامات التجارية، أو نطاقات الحزم، أو معرّفات المالكين، أو النطاقات الاسمية
عملية [مطالبات المؤسسة والنطاق الاسمي](/ar/clawhub/namespace-claims)، لا
تدفق الإبلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم هذه العملية عندما تحتاج إلى أن يراجع موظفو ClawHub دليلًا غير حساس على أن
نطاقًا اسميًا يجب أن يُحجز، أو يُنقل، أو يُعاد تسميته، أو يُخفى، أو يُحجر، أو يُضاف له اسم مستعار،
أو يخضع لمراجعة أخرى. لا تُضمّن أسرارًا، أو مستندات خاصة، أو ملفات قانونية خاصة،
أو وثائق هوية شخصية، أو رموز API، أو رموز تحدي DNS في
قضية عامة.

## التعليق للمراجعة

يمكن لبعض النتائج الخطيرة أو مشكلات السياسة أن تضع ناشرًا أو قائمة تحت
تعليق للمراجعة. عندما يحدث ذلك، قد يُخفى المحتوى المتأثر من
الاكتشاف العام أو قد تبدأ عمليات النشر المستقبلية مخفية إلى أن تتم مراجعة المشكلة.

تهدف حالات التعليق للمراجعة إلى حماية المستخدمين بينما يحل ClawHub الحالات عالية المخاطر.
ويمكن أيضًا رفعها عند تأكيد وجود نتيجة إيجابية كاذبة.

## القوائم المخفية أو المحظورة

قد تكون القائمة معلقة، أو مخفية، أو محجورة، أو ملغاة، أو غير متاحة بطريقة أخرى على
واجهات التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبت الإصدار إلا إذا حل المالك
المشكلة أو أعاد الإشراف إتاحته.

قد يظل بإمكان المالكين رؤية التشخيصات لقوائمهم المعلقة أو المخفية. تساعد هذه
التشخيصات في شرح ما حدث وما الذي يجب تغييره قبل أن
تعود القائمة إلى الواجهات العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub حق الوصول إلى النشر. يمكن أن يؤدي سوء الاستخدام الشديد إلى
حظر الحساب، أو إبطال الرموز، أو إخفاء المحتوى، أو إزالة القوائم.
تُفحص إشارات ضغط إساءة استخدام الناشر يوميًا. يمكن للإشارات التي تصل إلى
عتبة الحظر المحتمل في ClawHub أن تؤدي إلى تحذير تلقائي. إذا كان الفحص
المؤهل التالي بعد مهلة التحذير لا يزال يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبّق ClawHub إجراء الحساب تلقائيًا.
تبقى إشارات المراجعة منخفضة الثقة والمقيدة زمنيًا خارج
الإنفاذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـ ClawHub. إذا بدأت مصادقة CLI
بالفشل بعد إجراء على الحساب، فسجّل الدخول إلى واجهة الويب لمراجعة
حالة الحساب. إذا كان تسجيل الدخول أو وصول CLI العادي محظورًا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا سمّت رسالة بريد إلكتروني مُشغّلة بواسطة الماسح إصدار مهارة أو plugin على أنه ضار،
فنزّل نتائج الفحص المخزنة للإصدار المقدم المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى plugins، أضف
`--kind plugin`. راجع مخرجات الفحص، وأصلح القائمة، وزد رقم الإصدار،
وارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل الإيجابيات الكاذبة وتحسين ثقة المستخدم:

- حافظ على دقة الأسماء، والملخصات، والوسوم، وسجلات التغييرات
- صرّح بمتغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنًا
- استخدم التشغيلات التجريبية قبل نشر plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
