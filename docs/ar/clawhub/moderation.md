# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف العام والتثبيت لا تزال
بحاجة إلى ضوابط حماية. تساعد البلاغات، وتعليقات الإشراف، والقوائم المخفية، وإجراءات الحساب
على حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن أو مضللا أو خارج
السياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. لتسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/clawhub/security-audits).

راجع أيضا [الأمان](/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). بالنسبة إلى مخاوف حقوق النشر أو غيرها من حقوق
المحتوى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن Skills وPlugins والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الخبيثة
- البيانات الوصفية المضللة
- بيانات اعتماد أو متطلبات أذونات غير مصرح بها
- تعليمات تثبيت مشبوهة
- انتحال الهوية
- التسجيلات سيئة النية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن Skill** في صفحة Skill، أو أمر/API
الإبلاغ عن الحزم.

لا تستخدم بلاغات ClawHub للثغرات في الشيفرة المصدرية الخاصة بـ Skill أو
Plugin تابع لطرف ثالث. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من القائمة. لا يصون ClawHub ولا يصلح
شيفرة Skills أو Plugins التابعة لأطراف ثالثة.

تخص GitHub Security Advisories لـ `openclaw/clawhub` الثغرات في
ClawHub نفسه. تشمل الأمثلة الأخطاء في الموقع الإلكتروني أو API أو CLI أو السجل أو المصادقة أو
الفحص أو الإشراف أو حدود الثقة للتنزيل/التثبيت. لا تستخدم استشارات ClawHub
للثغرات في Skills أو Plugins التابعة لأطراف ثالثة.

البلاغات الجيدة محددة وقابلة للتنفيذ. وقد تؤدي إساءة استخدام الإبلاغ بحد ذاتها إلى
إجراء على الحساب.

## مطالبات المؤسسات ومساحات الأسماء

ينبغي أن تستخدم نزاعات ملكية المؤسسة أو العلامة التجارية أو نطاق الحزمة أو معرّف المالك أو مساحة الاسم
عملية [مطالبات المؤسسات ومساحات الأسماء](/clawhub/namespace-claims)، وليس مسار
الإبلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم هذه العملية عندما تحتاج إلى أن يراجع موظفو ClawHub دليلا غير حساس على أن
مساحة اسم ينبغي حجزها أو نقلها أو إعادة تسميتها أو إخفاؤها أو عزلها أو إضافة اسم مستعار لها
أو مراجعتها بطريقة أخرى. لا تضمّن أسرارا أو مستندات خاصة أو ملفات قانونية خاصة أو
مستندات هوية شخصية أو رموز API أو رموز تحدي DNS في
تذكرة عامة.

## تعليقات الإشراف

يمكن لبعض النتائج الشديدة أو مشكلات السياسة أن تضع ناشرا أو قائمة تحت
تعليق إشراف. عند حدوث ذلك، قد يخفى المحتوى المتأثر من
الاكتشاف العام، أو قد تبدأ عمليات النشر المستقبلية مخفية حتى تتم مراجعة المشكلة.

تهدف تعليقات الإشراف إلى حماية المستخدمين بينما يحل ClawHub الحالات عالية المخاطر.
كما يمكن رفعها عندما يتأكد وجود نتيجة إيجابية كاذبة.

## القوائم المخفية أو المحظورة

قد تكون القائمة معلقة أو مخفية أو معزولة أو ملغاة أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبت الإصدار ما لم يحل المالك
المشكلة أو يعيده الإشراف.

قد يظل بإمكان المالكين رؤية التشخيصات لقوائمهم المعلقة أو المخفية. تساعد هذه
التشخيصات على شرح ما حدث وما يلزم تغييره قبل أن
تعود القائمة إلى الأسطح العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub حق الوصول إلى النشر. يمكن أن تؤدي الإساءة الشديدة إلى
حظر الحساب أو إلغاء الرموز أو إخفاء المحتوى أو إزالة القوائم.
تُفحص إشارات ضغط إساءة الناشرين يوميا. يمكن للإشارات التي تصل إلى
عتبة الحظر المحتمل في ClawHub أن تطلق تحذيرا تلقائيا. إذا كان الفحص
المؤهل التالي بعد مهلة التحذير لا يزال يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيا.
وتبقى إشارات المراجعة الزمنية الأقل ثقة والمحدودة خارج
التنفيذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـ ClawHub. إذا بدأت مصادقة CLI
تفشل بعد إجراء على الحساب، فسجل الدخول إلى واجهة الويب لمراجعة
حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محظورا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا ذكر بريد إلكتروني ناتج عن فاحص إصدار Skill أو Plugin باعتباره خبيثا،
فنزّل نتائج الفحص المخزنة للإصدار المقدم المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى Plugins، أضف
`--kind plugin`. راجع مخرجات الفحص، وأصلح القائمة، وزد رقم
الإصدار، وارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الكاذبة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغييرات
- صرّح بمتغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنا
- استخدم التشغيل التجريبي قبل نشر Plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
