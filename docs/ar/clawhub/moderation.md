# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف العام والتثبيت ما زالت تحتاج إلى
ضوابط حماية. تساعد البلاغات، وتعليقات الإشراف، والقوائم المخفية، وإجراءات الحساب
على حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن أو مضللاً أو مخالفاً
للسياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. لتسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/clawhub/security-audits).

راجع أيضاً [الأمان](/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). لمخاوف حقوق النشر أو غيرها من
حقوق المحتوى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن المهارات، وPlugins، والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الخبيثة
- البيانات الوصفية المضللة
- بيانات الاعتماد أو متطلبات الأذونات غير المعلنة
- تعليمات التثبيت المشبوهة
- انتحال الهوية
- التسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن المهارة** في صفحة المهارة، أو أمر/API الإبلاغ عن الحزم.

لا تستخدم بلاغات ClawHub للثغرات في الشيفرة المصدرية الخاصة بمهارة أو
Plugin تابع لجهة خارجية. أبلغ عنها مباشرةً إلى الناشر أو مستودع المصدر المرتبط
من القائمة. لا يصون ClawHub شيفرة المهارات أو Plugins التابعة لجهات خارجية ولا
يرقّعها.

تخص GitHub Security Advisories الخاصة بـ `openclaw/clawhub` الثغرات في
ClawHub نفسه. تشمل الأمثلة الأخطاء في الموقع الإلكتروني، أو API، أو CLI، أو
السجل، أو المصادقة، أو الفحص، أو الإشراف، أو حدود الثقة الخاصة بالتنزيل/التثبيت.
لا تستخدم Advisories الخاصة بـ ClawHub للثغرات في المهارات أو Plugins التابعة
لجهات خارجية.

البلاغات الجيدة محددة وقابلة للتنفيذ. ويمكن أن تؤدي إساءة استخدام الإبلاغ نفسها
إلى إجراء على الحساب.

## مطالبات المؤسسة والنطاق الاسمي

ينبغي أن تستخدم نزاعات ملكية المؤسسة، أو العلامة التجارية، أو نطاق الحزمة، أو
معرّف المالك، أو النطاق الاسمي عملية [مطالبات المؤسسة والنطاق الاسمي](/clawhub/namespace-claims)،
وليس مسار الإبلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم هذه العملية عندما تحتاج إلى أن يراجع فريق ClawHub دليلاً غير حساس على أن
نطاقاً اسمياً يجب حجزه، أو نقله، أو إعادة تسميته، أو إخفاؤه، أو عزله، أو إضافة
اسم مستعار له، أو مراجعته بطريقة أخرى. لا تدرج أسراراً، أو مستندات خاصة، أو
ملفات قانونية خاصة، أو وثائق هوية شخصية، أو رموز API، أو رموز تحدي DNS في
مسألة عامة.

## تعليقات الإشراف

يمكن لبعض النتائج الشديدة أو مشكلات السياسة أن تضع ناشراً أو قائمة ضمن تعليق
إشراف. عندما يحدث ذلك، قد يُخفى المحتوى المتأثر من الاكتشاف العام أو قد تبدأ
عمليات النشر المستقبلية مخفية إلى أن تُراجع المشكلة.

تهدف تعليقات الإشراف إلى حماية المستخدمين أثناء حل ClawHub للحالات عالية
المخاطر. ويمكن أيضاً رفعها عند تأكيد وجود نتيجة إيجابية كاذبة.

## القوائم المخفية أو المحظورة

قد تكون القائمة معلقة، أو مخفية، أو معزولة، أو ملغاة، أو غير متاحة بطريقة أخرى
على أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبت الإصدار إلا إذا حل المالك المشكلة أو أعاده
الإشراف.

قد يظل المالكون قادرين على رؤية التشخيصات الخاصة بقوائمهم المعلقة أو المخفية.
تساعد هذه التشخيصات في شرح ما حدث وما الذي يحتاج إلى تغيير قبل أن تعود القائمة
إلى الأسطح العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub صلاحية النشر. ويمكن أن تؤدي إساءة
الاستخدام الشديدة إلى حظر الحسابات، أو إلغاء الرموز، أو إخفاء المحتوى، أو إزالة
القوائم. تُفحص إشارات ضغط إساءة استخدام الناشرين يومياً. ويمكن للإشارات التي تصل
إلى عتبة الحظر المحتمل في ClawHub أن تطلق تحذيراً تلقائياً. إذا ظل الفحص المؤهل
التالي بعد موعد التحذير يضع الناشر ضمن عتبة الحظر المحتمل، فقد يطبق ClawHub
إجراء الحساب تلقائياً. تبقى إشارات المراجعة الأقل ثقة والمحدودة زمنياً خارج
التنفيذ التلقائي.

لا يمكن للحسابات المحذوفة، أو المحظورة، أو المعطلة استخدام رموز API الخاصة بـ
ClawHub. إذا بدأت مصادقة CLI بالفشل بعد إجراء على الحساب، فسجّل الدخول إلى واجهة
الويب لمراجعة حالة الحساب. إذا كان تسجيل الدخول أو وصول CLI العادي محظوراً بسبب
حظر أو حساب معطل، فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/)
لمراجعة الاسترداد.

إذا ذكر بريد إلكتروني أطلقه الماسح إصدار مهارة أو Plugin على أنه خبيث، فقم بتنزيل
نتائج الفحص المخزنة للإصدار المقدم المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى Plugins، أضف
`--kind plugin`. راجع مخرجات الفحص، وأصلح القائمة، وزد رقم الإصدار، وارفع الإصدار
المصحح.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الكاذبة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغييرات
- أعلن متغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المموهة
- اربط بالمصدر عندما يكون ذلك ممكناً
- استخدم عمليات التشغيل التجريبي قبل نشر Plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
