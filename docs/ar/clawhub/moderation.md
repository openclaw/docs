# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف والتثبيت العامة لا تزال
تحتاج إلى ضوابط حماية. تساعد البلاغات، وإيقافات الإشراف، والقوائم المخفية، وإجراءات الحساب
على حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن أو مضللا أو خارج
السياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. لتسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/clawhub/security-audits).

راجع أيضا [الأمان](/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). ولمخاوف حقوق النشر أو حقوق المحتوى
الأخرى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن Skills وplugins والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الخبيثة
- البيانات الوصفية المضللة
- بيانات الاعتماد أو متطلبات الأذونات غير المعلنة
- تعليمات التثبيت المشبوهة
- انتحال الهوية
- التسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن Skill** في صفحة Skill، أو أمر/API الإبلاغ عن الحزم
للحزم.

لا تستخدم بلاغات ClawHub للثغرات في شيفرة المصدر الخاصة بجهة خارجية في
Skill أو plugin. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من القائمة. لا يصون ClawHub شيفرة Skills أو plugins
الخاصة بجهات خارجية ولا يرقعها.

تنبيهات GitHub Security Advisories الخاصة بـ `openclaw/clawhub` مخصصة للثغرات في
ClawHub نفسه. تشمل الأمثلة الأخطاء في الموقع الإلكتروني وAPI وCLI والسجل والمصادقة
والمسح والإشراف وحدود الثقة في التنزيل/التثبيت. لا تستخدم تنبيهات ClawHub
للثغرات في Skills أو plugins الخاصة بجهات خارجية.

البلاغات الجيدة محددة وقابلة للتنفيذ. وقد يؤدي إساءة استخدام الإبلاغ نفسها إلى
إجراء على الحساب.

## مطالبات المؤسسات والنطاقات الاسمية

يجب أن تستخدم نزاعات ملكية المؤسسات أو العلامات التجارية أو نطاقات الحزم أو مقابض المالكين أو النطاقات الاسمية
عملية [مطالبات المؤسسات والنطاقات الاسمية](/clawhub/namespace-claims)، وليس
تدفق البلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم تلك العملية عندما تحتاج إلى أن يراجع فريق ClawHub دليلا غير حساس على أن
نطاقا اسميا يجب حجزه أو نقله أو إعادة تسميته أو إخفاؤه أو عزله أو تسميته كاسم بديل
أو مراجعته بطريقة أخرى. لا تضمّن أسرارا أو مستندات خاصة أو ملفات قانونية خاصة
أو وثائق هوية شخصية أو رموز API أو رموز تحدي DNS في
قضية عامة.

## إيقافات الإشراف

يمكن لبعض النتائج الخطيرة أو مشكلات السياسة أن تضع ناشرا أو قائمة تحت
إيقاف إشراف. عند حدوث ذلك، قد يخفى المحتوى المتأثر من
الاكتشاف العام، أو قد تبدأ عمليات النشر المستقبلية مخفية إلى أن تتم مراجعة المشكلة.

تهدف إيقافات الإشراف إلى حماية المستخدمين بينما يحل ClawHub الحالات عالية المخاطر.
ويمكن أيضا رفعها عند تأكيد أن النتيجة إيجابية كاذبة.

## القوائم المخفية أو المحظورة

قد تكون القائمة موقوفة أو مخفية أو معزولة أو ملغاة أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبت الإصدار ما لم يحل المالك
المشكلة أو يعيده الإشراف.

قد يظل بإمكان المالكين رؤية التشخيصات الخاصة بقوائمهم الموقوفة أو المخفية. تساعد هذه
التشخيصات في توضيح ما حدث وما الذي يجب تغييره قبل أن
تعود القائمة إلى الأسطح العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub حق الوصول إلى النشر. ويمكن أن يؤدي سوء الاستخدام الشديد إلى
حظر الحسابات أو إلغاء الرموز أو إخفاء المحتوى أو إزالة القوائم.
تتحقق ClawHub يوميا من إشارات ضغط إساءة استخدام الناشرين. ويمكن للإشارات التي تبلغ
عتبة الحظر المحتمل في ClawHub أن تطلق تحذيرا تلقائيا. إذا ظل الفحص
المؤهل التالي بعد مهلة التحذير يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيا.
تبقى إشارات المراجعة الأقل ثقة والمحدودة زمنيا خارج
الإنفاذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـ ClawHub. إذا بدأت مصادقة CLI
تفشل بعد إجراء على الحساب، فسجل الدخول إلى واجهة الويب لمراجعة
حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محظورا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا سمى بريد إلكتروني أطلقه ماسح إصدار Skill أو plugin بأنه خبيث،
فنزل نتائج المسح المخزنة للإصدار المقدم المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى plugins، أضف
`--kind plugin`. راجع مخرجات المسح، وأصلح القائمة، وزد رقم الإصدار،
وارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل الإيجابيات الكاذبة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغييرات
- أعلن متغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنا
- استخدم التشغيلات التجريبية قبل نشر plugins
- رد بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
