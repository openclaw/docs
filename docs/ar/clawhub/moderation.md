# الإشراف وسلامة الحسابات

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف العام والتثبيت لا تزال
تحتاج إلى ضوابط حماية. تساعد البلاغات، وقيود الإشراف، والقوائم المخفية، وإجراءات الحسابات
على حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن أو مضللا أو خارجا
عن السياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. لتسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/clawhub/security-audits).

راجع أيضا [الأمان](/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). بالنسبة إلى مخاوف حقوق النشر أو غيرها من
حقوق المحتوى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الدخول الإبلاغ عن skills وplugins والحزم.

استخدم بلاغات ClawHub فقط للمحتوى غير الآمن في سوق التطبيقات، مثل:

- القوائم الخبيثة
- البيانات الوصفية المضللة
- بيانات الاعتماد أو متطلبات الأذونات غير المعلنة
- تعليمات التثبيت المشبوهة
- انتحال الهوية
- التسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن skill** في صفحة skill، أو أمر/واجهة API الإبلاغ عن الحزمة
للحزم.

لا تستخدم بلاغات ClawHub للثغرات الأمنية في الشفرة المصدرية الخاصة بطرف ثالث لـskill أو
plugin. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من القائمة. لا يصون ClawHub أو يرقع
شفرة skills أو plugins التابعة لأطراف ثالثة.

GitHub Security Advisories الخاصة بـ`openclaw/clawhub` مخصصة للثغرات الأمنية في
ClawHub نفسه. تشمل الأمثلة العلل في موقع الويب أو API أو CLI أو السجل أو المصادقة أو
الفحص أو الإشراف أو حدود الثقة للتنزيل/التثبيت. لا تستخدم Advisories الخاصة بـClawHub
للثغرات الأمنية في skills أو plugins التابعة لأطراف ثالثة.

البلاغات الجيدة محددة وقابلة للتنفيذ. يمكن أن تؤدي إساءة استخدام الإبلاغ بحد ذاتها إلى
إجراء على الحساب.

## مطالبات المؤسسات والنطاقات الاسمية

ينبغي أن تستخدم نزاعات ملكية المؤسسة أو العلامة التجارية أو نطاق الحزمة أو معرف المالك أو النطاق الاسمي
عملية [مطالبات المؤسسات والنطاقات الاسمية](/clawhub/namespace-claims)، وليس
مسار الإبلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم تلك العملية عندما تحتاج إلى أن يراجع موظفو ClawHub دليلا غير حساس على أن
النطاق الاسمي ينبغي حجزه أو نقله أو إعادة تسميته أو إخفاؤه أو عزله أو تسميته باسم مستعار
أو مراجعته بطريقة أخرى. لا تدرج أسرارا أو مستندات خاصة أو ملفات قانونية خاصة أو
وثائق هوية شخصية أو رموز API أو رموز تحدي DNS في
مشكلة عامة.

## قيود الإشراف

يمكن لبعض النتائج الشديدة أو مشكلات السياسة أن تضع ناشرا أو قائمة تحت
قيد إشراف. عند حدوث ذلك، قد يخفى المحتوى المتأثر من
الاكتشاف العام، أو قد تبدأ عمليات النشر المستقبلية مخفية حتى تتم مراجعة المشكلة.

تهدف قيود الإشراف إلى حماية المستخدمين أثناء حل ClawHub للحالات عالية المخاطر.
كما يمكن رفعها عند تأكيد نتيجة إيجابية كاذبة.

## القوائم المخفية أو المحظورة

قد تكون قائمة ما قيد الانتظار أو مخفية أو معزولة أو ملغاة أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبت الإصدار إلا إذا حل المالك
المشكلة أو أعاده الإشراف.

قد يظل المالكون قادرين على رؤية التشخيصات لقوائمهم الخاصة الموضوعة قيد الانتظار أو المخفية. تساعد هذه
التشخيصات في شرح ما حدث وما الذي يحتاج إلى تغيير قبل أن
تعود القائمة إلى الأسطح العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub حق الوصول إلى النشر. يمكن أن تؤدي إساءة الاستخدام الشديدة إلى
حظر الحسابات أو إلغاء الرموز أو إخفاء المحتوى أو إزالة القوائم.
تتحقق إشارات ضغط إساءة استخدام الناشرين يوميا. يمكن للإشارات التي تصل إلى
عتبة الحظر المحتمل في ClawHub أن تطلق تحذيرا تلقائيا. إذا كان الفحص التالي
المؤهل بعد الموعد النهائي للتحذير لا يزال يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيا.
تبقى إشارات المراجعة الزمنية محدودة النطاق ومنخفضة الثقة خارج
الإنفاذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـClawHub. إذا بدأت مصادقة CLI
تفشل بعد إجراء على الحساب، فسجل الدخول إلى واجهة الويب لمراجعة
حالة الحساب. إذا كان تسجيل الدخول أو وصول CLI العادي محظورا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا سمى بريد إلكتروني ناتج عن ماسح ضوئي إصدار skill أو plugin على أنه خبيث،
فنزل نتائج الفحص المخزنة للإصدار المقدم المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى plugins، أضف
`--kind plugin`. راجع مخرجات الفحص، وأصلح القائمة، وزد رقم الإصدار،
وارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل الإيجابيات الكاذبة وتحسين ثقة المستخدمين:

- أبق الأسماء والملخصات والوسوم وسجلات التغييرات دقيقة
- أعلن عن متغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المعماة
- ضع رابطا إلى المصدر عندما يكون ذلك ممكنا
- استخدم التشغيلات التجريبية قبل نشر plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
