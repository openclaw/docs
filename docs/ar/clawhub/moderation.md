# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف والتثبيت العامة لا تزال
تحتاج إلى ضوابط حماية. تساعد البلاغات، وتعليقات الإشراف، والقوائم المخفية، وإجراءات الحساب
في حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن أو مضللا أو خارجا
عن السياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. بالنسبة إلى تسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/clawhub/security-audits).

راجع أيضا [الأمان](/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). بالنسبة إلى مخاوف حقوق النشر أو غيرها من
حقوق المحتوى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن المهارات، والإضافات، والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الخبيثة
- البيانات الوصفية المضللة
- بيانات الاعتماد أو متطلبات الأذونات غير المصرح بها
- تعليمات التثبيت المشبوهة
- انتحال الهوية
- التسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن مهارة** في صفحة المهارة، أو أمر/واجهة API
الإبلاغ عن الحزم للحزم.

لا تستخدم بلاغات ClawHub للثغرات في الشيفرة المصدرية الخاصة بمهارة أو
إضافة تابعة لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من القائمة. لا يصون ClawHub شيفرة مهارات أو إضافات
الجهات الخارجية ولا يرقعها.

إرشادات أمان GitHub الخاصة بـ `openclaw/clawhub` مخصصة للثغرات في
ClawHub نفسه. تشمل الأمثلة الأخطاء في الموقع الإلكتروني، أو API، أو CLI، أو السجل، أو المصادقة،
أو المسح، أو الإشراف، أو حدود الثقة للتنزيل/التثبيت. لا تستخدم إرشادات
ClawHub للثغرات في مهارات أو إضافات الجهات الخارجية.

البلاغات الجيدة محددة وقابلة للتنفيذ. إساءة استخدام الإبلاغ قد تؤدي هي نفسها إلى
إجراء على الحساب.

## مطالبات المؤسسة ونطاقات الأسماء

ينبغي لنزاعات ملكية المؤسسة، أو العلامة التجارية، أو نطاق الحزمة، أو مقبض المالك، أو نطاق الاسم
استخدام عملية [مطالبات المؤسسة ونطاقات الأسماء](/clawhub/namespace-claims)، وليس
تدفق البلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم تلك العملية عندما تحتاج إلى أن يراجع فريق ClawHub دليلا غير حساس على أن
نطاق اسم ينبغي حجزه، أو نقله، أو إعادة تسميته، أو إخفاؤه، أو عزله، أو جعله اسما مستعارا،
أو مراجعته بطريقة أخرى. لا تدرج أسرارا، أو مستندات خاصة، أو ملفات قانونية خاصة،
أو وثائق هوية شخصية، أو رموز API، أو رموز تحدي DNS في
قضية عامة.

## تعليقات الإشراف

يمكن لبعض النتائج الشديدة أو مشكلات السياسة أن تضع ناشرا أو قائمة تحت
تعليق إشراف. عندما يحدث ذلك، قد يخفى المحتوى المتأثر من
الاكتشاف العام أو قد تبدأ النشرات المستقبلية مخفية حتى تتم مراجعة المشكلة.

تهدف تعليقات الإشراف إلى حماية المستخدمين بينما يحل ClawHub الحالات عالية المخاطر.
ويمكن أيضا رفعها عند تأكيد نتيجة إيجابية كاذبة.

## القوائم المخفية أو المحظورة

قد تكون قائمة ما معلقة، أو مخفية، أو معزولة، أو ملغاة، أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبت الإصدار ما لم يحل المالك
المشكلة أو يعيده الإشراف.

قد يظل المالكون يرون تشخيصات لقوائمهم المعلقة أو المخفية. تساعد هذه
التشخيصات في شرح ما حدث وما الذي يجب تغييره قبل أن
تعود القائمة إلى الأسطح العامة.

## الحظر وحالة الحساب

الحسابات التي تنتهك سياسة ClawHub قد تفقد صلاحية النشر. قد تؤدي إساءة الاستخدام الشديدة إلى
حظر الحساب، أو إبطال الرموز، أو إخفاء المحتوى، أو إزالة القوائم.
تتحقق إشارات ضغط إساءة استخدام الناشرين يوميا. الإشارات التي تبلغ
عتبة الحظر المحتمل في ClawHub يمكن أن تطلق تحذيرا تلقائيا. إذا ظل الفحص
المؤهل التالي بعد مهلة التحذير يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيا.
تبقى إشارات المراجعة الزمنية منخفضة الثقة والمحدودة خارج
التنفيذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـ ClawHub. إذا بدأت مصادقة CLI
بالفشل بعد إجراء على الحساب، فسجل الدخول إلى واجهة الويب لمراجعة
حالة الحساب. إذا كان تسجيل الدخول أو وصول CLI العادي محظورا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا ذكر بريد إلكتروني أطلقه الماسح إصدار مهارة أو إضافة على أنه خبيث،
فنزّل نتائج المسح المخزنة للإصدار المرسل المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى الإضافات، أضف
`--kind plugin`. راجع مخرجات المسح، وأصلح القائمة، وزد رقم
الإصدار، ثم ارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الكاذبة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغيير
- صرّح بمتغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنا
- استخدم التشغيلات التجريبية قبل نشر الإضافات
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
