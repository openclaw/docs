# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف العام والتثبيت ما زالت
تحتاج إلى حواجز حماية. تساعد البلاغات، وتعليقات الإشراف، والقوائم المخفية، وإجراءات الحساب
في حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن أو مضللا أو خارجا
عن السياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. لتسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/ar/clawhub/security-audits).

راجع أيضا [الأمان](/clawhub/security) و
[الاستخدام المقبول](/ar/clawhub/acceptable-usage). لمخاوف حقوق النشر أو غيرها من
حقوق المحتوى، استخدم [طلبات حقوق المحتوى](/ar/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الدخول الإبلاغ عن Skills وPlugins والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الخبيثة
- البيانات الوصفية المضللة
- بيانات الاعتماد أو متطلبات الأذونات غير المعلنة
- تعليمات التثبيت المشبوهة
- انتحال الهوية
- التسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/ar/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن Skill** في صفحة Skill، أو أمر/API الإبلاغ عن الحزم
للحزم.

لا تستخدم بلاغات ClawHub للثغرات الأمنية في الشيفرة المصدرية الخاصة بـSkill أو
Plugin تابع لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من القائمة. لا يصون ClawHub شيفرة Skills أو Plugins التابعة
لجهات خارجية ولا يرقعها.

إرشادات GitHub الأمنية الخاصة بـ`openclaw/clawhub` مخصصة للثغرات الأمنية في
ClawHub نفسه. تشمل الأمثلة الأخطاء في الموقع الإلكتروني أو API أو CLI أو السجل أو المصادقة أو
الفحص أو الإشراف أو حدود الثقة الخاصة بالتنزيل/التثبيت. لا تستخدم إرشادات ClawHub
للثغرات الأمنية في Skills أو Plugins التابعة لجهات خارجية.

البلاغات الجيدة محددة وقابلة للتنفيذ. يمكن أن تؤدي إساءة استخدام الإبلاغ بحد ذاتها إلى
إجراء على الحساب.

## مطالبات المؤسسات ونطاقات الأسماء

يجب أن تستخدم نزاعات ملكية المؤسسة أو العلامة التجارية أو نطاق الحزمة أو معرف المالك أو نطاق الاسم
عملية [مطالبات المؤسسات ونطاقات الأسماء](/ar/clawhub/namespace-claims)، وليس
تدفق الإبلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم تلك العملية عندما تحتاج إلى أن يراجع موظفو ClawHub دليلا غير حساس على أن
نطاق اسم ينبغي حجزه أو نقله أو إعادة تسميته أو إخفاؤه أو عزله أو جعله اسما مستعارا
أو مراجعته بطريقة أخرى. لا تدرج أسرارا أو مستندات خاصة أو ملفات قانونية خاصة
أو مستندات هوية شخصية أو رموز API أو رموز تحدي DNS في
قضية عامة.

## تعليقات الإشراف

قد تؤدي بعض النتائج الشديدة أو مشكلات السياسة إلى وضع ناشر أو قائمة تحت
تعليق إشراف. عندما يحدث ذلك، قد يخفى المحتوى المتأثر من
الاكتشاف العام أو قد تبدأ عمليات النشر المستقبلية مخفية إلى أن تتم مراجعة المشكلة.

تهدف تعليقات الإشراف إلى حماية المستخدمين بينما يحل ClawHub الحالات عالية المخاطر.
ويمكن أيضا رفعها عندما تتأكد نتيجة إيجابية كاذبة.

## القوائم المخفية أو المحظورة

قد تكون القائمة معلقة أو مخفية أو معزولة أو ملغاة أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبت الإصدار ما لم يحل المالك
المشكلة أو يعيده الإشراف.

قد يظل بإمكان المالكين رؤية التشخيصات لقوائمهم المعلقة أو المخفية. تساعد هذه
التشخيصات في شرح ما حدث وما الذي يجب تغييره قبل أن
تعود القائمة إلى الأسطح العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub إمكانية النشر. يمكن أن تؤدي إساءة الاستخدام الشديدة إلى
حظر الحساب أو إلغاء الرموز أو إخفاء المحتوى أو إزالة القوائم.
تتحقق إشارات ضغط إساءة استخدام الناشرين يوميا. يمكن للإشارات التي تصل إلى
عتبة الحظر المحتمل في ClawHub أن تطلق تحذيرا تلقائيا. إذا ظل الفحص المؤهل التالي
بعد مهلة التحذير يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيا.
تبقى إشارات المراجعة الأقل ثقة والمحدودة زمنيا خارج
الإنفاذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـClawHub. إذا بدأت مصادقة CLI
تفشل بعد إجراء على الحساب، فسجل الدخول إلى واجهة الويب لمراجعة
حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محظورا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا ذكر بريد إلكتروني أطلقه ماسح أن إصدار Skill أو Plugin خبيث،
فنزل نتائج الفحص المخزنة للإصدار المرسل المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى Plugins، أضف
`--kind plugin`. راجع مخرجات الفحص، وأصلح القائمة، وزد رقم الإصدار،
وارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الكاذبة وتحسين ثقة المستخدمين:

- أبق الأسماء والملخصات والوسوم وسجلات التغيير دقيقة
- صرح بمتغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المموهة
- اربط بالمصدر عندما يكون ذلك ممكنا
- استخدم التشغيلات الجافة قبل نشر Plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
