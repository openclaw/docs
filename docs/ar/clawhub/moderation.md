# الإشراف وسلامة الحساب

يتيح ClawHub النشر للجميع، لكن واجهات الاكتشاف العام والتثبيت لا تزال
تحتاج إلى ضوابط وقائية. تساعد البلاغات، وقيود الإشراف، والقوائم المخفية، وإجراءات الحساب
في حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن أو مضللًا أو
مخالفًا للسياسة.

تتناول هذه الصفحة الإشراف وحالة الحساب. للاطلاع على تسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[عمليات التدقيق الأمني](/clawhub/security-audits).

راجع أيضًا [الأمان](/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). للمخاوف المتعلقة بحقوق النشر أو غيرها من حقوق
المحتوى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجّلين الدخول الإبلاغ عن Skills وplugins والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الضارة
- البيانات الوصفية المضللة
- بيانات الاعتماد أو متطلبات الأذونات غير المعلنة
- تعليمات التثبيت المشبوهة
- انتحال الهوية
- التسجيلات سيئة النية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن Skill** في صفحة Skill، أو أمر/API الإبلاغ عن الحزم
بالنسبة إلى الحزم.

لا تستخدم بلاغات ClawHub للإبلاغ عن الثغرات الأمنية في الشيفرة المصدرية الخاصة بـSkill أو
plugin تابع لجهة خارجية. أبلغ عنها مباشرةً إلى الناشر أو مستودع الشيفرة المصدرية
المرتبط من القائمة. لا يتولى ClawHub صيانة أو تصحيح شيفرة Skills أوplugins
التابعة لجهات خارجية.

تنبيهات GitHub الأمنية الخاصة بـ`openclaw/clawhub` مخصصة للثغرات الأمنية في
ClawHub نفسه. تشمل الأمثلة الأخطاء في موقع الويب أو API أو CLI أو السجل أو المصادقة أو
الفحص أو الإشراف أو حدود الثقة المتعلقة بالتنزيل/التثبيت. لا تستخدم تنبيهات ClawHub
للثغرات الأمنية في Skills أوplugins التابعة لجهات خارجية.

تكون البلاغات الجيدة محددة وقابلة لاتخاذ إجراء بشأنها. وقد تؤدي إساءة استخدام الإبلاغ في حد ذاتها إلى
اتخاذ إجراء بحق الحساب.

## المطالبات بالمؤسسات ومساحات الأسماء

ينبغي استخدام عملية [المطالبات بالمؤسسات ومساحات الأسماء](/clawhub/namespace-claims) في النزاعات المتعلقة
بملكية المؤسسة أو العلامة التجارية أو نطاق الحزمة أو معرّف المالك أو مساحة الاسم، وليس
مسار الإبلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم هذه العملية عندما تحتاج إلى أن يراجع موظفو ClawHub إثباتًا غير حساس يفيد بأن
مساحة اسم ينبغي حجزها أو نقلها أو إعادة تسميتها أو إخفاؤها أو عزلها أو إنشاء اسم مستعار لها
أو مراجعتها بأي طريقة أخرى. لا تُدرج أسرارًا أو مستندات خاصة أو ملفات قانونية خاصة
أو وثائق هوية شخصية أو رموز API المميزة أو رموز تحدي DNS في
مشكلة عامة.

## قيود الإشراف

يمكن لبعض النتائج الخطيرة أو مشكلات السياسة أن تضع ناشرًا أو قائمة تحت
قيد الإشراف. عند حدوث ذلك، قد يُخفى المحتوى المتأثر عن
الاكتشاف العام، أو قد تبدأ عمليات النشر المستقبلية مخفية حتى تُراجع المشكلة.

تهدف قيود الإشراف إلى حماية المستخدمين بينما يعالج ClawHub الحالات
عالية المخاطر. ويمكن أيضًا رفعها عند التأكد من وجود نتيجة إيجابية زائفة.

## القوائم المخفية أو المحظورة

قد تكون القائمة موقوفة أو مخفية أو معزولة أو ملغاة أو غير متاحة بطريقة أخرى على
واجهات التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبّت الإصدار ما لم يعالج المالك
المشكلة أو يعِدْه الإشراف إلى حالته السابقة.

قد يظل بإمكان المالكين رؤية معلومات التشخيص الخاصة بقوائمهم الموقوفة أو المخفية. تساعد هذه
المعلومات في توضيح ما حدث وما يجب تغييره قبل أن تتمكن
القائمة من العودة إلى الواجهات العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub صلاحية النشر. ويمكن أن تؤدي إساءة الاستخدام الجسيمة إلى
حظر الحساب أو إلغاء الرموز المميزة أو إخفاء المحتوى أو إزالة القوائم.
تُفحص يوميًا إشارات الضغط المرتبطة بإساءة استخدام الناشر. ويمكن للإشارات التي تبلغ
حد الحظر المحتمل في ClawHub أن تؤدي إلى تحذير تلقائي. وإذا ظل الفحص التالي
المؤهل بعد الموعد النهائي للتحذير يضع الناشر ضمن
حد الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيًا.
تظل إشارات المراجعة الزمنية المحدودة والأقل موثوقية خارج
نطاق الإنفاذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطّلة استخدام رموز API المميزة الخاصة بـClawHub. إذا بدأت مصادقة CLI
بالفشل بعد اتخاذ إجراء بحق الحساب، فسجّل الدخول إلى واجهة الويب لمراجعة
حالة الحساب. إذا كان تسجيل الدخول أو الوصول المعتاد عبر CLI محظورًا بسبب حظر الحساب أو تعطيله،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا ذكرت رسالة بريد إلكتروني ناتجة عن الماسح إصدار Skill أوplugin بوصفه ضارًا،
فنزّل نتائج الفحص المخزنة للإصدار المرسَل المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى plugins، أضف
`--kind plugin`. راجع مخرجات الفحص، وأصلح القائمة، وزِد رقم
الإصدار، وارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الزائفة وتعزيز ثقة المستخدمين:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغييرات
- أعلن عن متغيرات البيئة والأذونات المطلوبة
- تجنّب أوامر التثبيت المبهمة
- أدرج رابطًا إلى الشيفرة المصدرية متى أمكن
- استخدم عمليات التشغيل التجريبي قبل نشر plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
