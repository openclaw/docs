# الإشراف وسلامة الحساب

ClawHub مفتوح للنشر، لكن أسطح الاكتشاف والتثبيت العامة ما زالت
تحتاج إلى ضوابط وقائية. تساعد البلاغات، وحالات التعليق الإشرافي، والقوائم المخفية، وإجراءات الحساب
على حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن أو مضللا أو خارجا
عن السياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. لتسميات التدقيق مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/clawhub/security-audits).

راجع أيضا [الأمان](/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). للمخاوف المتعلقة بحقوق النشر أو حقوق المحتوى
الأخرى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين المسجلين الإبلاغ عن المهارات والمكونات الإضافية والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الخبيثة
- البيانات الوصفية المضللة
- بيانات الاعتماد أو متطلبات الأذونات غير المعلنة
- تعليمات التثبيت المشبوهة
- انتحال الهوية
- التسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن المهارة** في صفحة المهارة، أو أمر/واجهة API
الإبلاغ عن الحزم.

لا تستخدم بلاغات ClawHub للثغرات في الشيفرة المصدرية الخاصة بمهارة أو
مكون إضافي تابع لجهة خارجية. أبلغ عنها مباشرة إلى الناشر أو مستودع المصدر
المرتبط من القائمة. لا يصون ClawHub شيفرة المهارات أو المكونات الإضافية
الخاصة بجهات خارجية ولا يرقعها.

إرشادات GitHub الأمنية الخاصة بـ `openclaw/clawhub` مخصصة للثغرات في
ClawHub نفسه. تشمل الأمثلة الأخطاء في الموقع الإلكتروني أو API أو CLI أو السجل أو المصادقة
أو الفحص أو الإشراف أو حدود الثقة للتنزيل/التثبيت. لا تستخدم إرشادات ClawHub
للثغرات في المهارات أو المكونات الإضافية الخاصة بجهات خارجية.

البلاغات الجيدة محددة وقابلة للتنفيذ. إساءة استخدام الإبلاغ يمكن أن تؤدي هي نفسها إلى
إجراء على الحساب.

## مطالبات المؤسسات والنطاقات الاسمية

ينبغي أن تستخدم نزاعات ملكية المؤسسة أو العلامة التجارية أو نطاق الحزمة أو معرّف المالك أو النطاق الاسمي
عملية [مطالبات المؤسسات والنطاقات الاسمية](/clawhub/namespace-claims)، وليس
تدفق الإبلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم تلك العملية عندما تحتاج إلى أن يراجع فريق ClawHub دليلا غير حساس على أن
نطاقا اسميا يجب حجزه أو نقله أو إعادة تسميته أو إخفاؤه أو عزله أو إنشاء اسم مستعار له
أو مراجعته بطريقة أخرى. لا تدرج أسرارا أو مستندات خاصة أو ملفات قانونية خاصة
أو مستندات هوية شخصية أو رموز API أو رموز تحدي DNS في
قضية عامة.

## التعليقات الإشرافية

يمكن لبعض النتائج الخطيرة أو مشكلات السياسة أن تضع ناشرا أو قائمة تحت
تعليق إشرافي. عند حدوث ذلك، قد يخفى المحتوى المتأثر من الاكتشاف العام
أو قد تبدأ عمليات النشر المستقبلية مخفية حتى تتم مراجعة المشكلة.

تهدف التعليقات الإشرافية إلى حماية المستخدمين بينما يحل ClawHub الحالات عالية المخاطر.
ويمكن رفعها أيضا عند تأكيد وجود نتيجة إيجابية كاذبة.

## القوائم المخفية أو المحظورة

قد تكون القائمة معلقة أو مخفية أو معزولة أو ملغاة أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبت الإصدار ما لم يحل المالك
المشكلة أو يعيده الإشراف.

قد يظل المالكون قادرين على رؤية التشخيصات لقوائمهم المعلقة أو المخفية. تساعد هذه
التشخيصات في شرح ما حدث وما يجب تغييره قبل أن
تعود القائمة إلى الأسطح العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub حق الوصول إلى النشر. يمكن أن تؤدي الإساءة الشديدة
إلى حظر الحسابات أو إبطال الرموز أو إخفاء المحتوى أو إزالة القوائم.
تتحقق إشارات ضغط إساءة الناشرين يوميا. يمكن للإشارات التي تصل إلى
عتبة الحظر المحتمل في ClawHub أن تطلق تحذيرا تلقائيا. إذا كان الفحص
المؤهل التالي بعد موعد التحذير ما يزال يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيا.
تبقى إشارات المراجعة الزمنية محدودة النطاق ومنخفضة الثقة خارج
الإنفاذ التلقائي.

لا يمكن للحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـ ClawHub. إذا بدأت مصادقة CLI
بالفشل بعد إجراء على الحساب، فسجل الدخول إلى واجهة الويب لمراجعة
حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محظورا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا سمى بريد إلكتروني ناتج عن ماسح إصدار مهارة أو مكون إضافي بأنه خبيث،
فنزل نتائج الفحص المخزنة للإصدار المرسل المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى المكونات الإضافية، أضف
`--kind plugin`. راجع مخرجات الفحص، وأصلح القائمة، وزد رقم الإصدار،
ثم ارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الكاذبة وتحسين ثقة المستخدمين:

- حافظ على دقة الأسماء والملخصات والوسوم وسجلات التغيير
- أعلن متغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنا
- استخدم التشغيل التجريبي قبل نشر المكونات الإضافية
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
