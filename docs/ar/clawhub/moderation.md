# الإشراف وسلامة الحساب

يفتح ClawHub باب النشر، لكن أسطح الاكتشاف والتثبيت العامة ما زالت
تحتاج إلى ضوابط. تساعد البلاغات، وتعليقات الإشراف، والقوائم المخفية، وإجراءات الحساب
على حماية المستخدمين عندما يبدو إصدار أو حساب غير آمن أو مضللاً أو خارج
السياسة.

تغطي هذه الصفحة الإشراف وحالة الحساب. للتسميات التدقيقية مثل
`Pass` و`Review` و`Warn` و`Malicious` ومستوى المخاطر، راجع
[تدقيقات الأمان](/clawhub/security-audits).

راجع أيضًا [الأمان](/clawhub/security) و
[الاستخدام المقبول](/clawhub/acceptable-usage). ولمخاوف حقوق النشر أو حقوق
المحتوى الأخرى، استخدم [طلبات حقوق المحتوى](/clawhub/content-rights).

## البلاغات

يمكن للمستخدمين الذين سجّلوا الدخول الإبلاغ عن Skills وPlugins والحزم.

استخدم بلاغات ClawHub فقط لمحتوى السوق غير الآمن، مثل:

- القوائم الخبيثة
- البيانات الوصفية المضللة
- بيانات الاعتماد أو متطلبات الأذونات غير المصرّح بها
- تعليمات التثبيت المشبوهة
- انتحال الهوية
- التسجيلات بسوء نية أو إساءة استخدام العلامات التجارية
- المحتوى الذي ينتهك [الاستخدام المقبول](/clawhub/acceptable-usage)

استخدم زر **الإبلاغ عن Skill** في صفحة Skill، أو أمر/واجهة API الإبلاغ عن الحزم
للحزم.

لا تستخدم بلاغات ClawHub للثغرات في شيفرة المصدر الخاصة بـ Skill أو
Plugin تابع لجهة خارجية. أبلغ عنها مباشرةً إلى الناشر أو مستودع المصدر
المرتبط من القائمة. لا يصون ClawHub شيفرة Skills أو Plugins التابعة لجهات خارجية
ولا يرقّعها.

استشارات أمان GitHub الخاصة بـ `openclaw/clawhub` مخصصة للثغرات في
ClawHub نفسه. تشمل الأمثلة الأخطاء في الموقع الإلكتروني، أو API، أو CLI، أو السجل، أو المصادقة،
أو الفحص، أو الإشراف، أو حدود الثقة الخاصة بالتنزيل/التثبيت. لا تستخدم استشارات ClawHub
للثغرات في Skills أو Plugins التابعة لجهات خارجية.

البلاغات الجيدة محددة وقابلة للتنفيذ. وقد تؤدي إساءة استخدام الإبلاغ بحد ذاتها إلى
إجراء على الحساب.

## مطالبات المؤسسة ونطاق الاسم

يجب أن تستخدم نزاعات ملكية المؤسسة أو العلامة التجارية أو نطاق الحزمة أو معرّف المالك أو نطاق الاسم
عملية [مطالبات المؤسسة ونطاق الاسم](/clawhub/namespace-claims)، وليس تدفق
الإبلاغ داخل المنتج أو نموذج استئناف الحساب.

استخدم هذه العملية عندما تحتاج إلى أن يراجع فريق ClawHub إثباتًا غير حساس بأن
نطاق اسم يجب أن يُحجز أو يُنقل أو يُعاد تسميته أو يُخفى أو يُحجر أو يُمنح اسمًا مستعارًا
أو يُراجع بطريقة أخرى. لا تدرج أسرارًا، أو مستندات خاصة، أو ملفات قانونية خاصة،
أو مستندات هوية شخصية، أو رموز API، أو رموز تحدي DNS في
مشكلة عامة.

## تعليقات الإشراف

يمكن لبعض النتائج الخطيرة أو مشكلات السياسة أن تضع ناشرًا أو قائمة تحت
تعليق إشراف. عند حدوث ذلك، قد يُخفى المحتوى المتأثر من الاكتشاف العام
أو قد تبدأ عمليات النشر المستقبلية مخفية حتى تتم مراجعة المشكلة.

تهدف تعليقات الإشراف إلى حماية المستخدمين بينما يحل ClawHub الحالات عالية المخاطر.
ويمكن رفعها أيضًا عند تأكيد نتيجة إيجابية كاذبة.

## القوائم المخفية أو المحظورة

قد تكون القائمة معلّقة أو مخفية أو محجورة أو ملغاة أو غير متاحة بطريقة أخرى على
أسطح التثبيت العامة.

إذا رأيت إحدى هذه الحالات، فلا تثبّت الإصدار ما لم يحل المالك
المشكلة أو يعيده الإشراف.

قد يظل بإمكان المالكين رؤية التشخيصات الخاصة بقوائمهم المعلّقة أو المخفية. تساعد هذه
التشخيصات في شرح ما حدث وما يجب تغييره قبل أن تتمكن
القائمة من العودة إلى الأسطح العامة.

## الحظر وحالة الحساب

قد تفقد الحسابات التي تنتهك سياسة ClawHub إمكانية النشر. وقد تؤدي الإساءة الجسيمة إلى
حظر الحساب، أو إلغاء الرموز، أو إخفاء المحتوى، أو إزالة القوائم.
تُفحص إشارات ضغط إساءة الناشرين يوميًا. ويمكن للإشارات التي تصل إلى
عتبة الحظر المحتمل في ClawHub أن تطلق تحذيرًا تلقائيًا. إذا ظل الفحص التالي
المؤهل بعد مهلة التحذير يضع الناشر ضمن
عتبة الحظر المحتمل، فقد يطبق ClawHub إجراء الحساب تلقائيًا.
وتبقى إشارات المراجعة منخفضة الثقة والمحدودة زمنيًا خارج
التنفيذ التلقائي.

لا تستطيع الحسابات المحذوفة أو المحظورة أو المعطلة استخدام رموز API الخاصة بـ ClawHub. إذا بدأت مصادقة CLI
بالفشل بعد إجراء على الحساب، فسجّل الدخول إلى واجهة الويب لمراجعة
حالة الحساب. إذا كان تسجيل الدخول أو الوصول العادي عبر CLI محظورًا بسبب حظر أو حساب معطل،
فاستخدم [نموذج استئناف ClawHub](https://appeals.openclaw.ai/) لمراجعة الاسترداد.

إذا سمّت رسالة بريد إلكتروني ناتجة عن ماسح إصدار Skill أو Plugin على أنه خبيث،
فنزّل نتائج الفحص المخزنة للإصدار المقدّم المحظور:
`clawhub scan download <slug> --version <version>`. بالنسبة إلى Plugins، أضف
`--kind plugin`. راجع مخرجات الفحص، وأصلح القائمة، وزد رقم الإصدار،
ثم ارفع الإصدار المصحح.

## إرشادات الناشرين

لتقليل النتائج الإيجابية الكاذبة وتحسين ثقة المستخدمين:

- أبقِ الأسماء والملخصات والوسوم وسجلات التغييرات دقيقة
- صرّح بمتغيرات البيئة والأذونات المطلوبة
- تجنب أوامر التثبيت المبهمة
- اربط بالمصدر عندما يكون ذلك ممكنًا
- استخدم التشغيلات التجريبية قبل نشر Plugins
- استجب بوضوح إذا سأل المستخدمون أو المشرفون عن سلوك الإصدار
