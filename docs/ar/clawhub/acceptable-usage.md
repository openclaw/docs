# الاستخدام المقبول

تصف هذه الصفحة أنواع Skills والمحتوى المقبولة لدى ClawHub، ومسارات إساءة الاستخدام التي لن تستضيفها.

هذه القواعد عملية عن قصد. ما يهمنا أكثر هو مسارات إساءة الاستخدام من البداية إلى النهاية، وليس مجرد كلمات مفتاحية معزولة. إذا كانت المهارة مبنية للتحايل على الدفاعات، أو إساءة استخدام المنصات، أو الاحتيال على الأشخاص، أو انتهاك الخصوصية، أو تمكين سلوك غير توافقي، فلا مكان لها على ClawHub.

## أنماط حديثة نقبلها صراحة

- أعمال الواجهة الأمامية وأنظمة التصميم التي تستخدم مكونات حقيقية، ورموزًا دلالية، وحالات قابلة للوصول، وتدفقات مستخدم مختبرة.
- تركيب shadcn/ui الذي يستخدم مكونات المصدر المثبتة، وأسماء المشروع المستعارة، والمتغيرات الموثقة بدلًا من ترميز مخصص لمرة واحدة.
- تحويل UI5 من JavaScript إلى TypeScript مع الحفاظ على التعليقات، واستخدام أنواع UI5 ملموسة، وإبقاء واجهات عناصر التحكم المولدة قابلة للمراجعة.
- مراجعة الأمان الدفاعية، وأدوات الإشراف، ومطالبات اكتشاف الإساءة التي تعرض الأدلة وتبقي حدود موافقة البشر واضحة.
- أتمتة مسارات العمل القائمة على الموافقة لحسابات شخصية أو حسابات فريق باستخدام بيانات اعتماد صريحة، وإعداد شفاف، وأوضاع تشغيل تجريبي أو معاينة.
- التوثيق، وأدلة تشغيل الترحيل، وأدوات المطورين، وتجهيزات الاختبار المحددة بالبرنامج الذي تدعمه.

## غير مقبول

- مسارات عمل تجاوز الأمان أو الوصول غير المصرح به.
  - أمثلة: تجاوز المصادقة، الاستيلاء على الحسابات، تجاوز CAPTCHA، التحايل على Cloudflare أو أنظمة مكافحة الروبوتات، تجاوز حدود المعدل، الاستخراج الخفي المصمم لهزيمة وسائل الحماية، الاستيلاء على مكالمة مباشرة أو وكيل، سرقة جلسات قابلة لإعادة الاستخدام، الموافقة التلقائية على تدفقات الاقتران للمستخدمين غير المعتمدين.

- إساءة استخدام المنصات والتحايل على الحظر.
  - أمثلة: حسابات خفية بعد الحظر، تهيئة الحسابات أو زراعتها، تفاعل مزيف، تنمية الكارما أو المتابعين، أتمتة متعددة الحسابات، النشر الجماعي، روبوتات البريد المزعج، أتمتة الأسواق أو الشبكات الاجتماعية المصممة لتجنب الاكتشاف.

- الاحتيال، وعمليات النصب، ومسارات العمل المالية الخادعة.
  - أمثلة: شهادات مزيفة، فواتير مزيفة، تدفقات دفع خادعة، تواصل احتيالي، دليل اجتماعي مزيف، أدوات تتيح الإنفاق أو التحصيل دون موافقة بشرية واضحة وضوابط شفافة، أو مسارات عمل هويات اصطناعية مبنية لإنشاء حسابات بغرض الاحتيال.

- الاستخراج، أو الإثراء، أو المراقبة المنتهكة للخصوصية.
  - أمثلة: استخراج تفاصيل الاتصال على نطاق واسع للبريد المزعج، التشهير بالمعلومات الشخصية، المطاردة، استخراج العملاء المحتملين مقرونًا بتواصل غير مطلوب، المراقبة السرية، البحث عن الوجوه أو المطابقة البيومترية المستخدمة دون موافقة واضحة، أو شراء بيانات مسربة أو تفريغات اختراق أو نشرها أو تنزيلها أو تحويلها إلى عمليات.

- الانتحال غير التوافقي أو التلاعب الخادع بالهوية.
  - أمثلة: تبديل الوجوه، التوائم الرقمية، الشخصيات المزيفة، المؤثرون المستنسخون، أو أدوات أخرى للتلاعب بالهوية تُستخدم للانتحال أو التضليل.

- المحتوى الجنسي الصريح وتوليد محتوى بالغين مع تعطيل السلامة.
  - أمثلة: توليد صور/فيديو/محتوى NSFW، أغلفة محتوى بالغين حول واجهات API تابعة لجهات خارجية، أو Skills يكون غرضها الأساسي المحتوى الجنسي الصريح.

- متطلبات تنفيذ مخفية أو غير آمنة أو مضللة.
  - أمثلة: أوامر تثبيت مبهمة، `curl | sh`، متطلبات أسرار غير معلنة، استخدام مفاتيح خاصة غير معلن، تنفيذ `npx @latest` عن بُعد دون قابلية واضحة للمراجعة، بيانات وصفية مضللة تخفي ما تحتاجه المهارة فعليًا للتشغيل.

## أنماط حديثة لا نقبلها صراحة

- “إنشاء حسابات بائع خفية بعد الحظر من الأسواق.”
- “تعديل اقتران Telegram بحيث يتلقى المستخدمون غير المعتمدين رموز الاقتران تلقائيًا.”
- “تنمية حسابات Reddit/Twitter بأتمتة غير قابلة للاكتشاف.”
- “إنشاء شهادات مهنية أو فواتير لاستخدام عشوائي.”
- “توليد محتوى NSFW مع تعطيل فحوصات السلامة.”
- “استخراج العملاء المحتملين، وإثراء جهات الاتصال، وإطلاق تواصل بارد على نطاق واسع.”
- “شراء بيانات مسربة أو تفريغات اختراق أو نشرها أو تنزيلها.”
- “إنشاء حسابات بريد إلكتروني أو حسابات اجتماعية بالجملة باستخدام هويات اصطناعية أو حل CAPTCHA.”

## ملاحظات للمراجعين

- السياق مهم. يمكن أن يكون الموضوع نفسه مشروعًا في إعداد دفاعي ضيق أو قائم على الموافقة، وغير مقبول عندما يُحزم كمسار عمل لإساءة الاستخدام.
- ينبغي أن نميل إلى اتخاذ إجراء عندما تكون المهارة محسّنة بوضوح للتحايل، أو الخداع، أو الاستخدام غير التوافقي.
- التحميلات المتكررة ضمن هذه الفئات تُعد سببًا لإخفاء المحتوى وحظر الحساب.

## الإنفاذ

- قد نخفي Skills المخالفة أو نزيلها أو نحذفها حذفًا نهائيًا.
- قد نلغي الرموز، ونحذف المحتوى المرتبط حذفًا ناعمًا، ونحظر المخالفين المتكررين أو الجسيمين.
- لا نضمن إنفاذًا يبدأ بالتحذير في حالات الإساءة الواضحة.
