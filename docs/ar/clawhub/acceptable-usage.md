# الاستخدام المقبول

تصف هذه الصفحة أنواع المهارات والمحتوى التي يقبلها ClawHub، وسير عمل إساءة الاستخدام التي لن يستضيفها.

هذه القواعد عملية عن قصد. ما يهمنا أكثر هو سير عمل إساءة الاستخدام من البداية إلى النهاية، وليس مجرد كلمات مفتاحية معزولة. إذا كانت المهارة مبنية لتجاوز الدفاعات، أو إساءة استخدام المنصات، أو الاحتيال على الأشخاص، أو انتهاك الخصوصية، أو تمكين سلوك غير رضائي، فمكانها ليس على ClawHub.

## أنماط حديثة نقبلها صراحة

- عمل الواجهة الأمامية وأنظمة التصميم الذي يستخدم مكونات حقيقية، ورموزًا دلالية، وحالات ميسّرة الوصول، وتدفقات مستخدم مختبرة.
- تركيب shadcn/ui الذي يستخدم مكونات المصدر المثبتة، وأسماء المشروع البديلة، والمتغيرات الموثقة بدلًا من ترميز لمرة واحدة.
- تحويل UI5 من JavaScript إلى TypeScript مع الحفاظ على التعليقات، واستخدام أنواع UI5 محددة، وإبقاء واجهات عناصر التحكم المُولّدة قابلة للمراجعة.
- مراجعة الأمن الدفاعية، وأدوات الإشراف، ومطالبات اكتشاف الإساءة التي تعرض الأدلة وتحافظ على حدود الموافقة البشرية واضحة.
- أتمتة سير العمل القائمة على الموافقة لحسابات شخصية أو حسابات فريق باستخدام بيانات اعتماد صريحة، وإعداد شفاف، وأوضاع تشغيل تجريبي أو معاينة.
- التوثيق، وأدلة تشغيل الترحيل، وأدوات المطورين، وتجهيزات الاختبار المحددة النطاق للبرمجيات التي تدعمها.

## غير مقبول

- سير عمل تجاوز الأمن أو الوصول غير المصرح به.
  - أمثلة: تجاوز المصادقة، الاستيلاء على الحسابات، تجاوز CAPTCHA، تفادي Cloudflare أو مكافحة الروبوتات، تجاوز حدود المعدل، الكشط الخفي المصمم لهزيمة الحمايات، الاستيلاء على مكالمة مباشرة أو وكيل مباشر، سرقة جلسات قابلة لإعادة الاستخدام، الموافقة التلقائية على تدفقات الاقتران للمستخدمين غير المعتمدين.

- إساءة استخدام المنصات وتفادي الحظر.
  - أمثلة: حسابات خفية بعد الحظر، تهيئة الحسابات أو زراعتها، تفاعل مزيف، تنمية الكارما أو المتابعين، أتمتة الحسابات المتعددة، النشر الجماعي، روبوتات الرسائل المزعجة، أتمتة الأسواق أو الشبكات الاجتماعية المصممة لتجنب الاكتشاف.

- الاحتيال، وعمليات النصب، وسير العمل المالي المخادع.
  - أمثلة: شهادات مزيفة، فواتير مزيفة، تدفقات دفع مخادعة، تواصل احتيالي، دليل اجتماعي مزيف، أدوات تمكّن الإنفاق أو تحصيل الرسوم دون موافقة بشرية واضحة وضوابط شفافة، أو سير عمل هويات اصطناعية مبنية لإنشاء حسابات للاحتيال.

- الكشط أو الإثراء أو المراقبة المنتهكة للخصوصية.
  - أمثلة: كشط تفاصيل الاتصال على نطاق واسع للرسائل المزعجة، التشهير بالمعلومات الشخصية، المطاردة، استخراج العملاء المحتملين مقرونًا بتواصل غير مرغوب فيه، المراقبة السرية، البحث بالوجه أو المطابقة الحيوية المستخدمة دون موافقة واضحة، أو شراء البيانات المسربة أو تفريغات الاختراق أو نشرها أو تنزيلها أو تحويلها إلى عمليات تشغيلية.

- انتحال الهوية غير الرضائي أو التلاعب المخادع بالهوية.
  - أمثلة: تبديل الوجوه، التوائم الرقمية، الشخصيات المزيفة، المؤثرون المستنسخون، أو أدوات تلاعب أخرى بالهوية تُستخدم للانتحال أو التضليل.

- المحتوى الجنسي الصريح وتوليد محتوى للبالغين مع تعطيل إجراءات السلامة.
  - أمثلة: توليد صور أو فيديوهات أو محتوى NSFW، أغلفة محتوى للبالغين حول واجهات API تابعة لجهات خارجية، أو مهارات يكون غرضها الأساسي محتوى جنسيًا صريحًا.

- متطلبات تنفيذ مخفية أو غير آمنة أو مضللة.
  - أمثلة: أوامر تثبيت مبهمة، `curl | sh`، متطلبات أسرار غير معلنة، استخدام مفاتيح خاصة غير معلن، تنفيذ `npx @latest` عن بُعد دون قابلية مراجعة واضحة، بيانات وصفية مضللة تخفي ما تحتاجه المهارة فعليًا للتشغيل.

## أنماط حديثة لا نقبلها صراحة

- "إنشاء حسابات بائع خفية بعد حظر الأسواق."
- "تعديل اقتران Telegram بحيث يتلقى المستخدمون غير المعتمدين رموز الاقتران تلقائيًا."
- "تنمية حسابات Reddit/Twitter باستخدام أتمتة غير قابلة للاكتشاف."
- "إنشاء شهادات أو فواتير مهنية لاستخدام عشوائي."
- "توليد محتوى NSFW مع تعطيل فحوصات السلامة."
- "كشط العملاء المحتملين، وإثراء جهات الاتصال، وإطلاق تواصل بارد على نطاق واسع."
- "شراء البيانات المسربة أو تفريغات الاختراق أو نشرها أو تنزيلها."
- "إنشاء حسابات بريد إلكتروني أو حسابات اجتماعية بكميات كبيرة باستخدام هويات اصطناعية أو حل CAPTCHA."

## ملاحظات للمراجعين

- السياق مهم. قد يكون الموضوع نفسه مشروعًا في إعداد دفاعي ضيق أو قائم على الموافقة، وغير مقبول عندما يُغلّف كسير عمل إساءة استخدام.
- ينبغي أن نميل إلى اتخاذ إجراء عندما تكون المهارة محسّنة بوضوح للتفادي أو الخداع أو الاستخدام غير الرضائي.
- التحميلات المتكررة ضمن هذه الفئات تُعد سببًا لإخفاء المحتوى وحظر الحساب.

## الإنفاذ

- قد نخفي المهارات المخالفة أو نزيلها أو نحذفها حذفًا نهائيًا.
- قد نلغي الرموز، ونحذف المحتوى المرتبط حذفًا مرنًا، ونحظر المخالفين المتكررين أو الجسيمين.
- لا نضمن إنفاذًا يسبقُه تحذير في حالات الإساءة الواضحة.
