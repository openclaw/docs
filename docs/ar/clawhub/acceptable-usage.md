# الاستخدام المقبول

يستضيف ClawHub Skills وPlugins والحزم وبيانات marketplace الوصفية لـ OpenClaw.
استخدم هذه الصفحة لتحديد ما إذا كان المحتوى أو سلوك النشر ينتمي إلى
ClawHub.

تنطبق هذه القواعد على ما يفعله الإدراج، وما يطلب من المستخدمين تشغيله، وكيف
يمثل نفسه، وكيف يستخدم الناشرون أسطح الاكتشاف والتثبيت والثقة في ClawHub. للاطلاع على حالات الإشراف ووضع الحساب، راجع
[الإشراف وسلامة الحساب](/ar/clawhub/moderation). بالنسبة لمطالبات حقوق النشر أو الحقوق الأخرى،
راجع [طلبات حقوق المحتوى](/ar/clawhub/content-rights).

## المحتوى المسموح به

يرحب ClawHub بالمحتوى المفيد والمفهوم والمنشور بحسن
نية.

| الفئة                                         | مسموح به عندما                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| إنتاجية المطورين                           | يساعد الإدراج المستخدمين على بناء البرمجيات أو اختبارها أو ترحيلها أو تصحيحها أو توثيقها أو تشغيلها.                                               |
| واجهة المستخدم والبيانات وتدفقات عمل الأتمتة               | يكون النطاق واضحًا، وبيانات الاعتماد المطلوبة صريحة، وتشمل الإجراءات الخطرة مسارات للمراجعة أو التشغيل التجريبي أو المعاينة أو التأكيد. |
| الأمن الدفاعي والإشراف ومراجعة الإساءة | تُقدَّم الأداة بوصفها مخصصة للمراجعة المصرح بها، وتحافظ على الأدلة، وتُبقي حدود الموافقة البشرية واضحة.                          |
| تدفقات العمل الشخصية أو الخاصة بالفريق                       | يستخدم تدفق العمل حسابات قائمة على الموافقة، وإعدادًا شفافًا، وأذونات صريحة.                                            |
| الكتالوجات المُصانة                              | يكون كل إدراج مميزًا ومفيدًا وموصوفًا بدقة ومُصانًا بدرجة معقولة.                                                |

السياق مهم. يمكن أن يكون الموضوع نفسه مقبولًا في بيئة دفاعية ضيقة أو
قائمة على الموافقة، وغير مقبول عندما يُغلَّف كتدفق عمل للإساءة.

أو انتهاك الحقوق.

| إساءة استخدام المنصة وتجاوز الحظر                              | حسابات خفية بعد الحظر، أو تهيئة الحسابات أو زراعتها، أو التفاعل الزائف، أو أتمتة الحسابات المتعددة، أو النشر الجماعي، أو روبوتات الرسائل المزعجة، أو الأتمتة المصممة لتجنب الاكتشاف.                                                                                                                                          |

## سلوك marketplace غير المسموح به

يراجع ClawHub أيضًا كيفية استخدام الناشرين لـ marketplace. لا تستخدم ClawHub
للتلاعب بالاكتشاف أو المقاييس أو إشارات الثقة أو أنظمة الإشراف أو انتباه
المستخدمين.

يتضمن سلوك marketplace غير المسموح به:

- نشر أعداد كبيرة بالجملة من الإدراجات منخفضة الجهد أو المكررة أو النائبة أو
  المولدة آليًا التي لا يبدو أن لها قيمة حقيقية للمستخدم
- إغراق أسطح البحث أو الفئات بمهارات أو Plugins شبه متطابقة
- نشر مئات الإدراجات مع استخدام أو صيانة أو وضوح مصدر أو تمييز ذي معنى
  قليل أو معدوم
- تضخيم عمليات التثبيت أو التنزيلات أو النجوم أو مقاييس التفاعل الأخرى
  بشكل مصطنع من خلال الأتمتة أو حلقات التثبيت الذاتي أو الحسابات المزيفة أو النشاط
  المنسق أو التفاعل المدفوع أو أي سلوك آخر غير عضوي
- إنشاء الحسابات أو تدويرها لتفادي الإشراف أو الحظر أو حدود الناشرين أو
  مراجعة marketplace
- تضليل المستخدمين بشأن الملكية أو المصدر أو القدرات أو الوضع الأمني أو
  متطلبات التثبيت أو الارتباط بمشروع أو ناشر آخر
- رفع محتوى مرارًا سبق إخفاؤه أو إزالته أو حظره
  دون إصلاح المشكلة الأساسية

النشر بكميات كبيرة ليس إساءة تلقائيًا. الكتالوجات الكبيرة مقبولة
عندما تكون الإدراجات مختلفة بشكل ذي معنى، وموصوفة بدقة، ومُصانة،
ويستخدمها مستخدمون حقيقيون. تصبح الكتالوجات الكبيرة مشكلة ثقة وسلامة عندما
يقترن الحجم بإدراجات ضعيفة أو مكررة أو مضللة أو غير مُصانة أو
مروَّج لها بشكل مصطنع.

## حقوق المحتوى

إذا كنت تعتقد أن محتوى على ClawHub ينتهك حقوق النشر الخاصة بك أو حقوقًا أخرى، فاستخدم
[طلبات حقوق المحتوى](/ar/clawhub/content-rights). لا تستخدم بلاغات marketplace
العادية لمطالبات حقوق النشر أو الحقوق إلا إذا كان الإدراج أيضًا غير آمن
أو ضارًا أو مضللًا.

## المراجعة والإنفاذ

قد يستخدم ClawHub فحوصات مؤتمتة، وإشارات إحصائية للإساءة، وبلاغات المستخدمين، و
مراجعة الموظفين لتحديد المحتوى غير الآمن أو سلوك النشر المسيء. لا تثبت الإشارة
الإساءة بحد ذاتها؛ بل تساعد ClawHub في تحديد ما يحتاج إلى مراجعة.

قد نقوم بما يلي:

- إخفاء الإدراجات المخالفة أو تعليقها أو إزالتها أو حذفها حذفًا مرنًا أو، حيث يكون ذلك مدعومًا لنوع المورد،
  حذفها حذفًا نهائيًا
- حظر التنزيلات أو عمليات التثبيت للإصدارات غير الآمنة
- إلغاء رموز API
- حذف المحتوى المرتبط حذفًا مرنًا
- تقييد إمكانية النشر
- حظر المخالفين المتكررين أو الجسيمين

لا نضمن الإنفاذ بعد التحذير أولًا في حالات الإساءة الواضحة. راجع
[الإشراف وسلامة الحساب](/ar/clawhub/moderation) للاطلاع على البلاغات، وتعليقات الإشراف،
والإدراجات المخفية، والحظر، ووضع الحساب.
