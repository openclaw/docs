# الاستخدام المقبول

تصف هذه الصفحة أنواع Skills والمحتوى الذي يقبله ClawHub، ومسارات إساءة الاستخدام التي لن يستضيفها.

هذه القواعد عملية عن قصد. ما يهمنا أكثر هو مسارات إساءة الاستخدام من البداية إلى النهاية، وليس الكلمات المفتاحية المعزولة فقط. إذا كانت Skill مبنية لتجاوز الدفاعات، أو إساءة استخدام المنصات، أو الاحتيال على الأشخاص، أو انتهاك الخصوصية، أو تمكين سلوك غير رضائي، فلا مكان لها على ClawHub.

## الأنماط الحديثة التي نقبلها صراحة

- أعمال الواجهة الأمامية وأنظمة التصميم التي تستخدم مكونات حقيقية، ورموزًا دلالية، وحالات قابلة للوصول، وتدفقات مستخدم مختبرة.
- تركيب shadcn/ui الذي يستخدم مكونات مصدرية مثبتة، وأسماء مستعارة للمشروع، ومتغيرات موثقة بدلًا من ترميز مخصص لمرة واحدة.
- تحويل UI5 من JavaScript إلى TypeScript مع الحفاظ على التعليقات، واستخدام أنواع UI5 ملموسة، وإبقاء واجهات عناصر التحكم المولدة قابلة للمراجعة.
- مراجعة الأمان الدفاعية، وأدوات الإشراف، ومطالبات اكتشاف إساءة الاستخدام التي تعرض الأدلة وتحافظ على حدود موافقة البشر بوضوح.
- أتمتة سير العمل القائمة على الموافقة للحسابات الشخصية أو حسابات الفريق مع بيانات اعتماد صريحة، وإعداد شفاف، وأوضاع تشغيل تجريبي أو معاينة.
- الوثائق، ودلائل الترحيل التشغيلية، وأدوات المطورين، وتجهيزات الاختبار المحددة النطاق للبرمجيات التي تدعمها.

## غير مقبول

- مسارات عمل تجاوز الأمان أو الوصول غير المصرح به.
  - أمثلة: تجاوز المصادقة، الاستيلاء على الحسابات، تجاوز CAPTCHA، تجاوز Cloudflare أو أنظمة مكافحة الروبوتات، تجاوز حدود المعدل، الكشط الخفي المصمم لهزيمة وسائل الحماية، الاستيلاء على مكالمة حية أو وكيل حي، سرقة جلسات قابلة لإعادة الاستخدام، الموافقة التلقائية على تدفقات الاقتران لمستخدمين غير معتمدين.

- إساءة استخدام المنصات وتجنب الحظر.
  - أمثلة: حسابات خفية بعد الحظر، تسخين الحسابات أو زراعتها، تفاعل مزيف، تنمية الكارما أو المتابعين، أتمتة متعددة الحسابات، نشر جماعي، روبوتات رسائل مزعجة، أتمتة الأسواق أو الشبكات الاجتماعية المصممة لتجنب الاكتشاف.

- الاحتيال، وعمليات النصب، ومسارات العمل المالية الخادعة.
  - أمثلة: شهادات مزيفة، فواتير مزيفة، تدفقات دفع خادعة، تواصل احتيالي، إثبات اجتماعي مزيف، أدوات تتيح الإنفاق أو التحصيل دون موافقة بشرية واضحة وضوابط شفافة، أو مسارات هويات اصطناعية مبنية لإنشاء حسابات بغرض الاحتيال.

- الكشط أو الإثراء أو المراقبة المنتهكة للخصوصية.
  - أمثلة: كشط تفاصيل الاتصال على نطاق واسع للرسائل المزعجة، كشف البيانات الشخصية، المطاردة، استخراج العملاء المحتملين مقرونًا بتواصل غير مرغوب فيه، المراقبة السرية، البحث عن الوجوه أو المطابقة البيومترية المستخدمة دون موافقة واضحة، أو شراء البيانات المسربة أو تفريغات الاختراق أو نشرها أو تنزيلها أو تحويلها إلى عمليات تشغيلية.

- انتحال الهوية غير الرضائي أو التلاعب الخادع بالهوية.
  - أمثلة: تبديل الوجوه، التوائم الرقمية، الشخصيات المزيفة، المؤثرون المستنسخون، أو أدوات تلاعب أخرى بالهوية تُستخدم للانتحال أو التضليل.

- المحتوى الجنسي الصريح وتوليد محتوى بالغين مع تعطيل السلامة.
  - أمثلة: توليد صور أو فيديوهات أو محتوى NSFW، واجهات محتوى بالغين حول واجهات API تابعة لجهات خارجية، أو Skills يكون غرضها الأساسي المحتوى الجنسي الصريح.

- متطلبات تنفيذ مخفية أو غير آمنة أو مضللة.
  - أمثلة: أوامر تثبيت مبهمة، `curl | sh`، متطلبات أسرار غير مصرح بها، استخدام مفاتيح خاصة غير مصرح به، تنفيذ `npx @latest` عن بعد دون قابلية مراجعة واضحة، بيانات وصفية مضللة تخفي ما تحتاجه Skill حقًا للتشغيل.

## الأنماط الحديثة التي لا نقبلها صراحة

- “إنشاء حسابات بائعين خفية بعد حظرها في الأسواق.”
- “تعديل اقتران Telegram بحيث يتلقى المستخدمون غير المعتمدين رموز الاقتران تلقائيًا.”
- “تنمية حسابات Reddit/Twitter بأتمتة غير قابلة للاكتشاف.”
- “توليد شهادات مهنية أو فواتير لاستخدام عشوائي.”
- “توليد محتوى NSFW مع تعطيل فحوصات السلامة.”
- “كشط العملاء المحتملين، وإثراء جهات الاتصال، وإطلاق تواصل بارد على نطاق واسع.”
- “شراء البيانات المسربة أو تفريغات الاختراق أو نشرها أو تنزيلها.”
- “إنشاء حسابات بريد إلكتروني أو حسابات اجتماعية بكميات كبيرة باستخدام هويات اصطناعية أو حل CAPTCHA.”

## ملاحظات للمراجعين

- السياق مهم. قد يكون الموضوع نفسه مشروعًا في نطاق دفاعي ضيق أو قائم على الموافقة، وغير مقبول عندما يُعبأ كمسار عمل لإساءة الاستخدام.
- ينبغي أن نميل إلى اتخاذ إجراء عندما تكون Skill محسنة بوضوح للتجنب أو الخداع أو الاستخدام غير الرضائي.
- التحميلات المتكررة في هذه الفئات تُعد سببًا لإخفاء المحتوى وحظر الحساب.

## الإنفاذ

- قد نخفي Skills المخالفة أو نزيلها أو نحذفها حذفًا نهائيًا.
- قد نسحب الرموز، ونحذف المحتوى المرتبط حذفًا مبدئيًا، ونحظر المخالفين المتكررين أو الجسيمين.
- لا نضمن الإنفاذ بعد تحذير مسبق في حالات إساءة الاستخدام الواضحة.
