# الاستخدام المقبول

يستضيف ClawHub الـ Skills والـ Plugins والحزم وبيانات السوق الوصفية الخاصة بـ OpenClaw.
استخدم هذه الصفحة لتحديد ما إذا كان المحتوى أو سلوك النشر ينتمي إلى
ClawHub.

تنطبق هذه القواعد على ما تفعله القائمة، وما تطلب من المستخدمين تشغيله، وكيف
تمثل نفسها، وكيف يستخدم الناشرون أسطح الاكتشاف والتثبيت والثقة في ClawHub.
بالنسبة إلى حالات الإشراف ووضع الحساب، راجع
[الإشراف وسلامة الحساب](/clawhub/moderation). وبالنسبة إلى مطالبات حقوق النشر أو الحقوق الأخرى،
راجع [طلبات حقوق المحتوى](/ar/clawhub/content-rights).

## المحتوى المسموح

يرحب ClawHub بالمحتوى المفيد والواضح والمنشور بحسن نية.

| الفئة | مسموح عندما |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| إنتاجية المطورين | تساعد القائمة المستخدمين على بناء البرامج أو اختبارها أو ترحيلها أو تصحيح أخطائها أو توثيقها أو تشغيلها. |
| واجهة المستخدم والبيانات وسير عمل الأتمتة | يكون النطاق واضحا، وتكون بيانات الاعتماد المطلوبة صريحة، وتتضمن الإجراءات الخطرة مسارات مراجعة أو تشغيل تجريبي أو معاينة أو تأكيد. |
| الأمان الدفاعي والإشراف ومراجعة إساءة الاستخدام | تكون الأداة مقدمة للمراجعة المصرح بها، وتحافظ على الأدلة، وتبقي حدود الموافقة البشرية واضحة. |
| سير العمل الشخصي أو الخاص بالفريق | يستخدم سير العمل حسابات قائمة على الموافقة، وإعدادا شفافا، وأذونات صريحة. |
| الكتالوجات المصانة | تكون كل قائمة مميزة ومفيدة وموصوفة بدقة ومصانة بدرجة معقولة. |

السياق مهم. يمكن أن يكون الموضوع نفسه مقبولا في سياق دفاعي ضيق أو قائم على
الموافقة، وغير مقبول عندما يعبأ كسير عمل لإساءة الاستخدام.

## المحتوى غير المسموح

لا يستضيف ClawHub محتوى يكون غرضه الأساسي إساءة الاستخدام أو الخداع أو التنفيذ
غير الآمن أو انتهاك الحقوق.

| الفئة | غير مسموح |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| الوصول غير المصرح به أو تجاوز الأمان | تجاوز المصادقة، أو الاستيلاء على الحسابات، أو إساءة استخدام حدود المعدل، أو الاستيلاء على مكالمة مباشرة أو وكيل، أو سرقة جلسات قابلة لإعادة الاستخدام، أو الموافقة التلقائية على تدفقات الاقتران لمستخدمين غير معتمدين. |
| إساءة استخدام المنصات وتجاوز الحظر | الحسابات الخفية بعد الحظر، أو تسخين الحسابات أو زراعتها، أو التفاعل المزيف، أو أتمتة الحسابات المتعددة، أو النشر الجماعي، أو روبوتات الرسائل المزعجة، أو الأتمتة المصممة لتجنب الاكتشاف. |
| الاحتيال وعمليات النصب وسير العمل المالي الخادع | الشهادات أو الفواتير المزيفة، أو تدفقات الدفع الخادعة، أو تواصل الاحتيال، أو الإثبات الاجتماعي المزيف، أو سير عمل الهوية الاصطناعية للاحتيال، أو أدوات الإنفاق/التحصيل دون موافقة بشرية واضحة. |
| الإثراء أو المراقبة المنتهكان للخصوصية | جمع جهات الاتصال للرسائل المزعجة، أو كشف المعلومات الشخصية، أو الملاحقة، أو استخراج العملاء المحتملين المقترن بالتواصل غير المرغوب فيه، أو المراقبة السرية، أو المطابقة البيومترية دون موافقة، أو استخدام البيانات المسربة أو تفريغات الاختراق. |
| الانتحال أو التلاعب بالهوية دون موافقة | تبديل الوجوه، أو التوائم الرقمية، أو المؤثرون المستنسخون، أو الشخصيات المزيفة، أو غيرها من الأدوات المستخدمة للانتحال أو التضليل. |
| المحتوى الجنسي الصريح أو توليد محتوى بالغين معطل السلامة | توليد صور أو فيديوهات أو محتوى NSFW؛ أو أغلفة محتوى بالغين حول واجهات API تابعة لجهات خارجية؛ أو القوائم التي يكون غرضها الأساسي المحتوى الجنسي الصريح. |
| متطلبات تنفيذ مخفية أو غير آمنة أو مضللة | أوامر تثبيت مبهمة، أو مثبتات تمرر إلى الصدفة مثل محتوى منزّل يشغل باستخدام `sh` أو `bash` دون قابلية مراجعة واضحة، أو متطلبات أسرار أو مفاتيح خاصة غير معلنة، أو تنفيذ `npx @latest` عن بعد دون قابلية مراجعة واضحة، أو بيانات وصفية تخفي ما تحتاجه القائمة فعليا للتشغيل. |
| مواد تنتهك حقوق النشر أو الحقوق | إعادة نشر Skill أو Plugin أو مستندات أو أصول علامة تجارية أو كود مملوك لشخص آخر دون إذن؛ أو انتهاك شروط الترخيص؛ أو انتحال المؤلف أو الناشر الأصلي. |

## سلوك السوق غير المسموح

يراجع ClawHub أيضا كيفية استخدام الناشرين للسوق. لا تستخدم ClawHub
للتلاعب بالاكتشاف أو المقاييس أو إشارات الثقة أو أنظمة الإشراف أو انتباه
المستخدمين.

يشمل سلوك السوق غير المسموح:

- نشر أعداد كبيرة بالجملة من القوائم منخفضة الجهد أو المكررة أو النائبة أو
  المولدة آليا التي لا يبدو أن لها قيمة حقيقية للمستخدم
- إغراق أسطح البحث أو الفئات بمهارات أو Plugins شبه متطابقة
- نشر مئات القوائم مع استخدام أو صيانة أو وضوح مصدر أو تمييز ذي معنى قليل أو معدوم
- تضخيم عمليات التثبيت أو التنزيلات أو النجوم أو مقاييس التفاعل الأخرى
  اصطناعيا عبر الأتمتة أو حلقات التثبيت الذاتي أو الحسابات المزيفة أو النشاط
  المنسق أو التفاعل المدفوع أو غير ذلك من السلوك غير العضوي
- إنشاء الحسابات أو تدويرها لتجاوز الإشراف أو الحظر أو حدود الناشرين أو
  مراجعة السوق
- تضليل المستخدمين بشأن الملكية أو المصدر أو القدرات أو الوضع الأمني أو
  متطلبات التثبيت أو الارتباط بمشروع أو ناشر آخر
- رفع محتوى بشكل متكرر سبق إخفاؤه أو إزالته أو حظره
  دون إصلاح المشكلة الأساسية

النشر كبير الحجم ليس إساءة استخدام تلقائيا. الكتالوجات الكبيرة مقبولة
عندما تكون القوائم مختلفة بشكل ذي معنى، وموصوفة بدقة، ومصانة،
ويستخدمها مستخدمون حقيقيون. تصبح الكتالوجات الكبيرة مشكلة ثقة وسلامة عندما
يقترن الحجم بقوائم سطحية أو مكررة أو مضللة أو غير مصانة أو
مروجة اصطناعيا.

## حقوق المحتوى

إذا كنت تعتقد أن محتوى على ClawHub ينتهك حقوق النشر الخاصة بك أو حقوقا أخرى، فاستخدم
[طلبات حقوق المحتوى](/ar/clawhub/content-rights). لا تستخدم تقارير السوق العادية
لمطالبات حقوق النشر أو الحقوق إلا إذا كانت القائمة أيضا غير آمنة أو
خبيثة أو مضللة.

## المراجعة والإنفاذ

قد يستخدم ClawHub فحوصات آلية، وإشارات إساءة استخدام إحصائية، وتقارير المستخدمين،
ومراجعة الموظفين لتحديد المحتوى غير الآمن أو سلوك النشر المسيء. لا تثبت الإشارة
إساءة الاستخدام بذاتها؛ بل تساعد ClawHub على تحديد ما يحتاج إلى مراجعة.

قد نقوم بما يلي:

- إخفاء القوائم المخالفة أو تعليقها أو إزالتها أو حذفها حذفا مرنا، أو، عندما يكون ذلك مدعوما لنوع المورد،
  حذفها حذفا نهائيا
- حظر التنزيلات أو عمليات التثبيت للإصدارات غير الآمنة
- إلغاء رموز API
- حذف المحتوى المرتبط حذفا مرنا
- تقييد صلاحية النشر
- حظر المخالفين المتكررين أو الجسيمين

لا نضمن الإنفاذ بعد التحذير أولا في حالات إساءة الاستخدام الواضحة. راجع
[الإشراف وسلامة الحساب](/clawhub/moderation) للاطلاع على التقارير وتعليقات الإشراف
والقوائم المخفية والحظر ووضع الحساب.
