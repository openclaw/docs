# الاستخدام المقبول

تصف هذه الصفحة أنواع المهارات والمحتوى المقبولة لدى ClawHub، وسير عمل إساءة الاستخدام التي لن يستضيفها.

هذه القواعد عملية عن قصد. ما يهمنا أكثر هو سير عمل إساءة الاستخدام من البداية إلى النهاية، وليس مجرد كلمات مفتاحية معزولة. إذا كانت المهارة مبنية لتجاوز الدفاعات، أو إساءة استخدام المنصات، أو الاحتيال على الأشخاص، أو انتهاك الخصوصية، أو تمكين سلوك غير توافقي، فهي لا تنتمي إلى ClawHub.

## أنماط حديثة نقبلها صراحةً

- عمل الواجهة الأمامية وأنظمة التصميم الذي يستخدم مكونات حقيقية، ورموزًا دلالية، وحالات قابلة للوصول، وتدفقات مستخدمين مختبرة.
- تركيب shadcn/ui الذي يستخدم مكونات مصدرية مثبتة، وأسماء مستعارة للمشروع، ومتغيرات موثقة بدلًا من ترميز مخصص لمرة واحدة.
- تحويل UI5 من JavaScript إلى TypeScript مع الحفاظ على التعليقات، واستخدام أنواع UI5 ملموسة، وإبقاء واجهات عناصر التحكم المولدة قابلة للمراجعة.
- مراجعة أمنية دفاعية، وأدوات إشراف، ومطالبات لاكتشاف إساءة الاستخدام تعرض الأدلة وتبقي حدود الموافقة البشرية واضحة.
- أتمتة سير عمل قائمة على الموافقة للحسابات الشخصية أو حسابات الفريق باستخدام بيانات اعتماد صريحة، وإعداد شفاف، وأوضاع تشغيل تجريبي أو معاينة.
- توثيق، وأدلة تشغيل للترحيل، وأدوات للمطورين، وتجهيزات اختبار محددة النطاق للبرمجيات التي تدعمها.

## غير مقبول

- سير عمل تجاوز الأمان أو الوصول غير المصرح به.
  - أمثلة: تجاوز المصادقة، الاستيلاء على الحسابات، تجاوز CAPTCHA، تجاوز Cloudflare أو أنظمة مكافحة الروبوتات، تجاوز حدود المعدل، الكشط الخفي المصمم لهزيمة الحمايات، الاستيلاء على مكالمة مباشرة أو وكيل مباشر، سرقة جلسات قابلة لإعادة الاستخدام، أو الموافقة التلقائية على تدفقات الاقتران لمستخدمين غير معتمدين.

- إساءة استخدام المنصات والتهرب من الحظر.
  - أمثلة: حسابات خفية بعد الحظر، تهيئة الحسابات أو زراعتها، تفاعل مزيف، تنمية الكارما أو المتابعين، أتمتة الحسابات المتعددة، النشر الجماعي، روبوتات البريد المزعج، أو أتمتة الأسواق أو الشبكات الاجتماعية المبنية لتجنب الاكتشاف.

- الاحتيال، والخدع، وسير العمل المالي الخادع.
  - أمثلة: شهادات مزيفة، فواتير مزيفة، تدفقات دفع خادعة، تواصل احتيالي، إثبات اجتماعي مزيف، أدوات تمكّن الإنفاق أو فرض الرسوم دون موافقة بشرية واضحة وضوابط شفافة، أو سير عمل هويات اصطناعية مبنية لإنشاء حسابات لأغراض الاحتيال.

- الكشط أو الإثراء أو المراقبة التي تنتهك الخصوصية.
  - أمثلة: كشط تفاصيل الاتصال على نطاق واسع للبريد المزعج، كشف البيانات الشخصية، الملاحقة، استخراج العملاء المحتملين مقترنًا بتواصل غير مرغوب فيه، المراقبة السرية، البحث عن الوجوه أو المطابقة البيومترية المستخدمة دون موافقة واضحة، أو شراء بيانات مسربة أو تفريغات خروقات أو نشرها أو تنزيلها أو تحويلها إلى عمليات تشغيلية.

- الانتحال غير التوافقي أو التلاعب الخادع بالهوية.
  - أمثلة: تبديل الوجوه، التوائم الرقمية، الشخصيات المزيفة، المؤثرون المستنسخون، أو أدوات أخرى للتلاعب بالهوية تُستخدم للانتحال أو التضليل.

- المحتوى الجنسي الصريح وتوليد محتوى للبالغين مع تعطيل وسائل السلامة.
  - أمثلة: توليد صور أو فيديوهات أو محتوى NSFW، أغلفة محتوى للبالغين حول APIs تابعة لجهات خارجية، أو مهارات يكون غرضها الأساسي محتوى جنسيًا صريحًا.

- متطلبات تنفيذ مخفية أو غير آمنة أو مضللة.
  - أمثلة: أوامر تثبيت مبهمة، `curl | sh`، متطلبات أسرار غير معلنة، استخدام مفاتيح خاصة غير معلن، تنفيذ `npx @latest` عن بُعد دون قابلية مراجعة واضحة، أو بيانات وصفية مضللة تخفي ما تحتاجه المهارة فعليًا للتشغيل.

## أنماط حديثة لا نقبلها صراحةً

- «إنشاء حسابات بائعين خفية بعد حظرهم من الأسواق.»
- «تعديل اقتران Telegram بحيث يتلقى المستخدمون غير المعتمدين رموز الاقتران تلقائيًا.»
- «تنمية حسابات Reddit/Twitter باستخدام أتمتة غير قابلة للاكتشاف.»
- «إنشاء شهادات أو فواتير مهنية لاستخدام عشوائي.»
- «توليد محتوى NSFW مع تعطيل فحوصات السلامة.»
- «كشط العملاء المحتملين، وإثراء جهات الاتصال، وإطلاق تواصل بارد على نطاق واسع.»
- «شراء بيانات مسربة أو تفريغات خروقات أو نشرها أو تنزيلها.»
- «إنشاء حسابات بريد إلكتروني أو حسابات اجتماعية بكميات كبيرة باستخدام هويات اصطناعية أو حل CAPTCHA.»

## ملاحظات للمراجعين

- السياق مهم. قد يكون الموضوع نفسه مشروعًا في إطار دفاعي ضيق أو قائم على الموافقة، وغير مقبول عندما يُغلف كسير عمل لإساءة الاستخدام.
- ينبغي أن نميل إلى اتخاذ إجراء عندما تكون المهارة محسّنة بوضوح للتهرب، أو الخداع، أو الاستخدام غير التوافقي.
- الرفع المتكرر ضمن هذه الفئات يشكل أساسًا لإخفاء المحتوى وحظر الحساب.

## الإنفاذ

- قد نخفي المهارات المخالفة أو نزيلها أو نحذفها حذفًا نهائيًا.
- قد نلغي الرموز، ونحذف المحتوى المرتبط حذفًا لينًا، ونحظر المخالفين المتكررين أو الجسيمين.
- لا نضمن الإنفاذ بعد تحذير أولًا في حالات إساءة الاستخدام الواضحة.
