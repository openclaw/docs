# الاستخدام المقبول

تصف هذه الصفحة أنواع المهارات والمحتوى المقبولة لدى ClawHub، وسير عمل إساءة الاستخدام التي لن يستضيفها.

هذه القواعد عملية عن قصد. ما نهتم به أكثر هو سير عمل إساءة الاستخدام من البداية إلى النهاية، وليس مجرد كلمات مفتاحية معزولة. إذا كانت المهارة مبنية للالتفاف على الدفاعات، أو إساءة استخدام المنصات، أو الاحتيال على الأشخاص، أو انتهاك الخصوصية، أو تمكين سلوك غير توافقي، فلا مكان لها على ClawHub.

## أنماط حديثة نصرّح بأنها مقبولة لدينا

- أعمال الواجهة الأمامية وأنظمة التصميم التي تستخدم مكونات حقيقية، ورموزًا دلالية، وحالات قابلة للوصول، وتدفقات مستخدمين مختبرة.
- تركيب shadcn/ui الذي يستخدم مكونات مصدرية مثبتة، وأسماء مستعارة للمشروع، ومتغيرات موثقة بدلًا من ترميز مخصص لمرة واحدة.
- تحويل UI5 من JavaScript إلى TypeScript مع الحفاظ على التعليقات، واستخدام أنواع UI5 محددة، وإبقاء واجهات عناصر التحكم المولدة قابلة للمراجعة.
- مراجعة أمنية دفاعية، وأدوات إشراف، ومطالبات لاكتشاف إساءة الاستخدام تعرض الأدلة وتبقي حدود الموافقة البشرية واضحة.
- أتمتة سير عمل قائمة على الموافقة للحسابات الشخصية أو حسابات الفريق باستخدام بيانات اعتماد صريحة، وإعداد شفاف، وأوضاع تشغيل تجريبي أو معاينة.
- التوثيق، وكتيبات تشغيل الترحيل، وأدوات المطورين، وتركيبات الاختبار المحددة ضمن نطاق البرنامج الذي تدعمه.

## غير مقبول

- سير عمل تجاوز الأمان أو الوصول غير المصرح به.
  - أمثلة: تجاوز المصادقة، الاستيلاء على الحسابات، تجاوز CAPTCHA، الالتفاف على Cloudflare أو أنظمة مكافحة الروبوتات، تجاوز حدود المعدل، الاستخراج الخفي المصمم لهزيمة وسائل الحماية، الاستيلاء على مكالمة أو وكيل مباشر، سرقة الجلسات القابلة لإعادة الاستخدام، الموافقة التلقائية على تدفقات الاقتران للمستخدمين غير المعتمدين.

- إساءة استخدام المنصات والالتفاف على الحظر.
  - أمثلة: حسابات خفية بعد الحظر، تهيئة الحسابات أو زراعتها، تفاعل مزيف، تنمية karma أو المتابعين، أتمتة الحسابات المتعددة، النشر الجماعي، روبوتات الرسائل المزعجة، أتمتة الأسواق أو الشبكات الاجتماعية المبنية لتجنب الاكتشاف.

- الاحتيال، وعمليات النصب، وسير العمل المالي المخادع.
  - أمثلة: شهادات مزيفة، فواتير مزيفة، تدفقات دفع مخادعة، تواصل احتيالي، دليل اجتماعي مزيف، أدوات تتيح الإنفاق أو التحصيل دون موافقة بشرية واضحة وضوابط شفافة، أو سير عمل هوية اصطناعية مبني لإنشاء حسابات بغرض الاحتيال.

- الاستخراج أو الإثراء أو المراقبة المنتهكة للخصوصية.
  - أمثلة: استخراج تفاصيل الاتصال على نطاق واسع للرسائل المزعجة، كشف المعلومات الشخصية، الملاحقة، استخراج العملاء المحتملين مع تواصل غير مطلوب، المراقبة السرية، البحث بالوجه أو المطابقة البيومترية المستخدمة دون موافقة واضحة، أو شراء بيانات مسربة أو حزم اختراق أو نشرها أو تنزيلها أو تشغيلها عمليًا.

- انتحال غير توافقي أو تلاعب مخادع بالهوية.
  - أمثلة: تبديل الوجوه، التوائم الرقمية، الشخصيات المزيفة، المؤثرون المستنسخون، أو أدوات تلاعب أخرى بالهوية تُستخدم للانتحال أو التضليل.

- محتوى جنسي صريح وتوليد محتوى للبالغين مع تعطيل السلامة.
  - أمثلة: توليد صور/فيديوهات/محتوى NSFW، أغلفة محتوى للبالغين حول واجهات API تابعة لأطراف ثالثة، أو مهارات يكون غرضها الأساسي محتوى جنسيًا صريحًا.

- متطلبات تنفيذ مخفية أو غير آمنة أو مضللة.
  - أمثلة: أوامر تثبيت مبهمة، `curl | sh`، متطلبات أسرار غير معلنة، استخدام مفاتيح خاصة غير معلن، تنفيذ `npx @latest` عن بُعد دون قابلية مراجعة واضحة، بيانات وصفية مضللة تخفي ما تحتاجه المهارة فعليًا للتشغيل.

## أنماط حديثة نصرّح بأنها غير مقبولة لدينا

- “إنشاء حسابات بائع خفية بعد الحظر من السوق.”
- “تعديل اقتران Telegram بحيث يتلقى المستخدمون غير المعتمدين رموز الاقتران تلقائيًا.”
- “تنمية حسابات Reddit/Twitter باستخدام أتمتة غير قابلة للاكتشاف.”
- “إنشاء شهادات مهنية أو فواتير لاستخدامات عشوائية.”
- “توليد محتوى NSFW مع تعطيل فحوصات السلامة.”
- “استخراج العملاء المحتملين، وإثراء جهات الاتصال، وإطلاق تواصل بارد على نطاق واسع.”
- “شراء بيانات مسربة أو حزم اختراق أو نشرها أو تنزيلها.”
- “إنشاء حسابات بريد إلكتروني أو حسابات اجتماعية بالجملة باستخدام هويات اصطناعية أو حل CAPTCHA.”

## ملاحظات للمراجعين

- السياق مهم. يمكن أن يكون الموضوع نفسه مشروعًا في إعداد دفاعي ضيق أو قائم على الموافقة، وغير مقبول عندما يُعبأ كسير عمل لإساءة الاستخدام.
- ينبغي أن نميل إلى اتخاذ إجراء عندما تكون المهارة محسّنة بوضوح للالتفاف أو الخداع أو الاستخدام غير التوافقي.
- التحميلات المتكررة ضمن هذه الفئات تُعد سببًا لإخفاء المحتوى وحظر الحساب.

## الإنفاذ

- قد نخفي المهارات المخالفة أو نزيلها أو نحذفها حذفًا نهائيًا.
- قد نسحب الرموز، ونحذف المحتوى المرتبط حذفًا ناعمًا، ونحظر المخالفين المتكررين أو الجسيمين.
- لا نضمن الإنفاذ مع التحذير أولًا في حالات إساءة الاستخدام الواضحة.
