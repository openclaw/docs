# الاستخدام المقبول

تصف هذه الصفحة أنواع Skills والمحتوى التي يقبلها ClawHub، وسير عمل إساءة الاستخدام التي لن يستضيفها.

هذه القواعد عملية عن قصد. نهتم أكثر بسير عمل إساءة الاستخدام من البداية إلى النهاية، وليس بالكلمات المفتاحية المعزولة فقط. إذا كانت إحدى المهارات مصممة لتجاوز وسائل الدفاع، أو إساءة استخدام المنصات، أو الاحتيال على الأشخاص، أو انتهاك الخصوصية، أو تمكين سلوك غير توافقي، فلا مكان لها على ClawHub.

## أنماط حديثة نقبلها صراحة

- أعمال الواجهة الأمامية وأنظمة التصميم التي تستخدم مكوّنات حقيقية، ورموزًا دلالية، وحالات قابلة للوصول، وتدفقات مستخدم مختبرة.
- تركيب shadcn/ui الذي يستخدم مكوّنات المصدر المثبتة، وأسماء المشاريع المستعارة، والمتغيرات الموثقة بدلًا من ترميز مخصص لمرة واحدة.
- تحويل UI5 من JavaScript إلى TypeScript مع الحفاظ على التعليقات، واستخدام أنواع UI5 محددة، وإبقاء واجهات عناصر التحكم المولّدة قابلة للمراجعة.
- مراجعة الأمان الدفاعية، وأدوات الإشراف، ومطالبات اكتشاف إساءة الاستخدام التي تعرض الأدلة وتُبقي حدود الموافقة البشرية واضحة.
- أتمتة سير العمل القائمة على الموافقة لحسابات شخصية أو فرق باستخدام بيانات اعتماد صريحة، وإعداد شفاف، وأوضاع تشغيل تجريبي أو معاينة.
- التوثيق، وأدلة الهجرة التشغيلية، وأدوات المطورين، وتركيبات الاختبار المحددة بنطاق البرنامج الذي تدعمه.

## غير مقبول

- سير عمل تجاوز الأمان أو الوصول غير المصرح به.
  - أمثلة: تجاوز المصادقة، الاستيلاء على الحسابات، تجاوز CAPTCHA، التهرب من Cloudflare أو أنظمة مكافحة الروبوتات، تجاوز حدود المعدل، الاستخراج الخفي المصمم لهزيمة وسائل الحماية، الاستيلاء على مكالمة مباشرة أو وكيل، سرقة الجلسات القابلة لإعادة الاستخدام، الموافقة التلقائية على تدفقات الاقتران للمستخدمين غير المعتمدين.

- إساءة استخدام المنصات والتهرب من الحظر.
  - أمثلة: حسابات خفية بعد الحظر، تهيئة الحسابات أو زراعتها، تفاعل مزيف، تنمية الكارما أو المتابعين، أتمتة متعددة الحسابات، النشر الجماعي، روبوتات الرسائل المزعجة، أتمتة الأسواق أو الشبكات الاجتماعية المصممة لتجنب الاكتشاف.

- الاحتيال، والخدع، وسير العمل المالي الخادع.
  - أمثلة: شهادات مزيفة، فواتير مزيفة، تدفقات دفع خادعة، تواصل احتيالي، إثبات اجتماعي مزيف، أدوات تمكّن الإنفاق أو فرض الرسوم دون موافقة بشرية واضحة وضوابط شفافة، أو سير عمل هوية اصطناعية مصممة لإنشاء حسابات بغرض الاحتيال.

- الاستخراج أو الإثراء أو المراقبة التي تنتهك الخصوصية.
  - أمثلة: استخراج تفاصيل الاتصال على نطاق واسع للرسائل المزعجة، التشهير بالمعلومات الشخصية، المطاردة، استخراج العملاء المحتملين مقترنًا بتواصل غير مرغوب فيه، المراقبة السرية، البحث عن الوجوه أو المطابقة البيومترية المستخدمة دون موافقة واضحة، أو شراء البيانات المسرّبة أو تفريغات الاختراقات أو نشرها أو تنزيلها أو تشغيلها عمليًا.

- انتحال غير توافقي أو تلاعب خادع بالهوية.
  - أمثلة: تبديل الوجوه، التوائم الرقمية، الشخصيات المزيفة، المؤثرون المستنسخون، أو أدوات تلاعب أخرى بالهوية تُستخدم للانتحال أو التضليل.

- محتوى جنسي صريح وتوليد محتوى للبالغين مع تعطيل السلامة.
  - أمثلة: توليد صور/فيديو/محتوى NSFW، أغلفة محتوى للبالغين حول واجهات API تابعة لجهات خارجية، أو Skills يكون غرضها الأساسي محتوى جنسيًا صريحًا.

- متطلبات تنفيذ مخفية أو غير آمنة أو مضللة.
  - أمثلة: أوامر تثبيت مبهمة، `curl | sh`، متطلبات أسرار غير معلنة، استخدام مفاتيح خاصة غير معلن، تنفيذ `npx @latest` عن بُعد دون قابلية واضحة للمراجعة، بيانات وصفية مضللة تخفي ما تحتاجه المهارة فعليًا للتشغيل.

## أنماط حديثة لا نقبلها صراحة

- "إنشاء حسابات بائع خفية بعد حظر الأسواق."
- "تعديل اقتران Telegram بحيث يتلقى المستخدمون غير المعتمدين رموز الاقتران تلقائيًا."
- "تنمية حسابات Reddit/Twitter بأتمتة غير قابلة للاكتشاف."
- "إنشاء شهادات مهنية أو فواتير لأي استخدام."
- "توليد محتوى NSFW مع تعطيل فحوصات السلامة."
- "استخراج العملاء المحتملين، وإثراء جهات الاتصال، وإطلاق تواصل بارد على نطاق واسع."
- "شراء بيانات مسرّبة أو تفريغات اختراقات أو نشرها أو تنزيلها."
- "إنشاء حسابات بريد إلكتروني أو حسابات اجتماعية بالجملة باستخدام هويات اصطناعية أو حل CAPTCHA."

## ملاحظات للمراجعين

- السياق مهم. يمكن أن يكون الموضوع نفسه مشروعًا في إعداد دفاعي ضيق أو قائم على الموافقة، وغير مقبول عندما يُعبأ كسير عمل لإساءة الاستخدام.
- ينبغي أن نميل إلى اتخاذ إجراء عندما تكون إحدى المهارات محسّنة بوضوح للتهرب أو الخداع أو الاستخدام غير التوافقي.
- التحميلات المتكررة ضمن هذه الفئات تُعد أسبابًا لإخفاء المحتوى وحظر الحساب.

## الإنفاذ

- قد نخفي Skills المخالفة أو نزيلها أو نحذفها نهائيًا.
- قد نلغي الرموز، ونحذف المحتوى المرتبط حذفًا ناعمًا، ونحظر المخالفين المتكررين أو الشديدين.
- لا نضمن الإنفاذ بعد التحذير أولًا في حالات إساءة الاستخدام الواضحة.
