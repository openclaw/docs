# الاستخدام المقبول

تصف هذه الصفحة أنواع المهارات والمحتوى المقبولة في ClawHub، وسير عمل إساءة الاستخدام التي لن يستضيفها.

هذه القواعد عملية عن قصد. ما يهمنا أكثر هو سير عمل إساءة الاستخدام من البداية إلى النهاية، وليس مجرد كلمات مفتاحية معزولة. إذا كانت المهارة مبنية لتجاوز الدفاعات، أو إساءة استخدام المنصات، أو الاحتيال على الأشخاص، أو انتهاك الخصوصية، أو تمكين سلوك غير رضائي، فلا مكان لها على ClawHub.

## أنماط حديثة نقبلها صراحة

- أعمال الواجهة الأمامية وأنظمة التصميم التي تستخدم مكونات حقيقية، ورموزًا دلالية، وحالات قابلة للوصول، وتدفقات مستخدم مختبرة.
- تركيب shadcn/ui الذي يستخدم مكونات مصدرية مثبتة، وأسماء مستعارة للمشروع، ومتغيرات موثقة بدلًا من ترميز أحادي الاستخدام.
- تحويل UI5 من JavaScript إلى TypeScript مع الحفاظ على التعليقات، واستخدام أنواع UI5 محددة، وإبقاء واجهات عناصر التحكم المولدة قابلة للمراجعة.
- مراجعة أمنية دفاعية، وأدوات إشراف، ومطالبات كشف إساءة الاستخدام التي تعرض أدلة وتبقي حدود موافقة الإنسان واضحة.
- أتمتة سير عمل قائمة على الموافقة لحسابات شخصية أو فرق، مع بيانات اعتماد صريحة، وإعداد شفاف، وأوضاع تشغيل تجريبي أو معاينة.
- وثائق، وكتيبات ترحيل تشغيلية، وأدوات للمطورين، وتجهيزات اختبار محددة النطاق للبرمجيات التي تدعمها.

## غير مقبول

- سير عمل تجاوز الأمان أو الوصول غير المصرح به.
  - أمثلة: تجاوز المصادقة، الاستيلاء على الحسابات، تجاوز CAPTCHA، التهرب من Cloudflare أو أنظمة مكافحة الروبوتات، تجاوز حدود المعدل، كشط خفي مصمم لهزيمة الحمايات، الاستيلاء على مكالمة مباشرة أو وكيل مباشر، سرقة جلسات قابلة لإعادة الاستخدام، الموافقة التلقائية على تدفقات الاقتران لمستخدمين غير معتمدين.

- إساءة استخدام المنصات والتهرب من الحظر.
  - أمثلة: حسابات خفية بعد الحظر، تهيئة الحسابات أو زراعتها، تفاعل زائف، تنمية الكارما أو المتابعين، أتمتة متعددة الحسابات، النشر الجماعي، روبوتات الرسائل المزعجة، أتمتة الأسواق أو الشبكات الاجتماعية المصممة لتجنب الكشف.

- الاحتيال، وعمليات النصب، وسير العمل المالي الخادع.
  - أمثلة: شهادات مزيفة، فواتير مزيفة، تدفقات دفع خادعة، تواصل احتيالي، إثبات اجتماعي مزيف، أدوات تتيح الإنفاق أو التحصيل دون موافقة بشرية واضحة وضوابط شفافة، أو سير عمل هويات اصطناعية مبنية لإنشاء حسابات بغرض الاحتيال.

- الكشط أو الإثراء أو المراقبة المنتهكة للخصوصية.
  - أمثلة: كشط تفاصيل الاتصال على نطاق واسع للرسائل المزعجة، نشر معلومات شخصية، الملاحقة، استخراج العملاء المحتملين مقرونًا بتواصل غير مرغوب فيه، المراقبة السرية، البحث بالوجه أو المطابقة البيومترية المستخدمة دون موافقة واضحة، أو شراء البيانات المسربة أو تفريغات الاختراق أو نشرها أو تنزيلها أو تفعيلها عمليًا.

- الانتحال غير الرضائي أو التلاعب الخادع بالهوية.
  - أمثلة: تبديل الوجوه، التوائم الرقمية، الشخصيات المزيفة، المؤثرون المستنسخون، أو أدوات أخرى للتلاعب بالهوية تُستخدم للانتحال أو التضليل.

- المحتوى الجنسي الصريح وتوليد محتوى البالغين مع تعطيل السلامة.
  - أمثلة: توليد صور/فيديو/محتوى NSFW، أغلفة محتوى البالغين حول واجهات API تابعة لجهات خارجية، أو مهارات يكون غرضها الأساسي محتوى جنسيًا صريحًا.

- متطلبات تنفيذ مخفية أو غير آمنة أو مضللة.
  - أمثلة: أوامر تثبيت مبهمة، `curl | sh`، متطلبات أسرار غير مصرح بها، استخدام مفتاح خاص غير مصرح به، تنفيذ `npx @latest` عن بُعد دون قابلية مراجعة واضحة، بيانات وصفية مضللة تخفي ما تحتاجه المهارة فعليًا للتشغيل.

## أنماط حديثة لا نقبلها صراحة

- "إنشاء حسابات بائع خفية بعد حظر السوق."
- "تعديل اقتران Telegram بحيث يتلقى المستخدمون غير المعتمدين رموز الاقتران تلقائيًا."
- "تنمية حسابات Reddit/Twitter بأتمتة غير قابلة للكشف."
- "توليد شهادات أو فواتير مهنية لأي استخدام."
- "توليد محتوى NSFW مع تعطيل فحوص السلامة."
- "كشط العملاء المحتملين، وإثراء جهات الاتصال، وإطلاق تواصل بارد على نطاق واسع."
- "شراء البيانات المسربة أو تفريغات الاختراق أو نشرها أو تنزيلها."
- "إنشاء حسابات بريد إلكتروني أو حسابات اجتماعية بالجملة بهويات اصطناعية أو حل CAPTCHA."

## ملاحظات للمراجعين

- السياق مهم. يمكن أن يكون الموضوع نفسه مشروعًا ضمن إعداد دفاعي ضيق أو قائم على الموافقة، وغير مقبول عندما يُغلف كسير عمل إساءة استخدام.
- ينبغي أن نميل إلى اتخاذ إجراء عندما تكون المهارة محسّنة بوضوح للتهرب أو الخداع أو الاستخدام غير الرضائي.
- التحميلات المتكررة ضمن هذه الفئات تُعد سببًا لإخفاء المحتوى وحظر الحساب.

## الإنفاذ

- قد نخفي المهارات المخالفة أو نزيلها أو نحذفها حذفًا نهائيًا.
- قد نسحب الرموز، ونحذف المحتوى المرتبط حذفًا مبدئيًا، ونحظر المخالفين المتكررين أو الجسيمين.
- لا نضمن الإنفاذ مع التحذير أولًا في حالات إساءة الاستخدام الواضحة.
