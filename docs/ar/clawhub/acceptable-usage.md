# الاستخدام المقبول

تصف هذه الصفحة أنواع المهارات والمحتوى المقبولة على ClawHub، وسير عمل إساءة الاستخدام التي لن يستضيفها.

هذه القواعد عملية عن قصد. ما نهتم به أكثر هو سير عمل إساءة الاستخدام من البداية إلى النهاية، وليس مجرد كلمات مفتاحية معزولة. إذا كانت المهارة مبنية لتجاوز الدفاعات، أو إساءة استخدام المنصات، أو الاحتيال على الناس، أو انتهاك الخصوصية، أو تمكين سلوك غير رضائي، فهي لا تنتمي إلى ClawHub.

## أنماط حديثة نقبلها صراحة

- عمل الواجهة الأمامية وأنظمة التصميم الذي يستخدم مكونات حقيقية، ورموزًا دلالية، وحالات ميسّرة، وتدفقات مستخدم مختبرة.
- تركيب shadcn/ui الذي يستخدم مكونات المصدر المثبتة، والأسماء المستعارة للمشروع، والمتغيرات الموثقة بدلًا من ترميز مخصص لمرة واحدة.
- تحويل UI5 من JavaScript إلى TypeScript مع الحفاظ على التعليقات، واستخدام أنواع UI5 ملموسة، وإبقاء واجهات عناصر التحكم المولدة قابلة للمراجعة.
- مراجعة الأمان الدفاعية، وأدوات الإشراف، ومطالبات كشف إساءة الاستخدام التي تعرض الأدلة وتبقي حدود الموافقة البشرية واضحة.
- أتمتة سير العمل القائمة على الموافقة للحسابات الشخصية أو حسابات الفرق، مع بيانات اعتماد صريحة، وإعداد شفاف، وأوضاع تشغيل تجريبي أو معاينة.
- التوثيق، وأدلة تشغيل الترحيل، وأدوات المطورين، وتجهيزات الاختبار المحددة بنطاق البرنامج الذي تدعمه.

## غير مقبول

- سير عمل تجاوز الأمان أو الوصول غير المصرح به.
  - أمثلة: تجاوز المصادقة، الاستيلاء على الحسابات، تجاوز CAPTCHA، التحايل على Cloudflare أو أنظمة مكافحة الروبوتات، تجاوز حدود المعدل، الكشط الخفي المصمم لهزيمة إجراءات الحماية، الاستيلاء على مكالمة مباشرة أو وكيل مباشر، سرقة جلسات قابلة لإعادة الاستخدام، الموافقة التلقائية على تدفقات الإقران لمستخدمين غير معتمدين.

- إساءة استخدام المنصات والتحايل على الحظر.
  - أمثلة: حسابات خفية بعد الحظر، تسخين الحسابات أو زراعتها، التفاعل المزيف، تنمية الكارما أو المتابعين، أتمتة الحسابات المتعددة، النشر الجماعي، روبوتات الرسائل المزعجة، أتمتة الأسواق أو الشبكات الاجتماعية المبنية لتجنب الكشف.

- الاحتيال، والخدع، وسير العمل المالي المضلل.
  - أمثلة: شهادات مزيفة، فواتير مزيفة، تدفقات دفع مضللة، تواصل احتيالي، إثبات اجتماعي مزيف، أدوات تمكّن الإنفاق أو تحصيل الرسوم دون موافقة بشرية واضحة وضوابط شفافة، أو سير عمل هويات اصطناعية مبني لإنشاء حسابات بغرض الاحتيال.

- الكشط أو الإثراء أو المراقبة المنتهكة للخصوصية.
  - أمثلة: كشط تفاصيل الاتصال على نطاق واسع للرسائل المزعجة، كشف معلومات الأشخاص، الملاحقة، استخراج العملاء المحتملين مع تواصل غير مرغوب فيه، المراقبة السرية، البحث عن الوجوه أو المطابقة البيومترية المستخدمة دون موافقة واضحة، أو شراء البيانات المسربة أو تفريغات الاختراق أو نشرها أو تنزيلها أو تشغيلها عمليًا.

- الانتحال غير الرضائي أو التلاعب المضلل بالهوية.
  - أمثلة: تبديل الوجوه، التوائم الرقمية، الشخصيات المزيفة، المؤثرون المستنسخون، أو أدوات أخرى للتلاعب بالهوية تُستخدم للانتحال أو التضليل.

- المحتوى الجنسي الصريح وتوليد محتوى للبالغين مع تعطيل السلامة.
  - أمثلة: توليد صور أو فيديوهات أو محتوى NSFW، أغلفة محتوى للبالغين حول واجهات برمجة تطبيقات تابعة لجهات خارجية، أو مهارات يكون هدفها الأساسي محتوى جنسيًا صريحًا.

- متطلبات تنفيذ مخفية أو غير آمنة أو مضللة.
  - أمثلة: أوامر تثبيت مبهمة، `curl | sh`، متطلبات أسرار غير معلنة، استخدام مفاتيح خاصة غير معلن، تنفيذ `npx @latest` عن بُعد دون قابلية مراجعة واضحة، بيانات وصفية مضللة تخفي ما تحتاجه المهارة فعليًا للتشغيل.

## أنماط حديثة لا نقبلها صراحة

- “أنشئ حسابات بائعين خفية بعد حظر الأسواق.”
- “عدّل إقران Telegram بحيث يتلقى المستخدمون غير المعتمدين رموز الإقران تلقائيًا.”
- “نمِّ حسابات Reddit/Twitter بأتمتة لا يمكن كشفها.”
- “ولّد شهادات مهنية أو فواتير لاستخدام عشوائي.”
- “ولّد محتوى NSFW مع تعطيل فحوصات السلامة.”
- “اكشط العملاء المحتملين، وأثرِ جهات الاتصال، وابدأ تواصلًا باردًا على نطاق واسع.”
- “اشترِ البيانات المسربة أو تفريغات الاختراق أو انشرها أو نزّلها.”
- “أنشئ حسابات بريد إلكتروني أو حسابات اجتماعية بكميات كبيرة باستخدام هويات اصطناعية أو حل CAPTCHA.”

## ملاحظات للمراجعين

- السياق مهم. يمكن أن يكون الموضوع نفسه مشروعًا في إطار دفاعي ضيق أو قائم على الموافقة، وغير مقبول عندما يُعبأ كسير عمل لإساءة الاستخدام.
- ينبغي أن نميل إلى اتخاذ إجراء عندما تكون المهارة محسّنة بوضوح للتحايل أو الخداع أو الاستخدام غير الرضائي.
- الرفع المتكرر ضمن هذه الفئات يُعد سببًا لإخفاء المحتوى وحظر الحساب.

## الإنفاذ

- قد نخفي المهارات المخالفة أو نزيلها أو نحذفها حذفًا نهائيًا.
- قد نلغي الرموز، ونحذف المحتوى المرتبط حذفًا مبدئيًا، ونحظر المخالفين المتكررين أو الجسيمين.
- لا نضمن الإنفاذ المسبوق بتحذير في حالات إساءة الاستخدام الواضحة.
