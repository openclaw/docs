# الاستخدام المقبول

تصف هذه الصفحة أنواع Skills والمحتوى الذي يقبله ClawHub، وسير عمل إساءة الاستخدام التي لن يستضيفها.

هذه القواعد عملية عن قصد. ما نهتم به أكثر هو سير عمل إساءة الاستخدام من البداية إلى النهاية، وليس مجرد كلمات مفتاحية معزولة. إذا كانت Skill مبنية لتجاوز الدفاعات، أو إساءة استخدام المنصات، أو الاحتيال على الناس، أو انتهاك الخصوصية، أو تمكين سلوك غير توافقي، فلا مكان لها على ClawHub.

## أنماط حديثة نقبلها صراحة

- أعمال الواجهة الأمامية وأنظمة التصميم التي تستخدم مكونات حقيقية، ورموزًا دلالية، وحالات قابلة للوصول، وتدفقات مستخدم مختبرة.
- تركيب shadcn/ui الذي يستخدم مكونات مصدرية مثبتة، وأسماء مستعارة للمشروع، ومتغيرات موثقة بدلًا من ترميز مخصص لمرة واحدة.
- تحويل UI5 من JavaScript إلى TypeScript مع الحفاظ على التعليقات، واستخدام أنواع UI5 محددة، وإبقاء واجهات عناصر التحكم المولدة قابلة للمراجعة.
- مراجعة الأمان الدفاعية، وأدوات الإشراف، ومطالبات كشف إساءة الاستخدام التي تعرض الأدلة وتبقي حدود موافقة البشر واضحة.
- أتمتة سير العمل القائمة على الموافقة لحسابات شخصية أو حسابات فريق باستخدام بيانات اعتماد صريحة، وإعداد شفاف، وأوضاع تشغيل تجريبي أو معاينة.
- الوثائق، وأدلة تنفيذ الترحيل، وأدوات المطورين، وتثبيتات الاختبار المحددة بنطاق البرنامج الذي تدعمه.

## غير مقبول

- سير عمل تجاوز الأمان أو الوصول غير المصرح به.
  - أمثلة: تجاوز المصادقة، الاستيلاء على الحسابات، تجاوز CAPTCHA، تجاوز Cloudflare أو أنظمة مكافحة الروبوتات، تجاوز حدود المعدل، الاستخراج الخفي المصمم لهزيمة الحمايات، الاستيلاء على مكالمة مباشرة أو وكيل، سرقة جلسات قابلة لإعادة الاستخدام، الموافقة التلقائية على تدفقات الاقتران للمستخدمين غير المعتمدين.

- إساءة استخدام المنصات والتهرب من الحظر.
  - أمثلة: حسابات خفية بعد الحظر، تهيئة الحسابات أو تربيتها، تفاعل مزيف، تنمية الكارما أو المتابعين، أتمتة حسابات متعددة، نشر جماعي، روبوتات رسائل مزعجة، أتمتة سوق أو شبكات اجتماعية مصممة لتجنب الاكتشاف.

- الاحتيال وعمليات النصب وسير العمل المالي المخادع.
  - أمثلة: شهادات مزيفة، فواتير مزيفة، تدفقات دفع مخادعة، تواصل احتيالي، دليل اجتماعي مزيف، أدوات تمكّن الإنفاق أو التحصيل دون موافقة بشرية واضحة وضوابط شفافة، أو سير عمل هويات اصطناعية مبنية لإنشاء حسابات بغرض الاحتيال.

- الاستخراج أو الإثراء أو المراقبة التي تنتهك الخصوصية.
  - أمثلة: استخراج تفاصيل الاتصال على نطاق واسع للرسائل المزعجة، التشهير بالمعلومات الشخصية، المطاردة، استخراج العملاء المحتملين مقرونًا بتواصل غير مرغوب فيه، المراقبة السرية، البحث عن الوجوه أو المطابقة البيومترية المستخدمة دون موافقة واضحة، أو شراء البيانات المسربة أو تفريغات الاختراق أو نشرها أو تنزيلها أو تحويلها إلى عمليات تشغيلية.

- انتحال الهوية غير التوافقي أو التلاعب المخادع بالهوية.
  - أمثلة: تبديل الوجوه، التوائم الرقمية، الشخصيات المزيفة، المؤثرون المستنسخون، أو أدوات أخرى للتلاعب بالهوية تُستخدم للانتحال أو التضليل.

- المحتوى الجنسي الصريح وتوليد محتوى البالغين مع تعطيل إجراءات السلامة.
  - أمثلة: توليد صور أو فيديوهات أو محتوى NSFW، أو أغلفة محتوى البالغين حول APIs تابعة لجهات خارجية، أو Skills يكون غرضها الأساسي محتوى جنسيًا صريحًا.

- متطلبات تشغيل مخفية أو غير آمنة أو مضللة.
  - أمثلة: أوامر تثبيت مبهمة، `curl | sh`، متطلبات أسرار غير مصرّح بها، استخدام مفاتيح خاصة غير مصرّح به، تنفيذ `npx @latest` عن بُعد دون قابلية مراجعة واضحة، بيانات وصفية مضللة تخفي ما تحتاجه Skill فعليًا للتشغيل.

## أنماط حديثة لا نقبلها صراحة

- "إنشاء حسابات بائعين خفية بعد حظرها في الأسواق."
- "تعديل اقتران Telegram بحيث يتلقى المستخدمون غير المعتمدين رموز الاقتران تلقائيًا."
- "تنمية حسابات Reddit/Twitter بأتمتة غير قابلة للاكتشاف."
- "توليد شهادات أو فواتير مهنية لأي استخدام."
- "توليد محتوى NSFW مع تعطيل فحوصات السلامة."
- "استخراج العملاء المحتملين، وإثراء جهات الاتصال، وإطلاق تواصل بارد على نطاق واسع."
- "شراء البيانات المسربة أو تفريغات الاختراق أو نشرها أو تنزيلها."
- "إنشاء حسابات بريد إلكتروني أو حسابات اجتماعية جماعيًا بهويات اصطناعية أو عبر حل CAPTCHA."

## ملاحظات للمراجعين

- السياق مهم. قد يكون الموضوع نفسه مشروعًا في إطار دفاعي ضيق أو قائم على الموافقة، وغير مقبول عندما يُغلّف كسير عمل لإساءة الاستخدام.
- ينبغي أن نميل إلى اتخاذ إجراء عندما تكون Skill محسّنة بوضوح للتهرب أو الخداع أو الاستخدام غير التوافقي.
- الرفع المتكرر ضمن هذه الفئات يعد سببًا لإخفاء المحتوى وحظر الحساب.

## الإنفاذ

- قد نخفي Skills المخالفة أو نزيلها أو نحذفها نهائيًا.
- قد نسحب الرموز، ونحذف المحتوى المرتبط حذفًا مبدئيًا، ونحظر المخالفين المتكررين أو الجسيمين.
- لا نضمن إنفاذًا يبدأ بالتحذير في حالات إساءة الاستخدام الواضحة.
