# الاستخدام المقبول

تصف هذه الصفحة أنواع Skills والمحتوى المقبولة لدى ClawHub، وسير عمل إساءة الاستخدام التي لن يستضيفها.

هذه القواعد عملية عن قصد. يهمنا أكثر سير عمل إساءة الاستخدام من البداية إلى النهاية، وليس مجرد كلمات مفتاحية معزولة. إذا كانت Skill مبنية لتجاوز الدفاعات، أو إساءة استخدام المنصات، أو الاحتيال على الأشخاص، أو انتهاك الخصوصية، أو تمكين سلوك غير توافقي، فلا مكان لها على ClawHub.

## أنماط حديثة نقبلها صراحة

- عمل الواجهة الأمامية وأنظمة التصميم الذي يستخدم مكونات حقيقية، ورموزًا دلالية، وحالات قابلة للوصول، وتدفقات مستخدم مختبرة.
- تركيب shadcn/ui الذي يستخدم مكونات مصدرية مثبتة، وأسماء مستعارة للمشروع، ومتغيرات موثقة بدلًا من ترميز مخصص لمرة واحدة.
- تحويل UI5 من JavaScript إلى TypeScript مع الحفاظ على التعليقات، واستخدام أنواع UI5 ملموسة، وإبقاء واجهات عناصر التحكم المولدة قابلة للمراجعة.
- مراجعة أمنية دفاعية، وأدوات إشراف، ومطالبات لاكتشاف إساءة الاستخدام تعرض أدلة وتبقي حدود موافقة الإنسان واضحة.
- أتمتة سير عمل قائمة على الموافقة للحسابات الشخصية أو حسابات الفرق مع بيانات اعتماد صريحة، وإعداد شفاف، وأوضاع تشغيل تجريبي أو معاينة.
- توثيق، ودفاتر تشغيل للترحيل، وأدوات للمطورين، وتجهيزات اختبار محددة النطاق للبرمجيات التي تدعمها.

## غير مقبول

- سير عمل لتجاوز الأمن أو الوصول غير المصرح به.
  - أمثلة: تجاوز المصادقة، الاستيلاء على الحسابات، تجاوز CAPTCHA، تجاوز Cloudflare أو أنظمة مكافحة الروبوتات، تجاوز حدود المعدل، استخراج بيانات خفي مصمم لهزيمة وسائل الحماية، الاستيلاء على مكالمة مباشرة أو وكيل مباشر، سرقة جلسات قابلة لإعادة الاستخدام، الموافقة التلقائية على تدفقات الاقتران لمستخدمين غير معتمدين.

- إساءة استخدام المنصات وتجنب الحظر.
  - أمثلة: حسابات خفية بعد الحظر، تهيئة الحسابات أو زراعتها، تفاعل مزيف، تنمية الكارما أو المتابعين، أتمتة متعددة الحسابات، نشر جماعي، روبوتات سبام، أتمتة أسواق أو شبكات اجتماعية مبنية لتجنب الاكتشاف.

- احتيال، وعمليات نصب، وسير عمل مالي مضلل.
  - أمثلة: شهادات مزيفة، فواتير مزيفة، تدفقات دفع مضللة، تواصل احتيالي، دليل اجتماعي مزيف، أدوات تمكّن الإنفاق أو التحصيل دون موافقة بشرية واضحة وضوابط شفافة، أو سير عمل لهويات اصطناعية مبنية لإنشاء حسابات لأغراض الاحتيال.

- استخراج بيانات أو إثراء أو مراقبة منتهكة للخصوصية.
  - أمثلة: استخراج تفاصيل الاتصال على نطاق واسع للسبام، كشف معلومات شخصية، ملاحقة، استخراج عملاء محتملين مقترن بتواصل غير مرغوب فيه، مراقبة سرية، بحث عن الوجوه أو مطابقة بيومترية تُستخدم دون موافقة واضحة، أو شراء بيانات مسربة أو تفريغات اختراق أو نشرها أو تنزيلها أو تفعيلها تشغيليًا.

- انتحال غير توافقي أو تلاعب مضلل بالهوية.
  - أمثلة: تبديل الوجوه، التوائم الرقمية، الشخصيات المزيفة، المؤثرون المستنسخون، أو أدوات أخرى للتلاعب بالهوية تُستخدم للانتحال أو التضليل.

- محتوى جنسي صريح وتوليد محتوى للبالغين مع تعطيل السلامة.
  - أمثلة: توليد صور/فيديو/محتوى NSFW، أغلفة محتوى للبالغين حول واجهات API تابعة لجهات خارجية، أو Skills يكون غرضها الأساسي محتوى جنسيًا صريحًا.

- متطلبات تنفيذ مخفية أو غير آمنة أو مضللة.
  - أمثلة: أوامر تثبيت مبهمة، `curl | sh`، متطلبات أسرار غير معلنة، استخدام مفاتيح خاصة غير معلن، تنفيذ `npx @latest` عن بُعد دون قابلية مراجعة واضحة، بيانات وصفية مضللة تخفي ما تحتاجه Skill فعليًا للتشغيل.

## أنماط حديثة لا نقبلها صراحة

- “إنشاء حسابات بائع خفية بعد الحظر في الأسواق.”
- “تعديل اقتران Telegram بحيث يتلقى المستخدمون غير المعتمدين رموز الاقتران تلقائيًا.”
- “تنمية حسابات Reddit/Twitter بأتمتة غير قابلة للاكتشاف.”
- “إنشاء شهادات مهنية أو فواتير لأي استخدام.”
- “توليد محتوى NSFW مع تعطيل فحوصات السلامة.”
- “استخراج عملاء محتملين، وإثراء جهات الاتصال، وإطلاق تواصل بارد على نطاق واسع.”
- “شراء بيانات مسربة أو تفريغات اختراق أو نشرها أو تنزيلها.”
- “إنشاء حسابات بريد إلكتروني أو حسابات اجتماعية بالجملة باستخدام هويات اصطناعية أو حل CAPTCHA.”

## ملاحظات للمراجعين

- السياق مهم. يمكن أن يكون الموضوع نفسه مشروعًا في إطار دفاعي ضيق أو قائم على الموافقة، وغير مقبول عندما يكون مغلفًا كسير عمل لإساءة الاستخدام.
- يجب أن نميل إلى اتخاذ إجراء عندما تكون Skill محسّنة بوضوح للتجنب أو الخداع أو الاستخدام غير التوافقي.
- الرفع المتكرر ضمن هذه الفئات يُعد سببًا لإخفاء المحتوى وحظر الحساب.

## الإنفاذ

- قد نخفي Skills المخالفة أو نزيلها أو نحذفها حذفًا نهائيًا.
- قد نسحب الرموز، ونحذف المحتوى المرتبط حذفًا مبدئيًا، ونحظر المخالفين المتكررين أو الجسيمين.
- لا نضمن إنفاذًا مسبوقًا بالتحذير في حالات إساءة الاستخدام الواضحة.
