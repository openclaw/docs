# الاستخدام المقبول

يستضيف ClawHub‏ Skills وPlugins والحزم وبيانات السوق الوصفية الخاصة بـ OpenClaw.
استخدم هذه الصفحة لتحديد ما إذا كان المحتوى أو سلوك النشر مناسبًا على
ClawHub.

تنطبق هذه القواعد على ما تفعله القائمة، وما تطلب من المستخدمين تشغيله، وكيف
تقدم نفسها، وكيف يستخدم الناشرون واجهات الاكتشاف والتثبيت والثقة في ClawHub.
للاطلاع على حالات الإشراف ووضع الحساب، راجع
[الإشراف وسلامة الحساب](/clawhub/moderation). ولمطالبات حقوق النشر أو الحقوق الأخرى،
راجع [طلبات حقوق المحتوى](/ar/clawhub/content-rights).

## المحتوى المسموح به

يرحب ClawHub بالمحتوى المفيد والواضح والمنشور بحسن نية.

| الفئة                                         | مسموح به عندما                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |

| واجهات المستخدم والبيانات وسير عمل الأتمتة               | يكون النطاق واضحًا، وبيانات الاعتماد المطلوبة صريحة، وتتضمن الإجراءات الخطرة مسارات مراجعة أو تشغيل تجريبي أو معاينة أو تأكيد. |
| الأمن الدفاعي والإشراف ومراجعة إساءة الاستخدام | تُعرض الأداة لغرض المراجعة المصرح بها، وتحافظ على الأدلة، وتُبقي حدود الموافقة البشرية واضحة.                          |

السياق مهم. يمكن أن يكون الموضوع نفسه مقبولًا في إعداد دفاعي ضيق أو قائم على
الموافقة، وغير مقبول عندما يُعبأ كسير عمل لإساءة الاستخدام.

## سلوك السوق المحظور

يراجع ClawHub أيضًا كيفية استخدام الناشرين للسوق. لا تستخدم ClawHub للتلاعب
بالاكتشاف أو المقاييس أو إشارات الثقة أو أنظمة الإشراف أو انتباه المستخدمين.

يشمل سلوك السوق المحظور ما يلي:

- نشر أعداد كبيرة دفعة واحدة من قوائم منخفضة الجهد أو مكررة أو نائبة أو
  مولدة آليًا ولا يبدو أن لها قيمة حقيقية للمستخدمين
- إغراق واجهات البحث أو الفئات بمهارات أو Plugins شبه متطابقة
- نشر مئات القوائم مع استخدام أو صيانة أو وضوح مصدر أو تمايز ذي معنى قليل أو معدوم
- تضخيم التثبيتات أو التنزيلات أو النجوم أو مقاييس التفاعل الأخرى اصطناعيًا
  عبر الأتمتة أو حلقات التثبيت الذاتي أو الحسابات المزيفة أو النشاط المنسق
  أو التفاعل المدفوع أو أي سلوك آخر غير عضوي
- إنشاء الحسابات أو تدويرها لتجنب الإشراف أو الحظر أو حدود الناشرين أو
  مراجعة السوق
- تضليل المستخدمين بشأن الملكية أو المصدر أو القدرات أو الوضع الأمني أو
  متطلبات التثبيت أو الانتماء إلى مشروع أو ناشر آخر
- رفع محتوى مرارًا سبق إخفاؤه أو إزالته أو حظره
  دون إصلاح المشكلة الأساسية

النشر بكميات كبيرة ليس إساءة استخدام تلقائيًا. تكون الكتالوجات الكبيرة مقبولة
عندما تكون القوائم مختلفة على نحو ذي معنى، وموصوفة بدقة، ومُصانة،
ويستخدمها مستخدمون حقيقيون. تصبح الكتالوجات الكبيرة مشكلة ثقة وسلامة عندما
يقترن الحجم بقوائم سطحية أو مكررة أو مضللة أو غير مُصانة أو
مُروّج لها اصطناعيًا.

## حقوق المحتوى

إذا كنت تعتقد أن محتوى على ClawHub ينتهك حقوق النشر الخاصة بك أو حقوقًا أخرى، فاستخدم
[طلبات حقوق المحتوى](/ar/clawhub/content-rights). لا تستخدم بلاغات السوق العادية
لمطالبات حقوق النشر أو الحقوق إلا إذا كانت القائمة غير آمنة أيضًا
أو خبيثة أو مضللة.

## المراجعة والإنفاذ

قد يستخدم ClawHub فحوصات آلية وإشارات إحصائية لإساءة الاستخدام وبلاغات المستخدمين
ومراجعة الموظفين لتحديد المحتوى غير الآمن أو سلوك النشر المسيء. لا تثبت الإشارة
إساءة الاستخدام بحد ذاتها؛ لكنها تساعد ClawHub على تحديد ما يحتاج إلى مراجعة.

قد نقوم بما يلي:

- إخفاء القوائم المخالفة أو تعليقها أو إزالتها أو حذفها حذفًا مرنًا أو، حيث يدعم نوع المورد ذلك،
  حذفها حذفًا نهائيًا
- حظر التنزيلات أو التثبيتات للإصدارات غير الآمنة
- إبطال رموز API
- حذف المحتوى المرتبط حذفًا مرنًا
- تقييد صلاحية النشر
- حظر المخالفين المتكررين أو الجسيمين

لا نضمن الإنفاذ بعد التحذير أولًا في حالات إساءة الاستخدام الواضحة. راجع
[الإشراف وسلامة الحساب](/clawhub/moderation) للاطلاع على البلاغات وتعليقات الإشراف
والقوائم المخفية والحظر ووضع الحساب.
