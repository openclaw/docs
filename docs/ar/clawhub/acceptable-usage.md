# الاستخدام المقبول

تستضيف ClawHub Skills وPlugin والحزم وبيانات تعريف السوق الخاصة بـ OpenClaw.
استخدم هذه الصفحة لتحديد ما إذا كان المحتوى أو سلوك النشر ينتمي إلى
ClawHub.

تنطبق هذه القواعد على ما يفعله الإدراج، وما يطلب من المستخدمين تشغيله، وكيف
يمثل نفسه، وكيف يستخدم الناشرون أسطح الاكتشاف والتثبيت والثقة في ClawHub. لحالات الإشراف ووضع الحساب، راجع
[الإشراف وسلامة الحساب](/ar/clawhub/moderation). لمطالبات حقوق النشر أو الحقوق الأخرى،
راجع [طلبات حقوق المحتوى](/ar/clawhub/content-rights).

## المحتوى المسموح

ترحب ClawHub بالمحتوى المفيد والمفهوم والمنشور بحسن نية.

| الفئة | مسموح عندما |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| إنتاجية المطورين | يساعد الإدراج المستخدمين على بناء البرامج أو اختبارها أو ترحيلها أو تصحيحها أو توثيقها أو تشغيلها. |
| واجهة المستخدم والبيانات وسير عمل الأتمتة | يكون النطاق واضحا، وبيانات الاعتماد المطلوبة صريحة، وتتضمن الإجراءات الخطرة مسارات مراجعة أو تشغيل تجريبي أو معاينة أو تأكيد. |
| الأمن الدفاعي والإشراف ومراجعة إساءة الاستخدام | تكون الأداة مؤطرة للمراجعة المصرح بها، وتحافظ على الأدلة، وتبقي حدود الموافقة البشرية واضحة. |
| سير العمل الشخصي أو الجماعي | يستخدم سير العمل حسابات قائمة على الموافقة، وإعدادا شفافا، وأذونات صريحة. |
| الفهارس المصانة | يكون كل إدراج مميزا ومفيدا وموصوفا بدقة ومصانا بدرجة معقولة. |

السياق مهم. قد يكون الموضوع نفسه مقبولا في إعداد دفاعي ضيق أو قائم على
الموافقة، وغير مقبول عندما يعبأ كسير عمل لإساءة الاستخدام.

## المحتوى غير المسموح

لا تستضيف ClawHub محتوى يكون غرضه الرئيسي إساءة الاستخدام أو الخداع أو التنفيذ
غير الآمن أو انتهاك الحقوق.

| الفئة | غير مسموح |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| الوصول غير المصرح به أو تجاوز الأمان | تجاوز المصادقة، أو الاستيلاء على الحساب، أو إساءة استخدام حدود المعدل، أو الاستيلاء على مكالمة مباشرة أو وكيل، أو سرقة جلسة قابلة لإعادة الاستخدام، أو الموافقة التلقائية على تدفقات الاقتران لمستخدمين غير معتمدين. |
| إساءة استخدام المنصات والالتفاف على الحظر | الحسابات المتخفية بعد الحظر، أو تهيئة الحسابات أو زراعتها، أو التفاعل الزائف، أو أتمتة الحسابات المتعددة، أو النشر الجماعي، أو روبوتات الرسائل المزعجة، أو الأتمتة المبنية لتجنب الاكتشاف. |
| الاحتيال وعمليات النصب وسير العمل المالي الخادع | شهادات أو فواتير مزيفة، أو تدفقات دفع خادعة، أو تواصل احتيالي، أو دليل اجتماعي مزيف، أو سير عمل هويات اصطناعية للاحتيال، أو أدوات إنفاق/تحصيل رسوم دون موافقة بشرية واضحة. |
| الإثراء أو المراقبة المتطفلة على الخصوصية | استخراج جهات الاتصال للرسائل المزعجة، أو كشف المعلومات الشخصية، أو المطاردة، أو استخراج العملاء المحتملين المقترن بتواصل غير مرغوب، أو المراقبة السرية، أو المطابقة البيومترية دون موافقة، أو استخدام بيانات مسربة أو تفريغات اختراق. |
| الانتحال أو التلاعب بالهوية دون موافقة | تبديل الوجوه، أو التوائم الرقمية، أو المؤثرون المستنسخون، أو الشخصيات المزيفة، أو أدوات أخرى تستخدم للانتحال أو التضليل. |
| المحتوى الجنسي الصريح أو توليد محتوى للبالغين مع تعطيل السلامة | توليد صور أو فيديو أو محتوى غير آمن للعمل؛ أو أغلفة محتوى للبالغين حول واجهات API تابعة لجهات خارجية؛ أو إدراجات يكون غرضها الأساسي محتوى جنسيا صريحا. |
| متطلبات تنفيذ مخفية أو غير آمنة أو مضللة | أوامر تثبيت مبهمة، أو مثبتات تمرر إلى shell مثل المحتوى المنزل الذي يشغل باستخدام `sh` أو `bash` دون قابلية مراجعة واضحة، أو متطلبات أسرار أو مفاتيح خاصة غير معلنة، أو تنفيذ `npx @latest` عن بعد دون قابلية مراجعة واضحة، أو بيانات تعريف تخفي ما يحتاجه الإدراج فعليا للتشغيل. |
| مواد منتهكة لحقوق النشر أو مخالفة للحقوق | إعادة نشر Skills أو Plugin أو وثائق أو أصول علامة تجارية أو كود مملوك لشخص آخر دون إذن؛ أو انتهاك شروط الترخيص؛ أو انتحال المؤلف أو الناشر الأصلي. |

## سلوك السوق غير المسموح

تراجع ClawHub أيضا كيفية استخدام الناشرين للسوق. لا تستخدم ClawHub
للتلاعب بالاكتشاف أو المقاييس أو إشارات الثقة أو أنظمة الإشراف أو انتباه
المستخدمين.

يشمل سلوك السوق غير المسموح:

- نشر أعداد كبيرة بالجملة من إدراجات قليلة الجهد أو مكررة أو مؤقتة أو
  مولدة آليا لا يبدو أن لها قيمة حقيقية للمستخدمين
- إغراق أسطح البحث أو الفئات بمهارات أو Plugin شبه متطابقة
- نشر مئات الإدراجات مع استخدام قليل أو معدوم، أو صيانة قليلة أو معدومة، أو وضوح
  مصدر قليل أو معدوم، أو تمايز ذي معنى قليل أو معدوم
- تضخيم عمليات التثبيت أو التنزيلات أو النجوم أو مقاييس التفاعل الأخرى
  اصطناعيا عبر الأتمتة، أو حلقات التثبيت الذاتي، أو الحسابات المزيفة، أو النشاط
  المنسق، أو التفاعل المدفوع، أو أي سلوك آخر غير عضوي
- إنشاء الحسابات أو تدويرها للالتفاف على الإشراف أو الحظر أو حدود الناشرين أو
  مراجعة السوق
- تضليل المستخدمين بشأن الملكية أو المصدر أو القدرات أو الوضع الأمني أو
  متطلبات التثبيت أو الارتباط بمشروع أو ناشر آخر
- رفع محتوى مرارا سبق إخفاؤه أو إزالته أو حظره
  دون إصلاح المشكلة الأساسية

النشر بكثافة عالية ليس إساءة استخدام تلقائيا. تكون الفهارس الكبيرة مقبولة
عندما تكون الإدراجات مختلفة بشكل ذي معنى، وموصوفة بدقة، ومصانة،
ويستخدمها مستخدمون حقيقيون. تصبح الفهارس الكبيرة مشكلة ثقة وسلامة عندما
يقترن الحجم بإدراجات سطحية أو مكررة أو مضللة أو غير مصانة أو
مروجة اصطناعيا.

## حقوق المحتوى

إذا كنت تعتقد أن محتوى على ClawHub ينتهك حقوق النشر الخاصة بك أو حقوقا أخرى، فاستخدم
[طلبات حقوق المحتوى](/ar/clawhub/content-rights). لا تستخدم تقارير السوق العادية
لمطالبات حقوق النشر أو الحقوق إلا إذا كان الإدراج أيضا غير آمن
أو خبيثا أو مضللا.

## المراجعة والإنفاذ

قد تستخدم ClawHub فحوصا آلية، وإشارات إحصائية لإساءة الاستخدام، وتقارير المستخدمين،
ومراجعة الموظفين لتحديد المحتوى غير الآمن أو سلوك النشر المسيء. لا تثبت الإشارة
إساءة الاستخدام بحد ذاتها؛ بل تساعد ClawHub على تحديد ما يحتاج إلى مراجعة.

قد نقوم بما يلي:

- إخفاء الإدراجات المخالفة أو تعليقها أو إزالتها أو حذفها حذفا لينا، أو، حيث يكون ذلك مدعوما لنوع المورد،
  حذفها حذفا نهائيا
- حظر التنزيلات أو عمليات التثبيت للإصدارات غير الآمنة
- إبطال رموز API
- حذف المحتوى المرتبط حذفا لينا
- تقييد صلاحية النشر
- حظر المخالفين المتكررين أو الجسيمين

لا نضمن الإنفاذ بعد تحذير أولا في حالات إساءة الاستخدام الواضحة. راجع
[الإشراف وسلامة الحساب](/ar/clawhub/moderation) للتقارير، وتعليقات الإشراف،
والإدراجات المخفية، والحظر، ووضع الحساب.
