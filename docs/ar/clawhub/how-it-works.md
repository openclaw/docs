# كيف يعمل ClawHub

ClawHub هو طبقة السجلّ لـ Skills وplugins في OpenClaw. فهو يوفّر للمستخدمين
مكانًا لاكتشاف الحِزم، ويوفّر للناشرين مكانًا لإصدار النُسخ، ويمنح
OpenClaw بيانات وصفية كافية لتثبيت تلك الحِزم وتحديثها بأمان.

## سجلات السجلّ

كل إدراج عام هو سجلّ في السجلّ يتضمن:

- مالكًا ومُعرّفًا قصيرًا أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية، وملخصًا، وملفات، ونَسبًا إلى المصدر
- سجل تغييرات ومعلومات وسوم مثل `latest`
- إشارات التنزيل، والتثبيت، والنجوم، والتعليقات
- حالة فحص الأمان والإشراف

صفحة الإدراج هي المكان المعياري الذي يستخدمه المستخدمون لفحص ما تزعم مهارة أو
plugin أنها تفعله قبل تثبيتها.

## Skills

المهارة هي حزمة نصية ذات نُسخ تتمحور حول `SKILL.md`. ويمكن أن تتضمن
ملفات داعمة، وأمثلة، وقوالب، ونصوصًا برمجية.

يقرأ ClawHub بيانات الواجهة في `SKILL.md` لفهم اسم المهارة،
ووصفها، ومتطلباتها، ومتغيرات البيئة، وبياناتها الوصفية. دقة
البيانات الوصفية مهمة لأنها تساعد المستخدمين على تحديد ما إذا كانوا سيثبتون المهارة،
وتساعد الفحوص الآلية على اكتشاف حالات عدم التطابق بين السلوك المعلن والمرصود.

راجع [تنسيق المهارة](/ar/clawhub/skill-format).

## Plugins

Plugins هي امتدادات OpenClaw مُغلّفة في حِزم. يخزّن ClawHub بيانات الحزمة الوصفية،
ومعلومات التوافق، وروابط المصدر، والآثار، وسجلات النُسخ.

عندما يثبّت OpenClaw plugin من ClawHub، فإنه يتحقق من بيانات التوافق
الوصفية المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحِزم توافق API،
والحد الأدنى لإصدار Gateway، وأهداف المضيف، ومتطلبات البيئة، وبصمات
الآثار.

استخدم مصدر تثبيت ClawHub صريحًا عندما تريد أن يكون السجلّ هو
مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل نسخة جديدًا غير قابل للتغيير. يستخدم الناشرون CLI الخاص بـ `clawhub`
لسير عمل السجلّ المُصادَق عليه:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم التشغيل التجريبي لمعاينة الحمولة المحلولة قبل الرفع. ثم تعرض الصفحات العامة
البيانات الوصفية المنشورة، والملفات، ونَسب المصدر، وحالة الفحص.

## التثبيتات والتحديثات

تستخدم أوامر التثبيت في OpenClaw ClawHub كمصدر للحِزم:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

يسجّل OpenClaw بيانات وصفية لمصدر التثبيت حتى تتمكن التحديثات من حل حزمة
السجلّ نفسها لاحقًا. كما يدعم CLI الخاص بـ ClawHub سير عمل تثبيت المهارات
وتحديثها مباشرةً للمستخدمين الذين يريدون مجلدات مهارات مُدارة عبر السجلّ خارج
مساحة عمل OpenClaw كاملة.

## حالة الأمان

ClawHub مفتوح للنشر، لكن الإصدارات تظل خاضعة لبوابات الرفع،
والفحوص الآلية، وبلاغات المستخدمين، وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. قد يختفي المحتوى المحتجز، أو المخفي،
أو المحظور من البحث العام وتدفقات التثبيت، مع بقائه
مرئيًا للمالك لأغراض التشخيص.

راجع [الأمان + الإشراف](/ar/clawhub/security) و
[الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## الوصول إلى API

يوفّر ClawHub واجهات API قراءة عامة للاكتشاف، والبحث، وتفاصيل الحِزم، وعمليات
التنزيل. يمكن للفهارس التابعة لجهات خارجية استخدام واجهات API هذه عندما تربط إلى
إدراج ClawHub المعياري، وتحترم حدود المعدل، وتتجنب الإيحاء بالتأييد.

راجع [API العام](/ar/clawhub/api) و[HTTP API](/ar/clawhub/http-api).
