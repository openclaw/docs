# كيف يعمل ClawHub

ClawHub هي طبقة السجل لـ Skills وPlugins في OpenClaw. فهي تمنح المستخدمين
مكانًا لاكتشاف الحزم، وتمنح الناشرين مكانًا لإصدار النسخ، وتمنح
OpenClaw بيانات وصفية كافية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يحتوي على:

- مالك واسم مختصر أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية وملخص وملفات ونسبة المصدر
- سجل تغييرات ومعلومات وسوم مثل `latest`
- مؤشرات التنزيل والتثبيت والنجوم والتعليقات
- حالة فحص الأمان والإشراف

صفحة الإدراج هي المكان المعتمد الذي يطّلع فيه المستخدمون على ما تزعم Skill أو
Plugin أنها تفعله قبل تثبيتها.

## Skills

Skill هي حزمة نصية ذات نسخ تتمحور حول `SKILL.md`. ويمكن أن تتضمن
ملفات داعمة وأمثلة وقوالب وسكربتات.

يقرأ ClawHub بيانات الواجهة الأمامية في `SKILL.md` لفهم اسم Skill
ووصفها ومتطلباتها ومتغيرات البيئة وبياناتها الوصفية. دقة البيانات
الوصفية مهمة لأنها تساعد المستخدمين على تحديد ما إذا كانوا سيثبتون Skill، كما
تساعد الفحوصات الآلية على اكتشاف حالات عدم التطابق بين السلوك المعلن والسلوك المرصود.

راجع [تنسيق Skill](/ar/clawhub/skill-format).

## Plugins

Plugins هي إضافات OpenClaw مُغلّفة. يخزن ClawHub بيانات الحزمة الوصفية
ومعلومات التوافق وروابط المصدر والقطع الأثرية وسجلات النسخ.

عندما يثبّت OpenClaw أحد Plugins من ClawHub، فإنه يتحقق من بيانات التوافق
الوصفية المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحزم توافق API،
والحد الأدنى لإصدار Gateway، وأهداف المضيف، ومتطلبات البيئة، وبصمات
القطع الأثرية.

استخدم مصدر تثبيت ClawHub صريحًا عندما تريد أن يكون السجل هو
مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل نسخة جديدًا غير قابل للتغيير. يستخدم الناشرون CLI الخاص بـ `clawhub`
لسير عمل السجل الموثّق:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم التشغيلات التجريبية لمعاينة الحمولة المحلولة قبل الرفع. تعرض الصفحات العامة بعد ذلك
البيانات الوصفية المنشورة والملفات ونسبة المصدر وحالة الفحص.

## التثبيتات والتحديثات

تستخدم أوامر تثبيت OpenClaw‏ ClawHub كمصدر حزم:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

يسجل OpenClaw بيانات وصفية لمصدر التثبيت حتى تتمكن التحديثات لاحقًا من حل
حزمة السجل نفسها. يدعم CLI الخاص بـ ClawHub أيضًا مسارات عمل تثبيت Skills
وتحديثها مباشرة للمستخدمين الذين يريدون مجلدات Skills مُدارة عبر السجل خارج
مساحة عمل OpenClaw كاملة.

## حالة الأمان

ClawHub مفتوح للنشر، لكن الإصدارات تظل خاضعة لبوابات الرفع
والفحوصات الآلية وبلاغات المستخدمين وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. قد يختفي المحتوى المحتجز أو المخفي
أو المحظور من البحث العام ومسارات التثبيت، مع بقائه مرئيًا للمالك لأغراض التشخيص.

راجع [الأمان + الإشراف](/ar/clawhub/security) و
[الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## الوصول إلى API

يوفر ClawHub واجهات API قراءة عامة للاكتشاف والبحث وتفاصيل الحزم
والتنزيلات. يمكن للفهرسات التابعة لأطراف ثالثة استخدام واجهات API هذه عندما تربط
مرة أخرى بإدراج ClawHub المعتمد، وتحترم حدود المعدل، وتتجنب الإيحاء بالتأييد.

راجع [API العامة](/ar/clawhub/api) و[HTTP API](/ar/clawhub/http-api).
