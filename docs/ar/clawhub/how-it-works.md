# كيف يعمل ClawHub

ClawHub هو طبقة السجل لمهارات OpenClaw وPlugins. يوفّر للمستخدمين
مكانًا لاكتشاف الحِزم، ويوفّر للناشرين مكانًا لإصدار النسخ، ويمنح
OpenClaw بيانات وصفية كافية لتثبيت تلك الحِزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يحتوي على:

- مالك واسم مختصر أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية وملخص وملفات ونسبة إلى المصدر
- سجل تغييرات ومعلومات وسوم مثل `latest`
- إشارات التنزيل والتثبيت والنجوم
- حالة الفحص الأمني والإشراف

صفحة الإدراج هي المكان المرجعي الذي يتيح للمستخدمين فحص ما تدّعي المهارة أو
Plugin أنها تفعله قبل تثبيتها.

## Skills

المهارة هي حزمة نصية ذات نسخ تتمحور حول `SKILL.md`. ويمكن أن تتضمن
ملفات داعمة وأمثلة وقوالب وسكربتات.

يقرأ ClawHub واجهة `SKILL.md` الأمامية لفهم اسم المهارة
ووصفها ومتطلباتها ومتغيرات البيئة والبيانات الوصفية. دقة
البيانات الوصفية مهمة لأنها تساعد المستخدمين على تحديد ما إذا كانوا سيثبتون المهارة
وتساعد الفحوصات الآلية على اكتشاف حالات عدم التطابق بين السلوك المعلن والمرصود.

راجع [تنسيق المهارة](/ar/clawhub/skill-format).

## Plugins

Plugins هي امتدادات OpenClaw معبّأة. يخزّن ClawHub بيانات الحزمة الوصفية،
ومعلومات التوافق، وروابط المصدر، والقطع الأثرية، وسجلات النسخ.

عندما يثبت OpenClaw Plugin من ClawHub، فإنه يتحقق من بيانات التوافق الوصفية
المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحِزم توافق API،
والحد الأدنى لإصدار Gateway، وأهداف المضيف، ومتطلبات البيئة، وملخصات
القطع الأثرية.

استخدم مصدر تثبيت صريحًا من ClawHub عندما تريد أن يكون السجل هو
مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل نسخة جديدًا غير قابل للتغيير. يستخدم الناشرون CLI الخاص بـ `clawhub`
لسير عمل السجل الموثّق:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم التشغيلات التجريبية لمعاينة الحمولة المحلولة قبل الرفع. بعد ذلك
تعرض الصفحات العامة البيانات الوصفية المنشورة، والملفات، ونسبة المصدر، وحالة الفحص.

## التثبيتات والتحديثات

تستخدم أوامر تثبيت OpenClaw ClawHub كمصدر للحِزم:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

يسجل OpenClaw بيانات وصفية لمصدر التثبيت كي تتمكن التحديثات من حل حزمة
السجل نفسها لاحقًا. يدعم ClawHub CLI أيضًا سير عمل تثبيت المهارات
وتحديثها مباشرة للمستخدمين الذين يريدون مجلدات مهارات مُدارة عبر السجل خارج
مساحة عمل OpenClaw كاملة.

## حالة الأمان

ClawHub مفتوح للنشر، لكن الإصدارات تظل خاضعة لبوابات الرفع،
والفحوصات الآلية، وبلاغات المستخدمين، وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. قد يختفي المحتوى المعلّق أو المخفي
أو المحظور من البحث العام وتدفقات التثبيت، مع بقائه
مرئيًا للمالك لأغراض التشخيص.

راجع [الأمان](/clawhub/security)، و[تدقيقات الأمان](/clawhub/security-audits)،
و[الإشراف وسلامة الحساب](/ar/clawhub/moderation)، و
[الاستخدام المقبول](/clawhub/acceptable-usage).

## الوصول إلى API

يوفّر ClawHub واجهات API عامة للقراءة من أجل الاكتشاف والبحث وتفاصيل الحِزم
والتنزيلات. يمكن للكتالوجات التابعة لجهات خارجية استخدام واجهات API هذه عندما تربط إلى
إدراج ClawHub المرجعي، وتحترم حدود المعدل، وتتجنب الإيحاء بالتزكية.

راجع [API العامة](/clawhub/api) و[HTTP API](/clawhub/http-api).
