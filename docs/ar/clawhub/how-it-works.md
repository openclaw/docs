# كيف يعمل ClawHub

ClawHub هو طبقة السجل لمهارات OpenClaw وإضافاته. يوفّر للمستخدمين
مكانًا لاكتشاف الحزم، ويوفّر للناشرين مكانًا لإصدار النسخ، ويوفّر
لـ OpenClaw ما يكفي من البيانات الوصفية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يتضمن:

- مالكًا ومعرّفًا مختصرًا أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية وملخصًا وملفات وإسنادًا للمصدر
- سجل تغييرات ومعلومات وسوم مثل `latest`
- إشارات التنزيل والتثبيت والنجوم
- حالة فحص الأمان والإشراف

صفحة الإدراج هي المكان المرجعي للمستخدمين لفحص ما تدّعي مهارة أو
إضافة أنها تفعله قبل تثبيتها.

## Skills

المهارة هي حزمة نصية ذات إصدارات تتمحور حول `SKILL.md`. ويمكن أن تتضمن
ملفات داعمة وأمثلة وقوالب ونصوصًا برمجية.

يقرأ ClawHub بيانات الواجهة في `SKILL.md` لفهم اسم المهارة
ووصفها ومتطلباتها ومتغيرات البيئة وبياناتها الوصفية. تهم دقة
البيانات الوصفية لأنها تساعد المستخدمين على تحديد ما إذا كانوا سيثبتون المهارة،
وتساعد الفحوصات الآلية على اكتشاف عدم التطابق بين السلوك المعلن والسلوك المرصود.

راجع [تنسيق المهارة](/clawhub/skill-format).

## الإضافات

الإضافات هي امتدادات OpenClaw مُغلّفة في حزم. يخزّن ClawHub بيانات الحزمة الوصفية
ومعلومات التوافق وروابط المصدر والقطع الأثرية وسجلات الإصدارات.

عندما يثبّت OpenClaw إضافة من ClawHub، فإنه يتحقق من بيانات التوافق الوصفية
المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحزم توافق API،
والحد الأدنى من إصدار Gateway، وأهداف المضيف، ومتطلبات البيئة، وملخصات
القطع الأثرية.

استخدم مصدر تثبيت ClawHub صريحًا عندما تريد أن يكون السجل هو
مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل إصدار جديدًا غير قابل للتغيير. يستخدم الناشرون CLI الخاص بـ `clawhub`
لسير عمل السجل الموثّق:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم عمليات التشغيل التجريبي لمعاينة الحمولة المحلولة قبل الرفع. ثم تعرض
الصفحات العامة البيانات الوصفية المنشورة والملفات وإسناد المصدر وحالة الفحص.

## التثبيتات والتحديثات

تستخدم أوامر تثبيت OpenClaw ClawHub كمصدر للحزم:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

يسجّل OpenClaw بيانات مصدر التثبيت الوصفية كي تتمكن التحديثات من حل
حزمة السجل نفسها لاحقًا. يدعم CLI الخاص بـ ClawHub أيضًا عمليات تثبيت المهارات
وتحديثها مباشرة للمستخدمين الذين يريدون مجلدات مهارات مُدارة عبر السجل خارج
مساحة عمل OpenClaw كاملة.

## حالة الأمان

ClawHub مفتوح للنشر، لكن الإصدارات تظل خاضعة لبوابات الرفع
والفحوصات الآلية وبلاغات المستخدمين وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. قد يختفي المحتوى المعلّق أو المخفي
أو المحظور من البحث العام ومسارات التثبيت، مع بقائه
مرئيًا للمالك لأغراض التشخيص.

راجع [الأمان](/ar/clawhub/security)، و[تدقيقات الأمان](/clawhub/security-audits)،
و[الإشراف وسلامة الحساب](/clawhub/moderation)، و
[الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## الوصول إلى API

يوفّر ClawHub واجهات API عامة للقراءة من أجل الاكتشاف والبحث وتفاصيل الحزم
والتنزيلات. يمكن للفهارس التابعة لجهات خارجية استخدام واجهات API هذه عندما تعود بروابط إلى
إدراج ClawHub المرجعي، وتحترم حدود المعدل، وتتجنب الإيحاء بالتأييد.

راجع [API العامة](/clawhub/api) و[HTTP API](/ar/clawhub/http-api).
