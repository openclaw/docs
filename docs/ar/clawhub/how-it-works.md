# كيف يعمل ClawHub

ClawHub هو طبقة السجل لـ Skills وPlugins في OpenClaw. يمنح المستخدمين
مكانًا لاكتشاف الحزم، ويمنح الناشرين مكانًا لإصدار النسخ، ويمنح
OpenClaw ما يكفي من البيانات الوصفية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يتضمن:

- مالكًا واسمًا مختصرًا أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية وملخصًا وملفات وإسنادًا للمصدر
- سجل تغييرات ومعلومات وسوم مثل `latest`
- إشارات التنزيل والتثبيت والنجوم والتعليقات
- حالة الفحص الأمني والإشراف

صفحة الإدراج هي المكان المعتمد الذي يفحص فيه المستخدمون ما تدّعي Skills أو
Plugins القيام به قبل تثبيتها.

## Skills

Skills هي حزمة نصية ذات إصدارات تتمحور حول `SKILL.md`. ويمكن أن تتضمن
ملفات داعمة وأمثلة وقوالب وسكربتات.

يقرأ ClawHub الواجهة الأمامية في `SKILL.md` لفهم اسم Skills
ووصفها ومتطلباتها ومتغيرات البيئة والبيانات الوصفية. تهم البيانات
الوصفية الدقيقة لأنها تساعد المستخدمين على تحديد ما إذا كانوا سيثبتون Skills
وتساعد عمليات الفحص الآلية على اكتشاف عدم التطابق بين السلوك المصرح به
والسلوك المرصود.

راجع [تنسيق Skills](/ar/clawhub/skill-format).

## Plugins

Plugins هي امتدادات OpenClaw مجمعة في حزم. يخزن ClawHub بيانات الحزمة
الوصفية ومعلومات التوافق وروابط المصدر والآثار وسجلات الإصدارات.

عندما يثبت OpenClaw Plugin من ClawHub، فإنه يتحقق من بيانات التوافق
الوصفية المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحزم توافق API،
والحد الأدنى لإصدار Gateway، وأهداف المضيف، ومتطلبات البيئة، وبصمات
الآثار.

استخدم مصدر تثبيت ClawHub صريحًا عندما تريد أن يكون السجل هو
مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل إصدار جديدًا غير قابل للتغيير. يستخدم الناشرون CLI باسم `clawhub`
لتدفقات عمل السجل الموثقة:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم التشغيل الجاف لمعاينة الحمولة النهائية قبل الرفع. ثم تعرض الصفحات العامة
البيانات الوصفية المنشورة والملفات وإسناد المصدر وحالة الفحص.

## التثبيتات والتحديثات

تستخدم أوامر تثبيت OpenClaw ClawHub كمصدر للحزم:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

يسجل OpenClaw بيانات مصدر التثبيت الوصفية حتى تتمكن التحديثات من حل
حزمة السجل نفسها لاحقًا. يدعم CLI الخاص بـ ClawHub أيضًا تدفقات عمل
مباشرة لتثبيت Skills وتحديثها للمستخدمين الذين يريدون مجلدات Skills
مدارة عبر السجل خارج مساحة عمل OpenClaw كاملة.

## حالة الأمان

ClawHub مفتوح للنشر، لكن الإصدارات تظل خاضعة لبوابات الرفع
والفحوصات الآلية وبلاغات المستخدمين وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. قد يختفي المحتوى المحتجز
أو المخفي أو المحظور من البحث العام وتدفقات التثبيت، مع بقائه مرئيًا
للمالك لأغراض التشخيص.

راجع [الأمان + الإشراف](/ar/clawhub/security) و
[الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## الوصول إلى API

يوفر ClawHub واجهات API عامة للقراءة من أجل الاكتشاف والبحث وتفاصيل الحزم
والتنزيلات. يمكن للفهارس الخارجية استخدام واجهات API هذه عندما تربط عائدًا
إلى إدراج ClawHub المعتمد، وتحترم حدود المعدل، وتتجنب الإيحاء بالتأييد.

راجع [API العامة](/ar/clawhub/api) و[HTTP API](/ar/clawhub/http-api).
