# كيف يعمل ClawHub

ClawHub هو طبقة السجل لمهارات OpenClaw وplugins الخاصة به. يوفّر للمستخدمين
مكانًا لاكتشاف الحزم، وللناشرين مكانًا لإصدار النسخ، ويمنح OpenClaw ما يكفي من
البيانات الوصفية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يتضمن:

- مالكًا وslug أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية وملخصًا وملفات ونسبة المصدر
- سجل تغييرات ومعلومات وسوم مثل `latest`
- إشارات التنزيل والتثبيت والنجوم والتعليقات
- حالة الفحص الأمني والإشراف

صفحة الإدراج هي المكان المرجعي الذي يمكن للمستخدمين فيه فحص ما تدّعي مهارة أو
plugin أنها تفعله قبل تثبيتها.

## Skills

المهارة هي حزمة نصية ذات إصدارات تتمحور حول `SKILL.md`. ويمكن أن تتضمن
ملفات داعمة وأمثلة وقوالب ونصوصًا برمجية.

يقرأ ClawHub frontmatter الخاص بملف `SKILL.md` لفهم اسم المهارة
ووصفها ومتطلباتها ومتغيرات البيئة والبيانات الوصفية. تهم البيانات الوصفية الدقيقة
لأنها تساعد المستخدمين على تحديد ما إذا كانوا سيثبتون المهارة، وتساعد الفحوصات
الآلية على اكتشاف أوجه عدم التطابق بين السلوك المعلن والسلوك المرصود.

راجع [تنسيق المهارة](/ar/clawhub/skill-format).

## Plugins

Plugins هي إضافات OpenClaw معبأة. يخزن ClawHub بيانات الحزمة الوصفية
ومعلومات التوافق وروابط المصدر والقطع الأثرية وسجلات النسخ.

عندما يثبت OpenClaw plugin من ClawHub، فإنه يتحقق من بيانات التوافق الوصفية
المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحزم توافق API، والحد الأدنى من إصدار
Gateway، وأهداف المضيف، ومتطلبات البيئة، وبصمات القطع الأثرية.

استخدم مصدر تثبيت ClawHub صريحًا عندما تريد أن يكون السجل هو
مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل نسخة جديدًا غير قابل للتغيير. يستخدم الناشرون CLI الخاص بـ`clawhub`
لسير عمل السجل الموثق:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم التشغيلات التجريبية لمعاينة الحمولة المحلولة قبل الرفع. بعد ذلك تعرض
الصفحات العامة البيانات الوصفية المنشورة والملفات ونسبة المصدر وحالة الفحص.

## التثبيتات والتحديثات

تستخدم أوامر تثبيت OpenClaw ClawHub كمصدر للحزم:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

يسجل OpenClaw بيانات وصفية لمصدر التثبيت كي تتمكن التحديثات لاحقًا من حل
حزمة السجل نفسها. يدعم CLI الخاص بـClawHub أيضًا سير عمل تثبيت المهارات
وتحديثها مباشرةً للمستخدمين الذين يريدون مجلدات مهارات مُدارة بالسجل خارج
مساحة عمل OpenClaw كاملة.

## الحالة الأمنية

ClawHub مفتوح للنشر، لكن الإصدارات تظل خاضعة لبوابات الرفع
والفحوصات الآلية وتقارير المستخدمين وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. قد يختفي المحتوى المحتجز أو
المخفي أو المحظور من البحث العام ومسارات التثبيت مع بقائه مرئيًا للمالك لأغراض
التشخيص أو الاستئناف.

راجع [الأمان + الإشراف](/ar/clawhub/security) و
[الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## الوصول إلى API

يوفر ClawHub واجهات API عامة للقراءة من أجل الاكتشاف والبحث وتفاصيل الحزم
والتنزيلات. يمكن للفهارس الخارجية استخدام واجهات API هذه عندما تربط إلى إدراج
ClawHub المرجعي، وتحترم حدود المعدل، وتتجنب الإيحاء بالمصادقة أو التأييد.

راجع [API العامة](/ar/clawhub/api) و[HTTP API](/ar/clawhub/http-api).
