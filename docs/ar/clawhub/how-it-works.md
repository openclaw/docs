# كيف يعمل ClawHub

يمثل ClawHub طبقة السجل الخاصة بـ Skills وPlugins في OpenClaw. فهو يتيح للمستخدمين
مكانًا لاكتشاف الحزم، ويتيح للناشرين مكانًا لإصدار النسخ، ويوفر لـ OpenClaw ما يكفي
من البيانات الوصفية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يتضمن:

- مالكًا ومعرّفًا مختصرًا أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية وملخصًا وملفات وإسنادًا إلى المصدر
- معلومات سجل التغييرات والوسوم، مثل `latest`
- مؤشرات التنزيل والتثبيت والنجوم
- حالة الفحص الأمني والإشراف

صفحة الإدراج هي المكان المرجعي الذي يمكن للمستخدمين فيه التحقق مما تزعم Skill أو
Plugin أنها تفعله قبل تثبيتها.

## Skills

Skill هي حزمة نصية ذات نسخ تتمحور حول `SKILL.md`. ويمكن أن تتضمن
ملفات داعمة وأمثلة وقوالب ونصوصًا برمجية.

يقرأ ClawHub بيانات الواجهة الأمامية في `SKILL.md` لفهم اسم Skill
ووصفها ومتطلباتها ومتغيرات البيئة وبياناتها الوصفية. وتُعد دقة
البيانات الوصفية مهمة لأنها تساعد المستخدمين على تحديد ما إذا كانوا يريدون تثبيت Skill،
كما تساعد عمليات الفحص الآلية على اكتشاف حالات عدم التطابق بين السلوك المعلن والمرصود.

راجع [تنسيق Skill](/clawhub/skill-format).

## Plugins

Plugins هي امتدادات OpenClaw مُحزّمة. يخزن ClawHub البيانات الوصفية للحزم
ومعلومات التوافق وروابط المصدر والعناصر المنشأة وسجلات النسخ.

عندما يثبّت OpenClaw إحدى Plugins من ClawHub، فإنه يتحقق من بيانات التوافق الوصفية
المعلنة قبل التثبيت. ويمكن أن تتضمن سجلات الحزم توافق API،
والحد الأدنى لإصدار Gateway، والأنظمة المضيفة المستهدفة، ومتطلبات البيئة، وبصمات
العناصر المنشأة.

استخدم مصدر تثبيت صريحًا من ClawHub عندما تريد أن يكون السجل هو
مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل نسخة جديدًا غير قابل للتغيير. يستخدم الناشرون CLI الخاص بـ `clawhub`
لسير عمل السجل الذي يتطلب المصادقة:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم عمليات التشغيل التجريبي لمعاينة الحمولة المحسومة قبل الرفع. بعد ذلك تعرض
الصفحات العامة البيانات الوصفية المنشورة والملفات وإسناد المصدر وحالة الفحص.

## عمليات التثبيت والتحديث

تستخدم أوامر تثبيت OpenClaw منصة ClawHub مصدرًا للحزم:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

يسجل OpenClaw البيانات الوصفية لمصدر التثبيت حتى تتمكن التحديثات لاحقًا من الوصول إلى
حزمة السجل نفسها. ويدعم CLI الخاص بـ ClawHub أيضًا عمليات تثبيت Skills وتحديثها
مباشرةً للمستخدمين الذين يريدون مجلدات Skills مُدارة بواسطة السجل خارج
مساحة عمل OpenClaw كاملة.

## الحالة الأمنية

يتيح ClawHub النشر للجميع، لكن الإصدارات تظل خاضعة لبوابات الرفع
والفحوصات الآلية وبلاغات المستخدمين وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. وقد يختفي المحتوى المحتجز أو المخفي
أو المحظور من البحث العام ومسارات التثبيت، مع بقائه
مرئيًا للمالك لأغراض التشخيص.

راجع [الأمان](/ar/clawhub/security)، و[عمليات التدقيق الأمني](/clawhub/security-audits)،
و[الإشراف وسلامة الحساب](/ar/clawhub/moderation)، و
[الاستخدام المقبول](/clawhub/acceptable-usage).

## الوصول إلى API

يوفر ClawHub واجهات API عامة للقراءة من أجل الاكتشاف والبحث وتفاصيل الحزم
والتنزيلات. ويمكن لكتالوجات الجهات الخارجية استخدام واجهات API هذه عندما تضع رابطًا
يعود إلى إدراج ClawHub المرجعي، وتحترم حدود المعدل، وتتجنب الإيحاء بالتأييد.

راجع [API العامة](/clawhub/api) و[HTTP API](/clawhub/http-api).
