# كيف يعمل ClawHub

ClawHub هو طبقة السجل لـ Skills وPlugins في OpenClaw. فهو يمنح المستخدمين
مكانًا لاكتشاف الحزم، ويمنح الناشرين مكانًا لإصدار النسخ، ويمنح
OpenClaw بيانات وصفية كافية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يتضمن:

- مالكًا ومعرّفًا مختصرًا أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية وملخصًا وملفات ونسبة للمصدر
- سجل تغييرات ومعلومات وسوم مثل `latest`
- مؤشرات التنزيل والتثبيت والنجوم
- حالة فحص الأمان والإشراف

صفحة الإدراج هي المكان المرجعي الذي يفحص فيه المستخدمون ما يدّعي Skill أو
Plugin أنه يفعله قبل تثبيته.

## Skills

Skill هي حزمة نصية ذات نسخ تتمحور حول `SKILL.md`. يمكن أن تتضمن
ملفات داعمة وأمثلة وقوالب وسكربتات.

يقرأ ClawHub بيانات الواجهة الأمامية في `SKILL.md` لفهم اسم Skill
ووصفها ومتطلباتها ومتغيرات البيئة والبيانات الوصفية. دقة البيانات الوصفية
مهمة لأنها تساعد المستخدمين على تحديد ما إذا كانوا سيثبّتون Skill وتساعد
الفحوصات الآلية على اكتشاف أوجه عدم التطابق بين السلوك المعلن والمرصود.

راجع [تنسيق Skill](/ar/clawhub/skill-format).

## Plugins

Plugins هي امتدادات OpenClaw مُحزّمة. يخزن ClawHub بيانات الحزمة الوصفية
ومعلومات التوافق وروابط المصدر والقطع الأثرية وسجلات النسخ.

عندما يثبّت OpenClaw Plugin من ClawHub، فإنه يتحقق من بيانات التوافق
الوصفية المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحزم توافق API،
والحد الأدنى من إصدار Gateway، وأهداف المضيف، ومتطلبات البيئة، وبصمات
القطع الأثرية.

استخدم مصدر تثبيت ClawHub صريحًا عندما تريد أن يكون السجل هو مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل نسخة جديدًا غير قابل للتغيير. يستخدم الناشرون CLI `clawhub`
لسير عمل السجل الموثّق:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم التشغيلات التجريبية لمعاينة الحمولة المحلولة قبل الرفع. ثم تعرض
الصفحات العامة البيانات الوصفية المنشورة والملفات ونسبة المصدر وحالة الفحص.

## التثبيتات والتحديثات

تستخدم أوامر تثبيت OpenClaw ClawHub كمصدر حزم:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

يسجل OpenClaw بيانات وصفية لمصدر التثبيت حتى تتمكن التحديثات من حل حزمة
السجل نفسها لاحقًا. يدعم CLI الخاص بـ ClawHub أيضًا سير عمل تثبيت Skill
وتحديثها مباشرة للمستخدمين الذين يريدون مجلدات Skills مُدارة بالسجل خارج
مساحة عمل OpenClaw كاملة.

## حالة الأمان

ClawHub مفتوح للنشر، لكن الإصدارات تظل خاضعة لبوابات الرفع،
والفحوصات الآلية، وبلاغات المستخدمين، وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. قد يختفي المحتوى المحتجز أو
المخفي أو المحظور من البحث العام وتدفقات التثبيت، مع بقائه مرئيًا للمالك
لأغراض التشخيص.

راجع [الأمان](/clawhub/security)، و[تدقيقات الأمان](/clawhub/security-audits)،
و[الإشراف وسلامة الحساب](/ar/clawhub/moderation)، و
[الاستخدام المقبول](/clawhub/acceptable-usage).

## الوصول إلى API

يوفر ClawHub واجهات API عامة للقراءة لأغراض الاكتشاف والبحث وتفاصيل الحزم
والتنزيلات. يمكن للفهرسات التابعة لجهات خارجية استخدام واجهات API هذه عندما
تضع رابطًا إلى إدراج ClawHub المرجعي، وتحترم حدود المعدل، وتتجنب الإيحاء
بوجود تأييد.

راجع [API العامة](/ar/clawhub/api) و[HTTP API](/clawhub/http-api).
