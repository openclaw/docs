# كيفية عمل ClawHub

ClawHub هو طبقة السجل لـ Skills وPlugins في OpenClaw. يمنح المستخدمين
مكانًا لاكتشاف الحزم، ويمنح الناشرين مكانًا لإصدار النسخ، ويمنح
OpenClaw بيانات وصفية كافية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يتضمن:

- مالكًا واسمًا مختصرًا أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية، وملخصًا، وملفات، ونَسبًا للمصدر
- سجل تغييرات ومعلومات وسوم مثل `latest`
- إشارات التنزيل والتثبيت والنجوم
- حالة الفحص الأمني والإشراف

صفحة الإدراج هي المكان المعتمد الذي يتيح للمستخدمين فحص ما تدّعي Skill أو
Plugin أنها تفعله قبل تثبيتها.

## Skills

Skill هي حزمة نصية ذات نسخ تتمحور حول `SKILL.md`. يمكن أن تتضمن
ملفات داعمة، وأمثلة، وقوالب، وسكربتات.

يقرأ ClawHub الواجهة الأمامية في `SKILL.md` لفهم اسم Skill،
ووصفها، ومتطلباتها، ومتغيرات البيئة، وبياناتها الوصفية. دقة
البيانات الوصفية مهمة لأنها تساعد المستخدمين على تحديد ما إذا كانوا سيثبتون Skill، كما
تساعد عمليات الفحص الآلية على اكتشاف حالات عدم التطابق بين السلوك المعلن والسلوك المرصود.

راجع [تنسيق Skill](/ar/clawhub/skill-format).

## Plugins

Plugins هي امتدادات OpenClaw مُحزّمة. يخزن ClawHub بيانات الحزمة الوصفية،
ومعلومات التوافق، وروابط المصدر، والقطع الأثرية، وسجلات النسخ.

عندما يثبّت OpenClaw Plugin من ClawHub، فإنه يتحقق من بيانات التوافق
الوصفية المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحزم توافق API،
والحد الأدنى من نسخة Gateway، وأهداف المضيف، ومتطلبات البيئة، وبصمات
القطع الأثرية.

استخدم مصدر تثبيت ClawHub صريحًا عندما تريد أن يكون السجل هو
مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل نسخة جديدًا غير قابل للتغيير. يستخدم الناشرون CLI `clawhub`
لسير عمل السجل المصادق عليه:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم عمليات التشغيل الجاف لمعاينة الحمولة المحلولة قبل الرفع. ثم
تعرض الصفحات العامة البيانات الوصفية المنشورة، والملفات، ونَسب المصدر، وحالة الفحص.

## التثبيتات والتحديثات

تستخدم أوامر تثبيت OpenClaw ClawHub كمصدر للحزم:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

يسجل OpenClaw البيانات الوصفية لمصدر التثبيت حتى تتمكن التحديثات من حل حزمة
السجل نفسها لاحقًا. يدعم CLI الخاص بـ ClawHub أيضًا سير عمل تثبيت Skills
وتحديثها مباشرةً للمستخدمين الذين يريدون مجلدات Skills مُدارة عبر السجل خارج
مساحة عمل OpenClaw كاملة.

## الحالة الأمنية

ClawHub مفتوح للنشر، لكن الإصدارات لا تزال خاضعة لبوابات الرفع،
والفحوصات الآلية، وبلاغات المستخدمين، وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. قد يختفي المحتوى المحتجز أو المخفي
أو المحظور من البحث العام وتدفقات التثبيت مع بقائه
مرئيًا للمالك لأغراض التشخيص.

راجع [الأمان](/clawhub/security)، و[تدقيقات الأمان](/clawhub/security-audits)،
و[الإشراف وسلامة الحساب](/ar/clawhub/moderation)، و
[الاستخدام المقبول](/clawhub/acceptable-usage).

## الوصول إلى API

يوفر ClawHub واجهات API قراءة عامة للاكتشاف والبحث وتفاصيل الحزم
والتنزيلات. يمكن للكتالوجات التابعة لجهات خارجية استخدام واجهات API هذه عندما تربط عائدًا إلى
إدراج ClawHub المعتمد، وتحترم حدود المعدل، وتتجنب الإيحاء بالتأييد.

راجع [API العامة](/ar/clawhub/api) و[HTTP API](/clawhub/http-api).
