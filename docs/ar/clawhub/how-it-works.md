# آلية عمل ClawHub

يمثّل ClawHub طبقة السجل لـ Skills وPlugins في OpenClaw. فهو يوفّر للمستخدمين
مكانًا لاكتشاف الحزم، وللناشرين مكانًا لإصدار الإصدارات، كما يزوّد
OpenClaw ببيانات وصفية كافية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يتضمن:

- مالكًا ومُعرّفًا مختصرًا أو اسم حزمة
- إصدارًا منشورًا واحدًا أو أكثر
- بيانات وصفية وملخصًا وملفات ونسبة المصدر
- سجل تغييرات ومعلومات وسوم مثل `latest`
- إشارات التنزيل والتثبيت والنجوم
- حالة الفحص الأمني والإشراف

صفحة الإدراج هي المكان المعتمد الذي يمكن للمستخدمين فيه التحقق مما تدّعي
Skill أو Plugin أنها تفعله قبل تثبيتها.

## Skills

Skill هي حزمة نصية ذات إصدارات تتمحور حول `SKILL.md`. ويمكن أن تتضمن
ملفات داعمة وأمثلة وقوالب وبرامج نصية.

يقرأ ClawHub البيانات التمهيدية في `SKILL.md` لفهم اسم Skill
ووصفها ومتطلباتها ومتغيرات البيئة وبياناتها الوصفية. وتكتسب دقة
البيانات الوصفية أهميتها لأنها تساعد المستخدمين على تحديد ما إذا كانوا سيثبّتون Skill،
وتساعد عمليات الفحص الآلية على اكتشاف حالات عدم التطابق بين السلوك المعلَن والمرصود.

راجع [تنسيق Skill](/ar/clawhub/skill-format).

## Plugins

Plugins هي امتدادات OpenClaw مُجمّعة في حزم. يخزّن ClawHub البيانات الوصفية للحزم
ومعلومات التوافق وروابط المصدر والعناصر المصاحبة وسجلات الإصدارات.

عندما يثبّت OpenClaw أحد Plugins من ClawHub، فإنه يتحقق من بيانات التوافق الوصفية
المعلَن عنها قبل التثبيت. ويمكن أن تتضمن سجلات الحزم توافق API،
والحد الأدنى لإصدار Gateway، والمضيفين المستهدفين، ومتطلبات البيئة، وبصمات
العناصر المصاحبة.

استخدم مصدر تثبيت صريحًا من ClawHub عندما تريد أن يكون السجل هو
مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل إصدار جديدًا غير قابل للتغيير. يستخدم الناشرون CLI ‏`clawhub`
لسير عمل السجل المُصادَق عليه:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم عمليات التشغيل التجريبي لمعاينة الحمولة المحسومة قبل الرفع. ثم تعرض
الصفحات العامة البيانات الوصفية المنشورة والملفات ونسبة المصدر وحالة الفحص.

## عمليات التثبيت والتحديث

تستخدم أوامر تثبيت OpenClaw منصة ClawHub مصدرًا للحزم:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

يسجّل OpenClaw البيانات الوصفية لمصدر التثبيت حتى تتمكن التحديثات لاحقًا من العثور على
حزمة السجل نفسها. ويدعم CLI الخاص بـ ClawHub أيضًا عمليات تثبيت Skill وتحديثها
مباشرةً للمستخدمين الذين يريدون مجلدات Skills يديرها السجل خارج
مساحة عمل OpenClaw كاملة.

## الحالة الأمنية

يتيح ClawHub النشر المفتوح، لكن الإصدارات تظل خاضعة لبوابات الرفع
والفحوص الآلية وبلاغات المستخدمين وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. وقد يختفي المحتوى المحتجز أو المخفي
أو المحظور من البحث العام ومسارات التثبيت، مع بقائه
ظاهرًا للمالك لأغراض التشخيص.

راجع [الأمان](/clawhub/security)، و[عمليات التدقيق الأمني](/clawhub/security-audits)،
و[الإشراف وسلامة الحساب](/ar/clawhub/moderation)، و
[الاستخدام المقبول](/clawhub/acceptable-usage).

## الوصول إلى API

يوفّر ClawHub واجهات API عامة للقراءة من أجل الاكتشاف والبحث وتفاصيل الحزم
والتنزيلات. ويمكن للكتالوجات التابعة لجهات خارجية استخدام واجهات API هذه عندما تربط
بالإدراج المعتمد في ClawHub، وتحترم حدود المعدل، وتتجنب الإيحاء بالتأييد.

راجع [واجهة API العامة](/clawhub/api) و[واجهة HTTP API](/clawhub/http-api).
