# كيف يعمل ClawHub

ClawHub هو طبقة السجل الخاصة بمهارات OpenClaw وPlugins. فهو يمنح المستخدمين
مكانًا لاكتشاف الحزم، ويمنح الناشرين مكانًا لإصدار النسخ، ويمنح
OpenClaw بيانات وصفية كافية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يحتوي على:

- مالك ومعرّف أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية، وملخص، وملفات، ونَسب للمصدر
- سجل تغييرات ومعلومات وسوم مثل `latest`
- إشارات التنزيل، والتثبيت، والنجوم، والتعليقات
- حالة الفحص الأمني والإشراف

صفحة الإدراج هي المكان المعياري الذي يراجع فيه المستخدمون ما يدّعي skill أو
plugin أنه يفعله قبل تثبيته.

## Skills

Skill هي حزمة نصية ذات نسخ تتمحور حول `SKILL.md`. ويمكن أن تتضمن
ملفات داعمة، وأمثلة، وقوالب، وسكربتات.

يقرأ ClawHub frontmatter في `SKILL.md` لفهم اسم skill
ووصفها، ومتطلباتها، ومتغيرات البيئة، وبياناتها الوصفية. تهم البيانات
الوصفية الدقيقة لأنها تساعد المستخدمين على تحديد ما إذا كانوا يريدون تثبيت skill
وتساعد الفحوصات المؤتمتة على اكتشاف أوجه عدم التطابق بين السلوك المعلن والسلوك المرصود.

راجع [تنسيق Skill](/ar/clawhub/skill-format).

## Plugins

Plugins هي امتدادات OpenClaw معبأة. يخزن ClawHub بيانات الحزمة الوصفية،
ومعلومات التوافق، وروابط المصدر، والآثار، وسجلات النسخ.

عندما يثبت OpenClaw plugin من ClawHub، فإنه يتحقق من بيانات التوافق
الوصفية المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحزم توافق API،
والحد الأدنى لإصدار Gateway، وأهداف المضيف، ومتطلبات البيئة، وملخصات
الآثار.

استخدم مصدر تثبيت ClawHub صريحًا عندما تريد أن يكون السجل هو
مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل نسخة جديدًا غير قابل للتغيير. يستخدم الناشرون CLI الخاص بـ `clawhub`
لسير عمل السجل المصادق عليه:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم التشغيلات التجريبية لمعاينة الحمولة المحلولة قبل الرفع. ثم تعرض الصفحات العامة
البيانات الوصفية المنشورة، والملفات، ونَسب المصدر، وحالة الفحص.

## التثبيتات والتحديثات

تستخدم أوامر تثبيت OpenClaw ClawHub كمصدر للحزم:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

يسجل OpenClaw بيانات وصفية لمصدر التثبيت حتى تتمكن التحديثات من حل حزمة
السجل نفسها لاحقًا. يدعم ClawHub CLI أيضًا مسارات عمل تثبيت وتحديث
Skills المباشرة للمستخدمين الذين يريدون مجلدات Skills مدارة بالسجل خارج
مساحة عمل OpenClaw كاملة.

## الحالة الأمنية

ClawHub مفتوح للنشر، لكن الإصدارات تظل خاضعة لبوابات الرفع،
والفحوصات المؤتمتة، وبلاغات المستخدمين، وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. قد يختفي المحتوى المحتجز أو المخفي
أو المحظور من البحث العام ومسارات التثبيت مع بقائه مرئيًا للمالك لأغراض التشخيص.

راجع [الأمان + الإشراف](/ar/clawhub/security) و
[الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## الوصول إلى API

يوفر ClawHub واجهات API قراءة عامة للاكتشاف، والبحث، وتفاصيل الحزم،
والتنزيلات. يجوز للفهارس التابعة لأطراف خارجية استخدام واجهات API هذه عندما تربط
مرة أخرى بإدراج ClawHub المعياري، وتحترم حدود المعدل، وتتجنب الإيحاء بالتأييد.

راجع [API العامة](/ar/clawhub/api) و[HTTP API](/ar/clawhub/http-api).
