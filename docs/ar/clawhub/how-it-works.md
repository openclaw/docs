# كيف يعمل ClawHub

ClawHub هو طبقة السجل لمهارات وإضافات OpenClaw. يمنح المستخدمين
مكانًا لاكتشاف الحزم، ويمنح الناشرين مكانًا لإصدار النسخ، ويمنح
OpenClaw ما يكفي من البيانات الوصفية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يتضمن:

- مالكًا ومعرّفًا مختصرًا أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية وملخصًا وملفات ونسبة إلى المصدر
- سجل تغييرات ومعلومات وسوم مثل `latest`
- إشارات التنزيل والتثبيت والنجوم والتعليقات
- حالة الفحص الأمني والإشراف

صفحة الإدراج هي المكان المرجعي للمستخدمين لفحص ما تزعم المهارة أو
الإضافة أنها تفعله قبل تثبيتها.

## Skills

المهارة هي حزمة نصية ذات إصدارات تتمحور حول `SKILL.md`. ويمكن أن تتضمن
ملفات داعمة وأمثلة وقوالب وسكربتات.

يقرأ ClawHub واجهة `SKILL.md` الأمامية لفهم اسم المهارة
ووصفها ومتطلباتها ومتغيرات البيئة والبيانات الوصفية الخاصة بها. دقة
البيانات الوصفية مهمة لأنها تساعد المستخدمين على تقرير ما إذا كانوا سيثبتون
المهارة، وتساعد الفحوصات الآلية على اكتشاف حالات عدم التطابق بين السلوك
المعلن والسلوك المرصود.

راجع [تنسيق المهارة](/ar/clawhub/skill-format).

## الإضافات

الإضافات هي امتدادات OpenClaw معبأة. يخزن ClawHub بيانات الحزمة الوصفية
ومعلومات التوافق وروابط المصدر والقطع الأثرية وسجلات الإصدارات.

عندما يثبت OpenClaw إضافة من ClawHub، فإنه يتحقق من بيانات التوافق
المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحزم توافق API،
والحد الأدنى لإصدار Gateway، وأهداف المضيف، ومتطلبات البيئة، وملخصات
القطع الأثرية.

استخدم مصدر تثبيت ClawHub صريحًا عندما تريد أن يكون السجل هو
مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل إصدار جديدًا غير قابل للتغيير. يستخدم الناشرون CLI الخاص بـ `clawhub`
لسير عمل السجل المصادق عليه:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم التشغيلات التجريبية لمعاينة الحمولة المحلولة قبل الرفع. ثم تعرض
الصفحات العامة البيانات الوصفية المنشورة والملفات ونسبة المصدر وحالة الفحص.

## التثبيتات والتحديثات

تستخدم أوامر تثبيت OpenClaw ClawHub كمصدر حزم:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

يسجل OpenClaw بيانات مصدر التثبيت الوصفية حتى تتمكن التحديثات لاحقًا من حل
حزمة السجل نفسها. يدعم CLI الخاص بـ ClawHub أيضًا سير عمل تثبيت المهارات
وتحديثها مباشرة للمستخدمين الذين يريدون مجلدات مهارات يديرها السجل خارج
مساحة عمل OpenClaw كاملة.

## حالة الأمان

ClawHub مفتوح للنشر، لكن الإصدارات لا تزال خاضعة لبوابات الرفع
والفحوصات الآلية وبلاغات المستخدمين وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. قد يختفي المحتوى المعلّق أو
المخفي أو المحظور من البحث العام ومسارات التثبيت، مع بقائه مرئيًا للمالك
لأغراض التشخيص أو الاستئناف.

راجع [الأمان + الإشراف](/ar/clawhub/security) و
[الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## الوصول إلى API

يوفر ClawHub واجهات API عامة للقراءة من أجل الاكتشاف والبحث وتفاصيل الحزم
والتنزيلات. يمكن للكتالوجات الخارجية استخدام واجهات API هذه عندما تربط
بالإدراج المرجعي في ClawHub، وتحترم حدود المعدل، وتتجنب الإيحاء بالمصادقة.

راجع [API العامة](/ar/clawhub/api) و [HTTP API](/ar/clawhub/http-api).
