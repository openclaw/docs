# كيف يعمل ClawHub

ClawHub هو طبقة السجل لـ Skills وPlugins في OpenClaw. يمنح المستخدمين
مكانًا لاكتشاف الحزم، ويمنح الناشرين مكانًا لإصدار النسخ، ويمنح
OpenClaw بيانات وصفية كافية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يتضمن:

- مالكًا واسمًا مختصرًا أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية، وملخصًا، وملفات، ونسبة إلى المصدر
- سجل تغييرات ومعلومات وسوم مثل `latest`
- إشارات التنزيل والتثبيت والنجوم
- حالة فحص الأمان والإشراف

صفحة الإدراج هي المكان المرجعي الذي يفحص فيه المستخدمون ما يدعي Skill أو
Plugin القيام به قبل تثبيته.

## Skills

Skill هو حزمة نصية ذات نسخ تتمحور حول `SKILL.md`. ويمكن أن يتضمن
ملفات داعمة، وأمثلة، وقوالب، ونصوصًا برمجية.

يقرأ ClawHub بيانات الواجهة الأمامية في `SKILL.md` لفهم اسم Skill
ووصفه ومتطلباته ومتغيرات البيئة والبيانات الوصفية. دقة البيانات الوصفية
مهمة لأنها تساعد المستخدمين على تحديد ما إذا كانوا سيثبتون Skill وتساعد
عمليات الفحص الآلية على اكتشاف حالات عدم التطابق بين السلوك المعلن
والسلوك المرصود.

راجع [تنسيق Skill](/ar/clawhub/skill-format).

## Plugins

Plugins هي امتدادات OpenClaw معبأة. يخزن ClawHub بيانات الحزمة الوصفية،
ومعلومات التوافق، وروابط المصدر، والملفات الناتجة، وسجلات النسخ.

عندما يثبت OpenClaw Plugin من ClawHub، فإنه يتحقق من بيانات التوافق
الوصفية المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحزم توافق API،
والحد الأدنى لإصدار Gateway، وأهداف المضيف، ومتطلبات البيئة، وملخصات
الملفات الناتجة.

استخدم مصدر تثبيت ClawHub صريحًا عندما تريد أن يكون السجل هو مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل نسخة جديدًا غير قابل للتغيير. يستخدم الناشرون CLI الخاص بـ `clawhub`
لسير عمل السجل المصادق عليه:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم التشغيل التجريبي لمعاينة الحمولة المحلولة قبل الرفع. بعد ذلك تعرض
الصفحات العامة البيانات الوصفية المنشورة، والملفات، ونسبة المصدر،
وحالة الفحص.

## التثبيتات والتحديثات

تستخدم أوامر تثبيت OpenClaw ClawHub كمصدر للحزم:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

يسجل OpenClaw بيانات مصدر التثبيت الوصفية حتى تتمكن التحديثات من حل
حزمة السجل نفسها لاحقًا. يدعم CLI الخاص بـ ClawHub أيضًا مسارات عمل
تثبيت وتحديث Skill مباشرة للمستخدمين الذين يريدون مجلدات Skills يديرها
السجل خارج مساحة عمل OpenClaw كاملة.

## حالة الأمان

ClawHub مفتوح للنشر، لكن الإصدارات ما زالت خاضعة لبوابات الرفع،
والفحوصات الآلية، وبلاغات المستخدمين، وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. وقد يختفي المحتوى المحتجز
أو المخفي أو المحظور من البحث العام ومسارات التثبيت مع بقائه مرئيًا
للمالك لأغراض التشخيص.

راجع [الأمان](/ar/clawhub/security)، و[تدقيقات الأمان](/ar/clawhub/security-audits)،
و[الإشراف وسلامة الحساب](/ar/clawhub/moderation)، و
[الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## الوصول إلى API

يوفر ClawHub واجهات API عامة للقراءة من أجل الاكتشاف، والبحث،
وتفاصيل الحزم، والتنزيلات. يمكن للفهارس الخارجية استخدام واجهات API هذه
عندما تربط إلى إدراج ClawHub المرجعي، وتلتزم بحدود المعدل، وتتجنب
الإيحاء بالتأييد.

راجع [API العام](/ar/clawhub/api) و[HTTP API](/ar/clawhub/http-api).
