# كيفية عمل ClawHub

ClawHub هو طبقة السجل لـ Skills وPlugin في OpenClaw. يمنح المستخدمين مكانًا لاكتشاف الحزم، ويمنح الناشرين مكانًا لإصدار النسخ، ويمنح OpenClaw بيانات وصفية كافية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يحتوي على:

- مالك ومعرّف قصير أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية وملخص وملفات وإسناد المصدر
- سجل تغييرات ومعلومات وسوم مثل `latest`
- إشارات التنزيل والتثبيت والنجوم
- حالة الفحص الأمني والإشراف

صفحة الإدراج هي المكان المرجعي الذي يطّلع فيه المستخدمون على ما تدّعي مهارة أو Plugin أنه يفعله قبل تثبيته.

## Skills

المهارة حزمة نصية ذات نسخ تتمحور حول `SKILL.md`. ويمكن أن تتضمن ملفات داعمة وأمثلة وقوالب وسكربتات.

يقرأ ClawHub بيانات الواجهة في `SKILL.md` لفهم اسم المهارة ووصفها ومتطلباتها ومتغيرات البيئة والبيانات الوصفية. تهم دقة البيانات الوصفية لأنها تساعد المستخدمين على تحديد ما إذا كانوا يريدون تثبيت المهارة، وتساعد الفحوصات الآلية على اكتشاف أوجه عدم التطابق بين السلوك المعلن والسلوك المرصود.

راجع [تنسيق المهارة](/ar/clawhub/skill-format).

## Plugin

Plugin هي امتدادات OpenClaw معبأة. يخزن ClawHub بيانات الحزمة الوصفية ومعلومات التوافق وروابط المصدر والقطع الأثرية وسجلات النسخ.

عندما يثبت OpenClaw ‏Plugin من ClawHub، فإنه يتحقق من بيانات التوافق الوصفية المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحزم توافق API، والحد الأدنى من إصدار Gateway، وأهداف المضيف، ومتطلبات البيئة، وبصمات القطع الأثرية.

استخدم مصدر تثبيت ClawHub صريحًا عندما تريد أن يكون السجل هو مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل نسخة جديدًا غير قابل للتغيير. يستخدم الناشرون CLI الخاص بـ `clawhub` لسير عمل السجل المصادق عليه:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم التشغيل التجريبي لمعاينة الحمولة المحلولة قبل الرفع. بعد ذلك تعرض الصفحات العامة البيانات الوصفية المنشورة والملفات وإسناد المصدر وحالة الفحص.

## عمليات التثبيت والتحديث

تستخدم أوامر تثبيت OpenClaw ‏ClawHub كمصدر حزم:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

يسجل OpenClaw بيانات مصدر التثبيت الوصفية حتى تتمكن التحديثات لاحقًا من حل حزمة السجل نفسها. يدعم CLI الخاص بـ ClawHub أيضًا مسارات عمل تثبيت المهارات وتحديثها مباشرة للمستخدمين الذين يريدون مجلدات مهارات يديرها السجل خارج مساحة عمل OpenClaw كاملة.

## الحالة الأمنية

ClawHub مفتوح للنشر، لكن الإصدارات لا تزال خاضعة لبوابات الرفع والفحوصات الآلية وبلاغات المستخدمين وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. قد يختفي المحتوى المحتجز أو المخفي أو المحظور من البحث العام ومسارات التثبيت، مع بقائه مرئيًا للمالك لأغراض التشخيص.

راجع [الأمان](/ar/clawhub/security)، و[تدقيقات الأمان](/ar/clawhub/security-audits)، و[الإشراف وسلامة الحساب](/ar/clawhub/moderation)، و[الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## الوصول إلى API

يوفر ClawHub واجهات API عامة للقراءة من أجل الاكتشاف والبحث وتفاصيل الحزم والتنزيلات. يمكن للفهارس التابعة لجهات خارجية استخدام واجهات API هذه عندما تربط بإدراج ClawHub المرجعي، وتحترم حدود المعدلات، وتتجنب الإيحاء بالتأييد.

راجع [API العام](/ar/clawhub/api) و[HTTP API](/ar/clawhub/http-api).
