# كيف يعمل ClawHub

ClawHub هو طبقة السجل لـ Skills وPlugins في OpenClaw. يمنح المستخدمين
مكانًا لاكتشاف الحزم، ويمنح الناشرين مكانًا لإصدار النسخ، ويمنح
OpenClaw بيانات وصفية كافية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يحتوي على:

- مالك واسم مختصر أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية وملخص وملفات وإسناد المصدر
- سجل تغييرات ومعلومات وسوم مثل `latest`
- إشارات التنزيل والتثبيت والنجوم
- حالة الفحص الأمني والإشراف

صفحة الإدراج هي المكان المرجعي الذي يستخدمه المستخدمون لفحص ما تزعم Skills أو
Plugin أنها تفعله قبل تثبيتها.

## Skills

Skill هي حزمة نصية ذات نسخ تتمحور حول `SKILL.md`. يمكن أن تتضمن
ملفات داعمة وأمثلة وقوالب وسكربتات.

يقرأ ClawHub بيانات الواجهة الأمامية في `SKILL.md` لفهم اسم Skill
ووصفها ومتطلباتها ومتغيرات البيئة والبيانات الوصفية. دقة البيانات الوصفية
مهمة لأنها تساعد المستخدمين على تحديد ما إذا كانوا سيثبتون Skill وتساعد
الفحوصات الآلية على اكتشاف أوجه عدم التطابق بين السلوك المعلن والسلوك المرصود.

راجع [تنسيق Skill](/ar/clawhub/skill-format).

## Plugins

Plugins هي إضافات OpenClaw معبأة. يخزن ClawHub البيانات الوصفية للحزمة،
ومعلومات التوافق، وروابط المصدر، والقطع الأثرية، وسجلات النسخ.

عندما يثبت OpenClaw Plugin من ClawHub، فإنه يتحقق من بيانات التوافق
الوصفية المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحزم توافق واجهة برمجة
التطبيقات، والحد الأدنى لإصدار Gateway، وأهداف المضيف، ومتطلبات البيئة،
وملخصات القطع الأثرية.

استخدم مصدر تثبيت ClawHub صريحًا عندما تريد أن يكون السجل هو
مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل نسخة جديدًا غير قابل للتغيير. يستخدم الناشرون CLI الخاص بـ
`clawhub` لسير عمل السجل المصادق عليه:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم التشغيل التجريبي لمعاينة الحمولة المحلولة قبل الرفع. تعرض الصفحات العامة
بعد ذلك البيانات الوصفية المنشورة، والملفات، وإسناد المصدر، وحالة الفحص.

## التثبيتات والتحديثات

تستخدم أوامر تثبيت OpenClaw ClawHub كمصدر حزم:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

يسجل OpenClaw البيانات الوصفية لمصدر التثبيت حتى تتمكن التحديثات لاحقًا من
حل حزمة السجل نفسها. يدعم CLI الخاص بـ ClawHub أيضًا سير عمل تثبيت
Skills وتحديثها مباشرة للمستخدمين الذين يريدون مجلدات Skills مُدارة عبر
السجل خارج مساحة عمل OpenClaw كاملة.

## الحالة الأمنية

ClawHub مفتوح للنشر، لكن الإصدارات تظل خاضعة لبوابات الرفع،
والفحوصات الآلية، وبلاغات المستخدمين، وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. قد يختفي المحتوى المحتجز أو
المخفي أو المحظور من البحث العام وسير عمل التثبيت، مع بقائه مرئيًا للمالك
لأغراض التشخيص.

راجع [الأمان](/clawhub/security)، و[عمليات التدقيق الأمني](/clawhub/security-audits)،
و[الإشراف وسلامة الحساب](/ar/clawhub/moderation)، و
[الاستخدام المقبول](/clawhub/acceptable-usage).

## الوصول إلى واجهة برمجة التطبيقات

يوفر ClawHub واجهات برمجة تطبيقات قراءة عامة للاكتشاف، والبحث، وتفاصيل الحزم،
والتنزيلات. يمكن للفهارس التابعة لجهات خارجية استخدام هذه الواجهات عندما تربط
مرة أخرى بإدراج ClawHub المرجعي، وتحترم حدود المعدل، وتتجنب الإيحاء بالتأييد.

راجع [واجهة برمجة التطبيقات العامة](/ar/clawhub/api) و[واجهة HTTP API](/clawhub/http-api).
