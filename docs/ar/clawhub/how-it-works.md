# كيف يعمل ClawHub

ClawHub هو طبقة السجل لـ Skills وPlugin في OpenClaw. يمنح المستخدمين
مكانًا لاكتشاف الحزم، ويمنح الناشرين مكانًا لإصدار النسخ، ويمنح
OpenClaw ما يكفي من البيانات الوصفية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يتضمن:

- مالكًا ومعرّفًا قصيرًا أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية، وملخصًا، وملفات، وإسنادًا للمصدر
- سجل تغييرات ومعلومات وسوم مثل `latest`
- إشارات التنزيل، والتثبيت، والنجوم، والتعليقات
- حالة الفحص الأمني والإشراف

صفحة الإدراج هي المكان الموثوق للمستخدمين لفحص ما تدّعي Skill أو
Plugin أنها تفعله قبل تثبيتها.

## Skills

Skill هي حزمة نصية ذات نسخ تتمحور حول `SKILL.md`. يمكن أن تتضمن
ملفات داعمة، وأمثلة، وقوالب، وسكربتات.

يقرأ ClawHub واجهة `SKILL.md` الأمامية لفهم اسم Skill،
ووصفها، ومتطلباتها، ومتغيرات البيئة، وبياناتها الوصفية. تهم البيانات
الوصفية الدقيقة لأنها تساعد المستخدمين على تقرير ما إذا كانوا سيثبتون
Skill، وتساعد الفحوصات الآلية على اكتشاف أوجه عدم التطابق بين السلوك
المعلن والسلوك المرصود.

راجع [تنسيق Skill](/ar/clawhub/skill-format).

## Plugin

Plugin هي امتدادات OpenClaw معبأة. يخزن ClawHub بيانات الحزمة
الوصفية، ومعلومات التوافق، وروابط المصدر، والقطع الأثرية، وسجلات النسخ.

عندما يثبت OpenClaw Plugin من ClawHub، فإنه يتحقق من بيانات التوافق
الوصفية المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحزم توافق API،
والحد الأدنى من إصدار Gateway، وأهداف المضيف، ومتطلبات البيئة، وبصمات
القطع الأثرية.

استخدم مصدر تثبيت ClawHub صريحًا عندما تريد أن يكون السجل هو مصدر
الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل نسخة جديدًا غير قابل للتغيير. يستخدم الناشرون CLI
`clawhub` لسير عمل السجل الموثق:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم التشغيل التجريبي لمعاينة الحمولة المحلولة قبل الرفع. ثم تعرض
الصفحات العامة البيانات الوصفية المنشورة، والملفات، وإسناد المصدر،
وحالة الفحص.

## عمليات التثبيت والتحديث

تستخدم أوامر تثبيت OpenClaw ClawHub كمصدر للحزم:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

يسجل OpenClaw بيانات وصفية لمصدر التثبيت بحيث يمكن للتحديثات حل حزمة
السجل نفسها لاحقًا. يدعم CLI الخاص بـ ClawHub أيضًا مسارات عمل تثبيت
وتحديث Skill مباشرة للمستخدمين الذين يريدون مجلدات Skill مُدارة عبر
السجل خارج مساحة عمل OpenClaw كاملة.

## الحالة الأمنية

ClawHub مفتوح للنشر، لكن الإصدارات تظل خاضعة لبوابات الرفع،
والفحوصات الآلية، وبلاغات المستخدمين، وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. قد يختفي المحتوى
المحتجز أو المخفي أو المحظور من البحث العام ومسارات التثبيت، مع بقائه
مرئيًا للمالك لأغراض التشخيص.

راجع [الأمان + الإشراف](/ar/clawhub/security) و
[الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## الوصول إلى API

يوفر ClawHub واجهات API عامة للقراءة من أجل الاكتشاف، والبحث،
وتفاصيل الحزم، والتنزيلات. يمكن للكتالوجات التابعة لجهات خارجية
استخدام واجهات API هذه عندما تربط عائدًا إلى إدراج ClawHub الموثوق،
وتحترم حدود المعدل، وتتجنب الإيحاء بالتأييد.

راجع [API العام](/ar/clawhub/api) و[HTTP API](/ar/clawhub/http-api).
