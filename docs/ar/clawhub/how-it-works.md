# كيف يعمل ClawHub

ClawHub هو طبقة السجل لمهارات OpenClaw وPluginاته. يمنح المستخدمين
مكانًا لاكتشاف الحزم، ويمنح الناشرين مكانًا لإصدار النسخ، ويمنح
OpenClaw بيانات وصفية كافية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يتضمن:

- مالكًا واسمًا مختصرًا أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية، وملخصًا، وملفات، ونسبة المصدر
- سجل تغييرات ومعلومات وسوم مثل `latest`
- إشارات التنزيل، والتثبيت، والنجوم
- حالة فحص الأمان والإشراف

صفحة الإدراج هي المكان المعتمد الذي يتيح للمستخدمين فحص ما تدعي المهارة أو
Plugin أنها تفعله قبل تثبيتها.

## Skills

المهارة هي حزمة نصية ذات إصدارات تتمحور حول `SKILL.md`. ويمكن أن تتضمن
ملفات داعمة، وأمثلة، وقوالب، وسكربتات.

يقرأ ClawHub بيانات الواجهة في `SKILL.md` لفهم اسم المهارة
ووصفها ومتطلباتها ومتغيرات البيئة والبيانات الوصفية. دقة البيانات الوصفية
مهمة لأنها تساعد المستخدمين على تقرير ما إذا كانوا سيثبتون المهارة، وتساعد
الفحوصات الآلية على اكتشاف حالات عدم التطابق بين السلوك المعلن والسلوك المرصود.

راجع [تنسيق المهارة](/ar/clawhub/skill-format).

## Plugins

Plugins هي امتدادات OpenClaw معبأة. يخزن ClawHub بيانات الحزم الوصفية،
ومعلومات التوافق، وروابط المصدر، والقطع الأثرية، وسجلات الإصدارات.

عندما يثبت OpenClaw Plugin من ClawHub، فإنه يتحقق من بيانات التوافق الوصفية
المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحزم توافق API،
وإصدار Gateway الأدنى، وأهداف المضيف، ومتطلبات البيئة، وبصمات القطع الأثرية.

استخدم مصدر تثبيت ClawHub صريحًا عندما تريد أن يكون السجل هو
مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل إصدار جديدًا غير قابل للتغيير. يستخدم الناشرون CLI `clawhub`
لسير عمل السجل المصادق عليه:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم التشغيلات التجريبية لمعاينة الحمولة المحلولة قبل الرفع. ثم تعرض الصفحات العامة
البيانات الوصفية المنشورة، والملفات، ونسبة المصدر، وحالة الفحص.

## التثبيتات والتحديثات

تستخدم أوامر تثبيت OpenClaw ClawHub كمصدر حزم:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

يسجل OpenClaw بيانات مصدر التثبيت الوصفية لكي تتمكن التحديثات لاحقًا من حل
حزمة السجل نفسها. يدعم CLI الخاص بـ ClawHub أيضًا مسارات عمل تثبيت المهارات
وتحديثها مباشرة للمستخدمين الذين يريدون مجلدات مهارات يديرها السجل خارج
مساحة عمل OpenClaw كاملة.

## حالة الأمان

ClawHub مفتوح للنشر، لكن الإصدارات تظل خاضعة لبوابات الرفع،
والفحوصات الآلية، وبلاغات المستخدمين، وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. قد يختفي المحتوى المحتجز أو المخفي
أو المحظور من البحث العام ومسارات التثبيت، مع بقائه مرئيًا
للمالك لأغراض التشخيص.

راجع [الأمان](/clawhub/security)، و[تدقيقات الأمان](/clawhub/security-audits)،
و[الإشراف وسلامة الحساب](/ar/clawhub/moderation)، و
[الاستخدام المقبول](/clawhub/acceptable-usage).

## الوصول إلى API

يوفر ClawHub واجهات API قراءة عامة للاكتشاف، والبحث، وتفاصيل الحزم، و
التنزيلات. يمكن للفهارس الخارجية استخدام واجهات API هذه عندما تربط مرة أخرى
بالإدراج المعتمد في ClawHub، وتحترم حدود المعدل، وتتجنب الإيحاء بالتأييد.

راجع [API العام](/ar/clawhub/api) و[HTTP API](/clawhub/http-api).
