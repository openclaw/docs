# كيف يعمل ClawHub

ClawHub هو طبقة السجل لـ OpenClaw Skills وplugins. يمنح المستخدمين
مكانًا لاكتشاف الحزم، ويمنح الناشرين مكانًا لإصدار النسخ، ويمنح
OpenClaw ما يكفي من البيانات الوصفية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يحتوي على:

- مالك وslug أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية، وملخص، وملفات، ونسبة المصدر
- سجل تغييرات ومعلومات وسوم مثل `latest`
- إشارات التنزيل، والتثبيت، والنجوم، والتعليقات
- حالة الفحص الأمني والإشراف

صفحة الإدراج هي المكان المعتمد للمستخدمين لفحص ما تزعم Skill أو
plugin أنها تفعله قبل تثبيتها.

## Skills

Skill هي حزمة نصية ذات نسخ تتمحور حول `SKILL.md`. يمكن أن تتضمن
ملفات داعمة، وأمثلة، وقوالب، وسكربتات.

يقرأ ClawHub بيانات frontmatter في `SKILL.md` لفهم اسم Skill
ووصفها ومتطلباتها ومتغيرات البيئة وبياناتها الوصفية. دقة البيانات
الوصفية مهمة لأنها تساعد المستخدمين على تحديد ما إذا كانوا سيثبتون Skill
وتساعد الفحوصات الآلية على اكتشاف حالات عدم التطابق بين السلوك المصرح به
والسلوك المرصود.

راجع [تنسيق Skill](/ar/clawhub/skill-format).

## Plugins

Plugins هي إضافات OpenClaw معبأة. يخزن ClawHub بيانات الحزمة الوصفية،
ومعلومات التوافق، وروابط المصدر، والملفات الناتجة، وسجلات النسخ.

عندما يثبت OpenClaw plugin من ClawHub، فإنه يتحقق من بيانات التوافق
الوصفية المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحزم توافق API،
والحد الأدنى من إصدار Gateway، وأهداف المضيف، ومتطلبات البيئة، وملخصات
الملفات الناتجة.

استخدم مصدر تثبيت ClawHub صريحًا عندما تريد أن يكون السجل هو
مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل نسخة جديدًا غير قابل للتغيير. يستخدم الناشرون CLI
`clawhub` لسير عمل السجل الموثق:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم التشغيلات التجريبية لمعاينة الحمولة المحلولة قبل الرفع. تعرض
الصفحات العامة بعد ذلك البيانات الوصفية المنشورة، والملفات، ونسبة المصدر،
وحالة الفحص.

## التثبيتات والتحديثات

تستخدم أوامر التثبيت في OpenClaw‏ ClawHub كمصدر حزم:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

يسجل OpenClaw بيانات مصدر التثبيت الوصفية لكي تتمكن التحديثات من حل
حزمة السجل نفسها لاحقًا. يدعم ClawHub CLI أيضًا مسارات عمل مباشرة
لتثبيت Skills وتحديثها للمستخدمين الذين يريدون مجلدات Skills مُدارة
بالسجل خارج مساحة عمل OpenClaw كاملة.

## الحالة الأمنية

ClawHub مفتوح للنشر، لكن الإصدارات تظل خاضعة لبوابات الرفع،
والفحوصات الآلية، وبلاغات المستخدمين، وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. قد يختفي المحتوى المحتجز
أو المخفي أو المحظور من البحث العام ومسارات التثبيت، مع بقائه مرئيًا
للمالك لأغراض التشخيص.

راجع [الأمان + الإشراف](/ar/clawhub/security) و
[الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## الوصول إلى API

يوفر ClawHub واجهات API عامة للقراءة من أجل الاكتشاف، والبحث، وتفاصيل
الحزم، والتنزيلات. يمكن للفهارس التابعة لجهات خارجية استخدام واجهات API
هذه عندما تربط بالعودة إلى إدراج ClawHub المعتمد، وتحترم حدود المعدل،
وتتجنب الإيحاء بالتأييد.

راجع [API العام](/ar/clawhub/api) و[HTTP API](/ar/clawhub/http-api).
