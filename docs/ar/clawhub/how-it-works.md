# كيف يعمل ClawHub

ClawHub هو طبقة السجل لمهارات Skills وPlugins في OpenClaw. يوفّر للمستخدمين
مكانًا لاكتشاف الحزم، وللناشرين مكانًا لإصدار النسخ، ويمنح OpenClaw ما يكفي من
البيانات الوصفية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يتضمن:

- مالكًا واسمًا مختصرًا أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية، وملخصًا، وملفات، وإسنادًا للمصدر
- سجل تغييرات ومعلومات وسوم مثل `latest`
- إشارات التنزيل، والتثبيت، والنجوم، والتعليقات
- حالة الفحص الأمني والإشراف

صفحة الإدراج هي المكان المعتمد الذي يراجع فيه المستخدمون ما تزعم Skills أو
Plugin أنها تفعله قبل تثبيتها.

## Skills

Skills هي حزمة نصية ذات نسخ تتمحور حول `SKILL.md`. يمكن أن تتضمن
ملفات داعمة، وأمثلة، وقوالب، وسكربتات.

يقرأ ClawHub بيانات الواجهة الأمامية في `SKILL.md` لفهم اسم Skills
والوصف، والمتطلبات، ومتغيرات البيئة، والبيانات الوصفية. دقة البيانات الوصفية
مهمة لأنها تساعد المستخدمين على تحديد ما إذا كانوا سيثبتون Skills وتساعد
الفحوصات الآلية على اكتشاف حالات عدم التطابق بين السلوك المعلن والسلوك المرصود.

راجع [تنسيق Skills](/ar/clawhub/skill-format).

## Plugins

Plugins هي إضافات OpenClaw محزّمة. يخزّن ClawHub بيانات الحزمة الوصفية،
ومعلومات التوافق، وروابط المصدر، والقطع الأثرية، وسجلات النسخ.

عندما يثبّت OpenClaw ‏Plugin من ClawHub، فإنه يتحقق من بيانات التوافق الوصفية
المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحزم توافق API، والحد الأدنى من نسخة
Gateway، وأهداف المضيف، ومتطلبات البيئة، وملخصات القطع الأثرية.

استخدم مصدر تثبيت ClawHub صريحًا عندما تريد أن يكون السجل هو مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل نسخة جديدًا غير قابل للتغيير. يستخدم الناشرون CLI الخاص بـ
`clawhub` لسير عمل السجل الموثق:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم التشغيلات الجافة لمعاينة الحمولة المحلولة قبل الرفع. بعد ذلك تعرض
الصفحات العامة البيانات الوصفية المنشورة، والملفات، وإسناد المصدر، وحالة الفحص.

## التثبيتات والتحديثات

تستخدم أوامر تثبيت OpenClaw ‏ClawHub كمصدر للحزم:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

يسجل OpenClaw بيانات وصفية لمصدر التثبيت كي تتمكن التحديثات لاحقًا من حل
حزمة السجل نفسها. يدعم CLI الخاص بـ ClawHub أيضًا مسارات عمل تثبيت Skills
وتحديثها مباشرة للمستخدمين الذين يريدون مجلدات Skills مُدارة بالسجل خارج مساحة
عمل OpenClaw كاملة.

## الحالة الأمنية

ClawHub مفتوح للنشر، لكن الإصدارات لا تزال خاضعة لبوابات الرفع،
والفحوصات الآلية، وبلاغات المستخدمين، وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. قد يختفي المحتوى المحتجز، أو
المخفي، أو المحظور من البحث العام ومسارات التثبيت مع بقائه مرئيًا للمالك لأغراض
التشخيص.

راجع [الأمان + الإشراف](/ar/clawhub/security) و
[الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## الوصول إلى API

يكشف ClawHub واجهات API عامة للقراءة من أجل الاكتشاف، والبحث، وتفاصيل الحزم،
والتنزيلات. يمكن للكتالوجات التابعة لجهات خارجية استخدام واجهات API هذه عندما
تربط عائدًا إلى إدراج ClawHub المعتمد، وتحترم حدود المعدل، وتتجنب الإيحاء
بالتأييد.

راجع [API العام](/ar/clawhub/api) و[HTTP API](/ar/clawhub/http-api).
