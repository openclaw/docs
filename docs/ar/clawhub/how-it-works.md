# كيف يعمل ClawHub

ClawHub هو طبقة السجل الخاصة بمهارات OpenClaw والـ Plugins. يمنح المستخدمين
مكانًا لاكتشاف الحزم، ويمنح الناشرين مكانًا لإصدار النسخ، ويمنح
OpenClaw ما يكفي من البيانات الوصفية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يتضمن:

- مالكًا ومعرّفًا نصيًا أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية وملخصًا وملفات ونسبة إلى المصدر
- سجل تغييرات ومعلومات وسوم مثل `latest`
- إشارات التنزيل والتثبيت والنجوم والتعليقات
- حالة الفحص الأمني والإشراف

صفحة الإدراج هي المكان الموثوق الذي يطّلع فيه المستخدمون على ما تزعم المهارة أو
الـ Plugin أنها تفعله قبل تثبيتها.

## Skills

المهارة هي حزمة نصية ذات إصدارات تتمحور حول `SKILL.md`. ويمكن أن تتضمن
ملفات داعمة وأمثلة وقوالب وبرامج نصية.

يقرأ ClawHub واجهة `SKILL.md` الأمامية لفهم اسم المهارة
ووصفها ومتطلباتها ومتغيرات البيئة والبيانات الوصفية. تهم دقة
البيانات الوصفية لأنها تساعد المستخدمين على تحديد ما إذا كانوا سيثبتون المهارة،
وتساعد الفحوصات المؤتمتة على اكتشاف حالات عدم التطابق بين السلوك المعلن والسلوك المرصود.

راجع [تنسيق المهارة](/ar/clawhub/skill-format).

## Plugins

الـ Plugins هي إضافات OpenClaw معبأة. يخزن ClawHub بيانات الحزمة الوصفية،
ومعلومات التوافق، وروابط المصدر، والآثار، وسجلات الإصدارات.

عندما يثبت OpenClaw ‏Plugin من ClawHub، فإنه يتحقق من بيانات التوافق
الوصفية المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحزم توافق API،
والحد الأدنى لإصدار Gateway، وأهداف المضيف، ومتطلبات البيئة، وبصمات
الآثار.

استخدم مصدر تثبيت ClawHub صريحًا عندما تريد أن يكون السجل هو
مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل إصدار جديدًا غير قابل للتغيير. يستخدم الناشرون CLI الخاص بـ `clawhub`
لسير عمل السجل الموثق:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم التشغيلات الجافة لمعاينة الحمولة المحلولة قبل الرفع. بعد ذلك تعرض
الصفحات العامة البيانات الوصفية المنشورة والملفات ونسبة المصدر وحالة الفحص.

## التثبيتات والتحديثات

تستخدم أوامر تثبيت OpenClaw ‏ClawHub كمصدر حزم:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

يسجل OpenClaw بيانات وصفية لمصدر التثبيت حتى تتمكن التحديثات من حل حزمة
السجل نفسها لاحقًا. يدعم CLI الخاص بـ ClawHub أيضًا سير عمل مباشرًا لتثبيت
المهارات وتحديثها للمستخدمين الذين يريدون مجلدات مهارات يديرها السجل خارج
مساحة عمل OpenClaw كاملة.

## الحالة الأمنية

ClawHub مفتوح للنشر، لكن الإصدارات تظل خاضعة لبوابات الرفع،
والفحوصات المؤتمتة، وبلاغات المستخدمين، وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. قد يختفي المحتوى المحتجز أو المخفي
أو المحظور من البحث العام وتدفقات التثبيت، مع بقائه
مرئيًا للمالك لأغراض التشخيص.

راجع [الأمان + الإشراف](/ar/clawhub/security) و
[الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## الوصول إلى API

يوفر ClawHub واجهات API قراءة عامة للاكتشاف والبحث وتفاصيل الحزم
والتنزيلات. يمكن للفهارس التابعة لجهات خارجية استخدام واجهات API هذه عندما
تضع رابطًا إلى إدراج ClawHub الموثوق، وتحترم حدود المعدل، وتتجنب الإيحاء
بالتأييد.

راجع [API العامة](/ar/clawhub/api) و [HTTP API](/ar/clawhub/http-api).
