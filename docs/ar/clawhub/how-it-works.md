# كيف يعمل ClawHub

ClawHub هو طبقة السجل لـ Skills وPlugin في OpenClaw. يمنح المستخدمين
مكانًا لاكتشاف الحزم، ويمنح الناشرين مكانًا لإصدار النسخ، ويمنح
OpenClaw ما يكفي من البيانات الوصفية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يحتوي على:

- مالك ومعرّف مختصر أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية وملخص وملفات وإسناد المصدر
- سجل تغييرات ومعلومات وسوم مثل `latest`
- إشارات التنزيل والتثبيت والنجوم
- حالة الفحص الأمني والإشراف

صفحة الإدراج هي المكان المعتمد الذي يمكن للمستخدمين فيه فحص ما تزعم Skill أو
Plugin أنها تفعله قبل تثبيتها.

## Skills

Skill هي حزمة نصية ذات إصدارات تتمحور حول `SKILL.md`. يمكن أن تتضمن
ملفات داعمة وأمثلة وقوالب وسكربتات.

يقرأ ClawHub الواجهة الأمامية في `SKILL.md` لفهم اسم Skill
ووصفها ومتطلباتها ومتغيرات البيئة والبيانات الوصفية. تهم البيانات
الوصفية الدقيقة لأنها تساعد المستخدمين على تحديد ما إذا كانوا سيثبتون Skill
وتساعد الفحوصات الآلية على اكتشاف حالات عدم التطابق بين السلوك المعلن
والسلوك المرصود.

راجع [تنسيق Skill](/ar/clawhub/skill-format).

## Plugins

Plugins هي إضافات OpenClaw معبأة. يخزن ClawHub بيانات الحزمة الوصفية
ومعلومات التوافق وروابط المصدر والقطع الأثرية وسجلات الإصدارات.

عندما يثبت OpenClaw إضافة من ClawHub، فإنه يتحقق من بيانات التوافق
الوصفية المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحزم توافق API،
والحد الأدنى لإصدار Gateway، وأهداف المضيف، ومتطلبات البيئة، وبصمات
القطع الأثرية.

استخدم مصدر تثبيت ClawHub صريحًا عندما تريد أن يكون السجل هو
مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل إصدار جديدًا غير قابل للتغيير. يستخدم الناشرون CLI باسم `clawhub`
لسير عمل السجل الموثق:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم التشغيلات التجريبية لمعاينة الحمولة المحلولة قبل الرفع. ثم تعرض
الصفحات العامة البيانات الوصفية المنشورة والملفات وإسناد المصدر وحالة الفحص.

## التثبيتات والتحديثات

تستخدم أوامر تثبيت OpenClaw ClawHub كمصدر للحزم:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

يسجل OpenClaw بيانات مصدر التثبيت الوصفية حتى تتمكن التحديثات من حل
حزمة السجل نفسها لاحقًا. يدعم CLI الخاص بـ ClawHub أيضًا سير عمل تثبيت
وتحديث Skills مباشرة للمستخدمين الذين يريدون مجلدات Skills يديرها السجل
خارج مساحة عمل OpenClaw كاملة.

## الحالة الأمنية

ClawHub مفتوح للنشر، لكن الإصدارات تظل خاضعة لبوابات الرفع،
والفحوصات الآلية، وبلاغات المستخدمين، وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. قد يختفي المحتوى المحتجز
أو المخفي أو المحظور من البحث العام ومسارات التثبيت، مع بقائه مرئيًا
للمالك لأغراض التشخيص.

راجع [الأمان](/clawhub/security)، و[عمليات التدقيق الأمني](/clawhub/security-audits)،
و[الإشراف وسلامة الحساب](/ar/clawhub/moderation)، و
[الاستخدام المقبول](/clawhub/acceptable-usage).

## الوصول إلى API

يوفر ClawHub واجهات API عامة للقراءة من أجل الاكتشاف والبحث وتفاصيل
الحزم والتنزيلات. يمكن للفهارس التابعة لجهات خارجية استخدام واجهات API هذه
عندما تربط عائدة إلى إدراج ClawHub المعتمد، وتحترم حدود المعدل، وتتجنب
الإيحاء بالمصادقة أو التزكية.

راجع [API العامة](/ar/clawhub/api) و[HTTP API](/clawhub/http-api).
