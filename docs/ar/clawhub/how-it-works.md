# كيف يعمل ClawHub

ClawHub هو طبقة السجل لـ Skills وPlugins في OpenClaw. يمنح المستخدمين
مكانًا لاكتشاف الحزم، ويمنح الناشرين مكانًا لإصدار النسخ، ويمنح
OpenClaw ما يكفي من البيانات الوصفية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يتضمن:

- مالكًا ومعرّفًا أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية وملخصًا وملفات وإسنادًا للمصدر
- معلومات سجل التغييرات والوسوم مثل `latest`
- إشارات التنزيل والتثبيت والنجوم
- حالة الفحص الأمني والإشراف

صفحة الإدراج هي المكان المرجعي للمستخدمين لفحص ما تدّعي Skill أو
Plugin أنها تفعله قبل تثبيتها.

## Skills

Skill هي حزمة نصية ذات نسخ تتمحور حول `SKILL.md`. ويمكن أن تتضمن
ملفات داعمة وأمثلة وقوالب وسكربتات.

يقرأ ClawHub واجهة `SKILL.md` الأمامية لفهم اسم Skill
ووصفها ومتطلباتها ومتغيرات البيئة والبيانات الوصفية. تهم دقة
البيانات الوصفية لأنها تساعد المستخدمين على تحديد ما إذا كانوا سيثبتون Skill،
وتساعد الفحوصات الآلية على اكتشاف أوجه عدم التطابق بين السلوك المعلن
والسلوك المرصود.

راجع [تنسيق Skill](/ar/clawhub/skill-format).

## Plugins

Plugins هي امتدادات OpenClaw مغلفة في حزم. يخزن ClawHub بيانات الحزمة
الوصفية ومعلومات التوافق وروابط المصدر والملفات الأثرية وسجلات النسخ.

عندما يثبّت OpenClaw Plugin من ClawHub، فإنه يتحقق من بيانات التوافق
الوصفية المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحزم توافق API،
والحد الأدنى لإصدار Gateway، وأهداف المضيف، ومتطلبات البيئة، وبصمات
الملفات الأثرية.

استخدم مصدر تثبيت ClawHub صريحًا عندما تريد أن يكون السجل هو
مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل نسخة جديدًا غير قابل للتغيير. يستخدم الناشرون CLI الخاص بـ `clawhub`
لسير عمل السجل الموثق:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم عمليات التشغيل التجريبي لمعاينة الحمولة المحلولة قبل الرفع. بعد ذلك
تعرض الصفحات العامة البيانات الوصفية المنشورة والملفات وإسناد المصدر
وحالة الفحص.

## التثبيتات والتحديثات

تستخدم أوامر تثبيت OpenClaw ClawHub كمصدر للحزم:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

يسجل OpenClaw بيانات وصفية لمصدر التثبيت حتى تتمكن التحديثات لاحقًا من حل
حزمة السجل نفسها. يدعم CLI الخاص بـ ClawHub أيضًا سير عمل تثبيت Skills
وتحديثها مباشرة للمستخدمين الذين يريدون مجلدات Skills مُدارة من السجل خارج
مساحة عمل OpenClaw كاملة.

## الحالة الأمنية

ClawHub مفتوح للنشر، لكن الإصدارات تظل خاضعة لبوابات الرفع،
والفحوصات الآلية، وبلاغات المستخدمين، وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. قد يختفي المحتوى المحتجز
أو المخفي أو المحظور من البحث العام ومسارات التثبيت، مع بقائه
مرئيًا للمالك لأغراض التشخيص.

راجع [الأمان](/clawhub/security)، و[عمليات تدقيق الأمان](/clawhub/security-audits)،
و[الإشراف وسلامة الحساب](/ar/clawhub/moderation)، و
[الاستخدام المقبول](/clawhub/acceptable-usage).

## الوصول إلى API

يوفر ClawHub واجهات API عامة للقراءة من أجل الاكتشاف والبحث وتفاصيل الحزم
والتنزيلات. يمكن للفهارس التابعة لجهات خارجية استخدام واجهات API هذه عندما
تربط عائدًا بإدراج ClawHub المرجعي، وتلتزم بحدود المعدل، وتتجنب الإيحاء
بوجود تأييد.

راجع [API العام](/ar/clawhub/api) و[واجهة HTTP API](/clawhub/http-api).
