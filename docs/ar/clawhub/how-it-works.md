# كيف يعمل ClawHub

ClawHub هو طبقة السجل لمهارات OpenClaw وPlugins الخاصة به. يمنح المستخدمين
مكانًا لاكتشاف الحزم، ويمنح الناشرين مكانًا لإصدار النسخ، ويمنح
OpenClaw بيانات وصفية كافية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل إدراج عام هو سجل في السجل يتضمن:

- مالكًا واسمًا مختصرًا أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية وملخصًا وملفات ونسبة إلى المصدر
- سجل تغييرات ومعلومات وسوم مثل `latest`
- إشارات التنزيل والتثبيت والنجوم
- حالة فحص الأمان والإشراف

صفحة الإدراج هي المكان المرجعي الذي يتيح للمستخدمين فحص ما تدعي المهارة أو
Plugin أنها تفعله قبل تثبيتها.

## Skills

المهارة هي حزمة نصية ذات نسخ تتمحور حول `SKILL.md`. ويمكن أن تتضمن
ملفات داعمة وأمثلة وقوالب ونصوصًا برمجية.

يقرأ ClawHub بيانات frontmatter في `SKILL.md` لفهم اسم المهارة
ووصفها ومتطلباتها ومتغيرات البيئة والبيانات الوصفية. البيانات الوصفية الدقيقة
مهمة لأنها تساعد المستخدمين على تحديد ما إذا كانوا سيثبتون المهارة، وتساعد
عمليات الفحص الآلية على اكتشاف عدم التطابق بين السلوك المعلن والسلوك المرصود.

راجع [تنسيق المهارة](/ar/clawhub/skill-format).

## Plugins

Plugins هي امتدادات OpenClaw مُغلّفة كحزم. يخزن ClawHub بيانات الحزمة الوصفية
ومعلومات التوافق وروابط المصدر والقطع الأثرية وسجلات النسخ.

عندما يثبت OpenClaw Plugin من ClawHub، فإنه يتحقق من بيانات التوافق الوصفية
المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحزم توافق API، والحد الأدنى لإصدار
Gateway، وأهداف المضيف، ومتطلبات البيئة، وبصمات القطع الأثرية.

استخدم مصدر تثبيت ClawHub صريحًا عندما تريد أن يكون السجل هو
مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل نسخة جديدًا غير قابل للتغيير. يستخدم الناشرون CLI الخاص بـ `clawhub`
لسير عمل السجل الموثق:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم التشغيل التجريبي لمعاينة الحمولة المحلولة قبل الرفع. ثم تعرض الصفحات العامة
البيانات الوصفية المنشورة والملفات ونسبة المصدر وحالة الفحص.

## التثبيتات والتحديثات

تستخدم أوامر تثبيت OpenClaw ClawHub كمصدر للحزم:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

يسجل OpenClaw بيانات مصدر التثبيت الوصفية بحيث يمكن للتحديثات حل حزمة السجل
نفسها لاحقًا. يدعم CLI الخاص بـ ClawHub أيضًا سير عمل تثبيت المهارات وتحديثها
مباشرة للمستخدمين الذين يريدون مجلدات مهارات مُدارة عبر السجل خارج مساحة عمل
OpenClaw كاملة.

## حالة الأمان

ClawHub مفتوح للنشر، لكن الإصدارات تظل خاضعة لبوابات الرفع
والفحوصات الآلية وبلاغات المستخدمين وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. قد يختفي المحتوى المحتجز أو المخفي
أو المحظور من البحث العام وسير عمل التثبيت، مع بقائه مرئيًا للمالك لأغراض
التشخيص.

راجع [الأمان](/ar/clawhub/security)، و[تدقيقات الأمان](/ar/clawhub/security-audits)،
و[الإشراف وسلامة الحساب](/ar/clawhub/moderation)، و
[الاستخدام المقبول](/ar/clawhub/acceptable-usage).

## الوصول إلى API

يوفر ClawHub واجهات API قراءة عامة للاكتشاف والبحث وتفاصيل الحزم
والتنزيلات. يمكن للفهارس الخارجية استخدام واجهات API هذه عندما تربط بإدراج
ClawHub المرجعي، وتحترم حدود المعدل، وتتجنب الإيحاء بالتأييد.

راجع [API العامة](/ar/clawhub/api) و[HTTP API](/ar/clawhub/http-api).
