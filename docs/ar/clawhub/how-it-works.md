# كيف يعمل ClawHub

ClawHub هو طبقة السجل لـ Skills وPlugin في OpenClaw. يمنح المستخدمين
مكانًا لاكتشاف الحزم، ويمنح الناشرين مكانًا لإصدار النسخ، ويمنح
OpenClaw بيانات وصفية كافية لتثبيت تلك الحزم وتحديثها بأمان.

## سجلات السجل

كل قائمة عامة هي سجل في السجل يتضمن:

- مالكًا ومعرّفًا مختصرًا أو اسم حزمة
- نسخة منشورة واحدة أو أكثر
- بيانات وصفية وملخصًا وملفات ونسبة المصدر
- معلومات سجل التغييرات والوسوم مثل `latest`
- إشارات التنزيل والتثبيت والنجوم
- حالة الفحص الأمني والمراجعة

صفحة القائمة هي المكان الموثوق للمستخدمين لفحص ما تدّعي Skills أو
Plugin أنها تفعله قبل تثبيتها.

## Skills

Skill هي حزمة نصية ذات نسخ تتمحور حول `SKILL.md`. يمكن أن تتضمن
ملفات داعمة وأمثلة وقوالب ونصوصًا برمجية.

يقرأ ClawHub بيانات الواجهة الأمامية في `SKILL.md` لفهم اسم Skill
ووصفها ومتطلباتها ومتغيرات البيئة والبيانات الوصفية. تهم دقة البيانات
الوصفية لأنها تساعد المستخدمين على تحديد ما إذا كانوا سيثبتون Skill،
وتساعد الفحوصات الآلية على اكتشاف عدم التطابق بين السلوك المعلن والسلوك المرصود.

راجع [تنسيق Skill](/ar/clawhub/skill-format).

## Plugin

Plugin هي امتدادات OpenClaw مُحزّمة. يخزن ClawHub بيانات الحزمة الوصفية،
ومعلومات التوافق، وروابط المصدر، والقطع الأثرية، وسجلات النسخ.

عندما يثبت OpenClaw ‏Plugin من ClawHub، فإنه يتحقق من بيانات التوافق
الوصفية المعلنة قبل التثبيت. يمكن أن تتضمن سجلات الحزم توافق API،
والحد الأدنى من إصدار Gateway، وأهداف المضيف، ومتطلبات البيئة، وبصمات
القطع الأثرية.

استخدم مصدر تثبيت ClawHub صريحًا عندما تريد أن يكون السجل هو مصدر الحقيقة:

```bash
openclaw plugins install clawhub:<package>
```

## النشر

ينشئ النشر سجل نسخة جديدًا غير قابل للتغيير. يستخدم الناشرون CLI الخاص بـ `clawhub`
لسير عمل السجل المصدق:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

استخدم التشغيلات التجريبية لمعاينة الحمولة النهائية قبل الرفع. بعد ذلك
تعرض الصفحات العامة البيانات الوصفية المنشورة والملفات ونسبة المصدر وحالة الفحص.

## التثبيتات والتحديثات

تستخدم أوامر تثبيت OpenClaw ‏ClawHub كمصدر للحزم:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

يسجل OpenClaw بيانات مصدر التثبيت الوصفية حتى تتمكن التحديثات لاحقًا من
حل الحزمة نفسها من السجل. يدعم CLI الخاص بـ ClawHub أيضًا سير عمل تثبيت
وتحديث مباشر لـ Skills للمستخدمين الذين يريدون مجلدات Skills مُدارة عبر
السجل خارج مساحة عمل OpenClaw كاملة.

## الحالة الأمنية

ClawHub مفتوح للنشر، لكن الإصدارات تظل خاضعة لبوابات الرفع،
والفحوصات الآلية، وبلاغات المستخدمين، وإجراءات المشرفين.

تعرض الصفحات العامة ملخصات الفحص عند توفرها. قد يختفي المحتوى المحتجز
أو المخفي أو المحظور من البحث العام ومسارات التثبيت مع بقائه
مرئيًا للمالك لأغراض التشخيص.

راجع [الأمان](/clawhub/security)، و[تدقيقات الأمان](/clawhub/security-audits)،
و[الإشراف وسلامة الحساب](/ar/clawhub/moderation)، و
[الاستخدام المقبول](/clawhub/acceptable-usage).

## الوصول إلى API

يوفر ClawHub واجهات API عامة للقراءة من أجل الاكتشاف والبحث وتفاصيل الحزم
والتنزيلات. يمكن للفهارس التابعة لجهات خارجية استخدام واجهات API هذه عندما
تضع رابطًا إلى قائمة ClawHub الموثوقة، وتلتزم بحدود المعدل، وتتجنب الإيحاء بالتأييد.

راجع [API العامة](/clawhub/api) و[HTTP API](/clawhub/http-api).
