Implementa OpenClaw en Northflank con una plantilla de un solo clic y accede a él a través de la interfaz web de Control UI. Esta es la ruta más sencilla "sin terminal en el servidor": Northflank ejecuta el Gateway por ti.

## Cómo empezar

1. Haz clic en [Implementar OpenClaw](https://northflank.com/stacks/deploy-openclaw) para abrir la plantilla.
2. Crea una [cuenta en Northflank](https://app.northflank.com/signup) si aún no tienes una.
3. Haz clic en **Implementar OpenClaw ahora**.
4. Define la variable de entorno obligatoria: `OPENCLAW_GATEWAY_TOKEN` (usa un valor aleatorio seguro).
5. Haz clic en **Implementar stack** para compilar y ejecutar la plantilla de OpenClaw.
6. Espera a que se complete la implementación y luego haz clic en **Ver recursos**.
7. Abre el servicio OpenClaw.
8. Abre la URL pública de OpenClaw en `/openclaw` y conéctate usando el secreto compartido configurado. Esta plantilla usa `OPENCLAW_GATEWAY_TOKEN` de forma predeterminada; si lo reemplazas con autenticación por contraseña, usa esa contraseña en su lugar.

## Qué obtienes

- Gateway + Control UI de OpenClaw alojados
- Almacenamiento persistente mediante un volumen de Northflank (`/data`) para que `openclaw.json`, `auth-profiles.json` por agente, el estado de canales/proveedores, las sesiones y el espacio de trabajo sobrevivan a las reimplementaciones

## Conecta un canal

Usa la Control UI en `/openclaw` o ejecuta `openclaw onboard` mediante SSH para obtener instrucciones de configuración de canales:

- [Telegram](/es/channels/telegram) (lo más rápido, solo un token de bot)
