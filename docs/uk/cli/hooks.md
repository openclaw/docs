# `openclaw hooks`

Керуйте хуками агента (автоматизаціями, керованими подіями, для команд на кшталт `/new`, `/reset` і запуску Gateway). Команда `openclaw hooks` без аргументів еквівалентна `openclaw hooks list`.

Пов’язане: [Хуки](/uk/automation/hooks) — [Хуки Plugin](/uk/plugins/hooks)

## Перегляд списку хуків

```bash
openclaw hooks list [--eligible] [--json] [-v|--verbose]
```

Виводить список хуків, виявлених у робочому просторі, керованих, додаткових і вбудованих каталогах.

- `--eligible`: лише хуки, вимоги яких виконано.
- `--json`: структурований вивід.
- `-v, --verbose`: додати стовпець «Відсутнє» з невиконаними вимогами.

```
Хуки (4/5 готові)

Готові:
  🚀 boot-md ✓ — Запускає BOOT.md під час запуску Gateway
  📎 bootstrap-extra-files ✓ — Впроваджує додаткові початкові файли робочого простору під час початкового налаштування агента
  📝 command-logger ✓ — Записує всі події команд у централізований файл аудиту
  💾 session-memory ✓ — Зберігає контекст сеансу в пам’яті після виконання команди /new або /reset
```

## Отримання інформації про хук

```bash
openclaw hooks info <name> [--json]
```

`<name>` — назва або ключ хука (наприклад, `session-memory`). Показує джерело, шляхи до файлу й обробника, домашню сторінку, події та стан кожної вимоги (двійкові файли, змінні середовища, конфігурація, ОС).

## Перевірка відповідності вимогам

```bash
openclaw hooks check [--json]
```

Виводить зведення про кількість готових і неготових хуків; якщо є неготові хуки, перелічує кожен із причиною блокування.

## Увімкнення хука

```bash
openclaw hooks enable <name>
```

Додає або оновлює `hooks.internal.entries.<name>.enabled = true` у конфігурації, а також вмикає головний перемикач `hooks.internal.enabled` (Gateway не завантажує жодного внутрішнього обробника хуків, доки не налаштовано принаймні один). Команда завершується помилкою, якщо хук не існує, керується Plugin або не відповідає вимогам (вимоги не виконано).

Хуки, керовані Plugin, позначаються як `plugin:<id>` у виводі `hooks list`, і їх не можна ввімкнути або вимкнути тут; натомість увімкніть або вимкніть Plugin-власник.

Після ввімкнення перезапустіть Gateway (перезапустіть застосунок у рядку меню macOS або процес Gateway у середовищі розробки), щоб він повторно завантажив хуки.

## Вимкнення хука

```bash
openclaw hooks disable <name>
```

Установлює `hooks.internal.entries.<name>.enabled = false`. Після цього перезапустіть Gateway.

## Встановлення й оновлення пакетів хуків

```bash
openclaw plugins install <package>        # типово npm
openclaw plugins install npm:<package>    # лише npm
openclaw plugins install <package> --pin  # закріпити визначену версію
openclaw plugins install <path>           # локальний каталог або архів
openclaw plugins install -l <path>        # створити посилання на локальний каталог замість копіювання

openclaw plugins update <id>
openclaw plugins update --all
openclaw plugins update --dry-run
```

Пакети хуків установлюються через уніфікований засіб установлення й оновлення плагінів; `openclaw hooks install` / `openclaw hooks update` досі працюють як застарілі псевдоніми, що виводять попередження та передають виконання командам `plugins`.

- Специфікації npm підтримують лише реєстр: назву пакета з необов’язковою точною версією або тегом дистрибутива. Специфікації Git, URL і файлів, а також діапазони semver відхиляються. Залежності встановлюються локально для проєкту з `--ignore-scripts`.
- Специфікації без версії та `@latest` залишаються на стабільному каналі; якщо npm визначає попередню версію, OpenClaw зупиняється та просить явно погодитися на її використання (`@beta`, `@rc` або точна попередня версія).
- Підтримувані архіви: `.zip`, `.tgz`, `.tar.gz`, `.tar`.
- `-l, --link` створює посилання на локальний каталог замість його копіювання (додає його до `hooks.internal.load.extraDirs`); пов’язані пакети хуків є керованими хуками з каталогу, налаштованого оператором, а не хуками робочого простору.
- `--pin` записує встановлення npm як точно визначене `name@version` у `hooks.internal.installs`.
- Під час встановлення пакет копіюється до `~/.openclaw/hooks/<id>`, його хуки вмикаються в `hooks.internal.entries.*`, а запис про встановлення додається до `hooks.internal.installs`.
- Якщо збережений хеш цілісності більше не відповідає отриманому артефакту, OpenClaw попереджає та запитує підтвердження перед продовженням; передайте глобальний параметр `--yes`, щоб пропустити запит (наприклад, у CI).

## Вбудовані хуки

| Хук                   | Події                                             | Що він робить                                                                                                     |
| --------------------- | ------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------- |
| boot-md               | `gateway:startup`                                 | Запускає `BOOT.md` під час запуску Gateway для кожної налаштованої області агента                                 |
| bootstrap-extra-files | `agent:bootstrap`                                 | Впроваджує додаткові початкові файли (наприклад, `AGENTS.md`/`TOOLS.md` монорепозиторію) під час запуску агента    |
| command-logger        | `command`                                         | Записує події команд до `~/.openclaw/logs/commands.log`                                                           |
| compaction-notifier   | `session:compact:before`, `session:compact:after` | Надсилає видимі сповіщення в чаті, коли ущільнення сеансу починається та завершується                              |
| session-memory        | `command:new`, `command:reset`                    | Зберігає контекст сеансу в пам’яті після виконання `/new` або `/reset`                                            |

Увімкніть будь-який вбудований хук за допомогою `openclaw hooks enable <hook-name>`. Повні відомості, ключі конфігурації та типові значення: [Вбудовані хуки](/uk/automation/hooks#bundled-hooks).

### Файл журналу command-logger

```bash
tail -n 20 ~/.openclaw/logs/commands.log        # останні команди
cat ~/.openclaw/logs/commands.log | jq .          # форматований вивід
grep '"action":"new"' ~/.openclaw/logs/commands.log | jq .   # фільтрування за дією
```

## Примітки

- `hooks list --json`, `info --json` і `check --json` записують структурований JSON безпосередньо до стандартного виводу.
