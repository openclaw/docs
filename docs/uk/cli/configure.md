# `openclaw configure`

Інтерактивний запит для налаштування облікових даних, пристроїв і типових параметрів агентів.

<Note>
Розділ **Модель** містить множинний вибір для списку дозволених `agents.defaults.models` (те, що відображається в `/model` і засобі вибору моделі). Варіанти налаштування в межах провайдера додають вибрані моделі до наявного списку дозволених, не замінюючи непов’язаних провайдерів, які вже є в конфігурації. Повторний запуск автентифікації провайдера з налаштування зберігає наявний `agents.defaults.model.primary`. Використовуйте `openclaw models auth login --provider <id> --set-default` або `openclaw models set <model>`, коли ви навмисно хочете змінити типову модель.
</Note>

Коли майстер налаштування запускається з вибору автентифікації провайдера, засоби вибору типової моделі та списку дозволених автоматично віддають перевагу цьому провайдеру. Для парних провайдерів, як-от Volcengine і BytePlus, така сама перевага також відповідає їхнім варіантам плану кодування (`volcengine-plan/*`, `byteplus-plan/*`). Якщо фільтр бажаного провайдера створив би порожній список, майстер налаштування повертається до нефільтрованого каталогу замість показу порожнього засобу вибору.

<Tip>
`openclaw config` без підкоманди відкриває той самий майстер. Використовуйте `openclaw config get|set|unset` для неінтерактивних змін.
</Tip>

Для вебпошуку `openclaw configure --section web` дає змогу вибрати провайдера
та налаштувати його облікові дані. Деякі провайдери також показують специфічні для провайдера
подальші запити:

- **Grok** може запропонувати необов’язкове налаштування `x_search` з тим самим `XAI_API_KEY` і
  дати змогу вибрати модель `x_search`.
- **Kimi** може запитати регіон Moonshot API (`api.moonshot.ai` або
  `api.moonshot.cn`) і типову модель вебпошуку Kimi.

Пов’язано:

- Довідник конфігурації Gateway: [Конфігурація](/uk/gateway/configuration)
- CLI конфігурації: [Конфігурація](/uk/cli/config)

## Параметри

- `--section <section>`: повторюваний фільтр розділу

Доступні розділи:

- `workspace`
- `model`
- `web`
- `gateway`
- `daemon`
- `channels`
- `plugins`
- `skills`
- `health`

Примітки:

- Вибір місця запуску Gateway завжди оновлює `gateway.mode`. Ви можете вибрати "Продовжити" без інших розділів, якщо це все, що вам потрібно.
- Після запису локальної конфігурації майстер налаштування встановлює вибрані завантажувані plugins, коли обраний шлях налаштування цього потребує. Конфігурація віддаленого gateway не встановлює локальні пакети plugin.
- Сервіси, орієнтовані на канали (Slack/Discord/Matrix/Microsoft Teams), під час налаштування запитують списки дозволених каналів/кімнат. Ви можете вводити назви або ID; майстер за можливості перетворює назви на ID.
- Якщо ви запускаєте крок встановлення демона, автентифікація за токеном вимагає токен, а `gateway.auth.token` керується через SecretRef, майстер налаштування перевіряє SecretRef, але не зберігає розв’язані відкриті значення токена в метадані середовища служби supervisor.
- Якщо автентифікація за токеном вимагає токен, а налаштований SecretRef токена не розв’язано, майстер налаштування блокує встановлення демона з практичними вказівками щодо виправлення.
- Якщо налаштовано і `gateway.auth.token`, і `gateway.auth.password`, а `gateway.auth.mode` не задано, майстер налаштування блокує встановлення демона, доки режим не буде задано явно.

## Приклади

```bash
openclaw configure
openclaw configure --section web
openclaw configure --section model --section channels
openclaw configure --section gateway --section daemon
```
