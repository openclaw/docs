# `openclaw configure`

Інтерактивний запит для налаштування облікових даних, пристроїв і стандартних параметрів агента.

<Note>
Розділ **Модель** містить множинний вибір для списку дозволених `agents.defaults.models` (те, що відображається в `/model` і засобі вибору моделі). Варіанти налаштування в межах провайдера об’єднують вибрані моделі з наявним списком дозволених, а не замінюють непов’язаних провайдерів, які вже є в конфігурації. Повторний запуск автентифікації провайдера з configure зберігає наявне значення `agents.defaults.model.primary`. Використовуйте `openclaw models auth login --provider <id> --set-default` або `openclaw models set <model>`, коли ви навмисно хочете змінити стандартну модель.
</Note>

Коли configure запускається з варіанта автентифікації провайдера, засоби вибору стандартної моделі та списку дозволених автоматично надають перевагу цьому провайдеру. Для парних провайдерів, як-от Volcengine і BytePlus, та сама перевага також охоплює їхні варіанти плану кодування (`volcengine-plan/*`, `byteplus-plan/*`). Якщо фільтр бажаного провайдера дав би порожній список, configure повертається до нефільтрованого каталогу замість показу порожнього засобу вибору.

<Tip>
`openclaw config` без підкоманди відкриває той самий майстер. Використовуйте `openclaw config get|set|unset` для неінтерактивного редагування.
</Tip>

Для вебпошуку `openclaw configure --section web` дає змогу вибрати провайдера
і налаштувати його облікові дані. Деякі провайдери також показують специфічні
для провайдера додаткові запити:

- **Grok** може запропонувати необов’язкове налаштування `x_search` з тим самим `XAI_API_KEY` і
  дати змогу вибрати модель `x_search`.
- **Kimi** може запитати регіон Moonshot API (`api.moonshot.ai` або
  `api.moonshot.cn`) і стандартну модель вебпошуку Kimi.

Пов’язано:

- Довідник конфігурації Gateway: [Конфігурація](/uk/gateway/configuration)
- Config CLI: [Config](/uk/cli/config)

## Параметри

- `--section <section>`: повторюваний фільтр розділів

Доступні розділи:

- `workspace`
- `model`
- `web`
- `gateway`
- `daemon`
- `channels`
- `plugins`
- `skills`
- `health`

Примітки:

- Вибір місця, де працює Gateway, завжди оновлює `gateway.mode`. Ви можете вибрати "Продовжити" без інших розділів, якщо це все, що вам потрібно.
- Після записів у локальну конфігурацію configure матеріалізує нові необхідні runtime-залежності вбудованих plugin. Це вузький крок відновлення менеджера пакетів, а не повний запуск `openclaw doctor`. Конфігурація віддаленого gateway не встановлює локальні залежності plugin.
- Сервіси, орієнтовані на канали (Slack/Discord/Matrix/Microsoft Teams), під час налаштування запитують списки дозволених каналів/кімнат. Ви можете вводити назви або ID; майстер за можливості зіставляє назви з ID.
- Якщо ви запускаєте крок встановлення daemon, token auth вимагає токен, а `gateway.auth.token` керується через SecretRef, configure перевіряє SecretRef, але не зберігає розв’язані значення токена у відкритому тексті в метаданих середовища служби supervisor.
- Якщо token auth вимагає токен, а налаштований SecretRef токена не розв’язано, configure блокує встановлення daemon з практичними вказівками щодо виправлення.
- Якщо налаштовано і `gateway.auth.token`, і `gateway.auth.password`, а `gateway.auth.mode` не задано, configure блокує встановлення daemon, доки режим не буде задано явно.

## Приклади

```bash
openclaw configure
openclaw configure --section web
openclaw configure --section model --section channels
openclaw configure --section gateway --section daemon
```
