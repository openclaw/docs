# `openclaw configure`

Інтерактивний запит для цільових змін в наявному налаштуванні: облікові дані, пристрої, стандартні параметри агентів, Gateway, канали, Plugin, Skills і перевірки справності.

Використовуйте `openclaw onboard` або `openclaw setup` для повного керованого першого запуску, `openclaw setup --baseline` лише для базової конфігурації/робочого простору, і `openclaw channels add`, коли потрібно налаштувати лише обліковий запис каналу.

<Note>
Розділ **Модель** містить множинний вибір для списку дозволених `agents.defaults.models` (те, що відображається в `/model` і засобі вибору моделі). Варіанти налаштування в межах постачальника додають вибрані моделі до наявного списку дозволених, а не замінюють непов’язаних постачальників, які вже є в конфігурації.

Повторний запуск автентифікації постачальника з configure зберігає наявне значення `agents.defaults.model.primary`, навіть коли крок автентифікації постачальника повертає патч конфігурації з власною рекомендованою стандартною моделлю. Це означає, що додавання або повторна автентифікація xAI, OpenRouter чи іншого постачальника має зробити нову модель доступною, не замінюючи вашу поточну основну модель. Використовуйте `openclaw models auth login --provider <id> --set-default` або `openclaw models set <model>`, коли навмисно хочете змінити стандартну модель.
</Note>

Коли configure запускається з вибору автентифікації постачальника, засоби вибору стандартної моделі та списку дозволених автоматично надають перевагу цьому постачальнику. Для парних постачальників, як-от Volcengine і BytePlus, ця сама перевага також відповідає їхнім варіантам плану кодування (`volcengine-plan/*`, `byteplus-plan/*`). Якщо фільтр бажаного постачальника дав би порожній список, configure повертається до нефільтрованого каталогу замість показу порожнього засобу вибору.

<Tip>
`openclaw config` без підкоманди відкриває той самий майстер. Використовуйте `openclaw config get|set|unset` для неінтерактивних змін.
</Tip>

Для вебпошуку `openclaw configure --section web` дає змогу вибрати постачальника
та налаштувати його облікові дані. Деякі постачальники також показують специфічні
для постачальника подальші запити:

- **Grok** може запропонувати необов’язкове налаштування `x_search` з тим самим профілем OAuth xAI
  або ключем API й дати змогу вибрати модель `x_search`.
- **Kimi** може запитати регіон API Moonshot (`api.moonshot.ai` чи
  `api.moonshot.cn`) і стандартну модель вебпошуку Kimi.

## Параметри

- `--section <section>`: повторюваний фільтр розділу

Доступні розділи:

- `workspace`
- `model`
- `web`
- `gateway`
- `daemon`
- `channels`
- `plugins`
- `skills`
- `health`

Примітки:

- Повний майстер і розділи, пов’язані з Gateway, запитують, де працює Gateway, і оновлюють `gateway.mode`. Фільтри розділів, які не містять `gateway`, `daemon` або `health`, переходять безпосередньо до запитаного налаштування.
- Після локальних записів конфігурації configure встановлює вибрані завантажувані Plugin, коли цього вимагає вибраний шлях налаштування. Віддалена конфігурація Gateway не встановлює локальні пакети Plugin.
- Сервіси, орієнтовані на канали (Slack/Discord/Matrix/Microsoft Teams), під час налаштування запитують списки дозволених каналів/кімнат. Можна ввести назви або ID; майстер за можливості перетворює назви на ID.
- Якщо ви запускаєте крок встановлення демона, автентифікація за токеном вимагає токен, а `gateway.auth.token` керується SecretRef, configure перевіряє SecretRef, але не зберігає розв’язані значення токена відкритим текстом у метаданих середовища служби супервізора.
- Якщо автентифікація за токеном вимагає токен, а налаштований SecretRef токена не розв’язано, configure блокує встановлення демона й надає практичні вказівки з виправлення.
- Якщо налаштовано і `gateway.auth.token`, і `gateway.auth.password`, а `gateway.auth.mode` не задано, configure блокує встановлення демона, доки режим не буде задано явно.

## Приклади

```bash
openclaw configure
openclaw configure --section web
openclaw configure --section model --section channels
openclaw configure --section gateway --section daemon
```
