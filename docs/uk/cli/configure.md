# `openclaw configure`

Інтерактивний запит для налаштування облікових даних, пристроїв і типових параметрів агента.

<Note>
Розділ **Модель** містить множинний вибір для списку дозволених `agents.defaults.models` (те, що відображається в `/model` і засобі вибору моделі). Варіанти налаштування в межах провайдера об’єднують вибрані моделі з наявним списком дозволених, а не замінюють непов’язаних провайдерів, які вже є в конфігурації.

Повторний запуск автентифікації провайдера з configure зберігає наявне значення `agents.defaults.model.primary`, навіть коли крок автентифікації провайдера повертає патч конфігурації з власною рекомендованою типовою моделлю. Це означає, що додавання або повторна автентифікація xAI, OpenRouter чи іншого провайдера має зробити нову модель доступною, не перебираючи керування у вашої поточної основної моделі. Використовуйте `openclaw models auth login --provider <id> --set-default` або `openclaw models set <model>`, коли ви навмисно хочете змінити типову модель.
</Note>

Коли configure запускається з варіанта автентифікації провайдера, засоби вибору типової моделі та списку дозволених автоматично надають перевагу цьому провайдеру. Для парних провайдерів, таких як Volcengine і BytePlus, та сама перевага також відповідає їхнім варіантам планів для кодування (`volcengine-plan/*`, `byteplus-plan/*`). Якщо фільтр бажаного провайдера дав би порожній список, configure повертається до нефільтрованого каталогу замість показу порожнього засобу вибору.

<Tip>
`openclaw config` без підкоманди відкриває той самий майстер. Використовуйте `openclaw config get|set|unset` для неінтерактивних змін.
</Tip>

Для вебпошуку `openclaw configure --section web` дає змогу вибрати провайдера
та налаштувати його облікові дані. Деякі провайдери також показують специфічні
для провайдера додаткові запити:

- **Grok** може запропонувати необов’язкове налаштування `x_search` з тим самим `XAI_API_KEY` і
  дати змогу вибрати модель `x_search`.
- **Kimi** може запитати регіон Moonshot API (`api.moonshot.ai` або
  `api.moonshot.cn`) і типову модель вебпошуку Kimi.

## Параметри

- `--section <section>`: повторюваний фільтр розділів

Доступні розділи:

- `workspace`
- `model`
- `web`
- `gateway`
- `daemon`
- `channels`
- `plugins`
- `skills`
- `health`

Примітки:

- Вибір місця запуску Gateway завжди оновлює `gateway.mode`. Ви можете вибрати «Продовжити» без інших розділів, якщо це все, що вам потрібно.
- Після запису локальної конфігурації configure встановлює вибрані завантажувані plugins, коли вибраний шлях налаштування їх потребує. Конфігурація віддаленого Gateway не встановлює локальні пакети plugin.
- Служби, орієнтовані на канали (Slack/Discord/Matrix/Microsoft Teams), під час налаштування запитують списки дозволених каналів/кімнат. Можна вводити назви або ідентифікатори; майстер за можливості зіставляє назви з ідентифікаторами.
- Якщо ви запускаєте крок установлення демона, автентифікація за токеном вимагає токен, а `gateway.auth.token` керується через SecretRef, configure перевіряє SecretRef, але не зберігає розв’язані значення токена у відкритому тексті в метаданих середовища служби supervisor.
- Якщо автентифікація за токеном вимагає токен, а налаштований SecretRef токена не розв’язано, configure блокує встановлення демона з практичними вказівками щодо виправлення.
- Якщо налаштовано і `gateway.auth.token`, і `gateway.auth.password`, а `gateway.auth.mode` не задано, configure блокує встановлення демона, доки режим не буде задано явно.

## Приклади

```bash
openclaw configure
openclaw configure --section web
openclaw configure --section model --section channels
openclaw configure --section gateway --section daemon
```
