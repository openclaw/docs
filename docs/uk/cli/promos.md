# `openclaw promos`

Знаходьте й активуйте рекламні пропозиції моделей, опубліковані на ClawHub. Активація
пропозиції налаштовує постачальника (автентифікацію та Plugin, коли потрібно) і реєструє
моделі пропозиції — без повторного проходження початкового налаштування та без зміни
моделі за замовчуванням, якщо ви цього не вкажете.

Пов’язані матеріали:

- Модель за замовчуванням і резервні моделі: [Моделі](/uk/cli/models)
- Налаштування автентифікації постачальника: [Початок роботи](/uk/start/getting-started)

## Команди

```bash
openclaw promos list
openclaw promos claim <slug>
openclaw promos claim <slug> --api-key <key> --set-default
```

## `openclaw promos list`

Виводить активні на цей момент пропозиції, їхні моделі, рекомендовану
модель за замовчуванням, час до завершення та точну команду активації. `--json` виводить необроблені
дані.

## `openclaw promos claim <slug>`

Активує чинну пропозицію:

1. Отримує пропозицію з ClawHub і перевіряє, чи триває період її дії.
2. Перевіряє постачальника пропозиції, спосіб автентифікації та заявлені пакети Plugin
   на сумісність із вашою встановленою версією OpenClaw. Невідомі ідентифікатори або невідповідності пакетів
   відхиляються — пропозиція ніколи не змусить CLI виконати те, чого він іще
   не вміє.
3. Повторно використовує наявні облікові дані постачальника, якщо вони у вас є. Інакше
   запускає звичайний процес автентифікації постачальника (спочатку виводячи URL-адресу реєстрації
   з пропозиції для отримання безплатного ключа). `--api-key <key>` завершує автентифікацію за ключем API без
   запитів, відповідно до неінтерактивних прапорців `openclaw onboard`; щоб не вказувати
   ключ у командному рядку, натомість експортуйте змінну середовища постачальника
   (наприклад, `OPENROUTER_API_KEY`) — наявні облікові дані в середовищі
   виявляються автоматично, тож прапорець не потрібен.
4. Реєструє моделі пропозиції з їхніми псевдонімами. Наявні псевдоніми
   ніколи не перезаписуються.
5. Пропонує встановити рекомендовану модель пропозиції як модель за замовчуванням —
   `--set-default` пропускає запитання; в іншому разі ваші налаштування за замовчуванням
   не змінюються.

Коли період дії пропозиції завершується, постачальник припиняє надавати безплатні моделі;
ваші налаштування та облікові дані залишаються незмінними. Будь-коли перемкніться назад за допомогою
`openclaw models set <model>`.

## Пасивне виявлення в `models list`

`openclaw models list` також показує пропозиції без прямого звернення
до ClawHub:

- Чинні пропозиції, моделі з яких ви ще не налаштували, відображаються в групі
  «Доступно за пропозицією» під таблицею, кожна зі своєю командою
  активації.
- Моделі, зареєстровані через `promos claim`, мають позначку `promo`, яка
  змінюється на `promo ended` після завершення періоду дії пропозиції.
- Коли нову пропозицію виявлено вперше, одноразове сповіщення вказує на
  `openclaw promos list`. Пропозиції, які ви вже переглянули або активували, більше
  не оголошуються.

Для цього використовується локально кешована копія розміщеного на ClawHub каналу пропозицій
(зазвичай вона оновлюється раз на день за допомогою умовного запиту або раніше, коли
термін дії кешованого знімка спливає; помилки оновлення непомітно пропускаються). Оновлення
застарілих даних очікується щонайбільше 2,5 секунди й ніколи не порушує виведення списку. Виведення `--json` і
`--plain` залишається придатним для машинної обробки: без розділів пропозицій і сповіщень.
Під час активації завжди виконується повторна перевірка через актуальний API ClawHub, тому достроково відкликану
пропозицію буде відхилено, навіть якщо кешована копія все ще її показує.
