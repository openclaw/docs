# `openclaw status`

Діагностика для каналів і сесій.

```bash
openclaw status
openclaw status --all
openclaw status --deep
openclaw status --usage
```

Примітки:

- `--deep` запускає живі перевірки (WhatsApp Web + Telegram + Discord + Slack + Signal).
- `--usage` виводить нормалізовані вікна використання провайдера як `X% left`.
- Вивід стану сесії тепер окремо показує `Runtime:` і `Runner:`. `Runtime` — це шлях виконання та стан sandbox (`direct`, `docker/*`), тоді як `Runner` показує, чи використовує сесія вбудований Pi, провайдера на основі CLI або бекенд ACP harness, наприклад `codex (acp/acpx)`.
- Необроблені поля `usage_percent` / `usagePercent` у MiniMax означають залишкову квоту, тому OpenClaw інвертує їх перед відображенням; поля на основі підрахунку мають пріоритет, коли вони присутні. Відповіді `model_remains` віддають перевагу запису chat-моделі, за потреби виводять мітку вікна з часових міток і включають назву моделі в мітку плану.
- Коли поточний знімок сесії є неповним, `/status` може дозаповнити лічильники токенів і кешу з найновішого журналу використання transcript. Наявні ненульові live-значення, як і раніше, мають пріоритет над запасними значеннями з transcript.
- Запасний варіант transcript також може відновити мітку активної runtime-моделі, коли в live-записі сесії її немає. Якщо ця модель у transcript відрізняється від вибраної моделі, status визначає контекстне вікно за відновленою runtime-моделлю, а не за вибраною.
- Для обліку розміру запиту запасний варіант transcript віддає перевагу більшому загальному значенню, орієнтованому на запит, коли метадані сесії відсутні або менші, щоб сесії custom provider не зводилися до відображення `0` токенів.
- Вивід включає сховища сесій для кожного агента, коли налаштовано кілька агентів.
- Огляд включає статус встановлення/виконання Gateway і служби хоста node, коли він доступний.
- Огляд включає канал оновлень і git SHA (для source checkout).
- Інформація про оновлення відображається в Overview; якщо оновлення доступне, status виводить підказку запустити `openclaw update` (див. [Оновлення](/uk/install/updating)).
- Поверхні status лише для читання (`status`, `status --json`, `status --all`) за можливості визначають підтримувані SecretRef для цільових шляхів конфігурації.
- Якщо підтримуваний SecretRef каналу налаштовано, але він недоступний у поточному шляху команди, status залишається лише для читання і повідомляє про деградований вивід замість аварійного завершення. У виводі для людини показуються попередження на кшталт “configured token unavailable in this command path”, а JSON-вивід включає `secretDiagnostics`.
- Коли локальне для команди визначення SecretRef успішне, status віддає перевагу визначеному знімку й прибирає тимчасові маркери каналу “secret unavailable” з фінального виводу.
- `status --all` включає рядок огляду Secrets і розділ діагностики, який підсумовує діагностику секретів (усічено для читабельності), не зупиняючи генерацію звіту.
