# `openclaw setup`

`openclaw setup` — це точка входу системного агента. У налаштованій системі команда
`openclaw setup` без аргументів відкриває інтерактивний чат OpenClaw. У новій системі вона
переходить до керованого початкового налаштування. Використовуйте `-m`/`--message` для одного запиту або
`--baseline`, щоб ініціалізувати папки конфігурації/робочого простору без майстра.

Порядок маршрутизації:

1. Будь-який параметр початкового налаштування (`--wizard`, `--baseline`, робочий простір, скидання,
   неінтерактивний режим, процес, режим, Gateway, фоновий процес, пропуск, імпорт, віддалений доступ або параметри
   автентифікації) запускає початкове налаштування так само, як `openclaw onboard`.
2. `-m`/`--message` або `--yes` запускає системного агента.
3. Якщо параметр маршрутизації не вказано, у налаштованій інтерактивній системі відкривається OpenClaw. У
   новій системі запускається початкове налаштування. У налаштованій системі `--json` виводить
   огляд системи навіть без TTY; параметр початкового налаштування зберігає
   підсумок початкового налаштування у форматі JSON.

У керованому режимі `--workspace <dir>` — це робочий простір, запропонований OpenClaw;
він зберігається лише після схвалення цієї пропозиції. Базове, класичне та
неінтерактивне налаштування зберігають наданий робочий простір у межах свого звичайного процесу.

Кероване виявлення інференсу виконується на хості Gateway у macOS або Linux. CLI
та застосунок macOS викликають той самий засіб виявлення, яким керує Gateway; він перевіряє налаштовані
моделі, підтримувані входи через CLI, змінні середовища з ключами API та вже
встановлені моделі Ollama або LM Studio. Під час цього автоматичного проходу локальні моделі ніколи не
завантажуються; вибраний кандидат має відповісти на реальний запит автодоповнення, перш ніж його
конфігурацію постачальника та моделі буде збережено.

`setup` приймає ті самі прапорці початкового налаштування, що й `openclaw onboard`, зокрема
автентифікації (`--auth-choice`, `--token`, прапорці ключів постачальників), Gateway
(`--gateway-port`, `--gateway-bind`, `--gateway-auth`, `--install-daemon`),
Tailscale (`--tailscale`), скидання (`--reset`, `--reset-scope`), процесу
(`--flow quickstart|advanced|manual|import`) і пропуску
(`--skip-channels`, `--skip-skills`, `--skip-bootstrap`, `--skip-search`,
`--skip-health`, `--skip-ui`, `--skip-hooks`). Повний довідник прапорців і
приклади неінтерактивного використання наведено в розділах [Початкове налаштування](/uk/cli/onboard) та
[Автоматизація CLI](/uk/start/wizard-cli-automation). `openclaw onboard --modern` залишається сумісною
точкою входу для того самого асистента OpenClaw із перевіркою можливості інференсу.

<Note>
`openclaw setup` призначено для встановлень зі змінюваною конфігурацією. У режимі Nix (`OPENCLAW_NIX_MODE=1`) OpenClaw відмовляється записувати налаштування, оскільки файлом конфігурації керує Nix. Скористайтеся офіційним [коротким посібником із nix-openclaw](https://github.com/openclaw/nix-openclaw#quick-start) або еквівалентною конфігурацією джерела для іншого пакета Nix.
</Note>

## Параметри

| Прапорець                  | Опис                                                                                                  |
| -------------------------- | ----------------------------------------------------------------------------------------------------- |
| `-m, --message <text>`     | Виконати один запит OpenClaw.                                                                         |
| `--yes`                    | Дозволити постійний запис конфігурації для одного запиту `--message`.                          |
| `--workspace <dir>`        | Пропозиція робочого простору в керованому режимі; безпосередньо зберігається під час базового, класичного та неінтерактивного налаштування. |
| `--baseline`               | Створити базові папки конфігурації/робочого простору/сеансів без початкового налаштування.             |
| `--wizard`                 | Примусово запустити інтерактивне початкове налаштування.                                               |
| `--non-interactive`        | Запустити початкове налаштування без запитів.                                                         |
| `--accept-risk`            | Підтвердити ризик доступу агента до всієї системи; обов’язково з `--non-interactive`.                   |
| `--mode <mode>`            | Режим початкового налаштування: `local` або `remote`.                            |
| `--flow <flow>`            | Процес початкового налаштування: `quickstart`, `advanced`, `manual` або `import`. |
| `--reset`                  | Скинути конфігурацію, облікові дані та сеанси перед початковим налаштуванням (робочий простір — лише з `--reset-scope full`). |
| `--reset-scope <scope>`    | Обсяг скидання: `config`, `config+creds+sessions` або `full`.                         |
| `--import-from <provider>` | Постачальник міграції, якого слід запустити під час початкового налаштування.                           |
| `--import-source <path>`   | Домашня папка вихідного агента для `--import-from`.                                                |
| `--import-secrets`         | Імпортувати підтримувані секрети під час міграції в межах початкового налаштування.                    |
| `--remote-url <url>`       | URL-адреса WebSocket віддаленого Gateway.                                                             |
| `--remote-token <token>`   | Токен віддаленого Gateway (необов’язково).                                                            |
| `--json`                   | Налаштована система: огляд OpenClaw. Маршрут початкового налаштування: підсумок початкового налаштування. |

`--classic` і `--non-interactive` взаємовиключні: класичний режим відкриває
майстер із запитами, а неінтерактивне налаштування використовує шлях автоматизації.

### Базовий режим

`openclaw setup --baseline` зберігає попередню поведінку лише базового налаштування: він
створює папки конфігурації, робочого простору та сеансів, а потім завершує роботу, не
запускаючи початкового налаштування.

## Приклади

```bash
openclaw setup
openclaw setup -m "status"
openclaw setup -m "restart gateway" --yes
openclaw setup --json
openclaw setup --wizard
openclaw setup --baseline
openclaw setup --workspace ~/.openclaw/workspace
openclaw setup --import-from hermes --import-source ~/.hermes
openclaw setup --non-interactive --accept-risk --mode remote --remote-url wss://gateway-host:18789 --remote-token <token>
```

## Примітки

- Після базового налаштування запустіть `openclaw onboard` для повного керованого процесу, `openclaw configure` для цільових змін або `openclaw channels add`, щоб додати облікові записи каналів.
- Якщо виявлено стан Hermes, інтерактивне початкове налаштування може автоматично запропонувати міграцію. Для імпорту під час початкового налаштування потрібна нова конфігурація; скористайтеся розділом [Міграція](/uk/cli/migrate) для планів пробного запуску, резервних копій і режиму перезапису поза початковим налаштуванням.
