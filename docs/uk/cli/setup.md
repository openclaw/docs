# `openclaw setup`

Запустіть повний потік початкового налаштування CLI. `openclaw setup` — це псевдонім для `openclaw onboard`; використовуйте `--baseline`, коли потрібно лише ініціалізувати теки конфігурації/робочої області без майстра.

<Note>
`openclaw setup` призначено для встановлень зі змінюваною конфігурацією. У режимі Nix (`OPENCLAW_NIX_MODE=1`) OpenClaw відмовляється виконувати записи налаштування, оскільки файл конфігурації керується Nix. Використовуйте офіційний [Короткий старт nix-openclaw](https://github.com/openclaw/nix-openclaw#quick-start) або еквівалентну конфігурацію з джерела для іншого пакета Nix.
</Note>

## Параметри

| Прапорець                  | Опис                                                                                                              |
| -------------------------- | ----------------------------------------------------------------------------------------------------------------- |
| `--workspace <dir>`        | Каталог робочої області агента (типово `~/.openclaw/workspace`; зберігається як `agents.defaults.workspace`).    |
| `--baseline`               | Створити базові теки конфігурації/робочої області/сеансу без початкового налаштування.                           |
| `--wizard`                 | Приймається для сумісності; setup типово запускає початкове налаштування.                                         |
| `--non-interactive`        | Запустити початкове налаштування без запитів.                                                                     |
| `--accept-risk`            | Підтвердити ризик доступу агента до всієї системи; обов’язково з `--non-interactive`.                            |
| `--mode <mode>`            | Режим початкового налаштування: `local` або `remote`.                                                             |
| `--import-from <provider>` | Постачальник міграції, який потрібно запустити під час початкового налаштування.                                  |
| `--import-source <path>`   | Домашній каталог агента-джерела для `--import-from`.                                                              |
| `--import-secrets`         | Імпортувати підтримувані секрети під час міграції в межах початкового налаштування.                               |
| `--remote-url <url>`       | URL WebSocket віддаленого Gateway.                                                                                |
| `--remote-token <token>`   | Токен віддаленого Gateway (необов’язково).                                                                        |

### Базовий режим

`openclaw setup --baseline` зберігає старішу поведінку лише для базового налаштування: створює каталоги конфігурації, робочої області та сеансів, а потім завершує роботу без запуску початкового налаштування.

## Приклади

```bash
openclaw setup
openclaw setup --baseline
openclaw setup --workspace ~/.openclaw/workspace
openclaw setup --import-from hermes --import-source ~/.hermes
openclaw setup --non-interactive --accept-risk --mode remote --remote-url wss://gateway-host:18789 --remote-token <token>
```

## Примітки

- Звичайна команда `openclaw setup` запускає той самий керований шлях, що й `openclaw onboard`.
- Після базового налаштування запустіть `openclaw setup` або `openclaw onboard` для повного керованого шляху, `openclaw configure` для цільових змін або `openclaw channels add`, щоб додати облікові записи каналів.
- Якщо виявлено стан Hermes, інтерактивне початкове налаштування може автоматично запропонувати міграцію. Імпорт у межах початкового налаштування потребує свіжого налаштування; використовуйте [Міграція](/uk/cli/migrate) для планів пробного запуску, резервних копій і режиму перезапису поза початковим налаштуванням.
