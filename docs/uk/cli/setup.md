# `openclaw setup`

`openclaw setup` запускає той самий керований процес початкового налаштування, що й `openclaw onboard`:
спочатку він перевіряє та зберігає налаштування інференсу, а потім запускає Crestodian для налаштування
робочого простору, Gateway, каналів, Skills і перевірки стану. Використовуйте `--baseline`, коли потрібно
лише ініціалізувати конфігурацію та каталоги робочого простору без майстра.

У керованому режимі `--workspace <dir>` — це робочий простір, запропонований Crestodian;
він зберігається лише після схвалення цієї пропозиції. Базове, класичне та
неінтерактивне налаштування зберігають указаний робочий простір у межах свого звичайного процесу.

`setup` приймає ті самі прапорці початкового налаштування, що й `openclaw onboard`, зокрема
для автентифікації (`--auth-choice`, `--token`, прапорці ключів постачальників), Gateway
(`--gateway-port`, `--gateway-bind`, `--gateway-auth`, `--install-daemon`),
Tailscale (`--tailscale`), скидання (`--reset`, `--reset-scope`), сценарію
(`--flow quickstart|advanced|manual|import`) і пропуску кроків
(`--skip-channels`, `--skip-skills`, `--skip-bootstrap`, `--skip-search`,
`--skip-health`, `--skip-ui`, `--skip-hooks`). Повний довідник прапорців і
приклади неінтерактивного використання див. у розділах [Початкове налаштування](/uk/cli/onboard) та
[Автоматизація CLI](/uk/start/wizard-cli-automation). `openclaw onboard --modern` — це псевдонім
сумісності для помічника Crestodian із перевіркою інференсу, якому немає відповідника в `setup`.

<Note>
`openclaw setup` призначено для встановлень зі змінюваною конфігурацією. У режимі Nix (`OPENCLAW_NIX_MODE=1`) OpenClaw відмовляється записувати зміни налаштування, оскільки файлом конфігурації керує Nix. Скористайтеся офіційним [коротким посібником із nix-openclaw](https://github.com/openclaw/nix-openclaw#quick-start) або еквівалентною конфігурацією з вихідного коду для іншого пакета Nix.
</Note>

## Параметри

| Прапорець                  | Опис                                                                                                  |
| -------------------------- | ----------------------------------------------------------------------------------------------------- |
| `--workspace <dir>`        | Пропозиція робочого простору в керованому режимі; зберігається безпосередньо під час базового, класичного й неінтерактивного налаштування. |
| `--baseline`               | Створити базову конфігурацію та каталоги робочого простору й сеансів без початкового налаштування.     |
| `--wizard`                 | Приймається для сумісності; за замовчуванням setup запускає початкове налаштування.                    |
| `--non-interactive`        | Запустити початкове налаштування без запитів.                                                          |
| `--accept-risk`            | Підтвердити ризик доступу агента до всієї системи; обов’язково з `--non-interactive`.                  |
| `--mode <mode>`            | Режим початкового налаштування: `local` або `remote`.                                                  |
| `--flow <flow>`            | Сценарій початкового налаштування: `quickstart`, `advanced`, `manual` або `import`.                    |
| `--reset`                  | Скинути конфігурацію, облікові дані та сеанси перед початковим налаштуванням (робочий простір — лише з `--reset-scope full`). |
| `--reset-scope <scope>`    | Обсяг скидання: `config`, `config+creds+sessions` або `full`.                                         |
| `--import-from <provider>` | Постачальник міграції, яку слід виконати під час початкового налаштування.                             |
| `--import-source <path>`   | Домашній каталог вихідного агента для `--import-from`.                                                |
| `--import-secrets`         | Імпортувати підтримувані секрети під час міграції в межах початкового налаштування.                    |
| `--remote-url <url>`       | URL WebSocket віддаленого Gateway.                                                                    |
| `--remote-token <token>`   | Токен віддаленого Gateway (необов’язково).                                                             |
| `--json`                   | Вивести підсумок у форматі JSON.                                                                      |

`--classic` і `--non-interactive` взаємовиключні: класичний режим відкриває
інтерактивний майстер, а неінтерактивне налаштування використовує шлях автоматизації.

### Базовий режим

`openclaw setup --baseline` зберігає попередню поведінку лише базового налаштування: команда
створює каталоги конфігурації, робочого простору та сеансів, а потім завершує роботу без
запуску початкового налаштування.

## Приклади

```bash
openclaw setup
openclaw setup --baseline
openclaw setup --workspace ~/.openclaw/workspace
openclaw setup --import-from hermes --import-source ~/.hermes
openclaw setup --non-interactive --accept-risk --mode remote --remote-url wss://gateway-host:18789 --remote-token <token>
```

## Примітки

- Після базового налаштування запустіть `openclaw setup` або `openclaw onboard` для повного керованого процесу, `openclaw configure` для цільових змін або `openclaw channels add`, щоб додати облікові записи каналів.
- Якщо виявлено стан Hermes, інтерактивне початкове налаштування може автоматично запропонувати міграцію. Імпорт під час початкового налаштування потребує чистого налаштування; для планів пробного запуску, резервних копій і режиму перезапису поза початковим налаштуванням використовуйте [Міграцію](/uk/cli/migrate).
