# `openclaw voicecall`

`voicecall` — це команда, яку надає плагін. Вона з’являється лише тоді, коли плагін голосових викликів установлено й увімкнено.

Коли Gateway працює, операційні команди (`call`, `start`, `continue`, `speak`, `dtmf`, `end`, `status`) спрямовуються до середовища виконання голосових викликів цього Gateway. Якщо жоден Gateway недоступний, вони використовують автономне середовище виконання CLI.

## Підкоманди

```bash
openclaw voicecall setup    [--json]
openclaw voicecall smoke    [-t <phone>] [--message <text>] [--mode <m>] [--yes] [--json]
openclaw voicecall call     -m <text> [-t <phone>] [--mode <m>]
openclaw voicecall start    --to <phone> [--message <text>] [--mode <m>]
openclaw voicecall continue --call-id <id> --message <text>
openclaw voicecall speak    --call-id <id> --message <text>
openclaw voicecall dtmf     --call-id <id> --digits <digits>
openclaw voicecall end      --call-id <id>
openclaw voicecall status   [--call-id <id>] [--json]
openclaw voicecall tail     [--file <path>] [--since <n>] [--poll <ms>]
openclaw voicecall latency  [--file <path>] [--last <n>]
openclaw voicecall expose   [--mode <m>] [--path <p>] [--port <port>] [--serve-path <p>]
```

| Підкоманда | Опис                                                                 |
| ---------- | -------------------------------------------------------------------- |
| `setup`    | Показує перевірки готовності провайдера та Webhook.                   |
| `smoke`    | Виконує перевірки готовності; здійснює реальний тестовий виклик лише з `--yes`. |
| `call`     | Ініціює вихідний голосовий виклик.                                    |
| `start`    | Псевдонім для `call` з обов’язковим `--to` і необов’язковим `--message`. |
| `continue` | Озвучує повідомлення та очікує наступної відповіді.                   |
| `speak`    | Озвучує повідомлення, не очікуючи відповіді.                          |
| `dtmf`     | Надсилає цифри DTMF в активний виклик.                                |
| `end`      | Завершує активний виклик.                                             |
| `status`   | Перевіряє активні виклики (або один за `--call-id`).                  |
| `tail`     | Відстежує `calls.jsonl` (корисно під час тестування провайдера).      |
| `latency`  | Підсумовує метрики затримки реплік із `calls.jsonl`.                  |
| `expose`   | Перемикає Tailscale Serve/Funnel для кінцевої точки Webhook.          |

## Налаштування та димове тестування

### `setup`

За замовчуванням виводить зрозумілі для людини результати перевірок готовності. Передайте `--json` для використання у сценаріях.

```bash
openclaw voicecall setup
openclaw voicecall setup --json
```

### `smoke`

Виконує ті самі перевірки готовності. Здійснює реальний телефонний виклик лише тоді, коли одночасно вказано `--to` і `--yes`.

| Прапорець          | Значення за замовчуванням          | Опис                                              |
| ------------------ | ---------------------------------- | ------------------------------------------------- |
| `-t, --to <phone>` | (немає)                            | Номер телефону для реального димового тестування. |
| `--message <text>` | `OpenClaw voice call smoke test.`  | Повідомлення для озвучення під час тестового виклику. |
| `--mode <mode>`    | `notify`                           | Режим виклику: `notify` або `conversation`.       |
| `--yes`            | `false`                            | Фактично здійснити реальний вихідний виклик.       |
| `--json`           | `false`                            | Вивести машинозчитуваний JSON.                    |

```bash
openclaw voicecall smoke
openclaw voicecall smoke --to "+15555550123"        # пробний запуск
openclaw voicecall smoke --to "+15555550123" --yes  # реальний виклик зі сповіщенням
```

<Note>
Для зовнішніх провайдерів (`plivo`, `telnyx`, `twilio`) командам `setup` і `smoke` потрібна загальнодоступна URL-адреса Webhook із `publicUrl`, тунелю або доступу через Tailscale. Резервний варіант із local loopback або приватним Serve відхиляється, оскільки оператори зв’язку не можуть до нього підключитися.
</Note>

## Життєвий цикл виклику

### `call`

Ініціює вихідний голосовий виклик.

| Прапорець              | Обов’язковий | Значення за замовчуванням | Опис                                                                       |
| ---------------------- | ------------ | ------------------------- | -------------------------------------------------------------------------- |
| `-m, --message <text>` | так          | (немає)                   | Повідомлення для озвучення після встановлення з’єднання.                   |
| `-t, --to <phone>`     | ні           | `toNumber` у конфігурації | Номер телефону у форматі E.164, на який потрібно зателефонувати.           |
| `--mode <mode>`        | ні           | `conversation`            | Режим виклику: `notify` (завершити після повідомлення) або `conversation` (залишити з’єднання відкритим). |

```bash
openclaw voicecall call --to "+15555550123" --message "Hello"
openclaw voicecall call -m "Heads up" --mode notify
```

### `start`

Псевдонім для `call` з іншим набором прапорців за замовчуванням.

| Прапорець          | Обов’язковий | Значення за замовчуванням | Опис                                                     |
| ------------------ | ------------ | ------------------------- | -------------------------------------------------------- |
| `--to <phone>`     | так          | (немає)                   | Номер телефону, на який потрібно зателефонувати.         |
| `--message <text>` | ні           | (немає)                   | Повідомлення для озвучення після встановлення з’єднання. |
| `--mode <mode>`    | ні           | `conversation`            | Режим виклику: `notify` або `conversation`.              |

### `continue`

Озвучує повідомлення та очікує відповіді.

| Прапорець          | Обов’язковий | Опис                         |
| ------------------ | ------------ | ---------------------------- |
| `--call-id <id>`   | так          | Ідентифікатор виклику.       |
| `--message <text>` | так          | Повідомлення для озвучення.  |

### `speak`

Озвучує повідомлення, не очікуючи відповіді.

| Прапорець          | Обов’язковий | Опис                         |
| ------------------ | ------------ | ---------------------------- |
| `--call-id <id>`   | так          | Ідентифікатор виклику.       |
| `--message <text>` | так          | Повідомлення для озвучення.  |

### `dtmf`

Надсилає цифри DTMF в активний виклик.

| Прапорець           | Обов’язковий | Опис                                                  |
| ------------------- | ------------ | ----------------------------------------------------- |
| `--call-id <id>`    | так          | Ідентифікатор виклику.                                |
| `--digits <digits>` | так          | Цифри DTMF (наприклад, `ww123456#` для пауз очікування). |

### `end`

Завершує активний виклик.

| Прапорець        | Обов’язковий | Опис                   |
| ---------------- | ------------ | ---------------------- |
| `--call-id <id>` | так          | Ідентифікатор виклику. |

### `status`

Перевіряє активні виклики.

| Прапорець        | Значення за замовчуванням | Опис                                  |
| ---------------- | ------------------------- | ------------------------------------- |
| `--call-id <id>` | (немає)                   | Обмежити виведення одним викликом.    |
| `--json`         | `false`                   | Вивести машинозчитуваний JSON.        |

```bash
openclaw voicecall status
openclaw voicecall status --json
openclaw voicecall status --call-id <id>
```

## Журнали та метрики

### `tail`

Відстежує журнал голосових викликів у форматі JSONL. Під час запуску виводить останні `--since` рядків, а потім показує нові рядки в міру їх запису.

| Прапорець       | Значення за замовчуванням       | Опис                                      |
| --------------- | -------------------------------- | ----------------------------------------- |
| `--file <path>` | визначається зі сховища плагіна  | Шлях до `calls.jsonl`.                    |
| `--since <n>`   | `25`                             | Кількість рядків перед відстеженням.      |
| `--poll <ms>`   | `250` (мінімум 50)               | Інтервал опитування в мілісекундах.       |

### `latency`

Підсумовує метрики затримки реплік і очікування прослуховування з `calls.jsonl`. Виводить JSON із підсумками `recordsScanned`, `turnLatency` і `listenWait`.

| Прапорець       | Значення за замовчуванням       | Опис                                          |
| --------------- | -------------------------------- | --------------------------------------------- |
| `--file <path>` | визначається зі сховища плагіна  | Шлях до `calls.jsonl`.                        |
| `--last <n>`    | `200` (мінімум 1)                | Кількість останніх записів для аналізу.       |

## Надання доступу до Webhook

### `expose`

Вмикає, вимикає або змінює конфігурацію Tailscale Serve/Funnel для голосового Webhook.

| Прапорець             | Значення за замовчуванням                     | Опис                                              |
| --------------------- | --------------------------------------------- | ------------------------------------------------- |
| `--mode <mode>`       | `funnel`                                      | `off`, `serve` (tailnet) або `funnel` (публічний). |
| `--path <path>`       | `tailscale.path` у конфігурації або `--serve-path` | Шлях Tailscale, до якого потрібно надати доступ. |
| `--port <port>`       | `serve.port` у конфігурації або `3334`        | Локальний порт Webhook.                           |
| `--serve-path <path>` | `serve.path` у конфігурації або `/voice/webhook` | Локальний шлях Webhook.                         |

```bash
openclaw voicecall expose --mode serve
openclaw voicecall expose --mode funnel
openclaw voicecall expose --mode off
```

<Warning>
Надавайте доступ до кінцевої точки Webhook лише мережам, яким довіряєте. За можливості віддавайте перевагу Tailscale Serve перед Funnel.
</Warning>
