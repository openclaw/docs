# `openclaw transcripts`

Інспектор лише для читання транскриптів, записаних інструментом агента `transcripts`.
Захоплення, імпорт і узагальнення виконуються через цей інструмент, а не через цей CLI.

Артефакти зберігаються в каталозі стану:

```text
$OPENCLAW_STATE_DIR/transcripts/YYYY-MM-DD/<session>/
  metadata.json
  transcript.jsonl
  summary.json
  summary.md
```

Стандартний каталог стану — `~/.openclaw`; перевизначте його за допомогою `OPENCLAW_STATE_DIR`.
Каталог дати визначається за часом початку сеансу; каталог сеансу — це
безпечний для файлової системи ідентифікатор, утворений з ідентифікатора сеансу.

## Команди

```bash
openclaw transcripts list
openclaw transcripts show <session>
openclaw transcripts show YYYY-MM-DD/<session>
openclaw transcripts path <session>
openclaw transcripts path YYYY-MM-DD/<session>
openclaw transcripts path <session> --dir
openclaw transcripts path <session> --metadata
openclaw transcripts path <session> --transcript
openclaw transcripts list --json
openclaw transcripts show <session> --json
openclaw transcripts path <session> --json
```

| Команда                       | Опис                                                   |
| ----------------------------- | ------------------------------------------------------ |
| `list`                        | Вивести список збережених сеансів.                     |
| `show <session>`              | Вивести збережений файл `summary.md`.                  |
| `path <session>`              | Вивести шлях до `summary.md`.                          |
| `path <session> --dir`        | Вивести каталог сеансу.                                |
| `path <session> --metadata`   | Вивести шлях до `metadata.json`.                       |
| `path <session> --transcript` | Вивести шлях до `transcript.jsonl`.                    |
| `--json`                      | Вивести машиночитаний результат (для будь-якої підкоманди). |

`<session>` приймає або звичайний ідентифікатор сеансу, або селектор із датою
(`YYYY-MM-DD/<session>`). Використовуйте форму з датою, якщо той самий ідентифікатор
сеансу трапляється більш ніж в один день, наприклад `openclaw transcripts show
2026-05-22/standup`. Стандартні ідентифікатори сеансів містять позначку часу та випадковий
суфікс; задавайте сеансу фіксований ідентифікатор лише тоді, коли він унікальний у межах дня.

## Вивід

`list` виводить для кожного сеансу один рядок із полями, розділеними табуляцією: селектор, час початку, назва,
шлях до підсумку.

```text
2026-05-22/standup  2026-05-22T09:00:00.000Z  Щотижнева нарада  /Users/user/.openclaw/transcripts/2026-05-22/standup/summary.md
```

Селектор — найбезпечніше значення для передавання назад у `show` або `path`.

`list --json` повертає об’єкти з полями `sessionId`, `selector`, `date`, `title`,
`startedAt`, `stoppedAt`, `source`, `path`, `summaryPath`, `hasSummary`.

`show --json` повертає збережені метадані сеансу, селектор, каталог сеансу,
шлях до підсумку та текст підсумку у форматі Markdown.

`path --json` повертає вибраний шлях і відомості про те, чи існує цей файл.

## Кілька сеансів на день

Сеанси групуються спочатку за датою, а потім за ідентифікатором сеансу. Десять зустрічей за один день утворюють
десять сусідніх каталогів:

```text
~/.openclaw/transcripts/2026-05-22/
  transcript-2026-05-22T09-00-00-000Z-a1b2c3d4/
  transcript-2026-05-22T10-30-00-000Z-b2c3d4e5/
  standup/
```

Для автоматизації використовуйте стандартні згенеровані ідентифікатори. Використовуйте фіксований ідентифікатор на кшталт `standup` лише
тоді, коли він не повторюватиметься в межах тієї самої дати.

## Відсутні підсумки

Активні сеанси записують `summary.md` після завершення сеансу; імпортовані транскрипти
записують його одразу після імпорту. Сеанс може з’явитися в `list` без
підсумку, поки захоплення ще триває, якщо сталася помилка постачальника під час зупинення або якщо
метадані було записано до надходження будь-яких висловлювань.

Використовуйте `path <session> --transcript`, щоб переглянути необроблений транскрипт, доступний лише для дописування,
або виконайте дію `summarize` інструмента `transcripts`, щоб повторно створити підсумок
у форматі Markdown.

## Конфігурація

Захоплення вмикається за бажанням (джерела наживо можуть приєднуватися та записувати аудіо зустрічі). Увімкніть його
так:

```json
{
  "transcripts": {
    "enabled": true,
    "maxUtterances": 2000
  }
}
```

- `enabled` (стандартно `false`): увімкнути інструмент.
- `maxUtterances` (стандартно `2000`, обмежується діапазоном 1–10000): розмір буфера висловлювань для кожного
  сеансу.

Налаштуйте джерела автоматичного запуску за допомогою `transcripts.autoStart`. Кожен запис
вмикається своєю наявністю; вилучіть запис, щоб вимкнути відповідне джерело. `discord-voice`
— вбудоване джерело з підтримкою автоматичного запуску, яке потребує `guildId` і
`channelId`:

```json
{
  "transcripts": {
    "enabled": true,
    "autoStart": [
      {
        "providerId": "discord-voice",
        "guildId": "1234567890",
        "channelId": "2345678901"
      }
    ]
  }
}
```
