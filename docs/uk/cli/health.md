# `openclaw health`

Отримує знімок стану працездатності запущеного Gateway через WebSocket RPC (без прямих сокетів каналів із CLI).

## Параметри

| Прапорець        | Типове значення | Опис                                                                                                                |
| ---------------- | --------------- | ------------------------------------------------------------------------------------------------------------------- |
| `--json`         | `false`         | Виводить машинозчитуваний JSON замість тексту.                                                                      |
| `--timeout <ms>` | `10000`         | Час очікування підключення в мілісекундах.                                                                          |
| `--verbose`      | `false`         | Примусово виконує перевірку в реальному часі та розгортає вивід для всіх налаштованих облікових записів і агентів. |
| `--debug`        | `false`         | Псевдонім для `--verbose`.                                                                                          |

Приклади:

```bash
openclaw health
openclaw health --json
openclaw health --timeout 2500
openclaw health --verbose
openclaw health --debug
```

## Поведінка

- Без `--verbose` Gateway може повернути кешований знімок (актуальний до 60 секунд і без змін порівняно зі станом середовища виконання каналів у реальному часі) та оновити його у фоновому режимі для наступного виклику.
- `--verbose` примусово виконує перевірку в реальному часі (перевірки облікових записів для кожного каналу), виводить відомості про підключення до Gateway та розгортає зручний для читання вивід для всіх налаштованих облікових записів і агентів, а не лише для типового агента.
- `--json` завжди повертає повний знімок: канали, перевірки кожного облікового запису, стан завантаження плагінів, стан карантину рушія контексту, стан кешу цін моделей, стан циклу подій і сховища сеансів кожного агента.
