# `openclaw daemon`

Застарілий псевдонім для керування службою Gateway. `openclaw daemon ...` відповідає тим самим командам керування службою, що й `openclaw gateway ...`. Для актуальної документації та прикладів використовуйте [`openclaw gateway`](/uk/cli/gateway).

## Використання

```bash
openclaw daemon status
openclaw daemon install
openclaw daemon start
openclaw daemon stop
openclaw daemon restart
openclaw daemon uninstall
```

## Підкоманди та параметри

| Підкоманда  | Параметри                                                                                          |
| ----------- | ------------------------------------------------------------------------------------------------ |
| `status`    | `--url`, `--token`, `--password`, `--timeout`, `--no-probe`, `--require-rpc`, `--deep`, `--json` |
| `install`   | `--port`, `--runtime <node\|bun>`, `--token`, `--wrapper <path>`, `--force`, `--json`            |
| `uninstall` | `--json`                                                                                         |
| `start`     | `--json`                                                                                         |
| `stop`      | `--json`, `--disable` (лише launchd: постійно вимикає KeepAlive/RunAtLoad до наступного запуску) |
| `restart`   | `--force`, `--safe`, `--skip-deferral`, `--wait <duration>`, `--json`                            |

- `status`: показує стан установлення служби (launchd/systemd/schtasks) і перевіряє працездатність Gateway.
- `install`: установлює службу; `--force` повторно встановлює або перезаписує наявне встановлення.
- `restart --safe`: просить запущений Gateway попередньо перевірити активні завдання й запланувати один об’єднаний перезапуск після їх завершення в межах часу, заданого `gateway.reload.deferralTimeoutMs` (типово 300000 мс/5 хвилин; установіть `0`, щоб чекати необмежено довго). Після вичерпання цього часу перезапуск однаково виконується примусово. Звичайна команда `restart` безпосередньо використовує диспетчер служб; `--force` негайно перевизначає відкладення.
- `restart --safe --skip-deferral`: обходить механізм відкладення через активні завдання, тому Gateway перезапускається негайно, навіть якщо повідомлено про перешкоди. Потребує `--safe`.

## Примітки

- `status` за можливості визначає налаштовані SecretRef для автентифікації під час перевірки. Якщо обов’язкове посилання SecretRef не вдається визначити, `status --json` повідомляє про `rpc.authWarning`; явно передайте `--token`/`--password` або спочатку усуньте проблему з джерелом секрету. Попередження про невизначені дані автентифікації приховуються, щойно перевірка успішно завершується за інших умов.
- `status --deep` додає систему перевірку на рівні системи з максимальними зусиллями для пошуку інших служб, подібних до Gateway (виводить підказки щодо очищення; як і раніше, рекомендовано використовувати один Gateway на комп’ютер), і запускає перевірку конфігурації в режимі з урахуванням плагінів, показуючи попередження маніфестів плагінів, які пропускає швидкий типовий режим.
- У встановленнях Linux із systemd перевірки розбіжностей токенів аналізують джерела модулів `Environment=` і `EnvironmentFile=`.
- Перевірки розбіжностей токенів визначають SecretRef для `gateway.auth.token` за допомогою об’єднаного середовища виконання (спочатку середовище команди служби, потім середовище процесу). Якщо автентифікація за токеном фактично не активна (`gateway.auth.mode` має значення `password`/`none`/`trusted-proxy` або не задано, а пароль може мати пріоритет), визначення токена з конфігурації пропускається.
- `install` перевіряє, чи можна визначити керований через SecretRef параметр `gateway.auth.token`, але ніколи не зберігає визначене значення в метаданих середовища служби; якщо визначити його неможливо, встановлення завершується відмовою.
- Якщо налаштовано і `gateway.auth.token`, і `gateway.auth.password`, а `gateway.auth.mode` не задано, `install` блокується, доки ви явно не задасте режим.
- У macOS команда `install` зберігає файли plist LaunchAgent і створений файл середовища/обгортку доступними лише власнику (режим `0600`/`0700`), замість вбудовування секретів у `EnvironmentVariables`.
- Запуск кількох екземплярів Gateway на одному хості: ізолюйте порти, конфігурацію/стан і робочі простори. Див. [Кілька екземплярів Gateway](/uk/gateway#multiple-gateways-same-host).
