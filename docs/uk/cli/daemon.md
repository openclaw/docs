# `openclaw daemon`

Застарілий псевдонім для керування службою Gateway. `openclaw daemon ...` відповідає тим самим командам керування службою, що й `openclaw gateway ...`. Для актуальної документації та прикладів використовуйте [`openclaw gateway`](/uk/cli/gateway).

## Використання

```bash
openclaw daemon status
openclaw daemon install
openclaw daemon start
openclaw daemon stop
openclaw daemon restart
openclaw daemon uninstall
```

## Підкоманди та параметри

| Підкоманда  | Параметри                                                                                          |
| ----------- | ------------------------------------------------------------------------------------------------ |
| `status`    | `--url`, `--token`, `--password`, `--timeout`, `--no-probe`, `--require-rpc`, `--deep`, `--json` |
| `install`   | `--port`, `--runtime <node>`, `--token`, `--wrapper <path>`, `--force`, `--json`                 |
| `uninstall` | `--json`                                                                                         |
| `start`     | `--json`                                                                                         |
| `stop`      | `--json`, `--disable` (лише launchd: постійно пригнічує KeepAlive/RunAtLoad до наступного запуску) |
| `restart`   | `--force`, `--safe`, `--skip-deferral`, `--wait <duration>`, `--json`                            |

- `status`: показує стан встановлення служби (launchd/systemd/schtasks) і перевіряє працездатність Gateway.
- `install`: установлює службу; `--force` повторно встановлює або перезаписує наявну інсталяцію.
- `restart --safe`: просить запущений Gateway попередньо перевірити активні завдання та запланувати один об’єднаний перезапуск після їх завершення в межах `gateway.reload.deferralTimeoutMs` (типово 300000ms/5 хвилин; установіть `0`, щоб чекати необмежено). Після вичерпання цього часу перезапуск усе одно виконується примусово. Звичайна команда `restart` безпосередньо використовує диспетчер служб; `--force` негайно перевизначає цю поведінку.
- `restart --safe --skip-deferral`: обходить механізм відкладення через активні завдання, тож Gateway перезапускається негайно, навіть якщо повідомлено про блокувальні чинники. Потребує `--safe`.

## Примітки

- `status` за можливості розв’язує налаштовані SecretRef автентифікації для автентифікації перевірки. Якщо обов’язковий SecretRef не розв’язано, `status --json` повідомляє `rpc.authWarning`; передайте `--token`/`--password` явно або спочатку забезпечте розв’язання джерела секрету. Попередження про нерозв’язану автентифікацію приховуються, щойно перевірка в іншому разі завершується успішно.
- `status --deep` додає виконуване за можливості загальносистемне сканування інших служб, подібних до Gateway (виводить підказки щодо очищення; як і раніше, рекомендовано використовувати один Gateway на машину), і запускає перевірку конфігурації в режимі з урахуванням плагінів, показуючи попередження маніфестів плагінів, які пропускає швидкий типовий шлях.
- У встановленнях Linux systemd перевірки розбіжності токенів аналізують джерела модулів `Environment=` і `EnvironmentFile=`.
- Перевірки розбіжності токенів розв’язують SecretRef `gateway.auth.token` за допомогою об’єднаного середовища виконання (спочатку середовище команди служби, потім середовище процесу). Якщо автентифікація за токеном фактично не активна (`gateway.auth.mode` зі значенням `password`/`none`/`trusted-proxy` або не задано, коли пріоритет може отримати пароль), розв’язання токена конфігурації пропускається.
- `install` перевіряє, чи можна розв’язати керований через SecretRef параметр `gateway.auth.token`, але ніколи не зберігає розв’язане значення в метаданих середовища служби; якщо розв’язати його неможливо, встановлення завершується відмовою.
- Якщо налаштовано і `gateway.auth.token`, і `gateway.auth.password`, а `gateway.auth.mode` не задано, `install` блокується, доки режим не буде задано явно.
- У macOS команда `install` зберігає файли plist LaunchAgent і згенерований файл середовища/обгортку доступними лише власнику (режим `0600`/`0700`), замість вбудовування секретів у `EnvironmentVariables`.
- Запуск кількох Gateway на одному хості: ізолюйте порти, конфігурацію/стан і робочі простори. Див. [Кілька Gateway](/uk/gateway#multiple-gateways-same-host).
