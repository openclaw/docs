# `openclaw system`

Допоміжні засоби системного рівня для Gateway: додавання системних подій до черги, керування
Heartbeat і перегляд присутності.

Усі підкоманди `system` використовують RPC Gateway і приймають спільні клієнтські прапорці:

| Прапорець         | Типове значення                      | Опис                                                                                                                                                                                                 |
| ----------------- | ------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--url <url>`     | `gateway.remote.url`, якщо налаштовано | URL WebSocket для Gateway.                                                                                                                                                                           |
| `--token <token>` | немає                                | Токен Gateway (якщо потрібен).                                                                                                                                                                       |
| `--timeout <ms>`  | `30000`                              | Час очікування RPC у мілісекундах.                                                                                                                                                                   |
| `--expect-final`  | вимкнено                              | Очікувати остаточну відповідь (агента).                                                                                                                                                               |
| `--json`          | вимкнено                              | Виводити JSON. `heartbeat last/enable/disable` і `system presence` завжди виводять необроблене корисне навантаження RPC у форматі JSON незалежно від цього прапорця; `system event` використовує його для перемикання між JSON і звичайним рядком `ok`. |

## Поширені команди

```bash
openclaw system event --text "Check for urgent follow-ups" --mode now
openclaw system event --text "Check for urgent follow-ups" --url ws://127.0.0.1:18789 --token "$OPENCLAW_GATEWAY_TOKEN"
openclaw system heartbeat enable
openclaw system heartbeat last
openclaw system presence
```

## `system event`

Типово додає системну подію до черги **головного** сеансу. Наступний
Heartbeat вставляє її в запит як рядок `System:`. Використовуйте `--mode now`, щоб
негайно запустити Heartbeat; `next-heartbeat` (типове значення) очікує
наступного запланованого такту.

Передайте `--session-key`, щоб спрямувати подію до певного сеансу, наприклад для передавання
повідомлення про завершення асинхронного завдання назад до каналу, який його запустив.

<Note>
**Виняток щодо часу з `--session-key`:** коли вказано `--session-key`,
`--mode next-heartbeat` перетворюється на негайне цільове пробудження замість
очікування наступного запланованого такту. Цільові пробудження використовують намір Heartbeat
`immediate`, тому вони обходять перевірку засобу виконання на те, що час іще не настав, яка інакше
відклала б (і фактично відкинула) пробудження з наміром `event`. Якщо потрібна відкладена
доставка, не вказуйте `--session-key`, щоб подія потрапила до головного сеансу та
була доставлена з наступним регулярним Heartbeat.
</Note>

Прапорці:

- `--text <text>`: обов’язковий текст системної події.
- `--mode <mode>`: `now` або `next-heartbeat` (типове значення).
- `--session-key <sessionKey>`: необов’язковий; спрямовує подію до певного сеансу агента
  замість головного сеансу агента. Ключі, які не належать
  визначеному агенту, спрямовуються до головного сеансу агента.

## `system heartbeat last|enable|disable`

- `last`: показати останню подію Heartbeat.
- `enable`: знову ввімкнути Heartbeat (використовуйте, якщо його було вимкнено).
- `disable`: призупинити Heartbeat.

## `system presence`

Показує поточні записи про присутність у системі, відомі Gateway (вузли,
екземпляри та подібні рядки стану).

## Примітки

- Потрібен запущений Gateway, доступний через поточну конфігурацію (локальну або
  віддалену).
- Системні події є тимчасовими й не зберігаються після перезапусків.
