# `openclaw update`

Безпечно оновлюйте OpenClaw і перемикайтеся між каналами stable/beta/dev.

Якщо ви встановили через **npm/pnpm/bun** (глобальне встановлення, без метаданих git),
оновлення відбуваються через потік менеджера пакетів у [Оновлення](/uk/install/updating).

## Використання

```bash
openclaw update
openclaw update status
openclaw update wizard
openclaw update --channel beta
openclaw update --channel dev
openclaw update --tag beta
openclaw update --tag main
openclaw update --dry-run
openclaw update --no-restart
openclaw update --yes
openclaw update --json
openclaw --update
```

## Параметри

- `--no-restart`: пропустити перезапуск служби Gateway після успішного оновлення. Оновлення через менеджер пакетів, які виконують перезапуск Gateway, перевіряють, що перезапущена служба повідомляє очікувану оновлену версію, перш ніж команда завершиться успішно.
- `--channel <stable|beta|dev>`: установити канал оновлення (git + npm; зберігається в конфігурації).
- `--tag <dist-tag|version|spec>`: перевизначити ціль пакета лише для цього оновлення. Для встановлень із пакетів `main` зіставляється з `github:openclaw/openclaw#main`.
- `--dry-run`: попередньо показати заплановані дії оновлення (канал/тег/ціль/потік перезапуску) без запису конфігурації, встановлення, синхронізації Plugin або перезапуску.
- `--json`: вивести машиночитний JSON `UpdateRunResult`, включно з
  `postUpdate.plugins.integrityDrifts`, коли під час післяоновлювальної синхронізації Plugin
  виявлено розбіжність артефактів npm Plugin.
- `--timeout <seconds>`: тайм-аут для кожного кроку (типово 1800 с).
- `--yes`: пропустити запити на підтвердження (наприклад, підтвердження зниження версії).

<Warning>
Зниження версії потребує підтвердження, оскільки старіші версії можуть пошкодити конфігурацію.
</Warning>

## `update status`

Показати активний канал оновлення + тег/гілку/SHA git (для checkout джерела), а також доступність оновлень.

```bash
openclaw update status
openclaw update status --json
openclaw update status --timeout 10
```

Параметри:

- `--json`: вивести машиночитний JSON стану.
- `--timeout <seconds>`: тайм-аут для перевірок (типово 3 с).

## `update wizard`

Інтерактивний потік для вибору каналу оновлення й підтвердження, чи перезапускати Gateway
після оновлення (типово виконується перезапуск). Якщо ви виберете `dev` без checkout git, він
запропонує створити його.

Параметри:

- `--timeout <seconds>`: тайм-аут для кожного кроку оновлення (типово `1800`)

## Що це робить

Коли ви явно перемикаєте канали (`--channel ...`), OpenClaw також підтримує
узгодженість методу встановлення:

- `dev` → забезпечує checkout git (типово: `~/openclaw`, перевизначається через `OPENCLAW_GIT_DIR`),
  оновлює його й встановлює глобальний CLI з цього checkout.
- `stable` → встановлює з npm за допомогою `latest`.
- `beta` → надає перевагу dist-tag npm `beta`, але повертається до `latest`, якщо beta
  відсутня або старіша за поточний stable-реліз.

Автооновлювач ядра Gateway (коли його ввімкнено через конфігурацію) повторно використовує цей самий шлях оновлення.

Для встановлень через менеджер пакетів `openclaw update` визначає цільову
версію пакета перед викликом менеджера пакетів. Навіть якщо встановлена версія
вже відповідає цільовій, команда оновлює глобальне встановлення пакета,
а потім виконує синхронізацію Plugin, оновлення completion і роботу з перезапуском. Це забезпечує узгодженість упакованих
sidecar-компонентів і записів Plugin, якими володіє канал, з установленою збіркою OpenClaw.

## Потік checkout git

### Вибір каналу

- `stable`: checkout останнього тегу без beta, потім build і doctor.
- `beta`: надавати перевагу останньому тегу `-beta`, але переходити до останнього stable-тегу, якщо beta відсутня або старіша.
- `dev`: checkout `main`, потім fetch і rebase.

### Кроки оновлення

<Steps>
  <Step title="Перевірити чисте робоче дерево">
    Потрібна відсутність незакомічених змін.
  </Step>
  <Step title="Перемкнути канал">
    Перемикає на вибраний канал (тег або гілку).
  </Step>
  <Step title="Отримати оновлення з upstream">
    Лише для dev.
  </Step>
  <Step title="Попередня build-перевірка (лише dev)">
    Запускає lint і build TypeScript у тимчасовому worktree. Якщо tip не проходить, відступає назад до 10 комітів, щоб знайти найновішу чисту build.
  </Step>
  <Step title="Rebase">
    Виконує rebase на вибраний коміт (лише dev).
  </Step>
  <Step title="Встановити залежності">
    Використовує менеджер пакетів репозиторію. Для checkout на pnpm засіб оновлення за потреби ініціалізує `pnpm` (спочатку через `corepack`, потім через тимчасовий резервний варіант `npm install pnpm@10`) замість запуску `npm run build` всередині workspace pnpm.
  </Step>
  <Step title="Зібрати Control UI">
    Збирає gateway і Control UI.
  </Step>
  <Step title="Запустити doctor">
    `openclaw doctor` запускається як фінальна безпечна перевірка оновлення.
  </Step>
  <Step title="Синхронізувати Plugin">
    Синхронізує Plugin з активним каналом. Dev використовує вбудовані Plugin; stable і beta використовують npm. Оновлює Plugin, встановлені через npm.
  </Step>
</Steps>

<Warning>
Якщо оновлення точно закріпленого npm Plugin визначає артефакт, цілісність якого відрізняється від збереженого запису встановлення, `openclaw update` перериває оновлення артефакту цього Plugin замість його встановлення. Перевстановлюйте або оновлюйте Plugin явно лише після перевірки, що ви довіряєте новому артефакту.
</Warning>

<Note>
Збої синхронізації Plugin після оновлення призводять до збою результату оновлення й зупиняють подальші дії з перезапуском. Виправте помилку встановлення або оновлення Plugin, а потім повторно запустіть `openclaw update`.

Коли оновлений Gateway запускається, залежності runtime увімкнених вбудованих Plugin готуються перед активацією Plugin. Перезапуски, ініційовані оновленням, завершують будь-яке активне розгортання залежностей runtime перед закриттям Gateway, щоб перезапуски через менеджер служб не переривали npm install, що триває.

Якщо ініціалізація pnpm все одно не вдається, засіб оновлення зупиняється рано з помилкою, специфічною для менеджера пакетів, замість спроби виконати `npm run build` всередині checkout.
</Note>

## Скорочення `--update`

`openclaw --update` переписується в `openclaw update` (зручно для оболонок і скриптів запуску).
