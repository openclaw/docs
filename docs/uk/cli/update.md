# `openclaw update`

Безпечно оновлюйте OpenClaw і перемикайтеся між каналами stable/beta/dev.

Якщо ви встановили через **npm/pnpm/bun** (глобальне встановлення, без метаданих git),
оновлення виконуються через потік менеджера пакетів у [Оновлення](/uk/install/updating).

## Використання

```bash
openclaw update
openclaw update status
openclaw update wizard
openclaw update --channel beta
openclaw update --channel dev
openclaw update --tag beta
openclaw update --tag main
openclaw update --dry-run
openclaw update --no-restart
openclaw update --yes
openclaw update --json
openclaw --update
```

## Параметри

- `--no-restart`: пропустити перезапуск служби Gateway після успішного оновлення. Оновлення через менеджер пакетів, які все ж перезапускають Gateway, перевіряють, що перезапущена служба повідомляє очікувану оновлену версію, перш ніж команда завершиться успішно.
- `--channel <stable|beta|dev>`: встановити канал оновлення (git + npm; зберігається в конфігурації).
- `--tag <dist-tag|version|spec>`: перевизначити цільовий пакет лише для цього оновлення. Для пакетних установок `main` зіставляється з `github:openclaw/openclaw#main`.
- `--dry-run`: попередньо показати заплановані дії оновлення (канал/тег/цільовий об’єкт/послідовність перезапуску) без запису конфігурації, встановлення, синхронізації Plugin або перезапуску.
- `--json`: вивести машиночитаний JSON `UpdateRunResult`, зокрема `postUpdate.plugins.integrityDrifts`, коли під час післяоновлювальної синхронізації Plugin виявлено дрейф артефактів npm Plugin.
- `--timeout <seconds>`: тайм-аут для кожного кроку (типово 1800 с).
- `--yes`: пропустити запити на підтвердження (наприклад, підтвердження пониження версії).

<Warning>
Пониження версії потребує підтвердження, оскільки старіші версії можуть пошкодити конфігурацію.
</Warning>

## `update status`

Показати активний канал оновлення + тег/гілку/SHA git (для вихідних копій), а також доступність оновлень.

```bash
openclaw update status
openclaw update status --json
openclaw update status --timeout 10
```

Параметри:

- `--json`: вивести машиночитаний JSON статусу.
- `--timeout <seconds>`: тайм-аут для перевірок (типово 3 с).

## `update wizard`

Інтерактивний потік для вибору каналу оновлення та підтвердження, чи перезапускати Gateway
після оновлення (типово — перезапускати). Якщо ви виберете `dev` без git-копії, буде
запропоновано створити її.

Параметри:

- `--timeout <seconds>`: тайм-аут для кожного кроку оновлення (типово `1800`)

## Що він робить

Коли ви явно перемикаєте канали (`--channel ...`), OpenClaw також
підтримує узгодженість методу встановлення:

- `dev` → забезпечує git-копію (типово: `~/openclaw`, перевизначається через `OPENCLAW_GIT_DIR`),
  оновлює її та встановлює глобальний CLI з цієї копії.
- `stable` → встановлює з npm за допомогою `latest`.
- `beta` → надає перевагу npm dist-tag `beta`, але повертається до `latest`, якщо beta
  відсутня або старіша за поточний стабільний реліз.

Автооновлювач ядра Gateway (коли увімкнений через конфігурацію) повторно використовує цей самий шлях оновлення.

Для встановлень через менеджер пакетів `openclaw update` визначає цільову версію пакета
перед викликом менеджера пакетів. Навіть якщо встановлена версія вже відповідає цільовій, команда оновлює глобальне встановлення пакета,
потім запускає синхронізацію Plugin, оновлення автодоповнення команд ядра та перезапуск. Це
дозволяє узгодити пакетовані сайдкари й записи Plugin, що належать каналу, зі
встановленою збіркою OpenClaw, залишаючи повне перебудування автодоповнення команд Plugin для
явних запусків `openclaw completion --write-state`.

## Потік git-копії

### Вибір каналу

- `stable`: перейти на найновіший тег без beta, потім виконати збірку і doctor.
- `beta`: надавати перевагу найновішому тегу `-beta`, але повертатися до найновішого стабільного тега, якщо beta відсутня або старіша.
- `dev`: перейти на `main`, потім виконати fetch і rebase.

### Кроки оновлення

<Steps>
  <Step title="Перевірити чистий робочий каталог">
    Потребує відсутності незакомічених змін.
  </Step>
  <Step title="Перемкнути канал">
    Перемикає на вибраний канал (тег або гілку).
  </Step>
  <Step title="Отримати зміни з upstream">
    Лише для dev.
  </Step>
  <Step title="Попередня збірка (лише dev)">
    Запускає lint і збірку TypeScript у тимчасовому worktree. Якщо вершина не проходить перевірку, відступає назад до 10 комітів, щоб знайти найновішу чисту збірку.
  </Step>
  <Step title="Rebase">
    Виконує rebase на вибраний коміт (лише dev).
  </Step>
  <Step title="Встановити залежності">
    Використовує менеджер пакетів репозиторію. Для копій pnpm засіб оновлення за потреби ініціалізує `pnpm` (спочатку через `corepack`, а потім через тимчасовий резервний варіант `npm install pnpm@10`) замість запуску `npm run build` усередині workspace pnpm.
  </Step>
  <Step title="Зібрати Control UI">
    Збирає gateway і Control UI.
  </Step>
  <Step title="Запустити doctor">
    `openclaw doctor` запускається як фінальна перевірка безпечного оновлення.
  </Step>
  <Step title="Синхронізувати Plugin">
    Синхронізує Plugin з активним каналом. Dev використовує вбудовані Plugin; stable і beta використовують npm. Оновлює Plugin, встановлені через npm.
  </Step>
</Steps>

<Warning>
Якщо оновлення точно закріпленого npm Plugin визначає артефакт, цілісність якого відрізняється від збереженого запису встановлення, `openclaw update` перериває оновлення артефакту цього Plugin замість його встановлення. Перевстановлюйте або оновлюйте Plugin явно лише після перевірки, що ви довіряєте новому артефакту.
</Warning>

<Note>
Збої післяоновлювальної синхронізації Plugin призводять до невдалого результату оновлення та зупиняють подальші дії з перезапуску. Виправте помилку встановлення або оновлення Plugin, а потім знову запустіть `openclaw update`.

Коли оновлений Gateway запускається, залежності середовища виконання ввімкнених вбудованих Plugin розміщуються перед активацією Plugin. Перезапуски, ініційовані оновленням, завершують будь-яке активне розміщення залежностей середовища виконання перед закриттям Gateway, тому перезапуски через менеджер служб не переривають npm-встановлення, що виконується.

Якщо ініціалізація pnpm усе ще не вдається, засіб оновлення зупиняється раніше з помилкою, специфічною для менеджера пакетів, замість спроби виконати `npm run build` усередині копії.
</Note>

## Скорочення `--update`

`openclaw --update` переписується в `openclaw update` (зручно для оболонок і скриптів запуску).
