# `openclaw pairing`

Схвалення або перегляд запитів на сполучення в особистих повідомленнях для каналів, які підтримують сполучення (лише особисті повідомлення в чаті — для сполучення Node або пристрою використовуйте `openclaw devices`).

Див. також: [Процес сполучення](/uk/channels/pairing)

## Команди

```bash
openclaw pairing list telegram
openclaw pairing list --channel telegram --account work
openclaw pairing list telegram --json

openclaw pairing approve <code>
openclaw pairing approve telegram <code>
openclaw pairing approve --channel telegram --account work <code> --notify
```

## `pairing list`

Виводить список запитів на сполучення, що очікують розгляду, для одного каналу.

| Параметр                | Опис                                             |
| ----------------------- | ------------------------------------------------ |
| `[channel]`             | позиційний ідентифікатор каналу                  |
| `--channel <channel>`   | явно вказаний ідентифікатор каналу               |
| `--account <accountId>` | ідентифікатор облікового запису для багатопрофільних каналів |
| `--json`                | формат виведення для машинної обробки             |

Якщо налаштовано кілька каналів із підтримкою сполучення, передайте канал як позиційний аргумент або за допомогою `--channel`. Канали розширень працюють, якщо ідентифікатор каналу дійсний.

## `pairing approve`

Схвалює код сполучення, що очікує розгляду, і надає відповідному відправнику доступ.

Використання:

- `openclaw pairing approve <channel> <code>`
- `openclaw pairing approve --channel <channel> <code>`
- `openclaw pairing approve <code>`, якщо налаштовано рівно один канал із підтримкою сполучення

Параметри: `--channel <channel>`, `--account <accountId>`, `--notify` (надсилає підтвердження запитувачу через той самий канал).

### Початкове налаштування власника

Якщо під час схвалення коду сполучення `commands.ownerAllowFrom` порожній, OpenClaw також записує схваленого відправника як власника команд, використовуючи запис у межах каналу, наприклад `telegram:123456789`. Так виконується лише початкове налаштування першого власника — подальші схвалення сполучення ніколи не замінюють і не розширюють `commands.ownerAllowFrom`.

Власник команд — це обліковий запис оператора-людини, якому дозволено виконувати команди лише для власника та схвалювати небезпечні дії, як-от `/diagnostics`, `/export-trajectory`, `/config` і схвалення виконання команд. Сполучення лише дає відправнику змогу спілкуватися з агентом; саме по собі воно не надає привілеїв власника, крім цього одноразового початкового налаштування.

Якщо ви схвалили відправника до появи цього механізму початкового налаштування, виконайте `openclaw doctor`; команда попередить, якщо власника команд не налаштовано, і покаже точну команду `openclaw config set commands.ownerAllowFrom ...` для виправлення.
