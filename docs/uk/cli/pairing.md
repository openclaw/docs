# `openclaw pairing`

Схвалюйте або перевіряйте запити на сполучення в особистих повідомленнях для каналів, які підтримують сполучення (лише особисті повідомлення в чаті — для сполучення вузлів/пристроїв використовується `openclaw devices`).

Пов’язана тема: [Процес сполучення](/uk/channels/pairing)

## Команди

```bash
openclaw pairing list telegram
openclaw pairing list --channel telegram --account work
openclaw pairing list telegram --json

openclaw pairing approve <code>
openclaw pairing approve telegram <code>
openclaw pairing approve --channel telegram --account work <code> --notify
```

## `pairing list`

Перелічує очікувані запити на сполучення для одного каналу.

| Параметр                | Опис                                  |
| ----------------------- | ------------------------------------- |
| `[channel]`             | позиційний ідентифікатор каналу       |
| `--channel <channel>`   | явний ідентифікатор каналу            |
| `--account <accountId>` | ідентифікатор облікового запису для каналів із кількома обліковими записами |
| `--json`                | машинозчитуваний формат виведення     |

Якщо налаштовано кілька каналів із підтримкою сполучення, передайте канал позиційно або за допомогою `--channel`. Канали розширень працюють, якщо ідентифікатор каналу дійсний.

## `pairing approve`

Схвалює очікуваний код сполучення та дозволяє цьому відправнику доступ.

Використання:

- `openclaw pairing approve <channel> <code>`
- `openclaw pairing approve --channel <channel> <code>`
- `openclaw pairing approve <code>`, якщо налаштовано рівно один канал із підтримкою сполучення

Параметри: `--channel <channel>`, `--account <accountId>`, `--notify` (надсилає підтвердження запитувачу через той самий канал).

### Початкове налаштування власника

Якщо `commands.ownerAllowFrom` порожній під час схвалення коду сполучення, OpenClaw також записує схваленого відправника як власника команд, використовуючи запис, обмежений каналом, наприклад `telegram:123456789`. Так виконується лише початкове налаштування першого власника — подальші схвалення сполучення ніколи не замінюють і не розширюють `commands.ownerAllowFrom`.

Власник команд — це обліковий запис оператора-людини, якому дозволено виконувати команди лише для власника та схвалювати небезпечні дії, як-от `/diagnostics`, `/export-session`, `/export-trajectory`, `/config` і схвалення виконання команд. Сполучення лише дає відправнику змогу спілкуватися з агентом; саме по собі воно не надає привілеїв власника, окрім цього одноразового початкового налаштування.

Якщо відправника було схвалено до появи цього початкового налаштування, виконайте `openclaw doctor`; команда попередить, якщо власника команд не налаштовано, і покаже точну команду `openclaw config set commands.ownerAllowFrom ...` для виправлення.
