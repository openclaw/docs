# `openclaw logs`

Виводить у реальному часі файлові журнали Gateway через RPC (працює у віддаленому режимі).

Пов’язане:

- Огляд журналювання: [Журналювання](/uk/logging)
- Gateway CLI: [gateway](/uk/cli/gateway)

## Параметри

- `--limit <n>`: максимальна кількість рядків журналу для повернення (за замовчуванням `200`)
- `--max-bytes <n>`: максимальна кількість байтів для читання з файлу журналу (за замовчуванням `250000`)
- `--follow`: відстежувати потік журналу
- `--interval <ms>`: інтервал опитування під час відстеження (за замовчуванням `1000`)
- `--json`: виводити JSON-події, розділені рядками
- `--plain`: виведення звичайним текстом без стилізованого форматування
- `--no-color`: вимкнути кольори ANSI
- `--local-time`: відображати часові позначки у вашому локальному часовому поясі

## Спільні параметри RPC Gateway

`openclaw logs` також приймає стандартні прапорці клієнта Gateway:

- `--url <url>`: URL WebSocket Gateway
- `--token <token>`: токен Gateway
- `--timeout <ms>`: час очікування в мс (за замовчуванням `30000`)
- `--expect-final`: чекати фінальної відповіді, коли виклик Gateway підтримується агентом

Коли ви передаєте `--url`, CLI не застосовує автоматично облікові дані з конфігурації або середовища. Додайте `--token` явно, якщо цільовий Gateway вимагає автентифікації.

## Приклади

```bash
openclaw logs
openclaw logs --follow
openclaw logs --follow --interval 2000
openclaw logs --limit 500 --max-bytes 500000
openclaw logs --json
openclaw logs --plain
openclaw logs --no-color
openclaw logs --limit 500
openclaw logs --local-time
openclaw logs --follow --local-time
openclaw logs --url ws://127.0.0.1:18789 --token "$OPENCLAW_GATEWAY_TOKEN"
```

## Примітки

- Використовуйте `--local-time`, щоб відображати часові позначки у вашому локальному часовому поясі.
- Якщо неявний local loopback Gateway запитує сполучення, закривається під час підключення або перевищує час очікування до відповіді `logs.tail`, `openclaw logs` автоматично повертається до налаштованого файлового журналу Gateway. Явні цілі `--url` не використовують цей резервний варіант.
- Під час використання `--follow` тимчасові роз’єднання gateway (закриття WebSocket, перевищення часу очікування, втрата з’єднання) запускають автоматичне повторне підключення з експоненційною затримкою (до 8 повторних спроб, максимум 30 с між спробами). Під час кожної повторної спроби в stderr виводиться попередження, а після успішного опитування виводиться сповіщення `[logs] gateway reconnected`. У режимі `--json` і попередження про повторну спробу, і перехід до повторного підключення виводяться в stderr як записи `{"type":"notice"}`. Невідновлювані помилки (збій автентифікації, неправильна конфігурація) все одно завершують виконання негайно.
