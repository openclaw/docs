# `openclaw logs`

Передавайте файлові журнали Gateway через RPC у режимі tail (працює у віддаленому режимі).

Пов’язано:

- Огляд журналювання: [Журналювання](/uk/logging)
- CLI Gateway: [gateway](/uk/cli/gateway)

## Параметри

- `--limit <n>`: максимальна кількість рядків журналу для повернення (за замовчуванням `200`)
- `--max-bytes <n>`: максимальна кількість байтів для читання з файлу журналу (за замовчуванням `250000`)
- `--follow`: стежити за потоком журналу
- `--interval <ms>`: інтервал опитування під час стеження (за замовчуванням `1000`)
- `--json`: виводити події JSON, розділені рядками
- `--plain`: вивід звичайного тексту без стилізованого форматування
- `--no-color`: вимкнути кольори ANSI
- `--local-time`: відображати часові позначки у вашому локальному часовому поясі (за замовчуванням)
- `--utc`: відображати часові позначки в UTC

## Спільні параметри RPC Gateway

`openclaw logs` також приймає стандартні прапорці клієнта Gateway:

- `--url <url>`: URL WebSocket Gateway
- `--token <token>`: токен Gateway
- `--timeout <ms>`: час очікування в мс (за замовчуванням `30000`)
- `--expect-final`: чекати на фінальну відповідь, коли виклик Gateway підтримується агентом

Коли ви передаєте `--url`, CLI не застосовує автоматично облікові дані з конфігурації або середовища. Явно додайте `--token`, якщо цільовий Gateway вимагає автентифікації.

## Приклади

```bash
openclaw logs
openclaw logs --follow
openclaw logs --follow --interval 2000
openclaw logs --limit 500 --max-bytes 500000
openclaw logs --json
openclaw logs --plain
openclaw logs --no-color
openclaw logs --limit 500
openclaw logs --local-time
openclaw logs --utc
openclaw logs --follow --local-time
openclaw logs --url ws://127.0.0.1:18789 --token "$OPENCLAW_GATEWAY_TOKEN"
```

## Примітки

- Часові позначки за замовчуванням відображаються у вашому локальному часовому поясі. Використовуйте `--utc` для виводу в UTC.
- Якщо неявний Gateway local loopback запитує сполучення, закривається під час підключення або час очікування спливає до відповіді `logs.tail`, `openclaw logs` автоматично повертається до налаштованого файлового журналу Gateway. Явні цілі `--url` не використовують цей резервний варіант.
- `openclaw logs --follow` не стежить за резервними варіантами налаштованого файлу після збоїв неявного локального RPC Gateway. У Linux він використовує активний журнал Gateway user-systemd за PID, коли він доступний, і друкує вибране джерело журналу; інакше він продовжує повторювати спроби з live Gateway замість читання потенційно застарілого суміжного файлу в режимі tail.
- Під час використання `--follow` тимчасові відключення gateway (закриття WebSocket, час очікування, розрив з’єднання) запускають автоматичне повторне підключення з експоненційною затримкою (до 8 повторних спроб, з обмеженням 30 с між спробами). Попередження друкується в stderr під час кожної повторної спроби, а сповіщення `[logs] gateway reconnected` друкується після успішного опитування. У режимі `--json` і попередження про повторну спробу, і перехід повторного підключення виводяться як записи `{"type":"notice"}` у stderr. Помилки, які неможливо відновити (збій автентифікації, неправильна конфігурація), усе одно завершують роботу негайно.
