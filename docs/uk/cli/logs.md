# `openclaw logs`

Відстежує файлові журнали Gateway через RPC (працює у віддаленому режимі).

Пов’язане:

- Огляд журналювання: [Журналювання](/uk/logging)
- CLI Gateway: [gateway](/uk/cli/gateway)

## Параметри

- `--limit <n>`: максимальна кількість рядків журналу для повернення (за замовчуванням `200`)
- `--max-bytes <n>`: максимальна кількість байтів для читання з файлу журналу (за замовчуванням `250000`)
- `--follow`: стежити за потоком журналу
- `--interval <ms>`: інтервал опитування під час стеження (за замовчуванням `1000`)
- `--json`: виводити події JSON, розділені рядками
- `--plain`: вивід звичайного тексту без стилізованого форматування
- `--no-color`: вимкнути кольори ANSI
- `--local-time`: відображати часові мітки у вашому локальному часовому поясі

## Спільні параметри RPC Gateway

`openclaw logs` також приймає стандартні прапорці клієнта Gateway:

- `--url <url>`: URL WebSocket Gateway
- `--token <token>`: токен Gateway
- `--timeout <ms>`: час очікування в мс (за замовчуванням `30000`)
- `--expect-final`: чекати остаточної відповіді, коли виклик Gateway підтримується агентом

Коли ви передаєте `--url`, CLI не застосовує автоматично конфігурацію або облікові дані з середовища. Додайте `--token` явно, якщо цільовий Gateway вимагає автентифікації.

## Приклади

```bash
openclaw logs
openclaw logs --follow
openclaw logs --follow --interval 2000
openclaw logs --limit 500 --max-bytes 500000
openclaw logs --json
openclaw logs --plain
openclaw logs --no-color
openclaw logs --limit 500
openclaw logs --local-time
openclaw logs --follow --local-time
openclaw logs --url ws://127.0.0.1:18789 --token "$OPENCLAW_GATEWAY_TOKEN"
```

## Примітки

- Використовуйте `--local-time`, щоб відображати часові мітки у вашому локальному часовому поясі.
- Якщо неявний local loopback Gateway запитує сполучення, закривається під час підключення або час очікування спливає до відповіді `logs.tail`, `openclaw logs` автоматично повертається до налаштованого файлового журналу Gateway. Явні цілі `--url` не використовують цей резервний механізм.
- Під час використання `--follow` тимчасові роз’єднання з gateway (закриття WebSocket, сплив часу очікування, розрив з’єднання) запускають автоматичне повторне підключення з експоненційною затримкою (до 8 повторних спроб, з обмеженням 30 с між спробами). Під час кожної повторної спроби до stderr виводиться попередження. Невідновлювані помилки (помилка автентифікації, неправильна конфігурація) усе одно завершуються негайно.
