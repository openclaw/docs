# Аудіо / голосові нотатки (2026-01-17)

## Що працює

- **Розуміння медіа (аудіо)**: Якщо розуміння аудіо ввімкнене (або автоматично визначене), OpenClaw:
  1. Знаходить перше аудіовкладення (локальний шлях або URL) і завантажує його, якщо потрібно.
  2. Застосовує `maxBytes` перед надсиланням до кожного запису моделі.
  3. Запускає перший придатний запис моделі за порядком (провайдер або CLI).
  4. Якщо це завершується помилкою або пропускається (розмір/тайм-аут), пробує наступний запис.
  5. У разі успіху замінює `Body` блоком `[Audio]` і встановлює `{{Transcript}}`.
- **Розбір команд**: Коли транскрибування успішне, `CommandBody`/`RawBody` встановлюються в текст транскрипту, щоб slash-команди й далі працювали.
- **Докладне журналювання**: У `--verbose` ми логуємо, коли запускається транскрибування і коли воно замінює тіло повідомлення.

## Автовизначення (типово)

Якщо ви **не налаштовуєте моделі** і `tools.media.audio.enabled` **не** встановлено в `false`,
OpenClaw автоматично визначає в такому порядку й зупиняється на першому робочому варіанті:

1. **Активна модель відповіді**, якщо її провайдер підтримує розуміння аудіо.
2. **Локальні CLI** (якщо встановлені)
   - `sherpa-onnx-offline` (потребує `SHERPA_ONNX_MODEL_DIR` з encoder/decoder/joiner/tokens)
   - `whisper-cli` (з `whisper-cpp`; використовує `WHISPER_CPP_MODEL` або вбудовану tiny-модель)
   - `whisper` (Python CLI; автоматично завантажує моделі)
3. **Gemini CLI** (`gemini`) з використанням `read_many_files`
4. **Авторизація провайдера**
   - Спочатку пробуються налаштовані записи `models.providers.*`, які підтримують аудіо
   - Порядок вбудованих запасних варіантів: OpenAI → Groq → xAI → Deepgram → Google → SenseAudio → ElevenLabs → Mistral

Щоб вимкнути автовизначення, встановіть `tools.media.audio.enabled: false`.
Щоб налаштувати вручну, задайте `tools.media.audio.models`.
Примітка: виявлення бінарних файлів виконується в режимі best-effort на macOS/Linux/Windows; переконайтеся, що CLI є в `PATH` (ми розгортаємо `~`), або задайте явну CLI-модель з повним шляхом до команди.

## Приклади конфігурації

### Провайдер + запасний CLI-варіант (OpenAI + Whisper CLI)

```json5
{
  tools: {
    media: {
      audio: {
        enabled: true,
        maxBytes: 20971520,
        models: [
          { provider: "openai", model: "gpt-4o-mini-transcribe" },
          {
            type: "cli",
            command: "whisper",
            args: ["--model", "base", "{{MediaPath}}"],
            timeoutSeconds: 45,
          },
        ],
      },
    },
  },
}
```

## Примітки та обмеження

- Авторизація провайдера виконується за стандартним порядком авторизації моделей (профілі авторизації, змінні середовища, `models.providers.*.apiKey`).
- Докладніше про налаштування Groq: [Groq](/uk/providers/groq).
- Deepgram підхоплює `DEEPGRAM_API_KEY`, коли використовується `provider: "deepgram"`.
- Докладніше про налаштування Deepgram: [Deepgram (транскрибування аудіо)](/uk/providers/deepgram).
- Докладніше про налаштування Mistral: [Mistral](/uk/providers/mistral).
- SenseAudio підхоплює `SENSEAUDIO_API_KEY`, коли використовується `provider: "senseaudio"`.
- Докладніше про налаштування SenseAudio: [SenseAudio](/uk/providers/senseaudio).
- Провайдери аудіо можуть перевизначати `baseUrl`, `headers` і `providerOptions` через `tools.media.audio`.
- Типове обмеження розміру — 20 МБ (`tools.media.audio.maxBytes`). Аудіо, що перевищує розмір, пропускається для цієї моделі, і пробується наступний запис.
- Дуже малі/порожні аудіофайли менше ніж 1024 байти пропускаються до транскрибування провайдером/CLI.
- Типове значення `maxChars` для аудіо **не задане** (повний транскрипт). Задайте `tools.media.audio.maxChars` або `maxChars` для окремого запису, щоб обрізати результат.
- Типове значення OpenAI — `gpt-4o-mini-transcribe`; задайте `model: "gpt-4o-transcribe"` для вищої точності.
- Використовуйте `tools.media.audio.attachments` для обробки кількох голосових нотаток (`mode: "all"` + `maxAttachments`).
- Транскрипт доступний у шаблонах як `{{Transcript}}`.
- `tools.media.audio.echoTranscript` типово вимкнено; увімкніть його, щоб надсилати підтвердження транскрипту назад у вихідний чат перед обробкою агентом.
- `tools.media.audio.echoFormat` налаштовує текст такого відлуння (заповнювач: `{transcript}`).
- Вивід CLI в stdout обмежено (5 МБ); робіть вивід CLI лаконічним.
- `args` для CLI мають використовувати `{{MediaPath}}` для локального шляху до аудіофайлу. Застарілі заповнювачі `{input}` зі старіших конфігурацій `audio.transcription.command` приймаються як сумісний псевдонім і мігрують до `{{MediaPath}}`.

### Підтримка проксі через змінні середовища

Транскрибування аудіо через провайдера враховує стандартні змінні середовища для вихідного проксі:

- `HTTPS_PROXY`
- `HTTP_PROXY`
- `https_proxy`
- `http_proxy`

Якщо змінні середовища проксі не задані, використовується прямий вихід у мережу. Якщо конфігурація проксі некоректна, OpenClaw логує попередження і повертається до прямого отримання.

## Виявлення згадок у групах

Коли для групового чату встановлено `requireMention: true`, OpenClaw тепер транскрибує аудіо **до** перевірки згадок. Це дає змогу обробляти голосові нотатки, навіть якщо вони містять згадки.

**Як це працює:**

1. Якщо голосове повідомлення не має текстового тіла і для групи потрібні згадки, OpenClaw виконує "попереднє" транскрибування.
2. Транскрипт перевіряється на шаблони згадок (наприклад, `@BotName`, тригери-емодзі).
3. Якщо згадку знайдено, повідомлення проходить через повний конвеєр відповіді.
4. Транскрипт використовується для виявлення згадок, щоб голосові нотатки могли пройти перевірку на згадку.

**Резервна поведінка:**

- Якщо транскрибування під час попередньої перевірки завершується помилкою (тайм-аут, помилка API тощо), повідомлення обробляється на основі виявлення згадок лише в тексті.
- Це гарантує, що змішані повідомлення (текст + аудіо) ніколи не буде помилково відкинуто.

**Вимкнення для окремої Telegram-групи/теми:**

- Встановіть `channels.telegram.groups.<chatId>.disableAudioPreflight: true`, щоб пропустити перевірки згадок у попередньому транскрипті для цієї групи.
- Встановіть `channels.telegram.groups.<chatId>.topics.<threadId>.disableAudioPreflight`, щоб перевизначати для окремої теми (`true` — пропустити, `false` — примусово ввімкнути).
- Типове значення — `false` (попереднє транскрибування ввімкнене, коли збігаються умови перевірки згадок).

**Приклад:** Користувач надсилає голосову нотатку зі словами "Привіт @Claude, яка погода?" у групі Telegram з `requireMention: true`. Голосову нотатку транскрибують, виявляють згадку, і агент відповідає.

## Підводні камені

- Для правил області дії використовується принцип first-match wins. `chatType` нормалізується до `direct`, `group` або `room`.
- Переконайтеся, що ваш CLI завершується з кодом 0 і виводить звичайний текст; JSON треба обробити через `jq -r .text`.
- Для `parakeet-mlx`, якщо ви передаєте `--output-dir`, OpenClaw читає `<output-dir>/<media-basename>.txt`, коли `--output-format` має значення `txt` (або не вказано); для форматів виводу, відмінних від `txt`, використовується резервний розбір stdout.
- Тримайте тайм-аути розумними (`timeoutSeconds`, типово 60 с), щоб не блокувати чергу відповідей.
- Попереднє транскрибування обробляє лише **перше** аудіовкладення для виявлення згадок. Додаткове аудіо обробляється під час основної фази розуміння медіа.
