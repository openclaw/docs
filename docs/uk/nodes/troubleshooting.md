Використовуйте цю сторінку, коли Node відображається у стані, але інструменти Node не працюють.

## Послідовність команд

```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
openclaw channels status --probe
```

Потім виконайте перевірки для конкретного Node:

```bash
openclaw nodes status
openclaw nodes describe --node <idOrNameOrIp>
openclaw approvals get --node <idOrNameOrIp>
```

Ознаки справного стану:

- Node підключений і спарений для ролі `node`.
- Вивід `nodes describe` містить можливість, яку ви викликаєте.
- Схвалення виконання містять очікуваний режим і список дозволених команд.

## Вимоги до роботи на передньому плані

`canvas.*`, `camera.*` і `screen.*` працюють лише тоді, коли застосунок Node перебуває на передньому плані на пристроях iOS/Android.

Швидка перевірка та виправлення:

```bash
openclaw nodes describe --node <idOrNameOrIp>
openclaw nodes canvas snapshot --node <idOrNameOrIp>
openclaw logs --follow
```

Якщо з’являється `NODE_BACKGROUND_UNAVAILABLE`, переведіть застосунок Node на передній план і повторіть спробу.

## Матриця дозволів

| Можливість                   | iOS                                             | Android                                             | Застосунок Node для macOS                    | Типовий код помилки                           |
| ---------------------------- | ----------------------------------------------- | --------------------------------------------------- | --------------------------------------------- | --------------------------------------------- |
| `camera.snap`, `camera.clip` | Камера (+ мікрофон для аудіо у відеофрагменті) | Камера (+ мікрофон для аудіо у відеофрагменті)      | Камера (+ мікрофон для аудіо у відеофрагменті) | `*_PERMISSION_REQUIRED`                       |
| `screen.record`              | Запис екрана (+ мікрофон за потреби)            | Запит на захоплення екрана (+ мікрофон за потреби)  | Запис екрана                                  | `*_PERMISSION_REQUIRED`                       |
| `computer.act`               | не застосовується                               | не застосовується                                   | Доступність + запис екрана                    | `COMPUTER_DISABLED`, `ACCESSIBILITY_REQUIRED` |
| `location.get`               | Під час використання або завжди (залежить від режиму) | Геолокація на передньому/задньому плані залежно від режиму | Дозвіл на геолокацію                          | `LOCATION_PERMISSION_REQUIRED`                |
| `system.run`                 | не застосовується (шлях хоста Node)             | не застосовується (шлях хоста Node)                  | Потрібні схвалення виконання                  | `SYSTEM_RUN_DENIED`                           |

## Спарювання та схвалення

Успішність команди Node контролюють три окремі рівні:

1. **Спарювання пристрою**: чи може цей Node підключитися до Gateway?
2. **Політика команд Node у Gateway**: чи дозволено ідентифікатор команди RPC параметрами `gateway.nodes.allowCommands` / `denyCommands` і типовими налаштуваннями платформи?
3. **Схвалення виконання**: чи може цей Node локально виконати певну команду оболонки?

Спарювання Node є перевіркою ідентичності й довіри, а не механізмом схвалення окремих команд. Для `system.run` політика конкретного Node міститься у файлі схвалень виконання цього Node (`openclaw approvals get --node ...`), а не в записі про спарювання Gateway.

Швидкі перевірки:

```bash
openclaw devices list
openclaw nodes status
openclaw approvals get --node <idOrNameOrIp>
openclaw approvals allowlist add --node <idOrNameOrIp> "/usr/bin/uname"
```

- Немає спарювання: спочатку схваліть пристрій Node.
- У виводі `nodes describe` немає команди: перевірте політику команд Node у Gateway і чи справді Node оголосив цю команду під час підключення.
- Спарювання справне, але `system.run` завершується помилкою: виправте схвалення виконання або список дозволених команд на цьому Node.

Для запусків `host=node`, що потребують схвалення, Gateway також прив’язує виконання до підготовленого канонічного `systemRunPlan`. Якщо наступний викликач змінить команду, робочий каталог або метадані сеансу до пересилання схваленого запуску, Gateway відхилить запуск через невідповідність схваленню, а не довірятиме зміненим даним.

## Поширені коди помилок Node

| Код                                    | Значення                                                                                                                                                                                                 |
| -------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `NODE_BACKGROUND_UNAVAILABLE`          | Застосунок працює у фоновому режимі; переведіть його на передній план.                                                                                                                                    |
| `CAMERA_DISABLED`                      | Перемикач камери вимкнено в налаштуваннях Node.                                                                                                                                                           |
| `*_PERMISSION_REQUIRED`                | Дозвіл ОС відсутній або його відхилено.                                                                                                                                                                   |
| `LOCATION_DISABLED`                    | Режим геолокації вимкнено.                                                                                                                                                                                |
| `LOCATION_PERMISSION_REQUIRED`         | Запитаний режим геолокації не дозволено.                                                                                                                                                                  |
| `LOCATION_BACKGROUND_UNAVAILABLE`      | Застосунок працює у фоновому режимі, але надано лише дозвіл на використання геолокації під час роботи із застосунком.                                                                                       |
| `COMPUTER_DISABLED`                    | Увімкніть **Allow Computer Control** у застосунку macOS, а потім схваліть оновлення спарювання.                                                                                                            |
| `ACCESSIBILITY_REQUIRED`               | Надайте дозвіл Accessibility поточному пакету застосунку OpenClaw у macOS System Settings.                                                                                                                |
| `SYSTEM_RUN_DENIED: approval required` | Запит на виконання потребує явного схвалення.                                                                                                                                                             |
| `SYSTEM_RUN_DENIED: allowlist miss`    | Команду заблоковано режимом списку дозволених команд. На хостах Node під керуванням Windows форми з оболонкою-обгорткою, як-от `cmd.exe /c ...`, у режимі списку дозволених команд вважаються відсутніми в цьому списку, якщо їх не схвалено через процес запиту. |

## Цикл швидкого відновлення

```bash
openclaw nodes status
openclaw nodes describe --node <idOrNameOrIp>
openclaw approvals get --node <idOrNameOrIp>
openclaw logs --follow
```

Якщо проблему все ще не усунуто:

- Повторно схваліть спарювання пристрою.
- Повторно відкрийте застосунок Node на передньому плані.
- Повторно надайте дозволи ОС.
- Повторно створіть або скоригуйте політику схвалення виконання.

Для керування комп’ютером також переконайтеся, що агент із можливістю аналізувати зображення надає інструмент `computer`, команда `screen.snapshot` успішно виконується з дозволом на запис екрана, а `/phone status` показує потрібну тимчасову або постійну авторизацію Gateway. Запис `gateway.nodes.denyCommands` завжди має перевагу над `allowCommands`.
