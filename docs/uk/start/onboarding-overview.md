OpenClaw має адаптацію в терміналі та застосунку macOS. Обидва варіанти спочатку налаштовують інференс:
вони виявляють наявний доступ до ШІ, вимагають успішного виконання реального запиту й лише після цього запускають
Crestodian для завершення решти налаштувань. Якщо налаштований Gateway доступний,
а для його агента за замовчуванням уже налаштовано модель, адаптація пропускається
й одразу відкривається звичайний інтерфейс агента. Термінальний процес також пропонує повний класичний майстер
для детального налаштування.

## Який варіант вибрати?

|                  | Адаптація через CLI                  | Адаптація в застосунку macOS      |
| ---------------- | ------------------------------------ | --------------------------------- |
| **Платформи**    | macOS, Linux, Windows (напряму або через WSL2) | Лише macOS                        |
| **Інтерфейс**    | Налаштування інференсу, потім Crestodian | Налаштування інференсу, потім Crestodian |
| **Найкраще для** | Серверів, систем без графічного інтерфейсу, повного контролю | Настільних комп’ютерів Mac, візуального налаштування |
| **Автоматизація** | `--non-interactive` для скриптів    | Лише вручну                       |
| **Команда**      | `openclaw onboard`                   | Запустити застосунок              |

Більшості користувачів варто почати з **адаптації через CLI** — вона працює
всюди та надає найбільше контролю.

## Що налаштовує адаптація

Керований етап налаштування інференсу визначає лише:

1. **Постачальника моделі та автентифікацію** — виявлений доступ або перевірений ключ API
2. **Перевірений інференс** — реальне виконання запиту з ефективною
   моделлю агента за замовчуванням

Після успішного виконання цього запиту Crestodian може налаштувати робочий простір, Gateway,
службу Gateway, канали, агентів, плагіни та інші необов’язкові функції.

Класичний майстер CLI може додатково налаштувати:

1. **Канали** (необов’язково) — вбудовані та включені до комплекту канали чату, як-от
   Discord, Feishu, Google Chat, iMessage, Mattermost, Microsoft Teams,
   Telegram, WhatsApp та інші
2. **Розширені засоби керування Gateway** — віддалений режим, параметри мережі та варіанти фонової служби

## Адаптація через CLI

Виконайте в будь-якому терміналі:

```bash
openclaw onboard
```

Керований процес виявляє наявний доступ до ШІ, послідовно перевіряє варіанти реальними запитами,
у разі невдачі переходить до наступного та пропонує вручну ввести прихований ключ. Він зберігає
модель і облікові дані лише після успішного виконання запиту, а потім запускає Crestodian,
щоб налаштувати робочий простір, Gateway, канали, агентів, плагіни та інші
необов’язкові функції. До налаштування інференсу Crestodian не запускається; немає шляху
з пропуском ШІ або переходу до класичного майстра під час процесу. Якщо натомість потрібен
класичний майстер, вийдіть і виконайте `openclaw onboard --classic`.

Після успішної перевірки інференсу Crestodian може передати налаштування каналів майстру
в терміналі з прихованим введенням. Він не відкриває кероване або класичне налаштування постачальника;
щоб змінити постачальника моделі чи спосіб його автентифікації, вийдіть із Crestodian
і виконайте `openclaw onboard`.

Для детального налаштування моделі й автентифікації, каналів, Skills,
віддаленого Gateway або імпорту використовуйте `openclaw onboard --classic`. Додавання
`--install-daemon` також вибирає класичний процес і встановлює
фонову службу за один крок. Для діалогового налаштування та виправлення параметрів,
не пов’язаних з інференсом, використовуйте `openclaw crestodian`. `openclaw
onboard --modern` — це псевдонім сумісності, який використовує таку саму
перевірку реального інференсу.

Повний довідник: [Адаптація (CLI)](/uk/start/wizard)
Документація команди CLI: [`openclaw onboard`](/uk/cli/onboard)

## Адаптація в застосунку macOS

Відкрийте застосунок OpenClaw. Якщо налаштований локальний або віддалений Gateway
доступний і для агента за замовчуванням уже налаштовано модель, застосунок пропускає адаптацію
та Crestodian і відразу відкриває звичайний інтерфейс агента.

Для нового або не повністю налаштованого Gateway процес першого запуску виявляє наявний доступ
до ШІ (Claude Code, Codex або ключі API), перевіряє найкращий
варіант реальним запитом і зберігає його лише після отримання справжньої відповіді — автоматично
переходячи до резервних варіантів і пропонуючи перевірений крок ручного введення ключа API,
якщо нічого не знайдено. Конфіденційні облікові дані вводяться в прихованому вигляді.
Після успішної перевірки інференсу запускається Crestodian і допомагає налаштувати решту.

Gemini CLI залишається доступним для звичайних агентів після налаштування, але не
пропонується для цієї перевірки інференсу, оскільки не може гарантувати виконання
перевірного запиту без інструментів.

Повний довідник: [Адаптація (застосунок macOS)](/uk/start/onboarding)

## Власні постачальники або постачальники поза списком

Якщо вашого постачальника немає в списку, виконайте `openclaw onboard --classic`, виберіть
**Власний постачальник** і введіть:

- Сумісність кінцевої точки: сумісна з OpenAI (`/chat/completions`), сумісна з OpenAI Responses (`/responses`), сумісна з Anthropic (`/messages`) або невідома (перевіряє всі три варіанти й визначає автоматично)
- Базову URL-адресу та ключ API (ключ API необов’язковий, якщо кінцева точка його не вимагає)
- Ідентифікатор моделі та необов’язковий псевдонім моделі

Кілька власних кінцевих точок можуть співіснувати — кожна отримує власний ідентифікатор кінцевої точки.
