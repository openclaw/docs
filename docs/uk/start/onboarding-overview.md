OpenClaw має початкове налаштування в терміналі та застосунку macOS. Обидва варіанти спочатку налаштовують інференс:
вони виявляють наявний доступ до ШІ, вимагають успішного виконання реального запиту й лише після цього запускають
OpenClaw для решти налаштувань. Якщо доступний налаштований Gateway,
у якому для типового агента вже налаштовано модель, початкове налаштування пропускається
й одразу відкривається звичайний інтерфейс агента. Термінальний процес також пропонує повний класичний майстер
для детального налаштування.

## Який варіант слід використовувати?

|                | Початкове налаштування через CLI              | Початкове налаштування в застосунку macOS |
| -------------- | --------------------------------------------- | ----------------------------------------- |
| **Платформи**  | macOS, Linux, Windows (безпосередньо або WSL2) | Лише macOS                               |
| **Інтерфейс**  | Налаштування інференсу, потім OpenClaw        | Налаштування інференсу, потім OpenClaw   |
| **Найкраще для** | Серверів, систем без графічного інтерфейсу, повного контролю | Настільних комп’ютерів Mac, візуального налаштування |
| **Автоматизація** | `--non-interactive` для скриптів             | Лише вручну                              |
| **Команда**    | `openclaw onboard`                            | Запустити застосунок                     |

Більшості користувачів варто почати з **початкового налаштування через CLI** — воно працює
всюди та забезпечує найповніший контроль.

## Що налаштовує початкове налаштування

Керований етап інференсу налаштовує лише:

1. **Постачальник моделі та автентифікація** — виявлений доступ або підтверджений вхід
   до постачальника, ключ API чи токен
2. **Перевірений інференс** — реальне виконання запиту ефективною моделлю
   типового агента

Після успішного виконання запиту OpenClaw може налаштувати робочий простір, Gateway,
службу Gateway, канали, агентів, плагіни та інші необов’язкові функції.

Класичний майстер CLI також може налаштувати:

1. **Канали** (необов’язково) — вбудовані та включені до комплекту канали чату, як-от
   Discord, Feishu, Google Chat, iMessage, Mattermost, Microsoft Teams,
   Telegram, WhatsApp тощо
2. **Розширені параметри керування Gateway** — віддалений режим, параметри мережі та варіанти демона

## Початкове налаштування через CLI

Виконайте в будь-якому терміналі:

```bash
openclaw onboard
```

Керований процес виявляє наявний доступ до ШІ, по черзі перевіряє можливі варіанти реальними запитами
та в разі невдачі переходить до наступного. Якщо всі варіанти виявлення вичерпано, спочатку відображаються OpenAI,
Anthropic, xAI (Grok), Google та OpenRouter. **More…** містить
решту постачальників, згрупованих за групами постачальників, а в другому меню наведено регіони, плани й підтримувані
методи через браузер, пристрій, ключ API або токен. Модель
і облікові дані зберігаються лише після успішного виконання запиту, після чого запускається OpenClaw для
налаштування робочого простору, Gateway, каналів, агентів, плагінів та інших необов’язкових
функцій. **Skip for now** завершує процес без запуску OpenClaw. Перехід
до класичного процесу безпосередньо під час налаштування не передбачений; вийдіть і виконайте `openclaw onboard --classic`, якщо натомість потрібен
класичний майстер.

Після успішної перевірки інференсу OpenClaw може передати налаштування каналів майстру
в терміналі з маскованим введенням. Він не відкриває кероване чи класичне налаштування постачальника; вийдіть з OpenClaw і
виконайте `openclaw onboard`, щоб змінити постачальника моделі або спосіб його автентифікації.

Використовуйте `openclaw onboard --classic` для детального налаштування моделі й автентифікації, каналів, навичок,
віддаленого Gateway або імпорту. Додавання `--install-daemon` також вибирає
класичний процес і встановлює фонову службу за один крок. Використовуйте `openclaw
openclaw` для діалогового налаштування та виправлення, не пов’язаного з інференсом. `openclaw
onboard --modern` — це псевдонім сумісності, який використовує той самий шлюз
перевірки реального інференсу.

Повна довідка: [Початкове налаштування (CLI)](/uk/start/wizard)
Документація команди CLI: [`openclaw onboard`](/uk/cli/onboard)

## Початкове налаштування в застосунку macOS

Відкрийте застосунок OpenClaw. Якщо налаштований локальний або віддалений Gateway доступний,
а для типового агента вже налаштовано модель, застосунок пропускає початкове налаштування
та OpenClaw й одразу відкриває звичайний інтерфейс агента.

Для нового або не повністю налаштованого Gateway процес першого запуску виявляє наявний доступ
до ШІ (Claude Code, Codex або ключі API), перевіряє найкращий
варіант реальним запитом і зберігає його лише після отримання справжньої відповіді — автоматично переходячи до резервних варіантів і
пропонуючи перевірений ручний етап введення ключа API, якщо нічого не знайдено. Конфіденційні
облікові дані вводяться з маскуванням. Після успішної перевірки інференсу запускається OpenClaw і
допомагає налаштувати решту параметрів.

Gemini CLI залишається доступним для звичайних агентів після налаштування, але він не
пропонується для цього шлюзу перевірки інференсу, оскільки не може гарантувати виконання перевірочного запиту без інструментів.

Повна довідка: [Початкове налаштування (застосунок macOS)](/uk/start/onboarding)

## Власні або відсутні в списку постачальники

Якщо вашого постачальника немає в списку, виконайте `openclaw onboard --classic`, виберіть
**Custom Provider** і введіть:

- Сумісність кінцевої точки: сумісна з OpenAI (`/chat/completions`), сумісна з OpenAI Responses (`/responses`), сумісна з Anthropic (`/messages`) або невідома (перевіряються всі три варіанти й сумісність визначається автоматично)
- Базова URL-адреса та ключ API (ключ API необов’язковий, якщо кінцева точка його не вимагає)
- Ідентифікатор моделі та необов’язковий псевдонім моделі

Можна одночасно використовувати кілька власних кінцевих точок — кожна отримує окремий ідентифікатор кінцевої точки.
