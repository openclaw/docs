OpenClaw має два шляхи початкового налаштування. Обидва налаштовують автентифікацію, Gateway і
необов’язкові канали чату — вони відрізняються лише способом взаємодії з налаштуванням.

## Який шлях варто обрати?

|                | Початкове налаштування через CLI       | Початкове налаштування через застосунок macOS |
| -------------- | -------------------------------------- | ------------------------- |
| **Платформи**  | macOS, Linux, Windows (нативно або WSL2) | лише macOS                |
| **Інтерфейс**  | Майстер у терміналі                    | Покроковий інтерфейс у застосунку |
| **Найкраще для** | Сервери, без графічного інтерфейсу, повний контроль | Настільний Mac, візуальне налаштування |
| **Автоматизація** | `--non-interactive` для скриптів     | Лише вручну               |
| **Команда**    | `openclaw onboard`                     | Запустіть застосунок      |

Більшості користувачів варто почати з **початкового налаштування через CLI** — воно працює всюди й дає
вам найбільше контролю.

## Що налаштовує початкове налаштування

Незалежно від обраного шляху, початкове налаштування налаштовує:

1. **Постачальник моделі та автентифікація** — API-ключ, OAuth або токен налаштування для вибраного постачальника
2. **Робочий простір** — каталог для файлів агента, шаблонів початкового запуску та пам’яті
3. **Gateway** — порт, адреса прив’язки, режим автентифікації
4. **Канали** (необов’язково) — вбудовані й комплектні канали чату, як-от
   iMessage, Discord, Feishu, Google Chat, Mattermost, Microsoft Teams,
   Telegram, WhatsApp та інші
5. **Демон** (необов’язково) — фонова служба, щоб Gateway запускався автоматично

## Початкове налаштування через CLI

Виконайте в будь-якому терміналі:

```bash
openclaw onboard
```

Додайте `--install-daemon`, щоб також установити фонову службу за один крок.

Повний довідник: [Початкове налаштування (CLI)](/uk/start/wizard)
Документація команди CLI: [`openclaw onboard`](/uk/cli/onboard)

## Початкове налаштування через застосунок macOS

Відкрийте застосунок OpenClaw. Майстер першого запуску проведе вас тими самими кроками
через візуальний інтерфейс.

Повний довідник: [Початкове налаштування (застосунок macOS)](/uk/start/onboarding)

## Користувацькі або відсутні у списку постачальники

Якщо вашого постачальника немає в початковому налаштуванні, виберіть **Користувацький постачальник** і
введіть:

- Режим сумісності API (сумісний з OpenAI, сумісний з Anthropic або автовизначення)
- Базову URL-адресу та API-ключ
- ID моделі та необов’язковий псевдонім

Кілька користувацьких кінцевих точок можуть співіснувати — кожна отримує власний ID кінцевої точки.
