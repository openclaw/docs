Процес першого запуску застосунку macOS: виберіть, де працюватиме Gateway, підключіть
перевірений сервер ШІ, надайте дозволи й перейдіть до власної
процедури початкового налаштування агента.
Щоб дізнатися про початкове налаштування через CLI та порівняти обидва способи, див. [Огляд початкового налаштування](/uk/start/onboarding-overview).

Модель довіри безпеки:

- За замовчуванням OpenClaw є персональним агентом з однією межею довіри для надійного оператора.
- Спільні та багатокористувацькі конфігурації потребують посилення захисту: розділяйте межі довіри, надавайте мінімальний доступ до інструментів і дотримуйтеся рекомендацій у розділі [Безпека](/uk/gateway/security).
- Під час локального початкового налаштування для нових конфігурацій за замовчуванням установлюється `tools.profile: "coding"`, щоб нові інсталяції мали інструменти файлової системи та середовища виконання без необмеженого профілю `full`.
- Якщо ввімкнено хуки, вебхуки або інші канали надходження ненадійного вмісту, використовуйте потужну сучасну модель і зберігайте сувору політику інструментів та ізоляцію.

</Step>
<Step title="Локально чи віддалено">
<Frame>
<img src="/assets/macos-onboarding/04-choose-gateway.png" alt="" />
</Frame>

Де працює **Gateway**?

- **Цей Mac (лише локально):** початкове налаштування конфігурує автентифікацію та записує облікові дані локально.
- **Віддалено (через SSH/Tailnet):** початкове налаштування **не** конфігурує локальну автентифікацію;
  облікові дані вже мають бути на вузлі Gateway. Поле токена віддаленого Gateway
  зберігає токен, який застосунок macOS використовує для підключення до цього Gateway;
  наявні значення SecretRef у `gateway.remote.token` зберігаються, доки ви
  їх не заміните.
- **Налаштувати пізніше:** пропустіть налаштування та залиште застосунок неналаштованим.

<Tip>
**Порада щодо автентифікації Gateway:**

- За замовчуванням режим автентифікації Gateway має значення `token` навіть для прив’язок local loopback, тому локальні клієнти WS мають проходити автентифікацію.
- Значення `gateway.auth.mode: "none"` дозволяє підключатися будь-якому локальному процесу; використовуйте його лише на повністю надійних комп’ютерах.
- Для доступу з кількох комп’ютерів або прив’язок, відмінних від local loopback, використовуйте токен.

</Tip>
</Step>
<Step title="CLI">
  Локальне налаштування встановлює глобальний CLI `openclaw` через npm, pnpm або bun,
  віддаючи перевагу npm. Node залишається рекомендованим середовищем виконання для самого
  Gateway. Наявні сумісні інсталяції використовуються повторно.
</Step>
<Step title="Підключіть свій ШІ">
  Підключений Gateway, у якому вже налаштовано модель агента, повністю пропускає цю
  сторінку та відкриває звичайний інтерфейс агента. Налаштування Crestodian і постачальника
  виконується лише для нового або неповністю налаштованого Gateway.

Коли Gateway буде готовий, процедура початкового налаштування шукає вже наявний у вас доступ до ШІ:
вхід у Claude Code або Codex чи `OPENAI_API_KEY` /
`ANTHROPIC_API_KEY`. Найкращий варіант перевіряється реальним запитом на доповнення
та зберігається лише після отримання відповіді; якщо перевірка не вдається, застосунок автоматично
пробує наступний варіант і показує, чому попередній завершився невдало. Якщо знайдено
кілька варіантів, перед продовженням можна перемикатися між ними.

Gemini CLI залишається доступним для звичайних агентів після налаштування, але тут він
не пропонується, оскільки не може гарантувати виконання перевірочного запиту без інструментів.

Також можна ввійти через власну процедуру OAuth або сполучення пристрою від постачальника.
Вбудовані варіанти включають OpenAI/ChatGPT, OpenRouter, GitHub Copilot, Google
Gemini CLI, xAI, MiniMax Global і CN та Chutes. Список формується з
активних Plugin постачальників текстового виведення Gateway, а не з фіксованого списку застосунку,
тому інший постачальник може долучитися без додавання специфічного для нього коду macOS.

Засіб ручного вибору ключа або токена використовує той самий реєстр постачальників. Для кожного способу
постачальник надає початкову модель і конфігурацію; OpenClaw перевіряє
облікові дані тим самим тестом у реальному часі, перш ніж зберегти профіль автентифікації. Кнопка Next
залишається заблокованою, доки один із серверів не пройде перевірку, тому перший чат з агентом не може
початися без працездатного виведення. Після успішного завершення цієї перевірки Crestodian стає
доступним для налаштування решти робочого простору, Gateway, каналів та
інших необов’язкових функцій; пізніше він також доступний у розділі Settings → Crestodian.
</Step>
<Step title="Дозволи">

<Frame caption="Виберіть, які дозволи ви хочете надати OpenClaw">
<img src="/assets/macos-onboarding/05-permissions.png" alt="" />

Початкове налаштування запитує дозволи TCC для таких функцій: Automation (AppleScript), Notifications, Accessibility, Screen Recording, Microphone, Speech Recognition, Camera та Location.

</Step>
<Step title="Завершення">
  Після успішної перевірки виведення Crestodian керує рештою необов’язкового налаштування та може
  перевести вас до звичайного чату з агентом. Завершення процедури надання дозволів
  відкриває той самий чат; застосунок не створює робочий простір і не запускає окрему
  розмову для налаштування агента перед Crestodian. Опис дій, що відбуваються на вузлі Gateway
  під час першого справжнього ходу агента, див. у розділі
  [Початкове завантаження](/uk/start/bootstrapping).
</Step>
