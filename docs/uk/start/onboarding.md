Цей документ описує **поточний** процес налаштування під час першого запуску. Мета — забезпечити
плавний досвід "дня 0": вибрати, де працює Gateway, підключити автентифікацію, запустити
майстер і дозволити агенту самостійно ініціалізуватися.
Загальний огляд шляхів онбордингу див. у [Огляд онбордингу](/uk/start/onboarding-overview).

Модель довіри безпеки:

- За замовчуванням OpenClaw — це персональний агент: одна довірена межа оператора.
- Спільні/багатокористувацькі налаштування потребують жорсткого обмеження (розділіть межі довіри, мінімізуйте доступ до інструментів і дотримуйтеся розділу [Безпека](/uk/gateway/security)).
- Локальний онбординг тепер за замовчуванням встановлює для нових конфігурацій `tools.profile: "coding"`, щоб нові локальні налаштування зберігали інструменти файлової системи/середовища виконання без примусового використання необмеженого профілю `full`.
- Якщо ввімкнено hooks/webhooks або інші канали недовіреного вмісту, використовуйте сильний сучасний рівень моделі та підтримуйте сувору політику інструментів/пісочниці.

</Step>
<Step title="Локально чи віддалено">
<Frame>
<img src="/assets/macos-onboarding/04-choose-gateway.png" alt="" />
</Frame>

Де працює **Gateway**?

- **Цей Mac (лише локально):** онбординг може налаштувати автентифікацію та записати облікові дані
  локально.
- **Віддалено (через SSH/Tailnet):** онбординг **не** налаштовує локальну автентифікацію;
  облікові дані мають існувати на хості Gateway. Поле токена віддаленого Gateway
  зберігає токен, який macOS-застосунок використовує для підключення до цього Gateway; наявні
  не відкритотекстові значення `gateway.remote.token` зберігаються, доки ви їх не заміните.
- **Налаштувати пізніше:** пропустити налаштування й залишити застосунок неналаштованим.

<Tip>
**Порада щодо автентифікації Gateway:**

- Майстер тепер генерує **токен** навіть для loopback, тому локальні WS-клієнти мають автентифікуватися.
- Якщо вимкнути автентифікацію, будь-який локальний процес зможе підключитися; використовуйте це лише на повністю довірених машинах.
- Використовуйте **токен** для доступу з кількох машин або прив’язок не до loopback.

</Tip>
</Step>
<Step title="Дозволи">
<Frame caption="Виберіть, які дозволи ви хочете надати OpenClaw">

Онбординг запитує дозволи TCC, потрібні для:

- Автоматизації (AppleScript)
- Сповіщень
- Доступності
- Запису екрана
- Мікрофона
- Розпізнавання мовлення
- Камери
- Місцезнаходження

</Step>
<Step title="CLI">
  <Info>Цей крок необов’язковий</Info>
  Застосунок може встановити глобальний CLI `openclaw` через npm, pnpm або bun.
  Спершу він віддає перевагу npm, потім pnpm, а потім bun, якщо це єдиний виявлений
  менеджер пакетів. Для середовища виконання Gateway рекомендованим шляхом залишається Node.
</Step>
<Step title="Онбординг-чат (окремий сеанс)">
  Після налаштування застосунок відкриває окремий сеанс онбординг-чату, щоб агент міг
  представитися й підказати наступні кроки. Це відокремлює підказки першого запуску
  від вашої звичайної розмови. Див. [Ініціалізація](/uk/start/bootstrapping), щоб дізнатися,
  що відбувається на хості Gateway під час першого запуску агента.
</Step>
