CLI-онбординг — **рекомендований** шлях налаштування OpenClaw у терміналі на
macOS, Linux або Windows. Користувачі робочого столу Windows також можуть почати з
[Windows Hub](/uk/platforms/windows).
Він налаштовує локальний Gateway або підключення до віддаленого Gateway, а також канали, Skills
і стандартні параметри робочого простору в одному керованому потоці.

```bash
openclaw onboard
```

QuickStart зазвичай займає лише кілька хвилин, але повний онбординг може тривати довше,
коли вхід до провайдера, сполучення каналу, встановлення демона, мережеві завантаження,
Skills або необов'язкові Plugin-и потребують додаткового налаштування. Майстер показує цей часовий план
заздалегідь, а необов'язкові кроки можна пропустити й повернутися до них пізніше за допомогою
`openclaw configure`.

## Локаль

Майстер CLI локалізує фіксований текст онбордингу. Він визначає локаль з
`OPENCLAW_LOCALE`, потім `LC_ALL`, потім `LC_MESSAGES`, потім `LANG`, і
повертається до англійської. Підтримувані локалі майстра: `en`, `zh-CN` і `zh-TW`.

```bash
OPENCLAW_LOCALE=zh-CN openclaw onboard
```

Назви й стабільні ідентифікатори залишаються дослівними: `OpenClaw`, `Gateway`, `Tailscale`,
команди, ключі конфігурації, URL-адреси, ID провайдерів, ID моделей і мітки Plugin/каналів
не перекладаються.

<Info>
Найшвидший перший чат: відкрийте Control UI (налаштування каналу не потрібне). Запустіть
`openclaw dashboard` і спілкуйтеся в браузері. Документація: [Dashboard](/uk/web/dashboard).
</Info>

Щоб переналаштувати пізніше:

```bash
openclaw configure
openclaw agents add <name>
```

<Note>
`--json` не означає неінтерактивний режим. Для скриптів використовуйте `--non-interactive`.
</Note>

<Tip>
CLI-онбординг містить крок вебпошуку, де можна вибрати провайдера,
наприклад Brave, DuckDuckGo, Exa, Firecrawl, Gemini, Grok, Kimi, MiniMax Search,
Ollama Web Search, Perplexity, SearXNG або Tavily. Деякі провайдери потребують
API-ключ, тоді як інші працюють без ключа. Це також можна налаштувати пізніше за допомогою
`openclaw configure --section web`. Документація: [Вебінструменти](/uk/tools/web).
</Tip>

## QuickStart проти Advanced

Онбординг починається з вибору **QuickStart** (стандартні параметри) або **Advanced** (повний контроль).

<Tabs>
  <Tab title="QuickStart (стандартні параметри)">
    - Локальний gateway (loopback)
    - Стандартний робочий простір (або наявний робочий простір)
    - Порт Gateway **18789**
    - Автентифікація Gateway **Token** (автоматично згенерований, навіть на loopback)
    - Стандартна політика інструментів для нових локальних налаштувань: `tools.profile: "coding"` (наявний явно заданий профіль зберігається)
    - Стандартна ізоляція DM: локальний онбординг записує `session.dmScope: "per-channel-peer"`, якщо значення не задане. Подробиці: [Довідник налаштування CLI](/uk/start/wizard-cli-reference#outputs-and-internals)
    - Експонування Tailscale **вимкнено**
    - DM у Telegram + WhatsApp стандартно працюють за **списком дозволених** (вам буде запропоновано ввести номер телефону)

  </Tab>
  <Tab title="Advanced (повний контроль)">
    - Відкриває кожен крок (режим, робочий простір, gateway, канали, демон, Skills).

  </Tab>
</Tabs>

## Що налаштовує онбординг

**Локальний режим (стандартний)** проводить вас цими кроками:

1. **Модель/автентифікація** — виберіть будь-який підтримуваний провайдер/потік автентифікації (API-ключ, OAuth або специфічна для провайдера ручна автентифікація), зокрема Custom Provider
   (сумісний з OpenAI, сумісний з Anthropic або Unknown з автовизначенням). Виберіть стандартну модель.
   Примітка з безпеки: якщо цей агент запускатиме інструменти або оброблятиме вміст webhook/hooks, віддавайте перевагу найсильнішій доступній моделі найновішого покоління й тримайте політику інструментів суворою. Слабші/старіші рівні легше піддаються prompt injection.
   Для неінтерактивних запусків `--secret-input-mode ref` зберігає посилання на основі змінних середовища в профілях автентифікації замість значень API-ключів відкритим текстом.
   У неінтерактивному режимі `ref` змінна середовища провайдера має бути задана; передавання inline-прапорів ключа без цієї змінної середовища швидко завершується помилкою.
   В інтерактивних запусках вибір режиму посилання на секрет дає змогу вказати або змінну середовища, або налаштоване посилання провайдера (`file` чи `exec`), зі швидкою попередньою перевіркою перед збереженням.
   Для Anthropic інтерактивний onboarding/configure пропонує **Anthropic Claude CLI** як бажаний локальний шлях і **Anthropic API key** як рекомендований production-шлях. Anthropic setup-token також залишається доступним як підтримуваний шлях автентифікації через token.
2. **Робочий простір** — розташування файлів агента (стандартно `~/.openclaw/workspace`). Створює початкові bootstrap-файли.
3. **Gateway** — порт, адреса прив'язки, режим автентифікації, експонування Tailscale.
   В інтерактивному режимі token виберіть стандартне зберігання token відкритим текстом або перейдіть на SecretRef.
   Неінтерактивний шлях token SecretRef: `--gateway-token-ref-env <ENV_VAR>`.
4. **Канали** — вбудовані й офіційні Plugin-канали чату, як-от iMessage, Discord, Feishu, Google Chat, Mattermost, Microsoft Teams, QQ Bot, Signal, Slack, Telegram, WhatsApp та інші.
5. **Демон** — встановлює LaunchAgent (macOS), користувацький systemd unit (Linux/WSL2) або нативне Windows Scheduled Task з резервним варіантом Startup-folder для кожного користувача.
   Якщо token-автентифікація потребує token і `gateway.auth.token` керується SecretRef, встановлення демона перевіряє його, але не зберігає розв'язаний token у метаданих середовища служби supervisor.
   Якщо token-автентифікація потребує token, а налаштований token SecretRef не розв'язується, встановлення демона блокується з практичними вказівками.
   Якщо налаштовано і `gateway.auth.token`, і `gateway.auth.password`, а `gateway.auth.mode` не задано, встановлення демона блокується, доки режим не буде задано явно.
6. **Перевірка працездатності** — запускає Gateway і перевіряє, що він працює.
7. **Skills** — встановлює рекомендовані Skills і необов'язкові залежності.

<Note>
Повторний запуск онбордингу **не** стирає нічого, якщо ви явно не виберете **Reset** (або не передасте `--reset`).
CLI `--reset` стандартно охоплює конфігурацію, облікові дані й сесії; використовуйте `--reset-scope full`, щоб включити робочий простір.
Якщо конфігурація недійсна або містить застарілі ключі, онбординг попросить спочатку запустити `openclaw doctor`.
</Note>

**Віддалений режим** лише налаштовує локальний клієнт для підключення до Gateway в іншому місці.
Він **не** встановлює й не змінює нічого на віддаленому хості.

## Додати іншого агента

Використовуйте `openclaw agents add <name>`, щоб створити окремого агента з власним робочим простором,
сесіями й профілями автентифікації. Запуск без `--workspace` відкриває онбординг.

Що він задає:

- `agents.list[].name`
- `agents.list[].workspace`
- `agents.list[].agentDir`

Примітки:

- Стандартні робочі простори відповідають `~/.openclaw/workspace-<agentId>`.
- Додайте `bindings`, щоб маршрутизувати вхідні повідомлення (онбординг може це зробити).
- Неінтерактивні прапорці: `--model`, `--agent-dir`, `--bind`, `--non-interactive`.
