OpenClaw підключається до Zalo ClawBot через указаний у каталозі зовнішній плагін `@zalo-platforms/openclaw-zaloclawbot`. Для входу використовується QR-код Zalo Mini App; ідентифікатор плагіна в конфігурації — `openclaw-zaloclawbot`.

## Сумісність

| Версія плагіна | Версія OpenClaw | dist-tag npm | Стан          |
| -------------- | ---------------- | ------------ | ------------- |
| 0.1.4          | >=2026.4.10      | `latest`     | Активний / Бета |

## Передумови

- Node.js >= 22
- Установлений [OpenClaw](https://docs.openclaw.ai/install) (доступний CLI `openclaw`)
- Обліковий запис Zalo на мобільному пристрої для сканування QR-коду входу

## Установлення через початкове налаштування (рекомендовано)

```bash
openclaw onboard
```

Виберіть **Zalo ClawBot** у меню каналів. Майстер установлює плагін з офіційного каталогу (з перевіркою цілісності), відображає QR-код входу в терміналі та завершує налаштування каналу після його сканування в застосунку Zalo.

## Ручне встановлення

Щоб додати канал до вже налаштованого Gateway:

### 1. Установіть плагін

```bash
openclaw plugins install "@zalo-platforms/openclaw-zaloclawbot@0.1.4"
```

Використовуйте точну зафіксовану версію, щоб під час установлення OpenClaw перевірив пакет за хешем цілісності з каталогу.

### 2. Увімкніть плагін у конфігурації

```bash
openclaw config set plugins.entries.openclaw-zaloclawbot.enabled true
```

### 3. Створіть QR-код і ввійдіть

```bash
openclaw channels login --channel openclaw-zaloclawbot
```

Відскануйте показаний у терміналі QR-код мобільним застосунком Zalo, прийміть Умови використання в Zalo Mini App і авторизуйте сеанс.

### 4. Перезапустіть Gateway

```bash
openclaw gateway restart
```

## Принцип роботи

На відміну від стандартного каналу Zalo, для якого потрібно зареєструвати власний офіційний обліковий запис Zalo (OA) і налаштувати статичні облікові дані розробника, Zalo ClawBot — це **персональний помічник, прив’язаний до власника**, у спільній офіційній інфраструктурі:

1. **Початкове налаштування:** QR-код веде до Zalo Mini App, який безпосередньо прив’язує новоствореного приватного бота в межах спільного офіційного OA до вашого ідентифікатора користувача Zalo.
2. **Конфіденційність завдяки прив’язці до власника:** бот взаємодіє лише зі своїм власником. Повідомлення від інших користувачів відкидаються на рівні платформи.
3. **Офіційний шлях API:** плагін використовує API Zalo Bot Platform, а не автоматизацію браузера чи вебсеансу.

## Внутрішня реалізація

Плагін взаємодіє із Zalo через постійний цикл тривалого опитування (`getUpdates`). Webhook за замовчуванням вимкнено для локальних запусків Gateway із настільного середовища або термінала. Повідомлення обробляються на боці клієнта й зіставляються з локальним середовищем виконання агента.

Плагін зберігає облікові дані бота в каталозі стану OpenClaw. Вважайте цей каталог конфіденційним і застосовуйте до нього ту саму політику контролю доступу та резервного копіювання, що й до решти стану OpenClaw.

Середовище виконання цього плагіна повністю міститься в зовнішньому пакеті `@zalo-platforms/openclaw-zaloclawbot`; наведені нижче відомості про поведінку, що виходять за межі встановлення та конфігурації, подано зі слів супровідників плагіна й не перевірено за вихідним кодом ядра OpenClaw.

## Усунення несправностей

- **Час очікування входу за QR-кодом минув:** з міркувань безпеки строк дії токена входу (`zbsk`) становить 5 хвилин. Якщо строк дії QR-коду минув до сканування, повторно виконайте команду входу, щоб створити новий.
- **Не вдається завантажити Gateway:** переконайтеся, що версія OpenClaw на вашому хості — `2026.4.10` або новіша. Старіші версії не підтримують реєстр установлення зовнішніх npm-плагінів, потрібний для цього ідентифікатора.
