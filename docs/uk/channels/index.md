OpenClaw може спілкуватися з вами в будь-якому чат-застосунку, яким ви вже користуєтеся. Кожен канал підключається через Gateway.
Текст підтримується всюди; підтримка медіафайлів і реакцій залежить від каналу.

iMessage, Telegram та інтерфейс WebChat постачаються разом з основним пакетом. Канали з позначкою
«офіційний плагін» установлюються однією командою (`openclaw plugins install @openclaw/<id>`)
або на вимогу під час виконання `openclaw onboard` / `openclaw channels add`, після чого потрібно
перезапустити Gateway. Канали з позначкою «зовнішній плагін» підтримуються поза репозиторієм OpenClaw.

## Підтримувані канали

- [Discord](/uk/channels/discord) — Discord Bot API + Gateway; підтримує сервери, канали та приватні повідомлення (офіційний плагін).
- [Feishu](/uk/channels/feishu) — бот Feishu/Lark через WebSocket (офіційний плагін).
- [Google Chat](/uk/channels/googlechat) — застосунок Google Chat API через HTTP-вебхук (офіційний плагін).
- [iMessage](/uk/channels/imessage) — входить до основного пакета. Нативна інтеграція з macOS через міст `imsg` на комп’ютері Mac із виконаним входом (або через обгортку SSH, якщо Gateway працює деінде), зокрема приватні дії API для відповідей, реакцій Tapback, ефектів, вкладень і керування групами.
- [IRC](/uk/channels/irc) — класичні сервери IRC; канали та приватні повідомлення з контролем сполучення й списків дозволених адресатів (офіційний плагін).
- [LINE](/uk/channels/line) — бот LINE Messaging API (офіційний плагін).
- [Matrix](/uk/channels/matrix) — протокол Matrix (офіційний плагін).
- [Mattermost](/uk/channels/mattermost) — Bot API + WebSocket; канали, групи та приватні повідомлення (офіційний плагін).
- [Microsoft Teams](/uk/channels/msteams) — Bot Framework; підтримка для підприємств (офіційний плагін).
- [Nextcloud Talk](/uk/channels/nextcloud-talk) — самостійно розміщуваний чат через Nextcloud Talk (офіційний плагін).
- [Nostr](/uk/channels/nostr) — децентралізовані приватні повідомлення через NIP-04 (офіційний плагін).
- [QQ Bot](/uk/channels/qqbot) — QQ Bot API; приватні й групові чати та мультимедійний вміст (офіційний плагін).
- [Raft](/uk/channels/raft) — міст пробудження Raft CLI для співпраці людей і агентів (офіційний плагін).
- [Signal](/uk/channels/signal) — signal-cli; орієнтований на конфіденційність (офіційний плагін).
- [Slack](/uk/channels/slack) — Bolt SDK; застосунки робочих просторів (офіційний плагін).
- [SMS](/uk/channels/sms) — SMS на базі Twilio через вебхук Gateway (офіційний плагін).
- [Synology Chat](/uk/channels/synology-chat) — Synology NAS Chat через вихідні та вхідні вебхуки (офіційний плагін).
- [Telegram](/uk/channels/telegram) — входить до основного пакета. Bot API через grammY; підтримує групи.
- [Tlon](/uk/channels/tlon) — месенджер на базі Urbit (офіційний плагін).
- [Twitch](/uk/channels/twitch) — чат Twitch через з’єднання IRC (офіційний плагін).
- [Голосові виклики](/uk/plugins/voice-call) — телефонія через Plivo, Telnyx або Twilio (офіційний плагін).
- [WebChat](/uk/web/webchat) — входить до основного пакета. Інтерфейс Gateway WebChat через WebSocket.
- [WeChat](/uk/channels/wechat) — бот Tencent iLink із входом за QR-кодом; лише приватні чати (зовнішній плагін).
- [WhatsApp](/uk/channels/whatsapp) — найпопулярніший; використовує Baileys і потребує сполучення за QR-кодом (офіційний плагін).
- [Yuanbao](/uk/channels/yuanbao) — бот Tencent Yuanbao (зовнішній плагін).
- [Zalo](/uk/channels/zalo) — Zalo Bot API; популярний у В’єтнамі месенджер (офіційний плагін).
- [Zalo ClawBot](/uk/channels/zaloclawbot) — персональний помічник Zalo із входом за QR-кодом; прив’язаний до власника (зовнішній плагін).
- [Zalo Personal](/uk/channels/zalouser) — особистий обліковий запис Zalo із входом за QR-кодом (офіційний плагін).

## Примітки щодо доставлення

- Відповіді Telegram, які містять синтаксис зображень Markdown, наприклад `![alt](url)`,
  за можливості перетворюються на відповіді з медіафайлами на завершальному етапі вихідного доставлення.
- Приватні повідомлення Slack із кількома учасниками маршрутизуються як групові чати, тому до розмов MPIM
  застосовуються групова політика, правила згадок і правила групових сеансів.
- Налаштування WhatsApp виконується шляхом установлення на вимогу: початкове налаштування може показати
  процес конфігурації до встановлення пакета плагіна, а Gateway завантажує зовнішній
  плагін ClawHub/npm лише тоді, коли канал фактично активний.
- Канали, які приймають вхідні повідомлення, створені ботами, можуть використовувати спільний
  [захист від зациклення ботів](/uk/channels/bot-loop-protection), щоб пари ботів не
  відповідали один одному нескінченно.
- Підтримувані постійно активні кімнати можуть використовувати [фонові події кімнати](/uk/channels/ambient-room-events),
  щоб розмова в кімнаті без згадок ставала ненав’язливим контекстом, якщо агент не надсилає повідомлення
  за допомогою інструмента `message`.

## Примітки

- Канали можуть працювати одночасно; налаштуйте кілька каналів, і OpenClaw виконуватиме маршрутизацію для кожного чату.
- Найшвидше зазвичай налаштувати **Telegram** (простий токен бота, установлення плагіна не потрібне). WhatsApp
  потребує сполучення за QR-кодом і зберігає більше стану на диску.
- Поведінка груп залежить від каналу; див. [Групи](/uk/channels/groups).
- Для безпеки застосовуються сполучення приватних повідомлень і списки дозволених адресатів; див. [Безпека](/uk/gateway/security).
- Усунення несправностей: [Усунення несправностей каналів](/uk/channels/troubleshooting).
- Постачальники моделей задокументовані окремо; див. [Постачальники моделей](/uk/providers/models).
