OpenClaw може спілкуватися з вами в будь-якому застосунку для чатів, яким ви вже користуєтеся. Кожен канал підключається через Gateway.
Текст підтримується всюди; підтримка медіафайлів і реакцій залежить від каналу.

iMessage, Telegram та інтерфейс WebChat входять до основного встановлення. Канали з позначкою
«офіційний плагін» установлюються однією командою (`openclaw plugins install @openclaw/<id>`)
або за потреби під час `openclaw onboard` / `openclaw channels add`, після чого потрібно
перезапустити Gateway. Канали з позначкою «зовнішній плагін» супроводжуються поза репозиторієм OpenClaw.

## Підтримувані канали

- [Discord](/uk/channels/discord) — Discord Bot API + Gateway; підтримує сервери, канали та особисті повідомлення (офіційний плагін).
- [Feishu](/uk/channels/feishu) — бот Feishu/Lark через WebSocket (офіційний плагін).
- [Google Chat](/uk/channels/googlechat) — застосунок Google Chat API через HTTP Webhook (офіційний плагін).
- [iMessage](/uk/channels/imessage) — входить до основного встановлення. Нативна інтеграція з macOS через міст `imsg` на Mac із виконаним входом (або через обгортку SSH, коли Gateway працює деінде), включно з діями приватного API для відповідей, реакцій Tapback, ефектів, вкладень і керування групами.
- [IRC](/uk/channels/irc) — класичні сервери IRC; канали й особисті повідомлення з керуванням сполученням і списком дозволених користувачів (офіційний плагін).
- [LINE](/uk/channels/line) — бот LINE Messaging API (офіційний плагін).
- [Matrix](/uk/channels/matrix) — протокол Matrix (офіційний плагін).
- [Mattermost](/uk/channels/mattermost) — Bot API + WebSocket; канали, групи та особисті повідомлення (офіційний плагін).
- [Microsoft Teams](/uk/channels/msteams) — Bot Framework; підтримка корпоративного рівня (офіційний плагін).
- [Nextcloud Talk](/uk/channels/nextcloud-talk) — самостійно розміщуваний чат через Nextcloud Talk (офіційний плагін).
- [Nostr](/uk/channels/nostr) — децентралізовані особисті повідомлення через NIP-04 (офіційний плагін).
- [QQ Bot](/uk/channels/qqbot) — QQ Bot API; приватні й групові чати та мультимедійний вміст (офіційний плагін).
- [Reef](/uk/channels/reef) — захищений наскрізним шифруванням обмін повідомленнями між агентами OpenClaw різних людей (вбудований плагін).
- [Raft](/uk/channels/raft) — міст пробудження Raft CLI для співпраці людей і агентів (офіційний плагін).
- [Signal](/uk/channels/signal) — signal-cli; орієнтований на конфіденційність (офіційний плагін).
- [Slack](/uk/channels/slack) — Bolt SDK; застосунки робочих просторів (офіційний плагін).
- [SMS](/uk/channels/sms) — SMS на базі Twilio через Webhook Gateway (офіційний плагін).
- [Synology Chat](/uk/channels/synology-chat) — Synology NAS Chat через вихідні та вхідні вебхуки (офіційний плагін).
- [Telegram](/uk/channels/telegram) — входить до основного встановлення. Bot API через grammY; підтримує групи.
- [Tlon](/uk/channels/tlon) — месенджер на базі Urbit (офіційний плагін).
- [Twitch](/uk/channels/twitch) — чат Twitch через з’єднання IRC (офіційний плагін).
- [Голосовий виклик](/uk/plugins/voice-call) — телефонний зв’язок через Plivo, Telnyx або Twilio (офіційний плагін).
- [WebChat](/uk/web/webchat) — входить до основного встановлення. Інтерфейс Gateway WebChat через WebSocket.
- [WeChat](/uk/channels/wechat) — бот Tencent iLink із входом за QR-кодом; лише приватні чати (зовнішній плагін).
- [WhatsApp](/uk/channels/whatsapp) — найпопулярніший; використовує Baileys і потребує сполучення за QR-кодом (офіційний плагін).
- [Yuanbao](/uk/channels/yuanbao) — бот Tencent Yuanbao (зовнішній плагін).
- [Zalo](/uk/channels/zalo) — Zalo Bot API; популярний месенджер у В’єтнамі (офіційний плагін).
- [Zalo ClawBot](/uk/channels/zaloclawbot) — персональний помічник Zalo із входом за QR-кодом; прив’язаний до власника (зовнішній плагін).
- [Zalo Personal](/uk/channels/zalouser) — особистий обліковий запис Zalo із входом за QR-кодом (офіційний плагін).

## Примітки щодо доставлення

- Відповіді Telegram, що містять синтаксис зображень Markdown, наприклад `![alt](url)`,
  за можливості перетворюються на відповіді з медіафайлами на завершальному етапі надсилання.
- Особисті повідомлення Slack за участю кількох людей спрямовуються як групові чати, тому до розмов MPIM
  застосовуються групова політика, поведінка згадок і правила групових сеансів.
- Налаштування WhatsApp виконується встановленням за потреби: початкове налаштування може показати процес конфігурації до
  встановлення пакета плагіна, а Gateway завантажує зовнішній
  плагін ClawHub/npm лише тоді, коли канал фактично активний.
- Канали, які приймають створені ботами вхідні повідомлення, можуть використовувати спільний
  [захист від зациклення ботів](/uk/channels/bot-loop-protection), щоб пари ботів не
  відповідали один одному нескінченно.
- Підтримувані постійно активні кімнати можуть використовувати [фонові події кімнат](/uk/channels/ambient-room-events),
  щоб розмови в кімнаті без згадування агента ставали тихим контекстом, якщо агент не надсилає повідомлення за допомогою
  інструмента `message`.

## Примітки

- Канали можуть працювати одночасно; налаштуйте кілька каналів, і OpenClaw виконуватиме маршрутизацію для кожного чату.
- Найшвидше зазвичай налаштовується **Telegram** (простий токен бота, встановлення плагіна не потрібне). WhatsApp
  потребує сполучення за QR-кодом і зберігає більше стану на диску.
- Поведінка груп залежить від каналу; див. [Групи](/uk/channels/groups).
- Сполучення особистих повідомлень і списки дозволених користувачів застосовуються задля безпеки; див. [Безпека](/uk/gateway/security).
- Усунення несправностей: [Усунення несправностей каналів](/uk/channels/troubleshooting).
- Постачальників моделей описано окремо; див. [Постачальники моделей](/uk/providers/models).
