Використовуйте цю сторінку, коли канал під’єднується, але поведінка неправильна.

## Драбина команд

Спершу виконайте їх по порядку:

```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
openclaw channels status --probe
```

Справний базовий стан:

- `Runtime: running`
- `Connectivity probe: ok`
- `Capability: read-only`, `write-capable` або `admin-capable`
- Перевірка каналу показує, що транспорт під’єднано, а де підтримується, `works` або `audit ok`

## WhatsApp

### Ознаки збоїв WhatsApp

| Симптом                         | Найшвидша перевірка                                | Виправлення                                             |
| ------------------------------- | -------------------------------------------------- | ------------------------------------------------------- |
| Під’єднано, але немає відповідей у DM | `openclaw pairing list whatsapp`                   | Схваліть відправника або змініть політику/allowlist DM. |
| Повідомлення групи ігноруються  | Перевірте `requireMention` + шаблони згадок у конфігу | Згадайте бота або послабте політику згадок для цієї групи. |
| Вхід через QR завершується тайм-аутом із 408 | Перевірте змінні середовища Gateway `HTTPS_PROXY` / `HTTP_PROXY` | Налаштуйте доступний проксі; використовуйте `NO_PROXY` лише для обходів. |
| Випадкові цикли від’єднання/повторного входу | `openclaw channels status --probe` + журнали       | Увійдіть повторно й перевірте справність каталогу облікових даних. |

Повне усунення несправностей: [усунення несправностей WhatsApp](/uk/channels/whatsapp#troubleshooting)

## Telegram

### Ознаки збоїв Telegram

| Симптом                              | Найшвидша перевірка                              | Виправлення                                                                                                                |
| ------------------------------------ | ------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------- |
| `/start`, але немає придатного потоку відповідей | `openclaw pairing list telegram`                 | Схваліть сполучення або змініть політику DM.                                                                               |
| Бот онлайн, але група мовчить        | Перевірте вимогу згадки та режим приватності бота | Вимкніть режим приватності для видимості в групі або згадайте бота.                                                        |
| Збої надсилання з мережевими помилками | Перегляньте журнали щодо збоїв викликів Telegram API | Виправте маршрутизацію DNS/IPv6/проксі до `api.telegram.org`.                                                              |
| Запуск повідомляє `getMe returned 401` | Перевірте налаштоване джерело токена             | Повторно скопіюйте або згенеруйте токен BotFather і оновіть `botToken`, `tokenFile` або `TELEGRAM_BOT_TOKEN` стандартного облікового запису. |
| Опитування зависає або повільно перепід’єднується | `openclaw logs --follow` для діагностики опитування | Оновіть; якщо перезапуски є хибними спрацьовуваннями, налаштуйте `pollingStallThresholdMs`. Постійні зависання все ще вказують на проксі/DNS/IPv6. |
| `setMyCommands` відхилено під час запуску | Перегляньте журнали щодо `BOT_COMMANDS_TOO_MUCH` | Зменште кількість команд Telegram від Plugin/Skills/користувача або вимкніть нативні меню.                                |
| Після оновлення allowlist блокує вас | `openclaw security audit` і allowlist у конфігу  | Виконайте `openclaw doctor --fix` або замініть `@username` на числові ID відправників.                                     |

Повне усунення несправностей: [усунення несправностей Telegram](/uk/channels/telegram#troubleshooting)

## Matrix

### Ознаки збоїв Matrix

| Симптом                             | Найшвидша перевірка                    | Виправлення                                                               |
| ----------------------------------- | -------------------------------------- | ------------------------------------------------------------------------- |
| Увійшов, але ігнорує повідомлення кімнати | `openclaw channels status --probe`     | Перевірте `groupPolicy`, allowlist кімнат і фільтрацію згадок.            |
| DM не обробляються                  | `openclaw pairing list matrix`         | Схваліть відправника або скоригуйте політику DM.                          |
| Зашифровані кімнати не працюють     | `openclaw matrix verify status`        | Повторно перевірте пристрій, потім перевірте `openclaw matrix verify backup status`. |
| Відновлення резервної копії очікує/зламане | `openclaw matrix verify backup status` | Виконайте `openclaw matrix verify backup restore` або повторіть із recovery key. |
| Cross-signing/bootstrap виглядає неправильно | `openclaw matrix verify bootstrap`     | Виправте secret storage, cross-signing і стан резервної копії за один прохід. |

Повне налаштування та конфігурація: [Matrix](/uk/channels/matrix)
