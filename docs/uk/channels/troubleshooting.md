Використовуйте цю сторінку, коли канал підключається, але поведінка неправильна.

## Послідовність команд

Спочатку виконайте їх по порядку:

```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
openclaw channels status --probe
```

Здоровий базовий стан:

- `Runtime: running`
- `Connectivity probe: ok`
- `Capability: read-only`, `write-capable` або `admin-capable`
- Перевірка каналу показує, що транспорт підключено, і, де підтримується, `works` або `audit ok`

## WhatsApp

### Ознаки збоїв WhatsApp

| Симптом                         | Найшвидша перевірка                                | Виправлення                                                                                                                               |
| ------------------------------- | -------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| Підключено, але немає відповідей на приватні повідомлення | `openclaw pairing list whatsapp`                   | Схваліть відправника або змініть політику приватних повідомлень/список дозволених.                                                        |
| Групові повідомлення ігноруються | Перевірте `requireMention` і шаблони згадок у конфігурації | Згадайте бота або послабте політику згадок для цієї групи.                                                                                |
| Вхід через QR завершується тайм-аутом із 408 | Перевірте змінні середовища Gateway `HTTPS_PROXY` / `HTTP_PROXY` | Налаштуйте доступний проксі; використовуйте `NO_PROXY` лише для обходів.                                                                  |
| Випадкові цикли відключення/повторного входу | `openclaw channels status --probe` + журнали       | Нещодавні повторні підключення позначаються навіть тоді, коли зараз підключення є; стежте за журналами, перезапустіть Gateway, а потім повторно прив’яжіть, якщо нестабільність триває. |

Повне усунення неполадок: [усунення неполадок WhatsApp](/uk/channels/whatsapp#troubleshooting)

## Telegram

### Ознаки збоїв Telegram

| Симптом                              | Найшвидша перевірка                                | Виправлення                                                                                                                          |
| ------------------------------------ | -------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| `/start`, але немає придатного потоку відповідей | `openclaw pairing list telegram`                  | Схваліть прив’язування або змініть політику приватних повідомлень.                                                                    |
| Бот онлайн, але група залишається мовчазною | Перевірте вимогу згадки та режим приватності бота | Вимкніть режим приватності для видимості в групі або згадайте бота.                                                                  |
| Помилки надсилання з мережевими помилками | Перегляньте журнали на збої викликів API Telegram | Виправте маршрутизацію DNS/IPv6/проксі до `api.telegram.org`.                                                                         |
| Під час запуску повідомляється `getMe returned 401` | Перевірте налаштоване джерело токена              | Повторно скопіюйте або згенеруйте заново токен BotFather і оновіть `botToken`, `tokenFile` або `TELEGRAM_BOT_TOKEN` облікового запису за замовчуванням. |
| Опитування зависає або повільно перепідключається | `openclaw logs --follow` для діагностики опитування | Оновіть; якщо перезапуски є хибними спрацюваннями, налаштуйте `pollingStallThresholdMs`. Постійні зависання все одно вказують на проксі/DNS/IPv6. |
| `setMyCommands` відхиляється під час запуску | Перегляньте журнали на `BOT_COMMANDS_TOO_MUCH`    | Зменште кількість команд Telegram від плагінів/Skills/користувацьких команд або вимкніть нативні меню.                              |
| Після оновлення список дозволених блокує вас | `openclaw security audit` і списки дозволених у конфігурації | Запустіть `openclaw doctor --fix` або замініть `@username` на числові ідентифікатори відправників.                                  |

Повне усунення неполадок: [усунення неполадок Telegram](/uk/channels/telegram#troubleshooting)

## Matrix

### Ознаки збоїв Matrix

| Симптом                             | Найшвидша перевірка                          | Виправлення                                                                    |
| ----------------------------------- | -------------------------------------------- | ------------------------------------------------------------------------------ |
| Вхід виконано, але повідомлення кімнати ігноруються | `openclaw channels status --probe`           | Перевірте `groupPolicy`, список дозволених кімнат і фільтрацію за згадками.    |
| Приватні повідомлення не обробляються | `openclaw pairing list matrix`               | Схваліть відправника або скоригуйте політику приватних повідомлень.            |
| Зашифровані кімнати не працюють     | `openclaw matrix verify status`              | Повторно перевірте пристрій, а потім перевірте `openclaw matrix verify backup status`. |
| Відновлення резервної копії очікує або зламане | `openclaw matrix verify backup status`       | Запустіть `openclaw matrix verify backup restore` або повторіть запуск із ключем відновлення. |
| Крос-підписування/початкове налаштування виглядає неправильно | `openclaw matrix verify bootstrap`           | Відновіть секретне сховище, крос-підписування та стан резервної копії за один прохід. |

Повне налаштування та конфігурація: [Matrix](/uk/channels/matrix)
