Використовуйте цю сторінку, коли канал підключається, але працює неправильно.

## Послідовність команд

Спочатку виконайте ці команди по черзі:

```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
openclaw channels status --probe
```

Ознаки справного стану:

- `Runtime: running`
- `Connectivity probe: ok`
- `Capability: read-only`, `write-capable` або `admin-capable`
- Перевірка каналу показує, що транспорт підключено, а там, де це підтримується, — `works` або `audit ok`

## Після оновлення

Скористайтеся цим, якщо після оновлення зникає Telegram, iMessage, конфігурації часів BlueBubbles або інший канал Plugin.

```bash
openclaw status --all
openclaw doctor --fix
openclaw gateway restart
openclaw status --all
```

Шукайте `plugin load failed: dependency tree corrupted; run openclaw doctor --fix` у виводі `openclaw
status --all`. Це означає, що канал налаштовано, але під час налаштування або завантаження Plugin виявлено пошкоджене дерево залежностей, тому канал не було зареєстровано. Команда `openclaw doctor --fix` видаляє застарілі символічні посилання залежностей середовища виконання Plugin і застарілі тіньові дані автентифікації, після чого `openclaw gateway restart` перезавантажує чистий стан.

## WhatsApp

### Ознаки збоїв WhatsApp

| Симптом                                      | Найшвидша перевірка                                 | Виправлення                                                                                                                                                            |
| -------------------------------------------- | --------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Підключено, але немає відповідей у приватних повідомленнях | `openclaw pairing list whatsapp`                    | Схваліть відправника або змініть політику приватних повідомлень чи список дозволених.                                                                                   |
| Групові повідомлення ігноруються             | Перевірте `requireMention` і шаблони згадок у конфігурації | Згадайте бота або послабте політику згадок для цієї групи.                                                                                                              |
| Вхід за QR-кодом завершується помилкою 408 через перевищення часу очікування | Перевірте змінні середовища Gateway `HTTPS_PROXY` / `HTTP_PROXY` | Укажіть доступний проксі; використовуйте `NO_PROXY` лише для обходу проксі.                                                                                              |
| Випадкові цикли відключення й повторного входу | `openclaw channels status --probe` і журнали        | Нещодавні повторні підключення позначаються навіть за поточного підключення; стежте за журналами, перезапустіть Gateway, а якщо нестабільність триває — повторно прив’яжіть обліковий запис. |
| Цикл `status=408 Request Time-out`            | Перевірка, журнали, doctor, потім стан Gateway      | Спочатку виправте проблеми з підключенням або часовими параметрами хоста; якщо цикл триває, створіть резервну копію даних автентифікації та повторно прив’яжіть обліковий запис. |
| Відповіді надходять із затримкою в секунди або хвилини | `openclaw doctor --fix`                             | Doctor зупиняє перевірені застарілі локальні клієнти TUI, якщо вони погіршують роботу циклу подій Gateway.                                                               |

Повний посібник з усунення несправностей: [Усунення несправностей WhatsApp](/uk/channels/whatsapp#troubleshooting)

## Telegram

### Ознаки збоїв Telegram

| Симптом                                      | Найшвидша перевірка                              | Виправлення                                                                                                                                                    |
| -------------------------------------------- | ------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `/start`, але немає придатного потоку відповідей | `openclaw pairing list telegram`                 | Схваліть сполучення або змініть політику приватних повідомлень.                                                                                                 |
| Бот у мережі, але група залишається без відповіді | Перевірте вимогу згадки та режим приватності бота | Вимкніть режим приватності, щоб бот бачив повідомлення групи, або згадайте бота.                                                                                 |
| Помилки надсилання через мережеві збої       | Перевірте журнали на наявність помилок викликів API Telegram | Виправте маршрутизацію DNS/IPv6/проксі до `api.telegram.org`.                                                                                                   |
| Під час запуску повідомляється `getMe returned 401` | Перевірте налаштоване джерело токена             | Повторно скопіюйте або згенеруйте токен BotFather і оновіть `botToken`, `tokenFile` або `TELEGRAM_BOT_TOKEN` стандартного облікового запису.                    |
| Опитування зависає або повільно перепідключається | Перегляньте діагностику опитування через `openclaw logs --follow` | Оновіть систему; якщо перезапуски є хибними спрацюваннями, налаштуйте `pollingStallThresholdMs`. Стійкі зависання й далі вказують на проблеми проксі/DNS/IPv6. |
| `setMyCommands` відхиляється під час запуску | Перевірте журнали на наявність `BOT_COMMANDS_TOO_MUCH` | Зменште кількість команд Telegram від Plugin, Skills або користувацьких команд чи вимкніть вбудовані меню.                                                       |
| Після оновлення список дозволених блокує вас | `openclaw security audit` і списки дозволених у конфігурації | Виконайте `openclaw doctor --fix` або замініть `@username` на числові ідентифікатори відправників.                                                               |

Повний посібник з усунення несправностей: [Усунення несправностей Telegram](/uk/channels/telegram#troubleshooting)
