`qa-channel` — це вбудований синтетичний транспорт повідомлень для автоматизованого QA OpenClaw.

Це не production-канал. Він існує для перевірки тієї самої межі Plugin каналу,
яку використовують реальні транспорти, зберігаючи стан детермінованим і повністю
придатним для інспекції.

## Що він робить зараз

- Граматика цілей класу Slack:
  - `dm:<user>`
  - `channel:<room>`
  - `thread:<room>/<thread>`
- Синтетична шина на основі HTTP для:
  - ін’єкції вхідних повідомлень
  - захоплення вихідної транскрипції
  - створення тредів
  - реакцій
  - редагувань
  - видалень
  - пошуку та дій читання
- Вбудований self-check runner на стороні хоста, який записує звіт у Markdown

## Конфігурація

```json
{
  "channels": {
    "qa-channel": {
      "baseUrl": "http://127.0.0.1:43123",
      "botUserId": "openclaw",
      "botDisplayName": "OpenClaw QA",
      "allowFrom": ["*"],
      "pollTimeoutMs": 1000
    }
  }
}
```

Підтримувані ключі облікового запису:

- `baseUrl`
- `botUserId`
- `botDisplayName`
- `pollTimeoutMs`
- `allowFrom`
- `defaultTo`
- `actions.messages`
- `actions.reactions`
- `actions.search`
- `actions.threads`

## Runner

Поточний вертикальний зріз:

```bash
pnpm qa:e2e
```

Тепер він маршрутизується через вбудоване розширення `qa-lab`. Воно запускає
QA bus у репозиторії, піднімає вбудований runtime-зріз `qa-channel`, виконує
детермінований self-check і записує звіт у Markdown до `.artifacts/qa-e2e/`.

Приватний UI для налагодження:

```bash
pnpm qa:lab:up
```

Ця одна команда збирає QA-сайт, запускає стек Gateway + QA Lab на основі Docker
і виводить URL QA Lab. На цьому сайті можна вибирати сценарії, обирати lane
моделі, запускати окремі прогони й у реальному часі стежити за результатами.

Повний QA-набір на основі репозиторію:

```bash
pnpm openclaw qa suite
```

Він запускає приватний QA-налагоджувач на локальному URL, окремо від
постачуваного bundle Control UI.

## Область застосування

Поточна область застосування навмисно вузька:

- bus + Plugin-транспорт
- граматика маршрутизації тредів
- дії повідомлень, що належать каналу
- звітність у Markdown
- QA-сайт на основі Docker з елементами керування прогонами

Подальша робота додасть:

- виконання матриці провайдерів/моделей
- багатше виявлення сценаріїв
- пізніше — оркестрацію нативно в OpenClaw
