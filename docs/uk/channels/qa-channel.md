# QA-канал

`qa-channel` — це вбудований синтетичний транспорт повідомлень для автоматизованого QA OpenClaw.

Це не production-канал. Він існує, щоб задіяти ту саму межу плагіна каналу,
яку використовують реальні транспорти, водночас зберігаючи стан детермінованим
і повністю придатним до інспекції.

## Що він робить сьогодні

- Граматика цілей класу Slack:
  - `dm:<user>`
  - `channel:<room>`
  - `thread:<room>/<thread>`
- Синтетична шина на основі HTTP для:
  - інʼєкції вхідних повідомлень
  - захоплення вихідних транскриптів
  - створення потоків
  - реакцій
  - редагувань
  - видалень
  - дій пошуку та читання
- Вбудований засіб host-side self-check, який записує звіт у Markdown

## Конфігурація

```json
{
  "channels": {
    "qa-channel": {
      "baseUrl": "http://127.0.0.1:43123",
      "botUserId": "openclaw",
      "botDisplayName": "OpenClaw QA",
      "allowFrom": ["*"],
      "pollTimeoutMs": 1000
    }
  }
}
```

Підтримувані ключі облікового запису:

- `baseUrl`
- `botUserId`
- `botDisplayName`
- `pollTimeoutMs`
- `allowFrom`
- `defaultTo`
- `actions.messages`
- `actions.reactions`
- `actions.search`
- `actions.threads`

## Засіб запуску

Поточний вертикальний зріз:

```bash
pnpm qa:e2e
```

Тепер це маршрутизується через вбудоване розширення `qa-lab`. Воно запускає
QA-шину в репозиторії, завантажує вбудований runtime-зріз `qa-channel`,
виконує детерміновану self-check і записує звіт Markdown у `.artifacts/qa-e2e/`.

Приватний UI для налагодження:

```bash
pnpm qa:lab:up
```

Ця одна команда збирає QA-сайт, запускає стек gateway + QA Lab на основі Docker
і виводить URL QA Lab. На цьому сайті можна вибирати сценарії, обирати доріжку
моделі, запускати окремі прогони й спостерігати результати в реальному часі.

Повний QA-набір на основі репозиторію:

```bash
pnpm openclaw qa suite
```

Це запускає приватний QA-налагоджувач за локальною URL-адресою, окремо від
пакунка shipped Control UI.

## Область охоплення

Поточна область охоплення навмисно вузька:

- шина + транспорт плагіна
- граматика маршрутизації потоків
- дії повідомлень, що належать каналу
- звітність у Markdown
- QA-сайт на основі Docker із засобами керування запусками

Подальша робота додасть:

- виконання матриці провайдерів/моделей
- ширше виявлення сценаріїв
- OpenClaw-native оркестрацію пізніше
