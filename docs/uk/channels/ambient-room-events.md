Фонові події кімнати дають OpenClaw змогу обробляти незгадані розмови в групі або каналі як тихий контекст. Агент може оновлювати пам’ять і стан сеансу, але кімната лишається мовчазною, доки агент явно не викличе інструмент `message`.

Для постійно ввімкнених групових чатів це рекомендований режим: поєднайте `messages.groupChat.unmentionedInbound: "room_event"` з `messages.groupChat.visibleReplies: "message_tool"`. Використовуйте його, коли агент має слухати, вирішувати, коли відповідь корисна, і уникати старого шаблону промпта з відповіддю `NO_REPLY`.

Підтримується сьогодні: канали гільдій Discord, канали Slack і приватні канали, багатокористувацькі DM у Slack, а також групи або супергрупи Telegram. Інші групові канали зберігають свою наявну групову поведінку, якщо на сторінці їхнього каналу не зазначено, що вони підтримують фонові події кімнати.

## Рекомендоване налаштування

Задайте глобальну поведінку групового чату:

```json5
{
  messages: {
    groupChat: {
      unmentionedInbound: "room_event",
      visibleReplies: "message_tool",
      historyLimit: 50,
    },
  },
}
```

Потім налаштуйте саму кімнату як постійно ввімкнену, вимкнувши для неї вимогу згадки. Канал усе одно має бути дозволений його звичайною `groupPolicy`, списком дозволених кімнат і списком дозволених відправників.

Після збереження конфігурації Gateway гаряче перезавантажує налаштування `messages`. Перезапускайте лише тоді, коли спостереження за файлами або перезавантаження конфігурації вимкнено.

## Що змінюється

З `messages.groupChat.unmentionedInbound: "room_event"`:

- незгадані дозволені повідомлення групи або каналу стають тихими подіями кімнати
- повідомлення зі згадками лишаються запитами користувача
- текстові команди та нативні команди лишаються запитами користувача
- запити на переривання або зупинку лишаються запитами користувача
- прямі повідомлення лишаються запитами користувача

Події кімнати використовують сувору видиму доставку. Фінальний текст асистента є приватним. Агент має викликати `message(action=send)`, щоб опублікувати в кімнаті.

## Приклад Discord

```json5
{
  messages: {
    groupChat: {
      unmentionedInbound: "room_event",
      visibleReplies: "message_tool",
      historyLimit: 50,
    },
  },
  channels: {
    discord: {
      groupPolicy: "allowlist",
      guilds: {
        "<DISCORD_SERVER_ID>": {
          requireMention: false,
          users: ["<YOUR_DISCORD_USER_ID>"],
        },
      },
    },
  },
}
```

Використовуйте поканальну конфігурацію Discord, коли фоновим має бути лише один канал:

```json5
{
  channels: {
    discord: {
      guilds: {
        "<DISCORD_SERVER_ID>": {
          channels: {
            "<DISCORD_CHANNEL_ID_OR_NAME>": {
              allow: true,
              requireMention: false,
            },
          },
        },
      },
    },
  },
}
```

## Приклад Slack

Списки дозволених каналів Slack насамперед використовують ID. Використовуйте ID каналів, як-от `C12345678`, а не `#channel-name`.

```json5
{
  messages: {
    groupChat: {
      unmentionedInbound: "room_event",
      visibleReplies: "message_tool",
      historyLimit: 50,
    },
  },
  channels: {
    slack: {
      groupPolicy: "allowlist",
      channels: {
        "<SLACK_CHANNEL_ID>": {
          allow: true,
          requireMention: false,
        },
      },
    },
  },
}
```

## Приклад Telegram

Для груп Telegram бот має бачити звичайні групові повідомлення. Якщо `requireMention: false`, вимкніть режим приватності BotFather або використайте інше налаштування Telegram, яке доставляє боту весь груповий трафік.

```json5
{
  messages: {
    groupChat: {
      unmentionedInbound: "room_event",
      visibleReplies: "message_tool",
      historyLimit: 50,
    },
  },
  channels: {
    telegram: {
      groups: {
        "<TELEGRAM_GROUP_CHAT_ID>": {
          groupPolicy: "open",
          requireMention: false,
        },
      },
    },
  },
}
```

ID груп Telegram зазвичай є від’ємними числами, як-от `-1001234567890`. Прочитайте `chat.id` з `openclaw logs --follow`, перешліть групове повідомлення боту-помічнику для ID або перевірте Bot API `getUpdates`.

## Політика для окремого агента

Використовуйте перевизначення агента, коли кілька агентів спільно використовують ту саму кімнату, але лише один має сприймати незгадані розмови як фоновий контекст:

```json5
{
  messages: {
    groupChat: {
      visibleReplies: "message_tool",
    },
  },
  agents: {
    list: [
      {
        id: "main",
        groupChat: {
          unmentionedInbound: "room_event",
          mentionPatterns: ["@openclaw", "openclaw"],
        },
      },
    ],
  },
}
```

Значення `agents.list[].groupChat.unmentionedInbound`, специфічне для агента, перевизначає `messages.groupChat.unmentionedInbound` для цього агента.

## Режими видимих відповідей

`messages.groupChat.visibleReplies` за замовчуванням має значення `"automatic"` для звичайних групових або канальних запитів користувача. Залишайте це значення за замовчуванням, коли хочете, щоб фінальний текст асистента публікувався видимо без потреби в явному виклику інструмента повідомлень.

Для фонових постійно ввімкнених кімнат `messages.groupChat.visibleReplies: "message_tool"` усе ще рекомендовано, особливо з надійними щодо інструментів моделями найновішого покоління, як-от GPT 5.5. Це дає агенту змогу вирішувати, коли говорити, викликаючи інструмент повідомлень. Якщо модель повертає фінальний текст без виклику інструмента, OpenClaw зберігає цей фінальний текст приватним і записує в журнал метадані пригніченої доставки.

Події кімнати лишаються суворими, навіть коли інші групові запити використовують автоматичні відповіді. Незгадані фонові події кімнати все одно потребують `message(action=send)` для видимого виводу.

## Історія

`messages.groupChat.historyLimit` керує глобальним типовим значенням історії груп. Канали можуть перевизначати його за допомогою `channels.<channel>.historyLimit`, а деякі канали також підтримують ліміти історії для окремих облікових записів.

Задайте `historyLimit: 0`, щоб вимкнути контекст групової історії.

Підтримувані канали подій кімнати зберігають нещодавні фонові повідомлення кімнати як контекст. Discord зберігає історію подій кімнати, доки видиме надсилання Discord не завершиться успішно, тому тихий контекст не втрачається до доставки через інструмент повідомлень.

## Усунення несправностей

Якщо кімната показує набір тексту або використання токенів, але видимого повідомлення немає:

1. Переконайтеся, що кімната дозволена списком дозволених каналів і списком дозволених відправників.
2. Переконайтеся, що `requireMention: false` задано на очікуваному рівні кімнати.
3. Перевірте, чи `messages.groupChat.unmentionedInbound` або перевизначення агента має значення `"room_event"`.
4. Перегляньте журнали на наявність метаданих пригніченого фінального payload або `didSendViaMessagingTool: false`.
5. Для звичайних групових запитів залиште або відновіть `messages.groupChat.visibleReplies: "automatic"`, якщо хочете, щоб фінальні відповіді публікувалися автоматично. Для фонових кімнат, що використовують `message_tool`, використовуйте модель або runtime, які надійно викликають інструменти.

Якщо фонові кімнати Telegram взагалі не спрацьовують, перевірте режим приватності BotFather і переконайтеся, що Gateway отримує звичайні групові повідомлення.

Якщо фонові кімнати Slack не спрацьовують, переконайтеся, що ключ каналу є ID каналу Slack, а застосунок має потрібний scope `channels:history` або `groups:history` для цього типу кімнати.
