# Як робити внесок у модель загроз OpenClaw

Дякуємо, що допомагаєте зробити OpenClaw безпечнішим. Ця модель загроз — живий документ, і ми вітаємо внески від будь-кого — вам не потрібно бути експертом із безпеки.

## Способи зробити внесок

### Додати загрозу

Помітили вектор атаки або ризик, який ми не охопили? Відкрийте issue в [openclaw/trust](https://github.com/openclaw/trust/issues) і опишіть його своїми словами. Вам не потрібно знати жодних фреймворків або заповнювати всі поля — просто опишіть сценарій.

**Корисно включити (але не обов’язково):**

- Сценарій атаки і як його можна використати
- Які частини OpenClaw зачіпаються (CLI, gateway, channels, ClawHub, MCP servers тощо)
- Наскільки серйозним ви це вважаєте (low / medium / high / critical)
- Будь-які посилання на пов’язані дослідження, CVE або приклади з реального світу

Ми самі виконаємо зіставлення з ATLAS, призначимо threat ID і проведемо оцінку ризику під час review. Якщо хочете включити ці деталі — чудово, але цього не очікується.

> **Це для додавання в модель загроз, а не для повідомлення про активні вразливості.** Якщо ви знайшли вразливість, яку можна експлуатувати, див. нашу [Trust page](https://trust.openclaw.ai) для інструкцій із відповідального розкриття.

### Запропонувати mitigation

Маєте ідею, як усунути наявну загрозу? Відкрийте issue або PR з посиланням на загрозу. Корисні mitigation-и є конкретними й придатними до дії — наприклад, «обмеження частоти по відправнику до 10 повідомлень/хвилину на gateway» краще, ніж «реалізувати rate limiting».

### Запропонувати attack chain

Attack chain-и показують, як кілька загроз поєднуються в реалістичний сценарій атаки. Якщо ви бачите небезпечну комбінацію, опишіть кроки та як атакувальник міг би поєднати їх в один ланцюг. Коротка розповідь про те, як атака розгортається на практиці, цінніша за формальний шаблон.

### Виправити або покращити наявний контент

Опечатки, уточнення, застаріла інформація, кращі приклади — PR вітаються, issue не потрібен.

## Що ми використовуємо

### MITRE ATLAS

Ця модель загроз побудована на [MITRE ATLAS](https://atlas.mitre.org/) (Adversarial Threat Landscape for AI Systems) — фреймворку, спеціально розробленому для загроз AI/ML, таких як prompt injection, неправильне використання інструментів і експлуатація агентів. Вам не потрібно знати ATLAS, щоб зробити внесок — ми зіставляємо подання з фреймворком під час review.

### Threat ID

Кожна загроза отримує ID на кшталт `T-EXEC-003`. Категорії такі:

| Код     | Категорія                                  |
| ------- | ------------------------------------------ |
| RECON   | Reconnaissance — збирання інформації       |
| ACCESS  | Initial access — отримання доступу         |
| EXEC    | Execution — виконання шкідливих дій        |
| PERSIST | Persistence — утримання доступу            |
| EVADE   | Defense evasion — уникнення виявлення      |
| DISC    | Discovery — вивчення середовища            |
| EXFIL   | Exfiltration — викрадення даних            |
| IMPACT  | Impact — шкода або порушення роботи        |

ID призначаються мейнтейнерами під час review. Вам не потрібно обирати його самостійно.

### Рівні ризику

| Рівень       | Значення                                                          |
| ------------ | ----------------------------------------------------------------- |
| **Critical** | Повна компрометація системи або висока ймовірність + критичний вплив |
| **High**     | Імовірна значна шкода або середня ймовірність + критичний вплив   |
| **Medium**   | Помірний ризик або низька ймовірність + високий вплив             |
| **Low**      | Малоймовірно і з обмеженим впливом                                |

Якщо ви не впевнені щодо рівня ризику, просто опишіть вплив, а ми його оцінимо.

## Процес review

1. **Triage** — ми переглядаємо нові подання протягом 48 годин
2. **Оцінка** — ми перевіряємо здійсненність, призначаємо зіставлення з ATLAS і threat ID, підтверджуємо рівень ризику
3. **Документування** — ми переконуємося, що все відформатовано й завершено
4. **Merge** — додаємо до моделі загроз і візуалізації
