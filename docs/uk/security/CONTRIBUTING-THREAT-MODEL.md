# Внесок у модель загроз OpenClaw

Дякуємо, що допомагаєте зробити OpenClaw безпечнішим. Ця модель загроз — живий документ, і ми вітаємо внески від будь-кого — вам не обов’язково бути експертом із безпеки.

## Способи зробити внесок

### Додати загрозу

Помітили вектор атаки або ризик, який ми ще не охопили? Відкрийте issue в [openclaw/trust](https://github.com/openclaw/trust/issues) і опишіть його своїми словами. Вам не потрібно знати жодних фреймворків або заповнювати кожне поле — просто опишіть сценарій.

**Корисно вказати (але не обов’язково):**

- Сценарій атаки й те, як його можна експлуатувати
- Які частини OpenClaw зачеплені (CLI, Gateway, канали, ClawHub, MCP servers тощо)
- Наскільки серйозним ви це вважаєте (low / medium / high / critical)
- Будь-які посилання на пов’язані дослідження, CVE або реальні приклади

Ми самі виконаємо зіставлення з ATLAS, призначення ID загроз і оцінку ризику під час рев’ю. Якщо хочете додати ці деталі — чудово, але цього не очікується.

> **Це для доповнення моделі загроз, а не для повідомлення про активні вразливості.** Якщо ви знайшли експлуатовану вразливість, перегляньте нашу [сторінку Trust](https://trust.openclaw.ai) для інструкцій щодо відповідального розкриття.

### Запропонувати пом’якшення

Маєте ідею, як усунути наявну загрозу? Відкрийте issue або PR із посиланням на цю загрозу. Корисні пом’якшення — конкретні й практичні; наприклад, «обмеження частоти на рівні gateway до 10 повідомлень/хвилину для кожного відправника» краще, ніж «реалізувати rate limiting».

### Запропонувати ланцюг атаки

Ланцюги атак показують, як кілька загроз поєднуються в реалістичний сценарій атаки. Якщо ви бачите небезпечну комбінацію, опишіть кроки й те, як зловмисник міг би об’єднати їх у ланцюг. Короткий опис того, як атака розгортається на практиці, цінніший за формальний шаблон.

### Виправити або покращити наявний вміст

Друкарські помилки, уточнення, застаріла інформація, кращі приклади — PR вітаються, issue не потрібен.

## Що ми використовуємо

### MITRE ATLAS

Ця модель загроз побудована на [MITRE ATLAS](https://atlas.mitre.org/) (Adversarial Threat Landscape for AI Systems) — фреймворку, спеціально розробленому для загроз AI/ML, таких як prompt injection, неправильне використання інструментів і експлуатація агентів. Вам не потрібно знати ATLAS, щоб зробити внесок — ми зіставляємо подані матеріали з цим фреймворком під час рев’ю.

### ID загроз

Кожна загроза отримує ID на кшталт `T-EXEC-003`. Категорії такі:

| Code    | Category                                       |
| ------- | ---------------------------------------------- |
| RECON   | Розвідка — збирання інформації                 |
| ACCESS  | Початковий доступ — отримання входу            |
| EXEC    | Виконання — запуск шкідливих дій               |
| PERSIST | Закріплення — збереження доступу               |
| EVADE   | Ухилення від захисту — уникнення виявлення     |
| DISC    | Дослідження — вивчення середовища              |
| EXFIL   | Екcфільтрація — викрадення даних               |
| IMPACT  | Вплив — шкода або порушення роботи             |

ID призначаються супроводжувачами під час рев’ю. Вам не потрібно вибирати їх самостійно.

### Рівні ризику

| Level        | Meaning                                                              |
| ------------ | -------------------------------------------------------------------- |
| **Critical** | Повний компроміс системи або висока ймовірність + критичний вплив    |
| **High**     | Імовірна значна шкода або середня ймовірність + критичний вплив      |
| **Medium**   | Помірний ризик або низька ймовірність + високий вплив                |
| **Low**      | Малоймовірно й з обмеженим впливом                                   |

Якщо ви не впевнені щодо рівня ризику, просто опишіть вплив, а ми його оцінимо.

## Процес рев’ю

1. **Тріаж** — ми переглядаємо нові подання протягом 48 годин
2. **Оцінка** — ми перевіряємо здійсненність, призначаємо зіставлення з ATLAS та ID загрози, перевіряємо рівень ризику
3. **Документування** — ми переконуємося, що все правильно оформлено й завершено
4. **Злиття** — додаємо до моделі загроз і візуалізації
