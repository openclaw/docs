OpenClaw постачається з вбудованим провайдером `runway` для хмарного генерування відео, який увімкнено за замовчуванням і зареєстровано відповідно до контракту `videoGenerationProviders`.

| Властивість              | Значення                                                                  |
| ------------------------ | ------------------------------------------------------------------------- |
| Ідентифікатор провайдера | `runway`                                                                  |
| Plugin                   | вбудований, `enabledByDefault: true`                                      |
| Змінні середовища автентифікації | `RUNWAYML_API_SECRET` (канонічна) або `RUNWAY_API_KEY`              |
| Прапорець початкового налаштування | `--auth-choice runway-api-key`                                     |
| Прямий прапорець CLI     | `--runway-api-key <key>`                                                   |
| API                      | Генерування відео Runway на основі завдань (опитування `GET /v1/tasks/{id}`) |
| Модель за замовчуванням  | `runway/gen4.5`                                                            |

## Початок роботи

<Steps>
  <Step title="Установіть ключ API">
    ```bash
    openclaw onboard --auth-choice runway-api-key
    ```
  </Step>
  <Step title="Установіть Runway як провайдера відео за замовчуванням">
    ```bash
    openclaw config set agents.defaults.videoGenerationModel.primary "runway/gen4.5"
    ```
  </Step>
  <Step title="Згенеруйте відео">
    Попросіть агента згенерувати відео. Runway буде використано автоматично.
  </Step>
</Steps>

## Підтримувані режими та моделі

Провайдер надає сім моделей Runway, розподілених між трьома режимами. Один і той самий ідентифікатор моделі може використовуватися в кількох режимах (наприклад, `gen4.5` працює як для перетворення тексту на відео, так і для перетворення зображення на відео).

| Режим                    | Моделі                                                                  | Вхідні референсні дані          |
| ------------------------ | ----------------------------------------------------------------------- | ------------------------------- |
| Перетворення тексту на відео | `gen4.5` (за замовчуванням), `veo3.1`, `veo3.1_fast`, `veo3`         | Немає                           |
| Перетворення зображення на відео | `gen4.5`, `gen4_turbo`, `gen3a_turbo`, `veo3.1`, `veo3.1_fast`, `veo3` | 1 локальне або віддалене зображення |
| Перетворення відео на відео | `gen4_aleph`                                                         | 1 локальне або віддалене відео |

Локальні посилання на зображення та відео підтримуються через URI даних.

| Співвідношення сторін           | Дозволені значення                           |
| ------------------------------- | -------------------------------------------- |
| Перетворення тексту на відео    | `16:9`, `9:16`                               |
| Редагування зображень і відео   | `1:1`, `16:9`, `9:16`, `3:4`, `4:3`, `21:9` |

<Warning>
  Для перетворення відео на відео наразі потрібна модель `runway/gen4_aleph`. Інші ідентифікатори моделей Runway відхиляють вхідні референсні відео.
</Warning>

<Note>
  Вибір ідентифікатора моделі Runway із неправильного стовпця спричиняє явну помилку ще до того, як запит API залишить OpenClaw. Провайдер перевіряє `model` за списком дозволених моделей режиму (`TEXT_ONLY_MODELS`, `IMAGE_MODELS`, `VIDEO_MODELS`) у `extensions/runway/video-generation-provider.ts`.
</Note>

## Конфігурація

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "runway/gen4.5",
      },
    },
  },
}
```

## Розширена конфігурація

<AccordionGroup>
  <Accordion title="Псевдоніми змінних середовища">
    OpenClaw розпізнає як `RUNWAYML_API_SECRET` (канонічну), так і `RUNWAY_API_KEY`.
    Будь-яка з цих змінних автентифікує провайдера Runway.
  </Accordion>

  <Accordion title="Опитування стану завдань">
    Runway використовує API на основі завдань. Після надсилання запиту на генерування OpenClaw
    опитує `GET /v1/tasks/{id}`, доки відео не буде готове. Для такої
    поведінки опитування додаткова конфігурація не потрібна.
  </Accordion>
</AccordionGroup>
