GMI Cloud — це розміщена платформа інференсу для frontier-моделей і моделей з відкритими вагами
за OpenAI-сумісним API. В OpenClaw це офіційний зовнішній провайдер
plugin, тобто ви встановлюєте його один раз, вибираєте через ідентифікатор провайдера `gmi`,
зберігаєте облікові дані через звичайну автентифікацію моделі та використовуєте посилання на моделі на кшталт
`gmi/google/gemini-3.1-flash-lite`.

Використовуйте GMI, коли вам потрібен один API-ключ для кількох сімейств розміщених моделей, зокрема
маршрутів Google, Anthropic, OpenAI, DeepSeek, Moonshot і Z.AI, які відкриває
каталог GMI. Це корисно як вторинний провайдер для резервного вибору моделі, для порівняння
розміщених маршрутів між постачальниками або коли GMI має модель, доступну раніше, ніж ваш
основний провайдер.

Цей провайдер використовує OpenAI-сумісну семантику чату. OpenClaw володіє
ідентифікатором провайдера, профілем автентифікації, псевдонімами, початковим каталогом моделей і базовою URL-адресою; GMI володіє поточною
доступністю моделей, білінгом, обмеженнями швидкості та будь-якою політикою маршрутизації на боці провайдера.

## Налаштування

Встановіть plugin, перезапустіть gateway, а потім створіть API-ключ у GMI Cloud:

```bash
openclaw plugins install @openclaw/gmi-provider
openclaw gateway restart
```

Потім виконайте:

```bash
openclaw onboard --auth-choice gmi-api-key
```

Або задайте:

```bash
export GMI_API_KEY="<your-gmi-api-key>" # pragma: allowlist secret
```

## Типові значення

- Провайдер: `gmi`
- Псевдоніми: `gmi-cloud`, `gmicloud`
- Базова URL-адреса: `https://api.gmi-serving.com/v1`
- Змінна середовища: `GMI_API_KEY`
- Типова модель: `gmi/google/gemini-3.1-flash-lite`

## Коли обирати GMI

- Вам потрібна розміщена OpenAI-сумісна кінцева точка, а не локальний сервер моделей.
- Ви хочете спробувати кілька комерційних сімейств моделей і сімейств моделей з відкритими вагами через один
  обліковий запис провайдера.
- Вам потрібен резервний провайдер із маршрутизацією upstream, відмінною від OpenRouter,
  DeepInfra, Together або прямих API постачальників.
- Вам потрібні специфічні для GMI ідентифікатори моделей, ціни або керування обліковим записом.

Натомість оберіть прямого провайдера постачальника, коли вам потрібні vendor-native функції,
які GMI не відкриває через свій OpenAI-сумісний маршрут. Оберіть локального
провайдера, як-от Ollama, LM Studio, vLLM або SGLang, коли локальність даних або локальне
керування GPU важливіші за зручність розміщеного сервісу.

## Моделі

Каталог plugin попередньо містить поширені ідентифікатори маршрутів GMI Cloud, зокрема:

- `gmi/zai-org/GLM-5.1-FP8`
- `gmi/deepseek-ai/DeepSeek-V3.2`
- `gmi/moonshotai/Kimi-K2.5`
- `gmi/google/gemini-3.1-flash-lite`
- `gmi/anthropic/claude-sonnet-4.6`
- `gmi/openai/gpt-5.4`

Каталог є початковим набором, а не обіцянкою, що кожен обліковий запис може викликати кожну модель
у будь-який час. Використовуйте команду OpenClaw для переліку моделей, щоб побачити, що налаштований
провайдер повідомляє у вашому середовищі:

```bash
openclaw models list --provider gmi
```

## Усунення несправностей

- `401` або `403`: перевірте, що `GMI_API_KEY` задано для процесу, який запускає
  OpenClaw, або повторно запустіть onboarding, щоб зберегти ключ у профілі автентифікації провайдера.
- Помилки невідомої моделі: підтвердьте, що модель існує у вашому обліковому записі GMI, і використовуйте
  повне посилання `gmi/<route-id>`, показане командою `openclaw models list --provider gmi`.
- Періодичні помилки провайдера: спробуйте інший маршрут GMI або налаштуйте GMI як
  резервний, а не єдиний основний провайдер моделей.
