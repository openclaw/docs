GMI Cloud — це розміщена платформа інференсу для передових моделей і моделей із відкритими вагами
за API, сумісним з OpenAI. В OpenClaw це офіційний зовнішній Plugin
постачальника: установіть його один раз, збережіть облікові дані через звичайну автентифікацію моделі та використовуйте
посилання на моделі на кшталт `gmi/google/gemini-3.1-flash-lite`.

Використовуйте GMI, якщо потрібен один ключ API для кількох сімейств розміщених моделей, зокрема
маршрутів Anthropic, DeepSeek, Google, Moonshot, OpenAI та Z.AI, доступних у
каталозі GMI. Він може слугувати додатковим постачальником для резервного перемикання моделей, порівняння
розміщених маршрутів різних постачальників або коли модель з’являється в GMI раніше, ніж
у вашого основного постачальника. OpenClaw керує ідентифікатором постачальника, профілем автентифікації, псевдонімами,
початковим каталогом моделей і базовою URL-адресою; GMI керує поточною доступністю моделей, виставленням рахунків,
обмеженнями частоти запитів і всіма політиками маршрутизації на боці постачальника.

| Властивість             | Значення                                 |
| ----------------------- | ---------------------------------------- |
| Ідентифікатор постачальника | `gmi` (псевдоніми: `gmi-cloud`, `gmicloud`) |
| Пакет                   | `@openclaw/gmi-provider`                 |
| Змінна середовища автентифікації | `GMI_API_KEY`                    |
| API                     | Сумісний з OpenAI (`openai-completions`) |
| Базова URL-адреса       | `https://api.gmi-serving.com/v1`         |
| Типова модель           | `gmi/google/gemini-3.1-flash-lite`       |

## Налаштування

Установіть Plugin, перезапустіть Gateway, а потім створіть ключ API у GMI Cloud
(`https://www.gmicloud.ai/`):

```bash
openclaw plugins install @openclaw/gmi-provider
openclaw gateway restart
```

Потім виконайте:

```bash
openclaw onboard --auth-choice gmi-api-key
```

Для неінтерактивного налаштування можна передати `--gmi-api-key <key>` або встановити:

```bash
export GMI_API_KEY="<your-gmi-api-key>" # pragma: allowlist secret
```

## Коли варто вибрати GMI

- Вам потрібна розміщена кінцева точка, сумісна з OpenAI, а не локальний сервер моделей.
- Ви хочете випробувати кілька комерційних сімейств моделей і сімейств моделей із відкритими вагами через один
  обліковий запис постачальника.
- Вам потрібен резервний постачальник з іншою висхідною маршрутизацією, ніж у DeepInfra,
  OpenRouter, Together або прямих API постачальників.
- Вам потрібні специфічні для GMI ідентифікатори моделей, ціни або засоби керування обліковим записом.

Натомість виберіть прямого постачальника виробника, якщо потрібні власні функції виробника,
які GMI не надає через свій маршрут, сумісний з OpenAI. Виберіть локального
постачальника, як-от LM Studio, Ollama, SGLang або vLLM, якщо локальність даних чи локальне
керування GPU важливіші за зручність розміщеного сервісу.

## Моделі

Каталог Plugin містить початковий набір ідентифікаторів загальнодоступних маршрутів GMI Cloud:

| Посилання на модель                | Вхідні дані        | Контекст  | Макс. вихід |
| ---------------------------------- | ------------------ | --------- | ----------- |
| `gmi/anthropic/claude-sonnet-4.6`  | текст + зображення | 200,000   | 64,000      |
| `gmi/deepseek-ai/DeepSeek-V3.2`    | текст              | 163,840   | 65,536      |
| `gmi/google/gemini-3.1-flash-lite` | текст + зображення | 1,048,576 | 65,536      |
| `gmi/moonshotai/Kimi-K2.5`         | текст + зображення | 262,144   | 65,536      |
| `gmi/openai/gpt-5.4`               | текст + зображення | 400,000   | 128,000     |
| `gmi/zai-org/GLM-5.1-FP8`          | текст              | 202,752   | 65,536      |

Каталог — це початковий набір, а не гарантія того, що кожен обліковий запис завжди може викликати кожну модель.
Перегляньте, що налаштований постачальник повідомляє у вашому середовищі:

```bash
openclaw models list --provider gmi
```

## Усунення несправностей

- `401` або `403`: перевірте, чи встановлено `GMI_API_KEY` для процесу, у якому працює
  OpenClaw, або повторно виконайте початкове налаштування, щоб зберегти ключ у профілі автентифікації постачальника.
- Помилки невідомої моделі: переконайтеся, що модель існує у вашому обліковому записі GMI, і використовуйте
  повне посилання `gmi/<route-id>`, показане командою `openclaw models list --provider gmi`.
- Періодичні помилки постачальника: спробуйте інший маршрут GMI або налаштуйте GMI як
  резервний, а не єдиний основний постачальник моделей.
