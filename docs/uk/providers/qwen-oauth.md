`qwen-oauth` — це ідентифікатор провайдера Qwen Portal, зареєстрований плагіном Qwen
(`@openclaw/qwen-provider`). Він використовує кінцеву точку Qwen Portal за адресою
`https://portal.qwen.ai/v1` і забезпечує доступ до старіших конфігурацій Qwen OAuth / Portal
через окремий ідентифікатор провайдера, відмінний від канонічного провайдера `qwen`.

Виберіть `qwen-oauth`, якщо у вас уже є робочий токен Qwen Portal, ви
мігруєте зі старого робочого процесу Qwen OAuth або Qwen CLI чи вам потрібно
протестувати саме кінцеву точку Qwen Portal. Для нових конфігурацій віддавайте перевагу
[Qwen](/uk/providers/qwen) зі стандартною кінцевою точкою ModelStudio: вона підтримує нові
конфігурації з ключем API, ширший вибір кінцевих точок, стандартну оплату за фактичне використання, Coding Plan
і повний каталог плагіна Qwen.

## Налаштування

Установіть плагін Qwen, якщо ви ще цього не зробили:

```bash
openclaw plugins install @openclaw/qwen-provider
openclaw gateway restart
```

Надайте токен Portal під час початкового налаштування:

```bash
openclaw onboard --auth-choice qwen-oauth
```

Неінтерактивні запуски зчитують токен із `--qwen-oauth-token <token>`; також можна встановити:

```bash
export QWEN_API_KEY="<your-qwen-portal-token>" # pragma: allowlist secret
```

Початкове налаштування зберігає токен у профілі автентифікації `qwen-oauth`, заповнює каталог
моделей Portal і встановлює `qwen-oauth/qwen3.5-plus` як модель за замовчуванням, якщо
жодну модель не налаштовано.

## Значення за замовчуванням

- Провайдер: `qwen-oauth`
- Псевдоніми: `qwen-portal`, `qwen-cli`
- Базова URL-адреса: `https://portal.qwen.ai/v1`
- Змінна середовища: `QWEN_API_KEY`
- Стиль API: сумісний з OpenAI
- Модель за замовчуванням: `qwen-oauth/qwen3.5-plus`

## Відмінності від Qwen

OpenClaw має два орієнтовані на Qwen ідентифікатори провайдерів:

| Провайдер    | Сімейство кінцевих точок                                 | Найкраще підходить для                                                                 |
| ------------ | -------------------------------------------------------- | -------------------------------------------------------------------------------------- |
| `qwen`       | Кінцеві точки Qwen Cloud / Alibaba DashScope і Coding Plan | Нових конфігурацій із ключем API, стандартної оплати за фактичне використання, Coding Plan, мультимодальних можливостей DashScope |
| `qwen-oauth` | Кінцева точка Qwen Portal за адресою `portal.qwen.ai/v1` | Наявних токенів Qwen Portal і старих конфігурацій Qwen OAuth / CLI                     |

Обидва провайдери використовують сумісні з OpenAI формати запитів, але мають окремі
механізми автентифікації. Токен, збережений для `qwen-oauth`, не слід вважати ключем
DashScope або ModelStudio, а для нового ключа DashScope натомість слід використовувати
канонічний провайдер `qwen`.

## Моделі

Плагін Qwen заповнює цей статичний каталог для кінцевої точки Qwen Portal. Усі
записи мають максимальний обсяг виведення 65 536 токенів; доступність залежить від поточного
облікового запису й токена Qwen Portal.

| Посилання на модель               | Вхідні дані  | Контекст  | Примітки              |
| --------------------------------- | ------------ | --------- | --------------------- |
| `qwen-oauth/qwen3.5-plus`         | текст, зображення | 1,000,000 | Модель за замовчуванням |
| `qwen-oauth/qwen3.6-plus`         | текст, зображення | 1,000,000 |                       |
| `qwen-oauth/qwen3-max-2026-01-23` | текст        | 262,144   |                       |
| `qwen-oauth/qwen3-coder-next`     | текст        | 262,144   |                       |
| `qwen-oauth/qwen3-coder-plus`     | текст        | 1,000,000 |                       |
| `qwen-oauth/MiniMax-M2.5`         | текст        | 1,000,000 | Міркування             |
| `qwen-oauth/glm-5`                | текст        | 202,752   |                       |
| `qwen-oauth/glm-4.7`              | текст        | 202,752   |                       |
| `qwen-oauth/kimi-k2.5`            | текст, зображення | 262,144   |                       |

Якщо ваш обліковий запис натомість використовує ключі API ModelStudio / DashScope, налаштуйте
канонічний провайдер `qwen`:

```bash
openclaw onboard --auth-choice qwen-standard-api-key
openclaw models set qwen/qwen3-coder-plus
```

## Міграція

Застарілі профілі OAuth Qwen Portal неможливо оновити; `openclaw doctor` позначає
їх. Якщо профіль Portal припинив працювати, повторно виконайте початкове налаштування з актуальним токеном
або перейдіть на стандартний провайдер Qwen:

```bash
openclaw onboard --auth-choice qwen-standard-api-key
```

Стандартний глобальний ModelStudio використовує:

```text
https://dashscope-intl.aliyuncs.com/compatible-mode/v1
```

## Усунення несправностей

- Помилки оновлення Portal OAuth: застарілі профілі OAuth Qwen Portal неможливо
  оновити. Повторно виконайте початкове налаштування з актуальним токеном.
- Помилки через неправильну кінцеву точку: переконайтеся, що посилання на модель починається з `qwen-oauth/`,
  коли використовується токен Portal. Посилання з `qwen/` використовуйте лише для канонічного провайдера Qwen.
- Плутанина щодо `QWEN_API_KEY`: на обох сторінках Qwen згадується ця змінна середовища, але початкове налаштування
  зберігає облікові дані під ідентифікатором вибраного провайдера. Віддавайте перевагу початковому налаштуванню, якщо ви
  хочете, щоб на одному комп’ютері були доступні обидва провайдери — `qwen` і `qwen-oauth`.
