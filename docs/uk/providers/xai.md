OpenClaw постачається з вбудованим плагіном провайдера `xai` для моделей Grok. Рекомендований
спосіб — Grok OAuth із відповідною передплатою SuperGrok або X Premium.
Gateway, конфігурація, маршрутизація та інструменти залишаються локальними; лише запити
Grok надсилаються до API xAI.

OAuth не потребує ключа API xAI або застосунку Grok Build. xAI усе одно може
показувати Grok Build на екрані згоди, оскільки OpenClaw використовує спільний
клієнт OAuth від xAI.

## Налаштування

<Steps>
  <Step title="Нове встановлення">
    Запустіть початкове налаштування зі встановленням фонової служби, а потім виберіть xAI/Grok OAuth на
    кроці вибору моделі й автентифікації:

    ```bash
    openclaw onboard --install-daemon
    ```

    На VPS або через SSH виберіть xAI OAuth безпосередньо; цей спосіб використовує перевірку
    за кодом пристрою й не потребує зворотного виклику на localhost:

    ```bash
    openclaw onboard --install-daemon --auth-choice xai-oauth
    ```

  </Step>
  <Step title="Наявне встановлення">
    Увійдіть лише в xAI; не запускайте повторно повне початкове налаштування тільки для підключення Grok:

    ```bash
    openclaw models auth login --provider xai --method oauth
    ```

    Окремо встановіть Grok як модель за замовчуванням:

    ```bash
    openclaw models set xai/grok-4.3
    ```

    Повторно запускайте повне початкове налаштування, лише якщо свідомо хочете змінити Gateway,
    фонову службу, канал, робочий простір або інші параметри налаштування.

  </Step>
  <Step title="Використання ключа API">
    Налаштування за допомогою ключа API й надалі працює для ключів xAI Console та мультимедійних поверхонь,
    яким потрібна конфігурація провайдера на основі ключа:

    ```bash
    openclaw models auth login --provider xai --method api-key
    export XAI_API_KEY=xai-...
    ```

  </Step>
  <Step title="Вибір моделі">
    ```json5
    {
      agents: { defaults: { model: { primary: "xai/grok-4.3" } } },
    }
    ```
  </Step>
</Steps>

<Note>
OpenClaw використовує xAI Responses API як вбудований транспорт xAI. Ті самі
облікові дані з `openclaw models auth login --provider xai --method oauth` або
`--method api-key` також забезпечують роботу `web_search` (ідентифікатор провайдера `grok`), `x_search`,
`code_execution`, мовлення й транскрибування, а також генерацію зображень і відео xAI. Якщо
зберегти ключ xAI у `plugins.entries.xai.config.webSearch.apiKey`,
вбудований провайдер моделей xAI також повторно використовуватиме його як резервний варіант.
</Note>

## Усунення несправностей OAuth

- Для SSH, Docker, VPS або інших віддалених середовищ використовуйте
  `openclaw models auth login --provider xai --method oauth`; цей спосіб використовує
  перевірку за кодом пристрою, а не зворотний виклик на localhost.
- Якщо вхід виконано успішно, але Grok не є моделлю за замовчуванням, виконайте
  `openclaw models set xai/grok-4.3`.
- Перегляньте збережені профілі автентифікації xAI:

  ```bash
  openclaw models auth list --provider xai
  openclaw models status
  ```

- xAI визначає, які облікові записи можуть отримувати токени API через OAuth. Якщо обліковий запис
  не відповідає вимогам, скористайтеся ключем API або перевірте передплату на боці xAI.

<Tip>
Використовуйте `xai-oauth` для входу через SSH, Docker або VPS. OpenClaw виводить
URL і короткий код; завершіть вхід у будь-якому локальному браузері, поки віддалений
процес опитує xAI щодо завершення обміну токенами.
</Tip>

## Вбудований каталог

Ідентифікатори, доступні для вибору в засобах вибору моделей. Плагін і надалі розпізнає старі ідентифікатори Grok 3,
Grok 4, Grok 4 Fast, Grok 4.1 Fast і Grok Code для наявних конфігурацій;
див. [сумісність зі старими версіями та рухомі псевдоніми](#legacy-compatibility-and-moving-aliases).

| Сімейство      | Ідентифікатори моделей                                      |
| -------------- | ------------------------------------------------------------ |
| Grok 4.5       | `grok-4.5` (псевдоніми: `grok-4.5-latest`, `grok-build-latest`) |
| Grok Build 0.1 | `grok-build-0.1`                                             |
| Grok 4.3       | `grok-4.3` (псевдоніми: `grok-4.3-latest`, `grok-latest`)       |
| Grok 4.20      | `grok-4.20-0309-reasoning`, `grok-4.20-0309-non-reasoning`   |

<Tip>
Використовуйте `grok-4.5` для загального спілкування, програмування та агентної роботи там, де ця модель доступна.
Grok 4.3 залишається регіонально безпечним варіантом налаштування за замовчуванням; `grok-build-0.1` і обидва
датовані варіанти Grok 4.20 залишаються доступними для вибору.
</Tip>

## Підтримка можливостей

Вбудований плагін зіставляє підтримувані API xAI зі спільними контрактами провайдерів та
інструментів OpenClaw. Можливості, які не відповідають спільному контракту, наведено
нижче або в розділі відомих обмежень.

| Можливість xAI             | Поверхня OpenClaw                       | Стан                                                 |
| -------------------------- | --------------------------------------- | ---------------------------------------------------- |
| Чат / Responses            | Провайдер моделей `xai/<model>`         | Так                                                  |
| Серверний вебпошук         | Провайдер `web_search` `grok`         | Так                                                  |
| Серверний пошук у X        | Інструмент `x_search`                  | Так                                                  |
| Серверне виконання коду    | Інструмент `code_execution`            | Так                                                  |
| Зображення                 | `image_generate`                        | Так                                                  |
| Відео                      | `video_generate`                        | Так                                                  |
| Пакетний синтез мовлення   | `messages.tts.provider: "xai"` / `tts`  | Так                                                  |
| Потоковий синтез мовлення  | `textToSpeechStream`                    | Так, через `wss://api.x.ai/v1/tts` (не голос у реальному часі) |
| Пакетне розпізнавання мовлення | Розуміння мультимедіа `tools.media.audio` | Так                                              |
| Потокове розпізнавання мовлення | Voice Call `streaming.provider: "xai"`  | Так                                                  |
| Голос у реальному часі     | Talk `talk.realtime.provider: "xai"`    | Так; ретрансляція через Gateway для власних вузлів Talk |
| Файли / пакети             | Лише сумісність із загальним API моделей | Не є повноцінним інструментом OpenClaw               |

<Note>
OpenClaw використовує REST API xAI для зображень, відео, TTS і STT для генерації мультимедіа та
пакетного транскрибування, потоковий WebSocket STT від xAI для транскрибування
голосових викликів наживо, WebSocket Grok Voice Agent від xAI для сеансів Talk у реальному часі,
а Responses API — для чату, пошуку та інструментів виконання коду.
</Note>

### Сумісність зі старим швидким режимом

`/fast on` або `agents.defaults.models["xai/<model>"].params.fastMode: true`
і надалі перетворює старі конфігурації xAI наведеним нижче чином. Ці цільові ідентифікатори
збережено лише для сумісності; для нових
конфігурацій використовуйте актуальні моделі, доступні для вибору.

| Вихідна модель | Ціль швидкого режиму |
| -------------- | -------------------- |
| `grok-3`      | `grok-3-fast`      |
| `grok-3-mini` | `grok-3-mini-fast` |
| `grok-4`      | `grok-4-fast`      |
| `grok-4-0709` | `grok-4-fast`      |

### Сумісність зі старими версіями та рухомі псевдоніми

Старі псевдоніми нормалізуються так:

| Старий псевдонім                                             | Нормалізований ідентифікатор |
| ------------------------------------------------------------- | ---------------- |
| `grok-code-fast-1`, `grok-code-fast`, `grok-code-fast-1-0825` | `grok-build-0.1` |

Датовані ідентифікатори 0309 є доступними для вибору записами каталогу. OpenClaw передає всі інші
актуальні псевдоніми Grok 4.20 без змін, щоб xAI зберігав контроль над семантикою стабільних, найновіших,
бета-, експериментальних і датованих псевдонімів. Глобальний псевдонім `grok-latest`
також зберігається без змін.

xAI припинила підтримку наведених нижче точних ідентифікаторів. OpenClaw зберігає їх як приховані записи
сумісності для випущених конфігурацій, з обмеженнями та цінами їхніх актуальних
цілей переспрямування:

| Виведені з експлуатації ідентифікатори                        | Поточна поведінка                |
| -------------------------------------------------------------------- | -------------------------------- |
| `grok-4-1-fast-reasoning`, `grok-4-fast-reasoning`, `grok-4-0709`    | Grok 4.3 із міркуванням `low` |
| `grok-4-1-fast-non-reasoning`, `grok-4-fast-non-reasoning`, `grok-3` | Grok 4.3 із вимкненим міркуванням |
| `grok-code-fast-1`                                                   | Grok Build 0.1                   |
| `grok-imagine-image-pro`                                             | Grok Imagine Image Quality       |

`openclaw doctor --fix` оновлює збережені параметри за замовчуванням для серверних інструментів xAI та
виведений з експлуатації ідентифікатор якісного зображення, видаляє застарілі згенеровані записи каталогу й виправляє
застарілі контекстні метадані в активних записах 4.20. Ця команда не прив’язує активні псевдоніми 4.20
`beta-latest` до датованого знімка.

## Можливості

<Warning>
  `x_search` і `code_execution` виконуються на серверах xAI. xAI стягує $5 за 1,000
  викликів інструментів, а також оплату за вхідні та вихідні токени моделі. Якщо параметр
  `enabled` кожного інструмента не вказано, OpenClaw надає його лише для активної моделі xAI.
  Відомий провайдер моделей, відмінний від xAI, потребує явного значення `enabled: true` для кожного інструмента;
  за відсутності або неможливості визначити провайдера доступ блокується. Автентифікація xAI потрібна завжди,
  а `enabled: false` вимикає інструмент для всіх провайдерів.
</Warning>

  <Accordion title="Вебпошук">

    ```bash
    openclaw models auth login --provider xai --method oauth
    openclaw config set tools.web.search.provider grok
    ```

  </Accordion>

  <Accordion title="Генерація відео">
    Вбудований плагін `xai` реєструє генерацію відео через спільний
    інструмент `video_generate`.

    - Модель за замовчуванням: `xai/grok-imagine-video`
    - Додаткова модель: `xai/grok-imagine-video-1.5`
    - Класичні режими: перетворення тексту на відео, зображення на відео, генерація за еталонними зображеннями,
      віддалене редагування відео та віддалене продовження відео
    - Режим Video 1.5: лише перетворення зображення на відео з рівно одним зображенням першого кадру
    - Співвідношення сторін: `1:1`, `16:9`, `9:16`, `4:3`, `3:4`, `3:2`, `2:3`;
      класичний режим і режим Video 1.5 для перетворення зображення на відео успадковують співвідношення сторін вихідного зображення, якщо
      його не вказано
    - Роздільна здатність: класичний режим `480P`/`720P`; Video 1.5 також підтримує `1080P`; у всіх
      режимах генерації типовим значенням є `480P`
    - Тривалість: 1-15 секунд для генерації або перетворення зображення на відео, 1-10 секунд за
      використання класичних ролей `reference_image`, 2-10 секунд для класичного продовження
    - Генерація за еталонними зображеннями: установіть `imageRoles` у значення `reference_image` для
      кожного наданого зображення; xAI приймає до 7 таких зображень
    - Редагування й продовження відео успадковують співвідношення сторін і роздільну здатність вхідного відео;
      ці операції не приймають перевизначення геометрії
    - Типовий час очікування операції: 600 секунд, якщо не встановлено `video_generate.timeoutMs`
      або `agents.defaults.videoGenerationModel.timeoutMs`

    <Warning>
    Локальні буфери відео не приймаються. Для вхідних даних редагування або продовження відео використовуйте віддалені URL-адреси `http(s)`.
    Перетворення зображення на відео приймає локальні буфери зображень, оскільки
    OpenClaw кодує їх як URL-адреси даних для xAI.
    </Warning>

    Video 1.5 також розпізнає ідентифікатори xAI `grok-imagine-video-1.5-preview` і
    `grok-imagine-video-1.5-2026-05-30`. OpenClaw передає
    вибраний ідентифікатор без змін, але застосовує ту саму перевірку, що дозволяє лише зображення.

    Щоб використовувати xAI як провайдера відео за замовчуванням:

    ```json5
    {
      agents: {
        defaults: {
          videoGenerationModel: {
            primary: "xai/grok-imagine-video",
          },
        },
      },
    }
    ```

    <Note>
    Див. [Генерація відео](/uk/tools/video-generation), щоб дізнатися про спільні параметри інструмента,
    вибір провайдера та поведінку в разі перемикання на резервний варіант.
    </Note>

  </Accordion>

  <Accordion title="Генерація зображень">
    Вбудований плагін `xai` реєструє генерацію зображень через спільний
    інструмент `image_generate`.

    - Модель зображень за замовчуванням: `xai/grok-imagine-image`
    - Додаткова модель: `xai/grok-imagine-image-quality`
    - Режими: перетворення тексту на зображення та редагування за еталонним зображенням
    - Еталонні вхідні дані: одне `image` або до трьох `images`
    - Співвідношення сторін: `1:1`, `16:9`, `9:16`, `4:3`, `3:4`, `3:2`, `2:3`, `2:1`,
      `1:2`, `19.5:9`, `9:19.5`, `20:9`, `9:20`
    - Роздільна здатність: `1K`, `2K`
    - Кількість: до 4 зображень
    - Час очікування операції за замовчуванням: 600 секунд, якщо не задано `image_generate.timeoutMs`
      або `agents.defaults.imageGenerationModel.timeoutMs`

    OpenClaw запитує в xAI відповіді із зображеннями у форматі `b64_json`, щоб створені медіафайли можна було
    зберігати й доставляти звичайним шляхом вкладень каналу. Локальні
    еталонні зображення перетворюються на URL-адреси даних; віддалені посилання `http(s)`
    передаються без змін.

    Щоб використовувати xAI як постачальника зображень за замовчуванням:

    ```json5
    {
      agents: {
        defaults: {
          imageGenerationModel: {
            primary: "xai/grok-imagine-image",
          },
        },
      },
    }
    ```

    <Note>
    xAI також документує `quality`, `mask`, `user` і співвідношення сторін `auto`.
    Наразі OpenClaw передає лише спільні для різних постачальників засоби керування зображеннями;
    ці параметри, властиві лише xAI, не доступні через `image_generate`.
    </Note>

  </Accordion>

  <Accordion title="Перетворення тексту на мовлення">
    Вбудований Plugin `xai` реєструє перетворення тексту на мовлення через спільний
    інтерфейс постачальника `tts`.

    - Голоси: автентифікований актуальний каталог від xAI; переглянути його можна за допомогою
      `openclaw infer tts voices --provider xai`
    - Резервні офлайн-голоси: `ara`, `eve`, `leo`, `rex`, `sal`
    - Голос за замовчуванням: `eve`
    - Ідентифікатори власних голосів облікового запису передаються, навіть якщо їх немає у
      відповіді вбудованого каталогу
    - Формати: `mp3`, `wav`, `pcm`, `mulaw`, `alaw`
    - Мова: код BCP-47 або `auto`
    - Швидкість: перевизначення швидкості, властиве постачальнику
    - Власний формат голосових повідомлень Opus не підтримується

    Щоб використовувати xAI як постачальника TTS за замовчуванням:

    ```json5
    {
      messages: {
        tts: {
          provider: "xai",
          providers: {
            xai: {
              voiceId: "eve",
            },
          },
        },
      },
    }
    ```

    <Note>
    OpenClaw використовує пакетну кінцеву точку xAI `/v1/tts` для буферизованого синтезу,
    автентифіковане виявлення каталогу `/v1/tts/voices` і власний
    `wss://api.x.ai/v1/tts` для потокового синтезу. Потокове передавання обмежене
    власним хостом `api.x.ai`, тому користувацькі значення `baseUrl` у цьому
    шляху відхиляються. Використовуються наявні засоби керування мовою, голосом, кодеком і швидкістю; для
    частоти дискретизації та бітрейту застосовуються стандартні значення xAI. Синтез аудіофайлів підтримує всі
    налаштовані кодеки. Для цільових голосових повідомлень використовується MP3 як для потокового передавання, так і для буферизованого
    резервного варіанта, оскільки необроблені кодеки xAI не містять метаданих кодека та частоти. Потік
    надсилає `text.delta`, а потім
    `text.done`, отримує `audio.delta`, `audio.done` або `error` і застосовує
    тайм-аут бездіяльності `timeoutMs`, який поновлюється для кожного фрагмента аудіо. Це окремий механізм від
    голосових сеансів реального часу. Див. контракт xAI [API потокового TTS](https://docs.x.ai/developers/rest-api-reference/inference/voice).
    </Note>

  </Accordion>

  <Accordion title="Перетворення мовлення на текст">
    Вбудований Plugin `xai` реєструє пакетне перетворення мовлення на текст через
    інтерфейс транскрибування для розпізнавання медіа в OpenClaw.

    - Кінцева точка: xAI REST `/v1/stt`
    - Шлях вхідних даних: завантаження аудіофайлу у форматі multipart
    - Вибір моделі: xAI вибирає модель транскрибування внутрішньо;
      кінцева точка не має селектора моделі
    - Використовується скрізь, де транскрибування вхідного аудіо читає `tools.media.audio`,
      зокрема для сегментів голосових каналів Discord і аудіовкладень каналів

    Щоб примусово використовувати xAI для транскрибування вхідного аудіо:

    ```json5
    {
      tools: {
        media: {
          audio: {
            models: [
              {
                type: "provider",
                provider: "xai",
              },
            ],
          },
        },
      },
    }
    ```

    Мову можна вказати через спільну конфігурацію аудіомедіа або окремий запит
    транскрибування. Спільний інтерфейс OpenClaw приймає підказки для запиту,
    але інтеграція xAI REST STT передає лише файл і мову,
    оскільки саме вони відповідають поточній публічній кінцевій точці xAI.

  </Accordion>

  <Accordion title="Потокове перетворення мовлення на текст">
    Вбудований Plugin `xai` також реєструє постачальника транскрибування в реальному часі
    для аудіо голосових викликів наживо.

    - Кінцева точка: xAI WebSocket `wss://api.x.ai/v1/stt`
    - Кодування за замовчуванням: `mulaw`
    - Частота дискретизації за замовчуванням: `8000`
    - Визначення кінця мовлення за замовчуванням: `800ms`
    - Проміжні транскрипції: увімкнено за замовчуванням

    Медіапотік Twilio у Voice Call надсилає аудіокадри G.711 mu-law, тому
    постачальник xAI передає ці кадри безпосередньо без перекодування:

    ```json5
    {
      plugins: {
        entries: {
          "voice-call": {
            config: {
              streaming: {
                enabled: true,
                provider: "xai",
                providers: {
                  xai: {
                    apiKey: "${XAI_API_KEY}",
                    endpointingMs: 800,
                    language: "en",
                  },
                },
              },
            },
          },
        },
      },
    }
    ```

    Конфігурація, що належить постачальнику, міститься в
    `plugins.entries.voice-call.config.streaming.providers.xai`. Підтримувані
    ключі: `apiKey`, `baseUrl`, `sampleRate`, `encoding` (`pcm`, `mulaw` або
    `alaw`), `interimResults`, `endpointingMs` і `language`.

    <Note>
    Цей постачальник потокового передавання призначений для шляху транскрибування Voice Call у реальному часі.
    Discord записує короткі сегменти й натомість використовує пакетний
    шлях транскрибування `tools.media.audio`.
    </Note>

  </Accordion>

  <Accordion title="Голос у реальному часі (Talk)">
    Вбудований Plugin `xai` реєструє сеанси Grok Voice Agent у реальному часі для
    режиму Talk через спільний контракт `registerRealtimeVoiceProvider`.

    - Кінцева точка: `wss://api.x.ai/v1/realtime?model=<voice-model>`
    - Модель за замовчуванням: `grok-voice-latest`
    - Голос за замовчуванням: `eve`
    - Транспорт: `gateway-relay` (шляхи ретрансляції iOS, Android і Control UI)
    - Аудіо: PCM16 24 кГц або G.711 µ-law 8 кГц
    - Переривання реплікою: серверний VAD xAI перериває відповідь; OpenClaw очищає чергу відтворення
      та скорочує історію постачальника, яку ще не було відтворено

    Налаштуйте Talk у Gateway:

    ```json5
    {
      talk: {
        realtime: {
          provider: "xai",
          mode: "realtime",
          transport: "gateway-relay",
          brain: "agent-consult",
          providers: {
            xai: {
              model: "grok-voice-latest",
              voice: "eve",
              // Увімкніть лише за умови, що повторне відтворення сеансу на боці постачальника є прийнятним.
              sessionResumption: false,
            },
          },
        },
      },
      env: { XAI_API_KEY: "xai-..." },
    }
    ```

    Конфігурація, що належить постачальнику, також визначається з
    `plugins.entries.voice-call.config.realtime.providers.xai`, коли Voice Call
    або спільні селектори реального часу повторно використовують ту саму мапу постачальників. Підтримувані ключі:
    `apiKey`, `baseUrl`, `model`, `voice`, `vadThreshold`, `silenceDurationMs`,
    `prefixPaddingMs`, `reasoningEffort` і `sessionResumption`.
    `reasoningEffort` приймає лише `high` або `none`, відповідно до API xAI Voice Agent.

    Серверний VAD xAI завжди створює відповіді й обробляє переривання аудіо.
    Використовуйте `consultRouting: "provider-direct"`; примусова маршрутизація транскрипції та вимкнення
    переривання вхідного аудіо не підтримуються протоколом xAI Voice Agent.

    <Note>
    Для автентифікації голосу в реальному часі можна використовувати xAI OAuth або `XAI_API_KEY`. WebRTC,
    керований браузером, поки що не входить до цього інтерфейсу постачальника; використовуйте Talk через gateway-relay на
    нативних Node або шлях ретрансляції Control UI.
    </Note>

    <Note>
    `sessionResumption` за замовчуванням має значення `false`. Якщо встановити `true`, OpenClaw просить
    xAI зберігати достатньо стану сеансу для відновлення тієї самої розмови після
    повторного підключення, а потім підключається повторно з повернутим ідентифікатором розмови. Залиште цей параметр
    вимкненим, якщо повторне відтворення або зберігання на боці постачальника є неприйнятним; у такому разі перервані
    сокети завершуються із закриттям доступу замість непомітного початку нової розмови.
    </Note>

  </Accordion>

  <Accordion title="Конфігурація x_search">
    Вбудований Plugin xAI надає `x_search` як інструмент OpenClaw для
    пошуку вмісту X (раніше Twitter) через Grok.

    Шлях конфігурації: `plugins.entries.xai.config.xSearch`

    | Ключ              | Тип     | За замовчуванням          | Опис                                             |
    | ----------------- | ------- | ------------------------- | ------------------------------------------------ |
    | `enabled`         | boolean | Автоматично для моделей xAI | Вимкнути або явно ввімкнути для відомого постачальника, відмінного від xAI |
    | `model`           | string  | `grok-4.3`                | Модель, що використовується для запитів x_search |
    | `baseUrl`         | string  | -                         | Перевизначення базової URL-адреси xAI Responses  |
    | `inlineCitations` | boolean | -                         | Додавати вбудовані посилання на джерела до результатів |
    | `maxTurns`        | number  | -                         | Максимальна кількість ходів розмови              |
    | `timeoutSeconds`  | number  | `30`                      | Час очікування запиту в секундах                  |
    | `cacheTtlMinutes` | number  | `15`                      | Час життя кешу в хвилинах                         |

    ```json5
    {
      plugins: {
        entries: {
          xai: {
            config: {
              xSearch: {
                enabled: true,
                model: "grok-4.3",
                baseUrl: "https://api.x.ai/v1",
                inlineCitations: true,
              },
            },
          },
        },
      },
    }
    ```

  </Accordion>

  <Accordion title="Конфігурація виконання коду">
    Вбудований Plugin xAI надає `code_execution` як інструмент OpenClaw для
    віддаленого виконання коду в ізольованому середовищі xAI.

    Шлях конфігурації: `plugins.entries.xai.config.codeExecution`

    | Ключ              | Тип     | За замовчуванням         | Опис                                             |
    | ---------------- | ------- | ------------------------ | ------------------------------------------------ |
    | `enabled`        | boolean | Автоматично для моделей xAI | Вимкнути або явно ввімкнути для відомого постачальника, відмінного від xAI |
    | `model`          | string  | `grok-4.3`               | Модель, що використовується для запитів виконання коду |
    | `maxTurns`       | number  | -                        | Максимальна кількість ходів розмови              |
    | `timeoutSeconds` | number  | `30`                     | Час очікування запиту в секундах                  |

    <Note>
    Це віддалене виконання в ізольованому середовищі xAI, а не локальний [`exec`](/uk/tools/exec).
    </Note>

    ```json5
    {
      plugins: {
        entries: {
          xai: {
            config: {
              codeExecution: {
                enabled: true,
                model: "grok-4.3",
              },
            },
          },
        },
      },
    }
    ```

  </Accordion>

  <Accordion title="Відомі обмеження">
    - Для автентифікації xAI можна використовувати ключ API, змінну середовища, резервну конфігурацію плагіна або OAuth із відповідним обліковим записом xAI. OAuth використовує перевірку за кодом пристрою без зворотного виклику через localhost. xAI визначає, які облікові записи можуть отримувати токени API OAuth, а на сторінці згоди може відображатися Grok Build, хоча OpenClaw не потребує застосунку Grok Build.
    - Наразі OpenClaw не надає доступу до сімейства багатоагентних моделей xAI. xAI обслуговує ці моделі через Responses API, але вони не приймають клієнтські чи спеціальні інструменти, які використовує спільний агентний цикл OpenClaw. Див.
      [обмеження багатоагентних моделей xAI](https://docs.x.ai/developers/model-capabilities/text/multi-agent#limitations).
    - Наразі голосовий режим xAI Realtime надає лише транспорт Talk через ретрансляцію Gateway. Сеанси WebSocket провайдера, якими керує браузер, у Control UI ще не підключено.
    - Зображення xAI `quality`, зображення `mask` і додаткові співвідношення сторін, доступні лише в нативному режимі, не надаються, доки спільний інструмент `image_generate` не матиме відповідних міжпровайдерних елементів керування.
  </Accordion>

  <Accordion title="Розширені примітки">
    - OpenClaw автоматично застосовує виправлення сумісності схем інструментів і викликів інструментів, специфічні для xAI, у спільному шляху виконання.
    - Нативні запити xAI типово використовують `tool_stream: true`. Установіть
      `agents.defaults.models["xai/<model>"].params.tool_stream` значення `false`,
      щоб вимкнути це.
    - Вбудована обгортка xAI вилучає непідтримувані межі кількості входжень у схемі та непідтримувані ключі *інтенсивності* міркування з корисного навантаження перед надсиланням нативних запитів xAI. Grok 4.5 підтримує низьку, середню та високу інтенсивність (типово високу). Grok 4.3 підтримує відсутню, низьку, середню та високу інтенсивність (типово низьку). Інші моделі xAI із підтримкою міркування не надають настроюваного керування інтенсивністю, але все одно запитують
      `include: ["reasoning.encrypted_content"]`, щоб попередні зашифровані міркування можна було повторно використати в наступних репліках.
    - `web_search`, `x_search` і `code_execution` надаються як інструменти OpenClaw. OpenClaw додає до запиту кожного інструмента лише конкретний вбудований засіб xAI, якого він потребує, замість додавання всіх нативних інструментів до кожної репліки чату.
    - Grok `web_search` зчитує `plugins.entries.xai.config.webSearch.baseUrl`.
      `x_search` зчитує `plugins.entries.xai.config.xSearch.baseUrl`, а потім
      використовує як резервний варіант базову URL-адресу вебпошуку Grok.
    - `x_search` і `code_execution` належать вбудованому плагіну xAI, а не жорстко закодовані в основному середовищі виконання моделей.
    - `code_execution` — це віддалене виконання в пісочниці xAI, а не локальний
      [`exec`](/uk/tools/exec).
  </Accordion>
</AccordionGroup>

## Тестування в реальному середовищі

Медіашляхи xAI охоплено модульними тестами та необов'язковими наборами тестів у реальному середовищі. Експортуйте
`XAI_API_KEY` у середовище процесу перед запуском перевірок у реальному середовищі.

```bash
pnpm test extensions/xai
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_TEST_QUIET=1 pnpm test:live -- extensions/xai/xai.live.test.ts
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_XAI_VIDEO=1 pnpm test:live -- extensions/xai/xai.live.test.ts -t "classic Grok Imagine"
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_XAI_VIDEO=1 pnpm test:live -- extensions/xai/xai.live.test.ts -t "Grok Imagine Video 1.5"
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_TEST_QUIET=1 pnpm test:live -- extensions/xai/x-search.live.test.ts
OPENCLAW_LIVE_GATEWAY_MODELS="xai/grok-4.5,xai/grok-build-0.1,xai/grok-4.3,xai/grok-4.20-0309-reasoning,xai/grok-4.20-0309-non-reasoning" OPENCLAW_LIVE_GATEWAY_MAX_MODELS=0 OPENCLAW_LIVE_GATEWAY_SMOKE=0 pnpm test:live -- src/gateway/gateway-models.profiles.live.test.ts
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_TEST_QUIET=1 OPENCLAW_LIVE_IMAGE_GENERATION_PROVIDERS=xai pnpm test:live -- test/image-generation.runtime.live.test.ts
```

Файл тестів у реальному середовищі, специфічний для провайдера, синтезує звичайне мовлення TTS і придатне для телефонії PCM-мовлення TTS, транскрибує аудіо через пакетний STT xAI, передає той самий PCM потоково через STT xAI у реальному часі, генерує зображення з тексту та редагує еталонне зображення.
Спільний файл тестування зображень у реальному середовищі перевіряє того самого провайдера xAI через вибір середовища виконання OpenClaw, резервний варіант, нормалізацію та шлях прикріплення медіафайлів. Необов'язковий тестовий випадок Video 1.5 надсилає одне згенероване зображення першого кадру з роздільністю 1080P і перевіряє завантаження завершеного відео.
