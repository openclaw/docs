OpenClaw постачається з вбудованим Plugin провайдера `xai` для моделей Grok. Рекомендований
шлях — OAuth Grok із відповідною передплатою SuperGrok або X Premium.
Gateway, конфігурація, маршрутизація та інструменти залишаються локальними; лише запити
Grok надсилаються до API xAI.

OAuth не потребує ключа API xAI або застосунку Grok Build. xAI усе одно може
показувати Grok Build на екрані згоди, оскільки OpenClaw використовує спільний
клієнт OAuth від xAI.

## Налаштування

<Steps>
  <Step title="Нове встановлення">
    Запустіть початкове налаштування зі встановленням фонової служби, а потім виберіть OAuth xAI/Grok на
    кроці вибору моделі та автентифікації:

    ```bash
    openclaw onboard --install-daemon
    ```

    На VPS або через SSH виберіть OAuth xAI безпосередньо; він використовує
    перевірку за кодом пристрою та не потребує зворотного виклику на localhost:

    ```bash
    openclaw onboard --install-daemon --auth-choice xai-oauth
    ```

  </Step>
  <Step title="Наявне встановлення">
    Увійдіть лише в xAI; не запускайте повне початкове налаштування повторно лише для підключення Grok:

    ```bash
    openclaw models auth login --provider xai --method oauth
    ```

    Окремо встановіть Grok як модель за замовчуванням:

    ```bash
    openclaw models set xai/grok-4.3
    ```

    Повторно запускайте повне початкове налаштування лише тоді, коли свідомо хочете змінити Gateway,
    фонову службу, канал, робочий простір або інші параметри налаштування.

  </Step>
  <Step title="Налаштування через ключ API">
    Налаштування через ключ API досі працює для ключів xAI Console та мультимедійних поверхонь,
    яким потрібна конфігурація провайдера на основі ключа:

    ```bash
    openclaw models auth login --provider xai --method api-key
    export XAI_API_KEY=xai-...
    ```

  </Step>
  <Step title="Виберіть модель">
    ```json5
    {
      agents: { defaults: { model: { primary: "xai/grok-4.3" } } },
    }
    ```
  </Step>
</Steps>

<Note>
OpenClaw використовує API Responses від xAI як вбудований транспорт xAI. Ті самі
облікові дані з `openclaw models auth login --provider xai --method oauth` або
`--method api-key` також забезпечують роботу `web_search` (ідентифікатор провайдера `grok`), `x_search`,
`code_execution`, синтезу й транскрибування мовлення, а також генерації зображень і відео через xAI. Якщо ви
зберігаєте ключ xAI у `plugins.entries.xai.config.webSearch.apiKey`,
вбудований провайдер моделей xAI також використовує його як резервний.
</Note>

## Усунення проблем з OAuth

- Для SSH, Docker, VPS або інших віддалених середовищ використовуйте
  `openclaw models auth login --provider xai --method oauth`; ця команда використовує
  перевірку за кодом пристрою, а не зворотний виклик на localhost.
- Якщо вхід успішний, але Grok не є моделлю за замовчуванням, виконайте
  `openclaw models set xai/grok-4.3`.
- Перегляньте збережені профілі автентифікації xAI:

  ```bash
  openclaw models auth list --provider xai
  openclaw models status
  ```

- xAI визначає, які облікові записи можуть отримувати токени API через OAuth. Якщо обліковий запис
  не відповідає вимогам, скористайтеся ключем API або перевірте передплату на боці xAI.

<Tip>
Використовуйте `xai-oauth` для входу через SSH, Docker або VPS. OpenClaw виводить
URL і короткий код; завершіть вхід у будь-якому локальному браузері, поки віддалений
процес опитує xAI щодо завершення обміну токенами.
</Tip>

## Вбудований каталог

Ідентифікатори, доступні для вибору в засобах вибору моделей. Plugin також розпізнає старі ідентифікатори Grok 3,
Grok 4, Grok 4 Fast, Grok 4.1 Fast і Grok Code для наявних конфігурацій;
див. [сумісність із застарілими версіями та рухомі псевдоніми](#legacy-compatibility-and-moving-aliases).

| Сімейство      | Ідентифікатори моделей                                       |
| -------------- | ------------------------------------------------------------ |
| Grok 4.5       | `grok-4.5` (псевдоніми: `grok-4.5-latest`, `grok-build-latest`) |
| Grok Build 0.1 | `grok-build-0.1`                                             |
| Grok 4.3       | `grok-4.3` (псевдоніми: `grok-4.3-latest`, `grok-latest`)    |
| Grok 4.20      | `grok-4.20-0309-reasoning`, `grok-4.20-0309-non-reasoning`   |

<Tip>
Використовуйте `grok-4.5` для загальних розмов, програмування та агентних завдань там, де ця модель доступна.
Grok 4.3 залишається безпечним для різних регіонів варіантом налаштування за замовчуванням; `grok-build-0.1` і обидва
датовані варіанти Grok 4.20 залишаються доступними для вибору.
</Tip>

## Підтримка можливостей

Вбудований Plugin зіставляє підтримувані API xAI зі спільними контрактами провайдерів та
інструментів OpenClaw. Можливості, які не відповідають спільному контракту, перелічено
нижче або в розділі відомих обмежень.

| Можливість xAI                    | Поверхня OpenClaw                       | Стан                                                                  |
| --------------------------------- | --------------------------------------- | --------------------------------------------------------------------- |
| Чат / Responses                   | провайдер моделей `xai/<model>`         | Так                                                                   |
| Серверний вебпошук                | провайдер `grok` для `web_search`       | Так                                                                   |
| Серверний пошук у X               | інструмент `x_search`                   | Так                                                                   |
| Серверне виконання коду           | інструмент `code_execution`             | Так                                                                   |
| Зображення                        | `image_generate`                        | Так                                                                   |
| Відео                             | `video_generate`                        | Повний класичний робочий процес; перетворення зображення на відео у Video 1.5 |
| Пакетний синтез мовлення          | `messages.tts.provider: "xai"` / `tts`  | Так                                                                   |
| Потоковий синтез мовлення         | -                                       | Ще не реалізовано провайдером xAI                                     |
| Пакетне розпізнавання мовлення    | розуміння медіа `tools.media.audio`     | Так                                                                   |
| Потокове розпізнавання мовлення   | Voice Call `streaming.provider: "xai"`  | Так                                                                   |
| Голосовий зв’язок у реальному часі | -                                     | Ще не надається; потребує іншого контракту сеансу/WebSocket           |
| Файли / пакети                    | Лише сумісність із загальним API моделей | Не є повноцінним інструментом OpenClaw                               |

<Note>
OpenClaw використовує REST API xAI для зображень, відео, синтезу й розпізнавання мовлення для генерації медіа та
пакетного транскрибування, потоковий WebSocket STT від xAI для транскрибування голосових викликів
наживо, а також API Responses для чату, пошуку та інструментів
виконання коду.
</Note>

### Сумісність із застарілим швидким режимом

`/fast on` або `agents.defaults.models["xai/<model>"].params.fastMode: true`
досі перетворює старі конфігурації xAI, як наведено нижче. Ці цільові ідентифікатори
збережено лише для сумісності; для нових конфігурацій використовуйте поточні
моделі, доступні для вибору.

| Початкова модель | Ціль швидкого режиму |
| ---------------- | -------------------- |
| `grok-3`         | `grok-3-fast`        |
| `grok-3-mini`    | `grok-3-mini-fast`   |
| `grok-4`         | `grok-4-fast`        |
| `grok-4-0709`    | `grok-4-fast`        |

### Сумісність із застарілими версіями та рухомі псевдоніми

Старі псевдоніми нормалізуються так:

| Застарілий псевдонім                                         | Нормалізований ідентифікатор |
| ------------------------------------------------------------ | ---------------------------- |
| `grok-code-fast-1`, `grok-code-fast`, `grok-code-fast-1-0825` | `grok-build-0.1`            |

Датовані ідентифікатори 0309 є записами каталогу, доступними для вибору. OpenClaw передає всі інші
поточні псевдоніми Grok 4.20 без змін, щоб xAI зберігав контроль над семантикою стабільних, найновіших,
бета-, експериментальних і датованих псевдонімів. Глобальний псевдонім `grok-latest`
також зберігається без змін.

xAI вилучила наведені нижче точні ідентифікатори. OpenClaw зберігає їх як приховані рядки сумісності
для випущених конфігурацій, з обмеженнями й цінами їхніх поточних
цілей перенаправлення:

| Вилучені ідентифікатори                                           | Поточна поведінка                         |
| ----------------------------------------------------------------- | ----------------------------------------- |
| `grok-4-1-fast-reasoning`, `grok-4-fast-reasoning`, `grok-4-0709` | Grok 4.3 із рівнем міркування `low`       |
| `grok-4-1-fast-non-reasoning`, `grok-4-fast-non-reasoning`, `grok-3` | Grok 4.3 із вимкненим міркуванням      |
| `grok-code-fast-1`                                                | Grok Build 0.1                            |
| `grok-imagine-image-pro`                                          | Grok Imagine Image Quality                |

`openclaw doctor --fix` оновлює збережені стандартні налаштування серверних інструментів xAI та
вилучений ідентифікатор якісного зображення, видаляє застарілі згенеровані рядки каталогу й виправляє
застарілі метадані контексту в активних рядках 4.20. Команда не прив’язує активні псевдоніми 4.20
`beta-latest` до датованого знімка.

## Можливості

<Warning>
  `x_search` і `code_execution` виконуються на серверах xAI. xAI стягує $5 за 1 000
  викликів інструментів, а також плату за вхідні й вихідні токени моделі. Якщо параметр
  `enabled` кожного інструмента не вказано, OpenClaw надає його лише для активної моделі xAI.
  Для відомого провайдера моделей, відмінного від xAI, потрібно явно вказати `enabled: true` для кожного інструмента;
  якщо провайдер відсутній або його не вдається визначити, доступ забороняється. Автентифікація xAI потрібна завжди,
  а `enabled: false` вимикає інструмент для всіх провайдерів.
</Warning>

    ```bash
    openclaw models auth login --provider xai --method oauth
    openclaw config set tools.web.search.provider grok
    ```

  <Accordion title="Генерація відео">
    Вбудований Plugin `xai` реєструє генерацію відео через спільний
    інструмент `video_generate`.

    - Модель за замовчуванням: `xai/grok-imagine-video`
    - Додаткова модель: `xai/grok-imagine-video-1.5`
    - Класичні режими: перетворення тексту на відео, зображення на відео, генерація за еталонним зображенням,
      віддалене редагування відео та віддалене продовження відео
    - Режим Video 1.5: лише перетворення зображення на відео, із рівно одним зображенням першого кадру
    - Співвідношення сторін: `1:1`, `16:9`, `9:16`, `4:3`, `3:4`, `3:2`, `2:3`;
      у класичному режимі та Video 1.5 перетворення зображення на відео успадковує співвідношення сторін вихідного зображення, якщо
      його не вказано
    - Роздільна здатність: у класичному режимі `480P`/`720P`; Video 1.5 також підтримує `1080P`; у всіх
      режимах генерації за замовчуванням використовується `480P`
    - Тривалість: 1–15 секунд для генерації/перетворення зображення на відео, 1–10 секунд за
      використання класичних ролей `reference_image`, 2–10 секунд для класичного продовження
    - Генерація за еталонним зображенням: установіть `imageRoles` у значення `reference_image` для
      кожного наданого зображення; xAI приймає до 7 таких зображень
    - Редагування/продовження відео успадковує співвідношення сторін і роздільну здатність вхідного відео;
      ці операції не приймають перевизначень геометрії
    - Стандартний час очікування операції: 600 секунд, якщо не встановлено `video_generate.timeoutMs`
      або `agents.defaults.videoGenerationModel.timeoutMs`

    <Warning>
    Локальні буфери відео не приймаються. Використовуйте віддалені URL `http(s)` для вхідних даних
    редагування/продовження відео. Перетворення зображення на відео приймає локальні буфери зображень, оскільки
    OpenClaw кодує їх як URL даних для xAI.
    </Warning>

    Video 1.5 також розпізнає ідентифікатори xAI `grok-imagine-video-1.5-preview` і
    `grok-imagine-video-1.5-2026-05-30`. OpenClaw передає вибраний
    ідентифікатор без змін, але застосовує таку саму перевірку, що дозволяє лише зображення.

    Щоб використовувати xAI як провайдера відео за замовчуванням:

    ```json5
    {
      agents: {
        defaults: {
          videoGenerationModel: {
            primary: "xai/grok-imagine-video",
          },
        },
      },
    }
    ```

    <Note>
    Див. [Генерація відео](/uk/tools/video-generation), щоб дізнатися про спільні параметри
    інструмента, вибір провайдера та поведінку резервного перемикання.
    </Note>

  </Accordion>

  <Accordion title="Генерація зображень">
    Вбудований Plugin `xai` реєструє генерацію зображень через спільний
    інструмент `image_generate`.

    - Модель зображень за замовчуванням: `xai/grok-imagine-image`
    - Додаткова модель: `xai/grok-imagine-image-quality`
    - Режими: перетворення тексту на зображення та редагування еталонного зображення
    - Еталонні вхідні дані: одне `image` або до трьох `images`
    - Співвідношення сторін: `1:1`, `16:9`, `9:16`, `4:3`, `3:4`, `3:2`, `2:3`, `2:1`,
      `1:2`, `19.5:9`, `9:19.5`, `20:9`, `9:20`
    - Роздільна здатність: `1K`, `2K`
    - Кількість: до 4 зображень
    - Стандартний час очікування операції: 600 секунд, якщо не задано
      `image_generate.timeoutMs` або `agents.defaults.imageGenerationModel.timeoutMs`

    OpenClaw запитує в xAI відповіді із зображеннями у форматі `b64_json`, щоб
    згенеровані медіафайли можна було зберігати й доставляти звичайним шляхом
    вкладень каналу. Локальні еталонні зображення перетворюються на URL-адреси
    даних; віддалені посилання `http(s)` передаються без змін.

    Щоб використовувати xAI як постачальника зображень за замовчуванням:

    ```json5
    {
      agents: {
        defaults: {
          imageGenerationModel: {
            primary: "xai/grok-imagine-image",
          },
        },
      },
    }
    ```

    <Note>
    xAI також документує `quality`, `mask`, `user` і співвідношення сторін
    `auto`. Наразі OpenClaw передає лише спільні для різних постачальників
    засоби керування зображеннями; ці параметри, властиві лише xAI, не
    доступні через `image_generate`.
    </Note>

  </Accordion>

  <Accordion title="Text-to-speech">
    Вбудований plugin `xai` реєструє синтез мовлення через спільну поверхню
    постачальника `tts`.

    - Голоси: автентифікований актуальний каталог від xAI; перегляньте його за
      допомогою `openclaw infer tts voices --provider xai`
    - Резервні офлайн-голоси: `ara`, `eve`, `leo`, `rex`, `sal`
    - Голос за замовчуванням: `eve`
    - Ідентифікатори власних голосів облікового запису передаються, навіть якщо
      їх немає у відповіді вбудованого каталогу
    - Формати: `mp3`, `wav`, `pcm`, `mulaw`, `alaw`
    - Мова: код BCP-47 або `auto`
    - Швидкість: перевизначення швидкості засобами постачальника
    - Власний формат голосових нотаток Opus не підтримується

    Щоб використовувати xAI як постачальника TTS за замовчуванням:

    ```json5
    {
      messages: {
        tts: {
          provider: "xai",
          providers: {
            xai: {
              voiceId: "eve",
            },
          },
        },
      },
    }
    ```

    <Note>
    OpenClaw використовує пакетну кінцеву точку xAI `/v1/tts` і
    автентифікований каталог `/v1/tts/voices`. xAI також пропонує потоковий TTS
    через WebSocket, але вбудований постачальник xAI поки не реалізує цей
    потоковий обробник.
    </Note>

  </Accordion>

  <Accordion title="Speech-to-text">
    Вбудований plugin `xai` реєструє пакетне розпізнавання мовлення через
    поверхню транскрибування для аналізу медіа в OpenClaw.

    - Кінцева точка: xAI REST `/v1/stt`
    - Шлях вхідних даних: завантаження аудіофайлу як multipart
    - Вибір моделі: xAI вибирає модель транскрибування внутрішньо; кінцева
      точка не має селектора моделі
    - Використовується всюди, де транскрибування вхідного аудіо читає
      `tools.media.audio`, зокрема для сегментів голосових каналів Discord і
      аудіовкладень каналів

    Щоб примусово використовувати xAI для транскрибування вхідного аудіо:

    ```json5
    {
      tools: {
        media: {
          audio: {
            models: [
              {
                type: "provider",
                provider: "xai",
              },
            ],
          },
        },
      },
    }
    ```

    Мову можна вказати через спільну конфігурацію аудіомедіа або в окремому
    запиті на транскрибування. Спільна поверхня OpenClaw приймає підказки
    промпту, але інтеграція xAI REST STT передає лише файл і мову, оскільки
    саме вони відповідають поточній загальнодоступній кінцевій точці xAI.

  </Accordion>

  <Accordion title="Streaming speech-to-text">
    Вбудований plugin `xai` також реєструє постачальника транскрибування в
    реальному часі для аудіо активних голосових викликів.

    - Кінцева точка: xAI WebSocket `wss://api.x.ai/v1/stt`
    - Кодування за замовчуванням: `mulaw`
    - Частота дискретизації за замовчуванням: `8000`
    - Визначення кінця мовлення за замовчуванням: `800ms`
    - Проміжні транскрипції: увімкнені за замовчуванням

    Медіапотік Twilio для Voice Call надсилає аудіокадри G.711 mu-law, тому
    постачальник xAI передає ці кадри безпосередньо без перекодування:

    ```json5
    {
      plugins: {
        entries: {
          "voice-call": {
            config: {
              streaming: {
                enabled: true,
                provider: "xai",
                providers: {
                  xai: {
                    apiKey: "${XAI_API_KEY}",
                    endpointingMs: 800,
                    language: "en",
                  },
                },
              },
            },
          },
        },
      },
    }
    ```

    Конфігурація, якою керує постачальник, розміщується в
    `plugins.entries.voice-call.config.streaming.providers.xai`. Підтримувані
    ключі: `apiKey`, `baseUrl`, `sampleRate`, `encoding` (`pcm`, `mulaw` або
    `alaw`), `interimResults`, `endpointingMs` і `language`.

    <Note>
    Цей потоковий постачальник призначений для шляху транскрибування Voice Call
    у реальному часі. Натомість Discord записує короткі сегменти й використовує
    пакетний шлях транскрибування `tools.media.audio`.
    </Note>

  </Accordion>

  <Accordion title="x_search configuration">
    Вбудований plugin xAI надає `x_search` як інструмент OpenClaw для пошуку
    вмісту X (раніше Twitter) через Grok.

    Шлях конфігурації: `plugins.entries.xai.config.xSearch`

    | Ключ              | Тип     | За замовчуванням         | Опис                                                   |
    | ----------------- | ------- | ------------------------ | ------------------------------------------------------ |
    | `enabled`         | boolean | Автоматично для моделей xAI | Вимкнути або явно ввімкнути для відомого постачальника не-xAI |
    | `model`           | string  | `grok-4.3`               | Модель для запитів x_search                            |
    | `baseUrl`         | string  | -                        | Перевизначення базової URL-адреси xAI Responses        |
    | `inlineCitations` | boolean | -                        | Додавати вбудовані посилання на джерела до результатів |
    | `maxTurns`        | number  | -                        | Максимальна кількість ходів розмови                    |
    | `timeoutSeconds`  | number  | `30`                     | Час очікування запиту в секундах                       |
    | `cacheTtlMinutes` | number  | `15`                     | Час життя кешу в хвилинах                              |

    ```json5
    {
      plugins: {
        entries: {
          xai: {
            config: {
              xSearch: {
                enabled: true,
                model: "grok-4.3",
                baseUrl: "https://api.x.ai/v1",
                inlineCitations: true,
              },
            },
          },
        },
      },
    }
    ```

  </Accordion>

  <Accordion title="Code execution configuration">
    Вбудований plugin xAI надає `code_execution` як інструмент OpenClaw для
    віддаленого виконання коду в ізольованому середовищі xAI.

    Шлях конфігурації: `plugins.entries.xai.config.codeExecution`

    | Ключ             | Тип     | За замовчуванням         | Опис                                                   |
    | ---------------- | ------- | ------------------------ | ------------------------------------------------------ |
    | `enabled`        | boolean | Автоматично для моделей xAI | Вимкнути або явно ввімкнути для відомого постачальника не-xAI |
    | `model`          | string  | `grok-4.3`               | Модель для запитів на виконання коду                   |
    | `maxTurns`       | number  | -                        | Максимальна кількість ходів розмови                    |
    | `timeoutSeconds` | number  | `30`                     | Час очікування запиту в секундах                       |

    <Note>
    Це віддалене виконання в ізольованому середовищі xAI, а не локальний
    [`exec`](/uk/tools/exec).
    </Note>

    ```json5
    {
      plugins: {
        entries: {
          xai: {
            config: {
              codeExecution: {
                enabled: true,
                model: "grok-4.3",
              },
            },
          },
        },
      },
    }
    ```

  </Accordion>

  <Accordion title="Known limits">
    - Автентифікація xAI може використовувати ключ API, змінну середовища,
      резервне значення конфігурації plugin або OAuth із відповідним обліковим
      записом xAI. OAuth використовує перевірку за кодом пристрою без зворотного
      виклику на localhost. xAI визначає, які облікові записи можуть отримувати
      токени API OAuth, а на сторінці згоди може відображатися Grok Build, хоча
      OpenClaw не потребує застосунку Grok Build.
    - Наразі OpenClaw не надає доступу до сімейства багатоагентних моделей xAI.
      xAI обслуговує ці моделі через Responses API, але вони не приймають
      клієнтські або власні інструменти, які використовує спільний агентний цикл
      OpenClaw. Див.
      [обмеження багатоагентних моделей xAI](https://docs.x.ai/developers/model-capabilities/text/multi-agent#limitations).
    - Голосовий режим xAI Realtime ще не зареєстровано як постачальника OpenClaw.
      Для нього потрібен інший контракт двонапрямленого голосового сеансу, ніж
      для пакетного STT або потокового транскрибування.
    - Параметри зображень xAI `quality`, `mask` і власне співвідношення сторін
      `auto` не надаються, доки спільний інструмент `image_generate` не матиме
      відповідних засобів керування для різних постачальників.
  </Accordion>

  <Accordion title="Advanced notes">
    - OpenClaw автоматично застосовує виправлення сумісності схем інструментів і
      викликів інструментів, специфічні для xAI, у спільному шляху виконавця.
    - Власні запити xAI за замовчуванням використовують `tool_stream: true`.
      Установіть
      `agents.defaults.models["xai/<model>"].params.tool_stream` у `false`, щоб
      вимкнути це.
    - Вбудована обгортка xAI видаляє непідтримувані обмеження кількості елементів
      у схемі та непідтримувані ключі корисного навантаження *рівня зусиль*
      міркування перед надсиланням власних запитів xAI. Grok 4.5 підтримує
      низький, середній і високий рівні зусиль (за замовчуванням — високий).
      Grok 4.3 підтримує відсутній, низький, середній і високий рівні зусиль
      (за замовчуванням — низький). Інші моделі xAI зі здатністю до міркування
      не надають налаштовуваного керування рівнем зусиль, але все одно запитують
      `include: ["reasoning.encrypted_content"]`, щоб попередні зашифровані
      міркування можна було повторно використати в наступних ходах.
    - `web_search`, `x_search` і `code_execution` надаються як інструменти
      OpenClaw. OpenClaw додає до запиту кожного інструмента лише конкретний
      вбудований інструмент xAI, який йому потрібен, замість додавання всіх
      власних інструментів до кожного ходу чату.
    - Grok `web_search` читає `plugins.entries.xai.config.webSearch.baseUrl`.
      `x_search` читає `plugins.entries.xai.config.xSearch.baseUrl`, а потім
      використовує базову URL-адресу вебпошуку Grok як резервну.
    - `x_search` і `code_execution` належать вбудованому plugin xAI, а не
      жорстко закодовані в основному середовищі виконання моделей.
    - `code_execution` — це віддалене виконання в ізольованому середовищі xAI,
      а не локальний [`exec`](/uk/tools/exec).
  </Accordion>
</AccordionGroup>

## Тестування наживо

Шляхи медіа xAI охоплено модульними тестами та необов’язковими наборами
тестування наживо. Експортуйте `XAI_API_KEY` у середовище процесу перед запуском
перевірок наживо.

```bash
pnpm test extensions/xai
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_TEST_QUIET=1 pnpm test:live -- extensions/xai/xai.live.test.ts
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_XAI_VIDEO=1 pnpm test:live -- extensions/xai/xai.live.test.ts -t "classic Grok Imagine"
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_XAI_VIDEO=1 pnpm test:live -- extensions/xai/xai.live.test.ts -t "Grok Imagine Video 1.5"
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_TEST_QUIET=1 pnpm test:live -- extensions/xai/x-search.live.test.ts
OPENCLAW_LIVE_GATEWAY_MODELS="xai/grok-4.5,xai/grok-build-0.1,xai/grok-4.3,xai/grok-4.20-0309-reasoning,xai/grok-4.20-0309-non-reasoning" OPENCLAW_LIVE_GATEWAY_MAX_MODELS=0 OPENCLAW_LIVE_GATEWAY_SMOKE=0 pnpm test:live -- src/gateway/gateway-models.profiles.live.test.ts
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_TEST_QUIET=1 OPENCLAW_LIVE_IMAGE_GENERATION_PROVIDERS=xai pnpm test:live -- test/image-generation.runtime.live.test.ts
```

Специфічний для провайдера файл перевірки в реальному середовищі синтезує звичайне TTS, придатне для телефонії PCM-
TTS, транскрибує аудіо через пакетне STT xAI, потоково передає той самий PCM через
STT xAI у реальному часі, генерує зображення з текстового опису та редагує еталонне зображення.
Спільний файл перевірки зображень у реальному середовищі перевіряє того самого провайдера xAI через
вибір середовища виконання OpenClaw, резервне перемикання, нормалізацію та шлях вкладення медіафайлів. Випадок
Video 1.5, який вмикається за бажанням, надсилає одне згенероване зображення першого кадру у форматі 1080P та
перевіряє завантаження готового відео.
