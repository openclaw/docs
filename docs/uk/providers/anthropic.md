Anthropic розробляє сімейство моделей **Claude**. OpenClaw підтримує два способи автентифікації:

- **Ключ API** — прямий доступ до API Anthropic з оплатою за використання (моделі `anthropic/*`)
- **Claude CLI** — повторне використання наявного входу в Claude Code на тому самому хості

## Відстеження використання та витрат

OpenClaw виявляє доступні облікові дані Anthropic і вибирає відповідний інтерфейс відомостей про використання:

- Облікові дані підписки або налаштування Claude показують періоди квот і необов’язковий бюджет додаткового використання.
- `ANTHROPIC_ADMIN_KEY` або `ANTHROPIC_ADMIN_API_KEY` показує у розділі **Використання** інтерфейсу керування витрати організації та використання Messages API за 30 днів за даними постачальника, зокрема щоденні витрати, загальні обсяги токенів і кешу, найпопулярніші моделі та категорії витрат.
- Облікові дані `sk-ant-admin...`, збережені у профілі постачальника Anthropic, автоматично визначаються як ключ Admin API.

Історія витрат Admin API надходить з [API використання та витрат](https://platform.claude.com/docs/en/manage-claude/usage-cost-api) Anthropic. Це фактичні рахунки постачальника, окремі від оцінки витрат OpenClaw, обчисленої на основі сеансів.

<Warning>
Серверна частина Claude CLI в OpenClaw запускає встановлений Claude Code CLI у
неінтерактивному режимі виведення (`claude -p`). У поточній документації Claude Code від Anthropic
цей режим описано як використання Agent SDK або програмний доступ. В оновленні служби підтримки Anthropic від 15 червня 2026 року
призупинено оголошену окрему зміну тарифікації Agent SDK: використання Claude
Agent SDK, `claude -p` і сторонніх застосунків усе ще враховується в межах
лімітів використання підписки, у яку виконано вхід, а раніше оголошений щомісячний кредит Agent SDK
недоступний, поки Anthropic переглядає цей план.

Інтерактивний Claude Code також використовує ліміти плану Claude, у який виконано вхід.
Автентифікація за ключем API передбачає пряму оплату за фактичне використання й не залежить від цього плану.
Для довготривалих хостів Gateway, спільної автоматизації та передбачуваних
виробничих витрат використовуйте ключ API Anthropic.

Поточні статті служби підтримки Anthropic можуть змінити цю поведінку без
випуску нової версії OpenClaw:

- [Довідник Claude Code CLI](https://code.claude.com/docs/en/cli-usage)
- [Використання Claude Agent SDK із планом Claude](https://support.claude.com/en/articles/15036540-use-the-claude-agent-sdk-with-your-claude-plan)
- [Використання Claude Code із планом Pro або Max](https://support.claude.com/en/articles/11145838-use-claude-code-with-your-pro-or-max-plan)
- [Використання Claude Code із планом Team або Enterprise](https://support.claude.com/en/articles/11845131-using-claude-code-with-your-team-or-enterprise-plan)
- [Керування витратами Claude Code](https://code.claude.com/docs/en/costs)

</Warning>

    ### Отримання токена налаштування

    Запустіть `claude setup-token` на будь-якому комп’ютері зі встановленим Claude Code. Команда виведе
    довготривалий токен, що починається з `sk-ant-oat01-`.

    Під час початкового налаштування вставте токен у застосунку macOS, вибравши
    **Anthropic setup-token** у розділі **Connect with an API key or token**, або скористайтеся командою:

    ```bash
    openclaw models auth login --provider anthropic --method setup-token
    ```

    ### Приклад конфігурації

    Надавайте перевагу канонічному посиланню на модель Anthropic із перевизначенням середовища виконання CLI:

    ```json5
    {
      agents: {
        defaults: {
          model: { primary: "anthropic/claude-opus-4-8" },
          models: {
            "anthropic/claude-opus-4-8": {
              agentRuntime: { id: "claude-cli" },
            },
          },
        },
      },
    }
    ```

    Застарілі посилання на моделі `claude-cli/claude-opus-4-7` усе ще працюють для
    сумісності, але в новій конфігурації вибір постачальника й моделі має залишатися у форматі
    `anthropic/*`, а серверну частину виконання слід задавати в політиці середовища виконання постачальника або моделі.

    ### Тарифікація та `claude -p`

    Для запусків Claude CLI OpenClaw використовує неінтерактивний шлях `claude -p` Claude Code.
    Наразі Anthropic вважає цей шлях використанням Agent SDK або програмним доступом:

    - В оновленні служби підтримки Anthropic від 15 червня 2026 року призупинено раніше оголошений
      план окремих кредитів Agent SDK.
    - Використання Claude Agent SDK у межах плану підписки, `claude -p` і сторонніх застосунків
      усе ще враховується в межах лімітів використання підписки, у яку виконано вхід.
    - Раніше оголошений щомісячний кредит Agent SDK недоступний, поки
      Anthropic переглядає цей план.
    - Для входів через консоль або ключ API застосовується оплата API за фактичне використання; такі входи не отримують
      кредит Agent SDK за підпискою.

    Повідомлення про призупинення наведено у [статті Anthropic про план Agent SDK](https://support.claude.com/en/articles/15036540-use-the-claude-agent-sdk-with-your-claude-plan),
    а поведінку підписок описано в статтях про плани Claude Code:
    [Pro/Max](https://support.claude.com/en/articles/11145838-use-claude-code-with-your-pro-or-max-plan)
    і
    [Team/Enterprise](https://support.claude.com/en/articles/11845131-use-claude-code-with-your-team-or-enterprise-plan).

    Anthropic може змінити тарифікацію Claude Code і поведінку обмеження частоти запитів без
    випуску нової версії OpenClaw. Якщо передбачуваність тарифікації важлива, перевіряйте `claude auth status`, `/status` і
    документацію Anthropic за наведеними посиланнями.

## Сеанси Claude на різних комп’ютерах

Вбудований Plugin Anthropic додає групу **Claude Code** до звичайної бічної панелі
сеансів. Рядки відкриваються у звичайній панелі чату. Він виявляє неархівовані сеанси Claude
Code на Gateway і підключених хостах Node:

- Сеанси Claude CLI надходять із дійсних записів індексу проєктів і поточних файлів JSONL,
  обмежений префікс метаданих яких визначає сеанс `sdk-cli`, що не є бічним ланцюжком,
  у каталозі `~/.claude/projects/`.
- Сеанси Claude Desktop використовують назву Desktop, час активності та
  стан архівування, коли його метадані вказують на той самий ідентифікатор сеансу Claude Code.
- Сеанс лише CLI не має прапорця архівування, тому залишається видимим, доки
  існує його стенограма.

Для виявлення не потрібна додаткова конфігурація OpenClaw. Plugin Anthropic
вбудовано й увімкнено за замовчуванням; нативний Node macOS оголошує доступні лише для читання
команди сеансів Claude, коли існує локальний каталог `~/.claude/projects/`.
Схваліть оновлення сполучення Node, коли ці команди з’являться вперше.

Бічна панель групує рядки за їхнім Gateway або хостом сполученого Node, спочатку показує
найновішу обмежену сторінку з кожного хоста й оновлюється зі звичайним
30-секундним інтервалом. Скористайтеся пунктом **Завантажити більше сеансів** під групою каталогу, щоб додати наступну сторінку
для кожного хоста, що має більше історії; додані рядки залишаються видимими й
повторно завантажуються до тієї самої глибини під час оновлень. Клієнти каталогу використовують
`sessions.catalog.list`; відкриття рядка використовує `sessions.catalog.read`.

Під час передавання керування терміналу `claude` визначається з PATH оболонки входу
користувача-власника хоста перед PATH служби або демона. Завдяки цьому сеанси, запущені із застосунку, використовують
той самий Claude CLI, який оператор отримує у звичайному терміналі.

Після вибору рядка спочатку зчитується найновіша сторінка стенограми. Пункт **Завантажити старіші
елементи стенограми** переходить за непрозорим байтовим курсором і зчитує ще одну обмежену частину
файлу JSONL замість завантаження всієї історії. Звичайний вміст користувача, асистента,
міркувань, викликів інструментів і результатів інструментів зберігається. Окремий елемент,
розмір якого перевищує граничне безпечне значення Node або Gateway, чітко позначається як обрізаний.

Якщо ввести текст у звичайному редакторі повідомлення для локального рядка Gateway `claude-cli`,
буде викликано `sessions.catalog.continue`. OpenClaw повторно визначає локальний запис каталогу,
створює або повторно використовує нативний сеанс, закріплений за моделлю, імпортує не більше ніж 200 видимих
елементів або 512 КіБ і ініціалізує прив’язку Claude CLI. Перший хід відновлюється за допомогою
`--fork-session`; Claude призначає відгалуженню новий ідентифікатор сеансу, тому наступні ходи використовують
відгалуження, а вихідний сеанс залишається незмінним.

Хост Node без графічного інтерфейсу також може дозволити продовження своїх рядків Claude CLI, увімкнувши
наведене нижче локальне налаштування Node й перезапустивши хост Node:

```json5
{
  nodeHost: {
    agentRuns: {
      claude: { enabled: true },
    },
  },
}
```

Node оголошує `agent.cli.claude.run.v1` лише тоді, коли налаштування ввімкнено
і його локальний виконуваний файл `claude` вдається знайти. OpenClaw повторно визначає запис каталогу
на цьому Node, імпортує ту саму обмежену історію та прив’язує прийнятий
сеанс до Node і робочого каталогу, вказаного каталогом. Кожен хід запускає
справжній процес `claude -p` цього Node, використовуючи файли Claude й обліковий запис цього Node. Політика
схвалення виконання Node продовжує діяти; Gateway не може примусово ввімкнути цю можливість.

Продовження на Node версії 1 є лише одноразовим. Воно не додає конфігурацію зворотного підключення MCP до Gateway і
аргументи Plugin Skills Gateway, не виконує повторну ініціалізацію зі стенограми Gateway та
відхиляє вкладення й зображення. Рядки Claude Desktop залишаються доступними лише для перегляду. Нативні
Node застосунку macOS також залишаються доступними лише для перегляду, доки застосунок не оголосить команду запуску.

<Note>
Сеанси Claude на сполучених Node залишаються доступними лише для читання, якщо Node без графічного інтерфейсу явно не
оголошує `agent.cli.claude.run.v1`. OpenClaw ніколи не змінює метадані Claude Desktop
і не архівує сеанси Claude. Сторінка потребує підключення оператора
з дозволом на запис, оскільки використовує автентифікований `node.invoke`; операції переліку й читання
залишаються доступними лише для читання навіть на Node з увімкненим продовженням.
</Note>

Докладніше про команду Node і межі безпеки див. у розділі [Node: сеанси та стенограми Claude](/uk/nodes#claude-sessions-and-transcripts).

## Типові налаштування мислення (Claude Sonnet 5, Mythos 5, Fable 5, 4.8 і 4.6)

`anthropic/claude-sonnet-5` за замовчуванням використовує адаптивне мислення з рівнем зусиль `high`.
Використовуйте `/think off`, щоб вимкнути мислення, або `/think xhigh|max`, щоб вибрати вищі
вбудовані рівні зусиль моделі. OpenClaw не задає вручну бюджети мислення, користувацькі
параметри семплювання, попереднє заповнення відповіді асистента та Priority Tier для Sonnet 5, оскільки
Anthropic не підтримує ці функції запитів для цієї моделі.
У каталозі до 31 серпня 2026 року включно застосовується початкова ціна Anthropic `$2/$10` за введення/виведення;
стандартна ціна `$3/$15` починає діяти 1 вересня 2026 року.

`anthropic/claude-fable-5` завжди використовує адаптивне мислення, а рівень зусиль за замовчуванням — `high`.
Anthropic не дає змоги вимкнути мислення для цієї моделі, тому
`/think off` і `/think minimal` натомість зіставляються з рівнем зусиль `low`. OpenClaw також
не передає користувацькі значення температури в запитах до Fable 5, оскільки Anthropic відхиляє
перевизначення температури в будь-якому запиті з увімкненим мисленням.

`anthropic/claude-mythos-5` — модель з обмеженим доступом і таким самим контрактом
постійно ввімкненого адаптивного мислення. У OpenClaw за замовчуванням використовується `high`,
`/think off` і `/think minimal` зіставляються з `low`, а вибрані викликачем параметри семплювання не передаються.
У каталозі зазначено її контекстне вікно на 1,000,000 токенів, обмеження виведення
на 128,000 токенів, підтримку зображень на вході та ціну `$10/$50` за введення/виведення.

У OpenClaw для Claude Opus 4.8 мислення за замовчуванням вимкнено. Коли адаптивне мислення
явно вмикається за допомогою `/think high|xhigh|max`, OpenClaw передає
значення рівня зусиль Anthropic для Opus 4.8; у моделях Claude 4.6 (Opus 4.6 і Sonnet 4.6)
за замовчуванням використовується `adaptive`.

Перевизначте налаштування для окремого повідомлення за допомогою `/think:<level>` або в параметрах моделі:

```json5
{
  agents: {
    defaults: {
      models: {
        "anthropic/claude-opus-4-8": {
          params: { thinking: "high" },
        },
      },
    },
  },
}
```

<Note>
Пов’язана документація Anthropic:
- [Адаптивне мислення](https://platform.claude.com/docs/en/build-with-claude/adaptive-thinking)
- [Розширене мислення](https://platform.claude.com/docs/en/build-with-claude/extended-thinking)

</Note>

## Резервна модель у разі відмови з міркувань безпеки (Claude Fable 5)

<Warning>
Використання Claude Fable 5 також означає використання Claude Opus 4.8. Fable 5 постачається з
класифікаторами безпеки, які можуть відхилити запит, а санкціонований Anthropic
спосіб відновлення полягає в тому, щоб цей хід обслуговувала `claude-opus-4-8`. OpenClaw вмикає цю можливість
автоматично для прямих запитів з ключем API, тому відповіді на деякі ходи Fable
надає Claude Opus 4.8, і вони тарифікуються як Claude Opus 4.8. Якщо ваша політика або бюджет не допускають
ходів, які обслуговує Opus, не вибирайте `anthropic/claude-fable-5`.
</Warning>

### Навіщо це потрібно

Класифікатори Fable 5 повертають `stop_reason: "refusal"` для запитів в обмежених
доменах, а також іноді помилково спрацьовують на суміжних безпечних завданнях (інструменти
безпеки, науки про життя або навіть прохання до моделі відтворити її необроблені
міркування). Без резервної моделі хід завершується помилкою, хоча
інша модель Claude без проблем могла б його обслугувати — власне повідомлення Anthropic про відмову
рекомендує інтеграторам API налаштувати резервну модель.

### Як це працює

1. Для кожного прямого запиту з ключем API до `anthropic/claude-fable-5` OpenClaw
   надсилає згоду на використання серверної резервної моделі Anthropic: бета-заголовок
   `server-side-fallback-2026-06-01` разом із
   `fallbacks: [{"model": "claude-opus-4-8"}]`. Claude Opus 4.8 — єдина
   резервна ціль, яку Anthropic дозволяє для Fable 5.
2. Резервна модель активується лише в разі відмови класифікатора безпеки. Обмеження частоти запитів,
   перевантаження та серверні помилки обробляються так само, як раніше, через
   звичайне [перемикання між моделями](/uk/concepts/model-failover) OpenClaw.
3. Відновлення відбувається в межах того самого виклику. Відмова до початку виведення
   непомітна, окрім затримки; усю відповідь надає Opus 4.8. У разі
   відмови посеред потоку частковий текст зберігається як префікс, з якого продовжує
   резервна модель, а міркування та виклики інструментів відхиленої моделі
   відкидаються згідно з правилами повторного відтворення Anthropic (їх не можна повертати у відповіді або
   виконувати).
4. Якщо Claude Opus 4.8 також відмовляє, хід повертає відмову як
   помилку — так само, як до впровадження цієї функції.

Перемикання на резервну модель відбувається на рівні API Anthropic, тому `claude-opus-4-8` не
потрібно додавати до налаштованого списку моделей або ланцюжка резервних моделей — ключ API
з доступом до Fable завжди може обслуговувати Opus.

### Спостережуваність і тарифікація

- Хід, обслугований резервною моделлю, записує діагностичне повідомлення `provider_fallback` у
  повідомленні асистента із зазначенням `fromModel` і `toModel`, а поле
  `responseModel` повідомлення містить `claude-opus-4-8`.
- Anthropic тарифікує кожну спробу окремо: відмова до початку виведення безкоштовна, а відновлення
  тарифікується за ставками Claude Opus 4.8 (наразі це половина ставок Fable 5). Оцінка
  вартості ходу в OpenClaw для ходів, обслугованих резервною моделлю, використовує ставки Opus.
- У разі відмови посеред потоку Anthropic додатково тарифікує вже передану потоково частину Fable;
  ця частина відображається в статистиці використання API для кожної спроби,
  але не включається в оцінку вартості ходу в OpenClaw.

### Область дії

Застосовується до `anthropic/claude-fable-5` з автентифікацією за ключем API через
`api.anthropic.com`. OAuth (повторне використання підписки Claude CLI), базові URL-адреси проксі,
запити Bedrock, Vertex і Foundry не змінюються та, як і раніше, повертають
відмови як помилки.

Перевірено наживо: безпечний запит із проханням до Fable 5 відтворити її необроблений ланцюжок
міркувань відхиляється з `category: "reasoning_extraction"`, якщо його надіслати без
резервних моделей, а той самий запит через OpenClaw повертає звичайну відповідь,
обслуговану Opus, з доданим діагностичним повідомленням `provider_fallback`.

Докладніше про базову поведінку див. в посібнику Anthropic щодо [відмов і резервних
моделей](https://platform.claude.com/docs/en/build-with-claude/refusals-and-fallback).

## Кешування запитів

OpenClaw підтримує функцію кешування запитів Anthropic для автентифікації за ключем API.

| Значення               | Тривалість кешу | Опис                            |
| ------------------- | -------------- | -------------------------------------- |
| `"short"` (за замовчуванням) | 5 хвилин      | Застосовується автоматично для автентифікації за ключем API |
| `"long"`            | 1 година         | Розширений кеш                         |
| `"none"`            | Без кешування     | Вимкнути кешування запитів                 |

```json5
{
  agents: {
    defaults: {
      models: {
        "anthropic/claude-opus-4-6": {
          params: { cacheRetention: "long" },
        },
      },
    },
  },
}
```

<AccordionGroup>
  <Accordion title="Перевизначення кешу для окремих агентів">
    Використовуйте параметри рівня моделі як базові, а потім перевизначайте їх для окремих агентів через `agents.list[].params`:

    ```json5
    {
      agents: {
        defaults: {
          model: { primary: "anthropic/claude-opus-4-6" },
          models: {
            "anthropic/claude-opus-4-6": {
              params: { cacheRetention: "long" },
            },
          },
        },
        list: [
          { id: "research", default: true },
          { id: "alerts", params: { cacheRetention: "none" } },
        ],
      },
    }
    ```

    Порядок об’єднання конфігурації:

    1. `agents.defaults.models["provider/model"].params`
    2. `agents.list[].params` (відповідний `id`, перевизначення за ключем)

    Завдяки цьому один агент може використовувати довготривалий кеш, а інший агент на тій самій моделі — вимкнути кешування для сплескового трафіку або трафіку з низьким рівнем повторного використання.

  </Accordion>

  <Accordion title="Примітки щодо Claude у Bedrock">
    - Моделі Anthropic Claude у Bedrock (`amazon-bedrock/*anthropic.claude*`) приймають наскрізний параметр `cacheRetention`, якщо його налаштовано.
    - Для моделей Bedrock не від Anthropic у середовищі виконання примусово встановлюється `cacheRetention: "none"`.
    - Інтелектуальні значення за замовчуванням для ключа API також задають `cacheRetention: "short"` для посилань на Claude у Bedrock, якщо явне значення не встановлено.

  </Accordion>
</AccordionGroup>

## Розширена конфігурація

<AccordionGroup>
  <Accordion title="Швидкий режим">
    Спільний перемикач OpenClaw `/fast` задає поле Anthropic `service_tier` для прямого трафіку з ключем API до `api.anthropic.com`.

    | Команда | Відповідність |
    |---------|---------|
    | `/fast on` | `service_tier: "auto"` |
    | `/fast off` | `service_tier: "standard_only"` |

    ```json5
    {
      agents: {
        defaults: {
          models: {
            "anthropic/claude-sonnet-4-6": {
              params: { fastMode: true },
            },
          },
        },
      },
    }
    ```

    <Note>
    - Застосовується лише до прямих запитів `api.anthropic.com`, виконаних із ключем API. Запити з OAuth/токеном підписки та маршрути через проксі ніколи не отримують поле `service_tier`.
    - Явні параметри `serviceTier` або `service_tier` перевизначають `/fast`, якщо задано обидва.
    - В облікових записах без доступної потужності Priority Tier значення `service_tier: "auto"` може бути перетворено на `standard`.

    </Note>

  </Accordion>

  <Accordion title="Розуміння медіа (зображення та PDF)">
    Вбудований Plugin Anthropic реєструє можливості розуміння зображень і PDF. OpenClaw
    автоматично визначає можливості обробки медіа на основі налаштованої автентифікації Anthropic;
    додаткова конфігурація не потрібна.

    | Властивість        | Значення                 |
    | --------------- | --------------------- |
    | Модель за замовчуванням   | `claude-opus-4-8`     |
    | Підтримувані вхідні дані | Зображення, документи PDF |

    Коли до розмови прикріплено зображення або PDF, OpenClaw автоматично
    спрямовує його через провайдера розуміння медіа Anthropic.

  </Accordion>

  <Accordion title="Контекстне вікно 1M">
    Claude Sonnet 5, Mythos 5 і Fable 5 мають точне вхідне вікно
    на 1,000,000 токенів і підтримують до 128,000 токенів виведення. Контекстне вікно 1M від Anthropic
    також загальнодоступне для моделей Claude 4.x з адаптивним мисленням: Opus 4.8,
    Opus 4.7, Opus 4.6 і Sonnet 4.6. OpenClaw визначає розміри цих моделей
    автоматично, тому `params.context1m` не потрібен:

    ```json5
    {
      agents: {
        defaults: {
          models: {
            "anthropic/claude-sonnet-5": {},
            "anthropic/claude-mythos-5": {},
            "anthropic/claude-opus-4-6": {},
          },
        },
      },
    }
    ```

    У старіших конфігураціях можна залишити `params.context1m: true`; для
    цих моделей він нічого не робить і не шкодить, а OpenClaw більше не надсилає застарілий
    бета-заголовок `context-1m-2025-08-07` за жодних обставин. Старіші записи конфігурації `anthropicBeta`
    із цим значенням вилучаються під час визначення заголовків запиту, а
    непідтримувані старіші моделі Claude зберігають своє звичайне контекстне вікно.

    `params.context1m: true` працює так само для бекенду Claude CLI
    (`claude-cli/*`): сумісні моделі Opus і Sonnet із загальнодоступною підтримкою вже автоматично отримують
    вікно 1M, тому цей параметр і там необов’язковий.

    <Warning>
    Потрібен доступ до розширеного контексту для ваших облікових даних Anthropic. Автентифікація через OAuth/токен підписки зберігає необхідні бета-заголовки Anthropic, але OpenClaw вилучає застарілий бета-заголовок 1M, якщо він залишився у старішій конфігурації.
    </Warning>

  </Accordion>

  <Accordion title="Контекст 1M у Claude Opus 4.8">
    `anthropic/claude-opus-4-8` і його варіант `claude-cli` за замовчуванням мають контекстне
    вікно 1M; `params.context1m: true` не потрібен.
  </Accordion>
</AccordionGroup>

## Усунення несправностей

<AccordionGroup>
  <Accordion title="Помилки 401 / токен раптово став недійсним">
    Автентифікаційні токени Anthropic мають термін дії та можуть бути відкликані. Для нових налаштувань натомість використовуйте ключ API Anthropic.
  </Accordion>

  <Accordion title='Ключ API для провайдера "anthropic" не знайдено'>
    Автентифікація Anthropic налаштовується **для кожного агента окремо**; нові агенти не успадковують ключі головного агента. Повторно виконайте початкове налаштування для цього агента (або налаштуйте ключ API на хості Gateway), а потім перевірте за допомогою `openclaw models status`.
  </Accordion>

  <Accordion title='Облікові дані для профілю "anthropic:default" не знайдено'>
    Виконайте `openclaw models status`, щоб дізнатися, який профіль автентифікації активний. Повторно виконайте початкове налаштування або налаштуйте ключ API для шляху цього профілю.
  </Accordion>

  <Accordion title="Немає доступного профілю автентифікації (усі в періоді очікування)">
    Перевірте `openclaw models status --json` для `auth.unusableProfiles`. Періоди очікування через обмеження частоти запитів Anthropic можуть діяти лише для певної моделі, тому інша модель Anthropic усе ще може бути доступною. Додайте ще один профіль Anthropic або зачекайте, доки завершиться період очікування.
  </Accordion>
</AccordionGroup>
