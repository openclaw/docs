Z.AI — це платформа API для моделей **GLM**. Вона надає REST API для GLM і
використовує ключі API для автентифікації. Створіть ключ API в консолі Z.AI.
OpenClaw використовує провайдера `zai` із ключем API Z.AI.

| Властивість   | Значення                                     |
| ------------- | -------------------------------------------- |
| Провайдер     | `zai`                                        |
| Пакет         | `@openclaw/zai-provider`                     |
| Автентифікація | `ZAI_API_KEY` (застарілий псевдонім: `Z_AI_API_KEY`) |
| API           | Z.AI Chat Completions (автентифікація Bearer) |

## Моделі GLM

GLM — це сімейство моделей, а не окремий провайдер. В OpenClaw моделі GLM
використовують посилання на кшталт `zai/glm-5.2`: провайдер `zai`, ідентифікатор моделі `glm-5.2`.

## Початок роботи

Спочатку встановіть Plugin провайдера:

```bash
openclaw plugins install @openclaw/zai-provider
```

### Кінцеві точки

| Варіант початкового налаштування | Базова URL-адреса                            | Модель за замовчуванням |
| -------------------------------- | -------------------------------------------- | ----------------------- |
| `zai-global`                     | `https://api.z.ai/api/paas/v4`                | `glm-5.1`               |
| `zai-cn`                         | `https://open.bigmodel.cn/api/paas/v4`        | `glm-5.1`               |
| `zai-coding-global`              | `https://api.z.ai/api/coding/paas/v4`         | `glm-5.2`               |
| `zai-coding-cn`                  | `https://open.bigmodel.cn/api/coding/paas/v4` | `glm-5.2`               |

`zai-api-key` автоматично визначає одну з цих чотирьох кінцевих точок, перевіряючи ваш ключ
через API завершень чату кожної кінцевої точки: спочатку перевіряються загальні кінцеві точки
(`zai-global`, потім `zai-cn`), а далі — кінцеві точки Coding Plan
(`zai-coding-global`, потім `zai-coding-cn`). Перевірка зупиняється на першій
кінцевій точці, яка приймає запит. Щоб примусово вибрати кінцеву точку Coding Plan,
якщо ваш ключ працює з обома типами, явно вкажіть `--auth-choice`.

## Приклад конфігурації

<Tip>
`zai-api-key` дає OpenClaw змогу визначити відповідну кінцеву точку Z.AI за ключем і
автоматично застосувати правильну базову URL-адресу. Використовуйте явно вказані регіональні варіанти,
коли потрібно примусово вибрати певний Coding Plan або загальний інтерфейс API.
</Tip>

```json5
{
  env: { ZAI_API_KEY: "sk-..." },
  models: {
    providers: {
      zai: {
        // GLM-5.2 використовує кінцеву точку Coding Plan.
        baseUrl: "https://api.z.ai/api/coding/paas/v4",
      },
    },
  },
  agents: { defaults: { model: { primary: "zai/glm-5.2" } } },
}
```

## Вбудований каталог

Plugin провайдера `zai` постачається з каталогом у маніфесті Plugin, тому під час
перегляду лише для читання можна показувати відомі рядки GLM без завантаження середовища виконання провайдера:

```bash
openclaw models list --all --provider zai
```

Каталог на основі маніфесту наразі містить:

| Посилання на модель | Примітки                                      |
| ------------------- | --------------------------------------------- |
| `zai/glm-5.2`        | Стандартна модель Coding Plan; контекст 1 млн |
| `zai/glm-5.1`        | Стандартна модель загального API              |
| `zai/glm-5`          |                                               |
| `zai/glm-5-turbo`    |                                               |
| `zai/glm-5v-turbo`   |                                               |
| `zai/glm-4.7`        |                                               |
| `zai/glm-4.7-flash`  |                                               |
| `zai/glm-4.7-flashx` |                                               |
| `zai/glm-4.6`        |                                               |
| `zai/glm-4.6v`       |                                               |
| `zai/glm-4.5`        |                                               |
| `zai/glm-4.5-air`    |                                               |
| `zai/glm-4.5-flash`  |                                               |
| `zai/glm-4.5v`       |                                               |

<Tip>
Моделі GLM доступні як `zai/<model>` (приклад: `zai/glm-5`).
</Tip>

<Note>
Початкове налаштування Coding Plan за замовчуванням використовує `zai/glm-5.2`, а налаштування
загального API — `zai/glm-5.1`. На кінцевих точках Coding Plan автоматичне визначення
переходить до `glm-5.1`, а потім до `glm-4.7`, якщо ключ або план не надає GLM-5.2.
Версії та доступність GLM можуть змінюватися; виконайте
`openclaw models list --all --provider zai`, щоб переглянути каталог, відомий
встановленій версії.
</Note>

## Рівні міркування

<Tabs>
  <Tab title="GLM-5.2">
    Повний діапазон: `off`, `low`, `high`, `max` (за замовчуванням `off`). OpenClaw зіставляє
    `low` і `high` із рівнем зусиль міркування `high` у Z.AI, а `max` — із рівнем
    `max` у Z.AI через `reasoning_effort` у корисному навантаженні запиту.
  </Tab>
  <Tab title="Інші моделі GLM">
    Лише двійковий перемикач: `off` і `low` (у засобах вибору відображається як `on`),
    за замовчуванням `off`. Якщо для міркування встановити `off`, надсилається
    `thinking: { type: "disabled" }`; будь-який інший рівень не змінює корисне
    навантаження запиту (застосовується власна стандартна поведінка міркування Z.AI).
  </Tab>
</Tabs>

Установлення міркування в `off` запобігає відповідям, які витрачають бюджет виведення
на `reasoning_content` до появи видимого тексту.

## Розширена конфігурація

<AccordionGroup>
  <Accordion title="Перспективне розпізнавання невідомих моделей GLM-5">
    Невідомі ідентифікатори `glm-5*` усе одно перспективно розпізнаються на шляху провайдера
    завдяки синтезу метаданих, якими керує провайдер, із шаблону `glm-4.7`, якщо ідентифікатор
    відповідає поточній формі сімейства GLM-5.
  </Accordion>

  <Accordion title="Потокове передавання викликів інструментів">
    `tool_stream` увімкнено за замовчуванням для потокового передавання викликів інструментів Z.AI. Щоб вимкнути його:

    ```json5
    {
      agents: {
        defaults: {
          models: {
            "zai/<model>": {
              params: { tool_stream: false },
            },
          },
        },
      },
    }
    ```

  </Accordion>

  <Accordion title="Збережене міркування">
    Збережене міркування потрібно вмикати явно, оскільки Z.AI вимагає повторного відтворення
    повного історичного `reasoning_content`, що збільшує кількість токенів запиту. Увімкніть
    його окремо для кожної моделі:

    ```json5
    {
      agents: {
        defaults: {
          models: {
            "zai/glm-5.2": {
              params: { preserveThinking: true },
            },
          },
        },
      },
    }
    ```

    Коли цю функцію ввімкнено й міркування активне, OpenClaw надсилає
    `thinking: { type: "enabled", clear_thinking: false }` і повторно відтворює попередній
    `reasoning_content` для тієї самої сумісної з OpenAI історії діалогу. Ключ параметра
    `preserve_thinking` у форматі snake_case працює як псевдонім.

    Досвідчені користувачі також можуть перевизначити точне корисне навантаження провайдера
    за допомогою `params.extra_body.thinking`.

  </Accordion>

  <Accordion title="Розпізнавання зображень">
    Plugin Z.AI реєструє розпізнавання зображень.

    | Властивість | Значення    |
    | ----------- | ----------- |
    | Модель      | `glm-4.6v`  |

    Розпізнавання зображень автоматично визначається з налаштованої автентифікації Z.AI —
    додаткова конфігурація не потрібна.

  </Accordion>

  <Accordion title="Відомості про автентифікацію">
    - Z.AI використовує автентифікацію Bearer із вашим ключем API.
    - Варіант початкового налаштування `zai-api-key` автоматично визначає відповідну кінцеву точку Z.AI, перевіряючи підтримувані кінцеві точки за допомогою вашого ключа.
    - Використовуйте явно вказані регіональні варіанти (`zai-coding-global`, `zai-coding-cn`, `zai-global`, `zai-cn`), коли потрібно примусово вибрати певний інтерфейс API.
    - Застаріла змінна середовища `Z_AI_API_KEY` усе ще підтримується; під час запуску OpenClaw копіює її в `ZAI_API_KEY`, якщо `ZAI_API_KEY` не задано.

  </Accordion>
</AccordionGroup>
