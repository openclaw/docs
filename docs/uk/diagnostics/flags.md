Прапорці діагностики дають змогу вмикати цільові журнали налагодження без увімкнення докладного логування всюди. Прапорці є опційними й не мають ефекту, доки підсистема не перевіряє їх.

## Як це працює

- Прапорці — це рядки (без урахування регістру).
- Ви можете вмикати прапорці в config або через перевизначення env.
- Підтримуються wildcard-и:
  - `telegram.*` відповідає `telegram.http`
  - `*` вмикає всі прапорці

## Увімкнення через config

```json
{
  "diagnostics": {
    "flags": ["telegram.http"]
  }
}
```

Кілька прапорців:

```json
{
  "diagnostics": {
    "flags": ["telegram.http", "gateway.*"]
  }
}
```

Після зміни прапорців перезапустіть gateway.

## Перевизначення env (одноразово)

```bash
OPENCLAW_DIAGNOSTICS=telegram.http,telegram.payload
```

Вимкнути всі прапорці:

```bash
OPENCLAW_DIAGNOSTICS=0
```

## Куди потрапляють журнали

Прапорці записують журнали до стандартного файла журналу діагностики. За замовчуванням:

```
/tmp/openclaw/openclaw-YYYY-MM-DD.log
```

Якщо ви задали `logging.file`, замість цього використовується цей шлях. Журнали мають формат JSONL (по одному JSON-об’єкту в рядку). Редагування чутливих даних усе ще застосовується відповідно до `logging.redactSensitive`.

## Витяг журналів

Вибрати найновіший файл журналу:

```bash
ls -t /tmp/openclaw/openclaw-*.log | head -n 1
```

Відфільтрувати діагностику Telegram HTTP:

```bash
rg "telegram http error" /tmp/openclaw/openclaw-*.log
```

Або переглядати в реальному часі під час відтворення:

```bash
tail -f /tmp/openclaw/openclaw-$(date +%F).log | rg "telegram http error"
```

Для віддалених gateway ви також можете використовувати `openclaw logs --follow` (див. [/cli/logs](/uk/cli/logs)).

## Примітки

- Якщо `logging.level` встановлено вище за `warn`, ці журнали можуть бути пригнічені. Типове значення `info` підходить.
- Прапорці безпечно залишати увімкненими; вони впливають лише на обсяг журналів для конкретної підсистеми.
- Використовуйте [/logging](/uk/logging), щоб змінювати місця призначення журналів, рівні та редагування чутливих даних.
