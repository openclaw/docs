## Що це таке

- Отримує дані про використання/квоти провайдера безпосередньо з їхніх endpoint використання.
- Жодних оцінених витрат; лише вікна, про які повідомляє сам провайдер.
- Зрозумілий для людини вивід стану нормалізується до `X% left`, навіть коли
  зовнішній API повідомляє використану квоту, залишок квоти або лише сирі лічильники.
- `/status` і `session_status` на рівні сесії можуть використовувати fallback до
  останнього запису використання в транскрипті, коли live snapshot сесії є розрідженим. Такий
  fallback заповнює відсутні лічильники токенів/кешу, може відновити мітку
  активної моделі runtime і віддає перевагу більшому загальному значенню, орієнтованому на prompt, коли метаданих сесії бракує або вони менші. Наявні ненульові live-значення все одно мають пріоритет.

## Де це відображається

- `/status` у чатах: картка стану з емодзі, токенами сесії + оціненою вартістю (лише для API-ключа). Використання провайдера показується для **поточного провайдера моделі**, коли доступне, у нормалізованому вигляді `X% left`.
- `/usage off|tokens|full` у чатах: нижній колонтитул використання для кожної відповіді (OAuth показує лише токени).
- `/usage cost` у чатах: локальний підсумок вартості, агрегований із логів сесій OpenClaw.
- CLI: `openclaw status --usage` виводить повну розбивку за провайдерами.
- CLI: `openclaw channels list` виводить той самий snapshot використання поруч із конфігурацією провайдера (використайте `--no-usage`, щоб пропустити).
- Рядок меню macOS: розділ “Usage” у Context (лише якщо доступно).

## Провайдери + облікові дані

- **Anthropic (Claude)**: OAuth-токени в профілях автентифікації.
- **GitHub Copilot**: OAuth-токени в профілях автентифікації.
- **Gemini CLI**: OAuth-токени в профілях автентифікації.
  - JSON-використання має fallback до `stats`; `stats.cached` нормалізується в
    `cacheRead`.
- **OpenAI Codex**: OAuth-токени в профілях автентифікації (використовується accountId, якщо він є).
- **MiniMax**: API-ключ або профіль автентифікації MiniMax OAuth. OpenClaw розглядає
  `minimax`, `minimax-cn` і `minimax-portal` як ту саму поверхню квоти MiniMax,
  віддає перевагу збереженому MiniMax OAuth, якщо він є, і в іншому випадку використовує fallback до
  `MINIMAX_CODE_PLAN_KEY`, `MINIMAX_CODING_API_KEY` або `MINIMAX_API_KEY`.
  Сирі поля MiniMax `usage_percent` / `usagePercent` означають **залишок**
  квоти, тому OpenClaw інвертує їх перед відображенням; поля на основі лічильників мають перевагу, якщо вони наявні.
  - Мітки вікон coding-plan беруться з полів годин/хвилин провайдера, якщо
    вони є, а потім мають fallback до інтервалу `start_time` / `end_time`.
  - Якщо endpoint coding-plan повертає `model_remains`, OpenClaw віддає перевагу
    запису chat-model, виводить мітку вікна з часових міток, коли явні поля
    `window_hours` / `window_minutes` відсутні, і включає назву моделі
    в мітку плану.
- **Xiaomi MiMo**: API-ключ через env/config/auth store (`XIAOMI_API_KEY`).
- **z.ai**: API-ключ через env/config/auth store.

Використання приховується, якщо не вдається розв’язати придатну автентифікацію для usage провайдера. Провайдери
можуть надавати Plugin-специфічну логіку автентифікації для usage; інакше OpenClaw використовує
fallback до відповідних облікових даних OAuth/API-ключа з профілів автентифікації, змінних середовища
або конфігурації.
