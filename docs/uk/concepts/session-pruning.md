Pruning session обрізає **старі результати інструментів** із контексту перед кожним викликом LLM. Це зменшує роздування контексту через накопичений вивід інструментів (результати exec, читання файлів, результати пошуку), не переписуючи звичайний текст розмови.

<Info>
Pruning виконується лише в пам’яті — він не змінює session transcript на диску.
Повна історія завжди зберігається.
</Info>

## Чому це важливо

У довгих sessions накопичується вивід інструментів, який роздуває вікно контексту. Це
збільшує вартість і може змусити виконувати [Compaction](/uk/concepts/compaction) раніше,
ніж потрібно.

Pruning особливо цінний для **Anthropic prompt caching**. Після завершення TTL кешу
наступний запит повторно кешує весь prompt. Pruning зменшує розмір запису в кеш,
що безпосередньо знижує вартість.

## Як це працює

1. Дочекатися завершення TTL кешу (типово 5 хвилин).
2. Знайти старі результати інструментів для звичайного pruning (текст розмови не чіпається).
3. **М’яко обрізати** завеликі результати — зберегти початок і кінець, вставити `...`.
4. **Жорстко очистити** решту — замінити placeholder-ом.
5. Скинути TTL, щоб наступні запити повторно використовували свіжий кеш.

## Очищення застарілих зображень

OpenClaw також виконує окреме ідемпотентне очищення для старих legacy sessions, які
зберігали сирі блоки зображень в історії.

- Воно зберігає **3 найновіші завершені ходи** байт-у-байт, щоб префікси prompt cache
  для недавніх follow-up залишалися стабільними.
- Старі вже оброблені блоки зображень в історії `user` або `toolResult` можуть бути
  замінені на `[image data removed - already processed by model]`.
- Це окремо від звичайного pruning за TTL кешу. Воно існує, щоб повторні
  payload-и зображень не ламали prompt cache на наступних ходах.

## Розумні типові значення

OpenClaw автоматично вмикає pruning для профілів Anthropic:

| Тип профілю                                            | Pruning увімкнено | Heartbeat |
| ------------------------------------------------------ | ----------------- | --------- |
| Anthropic OAuth/token auth (включно з повторним використанням Claude CLI) | Так               | 1 година  |
| API key                                                | Так               | 30 хв     |

Якщо ви задаєте явні значення, OpenClaw їх не перевизначає.

## Увімкнення або вимкнення

Для provider-ів не Anthropic pruning типово вимкнений. Щоб увімкнути:

```json5
{
  agents: {
    defaults: {
      contextPruning: { mode: "cache-ttl", ttl: "5m" },
    },
  },
}
```

Щоб вимкнути: задайте `mode: "off"`.

## Pruning і Compaction

|            | Pruning                 | Compaction               |
| ---------- | ----------------------- | ------------------------ |
| **Що**     | Обрізає результати інструментів | Підсумовує розмову       |
| **Зберігається?** | Ні (для кожного запиту)   | Так (у transcript)       |
| **Область**| Лише результати інструментів | Уся розмова              |

Вони доповнюють одне одного — pruning підтримує компактність виводу
інструментів між циклами Compaction.
