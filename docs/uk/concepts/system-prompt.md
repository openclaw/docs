OpenClaw збирає власний системний промпт для кожного запуску агента. Промпт **належить OpenClaw** і не використовує стандартний промпт pi-coding-agent.

Промпт збирається OpenClaw і вбудовується в кожен запуск агента.

Plugin провайдерів можуть додавати підказки для промпту з урахуванням кешу, не замінюючи
повний промпт, яким володіє OpenClaw. Runtime провайдера може:

- замінювати невеликий набір названих базових розділів (`interaction_style`,
  `tool_call_style`, `execution_bias`)
- вставляти **стабільний префікс** над межею кешу промпту
- вставляти **динамічний суфікс** під межею кешу промпту

Використовуйте внески, якими володіє провайдер, для налаштування під конкретні сімейства моделей. Зберігайте застарілу
мутацію промпту `before_prompt_build` для сумісності або справді глобальних змін
промпту, а не для звичайної поведінки провайдера.

Оверлей для сімейства OpenAI GPT-5 зберігає базове правило виконання компактним і додає
специфічні для моделі вказівки щодо фіксації на персоні, стислого виводу, дисципліни інструментів,
паралельного пошуку, покриття результатів, перевірки, відсутнього контексту та
гігієни terminal-tool.

## Структура

Промпт навмисно компактний і використовує фіксовані розділи:

- **Tooling**: нагадування про структуровані інструменти як джерело істини плюс вказівки щодо використання інструментів під час виконання.
- **Execution Bias**: стислі вказівки щодо доведення справи до кінця: діяти в межах
  поточного ходу для запитів, які можна виконати, продовжувати до завершення або блокування, відновлюватися після слабких
  результатів інструментів, перевіряти змінний стан наживо й виконувати перевірку перед фіналізацією.
- **Safety**: коротке нагадування про запобіжники, щоб уникати поведінки, спрямованої на захоплення влади, або обходу нагляду.
- **Skills** (коли доступні): пояснює моделі, як завантажувати інструкції Skills за потреби.
- **OpenClaw Self-Update**: як безпечно перевіряти конфігурацію за допомогою
  `config.schema.lookup`, змінювати конфігурацію через `config.patch`, замінювати всю
  конфігурацію через `config.apply` і запускати `update.run` лише за явним
  запитом користувача. Інструмент `gateway`, доступний лише власнику, також відмовляється переписувати
  `tools.exec.ask` / `tools.exec.security`, включно із застарілими
  псевдонімами `tools.bash.*`, які нормалізуються до цих захищених шляхів exec.
- **Workspace**: робочий каталог (`agents.defaults.workspace`).
- **Documentation**: локальний шлях до документації OpenClaw (репозиторій або npm-пакет) і коли її слід читати.
- **Workspace Files (injected)**: вказує, що bootstrap-файли включено нижче.
- **Sandbox** (коли ввімкнено): вказує на sandboxed runtime, шляхи sandbox і те, чи доступний exec із підвищенням привілеїв.
- **Current Date & Time**: локальний для користувача час, часовий пояс і формат часу.
- **Reply Tags**: необов’язковий синтаксис тегів відповіді для підтримуваних провайдерів.
- **Heartbeats**: промпт Heartbeat і поведінка підтвердження, коли Heartbeat увімкнено для агента за замовчуванням.
- **Runtime**: хост, ОС, node, модель, корінь репозиторію (коли виявлено), рівень міркування (один рядок).
- **Reasoning**: поточний рівень видимості + підказка щодо перемикання `/reasoning`.

Розділ Tooling також містить вказівки під час виконання для довготривалої роботи:

- використовуйте cron для майбутніх подальших дій (`check back later`, нагадування, повторювана робота)
  замість циклів sleep через `exec`, трюків із затримкою `yieldMs` або повторного опитування `process`
- використовуйте `exec` / `process` лише для команд, які запускаються зараз і продовжують працювати
  у фоновому режимі
- коли ввімкнено автоматичне пробудження після завершення, запускайте команду один раз і покладайтеся на
  push-орієнтований шлях пробудження, коли вона виводить результат або завершується помилкою
- використовуйте `process` для журналів, стану, введення або втручання, коли вам потрібно
  перевірити команду, що виконується
- якщо завдання більше, віддавайте перевагу `sessions_spawn`; завершення підлеглого агента працює через push і автоматично анонсується назад запитувачу
- не опитуйте `subagents list` / `sessions_list` у циклі лише для очікування
  завершення

Коли ввімкнено експериментальний інструмент `update_plan`, Tooling також вказує
моделі використовувати його лише для нетривіальної багатокрокової роботи, тримати рівно один
крок `in_progress` і не повторювати весь план після кожного оновлення.

Запобіжники безпеки в системному промпті мають рекомендаційний характер. Вони спрямовують поведінку моделі, але не забезпечують примусове виконання політики. Для жорсткого забезпечення використовуйте політику інструментів, схвалення exec, sandboxing і списки дозволених каналів; оператори можуть вимикати це за задумом.

На каналах із нативними картками/кнопками схвалення промпт runtime тепер вказує
агенту спочатку покладатися на цей нативний UI схвалення. Він має включати ручну
команду `/approve` лише тоді, коли результат інструмента каже, що схвалення в чаті недоступні або
ручне схвалення — єдиний шлях.

## Режими промпту

OpenClaw може рендерити менші системні промпти для підлеглих агентів. Runtime встановлює
`promptMode` для кожного запуску (це не користувацька конфігурація):

- `full` (за замовчуванням): включає всі розділи вище.
- `minimal`: використовується для підлеглих агентів; пропускає **Skills**, **Memory Recall**, **OpenClaw
  Self-Update**, **Model Aliases**, **User Identity**, **Reply Tags**,
  **Messaging**, **Silent Replies** і **Heartbeats**. Tooling, **Safety**,
  Workspace, Sandbox, Current Date & Time (коли відомо), Runtime і вбудований
  контекст залишаються доступними.
- `none`: повертає лише базовий рядок ідентичності.

Коли `promptMode=minimal`, додаткові вбудовані промпти позначаються як **Subagent
Context** замість **Group Chat Context**.

## Вбудовування bootstrap робочого простору

Bootstrap-файли обрізаються й додаються під **Project Context**, щоб модель бачила контекст ідентичності та профілю без потреби в явному читанні:

- `AGENTS.md`
- `SOUL.md`
- `TOOLS.md`
- `IDENTITY.md`
- `USER.md`
- `HEARTBEAT.md`
- `BOOTSTRAP.md` (лише в абсолютно нових робочих просторах)
- `MEMORY.md`, коли присутній

Усі ці файли **вбудовуються у вікно контексту** на кожному ході, якщо тільки
не застосовується специфічне обмеження для файлу. `HEARTBEAT.md` пропускається під час звичайних запусків, коли
Heartbeat вимкнено для агента за замовчуванням або
`agents.defaults.heartbeat.includeSystemPromptSection` має значення false. Зберігайте вбудовані
файли стислими — особливо `MEMORY.md`, який із часом може зростати й призводити до
неочікувано високого використання контексту та частішої Compaction.

<Note>
Щоденні файли `memory/*.md` **не** є частиною звичайного bootstrap Project Context. У звичайних ходах до них звертаються за потреби через інструменти `memory_search` і `memory_get`, тому вони не займають місце у вікні контексту, якщо модель явно їх не читає. Виняток — прості ходи `/new` і `/reset`: runtime може додати нещодавню щоденну пам’ять як одноразовий блок стартового контексту для цього першого ходу.
</Note>

Великі файли обрізаються з маркером. Максимальний розмір на файл контролюється через
`agents.defaults.bootstrapMaxChars` (типово: 12000). Загальний обсяг вбудованого bootstrap-вмісту
в усіх файлах обмежується через `agents.defaults.bootstrapTotalMaxChars`
(типово: 60000). Відсутні файли вбудовують короткий маркер відсутнього файлу. Коли відбувається обрізання,
OpenClaw може вбудовувати блок попередження в Project Context; це контролюється через
`agents.defaults.bootstrapPromptTruncationWarning` (`off`, `once`, `always`;
типово: `once`).

Сеанси підлеглих агентів вбудовують лише `AGENTS.md` і `TOOLS.md` (інші bootstrap-файли
відфільтровуються, щоб контекст підлеглого агента залишався малим).

Внутрішні хуки можуть перехоплювати цей крок через `agent:bootstrap`, щоб змінювати або замінювати
вбудовані bootstrap-файли (наприклад, замінювати `SOUL.md` на альтернативну персону).

Якщо ви хочете, щоб агент звучав менш узагальнено, почніть із
[Посібник із персональності SOUL.md](/uk/concepts/soul).

Щоб перевірити, який внесок робить кожен вбудований файл (сирий проти вбудованого, обрізання, а також накладні витрати схеми інструментів), використовуйте `/context list` або `/context detail`. Див. [Контекст](/uk/concepts/context).

## Обробка часу

Системний промпт містить окремий розділ **Current Date & Time**, коли
часовий пояс користувача відомий. Щоб зробити кеш промпту стабільним, тепер він містить лише
**часовий пояс** (без динамічного годинника чи формату часу).

Використовуйте `session_status`, коли агенту потрібен поточний час; картка стану
містить рядок із часовою міткою. Той самий інструмент також може за бажанням встановлювати
перевизначення моделі для сеансу (`model=default` очищає його).

Налаштовується через:

- `agents.defaults.userTimezone`
- `agents.defaults.timeFormat` (`auto` | `12` | `24`)

Див. [Дата й час](/uk/date-time) для повних деталей поведінки.

## Skills

Коли існують допустимі Skills, OpenClaw вбудовує компактний **список доступних Skills**
(`formatSkillsForPrompt`), який включає **шлях до файлу** для кожного Skill. Промпт
вказує моделі використовувати `read`, щоб завантажити SKILL.md у зазначеному
розташуванні (workspace, керованому або вбудованому). Якщо допустимих Skills немає, розділ
Skills пропускається.

Допустимість включає обмеження метаданих Skill, перевірки runtime-середовища/конфігурації
та ефективний список дозволених Skills агента, коли налаштовано `agents.defaults.skills` або
`agents.list[].skills`.

Skills, вбудовані в Plugin, є допустимими лише тоді, коли їхній Plugin-власник увімкнений.
Це дозволяє Plugin інструментів відкривати глибші операційні посібники без вбудовування
усіх цих вказівок безпосередньо в опис кожного інструмента.

```
<available_skills>
  <skill>
    <name>...</name>
    <description>...</description>
    <location>...</location>
  </skill>
</available_skills>
```

Це зберігає базовий промпт малим, водночас даючи змогу націленого використання Skills.

Бюджет списку Skills належить підсистемі Skills:

- Глобальне значення за замовчуванням: `skills.limits.maxSkillsPromptChars`
- Перевизначення для агента: `agents.list[].skillsLimits.maxSkillsPromptChars`

Загальні обмежені уривки runtime використовують іншу поверхню:

- `agents.defaults.contextLimits.*`
- `agents.list[].contextLimits.*`

Такий поділ зберігає розмір Skills окремо від розміру читання/вбудовування під час runtime, наприклад
`memory_get`, живих результатів інструментів і оновлення AGENTS.md після Compaction.

## Documentation

Системний промпт включає розділ **Documentation**. Коли локальна документація доступна, він
вказує на локальний каталог документації OpenClaw (`docs/` у checkout Git або документація,
вбудована в npm-пакет). Якщо локальна документація недоступна, використовується резервний варіант
[https://docs.openclaw.ai](https://docs.openclaw.ai).

Цей самий розділ також включає розташування джерел OpenClaw. Checkout Git показують локальний
корінь джерел, щоб агент міг безпосередньо перевіряти код. Встановлення з пакетів включають URL
джерел на GitHub і кажуть агенту переглядати джерела там, коли документація неповна або
застаріла. Промпт також згадує публічне дзеркало документації, спільноту Discord і ClawHub
([https://clawhub.ai](https://clawhub.ai)) для пошуку Skills. Він вказує моделі
спочатку звертатися до документації щодо поведінки, команд, конфігурації або архітектури OpenClaw і
самостійно запускати `openclaw status`, коли це можливо (звертаючись до користувача лише тоді, коли доступу немає).
Зокрема для конфігурації він спрямовує агентів до дії інструмента `gateway`
`config.schema.lookup` для точної документації та обмежень на рівні полів, а потім до
`docs/gateway/configuration.md` і `docs/gateway/configuration-reference.md`
для ширших вказівок.
