OpenClaw створює власний системний промпт для кожного запуску агента. Промпт **належить OpenClaw** і не використовує стандартний промпт pi-coding-agent.

Промпт збирається OpenClaw і впроваджується в кожен запуск агента.

Plugin-и постачальників можуть додавати cache-aware інструкції для промпта, не замінюючи
повний промпт, що належить OpenClaw. Середовище виконання постачальника може:

- замінювати невеликий набір іменованих основних розділів (`interaction_style`,
  `tool_call_style`, `execution_bias`)
- впроваджувати **стабільний префікс** над межею кешу промпта
- впроваджувати **динамічний суфікс** під межею кешу промпта

Використовуйте внески, що належать постачальнику, для налаштування під конкретні сімейства моделей. Залишайте застарілу
мутацію промпта `before_prompt_build` для сумісності або справді глобальних змін
промпта, а не для звичайної поведінки постачальника.

Оверлей сімейства OpenAI GPT-5 тримає основне правило виконання невеликим і додає
модель-специфічні інструкції для закріплення персони, лаконічного виводу, дисципліни інструментів,
паралельного пошуку, покриття результатів, перевірки, відсутнього контексту та
гігієни термінальних інструментів.

## Структура

Промпт навмисно компактний і використовує фіксовані розділи:

- **Інструменти**: нагадування про джерело істини для структурованих інструментів плюс runtime-інструкції щодо використання інструментів.
- **Схильність до виконання**: компактні інструкції для доведення справ до кінця: діяти в межах ходу за
  придатними до виконання запитами, продовжувати до завершення або блокування, відновлюватися після слабких
  результатів інструментів, перевіряти змінний стан наживо та перевіряти перед фіналізацією.
- **Безпека**: коротке нагадування про запобіжники, щоб уникати поведінки, спрямованої на здобуття влади, або обходу нагляду.
- **Skills** (коли доступні): пояснює моделі, як завантажувати інструкції Skills на вимогу.
- **Самооновлення OpenClaw**: як безпечно перевіряти конфігурацію за допомогою
  `config.schema.lookup`, змінювати конфігурацію через `config.patch`, замінювати повну
  конфігурацію через `config.apply` і запускати `update.run` лише за явним запитом
  користувача. Інструмент лише для власника `gateway` також відмовляється перезаписувати
  `tools.exec.ask` / `tools.exec.security`, включно із застарілими псевдонімами `tools.bash.*`,
  які нормалізуються до цих захищених шляхів exec.
- **Робочий простір**: робоча директорія (`agents.defaults.workspace`).
- **Документація**: локальний шлях до документації OpenClaw (репозиторій або npm-пакет) і коли її читати.
- **Файли робочого простору (впроваджені)**: вказує, що bootstrap-файли включені нижче.
- **Пісочниця** (коли ввімкнена): вказує sandboxed runtime, шляхи пісочниці та чи доступний elevated exec.
- **Поточна дата й час**: локальний час користувача, часовий пояс і формат часу.
- **Теги відповіді**: необов'язковий синтаксис тегів відповіді для підтримуваних постачальників.
- **Heartbeats**: промпт Heartbeat і поведінка ack, коли Heartbeats увімкнені для стандартного агента.
- **Runtime**: host, ОС, node, модель, корінь репозиторію (коли виявлено), рівень мислення (один рядок).
- **Міркування**: поточний рівень видимості + підказка перемикача /reasoning.

OpenClaw тримає великий стабільний вміст, включно з **Контекстом проєкту**, над
внутрішньою межею кешу промпта. Мінливі розділи каналу/сесії, як-от
інструкції для вбудовування Control UI, **Повідомлення**, **Голос**, **Контекст групового чату**,
**Реакції**, **Heartbeats** і **Runtime**, додаються під цією межею,
щоб локальні бекенди з префіксними кешами могли повторно використовувати стабільний префікс робочого простору
між ходами каналу. Описи інструментів так само мають уникати вбудовування поточних
назв каналів, коли прийнята схема вже передає цю runtime-деталь.

Розділ Інструменти також містить runtime-інструкції для довготривалої роботи:

- використовуйте cron для майбутнього follow-up (`check back later`, нагадування, повторювана робота)
  замість циклів sleep в `exec`, трюків із затримкою `yieldMs` або повторного опитування `process`
- використовуйте `exec` / `process` лише для команд, які стартують зараз і продовжують працювати
  у фоновому режимі
- коли ввімкнене автоматичне пробудження після завершення, запустіть команду один раз і покладайтеся на
  push-based шлях пробудження, коли вона виводить дані або завершується з помилкою
- використовуйте `process` для журналів, статусу, введення або втручання, коли потрібно
  перевірити запущену команду
- якщо завдання більше, надавайте перевагу `sessions_spawn`; завершення субагента є
  push-based і автоматично оголошується назад запитувачу
- не опитуйте `subagents list` / `sessions_list` у циклі лише для очікування
  завершення

Коли експериментальний інструмент `update_plan` увімкнено, Інструменти також вказують
моделі використовувати його лише для нетривіальної багатоетапної роботи, тримати рівно один
крок `in_progress` і уникати повторення всього плану після кожного оновлення.

Запобіжники безпеки в системному промпті є рекомендаційними. Вони спрямовують поведінку моделі, але не забезпечують примусове застосування політики. Для жорсткого примусу використовуйте політики інструментів, схвалення exec, пісочницю та allowlist-и каналів; оператори можуть вимикати їх за дизайном.

На каналах із нативними картками/кнопками схвалення runtime-промпт тепер вказує
агенту спершу покладатися на цей нативний UI схвалення. Він має включати ручну
команду `/approve` лише коли результат інструмента каже, що схвалення в чаті недоступні або
ручне схвалення є єдиним шляхом.

## Режими промпта

OpenClaw може рендерити менші системні промпти для субагентів. Runtime задає
`promptMode` для кожного запуску (це не користувацька конфігурація):

- `full` (стандартно): містить усі розділи вище.
- `minimal`: використовується для субагентів; пропускає **Skills**, **Memory Recall**, **Самооновлення OpenClaw**,
  **Псевдоніми моделей**, **Ідентичність користувача**, **Теги відповіді**,
  **Повідомлення**, **Тихі відповіді** та **Heartbeats**. Інструменти, **Безпека**,
  Робочий простір, Пісочниця, Поточна дата й час (коли відомі), Runtime і впроваджений
  контекст залишаються доступними.
- `none`: повертає лише базовий рядок ідентичності.

Коли `promptMode=minimal`, додаткові впроваджені промпти позначаються як **Контекст субагента**
замість **Контекст групового чату**.

Для запусків автоматичної відповіді в каналі OpenClaw може пропускати загальний розділ **Тихі відповіді**,
коли контекст прямого/групового чату вже містить визначену для конкретної розмови
поведінку `NO_REPLY`. Це уникає повторення механіки токенів
і в глобальному системному промпті, і в контексті каналу.

## Знімки промптів

OpenClaw зберігає закомічені happy-path знімки промптів для runtime Codex/message-tool
у `test/fixtures/agents/prompt-snapshots/happy-path/`. Вони рендерять
інструкції розробника для app-server Codex, що належать OpenClaw, вибрані параметри
старту/відновлення треду, користувацький ввід ходу та динамічні специфікації інструментів для прямих повідомлень Telegram,
груп Discord і ходів Heartbeat. Прихований базовий системний промпт Codex і
turn-scoped інструкції режиму співпраці Codex належать runtime Codex
і не рендеряться OpenClaw.

Перегенеруйте їх за допомогою `pnpm prompt:snapshots:gen` і перевірте drift через
`pnpm prompt:snapshots:check`.

## Впровадження bootstrap робочого простору

Bootstrap-файли обрізаються та додаються в **Контекст проєкту**, щоб модель бачила контекст ідентичності та профілю без потреби в явному читанні:

- `AGENTS.md`
- `SOUL.md`
- `TOOLS.md`
- `IDENTITY.md`
- `USER.md`
- `HEARTBEAT.md`
- `BOOTSTRAP.md` (лише в абсолютно нових робочих просторах)
- `MEMORY.md`, коли наявний

Усі ці файли **впроваджуються у вікно контексту** на кожному ході, якщо
не застосовується специфічний для файлу gate. `HEARTBEAT.md` пропускається у звичайних запусках, коли
Heartbeats вимкнені для стандартного агента або
`agents.defaults.heartbeat.includeSystemPromptSection` дорівнює false. Тримайте впроваджені
файли лаконічними — особливо `MEMORY.md`, який може зростати з часом і призводити до
неочікувано високого використання контексту та частішої Compaction.

<Note>
Щоденні файли `memory/*.md` **не** є частиною звичайного bootstrap Контексту проєкту. У звичайних ходах до них звертаються на вимогу через інструменти `memory_search` і `memory_get`, тож вони не рахуються у вікно контексту, якщо модель явно їх не читає. Виняток — чисті ходи `/new` і `/reset`: runtime може додати недавню щоденну пам'ять на початок як одноразовий блок стартового контексту для цього першого ходу.
</Note>

Великі файли обрізаються з маркером. Максимальний розмір на файл керується
`agents.defaults.bootstrapMaxChars` (стандартно: 12000). Загальний впроваджений bootstrap-вміст
по всіх файлах обмежується `agents.defaults.bootstrapTotalMaxChars`
(стандартно: 60000). Відсутні файли впроваджують короткий маркер відсутнього файлу. Коли відбувається обрізання,
OpenClaw може впровадити блок попередження в Контекст проєкту; керуйте цим через
`agents.defaults.bootstrapPromptTruncationWarning` (`off`, `once`, `always`;
стандартно: `once`).

Сесії субагентів впроваджують лише `AGENTS.md` і `TOOLS.md` (інші bootstrap-файли
відфільтровуються, щоб контекст субагента був невеликим).

Внутрішні hooks можуть перехоплювати цей крок через `agent:bootstrap`, щоб змінити або замінити
впроваджені bootstrap-файли (наприклад, замінивши `SOUL.md` на альтернативну персону).

Якщо ви хочете, щоб агент звучав менш загально, почніть із
[Посібника з особистості SOUL.md](/uk/concepts/soul).

Щоб перевірити, скільки додає кожен впроваджений файл (raw проти injected, обрізання, плюс overhead схеми інструментів), використовуйте `/context list` або `/context detail`. Див. [Контекст](/uk/concepts/context).

## Обробка часу

Системний промпт містить окремий розділ **Поточна дата й час**, коли
часовий пояс користувача відомий. Щоб промпт залишався cache-stable, тепер він містить лише
**часовий пояс** (без динамічного годинника або формату часу).

Використовуйте `session_status`, коли агенту потрібен поточний час; картка статусу
містить рядок із timestamp. Той самий інструмент може необов'язково встановити для сесії
перевизначення моделі (`model=default` очищає його).

Налаштовується через:

- `agents.defaults.userTimezone`
- `agents.defaults.timeFormat` (`auto` | `12` | `24`)

Повні подробиці поведінки див. у [Дата й час](/uk/date-time).

## Skills

Коли існують придатні Skills, OpenClaw впроваджує компактний **список доступних Skills**
(`formatSkillsForPrompt`), який містить **шлях до файлу** для кожного Skill. Промпт
інструктує модель використовувати `read`, щоб завантажити SKILL.md у вказаному
розташуванні (робочий простір, керований або bundled). Якщо жодні Skills не придатні, розділ
Skills пропускається.

Придатність включає gates метаданих Skill, перевірки runtime-середовища/конфігурації
та ефективний allowlist Skills агента, коли налаштовано `agents.defaults.skills` або
`agents.list[].skills`.

Skills, bundled із Plugin, придатні лише коли їхній Plugin-власник увімкнений.
Це дає змогу Plugin-ам інструментів надавати глибші операційні посібники без вбудовування всіх
цих інструкцій безпосередньо в кожен опис інструмента.

```
<available_skills>
  <skill>
    <name>...</name>
    <description>...</description>
    <location>...</location>
  </skill>
</available_skills>
```

Це тримає базовий промпт малим і водночас уможливлює цільове використання Skills.

Бюджетом списку Skills володіє підсистема Skills:

- Глобальне стандартне значення: `skills.limits.maxSkillsPromptChars`
- Перевизначення для агента: `agents.list[].skillsLimits.maxSkillsPromptChars`

Загальні обмежені runtime-витяги використовують іншу поверхню:

- `agents.defaults.contextLimits.*`
- `agents.list[].contextLimits.*`

Це розділення тримає розмір Skills окремо від розміру runtime-читання/впровадження, як-от
`memory_get`, результати live-інструментів і оновлення AGENTS.md після Compaction.

## Документація

Системний промпт містить розділ **Документація**. Коли локальна документація доступна, він
вказує на локальну директорію документації OpenClaw (`docs/` у Git checkout або документацію bundled npm
пакета). Якщо локальна документація недоступна, він повертається до
[https://docs.openclaw.ai](https://docs.openclaw.ai).

Той самий розділ також містить розташування джерельного коду OpenClaw. Git checkout-и показують локальний
корінь джерел, щоб агент міг безпосередньо перевіряти код. Встановлення з пакета містять GitHub
URL джерел і вказують агенту переглядати джерела там щоразу, коли документація неповна або
застаріла. Промпт також згадує публічне дзеркало документації, community Discord і ClawHub
([https://clawhub.ai](https://clawhub.ai)) для пошуку Skills. Він вказує моделі
спершу звертатися до документації щодо поведінки, команд, конфігурації або архітектури OpenClaw і
самостійно запускати `openclaw status`, коли можливо (питаючи користувача лише коли не має доступу).
Для конфігурації зокрема він спрямовує агентів до дії інструмента `gateway`
`config.schema.lookup` для точних field-level документів і обмежень, а потім до
`docs/gateway/configuration.md` і `docs/gateway/configuration-reference.md`
для ширших інструкцій.
