OpenClaw будує кастомний системний prompt для кожного запуску агента. Prompt **належить OpenClaw** і не використовує типовий prompt `pi-coding-agent`.

Prompt збирається OpenClaw і впроваджується в кожен запуск агента.

Plugin-и provider-ів можуть додавати підказки для prompt, сумісні з кешуванням, не замінюючи
повністю prompt, який належить OpenClaw. Runtime provider-а може:

- замінювати невеликий набір іменованих базових секцій (`interaction_style`,
  `tool_call_style`, `execution_bias`)
- впроваджувати **стабільний префікс** над межею prompt cache
- впроваджувати **динамічний суфікс** під межею prompt cache

Використовуйте внески provider-а для налаштування під конкретні сімейства моделей. Зберігайте застарілу
мутацію prompt `before_prompt_build` для сумісності або для справді глобальних змін prompt, а не для звичайної поведінки provider-а.

Overlay для сімейства OpenAI GPT-5 тримає базове правило виконання компактним і додає
специфічні для моделі настанови щодо фіксації persona, стислого виводу, дисципліни інструментів,
паралельного пошуку, повноти deliverable, перевірки, відсутнього контексту та
гігієни terminal-інструментів.

## Структура

Prompt навмисно компактний і використовує фіксовані секції:

- **Tooling**: нагадування про source-of-truth для структурованих інструментів плюс настанови runtime щодо використання інструментів.
- **Execution Bias**: компактні настанови щодо доведення справи до кінця: діяти в тому самому ході для
  запитів, з якими можна працювати, продовжувати до завершення або блокування, відновлюватися після слабких результатів
  інструментів, перевіряти змінний стан наживо та верифікувати перед фіналізацією.
- **Safety**: коротке нагадування про guardrail-и, щоб уникати поведінки, спрямованої на здобуття влади, або обходу нагляду.
- **Skills** (коли доступні): повідомляє моделі, як завантажувати інструкції Skill на вимогу.
- **OpenClaw Self-Update**: як безпечно переглядати config через
  `config.schema.lookup`, змінювати config через `config.patch`, замінювати всю
  config через `config.apply` і запускати `update.run` лише за явним запитом
  користувача. Інструмент `gateway`, доступний лише власнику, також відмовляється переписувати
  `tools.exec.ask` / `tools.exec.security`, включно із застарілими
  псевдонімами `tools.bash.*`, які нормалізуються до цих захищених exec-шляхів.
- **Workspace**: робочий каталог (`agents.defaults.workspace`).
- **Documentation**: локальний шлях до документації OpenClaw (репозиторій або npm package) і коли її читати.
- **Workspace Files (injected)**: вказує, що bootstrap-файли включені нижче.
- **Sandbox** (коли ввімкнено): вказує sandboxed runtime, шляхи пісочниці та те, чи доступний elevated exec.
- **Current Date & Time**: локальний час користувача, часовий пояс і формат часу.
- **Reply Tags**: необов’язковий синтаксис тегів відповіді для підтримуваних provider-ів.
- **Heartbeats**: prompt Heartbeat і поведінка ack, коли Heartbeat-и ввімкнено для типового агента.
- **Runtime**: host, OS, Node, модель, корінь репозиторію (якщо виявлено), рівень thinking (одним рядком).
- **Reasoning**: поточний рівень видимості + підказка щодо перемикання `/reasoning`.

Секція Tooling також містить настанови runtime для довготривалої роботи:

- використовуйте Cron для майбутніх follow-up (`check back later`, нагадування, періодична робота)
  замість sleep-циклів через `exec`, трюків затримки `yieldMs` або повторного
  polling через `process`
- використовуйте `exec` / `process` лише для команд, які стартують зараз і продовжують працювати
  у фоновому режимі
- коли увімкнено автоматичне пробудження після завершення, запускайте команду один раз і покладайтеся на
  push-базований шлях пробудження, коли вона виводить результат або зазнає помилки
- використовуйте `process` для логів, стану, вводу або втручання, коли вам потрібно
  перевірити запущену команду
- якщо завдання більше, надавайте перевагу `sessions_spawn`; завершення субагента працює на основі push і автоматично оголошується назад запитувачу
- не виконуйте `subagents list` / `sessions_list` у циклі лише для очікування
  завершення

Коли ввімкнено експериментальний інструмент `update_plan`, секція Tooling також повідомляє
моделі використовувати його лише для нетривіальної багатокрокової роботи, тримати рівно один крок
`in_progress` і не повторювати весь план після кожного оновлення.

Safety guardrail-и в системному prompt мають рекомендаційний характер. Вони скеровують поведінку моделі, але не забезпечують політику примусово. Для жорсткого забезпечення використовуйте політику інструментів, exec approvals, sandboxing і allowlist-и каналів; оператори за задумом можуть вимикати їх.

На каналах із нативними картками/кнопками схвалення runtime prompt тепер повідомляє
агенту спочатку покладатися на цей нативний UI схвалення. Він має включати ручну
команду `/approve` лише коли результат інструмента каже, що chat approvals недоступні або
ручне схвалення є єдиним шляхом.

## Режими prompt

OpenClaw може рендерити менші системні prompt-и для субагентів. Runtime встановлює
`promptMode` для кожного запуску (це не користувацький параметр конфігурації):

- `full` (типово): включає всі секції вище.
- `minimal`: використовується для субагентів; опускає **Skills**, **Memory Recall**, **OpenClaw
  Self-Update**, **Model Aliases**, **User Identity**, **Reply Tags**,
  **Messaging**, **Silent Replies** і **Heartbeats**. Tooling, **Safety**,
  Workspace, Sandbox, Current Date & Time (коли відомо), Runtime та впроваджений
  контекст залишаються доступними.
- `none`: повертає лише базовий рядок ідентичності.

Коли `promptMode=minimal`, додаткові впроваджені prompt-и позначаються як **Subagent
Context**, а не **Group Chat Context**.

## Впровадження bootstrap workspace

Bootstrap-файли обрізаються і додаються в розділ **Project Context**, щоб модель бачила контекст ідентичності та профілю без явних читань:

- `AGENTS.md`
- `SOUL.md`
- `TOOLS.md`
- `IDENTITY.md`
- `USER.md`
- `HEARTBEAT.md`
- `BOOTSTRAP.md` (лише для абсолютно нових workspace)
- `MEMORY.md`, якщо він присутній

Усі ці файли **впроваджуються у вікно контексту** на кожному ході, якщо
не діє специфічний для файла шлюз. `HEARTBEAT.md` пропускається у звичайних запусках, коли
Heartbeat-и вимкнено для типового агента або
`agents.defaults.heartbeat.includeSystemPromptSection` має значення false. Тримайте впроваджені
файли стислими — особливо `MEMORY.md`, який із часом може розростатися і призводити до
неочікувано високого використання контексту та частішого Compaction.

> **Примітка:** щоденні файли `memory/*.md` **не** є частиною звичайного bootstrap
> Project Context. На звичайних ходах доступ до них відбувається на вимогу через
> інструменти `memory_search` і `memory_get`, тож вони не враховуються у
> вікні контексту, якщо модель явно їх не читає. Виняток — прості ходи `/new` і
> `/reset`: runtime може додавати недавню щоденну memory як одноразовий блок startup-контексту для цього першого ходу.

Великі файли обрізаються з маркером. Максимальний розмір для кожного файла керується
через `agents.defaults.bootstrapMaxChars` (типово: 12000). Загальний обсяг впровадженого bootstrap-вмісту
для всіх файлів обмежується через `agents.defaults.bootstrapTotalMaxChars`
(типово: 60000). Відсутні файли впроваджують короткий маркер відсутнього файла. Коли відбувається обрізання,
OpenClaw може впроваджувати попереджувальний блок у Project Context; це керується через
`agents.defaults.bootstrapPromptTruncationWarning` (`off`, `once`, `always`;
типово: `once`).

Sessions субагентів впроваджують лише `AGENTS.md` і `TOOLS.md` (інші bootstrap-файли
відфільтровуються, щоб зберегти малий контекст субагента).

Внутрішні хуки можуть перехоплювати цей крок через `agent:bootstrap`, щоб змінювати або замінювати
впроваджені bootstrap-файли (наприклад, підміняти `SOUL.md` альтернативною persona).

Якщо ви хочете, щоб агент звучав менш шаблонно, почніть із
[Посібника з особистості SOUL.md](/uk/concepts/soul).

Щоб перевірити, який внесок робить кожен впроваджений файл (raw проти injected, обрізання, а також накладні витрати schema інструментів), використовуйте `/context list` або `/context detail`. Див. [Контекст](/uk/concepts/context).

## Обробка часу

Системний prompt містить окрему секцію **Current Date & Time**, коли
часовий пояс користувача відомий. Щоб зберегти стабільність prompt cache, тепер вона включає лише
**часовий пояс** (без динамічного годинника або формату часу).

Використовуйте `session_status`, коли агенту потрібен поточний час; картка стану
містить рядок timestamp. Той самий інструмент також може за бажанням встановити перевизначення моделі для session
(`model=default` очищує його).

Налаштовується через:

- `agents.defaults.userTimezone`
- `agents.defaults.timeFormat` (`auto` | `12` | `24`)

Повні деталі поведінки див. в [Дата й час](/uk/date-time).

## Skills

Коли існують придатні Skills, OpenClaw впроваджує компактний **список доступних Skills**
(`formatSkillsForPrompt`), який містить **шлях до файла** для кожного Skill. Prompt
інструктує модель використовувати `read`, щоб завантажити `SKILL.md` у вказаному
розташуванні (workspace, managed або bundled). Якщо придатних Skills немає, секція
Skills пропускається.

Придатність включає шлюзи метаданих Skill, перевірки runtime environment/config
і ефективний allowlist Skills агента, коли налаштовано `agents.defaults.skills` або
`agents.list[].skills`.

```
<available_skills>
  <skill>
    <name>...</name>
    <description>...</description>
    <location>...</location>
  </skill>
</available_skills>
```

Це зберігає малий базовий prompt, водночас дозволяючи цільове використання Skills.

Бюджет списку Skills належить підсистемі Skills:

- Глобальне типове значення: `skills.limits.maxSkillsPromptChars`
- Перевизначення для окремого агента: `agents.list[].skillsLimits.maxSkillsPromptChars`

Звичайні обмежені runtime-excerpts використовують іншу поверхню:

- `agents.defaults.contextLimits.*`
- `agents.list[].contextLimits.*`

Такий поділ утримує розмір Skills окремо від обмеження runtime для читання/впровадження, таких як
`memory_get`, живі результати інструментів і оновлення `AGENTS.md` після Compaction.

## Documentation

Коли це доступно, системний prompt містить секцію **Documentation**, яка вказує на
локальний каталог документації OpenClaw (або `docs/` у workspace репозиторію, або документацію
з bundled npm package), а також згадує публічне дзеркало, source repo, спільноту Discord і
ClawHub ([https://clawhub.ai](https://clawhub.ai)) для пошуку Skills. Prompt інструктує модель спершу звертатися до локальної документації
щодо поведінки, команд, конфігурації або архітектури OpenClaw, а також самостійно запускати
`openclaw status`, коли це можливо (і звертатися до користувача лише тоді, коли в нього немає доступу).
