# Системний промпт

OpenClaw створює власний системний промпт для кожного запуску агента. Промпт **належить OpenClaw** і не використовує стандартний промпт pi-coding-agent.

Промпт збирається OpenClaw і впроваджується в кожен запуск агента.

Плагіни провайдерів можуть додавати cache-aware настанови для промпту, не замінюючи весь промпт, що належить OpenClaw. Середовище виконання провайдера може:

- замінювати невеликий набір іменованих базових розділів (`interaction_style`,
  `tool_call_style`, `execution_bias`)
- впроваджувати **стабільний префікс** над межею кешу промпту
- впроваджувати **динамічний суфікс** під межею кешу промпту

Використовуйте внески, що належать провайдеру, для налаштування під конкретне сімейство моделей. Зберігайте застарілу мутацію промпту `before_prompt_build` для сумісності або справді глобальних змін промпту, а не для звичайної поведінки провайдера.

## Структура

Промпт навмисно компактний і використовує фіксовані розділи:

- **Інструменти**: нагадування про джерело істини для структурованих інструментів і настанови щодо використання інструментів під час виконання.
- **Безпека**: коротке нагадування про запобіжні обмеження, щоб уникати поведінки, спрямованої на здобуття влади, або обходу нагляду.
- **Skills** (коли доступні): пояснює моделі, як завантажувати інструкції Skills за потреби.
- **Самооновлення OpenClaw**: як безпечно перевіряти конфігурацію за допомогою
  `config.schema.lookup`, вносити зміни до конфігурації через `config.patch`, замінювати повну
  конфігурацію через `config.apply` і запускати `update.run` лише за явним запитом
  користувача. Інструмент `gateway`, доступний лише власнику, також відмовляється переписувати
  `tools.exec.ask` / `tools.exec.security`, зокрема застарілі
  псевдоніми `tools.bash.*`, які нормалізуються до цих захищених шляхів exec.
- **Робочий простір**: робочий каталог (`agents.defaults.workspace`).
- **Документація**: локальний шлях до документації OpenClaw (репозиторій або npm package) і коли її слід читати.
- **Файли робочого простору (впроваджено)**: вказує, що файли bootstrap включено нижче.
- **Пісочниця** (коли ввімкнено): вказує на ізольоване середовище виконання, шляхи пісочниці й те, чи доступний exec з підвищеними правами.
- **Поточна дата й час**: локальний час користувача, часовий пояс і формат часу.
- **Теги відповіді**: необов’язковий синтаксис тегів відповіді для підтримуваних провайдерів.
- **Heartbeat**: промпт Heartbeat і поведінка підтвердження, коли Heartbeat увімкнено для стандартного агента.
- **Середовище виконання**: хост, ОС, node, модель, корінь репозиторію (коли виявлено), рівень thinking (один рядок).
- **Міркування**: поточний рівень видимості + підказка для перемикання `/reasoning`.

Розділ «Інструменти» також містить настанови для тривалої роботи під час виконання:

- використовуйте Cron для майбутніх повторних дій (`check back later`, нагадування, періодична робота),
  а не цикли сну через `exec`, трюки із затримкою `yieldMs` або повторне опитування `process`
- використовуйте `exec` / `process` лише для команд, які запускаються зараз і продовжують працювати
  у фоновому режимі
- коли ввімкнено автоматичне пробудження після завершення, запускайте команду один раз і покладайтеся на
  push-based механізм пробудження, коли вона виводить результат або завершується помилкою
- використовуйте `process` для журналів, стану, введення або втручання, коли потрібно
  перевірити виконувану команду
- якщо завдання більше, віддавайте перевагу `sessions_spawn`; завершення субагента відбувається через
  push-based механізм і автоматично повідомляється назад запитувачу
- не опитуйте `subagents list` / `sessions_list` у циклі лише для очікування
  завершення

Коли ввімкнено експериментальний інструмент `update_plan`, розділ «Інструменти» також повідомляє
моделі використовувати його лише для нетривіальної багатокрокової роботи, підтримувати рівно один крок
`in_progress` і не повторювати весь план після кожного оновлення.

Запобіжні обмеження безпеки в системному промпті мають рекомендаційний характер. Вони спрямовують поведінку моделі, але не забезпечують примусове виконання політики. Для жорсткого забезпечення використовуйте політику інструментів, погодження exec, пісочницю та allowlists каналів; оператори можуть вимкнути це за задумом.

У каналах із вбудованими картками/кнопками погодження промпт середовища виконання тепер повідомляє
агенту спочатку покладатися на цей вбудований інтерфейс погодження. Він має включати ручну
команду `/approve` лише тоді, коли результат інструмента повідомляє, що погодження в чаті недоступні або
ручне погодження — єдиний шлях.

## Режими промпту

OpenClaw може формувати менші системні промпти для субагентів. Середовище виконання встановлює
`promptMode` для кожного запуску (це не користувацьке налаштування):

- `full` (типово): включає всі розділи вище.
- `minimal`: використовується для субагентів; пропускає **Skills**, **Виклик пам’яті**, **Самооновлення OpenClaw**, **Псевдоніми моделей**, **Ідентичність користувача**, **Теги відповіді**,
  **Обмін повідомленнями**, **Тихі відповіді** і **Heartbeat**. Розділи «Інструменти», **Безпека**,
  «Робочий простір», «Пісочниця», «Поточна дата й час» (коли відомо), «Середовище виконання» та впроваджений
  контекст залишаються доступними.
- `none`: повертає лише базовий рядок ідентичності.

Коли `promptMode=minimal`, додаткові впроваджені промпти позначаються як **Контекст субагента**, а не **Контекст групового чату**.

## Впровадження bootstrap робочого простору

Файли bootstrap обрізаються й додаються в розділ **Контекст проєкту**, щоб модель бачила контекст ідентичності та профілю без потреби в явному читанні:

- `AGENTS.md`
- `SOUL.md`
- `TOOLS.md`
- `IDENTITY.md`
- `USER.md`
- `HEARTBEAT.md`
- `BOOTSTRAP.md` (лише у зовсім нових робочих просторах)
- `MEMORY.md`, якщо наявний, інакше `memory.md` як резервний варіант у нижньому регістрі

Усі ці файли **впроваджуються у вікно контексту** на кожному ході, якщо не застосовується окремий файловий бар’єр. `HEARTBEAT.md` пропускається під час звичайних запусків, коли Heartbeat вимкнено для стандартного агента або
`agents.defaults.heartbeat.includeSystemPromptSection` має значення false. Зберігайте впроваджені
файли стислими — особливо `MEMORY.md`, який з часом може зростати й призводити до
неочікувано високого використання контексту та частішого Compaction.

> **Примітка:** щоденні файли `memory/*.md` **не** входять до звичайного bootstrap
> Контексту проєкту. Під час звичайних ходів доступ до них відбувається за потреби через
> інструменти `memory_search` і `memory_get`, тому вони не займають
> вікно контексту, доки модель явно їх не прочитає. Винятком є прості ходи `/new` і
> `/reset`: середовище виконання може додавати недавню щоденну пам’ять як одноразовий блок
> стартового контексту для цього першого ходу.

Великі файли обрізаються з маркером. Максимальний розмір на файл контролюється через
`agents.defaults.bootstrapMaxChars` (типово: 20000). Загальний обсяг упровадженого bootstrap-вмісту
для всіх файлів обмежується `agents.defaults.bootstrapTotalMaxChars`
(типово: 150000). Для відсутніх файлів упроваджується короткий маркер відсутнього файла. Коли відбувається обрізання,
OpenClaw може впровадити блок попередження в Контекст проєкту; це контролюється через
`agents.defaults.bootstrapPromptTruncationWarning` (`off`, `once`, `always`;
типово: `once`).

Сеанси субагентів упроваджують лише `AGENTS.md` і `TOOLS.md` (інші bootstrap-файли
відфільтровуються, щоб контекст субагента залишався малим).

Внутрішні hooks можуть перехоплювати цей крок через `agent:bootstrap`, щоб змінювати або замінювати
впроваджені bootstrap-файли (наприклад, замінювати `SOUL.md` на альтернативну персону).

Якщо ви хочете, щоб агент звучав менш узагальнено, почніть із
[Посібника з особистості SOUL.md](/uk/concepts/soul).

Щоб перевірити, який внесок робить кожен впроваджений файл (сирий та впроваджений розмір, обрізання, а також накладні витрати схеми інструментів), використовуйте `/context list` або `/context detail`. Див. [Контекст](/uk/concepts/context).

## Обробка часу

Системний промпт містить окремий розділ **Поточна дата й час**, коли відомий
часовий пояс користувача. Щоб зберегти стабільність кешу промпту, тепер він містить лише
**часовий пояс** (без динамічного годинника чи формату часу).

Використовуйте `session_status`, коли агенту потрібен поточний час; картка стану
містить рядок із часовою позначкою. Той самий інструмент також може за потреби встановити перевизначення моделі для сеансу
(`model=default` очищає його).

Налаштовується через:

- `agents.defaults.userTimezone`
- `agents.defaults.timeFormat` (`auto` | `12` | `24`)

Деталі повної поведінки див. у [Дата й час](/uk/date-time).

## Skills

Коли існують відповідні Skills, OpenClaw впроваджує компактний **список доступних Skills**
(`formatSkillsForPrompt`), який включає **шлях до файла** для кожного skill. Промпт
інструктує модель використовувати `read` для завантаження SKILL.md за вказаним
розташуванням (робочий простір, кероване або вбудоване). Якщо відповідних Skills немає,
розділ Skills пропускається.

Відповідність включає перевірки метаданих skill, перевірки середовища виконання/конфігурації
та ефективний allowlist Skills агента, коли налаштовано `agents.defaults.skills` або
`agents.list[].skills`.

```
<available_skills>
  <skill>
    <name>...</name>
    <description>...</description>
    <location>...</location>
  </skill>
</available_skills>
```

Це зберігає базовий промпт малим, водночас усе ще даючи змогу цілеспрямовано використовувати skills.

Бюджет списку Skills належить підсистемі Skills:

- Глобальне типове значення: `skills.limits.maxSkillsPromptChars`
- Перевизначення для окремого агента: `agents.list[].skillsLimits.maxSkillsPromptChars`

Загальні обмежені витяги середовища виконання використовують іншу поверхню:

- `agents.defaults.contextLimits.*`
- `agents.list[].contextLimits.*`

Такий поділ зберігає окреме керування розміром Skills і розміром читання/впровадження під час виконання, як-от
`memory_get`, живі результати інструментів і оновлення AGENTS.md після Compaction.

## Документація

Коли доступно, системний промпт містить розділ **Документація**, який указує на
локальний каталог документації OpenClaw (або `docs/` у робочому просторі репозиторію, або документацію у вбудованому npm
package), а також згадує публічне дзеркало, вихідний репозиторій, спільноту Discord і
ClawHub ([https://clawhub.ai](https://clawhub.ai)) для пошуку Skills. Промпт інструктує модель спочатку звертатися до локальної документації
щодо поведінки OpenClaw, команд, конфігурації чи архітектури, а також
самостійно запускати `openclaw status`, коли це можливо (звертаючись до користувача лише тоді, коли доступу немає).
