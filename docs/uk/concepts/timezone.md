OpenClaw стандартизує часові позначки, щоб модель бачила **єдиний еталонний час**, а не суміш локальних годинників різних провайдерів. Часові пояси відображаються в трьох областях, кожна з яких має власне призначення:

## Три області часових поясів

| Область                   | Що відображається                                                                                                             | Типове значення                                           | Налаштовується через                                     |
| ------------------------- | ----------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------- | ------------------------------------------------------- |
| Конверти повідомлень      | Обгортають вхідні повідомлення каналів: `[Signal +1555 Sun 2026-01-18 00:19:42 PST] hello`                                     | Локальний часовий пояс хоста                              | `agents.defaults.envelopeTimezone`                      |
| Корисні дані інструментів | Канальні інструменти на кшталт `readMessages` повертають необроблений час провайдера та нормалізовані `timestampMs` / `timestampUtc` | Поля UTC наявні завжди                                    | Не налаштовується; зберігає власні часові позначки провайдера |
| Системний промпт          | Невеликий блок `Current Date & Time` лише з **часовим поясом** (без значення часу для стабільності кешу)                       | Часовий пояс хоста, якщо `userTimezone` не встановлено    | `agents.defaults.userTimezone`                          |

У системному промпті навмисно не вказано поточний час, щоб кешування промпту залишалося стабільним між ходами. Коли агенту потрібен поточний час, він викликає `session_status`.

## Налаштування часового поясу користувача

```json5
{
  agents: {
    defaults: {
      userTimezone: "America/Chicago",
    },
  },
}
```

Якщо `userTimezone` не встановлено, OpenClaw під час виконання визначає часовий пояс хоста за допомогою `Intl.DateTimeFormat().resolvedOptions().timeZone` (без запису до конфігурації). `agents.defaults.timeFormat` (`auto` | `12` | `24`) керує відображенням часу у 12- або 24-годинному форматі в конвертах і наступних областях, але не в розділі системного промпту.

## Значення часового поясу конверта

`agents.defaults.envelopeTimezone` приймає:

- `"local"` (типове значення) або `"host"` — часовий пояс хост-машини.
- `"utc"` або `"gmt"` — UTC.
- `"user"` — визначений `agents.defaults.userTimezone` (якщо не встановлено, використовується часовий пояс хоста).
- Будь-який явно вказаний рядок часового поясу IANA, наприклад `"Europe/Vienna"`.

## Коли слід перевизначати

- **Використовуйте `"utc"`** для стабільних часових позначок на хостах у різних регіонах або для узгодження з діагностичними даними чи журналами, орієнтованими на UTC.
- **Використовуйте `"user"`**, щоб часові позначки в конвертах відповідали налаштованому часовому поясу користувача незалежно від часового поясу хоста Gateway.
- **Використовуйте фіксований часовий пояс IANA**, якщо хост Gateway перебуває в одному часовому поясі, але конверт завжди має відображати час в іншому незалежно від міграції хоста.
- **Установіть `envelopeTimestamp: "off"`**, якщо контекст часових позначок не корисний для розмови. Це вилучає абсолютні часові позначки з конвертів, прямих префіксів промпту агента та вбудованих префіксів вхідних даних моделі.
