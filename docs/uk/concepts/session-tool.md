OpenClaw надає агентам інструменти для роботи між сесіями, перевірки статусу та
оркестрації субагентів.

## Доступні інструменти

| Інструмент        | Що він робить                                                              |
| ----------------- | -------------------------------------------------------------------------- |
| `sessions_list`   | Показує список сесій із необов’язковими фільтрами (тип, мітка, агент, нещодавність, попередній перегляд) |
| `sessions_history`| Читає транскрипт конкретної сесії                                           |
| `sessions_send`   | Надсилає повідомлення до іншої сесії та за потреби очікує                  |
| `sessions_spawn`  | Створює ізольовану сесію субагента для фонової роботи                      |
| `sessions_yield`  | Завершує поточний хід і очікує на подальші результати субагента            |
| `subagents`       | Показує список, керує або завершує запущених субагентів для цієї сесії     |
| `session_status`  | Показує картку в стилі `/status` і за потреби встановлює перевизначення моделі для сесії |

## Перегляд списку та читання сесій

`sessions_list` повертає сесії з їхнім ключем, agentId, типом, каналом, моделлю,
лічильниками токенів і часовими мітками. Можна фільтрувати за типом (`main`, `group`, `cron`, `hook`,
`node`), точною `label`, точним `agentId`, пошуковим текстом або нещодавністю
(`activeMinutes`). Якщо вам потрібен розбір у стилі поштової скриньки, інструмент також може
запитати похідний заголовок з урахуванням області видимості, фрагмент попереднього перегляду останнього повідомлення
або обмежені недавні повідомлення в кожному рядку. Похідні заголовки та попередні перегляди створюються лише для
сесій, які викликач уже може бачити згідно з налаштованою політикою видимості
інструментів сесії, тому непов’язані сесії залишаються прихованими.

`sessions_history` отримує транскрипт розмови для конкретної сесії.
Типово результати інструментів виключаються — передайте `includeTools: true`, щоб їх побачити.
Повернутий вигляд навмисно обмежений і відфільтрований для безпеки:

- текст асистента нормалізується перед відновленням:
  - теги thinking видаляються
  - блоки каркаса `<relevant-memories>` / `<relevant_memories>` видаляються
  - блоки XML-навантаження викликів інструментів у звичайному тексті, такі як `<tool_call>...</tool_call>`,
    `<function_call>...</function_call>`, `<tool_calls>...</tool_calls>` і
    `<function_calls>...</function_calls>`, видаляються, включно з обрізаними
    навантаженнями, які так і не закриваються коректно
  - спрощений каркас викликів/результатів інструментів, такий як `[Tool Call: ...]`,
    `[Tool Result ...]` і `[Historical context ...]`, видаляється
  - витоки керівних токенів моделі, такі як `<|assistant|>`, інші ASCII
    токени `<|...|>` і повноширинні варіанти `<｜...｜>`, видаляються
  - некоректний XML викликів інструментів MiniMax, такий як `<invoke ...>` /
    `</minimax:tool_call>`, видаляється
- текст, схожий на облікові дані/токени, редагується перед поверненням
- довгі текстові блоки обрізаються
- у дуже великих історіях старіші рядки можуть випадати або надто великий рядок може бути замінений на
  `[sessions_history omitted: message too large]`
- інструмент повідомляє підсумкові прапорці, такі як `truncated`, `droppedMessages`,
  `contentTruncated`, `contentRedacted` і `bytes`

Обидва інструменти приймають або **ключ сесії** (наприклад, `"main"`), або **ID сесії**
з попереднього виклику list.

Якщо вам потрібен точний транскрипт байт у байт, перевіряйте файл транскрипту на
диску, а не розглядайте `sessions_history` як сирий дамп.

## Надсилання повідомлень між сесіями

`sessions_send` доставляє повідомлення до іншої сесії та за потреби очікує
на відповідь:

- **Надіслати без очікування:** установіть `timeoutSeconds: 0`, щоб поставити в чергу та
  одразу повернутися.
- **Очікувати на відповідь:** установіть тайм-аут і отримайте відповідь inline.

Після відповіді цільової сторони OpenClaw може запустити **цикл зворотної відповіді**,
у якому агенти почергово обмінюються повідомленнями (до 5 ходів). Цільовий агент може відповісти
`REPLY_SKIP`, щоб зупинити це раніше.

## Допоміжні засоби статусу та оркестрації

`session_status` — це полегшений інструмент, еквівалентний `/status`, для поточної
або іншої видимої сесії. Він повідомляє про використання, час, стан моделі/середовища виконання та
пов’язаний контекст фонових завдань, якщо він є. Як і `/status`, він може дозаповнювати
розріджені лічильники токенів/кешу з останнього запису використання в транскрипті, а
`model=default` очищає перевизначення для окремої сесії. Використовуйте `sessionKey="current"` для
поточної сесії викликача; видимі мітки клієнта, такі як `openclaw-tui`, не є ключами сесії.

`sessions_yield` навмисно завершує поточний хід, щоб наступним повідомленням могла бути
подія продовження, на яку ви очікуєте. Використовуйте його після запуску субагентів, коли
хочете, щоб результати завершення надійшли наступним повідомленням, замість побудови циклів опитування.

`subagents` — це допоміжний інструмент площини керування для вже запущених
субагентів OpenClaw. Він підтримує:

- `action: "list"` для перегляду активних/недавніх запусків
- `action: "steer"` для надсилання додаткових вказівок запущеному дочірньому агенту
- `action: "kill"` для зупинки одного дочірнього агента або `all`

## Запуск субагентів

`sessions_spawn` типово створює ізольовану сесію для фонового завдання.
Він завжди не блокує виконання — одразу повертає `runId` і
`childSessionKey`.

Основні параметри:

- `runtime: "subagent"` (типово) або `"acp"` для зовнішніх harness-агентів.
- Перевизначення `model` і `thinking` для дочірньої сесії.
- `thread: true`, щоб прив’язати запуск до гілки чату (Discord, Slack тощо).
- `sandbox: "require"`, щоб примусово застосувати sandbox до дочірнього агента.
- `context: "fork"` для нативних субагентів, коли дочірньому агенту потрібен транскрипт
  поточного запитувача; не вказуйте це або використовуйте `context: "isolated"` для чистого дочірнього агента.

Типові листові субагенти не отримують інструменти сесії. Коли
`maxSpawnDepth >= 2`, субагенти-оркестратори рівня глибини 1 додатково отримують
`sessions_spawn`, `subagents`, `sessions_list` і `sessions_history`, щоб вони
могли керувати власними дочірніми агентами. Листові запуски все одно не отримують рекурсивних
інструментів оркестрації.

Після завершення крок анонсування надсилає результат у канал запитувача.
Доставка після завершення зберігає прив’язану маршрутизацію гілки/теми, коли це можливо, а якщо
джерело завершення вказує лише канал, OpenClaw усе одно може повторно використати збережений маршрут
сесії запитувача (`lastChannel` / `lastTo`) для прямої
доставки.

Специфічну поведінку ACP дивіться в [ACP Agents](/uk/tools/acp-agents).

## Видимість

Інструменти сесії мають область видимості, щоб обмежити те, що агент може бачити:

| Рівень  | Область видимості                        |
| ------- | ---------------------------------------- |
| `self`  | Лише поточна сесія                       |
| `tree`  | Поточна сесія + запущені субагенти       |
| `agent` | Усі сесії для цього агента               |
| `all`   | Усі сесії (між агентами, якщо налаштовано) |

Типове значення — `tree`. Сесії в sandbox примусово обмежуються до `tree` незалежно від
конфігурації.
