[Honcho](https://honcho.dev) додає до OpenClaw AI-native пам’ять. Він зберігає
розмови в окремому сервісі та з часом будує моделі користувача й агента,
надаючи вашому агенту міжсесійний контекст, який виходить за межі Markdown-файлів
робочого простору.

## Що він надає

- **Міжсесійну пам’ять** -- розмови зберігаються після кожного ходу, тож
  контекст переноситься між скиданнями сесій, Compaction і перемиканням каналів.
- **Моделювання користувача** -- Honcho підтримує профіль для кожного користувача (уподобання,
  факти, стиль спілкування) і для агента (особистість, засвоєні
  моделі поведінки).
- **Семантичний пошук** -- пошук за спостереженнями з минулих розмов, а не
  лише в межах поточної сесії.
- **Обізнаність про кількох агентів** -- батьківські агенти автоматично відстежують запущених
  підлеглих агентів, а батьків додають як спостерігачів у дочірніх сесіях.

## Доступні інструменти

Honcho реєструє інструменти, які агент може використовувати під час розмови:

**Отримання даних (швидко, без виклику LLM):**

| Tool                        | What it does                                             |
| --------------------------- | -------------------------------------------------------- |
| `honcho_context`            | Повне представлення користувача в різних сесіях          |
| `honcho_search_conclusions` | Семантичний пошук по збережених висновках                |
| `honcho_search_messages`    | Пошук повідомлень у різних сесіях (фільтр за відправником, датою) |
| `honcho_session`            | Історія та підсумок поточної сесії                       |

**Запитання й відповіді (на базі LLM):**

| Tool         | What it does                                                                |
| ------------ | --------------------------------------------------------------------------- |
| `honcho_ask` | Поставити запитання про користувача. `depth='quick'` для фактів, `'thorough'` для синтезу |

## Початок роботи

Установіть Plugin і запустіть налаштування:

```bash
openclaw plugins install @honcho-ai/openclaw-honcho
openclaw honcho setup
openclaw gateway --force
```

Команда налаштування запитає ваші облікові дані API, запише конфігурацію і,
за потреби, запропонує перенести наявні файли пам’яті робочого простору.

<Info>
Honcho може працювати повністю локально (self-hosted) або через керований API за адресою
`api.honcho.dev`. Для варіанта self-hosted зовнішні залежності не потрібні.
</Info>

## Конфігурація

Параметри розміщуються в `plugins.entries["openclaw-honcho"].config`:

```json5
{
  plugins: {
    entries: {
      "openclaw-honcho": {
        config: {
          apiKey: "your-api-key", // пропустіть для self-hosted
          workspaceId: "openclaw", // ізоляція пам’яті
          baseUrl: "https://api.honcho.dev",
        },
      },
    },
  },
}
```

Для self-hosted екземплярів укажіть у `baseUrl` адресу локального сервера (наприклад,
`http://localhost:8000`) і не задавайте API key.

## Міграція наявної пам’яті

Якщо у вас уже є файли пам’яті робочого простору (`USER.md`, `MEMORY.md`,
`IDENTITY.md`, `memory/`, `canvas/`), `openclaw honcho setup` виявить їх і
запропонує виконати міграцію.

<Info>
Міграція є неруйнівною -- файли вивантажуються в Honcho. Оригінали
ніколи не видаляються й не переміщуються.
</Info>

## Як це працює

Після кожного AI-ходу розмова зберігається в Honcho. Спостерігаються і повідомлення користувача, і
повідомлення агента, що дає Honcho змогу з часом будувати й уточнювати свої моделі.

Під час розмови інструменти Honcho звертаються до сервісу у фазі `before_prompt_build`,
впроваджуючи релевантний контекст до того, як модель побачить prompt. Це забезпечує
точні межі ходів і релевантне відновлення контексту.

## Honcho проти вбудованої пам’яті

|                   | Builtin / QMD                  | Honcho                              |
| ----------------- | ------------------------------ | ----------------------------------- |
| **Storage**       | Markdown-файли робочого простору | Окремий сервіс (локальний або розміщений) |
| **Cross-session** | Через файли пам’яті            | Автоматично, вбудовано              |
| **User modeling** | Вручну (запис у MEMORY.md)     | Автоматичні профілі                 |
| **Search**        | Векторний + за ключовими словами (гібридний) | Семантичний за спостереженнями      |
| **Multi-agent**   | Не відстежується               | Обізнаність про батьківські/дочірні зв’язки |
| **Dependencies**  | Немає (builtin) або бінарник QMD | Установлення Plugin                 |

Honcho і вбудована система пам’яті можуть працювати разом. Коли налаштовано QMD,
стають доступними додаткові інструменти для пошуку в локальних Markdown-файлах поряд із
міжсесійною пам’яттю Honcho.

## Команди CLI

```bash
openclaw honcho setup                        # Налаштувати API key і перенести файли
openclaw honcho status                       # Перевірити стан з’єднання
openclaw honcho ask <question>               # Запитати Honcho про користувача
openclaw honcho search <query> [-k N] [-d D] # Семантичний пошук у пам’яті
```
