OpenClaw запускає **єдине вбудоване середовище виконання агента** — один процес агента на
Gateway, із власною робочою областю, файлами початкового завантаження та сховищем сесій. На цій сторінці
описано контракт цього середовища виконання: що має містити робоча область, які файли
інжектуються і як сесії виконують початкове завантаження відносно неї.

## Робоча область (обов’язково)

OpenClaw використовує єдиний каталог робочої області агента (`agents.defaults.workspace`) як **єдиний** робочий каталог агента (`cwd`) для інструментів і контексту.

Рекомендовано: використовуйте `openclaw setup`, щоб створити `~/.openclaw/openclaw.json`, якщо його немає, та ініціалізувати файли робочої області.

Повний макет робочої області + посібник із резервного копіювання: [Робоча область агента](/uk/concepts/agent-workspace)

Якщо `agents.defaults.sandbox` увімкнено, неосновні сесії можуть перевизначити це
робочими областями на рівні сесії в `agents.defaults.sandbox.workspaceRoot` (див.
[Конфігурація Gateway](/uk/gateway/configuration)).

## Файли початкового завантаження (інжектуються)

Усередині `agents.defaults.workspace` OpenClaw очікує ці редаговані користувачем файли:

- `AGENTS.md` — робочі інструкції + “пам’ять”
- `SOUL.md` — персона, межі, тон
- `TOOLS.md` — нотатки про інструменти, які підтримує користувач (наприклад, `imsg`, `sag`, домовленості)
- `BOOTSTRAP.md` — одноразовий ритуал першого запуску (видаляється після завершення)
- `IDENTITY.md` — ім’я/настрій/емодзі агента
- `USER.md` — профіль користувача + бажана форма звертання

На першому ході нової сесії OpenClaw інжектує вміст цих файлів безпосередньо в контекст агента.

Порожні файли пропускаються. Великі файли обрізаються й усікаються з маркером, щоб підказки залишалися компактними (прочитайте файл, щоб побачити повний вміст).

Якщо файл відсутній, OpenClaw інжектує один рядок-маркер “відсутній файл” (а `openclaw setup` створить безпечний стандартний шаблон).

`BOOTSTRAP.md` створюється лише для **цілком нової робочої області** (коли немає інших файлів початкового завантаження). Якщо ви видалите його після завершення ритуалу, він не повинен створюватися повторно під час наступних перезапусків.

Щоб повністю вимкнути створення файлів початкового завантаження (для попередньо наповнених робочих областей), задайте:

```json5
{ agents: { defaults: { skipBootstrap: true } } }
```

## Вбудовані інструменти

Основні інструменти (read/exec/edit/write і пов’язані системні інструменти) доступні завжди,
з урахуванням політики інструментів. `apply_patch` є необов’язковим і керується
`tools.exec.applyPatch`. `TOOLS.md` **не** контролює, які інструменти існують; це
інструкції щодо того, як _ви_ хочете їх використовувати.

## Skills

OpenClaw завантажує Skills із цих розташувань (від найвищого пріоритету):

- Робоча область: `<workspace>/skills`
- Skills агента проєкту: `<workspace>/.agents/skills`
- Особисті Skills агента: `~/.agents/skills`
- Керовані/локальні: `~/.openclaw/skills`
- Вбудовані (постачаються з інсталяцією)
- Додаткові каталоги Skills: `skills.load.extraDirs`

Skills можуть керуватися конфігурацією/env (див. `skills` у [Конфігурація Gateway](/uk/gateway/configuration)).

## Межі середовища виконання

Вбудоване середовище виконання агента побудоване на ядрі агента Pi (моделі, інструменти та
конвеєр підказок). Керування сесіями, виявлення, підключення інструментів і доставка
в канали — це шари, якими володіє OpenClaw поверх цього ядра.

## Сесії

Транскрипти сесій зберігаються як JSONL за адресою:

- `~/.openclaw/agents/<agentId>/sessions/<SessionId>.jsonl`

ID сесії стабільний і вибирається OpenClaw.
Застарілі каталоги сесій з інших інструментів не читаються.

## Скерування під час потокової передачі

Коли режим черги — `steer`, вхідні повідомлення інжектуються в поточний запуск.
Скерування в черзі доставляється **після того, як поточний хід асистента завершить
виконання своїх викликів інструментів**, перед наступним викликом LLM. Pi разом обробляє всі очікувані
повідомлення скерування для `steer`; застарілий `queue` обробляє одне повідомлення на
межу моделі. Скерування більше не пропускає решту викликів інструментів із поточного
повідомлення асистента.

Коли режим черги — `followup` або `collect`, вхідні повідомлення утримуються до
завершення поточного ходу, а потім новий хід агента починається з поставленими в чергу корисними навантаженнями. Див.
[Черга](/uk/concepts/queue) і [Черга скерування](/uk/concepts/queue-steering), щоб дізнатися про режими
та поведінку на межах.

Блокова потокова передача надсилає завершені блоки асистента щойно вони завершуються; вона
**вимкнена за замовчуванням** (`agents.defaults.blockStreamingDefault: "off"`).
Налаштуйте межу через `agents.defaults.blockStreamingBreak` (`text_end` проти `message_end`; за замовчуванням text_end).
Керуйте м’яким розбиттям блоків на фрагменти за допомогою `agents.defaults.blockStreamingChunk` (за замовчуванням
800–1200 символів; віддає перевагу розривам абзаців, потім новим рядкам; речення — останні).
Об’єднуйте потокові фрагменти через `agents.defaults.blockStreamingCoalesce`, щоб зменшити
спам окремими рядками (об’єднання перед надсиланням на основі простою). Канали, відмінні від Telegram, потребують
явного `*.blockStreaming: true`, щоб увімкнути блокові відповіді.
Докладні підсумки інструментів виводяться під час запуску інструмента (без debounce); Control UI
транслює вивід інструментів через події агента, коли вони доступні.
Докладніше: [Потокова передача + розбиття на фрагменти](/uk/concepts/streaming).

## Посилання на моделі

Посилання на моделі в конфігурації (наприклад, `agents.defaults.model` і `agents.defaults.models`) розбираються поділом за **першим** `/`.

- Використовуйте `provider/model` під час налаштування моделей.
- Якщо сам ID моделі містить `/` (у стилі OpenRouter), додайте префікс провайдера (приклад: `openrouter/moonshotai/kimi-k2`).
- Якщо ви опустите провайдера, OpenClaw спершу спробує псевдонім, потім унікальний
  збіг серед налаштованих провайдерів для саме цього ID моделі, і лише після цього повернеться
  до налаштованого стандартного провайдера. Якщо цей провайдер більше не надає
  налаштовану стандартну модель, OpenClaw повернеться до першого налаштованого
  провайдера/моделі замість показу застарілого стандартного значення видаленого провайдера.
