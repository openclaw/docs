# Індикатори набору тексту

Індикатори набору тексту надсилаються до каналу чату, поки виконання активне. Використовуйте
`agents.defaults.typingMode`, щоб керувати, **коли** починається набір, а `typingIntervalSeconds`,
щоб керувати, **як часто** він оновлюється.

## Стандартні значення

Коли `agents.defaults.typingMode` **не задано**, OpenClaw зберігає попередню поведінку:

- **Прямі чати**: набір починається одразу, щойно починається цикл моделі.
- **Групові чати зі згадкою**: набір починається одразу.
- **Групові чати без згадки**: набір починається лише тоді, коли починає транслюватися текст повідомлення.
- **Запуски Heartbeat**: набір починається, коли починається запуск heartbeat, якщо
  визначена ціль Heartbeat — це чат із підтримкою набору тексту і набір не вимкнено.

## Режими

Установіть `agents.defaults.typingMode` в одне з таких значень:

- `never` — індикатор набору тексту ніколи не показується.
- `instant` — почати показувати набір **щойно починається цикл моделі**, навіть якщо виконання
  пізніше поверне лише маркер тихої відповіді.
- `thinking` — почати показувати набір на **першій дельті міркування** (потрібен
  `reasoningLevel: "stream"` для цього виконання).
- `message` — почати показувати набір на **першій нетихій текстовій дельті** (ігнорує
  тихий маркер `NO_REPLY`).

Порядок «наскільки рано спрацьовує»:
`never` → `message` → `thinking` → `instant`

## Конфігурація

```json5
{
  agent: {
    typingMode: "thinking",
    typingIntervalSeconds: 6,
  },
}
```

Ви можете перевизначити режим або частоту для окремої сесії:

```json5
{
  session: {
    typingMode: "message",
    typingIntervalSeconds: 4,
  },
}
```

## Примітки

- Режим `message` не показуватиме набір для відповідей, що містять лише тихий маркер, коли весь
  вміст точно дорівнює тихому маркеру (наприклад, `NO_REPLY` / `no_reply`,
  збіг без урахування регістру).
- `thinking` спрацьовує лише якщо виконання транслює міркування (`reasoningLevel: "stream"`).
  Якщо модель не надсилає дельти міркування, набір не почнеться.
- Набір у Heartbeat є сигналом живучості для визначеної цілі доставки. Він
  починається на старті виконання heartbeat замість того, щоб слідувати таймінгу потоків `message` або `thinking`. Установіть `typingMode: "never"`, щоб вимкнути його.
- Heartbeat не показує набір, коли `target: "none"`, коли ціль не вдається
  визначити, коли доставку в чат для heartbeat вимкнено, або коли канал не підтримує набір.
- `typingIntervalSeconds` керує **частотою оновлення**, а не часом початку.
  Стандартне значення — 6 секунд.
