Чернетки прогресу роблять тривалі кроки агента живими в чаті, не перетворюючи
розмову на стос тимчасових відповідей зі статусом.

Коли чернетки прогресу ввімкнено, OpenClaw створює одне видиме повідомлення
про роботу в процесі лише після того, як крок доводить, що виконує реальну
роботу, оновлює його, поки агент читає, планує, викликає інструменти або чекає
на схвалення, а потім перетворює цю чернетку на фінальну відповідь, коли канал
може зробити це безпечно.

```text
Shelling...
📖 Read: from docs/concepts/progress-drafts.md
🔎 Web Search: for "discord edit message"
🛠️ Exec: run tests
```

Використовуйте чернетки прогресу, коли потрібне одне охайне статусне
повідомлення під час роботи з великою кількістю інструментів і фінальна
відповідь після завершення кроку.

## Швидкий старт

Увімкніть чернетки прогресу для кожного каналу через `streaming.mode: "progress"`:

```json5
{
  channels: {
    discord: {
      streaming: {
        mode: "progress",
      },
    },
  },
}
```

Зазвичай цього достатньо. OpenClaw вибере автоматичну мітку з одного слова,
зачекає, доки робота триватиме щонайменше п’ять секунд або створить другу
робочу подію, додаватиме компактні рядки прогресу під час корисної роботи та
пригнічуватиме дублікати окремого службового прогресу для цього кроку.

## Що бачать користувачі

Чернетка прогресу має дві частини:

| Частина         | Призначення                                                                 |
| --------------- | --------------------------------------------------------------------------- |
| Мітка           | Короткий заголовок, наприклад `Thinking...` або `Shelling...`.              |
| Рядки прогресу  | Компактні оновлення запуску з тими самими мітками й іконками інструментів, що й у докладному виводі. |

Мітка з’являється після того, як агент починає змістовну роботу і або
залишається зайнятим п’ять секунд, або створює другу робочу подію. Відповіді
лише звичайним текстом не показують чернетку прогресу. Рядки прогресу
додаються лише тоді, коли агент створює корисні оновлення роботи, наприклад
`🛠️ Exec`, `🔎 Web Search` або `✍️ Write: to /tmp/file`.
За замовчуванням вони використовують той самий компактний режим пояснення, що й
`/verbose`; установіть `agents.defaults.toolProgressDetail: "raw"` під час
налагодження, якщо також потрібне додавання сирих команд або деталей.
Фінальна відповідь замінює чернетку, коли це можливо; інакше OpenClaw надсилає
фінальну відповідь звичайним способом і очищає або припиняє оновлювати чернетку
відповідно до транспорту каналу.

## Вибір режиму

`channels.<channel>.streaming.mode` керує видимою поведінкою роботи в процесі:

| Режим      | Найкраще для                    | Що з’являється в чаті                            |
| ---------- | -------------------------------- | ------------------------------------------------- |
| `off`      | Тихі канали                      | Лише фінальна відповідь.                          |
| `partial`  | Спостереження за появою тексту відповіді | Одна чернетка, що редагується найновішим текстом відповіді. |
| `block`    | Більші фрагменти попереднього перегляду відповіді | Один попередній перегляд, що оновлюється або доповнюється більшими фрагментами. |
| `progress` | Кроки з великою кількістю інструментів або тривалі кроки | Одна статусна чернетка, потім фінальна відповідь. |

Вибирайте `progress`, коли користувачам важливіше бачити, «що відбувається», ніж
спостерігати за потоковою передачею тексту відповіді токен за токеном.

Вибирайте `partial`, коли сама відповідь є сигналом прогресу.

Вибирайте `block`, коли потрібні оновлення чернетки попереднього перегляду
більшими текстовими фрагментами. У Discord і Telegram `streaming.mode: "block"`
досі є потоковим попереднім переглядом, а не звичайною блочною доставкою.
Використовуйте `streaming.block.enabled` або застарілий `blockStreaming`, коли
потрібні звичайні блочні відповіді.

## Налаштування міток

Мітки прогресу розміщено в `channels.<channel>.streaming.progress`.

Типова мітка — `auto`, що вибирає з вбудованого в OpenClaw набору міток
«одне слово з трьома крапками»:

```text
Thinking...
Shelling...
Scuttling...
Clawing...
Pinching...
Molting...
Bubbling...
Tiding...
Reefing...
Cracking...
Sifting...
Brining...
Nautiling...
Krilling...
Barnacling...
Lobstering...
Tidepooling...
Pearling...
Snapping...
Surfacing...
```

Використайте фіксовану мітку:

```json5
{
  channels: {
    discord: {
      streaming: {
        mode: "progress",
        progress: {
          label: "Investigating",
        },
      },
    },
  },
}
```

Використайте власний автоматичний набір міток:

```json5
{
  channels: {
    discord: {
      streaming: {
        mode: "progress",
        progress: {
          label: "auto",
          labels: ["Checking", "Reading", "Testing", "Finishing"],
        },
      },
    },
  },
}
```

Сховайте мітку й показуйте лише рядки прогресу:

```json5
{
  channels: {
    discord: {
      streaming: {
        mode: "progress",
        progress: {
          label: false,
        },
      },
    },
  },
}
```

## Керування рядками прогресу

Рядки прогресу ввімкнено за замовчуванням у режимі прогресу. Вони надходять із
реальних подій запуску: стартів інструментів, оновлень елементів, планів
завдань, схвалень, виводу команд, підсумків патчів і подібної активності агента.

OpenClaw використовує той самий форматувальник для чернеток прогресу й
`/verbose`:

```json5
{
  agents: {
    defaults: {
      toolProgressDetail: "explain", // explain | raw
    },
  },
}
```

`"explain"` є типовим значенням і зберігає чернетки стабільними завдяки стислим
міткам на кшталт `🛠️ Exec: check JS syntax for /tmp/app.js`. `"raw"` додає
базову команду або деталь, коли вона доступна; це корисно під час налагодження,
але створює більше шуму в чаті.

Наприклад, та сама команда виглядає по-різному залежно від режиму деталізації:

| Режим     | Рядок прогресу                                                       |
| --------- | -------------------------------------------------------------------- |
| `explain` | `🛠️ Exec: check JS syntax for /tmp/app.js`                           |
| `raw`     | `🛠️ Exec: check JS syntax for /tmp/app.js, node --check /tmp/app.js` |

Обмежте кількість рядків, що залишаються видимими:

```json5
{
  channels: {
    discord: {
      streaming: {
        mode: "progress",
        progress: {
          maxLines: 4,
        },
      },
    },
  },
}
```

Рядки прогресу автоматично ущільнюються, щоб зменшити переформатування бульбашки чату під час редагування чернетки.

OpenClaw за замовчуванням обрізає довгі рядки прогресу, щоб повторні
редагування чернетки не переносилися по-різному під час кожного оновлення.
Префікс залишається читабельним, а довгі деталі, як-от шляхи або сирі команди,
скорочуються з трьома крапками.

Slack може відтворювати рядки прогресу як структуровані поля Block Kit замість
одного текстового тіла:

```json5
{
  channels: {
    slack: {
      streaming: {
        mode: "progress",
        progress: {
          render: "rich",
        },
      },
    },
  },
}
```

Розширене відтворення зберігає той самий резервний варіант простим текстом, щоб
канали й клієнти, які не підтримують багатшу форму, усе одно могли показувати
компактний текст прогресу.

Збережіть одну чернетку прогресу, але сховайте рядки інструментів і завдань:

```json5
{
  channels: {
    discord: {
      streaming: {
        mode: "progress",
        progress: {
          toolProgress: false,
        },
      },
    },
  },
}
```

З `toolProgress: false` OpenClaw усе одно пригнічує старі окремі повідомлення
про прогрес інструментів для цього кроку. Канал залишається візуально тихим до
фінальної відповіді, за винятком мітки, якщо її налаштовано.

## Поведінка каналів

Кожен канал використовує найчистіший транспорт, який підтримує:

| Канал           | Транспорт прогресу                    | Примітки                                                              |
| --------------- | -------------------------------------- | --------------------------------------------------------------------- |
| Discord         | Надсилає одне повідомлення, потім редагує його. | Фінальний текст редагується на місці, коли вміщується в одне безпечне повідомлення попереднього перегляду. |
| Matrix          | Надсилає одну подію, потім редагує її. | Конфігурація потокової передачі на рівні облікового запису керує чернетками на рівні облікового запису. |
| Microsoft Teams | Нативний потік Teams у приватних чатах. | `streaming.mode: "block"` відповідає блочній доставці Teams.          |
| Slack           | Нативний потік або редагований допис-чернетка. | Доступність гілки впливає на те, чи можна використовувати нативну потокову передачу. |
| Telegram        | Надсилає одне повідомлення, потім редагує його. | Старі видимі чернетки можуть бути замінені, щоб фінальні часові мітки залишалися корисними. |
| Mattermost      | Редагований допис-чернетка.            | Активність інструментів згортається в той самий допис у стилі чернетки. |

Канали без безпечної підтримки редагування зазвичай повертаються до індикаторів
набору тексту або доставки лише фінальної відповіді.

## Завершення

Коли фінальна відповідь готова, OpenClaw намагається зберегти чат чистим:

- Якщо чернетка може безпечно стати фінальною відповіддю, OpenClaw редагує її на місці.
- Якщо канал використовує нативну потокову передачу прогресу, OpenClaw
  завершує цей потік, коли нативний транспорт приймає фінальний текст.
- Якщо фінальна відповідь має медіа, запит на схвалення, явну ціль відповіді,
  забагато фрагментів або невдале редагування/надсилання, OpenClaw надсилає
  фінальну відповідь через звичайний шлях доставки каналу.

Резервний шлях є навмисним. Краще надіслати нову фінальну відповідь, ніж
втратити текст, помилково прив’язати відповідь до гілки або перезаписати
чернетку вмістом, який канал не може безпечно представити.

## Усунення несправностей

**Я бачу лише фінальну відповідь.**

Перевірте, що `channels.<channel>.streaming.mode` встановлено на `progress` для
облікового запису або каналу, який обробив повідомлення. Деякі шляхи груп або
відповідей із цитуванням можуть вимикати попередні перегляди чернеток для кроку,
коли канал не може безпечно редагувати потрібне повідомлення.

**Я бачу мітку, але не бачу рядків інструментів.**

Перевірте `streaming.progress.toolProgress`. Якщо значення `false`, OpenClaw
зберігає поведінку однієї чернетки, але ховає рядки прогресу інструментів і
завдань.

**Я бачу нове фінальне повідомлення замість відредагованої чернетки.**

Це безпечний резервний варіант. Таке може статися для відповідей із медіа,
довгих відповідей, явних цілей відповіді, старих чернеток Telegram, відсутніх
цілей гілок Slack, видалених повідомлень попереднього перегляду або невдалого
завершення нативного потоку.

**Я все ще бачу окремі повідомлення про прогрес.**

Режим прогресу пригнічує типові окремі повідомлення про прогрес інструментів,
коли чернетка активна. Якщо окремі повідомлення все ще з’являються, перевірте,
що крок справді використовує режим прогресу, а не `streaming.mode: "off"` або
шлях каналу, який не може створити чернетку для цього повідомлення.

**Teams поводиться інакше, ніж Discord або Telegram.**

Microsoft Teams використовує нативний потік у приватних чатах замість
універсального транспорту попереднього перегляду «надіслати й редагувати».
Teams також трактує `streaming.mode: "block"` як блочну доставку Teams, тому що
в ньому немає такого самого блочного режиму попереднього перегляду чернетки, який
використовують Discord і Telegram.
