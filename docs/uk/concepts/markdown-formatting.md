OpenClaw форматує вихідний Markdown, перетворюючи його на спільне проміжне
представлення (IR) перед рендерингом вихідного формату для конкретного каналу. IR зберігає
початковий текст без змін, водночас містячи діапазони стилів/посилань, щоб розбиття на фрагменти та рендеринг
залишалися узгодженими між каналами.

## Цілі

- **Узгодженість:** один крок парсингу, кілька renderer.
- **Безпечне розбиття на фрагменти:** розбивати текст до рендерингу, щоб вбудоване форматування ніколи
  не ламалося між фрагментами.
- **Відповідність каналу:** зіставляти той самий IR із Slack mrkdwn, Telegram HTML і діапазонами стилів
  Signal без повторного парсингу Markdown.

## Конвеєр

1. **Parse Markdown -> IR**
   - IR — це звичайний текст плюс діапазони стилів (bold/italic/strike/code/spoiler) і діапазони посилань.
   - Зміщення задаються в кодових одиницях UTF-16, щоб діапазони стилів Signal узгоджувалися з його API.
   - Таблиці парсяться лише тоді, коли канал явно погоджується на перетворення таблиць.
2. **Chunk IR (спочатку форматування)**
   - Розбиття на фрагменти відбувається за текстом IR до рендерингу.
   - Вбудоване форматування не розривається між фрагментами; діапазони обрізаються для кожного фрагмента.
3. **Render для кожного каналу**
   - **Slack:** токени mrkdwn (bold/italic/strike/code), посилання як `<url|label>`.
   - **Telegram:** HTML-теги (`<b>`, `<i>`, `<s>`, `<code>`, `<pre><code>`, `<a href>`).
   - **Signal:** звичайний текст + діапазони `text-style`; посилання стають `label (url)`, коли label відрізняється.

## Приклад IR

Вхідний Markdown:

```markdown
Hello **world** — see [docs](https://docs.openclaw.ai).
```

IR (схематично):

```json
{
  "text": "Hello world — see docs.",
  "styles": [{ "start": 6, "end": 11, "style": "bold" }],
  "links": [{ "start": 19, "end": 23, "href": "https://docs.openclaw.ai" }]
}
```

## Де це використовується

- Вихідні адаптери Slack, Telegram і Signal рендерять з IR.
- Інші канали (WhatsApp, iMessage, Microsoft Teams, Discord) і далі використовують звичайний текст або
  власні правила форматування, при цьому перетворення таблиць Markdown застосовується до
  розбиття на фрагменти, коли це ввімкнено.

## Обробка таблиць

Таблиці Markdown підтримуються в чат-клієнтах непослідовно. Використовуйте
`markdown.tables`, щоб керувати перетворенням для кожного каналу (і для кожного облікового запису).

- `code`: рендерити таблиці як блоки коду (типово для більшості каналів).
- `bullets`: перетворювати кожен рядок на марковані пункти (типово для Signal + WhatsApp).
- `off`: вимкнути парсинг і перетворення таблиць; сирий текст таблиці проходить без змін.

Ключі конфігурації:

```yaml
channels:
  discord:
    markdown:
      tables: code
    accounts:
      work:
        markdown:
          tables: off
```

## Правила розбиття на фрагменти

- Обмеження фрагментів надходять з адаптерів/конфігурації каналів і застосовуються до тексту IR.
- Огороджені блоки коду зберігаються як єдиний блок із завершальним символом нового рядка, щоб канали
  рендерили їх правильно.
- Префікси списків і blockquote є частиною тексту IR, тому розбиття на фрагменти
  не розриває їх посеред префікса.
- Вбудовані стилі (bold/italic/strike/inline-code/spoiler) ніколи не розриваються між
  фрагментами; renderer повторно відкриває стилі всередині кожного фрагмента.

Якщо вам потрібно більше деталей про поведінку розбиття на фрагменти в різних каналах, див.
[Streaming + chunking](/uk/concepts/streaming).

## Політика посилань

- **Slack:** `[label](url)` -> `<url|label>`; звичайні URL залишаються як є. Autolink
  вимикається під час парсингу, щоб уникнути подвійного створення посилань.
- **Telegram:** `[label](url)` -> `<a href="url">label</a>` (режим парсингу HTML).
- **Signal:** `[label](url)` -> `label (url)`, якщо label не збігається з URL.

## Spoilers

Маркери spoiler (`||spoiler||`) парсяться лише для Signal, де вони зіставляються з
діапазонами стилю SPOILER. Інші канали сприймають їх як звичайний текст.

## Як додати або оновити formatter каналу

1. **Парсити один раз:** використовуйте спільний допоміжний засіб `markdownToIR(...)` з параметрами,
   відповідними для каналу (autolink, стиль heading, префікс blockquote).
2. **Рендерити:** реалізуйте renderer за допомогою `renderMarkdownWithMarkers(...)` і
   зіставлення маркерів стилю (або діапазонів стилю Signal).
3. **Розбити на фрагменти:** викличте `chunkMarkdownIR(...)` до рендерингу; рендерте кожен фрагмент.
4. **Підключити адаптер:** оновіть вихідний адаптер каналу, щоб він використовував новий chunker
   і renderer.
5. **Протестувати:** додайте або оновіть тести форматування й тест вихідної доставки, якщо
   канал використовує розбиття на фрагменти.

## Типові підводні камені

- Токени Slack у кутових дужках (`<@U123>`, `<#C123>`, `<https://...>`) мають
  зберігатися; безпечно екрануйте сирий HTML.
- Telegram HTML вимагає екранування тексту поза тегами, щоб уникнути зламаної розмітки.
- Діапазони стилів Signal залежать від зміщень UTF-16; не використовуйте зміщення кодових точок.
- Зберігайте завершальні символи нового рядка для огороджених блоків коду, щоб маркери закриття опинялися
  на окремому рядку.
