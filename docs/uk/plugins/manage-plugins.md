Control UI охоплює типовий робочий процес пошуку, встановлення, увімкнення та вимкнення. CLI також надає засоби оновлення, видалення, розширеного налаштування та явного вибору джерела встановлення. Повний опис командного контракту, прапорців, правил вибору джерела та крайових випадків див. у розділі [`openclaw plugins`](/uk/cli/plugins).

Типовий робочий процес у CLI: знайти пакет, установити його з ClawHub, npm, git або локального шляху, дозволити керованому Gateway автоматично перезапуститися (або перезапустити його вручну), а потім перевірити реєстрації плагіна в середовищі виконання.

## Використання Control UI

Відкрийте **Плагіни** в Control UI або перейдіть до `/settings/plugins` відносно налаштованого базового шляху Control UI. Наприклад, для базового шляху `/openclaw` використовується `/openclaw/settings/plugins`. Сторінка має дві вкладки:

- **Установлені** показує повний локальний перелік, згрупований за категоріями (канали, постачальники моделей, пам’ять, інструменти). Кожен рядок відкриває докладне подання; його меню додаткових дій (`…`) дає змогу ввімкнути або вимкнути плагін, а для плагінів, установлених із зовнішніх джерел, також пропонує дію **Видалити**. На вкладці також перелічено налаштовані [сервери MCP](/uk/cli/mcp) з такими самими доступними через меню діями ввімкнення, вимкнення та видалення, які змінюють `mcp.servers` у конфігурації Gateway.
- **Огляд** — це каталог: рекомендовані плагіни, що входять до OpenClaw, офіційні зовнішні плагіни та добірка конекторів. Картки конекторів або додають розміщений сервер MCP одним натисканням (GitHub, Notion, Linear, Sentry, Home Assistant), або відкривають попередньо заповнений пошук ClawHub. Введення тексту в поле пошуку надсилає запит до [ClawHub](https://clawhub.ai/plugins) безпосередньо на сторінці та додає розділ **Із ClawHub** із кількістю завантажень і позначками перевірки джерела.

Плагіни, що входять до комплекту, не потребують установлення пакета. У їхньому меню доступна дія **Увімкнути** або **Вимкнути**. Наприклад, Workboard входить до OpenClaw і за замовчуванням вимкнений, тому виберіть **Увімкнути**, щоб активувати його. Вбудовані плагіни не можна видалити — лише вимкнути.

Для доступу до каталогу та пошуку потрібен дозвіл `operator.read`. Для встановлення, увімкнення, вимкнення, видалення та зміни серверів MCP потрібен дозвіл `operator.admin`. Установлення з ClawHub виконує Gateway зі збереженням перевірок політик довіри, цілісності та встановлення плагінів.

Установлення або видалення коду плагіна потребує перезапуску Gateway. Зміни стану ввімкнення можна застосувати без перезапуску, якщо це підтримують установлений плагін і поточне середовище виконання Gateway; інакше інтерфейс повідомляє про необхідність перезапуску. Для конекторів MCP на основі OAuth після їх додавання все одно потрібно один раз виконати в CLI команду `openclaw mcp login <name>`.

Control UI не встановлює плагіни з довільних джерел npm, git або локальних шляхів, не оновлює їх і не надає розширеного налаштування плагінів. Для цих операцій використовуйте наведені нижче робочі процеси CLI.

## Перегляд і пошук плагінів

```bash
openclaw plugins list
openclaw plugins list --enabled
openclaw plugins list --verbose
openclaw plugins list --json
openclaw plugins search "calendar"
```

`--json` для скриптів:

```bash
openclaw plugins list --json \
  | jq '.plugins[] | {id, enabled, format, source, dependencyStatus}'
```

`plugins list` виконує холодну перевірку переліку: що OpenClaw може виявити за конфігурацією, маніфестами та збереженим реєстром плагінів. Вона не підтверджує, що вже запущений Gateway імпортував середовище виконання плагіна. Виведення JSON містить діагностику реєстру та `dependencyStatus` кожного плагіна (чи доступні на диску оголошені `dependencies`/`optionalDependencies`).

`plugins search` виконує пошук доступних для встановлення пакетів плагінів у ClawHub і для кожного результату виводить підказку щодо встановлення (`openclaw plugins install clawhub:<package>`).

## Увімкнення та вимкнення плагінів

```bash
openclaw plugins enable <plugin-id>
openclaw plugins disable <plugin-id>
```

Змінює стан запису плагіна в конфігурації, не торкаючись установлених файлів. Деякі вбудовані плагіни (вбудовані постачальники моделей і мовлення, а також вбудований плагін браузера) увімкнені за замовчуванням; для інших після встановлення потрібно виконати `enable`.

## Установлення плагінів

```bash
# Search ClawHub for plugin packages.
openclaw plugins search "calendar"

# Install from ClawHub.
openclaw plugins install clawhub:<package>
openclaw plugins install clawhub:<package>@1.2.3
openclaw plugins install clawhub:<package>@beta

# Install from npm.
openclaw plugins install npm:<package>
openclaw plugins install npm:@scope/openclaw-plugin@1.2.3
openclaw plugins install npm:@openclaw/codex

# Install from a local npm-pack artifact.
openclaw plugins install npm-pack:<path.tgz>

# Install from git or a local development checkout.
openclaw plugins install git:github.com/acme/openclaw-plugin@v1.0.0
openclaw plugins install ./my-plugin
openclaw plugins install --link ./my-plugin
```

Під час переходу на запуск специфікації пакетів без префікса встановлюються з npm, якщо назва не збігається з ідентифікатором вбудованого або офіційного плагіна; у такому разі OpenClaw натомість використовує відповідну локальну або офіційну копію. Для детермінованого вибору джерела використовуйте `clawhub:`, `npm:`, `git:` або `npm-pack:`.

Використовуйте `--force` лише для перезапису наявної цілі встановлення з іншого джерела. Для звичайного оновлення відстежуваного встановлення з npm, ClawHub або пакета обробників використовуйте натомість `openclaw plugins update`; `--force` не підтримується разом із `--link`.

## Перезапуск і перевірка

Запущений керований Gateway з увімкненим перезавантаженням конфігурації автоматично перезапускається після встановлення, оновлення або видалення коду плагіна. Якщо Gateway некерований або перезавантаження вимкнене, перезапустіть його самостійно, перш ніж перевіряти активні поверхні середовища виконання:

```bash
openclaw gateway restart
openclaw plugins inspect <plugin-id> --runtime --json
```

`inspect --runtime` завантажує модуль плагіна й підтверджує, що він зареєстрував поверхні середовища виконання (інструменти, обробники, служби, методи Gateway, маршрути HTTP, команди CLI, що належать плагіну). Звичайні `inspect` і `list` виконують лише холодні перевірки маніфесту, конфігурації та реєстру.

## Оновлення плагінів

```bash
openclaw plugins update <plugin-id>
openclaw plugins update <npm-package-or-spec>
openclaw plugins update --all
openclaw plugins update <plugin-id> --dry-run
```

Якщо передати ідентифікатор плагіна, повторно використовується його відстежувана специфікація встановлення: збережені теги дистрибутива (`@beta`) і точно закріплені версії переносяться до наступних запусків `update <plugin-id>`.

`openclaw plugins update --all` — це шлях масового обслуговування. Він так само дотримується звичайних відстежуваних специфікацій установлення, але записи довірених офіційних плагінів OpenClaw синхронізуються з поточною цільовою версією офіційного каталогу замість збереження прив’язки до застарілого точного офіційного пакета; коли `update.channel` має значення `beta`, така синхронізація надає перевагу гілці бета-випусків. Використовуйте цільову команду `update <plugin-id>`, щоб не змінювати точну або позначену тегом офіційну специфікацію.

Для встановлень із npm передайте явну специфікацію пакета, щоб змінити відстежуваний запис:

```bash
openclaw plugins update @scope/openclaw-plugin@beta
openclaw plugins update @scope/openclaw-plugin
```

Друга команда повертає плагін до стандартної гілки випусків реєстру, якщо раніше його було закріплено за точною версією або тегом.

Точні правила резервного вибору та закріплення див. у розділі [`openclaw plugins`](/uk/cli/plugins#update).

## Видалення плагінів

```bash
openclaw plugins uninstall <plugin-id> --dry-run
openclaw plugins uninstall <plugin-id>
openclaw plugins uninstall <plugin-id> --keep-files
```

Видалення прибирає запис плагіна з конфігурації, збережений запис індексу плагінів, записи списків дозволів і заборон, а також пов’язані записи `plugins.load.paths`, якщо вони застосовні. Керований каталог установлення видаляється, якщо не передано `--keep-files`. Запущений керований Gateway автоматично перезапускається, якщо видалення змінює джерело плагіна.

У режимі Nix (`OPENCLAW_NIX_MODE=1`) установлення, оновлення, видалення, увімкнення та вимкнення плагінів недоступні; керуйте цими параметрами в джерелі Nix для відповідного встановлення.

## Вибір джерела

| Джерело      | Коли використовувати                                                                    | Приклад                                                        |
| ----------- | --------------------------------------------------------------------------- | -------------------------------------------------------------- |
| ClawHub     | Потрібні вбудовані в OpenClaw пошук, підсумки сканування, версії та підказки     | `openclaw plugins install clawhub:<package>`                   |
| git         | Потрібна гілка, тег або коміт із репозиторію                         | `openclaw plugins install git:github.com/<owner>/<repo>@<ref>` |
| локальний шлях  | Ви розробляєте або тестуєте плагін на тому самому комп’ютері                  | `openclaw plugins install --link ./my-plugin`                  |
| каталог | Ви встановлюєте сумісний із Claude плагін із каталогу                   | `openclaw plugins install <plugin> --marketplace <source>`     |
| пакет npm    | Ви перевіряєте локальний артефакт пакета із застосуванням семантики встановлення npm      | `openclaw plugins install npm-pack:<path.tgz>`                 |
| npmjs.com   | Ви вже постачаєте пакети JavaScript або потребуєте тегів дистрибутива npm чи приватного реєстру | `openclaw plugins install npm:@acme/openclaw-plugin`           |

Керовані встановлення з локального шляху мають указувати на каталоги або архіви плагінів. Окремі файли плагінів додавайте до `plugins.load.paths`, а не встановлюйте їх через `plugins install`.

## Публікація плагінів

ClawHub — основна загальнодоступна поверхня пошуку плагінів OpenClaw. Публікуйте їх там, якщо хочете, щоб користувачі могли знайти метадані плагіна, історію версій, результати сканування реєстру та підказки щодо встановлення ще до самого встановлення.

```bash
npm i -g clawhub
clawhub login
clawhub package publish your-org/your-plugin --dry-run
clawhub package publish your-org/your-plugin
clawhub package publish your-org/your-plugin@v1.0.0
```

Перед публікацією нативні плагіни npm мають містити маніфест плагіна (`openclaw.plugin.json`) і метадані `package.json`:

```json package.json
{
  "name": "@acme/openclaw-plugin",
  "version": "1.0.0",
  "type": "module",
  "openclaw": {
    "extensions": ["./dist/index.js"]
  }
}
```

```bash
npm publish --access public
openclaw plugins install npm:@acme/openclaw-plugin
openclaw plugins install npm:@acme/openclaw-plugin@beta
openclaw plugins install npm:@acme/openclaw-plugin@1.0.0
```

Для повного опису контракту публікації використовуйте наведені нижче сторінки, а не цю сторінку як довідник із публікації:

- [Публікація в ClawHub](/uk/clawhub/publishing) пояснює власників, області видимості, випуски, перевірку, валідацію пакетів і передавання пакетів.
- [Створення плагінів](/uk/plugins/building-plugins) показує повну структуру пакета плагіна (зокрема `openclaw.plugin.json`) і робочий процес першої публікації.
- [Маніфест плагіна](/uk/plugins/manifest) визначає поля нативного маніфесту плагіна.

Якщо той самий пакет доступний і в ClawHub, і в npm, використовуйте явний префікс `clawhub:` або `npm:`, щоб примусово вибрати одне джерело.
