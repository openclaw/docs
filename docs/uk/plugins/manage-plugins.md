Інтерфейс керування охоплює типовий робочий процес пошуку, встановлення, увімкнення та вимкнення. CLI додає оновлення, видалення, розширене налаштування та явні засоби керування джерелом встановлення. Повний опис команд, прапорців, правил вибору джерела та граничних випадків див. у [`openclaw plugins`](/uk/cli/plugins).

Типовий робочий процес CLI: знайдіть пакет, установіть його з ClawHub, npm, git або локального шляху, дозвольте керованому Gateway автоматично перезапуститися (або перезапустіть його вручну), а потім перевірте реєстрації плагіна в середовищі виконання.

## Використання інтерфейсу керування

Відкрийте **Плагіни** в інтерфейсі керування або перейдіть за шляхом `/settings/plugins` відносно налаштованого базового шляху інтерфейсу керування. Наприклад, для базового шляху `/openclaw` використовується `/openclaw/settings/plugins`. Сторінка має дві вкладки:

- **Установлені** показує повний локальний перелік, згрупований за категоріями (канали, постачальники моделей, пам’ять, інструменти). Кожен рядок відкриває докладний перегляд; його меню додаткових дій (`…`) дає змогу ввімкнути або вимкнути плагін, а для встановлених зовнішніх плагінів пропонує **Видалити**. На вкладці також наведено налаштовані [сервери MCP](/uk/cli/mcp) з такими самими доступними через меню діями ввімкнення, вимкнення та видалення, які змінюють `mcp.servers` у конфігурації Gateway.
- **Огляд** — це магазин: рекомендовані плагіни, включені до OpenClaw, офіційні зовнішні плагіни та добірка конекторів. Картки конекторів дають змогу одним натисканням додати розміщений сервер MCP (GitHub, Notion, Linear, Sentry, Home Assistant) або перейти до попередньо заповненого пошуку ClawHub. Введення тексту в поле пошуку надсилає запит до [ClawHub](https://clawhub.ai/plugins) безпосередньо на сторінці та додає розділ **Із ClawHub** із кількістю завантажень і позначками перевірки джерела.

Включені плагіни не потребують установлення пакета. У їхньому меню доступна дія **Увімкнути** або **Вимкнути**. Наприклад, Workboard включено до OpenClaw і типово вимкнено, тому виберіть **Увімкнути**, щоб активувати його. Вбудовані плагіни не можна видалити — їх можна лише вимкнути.

Для доступу до каталогу та пошуку потрібен `operator.read`. Для встановлення, увімкнення, вимкнення, видалення та зміни серверів MCP потрібен `operator.admin`. Установлення з ClawHub виконує Gateway зі збереженням перевірок довіри, цілісності та політики встановлення плагінів. Коли адміністратор вмикає встановлений плагін, ця явна довіра також фіксується додаванням вибраного плагіна до наявного обмежувального списку `plugins.allow`. Явний запис `plugins.deny` залишається визначальним, і його потрібно видалити перед увімкненням плагіна.

Установлення або видалення коду плагіна потребує перезапуску Gateway. Зміни стану ввімкнення можна застосувати без перезапуску, якщо встановлений плагін і поточне середовище виконання Gateway це підтримують; інакше інтерфейс повідомляє про потребу перезапуску. Для конекторів MCP на основі OAuth після додавання все одно потрібно один раз виконати `openclaw mcp login <name>` через CLI.

Інтерфейс керування не встановлює плагіни з довільних джерел npm, git або локальних шляхів, не оновлює їх і не надає розширених засобів налаштування плагінів. Для цих операцій використовуйте наведені нижче робочі процеси CLI.

## Перегляд і пошук плагінів

```bash
openclaw plugins list
openclaw plugins list --enabled
openclaw plugins list --verbose
openclaw plugins list --json
openclaw plugins search "calendar"
```

`--json` для скриптів:

```bash
openclaw plugins list --json \
  | jq '.plugins[] | {id, enabled, format, source, dependencyStatus}'
```

`plugins list` — це автономна перевірка переліку: що OpenClaw може виявити з конфігурації, маніфестів і збереженого реєстру плагінів. Вона не доводить, що вже запущений Gateway імпортував середовище виконання плагіна. Виведення JSON містить діагностику реєстру та значення `dependencyStatus` для кожного плагіна (чи доступні на диску оголошені `dependencies`/`optionalDependencies`).

`plugins search` шукає в ClawHub доступні для встановлення пакети плагінів і виводить підказку щодо встановлення (`openclaw plugins install clawhub:<package>`) для кожного результату.

## Увімкнення та вимкнення плагінів

```bash
openclaw plugins enable <plugin-id>
openclaw plugins disable <plugin-id>
```

Перемикає запис плагіна в конфігурації, не змінюючи встановлені файли. Деякі вбудовані плагіни (вбудовані постачальники моделей і синтезу мовлення, вбудований плагін браузера) типово ввімкнено; для інших після встановлення потрібно виконати `enable`.

## Установлення плагінів

```bash
# Пошук пакетів плагінів у ClawHub.
openclaw plugins search "calendar"

# Установлення з ClawHub.
openclaw plugins install clawhub:<package>
openclaw plugins install clawhub:<package>@1.2.3
openclaw plugins install clawhub:<package>@beta

# Установлення з npm.
openclaw plugins install npm:<package>
openclaw plugins install npm:@scope/openclaw-plugin@1.2.3
openclaw plugins install npm:@openclaw/codex

# Установлення з локального артефакту npm-pack.
openclaw plugins install npm-pack:<path.tgz>

# Установлення з git або локальної робочої копії для розробки.
openclaw plugins install git:github.com/acme/openclaw-plugin@v1.0.0
openclaw plugins install ./my-plugin
openclaw plugins install --link ./my-plugin
```

Під час переходу на запуск специфікації пакетів без префікса встановлюються з npm, якщо назва не відповідає ідентифікатору вбудованого або офіційного плагіна; у такому разі OpenClaw натомість використовує відповідну локальну або офіційну копію. Для детермінованого вибору джерела використовуйте `clawhub:`, `npm:`, `git:` або `npm-pack:`. Вбудовані й офіційні пакети каталогу OpenClaw вважаються довіреними нарівні з пакетами ClawHub. Для нових довільних джерел npm, git, локального шляху чи архіву, `npm-pack:` або маркетплейсу під час неінтерактивного встановлення потрібен `--force` після перевірки джерела та підтвердження довіри до нього.

`--force` підтверджує джерело, відмінне від ClawHub, без запиту та за потреби перезаписує наявну ціль установлення. Для звичайного оновлення відстежуваного встановлення з npm, ClawHub або hook-pack натомість використовуйте `openclaw plugins update`. Разом із `--link` параметр `--force` лише підтверджує джерело; пов’язаний каталог не копіюється й не перезаписується.

## Перезапуск і перевірка

Запущений керований Gateway з увімкненим перезавантаженням конфігурації автоматично перезапускається після встановлення, оновлення або видалення коду плагіна. Якщо Gateway некерований або перезавантаження вимкнено, перезапустіть його самостійно перед перевіркою активних поверхонь середовища виконання:

```bash
openclaw gateway restart
openclaw plugins inspect <plugin-id> --runtime --json
```

`inspect --runtime` завантажує модуль плагіна й підтверджує реєстрацію поверхонь середовища виконання (інструментів, перехоплювачів, служб, методів Gateway, маршрутів HTTP, команд CLI, що належать плагіну). Звичайні `inspect` та `list` виконують лише автономні перевірки маніфесту, конфігурації та реєстру.

## Оновлення плагінів

```bash
openclaw plugins update <plugin-id>
openclaw plugins update <npm-package-or-spec>
openclaw plugins update --all
openclaw plugins update <plugin-id> --dry-run
```

Якщо передати ідентифікатор плагіна, повторно використовується відстежувана специфікація його встановлення: збережені dist-теги (`@beta`) і точні зафіксовані версії переносяться до наступних запусків `update <plugin-id>`.

`openclaw plugins update --all` — це шлях масового обслуговування. Він так само дотримується звичайних відстежуваних специфікацій установлення, але записи довірених офіційних плагінів OpenClaw синхронізуються з поточною ціллю офіційного каталогу замість збереження прив’язки до застарілого точного офіційного пакета; коли `update.channel` має значення `beta`, така синхронізація надає перевагу гілці бета-випусків. Використовуйте цільовий `update <plugin-id>`, щоб залишити точну або позначену тегом офіційну специфікацію без змін.

Для встановлень із npm передайте явну специфікацію пакета, щоб змінити відстежуваний запис:

```bash
openclaw plugins update @scope/openclaw-plugin@beta
openclaw plugins update @scope/openclaw-plugin
```

Друга команда повертає плагін до типової гілки випусків реєстру, якщо раніше його було прив’язано до точної версії або тегу.

Точні правила резервного вибору та фіксації версій див. у [`openclaw plugins`](/uk/cli/plugins#update).

## Видалення плагінів

```bash
openclaw plugins uninstall <plugin-id> --dry-run
openclaw plugins uninstall <plugin-id>
openclaw plugins uninstall <plugin-id> --keep-files
```

Під час видалення вилучаються запис плагіна з конфігурації, збережений запис індексу плагінів, записи списків дозволів і заборон, а також пов’язані записи `plugins.load.paths`, якщо це застосовно. Каталог керованого встановлення видаляється, якщо не передано `--keep-files`. Запущений керований Gateway автоматично перезапускається, коли видалення змінює джерело плагіна.

У режимі Nix (`OPENCLAW_NIX_MODE=1`) встановлення, оновлення, видалення, увімкнення та вимкнення плагінів недоступні; керуйте цими параметрами в джерелі Nix для відповідного встановлення.

## Вибір джерела

| Джерело    | Коли використовувати                                                         | Приклад                                                        |
| ----------- | --------------------------------------------------------------------------- | -------------------------------------------------------------- |
| ClawHub     | Потрібні орієнтовані на OpenClaw пошук, підсумки сканування, версії та підказки | `openclaw plugins install clawhub:<package>`                   |
| git         | Потрібна гілка, тег або коміт із репозиторію                                | `openclaw plugins install git:github.com/<owner>/<repo>@<ref>` |
| локальний шлях | Плагін розробляється або тестується на тому самому комп’ютері            | `openclaw plugins install --link ./my-plugin`                  |
| маркетплейс | Установлюється сумісний із Claude плагін із маркетплейсу                     | `openclaw plugins install <plugin> --marketplace <source>`     |
| npm pack    | Локальний артефакт пакета перевіряється за допомогою семантики встановлення npm | `openclaw plugins install npm-pack:<path.tgz>`                 |
| npmjs.com   | Пакети JavaScript уже публікуються або потрібні dist-теги npm чи приватний реєстр | `openclaw plugins install npm:@acme/openclaw-plugin`           |

Керовані встановлення з локального шляху мають вказувати на каталоги або архіви плагінів. Окремі файли плагінів розміщуйте в `plugins.load.paths`, а не встановлюйте їх за допомогою `plugins install`.

## Публікація плагінів

ClawHub — основний загальнодоступний засіб пошуку плагінів OpenClaw. Публікуйте їх там, якщо потрібно, щоб користувачі могли знайти метадані плагіна, історію версій, результати сканування реєстру та підказки щодо встановлення ще до встановлення.

```bash
npm i -g clawhub
clawhub login
clawhub package publish your-org/your-plugin --dry-run
clawhub package publish your-org/your-plugin
clawhub package publish your-org/your-plugin@v1.0.0
```

Перед публікацією нативні плагіни npm мають містити маніфест плагіна (`openclaw.plugin.json`) і метадані `package.json`:

```json package.json
{
  "name": "@acme/openclaw-plugin",
  "version": "1.0.0",
  "type": "module",
  "openclaw": {
    "extensions": ["./dist/index.js"]
  }
}
```

```bash
npm publish --access public
openclaw plugins install npm:@acme/openclaw-plugin
openclaw plugins install npm:@acme/openclaw-plugin@beta
openclaw plugins install npm:@acme/openclaw-plugin@1.0.0
```

Для повного опису вимог до публікації використовуйте ці сторінки, а не вважайте цю сторінку довідником із публікації:

- [Публікація в ClawHub](/uk/clawhub/publishing) пояснює власників, області видимості, випуски, перевірку, валідацію пакетів і передавання пакетів.
- [Створення плагінів](/uk/plugins/building-plugins) показує повну структуру пакета плагіна (включно з `openclaw.plugin.json`) і робочий процес першої публікації.
- [Маніфест плагіна](/uk/plugins/manifest) визначає поля маніфесту нативного плагіна.

Якщо той самий пакет доступний і в ClawHub, і в npm, використовуйте явний префікс `clawhub:` або `npm:`, щоб примусово вибрати одне джерело.
