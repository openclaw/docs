Плагіни спільноти — це сторонні пакети, які розширюють OpenClaw
каналами, інструментами, провайдерами, хуками або іншими можливостями. Використовуйте
[ClawHub](/clawhub) як основний засіб пошуку публічних плагінів
спільноти.

## Пошук плагінів

Шукайте в ClawHub за допомогою CLI:

```bash
openclaw plugins search "calendar"
```

Установіть плагін із ClawHub, явно вказавши префікс джерела:

```bash
openclaw plugins install clawhub:<package-name>
```

Під час переходу на нову систему запуску npm залишається підтримуваним способом прямого встановлення:

```bash
openclaw plugins install npm:<package-name>
```

Скористайтеся розділом [Керування плагінами](/uk/plugins/manage-plugins), щоб переглянути типові приклади встановлення, оновлення,
перевірки та видалення. Повний довідник команд і правила вибору джерела див. у розділі
[`openclaw plugins`](/uk/cli/plugins).

## Публікація плагінів

Публікуйте загальнодоступні плагіни спільноти в ClawHub, щоб користувачі OpenClaw могли їх знаходити
та встановлювати. ClawHub керує актуальним списком пакетів, історією випусків,
станом сканування та підказками щодо встановлення; документація не підтримує статичного
каталогу сторонніх плагінів.

```bash
clawhub package publish your-org/your-plugin --dry-run
clawhub package publish your-org/your-plugin
```

Перед публікацією переконайтеся, що плагін має метадані пакета, маніфест
плагіна, документацію з налаштування та чітко визначеного відповідального за супровід. Перед
створенням випуску ClawHub перевіряє область власника, назву пакета, версію,
обмеження файлів і метадані джерела, а потім приховує нові випуски від звичайних засобів установлення та
завантаження до завершення перевірки й підтвердження.

Контрольний список перед публікацією:

| Вимога                  | Навіщо                                                        |
| ----------------------- | ------------------------------------------------------------- |
| Опубліковано в ClawHub  | Підказки `openclaw plugins install` мають працювати для користувачів |
| Публічний репозиторій GitHub | Перегляд вихідного коду, відстеження проблем, прозорість       |
| Документація з налаштування та використання | Користувачі мають знати, як його налаштувати                   |
| Активний супровід       | Нещодавні оновлення або оперативне опрацювання проблем         |
