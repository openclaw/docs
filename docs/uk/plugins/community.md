Community plugins — це сторонні пакети, які розширюють OpenClaw каналами,
інструментами, провайдерами, хуками або іншими можливостями. Використовуйте [ClawHub](/uk/clawhub) як
основну поверхню виявлення для публічних Community plugins.

## Пошук plugins

Шукайте в ClawHub із CLI:

```bash
openclaw plugins search "calendar"
```

Встановіть Plugin із ClawHub з явним префіксом джерела:

```bash
openclaw plugins install clawhub:<package-name>
```

npm залишається підтримуваним шляхом прямого встановлення під час перехідного запуску:

```bash
openclaw plugins install npm:<package-name>
```

Використовуйте [Керування plugins](/uk/plugins/manage-plugins) для поширених прикладів встановлення, оновлення,
перевірки та видалення. Використовуйте [`openclaw plugins`](/uk/cli/plugins) для
повного довідника команд і правил вибору джерела.

## Публікація plugins

Публікуйте публічні Community plugins у ClawHub, коли хочете, щоб користувачі OpenClaw могли
знаходити й встановлювати їх. ClawHub керує актуальним переліком пакетів, історією
випусків, статусом сканування та підказками встановлення; документація не підтримує статичний
каталог сторонніх plugins.

```bash
clawhub package publish your-org/your-plugin --dry-run
clawhub package publish your-org/your-plugin
```

Перед публікацією переконайтеся, що Plugin має метадані пакета, маніфест Plugin,
документацію з налаштування та чітко визначеного власника супроводу. ClawHub перевіряє область власника,
назву пакета, версію, обмеження файлів і метадані джерела перед створенням
випуску, а потім приховує нові випуски від звичайних поверхонь встановлення й завантаження
до завершення огляду та перевірки.

Використовуйте цей контрольний список перед публікацією:

| Вимога              | Чому                                                |
| -------------------- | --------------------------------------------------- |
| Опубліковано в ClawHub | Користувачам потрібні підказки `openclaw plugins install`, щоб це працювало |
| Публічний репозиторій GitHub | Огляд вихідного коду, відстеження питань, прозорість |
| Документація з налаштування та використання | Користувачам потрібно знати, як це налаштувати |
| Активний супровід   | Нещодавні оновлення або оперативна обробка питань |

Використовуйте ці сторінки для повного контракту публікації:

- [Публікація ClawHub](/uk/clawhub/publishing) пояснює власників, області, випуски,
  огляд, перевірку пакетів і передавання пакетів.
- [Створення plugins](/uk/plugins/building-plugins) показує форму пакета Plugin
  і перший робочий процес публікації.
- [Маніфест Plugin](/uk/plugins/manifest) визначає поля маніфесту native Plugin.
