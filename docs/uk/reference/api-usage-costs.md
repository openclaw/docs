Карта функцій OpenClaw, які можуть викликати платні API постачальників, із зазначенням того, звідки кожна з них отримує облікові дані та де відображаються відповідні витрати.

## Де відображаються витрати

**`/status`** (знімок окремого сеансу)

- Показує поточну модель сеансу, використання контексту та кількість токенів останньої відповіді.
- Додає **орієнтовну вартість** останньої відповіді, коли OpenClaw має метадані використання й локальні ціни для активної моделі, зокрема для постачальників без ключів API з явно заданими цінами, як-от моделі Bedrock `aws-sdk`.
- Якщо актуальний знімок сеансу містить мало даних, `/status` відновлює лічильники токенів і кешу та мітку активної моделі з останнього запису про використання в транскрипті. Наявні ненульові актуальні значення мають пріоритет над даними транскрипту; загальна кількість із транскрипту, що відповідає розміру запиту, усе одно може мати пріоритет, якщо збережена загальна кількість відсутня або менша.

**`/usage`** (нижній колонтитул кожного повідомлення)

- `/usage full` додає нижній колонтитул із даними про використання до кожної відповіді, зокрема **орієнтовну вартість**, якщо налаштовано локальні ціни й доступні метадані використання.
- `/usage tokens` показує лише токени. Середовища виконання з OAuth/токенами передплатного типу та CLI показують лише токени, якщо не надають сумісні метадані використання разом із явно заданою локальною ціною.
- `/usage cost` виводить локальне зведення витрат; `/usage off` вимикає нижній колонтитул.
- Примітка щодо Gemini CLI: і `stream-json`, і застарілий формат виведення `json` містять дані про використання в `stats`. OpenClaw нормалізує `stats.cached` у `cacheRead` і за потреби обчислює вхідні токени як `stats.input_tokens - stats.cached`.

**Control UI → Використання** (аналіз кількох сеансів)

- Показує отримані з транскриптів загальні кількості токенів і орієнтовні витрати за вибраний діапазон дат із розподілом за постачальниками, моделями, агентами, каналами й типами токенів.
- Порівнює коротші календарні проміжки, що завершуються в кінцеву дату вибраного діапазону. Відсутні дати вважаються календарними днями з нульовим використанням; їх не пропускають для створення щільнішого проміжку.
- Безпосередньо позначає масштаб щоденного графіка. Значок `√` означає, що стиснення за квадратним коренем зберігає видимість днів із низьким використанням.
- Ці підсумки описують доступну локальну історію сеансів, а не рахунок постачальника чи журнал витрат за весь час. Інтерфейс попереджає, якщо для деяких записів немає цін.

**Проміжки використання в CLI** (квоти постачальників, а не вартість окремих повідомлень)

- `openclaw status --usage` і `openclaw channels list` показують **проміжки використання** постачальників у форматі `X% left`.
- Поточні постачальники даних про проміжки використання: Anthropic, ClawRouter, DeepSeek, GitHub Copilot, Gemini CLI, MiniMax, OpenAI (охоплює автентифікацію ChatGPT/Codex через OAuth або токен), Xiaomi та z.ai. Повний список постачальників і прапорців наведено в розділах [CLI моделей](/uk/cli/models) і [CLI каналів](/uk/cli/channels).
- Необроблені поля MiniMax `usage_percent` / `usagePercent` повідомляють про залишок квоти, тому OpenClaw інвертує їх; поля на основі кількості мають пріоритет, якщо вони наявні. Якщо відповідь містить масив `model_remains`, OpenClaw вибирає запис моделі чату, за потреби визначає мітку проміжку з часових позначок і додає назву моделі до мітки плану.
- Автентифікація для отримання даних про використання надходить зі спеціальних хуків постачальника, якщо вони доступні; інакше OpenClaw використовує відповідні облікові дані OAuth або ключі API з профілів автентифікації, змінних середовища чи конфігурації.

Докладні приклади наведено в розділі [Використання токенів і витрати](/uk/reference/token-use).

<Note>
Anthropic підтвердила, що повторне використання Claude CLI (зокрема `claude -p`) є дозволеним шаблоном інтеграції, доки компанія не опублікує нову політику. Anthropic не надає оцінку вартості в доларах для окремого повідомлення, тому `/usage full` не може показувати вартість використання Claude CLI.
</Note>

## Як виявляються ключі

- **Профілі автентифікації**: окремі для кожного агента, зберігаються в `auth-profiles.json`.
- **Змінні середовища**: наприклад, `OPENAI_API_KEY`, `BRAVE_API_KEY`, `FIRECRAWL_API_KEY`.
- **Конфігурація**: `models.providers.*.apiKey`, `plugins.entries.*.config.webSearch.apiKey`, `plugins.entries.firecrawl.config.webFetch.apiKey`, `agents.defaults.memorySearch.*`, `talk.providers.*.apiKey`.
- **Skills**: `skills.entries.<name>.apiKey`, звідки ключ може експортуватися до середовища процесу Skills.

## Функції, які можуть витрачати кошти за ключами

### Основні відповіді моделі (чат + інструменти)

Кожна відповідь або виклик інструмента виконується через постачальника поточної моделі. Це основне джерело використання й витрат, зокрема для розміщених планів передплатного типу, рахунки за які виставляються поза локальним інтерфейсом OpenClaw: OpenAI Codex, Alibaba Cloud Model Studio Coding Plan, MiniMax Coding Plan, Z.AI/GLM Coding Plan і шлях входу через Claude від Anthropic з увімкненою функцією Extra Usage.

Налаштування цін описано в розділі [Моделі](/uk/providers/models), а їх відображення — у розділі [Використання токенів і витрати](/uk/reference/token-use).

### Розпізнавання медіаданих (аудіо/зображення/відео)

Вхідні медіадані можуть бути підсумовані або транскрибовані через API постачальника до запуску конвеєра відповіді. Підтримка постачальників реєструється окремо для кожного Plugin і змінюється з додаванням нових Plugin; поточний список і конфігурацію наведено в розділі [Розпізнавання медіаданих](/uk/nodes/media-understanding).

### Генерування зображень і відео

`image_generate` і `video_generate` спрямовують запити до будь-якого доступного налаштованого постачальника. Якщо `agents.defaults.imageGenerationModel` не задано, для генерування зображень може бути автоматично визначено типового постачальника на основі автентифікації; для генерування відео потрібно явно задати `agents.defaults.videoGenerationModel` (наприклад, `qwen/wan2.6-t2v`).

Поточний список постачальників наведено в розділах [Генерування зображень](/uk/tools/image-generation) і [Генерування відео](/uk/tools/video-generation).

### Векторні представлення пам’яті та семантичний пошук

Семантичний пошук у пам’яті використовує API векторних представлень, коли `agents.defaults.memorySearch.provider` визначає віддалений адаптер (наприклад, `openai`, `gemini`, `voyage`, `mistral`, `deepinfra`, `github-copilot`, `amazon-bedrock`). `memorySearch.provider = "lmstudio"` або `"ollama"` працює з локальним або самостійно розміщеним сервером і зазвичай не передбачає оплату розміщеного сервісу. `memorySearch.provider = "local"` виконує все на пристрої без використання API. Необов’язковий постачальник `memorySearch.fallback` може використовуватися в разі помилок локального створення векторних представлень.

Див. [Пам’ять](/uk/concepts/memory).

### Інструмент вебпошуку

`web_search` може спричиняти плату за використання залежно від вибраного постачальника. Кожен постачальник спочатку зчитує ключ зі змінної середовища, а потім із `plugins.entries.<id>.config.webSearch.apiKey`:

| Постачальник          | Змінні середовища                                                                                                                                                              |
| --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Brave Search          | `BRAVE_API_KEY`                                                                                                                                                                 |
| DuckDuckGo            | без ключа; неофіційний, на основі HTML, без оплати                                                                                                                              |
| Exa                   | `EXA_API_KEY`                                                                                                                                                                   |
| Firecrawl             | `FIRECRAWL_API_KEY`                                                                                                                                                             |
| Gemini (Google Search) | `GEMINI_API_KEY`                                                                                                                                                               |
| Grok (xAI)            | профіль OAuth xAI або `XAI_API_KEY`                                                                                                                                             |
| Kimi (Moonshot)       | `KIMI_API_KEY` або `MOONSHOT_API_KEY`                                                                                                                                           |
| MiniMax Search        | `MINIMAX_CODE_PLAN_KEY`, `MINIMAX_CODING_API_KEY`, `MINIMAX_OAUTH_TOKEN` або `MINIMAX_API_KEY`                                                                                  |
| Ollama Web Search     | без ключа для доступного локального хоста з виконаним входом; прямий пошук через `https://ollama.com` використовує `OLLAMA_API_KEY`; захищені автентифікацією хости повторно використовують звичайну автентифікацію постачальника Ollama за токеном-носієм |
| Parallel              | `PARALLEL_API_KEY`                                                                                                                                                              |
| Perplexity Search API | `PERPLEXITY_API_KEY` або `OPENROUTER_API_KEY`                                                                                                                                   |
| SearXNG               | `SEARXNG_BASE_URL`; без ключа/самостійно розміщений, без оплати розміщеного сервісу                                                                                             |
| Tavily                | `TAVILY_API_KEY`                                                                                                                                                                |

Застарілі шляхи конфігурації `tools.web.search.*` усе ще завантажуються через шар сумісності, але більше не є рекомендованим інтерфейсом.

**Безкоштовний кредит Brave Search**: кожен план містить поновлюваний безкоштовний кредит у розмірі 5 доларів на місяць. План Search коштує 5 доларів за 1 000 запитів, тому кредит безкоштовно покриває 1 000 запитів на місяць. Установіть обмеження використання на панелі Brave, щоб уникнути неочікуваних витрат.

Див. [Вебінструменти](/uk/tools/web).

### Інструмент отримання вебвмісту (Firecrawl)

`web_fetch` може викликати Firecrawl із початковим доступом без ключа; додайте `FIRECRAWL_API_KEY` (або `plugins.entries.firecrawl.config.webFetch.apiKey`) для вищих лімітів. Якщо Firecrawl не налаштовано, інструмент переходить до прямого отримання даних разом із вбудованим Plugin `web-readability` (без платного API). Вимкніть `plugins.entries.web-readability.enabled`, щоб пропустити локальне видобування за допомогою Readability.

Див. [Вебінструменти](/uk/tools/web).

### Знімки використання постачальників (стан/працездатність)

`openclaw status --usage` і `openclaw models status --json` викликають кінцеві точки використання постачальників, щоб показати проміжки квот або стан автентифікації. Кількість викликів невелика, але вони все одно звертаються до API постачальників.

Див. [CLI моделей](/uk/cli/models).

### Захисне підсумовування під час Compaction

Захисний механізм Compaction може підсумовувати історію сеансу за допомогою поточної моделі, що під час виконання спричиняє виклики API постачальника.

Див. [Керування сеансами та Compaction](/uk/reference/session-management-compaction).

### Сканування та перевірка моделей

`openclaw models scan` може перевіряти моделі OpenRouter і використовує `OPENROUTER_API_KEY`, коли перевірку ввімкнено.

Див. [CLI моделей](/uk/cli/models).

### Розмовний режим (мовлення)

Розмовний режим може викликати ElevenLabs, якщо його налаштовано: `ELEVENLABS_API_KEY` або `talk.providers.elevenlabs.apiKey`.

Див. [Розмовний режим](/uk/nodes/talk).

### Skills (сторонні API)

Skills можуть зберігати `apiKey` у `skills.entries.<name>.apiKey`. Якщо Skills використовує цей ключ для зовнішнього API, вартість визначається постачальником відповідного Skills.

Див. [Skills](/uk/tools/skills).
