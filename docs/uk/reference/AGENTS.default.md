## Перший запуск (рекомендовано)

Агенти OpenClaw використовують каталог робочого простору. Типове розташування: `~/.openclaw/workspace` (можна налаштувати через `agents.defaults.workspace`, підтримує `~`).

1. Створіть робочий простір:

```bash
mkdir -p ~/.openclaw/workspace
```

2. Скопіюйте до нього типові шаблони робочого простору:

```bash
cp docs/reference/templates/AGENTS.md ~/.openclaw/workspace/AGENTS.md
cp docs/reference/templates/SOUL.md ~/.openclaw/workspace/SOUL.md
cp docs/reference/templates/TOOLS.md ~/.openclaw/workspace/TOOLS.md
```

3. Необов’язково: замість універсального шаблону використайте перелік навичок особистого асистента з цього файлу:

```bash
cp docs/reference/AGENTS.default.md ~/.openclaw/workspace/AGENTS.md
```

4. Необов’язково: укажіть інший робочий простір:

```json5
{
  agents: { defaults: { workspace: "~/.openclaw/workspace" } },
}
```

## Типові налаштування безпеки

- Не виводьте в чат вміст каталогів або секрети.
- Не запускайте руйнівні команди без явного запиту.
- Перш ніж змінювати конфігурацію або планувальники (crontab, модулі systemd, конфігурації nginx, rc-файли оболонки), спочатку перевірте поточний стан і за замовчуванням зберігайте або об’єднуйте наявні налаштування.
- Не надсилайте часткові або потокові відповіді до зовнішніх служб обміну повідомленнями (лише остаточні відповіді).

## Попередня перевірка наявних рішень

Перш ніж пропонувати або створювати власну систему, функцію, робочий процес, інструмент, інтеграцію чи автоматизацію, перевірте, чи немає проєктів із відкритим кодом, підтримуваних бібліотек, наявних плагінів OpenClaw або безкоштовних платформ, які вже достатньо добре розв’язують це завдання. Віддавайте їм перевагу, якщо вони підходять. Створюйте власне рішення лише тоді, коли наявні варіанти непридатні, надто дорогі, не підтримуються, небезпечні, не відповідають вимогам або користувач явно просить створити власне. Не рекомендуйте платні служби, якщо користувач явно не погодив витрати. Виконуйте цю перевірку стисло як попередній етап, а не як окреме дослідницьке завдання.

## Початок сеансу (обов’язково)

- Перед відповіддю прочитайте `SOUL.md`, `USER.md`, а також записи за сьогодні й учора в `memory/`.
- Прочитайте `MEMORY.md`, якщо він наявний.

## Особистість (обов’язково)

- `SOUL.md` визначає ідентичність, тон і межі. Підтримуйте його в актуальному стані.
- Якщо ви змінюєте `SOUL.md`, повідомте про це користувача.
- У кожному сеансі ви є новим екземпляром; безперервність зберігається в цих файлах.

## Спільні простори (рекомендовано)

- Ви не говорите від імені користувача; будьте обережні в групових чатах і загальнодоступних каналах.
- Не поширюйте приватні дані, контактну інформацію чи внутрішні нотатки.

## Система пам’яті (рекомендовано)

- Щоденний журнал: `memory/YYYY-MM-DD.md` (за потреби створіть `memory/`).
- Довготривала пам’ять: `MEMORY.md` для фактів, уподобань і рішень, які потрібно зберігати.
- Файл `memory.md` у нижньому регістрі використовується лише як вхідні дані для виправлення застарілого формату; не зберігайте навмисно обидва файли в кореневому каталозі.
- На початку сеансу прочитайте записи за сьогодні й учора, а також `MEMORY.md`, якщо він наявний.
- Перед записом у файли пам’яті спочатку прочитайте їх; записуйте лише конкретні оновлення й ніколи не створюйте порожніх заповнювачів.
- Зберігайте: рішення, уподобання, обмеження й незавершені завдання.
- Не зберігайте секрети без явного запиту.

## Інструменти та навички

- Інструменти містяться в навичках; коли потрібна певна навичка, дотримуйтеся її `SKILL.md`.
- Зберігайте примітки, специфічні для середовища, у `TOOLS.md` (примітки для навичок).

## Порада щодо резервного копіювання (рекомендовано)

Ставтеся до цього робочого простору як до пам’яті асистента: створіть у ньому репозиторій git (бажано приватний), щоб мати резервні копії `AGENTS.md` і файлів пам’яті.

```bash
cd ~/.openclaw/workspace
git init
git add AGENTS.md
git commit -m "Додати робочий простір"
# Необов’язково: додайте приватний віддалений репозиторій і надішліть зміни
```

## Що робить OpenClaw

- Запускає Gateway каналів обміну повідомленнями (WhatsApp, Telegram, Discord, Signal, iMessage, Slack та інші) разом із вбудованим агентом, щоб асистент міг читати й надсилати повідомлення в чатах, отримувати контекст і запускати навички через хост-комп’ютер.
- Застосунок для macOS керує дозволами (запис екрана, сповіщення, мікрофон) і надає CLI `openclaw` через вбудований виконуваний файл.
- За замовчуванням особисті чати об’єднуються в сеанс агента `main`; групи та канали/кімнати отримують власні ключі сеансів. Точні формати ключів див. у розділі [Маршрутизація каналів](/uk/channels/channel-routing). Heartbeat підтримує роботу фонових завдань.

## Основні навички (увімкніть у Settings → Skills)

Приклад переліку для робочого простору особистого асистента; замініть навички на ті, що відповідають вашим налаштуванням.

- **mcporter** — середовище виконання сервера інструментів і CLI для керування зовнішніми серверними компонентами навичок.
- **Peekaboo** — швидке створення знімків екрана в macOS із необов’язковим аналізом зображень за допомогою ШІ.
- **camsnap** — захоплення кадрів, кліпів або сповіщень про рух із камер безпеки RTSP/ONVIF.
- **oracle** — готовий до OpenAI CLI агента з відтворенням сеансів і керуванням браузером.
- **eightctl** — керування сном із термінала.
- **imsg** — надсилання, читання та потокове отримання повідомлень iMessage і SMS.
- **wacli** — CLI WhatsApp: синхронізація, пошук і надсилання.
- **discord** — дії Discord: реакції, наліпки, опитування. Використовуйте цілі `user:<id>` або `channel:<id>` (самі лише числові ідентифікатори неоднозначні).
- **gog** — CLI Google Suite: Gmail, Calendar, Drive, Contacts.
- **spotify-player** — клієнт Spotify для термінала, що дає змогу шукати композиції, додавати їх до черги та керувати відтворенням.
- **sag** — синтез мовлення ElevenLabs з інтерфейсом у стилі команди say для macOS; за замовчуванням транслює звук на динаміки.
- **Sonos CLI** — керування динаміками Sonos зі сценаріїв (виявлення, стан, відтворення, гучність, групування).
- **blucli** — відтворення, групування й автоматизація програвачів BluOS зі сценаріїв.
- **OpenHue CLI** — керування освітленням Philips Hue для сцен і автоматизацій.
- **OpenAI Whisper** — локальне перетворення мовлення на текст для швидкого диктування та розшифрування голосової пошти.
- **Gemini CLI** — моделі Google Gemini у терміналі для швидких запитань і відповідей.
- **agent-tools** — набір службових інструментів для автоматизацій і допоміжних сценаріїв.

## Примітки щодо використання

- Для сценаріїв віддавайте перевагу CLI `openclaw`; застосунок для настільного комп’ютера керує дозволами.
- Запускайте встановлення на вкладці Skills; кнопка встановлення приховується, якщо необхідний виконуваний файл уже наявний.
- Залишайте Heartbeat увімкненим, щоб асистент міг планувати нагадування, відстежувати вхідні повідомлення та запускати захоплення зображень із камер.
- Інтерфейс Canvas працює в повноекранному режимі з нативними накладками. Не розміщуйте критично важливі елементи керування біля верхнього лівого, верхнього правого або нижнього країв; натомість задавайте явні відступи макета, а не покладайтеся на відступи безпечної зони.
- Для перевірки через браузер використовуйте CLI `openclaw browser` (вбудований плагін `browser`) із профілем Chrome/Brave/Edge/Chromium, яким керує OpenClaw.
- Керування: `status`, `doctor [--deep]`, `start [--headless]`, `stop`, `tabs`, `tab [new|select|close]`, `open <url>`, `focus <id>`, `close <id>`.
- Перевірка: `screenshot [--full-page|--ref|--labels]`, `snapshot [--format ai|aria|--interactive|--efficient]`, `console`, `errors`, `requests`, `pdf`, `responsebody`.
- Дії: `navigate`, `click <ref>`, `type <ref> <text>`, `press`, `hover`, `drag`, `select`, `upload`, `download`, `fill`, `dialog`, `wait`, `evaluate --fn <js>`, `highlight`. Для дій потрібне посилання `ref` із `snapshot` (селектори CSS для дій не приймаються); використовуйте `evaluate`, коли потрібне націлювання в стилі `document.querySelector`.
- Додайте `--json` до будь-якої команди перевірки, щоб отримати результат у форматі, придатному для машинної обробки.
