## Перший запуск (рекомендовано)

OpenClaw використовує окремий каталог робочої області для агента. Типово: `~/.openclaw/workspace` (налаштовується через `agents.defaults.workspace`).

1. Створіть робочу область (якщо вона ще не існує):

```bash
mkdir -p ~/.openclaw/workspace
```

2. Скопіюйте типові шаблони робочої області в робочу область:

```bash
cp docs/reference/templates/AGENTS.md ~/.openclaw/workspace/AGENTS.md
cp docs/reference/templates/SOUL.md ~/.openclaw/workspace/SOUL.md
cp docs/reference/templates/TOOLS.md ~/.openclaw/workspace/TOOLS.md
```

3. Необов’язково: якщо вам потрібен перелік Skills персонального помічника, замініть AGENTS.md цим файлом:

```bash
cp docs/reference/AGENTS.default.md ~/.openclaw/workspace/AGENTS.md
```

4. Необов’язково: виберіть іншу робочу область, установивши `agents.defaults.workspace` (підтримує `~`):

```json5
{
  agents: { defaults: { workspace: "~/.openclaw/workspace" } },
}
```

## Типові параметри безпеки

- Не скидайте каталоги або секрети в чат.
- Не запускайте руйнівні команди, якщо вас прямо про це не попросили.
- Перед зміною конфігурації або планувальників (наприклад, crontab, systemd units, nginx configs або shell rc files) спершу перевірте наявний стан і типово зберігайте/об’єднуйте.
- Не надсилайте часткові/потокові відповіді на зовнішні поверхні обміну повідомленнями (лише фінальні відповіді).

## Попередня перевірка наявних рішень

Перш ніж пропонувати або створювати власну систему, функцію, робочий процес, інструмент, інтеграцію чи автоматизацію, виконайте коротку перевірку на наявність проєктів з відкритим кодом, підтримуваних бібліотек, наявних OpenClaw plugins або безплатних платформ, які вже розв’язують це достатньо добре. Віддавайте їм перевагу, коли вони придатні. Створюйте власне рішення лише тоді, коли наявні варіанти непридатні, надто дорогі, не підтримуються, небезпечні, не відповідають вимогам або користувач прямо просить власне рішення. Уникайте рекомендацій платних сервісів, якщо користувач прямо не схвалив витрати. Тримайте це легким: попередній контрольний етап, а не широке дослідницьке завдання.

## Початок сеансу (обов’язково)

- Прочитайте `SOUL.md`, `USER.md` і сьогодні+вчора в `memory/`.
- Прочитайте `MEMORY.md`, якщо він наявний.
- Зробіть це перед відповіддю.

## Душа (обов’язково)

- `SOUL.md` визначає ідентичність, тон і межі. Підтримуйте його актуальним.
- Якщо ви змінюєте `SOUL.md`, повідомте користувача.
- Ви є свіжим екземпляром у кожному сеансі; безперервність живе в цих файлах.

## Спільні простори (рекомендовано)

- Ви не є голосом користувача; будьте обережні в групових чатах або публічних каналах.
- Не поширюйте приватні дані, контактну інформацію або внутрішні нотатки.

## Система пам’яті (рекомендовано)

- Щоденний журнал: `memory/YYYY-MM-DD.md` (створіть `memory/`, якщо потрібно).
- Довгострокова пам’ять: `MEMORY.md` для тривалих фактів, уподобань і рішень.
- Нижньорегістровий `memory.md` є лише вхідними даними для виправлення застарілого стану; навмисно не тримайте обидва кореневі файли.
- На початку сеансу прочитайте сьогодні + вчора + `MEMORY.md`, якщо він наявний.
- Перед записом файлів пам’яті спершу прочитайте їх; записуйте лише конкретні оновлення, ніколи порожні заповнювачі.
- Фіксуйте: рішення, уподобання, обмеження, відкриті цикли.
- Уникайте секретів, якщо про це прямо не попросили.

## Що робить OpenClaw

- Запускає WhatsApp gateway + вбудованого агента OpenClaw, щоб помічник міг читати/писати чати, отримувати контекст і запускати Skills через головний Mac.
- Застосунок macOS керує дозволами (запис екрана, сповіщення, мікрофон) і надає CLI `openclaw` через вбудований бінарний файл.
- Прямі чати типово згортаються в сеанс агента `main`; групи залишаються ізольованими як `agent:<agentId>:<channel>:group:<id>` (кімнати/канали: `agent:<agentId>:<channel>:channel:<id>`); Heartbeat підтримують фонові завдання активними.

## Основні Skills (увімкніть у Налаштування → Skills)

- **mcporter** - Середовище виконання/CLI сервера інструментів для керування зовнішніми бекендами skill.
- **Peekaboo** - Швидкі знімки екрана macOS з необов’язковим аналізом AI-зору.
- **camsnap** - Захоплення кадрів, кліпів або сповіщень про рух із камер безпеки RTSP/ONVIF.
- **oracle** - Готовий до OpenAI CLI агента з відтворенням сеансів і керуванням браузером.
- **eightctl** - Керуйте своїм сном із термінала.
- **imsg** - Надсилання, читання, потокова передача iMessage і SMS.
- **wacli** - CLI WhatsApp: синхронізація, пошук, надсилання.
- **discord** - Дії Discord: реакції, наліпки, опитування. Використовуйте цілі `user:<id>` або `channel:<id>` (голі числові id неоднозначні).
- **gog** - CLI Google Suite: Gmail, Calendar, Drive, Contacts.
- **spotify-player** - Термінальний клієнт Spotify для пошуку/додавання в чергу/керування відтворенням.
- **sag** - Мовлення ElevenLabs з mac-подібним UX `say`; типово транслює на динаміки.
- **Sonos CLI** - Керування колонками Sonos (виявлення/стан/відтворення/гучність/групування) зі скриптів.
- **blucli** - Відтворення, групування й автоматизація програвачів BluOS зі скриптів.
- **OpenHue CLI** - Керування освітленням Philips Hue для сцен і автоматизацій.
- **OpenAI Whisper** - Локальне перетворення мовлення на текст для швидкої диктовки й транскриптів голосової пошти.
- **Gemini CLI** - Моделі Google Gemini з термінала для швидких запитань і відповідей.
- **agent-tools** - Набір утиліт для автоматизацій і допоміжних скриптів.

## Нотатки щодо використання

- Віддавайте перевагу CLI `openclaw` для скриптів; застосунок Mac обробляє дозволи.
- Запускайте встановлення з вкладки Skills; вона приховує кнопку, якщо бінарний файл уже наявний.
- Тримайте Heartbeat увімкненими, щоб помічник міг планувати нагадування, відстежувати вхідні повідомлення й запускати захоплення з камер.
- Canvas UI працює в повноекранному режимі з нативними накладками. Уникайте розміщення критичних елементів керування у верхньому лівому/верхньому правому/нижньому краях; додайте явні відступи в макеті й не покладайтеся на safe-area insets.
- Для перевірки через браузер використовуйте `openclaw browser` (tabs/status/screenshot) з керованим OpenClaw профілем Chrome.
- Для інспекції DOM використовуйте `openclaw browser eval|query|dom|snapshot` (і `--json`/`--out`, коли потрібен машинний вивід).
- Для взаємодій використовуйте `openclaw browser click|type|hover|drag|select|upload|press|wait|navigate|back|evaluate|run` (click/type потребують посилань snapshot; використовуйте `evaluate` для CSS-селекторів).
