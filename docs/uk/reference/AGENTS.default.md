## Перший запуск (рекомендовано)

OpenClaw використовує окремий каталог робочого простору для агента. Типово: `~/.openclaw/workspace` (налаштовується через `agents.defaults.workspace`).

1. Створіть робочий простір (якщо його ще не існує):

```bash
mkdir -p ~/.openclaw/workspace
```

2. Скопіюйте стандартні шаблони робочого простору в робочий простір:

```bash
cp docs/reference/templates/AGENTS.md ~/.openclaw/workspace/AGENTS.md
cp docs/reference/templates/SOUL.md ~/.openclaw/workspace/SOUL.md
cp docs/reference/templates/TOOLS.md ~/.openclaw/workspace/TOOLS.md
```

3. Необов'язково: якщо вам потрібен перелік Skills персонального асистента, замініть AGENTS.md цим файлом:

```bash
cp docs/reference/AGENTS.default.md ~/.openclaw/workspace/AGENTS.md
```

4. Необов'язково: виберіть інший робочий простір, задавши `agents.defaults.workspace` (підтримує `~`):

```json5
{
  agents: { defaults: { workspace: "~/.openclaw/workspace" } },
}
```

## Безпечні типові налаштування

- Не вивантажуйте каталоги або секрети в чат.
- Не виконуйте руйнівні команди, якщо про це явно не попросили.
- Не надсилайте часткові/потокові відповіді на зовнішні поверхні обміну повідомленнями (лише фінальні відповіді).

## Початок сеансу (обов'язково)

- Прочитайте `SOUL.md`, `USER.md` і сьогоднішній+вчорашній файли в `memory/`.
- Прочитайте `MEMORY.md`, якщо він є.
- Зробіть це перед відповіддю.

## Душа (обов'язково)

- `SOUL.md` визначає ідентичність, тон і межі. Підтримуйте його актуальним.
- Якщо ви змінюєте `SOUL.md`, повідомте користувача.
- Ви є свіжим екземпляром у кожному сеансі; безперервність зберігається в цих файлах.

## Спільні простори (рекомендовано)

- Ви не є голосом користувача; будьте обережні в групових чатах або публічних каналах.
- Не поширюйте приватні дані, контактну інформацію або внутрішні нотатки.

## Система пам'яті (рекомендовано)

- Щоденний журнал: `memory/YYYY-MM-DD.md` (створіть `memory/`, якщо потрібно).
- Довгострокова пам'ять: `MEMORY.md` для сталих фактів, уподобань і рішень.
- Нижньорегістровий `memory.md` є лише застарілим вхідним файлом для відновлення; не тримайте обидва кореневі файли навмисно.
- На початку сеансу прочитайте сьогодні + вчора + `MEMORY.md`, якщо він є.
- Фіксуйте: рішення, уподобання, обмеження, відкриті цикли.
- Уникайте секретів, якщо про це явно не попросили.

## Інструменти та Skills

- Інструменти містяться в Skills; дотримуйтеся `SKILL.md` кожного skill, коли він потрібен.
- Зберігайте нотатки, специфічні для середовища, у `TOOLS.md` (Нотатки для Skills).

## Порада щодо резервного копіювання (рекомендовано)

Якщо ви розглядаєте цей робочий простір як "пам'ять" Clawd, зробіть його git-репозиторієм (бажано приватним), щоб `AGENTS.md` і ваші файли пам'яті мали резервні копії.

```bash
cd ~/.openclaw/workspace
git init
git add AGENTS.md
git commit -m "Add Clawd workspace"
# Optional: add a private remote + push
```

## Що робить OpenClaw

- Запускає WhatsApp gateway + Pi агента для кодування, щоб асистент міг читати/писати чати, отримувати контекст і запускати skills через хостовий Mac.
- Застосунок macOS керує дозволами (запис екрана, сповіщення, мікрофон) і надає CLI `openclaw` через свій вбудований бінарний файл.
- Прямі чати типово згортаються в сеанс `main` агента; групи залишаються ізольованими як `agent:<agentId>:<channel>:group:<id>` (кімнати/канали: `agent:<agentId>:<channel>:channel:<id>`); Heartbeat-и підтримують фонові завдання активними.

## Основні Skills (увімкніть у Settings → Skills)

- **mcporter** - Середовище виконання/CLI сервера інструментів для керування зовнішніми бекендами skills.
- **Peekaboo** - Швидкі знімки екрана macOS з необов'язковим аналізом зору ШІ.
- **camsnap** - Захоплення кадрів, кліпів або сповіщень про рух із камер безпеки RTSP/ONVIF.
- **oracle** - Готовий до OpenAI CLI агента з відтворенням сеансу та керуванням браузером.
- **eightctl** - Керуйте своїм сном із термінала.
- **imsg** - Надсилання, читання й потокова передача iMessage і SMS.
- **wacli** - CLI WhatsApp: синхронізація, пошук, надсилання.
- **discord** - Дії Discord: реакції, наліпки, опитування. Використовуйте цілі `user:<id>` або `channel:<id>` (голі числові id неоднозначні).
- **gog** - CLI Google Suite: Gmail, Calendar, Drive, Contacts.
- **spotify-player** - Термінальний клієнт Spotify для пошуку/додавання в чергу/керування відтворенням.
- **sag** - Мовлення ElevenLabs з UX say у стилі Mac; типово передає потік на динаміки.
- **Sonos CLI** - Керуйте динаміками Sonos (виявлення/статус/відтворення/гучність/групування) зі скриптів.
- **blucli** - Відтворюйте, групуйте й автоматизуйте програвачі BluOS зі скриптів.
- **OpenHue CLI** - Керування освітленням Philips Hue для сцен і автоматизацій.
- **OpenAI Whisper** - Локальне перетворення мовлення на текст для швидкого диктування й транскриптів голосової пошти.
- **Gemini CLI** - Моделі Google Gemini з термінала для швидких запитань і відповідей.
- **agent-tools** - Набір утиліт для автоматизацій і допоміжних скриптів.

## Нотатки щодо використання

- Надавайте перевагу CLI `openclaw` для скриптів; застосунок Mac керує дозволами.
- Запускайте встановлення з вкладки Skills; вона приховує кнопку, якщо бінарний файл уже наявний.
- Тримайте Heartbeat-и увімкненими, щоб асистент міг планувати нагадування, моніторити вхідні та запускати захоплення з камер.
- Canvas UI працює на весь екран із нативними накладеннями. Уникайте розміщення критичних елементів керування у верхньому лівому/верхньому правому/нижніх краях; додайте явні відступи в макеті й не покладайтеся на safe-area insets.
- Для перевірки через браузер використовуйте `openclaw browser` (вкладки/статус/знімок екрана) з профілем Chrome, керованим OpenClaw.
- Для інспекції DOM використовуйте `openclaw browser eval|query|dom|snapshot` (і `--json`/`--out`, коли потрібен машинний вивід).
- Для взаємодій використовуйте `openclaw browser click|type|hover|drag|select|upload|press|wait|navigate|back|evaluate|run` (click/type потребують посилань на snapshot; використовуйте `evaluate` для CSS-селекторів).
