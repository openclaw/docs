# AGENTS.md - Персональний асистент OpenClaw (типово)

## Перший запуск (рекомендовано)

OpenClaw використовує окрему директорію робочого простору для агента. Типово: `~/.openclaw/workspace` (можна налаштувати через `agents.defaults.workspace`).

1. Створіть робочий простір (якщо він ще не існує):

```bash
mkdir -p ~/.openclaw/workspace
```

2. Скопіюйте типові шаблони робочого простору в робочий простір:

```bash
cp docs/reference/templates/AGENTS.md ~/.openclaw/workspace/AGENTS.md
cp docs/reference/templates/SOUL.md ~/.openclaw/workspace/SOUL.md
cp docs/reference/templates/TOOLS.md ~/.openclaw/workspace/TOOLS.md
```

3. Необов’язково: якщо вам потрібен список Skills для персонального асистента, замініть AGENTS.md цим файлом:

```bash
cp docs/reference/AGENTS.default.md ~/.openclaw/workspace/AGENTS.md
```

4. Необов’язково: виберіть інший робочий простір, задавши `agents.defaults.workspace` (підтримує `~`):

```json5
{
  agents: { defaults: { workspace: "~/.openclaw/workspace" } },
}
```

## Типові параметри безпеки

- Не виводьте списки директорій або секрети в чат.
- Не запускайте руйнівні команди, якщо про це прямо не попросили.
- Не надсилайте часткові/потокові відповіді до зовнішніх поверхонь обміну повідомленнями (лише фінальні відповіді).

## Початок сеансу (обов’язково)

- Прочитайте `SOUL.md`, `USER.md` і файли за сьогодні+вчора в `memory/`.
- Прочитайте `MEMORY.md`, якщо він є.
- Зробіть це до того, як відповідати.

## Soul (обов’язково)

- `SOUL.md` визначає ідентичність, тон і межі. Підтримуйте його в актуальному стані.
- Якщо ви змінюєте `SOUL.md`, повідомте користувача.
- Ви є новим екземпляром у кожному сеансі; безперервність зберігається в цих файлах.

## Спільні простори (рекомендовано)

- Ви не є голосом користувача; будьте обережні в групових чатах або публічних каналах.
- Не поширюйте приватні дані, контактну інформацію або внутрішні нотатки.

## Система пам’яті (рекомендовано)

- Щоденний журнал: `memory/YYYY-MM-DD.md` (створіть `memory/`, якщо потрібно).
- Довготривала пам’ять: `MEMORY.md` для сталих фактів, уподобань і рішень.
- `memory.md` у нижньому регістрі — лише вхід для legacy-відновлення; не зберігайте обидва кореневі файли навмисно.
- На початку сеансу прочитайте сьогоднішній + вчорашній файл + `MEMORY.md`, якщо він є.
- Фіксуйте: рішення, уподобання, обмеження, незавершені цикли.
- Уникайте секретів, якщо цього прямо не попросили.

## Інструменти та Skills

- Інструменти живуть у Skills; дотримуйтеся `SKILL.md` кожного Skill, коли він вам потрібен.
- Зберігайте примітки, специфічні для середовища, у `TOOLS.md` (Нотатки для Skills).

## Порада щодо резервного копіювання (рекомендовано)

Якщо ви розглядаєте цей робочий простір як «пам’ять» Clawd, зробіть його git-репозиторієм (ідеально — приватним), щоб `AGENTS.md` і ваші файли пам’яті мали резервні копії.

## Що робить OpenClaw

- Запускає gateway WhatsApp + coding agent на Pi, щоб асистент міг читати/писати чати, отримувати контекст і запускати Skills через хостовий Mac.
- Застосунок macOS керує дозволами (запис екрана, сповіщення, мікрофон) і надає CLI `openclaw` через свій вбудований бінарний файл.
- Прямі чати типово згортаються в сеанс `main` агента; групи лишаються ізольованими як `agent:<agentId>:<channel>:group:<id>` (кімнати/канали: `agent:<agentId>:<channel>:channel:<id>`); Heartbeat підтримують фонові завдання активними.

## Основні Skills (увімкніть у Settings → Skills)

- **mcporter** — runtime/CLI сервера інструментів для керування зовнішніми бекендами Skills.
- **Peekaboo** — швидкі знімки екрана macOS з необов’язковим AI-аналізом зображення.
- **camsnap** — захоплення кадрів, кліпів або сповіщень про рух із RTSP/ONVIF камер безпеки.
- **oracle** — готовий до OpenAI CLI агента з повтором сеансу та керуванням браузером.
- **eightctl** — керуйте своїм сном із термінала.
- **imsg** — надсилання, читання, потокова передача iMessage і SMS.
- **wacli** — CLI WhatsApp: синхронізація, пошук, надсилання.
- **discord** — дії в Discord: реакції, стікери, опитування. Використовуйте цілі `user:<id>` або `channel:<id>` (`bare` числові id неоднозначні).
- **gog** — CLI Google Suite: Gmail, Calendar, Drive, Contacts.
- **spotify-player** — термінальний клієнт Spotify для пошуку/додавання в чергу/керування відтворенням.
- **sag** — мовлення ElevenLabs зі схожим на mac UX команди say; типово транслює на динаміки.
- **Sonos CLI** — керування колонками Sonos (виявлення/статус/відтворення/гучність/групування) зі скриптів.
- **blucli** — відтворення, групування й автоматизація програвачів BluOS зі скриптів.
- **OpenHue CLI** — керування освітленням Philips Hue для сцен та автоматизацій.
- **OpenAI Whisper** — локальне перетворення мовлення в текст для швидкої диктовки та розшифрування голосової пошти.
- **Gemini CLI** — моделі Google Gemini з термінала для швидких запитань і відповідей.
- **agent-tools** — набір утиліт для автоматизацій і допоміжних скриптів.

## Примітки щодо використання

- Для скриптів віддавайте перевагу CLI `openclaw`; застосунок Mac керує дозволами.
- Виконуйте встановлення з вкладки Skills; вона приховує кнопку, якщо бінарний файл уже наявний.
- Тримайте Heartbeat увімкненими, щоб асистент міг планувати нагадування, стежити за вхідними повідомленнями й запускати захоплення з камер.
- Canvas UI працює в повноекранному режимі з нативними накладками. Уникайте розміщення критичних елементів керування у верхньому лівому/верхньому правому/нижніх краях; додавайте явні відступи в макет і не покладайтеся на safe-area insets.
- Для перевірки через браузер використовуйте `openclaw browser` (tabs/status/screenshot) з профілем Chrome, яким керує OpenClaw.
- Для інспекції DOM використовуйте `openclaw browser eval|query|dom|snapshot` (а також `--json`/`--out`, коли потрібен машинний вивід).
- Для взаємодій використовуйте `openclaw browser click|type|hover|drag|select|upload|press|wait|navigate|back|evaluate|run` (`click`/`type` потребують refs зі snapshot; використовуйте `evaluate` для CSS-селекторів).
