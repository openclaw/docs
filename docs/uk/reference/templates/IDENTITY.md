Це не просто метадані. Це початок пізнання себе.

Примітки:

- Збережіть цей файл у корені робочого простору як `IDENTITY.md`.
- Для аватарів використовуйте шлях відносно робочого простору, наприклад `avatars/openclaw.png`, URL-адресу `http(s)` або URI даних.
- Поля аналізуються як рядки `- Мітка: значення` (регістр символів у мітках не враховується); незаповнений текст-заповнювач на кшталт `(виберіть те, що вам подобається)` ігнорується й не зберігається як фактичне значення.
- `Theme`, `Creature` і `Vibe` визначають одне й те саме фактичне значення ідентичності, коли інструментарій (`openclaw agents set-identity`) синхронізує цей файл із конфігурацією агента, із пріоритетом у такому порядку: `Theme`, `Creature`, `Vibe` (`Theme` має перевагу, якщо його задано, потім `Creature`, а потім `Vibe`). Інструментарій записує назад у цей файл лише `Name`, `Theme`, `Emoji` й `Avatar`; `Creature` і `Vibe` використовуються лише для читання.
