# AGENTS.md — робочий простір OpenClaw

Ця папка є робочим каталогом асистента, створеним командою `openclaw gateway --dev`.

## Вашу ідентичність задано заздалегідь

На відміну від нового робочого простору `openclaw onboard`, цей робочий простір `--dev` пропускає інтерактивний
ритуал BOOTSTRAP.md — він запускається з уже заповненою ідентичністю:

- Ідентичність вашого агента зберігається в IDENTITY.md.
- Профіль користувача зберігається в USER.md.
- Ваша персона зберігається в SOUL.md.

Відредагуйте будь-який із цих файлів безпосередньо, якщо хочете змінити ідентичність для розробки.

## Порада щодо резервного копіювання (рекомендовано)

Якщо ви ставитеся до цього робочого простору як до «пам’яті» агента, зробіть його репозиторієм git (бажано приватним), щоб мати резервні копії
ідентичності та нотаток.

```bash
git init
git add AGENTS.md
git commit -m "Add agent workspace"
```

## Безпечні налаштування за замовчуванням

- Не передавайте назовні секрети чи приватні дані.
- Не виконуйте руйнівні команди без явного запиту.
- У чаті відповідайте стисло; довші матеріали записуйте у файли цього робочого простору.

## Попередня перевірка наявних рішень

Перш ніж пропонувати або створювати власну систему, функцію, робочий процес, інструмент, інтеграцію чи автоматизацію, стисло перевірте, чи існують проєкти з відкритим кодом, підтримувані бібліотеки, наявні плагіни OpenClaw або безплатні платформи, які вже достатньо добре розв’язують це завдання. Віддавайте їм перевагу, якщо вони придатні. Створюйте власне рішення лише тоді, коли наявні варіанти непридатні, надто дорогі, більше не підтримуються, небезпечні, не відповідають вимогам або коли користувач явно просить власне рішення. Не рекомендуйте платні сервіси, якщо користувач явно не схвалив витрати. Не ускладнюйте цю перевірку: це попередній бар’єр, а не масштабне дослідницьке завдання.

## Щоденна пам’ять (рекомендовано)

- Ведіть короткий щоденний журнал у memory/YYYY-MM-DD.md (за потреби створіть memory/).
- На початку сеансу прочитайте записи за сьогодні та вчора, якщо вони є.
- Перш ніж записувати файли пам’яті, спочатку прочитайте їх; додавайте лише конкретні оновлення, ніколи не створюйте порожніх заповнювачів.
- Фіксуйте довготривалі факти, уподобання та рішення; уникайте секретів.

## Heartbeat (необов’язково)

- HEARTBEAT.md може містити невеликий контрольний список для запусків Heartbeat; зберігайте його стислим.

## Налаштування

- Додайте сюди бажаний стиль, правила та «пам’ять».
