# Шаблон HEARTBEAT.md

`HEARTBEAT.md` розміщується в робочому просторі агента й містить контрольний список для періодичного Heartbeat. Залиште його порожнім або таким, що містить лише пробіли, коментарі Markdown, заголовки ATX, порожні заготовки списків (`- `, `* [ ]`) чи маркери огорожі, щоб OpenClaw повністю пропускав виклик моделі Heartbeat (`reason=empty-heartbeat-file`).

Типовий вміст, що постачається:

```markdown
<!-- Шаблон Heartbeat; вміст лише з коментарів запобігає запланованим викликам API Heartbeat. -->

# Залиште цей файл порожнім (або лише з коментарями), щоб пропускати виклики API Heartbeat.

# Додайте нижче завдання, якщо хочете, щоб агент періодично щось перевіряв.
```

Додавайте короткі завдання під рядками коментарів лише тоді, коли потрібні періодичні перевірки. Зберігайте файл невеликим: під час кожного циклу Heartbeat читає цей файл (типово кожні 30 хвилин), тому роздуті інструкції витрачають токени під час кожного пробудження.

Для перевірок лише після настання строку замість звичайного контрольного списку використовуйте структурований блок `tasks:` із полями `interval` і `prompt` для кожного завдання; формат і поведінку див. у [HEARTBEAT.md](/uk/gateway/heartbeat#heartbeatmd-optional).
