Збирання траєкторії — це бортовий самописець OpenClaw для кожного сеансу. Воно записує
структуровану часову шкалу для кожного запуску агента, після чого `/export-trajectory` пакує
поточний сеанс у відредагований пакет підтримки, що охоплює:

- Запит, системний запит та інструменти, надіслані моделі
- Які повідомлення транскрипту й виклики інструментів привели до відповіді
- Чи завершився запуск через перевищення часу очікування, чи був перерваний, ущільнений або зіткнувся з помилкою провайдера
- Які модель, плагіни, Skills і параметри середовища виконання були активні
- Метадані використання та кешу запитів, повернуті провайдером

Для загального звіту служби підтримки Gateway спочатку скористайтеся
[`/diagnostics`](/uk/gateway/diagnostics#chat-command); ця команда збирає
санітизований пакет Gateway, а для сеансів середовища OpenAI Codex може після
схвалення надіслати відгук про Codex до OpenAI. Використовуйте `/export-trajectory`, коли потрібна
докладна часова шкала запитів, інструментів і транскрипту для конкретного сеансу.

## Швидкий початок

Надішліть в активному сеансі (псевдонім `/trajectory`):

```text
/export-trajectory
```

OpenClaw записує пакет у робочому просторі:

```text
.openclaw/trajectory-exports/openclaw-trajectory-<session>-<timestamp>/
```

Передайте відносну назву вихідного каталогу, щоб перевизначити її:

```text
/export-trajectory bug-1234
```

Назва визначається всередині `.openclaw/trajectory-exports/`. Абсолютні шляхи та
шляхи `~` відхиляються.

Пакети траєкторій можуть містити запити, повідомлення моделі, схеми інструментів, результати
інструментів, події середовища виконання та локальні шляхи, тому команда чату завжди проходить
через схвалення виконання. Схваліть експорт один раз, коли маєте намір створити
пакет; не використовуйте дозвіл для всього. У групових чатах OpenClaw надсилає запит
на схвалення та результат експорту власнику приватно, а не публікує відомості про траєкторію
в спільній кімнаті.

Для локальної перевірки або робочих процесів підтримки запустіть базову команду CLI
безпосередньо:

```bash
openclaw sessions export-trajectory --session-key "agent:main:telegram:direct:123" --workspace .
```

Інші прапорці: `--output <path>` (назва каталогу всередині
`.openclaw/trajectory-exports`), `--store <path>` (перевизначення сховища сеансів),
`--agent <id>` (ідентифікатор агента для визначення сховища), `--json` (структурований вивід).

## Доступ

Експорт траєкторії — це команда власника. Відправник має пройти звичайні перевірки
авторизації команд, а також перевірку власника для каналу.

## Що записується

Збирання траєкторії типово ввімкнене для запусків агентів OpenClaw.

Події середовища виконання охоплюють:

- `session.started`
- `trace.metadata`
- `context.compiled`
- `prompt.submitted`
- `model.fallback_step`, зокрема початкову модель, наступну модель, причину/подробиці збою, позицію в ланцюжку та те, чи просунувся ланцюжок далі, завершився успішно або був вичерпаний
- `model.completed`
- `trace.artifacts`
- `session.ended`

Події транскрипту відтворюються з активної гілки сеансу: повідомлення
користувача, повідомлення асистента, виклики інструментів, результати інструментів, ущільнення, зміни
моделей, мітки та власні записи сеансу.

Події записуються у форматі JSON Lines із таким маркером схеми:

```json
{
  "traceSchema": "openclaw-trajectory",
  "schemaVersion": 1
}
```

## Файли пакета

| Файл                  | Вміст                                                                                       |
| --------------------- | ---------------------------------------------------------------------------------------------- |
| `manifest.json`       | Схема пакета, вихідні файли, кількість подій і список створених файлів                             |
| `events.jsonl`        | Упорядкована часова шкала середовища виконання та транскрипту                                                        |
| `session-branch.json` | Відредагована активна гілка транскрипту та заголовок сеансу                                           |
| `metadata.json`       | Версія OpenClaw, ОС/середовище виконання, модель, знімок конфігурації, плагіни, Skills і метадані запитів     |
| `artifacts.json`      | Остаточний стан, помилки, використання, кеш запитів, кількість ущільнень, текст асистента та метадані інструментів |
| `prompts.json`        | Надіслані запити та вибрані подробиці побудови запитів                                         |
| `system-prompt.txt`   | Останній скомпільований системний запит, якщо його було записано                                                   |
| `tools.json`          | Визначення інструментів, надіслані моделі, якщо їх було записано                                              |

`manifest.json` перелічує файли, наявні в певному пакеті; деякі файли
пропускаються, якщо сеанс не записав відповідні дані середовища виконання.

## Сховище записів

Події траєкторії середовища виконання зберігаються разом із сеансом у базі даних SQLite
відповідного агента. Під час експорту траєкторії створюється відредагований пакет підтримки JSONL;
активний запис середовища виконання не є додатковим файлом JSONL поруч із сеансом.

Застарілі файли `.trajectory.jsonl` і `.trajectory-path.json` усе ще можуть з'являтися
зі старіших випусків або явних експортів у застарілий файл. Обслуговування сеансів розглядає
ці файли як цілі для очищення; активне збирання записує рядки бази даних.

## Вимкнення збирання

```bash
export OPENCLAW_TRAJECTORY=0
```

Це вимикає збирання траєкторії середовища виконання перед запуском OpenClaw.
`/export-trajectory` усе ще може експортувати гілку транскрипту, але дані, доступні лише
під час виконання, як-от скомпільований контекст, артефакти провайдера та метадані запитів, можуть
бути відсутні.

## Налаштування часу очікування скидання

OpenClaw скидає рядки траєкторії середовища виконання під час очищення агента. Типовий
час очікування очищення становить 10,000 мс. На повільних дисках або у великих сховищах задайте
`OPENCLAW_TRAJECTORY_FLUSH_TIMEOUT_MS` перед запуском OpenClaw:

```bash
export OPENCLAW_TRAJECTORY_FLUSH_TIMEOUT_MS=30000
```

Це визначає, коли OpenClaw реєструє перевищення часу очікування `openclaw-trajectory-flush` і
продовжує роботу; це не змінює обмежень розміру траєкторії. Щоб налаштувати всі етапи
очищення агента, яким явно не передано час очікування, задайте
`OPENCLAW_AGENT_CLEANUP_TIMEOUT_MS`.

## Конфіденційність і обмеження

Пакети траєкторій призначені для підтримки та налагодження, а не для публікації. OpenClaw
редагує конфіденційні значення перед записуванням файлів експорту:

- облікові дані та відомі поля корисного навантаження, схожі на секрети
- дані зображень
- шляхи локального стану
- шляхи робочого простору, замінені на `$WORKSPACE_DIR`
- шляхи домашнього каталогу, якщо їх виявлено

Експортер також обмежує розмір вхідних даних:

- збирання під час виконання: активне збирання є рухомим вікном з обмеженням 10 MiB, яке відкидає найстаріші події, щоб звільнити місце для нових; експорт приймає наявні застарілі додаткові файли середовища виконання розміром до 50 MiB
- файли сеансу: 50 MiB
- події середовища виконання на один експорт: 200,000
- загальна кількість експортованих подій: 250,000
- окремі рядки подій середовища виконання обрізаються, якщо їх розмір перевищує 256 KiB

Перевіряйте пакети перед передаванням за межі команди. Редагування виконується за принципом
докладання максимальних зусиль і не може розпізнати кожен секрет, специфічний для програми.

## Усунення несправностей

Якщо експорт не містить подій середовища виконання:

- переконайтеся, що OpenClaw було запущено без `OPENCLAW_TRAJECTORY=0`
- надішліть ще одне повідомлення в сеансі, а потім повторіть експорт
- перевірте `manifest.json` на наявність `runtimeEventCount`

Якщо команда відхиляє вихідний шлях:

- використовуйте відносну назву, наприклад `bug-1234`
- не передавайте `/tmp/...` або `~/...`
- зберігайте експорт усередині `.openclaw/trajectory-exports/`

Якщо експорт завершується помилкою розміру, сеанс або додатковий файл перевищив
наведені вище безпечні обмеження експорту. Почніть новий сеанс або експортуйте менший
приклад відтворення.
