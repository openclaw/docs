Захоплення траєкторії — це бортовий самописець сесії в OpenClaw. Воно записує
структуровану часову шкалу для кожного запуску агента, а потім `/export-trajectory` пакує
поточну сесію у відредагований support-bundle.

Використовуйте це, коли потрібно відповісти на такі питання, як:

- Який prompt, системний prompt і які інструменти були надіслані моделі?
- Які повідомлення транскрипту й виклики інструментів привели до цієї відповіді?
- Чи був запуск перерваний через timeout, abort, Compaction або помилку provider?
- Які модель, plugins, Skills і налаштування runtime були активні?
- Які metadata usage і prompt-cache повернув provider?

## Швидкий старт

Надішліть це в активній сесії:

```text
/export-trajectory
```

Псевдонім:

```text
/trajectory
```

OpenClaw записує bundle у workspace:

```text
.openclaw/trajectory-exports/openclaw-trajectory-<session>-<timestamp>/
```

Ви можете вибрати відносну назву каталогу виводу:

```text
/export-trajectory bug-1234
```

Користувацький шлях розв’язується всередині `.openclaw/trajectory-exports/`. Абсолютні
шляхи та шляхи з `~` відхиляються.

## Доступ

Експорт траєкторії — це команда власника. Відправник має пройти звичайні перевірки
авторизації команд і перевірки власника для каналу.

## Що записується

Захоплення траєкторії типово ввімкнене для запусків агентів OpenClaw.

Події runtime включають:

- `session.started`
- `trace.metadata`
- `context.compiled`
- `prompt.submitted`
- `model.completed`
- `trace.artifacts`
- `session.ended`

Події транскрипту також реконструюються з активної гілки сесії:

- повідомлення користувача
- повідомлення assistant
- виклики інструментів
- результати інструментів
- Compaction
- зміни моделі
- labels і користувацькі записи сесії

Події записуються як JSON Lines з таким маркером схеми:

```json
{
  "traceSchema": "openclaw-trajectory",
  "schemaVersion": 1
}
```

## Файли bundle

Експортований bundle може містити:

| Файл                 | Вміст                                                                                         |
| -------------------- | ---------------------------------------------------------------------------------------------- |
| `manifest.json`      | Схема bundle, вихідні файли, кількість подій і список згенерованих файлів                     |
| `events.jsonl`       | Упорядкована часова шкала runtime і транскрипту                                                |
| `session-branch.json` | Відредагована активна гілка транскрипту і заголовок сесії                                     |
| `metadata.json`      | Версія OpenClaw, OS/runtime, модель, знімок config, plugins, Skills і metadata prompt         |
| `artifacts.json`     | Фінальний статус, помилки, usage, prompt cache, кількість Compaction, текст assistant і metadata інструментів |
| `prompts.json`       | Надіслані prompts і вибрані подробиці побудови prompt                                          |
| `system-prompt.txt`  | Останній скомпільований системний prompt, якщо був захоплений                                  |
| `tools.json`         | Визначення інструментів, надіслані моделі, якщо були захоплені                                 |

`manifest.json` перелічує файли, присутні в цьому bundle. Деякі файли пропускаються,
коли сесія не захопила відповідні runtime-дані.

## Розташування захоплення

За замовчуванням runtime-події траєкторії записуються поруч із файлом сесії:

```text
<session>.trajectory.jsonl
```

OpenClaw також записує найкращим можливим способом pointer-файл поруч із сесією:

```text
<session>.trajectory-path.json
```

Задайте `OPENCLAW_TRAJECTORY_DIR`, щоб зберігати runtime-sidecar-файли траєкторії в
окремому каталозі:

```bash
export OPENCLAW_TRAJECTORY_DIR=/var/lib/openclaw/trajectories
```

Коли цю змінну задано, OpenClaw записує в цей каталог по одному JSONL-файлу на кожен id сесії.

## Вимкнення захоплення

Задайте `OPENCLAW_TRAJECTORY=0` до запуску OpenClaw:

```bash
export OPENCLAW_TRAJECTORY=0
```

Це вимикає runtime-захоплення траєкторії. `/export-trajectory` усе ще може експортувати
гілку транскрипту, але файли лише runtime, такі як compiled context,
артефакти provider і metadata prompt, можуть бути відсутні.

## Конфіденційність і обмеження

Bundles траєкторії призначені для підтримки й налагодження, а не для публічного розміщення.
OpenClaw редагує чутливі значення перед записом файлів експорту:

- credentials і відомі поля payload, схожі на секрети
- дані зображень
- шляхи до локального state
- шляхи workspace, замінені на `$WORKSPACE_DIR`
- шляхи до домашнього каталогу, де їх виявлено

Експортер також обмежує розмір входу:

- runtime-sidecar-файли: 50 MiB
- файли сесій: 50 MiB
- runtime-події: 200,000
- загальна кількість експортованих подій: 250,000
- окремі рядки runtime-подій обрізаються понад 256 KiB

Переглядайте bundles перед тим, як ділитися ними поза межами своєї команди. Редагування працює в режимі best-effort
і не може знати про всі специфічні для вашого застосунку секрети.

## Усунення несправностей

Якщо в експорті немає runtime-подій:

- переконайтеся, що OpenClaw запущено без `OPENCLAW_TRAJECTORY=0`
- перевірте, чи `OPENCLAW_TRAJECTORY_DIR` вказує на каталог, у який можна записувати
- надішліть ще одне повідомлення в сесії, а потім знову експортуйте
- перевірте `manifest.json` на `runtimeEventCount`

Якщо команда відхиляє шлях виводу:

- використовуйте відносну назву, наприклад `bug-1234`
- не передавайте `/tmp/...` або `~/...`
- зберігайте експорт усередині `.openclaw/trajectory-exports/`

Якщо експорт завершується помилкою розміру, сесія або sidecar перевищили
безпечні ліміти експорту. Почніть нову сесію або експортуйте менше відтворення.
