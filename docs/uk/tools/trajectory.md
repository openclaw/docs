Захоплення траєкторії — це посесійний бортовий самописець OpenClaw. Воно записує
структуровану часову шкалу для кожного запуску агента, а потім `/export-trajectory` пакує
поточну сесію в редагований пакет підтримки.

Використовуйте його, коли потрібно відповісти на такі запитання:

- Який prompt, системний prompt і які інструменти було надіслано моделі?
- Які повідомлення transcript і виклики інструментів привели до цієї відповіді?
- Чи завершився запуск через тайм-аут, перервався, виконав compact, або отримав помилку провайдера?
- Які модель, плагіни, Skills і налаштування runtime були активними?
- Які метадані використання та prompt-cache повернув провайдер?

Якщо ви подаєте широкий звіт підтримки щодо live-проблеми Gateway, почніть із
[`/diagnostics`](/uk/gateway/diagnostics#chat-command). Diagnostics збирає
санітизований пакет Gateway і, для сесій harness OpenAI Codex, також може надіслати
відгук Codex на сервери OpenAI після схвалення. Використовуйте `/export-trajectory`, коли
вам потрібна саме детальна посесійна часова шкала prompt, інструментів і transcript.

## Швидкий старт

Надішліть це в активній сесії:

```text
/export-trajectory
```

Псевдонім:

```text
/trajectory
```

OpenClaw записує пакет у workspace:

```text
.openclaw/trajectory-exports/openclaw-trajectory-<session>-<timestamp>/
```

Можна вибрати назву відносного вихідного каталогу:

```text
/export-trajectory bug-1234
```

Користувацький шлях розв’язується всередині `.openclaw/trajectory-exports/`. Абсолютні
шляхи та шляхи `~` відхиляються.

Пакети траєкторії можуть містити prompts, повідомлення моделі, схеми інструментів, результати
інструментів, runtime-події та локальні шляхи. Тому chat slash command щоразу проходить
через схвалення exec. Схвалюйте експорт один раз, коли маєте намір
створити пакет; не використовуйте allow-all. У групових чатах OpenClaw надсилає
prompt схвалення та результат експорту власнику приватно, замість того щоб публікувати
деталі траєкторії назад у спільну кімнату.

Для локальної перевірки або робочих процесів підтримки також можна запустити схвалений command
path напряму:

```bash
openclaw sessions export-trajectory --session-key "agent:main:telegram:direct:123" --workspace .
```

## Доступ

Експорт траєкторії — це команда власника. Відправник має пройти звичайні перевірки
авторизації команд і перевірки власника для каналу.

## Що записується

Захоплення траєкторії ввімкнене за замовчуванням для запусків агентів OpenClaw.

Runtime-події включають:

- `session.started`
- `trace.metadata`
- `context.compiled`
- `prompt.submitted`
- `model.fallback_step`, зокрема вихідну модель, наступну модель, причину/деталі помилки, позицію в ланцюжку та те, чи fallback просунувся, завершився успішно або вичерпав ланцюжок
- `model.completed`
- `trace.artifacts`
- `session.ended`

Події transcript також реконструюються з активної гілки сесії:

- повідомлення користувача
- повідомлення асистента
- виклики інструментів
- результати інструментів
- compactions
- зміни моделі
- мітки та користувацькі записи сесії

Події записуються як JSON Lines із таким маркером схеми:

```json
{
  "traceSchema": "openclaw-trajectory",
  "schemaVersion": 1
}
```

## Файли пакета

Експортований пакет може містити:

| Файл                  | Вміст                                                                                       |
| --------------------- | ---------------------------------------------------------------------------------------------- |
| `manifest.json`       | Схема пакета, вихідні файли, кількість подій і згенерований список файлів                             |
| `events.jsonl`        | Впорядкована часова шкала runtime і transcript                                                        |
| `session-branch.json` | Редагована активна гілка transcript і заголовок сесії                                           |
| `metadata.json`       | Версія OpenClaw, OS/runtime, модель, знімок config, плагіни, Skills і метадані prompt     |
| `artifacts.json`      | Фінальний статус, помилки, використання, prompt cache, кількість compaction, текст асистента та метадані інструментів |
| `prompts.json`        | Надіслані prompts і вибрані деталі побудови prompt                                         |
| `system-prompt.txt`   | Останній скомпільований системний prompt, якщо захоплено                                                   |
| `tools.json`          | Визначення інструментів, надіслані моделі, якщо захоплено                                              |

`manifest.json` перелічує файли, наявні в цьому пакеті. Деякі файли пропускаються,
коли сесія не захопила відповідні runtime-дані.

## Розташування захоплення

За замовчуванням runtime-події траєкторії записуються поруч із файлом сесії:

```text
<session>.trajectory.jsonl
```

OpenClaw також записує best-effort файл-вказівник поруч із сесією:

```text
<session>.trajectory-path.json
```

Установіть `OPENCLAW_TRAJECTORY_DIR`, щоб зберігати runtime sidecars траєкторії у
виділеному каталозі:

```bash
export OPENCLAW_TRAJECTORY_DIR=/var/lib/openclaw/trajectories
```

Коли цю змінну встановлено, OpenClaw записує один JSONL-файл на id сесії в цьому
каталозі.

Обслуговування сесій видаляє sidecars траєкторії, коли їхній запис сесії-власника
обрізається, обмежується або витісняється дисковим бюджетом сесій. Runtime-файли поза
каталогом сесій видаляються лише тоді, коли ціль вказівника все ще доводить, що
належить цій сесії.

## Вимкнення захоплення

Установіть `OPENCLAW_TRAJECTORY=0` перед запуском OpenClaw:

```bash
export OPENCLAW_TRAJECTORY=0
```

Це вимикає runtime-захоплення траєкторії. `/export-trajectory` усе ще може експортувати
гілку transcript, але runtime-only файли, як-от скомпільований context,
provider artifacts і метадані prompt, можуть бути відсутні.

## Налаштування тайм-ауту flush

OpenClaw виконує flush runtime sidecars траєкторії під час cleanup агента. Стандартний
тайм-аут cleanup становить 10 000 мс. На повільних дисках або у великих сховищах установіть
`OPENCLAW_TRAJECTORY_FLUSH_TIMEOUT_MS` перед запуском OpenClaw:

```bash
export OPENCLAW_TRAJECTORY_FLUSH_TIMEOUT_MS=30000
```

Це керує тим, коли OpenClaw логує тайм-аут `openclaw-trajectory-flush` і продовжує.
Це не змінює обмеження розміру траєкторії. Щоб налаштувати всі кроки cleanup агента,
які не передають явний тайм-аут, установіть `OPENCLAW_AGENT_CLEANUP_TIMEOUT_MS`.

## Приватність і обмеження

Пакети траєкторії призначені для підтримки та налагодження, а не для публічної публікації.
OpenClaw редагує чутливі значення перед записом файлів експорту:

- credentials і відомі поля payload, схожі на секрети
- дані зображень
- локальні шляхи state
- шляхи workspace, замінені на `$WORKSPACE_DIR`
- шляхи домашнього каталогу, якщо виявлено

Експортер також обмежує розмір вхідних даних:

- runtime sidecar файли: live-захоплення зупиняється на 10 MiB і записує подію truncation, коли місце залишається; експорт приймає наявні runtime sidecars до 50 MiB
- файли сесій: 50 MiB
- runtime-події: 200 000
- загальна кількість експортованих подій: 250 000
- окремі рядки runtime-подій обрізаються понад 256 KiB

Переглядайте пакети перед тим, як ділитися ними поза своєю командою. Редагування є best-effort
і не може знати кожен секрет, специфічний для застосунку.

## Усунення несправностей

Якщо експорт не має runtime-подій:

- підтвердьте, що OpenClaw було запущено без `OPENCLAW_TRAJECTORY=0`
- перевірте, чи `OPENCLAW_TRAJECTORY_DIR` вказує на доступний для запису каталог
- запустіть ще одне повідомлення в сесії, потім експортуйте знову
- перевірте `manifest.json` на `runtimeEventCount`

Якщо команда відхиляє вихідний шлях:

- використовуйте відносну назву, наприклад `bug-1234`
- не передавайте `/tmp/...` або `~/...`
- тримайте експорт усередині `.openclaw/trajectory-exports/`

Якщо експорт завершується помилкою розміру, сесія або sidecar перевищили
безпечні обмеження експорту. Почніть нову сесію або експортуйте менше відтворення.
