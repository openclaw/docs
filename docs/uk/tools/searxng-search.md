OpenClaw підтримує [SearXNG](https://docs.searxng.org/) як **самостійно розгорнутого
постачальника `web_search` без ключа**. SearXNG — це пошуковий метарушій із відкритим кодом,
який агрегує результати з Google, Bing, DuckDuckGo та інших джерел.

Переваги:

- **Безкоштовно й без обмежень** -- ключ API або комерційна підписка не потрібні
- **Приватність / ізоляція** -- запити ніколи не залишають вашу мережу
- **Працює будь-де** -- немає регіональних обмежень комерційних пошукових API

## Налаштування

<Steps>
  <Step title="Запустіть інстанс SearXNG">
    ```bash
    docker run -d -p 8888:8080 searxng/searxng
    ```

    Або використайте будь-яке наявне розгортання SearXNG, до якого маєте доступ. Див.
    [документацію SearXNG](https://docs.searxng.org/) щодо налаштування для продакшну.

  </Step>
  <Step title="Налаштуйте">
    ```bash
    openclaw configure --section web
    # Select "searxng" as the provider
    ```

    Або задайте змінну середовища й дозвольте автовиявленню знайти її:

    ```bash
    export SEARXNG_BASE_URL="http://localhost:8888"
    ```

  </Step>
</Steps>

## Конфігурація

```json5
{
  tools: {
    web: {
      search: {
        provider: "searxng",
      },
    },
  },
}
```

Налаштування рівня Plugin для інстансу SearXNG:

```json5
{
  plugins: {
    entries: {
      searxng: {
        config: {
          webSearch: {
            baseUrl: "http://localhost:8888",
            categories: "general,news", // optional
            language: "en", // optional
          },
        },
      },
    },
  },
}
```

Поле `baseUrl` також приймає об’єкти SecretRef.

Правила транспорту:

- `https://` працює для публічних або приватних хостів SearXNG
- `http://` приймається лише для довірених хостів приватної мережі або loopback-хостів
- публічні хости SearXNG мають використовувати `https://`
- приватні/внутрішні хости використовують мережевий захист для самостійно розгорнутих інстансів; публічні хости `https://`
  залишаються на строгому захисті вебпошуку й не можуть перенаправляти на приватні
  адреси

## Змінна середовища

Задайте `SEARXNG_BASE_URL` як альтернативу конфігурації:

```bash
export SEARXNG_BASE_URL="http://localhost:8888"
```

Коли `SEARXNG_BASE_URL` задано й явного постачальника не налаштовано, автовиявлення
автоматично вибирає SearXNG (з найнижчим пріоритетом -- будь-який постачальник на основі API з
ключем перемагає першим).

## Довідка з конфігурації Plugin

| Поле         | Опис                                                               |
| ------------ | ------------------------------------------------------------------ |
| `baseUrl`    | Базовий URL вашого інстансу SearXNG (обов’язково)                  |
| `categories` | Категорії через кому, як-от `general`, `news` або `science`        |
| `language`   | Код мови для результатів, як-от `en`, `de` або `fr`                |

## Нотатки

- **JSON API** -- використовує нативну кінцеву точку SearXNG `format=json`, а не HTML-скрейпінг
- **Без ключа API** -- працює з будь-яким інстансом SearXNG одразу після налаштування
- **Перевірка базового URL** -- `baseUrl` має бути чинним URL `http://` або `https://`;
  публічні хости мають використовувати `https://`
- **Мережевий захист** -- приватні/внутрішні кінцеві точки SearXNG явно вмикають
  доступ до приватної мережі; публічні кінцеві точки SearXNG `https://` зберігають строгий
  захист від SSRF
- **Порядок автовиявлення** -- SearXNG перевіряється останнім (порядок 200) під час
  автовиявлення. Постачальники на основі API з налаштованими ключами запускаються першими, потім
  DuckDuckGo (порядок 100), потім Ollama Web Search (порядок 110)
- **Самостійно розгорнуто** -- ви контролюєте інстанс, запити та вихідні пошукові рушії
- **Категорії** за замовчуванням мають значення `general`, якщо їх не налаштовано

<Tip>
  Щоб JSON API SearXNG працював, переконайтеся, що у вашому інстансі SearXNG увімкнено формат `json`
  у `settings.yml` в розділі `search.formats`.
</Tip>
