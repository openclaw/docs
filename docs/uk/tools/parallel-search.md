Plugin Parallel надає два провайдери `web_search` від [Parallel](https://parallel.ai/), обидва з яких повертають ранжовані, оптимізовані для LLM уривки з вебіндексу, створеного для ШІ-агентів:

| Провайдер                  | id              | Автентифікація                                                                                          |
| -------------------------- | --------------- | ------------------------------------------------------------------------------------------------------- |
| Parallel Search (безплатний) | `parallel-free` | Не потрібна — безплатний [Search MCP](https://docs.parallel.ai/integrations/mcp/search-mcp) від Parallel |
| Parallel Search            | `parallel`      | `PARALLEL_API_KEY` — платний Search API, вищі обмеження частоти запитів і налаштування цільової функції  |

Щоб явно вибрати один із них, установіть для `tools.web.search.provider` значення `parallel-free` або `parallel`; жоден із них не визначається автоматично.

<Note>
  Моделі, що безпосередньо використовують OpenAI Responses (`api: "openai-responses"`, провайдер
  `openai`, офіційна базова URL-адреса API), автоматично використовують вбудований вебпошук,
  розміщений на інфраструктурі OpenAI, коли `tools.web.search.provider` не задано, має порожнє
  значення, `"auto"` або `"openai"`, тому за замовчуванням вони оминають Parallel. Щоб натомість
  спрямовувати їх через Parallel, установіть для `tools.web.search.provider` значення
  `parallel-free` або `parallel`. Див. [огляд вебпошуку](/uk/tools/web).
</Note>

## Установлення Plugin

```bash
openclaw plugins install @openclaw/parallel-plugin
openclaw gateway restart
```

## Ключ API (платний провайдер)

Для `parallel-free` ключ не потрібен, але цей провайдер усе одно необхідно вибрати явно. Для платного провайдера `parallel` потрібен ключ API:

<Steps>
  <Step title="Створення облікового запису">
    Зареєструйтеся на [platform.parallel.ai](https://platform.parallel.ai) і
    згенеруйте ключ API на панелі керування.
  </Step>
  <Step title="Збереження ключа">
    Установіть `PARALLEL_API_KEY` у середовищі Gateway або налаштуйте за допомогою:

    ```bash
    openclaw configure --section web
    ```

  </Step>
</Steps>

## Конфігурація

```json5
{
  plugins: {
    entries: {
      parallel: {
        config: {
          webSearch: {
            apiKey: "par-...", // необов’язково, якщо задано PARALLEL_API_KEY
            baseUrl: "https://api.parallel.ai", // необов’язково; OpenClaw додає /v1/search
          },
        },
      },
    },
  },
  tools: {
    web: {
      search: {
        // "parallel-free" для безплатного Search MCP або "parallel" для
        // показаного тут платного провайдера на основі API.
        provider: "parallel",
      },
    },
  },
}
```

**Альтернатива через змінну середовища:** установіть `PARALLEL_API_KEY` у середовищі Gateway. Для інсталяції Gateway додайте її до `~/.openclaw/.env`.

## Перевизначення базової URL-адреси

Застосовується лише до платного провайдера `parallel`; `parallel-free` завжди використовує `https://search.parallel.ai/mcp` та ігнорує це налаштування.

Установіть `plugins.entries.parallel.config.webSearch.baseUrl`, щоб спрямовувати платні запити через сумісний проксі-сервер або альтернативну кінцеву точку (наприклад, Cloudflare AI Gateway). OpenClaw нормалізує адреси хостів без схеми, додаючи на початку `https://`, і додає `/v1/search`, якщо шлях ще не закінчується цим фрагментом. Визначена кінцева точка є частиною ключа кешу пошуку, тому результати з різних кінцевих точок ніколи не використовуються спільно.

## Параметри інструмента

Обидва провайдери надають власну структуру пошуку Parallel, щоб модель заповнювала ціль природною мовою та кілька коротких запитів за ключовими словами — це поєднання Parallel [рекомендує](https://docs.parallel.ai/search/best-practices) для отримання найкращих результатів.

<ParamField path="objective" type="string" required>
Опис основного питання або цілі природною мовою (не більше ніж 5000 символів). Має бути самодостатнім.
</ParamField>

<ParamField path="search_queries" type="string[]" required>
Стислі пошукові запити за ключовими словами, по 3–6 слів кожен (1–5 елементів, не більше ніж 200 символів кожен). Для найкращих результатів надайте 2–3 різнопланові запити.
</ParamField>

<ParamField path="count" type="number">
Кількість результатів для повернення (1–40).
</ParamField>

<ParamField path="session_id" type="string">
Необов’язковий ідентифікатор сеансу Parallel зі значення `sessionId` попереднього результату. Передавайте його в наступних пошуках у межах того самого завдання, щоб Parallel групував пов’язані виклики та покращував подальші результати. Максимум 1000 символів для `parallel`; безплатний Search MCP `parallel-free` обмежує його до 100. Ідентифікатор, що перевищує обмеження, відкидається (платний провайдер) або замінюється новим (безплатний).
</ParamField>

<ParamField path="client_model" type="string">
Необов’язковий ідентифікатор моделі, яка виконує виклик (наприклад, `claude-opus-4-7`, `gpt-5.6-sol`), не більше ніж 100 символів. Дає змогу Parallel адаптувати стандартні налаштування до можливостей вашої моделі. Передавайте точний slug активної моделі; не скорочуйте його до псевдоніма сімейства.
</ParamField>

## Примітки

- Parallel ранжує та стискає результати для зручності міркувань LLM, а не для переходів користувачів за посиланнями; очікуйте щільних уривків для кожного результату, а не вмісту цілих сторінок.
- Уривки результатів повертаються як масив `excerpts`, а також об’єднуються в `description` для сумісності із загальним контрактом `web_search`.
- Обидва провайдери повертають `session_id`; OpenClaw надає його як `sessionId` у корисному навантаженні інструмента, щоб виклики могли групувати наступні пошуки. Згенерований Parallel ідентифікатор сеансу (який не був наданий викликом) не включається до запису кешу, оскільки непов’язані завдання з однаковими запитами не повинні його успадковувати.
- Значення `searchId`, `warnings` та `usage` від Parallel передаються без змін, якщо вони наявні.
- OpenClaw завжди передає Parallel визначену кількість результатів як `advanced_settings.max_results` (`parallel`) або застосовує `count` на боці клієнта після відповіді Parallel із фіксованою кількістю результатів (`parallel-free`). Аргумент `count` виклику має найвищий пріоритет, потім `tools.web.search.maxResults`, а інакше використовується загальне стандартне значення `web_search` у OpenClaw (5); власне стандартне значення API Parallel дорівнює 10.
- За замовчуванням результати кешуються на 15 хвилин (`cacheTtlMinutes`).
- `parallel-free` створює новий `session_id` для кожного виклику через узгодження MCP, коли виклик не надає його; `parallel` у такому разі залишає його незаданим.
