`tokenjuice` — це необов’язковий зовнішній Plugin, який стискає зашумлені результати інструментів `exec` і `bash`
після виконання команди.

Він змінює повернений `tool_result`, а не саму команду. Tokenjuice не
переписує введення оболонки, не запускає команди повторно й не змінює коди завершення.

Наразі це стосується вбудованих запусків OpenClaw і динамічних інструментів OpenClaw у
середовищі app-server Codex. Tokenjuice підключається до проміжного ПЗ OpenClaw для обробки результатів інструментів і
скорочує виведення, перш ніж воно повернеться до активного сеансу середовища.

## Увімкнення Plugin

Установіть один раз:

```bash
openclaw plugins install clawhub:@openclaw/tokenjuice
```

Потім увімкніть його:

```bash
openclaw config set plugins.entries.tokenjuice.enabled true
```

Еквівалентна команда:

```bash
openclaw plugins enable tokenjuice
```

Якщо ви віддаєте перевагу безпосередньому редагуванню конфігурації:

```json5
{
  plugins: {
    entries: {
      tokenjuice: {
        enabled: true,
      },
    },
  },
}
```

## Що змінює tokenjuice

- Стискає зашумлені результати `exec` і `bash`, перш ніж вони повернуться до сеансу.
- Не змінює початкове виконання команди.
- Застосовує політику безпечної інвентаризації: точне читання вмісту файлів залишається без змін, окремі команди інвентаризації репозиторію можна стискати, а небезпечні змішані послідовності команд залишаються без змін.
- Залишається функцією, яку потрібно вмикати явно: вимкніть Plugin, якщо вам усюди потрібне дослівне виведення.

## Перевірка роботи

1. Увімкніть Plugin.
2. Запустіть сеанс, який може викликати `exec`.
3. Виконайте зашумлену команду, наприклад `git status`.
4. Переконайтеся, що повернений результат інструмента коротший і структурованіший за необроблене виведення оболонки.

## Вимкнення Plugin

```bash
openclaw config set plugins.entries.tokenjuice.enabled false
```

Або:

```bash
openclaw plugins disable tokenjuice
```
