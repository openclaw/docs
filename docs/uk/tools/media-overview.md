# Генерація та розуміння медіа

OpenClaw генерує зображення, відео та музику, розуміє вхідні медіа (зображення, аудіо, відео) і озвучує відповіді за допомогою перетворення тексту на мовлення. Усі медіаможливості керуються інструментами: агент вирішує, коли їх використовувати, на основі розмови, і кожен інструмент з’являється лише тоді, коли налаштовано принаймні одного базового постачальника.

## Як працює асинхронна генерація

Генерація відео та музики виконується як фонові завдання, оскільки обробка у постачальників зазвичай триває від 30 секунд до кількох хвилин. Коли агент викликає `video_generate` або `music_generate`, OpenClaw надсилає запит постачальнику, одразу повертає ідентифікатор завдання та відстежує роботу в журналі завдань. Поки завдання виконується, агент продовжує відповідати на інші повідомлення. Коли постачальник завершує обробку, OpenClaw пробуджує агента, щоб той міг опублікувати готовий медіарезультат назад у вихідний канал. Генерація зображень і TTS є синхронними та завершуються безпосередньо під час відповіді.

Deepgram, ElevenLabs, Mistral, OpenAI і xAI можуть транскрибувати вхідне
аудіо через пакетний шлях `tools.media.audio`, якщо їх налаштовано. Deepgram,
ElevenLabs, Mistral, OpenAI і xAI також реєструють постачальників потокового STT для
Voice Call, тож живе телефонне аудіо може бути переспрямоване до вибраного постачальника
без очікування завершення запису.

Google відповідає поверхням OpenClaw для зображень, відео, музики, пакетного TTS, backend realtime
voice і розуміння медіа. OpenAI відповідає поверхням OpenClaw для зображень,
відео, пакетного TTS, пакетного STT, потокового STT для Voice Call, backend realtime voice
та вбудовувань пам’яті. xAI наразі відповідає поверхням OpenClaw для зображень, відео,
пошуку, виконання коду, пакетного TTS, пакетного STT і потокового STT для Voice Call.
Голос xAI Realtime є можливістю на стороні постачальника, але він не
реєструється в OpenClaw, доки спільний контракт голосу в реальному часі не зможе це виразити.
