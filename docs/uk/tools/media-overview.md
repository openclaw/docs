# Генерація та розуміння медіа

OpenClaw генерує зображення, відео та музику, розуміє вхідні медіа (зображення, аудіо, відео) і озвучує відповіді за допомогою перетворення тексту на мовлення. Усі медіаможливості керуються інструментами: агент вирішує, коли їх використовувати, на основі розмови, і кожен інструмент з’являється лише тоді, коли налаштовано принаймні одного відповідного постачальника.

| -------------------- | ---------------- | -------------------------------------------------------------------------------------------- | ------------------------------------------------------- |
| Генерація зображень  | `image_generate` | ComfyUI, fal, Google, MiniMax, OpenAI, Vydra, xAI                                            | Створює або редагує зображення з текстових запитів або референсів |
| Генерація відео      | `video_generate` | Alibaba, BytePlus, ComfyUI, fal, Google, MiniMax, OpenAI, Qwen, Runway, Together, Vydra, xAI | Створює відео з тексту, зображень або наявних відео     |
| Генерація музики     | `music_generate` | ComfyUI, Google, MiniMax                                                                     | Створює музику або аудіодоріжки з текстових запитів     |

## Як працює асинхронна генерація

Генерація відео та музики виконується як фонові завдання, оскільки обробка у постачальника зазвичай триває від 30 секунд до кількох хвилин. Коли агент викликає `video_generate` або `music_generate`, OpenClaw надсилає запит постачальнику, одразу повертає ідентифікатор завдання та відстежує завдання в журналі завдань. Поки завдання виконується, агент продовжує відповідати на інші повідомлення. Коли постачальник завершує обробку, OpenClaw активує агента, щоб він міг опублікувати готовий медіафайл назад у вихідний канал. Генерація зображень і TTS є синхронними та завершуються в межах відповіді.

Deepgram, ElevenLabs, Mistral, OpenAI та xAI можуть транскрибувати вхідне
аудіо через пакетний шлях `tools.media.audio`, якщо їх налаштовано. Deepgram,
ElevenLabs, Mistral, OpenAI та xAI також реєструють постачальників потокового STT
для Voice Call, тож живе телефонне аудіо можна пересилати вибраному постачальнику
без очікування завершення запису.

OpenAI відповідає поверхням OpenClaw для зображень, відео, пакетного TTS, пакетного STT, потокового STT для Voice Call, голосу в реальному часі та вбудовування пам’яті. xAI наразі відповідає поверхням OpenClaw для зображень, відео, пошуку, виконання коду, пакетного TTS, пакетного STT
і потокового STT для Voice Call. Голос xAI Realtime є можливістю на стороні
апстріму, але не реєструється в OpenClaw, доки спільний контракт голосу в реальному
часі не зможе її представити.
