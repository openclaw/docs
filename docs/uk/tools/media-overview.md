# Генерація та розуміння медіа

OpenClaw генерує зображення, відео й музику, розуміє вхідні медіа (зображення, аудіо, відео) і озвучує відповіді за допомогою text-to-speech. Усі медіаможливості керуються інструментами: агент сам вирішує, коли їх використовувати залежно від розмови, і кожен інструмент з’являється лише тоді, коли налаштовано принаймні одного базового провайдера.

## Можливості з першого погляду

| Можливість           | Інструмент       | Провайдери                                                                                   | Що робить                                              |
| -------------------- | ---------------- | -------------------------------------------------------------------------------------------- | ------------------------------------------------------ |
| Генерація зображень  | `image_generate` | ComfyUI, fal, Google, MiniMax, OpenAI, Vydra, xAI                                            | Створює або редагує зображення з текстових prompt або референсів |
| Генерація відео      | `video_generate` | Alibaba, BytePlus, ComfyUI, fal, Google, MiniMax, OpenAI, Qwen, Runway, Together, Vydra, xAI | Створює відео з тексту, зображень або наявних відео    |
| Генерація музики     | `music_generate` | ComfyUI, Google, MiniMax                                                                     | Створює музику або аудіодоріжки з текстових prompt     |
| Text-to-speech (TTS) | `tts`            | ElevenLabs, Google, Gradium, Microsoft, MiniMax, OpenAI, Vydra, xAI                          | Перетворює вихідні відповіді на озвучене аудіо         |

## Як працює асинхронна генерація

Генерація відео й музики виконується як фонові завдання, оскільки обробка у провайдера зазвичай займає від 30 секунд до кількох хвилин. Коли агент викликає `video_generate` або `music_generate`, OpenClaw надсилає запит провайдеру, одразу повертає ID завдання і відстежує роботу в ledger завдань. Агент продовжує відповідати на інші повідомлення, поки виконується завдання. Коли провайдер завершує роботу, OpenClaw пробуджує агента, щоб він міг опублікувати готове медіа назад в оригінальний канал. Генерація зображень і TTS є синхронними та завершуються inline разом із відповіддю.

Deepgram, ElevenLabs, Mistral, OpenAI і xAI можуть транскрибувати вхідне
аудіо через шлях batch `tools.media.audio`, якщо їх налаштовано. Deepgram,
ElevenLabs, Mistral, OpenAI і xAI також реєструють провайдерів streaming STT для Voice Call, тож live phone audio можна пересилати вибраному постачальнику
без очікування завершення запису.

Google зіставляється з поверхнями OpenClaw для зображень, відео, музики, batch TTS, backend realtime
voice і розуміння медіа. OpenAI зіставляється з поверхнями OpenClaw для зображень,
відео, batch TTS, batch STT, streaming STT для Voice Call, backend realtime voice
і memory embedding. xAI наразі зіставляється з поверхнями OpenClaw для зображень, відео,
пошуку, виконання коду, batch TTS, batch STT і streaming STT для Voice Call.
Голос xAI Realtime є можливістю upstream, але він не
зареєстрований в OpenClaw, доки спільний контракт realtime voice не зможе його представити.
