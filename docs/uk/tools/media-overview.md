# Генерація та розуміння медіа

OpenClaw генерує зображення, відео й музику, розуміє вхідні медіа (зображення, аудіо, відео) і озвучує відповіді за допомогою перетворення тексту на мовлення. Усі медіаможливості керуються інструментами: агент вирішує, коли їх використовувати, залежно від розмови, і кожен інструмент з’являється лише тоді, коли налаштовано принаймні одного провайдера, який його підтримує.

## Можливості з першого погляду

| Capability           | Tool             | Providers                                                                                     | What it does                                              |
| -------------------- | ---------------- | --------------------------------------------------------------------------------------------- | --------------------------------------------------------- |
| Генерація зображень  | `image_generate` | ComfyUI, fal, Google, MiniMax, OpenAI, Vydra, xAI                                             | Створює або редагує зображення з текстових prompt чи reference |
| Генерація відео      | `video_generate` | Alibaba, BytePlus, ComfyUI, fal, Google, MiniMax, OpenAI, Qwen, Runway, Together, Vydra, xAI | Створює відео з тексту, зображень або наявних відео       |
| Генерація музики     | `music_generate` | ComfyUI, Google, MiniMax                                                                      | Створює музику або аудіодоріжки з текстових prompt        |
| Перетворення тексту на мовлення (TTS) | `tts`            | ElevenLabs, Microsoft, MiniMax, OpenAI, xAI                                                   | Перетворює вихідні відповіді на озвучене аудіо            |
| Розуміння медіа      | (автоматично)    | Будь-який провайдер моделей із підтримкою vision/audio, а також CLI fallback                  | Узагальнює вхідні зображення, аудіо та відео              |

## Як працює асинхронна генерація

Генерація відео та музики виконується як фонові завдання, оскільки обробка провайдерами зазвичай триває від 30 секунд до кількох хвилин. Коли агент викликає `video_generate` або `music_generate`, OpenClaw надсилає запит провайдеру, одразу повертає ID завдання та відстежує його в журналі завдань. Поки завдання виконується, агент продовжує відповідати на інші повідомлення. Коли провайдер завершує обробку, OpenClaw пробуджує агента, щоб той міг опублікувати готове медіа назад у початковий канал. Генерація зображень і TTS є синхронними й завершуються inline разом із відповіддю.

Deepgram, ElevenLabs, Mistral, OpenAI і xAI можуть транскрибувати вхідне
аудіо через пакетний шлях `tools.media.audio`, якщо їх налаштовано. Deepgram,
ElevenLabs, Mistral, OpenAI і xAI також реєструють провайдерів потокового STT для Voice Call,
тому live-аудіо телефонних дзвінків можна пересилати до вибраного вендора
без очікування на завершений запис.

OpenAI зіставляється в OpenClaw з поверхнями генерації зображень, відео, пакетного TTS, пакетного STT, потокового STT Voice Call, realtime voice і memory embedding. xAI наразі
зіставляється в OpenClaw з поверхнями зображень, відео, пошуку, виконання коду, пакетного TTS, пакетного STT
і потокового STT Voice Call. Realtime voice у xAI є
можливістю на стороні upstream, але не реєструється в OpenClaw, доки спільний контракт
realtime voice не зможе її представити.
