## Ручний вхід (рекомендовано)

Коли сайт вимагає входу, увійдіть вручну в профілі `openclaw` браузера хоста. Не надавайте моделі свої облікові дані: автоматизовані спроби входу часто активують захист від ботів і можуть призвести до блокування облікового запису.

Використовуйте браузер хоста (з ручним входом) як для читання (пошуку/гілок обговорень), так і для публікації дописів у X/Twitter та на інших сайтах із посиленим захистом від ботів. Сеанси браузера в пісочниці частіше активують виявлення ботів.

Повернутися до основної документації браузера: [Браузер](/uk/tools/browser).

## Який профіль Chrome використовується?

OpenClaw керує окремим профілем Chrome із назвою `openclaw` (з інтерфейсом у помаранчевих тонах), відокремленим від вашого повсякденного профілю браузера.

Для викликів інструмента браузера агентом:

- Типовий вибір: агент використовує свій ізольований браузер `openclaw`.
- Використовуйте `profile="user"` лише тоді, коли важливі наявні активні сеанси входу й ви перебуваєте за комп’ютером, щоб натиснути або підтвердити будь-який запит на підключення.
- Якщо у вас кілька профілів користувацького браузера, вкажіть профіль явно, а не намагайтеся вгадати.

Існує два способи отримати доступ до профілю `openclaw`:

1. Попросіть агента відкрити браузер, а потім увійдіть самостійно.
2. Відкрийте його через CLI:

```bash
openclaw browser start
openclaw browser open https://x.com
```

Для профілю, відмінного від типового, розмістіть `--browser-profile <name>` перед підкомандою (типовим є `openclaw`):

```bash
openclaw browser --browser-profile <name> open https://x.com
```

## Пісочниця: дозвіл доступу до браузера хоста

Якщо агент працює в пісочниці, його виклики інструмента `browser` типово спрямовуються до браузера в пісочниці, а не на хості. Щоб натомість дозволити агенту звертатися до браузера хоста:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

Виклики CLI завжди спрямовуються до браузера хоста, а не до пісочниці, тому ви можете самостійно відкрити браузер хоста незалежно від цього налаштування:

```bash
openclaw browser --browser-profile openclaw open https://x.com
```

Після встановлення `sandbox.browser.allowHostControl: true` виклики інструмента `browser` агентом також можуть спрямовуватися до хоста. Як альтернативу, вимкніть пісочницю для агента, який публікує оновлення.
