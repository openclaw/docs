Gateway обробляє команди, надіслані як окремі повідомлення, що починаються з `/`.
Bash-команди лише для хоста використовують `! <cmd>` (із `/bash <cmd>` як псевдонімом).

Коли розмову прив’язано до сеансу ACP, звичайний текст спрямовується до
середовища ACP. Команди керування Gateway залишаються локальними: `/acp ...` завжди надходить
до обробника команд OpenClaw, а `/status` і `/unfocus` залишаються локальними, коли
обробку команд увімкнено для цієї поверхні.

## Три типи команд

<CardGroup cols={3}>
  <Card title="Команди" icon="terminal">
    Окремі повідомлення `/...`, які обробляє Gateway. Вони мають бути
    єдиним вмістом повідомлення.
  </Card>
  <Card title="Директиви" icon="sliders">
    `/think`, `/fast`, `/verbose`, `/trace`, `/reasoning`, `/elevated`,
    `/exec`, `/model`, `/queue` — вилучаються з повідомлення до того, як його
    побачить модель. Зберігають налаштування сеансу, якщо надіслані окремо; діють як вбудовані підказки,
    якщо надіслані з іншим текстом.
  </Card>
  <Card title="Вбудовані скорочення" icon="bolt">
    `/help`, `/commands`, `/status`, `/whoami` — виконуються негайно та
    вилучаються до того, як модель побачить решту тексту. Лише для авторизованих відправників.
  </Card>
</CardGroup>

<AccordionGroup>
  <Accordion title="Докладно про поведінку директив">
    - Директиви вилучаються з повідомлення до того, як його побачить модель.
    - У повідомленнях, що містять **лише директиви** (повідомлення складається лише з директив), вони
      зберігаються в сеансі, а у відповідь надходить підтвердження.
    - У повідомленнях **звичайного чату** з іншим текстом вони діють як вбудовані підказки та
      **не** зберігають налаштування сеансу.
    - Директиви застосовуються лише для **авторизованих відправників**. Якщо встановлено `commands.allowFrom`,
      використовується лише цей список дозволених користувачів; інакше авторизація визначається
      списками дозволених користувачів каналу/спарюванням разом із `commands.useAccessGroups`. Для неавторизованих
      відправників директиви обробляються як звичайний текст.
  </Accordion>
</AccordionGroup>

## Конфігурація

```json5
{
  commands: {
    native: "auto",
    nativeSkills: "auto",
    text: true,
    bash: false,
    bashForegroundMs: 2000,
    config: false,
    mcp: false,
    plugins: false,
    debug: false,
    restart: true,
    ownerAllowFrom: ["discord:123456789012345678"],
    ownerDisplay: "raw",
    ownerDisplaySecret: "${OWNER_ID_HASH_SECRET}",
    allowFrom: {
      "*": ["user1"],
      discord: ["user:123"],
    },
    useAccessGroups: true,
  },
}
```

<ParamField path="commands.text" type="boolean" default="true">
  Вмикає розбір `/...` у повідомленнях чату. На поверхнях без нативних команд
  (WhatsApp, WebChat, Signal, iMessage, Google Chat, Microsoft Teams) текстові
  команди працюють, навіть якщо встановлено значення `false`.
</ParamField>

<ParamField path="commands.native" type='boolean | "auto"' default='"auto"'>
  Реєструє нативні команди. Автоматичний режим: увімкнено для Discord/Telegram; вимкнено для Slack;
  ігнорується для постачальників без нативної підтримки. Перевизначається для окремого каналу за допомогою
  `channels.<provider>.commands.native`. У Discord значення `false` пропускає реєстрацію
  команд із косою рискою; раніше зареєстровані команди можуть залишатися видимими до видалення.
</ParamField>

<ParamField path="commands.nativeSkills" type='boolean | "auto"' default='"auto"'>
  Реєструє команди Skills як нативні, якщо це підтримується. Автоматичний режим: увімкнено для
  Discord/Telegram; вимкнено для Slack. Перевизначається за допомогою
  `channels.<provider>.commands.nativeSkills`.
</ParamField>

<ParamField path="commands.bash" type="boolean" default="false">
  Дозволяє `! <cmd>` запускати команди оболонки хоста (псевдонім `/bash <cmd>`). Потребує
  списків дозволених користувачів `tools.elevated`.
</ParamField>

<ParamField path="commands.bashForegroundMs" type="number" default="2000">
  Час очікування bash перед переходом у фоновий режим (`0` означає
  негайний перехід у фоновий режим).
</ParamField>

<ParamField path="commands.config" type="boolean" default="false">
  Вмикає `/config` (читання/запис `openclaw.json`). Лише для власника.
</ParamField>

<ParamField path="commands.mcp" type="boolean" default="false">
  Вмикає `/mcp` (читання/запис керованої OpenClaw конфігурації MCP у `mcp.servers`). Лише для власника.
</ParamField>

<ParamField path="commands.plugins" type="boolean" default="false">
  Вмикає `/plugins` (виявлення/стан плагінів, а також встановлення й увімкнення/вимкнення). Запис доступний лише власнику.
</ParamField>

<ParamField path="commands.debug" type="boolean" default="false">
  Вмикає `/debug` (перевизначення конфігурації лише під час виконання). Лише для власника.
</ParamField>

<ParamField path="commands.restart" type="boolean" default="true">
  Вмикає `/restart` і дії інструментів для перезапуску Gateway.
</ParamField>

<ParamField path="commands.ownerAllowFrom" type="string[]">
  Явний список дозволених власників для поверхонь команд, доступних лише власнику. Окремий від
  `commands.allowFrom` і доступу через спарювання в особистих повідомленнях.
</ParamField>

<ParamField path="channels.<channel>.commands.enforceOwnerForCommands" type="boolean" default="false">
  Для кожного каналу: вимагає ідентичність власника для команд, доступних лише власнику. Якщо значення `true`,
  відправник має відповідати `commands.ownerAllowFrom` або мати внутрішню область дії `operator.admin`.
  Запис із символом узагальнення в `allowFrom` **не** є достатнім.
</ParamField>

<ParamField path="commands.ownerDisplay" type='"raw" | "hash"'>
  Керує тим, як ідентифікатори власників відображаються в системному запиті.
</ParamField>

<ParamField path="commands.ownerDisplaySecret" type="string">
  Секрет HMAC, що використовується, коли задано `commands.ownerDisplay: "hash"`.
</ParamField>

<ParamField path="commands.allowFrom" type="object">
  Окремий для кожного провайдера список дозволених джерел для авторизації команд. Якщо його налаштовано, він є
  **єдиним** джерелом авторизації для команд і директив. Використовуйте `"*"` як
  глобальне значення за замовчуванням; ключі конкретних провайдерів мають вищий пріоритет.
</ParamField>

<ParamField path="commands.useAccessGroups" type="boolean" default="true">
  Забезпечує дотримання списків дозволених джерел і політик для команд, коли `commands.allowFrom` не задано.
</ParamField>

## Список команд

Команди надходять із трьох джерел:

- **Вбудовані команди ядра:** `src/auto-reply/commands-registry.shared.ts`
- **Згенеровані команди панелі:** `src/auto-reply/commands-registry.data.ts`
- **Команди Plugin:** виклики `registerCommand()` у Plugin

Доступність залежить від прапорців конфігурації, можливостей каналу та встановлених і ввімкнених
Plugin.

### Команди стикування

Команди стикування перемикають маршрут відповіді активного сеансу на інший пов’язаний канал.
Відомості про налаштування та усунення несправностей див. у розділі [Стикування каналів](/uk/concepts/channel-docking).

Створюються з плагінів каналів, які підтримують нативні команди:

- `/dock-discord` (псевдонім: `/dock_discord`)
- `/dock-mattermost` (псевдонім: `/dock_mattermost`)
- `/dock-slack` (псевдонім: `/dock_slack`)
- `/dock-telegram` (псевдонім: `/dock_telegram`)

Для команд стикування потрібен параметр `session.identityLinks`. Відправник із вихідного каналу та цільовий співрозмовник
мають належати до однієї групи ідентичностей.

### Команди вбудованих плагінів

| Команда                                                 | Опис                                                                                                                                                                                    |
| ------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `/dreaming [on\|off\|status\|help]`                     | Увімкнути або вимкнути Dreaming для пам’яті (власник або адміністратор Gateway). Див. [Dreaming](/uk/concepts/dreaming)                                                                                                            |

| `/phone status\|arm ...\|disarm`                        | Тимчасово дозволити команди Node із високим ризиком (камера, екран, комп’ютер, операції запису). Див. [Використання комп’ютера](/uk/nodes/computer-use)                                                                               |
| `/voice status\|list\|set <voiceId>`                    | Керувати конфігурацією голосу Talk. Нативна назва в Discord: `/talkvoice`                                                                                                                                    |
| `/card ...`                                             | Надсилати шаблони розширених карток LINE. Див. [LINE](/uk/channels/line)                                                                                                                                        |
| `/codex <action> ...`                                   | Прив’язувати, спрямовувати й перевіряти обв’язку сервера застосунку Codex (стан, гілки, відновлення, модель, швидкий режим, дозволи, ущільнення, перевірка, mcp, Skills тощо). Див. [Обв’язка Codex](/uk/plugins/codex-harness) |

### Команди Skills

Skills, які може викликати користувач, доступні як команди з косою рискою:

- `/skill <name> [input]` завжди працює як універсальна точка входу.
- Skills можуть реєструватися як безпосередні команди (наприклад, `/prose` для OpenProse).
- Реєстрація нативних команд Skills керується параметрами `commands.nativeSkills` і
  `channels.<provider>.commands.nativeSkills`.
- Назви очищуються до `a-z0-9_` (щонайбільше 32 символи); у разі конфліктів додаються числові суфікси.

<AccordionGroup>
  <Accordion title="Маршрутизація команд Skills">
    Типово команди Skills спрямовуються до моделі як звичайний запит.

    Skills можуть оголосити `command-dispatch: tool`, щоб спрямовувати запит безпосередньо до інструмента
    (детерміновано, без участі моделі). Приклад: `/prose` (плагін OpenProse)
    — див. [OpenProse](/uk/prose).

  </Accordion>
  <Accordion title="Аргументи нативних команд">
    Discord використовує автодоповнення для динамічних параметрів і меню кнопок, коли обов’язкові
    аргументи не вказано. Telegram і Slack показують меню кнопок для команд із
    варіантами вибору. Динамічні варіанти визначаються за моделлю цільового сеансу, тому залежні від
    моделі параметри, як-от рівні `/think`, враховують перевизначення `/model` у сеансі.
  </Accordion>
</AccordionGroup>

## `/tools`: що агент може використовувати зараз

`/tools` відповідає на запитання про середовище виконання: **що цей агент може використовувати просто зараз у цій
розмові**, а не показує статичний каталог конфігурації.

```text
/tools         # компактне подання
/tools verbose # із короткими описами
```

Результати обмежені поточним сеансом. Зміна агента, каналу, гілки, авторизації
відправника або моделі може змінити результат. Щоб редагувати профілі та перевизначення,
скористайтеся панеллю інструментів у Control UI або засобами конфігурації.

## `/model`: вибір моделі

```text
/model             # показати засіб вибору моделі
/model list        # те саме
/model 3           # вибрати за номером із засобу вибору
/model openai/gpt-5.4
/model opus@anthropic:default
/model default     # очистити вибір моделі для сеансу
/model status      # докладне подання з кінцевою точкою та режимом API
```

У Discord команди `/model` і `/models` відкривають інтерактивний засіб вибору з розкривними списками
постачальників і моделей. Засіб вибору враховує `agents.defaults.models`, зокрема
записи `provider/*`.

## `/config`: запис конфігурації на диск

<Note>
  Лише для власника. Типово вимкнено — увімкніть за допомогою `commands.config: true`.
</Note>

```text
/config show
/config show messages.responsePrefix
/config get messages.responsePrefix
/config set messages.responsePrefix="[openclaw]"
/config unset messages.responsePrefix
```

Перед записом конфігурація перевіряється. Недійсні зміни відхиляються. Оновлення через `/config`
зберігаються після перезапусків.

## `/mcp`: конфігурація сервера MCP

<Note>
  Лише для власника. Типово вимкнено — увімкніть за допомогою `commands.mcp: true`.
</Note>

```text
/mcp show
/mcp show context7
/mcp set context7={"command":"uvx","args":["context7-mcp"]}
/mcp unset context7
```

`/mcp` зберігає конфігурацію в конфігурації OpenClaw, а не в налаштуваннях проєкту вбудованого агента.
`/mcp show` приховує поля з обліковими даними, значення розпізнаних прапорців облікових даних
і відомі аргументи, схожі на секрети. Якщо команду виконано з групи,
конфігурація надсилається власнику приватно; якщо приватний маршрут до власника
недоступний, команда безпечно завершується помилкою та просить власника повторити спробу в особистому
чаті.

## `/debug`: перевизначення лише для середовища виконання

<Note>
  Лише для власника. Типово вимкнено — увімкніть за допомогою `commands.debug: true`.
  Перевизначення негайно застосовуються до нових операцій читання конфігурації, але **не** записуються на диск.
</Note>

```text
/debug show
/debug set messages.responsePrefix="[openclaw]"
/debug set channels.whatsapp.allowFrom=["+1555","+4477"]
/debug unset messages.responsePrefix
/debug reset
```

## `/plugins`: керування плагінами

<Note>
  Запис — лише для власника. Типово вимкнено — увімкніть за допомогою `commands.plugins: true`.
</Note>

```text
/plugins
/plugins list
/plugin show context7
/plugins enable context7
/plugins disable context7
/plugins install ./path/to/plugin
```

`/plugins enable|disable` оновлює конфігурацію плагінів і виконує гаряче перезавантаження
середовища виконання плагінів Gateway для нових ходів агента. `/plugins install` автоматично
перезапускає керовані Gateway, оскільки змінилися модулі вихідного коду плагінів.

## `/trace`: виведення трасування плагінів

```text
/trace          # показати поточний стан трасування
/trace on
/trace off
```

`/trace` показує обмежені сеансом рядки трасування й налагодження плагінів без повного докладного
режиму. Він не замінює `/debug` (перевизначення середовища виконання) або `/verbose` (звичайне
виведення інструментів).

## `/btw`: побічні запитання

`/btw` — це швидке побічне запитання щодо контексту поточного сеансу. Псевдонім: `/side`.

```text
/btw що ми зараз робимо?
/side що змінилося, поки тривав основний запуск?
```

На відміну від звичайного повідомлення:

- Використовує поточний сеанс як фоновий контекст.
- У сеансах обв’язки Codex запускається як тимчасова побічна гілка Codex.
- **Не** змінює майбутній контекст сеансу.
- Не записується до історії стенограми.

Повний опис поведінки див. у розділі [Побічні запитання BTW](/uk/tools/btw).

## Примітки щодо поверхонь

<AccordionGroup>
  <Accordion title="Область сеансу для кожної поверхні">
    - **Текстові команди:** виконуються у звичайному сеансі чату (особисті повідомлення використовують спільний `main`, групи мають власний сеанс).
    - **Нативні команди Discord:** `agent:<agentId>:discord:slash:<userId>`
    - **Нативні команди Slack:** `agent:<agentId>:slack:slash:<userId>` (префікс налаштовується через `channels.slack.slashCommand.sessionPrefix`)
    - **Нативні команди Telegram:** `telegram:slash:<userId>` (цільовим є сеанс чату через `CommandTargetSessionKey`)
    - **`/login codex`** надсилає коди сполучення пристрою лише через особистий чат або канали відповіді Web UI. У разі виклику в групі або темі Telegram власнику пропонується натомість написати боту особисте повідомлення.
    - **`/stop`** спрямовується до активного сеансу чату, щоб перервати поточний запуск.

  </Accordion>
  <Accordion title="Особливості Slack">
    `channels.slack.slashCommand` підтримує одну команду у стилі `/openclaw`.
    Якщо задано `commands.native: true`, створіть окрему команду Slack із косою рискою для кожної вбудованої
    команди. Зареєструйте `/agentstatus` (не `/status`), оскільки Slack резервує
    `/status`. Текстова команда `/status` і надалі працює в повідомленнях Slack.
  </Accordion>
  <Accordion title="Швидкий шлях і вбудовані скорочення">
    - Повідомлення, що містять лише команди й надіслані відправниками зі списку дозволених, обробляються негайно (в обхід черги та моделі).
    - Вбудовані скорочення (`/help`, `/commands`, `/status`, `/whoami`) також працюють усередині звичайних повідомлень і вилучаються, перш ніж модель побачить решту тексту.
    - Неавторизовані повідомлення, що містять лише команди, мовчки ігноруються; вбудовані токени `/...` розглядаються як звичайний текст.

  </Accordion>
  <Accordion title="Примітки щодо аргументів">
    - Команди допускають необов’язковий символ `:` між командою та аргументами (`/think: high`, `/send: on`).
    - `/new <model>` приймає псевдонім моделі, `provider/model` або назву постачальника (нечітка відповідність); якщо відповідності немає, текст розглядається як тіло повідомлення.
    - Для `/allowlist add|remove` потрібно `commands.config: true`; команда враховує параметр каналу `configWrites`.

  </Accordion>
</AccordionGroup>

## Використання постачальника та стан

- **Використання/квота провайдера** (наприклад, «Claude: залишилося 80%») відображається в `/status` для провайдера поточної моделі, коли ввімкнено відстеження використання.
- **Рядки токенів/кешу** в `/status` можуть використовувати останній запис про використання з журналу сеансу, якщо поточний знімок сеансу містить недостатньо даних.
- **Виконання та середовище виконання:** `/status` показує `Execution` для фактичного шляху пісочниці та `Runtime` для виконавця сеансу: `OpenClaw Default`, `OpenAI Codex`, серверна частина CLI або серверна частина ACP.
- **Токени/вартість для кожної відповіді:** керуються командою `/usage off|tokens|full`.
- `/model status` стосується моделей, автентифікації та кінцевих точок, а не використання.
