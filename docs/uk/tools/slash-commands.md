Gateway обробляє команди, надіслані як окремі повідомлення, що починаються з `/`.
Команди bash лише для хоста використовують `! <cmd>` (із `/bash <cmd>` як псевдонімом).

Коли розмову прив’язано до сеансу ACP, звичайний текст спрямовується до
обв’язки ACP. Команди керування Gateway залишаються локальними: `/acp ...` завжди надходить
до обробника команд OpenClaw, а `/status` та `/unfocus` залишаються локальними, коли
для цієї поверхні ввімкнено обробку команд.

## Три типи команд

<CardGroup cols={3}>
  <Card title="Команди" icon="terminal">
    Окремі повідомлення `/...`, які обробляє Gateway. Вони мають бути
    єдиним вмістом повідомлення.
  </Card>
  <Card title="Директиви" icon="sliders">
    `/think`, `/fast`, `/verbose`, `/trace`, `/reasoning`, `/elevated`,
    `/exec`, `/model`, `/queue` — вилучаються з повідомлення до того, як його
    побачить модель. Якщо їх надіслано окремо, зберігають налаштування сеансу; якщо
    надіслано з іншим текстом, діють як вбудовані підказки.
  </Card>
  <Card title="Вбудовані скорочення" icon="bolt">
    `/help`, `/commands`, `/status`, `/whoami` — виконуються негайно та
    вилучаються до того, як модель побачить решту тексту. Лише для авторизованих відправників.
  </Card>
</CardGroup>

<AccordionGroup>
  <Accordion title="Докладно про поведінку директив">
    - Директиви вилучаються з повідомлення до того, як його побачить модель.
    - У повідомленнях **лише з директивами** (повідомлення містить тільки директиви) вони
      зберігаються в сеансі, а у відповідь надсилається підтвердження.
    - У повідомленнях **звичайного чату** з іншим текстом вони діють як вбудовані підказки й
      **не** зберігають налаштування сеансу.
    - Директиви застосовуються лише для **авторизованих відправників**. Якщо встановлено `commands.allowFrom`,
      використовується лише цей список дозволених; інакше авторизація визначається
      списками дозволених каналів/сполученням разом із `commands.useAccessGroups`. Для неавторизованих
      відправників директиви обробляються як звичайний текст.
  </Accordion>
</AccordionGroup>

## Конфігурація

```json5
{
  commands: {
    native: "auto",
    nativeSkills: "auto",
    text: true,
    bash: false,
    bashForegroundMs: 2000,
    config: false,
    mcp: false,
    plugins: false,
    debug: false,
    restart: true,
    ownerAllowFrom: ["discord:123456789012345678"],
    ownerDisplay: "raw",
    ownerDisplaySecret: "${OWNER_ID_HASH_SECRET}",
    allowFrom: {
      "*": ["user1"],
      discord: ["user:123"],
    },
    useAccessGroups: true,
  },
}
```

<ParamField path="commands.text" type="boolean" default="true">
  Вмикає розбір `/...` у повідомленнях чату. На поверхнях без нативних команд
  (WhatsApp, WebChat, Signal, iMessage, Google Chat, Microsoft Teams) текстові
  команди працюють, навіть якщо встановлено `false`.
</ParamField>

<ParamField path="commands.native" type='boolean | "auto"' default='"auto"'>
  Реєструє нативні команди. Автоматичний режим: увімкнено для Discord/Telegram; вимкнено для Slack;
  ігнорується для постачальників без нативної підтримки. Перевизначається для кожного каналу через
  `channels.<provider>.commands.native`. У Discord `false` пропускає реєстрацію
  команд із косою рискою; раніше зареєстровані команди можуть залишатися видимими до видалення.
</ParamField>

<ParamField path="commands.nativeSkills" type='boolean | "auto"' default='"auto"'>
  Реєструє команди навичок як нативні, якщо це підтримується. Автоматичний режим: увімкнено для
  Discord/Telegram; вимкнено для Slack. Перевизначається через
  `channels.<provider>.commands.nativeSkills`.
</ParamField>

<ParamField path="commands.bash" type="boolean" default="false">
  Вмикає `! <cmd>` для виконання команд оболонки хоста (псевдонім `/bash <cmd>`). Потребує
  списків дозволених `tools.elevated`.
</ParamField>

<ParamField path="commands.bashForegroundMs" type="number" default="2000">
  Визначає, скільки часу bash очікує перед переходом у фоновий режим (`0` переходить
  у фоновий режим негайно).
</ParamField>

<ParamField path="commands.config" type="boolean" default="false">
  Вмикає `/config` (читання/запис `openclaw.json`). Лише для власника.
</ParamField>

<ParamField path="commands.mcp" type="boolean" default="false">
  Вмикає `/mcp` (читання/запис керованої OpenClaw конфігурації MCP у `mcp.servers`). Лише для власника.
</ParamField>

<ParamField path="commands.plugins" type="boolean" default="false">
  Вмикає `/plugins` (виявлення/стан плагінів, а також установлення та ввімкнення/вимкнення). Запис — лише для власника.
</ParamField>

<ParamField path="commands.debug" type="boolean" default="false">
  Вмикає `/debug` (перевизначення конфігурації лише під час виконання). Лише для власника.
</ParamField>

<ParamField path="commands.restart" type="boolean" default="true">
  Вмикає `/restart` та зовнішні запити на перезапуск `SIGUSR1`.
</ParamField>

<ParamField path="commands.ownerAllowFrom" type="string[]">
  Явний список дозволених власників для поверхонь команд, доступних лише власнику. Окремий від
  `commands.allowFrom` і доступу через сполучення в особистих повідомленнях.
</ParamField>

<ParamField path="channels.<channel>.commands.enforceOwnerForCommands" type="boolean" default="false">
  Для кожного каналу: вимагає ідентичності власника для команд, доступних лише власнику. Коли `true`,
  відправник має відповідати `commands.ownerAllowFrom` або мати внутрішню область дії `operator.admin`.
  Запис із символом узагальнення `allowFrom` **недостатній**.
</ParamField>

<ParamField path="commands.ownerDisplay" type='"raw" | "hash"'>
  Керує тим, як ідентифікатори власника відображаються в системній підказці.
</ParamField>

<ParamField path="commands.ownerDisplaySecret" type="string">
  Секрет HMAC, що використовується, коли `commands.ownerDisplay: "hash"`.
</ParamField>

<ParamField path="commands.allowFrom" type="object">
  Список дозволених для кожного постачальника для авторизації команд. Якщо його налаштовано, він є
  **єдиним** джерелом авторизації для команд і директив. Використовуйте `"*"` як
  глобальне типове значення; ключі конкретних постачальників його перевизначають.
</ParamField>

<ParamField path="commands.useAccessGroups" type="boolean" default="true">
  Застосовує списки дозволених/політики для команд, коли `commands.allowFrom` не встановлено.
</ParamField>

## Список команд

Команди надходять із трьох джерел:

- **Вбудовані команди ядра:** `src/auto-reply/commands-registry.shared.ts`
- **Згенеровані команди доку:** `src/auto-reply/commands-registry.data.ts`
- **Команди плагінів:** виклики `registerCommand()` плагіна

Доступність залежить від прапорців конфігурації, поверхні каналу та встановлених/увімкнених
плагінів.

### Команди стикування

Команди стикування перемикають маршрут відповіді активного сеансу на інший пов’язаний канал.
Відомості про налаштування та усунення несправностей див. у розділі [Стикування каналів](/uk/concepts/channel-docking).

Згенеровано з плагінів каналів із підтримкою нативних команд:

- `/dock-discord` (псевдонім: `/dock_discord`)
- `/dock-mattermost` (псевдонім: `/dock_mattermost`)
- `/dock-slack` (псевдонім: `/dock_slack`)
- `/dock-telegram` (псевдонім: `/dock_telegram`)

Для команд стикування потрібно `session.identityLinks`. Відправник у вихідному каналі та цільовий співрозмовник
мають належати до однієї групи ідентичностей.

### Команди комплектних плагінів

| Команда                                                 | Опис                                                                                                                                                                                    |
| ------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `/dreaming [on\|off\|status\|help]`                     | Увімкнути або вимкнути Dreaming пам’яті (власник або адміністратор Gateway). Див. [Dreaming](/uk/concepts/dreaming)                                                                                                            |
| `/pair [qr\|status\|pending\|approve\|cleanup\|notify]` | Керувати сполученням пристроїв. Див. [Сполучення](/uk/channels/pairing)                                                                                                                                        |
| `/phone status\|arm ...\|disarm`                        | Тимчасово дозволити високоризикові команди вузла (камера/екран/комп’ютер/запис). Див. [Використання комп’ютера](/uk/nodes/computer-use)                                                                               |
| `/voice status\|list\|set <voiceId>`                    | Керувати конфігурацією голосу Talk. Нативна назва в Discord: `/talkvoice`                                                                                                                                    |
| `/card ...`                                             | Надсилати шаблони розширених карток LINE. Див. [LINE](/uk/channels/line)                                                                                                                                        |
| `/codex <action> ...`                                   | Прив’язувати, спрямовувати та перевіряти оснастку сервера застосунку Codex (стан, гілки, відновлення, модель, швидкий режим, дозволи, стискання, перевірка, mcp, skills тощо). Див. [Оснастка Codex](/uk/plugins/codex-harness) |

Лише для QQBot: `/bot-ping`, `/bot-version`, `/bot-help`, `/bot-upgrade`, `/bot-logs`

### Команди skills

Доступні для виклику користувачем skills надаються як команди зі скісною рискою:

- `/skill <name> [input]` завжди працює як універсальна точка входу.
- Skills можуть реєструватися як безпосередні команди (наприклад, `/prose` для OpenProse).
- Реєстрація нативних команд skills керується параметрами `commands.nativeSkills` і
  `channels.<provider>.commands.nativeSkills`.
- Назви очищуються до `a-z0-9_` (щонайбільше 32 символи); у разі конфліктів додаються числові суфікси.

<AccordionGroup>
  <Accordion title="Диспетчеризація команд skills">
    Типово команди skills передаються моделі як звичайний запит.

    Skills можуть оголошувати `command-dispatch: tool`, щоб передавати запит безпосередньо інструменту
    (детерміновано, без залучення моделі). Приклад: `/prose` (плагін OpenProse)
    — див. [OpenProse](/uk/prose).

  </Accordion>
  <Accordion title="Аргументи нативних команд">
    Discord використовує автодоповнення для динамічних параметрів і меню кнопок, коли обов’язкові
    аргументи не вказано. Telegram і Slack показують меню кнопок для команд із
    варіантами. Динамічні варіанти визначаються відповідно до моделі цільового сеансу, тому залежні від
    моделі параметри, як-от рівні `/think`, враховують перевизначення `/model` сеансу.
  </Accordion>
</AccordionGroup>

## `/tools`: що агент може використовувати зараз

`/tools` відповідає на запитання про середовище виконання: **що цей агент може використовувати просто зараз у цій
розмові**, а не надає статичний каталог конфігурації.

```text
/tools         # стислий перегляд
/tools verbose # з короткими описами
```

Результати обмежено сеансом. Зміна агента, каналу, гілки, авторизації
відправника або моделі може змінити вивід. Для редагування профілю та перевизначень
використовуйте панель Tools в інтерфейсі керування або поверхні конфігурації.

## `/model`: вибір моделі

```text
/model             # показати засіб вибору моделі
/model list        # те саме
/model 3           # вибрати за номером із засобу вибору
/model openai/gpt-5.4
/model opus@anthropic:default
/model default     # скинути вибір моделі для сеансу
/model status      # докладний перегляд із кінцевою точкою та режимом API
```

У Discord команди `/model` і `/models` відкривають інтерактивний засіб вибору зі спадними списками
провайдерів і моделей. Засіб вибору враховує `agents.defaults.models`, зокрема
записи `provider/*`.

## `/config`: запис конфігурації на диск

<Note>
  Лише для власника. Типово вимкнено — увімкніть за допомогою `commands.config: true`.
</Note>

```text
/config show
/config show messages.responsePrefix
/config get messages.responsePrefix
/config set messages.responsePrefix="[openclaw]"
/config unset messages.responsePrefix
```

Конфігурація перевіряється перед записом. Недійсні зміни відхиляються. Оновлення `/config`
зберігаються після перезапусків.

## `/mcp`: конфігурація сервера MCP

<Note>
  Лише для власника. Типово вимкнено — увімкніть за допомогою `commands.mcp: true`.
</Note>

```text
/mcp show
/mcp show context7
/mcp set context7={"command":"uvx","args":["context7-mcp"]}
/mcp unset context7
```

`/mcp` зберігає конфігурацію в конфігурації OpenClaw, а не в налаштуваннях проєкту вбудованого агента.
`/mcp show` приховує поля з обліковими даними, розпізнані значення прапорців
облікових даних і відомі аргументи, схожі на секрети. Під час виконання з групи
конфігурація приватно надсилається власнику; якщо приватний маршрут до власника
недоступний, команда безпечно завершується відмовою та просить власника повторити спробу в особистому
чаті.

## `/debug`: перевизначення лише для середовища виконання

<Note>
  Лише для власника. Типово вимкнено — увімкніть за допомогою `commands.debug: true`.
  Перевизначення негайно застосовуються до нових зчитувань конфігурації, але **не** записуються на диск.
</Note>

```text
/debug show
/debug set messages.responsePrefix="[openclaw]"
/debug set channels.whatsapp.allowFrom=["+1555","+4477"]
/debug unset messages.responsePrefix
/debug reset
```

## `/plugins`: керування плагінами

<Note>
  Запис — лише для власника. Типово вимкнено — увімкніть за допомогою `commands.plugins: true`.
</Note>

```text
/plugins
/plugins list
/plugin show context7
/plugins enable context7
/plugins disable context7
/plugins install clawhub:<package>
/plugins install npm:@openclaw/<official-package>
/plugins install npm:<package> --force
/plugins install git:<repository>@<ref> --force
```

`/plugins enable|disable` оновлює конфігурацію плагіна та виконує гаряче перезавантаження
середовища виконання плагінів Gateway для нових звернень агента. `/plugins install` автоматично перезапускає керовані
Gateway, оскільки вихідні модулі плагіна змінилися. Установлення з довіреного ClawHub
та офіційного каталогу не потребують додаткового підтвердження. Довільні джерела npm,
git, архіви, `npm-pack:` та локальні шляхи показують попередження про походження й
потребують кінцевого `--force` після перевірки джерела. Цей прапорець підтверджує
джерело та дозволяє замінити наявне встановлення; він не обходить
`security.installPolicy` або перевірки безпеки інсталятора. Випуски ClawHub із
попередженнями про ризики все одно потребують окремого, доступного лише в оболонці
прапорця `--acknowledge-clawhub-risk`. Установлення з маркетплейсу, пов’язані та закріплені встановлення також
залишаються доступними лише в оболонці.

## `/trace`: вивід трасування плагіна

```text
/trace          # показати поточний стан трасування
/trace on
/trace off
```

`/trace` показує обмежені сеансом рядки трасування й налагодження плагіна без повного докладного
режиму. Це не замінює `/debug` (перевизначення середовища виконання) або `/verbose` (звичайний
вивід інструменту).

## `/btw`: побічні запитання

`/btw` — це коротке побічне запитання про контекст поточного сеансу. Псевдонім: `/side`.

```text
/btw що ми зараз робимо?
/side що змінилося, поки основний запуск тривав?
```

На відміну від звичайного повідомлення:

- Використовує поточний сеанс як фоновий контекст.
- У сеансах оснастки Codex виконується як тимчасова побічна гілка Codex.
- **Не** змінює майбутній контекст сеансу.
- Не записується до історії розшифрування.

Повний опис поведінки див. у розділі [Побічні запитання BTW](/uk/tools/btw).

## Примітки щодо поверхонь

<AccordionGroup>
  <Accordion title="Область сеансу для кожної поверхні">
    - **Текстові команди:** виконуються у звичайному сеансі чату (особисті повідомлення спільно використовують `main`, групи мають власний сеанс).
    - **Нативні команди Discord:** `agent:<agentId>:discord:slash:<userId>`
    - **Нативні команди Slack:** `agent:<agentId>:slack:slash:<userId>` (префікс налаштовується через `channels.slack.slashCommand.sessionPrefix`)
    - **Нативні команди Telegram:** `telegram:slash:<userId>` (спрямовуються до сеансу чату через `CommandTargetSessionKey`)
    - **`/login codex`** надсилає коди сполучення пристроїв лише через приватний чат або канали відповіді вебінтерфейсу. У разі виклику в групі або темі Telegram власника просять натомість написати боту особисте повідомлення.
    - **`/stop`** спрямовується до активного сеансу чату, щоб перервати поточний запуск.

  </Accordion>
  <Accordion title="Особливості Slack">
    `channels.slack.slashCommand` підтримує одну команду в стилі `/openclaw`.
    З `commands.native: true` створіть окрему команду зі скісною рискою Slack для кожної вбудованої
    команди. Зареєструйте `/agentstatus` (а не `/status`), оскільки Slack резервує
    `/status`. Текст `/status` і далі працює в повідомленнях Slack.
  </Accordion>
  <Accordion title="Швидкий шлях і вбудовані скорочення">
    - Повідомлення, що містять лише команди, від відправників зі списку дозволених обробляються негайно (в обхід черги та моделі).
    - Вбудовані скорочення (`/help`, `/commands`, `/status`, `/whoami`) також працюють усередині звичайних повідомлень і вилучаються до того, як модель побачить решту тексту.
    - Неавторизовані повідомлення, що містять лише команди, мовчки ігноруються; вбудовані токени `/...` сприймаються як звичайний текст.

  </Accordion>
  <Accordion title="Примітки щодо аргументів">
    - Команди приймають необов’язковий `:` між командою та аргументами (`/think: high`, `/send: on`).
    - `/new <model>` приймає псевдонім моделі, `provider/model` або назву постачальника (нечітке зіставлення); якщо збігу немає, текст сприймається як тіло повідомлення.
    - `/allowlist add|remove` вимагає `commands.config: true` і враховує параметр каналу `configWrites`.

  </Accordion>
</AccordionGroup>

## Використання та стан постачальника

- **Використання/квота постачальника** (наприклад, «Claude: залишилося 80%») відображається в `/status` для постачальника поточної моделі, коли відстеження використання ввімкнено.
- **Рядки токенів/кешу** в `/status` можуть використовувати останній запис про використання зі стенограми, якщо актуальний знімок сеансу містить недостатньо даних.
- **Середовище виконання та рантайм:** `/status` повідомляє `Execution` для фактичного шляху пісочниці та `Runtime` для того, хто виконує сеанс: `OpenClaw Default`, `OpenAI Codex`, бекенд CLI або бекенд ACP.
- **Токени/вартість для кожної відповіді:** керуються параметром `/usage off|tokens|full`.
- `/model status` стосується моделей, автентифікації та кінцевих точок, а не використання.
