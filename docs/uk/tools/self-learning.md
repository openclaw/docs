Самонавчання дає змогу OpenClaw перетворювати корисні свідчення з розмов на пропозиції, що очікують розгляду, у
[Skill Workshop](/uk/tools/skill-workshop). Воно не навчає ваги моделі,
не редагує активні навички й не змінює непомітно поведінку агента. Кожна вивчена
процедура залишається в очікуванні, доки оператор не перегляне та не застосує її.

Самонавчання **вимкнено за замовчуванням**. Вмикайте його лише тоді, коли додатковий
фоновий запуск моделі та перегляд стенограми доречні для вашого робочого простору.

## Увімкнення самонавчання

У Control UI відкрийте **Plugins → Workshop** і ввімкніть **Self-learning**. Зміна
набирає чинності негайно; якщо інший засіб запису конфігурації оновив
файл, Control UI оновлює знімок конфігурації та повторює перемикання без
перезавантаження сторінки чи Gateway.

Скористайтеся CLI:

```bash
openclaw config set skills.workshop.autonomous.enabled true --strict-json
```

Або відредагуйте `~/.openclaw/openclaw.json`:

```json5
{
  skills: {
    workshop: {
      autonomous: {
        enabled: true,
      },
    },
  },
}
```

Щоб знову вимкнути самонавчання, виконайте:

```bash
openclaw config set skills.workshop.autonomous.enabled false --strict-json
```

Створення навичок за запитом користувача, `/learn` і ручні операції Skill Workshop
продовжують працювати, коли самонавчання вимкнено.

## Ручний перегляд минулих сеансів

Ручний перегляд історії — консервативна альтернатива автономному збиранню.
Відкрийте **Plugins → Workshop** у Control UI та виберіть **Find skill ideas**.
Це не змінює `skills.workshop.autonomous.enabled`.

Кожне сканування:

- починається з найновіших непереглянутих сеансів і рухається назад;
- переглядає до 20 змістовних сеансів, що містять щонайменше шість ходів моделі;
- пропускає сеанси Cron, Heartbeat, хуків, субагентів, ACP, сеанси, що належать плагінам, і внутрішні сеанси перегляду;
- редагує розпізнані секрети й обмежує пакет стенограм перед надсиланням
  налаштованій моделі вибраного агента;
- застосовує такі самі високі вимоги, як і автономний перегляд досвіду; і
- може створити або переглянути не більше трьох пропозицій, що очікують розгляду, але ніколи не активні навички.

Workshop повідомляє сукупну кількість сеансів, охоплення дат та кількість знайдених ідей.
Виберіть **Scan earlier work**, щоб перейти до наступного старішого вікна. Коли курсор досягне
початку придатної історії, дія зміниться на **Scan new work**.
OpenClaw зберігає в спільній базі даних стану лише курсор і метадані охоплення;
він не створює другого архіву стенограм.

Сеанси скануються лише тоді, коли OpenClaw може підтвердити їхню належність і виключити
вміст зовнішніх хуків. Після оновлення поточну стенограму, створену до оновлення, можна
класифікувати локально, але ротовані стенограми до оновлення без даних про походження
кожного запуску пропускаються. Нові стенограми зберігають ці дані про походження після ротації.

Ручні сканування все одно спричиняють витрати на постачальника моделі й надсилають придатний
вміст розмови налаштованому постачальнику. Використовуйте їх лише тоді, коли такий перегляд
відповідає вимогам робочого простору щодо конфіденційності й обробки даних.

## Чого може навчитися OpenClaw

Самонавчання має два консервативні шляхи:

1. **Прямі інструкції та виправлення.** OpenClaw виявляє довготривалі вказівки,
   як-от «відтепер», «наступного разу», а також виправлення невдалого підходу.
   Коли самонавчання ввімкнено, він може перетворити ці сигнали на пропозиції, що очікують розгляду,
   не чекаючи іншого запиту. Цей детермінований шлях може згрупувати пов’язані
   інструкції щонайбільше в три пропозиції, націлити їх на доступну для запису навичку робочого простору
   або переглянути власну пов’язану пропозицію, що очікує розгляду. Він також запускається після невдалих ходів,
   оскільки фіксує інструкції користувача, а не оцінює завершення.
2. **Перегляд досвіду.** Після успішного змістовного ходу переднього плану
   OpenClaw може переглянути виконану роботу, щоб знайти придатний для повторного використання спосіб відновлення або
   стабільну процедуру, яка усуне щонайменше два майбутні цикли
   взаємодії з моделлю чи інструментом.

До вдалих кандидатів належать:

- надійне відновлення після повторних збоїв інструмента або моделі;
- неочевидне обмеження порядку, що запобігло повторюваній помилці;
- стабільний багатоетапний робочий процес, який вимагав повторного пошуку; або
- придатна для повторного використання попередня перевірка, яка дасть змогу уникнути кількох майбутніх викликів.

Рецензент має утримуватися від створення пропозицій для звичайної успішної роботи, одноразових запитів,
особистих фактів, простих уподобань, тимчасових збоїв середовища, загальних
порад, непідтверджених негативних тверджень і секретів.

## Коли запускається перегляд досвіду

Перегляд досвіду навмисно відкладено й обмежено:

- Хід переднього плану має успішно завершитися.
- Поточний хід має містити щонайменше десять ітерацій моделі.
- Сеанси Cron, Heartbeat, пам’яті, переповнення, хуків, субагентів і перегляду
  виключаються.
- Запуск переднього плану має визначити постачальника й модель та фактично
  мати доступ до `skill_workshop`.
- OpenClaw очікує 30 секунд після завершення. Пізніше завершення переднього плану в
  тому самому сеансі перезапускає цей період спокою.
- Якщо будь-який запуск агента або відповіді все ще активний, перегляд очікує ще 30 секунд.
- Одночасно виконується лише один перегляд досвіду.
- Відкладений перегляд — це локальна для процесу робота Gateway. Gateway має продовжувати працювати
  протягом вікна бездіяльності; одноразові локальні середовища виконання та середовища на основі CLI не зберігають
  достатнього контексту траєкторії й доступності інструментів для його планування.

Навчання ніколи не затримує відповідь переднього плану. Невдалий або непридатний
хід не запускає перегляд досвіду, хоча прямі виправлення користувача
все одно можуть пропонуватися як рекомендація, коли автономність вимкнено.

## Що отримує рецензент

Фоновий рецензент отримує лише поточний хід, починаючи з його останнього
повідомлення користувача. Відтворену траєкторію обмежено 60,000 символами;
за потреби OpenClaw зберігає перше повідомлення та найновіші свідчення й
позначає пропущену середину.

Рецензент повторно використовує визначених постачальника та модель. Він повторно використовує профіль
автентифікації переднього плану, коли ця ідентичність доступна, і вимикає резервні моделі. Тому
перегляд запускає додатковий запуск моделі в налаштованого постачальника.
Цей запуск може виконати більше одного запиту до постачальника під час перевірки або підготовки
пропозиції. Умови ціноутворення та обробки даних постачальника застосовуються так само, як і до
ходу переднього плану.

Перед запуском OpenClaw повторно завантажує поточну конфігурацію середовища виконання й перевіряє
чинну політику пісочниці та інструментів для початкової розмови. Якщо запуск
виконується в пісочниці, політика більше не дозволяє `skill_workshop` або відсутні необхідні факти
середовища виконання, перегляд безпечно завершується відмовою й нічого не створює.

<Warning>
  Увімкнення самонавчання дозволяє надсилати вибраному постачальнику
  моделі придатний вміст розмови, зокрема вхідні дані інструментів і результати
  поточного ходу, для одного додаткового перегляду. Не вмикайте його в робочому просторі, де
  такий перегляд порушуватиме вимоги щодо обробки даних.
</Warning>

## Безпека пропозицій

Рецензент працює в ізольованому сеансі з навмисно вузьким набором
інструментів:

- Він може лише перелічувати або перевіряти пропозиції Workshop і створювати або переглядати одну
  пропозицію, що очікує розгляду.
- Він не може оновлювати активну навичку, застосовувати, відхиляти чи ізолювати
  пропозицію, надсилати повідомлення або використовувати загальні інструменти агента.
- Один бюджет змін спільний для всіх повторних спроб моделі, тому перегляд може створити або
  переглянути щонайбільше одну пропозицію.
- Переглянута траєкторія розглядається як ненадійні свідчення, а не як інструкції
  для фонового агента.
- Skill Workshop сканує вміст пропозиції та відхиляє розпізнані буквальні
  облікові дані до запису стану пропозиції.

Звичайні обмеження Workshop і надалі застосовуються, зокрема `maxPending`, `maxSkillBytes`,
обмеження допоміжних файлів, перевірки сканера та запис лише в робочий простір. Налаштування
`approvalPolicy: "auto"` не надає фоновому рецензенту доступу
до дій життєвого циклу.

## Перегляд вивчених пропозицій

Самонавчання створює такі самі пропозиції, що очікують розгляду, як і ручне використання Workshop.
Перевірте їх перед застосуванням:

```bash
openclaw skills workshop list
openclaw skills workshop inspect <proposal-id>
openclaw skills workshop apply <proposal-id>
```

Переглядайте, відхиляйте або ізолюйте корисні, але ще не готові пропозиції:

```bash
openclaw skills workshop revise <proposal-id> --proposal ./PROPOSAL.md
openclaw skills workshop reject <proposal-id> --reason "Too specific"
openclaw skills workshop quarantine <proposal-id> --reason "Needs security review"
```

Застосування — єдина операція, що записує активну `SKILL.md`. Повну модель
життєвого циклу та зберігання див. у
[Skill Workshop](/uk/tools/skill-workshop).

## Конфігурація

| Налаштування                               | Значення за замовчуванням | Вплив самонавчання                                                                                                                 |
| ------------------------------------------ | ------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| `skills.workshop.autonomous.enabled`       | `false`  | Вмикає фіксацію прямих виправлень і відкладений перегляд досвіду.                                                                  |
| `skills.workshop.approvalPolicy`           | `"auto"` | Керує запитами на схвалення для звичайних ініційованих агентом дій життєвого циклу; не розширює дозволи фонового рецензента. |
| `skills.workshop.maxPending`               | `50`     | Обмежує кількість пропозицій, що очікують розгляду або перебувають в ізоляції, для кожного робочого простору.                                                                             |
| `skills.workshop.maxSkillBytes`            | `40000`  | Обмежує розмір тіла пропозиції в байтах.                                                                                                 |
| `skills.workshop.allowSymlinkTargetWrites` | `false`  | Впливає лише на застосування; саме самонавчання записує стан пропозиції, а не активні цільові навички.                                  |

Повну схему, діапазони та пов’язані налаштування навичок див. у
[конфігурації Skills](/uk/tools/skills-config#workshop-skills-workshop).

## Усунення несправностей

### Після тривалого ходу пропозиція не з’являється

Перевірте все наведене нижче:

1. `skills.workshop.autonomous.enabled` має значення `true` в активній конфігурації Gateway.
2. Хід завершився успішно й містив щонайменше десять ітерацій моделі після останнього
   повідомлення користувача.
3. Розмова була звичайним запуском переднього плану, а не запланованим запуском, запуском пам’яті,
   хука чи субагента.
4. Початковий запуск мав доступ до `skill_workshop` і не виконувався в пісочниці.
5. Система залишалася бездіяльною достатньо довго для відкладеного перегляду.
6. Довготривалий процес Gateway залишався активним протягом вікна бездіяльності; одноразова
   локальна команда не очікує відкладеного перегляду.

Навіть придатний перегляд може не створити пропозиції. Утримання від створення — очікуваний
результат, коли свідчення не відповідають вимогам до придатної для повторного використання процедури.

### Doctor повідомляє, що інструмент Workshop приховано

Коли самонавчання ввімкнено, `openclaw doctor` перевіряє, чи дозволяє чинна
політика інструментів агента за замовчуванням `skill_workshop`. Застосуйте повідомлену
зміну `tools.allow` або `tools.alsoAllow` чи вимкніть самонавчання.

### З’являється забагато малоцінних пропозицій

Вимкніть самонавчання та продовжуйте використовувати `/learn` або явні запити Workshop:

```bash
openclaw config set skills.workshop.autonomous.enabled false --strict-json
```

Пропозиції, що очікують розгляду, залишаються доступними для перегляду після вимкнення функції. Вимкнення
самонавчання не застосовує, не відхиляє й не видаляє їх.
