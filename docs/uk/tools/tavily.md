[Tavily](https://tavily.com) — це пошуковий API, розроблений для застосунків ШІ. OpenClaw надає до нього доступ двома способами:

- як до провайдера `web_search` для універсального інструмента пошуку
- як до окремих інструментів плагіна: `tavily_search` і `tavily_extract`

Tavily повертає структуровані результати, оптимізовані для використання LLM, із налаштовуваною глибиною пошуку, фільтрацією за темами й доменами, зведеннями відповідей, створеними ШІ, та видобуванням вмісту з URL-адрес (зокрема зі сторінок, відтворюваних за допомогою JavaScript).

| Властивість | Значення                                                                                              |
| ----------- | ----------------------------------------------------------------------------------------------------- |
| Ідентифікатор плагіна | `tavily`                                                                                      |
| Пакет       | `@openclaw/tavily-plugin`                                                                             |
| Автентифікація | змінна середовища `TAVILY_API_KEY` або параметр конфігурації `apiKey`                              |
| Базова URL-адреса | `https://api.tavily.com` (типово); змінна середовища `TAVILY_BASE_URL` або параметр конфігурації `baseUrl` для перевизначення |
| Час очікування | 30 с для пошуку, 60 с для видобування (типово)                                                     |
| Інструменти | `tavily_search`, `tavily_extract`                                                                     |

## Початок роботи

<Steps>
  <Step title="Установіть плагін">
    ```bash
    openclaw plugins install @openclaw/tavily-plugin
    ```
  </Step>
  <Step title="Отримайте ключ API">
    Створіть обліковий запис Tavily на [tavily.com](https://tavily.com), а потім згенеруйте ключ API на панелі керування.
  </Step>
  <Step title="Налаштуйте плагін і провайдера">
    ```json5
    {
      plugins: {
        entries: {
          tavily: {
            enabled: true,
            config: {
              webSearch: {
                apiKey: "tvly-...", // необов’язково, якщо задано TAVILY_API_KEY
                baseUrl: "https://api.tavily.com",
              },
            },
          },
        },
      },
      tools: {
        web: {
          search: {
            provider: "tavily",
          },
        },
      },
    }
    ```
  </Step>
  <Step title="Перевірте виконання пошуку">
    Запустіть `web_search` із будь-якого агента або викличте `tavily_search` безпосередньо.
  </Step>
</Steps>

<Tip>
Якщо вибрати Tavily під час початкового налаштування або в `openclaw configure --section web`, офіційний плагін Tavily буде встановлено й увімкнено за потреби.
</Tip>

## Довідник інструментів

### `tavily_search`

Використовуйте його, коли замість універсального `web_search` потрібні спеціальні параметри керування пошуком Tavily.

| Параметр          | Тип          | Обмеження / типове значення            | Опис                                               |
| ----------------- | ------------ | -------------------------------------- | -------------------------------------------------- |
| `query`           | рядок        | обов’язковий                           | Рядок пошукового запиту.                           |
| `search_depth`    | перелік      | `basic` (типово), `advanced`           | `advanced` працює повільніше, але дає релевантніші результати. |
| `topic`           | перелік      | `general` (типово), `news`, `finance`  | Фільтрування за категорією теми.                   |
| `max_results`     | ціле число   | 1–20, типово `5`                       | Кількість результатів.                             |
| `include_answer`  | логічне значення | типово `false`                     | Додати зведення відповіді, створене ШІ Tavily.     |
| `time_range`      | перелік      | `day`, `week`, `month`, `year`         | Фільтрувати результати за давністю.                 |
| `include_domains` | масив рядків | (немає)                                | Додавати лише результати із цих доменів.           |
| `exclude_domains` | масив рядків | (немає)                                | Виключати результати із цих доменів.               |

Компроміс між режимами глибини пошуку:

| Глибина   | Швидкість | Релевантність | Найкраще підходить для                         |
| --------- | --------- | ------------- | ---------------------------------------------- |
| `basic`   | Вища      | Висока        | Запитів загального призначення (типово).       |
| `advanced` | Нижча    | Найвища       | Точних досліджень і пошуку фактів.             |

### `tavily_extract`

Використовуйте його для видобування очищеного вмісту з однієї чи кількох URL-адрес. Він обробляє сторінки, відтворювані за допомогою JavaScript, і підтримує поділ на фрагменти з орієнтацією на запит для цільового видобування.

| Параметр            | Тип          | Обмеження / типове значення   | Опис                                                        |
| ------------------- | ------------ | ----------------------------- | ----------------------------------------------------------- |
| `urls`              | масив рядків | обов’язковий, 1–20            | URL-адреси, з яких потрібно видобути вміст.                  |
| `query`             | рядок        | (необов’язковий)              | Повторно впорядкувати видобуті фрагменти за релевантністю цьому запиту. |
| `extract_depth`     | перелік      | `basic` (типово), `advanced`  | Використовуйте `advanced` для сторінок із великою кількістю JS, SPA або динамічних таблиць. |
| `chunks_per_source` | ціле число   | 1–5; **потребує `query`**     | Кількість фрагментів, що повертаються для кожної URL-адреси. Спричиняє помилку, якщо задано без `query`. |
| `include_images`    | логічне значення | типово `false`            | Додати URL-адреси зображень до результатів.                  |

Компроміс між режимами глибини видобування:

| Глибина    | Коли використовувати                              |
| ---------- | ------------------------------------------------- |
| `basic`    | Прості сторінки. Спочатку спробуйте цей режим.    |
| `advanced` | SPA, відтворювані за допомогою JS, динамічний вміст і таблиці. |

<Tip>
Розділяйте великі списки URL-адрес на кілька викликів `tavily_extract` (не більше 20 на запит). Використовуйте `query` разом із `chunks_per_source`, щоб отримати лише релевантний вміст замість повних сторінок.
</Tip>

## Вибір відповідного інструмента

| Потреба                                      | Інструмент        |
| -------------------------------------------- | ----------------- |
| Швидкий вебпошук без спеціальних параметрів  | `web_search`      |
| Пошук із налаштуванням глибини, теми та відповідями ШІ | `tavily_search` |
| Видобування вмісту з певних URL-адрес        | `tavily_extract`  |

<Note>
Універсальний інструмент `web_search` із Tavily як провайдером підтримує `query` і `count` (до 20 результатів). Для спеціальних параметрів Tavily (`search_depth`, `topic`, `include_answer`, фільтрів доменів і часового діапазону) натомість використовуйте `tavily_search`.
</Note>

## Розширена конфігурація

<AccordionGroup>
  <Accordion title="Порядок визначення ключа API">
    Клієнт Tavily шукає ключ API в такому порядку:

    1. `plugins.entries.tavily.config.webSearch.apiKey` (визначається через SecretRefs).
    2. `TAVILY_API_KEY` із середовища Gateway.

    І `tavily_search`, і `tavily_extract` повертають помилку налаштування, якщо немає жодного з них.

  </Accordion>

  <Accordion title="Власна базова URL-адреса">
    Перевизначте `plugins.entries.tavily.config.webSearch.baseUrl` або задайте `TAVILY_BASE_URL`, якщо доступ до Tavily здійснюється через проксі. Конфігурація має пріоритет над змінною середовища. Типове значення — `https://api.tavily.com`.
  </Accordion>

  <Accordion title="`chunks_per_source` потребує `query`">
    `tavily_extract` відхиляє виклики, які передають `chunks_per_source` без `query`. Tavily упорядковує фрагменти за релевантністю запиту, тому без нього цей параметр не має сенсу.
  </Accordion>
</AccordionGroup>
