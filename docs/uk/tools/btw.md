`/btw` (псевдонім `/side`) дає змогу поставити коротке побічне запитання про **поточний
сеанс**, не додаючи його до історії розмови. Цю функцію створено за зразком
`/btw` у Claude Code й адаптовано до Gateway та багатоканальної
архітектури OpenClaw.

```text
/btw що змінилося?
/side що означає ця помилка?
```

## Що вона робить

1. Створює знімок поточного сеансу як фоновий контекст (зокрема й запит
   основного виконання, що наразі обробляється).
2. Виконує окремий одноразовий побічний запит, наказуючи моделі відповісти лише на
   побічне запитання й не відновлювати та не спрямовувати основне завдання.
3. Доставляє відповідь як оперативний побічний результат, а не як звичайне повідомлення асистента.
4. Ніколи не записує запитання чи відповідь до історії сеансу або `chat.history`.

Основне виконання, якщо воно активне, залишається без змін.

Для сеансів середовища Codex функція BTW відгалужує активний потік сервера застосунку Codex
у тимчасовий дочірній потік замість виконання окремого виклику постачальника. Завдяки цьому
OAuth Codex і вбудована поведінка інструментів та потоків залишаються без змін, а відгалужений
потік зберігає поточну політику схвалення, пісочницю та набір вбудованих
інструментів батьківського потоку. Відгалужений потік отримує граничний запит, який повідомляє моделі, що
все перед ним є успадкованим довідковим контекстом, а не активними інструкціями,
і що активними є лише повідомлення після межі. Для `/btw` потрібен
наявний потік Codex; спочатку надішліть звичайне повідомлення.

Для псевдонімів середовища виконання CLI функція BTW викликає відповідний серверний компонент CLI в режимі
одноразового побічного запитання: вона передає очищений контекст розмови до нового виклику CLI
з вимкненими об’єднанням інструментів і повторно використовуваним станом сеансу та додає
підтримувані серверним компонентом прапорці, що забороняють відновлення та використання інструментів. Безпосередні середовища виконання
(не CLI) натомість використовують прямий одноразовий виклик постачальника.

## Чого вона не робить

`/btw` не створює постійний сеанс, не продовжує незавершене основне завдання,
не зберігає дані запитання й відповіді в історії стенограми та не відновлюється після перезавантаження.

## Модель доставки

Звичайний чат з асистентом використовує подію Gateway `chat`. BTW використовує окрему
подію `chat.side_result`, щоб клієнти не могли сплутати її зі звичайною
історією розмови. Оскільки вона не відтворюється з `chat.history`, то
зникає після перезавантаження.

## Поведінка в інтерфейсах

| Інтерфейс         | Поведінка                                                                                                                                                                                                                                                                            |
| ----------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| TUI               | Відображається безпосередньо в журналі чату, помітно відрізняється від звичайної відповіді; її можна закрити за допомогою `Enter` або `Esc`.                                                                                                                                                                           |
| Зовнішні канали   | Доставляється як чітко позначена одноразова відповідь (Telegram, WhatsApp і Discord не мають локального тимчасового накладеного елемента).                                                                                                                                                                         |
| Control UI / веб  | Відображається як рухома панель "Побічний чат", закріплена за потоком. Відповіді накопичуються як репліки, а поле "Подальше запитання" дає змогу поставити наступне побічне запитання. Закриття (`Esc` або X) зберігає розмову, і панель знову відкривається після наступної відповіді; кнопка кошика видаляє її та зупиняє запит, що виконується. |

## Спливне меню виділення (Control UI)

Після виділення тексту в повідомленні чату в Control UI відкривається невелике
спливне меню виділення з двома діями:

- **Докладніше** негайно надсилає неявне запитання `/btw` із проханням до
  моделі пояснити виділений текст у контексті поточного
  сеансу. Відповідь надходить до рухомої панелі побічного чату.
- **Запитати в побічному чаті** попередньо заповнює редактор чернеткою `/btw`, що цитує
  виділений текст, щоб можна було ввести власне запитання про нього.

Обидві дії відповідають звичайній семантиці `/btw`: запитання та відповідь не потрапляють
до історії сеансу, а основне виконання залишається без змін.

## Коли її використовувати

Використовуйте `/btw` для швидкого уточнення, отримання побічної фактичної відповіді, поки тривале виконання
ще триває, або тимчасової відповіді, яка не повинна потрапити до майбутнього
контексту сеансу.

```text
/btw який файл ми редагуємо?
/btw підсумуй поточне завдання одним реченням
/btw скільки буде 17 * 19?
```

Усе, що має стати частиною майбутнього робочого
контексту сеансу, натомість запитуйте звичайним способом в основному сеансі.
