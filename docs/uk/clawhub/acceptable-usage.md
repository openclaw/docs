# Прийнятне використання

ClawHub розміщує skills, plugins, пакети та метадані marketplace для OpenClaw.
Використовуйте цю сторінку, щоб вирішити, чи належить контент або поведінка публікації
до ClawHub.

Ці правила застосовуються до того, що робить запис, що він просить користувачів запускати, як він
представляє себе та як видавці використовують поверхні виявлення, встановлення та
довіри ClawHub. Щодо станів модерації та статусу облікового запису див.
[Модерація та безпека облікового запису](/clawhub/moderation). Щодо авторського права чи інших претензій
щодо прав див. [Запити щодо прав на контент](/uk/clawhub/content-rights).

## Дозволений контент

ClawHub вітає контент, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія | Дозволено, коли |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробників | Запис допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення. |
| Робочі процеси UI, даних та автоматизації | Обсяг чіткий, потрібні облікові дані явно вказані, а ризиковані дії включають перегляд, сухий запуск, попередній перегляд або шляхи підтвердження. |
| Захисна безпека, модерація та перевірка зловживань | Інструмент подано як призначений для авторизованої перевірки, він зберігає докази та чітко окреслює межі схвалення людиною. |
| Особисті або командні робочі процеси | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи. |
| Підтримувані каталоги | Кожен запис є окремим, корисним, точно описаним і належно підтримуваним. |

Контекст має значення. Та сама тема може бути прийнятною у вузькому захисному або
заснованому на згоді середовищі й неприйнятною, коли її упаковано як робочий процес зловживання.

## Заборонений контент

ClawHub не розміщує контент, головною метою якого є зловживання, обман, небезпечне
виконання або порушення прав.

| Категорія | Не дозволено |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Несанкціонований доступ або обхід безпеки | Обхід автентифікації, захоплення облікового запису, зловживання обмеженнями швидкості, захоплення живого виклику або агента, багаторазово використовуване викрадення сесії чи автоматичне схвалення потоків сполучення для несхвалених користувачів. |
| Зловживання платформою та обхід банів | Приховані облікові записи після банів, прогрівання або ферми облікових записів, фальшива взаємодія, автоматизація кількох облікових записів, масове публікування, спам-боти або автоматизація, створена для уникнення виявлення. |
| Шахрайство, афери та оманливі фінансові робочі процеси | Фальшиві сертифікати або рахунки-фактури, оманливі платіжні потоки, шахрайські розсилки, фальшиві соціальні докази, робочі процеси синтетичної ідентичності для шахрайства або інструменти витрачання/стягнення коштів без чіткого схвалення людиною. |
| Збагачення даних або стеження, що порушує приватність | Збирання контактів для спаму, доксинг, переслідування, витягування лідів у поєднанні з небажаними зверненнями, прихований моніторинг, біометричне зіставлення без згоди або використання витоків даних чи дампів зламів. |
| Імітація особи або маніпуляція ідентичністю без згоди | Заміна облич, цифрові двійники, клоновані інфлюенсери, фальшиві персони або інші інструменти, що використовуються для імітації особи чи введення в оману. |
| Відвертий сексуальний контент або генерація дорослого контенту з вимкненими запобіжниками | Генерація NSFW-зображень, відео або контенту; обгортки для дорослого контенту навколо сторонніх API; або записи, основною метою яких є відвертий сексуальний контент. |
| Приховані, небезпечні або оманливі вимоги до виконання | Обфусковані команди встановлення, інсталятори pipe-to-shell, як-от завантажений контент, що запускається через `sh` або `bash` без чіткої можливості перевірки, незадекларовані вимоги до секретів або приватних ключів, віддалене виконання `npx @latest` без чіткої можливості перевірки або метадані, що приховують, що насправді потрібно запису для запуску. |
| Матеріал, що порушує авторське право або інші права | Повторне публікування чужого skill, plugin, документації, бренд-активів або пропрієтарного коду без дозволу; порушення умов ліцензії; або імітація оригінального автора чи видавця. |

## Заборонена поведінка marketplace

ClawHub також перевіряє, як видавці використовують marketplace. Не використовуйте ClawHub для
маніпулювання виявленням, метриками, сигналами довіри, системами модерації або
увагою користувачів.

Заборонена поведінка marketplace включає:

- масове публікування великої кількості малозусильних, дублювальних, заповнювальних або
  машинно згенерованих записів, які, схоже, не мають реальної цінності для користувачів
- переповнення пошуку або поверхонь категорій майже ідентичними skills або plugins
- публікування сотень записів із мінімальним або відсутнім використанням, підтримкою, прозорістю
  джерела чи змістовною диференціацією
- штучне завищення встановлень, завантажень, зірок або інших метрик
  взаємодії через автоматизацію, цикли самовстановлення, фальшиві облікові записи, скоординовану
  активність, платну взаємодію або іншу неорганічну поведінку
- створення або ротація облікових записів для обходу модерації, банів, лімітів видавців або
  перевірки marketplace
- введення користувачів в оману щодо власності, джерела, можливостей, стану безпеки,
  вимог до встановлення або афіліації з іншим проєктом чи видавцем
- повторне завантаження контенту, який уже було приховано, видалено або заблоковано,
  без усунення базової проблеми

Масове публікування не є автоматично зловживанням. Великі каталоги прийнятні,
коли записи суттєво відрізняються, точно описані, підтримуються
та використовуються реальними користувачами. Великі каталоги стають проблемою довіри та безпеки, коли
обсяг поєднується з поверховими, дублювальними, оманливими, непідтримуваними або
штучно просунутими записами.

## Права на контент

Якщо ви вважаєте, що контент на ClawHub порушує ваше авторське право чи інші права, використовуйте
[Запити щодо прав на контент](/uk/clawhub/content-rights). Не використовуйте звичайні звіти marketplace
для претензій щодо авторського права чи прав, якщо запис також не є небезпечним,
зловмисним або оманливим.

## Перевірка та застосування правил

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали зловживань, звіти користувачів і
перевірку персоналом для виявлення небезпечного контенту або зловмисної поведінки публікації. Сигнал
сам по собі не доводить зловживання; він допомагає ClawHub вирішити, що потребує перевірки.

Ми можемо:

- приховати, затримати, видалити, м’яко видалити або, якщо це підтримується для типу ресурсу,
  безповоротно видалити записи, що порушують правила
- заблокувати завантаження або встановлення небезпечних релізів
- відкликати API-токени
- м’яко видалити пов’язаний контент
- обмежити доступ до публікації
- забанити повторних або серйозних порушників

Ми не гарантуємо застосування правил спочатку з попередженням для очевидних зловживань. Див.
[Модерація та безпека облікового запису](/clawhub/moderation) щодо звітів, утримань модерацією,
прихованих записів, банів і статусу облікового запису.
