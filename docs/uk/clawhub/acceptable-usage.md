# Прийнятне використання

ClawHub розміщує Skills, плагіни, пакети та метадані маркетплейсу для OpenClaw.
Використовуйте цю сторінку, щоб визначити, чи належать вміст або поведінка публікації до
ClawHub.

Ці правила стосуються того, що робить запис, що він просить користувачів запускати, як він
представляє себе та як видавці використовують поверхні ClawHub для пошуку, встановлення й
довіри. Щодо станів модерації та статусу облікового запису див.
[Модерація та безпека облікового запису](/clawhub/moderation). Щодо авторського права або інших претензій
щодо прав див. [Запити щодо прав на вміст](/uk/clawhub/content-rights).

## Дозволений вміст

ClawHub вітає вміст, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія                                       | Дозволено, коли                                                                                                                   |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробників                       | Запис допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення. |
| Робочі процеси UI, даних та автоматизації        | Обсяг чіткий, потрібні облікові дані вказані явно, а ризиковані дії мають шляхи перевірки, пробного запуску, попереднього перегляду або підтвердження. |
| Захисна безпека, модерація та перевірка зловживань | Інструмент подано як призначений для авторизованої перевірки, він зберігає докази та чітко визначає межі людського схвалення. |
| Особисті або командні робочі процеси             | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи. |
| Підтримувані каталоги                            | Кожен запис є окремим, корисним, точно описаним і належно підтримуваним. |

Контекст має значення. Та сама тема може бути прийнятною у вузькому захисному або
заснованому на згоді середовищі й неприйнятною, коли її упаковано як робочий процес для зловживань.

## Заборонена поведінка в маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не використовуйте ClawHub, щоб
маніпулювати пошуком, метриками, сигналами довіри, системами модерації або
увагою користувачів.

Заборонена поведінка в маркетплейсі охоплює:

- масову публікацію великої кількості низькоякісних, дублювальних, заповнювальних або
  машинно згенерованих записів, які, схоже, не мають реальної цінності для користувачів
- засмічення поверхонь пошуку або категорій майже ідентичними Skills чи плагінами
- публікацію сотень записів із незначним або відсутнім використанням, підтримкою, прозорістю
  джерела або змістовною диференціацією
- штучне збільшення встановлень, завантажень, зірок або інших метрик взаємодії
  через автоматизацію, цикли самовстановлення, фальшиві облікові записи, скоординовану
  активність, платну взаємодію або іншу неорганічну поведінку
- створення або ротацію облікових записів для обходу модерації, блокувань, лімітів видавця або
  перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, стану безпеки,
  вимог до встановлення або афіліації з іншим проєктом чи видавцем
- повторне завантаження вмісту, який уже було приховано, видалено або заблоковано,
  без виправлення основної проблеми

Публікація великого обсягу не є автоматично зловживанням. Великі каталоги прийнятні,
коли записи змістовно відрізняються, точно описані, підтримуються
та використовуються реальними користувачами. Великі каталоги стають проблемою довіри й безпеки, коли
обсяг поєднується з поверховими, дублювальними, оманливими, непідтримуваними або
штучно просуваними записами.

## Права на вміст

Якщо ви вважаєте, що вміст на ClawHub порушує ваші авторські або інші права, використовуйте
[Запити щодо прав на вміст](/uk/clawhub/content-rights). Не використовуйте звичайні звіти маркетплейсу
для претензій щодо авторських або інших прав, якщо запис також не є небезпечним,
зловмисним або оманливим.

## Перевірка та застосування правил

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали зловживань, звіти користувачів і
перевірку персоналом, щоб виявляти небезпечний вміст або зловживальну поведінку публікації. Сигнал
сам по собі не доводить зловживання; він допомагає ClawHub визначити, що потребує перевірки.

Ми можемо:

- приховувати, утримувати, видаляти, м’яко видаляти або, де це підтримується для типу ресурсу,
  остаточно видаляти записи, що порушують правила
- блокувати завантаження або встановлення небезпечних релізів
- відкликати API-токени
- м’яко видаляти пов’язаний вміст
- обмежувати доступ до публікації
- блокувати повторних або серйозних порушників

Ми не гарантуємо застосування правил із попередженням спочатку для очевидних зловживань. Див.
[Модерація та безпека облікового запису](/clawhub/moderation) щодо звітів, модераційних утримань,
прихованих записів, блокувань і статусу облікового запису.
