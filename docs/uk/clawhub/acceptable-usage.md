# Прийнятне використання

Ця сторінка описує типи Skills і контенту, які є прийнятними для ClawHub, а також робочі процеси зловживання, які він не розміщуватиме.

Ці правила навмисно практичні. Насамперед нас цікавлять наскрізні робочі процеси зловживання, а не лише ізольовані ключові слова. Якщо Skill створено для обходу захистів, зловживання платформами, шахрайства щодо людей, вторгнення в приватність або уможливлення поведінки без згоди, йому не місце на ClawHub.

## Нещодавні шаблони, які ми явно вважаємо прийнятними

- Робота з фронтендом і дизайн-системами, що використовує реальні компоненти, семантичні токени, доступні стани та протестовані користувацькі потоки.
- Композиція shadcn/ui, що використовує встановлені вихідні компоненти, псевдоніми проєкту та задокументовані варіанти замість одноразової розмітки.
- Перетворення UI5 JavaScript на TypeScript, яке зберігає коментарі, використовує конкретні типи UI5 і залишає згенеровані інтерфейси елементів керування придатними для ревʼю.
- Огляд захисної безпеки, інструменти модерації та підказки для виявлення зловживань, які показують докази й чітко зберігають межі людського схвалення.
- Автоматизація робочих процесів на основі згоди для особистих або командних облікових записів з явними обліковими даними, прозорим налаштуванням і режимами сухого запуску або попереднього перегляду.
- Документація, інструкції з міграції, утиліти для розробників і тестові фікстури, обмежені програмним забезпеченням, яке вони підтримують.

## Неприйнятно

- Робочі процеси обходу безпеки або несанкціонованого доступу.
  - Приклади: обхід автентифікації, захоплення облікового запису, обхід CAPTCHA, обхід Cloudflare або антибот-захисту, обхід обмежень швидкості, приховане збирання даних, спроєктоване для подолання захистів, захоплення живого дзвінка або агента, повторно використовуване викрадення сесії, автоматичне схвалення потоків спарювання для несхвалених користувачів.

- Зловживання платформами та обхід банів.
  - Приклади: приховані облікові записи після банів, «прогрівання» або ферми облікових записів, фальшива взаємодія, нарощування карми або підписників, автоматизація багатьох облікових записів, масові публікації, спам-боти, автоматизація маркетплейсів або соціальних платформ, створена для уникнення виявлення.

- Шахрайство, скамерство та оманливі фінансові робочі процеси.
  - Приклади: фальшиві сертифікати, фальшиві рахунки, оманливі платіжні потоки, скам-розсилки, фальшиві соціальні докази, інструменти, що дають змогу витрачати або стягувати кошти без чіткого людського схвалення та прозорих контролів, або робочі процеси синтетичної ідентичності, створені для відкриття облікових записів з метою шахрайства.

- Збирання, збагачення даних або стеження, що порушують приватність.
  - Приклади: масове збирання контактних даних для спаму, доксинг, переслідування, витягування лідів у поєднанні з небажаними зверненнями, прихований моніторинг, пошук облич або біометричне зіставлення без чіткої згоди, або купівля, публікація, завантаження чи операціоналізація витеклих даних або дампів зламів.

- Імперсонація без згоди або оманливе маніпулювання ідентичністю.
  - Приклади: заміна облич, цифрові двійники, фальшиві персони, клоновані інфлюенсери або інші інструменти маніпулювання ідентичністю, що використовуються для імперсонації чи введення в оману.

- Відвертий сексуальний контент і генерація дорослого контенту з вимкненими захистами.
  - Приклади: генерація NSFW-зображень/відео/контенту, обгортки для дорослого контенту навколо сторонніх API або Skills, основною метою яких є відвертий сексуальний контент.

- Приховані, небезпечні або оманливі вимоги до виконання.
  - Приклади: обфусковані команди встановлення, `curl | sh`, незадекларовані вимоги до секретів, незадеклароване використання приватних ключів, віддалене виконання `npx @latest` без чіткої можливості ревʼю, оманливі метадані, що приховують, що насправді потрібно Skill для запуску.

## Нещодавні шаблони, які ми явно вважаємо неприйнятними

- «Створити приховані облікові записи продавців після банів на маркетплейсі».
- «Змінити спарювання Telegram, щоб несхвалені користувачі автоматично отримували коди спарювання».
- «Розвивати облікові записи Reddit/Twitter за допомогою невиявної автоматизації».
- «Генерувати професійні сертифікати або рахунки для довільного використання».
- «Генерувати NSFW-контент із вимкненими перевірками безпеки».
- «Збирати ліди, збагачувати контакти та запускати холодні розсилки у масштабі».
- «Купувати, публікувати або завантажувати витеклі дані чи дампи зламів».
- «Масово створювати електронні або соціальні облікові записи із синтетичними ідентичностями чи розвʼязанням CAPTCHA».

## Примітки для рецензентів

- Контекст має значення. Та сама тема може бути легітимною у вузькому захисному або заснованому на згоді середовищі й неприйнятною, коли її запаковано як робочий процес зловживання.
- Нам слід схилятися до дії, коли Skill явно оптимізовано для обходу, обману або використання без згоди.
- Повторні завантаження в цих категоріях є підставою для приховування контенту та бану облікового запису.

## Застосування правил

- Ми можемо приховувати, видаляти або остаточно видаляти Skills, що порушують правила.
- Ми можемо відкликати токени, мʼяко видаляти повʼязаний контент і банити повторних або серйозних порушників.
- Ми не гарантуємо попередження перед застосуванням заходів у випадках очевидного зловживання.
