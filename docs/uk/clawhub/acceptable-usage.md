# Прийнятне використання

ClawHub розміщує Skills, плагіни, пакети й метадані маркетплейсу для OpenClaw.
Використовуйте цю сторінку, щоб визначити, чи належать контент або поведінка
публікації до ClawHub.

Ці правила застосовуються до того, що робить запис, що він просить користувачів
запускати, як він представляє себе та як видавці використовують поверхні
пошуку, встановлення й довіри ClawHub. Щодо станів модерації та статусу
облікового запису див. [Модерація та безпека облікового запису](/uk/clawhub/moderation).
Щодо претензій про авторське право або інші права див.
[Запити щодо прав на контент](/uk/clawhub/content-rights).

## Дозволений контент

ClawHub вітає контент, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія                                         | Дозволено, коли                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробників                           | Запис допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення.                                               |
| UI, дані та робочі процеси автоматизації               | Обсяг чіткий, потрібні облікові дані явно зазначені, а ризиковані дії мають шляхи перевірки, пробного запуску, попереднього перегляду або підтвердження. |
| Захисна безпека, модерація та перевірка зловживань | Інструмент подано як призначений для авторизованої перевірки, він зберігає докази та чітко визначає межі людського схвалення.                          |
| Особисті або командні робочі процеси                       | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи.                                            |
| Підтримувані каталоги                              | Кожен запис є окремим, корисним, точно описаним і розумно підтримуваним.                                                |

Контекст має значення. Та сама тема може бути прийнятною у вузькому захисному
сценарії або сценарії на основі згоди й неприйнятною, коли її упаковано як
робочий процес для зловживань.

## Заборонена поведінка в маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не використовуйте
ClawHub для маніпулювання пошуком, метриками, сигналами довіри, системами
модерації або увагою користувачів.

Заборонена поведінка в маркетплейсі включає:

- масову публікацію великої кількості низькоякісних, дублювальних, шаблонних або
  машинно згенерованих записів, які не мають очевидної реальної цінності для користувачів
- засмічення поверхонь пошуку або категорій майже ідентичними Skills чи плагінами
- публікацію сотень записів із малим використанням або без нього, без підтримки,
  ясності джерела чи змістовної відмінності
- штучне завищення встановлень, завантажень, зірок або інших метрик
  взаємодії через автоматизацію, цикли самовстановлення, фальшиві облікові записи, скоординовану
  активність, оплачувану взаємодію або іншу неорганічну поведінку
- створення або ротацію облікових записів для обходу модерації, блокувань, лімітів видавця або
  перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, стану безпеки,
  вимог до встановлення або афіліації з іншим проєктом чи видавцем
- повторне завантаження контенту, який уже було приховано, видалено або заблоковано,
  без виправлення базової проблеми

Публікація у великому обсязі не є автоматично зловживанням. Великі каталоги
прийнятні, коли записи змістовно відрізняються, точно описані, підтримуються
та використовуються реальними користувачами. Великі каталоги стають проблемою
довіри й безпеки, коли обсяг поєднується з поверховими, дублювальними,
оманливими, непідтримуваними або штучно просуваними записами.

## Права на контент

Якщо ви вважаєте, що контент на ClawHub порушує ваше авторське право або інші
права, використовуйте [Запити щодо прав на контент](/uk/clawhub/content-rights).
Не використовуйте звичайні звіти маркетплейсу для претензій щодо авторського
права або прав, якщо запис також не є небезпечним, шкідливим чи оманливим.

## Перевірка та застосування правил

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали
зловживань, звіти користувачів і перевірку персоналом, щоб виявляти небезпечний
контент або зловживальну поведінку публікації. Сигнал сам по собі не доводить
зловживання; він допомагає ClawHub визначити, що потребує перевірки.

Ми можемо:

- приховувати, утримувати, видаляти, м’яко видаляти або, де це підтримується для типу ресурсу,
  остаточно видаляти записи, що порушують правила
- блокувати завантаження або встановлення небезпечних релізів
- відкликати API-токени
- м’яко видаляти пов’язаний контент
- обмежувати доступ до публікації
- блокувати повторних або серйозних порушників

Ми не гарантуємо застосування правил спершу з попередженням для очевидних
зловживань. Див. [Модерація та безпека облікового запису](/uk/clawhub/moderation)
щодо звітів, модераційних утримань, прихованих записів, блокувань і статусу
облікового запису.
