# Прийнятне використання

ClawHub розміщує Skills, плагіни, пакети та метадані маркетплейсу для OpenClaw.
Використовуйте цю сторінку, щоб визначити, чи належить контент або поведінка публікації
до ClawHub.

Ці правила застосовуються до того, що робить оголошення, що воно просить користувачів запускати, як воно
представляє себе та як видавці використовують поверхні виявлення, встановлення й довіри ClawHub. Щодо станів модерації та статусу облікового запису див.
[Модерація та безпека облікового запису](/clawhub/moderation). Щодо авторського права або інших правових
претензій див. [Запити щодо прав на контент](/uk/clawhub/content-rights).

## Дозволений контент

ClawHub вітає контент, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія                                         | Дозволено, коли                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробників                           | Оголошення допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення.                                               |
| Робочі процеси UI, даних та автоматизації               | Обсяг чіткий, необхідні облікові дані явно вказані, а ризиковані дії мають шляхи перегляду, пробного запуску, попереднього перегляду або підтвердження. |
| Захисна безпека, модерація та перевірка зловживань | Інструмент подано як призначений для авторизованої перевірки, він зберігає докази та чітко визначає межі схвалення людиною.                          |
| Особисті або командні робочі процеси                       | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи.                                            |

Контекст має значення. Та сама тема може бути прийнятною у вузькому захисному або
заснованому на згоді середовищі й неприйнятною, коли її упаковано як робочий процес для зловживань.

## Заборонена поведінка в маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не використовуйте ClawHub для
маніпулювання виявленням, метриками, сигналами довіри, системами модерації або
увагою користувачів.

Заборонена поведінка в маркетплейсі включає:

- масову публікацію великої кількості низькоякісних, дублікативних, шаблонних або
  згенерованих машиною оголошень, які не мають очевидної реальної цінності для користувачів
- переповнення пошуку або категорій майже ідентичними Skills чи плагінами
- публікацію сотень оголошень із незначним використанням або без нього, без підтримки, чіткості
  джерела або змістовної диференціації
- штучне збільшення встановлень, завантажень, зірок або інших метрик
  взаємодії через автоматизацію, цикли самовстановлення, фальшиві облікові записи, скоординовану
  активність, оплачену взаємодію або іншу неорганічну поведінку
- створення або ротацію облікових записів для обходу модерації, блокувань, обмежень видавця або
  перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, стану безпеки,
  вимог до встановлення або афіліації з іншим проєктом чи видавцем
- повторне завантаження контенту, який уже було приховано, видалено або заблоковано,
  без виправлення базової проблеми

Публікація великих обсягів не є автоматично зловживанням. Великі каталоги прийнятні,
коли оголошення суттєво відрізняються, точно описані, підтримуються
та використовуються реальними користувачами. Великі каталоги стають проблемою довіри й безпеки, коли
обсяг поєднується з поверховими, дублікативними, оманливими, непідтримуваними або
штучно просуваними оголошеннями.

## Права на контент

Якщо ви вважаєте, що контент на ClawHub порушує ваше авторське право або інші права, використовуйте
[Запити щодо прав на контент](/uk/clawhub/content-rights). Не використовуйте звичайні звіти маркетплейсу
для претензій щодо авторського права або прав, якщо оголошення також не є небезпечним,
зловмисним або оманливим.

## Перевірка та правозастосування

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали зловживань, звіти користувачів і
перевірку співробітниками, щоб виявляти небезпечний контент або зловживальну поведінку публікації. Сигнал
сам по собі не доводить зловживання; він допомагає ClawHub визначити, що потребує перевірки.

Ми можемо:

- приховувати, утримувати, видаляти, м’яко видаляти або, якщо це підтримується для типу ресурсу,
  остаточно видаляти оголошення, що порушують правила
- блокувати завантаження або встановлення небезпечних релізів
- відкликати API-токени
- м’яко видаляти пов’язаний контент
- обмежувати доступ до публікації
- блокувати повторних або серйозних порушників

Ми не гарантуємо правозастосування з попередженням спочатку для очевидних зловживань. Див.
[Модерація та безпека облікового запису](/clawhub/moderation) щодо звітів, модераційних утримань,
прихованих оголошень, блокувань і статусу облікового запису.
