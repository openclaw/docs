# Прийнятне використання

ClawHub розміщує Skills, плагіни, пакети та метадані маркетплейсу для OpenClaw.
Використовуйте цю сторінку, щоб визначити, чи належить контент або поведінка публікації до
ClawHub.

Ці правила застосовуються до того, що робить список, що він просить користувачів запускати, як він
представляє себе та як видавці використовують поверхні виявлення, встановлення й
довіри ClawHub. Про стани модерації та статус облікового запису див.
[Модерація та безпека облікового запису](/uk/clawhub/moderation). Про авторські права або інші претензії щодо прав
див. [Запити щодо прав на контент](/uk/clawhub/content-rights).

## Дозволений контент

ClawHub вітає контент, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія                                         | Дозволено, коли                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробників                           | Список допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення.                                               |
| Робочі процеси UI, даних та автоматизації               | Обсяг чіткий, потрібні облікові дані явно вказані, а ризиковані дії мають шляхи перегляду, пробного запуску, попереднього перегляду або підтвердження. |
| Захисна безпека, модерація та розгляд зловживань | Інструмент подано як призначений для авторизованого розгляду, він зберігає докази та чітко підтримує межі людського схвалення.                          |
| Особисті або командні робочі процеси                       | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи.                                            |
| Підтримувані каталоги                              | Кожен список є окремим, корисним, точно описаним і належно підтримуваним.                                                |

Контекст має значення. Та сама тема може бути прийнятною у вузькому захисному або
заснованому на згоді середовищі та неприйнятною, коли її упаковано як робочий процес зловживання.

виконання або порушення прав.

## Заборонена поведінка в маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не використовуйте ClawHub, щоб
маніпулювати виявленням, метриками, сигналами довіри, системами модерації або
увагою користувачів.

Заборонена поведінка в маркетплейсі включає:

- масову публікацію великої кількості низькоякісних, дублювальних, шаблонних або
  машинно згенерованих списків, які, схоже, не мають реальної цінності для користувачів
- переповнення пошуку або поверхонь категорій майже ідентичними Skills чи плагінами
- публікацію сотень списків із незначним або відсутнім використанням, підтримкою, прозорістю джерела
  чи змістовною відмінністю
- штучне завищення встановлень, завантажень, зірок або інших метрик
  взаємодії через автоматизацію, цикли самовстановлення, фальшиві облікові записи, скоординовану
  активність, оплачену взаємодію або іншу неорганічну поведінку
- створення або ротацію облікових записів для обходу модерації, банів, обмежень видавців або
  перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, стану безпеки,
  вимог до встановлення або зв’язку з іншим проєктом чи видавцем
- повторне завантаження контенту, який уже був прихований, видалений або заблокований,
  без виправлення основної проблеми

Публікація великого обсягу сама по собі не є зловживанням. Великі каталоги прийнятні,
коли списки змістовно відрізняються, точно описані, підтримуються
та використовуються реальними користувачами. Великі каталоги стають проблемою довіри й безпеки, коли
обсяг поєднується з поверхневими, дублювальними, оманливими, непідтримуваними або
штучно просунутими списками.

## Права на контент

Якщо ви вважаєте, що контент на ClawHub порушує ваші авторські або інші права, використовуйте
[Запити щодо прав на контент](/uk/clawhub/content-rights). Не використовуйте звичайні звіти маркетплейсу
для авторських або інших претензій щодо прав, якщо список також не є небезпечним,
зловмисним або оманливим.

## Перевірка та застосування правил

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали зловживань, звіти користувачів і
перевірку співробітниками, щоб виявляти небезпечний контент або зловживальну поведінку публікації. Сигнал
сам по собі не доводить зловживання; він допомагає ClawHub вирішити, що потребує перевірки.

Ми можемо:

- приховувати, утримувати, видаляти, м’яко видаляти або, якщо це підтримується для типу ресурсу,
  остаточно видаляти списки, що порушують правила
- блокувати завантаження або встановлення небезпечних релізів
- відкликати API-токени
- м’яко видаляти пов’язаний контент
- обмежувати доступ до публікації
- банити повторних або серйозних порушників

Ми не гарантуємо застосування правил спочатку з попередженням для очевидних зловживань. Див.
[Модерація та безпека облікового запису](/uk/clawhub/moderation) щодо звітів, модераційних утримань,
прихованих списків, банів і статусу облікового запису.
