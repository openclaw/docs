# Прийнятне використання

ClawHub розміщує Skills, плагіни, пакети та метадані маркетплейсу для OpenClaw.
Використовуйте цю сторінку, щоб визначити, чи належить вміст або поведінка
публікації до ClawHub.

Ці правила стосуються того, що робить сторінка розміщення, що вона просить
користувачів запускати, як вона себе представляє та як видавці використовують
поверхні ClawHub для виявлення, встановлення й довіри. Стани модерації та
статус облікового запису див. у розділі [Модерація та безпека облікового запису](/clawhub/moderation).
Щодо претензій про авторські права або інші права див.
[Запити щодо прав на вміст](/uk/clawhub/content-rights).

## Дозволений вміст

ClawHub вітає вміст, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія                                        | Дозволено, коли                                                                                                                   |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробників                       | Сторінка розміщення допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення. |
| Робочі процеси UI, даних і автоматизації         | Обсяг чіткий, потрібні облікові дані вказані явно, а ризикові дії містять шляхи перегляду, пробного запуску, попереднього перегляду або підтвердження. |
| Захисна безпека, модерація та перевірка зловживань | Інструмент подано як призначений для авторизованої перевірки, він зберігає докази й чітко підтримує межі людського схвалення. |
| Особисті або командні робочі процеси             | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи. |
| Підтримувані каталоги                            | Кожна сторінка розміщення є окремою, корисною, точно описаною та належно підтримуваною. |

Контекст має значення. Та сама тема може бути прийнятною у вузькому захисному
або заснованому на згоді контексті й неприйнятною, коли її упаковано як робочий
процес для зловживань.

## Заборонена поведінка в маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не
використовуйте ClawHub для маніпулювання виявленням, метриками, сигналами
довіри, системами модерації або увагою користувачів.

Заборонена поведінка в маркетплейсі включає:

- масову публікацію великої кількості низькоякісних, дублювальних, шаблонних
  або машинно згенерованих сторінок розміщення, які не мають очевидної
  реальної цінності для користувачів
- засмічення пошуку або категорій майже ідентичними Skills чи плагінами
- публікацію сотень сторінок розміщення з малою або відсутньою кількістю
  використань, підтримкою, прозорістю джерела чи змістовною відмінністю
- штучне завищення встановлень, завантажень, зірок або інших метрик
  залучення через автоматизацію, цикли самовстановлення, фальшиві облікові
  записи, скоординовану активність, оплачене залучення або іншу неорганічну
  поведінку
- створення або ротацію облікових записів для обходу модерації, блокувань,
  обмежень видавця або перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, стану
  безпеки, вимог до встановлення або пов’язаності з іншим проєктом чи видавцем
- повторне завантаження вмісту, який уже було приховано, видалено або
  заблоковано, без виправлення основної проблеми

Публікація великого обсягу не є автоматично зловживанням. Великі каталоги
прийнятні, коли сторінки розміщення суттєво відрізняються, точно описані,
підтримуються та використовуються реальними користувачами. Великі каталоги
стають проблемою довіри й безпеки, коли обсяг поєднується з поверховими,
дублювальними, оманливими, непідтримуваними або штучно просуваними сторінками
розміщення.

## Права на вміст

Якщо ви вважаєте, що вміст на ClawHub порушує ваші авторські або інші права,
скористайтеся [Запитами щодо прав на вміст](/uk/clawhub/content-rights). Не
використовуйте звичайні звіти маркетплейсу для претензій щодо авторських або
інших прав, якщо сторінка розміщення не є також небезпечною, шкідливою або
оманливою.

## Перевірка та застосування правил

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали
зловживань, звіти користувачів і перевірку персоналом для виявлення
небезпечного вмісту або зловживальної поведінки публікації. Сигнал сам по собі
не доводить зловживання; він допомагає ClawHub вирішити, що потребує перевірки.

Ми можемо:

- приховати, утримати, видалити, м’яко видалити або, якщо це підтримується для
  типу ресурсу, остаточно видалити сторінки розміщення, що порушують правила
- заблокувати завантаження або встановлення небезпечних випусків
- відкликати API-токени
- м’яко видалити пов’язаний вміст
- обмежити доступ до публікації
- заблокувати повторних або серйозних порушників

Ми не гарантуємо застосування правила «спочатку попередження» для очевидних
зловживань. Див. [Модерація та безпека облікового запису](/clawhub/moderation)
щодо звітів, модераційних утримань, прихованих сторінок розміщення, блокувань
і статусу облікового запису.
