# Прийнятне використання

Ця сторінка описує, які види навичок і контенту ClawHub приймає, а також які зловживальні робочі процеси він не розміщуватиме.

Ці правила навмисно практичні. Нас найбільше цікавлять наскрізні робочі процеси зловживань, а не лише ізольовані ключові слова. Якщо навичка створена для обходу захистів, зловживання платформами, шахрайства проти людей, порушення приватності або уможливлення поведінки без згоди, їй не місце на ClawHub.

## Нещодавні патерни, які ми явно приймаємо

- Робота з фронтендом і дизайн-системами, що використовує реальні компоненти, семантичні токени, доступні стани та протестовані користувацькі потоки.
- Компонування shadcn/ui, що використовує встановлені вихідні компоненти, псевдоніми проєкту та задокументовані варіанти замість одноразової розмітки.
- Перетворення UI5 JavaScript на TypeScript, яке зберігає коментарі, використовує конкретні типи UI5 і залишає згенеровані інтерфейси елементів керування придатними для рев’ю.
- Оборонний аналіз безпеки, інструменти модерації та промпти для виявлення зловживань, які показують докази й чітко зберігають межі людського схвалення.
- Автоматизація робочих процесів на основі згоди для особистих або командних облікових записів із явними обліковими даними, прозорим налаштуванням і режимами пробного запуску або попереднього перегляду.
- Документація, інструкції з міграції, утиліти для розробників і тестові фікстури, обмежені програмним забезпеченням, яке вони підтримують.

## Неприйнятно

- Робочі процеси обходу безпеки або несанкціонованого доступу.
  - Приклади: обхід автентифікації, захоплення облікового запису, обхід CAPTCHA, обхід Cloudflare або антибот-захисту, обхід обмежень частоти, прихований скрейпінг, спроєктований для подолання захистів, захоплення живого дзвінка або агента, багаторазова крадіжка сесій, автоматичне схвалення потоків сполучення для несхвалених користувачів.

- Зловживання платформами та обхід банів.
  - Приклади: приховані облікові записи після банів, прогрівання або фермерство облікових записів, фальшива залученість, нарощування карми або підписників, автоматизація багатьох облікових записів, масове публікування, спам-боти, автоматизація маркетплейсів або соціальних платформ, створена для уникнення виявлення.

- Шахрайство, сками та оманливі фінансові робочі процеси.
  - Приклади: фальшиві сертифікати, фальшиві рахунки, оманливі платіжні потоки, шахрайські розсилки, фальшиве соціальне підтвердження, інструменти, що уможливлюють витрати або списання без чіткого людського схвалення та прозорих засобів контролю, або робочі процеси із синтетичною ідентичністю, створені для відкриття облікових записів для шахрайства.

- Скрейпінг, збагачення даних або стеження, що порушують приватність.
  - Приклади: масовий скрейпінг контактних даних для спаму, доксинг, переслідування, витягування лідів разом із небажаними зверненнями, прихований моніторинг, пошук облич або біометричне зіставлення без чіткої згоди, або купівля, публікація, завантаження чи операціоналізація витоків даних або дампів зламів.

- Імперсонація без згоди або оманлива маніпуляція ідентичністю.
  - Приклади: заміна облич, цифрові двійники, фальшиві персоналії, клоновані інфлюенсери або інші інструменти маніпуляції ідентичністю, що використовуються для імперсонації або введення в оману.

- Відвертий сексуальний контент і генерація дорослого контенту з вимкненими запобіжниками.
  - Приклади: генерація NSFW-зображень/відео/контенту, обгортки для дорослого контенту навколо сторонніх API або навички, основна мета яких — відвертий сексуальний контент.

- Приховані, небезпечні або оманливі вимоги до виконання.
  - Приклади: обфусковані команди встановлення, `curl | sh`, незадекларовані вимоги до секретів, незадеклароване використання приватного ключа, віддалене виконання `npx @latest` без чіткої можливості рев’ю, оманливі метадані, що приховують, що навичці справді потрібно для запуску.

## Нещодавні патерни, які ми явно не приймаємо

- “Створити приховані облікові записи продавців після банів на маркетплейсі.”
- “Змінити сполучення Telegram так, щоб несхвалені користувачі автоматично отримували коди сполучення.”
- “Розвивати облікові записи Reddit/Twitter за допомогою невиявної автоматизації.”
- “Генерувати професійні сертифікати або рахунки для довільного використання.”
- “Генерувати NSFW-контент із вимкненими перевірками безпеки.”
- “Скрейпити ліди, збагачувати контакти та запускати холодні звернення у великому масштабі.”
- “Купувати, публікувати або завантажувати витоки даних чи дампи зламів.”
- “Масово створювати електронні або соціальні облікові записи із синтетичними ідентичностями чи розв’язуванням CAPTCHA.”

## Примітки для рев’юерів

- Контекст має значення. Та сама тема може бути легітимною у вузькому оборонному або заснованому на згоді середовищі й неприйнятною, коли її запаковано як робочий процес зловживання.
- Ми маємо схилятися до дії, коли навичка явно оптимізована для обходу, обману або використання без згоди.
- Повторні завантаження в цих категоріях є підставою для приховування контенту й бану облікового запису.

## Застосування правил

- Ми можемо приховувати, видаляти або остаточно видаляти навички, що порушують правила.
- Ми можемо відкликати токени, м’яко видаляти пов’язаний контент і банити повторних або серйозних порушників.
- Ми не гарантуємо застосування правила “спершу попередження” для очевидних зловживань.
