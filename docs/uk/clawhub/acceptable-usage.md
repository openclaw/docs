# Прийнятне використання

ClawHub розміщує Skills, Plugin-и, пакети та метадані маркетплейса для OpenClaw.
Скористайтеся цією сторінкою, щоб вирішити, чи належать вміст або поведінка публікації до
ClawHub.

Ці правила застосовуються до того, що робить запис, що він просить користувачів запускати, як він
представляє себе та як видавці використовують поверхні виявлення, встановлення й
довіри ClawHub. Щодо станів модерації та статусу облікового запису див.
[Модерація та безпека облікового запису](/clawhub/moderation). Щодо претензій про авторські права або інші права
див. [Запити щодо прав на вміст](/uk/clawhub/content-rights).

## Дозволений вміст

ClawHub вітає вміст, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія | Дозволено, коли |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробників | Запис допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення. |
| Робочі процеси UI, даних та автоматизації | Обсяг чіткий, потрібні облікові дані явно вказані, а ризиковані дії мають шляхи перегляду, пробного запуску, попереднього перегляду або підтвердження. |
| Захисна безпека, модерація та перевірка зловживань | Інструмент позиціонується для авторизованої перевірки, зберігає докази та чітко визначає межі людського схвалення. |
| Особисті або командні робочі процеси | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи. |
| Підтримувані каталоги | Кожен запис є окремим, корисним, точно описаним і розумно підтримуваним. |

Контекст має значення. Та сама тема може бути прийнятною у вузькому захисному або
заснованому на згоді середовищі та неприйнятною, коли її запаковано як робочий процес зловживання.

## Заборонений вміст

ClawHub не розміщує вміст, головною метою якого є зловживання, обман, небезпечне
виконання або порушення прав.

| Категорія | Не дозволено |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Несанкціонований доступ або обхід безпеки | Обхід автентифікації, захоплення облікового запису, зловживання обмеженнями частоти, захоплення живого виклику або агента, повторно використовуване викрадення сеансу або автоматичне схвалення потоків спарювання для несхвалених користувачів. |
| Зловживання платформою та обхід блокувань | Приховані облікові записи після блокувань, прогрівання або ферми облікових записів, фальшива взаємодія, автоматизація багатьох облікових записів, масові публікації, спам-боти або автоматизація, створена для уникнення виявлення. |
| Шахрайство, афери та оманливі фінансові робочі процеси | Фальшиві сертифікати або рахунки-фактури, оманливі платіжні потоки, шахрайські звернення, фальшивий соціальний доказ, робочі процеси синтетичної ідентичності для шахрайства або інструменти витрачання/стягнення коштів без чіткого людського схвалення. |
| Збагачення даних або стеження з порушенням приватності | Збирання контактів для спаму, доксинг, переслідування, отримання лідів у поєднанні з небажаними зверненнями, прихований моніторинг, біометричне зіставлення без згоди або використання витеклих даних чи дампів зламів. |
| Імітація особи або маніпуляція ідентичністю без згоди | Заміна облич, цифрові двійники, клоновані інфлюенсери, фальшиві персони або інші інструменти, що використовуються для імітації чи введення в оману. |
| Відверто сексуальний вміст або генерація дорослого вмісту з вимкненою безпекою | Генерація NSFW-зображень, відео або вмісту; обгортки дорослого вмісту навколо сторонніх API; або записи, основною метою яких є відверто сексуальний вміст. |
| Приховані, небезпечні або оманливі вимоги до виконання | Обфусковані команди встановлення, інсталятори pipe-to-shell, як-от завантажений вміст, запущений через `sh` або `bash` без чіткої можливості перевірки, незадекларовані вимоги до секретів або приватних ключів, віддалене виконання `npx @latest` без чіткої можливості перевірки або метадані, що приховують, що насправді потрібно запису для запуску. |
| Матеріали, що порушують авторські або інші права | Повторна публікація чужих Skills, Plugin-ів, документації, бренд-активів або пропрієтарного коду без дозволу; порушення умов ліцензії; або імітація оригінального автора чи видавця. |

## Заборонена поведінка в маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не використовуйте ClawHub для
маніпулювання виявленням, метриками, сигналами довіри, системами модерації або увагою
користувачів.

Заборонена поведінка в маркетплейсі включає:

- масову публікацію великої кількості низькоякісних, дублювальних, заповнювальних або
  згенерованих машиною записів, які, схоже, не мають реальної цінності для користувачів
- заповнення пошуку або поверхонь категорій майже ідентичними Skills чи Plugin-ами
- публікацію сотень записів із малим або відсутнім використанням, підтримкою, ясністю
  джерела або значущою диференціацією
- штучне збільшення встановлень, завантажень, зірок або інших метрик
  взаємодії через автоматизацію, цикли самовстановлення, фальшиві облікові записи, скоординовану
  активність, платну взаємодію або іншу неорганічну поведінку
- створення або ротацію облікових записів для обходу модерації, блокувань, обмежень видавця або
  перевірки маркетплейса
- введення користувачів в оману щодо власності, джерела, можливостей, стану безпеки,
  вимог до встановлення або афіліації з іншим проєктом чи видавцем
- повторне завантаження вмісту, який уже було приховано, видалено або заблоковано,
  без виправлення основної проблеми

Публікація великого обсягу не є автоматично зловживанням. Великі каталоги прийнятні,
коли записи суттєво відрізняються, точно описані, підтримуються
та використовуються реальними користувачами. Великі каталоги стають проблемою довіри та безпеки, коли
обсяг поєднується з поверхневими, дублювальними, оманливими, непідтримуваними або
штучно просуваними записами.

## Права на вміст

Якщо ви вважаєте, що вміст на ClawHub порушує ваші авторські або інші права, використовуйте
[Запити щодо прав на вміст](/uk/clawhub/content-rights). Не використовуйте звичайні звіти маркетплейса
для претензій щодо авторських або інших прав, якщо запис також не є небезпечним,
зловмисним або оманливим.

## Перевірка та застосування правил

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали зловживань, звіти користувачів і
перевірку персоналом для виявлення небезпечного вмісту або зловживальної поведінки публікації. Сигнал
сам по собі не доводить зловживання; він допомагає ClawHub вирішити, що потребує перевірки.

Ми можемо:

- приховувати, утримувати, видаляти, м’яко видаляти або, коли це підтримується для типу ресурсу,
  остаточно видаляти записи, що порушують правила
- блокувати завантаження або встановлення небезпечних релізів
- відкликати API-токени
- м’яко видаляти пов’язаний вміст
- обмежувати доступ до публікації
- блокувати повторних або серйозних порушників

Ми не гарантуємо застосування правил спочатку з попередженням для очевидних зловживань. Див.
[Модерація та безпека облікового запису](/clawhub/moderation) щодо звітів, модераційних утримань,
прихованих записів, блокувань і статусу облікового запису.
