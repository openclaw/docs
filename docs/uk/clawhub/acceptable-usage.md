# Прийнятне використання

ClawHub розміщує Skills, Plugin, пакети та метадані маркетплейсу для OpenClaw.
Використовуйте цю сторінку, щоб визначити, чи належить вміст або поведінка публікації
до ClawHub.

Ці правила застосовуються до того, що робить запис, що він просить користувачів запускати, як він
представляє себе та як видавці використовують поверхні виявлення, встановлення й довіри
ClawHub. Стани модерації та статус облікового запису див. у
[Модерація та безпека облікового запису](/clawhub/moderation). Для претензій щодо авторського права або інших прав
див. [Запити щодо прав на вміст](/uk/clawhub/content-rights).

## Дозволений вміст

ClawHub вітає вміст, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія                                        | Дозволено, коли                                                                                                                   |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробників                       | Запис допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення. |
| Робочі процеси UI, даних і автоматизації         | Обсяг чіткий, потрібні облікові дані явно вказані, а ризиковані дії мають шляхи перегляду, пробного запуску, попереднього перегляду або підтвердження. |
| Оборонна безпека, модерація та розгляд зловживань | Інструмент подано як призначений для авторизованого розгляду, він зберігає докази й чітко визначає межі людського схвалення.      |
| Особисті або командні робочі процеси             | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи.                               |
| Підтримувані каталоги                            | Кожен запис є окремим, корисним, точно описаним і належно підтримуваним.                                                          |

Контекст має значення. Та сама тема може бути прийнятною у вузькому оборонному або
заснованому на згоді середовищі й неприйнятною, коли її упаковано як робочий процес зловживання.

## Заборонена поведінка в маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не використовуйте ClawHub для
маніпулювання виявленням, метриками, сигналами довіри, системами модерації або
увагою користувачів.

Заборонена поведінка в маркетплейсі включає:

- масову публікацію великої кількості низькоякісних, дублювальних, заповнювальних або
  машинно згенерованих записів, які, схоже, не мають реальної цінності для користувачів
- засмічення поверхонь пошуку або категорій майже ідентичними Skills чи Plugin
- публікацію сотень записів із незначним використанням або без нього, без підтримки, прозорості
  джерела чи змістовної відмінності
- штучне завищення встановлень, завантажень, зірок або інших метрик взаємодії
  через автоматизацію, цикли самостійного встановлення, фейкові облікові записи, скоординовану
  активність, оплачену взаємодію або іншу неорганічну поведінку
- створення або ротацію облікових записів для обходу модерації, банів, обмежень видавця або
  перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, стану безпеки,
  вимог до встановлення або зв’язку з іншим проєктом чи видавцем
- повторне завантаження вмісту, який уже було приховано, видалено або заблоковано,
  без усунення базової проблеми

Публікація великих обсягів не є автоматично зловживанням. Великі каталоги прийнятні,
коли записи суттєво відрізняються, точно описані, підтримуються
та використовуються реальними користувачами. Великі каталоги стають проблемою довіри й безпеки, коли
обсяг поєднується з поверховими, дублювальними, оманливими, непідтримуваними або
штучно просунутими записами.

## Права на вміст

Якщо ви вважаєте, що вміст у ClawHub порушує ваше авторське право або інші права, використовуйте
[Запити щодо прав на вміст](/uk/clawhub/content-rights). Не використовуйте звичайні звіти маркетплейсу
для претензій щодо авторського права або прав, якщо запис також не є небезпечним,
шкідливим або оманливим.

## Перевірка та застосування правил

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали зловживань, звіти користувачів і
перевірку персоналом, щоб виявляти небезпечний вміст або зловживальну поведінку публікації. Сигнал
сам по собі не доводить зловживання; він допомагає ClawHub вирішити, що потребує перевірки.

Ми можемо:

- приховувати, утримувати, видаляти, м’яко видаляти або, якщо це підтримується для типу ресурсу,
  остаточно видаляти записи, що порушують правила
- блокувати завантаження або встановлення небезпечних релізів
- відкликати API-токени
- м’яко видаляти пов’язаний вміст
- обмежувати доступ до публікації
- банити повторних або серйозних порушників

Ми не гарантуємо застосування правил із попередженням спочатку для очевидних зловживань. Див.
[Модерація та безпека облікового запису](/clawhub/moderation) щодо звітів, утримань модерації,
прихованих записів, банів і статусу облікового запису.
