# Прийнятне використання

Ця сторінка описує, які типи Skills і контенту прийнятні для ClawHub, а також які зловживальні робочі процеси він не розміщуватиме.

Ці правила навмисно практичні. Нас насамперед цікавлять наскрізні зловживальні робочі процеси, а не лише ізольовані ключові слова. Якщо Skill створено для обходу захистів, зловживання платформами, шахрайства проти людей, порушення приватності або сприяння поведінці без згоди, йому не місце на ClawHub.

## Нещодавні шаблони, які ми явно вважаємо прийнятними

- Робота з фронтендом і дизайн-системами, що використовує реальні компоненти, семантичні токени, доступні стани та протестовані користувацькі потоки.
- Композиція shadcn/ui, що використовує встановлені вихідні компоненти, псевдоніми проєкту та задокументовані варіанти замість одноразової розмітки.
- Перетворення UI5 з JavaScript на TypeScript, що зберігає коментарі, використовує конкретні типи UI5 і залишає згенеровані інтерфейси елементів керування придатними для рев’ю.
- Оборонний безпековий аналіз, інструменти модерації та промпти для виявлення зловживань, які показують докази й чітко зберігають межі людського схвалення.
- Автоматизація робочих процесів на основі згоди для особистих або командних облікових записів з явними обліковими даними, прозорим налаштуванням і режимами dry-run або попереднього перегляду.
- Документація, міграційні runbook-и, утиліти для розробників і тестові фікстури, обмежені програмним забезпеченням, яке вони підтримують.

## Неприйнятно

- Робочі процеси для обходу безпеки або несанкціонованого доступу.
  - Приклади: обхід автентифікації, захоплення облікового запису, обхід CAPTCHA, обхід Cloudflare або антибот-захисту, обхід обмежень частоти запитів, прихований скрейпінг, спроєктований для подолання захистів, захоплення живого дзвінка або агента, багаторазова крадіжка сеансів, автоматичне схвалення потоків спарювання для несхвалених користувачів.

- Зловживання платформами та обхід банів.
  - Приклади: приховані облікові записи після банів, прогрівання або ферми облікових записів, фальшива взаємодія, нарощування карми або підписників, автоматизація кількох облікових записів, масові публікації, спам-боти, маркетплейсна або соціальна автоматизація, створена для уникнення виявлення.

- Шахрайство, сками та оманливі фінансові робочі процеси.
  - Приклади: фальшиві сертифікати, фальшиві рахунки, оманливі платіжні потоки, скам-розсилки, фальшивий соціальний доказ, інструменти, що дають змогу витрачати кошти або списувати оплату без чіткого людського схвалення та прозорих засобів контролю, або робочі процеси із синтетичними ідентичностями, створені для відкриття облікових записів з метою шахрайства.

- Скрейпінг, збагачення даних або стеження, що порушують приватність.
  - Приклади: масовий скрейпінг контактних даних для спаму, доксинг, переслідування, витягування лідів у поєднанні з небажаними зверненнями, прихований моніторинг, пошук облич або біометричне зіставлення без чіткої згоди, або купівля, публікація, завантаження чи операціоналізація витоків даних або дампів зламів.

- Імітація особи без згоди або оманлива маніпуляція ідентичністю.
  - Приклади: заміна облич, цифрові двійники, фальшиві персонажі, клоновані інфлюенсери або інші інструменти маніпуляції ідентичністю, що використовуються для імітації особи або введення в оману.

- Відвертий сексуальний контент і генерація дорослого контенту з вимкненою безпекою.
  - Приклади: генерація NSFW-зображень, відео або контенту, обгортки для дорослого контенту навколо сторонніх API або Skills, основною метою яких є відвертий сексуальний контент.

- Приховані, небезпечні або оманливі вимоги до виконання.
  - Приклади: обфусковані команди встановлення, `curl | sh`, незадекларовані вимоги до секретів, незадеклароване використання приватних ключів, віддалене виконання `npx @latest` без чіткої можливості рев’ю, оманливі метадані, що приховують, що насправді потрібно Skill для запуску.

## Нещодавні шаблони, які ми явно вважаємо неприйнятними

- “Створювати приховані облікові записи продавців після банів на маркетплейсах.”
- “Змінювати спарювання Telegram, щоб несхвалені користувачі автоматично отримували коди спарювання.”
- “Розвивати облікові записи Reddit/Twitter за допомогою невиявної автоматизації.”
- “Генерувати професійні сертифікати або рахунки для довільного використання.”
- “Генерувати NSFW-контент із вимкненими перевірками безпеки.”
- “Скрейпити ліди, збагачувати контакти та запускати масові холодні звернення.”
- “Купувати, публікувати або завантажувати витоки даних чи дампи зламів.”
- “Масово створювати електронні поштові або соціальні облікові записи із синтетичними ідентичностями чи розв’язанням CAPTCHA.”

## Нотатки для рев’юерів

- Контекст має значення. Одна й та сама тема може бути легітимною у вузькому оборонному середовищі або середовищі на основі згоди й неприйнятною, коли її пакують як зловживальний робочий процес.
- Нам слід схилятися до дії, коли Skill явно оптимізовано для обходу, обману або використання без згоди.
- Повторні завантаження в цих категоріях є підставою для приховування контенту та бану облікового запису.

## Застосування правил

- Ми можемо приховувати, видаляти або остаточно видаляти Skills, що порушують правила.
- Ми можемо відкликати токени, м’яко видаляти пов’язаний контент і банити повторних або серйозних порушників.
- Ми не гарантуємо застосування правил із попереднім попередженням для очевидних зловживань.
