# Прийнятне використання

ClawHub розміщує Skills, plugins, пакети та метадані маркетплейсу для OpenClaw.
Використовуйте цю сторінку, щоб вирішити, чи належить контент або поведінка публікації
до ClawHub.

Ці правила застосовуються до того, що робить лістинг, що він просить користувачів запускати, як він
представляє себе та як видавці використовують поверхні ClawHub для виявлення, встановлення й
довіри. Щодо станів модерації та статусу облікового запису див.
[Модерація та безпека облікового запису](/clawhub/moderation). Щодо претензій про авторське право або інші права
див. [Запити щодо прав на контент](/clawhub/content-rights).

## Дозволений контент

ClawHub вітає контент, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія                                         | Дозволено, коли                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробників                           | Лістинг допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення.                                               |
| UI, дані та автоматизаційні робочі процеси               | Обсяг чіткий, потрібні облікові дані вказані явно, а ризиковані дії мають шляхи перегляду, пробного запуску, попереднього перегляду або підтвердження. |
| Захисна безпека, модерація та перегляд зловживань | Інструмент подано для авторизованого перегляду, він зберігає докази та чітко підтримує межі людського схвалення.                          |
| Особисті або командні робочі процеси                       | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи.                                            |
| Підтримувані каталоги                              | Кожен лістинг є окремим, корисним, точно описаним і належно підтримуваним.                                                |

Контекст має значення. Та сама тема може бути прийнятною у вузькому захисному або
заснованому на згоді середовищі та неприйнятною, коли її упаковано як робочий процес зловживань.

ClawHub не розміщує контент, основною метою якого є зловживання, обман, небезпечне
виконання або порушення прав.

| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| Зловживання платформою та обхід блокувань                              | Приховані облікові записи після блокувань, «прогрівання» або фермерство облікових записів, фальшива взаємодія, автоматизація багатьох облікових записів, масове публікування, спам-боти або автоматизація, створена для уникнення виявлення.                                                                                                                                          |

## Заборонена поведінка в маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не використовуйте ClawHub для
маніпулювання виявленням, метриками, сигналами довіри, системами модерації або увагою
користувачів.

Заборонена поведінка в маркетплейсі включає:

- масову публікацію великої кількості низькоякісних, дублювальних, шаблонних або
  згенерованих машиною лістингів, які, схоже, не мають реальної цінності для користувачів
- переповнення пошуку або категорій майже ідентичними skills або plugins
- публікацію сотень лістингів із незначним або відсутнім використанням, підтримкою, ясністю
  джерела або змістовною диференціацією
- штучне завищення встановлень, завантажень, зірок або інших метрик взаємодії
  через автоматизацію, цикли самостійного встановлення, фальшиві облікові записи, скоординовану
  активність, оплачену взаємодію або іншу неорганічну поведінку
- створення або ротацію облікових записів для обходу модерації, блокувань, лімітів видавця або
  перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, стану безпеки,
  вимог до встановлення або афіліації з іншим проєктом чи видавцем
- повторне завантаження контенту, який уже було приховано, видалено або заблоковано,
  без виправлення основної проблеми

Високий обсяг публікацій не є автоматично зловживанням. Великі каталоги прийнятні,
коли лістинги суттєво відрізняються, точно описані, підтримуються
та використовуються реальними користувачами. Великі каталоги стають проблемою довіри й безпеки, коли
обсяг поєднується з поверхневими, дублювальними, оманливими, непідтримуваними або
штучно просуваними лістингами.

## Права на контент

Якщо ви вважаєте, що контент на ClawHub порушує ваше авторське право або інші права, використовуйте
[Запити щодо прав на контент](/clawhub/content-rights). Не використовуйте звичайні звіти маркетплейсу
для претензій щодо авторського права або прав, якщо лістинг також не є небезпечним,
зловмисним або оманливим.

## Перегляд і застосування правил

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали зловживань, звіти користувачів і
перегляд персоналом, щоб виявляти небезпечний контент або зловживальну поведінку публікації. Сигнал
сам по собі не доводить зловживання; він допомагає ClawHub вирішити, що потребує перегляду.

Ми можемо:

- приховувати, утримувати, видаляти, м’яко видаляти або, якщо це підтримується для типу ресурсу,
  остаточно видаляти лістинги, що порушують правила
- блокувати завантаження або встановлення небезпечних релізів
- відкликати API-токени
- м’яко видаляти пов’язаний контент
- обмежувати доступ до публікації
- блокувати повторних або серйозних порушників

Ми не гарантуємо застосування правил із попередженням спочатку для очевидних зловживань. Див.
[Модерація та безпека облікового запису](/clawhub/moderation) щодо звітів, модераційних утримань,
прихованих лістингів, блокувань і статусу облікового запису.
