# Прийнятне використання

ClawHub розміщує Skills, плагіни, пакети та метадані маркетплейсу для OpenClaw.
Використовуйте цю сторінку, щоб вирішити, чи належить вміст або поведінка
публікації до ClawHub.

Ці правила застосовуються до того, що робить запис, що він просить користувачів
запустити, як він себе представляє та як видавці використовують поверхні
виявлення, встановлення й довіри ClawHub. Про стани модерації та статус
облікового запису див. [Модерація та безпека облікового запису](/clawhub/moderation).
Про претензії щодо авторського права або інших прав див.
[Запити щодо прав на вміст](/clawhub/content-rights).

## Дозволений вміст

ClawHub вітає вміст, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія                                         | Дозволено, коли                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробника                           | Запис допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення.                                               |
| Робочі процеси UI, даних і автоматизації               | Обсяг чіткий, потрібні облікові дані явно вказані, а ризиковані дії мають шляхи перевірки, пробного запуску, попереднього перегляду або підтвердження. |
| Захисна безпека, модерація та перевірка зловживань | Інструмент подано як засіб для авторизованої перевірки, він зберігає докази та чітко визначає межі людського схвалення.                          |
| Особисті або командні робочі процеси                       | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи.                                            |
| Підтримувані каталоги                              | Кожен запис є окремим, корисним, точно описаним і розумно підтримуваним.                                                |

Контекст має значення. Та сама тема може бути прийнятною у вузькому захисному
або заснованому на згоді середовищі й неприйнятною, коли її запаковано як
робочий процес для зловживань.

## Заборонена поведінка в маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не використовуйте
ClawHub для маніпулювання виявленням, метриками, сигналами довіри, системами
модерації або увагою користувачів.

Заборонена поведінка в маркетплейсі включає:

- масову публікацію великої кількості низькоякісних, дубльованих, заповнювальних
  або машинно згенерованих записів, які, схоже, не мають реальної цінності для користувачів
- засмічення пошуку або категорій майже ідентичними skills чи плагінами
- публікацію сотень записів із незначним використанням або без нього, без
  обслуговування, прозорості джерела чи meaningful differentiation
- штучне збільшення встановлень, завантажень, зірок або інших метрик взаємодії
  через автоматизацію, цикли самовстановлення, фальшиві облікові записи, скоординовану
  активність, оплачену взаємодію або іншу неорганічну поведінку
- створення або ротацію облікових записів для обходу модерації, блокувань, лімітів видавців або
  перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, стану безпеки,
  вимог до встановлення або пов’язаності з іншим проєктом чи видавцем
- повторне завантаження вмісту, який уже було приховано, видалено або заблоковано,
  без усунення основної проблеми

Публікація великих обсягів сама по собі не є зловживанням. Великі каталоги
прийнятні, коли записи суттєво відрізняються, точно описані, підтримуються
та використовуються реальними користувачами. Великі каталоги стають проблемою
довіри й безпеки, коли обсяг поєднується з поверхневими, дубльованими, оманливими,
непідтримуваними або штучно просуваними записами.

## Права на вміст

Якщо ви вважаєте, що вміст на ClawHub порушує ваше авторське право або інші
права, використовуйте [Запити щодо прав на вміст](/clawhub/content-rights).
Не використовуйте звичайні звіти маркетплейсу для претензій щодо авторського
права або прав, якщо запис також не є небезпечним, зловмисним або оманливим.

## Перевірка та застосування правил

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали
зловживань, звіти користувачів і перевірку персоналом для виявлення небезпечного
вмісту або зловживальної поведінки публікації. Сигнал сам по собі не доводить
зловживання; він допомагає ClawHub вирішити, що потребує перевірки.

Ми можемо:

- приховувати, утримувати, видаляти, м’яко видаляти або, якщо це підтримується
  для типу ресурсу, остаточно видаляти записи, що порушують правила
- блокувати завантаження або встановлення небезпечних релізів
- відкликати API-токени
- м’яко видаляти пов’язаний вміст
- обмежувати доступ до публікації
- блокувати повторних або серйозних порушників

Ми не гарантуємо застосування правил спочатку з попередженням для очевидних
зловживань. Див. [Модерація та безпека облікового запису](/clawhub/moderation)
щодо звітів, модераційних утримань, прихованих записів, блокувань і статусу
облікового запису.
