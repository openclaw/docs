# Прийнятне використання

ClawHub розміщує Skills, plugins, пакети та метадані маркетплейсу для OpenClaw.
Використовуйте цю сторінку, щоб визначити, чи належать вміст або поведінка
публікації до ClawHub.

Ці правила застосовуються до того, що робить оголошення, що воно просить
користувачів запускати, як воно представляє себе, і як видавці використовують
поверхні ClawHub для пошуку, встановлення та довіри. Щодо станів модерації та
статусу облікового запису див. [Модерація та безпека облікового запису](/uk/clawhub/moderation).
Щодо заяв про авторські права або інші права див.
[Запити щодо прав на вміст](/uk/clawhub/content-rights).

## Дозволений вміст

ClawHub вітає вміст, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія                                        | Дозволено, коли                                                                                                                   |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробників                       | Оголошення допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення. |
| Робочі процеси UI, даних та автоматизації        | Обсяг чіткий, потрібні облікові дані явно зазначені, а ризиковані дії мають шляхи перевірки, пробного запуску, попереднього перегляду або підтвердження. |
| Захисна безпека, модерація та перевірка зловживань | Інструмент подано для авторизованої перевірки, він зберігає докази та чітко утримує межі людського схвалення.                   |
| Особисті або командні робочі процеси             | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи.                              |
| Підтримувані каталоги                            | Кожне оголошення є окремим, корисним, точно описаним і належно підтримуваним.                                                    |

Контекст має значення. Та сама тема може бути прийнятною у вузькому захисному
або заснованому на згоді середовищі й неприйнятною, коли її упаковано як робочий
процес для зловживань.

## Заборонена поведінка в маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не використовуйте
ClawHub для маніпулювання пошуком, метриками, сигналами довіри, системами
модерації або увагою користувачів.

Заборонена поведінка в маркетплейсі включає:

- масову публікацію великої кількості малозмістовних, дублювальних,
  заповнювальних або машинно згенерованих оголошень, які не мають очевидної
  реальної цінності для користувачів
- переповнення поверхонь пошуку або категорій майже ідентичними Skills чи plugins
- публікацію сотень оголошень із малою або відсутньою кількістю використань,
  обслуговуванням, ясністю джерела або змістовною диференціацією
- штучне збільшення встановлень, завантажень, зірок або інших метрик
  взаємодії через автоматизацію, цикли самовстановлення, фальшиві облікові
  записи, скоординовану активність, оплачену взаємодію або іншу неорганічну
  поведінку
- створення або ротацію облікових записів для ухилення від модерації,
  блокувань, лімітів видавця або перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, стану
  безпеки, вимог до встановлення або афіліації з іншим проєктом чи видавцем
- повторне завантаження вмісту, який уже було приховано, видалено або
  заблоковано, без виправлення основної проблеми

Публікація у великому обсязі не є автоматично зловживанням. Великі каталоги
прийнятні, коли оголошення суттєво відрізняються, точно описані, підтримуються
та використовуються реальними користувачами. Великі каталоги стають проблемою
довіри й безпеки, коли обсяг поєднується з поверхневими, дублювальними,
оманливими, непідтримуваними або штучно просуваними оголошеннями.

## Права на вміст

Якщо ви вважаєте, що вміст на ClawHub порушує ваші авторські або інші права,
скористайтеся [Запитами щодо прав на вміст](/uk/clawhub/content-rights). Не використовуйте
звичайні звіти маркетплейсу для заяв про авторські або інші права, якщо
оголошення також не є небезпечним, зловмисним або оманливим.

## Перевірка та застосування правил

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали
зловживань, звіти користувачів і перевірку персоналом, щоб виявляти небезпечний
вміст або зловживальну поведінку публікації. Сигнал сам по собі не доводить
зловживання; він допомагає ClawHub вирішити, що потребує перевірки.

Ми можемо:

- приховувати, утримувати, видаляти, м'яко видаляти або, якщо це підтримується
  для типу ресурсу, остаточно видаляти оголошення, що порушують правила
- блокувати завантаження або встановлення небезпечних релізів
- відкликати API-токени
- м'яко видаляти пов'язаний вміст
- обмежувати доступ до публікації
- блокувати повторних або серйозних порушників

Ми не гарантуємо застосування правил із попередженням спочатку для очевидних
зловживань. Див. [Модерація та безпека облікового запису](/uk/clawhub/moderation) щодо
звітів, модераційних утримань, прихованих оголошень, блокувань і статусу
облікового запису.
