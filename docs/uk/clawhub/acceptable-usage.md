# Прийнятне використання

ClawHub розміщує Skills, Plugin, пакети та метадані маркетплейсу для OpenClaw.
Використовуйте цю сторінку, щоб визначити, чи належить вміст або поведінка
публікації до ClawHub.

Ці правила стосуються того, що робить запис, що він просить користувачів
запустити, як він себе представляє та як видавці використовують поверхні
виявлення, встановлення й довіри ClawHub. Стан модерації та статус облікового
запису описано в [Модерація та безпека облікового запису](/uk/clawhub/moderation).
Щодо авторських прав або інших претензій на права див.
[Запити щодо прав на вміст](/uk/clawhub/content-rights).

## Дозволений вміст

ClawHub вітає вміст, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія                                         | Дозволено, коли                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробників                           | Запис допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення.                                               |
| Робочі процеси UI, даних і автоматизації               | Обсяг чіткий, потрібні облікові дані явно вказані, а ризиковані дії мають шляхи перевірки, пробного запуску, попереднього перегляду або підтвердження. |
| Захисна безпека, модерація та перевірка зловживань | Інструмент подано для авторизованої перевірки, він зберігає докази та чітко визначає межі людського схвалення.                          |
| Особисті або командні робочі процеси                       | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи.                                            |
| Підтримувані каталоги                              | Кожен запис є окремим, корисним, точно описаним і належно підтримуваним.                                                |

Контекст має значення. Одна й та сама тема може бути прийнятною у вузькому
захисному або заснованому на згоді середовищі й неприйнятною, якщо її
упаковано як робочий процес для зловживань.

## Заборонена поведінка на маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не
використовуйте ClawHub для маніпулювання виявленням, метриками, сигналами
довіри, системами модерації або увагою користувачів.

Заборонена поведінка на маркетплейсі включає:

- масову публікацію великої кількості низькоякісних, дублювальних,
  заповнювальних або машинно згенерованих записів, які не мають очевидної
  реальної цінності для користувачів
- засмічення пошуку або категорій майже ідентичними Skills чи Plugin
- публікацію сотень записів із малою або відсутньою використаністю, підтримкою,
  прозорістю джерела чи змістовною диференціацією
- штучне завищення встановлень, завантажень, зірок або інших метрик
  взаємодії через автоматизацію, цикли самовстановлення, фейкові облікові
  записи, координовану активність, оплачену взаємодію або іншу неорганічну
  поведінку
- створення або ротацію облікових записів для обходу модерації, блокувань,
  лімітів видавця або перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, стану
  безпеки, вимог до встановлення або афіліації з іншим проєктом чи видавцем
- повторне завантаження вмісту, який уже було приховано, видалено або
  заблоковано, без виправлення базової проблеми

Публікація у великому обсязі не є автоматично зловживанням. Великі каталоги
прийнятні, коли записи суттєво відрізняються, точно описані, підтримуються та
використовуються реальними користувачами. Великі каталоги стають проблемою
довіри й безпеки, коли обсяг поєднується з поверхневими, дублювальними,
оманливими, непідтримуваними або штучно просуваними записами.

## Права на вміст

Якщо ви вважаєте, що вміст на ClawHub порушує ваші авторські або інші права,
скористайтеся [Запитами щодо прав на вміст](/uk/clawhub/content-rights). Не
використовуйте звичайні звіти маркетплейсу для претензій щодо авторських або
інших прав, якщо запис також не є небезпечним, шкідливим або оманливим.

## Перевірка та застосування правил

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали
зловживань, звіти користувачів і перевірку співробітниками, щоб виявляти
небезпечний вміст або зловживальну поведінку публікації. Сигнал сам по собі не
доводить зловживання; він допомагає ClawHub визначити, що потребує перевірки.

Ми можемо:

- приховувати, утримувати, видаляти, soft-delete або, якщо це підтримується для
  типу ресурсу, hard-delete записи, що порушують правила
- блокувати завантаження або встановлення небезпечних релізів
- відкликати API-токени
- soft-delete пов’язаний вміст
- обмежувати доступ до публікації
- блокувати повторних або серйозних порушників

Ми не гарантуємо спершу попередження для очевидних зловживань. Див.
[Модерація та безпека облікового запису](/uk/clawhub/moderation) щодо звітів,
модераційних утримань, прихованих записів, блокувань і статусу облікового
запису.
