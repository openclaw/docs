# Прийнятне використання

Ця сторінка описує типи Skills і контенту, які ClawHub приймає, а також зловживальні робочі процеси, які він не розміщуватиме.

Ці правила навмисно практичні. Нас найбільше цікавлять наскрізні робочі процеси зловживання, а не лише окремі ключові слова. Якщо Skill створено для обходу захистів, зловживання платформами, шахрайства щодо людей, вторгнення в приватність або уможливлення поведінки без згоди, йому не місце на ClawHub.

## Нещодавні патерни, які ми явно приймаємо

- Робота з фронтендом і дизайн-системами, що використовує реальні компоненти, семантичні токени, доступні стани та протестовані користувацькі сценарії.
- Компонування shadcn/ui, що використовує встановлені вихідні компоненти, псевдоніми проєкту та задокументовані варіанти замість одноразової розмітки.
- Перетворення UI5 з JavaScript на TypeScript, яке зберігає коментарі, використовує конкретні типи UI5 і залишає згенеровані інтерфейси елементів керування придатними для ревʼю.
- Захисне ревʼю безпеки, інструменти модерації та промпти для виявлення зловживань, які показують докази й чітко зберігають межі людського схвалення.
- Автоматизація робочих процесів на основі згоди для особистих або командних облікових записів з явними обліковими даними, прозорим налаштуванням і режимами пробного запуску або попереднього перегляду.
- Документація, інструкції з міграції, утиліти для розробників і тестові фікстури, обмежені програмним забезпеченням, яке вони підтримують.

## Неприйнятно

- Робочі процеси обходу безпеки або неавторизованого доступу.
  - Приклади: обхід автентифікації, захоплення облікового запису, обхід CAPTCHA, обхід Cloudflare або засобів протидії ботам, обхід обмежень частоти запитів, прихований скрейпінг, розроблений для подолання захистів, захоплення живого дзвінка або агента, повторно використовуване викрадення сесії, автоматичне схвалення потоків спарювання для несхвалених користувачів.

- Зловживання платформами та обхід банів.
  - Приклади: приховані облікові записи після банів, прогрівання або фермерство облікових записів, фальшива взаємодія, нарощування карми або підписників, автоматизація кількох облікових записів, масові публікації, спам-боти, автоматизація маркетплейсів або соціальних платформ, створена для уникнення виявлення.

- Шахрайство, скамерство та оманливі фінансові робочі процеси.
  - Приклади: фальшиві сертифікати, фальшиві рахунки-фактури, оманливі платіжні потоки, скам-розсилки, фальшиві соціальні докази, інструменти, що уможливлюють витрачання коштів або стягнення платежів без чіткого людського схвалення та прозорих засобів контролю, або робочі процеси синтетичної ідентичності, створені для відкриття облікових записів з метою шахрайства.

- Скрейпінг, збагачення даних або стеження, що порушує приватність.
  - Приклади: масовий скрейпінг контактних даних для спаму, доксинг, переслідування, витягування лідів у поєднанні з небажаними зверненнями, прихований моніторинг, пошук за обличчям або біометричне зіставлення без чіткої згоди, або купівля, публікація, завантаження чи операціоналізація витоків даних або дампів зламів.

- Імітація особи без згоди або оманлива маніпуляція ідентичністю.
  - Приклади: заміна облич, цифрові двійники, фальшиві персонажі, клоновані інфлюенсери або інші інструменти маніпуляції ідентичністю, що використовуються для видавання себе за когось або введення в оману.

- Відвертий сексуальний контент і генерація дорослого контенту з вимкненою безпекою.
  - Приклади: генерація NSFW-зображень, відео або контенту, обгортки для дорослого контенту навколо сторонніх API або Skills, основною метою яких є відвертий сексуальний контент.

- Приховані, небезпечні або оманливі вимоги до виконання.
  - Приклади: обфусковані команди встановлення, `curl | sh`, незадекларовані вимоги до секретів, незадеклароване використання приватних ключів, віддалене виконання `npx @latest` без чіткої можливості ревʼю, оманливі метадані, які приховують, що насправді потрібно Skill для запуску.

## Нещодавні патерни, які ми явно не приймаємо

- «Створити приховані облікові записи продавців після банів на маркетплейсі».
- «Змінити спарювання Telegram так, щоб несхвалені користувачі автоматично отримували коди спарювання».
- «Розвивати облікові записи Reddit/Twitter за допомогою невиявної автоматизації».
- «Генерувати професійні сертифікати або рахунки-фактури для довільного використання».
- «Генерувати NSFW-контент із вимкненими перевірками безпеки».
- «Скрейпити ліди, збагачувати контакти й запускати холодні звернення у великому масштабі».
- «Купувати, публікувати або завантажувати витоки даних чи дампи зламів».
- «Масово створювати облікові записи електронної пошти або соціальних платформ із синтетичними ідентичностями чи розвʼязуванням CAPTCHA».

## Нотатки для ревʼюерів

- Контекст має значення. Та сама тема може бути легітимною у вузькому захисному або заснованому на згоді середовищі й неприйнятною, коли її упаковано як робочий процес зловживання.
- Ми маємо схилятися до дії, коли Skill явно оптимізовано для обходу, обману або використання без згоди.
- Повторні завантаження в цих категоріях є підставою для приховування контенту й бану облікового запису.

## Застосування правил

- Ми можемо приховувати, видаляти або остаточно видаляти Skills, що порушують правила.
- Ми можемо відкликати токени, мʼяко видаляти повʼязаний контент і банити повторних або серйозних порушників.
- Ми не гарантуємо застосування правил із попереднім попередженням у випадках очевидного зловживання.
