# Прийнятне використання

ClawHub розміщує Skills, Plugin-и, пакети та метадані маркетплейсу для OpenClaw.
Використовуйте цю сторінку, щоб визначити, чи належать вміст або поведінка публікації
до ClawHub.

Ці правила застосовуються до того, що робить запис, що він просить користувачів запускати, як він
представляє себе та як видавці використовують поверхні ClawHub для пошуку, встановлення й
довіри. Щодо станів модерації та статусу облікового запису див.
[Модерація та безпека облікового запису](/clawhub/moderation). Щодо скарг про авторські права або інші права
див. [Запити щодо прав на вміст](/clawhub/content-rights).

## Дозволений вміст

ClawHub вітає вміст, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія                                         | Дозволено, коли                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробників                           | Запис допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення.                                               |
| Робочі процеси UI, даних та автоматизації               | Обсяг чіткий, потрібні облікові дані явно вказані, а ризикові дії мають шляхи перегляду, пробного запуску, попереднього перегляду або підтвердження. |
| Захисна безпека, модерація та перевірка зловживань | Інструмент подано як призначений для авторизованої перевірки, він зберігає докази та чітко визначає межі людського схвалення.                          |
| Особисті або командні робочі процеси                       | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи.                                            |
| Підтримувані каталоги                              | Кожен запис є окремим, корисним, точно описаним і належно підтримуваним.                                                |

Контекст має значення. Та сама тема може бути прийнятною у вузькому захисному або
заснованому на згоді контексті й неприйнятною, коли її упаковано як робочий процес для зловживань.

виконання або порушення прав.

| Зловживання платформою та обхід блокувань                              | Приховані облікові записи після блокувань, прогрівання або ферми облікових записів, фальшива взаємодія, автоматизація кількох облікових записів, масові публікації, спам-боти або автоматизація, створена для уникнення виявлення.                                                                                                                                          |

| Інвазивне щодо приватності збагачення або стеження                 | Збирання контактів для спаму, доксинг, переслідування, витягування лідів у поєднанні з небажаними зверненнями, прихований моніторинг, біометричне зіставлення без згоди або використання витоку даних чи дампів зламу.                                                                                                                  |

## Заборонена поведінка в маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не використовуйте ClawHub для
маніпулювання пошуком, метриками, сигналами довіри, системами модерації або
увагою користувачів.

Заборонена поведінка в маркетплейсі включає:

- масову публікацію великої кількості низькоякісних, дублювальних, заповнювальних або
  машинно згенерованих записів, які, схоже, не мають реальної цінності для користувачів
- засмічення поверхонь пошуку або категорій майже ідентичними Skills чи Plugin-ами
- публікацію сотень записів із незначним використанням або без нього, без підтримки, ясності
  джерела або змістовної диференціації
- штучне збільшення встановлень, завантажень, зірок або інших метрик
  взаємодії через автоматизацію, цикли самостійного встановлення, фальшиві облікові записи, скоординовану
  активність, платну взаємодію або іншу неорганічну поведінку
- створення або ротацію облікових записів для обходу модерації, блокувань, обмежень видавця або
  перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, безпекової позиції,
  вимог до встановлення або зв'язку з іншим проєктом чи видавцем
- повторне завантаження вмісту, який уже було приховано, видалено або заблоковано,
  без виправлення базової проблеми

Великі обсяги публікацій не є автоматично зловживанням. Великі каталоги прийнятні,
коли записи змістовно відрізняються, точно описані, підтримуються
та використовуються реальними користувачами. Великі каталоги стають проблемою довіри й безпеки, коли
обсяг поєднується з поверхневими, дублювальними, оманливими, непідтримуваними або
штучно просуваними записами.

## Права на вміст

Якщо ви вважаєте, що вміст на ClawHub порушує ваші авторські або інші права, скористайтеся
[Запитами щодо прав на вміст](/clawhub/content-rights). Не використовуйте звичайні звіти маркетплейсу
для скарг щодо авторських або інших прав, якщо запис також не є небезпечним,
зловмисним або оманливим.

## Перевірка та застосування правил

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали зловживань, звіти користувачів і
перевірку персоналом, щоб виявляти небезпечний вміст або зловживальну поведінку публікації. Сигнал
сам по собі не доводить зловживання; він допомагає ClawHub визначити, що потребує перевірки.

Ми можемо:

- приховувати, утримувати, видаляти, м'яко видаляти або, де це підтримується для типу ресурсу,
  остаточно видаляти записи, що порушують правила
- блокувати завантаження або встановлення небезпечних релізів
- відкликати API-токени
- м'яко видаляти пов'язаний вміст
- обмежувати доступ до публікації
- блокувати повторних або серйозних порушників

Ми не гарантуємо попередження перед застосуванням правил у випадках очевидного зловживання. Див.
[Модерація та безпека облікового запису](/clawhub/moderation) щодо звітів, модераційних утримань,
прихованих записів, блокувань і статусу облікового запису.
