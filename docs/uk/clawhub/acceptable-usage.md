# Прийнятне використання

Ця сторінка описує, які види навичок і контенту прийнятні для ClawHub, а також які зловживальні робочі процеси він не розміщуватиме.

Ці правила навмисно практичні. Нас найбільше цікавлять наскрізні робочі процеси зловживань, а не лише окремі ключові слова. Якщо навичка створена для обходу захистів, зловживання платформами, шахрайства, порушення приватності або уможливлення дій без згоди, їй не місце на ClawHub.

## Нещодавні патерни, які ми явно вважаємо прийнятними

- Робота з фронтендом і дизайн-системами, що використовує реальні компоненти, семантичні токени, доступні стани та протестовані користувацькі потоки.
- Компонування shadcn/ui, що використовує встановлені вихідні компоненти, псевдоніми проєкту та документовані варіанти замість одноразової розмітки.
- Перетворення UI5 з JavaScript на TypeScript, що зберігає коментарі, використовує конкретні типи UI5 і залишає згенеровані інтерфейси елементів керування придатними для рев’ю.
- Захисний аудит безпеки, інструменти модерації та підказки для виявлення зловживань, які показують докази й чітко зберігають межі людського схвалення.
- Автоматизація робочих процесів на основі згоди для особистих або командних облікових записів із явними обліковими даними, прозорим налаштуванням і режимами пробного запуску або попереднього перегляду.
- Документація, міграційні інструкції, утиліти для розробників і тестові фікстури, обмежені програмним забезпеченням, яке вони підтримують.

## Неприйнятно

- Робочі процеси обходу безпеки або несанкціонованого доступу.
  - Приклади: обхід автентифікації, захоплення облікового запису, обхід CAPTCHA, обхід Cloudflare або антибот-захисту, обхід обмежень частоти запитів, прихований скрейпінг, спроєктований для подолання захистів, захоплення живого дзвінка або агента, повторно використовувана крадіжка сесій, автоматичне схвалення потоків сполучення для несхвалених користувачів.

- Зловживання платформами та обхід банів.
  - Приклади: приховані облікові записи після банів, «прогрівання» або фермерство облікових записів, фальшива взаємодія, нарощування карми або підписників, автоматизація багатьох облікових записів, масове публікування, спам-боти, автоматизація маркетплейсів або соціальних платформ, створена для уникнення виявлення.

- Шахрайство, сками та оманливі фінансові робочі процеси.
  - Приклади: фальшиві сертифікати, фальшиві рахунки, оманливі платіжні потоки, скам-розсилки, фальшивий соціальний доказ, інструменти, що дають змогу витрачати кошти або списувати оплату без чіткого людського схвалення й прозорих засобів контролю, або робочі процеси синтетичної ідентичності, створені для реєстрації облікових записів з метою шахрайства.

- Скрейпінг, збагачення даних або стеження, що порушують приватність.
  - Приклади: масовий скрейпінг контактних даних для спаму, доксинг, переслідування, витягування лідів у поєднанні з небажаними зверненнями, прихований моніторинг, пошук за обличчям або біометричне зіставлення без чіткої згоди, або купівля, публікація, завантаження чи операціоналізація витоку даних або дампів після зламу.

- Несанкціоноване уособлення або оманлива маніпуляція ідентичністю.
  - Приклади: заміна облич, цифрові двійники, фальшиві персони, клоновані інфлюенсери або інші інструменти маніпуляції ідентичністю, що використовуються для уособлення або введення в оману.

- Відвертий сексуальний контент і генерація дорослого контенту з вимкненими засобами безпеки.
  - Приклади: генерація NSFW-зображень/відео/контенту, обгортки для дорослого контенту навколо сторонніх API або навички, основною метою яких є відвертий сексуальний контент.

- Приховані, небезпечні або оманливі вимоги до виконання.
  - Приклади: обфусковані команди встановлення, `curl | sh`, незадекларовані вимоги до секретів, незадеклароване використання приватних ключів, віддалене виконання `npx @latest` без чіткої можливості рев’ю, оманливі метадані, що приховують, що насправді потрібно навичці для запуску.

## Нещодавні патерни, які ми явно вважаємо неприйнятними

- “Створити приховані облікові записи продавців після банів на маркетплейсі.”
- “Змінити сполучення Telegram так, щоб несхвалені користувачі автоматично отримували коди сполучення.”
- “Розвивати облікові записи Reddit/Twitter за допомогою невиявної автоматизації.”
- “Генерувати професійні сертифікати або рахунки для довільного використання.”
- “Генерувати NSFW-контент із вимкненими перевірками безпеки.”
- “Скрейпити ліди, збагачувати контакти й запускати масові холодні звернення.”
- “Купувати, публікувати або завантажувати витоки даних чи дампи після зламу.”
- “Масово створювати поштові або соціальні облікові записи із синтетичними ідентичностями чи розв’язанням CAPTCHA.”

## Примітки для рецензентів

- Контекст має значення. Та сама тема може бути легітимною у вузькому захисному або заснованому на згоді сценарії й неприйнятною, коли її запаковано як робочий процес зловживання.
- Ми маємо схилятися до дії, коли навичка явно оптимізована для обходу, обману або використання без згоди.
- Повторні завантаження в цих категоріях є підставою для приховування контенту та бану облікового запису.

## Застосування правил

- Ми можемо приховувати, вилучати або безповоротно видаляти навички, що порушують правила.
- Ми можемо відкликати токени, м’яко видаляти пов’язаний контент і банити повторних або серйозних порушників.
- Ми не гарантуємо попередження перед застосуванням санкцій у випадках очевидного зловживання.
