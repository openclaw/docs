# Прийнятне використання

Ця сторінка описує, які типи skills і контенту ClawHub допускає, а також зловживальні робочі процеси, які він не розміщуватиме.

Ці правила навмисно практичні. Нас найбільше цікавлять наскрізні робочі процеси зловживань, а не лише ізольовані ключові слова. Якщо skill створено, щоб обходити захист, зловживати платформами, ошукувати людей, порушувати приватність або уможливлювати поведінку без згоди, йому не місце на ClawHub.

## Останні патерни, які ми явно допускаємо

- Робота з фронтендом і дизайн-системами, що використовує реальні компоненти, семантичні токени, доступні стани та протестовані користувацькі потоки.
- Компонування shadcn/ui, що використовує встановлені вихідні компоненти, псевдоніми проєкту та задокументовані варіанти замість одноразової розмітки.
- Перетворення UI5 JavaScript на TypeScript, яке зберігає коментарі, використовує конкретні типи UI5 і залишає згенеровані інтерфейси елементів керування придатними для рев’ю.
- Захисне рев’ю безпеки, інструменти модерації та промпти для виявлення зловживань, які показують докази й чітко зберігають межі людського схвалення.
- Автоматизація робочих процесів на основі згоди для особистих або командних облікових записів з явними обліковими даними, прозорим налаштуванням і режимами dry-run або попереднього перегляду.
- Документація, інструкції з міграції, утиліти для розробників і тестові фікстури, обмежені програмним забезпеченням, яке вони підтримують.

## Неприпустимо

- Робочі процеси обходу безпеки або несанкціонованого доступу.
  - Приклади: обхід автентифікації, захоплення облікового запису, обхід CAPTCHA, обхід Cloudflare або антибот-захисту, обхід обмежень частоти запитів, прихований скрейпінг, спроєктований для подолання захистів, захоплення live-дзвінка або агента, багаторазово використовувана крадіжка сесії, автоматичне схвалення pairing-процесів для несхвалених користувачів.

- Зловживання платформами та обхід банів.
  - Приклади: приховані облікові записи після банів, прогрівання або ферми облікових записів, фальшива взаємодія, накопичення карми або підписників, автоматизація багатьох облікових записів, масові публікації, спам-боти, автоматизація маркетплейсів або соціальних платформ, створена для уникнення виявлення.

- Шахрайство, сками та оманливі фінансові робочі процеси.
  - Приклади: фальшиві сертифікати, фальшиві рахунки, оманливі платіжні потоки, скам-розсилки, фальшивий соціальний доказ, інструменти, що уможливлюють витрати або стягнення коштів без чіткого людського схвалення та прозорих елементів керування, або робочі процеси синтетичної ідентичності, створені для реєстрації облікових записів з метою шахрайства.

- Скрейпінг, збагачення даних або стеження з порушенням приватності.
  - Приклади: масовий скрейпінг контактних даних для спаму, доксинг, переслідування, витягування лідів у поєднанні з небажаними зверненнями, прихований моніторинг, пошук облич або біометричне зіставлення без чіткої згоди, або купівля, публікація, завантаження чи операціоналізація витоків даних або дампів зламаних баз.

- Імітація особи без згоди або оманливе маніпулювання ідентичністю.
  - Приклади: face swap, цифрові двійники, фальшиві персонажі, клоновані інфлюенсери або інші інструменти маніпулювання ідентичністю, що використовуються для імітації особи або введення в оману.

- Відвертий сексуальний контент і генерація дорослого контенту з вимкненою безпекою.
  - Приклади: генерація NSFW-зображень, відео або контенту, обгортки для дорослого контенту навколо сторонніх API або skills, основною метою яких є відвертий сексуальний контент.

- Приховані, небезпечні або оманливі вимоги до виконання.
  - Приклади: обфусковані команди встановлення, `curl | sh`, незадекларовані вимоги до секретів, незадеклароване використання приватних ключів, віддалене виконання `npx @latest` без чіткої можливості рев’ю, оманливі метадані, що приховують, що насправді потрібно skill для запуску.

## Останні патерни, які ми явно не допускаємо

- «Створювати приховані облікові записи продавців після банів на маркетплейсі.»
- «Змінити pairing у Telegram так, щоб несхвалені користувачі автоматично отримували pairing-коди.»
- «Розвивати облікові записи Reddit/Twitter за допомогою невиявної автоматизації.»
- «Генерувати професійні сертифікати або рахунки для довільного використання.»
- «Генерувати NSFW-контент із вимкненими перевірками безпеки.»
- «Скрейпити ліди, збагачувати контакти та запускати масові холодні звернення.»
- «Купувати, публікувати або завантажувати витоки даних чи дампи зламаних баз.»
- «Масово створювати електронні або соціальні облікові записи із синтетичними ідентичностями або розв’язуванням CAPTCHA.»

## Примітки для рецензентів

- Контекст має значення. Та сама тема може бути легітимною у вузькому захисному або заснованому на згоді середовищі й неприйнятною, коли її пакують як робочий процес зловживань.
- Ми маємо схилятися до дії, коли skill явно оптимізований для обходу, обману або використання без згоди.
- Повторні завантаження в цих категоріях є підставою для приховування контенту та бану облікового запису.

## Застосування правил

- Ми можемо приховувати, видаляти або остаточно видаляти skills, що порушують правила.
- Ми можемо відкликати токени, виконувати soft-delete пов’язаного контенту та банити повторних або серйозних порушників.
- Ми не гарантуємо застосування правил із попереднім попередженням для очевидних зловживань.
