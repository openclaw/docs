# Допустиме використання

ClawHub розміщує Skills, Plugins, пакети та метадані маркетплейсу для OpenClaw.
Використовуйте цю сторінку, щоб визначити, чи належить вміст або поведінка публікації
до ClawHub.

Ці правила застосовуються до того, що робить запис, що він просить користувачів запускати, як він
представляє себе та як видавці використовують поверхні виявлення, встановлення й
довіри ClawHub. Щодо станів модерації та статусу облікового запису див.
[Модерація та безпека облікового запису](/uk/clawhub/moderation). Щодо авторських прав або інших заяв про права
див. [Запити щодо прав на вміст](/uk/clawhub/content-rights).

## Дозволений вміст

ClawHub вітає вміст, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія                                         | Дозволено, коли                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробників                           | Запис допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення.                                               |
| Робочі процеси UI, даних та автоматизації               | Обсяг чіткий, потрібні облікові дані явно зазначені, а ризиковані дії мають шляхи перевірки, пробного запуску, попереднього перегляду або підтвердження. |
| Захисна безпека, модерація та перевірка зловживань | Інструмент подано як засіб для авторизованої перевірки, він зберігає докази й чітко визначає межі людського схвалення.                          |
| Особисті або командні робочі процеси                       | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи.                                            |
| Підтримувані каталоги                              | Кожен запис є окремим, корисним, точно описаним і належно підтримуваним.                                                |

Контекст має значення. Та сама тема може бути прийнятною у вузькому захисному середовищі або
середовищі на основі згоди й неприйнятною, коли її упаковано як робочий процес для зловживань.

## Заборонена поведінка на маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не використовуйте ClawHub для
маніпулювання виявленням, метриками, сигналами довіри, системами модерації або
увагою користувачів.

Заборонена поведінка на маркетплейсі включає:

- масову публікацію великої кількості низькоякісних, дублювальних, заповнювальних або
  машинно згенерованих записів, які, схоже, не мають реальної цінності для користувачів
- заповнення пошукових або категорійних поверхонь майже ідентичними Skills або Plugins
- публікацію сотень записів із малою або відсутньою кількістю використань, підтримки, ясності
  джерела або змістовної відмінності
- штучне збільшення кількості встановлень, завантажень, зірок або інших метрик
  взаємодії через автоматизацію, цикли самовстановлення, фальшиві облікові записи, скоординовану
  активність, платну взаємодію або іншу неорганічну поведінку
- створення або ротацію облікових записів для обходу модерації, заборон, обмежень видавців або
  перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, стану безпеки,
  вимог до встановлення або афіліації з іншим проєктом чи видавцем
- повторне завантаження вмісту, який уже було приховано, видалено або заблоковано,
  без виправлення основної проблеми

Публікація великого обсягу не є автоматично зловживанням. Великі каталоги прийнятні,
коли записи мають суттєві відмінності, точно описані, підтримуються
та використовуються реальними користувачами. Великі каталоги стають проблемою довіри й безпеки, коли
обсяг поєднується з поверхневими, дублювальними, оманливими, непідтримуваними або
штучно просуваними записами.

## Права на вміст

Якщо ви вважаєте, що вміст на ClawHub порушує ваші авторські чи інші права, скористайтеся
[Запитами щодо прав на вміст](/uk/clawhub/content-rights). Не використовуйте звичайні звіти маркетплейсу
для заяв про авторські права або права, якщо запис також не є небезпечним,
шкідливим або оманливим.

## Перевірка та застосування правил

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали зловживань, звіти користувачів і
перевірку персоналом для виявлення небезпечного вмісту або зловживальної поведінки публікації. Сигнал
сам по собі не доводить зловживання; він допомагає ClawHub вирішити, що потребує перевірки.

Ми можемо:

- приховувати, утримувати, видаляти, м’яко видаляти або, якщо підтримується для типу ресурсу,
  остаточно видаляти записи, що порушують правила
- блокувати завантаження або встановлення небезпечних релізів
- відкликати токени API
- м’яко видаляти пов’язаний вміст
- обмежувати доступ до публікації
- банити повторних або серйозних порушників

Ми не гарантуємо застосування правила з попередженням спочатку для очевидних зловживань. Див.
[Модерація та безпека облікового запису](/uk/clawhub/moderation) щодо звітів, модераційних утримань,
прихованих записів, заборон і статусу облікового запису.
