# Прийнятне використання

На цій сторінці описано, які типи навичок і контенту ClawHub допускає, а також сценарії зловживань, які він не розміщуватиме.

Ці правила навмисно практичні. Нас найбільше цікавлять наскрізні сценарії зловживань, а не лише окремі ключові слова. Якщо навичку створено для обходу захистів, зловживання платформами, шахрайства проти людей, порушення приватності або сприяння поведінці без згоди, їй не місце на ClawHub.

## Нещодавні патерни, які ми явно допускаємо

- Робота з фронтендом і дизайн-системами, що використовує реальні компоненти, семантичні токени, доступні стани та протестовані користувацькі потоки.
- Компонування shadcn/ui, що використовує встановлені вихідні компоненти, псевдоніми проєкту та задокументовані варіанти замість одноразової розмітки.
- Перетворення UI5 з JavaScript на TypeScript, що зберігає коментарі, використовує конкретні типи UI5 і залишає згенеровані інтерфейси елементів керування придатними для рев’ю.
- Огляд захисної безпеки, інструменти модерації та промпти для виявлення зловживань, які показують докази й чітко зберігають межі людського схвалення.
- Автоматизація робочих процесів на основі згоди для особистих або командних облікових записів із явними обліковими даними, прозорим налаштуванням і режимами пробного запуску або попереднього перегляду.
- Документація, плани міграції, утиліти для розробників і тестові фікстури, обмежені програмним забезпеченням, яке вони підтримують.

## Неприпустимо

- Сценарії обходу безпеки або несанкціонованого доступу.
  - Приклади: обхід автентифікації, захоплення облікового запису, обхід CAPTCHA, обхід Cloudflare або антибот-захисту, обхід обмежень частоти запитів, прихований скрейпінг, розроблений для подолання захистів, захоплення живого дзвінка або агента, багаторазово придатне викрадення сеансу, автоматичне схвалення потоків сполучення для несхвалених користувачів.

- Зловживання платформами та обхід блокувань.
  - Приклади: приховані облікові записи після блокувань, «прогрівання» або фармінг облікових записів, фальшива взаємодія, нарощування карми або підписників, автоматизація кількох облікових записів, масові публікації, спам-боти, автоматизація маркетплейсів або соціальних платформ, створена для уникнення виявлення.

- Шахрайство, афери та оманливі фінансові робочі процеси.
  - Приклади: фальшиві сертифікати, фальшиві рахунки-фактури, оманливі платіжні потоки, шахрайські звернення, фальшивий соціальний доказ, інструменти, що дають змогу витрачати кошти або стягувати плату без чіткого людського схвалення та прозорих засобів контролю, або робочі процеси із синтетичною ідентичністю, створені для реєстрації облікових записів із метою шахрайства.

- Скрейпінг, збагачення даних або стеження з порушенням приватності.
  - Приклади: масовий скрейпінг контактних даних для спаму, доксинг, переслідування, витягування лідів у поєднанні з небажаними зверненнями, прихований моніторинг, пошук облич або біометричне зіставлення без чіткої згоди, або купівля, публікація, завантаження чи операціоналізація витеклих даних або дампів зламів.

- Імперсонація без згоди або оманливе маніпулювання ідентичністю.
  - Приклади: заміна облич, цифрові двійники, фальшиві персони, клоновані інфлюенсери або інші інструменти маніпулювання ідентичністю, що використовуються для імперсонації або введення в оману.

- Відвертий сексуальний контент і генерація дорослого контенту з вимкненими засобами безпеки.
  - Приклади: генерація NSFW-зображень/відео/контенту, обгортки для дорослого контенту навколо API сторонніх розробників або навички, основною метою яких є відвертий сексуальний контент.

- Приховані, небезпечні або оманливі вимоги до виконання.
  - Приклади: обфусковані команди встановлення, `curl | sh`, незадекларовані вимоги до секретів, незадеклароване використання приватних ключів, віддалене виконання `npx @latest` без чіткої можливості рев’ю, оманливі метадані, що приховують, що насправді потрібно навичці для запуску.

## Нещодавні патерни, які ми явно не допускаємо

- «Створювати приховані облікові записи продавців після блокувань на маркетплейсі».
- «Змінити сполучення Telegram так, щоб несхвалені користувачі автоматично отримували коди сполучення».
- «Розвивати облікові записи Reddit/Twitter за допомогою невиявної автоматизації».
- «Генерувати професійні сертифікати або рахунки-фактури для довільного використання».
- «Генерувати NSFW-контент із вимкненими перевірками безпеки».
- «Збирати ліди скрейпінгом, збагачувати контакти та запускати масові холодні звернення».
- «Купувати, публікувати або завантажувати витеклі дані чи дампи зламів».
- «Масово створювати електронні або соціальні облікові записи із синтетичними ідентичностями чи розв’язанням CAPTCHA».

## Нотатки для рев’юерів

- Контекст має значення. Одна й та сама тема може бути легітимною у вузькому захисному або заснованому на згоді середовищі й неприйнятною, коли її упаковано як сценарій зловживання.
- Ми маємо схилятися до дії, коли навичку явно оптимізовано для обходу, обману або використання без згоди.
- Повторні завантаження в цих категоріях є підставою для приховування контенту та блокування облікового запису.

## Застосування правил

- Ми можемо приховувати, видаляти або безповоротно видаляти навички, що порушують правила.
- Ми можемо відкликати токени, м’яко видаляти пов’язаний контент і блокувати повторних або серйозних порушників.
- Ми не гарантуємо попередження перед застосуванням санкцій у разі очевидних зловживань.
