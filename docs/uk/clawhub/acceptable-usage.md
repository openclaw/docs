# Допустиме використання

ClawHub розміщує Skills, Plugin, пакети та метадані маркетплейсу для OpenClaw.
Використовуйте цю сторінку, щоб вирішити, чи належить вміст або поведінка публікації
до ClawHub.

Ці правила застосовуються до того, що робить запис, що він просить користувачів запускати, як він
представляє себе та як видавці використовують поверхні виявлення, встановлення й
довіри ClawHub. Щодо станів модерації та репутації облікового запису див.
[Модерація та безпека облікового запису](/clawhub/moderation). Щодо авторських прав або інших
претензій щодо прав див. [Запити щодо прав на вміст](/clawhub/content-rights).

## Дозволений вміст

ClawHub вітає вміст, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія                                       | Дозволено, коли                                                                                                                |
| ----------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| Продуктивність розробників                      | Запис допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення. |
| Робочі процеси UI, даних та автоматизації       | Обсяг чіткий, потрібні облікові дані вказані явно, а ризиковані дії містять шляхи перегляду, пробного запуску, попереднього перегляду або підтвердження. |
| Захисна безпека, модерація та перевірка зловживань | Інструмент подано як призначений для авторизованої перевірки, він зберігає докази та чітко визначає межі людського схвалення. |
| Особисті або командні робочі процеси            | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи.                            |
| Підтримувані каталоги                           | Кожен запис є окремим, корисним, точно описаним і належно підтримуваним.                                                       |

Контекст має значення. Та сама тема може бути прийнятною у вузькому захисному або
заснованому на згоді контексті й неприйнятною, коли її упаковано як робочий процес для зловживань.

## Заборонений вміст

ClawHub не розміщує вміст, основною метою якого є зловживання, обман, небезпечне
виконання або порушення прав.

| Категорія                                                | Заборонено                                                                                                                                                                                                                                                                                         |
| -------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Несанкціонований доступ або обхід безпеки                | Обхід автентифікації, захоплення облікового запису, зловживання лімітами частоти, захоплення живого виклику або агента, повторно використовувана крадіжка сесій чи автоматичне схвалення потоків сполучення для несхвалених користувачів. |
| Зловживання платформою та обхід блокувань                | Приховані облікові записи після блокувань, прогрівання або ферми облікових записів, фальшива взаємодія, автоматизація багатьох облікових записів, масові публікації, спам-боти або автоматизація, створена для уникнення виявлення. |
| Шахрайство, афери та оманливі фінансові робочі процеси  | Підроблені сертифікати або рахунки, оманливі платіжні потоки, шахрайські розсилки, фальшиві соціальні докази, робочі процеси синтетичної ідентичності для шахрайства або інструменти витрачання/стягнення коштів без чіткого людського схвалення. |
| Інвазивне щодо приватності збагачення або стеження       | Збирання контактів для спаму, доксинг, переслідування, витягування лідів у поєднанні з небажаними зверненнями, прихований моніторинг, біометричне зіставлення без згоди або використання витеклих даних чи дампів зламів. |

| Приховані, небезпечні або оманливі вимоги до виконання   | Обфусковані команди встановлення, інсталятори pipe-to-shell, як-от завантажений вміст, запущений через `sh` або `bash` без чіткої можливості перевірки, незадекларовані вимоги до секретів або приватних ключів, віддалене виконання `npx @latest` без чіткої можливості перевірки або метадані, які приховують, що насправді потрібно запису для запуску. |

## Заборонена поведінка на маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не використовуйте ClawHub для
маніпулювання виявленням, метриками, сигналами довіри, системами модерації або
увагою користувачів.

Заборонена поведінка на маркетплейсі включає:

- масову публікацію великої кількості низькоякісних, дубльованих, шаблонних або
  машинно згенерованих записів, які не мають очевидної реальної цінності для користувачів
- заповнення поверхонь пошуку або категорій майже ідентичними Skills чи Plugin
- публікацію сотень записів із малим або нульовим використанням, підтримкою, прозорістю
  джерела або змістовною диференціацією
- штучне завищення встановлень, завантажень, зірок або інших метрик взаємодії
  через автоматизацію, цикли самовстановлення, фальшиві облікові записи, скоординовану
  активність, платну взаємодію або іншу неорганічну поведінку
- створення або ротацію облікових записів для обходу модерації, блокувань, лімітів видавця або
  перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, стану безпеки,
  вимог до встановлення або афіліації з іншим проєктом чи видавцем
- повторне завантаження вмісту, який уже було приховано, видалено або заблоковано,
  без виправлення базової проблеми

Публікація великих обсягів сама по собі не є зловживанням. Великі каталоги прийнятні,
коли записи змістовно відрізняються, точно описані, підтримуються
та використовуються реальними користувачами. Великі каталоги стають проблемою довіри та безпеки, коли
обсяг поєднується з поверховими, дубльованими, оманливими, непідтримуваними або
штучно просуваними записами.

## Права на вміст

Якщо ви вважаєте, що вміст на ClawHub порушує ваші авторські або інші права, використовуйте
[Запити щодо прав на вміст](/clawhub/content-rights). Не використовуйте звичайні звіти маркетплейсу
для претензій щодо авторських або інших прав, якщо запис також не є небезпечним,
зловмисним або оманливим.

## Перевірка та застосування правил

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали зловживань, звіти користувачів і
перевірку персоналом, щоб виявляти небезпечний вміст або зловживальну поведінку публікації. Сигнал
сам по собі не доводить зловживання; він допомагає ClawHub вирішити, що потребує перевірки.

Ми можемо:

- приховувати, утримувати, видаляти, м’яко видаляти або, якщо це підтримується для типу ресурсу,
  остаточно видаляти записи, що порушують правила
- блокувати завантаження або встановлення небезпечних релізів
- відкликати API-токени
- м’яко видаляти пов’язаний вміст
- обмежувати доступ до публікації
- блокувати повторних або серйозних порушників

Ми не гарантуємо спершу попереджувальне застосування правил для очевидних зловживань. Див.
[Модерація та безпека облікового запису](/clawhub/moderation) щодо звітів, модераційних утримань,
прихованих записів, блокувань і репутації облікового запису.
