# Прийнятне використання

ClawHub розміщує Skills, плагіни, пакети та метадані маркетплейсу для OpenClaw.
Використовуйте цю сторінку, щоб визначити, чи належать контент або поведінка
публікації до ClawHub.

Ці правила застосовуються до того, що робить лістинг, що він просить користувачів
запустити, як він себе представляє, і як видавці використовують поверхні
виявлення, встановлення та довіри ClawHub. Щодо станів модерації та статусу
облікового запису див. [Модерація та безпека облікового запису](/clawhub/moderation).
Щодо авторського права або інших претензій щодо прав див.
[Запити щодо прав на контент](/uk/clawhub/content-rights).

## Дозволений контент

ClawHub вітає контент, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія                                        | Дозволено, коли                                                                                                                   |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробників                       | Лістинг допомагає користувачам будувати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення. |
| Робочі процеси UI, даних та автоматизації        | Обсяг чіткий, потрібні облікові дані явно вказані, а ризиковані дії мають шляхи перегляду, dry-run, попереднього перегляду або підтвердження. |
| Захисна безпека, модерація та перевірка зловживань | Інструмент подано як засіб для авторизованої перевірки, він зберігає докази та чітко визначає межі людського схвалення.          |
| Особисті або командні робочі процеси             | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи.                              |
| Підтримувані каталоги                            | Кожен лістинг є окремим, корисним, точно описаним і належно підтримуваним.                                                       |

Контекст має значення. Одна й та сама тема може бути прийнятною у вузькому
захисному або заснованому на згоді контексті й неприйнятною, якщо її упаковано
як робочий процес для зловживань.

## Заборонений контент

ClawHub не розміщує контент, основною метою якого є зловживання, обман,
небезпечне виконання або порушення прав.

| Категорія                                                   | Не дозволено                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Несанкціонований доступ або обхід безпеки                   | Обхід автентифікації, захоплення облікового запису, зловживання обмеженнями швидкості, захоплення live-виклику або агента, багаторазово використовувана крадіжка сесії або автоматичне схвалення потоків сполучення для несанкціонованих користувачів. |
| Зловживання платформою та обхід банів                       | Приховані облікові записи після банів, прогрівання або ферми облікових записів, фальшива взаємодія, автоматизація багатьох облікових записів, масове публікування, спам-боти або автоматизація, створена для уникнення виявлення.             |
| Шахрайство, скамерство та оманливі фінансові робочі процеси | Фальшиві сертифікати або рахунки, оманливі платіжні потоки, скам-розсилки, фальшиві соціальні докази, робочі процеси синтетичної ідентичності для шахрайства або інструменти витрачання/стягнення коштів без чіткого людського схвалення.      |
| Інвазивне збагачення даних або стеження                     | Збір контактів для спаму, доксинг, переслідування, витягування лідів у поєднанні з небажаними зверненнями, прихований моніторинг, біометричне зіставлення без згоди або використання витеклих даних чи дампів зламів.                         |

## Заборонена поведінка в маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не використовуйте
ClawHub для маніпулювання виявленням, метриками, сигналами довіри, системами
модерації або увагою користувачів.

Заборонена поведінка в маркетплейсі включає:

- масову публікацію великої кількості низькоякісних, дублювальних, шаблонних або
  машинно згенерованих лістингів, які, схоже, не мають реальної цінності для користувачів
- засмічення пошуку або категорій майже ідентичними Skills чи плагінами
- публікацію сотень лістингів із малою або відсутньою кількістю використань,
  підтримки, прозорості джерела або змістовної диференціації
- штучне збільшення встановлень, завантажень, зірок або інших метрик взаємодії
  через автоматизацію, цикли самовстановлення, фальшиві облікові записи,
  скоординовану активність, платну взаємодію або іншу неорганічну поведінку
- створення або ротацію облікових записів для обходу модерації, банів, обмежень
  видавця або перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, стану
  безпеки, вимог до встановлення або зв’язку з іншим проєктом чи видавцем
- повторне завантаження контенту, який уже було приховано, видалено або
  заблоковано, без усунення базової проблеми

Публікація великого обсягу сама по собі не є зловживанням. Великі каталоги
прийнятні, коли лістинги суттєво відрізняються, точно описані, підтримуються
та використовуються реальними користувачами. Великі каталоги стають проблемою
довіри та безпеки, коли обсяг поєднується з поверхневими, дублювальними,
оманливими, непідтримуваними або штучно просуваними лістингами.

## Права на контент

Якщо ви вважаєте, що контент на ClawHub порушує ваше авторське право або інші
права, використовуйте [Запити щодо прав на контент](/uk/clawhub/content-rights).
Не використовуйте звичайні звіти маркетплейсу для претензій щодо авторського
права або прав, якщо лістинг водночас не є небезпечним, шкідливим або оманливим.

## Перевірка та застосування правил

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали
зловживань, звіти користувачів і перевірку персоналом, щоб виявляти небезпечний
контент або зловмисну поведінку публікації. Сигнал сам по собі не доводить
зловживання; він допомагає ClawHub вирішити, що потребує перевірки.

Ми можемо:

- приховувати, утримувати, видаляти, soft-delete або, якщо це підтримується для
  типу ресурсу, hard-delete лістинги, що порушують правила
- блокувати завантаження або встановлення небезпечних релізів
- відкликати API-токени
- soft-delete пов’язаний контент
- обмежувати доступ до публікації
- банити повторних або серйозних порушників

Ми не гарантуємо застосування правила “спочатку попередження” для очевидних
зловживань. Див. [Модерація та безпека облікового запису](/clawhub/moderation)
щодо звітів, модераційних утримань, прихованих лістингів, банів і статусу
облікового запису.
