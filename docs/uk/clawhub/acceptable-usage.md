# Прийнятне використання

Ця сторінка описує типи навичок і контенту, які ClawHub дозволяє, а також сценарії зловживань, які він не розміщуватиме.

Ці правила навмисно практичні. Нас найбільше цікавлять наскрізні сценарії зловживань, а не лише окремі ключові слова. Якщо навичка створена для обходу захисту, зловживання платформами, шахрайства щодо людей, порушення приватності або уможливлення дій без згоди, їй не місце на ClawHub.

## Нещодавні шаблони, які ми явно дозволяємо

- Робота з фронтендом і дизайн-системами, що використовує реальні компоненти, семантичні токени, доступні стани та протестовані користувацькі сценарії.
- Композиція shadcn/ui, що використовує встановлені вихідні компоненти, псевдоніми проєкту та задокументовані варіанти замість одноразової розмітки.
- Перетворення UI5 JavaScript на TypeScript зі збереженням коментарів, використанням конкретних типів UI5 і підтримкою придатності згенерованих інтерфейсів керування до рев’ю.
- Захисний аудит безпеки, інструменти модерації та підказки для виявлення зловживань, які показують докази й чітко зберігають межі людського схвалення.
- Автоматизація робочих процесів на основі згоди для особистих або командних облікових записів із явними обліковими даними, прозорим налаштуванням і режимами dry-run або попереднього перегляду.
- Документація, інструкції з міграції, утиліти для розробників і тестові фікстури, обмежені програмним забезпеченням, яке вони підтримують.

## Неприйнятно

- Сценарії обходу безпеки або несанкціонованого доступу.
  - Приклади: обхід автентифікації, захоплення облікового запису, обхід CAPTCHA, ухилення від Cloudflare або антибот-захисту, обхід обмежень частоти запитів, прихований скрейпінг, розроблений для подолання захисту, захоплення живого дзвінка або агента, повторно використовувана крадіжка сесії, автоматичне схвалення процесів сполучення для несхвалених користувачів.

- Зловживання платформами та ухилення від банів.
  - Приклади: приховані облікові записи після банів, прогрівання або фермерство облікових записів, фальшива взаємодія, нарощування карми або підписників, автоматизація кількох облікових записів, масові публікації, спам-боти, автоматизація маркетплейсів або соціальних платформ, створена для уникнення виявлення.

- Шахрайство, сками та оманливі фінансові процеси.
  - Приклади: фальшиві сертифікати, фальшиві рахунки, оманливі платіжні процеси, скам-розсилки, фальшивий соціальний доказ, інструменти, що дозволяють витрачати кошти або стягувати оплату без чіткого людського схвалення та прозорих засобів контролю, або сценарії синтетичної ідентичності, створені для реєстрації облікових записів з метою шахрайства.

- Скрейпінг, збагачення даних або стеження, що порушують приватність.
  - Приклади: масштабний скрейпінг контактних даних для спаму, доксинг, переслідування, витягнення лідів у поєднанні з небажаними зверненнями, прихований моніторинг, пошук за обличчям або біометричне зіставлення без чіткої згоди, або купівля, публікація, завантаження чи операціоналізація витеклих даних або дампів зламів.

- Несанкціоноване уособлення або оманлива маніпуляція ідентичністю.
  - Приклади: заміна облич, цифрові двійники, фальшиві персони, клоновані інфлюенсери або інші інструменти маніпуляції ідентичністю, що використовуються для уособлення чи введення в оману.

- Явний сексуальний контент і генерація дорослого контенту з вимкненими механізмами безпеки.
  - Приклади: генерація NSFW-зображень/відео/контенту, обгортки для дорослого контенту навколо сторонніх API або навички, основною метою яких є явний сексуальний контент.

- Приховані, небезпечні або оманливі вимоги до виконання.
  - Приклади: обфусковані команди встановлення, `curl | sh`, незадекларовані вимоги до секретів, незадеклароване використання приватних ключів, віддалене виконання `npx @latest` без чіткої можливості рев’ю, оманливі метадані, що приховують справжні вимоги навички для запуску.

## Нещодавні шаблони, які ми явно не дозволяємо

- «Створити приховані облікові записи продавців після банів на маркетплейсі».
- «Змінити сполучення Telegram так, щоб несхвалені користувачі автоматично отримували коди сполучення».
- «Нарощувати облікові записи Reddit/Twitter за допомогою невиявної автоматизації».
- «Генерувати професійні сертифікати або рахунки для довільного використання».
- «Генерувати NSFW-контент із вимкненими перевірками безпеки».
- «Скрейпити ліди, збагачувати контакти й запускати холодні розсилки у великому масштабі».
- «Купувати, публікувати або завантажувати витеклі дані чи дампи зламів».
- «Масово створювати поштові або соціальні облікові записи із синтетичними ідентичностями чи розв’язуванням CAPTCHA».

## Примітки для рецензентів

- Контекст має значення. Та сама тема може бути легітимною у вузькому захисному або заснованому на згоді середовищі й неприйнятною, коли її упаковано як сценарій зловживання.
- Нам слід схилятися до дії, коли навичка явно оптимізована для ухилення, обману або використання без згоди.
- Повторні завантаження в цих категоріях є підставою для приховування контенту та бану облікового запису.

## Правозастосування

- Ми можемо приховувати, видаляти або безповоротно видаляти навички, що порушують правила.
- Ми можемо відкликати токени, м’яко видаляти пов’язаний контент і банити повторних або серйозних порушників.
- Ми не гарантуємо попередження перед застосуванням заходів для очевидних зловживань.
