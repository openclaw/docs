# Прийнятне використання

ClawHub розміщує Skills, плагіни, пакети та метадані маркетплейсу для OpenClaw.
Використовуйте цю сторінку, щоб визначити, чи належить вміст або поведінка
публікації до ClawHub.

Ці правила застосовуються до того, що робить лістинг, що він просить користувачів
запускати, як він представляє себе та як видавці використовують поверхні
виявлення, встановлення й довіри ClawHub. Щодо станів модерації та статусу
облікового запису див. [Модерація та безпека облікового запису](/uk/clawhub/moderation).
Щодо заяв про авторські або інші права див.
[Запити щодо прав на вміст](/uk/clawhub/content-rights).

## Дозволений вміст

ClawHub вітає вміст, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія                                      | Дозволено, коли                                                                                                                    |
| ---------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробника                      | Лістинг допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення. |
| Робочі процеси UI, даних та автоматизації      | Обсяг чіткий, необхідні облікові дані явно зазначені, а ризиковані дії мають шляхи перегляду, пробного запуску, попереднього перегляду або підтвердження. |
| Захисна безпека, модерація та перевірка зловживань | Інструмент подано як призначений для авторизованої перевірки, він зберігає докази та чітко окреслює межі людського схвалення. |
| Особисті або командні робочі процеси           | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи.                                |
| Підтримувані каталоги                          | Кожен лістинг є окремим, корисним, точно описаним і належно підтримується.                                                        |

Контекст має значення. Та сама тема може бути прийнятною у вузькому захисному
або заснованому на згоді середовищі та неприйнятною, коли її упаковано як
робочий процес для зловживань.

## Заборонена поведінка в маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не використовуйте
ClawHub для маніпулювання виявленням, метриками, сигналами довіри, системами
модерації або увагою користувачів.

Заборонена поведінка в маркетплейсі включає:

- масову публікацію великої кількості низькоякісних, дублювальних, заповнювальних
  або згенерованих машиною лістингів, які, схоже, не мають реальної користі для
  користувачів
- засмічення пошуку або категорій майже ідентичними Skills чи плагінами
- публікацію сотень лістингів із незначним використанням або без нього,
  обслуговуванням, прозорістю джерела чи змістовною відмінністю
- штучне завищення встановлень, завантажень, зірок або інших метрик взаємодії
  через автоматизацію, цикли самовстановлення, фейкові облікові записи,
  скоординовану активність, оплачену взаємодію або іншу неорганічну поведінку
- створення або ротацію облікових записів для обходу модерації, банів,
  обмежень видавця або перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, стану
  безпеки, вимог до встановлення або афіліації з іншим проєктом чи видавцем
- повторне завантаження вмісту, який уже було приховано, видалено або заблоковано,
  без виправлення базової проблеми

Публікація у великому обсязі не є автоматично зловживанням. Великі каталоги
прийнятні, коли лістинги суттєво відрізняються, точно описані, підтримуються та
використовуються реальними користувачами. Великі каталоги стають проблемою
довіри й безпеки, коли обсяг поєднується з поверховими, дублювальними,
оманливими, непідтримуваними або штучно просуваними лістингами.

## Права на вміст

Якщо ви вважаєте, що вміст на ClawHub порушує ваші авторські або інші права,
скористайтеся [Запитами щодо прав на вміст](/uk/clawhub/content-rights). Не
використовуйте звичайні звіти маркетплейсу для заяв про авторські або інші
права, якщо лістинг також не є небезпечним, шкідливим або оманливим.

## Перевірка та застосування правил

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали
зловживань, звіти користувачів і перевірку персоналом, щоб виявляти небезпечний
вміст або зловживальну поведінку публікації. Сигнал сам по собі не доводить
зловживання; він допомагає ClawHub визначити, що потребує перевірки.

Ми можемо:

- приховувати, утримувати, видаляти, м’яко видаляти або, якщо це підтримується
  для типу ресурсу, остаточно видаляти лістинги, що порушують правила
- блокувати завантаження або встановлення небезпечних релізів
- відкликати API-токени
- м’яко видаляти пов’язаний вміст
- обмежувати доступ до публікації
- банити повторних або серйозних порушників

Ми не гарантуємо застосування правил спочатку з попередженням у випадках
очевидного зловживання. Див.
[Модерація та безпека облікового запису](/uk/clawhub/moderation) щодо звітів,
модераційних утримань, прихованих лістингів, банів і статусу облікового запису.
