# Прийнятне використання

Ця сторінка описує, які види навичок і контенту ClawHub приймає, а також зловживальні робочі процеси, які він не розміщуватиме.

Ці правила навмисно практичні. Нас найбільше цікавлять наскрізні робочі процеси зловживання, а не лише ізольовані ключові слова. Якщо навичку створено для обходу захистів, зловживання платформами, шахрайства проти людей, вторгнення в приватність або уможливлення поведінки без згоди, їй не місце на ClawHub.

## Нещодавні патерни, які ми явно приймаємо

- Робота над фронтендом і дизайн-системою, що використовує реальні компоненти, семантичні токени, доступні стани та протестовані користувацькі сценарії.
- Компонування shadcn/ui, що використовує встановлені вихідні компоненти, аліаси проєкту та задокументовані варіанти замість одноразової розмітки.
- Перетворення UI5 з JavaScript на TypeScript, що зберігає коментарі, використовує конкретні типи UI5 і залишає згенеровані інтерфейси контролів придатними для ревʼю.
- Захисний аудит безпеки, інструменти модерації та промпти для виявлення зловживань, які показують докази й зберігають чіткі межі людського схвалення.
- Автоматизація робочих процесів на основі згоди для особистих або командних облікових записів із явними обліковими даними, прозорим налаштуванням і режимами пробного запуску або попереднього перегляду.
- Документація, посібники з міграції, утиліти для розробників і тестові фікстури, обмежені програмним забезпеченням, яке вони підтримують.

## Неприйнятно

- Робочі процеси обходу безпеки або несанкціонованого доступу.
  - Приклади: обхід автентифікації, захоплення облікового запису, обхід CAPTCHA, обхід Cloudflare або антибот-захисту, обхід обмежень частоти, приховане збирання даних, спроєктоване для подолання захистів, захоплення живого дзвінка або агента, багаторазова крадіжка сесій, автоматичне схвалення потоків сполучення для несхвалених користувачів.

- Зловживання платформами та обхід банів.
  - Приклади: приховані облікові записи після банів, прогрівання або ферми облікових записів, фальшива залученість, нарощування карми або підписників, автоматизація кількох облікових записів, масові публікації, спам-боти, маркетплейсова або соціальна автоматизація, створена для уникнення виявлення.

- Шахрайство, афери та оманливі фінансові робочі процеси.
  - Приклади: фальшиві сертифікати, фальшиві рахунки-фактури, оманливі платіжні потоки, шахрайські звернення, фальшиві соціальні докази, інструменти, що дають змогу витрачати кошти або стягувати оплату без чіткого людського схвалення та прозорих контролів, або робочі процеси синтетичної ідентичності, створені для відкриття облікових записів з метою шахрайства.

- Скрейпінг, збагачення даних або стеження, що порушують приватність.
  - Приклади: масовий скрейпінг контактних даних для спаму, доксинг, переслідування, витяг лідів у поєднанні з небажаними зверненнями, прихований моніторинг, пошук за обличчям або біометричне зіставлення без чіткої згоди, або купівля, публікація, завантаження чи операціоналізація витоків даних або дампів зламів.

- Імітація особи без згоди або оманливе маніпулювання ідентичністю.
  - Приклади: заміна облич, цифрові двійники, фальшиві персони, клоновані інфлюенсери або інші інструменти маніпулювання ідентичністю, що використовуються для видавання себе за когось іншого або введення в оману.

- Відвертий сексуальний контент і генерація дорослого контенту з вимкненою безпекою.
  - Приклади: генерація NSFW-зображень, відео або контенту, обгортки для дорослого контенту навколо сторонніх API або навички, основна мета яких — відвертий сексуальний контент.

- Приховані, небезпечні або оманливі вимоги до виконання.
  - Приклади: обфусковані команди встановлення, `curl | sh`, незадекларовані вимоги до секретів, незадеклароване використання приватного ключа, віддалене виконання `npx @latest` без чіткої можливості перевірки, оманливі метадані, що приховують, що насправді потрібно навичці для запуску.

## Нещодавні патерни, які ми явно не приймаємо

- «Створити приховані облікові записи продавців після банів на маркетплейсі».
- «Змінити сполучення Telegram, щоб несхвалені користувачі автоматично отримували коди сполучення».
- «Розвивати облікові записи Reddit/Twitter за допомогою непомітної автоматизації».
- «Генерувати професійні сертифікати або рахунки-фактури для довільного використання».
- «Генерувати NSFW-контент із вимкненими перевірками безпеки».
- «Скрейпити ліди, збагачувати контакти й запускати масові холодні звернення».
- «Купувати, публікувати або завантажувати витоки даних чи дампи зламів».
- «Масово створювати електронні або соціальні облікові записи із синтетичними ідентичностями чи розвʼязуванням CAPTCHA».

## Примітки для рецензентів

- Контекст має значення. Та сама тема може бути легітимною у вузькому захисному або заснованому на згоді середовищі й неприйнятною, коли її упаковано як робочий процес зловживання.
- Нам слід схилятися до дії, коли навичку явно оптимізовано для обходу, обману або використання без згоди.
- Повторні завантаження в цих категоріях є підставою для приховування контенту та блокування облікового запису.

## Застосування правил

- Ми можемо приховувати, вилучати або остаточно видаляти навички, що порушують правила.
- Ми можемо відкликати токени, мʼяко видаляти повʼязаний контент і блокувати повторних або серйозних порушників.
- Ми не гарантуємо спершу попередження для очевидних зловживань.
