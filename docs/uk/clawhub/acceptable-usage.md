# Прийнятне використання

ClawHub розміщує Skills, плагіни, пакети та метадані маркетплейсу для OpenClaw.
Використовуйте цю сторінку, щоб визначити, чи належать вміст або поведінка публікації до
ClawHub.

Ці правила застосовуються до того, що робить запис, що він просить користувачів запускати, як він
представляє себе та як видавці використовують поверхні ClawHub для виявлення, встановлення й
довіри. Стани модерації та репутацію облікового запису див.
[Модерація та безпека облікового запису](/clawhub/moderation). Щодо авторських прав або інших
правових претензій див. [Запити щодо прав на вміст](/uk/clawhub/content-rights).

## Дозволений вміст

ClawHub вітає вміст, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія                                       | Дозволено, коли                                                                                                                      |
| ----------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| Продуктивність розробників                      | Запис допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення. |
| Робочі процеси UI, даних та автоматизації       | Обсяг чіткий, потрібні облікові дані зазначені явно, а ризиковані дії мають шляхи перегляду, пробного запуску, попереднього перегляду або підтвердження. |
| Захисна безпека, модерація та перевірка зловживань | Інструмент подано для авторизованої перевірки, він зберігає докази та чітко позначає межі людського схвалення.                       |
| Особисті або командні робочі процеси            | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи.                                   |
| Підтримувані каталоги                           | Кожен запис є окремим, корисним, точно описаним і належно підтримуваним.                                                             |

Контекст має значення. Та сама тема може бути прийнятною у вузькому захисному або
заснованому на згоді контексті й неприйнятною, коли її упаковано як робочий процес для зловживань.

## Заборонена поведінка в маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не використовуйте ClawHub для
маніпулювання виявленням, метриками, сигналами довіри, системами модерації або
увагою користувачів.

Заборонена поведінка в маркетплейсі включає:

- масову публікацію великої кількості малозмістовних, дублювальних, шаблонних або
  машинно згенерованих записів, які не мають очевидної реальної цінності для користувачів
- засмічення пошуку або категорій майже ідентичними Skills чи плагінами
- публікацію сотень записів із малим використанням або без нього, без підтримки, прозорості
  джерела чи змістовної відмінності
- штучне завищення встановлень, завантажень, зірок або інших метрик
  взаємодії через автоматизацію, цикли самовстановлення, фальшиві облікові записи, скоординовану
  активність, платну взаємодію або іншу неорганічну поведінку
- створення або ротацію облікових записів для обходу модерації, банів, обмежень видавця або
  перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, стану безпеки,
  вимог до встановлення або зв’язку з іншим проєктом чи видавцем
- повторне завантаження вмісту, який уже було приховано, видалено або заблоковано,
  без виправлення основної проблеми

Публікація у великому обсязі не є автоматично зловживанням. Великі каталоги прийнятні,
коли записи суттєво відрізняються, точно описані, підтримуються
та використовуються реальними користувачами. Великі каталоги стають проблемою довіри й безпеки, коли
обсяг поєднується з поверховими, дублювальними, оманливими, непідтримуваними або
штучно просуваними записами.

## Права на вміст

Якщо ви вважаєте, що вміст на ClawHub порушує ваші авторські або інші права, скористайтеся
[Запитами щодо прав на вміст](/uk/clawhub/content-rights). Не використовуйте звичайні звіти маркетплейсу
для претензій щодо авторських або інших прав, якщо запис також не є небезпечним,
шкідливим або оманливим.

## Перевірка та застосування правил

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали зловживань, звіти користувачів і
перевірку персоналом, щоб виявляти небезпечний вміст або зловживальну поведінку публікації. Сигнал
сам по собі не доводить зловживання; він допомагає ClawHub вирішити, що потребує перевірки.

Ми можемо:

- приховувати, утримувати, видаляти, м’яко видаляти або, якщо це підтримується для типу ресурсу,
  остаточно видаляти записи, що порушують правила
- блокувати завантаження або встановлення небезпечних релізів
- відкликати API-токени
- м’яко видаляти пов’язаний вміст
- обмежувати доступ до публікації
- банити повторних або серйозних порушників

Ми не гарантуємо застосування правил із попереднім попередженням для очевидних зловживань. Див.
[Модерація та безпека облікового запису](/clawhub/moderation) щодо звітів, модераційних утримань,
прихованих записів, банів і репутації облікового запису.
