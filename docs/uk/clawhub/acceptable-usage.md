# Прийнятне використання

ClawHub розміщує Skills, plugins, пакети та метадані marketplace для OpenClaw.
Використовуйте цю сторінку, щоб визначити, чи належить вміст або поведінка публікації до
ClawHub.

Ці правила застосовуються до того, що робить listing, що він просить користувачів запускати, як він
представляє себе та як видавці використовують поверхні виявлення, встановлення й
довіри ClawHub. Для станів модерації та статусу акаунта див.
[Модерація та безпека акаунта](/clawhub/moderation). Для претензій щодо авторського права або інших прав
див. [Запити щодо прав на вміст](/uk/clawhub/content-rights).

## Дозволений вміст

ClawHub вітає вміст, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія                                         | Дозволено, коли                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробників                           | Listing допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення.                                               |
| Робочі процеси UI, даних та автоматизації               | Обсяг чіткий, необхідні облікові дані явно зазначені, а ризиковані дії включають перевірку, dry-run, попередній перегляд або шляхи підтвердження. |
| Захисна безпека, модерація та перевірка зловживань | Інструмент позиціонується для авторизованої перевірки, зберігає докази та чітко визначає межі схвалення людиною.                          |
| Особисті або командні робочі процеси                       | Робочий процес використовує акаунти на основі згоди, прозоре налаштування та явні дозволи.                                            |
| Підтримувані каталоги                              | Кожен listing є окремим, корисним, точно описаним і належно підтримуваним.                                                |

Контекст має значення. Та сама тема може бути прийнятною у вузькому захисному або
заснованому на згоді контексті й неприйнятною, коли її упаковано як робочий процес для зловживань.

## Заборонена поведінка в marketplace

ClawHub також перевіряє, як видавці використовують marketplace. Не використовуйте ClawHub, щоб
маніпулювати виявленням, метриками, сигналами довіри, системами модерації або увагою
користувачів.

Заборонена поведінка в marketplace включає:

- масову публікацію великої кількості низькоякісних, дублювальних, placeholder або
  машинно згенерованих listing, які не мають очевидної реальної цінності для користувачів
- заповнення поверхонь пошуку або категорій майже ідентичними skills або plugins
- публікацію сотень listing з малою кількістю або відсутністю використання, підтримки, прозорості
  джерела чи змістовної диференціації
- штучне збільшення встановлень, завантажень, зірок або інших метрик
  взаємодії через автоматизацію, цикли самовстановлення, фейкові акаунти, скоординовану
  активність, платну взаємодію або іншу неорганічну поведінку
- створення або ротацію акаунтів для обходу модерації, банів, лімітів видавців або
  перевірки marketplace
- введення користувачів в оману щодо власності, джерела, можливостей, стану безпеки,
  вимог до встановлення або зв'язку з іншим проєктом чи видавцем
- повторне завантаження вмісту, який уже було приховано, видалено або заблоковано,
  без виправлення базової проблеми

Публікація у великому обсязі не є автоматично зловживанням. Великі каталоги прийнятні,
коли listing суттєво відрізняються, точно описані, підтримуються
та використовуються реальними користувачами. Великі каталоги стають проблемою довіри й безпеки, коли
обсяг поєднується з поверхневими, дублювальними, оманливими, непідтримуваними або
штучно просуваними listing.

## Права на вміст

Якщо ви вважаєте, що вміст на ClawHub порушує ваше авторське право або інші права, скористайтеся
[Запити щодо прав на вміст](/uk/clawhub/content-rights). Не використовуйте звичайні звіти marketplace
для претензій щодо авторського права або прав, якщо listing також не є небезпечним,
шкідливим або оманливим.

## Перевірка та застосування заходів

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали зловживань, звіти користувачів і
перевірку співробітниками, щоб виявляти небезпечний вміст або зловживальну поведінку публікації. Сигнал
сам по собі не доводить зловживання; він допомагає ClawHub вирішити, що потребує перевірки.

Ми можемо:

- приховувати, утримувати, видаляти, soft-delete або, якщо це підтримується для типу ресурсу,
  hard-delete listing, що порушують правила
- блокувати завантаження або встановлення небезпечних релізів
- відкликати API tokens
- soft-delete пов'язаний вміст
- обмежувати доступ до публікації
- банити повторних або серйозних порушників

Ми не гарантуємо застосування заходів спочатку з попередженням для очевидних зловживань. Див.
[Модерація та безпека акаунта](/clawhub/moderation) щодо звітів, модераційних утримань,
прихованих listing, банів і статусу акаунта.
