# Прийнятне використання

ClawHub розміщує Skills, Plugins, пакети та метадані маркетплейсу для OpenClaw.
Використовуйте цю сторінку, щоб визначити, чи належить вміст або поведінка
публікації до ClawHub.

Ці правила застосовуються до того, що робить запис, що він просить користувачів
запускати, як він представляє себе та як видавці використовують поверхні
виявлення, встановлення й довіри ClawHub. Про стани модерації та статус
облікового запису див. [Модерація та безпека облікового запису](/clawhub/moderation).
Про авторські права або інші правові претензії див.
[Запити щодо прав на вміст](/clawhub/content-rights).

## Дозволений вміст

ClawHub вітає вміст, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія                                        | Дозволено, коли                                                                                                                    |
| ------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробників                       | Запис допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення. |
| Робочі процеси UI, даних та автоматизації        | Обсяг чіткий, потрібні облікові дані вказані явно, а ризиковані дії мають шляхи перегляду, пробного запуску, попереднього перегляду або підтвердження. |
| Захисна безпека, модерація та перевірка зловживань | Інструмент подано для авторизованої перевірки, він зберігає докази та чітко підтримує межі схвалення людиною.                    |
| Особисті або командні робочі процеси             | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи.                                |
| Підтримувані каталоги                            | Кожен запис є окремим, корисним, точно описаним і належно підтримуваним.                                                           |

Контекст має значення. Та сама тема може бути прийнятною у вузькому захисному
або заснованому на згоді середовищі й неприйнятною, коли її упаковано як
робочий процес для зловживань.

## Заборонений вміст

ClawHub не розміщує вміст, основною метою якого є зловживання, обман,
небезпечне виконання або порушення прав.

| Категорія                                                   | Не дозволено                                                                                                                                                                                                                                                                                                  |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Несанкціонований доступ або обхід безпеки                   | Обхід автентифікації, захоплення облікового запису, зловживання обмеженнями швидкості, захоплення живого виклику або агента, багаторазове викрадення сесії або автоматичне схвалення потоків сполучення для несхвалених користувачів.                                                                         |
| Зловживання платформою та обхід блокувань                   | Приховані облікові записи після блокувань, «прогрів» або ферми облікових записів, фальшива взаємодія, автоматизація багатьох облікових записів, масові публікації, спам-боти або автоматизація, створена для уникнення виявлення.                                                                            |
| Шахрайство, афери та оманливі фінансові робочі процеси      | Фальшиві сертифікати або рахунки, оманливі платіжні потоки, шахрайські звернення, фальшиві соціальні докази, робочі процеси із синтетичною ідентичністю для шахрайства або інструменти витрат/стягнення коштів без чіткого схвалення людиною.                                                                |
| Інвазивне щодо приватності збагачення або стеження          | Збирання контактів для спаму, доксинг, переслідування, витяг лідів у поєднанні з небажаними зверненнями, прихований моніторинг, біометричне зіставлення без згоди або використання витоку даних чи дампів зламу.                                                                                               |

## Заборонена поведінка в маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не використовуйте
ClawHub для маніпулювання виявленням, метриками, сигналами довіри, системами
модерації або увагою користувачів.

Заборонена поведінка в маркетплейсі включає:

- масову публікацію великої кількості низькоякісних, дубльованих, заповнювальних
  або згенерованих машиною записів, які, схоже, не мають реальної цінності для користувачів
- заповнення пошуку або поверхонь категорій майже ідентичними Skills чи Plugins
- публікацію сотень записів із малим або відсутнім використанням, підтримкою,
  прозорістю джерела або змістовною відмінністю
- штучне збільшення встановлень, завантажень, зірок або інших метрик взаємодії
  через автоматизацію, цикли самовстановлення, фальшиві облікові записи,
  скоординовану активність, оплачувану взаємодію або іншу неорганічну поведінку
- створення або ротацію облікових записів для обходу модерації, блокувань,
  лімітів видавців або перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, стану
  безпеки, вимог до встановлення або афіліації з іншим проєктом чи видавцем
- повторне завантаження вмісту, який уже було приховано, видалено або заблоковано,
  без виправлення базової проблеми

Великий обсяг публікацій не є автоматично зловживанням. Великі каталоги
прийнятні, коли записи змістовно відрізняються, точно описані, підтримуються
та використовуються реальними користувачами. Великі каталоги стають проблемою
довіри й безпеки, коли обсяг поєднується з поверхневими, дубльованими,
оманливими, непідтримуваними або штучно просунутими записами.

## Права на вміст

Якщо ви вважаєте, що вміст на ClawHub порушує ваші авторські права або інші
права, використовуйте [Запити щодо прав на вміст](/clawhub/content-rights).
Не використовуйте звичайні звіти маркетплейсу для претензій щодо авторських
прав або інших прав, якщо запис також не є небезпечним, зловмисним або оманливим.

## Перевірка та застосування правил

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали
зловживань, звіти користувачів і перевірку персоналом, щоб виявляти небезпечний
вміст або зловживальну поведінку публікації. Сигнал сам по собі не доводить
зловживання; він допомагає ClawHub визначити, що потребує перевірки.

Ми можемо:

- приховувати, утримувати, видаляти, soft-delete або, де це підтримується для
  типу ресурсу, hard-delete записи, що порушують правила
- блокувати завантаження або встановлення небезпечних релізів
- відкликати API-токени
- soft-delete пов’язаний вміст
- обмежувати доступ до публікації
- блокувати повторних або серйозних порушників

Ми не гарантуємо застосування правил із попередженням спочатку для очевидних
зловживань. Див.
[Модерація та безпека облікового запису](/clawhub/moderation) щодо звітів,
модераційних утримань, прихованих записів, блокувань і статусу облікового запису.
