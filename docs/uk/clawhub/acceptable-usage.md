# Допустиме використання

Ця сторінка описує, які види навичок і контенту ClawHub приймає, а також робочі процеси зловживання, які він не розміщуватиме.

Ці правила навмисно практичні. Нас найбільше цікавлять наскрізні робочі процеси зловживання, а не лише ізольовані ключові слова. Якщо навичку створено для обходу захистів, зловживання платформами, шахрайства проти людей, вторгнення в приватність або уможливлення поведінки без згоди, їй не місце на ClawHub.

## Останні патерни, які ми явно приймаємо

- Робота з фронтендом і дизайн-системами, що використовує реальні компоненти, семантичні токени, доступні стани та протестовані користувацькі сценарії.
- Композиція shadcn/ui, що використовує встановлені вихідні компоненти, псевдоніми проєкту та задокументовані варіанти замість одноразової розмітки.
- Перетворення UI5 JavaScript на TypeScript зі збереженням коментарів, використанням конкретних типів UI5 і підтримкою згенерованих інтерфейсів елементів керування у придатному для ревʼю стані.
- Захисне ревʼю безпеки, інструменти модерації та промпти для виявлення зловживань, які показують докази й чітко зберігають межі людського схвалення.
- Автоматизація робочих процесів на основі згоди для особистих або командних облікових записів із явними обліковими даними, прозорим налаштуванням і режимами пробного запуску або попереднього перегляду.
- Документація, інструкції з міграції, утиліти для розробників і тестові фікстури, обмежені програмним забезпеченням, яке вони підтримують.

## Неприпустимо

- Робочі процеси обходу безпеки або несанкціонованого доступу.
  - Приклади: обхід автентифікації, захоплення облікового запису, обхід CAPTCHA, обхід Cloudflare або антибот-захисту, обхід обмежень частоти запитів, прихований скрейпінг, розроблений для подолання захистів, захоплення живого дзвінка або агента, багаторазова крадіжка сесій, автоматичне схвалення процесів спарювання для несхвалених користувачів.

- Зловживання платформами та обхід банів.
  - Приклади: приховані облікові записи після банів, прогрівання або фермерство облікових записів, фальшива взаємодія, нарощування карми чи підписників, автоматизація кількох облікових записів, масові публікації, спам-боти, автоматизація маркетплейсів або соціальних платформ, створена для уникнення виявлення.

- Шахрайство, скамерство та оманливі фінансові робочі процеси.
  - Приклади: фальшиві сертифікати, фальшиві рахунки-фактури, оманливі платіжні процеси, скам-розсилки, фальшиві соціальні докази, інструменти, що дають змогу витрачати кошти або стягувати плату без чіткого людського схвалення та прозорих засобів контролю, або робочі процеси синтетичної ідентичності, створені для створення облікових записів з метою шахрайства.

- Скрейпінг, збагачення даних або стеження, що порушують приватність.
  - Приклади: масовий скрейпінг контактних даних для спаму, доксинг, переслідування, витягування лідів у поєднанні з небажаними зверненнями, прихований моніторинг, пошук за обличчям або біометричне зіставлення без чіткої згоди, або купівля, публікація, завантаження чи операціоналізація витоків даних або дампів зламів.

- Імперсонація без згоди або оманливе маніпулювання ідентичністю.
  - Приклади: заміна облич, цифрові двійники, фальшиві персони, клоновані інфлюенсери або інші інструменти маніпулювання ідентичністю, що використовуються для імперсонації або введення в оману.

- Відвертий сексуальний контент і генерація дорослого контенту з вимкненими засобами безпеки.
  - Приклади: генерація NSFW-зображень, відео або контенту, обгортки для дорослого контенту навколо сторонніх APIs або навички, основним призначенням яких є відвертий сексуальний контент.

- Приховані, небезпечні або оманливі вимоги до виконання.
  - Приклади: обфусковані команди встановлення, `curl | sh`, незадекларовані вимоги до секретів, незадеклароване використання приватних ключів, віддалене виконання `npx @latest` без чіткої можливості ревʼю, оманливі метадані, що приховують, що насправді потрібно навичці для запуску.

## Останні патерни, які ми явно не приймаємо

- «Створювати приховані облікові записи продавців після банів на маркетплейсах».
- «Змінювати спарювання Telegram так, щоб несхвалені користувачі автоматично отримували коди спарювання».
- «Розвивати облікові записи Reddit/Twitter за допомогою невиявної автоматизації».
- «Генерувати професійні сертифікати або рахунки-фактури для довільного використання».
- «Генерувати NSFW-контент із вимкненими перевірками безпеки».
- «Скрейпити ліди, збагачувати контакти та запускати холодні розсилки в масштабі».
- «Купувати, публікувати або завантажувати витоки даних чи дампи зламів».
- «Масово створювати електронні поштові або соціальні облікові записи із синтетичними ідентичностями чи розвʼязанням CAPTCHA».

## Примітки для рецензентів

- Контекст має значення. Та сама тема може бути легітимною у вузькому захисному або заснованому на згоді середовищі й неприйнятною, коли її упаковано як робочий процес зловживання.
- Нам варто схилятися до дії, коли навичку явно оптимізовано для обходу, обману або використання без згоди.
- Повторні завантаження в цих категоріях є підставою для приховування контенту та бану облікового запису.

## Застосування правил

- Ми можемо приховувати, видаляти або безповоротно видаляти навички, що порушують правила.
- Ми можемо відкликати токени, мʼяко видаляти повʼязаний контент і банити повторних або серйозних порушників.
- Ми не гарантуємо застосування правил із попередженням спочатку у випадках очевидного зловживання.
