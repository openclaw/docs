# Прийнятне використання

Ця сторінка описує типи навичок і вмісту, які ClawHub допускає, а також сценарії зловживань, які він не розміщуватиме.

Ці правила навмисно практичні. Для нас найважливіші наскрізні сценарії зловживань, а не лише окремі ключові слова. Якщо навичку створено для обходу захистів, зловживання платформами, шахрайства щодо людей, вторгнення в приватність або уможливлення поведінки без згоди, їй не місце на ClawHub.

## Нещодавні патерни, які ми явно дозволяємо

- Робота з фронтендом і дизайн-системами, що використовує реальні компоненти, семантичні токени, доступні стани та протестовані користувацькі сценарії.
- Компонування shadcn/ui, що використовує встановлені вихідні компоненти, псевдоніми проєкту та задокументовані варіанти замість одноразової розмітки.
- Перетворення UI5 з JavaScript на TypeScript, що зберігає коментарі, використовує конкретні типи UI5 і залишає згенеровані інтерфейси елементів керування придатними для рев’ю.
- Оборонне рев’ю безпеки, інструменти модерації та промпти для виявлення зловживань, які показують докази й чітко зберігають межі людського схвалення.
- Автоматизація робочих процесів на основі згоди для особистих або командних облікових записів із явними обліковими даними, прозорим налаштуванням і режимами dry-run або попереднього перегляду.
- Документація, інструкції з міграції, утиліти для розробників і тестові фікстури, обмежені програмним забезпеченням, яке вони підтримують.

## Неприйнятно

- Сценарії обходу безпеки або несанкціонованого доступу.
  - Приклади: обхід автентифікації, захоплення облікового запису, обхід CAPTCHA, обхід Cloudflare або антибот-захисту, обхід обмежень частоти запитів, прихований скрейпінг, розроблений для подолання захистів, захоплення live-дзвінка або агента, повторно використовуване викрадення сесій, автоматичне схвалення потоків сполучення для несхвалених користувачів.

- Зловживання платформами та обхід банів.
  - Приклади: приховані облікові записи після банів, прогрівання/фермерство облікових записів, фейкова взаємодія, нарощування карми або підписників, автоматизація кількох облікових записів, масові публікації, спам-боти, автоматизація маркетплейсів або соціальних платформ, створена для уникнення виявлення.

- Шахрайство, сками та оманливі фінансові сценарії.
  - Приклади: фейкові сертифікати, фейкові рахунки, оманливі платіжні потоки, шахрайські звернення, фейкові соціальні докази, інструменти, що дають змогу витрачати кошти або стягувати плату без чіткого людського схвалення та прозорих контролів, або сценарії синтетичної ідентичності, створені для реєстрації облікових записів із метою шахрайства.

- Скрейпінг, збагачення або спостереження, що втручаються в приватність.
  - Приклади: масштабний скрейпінг контактних даних для спаму, доксинг, переслідування, витягування лідів у поєднанні з небажаними зверненнями, прихований моніторинг, пошук облич або біометричне зіставлення без чіткої згоди, або купівля, публікація, завантаження чи операціоналізація витеклих даних або дампів зламів.

- Імітація особи без згоди або оманливе маніпулювання ідентичністю.
  - Приклади: заміна облич, цифрові двійники, фейкові персони, клоновані інфлюенсери або інші інструменти маніпулювання ідентичністю, що використовуються для імітації особи чи введення в оману.

- Відвертий сексуальний контент і генерація дорослого контенту з вимкненою безпекою.
  - Приклади: генерація NSFW-зображень/відео/контенту, обгортки для дорослого контенту навколо сторонніх API або навички, основною метою яких є відвертий сексуальний контент.

- Приховані, небезпечні або оманливі вимоги до виконання.
  - Приклади: обфусковані команди встановлення, `curl | sh`, незадекларовані вимоги до секретів, незадеклароване використання приватних ключів, віддалене виконання `npx @latest` без чіткої придатності до рев’ю, оманливі метадані, що приховують, що насправді потрібно навичці для запуску.

## Нотатки для рев’юерів

- Контекст має значення. Та сама тема може бути легітимною у вузькому оборонному або заснованому на згоді середовищі й неприйнятною, коли її запаковано як сценарій зловживання.
- Ми маємо схилятися до дії, коли навичку явно оптимізовано для обходу, обману або використання без згоди.
- Повторні завантаження в цих категоріях є підставою для приховування вмісту та бану облікового запису.

## Застосування правил

- Ми можемо приховувати, видаляти або остаточно видаляти навички, що порушують правила.
- Ми можемо відкликати токени, м’яко видаляти пов’язаний вміст і банити повторних або серйозних порушників.
- Ми не гарантуємо застосування правил із попереднім попередженням у випадках очевидного зловживання.
