# Прийнятне використання

ClawHub розміщує навички, плагіни, пакети й метадані маркетплейсу для OpenClaw.
Використовуйте цю сторінку, щоб визначити, чи належать певний вміст або поведінка публікації до
ClawHub.

Ці правила застосовуються до того, що робить запис, що він просить користувачів запускати, як він
представляє себе та як видавці використовують поверхні ClawHub для пошуку, встановлення й
довіри. Стани модерації та статус облікового запису див. у
[Модерація та безпека облікового запису](/clawhub/moderation). Щодо заяв про авторське право або інші права
див. [Запити щодо прав на вміст](/uk/clawhub/content-rights).

## Дозволений вміст

ClawHub вітає вміст, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія                                         | Дозволено, коли                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробників                           | Запис допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення.                                               |
| Робочі процеси UI, даних та автоматизації               | Обсяг чіткий, потрібні облікові дані вказані явно, а ризиковані дії мають шляхи перегляду, пробного запуску, попереднього перегляду або підтвердження. |
| Захисна безпека, модерація та перегляд зловживань | Інструмент подано як призначений для авторизованої перевірки, він зберігає докази та чітко визначає межі людського схвалення.                          |
| Особисті або командні робочі процеси                       | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи.                                            |

Контекст має значення. Та сама тема може бути прийнятною у вузькому захисному або
заснованому на згоді середовищі та неприйнятною, якщо її упаковано як робочий процес для зловживань.

## Заборонена поведінка в маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не використовуйте ClawHub для
маніпулювання пошуком, метриками, сигналами довіри, системами модерації або увагою
користувачів.

Заборонена поведінка в маркетплейсі включає:

- масову публікацію великої кількості низькоякісних, дубльованих, заповнювальних або
  машинно згенерованих записів, які не мають очевидної реальної цінності для користувачів
- переповнення поверхонь пошуку або категорій майже ідентичними навичками чи плагінами
- публікацію сотень записів із незначним використанням або без нього, без підтримки, прозорості
  джерела чи змістовної відмінності
- штучне завищення встановлень, завантажень, зірок або інших метрик взаємодії
  через автоматизацію, цикли самовстановлення, фальшиві облікові записи, скоординовану
  активність, платну взаємодію або іншу неорганічну поведінку
- створення або ротацію облікових записів для обходу модерації, блокувань, лімітів видавця або
  перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, стану безпеки,
  вимог до встановлення або зв’язку з іншим проєктом чи видавцем
- повторне завантаження вмісту, який уже було приховано, видалено або заблоковано,
  без виправлення основної проблеми

Публікація великих обсягів сама по собі не є зловживанням. Великі каталоги прийнятні,
коли записи змістовно відрізняються, точно описані, підтримуються
та використовуються реальними користувачами. Великі каталоги стають проблемою довіри й безпеки, коли
обсяг поєднується з поверховими, дубльованими, оманливими, непідтримуваними або
штучно просуваними записами.

## Права на вміст

Якщо ви вважаєте, що вміст на ClawHub порушує ваше авторське право або інші права, використовуйте
[Запити щодо прав на вміст](/uk/clawhub/content-rights). Не використовуйте звичайні звіти маркетплейсу
для заяв про авторське право або інші права, якщо запис також не є небезпечним,
шкідливим або оманливим.

## Перевірка та застосування правил

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали зловживань, звіти користувачів і
перевірку персоналом, щоб виявляти небезпечний вміст або зловживальну поведінку публікації. Сигнал
сам по собі не доводить зловживання; він допомагає ClawHub вирішити, що потребує перевірки.

Ми можемо:

- приховувати, утримувати, видаляти, м’яко видаляти або, якщо це підтримується для типу ресурсу,
  остаточно видаляти записи, що порушують правила
- блокувати завантаження або встановлення небезпечних релізів
- відкликати API-токени
- м’яко видаляти пов’язаний вміст
- обмежувати доступ до публікації
- блокувати повторних або серйозних порушників

Ми не гарантуємо застосування правил із попередженням спершу для очевидних зловживань. Див.
[Модерація та безпека облікового запису](/clawhub/moderation) щодо звітів, модераційних утримань,
прихованих записів, блокувань і статусу облікового запису.
