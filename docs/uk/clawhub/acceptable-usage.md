# Прийнятне використання

ClawHub розміщує Skills, plugins, пакети та метадані маркетплейсу для OpenClaw.
Використовуйте цю сторінку, щоб визначити, чи належить вміст або поведінка публікації до
ClawHub.

Ці правила застосовуються до того, що робить запис, що він просить користувачів запускати, як він
представляє себе та як видавці використовують поверхні ClawHub для виявлення, встановлення й
довіри. Про стани модерації та статус облікового запису див.
[Модерація та безпека облікового запису](/clawhub/moderation). Щодо скарг про авторські або інші права
див. [Запити щодо прав на вміст](/uk/clawhub/content-rights).

## Дозволений вміст

ClawHub вітає вміст, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія                                        | Дозволено, коли                                                                                                                    |
| ------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробників                       | Запис допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення. |
| UI, дані та автоматизовані робочі процеси        | Обсяг чіткий, потрібні облікові дані явно вказані, а ризиковані дії мають шляхи перевірки, dry-run, попереднього перегляду або підтвердження. |
| Захисна безпека, модерація та перевірка зловживань | Інструмент подано для авторизованої перевірки, він зберігає докази й чітко підтримує межі людського схвалення.                    |
| Особисті або командні робочі процеси             | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи.                                |
| Підтримувані каталоги                            | Кожен запис є окремим, корисним, точно описаним і належно підтримуваним.                                                          |

Контекст має значення. Та сама тема може бути прийнятною у вузькому захисному або
заснованому на згоді середовищі й неприйнятною, коли її запаковано як робочий процес для зловживань.

## Заборонена поведінка в маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не використовуйте ClawHub, щоб
маніпулювати виявленням, метриками, сигналами довіри, системами модерації або
увагою користувачів.

Заборонена поведінка в маркетплейсі включає:

- масову публікацію великої кількості низькоякісних, дублювальних, шаблонних або
  машинно згенерованих записів, які, схоже, не мають реальної цінності для користувачів
- заповнення поверхонь пошуку або категорій майже ідентичними skills або plugins
- публікацію сотень записів із малим використанням або без нього, без підтримки, прозорості
  джерела чи значущої відмінності
- штучне завищення встановлень, завантажень, зірок або інших метрик взаємодії
  через автоматизацію, цикли самовстановлення, фальшиві облікові записи, скоординовану
  активність, оплачену взаємодію або іншу неорганічну поведінку
- створення або ротацію облікових записів для обходу модерації, банів, лімітів видавця або
  перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, стану безпеки,
  вимог до встановлення або пов’язаності з іншим проєктом чи видавцем
- повторне завантаження вмісту, який уже було приховано, видалено або заблоковано,
  без виправлення базової проблеми

Публікація у великих обсягах не є автоматично зловживанням. Великі каталоги прийнятні,
коли записи суттєво відрізняються, точно описані, підтримуються
і використовуються реальними користувачами. Великі каталоги стають проблемою довіри та безпеки, коли
обсяг поєднується з поверхневими, дублювальними, оманливими, непідтримуваними або
штучно просуваними записами.

## Права на вміст

Якщо ви вважаєте, що вміст на ClawHub порушує ваші авторські або інші права, використовуйте
[Запити щодо прав на вміст](/uk/clawhub/content-rights). Не використовуйте звичайні скарги маркетплейсу
для претензій щодо авторських або інших прав, якщо запис також не є небезпечним,
шкідливим або оманливим.

## Перевірка та застосування правил

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали зловживань, скарги користувачів і
перевірку персоналом, щоб виявляти небезпечний вміст або зловживальну поведінку публікації. Сигнал
сам собою не доводить зловживання; він допомагає ClawHub визначити, що потребує перевірки.

Ми можемо:

- приховати, призупинити, видалити, soft-delete або, якщо це підтримується для типу ресурсу,
  hard-delete записи, що порушують правила
- заблокувати завантаження або встановлення небезпечних релізів
- відкликати API-токени
- soft-delete пов’язаний вміст
- обмежити доступ до публікації
- забанити повторних або серйозних порушників

Ми не гарантуємо застосування правила «спочатку попередження» для очевидних зловживань. Див.
[Модерація та безпека облікового запису](/clawhub/moderation) щодо скарг, модераційних утримань,
прихованих записів, банів і статусу облікового запису.
