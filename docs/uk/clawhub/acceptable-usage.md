# Допустиме використання

ClawHub розміщує Skills, плагіни, пакети та метадані маркетплейсу для OpenClaw.
Скористайтеся цією сторінкою, щоб визначити, чи належать матеріали або спосіб публікації
до ClawHub.

Ці правила стосуються того, що робить публікація, що вона пропонує користувачам запустити, як
вона представляє себе та як видавці використовують можливості ClawHub для пошуку, установлення й
формування довіри. Відомості про стани модерації та статус облікового запису див.
у розділі [Модерація та безпека облікового запису](/clawhub/moderation). Щодо заяв про авторські чи інші права
див. [Запити щодо прав на матеріали](/clawhub/content-rights).

## Дозволені матеріали

ClawHub вітає корисні, зрозумілі та добросовісно
опубліковані матеріали.

| Категорія                                         | Дозволено, якщо                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробників                           | Публікація допомагає користувачам створювати, тестувати, переносити, налагоджувати, документувати або експлуатувати програмне забезпечення.                                               |
| Робочі процеси інтерфейсу, даних і автоматизації               | Обсяг застосування чіткий, необхідні облікові дані явно зазначені, а ризиковані дії передбачають перевірку, пробний запуск, попередній перегляд або підтвердження. |
| Захисна безпека, модерація та перевірка зловживань | Інструмент призначений для авторизованої перевірки, зберігає докази та чітко визначає межі людського схвалення.                          |
| Особисті або командні робочі процеси                       | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи.                                            |
| Підтримувані каталоги                              | Кожна публікація є окремою, корисною, точно описаною та належно підтримується.                                                |

Контекст має значення. Одна й та сама тема може бути прийнятною у вузькому захисному
середовищі або середовищі на основі згоди й неприйнятною, якщо її оформлено як робочий процес для зловживань.

## Заборонена поведінка на маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не використовуйте ClawHub для
маніпулювання пошуком, показниками, сигналами довіри, системами модерації або увагою
користувачів.

До забороненої поведінки на маркетплейсі належать:

- масова публікація великої кількості низькоякісних, дубльованих, шаблонних або
  згенерованих машиною публікацій, які, схоже, не мають реальної цінності для користувачів
- переповнення результатів пошуку чи категорій майже ідентичними Skills або плагінами
- публікація сотень матеріалів із незначним використанням або без нього, без підтримки, чіткого зазначення джерела
  чи суттєвих відмінностей
- штучне збільшення кількості встановлень, завантажень, зірок або інших показників
  взаємодії за допомогою автоматизації, циклів самостійного встановлення, фальшивих облікових записів, скоординованої
  активності, оплаченої взаємодії чи іншої неорганічної поведінки
- створення або зміна облікових записів для обходу модерації, блокувань, обмежень видавця чи
  перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, стану безпеки,
  вимог до встановлення або зв’язку з іншим проєктом чи видавцем
- повторне завантаження матеріалів, які вже були приховані, видалені або заблоковані,
  без усунення основної проблеми

Публікація у великих обсягах не є автоматично зловживанням. Великі каталоги прийнятні,
якщо публікації суттєво відрізняються, точно описані, підтримуються
та використовуються реальними користувачами. Великі каталоги стають проблемою довіри й безпеки, коли
великий обсяг поєднується з поверхневими, дубльованими, оманливими, непідтримуваними або
штучно просуваними публікаціями.

## Права на матеріали

Якщо ви вважаєте, що матеріали на ClawHub порушують ваші авторські чи інші права, скористайтеся
розділом [Запити щодо прав на матеріали](/clawhub/content-rights). Не використовуйте звичайні скарги маркетплейсу
для заяв про авторські чи інші права, якщо публікація водночас не є небезпечною,
зловмисною або оманливою.

## Перевірка та застосування заходів

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали зловживань, повідомлення користувачів і
перевірку співробітниками для виявлення небезпечних матеріалів або зловживань під час публікації. Окремий сигнал
сам по собі не доводить зловживання; він допомагає ClawHub визначити, що потребує перевірки.

Ми можемо:

- приховувати, затримувати, видаляти, виконувати м’яке видалення або, якщо це підтримується для відповідного типу ресурсу,
  остаточно видаляти публікації, що порушують правила
- блокувати завантаження або встановлення небезпечних випусків
- відкликати токени API
- виконувати м’яке видалення пов’язаних матеріалів
- обмежувати доступ до публікації
- блокувати повторних або серйозних порушників

Ми не гарантуємо попередження перед застосуванням заходів у випадках очевидного зловживання. Відомості про
скарги, призупинення через модерацію, приховані публікації, блокування та статус облікового запису див.
у розділі [Модерація та безпека облікового запису](/clawhub/moderation).
