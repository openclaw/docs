# Прийнятне використання

ClawHub розміщує Skills, plugins, пакети й метадані маркетплейсу для OpenClaw.
Скористайтеся цією сторінкою, щоб визначити, чи належить вміст або поведінка публікації до
ClawHub.

Ці правила поширюються на те, що робить сторінка пропозиції, що вона просить користувачів запустити, як вона
представляє себе та як видавці використовують засоби пошуку, встановлення й
довіри ClawHub. Відомості про стани модерації та статус облікового запису див.
у розділі [Модерація та безпека облікового запису](/clawhub/moderation). Відомості про претензії щодо авторських або інших прав
див. у розділі [Запити щодо прав на вміст](/clawhub/content-rights).

## Дозволений вміст

ClawHub вітає корисний, зрозумілий і добросовісно опублікований
вміст.

| Категорія                                         | Дозволено, якщо                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробників                           | Сторінка пропозиції допомагає користувачам створювати, тестувати, переносити, налагоджувати, документувати або експлуатувати програмне забезпечення.                                               |
| Робочі процеси інтерфейсу, даних і автоматизації               | Обсяг чітко визначений, необхідні облікові дані явно зазначені, а для ризикованих дій передбачено перевірку, пробний запуск, попередній перегляд або підтвердження. |
| Захисна безпека, модерація та перевірка зловживань | Інструмент призначений для авторизованої перевірки, зберігає докази та чітко визначає межі, у яких потрібне схвалення людини.                          |
| Особисті або командні робочі процеси                       | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явно визначені дозволи.                                            |
| Підтримувані каталоги                              | Кожна сторінка пропозиції є окремою, корисною, точно описаною та належно підтримується.                                                |

Контекст має значення. Та сама тематика може бути прийнятною у вузькому захисному середовищі або
середовищі, що ґрунтується на згоді, і неприйнятною, коли її оформлено як робочий процес для зловживань.

| Зловживання платформою та обхід блокування                              | Приховані облікові записи після блокування, підготовка або масове вирощування облікових записів, фальшива взаємодія, автоматизація багатьох облікових записів, масові публікації, спам-боти або автоматизація, створена для уникнення виявлення.                                                                                                                                          |

## Заборонена поведінка на маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не використовуйте ClawHub для
маніпулювання пошуком, показниками, сигналами довіри, системами модерації або увагою
користувачів.

Заборонена поведінка на маркетплейсі включає:

- масове публікування великої кількості недбало створених, дубльованих, шаблонних або
  машинно згенерованих сторінок пропозицій, які не мають очевидної реальної цінності для користувачів
- заповнення результатів пошуку або категорій майже ідентичними Skills чи plugins
- публікування сотень сторінок пропозицій, які майже або зовсім не використовуються, не підтримуються, не мають зрозумілого
  джерела чи суттєвих відмінностей
- штучне збільшення кількості встановлень, завантажень, зірок або інших показників
  взаємодії за допомогою автоматизації, циклів самовстановлення, фальшивих облікових записів, скоординованої
  активності, платної взаємодії чи іншої неорганічної поведінки
- створення або ротація облікових записів для обходу модерації, блокувань, обмежень видавців або
  перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, стану безпеки,
  вимог до встановлення або зв’язку з іншим проєктом чи видавцем
- повторне завантаження вмісту, який уже було приховано, видалено або заблоковано,
  без усунення основної проблеми

Публікація у великих обсягах не є автоматично зловживанням. Великі каталоги прийнятні,
якщо сторінки пропозицій суттєво відрізняються, точно описані, підтримуються
та використовуються реальними користувачами. Великі каталоги стають проблемою довіри й безпеки, коли
великий обсяг поєднується з поверхневими, дубльованими, оманливими, непідтримуваними або
штучно просуваними сторінками пропозицій.

## Права на вміст

Якщо ви вважаєте, що вміст на ClawHub порушує ваші авторські або інші права, скористайтеся
розділом [Запити щодо прав на вміст](/clawhub/content-rights). Не використовуйте звичайні скарги маркетплейсу
для претензій щодо авторських або інших прав, якщо сторінка пропозиції водночас не є небезпечною,
зловмисною чи оманливою.

## Перевірка та правозастосування

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали зловживань, повідомлення користувачів і
перевірку працівниками для виявлення небезпечного вмісту або зловживань під час публікації. Сам по собі сигнал
не доводить зловживання; він допомагає ClawHub визначити, що потребує перевірки.

Ми можемо:

- приховувати, затримувати, видаляти, програмно видаляти або, якщо це підтримується для цього типу ресурсу,
  остаточно видаляти сторінки пропозицій, що порушують правила
- блокувати завантаження або встановлення небезпечних випусків
- відкликати токени API
- програмно видаляти пов’язаний вміст
- обмежувати доступ до публікації
- блокувати повторних або серйозних порушників

Ми не гарантуємо попередження перед застосуванням заходів у разі очевидного зловживання. Відомості про
скарги, призупинення модерацією, приховані сторінки пропозицій, блокування та статус облікового запису див.
у розділі [Модерація та безпека облікового запису](/clawhub/moderation).
