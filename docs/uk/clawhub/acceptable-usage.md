# Прийнятне використання

ClawHub розміщує Skills, плагіни, пакети та метадані маркетплейсу для OpenClaw.
Використовуйте цю сторінку, щоб визначити, чи належить вміст або поведінка
публікації до ClawHub.

Ці правила застосовуються до того, що робить запис, що він просить користувачів
запускати, як він представляє себе та як видавці використовують поверхні
виявлення, встановлення й довіри ClawHub. Про стани модерації та статус
облікового запису див. [Модерація та безпека облікового запису](/clawhub/moderation).
Про претензії щодо авторського права або інших прав див.
[Запити щодо прав на вміст](/uk/clawhub/content-rights).

## Дозволений вміст

ClawHub вітає вміст, який є корисним, зрозумілим і опублікованим добросовісно.

| Категорія                                        | Дозволено, коли                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| Продуктивність розробників                       | Запис допомагає користувачам створювати, тестувати, мігрувати, налагоджувати, документувати або експлуатувати програмне забезпечення. |
| Робочі процеси UI, даних та автоматизації        | Обсяг чіткий, потрібні облікові дані явно вказані, а ризиковані дії мають шляхи перегляду, пробного запуску, попереднього перегляду або підтвердження. |
| Захисна безпека, модерація та перевірка зловживань | Інструмент подано для авторизованої перевірки, він зберігає докази та чітко визначає межі людського схвалення. |
| Особисті або командні робочі процеси             | Робочий процес використовує облікові записи на основі згоди, прозоре налаштування та явні дозволи. |
| Підтримувані каталоги                            | Кожен запис є окремим, корисним, точно описаним і належно підтримуваним. |

Контекст має значення. Та сама тема може бути прийнятною у вузькому захисному
або заснованому на згоді середовищі й неприйнятною, якщо її упаковано як
робочий процес для зловживань.

виконання або порушення прав.

| Категорія                                                   | Не дозволено                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Несанкціонований доступ або обхід безпеки                   | Обхід автентифікації, захоплення облікового запису, зловживання обмеженнями частоти, захоплення живого виклику або агента, повторно придатна крадіжка сесії чи автоматичне схвалення потоків сполучення для не затверджених користувачів. |
| Зловживання платформою та обхід блокувань                   | Приховані облікові записи після блокувань, прогрівання або ферми облікових записів, фальшива взаємодія, автоматизація кількох облікових записів, масове публікування, спам-боти або автоматизація, створена для уникнення виявлення. |

| Інвазивне щодо приватності збагачення або стеження          | Збирання контактів для спаму, доксинг, переслідування, витяг лідів у поєднанні з небажаними зверненнями, прихований моніторинг, біометричне зіставлення без згоди або використання витеклих даних чи дампів зламів. |

## Заборонена поведінка в маркетплейсі

ClawHub також перевіряє, як видавці використовують маркетплейс. Не використовуйте
ClawHub для маніпулювання виявленням, метриками, сигналами довіри, системами
модерації або увагою користувачів.

Заборонена поведінка в маркетплейсі включає:

- масову публікацію великої кількості низькоякісних, дублювальних, шаблонних або
  згенерованих машиною записів, які, схоже, не мають реальної цінності для користувачів
- засмічення пошуку або поверхонь категорій майже ідентичними Skills або плагінами
- публікацію сотень записів із незначним або відсутнім використанням, підтримкою,
  прозорістю джерела чи змістовною відмінністю
- штучне завищення встановлень, завантажень, зірок або інших метрик взаємодії
  через автоматизацію, цикли самовстановлення, фальшиві облікові записи,
  скоординовану активність, оплачену взаємодію або іншу неорганічну поведінку
- створення або ротацію облікових записів для обходу модерації, блокувань,
  лімітів видавця або перевірки маркетплейсу
- введення користувачів в оману щодо власності, джерела, можливостей, стану
  безпеки, вимог до встановлення або афіліації з іншим проєктом чи видавцем
- повторне завантаження вмісту, який уже було приховано, видалено або заблоковано,
  без усунення першопричини

Публікація у великому обсязі не є автоматично зловживанням. Великі каталоги
прийнятні, коли записи змістовно відрізняються, точно описані, підтримуються
та використовуються реальними користувачами. Великі каталоги стають проблемою
довіри й безпеки, коли обсяг поєднується з поверхневими, дублювальними,
оманливими, непідтримуваними або штучно просуваними записами.

## Права на вміст

Якщо ви вважаєте, що вміст на ClawHub порушує ваше авторське право або інші
права, скористайтеся [Запитами щодо прав на вміст](/uk/clawhub/content-rights).
Не використовуйте звичайні звіти маркетплейсу для претензій щодо авторського
права або прав, якщо запис також не є небезпечним, зловмисним або оманливим.

## Перевірка та застосування правил

ClawHub може використовувати автоматизовані перевірки, статистичні сигнали
зловживань, звіти користувачів і перевірку персоналом, щоб виявляти небезпечний
вміст або зловживальну поведінку публікації. Сигнал сам по собі не доводить
зловживання; він допомагає ClawHub визначити, що потребує перевірки.

Ми можемо:

- приховувати, утримувати, видаляти, м’яко видаляти або, якщо це підтримується
  для типу ресурсу, остаточно видаляти записи, що порушують правила
- блокувати завантаження або встановлення небезпечних релізів
- відкликати API-токени
- м’яко видаляти пов’язаний вміст
- обмежувати доступ до публікації
- блокувати повторних або серйозних порушників

Ми не гарантуємо застосування правил із попереднім попередженням для очевидних
зловживань. Див. [Модерацію та безпеку облікового запису](/clawhub/moderation)
щодо звітів, модераційних утримань, прихованих записів, блокувань і статусу
облікового запису.
