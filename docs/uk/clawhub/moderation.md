# Модерація та безпека облікових записів

ClawHub відкритий для публікації, але загальнодоступні засоби пошуку та встановлення все одно
потребують запобіжних заходів. Скарги, блокування на час модерації, приховані публікації та дії
щодо облікових записів допомагають захистити користувачів, коли випуск або обліковий запис видається небезпечним, оманливим чи таким,
що порушує правила.

На цій сторінці описано модерацію та статус облікового запису. Відомості про позначки аудиту, як-от
`Pass`, `Review`, `Warn`, `Malicious`, і рівень ризику див. в розділі
[Аудити безпеки](/clawhub/security-audits).

Див. також [Безпека](/clawhub/security) і
[Прийнятне використання](/clawhub/acceptable-usage). Щодо авторських прав або інших питань,
пов’язаних із правами на вміст, використовуйте [Запити щодо прав на вміст](/clawhub/content-rights).

## Скарги

Користувачі, які ввійшли в систему, можуть повідомляти про навички, плагіни та пакети.

Використовуйте скарги ClawHub лише щодо небезпечного вмісту маркетплейсу, наприклад:

- шкідливих публікацій
- оманливих метаданих
- незадекларованих облікових даних або вимог до дозволів
- підозрілих інструкцій зі встановлення
- видавання себе за іншу особу чи організацію
- недобросовісних реєстрацій або неправомірного використання торговельної марки
- вмісту, що порушує правила [Прийнятного використання](/clawhub/acceptable-usage)

Скористайтеся кнопкою **Report skill** на сторінці навички або командою/API
для повідомлення про пакети.

Не використовуйте скарги ClawHub для повідомлення про вразливості у власному вихідному коді
сторонньої навички або плагіна. Повідомляйте про них безпосередньо видавцю або в репозиторії
вихідного коду, посилання на який наведено в публікації. ClawHub не супроводжує й не виправляє
код сторонніх навичок або плагінів.

Рекомендації із безпеки GitHub для `openclaw/clawhub` призначені для вразливостей у
самому ClawHub. До них належать, наприклад, помилки на вебсайті, в API, CLI, реєстрі, автентифікації,
скануванні, модерації або межах довіри під час завантаження чи встановлення. Не використовуйте рекомендації
ClawHub для повідомлення про вразливості у сторонніх навичках або плагінах.

Якісні скарги конкретні та придатні для вжиття заходів. Зловживання системою скарг також може призвести до
дій щодо облікового запису.

## Претензії щодо організацій і просторів імен

Спори щодо права власності на організацію, бренд, область пакетів, ідентифікатор власника або простір імен слід
вирішувати за допомогою процедури [Претензії щодо організацій і просторів імен](/clawhub/namespace-claims), а не через
вбудований засіб подання скарг або форму оскарження дій щодо облікового запису.

Використовуйте цю процедуру, коли потрібно, щоб працівники ClawHub розглянули неконфіденційні докази того, що
простір імен слід зарезервувати, передати, перейменувати, приховати, помістити на карантин, призначити йому псевдонім
або іншим чином переглянути його статус. Не додавайте секрети, приватні документи, конфіденційні юридичні
файли, особисті документи, що посвідчують особу, токени API або токени перевірки DNS
до загальнодоступного обговорення.

## Блокування на час модерації

Деякі серйозні виявлені проблеми або порушення правил можуть призвести до блокування видавця чи публікації
на час модерації. У такому разі відповідний вміст може бути приховано від загальнодоступного
пошуку, а майбутні публікації можуть спочатку бути прихованими до розгляду проблеми.

Блокування на час модерації призначені для захисту користувачів, поки ClawHub розглядає випадки
високого ризику. Їх також можна зняти після підтвердження хибного спрацювання.

## Приховані або заблоковані публікації

Публікація може бути призупинена, прихована, поміщена на карантин, відкликана або з інших причин недоступна в
загальнодоступних засобах встановлення.

Якщо ви бачите один із цих статусів, не встановлюйте випуск, доки власник
не розв’яже проблему або модерація не відновить його доступність.

Власники можуть і надалі бачити діагностичні дані щодо власних призупинених або прихованих публікацій. Ці
дані допомагають зрозуміти, що сталося і що потрібно змінити, перш ніж
публікація зможе повернутися до загальнодоступних засобів.

## Заборони та статус облікового запису

Облікові записи, що порушують правила ClawHub, можуть втратити доступ до публікації. Серйозні зловживання можуть
призвести до заборони облікового запису, відкликання токенів, приховування вмісту або видалення публікацій.
Сигнали про ймовірне зловживання з боку видавця перевіряються щодня. Сигнали, що досягають
порогового значення ClawHub для потенційної заборони, можуть спричинити автоматичне попередження. Якщо наступне
відповідне сканування після завершення терміну попередження й надалі відносить видавця до
порогової категорії потенційної заборони, ClawHub може автоматично застосувати дію щодо облікового запису.
Сигнали з нижчим рівнем достовірності та обмеженим часовим періодом перевірки не підлягають автоматичному
застосуванню заходів.

Видалені, заблоковані або вимкнені облікові записи не можуть використовувати токени API ClawHub. Якщо автентифікація CLI
починає завершуватися помилкою після дії щодо облікового запису, увійдіть до вебінтерфейсу, щоб перевірити
його статус. Якщо вхід або звичайний доступ через CLI заблоковано через заборону чи вимкнення облікового запису,
скористайтеся [формою оскарження ClawHub](https://appeals.openclaw.ai/) для перегляду можливості відновлення.

Якщо в електронному листі, надісланому після спрацювання сканера, версію навички або плагіна названо шкідливою,
завантажте збережені результати сканування заблокованої надісланої версії:
`clawhub scan download <slug> --version <version>`. Для плагінів додайте
`--kind plugin`. Перегляньте результати сканування, виправте публікацію, збільште номер
версії та завантажте виправлену версію.

## Рекомендації для видавців

Щоб зменшити кількість хибних спрацювань і підвищити довіру користувачів:

- підтримуйте точність назв, описів, тегів і журналів змін
- зазначайте необхідні змінні середовища та дозволи
- уникайте обфускованих команд встановлення
- за можливості додавайте посилання на вихідний код
- використовуйте пробні запуски перед публікацією плагінів
- відповідайте чітко, якщо користувачі або модератори запитують про поведінку випуску
