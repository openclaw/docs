# Телеметрія

ClawHub використовує мінімальну телеметрію CLI для обчислення сукупної кількості встановлень.

## Коли збирається телеметрія

Телеметрія надсилається лише за таких умов:

- Ви ввійшли в обліковий запис у CLI.
- Ви запускаєте `clawhub install <slug>`.
- Телеметрію **не вимкнено** (див. «Як вимкнути» нижче).

Якщо ви не ввійшли в обліковий запис, жодні дані не передаються.

## Що ми збираємо

Під час кожного запуску `clawhub install`, про який надсилаються дані, CLI робить одну спробу надіслати подію встановлення.

Подія містить:

- `slug`: ідентифікатор установленого Skill.
- `version`: установлену версію, якщо вона відома.

### Чого ми _не_ збираємо

- Жодних шляхів до папок або ідентифікаторів, утворених на їх основі.
- Жодного вмісту файлів.
- Жодних журналів окремих запусків, запитів чи інших результатів роботи CLI.

## Кількість встановлень

ClawHub зберігає сукупні лічильники для кожного Skill:

- `installsAllTime`: унікальні користувачі, які повідомили принаймні про одне встановлення Skill через CLI.
- `installsCurrent`: унікальні користувачі, які повідомили про встановлення та не видалили свої
  телеметричні дані.

## Прозорість і керування користувачем

Усі бачать лише **сукупні лічильники встановлень**.

Під час видалення облікового запису також видаляються ваші телеметричні дані.

## Як вимкнути телеметрію

Установіть змінну середовища:

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

Якщо цю змінну встановлено, CLI не надсилатиме телеметричні дані про встановлення.
