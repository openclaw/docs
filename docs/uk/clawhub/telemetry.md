# Телеметрія

ClawHub використовує **мінімальну телеметрію**, щоб обчислювати **кількість встановлень** (що фактично використовується) і забезпечувати кращі сортування та фільтрування.
Це базується на команді CLI `clawhub sync`.

## Коли збирається телеметрія

Телеметрія надсилається лише коли:

- Ви **увійшли в систему** в CLI (ми вже вимагаємо автентифікацію для потоків sync/publish).
- Ви запускаєте `clawhub sync`.
- Телеметрію **не вимкнено** (див. «Як вимкнути» нижче).

Якщо ви не увійшли в систему, нічого не передається.

## Що ми збираємо

Під час кожного `clawhub sync` CLI повідомляє **повний знімок** того, що він знайшов, згрупований за коренем сканування («папка/корінь»).

Для кожного кореня ми зберігаємо:

- `rootId`: **SHA-256-хеш** канонічного шляху кореня (сервер ніколи не бачить необроблений шлях).
- `label`: зручна для читання мітка, отримана з двох останніх сегментів шляху (домашні шляхи показуються з `~`).
- `firstSeenAt`, `lastSeenAt`, необов’язково `expiredAt`.

Для кожної навички, знайденої під коренем, ми зберігаємо:

- `skillId` (визначається за slug; відстежуються лише навички, що існують у реєстрі).
- `firstSeenAt`, `lastSeenAt`.
- `lastVersion` (на основі найкращої спроби; наразі це версія, зіставлена з реєстром, якщо вона відома).
- необов’язково `removedAt`, коли раніше повідомлене встановлення зникає з кореня.

### Що ми _не_ збираємо

- Жодних необроблених абсолютних шляхів до папок (лише хешований `rootId` + коротка мітка для відображення).
- Жодного вмісту файлів.
- Жодних журналів окремих запусків, запитів або іншого виводу CLI.
- Жодного відстеження навичок, які не завантажені до реєстру (невідомі slugs ігноруються).

## Кількість встановлень

Ми підтримуємо два лічильники для кожної навички:

- `installsCurrent`: унікальні користувачі, у яких зараз навичку встановлено принаймні в одному активному корені.
- `installsAllTime`: унікальні користувачі, які будь-коли повідомляли про встановлену навичку.

### Кілька коренів

Якщо ви синхронізуєтеся з кількох папок, ми обробляємо кожен корінь сканування незалежно. Навичка вважається «зараз встановленою», якщо вона існує в **будь-якому** активному корені.

### Виявлення видалення

Оскільки `sync` повідомляє повний набір для кожного кореня:

- Якщо навичка зникає з кореня під час наступної синхронізації, ми позначаємо її як видалену для цього кореня.
- Якщо навичку видалено з усіх ваших коренів, вона більше не зараховується до `installsCurrent`.
- `installsAllTime` ніколи не зменшується, якщо ви не видалите телеметрію (див. нижче).

### Застарівання (120 днів)

Корені, які не повідомляють телеметрію протягом **120 днів**, позначаються як застарілі, і їхні встановлення перестають зараховуватися до `installsCurrent`.
Це оцінюється ліниво (під час наступного звіту телеметрії), щоб уникнути фонових завдань.

## Прозорість + засоби керування для користувача

ClawHub надає приватну вкладку «Встановлено» у вашому власному профілі:

- Показує точні корені + встановлені навички, які ми зберігаємо.
- Містить подання **експорту JSON**.
- Містить дію **Видалити телеметрію**, щоб видалити всю збережену телеметрію для вашого облікового запису.

Усі інші бачать лише **агреговані лічильники встановлень**; ніхто інший не може бачити ваші корені/папки.

Видалення вашого облікового запису також видаляє ваші дані телеметрії.

## Як вимкнути телеметрію

Установіть змінну середовища:

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

Коли її встановлено, CLI не надсилатиме телеметрію під час `clawhub sync`.
