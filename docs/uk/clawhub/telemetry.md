# Телеметрія

ClawHub використовує **мінімальну телеметрію**, щоб обчислювати **кількість встановлень** (що фактично використовується) і забезпечувати краще сортування/фільтрування.
Це базується на команді CLI `clawhub sync`.

## Коли збирається телеметрія

Телеметрія надсилається лише коли:

- Ви **увійшли в систему** в CLI (ми вже вимагаємо автентифікацію для процесів sync/publish).
- Ви запускаєте `clawhub sync`.
- Телеметрію **не вимкнено** (див. «Як вимкнути» нижче).

Якщо ви не ввійшли в систему, нічого не повідомляється.

## Що ми збираємо

Під час кожного `clawhub sync` CLI повідомляє **повний знімок** того, що знайшов, згрупований за коренем сканування («папка/корінь»).

Для кожного кореня ми зберігаємо:

- `rootId`: **SHA-256 хеш** канонічного шляху кореня (сервер ніколи не бачить необроблений шлях).
- `label`: зрозуміла для людини мітка, отримана з останніх двох сегментів шляху (домашні шляхи показуються з `~`).
- `firstSeenAt`, `lastSeenAt`, необов’язковий `expiredAt`.

Для кожного skill, знайденого під коренем, ми зберігаємо:

- `skillId` (визначається за slug; відстежуються лише skills, які існують у registry).
- `firstSeenAt`, `lastSeenAt`.
- `lastVersion` (наскільки можливо; наразі версія, зіставлена з registry, якщо відома).
- необов’язковий `removedAt`, коли раніше повідомлене встановлення зникає з кореня.

### Що ми _не_ збираємо

- Жодних необроблених абсолютних шляхів до папок (лише хешований `rootId` + коротка мітка для відображення).
- Жодного вмісту файлів.
- Жодних журналів окремих запусків, prompts або іншого виводу CLI.
- Жодного відстеження skills, які не завантажені до registry (невідомі slugs ігноруються).

## Кількість встановлень

Ми підтримуємо два лічильники для кожного skill:

- `installsCurrent`: унікальні користувачі, у яких зараз встановлено skill принаймні в одному активному корені.
- `installsAllTime`: унікальні користувачі, які будь-коли повідомляли про встановлений skill.

### Кілька коренів

Якщо ви синхронізуєте з кількох папок, ми обробляємо кожен корінь сканування незалежно. Skill вважається «зараз встановленим», якщо він існує в **будь-якому** активному корені.

### Виявлення видалення

Оскільки `sync` повідомляє повний набір для кожного кореня:

- Якщо skill зникає з кореня під час наступної синхронізації, ми позначаємо його як видалений для цього кореня.
- Якщо skill видалено з усіх ваших коренів, він більше не враховується в `installsCurrent`.
- `installsAllTime` ніколи не зменшується, якщо ви не видалите телеметрію (див. нижче).

### Застарілість (120 днів)

Корені, які не повідомляють телеметрію протягом **120 днів**, позначаються як застарілі, і їхні встановлення перестають враховуватися в `installsCurrent`.
Це оцінюється ліниво (під час наступного звіту телеметрії), щоб уникнути фонових завдань.

## Прозорість + керування користувачем

ClawHub надає приватну вкладку «Встановлено» у вашому власному профілі:

- Показує точні корені + встановлені skills, які ми зберігаємо.
- Містить подання **експорту JSON**.
- Містить дію **Видалити телеметрію**, щоб видалити всю збережену телеметрію для вашого облікового запису.

Усі інші бачать лише **агреговані лічильники встановлень**; ніхто інший не може бачити ваші корені/папки.

Видалення вашого облікового запису також видаляє ваші дані телеметрії.

## Як вимкнути телеметрію

Установіть змінну середовища:

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

Коли її встановлено, CLI не надсилатиме телеметрію під час `clawhub sync`.
