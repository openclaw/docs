# HTTP API

Базова URL-адреса: `https://clawhub.ai` (за замовчуванням).

Усі шляхи v1 розташовані в `/api/v1/...`.
Застарілі `/api/...` і `/api/cli/...` збережено для сумісності (див. `DEPRECATIONS.md`).
OpenAPI: `/api/v1/openapi.json`.

## Повторне використання публічного каталогу

Сторонні каталоги можуть використовувати публічні кінцеві точки читання, щоб виводити список або виконувати пошук Skills у ClawHub. Кешуйте результати, дотримуйтеся `429`/`Retry-After`, спрямовуйте користувачів на канонічну сторінку ClawHub (`https://clawhub.ai/<owner>/skills/<slug>`) і не створюйте враження, що ClawHub підтримує сторонній сайт. Не намагайтеся віддзеркалювати прихований, приватний або заблокований модерацією вміст поза публічною поверхнею API.

Вебскорочення ідентифікаторів у URL розпізнаються в усіх сімействах реєстру, але клієнтам API слід використовувати канонічні URL-адреси, повернуті кінцевими точками читання, замість самостійного відтворення пріоритету маршрутів.

## Обмеження частоти запитів

Модель застосування:

- Анонімні запити: обмеження застосовується до кожної IP-адреси.
- Автентифіковані запити (дійсний токен Bearer): обмеження застосовується до кошика кожного користувача.
- Якщо токен відсутній або недійсний, застосовується обмеження за IP-адресою.
- Автентифіковані кінцеві точки запису не повинні повертати лише `Unauthorized`, коли сервер знає причину. Для відсутніх токенів, недійсних або відкликаних токенів, а також видалених, заблокованих або вимкнених облікових записів слід повертати окремі дієві повідомлення, щоб клієнти CLI могли пояснити користувачам, що саме заблокувало їх.

- Читання: 3000/хв на IP-адресу, 12000/хв на ключ
- Запис: 300/хв на IP-адресу, 3000/хв на ключ
- Завантаження: 1200/хв на IP-адресу, 6000/хв на ключ (кінцеві точки завантаження)

Заголовки:

- Сумісність із застарілими клієнтами: `X-RateLimit-Limit`, `X-RateLimit-Reset`
- Стандартизовані: `RateLimit-Limit`, `RateLimit-Reset`
- У разі `429`: `X-RateLimit-Remaining: 0` і `RateLimit-Remaining: 0`
- У разі `429`: `Retry-After`

Семантика заголовків:

- `X-RateLimit-Reset`: абсолютний час у секундах епохи Unix
- `RateLimit-Reset`: кількість секунд до скидання (затримка)
- `X-RateLimit-Remaining` / `RateLimit-Remaining`: точний залишок ліміту, якщо вказано.
  Для успішних сегментованих запитів цей заголовок не повертається замість надання приблизного глобального значення.
- `Retry-After`: кількість секунд очікування перед повторною спробою (затримка) у разі `429`

Приклад відповіді `429`:

```http
HTTP/2 429
content-type: text/plain; charset=utf-8
x-ratelimit-limit: 20
x-ratelimit-remaining: 0
x-ratelimit-reset: 1771404540
ratelimit-limit: 20
ratelimit-remaining: 0
ratelimit-reset: 34
retry-after: 34

Rate limit exceeded
```

Рекомендації для клієнтів:

- Якщо наявний `Retry-After`, зачекайте вказану кількість секунд перед повторною спробою.
- Використовуйте затримку з випадковим відхиленням, щоб уникнути синхронізованих повторних спроб.
- Якщо `Retry-After` відсутній, використовуйте `RateLimit-Reset` (або обчисліть значення з `X-RateLimit-Reset`).

Джерело IP-адреси:

- Довірені заголовки IP-адреси клієнта, зокрема `cf-connecting-ip`, використовуються лише тоді, коли розгортання явно дозволяє довірені переслані заголовки.
- ClawHub використовує довірені заголовки пересилання для визначення IP-адрес клієнтів на межі мережі.
- Якщо довірена IP-адреса клієнта недоступна, анонімні запити використовують резервні кошики, область дії яких обмежена лише типом обмеження частоти запитів. Ці резервні кошики не включають надані викликачем шляхи, ідентифікатори в URL, назви пакетів, версії, рядки запиту або інші параметри артефактів.

## Відповіді з помилками

Публічні відповіді v1 з помилками є звичайним текстом із `content-type: text/plain; charset=utf-8`.
Це стосується помилок перевірки (`400`), відсутніх публічних ресурсів (`404`), помилок автентифікації та дозволів (`401`/`403`), обмежень частоти запитів (`429`) і заблокованих завантажень. Клієнтам слід читати тіло відповіді як зрозумілий людині рядок. Невідомі параметри запиту ігноруються задля сумісності, але розпізнані параметри запиту з недійсними значеннями повертають `400`.

## Публічні кінцеві точки (без автентифікації)

### `GET /api/v1/search`

Параметри запиту:

- `q` (обов’язковий): рядок запиту
- `limit` (необов’язковий): ціле число
- `highlightedOnly` (необов’язковий): `true`, щоб відфільтрувати лише виділені Skills
- `nonSuspiciousOnly` (необов’язковий): `true`, щоб приховати підозрілі (`flagged.suspicious`) Skills
- `nonSuspicious` (необов’язковий): застарілий псевдонім для `nonSuspiciousOnly`

Відповідь:

```json
{
  "results": [
    {
      "score": 0.123,
      "slug": "gifgrep",
      "displayName": "GifGrep",
      "summary": "…",
      "version": "1.2.3",
      "updatedAt": 1730000000000,
      "ownerHandle": "openclaw",
      "owner": {
        "handle": "openclaw",
        "displayName": "OpenClaw",
        "image": "https://example.com/avatar.png"
      }
    }
  ]
}
```

Примітки:

- Результати повертаються в порядку релевантності (подібність векторних представлень + підсилення точних збігів токенів ідентифікатора в URL або назви + невеликий початковий коефіцієнт популярності).
- Релевантність має більшу вагу, ніж популярність. Точний збіг токена ідентифікатора в URL або відображуваної назви може мати вищий рейтинг, ніж менш точний збіг із набагато вищою залученістю.
- Текст ASCII токенізується за межами слів і розділових знаків. Наприклад, `personal-map` містить окремий токен `map`, тоді як `amap-jsapi-skill` містить `amap`, `jsapi` і `skill`; тому пошук `map` надає `personal-map` сильніший лексичний збіг, ніж `amap-jsapi-skill`.
- Популярність масштабується логарифмічно й має верхню межу. Skills із високою залученістю можуть мати нижчий рейтинг, якщо текст запиту збігається слабше.
- Підозрілий або прихований стан модерації може вилучити Skills із публічного пошуку залежно від фільтрів викликача та поточного стану модерації.

Рекомендації щодо доступності для пошуку для видавців:

- Додавайте терміни, які користувачі буквально шукатимуть, до відображуваної назви, опису й тегів. Використовуйте окремий токен ідентифікатора в URL лише тоді, коли він також є стабільним ідентифікатором, який ви хочете зберегти.
- Не перейменовуйте ідентифікатор в URL лише заради одного запиту, якщо новий ідентифікатор не є кращою довгостроковою канонічною назвою. Старі ідентифікатори в URL стають псевдонімами переспрямування, але канонічна URL-адреса, показуваний ідентифікатор і майбутні огляди пошуку використовують новий ідентифікатор.
- Псевдоніми перейменування зберігають розпізнавання старих URL-адрес та інсталяцій, які визначають адресу через реєстр, але рейтинг пошуку після індексування перейменування ґрунтується на канонічних метаданих Skills. Наявна статистика залишається зі Skills.
- Якщо Skills несподівано не відображаються, перш ніж змінювати метадані, пов’язані з рейтингом, спочатку перевірте стан модерації за допомогою `clawhub inspect @owner/slug`, увійшовши до облікового запису.

### `GET /api/v1/skills`

Параметри запиту:

- `limit` (необов’язковий): ціле число (1–200)
- `cursor` (необов’язковий): курсор пагінації для будь-якого сортування, крім `trending`
- `sort` (необов’язковий): `updated` (за замовчуванням), `recommended` (псевдонім: `default`), `createdAt` (псевдонім: `newest`), `downloads`, `stars` (псевдонім: `rating`), застарілі псевдоніми інсталяцій `installsCurrent`/`installs`/`installsAllTime` зіставляються з `downloads`, `trending`
- `nonSuspiciousOnly` (необов’язковий): `true`, щоб приховати підозрілі (`flagged.suspicious`) Skills
- `nonSuspicious` (необов’язковий): застарілий псевдонім для `nonSuspiciousOnly`

Недійсні значення `sort` повертають `400`.

Примітки:

- `recommended` використовує сигнали залученості й актуальності.
- `trending` ранжує за кількістю інсталяцій протягом останніх 7 днів (на основі телеметрії).
- `createdAt` є стабільним для обходу нових Skills; `updated` змінюється після повторної публікації наявних Skills.
- Якщо `nonSuspiciousOnly=true`, сортування на основі курсора може повертати на сторінці менше ніж `limit` елементів, оскільки підозрілі Skills фільтруються після отримання сторінки.
- Використовуйте `nextCursor`, щоб продовжити пагінацію, якщо він наявний. Коротка сторінка сама по собі не означає завершення результатів.

Відповідь:

```json
{
  "items": [
    {
      "slug": "gifgrep",
      "displayName": "GifGrep",
      "summary": "…",
      "topics": ["Productivity"],
      "tags": { "latest": "1.2.3" },
      "stats": {},
      "createdAt": 0,
      "updatedAt": 0,
      "latestVersion": { "version": "1.2.3", "createdAt": 0, "changelog": "…" },
      "metadata": { "os": ["macos"], "systems": ["aarch64-darwin"] }
    }
  ],
  "nextCursor": null
}
```

Відповідь:

```json
{
  "skill": {
    "slug": "gifgrep",

    "topics": ["Productivity"],
    "tags": { "latest": "1.2.3" },
    "stats": {},
    "createdAt": 0,
    "updatedAt": 0
  },
  "latestVersion": { "version": "1.2.3", "createdAt": 0, "changelog": "…" },
  "metadata": { "os": ["macos"], "systems": ["aarch64-darwin"] },
  "owner": { "handle": "steipete", "displayName": "Peter", "image": null },
  "moderation": {
    "isSuspicious": false,
    "isMalwareBlocked": false,
    "verdict": "clean",
    "reasonCodes": [],
    "summary": null,
    "engineVersion": "v2.0.0",
    "updatedAt": 0
  }
}
```

Примітки:

- Старі ідентифікатори в URL, створені внаслідок перейменування або об’єднання власником, розпізнаються як канонічні Skills.
- `metadata.os`: обмеження ОС, оголошені у вступному блоці Skills (наприклад, `["macos"]`, `["linux"]`). `null`, якщо не оголошено.
- `metadata.systems`: цільові системи Nix (наприклад, `["aarch64-darwin", "x86_64-linux"]`). `null`, якщо не оголошено.
- `metadata` має значення `null`, якщо Skills не мають метаданих платформи.
- `moderation` включається лише тоді, коли Skills позначено або їх переглядає власник.

- Докази редагуються для публічних викликачів і містять необроблені фрагменти лише для власників і модераторів.

### `POST /api/v1/skills/{slug}/report`

Повідомляє про Skills для перевірки модератором. Повідомлення стосуються Skills загалом, за потреби пов’язуються з версією та надходять до черги повідомлень про Skills.

Автентифікація:

- Потрібен токен API.

Запит:

```json
{ "reason": "Suspicious install step", "version": "1.2.3" }
```

Відповідь:

```json
{
  "ok": true,
  "reported": true,
  "alreadyReported": false,
  "reportId": "skillReports:...",
  "skillId": "skills:...",
  "reportCount": 1
}
```

### `GET /api/v1/skills/-/reports`

Кінцева точка модератора або адміністратора для приймання повідомлень про Skills.

Параметри запиту:

- `status` (необов’язковий): `open` (за замовчуванням), `confirmed`, `dismissed` або `all`
- `limit` (необов’язковий): ціле число (1–200)
- `cursor` (необов’язковий): курсор пагінації

Відповідь:

```json
{
  "items": [

      "skillVersionId": "skillVersions:...",

      "displayName": "GifGrep",
      "version": "1.2.3",
      "reason": "Suspicious install step",
      "status": "open",
      "createdAt": 1730000000000,
      "reporter": {
        "userId": "users:...",
        "handle": "reporter",
        "displayName": "Reporter"
      },
      "triagedAt": null,
      "triagedBy": null,
      "triageNote": null
    }
  ],
  "nextCursor": null,
  "done": true
}
```

### `POST /api/v1/skills/-/reports/{reportId}/triage`

Кінцева точка модератора або адміністратора для вирішення або повторного відкриття повідомлень про Skills.

Запит:

```json
{ "status": "confirmed", "note": "Reviewed and hid affected version.", "finalAction": "hide" }
```

`note` є обов’язковим для `confirmed` і `dismissed`; його можна опустити, коли `status` знову встановлюється на `open`. Передайте `finalAction: "hide"` разом з опрацьованим повідомленням, щоб приховати Skills у межах того самого робочого процесу з можливістю аудиту.

### `GET /api/v1/skills/{slug}/versions`

Параметри запиту:

- `limit` (необов’язковий): ціле число
- `cursor` (необов’язковий): курсор пагінації

### `GET /api/v1/skills/{slug}/versions/{version}`

Повертає метадані версії та список файлів.

- `version.security` містить нормалізований стан перевірки сканування й відомості про сканери (VirusTotal + LLM), якщо вони доступні.

### `GET /api/v1/skills/{slug}/scan`

Повертає подробиці перевірки сканування безпеки для версії Skills.

Параметри запиту:

- `version` (необов’язковий): конкретний рядок версії.
- `tag` (необов’язковий): визначити позначену версію (наприклад, `latest`).

Примітки:

- Якщо не вказано ні `version`, ні `tag`, використовується найновіша версія.
- Містить нормалізований статус перевірки та відомості, специфічні для кожного сканера.
- `security.hasScanResult` має значення `true`, лише коли сканер надав однозначний вердикт (`clean`, `suspicious` або `malicious`).
- `moderation` — це поточний знімок модерації на рівні Skills, сформований на основі найновішої версії.
- Під час запиту історичної версії перевіряйте `moderation.matchesRequestedVersion` і `moderation.sourceVersion`, перш ніж вважати `moderation` і `security` контекстом тієї самої версії.

### `POST /api/v1/skills/-/scan`

Автентифікована кінцева точка надсилання нових завдань ClawScan.

Сканування локально завантажених файлів більше не підтримується. Запити з
`multipart/form-data` або `{ "source": { "kind": "upload" } }` повертають `410`.

Для сканування опублікованих версій використовується JSON:

```json
{
  "source": { "kind": "published", "slug": "gifgrep", "version": "1.2.3" },
  "update": false
}
```

Примітки:

- Корисні навантаження запитів на сканування та звіти для завантаження видаляються зі сховища запитів на сканування після завершення строку зберігання.
- Для сканування опублікованих версій потрібен керівний доступ власника або видавця чи повноваження модератора або адміністратора платформи.
- Результати сканування опублікованих версій записуються назад, лише якщо `update: true` і сканування успішно завершено.
- Відповідь має статус `202` і містить `{ "ok": true, "scanId": "...", "jobId": "...", "status": "queued", "sourceKind": "published", "update": false, "queue": { "queuedAhead": 0, "queuedAheadIsEstimate": false, "position": 1, "running": 0, "runningIsEstimate": false, "note": "Scans are asynchronous and may take time to complete." } }`.
- Завдання сканування виконуються асинхронно. Ручні запити на сканування мають пріоритет над звичайними завданнями публікації та ретроспективної обробки, але їх завершення все одно залежить від доступності робочих процесів.

### `GET /api/v1/skills/-/scan/{scanId}`

Автентифікована кінцева точка опитування стану надісланого сканування.

- Повертає стан у черзі, виконання, успішного завершення або помилки.
- Поки запит перебуває в черзі, повертає `queue.queuedAhead` і `queue.position`, щоб клієнти могли показати кількість пріоритетних ручних сканувань перед цим запитом. Розмір дуже великих черг обмежується, а дані про них повертаються з `queuedAheadIsEstimate: true`.
- За наявності `report` містить розділи `clawscan`, `skillspector`, `staticAnalysis` і `virustotal`.
- Для невдалих завдань сканування повертається `status: "failed"` із `lastError`.

### `GET /api/v1/skills/-/scan/{scanId}/download`

Автентифікована кінцева точка архіву звіту.

- Потребує успішно завершеного сканування; сканування, які ще не досягли кінцевого стану, повертають `409`.
- Повертає ZIP-файл із `manifest.json`, `clawscan.json`, `skillspector.json`, `static-analysis.json`, `virustotal.json` і `README.md`.

### `GET /api/v1/skills/-/scan/download/{name}?version=<version>&kind=skill|plugin`

Автентифікована кінцева точка архіву збережених звітів для надісланих версій.

- Потребує керівного доступу власника або видавця до Skills чи плагіна або повноважень модератора чи адміністратора платформи.
- Повертає збережені результати сканування для точної надісланої версії, зокрема заблокованих або прихованих версій.
- Типовим значенням `kind` є `skill`; для сканування плагінів або пакетів використовуйте `kind=plugin`.
- Повертає ZIP-файл тієї самої структури, що й завантаження для запитів на сканування.

### `GET /api/v1/skills/{slug}/verify`

Повертає конверт перевірки картки Skills, який використовує `clawhub skill verify`.

Параметри запиту:

- `version` (необов’язковий): рядок конкретної версії.
- `tag` (необов’язковий): визначити версію за тегом (наприклад, `latest`).

Примітки:

- `ok` має значення `true`, лише коли для вибраної версії створено картку Skills, версію не заблоковано модерацією через шкідливе програмне забезпечення, а перевірка ClawScan не виявила загроз.
- Ідентичність Skills, ідентичність видавця та метадані вибраної версії є полями верхнього рівня конверта (`slug`, `displayName`, `publisherHandle`, `version`, `resolvedFrom`, `tag`, `createdAt`), щоб автоматизація оболонки могла читати їх без розпакування вкладених обгорток.
- `security` — це вердикт ClawScan або перевірки безпеки верхнього рівня. Автоматизація має орієнтуватися на `ok`, `decision`, `reasons` і `security.status`.
- `security.signals` містить допоміжні свідчення сканерів, як-от `staticScan`, `virusTotal` і `skillSpector`.
- `security.signals.dependencyRegistry` збережено для сумісності відповіді v1, але сканер наявності в реєстрі залежностей виведено з експлуатації, тому значення цього ключа завжди дорівнює `null`.
- `provenance` має значення `server-resolved-github-import`, лише коли ClawHub визначив і зберіг репозиторій, посилання, коміт і шлях GitHub під час публікації чи імпорту; інакше значенням є `unavailable`.

### `POST /api/v1/skills/-/security-verdicts`

Повертає поточні компактні вердикти безпеки для точних версій Skills. Ця
кінцева точка колекції призначена для клієнтів, які вже знають, які версії
встановлених Skills ClawHub їм потрібно відобразити, наприклад OpenClaw Control UI.

Запит:

```json
{
  "items": [{ "slug": "gifgrep", "version": "1.2.3" }]
}
```

Примітки:

- `items` має містити від 1 до 100 унікальних пар `{ slug, version }`.
- Результати повертаються для кожного елемента окремо; відсутність однієї Skills або версії не спричиняє помилку всієї відповіді.
- Відповідь містить лише дані безпеки. Вона не містить даних картки Skills, стану створення картки, списків файлів артефакту чи докладних корисних навантажень сканерів.
- `security.signals` містить лише допоміжні свідчення на рівні стану; для отримання повних відомостей сканерів використовуйте `/scan` або сторінку аудиту безпеки ClawHub.
- `security.signals.dependencyRegistry` збережено для сумісності відповіді v1, але сканер наявності в реєстрі залежностей виведено з експлуатації, тому значення цього ключа завжди дорівнює `null`.
- Відсутність картки Skills не впливає на `ok`, `decision` або `reasons` цієї кінцевої точки; коли клієнтам потрібен вміст картки, вони мають локально читати встановлений файл `skill-card.md`.
- Використовуйте `/verify`, коли потрібен конверт перевірки картки однієї Skills, `/card`, коли потрібен створений Markdown картки, і `/scan`, коли потрібні докладні дані сканерів.

Відповідь:

```json
{
  "schema": "clawhub.skill.security-verdicts.v1",
  "items": [
    {
      "ok": true,
      "decision": "pass",
      "reasons": [],
      "requestedSlug": "gifgrep",
      "slug": "gifgrep",
      "displayName": "GifGrep",
      "publisherHandle": "steipete",
      "publisherDisplayName": "Peter",
      "requestedVersion": "1.2.3",
      "version": "1.2.3",
      "createdAt": 0,
      "checkedAt": 0,
      "skillUrl": "https://clawhub.ai/steipete/skills/gifgrep",
      "securityAuditUrl": "https://clawhub.ai/steipete/skills/gifgrep/security-audit?version=1.2.3",
      "security": {
        "status": "clean",
        "passed": true,
        "signals": {
          "staticScan": { "status": "clean", "reasonCodes": [] },
          "virusTotal": null,
          "skillSpector": null,
          "dependencyRegistry": null
        }
      }
    },
    {
      "ok": false,
      "decision": "fail",
      "reasons": ["version.not_found"],
      "requestedSlug": "missing-version",
      "requestedVersion": "1.0.0",
      "error": { "code": "version_not_found", "message": "Version not found" },
      "security": null
    }
  ]

```

### `GET /api/v1/packages`

- `GET /api/v1/code-plugins` і `GET /api/v1/bundle-plugins` залишаються псевдонімами з фіксованим сімейством.
- Записи Skills і надалі зберігаються в реєстрі Skills та можуть публікуватися лише через `POST /api/v1/skills`.
- `POST /api/v1/packages` і надалі призначений лише для випусків плагінів коду та пакетних плагінів.
- Анонімні клієнти бачать лише загальнодоступні канали пакетів.
- Автентифіковані клієнти можуть бачити в результатах переліку й пошуку приватні пакети видавців, до яких вони належать.

Повертає одну версію пакета, включно з метаданими файлів, сумісністю,
верифікацією, метаданими артефакту та даними сканування.

Примітки:

- `version.artifact.kind` має значення `legacy-zip` для архівів пакетів старого формату або
  `npm-pack` для випусків на основі ClawPack.
- Випуски ClawPack містять сумісні з npm поля `npmIntegrity`, `npmShasum` і
  `npmTarballName`.
- `version.sha256hash` — застарілі метадані сумісності для старих клієнтів. Це поле
  містить хеш точних байтів ZIP, повернених `/api/v1/packages/{name}/download`.
  Сучасним клієнтам слід використовувати `version.artifact.sha256`, яке ідентифікує
  канонічний артефакт випуску.
- `version.vtAnalysis`, `version.llmAnalysis` і `version.staticScan`
  додаються за наявності даних сканування.
- Для приватних пакетів повертається `404`, якщо викликач не має права читати дані видавця-власника.

### `GET /api/v1/packages/{name}/versions/{version}/security`

Повертає точне зведення про безпеку й довіру для випуску пакета, призначене для клієнтів
встановлення. Це публічна поверхня OpenClaw для визначення можливості встановлення
визначеного випуску.

Автентифікація:

- Публічна кінцева точка для читання. Токен власника, видавця, модератора чи адміністратора
  не потрібен.

Відповідь:

```json
{
  "package": {
    "name": "@openclaw/example-plugin",
    "displayName": "Example Plugin",
    "family": "code-plugin"
  },
  "release": {
    "releaseId": "packageReleases:...",
    "version": "1.2.3",
    "artifactKind": "npm-pack",
    "artifactSha256": "0123456789abcdef...",
    "npmIntegrity": "sha512-...",
    "npmShasum": "0123456789abcdef0123456789abcdef01234567",
    "npmTarballName": "example-plugin-1.2.3.tgz",
    "createdAt": 1730000000000
  },
  "trust": {
    "scanStatus": "malicious",
    "moderationState": "quarantined",
    "blockedFromDownload": true,
    "reasons": ["manual:quarantined", "scan:malicious"],
    "pending": false,
    "stale": false
  }
}
```

Поля відповіді:

- `package.name`, `package.displayName` і `package.family` ідентифікують
  визначений пакет реєстру.
- `release.releaseId`, `release.version` і `release.createdAt` ідентифікують
  конкретний оцінений випуск.
- `release.artifactKind`, `release.artifactSha256`, `release.npmIntegrity`,
  `release.npmShasum` і `release.npmTarballName` наявні, якщо вони відомі для
  артефакту випуску.
- `trust.scanStatus` — фактичний статус довіри, визначений на основі даних сканерів
  і ручної модерації випуску.
- `trust.moderationState` може мати значення `null`. Воно дорівнює `null`, якщо ручна
  модерація випуску відсутня.
- `trust.blockedFromDownload` — сигнал блокування встановлення. OpenClaw та інші
  клієнти встановлення мають блокувати встановлення, коли це значення дорівнює `true`, замість
  повторного виведення правил блокування з полів сканування або модерації.
- `trust.reasons` — список пояснень для користувача й аудиту. Коди причин —
  стабільні компактні рядки, як-от `manual:quarantined`, `scan:malicious`
  і `package:malicious`.
- `trust.pending` означає, що один або кілька вхідних сигналів довіри все ще очікують завершення.
- `trust.stale` означає, що зведення про довіру обчислено на основі застарілих даних і
  перед ухваленням рішення про дозвіл із високою впевненістю його потрібно оновити.

Примітки:

- Ця кінцева точка прив’язана до конкретної версії. Клієнтам слід викликати її після визначення
  версії пакета, яку вони планують встановити, а не лише після читання метаданих
  останньої версії пакета.
- Для приватних пакетів повертається `404`, якщо викликач не має права читати дані видавця-власника.
- Ця кінцева точка навмисно має вужчу область, ніж кінцеві точки модерації
  для власників і модераторів. Вона надає рішення щодо встановлення та публічне пояснення, але не
  особи авторів скарг, вміст скарг, приватні докази чи внутрішні часові межі
  перевірки.

- `tag` (необов’язковий)

Відповідь:

```json
{ "slug": "gifgrep", "match": { "version": "1.2.2" }, "latestVersion": { "version": "1.2.3" } }
```

### `GET /api/v1/download`

Завантажує ZIP розміщеної версії Skills або повертає перенаправлення до джерела GitHub для
поточної Skills на основі GitHub зі скануванням `clean` чи `suspicious` і без розміщеної
версії.

Параметри запиту:

- `slug` (обов’язковий)
- `version` (необов’язковий): рядок semver
- `tag` (необов’язковий): назва тегу (наприклад, `latest`)

Примітки:

- Якщо не вказано ні `version`, ні `tag`, використовується найновіша версія.
- Для версій із м’яким видаленням повертається `410`.
- Перенаправлення Skills на основі GitHub не проксіюють і не дзеркалюють байти. JSON-відповідь
  містить `sourceRef: "public-github"`, `repo`, `commit`, `path`, `contentHash`
  і `archiveUrl`; стан сканування/актуальності є умовою доступу й не включається до метаданих
  успішної відповіді.
- Статистика завантажень підраховується за унікальними ідентичностями протягом кожної доби UTC (`userId`, якщо API-токен дійсний, інакше IP-адреса).

## Кінцеві точки автентифікації (токен Bearer)

Для всіх кінцевих точок потрібно:

```
Authorization: Bearer clh_...
```

### `GET /api/v1/whoami`

Перевіряє токен і повертає псевдонім користувача.

### `POST /api/v1/skills`

Публікує нову версію.

- Рекомендовано: `multipart/form-data` з JSON у полі `payload` і бінарними об’єктами `files[]`.
- Також приймається тіло JSON з `files` (на основі storageId).
- Необов’язкове поле корисного навантаження: `ownerHandle`. Якщо воно наявне, API визначає цього
  видавця на сервері та вимагає, щоб виконавець мав доступ до видавця.
- Необов’язкове поле корисного навантаження: `migrateOwner`. Якщо значення `true` використовується з `ownerHandle`,
  наявну Skills можна перемістити до цього власника, якщо виконавець є адміністратором/власником обох
  видавців — поточного й цільового. Без явної згоди на це зміну власника
  буде відхилено.

### `POST /api/v1/packages`

Публікує випуск `code-plugin` або `bundle-plugin`.

- Потребує автентифікації за допомогою токена Bearer.
- Потребує `multipart/form-data`.
- Дозволені поля форми: `payload`, повторювані бінарні об’єкти `files` або одне посилання на tarball-архів
  `clawpack`. `clawpack` може бути бінарним об’єктом `.tgz` або ідентифікатором сховища, повернутим
  процесом отримання URL завантаження. Поетапні публікації за ідентифікатором сховища також мають містити
  `clawpackUploadTicket`, повернений разом із цим URL завантаження.
- Використовуйте або `files`, або `clawpack`, але ніколи не обидва варіанти в одному запиті.
- Тіла JSON і надані викликачем метадані `payload.files` / `payload.artifact`
  відхиляються.
- Прямі запити публікації multipart обмежено 18 МБ. Tarball-архіви ClawPack можуть
  використовувати процес отримання URL завантаження до граничного розміру tarball-архіву 120 МБ.
- Необов’язкове поле корисного навантаження: `ownerHandle`. Якщо воно наявне, лише адміністратори можуть публікувати від імені цього власника.

Основні правила перевірки:

- `family` має бути `code-plugin` або `bundle-plugin`.
- Пакети Plugin потребують `openclaw.plugin.json`. Завантаження ClawPack `.tgz` мають
  містити його за шляхом `package/openclaw.plugin.json`.
- Plugin коду потребують `package.json`, метадані репозиторію джерельного коду, метадані
  коміту джерельного коду, метадані схеми конфігурації, `openclaw.compat.pluginApi` і
  `openclaw.build.openclawVersion`.
- `openclaw.hostTargets` і `openclaw.environment` є необов’язковими метаданими.
- Публікувати в каналі `official` можуть лише видавець організації `openclaw` та особисті
  видавці поточних учасників організації `openclaw`.
- Для публікацій від імені іншого користувача право на офіційний канал усе одно перевіряється за обліковим записом цільового власника.

### `DELETE /api/v1/skills/{slug}` / `POST /api/v1/skills/{slug}/undelete`

М’яко видаляє / відновлює Skills (власник, модератор або адміністратор).

Необов’язкове тіло JSON:

```json
{ "reason": "Held for moderation pending legal review." }
```

Якщо `reason` вказано, значення зберігається як примітка модерації Skills і копіюється до журналу аудиту.
М’які видалення, ініційовані власником, резервують slug на 30 днів, після чого його може зареєструвати
інший видавець. Відповідь на видалення містить `slugReservedUntil`, коли застосовується цей термін.
Приховування модератором/адміністратором і видалення з міркувань безпеки не мають такого терміну дії.

Відповідь на видалення:

```json
{ "ok": true, "slugReservedUntil": 1730000000000 }
```

Коди стану:

- `200`: успішно
- `401`: не автентифіковано
- `403`: доступ заборонено
- `404`: Skills/користувача не знайдено
- `500`: внутрішня помилка сервера

### `POST /api/v1/users/publisher`

Лише для адміністраторів. Забезпечує наявність видавця організації для псевдоніма. Якщо псевдонім усе ще вказує на
застарілого спільного користувача/особистого видавця, кінцева точка спочатку перетворює його на видавця організації.
Для новоствореної організації вкажіть `memberHandle`; активний адміністратор не додається як учасник.
Типове значення `memberRole` — `owner`.

- Тіло: `{ "handle": "openclaw", "displayName": "OpenClaw", "memberHandle": "alice", "memberRole": "owner", "trusted": true }`
- Відповідь: `{ "ok": true, "publisherId": "...", "handle": "openclaw", "created": true, "migrated": false, "trusted": true, "member": { "userId": "...", "handle": "alice", "role": "owner" } }`

### `POST /api/v1/publishers`

Автентифіковане самостійне створення видавця організації. Створює нового видавця організації та додає
викликача як власника. Ця кінцева точка не переносить наявні користувацькі/особисті псевдоніми та
не позначає видавця як довіреного/офіційного.

- Тіло: `{ "handle": "opik", "displayName": "Opik" }`
- Відповідь: `{ "ok": true, "publisherId": "...", "handle": "opik", "created": true, "trusted": false }`
- Повертає `409`, якщо псевдонім уже використовується видавцем, користувачем або особистим видавцем.

### `POST /api/v1/users/reserve`

Лише для адміністраторів. Резервує кореневі slug і назви пакетів для законного власника без публікації
випуску. Назви пакетів стають приватними пакетами-заповнювачами без рядків випусків, щоб той самий
власник згодом міг опублікувати справжній випуск `code-plugin` або `bundle-plugin` із цією назвою.

- Тіло: `{ "handle": "openclaw", "slugs": ["diffs"], "packageNames": ["@openclaw/diffs"], "reason": "reserved for official OpenClaw plugin" }`
- Відповідь: `{ "ok": true, "succeeded": 2, "failed": 0, "results": [{ "kind": "slug", "name": "diffs", "ok": true, "action": "reserved" }] }`

### `POST /api/v1/users/publisher-recovery`

Лише для адміністраторів. Відновлює особистого видавця для перевіреного нового суб’єкта GitHub OAuth
без редагування рядків облікових записів Convex Auth. У запиті потрібно вказати обидва незмінні
ідентифікатори облікових записів провайдера GitHub; змінювані псевдоніми використовуються лише як перевірка для оператора.

Кінцева точка за замовчуванням працює в режимі пробного запуску. Для застосування відновлення потрібно вказати `dryRun: false` і
`confirmIdentityVerified: true` після того, як співробітники незалежно перевірять безперервність зв’язку між обома
обліковими записами GitHub. Відновлення завершується без внесення змін, якщо поточний особистий
видавець користувача призначення має навички, пакети або джерела навичок GitHub.
Відновлення також переносить застарілі поля `ownerUserId` для навичок відновленого видавця,
псевдонімів ідентифікаторів навичок, пакетів, попереджень інспектора пакетів і похідних рядків дайджесту пошуку, щоб
шляхи безпосереднього власника відповідали новим повноваженням видавця. Активне резервування
захищеного імені для відновленого імені також перепризначається користувачеві-заміні, щоб подальша
синхронізація профілю не могла відновити конкуруючі повноваження попереднього користувача. Для кожної основної таблиці діє обмеження
у 100 рядків на транзакцію застосування; для більших відновлень спочатку потрібно використати відновлюване перенесення власника.
Джерела навичок GitHub мають область дії видавця та позначаються як перевірені, а не перезаписані.

- Тіло: `{ "handle": "gingiris", "nextUserHandle": "gingiris-1031", "previousGitHubProviderAccountId": "123", "nextGitHubProviderAccountId": "456", "reason": "Verified account continuity for issue #2555", "confirmIdentityVerified": true, "dryRun": false }`
- Відповідь: `{ "ok": true, "dryRun": false, "recovered": true, "publisherId": "...", "handle": "gingiris", "previousUser": { "userId": "...", "handle": "gingiris", "nextHandle": "gingiris-recovered", "githubProviderAccountId": "123", "authAccountCount": 1 }, "nextUser": { "userId": "...", "handle": "gingiris-1031", "nextHandle": "gingiris", "githubProviderAccountId": "456", "authAccountCount": 1 }, "retiredPersonalPublisher": null, "resourceOwnerMigration": { "limitPerTable": 100, "skills": 1, "skillSlugAliases": 1, "packages": 0, "packageInspectorWarnings": 0, "githubSourcesChecked": 1, "handleReservations": 1 }, "identityVerified": true, "reason": "Verified account continuity for issue #2555" }`

### Кінцеві точки керування ідентифікаторами власника

- `POST /api/v1/skills/{slug}/rename`
  - Тіло: `{ "newSlug": "new-canonical-slug" }`
  - Відповідь: `{ "ok": true, "slug": "new-canonical-slug", "previousSlug": "old-slug" }`
- `POST /api/v1/skills/{slug}/merge`
  - Тіло: `{ "targetSlug": "canonical-target-slug" }`
  - Відповідь: `{ "ok": true, "sourceSlug": "old-slug", "targetSlug": "canonical-target-slug" }`

Примітки:

- Обидві кінцеві точки потребують автентифікації за токеном API та працюють лише для власника навички.
- `rename` зберігає попередній ідентифікатор як псевдонім переспрямування.
- `merge` приховує вихідний запис і переспрямовує вихідний ідентифікатор на цільовий запис.

## Виявлення реєстру (`/.well-known/clawhub.json`)

CLI може отримувати налаштування реєстру й автентифікації із сайту:

- `/.well-known/clawhub.json` (JSON, бажаний)
- `/.well-known/clawdhub.json` (застарілий)

Схема:

```json
{ "apiBase": "https://clawhub.ai", "authBase": "https://clawhub.ai", "minCliVersion": "0.0.5" }
```

Якщо ви розгортаєте систему самостійно, надайте цей файл (або явно встановіть `CLAWHUB_REGISTRY`; застарілий варіант — `CLAWDHUB_REGISTRY`).
