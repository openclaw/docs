# HTTP API

Базова URL-адреса: `https://clawhub.ai` (за замовчуванням).

Усі шляхи v1 розташовані під `/api/v1/...`.
Застарілі `/api/...` і `/api/cli/...` зберігаються для сумісності (див. `DEPRECATIONS.md`).
OpenAPI: `/api/v1/openapi.json`.

## Повторне використання публічного каталогу

Сторонні каталоги можуть використовувати загальнодоступні кінцеві точки читання для відображення списку або пошуку Skills ClawHub. Кешуйте результати, дотримуйтеся `429`/`Retry-After`, спрямовуйте користувачів на канонічну сторінку ClawHub (`https://clawhub.ai/<owner>/skills/<slug>`) і не створюйте враження, що ClawHub підтримує сторонній сайт. Не намагайтеся віддзеркалювати прихований, приватний або заблокований модерацією вміст поза межами загальнодоступної поверхні API.

Скорочені вебадреси з ідентифікаторами розв’язуються в усіх сімействах реєстру, але клієнти API мають використовувати
канонічні URL-адреси, повернуті кінцевими точками читання, замість самостійного відтворення
пріоритету маршрутів.

## Обмеження частоти запитів

Модель застосування:

- Анонімні запити: обмеження застосовується для кожної IP-адреси.
- Автентифіковані запити (дійсний токен Bearer): обмеження застосовується для кошика кожного користувача.
- Якщо токен відсутній або недійсний, застосовується обмеження за IP-адресою.
- Автентифіковані кінцеві точки запису не повинні повертати лише `Unauthorized`, якщо
  сервер знає причину. Для відсутніх, недійсних або відкликаних токенів, а також
  видалених, заблокованих або вимкнених облікових записів слід повертати дієвий текст, щоб клієнти
  CLI могли повідомити користувачам, що саме їх заблокувало.

- Читання: 3000/хв на IP-адресу, 12000/хв на ключ
- Запис: 300/хв на IP-адресу, 3000/хв на ключ
- Завантаження: 1200/хв на IP-адресу, 6000/хв на ключ (кінцеві точки завантаження)

Заголовки:

- Застаріла сумісність: `X-RateLimit-Limit`, `X-RateLimit-Reset`
- Стандартизовані: `RateLimit-Limit`, `RateLimit-Reset`
- Для `429`: `X-RateLimit-Remaining: 0` і `RateLimit-Remaining: 0`
- Для `429`: `Retry-After`

Семантика заголовків:

- `X-RateLimit-Reset`: абсолютний час у секундах епохи Unix
- `RateLimit-Reset`: кількість секунд до скидання (затримка)
- `X-RateLimit-Remaining` / `RateLimit-Remaining`: точний залишок бюджету, якщо вказано.
  Успішні сегментовані запити не містять цього заголовка замість повернення приблизного глобального значення.
- `Retry-After`: кількість секунд очікування перед повторною спробою (затримка) для `429`

Приклад відповіді `429`:

```http
HTTP/2 429
content-type: text/plain; charset=utf-8
x-ratelimit-limit: 20
x-ratelimit-remaining: 0
x-ratelimit-reset: 1771404540
ratelimit-limit: 20
ratelimit-remaining: 0
ratelimit-reset: 34
retry-after: 34

Перевищено обмеження частоти запитів
```

Рекомендації для клієнта:

- Якщо існує `Retry-After`, зачекайте вказану кількість секунд перед повторною спробою.
- Використовуйте експоненційне відкладення з випадковим відхиленням, щоб уникнути синхронізованих повторних спроб.
- Якщо `Retry-After` відсутній, використовуйте `RateLimit-Reset` як резервний варіант (або обчисліть значення з `X-RateLimit-Reset`).

Джерело IP-адреси:

- Довірені заголовки IP-адреси клієнта, зокрема `cf-connecting-ip`, використовуються лише тоді, коли
  в розгортанні явно ввімкнено довірені переспрямовані заголовки.
- ClawHub використовує довірені переспрямовані заголовки для визначення IP-адрес клієнтів на межі мережі.
- Якщо довірена IP-адреса клієнта недоступна, для анонімних запитів використовуються резервні кошики,
  область дії яких визначається лише типом обмеження частоти запитів. Ці резервні кошики не містять
  наданих викликачем шляхів, ідентифікаторів, назв пакетів, версій, рядків запиту чи інших
  параметрів артефактів.

## Відповіді про помилки

Загальнодоступні відповіді про помилки v1 є звичайним текстом із `content-type: text/plain; charset=utf-8`.
Це охоплює помилки перевірки (`400`), відсутні загальнодоступні ресурси (`404`), помилки автентифікації та
дозволів (`401`/`403`), обмеження частоти запитів (`429`) і заблоковані завантаження. Клієнти
мають читати тіло відповіді як зрозумілий людині рядок. Невідомі параметри запиту
ігноруються для сумісності, але розпізнані параметри запиту з недійсними значеннями повертають
`400`.

## Загальнодоступні кінцеві точки (без автентифікації)

### `GET /api/v1/search`

Параметри запиту:

- `q` (обов’язковий): рядок пошукового запиту
- `limit` (необов’язковий): ціле число
- `highlightedOnly` (необов’язковий): `true`, щоб відфільтрувати лише рекомендовані Skills
- `nonSuspiciousOnly` (необов’язковий): `true`, щоб приховати підозрілі (`flagged.suspicious`) Skills
- `nonSuspicious` (необов’язковий): застарілий псевдонім для `nonSuspiciousOnly`

Відповідь:

```json
{
  "results": [
    {
      "score": 0.123,
      "slug": "gifgrep",
      "displayName": "GifGrep",
      "summary": "…",
      "version": "1.2.3",
      "updatedAt": 1730000000000,
      "ownerHandle": "openclaw",
      "owner": {
        "handle": "openclaw",
        "displayName": "OpenClaw",
        "image": "https://example.com/avatar.png"
      }
    }
  ]
}
```

Примітки:

- Результати повертаються в порядку релевантності (подібність векторних представлень + підвищення для точних збігів токенів ідентифікатора/назви + невеликий апріорний вплив популярності).
- Релевантність має більшу вагу, ніж популярність. Точний збіг токена ідентифікатора або відображуваної назви може випередити менш точний збіг зі значно вищою залученістю.
- Текст ASCII токенізується за межами слів і розділових знаків. Наприклад, `personal-map` містить окремий токен `map`, тоді як `amap-jsapi-skill` містить `amap`, `jsapi` і `skill`; тому пошук за `map` забезпечує для `personal-map` сильніший лексичний збіг, ніж для `amap-jsapi-skill`.
- Популярність масштабується логарифмічно й має верхню межу. Skills із високою залученістю можуть мати нижчий рейтинг, якщо текст запиту збігається гірше.
- Підозрілий або прихований стан модерації може вилучити Skill із загальнодоступного пошуку залежно від фільтрів викликача та поточного стану модерації.

Рекомендації щодо доступності для пошуку для видавців:

- Додавайте терміни, які користувачі буквально шукатимуть, до відображуваної назви, опису й тегів. Використовуйте окремий токен ідентифікатора лише тоді, коли це також стабільний ідентифікатор, який потрібно зберегти.
- Не перейменовуйте ідентифікатор лише заради одного запиту, якщо новий ідентифікатор не є кращою довгостроковою канонічною назвою. Старі ідентифікатори стають псевдонімами переспрямування, але канонічна URL-адреса, відображуваний ідентифікатор і майбутні пошукові зведення використовують новий ідентифікатор.
- Псевдоніми перейменування зберігають розв’язання старих URL-адрес і встановлень, які розв’язуються через реєстр, але рейтинг пошуку ґрунтується на канонічних метаданих Skill після індексування перейменування. Наявна статистика залишається пов’язаною зі Skill.
- Якщо Skill несподівано не відображається, спершу перевірте стан модерації за допомогою `clawhub inspect @owner/slug`, увійшовши в систему, і лише потім змінюйте метадані, пов’язані з рейтингом.

### `GET /api/v1/skills`

Параметри запиту:

- `limit` (необов’язковий): ціле число (1–200)
- `cursor` (необов’язковий): курсор пагінації для будь-якого сортування, відмінного від `trending`
- `sort` (необов’язковий): `updated` (за замовчуванням), `recommended` (псевдонім: `default`), `createdAt` (псевдонім: `newest`), `downloads`, `stars` (псевдонім: `rating`), застарілі псевдоніми встановлення `installsCurrent`/`installs`/`installsAllTime` відповідають `downloads`, `trending`
- `nonSuspiciousOnly` (необов’язковий): `true`, щоб приховати підозрілі (`flagged.suspicious`) Skills
- `nonSuspicious` (необов’язковий): застарілий псевдонім для `nonSuspiciousOnly`

Недійсні значення `sort` повертають `400`.

Примітки:

- `recommended` використовує сигнали залученості та актуальності.
- `trending` упорядковує за кількістю встановлень за останні 7 днів (на основі телеметрії).
- `createdAt` є стабільним для сканування нових Skills; `updated` змінюється, коли наявні Skills публікуються повторно.
- Коли `nonSuspiciousOnly=true`, сортування на основі курсора може повертати на сторінці менше ніж `limit` елементів, оскільки підозрілі Skills фільтруються після отримання сторінки.
- Використовуйте `nextCursor` для продовження пагінації, якщо він наявний. Коротка сторінка сама по собі не означає кінець результатів.

Відповідь:

```json
{
  "items": [
    {
      "slug": "gifgrep",
      "displayName": "GifGrep",
      "summary": "…",
      "topics": ["Productivity"],
      "tags": { "latest": "1.2.3" },
      "stats": {},
      "createdAt": 0,
      "updatedAt": 0,
      "latestVersion": { "version": "1.2.3", "createdAt": 0, "changelog": "…" },
      "metadata": { "os": ["macos"], "systems": ["aarch64-darwin"] }
    }
  ],
  "nextCursor": null
}
```

### `GET /api/v1/skills/{slug}`

Відповідь:

```json
{
  "skill": {
    "slug": "gifgrep",
    "displayName": "GifGrep",
    "summary": "…",
    "topics": ["Productivity"],
    "tags": { "latest": "1.2.3" },
    "stats": {},
    "createdAt": 0,
    "updatedAt": 0
  },
  "latestVersion": { "version": "1.2.3", "createdAt": 0, "changelog": "…" },
  "metadata": { "os": ["macos"], "systems": ["aarch64-darwin"] },
  "owner": { "handle": "steipete", "displayName": "Peter", "image": null },
  "moderation": {
    "isSuspicious": false,
    "isMalwareBlocked": false,
    "verdict": "clean",
    "reasonCodes": [],
    "summary": null,
    "engineVersion": "v2.0.0",
    "updatedAt": 0
  }
}
```

Примітки:

- Старі ідентифікатори, створені під час перейменування або об’єднання власником, розв’язуються до канонічного Skill.
- `metadata.os`: обмеження ОС, оголошені у frontmatter Skill (наприклад, `["macos"]`, `["linux"]`). `null`, якщо їх не оголошено.
- `metadata.systems`: цільові системи Nix (наприклад, `["aarch64-darwin", "x86_64-linux"]`). `null`, якщо їх не оголошено.
- `metadata` має значення `null`, якщо Skill не має метаданих платформи.
- `moderation` додається лише тоді, коли Skill позначено або його переглядає власник.

### `POST /api/v1/skills/{slug}/report`

з версією та надходять до черги повідомлень про Skills.

Автентифікація:

- Потрібен токен API.

Запит:

```json
{ "reason": "Підозрілий крок установлення", "version": "1.2.3" }
```

Відповідь:

```json
{
  "ok": true,
  "reported": true,
  "alreadyReported": false,
  "reportId": "skillReports:...",
  "skillId": "skills:...",
  "reportCount": 1
}
```

- `limit` (необов’язковий): ціле число (1-200)

### `POST /api/v1/skills/-/reports/{reportId}/triage`

Кінцева точка модератора/адміністратора для вирішення або повторного відкриття повідомлень про Skills.

Запит:

```json
{ "status": "confirmed", "note": "Перевірено та приховано відповідну версію.", "finalAction": "hide" }
```

`note` є обов’язковим для `confirmed` і `dismissed`; його можна не вказувати під час
повернення `status` до `open`. Передайте `finalAction: "hide"` разом із розглянутим
повідомленням, щоб приховати Skills у межах того самого робочого процесу з можливістю аудиту.

### `GET /api/v1/skills/{slug}/versions`

Параметри запиту:

- `limit` (необов’язковий): ціле число
- `cursor` (необов’язковий): курсор пагінації

### `GET /api/v1/skills/{slug}/versions/{version}`

Повертає метадані версії та список файлів.

- `version.security` містить нормалізований стан перевірки сканування та відомості про сканери
  (VirusTotal + LLM), якщо вони доступні.

### `GET /api/v1/skills/{slug}/scan`

Повертає відомості про перевірку сканування безпеки для версії Skills.

Параметри запиту:

- `version` (необов’язковий): конкретний рядок версії.
- `tag` (необов’язковий): визначити версію за тегом (наприклад, `latest`).

Примітки:

- Якщо не вказано ані `version`, ані `tag`, використовується найновіша версія.
- Містить нормалізований стан перевірки та відомості для кожного сканера.
- `security.hasScanResult` має значення `true`, лише коли сканер надав однозначний вердикт (`clean`, `suspicious` або `malicious`).
- `moderation` — це поточний знімок модерації на рівні Skills, отриманий із найновішої версії.
- Під час запиту історичної версії перевірте `moderation.matchesRequestedVersion` і `moderation.sourceVersion`, перш ніж вважати, що `moderation` та `security` стосуються контексту тієї самої версії.

### `POST /api/v1/skills/-/scan`

Автентифікована кінцева точка надсилання нових завдань ClawScan.

Сканування локально завантажених файлів більше не підтримується. Запити з
`multipart/form-data` або `{ "source": { "kind": "upload" } }` повертають `410`.

Для опублікованих сканувань використовується JSON:

```json
{
  "source": { "kind": "published", "slug": "gifgrep", "version": "1.2.3" },
  "update": false
}
```

Примітки:

- Корисні навантаження запитів на сканування та звіти для завантаження видаляються зі сховища запитів на сканування після завершення періоду зберігання.
- Опубліковані сканування потребують управлінського доступу власника/видавця або повноважень модератора/адміністратора платформи.
- Результати опублікованих сканувань записуються назад, лише коли `update: true` і сканування успішно завершено.
- Відповідь — `202` з `{ "ok": true, "scanId": "...", "jobId": "...", "status": "queued", "sourceKind": "published", "update": false, "queue": { "queuedAhead": 0, "queuedAheadIsEstimate": false, "position": 1, "running": 0, "runningIsEstimate": false, "note": "Scans are asynchronous and may take time to complete." } }`.
- Завдання сканування виконуються асинхронно. Ручні запити на сканування мають вищий пріоритет за звичайні операції публікації/ретроспективного заповнення, але завершення все одно залежить від доступності виконавців.

### `GET /api/v1/skills/-/scan/{scanId}`

Автентифікована кінцева точка опитування для надісланого сканування.

- Повертає стан у черзі/виконується/успішно завершено/невдало.
- Поки запит перебуває в черзі, повертає `queue.queuedAhead` і `queue.position`, щоб клієнти могли показати кількість пріоритетних ручних сканувань попереду нього. Розмір дуже великих черг обмежується та повідомляється за допомогою `queuedAheadIsEstimate: true`.
- Якщо доступно, `report` містить розділи `clawscan`, `skillspector`, `staticAnalysis` і `virustotal`.
- Невдалі завдання сканування повертають `status: "failed"` з `lastError`.

### `GET /api/v1/skills/-/scan/download/{name}?version=<version>&kind=skill|plugin`

- Повертає збережені результати сканування для точної надісланої версії, зокрема заблокованих або прихованих версій.
- `kind` за замовчуванням має значення `skill`; використовуйте `kind=plugin` для сканувань Plugin/пакетів.

### `GET /api/v1/skills/{slug}/verify`

Повертає конверт перевірки картки Skills, який використовує `clawhub skill verify`.

Параметри запиту:

- `version` (необов’язковий): конкретний рядок версії.
- `tag` (необов’язковий): визначити версію за тегом (наприклад, `latest`).

Примітки:

- `ok` має значення `true`, лише коли для вибраної версії створено картку Skills, версію не заблоковано модерацією через шкідливе ПЗ, а перевірка ClawScan не виявила проблем.
- Ідентичність Skills, ідентичність видавця та метадані вибраної версії є полями верхнього рівня конверта (`slug`, `displayName`, `publisherHandle`, `version`, `resolvedFrom`, `tag`, `createdAt`), щоб автоматизація оболонки могла зчитувати їх без розпакування вкладених обгорток.
- `security` — це вердикт ClawScan/безпеки верхнього рівня. Автоматизація має орієнтуватися на `ok`, `decision`, `reasons` і `security.status`.
- `security.signals` містить допоміжні докази від сканерів, як-от `staticScan`, `virusTotal` і `skillSpector`.
- `security.signals.dependencyRegistry` збережено для сумісності відповіді v1, але сканер наявності в реєстрі залежностей виведено з експлуатації, і цей ключ завжди має значення `null`.
- `provenance` має значення `server-resolved-github-import`, лише коли ClawHub визначив і зберіг репозиторій/ref/коміт/шлях GitHub під час публікації або імпорту; інакше він має значення `unavailable`.

### `POST /api/v1/skills/-/security-verdicts`

Повертає поточні компактні вердикти безпеки для точних версій Skills. Ця
кінцева точка колекції призначена для клієнтів, які вже знають, які встановлені
версії Skills ClawHub їм потрібно показати, наприклад інтерфейс керування OpenClaw.

Запит:

```json
{
  "items": [{ "slug": "gifgrep", "version": "1.2.3" }]
}
```

Примітки:

- `items` має містити 1-100 унікальних пар `{ slug, version }`.
- Результати повертаються для кожного елемента окремо; відсутність одного Skills або версії не спричиняє помилку всієї відповіді.
- Відповідь містить лише дані безпеки. Вона не містить даних картки Skills, стану створеної картки, списків файлів артефакту чи докладних корисних навантажень сканерів.
- `security.signals` містить лише допоміжні докази на рівні стану; використовуйте `/scan` або сторінку аудиту безпеки ClawHub для отримання повних відомостей про сканери.
- `security.signals.dependencyRegistry` збережено для сумісності відповіді v1, але сканер наявності в реєстрі залежностей виведено з експлуатації, і цей ключ завжди має значення `null`.
- Відсутність картки Skills не впливає на `ok`, `decision` або `reasons` цієї кінцевої точки; коли клієнтам потрібен вміст картки, вони мають локально читати встановлений `skill-card.md`.
- Використовуйте `/verify`, коли потрібен конверт перевірки картки Skills для одного Skills, `/card`, коли потрібен Markdown створеної картки, і `/scan`, коли потрібні докладні дані сканерів.

Відповідь:

### `GET /api/v1/packages/{name}/versions/{version}`

Повертає одну версію пакета, зокрема метадані файлів, сумісність,
перевірку, метадані артефакту та дані сканування.

Примітки:

- `version.artifact.kind` має значення `legacy-zip` для архівів пакетів старого формату або
  `npm-pack` для випусків на основі ClawPack.
- Випуски ClawPack містять сумісні з npm поля `npmIntegrity`, `npmShasum` і
  `npmTarballName`.
- `version.sha256hash` — застарілі метадані сумісності для старих клієнтів. Вони
  хешують точні байти ZIP-архіву, повернуті `/api/v1/packages/{name}/download`.
  Сучасним клієнтам слід використовувати `version.artifact.sha256`, який ідентифікує
  канонічний артефакт випуску.
- `version.vtAnalysis`, `version.llmAnalysis` і `version.staticScan`
  додаються за наявності даних сканування.
- Приватні пакети повертають `404`, якщо клієнт не має доступу на читання видавця-власника.

### `GET /api/v1/packages/{name}/versions/{version}/security`

Повертає точне зведення про безпеку й довіру для випуску пакета, призначене для клієнтів
встановлення. Це загальнодоступна поверхня споживання OpenClaw для визначення того, чи можна
встановити визначений випуск.

Автентифікація:

- Загальнодоступна кінцева точка читання. Токен власника, видавця, модератора чи адміністратора
  не потрібен.

Відповідь:

```json
{
  "package": {
    "name": "@openclaw/example-plugin",
    "displayName": "Приклад плагіна",
    "family": "code-plugin"
  },
  "release": {
    "releaseId": "packageReleases:...",
    "version": "1.2.3",
    "artifactKind": "npm-pack",
    "artifactSha256": "0123456789abcdef...",
    "npmIntegrity": "sha512-...",
    "npmShasum": "0123456789abcdef0123456789abcdef01234567",
    "npmTarballName": "example-plugin-1.2.3.tgz",
    "createdAt": 1730000000000
  },
  "trust": {
    "scanStatus": "malicious",
    "moderationState": "quarantined",
    "blockedFromDownload": true,
    "reasons": ["manual:quarantined", "scan:malicious"],
    "pending": false,
    "stale": false
  }
}
```

Поля відповіді:

- `package.name`, `package.displayName` і `package.family` ідентифікують
  визначений пакет реєстру.
- `release.releaseId`, `release.version` і `release.createdAt` ідентифікують
  точний випуск, який було оцінено.
- `release.artifactKind`, `release.artifactSha256`, `release.npmIntegrity`,
  `release.npmShasum` і `release.npmTarballName` наводяться, коли вони відомі для
  артефакту випуску.
- `trust.scanStatus` — це фактичний стан довіри, отриманий із вхідних даних сканера
  та ручної модерації випуску.
- `trust.moderationState` допускає значення null. Він має значення `null`, коли ручна
  модерація випуску відсутня.
- `trust.blockedFromDownload` — це сигнал блокування встановлення. OpenClaw та інші
  клієнти встановлення мають блокувати встановлення, коли це значення дорівнює `true`, замість
  повторного виведення правил блокування з полів сканера чи модерації.
- `trust.reasons` — це список пояснень для користувача та аудиту. Коди причин —
  стабільні компактні рядки, як-от `manual:quarantined`, `scan:malicious`
  і `package:malicious`.
- `trust.pending` означає, що один або кілька вхідних сигналів довіри все ще очікують на завершення.
- `trust.stale` означає, що зведення про довіру обчислено на основі застарілих вхідних даних і
  його слід вважати таким, що потребує оновлення перед ухваленням рішення про дозвіл із високим рівнем упевненості.

Примітки:

- Ця кінцева точка прив’язана до точної версії. Клієнтам слід викликати її після визначення
  версії пакета, яку вони мають намір встановити, а не лише після читання метаданих
  найновішого пакета.
- Приватні пакети повертають `404`, якщо клієнт не має доступу на читання видавця-власника.
- Ця кінцева точка навмисно вужча за кінцеві точки модерації для власників і модераторів.
  Вона надає рішення щодо встановлення та загальнодоступне пояснення, але не розкриває
  особи заявників, тексти скарг, приватні докази або внутрішні часові шкали
  перевірки.

- `tag` (необов’язково)

- Перевірки модерації та доступу до приватних пакетів усе ще застосовуються.

### `GET /api/v1/resolve`

Використовується CLI для зіставлення локального відбитка з відомою версією.

Параметри запиту:

- `slug` (обов’язково)
- `hash` (обов’язково): 64-символьний шістнадцятковий sha256 відбитка пакета

Відповідь:

```json
{ "slug": "gifgrep", "match": { "version": "1.2.2" }, "latestVersion": { "version": "1.2.3" } }
```

### `GET /api/v1/download`

Завантажує ZIP розміщеної версії Skills або повертає передавання до джерельного коду GitHub для
поточної Skills на основі GitHub зі скануванням `clean` або `suspicious` і без розміщеної
версії.

Параметри запиту:

- `slug` (обов’язково)
- `version` (необов’язково): рядок semver
- `tag` (необов’язково): назва тегу (наприклад, `latest`)

Примітки:

- Якщо не вказано ані `version`, ані `tag`, використовується найновіша версія.
- Версії, видалені м’яко, повертають `410`.
- Передавання Skills на основі GitHub не проксіює й не дзеркалює байти. Відповідь JSON
  містить `sourceRef: "public-github"`, `repo`, `commit`, `path`, `contentHash`
  та `archiveUrl`; сканування/поточний стан є умовою допуску й не включається
  до метаданих успішної відповіді.
- Статистика завантажень підраховує унікальні ідентичності за кожну добу UTC (`userId`, якщо токен API дійсний, інакше — IP-адресу).

## Кінцеві точки автентифікації (токен Bearer)

Усі кінцеві точки потребують:

```
Authorization: Bearer clh_...
```

### `GET /api/v1/whoami`

Перевіряє токен і повертає ідентифікатор користувача.

### `POST /api/v1/skills`

Публікує нову версію.

- Рекомендовано: `multipart/form-data` з JSON `payload` та двійковими об’єктами `files[]`.
- Також приймається тіло JSON із `files` (на основі storageId).
- Необов’язкове поле корисного навантаження: `ownerHandle`. Якщо його вказано, API визначає цього
  видавця на сервері та вимагає, щоб суб’єкт мав доступ видавця.
- Необов’язкове поле корисного навантаження: `migrateOwner`. Коли `true` із `ownerHandle`, наявний
  Skill можна передати цьому власнику, якщо суб’єкт є адміністратором/власником як поточного,
  так і цільового видавця. Без цієї явної згоди зміни власника
  відхиляються.

### `POST /api/v1/packages`

Публікує випуск плагіна з кодом або пакетного плагіна.

- Потребує автентифікації за допомогою токена Bearer.
- Потребує `multipart/form-data`.
- Дозволені поля форми: `payload`, повторювані двійкові об’єкти `files` або одне посилання на tar-архів `clawpack`.
  `clawpack` може бути двійковим об’єктом `.tgz` або ідентифікатором сховища, повернутим
  процесом отримання URL завантаження. Поетапні публікації за ідентифікатором сховища також мають містити
  `clawpackUploadTicket`, повернений разом із цим URL завантаження.
- Використовуйте або `files`, або `clawpack`, але ніколи обидва в одному запиті.
- Тіла JSON і надані клієнтом метадані `payload.files` / `payload.artifact`
  відхиляються.
- Розмір прямих багатокомпонентних запитів публікації обмежено 18MB. Для tar-архівів ClawPack
  можна використовувати процес отримання URL завантаження з обмеженням розміру tar-архіву 120MB.
- Необов’язкове поле корисного навантаження: `ownerHandle`. Якщо його вказано, публікувати від імені цього власника можуть лише адміністратори.

Основні аспекти перевірки:

- `family` має бути `code-plugin` або `bundle-plugin`.
- Пакети плагінів потребують `openclaw.plugin.json`. Завантаження ClawPack `.tgz` мають
  містити його в `package/openclaw.plugin.json`.
- Плагіни з кодом потребують `package.json`, метаданих репозиторію вихідного коду, метаданих
  коміту вихідного коду, метаданих схеми конфігурації, `openclaw.compat.pluginApi` та
  `openclaw.build.openclawVersion`.
- `openclaw.hostTargets` та `openclaw.environment` є необов’язковими метаданими.
- Публікувати в канал `official` можуть лише видавець організації `openclaw` та особисті
  видавці поточних учасників організації `openclaw`.
- Під час публікації від імені іншої сторони право на офіційний канал усе одно перевіряється для облікового запису цільового власника.

### `DELETE /api/v1/skills/{slug}` / `POST /api/v1/skills/{slug}/undelete`

М’яко видаляє / відновлює Skill (власник, модератор або адміністратор).

Необов’язкове тіло JSON:

```json
{ "reason": "Утримується на модерації до завершення юридичної перевірки." }
```

Якщо вказано, `reason` зберігається як примітка модерації Skill і копіюється до журналу аудиту.
Після м’якого видалення, ініційованого власником, коротка назва резервується на 30 днів, після чого її може отримати
інший видавець. Відповідь на видалення містить `slugReservedUntil`, коли застосовується цей строк дії.
Приховування модератором/адміністратором і видалення з міркувань безпеки не мають такого строку дії.

Відповідь на видалення:

```json
{ "ok": true, "slugReservedUntil": 1730000000000 }
```

Коди стану:

- `200`: успішно
- `401`: не автентифіковано
- `403`: заборонено
- `404`: Skill/користувача не знайдено
- `500`: внутрішня помилка сервера

### `POST /api/v1/users/publisher`

Лише для адміністраторів. Забезпечує наявність видавця організації для ідентифікатора. Якщо ідентифікатор досі вказує на
застарілого спільного користувача/особистого видавця, кінцева точка спочатку переносить його до видавця організації.
Для новоствореної організації вкажіть `memberHandle`; адміністратор, який виконує дію, не додається як учасник.
Типове значення `memberRole` — `owner`.

- Тіло: `{ "handle": "openclaw", "displayName": "OpenClaw", "memberHandle": "alice", "memberRole": "owner", "trusted": true }`
- Відповідь: `{ "ok": true, "publisherId": "...", "handle": "openclaw", "created": true, "migrated": false, "trusted": true, "member": { "userId": "...", "handle": "alice", "role": "owner" } }`

### `POST /api/v1/publishers`

Автентифіковане самостійне створення видавця організації. Створює нового видавця організації та додає
ініціатора виклику як власника. Ця кінцева точка не переносить наявні користувацькі/особисті ідентифікатори
та не позначає видавця як довіреного/офіційного.

- Тіло: `{ "handle": "opik", "displayName": "Opik" }`
- Відповідь: `{ "ok": true, "publisherId": "...", "handle": "opik", "created": true, "trusted": false }`
- Повертає `409`, якщо ідентифікатор уже використовується видавцем, користувачем або особистим видавцем.

### `POST /api/v1/users/reserve`

Лише для адміністраторів. Резервує кореневі короткі назви та назви пакетів для законного власника без публікації
випуску. Назви пакетів стають приватними пакетами-заповнювачами без рядків випусків, тому той самий
власник згодом може опублікувати справжній випуск плагіна з кодом або пакетного плагіна під цією назвою.

- Тіло: `{ "handle": "openclaw", "slugs": ["diffs"], "packageNames": ["@openclaw/diffs"], "reason": "reserved for official OpenClaw plugin" }`
- Відповідь: `{ "ok": true, "succeeded": 2, "failed": 0, "results": [{ "kind": "slug", "name": "diffs", "ok": true, "action": "reserved" }] }`

### `POST /api/v1/users/publisher-recovery`

Лише для адміністраторів. Відновлює особистого видавця для перевіреного замінного суб’єкта GitHub OAuth
без редагування рядків облікових записів Convex Auth. У запиті необхідно вказати обидва незмінні ідентифікатори
облікових записів провайдера GitHub; змінювані ідентифікатори використовуються лише як запобіжна перевірка для оператора.

Типово кінцева точка виконує пробний запуск. Для застосування відновлення потрібні `dryRun: false` і
`confirmIdentityVerified: true` після того, як працівники незалежно перевірять безперервність між обома
суб’єктами GitHub. Відновлення безпечно завершується помилкою, якщо поточний особистий
видавець цільового користувача має Skills, пакети або джерела Skills GitHub.
Відновлення також переносить застарілі поля `ownerUserId` для Skills відновленого видавця,
псевдонімів коротких назв Skills, пакетів, попереджень інспектора пакетів і похідних рядків дайджесту пошуку, щоб
шляхи безпосереднього власника узгоджувалися з повноваженнями нового видавця. Активне резервування
захищеного ідентифікатора для відновленого ідентифікатора також перепризначається замінному користувачеві, щоб подальша
синхронізація профілю не могла відновити конкуруючі повноваження колишнього користувача. Кожна основна таблиця обмежена
100 рядками на транзакцію застосування; для більших відновлень спочатку потрібно використати відновлюване перенесення власника.
Джерела Skills GitHub мають область дії видавця й позначаються у звіті як перевірені, а не переписані.

- Тіло: `{ "handle": "gingiris", "nextUserHandle": "gingiris-1031", "previousGitHubProviderAccountId": "123", "nextGitHubProviderAccountId": "456", "reason": "Verified account continuity for issue #2555", "confirmIdentityVerified": true, "dryRun": false }`
- Відповідь: `{ "ok": true, "dryRun": false, "recovered": true, "publisherId": "...", "handle": "gingiris", "previousUser": { "userId": "...", "handle": "gingiris", "nextHandle": "gingiris-recovered", "githubProviderAccountId": "123", "authAccountCount": 1 }, "nextUser": { "userId": "...", "handle": "gingiris-1031", "nextHandle": "gingiris", "githubProviderAccountId": "456", "authAccountCount": 1 }, "retiredPersonalPublisher": null, "resourceOwnerMigration": { "limitPerTable": 100, "skills": 1, "skillSlugAliases": 1, "packages": 0, "packageInspectorWarnings": 0, "githubSourcesChecked": 1, "handleReservations": 1 }, "identityVerified": true, "reason": "Verified account continuity for issue #2555" }`

### Кінцеві точки керування короткими назвами власника

- `POST /api/v1/skills/{slug}/rename`
  - Тіло: `{ "newSlug": "new-canonical-slug" }`
  - Відповідь: `{ "ok": true, "slug": "new-canonical-slug", "previousSlug": "old-slug" }`
- `POST /api/v1/skills/{slug}/merge`
  - Тіло: `{ "targetSlug": "canonical-target-slug" }`
  - Відповідь: `{ "ok": true, "sourceSlug": "old-slug", "targetSlug": "canonical-target-slug" }`

Примітки:

- Обидві кінцеві точки потребують автентифікації за допомогою токена API та працюють лише для власника Skill.
- `rename` зберігає попередню коротку назву як псевдонім переспрямування.
- `merge` приховує початковий запис і переспрямовує початкову коротку назву на цільовий запис.

### Кінцеві точки передавання права власності

- `POST /api/v1/skills/{slug}/transfer`
  - Тіло: `{ "toUserHandle": "target_handle", "message": "optional" }`
  - Відповідь: `{ "ok": true, "transferId": "skillOwnershipTransfers:...", "toUserHandle": "target_handle", "expiresAt": 1730000000000 }`
- `POST /api/v1/skills/{slug}/transfer/accept`
- `POST /api/v1/skills/{slug}/transfer/reject`
- `POST /api/v1/skills/{slug}/transfer/cancel`
  - Відповідь (прийняття/відхилення/скасування): `{ "ok": true, "skillSlug": "demo-skill?" }`
- `GET /api/v1/transfers/incoming`
- `GET /api/v1/transfers/outgoing`
  - Структура відповіді: `{ "transfers": [{ "_id": "...", "skill": { "slug": "demo", "displayName": "Demo" }, "fromUser"|"toUser": { "handle": "..." }, "message": "...", "requestedAt": 0, "expiresAt": 0 }] }`

### `POST /api/v1/users/ban`

Блокує користувача та безповоротно видаляє належні йому Skills (лише для модераторів/адміністраторів).

Тіло:

```json
{ "handle": "user_handle", "reason": "необов’язкова причина блокування" }
```

або

```json
{ "userId": "users_...", "reason": "необов’язкова причина блокування" }
```

Відповідь:

```json
{ "ok": true, "alreadyBanned": false, "deletedSkills": 3 }
```

### `POST /api/v1/users/unban`

Розблоковує користувача та відновлює придатні Skills (лише для адміністраторів).

Тіло:

```json
{ "handle": "user_handle", "reason": "необов’язкова причина розблокування" }
```

або

```json
{ "userId": "users_...", "reason": "необов’язкова причина розблокування" }
```

Відповідь:

```json
{ "ok": true, "alreadyUnbanned": false, "restoredSkills": 3 }
```

### `POST /api/v1/users/reclassify-ban`

Змінює збережену причину наявного блокування, не розблоковуючи користувача й не відновлюючи
вміст (лише для адміністраторів). Типово виконується пробний запуск, якщо `dryRun` не має значення `false`.

Тіло:

```json
{ "handle": "user_handle", "reason": "масове публікування спаму", "dryRun": true }
```

або

```json
{ "userId": "users_...", "reason": "масове публікування спаму", "dryRun": false }
```

Відповідь:

```json
{
  "ok": true,
  "dryRun": false,
  "userId": "users_...",
  "handle": "user_handle",
  "previousReason": "автоматичне блокування через шкідливе ПЗ",
  "nextReason": "масове публікування спаму",
  "changed": true
}
```

### `POST /api/v1/users/role`

Змінює роль користувача (лише для адміністраторів).

Тіло:

```json
{ "handle": "user_handle", "role": "moderator" }
```

або

```json
{ "userId": "users_...", "role": "admin" }
```

Відповідь:

```json
{ "ok": true, "role": "moderator" }
```

### `GET /api/v1/users`

Виводить список користувачів або виконує їх пошук (лише для адміністраторів).

Параметри запиту:

- `q` (необов’язково): пошуковий запит
- `query` (необов’язково): псевдонім для `q`
- `limit` (необов’язково): максимальна кількість результатів (типово 20, максимум 200)

Відповідь:

```json
{
  "items": [
    {
      "userId": "users_...",
      "handle": "user_handle",
      "displayName": "Користувач",
      "name": "Користувач",
      "role": "moderator"
    }
  ],
  "total": 1
}
```

### `POST /api/v1/stars/{slug}` / `DELETE /api/v1/stars/{slug}`

Додає/видаляє зірочку (виділення). Обидві кінцеві точки ідемпотентні.

Відповіді:

```json
{ "ok": true, "starred": true, "alreadyStarred": false }
```

```json
{ "ok": true, "unstarred": true, "alreadyUnstarred": false }
```

## Застарілі кінцеві точки CLI (не рекомендовані)

Усе ще підтримуються для старіших версій CLI:

- `GET /api/cli/whoami`
- `POST /api/cli/upload-url`
- `POST /api/cli/publish`
- `POST /api/cli/telemetry/install`
- `POST /api/cli/skill/delete`
- `POST /api/cli/skill/undelete`

План видалення наведено в `DEPRECATIONS.md`.

`POST /api/cli/upload-url` повертає `uploadUrl` та `uploadTicket`. Публікації
пакетів, що поетапно завантажують tar-архів ClawPack, мають передавати отриманий ідентифікатор сховища як
`clawpack`, а повернений квиток — як `clawpackUploadTicket`.

## Виявлення реєстру (`/.well-known/clawhub.json`)

CLI може визначати параметри реєстру/автентифікації із сайту:

- `/.well-known/clawhub.json` (JSON, рекомендовано)
- `/.well-known/clawdhub.json` (застарілий формат)

Схема:

```json
{ "apiBase": "https://clawhub.ai", "authBase": "https://clawhub.ai", "minCliVersion": "0.0.5" }
```

У разі самостійного розміщення надавайте цей файл (або явно задайте `CLAWHUB_REGISTRY`; застарілий варіант — `CLAWDHUB_REGISTRY`).
