# Усунення несправностей

## `clawhub login` відкриває браузер, але не завершується

CLI запускає короткочасний локальний сервер зворотного виклику під час входу через браузер.

- Переконайтеся, що ваш браузер може відкрити `http://127.0.0.1:<port>/callback`.
- Перевірте правила локального брандмауера, VPN і проксі, якщо зворотний виклик не надходить.
- У середовищах без графічного інтерфейсу створіть API-токен у вебінтерфейсі ClawHub і виконайте:

```bash
clawhub login --token clh_...
```

## `whoami` або `publish` повертає `Unauthorized` (401)

- Увійдіть знову за допомогою `clawhub login`.
- Якщо ви використовуєте власний шлях до конфігурації, переконайтеся, що `CLAWHUB_CONFIG_PATH` вказує на
  файл, який містить ваш поточний токен.
- Якщо ви використовуєте API-токен, переконайтеся, що його не відкликано у вебінтерфейсі.

## Пошук або встановлення повертає `Rate limit exceeded` (429)

Прочитайте інформацію про повторну спробу у відповіді:

- `Retry-After`: кількість секунд очікування перед повторною спробою.
- `RateLimit-Limit`: ліміт, застосований до цього запиту.
- `RateLimit-Remaining`: ваш точний залишковий бюджет, коли цей заголовок присутній. Для `429` він дорівнює `0`.
- `RateLimit-Reset` або `X-RateLimit-Reset`: час скидання.

Якщо багато користувачів мають одну вихідну IP-адресу, анонімні IP-ліміти можуть спрацювати навіть тоді, коли кожна
особа надсилає лише кілька запитів. Увійдіть, якщо можливо, і повторіть спробу після
повідомленої затримки.

## Пошук або встановлення не працює за проксі

CLI враховує стандартні змінні проксі:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

Підтримувані назви включають `HTTPS_PROXY`, `HTTP_PROXY`, `https_proxy` і
`http_proxy`.

## Навичка не з’являється в пошуку

- Перевірте точний слаг або сторінку власника, якщо ви її знаєте.
- Переконайтеся, що випуск публічний і не затриманий скануванням або модерацією.
- Якщо ви є власником навички, увійдіть і перевірте її:

```bash
clawhub inspect @openclaw/demo
```

Діагностика, видима власнику, може пояснити стан сканування, шлюзу завантаження або модерації.

## Публікація не вдається через відсутність обов’язкових метаданих

Для навичок перевірте frontmatter у `SKILL.md`. Обов’язкові змінні середовища та
інструменти слід оголосити, щоб користувачі й сканери могли зрозуміти пакет.

Для плагінів перевірте метадані сумісності в `package.json`. Публікації
code-plugin потребують полів сумісності OpenClaw, таких як `openclaw.compat.pluginApi` і
`openclaw.build.openclawVersion`.

Спершу перегляньте payload публікації:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## Публікація не вдається через помилку власника GitHub або джерела

ClawHub використовує ідентичність GitHub та атрибуцію джерела, щоб пов’язувати пакети з їхніми
видавцями.

- Переконайтеся, що ви ввійшли з обліковим записом GitHub, який володіє пакетом або може його публікувати.
- Перевірте, що URL джерела є публічним або доступним для ClawHub.
- Для джерел GitHub використовуйте `owner/repo`, `owner/repo@ref` або повний URL GitHub.

## Публікація не вдається, бо простір назв заявлений або зарезервований

Якщо публікація не вдається, бо handle власника, простір назв організації, scope пакета, слаг навички
або назва пакета вже заявлені чи зарезервовані, спершу переконайтеся, що ви
публікуєте від імені власника, який відповідає простору назв. Для пакетів плагінів
scoped-назви, такі як `@example-org/example-plugin`, мають публікуватися від імені
відповідного власника `example-org`.

Якщо ви вважаєте, що ваша організація, проєкт або бренд є правомірним власником простору назв, але
ви не можете керувати поточним власником ClawHub, відкрийте
[issue для заявки на організацію / простір назв](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
з публічними, нечутливими доказами. Див.
[Заявки на організації та простори назв](/clawhub/namespace-claims), щоб дізнатися, які докази надати та що
не слід включати в публічні issue.

## `sync` повідомляє, що навичок не знайдено

`sync` шукає папки, що містять `SKILL.md` або `skill.md`.

Вкажіть корені, які потрібно просканувати:

```bash
clawhub sync --root /path/to/skills
```

Спершу перегляньте результат, якщо не впевнені, що буде опубліковано:

```bash
clawhub sync --all --dry-run --no-input
```

## `update` відмовляється виконуватися через локальні зміни

Локальні файли не відповідають жодній версії, про яку знає ClawHub. Виберіть один варіант:

- Зберегти локальні зміни й пропустити оновлення.
- Перезаписати опублікованою версією:

```bash
clawhub update @openclaw/demo --force
```

- Опублікувати відредаговану копію як новий слаг або форк.

## Встановлення плагіна в OpenClaw не вдається

- Використайте явне джерело ClawHub:

```bash
openclaw plugins install clawhub:<package>
```

- Перевірте сторінку деталей пакета щодо стану сканування та метаданих сумісності.
- Переконайтеся, що ваша версія OpenClaw відповідає заявленому пакетом
  діапазону сумісності.
- Якщо пакет прихований, затриманий або заблокований, його може бути неможливо встановити, доки
  власник не вирішить проблему.

## Публічні API-запити не вдаються

- Дотримуйтеся заголовків повторної спроби `429` і кешуйте публічні відповіді списків/пошуку.
- Спрямовуйте користувачів назад до канонічного списингу ClawHub.
- Не дублюйте прихований, приватний, затриманий або заблокований модерацією вміст поза межами
  публічної API-поверхні.

Див. [HTTP API](/clawhub/http-api) для деталей endpoint.
