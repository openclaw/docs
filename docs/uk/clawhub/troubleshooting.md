# Усунення несправностей

## `clawhub login` відкриває браузер, але ніколи не завершується

CLI запускає короткочасний локальний сервер зворотного виклику під час входу через браузер.

- Переконайтеся, що ваш браузер може відкрити `http://127.0.0.1:<port>/callback`.
- Перевірте правила локального файрвола, VPN і проксі, якщо зворотний виклик не надходить.
- У середовищах без графічного інтерфейсу створіть API-токен у вебінтерфейсі ClawHub і виконайте:

```bash
clawhub login --token clh_...
```

## `whoami` або `publish` повертає `Unauthorized` (401)

- Увійдіть знову за допомогою `clawhub login`.
- Якщо ви використовуєте власний шлях до конфігурації, переконайтеся, що `CLAWHUB_CONFIG_PATH` вказує на
  файл, який містить ваш поточний токен.
- Якщо ви використовуєте API-токен, переконайтеся, що його не відкликано у вебінтерфейсі.

## Пошук або встановлення повертає `Rate limit exceeded` (429)

Прочитайте інформацію про повторну спробу у відповіді:

- `Retry-After`: секунди очікування перед повторною спробою.
- `RateLimit-Limit`: ліміт, застосований до цього запиту.
- `RateLimit-Remaining`: ваш точний залишковий ліміт, коли заголовок присутній. Для `429` це `0`.
- `RateLimit-Reset` або `X-RateLimit-Reset`: час скидання.

Якщо багато користувачів спільно використовують одну вихідну IP-адресу, анонімні IP-ліміти можуть спрацювати навіть тоді, коли кожна
людина надсилає лише кілька запитів. Увійдіть, де це можливо, і повторіть спробу після
повідомленої затримки.

## Пошук або встановлення не працює за проксі

CLI враховує стандартні змінні проксі:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

Підтримувані назви включають `HTTPS_PROXY`, `HTTP_PROXY`, `https_proxy` і
`http_proxy`.

## Skill не з’являється в пошуку

- Перевірте точний slug або сторінку власника, якщо ви їх знаєте.
- Переконайтеся, що реліз є публічним і не утримується через сканування або модерацію.
- Якщо Skill належить вам, увійдіть і перевірте його:

```bash
clawhub inspect @openclaw/demo
```

Діагностика, видима власнику, може пояснити стан сканування, блокування завантаження або модерації.

## Публікація не вдається, бо відсутні обов’язкові метадані

Для Skills перевірте frontmatter у `SKILL.md`. Обов’язкові змінні середовища й
інструменти мають бути оголошені, щоб користувачі та сканери могли зрозуміти пакет.

Для plugins перевірте метадані сумісності в `package.json`. Публікації code-plugin
потребують полів сумісності OpenClaw, як-от `openclaw.compat.pluginApi` і
`openclaw.build.openclawVersion`.

Спершу перегляньте payload публікації:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## Публікація не вдається через помилку власника GitHub або джерела

ClawHub використовує ідентичність GitHub та атрибуцію джерела, щоб пов’язувати пакети з їхніми
публікаторами.

- Переконайтеся, що ви ввійшли з обліковим записом GitHub, який володіє пакетом або може його публікувати.
- Перевірте, що URL джерела є публічним або доступним для ClawHub.
- Для джерел GitHub використовуйте `owner/repo`, `owner/repo@ref` або повний URL GitHub.

## Публікація не вдається, бо простір імен зайнятий або зарезервований

Якщо публікація не вдається, бо handle власника, простір імен організації, scope пакета, slug Skill
або назва пакета вже зайняті чи зарезервовані, спершу переконайтеся, що ви
публікуєте від власника, який відповідає простору імен. Для пакетів plugin
scoped-імена, як-от `@example-org/example-plugin`, мають публікуватися від
відповідного власника `example-org`.

Якщо ви вважаєте, що ваша організація, проєкт або бренд є законним власником простору імен, але
ви не можете керувати поточним власником ClawHub, відкрийте
[звернення щодо організації / простору імен](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
з публічними, нечутливими доказами. Див.
[Претензії щодо організацій і просторів імен](/clawhub/namespace-claims), щоб дізнатися, які докази надати та що
не варто включати в публічні issues.

## `sync` повідомляє, що Skills не знайдено

`sync` шукає папки, які містять `SKILL.md` або `skill.md`.

Вкажіть корені, які потрібно просканувати:

```bash
clawhub sync --root /path/to/skills
```

Спершу перегляньте попередній результат, якщо не впевнені, що буде опубліковано:

```bash
clawhub sync --all --dry-run --no-input
```

## `update` відмовляється через локальні зміни

Локальні файли не відповідають жодній версії, відомій ClawHub. Виберіть один варіант:

- Залишити локальні правки й пропустити оновлення.
- Перезаписати опублікованою версією:

```bash
clawhub update @openclaw/demo --force
```

- Опублікувати відредаговану копію як новий slug або fork.

## Встановлення plugin в OpenClaw не вдається

- Використайте явне джерело ClawHub:

```bash
openclaw plugins install clawhub:<package>
```

- Перевірте сторінку деталей пакета щодо статусу сканування та метаданих сумісності.
- Переконайтеся, що ваша версія OpenClaw відповідає заявленому пакетом
  діапазону сумісності.
- Якщо пакет прихований, утримується або заблокований, його може бути неможливо встановити, доки
  власник не вирішить проблему.

## Запити до публічного API не вдаються

- Дотримуйтеся заголовків повторної спроби `429` і кешуйте публічні відповіді списку/пошуку.
- Спрямовуйте користувачів назад до канонічного запису ClawHub.
- Не дзеркальте прихований, приватний, утримуваний або заблокований модерацією вміст поза
  поверхнею публічного API.

Див. [HTTP API](/clawhub/http-api) для деталей endpoint.
