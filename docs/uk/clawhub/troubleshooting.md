# Усунення неполадок

## `clawhub login` відкриває браузер, але не завершується

CLI запускає короткочасний локальний сервер зворотного виклику під час входу через браузер.

- Переконайтеся, що ваш браузер може відкрити `http://127.0.0.1:<port>/callback`.
- Перевірте правила локального брандмауера, VPN і проксі, якщо зворотний виклик не надходить.
- У безголових середовищах створіть API-токен у вебінтерфейсі ClawHub і виконайте:

```bash
clawhub login --token clh_...
```

## `whoami` або `publish` повертає `Unauthorized` (401)

- Увійдіть знову за допомогою `clawhub login`.
- Якщо ви використовуєте власний шлях до конфігурації, переконайтеся, що `CLAWHUB_CONFIG_PATH` вказує на
  файл, який містить ваш поточний токен.
- Якщо ви використовуєте API-токен, переконайтеся, що його не відкликано у вебінтерфейсі.

## Пошук або встановлення повертає `Rate limit exceeded` (429)

Прочитайте інформацію про повторну спробу у відповіді:

- `Retry-After`: скільки секунд чекати перед повторною спробою.
- `RateLimit-Limit`: ліміт, застосований до цього запиту.
- `RateLimit-Remaining`: ваш точний залишковий бюджет, коли цей заголовок присутній. Для `429` він дорівнює `0`.
- `RateLimit-Reset` або `X-RateLimit-Reset`: час скидання.

Якщо багато користувачів спільно використовують одну вихідну IP-адресу, анонімні IP-ліміти можуть спрацьовувати навіть тоді, коли кожна
людина надсилає лише кілька запитів. Увійдіть, якщо це можливо, і повторіть спробу після
повідомленої затримки.

## Пошук або встановлення не працює за проксі

CLI враховує стандартні змінні проксі:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

Підтримувані назви включають `HTTPS_PROXY`, `HTTP_PROXY`, `https_proxy` і
`http_proxy`.

## Skills не з'являється в пошуку

- Перевірте точний слаг або сторінку власника, якщо ви її знаєте.
- Переконайтеся, що реліз публічний і не затриманий скануванням або модерацією.
- Якщо ви володієте цим Skills, увійдіть і перевірте його:

```bash
clawhub inspect @openclaw/demo
```

Діагностика, видима власнику, може пояснити стан сканування, блокування завантаження або модерації.

## Публікація не вдається, бо відсутні обов'язкові метадані

Для Skills перевірте frontmatter у `SKILL.md`. Обов'язкові змінні середовища та
інструменти слід оголошувати, щоб користувачі й сканери могли зрозуміти пакет.

Для Plugin перевірте метадані сумісності в `package.json`. Публікації кодових Plugin
потребують полів сумісності OpenClaw, таких як `openclaw.compat.pluginApi` і
`openclaw.build.openclawVersion`.

Спершу перегляньте payload публікації:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## Публікація не вдається через помилку власника GitHub або джерела

ClawHub використовує ідентичність GitHub та атрибуцію джерела, щоб пов'язувати пакети з їхніми
видавцями.

- Переконайтеся, що ви ввійшли з обліковим записом GitHub, який володіє пакетом або може його публікувати.
- Перевірте, що URL джерела є публічним або доступним для ClawHub.
- Для джерел GitHub використовуйте `owner/repo`, `owner/repo@ref` або повний URL GitHub.

## Публікація не вдається, бо простір імен зайнятий або зарезервований

Якщо публікація не вдається, бо хендл власника, простір імен організації, область пакета, слаг Skills
або назва пакета вже зайняті чи зарезервовані, спершу переконайтеся, що ви
публікуєте від імені власника, який відповідає простору імен. Для пакетів Plugin
назви з областю, такі як `@example-org/example-plugin`, мають публікуватися від імені
відповідного власника `example-org`.

Якщо ви вважаєте, що ваша організація, проєкт або бренд є законним власником простору імен, але
не можете керувати поточним власником ClawHub, відкрийте
[запит на організацію / простір імен](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
з публічним, неконфіденційним доказом. Див.
[Заявки на організації та простори імен](/clawhub/namespace-claims), щоб дізнатися, які докази надати і що
не слід додавати до публічних issue.

## `sync` повідомляє, що Skills не знайдено

`sync` шукає папки, що містять `SKILL.md` або `skill.md`.

Укажіть корені, які потрібно просканувати:

```bash
clawhub sync --root /path/to/skills
```

Спершу перегляньте результат, якщо не впевнені, що буде опубліковано:

```bash
clawhub sync --all --dry-run --no-input
```

## `update` відмовляється виконуватися через локальні зміни

Локальні файли не збігаються з жодною версією, відомою ClawHub. Виберіть один варіант:

- Зберегти локальні зміни й пропустити оновлення.
- Перезаписати опублікованою версією:

```bash
clawhub update @openclaw/demo --force
```

- Опублікувати свою відредаговану копію як новий слаг або форк.

## Установлення Plugin в OpenClaw не вдається

- Використовуйте явне джерело ClawHub:

```bash
openclaw plugins install clawhub:<package>
```

- Перевірте сторінку деталей пакета щодо стану сканування та метаданих сумісності.
- Переконайтеся, що ваша версія OpenClaw відповідає заявленому пакетом
  діапазону сумісності.
- Якщо пакет прихований, затриманий або заблокований, його може бути неможливо встановити, доки
  власник не розв'яже проблему.

## Публічні API-запити не вдаються

- Дотримуйтеся заголовків повтору для `429` і кешуйте публічні відповіді списків/пошуку.
- Посилайте користувачів назад на канонічний список ClawHub.
- Не дзеркальте прихований, приватний, затриманий або заблокований модерацією вміст поза
  поверхнею публічного API.

Див. [HTTP API](/clawhub/http-api) для деталей кінцевих точок.
