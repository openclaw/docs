# Усунення несправностей

## `clawhub login` відкриває браузер, але не завершується

Під час входу через браузер CLI запускає короткочасний локальний сервер зворотного виклику.

- Переконайтеся, що браузер має доступ до `http://127.0.0.1:<port>/callback`.
- Якщо зворотний виклик не надходить, перевірте правила локального брандмауера, VPN і проксі.
- У середовищах без графічного інтерфейсу створіть токен API у вебінтерфейсі ClawHub і виконайте:

```bash
clawhub login --token clh_...
```

## `whoami` або `publish` повертає `Unauthorized` (401)

- Увійдіть знову за допомогою `clawhub login`.
- Якщо ви використовуєте власний шлях до конфігурації, переконайтеся, що `CLAWHUB_CONFIG_PATH` указує на
  файл із вашим поточним токеном.
- Якщо ви використовуєте токен API, переконайтеся, що його не було відкликано у вебінтерфейсі.

## Пошук або встановлення повертає `Rate limit exceeded` (429)

Ознайомтеся з інформацією про повторну спробу у відповіді:

- `Retry-After`: кількість секунд очікування перед повторною спробою.
- `RateLimit-Limit`: ліміт, застосований до цього запиту.
- `RateLimit-Remaining`: точний залишок вашої квоти, якщо цей заголовок наявний. Для `429` він дорівнює `0`.
- `RateLimit-Reset` або `X-RateLimit-Reset`: час скидання ліміту.

Якщо багато користувачів спільно використовують одну вихідну IP-адресу, анонімні обмеження за IP можуть спрацювати, навіть коли кожна
особа надсилає лише кілька запитів. За можливості увійдіть у систему та повторіть спробу після
зазначеної затримки.

## Пошук або встановлення не працює за проксі

CLI враховує стандартні змінні проксі:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

Підтримувані назви включають `HTTPS_PROXY`, `HTTP_PROXY`, `https_proxy` і
`http_proxy`.

## Skill не відображається в результатах пошуку

- Перевірте точний ідентифікатор або сторінку власника, якщо вони вам відомі.
- Переконайтеся, що випуск загальнодоступний і не затриманий перевіркою чи модерацією.
- Якщо ви володієте цим Skill, увійдіть у систему та перевірте його:

```bash
clawhub inspect @openclaw/demo
```

Діагностичні відомості, доступні власнику, можуть пояснити стан перевірки, обмеження завантаження або модерації.

## Публікація не вдається через відсутність обов’язкових метаданих

Для Skills перевірте frontmatter у `SKILL.md`. Обов’язкові змінні середовища та
інструменти мають бути оголошені, щоб користувачі й сканери могли зрозуміти пакет.

Для plugins перевірте метадані сумісності в `package.json`. Для публікації plugins із кодом
потрібні поля сумісності з OpenClaw, як-от `openclaw.compat.pluginApi` і
`openclaw.build.openclawVersion`.

Спочатку перегляньте дані публікації:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## Публікація не вдається через помилку власника або джерела GitHub

ClawHub використовує ідентичність GitHub і атрибуцію джерела, щоб пов’язувати пакети з їхніми
видавцями.

- Переконайтеся, що ви ввійшли за допомогою облікового запису GitHub, який володіє пакетом або може його публікувати.
- Перевірте, чи URL-адреса джерела загальнодоступна або доступна для ClawHub.
- Для джерел GitHub використовуйте `owner/repo`, `owner/repo@ref` або повну URL-адресу GitHub.

## Публікація не вдається, оскільки простір імен зайнятий або зарезервований

Якщо публікація не вдається через те, що ідентифікатор власника, простір імен організації, область пакета, ідентифікатор Skill
або назва пакета вже зайняті чи зарезервовані, спочатку переконайтеся, що ви
публікуєте від імені власника, який відповідає простору імен. Для пакетів plugins
назви з областю, як-от `@example-org/example-plugin`, потрібно публікувати від імені
відповідного власника `example-org`.

Якщо ви вважаєте, що ваша організація, проєкт або бренд є законним власником простору імен, але
не можете керувати поточним власником у ClawHub, створіть
[звернення щодо права на простір імен організації](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
із загальнодоступними доказами, що не містять конфіденційних даних. Рекомендації щодо доказів і відомостей,
які не слід додавати до загальнодоступних звернень, див. у розділі
[Права на простори імен організацій](/clawhub/namespace-claims).

## `sync` повідомляє, що Skills не знайдено

`sync` шукає папки, що містять `SKILL.md` або `skill.md`.

Укажіть кореневі каталоги, які потрібно просканувати:

```bash
clawhub sync --root /path/to/skills
```

Якщо ви не впевнені, що буде опубліковано, спочатку перегляньте результат:

```bash
clawhub sync --all --dry-run --no-input
```

## `update` відмовляється працювати через локальні зміни

Локальні файли не відповідають жодній версії, відомій ClawHub. Виберіть один із варіантів:

- Зберегти локальні зміни та пропустити оновлення.
- Перезаписати файли опублікованою версією:

```bash
clawhub update @openclaw/demo --force
```

- Опублікувати відредаговану копію з новим ідентифікатором або як форк.

## Не вдається встановити Plugin в OpenClaw

- Використайте явне джерело ClawHub:

```bash
openclaw plugins install clawhub:<package>
```

- Перевірте на сторінці відомостей про пакет стан перевірки та метадані сумісності.
- Переконайтеся, що ваша версія OpenClaw відповідає заявленому пакетом
  діапазону сумісності.
- Якщо пакет прихований, затриманий або заблокований, його встановлення може бути неможливим, доки
  власник не розв’яже проблему.

## Запити до загальнодоступного API не виконуються

- Дотримуйтеся заголовків повторної спроби для `429` і кешуйте загальнодоступні відповіді зі списками та результатами пошуку.
- Спрямовуйте користувачів до канонічної сторінки ClawHub.
- Не дублюйте прихований, приватний, затриманий або заблокований модерацією вміст поза межами
  загальнодоступного API.

Докладні відомості про кінцеві точки див. у розділі [HTTP API](/clawhub/http-api).
