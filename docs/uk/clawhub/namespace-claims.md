# Претензії щодо організацій і просторів імен

ClawHub використовує ідентифікатори власників, ідентифікатори організацій, slug-и Skills, назви пакетів Plugin і
області пакетів як публічні простори імен. Якщо простір імен виглядає таким, що належить
реальному проєкту, бренду, екосистемі пакетів або організації, але вже
заявлений, зарезервований, вводить в оману або оскаржується в ClawHub, попросіть персонал переглянути його
за допомогою
[форми issue для претензії щодо організації / простору імен](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml).

Використовуйте цей шлях для публічного, неконфіденційного перегляду права власності. Не використовуйте вбудовані в продукт
звіти або форму апеляції щодо облікового запису для претензій щодо просторів імен.

## Коли відкривати претензію

Відкривайте претензію щодо простору імен, коли вважаєте, що персонал ClawHub має перевірити, чи
простір імен потрібно зарезервувати, передати, перейменувати, приховати, помістити в карантин, зробити псевдонімом
або інакше змінити через право власності в реальному світі.

Приклади:

- ідентифікатор організації, який збігається з вашою організацією GitHub, проєктом, компанією або спільнотою
- область пакета, як-от `@example-org/*`, яка має публікуватися лише під
  відповідним власником ClawHub
- slug Skills або назва пакета Plugin, що виглядає як імітація проєкту
- спір щодо бренду, торговельної марки, перейменування проєкту або історії пакета
- видалений, неактивний або недоступний власник, який блокує законного власника
  простору імен

Якщо список небезпечний, зловмисний або вводить в оману поза межами спору про власність,
також дотримуйтеся відповідних настанов щодо модерації або безпеки. Форма претензії щодо простору імен
призначена для перегляду права власності, а не для екстреного розкриття вразливостей.

## Перед поданням

Спершу переконайтеся, що ви публікуєте від імені власника, який відповідає простору імен.
Для пакетів Plugin області назв, як-от `@example-org/example-plugin`, мають
публікуватися від імені відповідного власника `example-org`.

Якщо ви можете керувати поточним власником, виправте простір імен напряму шляхом публікації,
перейменування, передавання, приховування або видалення відповідного ресурсу. Використовуйте претензію,
коли не можете керувати поточним власником або коли персонал має розв’язати
спір.

## Докази, які слід додати

Використовуйте публічні, неконфіденційні докази. Корисні докази включають:

- історію організації GitHub, репозиторію, релізів або супровідників
- офіційну документацію проєкту, де названо простір імен
- доказ домену або офіційного домену електронної пошти
- контроль області в npm, PyPI, crates.io або іншому реєстрі пакетів
- докази права власності на торговельну марку, бренд або проєкт, які безпечно обговорювати
  публічно
- історію вихідного репозиторію, історію пакета або публічні повідомлення про перейменування
- посилання на спірного власника ClawHub, Skills, Plugin, пакет або issue

Поясніть, що доводить кожне посилання. Персонал має мати змогу зрозуміти
зв’язок без потреби в приватних облікових даних або секретах.

## Що не слід додавати

Не додавайте секрети або приватні докази в публічний issue GitHub. Не включайте:

- API-токени, ключі підпису або облікові дані
- DNS challenge tokens
- приватні юридичні файли або договори
- особисті документи, що посвідчують особу
- приватні електронні листи, приватні звіти з безпеки або конфіденційні дані клієнтів

Форма претензії запитує, чи потрібен приватний канал персоналу для конфіденційних доказів.
Використовуйте цей варіант замість публікації конфіденційних матеріалів у відкритому доступі.

## Можливі результати

Залежно від доказів і ризику, персонал ClawHub може зарезервувати простір імен,
передати право власності, перейменувати ресурс, приховати або помістити наявний список у карантин,
додати псевдонім або перенаправлення, попросити більше доказів або відхилити запит.

Перегляд простору імен не гарантує, що кожну відповідну назву буде передано.
Персонал зважує публічні докази, наявне використання, ризик безпеки та вплив на користувачів.
