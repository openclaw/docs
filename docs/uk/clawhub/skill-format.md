# Формат Skill

## На диску

Skill — це папка.

Обов’язково:

- `SKILL.md` (або `skill.md`; застарілий `skills.md` також підтримується)

Необов’язково:

- будь-які допоміжні _текстові_ файли (див. «Дозволені файли»)
- `.clawhubignore` (шаблони ігнорування для публікації, застарілий `.clawdhubignore`)
- `.gitignore` (також ураховується)

## Імпорт із GitHub

Вебімпортер GitHub суворіший за локальну публікацію/синхронізацію. Він виявляє лише
файли `SKILL.md` або застарілі `skills.md` у загальнодоступних репозиторіях, що не є форками та належать
авторизованому обліковому запису GitHub. Він не імпортує приватні репозиторії, форки,
архівовані/вимкнені репозиторії або сторонні загальнодоступні репозиторії.

Метадані локального встановлення (записуються CLI):

- `<skill>/.clawhub/origin.json` (застарілий `.clawdhub`)

Стан встановлення в робочому каталозі (записується CLI):

- `<workdir>/.clawhub/lock.json` (застарілий `.clawdhub`)

## `SKILL.md`

- Markdown із необов’язковим YAML frontmatter.
- Під час публікації сервер видобуває метадані з frontmatter.
- `description` використовується як опис Skill в інтерфейсі/пошуку.

Для переносних Agent Skills значення `name` має відповідати назві батьківського каталогу й містити
1–64 малі літери, цифри або дефіси. ClawHub зберігає маршрутизований слаг і
відображувану назву каталогу окремо, тому наявні назви з інших клієнтів залишаються
придатними для публікації й не переписуються без попередження. У списках каталогу довгі назви можуть
візуально скорочуватися без зміни збереженої назви.

## Метадані frontmatter

Метадані Skill оголошуються в YAML frontmatter на початку вашого `SKILL.md`. Вони повідомляють реєстру (і засобам аналізу безпеки), що потрібно вашому Skill для роботи.

### Базовий frontmatter

```yaml
---
name: my-skill
description: Короткий опис призначення цього Skill.
version: 1.0.0
---
```

### Метадані середовища виконання (`metadata.openclaw`)

Оголосіть вимоги Skill до середовища виконання в `metadata.openclaw` (псевдоніми: `metadata.clawdbot`, `metadata.clawdis`).

```yaml
---
name: my-skill
description: Керування завданнями через API Todoist.
metadata:
  openclaw:
    requires:
      env:
        - TODOIST_API_KEY
      bins:
        - curl
    primaryEnv: TODOIST_API_KEY
---
```

Використовуйте `requires.env` для змінних середовища, які мають бути наявні до запуску Skill. Використовуйте `envVars`, коли потрібні метадані для кожної змінної, зокрема для необов’язкових змінних із `required: false`.

### Повний довідник полів

| Поле               | Тип        | Опис                                                                                                                                         |
| ------------------ | ---------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| `requires.env`     | `string[]` | Обов’язкові змінні середовища, на які розраховує ваш Skill.                                                                                   |
| `requires.bins`    | `string[]` | Виконувані файли CLI, які мають бути встановлені всі.                                                                                         |
| `requires.anyBins` | `string[]` | Виконувані файли CLI, з яких має бути наявний принаймні один.                                                                                 |
| `requires.config`  | `string[]` | Шляхи до файлів конфігурації, які читає ваш Skill.                                                                                            |
| `primaryEnv`       | `string`   | Основна змінна середовища з обліковими даними для вашого Skill.                                                                               |
| `envVars`          | `array`    | Оголошення змінних середовища з `name`, необов’язковим `required` і необов’язковим `description`. Установіть `required: false` для необов’язкових змінних середовища. |
| `always`           | `boolean`  | Якщо `true`, Skill завжди активний (явне встановлення не потрібне).                                                               |
| `skillKey`         | `string`   | Перевизначає ключ виклику Skill.                                                                                                              |
| `emoji`            | `string`   | Емодзі, що відображається для Skill.                                                                                                          |
| `homepage`         | `string`   | URL домашньої сторінки або документації Skill.                                                                                               |
| `os`               | `string[]` | Обмеження щодо ОС (наприклад, `["macos"]`, `["linux"]`).                                                                        |
| `install`          | `array`    | Специфікації встановлення залежностей (див. нижче).                                                                                           |
| `nix`              | `object`   | Специфікація плагіна Nix (див. README).                                                                                                       |
| `config`           | `object`   | Специфікація конфігурації Clawdbot (див. README).                                                                                             |

### Специфікації встановлення

Якщо для вашого Skill потрібно встановити залежності, оголосіть їх у масиві `install`:

```yaml
metadata:
  openclaw:
    install:
      - kind: brew
        formula: jq
        bins: [jq]
      - kind: node
        package: typescript
        bins: [tsc]
```

Підтримувані типи встановлення: `brew`, `node`, `go`, `uv`.

### Необов’язкові змінні середовища

Оголошуйте необов’язкові змінні середовища в `metadata.openclaw.envVars` та встановлюйте `required: false`. Не додавайте необов’язкові записи до `requires.env`, оскільки `requires.env` означає, що без них Skill не може працювати.

```yaml
metadata:
  openclaw:
    primaryEnv: TODOIST_API_KEY
    envVars:
      - name: TODOIST_API_KEY
        required: true
        description: Токен API Todoist для автентифікованих запитів.
      - name: TODOIST_PROJECT_ID
        required: false
        description: Необов’язковий ідентифікатор проєкту за замовчуванням, якщо користувач не вказав його.
```

### Чому це важливо

Аналіз безпеки ClawHub перевіряє, чи відповідають оголошення вашого Skill його фактичній поведінці. Якщо ваш код посилається на `TODOIST_API_KEY`, але frontmatter не оголошує його в `requires.env`, `primaryEnv` або `envVars`, аналіз позначить невідповідність метаданих. Точні оголошення допомагають вашому Skill пройти перевірку, а користувачам — зрозуміти, що вони встановлюють.

### Приклад: повний frontmatter

```yaml
---
name: todoist-cli
description: Керування завданнями, проєктами та мітками Todoist із командного рядка.
version: 1.2.0
metadata:
  openclaw:
    requires:
      env:
        - TODOIST_API_KEY
      bins:
        - curl
    primaryEnv: TODOIST_API_KEY
    envVars:
      - name: TODOIST_API_KEY
        required: true
        description: Токен API Todoist.
      - name: TODOIST_PROJECT_ID
        required: false
        description: Необов’язковий ідентифікатор проєкту за замовчуванням.
    emoji: "\u2705"
    homepage: https://github.com/example/todoist-cli
---
```

## Дозволені файли

Під час публікації приймаються лише «текстові» файли.

- Список дозволених розширень міститься в `packages/schema/src/textFiles.ts` (`TEXT_FILE_EXTENSIONS`).
- Файли сценаріїв усе одно скануються після завантаження; файли PowerShell `.ps1`, `.psm1` і `.psd1` приймаються як текстові.
- Типи вмісту, що починаються з `text/`, вважаються текстовими; також діє невеликий список дозволених типів (JSON/YAML/TOML/JS/TS/Markdown/SVG).

Обмеження (на боці сервера):

- Загальний розмір пакета: 50MB.
- Текст для вбудовування охоплює `SKILL.md` + до ~40 файлів, відмінних від `.md` (орієнтовне обмеження).

## Слаги

- За замовчуванням утворюються з назви папки.
- Області пакетів мають точно відповідати ідентифікатору видавця ClawHub. Ідентифікатори видавців можуть містити малі літери, цифри, дефіси, крапки та символи підкреслення; вони мають починатися й закінчуватися малою літерою або цифрою.
- Слаги пакетів мають бути в нижньому регістрі та сумісними з npm, наприклад `@example.tools/demo-plugin` або `demo-plugin`.

## Версії та теги

- Кожна публікація створює нову версію (semver).
- Теги — це рядкові вказівники на версію; зазвичай використовується `latest`.

## Ліцензія

- Усі Skills, опубліковані в ClawHub, ліцензуються за умовами `MIT-0`.
- Будь-хто може використовувати, змінювати й повторно розповсюджувати опубліковані Skills, зокрема в комерційних цілях.
- Зазначення авторства не вимагається.
- Не додавайте суперечливих умов ліцензування до `SKILL.md`; ClawHub не підтримує перевизначення ліцензії для окремих Skills.

## Платні Skills

- ClawHub не підтримує платні Skills, індивідуальне ціноутворення для Skills, платний доступ або розподіл доходів.
- Не додавайте метадані ціноутворення до `SKILL.md`; вони не є частиною формату Skill і не зроблять опублікований Skill платним.
- Якщо ваш Skill інтегрується з платним стороннім сервісом, чітко задокументуйте зовнішню вартість і потрібний обліковий запис в інструкціях Skill та оголошеннях змінних середовища (`requires.env` для обов’язкових змінних або `envVars` з `required: false` для необов’язкових змінних).
