# Формат Skills

## На диску

Skill — це папка.

Обов’язково:

- `SKILL.md` (або `skill.md`; застарілий файл `skills.md` також підтримується)

Необов’язково:

- будь-які допоміжні _текстові_ файли (див. «Дозволені файли»)
- `.clawhubignore` (шаблони ігнорування для публікації; застарілий варіант — `.clawdhubignore`)
- `.gitignore` (також враховується)

## Імпорт із GitHub

Вебімпортер GitHub має суворіші вимоги, ніж локальна публікація або синхронізація. Він знаходить лише файли `SKILL.md` або застарілі файли `skills.md` у загальнодоступних репозиторіях, які не є форками та належать поточному обліковому запису GitHub. Він не імпортує приватні репозиторії, форки, архівовані або вимкнені репозиторії чи загальнодоступні репозиторії третіх сторін.

Метадані локального встановлення (записуються CLI):

- `<skill>/.clawhub/origin.json` (застарілий варіант — `.clawdhub`)

Стан встановлення в робочому каталозі (записується CLI):

- `<workdir>/.clawhub/lock.json` (застарілий варіант — `.clawdhub`)

## `SKILL.md`

- Markdown із необов’язковим YAML-блоком метаданих.
- Під час публікації сервер видобуває метадані з блоку метаданих.
- `description` використовується як короткий опис Skill в інтерфейсі та пошуку.

Для переносних Agent Skills значення `name` має збігатися з назвою батьківського каталогу та містити від 1 до 64 малих літер, цифр або дефісів. ClawHub зберігає маршрутизований ідентифікатор і відображувану назву в каталозі окремо, тому наявні назви з інших клієнтів можна публікувати без прихованого переписування. У списках каталогу довгі назви можуть візуально скорочуватися без зміни збереженої назви.

## Метадані блоку метаданих

Метадані Skill оголошуються в YAML-блоці метаданих на початку файлу `SKILL.md`. Вони повідомляють реєстру та системі аналізу безпеки, що потрібно вашому Skill для роботи.

### Базовий блок метаданих

```yaml
---
name: my-skill
description: Короткий опис того, що робить цей Skill.
version: 1.0.0
---
```

### Метадані середовища виконання (`metadata.openclaw`)

Оголошуйте вимоги Skill до середовища виконання в `metadata.openclaw` (псевдоніми: `metadata.clawdbot`, `metadata.clawdis`).

```yaml
---
name: my-skill
description: Керування завданнями через API Todoist.
metadata:
  openclaw:
    requires:
      env:
        - TODOIST_API_KEY
      bins:
        - curl
    primaryEnv: TODOIST_API_KEY
---
```

Використовуйте `requires.env` для змінних середовища, які мають бути доступні до запуску Skill. Використовуйте `envVars`, коли потрібні метадані для кожної змінної, зокрема необов’язкові змінні з `required: false`.

### Повний довідник полів

| Поле               | Тип        | Опис                                                                                                                                                                  |
| ------------------ | ---------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `requires.env`     | `string[]` | Обов’язкові змінні середовища, які очікує ваш Skill.                                                                                                                   |
| `requires.bins`    | `string[]` | Виконувані файли CLI, які всі мають бути встановлені.                                                                                                                  |
| `requires.anyBins` | `string[]` | Виконувані файли CLI, з яких має бути доступний принаймні один.                                                                                                        |
| `requires.config`  | `string[]` | Шляхи до конфігураційних файлів, які читає ваш Skill.                                                                                                                  |
| `primaryEnv`       | `string`   | Основна змінна середовища з обліковими даними для вашого Skill.                                                                                                        |
| `envVars`          | `array`    | Оголошення змінних середовища з `name`, необов’язковим `required` і необов’язковим `description`. Для необов’язкових змінних середовища встановіть `required: false`. |
| `always`           | `boolean`  | Якщо `true`, Skill завжди активний і не потребує явного встановлення.                                                                                                  |
| `skillKey`         | `string`   | Перевизначає ключ виклику Skill.                                                                                                                                       |
| `emoji`            | `string`   | Емодзі, що відображається для Skill.                                                                                                                                   |
| `homepage`         | `string`   | URL домашньої сторінки або документації Skill.                                                                                                                         |
| `os`               | `string[]` | Обмеження за ОС (наприклад, `["macos"]`, `["linux"]`).                                                                                                                 |
| `install`          | `array`    | Специфікації встановлення залежностей (див. нижче).                                                                                                                    |
| `nix`              | `object`   | Специфікація Plugin Nix (див. README).                                                                                                                                 |
| `config`           | `object`   | Специфікація конфігурації Clawdbot (див. README).                                                                                                                      |

### Специфікації встановлення

Якщо Skill потребує встановлення залежностей, оголосіть їх у масиві `install`:

```yaml
metadata:
  openclaw:
    install:
      - kind: brew
        formula: jq
        bins: [jq]
      - kind: node
        package: typescript
        bins: [tsc]
```

Підтримувані типи встановлення: `brew`, `node`, `go`, `uv`.

### Необов’язкові змінні середовища

Оголошуйте необов’язкові змінні середовища в `metadata.openclaw.envVars` і встановлюйте `required: false`. Не додавайте необов’язкові записи до `requires.env`, оскільки `requires.env` означає, що Skill не може працювати без них.

```yaml
metadata:
  openclaw:
    primaryEnv: TODOIST_API_KEY
    envVars:
      - name: TODOIST_API_KEY
        required: true
        description: Токен API Todoist, що використовується для автентифікованих запитів.
      - name: TODOIST_PROJECT_ID
        required: false
        description: Необов’язковий ідентифікатор проєкту за замовчуванням, якщо користувач не вказав інший.
```

### Чому це важливо

Аналіз безпеки ClawHub перевіряє, чи відповідають оголошення Skill його фактичній поведінці. Якщо ваш код посилається на `TODOIST_API_KEY`, але блок метаданих не оголошує її в `requires.env`, `primaryEnv` або `envVars`, аналіз позначить невідповідність метаданих. Точні оголошення допомагають Skill пройти перевірку, а користувачам — зрозуміти, що саме вони встановлюють.

### Приклад: повний блок метаданих

```yaml
---
name: todoist-cli
description: Керування завданнями, проєктами та мітками Todoist із командного рядка.
version: 1.2.0
metadata:
  openclaw:
    requires:
      env:
        - TODOIST_API_KEY
      bins:
        - curl
    primaryEnv: TODOIST_API_KEY
    envVars:
      - name: TODOIST_API_KEY
        required: true
        description: Токен API Todoist.
      - name: TODOIST_PROJECT_ID
        required: false
        description: Необов’язковий ідентифікатор проєкту за замовчуванням.
    emoji: "\u2705"
    homepage: https://github.com/example/todoist-cli
---
```

## Дозволені файли

Під час публікації приймаються лише «текстові» файли.

- Список дозволених розширень міститься в `packages/schema/src/textFiles.ts` (`TEXT_FILE_EXTENSIONS`).
- Після завантаження файли сценаріїв усе одно скануються; файли PowerShell `.ps1`, `.psm1` і `.psd1` приймаються як текстові.
- Типи вмісту, що починаються з `text/`, вважаються текстовими; також діє невеликий список дозволених типів (JSON/YAML/TOML/JS/TS/Markdown/SVG).

Обмеження на сервері:

- Загальний розмір пакета: 50 МБ.
- Текст для векторного представлення включає `SKILL.md` і до приблизно 40 файлів, що не мають розширення `.md` (обмеження застосовується за можливості).

## Ідентифікатори

- За замовчуванням утворюються з назви папки.
- Області пакетів мають точно збігатися з ідентифікатором видавця ClawHub. Ідентифікатори видавців можуть містити малі літери, цифри, дефіси, крапки та символи підкреслення; вони мають починатися й закінчуватися малою літерою або цифрою.
- Ідентифікатори пакетів мають бути в нижньому регістрі та сумісними з npm, наприклад `@example.tools/demo-plugin` або `demo-plugin`.

## Версії та теги

- Кожна публікація створює нову версію (semver).
- Теги — це текстові вказівники на версію; зазвичай використовується `latest`.

## Ліцензія

- Усі Skills, опубліковані в ClawHub, ліцензуються за умовами `MIT-0`.
- Будь-хто може використовувати, змінювати й поширювати опубліковані Skills, зокрема з комерційною метою.
- Зазначення авторства не вимагається.
- Не додавайте суперечливі умови ліцензування до `SKILL.md`; ClawHub не підтримує перевизначення ліцензії для окремих Skills.

## Платні Skills

- ClawHub не підтримує платні Skills, окреме ціноутворення для Skills, платний доступ або розподіл доходу.
- Не додавайте метадані ціни до `SKILL.md`; вони не є частиною формату Skill і не зроблять опублікований Skill платним.
- Якщо Skill інтегрується з платною службою третьої сторони, чітко задокументуйте зовнішню вартість і потрібний обліковий запис в інструкціях Skill та оголошеннях змінних середовища (`requires.env` для обов’язкових змінних або `envVars` із `required: false` для необов’язкових змінних).
