# Як працює ClawHub

ClawHub — це шар реєстру для OpenClaw Skills і плагінів. Він дає користувачам
місце для пошуку пакетів, видавцям — місце для випуску версій, а
OpenClaw — достатньо метаданих, щоб безпечно встановлювати й оновлювати ці пакети.

## Записи реєстру

Кожен публічний запис є записом реєстру з:

- власником і слагом або назвою пакета
- однією або кількома опублікованими версіями
- метаданими, коротким описом, файлами та атрибуцією джерела
- журналом змін та інформацією про теги, як-от `latest`
- сигналами завантажень, установлень і зірок
- статусом сканування безпеки та модерації

Сторінка запису є канонічним місцем, де користувачі можуть перевірити, що саме
Skill або плагін заявляє про себе перед установленням.

## Skills

Skill — це версійований текстовий пакет, зосереджений навколо `SKILL.md`. Він може містити
допоміжні файли, приклади, шаблони та скрипти.

ClawHub читає frontmatter у `SKILL.md`, щоб зрозуміти назву Skill,
опис, вимоги, змінні середовища та метадані. Точні
метадані важливі, бо вони допомагають користувачам вирішити, чи встановлювати Skill, і
допомагають автоматизованим перевіркам виявляти невідповідності між заявленою та спостережуваною поведінкою.

Див. [Формат Skill](/uk/clawhub/skill-format).

## Плагіни

Плагіни — це упаковані розширення OpenClaw. ClawHub зберігає метадані пакетів,
інформацію про сумісність, посилання на джерела, артефакти та записи версій.

Коли OpenClaw встановлює плагін із ClawHub, він перевіряє заявлені метадані
сумісності перед установленням. Записи пакетів можуть містити сумісність API,
мінімальну версію Gateway, цільові хости, вимоги до середовища та дайджести
артефактів.

Використовуйте явне джерело встановлення ClawHub, коли хочете, щоб реєстр був
джерелом істини:

```bash
openclaw plugins install clawhub:<package>
```

## Публікація

Публікація створює новий незмінний запис версії. Видавці використовують CLI `clawhub`
для автентифікованих робочих процесів реєстру:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Використовуйте пробні запуски, щоб переглянути розв’язане навантаження перед завантаженням. Публічні сторінки потім
показують опубліковані метадані, файли, атрибуцію джерела та статус сканування.

## Установлення й оновлення

Команди встановлення OpenClaw використовують ClawHub як джерело пакетів:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw записує метадані джерела встановлення, щоб оновлення могли пізніше розв’язувати той самий
пакет реєстру. CLI ClawHub також підтримує прямі робочі процеси встановлення й
оновлення Skill для користувачів, які хочуть керовані реєстром теки Skill поза
повним робочим простором OpenClaw.

## Стан безпеки

ClawHub відкритий для публікації, але випуски все одно підлягають шлюзам завантаження,
автоматизованим перевіркам, звітам користувачів і діям модераторів.

Публічні сторінки показують підсумки сканування, коли вони доступні. Вміст, який утримано, приховано
або заблоковано, може зникнути з публічного пошуку та потоків установлення, залишаючись
видимим для власника для діагностики.

Див. [Безпека](/clawhub/security), [Аудити безпеки](/clawhub/security-audits),
[Модерація та безпека облікового запису](/uk/clawhub/moderation) і
[Прийнятне використання](/clawhub/acceptable-usage).

## Доступ до API

ClawHub надає публічні API читання для виявлення, пошуку, відомостей про пакети та
завантажень. Сторонні каталоги можуть використовувати ці API, коли вони посилаються на
канонічний запис ClawHub, дотримуються обмежень частоти запитів і не створюють враження схвалення.

Див. [Публічний API](/uk/clawhub/api) і [HTTP API](/clawhub/http-api).
