# Як працює ClawHub

ClawHub — це рівень реєстру для Skills і плагінів OpenClaw. Він надає користувачам
місце для пошуку пакетів, видавцям — місце для випуску версій, а OpenClaw —
достатньо метаданих для безпечного встановлення й оновлення цих пакетів.

## Записи реєстру

Кожен публічний запис є записом реєстру з:

- власником і слагом або назвою пакета
- однією або кількома опублікованими версіями
- метаданими, коротким описом, файлами та атрибуцією джерела
- журналом змін та інформацією про теги, як-от `latest`
- сигналами завантажень, встановлень і зірок
- статусом безпекового сканування та модерації

Сторінка запису є канонічним місцем, де користувачі можуть перевірити, що вміння або
plugin заявляє про свої можливості перед встановленням.

## Skills

Навичка — це версійований текстовий пакет, зосереджений на `SKILL.md`. Він може містити
допоміжні файли, приклади, шаблони та скрипти.

ClawHub читає frontmatter `SKILL.md`, щоб визначити назву навички,
опис, вимоги, змінні середовища та метадані. Точні
метадані важливі, бо вони допомагають користувачам вирішити, чи встановлювати навичку, і
допомагають автоматизованим скануванням виявляти невідповідності між заявленою та спостережуваною поведінкою.

Див. [Формат навички](/clawhub/skill-format).

## Плагіни

Плагіни — це упаковані розширення OpenClaw. ClawHub зберігає метадані пакетів,
інформацію про сумісність, посилання на джерела, артефакти та записи версій.

Коли OpenClaw встановлює plugin з ClawHub, він перевіряє заявлені метадані сумісності
перед встановленням. Записи пакетів можуть містити сумісність API,
мінімальну версію Gateway, цільові хости, вимоги до середовища та
дайджести артефактів.

Використовуйте явне джерело встановлення ClawHub, коли хочете, щоб реєстр був
джерелом істини:

```bash
openclaw plugins install clawhub:<package>
```

## Публікація

Публікація створює новий незмінний запис версії. Видавці використовують CLI `clawhub`
для автентифікованих робочих процесів реєстру:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Використовуйте пробні запуски, щоб попередньо переглянути розв’язане корисне навантаження перед завантаженням. Потім публічні сторінки
показують опубліковані метадані, файли, атрибуцію джерела та статус сканування.

## Встановлення та оновлення

Команди встановлення OpenClaw використовують ClawHub як джерело пакетів:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw записує метадані джерела встановлення, щоб оновлення могли пізніше знаходити той самий
пакет реєстру. CLI ClawHub також підтримує прямі робочі процеси встановлення й
оновлення навичок для користувачів, які хочуть керовані реєстром папки навичок поза
повним робочим простором OpenClaw.

## Стан безпеки

ClawHub відкритий для публікації, але випуски все одно підлягають шлюзам завантаження,
автоматизованим перевіркам, звітам користувачів і діям модераторів.

Публічні сторінки показують зведення сканування, коли вони доступні. Вміст, який утримується, приховано
або заблоковано, може зникати з публічного пошуку й потоків встановлення, але залишатися
видимим власнику для діагностики.

Див. [Безпека](/uk/clawhub/security), [Аудити безпеки](/clawhub/security-audits),
[Модерація та безпека облікового запису](/uk/clawhub/moderation) і
[Прийнятне використання](/clawhub/acceptable-usage).

## Доступ до API

ClawHub надає публічні API читання для виявлення, пошуку, деталей пакетів і
завантажень. Сторонні каталоги можуть використовувати ці API, коли вони посилаються на
канонічний запис ClawHub, дотримуються обмежень швидкості та не створюють враження схвалення.

Див. [Публічний API](/clawhub/api) і [HTTP API](/clawhub/http-api).
