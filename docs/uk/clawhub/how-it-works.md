# Як працює ClawHub

ClawHub — це рівень реєстру для Skills і плагінів OpenClaw. Він дає користувачам
місце для пошуку пакетів, видавцям — місце для випуску версій, а
OpenClaw — достатньо метаданих, щоб безпечно встановлювати й оновлювати ці пакети.

## Записи реєстру

Кожна публічна сторінка є записом реєстру із:

- власником і slug або назвою пакета
- однією або кількома опублікованими версіями
- метаданими, описом, файлами та атрибуцією джерела
- журналом змін та інформацією про теги, як-от `latest`
- сигналами завантажень, встановлень, зірок і коментарів
- статусом сканування безпеки та модерації

Сторінка публікації є канонічним місцем, де користувачі можуть переглянути,
що, за твердженням навички або плагіна, вона чи він робить, перш ніж встановлювати.

## Skills

Навичка — це версіонований текстовий пакет, зосереджений навколо `SKILL.md`. Він може містити
допоміжні файли, приклади, шаблони та скрипти.

ClawHub читає frontmatter у `SKILL.md`, щоб зрозуміти назву навички,
опис, вимоги, змінні середовища та метадані. Точні
метадані важливі, бо вони допомагають користувачам вирішити, чи встановлювати навичку, і
допомагають автоматизованим скануванням виявляти невідповідності між заявленою та спостережуваною поведінкою.

Див. [Формат навички](/uk/clawhub/skill-format).

## Плагіни

Плагіни — це упаковані розширення OpenClaw. ClawHub зберігає метадані пакета,
інформацію про сумісність, посилання на джерело, артефакти та записи версій.

Коли OpenClaw встановлює плагін із ClawHub, він перевіряє заявлені метадані
сумісності перед встановленням. Записи пакетів можуть містити сумісність API,
мінімальну версію Gateway, цільові хости, вимоги до середовища та дайджести
артефактів.

Використовуйте явне джерело встановлення ClawHub, коли потрібно, щоб реєстр був
джерелом істини:

```bash
openclaw plugins install clawhub:<package>
```

## Публікація

Публікація створює новий незмінний запис версії. Видавці використовують CLI `clawhub`
для автентифікованих робочих процесів реєстру:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Використовуйте пробні запуски, щоб попередньо переглянути сформоване корисне навантаження перед завантаженням. Потім публічні сторінки
показують опубліковані метадані, файли, атрибуцію джерела та статус сканування.

## Встановлення та оновлення

Команди встановлення OpenClaw використовують ClawHub як джерело пакетів:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

OpenClaw записує метадані джерела встановлення, щоб оновлення могли згодом знайти той самий
пакет реєстру. CLI ClawHub також підтримує прямі робочі процеси встановлення й
оновлення навичок для користувачів, які хочуть мати керовані реєстром папки навичок поза
повним робочим простором OpenClaw.

## Стан безпеки

ClawHub відкритий для публікації, але випуски все одно підлягають шлюзам завантаження,
автоматизованим перевіркам, звітам користувачів і діям модераторів.

Публічні сторінки показують зведення сканування, коли вони доступні. Вміст, який утримано, приховано
або заблоковано, може зникнути з публічного пошуку та потоків встановлення, залишаючись
видимим власнику для діагностики.

Див. [Безпека + модерація](/uk/clawhub/security) і
[Прийнятне використання](/uk/clawhub/acceptable-usage).

## Доступ до API

ClawHub надає публічні API для читання, призначені для виявлення, пошуку, деталей пакетів і
завантажень. Сторонні каталоги можуть використовувати ці API, коли вони посилаються на
канонічну сторінку ClawHub, дотримуються обмежень частоти запитів і не створюють враження схвалення.

Див. [Публічний API](/uk/clawhub/api) і [HTTP API](/uk/clawhub/http-api).
