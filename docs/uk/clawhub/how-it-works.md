# Як працює ClawHub

ClawHub — це шар реєстру для Skills і plugins OpenClaw. Він надає користувачам
місце для пошуку пакетів, видавцям — місце для випуску версій, а
OpenClaw — достатньо метаданих для безпечного встановлення й оновлення цих пакетів.

## Записи реєстру

Кожна публічна публікація є записом реєстру з:

- власником і slug або назвою пакета
- однією або кількома опублікованими версіями
- метаданими, коротким описом, файлами та атрибуцією джерела
- журналом змін та інформацією про теги, як-от `latest`
- сигналами завантаження, встановлення та зірок
- статусом сканування безпеки та модерації

Сторінка публікації є канонічним місцем, де користувачі можуть перевірити, що skill або
plugin заявляє про себе перед установленням.

## Skills

Skill — це версійований текстовий набір із центром у `SKILL.md`. Він може містити
допоміжні файли, приклади, шаблони та скрипти.

ClawHub читає frontmatter `SKILL.md`, щоб зрозуміти назву skill,
опис, вимоги, змінні середовища та метадані. Точні
метадані важливі, бо вони допомагають користувачам вирішити, чи встановлювати skill, і
допомагають автоматизованим скануванням виявляти невідповідності між заявленою та спостережуваною поведінкою.

Див. [Формат skill](/uk/clawhub/skill-format).

## Plugins

Plugins — це упаковані розширення OpenClaw. ClawHub зберігає метадані пакетів,
інформацію про сумісність, посилання на джерела, артефакти та записи версій.

Коли OpenClaw встановлює plugin із ClawHub, він перевіряє заявлені метадані сумісності
перед установленням. Записи пакетів можуть містити сумісність API,
мінімальну версію gateway, цільові хости, вимоги до середовища та дайджести артефактів.

Використовуйте явне джерело встановлення ClawHub, коли хочете, щоб реєстр був
джерелом істини:

```bash
openclaw plugins install clawhub:<package>
```

## Публікація

Публікація створює новий незмінний запис версії. Видавці використовують CLI `clawhub`
для автентифікованих робочих процесів реєстру:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Використовуйте пробні запуски, щоб попередньо переглянути вирішене корисне навантаження перед завантаженням. Публічні сторінки потім
показують опубліковані метадані, файли, атрибуцію джерела та статус сканування.

## Встановлення та оновлення

Команди встановлення OpenClaw використовують ClawHub як джерело пакетів:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw записує метадані джерела встановлення, щоб оновлення могли пізніше знаходити той самий
пакет реєстру. CLI ClawHub також підтримує прямі робочі процеси встановлення та
оновлення skill для користувачів, які хочуть керовані реєстром теки skill поза
повним робочим простором OpenClaw.

## Стан безпеки

ClawHub відкритий для публікації, але випуски все одно підлягають шлюзам завантаження,
автоматизованим перевіркам, звітам користувачів і діям модераторів.

Публічні сторінки показують підсумки сканування, коли вони доступні. Вміст, який утримано, приховано
або заблоковано, може зникнути з публічного пошуку та потоків установлення, але залишатися
видимим для власника для діагностики.

Див. [Безпека](/clawhub/security), [Аудити безпеки](/clawhub/security-audits),
[Модерація та безпека облікового запису](/uk/clawhub/moderation) і
[Прийнятне використання](/clawhub/acceptable-usage).

## Доступ до API

ClawHub надає публічні API для читання для виявлення, пошуку, відомостей про пакети та
завантажень. Сторонні каталоги можуть використовувати ці API, коли вони посилаються на
канонічну публікацію ClawHub, дотримуються обмежень частоти запитів і не створюють враження схвалення.

Див. [Публічний API](/uk/clawhub/api) і [HTTP API](/clawhub/http-api).
