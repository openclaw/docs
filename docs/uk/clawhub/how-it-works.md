# Як працює ClawHub

ClawHub — це рівень реєстру для OpenClaw Skills і плагінів. Він дає користувачам
місце для пошуку пакетів, видавцям — місце для випуску версій, а
OpenClaw — достатньо метаданих, щоб безпечно встановлювати й оновлювати ці пакети.

## Записи реєстру

Кожен публічний запис — це запис реєстру, що містить:

- власника і slug або назву пакета
- одну або кілька опублікованих версій
- метадані, опис, файли та атрибуцію джерела
- журнал змін і інформацію про теги, як-от `latest`
- сигнали завантажень, встановлень, зірок і коментарів
- стан сканування безпеки та модерації

Сторінка запису — це канонічне місце, де користувачі можуть перевірити, що skill або
плагін заявляє про свої можливості, перш ніж його встановлювати.

## Skills

Skill — це версійний текстовий пакет, зосереджений навколо `SKILL.md`. Він може містити
допоміжні файли, приклади, шаблони та скрипти.

ClawHub читає frontmatter у `SKILL.md`, щоб зрозуміти назву skill,
опис, вимоги, змінні середовища та метадані. Точні
метадані важливі, бо вони допомагають користувачам вирішити, чи встановлювати skill, і
допомагають автоматизованим скануванням виявляти невідповідності між заявленою та спостережуваною поведінкою.

Див. [Формат skill](/uk/clawhub/skill-format).

## Плагіни

Плагіни — це запаковані розширення OpenClaw. ClawHub зберігає метадані пакетів,
інформацію про сумісність, посилання на джерела, артефакти та записи версій.

Коли OpenClaw встановлює плагін із ClawHub, він перевіряє заявлені метадані
сумісності перед встановленням. Записи пакетів можуть містити сумісність API,
мінімальну версію Gateway, цільові хости, вимоги до середовища та
дайджести артефактів.

Використовуйте явне джерело встановлення ClawHub, коли потрібно, щоб реєстр був
джерелом істини:

```bash
openclaw plugins install clawhub:<package>
```

## Публікація

Публікація створює новий незмінний запис версії. Видавці використовують `clawhub`
CLI для автентифікованих робочих процесів реєстру:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Використовуйте пробні запуски, щоб попередньо переглянути визначене корисне навантаження перед завантаженням. Потім публічні сторінки
показують опубліковані метадані, файли, атрибуцію джерела та стан сканування.

## Встановлення й оновлення

Команди встановлення OpenClaw використовують ClawHub як джерело пакетів:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

OpenClaw записує метадані джерела встановлення, щоб оновлення згодом могли визначити той самий
пакет реєстру. ClawHub CLI також підтримує прямі робочі процеси встановлення й
оновлення skill для користувачів, яким потрібні керовані реєстром папки skill поза
повним робочим простором OpenClaw.

## Стан безпеки

ClawHub відкритий для публікації, але випуски все одно проходять через шлюзи завантаження,
автоматизовані перевірки, звіти користувачів і дії модераторів.

Публічні сторінки показують зведення сканування, коли вони доступні. Вміст, який утримано, приховано
або заблоковано, може зникнути з публічного пошуку та потоків встановлення, але залишатися
видимим власнику для діагностики.

Див. [Безпека + модерація](/uk/clawhub/security) і
[Прийнятне використання](/uk/clawhub/acceptable-usage).

## Доступ до API

ClawHub надає публічні API читання для виявлення, пошуку, деталей пакетів і
завантажень. Сторонні каталоги можуть використовувати ці API, якщо вони посилаються назад на
канонічний запис ClawHub, дотримуються обмежень частоти запитів і не створюють враження схвалення.

Див. [Публічний API](/uk/clawhub/api) і [HTTP API](/uk/clawhub/http-api).
