# Як працює ClawHub

ClawHub — це шар реєстру для навичок і плагінів OpenClaw. Він дає користувачам
місце для пошуку пакетів, видавцям — місце для випуску версій, а
OpenClaw — достатньо метаданих, щоб безпечно встановлювати й оновлювати ці пакети.

## Записи реєстру

Кожна публічна сторінка є записом реєстру з такими даними:

- власник і slug або назва пакета
- одна або кілька опублікованих версій
- метадані, опис, файли й зазначення джерела
- журнал змін та інформація про теги, як-от `latest`
- сигнали завантаження, встановлення й зірок
- стан сканування безпеки та модерації

Сторінка публікації є канонічним місцем, де користувачі можуть перевірити, що
навичка або плагін заявляє про свої можливості, перш ніж встановлювати її чи його.

## Навички

Навичка — це версіонований текстовий пакет, зосереджений навколо `SKILL.md`. Вона може містити
допоміжні файли, приклади, шаблони та скрипти.

ClawHub читає frontmatter у `SKILL.md`, щоб зрозуміти назву навички,
опис, вимоги, змінні середовища та метадані. Точні
метадані важливі, бо вони допомагають користувачам вирішити, чи встановлювати навичку, і
допомагають автоматизованим перевіркам виявляти невідповідності між заявленою та спостережуваною поведінкою.

Див. [Формат навички](/uk/clawhub/skill-format).

## Плагіни

Плагіни — це упаковані розширення OpenClaw. ClawHub зберігає метадані пакетів,
інформацію про сумісність, посилання на джерела, артефакти та записи версій.

Коли OpenClaw встановлює плагін із ClawHub, він перевіряє заявлені метадані
сумісності перед встановленням. Записи пакетів можуть містити сумісність API,
мінімальну версію Gateway, цільові хости, вимоги до середовища та дайджести
артефактів.

Використовуйте явне джерело встановлення ClawHub, коли хочете, щоб реєстр був
авторитетним джерелом:

```bash
openclaw plugins install clawhub:<package>
```

## Публікація

Публікація створює новий незмінний запис версії. Видавці використовують CLI `clawhub`
для автентифікованих робочих процесів реєстру:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Використовуйте пробні запуски, щоб попередньо переглянути вирішене корисне навантаження перед завантаженням. Публічні сторінки потім
показують опубліковані метадані, файли, зазначення джерела та стан сканування.

## Встановлення й оновлення

Команди встановлення OpenClaw використовують ClawHub як джерело пакетів:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw записує метадані джерела встановлення, щоб оновлення могли згодом визначити той самий
пакет реєстру. CLI ClawHub також підтримує прямі робочі процеси встановлення й
оновлення навичок для користувачів, які хочуть мати керовані реєстром папки навичок поза
повноцінним робочим простором OpenClaw.

## Стан безпеки

ClawHub відкритий для публікації, але випуски все одно проходять через шлюзи завантаження,
автоматизовані перевірки, звіти користувачів і дії модераторів.

Публічні сторінки показують підсумки сканування, коли вони доступні. Вміст, який утримано, приховано
або заблоковано, може зникнути з публічного пошуку та процесів встановлення, але залишатися
видимим для власника для діагностики.

Див. [Безпека](/uk/clawhub/security), [Аудити безпеки](/uk/clawhub/security-audits),
[Модерація та безпека облікового запису](/uk/clawhub/moderation) і
[Допустиме використання](/uk/clawhub/acceptable-usage).

## Доступ до API

ClawHub надає публічні API читання для виявлення, пошуку, деталей пакетів і
завантажень. Сторонні каталоги можуть використовувати ці API, коли вони посилаються назад на
канонічну сторінку ClawHub, дотримуються лімітів частоти запитів і не створюють враження схвалення.

Див. [Публічний API](/uk/clawhub/api) і [HTTP API](/uk/clawhub/http-api).
