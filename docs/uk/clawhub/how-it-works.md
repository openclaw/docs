# Як працює ClawHub

ClawHub — це шар реєстру для Skills і Plugins OpenClaw. Він дає користувачам
місце для пошуку пакетів, видавцям — місце для випуску версій, а OpenClaw —
достатньо метаданих, щоб безпечно встановлювати й оновлювати ці пакети.

## Записи реєстру

Кожен публічний запис є записом реєстру з:

- власником і slug або назвою пакета
- однією або кількома опублікованими версіями
- метаданими, коротким описом, файлами та зазначенням джерела
- журналом змін і інформацією про теги, як-от `latest`
- сигналами завантажень, встановлень і зірок
- станом сканування безпеки та модерації

Сторінка запису є канонічним місцем, де користувачі можуть перевірити, що саме
skill або plugin заявляє, що робить, перш ніж його встановити.

## Skills

skill — це версійований текстовий пакет, зосереджений на `SKILL.md`. Він може
містити допоміжні файли, приклади, шаблони та скрипти.

ClawHub читає frontmatter `SKILL.md`, щоб зрозуміти назву skill, опис,
вимоги, змінні середовища та метадані. Точні метадані важливі, бо вони
допомагають користувачам вирішити, чи встановлювати skill, і допомагають
автоматизованим скануванням виявляти невідповідності між заявленою та
спостережуваною поведінкою.

Див. [Формат skill](/uk/clawhub/skill-format).

## Plugins

Plugins — це запаковані розширення OpenClaw. ClawHub зберігає метадані пакета,
інформацію про сумісність, посилання на джерела, артефакти та записи версій.

Коли OpenClaw встановлює plugin із ClawHub, він перевіряє заявлені метадані
сумісності перед встановленням. Записи пакетів можуть містити сумісність API,
мінімальну версію gateway, цільові хости, вимоги до середовища та дайджести
артефактів.

Використовуйте явне джерело встановлення ClawHub, коли хочете, щоб реєстр був
джерелом істини:

```bash
openclaw plugins install clawhub:<package>
```

## Публікація

Публікація створює новий незмінний запис версії. Видавці використовують CLI
`clawhub` для автентифікованих робочих процесів реєстру:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Використовуйте пробні запуски, щоб попередньо переглянути розв’язане корисне
навантаження перед завантаженням. Після цього публічні сторінки показують
опубліковані метадані, файли, зазначення джерела та стан сканування.

## Встановлення та оновлення

Команди встановлення OpenClaw використовують ClawHub як джерело пакетів:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw записує метадані джерела встановлення, щоб оновлення могли пізніше
розв’язати той самий пакет реєстру. CLI ClawHub також підтримує прямі робочі
процеси встановлення та оновлення skills для користувачів, які хочуть мати
керовані реєстром папки skills поза повним робочим простором OpenClaw.

## Стан безпеки

ClawHub відкритий для публікації, але випуски все одно проходять через шлюзи
завантаження, автоматизовані перевірки, звіти користувачів і дії модераторів.

Публічні сторінки показують підсумки сканування, коли вони доступні. Вміст,
який утримується, прихований або заблокований, може зникнути з публічного
пошуку та потоків встановлення, але залишатися видимим власнику для
діагностики.

Див. [Безпека](/uk/clawhub/security), [Аудити безпеки](/uk/clawhub/security-audits),
[Модерація та безпека облікового запису](/uk/clawhub/moderation) і
[Прийнятне використання](/uk/clawhub/acceptable-usage).

## Доступ до API

ClawHub надає публічні API читання для виявлення, пошуку, відомостей про
пакети та завантажень. Сторонні каталоги можуть використовувати ці API, коли
вони посилаються на канонічний запис ClawHub, дотримуються обмежень частоти
запитів і не створюють враження схвалення.

Див. [Публічний API](/uk/clawhub/api) і [HTTP API](/uk/clawhub/http-api).
