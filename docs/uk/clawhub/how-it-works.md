# Як працює ClawHub

ClawHub — це шар реєстру для OpenClaw Skills і plugins. Він дає користувачам
місце для пошуку пакетів, видавцям — місце для випуску версій, а OpenClaw —
достатньо метаданих, щоб безпечно встановлювати й оновлювати ці пакети.

## Записи реєстру

Кожна публічна сторінка є записом реєстру з:

- власником і slug або назвою пакета
- однією чи кількома опублікованими версіями
- метаданими, коротким описом, файлами та атрибуцією джерела
- журналом змін та інформацією про теги, як-от `latest`
- сигналами завантажень, встановлень і зірок
- станом сканування безпеки та модерації

Сторінка запису є канонічним місцем, де користувачі можуть перевірити, що
Skill або plugin заявляє про себе перед встановленням.

## Skills

Skill — це версійований текстовий пакет, зосереджений навколо `SKILL.md`. Він
може містити допоміжні файли, приклади, шаблони та скрипти.

ClawHub читає frontmatter у `SKILL.md`, щоб зрозуміти назву Skill,
опис, вимоги, змінні середовища та метадані. Точні метадані важливі, бо вони
допомагають користувачам вирішити, чи встановлювати Skill, і допомагають
автоматизованим скануванням виявляти невідповідності між заявленою та
спостережуваною поведінкою.

Див. [Формат Skill](/uk/clawhub/skill-format).

## Plugins

Plugins — це пакетовані розширення OpenClaw. ClawHub зберігає метадані пакета,
інформацію про сумісність, посилання на джерела, артефакти та записи версій.

Коли OpenClaw встановлює plugin із ClawHub, він перевіряє оголошені метадані
сумісності перед встановленням. Записи пакета можуть містити сумісність API,
мінімальну версію Gateway, цільові хости, вимоги до середовища та дайджести
артефактів.

Використовуйте явне джерело встановлення ClawHub, коли потрібно, щоб реєстр був
джерелом істини:

```bash
openclaw plugins install clawhub:<package>
```

## Публікація

Публікація створює новий незмінний запис версії. Видавці використовують CLI
`clawhub` для автентифікованих робочих процесів реєстру:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Використовуйте пробні запуски, щоб попередньо переглянути визначене корисне
навантаження перед завантаженням. Після цього публічні сторінки показують
опубліковані метадані, файли, атрибуцію джерела та стан сканування.

## Встановлення й оновлення

Команди встановлення OpenClaw використовують ClawHub як джерело пакетів:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw записує метадані джерела встановлення, щоб оновлення могли пізніше
визначити той самий пакет реєстру. CLI ClawHub також підтримує прямі робочі
процеси встановлення й оновлення Skill для користувачів, які хочуть мати
керовані реєстром папки Skill поза повним робочим простором OpenClaw.

## Стан безпеки

ClawHub відкритий для публікації, але випуски все одно підлягають шлюзам
завантаження, автоматизованим перевіркам, звітам користувачів і діям
модераторів.

Публічні сторінки показують зведення сканування, коли вони доступні. Вміст,
який утримується, прихований або заблокований, може зникнути з публічного
пошуку та потоків встановлення, залишаючись видимим власнику для діагностики.

Див. [Безпека](/clawhub/security), [Аудити безпеки](/clawhub/security-audits),
[Модерація та безпека облікового запису](/uk/clawhub/moderation) і
[Прийнятне використання](/clawhub/acceptable-usage).

## Доступ до API

ClawHub надає публічні API читання для виявлення, пошуку, деталей пакетів і
завантажень. Сторонні каталоги можуть використовувати ці API, коли вони
посилаються назад на канонічну сторінку ClawHub, дотримуються обмежень частоти
запитів і не створюють враження схвалення.

Див. [Публічний API](/uk/clawhub/api) і [HTTP API](/clawhub/http-api).
