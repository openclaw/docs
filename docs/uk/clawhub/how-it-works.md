# Як працює ClawHub

ClawHub — це реєстровий рівень для Skills і плагінів OpenClaw. Він дає користувачам
місце для пошуку пакетів, видавцям — місце для випуску версій, а
OpenClaw — достатньо метаданих, щоб безпечно встановлювати й оновлювати ці пакети.

## Записи реєстру

Кожна публічна сторінка є записом реєстру з:

- власником і slug або назвою пакета
- однією або кількома опублікованими версіями
- метаданими, коротким описом, файлами та зазначенням джерела
- журналом змін і інформацією про теги, як-от `latest`
- сигналами завантаження, встановлення та зірок
- статусом сканування безпеки та модерації

Сторінка списку є канонічним місцем, де користувачі можуть перевірити, що, за заявою, робить skill або
плагін, перш ніж установлювати його.

## Skills

Skill — це версійний текстовий пакет із центром у `SKILL.md`. Він може містити
допоміжні файли, приклади, шаблони та скрипти.

ClawHub читає frontmatter у `SKILL.md`, щоб зрозуміти назву skill,
опис, вимоги, змінні середовища та метадані. Точні
метадані важливі, оскільки вони допомагають користувачам вирішити, чи встановлювати skill, і
допомагають автоматизованим перевіркам виявляти невідповідності між заявленою та спостережуваною поведінкою.

Див. [Формат Skill](/uk/clawhub/skill-format).

## Плагіни

Плагіни — це запаковані розширення OpenClaw. ClawHub зберігає метадані пакетів,
інформацію про сумісність, посилання на джерела, артефакти та записи версій.

Коли OpenClaw встановлює плагін із ClawHub, він перевіряє заявлені метадані
сумісності перед встановленням. Записи пакетів можуть містити сумісність API,
мінімальну версію gateway, цільові хости, вимоги до середовища та дайджести
артефактів.

Використовуйте явне джерело встановлення ClawHub, коли хочете, щоб реєстр був
джерелом істини:

```bash
openclaw plugins install clawhub:<package>
```

## Публікація

Публікація створює новий незмінний запис версії. Видавці використовують CLI `clawhub`
для автентифікованих робочих процесів реєстру:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Використовуйте пробні запуски, щоб переглянути розв’язане корисне навантаження перед завантаженням. Після цього публічні сторінки
показують опубліковані метадані, файли, зазначення джерела та статус сканування.

## Встановлення та оновлення

Команди встановлення OpenClaw використовують ClawHub як джерело пакетів:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw записує метадані джерела встановлення, щоб оновлення могли згодом розв’язати той самий
пакет реєстру. CLI ClawHub також підтримує прямі робочі процеси встановлення й
оновлення skill для користувачів, які хочуть керовані реєстром папки skill поза
повним робочим простором OpenClaw.

## Стан безпеки

ClawHub відкритий для публікації, але випуски все одно підлягають шлюзам завантаження,
автоматизованим перевіркам, звітам користувачів і діям модераторів.

Публічні сторінки показують підсумки сканування, коли вони доступні. Вміст, який утримується, прихований
або заблокований, може зникати з публічного пошуку та потоків установлення, водночас залишаючись
видимим для власника для діагностики.

Див. [Безпека](/clawhub/security), [Аудити безпеки](/clawhub/security-audits),
[Модерація та безпека облікового запису](/uk/clawhub/moderation) і
[Прийнятне використання](/clawhub/acceptable-usage).

## Доступ до API

ClawHub надає публічні API читання для виявлення, пошуку, деталей пакетів і
завантажень. Сторонні каталоги можуть використовувати ці API, коли вони посилаються назад на
канонічну сторінку ClawHub, дотримуються обмежень частоти та не створюють враження схвалення.

Див. [Публічний API](/uk/clawhub/api) і [HTTP API](/clawhub/http-api).
