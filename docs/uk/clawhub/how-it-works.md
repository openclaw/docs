# Як працює ClawHub

ClawHub — це шар реєстру для навичок і плагінів OpenClaw. Він дає користувачам
місце для пошуку пакетів, видавцям — місце для випуску версій, а OpenClaw —
достатньо метаданих, щоб безпечно встановлювати й оновлювати ці пакети.

## Записи реєстру

Кожна публічна сторінка є записом реєстру з:

- власником і слагом або назвою пакета
- однією або кількома опублікованими версіями
- метаданими, зведенням, файлами й зазначенням джерела
- журналом змін та інформацією про теги, як-от `latest`
- сигналами завантаження, встановлення й зірок
- статусом сканування безпеки та модерації

Сторінка запису є канонічним місцем, де користувачі можуть перевірити, що
навичка або плагін заявляє про себе перед встановленням.

## Skills

Навичка — це версійований текстовий пакет, зосереджений навколо `SKILL.md`. Вона
може містити допоміжні файли, приклади, шаблони та скрипти.

ClawHub читає frontmatter `SKILL.md`, щоб зрозуміти назву навички, опис, вимоги,
змінні середовища та метадані. Точні метадані важливі, бо вони допомагають
користувачам вирішити, чи встановлювати навичку, і допомагають автоматизованим
скануванням виявляти невідповідності між заявленою та спостережуваною
поведінкою.

Див. [Формат навички](/uk/clawhub/skill-format).

## Плагіни

Плагіни — це упаковані розширення OpenClaw. ClawHub зберігає метадані пакетів,
інформацію про сумісність, посилання на джерела, артефакти та записи версій.

Коли OpenClaw встановлює плагін із ClawHub, він перевіряє заявлені метадані
сумісності перед встановленням. Записи пакетів можуть містити сумісність API,
мінімальну версію Gateway, цільові хости, вимоги до середовища й дайджести
артефактів.

Використовуйте явне джерело встановлення ClawHub, коли хочете, щоб реєстр був
джерелом істини:

```bash
openclaw plugins install clawhub:<package>
```

## Публікація

Публікація створює новий незмінний запис версії. Видавці використовують CLI
`clawhub` для автентифікованих робочих процесів реєстру:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Використовуйте пробні запуски, щоб переглянути розв’язане корисне навантаження
перед завантаженням. Після цього публічні сторінки показують опубліковані
метадані, файли, зазначення джерела та статус сканування.

## Встановлення й оновлення

Команди встановлення OpenClaw використовують ClawHub як джерело пакетів:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw записує метадані джерела встановлення, щоб оновлення згодом могли
розв’язати той самий пакет реєстру. CLI ClawHub також підтримує безпосередні
робочі процеси встановлення й оновлення навичок для користувачів, які хочуть
папки навичок, керовані реєстром, поза повною робочою областю OpenClaw.

## Стан безпеки

ClawHub відкритий для публікації, але випуски все одно підлягають шлюзам
завантаження, автоматизованим перевіркам, звітам користувачів і діям
модераторів.

Публічні сторінки показують зведення сканування, коли вони доступні. Вміст, який
утримується, прихований або заблокований, може зникнути з публічного пошуку та
процесів встановлення, водночас залишаючись видимим для власника для діагностики.

Див. [Безпека](/uk/clawhub/security), [Аудити безпеки](/uk/clawhub/security-audits),
[Модерація та безпека облікового запису](/uk/clawhub/moderation) і
[Прийнятне використання](/uk/clawhub/acceptable-usage).

## Доступ до API

ClawHub надає публічні API читання для виявлення, пошуку, деталей пакетів і
завантажень. Сторонні каталоги можуть використовувати ці API, коли вони
посилаються назад на канонічну сторінку ClawHub, дотримуються обмежень частоти
запитів і не створюють враження схвалення.

Див. [Публічний API](/uk/clawhub/api) і [HTTP API](/uk/clawhub/http-api).
