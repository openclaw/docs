# Як працює ClawHub

ClawHub — це рівень реєстру для Skills і плагінів OpenClaw. Він надає користувачам
місце для пошуку пакетів, видавцям — місце для публікації версій, а OpenClaw —
достатньо метаданих для безпечного встановлення й оновлення цих пакетів.

## Записи реєстру

Кожна загальнодоступна сторінка пакета є записом реєстру, що містить:

- власника та коротку назву або назву пакета
- одну чи кілька опублікованих версій
- метадані, стислий опис, файли та відомості про джерело
- журнал змін і відомості про теги, як-от `latest`
- показники завантажень, установлень і позначок «зіркою»
- результати перевірки безпеки та статус модерації

Сторінка пакета — це канонічне місце, де користувачі можуть перевірити заявлене
призначення Skills або плагіна перед його встановленням.

## Skills

Skills — це версійований текстовий пакет, основою якого є `SKILL.md`. Він може
містити допоміжні файли, приклади, шаблони та скрипти.

ClawHub читає frontmatter файлу `SKILL.md`, щоб визначити назву Skills,
опис, вимоги, змінні середовища та метадані. Точні метадані важливі, оскільки
допомагають користувачам вирішити, чи варто встановлювати Skills, а
автоматизованим перевіркам — виявляти невідповідності між заявленою та фактично
спостережуваною поведінкою.

Див. [Формат Skills](/clawhub/skill-format).

## Плагіни

Плагіни — це запаковані розширення OpenClaw. ClawHub зберігає метадані пакетів,
відомості про сумісність, посилання на джерела, артефакти та записи версій.

Коли OpenClaw встановлює плагін із ClawHub, перед установленням він перевіряє
заявлені метадані сумісності. Записи пакетів можуть містити відомості про
сумісність API, мінімальну версію Gateway, цільові хости, вимоги до середовища
та дайджести артефактів.

Явно вказуйте ClawHub як джерело встановлення, якщо реєстр має бути
джерелом істини:

```bash
openclaw plugins install clawhub:<package>
```

## Публікація

Публікація створює новий незмінний запис версії. Видавці використовують CLI
`clawhub` для автентифікованих операцій із реєстром:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Використовуйте пробні запуски, щоб переглянути сформовані дані перед
завантаженням. Після цього загальнодоступні сторінки показують опубліковані
метадані, файли, відомості про джерело та статус перевірки.

## Установлення й оновлення

Команди встановлення OpenClaw використовують ClawHub як джерело пакетів:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw записує метадані джерела встановлення, щоб під час наступних оновлень
можна було знайти той самий пакет у реєстрі. CLI ClawHub також підтримує прямі
операції встановлення й оновлення Skills для користувачів, яким потрібні
керовані реєстром папки Skills поза повним робочим простором OpenClaw.

## Стан безпеки

ClawHub відкритий для публікацій, але випуски все одно проходять перевірки під
час завантаження, автоматизовані перевірки, розгляд повідомлень користувачів і
модерацію.

Загальнодоступні сторінки показують зведення результатів перевірок, коли вони
доступні. Вміст, який затримано, приховано або заблоковано, може зникнути із
загальнодоступного пошуку та процесів установлення, але залишатися видимим
власнику для діагностики.

Див. [Безпека](/uk/clawhub/security), [Аудити безпеки](/clawhub/security-audits),
[Модерація та безпека облікового запису](/uk/clawhub/moderation) і
[Допустиме використання](/clawhub/acceptable-usage).

## Доступ до API

ClawHub надає загальнодоступні API читання для пошуку, огляду, отримання
відомостей про пакети та завантажень. Сторонні каталоги можуть використовувати
ці API, якщо вони посилаються на канонічну сторінку пакета в ClawHub,
дотримуються обмежень частоти запитів і не створюють враження офіційного
схвалення.

Див. [Загальнодоступний API](/clawhub/api) і [HTTP API](/clawhub/http-api).
