# Як працює ClawHub

ClawHub — це рівень реєстру для OpenClaw Skills і Plugin. Він дає користувачам
місце для пошуку пакетів, видавцям — місце для випуску версій, а OpenClaw —
достатньо метаданих, щоб безпечно встановлювати й оновлювати ці пакети.

## Записи реєстру

Кожна публічна сторінка є записом реєстру із:

- власником і slug або назвою пакета
- однією чи кількома опублікованими версіями
- метаданими, коротким описом, файлами та атрибуцією джерела
- журналом змін і інформацією про теги, як-от `latest`
- сигналами завантажень, встановлень і зірок
- статусом сканування безпеки та модерації

Сторінка списку є канонічним місцем, де користувачі можуть перевірити, що
навичка або plugin заявляє про свої можливості, перш ніж установлювати її.

## Skills

Навичка — це версіонований текстовий пакет, зосереджений навколо `SKILL.md`. Він може містити
допоміжні файли, приклади, шаблони та скрипти.

ClawHub читає frontmatter у `SKILL.md`, щоб зрозуміти назву навички,
опис, вимоги, змінні середовища та метадані. Точні
метадані важливі, бо вони допомагають користувачам вирішити, чи встановлювати навичку, і
допомагають автоматизованим скануванням виявляти невідповідності між заявленою та спостережуваною поведінкою.

Див. [Формат навички](/uk/clawhub/skill-format).

## Plugins

Plugins — це запаковані розширення OpenClaw. ClawHub зберігає метадані пакета,
інформацію про сумісність, посилання на джерела, артефакти та записи версій.

Коли OpenClaw встановлює plugin із ClawHub, він перевіряє заявлені метадані
сумісності перед встановленням. Записи пакетів можуть містити сумісність API,
мінімальну версію gateway, цільові хости, вимоги до середовища та дайджести
артефактів.

Використовуйте явне джерело встановлення ClawHub, коли хочете, щоб реєстр був
джерелом істини:

```bash
openclaw plugins install clawhub:<package>
```

## Публікація

Публікація створює новий незмінний запис версії. Видавці використовують CLI `clawhub`
для автентифікованих робочих процесів реєстру:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Використовуйте пробні запуски, щоб попередньо переглянути розв’язане корисне навантаження перед завантаженням. Потім публічні сторінки
показують опубліковані метадані, файли, атрибуцію джерела та статус сканування.

## Встановлення та оновлення

Команди встановлення OpenClaw використовують ClawHub як джерело пакетів:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw записує метадані джерела встановлення, щоб оновлення могли пізніше розв’язати той самий
пакет реєстру. CLI ClawHub також підтримує прямі робочі процеси встановлення та
оновлення навичок для користувачів, які хочуть керовані реєстром папки навичок поза
повним робочим простором OpenClaw.

## Стан безпеки

ClawHub відкритий для публікації, але випуски все одно підлягають шлюзам завантаження,
автоматизованим перевіркам, звітам користувачів і діям модераторів.

Публічні сторінки показують підсумки сканування, коли вони доступні. Вміст, який утримано, приховано
або заблоковано, може зникнути з публічного пошуку та потоків встановлення, але залишатися
видимим для власника для діагностики.

Див. [Безпека](/uk/clawhub/security), [Аудити безпеки](/uk/clawhub/security-audits),
[Модерація та безпека облікового запису](/uk/clawhub/moderation) і
[Прийнятне використання](/uk/clawhub/acceptable-usage).

## Доступ до API

ClawHub надає публічні API читання для виявлення, пошуку, деталей пакетів і
завантажень. Сторонні каталоги можуть використовувати ці API, коли вони посилаються назад на
канонічний список ClawHub, дотримуються обмежень швидкості та уникають натяків на схвалення.

Див. [Публічний API](/uk/clawhub/api) і [HTTP API](/uk/clawhub/http-api).
