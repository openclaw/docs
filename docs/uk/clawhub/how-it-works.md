# Як працює ClawHub

ClawHub — це реєстровий рівень для OpenClaw Skills і plugins. Він надає користувачам
місце для пошуку пакетів, видавцям — місце для випуску версій, а
OpenClaw — достатньо метаданих, щоб безпечно встановлювати й оновлювати ці пакети.

## Записи реєстру

Кожна публічна сторінка є записом реєстру з:

- власником і slug або назвою пакета
- однією або кількома опублікованими версіями
- метаданими, коротким описом, файлами та атрибуцією джерела
- журналом змін та інформацією про теги, як-от `latest`
- сигналами завантаження, встановлення та зірок
- станом сканування безпеки та модерації

Сторінка запису є канонічним місцем, де користувачі можуть перевірити, що skill або
plugin заявляє про свою функціональність перед встановленням.

## Skills

Skill — це версійований текстовий пакет, побудований навколо `SKILL.md`. Він може містити
допоміжні файли, приклади, шаблони та скрипти.

ClawHub читає frontmatter у `SKILL.md`, щоб зрозуміти назву skill,
опис, вимоги, змінні середовища та метадані. Точні
метадані важливі, бо вони допомагають користувачам вирішити, чи встановлювати skill, і
допомагають автоматизованим скануванням виявляти невідповідності між заявленою та спостережуваною поведінкою.

Див. [Формат Skill](/uk/clawhub/skill-format).

## Plugins

Plugins — це пакетовані розширення OpenClaw. ClawHub зберігає метадані пакета,
інформацію про сумісність, посилання на джерело, артефакти та записи версій.

Коли OpenClaw встановлює plugin із ClawHub, він перевіряє заявлені метадані
сумісності перед встановленням. Записи пакетів можуть містити сумісність API,
мінімальну версію Gateway, цільові хости, вимоги до середовища та дайджести
артефактів.

Використовуйте явне джерело встановлення ClawHub, коли хочете, щоб реєстр був
джерелом істини:

```bash
openclaw plugins install clawhub:<package>
```

## Публікація

Публікація створює новий незмінний запис версії. Видавці використовують `clawhub`
CLI для автентифікованих реєстрових робочих процесів:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Використовуйте пробні запуски, щоб попередньо переглянути розв’язане корисне навантаження перед завантаженням. Публічні сторінки потім
показують опубліковані метадані, файли, атрибуцію джерела та стан сканування.

## Встановлення та оновлення

Команди встановлення OpenClaw використовують ClawHub як джерело пакетів:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw записує метадані джерела встановлення, щоб оновлення могли згодом розв’язувати той самий
пакет реєстру. ClawHub CLI також підтримує прямі робочі процеси встановлення й
оновлення skill для користувачів, які хочуть мати керовані реєстром теки skill поза
повним робочим простором OpenClaw.

## Стан безпеки

ClawHub відкритий для публікації, але випуски все одно підлягають шлюзам завантаження,
автоматизованим перевіркам, звітам користувачів і діям модераторів.

Публічні сторінки показують підсумки сканування, коли вони доступні. Вміст, який утримано, приховано
або заблоковано, може зникнути з публічного пошуку та потоків встановлення, але залишатися
видимим для власника з діагностичною метою.

Див. [Безпека](/clawhub/security), [Аудити безпеки](/clawhub/security-audits),
[Модерація та безпека облікового запису](/uk/clawhub/moderation) і
[Прийнятне використання](/clawhub/acceptable-usage).

## Доступ до API

ClawHub надає публічні API для читання, призначені для пошуку, огляду, деталей пакетів і
завантажень. Сторонні каталоги можуть використовувати ці API, якщо вони посилаються назад на
канонічну сторінку ClawHub, дотримуються обмежень частоти запитів і не створюють враження схвалення.

Див. [Публічний API](/uk/clawhub/api) і [HTTP API](/clawhub/http-api).
