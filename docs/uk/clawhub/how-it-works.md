# Як працює ClawHub

ClawHub — це реєстровий рівень для OpenClaw Skills і plugins. Він дає користувачам
місце для пошуку пакетів, видавцям — місце для випуску версій, а
OpenClaw — достатньо метаданих, щоб безпечно встановлювати й оновлювати ці пакети.

## Записи реєстру

Кожен публічний запис є записом реєстру із:

- власником і слагом або назвою пакета
- однією чи кількома опублікованими версіями
- метаданими, коротким описом, файлами й атрибуцією джерела
- журналом змін та інформацією про теги, наприклад `latest`
- сигналами завантажень, встановлень, зірок і коментарів
- станом безпекового сканування та модерації

Сторінка запису є канонічним місцем, де користувачі можуть перевірити, що skill або
plugin заявляє про себе перед встановленням.

## Skills

Skill — це версійований текстовий пакет, зосереджений на `SKILL.md`. Він може містити
допоміжні файли, приклади, шаблони та скрипти.

ClawHub читає frontmatter у `SKILL.md`, щоб зрозуміти назву skill,
опис, вимоги, змінні середовища й метадані. Точні
метадані важливі, бо вони допомагають користувачам вирішити, чи встановлювати skill, і
допомагають автоматизованим скануванням виявляти невідповідності між заявленою та спостережуваною поведінкою.

Див. [Формат skill](/uk/clawhub/skill-format).

## Plugins

Plugins — це упаковані розширення OpenClaw. ClawHub зберігає метадані пакетів,
інформацію про сумісність, посилання на джерело, артефакти й записи версій.

Коли OpenClaw встановлює plugin із ClawHub, він перевіряє заявлені метадані сумісності
перед встановленням. Записи пакетів можуть містити сумісність API,
мінімальну версію Gateway, цільові хости, вимоги до середовища й дайджести
артефактів.

Використовуйте явне джерело встановлення ClawHub, коли хочете, щоб реєстр був
джерелом істини:

```bash
openclaw plugins install clawhub:<package>
```

## Публікація

Публікація створює новий незмінний запис версії. Видавці використовують `clawhub`
CLI для автентифікованих робочих процесів реєстру:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Використовуйте пробні запуски, щоб попередньо переглянути розв’язане корисне навантаження перед завантаженням. Після цього публічні сторінки
показують опубліковані метадані, файли, атрибуцію джерела та стан сканування.

## Встановлення й оновлення

Команди встановлення OpenClaw використовують ClawHub як джерело пакетів:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

OpenClaw записує метадані джерела встановлення, щоб оновлення згодом могли розв’язати той самий
пакет реєстру. CLI ClawHub також підтримує прямі робочі процеси встановлення й
оновлення skill для користувачів, які хочуть керовані реєстром теки skill поза
повним робочим простором OpenClaw.

## Стан безпеки

ClawHub відкритий для публікації, але випуски все одно підпадають під шлюзи завантаження,
автоматизовані перевірки, звіти користувачів і дії модераторів.

Публічні сторінки показують зведення сканування, коли вони доступні. Вміст, який утримано, приховано
або заблоковано, може зникати з публічного пошуку та потоків встановлення, залишаючись
видимим для власника з діагностичною метою.

Див. [Безпека + модерація](/uk/clawhub/security) і
[Прийнятне використання](/uk/clawhub/acceptable-usage).

## Доступ до API

ClawHub надає публічні API для читання для виявлення, пошуку, деталей пакетів і
завантажень. Сторонні каталоги можуть використовувати ці API, коли вони посилаються назад на
канонічний запис ClawHub, дотримуються обмежень частоти запитів і не створюють враження схвалення.

Див. [Публічний API](/uk/clawhub/api) і [HTTP API](/uk/clawhub/http-api).
