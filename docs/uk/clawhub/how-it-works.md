# Як працює ClawHub

ClawHub — це рівень реєстру для навичок і плагінів OpenClaw. Він надає користувачам
місце для пошуку пакетів, видавцям — місце для публікації версій, а
OpenClaw — достатньо метаданих для безпечного встановлення й оновлення цих пакетів.

## Записи реєстру

Кожен загальнодоступний запис у каталозі є записом реєстру, що містить:

- власника та коротку назву або назву пакета
- одну чи кілька опублікованих версій
- метадані, опис, файли та відомості про джерело
- журнал змін та інформацію про теги, як-от `latest`
- показники завантажень, установлень і вподобань
- результати перевірки безпеки та статус модерації

Сторінка запису в каталозі — це канонічне місце, де користувачі можуть перевірити заявлене
призначення навички або плагіна перед його встановленням.

## Навички

Навичка — це версіонований текстовий пакет, зосереджений навколо `SKILL.md`. Він може містити
допоміжні файли, приклади, шаблони та сценарії.

ClawHub зчитує вступні метадані `SKILL.md`, щоб визначити назву навички,
опис, вимоги, змінні середовища та метадані. Точні
метадані важливі, оскільки допомагають користувачам вирішити, чи варто встановлювати навичку, а
автоматизованим перевіркам — виявляти невідповідності між заявленою та фактичною поведінкою.

Див. [Формат навички](/uk/clawhub/skill-format).

## Плагіни

Плагіни — це упаковані розширення OpenClaw. ClawHub зберігає метадані пакетів,
інформацію про сумісність, посилання на джерела, артефакти та записи версій.

Коли OpenClaw встановлює плагін із ClawHub, перед установленням він перевіряє заявлені
метадані сумісності. Записи пакетів можуть містити дані про сумісність API,
мінімальну версію Gateway, цільові хост-системи, вимоги до середовища та контрольні суми
артефактів.

Використовуйте явно вказане джерело встановлення ClawHub, якщо реєстр має бути
джерелом істини:

```bash
openclaw plugins install clawhub:<package>
```

## Публікація

Публікація створює новий незмінний запис версії. Видавці використовують CLI `clawhub`
для автентифікованих операцій із реєстром:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Використовуйте пробні запуски, щоб переглянути сформовані дані перед передаванням. Після цього загальнодоступні сторінки
відображають опубліковані метадані, файли, відомості про джерело та статус перевірки.

## Установлення та оновлення

Команди встановлення OpenClaw використовують ClawHub як джерело пакетів:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw записує метадані джерела встановлення, щоб під час подальших оновлень можна було знайти той самий
пакет у реєстрі. CLI ClawHub також підтримує безпосереднє встановлення та
оновлення навичок для користувачів, які хочуть мати керовані реєстром папки навичок поза
повним робочим простором OpenClaw.

## Стан безпеки

ClawHub відкритий для публікації, однак випуски все одно проходять перевірки під час передавання,
автоматизовані перевірки, розгляд повідомлень користувачів і дії модераторів.

Загальнодоступні сторінки показують зведені результати перевірок, коли вони доступні. Матеріали, які призупинено, приховано
або заблоковано, можуть зникнути із загальнодоступного пошуку та процесів установлення, водночас залишаючись
видимими власнику для діагностики.

Див. [Безпека](/clawhub/security), [Аудити безпеки](/clawhub/security-audits),
[Модерація та безпека облікового запису](/uk/clawhub/moderation) і
[Допустиме використання](/clawhub/acceptable-usage).

## Доступ до API

ClawHub надає загальнодоступні API читання для виявлення, пошуку, отримання відомостей про пакети та
завантажень. Сторонні каталоги можуть використовувати ці API, якщо вони посилаються на
канонічний запис ClawHub, дотримуються обмежень частоти запитів і не створюють враження офіційного схвалення.

Див. [Загальнодоступний API](/clawhub/api) і [HTTP API](/clawhub/http-api).
