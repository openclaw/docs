# Як працює ClawHub

ClawHub — це рівень реєстру для Skills і plugins OpenClaw. Він дає користувачам
місце для пошуку пакетів, видавцям — місце для випуску версій, а OpenClaw —
достатньо метаданих, щоб безпечно встановлювати й оновлювати ці пакети.

## Записи реєстру

Кожна публічна сторінка є записом реєстру з:

- власником і slug або назвою пакета
- однією або кількома опублікованими версіями
- метаданими, коротким описом, файлами та зазначенням джерела
- журналом змін і інформацією про теги, як-от `latest`
- сигналами завантажень, встановлень і зірок
- станом сканування безпеки та модерації

Сторінка публікації є канонічним місцем, де користувачі можуть перевірити, що
Skill або plugin заявляє про себе, перш ніж установлювати його.

## Skills

Skill — це версіонований текстовий пакет, зосереджений навколо `SKILL.md`. Він може містити
допоміжні файли, приклади, шаблони та скрипти.

ClawHub читає frontmatter `SKILL.md`, щоб зрозуміти назву Skill,
опис, вимоги, змінні середовища та метадані. Точні
метадані важливі, бо вони допомагають користувачам вирішити, чи встановлювати Skill, і
допомагають автоматизованим скануванням виявляти невідповідності між заявленою та спостережуваною поведінкою.

Див. [Формат Skill](/uk/clawhub/skill-format).

## Plugins

Plugins — це пакетовані розширення OpenClaw. ClawHub зберігає метадані пакетів,
інформацію про сумісність, посилання на джерела, артефакти та записи версій.

Коли OpenClaw встановлює plugin із ClawHub, він перевіряє заявлені метадані сумісності
перед встановленням. Записи пакетів можуть містити сумісність API,
мінімальну версію Gateway, цільові хости, вимоги до середовища та дайджести
артефактів.

Використовуйте явне джерело встановлення ClawHub, коли хочете, щоб реєстр був
джерелом істини:

```bash
openclaw plugins install clawhub:<package>
```

## Публікація

Публікація створює новий незмінний запис версії. Видавці використовують CLI `clawhub`
для автентифікованих робочих процесів реєстру:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

Використовуйте пробні запуски, щоб попередньо переглянути визначене корисне навантаження перед завантаженням. Публічні сторінки потім
показують опубліковані метадані, файли, зазначення джерела та стан сканування.

## Встановлення та оновлення

Команди встановлення OpenClaw використовують ClawHub як джерело пакетів:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw записує метадані джерела встановлення, щоб оновлення могли згодом визначити той самий
пакет реєстру. CLI ClawHub також підтримує прямі робочі процеси встановлення та
оновлення Skill для користувачів, які хочуть керовані реєстром папки Skill поза
повною робочою областю OpenClaw.

## Стан безпеки

ClawHub відкритий для публікації, але випуски все одно підлягають шлюзам завантаження,
автоматизованим перевіркам, звітам користувачів і діям модераторів.

Публічні сторінки показують підсумки сканування, коли вони доступні. Вміст, який утримано, приховано
або заблоковано, може зникнути з публічного пошуку та потоків встановлення, але залишатися
видимим для власника для діагностики.

Див. [Безпека](/clawhub/security), [Аудити безпеки](/clawhub/security-audits),
[Модерація та безпека акаунта](/uk/clawhub/moderation) і
[Прийнятне використання](/clawhub/acceptable-usage).

## Доступ до API

ClawHub надає публічні API для читання для виявлення, пошуку, деталей пакетів і
завантажень. Сторонні каталоги можуть використовувати ці API, коли вони посилаються на
канонічну сторінку ClawHub, дотримуються обмежень частоти запитів і не створюють враження схвалення.

Див. [Публічний API](/clawhub/api) і [HTTP API](/clawhub/http-api).
