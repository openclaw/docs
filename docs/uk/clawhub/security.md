# Безпека

Про проблеми безпеки ClawHub можна повідомляти через GitHub Security Advisories для
`openclaw/clawhub`.

Використовуйте GitHub Security Advisories для вразливостей у самому ClawHub. Якісні
звіти про вразливості ClawHub включають помилки в:

- вебсайті ClawHub, API або CLI
- публікації в реєстрі, завантаженнях, встановленнях або цілісності артефактів
- автентифікації, авторизації або токенах API
- скануванні, модерації або обробці звітів

Не використовуйте порадники ClawHub для вразливостей у власному вихідному коді сторонньої навички або
Plugin. Повідомляйте про них безпосередньо видавцю або до вихідного
репозиторію, посилання на який наведено в записі ClawHub.

## Розкриття вразливостей

Оскільки ClawHub є розміщеною хмарною програмою, вразливості сервісу ClawHub
типово не розкриваються публічно. Їх розкривають публічно, коли є
докази реального впливу на користувачів або коли користувачам потрібно вжити дій.

Приклади реального впливу на користувачів включають підтверджену експлуатацію, розкриття даних
або секретів користувачів, потрапляння шкідливого вмісту до користувачів через збій платформи
або будь-яку проблему, що вимагає від користувачів ротації облікових даних, оновлення локального програмного забезпечення чи
вжиття інших захисних заходів.

Вразливості в установленому користувачами програмному забезпеченні розкриваються публічно, як-от
пакети CLI ClawHub, двійкові файли, бібліотеки або інші артефакти випуску, які користувачам
потрібно оновити локально.

## Пов’язані сторінки

Про мітки аудиту під час встановлення, рівні ризику, виявлені проблеми та тлумачення див.
[Аудити безпеки](/uk/clawhub/security-audits).

Про звіти маркетплейсу, модераційні утримання, приховані записи, блокування та стан
облікового запису див. [Модерація та безпека облікового запису](/uk/clawhub/moderation).
