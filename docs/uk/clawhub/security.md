# Безпека + модерація

ClawHub відкритий для публікації, але публічні оголошення все одно проходять через засоби довіри,
сканування, звітування та модерації. Мета практична: допомогти користувачам
перевіряти, що вони встановлюють, дати видавцям шлях відновлення після хибних спрацювань
і не допускати шкідливі пакети до публічного виявлення.

Див. також [Прийнятне використання](/uk/clawhub/acceptable-usage).

## Що можуть перевірити користувачі

Перед встановленням Skills або Plugin перевірте його сторінку ClawHub щодо:

- власника й атрибуції джерела
- останньої версії та журналу змін
- потрібних змінних середовища або дозволів
- метаданих сумісності для plugins
- стану сканування або модерації
- звітів, коментарів, зірок, завантажень і сигналів встановлення, де вони показані

Встановлюйте лише вміст, який ви розумієте й якому довіряєте.

## Стани сканування

ClawHub може показувати результати сканування або модерації на публічних сторінках і в діагностиці,
видимій власнику.

Поширені результати:

- `clean`: блокувальної проблеми не знайдено.
- `suspicious`: реліз потребує обережності або перевірки.
- `malicious`: реліз вважається небезпечним.
- `pending`: перевірки ще не завершено.
- `held`, `quarantined`, `revoked` або `hidden`: реліз не повністю
  доступний на публічних поверхнях встановлення.

Точне формулювання може відрізнятися залежно від поверхні, але практичне значення те саме: якщо
реліз утримано або заблоковано, користувачам не слід встановлювати його, доки власник не розв'яже
проблему або модерація не відновить реліз.

## Skills

Сканування Skills перевіряє опублікований пакет skill, метадані, заявлені
вимоги та підозрілі інструкції.

ClawHub приділяє особливу увагу невідповідностям між тим, що skill заявляє, і
тим, що він, схоже, робить. Наприклад, skill, який посилається на потрібний ключ API,
має задекларувати цю вимогу в `SKILL.md`, щоб користувачі могли побачити її перед
встановленням.

Висновки сканування базуються на артефактах. Очікувана поведінка провайдера, така як заявлені
облікові дані API, localhost OAuth callbacks, scoped uninstall cleanup, Basic Auth
encoding або вибрані користувачем завантаження файлів до заявленого провайдера, трактується
інакше, ніж приховане пересилання облікових даних, широкий доступ до приватних файлів,
непов'язані мережеві призначення або приховане зловживання браузером.

Див. [Формат skill](/uk/clawhub/skill-format).

## Plugins

Релізи Plugin містять метадані пакета, атрибуцію джерела, поля сумісності
та інформацію про цілісність артефакта.

OpenClaw перевіряє сумісність перед встановленням plugins, розміщених у ClawHub. Записи пакетів
також можуть надавати метадані дайджесту, щоб OpenClaw міг перевіряти завантажені
артефакти. ClawScan включає заявлені метадані env/config пакета `openclaw.environment`
під час перевірки релізів Plugin, щоб заявлені вимоги виконання
порівнювалися зі спостережуваною поведінкою.

## Звіти

Користувачі, що ввійшли в систему, можуть повідомляти про Skills, пакети та коментарі.

Звіти мають бути конкретними й придатними до дії. Зловживання звітуванням саме по собі може призвести до
дій щодо облікового запису.

Приклади звітів:

- оманливі метадані
- незадекларовані вимоги до облікових даних або дозволів
- підозрілі інструкції встановлення
- шахрайські коментарі або видавання себе за іншу особу
- недобросовісні реєстрації або зловживання торговельними марками
- вміст, що порушує [Прийнятне використання](/uk/clawhub/acceptable-usage)

## Примітки ClawScan для видавців

Видавці можуть надати необов'язкову примітку ClawScan під час публікації Skills або
Plugin. Ця примітка дає ClawScan контекст для поведінки, яка інакше може виглядати
незвичною, наприклад доступ до мережі, доступ до нативного хоста або облікові дані,
специфічні для провайдера.

## Модераційні утримання

Коли статичний сканер позначає завантажений skill як malicious, видавця
автоматично переводять під модераційне утримання (`requiresModerationAt` встановлюється для
користувача). Це приховує всі Skills видавця, призводить до того, що майбутні публікації
починаються прихованими, і створює запис журналу аудиту `user.moderation.auto`.

Статичні підозрілі висновки зберігаються як докази файл/рядок для модераторів,
але самі по собі вони не приховують вміст і не визначають публічний вердикт сканування.
Нові завантаження залишаються у стані review/pending, доки не завершиться перевірка LLM. Статичне
сканування блокує негайно лише за malicious signatures. Спрацювання рушія VirusTotal
залишаються видимими доказами безпеки, але вердикти VirusTotal Code Insight/Palm
є рекомендаційними й самі по собі не приховують Skills. Огляди ClawScan LLM
зберігають нотатки, узгоджені з призначенням, як настанови. Висновки середнього рівня перевірки залишаються видимими на
артефакті, тоді як suspicious filter зарезервовано для високовпливових занепокоєнь LLM,
malicious findings або підтверджених виявлень AV-engine.

Адміністратори можуть зняти хибнопозитивне утримання:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

Це очищає `requiresModerationAt` і `requiresModerationReason`, відновлює
Skills, приховані утриманням на рівні користувача, і записує запис журналу аудиту `user.moderation.lift`.
Skills, приховані з інших причин або ті, чий власний статичний скан залишається
malicious, залишаються прихованими.

## Заборони та стан облікового запису

Облікові записи, що порушують політику ClawHub, можуть втратити доступ до публікації. Серйозні зловживання
можуть призвести до заборони облікового запису, відкликання токенів, приховання вмісту або видалення
оголошень.

Видалені, забанені або вимкнені облікові записи не можуть використовувати токени API ClawHub. Якщо автентифікація CLI
починає не вдаватися після дій щодо облікового запису, увійдіть у вебінтерфейс, щоб переглянути
стан облікового запису. Якщо вхід або звичайний доступ CLI заблоковано, зверніться до
security@openclaw.ai для перевірки відновлення.

## Настанови для видавців

Щоб зменшити хибні спрацювання та підвищити довіру користувачів:

- підтримуйте точність назв, коротких описів, тегів і журналів змін
- декларуйте потрібні змінні середовища та дозволи
- додавайте примітку ClawScan для видавця, коли реліз має незвичну, але навмисну поведінку
- уникайте обфускованих команд встановлення
- додавайте посилання на джерело, коли можливо
- використовуйте пробні запуски перед публікацією plugins
- відповідайте чітко, якщо користувачі або модератори запитують про поведінку пакета
