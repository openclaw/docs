# Безпека

Про проблеми безпеки ClawHub можна повідомляти через GitHub Security Advisories для
`openclaw/clawhub`.

Використовуйте GitHub Security Advisories для вразливостей у самому ClawHub. Добрі
звіти advisory для ClawHub охоплюють помилки в:

- вебсайті, API або CLI ClawHub
- публікації в реєстрі, завантаженнях, інсталяціях або цілісності артефактів
- автентифікації, авторизації або токенах API
- скануванні, модерації або обробці звітів

Не використовуйте ClawHub advisories для вразливостей у власному вихідному коді сторонньої навички або плагіна. Повідомляйте про них безпосередньо видавцю або до вихідного
репозиторію, посилання на який наведено в списку ClawHub.

## Розкриття вразливостей

Оскільки ClawHub є хмарною застосунком, що хоститься, вразливості сервісу ClawHub
за замовчуванням не розкриваються публічно. Їх розкривають публічно, коли є
докази реального впливу на користувачів або коли користувачам потрібно вжити дій.

Приклади реального впливу на користувачів включають підтверджену експлуатацію, розкриття даних або секретів користувачів, потрапляння шкідливого вмісту до користувачів через збій платформи
або будь-яку проблему, що вимагає від користувачів ротації облікових даних, оновлення локального програмного забезпечення чи
інших захисних дій.

Вразливості у встановленому користувачами програмному забезпеченні розкриваються публічно, наприклад
пакети ClawHub CLI, бінарні файли, бібліотеки або інші артефакти релізу, які користувачам
потрібно оновити локально.
