# Безпека + модерація

ClawHub відкритий для публікації, але публічні списки все одно проходять через засоби контролю довіри,
сканування, звітування та модерації. Мета практична: допомогти користувачам
перевіряти те, що вони встановлюють, надати видавцям шлях відновлення після хибних спрацювань
і не допускати зловживальних пакетів до публічного пошуку.

Дивіться також [Прийнятне використання](/uk/clawhub/acceptable-usage).

## Що користувачі можуть перевірити

Перед встановленням skill або plugin перевірте його список у ClawHub на наявність:

- власника та атрибуції джерела
- останньої версії та журналу змін
- обов’язкових змінних середовища або дозволів
- метаданих сумісності для plugins
- статусу сканування або модерації
- звітів, коментарів, зірок, завантажень і сигналів встановлення, де вони показані

Встановлюйте лише той вміст, який ви розумієте й якому довіряєте.

## Стани сканування

ClawHub може показувати результати сканування або модерації на публічних сторінках і в діагностиці,
видимій власнику.

Поширені результати включають:

- `clean`: блокувальних проблем не знайдено.
- `suspicious`: реліз потребує обережності або перевірки.
- `malicious`: реліз вважається небезпечним.
- `pending`: перевірки ще не завершені.
- `held`, `quarantined`, `revoked` або `hidden`: реліз не повністю
  доступний на публічних поверхнях встановлення.

Точне формулювання може відрізнятися залежно від поверхні, але практичне значення однакове: якщо
реліз утримано або заблоковано, користувачам не слід встановлювати його, доки власник не усуне
проблему або модерація не відновить його.

## Skills

Сканування skill перевіряє опублікований пакет skill, метадані, заявлені
вимоги та підозрілі інструкції.

ClawHub приділяє особливу увагу невідповідностям між тим, що skill заявляє, і
тим, що він, схоже, робить. Наприклад, skill, який посилається на обов’язковий ключ API,
має задекларувати цю вимогу в `SKILL.md`, щоб користувачі могли побачити її перед
встановленням.

Результати сканування ґрунтуються на артефактах. Очікувана поведінка провайдера, як-от заявлені
облікові дані API, localhost OAuth callbacks, scoped uninstall cleanup, кодування Basic Auth
або вибрані користувачем завантаження файлів до заявленого провайдера, розглядається
інакше, ніж приховане пересилання облікових даних, широкий доступ до приватних файлів,
непов’язані мережеві призначення або приховане зловживання браузером.

Дивіться [Формат skill](/uk/clawhub/skill-format).

## Plugins

Релізи Plugin включають метадані пакета, атрибуцію джерела, поля сумісності
та інформацію про цілісність артефакту.

OpenClaw перевіряє сумісність перед встановленням plugins, розміщених у ClawHub. Записи пакетів
також можуть надавати метадані digest, щоб OpenClaw міг перевіряти завантажені
артефакти. ClawScan включає заявлені метадані env/config пакета `openclaw.environment`
під час перевірки релізів Plugin, щоб заявлені вимоги часу виконання
порівнювалися зі спостережуваною поведінкою.

## Звіти

Користувачі, які ввійшли в систему, можуть повідомляти про skills, пакети та коментарі.

Звіти мають бути конкретними й придатними до дії. Зловживання звітуванням саме може призвести до
дій щодо облікового запису.

Приклади звітів:

- оманливі метадані
- незадекларовані вимоги до облікових даних або дозволів
- підозрілі інструкції зі встановлення
- шахрайські коментарі або видавання себе за іншу особу
- недобросовісні реєстрації або зловживання торговельною маркою
- вміст, що порушує [Прийнятне використання](/uk/clawhub/acceptable-usage)

## Нотатки ClawScan від видавця

Видавці можуть надати необов’язкову нотатку ClawScan під час публікації skill або
plugin. Ця нотатка надає ClawScan контекст для поведінки, яка інакше може виглядати
незвично, як-от мережевий доступ, доступ до нативного хоста або специфічні для провайдера
облікові дані.

## Утримання модерацією

Коли статичний сканер позначає завантажений skill як зловмисний, видавець
автоматично поміщається під утримання модерацією (`requiresModerationAt`, встановлений для
користувача). Це приховує всі skills видавця, змушує майбутні публікації
починатися прихованими та створює запис журналу аудиту `user.moderation.auto`.

Статичні підозрілі знахідки зберігаються як докази файл/рядок для модераторів,
але самі по собі вони не приховують вміст і не визначають публічний вердикт сканування.
Нові завантаження залишаються в стані review/pending, доки перевірка LLM не завершиться. Статичне
сканування блокує негайно лише за зловмисними сигнатурами. Спрацювання рушія
VirusTotal залишаються видимими доказами безпеки, але вердикти VirusTotal Code Insight/Palm
є дорадчими й самі по собі не приховують skills. Огляди ClawScan LLM
зберігають нотатки, узгоджені з призначенням, як рекомендації. Середні знахідки перевірки залишаються видимими на
артефакті, тоді як фільтр підозрілості зарезервований для значних проблем LLM,
зловмисних знахідок або підтверджених виявлень AV-engine.

Адміністратори можуть зняти хибнопозитивне утримання:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

Це очищає `requiresModerationAt` і `requiresModerationReason`, відновлює
skills, приховані утриманням на рівні користувача, і записує запис журналу аудиту
`user.moderation.lift`. Skills, приховані з інших причин, або ті, чий власний статичний скан залишається
зловмисним, залишаються прихованими.

## Блокування та стан облікового запису

Облікові записи, що порушують політику ClawHub, можуть втратити доступ до публікації. Серйозні зловживання
можуть призвести до блокування облікового запису, відкликання токенів, прихованого вмісту або вилучених
списків.

Видалені, заблоковані або вимкнені облікові записи не можуть використовувати токени API ClawHub. Якщо CLI auth
починає давати збої після дії щодо облікового запису, увійдіть у вебінтерфейс, щоб переглянути стан
облікового запису. Якщо вхід або звичайний доступ CLI заблоковано, зверніться до
security@openclaw.ai для перевірки відновлення.

## Рекомендації для видавців

Щоб зменшити кількість хибних спрацювань і підвищити довіру користувачів:

- підтримуйте точність назв, коротких описів, тегів і журналів змін
- декларуйте обов’язкові змінні середовища та дозволи
- додавайте нотатку ClawScan від видавця, коли реліз має незвичну, але навмисну поведінку
- уникайте обфускованих команд встановлення
- посилайтеся на джерело, коли це можливо
- використовуйте dry runs перед публікацією plugins
- відповідайте чітко, якщо користувачі або модератори запитують про поведінку пакета
