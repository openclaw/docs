# Безпека + модерація

ClawHub відкритий до публікації, але публічні списки все одно проходять через засоби контролю довіри,
сканування, звітування та модерації. Мета практична: допомогти користувачам
перевіряти те, що вони встановлюють, надати видавцям шлях відновлення після хибних спрацювань
і не допускати зловживальних пакетів до публічного виявлення.

Див. також [Допустиме використання](/uk/clawhub/acceptable-usage).

## Що користувачі можуть перевірити

Перед установленням skill або plugin перевірте його список у ClawHub на наявність:

- власника й атрибуції джерела
- останньої версії та журналу змін
- потрібних змінних середовища або дозволів
- метаданих сумісності для plugins
- стану сканування або модерації
- звітів, коментарів, зірок, завантажень і сигналів установлення, де їх показано

Встановлюйте лише той вміст, який ви розумієте й якому довіряєте.

## Стани сканування

ClawHub може показувати результати сканування або модерації на публічних сторінках і в діагностиці,
видимій власнику.

Поширені результати включають:

- `clean`: блокувальної проблеми не знайдено.
- `suspicious`: реліз потребує обережності або перевірки.
- `malicious`: реліз вважається небезпечним.
- `pending`: перевірки ще не завершено.
- `held`, `quarantined`, `revoked` або `hidden`: реліз не повністю
  доступний на публічних поверхнях установлення.

Точне формулювання може відрізнятися залежно від поверхні, але практичний зміст той самий: якщо
реліз утримано або заблоковано, користувачам не слід установлювати його, доки власник не вирішить
проблему або модерація не відновить його.

## Skills

Сканування skill перевіряє опублікований пакет skill, метадані, заявлені
вимоги та підозрілі інструкції.

ClawHub приділяє особливу увагу невідповідностям між тим, що skill декларує, і
тим, що він, схоже, робить. Наприклад, skill, який посилається на потрібний ключ API,
має задекларувати цю вимогу в `SKILL.md`, щоб користувачі могли побачити її перед
установленням.

Знахідки сканування базуються на артефактах. Очікувана поведінка провайдера, як-от задекларовані
облікові дані API, localhost OAuth callback, scoped uninstall cleanup, Basic Auth
encoding або вибрані користувачем завантаження файлів до заявленого провайдера, розглядається
інакше, ніж приховане пересилання облікових даних, широкий доступ до приватних файлів,
непов’язані мережеві призначення або приховане зловживання браузером.

Див. [Формат skill](/uk/clawhub/skill-format).

## Plugins

Релізи Plugin містять метадані пакета, атрибуцію джерела, поля сумісності
та інформацію про цілісність артефакта.

OpenClaw перевіряє сумісність перед установленням plugins, розміщених у ClawHub. Записи пакетів
також можуть надавати метадані digest, щоб OpenClaw міг перевіряти завантажені
артефакти. ClawScan включає заявлені метадані env/config пакета `openclaw.environment`
під час перевірки релізів plugin, щоб заявлені вимоги runtime порівнювалися
зі спостережуваною поведінкою.

## Звіти

Користувачі, які ввійшли в систему, можуть повідомляти про skills, пакети та коментарі.

Звіти мають бути конкретними й такими, за якими можна діяти. Зловживання звітуванням саме по собі може призвести до
дій щодо облікового запису.

Приклади звітів:

- оманливі метадані
- незадекларовані вимоги до облікових даних або дозволів
- підозрілі інструкції з установлення
- шахрайські коментарі або видавання себе за іншу особу
- недобросовісні реєстрації або неправомірне використання торговельної марки
- вміст, що порушує [Допустиме використання](/uk/clawhub/acceptable-usage)

## Нотатки ClawScan від видавця

Видавці можуть надати необов’язкову нотатку ClawScan під час публікації skill або
plugin. Ця нотатка дає ClawScan контекст для поведінки, яка інакше може виглядати
незвично, як-от доступ до мережі, доступ до native host або облікові дані,
специфічні для провайдера.

## Модераційні утримання

Коли статичний сканер позначає завантажений skill як шкідливий, видавця
автоматично переводять у стан модераційного утримання (`requiresModerationAt` встановлюється для
користувача). Це приховує всі skills видавця, змушує майбутні публікації
починатися прихованими та створює запис журналу аудиту `user.moderation.auto`.

Статичні підозрілі знахідки зберігаються як докази файл/рядок для модераторів,
але самі по собі вони не приховують вміст і не визначають публічний вердикт сканування.
Нові завантаження залишаються у стані review/pending, доки не завершиться LLM-перевірка. Статичне
сканування негайно блокує лише за шкідливими signatures. Спрацювання рушіїв
VirusTotal залишаються видимими доказами безпеки, але вердикти VirusTotal Code Insight/Palm
є рекомендаційними й самі по собі не приховують skills. LLM-перевірки ClawScan
зберігають нотатки, узгоджені з призначенням, як настанови. Середні знахідки перевірки залишаються видимими на
артефакті, тоді як фільтр підозрілості зарезервований для LLM-занепокоєнь із великим впливом,
шкідливих знахідок або підтверджених виявлень AV-рушіями.

Адміністратори можуть зняти хибнопозитивне утримання:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

Це очищає `requiresModerationAt` і `requiresModerationReason`, відновлює
skills, приховані утриманням на рівні користувача, і записує запис журналу аудиту `user.moderation.lift`.
Skills, приховані з інших причин, або ті, чий власний статичний скан залишається
шкідливим, залишаються прихованими.

## Блокування та стан облікового запису

Облікові записи, які порушують політику ClawHub, можуть втратити доступ до публікації. Серйозні зловживання
можуть призвести до блокування облікового запису, відкликання токенів, приховання вмісту або видалення
списків.

Видалені, заблоковані або вимкнені облікові записи не можуть використовувати API-токени ClawHub. Якщо автентифікація CLI
починає давати збої після дії щодо облікового запису, увійдіть у вебінтерфейс, щоб переглянути стан
облікового запису. Якщо вхід або звичайний доступ CLI заблоковано, зверніться до
security@openclaw.ai для перевірки відновлення.

## Настанови для видавців

Щоб зменшити хибні спрацювання та підвищити довіру користувачів:

- підтримуйте точність назв, резюме, тегів і журналів змін
- декларуйте потрібні змінні середовища та дозволи
- додайте нотатку ClawScan від видавця, якщо реліз має незвичну, але навмисну поведінку
- уникайте обфускованих команд установлення
- посилайтеся на джерело, коли це можливо
- використовуйте пробні запуски перед публікацією plugins
- відповідайте чітко, якщо користувачі або модератори запитують про поведінку пакета
