# Безпека

Про проблеми безпеки ClawHub можна повідомляти через GitHub Security Advisories для
`openclaw/clawhub`.

Використовуйте GitHub Security Advisories для вразливостей у самому ClawHub. Якісні
звіти advisory для ClawHub охоплюють помилки в:

- вебсайті, API або CLI ClawHub
- публікації в реєстрі, завантаженнях, установленнях або цілісності артефактів
- автентифікації, авторизації або токенах API
- скануванні, модерації або обробці звітів

Не використовуйте advisories ClawHub для вразливостей у власному вихідному коді стороннього skill або
plugin. Повідомляйте про них безпосередньо видавцю або в репозиторій вихідного коду,
посилання на який наведено в записі ClawHub.

## Розкриття вразливостей

Оскільки ClawHub є розміщеною хмарною програмою, вразливості сервісу ClawHub
типово не розкриваються публічно. Їх розкривають публічно, коли є
докази реального впливу на користувачів або коли користувачам потрібно вжити дій.

Приклади реального впливу на користувачів включають підтверджену експлуатацію, розкриття даних
або секретів користувачів, потрапляння шкідливого вмісту до користувачів через збій платформи
або будь-яку проблему, яка вимагає від користувачів змінити облікові дані, оновити локальне програмне забезпечення чи
вжити інших захисних заходів.

Вразливості у встановленому користувачем програмному забезпеченні розкриваються публічно, зокрема в
пакетах CLI ClawHub, бінарних файлах, бібліотеках або інших артефактах релізу, які користувачам
потрібно оновити локально.

## Пов’язані сторінки

Про мітки аудиту під час установлення, рівні ризику, знахідки та інтерпретацію див.
[Аудити безпеки](/clawhub/security-audits).

Про звіти marketplace, модераційні блокування, приховані записи, заборони та
стан облікового запису див. [Модерація та безпека облікового запису](/uk/clawhub/moderation).
