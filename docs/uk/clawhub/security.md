# Безпека

Про проблеми безпеки ClawHub можна повідомляти через GitHub Security Advisories для
`openclaw/clawhub`.

Використовуйте GitHub Security Advisories для вразливостей у самому ClawHub. Якісні
звіти advisory для ClawHub містять помилки в:

- вебсайті, API або CLI ClawHub
- публікації в реєстрі, завантаженнях, інсталяціях або цілісності артефактів
- автентифікації, авторизації або API-токенах
- скануванні, модерації або обробці звітів

Не використовуйте advisory ClawHub для вразливостей у власному вихідному коді стороннього skill або
плагіна. Повідомляйте про них безпосередньо видавцю або у вихідний
репозиторій, посилання на який наведено в записі ClawHub.

## Розкриття вразливостей

Оскільки ClawHub є хмарним застосунком, що хоститься, вразливості сервісу ClawHub
типово не розкриваються публічно. Вони розкриваються публічно, коли є
докази реального впливу на користувачів або коли користувачам потрібно вжити дій.

Приклади реального впливу на користувачів включають підтверджену експлуатацію, розкриття даних
користувачів або секретів, потрапляння шкідливого вмісту до користувачів через збій платформи
або будь-яку проблему, що вимагає від користувачів змінити облікові дані, оновити локальне програмне забезпечення чи
вжити інших захисних заходів.

Вразливості в установленому користувачами програмному забезпеченні розкриваються публічно, наприклад
пакети CLI ClawHub, бінарні файли, бібліотеки або інші артефакти релізів, які користувачам
потрібно оновити локально.

## Пов’язані сторінки

Про мітки аудиту під час інсталяції, рівні ризику, виявлені проблеми та їх інтерпретацію див.
[Аудити безпеки](/clawhub/security-audits).

Про звіти marketplace, модераційні утримання, приховані записи, блокування та статус
облікового запису див. [Модерація та безпека облікового запису](/clawhub/moderation).
