# Безпека + модерація

ClawHub відкритий для публікації, але публічні списки все одно проходять перевірки довіри,
сканування, звітування й модерації. Мета практична: допомогти користувачам
перевіряти, що вони встановлюють, дати видавцям шлях відновлення після хибних спрацювань
і не допускати зловживальних пакетів до публічного виявлення.

Див. також [Прийнятне використання](/uk/clawhub/acceptable-usage).

## Що можуть перевірити користувачі

Перед встановленням Skills або плагіна перевірте його сторінку в ClawHub на наявність:

- власника й атрибуції джерела
- останньої версії та журналу змін
- потрібних змінних середовища або дозволів
- метаданих сумісності для плагінів
- статусу сканування або модерації
- звітів, коментарів, зірок, завантажень і сигналів встановлення, якщо вони показані

Встановлюйте лише той вміст, який ви розумієте й якому довіряєте.

## Стани сканування

ClawHub може показувати результати сканування або модерації на публічних сторінках і в
діагностиці, видимій власнику.

Поширені результати:

- `clean`: блокувальної проблеми не знайдено.
- `suspicious`: реліз потребує обережності або перевірки.
- `malicious`: реліз вважається небезпечним.
- `pending`: перевірки ще не завершено.
- `held`, `quarantined`, `revoked` або `hidden`: реліз не повністю
  доступний на публічних поверхнях встановлення.

Точне формулювання може відрізнятися залежно від поверхні, але практичне значення те саме: якщо
реліз утримано або заблоковано, користувачам не слід встановлювати його, доки власник не усуне
проблему або модерація не відновить його.

## Skills

Сканування Skills перевіряє опублікований пакет Skills, метадані, заявлені
вимоги та підозрілі інструкції.

ClawHub приділяє особливу увагу невідповідностям між тим, що Skills заявляє, і
тим, що він, схоже, робить. Наприклад, Skills, який посилається на обов’язковий API-ключ,
має заявити цю вимогу в `SKILL.md`, щоб користувачі могли побачити її перед
встановленням.

Знахідки сканування базуються на артефактах. Очікувана поведінка провайдера, як-от заявлені
облікові дані API, OAuth-зворотні виклики localhost, очищення під час видалення в межах області,
кодування Basic Auth або вибрані користувачем завантаження файлів до заявленого провайдера, розглядається
інакше, ніж приховане пересилання облікових даних, широкий доступ до приватних файлів,
непов’язані мережеві призначення або приховане зловживання браузером.

Див. [Формат Skills](/uk/clawhub/skill-format).

## Плагіни

Релізи плагінів містять метадані пакета, атрибуцію джерела, поля сумісності
та інформацію про цілісність артефактів.

OpenClaw перевіряє сумісність перед встановленням плагінів, розміщених у ClawHub. Записи пакетів
також можуть містити метадані дайджесту, щоб OpenClaw міг перевіряти завантажені
артефакти. ClawScan включає заявлені метадані env/config пакета `openclaw.environment`
під час перевірки релізів плагінів, щоб заявлені вимоги часу виконання
порівнювалися зі спостережуваною поведінкою.

## Звіти

Користувачі, які ввійшли в систему, можуть повідомляти про Skills, пакети й коментарі.

Звіти мають бути конкретними й придатними до дії. Зловживання звітуванням саме по собі може призвести до
дій щодо облікового запису.

Приклади звітів:

- оманливі метадані
- незаявлені вимоги до облікових даних або дозволів
- підозрілі інструкції зі встановлення
- шахрайські коментарі або видавання себе за іншу особу
- недобросовісні реєстрації або неправомірне використання торговельних марок
- вміст, що порушує [Прийнятне використання](/uk/clawhub/acceptable-usage)

## Примітки ClawScan для видавців

Видавці можуть надати необов’язкову примітку ClawScan під час публікації Skills або
плагіна. Ця примітка дає ClawScan контекст для поведінки, яка інакше може виглядати
незвичною, як-от мережевий доступ, доступ до нативного хоста або облікові дані,
специфічні для провайдера.

## Модераційні утримання

Коли статичний сканер позначає завантажений Skills як шкідливий, видавця
автоматично ставлять під модераційне утримання (`requiresModerationAt`, задане для
користувача). Це приховує всі Skills видавця, змушує майбутні публікації
починатися прихованими та створює запис журналу аудиту `user.moderation.auto`.

Статичні підозрілі знахідки зберігаються як докази файл/рядок для модераторів,
але самі по собі вони не приховують вміст і не визначають публічний вердикт сканування.
Нові завантаження залишаються в стані перевірки/очікування, доки LLM-перевірка не завершиться. Статичне
сканування блокує негайно лише за шкідливими сигнатурами. Спрацювання рушіїв
VirusTotal залишаються видимими доказами безпеки, але вердикти VirusTotal Code Insight/Palm
є рекомендаційними й самі по собі не приховують Skills. LLM-перевірки ClawScan
зберігають примітки, узгоджені з призначенням, як настанови. Знахідки середнього рівня перевірки залишаються видимими на
артефакті, тоді як фільтр підозрілості зарезервований для високовпливових
занепокоєнь LLM, шкідливих знахідок або підтверджених виявлень AV-рушіїв.

Адміністратори можуть зняти утримання через хибне спрацювання:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

Це очищує `requiresModerationAt` і `requiresModerationReason`, відновлює
Skills, приховані утриманням на рівні користувача, і записує запис журналу аудиту `user.moderation.lift`.
Skills, приховані з інших причин, або ті, чий власний статичний скан залишається
шкідливим, залишаються прихованими.

## Блокування й стан облікового запису

Облікові записи, що порушують політику ClawHub, можуть втратити доступ до публікації. Серйозні зловживання
можуть призвести до блокування облікового запису, відкликання токенів, прихованого вмісту або видалених
списків.

Видалені, заблоковані або вимкнені облікові записи не можуть використовувати API-токени ClawHub. Якщо автентифікація CLI
починає не працювати після дії щодо облікового запису, увійдіть у вебінтерфейс, щоб переглянути стан
облікового запису. Якщо вхід або звичайний доступ CLI заблоковано, зверніться на
security@openclaw.ai для перевірки відновлення.

## Настанови для видавців

Щоб зменшити кількість хибних спрацювань і підвищити довіру користувачів:

- підтримуйте точність назв, коротких описів, тегів і журналів змін
- заявляйте потрібні змінні середовища й дозволи
- додавайте примітку ClawScan видавця, коли реліз має незвичну, але навмисну поведінку
- уникайте обфускованих команд встановлення
- додавайте посилання на джерело, коли можливо
- використовуйте пробні запуски перед публікацією плагінів
- відповідайте чітко, якщо користувачі або модератори запитують про поведінку пакета
