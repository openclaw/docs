# Безпека + модерація

ClawHub відкритий до публікації, але публічні списки все одно проходять через контроль довіри,
сканування, звітування та модерації. Мета практична: допомогти користувачам
перевіряти те, що вони встановлюють, дати видавцям шлях відновлення після хибних спрацювань
і не допускати зловмисні пакети до публічного пошуку.

Див. також [Прийнятне використання](/uk/clawhub/acceptable-usage).

## Що користувачі можуть перевірити

Перед встановленням навички або плагіна перевірте її список у ClawHub на:

- власника та атрибуцію джерела
- останню версію та журнал змін
- необхідні змінні середовища або дозволи
- метадані сумісності для плагінів
- стан сканування або модерації
- звіти, коментарі, зірки, завантаження та сигнали встановлення, якщо вони показані

Встановлюйте лише вміст, який ви розумієте й якому довіряєте.

## Стани сканування

ClawHub може показувати результати сканування або модерації на публічних сторінках і в
діагностиці, видимій власнику.

Поширені результати включають:

- `clean`: блокувальної проблеми не знайдено.
- `suspicious`: реліз потребує обережності або перевірки.
- `malicious`: реліз вважається небезпечним.
- `pending`: перевірки ще не завершено.
- `held`, `quarantined`, `revoked` або `hidden`: реліз не повністю
  доступний на публічних поверхнях встановлення.

Точне формулювання може різнитися залежно від поверхні, але практичне значення те саме: якщо
реліз утримано або заблоковано, користувачам не слід встановлювати його, доки власник не розв'яже
проблему або модерація не відновить його.

## Skills

Сканування навичок перевіряє опублікований пакет навички, метадані, оголошені
вимоги та підозрілі інструкції.

ClawHub приділяє особливу увагу невідповідностям між тим, що навичка декларує, і
тим, що вона, схоже, робить. Наприклад, навичка, яка посилається на обов'язковий API-ключ,
має оголосити цю вимогу в `SKILL.md`, щоб користувачі могли побачити її перед
встановленням.

Результати сканування базуються на артефактах. Очікувана поведінка провайдера, як-от оголошені
облікові дані API, OAuth-зворотні виклики localhost, очищення під час видалення в межах області дії, кодування Basic Auth
або вибрані користувачем завантаження файлів до вказаного провайдера, обробляється
інакше, ніж приховане пересилання облікових даних, широкий доступ до приватних файлів,
непов'язані мережеві призначення або приховане зловживання браузером.

Див. [Формат навички](/uk/clawhub/skill-format).

## Плагіни

Релізи плагінів включають метадані пакета, атрибуцію джерела, поля
сумісності та інформацію про цілісність артефактів.

OpenClaw перевіряє сумісність перед встановленням плагінів, розміщених у ClawHub. Записи пакетів
також можуть надавати метадані digest, щоб OpenClaw міг перевіряти завантажені
артефакти. ClawScan включає оголошені метадані env/config пакета `openclaw.environment`
під час перевірки релізів плагінів, щоб оголошені runtime-вимоги
порівнювалися зі спостережуваною поведінкою.

## Звіти

Користувачі, які ввійшли в систему, можуть повідомляти про навички, пакети та коментарі.

Звіти мають бути конкретними й придатними до дії. Зловживання звітуванням саме по собі може призвести до
дій щодо облікового запису.

Приклади звітів:

- оманливі метадані
- неоголошені вимоги до облікових даних або дозволів
- підозрілі інструкції зі встановлення
- шахрайські коментарі або видавання себе за іншу особу
- недобросовісні реєстрації або зловживання торговельною маркою
- вміст, що порушує [Прийнятне використання](/uk/clawhub/acceptable-usage)

## Примітки ClawScan від видавця

Видавці можуть надати необов'язкову примітку ClawScan під час публікації навички або
плагіна. Ця примітка дає ClawScan контекст щодо поведінки, яка інакше може виглядати
незвично, наприклад мережевого доступу, доступу до нативного хоста або специфічних для провайдера
облікових даних.

## Утримання модерацією

Коли статичний сканер позначає завантажену навичку як зловмисну, видавця
автоматично поміщають під утримання модерацією (`requiresModerationAt` встановлюється для
користувача). Це приховує всі навички видавця, змушує майбутні публікації
починатися прихованими та створює запис журналу аудиту `user.moderation.auto`.

Статичні підозрілі знахідки зберігаються як докази файл/рядок для модераторів,
але самі по собі вони не приховують вміст і не визначають публічний вердикт сканування.
Нові завантаження залишаються у стані review/pending, доки не завершиться LLM-перевірка. Статичне
сканування негайно блокує лише за зловмисними сигнатурами. Спрацювання рушія
VirusTotal залишаються видимими доказами безпеки, але вердикти VirusTotal Code Insight/Palm
є рекомендаційними й самі по собі не приховують навички. LLM-перевірки ClawScan
зберігають узгоджені з призначенням примітки як настанови. Знахідки середнього рівня перевірки залишаються видимими на
артефакті, тоді як підозрілий фільтр зарезервовано для високовпливових LLM-
занепокоєнь, зловмисних знахідок або підтверджених виявлень AV-рушіями.

Адміністратори можуть зняти утримання через хибне спрацювання:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

Це очищає `requiresModerationAt` і `requiresModerationReason`, відновлює
навички, приховані утриманням на рівні користувача, і записує запис журналу аудиту `user.moderation.lift`.
Навички, приховані з інших причин, або ті, власне статичне сканування яких усе ще
є зловмисним, залишаються прихованими.

## Блокування та стан облікового запису

Облікові записи, що порушують політику ClawHub, можуть втратити доступ до публікації. Серйозні зловживання
можуть призвести до блокування облікового запису, відкликання токена, приховання вмісту або видалення
списків.

Видалені, заблоковані або вимкнені облікові записи не можуть використовувати API-токени ClawHub. Якщо автентифікація CLI
починає давати збій після дій щодо облікового запису, увійдіть у веб UI, щоб переглянути стан
облікового запису. Якщо вхід або звичайний доступ CLI заблоковано, зверніться до
security@openclaw.ai для перевірки відновлення.

## Настанови для видавців

Щоб зменшити хибні спрацювання та підвищити довіру користувачів:

- підтримуйте точність назв, резюме, тегів і журналів змін
- оголошуйте необхідні змінні середовища та дозволи
- додайте примітку ClawScan від видавця, коли реліз має незвичну, але навмисну поведінку
- уникайте обфускованих команд встановлення
- посилайтеся на джерело, коли це можливо
- використовуйте пробні запуски перед публікацією плагінів
- відповідайте чітко, якщо користувачі або модератори запитують про поведінку пакета
