# Безпека

Про проблеми безпеки ClawHub можна повідомляти через GitHub Security Advisories для
`openclaw/clawhub`.

Використовуйте GitHub Security Advisories для вразливостей у самому ClawHub. Якісні
звіти advisory для ClawHub охоплюють помилки в:

- вебсайті, API або CLI ClawHub
- публікації в реєстрі, завантаженнях, інсталяціях або цілісності артефактів
- автентифікації, авторизації або API-токенах
- скануванні, модерації або обробці звітів

Не використовуйте advisory ClawHub для вразливостей у власному вихідному коді стороннього skill або
plugin. Повідомляйте про них безпосередньо видавцю або в репозиторій вихідного коду,
посилання на який наведено в записі ClawHub.

## Розкриття вразливостей

Оскільки ClawHub є хмарним застосунком із хостингом, вразливості сервісу ClawHub
за замовчуванням не розкриваються публічно. Їх розкривають публічно, коли є
докази реального впливу на користувачів або коли користувачам потрібно вжити заходів.

Приклади реального впливу на користувачів включають підтверджену експлуатацію, розкриття даних
або секретів користувачів, потрапляння шкідливого вмісту до користувачів через збій платформи
або будь-яку проблему, що вимагає від користувачів ротації облікових даних, оновлення локального програмного забезпечення або
вжиття інших захисних заходів.

Вразливості в програмному забезпеченні, встановленому користувачами, розкриваються публічно, зокрема
пакети CLI ClawHub, бінарні файли, бібліотеки або інші релізні артефакти, які користувачам
потрібно оновити локально.

## Пов’язані сторінки

Про audit labels під час інсталяції, рівні ризику, findings та інтерпретацію див.
[Security Audits](/clawhub/security-audits).

Про звіти marketplace, утримання на модерації, приховані записи, блокування та стан облікового запису
див. [Moderation and Account Safety](/clawhub/moderation).
