# Безпека

Про проблеми безпеки ClawHub можна повідомляти через GitHub Security Advisories для
`openclaw/clawhub`.

Використовуйте GitHub Security Advisories для вразливостей у самому ClawHub. Якісні
звіти ClawHub advisory охоплюють помилки в:

- вебсайті ClawHub, API або CLI
- публікації в реєстрі, завантаженнях, встановленнях або цілісності артефактів
- автентифікації, авторизації або токенах API
- скануванні, модерації або обробці звітів

Не використовуйте ClawHub advisories для вразливостей у власному вихідному коді стороннього skill або
plugin. Повідомляйте про них безпосередньо видавцю або в репозиторій вихідного коду,
посилання на який наведено в записі ClawHub.

## Розкриття вразливостей

Оскільки ClawHub є розміщеним у хмарі застосунком, вразливості сервісу ClawHub
за замовчуванням не розкриваються публічно. Їх розкривають публічно, коли є
докази реального впливу на користувачів або коли користувачам потрібно вжити дій.

Приклади реального впливу на користувачів включають підтверджену експлуатацію, розкриття користувацьких
даних або секретів, потрапляння шкідливого вмісту до користувачів через збій платформи
або будь-яку проблему, яка вимагає від користувачів ротації облікових даних, оновлення локального програмного забезпечення чи
вжиття інших захисних дій.

Вразливості в установленому користувачами програмному забезпеченні розкриваються публічно, зокрема
пакети ClawHub CLI, бінарні файли, бібліотеки або інші артефакти релізу, які користувачам
потрібно оновити локально.

## Пов’язані сторінки

Про мітки аудиту під час встановлення, рівні ризику, знахідки та тлумачення див.
[Аудити безпеки](/clawhub/security-audits).

Про звіти маркетплейсу, затримки модерації, приховані записи, блокування та стан
облікового запису див. [Модерація та безпека облікового запису](/clawhub/moderation).
