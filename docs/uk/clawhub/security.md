# Безпека

Про проблеми безпеки ClawHub можна повідомляти через GitHub Security Advisories для
`openclaw/clawhub`.

Використовуйте GitHub Security Advisories для вразливостей у самому ClawHub. Хороші
звіти про вразливості ClawHub містять помилки в:

- вебсайті, API або CLI ClawHub
- публікації в реєстрі, завантаженнях, встановленнях або цілісності артефактів
- автентифікації, авторизації або токенах API
- скануванні, модерації або обробці звітів

Не використовуйте advisories ClawHub для вразливостей у власному вихідному коді сторонньої навички або
Plugin. Повідомляйте про них безпосередньо видавцю або в репозиторій вихідного коду,
посилання на який є в записі ClawHub.

## Розкриття вразливостей

Оскільки ClawHub є хмарним застосунком, розміщеним як сервіс, вразливості сервісу ClawHub
за замовчуванням не розкриваються публічно. Їх розкривають публічно, коли є
докази реального впливу на користувачів або коли користувачам потрібно вжити дій.

Приклади реального впливу на користувачів включають підтверджену експлуатацію, розкриття
даних або секретів користувачів, потрапляння шкідливого вмісту до користувачів через збій платформи
або будь-яку проблему, що вимагає від користувачів ротації облікових даних, оновлення локального програмного забезпечення чи
вжиття інших захисних заходів.

Вразливості в програмному забезпеченні, встановленому користувачами, розкриваються публічно, наприклад у
пакетах ClawHub CLI, двійкових файлах, бібліотеках або інших артефактах випуску, які користувачам
потрібно оновити локально.

## Пов’язані сторінки

Про мітки аудиту під час встановлення, рівні ризику, знахідки та інтерпретацію див.
[Аудити безпеки](/clawhub/security-audits).

Про звіти маркетплейсу, модераційні утримання, приховані записи, блокування та стан
облікового запису див. [Модерація та безпека облікового запису](/clawhub/moderation).
