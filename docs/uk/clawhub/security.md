# Безпека

Про проблеми безпеки ClawHub можна повідомляти через GitHub Security Advisories для
`openclaw/clawhub`.

Використовуйте GitHub Security Advisories для вразливостей у самому ClawHub. Якісні
звіти advisory для ClawHub охоплюють помилки в:

- вебсайті, API або CLI ClawHub
- публікації в реєстрі, завантаженнях, установленнях або цілісності артефактів
- автентифікації, авторизації або API-токенах
- скануванні, модерації або обробці звітів

Не використовуйте advisories ClawHub для вразливостей у власному вихідному коді сторонньої навички або
Plugin. Повідомляйте про них безпосередньо видавцю або до вихідного
репозиторію, посилання на який наведено в записі ClawHub.

## Розкриття вразливостей

Оскільки ClawHub є розміщеним хмарним застосунком, вразливості сервісу ClawHub
за замовчуванням не розкриваються публічно. Вони розкриваються публічно, коли є
докази реального впливу на користувачів або коли користувачам потрібно вжити дій.

Приклади реального впливу на користувачів включають підтверджену експлуатацію, розкриття користувацьких
даних або секретів, потрапляння шкідливого вмісту до користувачів через збій платформи
або будь-яку проблему, що вимагає від користувачів ротації облікових даних, оновлення локального програмного забезпечення чи
вжиття інших захисних заходів.

Вразливості в установленому користувачами програмному забезпеченні розкриваються публічно, наприклад
пакети CLI ClawHub, бінарні файли, бібліотеки або інші артефакти релізу, які користувачам
потрібно оновити локально.

## Пов’язані сторінки

Про мітки аудиту під час установлення, рівні ризику, знахідки та інтерпретацію див.
[Аудити безпеки](/clawhub/security-audits).

Про звіти marketplace, утримання на модерації, приховані записи, блокування та
статус облікового запису див. [Модерація та безпека облікового запису](/clawhub/moderation).
