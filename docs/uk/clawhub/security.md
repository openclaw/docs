# Безпека + модерація

ClawHub відкритий для публікації, але публічні списки все одно проходять через
контроль довіри, сканування, звітності та модерації. Мета практична: допомогти
користувачам перевіряти те, що вони встановлюють, дати видавцям шлях відновлення
після хибних спрацьовувань і не допускати зловмисні пакети до публічного пошуку.

Див. також [Прийнятне використання](/uk/clawhub/acceptable-usage).

## Що можуть перевірити користувачі

Перед встановленням skill або плагіна перевірте його список у ClawHub на:

- власника та атрибуцію джерела
- останню версію та журнал змін
- потрібні змінні середовища або дозволи
- метадані сумісності для плагінів
- стан сканування або модерації
- звіти, коментарі, зірки, завантаження та сигнали встановлення, якщо вони показані

Встановлюйте лише той вміст, який ви розумієте і якому довіряєте.

## Стани сканування

ClawHub може показувати результати сканування або модерації на публічних сторінках
і в діагностиці, видимій власнику.

Поширені результати:

- `clean`: блокувальних проблем не знайдено.
- `suspicious`: реліз потребує обережності або перевірки.
- `malicious`: реліз вважається небезпечним.
- `pending`: перевірки ще не завершилися.
- `held`, `quarantined`, `revoked` або `hidden`: реліз не повністю доступний
  на публічних поверхнях встановлення.

Точне формулювання може різнитися залежно від поверхні, але практичне значення
те саме: якщо реліз утримано або заблоковано, користувачам не слід встановлювати
його, доки власник не вирішить проблему або модерація не відновить його.

## Skills

Сканування Skills перевіряє опублікований пакет skill, метадані, заявлені
вимоги та підозрілі інструкції.

ClawHub приділяє особливу увагу невідповідностям між тим, що skill заявляє, і
тим, що він, схоже, робить. Наприклад, skill, який посилається на потрібний
ключ API, має заявити цю вимогу в `SKILL.md`, щоб користувачі могли побачити її
перед встановленням.

Результати сканування базуються на артефактах. Очікувана поведінка провайдера,
як-от заявлені облікові дані API, зворотні виклики OAuth на localhost, очищення
під час видалення в межах області дії, кодування Basic Auth або вибрані
користувачем завантаження файлів до заявленого провайдера, розглядається інакше,
ніж приховане пересилання облікових даних, широкий доступ до приватних файлів,
непов’язані мережеві призначення або приховане зловживання браузером.

Див. [Формат Skill](/uk/clawhub/skill-format).

## Плагіни

Релізи плагінів містять метадані пакета, атрибуцію джерела, поля сумісності та
інформацію про цілісність артефактів.

OpenClaw перевіряє сумісність перед встановленням плагінів, розміщених у
ClawHub. Записи пакетів також можуть надавати метадані digest, щоб OpenClaw міг
перевіряти завантажені артефакти. ClawScan включає заявлені метадані env/config
пакета `openclaw.environment` під час перевірки релізів плагінів, щоб заявлені
вимоги до виконання порівнювалися зі спостережуваною поведінкою.

## Звіти

Користувачі, які ввійшли в систему, можуть повідомляти про skills, пакети та
коментарі.

Звіти мають бути конкретними й придатними до дії. Зловживання звітністю саме по
собі може призвести до дій щодо облікового запису.

Приклади звітів:

- оманливі метадані
- незаявлені вимоги до облікових даних або дозволів
- підозрілі інструкції зі встановлення
- шахрайські коментарі або видавання себе за інших
- недобросовісні реєстрації або неправомірне використання торговельних марок
- вміст, що порушує [Прийнятне використання](/uk/clawhub/acceptable-usage)

## Нотатки видавця для ClawScan

Видавці можуть надати необов’язкову нотатку ClawScan під час публікації skill
або плагіна. Ця нотатка дає ClawScan контекст для поведінки, яка інакше може
виглядати незвично, наприклад мережевий доступ, доступ до нативного хоста або
облікові дані, специфічні для провайдера.

## Модераційні утримання

Коли статичний сканер позначає завантажений skill як зловмисний, видавець
автоматично потрапляє під модераційне утримання (`requiresModerationAt`,
установлене для користувача). Це приховує всі skills видавця, призводить до
того, що майбутні публікації починаються прихованими, і створює запис журналу
аудиту `user.moderation.auto`.

Статичні підозрілі знахідки зберігаються як докази файл/рядок для модераторів,
але самі по собі вони не приховують вміст і не визначають публічний вердикт
сканування. Нові завантаження залишаються в стані перевірки/очікування, доки
LLM-перевірка не завершиться. Статичне сканування негайно блокує лише
зловмисні сигнатури. Спрацьовування рушіїв VirusTotal залишаються видимими
доказами безпеки, але вердикти VirusTotal Code Insight/Palm є рекомендаційними
і самі по собі не приховують skills. LLM-перевірки ClawScan зберігають
нотатки, узгоджені з призначенням, як настанови. Знахідки середньої важливості
залишаються видимими в артефакті, тоді як підозрілий фільтр зарезервований для
високовпливових зауважень LLM, зловмисних знахідок або підтверджених виявлень
AV-рушіями.

Адміністратори можуть зняти утримання через хибне спрацьовування:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

Це очищає `requiresModerationAt` і `requiresModerationReason`, відновлює
skills, приховані утриманням на рівні користувача, і записує запис журналу
аудиту `user.moderation.lift`. Skills, приховані з інших причин, або ті, чий
власний статичний скан залишається зловмисним, лишаються прихованими.

## Блокування та стан облікового запису

Облікові записи, що порушують політику ClawHub, можуть втратити доступ до
публікації. Серйозні зловживання можуть призвести до блокування облікового
запису, відкликання токенів, прихованого вмісту або видалених списків.

Видалені, заблоковані або вимкнені облікові записи не можуть використовувати
API-токени ClawHub. Якщо автентифікація CLI починає давати збої після дій щодо
облікового запису, увійдіть у вебінтерфейс, щоб переглянути стан облікового
запису. Якщо вхід або звичайний доступ CLI заблоковано, зверніться до
security@openclaw.ai для перевірки відновлення.

## Настанови для видавців

Щоб зменшити кількість хибних спрацьовувань і підвищити довіру користувачів:

- підтримуйте точність назв, коротких описів, тегів і журналів змін
- заявляйте потрібні змінні середовища та дозволи
- додавайте нотатку видавця для ClawScan, коли реліз має незвичну, але навмисну поведінку
- уникайте обфускованих команд встановлення
- посилайтеся на джерело, коли це можливо
- використовуйте пробні запуски перед публікацією плагінів
- відповідайте чітко, якщо користувачі або модератори запитують про поведінку пакета
