# Безпека

Про проблеми безпеки ClawHub можна повідомляти через GitHub Security Advisories для
`openclaw/clawhub`.

Використовуйте GitHub Security Advisories для вразливостей безпосередньо в ClawHub. Належні
звіти про вразливості ClawHub охоплюють помилки у:

- вебсайті, API або CLI ClawHub
- публікації в реєстрі, завантаженнях, установленнях або цілісності артефактів
- автентифікації, авторизації або токенах API
- скануванні, модерації або опрацюванні звітів

Не використовуйте сповіщення ClawHub для вразливостей у власному вихідному коді стороннього skill або
Plugin. Повідомляйте про них безпосередньо видавцю або в репозиторій вихідного коду,
посилання на який наведено в записі ClawHub.

## Розкриття інформації про вразливості

Оскільки ClawHub є хмарним застосунком із хостингом, вразливості сервісу ClawHub
за замовчуванням не розкриваються публічно. Їх розкривають публічно, коли є
докази реального впливу на користувачів або коли користувачам потрібно вжити заходів.

Приклади реального впливу на користувачів: підтверджена експлуатація, розкриття даних
користувачів або секретів, потрапляння шкідливого вмісту до користувачів через збій платформи
або будь-яка проблема, через яку користувачам потрібно змінити облікові дані, оновити локальне програмне забезпечення чи
вжити інших захисних заходів.

Вразливості в установленому користувачами програмному забезпеченні розкриваються публічно, зокрема в
пакетах CLI ClawHub, бінарних файлах, бібліотеках або інших артефактах випуску, які користувачам
потрібно оновити локально.

## Пов’язані сторінки

Відомості про мітки аудиту під час установлення, рівні ризику, виявлені проблеми та їх тлумачення див.
у розділі [Аудити безпеки](/clawhub/security-audits).

Відомості про скарги в маркетплейсі, блокування на час модерації, приховані записи, заборони та
статус облікового запису див. у розділі [Модерація та безпека облікового запису](/uk/clawhub/moderation).
