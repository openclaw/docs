# Безпека

Проблеми безпеки ClawHub можна повідомляти через GitHub Security Advisories для
`openclaw/clawhub`.

Використовуйте GitHub Security Advisories для вразливостей у самому ClawHub. Якісні
звіти ClawHub про вразливості включають помилки в:

- вебсайті, API або CLI ClawHub
- публікації в реєстрі, завантаженнях, встановленнях або цілісності артефактів
- автентифікації, авторизації або API-токенах
- скануванні, модерації або обробці звітів

Не використовуйте рекомендації ClawHub для вразливостей у власному вихідному коді стороннього skill або
Plugin. Повідомляйте про них безпосередньо видавцю або в репозиторій вихідного коду,
посилання на який наведено в записі ClawHub.

## Розкриття вразливостей

Оскільки ClawHub є розміщеною хмарною програмою, вразливості сервісу ClawHub
за замовчуванням публічно не розкриваються. Їх розкривають публічно, коли є
докази реального впливу на користувачів або коли користувачам потрібно вжити дій.

Приклади реального впливу на користувачів включають підтверджену експлуатацію, розкриття користувацьких
даних або секретів, потрапляння шкідливого вмісту до користувачів через збій платформи
або будь-яку проблему, яка вимагає від користувачів ротувати облікові дані, оновити локальне програмне забезпечення або
вжити інших захисних дій.

Вразливості в програмному забезпеченні, встановленому користувачами, розкриваються публічно, зокрема
пакети CLI ClawHub, двійкові файли, бібліотеки або інші артефакти випуску, які користувачам
потрібно оновити локально.

## Пов’язані сторінки

Про мітки аудиту під час встановлення, рівні ризику, виявлені проблеми та інтерпретацію див.
[Аудити безпеки](/clawhub/security-audits).

Про звіти маркетплейса, модераційні утримання, приховані записи, блокування та
стан облікового запису див. [Модерація та безпека облікового запису](/clawhub/moderation).
