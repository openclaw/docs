# Безпека

Про проблеми безпеки ClawHub можна повідомляти через GitHub Security Advisories для
`openclaw/clawhub`.

Використовуйте GitHub Security Advisories для вразливостей у самому ClawHub. Якісні
звіти advisories для ClawHub містять помилки в:

- вебсайті, API або CLI ClawHub
- публікації в реєстрі, завантаженнях, установленнях або цілісності артефактів
- автентифікації, авторизації або API-токенах
- скануванні, модерації або обробці звітів

Не використовуйте advisories ClawHub для вразливостей у власному вихідному коді стороннього skill або
plugin. Повідомляйте про них безпосередньо видавцю або в репозиторій вихідного коду,
посилання на який наведено в записі ClawHub.

## Розкриття вразливостей

Оскільки ClawHub є розміщеним у хмарі застосунком, вразливості сервісу ClawHub
за замовчуванням не розкриваються публічно. Вони розкриваються публічно, коли є
докази реального впливу на користувачів або коли користувачам потрібно вжити дій.

Приклади реального впливу на користувачів включають підтверджену експлуатацію, розкриття користувацьких
даних або секретів, потрапляння шкідливого вмісту до користувачів через збій платформи
або будь-яку проблему, яка вимагає від користувачів змінити облікові дані, оновити локальне програмне забезпечення чи
вжити інших захисних заходів.

Вразливості в установленому користувачами програмному забезпеченні розкриваються публічно, наприклад у
пакетах CLI ClawHub, бінарних файлах, бібліотеках або інших релізних артефактах, які користувачам
потрібно оновити локально.

## Пов’язані сторінки

Про мітки аудиту під час установлення, рівні ризику, знахідки та тлумачення див.
[Аудити безпеки](/clawhub/security-audits).

Про звіти маркетплейсу, утримання на модерації, приховані записи, блокування та стан
облікового запису див. [Модерація та безпека облікового запису](/clawhub/moderation).
