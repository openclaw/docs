# Безпека

Про проблеми безпеки ClawHub можна повідомляти через GitHub Security Advisories для
`openclaw/clawhub`.

Використовуйте GitHub Security Advisories для вразливостей у самому ClawHub. Якісні
звіти про вразливості ClawHub охоплюють помилки в:

- вебсайті, API або CLI ClawHub
- публікації в реєстрі, завантаженнях, встановленнях або цілісності артефактів
- автентифікації, авторизації або API-токенах
- скануванні, модерації або обробці звітів

Не використовуйте advisories ClawHub для вразливостей у власному вихідному коді стороннього skill або
plugin. Повідомляйте про них безпосередньо видавцю або в репозиторій вихідного коду,
пов’язаний зі сторінки ClawHub.

## Розкриття вразливостей

Оскільки ClawHub є хмарним застосунком, розміщеним як сервіс, вразливості сервісу ClawHub
за замовчуванням не розкриваються публічно. Їх розкривають публічно, коли є
докази реального впливу на користувачів або коли користувачам потрібно вжити заходів.

Приклади реального впливу на користувачів включають підтверджену експлуатацію, розкриття користувацьких
даних або секретів, потрапляння шкідливого вмісту до користувачів через збій платформи
або будь-яку проблему, що вимагає від користувачів ротації облікових даних, оновлення локального програмного забезпечення чи
вжиття інших захисних заходів.

Вразливості в програмному забезпеченні, встановленому користувачами, розкриваються публічно, наприклад
пакети CLI ClawHub, бінарні файли, бібліотеки або інші релізні артефакти, які користувачам
потрібно оновити локально.

## Пов’язані сторінки

Для міток аудиту під час встановлення, рівнів ризику, знахідок та інтерпретації див.
[Аудити безпеки](/uk/clawhub/security-audits).

Для звітів marketplace, модераційних утримань, прихованих списків, блокувань і стану облікового запису
див. [Модерація та безпека облікового запису](/uk/clawhub/moderation).
