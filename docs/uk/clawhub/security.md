# Безпека

Проблеми безпеки ClawHub можна повідомляти через GitHub Security Advisories для
`openclaw/clawhub`.

Використовуйте GitHub Security Advisories для вразливостей у самому ClawHub. Якісні
звіти щодо ClawHub містять помилки в:

- вебсайті, API або CLI ClawHub
- публікації в реєстрі, завантаженнях, встановленнях або цілісності артефактів
- автентифікації, авторизації або API-токенах
- скануванні, модерації або обробці звітів

Не використовуйте advisories ClawHub для вразливостей у власному вихідному коді стороннього skill або
Plugin. Повідомляйте про них безпосередньо видавцю або до вихідного
репозиторію, посилання на який наведено в записі ClawHub.

## Розкриття вразливостей

Оскільки ClawHub є хмарним застосунком із хостингом, вразливості сервісу ClawHub
за замовчуванням не розкриваються публічно. Їх розкривають публічно, коли є
докази реального впливу на користувачів або коли користувачам потрібно вжити дій.

Приклади реального впливу на користувачів охоплюють підтверджену експлуатацію, розкриття користувацьких
даних або секретів, потрапляння шкідливого вмісту до користувачів через збій платформи
або будь-яку проблему, що вимагає від користувачів ротації облікових даних, оновлення локального програмного забезпечення чи
вжиття інших захисних заходів.

Вразливості в програмному забезпеченні, встановленому користувачами, розкриваються публічно, зокрема
пакети ClawHub CLI, двійкові файли, бібліотеки або інші релізні артефакти, які користувачам
потрібно оновлювати локально.
