# Безпека

Про проблеми безпеки ClawHub можна повідомляти через GitHub Security Advisories для
`openclaw/clawhub`.

Використовуйте GitHub Security Advisories для вразливостей у самому ClawHub. Добрі
звіти advisory для ClawHub охоплюють помилки в:

- вебсайті, API або CLI ClawHub
- публікації в реєстрі, завантаженнях, установленнях або цілісності артефактів
- автентифікації, авторизації або API-токенах
- скануванні, модерації або обробці звітів

Не використовуйте advisories ClawHub для вразливостей у власному вихідному коді сторонньої навички або
Plugin. Повідомляйте про них безпосередньо видавцю або у вихідний
репозиторій, посилання на який наведено в картці ClawHub.

## Розкриття вразливостей

Оскільки ClawHub є хмарною застосунком із хостингом, вразливості сервісу ClawHub
типово не розкриваються публічно. Їх розкривають публічно, коли є
докази реального впливу на користувачів або коли користувачам потрібно вжити дій.

Приклади реального впливу на користувачів включають підтверджену експлуатацію, розкриття даних
користувачів або секретів, потрапляння шкідливого вмісту до користувачів через збій платформи
або будь-яку проблему, що вимагає від користувачів ротації облікових даних, оновлення локального програмного забезпечення чи
інших захисних дій.

Вразливості в установленому користувачем програмному забезпеченні розкриваються публічно, зокрема
пакети CLI ClawHub, бінарні файли, бібліотеки або інші релізні артефакти, які користувачам
потрібно оновити локально.

## Пов’язані сторінки

Про мітки аудиту під час установлення, рівні ризику, знахідки та інтерпретацію див.
[Аудити безпеки](/uk/clawhub/security-audits).

Про звіти marketplace, модераційні утримання, приховані картки, блокування та стан облікового запису
див. [Модерація та безпека облікового запису](/uk/clawhub/moderation).
