# Безпека

Про проблеми безпеки ClawHub можна повідомляти через GitHub Security Advisories для
`openclaw/clawhub`.

Використовуйте GitHub Security Advisories для вразливостей у самому ClawHub. Якісні
звіти про ClawHub advisory містять помилки в:

- вебсайті, API або CLI ClawHub
- публікації в реєстрі, завантаженнях, встановленнях або цілісності артефактів
- автентифікації, авторизації або токенах API
- скануванні, модерації або обробці звітів

Не використовуйте ClawHub advisories для вразливостей у власному вихідному коді сторонньої навички або
Plugin. Повідомляйте про них безпосередньо видавцю або в репозиторій вихідного коду,
посилання на який наведено в записі ClawHub.

## Розкриття вразливостей

Оскільки ClawHub є хмарним застосунком, що розміщується як сервіс, вразливості сервісу ClawHub
не розкриваються публічно за замовчуванням. Вони розкриваються публічно, коли є
докази реального впливу на користувачів або коли користувачам потрібно вжити дій.

Приклади реального впливу на користувачів включають підтверджену експлуатацію, розкриття користувацьких
даних або секретів, потрапляння шкідливого вмісту до користувачів через збій платформи
або будь-яку проблему, яка вимагає від користувачів змінити облікові дані, оновити локальне програмне забезпечення чи
вжити інших захисних дій.

Вразливості у встановленому користувачем програмному забезпеченні розкриваються публічно, зокрема
пакети ClawHub CLI, бінарні файли, бібліотеки або інші релізні артефакти, які користувачам
потрібно оновити локально.

## Пов’язані сторінки

Про мітки аудиту під час встановлення, рівні ризику, знахідки та інтерпретацію див.
[Аудити безпеки](/clawhub/security-audits).

Про звіти marketplace, утримання модерацією, приховані записи, блокування та стан облікового запису
див. [Модерація та безпека облікового запису](/clawhub/moderation).
