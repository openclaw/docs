# Безпека

Про проблеми безпеки ClawHub можна повідомляти через GitHub Security Advisories для
`openclaw/clawhub`.

Використовуйте GitHub Security Advisories для вразливостей у самому ClawHub. Якісні
звіти про вразливості ClawHub включають помилки в:

- вебсайті, API або CLI ClawHub
- публікації в реєстрі, завантаженнях, установленнях або цілісності артефактів
- автентифікації, авторизації або API-токенах
- скануванні, модерації або обробці звітів

Не використовуйте advisory ClawHub для вразливостей у власному вихідному коді сторонньої навички або Plugin. Повідомляйте про них безпосередньо видавцю або в репозиторій вихідного коду, посилання на який є в записі ClawHub.

## Розкриття вразливостей

Оскільки ClawHub є хмарним застосунком із хостингом, вразливості сервісу ClawHub
за замовчуванням не розкриваються публічно. Їх розкривають публічно, коли є
докази реального впливу на користувачів або коли користувачам потрібно вжити дій.

Приклади реального впливу на користувачів включають підтверджену експлуатацію,
розкриття користувацьких даних або секретів, потрапляння шкідливого вмісту до
користувачів через збій платформи або будь-яку проблему, яка вимагає від
користувачів ротації облікових даних, оновлення локального програмного
забезпечення чи вжиття інших захисних заходів.

Вразливості в програмному забезпеченні, встановленому користувачами, розкриваються
публічно, зокрема в пакетах CLI ClawHub, двійкових файлах, бібліотеках або інших
артефактах випуску, які користувачам потрібно оновити локально.

## Пов’язані сторінки

Про мітки аудиту під час установлення, рівні ризику, знахідки та інтерпретацію див.
[Аудити безпеки](/clawhub/security-audits).

Про звіти маркетплейсу, утримання на модерації, приховані записи, блокування та
стан облікового запису див. [Модерація та безпека облікового запису](/clawhub/moderation).
