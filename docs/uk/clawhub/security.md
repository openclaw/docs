# Безпека

Про проблеми безпеки ClawHub можна повідомляти через GitHub Security Advisories для
`openclaw/clawhub`.

Використовуйте GitHub Security Advisories для вразливостей у самому ClawHub. Якісні
звіти про вразливості ClawHub містять помилки в:

- вебсайті, API або CLI ClawHub
- публікації в реєстрі, завантаженнях, установленнях або цілісності артефактів
- автентифікації, авторизації або API-токенах
- скануванні, модерації або обробці звітів

Не використовуйте advisories ClawHub для вразливостей у власному вихідному коді стороннього skill або
plugin. Повідомляйте про них безпосередньо видавцю або до вихідного
репозиторію, посилання на який наведено в записі ClawHub.

## Розкриття вразливостей

Оскільки ClawHub є хмарною застосунком, що хоститься, вразливості сервісу ClawHub
типово не розкриваються публічно. Їх розкривають публічно, коли є
докази реального впливу на користувачів або коли користувачам потрібно вжити заходів.

Приклади реального впливу на користувачів включають підтверджену експлуатацію, розкриття користувацьких
даних або секретів, потрапляння шкідливого вмісту до користувачів через збій платформи
або будь-яку проблему, яка вимагає від користувачів змінити облікові дані, оновити локальне ПЗ або
вжити інших захисних заходів.

Вразливості в установленому користувачами ПЗ розкриваються публічно, наприклад
пакети CLI ClawHub, бінарні файли, бібліотеки або інші артефакти релізів, які користувачам
потрібно оновити локально.

## Пов’язані сторінки

Про мітки аудиту під час установлення, рівні ризику, знахідки та інтерпретацію див.
[Аудити безпеки](/clawhub/security-audits).

Про повідомлення маркетплейсу, модераційні утримання, приховані записи, блокування та стан
облікового запису див. [Модерація та безпека облікового запису](/clawhub/moderation).
