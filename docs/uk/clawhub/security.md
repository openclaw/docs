# Безпека

Про проблеми безпеки ClawHub можна повідомляти через GitHub Security Advisories для
`openclaw/clawhub`.

Використовуйте GitHub Security Advisories для вразливостей у самому ClawHub. Належні
звіти про вразливості ClawHub охоплюють помилки в таких компонентах:

- вебсайт, API або CLI ClawHub
- публікація в реєстрі, завантаження, встановлення або цілісність артефактів
- автентифікація, авторизація або токени API
- сканування, модерація або опрацювання повідомлень

Не використовуйте рекомендації ClawHub з безпеки для повідомлення про вразливості у власному
вихідному коді стороннього skill або плагіна. Повідомляйте про них безпосередньо видавцю або в
репозиторій вихідного коду, посилання на який наведено в записі ClawHub.

## Розкриття інформації про вразливості

Оскільки ClawHub є хмарним застосунком, вразливості служби ClawHub
за замовчуванням публічно не розкриваються. Їх розкривають публічно, коли є
докази реального впливу на користувачів або коли користувачам необхідно вжити заходів.

Приклади реального впливу на користувачів: підтверджена експлуатація, розкриття даних
або секретів користувачів, потрапляння шкідливого вмісту до користувачів через збій платформи
чи будь-яка проблема, яка вимагає від користувачів змінити облікові дані, оновити локальне програмне забезпечення або
вжити інших захисних заходів.

Вразливості в установленому користувачами програмному забезпеченні розкриваються публічно, зокрема
в пакетах CLI ClawHub, виконуваних файлах, бібліотеках або інших артефактах випуску, які користувачам
потрібно оновити локально.

## Пов’язані сторінки

Відомості про позначки аудиту під час установлення, рівні ризику, виявлені проблеми та їх тлумачення див.
в розділі [Аудити безпеки](/clawhub/security-audits).

Відомості про повідомлення на майданчику, призупинення через модерацію, приховані записи, блокування та
стан облікового запису див. у розділі [Модерація та безпека облікових записів](/clawhub/moderation).
