# Безпека + модерація

ClawHub відкритий для публікації, але публічні списки все одно проходять через засоби довіри,
сканування, звітування та модерації. Мета практична: допомогти користувачам
перевіряти те, що вони встановлюють, надати видавцям шлях відновлення після хибних спрацювань
і не допускати зловживальні пакети до публічного пошуку.

Див. також [Прийнятне використання](/uk/clawhub/acceptable-usage).

## Що можуть перевірити користувачі

Перед установленням Skills або plugin перевірте його список у ClawHub на:

- власника й атрибуцію джерела
- останню версію та журнал змін
- потрібні змінні середовища або дозволи
- метадані сумісності для plugins
- стан сканування або модерації
- звіти, коментарі, зірки, завантаження та сигнали встановлення, якщо вони показані

Встановлюйте лише той вміст, який ви розумієте й якому довіряєте.

## Стани сканування

ClawHub може показувати результати сканування або модерації на публічних сторінках і в діагностиці,
видимій власнику.

Поширені результати:

- `clean`: блокувальної проблеми не знайдено.
- `suspicious`: реліз потребує обережності або перевірки.
- `malicious`: реліз вважається небезпечним.
- `pending`: перевірки ще не завершилися.
- `held`, `quarantined`, `revoked` або `hidden`: реліз не повністю
  доступний на публічних поверхнях установлення.

Точне формулювання може відрізнятися залежно від поверхні, але практичне значення те саме: якщо
реліз утримано або заблоковано, користувачам не слід установлювати його, доки власник не вирішить
проблему або модерація не відновить його.

## Skills

Сканування Skills аналізує опублікований пакет Skills, метадані, заявлені
вимоги та підозрілі інструкції.

ClawHub приділяє особливу увагу невідповідностям між тим, що Skills заявляє, і
тим, що він, схоже, робить. Наприклад, Skills, який посилається на потрібний ключ API,
має заявити цю вимогу в `SKILL.md`, щоб користувачі могли побачити її перед
установленням.

Результати сканування базуються на артефактах. Очікувана поведінка провайдера, як-от заявлені
облікові дані API, зворотні виклики OAuth localhost, очищення в межах області під час видалення, кодування Basic Auth
або завантаження вибраних користувачем файлів до заявленого провайдера, трактується
інакше, ніж приховане пересилання облікових даних, широкий доступ до приватних файлів,
непов’язані мережеві призначення або приховане зловживання браузером.

Див. [Формат Skills](/uk/clawhub/skill-format).

## Plugins

Релізи Plugin містять метадані пакета, атрибуцію джерела, поля сумісності
та інформацію про цілісність артефактів.

OpenClaw перевіряє сумісність перед установленням plugins, розміщених у ClawHub. Записи пакетів
також можуть містити метадані дайджесту, щоб OpenClaw міг перевіряти завантажені
артефакти. ClawScan включає заявлені метадані env/config пакета `openclaw.environment`
під час перевірки релізів Plugin, щоб заявлені вимоги часу виконання
порівнювалися зі спостережуваною поведінкою.

## Звіти

Користувачі, які ввійшли в систему, можуть повідомляти про Skills, пакети та коментарі.

Звіти мають бути конкретними й придатними до дії. Зловживання звітуванням саме по собі може призвести до
дій щодо облікового запису.

Приклади звітів:

- оманливі метадані
- незаявлені вимоги до облікових даних або дозволів
- підозрілі інструкції зі встановлення
- шахрайські коментарі або видавання себе за іншу особу
- недобросовісні реєстрації або неправомірне використання торговельних марок
- вміст, що порушує [Прийнятне використання](/uk/clawhub/acceptable-usage)

## Нотатки видавця ClawScan

Видавці можуть надати необов’язкову нотатку ClawScan під час публікації Skills або
Plugin. Ця нотатка дає ClawScan контекст для поведінки, яка інакше може виглядати
незвичною, як-от мережевий доступ, доступ до нативного хоста або облікові дані,
специфічні для провайдера.

## Утримання модерацією

Коли статичний сканер позначає завантажений Skills як шкідливий, видавець
автоматично потрапляє під утримання модерацією (`requiresModerationAt`, установлене для
користувача). Це приховує всі Skills видавця, змушує майбутні публікації
починатися прихованими та створює запис журналу аудиту `user.moderation.auto`.

Статичні підозрілі результати зберігаються як докази файл/рядок для модераторів,
але самі по собі вони не приховують вміст і не визначають публічний вердикт сканування.
Нові завантаження залишаються в стані review/pending, доки не завершиться перевірка LLM. Статичне
сканування блокує негайно лише за шкідливими сигнатурами. Спрацювання рушія
VirusTotal залишаються видимими доказами безпеки, але вердикти VirusTotal Code Insight/Palm
є рекомендаційними й самі по собі не приховують Skills. Перевірки ClawScan LLM
залишають нотатки, узгоджені з призначенням, як рекомендації. Результати перевірки середнього рівня залишаються видимими на
артефакті, тоді як фільтр підозрілості зарезервовано для високовпливових занепокоєнь LLM,
шкідливих результатів або підтверджених виявлень AV-рушіями.

Адміністратори можуть зняти хибнопозитивне утримання:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

Це очищає `requiresModerationAt` і `requiresModerationReason`, відновлює
Skills, приховані утриманням на рівні користувача, і записує запис журналу аудиту `user.moderation.lift`.
Skills, приховані з інших причин, або ті, чиє власне статичне сканування залишається
шкідливим, залишаються прихованими.

## Блокування та стан облікового запису

Облікові записи, що порушують політику ClawHub, можуть втратити доступ до публікації. Серйозні зловживання
можуть призвести до блокування облікового запису, відкликання токена, прихованого вмісту або видалених
списків.

Видалені, заблоковані або вимкнені облікові записи не можуть використовувати токени ClawHub API. Якщо автентифікація CLI
починає відмовляти після дії щодо облікового запису, увійдіть у вебінтерфейс, щоб переглянути стан облікового
запису. Якщо вхід або звичайний доступ CLI заблоковано, зверніться на
security@openclaw.ai для перевірки відновлення.

## Рекомендації для видавців

Щоб зменшити хибні спрацювання та підвищити довіру користувачів:

- підтримуйте точність назв, коротких описів, тегів і журналів змін
- заявляйте потрібні змінні середовища та дозволи
- додавайте нотатку видавця ClawScan, коли реліз має незвичну, але навмисну поведінку
- уникайте обфускованих команд установлення
- посилайтеся на джерело, коли це можливо
- використовуйте пробні запуски перед публікацією plugins
- відповідайте чітко, якщо користувачі або модератори запитують про поведінку пакета
