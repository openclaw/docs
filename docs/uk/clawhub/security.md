# Безпека + модерація

ClawHub відкритий для публікації, але публічні списки все одно проходять через контроль довіри,
сканування, звітування та модерації. Мета практична: допомогти користувачам
перевіряти те, що вони встановлюють, дати видавцям шлях відновлення після хибних спрацювань
і не допускати зловживальні пакети до публічного пошуку.

Див. також [Прийнятне використання](/uk/clawhub/acceptable-usage).

## Що можуть перевірити користувачі

Перед встановленням Skills або плагіна перевірте його запис у ClawHub щодо:

- власника та атрибуції джерела
- останньої версії та журналу змін
- потрібних змінних середовища або дозволів
- метаданих сумісності для плагінів
- статусу сканування або модерації
- звітів, коментарів, зірок, завантажень і сигналів встановлення, де вони показані

Встановлюйте лише той вміст, який ви розумієте й якому довіряєте.

## Стани сканування

ClawHub може показувати результати сканування або модерації на публічних сторінках і в
діагностиці, видимій власнику.

Поширені результати включають:

- `clean`: блокувальної проблеми не знайдено.
- `suspicious`: випуск потребує обережності або перевірки.
- `malicious`: випуск вважається небезпечним.
- `pending`: перевірки ще не завершилися.
- `held`, `quarantined`, `revoked` або `hidden`: випуск не повністю
  доступний на публічних поверхнях встановлення.

Точне формулювання може відрізнятися залежно від поверхні, але практичний зміст той самий: якщо
випуск утримано або заблоковано, користувачам не слід встановлювати його, доки власник не усуне
проблему або модерація не відновить його.

## Skills

Сканування Skills перевіряє опублікований пакет Skills, метадані, оголошені
вимоги та підозрілі інструкції.

ClawHub приділяє особливу увагу невідповідностям між тим, що Skills декларує, і
тим, що він, схоже, робить. Наприклад, Skills, який посилається на потрібний API-ключ,
має оголосити цю вимогу в `SKILL.md`, щоб користувачі могли побачити її перед
встановленням.

Результати сканування базуються на артефактах. Очікувана поведінка провайдера, як-от оголошені
облікові дані API, зворотні виклики OAuth на localhost, scoped uninstall cleanup, кодування Basic Auth
або вибрані користувачем завантаження файлів до заявленого провайдера, розглядається
інакше, ніж приховане пересилання облікових даних, широкий доступ до приватних файлів,
непов’язані мережеві призначення або приховане зловживання браузером.

Див. [Формат Skills](/uk/clawhub/skill-format).

## Плагіни

Випуски плагінів включають метадані пакета, атрибуцію джерела, поля сумісності
та інформацію про цілісність артефакта.

OpenClaw перевіряє сумісність перед встановленням плагінів, розміщених у ClawHub. Записи пакетів
також можуть надавати метадані дайджесту, щоб OpenClaw міг перевіряти завантажені
артефакти. ClawScan включає оголошені метадані env/config пакета `openclaw.environment`
під час перевірки випусків плагінів, щоб оголошені вимоги часу виконання
порівнювалися зі спостережуваною поведінкою.

## Звіти

Користувачі, що ввійшли в систему, можуть повідомляти про Skills, пакети та коментарі.

Звіти мають бути конкретними й придатними до дії. Зловживання звітуванням саме по собі може призвести до
дій щодо облікового запису.

Приклади звітів:

- оманливі метадані
- неоголошені вимоги до облікових даних або дозволів
- підозрілі інструкції зі встановлення
- шахрайські коментарі або видавання себе за іншу особу
- недобросовісні реєстрації або зловживання торговельною маркою
- вміст, що порушує [Прийнятне використання](/uk/clawhub/acceptable-usage)

## Нотатки видавця для ClawScan

Видавці можуть надати необов’язкову нотатку ClawScan під час публікації Skills або
плагіна. Ця нотатка дає ClawScan контекст для поведінки, яка інакше може виглядати
незвичною, наприклад мережевий доступ, доступ до нативного хоста або специфічні для провайдера
облікові дані.

## Утримання модерацією

Коли статичний сканер позначає завантажений Skills як шкідливий, видавець
автоматично потрапляє під утримання модерацією (`requiresModerationAt` встановлюється для
користувача). Це приховує всі Skills видавця, змушує майбутні публікації
починатися прихованими та створює запис журналу аудиту `user.moderation.auto`.

Статичні підозрілі знахідки зберігаються як докази з файлом/рядком для модераторів,
але самі по собі вони не приховують вміст і не визначають публічний вердикт сканування.
Нові завантаження залишаються в стані перевірки/очікування, доки не завершиться LLM-перевірка. Статичне
сканування негайно блокує лише за шкідливими сигнатурами. Спрацювання рушіїв VirusTotal
залишаються видимими доказами безпеки, але вердикти VirusTotal Code Insight/Palm
є рекомендаційними й самі по собі не приховують Skills. LLM-перевірки ClawScan
зберігають нотатки, узгоджені з призначенням, як орієнтир. Знахідки середнього рівня перевірки залишаються видимими на
артефакті, тоді як підозрілий фільтр зарезервовано для високовпливових LLM
занепокоєнь, шкідливих знахідок або підтверджених виявлень AV-рушія.

Адміністратори можуть зняти хибнопозитивне утримання:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

Це очищає `requiresModerationAt` і `requiresModerationReason`, відновлює
Skills, приховані утриманням на рівні користувача, і записує запис журналу аудиту `user.moderation.lift`.
Skills, приховані з інших причин, або ті, чий власний статичний скан залишається
шкідливим, залишаються прихованими.

## Блокування та стан облікового запису

Облікові записи, що порушують політику ClawHub, можуть втратити доступ до публікації. Серйозні зловживання
можуть призвести до блокування облікового запису, відкликання токенів, прихованого вмісту або вилучених
списків.

Видалені, заблоковані або вимкнені облікові записи не можуть використовувати API-токени ClawHub. Якщо CLI-автентифікація
починає давати збій після дії щодо облікового запису, увійдіть у вебінтерфейс, щоб переглянути
стан облікового запису. Якщо вхід або звичайний доступ CLI заблоковано, зверніться до
security@openclaw.ai для перевірки відновлення.

## Настанови для видавців

Щоб зменшити кількість хибних спрацювань і підвищити довіру користувачів:

- підтримуйте точність назв, описів, тегів і журналів змін
- оголошуйте потрібні змінні середовища та дозволи
- додавайте нотатку ClawScan від видавця, коли випуск має незвичну, але навмисну поведінку
- уникайте обфускованих команд встановлення
- додавайте посилання на джерело, коли це можливо
- використовуйте пробні запуски перед публікацією плагінів
- відповідайте чітко, якщо користувачі або модератори запитують про поведінку пакета
