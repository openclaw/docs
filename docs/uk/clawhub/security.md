# Безпека

Про проблеми безпеки ClawHub можна повідомляти через GitHub Security Advisories для
`openclaw/clawhub`.

Використовуйте GitHub Security Advisories для вразливостей у самому ClawHub. Хороші
звіти advisory ClawHub охоплюють помилки в:

- вебсайті, API або CLI ClawHub
- публікації в реєстрі, завантаженнях, інсталяціях або цілісності артефактів
- автентифікації, авторизації або токенах API
- скануванні, модерації або обробці звітів

Не використовуйте advisories ClawHub для вразливостей у власному вихідному коді стороннього skill або
Plugin. Повідомляйте про них безпосередньо видавцю або в репозиторій вихідного коду,
посилання на який наведено в записі ClawHub.

## Розкриття вразливостей

Оскільки ClawHub є розміщеним у хмарі застосунком, вразливості сервісу ClawHub
типово не розкриваються публічно. Їх розкривають публічно, коли є
докази реального впливу на користувачів або коли користувачам потрібно вжити дій.

Приклади реального впливу на користувачів включають підтверджену експлуатацію, розкриття даних
користувачів або секретів, потрапляння шкідливого вмісту до користувачів через збій платформи
або будь-яку проблему, що вимагає від користувачів ротації облікових даних, оновлення локального програмного забезпечення чи
вжиття інших захисних дій.

Вразливості у встановленому користувачем програмному забезпеченні розкриваються публічно, зокрема
пакети CLI ClawHub, бінарні файли, бібліотеки або інші артефакти випуску, які користувачам
потрібно оновити локально.

## Пов’язані сторінки

Про мітки аудиту під час інсталяції, рівні ризику, findings та інтерпретацію див.
[Аудити безпеки](/uk/clawhub/security-audits).

Про звіти маркетплейсу, модераційні утримання, приховані записи, блокування та стан
облікового запису див. [Модерація та безпека облікового запису](/uk/clawhub/moderation).
