# Аудити безпеки

Аудити безпеки ClawHub допомагають вирішити, чи достатньо безпечно встановлювати навичку або плагін. Вони показують, що робить реліз, які повноваження запитує, і чи щось потребує додаткової уваги, перш ніж отримати доступ до файлів, облікових записів, облікових даних, коду або зовнішніх сервісів.

Аудити є сильними сигналами безпеки, але вони не гарантують, що реліз не має ризиків. Завжди зважуйте рішення, перш ніж надавати чутливий доступ.

Див. також [Безпека](/uk/clawhub/security), [Прийнятне використання](/uk/clawhub/acceptable-usage) і [Модерація та безпека облікового запису](/uk/clawhub/moderation).

## Що перевірити перед встановленням

Перед встановленням перегляньте:

- загальний статус аудиту
- рівень ризику
- усі перелічені знахідки
- потрібні облікові дані, дозволи або змінні середовища
- власника, джерело, версію, журнал змін, завантаження, зірки та інші сигнали довіри

Встановлюйте лише той вміст, який розумієте й якому довіряєте.

## Статус аудиту

Статус аудиту підказує, як реагувати на результат аудиту:

| Статус      | Значення                                                                 |
| ----------- | ------------------------------------------------------------------------ |
| `Pass`      | Видимої проблеми вище низького ризику не знайдено.                      |
| `Review`    | Прочитайте знахідки перед встановленням. Реліз усе ще може бути легітимним. |
| `Warn`      | Будьте особливо обережні. ClawHub знайшов суттєву проблему або попереджувальний сигнал. |
| `Malicious` | Не встановлюйте.                                                         |
| `Pending`   | Аудити ще не завершено.                                                  |
| `Error`     | Аудит не вдалося завершити.                                              |

`Pass` заспокоює, але не замінює власного судження. Це найважливіше для інструментів, які можуть публікувати вміст, редагувати дані, виконувати команди, читати файли або отримувати доступ до виробничих систем.

## Рівень ризику

Рівень ризику описує радіус впливу: наскільки великі повноваження, схоже, має реліз, якщо використовувати його за призначенням.

| Рівень ризику | Значення                                                                     |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | Виявлено мало чутливих повноважень або впливу на користувача.                 |
| `Medium`   | Реліз має суттєві повноваження, наприклад доступ до облікового запису або зміну даних. |
| `High`     | Реліз має повноваження з великим впливом, серйозні знахідки або шкідливі сигнали. |

Рівень ризику й статус аудиту відповідають на різні запитання:

- Рівень ризику питає: "Скільки повноважень тут є?"
- Статус аудиту питає: "Що мені робити з цим результатом?"

Наприклад, навичка для публікації може показувати `Review` із ризиком `Medium`. Це не означає, що вона шкідлива. Це означає, що навичка виглядає узгодженою з призначенням, але може діяти зі значущими повноваженнями облікового запису.

## Знахідки

Знахідки пояснюють, чому було показано результат аудиту. Кожна знахідка зазвичай містить:

- що вона означає
- чому її позначено
- відповідний вміст навички або плагіна
- рекомендацію

Знахідки можуть мати мітки `Info`, `Low`, `Medium`, `High` або `Critical`. Знахідки вищої серйозності сильніше впливають на рівень ризику та статус аудиту.

Знахідки з низькою впевненістю приховані з публічного підсумку аудиту, щоб сторінка залишалася зосередженою на корисних доказах.

## Що перевіряє ClawHub

ClawHub аудіює надіслані артефакти релізу, зокрема:

- інструкції навички або метадані плагіна
- заявлені змінні середовища й дозволи
- інструкції зі встановлення та метадані пакета
- включені файли й маніфести файлів
- метадані сумісності та можливостей

Головне питання - узгодженість: чи відповідають назва, короткий опис, метадані, запитані повноваження та фактичний вміст тому, чого користувачі обґрунтовано очікували б?

Потужна поведінка не є автоматично поганою. Багатьом корисним інструментам потрібні облікові дані, локальні команди, API провайдерів або встановлення пакетів. Аудит перевіряє, чи така потужність очікувана, розкрита й пропорційна.

Сторінки артефактів посилаються на повний аудит за адресою:

```text
/<owner>/skills/<slug>/security-audit
```

Сторінка аудиту поєднує:

1. SkillSpector
2. VirusTotal
3. аналіз ризиків

## VirusTotal

ClawHub використовує VirusTotal як телеметрію шкідливого ПЗ у стеку аудиту. VirusTotal є довіреним галузевим стандартом для репутації файлів і сканування шкідливого ПЗ, а наше партнерство дає змогу ClawHub додавати ширшу безпекову аналітику до перевірки навичок і плагінів.

VirusTotal особливо корисний для відомих шкідливих артефактів, спрацювань рушіїв і сигналів репутації, які доповнюють агентно-обізнану перевірку ClawHub. Коли доступні кількості спрацювань від постачальників, аудит підсумовує їх простою мовою, наприклад:

```text
62/62 vendors flagged this skill as clean.
```

або:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

Коли ClawHub не має телеметрії кількості постачальників для підсумування, аудит каже:

```text
No VirusTotal findings
```

VirusTotal залишається телеметрією. Він не замінює власний, обізнаний про артефакти аналіз ризиків ClawHub.

## Аналіз ризиків

Аналіз ризиків внутрішньо забезпечує ClawScan, власна система аудиту безпеки ClawHub. Вона переглядає кожен реліз як артефакт, призначений для агента: інструкції, метадані, заявлені дозволи, файли, сигнали можливостей, сигнали статичного сканування, знахідки SkillSpector, телеметрію VirusTotal і контекст, наданий видавцем. Сигнали статичного сканування є внутрішнім контекстом для цієї перевірки; вони не є самостійним публічним розділом аудиту або вердиктом, що блокує встановлення.

Аналіз ризиків використовує
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
як призму для ризиків, як-от ін'єкція промптів, неправильне використання інструментів, розкриття облікових даних, небезпечне виконання, отруєння пам'яті або контексту та надмірна автономність.

ClawScan не вважає лячну на вигляд можливість автоматично шкідливою. Він питає, чи можливість розкрита, узгоджена з призначенням і підтримана заявленим сценарієм використання релізу.
