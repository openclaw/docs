# Аудити безпеки

Аудити безпеки ClawHub допомагають вирішити, чи достатньо безпечно встановлювати навичку або плагін. Вони показують, що робить реліз, які повноваження він запитує і чи є щось, що потребує додаткової уваги, перш ніж він отримає доступ до файлів, облікових записів, облікових даних, коду або зовнішніх сервісів.

Аудити є сильними сигналами безпеки, але вони не гарантують, що реліз не має ризиків. Завжди застосовуйте власне судження, перш ніж надавати чутливий доступ.

Див. також [Безпека](/clawhub/security), [Прийнятне використання](/clawhub/acceptable-usage) і [Модерація та безпека облікового запису](/clawhub/moderation).

## Що перевірити перед встановленням

Перед встановленням перегляньте:

- загальний статус аудиту
- рівень ризику
- усі перелічені знахідки
- потрібні облікові дані, дозволи або змінні середовища
- власника, джерело, версію, журнал змін, завантаження, зірки та інші сигнали довіри

Встановлюйте лише той вміст, який розумієте і якому довіряєте.

## Статус аудиту

Статус аудиту підказує, як реагувати на результат аудиту:

| Статус      | Значення                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | Не знайдено видимої проблеми вище низького ризику.                                |
| `Review`    | Прочитайте знахідки перед встановленням. Реліз усе ще може бути легітимним. |
| `Warn`      | Будьте особливо обережні. ClawHub знайшов занепокоєння з великим впливом або попереджувальний сигнал. |
| `Malicious` | Не встановлюйте.                                                           |
| `Pending`   | Аудити ще не завершено.                                             |
| `Error`     | Аудит не вдалося завершити.                                         |

`Pass` заспокоює, але не замінює вашого власного судження. Це найважливіше для інструментів, які можуть публікувати вміст, редагувати дані, виконувати команди, читати файли або отримувати доступ до виробничих систем.

## Рівень ризику

Рівень ризику описує радіус ураження: скільки повноважень, схоже, має реліз, якщо використовувати його за призначенням.

| Рівень ризику | Значення                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | Виявлено мало чутливих повноважень або впливу на користувача.                          |
| `Medium`   | Реліз має суттєві повноваження, наприклад доступ до облікового запису або зміну даних. |
| `High`     | Реліз має повноваження з великим впливом, серйозні знахідки або шкідливі сигнали. |

Рівень ризику і статус аудиту відповідають на різні запитання:

- Рівень ризику запитує: "Скільки тут повноважень?"
- Статус аудиту запитує: "Що мені робити з цим результатом?"

Наприклад, навичка для публікації може показувати `Review` із ризиком `Medium`. Це не означає, що вона шкідлива. Це означає, що навичка виглядає узгодженою з призначенням, але може діяти з суттєвими повноваженнями облікового запису.

## Знахідки

Знахідки пояснюють, чому було показано результат аудиту. Кожна знахідка зазвичай містить:

- що вона означає
- чому її позначено
- відповідний вміст навички або плагіна
- рекомендацію

Знахідки можуть мати мітки `Info`, `Low`, `Medium`, `High` або `Critical`. Знахідки з вищою серйозністю сильніше впливають на рівень ризику і статус аудиту.

Знахідки з низькою впевненістю приховано з публічного підсумку аудиту, щоб сторінка залишалася зосередженою на корисних доказах.

## Що перевіряє ClawHub

ClawHub аудіює надіслані артефакти релізу, зокрема:

- інструкції навички або метадані плагіна
- оголошені змінні середовища та дозволи
- інструкції встановлення та метадані пакета
- включені файли та маніфести файлів
- метадані сумісності та можливостей

Головне питання — узгодженість: чи відповідають назва, резюме, метадані, запитані повноваження і фактичний вміст тому, чого користувачі могли б розумно очікувати?

Потужна поведінка не є автоматично поганою. Багатьом корисним інструментам потрібні облікові дані, локальні команди, API провайдерів або встановлення пакетів. Аудит перевіряє, чи ці повноваження очікувані, розкриті та пропорційні.

Сторінки артефактів посилаються на повний аудит за адресою:

```text
/<owner>/skills/<slug>/security-audit
```

Сторінка аудиту поєднує:

1. SkillSpector
2. VirusTotal
3. Аналіз ризиків

## VirusTotal

ClawHub використовує VirusTotal як телеметрію шкідливого ПЗ у стеку аудиту. VirusTotal є надійним галузевим стандартом для репутації файлів і сканування шкідливого ПЗ, а наше партнерство дає ClawHub змогу додавати ширшу безпекову аналітику до перевірки навичок і плагінів.

VirusTotal особливо корисний для відомих шкідливих артефактів, спрацювань рушіїв і сигналів репутації, які доповнюють агентно-орієнтовану перевірку ClawHub. Коли доступні підрахунки рушіїв постачальників, аудит підсумовує їх простою мовою, наприклад:

```text
62/62 vendors flagged this skill as clean.
```

або:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

Коли ClawHub не має телеметрії підрахунку постачальників для підсумку, аудит повідомляє:

```text
No VirusTotal findings
```

VirusTotal залишається телеметрією. Він не замінює власний аналіз ризиків ClawHub, який враховує артефакти.

## Аналіз ризиків

Аналіз ризиків внутрішньо працює на ClawScan, власній системі аудиту безпеки ClawHub. Вона перевіряє кожен реліз як артефакт, орієнтований на агента: інструкції, метадані, оголошені дозволи, файли, сигнали можливостей, сигнали статичного сканування, знахідки SkillSpector, телеметрію VirusTotal і контекст, наданий видавцем. Сигнали статичного сканування є внутрішнім контекстом для цієї перевірки; вони не є окремим публічним розділом аудиту або вердиктом, що блокує встановлення.

Аналіз ризиків використовує
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
як призму для ризиків, таких як ін’єкція промптів, неправильне використання інструментів, розкриття облікових даних, небезпечне виконання, отруєння пам’яті або контексту та надмірна автономність.

ClawScan не вважає можливість із загрозливим виглядом автоматично шкідливою. Він перевіряє, чи можливість розкрита, узгоджена з призначенням і підтримана заявленим сценарієм використання релізу.
