# Аудити безпеки

Аудити безпеки ClawHub допомагають вирішити, чи навичка або плагін є достатньо
безпечними для встановлення. Вони показують, що робить реліз, які повноваження
він запитує, і чи є щось, що потребує додаткової уваги, перш ніж він зможе
отримати доступ до файлів, облікових записів, облікових даних, коду або
зовнішніх сервісів.

Аудити є сильними сигналами безпеки, але вони не гарантують, що реліз не має
ризиків. Завжди керуйтеся власним судженням, перш ніж надавати чутливий доступ.

Див. також [Безпека](/clawhub/security), [Прийнятне використання](/clawhub/acceptable-usage)
і [Модерація та безпека облікових записів](/clawhub/moderation).

## Що перевірити перед встановленням

Перед встановленням перегляньте:

- загальний статус аудиту
- рівень ризику
- будь-які перелічені знахідки
- потрібні облікові дані, дозволи або змінні середовища
- власника, джерело, версію, журнал змін, завантаження, зірки та інші сигнали довіри

Встановлюйте лише той вміст, який ви розумієте і якому довіряєте.

## Статус аудиту

Статус аудиту вказує, як реагувати на результат аудиту:

| Статус      | Значення                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | Не знайдено видимих проблем вище низького ризику.                         |
| `Review`    | Прочитайте знахідки перед встановленням. Реліз усе ще може бути легітимним. |
| `Warn`      | Будьте особливо обережні. ClawHub виявив серйозне занепокоєння або попереджувальний сигнал. |
| `Malicious` | Не встановлюйте.                                                           |
| `Pending`   | Аудити ще не завершено.                                                    |
| `Error`     | Аудит не вдалося завершити.                                                |

`Pass` заспокоює, але не замінює власного судження. Це найважливіше для
інструментів, які можуть публікувати вміст, редагувати дані, запускати команди,
читати файли або отримувати доступ до production-систем.

## Рівень ризику

Рівень ризику описує масштаб впливу: наскільки багато повноважень, схоже, має
реліз, якщо використовувати його за призначенням.

| Рівень ризику | Значення                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | Виявлено мало чутливих повноважень або впливу на користувача.                 |
| `Medium`   | Реліз має суттєві повноваження, як-от доступ до облікового запису або зміни даних. |
| `High`     | Реліз має повноваження з високим впливом, серйозні знахідки або шкідливі сигнали. |

Рівень ризику і статус аудиту відповідають на різні запитання:

- Рівень ризику запитує: "Скільки повноважень тут є?"
- Статус аудиту запитує: "Що мені робити з цим результатом?"

Наприклад, навичка для публікації може показувати `Review` із ризиком `Medium`.
Це не означає, що вона шкідлива. Це означає, що навичка, схоже, відповідає своїй
меті, але може діяти зі значними повноваженнями облікового запису.

## Знахідки

Знахідки пояснюють, чому було показано результат аудиту. Кожна знахідка зазвичай
містить:

- що вона означає
- чому її позначено
- відповідний вміст навички або плагіна
- рекомендацію

Знахідки можуть мати мітки `Info`, `Low`, `Medium`, `High` або `Critical`. Знахідки
з вищою серйозністю сильніше впливають на рівень ризику і статус аудиту.

Знахідки з низькою впевненістю приховано з публічного підсумку аудиту, щоб
сторінка залишалася зосередженою на корисних доказах.

## Що перевіряє ClawHub

ClawHub аудіює надіслані артефакти релізів, зокрема:

- інструкції навички або метадані плагіна
- оголошені змінні середовища та дозволи
- інструкції зі встановлення та метадані пакета
- включені файли та маніфести файлів
- метадані сумісності та можливостей

Головне питання — узгодженість: чи відповідають назва, короткий опис, метадані,
запитувані повноваження і фактичний вміст тому, чого користувачі могли б
обґрунтовано очікувати?

Потужна поведінка не є автоматично поганою. Багатьом корисним інструментам
потрібні облікові дані, локальні команди, API провайдерів або встановлення
пакетів. Аудит перевіряє, чи ці повноваження очікувані, розкриті та пропорційні.

Сторінки артефактів посилаються на повний аудит за адресою:

```text
/<owner>/skills/<slug>/security-audit
```

Сторінка аудиту поєднує:

1. SkillSpector
2. VirusTotal
3. Аналіз ризиків

## VirusTotal

ClawHub використовує VirusTotal як телеметрію шкідливого ПЗ у стеку аудиту.
VirusTotal є довіреним галузевим стандартом для репутації файлів і сканування
на шкідливе ПЗ, а наше партнерство дає ClawHub змогу додавати ширший контекст
безпекової розвідки до перевірки навичок і плагінів.

VirusTotal особливо корисний для відомих шкідливих артефактів, спрацювань
рушіїв і репутаційних сигналів, які доповнюють агентно-орієнтовану перевірку
ClawHub. Коли доступні кількості спрацювань рушіїв постачальників, аудит
підсумовує їх простою мовою, наприклад:

```text
62/62 vendors flagged this skill as clean.
```

або:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

Коли ClawHub не має телеметрії кількості постачальників для підсумування, аудит
повідомляє:

```text
No VirusTotal findings
```

VirusTotal залишається телеметрією. Він не замінює власний аналіз ризиків
ClawHub, який враховує артефакти.

## Аналіз ризиків

Аналіз ризиків внутрішньо працює на ClawScan, власній системі аудиту безпеки
ClawHub. Вона перевіряє кожен реліз як артефакт, призначений для агента:
інструкції, метадані, оголошені дозволи, файли, сигнали можливостей, сигнали
статичного сканування, знахідки SkillSpector, телеметрію VirusTotal і контекст,
наданий видавцем. Сигнали статичного сканування є внутрішнім контекстом для цієї
перевірки; вони не є окремим публічним розділом аудиту або вердиктом, що блокує
встановлення.

Аналіз ризиків використовує
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
як призму для ризиків, таких як інʼєкція промптів, неправильне використання
інструментів, розкриття облікових даних, небезпечне виконання, отруєння памʼяті
або контексту та надмірна агентність.

ClawScan не вважає можливість зі страшним виглядом автоматично шкідливою. Він
запитує, чи можливість розкрита, відповідає призначенню і підтримується
заявленим сценарієм використання релізу.
