# Аудити безпеки

Аудити безпеки ClawHub допомагають вирішити, чи достатньо безпечні skill або Plugin
для встановлення. Вони показують, що робить випуск, які повноваження він запитує та
чи потребує щось додаткової уваги, перш ніж він отримає доступ до файлів, облікових записів,
облікових даних, коду або зовнішніх служб.

Аудити є вагомими сигналами безпеки, але не гарантують, що випуск
не містить ризиків. Завжди зважуйте ризики, перш ніж надавати доступ до конфіденційних ресурсів.

Див. також [Безпека](/clawhub/security), [Прийнятне використання](/uk/clawhub/acceptable-usage)
та [Модерація й безпека облікових записів](/clawhub/moderation).

## Що перевірити перед установленням

Перед установленням перегляньте:

- загальний статус аудиту
- рівень ризику
- усі наведені висновки
- необхідні облікові дані, дозволи або змінні середовища
- власника, джерело, версію, журнал змін, кількість завантажень, зірки та інші сигнали довіри

Установлюйте лише той вміст, який ви розумієте і якому довіряєте.

## Статус аудиту

Статус аудиту вказує, як реагувати на його результат:

| Статус      | Значення                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | Видимих проблем із рівнем ризику вище низького не виявлено.                                |
| `Review`    | Прочитайте висновки перед установленням. Випуск усе ще може бути легітимним. |
| `Warn`      | Будьте особливо обережні. ClawHub виявив проблему з істотними наслідками або попереджувальний сигнал. |
| `Malicious` | Не встановлюйте.                                                           |
| `Pending`   | Аудити ще не завершено.                                             |
| `Error`     | Аудит не вдалося завершити.                                         |

Статус `Pass` вселяє впевненість, але не замінює власного оцінювання. Це особливо
важливо для інструментів, які можуть публікувати вміст, редагувати дані, виконувати команди, читати файли або
отримувати доступ до промислових систем.

## Рівень ризику

Рівень ризику описує масштаб потенційних наслідків: наскільки широкі повноваження, імовірно, має випуск, якщо
використовувати його за призначенням.

| Рівень ризику | Значення                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | Виявлено незначні повноваження щодо конфіденційних ресурсів або вплив на користувача.                          |
| `Medium`   | Випуск має істотні повноваження, як-от доступ до облікового запису або внесення змін до даних. |
| `High`     | Випуск має повноваження зі значними наслідками, критичні висновки або ознаки зловмисності. |

Рівень ризику та статус аудиту відповідають на різні запитання:

- Рівень ризику відповідає на запитання: «Наскільки широкі тут повноваження?»
- Статус аудиту відповідає на запитання: «Що робити з цим результатом?»

Наприклад, skill для публікування може мати статус `Review` і рівень ризику `Medium`. Це
не означає, що він зловмисний. Це означає, що skill, імовірно, відповідає своєму призначенню, але може
виконувати дії зі значними повноваженнями в обліковому записі.

## Висновки

Висновки пояснюють, чому було показано певний результат аудиту. Кожен висновок зазвичай містить:

- що це означає
- чому це було позначено
- відповідний вміст skill або Plugin
- рекомендацію

Висновки можуть мати позначки `Info`, `Low`, `Medium`, `High` або `Critical`. Висновки з вищим
рівнем серйозності сильніше впливають на рівень ризику та статус аудиту.

Висновки з низьким рівнем упевненості приховано із загальнодоступного зведення аудиту, щоб сторінка
залишалася зосередженою на корисних доказах.

## Що перевіряє ClawHub

ClawHub перевіряє подані артефакти випуску, зокрема:

- інструкції навички або метадані плагіна
- задекларовані змінні середовища та дозволи
- інструкції зі встановлення та метадані пакета
- включені файли та маніфести файлів
- метадані сумісності та можливостей

Головне питання — узгодженість: чи відповідають назва, короткий опис, метадані, запитувані
повноваження та фактичний вміст тому, чого користувачі можуть обґрунтовано очікувати?

Потужні можливості не є автоматично поганими. Багатьом корисним інструментам потрібні облікові дані,
локальні команди, API провайдерів або встановлення пакетів. Аудит перевіряє, чи є такі
можливості очікуваними, задекларованими та пропорційними.

Сторінки артефактів містять посилання на повний аудит за адресою:

```text
/<owner>/skills/<slug>/security-audit
```

Сторінка аудиту об’єднує:

1. SkillSpector
2. VirusTotal
3. Аналіз ризиків

## VirusTotal

ClawHub використовує VirusTotal як джерело телеметрії про шкідливе програмне забезпечення в системі аудиту. VirusTotal —
надійний галузевий стандарт для визначення репутації файлів і сканування на шкідливе програмне забезпечення, а наше
партнерство дає ClawHub змогу доповнювати перевірку навичок і плагінів ширшими даними
про безпеку.

VirusTotal особливо корисний для виявлення відомих шкідливих артефактів, спрацювань рушіїв і
сигналів репутації, які доповнюють перевірку ClawHub з урахуванням специфіки агентів. Коли доступна
кількість спрацювань рушіїв постачальників, аудит узагальнює її зрозумілою мовою, наприклад:

```text
62/62 постачальників позначили цю навичку як безпечну.
```

або:

```text
2/64 постачальників позначили цю навичку як шкідливу, 1/64 — як підозрілу, а 61/64 — як безпечну.
```

Коли ClawHub не має телеметрії щодо кількості спрацювань постачальників для узагальнення, в аудиті зазначено:

```text
Висновків VirusTotal немає
```

VirusTotal залишається джерелом телеметрії. Він не замінює власний аналіз ризиків ClawHub
з урахуванням специфіки артефактів.

## Аналіз ризиків

Внутрішньою основою аналізу ризиків є ClawScan — власна система аудиту безпеки
ClawHub. Вона перевіряє кожен випуск як артефакт, призначений для агента: інструкції,
метадані, задекларовані дозволи, файли, сигнали можливостей, сигнали статичного сканування,
висновки SkillSpector, телеметрію VirusTotal і контекст, наданий видавцем.
Сигнали статичного сканування є внутрішнім контекстом для цієї перевірки; вони не є
окремим загальнодоступним розділом аудиту чи вердиктом, що блокує встановлення.

Для аналізу ризиків використовується
[десятка найпоширеніших ризиків агентних навичок OWASP](https://owasp.org/www-project-agentic-skills-top-10/)
як основа для оцінювання таких ризиків, як ін’єкція підказок, неналежне використання інструментів, розкриття облікових даних,
небезпечне виконання, отруєння пам’яті чи контексту та надмірна автономність.

ClawScan не вважає можливість зі страхітливим виглядом автоматично шкідливою.
Він перевіряє, чи задекларовано цю можливість, чи відповідає вона призначенню та чи обґрунтована
заявленим сценарієм використання випуску.
