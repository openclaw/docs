# Аудити безпеки

Аудити безпеки ClawHub допомагають вирішити, чи достатньо безпечні Skills або Plugin для встановлення. Вони показують, що робить випуск, які повноваження він запитує та чи потребує щось додаткової уваги, перш ніж отримати доступ до файлів, облікових записів, облікових даних, коду або зовнішніх служб.

Аудити є вагомими показниками безпеки, але не гарантують, що випуск не містить ризиків. Завжди ретельно оцінюйте ситуацію, перш ніж надавати доступ до конфіденційних ресурсів.

Див. також [Безпека](/clawhub/security), [Допустиме використання](/clawhub/acceptable-usage) та [Модерація й безпека облікового запису](/clawhub/moderation).

## Що перевірити перед встановленням

Перед встановленням перегляньте:

- загальний статус аудиту
- рівень ризику
- усі зазначені результати перевірки
- необхідні облікові дані, дозволи або змінні середовища
- власника, джерело, версію, журнал змін, кількість завантажень, зірки та інші ознаки надійності

Встановлюйте лише той вміст, який ви розумієте і якому довіряєте.

## Статус аудиту

Статус аудиту вказує, як слід реагувати на його результат:

| Статус      | Значення                                                                         |
| ----------- | -------------------------------------------------------------------------------- |
| `Pass`      | Видимих проблем із рівнем ризику вище низького не виявлено.                      |
| `Review`    | Прочитайте результати перевірки перед встановленням. Випуск усе ще може бути легітимним. |
| `Warn`      | Будьте особливо обережні. ClawHub виявив проблему зі значним впливом або попереджувальну ознаку. |
| `Malicious` | Не встановлюйте.                                                                 |
| `Pending`   | Аудити ще не завершено.                                                          |
| `Error`     | Не вдалося завершити аудит.                                                      |

Статус `Pass` додає впевненості, але не замінює вашої власної оцінки. Це особливо важливо для інструментів, які можуть публікувати вміст, змінювати дані, виконувати команди, читати файли або отримувати доступ до робочих систем.

## Рівень ризику

Рівень ризику описує масштаб потенційних наслідків: якими повноваженнями, ймовірно, володіє випуск, якщо використовувати його за призначенням.

| Рівень ризику | Значення                                                                     |
| ------------- | ---------------------------------------------------------------------------- |
| `Low`         | Виявлено незначні конфіденційні повноваження або незначний вплив на користувача. |
| `Medium`      | Випуск має суттєві повноваження, як-от доступ до облікового запису або зміна даних. |
| `High`        | Випуск має повноваження зі значним впливом, серйозні результати перевірки або ознаки шкідливості. |

Рівень ризику та статус аудиту відповідають на різні запитання:

- Рівень ризику відповідає на запитання: «Наскільки значні тут повноваження?»
- Статус аудиту відповідає на запитання: «Що мені робити з цим результатом?»

Наприклад, Skills для публікації можуть мати статус `Review` із ризиком `Medium`. Це не означає, що вони шкідливі. Це означає, що призначення Skills, імовірно, відповідає їхній поведінці, але вони можуть діяти зі значними повноваженнями в обліковому записі.

## Результати перевірки

Результати перевірки пояснюють, чому було показано відповідний результат аудиту. Кожен результат зазвичай містить:

- що він означає
- чому його було позначено
- відповідний вміст Skills або Plugin
- рекомендацію

Результати перевірки можуть мати позначки `Info`, `Low`, `Medium`, `High` або `Critical`. Результати з вищим рівнем серйозності сильніше впливають на рівень ризику та статус аудиту.

Результати з низькою впевненістю приховано із загальнодоступного зведення аудиту, щоб сторінка залишалася зосередженою на корисних доказах.

## Що перевіряє ClawHub

ClawHub перевіряє подані артефакти випуску, зокрема:

- інструкції Skills або метадані Plugin
- оголошені змінні середовища та дозволи
- інструкції зі встановлення та метадані пакета
- включені файли та файлові маніфести
- метадані сумісності та можливостей

Головне питання — узгодженість: чи відповідають назва, опис, метадані, запитувані повноваження та фактичний вміст тому, чого користувачі могли б обґрунтовано очікувати?

Потужні можливості не є автоматично небезпечними. Багатьом корисним інструментам потрібні облікові дані, локальні команди, API постачальників або встановлення пакетів. Аудит перевіряє, чи є такі повноваження очікуваними, розкритими та пропорційними.

Сторінки артефактів містять посилання на повний аудит за адресою:

```text
/<owner>/skills/<slug>/security-audit
```

Сторінка аудиту об’єднує:

1. SkillSpector
2. VirusTotal
3. Аналіз ризиків

## VirusTotal

ClawHub використовує VirusTotal як джерело телеметрії про шкідливе програмне забезпечення в системі аудиту. VirusTotal — це надійний галузевий стандарт для оцінювання репутації файлів і сканування на наявність шкідливого програмного забезпечення, а наше партнерство дає змогу ClawHub додавати ширші відомості про безпеку до перевірки Skills і Plugin.

VirusTotal особливо корисний для виявлення відомих шкідливих артефактів, спрацювань рушіїв і репутаційних сигналів, які доповнюють перевірку ClawHub з урахуванням особливостей агентів. Коли доступні кількісні дані від рушіїв постачальників, аудит узагальнює їх зрозумілою мовою, наприклад:

```text
62/62 постачальники позначили ці Skills як безпечні.
```

або:

```text
2/64 постачальники позначили ці Skills як шкідливі, 1/64 — як підозрілі, а 61/64 — як безпечні.
```

Коли ClawHub не має кількісної телеметрії від постачальників для узагальнення, в аудиті зазначено:

```text
Немає результатів VirusTotal
```

VirusTotal залишається джерелом телеметрії. Він не замінює власний аналіз ризиків ClawHub з урахуванням особливостей артефактів.

## Аналіз ризиків

Аналіз ризиків виконується внутрішньою системою ClawScan — власною системою аудиту безпеки ClawHub. Вона перевіряє кожен випуск як артефакт, призначений для агентів: інструкції, метадані, оголошені дозволи, файли, сигнали можливостей, сигнали статичного сканування, результати SkillSpector, телеметрію VirusTotal і контекст, наданий видавцем. Сигнали статичного сканування є внутрішнім контекстом для цієї перевірки; вони не становлять окремого загальнодоступного розділу аудиту або вердикту, що блокує встановлення.

Аналіз ризиків використовує
[Десятку найпоширеніших ризиків агентних Skills за версією OWASP](https://owasp.org/www-project-agentic-skills-top-10/)
як основу для оцінювання таких ризиків, як ін’єкція підказок, неналежне використання інструментів, розкриття облікових даних, небезпечне виконання, отруєння пам’яті чи контексту та надмірна автономність.

ClawScan не вважає можливість, що виглядає небезпечною, автоматично шкідливою. Система перевіряє, чи розкрито цю можливість, чи відповідає вона призначенню та чи обґрунтована заявленим сценарієм використання випуску.
