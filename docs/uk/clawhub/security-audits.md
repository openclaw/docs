# Аудити безпеки

Аудити безпеки ClawHub допомагають вирішити, чи є навичка або плагін достатньо безпечними
для встановлення. Вони показують, що робить випуск, які повноваження він запитує, і
чи є щось, що потребує додаткової уваги, перш ніж він зможе отримати доступ до файлів, облікових записів,
облікових даних, коду або зовнішніх сервісів.

Аудити є сильними сигналами безпеки, але вони не гарантують, що випуск
не має ризиків. Завжди зважуйте рішення, перш ніж надавати чутливий доступ.

Див. також [Безпека](/uk/clawhub/security), [Прийнятне використання](/uk/clawhub/acceptable-usage)
і [Модерація та безпека облікового запису](/uk/clawhub/moderation).

## Що перевірити перед встановленням

Перед встановленням перегляньте:

- загальний статус аудиту
- рівень ризику
- усі перелічені висновки
- потрібні облікові дані, дозволи або змінні середовища
- власника, джерело, версію, журнал змін, завантаження, зірки та інші сигнали довіри

Встановлюйте лише той вміст, який ви розумієте і якому довіряєте.

## Статус аудиту

Статус аудиту підказує, як реагувати на результат аудиту:

| Статус      | Значення                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | Не знайдено видимої проблеми вище низького ризику.                                |
| `Review`    | Прочитайте висновки перед встановленням. Випуск усе ще може бути легітимним. |
| `Warn`      | Будьте особливо обережні. ClawHub знайшов проблему з високим впливом або попереджувальний сигнал. |
| `Malicious` | Не встановлюйте.                                                           |
| `Pending`   | Аудити ще не завершилися.                                             |
| `Error`     | Аудит не вдалося завершити.                                         |

`Pass` заспокоює, але не замінює вашого власного судження. Це найважливіше
для інструментів, які можуть публікувати вміст, редагувати дані, виконувати команди, читати файли або
отримувати доступ до виробничих систем.

## Рівень ризику

Рівень ризику описує зону впливу: скільки повноважень, схоже, має випуск, якщо
використовувати його за призначенням.

| Рівень ризику | Значення                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | Виявлено небагато чутливих повноважень або впливу на користувача.                          |
| `Medium`   | Випуск має суттєві повноваження, як-от доступ до облікового запису або зміни даних. |
| `High`     | Випуск має повноваження з високим впливом, серйозні висновки або шкідливі сигнали. |

Рівень ризику і статус аудиту відповідають на різні запитання:

- Рівень ризику запитує: "Скільки тут повноважень?"
- Статус аудиту запитує: "Що мені робити з цим результатом?"

Наприклад, навичка для публікації може показувати `Review` із ризиком `Medium`. Це
не означає, що вона шкідлива. Це означає, що навичка, схоже, відповідає призначенню, але може
діяти з суттєвими повноваженнями облікового запису.

## Висновки

Висновки пояснюють, чому було показано результат аудиту. Кожен висновок зазвичай містить:

- що він означає
- чому його позначили
- відповідний вміст навички або плагіна
- рекомендацію

Висновки можуть мати позначки `Info`, `Low`, `Medium`, `High` або `Critical`. Висновки з вищою
серйозністю сильніше впливають на рівень ризику і статус аудиту.

Висновки з низькою впевненістю приховано з публічного зведення аудиту, щоб сторінка
залишалася зосередженою на корисних доказах.

## Що перевіряє ClawHub

ClawHub аудіює подані артефакти випуску, зокрема:

- інструкції навички або метадані плагіна
- оголошені змінні середовища та дозволи
- інструкції зі встановлення і метадані пакета
- включені файли та файлові маніфести
- метадані сумісності та можливостей

Головне питання — узгодженість: чи відповідають назва, короткий опис, метадані, запитані
повноваження і фактичний вміст тому, чого користувачі могли б обґрунтовано очікувати?

Потужна поведінка не є автоматично поганою. Багатьом корисним інструментам потрібні облікові дані,
локальні команди, API провайдерів або встановлення пакетів. Аудит перевіряє, чи така
потужність очікувана, розкрита і пропорційна.

Сторінки артефактів посилаються на повний аудит за адресою:

```text
/<owner>/skills/<slug>/security-audit
```

Сторінка аудиту поєднує:

1. SkillSpector
2. VirusTotal
3. аналіз ризиків

## VirusTotal

ClawHub використовує VirusTotal як телеметрію шкідливого ПЗ у стеку аудиту. VirusTotal є
надійним галузевим стандартом для репутації файлів і сканування шкідливого ПЗ, а наше
партнерство дає ClawHub змогу додавати ширшу безпекову аналітику до перевірки навичок і плагінів.

VirusTotal особливо корисний для відомих шкідливих артефактів, спрацьовувань рушіїв і
сигналів репутації, які доповнюють агентно-орієнтовану перевірку ClawHub. Коли доступні
кількості спрацьовувань рушіїв постачальників, аудит підсумовує їх простою мовою, наприклад:

```text
62/62 vendors flagged this skill as clean.
```

або:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

Коли ClawHub не має телеметрії кількості постачальників для підсумку, аудит каже:

```text
No VirusTotal findings
```

VirusTotal залишається телеметрією. Він не замінює власний аналіз ризиків ClawHub,
орієнтований на артефакти.

## Аналіз ризиків

Аналіз ризиків внутрішньо забезпечує ClawScan, власна система аудиту безпеки
ClawHub. Вона перевіряє кожен випуск як артефакт, призначений для агента: інструкції,
метадані, оголошені дозволи, файли, сигнали можливостей, сигнали статичного сканування,
висновки SkillSpector, телеметрію VirusTotal і контекст, наданий видавцем.
Сигнали статичного сканування є внутрішнім контекстом для цієї перевірки; вони не є
окремим публічним розділом аудиту або вердиктом, що блокує встановлення.

Аналіз ризиків використовує
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
як призму для ризиків, як-от ін'єкція промптів, неправильне використання інструментів, розкриття облікових даних,
небезпечне виконання, отруєння пам'яті або контексту та надмірна агентність.

ClawScan не вважає можливість, що виглядає лячно, автоматично шкідливою.
Він запитує, чи можливість розкрита, відповідає призначенню і підтримується
заявленим сценарієм використання випуску.
