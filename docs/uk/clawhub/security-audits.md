# Аудити безпеки

Аудити безпеки ClawHub допомагають вирішити, чи достатньо безпечно
встановлювати навичку або плагін. Вони показують, що робить реліз, які
повноваження він запитує і чи потребує щось додаткової уваги, перш ніж отримати
доступ до файлів, облікових записів, облікових даних, коду або зовнішніх
сервісів.

Аудити є сильними сигналами безпеки, але вони не гарантують, що реліз не має
ризиків. Завжди зважуйте рішення, перш ніж надавати чутливий доступ.

Див. також [Безпека](/clawhub/security), [Прийнятне використання](/clawhub/acceptable-usage)
і [Модерація та безпека облікового запису](/clawhub/moderation).

## Що перевірити перед встановленням

Перед встановленням перегляньте:

- загальний статус аудиту
- рівень ризику
- усі перелічені висновки
- потрібні облікові дані, дозволи або змінні середовища
- власника, джерело, версію, журнал змін, завантаження, зірки та інші сигнали довіри

Встановлюйте лише той вміст, який ви розумієте й якому довіряєте.

## Статус аудиту

Статус аудиту підказує, як реагувати на результат аудиту:

| Статус      | Значення                                                                  |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | Видимих проблем вище низького ризику не знайдено.                         |
| `Review`    | Перед встановленням прочитайте висновки. Реліз усе ще може бути легітимним. |
| `Warn`      | Будьте особливо обережні. ClawHub виявив проблему з великим впливом або попереджувальний сигнал. |
| `Malicious` | Не встановлюйте.                                                          |
| `Pending`   | Аудити ще не завершено.                                                   |
| `Error`     | Не вдалося завершити аудит.                                               |

`Pass` заспокоює, але не замінює вашого власного судження. Це найважливіше для
інструментів, які можуть публікувати вміст, редагувати дані, виконувати команди,
читати файли або отримувати доступ до виробничих систем.

## Рівень ризику

Рівень ризику описує радіус ураження: наскільки широкі повноваження, схоже, має
реліз, якщо використовувати його за призначенням.

| Рівень ризику | Значення                                                                    |
| ------------- | --------------------------------------------------------------------------- |
| `Low`         | Виявлено мало чутливих повноважень або впливу на користувача.               |
| `Medium`      | Реліз має значущі повноваження, як-от доступ до облікового запису або зміни даних. |
| `High`        | Реліз має повноваження з великим впливом, серйозні висновки або ознаки шкідливості. |

Рівень ризику та статус аудиту відповідають на різні запитання:

- Рівень ризику запитує: "Скільки повноважень тут є?"
- Статус аудиту запитує: "Що мені робити з цим результатом?"

Наприклад, навичка для публікації може показувати `Review` із ризиком `Medium`.
Це не означає, що вона шкідлива. Це означає, що навичка, схоже, відповідає своїй
меті, але може діяти зі значущими повноваженнями облікового запису.

## Висновки

Висновки пояснюють, чому було показано результат аудиту. Кожен висновок зазвичай
містить:

- що це означає
- чому це було позначено
- відповідний вміст навички або плагіна
- рекомендацію

Висновки можуть мати позначки `Info`, `Low`, `Medium`, `High` або `Critical`.
Висновки з вищою серйозністю сильніше впливають на рівень ризику та статус
аудиту.

Висновки з низькою впевненістю приховано з публічного підсумку аудиту, щоб
сторінка залишалася зосередженою на корисних доказах.

## Що перевіряє ClawHub

ClawHub аудитує надіслані артефакти релізу, зокрема:

- інструкції навички або метадані плагіна
- оголошені змінні середовища та дозволи
- інструкції зі встановлення та метадані пакета
- включені файли та маніфести файлів
- метадані сумісності та можливостей

Головне питання — узгодженість: чи відповідають назва, короткий опис, метадані,
запитані повноваження та фактичний вміст тому, чого користувачі могли б
розумно очікувати?

Потужна поведінка не є автоматично поганою. Багатьом корисним інструментам
потрібні облікові дані, локальні команди, API провайдерів або встановлення
пакетів. Аудит перевіряє, чи ця потужність очікувана, розкрита та пропорційна.

Сторінки артефактів посилаються на повний аудит за адресою:

```text
/<owner>/skills/<slug>/security-audit
```

Сторінка аудиту поєднує:

1. SkillSpector
2. VirusTotal
3. Аналіз ризиків

## VirusTotal

ClawHub використовує VirusTotal як телеметрію шкідливого ПЗ у стеку аудиту.
VirusTotal — це надійний галузевий стандарт для репутації файлів і сканування
на шкідливе ПЗ, а наше партнерство дає ClawHub змогу додавати ширший
безпековий контекст до перевірки навичок і плагінів.

VirusTotal особливо корисний для відомих шкідливих артефактів, спрацювань
рушіїв і сигналів репутації, які доповнюють агентно-обізнану перевірку ClawHub.
Коли доступні кількості спрацювань рушіїв постачальників, аудит підсумовує їх
простою мовою, наприклад:

```text
62/62 vendors flagged this skill as clean.
```

або:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

Коли ClawHub не має телеметрії кількості постачальників для підсумування, аудит
повідомляє:

```text
No VirusTotal findings
```

VirusTotal залишається телеметрією. Він не замінює власний, обізнаний щодо
артефактів аналіз ризиків ClawHub.

## Аналіз ризиків

Аналіз ризиків внутрішньо забезпечує ClawScan, власна система аудиту безпеки
ClawHub. Вона перевіряє кожен реліз як артефакт, призначений для агента:
інструкції, метадані, оголошені дозволи, файли, сигнали можливостей, сигнали
статичного сканування, висновки SkillSpector, телеметрію VirusTotal і контекст,
наданий видавцем. Сигнали статичного сканування є внутрішнім контекстом для цієї
перевірки; вони не є окремим публічним розділом аудиту або вердиктом, що блокує
встановлення.

Аналіз ризиків використовує
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
як призму для ризиків, таких як інʼєкція промптів, неправильне використання
інструментів, розкриття облікових даних, небезпечне виконання, отруєння памʼяті
або контексту та надмірна агентність.

ClawScan не вважає можливість, яка виглядає загрозливо, автоматично шкідливою.
Він перевіряє, чи можливість розкрита, відповідає меті та підтримується заявленим
сценарієм використання релізу.
