# Аудити безпеки

Аудити безпеки ClawHub допомагають вирішити, чи достатньо безпечно
встановлювати навичку або Plugin. Вони показують, що робить реліз, які повноваження він запитує та
чи є щось, що потребує додаткової уваги, перш ніж він отримає доступ до файлів, облікових записів,
облікових даних, коду або зовнішніх сервісів.

Аудити є сильними сигналами безпеки, але вони не гарантують, що реліз
не містить ризиків. Завжди використовуйте власне судження перед наданням чутливого доступу.

Див. також [Безпека](/uk/clawhub/security), [Прийнятне використання](/uk/clawhub/acceptable-usage)
і [Модерація та безпека облікового запису](/uk/clawhub/moderation).

## Що перевірити перед встановленням

Перед встановленням перегляньте:

- загальний статус аудиту
- рівень ризику
- усі перелічені висновки
- потрібні облікові дані, дозволи або змінні середовища
- власника, джерело, версію, журнал змін, завантаження, зірки та інші сигнали довіри

Встановлюйте лише той вміст, який ви розумієте і якому довіряєте.

## Статус аудиту

Статус аудиту підказує, як реагувати на результат аудиту:

| Статус     | Значення                                                                 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | Видимих проблем вище низького ризику не знайдено.                       |
| `Review`    | Прочитайте висновки перед встановленням. Реліз усе ще може бути легітимним. |
| `Warn`      | Будьте особливо обережні. ClawHub виявив серйозне занепокоєння або попереджувальний сигнал. |
| `Malicious` | Не встановлюйте.                                                        |
| `Pending`   | Аудити ще не завершено.                                                  |
| `Error`     | Аудит не вдалося завершити.                                              |

`Pass` заспокоює, але не замінює вашого власного судження. Це найважливіше
для інструментів, які можуть публікувати вміст, редагувати дані, виконувати команди, читати файли або
отримувати доступ до виробничих систем.

## Рівень ризику

Рівень ризику описує радіус впливу: яку потужність, схоже, має реліз, якщо
використовувати його за призначенням.

| Рівень ризику | Значення                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | Виявлено мало чутливих повноважень або впливу на користувача.                 |
| `Medium`   | Реліз має суттєві повноваження, наприклад доступ до облікового запису або зміну даних. |
| `High`     | Реліз має високовпливові повноваження, серйозні висновки або шкідливі сигнали. |

Рівень ризику і статус аудиту відповідають на різні запитання:

- Рівень ризику запитує: "Скільки тут потужності?"
- Статус аудиту запитує: "Що мені робити з цим результатом?"

Наприклад, навичка для публікації може показувати `Review` із ризиком `Medium`. Це
не означає, що вона шкідлива. Це означає, що навичка виглядає узгодженою з метою, але може
діяти з суттєвими повноваженнями облікового запису.

## Висновки

Висновки пояснюють, чому було показано певний результат аудиту. Кожен висновок зазвичай містить:

- що це означає
- чому це було позначено
- відповідний вміст навички або Plugin
- рекомендацію

Висновки можуть мати позначки `Info`, `Low`, `Medium`, `High` або `Critical`. Висновки з вищою
серйозністю сильніше впливають на рівень ризику та статус аудиту.

Висновки з низькою впевненістю приховано з публічного зведення аудиту, щоб сторінка
залишалася зосередженою на корисних доказах.

## Що перевіряє ClawHub

ClawHub аудіює надіслані артефакти релізу, зокрема:

- інструкції навички або метадані Plugin
- оголошені змінні середовища та дозволи
- інструкції зі встановлення та метадані пакета
- включені файли та маніфести файлів
- метадані сумісності та можливостей

Головне питання — узгодженість: чи відповідають назва, короткий опис, метадані, запитані
повноваження та фактичний вміст тому, чого користувачі могли б обґрунтовано очікувати?

Потужна поведінка не є автоматично поганою. Багатьом корисним інструментам потрібні облікові дані,
локальні команди, API постачальників або встановлення пакетів. Аудит перевіряє, чи ця
потужність очікувана, розкрита та пропорційна.

Сторінки артефактів посилаються на повний аудит за адресою:

```text
/<owner>/skills/<slug>/security-audit
```

Сторінка аудиту поєднує:

1. SkillSpector
2. VirusTotal
3. аналіз ризику

## VirusTotal

ClawHub використовує VirusTotal як телеметрію шкідливого ПЗ у стеку аудиту. VirusTotal є
довіреним галузевим стандартом для репутації файлів і сканування шкідливого ПЗ, а наше
партнерство дає змогу ClawHub додавати ширшу аналітику безпеки до перевірки навичок і Plugin.

VirusTotal особливо корисний для відомих шкідливих артефактів, спрацювань рушіїв і
сигналів репутації, які доповнюють агентно-обізнану перевірку ClawHub. Коли доступні
кількості спрацювань рушіїв постачальників, аудит підсумовує їх простою мовою, наприклад:

```text
62/62 vendors flagged this skill as clean.
```

або:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

Коли ClawHub не має телеметрії кількості постачальників для підсумку, аудит повідомляє:

```text
No VirusTotal findings
```

VirusTotal залишається телеметрією. Він не замінює власний, обізнаний про артефакти
аналіз ризику ClawHub.

## Аналіз ризику

Аналіз ризику внутрішньо працює на базі ClawScan, власної системи аудиту безпеки
ClawHub. Вона перевіряє кожен реліз як артефакт, звернений до агента: інструкції,
метадані, оголошені дозволи, файли, сигнали можливостей, сигнали статичного сканування,
висновки SkillSpector, телеметрію VirusTotal і контекст, наданий видавцем.
Сигнали статичного сканування є внутрішнім контекстом для цієї перевірки; вони не є
окремим публічним розділом аудиту або вердиктом, що блокує встановлення.

Аналіз ризику використовує
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
як призму для ризиків, таких як ін'єкція підказок, неправильне використання інструментів, розкриття облікових даних,
небезпечне виконання, отруєння пам'яті або контексту та надмірна автономність.

ClawScan не вважає можливість, яка виглядає загрозливо, автоматично шкідливою.
Він запитує, чи можливість розкрита, узгоджена з метою та підтримана
заявленим сценарієм використання релізу.
