# Аудити безпеки

Аудити безпеки ClawHub допомагають вирішити, чи достатньо безпечно
встановлювати навичку або плагін. Вони показують, що робить випуск, які права
він запитує, і чи є щось, що потребує додаткової уваги, перш ніж він отримає
доступ до файлів, облікових записів, облікових даних, коду або зовнішніх сервісів.

Аудити є сильними сигналами безпеки, але вони не гарантують, що випуск не має
ризиків. Завжди зважуйте рішення, перш ніж надавати чутливий доступ.

Див. також [Безпека](/uk/clawhub/security), [Допустиме використання](/uk/clawhub/acceptable-usage)
і [Модерація та безпека облікового запису](/uk/clawhub/moderation).

## Що перевірити перед встановленням

Перед встановленням перегляньте:

- загальний статус аудиту
- рівень ризику
- усі перелічені знахідки
- потрібні облікові дані, дозволи або змінні середовища
- власника, джерело, версію, журнал змін, завантаження, зірки та інші сигнали довіри

Встановлюйте лише той вміст, який ви розумієте і якому довіряєте.

## Статус аудиту

Статус аудиту підказує, як реагувати на результат аудиту:

| Статус      | Значення                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | Не знайдено видимої проблеми з ризиком вище низького.                    |
| `Review`    | Прочитайте знахідки перед встановленням. Випуск усе ще може бути легітимним. |
| `Warn`      | Будьте особливо обережні. ClawHub знайшов проблему з високим впливом або попереджувальний сигнал. |
| `Malicious` | Не встановлюйте.                                                           |
| `Pending`   | Аудити ще не завершено.                                                    |
| `Error`     | Аудит не вдалося завершити.                                                |

`Pass` заспокоює, але не замінює власного судження. Це найважливіше для
інструментів, які можуть публікувати вміст, редагувати дані, виконувати команди,
читати файли або отримувати доступ до виробничих систем.

## Рівень ризику

Рівень ризику описує радіус впливу: скільки влади, схоже, має випуск, якщо
використовувати його за призначенням.

| Рівень ризику | Значення                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | Виявлено мало чутливих повноважень або впливу на користувача.                 |
| `Medium`   | Випуск має суттєві повноваження, як-от доступ до облікового запису або зміни даних. |
| `High`     | Випуск має повноваження з високим впливом, серйозні знахідки або шкідливі сигнали. |

Рівень ризику і статус аудиту відповідають на різні запитання:

- Рівень ризику запитує: "Скільки влади тут є?"
- Статус аудиту запитує: "Що мені робити з цим результатом?"

Наприклад, навичка публікації може показувати `Review` із ризиком `Medium`. Це
не означає, що вона шкідлива. Це означає, що навичка виглядає узгодженою з метою,
але може діяти зі значущими повноваженнями облікового запису.

## Знахідки

Знахідки пояснюють, чому було показано результат аудиту. Кожна знахідка зазвичай містить:

- що вона означає
- чому її було позначено
- відповідний вміст навички або плагіна
- рекомендацію

Знахідки можуть мати мітки `Info`, `Low`, `Medium`, `High` або `Critical`. Знахідки
з вищою серйозністю сильніше впливають на рівень ризику і статус аудиту.

Знахідки з низькою впевненістю приховано з публічного зведення аудиту, щоб
сторінка залишалася зосередженою на корисних доказах.

## Що перевіряє ClawHub

ClawHub аудіює надіслані артефакти випусків, зокрема:

- інструкції навички або метадані плагіна
- оголошені змінні середовища та дозволи
- інструкції встановлення і метадані пакета
- включені файли та маніфести файлів
- метадані сумісності та можливостей

Головне запитання — узгодженість: чи назва, короткий опис, метадані, запитувані
повноваження і фактичний вміст відповідають тому, чого користувачі могли б
розумно очікувати?

Потужна поведінка не є автоматично поганою. Багатьом корисним інструментам
потрібні облікові дані, локальні команди, API провайдерів або встановлення
пакетів. Аудит перевіряє, чи така влада очікувана, розкрита і пропорційна.

Сторінки артефактів посилаються на повний аудит за адресою:

```text
/<owner>/skills/<slug>/security-audit
```

Сторінка аудиту поєднує:

1. SkillSpector
2. VirusTotal
3. Аналіз ризику

## VirusTotal

ClawHub використовує VirusTotal як телеметрію зловмисного ПЗ у стеку аудиту.
VirusTotal є надійним галузевим стандартом для репутації файлів і сканування
зловмисного ПЗ, а наше партнерство дає ClawHub змогу додати ширшу безпекову
аналітику до перевірки навичок і плагінів.

VirusTotal особливо корисний для відомих шкідливих артефактів, спрацювань
рушіїв і репутаційних сигналів, які доповнюють агентно-орієнтовану перевірку
ClawHub. Коли доступні підрахунки рушіїв постачальників, аудит підсумовує їх
простою мовою, наприклад:

```text
62/62 vendors flagged this skill as clean.
```

або:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

Коли ClawHub не має телеметрії підрахунку постачальників для підсумку, аудит каже:

```text
No VirusTotal findings
```

VirusTotal залишається телеметрією. Він не замінює власний аналіз ризиків ClawHub
з урахуванням артефактів.

## Аналіз ризику

Аналіз ризику внутрішньо працює на ClawScan, власній системі аудиту безпеки
ClawHub. Вона переглядає кожен випуск як артефакт, призначений для агента:
інструкції, метадані, оголошені дозволи, файли, сигнали можливостей, сигнали
статичного сканування, знахідки SkillSpector, телеметрію VirusTotal і контекст,
наданий видавцем. Сигнали статичного сканування є внутрішнім контекстом для цієї
перевірки; вони не є окремим публічним розділом аудиту або вердиктом, що блокує
встановлення.

Аналіз ризику використовує
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
як призму для ризиків, як-от prompt injection, неправильне використання
інструментів, розкриття облікових даних, небезпечне виконання, отруєння пам’яті
або контексту та надмірна автономність.

ClawScan не вважає можливість, яка виглядає загрозливо, автоматично шкідливою.
Він запитує, чи можливість розкрита, узгоджена з метою і підтримана заявленим
сценарієм використання випуску.
