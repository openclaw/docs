# Аудити безпеки

Аудити безпеки ClawHub допомагають вирішити, чи достатньо безпечно встановлювати skill або plugin. Вони показують, що робить реліз, які повноваження він запитує, і чи є щось, що потребує додаткової уваги, перш ніж він отримає доступ до файлів, облікових записів, облікових даних, коду або зовнішніх сервісів.

Аудити є сильними сигналами безпеки, але вони не гарантують, що реліз не має ризиків. Завжди керуйтеся власним судженням, перш ніж надавати чутливий доступ.

Див. також [Безпека](/clawhub/security), [Допустиме використання](/clawhub/acceptable-usage) і [Модерація та безпека облікового запису](/clawhub/moderation).

## Що перевірити перед встановленням

Перед встановленням перегляньте:

- загальний статус аудиту
- рівень ризику
- усі перелічені findings
- потрібні облікові дані, дозволи або змінні середовища
- власника, джерело, версію, журнал змін, завантаження, зірки та інші сигнали довіри

Встановлюйте лише той вміст, який ви розумієте і якому довіряєте.

## Статус аудиту

Статус аудиту підказує, як реагувати на результат аудиту:

| Статус      | Значення                                                                   |
| ----------- | -------------------------------------------------------------------------- |
| `Pass`      | Не знайдено видимих проблем із ризиком вище низького.                      |
| `Review`    | Прочитайте findings перед встановленням. Реліз усе ще може бути легітимним. |
| `Warn`      | Будьте особливо обережні. ClawHub знайшов занепокоєння з великим впливом або попереджувальний сигнал. |
| `Malicious` | Не встановлюйте.                                                           |
| `Pending`   | Аудити ще не завершено.                                                    |
| `Error`     | Аудит не вдалося завершити.                                                |

`Pass` заспокоює, але не замінює вашого власного судження. Це найважливіше для інструментів, які можуть публікувати вміст, редагувати дані, виконувати команди, читати файли або отримувати доступ до продакшн-систем.

## Рівень ризику

Рівень ризику описує радіус впливу: скільки влади реліз, схоже, має, якщо використовувати його за призначенням.

| Рівень ризику | Значення                                                                       |
| ------------- | ------------------------------------------------------------------------------ |
| `Low`         | Виявлено мало чутливих повноважень або впливу на користувача.                  |
| `Medium`      | Реліз має значущі повноваження, як-от доступ до облікового запису або зміни даних. |
| `High`        | Реліз має повноваження з великим впливом, серйозні findings або шкідливі сигнали. |

Рівень ризику і статус аудиту відповідають на різні запитання:

- Рівень ризику питає: "Скільки тут влади?"
- Статус аудиту питає: "Що мені робити з цим результатом?"

Наприклад, publishing skill може показувати `Review` із ризиком `Medium`. Це не означає, що він шкідливий. Це означає, що skill, схоже, відповідає своїй меті, але може діяти зі значущими повноваженнями облікового запису.

## Findings

Findings пояснюють, чому було показано результат аудиту. Кожен finding зазвичай містить:

- що це означає
- чому це було позначено
- відповідний вміст skill або plugin
- рекомендацію

Findings можуть мати мітки `Info`, `Low`, `Medium`, `High` або `Critical`. Findings з вищою серйозністю сильніше впливають на рівень ризику і статус аудиту.

Findings із низькою впевненістю приховано з публічного підсумку аудиту, щоб сторінка зосереджувалася на корисних доказах.

## Що перевіряє ClawHub

ClawHub проводить аудит надісланих артефактів релізу, зокрема:

- інструкцій skill або метаданих plugin
- оголошених змінних середовища та дозволів
- інструкцій встановлення та метаданих пакета
- включених файлів і маніфестів файлів
- метаданих сумісності та можливостей

Головне запитання — узгодженість: чи відповідають назва, короткий опис, метадані, запитані повноваження і фактичний вміст тому, чого користувачі могли б обґрунтовано очікувати?

Потужна поведінка не є автоматично поганою. Багатьом корисним інструментам потрібні облікові дані, локальні команди, API провайдерів або встановлення пакетів. Аудит перевіряє, чи така влада очікувана, розкрита і пропорційна.

Сторінки артефактів містять посилання на повний аудит за адресою:

```text
/<owner>/skills/<slug>/security-audit
```

Сторінка аудиту поєднує:

1. SkillSpector
2. VirusTotal
3. Аналіз ризиків

## VirusTotal

ClawHub використовує VirusTotal як телеметрію шкідливого ПЗ у стеку аудиту. VirusTotal є довіреним галузевим стандартом для репутації файлів і сканування шкідливого ПЗ, а наше партнерство дає ClawHub змогу додавати ширшу розвідку безпеки до перевірки skill і plugin.

VirusTotal особливо корисний для відомих шкідливих артефактів, спрацьовувань рушіїв і сигналів репутації, які доповнюють агентно-орієнтовану перевірку ClawHub. Коли доступні підрахунки рушіїв постачальників, аудит підсумовує їх простою мовою, наприклад:

```text
62/62 vendors flagged this skill as clean.
```

або:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

Коли ClawHub не має телеметрії підрахунку постачальників для підсумку, аудит повідомляє:

```text
No VirusTotal findings
```

VirusTotal залишається телеметрією. Він не замінює власний артефактно-орієнтований аналіз ризиків ClawHub.

## Аналіз ризиків

Аналіз ризиків внутрішньо працює на ClawScan, власній системі аудиту безпеки ClawHub. Вона перевіряє кожен реліз як артефакт, орієнтований на агента: інструкції, метадані, оголошені дозволи, файли, сигнали можливостей, сигнали статичного сканування, findings SkillSpector, телеметрію VirusTotal і контекст, наданий видавцем. Сигнали статичного сканування є внутрішнім контекстом для цієї перевірки; вони не є окремим публічним розділом аудиту або вердиктом, що блокує встановлення.

Аналіз ризиків використовує [OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/) як призму для ризиків, таких як ін’єкція промптів, неправильне використання інструментів, розкриття облікових даних, небезпечне виконання, отруєння пам’яті або контексту та надмірна автономність.

ClawScan не вважає можливість, що виглядає лячно, автоматично шкідливою. Він запитує, чи ця можливість розкрита, відповідає меті та підтримується заявленим сценарієм використання релізу.
