# Аудити безпеки

Аудити безпеки ClawHub допомагають вирішити, чи skill або Plugin достатньо безпечні
для встановлення. Вони показують, що робить реліз, які повноваження він запитує та
чи є щось, що потребує додаткової уваги, перш ніж він отримає доступ до файлів, облікових записів,
облікових даних, коду або зовнішніх сервісів.

Аудити є сильними сигналами безпеки, але вони не гарантують, що реліз
не має ризиків. Завжди користуйтеся власним судженням, перш ніж надавати чутливий доступ.

Див. також [Безпека](/clawhub/security), [Допустиме використання](/clawhub/acceptable-usage)
та [Модерація й безпека облікового запису](/clawhub/moderation).

## Що перевірити перед встановленням

Перед встановленням перегляньте:

- загальний статус аудиту
- рівень ризику
- усі перелічені висновки
- потрібні облікові дані, дозволи або змінні середовища
- власника, джерело, версію, журнал змін, завантаження, зірки та інші сигнали довіри

Встановлюйте лише той вміст, який ви розумієте і якому довіряєте.

## Статус аудиту

Статус аудиту підказує, як реагувати на результат аудиту:

| Статус      | Значення                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | Не знайдено видимих проблем вище низького ризику.                                |
| `Review`    | Прочитайте висновки перед встановленням. Реліз усе ще може бути легітимним. |
| `Warn`      | Будьте особливо обережні. ClawHub знайшов проблему з високим впливом або попереджувальний сигнал. |
| `Malicious` | Не встановлюйте.                                                           |
| `Pending`   | Аудити ще не завершено.                                             |
| `Error`     | Аудит не вдалося завершити.                                         |

`Pass` заспокоює, але не замінює вашого власного судження. Це особливо важливо
для інструментів, які можуть публікувати вміст, редагувати дані, виконувати команди, читати файли або
отримувати доступ до виробничих систем.

## Рівень ризику

Рівень ризику описує радіус впливу: скільки повноважень, схоже, має реліз, якщо
використовувати його за призначенням.

| Рівень ризику | Значення                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | Виявлено мало чутливих повноважень або впливу на користувача.                          |
| `Medium`   | Реліз має суттєві повноваження, як-от доступ до облікового запису або зміни даних. |
| `High`     | Реліз має повноваження з високим впливом, серйозні висновки або шкідливі сигнали. |

Рівень ризику й статус аудиту відповідають на різні запитання:

- Рівень ризику запитує: "Скільки тут повноважень?"
- Статус аудиту запитує: "Що мені робити з цим результатом?"

Наприклад, навичка для публікації може показувати `Review` із ризиком `Medium`. Це
не означає, що вона шкідлива. Це означає, що навичка, схоже, відповідає заявленій меті, але може
діяти зі значущими повноваженнями облікового запису.

## Висновки

Висновки пояснюють, чому було показано результат аудиту. Кожен висновок зазвичай містить:

- що він означає
- чому його позначено
- відповідний вміст skill або Plugin
- рекомендацію

Висновки можуть мати мітки `Info`, `Low`, `Medium`, `High` або `Critical`. Висновки з вищою
серйозністю сильніше впливають на рівень ризику та статус аудиту.

Висновки з низькою впевненістю приховані з публічного підсумку аудиту, щоб сторінка
залишалася зосередженою на корисних доказах.

## Що перевіряє ClawHub

ClawHub аудіює надіслані артефакти релізу, зокрема:

- інструкції skill або метадані Plugin
- оголошені змінні середовища та дозволи
- інструкції зі встановлення та метадані пакета
- включені файли та маніфести файлів
- метадані сумісності й можливостей

Головне запитання — узгодженість: чи відповідають назва, підсумок, метадані, запитані
повноваження та фактичний вміст тому, чого користувачі розумно очікували б?

Потужна поведінка не є автоматично поганою. Багатьом корисним інструментам потрібні облікові дані,
локальні команди, API провайдерів або встановлення пакетів. Аудит перевіряє, чи така
сила очікувана, розкрита й пропорційна.

Сторінки артефактів посилаються на повний аудит за адресою:

```text
/<owner>/skills/<slug>/security-audit
```

Сторінка аудиту поєднує:

1. SkillSpector
2. VirusTotal
3. Аналіз ризику

## VirusTotal

ClawHub використовує VirusTotal як телеметрію шкідливого ПЗ у стеку аудиту. VirusTotal є
довіреним галузевим стандартом для репутації файлів і сканування шкідливого ПЗ, а наше
партнерство дає ClawHub змогу додавати ширшу безпекову аналітику до перевірки skill і Plugin.

VirusTotal особливо корисний для відомих шкідливих артефактів, спрацювань рушіїв і
репутаційних сигналів, які доповнюють агентно-обізнану перевірку ClawHub. Коли доступні
підрахунки рушіїв постачальників, аудит підсумовує їх простою мовою, наприклад:

```text
62/62 vendors flagged this skill as clean.
```

або:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

Коли ClawHub не має телеметрії з підрахунками постачальників для підсумку, аудит повідомляє:

```text
No VirusTotal findings
```

VirusTotal залишається телеметрією. Він не замінює власний артефактно-обізнаний
аналіз ризику ClawHub.

## Аналіз ризику

Аналіз ризику внутрішньо працює на ClawScan, власній системі аудиту безпеки
ClawHub. Вона переглядає кожен реліз як артефакт, призначений для агента: інструкції,
метадані, оголошені дозволи, файли, сигнали можливостей, сигнали статичного сканування,
висновки SkillSpector, телеметрію VirusTotal і контекст, наданий видавцем.
Сигнали статичного сканування є внутрішнім контекстом для цієї перевірки; вони не є
окремим публічним розділом аудиту або вердиктом, що блокує встановлення.

Аналіз ризику використовує
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
як призму для ризиків, як-от prompt injection, неправильне використання інструментів, розкриття облікових даних,
небезпечне виконання, отруєння пам’яті або контексту та надмірна автономність.

ClawScan не вважає можливість, яка виглядає загрозливо, автоматично шкідливою.
Він запитує, чи можливість розкрита, відповідає меті та підтримується
заявленим сценарієм використання релізу.
