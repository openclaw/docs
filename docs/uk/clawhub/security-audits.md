# Аудити безпеки

Аудити безпеки ClawHub допомагають вирішити, чи є skill або plugin достатньо безпечним
для встановлення. Вони показують, що робить реліз, які повноваження він запитує та
чи є щось, що потребує додаткової уваги, перш ніж він зможе отримати доступ до файлів, облікових записів,
облікових даних, коду або зовнішніх сервісів.

Аудити є сильними сигналами безпеки, але вони не гарантують, що реліз
не має ризиків. Завжди керуйтеся власним судженням, перш ніж надавати чутливий доступ.

Див. також [Безпека](/clawhub/security), [Допустиме використання](/clawhub/acceptable-usage)
та [Модерація і безпека облікового запису](/clawhub/moderation).

## Що перевірити перед встановленням

Перед встановленням перегляньте:

- загальний статус аудиту
- рівень ризику
- усі перелічені знахідки
- потрібні облікові дані, дозволи або змінні середовища
- власника, джерело, версію, журнал змін, завантаження, зірки та інші сигнали довіри

Встановлюйте лише той вміст, який ви розумієте і якому довіряєте.

## Статус аудиту

Статус аудиту підказує, як реагувати на результат аудиту:

| Статус      | Значення                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | Не знайдено видимих проблем вище низького ризику.                                |
| `Review`    | Прочитайте знахідки перед встановленням. Реліз усе ще може бути легітимним. |
| `Warn`      | Будьте особливо обережні. ClawHub знайшов проблему з великим впливом або попереджувальний сигнал. |
| `Malicious` | Не встановлюйте.                                                           |
| `Pending`   | Аудити ще не завершено.                                             |
| `Error`     | Аудит не вдалося завершити.                                         |

`Pass` заспокоює, але не замінює вашого власного судження. Це найважливіше
для інструментів, які можуть публікувати вміст, редагувати дані, виконувати команди, читати файли або
отримувати доступ до продакшн-систем.

## Рівень ризику

Рівень ризику описує радіус ураження: скільки влади, схоже, має реліз, якщо
ви використовуєте його за призначенням.

| Рівень ризику | Значення                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | Виявлено мало чутливих повноважень або впливу на користувача.                          |
| `Medium`   | Реліз має суттєві повноваження, як-от доступ до облікового запису або зміни даних. |
| `High`     | Реліз має повноваження з великим впливом, серйозні знахідки або зловмисні сигнали. |

Рівень ризику і статус аудиту відповідають на різні запитання:

- Рівень ризику запитує: «Скільки тут влади?»
- Статус аудиту запитує: «Що мені робити з цим результатом?»

Наприклад, skill для публікації може показувати `Review` із ризиком `Medium`. Це
не означає, що він зловмисний. Це означає, що skill виглядає узгодженим із призначенням, але може
діяти з суттєвими повноваженнями облікового запису.

## Знахідки

Знахідки пояснюють, чому було показано результат аудиту. Кожна знахідка зазвичай містить:

- що це означає
- чому це було позначено
- відповідний вміст skill або plugin
- рекомендацію

Знахідки можуть мати мітки `Info`, `Low`, `Medium`, `High` або `Critical`. Знахідки з вищою
серйозністю сильніше впливають на рівень ризику та статус аудиту.

Знахідки з низькою впевненістю приховані з публічного підсумку аудиту, щоб сторінка
залишалася зосередженою на корисних доказах.

## Що перевіряє ClawHub

ClawHub аудіює надіслані артефакти релізу, зокрема:

- інструкції skill або метадані plugin
- оголошені змінні середовища та дозволи
- інструкції зі встановлення та метадані пакета
- включені файли та маніфести файлів
- метадані сумісності та можливостей

Головне питання - узгодженість: чи відповідають назва, підсумок, метадані, запитані
повноваження та фактичний вміст тому, чого користувачі обґрунтовано очікували б?

Потужна поведінка не є автоматично поганою. Багатьом корисним інструментам потрібні облікові дані,
локальні команди, API провайдерів або встановлення пакетів. Аудит перевіряє, чи ця
сила очікувана, розкрита та пропорційна.

Сторінки артефактів посилаються на повний аудит за адресою:

```text
/<owner>/skills/<slug>/security-audit
```

Сторінка аудиту поєднує:

1. SkillSpector
2. VirusTotal
3. Аналіз ризиків

## VirusTotal

ClawHub використовує VirusTotal як телеметрію шкідливого ПЗ у стеку аудиту. VirusTotal є
надійним галузевим стандартом для репутації файлів і сканування шкідливого ПЗ, а наше
партнерство дає ClawHub змогу додавати ширшу безпекову аналітику до перевірки skill і plugin.

VirusTotal особливо корисний для відомих зловмисних артефактів, спрацювань рушіїв і
репутаційних сигналів, які доповнюють агентно-обізнану перевірку ClawHub. Коли доступні
підрахунки рушіїв постачальників, аудит підсумовує їх простою мовою, наприклад:

```text
62/62 vendors flagged this skill as clean.
```

або:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

Коли ClawHub не має телеметрії підрахунку постачальників для підсумування, аудит повідомляє:

```text
No VirusTotal findings
```

VirusTotal залишається телеметрією. Він не замінює власний, обізнаний про артефакти
аналіз ризику ClawHub.

## Аналіз ризиків

Аналіз ризиків внутрішньо працює на ClawScan, власній системі аудиту безпеки ClawHub.
Він перевіряє кожен реліз як артефакт, звернений до агента: інструкції,
метадані, оголошені дозволи, файли, сигнали можливостей, сигнали статичного сканування,
знахідки SkillSpector, телеметрію VirusTotal і контекст, наданий видавцем.
Сигнали статичного сканування є внутрішнім контекстом для цієї перевірки; вони не є
окремим публічним розділом аудиту або вердиктом, що блокує встановлення.

Аналіз ризиків використовує
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
як призму для ризиків, як-от інʼєкція підказок, неправильне використання інструментів, розкриття облікових даних,
небезпечне виконання, отруєння памʼяті або контексту та надмірна агентність.

ClawScan не вважає можливість, що виглядає лячно, автоматично зловмисною.
Він запитує, чи можливість розкрита, узгоджена з призначенням і підтримана
заявленим сценарієм використання релізу.
