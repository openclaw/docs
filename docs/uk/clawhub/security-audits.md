# Аудити безпеки

Аудити безпеки ClawHub допомагають вирішити, чи є навичка або Plugin достатньо безпечними
для встановлення. Вони показують, що робить реліз, які повноваження він запитує і
чи є щось, що потребує додаткової уваги, перш ніж він отримає доступ до файлів, облікових записів,
облікових даних, коду або зовнішніх сервісів.

Аудити є сильними сигналами безпеки, але вони не гарантують, що реліз
не має ризиків. Завжди користуйтеся власним судженням, перш ніж надавати чутливий доступ.

Див. також [Безпека](/clawhub/security), [Прийнятне використання](/uk/clawhub/acceptable-usage)
і [Модерація та безпека облікового запису](/clawhub/moderation).

## Що перевірити перед встановленням

Перед встановленням перегляньте:

- загальний статус аудиту
- рівень ризику
- усі перелічені висновки
- потрібні облікові дані, дозволи або змінні середовища
- власника, джерело, версію, changelog, завантаження, зірки та інші сигнали довіри

Встановлюйте лише той вміст, який ви розумієте і якому довіряєте.

## Статус аудиту

Статус аудиту підказує, як реагувати на результат аудиту:

| Статус      | Значення                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | Видимих проблем вище низького ризику не знайдено.                         |
| `Review`    | Прочитайте висновки перед встановленням. Реліз усе ще може бути легітимним. |
| `Warn`      | Будьте особливо обережні. ClawHub знайшов проблему з великим впливом або попереджувальний сигнал. |
| `Malicious` | Не встановлюйте.                                                           |
| `Pending`   | Аудити ще не завершено.                                                    |
| `Error`     | Аудит не вдалося завершити.                                                |

`Pass` заспокоює, але не замінює вашого власного судження. Це найважливіше
для інструментів, які можуть публікувати вміст, редагувати дані, виконувати команди, читати файли або
отримувати доступ до production-систем.

## Рівень ризику

Рівень ризику описує радіус ураження: скільки повноважень реліз, схоже, має, якщо
ви використовуєте його за призначенням.

| Рівень ризику | Значення                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | Виявлено мало чутливих повноважень або впливу на користувача.                 |
| `Medium`   | Реліз має значущі повноваження, як-от доступ до облікового запису або зміни даних. |
| `High`     | Реліз має повноваження з великим впливом, серйозні висновки або зловмисні сигнали. |

Рівень ризику і статус аудиту відповідають на різні запитання:

- Рівень ризику запитує: "Скільки тут повноважень?"
- Статус аудиту запитує: "Що мені робити з цим результатом?"

Наприклад, навичка для публікації може показувати `Review` із ризиком `Medium`. Це
не означає, що вона зловмисна. Це означає, що навичка, схоже, відповідає своїй меті, але може
діяти зі значущими повноваженнями облікового запису.

## Висновки

Висновки пояснюють, чому було показано результат аудиту. Кожен висновок зазвичай містить:

- що це означає
- чому це було позначено
- відповідний вміст навички або Plugin
- рекомендацію

Висновки можуть мати позначки `Info`, `Low`, `Medium`, `High` або `Critical`. Висновки з вищою
серйозністю сильніше впливають на рівень ризику та статус аудиту.

Висновки з низькою впевненістю приховано з публічного зведення аудиту, щоб сторінка
зосереджувалася на корисних доказах.

## Що перевіряє ClawHub

ClawHub аудіює надіслані артефакти релізу, зокрема:

- інструкції навички або метадані Plugin
- оголошені змінні середовища та дозволи
- інструкції встановлення і метадані пакета
- включені файли та маніфести файлів
- метадані сумісності та можливостей

Головне питання — узгодженість: чи назва, короткий опис, метадані, запитані
повноваження і фактичний вміст відповідають тому, чого користувачі могли б обґрунтовано очікувати?

Потужна поведінка не є автоматично поганою. Багатьом корисним інструментам потрібні облікові дані,
локальні команди, API провайдерів або встановлення пакетів. Аудит перевіряє, чи така
потужність є очікуваною, розкритою і пропорційною.

Сторінки артефактів посилаються на повний аудит за адресою:

```text
/<owner>/skills/<slug>/security-audit
```

Сторінка аудиту поєднує:

1. SkillSpector
2. VirusTotal
3. Аналіз ризиків

## VirusTotal

ClawHub використовує VirusTotal як телеметрію шкідливого ПЗ у стеку аудиту. VirusTotal є
довіреним галузевим стандартом для репутації файлів і сканування шкідливого ПЗ, а наше
партнерство дає ClawHub змогу додавати ширшу безпекову аналітику до перевірки навичок і Plugin.

VirusTotal особливо корисний для відомих зловмисних артефактів, спрацювань рушіїв і
репутаційних сигналів, які доповнюють agent-aware перевірку ClawHub. Коли доступні
кількості спрацювань рушіїв постачальників, аудит підсумовує їх простою мовою, наприклад:

```text
62/62 vendors flagged this skill as clean.
```

або:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

Коли ClawHub не має телеметрії кількості постачальників для підсумовування, аудит повідомляє:

```text
No VirusTotal findings
```

VirusTotal залишається телеметрією. Він не замінює власний artifact-aware
аналіз ризиків ClawHub.

## Аналіз ризиків

Аналіз ризиків внутрішньо працює на ClawScan, власній системі аудиту безпеки
ClawHub. Він перевіряє кожен реліз як артефакт, призначений для агента: інструкції,
метадані, оголошені дозволи, файли, сигнали можливостей, сигнали статичного сканування,
висновки SkillSpector, телеметрію VirusTotal і контекст, наданий видавцем.
Сигнали статичного сканування є внутрішнім контекстом для цієї перевірки; вони не є
окремим публічним розділом аудиту або вердиктом, що блокує встановлення.

Аналіз ризиків використовує
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
як призму для ризиків, таких як prompt injection, неправильне використання інструментів, розкриття облікових даних,
небезпечне виконання, отруєння пам’яті або контексту та надмірна агентність.

ClawScan не вважає можливість зі страшним виглядом автоматично зловмисною.
Він запитує, чи можливість розкрита, відповідає меті та підтримується
заявленим сценарієм використання релізу.
