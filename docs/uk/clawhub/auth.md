# Автентифікація

ClawHub використовує GitHub для входу у вебінтерфейс. CLI використовує API-токени ClawHub, створені
через цей обліковий запис після входу.

## Вхід у вебінтерфейс

Використовуйте GitHub, щоб увійти на [clawhub.ai](https://clawhub.ai).

Видалені, забанені або вимкнені облікові записи не можуть завершити звичайний вхід у ClawHub.
Якщо після входу вас повертає до стану без входу, ваш обліковий запис може бути не в належному
стані.

## Вхід у CLI

Стандартний потік входу в CLI відкриває ваш браузер:

```bash
clawhub login
clawhub whoami
```

Що відбувається:

1. CLI запускає тимчасовий сервер зворотного виклику на `127.0.0.1`.
2. Ваш браузер відкриває сторінку входу ClawHub.
3. Після входу через GitHub ClawHub створює API-токен.
4. Браузер перенаправляє назад до локального зворотного виклику.
5. CLI зберігає токен у вашому файлі конфігурації ClawHub.

Якщо ваш браузер не може дістатися локального зворотного виклику через правила брандмауера, VPN або
проксі, використовуйте потік токена без браузера.

## Вхід без браузера

Створіть токен у вебінтерфейсі ClawHub, а потім передайте його в CLI:

```bash
clawhub login --token clh_...
```

Використовуйте цей потік для серверів, завдань CI або середовищ лише з терміналом.

Для віддалених оболонок, де ви можете відкрити браузер в іншому місці, виконайте:

```bash
clawhub login --device
```

CLI виводить одноразовий код і чекає, доки ви авторизуєте його на
`https://clawhub.ai/cli/device`.

## Зберігання токенів

Стандартні шляхи конфігурації:

- macOS: `~/Library/Application Support/clawhub/config.json`
- Linux/XDG: `$XDG_CONFIG_HOME/clawhub/config.json` або `~/.config/clawhub/config.json`
- Windows: `%APPDATA%\\clawhub\\config.json`

Перевизначте шлях за допомогою:

```bash
export CLAWHUB_CONFIG_PATH=/path/to/config.json
```

## Відкликання

Ви можете відкликати API-токени у вебінтерфейсі ClawHub.

Відкликані, недійсні або відсутні токени повертають `401 Unauthorized`. Увійдіть знову
за допомогою `clawhub login` або надайте новий токен за допомогою `clawhub login --token`.

Видалені, забанені або вимкнені облікові записи не можуть продовжувати використовувати наявні API-токени.
