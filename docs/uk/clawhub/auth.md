# Автентифікація

ClawHub використовує GitHub для входу через вебінтерфейс. CLI використовує API-токени ClawHub, створені
через обліковий запис, у який виконано вхід.

## Вхід через вебінтерфейс

Використовуйте GitHub, щоб увійти на [clawhub.ai](https://clawhub.ai).

Видалені, заблоковані або вимкнені облікові записи не можуть завершити звичайний вхід у ClawHub.
Якщо після входу ви повертаєтеся до стану без авторизації, можливо, ваш обліковий запис
має обмеження. Якщо ваш обліковий запис заблоковано або вимкнено, скористайтеся
[формою оскарження ClawHub](https://appeals.openclaw.ai/), якщо вважаєте це
помилкою.

## Вхід через CLI

Стандартний процес входу через CLI відкриває браузер:

```bash
clawhub login
clawhub whoami
```

Що відбувається:

1. CLI запускає тимчасовий сервер зворотного виклику на `127.0.0.1`.
2. У браузері відкривається сторінка входу ClawHub.
3. Після входу через GitHub ClawHub створює API-токен.
4. Браузер переспрямовує запит назад до локального зворотного виклику.
5. CLI зберігає токен у вашому файлі конфігурації ClawHub.

Якщо браузер не може звернутися до локального зворотного виклику через правила брандмауера, VPN або
проксі-сервера, скористайтеся процесом входу за токеном без графічного інтерфейсу.

## Вхід без графічного інтерфейсу

Створіть токен у вебінтерфейсі ClawHub, а потім передайте його до CLI:

```bash
clawhub login --token clh_...
```

Використовуйте цей спосіб для серверів, завдань CI або середовищ, доступних лише через термінал.

Для віддалених оболонок, де браузер можна відкрити на іншому пристрої, виконайте:

```bash
clawhub login --device
```

CLI виведе одноразовий код і чекатиме, доки ви авторизуєте його на
`https://clawhub.ai/cli/device`.

## Зберігання токенів

Стандартні шляхи до конфігурації:

- macOS: `~/Library/Application Support/clawhub/config.json`
- Linux/XDG: `$XDG_CONFIG_HOME/clawhub/config.json` або `~/.config/clawhub/config.json`
- Windows: `%APPDATA%\\clawhub\\config.json`

Перевизначте шлях за допомогою:

```bash
export CLAWHUB_CONFIG_PATH=/path/to/config.json
```

Виведіть збережений токен для налаштування CI за допомогою:

```bash
clawhub token
```

## Відкликання

Ви можете відкликати API-токени у вебінтерфейсі ClawHub.

Відкликані, недійсні або відсутні токени повертають `401 Unauthorized`. Увійдіть знову
за допомогою `clawhub login` або надайте новий токен за допомогою `clawhub login --token`.

Видалені, заблоковані або вимкнені облікові записи не можуть надалі використовувати наявні API-токени.
Якщо ваш обліковий запис заблоковано або вимкнено, скористайтеся
[формою оскарження ClawHub](https://appeals.openclaw.ai/), якщо вважаєте це
помилкою.
