# Дата й час

OpenClaw типово використовує **локальний час хоста для часових міток транспорту** і **часовий пояс користувача лише в system prompt**.
Часові мітки провайдера зберігаються, щоб tools зберігали свою рідну семантику (поточний час доступний через `session_status`).

## Envelopes повідомлень (локально за замовчуванням)

Вхідні повідомлення обгортаються часовою міткою (точність до хвилини):

```
[Provider ... 2026-01-05 16:26 PST] message text
```

Ця часова мітка envelope **типово є локальною для хоста**, незалежно від часового поясу провайдера.

Ви можете перевизначити цю поведінку:

```json5
{
  agents: {
    defaults: {
      envelopeTimezone: "local", // "utc" | "local" | "user" | IANA timezone
      envelopeTimestamp: "on", // "on" | "off"
      envelopeElapsed: "on", // "on" | "off"
    },
  },
}
```

- `envelopeTimezone: "utc"` використовує UTC.
- `envelopeTimezone: "local"` використовує часовий пояс хоста.
- `envelopeTimezone: "user"` використовує `agents.defaults.userTimezone` (резервно використовується часовий пояс хоста).
- Використовуйте явний часовий пояс IANA (наприклад, `"America/Chicago"`) для фіксованої зони.
- `envelopeTimestamp: "off"` прибирає абсолютні часові мітки із заголовків envelope.
- `envelopeElapsed: "off"` прибирає суфікси минулого часу (стиль `+2m`).

### Приклади

**Локально (типово):**

```
[WhatsApp +1555 2026-01-18 00:19 PST] hello
```

**Часовий пояс користувача:**

```
[WhatsApp +1555 2026-01-18 00:19 CST] hello
```

**Із увімкненим минулим часом:**

```
[WhatsApp +1555 +30s 2026-01-18T05:19Z] follow-up
```

## System prompt: Current Date & Time

Якщо часовий пояс користувача відомий, system prompt містить окремий
розділ **Current Date & Time** лише з **часовим поясом** (без формату годинника/часу),
щоб зберегти стабільність кешування prompt:

```
Time zone: America/Chicago
```

Коли агенту потрібен поточний час, використовуйте інструмент `session_status`; картка status
містить рядок із часовою міткою.

## Рядки системних подій (локально за замовчуванням)

Системні події з черги, які вставляються в контекст агента, мають префікс із часовою міткою, що використовує
той самий вибір часового поясу, що й envelopes повідомлень (типово: локальний час хоста).

```
System: [2026-01-12 12:19:17 PST] Model switched.
```

### Налаштування часового поясу користувача + формату

```json5
{
  agents: {
    defaults: {
      userTimezone: "America/Chicago",
      timeFormat: "auto", // auto | 12 | 24
    },
  },
}
```

- `userTimezone` задає **локальний часовий пояс користувача** для контексту prompt.
- `timeFormat` керує **форматом 12h/24h** у prompt. `auto` слідує параметрам ОС.

## Визначення формату часу (auto)

Коли `timeFormat: "auto"`, OpenClaw перевіряє налаштування ОС (macOS/Windows)
і резервно використовує форматування локалі. Визначене значення **кешується на процес**
щоб уникати повторних системних викликів.

## Payload tools + connectors (сирий час провайдера + нормалізовані поля)

Інструменти каналів повертають **часові мітки, рідні для провайдера**, і додають нормалізовані поля для узгодженості:

- `timestampMs`: мілісекунди епохи (UTC)
- `timestampUtc`: рядок ISO 8601 UTC

Сирі поля провайдера зберігаються, щоб нічого не втрачалося.

- Slack: рядки в стилі epoch з API
- Discord: часові мітки UTC ISO
- Telegram/WhatsApp: числові/ISO часові мітки, специфічні для провайдера

Якщо вам потрібен локальний час, перетворюйте його далі, використовуючи відомий часовий пояс.
