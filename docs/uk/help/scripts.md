`scripts/` містить допоміжні скрипти для локальних робочих процесів і операційних завдань. Використовуйте їх, коли завдання явно пов’язане зі скриптом; в інших випадках віддавайте перевагу CLI.

## Умовні позначення

- Скрипти є **необов’язковими**, якщо на них немає посилань у документації чи контрольних списках випуску.
- Віддавайте перевагу інтерфейсам CLI, коли вони доступні (наприклад: `openclaw models status --check`).
- Вважайте скрипти залежними від хоста; прочитайте їх перед запуском на новій машині.

## Скрипти моніторингу автентифікації

Загальну автентифікацію моделей описано в розділі [Автентифікація](/uk/gateway/authentication). Наведені нижче скрипти є окремою необов’язковою системою для моніторингу **токена підписки Claude Code CLI** на віддаленому хості або хості без графічного інтерфейсу та повторної автентифікації з телефона:

- `scripts/setup-auth-system.sh` — одноразове налаштування: перевіряє поточну автентифікацію, допомагає створити довгостроковий `claude setup-token` і виводить кроки встановлення systemd/Termux.
- `scripts/claude-auth-status.sh [full|json|simple]` — перевіряє стан автентифікації Claude Code і OpenClaw.
- `scripts/auth-monitor.sh` — періодично перевіряє стан і надсилає сповіщення (через надсилання OpenClaw та/або ntfy.sh), коли наближається завершення терміну дії токена. Змінні середовища: `WARN_HOURS` (типове значення `2`), `NOTIFY_PHONE`, `NOTIFY_NTFY`. Запускайте за розкладом за допомогою доданих `scripts/systemd/openclaw-auth-monitor.{service,timer}` (кожні 30 хвилин).
- `scripts/mobile-reauth.sh` — повторно запускає `claude setup-token` і виводить URL-адреси, які потрібно відкрити на телефоні, для використання через SSH із Termux.
- `scripts/termux-quick-auth.sh`, `scripts/termux-auth-widget.sh`, `scripts/termux-sync-widget.sh` — скрипти Termux:Widget, які підключаються до хоста через SSH, показують спливне повідомлення про стан і відкривають консоль або інструкції для повторної автентифікації, коли термін дії автентифікації минув.

## Допоміжний скрипт читання GitHub

Використовуйте `scripts/gh-read`, коли потрібно, щоб `gh` використовував токен інсталяції GitHub App для викликів читання в межах репозиторію, водночас залишаючи звичайний `gh` із вашим особистим входом для операцій запису.

Обов’язкові змінні середовища:

- `OPENCLAW_GH_READ_APP_ID`
- `OPENCLAW_GH_READ_PRIVATE_KEY_FILE`

Необов’язкові змінні середовища:

- `OPENCLAW_GH_READ_INSTALLATION_ID`, якщо потрібно пропустити пошук інсталяції на основі репозиторію
- `OPENCLAW_GH_READ_PERMISSIONS` як перевизначення запитуваної підмножини дозволів на читання у вигляді списку, розділеного комами

Порядок визначення репозиторію:

- `gh ... -R owner/repo`
- `GH_REPO`
- `git remote origin`

Приклади:

- `scripts/gh-read pr view 123`
- `scripts/gh-read run list -R openclaw/openclaw`
- `scripts/gh-read api repos/openclaw/openclaw/pulls/123`

## Під час додавання скриптів

- Додайте короткий запис до відповідного документа (або створіть його, якщо він відсутній).
