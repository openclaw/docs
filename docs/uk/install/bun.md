<Warning>
Bun не може запускати CLI або Gateway OpenClaw, оскільки не надає необхідний API `node:sqlite`. Установіть підтримувану версію Node для всіх команд середовища виконання OpenClaw.
</Warning>

Bun можна й надалі використовувати як необов’язковий інсталятор залежностей і засіб запуску скриптів пакетів. Стандартним менеджером пакетів залишається `pnpm`, який повністю підтримується та використовується інструментами документації. Bun не може використовувати `pnpm-lock.yaml` та ігнорує його.

## Установлення

<Steps>
  <Step title="Установлення залежностей">
    ```sh
    bun install
    ```

    `bun.lock` / `bun.lockb` ігноруються Git, тому зміни в репозиторії не виникають. Щоб повністю пропустити запис файлу блокування:

    ```sh
    bun install --no-save
    ```

  </Step>
  <Step title="Збирання та тестування">
    ```sh
    bun run build
    bun run vitest run
    ```

    Команди, які запускають сам OpenClaw, усе одно потрібно виконувати через Node.

  </Step>
</Steps>

## Скрипти життєвого циклу

Bun блокує скрипти життєвого циклу залежностей, якщо їм явно не надано довіру. Для цього репозиторію скрипти, які зазвичай блокуються, не потрібні:

- `baileys` `preinstall`: перевіряє, що основна версія Node >= 20 (OpenClaw потребує Node 22.22.3+, 24.15+ або 25.9+; рекомендовано Node 24)
- `protobufjs` `postinstall`: виводить попередження про несумісні схеми версій (без артефактів збирання)

Якщо виникне проблема під час виконання, для розв’язання якої потрібні ці скрипти, явно надайте їм довіру:

```sh
bun pm trust baileys protobufjs
```

## Застереження

Деякі скрипти пакетів містять жорстко заданий `pnpm` (наприклад, `check:docs`, `ui:*`, `protocol:check`). Їх запуск через `bun run` усе одно викликає `pnpm` в оболонці, тому просто запускайте їх безпосередньо через `pnpm`.
