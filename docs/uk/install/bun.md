<Warning>
Bun не рекомендовано для середовища виконання Gateway (відомі проблеми з WhatsApp і Telegram). Для робочого середовища використовуйте Node.
</Warning>

Bun — це необов’язкове локальне середовище виконання для безпосереднього запуску TypeScript (`bun run ...`, `bun --watch ...`). Стандартним менеджером пакетів залишається `pnpm`, який повністю підтримується та використовується інструментами документації. Bun не може використовувати `pnpm-lock.yaml` та ігнорує його.

## Установлення

<Steps>
  <Step title="Установлення залежностей">
    ```sh
    bun install
    ```

    `bun.lock` / `bun.lockb` додано до gitignore, тому зайвих змін у репозиторії не виникає. Щоб повністю вимкнути запис файлу блокування:

    ```sh
    bun install --no-save
    ```

  </Step>
  <Step title="Збирання та тестування">
    ```sh
    bun run build
    bun run vitest run
    ```
  </Step>
</Steps>

## Скрипти життєвого циклу

Bun блокує скрипти життєвого циклу залежностей, якщо їм явно не надано довіру. Для цього репозиторію скрипти, які зазвичай блокуються, не потрібні:

- `baileys` `preinstall`: перевіряє, що основна версія Node >= 20 (OpenClaw потребує Node 22.19+ або 23.11+, рекомендовано Node 24)
- `protobufjs` `postinstall`: виводить попередження про несумісні схеми версій (без артефактів збирання)

Якщо виникне проблема під час виконання, для розв’язання якої потрібні ці скрипти, явно надайте їм довіру:

```sh
bun pm trust baileys protobufjs
```

## Застереження

У деяких скриптах пакетів усередині жорстко задано `pnpm` (наприклад, `check:docs`, `ui:*`, `protocol:check`). Їх запуск через `bun run` однаково викликає `pnpm` в оболонці, тому запускайте їх безпосередньо через `pnpm`.
