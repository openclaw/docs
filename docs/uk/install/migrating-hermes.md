Вбудований провайдер міграції Hermes виявляє стан у `~/.hermes`, показує попередній перегляд кожної зміни перед застосуванням, приховує секрети в планах і звітах та створює перевірену резервну копію OpenClaw, перш ніж щось змінювати.

<Note>
Імпорт потребує нового налаштування OpenClaw. Якщо у вас уже є локальний стан OpenClaw, спочатку скиньте конфігурацію, облікові дані, сеанси та робочий простір або скористайтеся безпосередньо `openclaw migrate apply hermes` із `--overwrite` після перегляду плану.
</Note>

## Два способи імпорту

<Tabs>
  <Tab title="Майстер початкового налаштування">
    Виявляє Hermes у `~/.hermes` і показує попередній перегляд перед застосуванням.

    ```bash
    openclaw onboard --flow import
    ```

    Або вкажіть конкретне джерело:

    ```bash
    openclaw onboard --import-from hermes --import-source ~/.hermes
    ```

  </Tab>
  <Tab title="CLI">
    Використовуйте `openclaw migrate` для сценарних або повторюваних запусків. Повний довідник див. у розділі [`openclaw migrate`](/uk/cli/migrate).

    ```bash
    openclaw migrate hermes --dry-run    # лише попередній перегляд
    openclaw migrate apply hermes --yes  # застосувати без запиту підтвердження
    ```

    Додайте `--from <path>`, якщо Hermes розташовано поза `~/.hermes`.

  </Tab>
</Tabs>

## Що імпортується

<AccordionGroup>
  <Accordion title="Конфігурація моделі">
    - Вибір стандартної моделі з Hermes `config.yaml`.
    - Налаштовані провайдери моделей і власні сумісні з OpenAI кінцеві точки з `providers` та `custom_providers`.

  </Accordion>
  <Accordion title="Сервери MCP">
    Визначення серверів MCP із `mcp_servers` або `mcp.servers`.
  </Accordion>
  <Accordion title="Файли робочого простору">
    - `SOUL.md` і `AGENTS.md` копіюються до робочого простору агента OpenClaw.
    - `memories/MEMORY.md` і `memories/USER.md` **додаються в кінець** відповідних файлів пам’яті OpenClaw замість їх перезаписування.

  </Accordion>
  <Accordion title="Конфігурація пам’яті">
    Стандартні параметри конфігурації файлової пам’яті OpenClaw. Зовнішні провайдери пам’яті, як-от Honcho, записуються як елементи архіву або ручної перевірки, щоб ви могли перенести їх свідомо.
  </Accordion>
  <Accordion title="Skills">
    Skills із файлом `SKILL.md` у `skills/<name>/` копіюються разом зі значеннями конфігурації кожного Skills із `skills.config`.
  </Accordion>
  <Accordion title="Облікові дані автентифікації">
    Інтерактивна команда `openclaw migrate` запитує підтвердження перед імпортом облікових даних автентифікації, причому стандартно вибрано «так». Після підтвердження імпортуються записи OpenCode OpenAI OAuth і GitHub Copilot із `auth.json` OpenCode, а також [підтримувані ключі Hermes `.env`](/uk/cli/migrate#supported-env-keys). Власні записи OAuth із `auth.json` Hermes є застарілим станом: замість імпорту до активної автентифікації вони відображаються як елемент для ручної повторної автентифікації або перевірки командою doctor. Використовуйте `--include-secrets`, щоб імпортувати облікові дані під час неінтерактивного запуску, `--no-auth-credentials`, щоб повністю пропустити імпорт облікових даних, або прапорець `--import-secrets` майстра початкового налаштування.
  </Accordion>
</AccordionGroup>

## Що залишається лише в архіві

Провайдер копіює наведені нижче елементи до каталогу звіту про міграцію для ручної перевірки, але **не** завантажує їх до активної конфігурації чи облікових даних OpenClaw:

- `plugins/`
- `sessions/`
- `logs/`
- `cron/`
- `mcp-tokens/`
- `state.db`

OpenClaw відмовляється автоматично виконувати цей стан або довіряти йому, оскільки формати та припущення щодо довіри можуть відрізнятися між системами. Після перегляду архіву перенесіть потрібні елементи вручну.

## Рекомендований процес

<Steps>
  <Step title="Перегляньте план">
    ```bash
    openclaw migrate hermes --dry-run
    ```

    План містить усе, що буде змінено, зокрема конфлікти, пропущені та конфіденційні елементи. Вкладені ключі, схожі на секрети, приховуються у виведенні.

  </Step>
  <Step title="Застосуйте зі створенням резервної копії">
    ```bash
    openclaw migrate apply hermes --yes
    ```

    OpenClaw створює та перевіряє резервну копію перед застосуванням. Цей неінтерактивний приклад імпортує лише несекретний стан. Запустіть без `--yes`, щоб відповісти на запит щодо облікових даних інтерактивно, або додайте `--include-secrets`, щоб включити підтримувані облікові дані під час автоматичного запуску.

  </Step>
  <Step title="Запустіть doctor">
    ```bash
    openclaw doctor
    ```

    [Doctor](/uk/gateway/doctor) повторно застосовує всі незавершені міграції конфігурації та перевіряє наявність проблем, що виникли під час імпорту.

  </Step>
  <Step title="Перезапустіть і перевірте">
    ```bash
    openclaw gateway restart
    openclaw status
    ```

    Переконайтеся, що Gateway працює належним чином, а імпортовані модель, пам’ять і Skills завантажено.

  </Step>
</Steps>

## Обробка конфліктів

Застосування не продовжується, якщо план повідомляє про конфлікти — файл або значення конфігурації вже існує в цільовому розташуванні.

<Warning>
Повторно запускайте з `--overwrite`, лише якщо ви навмисно замінюєте наявний цільовий елемент. Провайдери все одно можуть створювати резервні копії окремих перезаписаних файлів у каталозі звіту про міграцію.
</Warning>

Під час нового встановлення конфлікти трапляються рідко. Зазвичай вони виникають, коли імпорт запускають повторно для налаштування, яке вже містить зміни користувача.

Якщо конфлікт виникає під час застосування (наприклад, через неочікувану конкуренцію за файл конфігурації), Hermes позначає решту залежних елементів конфігурації як `skipped` із причиною `blocked by earlier apply conflict`, замість того щоб записувати їх частково. Звіт про міграцію фіксує кожен заблокований елемент, щоб ви могли усунути початковий конфлікт і повторно запустити імпорт.

## Секрети

Інтерактивна команда `openclaw migrate` запитує, чи імпортувати виявлені облікові дані автентифікації, причому стандартно вибрано «так».

- Після підтвердження імпортуються записи OpenCode OpenAI OAuth і GitHub Copilot із `auth.json` OpenCode, а також [підтримувані ключі `.env`](/uk/cli/migrate#supported-env-keys). Власні записи OAuth із `auth.json` Hermes натомість включаються до звіту для ручної повторної автентифікації OpenAI або виправлення командою doctor.
- Використовуйте `--no-auth-credentials` або дайте негативну відповідь на запит, щоб імпортувати лише несекретний стан.
- Використовуйте `--include-secrets`, щоб імпортувати облікові дані під час автоматичного запуску з `--yes`.
- Використовуйте прапорець `--import-secrets` майстра початкового налаштування, щоб імпортувати облікові дані за допомогою майстра.

## Виведення JSON для автоматизації

```bash
openclaw migrate hermes --dry-run --json
openclaw migrate apply hermes --json --yes
```

З `--json` і без `--yes` команда застосування виводить план і не змінює стан — це найбезпечніший режим для CI та спільних сценаріїв.

## Усунення проблем

<AccordionGroup>
  <Accordion title="Застосування не виконується через конфлікти">
    Перегляньте виведення плану. Для кожного конфлікту вказано шлях до джерела та наявний цільовий елемент. Для кожного елемента вирішіть, чи пропустити його, змінити цільовий елемент або повторно запустити команду з `--overwrite`.
  </Accordion>
  <Accordion title="Hermes розташовано поза ~/.hermes">
    Передайте `--from /actual/path` (CLI) або `--import-source /actual/path` (початкове налаштування).
  </Accordion>
  <Accordion title="Майстер початкового налаштування відмовляється імпортувати до наявного налаштування">
    Імпорт за допомогою майстра початкового налаштування потребує нового налаштування. Або скиньте стан і повторно виконайте початкове налаштування, або скористайтеся безпосередньо `openclaw migrate apply hermes`, що підтримує `--overwrite` і явне керування резервними копіями.
  </Accordion>
  <Accordion title="Ключі API не імпортувалися">
    Інтерактивна команда `openclaw migrate` імпортує ключі API, лише якщо ви підтвердите це у запиті щодо облікових даних. Для неінтерактивних запусків із `--yes` потрібен `--include-secrets`, а для імпорту за допомогою майстра початкового налаштування — `--import-secrets`. Розпізнаються лише [підтримувані ключі `.env`](/uk/cli/migrate#supported-env-keys) — інші змінні `.env` ігноруються.
  </Accordion>
</AccordionGroup>
