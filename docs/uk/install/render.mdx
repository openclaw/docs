Розгорніть OpenClaw на [Render](https://render.com), використовуючи Blueprint `render.yaml` із репозиторію. У ньому в одному файлі оголошено сервіс, диск і змінні середовища.

## Передумови

- [Обліковий запис Render](https://render.com) (доступний безкоштовний тариф)
- Ключ API від бажаного [постачальника моделей](/uk/providers)

## Розгортання

[Розгорнути на Render](https://render.com/deploy?repo=https://github.com/openclaw/openclaw)

Це створює сервіс Render із `render.yaml`, збирає образ Docker і розгортає його. URL-адреса вашого сервісу має формат `https://<service-name>.onrender.com`.

## Blueprint

```yaml
services:
  - type: web
    name: openclaw
    runtime: docker
    plan: starter
    healthCheckPath: /health
    envVars:
      - key: OPENCLAW_GATEWAY_PORT
        value: "8080"
      - key: OPENCLAW_STATE_DIR
        value: /data/.openclaw
      - key: OPENCLAW_WORKSPACE_DIR
        value: /data/workspace
      - key: OPENCLAW_GATEWAY_TOKEN
        generateValue: true # автоматично генерує захищений токен
    disk:
      name: openclaw-data
      mountPath: /data
      sizeGB: 1
```

| Функція               | Призначення                                                        |
| --------------------- | ------------------------------------------------------------------ |
| `runtime: docker`     | Збирає образ із Dockerfile репозиторію                             |
| `healthCheckPath`     | Render перевіряє `/health` і перезапускає несправні екземпляри     |
| `generateValue: true` | Автоматично генерує криптографічно захищене значення               |
| `disk`                | Постійне сховище, дані якого зберігаються після повторних розгортань |

## Вибір тарифу

| Тариф    | Зупинення в разі простою       | Диск        | Найкраще підходить для                |
| -------- | ------------------------------ | ----------- | ------------------------------------- |
| Free     | Після 15 хвилин бездіяльності  | Недоступний | Тестування, демонстрації              |
| Starter  | Ніколи                         | 1 ГБ+       | Особисте використання, невеликі команди |
| Standard+ | Ніколи                        | 1 ГБ+       | Робоче середовище, кілька каналів     |

У Blueprint типовим є тариф `starter`. Щоб використовувати безкоштовний тариф, змініть `plan: free` у файлі `render.yaml` вашого форку. Зверніть увагу: без постійного диска стан OpenClaw скидатиметься після кожного розгортання.

## Після розгортання

### Доступ до інтерфейсу керування

Вебпанель доступна за адресою `https://<your-service>.onrender.com/`. Підключіться за допомогою спільного секрету — автоматично згенерованого `OPENCLAW_GATEWAY_TOKEN` (його можна знайти в **Dashboard → your service → Environment**) — або пароля, якщо ви перейшли на автентифікацію за паролем.

### Журнали

У розділі **Dashboard → your service → Logs** відображаються журнали збирання (створення образу Docker), журнали розгортання (запуск сервісу) і журнали виконання (вивід застосунку).

### Доступ до оболонки

**Dashboard → your service → Shell** відкриває сеанс оболонки. Постійний диск змонтовано в `/data`.

### Змінні середовища

Редагуйте змінні в **Dashboard → your service → Environment**. Зміни запускають автоматичне повторне розгортання.

### Автоматичне розгортання

Render автоматично виконує повторне розгортання, коли в підключеній гілці репозиторію з’являється новий коміт. Якщо ви виконали розгортання безпосередньо з `openclaw/openclaw`, а не з власного форку, ви не маєте доступу для надсилання змін, щоб запустити цей процес. Тому для оновлення виконайте ручну синхронізацію Blueprint у Dashboard або спрямуйте сервіс на власний форк.

## Власний домен

1. **Dashboard → your service → Settings → Custom Domains**
2. Додайте свій домен
3. Налаштуйте DNS відповідно до вказівок (CNAME на `*.onrender.com`)
4. Render автоматично надасть сертифікат TLS

## Масштабування

- **Вертикальне**: змініть тариф, щоб отримати більше ресурсів CPU та RAM. Зазвичай цього достатньо для OpenClaw.
- **Горизонтальне**: збільште кількість екземплярів (тариф Standard або вищий). Потрібні прив’язані сеанси або зовнішнє керування станом, оскільки OpenClaw зберігає стан виконання на локальному диску.

## Резервне копіювання та міграція

З оболонки Render Dashboard можна будь-коли експортувати стан, конфігурацію, профілі автентифікації та робочий простір:

```bash
openclaw backup create
```

Ця команда створює переносний архів резервної копії. Див. [Резервне копіювання](/uk/cli/backup).

## Усунення несправностей

### Сервіс не запускається

Перевірте журнали розгортання в Render Dashboard. Поширені проблеми:

- Відсутній `OPENCLAW_GATEWAY_TOKEN` — переконайтеся, що його задано в **Dashboard → Environment**
- Невідповідність порту — переконайтеся, що задано `OPENCLAW_GATEWAY_PORT=8080`, щоб Gateway прив’язувався до порту, якого очікує Render

### Повільний запуск після простою (безкоштовний тариф)

Сервіси безкоштовного тарифу зупиняються після 15 хвилин бездіяльності; перший запит після зупинення виконується кілька секунд, поки запускається контейнер. Перейдіть на Starter, щоб сервіс працював постійно.

### Втрата даних після повторного розгортання

Це відбувається на безкоштовному тарифі (немає постійного диска). Перейдіть на платний тариф або регулярно експортуйте резервну копію командою `openclaw backup create` з оболонки Render.

### Помилки перевірки працездатності

Якщо збирання завершується успішно, але розгортання не вдається, можливо, сервіс запускається надто довго або `/health` недоступний. Перевірте:

- Журнали збирання на наявність помилок
- Чи запускається контейнер локально за допомогою `docker build && docker run`
