## Швидкий початок

### Режим Gateway

1. Запустіть Gateway.

```bash
openclaw gateway
```

2. Відкрийте TUI.

```bash
openclaw tui
```

3. Введіть повідомлення та натисніть Enter.

Віддалений Gateway:

```bash
openclaw tui --url ws://<host>:<port> --token <gateway-token>
```

Використовуйте `--password`, якщо ваш Gateway використовує автентифікацію за паролем.

### Локальний режим

Запустіть TUI без Gateway:

```bash
openclaw chat
# або
openclaw tui --local
```

- `openclaw chat` і `openclaw terminal` — псевдоніми для `openclaw tui --local`.
- `--local` не можна поєднувати з `--url`, `--token` або `--password`.
- Локальний режим безпосередньо використовує вбудоване середовище виконання агента. Більшість локальних інструментів працюють, але функції, доступні лише через Gateway, недоступні.
- Команда `openclaw` без підкоманди автоматично вибирає ціль: у неналаштованій інсталяції запускається початкове налаштування інференсу; за недійсної конфігурації відкриваються класичні рекомендації Doctor; якщо налаштований Gateway доступний, ця оболонка TUI відкривається в режимі Gateway; інакше, якщо налаштовано локальну модель, вона відкривається в локальному режимі.

## Що відображається

- Заголовок: URL-адреса підключення, поточний агент, поточний сеанс.
- Журнал чату: повідомлення користувача, відповіді асистента, системні сповіщення, картки інструментів.
- Рядок стану: стан підключення або виконання (підключення, виконання, потокове передавання, очікування, помилка).
- Нижній колонтитул: агент + сеанс + модель + стан цілі + think/fast/verbose/trace/reasoning + кількість токенів + доставлення. Коли ввімкнено `tui.footer.showRemoteHost`, для віддалених підключень до Gateway також відображається хост підключення.
- Введення: текстовий редактор з автодоповненням.

## Ментальна модель: агенти та сеанси

- Агенти мають унікальні короткі ідентифікатори (наприклад, `main`, `research`). Gateway надає їхній список.
- Сеанси належать поточному агенту.
- Ключі сеансів зберігаються як `agent:<agentId>:<sessionKey>`.
  - Якщо ввести `/session main`, TUI розгорне його до `agent:<currentAgent>:main`.
  - Якщо ввести `/session agent:other:main`, буде явно виконано перехід до цього сеансу агента.
- Область сеансу:
  - `per-sender` (типово): кожен агент має багато сеансів.
  - `global`: TUI завжди використовує сеанс `global` (список вибору може бути порожнім).
- Поточні агент і сеанс завжди відображаються в нижньому колонтитулі.
- Щоб відображати хост Gateway для нелокальних підключень за URL-адресою, увімкніть цю функцію командою:

  ```bash
  openclaw config set tui.footer.showRemoteHost true
  ```

  Типове значення — `false`. Для зворотних і вбудованих локальних підключень мітка хоста ніколи не відображається.

- Якщо сеанс має [ціль](/uk/tools/goal), у нижньому колонтитулі відображається її стислий стан:
  `Pursuing goal`, `Goal paused (/goal resume)`, `Goal blocked (/goal resume)` або `Goal achieved`.
- Якщо TUI в режимі Gateway запущено без `--session`, він відновлює останній вибраний сеанс для тих самих Gateway, агента й області сеансу, якщо цей сеанс досі існує. Передавання `--session`, `/session`, `/new` або `/reset` залишається явним.

## Надсилання та доставлення

- Повідомлення завжди надсилаються до Gateway (або вбудованого середовища виконання в локальному режимі); доставлення відповіді асистента назад до постачальника чату — це окремий крок, типово вимкнений.
- TUI — це внутрішня поверхня джерела, як і WebChat, а не універсальний вихідний канал. Середовища, яким для видимих відповідей потрібен `tools.message`, можуть виконати активний хід TUI за допомогою `message.send` без цілі; явне доставлення через постачальника й надалі використовує звичайні налаштовані канали й ніколи не повертається до `lastChannel`.
- Доставлення задається під час запуску для всього сеансу TUI: запустіть із `openclaw tui --deliver`, щоб увімкнути його. Немає slash-команди `/deliver` або перемикача Settings для зміни цього параметра посеред сеансу; щоб змінити його, перезапустіть TUI.

## Списки вибору та накладні панелі

- Вибір моделі: перегляд доступних моделей і встановлення перевизначення для сеансу.
- Вибір агента: вибір іншого агента.
- Вибір сеансу: відображає до 50 сеансів поточного агента, оновлених протягом останніх 7 днів. Використовуйте `/session <key>`, щоб перейти до старішого відомого сеансу.
- Settings (`/settings`): перемикання розгортання виводу інструментів і видимості міркувань. Ця панель не керує доставленням.

## Сполучення клавіш

- Enter: надіслати повідомлення
- Esc: перервати активне виконання
- Ctrl+C: очистити поле введення (натисніть двічі, щоб вийти)
- Ctrl+D: вийти
- Ctrl+L: вибір моделі
- Ctrl+G: вибір агента
- Ctrl+P: вибір сеансу
- Ctrl+O: перемкнути розгортання виводу інструментів
- Ctrl+T: перемкнути видимість міркувань (перезавантажує історію)

## Slash-команди

Основні:

- `/help`
- `/status` (пересилається до Gateway; показує зведення про сеанс і модель)
- `/gateway-status` (псевдонім `/gwstatus`; безпосередньо показує стан підключення до Gateway)
- `/agent <id>` (або `/agents`)
- `/session <key>` (або `/sessions`)
- `/model <provider/model>` (або `/models`)

Керування сеансом:

- `/think <off|minimal|low|medium|high>` (вищі рівні можуть додавати такі рівні, як `xhigh`/`max`, залежно від моделі)
- `/fast <status|auto|on|off>`
- `/verbose <on|full|off>`
- `/trace <on|off>`
- `/reasoning <on|off|stream>`
- `/usage <off|tokens|full|reset>` (`reset`/`inherit`/`clear`/`default` очищає перевизначення сеансу)
- `/goal [status] | /goal start <objective> | /goal edit <objective> | /goal pause|resume|complete|block|clear`
- `/elevated <on|off|ask|full>` (псевдонім: `/elev`)
- `/activation <mention|always>`

Життєвий цикл сеансу:

- `/new` (створити новий ізольований сеанс із новим ключем; не впливає на інші клієнти TUI в старому сеансі)
- `/reset` (скинути поточний ключ сеансу на місці)
- `/abort` (перервати активне виконання)
- `/settings`
- `/exit` (або `/quit`)

Лише локальний режим:

- `/auth [provider]` відкриває в TUI процес автентифікації або входу до постачальника.

OpenClaw:

- `/openclaw [request]` повертає зі звичайного TUI агента до чату налаштування або відновлення [OpenClaw](#openclaw-setup-and-repair-helper), за бажанням пересилаючи один запит.

Інші slash-команди Gateway (наприклад, `/context`) пересилаються до Gateway і відображаються як системний вивід. Див. [Slash-команди](/uk/tools/slash-commands).

## Команди локальної оболонки

- Додайте на початку рядка `!`, щоб виконати команду локальної оболонки на хості TUI.
- TUI один раз за сеанс запитує дозвіл на локальне виконання; у разі відмови `!` залишається вимкненим протягом сеансу.
- Команди виконуються в новій неінтерактивній оболонці в робочому каталозі TUI (без постійного `cd`/середовища).
- Команди локальної оболонки отримують `OPENCLAW_SHELL=tui-local` у своєму середовищі.
- Окремий `!` надсилається як звичайне повідомлення; початкові пробіли не запускають локальне виконання.

## Помічник із налаштування та відновлення OpenClaw

OpenClaw — це помічник нульового рівня для налаштування та відновлення, доступний як `openclaw setup` після того, як налаштована типова модель успішно пройде перевірку інференсу наживо. Якщо інференс недоступний, інтерактивний виклик повертає до початкового налаштування інференсу, а автоматизація завершується помилкою з рекомендаціями щодо відновлення. Він працює в тій самій локальній оболонці TUI, що й `openclaw tui --local`, і використовує агента ШІ, обмеженого типізованими операціями OpenClaw, які потребують схвалення:

```bash
openclaw setup                       # запустити інтерактивно
openclaw setup -m "status"           # виконати один запит і вийти
openclaw setup -m "set default model openai/gpt-5.2" --yes   # застосувати запис конфігурації
```

- Постійні записи до конфігурації потребують схвалення: підтвердьте їх інтерактивно або передайте `--yes`.
- `--json` виводить огляд запуску у форматі JSON замість запуску чату.
- Усередині OpenClaw запит `open-tui` (наприклад, прохання поспілкуватися зі звичайним агентом) завершує роботу OpenClaw і відкриває звичайний TUI агента; використовуйте там `/openclaw`, щоб повернутися.

Використовуйте локальний режим, коли поточна конфігурація вже проходить перевірку й потрібно, щоб вбудований агент перевірив її на тому самому комп'ютері, порівняв із документацією та допоміг усунути розбіжності без залежності від запущеного Gateway.

Якщо `openclaw config validate` уже завершується помилкою, спочатку запустіть `openclaw configure` або `openclaw doctor --fix`; для запуску `openclaw chat` усе одно потрібна конфігурація, яку можна завантажити.

Типовий цикл:

1. Запустіть локальний режим:

```bash
openclaw chat
```

2. Попросіть агента виконати потрібну перевірку, наприклад:

```text
Порівняй мою конфігурацію автентифікації Gateway із документацією та запропонуй найменше виправлення.
```

3. Використовуйте команди локальної оболонки для отримання точних даних і перевірки:

```text
!openclaw config file
!openclaw docs gateway auth token secretref
!openclaw config validate
!openclaw doctor
```

4. Застосуйте точкові зміни за допомогою `openclaw config set` або `openclaw configure`, а потім знову запустіть `!openclaw config validate`.
5. Якщо Doctor рекомендує автоматичну міграцію або відновлення, перегляньте їх і запустіть `!openclaw doctor --fix`.

Поради:

- Віддавайте перевагу `openclaw config set` або `openclaw configure` замість редагування `openclaw.json` вручну.
- `openclaw docs "<query>"` виконує пошук в актуальному індексі документації з того самого комп'ютера.
- `openclaw config validate --json` корисний, коли потрібні структурована схема та помилки SecretRef або можливості розв'язання.

## Вивід інструментів

- Виклики інструментів відображаються як картки з аргументами та результатами.
- Ctrl+O перемикає між згорнутим і розгорнутим поданнями.
- Під час роботи інструментів часткові оновлення потоково надходять до тієї самої картки.

## Кольори термінала

- TUI відображає основний текст асистента типовим кольором переднього плану термінала, тому він залишається читабельним як у темних, так і у світлих терміналах.
- Якщо термінал використовує світле тло, а автовизначення працює неправильно, установіть `OPENCLAW_THEME=light` перед запуском `openclaw tui`.
- Щоб натомість примусово використовувати початкову темну палітру, установіть `OPENCLAW_THEME=dark`.

## Історія та потокове передавання

- Після підключення TUI завантажує найновішу історію (типово 200 повідомлень).
- Потокові відповіді оновлюються на місці до завершення.
- TUI також відстежує події інструментів агента для створення докладніших карток інструментів.

## Відомості про підключення

- TUI підключається з ідентифікатором клієнта `openclaw-tui` у загальному режимі клієнта `ui` (цей самий режим використовують Control UI і WebChat для політик Gateway).
- Повторні підключення відображаються системним повідомленням; пропуски подій показуються в журналі.

## Параметри

- `--local`: Запустити з локальним вбудованим середовищем виконання агента
- `--url <url>`: URL WebSocket Gateway (типово — `gateway.remote.url` із конфігурації або `ws://127.0.0.1:<port>` у loopback)
- `--token <token>`: Токен Gateway (якщо потрібен)
- `--password <password>`: Пароль Gateway (якщо потрібен)
- `--tls-fingerprint <sha256>`: Очікуваний відбиток TLS-сертифіката для Gateway із закріпленим `wss://`
- `--session <key>`: Ключ сеансу (типово: `main` або `global`, коли область дії глобальна)
- `--deliver`: Доставляти відповіді асистента провайдеру (типово вимкнено)
- `--thinking <level>`: Перевизначити рівень міркування для надсилань
- `--message <text>`: Надіслати початкове повідомлення після підключення
- `--timeout-ms <ms>`: Час очікування агента в мс (типово — `agents.defaults.timeoutSeconds`)
- `--history-limit <n>`: Кількість записів історії для завантаження (типово — `200`)

<Warning>
Коли встановлено `--url`, TUI не використовує резервно облікові дані з конфігурації або змінних середовища. Явно передайте `--token` або `--password`, а також `--tls-fingerprint`, якщо ціль використовує закріплений сертифікат. Відсутність явно заданих облікових даних є помилкою. У локальному режимі не передавайте `--url`, `--token`, `--password` або `--tls-fingerprint`.
</Warning>

## Усунення несправностей

Немає виводу після надсилання повідомлення:

- Виконайте `/status` у TUI, щоб переконатися, що Gateway підключений і перебуває в стані очікування або зайнятості.
- Перевірте журнали Gateway: `openclaw logs --follow`.
- Переконайтеся, що агент може запускатися: `openclaw status` і `openclaw models status`.
- Якщо повідомлення мають надходити в канал чату, переконайтеся, що TUI запущено з `--deliver` (це неможливо ввімкнути пізніше без перезапуску).

## Усунення несправностей підключення

- `disconnected`: переконайтеся, що Gateway працює, а ваші `--url/--token/--password` правильні.
- У засобі вибору немає агентів: перевірте `openclaw agents list` і конфігурацію маршрутизації.
- Засіб вибору сеансів порожній: можливо, ви перебуваєте в глобальній області дії або ще не маєте сеансів.
