## Швидкий старт

### Режим Gateway

1. Запустіть Gateway.

```bash
openclaw gateway
```

2. Відкрийте TUI.

```bash
openclaw tui
```

3. Введіть повідомлення й натисніть Enter.

Віддалений Gateway:

```bash
openclaw tui --url ws://<host>:<port> --token <gateway-token>
```

Використовуйте `--password`, якщо ваш Gateway застосовує автентифікацію за паролем.

### Локальний режим

Запустіть TUI без Gateway:

```bash
openclaw chat
# або
openclaw tui --local
```

- `openclaw chat` і `openclaw terminal` — псевдоніми для `openclaw tui --local`.
- `--local` не можна поєднувати з `--url`, `--token` або `--password`.
- Локальний режим безпосередньо використовує вбудоване середовище виконання агента. Більшість локальних інструментів працюють, але функції, доступні лише через Gateway, недоступні.
- Команда `openclaw` без підкоманди автоматично вибирає ціль: у неналаштованій інсталяції запускається початкове налаштування інференсу; за некоректної конфігурації відкриваються класичні рекомендації Doctor; якщо налаштований Gateway доступний, ця оболонка TUI відкривається в режимі Gateway; інакше, якщо налаштовано локальну модель, вона відкривається в локальному режимі.

## Що відображається

- Заголовок: URL з’єднання, поточний агент, поточний сеанс.
- Журнал чату: повідомлення користувача, відповіді асистента, системні сповіщення, картки інструментів.
- Рядок стану: стан з’єднання/виконання (підключення, виконання, потокове передавання, очікування, помилка).
- Нижній рядок: агент + сеанс + модель + стан цілі + think/fast/verbose/trace/reasoning + кількість токенів + доставлення. Коли ввімкнено `tui.footer.showRemoteHost`, для підключень до віддаленого Gateway також відображається хост з’єднання.
- Поле введення: текстовий редактор з автодоповненням.

## Ментальна модель: агенти + сеанси

- Агенти мають унікальні слаги (наприклад, `main`, `research`). Gateway надає їх список.
- Сеанси належать поточному агенту.
- Ключі сеансів зберігаються у форматі `agent:<agentId>:<sessionKey>`.
  - Якщо ввести `/session main`, TUI розгорне це до `agent:<currentAgent>:main`.
  - Якщо ввести `/session agent:other:main`, ви явно перемкнетеся на сеанс іншого агента.
- Область сеансів:
  - `per-sender` (типово): кожен агент має багато сеансів.
  - `global`: TUI завжди використовує сеанс `global` (список вибору може бути порожнім).
- Поточні агент і сеанс завжди відображаються в нижньому рядку.
- Щоб показувати хост Gateway для нелокальних підключень через URL, увімкніть цю функцію:

  ```bash
  openclaw config set tui.footer.showRemoteHost true
  ```

  Типове значення — `false`. Для з’єднань через local loopback і вбудованих локальних з’єднань мітка хоста ніколи не відображається.

- Якщо сеанс має [ціль](/uk/tools/goal), у нижньому рядку відображається її стислий стан:
  `Виконання цілі`, `Ціль призупинено (/goal resume)`, `Ціль заблоковано (/goal resume)` або `Цілі досягнуто`.
- Якщо TUI запускається без `--session` у режимі Gateway, він відновлює останній вибраний сеанс для того самого Gateway, агента й області сеансів, якщо цей сеанс досі існує. Передавання `--session`, `/session`, `/new` або `/reset` залишається явною дією.

## Надсилання + доставлення

- Повідомлення завжди надходять до Gateway (або до вбудованого середовища виконання в локальному режимі); доставлення відповіді асистента назад до постачальника чату — окремий крок, типово вимкнений.
- TUI — це внутрішня поверхня джерела, як WebChat, а не універсальний вихідний канал. Середовища, яким для видимих відповідей потрібен `tools.message`, можуть обробити активний хід TUI за допомогою `message.send` без цілі; явне доставлення через постачальника й надалі використовує звичайні налаштовані канали й ніколи не повертається до `lastChannel`.
- Доставлення фіксується для всього сеансу TUI під час запуску: щоб увімкнути його, запустіть `openclaw tui --deliver`. Команди `/deliver` або перемикача в налаштуваннях для зміни цього параметра посеред сеансу немає; щоб змінити його, перезапустіть TUI.

## Списки вибору + накладні панелі

- Вибір моделі: перегляд доступних моделей і встановлення перевизначення для сеансу.
- Вибір агента: вибір іншого агента.
- Вибір сеансу: показує до 50 сеансів поточного агента, оновлених за останні 7 днів. Щоб перейти до старішого відомого сеансу, скористайтеся `/session <key>`.
- Налаштування (`/settings`): перемикання розгортання виводу інструментів і видимості міркувань. Ця панель не керує доставленням.

## Сполучення клавіш

- Enter: надіслати повідомлення
- Esc: перервати активне виконання
- Ctrl+C: очистити поле введення (натисніть двічі для виходу)
- Ctrl+D: вийти
- Ctrl+L: вибір моделі
- Ctrl+G: вибір агента
- Ctrl+P: вибір сеансу
- Ctrl+O: перемкнути розгортання виводу інструментів
- Ctrl+T: перемкнути видимість міркувань (перезавантажує історію)

## Команди з косою рискою

Основні:

- `/help`
- `/status` (пересилається до Gateway; показує зведення сеансу/моделі)
- `/gateway-status` (псевдонім `/gwstatus`; безпосередньо показує стан з’єднання з Gateway)
- `/agent <id>` (або `/agents`)
- `/session <key>` (або `/sessions`)
- `/model <provider/model>` (або `/models`)

Керування сеансом:

- `/think <off|minimal|low|medium|high>` (вищі рівні можуть додавати такі значення, як `xhigh`/`max`, залежно від моделі)
- `/fast <status|auto|on|off>`
- `/verbose <on|full|off>`
- `/trace <on|off>`
- `/reasoning <on|off|stream>`
- `/usage <off|tokens|full|reset>` (`reset`/`inherit`/`clear`/`default` очищає перевизначення сеансу)
- `/goal [status] | /goal start <objective> | /goal edit <objective> | /goal pause|resume|complete|block|clear`
- `/elevated <on|off|ask|full>` (псевдонім: `/elev`)
- `/activation <mention|always>`

Життєвий цикл сеансу:

- `/new` (створити новий ізольований сеанс із новим ключем; не впливає на інші клієнти TUI у старому сеансі)
- `/reset` (скинути поточний ключ сеансу без його зміни)
- `/abort` (перервати активне виконання)
- `/settings`
- `/exit` (або `/quit`)

Лише локальний режим:

- `/auth [provider]` відкриває процес автентифікації/входу постачальника всередині TUI.

Crestodian:

- `/crestodian [request]` повертає зі звичайного TUI агента до чату [Crestodian](#crestodian-setup-and-repair-helper) для налаштування/відновлення та за потреби пересилає один запит.

Інші команди Gateway із косою рискою (наприклад, `/context`) пересилаються до Gateway і відображаються як системний вивід. Див. [Команди з косою рискою](/uk/tools/slash-commands).

## Локальні команди оболонки

- Додайте `!` на початку рядка, щоб виконати локальну команду оболонки на хості TUI.
- TUI один раз за сеанс запитує дозвіл на локальне виконання; у разі відмови `!` залишається вимкненим протягом сеансу.
- Команди виконуються в новій неінтерактивній оболонці в робочому каталозі TUI (без збереження `cd`/змінних середовища).
- Локальні команди оболонки отримують у своєму середовищі `OPENCLAW_SHELL=tui-local`.
- Окремий символ `!` надсилається як звичайне повідомлення; початкові пробіли не запускають локальне виконання.

## Помічник Crestodian для налаштування та відновлення

Crestodian — це помічник нульового рівня для налаштування/відновлення, доступний як `openclaw crestodian` після того, як налаштована типова модель успішно пройде перевірку інференсу наживо. Якщо інференс недоступний, інтерактивний запуск повертає до початкового налаштування інференсу, а автоматизація завершується помилкою з рекомендаціями щодо відновлення. Він працює в тій самій локальній оболонці TUI, що й `openclaw tui --local`, і використовує ШІ-агента, обмеженого типізованими операціями Crestodian, які потребують схвалення:

```bash
openclaw crestodian                       # запустити інтерактивно
openclaw crestodian -m "status"           # виконати один запит і вийти
openclaw crestodian -m "set default model openai/gpt-5.2" --yes   # застосувати запис конфігурації
```

- Постійні зміни конфігурації потребують схвалення: підтвердьте їх інтерактивно або передайте `--yes`.
- `--json` виводить огляд запуску у форматі JSON замість запуску чату.
- Усередині Crestodian запит `open-tui` (наприклад, прохання поспілкуватися зі звичайним агентом) закриває Crestodian і відкриває звичайний TUI агента; щоб повернутися, скористайтеся там `/crestodian`.

Використовуйте локальний режим, якщо поточна конфігурація вже проходить перевірку й ви хочете, щоб вбудований агент перевірив її на тому самому комп’ютері, порівняв із документацією та допоміг виправити розбіжності без залежності від запущеного Gateway.

Якщо `openclaw config validate` уже завершується помилкою, спочатку запустіть `openclaw configure` або `openclaw doctor --fix`; для запуску `openclaw chat` усе одно потрібна конфігурація, яку можна завантажити.

Типовий цикл:

1. Запустіть локальний режим:

```bash
openclaw chat
```

2. Попросіть агента перевірити потрібне, наприклад:

```text
Порівняй мою конфігурацію автентифікації Gateway із документацією та запропонуй найменше виправлення.
```

3. Використовуйте локальні команди оболонки для отримання точних даних і перевірки:

```text
!openclaw config file
!openclaw docs gateway auth token secretref
!openclaw config validate
!openclaw doctor
```

4. Внесіть точкові зміни за допомогою `openclaw config set` або `openclaw configure`, а потім повторно запустіть `!openclaw config validate`.
5. Якщо Doctor рекомендує автоматичну міграцію або відновлення, перегляньте рекомендацію та запустіть `!openclaw doctor --fix`.

Поради:

- Надавайте перевагу `openclaw config set` або `openclaw configure` замість ручного редагування `openclaw.json`.
- `openclaw docs "<query>"` виконує пошук в актуальному індексі документації з того самого комп’ютера.
- `openclaw config validate --json` корисна, коли потрібні структуровані помилки схеми та SecretRef/можливості розв’язання.

## Вивід інструментів

- Виклики інструментів відображаються як картки з аргументами й результатами.
- Ctrl+O перемикає між згорнутим і розгорнутим виглядом.
- Під час виконання інструментів часткові оновлення потоково надходять до тієї самої картки.

## Кольори термінала

- TUI відображає основний текст асистента стандартним кольором переднього плану вашого термінала, щоб він залишався читабельним як у темних, так і у світлих терміналах.
- Якщо ваш термінал має світле тло, а автоматичне визначення працює неправильно, перед запуском `openclaw tui` установіть `OPENCLAW_THEME=light`.
- Щоб натомість примусово використовувати початкову темну палітру, установіть `OPENCLAW_THEME=dark`.

## Історія + потокове передавання

- Після підключення TUI завантажує найновішу історію (типово 200 повідомлень).
- Потокові відповіді оновлюються на місці до завершення.
- TUI також відстежує події інструментів агента для відображення докладніших карток інструментів.

## Відомості про з’єднання

- TUI підключається з ідентифікатором клієнта `openclaw-tui` у загальному режимі клієнта `ui` (тому самому режимі, який Control UI і WebChat використовують для політик Gateway).
- Повторні підключення супроводжуються системним повідомленням; пропуски подій відображаються в журналі.

## Параметри

- `--local`: працювати з локальним вбудованим середовищем виконання агента
- `--url <url>`: URL WebSocket Gateway (типово `gateway.remote.url` із конфігурації або `ws://127.0.0.1:<port>` через local loopback)
- `--token <token>`: токен Gateway (якщо потрібен)
- `--password <password>`: пароль Gateway (якщо потрібен)
- `--tls-fingerprint <sha256>`: очікуваний відбиток TLS-сертифіката для Gateway із закріпленим `wss://`
- `--session <key>`: ключ сеансу (типово `main` або `global`, коли область глобальна)
- `--deliver`: доставляти відповіді асистента постачальнику (типово вимкнено)
- `--thinking <level>`: перевизначити рівень міркувань для надсилань
- `--message <text>`: надіслати початкове повідомлення після підключення
- `--timeout-ms <ms>`: час очікування агента в мс (типово `agents.defaults.timeoutSeconds`)
- `--history-limit <n>`: кількість записів історії для завантаження (типово `200`)

<Warning>
Коли ви задаєте `--url`, TUI не використовує резервні облікові дані з конфігурації або змінних середовища. Явно передайте `--token` або `--password`, а також `--tls-fingerprint`, якщо ціль використовує закріплений сертифікат. Відсутність явних облікових даних є помилкою. У локальному режимі не передавайте `--url`, `--token`, `--password` або `--tls-fingerprint`.
</Warning>

## Усунення несправностей

Після надсилання повідомлення немає виводу:

- Виконайте `/status` у TUI, щоб переконатися, що Gateway підключений і перебуває в стані очікування/виконання.
- Перевірте журнали Gateway: `openclaw logs --follow`.
- Переконайтеся, що агент може виконуватися: `openclaw status` і `openclaw models status`.
- Якщо ви очікуєте повідомлення в каналі чату, переконайтеся, що TUI було запущено з `--deliver` (увімкнути це пізніше без перезапуску неможливо).

## Усунення проблем зі з’єднанням

- `disconnected`: переконайтеся, що Gateway запущений, а ваші `--url/--token/--password` правильні.
- У списку вибору немає агентів: перевірте `openclaw agents list` і конфігурацію маршрутизації.
- Порожній список вибору сеансів: можливо, ви перебуваєте в глобальній області або ще не маєте сеансів.
