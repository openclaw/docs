OpenClaw може розміщувати **PeekabooBridge** як локальний посередник автоматизації інтерфейсу з урахуванням дозволів (`PeekabooBridgeHostCoordinator`, що використовує пакет Swift `steipete/Peekaboo`). Завдяки цьому CLI `peekaboo` може керувати автоматизацією інтерфейсу, повторно використовуючи дозволи TCC застосунку macOS.

## Що це таке (і чим воно не є)

- **Хост**: OpenClaw.app може працювати як хост PeekabooBridge.
- **Клієнт**: CLI `peekaboo` (окремого інтерфейсу `openclaw ui ...` немає).
- **Інтерфейс**: візуальні накладки залишаються в Peekaboo.app; OpenClaw — це легкий хост-посередник.

## Зв’язок з іншими способами керування робочим столом

OpenClaw має чотири способи керування робочим столом, які навмисно залишаються окремими:

- **Хост PeekabooBridge**: OpenClaw.app розміщує локальний сокет PeekabooBridge. CLI `peekaboo` є клієнтом і використовує дозволи macOS застосунку OpenClaw.app для створення знімків екрана, клацань, роботи з меню й діалоговими вікнами, дій у Dock і керування вікнами.
- **Кероване агентом використання комп’ютера (`computer.act`)**: вбудований інструмент `computer` агента Gateway створює знімки екрана через `screen.snapshot` і керує вказівником та клавіатурою за допомогою небезпечної команди Node `computer.act`. Node macOS виконує `computer.act` у межах процесу, використовуючи вбудовані служби автоматизації Peekaboo, які надає цей міст, а також вузькоспеціалізовані примітиви CoreGraphics, не звертаючись до сокета PeekabooBridge або CLI `peekaboo`. Див. [Використання комп’ютера](/nodes/computer-use).
- **Codex Computer Use**: вбудований plugin `codex` перевіряє та може встановити MCP-плагін `computer-use` для Codex (`extensions/codex/src/app-server/computer-use.ts`), а потім дає Codex змогу самостійно виконувати виклики інструментів нативного керування робочим столом під час сеансів у режимі Codex. OpenClaw не проксіює ці дії через PeekabooBridge.
- **Безпосередній MCP `cua-driver`**: OpenClaw може зареєструвати сервер `cua-driver mcp` від TryCua як звичайний сервер MCP, надаючи агентам власні схеми драйвера CUA та робочий процес із PID, вікнами й індексами елементів без маршрутизації через каталог Codex або сокет PeekabooBridge.

Використовуйте Peekaboo для широких можливостей автоматизації macOS через хост-мост OpenClaw.app з урахуванням дозволів. Використовуйте кероване агентом використання комп’ютера, коли агент Gateway має бачити робочий стіл і керувати ним за допомогою уніфікованої команди Node `computer.act`, якою може керувати будь-яка модель із підтримкою зору. Використовуйте Codex Computer Use, коли агент у режимі Codex має покладатися на нативний plugin Codex. Використовуйте безпосередній `cua-driver mcp`, щоб надати драйвер CUA будь-якому середовищу виконання під керуванням OpenClaw як звичайний сервер MCP.

## Увімкнення мосту

У застосунку macOS: **Settings -> Enable Peekaboo Bridge**.

Коли міст увімкнено, OpenClaw запускає локальний сервер сокета UNIX за адресою `~/Library/Application Support/OpenClaw/<socket-name>`. Коли його вимкнено, хост зупиняється, а `peekaboo` переходить до інших доступних хостів. Координатор також підтримує застарілі символічні посилання на сокет (`clawdbot`, `clawdis`, `moltbot` у каталозі Application Support), які вказують на поточний сокет для старіших інсталяцій `peekaboo`.

## Порядок виявлення клієнтом

Клієнти Peekaboo зазвичай перевіряють хости в такому порядку:

1. Peekaboo.app (повноцінний інтерфейс)
2. Claude.app (якщо встановлено)
3. OpenClaw.app (легкий посередник)

Щоб дізнатися, який хост активний і який шлях до сокета використовується, виконайте `peekaboo bridge status --verbose`. Щоб перевизначити його, скористайтеся:

```bash
export PEEKABOO_BRIDGE_SOCKET=/path/to/bridge.sock
```

## Безпека та дозволи

- Міст перевіряє **підписи коду викликачів**; застосовується список дозволених TeamID (TeamID хоста Peekaboo та власний TeamID запущеного застосунку).
- Для спеціальних можливостей надавайте перевагу підписаній ідентичності мосту або застосунку, а не універсальному середовищу виконання `node`. Надання `node` доступу до спеціальних можливостей дає змогу будь-якому пакету, запущеному цим виконуваним файлом Node, успадкувати доступ до автоматизації графічного інтерфейсу; див. [Дозволи macOS](/uk/platforms/mac/permissions#accessibility-grants-for-node-and-cli-runtimes).
- Час очікування запитів спливає через 10 секунд (`requestTimeoutSec: 10`).
- Якщо необхідні дозволи відсутні, міст повертає зрозуміле повідомлення про помилку замість запуску System Settings.

## Поведінка знімків (автоматизація)

Знімки зберігаються в пам’яті протягом 10 хвилин, а їхня кількість обмежена 50 (`InMemorySnapshotManager`); під час очищення артефакти не видаляються. Якщо потрібно зберігати їх довше, повторно створіть знімок із клієнта.

## Усунення несправностей

- Якщо `peekaboo` повідомляє «bridge client is not authorized», переконайтеся, що клієнт належним чином підписаний, або запускайте хост із `PEEKABOO_ALLOW_UNSIGNED_SOCKET_CLIENTS=1` лише в режимі **налагодження**.
- Якщо хости не знайдено, відкрийте один із хост-застосунків (Peekaboo.app або OpenClaw.app) і переконайтеся, що дозволи надано.
