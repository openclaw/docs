# Дозволи macOS (TCC)

Надання дозволів у macOS є крихким. TCC пов’язує наданий дозвіл із
code signature застосунку, bundle identifier і шляхом на диску. Якщо щось із цього змінюється,
macOS вважає застосунок новим і може скинути або приховати запити дозволів.

## Вимоги для стабільних дозволів

- Той самий шлях: запускайте застосунок із фіксованого розташування (для OpenClaw це `dist/OpenClaw.app`).
- Той самий bundle identifier: зміна bundle ID створює нову ідентичність дозволів.
- Підписаний застосунок: непідписані або ad-hoc signed збірки не зберігають дозволи.
- Узгоджений підпис: використовуйте справжній сертифікат Apple Development або Developer ID,
  щоб підпис залишався стабільним між перебудовами.

Ad-hoc signatures створюють нову ідентичність для кожної збірки. macOS забуватиме попередні
дозволи, а запити можуть повністю зникати, доки застарілі записи не буде очищено.

## Контрольний список відновлення, коли запити зникають

1. Завершіть роботу застосунку.
2. Видаліть запис застосунку в System Settings -> Privacy & Security.
3. Знову запустіть застосунок із того самого шляху й повторно надайте дозволи.
4. Якщо запит усе ще не з’являється, скиньте записи TCC через `tccutil` і спробуйте ще раз.
5. Деякі дозволи знову з’являються лише після повного перезапуску macOS.

Приклади скидання (за потреби замініть bundle ID):

```bash
sudo tccutil reset Accessibility ai.openclaw.mac
sudo tccutil reset ScreenCapture ai.openclaw.mac
sudo tccutil reset AppleEvents
```

## Дозволи на файли та папки (Desktop/Documents/Downloads)

macOS також може обмежувати доступ до Desktop, Documents і Downloads для процесів термінала/фонових процесів. Якщо читання файлів або перелік каталогів зависають, надайте доступ тому самому контексту процесу, який виконує операції з файлами (наприклад, Terminal/iTerm, застосунок, запущений через LaunchAgent, або процес SSH).

Обхідний варіант: перемістіть файли до workspace OpenClaw (`~/.openclaw/workspace`), якщо хочете уникнути надання окремих дозволів для папок.

Якщо ви тестуєте дозволи, завжди підписуйте застосунок справжнім сертифікатом. Ad-hoc
збірки прийнятні лише для швидких локальних запусків, де дозволи не мають значення.
