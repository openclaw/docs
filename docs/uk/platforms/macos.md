Програма macOS є **супутником у смузі меню** для OpenClaw. Вона відповідає за дозволи,
керує Gateway або під’єднується до нього локально (launchd або вручну) і надає агенту
можливості macOS як Node.

## Що вона робить

- Показує нативні сповіщення та статус у смузі меню.
- Відповідає за запити TCC (Notifications, Accessibility, Screen Recording, Microphone,
  Speech Recognition, Automation/AppleScript).
- Запускає або під’єднується до Gateway (локального чи віддаленого).
- Надає інструменти, доступні лише в macOS (Canvas, Camera, Screen Recording, `system.run`).
- Запускає локальну службу хоста Node у **віддаленому** режимі (launchd) і зупиняє її в **локальному** режимі.
- Опційно розміщує **PeekabooBridge** для автоматизації UI.
- Установлює глобальний CLI (`openclaw`) на запит через npm, pnpm або bun (програма надає перевагу npm, потім pnpm, потім bun; Node залишається рекомендованим середовищем виконання Gateway).

## Локальний і віддалений режим

- **Локальний** (типово): програма під’єднується до запущеного локального Gateway, якщо він є;
  інакше вмикає службу launchd через `openclaw gateway install`.
- **Віддалений**: програма під’єднується до Gateway через SSH/Tailscale і ніколи не запускає
  локальний процес.
  Програма запускає локальну **службу хоста Node**, щоб віддалений Gateway міг досягати цього Mac.
  Програма не породжує Gateway як дочірній процес.
  Виявлення Gateway тепер надає перевагу іменам Tailscale MagicDNS замість необроблених IP-адрес tailnet,
  тож програма Mac надійніше відновлюється, коли IP-адреси tailnet змінюються.

## Керування launchd

Програма керує користувацьким LaunchAgent із міткою `ai.openclaw.gateway`
(або `ai.openclaw.<profile>` під час використання `--profile`/`OPENCLAW_PROFILE`; застарілий `com.openclaw.*` усе ще вивантажується).

```bash
launchctl kickstart -k gui/$UID/ai.openclaw.gateway
launchctl bootout gui/$UID/ai.openclaw.gateway
```

Замініть мітку на `ai.openclaw.<profile>`, коли запускаєте іменований профіль.

Якщо LaunchAgent не встановлено, увімкніть його з програми або виконайте
`openclaw gateway install`.

## Можливості Node (mac)

Програма macOS представляє себе як Node. Поширені команди:

- Canvas: `canvas.present`, `canvas.navigate`, `canvas.eval`, `canvas.snapshot`, `canvas.a2ui.*`
- Camera: `camera.snap`, `camera.clip`
- Screen: `screen.snapshot`, `screen.record`
- System: `system.run`, `system.notify`

Node повідомляє мапу `permissions`, щоб агенти могли вирішувати, що дозволено.

Служба Node + IPC програми:

- Коли безголова служба хоста Node запущена (віддалений режим), вона під’єднується до Gateway WS як Node.
- `system.run` виконується в програмі macOS (контекст UI/TCC) через локальний Unix-сокет; запити й вивід залишаються в програмі.

Діаграма (SCI):

```
Gateway -> Node Service (WS)
                 |  IPC (UDS + token + HMAC + TTL)
                 v
             Mac App (UI + TCC + system.run)
```

## Підтвердження виконання (system.run)

`system.run` керується через **підтвердження виконання** у програмі macOS (Settings → Exec approvals).
Безпека + запити + allowlist зберігаються локально на Mac у:

```
~/.openclaw/exec-approvals.json
```

Приклад:

```json
{
  "version": 1,
  "defaults": {
    "security": "deny",
    "ask": "on-miss"
  },
  "agents": {
    "main": {
      "security": "allowlist",
      "ask": "on-miss",
      "allowlist": [{ "pattern": "/opt/homebrew/bin/rg" }]
    }
  }
}
```

Примітки:

- Записи `allowlist` є glob-шаблонами для розпізнаних шляхів до бінарних файлів або простими іменами команд для команд, викликаних через PATH.
- Необроблений текст shell-команди, що містить синтаксис керування shell або підстановки (`&&`, `||`, `;`, `|`, `` ` ``, `$`, `<`, `>`, `(`, `)`), вважається промахом allowlist і потребує явного підтвердження (або додавання бінарного файлу shell до allowlist).
- Вибір “Always Allow” у запиті додає цю команду до allowlist.
- Перевизначення середовища `system.run` фільтруються (відкидаються `PATH`, `DYLD_*`, `LD_*`, `NODE_OPTIONS`, `PYTHON*`, `PERL*`, `RUBYOPT`, `SHELLOPTS`, `PS4`), а потім об’єднуються із середовищем програми.
- Для shell-обгорток (`bash|sh|zsh ... -c/-lc`) перевизначення середовища в межах запиту зводяться до невеликого явного allowlist (`TERM`, `LANG`, `LC_*`, `COLORTERM`, `NO_COLOR`, `FORCE_COLOR`).
- Для рішень «завжди дозволяти» в режимі allowlist відомі обгортки диспетчеризації (`env`, `nice`, `nohup`, `stdbuf`, `timeout`) зберігають шляхи внутрішніх виконуваних файлів замість шляхів обгорток. Якщо розгортання небезпечне, запис allowlist автоматично не зберігається.

## Глибинні посилання

Програма реєструє схему URL `openclaw://` для локальних дій.

### `openclaw://agent`

Запускає запит `agent` до Gateway.
__OC_I18N_900004__
Параметри запиту:

- `message` (обов’язково)
- `sessionKey` (опційно)
- `thinking` (опційно)
- `deliver` / `to` / `channel` (опційно)
- `timeoutSeconds` (опційно)
- `key` (опційний ключ автономного режиму)

Безпека:

- Без `key` програма запитує підтвердження.
- Без `key` програма застосовує коротке обмеження повідомлення для запиту підтвердження та ігнорує `deliver` / `to` / `channel`.
- Із дійсним `key` запуск відбувається автономно (призначено для особистих автоматизацій).

## Потік початкового налаштування (типовий)

1. Установіть і запустіть **OpenClaw.app**.
2. Завершіть контрольний список дозволів (запити TCC).
3. Переконайтеся, що режим **Локальний** активний, а Gateway запущено.
4. Установіть CLI, якщо потрібен доступ із термінала.

## Розміщення каталогу стану (macOS)

Уникайте розміщення каталогу стану OpenClaw в iCloud або інших папках, синхронізованих із хмарою.
Шляхи з підтримкою синхронізації можуть додавати затримку й іноді спричиняти перегони блокування файлів/синхронізації для
сесій і облікових даних.

Надавайте перевагу локальному несинхронізованому шляху стану, наприклад:
__OC_I18N_900005__
Якщо `openclaw doctor` виявить стан у:

- `~/Library/Mobile Documents/com~apple~CloudDocs/...`
- `~/Library/CloudStorage/...`

він попередить і порекомендує повернутися до локального шляху.

## Налагодження підключення Gateway (CLI macOS)

Використовуйте CLI для налагодження, щоб перевірити ті самі логіку рукостискання й виявлення
Gateway WebSocket, які використовує програма macOS, без запуску програми.
__OC_I18N_900006__
Параметри підключення:

- `--url <ws://host:port>`: перевизначити конфігурацію
- `--mode <local|remote>`: визначити з конфігурації (типово: конфігурація або локальний)
- `--probe`: примусово виконати нову перевірку працездатності
- `--timeout <ms>`: тайм-аут запиту (типово: `15000`)
- `--json`: структурований вивід для порівняння

Параметри виявлення:

- `--include-local`: включити Gateway, які було б відфільтровано як «локальні»
- `--timeout <ms>`: загальне вікно виявлення (типово: `2000`)
- `--json`: структурований вивід для порівняння

<Tip>
Порівняйте з `openclaw gateway discover --json`, щоб побачити, чи конвеєр виявлення програми macOS (`local.` плюс налаштований широкозонний домен, із fallback для wide-area та Tailscale Serve) відрізняється від виявлення CLI Node на основі `dns-sd`.
</Tip>

## Сантехніка віддаленого підключення (SSH-тунелі)

Коли програма macOS працює у **віддаленому** режимі, вона відкриває SSH-тунель, щоб локальні UI
компоненти могли спілкуватися з віддаленим Gateway так, ніби він був на localhost.

### Тунель керування (порт Gateway WebSocket)

- **Призначення:** перевірки працездатності, статус, вебчат, конфігурація та інші виклики площини керування.
- **Локальний порт:** порт Gateway (типово `18789`), завжди стабільний.
- **Віддалений порт:** той самий порт Gateway на віддаленому хості.
- **Поведінка:** без випадкового локального порту; програма повторно використовує наявний працездатний тунель
  або перезапускає його за потреби.
- **Форма SSH:** `ssh -N -L <local>:127.0.0.1:<remote>` з BatchMode +
  ExitOnForwardFailure + параметрами keepalive.
- **Звітування IP:** SSH-тунель використовує loopback, тому Gateway бачитиме IP Node
  як `127.0.0.1`. Використовуйте транспорт **Direct (ws/wss)**, якщо хочете, щоб відображався справжній IP клієнта
  (див. [віддалений доступ macOS](/uk/platforms/mac/remote)).

Кроки налаштування див. у [віддаленому доступі macOS](/uk/platforms/mac/remote). Деталі протоколу
див. у [протоколі Gateway](/uk/gateway/protocol).
