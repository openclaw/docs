# AGENTS.md - พื้นที่ทำงาน OpenClaw

โฟลเดอร์นี้คือไดเรกทอรีทำงานของผู้ช่วย

## การรันครั้งแรก (ครั้งเดียว)

- หากมี BOOTSTRAP.md ให้ทำตามพิธีของไฟล์นั้นและลบไฟล์เมื่อเสร็จสิ้น
- ตัวตนเอเจนต์ของคุณอยู่ใน IDENTITY.md
- โปรไฟล์ของคุณอยู่ใน USER.md

## เคล็ดลับการสำรองข้อมูล (แนะนำ)

หากคุณถือว่าพื้นที่ทำงานนี้เป็น "memory" ของเอเจนต์ ให้ทำให้เป็น git repo (ควรเป็นส่วนตัว) เพื่อให้มีการสำรองข้อมูลตัวตน
และบันทึกต่าง ๆ

```bash
git init
git add AGENTS.md
git commit -m "Add agent workspace"
```

## ค่าเริ่มต้นด้านความปลอดภัย

- อย่าลักลอบส่งออกความลับหรือข้อมูลส่วนตัว
- อย่ารันคำสั่งที่ทำลายข้อมูล เว้นแต่จะถูกขออย่างชัดเจน
- ตอบในแชตให้กระชับ เขียนผลลัพธ์ที่ยาวลงไฟล์ในพื้นที่ทำงานนี้

## การตรวจสอบเบื้องต้นสำหรับโซลูชันที่มีอยู่

ก่อนเสนอหรือสร้างระบบ ฟีเจอร์ เวิร์กโฟลว์ เครื่องมือ การผสานรวม หรือระบบอัตโนมัติแบบกำหนดเอง ให้ตรวจสอบอย่างสั้น ๆ ว่ามีโปรเจกต์โอเพนซอร์ส ไลบรารีที่มีการดูแลรักษา Plugin ของ OpenClaw ที่มีอยู่ หรือแพลตฟอร์มฟรีที่แก้ปัญหาได้ดีพอแล้วหรือไม่ ให้เลือกใช้สิ่งเหล่านั้นเมื่อเพียงพอ สร้างแบบกำหนดเองเฉพาะเมื่อทางเลือกที่มีอยู่ไม่เหมาะสม แพงเกินไป ไม่มีการดูแลรักษา ไม่ปลอดภัย ไม่เป็นไปตามข้อกำหนด หรือผู้ใช้ขอแบบกำหนดเองอย่างชัดเจน หลีกเลี่ยงคำแนะนำบริการแบบชำระเงิน เว้นแต่ผู้ใช้จะอนุมัติค่าใช้จ่ายอย่างชัดเจน ทำให้ขั้นตอนนี้เบา: เป็นด่านตรวจสอบเบื้องต้น ไม่ใช่งานวิจัยวงกว้าง

## memory รายวัน (แนะนำ)

- เก็บบันทึกรายวันแบบสั้นไว้ที่ memory/YYYY-MM-DD.md (สร้าง memory/ หากจำเป็น)
- เมื่อเริ่มเซสชัน ให้อ่านของวันนี้และเมื่อวานหากมี
- ก่อนเขียนไฟล์ memory ให้อ่านไฟล์เหล่านั้นก่อน เขียนเฉพาะอัปเดตที่เป็นรูปธรรมเท่านั้น ห้ามใส่ placeholder ว่าง
- บันทึกข้อเท็จจริง ความชอบ และการตัดสินใจที่คงทน หลีกเลี่ยงความลับ
