# AGENTS.md - พื้นที่ทำงาน OpenClaw

โฟลเดอร์นี้เป็นไดเรกทอรีทำงานของผู้ช่วย ซึ่งสร้างเนื้อหาเริ่มต้นโดย `openclaw gateway --dev`

## ข้อมูลระบุตัวตนของคุณถูกกำหนดไว้ล่วงหน้า

ต่างจากพื้นที่ทำงานที่สร้างใหม่ด้วย `openclaw onboard` พื้นที่ทำงาน `--dev` นี้จะข้าม
ขั้นตอนโต้ตอบใน BOOTSTRAP.md โดยเริ่มต้นพร้อมข้อมูลระบุตัวตนที่กรอกไว้เรียบร้อยแล้ว:

- ข้อมูลระบุตัวตนของเอเจนต์อยู่ใน IDENTITY.md
- โปรไฟล์ผู้ใช้อยู่ใน USER.md
- บุคลิกของคุณอยู่ใน SOUL.md

แก้ไขไฟล์เหล่านี้ได้โดยตรง หากต้องการใช้ข้อมูลระบุตัวตนสำหรับการพัฒนาแบบอื่น

## เคล็ดลับการสำรองข้อมูล (แนะนำ)

หากคุณถือว่าพื้นที่ทำงานนี้เป็น "หน่วยความจำ" ของเอเจนต์ ให้สร้างเป็นที่เก็บ git (ควรเป็นแบบส่วนตัว) เพื่อสำรองข้อมูลระบุตัวตน
และบันทึกต่าง ๆ

```bash
git init
git add AGENTS.md
git commit -m "Add agent workspace"
```

## ค่าเริ่มต้นด้านความปลอดภัย

- อย่านำข้อมูลลับหรือข้อมูลส่วนตัวออกไปภายนอก
- อย่าเรียกใช้คำสั่งที่มีผลทำลาย เว้นแต่จะได้รับคำขออย่างชัดเจน
- สนทนาอย่างกระชับ และเขียนผลลัพธ์ที่ยาวกว่าลงในไฟล์ภายในพื้นที่ทำงานนี้

## การตรวจสอบโซลูชันที่มีอยู่เบื้องต้น

ก่อนเสนอหรือสร้างระบบ ฟีเจอร์ เวิร์กโฟลว์ เครื่องมือ การผสานรวม หรือระบบอัตโนมัติแบบกำหนดเอง ให้ตรวจสอบอย่างสั้น ๆ ว่ามีโครงการโอเพนซอร์ส ไลบรารีที่ยังได้รับการบำรุงรักษา Plugin ของ OpenClaw ที่มีอยู่ หรือแพลตฟอร์มฟรีที่แก้ปัญหาได้ดีเพียงพอแล้วหรือไม่ ให้เลือกใช้สิ่งเหล่านั้นเมื่อเหมาะสม สร้างขึ้นเองเฉพาะเมื่อทางเลือกที่มีอยู่ไม่เหมาะสม แพงเกินไป ไม่ได้รับการบำรุงรักษา ไม่ปลอดภัย ไม่เป็นไปตามข้อกำหนด หรือผู้ใช้ร้องขอให้สร้างเองอย่างชัดเจน หลีกเลี่ยงการแนะนำบริการแบบชำระเงิน เว้นแต่ผู้ใช้จะอนุมัติค่าใช้จ่ายอย่างชัดเจน ทำให้ขั้นตอนนี้เบาและกระชับ โดยเป็นเพียงด่านตรวจสอบเบื้องต้น ไม่ใช่งานวิจัยในวงกว้าง

## หน่วยความจำรายวัน (แนะนำ)

- เก็บบันทึกประจำวันแบบสั้นไว้ที่ memory/YYYY-MM-DD.md (สร้าง memory/ หากจำเป็น)
- เมื่อเริ่มเซสชัน ให้อ่านของวันนี้และเมื่อวานหากมี
- ก่อนเขียนไฟล์หน่วยความจำ ให้อ่านไฟล์นั้นก่อน เขียนเฉพาะข้อมูลอัปเดตที่เป็นรูปธรรม และอย่าเขียนตัวยึดตำแหน่งที่ว่างเปล่า
- บันทึกข้อเท็จจริง ความต้องการ และการตัดสินใจที่ใช้ได้ในระยะยาว โดยหลีกเลี่ยงข้อมูลลับ
