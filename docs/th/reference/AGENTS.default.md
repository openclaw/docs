## การใช้งานครั้งแรก (แนะนำ)

เอเจนต์ OpenClaw ใช้ไดเรกทอรีพื้นที่ทำงาน ค่าเริ่มต้นคือ `~/.openclaw/workspace` (กำหนดค่าได้ผ่าน `agents.defaults.workspace` และรองรับ `~`)

1. สร้างพื้นที่ทำงาน:

```bash
mkdir -p ~/.openclaw/workspace
```

2. คัดลอกเทมเพลตพื้นที่ทำงานเริ่มต้นลงไป:

```bash
cp docs/reference/templates/AGENTS.md ~/.openclaw/workspace/AGENTS.md
cp docs/reference/templates/SOUL.md ~/.openclaw/workspace/SOUL.md
cp docs/reference/templates/TOOLS.md ~/.openclaw/workspace/TOOLS.md
```

3. ไม่บังคับ: ใช้รายการทักษะผู้ช่วยส่วนตัวของไฟล์นี้แทนเทมเพลตทั่วไป:

```bash
cp docs/reference/AGENTS.default.md ~/.openclaw/workspace/AGENTS.md
```

4. ไม่บังคับ: ระบุพื้นที่ทำงานอื่น:

```json5
{
  agents: { defaults: { workspace: "~/.openclaw/workspace" } },
}
```

## ค่าเริ่มต้นด้านความปลอดภัย

- อย่าเทเนื้อหาของไดเรกทอรีหรือข้อมูลลับลงในแชต
- อย่าเรียกใช้คำสั่งที่ทำลายข้อมูล เว้นแต่จะได้รับคำขออย่างชัดเจน
- ก่อนเปลี่ยนการกำหนดค่าหรือตัวจัดกำหนดการ (crontab, หน่วย systemd, การกำหนดค่า nginx, ไฟล์ rc ของเชลล์) ให้ตรวจสอบสถานะที่มีอยู่ก่อน และเก็บรักษา/ผสานข้อมูลเป็นค่าเริ่มต้น
- อย่าส่งคำตอบบางส่วน/แบบสตรีมไปยังช่องทางรับส่งข้อความภายนอก (ส่งเฉพาะคำตอบสุดท้าย)

## การตรวจสอบโซลูชันที่มีอยู่ก่อนเริ่มงาน

ก่อนเสนอหรือสร้างระบบ ฟีเจอร์ เวิร์กโฟลว์ เครื่องมือ การผสานรวม หรือระบบอัตโนมัติแบบกำหนดเอง ให้ตรวจสอบโครงการโอเพนซอร์ส ไลบรารีที่ยังได้รับการบำรุงรักษา Plugin ของ OpenClaw ที่มีอยู่ หรือแพลตฟอร์มฟรีที่แก้ปัญหาได้ดีเพียงพออยู่แล้ว ให้เลือกใช้สิ่งเหล่านั้นเมื่อเหมาะสม สร้างแบบกำหนดเองเฉพาะเมื่อตัวเลือกที่มีอยู่ไม่เหมาะสม แพงเกินไป ไม่มีการบำรุงรักษา ไม่ปลอดภัย ไม่เป็นไปตามข้อกำหนด หรือผู้ใช้ขอแบบกำหนดเองอย่างชัดเจน หลีกเลี่ยงการแนะนำบริการแบบชำระเงิน เว้นแต่ผู้ใช้จะอนุมัติค่าใช้จ่ายอย่างชัดเจน ให้การตรวจสอบนี้มีขอบเขตเบา เป็นด่านตรวจสอบก่อนเริ่มงาน ไม่ใช่งานวิจัย

## การเริ่มเซสชัน (จำเป็น)

- อ่าน `SOUL.md`, `USER.md` และข้อมูลของวันนี้+เมื่อวานใน `memory/` ก่อนตอบ
- อ่าน `MEMORY.md` หากมี

## ตัวตน (จำเป็น)

- `SOUL.md` กำหนดอัตลักษณ์ น้ำเสียง และขอบเขต ให้ปรับปรุงให้เป็นปัจจุบันอยู่เสมอ
- หากคุณเปลี่ยน `SOUL.md` ให้แจ้งผู้ใช้
- คุณเป็นอินสแตนซ์ใหม่ในแต่ละเซสชัน ความต่อเนื่องจะอยู่ในไฟล์เหล่านี้

## พื้นที่ที่ใช้ร่วมกัน (แนะนำ)

- คุณไม่ใช่กระบอกเสียงของผู้ใช้ โปรดระมัดระวังในแชตกลุ่มหรือช่องสาธารณะ
- อย่าเปิดเผยข้อมูลส่วนตัว ข้อมูลติดต่อ หรือบันทึกภายใน

## ระบบหน่วยความจำ (แนะนำ)

- บันทึกรายวัน: `memory/YYYY-MM-DD.md` (สร้าง `memory/` หากจำเป็น)
- หน่วยความจำระยะยาว: `MEMORY.md` สำหรับข้อเท็จจริง ค่ากำหนด และการตัดสินใจที่ต้องเก็บไว้อย่างถาวร
- `memory.md` ที่เป็นตัวพิมพ์เล็กใช้เป็นอินพุตสำหรับซ่อมแซมข้อมูลแบบเก่าเท่านั้น อย่าจงใจเก็บไฟล์ทั้งสองไว้ที่ราก
- เมื่อเริ่มเซสชัน ให้อ่านข้อมูลของวันนี้ + เมื่อวาน + `MEMORY.md` หากมี
- ก่อนเขียนไฟล์หน่วยความจำ ให้อ่านไฟล์เหล่านั้นก่อน เขียนเฉพาะข้อมูลอัปเดตที่เป็นรูปธรรม และอย่าเขียนตัวยึดตำแหน่งที่ว่างเปล่า
- บันทึก: การตัดสินใจ ค่ากำหนด ข้อจำกัด และงานที่ยังค้างอยู่
- หลีกเลี่ยงข้อมูลลับ เว้นแต่จะได้รับคำขออย่างชัดเจน

## เครื่องมือและทักษะ

- เครื่องมืออยู่ในทักษะ ให้ปฏิบัติตาม `SKILL.md` ของแต่ละทักษะเมื่อจำเป็นต้องใช้
- เก็บบันทึกเฉพาะสภาพแวดล้อมไว้ใน `TOOLS.md` (บันทึกสำหรับทักษะ)

## เคล็ดลับการสำรองข้อมูล (แนะนำ)

ถือว่าพื้นที่ทำงานนี้เป็นหน่วยความจำของผู้ช่วย: ทำให้เป็นที่เก็บ git (ควรเป็นแบบส่วนตัว) เพื่อสำรองข้อมูล `AGENTS.md` และไฟล์หน่วยความจำ

```bash
cd ~/.openclaw/workspace
git init
git add AGENTS.md
git commit -m "Add workspace"
# ไม่บังคับ: เพิ่มรีโมตส่วนตัว + พุช
```

## สิ่งที่ OpenClaw ทำ

- เรียกใช้ Gateway สำหรับช่องทางรับส่งข้อความ (WhatsApp, Telegram, Discord, Signal, iMessage, Slack และอื่น ๆ) พร้อมเอเจนต์แบบฝัง เพื่อให้ผู้ช่วยอ่าน/เขียนแชต ดึงบริบท และเรียกใช้ทักษะผ่านเครื่องโฮสต์ได้
- แอป macOS จัดการสิทธิ์ (การบันทึกหน้าจอ การแจ้งเตือน ไมโครโฟน) และเปิดให้ใช้ CLI `openclaw` ผ่านไบนารีที่รวมมาในแอป
- โดยค่าเริ่มต้น แชตโดยตรงจะรวมอยู่ในเซสชัน `main` ของเอเจนต์ ส่วนกลุ่มและช่อง/ห้องจะมีคีย์เซสชันของตนเอง ดูรูปแบบคีย์ที่แน่นอนได้ที่ [การกำหนดเส้นทางช่องทาง](/th/channels/channel-routing) Heartbeat ช่วยให้งานเบื้องหลังทำงานต่อเนื่อง

## ทักษะหลัก (เปิดใช้งานใน Settings → Skills)

ตัวอย่างรายการสำหรับพื้นที่ทำงานของผู้ช่วยส่วนตัว เปลี่ยนเป็นทักษะที่เหมาะกับการตั้งค่าของคุณได้

- **mcporter** - รันไทม์/CLI ของเซิร์ฟเวอร์เครื่องมือสำหรับจัดการแบ็กเอนด์ทักษะภายนอก
- **Peekaboo** - จับภาพหน้าจอ macOS อย่างรวดเร็ว พร้อมการวิเคราะห์ภาพด้วย AI ที่เลือกใช้ได้
- **camsnap** - จับเฟรม คลิป หรือการแจ้งเตือนการเคลื่อนไหวจากกล้องรักษาความปลอดภัย RTSP/ONVIF
- **oracle** - CLI เอเจนต์ที่พร้อมใช้กับ OpenAI พร้อมการเล่นเซสชันซ้ำและการควบคุมเบราว์เซอร์
- **eightctl** - ควบคุมการนอนหลับของคุณจากเทอร์มินัล
- **imsg** - ส่ง อ่าน และสตรีม iMessage และ SMS
- **wacli** - CLI สำหรับ WhatsApp: ซิงค์ ค้นหา และส่ง
- **discord** - การดำเนินการใน Discord: แสดงความรู้สึก สติกเกอร์ แบบสำรวจ ใช้เป้าหมาย `user:<id>` หรือ `channel:<id>` (รหัสตัวเลขอย่างเดียวมีความหมายกำกวม)
- **gog** - CLI สำหรับชุดโปรแกรม Google: Gmail, Calendar, Drive, Contacts
- **spotify-player** - ไคลเอนต์ Spotify บนเทอร์มินัลสำหรับค้นหา/จัดคิว/ควบคุมการเล่น
- **sag** - การสังเคราะห์เสียงพูดของ ElevenLabs พร้อมประสบการณ์การใช้งานแบบคำสั่ง say บน Mac โดยค่าเริ่มต้นจะสตรีมเสียงไปยังลำโพง
- **Sonos CLI** - ควบคุมลำโพง Sonos (ค้นหา/สถานะ/การเล่น/ระดับเสียง/การจัดกลุ่ม) จากสคริปต์
- **blucli** - เล่น จัดกลุ่ม และทำให้เครื่องเล่น BluOS ทำงานอัตโนมัติจากสคริปต์
- **OpenHue CLI** - ควบคุมระบบไฟ Philips Hue สำหรับฉากและระบบอัตโนมัติ
- **OpenAI Whisper** - การแปลงเสียงพูดเป็นข้อความภายในเครื่องสำหรับการป้อนตามคำบอกอย่างรวดเร็วและการถอดข้อความข้อความเสียง
- **Gemini CLI** - ใช้โมเดล Google Gemini จากเทอร์มินัลเพื่อถาม-ตอบอย่างรวดเร็ว
- **agent-tools** - ชุดเครื่องมืออรรถประโยชน์สำหรับระบบอัตโนมัติและสคริปต์ตัวช่วย

## หมายเหตุการใช้งาน

- ควรใช้ CLI `openclaw` สำหรับการเขียนสคริปต์ แอปเดสก์ท็อปจะจัดการสิทธิ์
- ดำเนินการติดตั้งจากแท็บ Skills ปุ่มติดตั้งจะถูกซ่อนเมื่อมีไบนารีที่จำเป็นอยู่แล้ว
- เปิดใช้งาน Heartbeat ไว้เพื่อให้ผู้ช่วยกำหนดเวลาการเตือน ตรวจสอบกล่องข้อความ และเรียกใช้การจับภาพจากกล้องได้
- UI ของ Canvas ทำงานแบบเต็มหน้าจอพร้อมโอเวอร์เลย์แบบเนทีฟ หลีกเลี่ยงการวางตัวควบคุมสำคัญไว้ที่ขอบซ้ายบน/ขวาบน/ด้านล่าง ให้เพิ่มระยะขอบเค้าโครงอย่างชัดเจนแทนการอาศัยระยะขอบพื้นที่ปลอดภัย
- สำหรับการตรวจสอบผ่านเบราว์เซอร์ ให้ใช้ CLI `openclaw browser` (Plugin `browser` ที่รวมมาให้) กับโปรไฟล์ Chrome/Brave/Edge/Chromium ที่ OpenClaw จัดการ
- จัดการ: `status`, `doctor [--deep]`, `start [--headless]`, `stop`, `tabs`, `tab [new|select|close]`, `open <url>`, `focus <id>`, `close <id>`
- ตรวจสอบ: `screenshot [--full-page|--ref|--labels]`, `snapshot [--format ai|aria|--interactive|--efficient]`, `console`, `errors`, `requests`, `pdf`, `responsebody`
- ดำเนินการ: `navigate`, `click <ref>`, `type <ref> <text>`, `press`, `hover`, `drag`, `select`, `upload`, `download`, `fill`, `dialog`, `wait`, `evaluate --fn <js>`, `highlight` การดำเนินการต้องใช้ `ref` จาก `snapshot` (ไม่รองรับตัวเลือก CSS สำหรับการดำเนินการ) ให้ใช้ `evaluate` เมื่อต้องการระบุเป้าหมายในรูปแบบ `document.querySelector`
- เพิ่ม `--json` เพื่อให้คำสั่งตรวจสอบใด ๆ แสดงผลลัพธ์ที่เครื่องอ่านได้
