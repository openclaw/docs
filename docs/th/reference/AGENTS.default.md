## การเรียกใช้ครั้งแรก (แนะนำ)

OpenClaw ใช้ไดเรกทอรีพื้นที่ทำงานเฉพาะสำหรับเอเจนต์ ค่าเริ่มต้น: `~/.openclaw/workspace` (กำหนดค่าได้ผ่าน `agents.defaults.workspace`)

1. สร้างพื้นที่ทำงาน (ถ้ายังไม่มี):

```bash
mkdir -p ~/.openclaw/workspace
```

2. คัดลอกเทมเพลตพื้นที่ทำงานเริ่มต้นเข้าไปในพื้นที่ทำงาน:

```bash
cp docs/reference/templates/AGENTS.md ~/.openclaw/workspace/AGENTS.md
cp docs/reference/templates/SOUL.md ~/.openclaw/workspace/SOUL.md
cp docs/reference/templates/TOOLS.md ~/.openclaw/workspace/TOOLS.md
```

3. ไม่บังคับ: หากคุณต้องการรายชื่อ Skills สำหรับผู้ช่วยส่วนตัว ให้แทนที่ AGENTS.md ด้วยไฟล์นี้:

```bash
cp docs/reference/AGENTS.default.md ~/.openclaw/workspace/AGENTS.md
```

4. ไม่บังคับ: เลือกพื้นที่ทำงานอื่นโดยตั้งค่า `agents.defaults.workspace` (รองรับ `~`):

```json5
{
  agents: { defaults: { workspace: "~/.openclaw/workspace" } },
}
```

## ค่าเริ่มต้นด้านความปลอดภัย

- อย่าเทไดเรกทอรีหรือความลับลงในแชต
- อย่ารันคำสั่งที่ทำลายข้อมูล เว้นแต่จะถูกขออย่างชัดเจน
- อย่าส่งคำตอบบางส่วน/แบบสตรีมไปยังพื้นผิวการส่งข้อความภายนอก (ส่งเฉพาะคำตอบสุดท้ายเท่านั้น)

## การเริ่มเซสชัน (จำเป็น)

- อ่าน `SOUL.md`, `USER.md` และวันนี้+เมื่อวานใน `memory/`
- อ่าน `MEMORY.md` เมื่อมีอยู่
- ทำสิ่งนี้ก่อนตอบกลับ

## Soul (จำเป็น)

- `SOUL.md` กำหนดตัวตน น้ำเสียง และขอบเขต ให้รักษาให้เป็นปัจจุบัน
- หากคุณเปลี่ยน `SOUL.md` ให้บอกผู้ใช้
- คุณคืออินสแตนซ์ใหม่ในแต่ละเซสชัน ความต่อเนื่องอยู่ในไฟล์เหล่านี้

## พื้นที่ที่ใช้ร่วมกัน (แนะนำ)

- คุณไม่ใช่เสียงของผู้ใช้ โปรดระมัดระวังในแชตกลุ่มหรือช่องสาธารณะ
- อย่าแชร์ข้อมูลส่วนตัว ข้อมูลติดต่อ หรือบันทึกภายใน

## ระบบหน่วยความจำ (แนะนำ)

- บันทึกรายวัน: `memory/YYYY-MM-DD.md` (สร้าง `memory/` หากจำเป็น)
- หน่วยความจำระยะยาว: `MEMORY.md` สำหรับข้อเท็จจริง ความชอบ และการตัดสินใจที่คงทน
- `memory.md` ตัวพิมพ์เล็กเป็นอินพุตซ่อมแซมแบบเดิมเท่านั้น อย่าเก็บไฟล์รากทั้งสองไฟล์ไว้โดยตั้งใจ
- เมื่อเริ่มเซสชัน ให้อ่านวันนี้ + เมื่อวาน + `MEMORY.md` เมื่อมีอยู่
- บันทึก: การตัดสินใจ ความชอบ ข้อจำกัด งานค้าง
- หลีกเลี่ยงความลับ เว้นแต่จะถูกขออย่างชัดเจน

## เครื่องมือและ Skills

- เครื่องมืออยู่ใน Skills ให้ทำตาม `SKILL.md` ของแต่ละ Skill เมื่อคุณจำเป็นต้องใช้
- เก็บบันทึกเฉพาะสภาพแวดล้อมไว้ใน `TOOLS.md` (บันทึกสำหรับ Skills)

## เคล็ดลับการสำรองข้อมูล (แนะนำ)

หากคุณถือว่าพื้นที่ทำงานนี้เป็น "หน่วยความจำ" ของ Clawd ให้ทำให้เป็นรีโพ git (ควรเป็นแบบส่วนตัว) เพื่อให้ `AGENTS.md` และไฟล์หน่วยความจำของคุณได้รับการสำรองข้อมูล

```bash
cd ~/.openclaw/workspace
git init
git add AGENTS.md
git commit -m "Add Clawd workspace"
# Optional: add a private remote + push
```

## สิ่งที่ OpenClaw ทำ

- รัน WhatsApp gateway + เอเจนต์เขียนโค้ด Pi เพื่อให้ผู้ช่วยสามารถอ่าน/เขียนแชต ดึงบริบท และรัน Skills ผ่าน Mac โฮสต์ได้
- แอป macOS จัดการสิทธิ์ (การบันทึกหน้าจอ การแจ้งเตือน ไมโครโฟน) และเปิดให้ใช้ `openclaw` CLI ผ่านไบนารีที่บันเดิลมา
- โดยค่าเริ่มต้น แชตโดยตรงจะถูกรวมเข้ากับเซสชัน `main` ของเอเจนต์ ส่วนกลุ่มจะแยกอยู่เป็น `agent:<agentId>:<channel>:group:<id>` (ห้อง/ช่อง: `agent:<agentId>:<channel>:channel:<id>`); Heartbeat ช่วยให้งานเบื้องหลังยังทำงานอยู่

## หมายเหตุการใช้งาน

- ควรใช้ `openclaw` CLI สำหรับการเขียนสคริปต์ แอป Mac จะจัดการสิทธิ์
- รันการติดตั้งจากแท็บ Skills; ระบบจะซ่อนปุ่มหากมีไบนารีอยู่แล้ว
- เปิดใช้ Heartbeat ไว้เพื่อให้ผู้ช่วยสามารถตั้งเวลาการเตือน ติดตามกล่องขาเข้า และทริกเกอร์การจับภาพจากกล้องได้
- Canvas UI ทำงานเต็มหน้าจอพร้อมโอเวอร์เลย์เนทีฟ หลีกเลี่ยงการวางคอนโทรลสำคัญไว้ที่มุมบนซ้าย/บนขวา/ขอบล่าง เพิ่มระยะขอบในเลย์เอาต์อย่างชัดเจน และอย่าพึ่งพา safe-area insets
- สำหรับการตรวจสอบที่ขับเคลื่อนด้วยเบราว์เซอร์ ให้ใช้ `openclaw browser` (แท็บ/สถานะ/ภาพหน้าจอ) กับโปรไฟล์ Chrome ที่ OpenClaw จัดการ
- สำหรับการตรวจสอบ DOM ให้ใช้ `openclaw browser eval|query|dom|snapshot` (และ `--json`/`--out` เมื่อคุณต้องการเอาต์พุตสำหรับเครื่อง)
- สำหรับการโต้ตอบ ให้ใช้ `openclaw browser click|type|hover|drag|select|upload|press|wait|navigate|back|evaluate|run` (click/type ต้องใช้ snapshot refs; ใช้ `evaluate` สำหรับ CSS selectors)
