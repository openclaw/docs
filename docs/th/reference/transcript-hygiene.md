OpenClaw ใช้ **การแก้ไขเฉพาะผู้ให้บริการ** กับทรานสคริปต์ก่อนการรัน (การสร้างบริบทโมเดล) การปรับส่วนใหญ่เป็นการปรับ **ในหน่วยความจำ** เพื่อให้ตรงตามข้อกำหนดที่เข้มงวดของผู้ให้บริการ อีกพาสซ่อมแซมไฟล์เซสชันแบบแยกต่างหากอาจเขียน JSONL ที่จัดเก็บไว้ใหม่ก่อนโหลดเซสชันด้วย โดยอาจลบบรรทัด JSONL ที่ผิดรูปแบบ หรือซ่อมเทิร์นที่บันทึกไว้ซึ่งถูกต้องตามไวยากรณ์แต่ทราบว่าจะถูก
ผู้ให้บริการปฏิเสธระหว่างการเล่นซ้ำ เมื่อมีการซ่อมแซม ไฟล์ต้นฉบับจะถูกสำรองไว้ข้าง
ไฟล์เซสชัน

ขอบเขตรวมถึง:

- บริบทพรอมป์เฉพาะรันไทม์ไม่เข้าไปอยู่ในเทิร์นทรานสคริปต์ที่ผู้ใช้มองเห็น
- การทำให้ id ของการเรียกเครื่องมือปลอดภัย
- การตรวจสอบอินพุตของการเรียกเครื่องมือ
- การซ่อมแซมการจับคู่ผลลัพธ์เครื่องมือ
- การตรวจสอบความถูกต้อง / การจัดลำดับเทิร์น
- การล้างลายเซ็นความคิด
- การล้างลายเซ็นการคิด
- การทำให้เพย์โหลดรูปภาพปลอดภัย
- การล้างบล็อกข้อความว่างก่อนเล่นซ้ำกับผู้ให้บริการ
- การติดแท็กที่มาของอินพุตผู้ใช้ (สำหรับพรอมป์ที่ถูกส่งต่อระหว่างเซสชัน)
- การซ่อมแซมเทิร์นข้อผิดพลาดของ assistant ที่ว่างเปล่าสำหรับการเล่นซ้ำ Bedrock Converse

หากคุณต้องการรายละเอียดการจัดเก็บทรานสคริปต์ โปรดดู:

- [เจาะลึกการจัดการเซสชัน](/th/reference/session-management-compaction)

## กฎสากล: บริบทรันไทม์ไม่ใช่ทรานสคริปต์ของผู้ใช้

สามารถเพิ่มบริบทรันไทม์/ระบบลงในพรอมป์โมเดลสำหรับเทิร์นหนึ่งได้ แต่บริบทนั้น
ไม่ใช่เนื้อหาที่ผู้ใช้ปลายทางเขียน OpenClaw เก็บเนื้อหาพรอมป์สำหรับทรานสคริปต์
แยกต่างหากสำหรับการตอบกลับของ Gateway, followup ที่เข้าคิว, ACP, CLI และการรัน Pi
แบบฝัง เทิร์นผู้ใช้ที่มองเห็นได้ซึ่งถูกจัดเก็บจะใช้เนื้อหาทรานสคริปต์นั้นแทน
พรอมป์ที่เติมบริบทรันไทม์แล้ว

สำหรับเซสชันเก่าที่บันทึก wrapper รันไทม์ไว้แล้ว พื้นผิวประวัติของ Gateway
จะใช้การฉายผลสำหรับการแสดงผลก่อนส่งข้อความกลับไปยังไคลเอนต์ WebChat,
TUI, REST หรือ SSE

## สิ่งนี้รันที่ไหน

การดูแลความสะอาดของทรานสคริปต์ทั้งหมดถูกรวมศูนย์ไว้ใน embedded runner:

- การเลือกนโยบาย: `src/agents/transcript-policy.ts`
- การใช้การทำให้ปลอดภัย/การซ่อมแซม: `sanitizeSessionHistory` ใน `src/agents/pi-embedded-runner/replay-history.ts`

นโยบายใช้ `provider`, `modelApi` และ `modelId` เพื่อตัดสินใจว่าจะใช้อะไร

แยกจากการดูแลความสะอาดของทรานสคริปต์ ไฟล์เซสชันจะถูกซ่อมแซม (หากจำเป็น) ก่อนโหลด:

- `repairSessionFileIfNeeded` ใน `src/agents/session-file-repair.ts`
- ถูกเรียกจาก `run/attempt.ts` และ `compact.ts` (embedded runner)

## กฎสากล: การทำให้รูปภาพปลอดภัย

เพย์โหลดรูปภาพจะถูกทำให้ปลอดภัยเสมอเพื่อป้องกันการถูกปฏิเสธฝั่งผู้ให้บริการเนื่องจากข้อจำกัด
ด้านขนาด (ลดขนาด/บีบอัดรูปภาพ base64 ที่ใหญ่เกินใหม่)

สิ่งนี้ยังช่วยควบคุมแรงกดดันด้านโทเคนจากรูปภาพสำหรับโมเดลที่รองรับวิชันด้วย
ขนาดสูงสุดที่ต่ำกว่ามักลดการใช้โทเคน ส่วนขนาดที่สูงกว่าจะรักษารายละเอียดไว้

การใช้งาน:

- `sanitizeSessionMessagesImages` ใน `src/agents/pi-embedded-helpers/images.ts`
- `sanitizeContentBlocksImages` ใน `src/agents/tool-images.ts`
- ด้านยาวสุดของรูปภาพกำหนดค่าได้ผ่าน `agents.defaults.imageMaxDimensionPx` (ค่าเริ่มต้น: `1200`)
- บล็อกข้อความว่างจะถูกลบระหว่างที่พาสนี้เดินผ่านเนื้อหาการเล่นซ้ำ เทิร์น assistant
  ที่กลายเป็นว่างจะถูกลบออกจากสำเนาการเล่นซ้ำ เทิร์นผู้ใช้และผลลัพธ์เครื่องมือ
  ที่กลายเป็นว่างจะได้รับ placeholder เนื้อหาที่ถูกละไว้ซึ่งไม่ว่างเปล่า

## กฎสากล: การเรียกเครื่องมือผิดรูปแบบ

บล็อกการเรียกเครื่องมือของ assistant ที่ไม่มีทั้ง `input` และ `arguments` จะถูกลบออก
ก่อนสร้างบริบทโมเดล สิ่งนี้ป้องกันการปฏิเสธจากผู้ให้บริการเนื่องจากการเรียกเครื่องมือที่บันทึกไว้
เพียงบางส่วน (ตัวอย่างเช่น หลังเกิดความล้มเหลวจาก rate limit)

การใช้งาน:

- `sanitizeToolCallInputs` ใน `src/agents/session-transcript-repair.ts`
- ใช้ใน `sanitizeSessionHistory` ใน `src/agents/pi-embedded-runner/replay-history.ts`

## กฎสากล: ที่มาของอินพุตระหว่างเซสชัน

เมื่อเอเจนต์ส่งพรอมป์เข้าไปยังอีกเซสชันผ่าน `sessions_send` (รวมถึง
ขั้นตอนตอบกลับ/ประกาศระหว่างเอเจนต์) OpenClaw จะบันทึกเทิร์นผู้ใช้ที่สร้างขึ้นพร้อมกับ:

- `message.provenance.kind = "inter_session"`

OpenClaw ยังเติม marker `[Inter-session message ... isUser=false]` ไว้ในเทิร์นเดียวกัน
ก่อนข้อความพรอมป์ที่ถูกส่งต่อ เพื่อให้การเรียกโมเดลที่ใช้งานอยู่แยกแยะ
เอาต์พุตจากเซสชันอื่นออกจากคำสั่งของผู้ใช้ปลายทางภายนอกได้ marker นี้รวม
เซสชันต้นทาง ช่องทาง และเครื่องมือเมื่อมี ทรานสคริปต์ยังคงใช้
`role: "user"` เพื่อความเข้ากันได้กับผู้ให้บริการ แต่ทั้งข้อความที่มองเห็นได้และเมตาดาต้าที่มา
ต่างระบุว่าเทิร์นนั้นเป็นข้อมูลระหว่างเซสชัน

ระหว่างการสร้างบริบทใหม่ OpenClaw จะใช้ marker เดียวกันกับเทิร์นผู้ใช้ระหว่างเซสชัน
ที่บันทึกไว้เก่ากว่าซึ่งมีเฉพาะเมตาดาต้าที่มา

## เมทริกซ์ผู้ให้บริการ (พฤติกรรมปัจจุบัน)

**OpenAI / OpenAI Codex**

- การทำให้รูปภาพปลอดภัยเท่านั้น
- ลบลายเซ็น reasoning กำพร้า (รายการ reasoning แบบเดี่ยวที่ไม่มีบล็อกเนื้อหาตามมา) สำหรับทรานสคริปต์ OpenAI Responses/Codex และลบ OpenAI reasoning ที่เล่นซ้ำได้หลังสลับเส้นทางโมเดล
- เก็บเพย์โหลดรายการ reasoning ของ OpenAI Responses ที่เล่นซ้ำได้ไว้ รวมถึงรายการสรุปว่างที่เข้ารหัส เพื่อให้การเล่นซ้ำแบบ manual/WebSocket เก็บสถานะ `rs_*` ที่จำเป็นให้จับคู่กับรายการเอาต์พุตของ assistant
- ไม่มีการทำให้ id ของการเรียกเครื่องมือปลอดภัย
- การซ่อมแซมการจับคู่ผลลัพธ์เครื่องมืออาจย้ายเอาต์พุตจริงที่จับคู่แล้ว และสร้างเอาต์พุต `aborted` แบบ Codex สำหรับการเรียกเครื่องมือที่ขาดหาย
- ไม่มีการตรวจสอบความถูกต้องหรือจัดลำดับเทิร์นใหม่
- เอาต์พุตเครื่องมือในตระกูล OpenAI Responses ที่ขาดหายจะถูกสร้างเป็น `aborted` เพื่อให้ตรงกับการทำให้การเล่นซ้ำ Codex เป็นมาตรฐาน
- ไม่มีการลบลายเซ็นความคิด

**Gemma 4 ที่เข้ากันได้กับ OpenAI**

- บล็อก thinking/reasoning ของ assistant ในอดีตจะถูกลบออกก่อนเล่นซ้ำ เพื่อให้เซิร์ฟเวอร์ Gemma 4
  ที่เข้ากันได้กับ OpenAI แบบโลคัลไม่ได้รับเนื้อหา reasoning จากเทิร์นก่อนหน้า
- continuation ของการเรียกเครื่องมือในเทิร์นเดียวกันปัจจุบันจะเก็บบล็อก reasoning ของ assistant
  ที่แนบกับการเรียกเครื่องมือไว้จนกว่าผลลัพธ์เครื่องมือจะถูกเล่นซ้ำแล้ว

**Google (Generative AI / Gemini CLI / Antigravity)**

- การทำให้ id ของการเรียกเครื่องมือปลอดภัย: ตัวอักษรและตัวเลขอย่างเข้มงวด
- การซ่อมแซมการจับคู่ผลลัพธ์เครื่องมือและผลลัพธ์เครื่องมือสังเคราะห์
- การตรวจสอบความถูกต้องของเทิร์น (การสลับเทิร์นแบบ Gemini)
- การแก้ไขลำดับเทิร์นของ Google (เติม user bootstrap ขนาดเล็กไว้ด้านหน้า หากประวัติเริ่มด้วย assistant)
- Antigravity Claude: ทำให้ลายเซ็นการคิดเป็นมาตรฐาน; ลบบล็อกการคิดที่ไม่มีลายเซ็น

**Anthropic / Minimax (ที่เข้ากันได้กับ Anthropic)**

- การซ่อมแซมการจับคู่ผลลัพธ์เครื่องมือและผลลัพธ์เครื่องมือสังเคราะห์
- การตรวจสอบความถูกต้องของเทิร์น (รวมเทิร์นผู้ใช้ที่ต่อเนื่องกันเพื่อให้ตรงตามการสลับที่เข้มงวด)
- เทิร์น prefill ของ assistant ท้ายสุดจะถูกลบออกจากเพย์โหลด Anthropic Messages
  ขาออกเมื่อเปิดใช้ thinking รวมถึงเส้นทาง Cloudflare AI Gateway
- บล็อกการคิดที่ลายเซ็นการเล่นซ้ำหายไป ว่างเปล่า หรือเป็นช่องว่างล้วน จะถูกลบออก
  ก่อนแปลงสำหรับผู้ให้บริการ หากทำให้เทิร์น assistant ว่าง OpenClaw จะคง
  รูปทรงเทิร์นไว้ด้วยข้อความ omitted-reasoning ที่ไม่ว่างเปล่า
- เทิร์น assistant แบบ thinking-only ที่เก่ากว่าซึ่งต้องถูกลบ จะถูกแทนที่ด้วย
  ข้อความ omitted-reasoning ที่ไม่ว่างเปล่า เพื่อให้อะแดปเตอร์ผู้ให้บริการไม่ลบเทิร์น
  การเล่นซ้ำ

**Amazon Bedrock (Converse API)**

- เทิร์นข้อผิดพลาดสตรีมของ assistant ที่ว่างเปล่าจะถูกซ่อมแซมเป็นบล็อกข้อความ fallback ที่ไม่ว่างเปล่า
  ก่อนเล่นซ้ำ Bedrock Converse ปฏิเสธข้อความ assistant ที่มี `content: []` ดังนั้น
  เทิร์น assistant ที่บันทึกไว้ซึ่งมี `stopReason: "error"` และเนื้อหาว่างจะถูก
  ซ่อมแซมบนดิสก์ก่อนโหลดด้วย
- เทิร์นข้อผิดพลาดสตรีมของ assistant ที่มีเพียงบล็อกข้อความว่างจะถูกลบ
  ออกจากสำเนาการเล่นซ้ำในหน่วยความจำ แทนการเล่นซ้ำบล็อกว่างที่ไม่ถูกต้อง
- บล็อกการคิดของ Claude ที่ลายเซ็นการเล่นซ้ำหายไป ว่างเปล่า หรือเป็นช่องว่างล้วน จะถูก
  ลบออกก่อนเล่นซ้ำ Converse หากทำให้เทิร์น assistant ว่าง OpenClaw
  จะคงรูปทรงเทิร์นไว้ด้วยข้อความ omitted-reasoning ที่ไม่ว่างเปล่า
- เทิร์น assistant แบบ thinking-only ที่เก่ากว่าซึ่งต้องถูกลบ จะถูกแทนที่ด้วย
  ข้อความ omitted-reasoning ที่ไม่ว่างเปล่า เพื่อให้การเล่นซ้ำ Converse คงรูปทรงเทิร์นที่เข้มงวด
- การเล่นซ้ำจะกรองเทิร์น assistant แบบ delivery-mirror ของ OpenClaw และที่ Gateway แทรกเข้ามา
- การทำให้รูปภาพปลอดภัยใช้ตามกฎสากล

**Mistral (รวมถึงการตรวจจับตาม model-id)**

- การทำให้ id ของการเรียกเครื่องมือปลอดภัย: strict9 (ตัวอักษรและตัวเลข ความยาว 9)

**OpenRouter Gemini**

- การล้างลายเซ็นความคิด: ลบค่า `thought_signature` ที่ไม่ใช่ base64 (เก็บ base64 ไว้)

**OpenRouter Anthropic**

- เทิร์น prefill ของ assistant ท้ายสุดจะถูกลบออกจากเพย์โหลดโมเดล Anthropic
  ที่เข้ากันได้กับ OpenAI ของ OpenRouter ที่ตรวจสอบแล้ว เมื่อเปิดใช้ reasoning ซึ่งตรงกับ
  พฤติกรรมการเล่นซ้ำของ Anthropic โดยตรงและ Cloudflare Anthropic

**อย่างอื่นทั้งหมด**

- การทำให้รูปภาพปลอดภัยเท่านั้น

## พฤติกรรมในอดีต (ก่อน 2026.1.22)

ก่อนรุ่น 2026.1.22 OpenClaw ใช้การดูแลความสะอาดของทรานสคริปต์หลายชั้น:

- **transcript-sanitize extension** ทำงานในการสร้างบริบททุกครั้งและสามารถ:
  - ซ่อมแซมการจับคู่ tool use/result
  - ทำให้ id ของการเรียกเครื่องมือปลอดภัย (รวมถึงโหมดไม่เข้มงวดที่คง `_`/`-` ไว้)
- runner ยังทำการทำให้ปลอดภัยเฉพาะผู้ให้บริการด้วย ซึ่งทำงานซ้ำซ้อน
- มีการกลายพันธุ์เพิ่มเติมเกิดขึ้นนอกนโยบายผู้ให้บริการ รวมถึง:
  - การลบแท็ก `<final>` ออกจากข้อความ assistant ก่อนบันทึก
  - การลบเทิร์นข้อผิดพลาดของ assistant ที่ว่างเปล่า
  - การตัดเนื้อหา assistant หลังการเรียกเครื่องมือ

ความซับซ้อนนี้ทำให้เกิด regression ข้ามผู้ให้บริการ (โดยเฉพาะการจับคู่ `call_id|fc_id`
ของ `openai-responses`) การล้างใน 2026.1.22 ลบ extension ออก รวมศูนย์
ตรรกะไว้ใน runner และทำให้ OpenAI เป็นแบบ **ไม่แตะต้อง** นอกเหนือจากการทำให้รูปภาพปลอดภัย
