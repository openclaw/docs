เอาต์พุตของ Assistant สามารถมีชุดคำสั่งกำกับการส่ง/การแสดงผลขนาดเล็กได้:

- `MEDIA:` สำหรับการส่งไฟล์แนบ
- `[[audio_as_voice]]` สำหรับคำใบ้การนำเสนอเสียง
- `[[reply_to_current]]` / `[[reply_to:<id>]]` สำหรับเมทาดาทาการตอบกลับ
- `[embed ...]` สำหรับการแสดงผลแบบริชของ Control UI

ไฟล์แนบ `MEDIA:` ระยะไกลต้องเป็น URL `https:` สาธารณะเท่านั้น `http:` แบบธรรมดา,
loopback, link-local, ส่วนตัว และชื่อโฮสต์ภายในจะถูกละเว้นในฐานะคำสั่งกำกับไฟล์แนบ;
ตัวดึงข้อมูลสื่อฝั่งเซิร์ฟเวอร์ยังคงบังคับใช้การป้องกันเครือข่ายของตนเอง

ไวยากรณ์รูปภาพ Markdown ธรรมดาจะยังคงเป็นข้อความโดยค่าเริ่มต้น ช่องทางที่ตั้งใจ
แมปการตอบกลับรูปภาพ Markdown เป็นไฟล์แนบสื่อจะเลือกใช้ที่อะแดปเตอร์ขาออกของตน
Telegram ทำเช่นนี้เพื่อให้ `![alt](url)` ยังคงกลายเป็นการตอบกลับแบบสื่อได้

คำสั่งกำกับเหล่านี้แยกจากกัน `MEDIA:` และแท็กตอบกลับ/เสียงยังคงเป็นเมทาดาทาการส่ง; `[embed ...]` คือเส้นทางการแสดงผลแบบริชเฉพาะเว็บ
สื่อจากผลลัพธ์เครื่องมือที่เชื่อถือได้ใช้ตัวแยกวิเคราะห์ `MEDIA:` / `[[audio_as_voice]]` เดียวกันก่อนการส่ง ดังนั้นเอาต์พุตเครื่องมือแบบข้อความยังคงทำเครื่องหมายไฟล์แนบเสียงเป็นโน้ตเสียงได้

เมื่อเปิดใช้การสตรีมแบบบล็อก `MEDIA:` จะยังคงเป็นเมทาดาทาการส่งครั้งเดียวสำหรับหนึ่ง
รอบ หาก URL สื่อเดียวกันถูกส่งในบล็อกที่สตรีมและซ้ำอีกครั้งในเพย์โหลดสุดท้ายของ
assistant OpenClaw จะส่งไฟล์แนบหนึ่งครั้งและตัดรายการซ้ำออกจากเพย์โหลดสุดท้าย

## `[embed ...]`

`[embed ...]` คือไวยากรณ์การแสดงผลแบบริชแบบเดียวที่ agent ใช้ได้สำหรับ Control UI

ตัวอย่างแบบปิดในตัว:

```text
[embed ref="cv_123" title="Status" /]
```

กฎ:

- `[view ...]` ไม่ถูกต้องสำหรับเอาต์พุตใหม่อีกต่อไป
- ชอร์ตโค้ด Embed จะแสดงผลเฉพาะในพื้นผิวข้อความของ assistant เท่านั้น
- จะแสดงผลเฉพาะ embed ที่มี URL รองรับ ใช้ `ref="..."` หรือ `url="..."`
- ชอร์ตโค้ด embed แบบ inline HTML ในรูปแบบบล็อกจะไม่ถูกแสดงผล
- เว็บ UI จะตัดชอร์ตโค้ดออกจากข้อความที่มองเห็นได้และแสดงผล embed แบบอินไลน์
- `MEDIA:` ไม่ใช่นามแฝงของ embed และไม่ควรใช้สำหรับการแสดงผล embed แบบริช

## รูปแบบการแสดงผลที่จัดเก็บไว้

บล็อกเนื้อหา assistant ที่ทำให้เป็นมาตรฐาน/จัดเก็บไว้คือรายการ `canvas` แบบมีโครงสร้าง:

```json
{
  "type": "canvas",
  "preview": {
    "kind": "canvas",
    "surface": "assistant_message",
    "render": "url",
    "viewId": "cv_123",
    "url": "/__openclaw__/canvas/documents/cv_123/index.html",
    "title": "Status",
    "preferredHeight": 320
  }
}
```

บล็อกแบบริชที่จัดเก็บ/แสดงผลใช้รูปแบบ `canvas` นี้โดยตรง ไม่รู้จัก `present_view`
