OpenClaw เป็นเจ้าของรันไทม์เอเจนต์ในตัว โค้ดรันไทม์อยู่ภายใต้ `src/agents/` การรับส่งข้อมูลของโมเดล/ผู้ให้บริการอยู่ภายใต้ `src/llm/` และสัญญาสำหรับ Plugin เปิดให้ใช้งานผ่าน barrel ของ `openclaw/plugin-sdk/*`

## โครงสร้างรันไทม์

| พาธ                                | รับผิดชอบ                                                                                                                                                                                                                      |
| ----------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `src/agents/embedded-agent-runner/` | ลูปการพยายามในตัว (`run.ts`, `run/`) การเลือกโมเดลและการปรับผู้ให้บริการให้อยู่ในรูปแบบมาตรฐาน (`model*.ts`) พารามิเตอร์คำขอแยกตามผู้ให้บริการ (`extra-params.*`) Compaction และการเชื่อมต่อทรานสคริปต์กับเซสชัน                            |
| `src/agents/sessions/`              | การคงอยู่ของเซสชัน (`session-manager.ts`) การค้นหาทรัพยากร (`package-manager.ts`, `resource-loader.ts`) การโหลด `extensions` ภายในเซสชัน เทมเพลตพรอมต์ Skills ธีม และตัวเรนเดอร์เครื่องมือที่ใช้ TUI (`tools/`) |
| `packages/agent-core/`              | แกนเอเจนต์ที่นำกลับมาใช้ซ้ำได้ (`@openclaw/agent-core`): ลูปเอเจนต์ ชนิดข้อมูลของฮาร์เนส ข้อความ ตัวช่วย Compaction เทมเพลตพรอมต์ Skills และสัญญาการจัดเก็บเซสชัน                                                           |
| `src/agents/runtime/`               | ฟาซาดของ OpenClaw ที่เชื่อม `@openclaw/agent-core` เข้ากับรันไทม์ LLM ของ SDK สำหรับ Plugin และส่งออกทั้งรันไทม์ดังกล่าวกับยูทิลิตีพร็อกซีภายในอีกครั้ง                                                                                             |
| `src/agents/agent-tools*.ts`        | นิยามเครื่องมือที่ OpenClaw เป็นเจ้าของ สคีมาพารามิเตอร์ นโยบายเครื่องมือ อะแดปเตอร์ก่อน/หลังการเรียกเครื่องมือ และเครื่องมือแก้ไขสำหรับโฮสต์/แซนด์บ็อกซ์                                                                                            |
| `src/agents/agent-hooks/`           | ฮุกของรันไทม์ในตัว: มาตรการป้องกัน Compaction คำสั่ง Compaction และการตัดแต่งบริบท                                                                                                                                   |
| `src/agents/harness/`               | รีจิสทรีฮาร์เนส นโยบายการเลือก และวงจรชีวิตสำหรับฮาร์เนสในตัวและฮาร์เนสที่ลงทะเบียนโดย Plugin                                                                                                                       |
| `src/llm/`                          | รีจิสทรีโมเดล/ผู้ให้บริการ ตัวช่วยการรับส่งข้อมูล และการใช้งานสตรีมเฉพาะผู้ให้บริการ (`src/llm/providers/`)                                                                                                          |

## ขอบเขต

แกนหลักเรียกรันไทม์ในตัวผ่านโมดูล OpenClaw และ barrel ของ SDK โดยไม่มีแพ็กเกจเฟรมเวิร์กเอเจนต์ภายนอกหลงเหลืออยู่ Plugin ใช้จุดเข้าใช้งาน `openclaw/plugin-sdk/*` ที่มีเอกสารกำกับ และไม่นำเข้าส่วนภายในของ `src/**`

`@earendil-works/pi-tui` ยังคงเป็นการขึ้นต่อกันของบุคคลที่สาม ซึ่งเป็นชุดเครื่องมือคอมโพเนนต์เทอร์มินัลที่ TUI ภายในและตัวเรนเดอร์เครื่องมือของเซสชันใช้ การนำมาไว้ภายในจะเป็นงานรวมซอร์สจากภายนอกอีกส่วนหนึ่งโดยเฉพาะ

## ไฟล์ Manifest

แพ็กเกจทรัพยากรประกาศทรัพยากรของ OpenClaw ในข้อมูลเมตา `package.json` รายการเป็นพาธไฟล์หรือรูปแบบ glob ที่สัมพันธ์กับรากของแพ็กเกจ:

```json
{
  "openclaw": {
    "extensions": ["extensions/index.ts"],
    "skills": ["skills/*.md"],
    "prompts": ["prompts/*.md"],
    "themes": ["themes/*.json"]
  }
}
```

ประเภททรัพยากรที่ไม่ได้ระบุใน manifest จะย้อนกลับไปใช้การค้นหาไดเรกทอรีมาตรฐาน `extensions/`, `skills/`, `prompts/` และ `themes/`

## การเลือกรันไทม์

- รหัสรันไทม์ในตัวคือ `openclaw` นามแฝงเดิม `pi` จะถูกปรับให้อยู่ในรูปแบบ `openclaw`; ส่วน `codex-app-server` จะถูกปรับให้อยู่ในรูปแบบ `codex`
- ฮาร์เนสของ Plugin ลงทะเบียนรหัสรันไทม์เพิ่มเติม (ตัวอย่างเช่น `codex`)
- นโยบายรันไทม์คือการกำหนดค่า `agentRuntime.id` ที่มีขอบเขตตามโมเดล/ผู้ให้บริการ (รายการโมเดลมีลำดับความสำคัญเหนือรายการผู้ให้บริการ) หากไม่ได้ตั้งค่าหรือเป็น `default` ระบบจะตีความเป็น `auto`
- `auto` จะเลือกฮาร์เนสของ Plugin ที่ลงทะเบียนไว้และรองรับเส้นทางผู้ให้บริการที่มีผล มิฉะนั้นจะเลือกรันไทม์ OpenClaw ในตัว การมีเพียงคำนำหน้าของผู้ให้บริการหรือโมเดลจะไม่เลือกฮาร์เนส
- OpenAI อาจเลือก `codex` โดยปริยายได้เฉพาะเส้นทาง HTTPS อย่างเป็นทางการของ Platform Responses หรือ ChatGPT Responses ที่ตรงกันทุกประการและไม่มีการเขียนทับคำขอไว้ อะแดปเตอร์ Completions จุดเชื่อมต่อที่กำหนดเอง และเส้นทางที่มีพฤติกรรมคำขอที่เขียนไว้จะยังคงใช้ `openclaw`; จุดเชื่อมต่อ HTTP แบบข้อความธรรมดาอย่างเป็นทางการจะถูกปฏิเสธ ดู [รันไทม์เอเจนต์โดยปริยายของ OpenAI](/th/providers/openai#implicit-agent-runtime)
