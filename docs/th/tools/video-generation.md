เอเจนต์ OpenClaw สร้างวิดีโอจากพรอมต์ข้อความ รูปภาพอ้างอิง หรือ
วิดีโอที่มีอยู่ผ่าน `video_generate` รองรับแบ็กเอนด์ของผู้ให้บริการสิบหกราย
โดยเอเจนต์จะเลือกแบ็กเอนด์ที่เหมาะสมโดยอัตโนมัติตามการกำหนดค่าและ
คีย์ API ที่พร้อมใช้งาน

<Note>
`video_generate` จะปรากฏเฉพาะเมื่อมีผู้ให้บริการสร้างวิดีโอพร้อมใช้งานอย่างน้อยหนึ่งราย
หากเครื่องมือนี้ไม่อยู่ในชุดเครื่องมือของเอเจนต์ ให้ตั้งค่าคีย์ API ของผู้ให้บริการหรือ
กำหนดค่า `agents.defaults.videoGenerationModel`
</Note>

`video_generate` มีโหมดรันไทม์สามโหมด ซึ่งกำหนดจากอินพุตอ้างอิง
ในการเรียกใช้:

- `generate` - ไม่มีสื่ออ้างอิง (ข้อความเป็นวิดีโอ)
- `imageToVideo` - รูปภาพอ้างอิงอย่างน้อยหนึ่งรูป
- `videoToVideo` - วิดีโออ้างอิงอย่างน้อยหนึ่งรายการ

ผู้ให้บริการอาจรองรับโหมดเหล่านี้เพียงบางส่วน เครื่องมือจะตรวจสอบ
โหมดที่ใช้งานอยู่ก่อนส่งคำขอ และรายงานโหมดที่รองรับใน `action=list`

## เริ่มต้นอย่างรวดเร็ว

<Steps>
  <Step title="กำหนดค่าการยืนยันตัวตน">
    ตั้งค่าคีย์ API สำหรับผู้ให้บริการที่รองรับรายใดก็ได้:

    ```bash
    export GEMINI_API_KEY="your-key"
    ```

  </Step>
  <Step title="เลือกโมเดลเริ่มต้น (ไม่บังคับ)">
    ```bash
    openclaw config set agents.defaults.videoGenerationModel.primary "google/veo-3.1-fast-generate-preview"
    ```
  </Step>
  <Step title="สั่งเอเจนต์">
    > สร้างวิดีโอแนวภาพยนตร์ความยาว 5 วินาที เป็นภาพกุ้งล็อบสเตอร์แสนเป็นมิตรกำลังโต้คลื่นยามพระอาทิตย์ตก

    เอเจนต์จะเรียก `video_generate` โดยอัตโนมัติ ไม่จำเป็นต้องเพิ่มเครื่องมือ
    ลงในรายการอนุญาต

  </Step>
</Steps>

## การสร้างแบบอะซิงโครนัสทำงานอย่างไร

การสร้างวิดีโอเป็นกระบวนการแบบอะซิงโครนัส:

1. OpenClaw ส่งคำขอไปยังผู้ให้บริการและส่งคืนรหัสงานทันที
2. ผู้ให้บริการประมวลผลงานในเบื้องหลัง (โดยทั่วไปใช้เวลา 30 วินาทีถึงหลายนาที ขึ้นอยู่กับผู้ให้บริการและความละเอียด ส่วนผู้ให้บริการที่ใช้คิวและทำงานช้าอาจใช้เวลาจนถึงค่าหมดเวลาที่กำหนดไว้)
3. เมื่อวิดีโอพร้อม OpenClaw จะปลุกเซสชันเดิมด้วยเหตุการณ์แจ้งการเสร็จสิ้นภายใน
4. เอเจนต์จะรายงานผ่านโหมดการตอบกลับที่มองเห็นได้ตามปกติของเซสชัน:
   ตอบกลับสุดท้ายโดยอัตโนมัติ หรือใช้ `message(action="send")` เมื่อเซสชันกำหนดให้
   ใช้เครื่องมือส่งข้อความ หากเซสชันของผู้ร้องขอไม่ได้ใช้งาน หรือการปลุกล้มเหลวและ
   สื่อที่สร้างยังไม่อยู่ในการตอบกลับเมื่อเสร็จสิ้น OpenClaw จะส่งข้อมูลสำรองโดยตรง
   ที่ทำซ้ำได้อย่างปลอดภัยพร้อมสื่อดังกล่าว

ระหว่างที่งานกำลังดำเนินอยู่ การเรียก `video_generate` ซ้ำใน
เซสชันเดียวกันจะส่งคืนสถานะงานปัจจุบันแทนการเริ่มสร้างงานใหม่
ใช้ `action: "status"` เพื่อตรวจสอบโดยไม่กระตุ้นให้เกิดการสร้างใหม่
หรือใช้ `openclaw tasks list` / `openclaw tasks show <lookup>` จาก
CLI (ดู [งานเบื้องหลัง](/th/automation/tasks))

นอกการทำงานของเอเจนต์ที่มีเซสชันรองรับ (ตัวอย่างเช่น การเรียกเครื่องมือโดยตรง)
เครื่องมือจะเปลี่ยนไปใช้การสร้างแบบอินไลน์และส่งคืนพาธของสื่อสุดท้าย
ภายในการทำงานรอบเดียวกัน

ไฟล์วิดีโอที่สร้างจะถูกบันทึกไว้ในพื้นที่จัดเก็บสื่อที่ OpenClaw จัดการ เมื่อ
ผู้ให้บริการส่งคืนข้อมูลแบบไบต์ ขีดจำกัดเริ่มต้นคือ 16MB (ขีดจำกัดสื่อวิดีโอ
ที่ใช้ร่วมกัน) โดย `agents.defaults.mediaMaxMb` สามารถเพิ่มขีดจำกัดสำหรับการเรนเดอร์ขนาดใหญ่ได้ เมื่อ
ผู้ให้บริการส่งคืน URL เอาต์พุตที่โฮสต์ไว้ด้วย OpenClaw จะส่ง URL นั้นแทน
การทำให้งานล้มเหลว หากการบันทึกในเครื่องปฏิเสธไฟล์ที่มีขนาดใหญ่เกินไป

### วงจรชีวิตของงาน

| สถานะ       | ความหมาย                                                                                                |
| ----------- | ------------------------------------------------------------------------------------------------------ |
| `queued`    | สร้างงานแล้วและกำลังรอให้ผู้ให้บริการยอมรับงาน                                                        |
| `running`   | ผู้ให้บริการกำลังประมวลผล (โดยทั่วไปใช้เวลา 30 วินาทีถึงหลายนาที ขึ้นอยู่กับผู้ให้บริการและความละเอียด) |
| `succeeded` | วิดีโอพร้อมแล้ว เอเจนต์จะตื่นและโพสต์วิดีโอลงในการสนทนา                                               |
| `failed`    | เกิดข้อผิดพลาดจากผู้ให้บริการหรือหมดเวลา เอเจนต์จะตื่นพร้อมรายละเอียดข้อผิดพลาด                        |

ตรวจสอบสถานะจาก CLI:

```bash
openclaw tasks list
openclaw tasks show <lookup>
openclaw tasks cancel <lookup>
```

คำใบ้บทบาทจะถูกส่งต่อไปยังผู้ให้บริการตามที่ระบุไว้โดยไม่มีการเปลี่ยนแปลง ค่ามาตรฐานมาจาก
ยูเนียน `VideoGenerationAssetRole` แต่ผู้ให้บริการอาจยอมรับสตริงบทบาท
เพิ่มเติม อาร์เรย์ `*Roles` ต้องมีจำนวนรายการไม่มากกว่า
รายการอ้างอิงที่สอดคล้องกัน ข้อผิดพลาดคลาดเคลื่อนหนึ่งตำแหน่งจะล้มเหลวพร้อมข้อความแสดงข้อผิดพลาดที่ชัดเจน
ใช้สตริงว่างเพื่อเว้นช่องโดยไม่กำหนดค่า สำหรับ xAI ให้ตั้งบทบาทของรูปภาพทุกภาพเป็น
`reference_image` เพื่อใช้โหมดการสร้าง `reference_images` ของผู้ให้บริการนั้น ให้ละเว้น
บทบาทหรือใช้ `first_frame` สำหรับการแปลงรูปภาพเดียวเป็นวิดีโอ
</Note>

  ผู้ให้บริการที่ประกาศสคีมาแบบระบุชนิดจะตรวจสอบความถูกต้องของคีย์และชนิดข้อมูล คีย์ที่ไม่รู้จัก
  หรือชนิดที่ไม่ตรงกันจะทำให้ข้ามตัวเลือกนั้นระหว่างการใช้ผู้ให้บริการสำรอง ผู้ให้บริการที่ไม่มี
  สคีมาที่ประกาศไว้จะได้รับตัวเลือกตามที่ระบุโดยไม่มีการเปลี่ยนแปลง เรียกใช้ `video_generate action=list`
  เพื่อดูว่าผู้ให้บริการแต่ละรายยอมรับค่าใดบ้าง
</ParamField>

<Note>
ผู้ให้บริการแต่ละรายไม่ได้รองรับพารามิเตอร์ทั้งหมด OpenClaw จะปรับระยะเวลาให้เป็น
ค่าที่ใกล้ที่สุดซึ่งผู้ให้บริการรองรับ และแมปคำใบ้เชิงเรขาคณิตที่แปลงแล้วใหม่
เช่น จากขนาดเป็นอัตราส่วนภาพ เมื่อผู้ให้บริการสำรองมี
พื้นผิวการควบคุมที่ต่างออกไป ค่าที่ระบุแทนซึ่งไม่รองรับจริง ๆ จะถูกเพิกเฉยโดยพยายาม
อย่างดีที่สุด และรายงานเป็นคำเตือนในผลลัพธ์ของเครื่องมือ ขีดจำกัดความสามารถแบบตายตัว
(เช่น มีอินพุตอ้างอิงมากเกินไป) จะทำให้ล้มเหลวก่อนส่งคำขอ ผลลัพธ์ของเครื่องมือ
จะรายงานการตั้งค่าที่ใช้จริง ส่วน `details.normalization` จะบันทึก
การแปลงจากค่าที่ขอเป็นค่าที่ใช้จริง
</Note>

อินพุตอ้างอิงจะเลือกโหมดรันไทม์ดังนี้:

- ไม่มีสื่ออ้างอิง -> `generate`
- มีการอ้างอิงรูปภาพใด ๆ -> `imageToVideo`
- มีการอ้างอิงวิดีโอใด ๆ -> `videoToVideo`
- อินพุตเสียงอ้างอิง **ไม่** เปลี่ยนโหมดที่กำหนดแล้ว แต่จะใช้
  เพิ่มเติมจากโหมดที่การอ้างอิงรูปภาพ/วิดีโอเลือกไว้ และทำงานได้เฉพาะ
  กับผู้ให้บริการที่ประกาศ `maxInputAudios`

การผสมการอ้างอิงรูปภาพและวิดีโอไม่ใช่พื้นผิวความสามารถร่วมที่เสถียร
ควรใช้การอ้างอิงเพียงประเภทเดียวต่อคำขอ

#### การใช้ผู้ให้บริการสำรองและตัวเลือกแบบระบุชนิด

การตรวจสอบความสามารถบางอย่างเกิดขึ้นที่ชั้นการใช้ผู้ให้บริการสำรองแทนที่จะเป็นขอบเขตของเครื่องมือ
ดังนั้นคำขอที่เกินขีดจำกัดของผู้ให้บริการหลักยังสามารถ
ทำงานกับผู้ให้บริการสำรองที่มีความสามารถเพียงพอได้:

- ตัวเลือกที่กำลังใช้งานซึ่งไม่ได้ประกาศ `maxInputAudios` (หรือประกาศเป็น `0`) จะถูกข้ามเมื่อ
  คำขอมีเสียงอ้างอิง จากนั้นจะลองตัวเลือกถัดไป การตรวจสอบแบบเดียวกันนี้
  ใช้กับจำนวนการอ้างอิงรูปภาพและวิดีโอเทียบกับ
  `maxInputImages`/`maxInputVideos`
- หาก `maxDurationSeconds` ของตัวเลือกที่กำลังใช้งานต่ำกว่า `durationSeconds` ที่ขอ
  และไม่มีรายการ `supportedDurationSeconds` ที่ประกาศไว้ -> ข้าม
- หากคำขอมี `providerOptions` และตัวเลือกที่กำลังใช้งานประกาศ
  สคีมา `providerOptions` แบบระบุชนิดไว้อย่างชัดเจน -> ข้ามหากคีย์ที่ระบุ
  ไม่อยู่ในสคีมาหรือชนิดค่าไม่ตรงกัน ผู้ให้บริการที่ไม่มี
  สคีมาที่ประกาศไว้จะได้รับตัวเลือกตามที่ระบุโดยไม่มีการเปลี่ยนแปลง (การส่งผ่านที่
  เข้ากันได้กับรุ่นก่อนหน้า) ผู้ให้บริการสามารถปฏิเสธตัวเลือกทั้งหมดของผู้ให้บริการได้โดย
  ประกาศสคีมาว่าง (`capabilities.providerOptions: {}`) ซึ่ง
  ทำให้ถูกข้ามเช่นเดียวกับกรณีชนิดไม่ตรงกัน

เหตุผลในการข้ามรายการแรกของคำขอจะถูกบันทึกที่ระดับ `warn` เพื่อให้ผู้ดูแลระบบเห็นว่า
ผู้ให้บริการหลักของตนถูกข้าม ส่วนการข้ามรายการถัด ๆ ไปจะถูกบันทึกที่ระดับ `debug` เพื่อ
ไม่ให้สายโซ่ผู้ให้บริการสำรองที่ยาวสร้างข้อมูลรบกวนมากเกินไป หากตัวเลือกทั้งหมดถูกข้าม
ข้อผิดพลาดแบบรวมจะมีเหตุผลในการข้ามของแต่ละตัวเลือก

## การดำเนินการ

| การดำเนินการ | สิ่งที่ทำ                                                                                             |
| ---------- | -------------------------------------------------------------------------------------------------------- |
| `generate` | ค่าปริยาย สร้างวิดีโอจากพรอมต์ที่ระบุและอินพุตอ้างอิงที่เป็นตัวเลือก                             |
| `status`   | ตรวจสอบสถานะงานวิดีโอที่กำลังดำเนินการของเซสชันปัจจุบันโดยไม่เริ่มการสร้างรายการใหม่ |
| `list`     | แสดงผู้ให้บริการ โมเดล และความสามารถที่พร้อมใช้งาน                                                |

## การเลือกโมเดล

OpenClaw จะกำหนดโมเดลตามลำดับต่อไปนี้:

1. **พารามิเตอร์เครื่องมือ `model`** - หากเอเจนต์ระบุไว้ในการเรียก
2. **`videoGenerationModel.primary`** จากการกำหนดค่า
3. **`videoGenerationModel.fallbacks`** ตามลำดับ
4. **การตรวจหาอัตโนมัติ** - ผู้ให้บริการที่มีข้อมูลรับรองความถูกต้อง โดยเริ่มจาก
   ผู้ให้บริการปริยายปัจจุบัน จากนั้นจึงเป็นผู้ให้บริการที่เหลือตามลำดับ
   ตัวอักษร

หากผู้ให้บริการล้มเหลว ระบบจะลองตัวเลือกถัดไปโดยอัตโนมัติ หาก
ตัวเลือกทั้งหมดล้มเหลว ข้อผิดพลาดจะมีรายละเอียดจากการลองแต่ละครั้ง

ตั้งค่า `agents.defaults.mediaGenerationAutoProviderFallback: false` เพื่อใช้
เฉพาะรายการ `model`, `primary` และ `fallbacks` ที่ระบุไว้อย่างชัดเจน

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "google/veo-3.1-fast-generate-preview",
        fallbacks: ["runway/gen4.5", "qwen/wan2.6-t2v"],
        timeoutMs: 180000, // optional per-tool provider request timeout override
      },
    },
  },
}
```

## โหมดความสามารถของผู้ให้บริการ

สัญญาการสร้างวิดีโอร่วมรองรับความสามารถเฉพาะโหมด
แทนที่จะรองรับเพียงขีดจำกัดรวมแบบแบน การติดตั้งใช้งานผู้ให้บริการใหม่
ควรเลือกใช้บล็อกโหมดที่ระบุชัดเจน:

```typescript
capabilities: {
  generate: {
    maxVideos: 1,
    maxDurationSeconds: 10,
    supportsResolution: true,
  },
  imageToVideo: {
    enabled: true,
    maxVideos: 1,
    maxInputImages: 1,
    maxInputImagesByModel: { "provider/reference-to-video": 9 },
    maxDurationSeconds: 5,
  },
  videoToVideo: {
    enabled: true,
    maxVideos: 1,
    maxInputVideos: 1,
    maxDurationSeconds: 5,
  },
}
```

ฟิลด์รวมแบบแบน เช่น `maxInputImages` และ `maxInputVideos`
**ไม่**เพียงพอสำหรับระบุว่ารองรับโหมดแปลง ผู้ให้บริการควร
ประกาศ `generate`, `imageToVideo` และ `videoToVideo` อย่างชัดเจน เพื่อให้การทดสอบ
แบบสด การทดสอบสัญญา และเครื่องมือ `video_generate` ที่ใช้ร่วมกันสามารถตรวจสอบ
การรองรับโหมดได้อย่างแน่นอน

เมื่อโมเดลหนึ่งของผู้ให้บริการรองรับอินพุตอ้างอิงได้กว้างกว่า
โมเดลอื่น ให้ใช้ `maxInputImagesByModel`, `maxInputVideosByModel` หรือ
`maxInputAudiosByModel` แทนการเพิ่มขีดจำกัดสำหรับทั้งโหมด

## การทดสอบแบบสด

เปิดใช้ความครอบคลุมการทดสอบแบบสดสำหรับผู้ให้บริการร่วมที่รวมมาให้ได้ตามต้องการ:

```bash
OPENCLAW_LIVE_TEST=1 pnpm test:live -- extensions/video-generation-providers.live.test.ts
```

ตัวครอบของคลัง:

```bash
pnpm test:live:media video
```

โดยค่าเริ่มต้น ไฟล์ทดสอบแบบสดนี้ใช้ตัวแปรสภาพแวดล้อมของผู้ให้บริการที่ส่งออกไว้แล้วก่อน
โปรไฟล์การยืนยันตัวตนที่จัดเก็บไว้ และเรียกใช้การทดสอบเบื้องต้นที่ปลอดภัยสำหรับการเผยแพร่โดยค่าเริ่มต้น:

- `generate` สำหรับผู้ให้บริการทุกรายที่ไม่ใช่ FAL ในการทดสอบรอบนี้
- พรอมต์กุ้งมังกรความยาวหนึ่งวินาที
- ขีดจำกัดเวลาการดำเนินการต่อผู้ให้บริการจาก
  `OPENCLAW_LIVE_VIDEO_GENERATION_TIMEOUT_MS` (ค่าเริ่มต้นคือ `180000`)

FAL ต้องเปิดใช้เอง เนื่องจากเวลาแฝงของคิวฝั่งผู้ให้บริการอาจกินเวลาส่วนใหญ่ของ
กระบวนการเผยแพร่:

```bash
pnpm test:live:media video --video-providers fal
```

ตั้งค่า `OPENCLAW_LIVE_VIDEO_GENERATION_FULL_MODES=1` เพื่อเรียกใช้
โหมดแปลงที่ประกาศไว้เพิ่มเติม ซึ่งการทดสอบร่วมสามารถทดสอบได้อย่างปลอดภัยด้วยสื่อในเครื่อง:

- `imageToVideo` เมื่อ `capabilities.imageToVideo.enabled`
- `videoToVideo` เมื่อ `capabilities.videoToVideo.enabled` และ
  ผู้ให้บริการ/โมเดลยอมรับอินพุตวิดีโอในเครื่องที่มีบัฟเฟอร์รองรับในการทดสอบ
  ร่วม

ปัจจุบันช่องทางทดสอบแบบสด `videoToVideo` ที่ใช้ร่วมกันครอบคลุมเฉพาะ `runway`
เมื่อเลือก `runway/gen4_aleph`

## การกำหนดค่า

ตั้งค่าโมเดลเริ่มต้นสำหรับการสร้างวิดีโอในการกำหนดค่า OpenClaw ของคุณ:

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "qwen/wan2.6-t2v",
        fallbacks: ["qwen/wan2.6-r2v-flash"],
      },
    },
  },
}
```

หรือผ่าน CLI:

```bash
openclaw config set agents.defaults.videoGenerationModel.primary "qwen/wan2.6-t2v"
```
