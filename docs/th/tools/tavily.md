[Tavily](https://tavily.com) คือ API การค้นหาที่ออกแบบมาสำหรับแอปพลิเคชัน AI โดย OpenClaw เปิดให้ใช้งานได้สองวิธี:

- เป็นผู้ให้บริการ `web_search` สำหรับเครื่องมือค้นหาทั่วไป
- เป็นเครื่องมือ Plugin โดยตรง ได้แก่ `tavily_search` และ `tavily_extract`

Tavily ส่งคืนผลลัพธ์แบบมีโครงสร้างที่ปรับให้เหมาะกับการใช้งานโดย LLM พร้อมความลึกของการค้นหาที่กำหนดค่าได้ การกรองตามหัวข้อ ตัวกรองโดเมน สรุปคำตอบที่สร้างโดย AI และการแยกเนื้อหาจาก URL (รวมถึงหน้าที่เรนเดอร์ด้วย JavaScript)

| คุณสมบัติ      | ค่า                                                                                                        |
| --------------- | ---------------------------------------------------------------------------------------------------------- |
| รหัส Plugin     | `tavily`                                                                                                   |
| แพ็กเกจ         | `@openclaw/tavily-plugin`                                                                                  |
| การยืนยันตัวตน | ตัวแปรสภาพแวดล้อม `TAVILY_API_KEY` หรือการกำหนดค่า `apiKey`                                               |
| URL ฐาน         | `https://api.tavily.com` (ค่าเริ่มต้น); ใช้ตัวแปรสภาพแวดล้อม `TAVILY_BASE_URL` หรือการกำหนดค่า `baseUrl` เพื่อแทนที่ |
| การหมดเวลา      | ค้นหา 30 วินาที แยกเนื้อหา 60 วินาที (ค่าเริ่มต้น)                                                       |
| เครื่องมือ      | `tavily_search`, `tavily_extract`                                                                          |

## เริ่มต้นใช้งาน

<Steps>
  <Step title="ติดตั้ง Plugin">
    ```bash
    openclaw plugins install @openclaw/tavily-plugin
    ```
  </Step>
  <Step title="รับคีย์ API">
    สร้างบัญชี Tavily ที่ [tavily.com](https://tavily.com) จากนั้นสร้างคีย์ API ในแดชบอร์ด
  </Step>
  <Step title="กำหนดค่า Plugin และผู้ให้บริการ">
    ```json5
    {
      plugins: {
        entries: {
          tavily: {
            enabled: true,
            config: {
              webSearch: {
                apiKey: "tvly-...", // ไม่บังคับหากตั้งค่า TAVILY_API_KEY แล้ว
                baseUrl: "https://api.tavily.com",
              },
            },
          },
        },
      },
      tools: {
        web: {
          search: {
            provider: "tavily",
          },
        },
      },
    }
    ```
  </Step>
  <Step title="ตรวจสอบว่าการค้นหาทำงาน">
    เรียกใช้ `web_search` จากเอเจนต์ใดก็ได้ หรือเรียก `tavily_search` โดยตรง
  </Step>
</Steps>

<Tip>
การเลือก Tavily ระหว่างการเริ่มต้นใช้งานหรือใน `openclaw configure --section web` จะติดตั้งและเปิดใช้งาน Plugin Tavily อย่างเป็นทางการเมื่อจำเป็น
</Tip>

## ข้อมูลอ้างอิงเครื่องมือ

### `tavily_search`

ใช้เครื่องมือนี้เมื่อต้องการควบคุมการค้นหาเฉพาะของ Tavily แทน `web_search` แบบทั่วไป

| พารามิเตอร์      | ชนิดข้อมูล        | ข้อจำกัด / ค่าเริ่มต้น                   | คำอธิบาย                                      |
| ----------------- | ----------------- | ---------------------------------------- | --------------------------------------------- |
| `query`           | สตริง             | จำเป็น                                   | สตริงคำค้นหา                                  |
| `search_depth`    | ค่าแจกแจง         | `basic` (ค่าเริ่มต้น), `advanced`        | `advanced` ช้ากว่าแต่มีความเกี่ยวข้องสูงกว่า |
| `topic`           | ค่าแจกแจง         | `general` (ค่าเริ่มต้น), `news`, `finance` | กรองตามกลุ่มหัวข้อ                            |
| `max_results`     | จำนวนเต็ม         | 1-20, ค่าเริ่มต้น `5`                    | จำนวนผลลัพธ์                                  |
| `include_answer`  | ค่าบูลีน          | ค่าเริ่มต้น `false`                      | รวมสรุปคำตอบที่ AI ของ Tavily สร้างขึ้น      |
| `time_range`      | ค่าแจกแจง         | `day`, `week`, `month`, `year`           | กรองผลลัพธ์ตามความใหม่                        |
| `include_domains` | อาร์เรย์ของสตริง | (ไม่มี)                                  | รวมเฉพาะผลลัพธ์จากโดเมนเหล่านี้              |
| `exclude_domains` | อาร์เรย์ของสตริง | (ไม่มี)                                  | ไม่รวมผลลัพธ์จากโดเมนเหล่านี้                |

ข้อแลกเปลี่ยนของความลึกในการค้นหา:

| ความลึก    | ความเร็ว | ความเกี่ยวข้อง | เหมาะที่สุดสำหรับ                              |
| ---------- | -------- | --------------- | ---------------------------------------------- |
| `basic`    | เร็วกว่า | สูง             | คำค้นหาสำหรับการใช้งานทั่วไป (ค่าเริ่มต้น)    |
| `advanced` | ช้ากว่า  | สูงที่สุด       | การวิจัยที่ต้องการความแม่นยำและการค้นหาข้อเท็จจริง |

### `tavily_extract`

ใช้เครื่องมือนี้เพื่อแยกเนื้อหาที่สะอาดจาก URL หนึ่งรายการขึ้นไป รองรับหน้าที่เรนเดอร์ด้วย JavaScript และการแบ่งเนื้อหาเป็นส่วนตามคำค้นหาเพื่อการแยกข้อมูลแบบเจาะจง

| พารามิเตอร์        | ชนิดข้อมูล        | ข้อจำกัด / ค่าเริ่มต้น          | คำอธิบาย                                                       |
| ------------------- | ----------------- | ------------------------------- | -------------------------------------------------------------- |
| `urls`              | อาร์เรย์ของสตริง | จำเป็น, 1-20                    | URL ที่ต้องการแยกเนื้อหา                                      |
| `query`             | สตริง             | (ไม่บังคับ)                     | จัดอันดับส่วนเนื้อหาที่แยกแล้วใหม่ตามความเกี่ยวข้องกับคำค้นหานี้ |
| `extract_depth`     | ค่าแจกแจง         | `basic` (ค่าเริ่มต้น), `advanced` | ใช้ `advanced` สำหรับหน้าที่ใช้ JS มาก, SPA หรือตารางแบบไดนามิก |
| `chunks_per_source` | จำนวนเต็ม         | 1-5; **ต้องมี `query`**         | จำนวนส่วนเนื้อหาที่ส่งคืนต่อ URL จะเกิดข้อผิดพลาดหากตั้งค่าโดยไม่มี `query` |
| `include_images`    | ค่าบูลีน          | ค่าเริ่มต้น `false`             | รวม URL ของรูปภาพไว้ในผลลัพธ์                                 |

ข้อแลกเปลี่ยนของความลึกในการแยกเนื้อหา:

| ความลึก    | ควรใช้เมื่อ                                       |
| ---------- | ------------------------------------------------ |
| `basic`    | หน้าเว็บแบบเรียบง่าย ให้ลองใช้ตัวเลือกนี้ก่อน    |
| `advanced` | SPA ที่เรนเดอร์ด้วย JS เนื้อหาแบบไดนามิก และตาราง |

<Tip>
แบ่งรายการ URL ขนาดใหญ่ออกเป็นการเรียก `tavily_extract` หลายครั้ง (สูงสุด 20 รายการต่อคำขอ) ใช้ `query` ร่วมกับ `chunks_per_source` เพื่อรับเฉพาะเนื้อหาที่เกี่ยวข้องแทนหน้าเว็บทั้งหน้า
</Tip>

## การเลือกเครื่องมือที่เหมาะสม

| ความต้องการ                                  | เครื่องมือ          |
| -------------------------------------------- | ------------------- |
| ค้นหาเว็บอย่างรวดเร็วโดยไม่มีตัวเลือกพิเศษ   | `web_search`        |
| ค้นหาพร้อมกำหนดความลึก หัวข้อ และคำตอบจาก AI | `tavily_search`     |
| แยกเนื้อหาจาก URL ที่ระบุ                    | `tavily_extract`    |

<Note>
เครื่องมือ `web_search` แบบทั่วไปที่ใช้ Tavily เป็นผู้ให้บริการรองรับ `query` และ `count` (สูงสุด 20 ผลลัพธ์) สำหรับการควบคุมเฉพาะของ Tavily (`search_depth`, `topic`, `include_answer`, ตัวกรองโดเมน และช่วงเวลา) ให้ใช้ `tavily_search` แทน
</Note>

## การกำหนดค่าขั้นสูง

<AccordionGroup>
  <Accordion title="ลำดับการค้นหาคีย์ API">
    ไคลเอนต์ Tavily จะค้นหาคีย์ API ตามลำดับต่อไปนี้:

    1. `plugins.entries.tavily.config.webSearch.apiKey` (แก้ไขค่าผ่าน SecretRefs)
    2. `TAVILY_API_KEY` จากสภาพแวดล้อมของ Gateway

    ทั้ง `tavily_search` และ `tavily_extract` จะแจ้งข้อผิดพลาดในการตั้งค่าหากไม่มีทั้งสองรายการ

  </Accordion>

  <Accordion title="URL ฐานแบบกำหนดเอง">
    แทนที่ `plugins.entries.tavily.config.webSearch.baseUrl` หรือตั้งค่า `TAVILY_BASE_URL` หากคุณเข้าถึง Tavily ผ่านพร็อกซี การกำหนดค่ามีลำดับความสำคัญเหนือกว่าตัวแปรสภาพแวดล้อม ค่าเริ่มต้นคือ `https://api.tavily.com`
  </Accordion>

  <Accordion title="`chunks_per_source` ต้องมี `query`">
    `tavily_extract` จะปฏิเสธการเรียกที่ส่ง `chunks_per_source` โดยไม่มี `query` Tavily จัดอันดับส่วนเนื้อหาตามความเกี่ยวข้องกับคำค้นหา ดังนั้นพารามิเตอร์นี้จึงไม่มีความหมายหากไม่มีคำค้นหา
  </Accordion>
</AccordionGroup>
