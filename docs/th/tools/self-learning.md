การเรียนรู้ด้วยตนเองช่วยให้ OpenClaw เปลี่ยนหลักฐานที่เป็นประโยชน์จากการสนทนาเป็นข้อเสนอ
[Skill Workshop](/th/tools/skill-workshop) ที่รอดำเนินการได้ โดยจะไม่ฝึกน้ำหนักโมเดล
แก้ไขสกิลที่ใช้งานอยู่ หรือเปลี่ยนพฤติกรรมของเอเจนต์โดยไม่แจ้งให้ทราบ ขั้นตอนที่เรียนรู้
ทั้งหมดจะยังคงรอดำเนินการจนกว่าผู้ดูแลจะตรวจสอบและนำไปใช้

การเรียนรู้ด้วยตนเอง **ปิดใช้งานโดยค่าเริ่มต้น** เปิดใช้งานเฉพาะเมื่อการเรียกใช้
โมเดลเบื้องหลังเพิ่มเติมและการตรวจสอบบันทึกการสนทนาเหมาะสมกับพื้นที่ทำงานของคุณ

## เปิดใช้งานการเรียนรู้ด้วยตนเอง

ใน Control UI ให้เปิด **Plugins → Workshop** แล้วเปิดสวิตช์ **Self-learning** การ
เปลี่ยนแปลงจะมีผลทันที เมื่อตัวเขียนการกำหนดค่าอื่นอัปเดตไฟล์แล้ว Control UI
จะรีเฟรชสแนปช็อตการกำหนดค่าและลองสลับอีกครั้งโดยไม่ต้องโหลดหน้าเว็บหรือ Gateway
ใหม่

ใช้ CLI:

```bash
openclaw config set skills.workshop.autonomous.enabled true --strict-json
```

หรือแก้ไข `~/.openclaw/openclaw.json`:

```json5
{
  skills: {
    workshop: {
      autonomous: {
        enabled: true,
      },
    },
  },
}
```

ปิดใช้งานอีกครั้งด้วย:

```bash
openclaw config set skills.workshop.autonomous.enabled false --strict-json
```

การสร้างสกิลตามคำขอของผู้ใช้, `/learn` และการดำเนินการ Skill Workshop
ด้วยตนเองยังคงทำงานได้ขณะปิดใช้งานการเรียนรู้ด้วยตนเอง

## ตรวจสอบเซสชันที่ผ่านมาโดยผู้ใช้

การตรวจสอบประวัติด้วยตนเองเป็นทางเลือกที่รัดกุมกว่าการเก็บข้อมูลแบบอัตโนมัติ
เปิด **Plugins → Workshop** ใน Control UI แล้วเลือก **Find skill ideas**
การดำเนินการนี้จะไม่เปลี่ยน `skills.workshop.autonomous.enabled`

การสแกนแต่ละครั้ง:

- เริ่มจากเซสชันที่ยังไม่ได้ตรวจสอบซึ่งใหม่ที่สุด แล้วย้อนกลับไป;
- ตรวจสอบเซสชันที่มีเนื้อหาสำคัญได้สูงสุด 20 เซสชัน โดยแต่ละเซสชันมีการโต้ตอบของโมเดลอย่างน้อยหกรอบ;
- ข้ามเซสชัน cron, heartbeat, hook, subagent, ACP, เซสชันที่ Plugin เป็นเจ้าของ และเซสชันตรวจสอบภายใน;
- ปกปิดความลับที่ตรวจพบและจำกัดขนาดชุดบันทึกการสนทนาก่อนส่งไปยังโมเดลที่กำหนดค่าไว้ของเอเจนต์ที่เลือก;
- ใช้เกณฑ์ที่เข้มงวดเช่นเดียวกับการตรวจสอบประสบการณ์แบบอัตโนมัติ; และ
- สามารถสร้างหรือแก้ไขข้อเสนอที่รอดำเนินการได้สูงสุดสามรายการ แต่จะไม่แก้ไขสกิลที่ใช้งานอยู่

Workshop จะแสดงจำนวนเซสชันสะสม ช่วงวันที่ที่ครอบคลุม และแนวคิดที่พบ
เลือก **Scan earlier work** เพื่อดูช่วงก่อนหน้าที่เก่ากว่าถัดไป เมื่อเคอร์เซอร์ไปถึง
จุดเริ่มต้นของประวัติที่เข้าเกณฑ์ การดำเนินการจะเปลี่ยนเป็น **Scan new work**
OpenClaw จะจัดเก็บเฉพาะเคอร์เซอร์และข้อมูลเมตาความครอบคลุมไว้ในฐานข้อมูลสถานะที่ใช้ร่วมกัน
โดยจะไม่สร้างคลังบันทึกการสนทนาชุดที่สอง

เซสชันจะถูกสแกนเฉพาะเมื่อ OpenClaw สามารถพิสูจน์ความเป็นเจ้าของและแยก
เนื้อหาจาก hook ภายนอกออกได้ หลังการอัปเกรด บันทึกการสนทนาปัจจุบันจากก่อนอัปเกรด
สามารถจำแนกได้ภายในเครื่อง แต่บันทึกการสนทนาก่อนอัปเกรดที่ถูกหมุนเวียนและไม่มี
ข้อมูลแหล่งที่มาระดับการเรียกใช้จะถูกข้าม บันทึกการสนทนาใหม่จะเก็บข้อมูลแหล่งที่มานี้
ไว้แม้ผ่านการหมุนเวียน

การสแกนด้วยตนเองยังคงมีค่าใช้จ่ายจากผู้ให้บริการโมเดล และส่งเนื้อหาการสนทนา
ที่เข้าเกณฑ์ไปยังผู้ให้บริการที่กำหนดค่าไว้ ใช้เฉพาะเมื่อการตรวจสอบดังกล่าวสอดคล้องกับ
ข้อกำหนดด้านความเป็นส่วนตัวและการจัดการข้อมูลของพื้นที่ทำงาน

## สิ่งที่ OpenClaw เรียนรู้ได้

การเรียนรู้ด้วยตนเองมีสองแนวทางที่รัดกุม:

1. **คำสั่งและการแก้ไขโดยตรง** OpenClaw ตรวจพบข้อความที่มีผลต่อเนื่อง
   เช่น “นับจากนี้ไป” “ครั้งถัดไป” และการแก้ไขแนวทางที่ล้มเหลว
   เมื่อเปิดใช้งานการเรียนรู้ด้วยตนเอง ระบบสามารถเปลี่ยนสัญญาณเหล่านั้นเป็นข้อเสนอ
   ที่รอดำเนินการได้โดยไม่ต้องรอพรอมต์อื่น เส้นทางแบบกำหนดแน่นอนนี้สามารถจัดกลุ่ม
   คำสั่งที่เกี่ยวข้องเป็นข้อเสนอได้สูงสุดสามรายการ กำหนดเป้าหมายเป็นสกิลในพื้นที่ทำงาน
   ที่เขียนได้ หรือแก้ไขข้อเสนอที่รอดำเนินการของตนเองซึ่งเกี่ยวข้อง นอกจากนี้ยังทำงาน
   หลังรอบที่ล้มเหลว เพราะเป็นการเก็บคำสั่งของผู้ใช้แทนที่จะตัดสินว่างานเสร็จสมบูรณ์หรือไม่
2. **การตรวจสอบประสบการณ์** หลังรอบเบื้องหน้าที่สำเร็จและมีเนื้อหาสำคัญ
   OpenClaw สามารถตรวจสอบงานที่เสร็จแล้วเพื่อหาเทคนิคการกู้คืนที่นำกลับมาใช้ใหม่ได้
   หรือขั้นตอนที่เสถียรซึ่งจะลดการโต้ตอบกับโมเดลหรือเครื่องมือในอนาคตได้อย่างน้อยสองรอบ

ตัวเลือกที่ดีได้แก่:

- การกู้คืนที่เชื่อถือได้หลังเครื่องมือหรือโมเดลล้มเหลวซ้ำหลายครั้ง;
- ข้อจำกัดด้านลำดับที่ไม่ชัดเจนซึ่งป้องกันข้อผิดพลาดที่เกิดซ้ำ;
- เวิร์กโฟลว์หลายขั้นตอนที่เสถียรซึ่งต้องค้นหาข้อมูลซ้ำ; หรือ
- การตรวจสอบล่วงหน้าที่นำกลับมาใช้ใหม่ได้และช่วยหลีกเลี่ยงการเรียกใช้หลายครั้งในอนาคต

ผู้ตรวจสอบควรงดสร้างข้อเสนอสำหรับงานประจำที่สำเร็จ คำขอครั้งเดียว ข้อเท็จจริงส่วนบุคคล
การตั้งค่าอย่างง่าย ความล้มเหลวชั่วคราวของสภาพแวดล้อม คำแนะนำทั่วไป
ข้อกล่าวอ้างเชิงลบที่ไม่มีหลักฐานรองรับ และความลับ

## การตรวจสอบประสบการณ์ทำงานเมื่อใด

การตรวจสอบประสบการณ์ถูกหน่วงเวลาและจำกัดขอบเขตโดยเจตนา:

- รอบเบื้องหน้าต้องเสร็จสิ้นโดยสำเร็จ
- รอบปัจจุบันต้องมีการวนซ้ำของโมเดลอย่างน้อยสิบครั้ง
- เซสชัน Cron, heartbeat, memory, overflow, hook, subagent และการตรวจสอบจะถูกยกเว้น
- การเรียกใช้เบื้องหน้าต้องระบุผู้ให้บริการและโมเดลได้ และต้องมีสิทธิ์เข้าถึง `skill_workshop` จริง
- OpenClaw จะรอ 30 วินาทีหลังเสร็จสิ้น หากมีการเสร็จสิ้นของงานเบื้องหน้าครั้งถัดไปในเซสชันเดียวกัน ระยะเวลาสงบนิ่งดังกล่าวจะเริ่มใหม่
- หากเอเจนต์หรือการเรียกใช้เพื่อตอบกลับใดยังทำงานอยู่ การตรวจสอบจะรออีก 30 วินาที
- การตรวจสอบประสบการณ์จะทำงานได้ครั้งละหนึ่งรายการเท่านั้น
- การตรวจสอบแบบหน่วงเวลาเป็นงาน Gateway ภายในกระบวนการ Gateway ต้องทำงานต่อไปตลอดช่วงว่าง รันไทม์ภายในเครื่องแบบครั้งเดียวและรันไทม์ที่ทำงานผ่าน CLI จะไม่เก็บบริบทเส้นทางการทำงานและความพร้อมของเครื่องมือมากพอสำหรับกำหนดเวลา

คำตอบเบื้องหน้าจะไม่ถูกหน่วงเวลาเพื่อการเรียนรู้ รอบที่ล้มเหลวหรือไม่เข้าเกณฑ์
จะไม่เริ่มการตรวจสอบประสบการณ์ แม้ว่าระบบยังสามารถเสนอการแก้ไขโดยตรงจากผู้ใช้
เป็นคำแนะนำได้เมื่อปิดใช้งานการทำงานแบบอัตโนมัติ

## สิ่งที่ผู้ตรวจสอบได้รับ

ผู้ตรวจสอบเบื้องหลังจะได้รับเฉพาะรอบปัจจุบัน โดยเริ่มจากข้อความล่าสุดของผู้ใช้
เส้นทางการทำงานที่เรนเดอร์แล้วถูกจำกัดไว้ที่ 60,000 อักขระ เมื่อจำเป็น OpenClaw
จะเก็บข้อความแรกและหลักฐานล่าสุดไว้ พร้อมทำเครื่องหมายส่วนกลางที่ถูกละไว้

ผู้ตรวจสอบจะใช้ผู้ให้บริการและโมเดลที่ระบุไว้ซ้ำ โดยใช้โปรไฟล์การยืนยันตัวตนของงาน
เบื้องหน้าซ้ำเมื่อข้อมูลประจำตัวนั้นพร้อมใช้งาน และปิดใช้งานโมเดลสำรอง
ดังนั้นการตรวจสอบจะเริ่มการเรียกใช้โมเดลเพิ่มเติมบนผู้ให้บริการที่กำหนดค่าไว้
การเรียกใช้นั้นอาจส่งคำขอไปยังผู้ให้บริการมากกว่าหนึ่งครั้งเมื่อมีการตรวจสอบหรือร่าง
ข้อเสนอ เงื่อนไขด้านราคาและการจัดการข้อมูลของผู้ให้บริการมีผลเช่นเดียวกับรอบเบื้องหน้า

ก่อนเริ่ม OpenClaw จะโหลดการกำหนดค่ารันไทม์ปัจจุบันอีกครั้ง และตรวจสอบนโยบาย
แซนด์บ็อกซ์และเครื่องมือที่มีผลสำหรับการสนทนาเดิมอีกครั้ง หากการเรียกใช้ทำงานใน
แซนด์บ็อกซ์ นโยบายไม่อนุญาต `skill_workshop` อีกต่อไป หรือไม่มีข้อมูลรันไทม์ที่จำเป็น
การตรวจสอบจะปฏิเสธการทำงานโดยค่าเริ่มต้นและไม่สร้างสิ่งใด

<Warning>
  การเปิดใช้งานการเรียนรู้ด้วยตนเองอนุญาตให้ส่งเนื้อหาการสนทนาที่เข้าเกณฑ์
  รวมถึงอินพุตและผลลัพธ์ของเครื่องมือจากรอบปัจจุบัน ไปยังผู้ให้บริการโมเดลที่เลือก
  เพื่อการตรวจสอบเพิ่มเติมหนึ่งครั้ง ห้ามเปิดใช้งานในพื้นที่ทำงานที่การตรวจสอบดังกล่าว
  จะละเมิดข้อกำหนดด้านการจัดการข้อมูล
</Warning>

## ความปลอดภัยของข้อเสนอ

ผู้ตรวจสอบทำงานในเซสชันแยกต่างหากโดยมีขอบเขตเครื่องมือที่จำกัดโดยเจตนา:

- สามารถแสดงรายการหรือตรวจสอบข้อเสนอของ Workshop และสร้างหรือแก้ไขข้อเสนอที่รอดำเนินการหนึ่งรายการเท่านั้น
- ไม่สามารถอัปเดตสกิลที่ใช้งานอยู่ นำข้อเสนอไปใช้ ปฏิเสธข้อเสนอ กักกันข้อเสนอ ส่งข้อความ หรือใช้เครื่องมือทั่วไปของเอเจนต์
- งบประมาณการแก้ไขหนึ่งครั้งจะใช้ร่วมกันในการลองใหม่ของโมเดล ดังนั้นการตรวจสอบหนึ่งครั้งจึงสร้างหรือแก้ไขข้อเสนอได้สูงสุดหนึ่งรายการ
- เส้นทางการทำงานที่ตรวจสอบถือเป็นหลักฐานที่ไม่น่าเชื่อถือ ไม่ใช่คำสั่งสำหรับเอเจนต์เบื้องหลัง
- Skill Workshop จะสแกนเนื้อหาข้อเสนอและปฏิเสธข้อมูลประจำตัวแบบตัวอักษรตรงที่ตรวจพบ ก่อนเขียนสถานะข้อเสนอ

ขีดจำกัดตามปกติของ Workshop ยังคงมีผล รวมถึง `maxPending`, `maxSkillBytes`,
ข้อจำกัดของไฟล์สนับสนุน การตรวจสอบของสแกนเนอร์ และการเขียนเฉพาะในพื้นที่ทำงาน
การตั้งค่า `approvalPolicy: "auto"` ไม่ได้ให้สิทธิ์เข้าถึงการดำเนินการในวงจรชีวิต
แก่ผู้ตรวจสอบเบื้องหลัง

## ตรวจสอบข้อเสนอที่เรียนรู้

การเรียนรู้ด้วยตนเองจะสร้างข้อเสนอที่รอดำเนินการแบบเดียวกับการใช้ Workshop ด้วยตนเอง
ตรวจสอบก่อนนำไปใช้:

```bash
openclaw skills workshop list
openclaw skills workshop inspect <proposal-id>
openclaw skills workshop apply <proposal-id>
```

แก้ไข ปฏิเสธ หรือกักกันข้อเสนอที่มีประโยชน์แต่ยังไม่พร้อม:

```bash
openclaw skills workshop revise <proposal-id> --proposal ./PROPOSAL.md
openclaw skills workshop reject <proposal-id> --reason "เฉพาะเจาะจงเกินไป"
openclaw skills workshop quarantine <proposal-id> --reason "ต้องตรวจสอบความปลอดภัย"
```

การนำไปใช้เป็นการดำเนินการเดียวที่จะเขียน `SKILL.md` ที่ใช้งานอยู่ ดู
[Skill Workshop](/th/tools/skill-workshop) สำหรับวงจรชีวิตและโมเดลการจัดเก็บฉบับสมบูรณ์

## การกำหนดค่า

| การตั้งค่า                                  | ค่าเริ่มต้น | ผลต่อการเรียนรู้ด้วยตนเอง                                                                                                         |
| ------------------------------------------ | -------- | --------------------------------------------------------------------------------------------------------------------------------- |
| `skills.workshop.autonomous.enabled`       | `false`  | เปิดใช้งานการเก็บการแก้ไขโดยตรงและการตรวจสอบประสบการณ์แบบหน่วงเวลา                                                               |
| `skills.workshop.approvalPolicy`           | `"auto"` | ควบคุมพรอมต์การอนุมัติสำหรับการดำเนินการตามวงจรชีวิตตามปกติที่เริ่มโดยเอเจนต์ แต่ไม่ขยายสิทธิ์ของผู้ตรวจสอบเบื้องหลัง |
| `skills.workshop.maxPending`               | `50`     | จำกัดจำนวนข้อเสนอที่รอดำเนินการและถูกกักกันต่อพื้นที่ทำงาน                                                                        |
| `skills.workshop.maxSkillBytes`            | `40000`  | จำกัดขนาดเนื้อหาข้อเสนอเป็นไบต์                                                                                                   |
| `skills.workshop.allowSymlinkTargetWrites` | `false`  | มีผลเฉพาะต่อพฤติกรรมการนำไปใช้ ตัวการเรียนรู้ด้วยตนเองจะเขียนสถานะข้อเสนอ ไม่ใช่เป้าหมายสกิลที่ใช้งานอยู่                         |

สำหรับสคีมาฉบับสมบูรณ์ ช่วงค่า และการตั้งค่าสกิลที่เกี่ยวข้อง โปรดดู
[การกำหนดค่า Skills](/th/tools/skills-config#workshop-skills-workshop)

## การแก้ไขปัญหา

### ไม่มีข้อเสนอปรากฏหลังรอบที่ยาวนาน

ตรวจสอบทั้งหมดต่อไปนี้:

1. `skills.workshop.autonomous.enabled` เป็น `true` ในการกำหนดค่า Gateway ที่ใช้งานอยู่
2. รอบสำเร็จและมีการวนซ้ำของโมเดลอย่างน้อยสิบครั้งหลังข้อความล่าสุดของผู้ใช้
3. การสนทนาเป็นการเรียกใช้เบื้องหน้าตามปกติ ไม่ใช่การเรียกใช้ตามกำหนดเวลา, memory, hook หรือ subagent
4. การเรียกใช้เดิมมีสิทธิ์เข้าถึง `skill_workshop` และไม่ได้ทำงานในแซนด์บ็อกซ์
5. ระบบไม่มีการทำงานนานพอสำหรับการตรวจสอบแบบหน่วงเวลา
6. กระบวนการ Gateway ที่ทำงานระยะยาวยังคงทำงานตลอดช่วงว่าง คำสั่งภายในเครื่องแบบครั้งเดียวจะไม่รอการตรวจสอบแบบหน่วงเวลา

การตรวจสอบที่เข้าเกณฑ์อาจยังไม่สร้างข้อเสนอ การงดสร้างข้อเสนอเป็นผลลัพธ์ที่คาดไว้
เมื่อหลักฐานไม่ผ่านเกณฑ์สำหรับขั้นตอนที่นำกลับมาใช้ใหม่ได้

### Doctor รายงานว่าเครื่องมือ Workshop ถูกซ่อน

เมื่อเปิดใช้งานการเรียนรู้ด้วยตนเอง `openclaw doctor` จะตรวจสอบว่านโยบายเครื่องมือ
ที่มีผลของเอเจนต์เริ่มต้นอนุญาต `skill_workshop` หรือไม่ ทำตามการเปลี่ยนแปลง
`tools.allow` หรือ `tools.alsoAllow` ที่รายงาน หรือปิดใช้งานการเรียนรู้ด้วยตนเอง

### มีข้อเสนอที่มีคุณค่าต่ำปรากฏมากเกินไป

ปิดใช้งานการเรียนรู้ด้วยตนเองและใช้ `/learn` หรือคำขอ Workshop โดยตรงต่อไป:

```bash
openclaw config set skills.workshop.autonomous.enabled false --strict-json
```

ข้อเสนอที่รอดำเนินการยังคงตรวจสอบได้หลังปิดใช้งานคุณลักษณะนี้ การปิดใช้งาน
การเรียนรู้ด้วยตนเองจะไม่นำข้อเสนอไปใช้ ปฏิเสธ หรือลบข้อเสนอเหล่านั้น
