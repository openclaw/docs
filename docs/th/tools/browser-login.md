## การเข้าสู่ระบบด้วยตนเอง (แนะนำ)

เมื่อเว็บไซต์กำหนดให้เข้าสู่ระบบ ให้ลงชื่อเข้าใช้ด้วยตนเองในโปรไฟล์ `openclaw`
ของเบราว์เซอร์บนโฮสต์ อย่าให้ข้อมูลประจำตัวของคุณแก่โมเดล เนื่องจากการเข้าสู่ระบบอัตโนมัติมัก
กระตุ้นระบบป้องกันบอตและอาจทำให้บัญชีถูกล็อก

ใช้เบราว์เซอร์บนโฮสต์ (เข้าสู่ระบบด้วยตนเอง) ทั้งสำหรับการอ่าน (การค้นหา/กระทู้) และ
การโพสต์บน X/Twitter รวมถึงเว็บไซต์อื่น ๆ ที่ไวต่อบอต เซสชันเบราว์เซอร์แบบแซนด์บ็อกซ์
มีแนวโน้มที่จะกระตุ้นการตรวจจับบอตมากกว่า

กลับไปยังเอกสารหลักเกี่ยวกับเบราว์เซอร์: [เบราว์เซอร์](/th/tools/browser)

## ใช้โปรไฟล์ Chrome ใด?

OpenClaw ควบคุมโปรไฟล์ Chrome เฉพาะชื่อ `openclaw` (ส่วนติดต่อผู้ใช้
โทนสีส้ม) ซึ่งแยกจากโปรไฟล์เบราว์เซอร์ที่คุณใช้ประจำ

สำหรับการเรียกใช้เครื่องมือเบราว์เซอร์ของเอเจนต์:

- ตัวเลือกเริ่มต้น: เอเจนต์ใช้เบราว์เซอร์ `openclaw` ที่แยกออกมาโดยเฉพาะ
- ใช้ `profile="user"` เฉพาะเมื่อจำเป็นต้องใช้เซสชันที่เข้าสู่ระบบไว้อยู่แล้ว และคุณ
  อยู่หน้าเครื่องเพื่อคลิก/อนุมัติข้อความแจ้งให้เชื่อมต่อ
- หากคุณมีโปรไฟล์เบราว์เซอร์ของผู้ใช้หลายโปรไฟล์ ให้ระบุโปรไฟล์อย่างชัดเจน
  แทนการคาดเดา

มีสองวิธีในการเข้าถึงโปรไฟล์ `openclaw`:

1. ขอให้เอเจนต์เปิดเบราว์เซอร์ แล้วเข้าสู่ระบบด้วยตนเอง
2. เปิดผ่าน CLI:

```bash
openclaw browser start
openclaw browser open https://x.com
```

สำหรับโปรไฟล์ที่ไม่ใช่ค่าเริ่มต้น ให้ใส่ `--browser-profile <name>` ก่อน
คำสั่งย่อย (ค่าเริ่มต้นคือ `openclaw`):

```bash
openclaw browser --browser-profile <name> open https://x.com
```

## การใช้แซนด์บ็อกซ์: อนุญาตให้เข้าถึงเบราว์เซอร์บนโฮสต์

หากเอเจนต์ทำงานในแซนด์บ็อกซ์ การเรียกใช้เครื่องมือ `browser` จะใช้
เบราว์เซอร์ในแซนด์บ็อกซ์เป็นค่าเริ่มต้น ไม่ใช่เบราว์เซอร์บนโฮสต์ หากต้องการให้เอเจนต์ใช้เบราว์เซอร์บนโฮสต์แทน:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

การเรียกใช้ผ่าน CLI จะใช้เบราว์เซอร์บนโฮสต์เสมอและไม่ใช้แซนด์บ็อกซ์ ดังนั้นคุณจึงสามารถ
เปิดเบราว์เซอร์บนโฮสต์ด้วยตนเองได้โดยไม่ขึ้นกับการตั้งค่านี้:

```bash
openclaw browser --browser-profile openclaw open https://x.com
```

เมื่อตั้งค่า `sandbox.browser.allowHostControl: true` แล้ว การเรียกใช้เครื่องมือ
`browser` ของเอเจนต์ก็สามารถใช้เบราว์เซอร์บนโฮสต์ได้เช่นกัน อีกทางเลือกหนึ่งคือปิดการใช้แซนด์บ็อกซ์สำหรับ
เอเจนต์ที่โพสต์ข้อมูลอัปเดต
