## การเข้าสู่ระบบด้วยตนเอง (แนะนำ)

เมื่อเว็บไซต์ต้องการการเข้าสู่ระบบ ให้ **ลงชื่อเข้าใช้ด้วยตนเอง** ในโปรไฟล์เบราว์เซอร์ของ **โฮสต์** (เบราว์เซอร์ openclaw)

อย่าให้ข้อมูลประจำตัวของคุณกับโมเดล การเข้าสู่ระบบอัตโนมัติมักกระตุ้นกลไกป้องกันบอทและอาจทำให้บัญชีถูกล็อกได้

กลับไปยังเอกสารเบราว์เซอร์หลัก: [เบราว์เซอร์](/th/tools/browser)

## ใช้โปรไฟล์ Chrome ใด?

OpenClaw ควบคุม **โปรไฟล์ Chrome เฉพาะ** (ชื่อ `openclaw` อินเทอร์เฟซผู้ใช้โทนสีส้ม) ซึ่งแยกจากโปรไฟล์เบราว์เซอร์ที่คุณใช้ประจำวัน

สำหรับการเรียกใช้เครื่องมือเบราว์เซอร์ของเอเจนต์:

- ตัวเลือกเริ่มต้น: เอเจนต์ควรใช้เบราว์เซอร์ `openclaw` ที่แยกออกมาของตนเอง
- ใช้ `profile="user"` เฉพาะเมื่อเซสชันที่ลงชื่อเข้าใช้อยู่แล้วมีความสำคัญ และผู้ใช้อยู่ที่คอมพิวเตอร์เพื่อคลิก/อนุมัติพรอมต์การแนบใด ๆ
- หากคุณมีโปรไฟล์เบราว์เซอร์ผู้ใช้หลายโปรไฟล์ ให้ระบุโปรไฟล์อย่างชัดเจนแทนการคาดเดา

สองวิธีง่าย ๆ ในการเข้าถึง:

1. **ขอให้เอเจนต์เปิดเบราว์เซอร์** แล้วลงชื่อเข้าใช้ด้วยตนเอง
2. **เปิดผ่าน CLI**:

```bash
openclaw browser start
openclaw browser open https://x.com
```

หากคุณมีหลายโปรไฟล์ ให้ส่ง `--browser-profile <name>` (ค่าเริ่มต้นคือ `openclaw`)

## X/Twitter: ขั้นตอนที่แนะนำ

- **อ่าน/ค้นหา/เธรด:** ใช้เบราว์เซอร์ของ **โฮสต์** (เข้าสู่ระบบด้วยตนเอง)
- **โพสต์การอัปเดต:** ใช้เบราว์เซอร์ของ **โฮสต์** (เข้าสู่ระบบด้วยตนเอง)

## แซนด์บ็อกซ์ + การเข้าถึงเบราว์เซอร์ของโฮสต์

เซสชันเบราว์เซอร์ในแซนด์บ็อกซ์มีแนวโน้ม **มากกว่า** ที่จะกระตุ้นการตรวจจับบอท สำหรับ X/Twitter (และเว็บไซต์อื่น ๆ ที่เข้มงวด) ให้เลือกใช้เบราว์เซอร์ของ **โฮสต์**

หากเอเจนต์อยู่ในแซนด์บ็อกซ์ เครื่องมือเบราว์เซอร์จะใช้แซนด์บ็อกซ์เป็นค่าเริ่มต้น หากต้องการอนุญาตให้ควบคุมโฮสต์:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

จากนั้นกำหนดเป้าหมายไปยังเบราว์เซอร์ของโฮสต์:

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

หรือปิดใช้งานแซนด์บ็อกซ์สำหรับเอเจนต์ที่โพสต์การอัปเดต
