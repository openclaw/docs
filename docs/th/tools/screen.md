เครื่องมือ `screen` ช่วยให้เอเจนต์จัดวาง Control UI บนเบราว์เซอร์ได้ โดยเป็น
พื้นผิวการจัดวางและการนำทางแบบมีชนิดข้อมูล ไม่ใช่การจับภาพหน้าจอหรือระบบอัตโนมัติ
ของเบราว์เซอร์

เครื่องมือนี้จะแสดงให้ใช้เฉพาะเมื่อไคลเอนต์ต้นทางประกาศความสามารถ
`ui-commands` เท่านั้น และขณะเครื่องมือทำงาน ต้องยังมี Control UI ที่รองรับอย่างน้อยหนึ่งรายการ
เชื่อมต่ออยู่ มิฉะนั้น Gateway จะส่งคืน `UNAVAILABLE`

## การดำเนินการ

| การดำเนินการ                            | ผลลัพธ์                                     | อินพุตที่ไม่บังคับ                                |
| --------------------------------- | ------------------------------------------ | ---------------------------------------------- |
| `split_right`                     | แบ่งบานหน้าต่างเซสชันเป้าหมายไปทางขวา | `sessionKey` (ค่าเริ่มต้นคือเซสชันปัจจุบัน) |
| `split_down`                      | แบ่งบานหน้าต่างเซสชันเป้าหมายลงด้านล่าง     | `sessionKey` (ค่าเริ่มต้นคือเซสชันปัจจุบัน) |
| `close_pane`                      | ปิดบานหน้าต่างเซสชันเป้าหมาย              | `sessionKey` (ค่าเริ่มต้นคือเซสชันปัจจุบัน) |
| `focus`                           | โฟกัสบานหน้าต่างเซสชันเป้าหมาย              | `sessionKey` (ค่าเริ่มต้นคือเซสชันปัจจุบัน) |
| `navigate`                        | เปิดเซสชันเป้าหมาย                    | `sessionKey` (ค่าเริ่มต้นคือเซสชันปัจจุบัน) |
| `sidebar_show` / `sidebar_hide`   | แสดงหรือซ่อนแถบด้านข้างหลัก              | -                                              |
| `terminal_show` / `terminal_hide` | แสดงหรือซ่อนแผงเทอร์มินัลของผู้ปฏิบัติงาน   | `dock` (`bottom` หรือ `right`) เมื่อแสดง      |
| `browser_show` / `browser_hide`   | แสดงหรือซ่อนแผงเบราว์เซอร์             | `dock` (`bottom` หรือ `right`) เมื่อแสดง      |

คำสั่งที่สำเร็จจะส่งคืน `{ "ok": true }` หลังจาก Gateway เผยแพร่
เหตุการณ์แบบมีชนิดข้อมูล `ui.command`

## การกำหนดเส้นทางและความปลอดภัย

โปรโตคอล v1 ตั้งใจส่งคำสั่งไปยัง Control UI ที่เชื่อมต่ออยู่ทุกตัวซึ่ง
ประกาศ `ui-commands` โดยไม่ได้กำหนดเป้าหมายเป็นแท็บเบราว์เซอร์ใดแท็บหนึ่ง ประเด็นนี้สำคัญเมื่อ
ผู้ปฏิบัติงานคนเดียวกันเปิดแดชบอร์ดไว้หลายรายการ

RPC ของ Gateway ต้องใช้ `operator.write` เครื่องมือนี้เปลี่ยนได้เฉพาะสถานะ
การนำเสนอเท่านั้น โดยไม่สามารถอ่านพิกเซล จับภาพหน้าจอ คลิกเนื้อหาใด ๆ
บนหน้าเว็บ หรือข้ามสิทธิ์ของเซสชันและแผงผู้ปฏิบัติงาน
ที่เลือกไว้ได้
