ใช้หน้านี้เพื่อเลือกพื้นผิวความสามารถที่เหมาะสม **เครื่องมือ** คือการดำเนินการที่เรียกใช้ได้ **Skills** สอนเอเจนต์ถึงวิธีทำงาน และ **Plugin** เพิ่มความสามารถขณะรัน เช่น เครื่องมือ ผู้ให้บริการ ช่องทาง ฮุก และ Skills ที่จัดแพ็กเกจไว้

หน้านี้เป็นภาพรวมและหน้าแนะนำเส้นทาง สำหรับนโยบายเครื่องมือ ค่าเริ่มต้น การเป็นสมาชิกกลุ่ม ข้อจำกัดของผู้ให้บริการ และฟิลด์การกำหนดค่าอย่างครบถ้วน โปรดดู [เครื่องมือและผู้ให้บริการแบบกำหนดเอง](/th/gateway/config-tools)

## เลือกเครื่องมือ Skills หรือ Plugin

<Steps>
  <Step title="ใช้เครื่องมือเมื่อเอเจนต์ต้องดำเนินการ">
    เครื่องมือคือฟังก์ชันแบบมีชนิดที่เอเจนต์สามารถเรียกใช้ได้ เช่น `exec`, `browser`,
    `web_search`, `message` หรือ `image_generate` ใช้เครื่องมือเมื่อเอเจนต์
    ต้องอ่านข้อมูล เปลี่ยนไฟล์ ส่งข้อความ เรียกผู้ให้บริการ หรือ
    ควบคุมระบบอื่น เครื่องมือที่มองเห็นได้จะถูกส่งไปยังโมเดลในรูปแบบ
    คำจำกัดความฟังก์ชันแบบมีโครงสร้าง

    โมเดลจะเห็นเฉพาะเครื่องมือที่ผ่านโปรไฟล์ที่ใช้งานอยู่ นโยบายอนุญาต/ปฏิเสธ
    ข้อจำกัดของผู้ให้บริการ สถานะแซนด์บ็อกซ์ สิทธิ์ของช่องทาง และ
    ความพร้อมใช้งานของ Plugin เท่านั้น

  </Step>

  <Step title="ใช้ Skill เมื่อเอเจนต์ต้องการคำสั่ง">
    Skill คือชุดคำสั่ง `SKILL.md` ที่โหลดเข้าสู่พรอมต์ของเอเจนต์ ใช้
    Skill เมื่อเอเจนต์มีเครื่องมือที่ต้องการอยู่แล้ว แต่ต้องการ
    เวิร์กโฟลว์ที่ทำซ้ำได้ เกณฑ์การตรวจสอบ ลำดับคำสั่ง หรือ
    ข้อจำกัดในการปฏิบัติงาน

    Skills สามารถอยู่ในเวิร์กสเปซ ไดเรกทอรี Skills ที่ใช้ร่วมกัน ราก Skills
    ของ OpenClaw ที่มีการจัดการ หรือแพ็กเกจ Plugin

    [Skills](/th/tools/skills) | [เวิร์กชอป Skills](/th/tools/skill-workshop) | [การสร้าง Skills](/th/tools/creating-skills) | [การกำหนดค่า Skills](/th/tools/skills-config)

  </Step>

  <Step title="ใช้ Plugin เมื่อ OpenClaw ต้องการความสามารถใหม่">
    Plugin สามารถเพิ่มเครื่องมือ Skills ช่องทาง ผู้ให้บริการโมเดล เสียงพูด
    เสียงแบบเรียลไทม์ การสร้างสื่อ การค้นหาเว็บ การดึงข้อมูลเว็บ ฮุก และ
    ความสามารถขณะรันอื่น ๆ ใช้ Plugin เมื่อความสามารถนั้นมีโค้ด
    ข้อมูลรับรอง ฮุกวงจรชีวิต เมทาดาทาของไฟล์กำกับ หรือแพ็กเกจ
    ที่ติดตั้งได้ Plugin ที่มีอยู่สามารถติดตั้งจาก ClawHub, npm, git,
    ไดเรกทอรีภายในเครื่อง หรือไฟล์บีบอัด

    [ติดตั้งและกำหนดค่า Plugin](/th/tools/plugin) | [สร้าง Plugin](/th/plugins/building-plugins) | [SDK ของ Plugin](/th/plugins/sdk-overview)

  </Step>
</Steps>

## เครื่องมือที่ Plugin จัดหาให้

Plugin สามารถลงทะเบียนเครื่องมือเพิ่มเติมได้ ผู้เขียน Plugin เชื่อมต่อเครื่องมือผ่าน
`api.registerTool(...)` และ `contracts.tools` ในไฟล์กำกับ โปรดดู
[SDK ของ Plugin](/th/plugins/sdk-overview) และ [ไฟล์กำกับ Plugin](/th/plugins/manifest)
สำหรับรายละเอียดสัญญา

เครื่องมือที่ Plugin จัดหาให้โดยทั่วไป ได้แก่:

- [ความแตกต่าง](/th/tools/diffs) สำหรับแสดงผลความแตกต่างของไฟล์และมาร์กดาวน์
- [แสดงวิดเจ็ต](/tools/show-widget) สำหรับ SVG และ HTML แบบอินไลน์ที่ทำงานได้ในตัวเองในเว็บแชต
- [งาน LLM](/th/tools/llm-task) สำหรับขั้นตอนเวิร์กโฟลว์ที่ใช้ JSON เท่านั้น
- [Lobster](/th/tools/lobster) สำหรับเวิร์กโฟลว์แบบมีชนิดพร้อมการอนุมัติที่กลับมาทำต่อได้
- [Tokenjuice](/th/tools/tokenjuice) สำหรับย่อเอาต์พุตที่มีข้อมูลรบกวนจากเครื่องมือ `exec` และ `bash`
- [การค้นหาเครื่องมือ](/th/tools/tool-search) สำหรับค้นหาและเรียกใช้แค็ตตาล็อกเครื่องมือ
  ขนาดใหญ่โดยไม่ใส่ทุกสคีมาไว้ในพรอมต์
- [Canvas](/th/plugins/reference/canvas) สำหรับควบคุม Canvas ของ Node และ
  การเรนเดอร์ A2UI

## กำหนดค่าสิทธิ์เข้าถึงและการอนุมัติ

นโยบายเครื่องมือจะถูกบังคับใช้ก่อนการเรียกโมเดล หากนโยบายนำเครื่องมือออก
โมเดลจะไม่ได้รับสคีมาของเครื่องมือนั้นสำหรับรอบดังกล่าว การรันอาจสูญเสียเครื่องมือ
เนื่องจากการกำหนดค่าส่วนกลาง การกำหนดค่ารายเอเจนต์ นโยบายช่องทาง ข้อจำกัดของผู้ให้บริการ
กฎแซนด์บ็อกซ์ นโยบายช่องทาง/รันไทม์ หรือความพร้อมใช้งานของ Plugin

- [เครื่องมือและผู้ให้บริการแบบกำหนดเอง](/th/gateway/config-tools) อธิบายโปรไฟล์เครื่องมือ
  รายการอนุญาต/ปฏิเสธ ข้อจำกัดเฉพาะผู้ให้บริการ การตรวจจับลูป และ
  การตั้งค่าเครื่องมือที่สนับสนุนโดยผู้ให้บริการ
- [การอนุมัติ Exec](/th/tools/exec-approvals) อธิบายนโยบายการอนุมัติ
  คำสั่งโฮสต์
- [Exec แบบยกระดับ](/th/tools/elevated) อธิบายการเรียกใช้งานที่มีการควบคุมนอก
  แซนด์บ็อกซ์
- [แซนด์บ็อกซ์ เทียบกับนโยบายเครื่องมือ เทียบกับแบบยกระดับ](/th/gateway/sandbox-vs-tool-policy-vs-elevated)
  อธิบายว่าเลเยอร์ใดควบคุมการเข้าถึงไฟล์และกระบวนการ
- [ข้อจำกัดแซนด์บ็อกซ์และเครื่องมือรายเอเจนต์](/th/tools/multi-agent-sandbox-tools)
  อธิบายข้อจำกัดเฉพาะเอเจนต์สำหรับการรันที่มอบหมาย

## ขยายความสามารถ

เลือกเส้นทางการขยายตามงานที่คุณต้องการให้ OpenClaw ทำ:

- ติดตั้งหรือจัดการ Plugin ที่มีอยู่ด้วย [Plugin](/th/tools/plugin)
- สร้างการผสานรวม ผู้ให้บริการ ช่องทาง เครื่องมือ หรือฮุกใหม่ด้วย
  [สร้าง Plugin](/th/plugins/building-plugins)
- เพิ่มหรือปรับแต่งคำสั่งเอเจนต์ที่นำกลับมาใช้ซ้ำได้ด้วย [Skills](/th/tools/skills) และ
  [การสร้าง Skills](/th/tools/creating-skills)
- ใช้ [SDK ของ Plugin](/th/plugins/sdk-overview) และ
  [ไฟล์กำกับ Plugin](/th/plugins/manifest) เมื่อคุณต้องการสัญญา
  การนำไปใช้

## แก้ไขปัญหาเครื่องมือที่หายไป

หากโมเดลมองไม่เห็นหรือเรียกใช้เครื่องมือไม่ได้ ให้เริ่มจากนโยบายที่มีผลสำหรับ
รอบปัจจุบัน:

1. ตรวจสอบโปรไฟล์ที่ใช้งานอยู่ `tools.allow` และ `tools.deny` ใน
   [เครื่องมือและผู้ให้บริการแบบกำหนดเอง](/th/gateway/config-tools)
2. ตรวจสอบข้อจำกัดเฉพาะผู้ให้บริการใน
   [เครื่องมือและผู้ให้บริการแบบกำหนดเอง](/th/gateway/config-tools) และยืนยันว่า
   [ผู้ให้บริการโมเดล](/th/concepts/model-providers) ที่เลือก รองรับรูปแบบของเครื่องมือ
3. ตรวจสอบสิทธิ์ของช่องทาง สถานะแซนด์บ็อกซ์ และสิทธิ์เข้าถึงแบบยกระดับด้วย
   [แซนด์บ็อกซ์ เทียบกับนโยบายเครื่องมือ เทียบกับแบบยกระดับ](/th/gateway/sandbox-vs-tool-policy-vs-elevated)
   และ [Exec แบบยกระดับ](/th/tools/elevated)
4. ตรวจสอบว่า Plugin เจ้าของได้รับการติดตั้งและเปิดใช้งานใน
   [Plugin](/th/tools/plugin) แล้วหรือไม่
5. สำหรับการรันที่มอบหมาย ให้ตรวจสอบข้อจำกัดรายเอเจนต์ใน
   [ข้อจำกัดแซนด์บ็อกซ์และเครื่องมือรายเอเจนต์](/th/tools/multi-agent-sandbox-tools)
6. สำหรับแค็ตตาล็อก OpenClaw ขนาดใหญ่ ให้ยืนยันว่าการรันใช้การเปิดเผยเครื่องมือ
   โดยตรงหรือ [การค้นหาเครื่องมือ](/th/tools/tool-search)

## ที่เกี่ยวข้อง

- [ระบบอัตโนมัติ](/th/automation) สำหรับ Cron, งาน, Heartbeat, ข้อผูกพัน, ฮุก,
  คำสั่งถาวร และ TaskFlow
- [เอเจนต์](/th/concepts/agent) สำหรับโมเดลเอเจนต์, เซสชัน, หน่วยความจำ และ
  การประสานงานระหว่างหลายเอเจนต์
- [เครื่องมือและผู้ให้บริการแบบกำหนดเอง](/th/gateway/config-tools) สำหรับเอกสารอ้างอิงมาตรฐานเกี่ยวกับ
  นโยบายเครื่องมือ
- [Plugin](/th/tools/plugin) สำหรับการติดตั้งและจัดการ Plugin
- [SDK สำหรับ Plugin](/th/plugins/sdk-overview) สำหรับเอกสารอ้างอิงของผู้สร้าง Plugin
- [Skills](/th/tools/skills) สำหรับลำดับการโหลด Skills, การควบคุมสิทธิ์ และการกำหนดค่า
- [เวิร์กช็อป Skills](/th/tools/skill-workshop) สำหรับการสร้าง Skills
  ที่ระบบสร้างและผ่านการตรวจสอบ
- [การค้นหาเครื่องมือ](/th/tools/tool-search) สำหรับการค้นหาแค็ตตาล็อกเครื่องมือ OpenClaw
  แบบกระชับ
