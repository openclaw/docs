ใช้หน้านี้เพื่อเลือกพื้นผิวความสามารถที่เหมาะสม **เครื่องมือ** คือการดำเนินการที่เรียกใช้ได้ **Skills** สอนเอเจนต์ให้ทำงาน และ **Plugin** เพิ่มความสามารถขณะรัน เช่น เครื่องมือ ผู้ให้บริการ ช่องทาง ฮุก และ Skills ที่จัดแพ็กเกจไว้

หน้านี้เป็นภาพรวมและหน้าสำหรับนำทาง หากต้องการรายละเอียดทั้งหมดเกี่ยวกับนโยบายเครื่องมือ ค่าเริ่มต้น การเป็นสมาชิกกลุ่ม ข้อจำกัดของผู้ให้บริการ และฟิลด์การกำหนดค่า โปรดดู [เครื่องมือและผู้ให้บริการแบบกำหนดเอง](/th/gateway/config-tools)

## เลือกเครื่องมือ Skills หรือ Plugin

<Steps>
  <Step title="ใช้เครื่องมือเมื่อเอเจนต์ต้องดำเนินการ">
    เครื่องมือคือฟังก์ชันแบบมีชนิดที่เอเจนต์เรียกใช้ได้ เช่น `exec`, `browser`,
    `web_search`, `message` หรือ `image_generate` ใช้เครื่องมือเมื่อเอเจนต์
    ต้องอ่านข้อมูล แก้ไขไฟล์ ส่งข้อความ เรียกผู้ให้บริการ หรือ
    ควบคุมระบบอื่น เครื่องมือที่มองเห็นได้จะถูกส่งไปยังโมเดลเป็น
    ข้อกำหนดฟังก์ชันแบบมีโครงสร้าง

    โมเดลจะเห็นเฉพาะเครื่องมือที่ผ่านโปรไฟล์ที่ใช้งานอยู่ นโยบาย
    อนุญาต/ปฏิเสธ ข้อจำกัดของผู้ให้บริการ สถานะแซนด์บ็อกซ์ สิทธิ์ของช่องทาง และ
    ความพร้อมใช้งานของ Plugin

  </Step>

  <Step title="ใช้ Skill เมื่อเอเจนต์ต้องการคำแนะนำ">
    Skill คือชุดคำแนะนำ `SKILL.md` ที่โหลดเข้าสู่พรอมต์ของเอเจนต์ ใช้
    Skill เมื่อเอเจนต์มีเครื่องมือที่ต้องการอยู่แล้ว แต่ต้องการ
    เวิร์กโฟลว์ที่ทำซ้ำได้ เกณฑ์การรีวิว ลำดับคำสั่ง หรือข้อจำกัด
    ในการปฏิบัติงาน

    Skills สามารถอยู่ในพื้นที่ทำงาน ไดเรกทอรี Skills ที่ใช้ร่วมกัน ราก Skills
    ของ OpenClaw ที่มีการจัดการ หรือแพ็กเกจ Plugin

    [Skills](/th/tools/skills) | [เวิร์กช็อป Skills](/th/tools/skill-workshop) | [การเรียนรู้ด้วยตนเอง](/th/tools/self-learning) | [การสร้าง Skills](/th/tools/creating-skills) | [การกำหนดค่า Skills](/th/tools/skills-config)

  </Step>

  <Step title="ใช้ Plugin เมื่อ OpenClaw ต้องการความสามารถใหม่">
    Plugin สามารถเพิ่มเครื่องมือ Skills ช่องทาง ผู้ให้บริการโมเดล เสียงพูด
    เสียงแบบเรียลไทม์ การสร้างสื่อ การค้นหาเว็บ การดึงข้อมูลเว็บ ฮุก และ
    ความสามารถขณะรันอื่นๆ ใช้ Plugin เมื่อความสามารถนั้นมีโค้ด
    ข้อมูลรับรอง ฮุกวงจรชีวิต เมทาดาทาของแมนิเฟสต์ หรือแพ็กเกจ
    ที่ติดตั้งได้ Plugin ที่มีอยู่สามารถติดตั้งจาก ClawHub, npm, git,
    ไดเรกทอรีในเครื่อง หรือไฟล์เก็บถาวร

    [ติดตั้งและกำหนดค่า Plugin](/th/tools/plugin) | [สร้าง Plugin](/th/plugins/building-plugins) | [Plugin SDK](/th/plugins/sdk-overview)

  </Step>
</Steps>

## เครื่องมือที่ Plugin จัดเตรียมให้

Plugin สามารถลงทะเบียนเครื่องมือเพิ่มเติมได้ ผู้เขียน Plugin เชื่อมต่อเครื่องมือผ่าน
`api.registerTool(...)` และ `contracts.tools` ของแมนิเฟสต์ โปรดดู
[Plugin SDK](/th/plugins/sdk-overview) และ [แมนิเฟสต์ Plugin](/th/plugins/manifest)
สำหรับรายละเอียดสัญญา

เครื่องมือทั่วไปที่ Plugin จัดเตรียมให้ ได้แก่:

- [Diffs](/th/tools/diffs) สำหรับแสดงผล diff ของไฟล์และ Markdown
- [แสดงวิดเจ็ต](/th/tools/show-widget) สำหรับ SVG และ HTML แบบอินไลน์ที่ทำงานได้ในตัวเองบนไคลเอนต์แชตที่รองรับ
- [หน้าจอ](/th/tools/screen) สำหรับจัดวาง Control UI ที่เชื่อมต่ออยู่
- [งาน LLM](/th/tools/llm-task) สำหรับขั้นตอนเวิร์กโฟลว์ที่ใช้เฉพาะ JSON
- [Lobster](/th/tools/lobster) สำหรับเวิร์กโฟลว์แบบมีชนิดพร้อมการอนุมัติที่ดำเนินการต่อได้
- [Tokenjuice](/th/tools/tokenjuice) สำหรับย่อเอาต์พุตเครื่องมือ `exec` และ `bash`
  ที่มีข้อมูลรบกวนให้กระชับ
- [ค้นหาเครื่องมือ](/th/tools/tool-search) สำหรับค้นหาและเรียกใช้แค็ตตาล็อกเครื่องมือขนาดใหญ่
  โดยไม่ต้องใส่ทุกสคีมาไว้ในพรอมต์
- [Canvas](/th/plugins/reference/canvas) สำหรับควบคุม Canvas ของ Node และ
  แสดงผล A2UI

## กำหนดค่าการเข้าถึงและการอนุมัติ

นโยบายเครื่องมือจะถูกบังคับใช้ก่อนเรียกโมเดล หากนโยบายนำเครื่องมือออก
โมเดลจะไม่ได้รับสคีมาของเครื่องมือนั้นสำหรับรอบนี้ การเรียกใช้งานอาจสูญเสียเครื่องมือ
เนื่องจากการกำหนดค่าส่วนกลาง การกำหนดค่ารายเอเจนต์ นโยบายช่องทาง ข้อจำกัดของผู้ให้บริการ
กฎแซนด์บ็อกซ์ นโยบายช่องทาง/รันไทม์ หรือความพร้อมใช้งานของ Plugin

- [เครื่องมือและผู้ให้บริการแบบกำหนดเอง](/th/gateway/config-tools) อธิบายโปรไฟล์เครื่องมือ
  รายการอนุญาต/ปฏิเสธ ข้อจำกัดเฉพาะผู้ให้บริการ การตรวจจับลูป และ
  การตั้งค่าเครื่องมือที่มีผู้ให้บริการรองรับ
- [การอนุมัติ Exec](/th/tools/exec-approvals) อธิบายนโยบายการอนุมัติ
  คำสั่งโฮสต์
- [Exec แบบยกระดับสิทธิ์](/th/tools/elevated) อธิบายการดำเนินการที่มีการควบคุมนอก
  แซนด์บ็อกซ์
- [แซนด์บ็อกซ์เทียบกับนโยบายเครื่องมือเทียบกับการยกระดับสิทธิ์](/th/gateway/sandbox-vs-tool-policy-vs-elevated)
  อธิบายว่าแต่ละชั้นควบคุมการเข้าถึงไฟล์และกระบวนการอย่างไร
- [ข้อจำกัดแซนด์บ็อกซ์และเครื่องมือรายเอเจนต์](/th/tools/multi-agent-sandbox-tools)
  อธิบายข้อจำกัดเฉพาะเอเจนต์สำหรับการเรียกใช้ที่มอบหมาย

## ขยายความสามารถ

เลือกแนวทางการขยายตามงานที่ต้องการให้ OpenClaw ทำ:

- ติดตั้งหรือจัดการ Plugin ที่มีอยู่ด้วย [Plugin](/th/tools/plugin)
- สร้างการผสานรวม ผู้ให้บริการ ช่องทาง เครื่องมือ หรือฮุกใหม่ด้วย
  [สร้าง Plugin](/th/plugins/building-plugins)
- เพิ่มหรือปรับแต่งคำสั่งเอเจนต์ที่นำกลับมาใช้ใหม่ได้ด้วย [Skills](/th/tools/skills) และ
  [การสร้าง Skills](/th/tools/creating-skills)
- ใช้ [Plugin SDK](/th/plugins/sdk-overview) และ
  [ไฟล์ Manifest ของ Plugin](/th/plugins/manifest) เมื่อต้องการสัญญา
  การนำไปใช้งาน

## แก้ไขปัญหาเครื่องมือที่หายไป

หากโมเดลมองไม่เห็นหรือเรียกใช้เครื่องมือไม่ได้ ให้เริ่มจากนโยบายที่มีผลสำหรับ
รอบปัจจุบัน:

1. ตรวจสอบโปรไฟล์ที่ใช้งานอยู่ `tools.allow` และ `tools.deny` ใน
   [เครื่องมือและผู้ให้บริการแบบกำหนดเอง](/th/gateway/config-tools)
2. ตรวจสอบข้อจำกัดเฉพาะผู้ให้บริการใน
   [เครื่องมือและผู้ให้บริการแบบกำหนดเอง](/th/gateway/config-tools) และยืนยันว่า
   [ผู้ให้บริการโมเดล](/th/concepts/model-providers) ที่เลือก รองรับรูปแบบ
   ของเครื่องมือ
3. ตรวจสอบสิทธิ์ของช่องทาง สถานะแซนด์บ็อกซ์ และการเข้าถึงแบบยกระดับสิทธิ์ด้วย
   [แซนด์บ็อกซ์เทียบกับนโยบายเครื่องมือเทียบกับการยกระดับสิทธิ์](/th/gateway/sandbox-vs-tool-policy-vs-elevated)
   และ [Exec แบบยกระดับสิทธิ์](/th/tools/elevated)
4. ตรวจสอบว่า Plugin ที่เป็นเจ้าของได้รับการติดตั้งและเปิดใช้งานแล้วใน
   [Plugin](/th/tools/plugin)
5. สำหรับการเรียกใช้ที่มอบหมาย ให้ตรวจสอบข้อจำกัดรายเอเจนต์ใน
   [ข้อจำกัดแซนด์บ็อกซ์และเครื่องมือรายเอเจนต์](/th/tools/multi-agent-sandbox-tools)
6. สำหรับแค็ตตาล็อก OpenClaw ขนาดใหญ่ ให้ยืนยันว่าการเรียกใช้เปิดเผยเครื่องมือโดยตรง
   ใช้ [โหมดโค้ด](/th/tools/code-mode) หรือใช้ [ค้นหาเครื่องมือ](/th/tools/tool-search)

## ที่เกี่ยวข้อง

- [ระบบอัตโนมัติ](/th/automation) สำหรับ Cron งาน Heartbeat ข้อผูกพัน ฮุก
  คำสั่งถาวร และ Task Flow
- [เอเจนต์](/th/concepts/agent) สำหรับโมเดลเอเจนต์ เซสชัน หน่วยความจำ และ
  การประสานงานแบบหลายเอเจนต์
- [เครื่องมือและผู้ให้บริการแบบกำหนดเอง](/th/gateway/config-tools) สำหรับข้อมูลอ้างอิงหลักของ
  นโยบายเครื่องมือ
- [Plugin](/th/tools/plugin) สำหรับการติดตั้งและจัดการ Plugin
- [Plugin SDK](/th/plugins/sdk-overview) สำหรับข้อมูลอ้างอิงของผู้สร้าง Plugin
- [Skills](/th/tools/skills) สำหรับลำดับการโหลดสกิล การควบคุมการเข้าถึง และการกำหนดค่า
- [เวิร์กช็อปสกิล](/th/tools/skill-workshop) สำหรับการสร้างสกิล
  ที่ผ่านการสร้างและตรวจทาน
- [ค้นหาเครื่องมือ](/th/tools/tool-search) สำหรับการค้นหาแค็ตตาล็อกเครื่องมือ OpenClaw
  แบบกระชับ
- [โหมดโค้ด](/th/tools/code-mode) สำหรับเวิร์กโฟลว์ JavaScript หรือ TypeScript แบบกระชับ
  บนแค็ตตาล็อกเครื่องมือ OpenClaw ที่ซ่อนอยู่
- [Swarm](/tools/swarm) สำหรับการกระจายงานและรวบรวมผลลัพธ์อย่างมีโครงสร้างจากโหมดโค้ด
