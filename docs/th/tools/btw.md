`/btw` (นามแฝง `/side`) ใช้ถามคำถามข้างเคียงสั้นๆ เกี่ยวกับ **เซสชัน
ปัจจุบัน** โดยไม่เพิ่มลงในประวัติการสนทนา รูปแบบนี้อ้างอิงจาก
`/btw` ของ Claude Code และปรับให้เข้ากับ Gateway และสถาปัตยกรรม
หลายช่องทางของ OpenClaw

```text
/btw มีอะไรเปลี่ยนแปลงบ้าง?
/side ข้อผิดพลาดนี้หมายความว่าอย่างไร?
```

## การทำงาน

1. บันทึกภาพรวมของเซสชันปัจจุบันเป็นบริบทเบื้องหลัง (รวมถึงพรอมต์
   ของการรันหลักที่กำลังดำเนินอยู่ หากมี)
2. เรียกใช้คำถามข้างเคียงแบบครั้งเดียวแยกต่างหาก โดยกำชับโมเดลให้ตอบเฉพาะ
   คำถามข้างเคียง และไม่กลับไปทำต่อหรือชี้นำงานหลัก
3. ส่งคำตอบเป็นผลลัพธ์ข้างเคียงแบบสด ไม่ใช่ข้อความปกติจากผู้ช่วย
4. ไม่เขียนคำถามหรือคำตอบลงในประวัติเซสชันหรือ `chat.history` โดยเด็ดขาด

หากมีการรันหลักอยู่ การรันนั้นจะไม่ถูกแตะต้อง

สำหรับเซสชันที่ใช้ชุดควบคุม Codex คำสั่ง BTW จะแยกเธรด app-server ของ Codex
ที่ใช้งานอยู่เป็นเธรดลูกชั่วคราว แทนการเรียกผู้ให้บริการแยกต่างหาก วิธีนี้
คงการทำงานของ Codex OAuth รวมถึงเครื่องมือและเธรดแบบเนทีฟไว้ และเธรดที่แยกออกมา
จะใช้นโยบายการอนุมัติ แซนด์บ็อกซ์ และพื้นผิวเครื่องมือแบบเนทีฟปัจจุบันของเธรดแม่
เธรดที่แยกออกมาจะได้รับพรอมต์กำหนดขอบเขต ซึ่งแจ้งโมเดลว่าทุกอย่างก่อนหน้า
เป็นบริบทอ้างอิงที่สืบทอดมา ไม่ใช่คำสั่งที่กำลังมีผล และมีเพียงข้อความ
หลังขอบเขตเท่านั้นที่ใช้งานอยู่ `/btw` ต้องมีเธรด Codex
อยู่แล้ว ให้ส่งข้อความปกติก่อน

สำหรับนามแฝงรันไทม์ CLI คำสั่ง BTW จะเรียกใช้แบ็กเอนด์ CLI เจ้าของในโหมด
คำถามข้างเคียงแบบครั้งเดียว โดยใส่บริบทการสนทนาที่ผ่านการกรองลงในการเรียก CLI
ครั้งใหม่ ปิดใช้งานการรวมเครื่องมือและสถานะเซสชันที่นำกลับมาใช้ใหม่ได้
พร้อมเพิ่มแฟล็กห้ามทำต่อ/ห้ามใช้เครื่องมือตามที่แบ็กเอนด์รองรับ ส่วนรันไทม์
แบบตรง (ที่ไม่ใช่ CLI) จะเรียกผู้ให้บริการโดยตรงแบบครั้งเดียวแทน

## สิ่งที่ไม่ทำ

`/btw` จะไม่สร้างเซสชันถาวร ทำงานหลักที่ยังไม่เสร็จต่อ
บันทึกข้อมูลคำถาม/คำตอบลงในประวัติทรานสคริปต์ หรือคงอยู่หลังการโหลดใหม่

## รูปแบบการส่งผลลัพธ์

แชตผู้ช่วยปกติใช้เหตุการณ์ `chat` ของ Gateway ส่วน BTW ใช้เหตุการณ์
`chat.side_result` แยกต่างหาก เพื่อให้ไคลเอนต์ไม่เข้าใจผิดว่าเป็น
ประวัติการสนทนาปกติ เนื่องจากไม่มีการเล่นซ้ำจาก `chat.history`
ผลลัพธ์จึงหายไปหลังการโหลดใหม่

## ป๊อปอัปการเลือก (Control UI)

เมื่อไฮไลต์ข้อความภายในข้อความแชตใน Control UI จะเปิด
ป๊อปอัปการเลือกขนาดเล็กพร้อมสองการทำงาน:

- **รายละเอียดเพิ่มเติม** จะส่งคำถาม `/btw` โดยนัยทันที เพื่อขอให้
  โมเดลอธิบายข้อความที่ไฮไลต์ในบริบทของเซสชันปัจจุบัน
  คำตอบจะปรากฏในแผงแชตข้างเคียงแบบลอย
- **ถามในแชตข้างเคียง** จะกรอกช่องเขียนข้อความล่วงหน้าด้วยร่าง `/btw`
  ที่ยกข้อความซึ่งไฮไลต์ไว้ เพื่อให้พิมพ์คำถามเกี่ยวกับข้อความนั้นได้เอง

ทั้งสองการทำงานเป็นไปตามความหมายปกติของ `/btw`: คำถามและคำตอบ
จะไม่เข้าสู่ประวัติเซสชัน และการรันหลักจะไม่ถูกแตะต้อง

## ควรใช้เมื่อใด

ใช้ `/btw` เมื่อต้องการคำชี้แจงสั้นๆ คำตอบข้อเท็จจริงข้างเคียง
ระหว่างที่การรันระยะยาวยังดำเนินอยู่ หรือคำตอบชั่วคราวที่ไม่ควรเข้าสู่
บริบทของเซสชันในอนาคต

```text
/btw เรากำลังแก้ไขไฟล์ใดอยู่?
/btw สรุปงานปัจจุบันเป็นหนึ่งประโยค
/btw 17 * 19 เท่ากับเท่าไร?
```

สำหรับสิ่งใดก็ตามที่ต้องการให้เป็นส่วนหนึ่งของบริบทการทำงานในอนาคต
ของเซสชัน ให้ถามตามปกติในเซสชันหลักแทน
