`openclaw agent` เรียกใช้การทำงานหนึ่งรอบของเอเจนต์จากบรรทัดคำสั่งโดยไม่มี
ข้อความแชทขาเข้า ใช้สำหรับเวิร์กโฟลว์แบบสคริปต์ การทดสอบ และ
การส่งข้อมูลด้วยโปรแกรม ข้อมูลอ้างอิงแฟล็กและลักษณะการทำงานฉบับเต็ม:
[ข้อมูลอ้างอิง Agent CLI](/th/cli/agent)

## เริ่มต้นอย่างรวดเร็ว

<Steps>
  <Step title="เรียกใช้การทำงานอย่างง่ายหนึ่งรอบของเอเจนต์">
    ```bash
    openclaw agent --agent main --message "วันนี้สภาพอากาศเป็นอย่างไร?"
    ```

    ส่งข้อความผ่าน Gateway และแสดงคำตอบ

  </Step>

  <Step title="ส่งพรอมต์หลายบรรทัดจากไฟล์">
    ```bash
    openclaw agent --agent ops --message-file ./task.md
    ```

    อ่านไฟล์ UTF-8 ที่ถูกต้องเป็นเนื้อหาข้อความของเอเจนต์

  </Step>

  <Step title="ระบุเอเจนต์หรือเซสชันที่เจาะจง">
    ```bash
    # ระบุเอเจนต์ที่เจาะจง
    openclaw agent --agent ops --message "สรุปบันทึก"

    # ระบุหมายเลขโทรศัพท์ (ใช้สร้างคีย์เซสชัน)
    openclaw agent --to +15555550123 --message "อัปเดตสถานะ"

    # ใช้เซสชันที่มีอยู่อีกครั้ง
    openclaw agent --session-id abc123 --message "ทำงานต่อ"

    # ระบุคีย์เซสชันที่แน่นอน
    openclaw agent --session-key agent:ops:incident-42 --message "สรุปสถานะ"
    ```

  </Step>

  <Step title="ส่งคำตอบไปยังช่องทาง">
    ```bash
    # ส่งไปยัง WhatsApp (ช่องทางเริ่มต้น)
    openclaw agent --to +15555550123 --message "รายงานพร้อมแล้ว" --deliver

    # ส่งไปยัง Slack
    openclaw agent --agent ops --message "สร้างรายงาน" \
      --deliver --reply-channel slack --reply-to "#reports"
    ```

  </Step>
</Steps>

## แฟล็ก

| แฟล็ก                       | คำอธิบาย                                                             |
| --------------------------- | -------------------------------------------------------------------- |
| `--message <text>`          | ข้อความแบบอินไลน์ที่จะส่ง                                            |
| `--message-file <path>`     | อ่านข้อความจากไฟล์ UTF-8 ที่ถูกต้อง (สูงสุด 4 MiB)                  |
| `--to <dest>`               | สร้างคีย์เซสชันจากเป้าหมาย (โทรศัพท์, รหัสแชท)                      |
| `--session-key <key>`       | ใช้คีย์เซสชันที่ระบุไว้อย่างชัดเจน                                  |
| `--agent <id>`              | ระบุเอเจนต์ที่กำหนดค่าไว้ (ใช้เซสชัน `main` ของเอเจนต์) |
| `--session-id <id>`         | ใช้เซสชันที่มีอยู่อีกครั้งตามรหัส                                   |
| `--model <id>`              | แทนที่โมเดลสำหรับการเรียกใช้ครั้งนี้ (`provider/model` หรือรหัสโมเดล) |
| `--local`                   | บังคับใช้รันไทม์แบบฝังภายในเครื่อง (ข้าม Gateway)                   |
| `--deliver`                 | ส่งคำตอบไปยังช่องทางแชท                                              |
| `--channel <name>`          | ช่องทางการส่ง; เมื่อใช้ร่วมกับ `--agent` + `--to` จะมีผลกับขอบเขต DM ด้วย |
| `--reply-to <target>`       | แทนที่เป้าหมายการส่ง                                                 |
| `--reply-channel <name>`    | แทนที่ช่องทางการส่ง                                                  |
| `--reply-account <id>`      | แทนที่รหัสบัญชีสำหรับการส่ง                                          |
| `--thinking <level>`        | กำหนดระดับการคิดสำหรับโปรไฟล์โมเดลที่เลือก                           |
| `--verbose <on\|full\|off>` | บันทึกระดับความละเอียดสำหรับเซสชัน (`full` จะบันทึกเอาต์พุตของเครื่องมือด้วย) |
| `--timeout <seconds>`       | แทนที่ระยะหมดเวลาของเอเจนต์ (ค่าเริ่มต้น 600 หรือค่าการกำหนดค่า)    |
| `--json`                    | แสดงผลเป็น JSON ที่มีโครงสร้าง                                      |

## ลักษณะการทำงาน

- ตามค่าเริ่มต้น CLI จะทำงาน **ผ่าน Gateway** เพิ่ม `--local` เพื่อบังคับใช้
  รันไทม์แบบฝังภายในเครื่องปัจจุบัน
- ส่งค่าเพียงหนึ่งรายการจาก `--message` หรือ `--message-file` ข้อความจากไฟล์จะคง
  เนื้อหาหลายบรรทัดไว้หลังจากนำ UTF-8 BOM ซึ่งเป็นตัวเลือกออก ไฟล์ที่มีขนาดใหญ่กว่า
  4 MiB จะถูกปฏิเสธก่อนส่งต่อ
- หลังจากลองจับมือเชื่อมต่อใหม่เนื่องจากข้อผิดพลาดชั่วคราว หาก Gateway หมดเวลาหรือการเชื่อมต่อปิดลง
  คำสั่งจะล้มเหลวพร้อมคำแนะนำใน stderr โดย CLI จะไม่เรียกใช้รอบการทำงานซ้ำ
  ด้วยรันไทม์แบบฝังโดยไม่แจ้งให้ทราบ Gateway อาจยังคงทำงานรอบที่ยอมรับไว้จนเสร็จ ดังนั้นให้ตรวจสอบสถานะของ Gateway
  และเซสชันก่อนลองอีกครั้งหรือเรียกใช้ซ้ำด้วย `--local`
- การเลือกเซสชัน: `--to` สร้างคีย์เซสชัน (เป้าหมายกลุ่ม/ช่องทาง
  จะรักษาการแยกจากกัน ส่วนแชทโดยตรงจะรวมเป็น `main`) เมื่อใช้ `--agent`,
  `--channel` และ `--to` ร่วมกัน การกำหนดเส้นทางจะเป็นไปตามผู้รับมาตรฐานของช่องทาง
  และ `session.dmScope` ข้อมูลประจำตัวที่เสถียรสำหรับขาออกเท่านั้นจะใช้เซสชัน
  ที่ผู้ให้บริการเป็นเจ้าของและแยกจากเซสชันหลักของเอเจนต์
- `--session-key` เลือกคีย์ที่ระบุไว้อย่างชัดเจน คีย์ที่ขึ้นต้นด้วยเอเจนต์ต้องใช้
  `agent:<agent-id>:<session-key>` และ `--agent` ต้องตรงกับรหัสเอเจนต์นั้นเมื่อ
  ระบุทั้งสองค่า คีย์เปล่าที่ไม่ใช่ค่า sentinel จะถูกกำหนดขอบเขตไว้ที่ `--agent` เมื่อ
  ระบุไว้ ตัวอย่างเช่น `--agent ops --session-key incident-42` จะกำหนดเส้นทางไปยัง
  `agent:ops:incident-42` หากไม่มี `--agent` คีย์เปล่าที่ไม่ใช่ค่า sentinel จะถูกกำหนดขอบเขต
  ไว้ที่เอเจนต์เริ่มต้นที่กำหนดค่าไว้ ค่าตรงตัว `global` และ `unknown` จะยังคง
  ไม่มีขอบเขตเฉพาะเมื่อไม่ได้ระบุ `--agent` เท่านั้น
- `--reply-channel` และ `--reply-account` มีผลเฉพาะกับการส่ง
- แฟล็กระดับการคิดและความละเอียดจะถูกบันทึกไว้ในที่จัดเก็บเซสชัน
- เอาต์พุต: ค่าเริ่มต้นเป็นข้อความธรรมดา หรือใช้ `--json` สำหรับเพย์โหลดที่มีโครงสร้างและเมทาดาทา
- เมื่อใช้ `--json --deliver` JSON จะมีสถานะการส่งสำหรับการส่งที่สำเร็จ
  ถูกระงับ สำเร็จบางส่วน และล้มเหลว ดู
  [สถานะการส่งใน JSON](/th/cli/agent#json-delivery-status)

## ตัวอย่าง

```bash
# การทำงานอย่างง่ายหนึ่งรอบพร้อมเอาต์พุต JSON
openclaw agent --to +15555550123 --message "ติดตามบันทึก" --verbose on --json

# การทำงานหนึ่งรอบพร้อมแทนที่โมเดล
openclaw agent --agent ops --model openai/gpt-5.4 --message "สรุปบันทึก"

# การทำงานหนึ่งรอบพร้อมระดับการคิด
openclaw agent --session-id 1234 --message "สรุปกล่องจดหมาย" --thinking medium

# พรอมต์หลายบรรทัดจากไฟล์
openclaw agent --agent ops --message-file ./task.md

# คีย์เซสชันที่แน่นอน
openclaw agent --session-key agent:ops:incident-42 --message "สรุปสถานะ"

# คีย์แบบเดิมที่กำหนดขอบเขตไว้ที่เอเจนต์
openclaw agent --agent ops --session-key incident-42 --message "สรุปสถานะ"

# ส่งไปยังช่องทางอื่นที่ไม่ใช่ช่องทางของเซสชัน
openclaw agent --agent ops --message "แจ้งเตือน" --deliver --reply-channel telegram --reply-to "@admin"
```
