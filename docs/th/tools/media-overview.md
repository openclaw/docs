OpenClaw สร้างรูปภาพ วิดีโอ และเพลง เข้าใจสื่อขาเข้า
(รูปภาพ เสียง วิดีโอ) และอ่านคำตอบออกเสียงด้วย text-to-speech ความสามารถด้าน
สื่อทั้งหมดขับเคลื่อนด้วยเครื่องมือ: เอเจนต์จะตัดสินใจว่าจะใช้เมื่อใดตาม
บทสนทนา และแต่ละเครื่องมือจะปรากฏเฉพาะเมื่อมีการกำหนดค่าผู้ให้บริการที่รองรับ
อย่างน้อยหนึ่งรายเท่านั้น

## อะซิงโครนัสเทียบกับซิงโครนัส

| ความสามารถ      | โหมด         | เหตุผล                                                                |
| --------------- | ------------ | ------------------------------------------------------------------ |
| รูปภาพ           | ซิงโครนัส  | การตอบกลับของผู้ให้บริการส่งคืนภายในไม่กี่วินาที เสร็จสมบูรณ์ในบรรทัดเดียวกับคำตอบ |
| Text-to-speech  | ซิงโครนัส  | การตอบกลับของผู้ให้บริการส่งคืนภายในไม่กี่วินาที แนบไปกับเสียงของคำตอบ |
| วิดีโอ           | อะซิงโครนัส | การประมวลผลของผู้ให้บริการใช้เวลา 30 วินาทีถึงหลายนาที                 |
| เพลง (ใช้ร่วมกัน)  | อะซิงโครนัส | มีลักษณะการประมวลผลโดยผู้ให้บริการเหมือนกับวิดีโอ                  |
| เพลง (ComfyUI) | ซิงโครนัส  | เวิร์กโฟลว์ภายในเครื่องทำงานในบรรทัดเดียวกับเซิร์ฟเวอร์ ComfyUI ที่กำหนดค่าไว้  |

สำหรับเครื่องมือแบบอะซิงโครนัส OpenClaw จะส่งคำขอไปยังผู้ให้บริการ ส่งคืน
id งานทันที และติดตามงานในสมุดบัญชีงาน เอเจนต์จะตอบกลับข้อความอื่นต่อไป
ระหว่างที่งานทำงาน เมื่อผู้ให้บริการทำงานเสร็จ OpenClaw จะปลุกเอเจนต์เพื่อให้
โพสต์สื่อที่เสร็จแล้วกลับไปยังช่องทางเดิมได้

## Speech-to-text และ Voice Call

Deepgram, DeepInfra, ElevenLabs, Mistral, OpenAI, SenseAudio และ xAI สามารถถอดเสียง
เสียงขาเข้าทั้งหมดผ่านพาธ `tools.media.audio` แบบแบตช์เมื่อมีการกำหนดค่า
Plugin ช่องทางที่ตรวจสอบข้อความเสียงล่วงหน้าเพื่อ gating การกล่าวถึงหรือการแยกวิเคราะห์
คำสั่ง จะทำเครื่องหมายไฟล์แนบที่ถอดเสียงแล้วในบริบทขาเข้า ดังนั้นรอบการทำความเข้าใจ
สื่อที่ใช้ร่วมกันจะนำทรานสคริปต์นั้นกลับมาใช้แทนการเรียก STT ครั้งที่สองสำหรับเสียงเดียวกัน

Deepgram, ElevenLabs, Mistral, OpenAI และ xAI ยังลงทะเบียนผู้ให้บริการ
STT สตรีมมิงสำหรับ Voice Call ด้วย ดังนั้นเสียงโทรศัพท์สดจึงถูกส่งต่อไปยัง
ผู้ให้บริการที่เลือกได้โดยไม่ต้องรอการบันทึกที่เสร็จสมบูรณ์

    การทำความเข้าใจสื่อ
  </Accordion>
  <Accordion title="OpenAI">

    และ memory-embedding
  </Accordion>
  <Accordion title="DeepInfra">
    พื้นผิวการกำหนดเส้นทางแชต/โมเดล การสร้าง/แก้ไขรูปภาพ ข้อความเป็นวิดีโอ TTS แบบแบตช์
    STT แบบแบตช์ การทำความเข้าใจสื่อรูปภาพ และ memory-embedding
    โมเดล rerank/classification/object-detection แบบเนทีฟของ DeepInfra จะยังไม่
    ลงทะเบียนจนกว่า OpenClaw จะมีสัญญาผู้ให้บริการเฉพาะสำหรับหมวดหมู่เหล่านั้น
  </Accordion>
  <Accordion title="xAI">

    Call เสียงเรียลไทม์ของ xAI เป็นความสามารถจากต้นน้ำ แต่ยังไม่
    ลงทะเบียนใน OpenClaw จนกว่าสัญญาเสียงเรียลไทม์ที่ใช้ร่วมกันจะสามารถ
    แสดงความสามารถนั้นได้
  </Accordion>
