OpenClaw สร้างรูปภาพ วิดีโอ และเพลง เข้าใจสื่อขาเข้า
(รูปภาพ เสียง วิดีโอ) และพูดคำตอบออกเสียงด้วยการแปลงข้อความเป็นเสียงพูด ความสามารถด้าน
สื่อทั้งหมดขับเคลื่อนด้วยเครื่องมือ: เอเจนต์จะตัดสินใจว่าจะใช้เมื่อใดตาม
บทสนทนา และแต่ละเครื่องมือจะปรากฏเฉพาะเมื่อมีการกำหนดค่า
ผู้ให้บริการรองรับอย่างน้อยหนึ่งราย

## อะซิงโครนัสเทียบกับซิงโครนัส

| ความสามารถ | โหมด | เหตุผล |
| --------------- | ------------ | ---------------------------------------------------------------------------------------------------- |
| รูปภาพ | ซิงโครนัส | การตอบสนองของผู้ให้บริการส่งกลับภายในไม่กี่วินาที; เสร็จสิ้นในบรรทัดเดียวกับคำตอบ |
| การแปลงข้อความเป็นเสียงพูด | ซิงโครนัส | การตอบสนองของผู้ให้บริการส่งกลับภายในไม่กี่วินาที; แนบไปกับเสียงของคำตอบ |
| วิดีโอ | อะซิงโครนัส | การประมวลผลของผู้ให้บริการใช้เวลา 30 วินาทีถึงหลายหลายนาที; คิวที่ช้าอาจทำงานจนถึงค่าหมดเวลาที่กำหนดไว้ |
| เพลง (ใช้ร่วมกัน) | อะซิงโครนัส | มีลักษณะการประมวลผลโดยผู้ให้บริการเหมือนกับวิดีโอ |
| เพลง (ComfyUI) | ซิงโครนัส | เวิร์กโฟลว์ในเครื่องทำงานในบรรทัดเดียวกับเซิร์ฟเวอร์ ComfyUI ที่กำหนดค่าไว้ |

สำหรับเครื่องมือแบบอะซิงโครนัส OpenClaw จะส่งคำขอไปยังผู้ให้บริการ ส่งคืน id
งานทันที และติดตามงานในบัญชีแยกประเภงาน เอเจนต์ยังคง
ตอบกลับข้อความอื่น ๆ ระหว่างที่งานกำลังทำงาน เมื่อผู้ให้บริการดำเนินการเสร็จ
OpenClaw จะปลุกเอเจนต์พร้อมพาธสื่อที่สร้างขึ้นเพื่อให้เอเจนต์บอก
ผู้ใช้ได้ และเมื่อจำเป็นตามนโยบายการส่งมอบจากต้นทาง จะส่งต่อผลลัพธ์ผ่าน
เครื่องมือข้อความ สำหรับเส้นทางกลุ่ม/ช่องทางที่ใช้เครื่องมือข้อความเท่านั้น OpenClaw ถือว่า
หลักฐานการส่งมอบด้วยเครื่องมือข้อความที่หายไปเป็นความพยายามเสร็จสิ้นที่ล้มเหลว และส่ง
สื่อสำรองที่สร้างขึ้นโดยตรงไปยังช่องทางเดิม

## การแปลงเสียงพูดเป็นข้อความและการโทรด้วยเสียง

Deepgram, DeepInfra, ElevenLabs, Mistral, OpenAI, SenseAudio และ xAI สามารถถอดความ
เสียงขาเข้าผ่านเส้นทาง `tools.media.audio` แบบแบตช์ได้ทั้งหมดเมื่อกำหนดค่าแล้ว
Plugin ช่องทางที่ตรวจสอบข้อความเสียงล่วงหน้าเพื่อการกั้นด้วยการกล่าวถึงหรือการแยกวิเคราะห์
คำสั่งจะทำเครื่องหมายไฟล์แนบที่ถอดความแล้วบนบริบทขาเข้า ดังนั้นรอบการทำความเข้าใจ
สื่อแบบใช้ร่วมกันจะใช้ทรานสคริปต์นั้นซ้ำแทนการเรียก
STT ครั้งที่สองสำหรับเสียงเดียวกัน

Deepgram, ElevenLabs, Mistral, OpenAI และ xAI ยังลงทะเบียนผู้ให้บริการ STT
แบบสตรีมมิงสำหรับการโทรด้วยเสียงด้วย ดังนั้นเสียงโทรศัพท์สดจึงสามารถส่งต่อไปยังผู้ให้บริการ
ที่เลือกได้โดยไม่ต้องรอให้การบันทึกเสร็จสมบูรณ์
