OpenClaw สร้างรูปภาพ วิดีโอ และเพลง เข้าใจสื่อขาเข้า
(รูปภาพ เสียง วิดีโอ) และพูดคำตอบออกเสียงด้วยข้อความเป็นเสียง ความสามารถด้าน
สื่อทั้งหมดขับเคลื่อนด้วยเครื่องมือ: เอเจนต์ตัดสินใจว่าจะใช้เมื่อใดตาม
บทสนทนา และแต่ละเครื่องมือจะปรากฏเฉพาะเมื่อมี Provider รองรับอย่างน้อยหนึ่งราย
ที่กำหนดค่าไว้

## อะซิงโครนัสเทียบกับซิงโครนัส

| ความสามารถ      | โหมด         | เหตุผล                                                                |
| --------------- | ------------ | ------------------------------------------------------------------ |
| รูปภาพ           | ซิงโครนัส  | คำตอบจาก Provider ส่งกลับในไม่กี่วินาที; เสร็จสมบูรณ์ในบรรทัดเดียวกับคำตอบ |
| ข้อความเป็นเสียง  | ซิงโครนัส  | คำตอบจาก Provider ส่งกลับในไม่กี่วินาที; แนบไปกับเสียงของคำตอบ |
| วิดีโอ           | อะซิงโครนัส | การประมวลผลของ Provider ใช้เวลา 30 วินาทีถึงหลายนาที                 |
| เพลง (ใช้ร่วมกัน)  | อะซิงโครนัส | มีลักษณะการประมวลผลโดย Provider เหมือนกับวิดีโอ                  |
| เพลง (ComfyUI) | ซิงโครนัส  | เวิร์กโฟลว์ในเครื่องทำงานในบรรทัดเดียวกับเซิร์ฟเวอร์ ComfyUI ที่กำหนดค่าไว้  |

สำหรับเครื่องมือแบบอะซิงโครนัส OpenClaw จะส่งคำขอไปยัง Provider ส่งคืนรหัสงาน
ทันที และติดตามงานในบัญชีแยกประเภทของงาน เอเจนต์จะตอบกลับข้อความอื่นต่อไป
ระหว่างที่งานทำงาน เมื่อ Provider ทำงานเสร็จ OpenClaw จะปลุกเอเจนต์พร้อมเส้นทาง
สื่อที่สร้างขึ้น เพื่อให้เอเจนต์บอกผู้ใช้ได้ และเมื่อจำเป็นตามนโยบายการส่งมอบ
จากแหล่งที่มา จะส่งต่อผลลัพธ์ผ่านเครื่องมือข้อความ

## เสียงเป็นข้อความและ Voice Call

Deepgram, DeepInfra, ElevenLabs, Mistral, OpenAI, SenseAudio และ xAI ทั้งหมดสามารถถอดเสียง
เสียงขาเข้าผ่านเส้นทาง `tools.media.audio` แบบแบตช์เมื่อกำหนดค่าไว้
Plugin ช่องทางที่ตรวจสอบข้อความเสียงล่วงหน้าสำหรับการควบคุมการกล่าวถึงหรือการแยกวิเคราะห์
คำสั่ง จะทำเครื่องหมายไฟล์แนบที่ถอดเสียงแล้วบนบริบทขาเข้า ดังนั้นรอบ
ความเข้าใจสื่อแบบใช้ร่วมกันจึงนำทรานสคริปต์นั้นกลับมาใช้แทนการเรียก
STT ครั้งที่สองสำหรับเสียงเดียวกัน

Deepgram, ElevenLabs, Mistral, OpenAI และ xAI ยังลงทะเบียน Provider STT แบบสตรีมมิง
ของ Voice Call ด้วย ดังนั้นเสียงโทรศัพท์สดจึงสามารถส่งต่อไปยังผู้ให้บริการ
ที่เลือกได้โดยไม่ต้องรอให้การบันทึกเสร็จสมบูรณ์

## การแมป Provider (วิธีที่ผู้ให้บริการแยกไปตามพื้นผิวต่างๆ)

<AccordionGroup>
  <Accordion title="Google">
    พื้นผิวรูปภาพ วิดีโอ เพลง TTS แบบแบตช์ เสียงแบบเรียลไทม์ฝั่งแบ็กเอนด์ และ
    ความเข้าใจสื่อ
  </Accordion>
  <Accordion title="OpenAI">
    พื้นผิวรูปภาพ วิดีโอ TTS แบบแบตช์ STT แบบแบตช์ STT แบบสตรีมมิงของ Voice Call เสียงแบบเรียลไทม์ฝั่งแบ็กเอนด์
    และการฝังหน่วยความจำ
  </Accordion>
  <Accordion title="DeepInfra">
    พื้นผิวการแชต/การกำหนดเส้นทางโมเดล การสร้าง/แก้ไขรูปภาพ ข้อความเป็นวิดีโอ TTS แบบแบตช์
    STT แบบแบตช์ ความเข้าใจสื่อรูปภาพ และการฝังหน่วยความจำ
    โมเดล rerank/classification/object-detection แบบเนทีฟของ DeepInfra จะยังไม่
    ลงทะเบียนจนกว่า OpenClaw จะมีสัญญา Provider เฉพาะสำหรับหมวดหมู่เหล่านั้น
  </Accordion>
  <Accordion title="xAI">
    รูปภาพ วิดีโอ การค้นหา การรันโค้ด TTS แบบแบตช์ STT แบบแบตช์ และ STT แบบสตรีมมิงของ Voice
    Call เสียง xAI Realtime เป็นความสามารถต้นทาง แต่ยังไม่
    ลงทะเบียนใน OpenClaw จนกว่าสัญญาเสียงแบบเรียลไทม์ที่ใช้ร่วมกันจะสามารถ
    แทนความสามารถนี้ได้
  </Accordion>
</AccordionGroup>
