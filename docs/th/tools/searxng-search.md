OpenClaw รองรับ [SearXNG](https://docs.searxng.org/) ในฐานะผู้ให้บริการ `web_search` แบบ **โฮสต์ด้วยตนเอง
และไม่ใช้คีย์** SearXNG เป็นเครื่องมือค้นหาแบบเมตาโอเพนซอร์ส
ที่รวบรวมผลลัพธ์จาก Google, Bing, DuckDuckGo และแหล่งอื่น ๆ

ข้อดี:

- **ฟรีและไม่จำกัด** -- ไม่ต้องใช้คีย์ API หรือสมัครสมาชิกเชิงพาณิชย์
- **ความเป็นส่วนตัว / แยกขาดจากเครือข่ายภายนอก** -- คำค้นหาจะไม่ออกจากเครือข่ายของคุณ
- **ใช้งานได้ทุกที่** -- ไม่มีข้อจำกัดด้านภูมิภาคของ API การค้นหาเชิงพาณิชย์

## การตั้งค่า

<Steps>
  <Step title="ติดตั้ง Plugin">
    ```bash
    openclaw plugins install @openclaw/searxng-plugin
    ```
  </Step>
  <Step title="เรียกใช้อินสแตนซ์ SearXNG">
    ```bash
    docker run -d -p 8888:8080 searxng/searxng
    ```

    หรือใช้การติดตั้งใช้งาน SearXNG ที่มีอยู่และคุณสามารถเข้าถึงได้ โปรดดู
    [เอกสาร SearXNG](https://docs.searxng.org/) สำหรับการตั้งค่าเพื่อใช้งานจริง

  </Step>
  <Step title="กำหนดค่า">
    ```bash
    openclaw configure --section web
    # Select "searxng" as the provider
    ```

    หรือตั้งค่าตัวแปรสภาพแวดล้อมและให้การตรวจหาอัตโนมัติค้นพบ:

    ```bash
    export SEARXNG_BASE_URL="http://localhost:8888"
    ```

  </Step>
</Steps>

## การกำหนดค่า

```json5
{
  tools: {
    web: {
      search: {
        provider: "searxng",
      },
    },
  },
}
```

การตั้งค่าระดับ Plugin สำหรับอินสแตนซ์ SearXNG:

```json5
{
  plugins: {
    entries: {
      searxng: {
        config: {
          webSearch: {
            baseUrl: "http://localhost:8888",
            categories: "general,news", // optional
            language: "en", // optional
          },
        },
      },
    },
  },
}
```

`baseUrl` ยอมรับออบเจ็กต์ SecretRef เช่นกัน (ตัวอย่างเช่น `{ source: "env", id: "SEARXNG_BASE_URL" }`)

## ตัวแปรสภาพแวดล้อม

ตั้งค่า `SEARXNG_BASE_URL` เพื่อใช้แทนการกำหนดค่า:

```bash
export SEARXNG_BASE_URL="http://localhost:8888"
```

ลำดับการเลือกใช้: สตริง `baseUrl` ที่กำหนดค่าไว้ ตามด้วย SecretRef ของตัวแปรสภาพแวดล้อมแบบอินไลน์บน
`baseUrl` แล้วจึง `SEARXNG_BASE_URL` เมื่อไม่ได้ตั้งค่าเส้นทางการกำหนดค่าใด ๆ และ
มี `SEARXNG_BASE_URL` โดยไม่ได้เลือกผู้ให้บริการไว้อย่างชัดเจน การตรวจหาอัตโนมัติ
จะเลือก SearXNG

## ข้อมูลอ้างอิงการกำหนดค่า Plugin

| ฟิลด์        | คำอธิบาย                                                        |
| ------------ | ------------------------------------------------------------------ |
| `baseUrl`    | URL ฐานของอินสแตนซ์ SearXNG ของคุณ (จำเป็น)                       |
| `categories` | หมวดหมู่ที่คั่นด้วยจุลภาค เช่น `general`, `news` หรือ `science` |
| `language`   | รหัสภาษาสำหรับผลลัพธ์ เช่น `en`, `de` หรือ `fr`              |

การเรียกใช้เครื่องมือ `web_search` ยังยอมรับ `count` (ผลลัพธ์ 1-10 รายการ), `categories`
และ `language` เพื่อใช้แทนค่าที่กำหนดไว้สำหรับการเรียกใช้แต่ละครั้ง

## หมายเหตุ

- **JSON API** -- ใช้ปลายทาง `format=json` แบบเนทีฟของ SearXNG ไม่ใช่การดึงข้อมูลจาก HTML
- **URL ของผลลัพธ์รูปภาพ** -- ผลลัพธ์ในหมวดหมู่รูปภาพจะมี `img_src` เมื่อ SearXNG
  ส่งคืน URL รูปภาพโดยตรง
- **ไม่ใช้คีย์ API** -- ใช้งานกับอินสแตนซ์ SearXNG ใดก็ได้ทันที
- **การตรวจสอบ URL ฐาน** -- `baseUrl` ต้องเป็น URL `http://` หรือ `https://`
  ที่ถูกต้อง
- **การป้องกันเครือข่าย** -- URL ฐานแบบ `http://` ต้องชี้ไปยังโฮสต์ส่วนตัวที่เชื่อถือได้หรือ
  โฮสต์ local loopback (โฮสต์สาธารณะต้องใช้ `https://`); URL ฐานแบบ `https://` ที่
  แปลงชื่อไปยังที่อยู่ส่วนตัว/ภายในจะได้รับอนุญาตสำหรับการโฮสต์ด้วยตนเองเช่นเดียวกัน
  ขณะที่ URL ฐานแบบ `https://` ที่แปลงชื่อไปยังที่อยู่สาธารณะยังคงใช้การป้องกัน SSRF อย่างเข้มงวด
- **ลำดับการตรวจหาอัตโนมัติ** -- SearXNG ต้องมี `baseUrl` ที่กำหนดค่าไว้ (ลำดับ
  200 ในบรรดาผู้ให้บริการที่มีข้อมูลรับรองที่จำเป็นแล้ว) ผู้ให้บริการที่ไม่ใช้คีย์
  เช่น DuckDuckGo หรือ Ollama Web Search จะไม่ถูกเลือกโดยการตรวจหาอัตโนมัติโดยปริยาย
  แต่จะทำงานเมื่อเลือก `provider` อย่างชัดเจนเท่านั้น
- **โฮสต์ด้วยตนเอง** -- คุณควบคุมอินสแตนซ์ คำค้นหา และเครื่องมือค้นหาต้นทาง
- **หมวดหมู่** จะมีค่าเริ่มต้นเป็น `general` เมื่อไม่ได้กำหนดค่า
- **การใช้หมวดหมู่สำรอง** -- หากคำขอหมวดหมู่ที่ไม่ใช่ `general` สำเร็จแต่
  ส่งคืนผลลัพธ์เป็นศูนย์ OpenClaw จะลองคำค้นหาเดิมอีกครั้งหนึ่งด้วย `general`
  ก่อนส่งคืนชุดผลลัพธ์ว่าง
- **การแคชผลลัพธ์** -- คำค้นหาที่เหมือนกัน (คำค้นหา จำนวน หมวดหมู่
  ภาษา และ URL ฐานเดียวกัน) จะถูกแคชภายในกระบวนการเป็นเวลาสั้น ๆ ตาม TTL
- **ข้อกำหนดด้านเวอร์ชัน** -- Plugin ประกาศ `minHostVersion: >=2026.6.9`

<Tip>
  เพื่อให้ JSON API ของ SearXNG ทำงานได้ โปรดตรวจสอบว่าอินสแตนซ์ SearXNG ของคุณเปิดใช้งานรูปแบบ `json`
  ใน `settings.yml` ภายใต้ `search.formats`
</Tip>
