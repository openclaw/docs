OpenClaw สามารถป้องกันไม่ให้เอเจนต์ติดอยู่ในรูปแบบการเรียกเครื่องมือซ้ำ ๆ ได้
การป้องกันนี้ **ปิดใช้งานเป็นค่าเริ่มต้น**

เปิดใช้เฉพาะในจุดที่จำเป็น เพราะการตั้งค่าที่เข้มงวดอาจบล็อกการเรียกซ้ำที่ถูกต้องได้

## ทำไมจึงมีสิ่งนี้

- ตรวจจับลำดับซ้ำ ๆ ที่ไม่ทำให้เกิดความคืบหน้า
- ตรวจจับลูปที่ไม่มีผลลัพธ์ด้วยความถี่สูง (เครื่องมือเดียวกัน อินพุตเดียวกัน ข้อผิดพลาดซ้ำ)
- ตรวจจับรูปแบบการเรียกซ้ำเฉพาะสำหรับเครื่องมือโพลลิงที่รู้จัก

## บล็อกการกำหนดค่า

ค่าเริ่มต้นแบบส่วนกลาง:

```json5
{
  tools: {
    loopDetection: {
      enabled: false,
      historySize: 30,
      warningThreshold: 10,
      criticalThreshold: 20,
      globalCircuitBreakerThreshold: 30,
      detectors: {
        genericRepeat: true,
        knownPollNoProgress: true,
        pingPong: true,
      },
    },
  },
}
```

การแทนที่รายเอเจนต์ (ไม่บังคับ):

```json5
{
  agents: {
    list: [
      {
        id: "safe-runner",
        tools: {
          loopDetection: {
            enabled: true,
            warningThreshold: 8,
            criticalThreshold: 16,
          },
        },
      },
    ],
  },
}
```

### ลักษณะการทำงานของฟิลด์

- `enabled`: สวิตช์หลัก `false` หมายความว่าจะไม่มีการตรวจจับลูป
- `historySize`: จำนวนการเรียกเครื่องมือล่าสุดที่เก็บไว้สำหรับการวิเคราะห์
- `warningThreshold`: เกณฑ์ก่อนจัดประเภทรูปแบบเป็นระดับเตือนเท่านั้น
- `criticalThreshold`: เกณฑ์สำหรับบล็อกรูปแบบลูปซ้ำ
- `globalCircuitBreakerThreshold`: เกณฑ์ตัวตัดวงจรแบบส่วนกลางเมื่อไม่มีความคืบหน้า
- `detectors.genericRepeat`: ตรวจจับรูปแบบเครื่องมือเดียวกัน + พารามิเตอร์เดียวกันซ้ำ
- `detectors.knownPollNoProgress`: ตรวจจับรูปแบบที่คล้ายการโพลลิงที่รู้จักซึ่งไม่มีการเปลี่ยนแปลงสถานะ
- `detectors.pingPong`: ตรวจจับรูปแบบ ping-pong สลับไปมา

สำหรับ `exec` การตรวจสอบว่าไม่มีความคืบหน้าจะเปรียบเทียบผลลัพธ์คำสั่งที่คงที่ และละเว้นเมตาดาตารันไทม์ที่เปลี่ยนแปลงง่าย เช่น ระยะเวลา PID, session ID และไดเรกทอรีทำงาน
เมื่อมี run id ประวัติการเรียกเครื่องมือล่าสุดจะถูกประเมินเฉพาะภายในรันนั้น เพื่อให้รอบ Heartbeat ตามกำหนดเวลาและรันใหม่ไม่สืบทอดจำนวนลูปที่ค้างจากรันก่อนหน้า

## การตั้งค่าที่แนะนำ

- สำหรับโมเดลขนาดเล็ก ให้เริ่มด้วย `enabled: true` และไม่เปลี่ยนค่าเริ่มต้น โมเดลระดับเรือธงแทบไม่จำเป็นต้องใช้การตรวจจับลูป และสามารถปล่อยให้ปิดใช้งานไว้ได้
- รักษาลำดับเกณฑ์เป็น `warningThreshold < criticalThreshold < globalCircuitBreakerThreshold`
- หากเกิดผลบวกลวง:
  - เพิ่ม `warningThreshold` และ/หรือ `criticalThreshold`
  - (ไม่บังคับ) เพิ่ม `globalCircuitBreakerThreshold`
  - ปิดใช้งานเฉพาะตัวตรวจจับที่ทำให้เกิดปัญหา
  - ลด `historySize` เพื่อให้บริบทประวัติเข้มงวดน้อยลง

## บันทึกและลักษณะการทำงานที่คาดไว้

เมื่อตรวจพบลูป OpenClaw จะรายงานเหตุการณ์ลูป และบล็อกหรือลดความแรงของรอบเครื่องมือถัดไปตามระดับความรุนแรง
สิ่งนี้ช่วยปกป้องผู้ใช้จากการใช้โทเคนเกินควบคุมและการค้าง ขณะยังคงรักษาการเข้าถึงเครื่องมือตามปกติไว้

- ควรใช้การเตือนและการระงับชั่วคราวก่อน
- ยกระดับเฉพาะเมื่อหลักฐานซ้ำสะสมมากขึ้น

## หมายเหตุ

- `tools.loopDetection` จะถูกรวมกับการแทนที่ระดับเอเจนต์
- การกำหนดค่ารายเอเจนต์จะแทนที่หรือขยายค่าทั่วโลกทั้งหมด
- หากไม่มีการกำหนดค่า guardrails จะยังคงปิดอยู่
