OpenClaw สามารถป้องกันไม่ให้เอเจนต์ติดอยู่ในรูปแบบการเรียกใช้เครื่องมือซ้ำ ๆ ได้
กลไกป้องกันนี้**ปิดใช้เป็นค่าเริ่มต้น**

เปิดใช้เฉพาะในจุดที่จำเป็น เพราะการตั้งค่าที่เข้มงวดอาจบล็อกการเรียกใช้ซ้ำที่ถูกต้องได้

## เหตุผลที่มีสิ่งนี้

- ตรวจจับลำดับซ้ำ ๆ ที่ไม่เกิดความคืบหน้า
- ตรวจจับลูปความถี่สูงที่ไม่มีผลลัพธ์ (เครื่องมือเดียวกัน อินพุตเดียวกัน ข้อผิดพลาดซ้ำ ๆ)
- ตรวจจับรูปแบบการเรียกใช้ซ้ำเฉพาะสำหรับเครื่องมือ polling ที่รู้จัก

## บล็อกการกำหนดค่า

ค่าเริ่มต้นส่วนกลาง:

```json5
{
  tools: {
    loopDetection: {
      enabled: false,
      historySize: 30,
      warningThreshold: 10,
      criticalThreshold: 20,
      globalCircuitBreakerThreshold: 30,
      detectors: {
        genericRepeat: true,
        knownPollNoProgress: true,
        pingPong: true,
      },
    },
  },
}
```

การแทนที่ต่อเอเจนต์ (ไม่บังคับ):

```json5
{
  agents: {
    list: [
      {
        id: "safe-runner",
        tools: {
          loopDetection: {
            enabled: true,
            warningThreshold: 8,
            criticalThreshold: 16,
          },
        },
      },
    ],
  },
}
```

### พฤติกรรมของฟิลด์

- `enabled`: สวิตช์หลัก `false` หมายความว่าจะไม่มีการตรวจจับลูป
- `historySize`: จำนวนการเรียกใช้เครื่องมือล่าสุดที่เก็บไว้สำหรับการวิเคราะห์
- `warningThreshold`: เกณฑ์ก่อนจัดประเภทให้รูปแบบเป็นเพียงคำเตือนเท่านั้น
- `criticalThreshold`: เกณฑ์สำหรับบล็อกรูปแบบลูปซ้ำ ๆ
- `globalCircuitBreakerThreshold`: เกณฑ์ตัวตัดวงจรแบบไม่มีความคืบหน้าส่วนกลาง
- `detectors.genericRepeat`: ตรวจจับรูปแบบเครื่องมือเดิม + พารามิเตอร์เดิมซ้ำ ๆ
- `detectors.knownPollNoProgress`: ตรวจจับรูปแบบคล้าย polling ที่รู้จักซึ่งไม่มีการเปลี่ยนแปลงสถานะ
- `detectors.pingPong`: ตรวจจับรูปแบบ ping-pong ที่สลับไปมา

สำหรับ `exec` การตรวจสอบแบบไม่มีความคืบหน้าจะเปรียบเทียบผลลัพธ์คำสั่งที่เสถียร และละเว้นเมตาดาต้ารันไทม์ที่ผันผวน เช่น ระยะเวลา, PID, ID ของเซสชัน และไดเรกทอรีทำงาน
เมื่อมี ID การรัน ระบบจะประเมินประวัติการเรียกใช้เครื่องมือล่าสุดเฉพาะภายในการรันนั้น เพื่อให้รอบ Heartbeat ที่ตั้งเวลาไว้และการรันใหม่ไม่สืบทอดจำนวนลูปค้างจากการรันก่อนหน้า

## การตั้งค่าที่แนะนำ

- เริ่มด้วย `enabled: true` โดยไม่เปลี่ยนค่าเริ่มต้น
- รักษาลำดับเกณฑ์เป็น `warningThreshold < criticalThreshold < globalCircuitBreakerThreshold`
- หากเกิดผลบวกเทียม:
  - เพิ่ม `warningThreshold` และ/หรือ `criticalThreshold`
  - เพิ่ม `globalCircuitBreakerThreshold` (ไม่บังคับ)
  - ปิดใช้เฉพาะตัวตรวจจับที่ทำให้เกิดปัญหา
  - ลด `historySize` เพื่อให้บริบทประวัติเข้มงวดน้อยลง

## บันทึกและพฤติกรรมที่คาดหวัง

เมื่อมีการตรวจพบลูป OpenClaw จะรายงานเหตุการณ์ลูป และบล็อกหรือลดความถี่รอบเครื่องมือถัดไปตามระดับความรุนแรง
สิ่งนี้ช่วยปกป้องผู้ใช้จากการใช้โทเคนเกินควบคุมและการค้าง ขณะยังคงการเข้าถึงเครื่องมือตามปกติไว้

- ควรใช้คำเตือนและการระงับชั่วคราวก่อน
- ยกระดับเฉพาะเมื่อมีหลักฐานซ้ำ ๆ สะสมขึ้น

## หมายเหตุ

- `tools.loopDetection` จะถูกรวมกับการแทนที่ระดับเอเจนต์
- การกำหนดค่าต่อเอเจนต์จะแทนที่หรือขยายค่าระดับส่วนกลางอย่างสมบูรณ์
- หากไม่มีการกำหนดค่า กลไกป้องกันจะยังคงปิดอยู่
