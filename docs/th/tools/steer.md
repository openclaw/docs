`/steer` จะพยายามส่งคำแนะนำไปยังการทำงานที่กำลังดำเนินอยู่ก่อน คำสั่งนี้ใช้สำหรับ
ช่วงเวลาที่ต้องการ "ปรับการทำงานนี้ขณะที่ยังทำงานอยู่" หากรันไทม์ปัจจุบัน
ไม่สามารถรับการชี้นำได้ OpenClaw จะส่งข้อความเป็นพรอมต์ปกติแทน
ที่จะละทิ้งข้อความนั้น

## เซสชันปัจจุบัน

ใช้ `/steer` ระดับบนสุดเพื่อกำหนดเป้าหมายเป็นการทำงานที่กำลังดำเนินอยู่ของเซสชันปัจจุบัน:

```text
/steer prefer the smaller patch and keep the tests focused
/tell summarize before making the next tool call
```

ลักษณะการทำงาน:

- กำหนดเป้าหมายเฉพาะการทำงานที่กำลังดำเนินอยู่ของเซสชันปัจจุบันเท่านั้น
- ทำงานโดยไม่ขึ้นกับโหมด `/queue` ของเซสชัน
- เริ่มรอบการสนทนาปกติด้วยข้อความเดียวกันเมื่อเซสชันไม่ได้ทำงาน หรือเมื่อ
  การทำงานที่กำลังดำเนินอยู่ไม่สามารถรับการชี้นำได้
- ใช้เส้นทางการชี้นำของรันไทม์ที่กำลังทำงาน เพื่อให้โมเดลเห็นคำแนะนำ ณ
  ขอบเขตรันไทม์ถัดไปที่รองรับ

## การชี้นำเทียบกับคิว

`/queue steer` ทำให้ข้อความขาเข้าปกติพยายามชี้นำการทำงานที่กำลังดำเนินอยู่ เมื่อ
ข้อความเหล่านั้นมาถึงขณะที่มีการทำงานกำลังดำเนินอยู่ `/steer <message>` เป็นคำสั่ง
ที่ระบุอย่างชัดเจน ซึ่งพยายามแทรกข้อความของคำสั่งนั้นเข้าไปในการทำงานที่กำลังดำเนินอยู่ ณ
ขอบเขตรันไทม์ถัดไปที่รองรับ โดยไม่คำนึงถึงการตั้งค่า `/queue` ที่จัดเก็บไว้ เมื่อ
ไม่สามารถแทรกข้อความได้ ระบบจะตัดคำนำหน้าคำสั่งออก และ `<message>`
จะดำเนินต่อไปเป็นพรอมต์ปกติ

การใช้งาน:

- ใช้ `/steer <message>` เมื่อต้องการชี้นำการทำงานที่กำลังดำเนินอยู่ทันที
- ใช้ `/queue steer` เมื่อต้องการให้ข้อความปกติในอนาคตชี้นำการทำงานที่กำลังดำเนินอยู่
  โดยค่าเริ่มต้น
- ใช้ `/queue collect` หรือ `/queue followup` เมื่อข้อความปกติในอนาคตควรรอ
  รอบการสนทนาภายหลังแทนการชี้นำการทำงานที่กำลังดำเนินอยู่
- ใช้ `/queue interrupt` เมื่อข้อความล่าสุดควรแทนที่การทำงานที่กำลังดำเนินอยู่
  แทนการชี้นำ

สำหรับโหมดคิวและขอบเขตการชี้นำ โปรดดู [คิวคำสั่ง](/th/concepts/queue) และ
[คิวการชี้นำ](/th/concepts/queue-steering)

## เอเจนต์ย่อย

`/steer` ระดับบนสุดกำหนดเป้าหมายเป็นการทำงานที่กำลังดำเนินอยู่ของเซสชันปัจจุบัน เอเจนต์ย่อยจะรายงาน
กลับไปยังเซสชันหลัก/เซสชันผู้ร้องขอ ส่วน `/subagents` ใช้เพื่อดูสถานะเท่านั้น

## เซสชัน ACP

ใช้ `/acp steer` เมื่อเป้าหมายเป็นเซสชันเครื่องมือควบคุม ACP:

```text
/acp steer --session agent:main:acp:codex tighten the repro
```

โปรดดู [เอเจนต์ ACP](/th/tools/acp-agents) สำหรับการเลือกเซสชัน ACP และลักษณะการทำงาน
ของรันไทม์
