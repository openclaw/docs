`/steer` จะพยายามส่งคำแนะนำไปยังการทำงานที่กำลังดำเนินอยู่ก่อน ใช้สำหรับช่วงเวลาที่ต้องการ
"ปรับการทำงานนี้ขณะที่ยังดำเนินอยู่" หากรันไทม์ปัจจุบัน
ไม่สามารถรับการกำกับทิศทางได้ OpenClaw จะส่งข้อความเป็นพรอมต์ปกติแทน
โดยไม่ทิ้งข้อความนั้น

## เซสชันปัจจุบัน

ใช้ `/steer` ระดับบนสุดเพื่อกำหนดเป้าหมายเป็นการทำงานที่กำลังดำเนินอยู่ของเซสชันปัจจุบัน:

```text
/steer เลือกใช้แพตช์ที่เล็กกว่าและให้การทดสอบมุ่งเน้นเฉพาะส่วน
/tell สรุปก่อนเรียกใช้เครื่องมือครั้งถัดไป
```

ลักษณะการทำงาน:

- กำหนดเป้าหมายเฉพาะการทำงานที่กำลังดำเนินอยู่ของเซสชันปัจจุบัน
- ทำงานโดยไม่ขึ้นกับโหมด `/queue` ของเซสชัน
- เริ่มรอบการทำงานปกติด้วยข้อความเดิมเมื่อเซสชันไม่ได้ทำงาน หรือเมื่อ
  การทำงานที่กำลังดำเนินอยู่ไม่สามารถรับการกำกับทิศทางได้
- ใช้เส้นทางการกำกับทิศทางของรันไทม์ที่กำลังทำงานอยู่ ดังนั้นโมเดลจะเห็นคำแนะนำที่
  ขอบเขตรันไทม์ถัดไปที่รองรับ

## การกำกับทิศทางเทียบกับคิว

`/queue steer` ทำให้ข้อความขาเข้าปกติพยายามกำกับทิศทางการทำงานที่กำลังดำเนินอยู่เมื่อ
ข้อความมาถึงระหว่างที่มีการทำงานอยู่ `/steer <message>` เป็นคำสั่งที่ระบุชัดเจน
ซึ่งพยายามแทรกข้อความของคำสั่งนั้นเข้าไปในการทำงานที่กำลังดำเนินอยู่ ณ
ขอบเขตรันไทม์ถัดไปที่รองรับ โดยไม่คำนึงถึงการตั้งค่า `/queue` ที่บันทึกไว้ เมื่อ
ไม่สามารถแทรกได้ ระบบจะตัดคำนำหน้าคำสั่งออกและ `<message>`
จะดำเนินต่อเป็นพรอมต์ปกติ

คำสั่ง `/steer` (และ `/tell`) ที่ระบุชัดเจนทำงานผ่าน Gateway ใน
`openclaw chat` หรือ `openclaw tui --local` ให้เลือก `/queue steer` แล้วส่ง
คำแนะนำเป็นข้อความปกติ รันไทม์แบบฝังจะใช้นโยบายการกำกับทิศทางเดียวกัน
โดยไม่ส่งต่อคำสั่ง Gateway

ใช้:

- `/steer <message>` เมื่อต้องการกำกับการทำงานที่กำลังดำเนินอยู่ทันที
- `/queue steer` เมื่อต้องการให้ข้อความปกติในอนาคตกำกับการทำงานที่กำลังดำเนินอยู่โดย
  ค่าเริ่มต้น
- `/queue collect` หรือ `/queue followup` เมื่อข้อความปกติในอนาคตควรรอ
  รอบการทำงานภายหลังแทนการกำกับการทำงานที่กำลังดำเนินอยู่
- `/queue interrupt` เมื่อต้องการให้ข้อความล่าสุดแทนที่การทำงานที่กำลังดำเนินอยู่
  แทนการกำกับทิศทาง

สำหรับโหมดคิวและขอบเขตการกำกับทิศทาง โปรดดู [คิวคำสั่ง](/th/concepts/queue) และ
[คิวการกำกับทิศทาง](/th/concepts/queue-steering)

## เอเจนต์ย่อย

`/steer` ระดับบนสุดกำหนดเป้าหมายเป็นการทำงานที่กำลังดำเนินอยู่ของเซสชันปัจจุบัน เอเจนต์ย่อยจะรายงาน
กลับไปยังเซสชันหลัก/เซสชันผู้ร้องขอ ส่วน `/subagents` ใช้เพื่อการมองเห็นเท่านั้น

## เซสชัน ACP

ใช้ `/acp steer` เมื่อเป้าหมายเป็นเซสชันของชุดทดสอบ ACP:

```text
/acp steer --session agent:main:acp:codex ทำให้ขั้นตอนการทำซ้ำปัญหากระชับขึ้น
```

โปรดดู [เอเจนต์ ACP](/th/tools/acp-agents) สำหรับการเลือกเซสชัน ACP และลักษณะการทำงานของ
รันไทม์
