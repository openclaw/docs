โหมดโค้ดเป็นฟีเจอร์รันไทม์เอเจนต์ของ OpenClaw ที่ยังอยู่ในขั้นทดลองและต้องเลือกเปิดใช้ เมื่อ
เปิดใช้แล้ว โมเดลจะไม่เห็นสคีมาของเครื่องมือที่เปิดใช้ทั้งหมดอีกต่อไป แต่จะเห็น
`exec`, `wait` และเครื่องมือแบบ direct-only ใดๆ ที่ผลลัพธ์แบบมีโครงสร้างไม่สามารถส่งผ่าน
บริดจ์เกสต์ที่รองรับเฉพาะ JSON ได้ โมเดลจะเขียนโปรแกรม JavaScript หรือ TypeScript ขนาดเล็ก
เพื่อค้นหา ดูคำอธิบาย และเรียกใช้แค็ตตาล็อกเครื่องมือที่ซ่อนอยู่

หน้านี้อธิบายโหมดโค้ดของ OpenClaw ไม่ใช่ Codex Code Mode ฟีเจอร์ทั้งสอง
ใช้ชื่อเดียวกันและชื่อเครื่องมือควบคุมเดียวกัน (`exec`, `wait`) แต่เป็น
การใช้งานแยกจากกัน:

- Codex Code Mode ทำงานภายในฮาร์เนสการเขียนโค้ด Codex เครื่องมือ `exec` ของมันเป็น
  เครื่องมือไวยากรณ์แบบอิสระ โดยโมเดลเขียนซอร์ส JavaScript ดิบ (เลือก
  เติมบรรทัด pragma `// @exec: {...}` ไว้ข้างหน้าสำหรับตัวเลือกการรันได้) ซึ่งจะถูกเรียกใช้
  ในรันไทม์ V8 Code Mode แบบในโพรเซสของ Codex
- โหมดโค้ดของ OpenClaw ทำงานในรันไทม์เอเจนต์ OpenClaw ทั่วไป และจะ
  ปิดใช้งานเว้นแต่มีการกำหนดค่า `tools.codeMode.enabled: true` เครื่องมือ `exec`
  รับเพย์โหลด JSON `{ code, language }` ซึ่งรันในเวิร์กเกอร์ QuickJS-WASI

ทั้งสองเป็นพื้นผิวการรัน JavaScript ไม่ใช่พื้นผิวคำสั่งเชลล์ ให้ถือว่าเป็น
ฟีเจอร์อิสระที่นำไปใช้ต่างกัน แต่บังเอิญเปิดเผยเครื่องมือ
`exec`/`wait` ที่มีชื่อเหมือนกัน

## การทำงาน

- รายการเครื่องมือที่โมเดลมองเห็นจะกลายเป็น `exec`, `wait` รวมถึงเครื่องมือแบบ direct-only ใดๆ
  เช่น `computer` หรือตัวโหลด `image` สำหรับการมองเห็นแบบเนทีฟ ซึ่งผลลัพธ์รูปภาพ
  ไม่สามารถส่งผ่านบริดจ์เกสต์ได้
- `exec` ประเมิน JavaScript หรือ TypeScript ที่โมเดลสร้างขึ้นใน
  เธรดเวิร์กเกอร์ QuickJS-WASI ที่แยกออกมา
- เครื่องมือที่เปิดใช้และมีสิทธิ์อยู่ในแค็ตตาล็อกทุกตัว (คอร์ OpenClaw, Plugin, MCP, ไคลเอนต์) จะถูกซ่อนในฐานะ
  เครื่องมือโมเดลแบบสแตนด์อโลน และเปิดเผยภายในโปรแกรมเกสต์ผ่าน `ALL_TOOLS`
  และ `tools`
- คำอธิบาย `exec` มีดัชนีด่วนแบบจำกัดขอบเขตของรหัสแค็ตตาล็อก OpenClaw/Plugin
  ที่ตรงกันทุกประการ คำใบ้อินพุตแบบย่อ และคำใบ้เอาต์พุตที่ประกาศไว้แบบย่อเมื่อ
  เครื่องมือที่เชื่อถือได้มีสคีมาเอาต์พุต โดยจะละคำอธิบาย สคีมาเต็ม
  รายการ MCP และรายการที่เกินขีดจำกัด การค้นหาแค็ตตาล็อกฝั่งเกสต์ยังคงเป็นทางเลือกสำรอง
- โค้ดเกสต์ค้นหาแค็ตตาล็อกที่ซ่อนอยู่ ดูคำอธิบายสคีมาของเครื่องมือ และเรียกใช้
  เครื่องมือผ่านเส้นทางการรันเดียวกับที่ใช้โดยรอบการทำงานปกติของเอเจนต์ (นโยบาย
  การอนุมัติ ฮุก และเทเลเมทรียังคงมีผลทั้งหมด)
- เครื่องมือ MCP จะถูกรวมกลุ่มภายใต้เนมสเปซ `MCP`; ในโหมดโค้ด นี่คือ
  วิธีเดียวที่รองรับสำหรับการเรียกใช้เครื่องมือเหล่านี้
- `wait` ดำเนินการโหมดโค้ดที่พักไว้ต่อ เมื่อการเรียกใช้เครื่องมือซ้อนยังคง
  รอดำเนินการ

โหมดโค้ดเปลี่ยนเฉพาะพื้นผิวการประสานงานที่แสดงต่อโมเดล โดยไม่ได้
แทนที่เครื่องมือ เครื่องมือ Plugin เครื่องมือ MCP การยืนยันตัวตน นโยบายการอนุมัติ พฤติกรรม
ของช่องทาง หรือการเลือกโมเดล

## เหตุผลที่ควรใช้

- พื้นผิวพรอมต์เล็กลง: ผู้ให้บริการได้รับเครื่องมือควบคุมสองรายการ ดัชนีเครื่องมือเนทีฟ
  แบบจำกัดขอบเขต และเฉพาะเครื่องมือโดยตรงที่จำเป็นเพียงไม่กี่รายการ แทนที่จะเป็นสคีมาเครื่องมือเต็มรูปแบบ
  หลายสิบหรือหลายร้อยรายการ
- การประสานงานที่ดีขึ้น: โมเดลสามารถใช้ลูป การรวมผล การแปลงขนาดเล็ก
  ตรรกะแบบมีเงื่อนไข และการเรียกใช้เครื่องมือซ้อนแบบขนานภายในเซลล์โค้ดเดียว
- จำนวนรอบการโต้ตอบกับโมเดลน้อยลง: สัญญาเอาต์พุตที่ประกาศไว้ช่วยให้โมเดลเรียกใช้และ
  แปลงผลลัพธ์ของเครื่องมือได้ใน `exec` เดียว ส่วนเอาต์พุตที่ไม่ทราบโครงสร้างจะยังคงส่งค่าดิบก่อน
- ไม่ผูกกับผู้ให้บริการ: ใช้งานได้กับเครื่องมือ OpenClaw, Plugin, MCP และไคลเอนต์ โดยไม่
  ขึ้นอยู่กับการรันโค้ดแบบเนทีฟของผู้ให้บริการ
- ล้มเหลวแบบปิด: หากเปิดใช้โหมดโค้ดแต่รันไทม์ QuickJS-WASI
  ไม่พร้อมใช้งาน การทำงานจะล้มเหลวแทนที่จะย้อนกลับอย่างเงียบๆ ไปเปิดเผยเครื่องมือโดยตรง
  ในวงกว้าง

มีประโยชน์สูงสุดสำหรับเอเจนต์ที่เปิดใช้แค็ตตาล็อกเครื่องมือขนาดใหญ่ หรือเวิร์กโฟลว์ที่
โมเดลต้องค้นหา รวม และเรียกใช้เครื่องมือหลายรายการก่อนตอบ

คงการเปิดเผยเครื่องมือโดยตรงไว้สำหรับแค็ตตาล็อกขนาดเล็กหรือโมเดลที่ไม่สามารถ
เขียนโปรแกรมสั้นๆ ได้อย่างน่าเชื่อถือ ใช้ [การค้นหาเครื่องมือ](/th/tools/tool-search) เมื่อต้องการ
แค็ตตาล็อกแบบกะทัดรัด แต่ต้องการตัวควบคุมค้นหา/ดูคำอธิบาย/เรียกใช้แบบมีโครงสร้างแทน
เกสต์ QuickJS-WASI

## เริ่มต้นอย่างรวดเร็ว

### เปิดใช้โหมดโค้ด

```json5
{
  tools: {
    codeMode: {
      enabled: true,
    },
  },
}
```

รูปแบบย่อ:

```json5
{
  tools: {
    codeMode: true,
  },
}
```

โหมดโค้ดยังคงปิดอยู่เมื่อไม่มี `tools.codeMode`, มีค่าเป็น `false` หรือเป็นออบเจ็กต์
ที่ไม่มี `enabled: true`

หากใช้เอเจนต์แบบแซนด์บ็อกซ์ร่วมกับเซิร์ฟเวอร์ MCP ที่กำหนดค่าไว้ ให้อนุญาต
Plugin MCP ที่รวมมาด้วยในนโยบายเครื่องมือของแซนด์บ็อกซ์ด้วย ตัวอย่างเช่น
`tools.sandbox.tools.alsoAllow: ["bundle-mcp"]` ดู
[การกำหนดค่า - เครื่องมือและผู้ให้บริการแบบกำหนดเอง](/th/gateway/config-tools#mcp-and-plugin-tools-inside-sandbox-tool-policy)

กำหนดขีดจำกัดอย่างชัดเจนเพื่อให้มีขอบเขตที่เข้มงวดยิ่งขึ้น:

```json5
{
  tools: {
    codeMode: {
      enabled: true,
      timeoutMs: 10000,
      memoryLimitBytes: 67108864,
      maxOutputBytes: 65536,
      maxSnapshotBytes: 10485760,
      maxPendingToolCalls: 16,
      snapshotTtlSeconds: 900,
      searchDefaultLimit: 8,
      maxSearchLimit: 50,
    },
  },
}
```

### สิ่งที่โมเดลทำ

สำหรับเครื่องมือที่มีเอาต์พุตที่ประกาศไว้ เช่น
`Array<{ id: string; paid: boolean; tons: number }>` โปรแกรมเกสต์หนึ่งโปรแกรมสามารถ
เลือก เรียกใช้ และแปลงผลลัพธ์ได้:

```javascript
const [shipmentTool] = await tools.search("list shipments");
const shipments = await tools.callValue(shipmentTool.id, {});
return shipments.filter((shipment) => !shipment.paid && shipment.tons > 10);
```

เมื่อบรรทัดดัชนีด่วนลงท้ายด้วย `-> ?` แสดงว่าไม่ทราบรูปร่างของเอาต์พุต การเรียก
`exec` ครั้งแรกต้องส่งคืน `await tools.callValue(...)` โดยไม่เปลี่ยนแปลง การเรียก `exec` ในภายหลังจึงจะ
แปลงค่าที่สังเกตได้ วิธีนี้ใช้รอบการทำงานของโมเดลเพิ่มอีกหนึ่งรอบ แต่ป้องกันไม่ให้
โมเดลเดาชื่อฟิลด์

### ตรวจสอบพื้นผิวที่ใช้งานอยู่

เพื่อยืนยันรูปร่างเพย์โหลดของโมเดลระหว่างการดีบัก ให้รัน Gateway พร้อม
การบันทึกที่เจาะจง:

```bash
OPENCLAW_DEBUG_CODE_MODE=1 \
OPENCLAW_DEBUG_MODEL_TRANSPORT=1 \
OPENCLAW_DEBUG_MODEL_PAYLOAD=tools \
openclaw gateway
```

เมื่อโหมดโค้ดทำงาน ชื่อเครื่องมือที่แสดงต่อโมเดลในบันทึกควรเป็น `exec` และ
`wait` สำหรับเพย์โหลดผู้ให้บริการฉบับเต็มที่ปกปิดข้อมูลแล้ว ให้เพิ่ม
`OPENCLAW_DEBUG_MODEL_PAYLOAD=full-redacted` สำหรับเซสชันดีบักระยะสั้น

## ภาพรวมทางเทคนิค

สำหรับผู้ดูแล ผู้เขียน Plugin ที่กำลังดีบักการเปิดเผยเครื่องมือ และผู้ปฏิบัติการที่
ตรวจสอบความถูกต้องของการปรับใช้ที่มีความเสี่ยงสูง

## ขอบเขต

โหมดโค้ดเป็นเจ้าของรูปร่างการประสานงานที่แสดงต่อโมเดลสำหรับการทำงานที่เตรียมไว้ แต่
ไม่ได้เป็นเจ้าของการเลือกโมเดล พฤติกรรมของช่องทาง การยืนยันตัวตน นโยบายเครื่องมือ หรือการนำ
เครื่องมือไปใช้

อยู่ในขอบเขต: คำจำกัดความของเครื่องมือควบคุม/เครื่องมือโดยตรงที่โมเดลมองเห็น การสร้างแค็ตตาล็อกเครื่องมือ
ที่ซ่อนอยู่ การรันเกสต์ JavaScript/TypeScript รันไทม์เวิร์กเกอร์ QuickJS-WASI
คอลแบ็กโฮสต์สำหรับค้นหา/ดูคำอธิบาย/เรียกใช้ สถานะที่ดำเนินการต่อได้สำหรับ
โปรแกรมเกสต์ที่พักไว้ ขีดจำกัดของเอาต์พุต/ระยะหมดเวลา/หน่วยความจำ/การเรียกที่รอดำเนินการ/สแนปช็อต
และการฉายข้อมูลเทเลเมทรี/เส้นทางการทำงานสำหรับการเรียกใช้เครื่องมือซ้อน

อยู่นอกขอบเขต: การรันโค้ดระยะไกลแบบเนทีฟของผู้ให้บริการ ความหมายเชิงพฤติกรรมของการรันเชลล์
การเปลี่ยนแปลงการให้สิทธิ์เครื่องมือที่มีอยู่ สคริปต์ถาวรที่ผู้ใช้เขียน
การเข้าถึงตัวจัดการแพ็กเกจ/ไฟล์/เครือข่าย/โมดูลในโค้ดเกสต์ และการนำส่วนภายในของ
Codex Code Mode กลับมาใช้โดยตรง

เครื่องมือที่ผู้ให้บริการเป็นเจ้าของ เช่น แซนด์บ็อกซ์ Python ระยะไกล เป็นเครื่องมือแยกต่างหาก ดู
[การรันโค้ด](/th/tools/code-execution)

## คำศัพท์

- **โหมดโค้ด**: โหมดรันไทม์ OpenClaw ที่ซ่อนเครื่องมือโมเดลที่เข้ากันได้กับแค็ตตาล็อก
  และเปิดเผย `exec`, `wait` รวมถึงเครื่องมือแบบ direct-only ที่จำเป็น
- **รันไทม์เกสต์**: VM JavaScript QuickJS-WASI ที่ประเมินโค้ดของโมเดล
- **บริดจ์โฮสต์**: พื้นผิวคอลแบ็กแบบแคบที่เข้ากันได้กับ JSON จากโค้ดเกสต์
  กลับเข้าสู่ OpenClaw
- **แค็ตตาล็อก**: รายการเครื่องมือที่มีผลภายในขอบเขตการทำงาน หลังจากการแก้ไข
  นโยบายเครื่องมือ Plugin, MCP และเครื่องมือไคลเอนต์ตามปกติ
- **การเรียกใช้เครื่องมือซ้อน**: การเรียกใช้เครื่องมือจากโค้ดเกสต์ผ่านบริดจ์โฮสต์
- **สแนปช็อต**: สถานะ VM QuickJS-WASI ที่ซีเรียลไลซ์และบันทึกไว้ เพื่อให้ `wait` สามารถดำเนิน
  การทำงานโหมดโค้ดที่พักไว้ต่อได้

## การกำหนดค่า

`tools.codeMode.enabled` เป็นเกตการเปิดใช้งาน การตั้งค่าฟิลด์อื่นไม่ได้
เปิดใช้ฟีเจอร์นี้ด้วยตัวเอง

| ฟิลด์                 | ค่าเริ่มต้น                        | ขอบเขต                                           |
| --------------------- | ------------------------------ | ----------------------------------------------- |
| `enabled`             | `false`                        | บูลีน; เฉพาะ `true` เท่านั้นที่เปิดใช้โหมดโค้ด          |
| `runtime`             | `"quickjs-wasi"`               | ค่าที่รองรับเพียงค่าเดียว                            |
| `mode`                | `"only"`                       | เปิดเผยเครื่องมือควบคุม/โดยตรง และจัดที่เหลือเข้าแค็ตตาล็อก |
| `languages`           | `["javascript", "typescript"]` | ชุดย่อยใดๆ ของทั้งสองรายการ                           |
| `timeoutMs`           | `10000`                        | `100`-`60000`                                   |
| `memoryLimitBytes`    | `67108864`                     | `1048576`-`1073741824`                          |
| `maxOutputBytes`      | `65536`                        | `1024`-`10485760`                               |
| `maxSnapshotBytes`    | `10485760`                     | `1024`-`268435456`                              |
| `maxPendingToolCalls` | `16`                           | `1`-`128`                                       |
| `snapshotTtlSeconds`  | `900`                          | `1`-`86400`                                     |
| `searchDefaultLimit`  | `8`                            | จำกัดไว้ที่ `maxSearchLimit`                     |
| `maxSearchLimit`      | `50`                           | `1`-`50`                                        |

หากเปิดใช้โหมดโค้ดแต่ไม่สามารถโหลด QuickJS-WASI ได้ OpenClaw จะล้มเหลวแบบปิด
สำหรับการทำงานนั้น โดยจะไม่เปิดเผยเครื่องมือปกติเป็นทางเลือกสำรองอย่างเงียบๆ

## การเปิดใช้งาน

โหมดโค้ดจะถูกประเมินหลังจากทราบนโยบายเครื่องมือที่มีผลแล้ว และก่อน
ประกอบคำขอสุดท้ายไปยังโมเดล:

1. กำหนดเอเจนต์ โมเดล ผู้ให้บริการ แซนด์บ็อกซ์ ช่องทาง ผู้ส่ง และนโยบาย
   การรัน
2. สร้างรายการเครื่องมือ OpenClaw ที่มีผล โดยเพิ่มเครื่องมือ Plugin, MCP และ
   ไคลเอนต์ที่เข้าเกณฑ์
3. ใช้นโยบายอนุญาต/ปฏิเสธ
4. หาก `tools.codeMode.enabled` เป็น false ให้ดำเนินการเปิดเผยเครื่องมือตามปกติต่อไป
5. หากเปิดใช้งานและมีเครื่องมือที่ใช้งานอยู่สำหรับการรัน ให้คงเครื่องมือที่จำเป็นซึ่งเรียกใช้โดยตรงเท่านั้น
   และลงทะเบียนเครื่องมือที่มีผลทุกตัวซึ่งเข้าเกณฑ์แค็ตตาล็อกไว้ในแค็ตตาล็อก
   โหมดโค้ด
6. นำเครื่องมือที่บันทึกในแค็ตตาล็อกออกจากรายการที่โมเดลมองเห็น เพิ่ม `exec` และ
   `wait` ควบคู่กับเครื่องมือที่คงไว้ซึ่งเรียกใช้โดยตรงเท่านั้น

การรันที่ตั้งใจให้ไม่มีเครื่องมือ (การเรียกโมเดลดิบ, `disableTools: true`,
หรือรายการ `tools.allow` ที่ว่างเปล่า) จะไม่เปิดใช้งานพื้นผิวโหมดโค้ด แม้
กำหนดค่า `tools.codeMode.enabled: true` ไว้ โหมดโค้ดและการค้นหาเครื่องมือของ OpenClaw
ไม่สามารถใช้ร่วมกันในการรันเดียวได้ หากโหมดโค้ดเปิดใช้งาน Compaction ของการค้นหาเครื่องมือ
จะไม่ทำงาน

แค็ตตาล็อกโหมดโค้ดมีขอบเขตเฉพาะการรัน และต้องไม่ทำให้เครื่องมือจากเอเจนต์
เซสชัน ผู้ส่ง หรือการรันอื่นรั่วไหลเข้ามา

## เครื่องมือที่โมเดลมองเห็น

เมื่อโหมดโค้ดทำงาน โมเดลจะเห็น `exec`, `wait` และเครื่องมือที่จำเป็น
ซึ่งเรียกใช้โดยตรงเท่านั้น เครื่องมืออื่นทุกตัวที่เปิดใช้งานจะถูกซ่อนจากรายการเครื่องมือ
ที่แสดงต่อโมเดล และลงทะเบียนไว้ในแค็ตตาล็อกโหมดโค้ด

ใช้ `exec` สำหรับการประสานงานเครื่องมือ การรวมข้อมูล ลูป การเรียกซ้อนแบบขนาน
และการแปลงข้อมูลแบบมีโครงสร้าง ใช้ `wait` เฉพาะเมื่อ `exec` ส่งคืนผลลัพธ์
`waiting` ที่ดำเนินการต่อได้

## `exec`

`exec` เริ่มเซลล์โหมดโค้ดและส่งคืนผลลัพธ์หนึ่งรายการ โค้ดอินพุตสร้างโดยโมเดล
และต้องถือว่าเป็นอันตราย

อินพุต:

```typescript
type CodeModeExecInput = {
  code?: string;
  command?: string;
  language?: "javascript" | "typescript";
};
```

กฎ:

- `code` หรือ `command` อย่างใดอย่างหนึ่งต้องไม่ว่างเปล่า
- `code` คือฟิลด์ที่แสดงต่อโมเดลตามเอกสาร
- ยอมรับ `command` เป็นนามแฝงที่เข้ากันได้กับ exec สำหรับนโยบายฮุกและ
  การเขียนใหม่ที่เชื่อถือได้ (เครื่องมือ shell exec ปกติของ OpenClaw ก็ใช้ฟิลด์ `command`
  เช่นกัน) เมื่อมีทั้งสองฟิลด์ ค่าต้องตรงกัน
- `language` มีค่าเริ่มต้นเป็น `"javascript"` โดยสคีมาจะแสดงเป็น enum สตริงแบบแบน
  (`"javascript" | "typescript"`) ไม่ใช่ยูเนียน `oneOf`/`anyOf`
  เนื่องจากผู้ให้บริการบางรายปฏิเสธรูปแบบเหล่านั้น
- หาก `language` เป็น `"typescript"` OpenClaw จะแปลงโค้ดก่อนประเมินผล
- `exec` ปฏิเสธ `import`, `require`, การนำเข้าแบบไดนามิก และรูปแบบ
  ตัวโหลดโมดูล
- `exec` จะไม่เปิดเผยการใช้งาน shell `exec` ปกติแบบเรียกซ้ำ
- เหตุการณ์ฮุก `exec` ของโหมดโค้ดชั้นนอกมี `toolKind: "code_mode_exec"` และ
  `toolInputKind: "javascript" | "typescript"` (เมื่อทราบ) เพื่อให้นโยบายสามารถ
  แยกเซลล์โหมดโค้ดออกจากการเรียก `exec` แบบ shell ที่ใช้ชื่อเครื่องมือ
  เดียวกันได้

ผลลัพธ์:

```typescript
type CodeModeResult = CodeModeCompletedResult | CodeModeWaitingResult | CodeModeFailedResult;

type CodeModeCompletedResult = {
  status: "completed";
  value: unknown;
  output?: CodeModeOutput[];
  telemetry: CodeModeTelemetry;
};

type CodeModeWaitingResult = {
  status: "waiting";
  runId: string;
  reason: "pending_tools" | "yield";
  pendingToolCalls?: CodeModePendingToolCall[];
  output?: CodeModeOutput[];
  telemetry: CodeModeTelemetry;
};

type CodeModeFailedResult = {
  status: "failed";
  error: string;
  code?: CodeModeErrorCode;
  output?: CodeModeOutput[];
  telemetry: CodeModeTelemetry;
};
```

`exec` ส่งคืน `waiting` เมื่อเกสต์หยุดชั่วคราวพร้อมสถานะที่ดำเนินการต่อได้ ซึ่งยัง
ต้องการการดำเนินการต่อที่โมเดลมองเห็น ได้แก่ `yield_control(...)` อย่างชัดเจน หรือ
การเรียกเครื่องมือบริดจ์ที่ยังไม่เสร็จภายในกำหนดเวลาของ exec ผลลัพธ์
มี `runId` สำหรับ `wait` การเรียกเครื่องมือบริดจ์ ได้แก่ `tools.search`/`describe`/
`call` และการเรียกผ่านเนมสเปซ รวมถึงการเรียกผ่านเนมสเปซ MCP จะถูกระบายโดยอัตโนมัติ
ภายในการเรียก `exec`/`wait` เดียวกัน ตราบใดที่เสร็จภายในกำหนดเวลา ดังนั้น
บล็อกโค้ดขนาดกะทัดรัดที่รอเครื่องมือหลายตัวจะทำงานจนเสร็จในหนึ่งเทิร์นของโมเดล
แทนที่จะบังคับให้เรียกเครื่องมือของโมเดลหนึ่งครั้งต่อการรอแต่ละครั้ง การรันที่ปลอดภัยต่อการรีสตาร์ต
จะไม่ระบายโดยอัตโนมัติ งานที่ค้างอยู่ยังคงผ่านการตรวจสอบที่ปลอดภัยต่อการเล่นซ้ำ

`exec` ส่งคืน `completed` เฉพาะเมื่อ VM ของเกสต์ไม่มีงานค้างอยู่ และ
ค่าสุดท้ายเข้ากันได้กับ JSON หลังจากอะแดปเตอร์เอาต์พุตของ OpenClaw ทำงานแล้ว

## `wait`

`wait` ดำเนินการ VM โหมดโค้ดที่หยุดชั่วคราวต่อ

อินพุต:

```typescript
type CodeModeWaitInput = {
  runId: string;
};
```

เอาต์พุตเป็นยูเนียน `CodeModeResult` เดียวกับที่ `exec` ส่งคืน

`wait` มีอยู่เนื่องจากเครื่องมือ OpenClaw ที่ซ้อนอยู่สามารถทำงานช้า ต้องโต้ตอบ
มีการควบคุมด้วยการอนุมัติ หรือสตรีมการอัปเดตบางส่วน โมเดลจึงไม่ควรต้องเปิด
การเรียก `exec` ที่ยาวนานค้างไว้ขณะที่โฮสต์รองานภายนอก

สแนปช็อต/การกู้คืน QuickJS-WASI คือกลไกการดำเนินการต่อ:

1. `exec` ประเมินโค้ดจนเสร็จสมบูรณ์ ล้มเหลว หรือหยุดชั่วคราว
2. เมื่อหยุดชั่วคราว OpenClaw จะสร้างสแนปช็อต VM ของ QuickJS และบันทึกงานโฮสต์
   ที่ค้างอยู่
3. เมื่องานที่ค้างอยู่เสร็จสิ้น `wait` จะกู้คืนสแนปช็อต VM และ
   ลงทะเบียนคอลแบ็กของโฮสต์อีกครั้งด้วยชื่อที่คงที่
4. OpenClaw ส่งผลลัพธ์ของเครื่องมือที่ซ้อนอยู่เข้าสู่ VM ที่กู้คืนแล้ว และระบาย
   งานที่ค้างอยู่ของ QuickJS
5. `wait` ส่งคืน `completed`, `failed` หรือผลลัพธ์ `waiting` อื่น

สแนปช็อตเป็นสถานะรันไทม์ ไม่ใช่อาร์ติแฟกต์ของผู้ใช้ โดยอยู่เฉพาะใน
แมปภายในกระบวนการเท่านั้น (ไม่มีการเขียนฐานข้อมูลหรือดิสก์) มีการจำกัดขนาด หมดอายุได้ และมี
ขอบเขตเฉพาะการรันและเซสชันที่สร้างสแนปช็อตนั้น

`wait` ล้มเหลว (เป็นผลลัพธ์ `failed`) เมื่อ:

- ไม่รู้จัก `runId` หรือสแนปช็อตหมดอายุแล้ว
- ผู้เรียกไม่ได้อยู่ในขอบเขตการรัน/เซสชันเดียวกับการรันที่หยุดชั่วคราว
- มี `wait` ที่กำลังดำเนินการอยู่สำหรับ `runId` นั้นแล้ว
- การกู้คืน QuickJS-WASI ล้มเหลว
- การดำเนินการต่อจะเกิน `maxOutputBytes` หรือ `maxSnapshotBytes`

## API รันไทม์ของเกสต์

```typescript
declare const ALL_TOOLS: ToolCatalogEntry[];
declare const tools: ToolCatalog;
declare const MCP: Record<string, unknown>;
declare const namespaces: Record<string, unknown>;

declare function text(value: unknown): void;
declare function json(value: unknown): void;
declare function yield_control(reason?: string): Promise<void>;
```

`ALL_TOOLS` เป็นเมทาดาทาขนาดกะทัดรัดสำหรับแค็ตตาล็อกที่มีขอบเขตเฉพาะการรัน โดยค่าเริ่มต้นจะไม่มี
สคีมาแบบเต็ม คำอธิบาย `exec` ที่โมเดลมองเห็นยังรวมถึง
ชุดย่อยที่มีขอบเขตจำกัดและกำหนดผลได้แน่นอนของรหัส OpenClaw/Plugin ที่ตรงกัน คำใบ้อินพุต
แบบกะทัดรัด และคำใบ้เอาต์พุตที่ประกาศไว้อย่างน่าเชื่อถือ คำอธิบายยังคงถูกเลื่อนการโหลดไว้ เพื่อให้
ข้อความในแค็ตตาล็อกที่เป็นอันตรายไม่สามารถชี้นำโมเดลได้ เมื่อดัชนีนั้นไม่รวมเครื่องมือ
ให้อ่าน `ALL_TOOLS` หรือเรียก `tools.search(...)` ภายในโปรแกรมเกสต์

ลูกศรในแต่ละบรรทัดของดัชนีด่วนอธิบายค่า `tools.callValue(...)`
`-> Array<{ id: string }>` คือคำใบ้เอาต์พุตที่ประกาศไว้ ส่วน `-> ?` หมายถึงไม่ทราบเอาต์พุต
เอาต์พุตที่ไม่ทราบจะคงค่าดิบไว้ก่อน: ส่งคืนค่าโดยไม่เปลี่ยนแปลง สังเกตค่า แล้วจึง
กรองหรือแมปใน `exec` ภายหลัง แทนการคาดเดาชื่อฟิลด์ หลักการนี้ยัง
ใช้เมื่อการอ่านเอาต์พุตที่ประกาศไว้ถูกส่งต่อไปยังการเรียก `-> ?` ขั้นสุดท้าย โดยให้ส่งคืน
ค่าดิบของการเรียกนั้นโดยไม่ห่อหุ้มให้อยู่ในรูปแบบคำตอบที่ร้องขอ

```typescript
type ToolCatalogEntry = {
  id: string;
  name: string;
  label?: string;
  description: string;
  source: "openclaw" | "mcp" | "client";
  sourceName?: string;
  input: string;
  output?: string;
};
```

`input` เป็นลายเซ็นแบบ TypeScript ที่มีขอบเขตจำกัดสำหรับกรณีทั่วไป ใช้
`tools.describe(...)` เมื่อยังต้องการสคีมาแบบเต็มที่ตรงทั้งหมด รายการ MCP ระยะไกล
และไคลเอนต์ใช้ `input: "unknown"` เพื่อให้สคีมาที่ไม่น่าเชื่อถือของรายการเหล่านั้นยังคง
ถูกเลื่อนการโหลดไว้จนถึง `describe` โดย `output`
จะมีเฉพาะเมื่อเป็นคำใบ้แบบกะทัดรัดที่สมบูรณ์ ซึ่งได้มาจากแกนหลัก OpenClaw
หรือ Plugin `outputSchema` ที่เชื่อถือได้เท่านั้น การอ้างสคีมาเอาต์พุตของ MCP และไคลเอนต์จะไม่ถูกยกระดับ
เป็นคำใบ้แค็ตตาล็อกที่เชื่อถือได้นี้

เครื่องมือ Plugin ใช้ `source: "openclaw"` โดยตั้งค่า `sourceName` เป็นรหัสของ
Plugin เจ้าของ และไม่มีค่าแหล่งที่มา `"plugin"` แยกต่างหาก `source: "mcp"` จะ
ใช้เฉพาะกับรายการ MCP ในเมทาดาทา `sourceName`/`mcp` (และถูกกรองออก
จาก `ALL_TOOLS`/`tools.*` โปรดดูด้านล่าง)

สคีมาแบบเต็มจะโหลดเฉพาะเมื่อร้องขอ:

```typescript
type ToolCatalogEntryWithSchema = ToolCatalogEntry & {
  parameters: unknown;
  outputSchema?: unknown;
};
```

ตัวช่วยแค็ตตาล็อก:

```typescript
type ToolCatalog = {
  search(query: string, options?: { limit?: number }): Promise<ToolCatalogEntry[]>;
  describe(id: string): Promise<ToolCatalogEntryWithSchema>;
  callValue(id: string, input?: unknown): Promise<unknown>;
  call(id: string, input?: unknown): Promise<unknown>;
  [safeToolName: string]: unknown;
};
```

ฟังก์ชันเครื่องมืออำนวยความสะดวกจะติดตั้งเฉพาะชื่อที่ปลอดภัยและไม่กำกวม:

```typescript
const files = await tools.search("read local file");
const fileRead = await tools.describe(files[0].id);
const content = await tools.callValue(fileRead.id, { path: "README.md" });

// หากแค็ตตาล็อกที่ซ่อนอยู่มีรายการ `web_search` ที่ไม่กำกวม:
const hits = await tools.web_search({ query: "OpenClaw code mode" });
```

`tools.callValue(...)` ส่งคืนค่า JSON `details` ของเครื่องมือปกติโดยตรง
`tools.call(...)` จะคงเอนเวโลป `{ tool, result }` ดิบไว้สำหรับผู้เรียก
ที่ต้องการบล็อกเนื้อหาหรือเมทาดาทาผลลัพธ์อื่น

## สัญญาเอาต์พุตที่ประกาศไว้

เครื่องมือ OpenClaw สามารถประกาศ `outputSchema` สำหรับค่าที่มีโครงสร้างซึ่งวางไว้ใน
`AgentToolResult.details` ซึ่งมีประโยชน์สำหรับโหมดโค้ดและการค้นหาเครื่องมือ แต่
ไม่ใช่สคีมาการตอบกลับของเครื่องมือแบบเนทีฟของผู้ให้บริการ และไม่เปลี่ยนแปลงการเปิดเผยเครื่องมือ
โดยตรง

สำหรับเครื่องมือที่สร้างด้วย `defineToolPlugin` ให้ประกาศสคีมาไว้ข้าง
`parameters`:

```typescript
import { Type } from "typebox";
import { defineToolPlugin } from "openclaw/plugin-sdk/tool-plugin";

const Shipment = Type.Object(
  {
    id: Type.String(),
    paid: Type.Boolean(),
    tons: Type.Number(),
  },
  { additionalProperties: false },
);

export default defineToolPlugin({
  id: "shipping",
  name: "Shipping",
  description: "Shipment tools.",
  tools: (tool) => [
    tool({
      name: "shipping_list",
      description: "List shipments.",
      parameters: Type.Object({}),
      outputSchema: Type.Array(Shipment),
      execute: async () => loadShipments(),
    }),
  ],
});
```

สำหรับ `api.registerTool(...)` หรือเครื่องมือแบบแฟกทอรี ให้ใส่พร็อพเพอร์ตี `outputSchema`
เดียวกันไว้ในออบเจ็กต์ `AnyAgentTool` ที่ส่งคืน

สัญญาในตัวที่มีอยู่ในปัจจุบันประกอบด้วย `agents_list`, `apply_patch`,
`conversations_list`, `conversations_send`, `conversations_turn`, `edit`,
`openclaw`, `read`, `screen`,
`sessions_history`, `sessions_list`, `sessions_search`, `sessions_send`,
`session_status`, `spawn_task`, `terminal`, `web_fetch` และ `web_search`
การส่งผ่านโดยตรงที่ตรงกันทุกประการสามารถนำสคีมาของโปรโตคอลเจ้าของกลับมาใช้แทน
การทำสัญญาเฉพาะโมเดลซ้ำได้ ตัวอย่างเช่น เครื่องมือการสนทนาเปิดเผย
สคีมาผลลัพธ์ Gateway ชุดเดียวกับที่ `conversations.list`,
`conversations.send` และ `conversations.turn` ใช้; `web_fetch` เป็นเจ้าของสคีมา
เฉพาะเครื่องมือซึ่งคำใบ้ของสคีมาเปิดเผยเมทาดาทาที่เสถียร ข้อความ สถานะแคช และเมทาดาทา
การล้นแบบซ้อน; `web_search` ประกาศยูเนียนผลลัพธ์/คำตอบ/ข้อผิดพลาด/ข้อมูลดิบ
ที่ปรับรูปแบบแล้วอย่างแม่นยำเป็นคำใบ้ดัชนีด่วนที่สมบูรณ์ สัญญาระบบไฟล์ส่งคืน
ผลลัพธ์แบบมีโครงสร้างสำหรับข้อความที่อ่าน รูปภาพ การตัดทอน และกรณีไม่พบแบบไม่บังคับ;
สถานะการเปลี่ยนแปลงจากการแก้ไขอย่างชัดเจนพร้อมข้อมูล diff/patch; และสรุปพาธของ apply-patch
เมื่อดัชนีด่วนประกาศฟิลด์ต่าง ๆ เซลล์เดียวสามารถประกอบการค้นหาและการส่งมอบ
โดยไม่ต้องมีรอบตรวจสอบแยกต่างหาก:

```javascript
const listed = await tools.conversations_list({ query: "บอตสร้าง" });
const target = listed.conversations.find((item) => item.label === "บอตสร้าง");
if (!target) throw new Error("ไม่พบการสนทนา");
return await tools.conversations_send({
  conversationRef: target.conversationRef,
  message: "สร้างเสร็จแล้ว",
});
```

การเรียกแบบซ้อนยังคงใช้นโยบายเครื่องมือ hooks และการอนุมัติตามปกติ หากสัญญาฉบับเต็ม
ตรงกันทุกประการแต่มีขนาดใหญ่เกินกว่าดัชนีด่วนแบบจำกัด สัญญานั้นยังคง
เข้าถึงได้ผ่าน `tools.describe(...)` และลูกศรยังคงเป็น `-> ?`

กฎของสัญญาเข้มงวดดังนี้:

- อธิบายค่า `details` ที่เข้ากันได้กับ JSON อย่างแม่นยำ ไม่ใช่บล็อก `content`
  ที่เรนเดอร์แล้วหรือเอนเวโลปของผู้ให้บริการ
- รวมทุกตัวแปรผลลัพธ์สำเร็จหรือข้อผิดพลาดที่ไม่ throw ละเว้น `outputSchema` เมื่อ
  เครื่องมือไม่มีผลลัพธ์แบบมีโครงสร้างที่เสถียร
- ปิดเลเยอร์ออบเจ็กต์ด้วย `{ additionalProperties: false }` เพื่อให้ได้
  คำใบ้ดัชนีด่วนที่สมบูรณ์ สคีมาที่เปิดอยู่ ใหญ่เกินไป หรือไม่สมบูรณ์ด้วยเหตุอื่น
  ยังคงเข้าถึงได้ผ่าน `tools.describe(...)` แต่ไม่เปิดให้ใช้ฟิลด์ได้ในรอบเดียว
- OpenClaw คอมไพล์สคีมาก่อนเรียกใช้เครื่องมือ จากนั้นตรวจสอบ `details`
  สุดท้ายหลังจาก hooks ของเครื่องมือตามปกติและก่อนที่การเรียกแค็ตตาล็อกจะส่งคืน
  สคีมาที่ไม่ถูกต้องไม่สามารถเรียกใช้เครื่องมือได้; ค่าที่ไม่ตรงกันจะล้มเหลวโดยไม่พิมพ์ค่า
- คำใบ้แบบกระชับเป็นแบบกำหนดแน่นอนและมีขอบเขตจำกัด `tools.describe(...)` เปิดเผย
  สคีมาที่เชื่อถือได้ฉบับเต็มเมื่อคำใบ้แบบกระชับไม่เพียงพอ
- โค้ด Plugin ที่ติดตั้งแล้วถือเป็นโค้ดภายในเครื่องที่เชื่อถือได้อยู่แล้ว เมทาดาทาของ MCP ระยะไกลและไคลเอนต์
  ยังคงไม่น่าเชื่อถือและไม่สามารถเลือกใช้คำใบ้ดัชนีด่วนเหล่านี้ได้

ดูรายละเอียดการสร้าง Plugin ที่ [Plugin เครื่องมือ](/th/plugins/tool-plugins#output-contracts)

รายการแค็ตตาล็อก MCP ไม่สามารถเรียกผ่าน `tools.callValue(...)`,
`tools.call(...)` หรือฟังก์ชันอำนวยความสะดวกในโหมดโค้ด; รายการเหล่านี้เปิดเผย
ผ่านเนมสเปซ `MCP` ที่สร้างขึ้นเท่านั้น ไฟล์ประกาศรูปแบบ TypeScript
เข้าถึงได้ผ่านพื้นผิวไฟล์เสมือน `API` แบบอ่านอย่างเดียว เพื่อให้เอเจนต์สามารถ
ตรวจสอบลายเซ็น MCP โดยไม่ต้องเพิ่มสคีมา MCP ลงในพรอมต์:

```typescript
const files = await API.list("mcp");
const githubApi = await API.read("mcp/github.d.ts");

const issue = await MCP.github.createIssue({
  owner: "openclaw",
  repo: "openclaw",
  title: "ตรวจสอบบันทึก Gateway",
});

const snapshot = await MCP.chromeDevtools.takeSnapshot({ output: "markdown" });
const resource = await MCP.docs.resources.read({ uri: "memo://one" });
const prompt = await MCP.docs.prompts.get({
  name: "brief",
  arguments: { topic: "รุ่นเผยแพร่" },
});
```

`API.read("mcp/<server>.d.ts")` ส่งคืนประกาศแบบกระชับที่อนุมานจากเมทาดาทาของ
เครื่องมือ MCP:

```typescript
type McpToolResult = {
  content?: unknown[];
  structuredContent?: unknown;
  isError?: boolean;
  [key: string]: unknown;
};

declare namespace MCP.github {
  /** ส่งคืนส่วนหัว API รูปแบบ TypeScript นี้ */
  function $api(toolName?: string, options?: { schema?: boolean }): Promise<McpApiHeader>;

  /**
   * สร้าง issue ของ GitHub
   * @param owner เจ้าของรีพอซิทอรี
   * @param repo ชื่อรีพอซิทอรี
   * @param title ชื่อ issue
   */
  function createIssue(input: {
    owner: string;
    repo: string;
    title: string;
    body?: string;
  }): Promise<McpToolResult>;
}
```

ไฟล์ประกาศเป็นไฟล์เสมือน ไม่ได้เขียนไว้ภายใต้ไดเรกทอรีพื้นที่ทำงานหรือไดเรกทอรี
สถานะ สำหรับการเรียก `exec` ในโหมดโค้ดแต่ละครั้ง OpenClaw จะสร้างแค็ตตาล็อก
เครื่องมือที่มีขอบเขตตามการรัน เก็บรายการ MCP ที่มองเห็นได้ เรนเดอร์ `mcp/index.d.ts` พร้อม
`mcp/<server>.d.ts` หนึ่งรายการต่อเซิร์ฟเวอร์ที่มองเห็นได้ และแทรกตารางแบบอ่านอย่างเดียวขนาดเล็กนั้น
ลงในเวิร์กเกอร์ QuickJS โค้ดเกสต์มองเห็นเฉพาะออบเจ็กต์ `API`:
`API.list(prefix?)` ส่งคืนเมทาดาทาของไฟล์ และ `API.read(path)` ส่งคืน
เนื้อหาประกาศที่เลือก พาธที่ไม่รู้จักและเซกเมนต์ `.`/`..`
จะถูกปฏิเสธ

วิธีนี้กันสคีมา MCP ขนาดใหญ่ออกจากพรอมต์ของโมเดล: เอเจนต์เรียนรู้ว่า
มี API เสมือนอยู่จากคำอธิบายเครื่องมือ `exec` อ่านเฉพาะ
ไฟล์ประกาศที่ต้องการ แล้วเรียก `MCP.<server>.<tool>()` ด้วยอาร์กิวเมนต์ออบเจ็กต์หนึ่งรายการ
`MCP.<server>.$api()` ยังคงใช้เป็นทางเลือกสำรองแบบอินไลน์สำหรับ
การตอบกลับสคีมาของเครื่องมือเดียวภายในโปรแกรมได้

รันไทม์เกสต์จะไม่เห็นออบเจ็กต์โฮสต์โดยตรง อินพุตและเอาต์พุตข้าม
บริดจ์ในรูปค่าที่เข้ากันได้กับ JSON พร้อมขีดจำกัดขนาดที่ระบุชัดเจน

## เนมสเปซภายใน

เนมสเปซภายในมอบ API ของโดเมนที่กระชับให้โหมดโค้ด โดยไม่เพิ่ม
เครื่องมือที่โมเดลมองเห็น การผสานรวมที่ตัวโหลดเป็นเจ้าของจะลงทะเบียนเนมสเปซ เช่น
`Issues` หรือ `Calendar`; จากนั้นโค้ดเกสต์จะเรียกเนมสเปซนั้นภายใน
โปรแกรม QuickJS ขณะที่โมเดลยังคงมองเห็นพื้นผิวควบคุม/โดยตรงแบบกระชับ

ขณะนี้เนมสเปซเป็นแบบภายใน ยังไม่มี API เนมสเปซ SDK ของ Plugin สาธารณะ:
เนมสเปซของ Plugin ภายนอกต้องมีสัญญาที่ตัวโหลดเป็นเจ้าของ เพื่อไม่ให้ข้อมูลประจำตัวของ Plugin,
ไฟล์ manifest ที่ติดตั้ง สถานะการยืนยันตัวตน และตัวอธิบายแค็ตตาล็อกที่แคชไว้คลาดเคลื่อน
จากเครื่องมือ Plugin ที่รองรับเนมสเปซ โหมดโค้ดหลักเป็นเจ้าของเฉพาะ
แซนด์บ็อกซ์ การซีเรียลไลซ์ การควบคุมแค็ตตาล็อก และการส่งต่อผ่านบริดจ์

โค้ดเกสต์สามารถใช้ได้ทั้งโกลบอลโดยตรงหรือแมป `namespaces`:

```javascript
const open = await Issues.list({ state: "open" });
const alsoOpen = await namespaces.Issues.list({ state: "open" });
return { count: open.length, alsoCount: alsoOpen.length };
```

### วงจรชีวิตของรีจิสทรี

รีจิสทรีของเนมสเปซอยู่ภายในโปรเซสและใช้ id ของเนมสเปซเป็นคีย์:

1. ตัวโหลดที่เชื่อถือได้เรียก `registerCodeModeNamespaceForPlugin(pluginId, registration)`
2. โหมดโค้ดสร้าง `ToolSearchRuntime` ที่ซ่อนไว้สำหรับการรันและอ่าน
   แค็ตตาล็อกที่มีขอบเขตตามการรัน
3. `createCodeModeNamespaceRuntime(ctx, catalog)` เก็บเฉพาะการลงทะเบียน
   ที่ `requiredToolNames` ทั้งหมดมองเห็นได้และเป็นของ `pluginId` เดียวกัน
4. แต่ละเนมสเปซที่มองเห็นได้เรียก `createScope(ctx)` สำหรับการรันปัจจุบัน
   โดยรับบริบทการรัน เช่น `agentId`, `sessionKey`, `sessionId`,
   `runId`, การกำหนดค่า และสถานะยกเลิก
5. ข้อมูลขอบเขตถูกซีเรียลไลซ์เป็นตัวอธิบายแบบธรรมดาและแทรกลงใน QuickJS
   เป็นโกลบอลโดยตรงและ `namespaces.<globalName>`
6. การเรียกจากเกสต์จะหยุดพักผ่านบริดจ์ของเวิร์กเกอร์ แก้พาธเนมสเปซ
   บนโฮสต์ แมปการเรียกไปยังเครื่องมือแค็ตตาล็อกที่ Plugin เจ้าของประกาศ และ
   เรียกใช้เครื่องมือนั้นผ่าน `ToolSearchRuntime.callExactId`
7. การเรียกบริดจ์เนมสเปซที่พร้อมจะถูกระบายโดยอัตโนมัติภายในการเรียก
   `exec`/`wait` ที่ใช้งานอยู่; หากงานเนมสเปซยังค้างอยู่เมื่อหมดเวลา หรือ
   เกสต์ยอมคืนการควบคุมอย่างชัดเจน `wait` จะดำเนินรันไทม์เนมสเปซเดิมต่อ
   ในภายหลัง
8. การย้อนกลับหรือถอนการติดตั้ง Plugin จะเรียก
   `clearCodeModeNamespacesForPlugin(pluginId)` เพื่อไม่ให้โกลบอลที่ล้าสมัย
   คงอยู่หลังการโหลด Plugin ล้มเหลว

การเรียกเนมสเปซคือการเรียกเครื่องมือแค็ตตาล็อก: การเรียกเหล่านี้ใช้นโยบาย hooks,
การอนุมัติ การจัดการการยกเลิก เทเลเมทรี การฉายทรานสคริปต์ และ
พฤติกรรมหยุดพัก/ดำเนินการต่อแบบเดียวกับ `tools.call(...)`

### รูปแบบการลงทะเบียน

ลงทะเบียนเนมสเปซจากการผสานรวมที่เป็นเจ้าของเครื่องมือเบื้องหลัง รักษา
ขอบเขตให้เล็ก และเปิดเผยเฉพาะกริยาของโดเมนที่แมปไปยังเครื่องมือแค็ตตาล็อก
ที่ประกาศไว้

```typescript
import {
  createCodeModeNamespaceTool,
  registerCodeModeNamespaceForPlugin,
} from "../agents/code-mode-namespaces.js";

const pluginId = "github";

registerCodeModeNamespaceForPlugin(pluginId, {
  id: "github-issues",
  globalName: "Issues",
  description: "ตัวช่วย issue ของ GitHub สำหรับรีพอซิทอรีปัจจุบัน",
  requiredToolNames: ["github_list_issues", "github_update_issue"],
  prompt: "ใช้ Issues.list(params) และ Issues.update(number, patch)",
  createScope: (ctx) => ({
    repository: ctx.config,
    list: createCodeModeNamespaceTool("github_list_issues", ([params]) => params ?? {}),
    update: createCodeModeNamespaceTool("github_update_issue", ([number, patch]) => ({
      number,
      patch,
    })),
  }),
});
```

`createCodeModeNamespaceTool(toolName, inputMapper)` ทำเครื่องหมายสมาชิกของขอบเขตเป็น
ฟังก์ชันเนมสเปซที่เรียกใช้ได้ `inputMapper` แบบไม่บังคับจะรับอาร์กิวเมนต์
จากเกสต์และส่งคืนออบเจ็กต์อินพุตสำหรับเครื่องมือแค็ตตาล็อกเบื้องหลัง; หากไม่มี
จะใช้อาร์กิวเมนต์แรกจากเกสต์ หรือใช้ `{}` หากไม่ได้ระบุ

ฟังก์ชันโฮสต์ดิบจะถูกปฏิเสธก่อนที่โค้ดเกสต์จะทำงาน:

```typescript
createScope: () => ({
  // ผิด: วิธีนี้ข้ามวงจรชีวิตของเครื่องมือแค็ตตาล็อกและจะถูกปฏิเสธ
  list: async () => githubClient.listIssues(),
});
```

### ความเป็นเจ้าของและการมองเห็น

ความเป็นเจ้าของเนมสเปซผูกกับ `pluginId` ของผู้เรียกที่ลงทะเบียน
`requiredToolNames` เป็นทั้งด่านควบคุมการมองเห็นและการตรวจสอบความเป็นเจ้าของ:

- เครื่องมือที่จำเป็นทุกเครื่องมือต้องมีอยู่ในแค็ตตาล็อกของการรัน
- เครื่องมือที่จำเป็นทุกเครื่องมือต้องมี `sourceName === pluginId`
- เนมสเปซจะถูกซ่อนเมื่อเครื่องมือที่จำเป็นรายการใดขาดหายหรือเป็นของ
  Plugin อื่น
- แต่ละพาธที่เรียกใช้ได้อาจกำหนดเป้าหมายได้เฉพาะเครื่องมือที่ระบุชื่อใน `requiredToolNames`

วิธีนี้ป้องกันไม่ให้ Plugin อื่นเปิดเผยเนมสเปซด้วยการลงทะเบียน
เครื่องมือชื่อเดียวกัน และทำให้เนมสเปซสอดคล้องกับนโยบายเอเจนต์ตามปกติ: หาก
การรันมองไม่เห็นเครื่องมือเบื้องหลัง ก็จะมองไม่เห็นเนมสเปซด้วย

ตัวอย่างเช่น เนมสเปซ GitHub ควรอยู่หลัง Plugin ที่ GitHub เป็นเจ้าของ ซึ่ง
เป็นเจ้าของการยืนยันตัวตน GitHub, ไคลเอนต์ REST/GraphQL, ขีดจำกัดอัตรา, การอนุมัติการเขียน และ
การทดสอบ โหมดโค้ดหลักไม่ควรฝัง API เฉพาะ GitHub, การจัดการโทเค็น
หรือนโยบายผู้ให้บริการ

### กฎการซีเรียลไลซ์ขอบเขต

`createScope(ctx)` อาจส่งคืนออบเจ็กต์ธรรมดาที่ประกอบด้วยค่า
ที่เข้ากันได้กับ JSON, อาร์เรย์, ออบเจ็กต์ซ้อน และมาร์กเกอร์การเรียก `createCodeModeNamespaceTool(...)`
ออบเจ็กต์โฮสต์จะไม่เข้าสู่ QuickJS โดยตรง

ตัวซีเรียลไลซ์จะปฏิเสธ:

- ฟังก์ชันดิบ
- กราฟออบเจ็กต์แบบวนซ้ำ
- เซกเมนต์พาธที่ไม่ปลอดภัย: `__proto__`, `constructor`, `prototype`, คีย์ว่าง
  หรือคีย์ที่มีตัวคั่นพาธภายใน
- ค่า `globalName` ที่ไม่ใช่ตัวระบุ JavaScript
- การชนกันของ `globalName` กับโกลบอลในตัวของโหมดโค้ด เช่น `tools`,
  `namespaces`, `text`, `json`, `yield_control`, `MCP`, `API`, `ALL_TOOLS` หรือ
  `__openclaw*`

ค่าที่ไม่สามารถซีเรียลไลซ์เป็น JSON จะถูกแปลงเป็นค่าทางเลือกสำรอง
ที่ปลอดภัยสำหรับ JSON ก่อนข้ามบริดจ์ ข้อมูลไบนารี แฮนเดิล ซ็อกเก็ต ไคลเอนต์ และ
อินสแตนซ์คลาสควรอยู่หลังเครื่องมือแค็ตตาล็อกตามปกติ

### พรอมต์

`description` ของเนมสเปซและ `prompt` แบบไม่บังคับจะถูกต่อท้ายสคีมา
`exec` ที่โมเดลมองเห็นเฉพาะเมื่อเนมสเปซมองเห็นได้สำหรับการรันนั้น ใช้
รายการเหล่านี้เพื่อสอนพื้นผิวที่มีประโยชน์ขนาดเล็กที่สุด:

```typescript
{
  description: "ตัวช่วยสำหรับบริการผลิตงานแต่งเรื่อง",
  prompt:
    "ใช้ Fictions.riskAudit(), Fictions.promoteIfReady(id, status) และ Fictions.unpaidOver(amount)",
}
```

ให้พรอมต์กล่าวถึงสัญญาของเนมสเปซ ไม่ใช่การตั้งค่าการยืนยันตัวตน ประวัติ
การติดตั้งใช้งาน หรือพฤติกรรมของ Plugin อื่นที่ไม่เกี่ยวข้อง

### การล้างข้อมูล

เนมสเปซเป็นการลงทะเบียนภายในโปรเซส ให้ลบเมื่อ Plugin เจ้าของ
ถูกปิดใช้งาน ถอนการติดตั้ง หรือย้อนกลับ:

```typescript
clearCodeModeNamespacesForPlugin(pluginId);
```

การล้างข้อมูลโหมดโค้ดเป็นความรับผิดชอบของ Plugin ให้ล้างการลงทะเบียนเนมสเปซของ Plugin
เมื่อวงจรชีวิตสิ้นสุดลง แทนที่จะเก็บแฮนเดิลการรื้อถอนแยกสำหรับแต่ละเนมสเปซ
การทดสอบสามารถเรียก `clearCodeModeNamespacesForTest()` เพื่อหลีกเลี่ยงการรั่วไหลของ
การลงทะเบียนข้ามกรณีทดสอบ

### รายการตรวจสอบการทดสอบ

การเปลี่ยนแปลงเนมสเปซควรครอบคลุมขอบเขตความปลอดภัยและพฤติกรรมของเกสต์:

- ข้อความพรอมต์ของเนมสเปซปรากฏเฉพาะเมื่อมองเห็นเครื่องมือที่รองรับอยู่
- เครื่องมือชื่อเดียวกันจาก `sourceName` อื่นจะไม่เปิดเผยเนมสเปซ
- ฟังก์ชันสโคปดิบถูกปฏิเสธ
- รหัสเนมสเปซปลอมและพาธปลอมถูกปฏิเสธ
- พาธที่เรียกใช้ได้ต้องไม่กำหนดเป้าหมายไปยังเครื่องมือที่ไม่ได้ประกาศ
- ออบเจ็กต์ซ้อนและการอ้างอิงร่วมได้รับการซีเรียลไลซ์อย่างถูกต้อง
- การเรียกเนมสเปซดำเนินการผ่านเครื่องมือในแค็ตตาล็อกและส่งคืนรายละเอียดที่ปลอดภัยสำหรับ JSON
- โค้ดเกสต์สามารถดักจับความล้มเหลวได้
- การเรียกเนมสเปซที่ถูกระงับดำเนินการต่อผ่าน `wait`
- การย้อนกลับ Plugin ล้างการลงทะเบียนเนมสเปซของเจ้าของ

เนมสเปซเสริมแค็ตตาล็อก `tools.search`/`tools.call` แบบทั่วไป: ใช้
แค็ตตาล็อกสำหรับเครื่องมือ OpenClaw, Plugin และไคลเอนต์ที่เปิดใช้งานตามต้องการ; ใช้ `MCP`
สำหรับเครื่องมือ MCP; ใช้เนมสเปซอื่นสำหรับ API โดเมนที่มีเอกสารกำกับและเป็นของ Plugin
ซึ่งโค้ดแบบกระชับเชื่อถือได้มากกว่าการค้นหาสคีมาซ้ำ ๆ

## API เอาต์พุต

- `text(value)` เพิ่มเอาต์พุตที่มนุษย์อ่านได้ต่อท้ายอาร์เรย์ `output`
- `json(value)` เพิ่มรายการเอาต์พุตแบบมีโครงสร้างต่อท้ายหลังจากการซีเรียลไลซ์
  ที่เข้ากันได้กับ JSON
- ค่าที่ส่งคืนสุดท้ายของโค้ดเกสต์จะกลายเป็น `value` ในผลลัพธ์ `completed`

```typescript
type CodeModeOutput = { type: "text"; text: string } | { type: "json"; value: unknown };
```

กฎ: ลำดับเอาต์พุตตรงกับลำดับการเรียกของเกสต์; เอาต์พุตถูกจำกัดด้วย
`maxOutputBytes`; ค่าที่ซีเรียลไลซ์ไม่ได้จะถูกแปลงเป็นสตริงธรรมดาหรือ
ข้อผิดพลาด; ไม่รองรับค่าไบนารี รูปภาพและไฟล์ส่งผ่าน
เครื่องมือ OpenClaw ตามปกติ ไม่ใช่ผ่านบริดจ์โหมดโค้ด

## แค็ตตาล็อกเครื่องมือ

แค็ตตาล็อกที่ซ่อนไว้ประกอบด้วยเครื่องมือหลังจากกรองด้วยนโยบายที่มีผลแล้ว ตาม
ลำดับนี้: เครื่องมือแกนหลักของ OpenClaw, เครื่องมือจาก Plugin ที่รวมมาให้, เครื่องมือจาก Plugin ภายนอก, เครื่องมือ MCP
จากนั้นเป็นเครื่องมือที่ไคลเอนต์จัดเตรียมให้สำหรับการรันปัจจุบัน

รหัสแค็ตตาล็อกมีเสถียรภาพภายในการรันหนึ่งครั้งและกำหนดซ้ำได้ในชุด
เครื่องมือที่เทียบเท่ากันเมื่อเป็นไปได้ รูปแบบจริง:

```text
<source>:<owner>:<tool-name>
```

โดย `<source>` คือ `openclaw`, `mcp` หรือ `client` (เครื่องมือ Plugin ใช้
`openclaw` โดยมีรหัส Plugin เป็น `<owner>`; เครื่องมือแกนหลักใช้ `openclaw:core:*`)
ตัวอย่าง:

```text
openclaw:core:message
openclaw:browser:browser_request
mcp:github:create_issue
client:app:select_file
```

แค็ตตาล็อกไม่รวมเครื่องมือควบคุมโหมดโค้ด (`exec`, `wait`, `tool_search_code`,
`tool_search`, `tool_describe`, `tool_call`) และเครื่องมือแบบโดยตรงเท่านั้น การควบคุม
ต้องไม่เรียกซ้ำผ่านแค็ตตาล็อก; เครื่องมือแบบโดยตรงเท่านั้นยังคงปรากฏต่อโมเดล
เพราะผลลัพธ์แบบมีโครงสร้างของเครื่องมือเหล่านั้นไม่สามารถข้ามบริดจ์ QuickJS ได้

รายการ MCP ยังคงอยู่ในแค็ตตาล็อกระดับการรัน เพื่อให้นโยบาย การอนุมัติ ฮุก
เทเลเมทรี การฉายทรานสคริปต์ และรหัสเครื่องมือที่แน่นอนยังคงใช้ร่วมกับ
การเรียกใช้เครื่องมือตามปกติ มุมมอง `ALL_TOOLS`, `tools.search(...)`,
`tools.describe(...)`, `tools.callValue(...)` และ `tools.call(...)` ที่แสดงต่อเกสต์จะไม่รวมรายการ MCP
เนมสเปซ `MCP.<server>.<tool>({ ...input })` ที่สร้างขึ้นจะแปลงกลับไปยัง
รหัสแค็ตตาล็อกที่แน่นอน และส่งงานผ่านพาธตัวดำเนินการเดียวกัน

## การทำงานร่วมกับ Tool Search

โหมดโค้ดจะแทนที่พื้นผิวโมเดล Tool Search ของ OpenClaw สำหรับการรันที่
เปิดใช้งานโหมดนี้

เมื่อ `tools.codeMode.enabled` เป็น true และโหมดโค้ดทำงาน:

- OpenClaw จะไม่เปิดเผย `tool_search_code`, `tool_search`, `tool_describe`
  หรือ `tool_call` เป็นเครื่องมือที่โมเดลมองเห็น
- แนวคิดการจัดทำแค็ตตาล็อกเดียวกันจะย้ายเข้าไปภายในรันไทม์เกสต์
- รันไทม์เกสต์ได้รับเมทาดาทา `ALL_TOOLS` แบบกระชับ และตัวช่วยค้นหา/อธิบาย/
  เรียกใช้สำหรับเครื่องมือที่ไม่ใช่ MCP
- การเรียก MCP ใช้เนมสเปซ `MCP` ที่สร้างขึ้นและส่วนหัว `$api()` ของเนมสเปซนั้น
  แทน `tools.call(...)`
- การเรียกซ้อนส่งงานผ่านพาธตัวดำเนินการ OpenClaw เดียวกับที่ Tool
  Search ใช้

ดู [Tool Search](/th/tools/tool-search) สำหรับบริดจ์แค็ตตาล็อกแบบกระชับของ OpenClaw
ที่โหมดโค้ดจะแทนที่ในการรันที่เปิดใช้งาน

## ชื่อเครื่องมือและการชนกัน

เครื่องมือ `exec` ที่โมเดลมองเห็นคือเครื่องมือโหมดโค้ด หากเปิดใช้งาน
เครื่องมือเชลล์ `exec` ตามปกติของ OpenClaw เครื่องมือนั้นจะถูกซ่อนจากโมเดลและจัดเข้าแค็ตตาล็อกเหมือน
เครื่องมืออื่น ๆ

ภายในรันไทม์เกสต์:

- `tools.call("openclaw:core:exec", input)` สามารถเรียกเครื่องมือดำเนินการคำสั่งเชลล์ได้ หาก
  นโยบายอนุญาต
- `tools.exec(...)` จะติดตั้งเฉพาะเมื่อรายการดำเนินการเชลล์ในแค็ตตาล็อกมี
  ชื่อที่ปลอดภัยและไม่กำกวม
- เครื่องมือโหมดโค้ด `exec` จะไม่พร้อมให้เรียกแบบวนซ้ำผ่าน `tools`

หากเครื่องมือสองรายการถูกปรับให้เป็นชื่ออำนวยความสะดวกที่ปลอดภัยเดียวกัน OpenClaw จะไม่สร้าง
ฟังก์ชันอำนวยความสะดวก และกำหนดให้ใช้ `tools.call(id, input)`

## การเรียกใช้เครื่องมือซ้อน

การเรียกเครื่องมือซ้อนทุกครั้งจะข้ามบริดจ์โฮสต์และกลับเข้าสู่ OpenClaw
โดยคงไว้ซึ่ง: รหัสเอเจนต์ที่ใช้งาน รหัสและคีย์เซสชัน บริบทผู้ส่งและช่องทาง
นโยบายแซนด์บ็อกซ์ นโยบายการอนุมัติ ฮุก `before_tool_call` ของ Plugin สัญญาณ
ยกเลิก การอัปเดตแบบสตรีมเมื่อพร้อมใช้งาน และเหตุการณ์วิถีการทำงาน/การตรวจสอบ

การเรียกซ้อนจะถูกฉายเข้าไปในทรานสคริปต์เป็นการเรียกเครื่องมือจริง เพื่อให้บันเดิล
การสนับสนุนแสดงสิ่งที่เกิดขึ้น โดยการฉายจะระบุการเรียกเครื่องมือ
โหมดโค้ดหลักและรหัสเครื่องมือซ้อน

อนุญาตให้เรียกซ้อนพร้อมกันได้สูงสุด `maxPendingToolCalls`

## วงจรชีวิตของการรันและสแนปช็อต

การรันโหมดโค้ดแต่ละครั้งถูกติดตามในแมปภายในโปรเซสที่ใช้ `runId` เป็นคีย์ (ไม่
บันทึกลงดิสก์หรือฐานข้อมูล) `exec`/`wait` ส่งคืนสถานะผลลัพธ์หนึ่งในสามแบบ:
`completed`, `waiting` หรือ `failed`

- ผลลัพธ์ `waiting` จะจัดเก็บสแนปช็อต QuickJS คำขอบริดจ์ที่รอดำเนินการ และ
  เมทาดาทาการกำหนดสโคป (รหัสการรันเอเจนต์ รหัส/คีย์เซสชัน) จนกว่า `wait` จะดำเนินการต่อหรือ
  หมดอายุ
- ค่าของ `runId` ที่หมดอายุ อยู่ผิดเซสชัน อยู่ผิดการรัน และไม่รู้จัก/กำลังดำเนินการต่ออยู่แล้ว
  จะไม่สร้างสถานะปลายทางแยกต่างหาก แต่จะแสดงเป็น
  ผลลัพธ์ `failed` (`code: "invalid_input"`) พร้อมข้อความ เช่น `code mode
run is unavailable or expired.` หรือ `code mode run belongs to a different
session.`
- สแนปช็อตของการรันจะถูกลบออกจากแมปทันทีที่สิ้นสุดเป็น
  `completed` หรือ `failed` หรือถูกทิ้งเมื่อ Gateway ปิดตัวลง (ไม่มีสิ่งใด
  คงอยู่หลังการรีสตาร์ต: นี่คือสถานะรันไทม์ชั่วคราว)
- สำหรับงานแบบอ่านอย่างเดียว `exec` สามารถตั้งค่า `restartSafe: true` จากนั้น OpenClaw จะปฏิเสธ
  การเรียกแค็ตตาล็อกที่มีผลข้างเคียงและเนมสเปซของ Plugin ก่อนดำเนินการ และ
  ทำเครื่องหมายผลลัพธ์ที่ถูกระงับว่าปลอดภัยสำหรับการเล่นซ้ำ หากการรีสตาร์ตขัดจังหวะ `wait`,
  [การกู้คืนหลังรีสตาร์ต](/th/gateway/restart-recovery) จะสร้างเทิร์นใหม่จาก
  ทรานสคริปต์ แทนการกู้คืนสแนปช็อตภายในโปรเซส เทิร์นการกู้คืน
  ยังคงถูกจำกัดไว้เฉพาะเครื่องมือแกนหลักแบบอ่านอย่างเดียวที่ผ่านการตรวจสอบ และเครื่องมือ Plugin ที่
  ระบุชัดเจนว่าปลอดภัยสำหรับการเล่นซ้ำ
- OpenClaw จำกัดจำนวนการรันที่ถูกระงับพร้อมกันต่อโปรเซสไว้ที่ (64) และ
  ปฏิเสธการระงับใหม่ที่เกินขีดจำกัดดังกล่าวด้วย `too many suspended code mode
runs.`

พื้นที่จัดเก็บสแนปช็อตถูกจำกัดด้วย `maxSnapshotBytes` ต่อการรัน ขีดจำกัด
การรันที่ถูกระงับต่อโปรเซสข้างต้น และ `snapshotTtlSeconds`

## รันไทม์ QuickJS-WASI

OpenClaw โหลด `quickjs-wasi` เป็นการขึ้นต่อกันโดยตรงในแพ็กเกจเจ้าของ โดย
ไม่พึ่งสำเนาแบบส่งผ่านที่ติดตั้งไว้สำหรับการขึ้นต่อกันที่ไม่เกี่ยวข้อง

ความรับผิดชอบของรันไทม์: คอมไพล์/โหลดโมดูล WebAssembly ของ QuickJS-WASI;
สร้าง VM ที่แยกจากกันหนึ่งชุดต่อการรันโหมดโค้ดหรือการดำเนินการต่อ; ลงทะเบียนคอลแบ็กโฮสต์
ด้วยชื่อที่เสถียร; ตั้งค่าขีดจำกัดหน่วยความจำและการขัดจังหวะ; ประเมิน JavaScript; ระบาย
งานที่รอดำเนินการ; สร้างสแนปช็อตสถานะ VM ที่ถูกระงับ; กู้คืนสแนปช็อตสำหรับ `wait`;
กำจัดแฮนเดิล VM และสแนปช็อตหลังสถานะปลายทาง

รันไทม์ทำงานในเธรดผู้ปฏิบัติงาน Node.js แยกจากลูป
เหตุการณ์หลักของ OpenClaw ลูปไม่สิ้นสุดของเกสต์ต้องไม่บล็อกโปรเซส Gateway
อย่างไม่มีกำหนด; ตัวจัดการการขัดจังหวะของผู้ปฏิบัติงานบังคับใช้การหมดเวลาตามเวลาจริง
โดยไม่ขึ้นกับการให้ความร่วมมือของโค้ดเกสต์

## TypeScript

การรองรับ TypeScript เป็นเพียงการแปลงซอร์ส: อินพุตที่ยอมรับคือสตริงโค้ด
TypeScript หนึ่งรายการ; เอาต์พุตคือสตริง JavaScript ที่ QuickJS-WASI ประเมินผล
ไม่มีการตรวจสอบชนิด ไม่มีการแก้ไขโมดูล และไม่มี
`import`/`require` การวินิจฉัยถูกส่งคืนเป็นผลลัพธ์ `failed`

คอมไพเลอร์ TypeScript จะถูกโหลดแบบล่าช้าเฉพาะเซลล์ TypeScript; เซลล์
JavaScript ธรรมดาและโหมดโค้ดที่ปิดใช้งานจะไม่โหลดคอมไพเลอร์

## ขอบเขตความปลอดภัย

โค้ดโมเดลไม่ควรได้รับความไว้วางใจ รันไทม์ใช้การป้องกันเชิงลึก:

- เรียกใช้ QuickJS-WASI นอกลูปเหตุการณ์หลัก ในเธรดผู้ปฏิบัติงาน
- โหลด `quickjs-wasi` เป็นการขึ้นต่อกันโดยตรง ไม่ใช่ผ่าน Codex หรือ
  แพ็กเกจแบบส่งผ่าน
- ไม่มีระบบไฟล์ เครือข่าย โปรเซสย่อย การนำเข้าโมดูล ตัวแปรสภาพแวดล้อม
  หรือออบเจ็กต์โกลบอลของโฮสต์ในเกสต์
- ใช้ขีดจำกัดหน่วยความจำและการขัดจังหวะของ QuickJS ร่วมกับการหมดเวลา
  ตามเวลาจริงของโปรเซสหลัก
- บังคับใช้ขีดจำกัดเอาต์พุต สแนปช็อต บันทึก และการเรียกที่รอดำเนินการ
- ซีเรียลไลซ์ค่าบริดจ์โฮสต์ผ่านอะแดปเตอร์ JSON ที่จำกัดขอบเขต
- แปลงข้อผิดพลาดของโฮสต์เป็นข้อผิดพลาดธรรมดาของเกสต์ โดยไม่ส่งออบเจ็กต์จากขอบเขตโฮสต์
- ทิ้งสแนปช็อตเมื่อหมดเวลา ยกเลิก เซสชันสิ้นสุด หรือหมดอายุ
- ปฏิเสธการเข้าถึงแบบวนซ้ำไปยัง `exec`, `wait` และเครื่องมือควบคุม Tool Search
- ป้องกันไม่ให้การชนกันของชื่ออำนวยความสะดวกบดบังตัวช่วยแค็ตตาล็อก

แซนด์บ็อกซ์เป็นหนึ่งในชั้นความปลอดภัย; ผู้ดำเนินการอาจยังจำเป็นต้องเสริมความแข็งแกร่ง
ในระดับระบบปฏิบัติการสำหรับการติดตั้งใช้งานที่มีความเสี่ยงสูง

## รหัสข้อผิดพลาด

```typescript
type CodeModeErrorCode =
  | "invalid_input"
  | "runtime_unavailable"
  | "timeout"
  | "output_limit_exceeded"
  | "snapshot_limit_exceeded"
  | "internal_error";
```

`invalid_input` ครอบคลุมอาร์กิวเมนต์ `exec`/`wait` ที่ไม่ถูกต้อง ภาษาที่ปิดใช้งาน
การเข้าถึงโมดูลที่ถูกปฏิเสธ ความล้มเหลวในการแปลง TypeScript ค่า `runId` ที่ไม่รู้จัก/หมดอายุ/
อยู่ผิดสโคป และการรันที่ถูกระงับมากเกินไป `runtime_unavailable`
ครอบคลุมผู้ปฏิบัติงาน QuickJS ที่เริ่มทำงานไม่สำเร็จหรือออกด้วยรหัสที่ไม่ใช่ศูนย์

ข้อผิดพลาดที่ส่งคืนให้เกสต์เป็นข้อมูลธรรมดา; อินสแตนซ์ `Error` ของโฮสต์ ออบเจ็กต์
สแต็ก โปรโตไทป์ และฟังก์ชันโฮสต์จะไม่ข้ามเข้าไปใน QuickJS

## เทเลเมทรี

ฟิลด์ `telemetry` ของแต่ละผลลัพธ์รายงาน: ขนาดแค็ตตาล็อกที่ซ่อนอยู่และการแจกแจง
ตามแหล่งที่มา (จำนวน `openclaw`/`mcp`/`client`) จำนวนการค้นหา/อธิบาย/เรียกใช้สะสม
สำหรับแค็ตตาล็อกของการรัน และชื่อเครื่องมือที่โมเดลมองเห็น (`exec`,
`wait` และเครื่องมือแบบโดยตรงเท่านั้นที่ยังคงไว้)

เทเลเมทรีต้องไม่มีข้อมูลลับ ค่าสภาพแวดล้อมดิบ หรืออินพุตเครื่องมือที่ไม่ผ่านการปกปิด
เกินกว่านโยบายวิถีการทำงานที่มีอยู่ของ OpenClaw

## การดีบัก

ใช้การบันทึกการขนส่งโมเดลแบบเจาะจง เมื่อโหมดโค้ดทำงานแตกต่างจาก
การรันเครื่องมือตามปกติ:

```bash
OPENCLAW_DEBUG_CODE_MODE=1 \
OPENCLAW_DEBUG_MODEL_TRANSPORT=1 \
OPENCLAW_DEBUG_MODEL_PAYLOAD=tools \
OPENCLAW_DEBUG_SSE=events \
openclaw gateway
```

สำหรับการดีบักรูปแบบเพย์โหลด ให้ใช้ `OPENCLAW_DEBUG_MODEL_PAYLOAD=full-redacted`
คำสั่งนี้จะบันทึกสแนปช็อต JSON ของคำขอโมเดลที่ผ่านการปกปิดข้อมูลและจำกัดขนาดแล้ว ควรใช้เฉพาะ
ระหว่างการดีบักเท่านั้น เนื่องจากพรอมป์และข้อความในข้อความสนทนาอาจยังคงปรากฏได้

สำหรับการดีบักสตรีม ให้ใช้ `OPENCLAW_DEBUG_SSE=peek` เพื่อบันทึกเหตุการณ์ SSE ที่ผ่านการปกปิดข้อมูลแล้วห้าเหตุการณ์แรก
โหมดโค้ดจะล้มเหลวแบบปิดด้วย หากเพย์โหลดสุดท้ายของผู้ให้บริการ
ไม่มี `exec` จำนวนหนึ่งรายการ, `wait` จำนวนหนึ่งรายการ และมีเฉพาะ
เครื่องมือที่อนุมัติให้เรียกใช้โดยตรงเท่านั้น หลังจากเปิดใช้งานพื้นผิวโหมดโค้ดแล้ว

## โครงสร้างการใช้งาน

- สัญญาการกำหนดค่า: `tools.codeMode`
- ตัวสร้างแค็ตตาล็อก: แปลงเครื่องมือที่มีผลเป็นรายการแบบกระชับและแมป ID
- อะแดปเตอร์พื้นผิวโมเดล: แทนที่เครื่องมือที่มองเห็นได้ด้วยเครื่องมือควบคุม/เครื่องมือโดยตรง
- อะแดปเตอร์รันไทม์ QuickJS-WASI: โหลด ประเมินผล สร้างสแนปช็อต กู้คืน กำจัด
- ตัวควบคุมเวิร์กเกอร์: การหมดเวลา การยกเลิก การแยกข้อขัดข้อง
- อะแดปเตอร์บริดจ์: คอลแบ็กโฮสต์ที่ปลอดภัยสำหรับ JSON และการส่งมอบผลลัพธ์
- อะแดปเตอร์แปลง TypeScript
- ที่เก็บสแนปช็อต: TTL ขีดจำกัดขนาด การกำหนดขอบเขตตามการรัน/เซสชัน
- การฉายเส้นทางการทำงานสำหรับการเรียกเครื่องมือแบบซ้อน
- ตัวนับเทเลเมทรีและการวินิจฉัย

การใช้งานนำแนวคิดแค็ตตาล็อกและตัวดำเนินการจากการค้นหาเครื่องมือมาใช้ซ้ำ แต่
ไม่ใช้ลูก `node:vm` เป็นแซนด์บ็อกซ์

## รายการตรวจสอบการตรวจสอบความถูกต้อง

ความครอบคลุมของโหมดโค้ดควรพิสูจน์ว่า:

- การกำหนดค่าที่ปิดใช้งานทำให้การเปิดเผยเครื่องมือที่มีอยู่ไม่เปลี่ยนแปลง
- การกำหนดค่าแบบออบเจ็กต์ที่ไม่มี `enabled: true` ทำให้โหมดโค้ดยังคงปิดใช้งาน
- การกำหนดค่าที่เปิดใช้งานจะแสดง `exec`, `wait` และเฉพาะเครื่องมือที่จำเป็น
  ซึ่งเรียกใช้โดยตรงเท่านั้นแก่โมเดล เมื่อเครื่องมือทำงานอยู่สำหรับการรันนั้น
- การรันแบบไม่มีเครื่องมือโดยตรง, `disableTools` และรายการอนุญาตที่ว่างเปล่า จะไม่เรียกใช้
  การบังคับใช้เพย์โหลดโหมดโค้ด
- เครื่องมือที่มีผลและไม่ใช่ MCP ทั้งหมดซึ่งมีสิทธิ์อยู่ในแค็ตตาล็อก จะปรากฏใน `ALL_TOOLS`
- เครื่องมือที่เรียกใช้โดยตรงเท่านั้นยังคงปรากฏแก่โมเดล และไม่ปรากฏใน `ALL_TOOLS`
- เครื่องมือที่ถูกปฏิเสธจะไม่ปรากฏใน `ALL_TOOLS`
- `tools.search`, `tools.describe`, `tools.callValue` และ `tools.call` ทำงานกับเครื่องมือ OpenClaw
- `API.list("mcp")` และ `API.read("mcp/<server>.d.ts")` แสดงการประกาศ MCP
  ในรูปแบบ TypeScript โดยไม่มีการเรียกบริดจ์/เครื่องมือ
- เนมสเปซ MCP `$api()` ยังคงพร้อมใช้งานเป็นทางเลือกสำรองแบบอินไลน์สำหรับสคีมา
- การเรียกเนมสเปซ MCP ทำงานกับเครื่องมือ MCP ที่มองเห็นได้ซึ่งมีอินพุตออบเจ็กต์หนึ่งรายการ ขณะที่
  รายการแค็ตตาล็อก MCP โดยตรงไม่ปรากฏใน `tools.*`
- เครื่องมือควบคุมการค้นหาเครื่องมือถูกซ่อนทั้งจากพื้นผิวโมเดลและ
  แค็ตตาล็อกที่ซ่อนอยู่
- การเรียกแบบซ้อนยังคงรักษาพฤติกรรมการอนุมัติและฮุก
- เชลล์ `exec` ถูกซ่อนจากโมเดล แต่สามารถเรียกด้วย ID แค็ตตาล็อกได้เมื่อ
  ได้รับอนุญาต
- ไม่สามารถเรียก `exec` และ `wait` ของโหมดโค้ดแบบเรียกซ้ำจากโค้ดเกสต์ได้
- อินพุต TypeScript ถูกแปลงและประเมินผลโดยไม่โหลด TypeScript บน
  เส้นทางที่ปิดใช้งานหรือเส้นทาง JavaScript เท่านั้น
- `import`, `require`, การเข้าถึงระบบไฟล์ เครือข่าย และสภาพแวดล้อมล้มเหลว
- ลูปไม่รู้จบหมดเวลาและไม่สามารถบล็อก Gateway ได้
- ความล้มเหลวจากขีดจำกัดหน่วยความจำจะยุติ VM ของเกสต์
- มีการบังคับใช้ขีดจำกัดเอาต์พุตและสแนปช็อตสำหรับการเรียกที่เสร็จสมบูรณ์และถูกระงับ
- `wait` ดำเนินการต่อจากสแนปช็อตที่ถูกระงับและคืนค่าสุดท้าย
- ค่า `runId` ที่หมดอายุ ถูกยกเลิก อยู่ผิดเซสชัน และไม่รู้จัก จะล้มเหลว
- การเล่นซ้ำและการคงอยู่ของทรานสคริปต์รักษาการเรียกควบคุมโหมดโค้ดไว้
- ทรานสคริปต์และเทเลเมทรีแสดงการเรียกเครื่องมือแบบซ้อนอย่างชัดเจน

## แผนการทดสอบ E2E

เรียกใช้รายการเหล่านี้เป็นการทดสอบการผสานรวมหรือการทดสอบตั้งแต่ต้นจนจบเมื่อเปลี่ยนแปลงรันไทม์:

1. เริ่ม Gateway ด้วย `tools.codeMode.enabled: false`
2. ส่งรอบการทำงานของเอเจนต์พร้อมชุดเครื่องมือโดยตรงขนาดเล็ก
3. ยืนยันว่าเครื่องมือที่โมเดลมองเห็นไม่เปลี่ยนแปลง
4. เริ่มใหม่ด้วย `tools.codeMode.enabled: true`
5. ส่งรอบการทำงานของเอเจนต์พร้อมเครื่องมือทดสอบ OpenClaw, Plugin, MCP และไคลเอนต์
6. ยืนยันว่ารายการเครื่องมือที่โมเดลมองเห็นคือ `exec`, `wait` รวมถึงเฉพาะ
   เครื่องมือที่กำหนดค่าให้เรียกใช้โดยตรงเท่านั้น
7. ใน `exec` ให้อ่าน `ALL_TOOLS` และยืนยันว่ามีเครื่องมือทดสอบที่มีผล
   ซึ่งมีสิทธิ์อยู่ในแค็ตตาล็อก ขณะที่ไม่มีเครื่องมือที่เรียกใช้โดยตรงเท่านั้น
8. ใน `exec` ให้เรียกเครื่องมือ OpenClaw/Plugin/ไคลเอนต์ผ่าน `tools.search`,
   `tools.describe` และ `tools.callValue` (หรือ `tools.call` แบบดิบ)
9. ใน `exec` ให้เรียก `API.list("mcp")` และ `API.read("mcp/<server>.d.ts")` และ
   ยืนยันว่าไฟล์ประกาศอธิบายเครื่องมือ MCP ที่มองเห็นได้
10. ใน `exec` ให้เรียกเครื่องมือ MCP ผ่าน `MCP.<server>.<tool>({ ...input })` และ
    ยืนยันว่ารายการแค็ตตาล็อก MCP โดยตรงไม่ปรากฏใน `ALL_TOOLS` และ
    `tools.*`
11. ยืนยันว่าเครื่องมือที่ถูกปฏิเสธไม่มีอยู่ และไม่สามารถเรียกด้วย ID ที่คาดเดาได้
12. เริ่มการเรียกเครื่องมือแบบซ้อนที่เสร็จสิ้นหลังจาก `exec` คืนค่า `waiting`
13. เรียก `wait` และยืนยันว่า VM ที่กู้คืนแล้วได้รับผลลัพธ์จากเครื่องมือ
14. ยืนยันว่าคำตอบสุดท้ายมีเอาต์พุตที่สร้างขึ้นหลังการกู้คืน
15. ยืนยันว่าการหมดเวลา การยกเลิก และการหมดอายุของสแนปช็อตล้างสถานะรันไทม์
16. ส่งออกเส้นทางการทำงานและยืนยันว่าการเรียกแบบซ้อนปรากฏอยู่ภายใต้การเรียก
    โหมดโค้ดระดับบน

การเปลี่ยนแปลงเฉพาะเอกสารในหน้านี้ยังคงควรเรียกใช้ `pnpm check:docs`
