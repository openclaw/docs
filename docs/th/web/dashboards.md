ทุกเธรดใน Control UI มีสองมุมมอง ได้แก่ การสนทนาที่คุณคุ้นเคย และ
**แดชบอร์ด** ซึ่งเป็นตารางวิดเจ็ตแบบสดที่เอเจนต์สร้างให้คุณ เธรดที่
ไม่มีวิดเจ็ตจะเป็นเพียงแชต ทันทีที่ปักหมุดวิดเจ็ต ตัวสลับ **แชต | แดชบอร์ด**
จะปรากฏในส่วนหัว และแดชบอร์ดจะกลายเป็นพื้นที่หลักโดยมี
แชตของคุณเทียบอยู่ด้านข้าง

ไม่ต้องตั้งค่าใดๆ และไม่มีแอปแยกต่างหากให้กำหนดค่า แดชบอร์ดเป็น
ฟีเจอร์หลักที่เธรดเป็นเจ้าของ จัดเก็บไว้กับเอเจนต์ และยังคงอยู่หลัง
`/new` และ `/reset` (บริบทการสนทนาจะถูกล้าง แต่บอร์ดยังคงอยู่)

## สร้างแดชบอร์ดด้วยการขอ

บอกเอเจนต์ว่าคุณต้องการเห็นอะไร:

> สร้างวิดเจ็ตชื่อ revenue-graph: แผนภูมิแท่งแบบโต้ตอบของรายได้
> รายเดือน เพิ่มปุ่ม "แท่ง" และ "แนวโน้ม" สำหรับสลับมุมมอง แล้วปักหมุดไว้บน
> แดชบอร์ดของฉัน

เอเจนต์จะแสดงวิดเจ็ตแบบอินไลน์ในแชตก่อน เพื่อให้คุณตรวจดูก่อน
นำไปไว้ที่อื่น จากนั้น:

- **คุณปักหมุดเอง**: วางเมาส์เหนือวิดเจ็ตอินไลน์แล้วเลือก **ปักหมุดไปยังแดชบอร์ด**
- **หรือให้เอเจนต์ปักหมุดโดยตรง** เมื่อคุณขอ และอัปเดตภายหลังโดยใช้
  ชื่อ วิดเจ็ตมีชื่อที่คงที่ ดังนั้น "อัปเดต revenue-graph ด้วย
  ตัวเลขของเดือนมิถุนายน" จะแทนที่เนื้อหาในตำแหน่งเดิมโดยที่บอร์ดยังคงอยู่

วิดเจ็ตเป็นแอปขนาดเล็กที่ทำงานได้ด้วยตัวเอง (HTML/JS/SVG ในแซนด์บ็อกซ์ที่เข้มงวด) ปุ่ม
และตัวสลับมุมมองภายในวิดเจ็ตใช้งานได้ทันที การสลับมุมมองแผนภูมิ
ไม่จำเป็นต้องใช้เอเจนต์

## บอร์ด

- **ตารางแบบยืดหยุ่น** ลากวิดเจ็ตด้วยที่จับ ทุกอย่างจะจัดเรียงใหม่และ
  กระชับโดยอัตโนมัติ ปรับขนาดด้วยที่จับหรือเลือกค่าขนาดสำเร็จรูป (เล็ก
  กลาง ใหญ่ ใหญ่มาก) จากเมนูวิดเจ็ต ไม่มีใครต้องจัดวางเป็นพิกเซล
  ทั้งคุณและเอเจนต์
- **แท็บ** บอร์ดหนึ่งรายการมีได้หลายหน้า เช่น แท็บภาพรวมและ
  แท็บเฉพาะที่มีวิดเจ็ตขนาดใหญ่เพียงรายการเดียว แต่ละแท็บจะจดจำตำแหน่ง
  ที่เทียบแชตของตนเอง
- **แชตแบบเทียบข้าง** ในมุมมองแดชบอร์ด การสนทนาของคุณจะเทียบไว้
  ทางซ้าย ขวา หรือด้านล่าง ปรับขนาดได้เหมือนแถบด้านข้าง และซ่อน
  ได้ทั้งหมด เอเจนต์ยังคงรับฟังคุณเมื่อคุณนำแชตกลับมา
- **ความสามารถเท่าเทียมของเอเจนต์** ทุกสิ่งที่คุณทำได้ เอเจนต์ก็ทำได้ด้วย
  เครื่องมือ `dashboard` ได้แก่ เพิ่ม อัปเดต ย้าย ปรับขนาด และนำวิดเจ็ตออก จัดการ
  แท็บ สลับแท็บที่มองเห็น และย้ายหรือซ่อนส่วนเทียบแชต ลองบอกว่า "ย้าย
  แชตไปทางซ้ายและแสดงแท็บการเงิน" แล้วดูการเปลี่ยนแปลงที่เกิดขึ้น

## สิ่งที่วิดเจ็ตได้รับอนุญาตให้ทำ

วิดเจ็ตที่มีเพียงการแสดงผลไม่ต้องได้รับการอนุมัติ โดยจะปรากฏขึ้นทันทีเหมือนกับ
วิดเจ็ตแชตแบบอินไลน์ทุกประการ และการเข้าถึงเครือข่ายจะถูกปิดใช้งานทั้งหมด

วิดเจ็ตที่ต้องการ **เข้าถึง** ต้องประกาศสิ่งที่ต้องการ และคุณอนุญาตให้ได้หนึ่งครั้งต่อวิดเจ็ต
ด้วยการแตะครั้งเดียว:

- **เครือข่าย** (`net`): ดึงข้อมูลจากต้นทาง HTTPS ที่ประกาศไว้โดยตรงจากแซนด์บ็อกซ์
  เช่น การ์ดสภาพอากาศที่รีเฟรชตัวเองจาก API
- **ข้อมูล Gateway** (`data`): ฟีดแบบอ่านอย่างเดียว เช่น เซสชัน การใช้งาน หรือสถานะ
  Cron ซึ่ง Gateway เป็นผู้ดำเนินการ วิดเจ็ตจะไม่เก็บโทเค็นของคุณ
- **การทำงานอัตโนมัติ** (`actions`): เรียกใช้งาน Cron ที่ระบุ เพื่อให้ปุ่มสามารถเรียกใช้
  งานจริง (ซึ่งอาจใช้โมเดลขนาดเล็กกว่า) โดยไม่ปลุกการสนทนา
  หลักของคุณ
- **พรอมต์** (`prompt`): ส่งข้อความเข้าเธรดของคุณโดยไม่ต้องยืนยันทุกครั้งที่คลิก
  ซึ่งวิดเจ็ตที่ไม่ได้รับการอนุมัติต้องใช้

Plugin ที่เปิดใช้งานสามารถเพิ่มฟีดแบบอ่านอย่างเดียวและการดำเนินการที่มีชื่อของตนเองลงในรายการความสามารถเหล่านี้ การปิดใช้งาน Plugin จะนำการผสานการทำงานเหล่านั้นออก

สิทธิ์ที่อนุญาตจะผูกกับไบต์และรุ่นแก้ไขของวิดเจ็ตที่คุณตรวจสอบโดยตรง หาก
เอเจนต์เปลี่ยนวิดเจ็ตและขอสิทธิ์ _เพิ่มเติม_ จากที่คุณอนุมัติไว้ สถานะจะกลับ
ไปเป็นรอดำเนินการ การรีเฟรชเนื้อหาภายใต้สิทธิ์เดิมจะยังคงการอนุญาตไว้
การโต้ตอบกับวิดเจ็ตที่เอเจนต์ควรทราบ (ตัวกรองที่คุณคลิก มุมมองที่
คุณสลับ) จะส่งไปถึงเอเจนต์อย่างเงียบๆ ในรูปแบบการแจ้งเตือนของเซสชัน เอเจนต์จึงรับรู้ข้อมูลอยู่เสมอโดยไม่
ถูกรบกวน

## แอป MCP บนบอร์ด

หาก Gateway ของคุณกำหนดค่าเซิร์ฟเวอร์ MCP ไว้ แอป MCP แบบโต้ตอบที่ปรากฏ
ในแชตสามารถปักหมุดได้เหมือนวิดเจ็ตทั่วไป แอปที่ปักหมุดจะกลับมาทำงานบน
บอร์ดพร้อมเซสชันใหม่ โดยค่าเริ่มต้นแอปจะแสดงผลอย่างเดียว และการอนุญาตให้
วิดเจ็ตใช้เครื่องมือเซิร์ฟเวอร์ที่ประกาศไว้จะทำให้โต้ตอบได้อย่างเต็มรูปแบบ โดยใช้การอนุมัติ
ด้วยการแตะครั้งเดียวและผูกกับรุ่นแก้ไขเช่นเดียวกับรายการอื่นทั้งหมด

## สิ่งที่ควรทราบ

- การรีเซ็ตเธรดที่มีบอร์ดจะขอการยืนยันและเก็บ
  บอร์ดไว้
- การลบเธรดจะลบบอร์ดของเธรดด้วย
- บอร์ดอยู่บน Gateway ของคุณ (ในฐานข้อมูลของเอเจนต์เจ้าของ) และปรากฏบน
  ทุกอุปกรณ์ที่คุณใช้เชื่อมต่อ
- โมเดลความปลอดภัย รายละเอียดการจัดเก็บ และเหตุผลในการออกแบบอยู่ใน
  [สถาปัตยกรรมแดชบอร์ด](/th/web/dashboard-architecture) ซึ่งรวมถึงข้อแลกเปลี่ยน
  ของแซนด์บ็อกซ์ที่จัดทำเป็นเอกสารไว้
