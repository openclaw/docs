## เริ่มต้นใช้งานอย่างรวดเร็ว

### โหมด Gateway

1. เริ่ม Gateway

```bash
openclaw gateway
```

2. เปิด TUI

```bash
openclaw tui
```

3. พิมพ์ข้อความแล้วกด Enter

Gateway ระยะไกล:

```bash
openclaw tui --url ws://<host>:<port> --token <gateway-token>
```

ใช้ `--password` หาก Gateway ของคุณใช้การยืนยันตัวตนด้วยรหัสผ่าน

### โหมดภายในเครื่อง

เรียกใช้ TUI โดยไม่ใช้ Gateway:

```bash
openclaw chat
# or
openclaw tui --local
```

หมายเหตุ:

- `openclaw chat` และ `openclaw terminal` เป็นชื่อแฝงของ `openclaw tui --local`
- `--local` ไม่สามารถใช้ร่วมกับ `--url`, `--token` หรือ `--password` ได้
- โหมดภายในเครื่องใช้รันไทม์เอเจนต์แบบฝังโดยตรง เครื่องมือภายในเครื่องส่วนใหญ่ใช้งานได้ แต่ฟีเจอร์ที่ใช้ได้เฉพาะ Gateway จะไม่พร้อมใช้งาน
- `openclaw` และ `openclaw crestodian` ยังใช้เชลล์ TUI นี้ด้วย โดยมี Crestodian เป็นแบ็กเอนด์แชตสำหรับตั้งค่าและซ่อมแซมภายในเครื่อง

## สิ่งที่คุณเห็น

- ส่วนหัว: URL การเชื่อมต่อ, เอเจนต์ปัจจุบัน, เซสชันปัจจุบัน
- บันทึกแชต: ข้อความผู้ใช้, คำตอบผู้ช่วย, ประกาศระบบ, การ์ดเครื่องมือ
- บรรทัดสถานะ: สถานะการเชื่อมต่อ/การรัน (กำลังเชื่อมต่อ, กำลังรัน, กำลังสตรีม, ว่าง, ข้อผิดพลาด)
- ส่วนท้าย: สถานะการเชื่อมต่อ + เอเจนต์ + เซสชัน + โมเดล + think/fast/verbose/trace/reasoning + จำนวนโทเค็น + deliver
- อินพุต: ตัวแก้ไขข้อความพร้อมการเติมคำอัตโนมัติ

## โมเดลความเข้าใจ: เอเจนต์ + เซสชัน

- เอเจนต์คือ slug ที่ไม่ซ้ำกัน (เช่น `main`, `research`) Gateway เปิดเผยรายการนี้
- เซสชันเป็นของเอเจนต์ปัจจุบัน
- คีย์เซสชันถูกจัดเก็บเป็น `agent:<agentId>:<sessionKey>`
  - หากคุณพิมพ์ `/session main` TUI จะขยายเป็น `agent:<currentAgent>:main`
  - หากคุณพิมพ์ `/session agent:other:main` คุณจะสลับไปยังเซสชันเอเจนต์นั้นอย่างชัดเจน
- ขอบเขตเซสชัน:
  - `per-sender` (ค่าเริ่มต้น): แต่ละเอเจนต์มีหลายเซสชัน
  - `global`: TUI จะใช้เซสชัน `global` เสมอ (ตัวเลือกอาจว่างได้)
- เอเจนต์ + เซสชันปัจจุบันจะแสดงในส่วนท้ายเสมอ
- เมื่อเริ่มโดยไม่มี `--session` TUI ในโหมด Gateway จะกลับมาใช้เซสชันที่เลือกล่าสุดสำหรับ Gateway, เอเจนต์ และขอบเขตเซสชันเดียวกัน หากเซสชันนั้นยังมีอยู่ การส่ง `--session`, `/session`, `/new` หรือ `/reset` ยังคงเป็นการระบุอย่างชัดเจน

## การส่ง + การนำส่ง

- ข้อความจะถูกส่งไปยัง Gateway; การนำส่งไปยังผู้ให้บริการปิดอยู่ตามค่าเริ่มต้น
- เปิดการนำส่ง:
  - `/deliver on`
  - หรือแผงการตั้งค่า
  - หรือเริ่มด้วย `openclaw tui --deliver`

## ตัวเลือก + โอเวอร์เลย์

- ตัวเลือกโมเดล: แสดงรายการโมเดลที่พร้อมใช้งานและตั้งค่าการแทนที่ของเซสชัน
- ตัวเลือกเอเจนต์: เลือกเอเจนต์อื่น
- ตัวเลือกเซสชัน: แสดงเฉพาะเซสชันสำหรับเอเจนต์ปัจจุบัน
- การตั้งค่า: สลับการนำส่ง, การขยายเอาต์พุตเครื่องมือ และการมองเห็นการคิด

## แป้นพิมพ์ลัด

- Enter: ส่งข้อความ
- Esc: ยกเลิกการรันที่ใช้งานอยู่
- Ctrl+C: ล้างอินพุต (กดสองครั้งเพื่อออก)
- Ctrl+D: ออก
- Ctrl+L: ตัวเลือกโมเดล
- Ctrl+G: ตัวเลือกเอเจนต์
- Ctrl+P: ตัวเลือกเซสชัน
- Ctrl+O: สลับการขยายเอาต์พุตเครื่องมือ
- Ctrl+T: สลับการมองเห็นการคิด (โหลดประวัติใหม่)

## คำสั่งสแลช

แกนหลัก:

- `/help`
- `/status`
- `/agent <id>` (หรือ `/agents`)
- `/session <key>` (หรือ `/sessions`)
- `/model <provider/model>` (หรือ `/models`)

การควบคุมเซสชัน:

- `/think <off|minimal|low|medium|high>`
- `/fast <status|on|off>`
- `/verbose <on|full|off>`
- `/trace <on|off>`
- `/reasoning <on|off|stream>`
- `/usage <off|tokens|full>`
- `/elevated <on|off|ask|full>` (ชื่อแฝง: `/elev`)
- `/activation <mention|always>`
- `/deliver <on|off>`

วงจรชีวิตเซสชัน:

- `/new` หรือ `/reset` (รีเซ็ตเซสชัน)
- `/abort` (ยกเลิกการรันที่ใช้งานอยู่)
- `/settings`
- `/exit`

เฉพาะโหมดภายในเครื่องเท่านั้น:

- `/auth [provider]` เปิดโฟลว์การยืนยันตัวตน/เข้าสู่ระบบของผู้ให้บริการภายใน TUI

คำสั่งสแลชอื่นของ Gateway (เช่น `/context`) จะถูกส่งต่อไปยัง Gateway และแสดงเป็นเอาต์พุตระบบ ดู [คำสั่งสแลช](/th/tools/slash-commands)

## คำสั่งเชลล์ภายในเครื่อง

- ใส่ `!` นำหน้าบรรทัดเพื่อเรียกใช้คำสั่งเชลล์ภายในเครื่องบนโฮสต์ TUI
- TUI จะถามหนึ่งครั้งต่อเซสชันเพื่ออนุญาตการดำเนินการภายในเครื่อง; หากปฏิเสธ `!` จะยังถูกปิดใช้งานสำหรับเซสชัน
- คำสั่งทำงานในเชลล์ใหม่แบบไม่โต้ตอบในไดเรกทอรีทำงานของ TUI (ไม่มี `cd`/env แบบคงอยู่)
- คำสั่งเชลล์ภายในเครื่องจะได้รับ `OPENCLAW_SHELL=tui-local` ในสภาพแวดล้อม
- `!` เพียงตัวเดียวจะถูกส่งเป็นข้อความปกติ; ช่องว่างนำหน้าไม่เรียกการดำเนินการภายในเครื่อง

## ซ่อมแซมคอนฟิกจาก TUI ภายในเครื่อง

ใช้โหมดภายในเครื่องเมื่อคอนฟิกปัจจุบันผ่านการตรวจสอบแล้ว และคุณต้องการให้
เอเจนต์แบบฝังตรวจสอบบนเครื่องเดียวกัน เปรียบเทียบกับเอกสาร
และช่วยซ่อมแซมความคลาดเคลื่อนโดยไม่ต้องพึ่ง Gateway ที่กำลังทำงานอยู่

หาก `openclaw config validate` ล้มเหลวอยู่แล้ว ให้เริ่มด้วย `openclaw configure`
หรือ `openclaw doctor --fix` ก่อน `openclaw chat` ไม่ได้ข้ามตัวป้องกันคอนฟิก-
ไม่ถูกต้อง

ลูปทั่วไป:

1. เริ่มโหมดภายในเครื่อง:

```bash
openclaw chat
```

2. ถามเอเจนต์ว่าคุณต้องการตรวจสอบอะไร เช่น:

```text
Compare my gateway auth config with the docs and suggest the smallest fix.
```

3. ใช้คำสั่งเชลล์ภายในเครื่องเพื่อหลักฐานและการตรวจสอบที่แม่นยำ:

```text
!openclaw config file
!openclaw docs gateway auth token secretref
!openclaw config validate
!openclaw doctor
```

4. ใช้การเปลี่ยนแปลงที่แคบด้วย `openclaw config set` หรือ `openclaw configure` แล้วเรียกใช้ `!openclaw config validate` อีกครั้ง
5. หาก Doctor แนะนำการย้ายข้อมูลหรือซ่อมแซมอัตโนมัติ ให้ตรวจทานและเรียกใช้ `!openclaw doctor --fix`

เคล็ดลับ:

- ควรใช้ `openclaw config set` หรือ `openclaw configure` แทนการแก้ไข `openclaw.json` ด้วยมือ
- `openclaw docs "<query>"` ค้นหาดัชนีเอกสารสดจากเครื่องเดียวกัน
- `openclaw config validate --json` มีประโยชน์เมื่อคุณต้องการข้อผิดพลาดของสคีมาแบบมีโครงสร้างและ SecretRef/ความสามารถในการแก้ค่า

## เอาต์พุตเครื่องมือ

- การเรียกเครื่องมือจะแสดงเป็นการ์ดพร้อมอาร์กิวเมนต์ + ผลลัพธ์
- Ctrl+O สลับระหว่างมุมมองย่อ/ขยาย
- ขณะเครื่องมือทำงาน การอัปเดตบางส่วนจะสตรีมเข้าไปในการ์ดเดียวกัน

## สีเทอร์มินัล

- TUI คงข้อความเนื้อหาของผู้ช่วยไว้เป็นสีพื้นหน้าค่าเริ่มต้นของเทอร์มินัล เพื่อให้เทอร์มินัลทั้งแบบมืดและสว่างยังอ่านได้
- หากเทอร์มินัลของคุณใช้พื้นหลังสว่างและการตรวจจับอัตโนมัติผิด ให้ตั้งค่า `OPENCLAW_THEME=light` ก่อนเปิด `openclaw tui`
- หากต้องการบังคับใช้พาเล็ตสีเข้มดั้งเดิมแทน ให้ตั้งค่า `OPENCLAW_THEME=dark`

## ประวัติ + การสตรีม

- เมื่อเชื่อมต่อ TUI จะโหลดประวัติล่าสุด (ค่าเริ่มต้น 200 ข้อความ)
- คำตอบแบบสตรีมจะอัปเดตอยู่กับที่จนกว่าจะสรุปเสร็จ
- TUI ยังฟังเหตุการณ์เครื่องมือของเอเจนต์เพื่อให้ได้การ์ดเครื่องมือที่สมบูรณ์ยิ่งขึ้น

## รายละเอียดการเชื่อมต่อ

- TUI ลงทะเบียนกับ Gateway เป็น `mode: "tui"`
- การเชื่อมต่อใหม่จะแสดงข้อความระบบ; ช่องว่างของเหตุการณ์จะแสดงในบันทึก

## ตัวเลือก

- `--local`: รันกับรันไทม์เอเจนต์แบบฝังภายในเครื่อง
- `--url <url>`: URL WebSocket ของ Gateway (ค่าเริ่มต้นจากคอนฟิกหรือ `ws://127.0.0.1:<port>`)
- `--token <token>`: โทเค็น Gateway (หากจำเป็น)
- `--password <password>`: รหัสผ่าน Gateway (หากจำเป็น)
- `--session <key>`: คีย์เซสชัน (ค่าเริ่มต้น: `main` หรือ `global` เมื่อขอบเขตเป็น global)
- `--deliver`: นำส่งคำตอบของผู้ช่วยไปยังผู้ให้บริการ (ค่าเริ่มต้นปิด)
- `--thinking <level>`: แทนที่ระดับการคิดสำหรับการส่ง
- `--message <text>`: ส่งข้อความเริ่มต้นหลังเชื่อมต่อ
- `--timeout-ms <ms>`: หมดเวลาเอเจนต์เป็นมิลลิวินาที (ค่าเริ่มต้นเป็น `agents.defaults.timeoutSeconds`)
- `--history-limit <n>`: รายการประวัติที่จะโหลด (ค่าเริ่มต้น `200`)

<Warning>
เมื่อคุณตั้งค่า `--url` TUI จะไม่ถอยกลับไปใช้ข้อมูลประจำตัวจากคอนฟิกหรือสภาพแวดล้อม ส่ง `--token` หรือ `--password` อย่างชัดเจน การขาดข้อมูลประจำตัวที่ระบุชัดเจนถือเป็นข้อผิดพลาด ในโหมดภายในเครื่อง อย่าส่ง `--url`, `--token` หรือ `--password`
</Warning>

## การแก้ไขปัญหา

ไม่มีเอาต์พุตหลังส่งข้อความ:

- เรียกใช้ `/status` ใน TUI เพื่อยืนยันว่า Gateway เชื่อมต่ออยู่และว่าง/ไม่ว่าง
- ตรวจสอบบันทึก Gateway: `openclaw logs --follow`
- ยืนยันว่าเอเจนต์รันได้: `openclaw status` และ `openclaw models status`
- หากคุณคาดว่าจะมีข้อความในช่องแชต ให้เปิดใช้งานการนำส่ง (`/deliver on` หรือ `--deliver`)

## การแก้ไขปัญหาการเชื่อมต่อ

- `disconnected`: ตรวจสอบให้แน่ใจว่า Gateway กำลังทำงาน และ `--url/--token/--password` ของคุณถูกต้อง
- ไม่มีเอเจนต์ในตัวเลือก: ตรวจสอบ `openclaw agents list` และคอนฟิกการกำหนดเส้นทางของคุณ
- ตัวเลือกเซสชันว่าง: คุณอาจอยู่ในขอบเขต global หรือยังไม่มีเซสชัน
