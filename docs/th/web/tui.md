## เริ่มต้นอย่างรวดเร็ว

### โหมด Gateway

1. เริ่ม Gateway

```bash
openclaw gateway
```

2. เปิด TUI

```bash
openclaw tui
```

3. พิมพ์ข้อความแล้วกด Enter

Gateway ระยะไกล:

```bash
openclaw tui --url ws://<host>:<port> --token <gateway-token>
```

ใช้ `--password` หาก Gateway ใช้การยืนยันตัวตนด้วยรหัสผ่าน

### โหมดภายในเครื่อง

เรียกใช้ TUI โดยไม่ใช้ Gateway:

```bash
openclaw chat
# หรือ
openclaw tui --local
```

- `openclaw chat` และ `openclaw terminal` เป็นนามแฝงของ `openclaw tui --local`
- `--local` ไม่สามารถใช้ร่วมกับ `--url`, `--token` หรือ `--password`
- โหมดภายในเครื่องใช้รันไทม์เอเจนต์แบบฝังโดยตรง เครื่องมือภายในเครื่องส่วนใหญ่ใช้งานได้ แต่ฟีเจอร์ที่ใช้ได้เฉพาะกับ Gateway จะไม่พร้อมใช้งาน
- `openclaw` แบบเดี่ยว (ไม่มีคำสั่งย่อย) จะเลือกเป้าหมายโดยอัตโนมัติ: การติดตั้งที่ยังไม่ได้กำหนดค่าจะเริ่มขั้นตอนเริ่มต้นใช้งานการอนุมาน; การกำหนดค่าที่ไม่ถูกต้องจะเปิดคำแนะนำ Doctor แบบคลาสสิก; Gateway ที่กำหนดค่าไว้และเข้าถึงได้จะเปิดเชลล์ TUI นี้ในโหมด Gateway; มิฉะนั้น โมเดลภายในเครื่องที่กำหนดค่าไว้จะเปิดในโหมดภายในเครื่อง

## สิ่งที่เห็น

- ส่วนหัว: URL การเชื่อมต่อ เอเจนต์ปัจจุบัน เซสชันปัจจุบัน
- บันทึกการสนทนา: ข้อความของผู้ใช้ คำตอบของผู้ช่วย การแจ้งเตือนจากระบบ การ์ดเครื่องมือ
- บรรทัดสถานะ: สถานะการเชื่อมต่อ/การทำงาน (กำลังเชื่อมต่อ กำลังทำงาน กำลังสตรีม ไม่ได้ใช้งาน ข้อผิดพลาด)
- ส่วนท้าย: เอเจนต์ + เซสชัน + โมเดล + สถานะเป้าหมาย + think/fast/verbose/trace/reasoning + จำนวนโทเค็น + การส่งต่อ
- ช่องป้อนข้อมูล: โปรแกรมแก้ไขข้อความพร้อมการเติมข้อความอัตโนมัติ

## แบบจำลองทางความคิด: เอเจนต์ + เซสชัน

- เอเจนต์คือ slug ที่ไม่ซ้ำกัน (เช่น `main`, `research`) Gateway จะแสดงรายการนี้
- เซสชันเป็นของเอเจนต์ปัจจุบัน
- คีย์เซสชันจัดเก็บในรูปแบบ `agent:<agentId>:<sessionKey>`
  - หากพิมพ์ `/session main` TUI จะขยายเป็น `agent:<currentAgent>:main`
  - หากพิมพ์ `/session agent:other:main` จะสลับไปยังเซสชันของเอเจนต์นั้นอย่างชัดเจน
- ขอบเขตเซสชัน:
  - `per-sender` (ค่าเริ่มต้น): แต่ละเอเจนต์มีหลายเซสชัน
  - `global`: TUI จะใช้เซสชัน `global` เสมอ (ตัวเลือกอาจว่างเปล่า)
- เอเจนต์และเซสชันปัจจุบันจะแสดงในส่วนท้ายเสมอ
- หากเซสชันมี[เป้าหมาย](/th/tools/goal) ส่วนท้ายจะแสดงสถานะแบบย่อ:
  `Pursuing goal`, `Goal paused (/goal resume)`, `Goal blocked (/goal resume)` หรือ `Goal achieved`
- เมื่อเริ่มโดยไม่มี `--session` TUI ในโหมด Gateway จะกลับมาใช้เซสชันที่เลือกล่าสุดสำหรับ Gateway เอเจนต์ และขอบเขตเซสชันเดียวกัน หากเซสชันนั้นยังมีอยู่ การระบุ `--session`, `/session`, `/new` หรือ `/reset` ยังคงเป็นการเลือกอย่างชัดเจน

## การส่งข้อความ + การส่งต่อ

- ข้อความจะถูกส่งไปยัง Gateway เสมอ (หรือรันไทม์แบบฝังในโหมดภายในเครื่อง) ส่วนการส่งคำตอบของผู้ช่วยกลับออกไปยังผู้ให้บริการแชตเป็นอีกขั้นตอนหนึ่ง ซึ่งปิดไว้โดยค่าเริ่มต้น
- TUI เป็นพื้นผิวต้นทางภายในเช่นเดียวกับ WebChat ไม่ใช่ช่องทางขาออกทั่วไป ชุดควบคุมที่ต้องใช้ `tools.message` เพื่อแสดงคำตอบ สามารถดำเนินรอบ TUI ที่ใช้งานอยู่ให้เสร็จด้วย `message.send` ที่ไม่มีเป้าหมายได้; การส่งต่อไปยังผู้ให้บริการอย่างชัดเจนยังคงใช้ช่องทางที่กำหนดค่าตามปกติ และจะไม่ย้อนกลับไปใช้ `lastChannel`
- การส่งต่อถูกกำหนดตายตัวสำหรับทั้งเซสชัน TUI ตั้งแต่เริ่มต้น: เริ่มด้วย `openclaw tui --deliver` เพื่อเปิดใช้งาน ไม่มีคำสั่งแบบสแลช `/deliver` หรือสวิตช์ Settings สำหรับสลับระหว่างเซสชัน; ให้เริ่ม TUI ใหม่เพื่อเปลี่ยนค่า

## ตัวเลือก + หน้าต่างซ้อน

- ตัวเลือกโมเดล: แสดงโมเดลที่พร้อมใช้งานและตั้งค่าการเขียนทับของเซสชัน
- ตัวเลือกเอเจนต์: เลือกเอเจนต์อื่น
- ตัวเลือกเซสชัน: แสดงเซสชันสูงสุด 50 รายการของเอเจนต์ปัจจุบันที่อัปเดตในช่วง 7 วันที่ผ่านมา ใช้ `/session <key>` เพื่อไปยังเซสชันเก่าที่ทราบ
- Settings (`/settings`): สลับการขยายผลลัพธ์เครื่องมือและการมองเห็นกระบวนการคิด แผงนี้ไม่ควบคุมการส่งต่อ

## คำสั่งแบบสแลช

คำสั่งหลัก:

- `/help`
- `/status` (ส่งต่อไปยัง Gateway; แสดงข้อมูลสรุปเซสชัน/โมเดล)
- `/gateway-status` (นามแฝง `/gwstatus`; แสดงสถานะการเชื่อมต่อ Gateway โดยตรง)
- `/agent <id>` (หรือ `/agents`)
- `/session <key>` (หรือ `/sessions`)
- `/model <provider/model>` (หรือ `/models`)

การควบคุมเซสชัน:

- `/think <off|minimal|low|medium|high>` (ระดับที่สูงกว่าอาจเพิ่มระดับ เช่น `xhigh`/`max` โดยขึ้นอยู่กับโมเดล)
- `/fast <status|auto|on|off>`
- `/verbose <on|full|off>`
- `/trace <on|off>`
- `/reasoning <on|off|stream>`
- `/usage <off|tokens|full|reset>` (`reset`/`inherit`/`clear`/`default` จะล้างการเขียนทับของเซสชัน)
- `/goal [status] | /goal start <objective> | /goal edit <objective> | /goal pause|resume|complete|block|clear`
- `/elevated <on|off|ask|full>` (นามแฝง: `/elev`)
- `/activation <mention|always>`
- `/queue <steer|followup|collect|interrupt> [debounce:<duration>] [cap:<n>] [drop:<summarize|old|new>]`
- `/queue default` (หรือ `/queue reset`) จะล้างการเขียนทับของเซสชัน

วงจรชีวิตเซสชัน:

- `/new` (สร้างเซสชันใหม่ที่แยกจากกันภายใต้คีย์ใหม่; ไม่ส่งผลต่อไคลเอนต์ TUI อื่นในเซสชันเดิม)
- `/reset` (รีเซ็ตคีย์เซสชันปัจจุบันโดยไม่เปลี่ยนตำแหน่ง)
- `/abort` (ยกเลิกการทำงานที่กำลังดำเนินอยู่)
- `/settings`
- `/exit` (หรือ `/quit`)

เฉพาะโหมดภายในเครื่อง:

- `/auth [provider]` เปิดขั้นตอนการยืนยันตัวตน/เข้าสู่ระบบของผู้ให้บริการภายใน TUI

โหมดภายในเครื่องใช้โหมดคิวแบบเดียวกันภายในรันไทม์แบบฝัง พรอมต์ที่ส่งระหว่าง
การทำงานจะเป็นไปตามนโยบาย `/queue` ของเซสชัน: `steer` จะแทรกเมื่อ
รันไทม์สามารถรับได้, `followup` จะรอรอบแยกต่างหาก, `collect` จะรวม
พรอมต์ที่รอดำเนินการ และ `interrupt` จะหยุดการทำงานปัจจุบันก่อนเริ่ม
รายการใหม่ `/steer <message>` แบบชัดเจนใช้ได้เฉพาะกับ Gateway; ในโหมดภายในเครื่องให้ใช้ `/queue steer` แล้วตามด้วย
ข้อความปกติ

OpenClaw:

- `/openclaw [request]` จะกลับจาก TUI ของเอเจนต์ปกติไปยังแชตตั้งค่า/ซ่อมแซม [OpenClaw](#openclaw-setup-and-repair-helper) และเลือกส่งต่อคำขอหนึ่งรายการได้

คำสั่งแบบสแลชอื่นของ Gateway (เช่น `/context`) จะถูกส่งต่อไปยัง Gateway และแสดงเป็นผลลัพธ์ของระบบ ดู[คำสั่งแบบสแลช](/th/tools/slash-commands)

## คำสั่งเชลล์ภายในเครื่อง

- เติม `!` หน้าบรรทัดเพื่อเรียกใช้คำสั่งเชลล์ภายในเครื่องบนโฮสต์ TUI
- TUI จะถามหนึ่งครั้งต่อเซสชันเพื่ออนุญาตการดำเนินการภายในเครื่อง; หากปฏิเสธ `!` จะยังคงปิดใช้งานตลอดเซสชัน
- คำสั่งจะทำงานในเชลล์ใหม่แบบไม่โต้ตอบในไดเรกทอรีทำงานของ TUI (ไม่มี `cd`/สภาพแวดล้อมที่คงอยู่)
- คำสั่งเชลล์ภายในเครื่องจะได้รับ `OPENCLAW_SHELL=tui-local` ในสภาพแวดล้อม
- `!` ที่อยู่เดี่ยวจะถูกส่งเป็นข้อความปกติ; ช่องว่างนำหน้าไม่ทำให้เกิดการดำเนินการภายในเครื่อง

## ตัวช่วยตั้งค่าและซ่อมแซม OpenClaw

OpenClaw คือผู้ช่วยตั้งค่า/ซ่อมแซมระดับวงแหวนศูนย์ ซึ่งเปิดให้ใช้งานเป็น `openclaw setup` หลังจากโมเดลเริ่มต้นที่กำหนดค่าผ่านการตรวจสอบการอนุมานจริง หากการอนุมานไม่พร้อมใช้งาน การเรียกใช้แบบโต้ตอบจะกลับไปยังขั้นตอนเริ่มต้นใช้งานการอนุมาน และระบบอัตโนมัติจะล้มเหลวพร้อมคำแนะนำการซ่อมแซม โดยทำงานภายในเชลล์ TUI ภายในเครื่องเดียวกับ `openclaw tui --local` และมีเอเจนต์ AI ที่ถูกจำกัดให้ใช้การดำเนินการแบบมีชนิดข้อมูลของ OpenClaw ซึ่งต้องผ่านการอนุมัติ:

```bash
openclaw setup                       # เริ่มแบบโต้ตอบ
openclaw setup -m "status"           # เรียกใช้หนึ่งคำขอแล้วออก
openclaw setup -m "set default model openai/gpt-5.2" --yes   # ใช้การเขียนค่าการกำหนดค่า
```

- การเขียนค่าการกำหนดค่าแบบถาวรต้องได้รับการอนุมัติ: ยืนยันแบบโต้ตอบหรือระบุ `--yes`
- `--json` จะแสดงภาพรวมการเริ่มต้นเป็น JSON แทนการเริ่มแชต
- จากภายใน OpenClaw คำขอ `open-tui` (เช่น ขอสนทนากับเอเจนต์ปกติ) จะออกจาก OpenClaw และเปิด TUI ของเอเจนต์ปกติ; ใช้ `/openclaw` ที่นั่นเพื่อกลับมา

ใช้โหมดภายในเครื่องเมื่อการกำหนดค่าปัจจุบันผ่านการตรวจสอบแล้ว และต้องการให้เอเจนต์แบบฝังตรวจสอบการกำหนดค่าบนเครื่องเดียวกัน เปรียบเทียบกับเอกสาร และช่วยแก้ไขค่าที่คลาดเคลื่อนโดยไม่ต้องพึ่ง Gateway ที่กำลังทำงาน

หาก `openclaw config validate` ล้มเหลวอยู่แล้ว ให้เริ่มด้วย `openclaw configure` หรือ `openclaw doctor --fix` ก่อน; `openclaw chat` ยังคงต้องใช้การกำหนดค่าที่โหลดได้เพื่อเริ่มทำงาน

วงจรการทำงานทั่วไป:

1. เริ่มโหมดภายในเครื่อง:

```bash
openclaw chat
```

2. บอกเอเจนต์ว่าต้องการตรวจสอบอะไร ตัวอย่างเช่น:

```text
เปรียบเทียบการกำหนดค่าการยืนยันตัวตนของ Gateway ของฉันกับเอกสาร และแนะนำวิธีแก้ไขที่เล็กที่สุด
```

3. ใช้คำสั่งเชลล์ภายในเครื่องเพื่อรับหลักฐานและตรวจสอบความถูกต้องอย่างแม่นยำ:

```text
!openclaw config file
!openclaw docs gateway auth token secretref
!openclaw config validate
!openclaw doctor
```

4. ใช้การเปลี่ยนแปลงเฉพาะจุดด้วย `openclaw config set` หรือ `openclaw configure` จากนั้นเรียกใช้ `!openclaw config validate` อีกครั้ง
5. หาก Doctor แนะนำการย้ายข้อมูลหรือซ่อมแซมอัตโนมัติ ให้ตรวจสอบแล้วเรียกใช้ `!openclaw doctor --fix`

เคล็ดลับ:

- ควรใช้ `openclaw config set` หรือ `openclaw configure` แทนการแก้ไข `openclaw.json` ด้วยตนเอง
- `openclaw docs "<query>"` ค้นหาดัชนีเอกสารที่ใช้งานจริงจากเครื่องเดียวกัน
- `openclaw config validate --json` มีประโยชน์เมื่อต้องการสคีมาที่มีโครงสร้างและข้อผิดพลาด SecretRef/ความสามารถในการแก้ไขค่า

## ผลลัพธ์เครื่องมือ

- การเรียกเครื่องมือจะแสดงเป็นการ์ดพร้อมอาร์กิวเมนต์และผลลัพธ์
- Ctrl+O สลับระหว่างมุมมองยุบ/ขยาย
- ขณะที่เครื่องมือทำงาน การอัปเดตบางส่วนจะสตรีมเข้าสู่การ์ดเดียวกัน

## สีของเทอร์มินัล

- TUI จะแสดงข้อความเนื้อหาของผู้ช่วยด้วยสีพื้นหน้าค่าเริ่มต้นของเทอร์มินัล เพื่อให้อ่านได้ทั้งบนเทอร์มินัลพื้นมืดและพื้นสว่าง
- หากเทอร์มินัลใช้พื้นหลังสีสว่างและการตรวจหาอัตโนมัติไม่ถูกต้อง ให้ตั้งค่า `OPENCLAW_THEME=light` ก่อนเปิด `openclaw tui`
- หากต้องการบังคับใช้ชุดสีเข้มดั้งเดิมแทน ให้ตั้งค่า `OPENCLAW_THEME=dark`

## ประวัติ + การสตรีม

- เมื่อเชื่อมต่อ TUI จะโหลดประวัติล่าสุด (ค่าเริ่มต้น 200 ข้อความ)
- คำตอบแบบสตรีมจะอัปเดตในตำแหน่งเดิมจนกว่าจะเสร็จสมบูรณ์
- TUI ยังรับฟังเหตุการณ์เครื่องมือของเอเจนต์เพื่อแสดงการ์ดเครื่องมือที่มีรายละเอียดมากขึ้น

## รายละเอียดการเชื่อมต่อ

- TUI เชื่อมต่อด้วยรหัสไคลเอนต์ `openclaw-tui` ภายใต้โหมดไคลเอนต์ระดับกว้าง `ui` (เป็นโหมดเดียวกับที่ Control UI และ WebChat ใช้สำหรับนโยบาย Gateway)
- เมื่อเชื่อมต่อใหม่ ระบบจะแสดงข้อความระบบ; ช่วงที่เหตุการณ์ขาดหายจะแสดงในบันทึก

## ตัวเลือก

- `--local`: เรียกใช้กับรันไทม์เอเจนต์แบบฝังในเครื่อง
- `--url <url>`: URL WebSocket ของ Gateway (ค่าเริ่มต้นคือ `gateway.remote.url` จากการกำหนดค่า หรือ `ws://127.0.0.1:<port>` บนลูปแบ็ก)
- `--token <token>`: โทเค็นของ Gateway (หากจำเป็น)
- `--password <password>`: รหัสผ่านของ Gateway (หากจำเป็น)
- `--tls-fingerprint <sha256>`: ลายนิ้วมือใบรับรอง TLS ที่คาดไว้สำหรับ Gateway `wss://` ที่ตรึงใบรับรอง
- `--session <key>`: คีย์เซสชัน (ค่าเริ่มต้น: `main` หรือ `global` เมื่อขอบเขตเป็นส่วนกลาง)
- `--deliver`: ส่งคำตอบของผู้ช่วยไปยังผู้ให้บริการ (ปิดเป็นค่าเริ่มต้น)
- `--thinking <level>`: แทนที่ระดับการคิดสำหรับการส่ง
- `--message <text>`: ส่งข้อความเริ่มต้นหลังจากเชื่อมต่อ
- `--timeout-ms <ms>`: ระยะหมดเวลาของเอเจนต์เป็นมิลลิวินาที (ค่าเริ่มต้นคือ `agents.defaults.timeoutSeconds`)
- `--history-limit <n>`: จำนวนรายการประวัติที่จะโหลด (ค่าเริ่มต้น `200`)

<Warning>
เมื่อตั้งค่า `--url` TUI จะไม่ย้อนกลับไปใช้ข้อมูลประจำตัวจากการกำหนดค่าหรือตัวแปรสภาพแวดล้อม ให้ระบุ `--token` หรือ `--password` อย่างชัดเจน รวมถึง `--tls-fingerprint` เมื่อปลายทางใช้ใบรับรองที่ตรึงไว้ การไม่ระบุข้อมูลประจำตัวอย่างชัดเจนถือเป็นข้อผิดพลาด ในโหมดภายในเครื่อง อย่าระบุ `--url`, `--token`, `--password` หรือ `--tls-fingerprint`
</Warning>

## การแก้ไขปัญหา

ไม่มีเอาต์พุตหลังจากส่งข้อความ:

- เรียกใช้ `/status` ใน TUI เพื่อยืนยันว่า Gateway เชื่อมต่อแล้วและอยู่ในสถานะว่าง/ไม่ว่าง
- ตรวจสอบบันทึกของ Gateway: `openclaw logs --follow`
- ยืนยันว่าเอเจนต์สามารถทำงานได้: `openclaw status` และ `openclaw models status`
- หากคาดว่าจะได้รับข้อความในช่องแชต ให้ยืนยันว่า TUI เริ่มทำงานด้วย `--deliver` (ไม่สามารถเปิดใช้ภายหลังได้หากไม่เริ่มระบบใหม่)

## การแก้ไขปัญหาการเชื่อมต่อ

- `disconnected`: ตรวจสอบว่า Gateway กำลังทำงานและ `--url/--token/--password` ของคุณถูกต้อง
- ไม่มีเอเจนต์ในตัวเลือก: ตรวจสอบ `openclaw agents list` และการกำหนดค่าการกำหนดเส้นทางของคุณ
- ตัวเลือกเซสชันว่างเปล่า: คุณอาจอยู่ในขอบเขตส่วนกลางหรือยังไม่มีเซสชัน
