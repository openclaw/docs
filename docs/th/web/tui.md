## เริ่มต้นอย่างรวดเร็ว

### โหมด Gateway

1. เริ่ม Gateway

```bash
openclaw gateway
```

2. เปิด TUI

```bash
openclaw tui
```

3. พิมพ์ข้อความแล้วกด Enter

Gateway ระยะไกล:

```bash
openclaw tui --url ws://<host>:<port> --token <gateway-token>
```

ใช้ `--password` หาก Gateway ของคุณใช้การยืนยันตัวตนด้วยรหัสผ่าน

### โหมดภายในเครื่อง

เรียกใช้ TUI โดยไม่ใช้ Gateway:

```bash
openclaw chat
# หรือ
openclaw tui --local
```

- `openclaw chat` และ `openclaw terminal` เป็นนามแฝงของ `openclaw tui --local`
- ไม่สามารถใช้ `--local` ร่วมกับ `--url`, `--token` หรือ `--password` ได้
- โหมดภายในเครื่องใช้รันไทม์เอเจนต์แบบฝังโดยตรง เครื่องมือภายในเครื่องส่วนใหญ่ใช้งานได้ แต่ฟีเจอร์ที่ใช้ได้เฉพาะกับ Gateway จะไม่พร้อมใช้งาน
- การเรียก `openclaw` เปล่า ๆ (ไม่มีคำสั่งย่อย) จะเลือกเป้าหมายโดยอัตโนมัติ: การติดตั้งที่ยังไม่ได้กำหนดค่าจะเริ่มขั้นตอนเริ่มต้นใช้งานการอนุมาน, การกำหนดค่าที่ไม่ถูกต้องจะเปิดคำแนะนำแบบดั้งเดิมของ Doctor, Gateway ที่กำหนดค่าไว้และเข้าถึงได้จะเปิดเชลล์ TUI นี้ในโหมด Gateway มิฉะนั้นโมเดลภายในเครื่องที่กำหนดค่าไว้จะเปิดเชลล์ในโหมดภายในเครื่อง

## สิ่งที่คุณจะเห็น

- ส่วนหัว: URL การเชื่อมต่อ เอเจนต์ปัจจุบัน และเซสชันปัจจุบัน
- บันทึกแชต: ข้อความของผู้ใช้ คำตอบจากผู้ช่วย การแจ้งเตือนของระบบ และการ์ดเครื่องมือ
- บรรทัดสถานะ: สถานะการเชื่อมต่อ/การทำงาน (กำลังเชื่อมต่อ, กำลังทำงาน, กำลังสตรีม, ว่าง, ข้อผิดพลาด)
- ส่วนท้าย: เอเจนต์ + เซสชัน + โมเดล + สถานะเป้าหมาย + การคิด/เร็ว/ละเอียด/ติดตาม/การให้เหตุผล + จำนวนโทเค็น + การนำส่ง เมื่อเปิดใช้ `tui.footer.showRemoteHost` การเชื่อมต่อ Gateway ระยะไกลจะแสดงโฮสต์ที่เชื่อมต่อด้วย
- ช่องป้อนข้อมูล: โปรแกรมแก้ไขข้อความพร้อมการเติมข้อความอัตโนมัติ

## แบบจำลองแนวคิด: เอเจนต์ + เซสชัน

- เอเจนต์เป็นสลักที่ไม่ซ้ำกัน (เช่น `main`, `research`) Gateway จะแสดงรายการดังกล่าว
- เซสชันเป็นของเอเจนต์ปัจจุบัน
- คีย์เซสชันจัดเก็บในรูปแบบ `agent:<agentId>:<sessionKey>`
  - หากคุณพิมพ์ `/session main` TUI จะขยายเป็น `agent:<currentAgent>:main`
  - หากคุณพิมพ์ `/session agent:other:main` คุณจะสลับไปยังเซสชันของเอเจนต์นั้นอย่างชัดเจน
- ขอบเขตเซสชัน:
  - `per-sender` (ค่าเริ่มต้น): แต่ละเอเจนต์มีได้หลายเซสชัน
  - `global`: TUI จะใช้เซสชัน `global` เสมอ (ตัวเลือกอาจว่างเปล่า)
- เอเจนต์และเซสชันปัจจุบันจะแสดงอยู่ในส่วนท้ายเสมอ
- หากต้องการแสดงโฮสต์ Gateway สำหรับการเชื่อมต่อที่ใช้ URL ซึ่งไม่ใช่ภายในเครื่อง ให้เลือกเปิดใช้ด้วย:

  ```bash
  openclaw config set tui.footer.showRemoteHost true
  ```

  ค่าเริ่มต้นคือ `false` การเชื่อมต่อแบบ local loopback และแบบฝังภายในเครื่องจะไม่แสดงป้ายกำกับโฮสต์

- หากเซสชันมี [เป้าหมาย](/th/tools/goal) ส่วนท้ายจะแสดงสถานะแบบย่อ:
  `กำลังดำเนินตามเป้าหมาย`, `หยุดเป้าหมายชั่วคราว (/goal resume)`, `เป้าหมายติดขัด (/goal resume)` หรือ `บรรลุเป้าหมายแล้ว`
- เมื่อเริ่มโดยไม่มี `--session` TUI ในโหมด Gateway จะกลับมาใช้เซสชันที่เลือกล่าสุดสำหรับ Gateway เอเจนต์ และขอบเขตเซสชันเดียวกัน หากเซสชันนั้นยังคงอยู่ การระบุ `--session`, `/session`, `/new` หรือ `/reset` ยังคงเป็นการเลือกอย่างชัดเจน

## การส่ง + การนำส่ง

- ข้อความจะถูกส่งไปยัง Gateway เสมอ (หรือรันไทม์แบบฝังในโหมดภายในเครื่อง) ส่วนการนำส่งคำตอบของผู้ช่วยกลับไปยังผู้ให้บริการแชตเป็นอีกขั้นตอนหนึ่งซึ่งปิดไว้โดยค่าเริ่มต้น
- TUI เป็นพื้นผิวต้นทางภายในเช่นเดียวกับ WebChat ไม่ใช่ช่องทางขาออกทั่วไป ชุดควบคุมที่กำหนดให้ใช้ `tools.message` สำหรับคำตอบที่มองเห็นได้ สามารถตอบสนองเทิร์น TUI ที่กำลังใช้งานด้วย `message.send` ที่ไม่ระบุเป้าหมาย ส่วนการนำส่งไปยังผู้ให้บริการอย่างชัดเจนยังคงใช้ช่องทางที่กำหนดค่าตามปกติ และจะไม่ย้อนกลับไปใช้ `lastChannel`
- การนำส่งจะถูกกำหนดคงที่ตลอดเซสชัน TUI ตั้งแต่เริ่มต้น: เริ่มด้วย `openclaw tui --deliver` เพื่อเปิดใช้ ไม่มีคำสั่งทับ `/deliver` หรือสวิตช์การตั้งค่าสำหรับเปลี่ยนระหว่างเซสชัน ให้เริ่ม TUI ใหม่เพื่อเปลี่ยนการตั้งค่านี้

## ตัวเลือก + หน้าต่างซ้อน

- ตัวเลือกโมเดล: แสดงรายการโมเดลที่พร้อมใช้งานและตั้งค่าการแทนที่สำหรับเซสชัน
- ตัวเลือกเอเจนต์: เลือกเอเจนต์อื่น
- ตัวเลือกเซสชัน: แสดงเซสชันของเอเจนต์ปัจจุบันสูงสุด 50 เซสชันที่อัปเดตในช่วง 7 วันที่ผ่านมา ใช้ `/session <key>` เพื่อไปยังเซสชันเก่าที่ทราบคีย์
- การตั้งค่า (`/settings`): สลับการขยายผลลัพธ์เครื่องมือและการมองเห็นกระบวนการคิด แผงนี้ไม่ควบคุมการนำส่ง

## แป้นพิมพ์ลัด

- Enter: ส่งข้อความ
- Esc: ยกเลิกการทำงานที่กำลังดำเนินอยู่
- Ctrl+C: ล้างช่องป้อนข้อมูล (กดสองครั้งเพื่อออก)
- Ctrl+D: ออก
- Ctrl+L: ตัวเลือกโมเดล
- Ctrl+G: ตัวเลือกเอเจนต์
- Ctrl+P: ตัวเลือกเซสชัน
- Ctrl+O: สลับการขยายผลลัพธ์เครื่องมือ
- Ctrl+T: สลับการมองเห็นกระบวนการคิด (โหลดประวัติใหม่)

## คำสั่งทับ

- `/help`

- `/session <key>` (หรือ `/sessions`)
- `/model <provider/model>` (หรือ `/models`)

- `/think <off|minimal|low|medium|high>` (ระดับที่สูงกว่าอาจมีระดับเพิ่มเติม เช่น `xhigh`/`max` โดยขึ้นอยู่กับโมเดล)
- `/fast <status|auto|on|off>`
- `/verbose <on|full|off>`
- `/trace <on|off>`
- `/reasoning <on|off|stream>`
- `/usage <off|tokens|full|reset>` (`reset`/`inherit`/`clear`/`default` จะล้างการแทนที่ของเซสชัน)
- `/goal [status] | /goal start <objective> | /goal edit <objective> | /goal pause|resume|complete|block|clear`
- `/elevated <on|off|ask|full>` (นามแฝง: `/elev`)
- `/activation <mention|always>`

วงจรชีวิตเซสชัน:

- `/new` (สร้างเซสชันใหม่ที่แยกออกจากกันภายใต้คีย์ใหม่ โดยไม่กระทบไคลเอ็นต์ TUI อื่นในเซสชันเดิม)
- `/reset` (รีเซ็ตคีย์เซสชันปัจจุบันโดยไม่เปลี่ยนคีย์)
- `/abort` (ยกเลิกการทำงานที่กำลังดำเนินอยู่)
- `/settings`
- `/exit` (หรือ `/quit`)

เฉพาะโหมดภายในเครื่อง:

- `/auth [provider]` เปิดขั้นตอนการยืนยันตัวตน/เข้าสู่ระบบของผู้ให้บริการภายใน TUI

Crestodian:

- `/crestodian [request]` กลับจาก TUI เอเจนต์ปกติไปยังแชต [Crestodian](#crestodian-setup-and-repair-helper) สำหรับการตั้งค่า/ซ่อมแซม โดยสามารถส่งต่อคำขอหนึ่งรายการได้

คำสั่งทับอื่น ๆ ของ Gateway (ตัวอย่างเช่น `/context`) จะถูกส่งต่อไปยัง Gateway และแสดงเป็นผลลัพธ์ของระบบ ดู [คำสั่งทับ](/th/tools/slash-commands)

## คำสั่งเชลล์ภายในเครื่อง

- เติม `!` หน้าบรรทัดเพื่อเรียกใช้คำสั่งเชลล์ภายในเครื่องบนโฮสต์ TUI
- TUI จะถามหนึ่งครั้งต่อเซสชันเพื่ออนุญาตการทำงานภายในเครื่อง หากปฏิเสธ `!` จะยังคงถูกปิดใช้สำหรับเซสชันนั้น
- คำสั่งจะทำงานในเชลล์ใหม่แบบไม่โต้ตอบภายในไดเรกทอรีทำงานของ TUI (ไม่มีการคงค่า `cd`/ตัวแปรสภาพแวดล้อม)
- คำสั่งเชลล์ภายในเครื่องจะได้รับ `OPENCLAW_SHELL=tui-local` ในสภาพแวดล้อม
- `!` เพียงตัวเดียวจะถูกส่งเป็นข้อความปกติ ช่องว่างนำหน้าจะไม่เรียกใช้คำสั่งภายในเครื่อง

## ผู้ช่วย Crestodian สำหรับการตั้งค่าและซ่อมแซม

Crestodian คือผู้ช่วยการตั้งค่า/ซ่อมแซมระดับวงแหวนศูนย์ ซึ่งเรียกใช้ผ่าน `openclaw crestodian` หลังจากโมเดลเริ่มต้นที่กำหนดค่าผ่านการตรวจสอบการอนุมานจริง หากการอนุมานไม่พร้อมใช้งาน การเรียกใช้แบบโต้ตอบจะกลับไปยังขั้นตอนเริ่มต้นใช้งานการอนุมาน และระบบอัตโนมัติจะล้มเหลวพร้อมคำแนะนำในการซ่อมแซม โดยทำงานภายในเชลล์ TUI ภายในเครื่องเดียวกับ `openclaw tui --local` และใช้เอเจนต์ AI ที่ถูกจำกัดให้ใช้เฉพาะการดำเนินการแบบมีชนิดของ Crestodian ซึ่งต้องผ่านการอนุมัติ:

```bash
openclaw crestodian                       # เริ่มแบบโต้ตอบ
openclaw crestodian -m "status"           # เรียกใช้หนึ่งคำขอแล้วออก
openclaw crestodian -m "set default model openai/gpt-5.2" --yes   # ใช้การเขียนค่าการกำหนดค่า
```

- การเขียนค่าการกำหนดค่าแบบถาวรต้องได้รับการอนุมัติ: ยืนยันแบบโต้ตอบหรือระบุ `--yes`
- `--json` จะแสดงภาพรวมการเริ่มต้นเป็น JSON แทนการเริ่มแชต
- จากภายใน Crestodian คำขอ `open-tui` (ตัวอย่างเช่น การขอสนทนากับเอเจนต์ปกติ) จะออกจาก Crestodian และเปิด TUI เอเจนต์ปกติ ใช้ `/crestodian` ที่นั่นเพื่อกลับมา

ใช้โหมดภายในเครื่องเมื่อการกำหนดค่าปัจจุบันผ่านการตรวจสอบแล้ว และคุณต้องการให้เอเจนต์แบบฝังตรวจสอบการกำหนดค่าบนเครื่องเดียวกัน เปรียบเทียบกับเอกสาร และช่วยแก้ไขความคลาดเคลื่อนโดยไม่ต้องพึ่งพา Gateway ที่กำลังทำงาน

หาก `openclaw config validate` ล้มเหลวอยู่แล้ว ให้เริ่มด้วย `openclaw configure` หรือ `openclaw doctor --fix` ก่อน เนื่องจาก `openclaw chat` ยังคงต้องใช้การกำหนดค่าที่โหลดได้เพื่อเริ่มทำงาน

วงจรการทำงานทั่วไป:

1. เริ่มโหมดภายในเครื่อง:

```bash
openclaw chat
```

2. ขอให้เอเจนต์ตรวจสอบสิ่งที่คุณต้องการ ตัวอย่างเช่น:

```text
เปรียบเทียบการกำหนดค่าการยืนยันตัวตน Gateway ของฉันกับเอกสาร และเสนอวิธีแก้ไขที่เล็กที่สุด
```

3. ใช้คำสั่งเชลล์ภายในเครื่องเพื่อรวบรวมหลักฐานและตรวจสอบความถูกต้องอย่างแม่นยำ:

```text
!openclaw config file
!openclaw docs gateway auth token secretref
!openclaw config validate
!openclaw doctor
```

4. ใช้การเปลี่ยนแปลงแบบจำกัดขอบเขตด้วย `openclaw config set` หรือ `openclaw configure` แล้วเรียก `!openclaw config validate` อีกครั้ง
5. หาก Doctor แนะนำการย้ายข้อมูลหรือการซ่อมแซมอัตโนมัติ ให้ตรวจสอบแล้วเรียกใช้ `!openclaw doctor --fix`

เคล็ดลับ:

- ควรใช้ `openclaw config set` หรือ `openclaw configure` แทนการแก้ไข `openclaw.json` ด้วยตนเอง
- `openclaw docs "<query>"` ค้นหาดัชนีเอกสารปัจจุบันจากเครื่องเดียวกัน
- `openclaw config validate --json` มีประโยชน์เมื่อต้องการข้อผิดพลาดของสคีมาและ SecretRef/ความสามารถในการแก้ไขค่าในรูปแบบมีโครงสร้าง

## ผลลัพธ์เครื่องมือ

- การเรียกใช้เครื่องมือจะแสดงเป็นการ์ดพร้อมอาร์กิวเมนต์ + ผลลัพธ์
- Ctrl+O สลับระหว่างมุมมองยุบ/ขยาย
- ขณะที่เครื่องมือทำงาน การอัปเดตบางส่วนจะสตรีมเข้าสู่การ์ดใบเดิม

## สีของเทอร์มินัล

- TUI จะคงข้อความเนื้อหาของผู้ช่วยไว้เป็นสีพื้นหน้าค่าเริ่มต้นของเทอร์มินัล เพื่อให้ทั้งเทอร์มินัลพื้นหลังมืดและสว่างยังคงอ่านได้
- หากเทอร์มินัลของคุณใช้พื้นหลังสว่างและการตรวจจับอัตโนมัติไม่ถูกต้อง ให้ตั้งค่า `OPENCLAW_THEME=light` ก่อนเปิด `openclaw tui`
- หากต้องการบังคับใช้ชุดสีมืดดั้งเดิมแทน ให้ตั้งค่า `OPENCLAW_THEME=dark`

## ประวัติ + การสตรีม

- เมื่อเชื่อมต่อ TUI จะโหลดประวัติล่าสุด (ค่าเริ่มต้น 200 ข้อความ)
- คำตอบแบบสตรีมจะอัปเดตในตำแหน่งเดิมจนกว่าจะเสร็จสมบูรณ์
- TUI ยังรับฟังเหตุการณ์เครื่องมือของเอเจนต์เพื่อแสดงการ์ดเครื่องมือที่มีข้อมูลสมบูรณ์ยิ่งขึ้น

## รายละเอียดการเชื่อมต่อ

- TUI เชื่อมต่อด้วยรหัสไคลเอ็นต์ `openclaw-tui` ภายใต้โหมดไคลเอ็นต์ระดับกว้าง `ui` (โหมดเดียวกับที่ Control UI และ WebChat ใช้สำหรับนโยบาย Gateway)
- การเชื่อมต่อใหม่จะแสดงข้อความระบบ ส่วนช่วงเหตุการณ์ที่ขาดหายจะแสดงในบันทึก

## ตัวเลือก

- `--local`: ทำงานกับรันไทม์เอเจนต์แบบฝังภายในเครื่อง
- `--url <url>`: URL WebSocket ของ Gateway (ค่าเริ่มต้นคือ `gateway.remote.url` จากการกำหนดค่า หรือ `ws://127.0.0.1:<port>` บน local loopback)
- `--token <token>`: โทเค็น Gateway (หากจำเป็น)
- `--password <password>`: รหัสผ่าน Gateway (หากจำเป็น)
- `--tls-fingerprint <sha256>`: ลายนิ้วมือใบรับรอง TLS ที่คาดไว้สำหรับ Gateway `wss://` ที่ตรึงใบรับรอง
- `--session <key>`: คีย์เซสชัน (ค่าเริ่มต้น: `main` หรือ `global` เมื่อขอบเขตเป็นแบบส่วนกลาง)
- `--deliver`: นำส่งคำตอบของผู้ช่วยไปยังผู้ให้บริการ (ปิดโดยค่าเริ่มต้น)
- `--thinking <level>`: แทนที่ระดับการคิดสำหรับการส่ง
- `--message <text>`: ส่งข้อความเริ่มต้นหลังจากเชื่อมต่อ
- `--timeout-ms <ms>`: ระยะหมดเวลาของเอเจนต์เป็นมิลลิวินาที (ค่าเริ่มต้นคือ `agents.defaults.timeoutSeconds`)
- `--history-limit <n>`: จำนวนรายการประวัติที่จะโหลด (ค่าเริ่มต้น `200`)

<Warning>
เมื่อคุณตั้งค่า `--url` TUI จะไม่ย้อนกลับไปใช้ข้อมูลประจำตัวจากการกำหนดค่าหรือตัวแปรสภาพแวดล้อม ระบุ `--token` หรือ `--password` อย่างชัดเจน รวมถึง `--tls-fingerprint` เมื่อเป้าหมายใช้ใบรับรองที่ตรึงไว้ การไม่ระบุข้อมูลประจำตัวอย่างชัดเจนถือเป็นข้อผิดพลาด ในโหมดภายในเครื่อง อย่าระบุ `--url`, `--token`, `--password` หรือ `--tls-fingerprint`
</Warning>

## การแก้ไขปัญหา

ไม่มีผลลัพธ์หลังจากส่งข้อความ:

- เรียกใช้ `/status` ใน TUI เพื่อยืนยันว่า Gateway เชื่อมต่ออยู่และอยู่ในสถานะว่าง/ไม่ว่าง
- ตรวจสอบบันทึก Gateway: `openclaw logs --follow`
- ยืนยันว่าเอเจนต์ทำงานได้: `openclaw status` และ `openclaw models status`
- หากคุณคาดหวังข้อความในช่องทางแชต ให้ยืนยันว่า TUI เริ่มต้นด้วย `--deliver` (ไม่สามารถเปิดใช้ภายหลังได้โดยไม่เริ่มใหม่)

## การแก้ไขปัญหาการเชื่อมต่อ

- `disconnected`: ตรวจสอบว่า Gateway กำลังทำงานและ `--url/--token/--password` ของคุณถูกต้อง
- ไม่มีเอเจนต์ในตัวเลือก: ตรวจสอบ `openclaw agents list` และการกำหนดค่าการกำหนดเส้นทางของคุณ
- ตัวเลือกเซสชันว่างเปล่า: คุณอาจอยู่ในขอบเขตส่วนกลางหรือยังไม่มีเซสชัน
