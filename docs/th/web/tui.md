## เริ่มต้นอย่างรวดเร็ว

### โหมด Gateway

1. เริ่ม Gateway

```bash
openclaw gateway
```

2. เปิด TUI

```bash
openclaw tui
```

3. พิมพ์ข้อความแล้วกด Enter

Gateway ระยะไกล:

```bash
openclaw tui --url ws://<host>:<port> --token <gateway-token>
```

ใช้ `--password` หาก Gateway ของคุณใช้การรับรองความถูกต้องด้วยรหัสผ่าน

### โหมดในเครื่อง

เรียกใช้ TUI โดยไม่มี Gateway:

```bash
openclaw chat
# or
openclaw tui --local
```

หมายเหตุ:

- `openclaw chat` และ `openclaw terminal` เป็นนามแฝงของ `openclaw tui --local`
- ไม่สามารถใช้ `--local` ร่วมกับ `--url`, `--token` หรือ `--password` ได้
- โหมดในเครื่องใช้รันไทม์เอเจนต์แบบฝังโดยตรง เครื่องมือในเครื่องส่วนใหญ่ใช้งานได้ แต่ฟีเจอร์ที่ใช้ได้เฉพาะ Gateway จะไม่พร้อมใช้งาน
- `openclaw` และ `openclaw crestodian` ก็ใช้เชลล์ TUI นี้ด้วย โดย Crestodian เป็นแบ็กเอนด์แชตสำหรับการตั้งค่าและซ่อมแซมในเครื่อง

## สิ่งที่คุณเห็น

- ส่วนหัว: URL การเชื่อมต่อ, เอเจนต์ปัจจุบัน, เซสชันปัจจุบัน
- บันทึกแชต: ข้อความของผู้ใช้, คำตอบของผู้ช่วย, การแจ้งเตือนระบบ, การ์ดเครื่องมือ
- บรรทัดสถานะ: สถานะการเชื่อมต่อ/การทำงาน (กำลังเชื่อมต่อ, กำลังทำงาน, กำลังสตรีม, ว่าง, ข้อผิดพลาด)
- ส่วนท้าย: สถานะการเชื่อมต่อ + เอเจนต์ + เซสชัน + โมเดล + think/fast/verbose/trace/reasoning + จำนวนโทเค็น + deliver
- ช่องป้อนข้อมูล: ตัวแก้ไขข้อความพร้อมการเติมข้อความอัตโนมัติ

## แบบจำลองความคิด: เอเจนต์ + เซสชัน

- เอเจนต์คือสลักเฉพาะ (เช่น `main`, `research`) Gateway จะแสดงรายการนี้
- เซสชันเป็นของเอเจนต์ปัจจุบัน
- คีย์เซสชันถูกจัดเก็บเป็น `agent:<agentId>:<sessionKey>`
  - หากคุณพิมพ์ `/session main` TUI จะขยายเป็น `agent:<currentAgent>:main`
  - หากคุณพิมพ์ `/session agent:other:main` คุณจะสลับไปยังเซสชันของเอเจนต์นั้นอย่างชัดเจน
- ขอบเขตเซสชัน:
  - `per-sender` (ค่าเริ่มต้น): แต่ละเอเจนต์มีหลายเซสชัน
  - `global`: TUI จะใช้เซสชัน `global` เสมอ (ตัวเลือกอาจว่าง)
- เอเจนต์ + เซสชันปัจจุบันจะแสดงในส่วนท้ายเสมอ

## การส่ง + การนำส่ง

- ข้อความจะถูกส่งไปยัง Gateway; การนำส่งไปยังผู้ให้บริการปิดไว้ตามค่าเริ่มต้น
- เปิดการนำส่ง:
  - `/deliver on`
  - หรือแผงการตั้งค่า
  - หรือเริ่มด้วย `openclaw tui --deliver`

## ตัวเลือก + โอเวอร์เลย์

- ตัวเลือกโมเดล: แสดงรายการโมเดลที่มีและตั้งค่าการแทนที่สำหรับเซสชัน
- ตัวเลือกเอเจนต์: เลือกเอเจนต์อื่น
- ตัวเลือกเซสชัน: แสดงเฉพาะเซสชันของเอเจนต์ปัจจุบัน
- การตั้งค่า: สลับการนำส่ง, การขยายผลลัพธ์เครื่องมือ และการแสดงความคิด

## แป้นพิมพ์ลัด

- Enter: ส่งข้อความ
- Esc: ยกเลิกการทำงานที่กำลังทำงานอยู่
- Ctrl+C: ล้างช่องป้อนข้อมูล (กดสองครั้งเพื่อออก)
- Ctrl+D: ออก
- Ctrl+L: ตัวเลือกโมเดล
- Ctrl+G: ตัวเลือกเอเจนต์
- Ctrl+P: ตัวเลือกเซสชัน
- Ctrl+O: สลับการขยายผลลัพธ์เครื่องมือ
- Ctrl+T: สลับการแสดงความคิด (โหลดประวัติใหม่)

## คำสั่งสแลช

หลัก:

- `/help`
- `/status`
- `/agent <id>` (หรือ `/agents`)
- `/session <key>` (หรือ `/sessions`)
- `/model <provider/model>` (หรือ `/models`)

การควบคุมเซสชัน:

- `/think <off|minimal|low|medium|high>`
- `/fast <status|on|off>`
- `/verbose <on|full|off>`
- `/trace <on|off>`
- `/reasoning <on|off|stream>`
- `/usage <off|tokens|full>`
- `/elevated <on|off|ask|full>` (นามแฝง: `/elev`)
- `/activation <mention|always>`
- `/deliver <on|off>`

วงจรชีวิตเซสชัน:

- `/new` หรือ `/reset` (รีเซ็ตเซสชัน)
- `/abort` (ยกเลิกการทำงานที่กำลังทำงานอยู่)
- `/settings`
- `/exit`

เฉพาะโหมดในเครื่อง:

- `/auth [provider]` เปิดโฟลว์การรับรองความถูกต้อง/เข้าสู่ระบบของผู้ให้บริการภายใน TUI

คำสั่งสแลชอื่นของ Gateway (เช่น `/context`) จะถูกส่งต่อไปยัง Gateway และแสดงเป็นผลลัพธ์ของระบบ ดู [คำสั่งสแลช](/th/tools/slash-commands)

## คำสั่งเชลล์ในเครื่อง

- เติม `!` นำหน้าบรรทัดเพื่อเรียกใช้คำสั่งเชลล์ในเครื่องบนโฮสต์ TUI
- TUI จะถามหนึ่งครั้งต่อเซสชันเพื่ออนุญาตการเรียกใช้ในเครื่อง; หากปฏิเสธ `!` จะถูกปิดใช้งานสำหรับเซสชัน
- คำสั่งทำงานในเชลล์ใหม่แบบไม่โต้ตอบในไดเรกทอรีทำงานของ TUI (ไม่มี `cd`/env ที่คงอยู่)
- คำสั่งเชลล์ในเครื่องจะได้รับ `OPENCLAW_SHELL=tui-local` ในสภาพแวดล้อม
- `!` เดี่ยวจะถูกส่งเป็นข้อความปกติ; ช่องว่างนำหน้าไม่ทริกเกอร์การเรียกใช้ในเครื่อง

## ซ่อมแซมการกำหนดค่าจาก TUI ในเครื่อง

ใช้โหมดในเครื่องเมื่อการกำหนดค่าปัจจุบันผ่านการตรวจสอบแล้ว และคุณต้องการให้
เอเจนต์แบบฝังตรวจสอบบนเครื่องเดียวกัน เปรียบเทียบกับเอกสาร
และช่วยซ่อมแซมความคลาดเคลื่อนโดยไม่ต้องพึ่ง Gateway ที่กำลังทำงาน

หาก `openclaw config validate` ล้มเหลวอยู่แล้ว ให้เริ่มด้วย `openclaw configure`
หรือ `openclaw doctor --fix` ก่อน `openclaw chat` จะไม่ข้ามตัวป้องกันการกำหนดค่า
ที่ไม่ถูกต้อง

ลูปทั่วไป:

1. เริ่มโหมดในเครื่อง:

```bash
openclaw chat
```

2. ถามเอเจนต์ว่าคุณต้องการตรวจสอบอะไร เช่น:

```text
Compare my gateway auth config with the docs and suggest the smallest fix.
```

3. ใช้คำสั่งเชลล์ในเครื่องเพื่อหลักฐานและการตรวจสอบที่แน่นอน:

```text
!openclaw config file
!openclaw docs gateway auth token secretref
!openclaw config validate
!openclaw doctor
```

4. ใช้การเปลี่ยนแปลงแบบแคบด้วย `openclaw config set` หรือ `openclaw configure` จากนั้นเรียกใช้ `!openclaw config validate` อีกครั้ง
5. หาก Doctor แนะนำการย้ายข้อมูลหรือการซ่อมแซมอัตโนมัติ ให้ตรวจทานแล้วเรียกใช้ `!openclaw doctor --fix`

เคล็ดลับ:

- ควรใช้ `openclaw config set` หรือ `openclaw configure` แทนการแก้ไข `openclaw.json` ด้วยมือ
- `openclaw docs "<query>"` ค้นหาดัชนีเอกสารสดจากเครื่องเดียวกัน
- `openclaw config validate --json` มีประโยชน์เมื่อคุณต้องการข้อผิดพลาดของสคีมาแบบมีโครงสร้างและ SecretRef/ความสามารถในการแก้ค่า

## ผลลัพธ์เครื่องมือ

- การเรียกเครื่องมือจะแสดงเป็นการ์ดพร้อมอาร์กิวเมนต์ + ผลลัพธ์
- Ctrl+O สลับระหว่างมุมมองยุบ/ขยาย
- ขณะที่เครื่องมือทำงาน การอัปเดตบางส่วนจะสตรีมเข้าไปในการ์ดเดียวกัน

## สีเทอร์มินัล

- TUI จะคงข้อความเนื้อหาของผู้ช่วยไว้ในสีพื้นหน้าค่าเริ่มต้นของเทอร์มินัล เพื่อให้เทอร์มินัลทั้งแบบมืดและสว่างยังอ่านได้
- หากเทอร์มินัลของคุณใช้พื้นหลังสีสว่างและการตรวจจับอัตโนมัติผิดพลาด ให้ตั้ง `OPENCLAW_THEME=light` ก่อนเปิด `openclaw tui`
- หากต้องการบังคับใช้พาเลตมืดแบบเดิมแทน ให้ตั้ง `OPENCLAW_THEME=dark`

## ประวัติ + การสตรีม

- เมื่อเชื่อมต่อ TUI จะโหลดประวัติล่าสุด (ค่าเริ่มต้น 200 ข้อความ)
- คำตอบแบบสตรีมจะอัปเดตในตำแหน่งเดิมจนกว่าจะเสร็จสมบูรณ์
- TUI ยังฟังเหตุการณ์เครื่องมือของเอเจนต์เพื่อการ์ดเครื่องมือที่สมบูรณ์ขึ้น

## รายละเอียดการเชื่อมต่อ

- TUI ลงทะเบียนกับ Gateway เป็น `mode: "tui"`
- การเชื่อมต่อใหม่จะแสดงข้อความระบบ; ช่องว่างของเหตุการณ์จะปรากฏในบันทึก

## ตัวเลือก

- `--local`: ทำงานกับรันไทม์เอเจนต์แบบฝังในเครื่อง
- `--url <url>`: URL WebSocket ของ Gateway (ค่าเริ่มต้นมาจากการกำหนดค่าหรือ `ws://127.0.0.1:<port>`)
- `--token <token>`: โทเค็น Gateway (หากจำเป็น)
- `--password <password>`: รหัสผ่าน Gateway (หากจำเป็น)
- `--session <key>`: คีย์เซสชัน (ค่าเริ่มต้น: `main` หรือ `global` เมื่อขอบเขตเป็น global)
- `--deliver`: นำส่งคำตอบของผู้ช่วยไปยังผู้ให้บริการ (ค่าเริ่มต้นปิด)
- `--thinking <level>`: แทนที่ระดับการคิดสำหรับการส่ง
- `--message <text>`: ส่งข้อความเริ่มต้นหลังจากเชื่อมต่อ
- `--timeout-ms <ms>`: เวลาหมดเขตของเอเจนต์เป็นมิลลิวินาที (ค่าเริ่มต้นมาจาก `agents.defaults.timeoutSeconds`)
- `--history-limit <n>`: รายการประวัติที่จะโหลด (ค่าเริ่มต้น `200`)

<Warning>
เมื่อคุณตั้ง `--url` TUI จะไม่ถอยกลับไปใช้ข้อมูลประจำตัวจากการกำหนดค่าหรือสภาพแวดล้อม ส่ง `--token` หรือ `--password` อย่างชัดเจน การไม่มีข้อมูลประจำตัวอย่างชัดเจนถือเป็นข้อผิดพลาด ในโหมดในเครื่อง อย่าส่ง `--url`, `--token` หรือ `--password`
</Warning>

## การแก้ไขปัญหา

ไม่มีผลลัพธ์หลังจากส่งข้อความ:

- เรียกใช้ `/status` ใน TUI เพื่อยืนยันว่า Gateway เชื่อมต่ออยู่และว่าง/ไม่ว่าง
- ตรวจสอบบันทึก Gateway: `openclaw logs --follow`
- ยืนยันว่าเอเจนต์ทำงานได้: `openclaw status` และ `openclaw models status`
- หากคุณคาดหวังข้อความในช่องแชต ให้เปิดการนำส่ง (`/deliver on` หรือ `--deliver`)

## การแก้ไขปัญหาการเชื่อมต่อ

- `disconnected`: ตรวจสอบให้แน่ใจว่า Gateway กำลังทำงาน และ `--url/--token/--password` ของคุณถูกต้อง
- ไม่มีเอเจนต์ในตัวเลือก: ตรวจสอบ `openclaw agents list` และการกำหนดค่าการกำหนดเส้นทางของคุณ
- ตัวเลือกเซสชันว่าง: คุณอาจอยู่ในขอบเขต global หรือยังไม่มีเซสชัน
