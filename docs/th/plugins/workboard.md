Plugin Workboard เพิ่มบอร์ดสไตล์คัมบังซึ่งเปิดใช้หรือไม่ก็ได้ให้กับ
[UI ควบคุม](/th/web/control-ui): การ์ดงานขนาดเหมาะสำหรับเอเจนต์ การมอบหมายให้เอเจนต์
และลิงก์กลับไปยังงาน การเรียกใช้ และเซสชันแดชบอร์ดของการ์ด

Workboard ได้รับการออกแบบให้มีขนาดเล็กโดยตั้งใจ โดยติดตามงานปฏิบัติการภายในเครื่องสำหรับ
OpenClaw Gateway หนึ่งรายการ ไม่ได้มีไว้แทน GitHub Issues, Linear, Jira หรือ
ระบบจัดการโครงการของทีมอื่นๆ

## เปิดใช้งาน

Workboard รวมมาให้แล้ว แต่ปิดใช้งานโดยค่าเริ่มต้น:

1. เปิด **Plugins** ใน UI ควบคุม หรือใช้ `/settings/plugins` โดยอ้างอิงจาก
   พาธฐานของ UI ควบคุมที่กำหนดค่าไว้ ตัวอย่างเช่น พาธฐาน `/openclaw`
   จะใช้ `/openclaw/settings/plugins`
2. ค้นหา **Workboard** แล้วเลือก **Enable** เนื่องจาก Workboard รวมอยู่ใน
   OpenClaw จึงไม่ต้องดำเนินการ **Install**
3. หาก UI รายงานว่าจำเป็นต้องเริ่มระบบใหม่ ให้เริ่ม Gateway ใหม่

แท็บ Workboard จะปรากฏในแถบนำทางของแดชบอร์ดหลังจากรันไทม์ของ Plugin โหลดแล้ว
ขณะที่ปิดใช้งาน แท็บจะถูกซ่อนจากการนำทาง การเปิดเส้นทาง
`/workboard` โดยตรงขณะที่ Plugin ปิดใช้งานหรือถูกบล็อกโดย
`plugins.allow`/`plugins.deny` จะแสดงสถานะ Plugin ไม่พร้อมใช้งานแทนข้อมูล
การ์ด

เวิร์กโฟลว์ CLI ที่เทียบเท่าคือ:

```bash
openclaw plugins enable workboard
openclaw gateway restart
openclaw dashboard
```

## การกำหนดค่า

Workboard ไม่มีการกำหนดค่าเฉพาะสำหรับ Plugin ให้เปิดหรือปิดใช้งานด้วยรายการ
Plugin มาตรฐาน:

```json5
{
  plugins: {
    entries: {
      workboard: {
        enabled: true,
        config: {},
      },
    },
  },
}
```

```bash
openclaw plugins disable workboard
openclaw gateway restart
```

## ฟิลด์ของการ์ด

| ฟิลด์       | ค่า                                                                                                           |
| ----------- | ------------------------------------------------------------------------------------------------------------- |
| `status`    | `triage`, `backlog`, `todo`, `scheduled`, `ready`, `running`, `review`, `blocked`, `done`                     |
| `priority`  | `low`, `normal`, `high`, `urgent`                                                                             |
| `labels`    | สตริงรูปแบบอิสระ                                                                                              |
| `agentId`   | เอเจนต์ที่ได้รับมอบหมายซึ่งระบุหรือไม่ก็ได้                                                                   |
| การอ้างอิงที่ลิงก์ไว้ | งาน การเรียกใช้ เซสชัน หรือ URL ต้นทางซึ่งระบุหรือไม่ก็ได้                                          |
| `execution` | เมทาดาทาซึ่งระบุหรือไม่ก็ได้สำหรับการเรียกใช้ Codex/Claude ที่เริ่มจากการ์ด (เอนจิน โหมด โมเดล เซสชัน รหัสการเรียกใช้ สถานะ) |

การ์ดยังมีเมทาดาทาแบบกระชับสำหรับความพยายาม ความคิดเห็น ลิงก์ หลักฐาน
อาร์ติแฟกต์ การตั้งค่าระบบอัตโนมัติ ไฟล์แนบ บันทึกของผู้ปฏิบัติงาน สถานะโพรโทคอล
ของผู้ปฏิบัติงาน การอ้างสิทธิ์ การวินิจฉัย การแจ้งเตือน รหัสเทมเพลต สถานะการเก็บถาวร
และการตรวจหาเซสชันที่ล้าสมัย รวมถึงรายการเหตุการณ์ล่าสุด (`created`, `edited`,
`moved`, `linked`, `specified`, `decomposed`, `claimed`, `heartbeat`,
`execution_updated`, `attempt_started`, `attempt_updated`, `comment_added`,
`link_added`, `proof_added`, `artifact_added`, `attachment_added`,
`diagnostic`, `notification`, `dispatch`, `orchestration`,
`protocol_violation`, `archived`, `unarchived`, `stale`) เมทาดาทานี้ช่วยให้
ผู้ปฏิบัติการเห็นว่าการ์ดเคลื่อนผ่านบอร์ดอย่างไรโดยไม่ต้องเปิดเซสชันที่ลิงก์ไว้
ข้อมูลนี้เป็นบริบทการปฏิบัติการภายในเครื่อง ไม่ได้มีไว้แทนบทถอดความของเซสชัน
หรือประวัติ GitHub Issues

การ์ดจัดเก็บอยู่ในสถานะ Gateway ของ Plugin เอง และย้ายไปพร้อมกับสถานะ OpenClaw
ส่วนที่เหลือของ Gateway นั้น (ดู[พื้นที่จัดเก็บ](#storage))

## การเริ่มงานจากการ์ด

การ์ดที่ไม่ได้ลิงก์สามารถเริ่มงานได้โดยตรง:

- **เรียกใช้ Codex** / **เรียกใช้ Claude** เริ่มการเรียกใช้เอเจนต์ที่ติดตามด้วยงานโดยใช้
  เอนจินที่ระบุอย่างชัดเจน ส่งพรอมต์ของการ์ด และทำเครื่องหมายการ์ดเป็น `running` การเรียกใช้ Codex
  ใช้ `openai/gpt-5.6-sol` ส่วนการเรียกใช้ Claude ใช้ `anthropic/claude-sonnet-4-6`
- **เปิด Codex** / **เปิด Claude** สร้างเซสชันแดชบอร์ดที่ลิงก์ไว้โดยไม่
  ส่งพรอมต์ของการ์ดหรือย้ายการ์ด สำหรับงานที่ดำเนินการด้วยตนเองและยังคง
  เชื่อมอยู่กับบอร์ด

การเริ่มโดยอัตโนมัติใช้เส้นทางการเรียกใช้เอเจนต์ที่ติดตามด้วยงานของ Gateway (ใช้เอเจนต์
และโมเดลเริ่มต้น เว้นแต่จะเลือก Codex/Claude อย่างชัดเจน) จากนั้น Workboard จะลิงก์
งานที่เกิดขึ้น รหัสการเรียกใช้ และคีย์เซสชันกลับไปยังการ์ด การดำเนินการที่ลิงก์ไว้แต่ละครั้ง
ยังบันทึกข้อมูลสรุปความพยายาม (เอนจิน โหมด โมเดล รหัสการเรียกใช้
การประทับเวลา สถานะ จำนวนความล้มเหลวสะสม) เพื่อให้ความล้มเหลวที่เกิดซ้ำยังคงมองเห็นได้

แดชบอร์ดรีเฟรชสถานะงานจากบัญชีแยกประเภทงานของ Gateway โดยจับคู่
งานกับการ์ดตามรหัสงาน รหัสการเรียกใช้ หรือคีย์เซสชันที่ลิงก์ไว้ งานที่อยู่ในคิวหรือกำลังทำงาน
จะทำให้วงจรชีวิตของการ์ดยังคงทำงานอยู่ ส่วนงานที่เสร็จสิ้น ล้มเหลว หมดเวลา หรือ
ถูกยกเลิกจะย้ายการ์ดไปยัง `review` หรือ `blocked` โดยใช้กฎการซิงค์เดียวกับ
เซสชันที่ลิงก์ไว้ (ดู[การซิงค์วงจรชีวิตของเซสชัน](#session-lifecycle-sync))

## เครื่องมือเอเจนต์

| เครื่องมือ                                                                                                                                         | วัตถุประสงค์                                                                                                                                                                                   |
| ------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `workboard_list`                                                                                                                                 | แสดงรายการการ์ดแบบย่อพร้อมสถานะการอ้างสิทธิ์/การวินิจฉัย และรองรับตัวกรองบอร์ดเพิ่มเติม                                                                                                                    |
| `workboard_read`                                                                                                                                 | ส่งคืนการ์ดหนึ่งใบพร้อมบริบทของผู้ปฏิบัติงานที่จำกัดขอบเขต (บันทึก ความพยายาม ความคิดเห็น ลิงก์ หลักฐาน อาร์ติแฟกต์ ผลลัพธ์จากรายการแม่ งานล่าสุดของผู้รับมอบหมาย และการวินิจฉัยที่กำลังทำงาน)                               |
| `workboard_create`                                                                                                                               | สร้างการ์ดโดยระบุรายการแม่ ผู้เช่า Skills บอร์ด เมทาดาทาของพื้นที่ทำงาน คีย์ idempotency ขีดจำกัดเวลารัน และงบประมาณการลองใหม่ได้ตามต้องการ                                                             |
| `workboard_link`                                                                                                                                 | เชื่อมโยงการ์ดแม่กับการ์ดลูก การ์ดลูกจะคงสถานะ `todo` จนกว่ารายการแม่ทั้งหมดจะมีสถานะ `done` จากนั้นการเลื่อนสถานะโดยการจัดสรรงานจะย้ายไปเป็น `ready`                                                     |
| `workboard_claim`                                                                                                                                | อ้างสิทธิ์การ์ดให้แก่เอเจนต์ที่เรียกใช้ และย้าย `backlog`/`todo`/`ready` ไปเป็น `running`                                                                                                        |
| `workboard_heartbeat`                                                                                                                            | รีเฟรช Heartbeat ของการอ้างสิทธิ์ระหว่างการรันที่ใช้เวลานาน                                                                                                                                          |
| `workboard_release`                                                                                                                              | ปล่อยการอ้างสิทธิ์หลังเสร็จสิ้น หยุดพัก หรือส่งมอบงาน และสามารถย้ายการ์ดไปยังสถานะถัดไปได้                                                                                                |
| `workboard_complete` / `workboard_block`                                                                                                         | เครื่องมือวงจรชีวิตแบบมีโครงสร้างสำหรับสรุปขั้นสุดท้าย หลักฐาน อาร์ติแฟกต์ และแมนิเฟสต์ของการ์ดที่สร้างขึ้น (ต้องอ้างอิงการ์ดที่เชื่อมโยงกลับมายังการ์ดที่เสร็จสมบูรณ์) หรือเหตุผลที่ขัดขวางงาน                 |
| `workboard_attachment_add` / `workboard_attachment_read` / `workboard_attachment_delete`                                                         | จัดเก็บไฟล์แนบขนาดเล็กของการ์ดในสถานะ SQLite ของ Plugin สร้างดัชนีบนการ์ด และแสดงในบริบทของผู้ปฏิบัติงาน                                                                                         |
| `workboard_worker_log` / `workboard_protocol_violation`                                                                                          | บันทึกบรรทัดล็อกของผู้ปฏิบัติงานและบล็อกการ์ดเมื่อผู้ปฏิบัติงานอัตโนมัติหยุดลงโดยไม่เรียก `workboard_complete`/`workboard_block`                                                           |
| `workboard_board_create` / `workboard_board_archive` / `workboard_board_delete`                                                                  | จัดการเมทาดาทาของบอร์ดที่จัดเก็บถาวร (ชื่อที่แสดง คำอธิบาย สถานะการเก็บถาวร และพื้นที่ทำงานเริ่มต้น)                                                                                            |
| `workboard_runs`                                                                                                                                 | ส่งคืนประวัติความพยายามรันที่จัดเก็บถาวรของการ์ด                                                                                                                                      |
| `workboard_specify`                                                                                                                              | เปลี่ยนการ์ดคัดแยก/งานค้างแบบคร่าว ๆ ให้เป็นการ์ด `todo` ที่ชัดเจน และบันทึกสรุปข้อกำหนดลงในการ์ด                                                                                      |
| `workboard_decompose`                                                                                                                            | แตกการ์ดแม่สำหรับการประสานงานออกเป็นการ์ดลูกที่เชื่อมโยงกัน โดยสืบทอดเมทาดาทาของบอร์ด/ผู้เช่า และสามารถทำให้การ์ดแม่เสร็จสมบูรณ์พร้อมแมนิเฟสต์ของการ์ดที่สร้างขึ้น                                             |
| `workboard_notify_subscribe` / `workboard_notify_list` / `workboard_notify_events` / `workboard_notify_advance` / `workboard_notify_unsubscribe` | จัดการการสมัครรับการแจ้งเตือน การอ่านเหตุการณ์สามารถเล่นซ้ำได้อย่างปลอดภัย ส่วน `advance` จะเลื่อนเคอร์เซอร์ถาวรเพื่อให้ผู้เรียกทำงานต่อได้โดยไม่สูญเสียหรืออ่านเหตุการณ์การ์ดที่เสร็จสิ้น/ล้มเหลว/ค้างซ้ำ |
| `workboard_boards` / `workboard_stats`                                                                                                           | ตรวจสอบเนมสเปซของบอร์ดและสถิติคิว                                                                                                                                                 |
| `workboard_promote` / `workboard_reassign` / `workboard_reclaim`                                                                                 | กู้คืนหรือส่งมอบงานที่ติดค้าง                                                                                                                                                           |
| `workboard_comment` / `workboard_proof`                                                                                                          | เพิ่มบันทึกการส่งมอบงานหรือแนบข้อมูลอ้างอิงหลักฐาน/อาร์ติแฟกต์                                                                                                                                    |
| `workboard_unblock`                                                                                                                              | ย้ายงานที่ถูกบล็อกกลับไปเป็น `todo`                                                                                                                                                         |
| `workboard_dispatch`                                                                                                                             | กระตุ้นการเลื่อนสถานะตามการขึ้นต่อกันหรือการล้างการอ้างสิทธิ์ที่ค้าง                                                                                                                                       |

การ์ดที่ถูกอ้างสิทธิ์จะปฏิเสธการเปลี่ยนแปลงผ่านเครื่องมือเอเจนต์จากเอเจนต์อื่น เว้นแต่ผู้เรียก
จะถือโทเค็นการอ้างสิทธิ์ที่ส่งคืนโดย `workboard_claim` การ์ดทุกใบที่ส่งคืนโดย
เครื่องมือเอเจนต์หรือการเรียก RPC ของ Gateway จะปกปิด `metadata.claim.token` เป็น `[redacted]`
(ตัวโทเค็นจะถูกส่งคืนเพียงครั้งเดียวที่ระดับบนสุด และส่งคืนจาก `workboard_claim` เท่านั้น)
ดังนั้นผู้ดูแลแดชบอร์ดและเอเจนต์อื่นจึงสามารถตรวจสอบสถานะการอ้างสิทธิ์ได้โดยไม่เคย
เห็นโทเค็นที่ใช้งานได้ การกู้คืนดำเนินการผ่าน
`workboard_promote`/`workboard_reassign`/`workboard_reclaim` ซึ่งไม่
ต้องใช้โทเค็น

## การจัดสรรงาน

การจัดสรรงานทำงานภายใน Gateway เท่านั้น โดยจะไม่สร้างโพรเซสระบบปฏิบัติการใด ๆ ตามอำเภอใจ เซสชัน
เอเจนต์ย่อยตามปกติของ OpenClaw ยังคงเป็นผู้ดำเนินการ การจัดสรรงานหนึ่งรอบจะ:

1. เลื่อนสถานะการ์ดที่การขึ้นต่อกันพร้อมแล้ว
2. บันทึกเมทาดาทาการจัดสรรงานลงในการ์ดที่พร้อม
3. บล็อกการอ้างสิทธิ์ที่หมดอายุหรือการรันที่หมดเวลา
4. ทำเครื่องหมายการ์ดคัดแยกที่กำหนดค่าไว้ในบอร์ดให้เป็นรายการที่อาจใช้สำหรับการประสานงาน
5. อ้างสิทธิ์การ์ดที่พร้อมจำนวนเล็กน้อยและเริ่มการรันของผู้ปฏิบัติงานผ่าน
   รันไทม์เอเจนต์ย่อยของ Gateway

ผู้ปฏิบัติงานจะได้รับบริบทของการ์ดที่จำกัดขอบเขต พร้อมโทเค็นการอ้างสิทธิ์ที่จำเป็นสำหรับส่ง Heartbeat
ทำให้การ์ดเสร็จสมบูรณ์ หรือบล็อกการ์ดผ่านเครื่องมือ Workboard

### การเลือกผู้ปฏิบัติงาน

แต่ละรอบจะเริ่ม **ผู้ปฏิบัติงานได้สูงสุด 3 รายโดยค่าเริ่มต้น** การ์ดที่พร้อมจะเรียงตาม
ลำดับความสำคัญ จากนั้นตามตำแหน่ง และเวลาที่สร้าง แต่ละรอบจะเริ่มการ์ดเพียงหนึ่งใบต่อ
เจ้าของ/เอเจนต์ และข้ามเจ้าของที่มีงานกำลังรันหรืองานตรวจสอบอยู่แล้วบน
บอร์ด การ์ดที่เก็บถาวร การ์ดที่มีการอ้างสิทธิ์อยู่ และการ์ดที่ไม่ได้อยู่ในสถานะ `ready`
จะไม่ถูกเลือกเพื่อเริ่มผู้ปฏิบัติงาน (แต่ยังอาจได้รับผลกระทบจาก
ด้านข้อมูลของการจัดสรรงาน ได้แก่ การล้างการอ้างสิทธิ์ที่ค้าง การเลื่อนสถานะตามการขึ้นต่อกัน และการล้าง
งานที่หมดเวลา)

คีย์เซสชันถูกกำหนดแบบตายตัวต่อบอร์ด/การ์ด ดังนั้นการจัดสรรงานซ้ำจะกำหนดเส้นทาง
กลับไปยังช่องทางผู้ปฏิบัติงานเดิมแทนที่จะสร้างเซสชันที่ไม่เกี่ยวข้อง:

- การ์ดที่ได้รับมอบหมาย: `agent:<agentId>:subagent:workboard-<boardId>-<cardId>`
- การ์ดที่ยังไม่ได้รับมอบหมาย: `subagent:workboard-<boardId>-<cardId>` (Gateway จะเลือก
  เอเจนต์เริ่มต้นที่กำหนดค่าไว้)

หากไม่สามารถเริ่มผู้ปฏิบัติงานได้หลังจากอ้างสิทธิ์การ์ดแล้ว Workboard จะบล็อก
การ์ด ล้างการอ้างสิทธิ์ บันทึกความล้มเหลวในการเริ่มรัน และเพิ่มบรรทัดล็อกของผู้ปฏิบัติงาน
ซึ่งมองเห็นได้ในแดชบอร์ด JSON ของ CLI เครื่องมือเอเจนต์ และข้อมูล
การวินิจฉัยของการ์ด

### จุดเริ่มต้น

- การดำเนินการจัดสรรงานในแดชบอร์ด
- `openclaw workboard dispatch`
- `/workboard dispatch` บนช่องทางที่รองรับคำสั่ง

ทั้งสามรายการใช้รันไทม์เอเจนต์ย่อยของ Gateway เมื่อ Gateway พร้อมใช้งาน
CLI มีทางเลือกสำรองสำหรับผู้ดำเนินการหนึ่งแบบ: หากการเรียก Gateway ล้มเหลวด้วยข้อผิดพลาด
การเชื่อมต่อ/ไม่พร้อมใช้งาน (หรือข้อผิดพลาด `unknown method` สำหรับ Gateway
รุ่นเก่า) และไม่มีเป้าหมาย `--url`/`--token` ที่ระบุอย่างชัดเจน รวมถึงไม่มี Gateway
ระยะไกลที่กำหนดค่าไว้ (`OPENCLAW_GATEWAY_URL` หรือ `gateway.mode: remote`)
CLI จะดำเนินการจัดสรรงานเฉพาะข้อมูลกับสถานะ SQLite ในเครื่อง โดยสามารถเลื่อนสถานะตามการขึ้นต่อกัน
ล้างการอ้างสิทธิ์ที่ค้าง และบล็อกการรันที่หมดเวลาได้ แต่ไม่สามารถเริ่มผู้ปฏิบัติงานได้ ความล้มเหลวด้านการยืนยันตัวตน
สิทธิ์ และการตรวจสอบความถูกต้องจาก Gateway ที่เข้าถึงได้จะไม่ถือว่าไม่พร้อมใช้งาน
แต่จะแสดงเป็นข้อผิดพลาดของคำสั่ง เช่นเดียวกับความล้มเหลวใด ๆ ของ Gateway
เมื่อมีการระบุเป้าหมาย `--url`/`--token` อย่างชัดเจน

เมทาดาทาของบอร์ดสามารถตั้งค่า `autoDecompose`, `autoDecomposePerDispatch`,
`defaultAssignee` และ `orchestratorProfile` ได้ OpenClaw จะบันทึกเจตนานี้และ
แสดงในบริบทของผู้ปฏิบัติงาน แต่การระบุข้อกำหนด/การแตกงานจริงยังคงดำเนินการ
ผ่านเครื่องมือ Workboard ตามปกติ

## CLI และคำสั่งแบบสแลช

```bash
openclaw workboard list [--board <id>] [--status <status>] [--include-archived] [--json]
openclaw workboard create "Fix stale card lifecycle" --priority high --labels bug,workboard
openclaw workboard show <card-id> [--json]
openclaw workboard dispatch [--board <id>] [--json]
```

เอาต์พุตข้อความของ `list` จะซ่อนการ์ดที่เก็บถาวรโดยค่าเริ่มต้น (`--include-archived`
จะเขียนทับพฤติกรรมนี้) ส่วน `--json` จะรวมการ์ดที่เก็บถาวรเสมอ ซึ่งสอดคล้องกับสัญญาการ์ดฉบับเต็ม
ที่สคริปต์เดิมใช้อยู่ `show` ยอมรับคำนำหน้ารหัสที่ไม่กำกวม
`list`, `create` และ `show` จะอ่าน/เขียนสถานะ Plugin ในเครื่องโดยตรงเสมอ
มีเพียง `dispatch` ที่เรียก Gateway ซึ่งกำลังทำงาน พร้อมทางเลือกสำรองตามที่อธิบายไว้ข้างต้น

ดูแฟล็กทั้งหมด เอาต์พุต JSON พฤติกรรมทางเลือกสำรองของ Gateway
การจัดการคำนำหน้ารหัส กฎการเลือกสำหรับการจัดสรรงาน และ
การแก้ไขปัญหาได้ที่ [CLI ของ Workboard](/th/cli/workboard)

`/workboard list`, `/workboard show <card-id>`, `/workboard create <title>`
และ `/workboard dispatch` ทำงานสอดคล้องกับ CLI คำสั่งแสดงรายการและแสดงรายละเอียดเป็นการดำเนินการอ่าน
สำหรับผู้ส่งคำสั่งที่ได้รับอนุญาตทุกคน การสร้างและการจัดสรรงานต้องมีสถานะเจ้าของบน
พื้นผิวแชต หรือใช้ไคลเอนต์ Gateway ที่มี `operator.write`/`operator.admin`

## การซิงค์วงจรชีวิตของเซสชัน

การ์ดสามารถลิงก์ไปยังเซสชันแดชบอร์ดที่มีอยู่ หรือเซสชันที่สร้างขึ้นเมื่อคุณ
เริ่มทำงานจากการ์ด การ์ดที่ลิงก์จะแสดงวงจรชีวิตของเซสชันในตำแหน่งเดียวกัน:
กำลังทำงาน, ค้าง, ลิงก์แล้วแต่ไม่ได้ใช้งาน, เสร็จสิ้น, ล้มเหลว หรือสูญหาย คุณยังสามารถบันทึก
เซสชันที่มีอยู่จากแท็บ Sessions ด้วย **เพิ่มไปยังบอร์ดงาน**; การ์ดจะ
ลิงก์ไปยังเซสชันนั้น ใช้ป้ายกำกับเซสชันหรือพรอมต์ล่าสุดของผู้ใช้เป็นชื่อเรื่อง
และเติมบันทึกเริ่มต้นจากพรอมต์ล่าสุดของผู้ใช้พร้อมคำตอบล่าสุดของผู้ช่วย
เมื่อมีข้อมูล

หากเซสชันที่ลิงก์สูญหาย การ์ดจะยังคงลิงก์ไว้เพื่อเป็นบริบทและ
ยังคงมีตัวควบคุมเริ่มต้นเพื่อเริ่มใหม่ในเซสชันใหม่ หากเซสชันที่ลิงก์ซึ่งกำลังใช้งาน
หยุดรายงานกิจกรรมล่าสุด บอร์ดงานจะทำเครื่องหมายการ์ดเป็น
`stale` และจัดเก็บสถานะนั้นเป็นข้อมูลเมตาจนกว่าวงจรชีวิตจะล้างออก

ขณะที่การ์ดอยู่ในสถานะการทำงานที่กำลังดำเนินอยู่ บอร์ดงานจะติดตามเซสชันที่ลิงก์:

| สถานะเซสชันที่ลิงก์                  | สถานะการ์ด |
| ------------------------------------- | ----------- |
| กำลังใช้งาน                           | `running`   |
| เสร็จสมบูรณ์                          | `review`    |
| ล้มเหลว, ถูกยุติ, หมดเวลา หรือถูกยกเลิก | `blocked`   |

**สถานะการตรวจสอบด้วยตนเองมีสิทธิ์เหนือกว่า** การย้ายการ์ดไปยัง `review`, `blocked` หรือ `done`
จะหยุดการซิงค์อัตโนมัติสำหรับการ์ดนั้นจนกว่าคุณจะย้ายกลับไปยัง `todo` หรือ `running`

การเริ่มการ์ดใช้เซสชัน Gateway ตามปกติ บอร์ดงานจัดเก็บเฉพาะ
ข้อมูลเมตาและลิงก์ของการ์ดเท่านั้น บันทึกบทสนทนา การเลือกโมเดล และวงจรชีวิต
ของการทำงานยังคงอยู่ภายใต้การดูแลของระบบเซสชันปกติ ใช้ **หยุด** บนการ์ดที่ลิงก์
และกำลังทำงานเพื่อยกเลิกการทำงานที่กำลังดำเนินอยู่ โดยบอร์ดงานจะทำเครื่องหมายการ์ดนั้นเป็น `blocked` เพื่อให้
ยังคงมองเห็นได้สำหรับการติดตามผล

การ์ดใหม่สามารถเริ่มต้นจากเทมเพลตบอร์ดงาน (`bugfix`, `docs`, `release`,
`pr_review`, `plugin`) เทมเพลตจะกรอกชื่อเรื่อง บันทึก ป้ายกำกับ และลำดับความสำคัญไว้ล่วงหน้า
โดยรหัสเทมเพลตจะถูกจัดเก็บเป็นข้อมูลเมตาของการ์ด

## ขั้นตอนการทำงานของแดชบอร์ด

1. เปิดแท็บบอร์ดงานใน Control UI
2. สร้างการ์ดโดยระบุชื่อเรื่อง บันทึก ลำดับความสำคัญ ป้ายกำกับ เอเจนต์ที่ไม่บังคับ และ
   เซสชันที่ลิงก์ซึ่งไม่บังคับ หรือเปิด Sessions แล้วเลือก **เพิ่มไปยังบอร์ดงาน**
   สำหรับเซสชันที่มีอยู่
3. ลากการ์ดระหว่างคอลัมน์ หรือโฟกัสตัวควบคุมสถานะแบบกะทัดรัดแล้วใช้
   เมนูหรือ ArrowLeft/ArrowRight
4. เริ่มงานจากการ์ดเพื่อสร้างหรือนำเซสชันแดชบอร์ดกลับมาใช้ใหม่
5. เปิดเซสชันที่ลิงก์จากการ์ดขณะที่เอเจนต์ทำงาน
6. ปล่อยให้การซิงค์วงจรชีวิตย้ายงานที่กำลังทำไปยัง `review`/`blocked` จากนั้นย้าย
   การ์ดไปยัง `done` ด้วยตนเองเมื่อยอมรับแล้ว

## การวินิจฉัย

การวินิจฉัยคำนวณจากข้อมูลเมตาของการ์ดในเครื่อง การตรวจสอบในตัวจะแจ้งเตือน:

| ประเภท                      | เงื่อนไข                                                                        |
| --------------------------- | ------------------------------------------------------------------------------ |
| `stranded_ready`            | การ์ด `todo`/`backlog`/`ready` ที่มอบหมายแล้วและไม่ได้อัปเดตมานานกว่า 1 ชั่วโมง |
| `running_without_heartbeat` | การ์ด `running` ที่ไม่มี Heartbeat การอ้างสิทธิ์หรือการอัปเดตการดำเนินงานนานกว่า 20 นาที |
| `blocked_too_long`          | การ์ด `blocked` ที่ไม่ได้อัปเดตมานานกว่า 24 ชั่วโมง |
| `repeated_failures`         | จำนวนความล้มเหลวที่ติดตามของการ์ดถึง 2 ครั้งขึ้นไป |
| `missing_proof`             | การ์ด `done` ที่ไม่มีหลักฐาน อาร์ติแฟกต์ หรือไฟล์แนบ |
| `orphaned_session`          | การ์ด `running` ที่มี `sessionKey` แต่ไม่มีข้อมูลเมตา `execution` |

## สิทธิ์

เมธอด RPC ของ Gateway อยู่ภายใต้ `workboard.*`:

| ขอบเขต           | เมธอด                                                                                                                                                                                                                                                                                                                                                                            |
| ---------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `operator.read`  | `cards.list`, `cards.export`, `cards.diagnostics`, การแสดงรายการ/รับไฟล์แนบ, การอ่านเหตุการณ์การแจ้งเตือน, `boards.list`, `cards.stats`, `cards.runs`                                                                                                                                                                                                                                       |
| `operator.write` | `cards.diagnostics.refresh`, สร้าง/อัปเดต/ย้าย/ลบ/แสดงความคิดเห็น/ลิงก์/linkDependency/หลักฐาน/อาร์ติแฟกต์, เพิ่ม/ลบไฟล์แนบ, บันทึกของผู้ปฏิบัติงาน, การละเมิดโปรโตคอล, อ้างสิทธิ์/Heartbeat/ปล่อย/promote/มอบหมายใหม่/เรียกคืน/ทำให้เสร็จ/บล็อก/เลิกบล็อก, `cards.dispatch`, `cards.bulk`, เก็บถาวร, `boards.upsert`/`archive`/`delete`, `cards.specify`/`decompose`, สมัครรับ/ลบ/เลื่อนการแจ้งเตือน |

ไม่มีเมธอด RPC ใดต้องใช้ `operator.admin` เบราว์เซอร์ที่เชื่อมต่อด้วยสิทธิ์
ผู้ปฏิบัติงานแบบอ่านอย่างเดียวสามารถตรวจสอบบอร์ดได้ แต่ไม่สามารถแก้ไขการ์ดได้

## พื้นที่จัดเก็บ

บอร์ดงานจัดเก็บข้อมูลถาวรในฐานข้อมูลเชิงสัมพันธ์ SQLite ที่ Plugin เป็นเจ้าของ
ภายใต้ไดเรกทอรีสถานะของ OpenClaw: บอร์ด การ์ด ป้ายกำกับ เหตุการณ์วงจรชีวิต
ความพยายามในการทำงาน ความคิดเห็น ลิงก์การขึ้นต่อกัน หลักฐาน การอ้างอิงอาร์ติแฟกต์
ข้อมูลเมตาและข้อมูลไบนารีของไฟล์แนบ การวินิจฉัย การแจ้งเตือน บันทึกของผู้ปฏิบัติงาน
สถานะโปรโตคอล และการสมัครรับข้อมูลทั้งหมดอยู่ในตารางของบอร์ดงาน (ไม่ใช่
รายการคีย์-ค่าของ Plugin) การส่งออกการ์ดจะคงเรื่องราวของบอร์ดไว้
โดยไม่ฝังเนื้อหาข้อมูลไบนารีของไฟล์แนบ

การติดตั้งที่เคยใช้บอร์ดงานในรุ่น `.28` สามารถเรียกใช้
`openclaw doctor --fix` เพื่อย้ายเนมสเปซสถานะ Plugin แบบเก่าที่เผยแพร่แล้ว
(`workboard.cards`, `workboard.boards`, `workboard.notify` และหากมี
`workboard.attachments`) ไปยังฐานข้อมูลเชิงสัมพันธ์

## การแก้ไขปัญหา

**แท็บแจ้งว่าบอร์ดงานไม่พร้อมใช้งาน**

```bash
openclaw plugins inspect workboard --runtime --json
```

หากกำหนดค่า `plugins.allow` ไว้ ให้เพิ่ม `workboard` ลงไป หาก `plugins.deny`
มี `workboard` ให้ลบออกก่อนเปิดใช้งาน Plugin

**บันทึกการ์ดไม่ได้**

ยืนยันว่าการเชื่อมต่อของเบราว์เซอร์มีสิทธิ์ `operator.write` เซสชันผู้ปฏิบัติงาน
แบบอ่านอย่างเดียวสามารถแสดงรายการการ์ดได้ แต่ไม่สามารถสร้าง แก้ไข ย้าย หรือลบการ์ดได้

**การเริ่มการ์ดไม่เปิดเซสชันที่คาดไว้**

ตรวจสอบรหัสเอเจนต์และเซสชันที่ลิงก์ของการ์ด จากนั้นเปิด Sessions หรือ Chat เพื่อ
ตรวจสอบสถานะการทำงานจริง

**การจัดส่งไม่เริ่มผู้ปฏิบัติงาน**

ยืนยันว่ามีการ์ด `ready` อย่างน้อยหนึ่งใบที่ไม่มีการอ้างสิทธิ์ซึ่งกำลังใช้งาน:

```bash
openclaw workboard list --status ready
```

หาก CLI รายงานการจัดส่งแบบข้อมูลเท่านั้น ให้เริ่มหรือเริ่ม Gateway ใหม่แล้ว
ลองอีกครั้ง การจัดส่งแบบข้อมูลเท่านั้นจะอัปเดตสถานะบอร์ดในเครื่อง แต่ไม่สามารถเริ่ม
การทำงานของผู้ปฏิบัติงานเอเจนต์ย่อยได้ นอกจากนี้ การ์ดอาจถูกข้ามเมื่อมีการ์ดอีกใบของ
เจ้าของหรือเอเจนต์เดียวกันกำลังทำงานหรือรอการตรวจสอบอยู่ ให้ทำงานที่กำลังดำเนินอยู่นั้น
ให้เสร็จ บล็อก หรือปล่อยก่อนจัดส่งงานเพิ่มเติมสำหรับเจ้าของรายเดิม
