การรองรับ Plugin ดั้งเดิมของ Codex ช่วยให้เอเจนต์ OpenClaw ในโหมด Codex ใช้ความสามารถของแอปและ Plugin ของ app-server ของ Codex เองได้ภายในเธรด Codex เดียวกันที่จัดการรอบการทำงานของ OpenClaw

OpenClaw จะไม่แปล Plugin ของ Codex เป็นเครื่องมือไดนามิก `codex_plugin_*` สังเคราะห์ของ OpenClaw การเรียก Plugin จะคงอยู่ในทรานสคริปต์ Codex ดั้งเดิม และ app-server ของ Codex จะเป็นเจ้าของการดำเนินการ MCP ที่มีแอปรองรับ

ใช้หน้านี้หลังจาก [ฮาร์เนส Codex](/th/plugins/codex-harness) พื้นฐานทำงานแล้ว

## ข้อกำหนด

- รันไทม์เอเจนต์ OpenClaw ที่เลือกต้องเป็นฮาร์เนส Codex ดั้งเดิม
- `plugins.entries.codex.enabled` ต้องเป็น true
- `plugins.entries.codex.config.codexPlugins.enabled` ต้องเป็น true
- V1 รองรับเฉพาะ Plugin `openai-curated` ที่การย้ายตรวจพบว่าติดตั้งจากซอร์สในโฮม Codex ต้นทาง
- app-server ของ Codex เป้าหมายต้องมองเห็นรายการมาร์เก็ตเพลส, Plugin และแอปที่คาดไว้ได้

`codexPlugins` ไม่มีผลต่อการรัน PI, การรันผู้ให้บริการ OpenAI ปกติ, การผูกการสนทนา ACP หรือฮาร์เนสอื่น ๆ เพราะเส้นทางเหล่านั้นไม่ได้สร้างเธรด app-server ของ Codex พร้อมคอนฟิก `apps` ดั้งเดิม

## เริ่มใช้งานอย่างรวดเร็ว

แสดงตัวอย่างการย้ายจากโฮม Codex ต้นทาง:

```bash
openclaw migrate codex --dry-run
```

ใช้การย้ายเมื่อแผนดูถูกต้องแล้ว:

```bash
openclaw migrate apply codex --yes
```

การย้ายจะเขียนรายการ `codexPlugins` แบบชัดเจนสำหรับ Plugin ที่เข้าเกณฑ์ และเรียก `plugin/install` ของ app-server ของ Codex สำหรับ Plugin ที่เลือก คอนฟิกที่ย้ายแล้วโดยทั่วไปมีลักษณะดังนี้:

```json5
{
  plugins: {
    entries: {
      codex: {
        enabled: true,
        config: {
          codexPlugins: {
            enabled: true,
            allow_destructive_actions: true,
            plugins: {
              "google-calendar": {
                enabled: true,
                marketplaceName: "openai-curated",
                pluginName: "google-calendar",
              },
            },
          },
        },
      },
    },
  },
}
```

หลังจากเปลี่ยน `codexPlugins` ให้ใช้ `/new`, `/reset` หรือรีสตาร์ต Gateway เพื่อให้เซสชันฮาร์เนส Codex ในอนาคตเริ่มด้วยชุดแอปที่อัปเดตแล้ว

## การตั้งค่า Plugin ดั้งเดิมทำงานอย่างไร

การผสานรวมมีสถานะแยกกันสามสถานะ:

- ติดตั้งแล้ว: Codex มีบันเดิล Plugin ในเครื่องภายในรันไทม์ app-server เป้าหมาย
- เปิดใช้แล้ว: คอนฟิก OpenClaw ยินยอมให้ Plugin พร้อมใช้งานกับรอบการทำงานของฮาร์เนส Codex
- เข้าถึงได้: app-server ของ Codex ยืนยันว่ารายการแอปของ Plugin พร้อมใช้งานสำหรับบัญชีที่ใช้งานอยู่ และสามารถแมปกับอัตลักษณ์ Plugin ที่ย้ายมาได้

การย้ายเป็นขั้นตอนติดตั้ง/ตรวจคุณสมบัติที่คงทน รายการแอปรันไทม์เป็นการตรวจสอบความสามารถในการเข้าถึง จากนั้นการตั้งค่าเซสชันฮาร์เนส Codex จะคำนวณคอนฟิกแอปของเธรดแบบจำกัดสำหรับแอป Plugin ที่เปิดใช้และเข้าถึงได้

คอนฟิกแอปของเธรดจะถูกคำนวณเมื่อ OpenClaw สร้างเซสชันฮาร์เนส Codex หรือแทนที่การผูกเธรด Codex ที่เก่าเกินไป ระบบจะไม่คำนวณใหม่ในทุกรอบการทำงาน

## ขอบเขตการรองรับ V1

V1 ถูกจำกัดโดยตั้งใจ:

- เฉพาะ Plugin `openai-curated` ที่ติดตั้งอยู่แล้วในรายการ app-server ของ Codex ต้นทางเท่านั้นที่มีสิทธิ์ย้าย
- การย้ายจะเขียนอัตลักษณ์ Plugin แบบชัดเจนด้วย `marketplaceName` และ `pluginName`; จะไม่เขียนเส้นทางแคช `marketplacePath` ในเครื่อง
- `codexPlugins.enabled` เป็นสวิตช์เปิดใช้ทั่วระบบ
- ไม่มีไวลด์การ์ด `plugins["*"]` และไม่มีคีย์คอนฟิกที่ให้สิทธิ์ติดตั้งตามอำเภอใจ
- มาร์เก็ตเพลสที่ไม่รองรับ, บันเดิล Plugin ที่แคชไว้, ฮุก และไฟล์คอนฟิก Codex จะถูกเก็บไว้ในรายงานการย้ายเพื่อตรวจสอบด้วยตนเอง

## รายการแอปและความเป็นเจ้าของ

OpenClaw อ่านรายการแอป Codex ผ่าน `app/list` ของ app-server, แคชไว้หนึ่งชั่วโมง และรีเฟรชรายการที่เก่าหรือหายไปแบบอะซิงโครนัส

แอป Plugin จะถูกเปิดเผยเฉพาะเมื่อ OpenClaw สามารถแมปกลับไปยัง Plugin ที่ย้ายมาได้ผ่านความเป็นเจ้าของที่เสถียร:

- id แอปที่ตรงกันจากรายละเอียด Plugin
- ชื่อเซิร์ฟเวอร์ MCP ที่รู้จัก
- เมตาดาต้าที่เสถียรและไม่ซ้ำกัน

ความเป็นเจ้าของที่ตรงกันเฉพาะชื่อที่แสดงหรือกำกวมจะถูกตัดออกจนกว่าการรีเฟรชรายการครั้งถัดไปจะพิสูจน์ความเป็นเจ้าของได้

## คอนฟิกแอปของเธรด

OpenClaw จะแทรกแพตช์ `config.apps` แบบจำกัดสำหรับเธรด Codex:
`_default` ถูกปิดใช้ และเปิดใช้เฉพาะแอปที่เป็นของ Plugin ที่ย้ายมาและเปิดใช้แล้วเท่านั้น

OpenClaw ตั้งค่า `destructive_enabled` ระดับแอปจากนโยบาย `allow_destructive_actions` ส่วนกลางหรือราย Plugin ที่มีผล และให้ Codex บังคับใช้เมตาดาต้าเครื่องมือทำลายล้างจากคำอธิบายประกอบเครื่องมือแอปดั้งเดิมของตน คอนฟิกแอป `_default` ถูกปิดใช้ด้วย `open_world_enabled: false` แอป Plugin ที่เปิดใช้แล้วจะถูกส่งออกด้วย `open_world_enabled: true`; OpenClaw ไม่เปิดเผยปุ่มปรับนโยบาย open-world แยกสำหรับ Plugin และไม่ดูแลรายการปฏิเสธชื่อเครื่องมือทำลายล้างราย Plugin

โหมดอนุมัติเครื่องมือเป็นแบบอัตโนมัติโดยค่าเริ่มต้นสำหรับแอป Plugin เพื่อให้เครื่องมืออ่านที่ไม่ทำลายล้างสามารถทำงานได้โดยไม่ต้องมี UI อนุมัติในเธรดเดียวกัน เครื่องมือทำลายล้างยังคงถูกควบคุมโดยนโยบาย `destructive_enabled` ของแต่ละแอป

## นโยบายการกระทำทำลายล้าง

การร้องขอการดำเนินการทำลายล้างของ Plugin ได้รับอนุญาตโดยค่าเริ่มต้นสำหรับ Plugin Codex ที่ย้ายมา ขณะที่สคีมาที่ไม่ปลอดภัยและความเป็นเจ้าของที่กำกวมยังคงล้มเหลวแบบปิด:

- `allow_destructive_actions` ส่วนกลางมีค่าเริ่มต้นเป็น `true`
- `allow_destructive_actions` ราย Plugin จะเขียนทับนโยบายส่วนกลางสำหรับ Plugin นั้น
- เมื่อนโยบายเป็น `false` OpenClaw จะส่งคืนการปฏิเสธแบบกำหนดผลได้
- เมื่อนโยบายเป็น `true` OpenClaw จะยอมรับอัตโนมัติเฉพาะสคีมาที่ปลอดภัยและสามารถแมปกับคำตอบการอนุมัติได้ เช่น ฟิลด์อนุมัติแบบบูลีน
- อัตลักษณ์ Plugin ที่หายไป, ความเป็นเจ้าของที่กำกวม, id รอบการทำงานที่หายไป, id รอบการทำงานที่ไม่ถูกต้อง หรือสคีมาการร้องขอที่ไม่ปลอดภัย จะถูกปฏิเสธแทนการแสดงพร้อมต์

## การแก้ไขปัญหา

**`auth_required`:** การย้ายติดตั้ง Plugin แล้ว แต่หนึ่งในแอปของ Plugin ยังต้องมีการยืนยันตัวตน รายการ Plugin แบบชัดเจนจะถูกเขียนเป็นปิดใช้จนกว่าคุณจะอนุญาตใหม่และเปิดใช้

**`marketplace_missing` หรือ `plugin_missing`:** app-server ของ Codex เป้าหมายมองไม่เห็นมาร์เก็ตเพลสหรือ Plugin `openai-curated` ที่คาดไว้ ให้รันการย้ายใหม่กับรันไทม์เป้าหมาย หรือตรวจสอบสถานะ Plugin ของ app-server ของ Codex

**`app_inventory_missing` หรือ `app_inventory_stale`:** ความพร้อมของแอปมาจากแคชที่ว่างเปล่าหรือเก่า OpenClaw จะจัดกำหนดการรีเฟรชแบบอะซิงโครนัส และตัดแอป Plugin ออกจนกว่าจะทราบความเป็นเจ้าของและความพร้อม

**`app_ownership_ambiguous`:** รายการแอปตรงกันเฉพาะชื่อที่แสดง ดังนั้นแอปจะไม่ถูกเปิดเผยต่อเธรด Codex

**คอนฟิกเปลี่ยนแล้วแต่เอเจนต์มองไม่เห็น Plugin:** ใช้ `/new`, `/reset` หรือรีสตาร์ต Gateway การผูกเธรด Codex ที่มีอยู่จะเก็บคอนฟิกแอปที่เริ่มต้นไว้ จนกว่า OpenClaw จะสร้างเซสชันฮาร์เนสใหม่หรือแทนที่การผูกที่เก่าเกินไป

**การกระทำทำลายล้างถูกปฏิเสธ:** ตรวจสอบค่า `allow_destructive_actions` ส่วนกลางและราย Plugin แม้นโยบายจะเป็น true สคีมาการร้องขอที่ไม่ปลอดภัยและอัตลักษณ์ Plugin ที่กำกวมยังคงล้มเหลวแบบปิด
