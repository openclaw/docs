Plugin `zoom-meetings` เข้าร่วมลิงก์การประชุม Zoom ในฐานะผู้เข้าร่วมผ่าน Zoom Web App ในโปรไฟล์ Chrome ของ OpenClaw โดยรองรับลิงก์การประชุมภายใต้ `zoom.us/j/...` และโดเมนย่อยของบัญชี เช่น `example.zoom.us/j/...` แต่จะไม่สร้างการประชุม โทรเข้าร่วม ใช้ Zoom Meeting SDK หรือบันทึกการประชุม

## การตั้งค่า

การพูดตอบกลับใช้ข้อกำหนดเบื้องต้นด้านเสียงภายในเครื่องเช่นเดียวกับ [Plugin Google Meet](/th/plugins/google-meet) ได้แก่ macOS, อุปกรณ์เสียงเสมือน `BlackHole 2ch` และ SoX

```bash
brew install blackhole-2ch sox
sudo reboot
system_profiler SPAudioDataType | grep -i BlackHole
command -v sox
```

เปิดใช้งาน Plugin แล้วตรวจสอบการตั้งค่า:

```json5
{
  plugins: {
    entries: {
      "zoom-meetings": {
        enabled: true,
        config: {
          defaultMode: "agent",
          chrome: { guestName: "OpenClaw Agent" },
        },
      },
    },
  },
}
```

```bash
openclaw zoommeetings setup
openclaw zoommeetings join 'https://zoom.us/j/1234567890'
```

ใช้ `chromeNode.node` เพื่อเรียกใช้ Chrome, BlackHole และ SoX บน Node macOS ที่จับคู่ไว้ โดย Node ต้องอนุญาต `zoommeetings.chrome` และ `browser.proxy`

## โหมด

| โหมด         | ลักษณะการทำงาน                                                                    |
| ------------ | --------------------------------------------------------------------------- |
| `agent`      | การถอดเสียงแบบเรียลไทม์ปรึกษาเอเจนต์ OpenClaw ที่กำหนดค่าไว้ และตอบกลับด้วย TTS |
| `bidi`       | โมเดลเสียงแบบเรียลไทม์รับฟังและตอบกลับโดยตรง                        |
| `transcribe` | เข้าร่วมเพื่อสังเกตการณ์เท่านั้น พร้อมสแนปช็อตข้อความถอดเสียงจากคำบรรยายสด                   |

โหมดถอดเสียงจะเปิดใช้คำบรรยายสดของ Zoom หลังได้รับอนุญาตให้เข้าร่วม และบันทึกการแสดงคำบรรยายภายในขอบเขตที่จำกัด การดำเนินการ `transcript` จะส่งคืนบัฟเฟอร์คำบรรยายสำหรับเซสชันการประชุม OpenClaw ที่กำลังใช้งานอยู่

## ข้อจำกัดในการเข้าร่วมในฐานะผู้เข้าร่วม

อะแดปเตอร์เบราว์เซอร์จะเลือก **Join from browser** กรอกชื่อผู้เข้าร่วม ปิดกล้อง กำหนดค่าไมโครโฟนสำหรับโหมดที่เลือก และคลิก **Join** โดย Zoom Web App ทำงานภายใต้ `app.zoom.us` และ Plugin จะให้สิทธิ์ต้นทางดังกล่าวในการใช้ไมโครโฟนและเลือกวิทยากรก่อนนำทาง สถานะระหว่างการโทรใช้ตัวควบคุม Leave ของ Zoom ส่วนสถานะล็อบบี้ การลงชื่อเข้าใช้ รหัสผ่าน CAPTCHA และสิทธิ์อุปกรณ์จะส่งคืนเหตุผลที่ระบุอย่างชัดเจนว่าต้องดำเนินการด้วยตนเอง

นโยบายของโฮสต์และบัญชี Zoom อาจปิดการเข้าร่วมผ่านเบราว์เซอร์ กำหนดให้ยืนยันตัวตนหรือยืนยันอีเมล แสดง CAPTCHA หรือกำหนดให้โฮสต์อนุญาตให้เข้าร่วม ดำเนินการขั้นตอนดังกล่าวให้เสร็จในโปรไฟล์ Chrome ของ OpenClaw แล้วลองตรวจสอบสถานะหรือใช้เสียงพูดอีกครั้ง Plugin จะไม่ข้ามนโยบายของ Zoom

Zoom Web App ผ่านการตรวจสอบการทำงานจริงด้วยการประชุมทดสอบอย่างเป็นทางการของ Zoom ครอบคลุมหน้าคั่นของแอป การป้อนชื่อผู้เข้าร่วมใน iframe ตัวควบคุมไมโครโฟนและกล้องก่อนเข้าร่วม การเข้าร่วม สิทธิ์สื่อของเบราว์เซอร์และ macOS การตรวจจับสถานะระหว่างการโทร การเปิดใช้คำบรรยายสด และการตรวจจับว่าโฮสต์ยุติการประชุมแล้ว สถานะล็อบบี้และการยืนยันตัวตนขึ้นอยู่กับนโยบายของโฮสต์ และยังคงใช้ข้อความสำรองเมื่อไม่มีตัวระบุ DOM ที่เสถียร

## พื้นผิวเครื่องมือและ Gateway

เครื่องมือเอเจนต์ `zoom_meetings` รองรับ `join`, `leave`, `status`, `transcript` และ `speak` เมธอด Gateway ใช้คำนำหน้า `zoommeetings.*` คำสั่งของ Node คือ `zoommeetings.chrome`
