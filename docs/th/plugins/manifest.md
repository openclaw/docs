หน้านี้ครอบคลุม **ไฟล์ manifest ของ Plugin แบบเนทีฟของ OpenClaw** ซึ่งคือ `openclaw.plugin.json` สำหรับโครงสร้างบันเดิลที่เข้ากันได้ (Codex, Claude, Cursor) โปรดดู [บันเดิล Plugin](/th/plugins/bundles)

รูปแบบบันเดิลที่เข้ากันได้จะใช้ไฟล์ manifest ของตนเองแทน:

- บันเดิล Codex: `.codex-plugin/plugin.json`
- บันเดิล Claude: `.claude-plugin/plugin.json` หรือโครงสร้างคอมโพเนนต์เริ่มต้นของ Claude ที่ไม่มีไฟล์ manifest
- บันเดิล Cursor: `.cursor-plugin/plugin.json`

OpenClaw ตรวจหาโครงสร้างเหล่านั้นโดยอัตโนมัติ แต่จะไม่ตรวจสอบความถูกต้องเทียบกับสคีมา `openclaw.plugin.json` ด้านล่าง สำหรับบันเดิลที่เข้ากันได้ OpenClaw จะอ่านเมทาดาทาของบันเดิล ราก Skills ที่ประกาศไว้ รากคำสั่ง Claude ค่าเริ่มต้น `settings.json` ของ Claude ค่าเริ่มต้น LSP ของ Claude และแพ็ก hook ที่รองรับ เมื่อโครงสร้างตรงตามความคาดหมายของรันไทม์ OpenClaw

Plugin แบบเนทีฟของ OpenClaw ทุกตัว **ต้อง** มี `openclaw.plugin.json` ใน **รากของ Plugin** OpenClaw จะอ่านไฟล์นี้เพื่อตรวจสอบความถูกต้องของการกำหนดค่า **โดยไม่เรียกใช้โค้ด Plugin** หากไฟล์ manifest หายไปหรือไม่ถูกต้อง การตรวจสอบความถูกต้องของการกำหนดค่าจะถูกบล็อกและถือว่าเป็นข้อผิดพลาดของ Plugin

โปรดดู [Plugin](/th/tools/plugin) สำหรับคู่มือระบบ Plugin ฉบับเต็ม และ [โมเดลความสามารถ](/th/plugins/architecture#public-capability-model) สำหรับโมเดลความสามารถแบบเนทีฟและคำแนะนำปัจจุบันด้านความเข้ากันได้กับระบบภายนอก

## ไฟล์นี้ทำหน้าที่อะไร

`openclaw.plugin.json` คือเมทาดาทาที่ OpenClaw อ่าน **ก่อนโหลดโค้ด Plugin ของคุณ** ทุกอย่างในไฟล์นี้ต้องสามารถตรวจสอบได้โดยใช้ทรัพยากรน้อยพอโดยไม่ต้องเริ่มรันไทม์ของ Plugin

**ใช้ไฟล์นี้สำหรับ:**

- ข้อมูลประจำตัวของ Plugin การตรวจสอบความถูกต้องของการกำหนดค่า และคำแนะนำ UI สำหรับการกำหนดค่า
- เมทาดาทาสำหรับการยืนยันตัวตน การเริ่มต้นใช้งาน และการตั้งค่า (นามแฝง การเปิดใช้อัตโนมัติ ตัวแปรสภาพแวดล้อมของผู้ให้บริการ ตัวเลือกการยืนยันตัวตน)
- คำแนะนำการเปิดใช้งานสำหรับพื้นผิวของระนาบควบคุม
- การระบุความเป็นเจ้าของตระกูลโมเดลด้วยชื่อย่อ
- สแนปช็อตแบบคงที่ของความเป็นเจ้าของความสามารถ (`contracts`)
- เมทาดาทาของตัวรัน QA ที่โฮสต์ `openclaw qa` แบบใช้ร่วมกันสามารถตรวจสอบได้
- เมทาดาทาการกำหนดค่าเฉพาะช่องทางที่ผสานเข้ากับแค็ตตาล็อกและพื้นผิวการตรวจสอบความถูกต้อง

**อย่าใช้ไฟล์นี้สำหรับ:** การลงทะเบียนพฤติกรรมขณะรันไทม์ การประกาศจุดเริ่มต้นของโค้ด หรือเมทาดาทาการติดตั้ง npm สิ่งเหล่านี้ต้องอยู่ในโค้ด Plugin และ `package.json`

## ตัวอย่างขั้นต่ำ

```json
{
  "id": "voice-call",
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {}
  }
}
```

## ข้อมูลอ้างอิงฟิลด์ระดับบนสุด

| ฟิลด์                                | จำเป็น | ชนิด                         | ความหมาย                                                                                                                                                                                                                                                              |
| ------------------------------------ | -------- | ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `id`                                 | ใช่      | `string`                     | ID มาตรฐานของ Plugin นี่คือ ID ที่ใช้ใน `plugins.entries.<id>`                                                                                                                                                                                                        |
| `configSchema`                       | ใช่      | `object`                     | JSON Schema แบบอินไลน์สำหรับการกำหนดค่าของ Plugin นี้                                                                                                                                                                                                                               |
| `requiresPlugins`                    | ไม่       | `string[]`                   | ID ของ Plugin ที่ต้องติดตั้งด้วยเพื่อให้ Plugin นี้มีผล การค้นหาจะยังคงทำให้โหลด Plugin ได้ แต่จะแจ้งเตือนเมื่อขาด Plugin ที่จำเป็นรายการใดก็ตาม                                                                                                               |
| `enabledByDefault`                   | ไม่       | `true`                       | ระบุให้ Plugin ที่รวมมาในชุดเปิดใช้งานตามค่าเริ่มต้น หากละเว้นหรือกำหนดเป็นค่าอื่นที่ไม่ใช่ `true` Plugin จะยังคงปิดใช้งานตามค่าเริ่มต้น                                                                                                                                               |
| `enabledByDefaultOnPlatforms`        | ไม่       | `string[]`                   | ระบุให้ Plugin ที่รวมมาในชุดเปิดใช้งานตามค่าเริ่มต้นเฉพาะบนแพลตฟอร์ม Node.js ที่ระบุไว้ เช่น `["darwin"]` การกำหนดค่าอย่างชัดเจนยังคงมีลำดับความสำคัญสูงกว่า                                                                                                                                   |
| `legacyPluginIds`                    | ไม่       | `string[]`                   | ID แบบเดิมที่จะถูกปรับให้เป็น ID มาตรฐานของ Plugin นี้                                                                                                                                                                                                                     |
| `autoEnableWhenConfiguredProviders`  | ไม่       | `string[]`                   | ID ของผู้ให้บริการที่ควรเปิดใช้งาน Plugin นี้โดยอัตโนมัติ เมื่อการยืนยันตัวตน การกำหนดค่า หรือการอ้างอิงโมเดลกล่าวถึง ID เหล่านั้น                                                                                                                                                                            |
| `kind`                               | ไม่       | `PluginKind \| PluginKind[]` | ประกาศประเภท Plugin แบบเอกสิทธิ์เฉพาะอย่างน้อยหนึ่งประเภท (`"memory"`, `"context-engine"`) ที่ใช้โดย `plugins.slots.*` Plugin ที่เป็นเจ้าของทั้งสองสล็อตจะประกาศทั้งสองประเภทไว้ในอาร์เรย์เดียว                                                                                                    |
| `channels`                           | ไม่       | `string[]`                   | ID ของช่องทางที่ Plugin นี้เป็นเจ้าของ ใช้สำหรับการค้นหาและตรวจสอบความถูกต้องของการกำหนดค่า                                                                                                                                                                                                |
| `providers`                          | ไม่       | `string[]`                   | ID ของผู้ให้บริการที่ Plugin นี้เป็นเจ้าของ                                                                                                                                                                                                                                         |
| `providerCatalogEntry`               | ไม่       | `string`                     | พาธโมดูลแค็ตตาล็อกผู้ให้บริการแบบน้ำหนักเบา ซึ่งสัมพันธ์กับรากของ Plugin สำหรับข้อมูลเมตาแค็ตตาล็อกผู้ให้บริการที่จำกัดขอบเขตตามไฟล์ manifest และสามารถโหลดได้โดยไม่ต้องเปิดใช้งานรันไทม์ทั้งหมดของ Plugin                                                                                        |
| `modelSupport`                       | ไม่       | `object`                     | ข้อมูลเมตากลุ่มโมเดลแบบย่อที่ไฟล์ manifest เป็นเจ้าของ ซึ่งใช้เพื่อโหลด Plugin โดยอัตโนมัติก่อนรันไทม์                                                                                                                                                                                |
| `modelCatalog`                       | ไม่       | `object`                     | ข้อมูลเมตาแค็ตตาล็อกโมเดลแบบประกาศสำหรับผู้ให้บริการที่ Plugin นี้เป็นเจ้าของ นี่คือสัญญาระดับ Control Plane สำหรับการแสดงรายการแบบอ่านอย่างเดียวในอนาคต การเริ่มต้นใช้งาน ตัวเลือกโมเดล นามแฝง และการระงับ โดยไม่ต้องโหลดรันไทม์ของ Plugin                                                |
| `modelPricing`                       | ไม่       | `object`                     | นโยบายค้นหาราคาภายนอกที่ผู้ให้บริการเป็นเจ้าของ ใช้เพื่อยกเว้นผู้ให้บริการภายในเครื่องหรือโฮสต์เองออกจากแค็ตตาล็อกราคาระยะไกล หรือแมปการอ้างอิงผู้ให้บริการกับ ID แค็ตตาล็อก OpenRouter/LiteLLM โดยไม่ฮาร์ดโค้ด ID ผู้ให้บริการไว้ในแกนหลัก                                                    |
| `modelIdNormalization`               | ไม่       | `object`                     | การล้างนามแฝง/คำนำหน้า ID โมเดลที่ผู้ให้บริการเป็นเจ้าของ ซึ่งต้องทำงานก่อนโหลดรันไทม์ของผู้ให้บริการ                                                                                                                                                                                  |
| `providerEndpoints`                  | ไม่       | `object[]`                   | ข้อมูลเมตาโฮสต์ปลายทาง/baseUrl ที่ไฟล์ manifest เป็นเจ้าของ สำหรับเส้นทางผู้ให้บริการที่แกนหลักต้องจำแนกประเภทก่อนโหลดรันไทม์ของผู้ให้บริการ                                                                                                                                                   |
| `providerRequest`                    | ไม่       | `object`                     | ข้อมูลเมตากลุ่มผู้ให้บริการและความเข้ากันได้ของคำขอแบบประหยัด ซึ่งนโยบายคำขอทั่วไปใช้ก่อนโหลดรันไทม์ของผู้ให้บริการ                                                                                                                                                     |
| `secretProviderIntegrations`         | ไม่       | `Record<string, object>`     | พรีเซ็ตผู้ให้บริการ exec สำหรับ SecretRef แบบประกาศ ซึ่งพื้นผิวการตั้งค่าหรือการติดตั้งสามารถนำเสนอได้โดยไม่ต้องฮาร์ดโค้ดการผสานรวมเฉพาะผู้ให้บริการไว้ในแกนหลัก                                                                                                                            |
| `cliBackends`                        | ไม่       | `string[]`                   | ID แบ็กเอนด์การอนุมานของ CLI ที่ Plugin นี้เป็นเจ้าของ ใช้สำหรับการเปิดใช้งานอัตโนมัติเมื่อเริ่มต้นระบบจากการอ้างอิงการกำหนดค่าที่ชัดเจน                                                                                                                                                                |
| `syntheticAuthRefs`                  | ไม่       | `string[]`                   | การอ้างอิงผู้ให้บริการหรือแบ็กเอนด์ CLI ซึ่งควรตรวจสอบฮุกการยืนยันตัวตนสังเคราะห์ที่ Plugin เป็นเจ้าของ ระหว่างการค้นหาโมเดลแบบ cold ก่อนโหลดรันไทม์                                                                                                                                     |
| `nonSecretAuthMarkers`               | ไม่       | `string[]`                   | ค่าคีย์ API ตัวยึดตำแหน่งที่ Plugin ซึ่งรวมมาในชุดเป็นเจ้าของ โดยแสดงถึงสถานะข้อมูลประจำตัวภายในเครื่อง OAuth หรือข้อมูลประจำตัวจากสภาพแวดล้อมที่ไม่เป็นความลับ                                                                                                                                                       |
| `commandAliases`                     | ไม่       | `object[]`                   | ชื่อคำสั่งที่ Plugin นี้เป็นเจ้าของ ซึ่งควรสร้างการวินิจฉัยการกำหนดค่าและ CLI ที่รับรู้ Plugin ก่อนโหลดรันไทม์                                                                                                                                                       |
| `providerAuthEnvVars`                | ไม่       | `Record<string, string[]>`   | ข้อมูลเมตาตัวแปรสภาพแวดล้อมเพื่อความเข้ากันได้แบบเลิกใช้แล้ว สำหรับการค้นหาการยืนยันตัวตน/สถานะของผู้ให้บริการ สำหรับ Plugin ใหม่ควรใช้ `setup.providers[].envVars` โดย OpenClaw ยังคงอ่านข้อมูลนี้ในช่วงเลิกใช้งาน                                                                                        |
| `providerUsageAuthEnvVars`           | ไม่       | `Record<string, string[]>`   | ข้อมูลประจำตัวของผู้ให้บริการสำหรับการใช้งาน/การเรียกเก็บเงินเท่านั้น OpenClaw ใช้ชื่อเหล่านี้ในการค้นหาการใช้งานและล้างข้อมูลลับ แต่จะไม่ใช้สำหรับการยืนยันตัวตนเพื่อการอนุมาน                                                                                                                                  |
| `providerAuthAliases`                | ไม่       | `Record<string, string>`     | ID ของผู้ให้บริการที่ควรใช้ ID ของผู้ให้บริการรายอื่นซ้ำสำหรับการค้นหาการยืนยันตัวตน เช่น ผู้ให้บริการเขียนโค้ดที่ใช้คีย์ API และโปรไฟล์การยืนยันตัวตนร่วมกับผู้ให้บริการพื้นฐาน                                                                                                                 |
| `channelEnvVars`                     | ไม่       | `Record<string, string[]>`   | ข้อมูลเมตาตัวแปรสภาพแวดล้อมของช่องทางแบบประหยัดที่ OpenClaw สามารถตรวจสอบได้โดยไม่ต้องโหลดโค้ด Plugin ใช้สำหรับพื้นผิวการตั้งค่าช่องทางหรือการยืนยันตัวตนที่ขับเคลื่อนด้วยตัวแปรสภาพแวดล้อม ซึ่งตัวช่วยเริ่มต้นระบบ/กำหนดค่าทั่วไปควรมองเห็น                                                                                   |

## ข้อมูลอ้างอิง catalog

`catalog` ให้คำแนะนำด้านการแสดงผลที่ไม่บังคับแก่เบราว์เซอร์ Plugin โฮสต์อาจไม่ใช้คำแนะนำเหล่านี้ คำแนะนำเหล่านี้จะไม่ติดตั้งหรือเปิดใช้งาน Plugin และไม่เปลี่ยนแปลงพฤติกรรมขณะรันหรือระดับความน่าเชื่อถือของ Plugin

```json
{
  "catalog": {
    "featured": true,
    "order": 10
  }
}
```

| ฟิลด์      | ชนิด      | ความหมาย                                                              |
| ---------- | --------- | -------------------------------------------------------------------------- |
| `featured` | `boolean` | กำหนดว่าพื้นที่แสดง catalog ควรนำเสนอ Plugin นี้หรือไม่                       |
| `order`    | `number`  | คำแนะนำลำดับการแสดงผลแบบน้อยไปมากในกลุ่ม Plugin ที่คัดสรรไว้ โดยค่าที่ต่ำกว่าจะแสดงก่อน |

## ข้อมูลอ้างอิงเมทาดาทาของผู้ให้บริการการสร้าง

ฟิลด์เมทาดาทาของผู้ให้บริการการสร้างอธิบายสัญญาณการตรวจสอบสิทธิ์แบบคงที่สำหรับผู้ให้บริการที่ประกาศในรายการ `contracts.*GenerationProviders` ที่ตรงกัน OpenClaw อ่านฟิลด์เหล่านี้ก่อนโหลดรันไทม์ของผู้ให้บริการ เพื่อให้เครื่องมือหลักตัดสินได้ว่าผู้ให้บริการการสร้างพร้อมใช้งานหรือไม่ โดยไม่ต้องนำเข้า Plugin ของผู้ให้บริการทุกรายการ

ใช้ฟิลด์เหล่านี้สำหรับข้อเท็จจริงเชิงประกาศที่ตรวจสอบได้โดยใช้ทรัพยากรต่ำเท่านั้น การรับส่งข้อมูล การแปลงคำขอ การรีเฟรชโทเค็น การตรวจสอบข้อมูลประจำตัว และพฤติกรรมการสร้างจริงยังคงอยู่ในรันไทม์ของ Plugin

```json
{
  "contracts": {
    "imageGenerationProviders": ["example-image"]
  },
  "imageGenerationProviderMetadata": {
    "example-image": {
      "aliases": ["example-image-oauth"],
      "authProviders": ["example-image"],
      "configSignals": [
        {
          "rootPath": "plugins.entries.example-image.config",
          "overlayPath": "image",
          "mode": {
            "path": "mode",
            "default": "local",
            "allowed": ["local"]
          },
          "requiredAny": ["workflow", "workflowPath"],
          "required": ["promptNodeId"]
        }
      ],
      "authSignals": [
        {
          "provider": "example-image"
        },
        {
          "provider": "example-image-oauth",
          "providerBaseUrl": {
            "provider": "example-image",
            "defaultBaseUrl": "https://api.example.com/v1",
            "allowedBaseUrls": ["https://api.example.com/v1"]
          }
        }
      ]
    }
  }
}
```

รายการเมทาดาทาแต่ละรายการรองรับ:

| ฟิลด์                  | จำเป็น | ชนิด       | ความหมาย                                                                                                                                       |
| ---------------------- | -------- | ---------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| `aliases`              | ไม่       | `string[]` | ID ผู้ให้บริการเพิ่มเติมที่ควรนับเป็นนามแฝงการตรวจสอบสิทธิ์แบบคงที่สำหรับผู้ให้บริการการสร้าง                                                       |
| `authProviders`        | ไม่       | `string[]` | ID ผู้ให้บริการที่ควรนับโปรไฟล์การตรวจสอบสิทธิ์ซึ่งกำหนดค่าไว้เป็นการตรวจสอบสิทธิ์สำหรับผู้ให้บริการการสร้างนี้                                                      |
| `configSignals`        | ไม่       | `object[]` | สัญญาณความพร้อมใช้งานจากการกำหนดค่าเท่านั้นซึ่งตรวจสอบได้โดยใช้ทรัพยากรต่ำ สำหรับผู้ให้บริการในเครื่องหรือที่โฮสต์เอง ซึ่งกำหนดค่าได้โดยไม่ต้องมีโปรไฟล์การตรวจสอบสิทธิ์หรือตัวแปรสภาพแวดล้อม                 |
| `authSignals`          | ไม่       | `object[]` | สัญญาณการตรวจสอบสิทธิ์ที่ระบุอย่างชัดเจน เมื่อมีฟิลด์นี้ สัญญาณเหล่านี้จะแทนที่ชุดสัญญาณเริ่มต้นจาก ID ผู้ให้บริการ, `aliases` และ `authProviders`                     |
| `referenceAudioInputs` | ไม่       | `boolean`  | สำหรับการสร้างวิดีโอเท่านั้น ตั้งค่าเป็น `true` เมื่อผู้ให้บริการยอมรับแอสเซ็ตเสียงอ้างอิง มิฉะนั้น `video_generate` จะซ่อนพารามิเตอร์การอ้างอิงเสียง |

รายการ `configSignals` แต่ละรายการรองรับ:

| ฟิลด์            | จำเป็น | ชนิด       | ความหมาย                                                                                                                                                                             |
| ---------------- | -------- | ---------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `rootPath`       | ใช่      | `string`   | พาธแบบจุดไปยังออบเจ็กต์การกำหนดค่าที่ Plugin เป็นเจ้าของซึ่งต้องตรวจสอบ ตัวอย่างเช่น `plugins.entries.example.config`                                                                                      |
| `overlayPath`    | ไม่       | `string`   | พาธแบบจุดภายในการกำหนดค่าราก ซึ่งออบเจ็กต์ที่พาธนั้นควรซ้อนทับออบเจ็กต์รากก่อนประเมินสัญญาณ ใช้สำหรับการกำหนดค่าเฉพาะความสามารถ เช่น `image`, `video` หรือ `music`   |
| `overlayMapPath` | ไม่       | `string`   | พาธแบบจุดภายในการกำหนดค่าราก ซึ่งค่าออบเจ็กต์แต่ละค่าที่พาธนั้นควรซ้อนทับออบเจ็กต์ราก ใช้สำหรับแมปบัญชีที่มีชื่อ เช่น `accounts` ซึ่งบัญชีใดก็ตามที่กำหนดค่าไว้ควรผ่านเกณฑ์ |
| `required`       | ไม่       | `string[]` | พาธแบบจุดภายในการกำหนดค่าที่มีผล ซึ่งต้องมีค่าที่กำหนดไว้ สตริงต้องไม่ว่าง และออบเจ็กต์กับอาร์เรย์ต้องไม่ว่าง                                                  |
| `requiredAny`    | ไม่       | `string[]` | พาธแบบจุดภายในการกำหนดค่าที่มีผล โดยอย่างน้อยหนึ่งพาธต้องมีค่าที่กำหนดไว้                                                                                                    |
| `mode`           | ไม่       | `object`   | เงื่อนไขตรวจสอบโหมดแบบสตริงที่ไม่บังคับภายในการกำหนดค่าที่มีผล ใช้เมื่อความพร้อมใช้งานจากการกำหนดค่าเท่านั้นใช้ได้กับโหมดเดียว                                                                  |

เงื่อนไขตรวจสอบ `mode` แต่ละรายการรองรับ:

| ฟิลด์        | จำเป็น | ชนิด       | ความหมาย                                                                      |
| ------------ | -------- | ---------- | ---------------------------------------------------------------------------------- |
| `path`       | ไม่       | `string`   | พาธแบบจุดภายในการกำหนดค่าที่มีผล ค่าเริ่มต้นคือ `mode`                          |
| `default`    | ไม่       | `string`   | ค่าโหมดที่จะใช้เมื่อการกำหนดค่าไม่มีพาธดังกล่าว                                  |
| `allowed`    | ไม่       | `string[]` | หากมีฟิลด์นี้ สัญญาณจะผ่านเฉพาะเมื่อโหมดที่มีผลเป็นหนึ่งในค่าเหล่านี้ |
| `disallowed` | ไม่       | `string[]` | หากมีฟิลด์นี้ สัญญาณจะไม่ผ่านเมื่อโหมดที่มีผลเป็นหนึ่งในค่าเหล่านี้       |

รายการ `authSignals` แต่ละรายการรองรับ:

| ฟิลด์             | จำเป็น | ชนิด     | ความหมาย                                                                                                                                                                 |
| ----------------- | -------- | -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `provider`        | ใช่      | `string` | ID ผู้ให้บริการที่จะตรวจสอบในโปรไฟล์การตรวจสอบสิทธิ์ที่กำหนดค่าไว้                                                                                                                             |
| `providerBaseUrl` | ไม่       | `object` | เงื่อนไขตรวจสอบที่ไม่บังคับ ซึ่งทำให้สัญญาณถูกนับเฉพาะเมื่อผู้ให้บริการที่กำหนดค่าไว้และถูกอ้างอิงใช้ URL ฐานที่อนุญาต ใช้เมื่อนามแฝงการตรวจสอบสิทธิ์ใช้ได้กับ API บางรายการเท่านั้น |

เงื่อนไขตรวจสอบ `providerBaseUrl` แต่ละรายการรองรับ:

| ฟิลด์             | จำเป็น | ชนิด       | ความหมาย                                                                                                                                        |
| ----------------- | -------- | ---------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| `provider`        | ใช่      | `string`   | ID การกำหนดค่าผู้ให้บริการที่ควรตรวจสอบ `baseUrl`                                                                                                |
| `defaultBaseUrl`  | ไม่       | `string`   | URL ฐานที่จะสมมติใช้เมื่อการกำหนดค่าผู้ให้บริการไม่มี `baseUrl`                                                                                         |
| `allowedBaseUrls` | ใช่      | `string[]` | URL ฐานที่อนุญาตสำหรับสัญญาณการตรวจสอบสิทธิ์นี้ สัญญาณจะถูกละเว้นเมื่อ URL ฐานที่กำหนดค่าไว้หรือค่าเริ่มต้นไม่ตรงกับค่าที่ปรับให้อยู่ในรูปแบบมาตรฐานเหล่านี้ค่าใดค่าหนึ่ง |

## ข้อมูลอ้างอิงเมทาดาทาของเครื่องมือ

`toolMetadata` ใช้รูปแบบ `configSignals` และ `authSignals` เดียวกับเมทาดาทาของผู้ให้บริการการสร้าง โดยใช้ชื่อเครื่องมือเป็นคีย์ `contracts.tools` ประกาศความเป็นเจ้าของ `toolMetadata` ประกาศหลักฐานความพร้อมใช้งานที่ตรวจสอบได้โดยใช้ทรัพยากรต่ำ เพื่อให้ OpenClaw หลีกเลี่ยงการนำเข้ารันไทม์ของ Plugin เพียงเพื่อให้แฟกทอรีเครื่องมือส่งคืน `null`

```json
{
  "setup": {
    "providers": [
      {
        "id": "example",
        "envVars": ["EXAMPLE_API_KEY"]
      }
    ]
  },
  "contracts": {
    "tools": ["example_search"]
  },
  "toolMetadata": {
    "example_search": {
      "authSignals": [
        {
          "provider": "example"
        }
      ],
      "configSignals": [
        {
          "rootPath": "plugins.entries.example.config",
          "overlayPath": "search",
          "required": ["apiKey"]
        }
      ]
    }
  }
}
```

รายการ `toolMetadata` ยังรองรับ `optional` (ทำเครื่องหมายว่าเครื่องมือนี้ไม่จำเป็นสำหรับการเปิดใช้งาน Plugin) และ `replaySafe` (ทำเครื่องหมายว่าการเรียกใช้เครื่องมือสามารถทำซ้ำได้อย่างปลอดภัยหลังจากเทิร์นของโมเดลที่ไม่สมบูรณ์) เพิ่มเติมจากฟิลด์ `configSignals`/`authSignals` ที่ใช้ร่วมกันข้างต้น

หากเครื่องมือไม่มี `toolMetadata` OpenClaw จะคงพฤติกรรมเดิมไว้และโหลด Plugin เจ้าของเมื่อสัญญาเครื่องมือตรงตามนโยบาย สำหรับเครื่องมือในเส้นทางที่ทำงานบ่อยซึ่งแฟกทอรีขึ้นอยู่กับการตรวจสอบสิทธิ์/การกำหนดค่า ผู้เขียน Plugin ควรประกาศ `toolMetadata` แทนการทำให้แกนหลักนำเข้ารันไทม์เพื่อสอบถาม

## ข้อมูลอ้างอิง providerAuthChoices

รายการ `providerAuthChoices` แต่ละรายการอธิบายตัวเลือกการเริ่มต้นใช้งานหรือการตรวจสอบสิทธิ์หนึ่งรายการ OpenClaw อ่านข้อมูลนี้ก่อนโหลดรันไทม์ของผู้ให้บริการ รายการการตั้งค่าผู้ให้บริการใช้ตัวเลือกใน manifest เหล่านี้ ตัวเลือกการตั้งค่าที่ได้มาจาก descriptor และเมทาดาทาของ catalog การติดตั้ง โดยไม่โหลดรันไทม์ของผู้ให้บริการ

| ฟิลด์                 | จำเป็น | ชนิด                                                                  | ความหมาย                                                                                             |
| --------------------- | -------- | --------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------- |
| `provider`            | ใช่      | `string`                                                              | รหัสผู้ให้บริการที่ตัวเลือกนี้สังกัด                                                                       |
| `method`              | ใช่      | `string`                                                              | รหัสวิธีการยืนยันตัวตนที่จะส่งต่อไป                                                                            |
| `choiceId`            | ใช่      | `string`                                                              | รหัสตัวเลือกการยืนยันตัวตนแบบคงที่ซึ่งใช้โดยขั้นตอนการเริ่มต้นใช้งานและโฟลว์ CLI                                                   |
| `choiceLabel`         | ไม่       | `string`                                                              | ป้ายกำกับที่แสดงต่อผู้ใช้ หากละไว้ OpenClaw จะใช้ `choiceId` แทน                                         |
| `choiceHint`          | ไม่       | `string`                                                              | ข้อความช่วยเหลือสั้นๆ สำหรับตัวเลือก                                                                         |
| `icon`                | ไม่       | URL แบบ HTTPS                                                             | ภาพประกอบที่แสดงข้างตัวเลือกนี้ในไคลเอนต์เริ่มต้นใช้งานที่รองรับ                                         |
| `website`             | ไม่       | URL แบบ HTTPS                                                             | หน้าผลิตภัณฑ์ การลงชื่อเข้าใช้ หรือการติดตั้งที่แสดงโดยไคลเอนต์เริ่มต้นใช้งานที่รองรับ                             |
| `assistantPriority`   | ไม่       | `number`                                                              | ค่าที่ต่ำกว่าจะเรียงก่อนในตัวเลือกแบบโต้ตอบที่ขับเคลื่อนโดยผู้ช่วย                                        |
| `assistantVisibility` | ไม่       | `"visible"` \| `"manual-only"`                                        | ซ่อนตัวเลือกจากตัวเลือกของผู้ช่วย แต่ยังอนุญาตให้เลือกด้วยตนเองผ่าน CLI                         |
| `deprecatedChoiceIds` | ไม่       | `string[]`                                                            | รหัสตัวเลือกแบบเดิมที่ควรเปลี่ยนเส้นทางผู้ใช้มายังตัวเลือกทดแทนนี้                                  |
| `groupId`             | ไม่       | `string`                                                              | รหัสกลุ่มที่ไม่บังคับสำหรับจัดกลุ่มตัวเลือกที่เกี่ยวข้อง                                                           |
| `groupLabel`          | ไม่       | `string`                                                              | ป้ายกำกับที่แสดงต่อผู้ใช้สำหรับกลุ่มนั้น                                                                         |
| `groupHint`           | ไม่       | `string`                                                              | ข้อความช่วยเหลือสั้นๆ สำหรับกลุ่ม                                                                          |
| `onboardingFeatured`  | ไม่       | `boolean`                                                             | แสดงกลุ่มนี้ในระดับแนะนำของตัวเลือกเริ่มต้นใช้งานแบบโต้ตอบ ก่อนรายการ "เพิ่มเติม..." |
| `optionKey`           | ไม่       | `string`                                                              | คีย์ตัวเลือกภายในสำหรับโฟลว์การยืนยันตัวตนแบบแฟล็กเดียวที่เรียบง่าย                                                       |
| `cliFlag`             | ไม่       | `string`                                                              | ชื่อแฟล็ก CLI เช่น `--openrouter-api-key`                                                            |
| `cliOption`           | ไม่       | `string`                                                              | รูปแบบตัวเลือก CLI แบบเต็ม เช่น `--openrouter-api-key <key>`                                              |
| `cliDescription`      | ไม่       | `string`                                                              | คำอธิบายที่ใช้ในวิธีใช้ CLI                                                                             |
| `appGuidedSecret`     | ไม่       | `boolean`                                                             | ข้อมูลลับที่วางหนึ่งรายการร่วมกับค่าเริ่มต้นของผู้ให้บริการเพียงพอสำหรับการตั้งค่าที่แอปแนะนำ                              |
| `appGuidedDiscovery`  | ไม่       | `boolean`                                                             | วิธีการยืนยันตัวตนของรันไทม์ที่ตรงกันเป็นเจ้าของการค้นหาในเครื่องแบบอ่านอย่างเดียวผ่าน `appGuidedSetup`                 |
| `appGuidedAuth`       | ไม่       | `"oauth"` \| `"device-code"`                                          | การเข้าสู่ระบบแบบโต้ตอบที่ผู้ให้บริการเป็นเจ้าของ ซึ่งไคลเอนต์ตั้งค่าแบบเนทีฟสามารถเรนเดอร์ในรูปแบบทั่วไปได้                        |
| `onboardingScopes`    | ไม่       | `Array<"text-inference" \| "image-generation" \| "music-generation">` | ตัวเลือกนี้ควรปรากฏในพื้นผิวการเริ่มต้นใช้งานใดบ้าง หากละไว้ ค่าเริ่มต้นคือ `["text-inference"]`  |

เมื่อ `appGuidedDiscovery` เป็น true วิธีการยืนยันตัวตนของผู้ให้บริการที่ตรงกันต้องเปิดเผย
`appGuidedSetup.detect` และ `appGuidedSetup.prepare` การตรวจหาต้องเป็นแบบ
อ่านอย่างเดียว: ห้ามเข้าสู่ระบบ ดึงโมเดล ดาวน์โหลด หรือเขียนการกำหนดค่า การเตรียมการจะตรวจสอบ
โมเดลที่เลือกไว้อย่างเจาะจงอีกครั้งและส่งคืนข้อเสนอการกำหนดค่า โดย OpenClaw จะทดสอบข้อเสนอนั้น
แบบสดโดยแยกส่วน และยืนยันการเปลี่ยนแปลงหลังจากสำเร็จเท่านั้น

## ข้อมูลอ้างอิง commandAliases

ใช้ `commandAliases` เมื่อ Plugin เป็นเจ้าของชื่อคำสั่งรันไทม์ที่ผู้ใช้อาจใส่ใน `plugins.allow` โดยเข้าใจผิด หรือพยายามเรียกใช้เป็นคำสั่ง CLI ระดับราก OpenClaw ใช้ข้อมูลเมตานี้เพื่อวินิจฉัยโดยไม่ต้องนำเข้าโค้ดรันไทม์ของ Plugin

```json
{
  "commandAliases": [
    {
      "name": "dreaming",
      "kind": "runtime-slash",
      "cliCommand": "memory"
    }
  ]
}
```

| ฟิลด์        | จำเป็น | ชนิด              | ความหมาย                                                           |
| ------------ | -------- | ----------------- | ----------------------------------------------------------------------- |
| `name`       | ใช่      | `string`          | ชื่อคำสั่งที่เป็นของ Plugin นี้                               |
| `kind`       | ไม่       | `"runtime-slash"` | ระบุว่านามแฝงนี้เป็นคำสั่งแชตแบบสแลช ไม่ใช่คำสั่ง CLI ระดับราก |
| `cliCommand` | ไม่       | `string`          | คำสั่ง CLI ระดับรากที่เกี่ยวข้องซึ่งจะแนะนำสำหรับการดำเนินการ CLI หากมี  |

## ข้อมูลอ้างอิง activation

ใช้ `activation` เมื่อ Plugin สามารถประกาศได้โดยมีต้นทุนต่ำว่าเหตุการณ์ในระนาบควบคุมใดควรรวม Plugin นี้ไว้ในแผนการเปิดใช้งาน/โหลด

บล็อกนี้เป็นข้อมูลเมตาสำหรับตัววางแผน ไม่ใช่ API วงจรชีวิต บล็อกนี้ไม่ลงทะเบียนพฤติกรรมรันไทม์ ไม่แทนที่ `register(...)` และไม่รับประกันว่าโค้ด Plugin ได้ทำงานแล้ว ตัววางแผนการเปิดใช้งานใช้ฟิลด์เหล่านี้เพื่อจำกัด Plugin ที่เป็นตัวเลือก ก่อนจะย้อนกลับไปใช้ข้อมูลเมตาความเป็นเจ้าของจากไฟล์ manifest ที่มีอยู่ เช่น `providers`, `channels`, `commandAliases`, `setup.providers`, `contracts.tools` และฮุก

ควรเลือกใช้ข้อมูลเมตาที่แคบที่สุดซึ่งอธิบายความเป็นเจ้าของอยู่แล้ว ใช้ `providers`, `channels`, `commandAliases`, ตัวอธิบายการตั้งค่า หรือ `contracts` เมื่อฟิลด์เหล่านั้นแสดงความสัมพันธ์ได้ ใช้ `activation` สำหรับคำใบ้เพิ่มเติมแก่ตัววางแผนซึ่งไม่สามารถแสดงด้วยฟิลด์ความเป็นเจ้าของเหล่านั้น ใช้ `cliBackends` ระดับบนสุดสำหรับนามแฝงรันไทม์ CLI เช่น `claude-cli`, `my-cli` หรือ `google-gemini-cli`; `activation.onAgentHarnesses` ใช้เฉพาะสำหรับรหัสชุดควบคุมเอเจนต์แบบฝังที่ยังไม่มีฟิลด์ความเป็นเจ้าของเท่านั้น

Plugin ทุกตัวควรกำหนด `activation.onStartup` อย่างตั้งใจ กำหนดเป็น `true` เฉพาะเมื่อ Plugin ต้องทำงานระหว่างการเริ่มต้น Gateway กำหนดเป็น `false` เมื่อ Plugin ไม่มีการทำงานขณะเริ่มต้นและควรโหลดจากทริกเกอร์ที่แคบกว่าเท่านั้น การละ `onStartup` จะไม่ทำให้ Plugin ถูกโหลดขณะเริ่มต้นโดยปริยายอีกต่อไป ให้ใช้ข้อมูลเมตาการเปิดใช้งานที่ชัดเจนสำหรับทริกเกอร์การเปิดใช้งานขณะเริ่มต้น ช่องทาง การกำหนดค่า ชุดควบคุมเอเจนต์ หน่วยความจำ หรือทริกเกอร์อื่นที่แคบกว่า

```json
{
  "activation": {
    "onStartup": false,
    "onProviders": ["openai"],
    "onCommands": ["models"],
    "onChannels": ["web"],
    "onRoutes": ["gateway-webhook"],
    "onConfigPaths": ["browser"],
    "onCapabilities": ["provider", "tool"]
  }
}
```

| ฟิลด์              | จำเป็น | ชนิด                                                 | ความหมาย                                                                                                                                                                               |
| ------------------ | -------- | ---------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `onStartup`        | ไม่       | `boolean`                                            | การเปิดใช้งานเมื่อเริ่มต้น Gateway อย่างชัดเจน Plugin ทุกตัวควรกำหนดค่านี้ `true` จะนำเข้า Plugin ระหว่างการเริ่มต้น ส่วน `false` จะคงการโหลดแบบหน่วงไว้ในช่วงเริ่มต้น เว้นแต่ทริกเกอร์อื่นที่ตรงกันจะกำหนดให้ต้องโหลด |
| `onProviders`      | ไม่       | `string[]`                                           | รหัสผู้ให้บริการที่ควรรวม Plugin นี้ไว้ในแผนการเปิดใช้งาน/โหลด                                                                                                                      |
| `onAgentHarnesses` | ไม่       | `string[]`                                           | รหัสรันไทม์ของชุดควบคุมเอเจนต์แบบฝังที่ควรรวม Plugin นี้ไว้ในแผนการเปิดใช้งาน/โหลด ใช้ `cliBackends` ระดับบนสุดสำหรับนามแฝงแบ็กเอนด์ CLI                                           |
| `onCommands`       | ไม่       | `string[]`                                           | รหัสคำสั่งที่ควรรวม Plugin นี้ไว้ในแผนการเปิดใช้งาน/โหลด                                                                                                                       |
| `onChannels`       | ไม่       | `string[]`                                           | รหัสช่องทางที่ควรรวม Plugin นี้ไว้ในแผนการเปิดใช้งาน/โหลด                                                                                                                       |
| `onRoutes`         | ไม่       | `string[]`                                           | ชนิดเส้นทางที่ควรรวม Plugin นี้ไว้ในแผนการเปิดใช้งาน/โหลด                                                                                                                       |
| `onConfigPaths`    | ไม่       | `string[]`                                           | พาธการกำหนดค่าที่สัมพันธ์กับราก ซึ่งเมื่อมีพาธนั้นอยู่และไม่ได้ถูกปิดใช้งานอย่างชัดเจน ควรรวม Plugin นี้ไว้ในแผนการเริ่มต้น/โหลด                                                      |
| `onCapabilities`   | ไม่       | `Array<"provider" \| "channel" \| "tool" \| "hook">` | คำใบ้ความสามารถแบบกว้างที่ใช้โดยการวางแผนการเปิดใช้งานในระนาบควบคุม ควรเลือกใช้ฟิลด์ที่แคบกว่าเมื่อทำได้                                                                                     |

ส่วนที่ใช้งานอยู่ในปัจจุบัน:

- การวางแผนการเริ่มต้น Gateway ใช้ `activation.onStartup` สำหรับการนำเข้าขณะเริ่มต้นอย่างชัดเจน
- การวางแผน CLI ที่ทริกเกอร์ด้วยคำสั่งจะย้อนกลับไปใช้ `commandAliases[].cliCommand` หรือ `commandAliases[].name` แบบเดิม
- การวางแผนการเริ่มต้นรันไทม์ของเอเจนต์ใช้ `activation.onAgentHarnesses` สำหรับชุดทดสอบแบบฝัง และใช้ `cliBackends[]` ระดับบนสุดสำหรับนามแฝงรันไทม์ CLI
- การวางแผนการตั้งค่า/ช่องทางที่ทริกเกอร์โดยช่องทางจะย้อนกลับไปใช้ความเป็นเจ้าของ `channels[]` แบบเดิม เมื่อไม่มีข้อมูลเมตาการเปิดใช้งานช่องทางที่ระบุไว้อย่างชัดเจน
- การวางแผน Plugin ขณะเริ่มต้นใช้ `activation.onConfigPaths` สำหรับพื้นผิวการกำหนดค่ารากที่ไม่ใช่ช่องทาง เช่น บล็อก `browser` ของ Plugin เบราว์เซอร์ที่รวมมาให้
- การวางแผนการตั้งค่า/รันไทม์ที่ทริกเกอร์โดยผู้ให้บริการจะย้อนกลับไปใช้ความเป็นเจ้าของ `providers[]` แบบเดิมและ `cliBackends[]` ระดับบนสุด เมื่อไม่มีข้อมูลเมตาการเปิดใช้งานผู้ให้บริการที่ระบุไว้อย่างชัดเจน

การวินิจฉัยของเครื่องมือวางแผนสามารถแยกคำใบ้การเปิดใช้งานที่ระบุอย่างชัดเจนออกจากการย้อนกลับไปใช้ความเป็นเจ้าของในไฟล์ manifest ได้ ตัวอย่างเช่น `activation-command-hint` หมายถึง `activation.onCommands` ตรงกัน ขณะที่ `manifest-command-alias` หมายถึงเครื่องมือวางแผนใช้ความเป็นเจ้าของ `commandAliases` แทน ป้ายกำกับเหตุผลเหล่านี้มีไว้สำหรับการวินิจฉัยของโฮสต์และการทดสอบ ผู้เขียน Plugin ควรประกาศข้อมูลเมตาที่อธิบายความเป็นเจ้าของได้เหมาะสมที่สุดต่อไป

## ข้อมูลอ้างอิง qaRunners

ใช้ `qaRunners` เมื่อ Plugin มีตัวเรียกใช้การขนส่งอย่างน้อยหนึ่งรายการภายใต้
ราก `openclaw qa` ที่ใช้ร่วมกัน เก็บข้อมูลเมตานี้ให้ประมวลผลได้รวดเร็วและเป็นแบบคงที่ โดยรันไทม์ของ Plugin
ยังคงเป็นเจ้าของการลงทะเบียน CLI จริงผ่านพื้นผิว `runtime-api.ts`
ขนาดเล็กที่ส่งออก `qaRunnerCliRegistrations` ที่ตรงกัน ส่วน `adapterFactory`
ที่เป็นทางเลือกจะเปิดเผยการขนส่งให้กับสถานการณ์ QA ที่ใช้ร่วมกัน โดยไม่เปลี่ยน
ตัวเรียกใช้ของคำสั่งที่ลงทะเบียนไว้

```json
{
  "qaRunners": [
    {
      "commandName": "matrix",
      "description": "เรียกใช้เลน QA แบบสดของ Matrix ที่รองรับด้วย Docker กับโฮมเซิร์ฟเวอร์ชั่วคราว"
    }
  ]
}
```

| ฟิลด์         | จำเป็น | ชนิด     | ความหมาย                                                      |
| ------------- | -------- | -------- | ------------------------------------------------------------------ |
| `commandName` | ใช่      | `string` | คำสั่งย่อยที่ติดตั้งไว้ภายใต้ `openclaw qa` เช่น `matrix`    |
| `description` | ไม่       | `string` | ข้อความช่วยเหลือสำรองที่ใช้เมื่อโฮสต์ร่วมต้องการคำสั่งชั่วคราว |

รหัส `adapterFactory` ต้องตรงกับ `commandName` อย่าส่งออกการลงทะเบียน
สำหรับคำสั่งที่ไม่มีอยู่ในไฟล์ manifest

## ข้อมูลอ้างอิง setup

ใช้ `setup` เมื่อพื้นผิวการตั้งค่าและการเริ่มต้นใช้งานต้องการข้อมูลเมตาที่ Plugin เป็นเจ้าของและประมวลผลได้รวดเร็วก่อนโหลดรันไทม์

```json
{
  "setup": {
    "providers": [
      {
        "id": "openai",
        "authMethods": ["api-key"],
        "envVars": ["OPENAI_API_KEY"],
        "authEvidence": [
          {
            "type": "local-file-with-env",
            "fileEnvVar": "OPENAI_CREDENTIALS_FILE",
            "requiresAllEnv": ["OPENAI_PROJECT"],
            "credentialMarker": "openai-local-credentials",
            "source": "ข้อมูลรับรองภายในเครื่องของ openai"
          }
        ]
      }
    ],
    "cliBackends": ["openai-cli"],
    "configMigrations": ["legacy-openai-auth"],
    "requiresRuntime": false
  }
}
```

`cliBackends` ระดับบนสุดยังคงใช้ได้และยังคงอธิบายแบ็กเอนด์การอนุมานของ CLI ส่วน `setup.cliBackends` เป็นพื้นผิวตัวอธิบายเฉพาะการตั้งค่าสำหรับโฟลว์ส่วนควบคุม/การตั้งค่าที่ควรใช้เฉพาะข้อมูลเมตา

เมื่อมี `setup.providers` และ `setup.cliBackends` ทั้งสองรายการนี้คือพื้นผิวการค้นหาที่ให้ความสำคัญกับตัวอธิบายสำหรับการค้นพบการตั้งค่า หากตัวอธิบายเพียงจำกัดขอบเขตของ Plugin ที่เป็นตัวเลือก แต่การตั้งค่ายังต้องใช้ฮุกของรันไทม์ขณะตั้งค่าที่สมบูรณ์กว่า ให้ตั้งค่า `requiresRuntime: true` และคง `setup-api` ไว้เป็นเส้นทางการดำเนินการสำรอง

OpenClaw ยังรวม `setup.providers[].envVars` ไว้ในการค้นหาการยืนยันตัวตนและตัวแปรสภาพแวดล้อมของผู้ให้บริการแบบทั่วไปด้วย `providerAuthEnvVars` ยังคงรองรับผ่านอะแดปเตอร์ความเข้ากันได้ในช่วงการเลิกใช้งาน แต่ Plugin ที่ไม่ได้รวมมาให้และยังคงใช้รายการนี้จะได้รับการวินิจฉัยไฟล์ manifest Plugin ใหม่ควรวางข้อมูลเมตาตัวแปรสภาพแวดล้อมสำหรับการตั้งค่า/สถานะไว้ที่ `setup.providers[].envVars`

ใช้ `providerUsageAuthEnvVars` เมื่อข้อมูลรับรองระดับการเรียกเก็บเงินหรือองค์กรจำเป็นต้องเปิดใช้งาน `resolveUsageAuth` โดยไม่กลายเป็นข้อมูลรับรองสำหรับการอนุมาน ชื่อเหล่านี้จะรวมอยู่ในการบล็อก dotenv ของพื้นที่ทำงาน การตัดออกจากกระบวนการลูก ACP การกรองข้อมูลลับของแซนด์บ็อกซ์ และการล้างข้อมูลลับในวงกว้าง รันไทม์ของผู้ให้บริการยังคงอ่านและจำแนกค่าภายใน `resolveUsageAuth`

OpenClaw ยังสามารถอนุมานตัวเลือกการตั้งค่าแบบง่ายจาก `setup.providers[].authMethods` เมื่อไม่มีรายการการตั้งค่า หรือเมื่อ `setup.requiresRuntime: false` ประกาศว่าไม่จำเป็นต้องใช้รันไทม์การตั้งค่า รายการ `providerAuthChoices` ที่ระบุอย่างชัดเจนยังคงได้รับความสำคัญสำหรับป้ายกำกับแบบกำหนดเอง แฟล็ก CLI ขอบเขตการเริ่มต้นใช้งาน และข้อมูลเมตาของผู้ช่วย

ตั้งค่า `requiresRuntime: false` เฉพาะเมื่อตัวอธิบายเหล่านั้นเพียงพอสำหรับพื้นผิวการตั้งค่า OpenClaw ถือว่า `false` ที่ระบุอย่างชัดเจนเป็นสัญญาที่ใช้เฉพาะตัวอธิบาย และจะไม่เรียกใช้ `setup-api` หรือ `openclaw.setupEntry` สำหรับการค้นหาการตั้งค่า หาก Plugin ที่ใช้เฉพาะตัวอธิบายยังคงจัดส่งรายการรันไทม์การตั้งค่ารายการใดรายการหนึ่งดังกล่าว OpenClaw จะรายงานการวินิจฉัยเพิ่มเติมและเพิกเฉยต่อรายการนั้นต่อไป การละ `requiresRuntime` จะคงพฤติกรรมการย้อนกลับแบบเดิมไว้ เพื่อไม่ให้ Plugin ที่มีอยู่ซึ่งเพิ่มตัวอธิบายโดยไม่มีแฟล็กเกิดความเสียหาย

เนื่องจากการค้นหาการตั้งค่าสามารถเรียกใช้โค้ด `setup-api` ที่ Plugin เป็นเจ้าของ ค่า `setup.providers[].id` และ `setup.cliBackends[]` ที่ปรับรูปแบบแล้วจึงต้องไม่ซ้ำกันในบรรดา Plugin ที่ค้นพบ ความเป็นเจ้าของที่กำกวมจะปฏิเสธการทำงานอย่างปลอดภัย แทนที่จะเลือกผู้ชนะตามลำดับการค้นพบ

เมื่อมีการเรียกใช้รันไทม์การตั้งค่า การวินิจฉัยรีจิสทรีการตั้งค่าจะรายงานความคลาดเคลื่อนของตัวอธิบาย หาก `setup-api` ลงทะเบียนผู้ให้บริการหรือแบ็กเอนด์ CLI ที่ตัวอธิบายในไฟล์ manifest ไม่ได้ประกาศไว้ หรือหากตัวอธิบายไม่มีการลงทะเบียนรันไทม์ที่ตรงกัน การวินิจฉัยเหล่านี้เป็นข้อมูลเพิ่มเติมและจะไม่ปฏิเสธ Plugin แบบเดิม

### ข้อมูลอ้างอิง setup.providers

| ฟิลด์          | จำเป็น | ชนิด       | ความหมาย                                                                                    |
| -------------- | -------- | ---------- | ------------------------------------------------------------------------------------------------ |
| `id`           | ใช่      | `string`   | รหัสผู้ให้บริการที่เปิดเผยระหว่างการตั้งค่าหรือการเริ่มต้นใช้งาน รักษารหัสที่ปรับรูปแบบแล้วไม่ให้ซ้ำกันทั่วทั้งระบบ             |
| `authMethods`  | ไม่       | `string[]` | รหัสวิธีการตั้งค่า/การยืนยันตัวตนที่ผู้ให้บริการนี้รองรับโดยไม่ต้องโหลดรันไทม์ทั้งหมด                       |
| `envVars`      | ไม่       | `string[]` | ตัวแปรสภาพแวดล้อมที่พื้นผิวการตั้งค่า/สถานะแบบทั่วไปสามารถตรวจสอบได้ก่อนโหลดรันไทม์ของ Plugin               |
| `authEvidence` | ไม่       | `object[]` | การตรวจสอบหลักฐานการยืนยันตัวตนภายในเครื่องที่ประมวลผลได้รวดเร็ว สำหรับผู้ให้บริการที่ยืนยันตัวตนผ่านตัวบ่งชี้ที่ไม่ใช่ข้อมูลลับได้ |

`authEvidence` มีไว้สำหรับตัวบ่งชี้ข้อมูลรับรองภายในเครื่องที่ผู้ให้บริการเป็นเจ้าของ ซึ่งสามารถตรวจสอบได้โดยไม่ต้องโหลดโค้ดรันไทม์ การตรวจสอบเหล่านี้ต้องประมวลผลได้รวดเร็วและทำงานภายในเครื่องเท่านั้น: ห้ามเรียกใช้เครือข่าย ห้ามอ่านพวงกุญแจหรือตัวจัดการข้อมูลลับ ห้ามใช้คำสั่งเชลล์ และห้ามตรวจสอบ API ของผู้ให้บริการ

รายการหลักฐานที่รองรับ:

| ฟิลด์              | จำเป็น | ชนิด       | ความหมาย                                                                                                  |
| ------------------ | -------- | ---------- | -------------------------------------------------------------------------------------------------------------- |
| `type`             | ใช่      | `string`   | ปัจจุบันคือ `local-file-with-env`                                                                               |
| `fileEnvVar`       | ไม่       | `string`   | ตัวแปรสภาพแวดล้อมที่มีพาธไฟล์ข้อมูลรับรองซึ่งระบุไว้อย่างชัดเจน                                                           |
| `fallbackPaths`    | ไม่       | `string[]` | พาธไฟล์ข้อมูลรับรองภายในเครื่องที่ตรวจสอบเมื่อไม่มี `fileEnvVar` หรือมีค่าว่าง รองรับ `${HOME}` และ `${APPDATA}` |
| `requiresAnyEnv`   | ไม่       | `string[]` | ตัวแปรสภาพแวดล้อมอย่างน้อยหนึ่งรายการที่ระบุต้องไม่ว่างเปล่า จึงจะถือว่าหลักฐานใช้ได้                                    |
| `requiresAllEnv`   | ไม่       | `string[]` | ตัวแปรสภาพแวดล้อมทุกรายการที่ระบุต้องไม่ว่างเปล่า จึงจะถือว่าหลักฐานใช้ได้                                           |
| `credentialMarker` | ใช่      | `string`   | ตัวบ่งชี้ที่ไม่ใช่ข้อมูลลับซึ่งส่งคืนเมื่อมีหลักฐาน                                                       |
| `source`           | ไม่       | `string`   | ป้ายกำกับแหล่งที่มาสำหรับผู้ใช้ในผลลัพธ์การยืนยันตัวตน/สถานะ                                                               |

### ฟิลด์ setup

| ฟิลด์              | จำเป็น | ชนิด       | ความหมาย                                                                                       |
| ------------------ | -------- | ---------- | --------------------------------------------------------------------------------------------------- |
| `providers`        | ไม่       | `object[]` | ตัวอธิบายการตั้งค่าผู้ให้บริการที่เปิดเผยระหว่างการตั้งค่าและการเริ่มต้นใช้งาน                                     |

## ข้อมูลอ้างอิง uiHints

`uiHints` คือแมปจากชื่อฟิลด์การกำหนดค่าไปยังคำใบ้การแสดงผลขนาดเล็ก คีย์สามารถใช้จุดสำหรับฟิลด์การกำหนดค่าที่ซ้อนกันได้ แต่ส่วนใดของพาธต้องไม่เป็น `__proto__`, `constructor` หรือ `prototype` มิฉะนั้นการตั้งค่าจะปฏิเสธชื่อเหล่านั้น

```json
{
  "uiHints": {
    "apiKey": {
      "label": "คีย์ API",
      "help": "ใช้สำหรับคำขอ OpenRouter",
      "placeholder": "sk-or-v1-...",
      "sensitive": true
    }
  }
}
```

คำใบ้ของแต่ละฟิลด์สามารถประกอบด้วย:

| ฟิลด์         | ชนิด       | ความหมาย                           |
| ------------- | ---------- | --------------------------------------- |
| `label`       | `string`   | ป้ายกำกับฟิลด์สำหรับผู้ใช้                |
| `help`        | `string`   | ข้อความช่วยเหลือแบบสั้น                      |
| `tags`        | `string[]` | แท็ก UI ที่เป็นทางเลือก                       |
| `advanced`    | `boolean`  | ทำเครื่องหมายฟิลด์ว่าเป็นขั้นสูง            |
| `sensitive`   | `boolean`  | ทำเครื่องหมายฟิลด์ว่าเป็นข้อมูลลับหรือข้อมูลละเอียดอ่อน |
| `placeholder` | `string`   | ข้อความตัวยึดตำแหน่งสำหรับช่องป้อนข้อมูลในฟอร์ม       |

## ข้อมูลอ้างอิง contracts

ใช้ `contracts` เฉพาะสำหรับข้อมูลเมตาความเป็นเจ้าของความสามารถแบบคงที่ที่ OpenClaw สามารถอ่านได้โดยไม่ต้องนำเข้ารันไทม์ของ Plugin

```json
{
  "contracts": {
    "agentToolResultMiddleware": ["openclaw", "codex"],
    "trustedToolPolicies": ["workflow-budget"],
```

`contracts.embeddedExtensionFactories` ยังคงเก็บไว้สำหรับแฟกทอรีส่วนขยาย Codex app-server-only ที่รวมมาในชุด การแปลงผลลัพธ์เครื่องมือที่รวมมาในชุดควรประกาศ `contracts.agentToolResultMiddleware` และลงทะเบียนด้วย `api.registerAgentToolResultMiddleware(...)` แทน Plugin ที่ติดตั้งอาจใช้จุดเชื่อมมิดเดิลแวร์เดียวกันได้เฉพาะเมื่อเปิดใช้อย่างชัดเจน และเฉพาะกับรันไทม์ที่ประกาศไว้ใน `contracts.agentToolResultMiddleware`

Plugin ที่ติดตั้งซึ่งต้องใช้ระดับนโยบายก่อนใช้เครื่องมือที่โฮสต์เชื่อถือ ต้องประกาศ ID ภายในที่ลงทะเบียนแต่ละรายการใน `contracts.trustedToolPolicies` และต้องได้รับการเปิดใช้อย่างชัดเจน Plugin ที่รวมมาในชุดยังคงใช้เส้นทางนโยบายที่เชื่อถือได้เดิม แต่ Plugin ที่ติดตั้งซึ่งมี ID นโยบายที่ไม่ได้ประกาศจะถูกปฏิเสธก่อนการลงทะเบียน ID นโยบายมีขอบเขตเฉพาะ Plugin ที่ลงทะเบียน ดังนั้น Plugin สองตัวอาจประกาศและลงทะเบียน `workflow-budget` เหมือนกันได้ แต่ Plugin เดียวไม่สามารถลงทะเบียน ID ภายในเดียวกันซ้ำสองครั้ง

การลงทะเบียนรันไทม์ `api.registerTool(...)` ต้องตรงกับ `contracts.tools` การค้นหาเครื่องมือใช้รายการนี้เพื่อโหลดเฉพาะรันไทม์ของ Plugin ที่สามารถเป็นเจ้าของเครื่องมือที่ร้องขอ

Plugin ผู้ให้บริการที่ใช้งาน `resolveExternalAuthProfiles` ควรประกาศ `contracts.externalAuthProviders` โดยฮุกการยืนยันตัวตนภายนอกที่ไม่ได้ประกาศจะถูกละเว้น

Plugin ผู้ให้บริการที่ใช้งานทั้ง `resolveUsageAuth` และ `fetchUsageSnapshot` ควรประกาศ ID ผู้ให้บริการที่ค้นพบอัตโนมัติแต่ละรายการใน `contracts.usageProviders` การค้นหาการใช้งานจะอ่านสัญญานี้ก่อนโหลดโค้ดรันไทม์ จากนั้นตรวจสอบฮุกทั้งสองหลังจากโหลดเฉพาะเจ้าของที่ประกาศไว้

ผู้ให้บริการ embedding ทั่วไปควรประกาศ `contracts.embeddingProviders` สำหรับอะแดปเตอร์แต่ละตัวที่ลงทะเบียนด้วย `api.registerEmbeddingProvider(...)` ใช้สัญญาทั่วไปสำหรับการสร้างเวกเตอร์ที่ใช้ซ้ำได้ รวมถึงผู้ให้บริการที่การค้นหาหน่วยความจำใช้งาน `contracts.memoryEmbeddingProviders` เป็นความเข้ากันได้เฉพาะหน่วยความจำที่เลิกใช้แล้ว และจะคงอยู่เฉพาะระหว่างที่ผู้ให้บริการเดิมย้ายไปยังจุดเชื่อมผู้ให้บริการ embedding แบบทั่วไป

ผู้ให้บริการเวิร์กเกอร์ต้องประกาศ ID `api.registerWorkerProvider(...)` แต่ละรายการใน `contracts.workerProviders` คอร์จะคงเจตนาที่ถาวรก่อนเรียก `provision` ผู้ให้บริการจะตรวจสอบการตั้งค่าก่อนการจัดสรรภายนอก และการเรียกซ้ำด้วย ID การดำเนินการเดียวกันต้องรับช่วงลีสเดียวกัน คอร์ยังคงเก็บสแนปช็อตการตั้งค่าที่ตรวจสอบแล้ว และส่งไปพร้อมกับ `leaseId` ให้กับ `inspect({ leaseId, profile })` และ `destroy({ leaseId, profile })` รวมถึงหลังจากมีการเปลี่ยนหรือลบโปรไฟล์ที่ระบุชื่อแล้ว การทำลายเป็นแบบ idempotent การตรวจสอบจะคืนค่ายูเนียนสถานะปิด `active` / `destroyed` / `unknown` และวัสดุคีย์ส่วนตัว SSH จะถูกอ้างอิงผ่าน `SecretRef` เท่านั้น ปลายทาง SSH ที่จัดเตรียมแล้วต้องมี `hostKey` สาธารณะจากผลลัพธ์การจัดเตรียมที่เชื่อถือได้เป็น `algorithm base64` อย่างตรงตัว โดยไม่มีชื่อโฮสต์หรือความคิดเห็น เพื่อให้คอร์สามารถตรึงโฮสต์ก่อนเชื่อมต่อ ผู้ให้บริการที่สร้างการอ้างอิงตัวตนแบบไดนามิกอาจใช้งาน `resolveSshIdentity({ leaseId, profile, keyRef })` ที่มีอำนาจตัดสินชี้ขาด ส่วนผู้ให้บริการที่ไม่มีจะใช้ตัวแก้ไขข้อมูลลับทั่วไปของคอร์ `unknown` ที่มีอำนาจตัดสินชี้ขาดจะทำให้ระเบียนภายในที่ใช้งานอยู่กลายเป็นกำพร้า และหลังจากคำขอทำลายถูกคงเก็บแล้ว จะยืนยันการรื้อถอน

ปัจจุบัน `contracts.gatewayMethodDispatch` ยอมรับ `"authenticated-request"` โดยเป็นด่านสุขอนามัย API สำหรับเส้นทาง HTTP ดั้งเดิมของ Plugin ที่ตั้งใจส่งต่อเมธอดระนาบควบคุมของ Gateway ภายในโปรเซส ไม่ใช่แซนด์บ็อกซ์ป้องกัน Plugin ดั้งเดิมที่เป็นอันตราย ใช้เฉพาะกับพื้นผิวที่รวมมาในชุดหรือของผู้ปฏิบัติการซึ่งผ่านการตรวจสอบอย่างเข้มงวดและกำหนดให้ใช้การยืนยันตัวตน HTTP ของ Gateway อยู่แล้ว เส้นทางที่ได้รับสิทธิ์จะยังเข้าถึงได้ขณะที่การรับงานระดับรากของ Gateway ปิดอยู่ เฉพาะเมื่อเส้นทางนั้นประกาศทั้ง `auth: "gateway"` และ `gatewayRuntimeScopeSurface: "trusted-operator"` ที่เฉพาะเจาะจงสำหรับเส้นทาง เส้นทางพี่น้องทั่วไปจาก Plugin เดียวกันจะยังอยู่หลังขอบเขตการรับงาน ซึ่งช่วยให้ยังเข้าถึงสถานะการระงับและการทำงานต่อได้โดยไม่ให้สิทธิ์ข้ามการรับงานแก่ Plugin ทั้งหมด จำกัดการแยกวิเคราะห์และการจัดรูปแบบการตอบกลับไว้นอกการส่งต่อ งานที่มีสาระสำคัญหรือแก้ไขข้อมูลต้องผ่านการส่งต่อเมธอด Gateway ซึ่งเป็นเจ้าของการบังคับใช้การรับงานและขอบเขต

## ข้อมูลอ้างอิง configContracts

ใช้ `configContracts` สำหรับพฤติกรรมการกำหนดค่าที่แมนิเฟสต์เป็นเจ้าของ ซึ่งตัวช่วยทั่วไปของคอร์ต้องใช้โดยไม่ต้องนำเข้ารันไทม์ของ Plugin ได้แก่ การตรวจหาแฟล็กอันตราย เป้าหมายการย้าย SecretRef และการจำกัดพาธการกำหนดค่าแบบเดิม

```json
{
  "configContracts": {
    "compatibilityMigrationPaths": ["legacyProvider"],
    "compatibilityRuntimePaths": ["legacyProvider.webhook"],
    "dangerousFlags": [
      {
        "path": "accounts.*.allowUnverifiedSenders",
        "equals": true
      }
    ],
    "secretInputs": {
      "bundledDefaultEnabled": false,
      "paths": [
        {
          "path": "routes.*.secret",
          "expected": "string",
          "ownerKind": "route"
        }
      ]
    }
  }
}
```

| ฟิลด์                         | จำเป็น | ชนิด       | ความหมาย                                                                                                                                                                                                                          |
| ----------------------------- | -------- | ---------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `compatibilityMigrationPaths` | ไม่       | `string[]` | พาธการกำหนดค่าที่สัมพันธ์กับราก ซึ่งบ่งชี้ว่าการย้ายความเข้ากันได้ระหว่างการตั้งค่าของ Plugin นี้อาจมีผล ช่วยให้การอ่านการกำหนดค่ารันไทม์ทั่วไปข้ามพื้นผิวการตั้งค่าทั้งหมดของ Plugin ได้เมื่อการกำหนดค่าไม่เคยอ้างถึง Plugin                 |
| `compatibilityRuntimePaths`   | ไม่       | `string[]` | พาธความเข้ากันได้ที่สัมพันธ์กับราก ซึ่ง Plugin นี้สามารถให้บริการระหว่างรันไทม์ก่อนที่โค้ด Plugin จะเปิดใช้งานอย่างสมบูรณ์ ใช้สำหรับพื้นผิวแบบเดิมที่ควรจำกัดชุดตัวเลือกที่รวมมาในชุดโดยไม่ต้องนำเข้ารันไทม์ Plugin ที่เข้ากันได้ทั้งหมด |
| `dangerousFlags`              | ไม่       | `object[]` | ลิเทอรัลการกำหนดค่าที่ `openclaw doctor` ควรแจ้งว่าไม่ปลอดภัยหรืออันตรายเมื่อเปิดใช้ ดูด้านล่าง                                                                                                                                   |
| `secretInputs`                | ไม่       | `object`   | พาธการกำหนดค่าภายใต้ `plugins.entries.<id>.config` สำหรับการย้าย SecretRef การตรวจสอบ การสร้างค่าเมื่อเริ่มต้นระบบ และการแยกเจ้าของรันไทม์แบบเลือกได้ ดูด้านล่าง                                                                             |

รายการ `dangerousFlags` แต่ละรายการรองรับ:

| ฟิลด์    | จำเป็น | ชนิด                                  | ความหมาย                                                                                                       |
| -------- | -------- | ------------------------------------- | ------------------------------------------------------------------------------------------------------------------- |
| `path`   | ใช่      | `string`                              | พาธการกำหนดค่าที่คั่นด้วยจุดและสัมพันธ์กับ `plugins.entries.<id>.config` รองรับไวลด์การ์ด `*` สำหรับเซกเมนต์ของแมป/อาร์เรย์ |
| `equals` | ใช่      | `string \| number \| boolean \| null` | ลิเทอรัลที่ตรงกันทุกประการซึ่งระบุว่าค่าการกำหนดค่านี้เป็นอันตราย                                                            |

`secretInputs` รองรับ:

| ฟิลด์                   | จำเป็น | ประเภท       | ความหมาย                                                                                                                                                                                                                                                                                                                                              |
| ----------------------- | -------- | ---------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `bundledDefaultEnabled` | ไม่       | `boolean`  | เขียนทับค่าเริ่มต้นการเปิดใช้งาน Plugin ที่รวมมาให้ เมื่อพิจารณาว่าพื้นผิว SecretRef นี้ทำงานอยู่หรือไม่ ใช้ตัวเลือกนี้เมื่อ Plugin รวมมาให้แล้ว แต่พื้นผิวควรไม่ทำงานจนกว่าจะเปิดใช้งานอย่างชัดเจนในการกำหนดค่า                                                                                                                                            |
| `paths`                 | ใช่      | `object[]` | พาธการกำหนดค่าที่มีรูปแบบเป็นข้อมูลลับ โดยแต่ละพาธมี `path` (คั่นด้วยจุด สัมพันธ์กับ `plugins.entries.<id>.config` และรองรับไวลด์การ์ด `*`) มี `expected` เป็นตัวเลือก (ปัจจุบันรองรับเฉพาะ `"string"`) และมี `ownerKind` เป็นตัวเลือก (ปัจจุบันรองรับเฉพาะ `"route"`) เจ้าของที่ประกาศไว้จะแยกเฉพาะพาธที่ตรงกันทุกประการนั้นเมื่อการแก้ไขค่าล้มเหลว โดยรหัสเจ้าของคือพาธการกำหนดค่าแบบเต็ม |

## ข้อมูลอ้างอิง mediaUnderstandingProviderMetadata

ใช้ `mediaUnderstandingProviderMetadata` เมื่อผู้ให้บริการทำความเข้าใจสื่อมีโมเดลเริ่มต้น ลำดับความสำคัญสำรองของการยืนยันตัวตนอัตโนมัติ หรือการรองรับเอกสารโดยตรง ซึ่งตัวช่วยทั่วไปของแกนหลักต้องใช้ก่อนโหลดรันไทม์ คีย์ต่างๆ ต้องประกาศไว้ใน `contracts.mediaUnderstandingProviders` ด้วย

```json
{
  "contracts": {
    "mediaUnderstandingProviders": ["example"]
  },
  "mediaUnderstandingProviderMetadata": {
    "example": {
      "capabilities": ["image", "audio"],
      "defaultModels": {
        "image": "example-vision-latest",
        "audio": "example-transcribe-latest"
      },
      "autoPriority": {
        "image": 40
      },
      "nativeDocumentInputs": ["pdf"],
      "documentModels": {
        "pdf": {
          "textExtraction": "example-doc-text-latest",
          "image": "example-doc-vision-latest"
        }
      }
    }
  }
}
```

รายการของผู้ให้บริการแต่ละรายสามารถประกอบด้วย:

| ฟิลด์                  | ประเภท                                                             | ความหมาย                                                                                                   |
| ---------------------- | ---------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------- |
| `capabilities`         | `("image" \| "audio" \| "video")[]`                              | ความสามารถด้านสื่อที่ผู้ให้บริการรายนี้เปิดให้ใช้                                                                    |
| `defaultModels`        | `Record<string, string>`                                         | ค่าเริ่มต้นของโมเดลตามความสามารถ ซึ่งใช้เมื่อการกำหนดค่าไม่ได้ระบุโมเดล                                         |
| `autoPriority`         | `Record<string, number>`                                         | ตัวเลขที่ต่ำกว่าจะถูกจัดไว้ก่อนสำหรับการสำรองผู้ให้บริการโดยอัตโนมัติตามข้อมูลประจำตัว                                    |
| `nativeDocumentInputs` | `"pdf"[]`                                                        | อินพุตเอกสารโดยตรงที่ผู้ให้บริการรองรับ                                                               |
| `documentModels`       | `{ pdf?: { textExtraction?: string; image?: string \| false } }` | การเขียนทับโมเดลแยกตามประเภทเอกสาร ตั้งค่า `image: false` เพื่อปิดใช้งานการแยกข้อมูลด้วยภาพสำหรับเอกสารประเภทนั้น |

## ข้อมูลอ้างอิง channelConfigs

ใช้ `channelConfigs` เมื่อ Plugin ช่องทางต้องการข้อมูลเมตาการกำหนดค่าที่เข้าถึงได้รวดเร็วก่อนโหลดรันไทม์ การค้นหาการตั้งค่า/สถานะแบบอ่านอย่างเดียวของช่องทางสามารถใช้ข้อมูลเมตานี้โดยตรงสำหรับช่องทางภายนอกที่กำหนดค่าไว้ เมื่อไม่มีรายการการตั้งค่า หรือเมื่อ `setup.requiresRuntime: false` ประกาศว่าไม่จำเป็นต้องใช้รันไทม์สำหรับการตั้งค่า

`channelConfigs` คือข้อมูลเมตาของไฟล์ manifest ของ Plugin ไม่ใช่ส่วนการกำหนดค่าระดับบนสุดใหม่สำหรับผู้ใช้ ผู้ใช้ยังคงกำหนดค่าอินสแตนซ์ช่องทางภายใต้ `channels.<channel-id>` OpenClaw อ่านข้อมูลเมตาของไฟล์ manifest เพื่อพิจารณาว่า Plugin ใดเป็นเจ้าของช่องทางที่กำหนดค่าไว้นั้น ก่อนที่โค้ดรันไทม์ของ Plugin จะทำงาน

สำหรับ Plugin ช่องทาง `configSchema` และ `channelConfigs` อธิบายพาธที่ต่างกัน:

- `configSchema` ตรวจสอบความถูกต้องของ `plugins.entries.<plugin-id>.config`
- `channelConfigs.<channel-id>.schema` ตรวจสอบความถูกต้องของ `channels.<channel-id>`

Plugin ที่ไม่ได้รวมมาให้ซึ่งประกาศ `channels[]` ควรประกาศรายการ `channelConfigs` ที่ตรงกันด้วย หากไม่มีรายการเหล่านี้ OpenClaw ยังคงโหลด Plugin ได้ แต่สคีมาการกำหนดค่าของพาธเริ่มต้น การตั้งค่า และพื้นผิว Control UI จะไม่สามารถทราบรูปแบบตัวเลือกที่ช่องทางเป็นเจ้าของได้จนกว่ารันไทม์ของ Plugin จะทำงาน

`channelConfigs.<channel-id>.commands.nativeCommandsAutoEnabled` และ `nativeSkillsAutoEnabled` สามารถประกาศค่าเริ่มต้น `auto` แบบคงที่สำหรับการตรวจสอบการกำหนดค่าคำสั่งที่ทำงานก่อนโหลดรันไทม์ของช่องทาง ช่องทางที่รวมมาให้ยังสามารถเผยแพร่ค่าเริ่มต้นเดียวกันผ่าน `package.json#openclaw.channel.commands` ควบคู่กับข้อมูลเมตาแค็ตตาล็อกช่องทางอื่นๆ ที่แพ็กเกจเป็นเจ้าของ

```json
{
  "channelConfigs": {
    "matrix": {
      "schema": {
        "type": "object",
        "additionalProperties": false,
        "properties": {
          "homeserverUrl": { "type": "string" }
        }
      },
      "uiHints": {
        "homeserverUrl": {
          "label": "URL ของโฮมเซิร์ฟเวอร์",
          "placeholder": "https://matrix.example.com"
        }
      },
      "label": "Matrix",
      "description": "การเชื่อมต่อโฮมเซิร์ฟเวอร์ Matrix",
      "commands": {
        "nativeCommandsAutoEnabled": true,
        "nativeSkillsAutoEnabled": true
      },
      "preferOver": ["matrix-legacy"]
    }
  }
}
```

รายการช่องทางแต่ละรายการสามารถประกอบด้วย:

| ฟิลด์         | ประเภท                     | ความหมาย                                                                             |
| ------------- | ------------------------ | ----------------------------------------------------------------------------------------- |
| `schema`      | `object`                 | JSON Schema สำหรับ `channels.<id>` จำเป็นสำหรับรายการการกำหนดค่าช่องทางแต่ละรายการที่ประกาศไว้         |
| `uiHints`     | `Record<string, object>` | ป้ายกำกับ UI/ข้อความตัวอย่าง/คำแนะนำเกี่ยวกับข้อมูลละเอียดอ่อนซึ่งเป็นตัวเลือกสำหรับส่วนการกำหนดค่าช่องทางนั้น          |
| `label`       | `string`                 | ป้ายกำกับช่องทางที่ผสานเข้ากับพื้นผิวตัวเลือกและการตรวจสอบ เมื่อข้อมูลเมตารันไทม์ยังไม่พร้อม |
| `description` | `string`                 | คำอธิบายช่องทางแบบสั้นสำหรับพื้นผิวการตรวจสอบและแค็ตตาล็อก                               |
| `commands`    | `object`                 | ค่าเริ่มต้นอัตโนมัติแบบคงที่สำหรับคำสั่งดั้งเดิมและ Skills ดั้งเดิม เพื่อใช้ตรวจสอบการกำหนดค่าก่อนรันไทม์       |
| `preferOver`  | `string[]`               | รหัส Plugin แบบเดิมหรือที่มีลำดับความสำคัญต่ำกว่า ซึ่งช่องทางนี้ควรมีอันดับเหนือกว่าในพื้นผิวการเลือก    |

### การแทนที่ Plugin ช่องทางอื่น

ใช้ `preferOver` เมื่อ Plugin ของคุณเป็นเจ้าของที่ต้องการสำหรับรหัสช่องทางซึ่ง Plugin อื่นสามารถให้บริการได้เช่นกัน กรณีทั่วไปได้แก่ รหัส Plugin ที่เปลี่ยนชื่อ Plugin แบบแยกอิสระที่มาแทน Plugin ซึ่งรวมมาให้ หรือ fork ที่ได้รับการบำรุงรักษาซึ่งคงรหัสช่องทางเดิมไว้เพื่อความเข้ากันได้ของการกำหนดค่า

```json
{
  "id": "acme-chat",
  "channels": ["chat"],
  "channelConfigs": {
    "chat": {
      "schema": {
        "type": "object",
        "additionalProperties": false,
        "properties": {
          "webhookUrl": { "type": "string" }
        }
      },
      "preferOver": ["chat"]
    }
  }
}
```

เมื่อกำหนดค่า `channels.chat` แล้ว OpenClaw จะพิจารณาทั้งรหัสช่องทางและรหัส Plugin ที่ต้องการ หาก Plugin ที่มีลำดับความสำคัญต่ำกว่าถูกเลือกเพียงเพราะรวมมาให้หรือเปิดใช้งานโดยค่าเริ่มต้น OpenClaw จะปิดใช้งาน Plugin นั้นในการกำหนดค่ารันไทม์ที่มีผล เพื่อให้ Plugin เดียวเป็นเจ้าของช่องทางและเครื่องมือของช่องทาง การเลือกอย่างชัดเจนของผู้ใช้ยังคงมีลำดับความสำคัญสูงสุด: หากผู้ใช้เปิดใช้งาน Plugin ทั้งสองอย่างชัดเจน (ผ่าน `plugins.allow` หรือการกำหนดค่า `plugins.entries` ที่มีผลจริง) OpenClaw จะรักษาตัวเลือกนั้นไว้และรายงานการวินิจฉัยช่องทาง/เครื่องมือที่ซ้ำกัน แทนที่จะเปลี่ยนชุด Plugin ที่ร้องขอโดยไม่แจ้งให้ทราบ

จำกัดขอบเขต `preferOver` ไว้เฉพาะรหัส Plugin ที่สามารถให้บริการช่องทางเดียวกันได้จริง ฟิลด์นี้ไม่ใช่ฟิลด์ลำดับความสำคัญทั่วไป และไม่ได้เปลี่ยนชื่อคีย์การกำหนดค่าของผู้ใช้

## ข้อมูลอ้างอิง modelSupport

ใช้ `modelSupport` เมื่อ OpenClaw ควรอนุมาน Plugin ผู้ให้บริการของคุณจากรหัสโมเดลแบบย่อ เช่น `gpt-5.6-sol` หรือ `claude-sonnet-4.6` ก่อนโหลดรันไทม์ของ Plugin

```json
{
  "modelSupport": {
    "modelPrefixes": ["gpt-", "o1", "o3", "o4"],
    "modelPatterns": ["^computer-use-preview"]
  }
}
```

OpenClaw ใช้ลำดับความสำคัญดังนี้:

- การอ้างอิง `provider/model` ที่ระบุอย่างชัดเจนจะใช้ข้อมูลเมตาของไฟล์ manifest `providers` ที่เป็นเจ้าของ
- `modelPatterns` มีลำดับความสำคัญเหนือ `modelPrefixes`
- หาก Plugin ที่ไม่ได้รวมมาให้หนึ่งรายการและ Plugin ที่รวมมาให้หนึ่งรายการตรงกันทั้งคู่ Plugin ที่ไม่ได้รวมมาให้จะได้รับเลือก
- ความกำกวมที่เหลือจะถูกละเว้นจนกว่าผู้ใช้หรือการกำหนดค่าจะระบุผู้ให้บริการ

ฟิลด์:

| ฟิลด์           | ประเภท       | ความหมาย                                                                   |
| --------------- | ---------- | ------------------------------------------------------------------------------- |
| `modelPrefixes` | `string[]` | คำนำหน้าที่จับคู่กับรหัสโมเดลแบบย่อโดยใช้ `startsWith`                 |
| `modelPatterns` | `string[]` | ซอร์ส regex ที่จับคู่กับรหัสโมเดลแบบย่อหลังจากนำส่วนต่อท้ายโปรไฟล์ออก |

รายการ `modelPatterns` จะถูกคอมไพล์ผ่าน `compileSafeRegex` ซึ่งปฏิเสธรูปแบบที่มีการทำซ้ำซ้อนกัน (ตัวอย่างเช่น `(a+)+$`) รูปแบบที่ไม่ผ่านการตรวจสอบความปลอดภัยจะถูกข้ามโดยไม่แจ้ง เช่นเดียวกับ regex ที่มีไวยากรณ์ไม่ถูกต้อง ควรรักษารูปแบบให้เรียบง่ายและหลีกเลี่ยงตัวระบุจำนวนแบบซ้อนกัน

## ข้อมูลอ้างอิง modelCatalog

ใช้ `modelCatalog` เมื่อ OpenClaw ควรทราบข้อมูลเมตาของโมเดลผู้ให้บริการก่อนโหลดรันไทม์ของ Plugin นี่คือแหล่งข้อมูลที่ไฟล์ manifest เป็นเจ้าของสำหรับแถวแค็ตตาล็อกแบบคงที่ นามแฝงของผู้ให้บริการ กฎการระงับ และโหมดการค้นหา การรีเฟรชขณะรันไทม์ยังคงเป็นหน้าที่ของโค้ดรันไทม์ผู้ให้บริการ แต่ไฟล์ manifest จะแจ้งให้แกนหลักทราบว่าเมื่อใดจำเป็นต้องใช้รันไทม์

```json
{
  "providers": ["openai"],
  "modelCatalog": {
    "providers": {
      "openai": {
        "baseUrl": "https://api.openai.com/v1",
        "api": "openai-responses",
        "models": [
          {
            "id": "gpt-5.4",
            "name": "GPT-5.4",
            "input": ["text", "image"],
            "reasoning": true,
            "contextWindow": 256000,
            "maxTokens": 128000,
            "cost": {
              "input": 1.25,
              "output": 10,
              "cacheRead": 0.125
            },
            "status": "available",
            "tags": ["default"]
          }
        ]
      }
    },
    "aliases": {
      "azure-openai-responses": {
        "provider": "openai",
        "api": "azure-openai-responses"
      }
    },
    "suppressions": [
      {
        "provider": "azure-openai-responses",
        "model": "gpt-5.3-codex-spark",
        "reason": "ไม่มีให้ใช้งานบน Azure OpenAI Responses"
      }
    ],
    "discovery": {
      "openai": "static"
    }
  }
}
```

ฟิลด์ระดับบนสุด:

| ฟิลด์            | ชนิด                                                     | ความหมาย                                                                                               |
| ---------------- | -------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------- |
| `providers`      | `Record<string, object>`                                 | แถวแค็ตตาล็อกสำหรับ ID ผู้ให้บริการที่ Plugin นี้เป็นเจ้าของ คีย์ควรปรากฏใน `providers` ระดับบนสุดด้วย       |
| `aliases`        | `Record<string, object>`                                 | นามแฝงผู้ให้บริการที่ควรถูกแปลงเป็นผู้ให้บริการที่เป็นเจ้าของสำหรับการวางแผนแค็ตตาล็อกหรือการระงับ              |
| `suppressions`   | `object[]`                                               | แถวโมเดลจากแหล่งอื่นที่ Plugin นี้ระงับด้วยเหตุผลเฉพาะของผู้ให้บริการ                  |
| `discovery`      | `Record<string, "static" \| "refreshable" \| "runtime">` | ระบุว่าสามารถอ่านแค็ตตาล็อกผู้ให้บริการจากเมทาดาทาของไฟล์ manifest, รีเฟรชลงในแคช หรือต้องใช้รันไทม์ |
| `runtimeAugment` | `boolean`                                                | ตั้งค่าเป็น `true` เฉพาะเมื่อรันไทม์ของผู้ให้บริการต้องเพิ่มแถวแค็ตตาล็อกหลังการวางแผน manifest/การกำหนดค่า       |

`aliases` มีส่วนร่วมในการค้นหาความเป็นเจ้าของผู้ให้บริการสำหรับการวางแผนแค็ตตาล็อกโมเดล เป้าหมายของนามแฝงต้องเป็นผู้ให้บริการระดับบนสุดที่ Plugin เดียวกันเป็นเจ้าของ เมื่อรายการที่กรองตามผู้ให้บริการใช้นามแฝง OpenClaw สามารถอ่าน manifest ของเจ้าของและใช้การแทนที่ API/URL ฐานของนามแฝงได้โดยไม่ต้องโหลดรันไทม์ของผู้ให้บริการ นามแฝงจะไม่ขยายรายการแค็ตตาล็อกที่ไม่ผ่านการกรอง โดยรายการแบบกว้างจะแสดงเฉพาะแถวผู้ให้บริการมาตรฐานของเจ้าของเท่านั้น

`suppressions` ใช้แทนฮุก `suppressBuiltInModel` ของรันไทม์ผู้ให้บริการแบบเก่า ระบบจะใช้รายการระงับเฉพาะเมื่อ Plugin เป็นเจ้าของผู้ให้บริการ หรือประกาศเป็นคีย์ `modelCatalog.aliases` ที่ชี้ไปยังผู้ให้บริการที่เป็นเจ้าของเท่านั้น ระบบจะไม่เรียกฮุกระงับของรันไทม์ระหว่างการแก้ไขโมเดลอีกต่อไป

ฟิลด์ผู้ให้บริการ:

| ฟิลด์                 | ชนิด                     | ความหมาย                                                                                                                                                                                                     |

| `api`                 | `ModelApi`               | อะแดปเตอร์ API เริ่มต้นที่เลือกกำหนดได้สำหรับโมเดลในแค็ตตาล็อกผู้ให้บริการนี้                                                                                                                                                 |

อย่าใส่ข้อมูลที่ใช้เฉพาะในรันไทม์ไว้ใน `modelCatalog` ใช้ `static` เฉพาะเมื่อแถวใน manifest สมบูรณ์เพียงพอให้พื้นผิวรายการที่กรองตามผู้ให้บริการและตัวเลือกข้ามการค้นหารีจิสทรี/รันไทม์ได้ ใช้ `refreshable` เมื่อแถวใน manifest เป็นข้อมูลตั้งต้นหรือส่วนเสริมที่มีประโยชน์และแสดงในรายการได้ แต่การรีเฟรช/แคชสามารถเพิ่มแถวเพิ่มเติมได้ในภายหลัง แถวที่รีเฟรชได้ไม่ถือเป็นแหล่งข้อมูลที่เชื่อถือได้ด้วยตัวเอง ใช้ `runtime` เมื่อ OpenClaw ต้องโหลดรันไทม์ของผู้ให้บริการเพื่อทราบรายการ

## ข้อมูลอ้างอิง modelIdNormalization

ใช้ `modelIdNormalization` สำหรับการปรับ ID โมเดลที่ผู้ให้บริการเป็นเจ้าของแบบต้นทุนต่ำ ซึ่งต้องเกิดขึ้นก่อนโหลดรันไทม์ของผู้ให้บริการ วิธีนี้จะเก็บนามแฝง เช่น ชื่อโมเดลแบบสั้น, ID เดิมภายในผู้ให้บริการ และกฎคำนำหน้าพร็อกซี ไว้ใน manifest ของ Plugin เจ้าของ แทนที่จะเก็บไว้ในตารางเลือกโมเดลของแกนหลัก

```json
{
  "providers": ["anthropic", "openrouter"],
  "modelIdNormalization": {
    "providers": {
      "anthropic": {
        "aliases": {
          "sonnet-4.6": "claude-sonnet-4-6"
        }
      },
      "openrouter": {
        "prefixWhenBare": "openrouter"
      }
    }
  }
}
```

ฟิลด์ผู้ให้บริการ:

| ฟิลด์                                | ชนิด                    | ความหมาย                                                                             |
| ------------------------------------ | ----------------------- | ----------------------------------------------------------------------------------------- |
| `aliases`                            | `Record<string,string>` | นามแฝง ID โมเดลที่ตรงกันทุกประการโดยไม่คำนึงถึงตัวพิมพ์เล็กหรือใหญ่ ระบบจะคืนค่าตามที่เขียนไว้                  |
| `stripPrefixes`                      | `string[]`              | คำนำหน้าที่จะลบออกก่อนค้นหานามแฝง มีประโยชน์สำหรับการซ้ำซ้อนของผู้ให้บริการ/โมเดลแบบเดิม     |
| `prefixWhenBare`                     | `string`                | คำนำหน้าที่จะเพิ่มเมื่อ ID โมเดลที่ปรับแล้วไม่มี `/` อยู่ก่อน                  |
| `prefixWhenBareAfterAliasStartsWith` | `object[]`              | กฎคำนำหน้า ID เปล่าแบบมีเงื่อนไขหลังค้นหานามแฝง โดยใช้ `modelPrefix` และ `prefix` เป็นคีย์ |

## ข้อมูลอ้างอิง providerEndpoints

ใช้ `providerEndpoints` สำหรับการจัดประเภทปลายทางที่นโยบายคำขอทั่วไปต้องทราบก่อนโหลดรันไทม์ของผู้ให้บริการ แกนหลักยังคงเป็นเจ้าของความหมายของ `endpointClass` แต่ละรายการ ส่วน manifest ของ Plugin เป็นเจ้าของเมทาดาทาโฮสต์และ URL ฐาน

Plugin ผู้ให้บริการที่แยกออกเป็นภายนอกอย่างเป็นทางการจะไม่รวมอยู่ในชุดแจกจ่ายของแกนหลัก ดังนั้น
manifest ของ Plugin เหล่านั้นจะมองไม่เห็นจนกว่าจะติดตั้ง `providerEndpoints` ของ Plugin เหล่านั้นต้อง
ถูกทำสำเนาไว้ใน `scripts/lib/official-external-provider-catalog.json` ด้วย เพื่อให้
การจัดประเภทปลายทางยังคงทำงานได้โดยไม่มี Plugin โดยมีการทดสอบสัญญา
บังคับให้ข้อมูลที่ทำสำเนาไว้ตรงกัน

ฟิลด์ปลายทาง:

| ฟิลด์                          | ชนิด       | ความหมาย                                                                                  |
| ------------------------------ | ---------- | ---------------------------------------------------------------------------------------------- |
| `endpointClass`                | `string`   | คลาสเอนด์พอยต์หลักที่รู้จัก เช่น `openrouter`, `moonshot-native` หรือ `google-vertex`        |
| `hosts`                        | `string[]` | ชื่อโฮสต์ที่ตรงกันทุกประการซึ่งแมปกับคลาสเอนด์พอยต์                                                |
| `hostSuffixes`                 | `string[]` | ส่วนต่อท้ายของโฮสต์ที่แมปกับคลาสเอนด์พอยต์ เติม `.` ไว้ข้างหน้าสำหรับการจับคู่เฉพาะส่วนต่อท้ายของโดเมน |
| `baseUrls`                     | `string[]` | URL ฐาน HTTP(S) ที่ปรับเป็นรูปแบบมาตรฐานและตรงกันทุกประการ ซึ่งแมปกับคลาสเอนด์พอยต์                             |
| `googleVertexRegion`           | `string`   | ภูมิภาค Google Vertex แบบคงที่สำหรับโฮสต์ส่วนกลางที่ตรงกันทุกประการ                                            |
| `googleVertexRegionHostSuffix` | `string`   | ส่วนต่อท้ายที่จะตัดออกจากโฮสต์ที่ตรงกัน เพื่อแสดงคำนำหน้าภูมิภาค Google Vertex                 |

## ข้อมูลอ้างอิง providerRequest

ใช้ `providerRequest` สำหรับเมทาดาทาความเข้ากันได้ของคำขอที่มีต้นทุนต่ำ ซึ่งนโยบายคำขอทั่วไปต้องใช้โดยไม่ต้องโหลดรันไทม์ของผู้ให้บริการ เก็บการเขียนเพย์โหลดใหม่ตามพฤติกรรมไว้ในฮุกของรันไทม์ผู้ให้บริการหรือตัวช่วยร่วมของตระกูลผู้ให้บริการ

```json
{
  "providerRequest": {
    "providers": {
      "vllm": {
        "family": "vllm",
        "openAICompletions": {
          "supportsStreamingUsage": true
        }
      }
    }
  }
}
```

ฟิลด์ผู้ให้บริการ:

| ฟิลด์                 | ชนิด         | ความหมาย                                                                          |
| --------------------- | ------------ | -------------------------------------------------------------------------------------- |
| `family`              | `string`     | ป้ายกำกับตระกูลผู้ให้บริการที่ใช้ในการตัดสินใจเกี่ยวกับความเข้ากันได้ของคำขอทั่วไปและการวินิจฉัย |
| `compatibilityFamily` | `"moonshot"` | กลุ่มความเข้ากันได้ของตระกูลผู้ให้บริการที่ไม่บังคับ สำหรับตัวช่วยคำขอร่วม              |
| `openAICompletions`   | `object`     | แฟล็กคำขอ completions ที่เข้ากันได้กับ OpenAI ซึ่งปัจจุบันคือ `supportsStreamingUsage`       |

## ข้อมูลอ้างอิง secretProviderIntegrations

ใช้ `secretProviderIntegrations` เมื่อ Plugin สามารถเผยแพร่ค่าที่ตั้งไว้ล่วงหน้าของผู้ให้บริการ exec สำหรับ SecretRef ที่นำกลับมาใช้ใหม่ได้ OpenClaw อ่านเมทาดาทานี้ก่อนโหลดรันไทม์ของ Plugin จัดเก็บความเป็นเจ้าของของ Plugin ไว้ใน `secrets.providers.<alias>.pluginIntegration` และปล่อยให้รันไทม์ SecretRef เป็นผู้แก้ไขค่าความลับจริง ค่าที่ตั้งไว้ล่วงหน้าจะแสดงเฉพาะสำหรับ Plugin ที่รวมมาในชุดและ Plugin ที่ติดตั้งซึ่งค้นพบจากรากการติดตั้ง Plugin ที่มีการจัดการ เช่น การติดตั้งจาก git และ ClawHub

```json
{
  "secretProviderIntegrations": {
    "secret-store": {
      "providerAlias": "team-secrets",
      "displayName": "Team secrets",
      "source": "exec",
      "command": "${node}",
      "args": ["./bin/resolve-secrets.mjs"]
    }
  }
}
```

คีย์ของแมปคือรหัสการผสานรวม หากละเว้น `providerAlias` OpenClaw จะใช้รหัสการผสานรวมเป็นนามแฝงผู้ให้บริการ SecretRef นามแฝงผู้ให้บริการต้องตรงกับรูปแบบนามแฝงผู้ให้บริการ SecretRef ตามปกติ เช่น `team-secrets` หรือ `onepassword-work`

เมื่อผู้ปฏิบัติงานเลือกค่าที่ตั้งไว้ล่วงหน้า OpenClaw จะเขียนการอ้างอิงผู้ให้บริการดังนี้:

```json
{
  "secrets": {
    "providers": {
      "team-secrets": {
        "source": "exec",
        "pluginIntegration": {
          "pluginId": "acme-secrets",
          "integrationId": "secret-store"
        }
      }
    }
  }
}
```

เมื่อเริ่มต้น/โหลดใหม่ OpenClaw จะแก้ไขผู้ให้บริการนั้นโดยโหลดเมทาดาทาไฟล์กำกับ Plugin ปัจจุบัน ตรวจสอบว่า Plugin เจ้าของได้รับการติดตั้งและทำงานอยู่ และสร้างคำสั่ง exec จากไฟล์กำกับ การปิดใช้งานหรือลบ Plugin จะเพิกถอนผู้ให้บริการสำหรับ SecretRef ที่ใช้งานอยู่ ผู้ปฏิบัติงานที่ต้องการการกำหนดค่า exec แบบสแตนด์อโลนยังสามารถเขียนผู้ให้บริการ `command`/`args` ด้วยตนเองได้โดยตรง

ขณะนี้รองรับเฉพาะค่าที่ตั้งไว้ล่วงหน้า `source: "exec"` เท่านั้น `command` ต้องเป็น `${node}` และ `args[0]` ต้องเป็นสคริปต์ตัวแก้ไข `./` ที่สัมพันธ์กับรากของ Plugin OpenClaw จะสร้างค่าเหล่านี้เมื่อเริ่มต้น/โหลดใหม่ให้เป็นไฟล์ปฏิบัติการ Node ปัจจุบันและพาธสัมบูรณ์ของสคริปต์ภายใน Plugin ตัวเลือก Node เช่น `--require`, `--import`, `--loader`, `--env-file`, `--eval` และ `--print` ไม่ได้เป็นส่วนหนึ่งของสัญญาค่าที่ตั้งไว้ล่วงหน้าในไฟล์กำกับ ผู้ปฏิบัติงานที่ต้องการคำสั่งที่ไม่ใช่ Node สามารถกำหนดค่าผู้ให้บริการ exec แบบสแตนด์อโลนด้วยตนเองได้โดยตรง

OpenClaw สร้าง `trustedDirs` สำหรับค่าที่ตั้งไว้ล่วงหน้าในไฟล์กำกับจากรากของ Plugin และสำหรับค่าที่ตั้งไว้ล่วงหน้า `${node}` จะใช้ไดเรกทอรีของไฟล์ปฏิบัติการ Node ปัจจุบัน ค่า `trustedDirs` ที่ระบุในไฟล์กำกับจะถูกละเว้น ตัวเลือกผู้ให้บริการ exec อื่นๆ เช่น `timeoutMs`, `noOutputTimeoutMs`, `maxOutputBytes`, `jsonOnly`, `env`, `passEnv` และ `allowInsecurePath` จะถูกส่งต่อไปยังการกำหนดค่าผู้ให้บริการ exec ของ SecretRef ตามปกติ

## ข้อมูลอ้างอิง modelPricing

ใช้ `modelPricing` เมื่อผู้ให้บริการต้องควบคุมพฤติกรรมการกำหนดราคาของระนาบควบคุมก่อนโหลดรันไทม์ แคชราคาของ Gateway อ่านเมทาดาทานี้โดยไม่ต้องนำเข้าโค้ดรันไทม์ของผู้ให้บริการ

```json
{
  "providers": ["ollama", "openrouter"],
  "modelPricing": {
    "providers": {
      "ollama": {
        "external": false
      },
      "openrouter": {
        "openRouter": {
          "passthroughProviderModel": true
        },
        "liteLLM": false
      }
    }
  }
}
```

ฟิลด์ผู้ให้บริการ:

| ฟิลด์        | ชนิด              | ความหมาย                                                                                      |
| ------------ | ----------------- | -------------------------------------------------------------------------------------------------- |
| `external`   | `boolean`         | ตั้งค่า `false` สำหรับผู้ให้บริการภายในเครื่อง/ที่โฮสต์เอง ซึ่งต้องไม่ดึงราคาจาก OpenRouter หรือ LiteLLM |
| `openRouter` | `false \| object` | การแมปการค้นหาราคาของ OpenRouter ค่า `false` จะปิดใช้งานการค้นหาจาก OpenRouter สำหรับผู้ให้บริการนี้           |
| `liteLLM`    | `false \| object` | การแมปการค้นหาราคาของ LiteLLM ค่า `false` จะปิดใช้งานการค้นหาจาก LiteLLM สำหรับผู้ให้บริการนี้                 |

ฟิลด์แหล่งที่มา:

| ฟิลด์                      | ชนิด               | ความหมาย                                                                                                        |
| -------------------------- | ------------------ | -------------------------------------------------------------------------------------------------------------------- |
| `provider`                 | `string`           | รหัสผู้ให้บริการแค็ตตาล็อกภายนอกเมื่อแตกต่างจากรหัสผู้ให้บริการ OpenClaw เช่น `z-ai` สำหรับผู้ให้บริการ `zai` |
| `passthroughProviderModel` | `boolean`          | ถือว่ารหัสโมเดลที่มีเครื่องหมายทับเป็นการอ้างอิงผู้ให้บริการ/โมเดลแบบซ้อน ซึ่งมีประโยชน์สำหรับผู้ให้บริการพร็อกซี เช่น OpenRouter       |
| `modelIdTransforms`        | `"version-dots"[]` | รูปแบบรหัสโมเดลของแค็ตตาล็อกภายนอกเพิ่มเติม `version-dots` จะลองใช้รหัสเวอร์ชันแบบมีจุด เช่น `claude-opus-4.6`            |

### ดัชนีผู้ให้บริการ OpenClaw

ดัชนีผู้ให้บริการ OpenClaw คือเมทาดาทาตัวอย่างที่ OpenClaw เป็นเจ้าของ สำหรับผู้ให้บริการที่อาจยังไม่ได้ติดตั้ง Plugin ข้อมูลนี้ไม่ใช่ส่วนหนึ่งของไฟล์กำกับ Plugin ไฟล์กำกับ Plugin ยังคงเป็นแหล่งข้อมูลหลักสำหรับ Plugin ที่ติดตั้งแล้ว ดัชนีผู้ให้บริการคือสัญญาสำรองภายในที่พื้นผิวสำหรับผู้ให้บริการที่ติดตั้งได้และตัวเลือกโมเดลก่อนการติดตั้งในอนาคตจะใช้ เมื่อยังไม่ได้ติดตั้ง Plugin ของผู้ให้บริการ

ลำดับอำนาจของแค็ตตาล็อก:

1. การกำหนดค่าของผู้ใช้
2. ไฟล์กำกับ Plugin ที่ติดตั้ง `modelCatalog`
3. แคชแค็ตตาล็อกโมเดลจากการรีเฟรชอย่างชัดเจน
4. แถวตัวอย่างในดัชนีผู้ให้บริการ OpenClaw

ดัชนีผู้ให้บริการต้องไม่มีค่าความลับ สถานะการเปิดใช้งาน ฮุกรันไทม์ หรือข้อมูลโมเดลแบบสดที่เฉพาะเจาะจงกับบัญชี แค็ตตาล็อกตัวอย่างใช้รูปแบบแถวผู้ให้บริการ `modelCatalog` เดียวกับไฟล์กำกับ Plugin แต่ควรจำกัดไว้เฉพาะเมทาดาทาการแสดงผลที่เสถียร เว้นแต่ฟิลด์อะแดปเตอร์รันไทม์ เช่น `api`, `baseUrl`, ราคา หรือแฟล็กความเข้ากันได้ จะถูกตั้งใจให้สอดคล้องกับไฟล์กำกับ Plugin ที่ติดตั้ง ผู้ให้บริการที่มีการค้นพบ `/models` แบบสดควรเขียนแถวที่รีเฟรชผ่านพาธแคชแค็ตตาล็อกโมเดลอย่างชัดเจน แทนที่จะให้การแสดงรายการหรือการเริ่มต้นใช้งานตามปกติเรียก API ของผู้ให้บริการ

รายการในดัชนีผู้ให้บริการอาจมีเมทาดาทาของ Plugin ที่ติดตั้งได้สำหรับผู้ให้บริการที่ย้ายออกจากแกนหลักหรือยังไม่ได้ติดตั้งด้วย เมทาดาทานี้สะท้อนรูปแบบแค็ตตาล็อกช่องทาง: ชื่อแพ็กเกจ ข้อกำหนดการติดตั้ง npm ค่าความสมบูรณ์ที่คาดไว้ และป้ายกำกับตัวเลือกการยืนยันตัวตนที่มีต้นทุนต่ำ เพียงพอสำหรับแสดงตัวเลือกการตั้งค่าที่ติดตั้งได้ เมื่อติดตั้ง Plugin แล้ว ไฟล์กำกับของ Plugin จะมีลำดับความสำคัญ และรายการในดัชนีผู้ให้บริการสำหรับผู้ให้บริการนั้นจะถูกละเว้น

`openclaw doctor --fix` ย้ายคีย์ความสามารถระดับบนสุดแบบเดิมของไฟล์กำกับจำนวนเล็กน้อยและเป็นชุดปิดไปยัง `contracts.*`: `speechProviders`, `mediaUnderstandingProviders`, `imageGenerationProviders` และ `tools` ปัจจุบันไม่มีคีย์เหล่านี้ (หรือรายการความสามารถอื่นใด) ที่ถูกอ่านเป็นฟิลด์ระดับบนสุดของไฟล์กำกับอีกต่อไป การโหลดไฟล์กำกับตามปกติจะรู้จักคีย์เหล่านี้เฉพาะภายใต้ `contracts` เท่านั้น

## ไฟล์กำกับเทียบกับ package.json

ไฟล์ทั้งสองมีหน้าที่ต่างกัน:

| ไฟล์                   | ใช้สำหรับ                                                                                                                       |
| ---------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| `openclaw.plugin.json` | การค้นพบ การตรวจสอบความถูกต้องของการกำหนดค่า เมทาดาทาตัวเลือกการยืนยันตัวตน และคำแนะนำ UI ที่ต้องมีอยู่ก่อนโค้ด Plugin ทำงาน                         |
| `package.json`         | เมทาดาทา npm การติดตั้งการขึ้นต่อกัน และบล็อก `openclaw` ที่ใช้สำหรับจุดเริ่มต้น เกณฑ์การติดตั้ง การตั้งค่า หรือเมทาดาทาแค็ตตาล็อก |

หากไม่แน่ใจว่าเมทาดาทาส่วนหนึ่งควรอยู่ที่ใด ให้ใช้กฎนี้:

- หาก OpenClaw ต้องทราบข้อมูลก่อนโหลดโค้ด Plugin ให้ใส่ไว้ใน `openclaw.plugin.json`
- หากเกี่ยวข้องกับการจัดแพ็กเกจ ไฟล์เริ่มต้น หรือพฤติกรรมการติดตั้ง npm ให้ใส่ไว้ใน `package.json`

### ฟิลด์ package.json ที่มีผลต่อการค้นพบ

เมทาดาทา Plugin ก่อนรันไทม์บางส่วนตั้งใจให้อยู่ใน `package.json` ภายใต้บล็อก `openclaw` แทนที่จะอยู่ใน `openclaw.plugin.json` ค่า `openclaw.bundle` และ `openclaw.bundle.json` ไม่ใช่สัญญา Plugin ของ OpenClaw โดย Plugin แบบเนทีฟต้องใช้ `openclaw.plugin.json` ร่วมกับฟิลด์ `package.json#openclaw` ที่รองรับด้านล่าง

ตัวอย่างสำคัญ:

| ฟิลด์                                                                                      | ความหมาย                                                                                                                                                                        |
| ------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `openclaw.extensions`                                                                      | ประกาศจุดเริ่มต้นของ Plugin แบบเนทีฟ ต้องอยู่ภายในไดเรกทอรีแพ็กเกจของ Plugin                                                                                                   |
| `openclaw.runtimeExtensions`                                                               | ประกาศจุดเริ่มต้นรันไทม์ JavaScript ที่บิลด์แล้วสำหรับแพ็กเกจที่ติดตั้ง ต้องอยู่ภายในไดเรกทอรีแพ็กเกจของ Plugin                                                                 |
| `openclaw.setupEntry`                                                                      | จุดเริ่มต้นน้ำหนักเบาสำหรับการตั้งค่าเท่านั้น ซึ่งใช้ระหว่างการเริ่มต้นใช้งาน การเลื่อนการเริ่มช่องทางออกไป และการตรวจหาสถานะช่องทาง/SecretRef แบบอ่านอย่างเดียว ต้องอยู่ภายในไดเรกทอรีแพ็กเกจของ Plugin |
| `openclaw.runtimeSetupEntry`                                                               | ประกาศจุดเริ่มต้นการตั้งค่า JavaScript ที่บิลด์แล้วสำหรับแพ็กเกจที่ติดตั้ง ต้องมี `setupEntry` ต้องมีอยู่จริง และต้องอยู่ภายในไดเรกทอรีแพ็กเกจของ Plugin                         |
| `openclaw.channel`                                                                         | เมทาดาทาแค็ตตาล็อกช่องทางที่มีต้นทุนต่ำ เช่น ป้ายกำกับ พาธเอกสาร นามแฝง และข้อความสำหรับการเลือก                                                                                                 |
| `openclaw.channel.approvalFlags`                                                           | แฟล็กลักษณะการอนุมัติแบบปิดที่ใช้ได้ก่อนโหลดรันไทม์ `native` หมายความว่าช่องทางเป็นเจ้าของ UI การอนุมัติแบบเนทีฟและการแก้ไขภายในเทิร์นเดียวกัน                                           |
| `openclaw.channel.commands`                                                                | เมทาดาทาค่าเริ่มต้นอัตโนมัติแบบคงที่สำหรับคำสั่งเนทีฟและ Skills แบบเนทีฟ ซึ่งใช้โดยพื้นผิวการกำหนดค่า การตรวจสอบ และรายการคำสั่งก่อนโหลดรันไทม์ช่องทาง                                          |
| `openclaw.channel.configuredState`                                                         | เมทาดาทาตัวตรวจสอบสถานะที่กำหนดค่าแล้วแบบน้ำหนักเบา ซึ่งตอบได้ว่า "มีการตั้งค่าโดยใช้เฉพาะตัวแปรสภาพแวดล้อมอยู่แล้วหรือไม่" โดยไม่ต้องโหลดรันไทม์ช่องทางฉบับเต็ม                                         |
| `openclaw.channel.persistedAuthState`                                                      | เมทาดาทาตัวตรวจสอบการยืนยันตัวตนที่คงอยู่แบบน้ำหนักเบา ซึ่งตอบได้ว่า "มีสิ่งใดลงชื่อเข้าใช้ไว้แล้วหรือไม่" โดยไม่ต้องโหลดรันไทม์ช่องทางฉบับเต็ม                                               |
| `openclaw.install.clawhubSpec` / `openclaw.install.npmSpec` / `openclaw.install.localPath` | คำแนะนำการติดตั้ง/อัปเดตสำหรับ Plugin ที่รวมมาให้และเผยแพร่ภายนอก                                                                                                                   |
| `openclaw.install.defaultChoice`                                                           | พาธการติดตั้งที่ต้องการเมื่อมีแหล่งติดตั้งหลายแหล่ง                                                                                                                  |
| `openclaw.install.minHostVersion`                                                          | เวอร์ชันโฮสต์ OpenClaw ขั้นต่ำที่รองรับ โดยใช้ขอบเขตล่างของ semver เช่น `>=2026.3.22` หรือ `>=2026.5.1-beta.1`                                                                             |
| `openclaw.compat.pluginApi`                                                                | ช่วง API ของ Plugin OpenClaw ขั้นต่ำที่แพ็กเกจนี้ต้องใช้ โดยใช้ขอบเขตล่างของ semver เช่น `>=2026.5.27`                                                                                 |
| `openclaw.install.expectedIntegrity`                                                       | สตริงความสมบูรณ์ของ npm dist ที่คาดไว้ เช่น `sha512-...`; ขั้นตอนการติดตั้งและอัปเดตจะตรวจสอบอาร์ติแฟกต์ที่ดึงมากับค่านี้                                                            |
| `openclaw.install.allowInvalidConfigRecovery`                                              | อนุญาตพาธการกู้คืนจากการติดตั้ง Plugin ที่รวมมาให้ซ้ำในขอบเขตจำกัด เมื่อการกำหนดค่าไม่ถูกต้อง                                                                                                       |
| `openclaw.install.requiredPlatformPackages`                                                | นามแฝงแพ็กเกจ npm ที่ต้องถูกสร้างขึ้นเมื่อข้อจำกัดด้านแพลตฟอร์มใน lockfile ตรงกับโฮสต์ปัจจุบัน                                                                           |
| `openclaw.startup.deferConfiguredChannelFullLoadUntilAfterListen`                          | ทำให้พื้นผิวช่องทางของรันไทม์การตั้งค่าโหลดได้ก่อนเริ่มรับฟัง แล้วเลื่อนการโหลด Plugin ช่องทางฉบับเต็มที่กำหนดค่าแล้วไปจนถึงการเปิดใช้งานหลังเริ่มรับฟัง                                                 |

เมทาดาทาของไฟล์รายการกำหนดตัวเลือกผู้ให้บริการ/ช่องทาง/การตั้งค่าที่ปรากฏในการเริ่มต้นใช้งานก่อนโหลดรันไทม์ `package.json#openclaw.install` บอกขั้นตอนการเริ่มต้นใช้งานว่าต้องดึงหรือเปิดใช้ Plugin นั้นอย่างไรเมื่อผู้ใช้เลือกหนึ่งในตัวเลือกเหล่านั้น อย่าย้ายคำแนะนำการติดตั้งไปไว้ใน `openclaw.plugin.json`

`openclaw.install.minHostVersion` ถูกบังคับใช้ระหว่างการติดตั้งและการโหลดรีจิสทรีไฟล์รายการสำหรับแหล่ง Plugin ที่ไม่ได้รวมมาให้ ค่าที่ไม่ถูกต้องจะถูกปฏิเสธ ส่วนค่าที่ใหม่กว่าแต่ถูกต้องจะทำให้ข้าม Plugin ภายนอกบนโฮสต์รุ่นเก่า ถือว่า Plugin จากซอร์สที่รวมมาให้ใช้เวอร์ชันเดียวกับเช็กเอาต์ของโฮสต์

`openclaw.install.requiredPlatformPackages` ใช้สำหรับแพ็กเกจ npm ที่เปิดเผยไบนารีเนทีฟที่จำเป็นผ่านนามแฝงแบบไม่บังคับซึ่งเจาะจงแพลตฟอร์ม ให้ระบุชื่อแพ็กเกจ npm แบบไม่มีส่วนขยายสำหรับนามแฝงของทุกแพลตฟอร์มที่รองรับ ระหว่างการติดตั้ง npm OpenClaw จะตรวจสอบเฉพาะนามแฝงที่ประกาศไว้ซึ่งข้อจำกัดใน lockfile ตรงกับโฮสต์ปัจจุบัน หาก npm รายงานว่าสำเร็จแต่ละเว้นนามแฝงนั้น OpenClaw จะลองใหม่หนึ่งครั้งด้วยแคชใหม่ และย้อนกลับการติดตั้งหากนามแฝงยังคงหายไป

`openclaw.compat.pluginApi` ถูกบังคับใช้ระหว่างการติดตั้งแพ็กเกจสำหรับแหล่ง Plugin ที่ไม่ได้รวมมาให้ ใช้ค่านี้เป็นขอบเขตล่างของ API รันไทม์/SDK ของ Plugin OpenClaw ที่ใช้บิลด์แพ็กเกจ ค่านี้อาจเข้มงวดกว่า `minHostVersion` เมื่อแพ็กเกจ Plugin ต้องใช้ API ที่ใหม่กว่า แต่ยังคงคำแนะนำการติดตั้งที่ต่ำกว่าไว้สำหรับขั้นตอนอื่น ตามค่าเริ่มต้น การซิงค์รุ่นเผยแพร่อย่างเป็นทางการของ OpenClaw จะปรับขอบเขตล่าง API ของ Plugin อย่างเป็นทางการที่มีอยู่ให้เป็นเวอร์ชันรุ่นเผยแพร่ของ OpenClaw แต่รุ่นเผยแพร่เฉพาะ Plugin สามารถคงขอบเขตล่างที่ต่ำกว่าไว้ได้เมื่อแพ็กเกจตั้งใจรองรับโฮสต์รุ่นเก่า อย่าใช้เวอร์ชันแพ็กเกจเพียงอย่างเดียวเป็นสัญญาความเข้ากันได้ `peerDependencies.openclaw` ยังคงเป็นเมทาดาทาแพ็กเกจ npm; OpenClaw ใช้สัญญา `openclaw.compat.pluginApi` สำหรับการตัดสินใจด้านความเข้ากันได้ในการติดตั้ง

เมทาดาทาการติดตั้งตามต้องการอย่างเป็นทางการควรใช้ `clawhubSpec` เมื่อ Plugin เผยแพร่บน ClawHub; ขั้นตอนการเริ่มต้นใช้งานจะถือว่านั่นเป็นแหล่งระยะไกลที่ต้องการ และบันทึกข้อมูลอาร์ติแฟกต์ของ ClawHub หลังการติดตั้ง `npmSpec` ยังคงเป็นทางเลือกสำรองด้านความเข้ากันได้สำหรับแพ็กเกจที่ยังไม่ได้ย้ายไปยัง ClawHub

การตรึงเวอร์ชัน npm แบบเจาะจงมีอยู่แล้วใน `npmSpec` ตัวอย่างเช่น `"npmSpec": "@wecom/wecom-openclaw-plugin@1.2.3"` รายการแค็ตตาล็อกภายนอกอย่างเป็นทางการควรจับคู่ข้อกำหนดแบบเจาะจงกับ `expectedIntegrity` เพื่อให้ขั้นตอนการอัปเดตปฏิเสธโดยค่าเริ่มต้นหากอาร์ติแฟกต์ npm ที่ดึงมาไม่ตรงกับรุ่นเผยแพร่ที่ตรึงไว้อีกต่อไป การเริ่มต้นใช้งานแบบโต้ตอบยังคงเสนอข้อกำหนด npm จากรีจิสทรีที่เชื่อถือได้ รวมถึงชื่อแพ็กเกจแบบไม่มีส่วนขยายและ dist-tags เพื่อความเข้ากันได้ การวินิจฉัยแค็ตตาล็อกสามารถแยกแยะแหล่งที่มาแบบเจาะจง แบบลอยตัว แบบตรึงความสมบูรณ์ ไม่มีความสมบูรณ์ ชื่อแพ็กเกจไม่ตรงกัน และตัวเลือกเริ่มต้นไม่ถูกต้อง นอกจากนี้ยังเตือนเมื่อมี `expectedIntegrity` แต่ไม่มีแหล่ง npm ที่ถูกต้องให้ตรึง เมื่อมี `expectedIntegrity` ขั้นตอนการติดตั้ง/อัปเดตจะบังคับใช้ค่านี้; เมื่อไม่ได้ระบุ ระบบจะบันทึกผลการแก้ไขรีจิสทรีโดยไม่มีการตรึงความสมบูรณ์

Plugin ช่องทางควรมี `openclaw.setupEntry` เมื่อการสแกนสถานะ รายการช่องทาง หรือ SecretRef จำเป็นต้องระบุบัญชีที่กำหนดค่าไว้โดยไม่โหลดรันไทม์ฉบับเต็ม จุดเริ่มต้นการตั้งค่าควรเปิดเผยเมทาดาทาช่องทาง รวมถึงอะแดปเตอร์การกำหนดค่า สถานะ และข้อมูลลับที่ปลอดภัยสำหรับการตั้งค่า; ให้คงไคลเอนต์เครือข่าย ตัวรับฟัง Gateway และรันไทม์การรับส่งไว้ในจุดเริ่มต้นส่วนขยายหลัก

ฟิลด์จุดเริ่มต้นรันไทม์ไม่แทนที่การตรวจสอบขอบเขตแพ็กเกจสำหรับฟิลด์จุดเริ่มต้นซอร์ส ตัวอย่างเช่น `openclaw.runtimeExtensions` ไม่สามารถทำให้พาธ `openclaw.extensions` ที่หลุดออกนอกขอบเขตโหลดได้

`openclaw.install.allowInvalidConfigRecovery` มีขอบเขตจำกัดโดยเจตนา ไม่ได้ทำให้การกำหนดค่าที่เสียหายใด ๆ ติดตั้งได้ ปัจจุบันอนุญาตให้ขั้นตอนการติดตั้งกู้คืนจากความล้มเหลวเฉพาะบางกรณีของการอัปเกรด Plugin ที่รวมมาให้ซึ่งล้าสมัยเท่านั้น เช่น พาธ Plugin ที่รวมมาให้หายไป หรือรายการ `channels.<id>` ที่ล้าสมัยสำหรับ Plugin ที่รวมมาให้ตัวเดียวกัน ข้อผิดพลาดการกำหนดค่าอื่นที่ไม่เกี่ยวข้องยังคงบล็อกการติดตั้งและส่งผู้ดูแลระบบไปยัง `openclaw doctor --fix`

`openclaw.channel.persistedAuthState` คือเมทาดาทาแพ็กเกจสำหรับโมดูลตัวตรวจสอบขนาดเล็ก:

```json
{
  "openclaw": {
    "channel": {
      "id": "whatsapp",
      "persistedAuthState": {
        "specifier": "./auth-presence",
        "exportName": "hasAnyWhatsAppAuth"
      }
    }
  }
}
```

ใช้เมื่อขั้นตอนการตั้งค่า doctor สถานะ หรือขั้นตอนตรวจสอบการมีอยู่แบบอ่านอย่างเดียว ต้องการโพรบการยืนยันตัวตนแบบใช่/ไม่ใช่ที่มีต้นทุนต่ำก่อนโหลด Plugin ช่องทางฉบับเต็ม สถานะการยืนยันตัวตนที่คงอยู่ไม่ใช่สถานะช่องทางที่กำหนดค่าไว้: อย่าใช้เมทาดาทานี้เพื่อเปิดใช้ Plugin โดยอัตโนมัติ ซ่อมแซมการขึ้นต่อกันของรันไทม์ หรือตัดสินใจว่าควรโหลดรันไทม์ช่องทางหรือไม่ เอ็กซ์พอร์ตเป้าหมายควรเป็นฟังก์ชันขนาดเล็กที่อ่านเฉพาะสถานะที่คงอยู่; อย่าส่งผ่าน barrel ของรันไทม์ช่องทางฉบับเต็ม

`openclaw.channel.configuredState` รองรับการตรวจสอบการกำหนดค่าที่มีต้นทุนต่ำ ควรเลือกใช้เมทาดาทาสภาพแวดล้อมแบบประกาศเมื่อใช้ตัวแปรสภาพแวดล้อมก็เพียงพอ:

```json
{
  "openclaw": {
    "channel": {
      "id": "telegram",
      "configuredState": {
        "env": {
          "allOf": ["TELEGRAM_BOT_TOKEN"]
        }
      }
    }
  }
}
```

ใช้ `env.allOf` เมื่อจำเป็นต้องมีตัวแปรที่ระบุไว้ทุกตัว และใช้ `env.anyOf` เมื่อตัวแปรใดตัวแปรหนึ่งที่ไม่ว่างก็เพียงพอ หากการตรวจสอบขนาดเล็กที่ไม่ใช่รันไทม์ต้องการมากกว่าเมทาดาทาสภาพแวดล้อม ให้ใช้ `specifier` ร่วมกับ `exportName` ดังที่แสดงสำหรับ `persistedAuthState`; เมื่อมี `env` OpenClaw จะใช้ค่านี้โดยไม่โหลดโมดูลนั้น หากการตรวจสอบต้องใช้การแก้ไขการกำหนดค่าฉบับเต็มหรือรันไทม์ช่องทางจริง ให้คงตรรกะนั้นไว้ในฮุก `config.hasConfiguredState` ของ Plugin แทน

## ลำดับความสำคัญในการค้นหา (รหัส Plugin ซ้ำกัน)

OpenClaw ค้นหา Plugin จากสามราก โดยตรวจสอบตามลำดับดังนี้: Plugin ที่รวมมากับ OpenClaw, รากการติดตั้งส่วนกลาง (`~/.openclaw/extensions`) และรากพื้นที่ทำงานปัจจุบัน (`<workspace>/.openclaw/extensions`) รวมถึงรายการ `plugins.load.paths` ที่ระบุไว้อย่างชัดเจน

หากผลการค้นหาสองรายการมี `id` เดียวกัน ระบบจะเก็บเฉพาะไฟล์รายการที่มี**ลำดับความสำคัญสูงสุด**; รายการซ้ำที่มีลำดับความสำคัญต่ำกว่าจะถูกทิ้งแทนที่จะโหลดควบคู่กัน ลำดับความสำคัญจากสูงสุดไปต่ำสุด:

1. **เลือกโดยการกำหนดค่า** — พาธที่ตรึงไว้อย่างชัดเจนใน `plugins.entries.<id>`
2. **การติดตั้งส่วนกลางที่ตรงกับระเบียนการติดตั้งที่ติดตามไว้** — Plugin ที่ติดตั้งผ่าน `openclaw plugin install`/`openclaw plugin update` ซึ่งระบบติดตามการติดตั้งของ OpenClaw จดจำได้สำหรับรหัสเดียวกันนั้น แม้ว่ารหัสดังกล่าวจะเป็นของ Plugin ที่รวมมาให้ด้วย
3. **รวมมาให้** — Plugin ที่จัดส่งมากับ OpenClaw
4. **พื้นที่ทำงาน** — Plugin ที่ค้นพบโดยอ้างอิงจากพื้นที่ทำงานปัจจุบัน
5. ตัวเลือกอื่นใดที่ค้นพบ

ผลที่ตามมา:

- สำเนาของ Plugin ที่รวมมาในชุดซึ่งถูก fork หรือเก่าเกินไปและค้างอยู่โดยไม่ได้ติดตามในพื้นที่ทำงานหรือรูทส่วนกลาง จะไม่บดบังบิลด์ที่รวมมาในชุด
- หากต้องการแทนที่ Plugin ที่รวมมาในชุด ให้เรียกใช้ `openclaw plugin install` สำหรับ id นั้น เพื่อให้การติดตั้งส่วนกลางที่ติดตามอยู่มีลำดับเหนือกว่าสำเนาที่รวมมาในชุด หรือปักหมุดพาธเฉพาะผ่าน `plugins.entries.<id>` เพื่อให้พาธนั้นชนะตามลำดับความสำคัญที่เลือกโดยการกำหนดค่า
- ระบบจะบันทึกสำเนาที่ซ้ำและถูกตัดทิ้ง เพื่อให้ Doctor และการวินิจฉัยเมื่อเริ่มต้นระบบระบุสำเนาที่ถูกละทิ้งได้
- การแทนที่รายการซ้ำที่เลือกโดยการกำหนดค่าจะแสดงในการวินิจฉัยว่าเป็นการแทนที่อย่างชัดเจน แต่ยังคงมีคำเตือนเพื่อให้มองเห็น fork ที่เก่าเกินไปและการบดบังโดยไม่ตั้งใจ

## ข้อกำหนด JSON Schema

- **Plugin ทุกตัวต้องมาพร้อม JSON Schema** แม้จะไม่รับการกำหนดค่าใดๆ
- อนุญาตให้ใช้สคีมาว่างได้ (ตัวอย่างเช่น `{ "type": "object", "additionalProperties": false }`)
- ระบบจะตรวจสอบความถูกต้องของสคีมาขณะอ่าน/เขียนการกำหนดค่า ไม่ใช่ขณะรันไทม์
- เมื่อขยายหรือ fork Plugin ที่รวมมาในชุดด้วยคีย์การกำหนดค่าใหม่ ให้อัปเดต `openclaw.plugin.json` `configSchema` ของ Plugin นั้นพร้อมกัน สคีมาของ Plugin ที่รวมมาในชุดมีความเข้มงวด ดังนั้นการเพิ่ม `plugins.entries.<id>.config.myNewKey` ในการกำหนดค่าของผู้ใช้โดยไม่เพิ่ม `myNewKey` ลงใน `configSchema.properties` จะถูกปฏิเสธก่อนโหลดรันไทม์ของ Plugin

ตัวอย่างการขยายสคีมา:

```json
{
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {
      "myNewKey": {
        "type": "string"
      }
    }
  }
}
```

## ลักษณะการตรวจสอบความถูกต้อง

- คีย์ `channels.*` ที่ไม่รู้จักถือเป็น **ข้อผิดพลาด** เว้นแต่ id ของช่องทางนั้นจะประกาศไว้ในแมนิเฟสต์ของ Plugin หาก id เดียวกันปรากฏใน `plugins.allow`, `plugins.entries` หรือ `plugins.installs` ด้วย (Plugin ที่มีการอ้างอิงถึงแต่ยังค้นพบไม่ได้ในขณะนี้) OpenClaw จะลดระดับเป็น **คำเตือน** แทน
- `plugins.entries.<id>`, `plugins.allow` และ `plugins.deny` ที่อ้างอิง id ของ Plugin ที่ไม่รู้จักถือเป็น **คำเตือน** ("ละเว้นรายการกำหนดค่าที่เก่าเกินไป") ไม่ใช่ข้อผิดพลาด เพื่อไม่ให้การอัปเกรดและ Plugin ที่ถูกนำออก/เปลี่ยนชื่อขัดขวางการเริ่มต้น Gateway
- `plugins.slots.memory` ที่อ้างอิง id ของ Plugin ที่ไม่รู้จักถือเป็น **ข้อผิดพลาด** ยกเว้น Plugin ภายนอกอย่างเป็นทางการ `memory-lancedb` ที่ทราบอยู่แล้ว ซึ่งจะแสดงคำเตือนแทน
- หากติดตั้ง Plugin แล้ว แต่แมนิเฟสต์หรือสคีมาเสียหายหรือหายไป การตรวจสอบความถูกต้องจะล้มเหลวและ Doctor จะรายงานข้อผิดพลาดของ Plugin
- หากมีการกำหนดค่าของ Plugin แต่ Plugin **ถูกปิดใช้งาน** ระบบจะเก็บการกำหนดค่าไว้และแสดง **คำเตือน** ใน Doctor และบันทึก

ดูสคีมา `plugins.*` ฉบับเต็มได้ที่ [ข้อมูลอ้างอิงการกำหนดค่า](/th/gateway/configuration)

## หมายเหตุ

- แมนิเฟสต์เป็น **ข้อกำหนดบังคับสำหรับ Plugin แบบเนทีฟของ OpenClaw** รวมถึงการโหลดจากระบบไฟล์ภายในเครื่อง รันไทม์ยังคงโหลดโมดูล Plugin แยกต่างหาก ส่วนแมนิเฟสต์ใช้สำหรับการค้นพบและการตรวจสอบความถูกต้องเท่านั้น
- แมนิเฟสต์แบบเนทีฟจะถูกแยกวิเคราะห์ด้วย JSON5 จึงรองรับความคิดเห็น เครื่องหมายจุลภาคปิดท้าย และคีย์ที่ไม่ใส่เครื่องหมายอัญประกาศ ตราบใดที่ค่าสุดท้ายยังเป็นออบเจ็กต์
- ตัวโหลดแมนิเฟสต์จะอ่านเฉพาะฟิลด์แมนิเฟสต์ที่มีการจัดทำเอกสารไว้เท่านั้น หลีกเลี่ยงคีย์ระดับบนสุดที่กำหนดเอง
- สามารถละเว้น `channels`, `providers`, `cliBackends` และ `skills` ได้ทั้งหมดเมื่อ Plugin ไม่จำเป็นต้องใช้
- `providerCatalogEntry` ต้องมีขนาดเบาและไม่ควรนำเข้าโค้ดรันไทม์ในวงกว้าง ให้ใช้สำหรับเมทาดาทาแค็ตตาล็อกผู้ให้บริการแบบคงที่หรือตัวอธิบายการค้นพบที่มีขอบเขตแคบ ไม่ใช่การดำเนินการในเวลาที่มีคำขอ
- ชนิด Plugin แบบเอกสิทธิ์จะถูกเลือกผ่าน `plugins.slots.*`: `kind: "memory"` ผ่าน `plugins.slots.memory` (ค่าเริ่มต้น `memory-core`), `kind: "context-engine"` ผ่าน `plugins.slots.contextEngine` (ค่าเริ่มต้น `legacy`)
- ประกาศชนิด Plugin แบบเอกสิทธิ์ในแมนิเฟสต์นี้ `OpenClawPluginDefinition.kind` ของจุดเข้ารันไทม์เลิกแนะนำให้ใช้แล้ว และคงไว้เพียงเป็นทางเลือกสำรองเพื่อความเข้ากันได้สำหรับ Plugin รุ่นเก่า
- เมทาดาทาของตัวแปรสภาพแวดล้อม (`setup.providers[].envVars`, `providerAuthEnvVars` ที่เลิกแนะนำให้ใช้แล้ว และ `channelEnvVars`) เป็นเพียงการประกาศเท่านั้น สถานะ การตรวจสอบ การตรวจสอบความถูกต้องของการส่งมอบ Cron และพื้นผิวแบบอ่านอย่างเดียวอื่นๆ ยังคงใช้นโยบายความน่าเชื่อถือและการเปิดใช้งานที่มีผลของ Plugin ก่อนถือว่าตัวแปรสภาพแวดล้อมได้รับการกำหนดค่าแล้ว
- สำหรับเมทาดาทาของตัวช่วยตั้งค่ารันไทม์ที่ต้องใช้โค้ดผู้ให้บริการ โปรดดู [ฮุกของรันไทม์ผู้ให้บริการ](/th/plugins/architecture-internals#provider-runtime-hooks)
- หาก Plugin ของคุณขึ้นอยู่กับโมดูลเนทีฟ ให้จัดทำเอกสารขั้นตอนการบิลด์และข้อกำหนดรายการอนุญาตของตัวจัดการแพ็กเกจ (ตัวอย่างเช่น pnpm `allow-build-scripts` + `pnpm rebuild <package>`)
