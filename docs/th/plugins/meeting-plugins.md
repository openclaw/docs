OpenClaw มี Plugin แยกกันสำหรับ Google Meet, การประชุม Microsoft Teams และ Zoom ทั้งสามสามารถเข้าร่วมผ่าน Chrome ใช้โหมดการเข้าร่วมแบบเดียวกัน และเรียกใช้ Chrome บนโฮสต์ Gateway หรือ Node ที่จับคู่ไว้ก็ได้ แต่ URL ของแพลตฟอร์ม รูปแบบการติดตั้ง และความสามารถเพิ่มเติมจะแตกต่างกัน

Plugin เหล่านี้ใช้สำหรับเข้าร่วมการประชุม โดยแยกจากช่องทางรับส่งข้อความ เช่น [ช่องทาง Microsoft Teams](/th/channels/msteams) และ [Plugin การโทรด้วยเสียง](/th/plugins/voice-call)

## เลือก Plugin

| แพลตฟอร์ม        | Plugin                                      | ลิงก์การประชุมที่รองรับ                                                                                      | การติดตั้ง                             | วิธีการเข้าร่วม                                      | ความสามารถเฉพาะแพลตฟอร์ม                                                                                |
| --------------- | ------------------------------------------- | ----------------------------------------------------------------------------------------------------------- | ---------------------------------------- | -------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| Google Meet     | [`google-meet`](/th/plugins/google-meet)       | `meet.google.com/...`                                                                                       | ติดตั้งจาก npm หรือ ClawHub แล้วเปิดใช้งาน | Chrome ภายในเครื่อง, Chrome บน Node ที่จับคู่ไว้ หรือโทรเข้าผ่าน Twilio | สร้างการประชุมผ่าน Meet API หรือเบราว์เซอร์ที่ลงชื่อเข้าใช้แล้วได้ และอ่านอาร์ติแฟกต์ Meet ที่รองรับด้วย OAuth ได้ |
| Microsoft Teams | [`teams-meetings`](/plugins/teams-meetings) | ลิงก์สำหรับที่ทำงานภายใต้ `teams.microsoft.com/l/meetup-join/...` และลิงก์สำหรับผู้บริโภคภายใต้ `teams.live.com/meet/...` | รวมมาให้แล้ว ให้เปิดใช้งาน                      | Chrome ภายในเครื่องหรือ Chrome บน Node ที่จับคู่ไว้                  | เข้าร่วมในฐานะผู้เยี่ยมชมได้ทั้งการประชุมสำหรับที่ทำงานและผู้บริโภค                                                                     |
| Zoom            | [`zoom-meetings`](/plugins/zoom-meetings)   | `zoom.us/j/...` และโดเมนย่อยของบัญชี เช่น `example.zoom.us/j/...`                                      | รวมมาให้แล้ว ให้เปิดใช้งาน                      | Chrome ภายในเครื่องหรือ Chrome บน Node ที่จับคู่ไว้                  | เข้าร่วมในฐานะผู้เยี่ยมชมผ่าน Zoom Web App                                                                           |

เลือก Google Meet เมื่อต้องการสร้างการประชุม ใช้อาร์ติแฟกต์จาก Google API หรือใช้เส้นทางโทรศัพท์ผ่าน Twilio เลือก Teams หรือ Zoom เมื่อต้องการเข้าร่วมในฐานะผู้เยี่ยมชมโดยตรงผ่านเบราว์เซอร์บนแพลตฟอร์มเหล่านั้น Plugin ของ Teams และ Zoom ไม่สร้างการประชุม ไม่โทรเข้า ไม่เรียกใช้ API ของผู้ให้บริการ และไม่บันทึกการประชุม

## เลือกโหมด

Plugin ทั้งสามใช้โหมดเดียวกัน:

| โหมด         | ลักษณะการทำงาน                                                                                              | ข้อกำหนดด้านเสียง                                      |
| ------------ | ----------------------------------------------------------------------------------------------------- | ------------------------------------------------------- |
| `agent`      | ส่งการถอดเสียงแบบเรียลไทม์ไปยังเอเจนต์ OpenClaw ที่กำหนดค่าไว้ และใช้ TTS ปกติของ OpenClaw พูดคำตอบ  | การส่งเสียงตอบกลับผ่าน Chrome ต้องใช้บริดจ์ BlackHole และ SoX |
| `bidi`       | โมเดลเสียงแบบเรียลไทม์รับฟังและตอบกลับโดยตรง                                                  | การส่งเสียงตอบกลับผ่าน Chrome ต้องใช้บริดจ์ BlackHole และ SoX |
| `transcribe` | เข้าร่วมแบบสังเกตการณ์เท่านั้น และแสดงบทถอดคำบรรยายสดที่มีขอบเขตจำกัดเมื่อแพลตฟอร์มมีคำบรรยาย | ไม่ต้องใช้บริดจ์ส่งเสียงตอบกลับ BlackHole หรือ SoX                   |

ใช้ `transcribe` เมื่อเอเจนต์ต้องการเฉพาะข้อความจากการประชุม ใช้ `agent` สำหรับการให้เหตุผลและเครื่องมือปกติของ OpenClaw ใช้ `bidi` เมื่อเสียงตอบกลับโดยตรงที่มีความหน่วงต่ำสำคัญกว่าการส่งแต่ละรอบผ่านเอเจนต์ปกติ

บทถอดคำบรรยายเป็นข้อมูลรันไทม์ที่จำกัดอยู่ในเซสชัน ไม่ใช่การบันทึกการประชุมแบบถาวร ความพร้อมใช้งานของคำบรรยายยังขึ้นอยู่กับแพลตฟอร์มการประชุม บัญชี ภาษา และนโยบายของโฮสต์ โปรดดูข้อจำกัดของบทถอดเสียงและฟิลด์สถานะในคู่มือของแต่ละแพลตฟอร์ม

## เตรียม Chrome และเสียง

Chrome สามารถทำงานบนโฮสต์ Gateway หรือ Node ที่จับคู่ไว้ได้ Node Chrome ระยะไกลต้องอนุญาต `browser.proxy` รวมถึงคำสั่งของแพลตฟอร์ม:

| Plugin          | คำสั่ง Node           |
| --------------- | ---------------------- |
| Google Meet     | `googlemeet.chrome`    |
| Microsoft Teams | `teamsmeetings.chrome` |
| Zoom            | `zoommeetings.chrome`  |

สำหรับโหมด `agent` หรือ `bidi` ผ่าน Chrome ให้เรียกใช้ Chrome บน macOS และติดตั้งการขึ้นต่อกันด้านเสียงที่ใช้ร่วมกันบนโฮสต์เดียวกัน:

```bash
brew install blackhole-2ch sox
sudo reboot
system_profiler SPAudioDataType | grep -i BlackHole
command -v sox
```

โฮสต์ Gateway ยังคงเป็นเจ้าของเอเจนต์ OpenClaw และข้อมูลประจำตัวของโมเดลเมื่อ Chrome ทำงานบน Node ที่จับคู่ไว้ สำหรับโหมด `agent` ให้กำหนดค่าผู้ให้บริการถอดเสียงแบบเรียลไทม์และ TTS ของ OpenClaw หรือกำหนดค่าผู้ให้บริการเสียงแบบเรียลไทม์สำหรับโหมด `bidi` คู่มือของแต่ละแพลตฟอร์มมีตัวเลือกผู้ให้บริการและคำสั่งเสียง

## เปิดใช้งาน Plugin

ติดตั้ง Google Meet ก่อนเปิดใช้งาน ส่วนการประชุม Teams และ Zoom รวมอยู่ใน OpenClaw แล้วและต้องเปิดใช้งานเท่านั้น:

```bash
# Google Meet เท่านั้น
openclaw plugins install npm:@openclaw/google-meet

# เปิดใช้งานเฉพาะ Plugin การประชุมที่ใช้
openclaw plugins enable google-meet
openclaw plugins enable teams-meetings
openclaw plugins enable zoom-meetings
```

รีสตาร์ต Gateway หากวิธีจัดการ Plugin ที่ใช้ไม่ได้รีสตาร์ตโดยอัตโนมัติ จากนั้นเรียกใช้การตรวจสอบการตั้งค่าของแพลตฟอร์มก่อนเข้าร่วม

## ตรวจสอบและเข้าร่วม

| แพลตฟอร์ม        | การตรวจสอบการตั้งค่า                    | คำสั่งเข้าร่วม                                                                  |
| --------------- | ------------------------------ | ----------------------------------------------------------------------------- |
| Google Meet     | `openclaw googlemeet setup`    | `openclaw googlemeet join 'https://meet.google.com/abc-defg-hij'`             |
| Microsoft Teams | `openclaw teamsmeetings setup` | `openclaw teamsmeetings join 'https://teams.microsoft.com/l/meetup-join/...'` |
| Zoom            | `openclaw zoommeetings setup`  | `openclaw zoommeetings join 'https://zoom.us/j/1234567890'`                   |

ให้ถือว่าการตรวจสอบการตั้งค่าที่ล้มเหลวเป็นตัวขัดขวางสำหรับช่องทางการรับส่งและโหมดนั้น สำหรับการทดสอบเบื้องต้นแบบสังเกตการณ์เท่านั้น ให้เลือกโหมด `transcribe` และยืนยันว่าสถานะรายงานเซสชันที่อยู่ในการโทรก่อนคาดหวังข้อความคำบรรยาย

## จัดการพรอมต์นโยบายของแพลตฟอร์ม

ระบบอัตโนมัติของเบราว์เซอร์จัดการส่วนควบคุมตามปกติสำหรับชื่อผู้เยี่ยมชม กล้องและไมโครโฟนก่อนเข้าร่วม การเข้าร่วม ขณะอยู่ในการโทร และการออกจากการโทร แต่ไม่หลีกเลี่ยงนโยบายของแพลตฟอร์มหรือผู้จัด

- Google Meet อาจกำหนดให้ลงชื่อเข้าใช้ Google, รอการอนุมัติจากโฮสต์ หรือตัดสินใจเกี่ยวกับสิทธิ์ของเบราว์เซอร์
- Microsoft Teams อาจกำหนดให้ลงชื่อเข้าใช้ผู้เช่า ยืนยันอีเมล หรือรอการอนุมัติจากผู้จัด
- Zoom อาจกำหนดให้ยืนยันตัวตน ยืนยันอีเมล ป้อนรหัสผ่าน ทำ CAPTCHA ให้เสร็จ หรือรอการอนุมัติจากโฮสต์ และบัญชียังสามารถปิดใช้งานการเข้าร่วมผ่านเบราว์เซอร์ได้

เมื่อผลลัพธ์การเข้าร่วมหรือสถานะรายงาน `manualActionRequired` ให้ดำเนินการตามขั้นตอนที่รายงานในโปรไฟล์ Chrome ของ OpenClaw เดิมให้เสร็จก่อนลองอีกครั้ง การเปิดแท็บใหม่ซ้ำ ๆ ไม่สามารถแก้ไขข้อจำกัดของบัญชี ผู้เช่า ล็อบบี้ หรือ CAPTCHA ได้

เข้าร่วมเฉพาะการประชุมที่ผู้ควบคุมได้รับอนุญาตให้เพิ่มเอเจนต์เท่านั้น แจ้งผู้เข้าร่วมเมื่อนโยบายภายในหรือกฎความยินยอมกำหนดให้ต้องเปิดเผยการเข้าร่วมอัตโนมัติ การถอดเสียง หรือเสียงสังเคราะห์

## การแชตด้วยเสียงใน Discord

[ช่องเสียง Discord](/th/channels/discord#voice-channels) รองรับการสนทนาแบบเรียลไทม์ที่มีเฉพาะเสียงโดยตรง โดยไม่ต้องใช้ระบบประชุมอัตโนมัติผ่านเบราว์เซอร์ OpenClaw สามารถเข้าร่วมช่องเสียง รับฟัง ส่งแต่ละรอบผ่านเอเจนต์ OpenClaw หรือโมเดลเสียงแบบเรียลไทม์ และพูดคำตอบได้ โดยจะไม่ส่งหรือรับวิดีโอจากกล้องหรือการแชร์หน้าจอ แม้ว่าผู้คนจะใช้วิดีโอในช่อง Discord เดียวกันก็ตาม ดังนั้นเสียงใน Discord จึงเป็นช่องทางการสนทนาสดที่เกี่ยวข้อง ไม่ใช่ Plugin การประชุมผ่านเบราว์เซอร์ตัวที่สี่
