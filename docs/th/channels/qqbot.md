QQ Bot เชื่อมต่อกับ OpenClaw ผ่าน QQ Bot API อย่างเป็นทางการ (WebSocket Gateway) Plugin รองรับแชทส่วนตัว C2C, @messages ในกลุ่ม และข้อความในช่องกิลด์พร้อมสื่อสมบูรณ์ (รูปภาพ, เสียง, วิดีโอ, ไฟล์)

สถานะ: Plugin ที่ดาวน์โหลดได้ รองรับข้อความโดยตรง, แชทกลุ่ม, ช่องกิลด์ และสื่อ ไม่รองรับรีแอ็กชันและเธรด

## ติดตั้ง

ติดตั้ง QQ Bot ก่อนตั้งค่า:

```bash
openclaw plugins install @openclaw/qqbot
```

## ตั้งค่า

1. ไปที่ [QQ Open Platform](https://q.qq.com/) แล้วสแกนคิวอาร์โค้ดด้วย QQ บนโทรศัพท์ของคุณเพื่อลงทะเบียน / เข้าสู่ระบบ
2. คลิก **Create Bot** เพื่อสร้าง QQ bot ใหม่
3. ค้นหา **AppID** และ **AppSecret** ในหน้าการตั้งค่าของบอต แล้วคัดลอกไว้

> AppSecret จะไม่ถูกจัดเก็บเป็นข้อความธรรมดา — หากคุณออกจากหน้านี้โดยไม่บันทึก
> คุณจะต้องสร้างรายการใหม่

4. เพิ่มช่องทาง:

```bash
openclaw channels add --channel qqbot --token "AppID:AppSecret"
```

5. รีสตาร์ท Gateway

เส้นทางการตั้งค่าแบบโต้ตอบ:

```bash
openclaw channels add
openclaw configure --section channels
```

## กำหนดค่า

การกำหนดค่าขั้นต่ำ:

```json5
{
  channels: {
    qqbot: {
      enabled: true,
      appId: "YOUR_APP_ID",
      clientSecret: "YOUR_APP_SECRET",
    },
  },
}
```

ตัวแปรสภาพแวดล้อมสำหรับบัญชีเริ่มต้น:

- `QQBOT_APP_ID`
- `QQBOT_CLIENT_SECRET`

AppSecret ที่อ้างอิงจากไฟล์:

```json5
{
  channels: {
    qqbot: {
      enabled: true,
      appId: "YOUR_APP_ID",
      clientSecretFile: "/path/to/qqbot-secret.txt",
    },
  },
}
```

หมายเหตุ:

- การใช้ค่าสำรองจากสภาพแวดล้อมมีผลกับบัญชี QQ Bot เริ่มต้นเท่านั้น
- `openclaw channels add --channel qqbot --token-file ...` ให้เฉพาะ
  AppSecret เท่านั้น; ต้องตั้งค่า AppID ไว้แล้วในการกำหนดค่าหรือ `QQBOT_APP_ID`
- `clientSecret` ยังรับอินพุต SecretRef ได้ ไม่ใช่เฉพาะสตริงข้อความธรรมดา

### การตั้งค่าหลายบัญชี

เรียกใช้ QQ bot หลายตัวภายใต้อินสแตนซ์ OpenClaw เดียว:

```json5
{
  channels: {
    qqbot: {
      enabled: true,
      appId: "111111111",
      clientSecret: "secret-of-bot-1",
      accounts: {
        bot2: {
          enabled: true,
          appId: "222222222",
          clientSecret: "secret-of-bot-2",
        },
      },
    },
  },
}
```

แต่ละบัญชีเริ่มการเชื่อมต่อ WebSocket ของตัวเองและดูแลแคชโทเคนที่เป็นอิสระ
(แยกตาม `appId`)

เพิ่มบอตตัวที่สองผ่าน CLI:

```bash
openclaw channels add --channel qqbot --account bot2 --token "222222222:secret-of-bot-2"
```

### แชทกลุ่ม

การรองรับแชทกลุ่มของ QQ Bot ใช้ QQ group OpenIDs ไม่ใช่ชื่อที่แสดง เพิ่มบอต
เข้าไปในกลุ่ม แล้วกล่าวถึงบอตหรือกำหนดค่ากลุ่มให้ทำงานโดยไม่ต้องกล่าวถึง

```json5
{
  channels: {
    qqbot: {
      groupPolicy: "allowlist",
      groupAllowFrom: ["member_openid"],
      groups: {
        "*": {
          requireMention: true,
          historyLimit: 50,
          toolPolicy: "restricted",
        },
        GROUP_OPENID: {
          name: "Release room",
          requireMention: false,
          ignoreOtherMentions: true,
          historyLimit: 20,
          prompt: "Keep replies short and operational.",
        },
      },
    },
  },
}
```

`groups["*"]` ตั้งค่าเริ่มต้นสำหรับทุกกลุ่ม และรายการ
`groups.GROUP_OPENID` ที่ระบุชัดเจนจะแทนที่ค่าเริ่มต้นเหล่านั้นสำหรับกลุ่มหนึ่ง การตั้งค่ากลุ่มประกอบด้วย:

- `requireMention`: ต้องมี @mention ก่อนที่บอตจะตอบกลับ ค่าเริ่มต้น: `true`
- `ignoreOtherMentions`: ทิ้งข้อความที่กล่าวถึงคนอื่นแต่ไม่ได้กล่าวถึงบอต
- `historyLimit`: เก็บข้อความกลุ่มล่าสุดที่ไม่ใช่การกล่าวถึงไว้เป็นบริบทสำหรับเทิร์นถัดไปที่มีการกล่าวถึง ตั้งค่าเป็น `0` เพื่อปิดใช้งาน
- `toolPolicy`: `full`, `restricted` หรือ `none` สำหรับเครื่องมือที่มีขอบเขตตามกลุ่ม
- `name`: ป้ายชื่อที่อ่านง่ายซึ่งใช้ในบันทึกและบริบทกลุ่ม
- `prompt`: พรอมป์พฤติกรรมเฉพาะกลุ่มที่ต่อท้ายบริบทของเอเจนต์

โหมดการเปิดใช้งานคือ `mention` และ `always` `requireMention: true` จะแมปไปยัง
`mention`; `requireMention: false` จะแมปไปยัง `always` หากมีการแทนที่การเปิดใช้งานระดับเซสชัน ค่านั้นจะมีผลเหนือการกำหนดค่า

คิวขาเข้าแยกตามเพียร์ เพียร์กลุ่มมีขีดจำกัดคิวที่ใหญ่กว่า รักษาข้อความจากมนุษย์
ไว้ก่อนข้อความสนทนาที่บอตเขียนเมื่อคิวเต็ม และรวมข้อความกลุ่มปกติที่เข้ามาเป็นชุด
ให้เป็นเทิร์นเดียวพร้อมการระบุผู้พูด คำสั่งสแลชยังคงทำงานทีละคำสั่ง

### เสียง (STT / TTS)

การรองรับ STT และ TTS ใช้การกำหนดค่าสองระดับพร้อมค่าสำรองตามลำดับความสำคัญ:

| การตั้งค่า | เฉพาะ Plugin                                             | ค่าสำรองของเฟรมเวิร์ก         |
| ------- | -------------------------------------------------------- | ----------------------------- |
| STT     | `channels.qqbot.stt`                                     | `tools.media.audio.models[0]` |
| TTS     | `channels.qqbot.tts`, `channels.qqbot.accounts.<id>.tts` | `messages.tts`                |

```json5
{
  channels: {
    qqbot: {
      stt: {
        provider: "your-provider",
        model: "your-stt-model",
      },
      tts: {
        provider: "your-provider",
        model: "your-tts-model",
        voice: "your-voice",
      },
      accounts: {
        qq-main: {
          tts: {
            providers: {
              openai: { voice: "shimmer" },
            },
          },
        },
      },
    },
  },
}
```

ตั้งค่า `enabled: false` กับรายการใดรายการหนึ่งเพื่อปิดใช้งาน
การแทนที่ TTS ระดับบัญชีใช้โครงสร้างเดียวกับ `messages.tts` และทำ deep-merge
ทับการกำหนดค่า TTS ระดับช่องทาง/ส่วนกลาง

ไฟล์แนบเสียงขาเข้าของ QQ จะแสดงต่อเอเจนต์เป็นเมตาดาต้าสื่อเสียง พร้อมกับ
กันไฟล์เสียงดิบออกจาก `MediaPaths` ทั่วไป การตอบกลับเป็นข้อความธรรมดา
`[[audio_as_voice]]` จะสังเคราะห์ TTS และส่งข้อความเสียง QQ แบบเนทีฟเมื่อมีการ
กำหนดค่า TTS

พฤติกรรมการอัปโหลด/ทรานส์โค้ดเสียงขาออกยังสามารถปรับได้ด้วย
`channels.qqbot.audioFormatPolicy`:

- `sttDirectFormats`
- `uploadDirectFormats`
- `transcodeEnabled`

## รูปแบบเป้าหมาย

| รูปแบบ                     | คำอธิบาย        |
| -------------------------- | ------------------ |
| `qqbot:c2c:OPENID`         | แชทส่วนตัว (C2C) |
| `qqbot:group:GROUP_OPENID` | แชทกลุ่ม         |
| `qqbot:channel:CHANNEL_ID` | ช่องกิลด์      |

> แต่ละบอตมีชุด user OpenIDs ของตัวเอง OpenID ที่ได้รับจาก Bot A **ไม่สามารถ**
> ใช้ส่งข้อความผ่าน Bot B ได้

## คำสั่งสแลช

คำสั่งในตัวที่ถูกดักก่อนคิว AI:

| คำสั่ง        | คำอธิบาย                                                                                              |
| -------------- | -------------------------------------------------------------------------------------------------------- |
| `/bot-ping`    | ทดสอบเวลาแฝง                                                                                             |
| `/bot-version` | แสดงเวอร์ชันของเฟรมเวิร์ก OpenClaw                                                                      |
| `/bot-help`    | แสดงรายการคำสั่งทั้งหมด                                                                                        |
| `/bot-me`      | แสดง QQ user ID (openid) ของผู้ส่งสำหรับการตั้งค่า `allowFrom`/`groupAllowFrom`                             |
| `/bot-upgrade` | แสดงลิงก์คู่มือการอัปเกรด QQBot                                                                        |
| `/bot-logs`    | ส่งออกบันทึก Gateway ล่าสุดเป็นไฟล์                                                                     |
| `/bot-approve` | อนุมัติการดำเนินการ QQ Bot ที่รอดำเนินการ (เช่น ยืนยันการอัปโหลด C2C หรือกลุ่ม) ผ่านโฟลว์เนทีฟ |

ต่อท้าย `?` กับคำสั่งใดก็ได้เพื่อดูวิธีใช้งาน (เช่น `/bot-upgrade ?`)

คำสั่งผู้ดูแลระบบ (`/bot-me`, `/bot-upgrade`, `/bot-logs`, `/bot-clear-storage`, `/bot-streaming`, `/bot-approve`) ใช้ได้เฉพาะข้อความโดยตรงและต้องมี openid ของผู้ส่งอยู่ในรายการ `allowFrom` แบบชัดเจนที่ไม่ใช่ไวลด์การ์ด ไวลด์การ์ด `allowFrom: ["*"]` อนุญาตให้แชทได้แต่ไม่ให้สิทธิ์เข้าถึงคำสั่งผู้ดูแลระบบ ข้อความกลุ่มจะเทียบกับ `groupAllowFrom` ก่อน แล้วจึงย้อนกลับไปใช้ `allowFrom` การเรียกใช้คำสั่งผู้ดูแลระบบในกลุ่มจะส่งคำใบ้กลับ แทนที่จะทิ้งข้อความโดยไม่แจ้งอะไร

## สถาปัตยกรรมเอนจิน

QQ Bot มาพร้อมเอนจินที่ครบถ้วนในตัวภายใน Plugin:

- แต่ละบัญชีเป็นเจ้าของสแต็กทรัพยากรที่แยกจากกัน (การเชื่อมต่อ WebSocket, ไคลเอนต์ API, แคชโทเคน, รูทที่เก็บสื่อ) ซึ่งมีคีย์ตาม `appId` บัญชีจะไม่แชร์สถานะขาเข้า/ขาออกกัน
- ตัวบันทึกหลายบัญชีติดแท็กบรรทัดบันทึกด้วยบัญชีเจ้าของ เพื่อให้การวินิจฉัยแยกจากกันได้เมื่อคุณเรียกใช้หลายบอตภายใต้ Gateway เดียว
- เส้นทางขาเข้า, ขาออก และบริดจ์ Gateway ใช้รูทเพย์โหลดสื่อเดียวกันภายใต้ `~/.openclaw/media` ดังนั้นการอัปโหลด, การดาวน์โหลด และแคชทรานส์โค้ดจะอยู่ภายใต้ไดเรกทอรีที่มีการป้องกันเดียว แทนที่จะเป็นต้นไม้แยกตามระบบย่อย
- การส่งสื่อสมบูรณ์ผ่านเส้นทาง `sendMedia` เดียวสำหรับเป้าหมาย C2C และกลุ่ม ไฟล์ภายในเครื่องและบัฟเฟอร์ที่เกินเกณฑ์ไฟล์ขนาดใหญ่จะใช้ปลายทางการอัปโหลดแบบแบ่งชิ้นของ QQ ส่วนเพย์โหลดที่เล็กกว่าจะใช้ media API แบบครั้งเดียว
- สามารถสำรองและกู้คืนข้อมูลประจำตัวเป็นส่วนหนึ่งของสแนปชอตข้อมูลประจำตัวมาตรฐานของ OpenClaw ได้ เอนจินจะเชื่อมสแต็กทรัพยากรของแต่ละบัญชีอีกครั้งเมื่อกู้คืน โดยไม่ต้องจับคู่คิวอาร์โค้ดใหม่

## การเริ่มต้นใช้งานด้วยคิวอาร์โค้ด

นอกจากการวาง `AppID:AppSecret` ด้วยตนเอง เอนจินยังรองรับโฟลว์การเริ่มต้นใช้งานด้วยคิวอาร์โค้ดสำหรับลิงก์ QQ Bot เข้ากับ OpenClaw:

1. เรียกใช้เส้นทางการตั้งค่า QQ Bot (เช่น `openclaw channels add --channel qqbot`) แล้วเลือกโฟลว์คิวอาร์โค้ดเมื่อมีพรอมป์
2. สแกนคิวอาร์โค้ดที่สร้างขึ้นด้วยแอปบนโทรศัพท์ที่ผูกกับ QQ Bot เป้าหมาย
3. อนุมัติการจับคู่บนโทรศัพท์ OpenClaw จะบันทึกข้อมูลประจำตัวที่ส่งกลับลงใน `credentials/` ภายใต้ขอบเขตบัญชีที่ถูกต้อง

พรอมป์การอนุมัติที่บอตสร้างขึ้นเอง (เช่น โฟลว์ "อนุญาตการดำเนินการนี้หรือไม่?" ที่ QQ Bot API เปิดเผย) จะแสดงเป็นพรอมป์ OpenClaw แบบเนทีฟที่คุณยอมรับได้ด้วย `/bot-approve` แทนการตอบผ่านไคลเอนต์ QQ ดิบ

## การแก้ไขปัญหา

- **บอตตอบว่า "gone to Mars":** ยังไม่ได้กำหนดค่าข้อมูลประจำตัวหรือยังไม่ได้เริ่ม Gateway
- **ไม่มีข้อความขาเข้า:** ตรวจสอบว่า `appId` และ `clientSecret` ถูกต้อง และ
  เปิดใช้งานบอตบน QQ Open Platform แล้ว
- **ตอบกลับตัวเองซ้ำ:** OpenClaw บันทึกดัชนีอ้างอิงขาออกของ QQ เป็น
  ข้อความที่บอตเขียน และเพิกเฉยต่อเหตุการณ์ขาเข้าที่ `msgIdx` ปัจจุบันตรงกับ
  บัญชีบอตเดียวกันนั้น วิธีนี้ป้องกันลูปเสียงสะท้อนของแพลตฟอร์ม ขณะที่ยังอนุญาตให้ผู้ใช้
  อ้างอิงหรือตอบกลับข้อความบอตก่อนหน้าได้
- **ตั้งค่าด้วย `--token-file` แล้วยังแสดงว่ายังไม่ได้กำหนดค่า:** `--token-file` ตั้งค่าเฉพาะ
  AppSecret เท่านั้น คุณยังต้องมี `appId` ในการกำหนดค่าหรือ `QQBOT_APP_ID`
- **ข้อความเชิงรุกไม่มาถึง:** QQ อาจดักข้อความที่บอตเริ่มส่งเองหาก
  ผู้ใช้ไม่ได้โต้ตอบเมื่อเร็ว ๆ นี้
- **เสียงไม่ได้รับการถอดความ:** ตรวจสอบว่าได้กำหนดค่า STT แล้วและผู้ให้บริการเข้าถึงได้
