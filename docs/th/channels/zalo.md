## Plugin ที่มาพร้อมแพ็กเกจ

Zalo จัดส่งเป็น Plugin ที่มาพร้อมแพ็กเกจใน OpenClaw รุ่นปัจจุบัน ดังนั้นบิลด์แบบแพ็กเกจปกติ
ไม่จำเป็นต้องติดตั้งแยกต่างหาก

หากคุณใช้บิลด์เก่ากว่าหรือการติดตั้งแบบกำหนดเองที่ไม่รวม Zalo ให้ติดตั้ง
แพ็กเกจ npm ปัจจุบันเมื่อมีการเผยแพร่:

- ติดตั้งผ่าน CLI: `openclaw plugins install @openclaw/zalo`
- หรือจากเช็กเอาต์ซอร์ส: `openclaw plugins install ./path/to/local/zalo-plugin`
- รายละเอียด: [Plugins](/th/tools/plugin)

หาก npm รายงานว่าแพ็กเกจที่ OpenClaw เป็นเจ้าของถูกทำเครื่องหมายว่าเลิกใช้แล้ว ให้ใช้บิลด์ OpenClaw
แบบแพ็กเกจปัจจุบันหรือพาธเช็กเอาต์ภายในเครื่องจนกว่าจะมีการเผยแพร่แพ็กเกจ npm ที่ใหม่กว่า

## การตั้งค่าอย่างรวดเร็ว (ผู้เริ่มต้น)

1. ตรวจสอบว่า Plugin Zalo พร้อมใช้งาน
   - OpenClaw รุ่นแบบแพ็กเกจปัจจุบันรวมไว้แล้ว
   - การติดตั้งเก่ากว่า/แบบกำหนดเองสามารถเพิ่มด้วยตนเองโดยใช้คำสั่งด้านบน
2. ตั้งค่าโทเค็น:
   - Env: `ZALO_BOT_TOKEN=...`
   - หรือการกำหนดค่า: `channels.zalo.accounts.default.botToken: "..."`.
3. รีสตาร์ต Gateway (หรือดำเนินการตั้งค่าให้เสร็จ)
4. การเข้าถึง DM เป็นการจับคู่โดยค่าเริ่มต้น อนุมัติรหัสจับคู่เมื่อมีการติดต่อครั้งแรก

การกำหนดค่าขั้นต่ำ:

```json5
{
  channels: {
    zalo: {
      enabled: true,
      accounts: {
        default: {
          botToken: "12345689:abc-xyz",
          dmPolicy: "pairing",
        },
      },
    },
  },
}
```

## คืออะไร

Zalo เป็นแอปส่งข้อความที่เน้นผู้ใช้ในเวียดนาม; Bot API ของแอปช่วยให้ Gateway เรียกใช้บอตสำหรับการสนทนาแบบ 1:1 ได้
เหมาะสำหรับงานสนับสนุนหรือการแจ้งเตือนที่คุณต้องการการกำหนดเส้นทางกลับไปยัง Zalo อย่างแน่นอน

หน้านี้สะท้อนพฤติกรรม OpenClaw ปัจจุบันสำหรับ **บอต Zalo Bot Creator / Marketplace**
**บอต Zalo Official Account (OA)** เป็นพื้นผิวผลิตภัณฑ์ Zalo อีกแบบหนึ่ง และอาจมีพฤติกรรมแตกต่างกัน

- ช่อง Zalo Bot API ที่ Gateway เป็นเจ้าของ
- การกำหนดเส้นทางแบบแน่นอน: การตอบกลับจะกลับไปยัง Zalo; โมเดลไม่เลือกช่องเอง
- DM ใช้เซสชันหลักของเอเจนต์ร่วมกัน
- ส่วน [ความสามารถ](#capabilities) ด้านล่างแสดงการรองรับบอต Marketplace ปัจจุบัน

## การตั้งค่า (เส้นทางเร็ว)

### 1) สร้างโทเค็นบอต (Zalo Bot Platform)

1. ไปที่ [https://bot.zaloplatforms.com](https://bot.zaloplatforms.com) แล้วลงชื่อเข้าใช้
2. สร้างบอตใหม่และกำหนดค่าการตั้งค่า
3. คัดลอกโทเค็นบอตแบบเต็ม (โดยทั่วไปคือ `numeric_id:secret`) สำหรับบอต Marketplace โทเค็นรันไทม์ที่ใช้งานได้อาจปรากฏในข้อความต้อนรับของบอตหลังจากสร้างแล้ว

### 2) กำหนดค่าโทเค็น (env หรือ config)

ตัวอย่าง:

```json5
{
  channels: {
    zalo: {
      enabled: true,
      accounts: {
        default: {
          botToken: "12345689:abc-xyz",
          dmPolicy: "pairing",
        },
      },
    },
  },
}
```

หากภายหลังคุณย้ายไปยังพื้นผิวบอต Zalo ที่มีกลุ่มให้ใช้งาน คุณสามารถเพิ่มการกำหนดค่าเฉพาะกลุ่ม เช่น `groupPolicy` และ `groupAllowFrom` อย่างชัดเจนได้ สำหรับพฤติกรรมบอต Marketplace ปัจจุบัน โปรดดู [ความสามารถ](#capabilities)

ตัวเลือก Env: `ZALO_BOT_TOKEN=...` (ใช้ได้กับบัญชีเริ่มต้นเท่านั้น)

การรองรับหลายบัญชี: ใช้ `channels.zalo.accounts` พร้อมโทเค็นต่อบัญชีและ `name` ที่เป็นทางเลือก

3. รีสตาร์ต Gateway Zalo จะเริ่มเมื่อระบุโทเค็นได้แล้ว (env หรือ config)
4. การเข้าถึง DM ใช้การจับคู่เป็นค่าเริ่มต้น อนุมัติรหัสเมื่อมีการติดต่อบอตครั้งแรก

## วิธีทำงาน (พฤติกรรม)

- ข้อความขาเข้าจะถูกทำให้เป็นมาตรฐานเป็นซองช่องที่ใช้ร่วมกันพร้อมตัวยึดสื่อ
- การตอบกลับจะกำหนดเส้นทางกลับไปยังแชต Zalo เดิมเสมอ
- ใช้ long-polling เป็นค่าเริ่มต้น; โหมด Webhook ใช้ได้ด้วย `channels.zalo.webhookUrl`

## ข้อจำกัด

- ข้อความขาออกถูกแบ่งเป็นส่วนละ 2000 อักขระ (ข้อจำกัดของ Zalo API)
- การดาวน์โหลด/อัปโหลดสื่อถูกจำกัดด้วย `channels.zalo.mediaMaxMb` (ค่าเริ่มต้น 5)
- การสตรีมถูกบล็อกโดยค่าเริ่มต้น เนื่องจากข้อจำกัด 2000 อักขระทำให้การสตรีมมีประโยชน์น้อยลง

## การควบคุมการเข้าถึง (DM)

### การเข้าถึง DM

- ค่าเริ่มต้น: `channels.zalo.dmPolicy = "pairing"` ผู้ส่งที่ไม่รู้จักจะได้รับรหัสจับคู่; ข้อความจะถูกละเว้นจนกว่าจะได้รับอนุมัติ (รหัสหมดอายุหลัง 1 ชั่วโมง)
- อนุมัติผ่าน:
  - `openclaw pairing list zalo`
  - `openclaw pairing approve zalo <CODE>`
- การจับคู่เป็นการแลกเปลี่ยนโทเค็นเริ่มต้น รายละเอียด: [การจับคู่](/th/channels/pairing)
- `channels.zalo.allowFrom` รับ ID ผู้ใช้แบบตัวเลข (ไม่มีการค้นหาชื่อผู้ใช้ให้ใช้งาน)

## การควบคุมการเข้าถึง (กลุ่ม)

สำหรับ **บอต Zalo Bot Creator / Marketplace** การรองรับกลุ่มไม่พร้อมใช้งานในทางปฏิบัติ เพราะไม่สามารถเพิ่มบอตเข้ากลุ่มได้เลย

หมายความว่าคีย์การกำหนดค่าที่เกี่ยวข้องกับกลุ่มด้านล่างมีอยู่ในสคีมา แต่ไม่สามารถใช้กับบอต Marketplace ได้:

- `channels.zalo.groupPolicy` ควบคุมการจัดการขาเข้าของกลุ่ม: `open | allowlist | disabled`
- `channels.zalo.groupAllowFrom` จำกัดว่า ID ผู้ส่งใดสามารถทริกเกอร์บอตในกลุ่มได้
- หากไม่ได้ตั้งค่า `groupAllowFrom` Zalo จะย้อนกลับไปใช้ `allowFrom` สำหรับการตรวจสอบผู้ส่ง
- หมายเหตุรันไทม์: หากไม่มี `channels.zalo` ทั้งหมด รันไทม์ยังคงย้อนกลับไปใช้ `groupPolicy="allowlist"` เพื่อความปลอดภัย

ค่าของนโยบายกลุ่ม (เมื่อการเข้าถึงกลุ่มพร้อมใช้งานบนพื้นผิวบอตของคุณ) คือ:

- `groupPolicy: "disabled"` — บล็อกข้อความกลุ่มทั้งหมด
- `groupPolicy: "open"` — อนุญาตสมาชิกกลุ่มใดก็ได้ (กำกับด้วยการกล่าวถึง)
- `groupPolicy: "allowlist"` — ค่าเริ่มต้นแบบปิดเมื่อไม่ผ่าน; ยอมรับเฉพาะผู้ส่งที่ได้รับอนุญาต

หากคุณใช้พื้นผิวผลิตภัณฑ์บอต Zalo แบบอื่นและได้ตรวจสอบแล้วว่าพฤติกรรมกลุ่มทำงานได้ ให้จัดทำเอกสารแยกต่างหากแทนการสันนิษฐานว่าตรงกับโฟลว์ของบอต Marketplace

## Long-polling เทียบกับ Webhook

- ค่าเริ่มต้น: long-polling (ไม่ต้องใช้ URL สาธารณะ)
- โหมด Webhook: ตั้งค่า `channels.zalo.webhookUrl` และ `channels.zalo.webhookSecret`
  - ความลับ Webhook ต้องมี 8-256 อักขระ
  - URL ของ Webhook ต้องใช้ HTTPS
  - Zalo ส่งเหตุการณ์พร้อมส่วนหัว `X-Bot-Api-Secret-Token` สำหรับการตรวจสอบ
  - HTTP ของ Gateway จัดการคำขอ Webhook ที่ `channels.zalo.webhookPath` (ค่าเริ่มต้นเป็นพาธของ URL Webhook)
  - คำขอต้องใช้ `Content-Type: application/json` (หรือชนิดสื่อ `+json`)
  - เหตุการณ์ซ้ำ (`event_name + message_id`) จะถูกละเว้นในหน้าต่างเล่นซ้ำสั้น ๆ
  - ทราฟฟิกแบบพุ่งสูงจะถูกจำกัดอัตราต่อพาธ/แหล่งที่มา และอาจคืนค่า HTTP 429

**หมายเหตุ:** getUpdates (polling) และ Webhook ไม่สามารถใช้ร่วมกันได้ต่อเอกสาร Zalo API

## ประเภทข้อความที่รองรับ

สำหรับภาพรวมการรองรับแบบรวดเร็ว โปรดดู [ความสามารถ](#capabilities) หมายเหตุด้านล่างเพิ่มรายละเอียดในจุดที่พฤติกรรมต้องการบริบทเพิ่มเติม

- **ข้อความตัวอักษร**: รองรับเต็มรูปแบบพร้อมการแบ่งส่วนที่ 2000 อักขระ
- **URL ธรรมดาในข้อความ**: ทำงานเหมือนอินพุตข้อความปกติ
- **ตัวอย่างลิงก์ / การ์ดลิงก์แบบ rich**: ดูสถานะบอต Marketplace ใน [ความสามารถ](#capabilities); ไม่ได้ทริกเกอร์การตอบกลับอย่างเชื่อถือได้
- **ข้อความรูปภาพ**: ดูสถานะบอต Marketplace ใน [ความสามารถ](#capabilities); การจัดการรูปภาพขาเข้าไม่น่าเชื่อถือ (แสดงตัวบ่งชี้ว่ากำลังพิมพ์โดยไม่มีการตอบกลับสุดท้าย)
- **สติกเกอร์**: ดูสถานะบอต Marketplace ใน [ความสามารถ](#capabilities)
- **บันทึกเสียง / ไฟล์เสียง / วิดีโอ / ไฟล์แนบทั่วไป**: ดูสถานะบอต Marketplace ใน [ความสามารถ](#capabilities)
- **ชนิดที่ไม่รองรับ**: ถูกบันทึกในล็อก (เช่น ข้อความจากผู้ใช้ที่ได้รับการป้องกัน)

## เป้าหมายการส่งมอบ (CLI/Cron)

- ใช้ ID แชตเป็นเป้าหมาย
- ตัวอย่าง: `openclaw message send --channel zalo --target 123456789 --message "hi"`

## การแก้ไขปัญหา

**บอตไม่ตอบสนอง:**

- ตรวจสอบว่าโทเค็นถูกต้อง: `openclaw channels status --probe`
- ตรวจสอบว่าผู้ส่งได้รับอนุมัติแล้ว (pairing หรือ allowFrom)
- ตรวจสอบล็อก Gateway: `openclaw logs --follow`

**Webhook ไม่ได้รับเหตุการณ์:**

- ตรวจสอบให้แน่ใจว่า URL ของ Webhook ใช้ HTTPS
- ตรวจสอบว่าโทเค็นความลับมี 8-256 อักขระ
- ยืนยันว่าเอ็นด์พอยต์ HTTP ของ Gateway เข้าถึงได้บนพาธที่กำหนดค่าไว้
- ตรวจสอบว่า getUpdates polling ไม่ได้ทำงานอยู่ (ทั้งสองอย่างใช้ร่วมกันไม่ได้)

## อ้างอิงการกำหนดค่า (Zalo)

การกำหนดค่าแบบเต็ม: [การกำหนดค่า](/th/gateway/configuration)

คีย์ระดับบนแบบแบน (`channels.zalo.botToken`, `channels.zalo.dmPolicy` และคีย์ที่คล้ายกัน) เป็นชวเลขบัญชีเดียวแบบเดิม แนะนำให้ใช้ `channels.zalo.accounts.<id>.*` สำหรับการกำหนดค่าใหม่ ทั้งสองรูปแบบยังคงมีเอกสารที่นี่เพราะมีอยู่ในสคีมา

ตัวเลือกผู้ให้บริการ:

- `channels.zalo.enabled`: เปิด/ปิดการเริ่มต้นช่อง
- `channels.zalo.botToken`: โทเค็นบอตจาก Zalo Bot Platform
- `channels.zalo.tokenFile`: อ่านโทเค็นจากพาธไฟล์ปกติ Symlink จะถูกปฏิเสธ
- `channels.zalo.dmPolicy`: `pairing | allowlist | open | disabled` (ค่าเริ่มต้น: pairing)
- `channels.zalo.allowFrom`: allowlist ของ DM (ID ผู้ใช้) `open` ต้องใช้ `"*"` วิซาร์ดจะถามหา ID แบบตัวเลข
- `channels.zalo.groupPolicy`: `open | allowlist | disabled` (ค่าเริ่มต้น: allowlist) มีอยู่ในการกำหนดค่า; ดู [ความสามารถ](#capabilities) และ [การควบคุมการเข้าถึง (กลุ่ม)](#access-control-groups) สำหรับพฤติกรรมบอต Marketplace ปัจจุบัน
- `channels.zalo.groupAllowFrom`: allowlist ผู้ส่งของกลุ่ม (ID ผู้ใช้) ย้อนกลับไปใช้ `allowFrom` เมื่อไม่ได้ตั้งค่า
- `channels.zalo.mediaMaxMb`: ขีดจำกัดสื่อขาเข้า/ขาออก (MB, ค่าเริ่มต้น 5)
- `channels.zalo.webhookUrl`: เปิดใช้งานโหมด Webhook (ต้องใช้ HTTPS)
- `channels.zalo.webhookSecret`: ความลับ Webhook (8-256 อักขระ)
- `channels.zalo.webhookPath`: พาธ Webhook บนเซิร์ฟเวอร์ HTTP ของ Gateway
- `channels.zalo.proxy`: URL พร็อกซีสำหรับคำขอ API

ตัวเลือกหลายบัญชี:

- `channels.zalo.accounts.<id>.botToken`: โทเค็นต่อบัญชี
- `channels.zalo.accounts.<id>.tokenFile`: ไฟล์โทเค็นปกติต่อบัญชี Symlink จะถูกปฏิเสธ
- `channels.zalo.accounts.<id>.name`: ชื่อที่แสดง
- `channels.zalo.accounts.<id>.enabled`: เปิด/ปิดบัญชี
- `channels.zalo.accounts.<id>.dmPolicy`: นโยบาย DM ต่อบัญชี
- `channels.zalo.accounts.<id>.allowFrom`: allowlist ต่อบัญชี
- `channels.zalo.accounts.<id>.groupPolicy`: นโยบายกลุ่มต่อบัญชี มีอยู่ในการกำหนดค่า; ดู [ความสามารถ](#capabilities) และ [การควบคุมการเข้าถึง (กลุ่ม)](#access-control-groups) สำหรับพฤติกรรมบอต Marketplace ปัจจุบัน
- `channels.zalo.accounts.<id>.groupAllowFrom`: allowlist ผู้ส่งกลุ่มต่อบัญชี
- `channels.zalo.accounts.<id>.webhookUrl`: URL Webhook ต่อบัญชี
- `channels.zalo.accounts.<id>.webhookSecret`: ความลับ Webhook ต่อบัญชี
- `channels.zalo.accounts.<id>.webhookPath`: พาธ Webhook ต่อบัญชี
- `channels.zalo.accounts.<id>.proxy`: URL พร็อกซีต่อบัญชี
