OpenClaw เชื่อมต่อกับ Zalo ClawBot ผ่าน Plugin ภายนอก `@zalo-platforms/openclaw-zaloclawbot` ที่แสดงอยู่ในแค็ตตาล็อก การเข้าสู่ระบบใช้คิวอาร์โค้ดของ Zalo Mini App โดยรหัส Plugin ในการกำหนดค่าคือ `openclaw-zaloclawbot`

## ความเข้ากันได้

| เวอร์ชัน Plugin | เวอร์ชัน OpenClaw | dist-tag ของ npm | สถานะ              |
| --------------- | ----------------- | ---------------- | ------------------ |
| 0.1.4           | >=2026.4.10       | `latest`         | ใช้งานอยู่ / เบต้า |

## ข้อกำหนดเบื้องต้น

- Node.js >= 22
- ติดตั้ง [OpenClaw](https://docs.openclaw.ai/install) แล้ว (พร้อมใช้งาน CLI `openclaw`)
- บัญชี Zalo บนอุปกรณ์เคลื่อนที่สำหรับสแกนคิวอาร์โค้ดเข้าสู่ระบบ

## ติดตั้งด้วยการเริ่มต้นใช้งาน (แนะนำ)

```bash
openclaw onboard
```

เลือก **Zalo ClawBot** จากเมนูช่องทาง ตัวช่วยจะติดตั้ง Plugin จากแค็ตตาล็อกอย่างเป็นทางการ (พร้อมตรวจสอบความสมบูรณ์) แสดงคิวอาร์โค้ดเข้าสู่ระบบในเทอร์มินัล และตั้งค่าช่องทางให้เสร็จสิ้นเมื่อคุณสแกนด้วยแอป Zalo

## การติดตั้งด้วยตนเอง

หากต้องการเพิ่มช่องทางลงใน Gateway ที่ผ่านการเริ่มต้นใช้งานแล้ว:

### 1. ติดตั้ง Plugin

```bash
openclaw plugins install "@zalo-platforms/openclaw-zaloclawbot@0.1.4"
```

ใช้เวอร์ชันที่ตรึงไว้นี้อย่างถูกต้อง เพื่อให้ OpenClaw ตรวจสอบแพ็กเกจเทียบกับแฮชความสมบูรณ์ในแค็ตตาล็อกระหว่างการติดตั้ง

### 2. เปิดใช้งาน Plugin ในการกำหนดค่า

```bash
openclaw config set plugins.entries.openclaw-zaloclawbot.enabled true
```

### 3. สร้างคิวอาร์โค้ดและเข้าสู่ระบบ

```bash
openclaw channels login --channel openclaw-zaloclawbot
```

สแกนคิวอาร์โค้ดที่แสดงในเทอร์มินัลด้วยแอป Zalo บนอุปกรณ์เคลื่อนที่ ยอมรับข้อกำหนดการใช้งานภายใน Zalo Mini App และอนุญาตเซสชัน

### 4. เริ่ม Gateway ใหม่

```bash
openclaw gateway restart
```

## หลักการทำงาน

ต่างจากช่องทาง Zalo มาตรฐานซึ่งกำหนดให้คุณลงทะเบียนบัญชีทางการของ Zalo (OA) ของตนเองและกำหนดข้อมูลประจำตัวแบบคงที่สำหรับนักพัฒนา Zalo ClawBot เป็น **ผู้ช่วยส่วนตัวที่ผูกกับเจ้าของ** บนโครงสร้างพื้นฐานอย่างเป็นทางการที่ใช้ร่วมกัน:

1. **การเริ่มต้นใช้งาน:** คิวอาร์โค้ดจะนำไปยัง Zalo Mini App ซึ่งผูกบอตส่วนตัวที่จัดเตรียมขึ้นใหม่ภายใต้ OA อย่างเป็นทางการที่ใช้ร่วมกัน เข้ากับรหัสผู้ใช้ Zalo ของคุณโดยตรง
2. **ความเป็นส่วนตัวที่ผูกกับเจ้าของ:** บอตจะสื่อสารกับเจ้าของเท่านั้น ข้อความจากผู้ใช้รายอื่นจะถูกละทิ้งในระดับแพลตฟอร์ม
3. **เส้นทาง API อย่างเป็นทางการ:** Plugin ใช้ API ของ Zalo Bot Platform ไม่ใช่ระบบอัตโนมัติผ่านเบราว์เซอร์หรือเซสชันเว็บ

## การทำงานภายใน

Plugin สื่อสารกับ Zalo ผ่านลูปการโพลระยะยาวแบบต่อเนื่อง (`getUpdates`) โดยค่าเริ่มต้น Webhook จะถูกปิดใช้งานสำหรับการเรียกใช้ Gateway บนเดสก์ท็อปหรือเทอร์มินัลภายในเครื่อง ข้อความจะถูกประมวลผลฝั่งไคลเอนต์และเชื่อมโยงกับรันไทม์เอเจนต์ภายในเครื่องของคุณ

Plugin จัดการข้อมูลประจำตัวของบอตภายใต้ไดเรกทอรีสถานะของ OpenClaw ให้ถือว่าไดเรกทอรีดังกล่าวเป็นข้อมูลละเอียดอ่อน และกำหนดให้อยู่ภายใต้นโยบายการควบคุมการเข้าถึงและการสำรองข้อมูลเดียวกับสถานะส่วนอื่นของ OpenClaw

รันไทม์ของ Plugin นี้อยู่ภายในแพ็กเกจภายนอก `@zalo-platforms/openclaw-zaloclawbot` ทั้งหมด รายละเอียดพฤติกรรมด้านล่างที่นอกเหนือจากการติดตั้งและการกำหนดค่าเป็นข้อมูลตามที่ผู้ดูแล Plugin รายงาน และยังไม่ได้รับการตรวจสอบเทียบกับซอร์สหลักของ OpenClaw

## การแก้ไขปัญหา

- **หมดเวลาเข้าสู่ระบบด้วยคิวอาร์โค้ด:** โทเค็นเข้าสู่ระบบ (`zbsk`) จะหมดอายุหลังจาก 5 นาทีเพื่อความปลอดภัย หากคิวอาร์โค้ดหมดอายุก่อนที่คุณจะสแกน ให้เรียกใช้คำสั่งเข้าสู่ระบบอีกครั้งเพื่อสร้างคิวอาร์โค้ดใหม่
- **Gateway โหลดไม่สำเร็จ:** ตรวจสอบว่าเวอร์ชันโฮสต์ OpenClaw ของคุณคือ `2026.4.10` หรือสูงกว่า เวอร์ชันที่เก่ากว่าไม่รองรับบัญชีรายการการติดตั้ง Plugin npm ภายนอกที่รหัสนี้จำเป็นต้องใช้
