ใช้หน้านี้เมื่อช่องทางเชื่อมต่อได้ แต่พฤติกรรมผิดปกติ

## ลำดับคำสั่ง

รันคำสั่งเหล่านี้ตามลำดับก่อน:

```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
openclaw channels status --probe
```

เกณฑ์พื้นฐานที่ปกติ:

- `Runtime: running`
- `Connectivity probe: ok`
- `Capability: read-only`, `write-capable`, หรือ `admin-capable`
- การตรวจสอบช่องทางแสดงว่าทรานสปอร์ตเชื่อมต่อแล้ว และในกรณีที่รองรับ แสดง `works` หรือ `audit ok`

## WhatsApp

### สัญญาณความล้มเหลวของ WhatsApp

| อาการ | การตรวจสอบที่เร็วที่สุด | วิธีแก้ |
| ----------------------------------- | --------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| เชื่อมต่อแล้วแต่ไม่มีการตอบกลับ DM | `openclaw pairing list whatsapp` | อนุมัติผู้ส่งหรือเปลี่ยนนโยบาย/allowlist ของ DM |
| ข้อความกลุ่มถูกเพิกเฉย | ตรวจสอบ `requireMention` + รูปแบบการกล่าวถึงใน config | กล่าวถึงบอตหรือผ่อนปรนนโยบายการกล่าวถึงสำหรับกลุ่มนั้น |
| การเข้าสู่ระบบด้วย QR หมดเวลาด้วย 408 | ตรวจสอบ env `HTTPS_PROXY` / `HTTP_PROXY` ของ Gateway | ตั้งค่าพร็อกซีที่เข้าถึงได้ ใช้ `NO_PROXY` เฉพาะสำหรับการข้ามเท่านั้น |
| หลุดการเชื่อมต่อ/วนเข้าสู่ระบบใหม่แบบสุ่ม | `openclaw channels status --probe` + logs | การเชื่อมต่อใหม่ล่าสุดจะถูกตั้งค่าสถานะแม้ขณะนี้เชื่อมต่ออยู่ ให้ดู logs, รีสตาร์ท Gateway แล้วลิงก์ใหม่หากยังแกว่งต่อเนื่อง |
| การตอบกลับมาถึงช้าเป็นวินาที/นาที | `openclaw doctor --fix` | Doctor หยุดไคลเอนต์ TUI ภายในเครื่องที่ตรวจยืนยันแล้วว่าค้างเก่า เมื่อไคลเอนต์เหล่านั้นทำให้ event loop ของ Gateway เสื่อมประสิทธิภาพ |

การแก้ไขปัญหาแบบเต็ม: [การแก้ไขปัญหา WhatsApp](/th/channels/whatsapp#troubleshooting)

## Telegram

### สัญญาณความล้มเหลวของ Telegram

| อาการ | การตรวจสอบที่เร็วที่สุด | วิธีแก้ |
| ------------------------------------ | ------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------- |
| `/start` แต่ไม่มีโฟลว์ตอบกลับที่ใช้งานได้ | `openclaw pairing list telegram` | อนุมัติการจับคู่หรือเปลี่ยนนโยบาย DM |
| บอตออนไลน์แต่กลุ่มยังเงียบ | ตรวจยืนยันข้อกำหนดการกล่าวถึงและโหมดความเป็นส่วนตัวของบอต | ปิดโหมดความเป็นส่วนตัวเพื่อให้มองเห็นกลุ่ม หรือกล่าวถึงบอต |
| ส่งล้มเหลวพร้อมข้อผิดพลาดเครือข่าย | ตรวจสอบ logs เพื่อหาความล้มเหลวของการเรียก Telegram API | แก้การกำหนดเส้นทาง DNS/IPv6/พร็อกซีไปยัง `api.telegram.org` |
| ตอนเริ่มต้นรายงาน `getMe returned 401` | ตรวจสอบแหล่งที่มาของโทเค็นที่ตั้งค่าไว้ | คัดลอกใหม่หรือสร้างโทเค็น BotFather ใหม่ แล้วอัปเดต `botToken`, `tokenFile`, หรือ `TELEGRAM_BOT_TOKEN` ของบัญชีเริ่มต้น |
| การ polling ค้างหรือเชื่อมต่อใหม่ช้า | `openclaw logs --follow` เพื่อดูการวินิจฉัย polling | อัปเกรด หากการรีสตาร์ทเป็นผลบวกเท็จ ให้ปรับ `pollingStallThresholdMs` การค้างต่อเนื่องยังคงชี้ไปที่พร็อกซี/DNS/IPv6 |
| `setMyCommands` ถูกปฏิเสธตอนเริ่มต้น | ตรวจสอบ logs เพื่อหา `BOT_COMMANDS_TOO_MUCH` | ลดคำสั่ง Telegram ของ Plugin/Skills/กำหนดเอง หรือปิดเมนูเนทีฟ |
| อัปเกรดแล้ว allowlist บล็อกคุณ | `openclaw security audit` และ allowlist ใน config | รัน `openclaw doctor --fix` หรือแทนที่ `@username` ด้วย ID ผู้ส่งแบบตัวเลข |

การแก้ไขปัญหาแบบเต็ม: [การแก้ไขปัญหา Telegram](/th/channels/telegram#troubleshooting)
