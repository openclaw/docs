GMI Cloud เป็นแพลตฟอร์มอนุมานแบบโฮสต์สำหรับโมเดลแนวหน้าและโมเดลแบบเปิดน้ำหนัก
ที่ให้บริการผ่าน API ซึ่งเข้ากันได้กับ OpenAI ใน OpenClaw แพลตฟอร์มนี้เป็น Plugin
ผู้ให้บริการภายนอกอย่างเป็นทางการ: ติดตั้งเพียงครั้งเดียว จัดเก็บข้อมูลประจำตัวผ่าน
การยืนยันตัวตนของโมเดลตามปกติ แล้วใช้การอ้างอิงโมเดล เช่น `gmi/google/gemini-3.1-flash-lite`

ใช้ GMI เมื่อต้องการใช้คีย์ API เดียวสำหรับโมเดลแบบโฮสต์หลายตระกูล รวมถึง
เส้นทางของ Anthropic, DeepSeek, Google, Moonshot, OpenAI และ Z.AI ที่แสดงอยู่ใน
แค็ตตาล็อกของ GMI แพลตฟอร์มนี้ใช้เป็นผู้ให้บริการสำรองสำหรับการสลับไปใช้โมเดลอื่น
เมื่อเกิดข้อผิดพลาด ใช้เปรียบเทียบเส้นทางแบบโฮสต์ระหว่างผู้จำหน่าย หรือใช้เมื่อ GMI
มีโมเดลให้บริการก่อนผู้ให้บริการหลักของคุณ OpenClaw เป็นผู้ดูแลรหัสผู้ให้บริการ
โปรไฟล์การยืนยันตัวตน นามแฝง ข้อมูลตั้งต้นของแค็ตตาล็อกโมเดล และ URL ฐาน ส่วน GMI
เป็นผู้ดูแลความพร้อมใช้งานจริงของโมเดล การเรียกเก็บเงิน ขีดจำกัดอัตรา และนโยบาย
การกำหนดเส้นทางฝั่งผู้ให้บริการทั้งหมด

| คุณสมบัติ              | ค่า                                      |
| ---------------------- | ---------------------------------------- |
| รหัสผู้ให้บริการ       | `gmi` (นามแฝง: `gmi-cloud`, `gmicloud`) |
| แพ็กเกจ                | `@openclaw/gmi-provider`                 |
| ตัวแปรสภาพแวดล้อมการยืนยันตัวตน | `GMI_API_KEY`                  |
| API                    | เข้ากันได้กับ OpenAI (`openai-completions`) |
| URL ฐาน                | `https://api.gmi-serving.com/v1`         |
| โมเดลเริ่มต้น          | `gmi/google/gemini-3.1-flash-lite`       |

## การตั้งค่า

ติดตั้ง Plugin รีสตาร์ต Gateway จากนั้นสร้างคีย์ API ใน GMI Cloud
(`https://www.gmicloud.ai/`):

```bash
openclaw plugins install @openclaw/gmi-provider
openclaw gateway restart
```

จากนั้นเรียกใช้:

```bash
openclaw onboard --auth-choice gmi-api-key
```

การตั้งค่าแบบไม่โต้ตอบสามารถส่ง `--gmi-api-key <key>` หรือกำหนดค่า:

```bash
export GMI_API_KEY="<your-gmi-api-key>" # pragma: allowlist secret
```

## เมื่อใดควรเลือก GMI

- คุณต้องการปลายทางแบบโฮสต์ที่เข้ากันได้กับ OpenAI แทนเซิร์ฟเวอร์โมเดลภายในเครื่อง
- คุณต้องการทดลองโมเดลเชิงพาณิชย์และโมเดลแบบเปิดน้ำหนักหลายตระกูลผ่านบัญชี
  ผู้ให้บริการบัญชีเดียว
- คุณต้องการผู้ให้บริการสำรองที่ใช้การกำหนดเส้นทางต้นทางแตกต่างจาก DeepInfra,
  OpenRouter, Together หรือ API โดยตรงของผู้จำหน่าย
- คุณต้องการรหัสโมเดล ราคา หรือการควบคุมบัญชีเฉพาะของ GMI

ให้เลือกผู้ให้บริการโดยตรงของผู้จำหน่ายแทน เมื่อคุณต้องการคุณสมบัติเฉพาะของผู้จำหน่าย
ที่ GMI ไม่ได้เปิดให้ใช้ผ่านเส้นทางซึ่งเข้ากันได้กับ OpenAI เลือกผู้ให้บริการภายในเครื่อง
เช่น LM Studio, Ollama, SGLang หรือ vLLM เมื่อความใกล้ชิดของข้อมูลหรือการควบคุม GPU
ภายในเครื่องสำคัญกว่าความสะดวกของบริการแบบโฮสต์

## โมเดล

แค็ตตาล็อกของ Plugin มีข้อมูลตั้งต้นของรหัสเส้นทาง GMI Cloud ที่มีให้ใช้งานโดยทั่วไป:

| การอ้างอิงโมเดล                    | อินพุต       | บริบท     | เอาต์พุตสูงสุด |
| ---------------------------------- | ------------ | --------- | -------------- |
| `gmi/anthropic/claude-sonnet-4.6`  | ข้อความ + รูปภาพ | 200,000   | 64,000         |
| `gmi/deepseek-ai/DeepSeek-V3.2`    | ข้อความ      | 163,840   | 65,536         |
| `gmi/google/gemini-3.1-flash-lite` | ข้อความ + รูปภาพ | 1,048,576 | 65,536         |
| `gmi/moonshotai/Kimi-K2.5`         | ข้อความ + รูปภาพ | 262,144   | 65,536         |
| `gmi/openai/gpt-5.4`               | ข้อความ + รูปภาพ | 400,000   | 128,000        |
| `gmi/zai-org/GLM-5.1-FP8`          | ข้อความ      | 202,752   | 65,536         |

แค็ตตาล็อกเป็นเพียงข้อมูลตั้งต้น ไม่ใช่การรับประกันว่าทุกบัญชีจะสามารถเรียกใช้ทุกโมเดล
ได้ตลอดเวลา แสดงรายการที่ผู้ให้บริการซึ่งกำหนดค่าไว้รายงานในสภาพแวดล้อมของคุณ:

```bash
openclaw models list --provider gmi
```

## การแก้ไขปัญหา

- `401` หรือ `403`: ตรวจสอบว่าได้กำหนด `GMI_API_KEY` ให้กับโปรเซสที่เรียกใช้
  OpenClaw แล้ว หรือเรียกใช้การเริ่มต้นใช้งานอีกครั้งเพื่อจัดเก็บคีย์ในโปรไฟล์
  การยืนยันตัวตนของผู้ให้บริการ
- ข้อผิดพลาดไม่รู้จักโมเดล: ตรวจสอบว่าโมเดลมีอยู่ในบัญชี GMI ของคุณ และใช้การอ้างอิง
  `gmi/<route-id>` แบบเต็มที่แสดงโดย `openclaw models list --provider gmi`
- ข้อผิดพลาดของผู้ให้บริการที่เกิดขึ้นเป็นครั้งคราว: ลองใช้เส้นทาง GMI อื่น หรือกำหนด
  ให้ GMI เป็นผู้ให้บริการสำรองแทนการใช้เป็นผู้ให้บริการโมเดลหลักเพียงรายเดียว
