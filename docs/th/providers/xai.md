OpenClaw มาพร้อมกับ Plugin ผู้ให้บริการ `xai` ที่รวมมาในตัวสำหรับโมเดล Grok แนวทางที่แนะนำคือใช้ Grok OAuth กับการสมัครสมาชิก SuperGrok หรือ X Premium ที่มีสิทธิ์ใช้งาน Gateway การกำหนดค่า การกำหนดเส้นทาง และเครื่องมือต่าง ๆ ยังคงทำงานภายในเครื่อง มีเพียงคำขอ Grok เท่านั้นที่ส่งไปยัง API ของ xAI

OAuth ไม่จำเป็นต้องใช้คีย์ API ของ xAI หรือแอป Grok Build แต่ xAI อาจยังแสดง Grok Build บนหน้าจอให้ความยินยอม เนื่องจาก OpenClaw ใช้ไคลเอนต์ OAuth ที่ใช้ร่วมกันของ xAI

## การตั้งค่า

<Steps>
  <Step title="การติดตั้งใหม่">
    เรียกใช้ขั้นตอนเริ่มต้นใช้งานพร้อมติดตั้งดีมอน จากนั้นเลือก xAI/Grok OAuth ในขั้นตอนโมเดล/การยืนยันตัวตน:

    ```bash
    openclaw onboard --install-daemon
    ```

    บน VPS หรือผ่าน SSH ให้เลือก xAI OAuth โดยตรง ซึ่งใช้การยืนยันด้วยรหัสอุปกรณ์และไม่ต้องใช้คอลแบ็ก localhost:

    ```bash
    openclaw onboard --install-daemon --auth-choice xai-oauth
    ```

  </Step>
  <Step title="การติดตั้งที่มีอยู่">
    ลงชื่อเข้าใช้ xAI เท่านั้น อย่าเรียกใช้ขั้นตอนเริ่มต้นใช้งานทั้งหมดอีกครั้งเพียงเพื่อเชื่อมต่อ Grok:

    ```bash
    openclaw models auth login --provider xai --method oauth
    ```

    ตั้งให้ Grok เป็นโมเดลเริ่มต้นแยกต่างหาก:

    ```bash
    openclaw models set xai/grok-4.3
    ```

    เรียกใช้ขั้นตอนเริ่มต้นใช้งานทั้งหมดอีกครั้งเฉพาะเมื่อคุณตั้งใจเปลี่ยน Gateway, ดีมอน, ช่องทาง, พื้นที่ทำงาน หรือตัวเลือกการตั้งค่าอื่น ๆ

  </Step>
  <Step title="แนวทางคีย์ API">
    การตั้งค่าด้วยคีย์ API ยังคงใช้ได้กับคีย์จาก xAI Console และพื้นผิวสื่อที่ต้องใช้การกำหนดค่าผู้ให้บริการซึ่งมีคีย์รองรับ:

    ```bash
    openclaw models auth login --provider xai --method api-key
    export XAI_API_KEY=xai-...
    ```

  </Step>
  <Step title="เลือกโมเดล">
    ```json5
    {
      agents: { defaults: { model: { primary: "xai/grok-4.3" } } },
    }
    ```
  </Step>
</Steps>

<Note>
OpenClaw ใช้ xAI Responses API เป็นการขนส่ง xAI ที่รวมมาในตัว ข้อมูลรับรองเดียวกันจาก `openclaw models auth login --provider xai --method oauth` หรือ `--method api-key` ยังใช้ขับเคลื่อน `web_search` (รหัสผู้ให้บริการ `grok`), `x_search`, `code_execution`, การสังเคราะห์เสียง/การถอดเสียง และการสร้างรูปภาพ/วิดีโอของ xAI ด้วย หากคุณจัดเก็บคีย์ xAI ไว้ที่ `plugins.entries.xai.config.webSearch.apiKey` ผู้ให้บริการโมเดล xAI ที่รวมมาในตัวจะนำคีย์นั้นมาใช้เป็นทางเลือกสำรองด้วย
</Note>

## การแก้ไขปัญหา OAuth

- สำหรับ SSH, Docker, VPS หรือการตั้งค่าระยะไกลอื่น ๆ ให้ใช้ `openclaw models auth login --provider xai --method oauth` ซึ่งใช้การยืนยันด้วยรหัสอุปกรณ์ ไม่ใช่คอลแบ็ก localhost
- หากลงชื่อเข้าใช้สำเร็จแล้วแต่ Grok ไม่ใช่โมเดลเริ่มต้น ให้เรียกใช้ `openclaw models set xai/grok-4.3`
- ตรวจสอบโปรไฟล์การยืนยันตัวตน xAI ที่บันทึกไว้:

  ```bash
  openclaw models auth list --provider xai
  openclaw models status
  ```

- xAI เป็นผู้กำหนดว่าบัญชีใดสามารถรับโทเค็น API ผ่าน OAuth ได้ หากบัญชีไม่มีสิทธิ์ ให้ใช้แนวทางคีย์ API หรือตรวจสอบการสมัครสมาชิกทางฝั่ง xAI

<Tip>
ใช้ `xai-oauth` เมื่อลงชื่อเข้าใช้จาก SSH, Docker หรือ VPS OpenClaw จะแสดง URL และรหัสสั้น ๆ ให้ดำเนินการลงชื่อเข้าใช้ให้เสร็จสิ้นในเบราว์เซอร์ภายในเครื่องใดก็ได้ ขณะที่กระบวนการระยะไกลตรวจสอบ xAI เป็นระยะเพื่อรอการแลกเปลี่ยนโทเค็นให้เสร็จสมบูรณ์
</Tip>

## แค็ตตาล็อกในตัว

รหัสที่เลือกได้ในตัวเลือกโมเดล Plugin ยังคงรองรับการแก้ไขรหัส Grok 3, Grok 4, Grok 4 Fast, Grok 4.1 Fast และ Grok Code รุ่นเก่าสำหรับการกำหนดค่าที่มีอยู่ โปรดดู[ความเข้ากันได้แบบเดิมและนามแฝงที่เปลี่ยนแปลง](#legacy-compatibility-and-moving-aliases)

| ตระกูล         | รหัสโมเดล                                                    |
| -------------- | ------------------------------------------------------------ |
| Grok 4.5       | `grok-4.5` (นามแฝง: `grok-4.5-latest`, `grok-build-latest`) |
| Grok Build 0.1 | `grok-build-0.1`                                             |
| Grok 4.3       | `grok-4.3` (นามแฝง: `grok-4.3-latest`, `grok-latest`)       |
| Grok 4.20      | `grok-4.20-0309-reasoning`, `grok-4.20-0309-non-reasoning`   |

<Tip>
ใช้ `grok-4.5` สำหรับการสนทนาทั่วไป การเขียนโค้ด และงานแบบเอเจนต์ในพื้นที่ที่พร้อมใช้งาน Grok 4.3 ยังคงเป็นค่าเริ่มต้นสำหรับการตั้งค่าที่ปลอดภัยในระดับภูมิภาค ส่วน `grok-build-0.1` และ Grok 4.20 ทั้งสองรูปแบบที่ระบุวันที่ยังคงเลือกได้
</Tip>

## ความครอบคลุมของคุณสมบัติ

Plugin ที่รวมมาในตัวจะแมป API ของ xAI ที่รองรับเข้ากับสัญญาผู้ให้บริการและเครื่องมือที่ใช้ร่วมกันของ OpenClaw ความสามารถที่ไม่เข้ากับสัญญาที่ใช้ร่วมกันจะแสดงไว้ด้านล่างหรือในส่วนข้อจำกัดที่ทราบ

| ความสามารถของ xAI             | พื้นผิวของ OpenClaw                        | สถานะ                                                        |
| -------------------------- | --------------------------------------- | ------------------------------------------------------------- |
| การสนทนา / Responses           | ผู้ให้บริการโมเดล `xai/<model>`            | รองรับ                                                           |
| การค้นหาเว็บฝั่งเซิร์ฟเวอร์     | ผู้ให้บริการ `grok` ของ `web_search`            | รองรับ                                                           |
| การค้นหา X ฝั่งเซิร์ฟเวอร์       | เครื่องมือ `x_search`                         | รองรับ                                                           |
| การเรียกใช้โค้ดฝั่งเซิร์ฟเวอร์ | เครื่องมือ `code_execution`                   | รองรับ                                                           |
| รูปภาพ                     | `image_generate`                        | รองรับ                                                           |
| วิดีโอ                     | `video_generate`                        | เวิร์กโฟลว์แบบคลาสสิกเต็มรูปแบบ; Video 1.5 แปลงรูปภาพเป็นวิดีโอ               |
| การแปลงข้อความเป็นเสียงแบบชุด       | `messages.tts.provider: "xai"` / `tts`  | รองรับ                                                           |
| TTS แบบสตรีม              | -                                       | ผู้ให้บริการ xAI ยังไม่ได้ดำเนินการ                       |
| การแปลงเสียงเป็นข้อความแบบชุด       | การทำความเข้าใจสื่อ `tools.media.audio` | รองรับ                                                           |
| การแปลงเสียงเป็นข้อความแบบสตรีม   | Voice Call `streaming.provider: "xai"`  | รองรับ                                                           |
| เสียงแบบเรียลไทม์             | -                                       | ยังไม่เปิดให้ใช้งาน ต้องใช้สัญญาเซสชัน/WebSocket แบบอื่น |
| ไฟล์ / งานแบบชุด            | รองรับความเข้ากันได้กับ API โมเดลทั่วไปเท่านั้น    | ไม่ใช่เครื่องมือ OpenClaw ระดับชั้นหนึ่ง                               |

<Note>
OpenClaw ใช้ API แบบ REST ของ xAI สำหรับรูปภาพ/วิดีโอ/TTS/STT เพื่อสร้างสื่อและถอดเสียงแบบชุด ใช้ WebSocket สำหรับ STT แบบสตรีมของ xAI เพื่อถอดเสียงการโทรด้วยเสียงแบบสด และใช้ Responses API สำหรับการสนทนา การค้นหา และเครื่องมือเรียกใช้โค้ด
</Note>

### ความเข้ากันได้กับโหมดเร็วแบบเดิม

`/fast on` หรือ `agents.defaults.models["xai/<model>"].params.fastMode: true` ยังคงเขียนการกำหนดค่า xAI รุ่นเก่าใหม่ดังต่อไปนี้ รหัสเป้าหมายเหล่านี้เก็บไว้เพื่อความเข้ากันได้เท่านั้น สำหรับการกำหนดค่าใหม่ให้ใช้โมเดลปัจจุบันที่เลือกได้

| โมเดลต้นทาง  | เป้าหมายโหมดเร็ว   |
| ------------- | ------------------ |
| `grok-3`      | `grok-3-fast`      |
| `grok-3-mini` | `grok-3-mini-fast` |
| `grok-4`      | `grok-4-fast`      |
| `grok-4-0709` | `grok-4-fast`      |

### ความเข้ากันได้แบบเดิมและนามแฝงที่เปลี่ยนแปลง

นามแฝงรุ่นเก่าจะถูกปรับให้เป็นมาตรฐานดังนี้:

| นามแฝงรุ่นเก่า                                                  | รหัสที่ปรับเป็นมาตรฐานแล้ว    |
| ------------------------------------------------------------- | ---------------- |
| `grok-code-fast-1`, `grok-code-fast`, `grok-code-fast-1-0825` | `grok-build-0.1` |

รหัส 0309 ที่ระบุวันที่เป็นรายการในแค็ตตาล็อกที่เลือกได้ OpenClaw จะส่งนามแฝง Grok 4.20 ปัจจุบันอื่น ๆ ทั้งหมดตามเดิม เพื่อให้ xAI ยังคงควบคุมความหมายของนามแฝงแบบเสถียร ล่าสุด เบต้า ทดลอง และแบบระบุวันที่ นามแฝงส่วนกลาง `grok-latest` จะถูกคงไว้ตามเดิมเช่นกัน

xAI เลิกใช้รหัสต่อไปนี้แล้ว OpenClaw เก็บรหัสเหล่านี้เป็นแถวความเข้ากันได้ที่ซ่อนไว้สำหรับการกำหนดค่าที่เผยแพร่แล้ว โดยใช้ข้อจำกัดและราคาของเป้าหมายการเปลี่ยนเส้นทางปัจจุบัน:

| รหัสที่เลิกใช้                                                          | พฤติกรรมปัจจุบัน                 |
| -------------------------------------------------------------------- | -------------------------------- |
| `grok-4-1-fast-reasoning`, `grok-4-fast-reasoning`, `grok-4-0709`    | Grok 4.3 พร้อมการใช้เหตุผลระดับ `low`    |
| `grok-4-1-fast-non-reasoning`, `grok-4-fast-non-reasoning`, `grok-3` | Grok 4.3 โดยปิดใช้การให้เหตุผล |
| `grok-code-fast-1`                                                   | Grok Build 0.1                   |
| `grok-imagine-image-pro`                                             | Grok Imagine Image Quality       |

`openclaw doctor --fix` จะอัปเดตค่าเริ่มต้นของเครื่องมือฝั่งเซิร์ฟเวอร์ xAI ที่คงอยู่และชื่อย่อรูปภาพคุณภาพที่เลิกใช้แล้ว ลบแถวแค็ตตาล็อกที่สร้างขึ้นและล้าสมัย และซ่อมแซมเมทาดาทาบริบทที่ล้าสมัยในแถว 4.20 ที่ใช้งานอยู่ โดยจะไม่ตรึงนามแฝง `beta-latest` ของ 4.20 ที่ใช้งานอยู่กับสแนปช็อตที่ระบุวันที่

## คุณสมบัติ

<Warning>
  `x_search` และ `code_execution` ทำงานบนเซิร์ฟเวอร์ของ xAI โดย xAI คิดค่าบริการ $5 ต่อการเรียกใช้เครื่องมือ 1,000 ครั้ง บวกกับโทเค็นอินพุตและเอาต์พุตของโมเดล เมื่อไม่ได้ระบุการตั้งค่า `enabled` ของแต่ละเครื่องมือ OpenClaw จะเปิดให้ใช้เครื่องมือนั้นเฉพาะกับโมเดล xAI ที่ใช้งานอยู่เท่านั้น ผู้ให้บริการโมเดลที่ทราบแน่ชัดว่าไม่ใช่ xAI ต้องกำหนด `enabled: true` อย่างชัดเจนเป็นรายเครื่องมือ ส่วนผู้ให้บริการที่หายไปหรือไม่สามารถระบุได้จะปฏิเสธการทำงานโดยค่าเริ่มต้น ต้องมีการยืนยันตัวตน xAI เสมอ และ `enabled: false` จะปิดใช้เครื่องมือสำหรับผู้ให้บริการทุกราย
</Warning>

    - ความละเอียด: แบบคลาสสิก `480P`/`720P`; Video 1.5 รองรับ `1080P` เพิ่มเติม; โหมดการสร้างทั้งหมดใช้ค่าเริ่มต้นเป็น `480P`
    - ระยะเวลา: 1-15 วินาทีสำหรับการสร้าง/แปลงรูปภาพเป็นวิดีโอ, 1-10 วินาทีเมื่อใช้บทบาท `reference_image` แบบคลาสสิก, 2-10 วินาทีสำหรับการต่อขยายแบบคลาสสิก
    - การสร้างจากรูปภาพอ้างอิง: ตั้งค่า `imageRoles` เป็น `reference_image` สำหรับรูปภาพทุกภาพที่ระบุ โดย xAI ยอมรับรูปภาพดังกล่าวได้สูงสุด 7 ภาพ
    - การแก้ไข/ต่อขยายวิดีโอจะสืบทอดอัตราส่วนภาพและความละเอียดของวิดีโออินพุต การดำเนินการเหล่านั้นไม่ยอมรับการแทนที่ค่ามิติ
    - ระยะหมดเวลาการดำเนินการเริ่มต้น: 600 วินาที เว้นแต่จะตั้งค่า `video_generate.timeoutMs` หรือ `agents.defaults.videoGenerationModel.timeoutMs`

    <Warning>
    ไม่รองรับบัฟเฟอร์วิดีโอภายในเครื่อง ให้ใช้ URL ระยะไกลแบบ `http(s)` สำหรับอินพุตการแก้ไข/ต่อขยายวิดีโอ การแปลงรูปภาพเป็นวิดีโอรองรับบัฟเฟอร์รูปภาพภายในเครื่อง เนื่องจาก OpenClaw เข้ารหัสบัฟเฟอร์เหล่านั้นเป็น URL ข้อมูลสำหรับ xAI
    </Warning>

    Video 1.5 ยังรองรับตัวระบุ `grok-imagine-video-1.5-preview` และ `grok-imagine-video-1.5-2026-05-30` ของ xAI ด้วย OpenClaw จะส่งต่อตัวระบุที่เลือกโดยไม่เปลี่ยนแปลง แต่ใช้การตรวจสอบแบบรับเฉพาะรูปภาพเช่นเดียวกัน

    หากต้องการใช้ xAI เป็นผู้ให้บริการวิดีโอเริ่มต้น:

    ```json5
    {
      agents: {
        defaults: {
          videoGenerationModel: {
            primary: "xai/grok-imagine-video",
          },
        },
      },
    }
    ```

    <Note>
    โปรดดู[การสร้างวิดีโอ](/th/tools/video-generation)สำหรับพารามิเตอร์เครื่องมือที่ใช้ร่วมกัน การเลือกผู้ให้บริการ และพฤติกรรมการสลับไปใช้ระบบสำรอง
    </Note>

  </Accordion>

  <Accordion title="การสร้างรูปภาพ">
    Plugin `xai` ที่รวมมาในตัวจะลงทะเบียนการสร้างรูปภาพผ่านเครื่องมือ `image_generate` ที่ใช้ร่วมกัน

    - โมเดลรูปภาพเริ่มต้น: `xai/grok-imagine-image`
    - โมเดลเพิ่มเติม: `xai/grok-imagine-image-quality`
    - โหมด: แปลงข้อความเป็นรูปภาพและแก้ไขรูปภาพอ้างอิง
    - อินพุตอ้างอิง: `image` หนึ่งรายการหรือ `images` สูงสุดสามรายการ
    - อัตราส่วนภาพ: `1:1`, `16:9`, `9:16`, `4:3`, `3:4`, `3:2`, `2:3`, `2:1`,
      `1:2`, `19.5:9`, `9:19.5`, `20:9`, `9:20`
    - ความละเอียด: `1K`, `2K`
    - จำนวน: สูงสุด 4 รูปภาพ
    - ระยะหมดเวลาการดำเนินการเริ่มต้น: 600 วินาที เว้นแต่ตั้งค่า `image_generate.timeoutMs`
      หรือ `agents.defaults.imageGenerationModel.timeoutMs`

    OpenClaw ขอให้ xAI ส่งการตอบกลับรูปภาพแบบ `b64_json` เพื่อให้สามารถจัดเก็บ
    และส่งสื่อที่สร้างขึ้นผ่านเส้นทางไฟล์แนบของช่องทางตามปกติได้ รูปภาพอ้างอิง
    ภายในเครื่องจะถูกแปลงเป็น URL ข้อมูล ส่วนการอ้างอิง `http(s)` ระยะไกล
    จะถูกส่งต่อโดยไม่เปลี่ยนแปลง

    หากต้องการใช้ xAI เป็นผู้ให้บริการรูปภาพเริ่มต้น:

    ```json5
    {
      agents: {
        defaults: {
          imageGenerationModel: {
            primary: "xai/grok-imagine-image",
          },
        },
      },
    }
    ```

    <Note>
    xAI ยังจัดทำเอกสารสำหรับ `quality`, `mask`, `user` และอัตราส่วนภาพ `auto`
    ปัจจุบัน OpenClaw ส่งต่อเฉพาะตัวควบคุมรูปภาพร่วมที่ใช้ข้ามผู้ให้บริการเท่านั้น
    ตัวเลือกเฉพาะของระบบเหล่านี้ยังไม่เปิดให้ใช้งานผ่าน `image_generate`
    </Note>

  </Accordion>

  <Accordion title="Text-to-speech">
    Plugin `xai` ที่รวมมาให้ลงทะเบียนการแปลงข้อความเป็นเสียงผ่านพื้นผิวผู้ให้บริการ
    `tts` ที่ใช้ร่วมกัน

    - เสียง: แค็ตตาล็อกแบบสดที่ยืนยันตัวตนจาก xAI; แสดงรายการด้วย
      `openclaw infer tts voices --provider xai`
    - เสียงสำรองแบบออฟไลน์: `ara`, `eve`, `leo`, `rex`, `sal`
    - เสียงเริ่มต้น: `eve`
    - ID เสียงแบบกำหนดเองของบัญชีจะถูกส่งต่อ แม้จะไม่มีอยู่ในการตอบกลับ
      แค็ตตาล็อกในตัว
    - รูปแบบ: `mp3`, `wav`, `pcm`, `mulaw`, `alaw`
    - ภาษา: รหัส BCP-47 หรือ `auto`
    - ความเร็ว: การแทนที่ความเร็วแบบเฉพาะของผู้ให้บริการ
    - ไม่รองรับรูปแบบข้อความเสียง Opus แบบเนทีฟ

    หากต้องการใช้ xAI เป็นผู้ให้บริการ TTS เริ่มต้น:

    ```json5
    {
      messages: {
        tts: {
          provider: "xai",
          providers: {
            xai: {
              voiceId: "eve",
            },
          },
        },
      },
    }
    ```

    <Note>
    OpenClaw ใช้ปลายทางแบบชุด `/v1/tts` และแค็ตตาล็อก `/v1/tts/voices`
    ที่ยืนยันตัวตนของ xAI นอกจากนี้ xAI ยังมี TTS แบบสตรีมผ่าน WebSocket
    แต่ผู้ให้บริการ xAI ที่รวมมาให้ยังไม่ได้ใช้ฮุกการสตรีมดังกล่าว
    </Note>

  </Accordion>

  <Accordion title="Speech-to-text">
    Plugin `xai` ที่รวมมาให้ลงทะเบียนการแปลงเสียงเป็นข้อความแบบชุด
    ผ่านพื้นผิวการถอดเสียงเพื่อทำความเข้าใจสื่อของ OpenClaw

    - ปลายทาง: xAI REST `/v1/stt`
    - เส้นทางอินพุต: การอัปโหลดไฟล์เสียงแบบหลายส่วน
    - การเลือกโมเดล: xAI เลือกโมเดลการถอดเสียงภายใน โดยปลายทาง
      ไม่มีตัวเลือกโมเดล
    - ใช้ในทุกตำแหน่งที่การถอดเสียงขาเข้าอ่าน `tools.media.audio`
      รวมถึงส่วนเสียงจากช่องเสียง Discord และไฟล์แนบเสียงของช่องทาง

    หากต้องการบังคับใช้ xAI สำหรับการถอดเสียงขาเข้า:

    ```json5
    {
      tools: {
        media: {
          audio: {
            models: [
              {
                type: "provider",
                provider: "xai",
              },
            ],
          },
        },
      },
    }
    ```

    สามารถระบุภาษาผ่านการกำหนดค่าสื่อเสียงที่ใช้ร่วมกัน หรือผ่านคำขอ
    ถอดเสียงแต่ละครั้งได้ พื้นผิว OpenClaw ที่ใช้ร่วมกันยอมรับคำแนะนำพรอมต์
    แต่การเชื่อมต่อ xAI REST STT จะส่งต่อเฉพาะไฟล์และภาษา เนื่องจากมีเพียง
    สองรายการนี้ที่สอดคล้องกับปลายทางสาธารณะของ xAI ในปัจจุบัน

  </Accordion>

  <Accordion title="Streaming speech-to-text">
    Plugin `xai` ที่รวมมาให้ยังลงทะเบียนผู้ให้บริการถอดเสียงแบบเรียลไทม์
    สำหรับเสียงจากการโทรสดด้วย

    - ปลายทาง: xAI WebSocket `wss://api.x.ai/v1/stt`
    - การเข้ารหัสเริ่มต้น: `mulaw`
    - อัตราการสุ่มตัวอย่างเริ่มต้น: `8000`
    - การตรวจจับจุดสิ้นสุดเริ่มต้น: `800ms`
    - ข้อความถอดเสียงระหว่างดำเนินการ: เปิดใช้งานโดยค่าเริ่มต้น

    สตรีมสื่อ Twilio ของ Voice Call ส่งเฟรมเสียง G.711 mu-law ดังนั้น
    ผู้ให้บริการ xAI จึงส่งต่อเฟรมเหล่านั้นโดยตรงโดยไม่แปลงรหัส:

    ```json5
    {
      plugins: {
        entries: {
          "voice-call": {
            config: {
              streaming: {
                enabled: true,
                provider: "xai",
                providers: {
                  xai: {
                    apiKey: "${XAI_API_KEY}",
                    endpointingMs: 800,
                    language: "en",
                  },
                },
              },
            },
          },
        },
      },
    }
    ```

    การกำหนดค่าที่ผู้ให้บริการเป็นเจ้าของอยู่ภายใต้
    `plugins.entries.voice-call.config.streaming.providers.xai` คีย์ที่รองรับ
    ได้แก่ `apiKey`, `baseUrl`, `sampleRate`, `encoding` (`pcm`, `mulaw` หรือ
    `alaw`), `interimResults`, `endpointingMs` และ `language`

    <Note>
    ผู้ให้บริการสตรีมนี้ใช้สำหรับเส้นทางการถอดเสียงแบบเรียลไทม์ของ Voice Call
    ส่วน Discord จะบันทึกช่วงเสียงสั้น ๆ และใช้เส้นทางการถอดเสียงแบบชุด
    `tools.media.audio` แทน
    </Note>

  </Accordion>

  <Accordion title="Known limits">
    - การยืนยันตัวตนกับ xAI สามารถใช้คีย์ API, ตัวแปรสภาพแวดล้อม, การกำหนดค่า
      สำรองของ Plugin หรือ OAuth ด้วยบัญชี xAI ที่มีสิทธิ์ OAuth ใช้การยืนยัน
      ด้วยรหัสอุปกรณ์โดยไม่มีการเรียกกลับไปยัง localhost โดย xAI เป็นผู้ตัดสินว่า
      บัญชีใดสามารถรับโทเค็น OAuth API ได้ และหน้าคำยินยอมอาจแสดง Grok Build
      แม้ว่า OpenClaw จะไม่ต้องใช้แอป Grok Build
    - ปัจจุบัน OpenClaw ยังไม่เปิดใช้ตระกูลโมเดลหลายเอเจนต์ของ xAI โดย xAI
      ให้บริการโมเดลเหล่านี้ผ่าน Responses API แต่โมเดลไม่ยอมรับเครื่องมือ
      ฝั่งไคลเอนต์หรือเครื่องมือแบบกำหนดเองที่ลูปเอเจนต์ร่วมของ OpenClaw ใช้
      ดู
      [ข้อจำกัดของระบบหลายเอเจนต์ xAI](https://docs.x.ai/developers/model-capabilities/text/multi-agent#limitations)
    - เสียงแบบเรียลไทม์ของ xAI ยังไม่ได้ลงทะเบียนเป็นผู้ให้บริการ OpenClaw
      เนื่องจากต้องใช้สัญญาเซสชันเสียงแบบสองทิศทางที่แตกต่างจาก STT แบบชุด
      หรือการถอดเสียงแบบสตรีม
    - `quality` ของรูปภาพ xAI, `mask` ของรูปภาพ และอัตราส่วนภาพ `auto`
      แบบเนทีฟจะยังไม่เปิดใช้จนกว่าเครื่องมือ `image_generate` ที่ใช้ร่วมกัน
      จะมีตัวควบคุมข้ามผู้ให้บริการที่สอดคล้องกัน
  </Accordion>

  <Accordion title="Advanced notes">
    - OpenClaw ใช้การแก้ไขความเข้ากันได้ของสคีมาเครื่องมือและการเรียกเครื่องมือ
      ที่เฉพาะสำหรับ xAI โดยอัตโนมัติบนเส้นทางตัวเรียกใช้ร่วม
    - คำขอ xAI แบบเนทีฟใช้ค่าเริ่มต้น `tool_stream: true` ตั้งค่า
      `agents.defaults.models["xai/<model>"].params.tool_stream` เป็น `false`
      เพื่อปิดใช้งาน
    - ตัวห่อ xAI ที่รวมมาให้จะตัดขอบเขตสคีมาจำนวนสมาชิกที่ไม่รองรับ
      และคีย์เพย์โหลด *effort* สำหรับการให้เหตุผลที่ไม่รองรับ ก่อนส่งคำขอ
      xAI แบบเนทีฟ Grok 4.5 รองรับระดับความพยายามต่ำ ปานกลาง และสูง
      (ค่าเริ่มต้นคือสูง) Grok 4.3 รองรับไม่มี ต่ำ ปานกลาง และสูง
      (ค่าเริ่มต้นคือต่ำ) โมเดล xAI อื่นที่รองรับการให้เหตุผลไม่มีตัวควบคุม
      ระดับความพยายามที่กำหนดค่าได้ แต่ยังคงส่งคำขอ
      `include: ["reasoning.encrypted_content"]` เพื่อให้สามารถนำการให้เหตุผล
      ที่เข้ารหัสไว้ก่อนหน้ามาใช้ซ้ำในรอบถัดไปได้
    - `web_search`, `x_search` และ `code_execution` เปิดใช้เป็นเครื่องมือ
      OpenClaw โดย OpenClaw จะแนบเฉพาะเครื่องมือในตัวของ xAI ที่แต่ละเครื่องมือ
      ต้องใช้ไปกับคำขอของเครื่องมือนั้น แทนการแนบเครื่องมือแบบเนทีฟทุกตัว
      ในทุกรอบการสนทนา
    - `web_search` ของ Grok อ่าน `plugins.entries.xai.config.webSearch.baseUrl`
      ส่วน `x_search` อ่าน `plugins.entries.xai.config.xSearch.baseUrl` แล้วจึง
      ย้อนกลับไปใช้ URL ฐานสำหรับการค้นหาเว็บของ Grok
    - `x_search` และ `code_execution` อยู่ภายใต้ความรับผิดชอบของ Plugin xAI
      ที่รวมมาให้ แทนการเขียนค่าตายตัวไว้ในรันไทม์โมเดลหลัก
    - `code_execution` คือการเรียกใช้ในแซนด์บ็อกซ์ xAI ระยะไกล ไม่ใช่
      [`exec`](/th/tools/exec) ภายในเครื่อง
  </Accordion>
</AccordionGroup>

## การทดสอบแบบสด

เส้นทางสื่อ xAI ครอบคลุมด้วยการทดสอบหน่วยและชุดทดสอบแบบสดที่ต้องเลือกเปิดใช้
ส่งออก `XAI_API_KEY` ในสภาพแวดล้อมของกระบวนการก่อนเรียกใช้การตรวจสอบแบบสด

```bash
pnpm test extensions/xai
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_TEST_QUIET=1 pnpm test:live -- extensions/xai/xai.live.test.ts
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_XAI_VIDEO=1 pnpm test:live -- extensions/xai/xai.live.test.ts -t "classic Grok Imagine"
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_XAI_VIDEO=1 pnpm test:live -- extensions/xai/xai.live.test.ts -t "Grok Imagine Video 1.5"
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_TEST_QUIET=1 pnpm test:live -- extensions/xai/x-search.live.test.ts
OPENCLAW_LIVE_GATEWAY_MODELS="xai/grok-4.5,xai/grok-build-0.1,xai/grok-4.3,xai/grok-4.20-0309-reasoning,xai/grok-4.20-0309-non-reasoning" OPENCLAW_LIVE_GATEWAY_MAX_MODELS=0 OPENCLAW_LIVE_GATEWAY_SMOKE=0 pnpm test:live -- src/gateway/gateway-models.profiles.live.test.ts
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_TEST_QUIET=1 OPENCLAW_LIVE_IMAGE_GENERATION_PROVIDERS=xai pnpm test:live -- test/image-generation.runtime.live.test.ts
```

ไฟล์ไลฟ์เฉพาะผู้ให้บริการสังเคราะห์ TTS แบบปกติ, TTS แบบ PCM
ที่เหมาะสำหรับระบบโทรศัพท์, ถอดเสียงผ่าน STT แบบแบตช์ของ xAI, สตรีม PCM เดียวกันผ่าน STT
แบบเรียลไทม์ของ xAI, สร้างผลลัพธ์ข้อความเป็นภาพ และแก้ไขภาพอ้างอิง
ไฟล์ไลฟ์สำหรับภาพที่ใช้ร่วมกันจะตรวจสอบผู้ให้บริการ xAI รายเดียวกันผ่านเส้นทาง
การเลือกรันไทม์ การใช้ตัวสำรอง การปรับให้อยู่ในรูปแบบมาตรฐาน และการแนบสื่อของ OpenClaw
กรณี Video 1.5 ที่ต้องเลือกเปิดใช้จะส่งภาพเฟรมแรกที่สร้างขึ้นหนึ่งภาพที่ความละเอียด 1080P และ
ตรวจสอบการดาวน์โหลดวิดีโอที่เสร็จสมบูรณ์
