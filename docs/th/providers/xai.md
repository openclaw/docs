OpenClaw มาพร้อม Plugin ผู้ให้บริการ `xai` แบบรวมชุดสำหรับโมเดล Grok แนวทางที่
แนะนำคือ Grok OAuth พร้อมการสมัครสมาชิก SuperGrok หรือ X Premium
ที่มีสิทธิ์ใช้งาน Gateway การกำหนดค่า การกำหนดเส้นทาง และเครื่องมือต่าง ๆ ยังคงทำงานภายในเครื่อง มีเพียงคำขอ
Grok เท่านั้นที่ส่งไปยัง API ของ xAI

OAuth ไม่ต้องใช้คีย์ API ของ xAI หรือแอป Grok Build แต่ xAI อาจยังคง
แสดง Grok Build บนหน้าจอยินยอม เนื่องจาก OpenClaw ใช้ไคลเอนต์
OAuth ที่ใช้ร่วมกันของ xAI

## การตั้งค่า

<Steps>
  <Step title="การติดตั้งใหม่">
    เรียกใช้ขั้นตอนเริ่มต้นพร้อมติดตั้งดีมอน จากนั้นเลือก xAI/Grok OAuth ในขั้นตอน
    โมเดล/การยืนยันตัวตน:

    ```bash
    openclaw onboard --install-daemon
    ```

    บน VPS หรือผ่าน SSH ให้เลือก xAI OAuth โดยตรง ซึ่งใช้การยืนยัน
    ด้วยรหัสอุปกรณ์และไม่ต้องใช้การเรียกกลับไปยัง localhost:

    ```bash
    openclaw onboard --install-daemon --auth-choice xai-oauth
    ```

  </Step>
  <Step title="การติดตั้งที่มีอยู่">
    ลงชื่อเข้าใช้ xAI เท่านั้น อย่าเรียกใช้ขั้นตอนเริ่มต้นทั้งหมดซ้ำเพียงเพื่อเชื่อมต่อ Grok:

    ```bash
    openclaw models auth login --provider xai --method oauth
    ```

    ตั้งค่า Grok เป็นโมเดลเริ่มต้นแยกต่างหาก:

    ```bash
    openclaw models set xai/grok-4.3
    ```

    เรียกใช้ขั้นตอนเริ่มต้นทั้งหมดซ้ำเฉพาะเมื่อคุณตั้งใจเปลี่ยน Gateway,
    ดีมอน ช่องทาง พื้นที่ทำงาน หรือตัวเลือกการตั้งค่าอื่น ๆ

  </Step>
  <Step title="เส้นทางคีย์ API">
    การตั้งค่าด้วยคีย์ API ยังคงใช้ได้กับคีย์จาก xAI Console และพื้นผิวสื่อ
    ที่ต้องใช้การกำหนดค่าผู้ให้บริการซึ่งรองรับด้วยคีย์:

    ```bash
    openclaw models auth login --provider xai --method api-key
    export XAI_API_KEY=xai-...
    ```

  </Step>
  <Step title="เลือกโมเดล">
    ```json5
    {
      agents: { defaults: { model: { primary: "xai/grok-4.3" } } },
    }
    ```
  </Step>
</Steps>

<Note>
OpenClaw ใช้ xAI Responses API เป็นการขนส่ง xAI แบบรวมชุด ข้อมูลประจำตัวเดียวกันจาก `openclaw models auth login --provider xai --method oauth` หรือ
`--method api-key` ยังขับเคลื่อน `web_search` (รหัสผู้ให้บริการ `grok`), `x_search`,
`code_execution`, เสียงพูด/การถอดเสียง และการสร้างภาพ/วิดีโอด้วย xAI หากคุณ
จัดเก็บคีย์ xAI ไว้ภายใต้ `plugins.entries.xai.config.webSearch.apiKey` ผู้ให้บริการโมเดล xAI
แบบรวมชุดจะนำคีย์นั้นมาใช้เป็นทางเลือกสำรองด้วย
</Note>

## การแก้ไขปัญหา OAuth

- สำหรับ SSH, Docker, VPS หรือการตั้งค่าระยะไกลอื่น ๆ ให้ใช้
  `openclaw models auth login --provider xai --method oauth` ซึ่งใช้
  การยืนยันด้วยรหัสอุปกรณ์ ไม่ใช่การเรียกกลับไปยัง localhost
- หากลงชื่อเข้าใช้สำเร็จแต่ Grok ไม่ใช่โมเดลเริ่มต้น ให้เรียกใช้
  `openclaw models set xai/grok-4.3`
- ตรวจสอบโปรไฟล์การยืนยันตัวตนของ xAI ที่บันทึกไว้:

  ```bash
  openclaw models auth list --provider xai
  openclaw models status
  ```

- xAI เป็นผู้ตัดสินว่าบัญชีใดสามารถรับโทเค็น API ผ่าน OAuth ได้ หากบัญชี
  ไม่มีสิทธิ์ ให้ใช้เส้นทางคีย์ API หรือตรวจสอบการสมัครสมาชิกทางฝั่ง xAI

<Tip>
ใช้ `xai-oauth` เมื่อลงชื่อเข้าใช้จาก SSH, Docker หรือ VPS โดย OpenClaw จะแสดง
URL และรหัสสั้น ๆ ให้ดำเนินการลงชื่อเข้าใช้ให้เสร็จสิ้นในเบราว์เซอร์ภายในเครื่องใดก็ได้ ขณะที่กระบวนการ
ระยะไกลสำรวจ xAI เพื่อรอให้การแลกเปลี่ยนโทเค็นเสร็จสมบูรณ์
</Tip>

## แค็ตตาล็อกในตัว

รหัสที่เลือกได้ในตัวเลือกโมเดล Plugin ยังคงตีความรหัส Grok 3,
Grok 4, Grok 4 Fast, Grok 4.1 Fast และ Grok Code รุ่นเก่าสำหรับการกำหนดค่าที่มีอยู่
ดู[ความเข้ากันได้กับระบบเดิมและนามแฝงที่เปลี่ยนแปลง](#legacy-compatibility-and-moving-aliases)

| ตระกูล         | รหัสโมเดล                                                    |
| -------------- | ------------------------------------------------------------ |
| Grok 4.5       | `grok-4.5` (นามแฝง: `grok-4.5-latest`, `grok-build-latest`) |
| Grok Build 0.1 | `grok-build-0.1`                                             |
| Grok 4.3       | `grok-4.3` (นามแฝง: `grok-4.3-latest`, `grok-latest`)       |
| Grok 4.20      | `grok-4.20-0309-reasoning`, `grok-4.20-0309-non-reasoning`   |

<Tip>
ใช้ `grok-4.5` สำหรับการสนทนาทั่วไป การเขียนโค้ด และงานแบบเอเจนต์ในพื้นที่ที่พร้อมให้ใช้งาน
Grok 4.3 ยังคงเป็นค่าเริ่มต้นในการตั้งค่าที่ปลอดภัยสำหรับแต่ละภูมิภาค ส่วน `grok-build-0.1` และ
Grok 4.20 ทั้งสองรุ่นที่ระบุวันที่ยังคงเลือกได้
</Tip>

## การครอบคลุมคุณสมบัติ

Plugin แบบรวมชุดจะแมป API ของ xAI ที่รองรับเข้ากับสัญญาผู้ให้บริการและ
เครื่องมือที่ใช้ร่วมกันของ OpenClaw ความสามารถที่ไม่เข้ากับสัญญาที่ใช้ร่วมกันจะแสดงไว้
ด้านล่างหรือในส่วนข้อจำกัดที่ทราบ

| ความสามารถของ xAI             | พื้นผิว OpenClaw                        | สถานะ                                               |
| -------------------------- | --------------------------------------- | ---------------------------------------------------- |
| แชต / Responses           | ผู้ให้บริการโมเดล `xai/<model>`            | รองรับ                                                  |
| การค้นหาเว็บฝั่งเซิร์ฟเวอร์     | ผู้ให้บริการ `web_search` `grok`            | รองรับ                                                  |
| การค้นหา X ฝั่งเซิร์ฟเวอร์       | เครื่องมือ `x_search`                         | รองรับ                                                  |
| การเรียกใช้โค้ดฝั่งเซิร์ฟเวอร์ | เครื่องมือ `code_execution`                   | รองรับ                                                  |
| ภาพ                     | `image_generate`                        | รองรับ                                                  |
| วิดีโอ                     | `video_generate`                        | รองรับ                                                  |
| การแปลงข้อความเป็นเสียงพูดแบบชุด       | `messages.tts.provider: "xai"` / `tts`  | รองรับ                                                  |
| TTS แบบสตรีม              | `textToSpeechStream`                    | รองรับผ่าน `wss://api.x.ai/v1/tts` (ไม่ใช่เสียงแบบเรียลไทม์) |
| การแปลงเสียงพูดเป็นข้อความแบบชุด       | การทำความเข้าใจสื่อ `tools.media.audio` | รองรับ                                                  |
| การแปลงเสียงพูดเป็นข้อความแบบสตรีม   | Plugin Voice Call `streaming.provider: "xai"`  | รองรับ                                                  |
| เสียงแบบเรียลไทม์             | Plugin Talk `talk.realtime.provider: "xai"`    | รองรับ; รีเลย์ผ่าน Gateway สำหรับโหนด Talk แบบเนทีฟ             |
| ไฟล์ / งานแบบชุด            | รองรับเฉพาะความเข้ากันได้กับ API โมเดลทั่วไป    | ไม่ใช่เครื่องมือ OpenClaw ระดับชั้นหนึ่ง                      |

<Note>
OpenClaw ใช้ API แบบ REST สำหรับภาพ/วิดีโอ/TTS/STT ของ xAI ในการสร้างสื่อและ
การถอดเสียงแบบชุด, WebSocket STT แบบสตรีมของ xAI สำหรับการถอดเสียงการโทรด้วยเสียง
แบบสด, WebSocket Grok Voice Agent ของ xAI สำหรับเซสชัน Talk แบบเรียลไทม์
และ Responses API สำหรับแชต การค้นหา และเครื่องมือเรียกใช้โค้ด
</Note>

### ความเข้ากันได้กับโหมดเร็วแบบเดิม

`/fast on` หรือ `agents.defaults.models["xai/<model>"].params.fastMode: true`
ยังคงเขียนการกำหนดค่า xAI รุ่นเก่าใหม่ดังต่อไปนี้ รหัสเป้าหมายเหล่านี้
เก็บไว้เพื่อความเข้ากันได้เท่านั้น สำหรับการกำหนดค่าใหม่ให้ใช้โมเดลปัจจุบันที่เลือกได้

| โมเดลต้นทาง  | เป้าหมายโหมดเร็ว   |
| ------------- | ------------------ |
| `grok-3`      | `grok-3-fast`      |
| `grok-3-mini` | `grok-3-mini-fast` |
| `grok-4`      | `grok-4-fast`      |
| `grok-4-0709` | `grok-4-fast`      |

### ความเข้ากันได้กับระบบเดิมและนามแฝงที่เปลี่ยนแปลง

นามแฝงรุ่นเก่าจะถูกปรับให้อยู่ในรูปแบบมาตรฐานดังนี้:

| นามแฝงรุ่นเก่า                                                  | รหัสที่ปรับเป็นมาตรฐาน    |
| ------------------------------------------------------------- | ---------------- |
| `grok-code-fast-1`, `grok-code-fast`, `grok-code-fast-1-0825` | `grok-build-0.1` |

รหัส 0309 ที่ระบุวันที่คือรายการในแค็ตตาล็อกที่เลือกได้ OpenClaw ส่งนามแฝง
Grok 4.20 ปัจจุบันอื่น ๆ ทั้งหมดตามเดิม เพื่อให้ xAI ยังคงควบคุมความหมายของนามแฝง
แบบเสถียร ล่าสุด เบตา ทดลอง และระบุวันที่ นามแฝงส่วนกลาง `grok-latest`
จะถูกเก็บไว้ตามเดิมเช่นกัน

xAI เลิกใช้รหัสต่อไปนี้โดยตรง OpenClaw เก็บรหัสเหล่านี้เป็นแถวความเข้ากันได้
ที่ซ่อนไว้สำหรับการกำหนดค่าที่เผยแพร่แล้ว พร้อมข้อจำกัดและราคาของเป้าหมาย
การเปลี่ยนเส้นทางปัจจุบัน:

| รหัสที่เลิกใช้                                                          | ลักษณะการทำงานปัจจุบัน                 |
| -------------------------------------------------------------------- | -------------------------------- |
| `grok-4-1-fast-reasoning`, `grok-4-fast-reasoning`, `grok-4-0709`    | Grok 4.3 พร้อมการให้เหตุผล `low`    |
| `grok-4-1-fast-non-reasoning`, `grok-4-fast-non-reasoning`, `grok-3` | Grok 4.3 โดยปิดการให้เหตุผล |
| `grok-code-fast-1`                                                   | Grok Build 0.1                   |
| `grok-imagine-image-pro`                                             | Grok Imagine Image Quality       |

`openclaw doctor --fix` อัปเดตค่าเริ่มต้นของเครื่องมือเซิร์ฟเวอร์ xAI ที่จัดเก็บถาวรและ
slug ภาพคุณภาพที่เลิกใช้ ลบแถวแค็ตตาล็อกที่สร้างขึ้นซึ่งล้าสมัย และซ่อมแซม
ข้อมูลเมตาบริบทที่ล้าสมัยในแถว 4.20 ที่ใช้งานอยู่ โดยจะไม่ตรึงนามแฝง

## คุณสมบัติ

<Warning>
  `x_search` และ `code_execution` ทำงานบนเซิร์ฟเวอร์ของ xAI โดย xAI คิดค่าบริการ $5 ต่อการเรียกใช้
  เครื่องมือ 1,000 ครั้ง บวกกับโทเค็นอินพุตและเอาต์พุตของโมเดล เมื่อไม่ได้ระบุการตั้งค่า
  `enabled` ของแต่ละเครื่องมือ OpenClaw จะแสดงเครื่องมือนั้นเฉพาะสำหรับโมเดล xAI ที่ใช้งานอยู่
  ผู้ให้บริการโมเดลที่ทราบว่าไม่ใช่ xAI ต้องระบุ `enabled: true` ต่อเครื่องมืออย่างชัดเจน
  หากผู้ให้บริการขาดหายไปหรือแก้ไขไม่ได้ ระบบจะปิดการทำงานโดยค่าเริ่มต้น ต้องมีการยืนยันตัวตนกับ xAI เสมอ
  และ `enabled: false` จะปิดใช้งานเครื่องมือสำหรับผู้ให้บริการทุกราย
</Warning>

    - การแก้ไข/ต่อวิดีโอจะรับช่วงอัตราส่วนภาพและความละเอียดของวิดีโออินพุต
      การดำเนินการเหล่านั้นไม่รองรับการกำหนดเรขาคณิตทับ
    - ระยะหมดเวลาการดำเนินการเริ่มต้น: 600 วินาที เว้นแต่จะตั้งค่า `video_generate.timeoutMs`
      หรือ `agents.defaults.videoGenerationModel.timeoutMs`

    <Warning>
    ไม่รองรับบัฟเฟอร์วิดีโอภายในเครื่อง ให้ใช้ URL `http(s)` ระยะไกลสำหรับอินพุต
    การแก้ไข/ต่อวิดีโอ การแปลงภาพเป็นวิดีโอรองรับบัฟเฟอร์ภาพภายในเครื่อง เนื่องจาก
    OpenClaw เข้ารหัสบัฟเฟอร์เหล่านั้นเป็น URL ข้อมูลสำหรับ xAI
    </Warning>

    Video 1.5 ยังรู้จักตัวระบุ `grok-imagine-video-1.5-preview` และ
    `grok-imagine-video-1.5-2026-05-30` ของ xAI ด้วย OpenClaw จะส่งต่อ
    ตัวระบุที่เลือกโดยไม่เปลี่ยนแปลง แต่ใช้การตรวจสอบที่อนุญาตเฉพาะภาพแบบเดียวกัน

    หากต้องการใช้ xAI เป็นผู้ให้บริการวิดีโอเริ่มต้น:

    ```json5
    {
      agents: {
        defaults: {
          videoGenerationModel: {
            primary: "xai/grok-imagine-video",
          },
        },
      },
    }
    ```

    <Note>
    ดู[การสร้างวิดีโอ](/th/tools/video-generation)สำหรับพารามิเตอร์เครื่องมือ
    ที่ใช้ร่วมกัน การเลือกผู้ให้บริการ และลักษณะการทำงานเมื่อสลับไปใช้ระบบสำรอง
    </Note>

  </Accordion>

  <Accordion title="การสร้างภาพ">
    Plugin `xai` แบบรวมชุดจะลงทะเบียนการสร้างภาพผ่านเครื่องมือ
    `image_generate` ที่ใช้ร่วมกัน

    - โมเดลรูปภาพเริ่มต้น: `xai/grok-imagine-image`
    - โมเดลเพิ่มเติม: `xai/grok-imagine-image-quality`
    - โหมด: แปลงข้อความเป็นรูปภาพและแก้ไขรูปภาพอ้างอิง
    - อินพุตอ้างอิง: `image` หนึ่งรายการ หรือ `images` สูงสุดสามรายการ
    - อัตราส่วนภาพ: `1:1`, `16:9`, `9:16`, `4:3`, `3:4`, `3:2`, `2:3`, `2:1`,
      `1:2`, `19.5:9`, `9:19.5`, `20:9`, `9:20`
    - ความละเอียด: `1K`, `2K`
    - จำนวน: สูงสุด 4 รูปภาพ
    - ระยะหมดเวลาการดำเนินการเริ่มต้น: 600 วินาที เว้นแต่จะตั้งค่า `image_generate.timeoutMs`
      หรือ `agents.defaults.imageGenerationModel.timeoutMs`

    OpenClaw ขอให้ xAI ส่งการตอบกลับรูปภาพแบบ `b64_json` เพื่อให้สามารถจัดเก็บ
    และส่งสื่อที่สร้างขึ้นผ่านเส้นทางไฟล์แนบตามปกติของช่องทาง รูปภาพอ้างอิง
    ภายในเครื่องจะถูกแปลงเป็น URL ข้อมูล ส่วนการอ้างอิง `http(s)` ระยะไกล
    จะถูกส่งต่อโดยไม่เปลี่ยนแปลง

    หากต้องการใช้ xAI เป็นผู้ให้บริการรูปภาพเริ่มต้น:

    ```json5
    {
      agents: {
        defaults: {
          imageGenerationModel: {
            primary: "xai/grok-imagine-image",
          },
        },
      },
    }
    ```

    <Note>
    xAI ยังระบุ `quality`, `mask`, `user` และอัตราส่วนภาพ `auto`
    ปัจจุบัน OpenClaw ส่งต่อเฉพาะตัวควบคุมรูปภาพที่ใช้ร่วมกันระหว่างผู้ให้บริการ
    เท่านั้น ตัวเลือกเฉพาะระบบเหล่านี้ยังไม่เปิดให้ใช้ผ่าน `image_generate`
    </Note>

  <Accordion title="การแปลงข้อความเป็นเสียงพูด">
    Plugin `xai` ที่ให้มาพร้อมระบบลงทะเบียนการแปลงข้อความเป็นเสียงพูดผ่านพื้นผิว
    ผู้ให้บริการ `tts` ที่ใช้ร่วมกัน

    - เสียง: แค็ตตาล็อกแบบสดที่ผ่านการยืนยันตัวตนจาก xAI; แสดงรายการด้วย
      `openclaw infer tts voices --provider xai`
    - เสียงสำรองแบบออฟไลน์: `ara`, `eve`, `leo`, `rex`, `sal`
    - เสียงเริ่มต้น: `eve`
    - ระบบจะส่งต่อรหัสเสียงแบบกำหนดเองของบัญชี แม้ว่ารหัสดังกล่าวจะไม่มีอยู่ใน
      การตอบกลับของแค็ตตาล็อกในตัว
    - รูปแบบ: `mp3`, `wav`, `pcm`, `mulaw`, `alaw`
    - ภาษา: รหัส BCP-47 หรือ `auto`
    - ความเร็ว: การแทนที่ความเร็วแบบเนทีฟของผู้ให้บริการ
    - ไม่รองรับรูปแบบบันทึกเสียง Opus แบบเนทีฟ

    หากต้องการใช้ xAI เป็นผู้ให้บริการ TTS เริ่มต้น:

    ```json5
    {
      messages: {
        tts: {
          provider: "xai",
          providers: {
            xai: {
              voiceId: "eve",
            },
          },
        },
      },
    }
    ```

    <Note>
    OpenClaw ใช้ปลายทาง `/v1/tts` แบบแบตช์ของ xAI สำหรับการสังเคราะห์แบบบัฟเฟอร์
    ใช้การค้นพบแค็ตตาล็อก `/v1/tts/voices` ที่ผ่านการยืนยันตัวตน และใช้
    `wss://api.x.ai/v1/tts` แบบเนทีฟสำหรับการสังเคราะห์แบบสตรีม การสตรีมจำกัดไว้เฉพาะ
    โฮสต์ `api.x.ai` แบบเนทีฟ ดังนั้นค่า `baseUrl` แบบกำหนดเองจะถูกปฏิเสธใน
    เส้นทางนี้ ระบบใช้ตัวควบคุมภาษา เสียง ตัวแปลงสัญญาณ และความเร็วที่มีอยู่ โดยใช้
    ค่าเริ่มต้นของ xAI สำหรับอัตราการสุ่มตัวอย่างและอัตราบิต การสังเคราะห์ไฟล์เสียงใช้
    ตัวแปลงสัญญาณทั้งหมดที่กำหนดค่าไว้ เป้าหมายบันทึกเสียงใช้ MP3 สำหรับการสตรีมและการ
    สำรองแบบบัฟเฟอร์ เนื่องจากตัวแปลงสัญญาณดิบของ xAI ไม่มีข้อมูลเมตาของตัวแปลงสัญญาณ/อัตรา
    สตรีมจะส่ง `text.delta` แล้วตามด้วย
    `text.done` รับ `audio.delta`, `audio.done` หรือ `error` และใช้
    `timeoutMs` เมื่อไม่มีการใช้งาน ซึ่งจะรีเฟรชเมื่อได้รับเสียงแต่ละส่วน การทำงานนี้
    แยกจากเซสชันเสียงแบบเรียลไทม์ ดูสัญญา [API TTS แบบสตรีม](https://docs.x.ai/developers/rest-api-reference/inference/voice) ของ xAI
    </Note>

  </Accordion>

  <Accordion title="การแปลงเสียงพูดเป็นข้อความ">

    - เส้นทางอินพุต: การอัปโหลดไฟล์เสียงแบบ multipart
    - การเลือกโมเดล: xAI เลือกโมเดลการถอดเสียงภายใน ปลายทางนี้
      ไม่มีตัวเลือกโมเดล
    - ใช้ทุกตำแหน่งที่การถอดเสียงขาเข้าอ่าน `tools.media.audio`

    ```json5
    {
      tools: {
        media: {
          audio: {
            models: [
              {
                type: "provider",
                provider: "xai",
              },
            ],
          },
        },
      },
    }
    ```

    สามารถระบุภาษาผ่านการกำหนดค่าสื่อเสียงที่ใช้ร่วมกันหรือคำขอถอดเสียง
    แต่ละครั้ง พื้นผิว OpenClaw ที่ใช้ร่วมกันยอมรับคำใบ้พรอมต์ แต่การผสานรวม
    STT แบบ REST ของ xAI จะส่งต่อเฉพาะไฟล์และภาษา เนื่องจากมีเพียงสองรายการนี้
    ที่ตรงกับปลายทางสาธารณะปัจจุบันของ xAI

  </Accordion>

  <Accordion title="การแปลงเสียงพูดเป็นข้อความแบบสตรีม">
    Plugin `xai` ที่ให้มาพร้อมระบบยังลงทะเบียนผู้ให้บริการถอดเสียงแบบเรียลไทม์
    สำหรับเสียงการโทรสดด้วย

    - ปลายทาง: WebSocket `wss://api.x.ai/v1/stt` ของ xAI
    - การเข้ารหัสเริ่มต้น: `mulaw`
    - อัตราการสุ่มตัวอย่างเริ่มต้น: `8000`
    - การตรวจจับจุดสิ้นสุดเริ่มต้น: `800ms`
    - ข้อความถอดเสียงชั่วคราว: เปิดใช้งานโดยค่าเริ่มต้น

    สตรีมสื่อ Twilio ของ Voice Call ส่งเฟรมเสียง G.711 mu-law ดังนั้น
    ผู้ให้บริการ xAI จึงส่งต่อเฟรมเหล่านั้นโดยตรงโดยไม่แปลงรหัส:

    ```json5
    {
      plugins: {
        entries: {
          "voice-call": {
            config: {
              streaming: {
                enabled: true,
                provider: "xai",
                providers: {
                  xai: {
                    apiKey: "${XAI_API_KEY}",
                    endpointingMs: 800,
                    language: "en",
                  },
                },
              },
            },
          },
        },
      },
    }
    ```

    การกำหนดค่าที่ผู้ให้บริการเป็นเจ้าของอยู่ภายใต้
    `plugins.entries.voice-call.config.streaming.providers.xai` คีย์ที่รองรับ
    ได้แก่ `apiKey`, `baseUrl`, `sampleRate`, `encoding` (`pcm`, `mulaw` หรือ
    `alaw`), `interimResults`, `endpointingMs` และ `language`

    <Note>
    ผู้ให้บริการสตรีมนี้ใช้สำหรับเส้นทางการถอดเสียงแบบเรียลไทม์ของ Voice Call
    Discord บันทึกเสียงเป็นช่วงสั้นๆ และใช้เส้นทางการถอดเสียง
    `tools.media.audio` แบบแบตช์แทน
    </Note>

  </Accordion>

  <Accordion title="เสียงแบบเรียลไทม์ (Talk)">
    Plugin `xai` ที่ให้มาพร้อมระบบลงทะเบียนเซสชัน Grok Voice Agent แบบเรียลไทม์สำหรับ
    โหมด Talk ผ่านสัญญา `registerRealtimeVoiceProvider` ที่ใช้ร่วมกัน

    - ปลายทาง: `wss://api.x.ai/v1/realtime?model=<voice-model>`
    - โมเดลเริ่มต้น: `grok-voice-latest`
    - เสียงเริ่มต้น: `eve`
    - การรับส่งข้อมูล: `gateway-relay` (เส้นทางรีเลย์ของ iOS, Android และ Control UI)
    - เสียง: PCM16 24 kHz หรือ G.711 µ-law 8 kHz
    - การพูดแทรก: VAD ของเซิร์ฟเวอร์ xAI ขัดจังหวะการตอบกลับ OpenClaw จะล้างการเล่นที่อยู่ในคิว
      และตัดประวัติของผู้ให้บริการส่วนที่ยังไม่ได้เล่นออก

    กำหนดค่า Talk บน Gateway:

    ```json5
    {
      talk: {
        realtime: {
          provider: "xai",
          mode: "realtime",
          transport: "gateway-relay",
          brain: "agent-consult",
          providers: {
            xai: {
              model: "grok-voice-latest",
              voice: "eve",
              // เลือกใช้เฉพาะเมื่อยอมรับการเล่นเซสชันซ้ำโดยผู้ให้บริการได้
              sessionResumption: false,
            },
          },
        },
      },
      env: { XAI_API_KEY: "xai-..." },
    }
    ```

    การกำหนดค่าที่ผู้ให้บริการเป็นเจ้าของยังแก้ไขค่าจาก
    `plugins.entries.voice-call.config.realtime.providers.xai` เมื่อ Voice Call
    หรือตัวเลือกแบบเรียลไทม์ที่ใช้ร่วมกันนำแผนที่ผู้ให้บริการเดียวกันมาใช้ซ้ำ คีย์ที่รองรับได้แก่
    `apiKey`, `baseUrl`, `model`, `voice`, `vadThreshold`, `silenceDurationMs`,
    `prefixPaddingMs`, `reasoningEffort` และ `sessionResumption`
    `reasoningEffort` ยอมรับเฉพาะ `high` หรือ `none` ซึ่งตรงกับ API Voice Agent ของ xAI

    VAD ของเซิร์ฟเวอร์ xAI จะสร้างการตอบกลับและจัดการการขัดจังหวะเสียงเสมอ
    ใช้ `consultRouting: "provider-direct"` ไม่รองรับการบังคับกำหนดเส้นทางข้อความถอดเสียงและการปิดใช้งาน
    การขัดจังหวะเสียงอินพุตในโปรโตคอล Voice Agent ของ xAI

    <Note>
    OAuth ของ xAI หรือ `XAI_API_KEY` สามารถยืนยันตัวตนสำหรับเสียงแบบเรียลไทม์ได้ WebRTC
    ที่เบราว์เซอร์เป็นเจ้าของยังไม่เป็นส่วนหนึ่งของพื้นผิวผู้ให้บริการนี้ ให้ใช้ Talk แบบ gateway-relay บน
    Node แบบเนทีฟหรือเส้นทางรีเลย์ของ Control UI
    </Note>

    <Note>
    `sessionResumption` มีค่าเริ่มต้นเป็น `false` เมื่อตั้งค่าเป็น `true` OpenClaw จะขอให้
    xAI เก็บสถานะเซสชันไว้เพียงพอสำหรับกลับมาสนทนาเดิมต่อหลัง
    เชื่อมต่อใหม่ จากนั้นจึงเชื่อมต่อใหม่ด้วยรหัสการสนทนาที่ส่งกลับมา ให้ปิดใช้งานไว้
    เมื่อไม่สามารถยอมรับการเล่นซ้ำ/การเก็บรักษาข้อมูลโดยผู้ให้บริการได้ ในกรณีนั้นซ็อกเก็ตที่ถูกขัดจังหวะ
    จะหยุดทำงานแบบปิดกั้น แทนที่จะเริ่มการสนทนาใหม่โดยไม่แจ้งให้ทราบ
    </Note>

  </Accordion>

  <Accordion title="การกำหนดค่า x_search">

    | `baseUrl`         | string  | -                         | การแทนที่ URL ฐาน Responses ของ xAI                  |
    | `inlineCitations` | boolean | -                         | รวมการอ้างอิงแบบอินไลน์ในผลลัพธ์              |
    | `maxTurns`        | number  | -                         | จำนวนรอบการสนทนาสูงสุด                       |
    | `timeoutSeconds`  | number  | `30`                      | ระยะหมดเวลาของคำขอเป็นวินาที                       |
    | `cacheTtlMinutes` | number  | `15`                      | อายุแคชเป็นนาที                    |

    ```json5
    {
      plugins: {
        entries: {
          xai: {
            config: {
              xSearch: {
                enabled: true,
                model: "grok-4.3",
                baseUrl: "https://api.x.ai/v1",
                inlineCitations: true,
              },
            },
          },
        },
      },
    }
    ```

  </Accordion>

  <Accordion title="การกำหนดค่าการเรียกใช้โค้ด">
    Plugin xAI ที่ให้มาพร้อมระบบเปิดให้ใช้ `code_execution` เป็นเครื่องมือ OpenClaw สำหรับ
    การเรียกใช้โค้ดระยะไกลในสภาพแวดล้อมแซนด์บ็อกซ์ของ xAI

    เส้นทางการกำหนดค่า: `plugins.entries.xai.config.codeExecution`

    | คีย์              | ชนิด    | ค่าเริ่มต้น                  | คำอธิบาย                                      |
    | ---------------- | ------- | ------------------------ | ------------------------------------------------ |
    | `enabled`        | boolean | อัตโนมัติสำหรับโมเดล xAI | ปิดใช้งาน หรือเลือกใช้สำหรับผู้ให้บริการที่ทราบว่าไม่ใช่ xAI |
    | `model`          | string  | `grok-4.3`               | โมเดลที่ใช้สำหรับคำขอเรียกใช้โค้ด           |
    | `maxTurns`       | number  | -                        | จำนวนรอบการสนทนาสูงสุด                       |
    | `timeoutSeconds` | number  | `30`                     | ระยะหมดเวลาของคำขอเป็นวินาที                       |

    <Note>
    นี่คือการเรียกใช้ในแซนด์บ็อกซ์ xAI ระยะไกล ไม่ใช่ [`exec`](/th/tools/exec) ภายในเครื่อง
    </Note>

    ```json5
    {
      plugins: {
        entries: {
          xai: {
            config: {
              codeExecution: {
                enabled: true,
                model: "grok-4.3",
              },
            },
          },
        },
      },
    }
    ```

  </Accordion>

  <Accordion title="ข้อจำกัดที่ทราบ">
    - การตรวจสอบสิทธิ์ xAI สามารถใช้คีย์ API, ตัวแปรสภาพแวดล้อม, การกำหนดค่า Plugin
      สำรอง หรือ OAuth ด้วยบัญชี xAI ที่มีสิทธิ์ OAuth ใช้การยืนยัน
      ด้วยรหัสอุปกรณ์โดยไม่มีการเรียกกลับไปยัง localhost xAI เป็นผู้กำหนดว่าบัญชีใด
      สามารถรับโทเค็น API ของ OAuth ได้ และหน้าความยินยอมอาจแสดง Grok Build
      แม้ว่า OpenClaw จะไม่จำเป็นต้องใช้แอป Grok Build
    - ปัจจุบัน OpenClaw ยังไม่เปิดให้ใช้ตระกูลโมเดลหลายเอเจนต์ของ xAI โดย xAI
      ให้บริการโมเดลเหล่านี้ผ่าน Responses API แต่โมเดลไม่รองรับ
      เครื่องมือฝั่งไคลเอนต์หรือเครื่องมือแบบกำหนดเองที่ลูปเอเจนต์ร่วมของ OpenClaw ใช้
      ดู
      [ข้อจำกัดของระบบหลายเอเจนต์ xAI](https://docs.x.ai/developers/model-capabilities/text/multi-agent#limitations)
    - ขณะนี้เสียงแบบเรียลไทม์ของ xAI เปิดให้ใช้เฉพาะการขนส่ง Talk ผ่านรีเลย์ของ Gateway
      เซสชัน WebSocket ของผู้ให้บริการที่เบราว์เซอร์เป็นเจ้าของยังไม่ได้เชื่อมต่อ
      ใน Control UI
    - รูปภาพ xAI `quality`, รูปภาพ `mask` และอัตราส่วนภาพเพิ่มเติมที่มีเฉพาะในระบบเนทีฟ
      จะยังไม่เปิดให้ใช้จนกว่าเครื่องมือ `image_generate` ที่ใช้ร่วมกันจะมี
      ตัวควบคุมข้ามผู้ให้บริการที่สอดคล้องกัน
  </Accordion>

  <Accordion title="หมายเหตุขั้นสูง">
    - OpenClaw ใช้การแก้ไขความเข้ากันได้ของสคีมาเครื่องมือและการเรียกใช้เครื่องมือ
      สำหรับ xAI โดยอัตโนมัติบนเส้นทางรันเนอร์ร่วม
    - คำขอ xAI แบบเนทีฟใช้ค่าเริ่มต้นเป็น `tool_stream: true` ตั้งค่า
      `agents.defaults.models["xai/<model>"].params.tool_stream` เป็น `false`
      เพื่อปิดใช้งาน
    - ตัวห่อหุ้ม xAI ที่รวมมาให้จะตัดขอบเขตจำนวน contains ที่ไม่รองรับ
      และคีย์เพย์โหลด *effort* สำหรับการให้เหตุผลที่ไม่รองรับก่อนส่งคำขอ
      xAI แบบเนทีฟ Grok 4.5 รองรับระดับ effort ต่ำ ปานกลาง และ
      สูง (ค่าเริ่มต้นคือสูง) Grok 4.3 รองรับไม่มี ต่ำ ปานกลาง และสูง
      (ค่าเริ่มต้นคือต่ำ) โมเดล xAI อื่นที่รองรับการให้เหตุผลไม่มี
      ตัวควบคุม effort ที่กำหนดค่าได้ แต่ยังคงร้องขอ
      `include: ["reasoning.encrypted_content"]` เพื่อให้สามารถนำการให้เหตุผลที่เข้ารหัสไว้ก่อนหน้า
      มาเล่นซ้ำในการโต้ตอบครั้งถัดไป
    - `web_search`, `x_search` และ `code_execution` เปิดให้ใช้เป็นเครื่องมือของ OpenClaw
      OpenClaw จะแนบเฉพาะเครื่องมือในตัวของ xAI ที่แต่ละเครื่องมือต้องใช้
      ไปกับคำขอของเครื่องมือนั้น แทนที่จะแนบเครื่องมือเนทีฟทั้งหมดไปกับ
      การโต้ตอบแชตทุกครั้ง
    - Grok `web_search` อ่าน `plugins.entries.xai.config.webSearch.baseUrl`
      `x_search` อ่าน `plugins.entries.xai.config.xSearch.baseUrl` จากนั้น
      จึงใช้ URL ฐานการค้นหาเว็บของ Grok เป็นทางเลือกสำรอง
    - `x_search` และ `code_execution` อยู่ภายใต้การดูแลของ Plugin xAI ที่รวมมาให้
      แทนที่จะถูกกำหนดตายตัวไว้ในรันไทม์โมเดลแกนหลัก
    - `code_execution` คือการเรียกใช้ในแซนด์บ็อกซ์ xAI ระยะไกล ไม่ใช่
      [`exec`](/th/tools/exec) ภายในเครื่อง
  </Accordion>
</AccordionGroup>

## การทดสอบแบบสด

เส้นทางสื่อของ xAI ครอบคลุมด้วยการทดสอบหน่วยและชุดทดสอบแบบสดที่ต้องเลือกเปิดใช้ ส่งออก
`XAI_API_KEY` ในสภาพแวดล้อมของกระบวนการก่อนเรียกใช้โพรบแบบสด

```bash
pnpm test extensions/xai
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_TEST_QUIET=1 pnpm test:live -- extensions/xai/xai.live.test.ts
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_XAI_VIDEO=1 pnpm test:live -- extensions/xai/xai.live.test.ts -t "classic Grok Imagine"
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_XAI_VIDEO=1 pnpm test:live -- extensions/xai/xai.live.test.ts -t "Grok Imagine Video 1.5"
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_TEST_QUIET=1 pnpm test:live -- extensions/xai/x-search.live.test.ts
OPENCLAW_LIVE_GATEWAY_MODELS="xai/grok-4.5,xai/grok-build-0.1,xai/grok-4.3,xai/grok-4.20-0309-reasoning,xai/grok-4.20-0309-non-reasoning" OPENCLAW_LIVE_GATEWAY_MAX_MODELS=0 OPENCLAW_LIVE_GATEWAY_SMOKE=0 pnpm test:live -- src/gateway/gateway-models.profiles.live.test.ts
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_TEST_QUIET=1 OPENCLAW_LIVE_IMAGE_GENERATION_PROVIDERS=xai pnpm test:live -- test/image-generation.runtime.live.test.ts
```

ไฟล์ทดสอบแบบสดเฉพาะผู้ให้บริการจะสังเคราะห์ TTS ปกติ, TTS แบบ PCM
ที่เหมาะสำหรับระบบโทรศัพท์, ถอดเสียงผ่าน STT แบบแบตช์ของ xAI, สตรีม PCM เดียวกันผ่าน STT
แบบเรียลไทม์ของ xAI, สร้างผลลัพธ์จากข้อความเป็นรูปภาพ และแก้ไขรูปภาพอ้างอิง
ไฟล์ทดสอบรูปภาพแบบสดที่ใช้ร่วมกันจะตรวจสอบผู้ให้บริการ xAI รายเดียวกันผ่าน
การเลือกรันไทม์ ทางเลือกสำรอง การปรับให้อยู่ในรูปแบบมาตรฐาน และเส้นทางแนบสื่อของ OpenClaw กรณี
Video 1.5 ที่ต้องเลือกเปิดใช้จะส่งรูปภาพเฟรมแรกที่สร้างขึ้นหนึ่งภาพที่ 1080P และ
ตรวจสอบการดาวน์โหลดวิดีโอที่เสร็จสมบูรณ์
