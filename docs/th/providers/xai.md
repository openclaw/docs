OpenClaw มาพร้อม Plugin ผู้ให้บริการ `xai` แบบรวมในตัวสำหรับโมเดล Grok แนวทางที่
แนะนำคือ Grok OAuth พร้อมการสมัครสมาชิก SuperGrok หรือ X Premium
ที่มีสิทธิ์ใช้งาน Gateway การกำหนดค่า การกำหนดเส้นทาง และเครื่องมือยังคงทำงานภายในเครื่อง มีเพียงคำขอ Grok
เท่านั้นที่ส่งไปยัง API ของ xAI

OAuth ไม่ต้องใช้คีย์ API ของ xAI หรือแอป Grok Build แต่ xAI อาจยังคง
แสดง Grok Build บนหน้าจอให้ความยินยอม เนื่องจาก OpenClaw ใช้ไคลเอ็นต์
OAuth ที่ใช้ร่วมกันของ xAI

## การตั้งค่า

<Steps>
  <Step title="การติดตั้งใหม่">
    เรียกใช้การเริ่มต้นใช้งานพร้อมติดตั้งดีมอน จากนั้นเลือก xAI/Grok OAuth ในขั้นตอน
    โมเดล/การยืนยันตัวตน:

    ```bash
    openclaw onboard --install-daemon
    ```

    บน VPS หรือผ่าน SSH ให้เลือก xAI OAuth โดยตรง ซึ่งใช้การยืนยัน
    ด้วยรหัสอุปกรณ์และไม่ต้องใช้คอลแบ็ก localhost:

    ```bash
    openclaw onboard --install-daemon --auth-choice xai-oauth
    ```

  </Step>
  <Step title="การติดตั้งที่มีอยู่">
    ลงชื่อเข้าใช้ xAI เท่านั้น อย่าเรียกใช้การเริ่มต้นใช้งานแบบเต็มอีกครั้งเพียงเพื่อเชื่อมต่อ Grok:

    ```bash
    openclaw models auth login --provider xai --method oauth
    ```

    กำหนด Grok เป็นโมเดลเริ่มต้นแยกต่างหาก:

    ```bash
    openclaw models set xai/grok-4.3
    ```

    เรียกใช้การเริ่มต้นใช้งานแบบเต็มอีกครั้งเฉพาะเมื่อตั้งใจจะเปลี่ยน Gateway,
    ดีมอน ช่องทาง พื้นที่ทำงาน หรือตัวเลือกการตั้งค่าอื่นๆ

  </Step>
  <Step title="แนวทางใช้คีย์ API">
    การตั้งค่าด้วยคีย์ API ยังคงใช้ได้กับคีย์จาก xAI Console และพื้นผิวสื่อ
    ที่ต้องใช้การกำหนดค่าผู้ให้บริการซึ่งอ้างอิงคีย์:

    ```bash
    openclaw models auth login --provider xai --method api-key
    export XAI_API_KEY=xai-...
    ```

  </Step>
  <Step title="เลือกโมเดล">
    ```json5
    {
      agents: { defaults: { model: { primary: "xai/grok-4.3" } } },
    }
    ```
  </Step>
</Steps>

<Note>
OpenClaw ใช้ xAI Responses API เป็นกลไกขนส่ง xAI แบบรวมในตัว ข้อมูลรับรองเดียวกันจาก
`openclaw models auth login --provider xai --method oauth` หรือ
`--method api-key` ยังขับเคลื่อน `web_search` (รหัสผู้ให้บริการ `grok`), `x_search`,
`code_execution`, เสียงพูด/การถอดเสียง และการสร้างรูปภาพ/วิดีโอด้วย xAI หาก
จัดเก็บคีย์ xAI ไว้ภายใต้ `plugins.entries.xai.config.webSearch.apiKey`
ผู้ให้บริการโมเดล xAI แบบรวมในตัวจะนำคีย์นั้นกลับมาใช้เป็นทางเลือกสำรองด้วย
</Note>

## การแก้ไขปัญหา OAuth

- สำหรับ SSH, Docker, VPS หรือการตั้งค่าระยะไกลอื่นๆ ให้ใช้
  `openclaw models auth login --provider xai --method oauth` ซึ่งใช้
  การยืนยันด้วยรหัสอุปกรณ์ ไม่ใช่คอลแบ็ก localhost
- หากลงชื่อเข้าใช้สำเร็จแต่ Grok ไม่ใช่โมเดลเริ่มต้น ให้เรียกใช้
  `openclaw models set xai/grok-4.3`
- ตรวจสอบโปรไฟล์การยืนยันตัวตน xAI ที่บันทึกไว้:

  ```bash
  openclaw models auth list --provider xai
  openclaw models status
  ```

- xAI เป็นผู้กำหนดว่าบัญชีใดสามารถรับโทเค็น API ผ่าน OAuth ได้ หากบัญชี
  ไม่มีสิทธิ์ ให้ใช้แนวทางคีย์ API หรือตรวจสอบการสมัครสมาชิกทางฝั่ง xAI

<Tip>
ใช้ `xai-oauth` เมื่อลงชื่อเข้าใช้จาก SSH, Docker หรือ VPS OpenClaw จะแสดง
URL และรหัสสั้นๆ ให้ลงชื่อเข้าใช้ให้เสร็จในเบราว์เซอร์ภายในเครื่องใดก็ได้ ขณะที่กระบวนการ
ระยะไกลสำรวจ xAI เพื่อรอการแลกเปลี่ยนโทเค็นให้เสร็จสมบูรณ์
</Tip>

## แค็ตตาล็อกในตัว

รหัสที่เลือกได้ในตัวเลือกโมเดล Plugin ยังคงตีความรหัส Grok 3,
Grok 4, Grok 4 Fast, Grok 4.1 Fast และ Grok Code รุ่นเก่าสำหรับการกำหนดค่าที่มีอยู่
ดู[ความเข้ากันได้กับรุ่นเก่าและนามแฝงที่เปลี่ยนตำแหน่ง](#legacy-compatibility-and-moving-aliases)

| ตระกูล         | รหัสโมเดล                                                    |
| -------------- | ------------------------------------------------------------ |
| Grok 4.5       | `grok-4.5` (นามแฝง: `grok-4.5-latest`, `grok-build-latest`) |
| Grok Build 0.1 | `grok-build-0.1`                                             |
| Grok 4.3       | `grok-4.3` (นามแฝง: `grok-4.3-latest`, `grok-latest`)       |
| Grok 4.20      | `grok-4.20-0309-reasoning`, `grok-4.20-0309-non-reasoning`   |

<Tip>
ใช้ `grok-4.5` สำหรับการแชตทั่วไป การเขียนโค้ด และงานแบบเอเจนต์ในพื้นที่ที่พร้อมใช้งาน
Grok 4.3 ยังคงเป็นค่าเริ่มต้นสำหรับการตั้งค่าที่ปลอดภัยในแต่ละภูมิภาค ส่วน `grok-build-0.1` และ
Grok 4.20 ทั้งสองรุ่นที่ระบุวันที่ยังคงเลือกได้
</Tip>

ข้อมูลบริบทและต้นทุนโทเค็นในแค็ตตาล็อกเป็นไปตาม
[หน้าโมเดล](https://docs.x.ai/developers/models) และ
[หน้าราคา](https://docs.x.ai/developers/pricing) แบบสดของ xAI โดย xAI ใช้อัตราที่สูงขึ้น
เมื่อคำขอเกินเกณฑ์บริบทยาวที่ระบุไว้ ช่องข้อมูลต้นทุนแบบคงที่ในแค็ตตาล็อกของ OpenClaw
บันทึกอัตราสำหรับบริบทสั้น Grok Build ซึ่งเป็น CLI เอเจนต์เขียนโค้ดแยกต่างหากของ xAI
พร้อมใช้งานที่ [x.ai/cli](https://x.ai/cli) และปัจจุบัน
ใช้ Grok 4.5

## ความครอบคลุมของฟีเจอร์

Plugin แบบรวมในตัวแมป API ของ xAI ที่รองรับเข้ากับสัญญาผู้ให้บริการและ
เครื่องมือที่ใช้ร่วมกันของ OpenClaw ความสามารถที่ไม่สอดคล้องกับสัญญาที่ใช้ร่วมกัน
แสดงรายการไว้ด้านล่างหรือในข้อจำกัดที่ทราบ

| ความสามารถของ xAI             | พื้นผิว OpenClaw                        | สถานะ                                               |
| -------------------------- | --------------------------------------- | ---------------------------------------------------- |
| แชต / Responses           | ผู้ให้บริการโมเดล `xai/<model>`            | รองรับ                                                  |
| การค้นหาเว็บฝั่งเซิร์ฟเวอร์     | ผู้ให้บริการ `web_search` `grok`            | รองรับ                                                  |
| การค้นหา X ฝั่งเซิร์ฟเวอร์       | เครื่องมือ `x_search`                         | รองรับ                                                  |
| การเรียกใช้โค้ดฝั่งเซิร์ฟเวอร์ | เครื่องมือ `code_execution`                   | รองรับ                                                  |
| รูปภาพ                     | `image_generate`                        | รองรับ                                                  |
| วิดีโอ                     | `video_generate`                        | รองรับ                                                  |
| การแปลงข้อความเป็นเสียงแบบกลุ่ม       | `messages.tts.provider: "xai"` / `tts`  | รองรับ                                                  |
| TTS แบบสตรีม              | `textToSpeechStream`                    | รองรับผ่าน `wss://api.x.ai/v1/tts` (ไม่ใช่เสียงแบบเรียลไทม์) |
| การแปลงเสียงเป็นข้อความแบบกลุ่ม       | การทำความเข้าใจสื่อ `tools.media.audio` | รองรับ                                                  |
| การแปลงเสียงเป็นข้อความแบบสตรีม   | Voice Call `streaming.provider: "xai"`  | รองรับ                                                  |
| เสียงแบบเรียลไทม์             | Talk `talk.realtime.provider: "xai"`    | รองรับ; ถ่ายทอดผ่าน Gateway สำหรับโหนด Talk แบบเนทีฟ             |
| ไฟล์ / งานแบบกลุ่ม            | ความเข้ากันได้กับ API โมเดลทั่วไปเท่านั้น    | ไม่ใช่เครื่องมือ OpenClaw ระดับชั้นหนึ่ง                      |

<Note>
OpenClaw ใช้ REST API ด้านรูปภาพ/วิดีโอ/TTS/STT ของ xAI สำหรับการสร้างสื่อและ
การถอดเสียงแบบกลุ่ม ใช้ WebSocket STT แบบสตรีมของ xAI สำหรับการถอดเสียงการโทรด้วยเสียง
แบบสด ใช้ WebSocket ของ Grok Voice Agent ของ xAI สำหรับเซสชัน Talk แบบเรียลไทม์
และใช้ Responses API สำหรับแชต การค้นหา และเครื่องมือเรียกใช้โค้ด
</Note>

### ความเข้ากันได้กับโหมดเร็วรุ่นเก่า

`/fast on` หรือ `agents.defaults.models["xai/<model>"].params.fastMode: true`
ยังคงเขียนการกำหนดค่า xAI รุ่นเก่าใหม่ดังต่อไปนี้ รหัสเป้าหมายเหล่านี้
เก็บไว้เพื่อความเข้ากันได้เท่านั้น สำหรับการกำหนดค่าใหม่ให้ใช้โมเดลปัจจุบัน
ที่เลือกได้

| โมเดลต้นทาง  | เป้าหมายโหมดเร็ว   |
| ------------- | ------------------ |
| `grok-3`      | `grok-3-fast`      |
| `grok-3-mini` | `grok-3-mini-fast` |
| `grok-4`      | `grok-4-fast`      |
| `grok-4-0709` | `grok-4-fast`      |

### ความเข้ากันได้กับรุ่นเก่าและนามแฝงที่เปลี่ยนตำแหน่ง

นามแฝงรุ่นเก่าจะถูกปรับให้อยู่ในรูปแบบมาตรฐานดังนี้:

| นามแฝงรุ่นเก่า                                                  | รหัสที่ปรับเป็นมาตรฐาน    |
| ------------------------------------------------------------- | ---------------- |
| `grok-code-fast-1`, `grok-code-fast`, `grok-code-fast-1-0825` | `grok-build-0.1` |

รหัส 0309 ที่ระบุวันที่คือรายการในแค็ตตาล็อกที่เลือกได้ OpenClaw ส่งนามแฝง
Grok 4.20 ปัจจุบันอื่นๆ ทั้งหมดตามข้อความเดิม เพื่อให้ xAI ยังคงควบคุมความหมายของ
นามแฝงแบบเสถียร ล่าสุด เบตา ทดลอง และระบุวันที่ นามแฝงสากล `grok-latest`
จะถูกเก็บไว้ตามข้อความเดิมเช่นกัน

xAI เลิกใช้รหัสที่ตรงกันต่อไปนี้ OpenClaw เก็บรหัสเหล่านี้เป็นแถวความเข้ากันได้
ที่ซ่อนไว้สำหรับการกำหนดค่าที่เผยแพร่แล้ว โดยใช้ข้อจำกัดและราคาของเป้าหมาย
การเปลี่ยนเส้นทางปัจจุบัน:

| รหัสที่เลิกใช้                                                          | ลักษณะการทำงานปัจจุบัน                 |
| -------------------------------------------------------------------- | -------------------------------- |
| `grok-4-1-fast-reasoning`, `grok-4-fast-reasoning`, `grok-4-0709`    | Grok 4.3 พร้อมการให้เหตุผล `low`    |
| `grok-4-1-fast-non-reasoning`, `grok-4-fast-non-reasoning`, `grok-3` | Grok 4.3 ที่ปิดใช้การให้เหตุผล |
| `grok-code-fast-1`                                                   | Grok Build 0.1                   |
| `grok-imagine-image-pro`                                             | คุณภาพรูปภาพ Grok Imagine       |

`openclaw doctor --fix` อัปเดตค่าเริ่มต้นของเครื่องมือเซิร์ฟเวอร์ xAI ที่จัดเก็บถาวรและ
slug รูปภาพคุณภาพที่เลิกใช้ ลบแถวแค็ตตาล็อกที่สร้างขึ้นซึ่งล้าสมัย และซ่อมแซม
ข้อมูลเมตาบริบทที่ล้าสมัยในแถว 4.20 ที่ใช้งานอยู่ โดยไม่ตรึงนามแฝง
`beta-latest` ของ 4.20 ที่ใช้งานอยู่กับสแนปช็อตที่ระบุวันที่

## ฟีเจอร์

<Warning>
  `x_search` และ `code_execution` ทำงานบนเซิร์ฟเวอร์ของ xAI โดย xAI เรียกเก็บ $5 ต่อการเรียกใช้
  เครื่องมือ 1,000 ครั้ง บวกโทเค็นอินพุตและเอาต์พุตของโมเดล เมื่อไม่ได้ระบุการตั้งค่า
  `enabled` ของแต่ละเครื่องมือ OpenClaw จะแสดงเครื่องมือนั้นเฉพาะเมื่อมีโมเดล xAI ที่ใช้งานอยู่
  ผู้ให้บริการโมเดลที่ไม่ใช่ xAI ซึ่งเป็นที่รู้จักต้องกำหนด `enabled: true` ต่อเครื่องมืออย่างชัดเจน
  หากไม่มีหรือไม่สามารถแก้ไขผู้ให้บริการได้ ระบบจะปิดไว้เพื่อความปลอดภัย ต้องมีการยืนยันตัวตน xAI เสมอ
  และ `enabled: false` จะปิดใช้เครื่องมือนี้สำหรับผู้ให้บริการทุกราย
</Warning>

      การแปลงรูปภาพเป็นวิดีโอทั้งแบบคลาสสิกและ Video 1.5 จะใช้อัตราส่วนของรูปภาพต้นทาง
      เมื่อไม่ได้ระบุ
    - ความละเอียด: แบบคลาสสิก `480P`/`720P`; Video 1.5 ยังรองรับ `1080P`; โหมด
      การสร้างทั้งหมดมีค่าเริ่มต้นเป็น `480P`
    - ระยะเวลา: 1-15 วินาทีสำหรับการสร้าง/รูปภาพเป็นวิดีโอ, 1-10 วินาทีเมื่อ
      ใช้บทบาท `reference_image` แบบคลาสสิก, 2-10 วินาทีสำหรับการขยายแบบคลาสสิก
    - การสร้างจากรูปภาพอ้างอิง: ตั้งค่า `imageRoles` เป็น `reference_image` สำหรับ
      รูปภาพที่ระบุทุกภาพ โดย xAI ยอมรับรูปภาพดังกล่าวได้สูงสุด 7 ภาพ
    - การแก้ไข/ขยายวิดีโอจะใช้อัตราส่วนภาพและความละเอียดของวิดีโออินพุต
      การดำเนินการเหล่านั้นไม่ยอมรับการแทนที่ค่ารูปทรง
    - ระยะหมดเวลาการดำเนินการเริ่มต้น: 600 วินาที เว้นแต่จะตั้งค่า `video_generate.timeoutMs`
      หรือ `agents.defaults.videoGenerationModel.timeoutMs`

    <Warning>
    ไม่รองรับบัฟเฟอร์วิดีโอภายในเครื่อง ให้ใช้ URL `http(s)` ระยะไกลสำหรับอินพุต
    การแก้ไข/ขยายวิดีโอ การแปลงรูปภาพเป็นวิดีโอรองรับบัฟเฟอร์รูปภาพภายในเครื่อง เนื่องจาก
    OpenClaw เข้ารหัสข้อมูลเหล่านั้นเป็น URL ข้อมูลสำหรับ xAI
    </Warning>

    Video 1.5 ยังรู้จักตัวระบุ `grok-imagine-video-1.5-preview` และ
    `grok-imagine-video-1.5-2026-05-30` ของ xAI ด้วย OpenClaw ส่งต่อตัวระบุ
    ที่เลือกโดยไม่เปลี่ยนแปลง แต่ใช้การตรวจสอบเฉพาะรูปภาพแบบเดียวกัน

    การเลือกผู้ให้บริการ และลักษณะการทำงานของการสลับไปใช้ระบบสำรอง
    </Note>

  <Accordion title="การสร้างรูปภาพ">
    Plugin `xai` ที่รวมมาด้วยลงทะเบียนการสร้างรูปภาพผ่านเครื่องมือ
    `image_generate` ที่ใช้ร่วมกัน

    - โมเดลรูปภาพเริ่มต้น: `xai/grok-imagine-image`
    - โมเดลเพิ่มเติม: `xai/grok-imagine-image-quality`
    - โหมด: การแปลงข้อความเป็นรูปภาพและการแก้ไขรูปภาพอ้างอิง
    - อินพุตอ้างอิง: `image` หนึ่งรายการ หรือ `images` สูงสุดสามรายการ
    - อัตราส่วนภาพ: `1:1`, `16:9`, `9:16`, `4:3`, `3:4`, `3:2`, `2:3`, `2:1`,
      `1:2`, `19.5:9`, `9:19.5`, `20:9`, `9:20`
    - ความละเอียด: `1K`, `2K`
    - จำนวน: สูงสุด 4 รูปภาพ
    - ระยะหมดเวลาเริ่มต้นของการดำเนินการ: 600 วินาที เว้นแต่จะตั้งค่า `image_generate.timeoutMs`
      หรือ `agents.defaults.imageGenerationModel.timeoutMs`

    OpenClaw ขอการตอบกลับรูปภาพแบบ `b64_json` จาก xAI เพื่อให้สามารถจัดเก็บ
    และส่งสื่อที่สร้างขึ้นผ่านเส้นทางไฟล์แนบของช่องทางตามปกติได้ รูปภาพอ้างอิง
    ภายในเครื่องจะถูกแปลงเป็น URL ข้อมูล ส่วนการอ้างอิง `http(s)` ระยะไกล
    จะถูกส่งผ่านโดยไม่มีการเปลี่ยนแปลง

    หากต้องการใช้ xAI เป็นผู้ให้บริการรูปภาพเริ่มต้น:

    ```json5
    {
      agents: {
        defaults: {
          imageGenerationModel: {
            primary: "xai/grok-imagine-image",
          },
        },
      },
    }
    ```

    <Note>
    xAI ยังระบุ `quality`, `mask`, `user` และอัตราส่วนภาพ `auto`
    ปัจจุบัน OpenClaw ส่งต่อเฉพาะตัวควบคุมรูปภาพข้ามผู้ให้บริการที่ใช้ร่วมกัน
    โดยไม่ได้เปิดเผยตัวเลือกที่ใช้ได้เฉพาะระบบเนทีฟเหล่านี้ผ่าน `image_generate`
    </Note>

  <Accordion title="การแปลงข้อความเป็นเสียงพูด">
    Plugin `xai` ที่รวมมาด้วยลงทะเบียนการแปลงข้อความเป็นเสียงพูดผ่านพื้นผิวผู้ให้บริการ
    `tts` ที่ใช้ร่วมกัน

    - เสียง: แค็ตตาล็อกสดจาก xAI ที่ผ่านการตรวจสอบสิทธิ์ แสดงรายการด้วย
      `openclaw infer tts voices --provider xai`
    - เสียงสำรองแบบออฟไลน์: `ara`, `eve`, `leo`, `rex`, `sal`
    - เสียงเริ่มต้น: `eve`
    - ระบบจะส่งต่อ ID เสียงแบบกำหนดเองของบัญชี แม้จะไม่มีอยู่ในการตอบกลับ
      ของแค็ตตาล็อกในตัว
    - รูปแบบ: `mp3`, `wav`, `pcm`, `mulaw`, `alaw`
    - ภาษา: รหัส BCP-47 หรือ `auto`
    - ความเร็ว: การแทนที่ความเร็วแบบเนทีฟของผู้ให้บริการ
    - ไม่รองรับรูปแบบบันทึกเสียง Opus แบบเนทีฟ

    หากต้องการใช้ xAI เป็นผู้ให้บริการ TTS เริ่มต้น:

    ```json5
    {
      messages: {
        tts: {
          provider: "xai",
          providers: {
            xai: {
              voiceId: "eve",
            },
          },
        },
      },
    }
    ```

    <Note>
    OpenClaw ใช้ปลายทางแบบแบตช์ `/v1/tts` ของ xAI สำหรับการสังเคราะห์แบบบัฟเฟอร์
    ใช้การค้นพบแค็ตตาล็อก `/v1/tts/voices` ที่ผ่านการตรวจสอบสิทธิ์ และใช้
    `wss://api.x.ai/v1/tts` แบบเนทีฟสำหรับการสังเคราะห์แบบสตรีม การสตรีมจำกัดไว้เฉพาะ
    โฮสต์ `api.x.ai` แบบเนทีฟ ดังนั้นค่า `baseUrl` แบบกำหนดเองจึงถูกปฏิเสธใน
    เส้นทางนี้ ระบบใช้ตัวควบคุมภาษา เสียง ตัวเข้ารหัส และความเร็วที่มีอยู่ ส่วนอัตรา
    การสุ่มตัวอย่างและอัตราบิตจะใช้ค่าเริ่มต้นของ xAI การสังเคราะห์ไฟล์เสียงรองรับ
    ตัวเข้ารหัสที่กำหนดค่าไว้ทั้งหมด เป้าหมายบันทึกเสียงใช้ MP3 สำหรับการสตรีมและ
    การสำรองแบบบัฟเฟอร์ เนื่องจากตัวเข้ารหัสแบบดิบของ xAI ไม่มีข้อมูลเมตาของตัวเข้ารหัส/อัตรา
    สตรีมจะส่ง `text.delta` แล้วจึงส่ง
    `text.done` รับ `audio.delta`, `audio.done` หรือ `error` และใช้
    `timeoutMs` เมื่อไม่มีการใช้งาน ซึ่งจะรีเฟรชทุกครั้งที่ได้รับชิ้นส่วนเสียง ระบบนี้แยกจาก
    เซสชันเสียงแบบเรียลไทม์ ดูสัญญา [API การสตรีม TTS](https://docs.x.ai/developers/rest-api-reference/inference/voice) ของ xAI
    </Note>

  </Accordion>

    - เส้นทางอินพุต: การอัปโหลดไฟล์เสียงแบบ multipart
    - การเลือกโมเดล: xAI เลือกโมเดลการถอดเสียงภายใน โดยปลายทาง
      ไม่มีตัวเลือกโมเดล
    - ใช้ทุกตำแหน่งที่การถอดเสียงขาเข้าอ่าน `tools.media.audio`

    ```json5
    {
      tools: {
        media: {
          audio: {
            models: [
              {
                type: "provider",
                provider: "xai",
              },
            ],
          },
        },
      },
    }
    ```

    สามารถระบุภาษาผ่านการกำหนดค่าสื่อเสียงที่ใช้ร่วมกันหรือคำขอถอดเสียง
    ในแต่ละครั้งได้ พื้นผิว OpenClaw ที่ใช้ร่วมกันยอมรับคำใบ้พรอมต์ แต่การผสานรวม
    REST STT ของ xAI จะส่งต่อเฉพาะไฟล์และภาษา เนื่องจากมีเพียงสองรายการนี้
    ที่แมปกับปลายทางสาธารณะปัจจุบันของ xAI

  <Accordion title="การแปลงเสียงพูดเป็นข้อความแบบสตรีม">
    Plugin `xai` ที่รวมมาด้วยยังลงทะเบียนผู้ให้บริการถอดเสียงแบบเรียลไทม์
    สำหรับเสียงการโทรสด

    - ปลายทาง: WebSocket `wss://api.x.ai/v1/stt` ของ xAI
    - การเข้ารหัสเริ่มต้น: `mulaw`
    - อัตราการสุ่มตัวอย่างเริ่มต้น: `8000`
    - การกำหนดจุดสิ้นสุดเริ่มต้น: `800ms`
    - ข้อความถอดเสียงระหว่างดำเนินการ: เปิดใช้งานโดยค่าเริ่มต้น

    สตรีมสื่อ Twilio ของ Voice Call ส่งเฟรมเสียง G.711 mu-law ดังนั้น
    ผู้ให้บริการ xAI จึงส่งต่อเฟรมเหล่านั้นโดยตรงโดยไม่แปลงรหัส:

    ```json5
    {
      plugins: {
        entries: {
          "voice-call": {
            config: {
              streaming: {
                enabled: true,
                provider: "xai",
                providers: {
                  xai: {
                    apiKey: "${XAI_API_KEY}",
                    endpointingMs: 800,
                    language: "en",
                  },
                },
              },
            },
          },
        },
      },
    }
    ```

    การกำหนดค่าที่ผู้ให้บริการเป็นเจ้าของอยู่ภายใต้
    `plugins.entries.voice-call.config.streaming.providers.xai` คีย์ที่รองรับ
    ได้แก่ `apiKey`, `baseUrl`, `sampleRate`, `encoding` (`pcm`, `mulaw` หรือ
    `alaw`), `interimResults`, `endpointingMs` และ `language`

    <Note>
    ผู้ให้บริการสตรีมนี้ใช้สำหรับเส้นทางการถอดเสียงแบบเรียลไทม์ของ Voice Call
    เสียงของ Discord จะบันทึกเป็นส่วนสั้นๆ และใช้เส้นทางการถอดเสียงแบบแบตช์
    `tools.media.audio` แทน
    </Note>

  </Accordion>

  <Accordion title="เสียงแบบเรียลไทม์ (Talk)">
    Plugin `xai` ที่รวมมาด้วยลงทะเบียนเซสชันแบบเรียลไทม์ของ Grok Voice Agent สำหรับ
    โหมด Talk ผ่านสัญญา `registerRealtimeVoiceProvider` ที่ใช้ร่วมกัน

    - ปลายทาง: `wss://api.x.ai/v1/realtime?model=<voice-model>`
    - โมเดลเริ่มต้น: `grok-voice-latest`
    - เสียงเริ่มต้น: `eve`
    - การรับส่งข้อมูล: `gateway-relay` (เส้นทางรีเลย์ของ iOS, Android และ Control UI)
    - เสียง: PCM16 24 kHz หรือ G.711 µ-law 8 kHz
    - การพูดแทรก: VAD ของเซิร์ฟเวอร์ xAI ขัดจังหวะการตอบกลับ OpenClaw จะล้างการเล่นที่อยู่ในคิว
      และตัดประวัติของผู้ให้บริการที่ยังไม่ได้เล่นออก

    กำหนดค่า Talk บน Gateway:

    ```json5
    {
      talk: {
        realtime: {
          provider: "xai",
          mode: "realtime",
          transport: "gateway-relay",
          brain: "agent-consult",
          providers: {
            xai: {
              model: "grok-voice-latest",
              voice: "eve",
              // เลือกใช้เฉพาะเมื่อยอมรับการเล่นเซสชันซ้ำในฝั่งผู้ให้บริการได้
              sessionResumption: false,
            },
          },
        },
      },
      env: { XAI_API_KEY: "xai-..." },
    }
    ```

    การกำหนดค่าที่ผู้ให้บริการเป็นเจ้าของยังถูกแก้ค่าจาก
    `plugins.entries.voice-call.config.realtime.providers.xai` เมื่อ Voice Call
    หรือตัวเลือกแบบเรียลไทม์ที่ใช้ร่วมกันนำแมปผู้ให้บริการเดียวกันกลับมาใช้ คีย์ที่รองรับได้แก่
    `apiKey`, `baseUrl`, `model`, `voice`, `vadThreshold`, `silenceDurationMs`,
    `prefixPaddingMs`, `reasoningEffort` และ `sessionResumption`
    `reasoningEffort` ยอมรับเฉพาะ `high` หรือ `none` ซึ่งตรงกับ API ของ xAI Voice Agent

    VAD ของเซิร์ฟเวอร์ xAI จะสร้างการตอบกลับและจัดการการขัดจังหวะเสียงเสมอ
    ใช้ `consultRouting: "provider-direct"` โดยโปรโตคอล xAI Voice Agent ไม่รองรับการบังคับ
    กำหนดเส้นทางข้อความถอดเสียงและการปิดใช้งานการขัดจังหวะเสียงอินพุต

    <Note>
    สามารถใช้ xAI OAuth หรือ `XAI_API_KEY` เพื่อตรวจสอบสิทธิ์เสียงแบบเรียลไทม์ได้
    WebRTC ที่เบราว์เซอร์เป็นเจ้าของยังไม่เป็นส่วนหนึ่งของพื้นผิวผู้ให้บริการนี้ ให้ใช้ Talk แบบ gateway-relay
    บน Node เนทีฟหรือเส้นทางรีเลย์ของ Control UI
    </Note>

    <Note>
    `sessionResumption` มีค่าเริ่มต้นเป็น `false` เมื่อตั้งเป็น `true` OpenClaw จะขอให้
    xAI เก็บสถานะเซสชันให้เพียงพอสำหรับดำเนินบทสนทนาเดิมต่อหลัง
    เชื่อมต่อใหม่ จากนั้นจึงเชื่อมต่อใหม่ด้วย ID บทสนทนาที่ได้รับ ปล่อยให้
    ปิดใช้งานไว้เมื่อไม่สามารถยอมรับการเล่นซ้ำ/การเก็บรักษาในฝั่งผู้ให้บริการได้ มิฉะนั้นซ็อกเก็ตที่ถูกขัดจังหวะ
    จะล้มเหลวแบบปิดกั้นแทนที่จะเริ่มบทสนทนาใหม่โดยไม่แจ้งให้ทราบ
    </Note>

  </Accordion>

  <Accordion title="การกำหนดค่า x_search">
    Plugin xAI ที่รวมมาด้วยเปิดเผย `x_search` เป็นเครื่องมือ OpenClaw สำหรับ
    ค้นหาเนื้อหาบน X (เดิมคือ Twitter) ผ่าน Grok

    เส้นทางการกำหนดค่า: `plugins.entries.xai.config.xSearch`

    | คีย์               | ชนิด    | ค่าเริ่มต้น                   | คำอธิบาย                                      |
    | ----------------- | ------- | ------------------------- | ------------------------------------------------ |
    | `enabled`         | บูลีน | อัตโนมัติสำหรับโมเดล xAI  | ปิดใช้งาน หรือเลือกใช้สำหรับผู้ให้บริการที่ทราบว่าไม่ใช่ xAI |
    | `model`           | สตริง  | `grok-4.3`                | โมเดลที่ใช้สำหรับคำขอ x_search                 |
    | `baseUrl`         | สตริง  | -                         | แทนที่ URL ฐานของ xAI Responses                  |
    | `inlineCitations` | บูลีน | -                         | รวมการอ้างอิงแบบอินไลน์ในผลลัพธ์              |
    | `maxTurns`        | ตัวเลข  | -                         | จำนวนรอบการสนทนาสูงสุด                       |
    | `timeoutSeconds`  | ตัวเลข  | `30`                      | ระยะหมดเวลาของคำขอเป็นวินาที                       |
    | `cacheTtlMinutes` | ตัวเลข  | `15`                      | ระยะเวลาคงอยู่ของแคชเป็นนาที                    |

    ```json5
    {
      plugins: {
        entries: {
          xai: {
            config: {
              xSearch: {
                enabled: true,
                model: "grok-4.3",
                baseUrl: "https://api.x.ai/v1",
                inlineCitations: true,
              },
            },
          },
        },
      },
    }
    ```

  </Accordion>

    | ---------------- | ------- | ------------------------ | ------------------------------------------------ |
    | `enabled`        | boolean | อัตโนมัติสำหรับโมเดล xAI | ปิดใช้งาน หรือเลือกเปิดใช้สำหรับผู้ให้บริการที่ทราบว่าไม่ใช่ xAI |
    | `model`          | string  | `grok-4.3`               | โมเดลที่ใช้สำหรับคำขอเรียกใช้โค้ด           |
    | `maxTurns`       | number  | -                        | จำนวนรอบการสนทนาสูงสุด                       |

    <Note>

  <Accordion title="ข้อจำกัดที่ทราบ">
    - การรับรองความถูกต้องของ xAI สามารถใช้คีย์ API, ตัวแปรสภาพแวดล้อม, การสำรองไปใช้การกำหนดค่าของ Plugin
      หรือ OAuth ด้วยบัญชี xAI ที่มีสิทธิ์ OAuth ใช้การยืนยันด้วยรหัสอุปกรณ์
      โดยไม่มีคอลแบ็ก localhost xAI เป็นผู้กำหนดว่าบัญชีใด
      สามารถรับโทเค็น API ของ OAuth ได้ และหน้าคำยินยอมอาจแสดง Grok Build
      แม้ว่า OpenClaw จะไม่ต้องใช้แอป Grok Build
    - ขณะนี้ OpenClaw ยังไม่เปิดให้ใช้ตระกูลโมเดลแบบหลายเอเจนต์ของ xAI โดย xAI
      ให้บริการโมเดลเหล่านี้ผ่าน Responses API แต่โมเดลเหล่านี้ไม่ยอมรับ
      เครื่องมือฝั่งไคลเอนต์หรือเครื่องมือแบบกำหนดเองที่ลูปเอเจนต์ร่วมของ OpenClaw ใช้
      โปรดดู
      [ข้อจำกัดของระบบหลายเอเจนต์ของ xAI](https://docs.x.ai/developers/model-capabilities/text/multi-agent#limitations)
    - ขณะนี้เสียงเรียลไทม์ของ xAI เปิดให้ใช้เฉพาะการรับส่ง Talk แบบรีเลย์ผ่าน Gateway
      เซสชัน WebSocket ของผู้ให้บริการที่เบราว์เซอร์เป็นเจ้าของยังไม่ได้เชื่อมต่อ
      ใน Control UI
    - xAI image `quality`, image `mask` และอัตราส่วนภาพเพิ่มเติมที่ใช้ได้เฉพาะแบบเนทีฟ
      จะยังไม่เปิดให้ใช้จนกว่าเครื่องมือ `image_generate` ร่วมจะมี
      ตัวควบคุมข้ามผู้ให้บริการที่สอดคล้องกัน
  </Accordion>

  <Accordion title="หมายเหตุขั้นสูง">
    - OpenClaw ใช้การแก้ไขความเข้ากันได้ของสคีมาเครื่องมือและการเรียกเครื่องมือ
      ที่เฉพาะเจาะจงสำหรับ xAI โดยอัตโนมัติบนเส้นทางรันเนอร์ร่วม
    - คำขอ xAI แบบเนทีฟใช้ค่าเริ่มต้นเป็น `tool_stream: true` ตั้งค่า
      `agents.defaults.models["xai/<model>"].params.tool_stream` เป็น `false`
      เพื่อปิดใช้งาน
    - ตัวห่อหุ้ม xAI ที่รวมมาให้จะตัดขอบเขตจำนวน contains ของสคีมาที่ไม่รองรับ
      และคีย์เพย์โหลด *effort* สำหรับการให้เหตุผลที่ไม่รองรับออกก่อนส่งคำขอ xAI
      แบบเนทีฟ Grok 4.5 รองรับ effort ระดับ low, medium และ
      high (ค่าเริ่มต้นคือ high) Grok 4.3 รองรับ none, low, medium และ high
      (ค่าเริ่มต้นคือ low) โมเดล xAI อื่นที่รองรับการให้เหตุผลไม่มี
      ตัวควบคุม effort ที่กำหนดค่าได้ แต่ยังคงขอ
      `include: ["reasoning.encrypted_content"]` เพื่อให้สามารถนำการให้เหตุผลที่เข้ารหัสไว้ก่อนหน้า
      มาเล่นซ้ำในรอบถัดไปได้
    - `web_search`, `x_search` และ `code_execution` เปิดให้ใช้เป็นเครื่องมือของ OpenClaw
      OpenClaw จะแนบเฉพาะเครื่องมือในตัวของ xAI ที่แต่ละเครื่องมือต้องใช้
      ไปกับคำขอของเครื่องมือนั้น แทนที่จะแนบเครื่องมือเนทีฟทั้งหมดไปกับ
      ทุกรอบการแชต
    - Grok `web_search` อ่าน `plugins.entries.xai.config.webSearch.baseUrl`
      `x_search` อ่าน `plugins.entries.xai.config.xSearch.baseUrl` จากนั้น
      สำรองไปใช้ URL ฐานสำหรับการค้นหาเว็บของ Grok
    - `x_search` และ `code_execution` อยู่ภายใต้การดูแลของ Plugin xAI ที่รวมมาให้
      แทนที่จะฮาร์ดโค้ดไว้ในรันไทม์โมเดลหลัก
    - `code_execution` คือการเรียกใช้ในแซนด์บ็อกซ์ xAI ระยะไกล ไม่ใช่
      [`exec`](/th/tools/exec) ภายในเครื่อง
  </Accordion>
</AccordionGroup>

## การทดสอบแบบสด

เส้นทางสื่อของ xAI ครอบคลุมด้วยการทดสอบหน่วยและชุดทดสอบแบบสดที่ต้องเลือกเปิดใช้ ส่งออก
`XAI_API_KEY` ในสภาพแวดล้อมของโพรเซสก่อนเรียกใช้โพรบแบบสด

```bash
pnpm test extensions/xai
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_TEST_QUIET=1 pnpm test:live -- extensions/xai/xai.live.test.ts
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_XAI_VIDEO=1 pnpm test:live -- extensions/xai/xai.live.test.ts -t "classic Grok Imagine"
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_XAI_VIDEO=1 pnpm test:live -- extensions/xai/xai.live.test.ts -t "Grok Imagine Video 1.5"
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_TEST_QUIET=1 pnpm test:live -- extensions/xai/x-search.live.test.ts
OPENCLAW_LIVE_GATEWAY_MODELS="xai/grok-4.5,xai/grok-build-0.1,xai/grok-4.3,xai/grok-4.20-0309-reasoning,xai/grok-4.20-0309-non-reasoning" OPENCLAW_LIVE_GATEWAY_MAX_MODELS=0 OPENCLAW_LIVE_GATEWAY_SMOKE=0 pnpm test:live -- src/gateway/gateway-models.profiles.live.test.ts
OPENCLAW_LIVE_TEST=1 OPENCLAW_LIVE_TEST_QUIET=1 OPENCLAW_LIVE_IMAGE_GENERATION_PROVIDERS=xai pnpm test:live -- test/image-generation.runtime.live.test.ts
```

ไฟล์ทดสอบแบบสดเฉพาะผู้ให้บริการสังเคราะห์ TTS ปกติ, TTS แบบ PCM
ที่เหมาะกับระบบโทรศัพท์, ถอดเสียงผ่าน STT แบบแบตช์ของ xAI, สตรีม PCM เดียวกันผ่าน
STT เรียลไทม์ของ xAI, สร้างผลลัพธ์แบบข้อความเป็นภาพ และแก้ไขภาพอ้างอิง
ไฟล์ทดสอบภาพแบบสดร่วมจะตรวจสอบผู้ให้บริการ xAI เดียวกันผ่านเส้นทาง
การเลือกรันไทม์ การสำรอง การทำให้เป็นมาตรฐาน และการแนบสื่อของ OpenClaw
กรณี Video 1.5 ที่ต้องเลือกเปิดใช้จะส่งภาพเฟรมแรกที่สร้างขึ้นหนึ่งภาพในความละเอียด 1080P และ
ตรวจสอบการดาวน์โหลดวิดีโอที่เสร็จสมบูรณ์
