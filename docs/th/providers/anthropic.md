Anthropic พัฒนาตระกูลโมเดล **Claude** OpenClaw รองรับช่องทางการยืนยันตัวตนสองแบบ:

- **คีย์ API** - เข้าถึง Anthropic API โดยตรงพร้อมการเรียกเก็บเงินตามการใช้งาน (โมเดล `anthropic/*`)
- **Claude CLI** - ใช้การเข้าสู่ระบบ Claude Code ที่มีอยู่แล้วบนโฮสต์เดียวกัน

## การติดตามการใช้งานและค่าใช้จ่าย

OpenClaw ตรวจหาข้อมูลประจำตัว Anthropic ที่พร้อมใช้งานและเลือกส่วนแสดงการใช้งานที่ตรงกัน:

- ข้อมูลประจำตัวสำหรับการสมัครสมาชิก/การตั้งค่า Claude จะแสดงช่วงโควตาและงบประมาณเสริมสำหรับการใช้งานเพิ่มเติมที่กำหนดหรือไม่ก็ได้
- `ANTHROPIC_ADMIN_KEY` หรือ `ANTHROPIC_ADMIN_API_KEY` จะแสดงค่าใช้จ่ายระดับองค์กรที่ผู้ให้บริการรายงานและการใช้งาน Messages API ย้อนหลัง 30 วันใน **การใช้งาน** ของ Control UI รวมถึงยอดใช้จ่ายรายวัน ยอดรวมโทเค็น/แคช โมเดลที่ใช้มากที่สุด และหมวดหมู่ค่าใช้จ่าย
- ระบบจะตรวจพบข้อมูลประจำตัว `sk-ant-admin...` ที่เก็บไว้ในโปรไฟล์ผู้ให้บริการ Anthropic เป็นคีย์ Admin API โดยอัตโนมัติ

ประวัติค่าใช้จ่ายของ Admin API มาจาก [Usage and Cost API ของ Anthropic](https://platform.claude.com/docs/en/manage-claude/usage-cost-api) ซึ่งเป็นยอดเรียกเก็บเงินจริงจากผู้ให้บริการ แยกจากค่าใช้จ่ายโดยประมาณที่ OpenClaw คำนวณจากเซสชัน

<Warning>
แบ็กเอนด์ Claude CLI ของ OpenClaw เรียกใช้ Claude Code CLI ที่ติดตั้งไว้ใน
โหมดพิมพ์แบบไม่โต้ตอบ (`claude -p`) เอกสาร Claude Code ปัจจุบันของ Anthropic
อธิบายโหมดดังกล่าวว่าเป็นการใช้งาน Agent SDK/แบบโปรแกรม การอัปเดตฝ่ายสนับสนุนของ Anthropic เมื่อวันที่ 15 มิถุนายน 2026
ได้ระงับการเปลี่ยนแปลงการเรียกเก็บเงินแยกต่างหากสำหรับ Agent SDK ที่ประกาศไว้: การใช้งาน Claude
Agent SDK, `claude -p` และแอปของบุคคลที่สามยังคงหักจากขีดจำกัดการใช้งาน
ของการสมัครสมาชิกที่เข้าสู่ระบบอยู่ และเครดิต Agent SDK รายเดือนที่ประกาศไว้ก่อนหน้านี้
ยังไม่พร้อมใช้งานระหว่างที่ Anthropic ปรับปรุงแผนดังกล่าว

Claude Code แบบโต้ตอบยังคงหักจากขีดจำกัดของแผน Claude ที่เข้าสู่ระบบอยู่
การยืนยันตัวตนด้วยคีย์ API เป็นการเรียกเก็บเงินตามการใช้งานจริงโดยตรงและไม่ขึ้นอยู่กับแผนดังกล่าว
สำหรับโฮสต์ Gateway ที่ทำงานระยะยาว ระบบอัตโนมัติที่ใช้ร่วมกัน และค่าใช้จ่ายในการผลิต
ที่คาดการณ์ได้ ให้ใช้คีย์ API ของ Anthropic

บทความฝ่ายสนับสนุนปัจจุบันของ Anthropic สามารถเปลี่ยนลักษณะการทำงานนี้ได้โดยไม่ต้องมี
OpenClaw รุ่นใหม่:

- [ข้อมูลอ้างอิง Claude Code CLI](https://code.claude.com/docs/en/cli-usage)
- [ใช้ Claude Agent SDK กับแผน Claude ของคุณ](https://support.claude.com/en/articles/15036540-use-the-claude-agent-sdk-with-your-claude-plan)
- [ใช้ Claude Code กับแผน Pro หรือ Max ของคุณ](https://support.claude.com/en/articles/11145838-use-claude-code-with-your-pro-or-max-plan)
- [ใช้ Claude Code กับแผน Team หรือ Enterprise ของคุณ](https://support.claude.com/en/articles/11845131-using-claude-code-with-your-team-or-enterprise-plan)
- [จัดการค่าใช้จ่ายของ Claude Code](https://code.claude.com/docs/en/costs)

</Warning>

    ผู้ให้บริการที่ใช้ OAuth ซึ่ง OpenClaw จัดการให้ เช่น

    ### รับโทเค็นการตั้งค่า

    เรียกใช้ `claude setup-token` บนเครื่องใดก็ได้ที่ติดตั้ง Claude Code คำสั่งนี้จะแสดง
    โทเค็นระยะยาวที่ขึ้นต้นด้วย `sk-ant-oat01-`

    ระหว่างการเริ่มต้นใช้งาน ให้วางโทเค็นในแอป macOS โดยเลือก
    **Anthropic setup-token** ภายใต้ **Connect with an API key or token** หรือใช้:

    ```bash
    openclaw models auth login --provider anthropic --method setup-token
    ```

    ### ตัวอย่างการกำหนดค่า

    ควรใช้การอ้างอิงโมเดล Anthropic แบบมาตรฐานร่วมกับการแทนที่รันไทม์ CLI:

    ```json5
    {
      agents: {
        defaults: {
          model: { primary: "anthropic/claude-opus-4-8" },
          models: {
            "anthropic/claude-opus-4-8": {
              agentRuntime: { id: "claude-cli" },
            },
          },
        },
      },
    }
    ```

    การอ้างอิงโมเดล `claude-cli/claude-opus-4-7` แบบเดิมยังคงใช้งานได้เพื่อ
    ความเข้ากันได้ แต่การกำหนดค่าใหม่ควรเก็บการเลือกผู้ให้บริการ/โมเดลเป็น
    `anthropic/*` และระบุแบ็กเอนด์การประมวลผลไว้ในนโยบายรันไทม์ของผู้ให้บริการ/โมเดล

    ### การเรียกเก็บเงินและ `claude -p`

    OpenClaw ใช้เส้นทาง `claude -p` แบบไม่โต้ตอบของ Claude Code สำหรับการทำงานของ Claude CLI
    ปัจจุบัน Anthropic ถือว่าเส้นทางดังกล่าวเป็นการใช้งาน Agent SDK/แบบโปรแกรม:

    - การอัปเดตฝ่ายสนับสนุนของ Anthropic เมื่อวันที่ 15 มิถุนายน 2026 ระงับแผน
      เครดิต Agent SDK แยกต่างหากที่ประกาศไว้ก่อนหน้านี้
    - การใช้งาน Claude Agent SDK, `claude -p` และแอปของบุคคลที่สาม
      ภายใต้แผนการสมัครสมาชิกยังคงหักจากขีดจำกัดการใช้งานของการสมัครสมาชิกที่เข้าสู่ระบบอยู่
    - เครดิต Agent SDK รายเดือนที่ประกาศไว้ก่อนหน้านี้ยังไม่พร้อมใช้งานระหว่างที่
      Anthropic ปรับปรุงแผนดังกล่าว
    - การเข้าสู่ระบบด้วย Console/คีย์ API ใช้การเรียกเก็บเงิน API ตามการใช้งานจริงและไม่ได้รับ
      เครดิต Agent SDK ของการสมัครสมาชิก

    โปรดดู[บทความเกี่ยวกับแผน Agent SDK
    ของ Anthropic](https://support.claude.com/en/articles/15036540-use-the-claude-agent-sdk-with-your-claude-plan)
    สำหรับประกาศการระงับ และบทความเกี่ยวกับแผน Claude Code สำหรับลักษณะการทำงานของ
    การสมัครสมาชิก [Pro/Max](https://support.claude.com/en/articles/11145838-use-claude-code-with-your-pro-or-max-plan)
    และ
    [Team/Enterprise](https://support.claude.com/en/articles/11845131-use-claude-code-with-your-team-or-enterprise-plan)

    Anthropic สามารถเปลี่ยนการเรียกเก็บเงินและลักษณะการจำกัดอัตราของ Claude Code ได้โดยไม่ต้องมี
    OpenClaw รุ่นใหม่ ตรวจสอบ `claude auth status`, `/status` และ
    เอกสาร Anthropic ที่ลิงก์ไว้เมื่อความสามารถในการคาดการณ์ค่าใช้จ่ายมีความสำคัญ

## เซสชัน Claude ระหว่างคอมพิวเตอร์

Plugin Anthropic ที่รวมมาให้จะเพิ่มกลุ่ม **Claude Code** ลงในแถบด้านข้างของเซสชัน
ตามปกติ แถวต่างๆ จะเปิดในบานหน้าต่างแชทตามปกติ ระบบจะค้นหาเซสชัน Claude
Code ที่ไม่ได้เก็บถาวรบน Gateway และบนโฮสต์ Node ที่เชื่อมต่อ:

- เซสชัน Claude CLI มาจากระเบียนดัชนีโปรเจกต์ที่ถูกต้องและไฟล์ JSONL
  ปัจจุบันซึ่งคำนำหน้าข้อมูลเมตาที่มีขอบเขตระบุว่าเป็นเซสชัน `sdk-cli`
  ที่ไม่ใช่ sidechain ภายใต้ `~/.claude/projects/`
- เซสชัน Claude Desktop ใช้ชื่อ Desktop เวลากิจกรรม และ
  สถานะการเก็บถาวร เมื่อข้อมูลเมตาชี้ไปยัง ID เซสชัน Claude Code เดียวกัน
- เซสชันที่มีเฉพาะ CLI ไม่มีแฟล็กการเก็บถาวร จึงยังคงมองเห็นได้ตราบใดที่
  มีทรานสคริปต์อยู่

ไม่จำเป็นต้องกำหนดค่า OpenClaw เพิ่มเติมสำหรับการค้นหา Plugin Anthropic
รวมมาให้และเปิดใช้งานโดยค่าเริ่มต้น Node macOS แบบเนทีฟจะประกาศคำสั่งเซสชัน
Claude แบบอ่านอย่างเดียวเมื่อมีไดเรกทอรี `~/.claude/projects/` ในเครื่อง
อนุมัติการอัปเกรดการจับคู่ Node เมื่อคำสั่งเหล่านั้นปรากฏเป็นครั้งแรก

แถบด้านข้างจัดกลุ่มแถวตาม Gateway หรือโฮสต์ Node ที่จับคู่ เริ่มด้วยหน้า
ที่มีขอบเขตและใหม่ที่สุดจากแต่ละโฮสต์ และรีเฟรชตามรอบปกติทุก 30 วินาที
ใช้ **โหลดเซสชันเพิ่มเติม** ใต้กลุ่มแค็ตตาล็อกเพื่อเพิ่มหน้าถัดไป
สำหรับทุกโฮสต์ที่มีประวัติเพิ่มเติม แถวที่เพิ่มเข้ามาจะยังคงมองเห็นได้และ
ถูกดึงใหม่ให้มีความลึกเท่าเดิมระหว่างการรีเฟรช ไคลเอนต์แค็ตตาล็อกใช้
`sessions.catalog.list`; การเปิดแถวใช้ `sessions.catalog.read`

การเข้าควบคุมเทอร์มินัลจะค้นหา `claude` จาก PATH ของเชลล์เข้าสู่ระบบ
ของผู้ใช้บนโฮสต์เจ้าของ ก่อน PATH ของบริการ/ดีมอน วิธีนี้ทำให้เซสชันที่เริ่มจากแอป
สอดคล้องกับ Claude CLI ที่ผู้ปฏิบัติงานได้รับในเทอร์มินัลปกติ

เมื่อเลือกแถว ระบบจะอ่านหน้าทรานสคริปต์ใหม่ล่าสุดก่อน **โหลดรายการทรานสคริปต์
ที่เก่ากว่า** จะตามเคอร์เซอร์ไบต์แบบทึบและอ่านส่วนที่มีขอบเขตอีกส่วนจาก
ไฟล์ JSONL แทนการโหลดประวัติทั้งหมด เนื้อหาปกติของผู้ใช้ ผู้ช่วย
การให้เหตุผล การเรียกใช้เครื่องมือ และผลลัพธ์จากเครื่องมือจะยังคงอยู่ รายการแต่ละรายการ
ที่ใหญ่กว่าเพดานความปลอดภัยของ Node/Gateway จะมีเครื่องหมายชัดเจนว่าถูกตัดทอน

สำหรับแถว `claude-cli` ที่อยู่ใน Gateway การพิมพ์ในช่องเขียนข้อความตามปกติจะเรียก
`sessions.catalog.continue` OpenClaw จะค้นหาระเบียนแค็ตตาล็อกในเครื่องอีกครั้ง
สร้างหรือใช้เซสชันแบบเนทีฟที่ล็อกโมเดลไว้ซ้ำ นำเข้ารายการที่มองเห็นได้สูงสุด 200
รายการหรือ 512 KiB และเตรียมการเชื่อมโยง Claude CLI เทิร์นแรกจะทำงานต่อด้วย
`--fork-session`; Claude จะกำหนด ID เซสชันใหม่ให้ฟอร์ก ดังนั้นเทิร์นต่อๆ ไปจึงใช้
ฟอร์กและไม่แก้ไขเซสชันต้นทาง

โฮสต์ Node แบบไม่มีส่วนติดต่อผู้ใช้ยังสามารถทำให้แถว Claude CLI ดำเนินต่อได้ด้วยการเปิดใช้
การตั้งค่าเฉพาะ Node ด้านล่างและรีสตาร์ตโฮสต์ Node:

```json5
{
  nodeHost: {
    agentRuns: {
      claude: { enabled: true },
    },
  },
}
```

Node จะประกาศ `agent.cli.claude.run.v1` เฉพาะเมื่อเปิดใช้การตั้งค่า
และสามารถค้นหาไฟล์ปฏิบัติการ `claude` ในเครื่องได้ OpenClaw จะค้นหาระเบียนแค็ตตาล็อก
บน Node นั้นอีกครั้ง นำเข้าประวัติที่มีขอบเขตเดียวกัน และเชื่อมโยงเซสชันที่รับช่วง
เข้ากับ Node และไดเรกทอรีทำงานที่แค็ตตาล็อกรายงาน แต่ละเทิร์นจะเรียกใช้
กระบวนการ `claude -p` จริงของ Node โดยใช้ไฟล์ Claude และการเข้าสู่ระบบของ Node นั้น
นโยบายการอนุมัติการดำเนินการของ Node ยังคงมีผล Gateway ไม่สามารถบังคับการเลือกเปิดใช้งานได้

การดำเนินต่อผ่าน Node v1 ใช้ได้เพียงครั้งเดียว โดยไม่รวมการกำหนดค่า MCP แบบวนกลับของ Gateway และ
อาร์กิวเมนต์ Plugin Skills ของ Gateway ไม่สร้างข้อมูลเริ่มต้นใหม่จากทรานสคริปต์ Gateway และ
ปฏิเสธไฟล์แนบและรูปภาพ แถว Claude Desktop ยังคงดูได้อย่างเดียว Node ของแอป
macOS แบบเนทีฟยังคงดูได้อย่างเดียวเช่นกันจนกว่าแอปจะประกาศคำสั่งเรียกใช้

<Note>
เซสชัน Claude ของ Node ที่จับคู่จะยังคงอ่านได้อย่างเดียว เว้นแต่ Node แบบไม่มีส่วนติดต่อผู้ใช้จะประกาศ
`agent.cli.claude.run.v1` อย่างชัดเจน OpenClaw จะไม่แก้ไขข้อมูลเมตาของ Claude Desktop
หรือเก็บเซสชัน Claude ถาวร หน้านี้ต้องใช้การเชื่อมต่อของผู้ปฏิบัติงาน
ที่มีขอบเขตการเขียน เนื่องจากใช้ `node.invoke` ที่ผ่านการยืนยันตัวตนแล้ว ส่วนการแสดงรายการและการอ่าน
ยังคงเป็นแบบอ่านอย่างเดียว แม้บน Node ที่เปิดใช้การดำเนินต่อ
</Note>

โปรดดู [Node: เซสชันและทรานสคริปต์ Claude](/th/nodes#claude-sessions-and-transcripts)
สำหรับคำสั่ง Node และขอบเขตความปลอดภัย

## ค่าเริ่มต้นการคิด (Claude Sonnet 5, Mythos 5, Fable 5, 4.8 และ 4.6)

`anthropic/claude-sonnet-5` ใช้การคิดแบบปรับเปลี่ยนได้ที่ระดับความพยายาม `high` ตามค่าเริ่มต้น
ใช้ `/think off` เพื่อปิดใช้การคิด หรือ `/think xhigh|max` สำหรับระดับความพยายามดั้งเดิม
ที่สูงขึ้นของโมเดล OpenClaw จะไม่ส่งงบประมาณการคิดแบบกำหนดเอง พารามิเตอร์
การสุ่มตัวอย่างแบบกำหนดเอง ข้อความเติมล่วงหน้าของผู้ช่วย และ Priority Tier สำหรับ Sonnet 5 เนื่องจาก
Anthropic ไม่รองรับคุณลักษณะคำขอเหล่านั้นในโมเดลนี้
แค็ตตาล็อกใช้ราคาอินพุต/เอาต์พุตช่วงเปิดตัวของ Anthropic ที่ `$2/$10` จนถึง
31 สิงหาคม 2026 และราคา `$3/$15` มาตรฐานจะเริ่มในวันที่ 1 กันยายน 2026

`anthropic/claude-fable-5` ใช้การคิดแบบปรับเปลี่ยนได้เสมอและใช้ระดับความพยายาม `high`
เป็นค่าเริ่มต้น Anthropic ไม่อนุญาตให้ปิดใช้การคิดสำหรับโมเดลนี้ ดังนั้น
`/think off` และ `/think minimal` จึงแมปไปยังระดับความพยายาม `low` แทน นอกจากนี้ OpenClaw
ยังไม่ส่งค่าอุณหภูมิแบบกำหนดเองสำหรับคำขอ Fable 5 เนื่องจาก Anthropic ปฏิเสธ
การแทนที่ค่าอุณหภูมิในคำขอใดก็ตามที่เปิดใช้การคิด

`anthropic/claude-mythos-5` เป็นโมเดลที่มีการจำกัดการเข้าถึงและใช้ข้อกำหนด
การคิดแบบปรับเปลี่ยนได้ที่เปิดใช้งานตลอดเวลาเหมือนกัน OpenClaw ใช้ `high` เป็นค่าเริ่มต้น แมป `/think off` และ
`/think minimal` ไปยัง `low` และไม่ส่งพารามิเตอร์การสุ่มตัวอย่างที่ผู้เรียกเลือก
แค็ตตาล็อกระบุหน้าต่างบริบทขนาด 1,000,000 โทเค็น ขีดจำกัดเอาต์พุต
128,000 โทเค็น การรับอินพุตรูปภาพ และราคาอินพุต/เอาต์พุต `$10/$50`

Claude Opus 4.8 ปิดการคิดไว้ตามค่าเริ่มต้นใน OpenClaw เมื่อเปิดใช้
การคิดแบบปรับเปลี่ยนได้อย่างชัดเจนด้วย `/think high|xhigh|max` OpenClaw จะส่ง
ค่าระดับความพยายามของ Opus 4.8 จาก Anthropic ส่วนโมเดล Claude 4.6 (Opus 4.6 และ Sonnet 4.6)
ใช้ `adaptive` เป็นค่าเริ่มต้น

แทนค่าต่อข้อความด้วย `/think:<level>` หรือกำหนดในพารามิเตอร์ของโมเดล:

```json5
{
  agents: {
    defaults: {
      models: {
        "anthropic/claude-opus-4-8": {
          params: { thinking: "high" },
        },
      },
    },
  },
}
```

<Note>
เอกสาร Anthropic ที่เกี่ยวข้อง:
- [การคิดแบบปรับเปลี่ยนได้](https://platform.claude.com/docs/en/build-with-claude/adaptive-thinking)
- [การคิดแบบขยาย](https://platform.claude.com/docs/en/build-with-claude/extended-thinking)

</Note>

## การสำรองเมื่อปฏิเสธด้วยเหตุผลด้านความปลอดภัย (Claude Fable 5)

<Warning>
การใช้ Claude Fable 5 หมายถึงการใช้ Claude Opus 4.8 ด้วย Fable 5 มาพร้อมกับ
ตัวจำแนกด้านความปลอดภัยที่สามารถปฏิเสธคำขอได้ และวิธีการกู้คืนที่ Anthropic รับรอง
คือให้ `claude-opus-4-8` ให้บริการคำขอรอบนั้น OpenClaw เลือกใช้กลไกนี้
โดยอัตโนมัติสำหรับคำขอที่ใช้คีย์ API โดยตรง ดังนั้นคำขอบางรอบของ Fable จึงได้รับคำตอบ
และเรียกเก็บค่าบริการในฐานะ Claude Opus 4.8 หากนโยบายหรืองบประมาณไม่สามารถยอมรับ
คำขอรอบที่ให้บริการโดย Opus ได้ อย่าเลือก `anthropic/claude-fable-5`
</Warning>

### เหตุผลที่มีกลไกนี้

ตัวจำแนกของ Fable 5 ส่งคืน `stop_reason: "refusal"` สำหรับคำขอในโดเมน
ที่ถูกจำกัด และยังเกิดผลบวกลวงกับงานที่ไม่เป็นอันตรายแต่ใกล้เคียงกับขอบเขตดังกล่าว
(เครื่องมือด้านความปลอดภัย วิทยาศาสตร์ชีวภาพ หรือแม้แต่การขอให้โมเดลสร้าง
กระบวนการให้เหตุผลดิบของตนซ้ำ) หากไม่มีโมเดลสำรอง คำขอรอบนั้นจะสิ้นสุดด้วยข้อผิดพลาด
แม้ว่าโมเดล Claude อื่นจะสามารถให้บริการได้ก็ตาม โดยข้อความปฏิเสธของ Anthropic เอง
จะแจ้งให้ผู้ผสานรวม API กำหนดค่าโมเดลสำรอง

### วิธีการทำงาน

1. สำหรับทุกคำขอที่ใช้คีย์ API โดยตรงไปยัง `anthropic/claude-fable-5` OpenClaw
   จะส่งการเลือกใช้โมเดลสำรองฝั่งเซิร์ฟเวอร์ของ Anthropic ได้แก่
   ส่วนหัวเบตา `server-side-fallback-2026-06-01` พร้อมกับ
   `fallbacks: [{"model": "claude-opus-4-8"}]` Claude Opus 4.8 เป็นเป้าหมาย
   สำรองเพียงรายการเดียวที่ Anthropic อนุญาตสำหรับ Fable 5
2. เฉพาะการปฏิเสธจากตัวจำแนกด้านความปลอดภัยเท่านั้นที่จะเรียกใช้โมเดลสำรอง ขีดจำกัดอัตรา
   ภาวะโหลดเกิน และข้อผิดพลาดของเซิร์ฟเวอร์จะทำงานเหมือนเดิมทุกประการและผ่าน
   [การสลับโมเดลเมื่อขัดข้อง](/th/concepts/model-failover) ตามปกติของ OpenClaw
3. การกู้คืนเกิดขึ้นภายในการเรียกเดียวกัน การปฏิเสธก่อนมีเอาต์พุตใด ๆ
   จะสังเกตไม่เห็นนอกจากเวลาแฝง โดยคำตอบทั้งหมดมาจาก Opus 4.8 หากปฏิเสธ
   ระหว่างการสตรีม ข้อความบางส่วนจะถูกเก็บไว้เป็นคำนำหน้าที่โมเดลสำรอง
   ใช้เขียนต่อ ขณะที่การให้เหตุผลและการเรียกเครื่องมือของโมเดลที่ปฏิเสธ
   จะถูกละทิ้งตามกฎการเล่นซ้ำของ Anthropic (ห้ามส่งข้อมูลเหล่านั้นกลับหรือ
   ดำเนินการ)
4. หาก Claude Opus 4.8 ปฏิเสธด้วย คำขอรอบนั้นจะแสดงการปฏิเสธเป็น
   ข้อผิดพลาดเหมือนก่อนมีคุณลักษณะนี้ทุกประการ

การสำรองเกิดขึ้นที่ระดับ API ของ Anthropic ดังนั้น `claude-opus-4-8` จึงไม่
จำเป็นต้องอยู่ในรายการโมเดลหรือสายโมเดลสำรองที่กำหนดค่าไว้ คีย์ API
ที่รองรับ Fable สามารถให้บริการ Opus ได้เสมอ

### การสังเกตการณ์และการเรียกเก็บเงิน

- คำขอรอบที่ให้บริการโดยโมเดลสำรองจะบันทึกข้อมูลวินิจฉัย `provider_fallback` ใน
  ข้อความของผู้ช่วย โดยระบุ `fromModel` และ `toModel` และ
  `responseModel` ของข้อความจะรายงาน `claude-opus-4-8`
- Anthropic เรียกเก็บเงินต่อความพยายาม: การปฏิเสธก่อนมีเอาต์พุตไม่มีค่าใช้จ่าย และการกู้คืน
  จะเรียกเก็บตามอัตราของ Claude Opus 4.8 (ปัจจุบันเป็นครึ่งหนึ่งของอัตรา Fable 5) การประมาณ
  ค่าใช้จ่ายต่อรอบของ OpenClaw จะคิดราคาคำขอรอบที่ให้บริการโดยโมเดลสำรองตามอัตราของ Opus
  ให้สอดคล้องกัน
- การปฏิเสธระหว่างการสตรีมจะเรียกเก็บเงินเพิ่มเติมสำหรับส่วนของ Fable ที่สตรีมไปแล้ว
  ทางฝั่ง Anthropic โดยส่วนดังกล่าวจะรายงานในการใช้งานต่อความพยายามของ API
  แต่จะไม่รวมอยู่ในการประมาณค่าใช้จ่ายต่อรอบของ OpenClaw

### ขอบเขต

ใช้กับ `anthropic/claude-fable-5` ที่ยืนยันตัวตนด้วยคีย์ API กับ
`api.anthropic.com` คำขอผ่าน OAuth (การใช้การสมัครสมาชิก Claude CLI ซ้ำ), URL ฐานของพร็อกซี,
Bedrock, Vertex และ Foundry จะไม่เปลี่ยนแปลงและยังคงแสดง
การปฏิเสธเป็นข้อผิดพลาดในระบบเหล่านั้น

ตรวจสอบแบบสดแล้ว: พรอมต์ที่ไม่เป็นอันตรายซึ่งขอให้ Fable 5 สร้างลำดับ
ความคิดดิบของตนซ้ำจะถูกปฏิเสธด้วย `category: "reasoning_extraction"` เมื่อส่งโดยไม่มี
โมเดลสำรอง ส่วนพรอมต์เดียวกันที่ส่งผ่าน OpenClaw จะได้รับคำตอบตามปกติจาก Opus
พร้อมแนบข้อมูลวินิจฉัย `provider_fallback`

ดูพฤติกรรมพื้นฐานได้จาก[คู่มือการปฏิเสธและการใช้โมเดลสำรอง
ของ Anthropic](https://platform.claude.com/docs/en/build-with-claude/refusals-and-fallback)

## การแคชพรอมต์

OpenClaw รองรับคุณลักษณะการแคชพรอมต์ของ Anthropic สำหรับการยืนยันตัวตนด้วยคีย์ API

| ค่า               | ระยะเวลาแคช | คำอธิบาย                            |
| ------------------- | -------------- | -------------------------------------- |
| `"short"` (ค่าเริ่มต้น) | 5 นาที      | ใช้โดยอัตโนมัติสำหรับการยืนยันตัวตนด้วยคีย์ API |
| `"long"`            | 1 ชั่วโมง         | แคชแบบขยาย                         |
| `"none"`            | ไม่มีการแคช     | ปิดใช้การแคชพรอมต์                 |

```json5
{
  agents: {
    defaults: {
      models: {
        "anthropic/claude-opus-4-6": {
          params: { cacheRetention: "long" },
        },
      },
    },
  },
}
```

<AccordionGroup>
  <Accordion title="การแทนค่าการแคชต่อเอเจนต์">
    ใช้พารามิเตอร์ระดับโมเดลเป็นค่าพื้นฐาน แล้วแทนค่าสำหรับเอเจนต์ที่ต้องการผ่าน `agents.list[].params`:

    ```json5
    {
      agents: {
        defaults: {
          model: { primary: "anthropic/claude-opus-4-6" },
          models: {
            "anthropic/claude-opus-4-6": {
              params: { cacheRetention: "long" },
            },
          },
        },
        list: [
          { id: "research", default: true },
          { id: "alerts", params: { cacheRetention: "none" } },
        ],
      },
    }
    ```

    ลำดับการผสานการกำหนดค่า:

    1. `agents.defaults.models["provider/model"].params`
    2. `agents.list[].params` (เมื่อ `id` ตรงกัน จะเขียนทับตามคีย์)

    วิธีนี้ช่วยให้เอเจนต์หนึ่งเก็บแคชที่มีอายุยาวไว้ได้ ขณะที่เอเจนต์อีกตัวซึ่งใช้โมเดลเดียวกันปิดใช้งานการแคชสำหรับทราฟฟิกแบบเป็นช่วงหรือมีการนำกลับมาใช้ซ้ำน้อย

  </Accordion>

  <Accordion title="หมายเหตุเกี่ยวกับ Bedrock Claude">
    - โมเดล Anthropic Claude บน Bedrock (`amazon-bedrock/*anthropic.claude*`) ยอมรับการส่งผ่าน `cacheRetention` เมื่อกำหนดค่าไว้
    - โมเดล Bedrock ที่ไม่ใช่ Anthropic จะถูกบังคับให้เป็น `cacheRetention: "none"` ขณะรันไทม์
    - ค่าเริ่มต้นอัจฉริยะสำหรับคีย์ API จะตั้งค่าเริ่มต้น `cacheRetention: "short"` ให้กับการอ้างอิง Claude-on-Bedrock ด้วย เมื่อไม่ได้ระบุค่าไว้อย่างชัดเจน

  </Accordion>
</AccordionGroup>

## การกำหนดค่าขั้นสูง

<AccordionGroup>
  <Accordion title="โหมดเร็ว">
    ตัวสลับ `/fast` ที่ใช้ร่วมกันของ OpenClaw จะตั้งค่าฟิลด์ `service_tier` ของ Anthropic สำหรับทราฟฟิกที่ส่งตรงโดยใช้คีย์ API เป็น `api.anthropic.com`

    | คำสั่ง | จับคู่กับ |
    |---------|---------|
    | `/fast on` | `service_tier: "auto"` |
    | `/fast off` | `service_tier: "standard_only"` |

    ```json5
    {
      agents: {
        defaults: {
          models: {
            "anthropic/claude-sonnet-4-6": {
              params: { fastMode: true },
            },
          },
        },
      },
    }
    ```

    <Note>
    - มีผลเฉพาะกับคำขอ `api.anthropic.com` โดยตรงที่ส่งด้วยคีย์ API เท่านั้น คำขอที่ใช้ OAuth/โทเค็นการสมัครใช้งานและเส้นทางพร็อกซีจะไม่มีฟิลด์ `service_tier`
    - พารามิเตอร์ `serviceTier` หรือ `service_tier` ที่ระบุอย่างชัดเจนจะเขียนทับ `/fast` เมื่อตั้งค่าทั้งคู่
    - สำหรับบัญชีที่ไม่มีความจุ Priority Tier ค่า `service_tier: "auto"` อาจถูกแปลงเป็น `standard`

    </Note>

  </Accordion>

  <Accordion title="การทำความเข้าใจสื่อ (รูปภาพและ PDF)">
    Plugin Anthropic ที่รวมมาให้จะลงทะเบียนความสามารถในการทำความเข้าใจรูปภาพและ PDF โดย OpenClaw
    จะตรวจหาความสามารถด้านสื่อโดยอัตโนมัติจากการยืนยันตัวตน Anthropic ที่กำหนดค่าไว้
    ไม่จำเป็นต้องกำหนดค่าเพิ่มเติม

    | คุณสมบัติ        | ค่า                 |
    | --------------- | --------------------- |
    | โมเดลเริ่มต้น   | `claude-opus-4-8`     |
    | อินพุตที่รองรับ | รูปภาพ, เอกสาร PDF |

    เมื่อแนบรูปภาพหรือ PDF กับการสนทนา OpenClaw จะกำหนดเส้นทาง
    ผ่านผู้ให้บริการทำความเข้าใจสื่อของ Anthropic โดยอัตโนมัติ

  </Accordion>

  <Accordion title="หน้าต่างบริบท 1M">
    Claude Sonnet 5, Mythos 5 และ Fable 5 มีหน้าต่างอินพุตขนาด
    1,000,000 โทเค็นพอดี และรองรับเอาต์พุตสูงสุด 128,000 โทเค็น หน้าต่างบริบท 1M
    ของ Anthropic พร้อมใช้งานทั่วไปบนโมเดล Claude 4.x ที่มีการคิดแบบปรับตัวได้ด้วย ได้แก่ Opus 4.8,
    Opus 4.7, Opus 4.6 และ Sonnet 4.6 OpenClaw กำหนดขนาดโมเดลเหล่านี้
    โดยอัตโนมัติ จึงไม่ต้องใช้ `params.context1m`:

    ```json5
    {
      agents: {
        defaults: {
          models: {
            "anthropic/claude-sonnet-5": {},
            "anthropic/claude-mythos-5": {},
            "anthropic/claude-opus-4-6": {},
          },
        },
      },
    }
    ```

    การกำหนดค่าเก่ายังคงใช้ `params.context1m: true` ได้ ซึ่งจะไม่ทำอะไรและไม่ก่อให้เกิดผลเสียสำหรับ
    โมเดลเหล่านี้ และ OpenClaw จะไม่ส่งส่วนหัวเบต้าที่เลิกใช้แล้ว
    `context-1m-2025-08-07` อีกต่อไปไม่ว่าในกรณีใด รายการกำหนดค่า `anthropicBeta` รุ่นเก่า
    ที่มีค่านี้จะถูกตัดทิ้งระหว่างการประมวลผลส่วนหัวของคำขอ และ
    โมเดล Claude รุ่นเก่าที่ไม่รองรับจะยังคงใช้หน้าต่างบริบทปกติ

    `params.context1m: true` ทำงานในลักษณะเดียวกันสำหรับแบ็กเอนด์ Claude CLI
    (`claude-cli/*`): โมเดล Opus และ Sonnet ที่มีความสามารถ GA และเข้าเกณฑ์จะได้รับ
    หน้าต่าง 1M โดยอัตโนมัติอยู่แล้ว ดังนั้นพารามิเตอร์นี้จึงเป็นทางเลือกสำหรับแบ็กเอนด์ดังกล่าวเช่นกัน

    <Warning>
    ต้องมีสิทธิ์เข้าถึงบริบทยาวในข้อมูลรับรอง Anthropic ของคุณ การยืนยันตัวตนด้วย OAuth/โทเค็นการสมัครใช้งานจะคงส่วนหัวเบต้าของ Anthropic ที่จำเป็นไว้ แต่ OpenClaw จะนำส่วนหัวเบต้า 1M ที่เลิกใช้แล้วออก หากยังเหลืออยู่ในการกำหนดค่าเก่า
    </Warning>

  </Accordion>

  <Accordion title="บริบท 1M ของ Claude Opus 4.8">
    `anthropic/claude-opus-4-8` และตัวแปร `claude-cli` มีหน้าต่างบริบท
    1M เป็นค่าเริ่มต้น จึงไม่ต้องใช้ `params.context1m: true`
  </Accordion>
</AccordionGroup>

## การแก้ไขปัญหา

<AccordionGroup>
  <Accordion title="ข้อผิดพลาด 401 / โทเค็นใช้ไม่ได้กะทันหัน">
    การยืนยันตัวตนด้วยโทเค็น Anthropic จะหมดอายุและอาจถูกเพิกถอนได้ สำหรับการตั้งค่าใหม่ ให้ใช้คีย์ API ของ Anthropic แทน
  </Accordion>

  <Accordion title='ไม่พบคีย์ API สำหรับผู้ให้บริการ "anthropic"'>
    การยืนยันตัวตน Anthropic เป็นแบบ **ต่อเอเจนต์** โดยเอเจนต์ใหม่จะไม่สืบทอดคีย์ของเอเจนต์หลัก เรียกใช้การเริ่มต้นใช้งานอีกครั้งสำหรับเอเจนต์นั้น (หรือกำหนดค่าคีย์ API บนโฮสต์ Gateway) แล้วตรวจสอบด้วย `openclaw models status`
  </Accordion>

  <Accordion title='ไม่พบข้อมูลรับรองสำหรับโปรไฟล์ "anthropic:default"'>
    เรียกใช้ `openclaw models status` เพื่อดูว่าโปรไฟล์การยืนยันตัวตนใดกำลังใช้งานอยู่ เรียกใช้การเริ่มต้นใช้งานอีกครั้ง หรือกำหนดค่าคีย์ API สำหรับพาธของโปรไฟล์นั้น
  </Accordion>

  <Accordion title="ไม่มีโปรไฟล์การตรวจสอบสิทธิ์ที่พร้อมใช้งาน (ทั้งหมดอยู่ในช่วงพักการใช้งาน)">
    ตรวจสอบ `openclaw models status --json` สำหรับ `auth.unusableProfiles` ช่วงพักการใช้งานจากการจำกัดอัตราของ Anthropic อาจกำหนดเฉพาะแต่ละโมเดล ดังนั้นโมเดล Anthropic อื่นในกลุ่มเดียวกันอาจยังใช้งานได้ เพิ่มโปรไฟล์ Anthropic อื่นหรือรอให้ช่วงพักการใช้งานสิ้นสุด
  </Accordion>
</AccordionGroup>
