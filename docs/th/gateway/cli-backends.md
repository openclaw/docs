OpenClaw สามารถเรียกใช้ AI CLI ภายในเครื่องเป็นตัวเลือกสำรองแบบข้อความเท่านั้น เมื่อผู้ให้บริการ API หยุดทำงาน ถูกจำกัดอัตรา หรือทำงานผิดปกติ โดยออกแบบให้ระมัดระวังโดยเจตนา:

- เครื่องมือของ OpenClaw จะไม่ถูกแทรกโดยตรง แต่แบ็กเอนด์ที่มี `bundleMcp: true` สามารถรับเครื่องมือ Gateway ผ่านบริดจ์ MCP แบบลูปแบ็กได้
- การสตรีม JSONL สำหรับ CLI ที่รองรับ
- รองรับเซสชัน ดังนั้นการโต้ตอบต่อเนื่องจึงยังคงสอดคล้องกัน
- รูปภาพจะถูกส่งผ่านหาก CLI ยอมรับพาธรูปภาพ

ใช้เป็นตาข่ายนิรภัยสำหรับการตอบกลับแบบข้อความที่ "ใช้งานได้เสมอ" ไม่ใช่เส้นทางหลัก สำหรับรันไทม์ฮาร์เนสแบบเต็มที่มีการควบคุมเซสชัน ACP, งานเบื้องหลัง, การผูกเธรด/การสนทนา และเซสชันการเขียนโค้ดภายนอกแบบถาวร ให้ใช้ [เอเจนต์ ACP](/th/tools/acp-agents) แทน เนื่องจากแบ็กเอนด์ CLI ไม่ใช่ ACP

<Tip>
  กำลังสร้าง Plugin แบ็กเอนด์ใหม่อยู่หรือไม่ ดู [Plugin แบ็กเอนด์ CLI](/th/plugins/cli-backend-plugins) หน้านี้ครอบคลุมการกำหนดค่าและการใช้งานแบ็กเอนด์ที่ลงทะเบียนไว้แล้ว
</Tip>

## เริ่มต้นอย่างรวดเร็ว

Plugin Anthropic ที่มาพร้อมระบบจะลงทะเบียนแบ็กเอนด์ `claude-cli` เริ่มต้น จึงใช้งานได้โดยไม่ต้องกำหนดค่าเพิ่มเติม นอกเหนือจากการติดตั้งและเข้าสู่ระบบ Claude Code แล้ว:

```bash
openclaw agent --agent main --message "hi" --model claude-cli/claude-sonnet-4-6
```

`main` คือรหัสเอเจนต์เริ่มต้นเมื่อไม่ได้กำหนดรายการเอเจนต์อย่างชัดเจน มิฉะนั้นให้เปลี่ยนเป็นรหัสเอเจนต์ของคุณเอง

หาก Gateway ทำงานภายใต้ launchd/systemd โดยมี `PATH` แบบจำกัด ให้ระบุไบนารีอย่างชัดเจน:

```json5
{
  agents: {
    defaults: {
      cliBackends: {
        "claude-cli": {
          command: "/opt/homebrew/bin/claude",
        },
      },
    },
  },
}
```

หากใช้แบ็กเอนด์ CLI ที่มาพร้อมระบบเป็นผู้ให้บริการข้อความหลักบนโฮสต์ Gateway OpenClaw จะโหลด Plugin ที่มาพร้อมระบบซึ่งเป็นเจ้าของโดยอัตโนมัติ เมื่อการกำหนดค่าของคุณอ้างอิงแบ็กเอนด์นั้นในข้อมูลอ้างอิงโมเดลหรือภายใต้ `agents.defaults.cliBackends`

## การใช้เป็นตัวเลือกสำรอง

เพิ่มแบ็กเอนด์ CLI ลงในรายการตัวเลือกสำรอง เพื่อให้ทำงานเฉพาะเมื่อโมเดลหลักล้มเหลว:

```json5
{
  agents: {
    defaults: {
      model: {
        primary: "anthropic/claude-opus-4-6",
        fallbacks: ["claude-cli/claude-sonnet-4-6"],
      },
      models: {
        "anthropic/claude-opus-4-6": { alias: "Opus" },
        "claude-cli/claude-sonnet-4-6": {},
      },
    },
  },
}
```

หากใช้ `agents.defaults.models` เป็นรายการอนุญาต ให้รวมโมเดลแบ็กเอนด์ CLI ของคุณไว้ด้วย เมื่อผู้ให้บริการหลักล้มเหลว (การยืนยันตัวตน การจำกัดอัตรา หรือหมดเวลา) OpenClaw จะลองใช้แบ็กเอนด์ CLI เป็นลำดับถัดไป

## การกำหนดค่า

แบ็กเอนด์ CLI ทั้งหมดอยู่ภายใต้ `agents.defaults.cliBackends` โดยใช้รหัสผู้ให้บริการเป็นคีย์ (เช่น `claude-cli`, `my-cli`) รหัสผู้ให้บริการจะกลายเป็นด้านซ้ายของข้อมูลอ้างอิงโมเดล: `<provider>/<model>`

```json5
{
  agents: {
    defaults: {
      cliBackends: {
        "my-cli": {
          command: "my-cli",
          args: ["--json"],
          output: "json",
          input: "arg",
          modelArg: "--model",
          modelAliases: {
            "claude-opus-4-6": "opus",
            "claude-sonnet-4-6": "sonnet",
          },
          sessionArg: "--session",
          sessionMode: "existing",
          sessionIdFields: ["session_id", "conversation_id"],
          systemPromptArg: "--system",
          // แฟล็กไฟล์พรอมต์เฉพาะ:
          // systemPromptFileArg: "--system-file",
          // หรือใช้แฟล็กเขียนทับการกำหนดค่าแบบ Codex:
          // systemPromptFileConfigArg: "-c",
          // systemPromptFileConfigKey: "model_instructions_file",
          systemPromptWhen: "first",
          imageArg: "--image",
          imageMode: "repeat",
          // เลือกใช้เฉพาะเมื่อแบ็กเอนด์นี้อาจเติมข้อมูลเริ่มต้นให้เซสชันที่ถูกทำให้ใช้ไม่ได้อีกครั้ง
          // จากประวัติทรานสคริปต์ดิบของ OpenClaw ที่มีขอบเขต ก่อน Compaction
          reseedFromRawTranscriptWhenUncompacted: true,
          serialize: true,
        },
      },
    },
  },
}
```

## วิธีการทำงาน

1. เลือกแบ็กเอนด์ตามคำนำหน้าผู้ให้บริการ (`claude-cli/...`)
2. สร้างพรอมต์ระบบโดยใช้พรอมต์และบริบทเวิร์กสเปซเดียวกันของ OpenClaw
3. เรียกใช้ CLI พร้อมรหัสเซสชัน (หากรองรับ) เพื่อให้ประวัติคงความสอดคล้อง แบ็กเอนด์ `claude-cli` ที่มาพร้อมระบบจะคงกระบวนการ stdio ของ Claude ไว้ต่อหนึ่งเซสชัน OpenClaw และส่งการโต้ตอบต่อเนื่องผ่าน stdin แบบ stream-json
4. แยกวิเคราะห์เอาต์พุต (JSON หรือข้อความธรรมดา) และส่งคืนข้อความสุดท้าย
5. จัดเก็บรหัสเซสชันต่อแบ็กเอนด์ เพื่อให้การโต้ตอบต่อเนื่องใช้เซสชัน CLI เดิมซ้ำ

### รายละเอียดเฉพาะของ Claude CLI

แบ็กเอนด์ `claude-cli` ที่มาพร้อมระบบจะให้ความสำคัญกับตัวแก้ไข Skills แบบเนทีฟของ Claude Code เมื่อสแนปช็อต Skills ปัจจุบันมี Skills ที่เลือกอย่างน้อยหนึ่งรายการซึ่งมีพาธที่สร้างขึ้นแล้ว OpenClaw จะส่ง Plugin Claude Code ชั่วคราวผ่าน `--plugin-dir` และละเว้นแค็ตตาล็อก Skills ของ OpenClaw ที่ซ้ำกันจากพรอมต์ระบบที่ต่อท้าย หากไม่มี Skills ของ Plugin ที่สร้างขึ้น OpenClaw จะคงแค็ตตาล็อกพรอมต์ไว้เป็นตัวเลือกสำรอง การเขียนทับสภาพแวดล้อม/คีย์ API ของ Skills ยังคงมีผลกับสภาพแวดล้อมของกระบวนการลูกสำหรับการเรียกใช้งาน

Claude CLI มีโหมดสิทธิ์แบบไม่โต้ตอบของตนเอง OpenClaw จะแมปโหมดดังกล่าวเข้ากับนโยบายการดำเนินการที่มีอยู่ แทนที่จะเพิ่มการกำหนดค่าเฉพาะ Claude สำหรับเซสชัน Claude แบบสดที่ OpenClaw จัดการ นโยบายการดำเนินการที่มีผลถือเป็นข้อกำหนดสูงสุด: โดยปกติ YOLO (`tools.exec.security: "full"` และ `tools.exec.ask: "off"`) จะเปิด Claude ด้วย `--permission-mode bypassPermissions` ขณะที่นโยบายแบบจำกัดจะเปิดด้วย `--permission-mode default` นอกจากนี้ Gateway ที่ทำงานด้วย root จะใช้ `default` เนื่องจาก Claude Code ปฏิเสธโหมดข้ามข้อจำกัดสำหรับ root แต่ OpenClaw ยังคงตอบคำขอควบคุมเครื่องมือผ่าน stdio ของ Claude ตามนโยบายการดำเนินการที่กำหนดไว้ การตั้งค่า `agents.list[].tools.exec` ต่อเอเจนต์จะแทนที่ `tools.exec` ส่วนกลางสำหรับเอเจนต์นั้น อาร์กิวเมนต์แบ็กเอนด์ดิบอาจยังมี `--permission-mode` แต่การเปิด Claude แบบสดจะปรับแฟล็กนั้นให้ตรงกับนโยบายที่มีผลและข้อจำกัดของโฮสต์

แบ็กเอนด์ยังแมประดับ `/think` ของ OpenClaw ไปยังแฟล็ก `--effort` แบบเนทีฟของ Claude Code: `minimal`/`low` -> `low`, `medium` -> `medium` และ `high`/`xhigh`/`max` จะถูกส่งผ่านโดยตรง ซึ่งทำให้ระดับ effort ของ Fable 5 ที่รองรับเหมือนกันทั้ง Claude CLI ที่อิงการสมัครสมาชิกและเส้นทางคีย์ API `adaptive` จะนำแฟล็ก `--effort` ที่กำหนดค่าไว้ออกและไม่ระบุค่าแทน เพื่อให้ Claude Code กำหนด effort ที่มีผลจากสภาพแวดล้อม การตั้งค่า และค่าเริ่มต้นของโมเดลเอง แบ็กเอนด์ CLI อื่นต้องให้ Plugin เจ้าของประกาศตัวแมป argv ที่เทียบเท่าก่อนที่ `/think` จะส่งผลต่อ CLI ที่ถูกเรียกใช้

ก่อนที่ OpenClaw จะใช้ `claude-cli` ได้ Claude Code ต้องเข้าสู่ระบบบนโฮสต์เดียวกันก่อน:

```bash
claude auth login
claude auth status --text
openclaw models auth login --provider anthropic --method cli --set-default
```

การติดตั้งผ่าน Docker จำเป็นต้องติดตั้งและเข้าสู่ระบบ Claude Code ภายในโฮมของคอนเทนเนอร์ที่จัดเก็บแบบถาวร ไม่ใช่เฉพาะบนโฮสต์ ดู [แบ็กเอนด์ Claude CLI ใน Docker](/th/install/docker#claude-cli-backend-in-docker)

ตั้งค่า `agents.defaults.cliBackends.claude-cli.command` เฉพาะเมื่อไบนารี `claude` ยังไม่อยู่ใน `PATH`

## เซสชัน

- หาก CLI รองรับเซสชัน ให้ตั้งค่า `sessionArg` (เช่น `--session-id`) หรือ `sessionArgs` (ตัวยึดตำแหน่ง `{sessionId}`) เมื่อจำเป็นต้องใส่รหัสในหลายแฟล็ก
- หาก CLI ใช้คำสั่งย่อยสำหรับดำเนินการต่อพร้อมแฟล็กที่แตกต่างกัน ให้ตั้งค่า `resumeArgs` (แทนที่ `args` เมื่อดำเนินการต่อ) และอาจตั้งค่า `resumeOutput` สำหรับการดำเนินการต่อที่ไม่ใช่ JSON
- `sessionMode`:
  - `always`: ส่งรหัสเซสชันเสมอ (UUID ใหม่หากยังไม่ได้จัดเก็บ)
  - `existing`: ส่งรหัสเซสชันเฉพาะเมื่อเคยจัดเก็บไว้ก่อนหน้า
  - `none`: ไม่ส่งรหัสเซสชัน
- `claude-cli` มีค่าเริ่มต้นเป็น `liveSession: "claude-stdio"`, `output: "jsonl"` และ `input: "stdin"` ดังนั้นการโต้ตอบต่อเนื่องจะใช้กระบวนการ Claude แบบสดซ้ำขณะที่ยังทำงานอยู่ รวมถึงการกำหนดค่าแบบกำหนดเองที่ละเว้นฟิลด์การขนส่ง หาก Gateway รีสตาร์ตหรือกระบวนการที่ไม่ได้ใช้งานสิ้นสุดลง OpenClaw จะดำเนินการต่อจากรหัสเซสชัน Claude ที่จัดเก็บไว้ รหัสเซสชันที่จัดเก็บไว้จะถูกตรวจสอบกับทรานสคริปต์โปรเจกต์ที่อ่านได้ก่อนดำเนินการต่อ หากไม่มีทรานสคริปต์ การผูกจะถูกล้าง (บันทึกเป็น `reason=transcript-missing`) แทนที่จะเริ่มเซสชันใหม่อย่างเงียบ ๆ ภายใต้ `--resume`
- เซสชัน Claude แบบสดจะใช้ขีดจำกัดเอาต์พุต JSONL: ค่าเริ่มต้นคือ 8 MiB และ 20,000 บรรทัด JSONL ดิบต่อการโต้ตอบ เพิ่มขีดจำกัดต่อแบ็กเอนด์ได้ด้วย `agents.defaults.cliBackends.claude-cli.reliability.outputLimits.maxTurnRawChars` และ `maxTurnLines`; OpenClaw จะจำกัดการตั้งค่าเหล่านั้นไว้ไม่เกิน 64 MiB และ 100,000 บรรทัด
- เซสชัน CLI ที่จัดเก็บไว้เป็นความต่อเนื่องที่ผู้ให้บริการเป็นเจ้าของ การรีเซ็ตเซสชันรายวันโดยปริยายจะไม่ตัดเซสชันเหล่านี้ แต่ `/reset` และนโยบาย `session.reset` ที่กำหนดอย่างชัดเจนยังคงตัดเซสชัน
- โดยปกติเซสชัน CLI ใหม่จะเติมข้อมูลเริ่มต้นอีกครั้งเฉพาะจากข้อมูลสรุป Compaction ของ OpenClaw รวมกับส่วนท้ายหลัง Compaction หากต้องการกู้คืนเซสชันสั้นที่ถูกทำให้ใช้ไม่ได้ก่อน Compaction แบ็กเอนด์สามารถเลือกใช้ `reseedFromRawTranscriptWhenUncompacted: true` ได้ การเติมข้อมูลจากทรานสคริปต์ดิบยังคงมีขอบเขตและจำกัดเฉพาะการทำให้ใช้ไม่ได้ที่ปลอดภัย เช่น ไม่มีทรานสคริปต์ CLI, ส่วนท้ายการใช้เครื่องมือที่ขาดคู่, การเปลี่ยนแปลงนโยบายข้อความ/พรอมต์ระบบ/cwd/MCP หรือการลองใหม่หลังเซสชันหมดอายุ ส่วนการเปลี่ยนโปรไฟล์การยืนยันตัวตนหรือยุคของข้อมูลประจำตัวจะไม่เติมข้อมูลจากประวัติทรานสคริปต์ดิบอีกครั้ง

การทำให้เป็นลำดับ: `serialize: true` จะรักษาลำดับการเรียกใช้ในเลนเดียวกัน (CLI ส่วนใหญ่ทำงานเป็นลำดับบนเลนผู้ให้บริการเดียว) OpenClaw ยังยกเลิกการใช้เซสชัน CLI ที่จัดเก็บไว้ซ้ำเมื่อข้อมูลระบุตัวตนสำหรับการยืนยันตัวตนที่เลือกเปลี่ยนแปลง รวมถึงรหัสโปรไฟล์การยืนยันตัวตนที่เปลี่ยน คีย์ API แบบคงที่ โทเค็นแบบคงที่ หรือข้อมูลระบุตัวตนบัญชี OAuth เมื่อ CLI เปิดเผยข้อมูลดังกล่าว การหมุนเวียนโทเค็นการเข้าถึง/รีเฟรช OAuth เพียงอย่างเดียวจะไม่ตัดเซสชัน หาก CLI ไม่มีรหัสบัญชี OAuth ที่เสถียร OpenClaw จะให้ CLI นั้นบังคับใช้สิทธิ์การดำเนินการต่อของตนเอง

## บทนำสำหรับการสำรองข้อมูลจากเซสชัน claude-cli

เมื่อความพยายาม `claude-cli` สลับไปยังตัวเลือกที่ไม่ใช่ CLI ใน [`agents.defaults.model.fallbacks`](/th/concepts/model-failover) OpenClaw จะเติมข้อมูลเริ่มต้นให้ความพยายามถัดไปด้วยบทนำบริบทที่รวบรวมจากทรานสคริปต์ JSONL ภายในเครื่องของ Claude Code (ภายใต้ `~/.claude/projects/` โดยแยกคีย์ตามเวิร์กสเปซ) หากไม่มีข้อมูลเริ่มต้นนี้ ผู้ให้บริการสำรองจะเริ่มต้นโดยไม่มีบริบท เนื่องจากทรานสคริปต์เซสชันของ OpenClaw เองว่างเปล่าสำหรับการเรียกใช้ `claude-cli`

- บทนำจะให้ความสำคัญกับข้อมูลสรุป `/compact` หรือเครื่องหมาย `compact_boundary` ล่าสุด จากนั้นต่อท้ายการโต้ตอบหลังขอบเขตล่าสุดจนถึงงบประมาณจำนวนอักขระ การโต้ตอบก่อนขอบเขตจะถูกละทิ้ง เนื่องจากข้อมูลสรุปได้แสดงเนื้อหาเหล่านั้นแล้ว
- บล็อกเครื่องมือจะถูกรวมเป็นคำใบ้ `(tool call: name)` และ `(tool result: …)` แบบกระชับ เพื่อรักษางบประมาณพรอมต์ให้ตรงตามจริง ส่วนข้อมูลสรุปที่ใหญ่เกินไปจะถูกตัดทอนและติดป้าย `(truncated)`
- ตัวเลือกสำรองจาก `claude-cli` ไปยัง `claude-cli` ภายในผู้ให้บริการเดียวกันจะพึ่งพา `--resume` ของ Claude เองและข้ามบทนำ
- ข้อมูลเริ่มต้นใช้การตรวจสอบพาธไฟล์เซสชัน Claude ที่มีอยู่ซ้ำ จึงไม่สามารถอ่านพาธใด ๆ โดยพลการได้

## รูปภาพ

หาก CLI ของคุณยอมรับพาธรูปภาพ ให้ตั้งค่า `imageArg`:

```json5
imageArg: "--image",
imageMode: "repeat"
```

OpenClaw จะเขียนรูปภาพ base64 ลงในไฟล์ชั่วคราว หากตั้งค่า `imageArg` พาธเหล่านั้นจะถูกส่งเป็นอาร์กิวเมนต์ CLI หากไม่ได้ตั้งค่า OpenClaw จะต่อท้ายพาธไฟล์เข้ากับพรอมต์ (การแทรกพาธ) ซึ่งใช้ได้กับ CLI ที่โหลดไฟล์ภายในเครื่องจากพาธแบบข้อความธรรมดาโดยอัตโนมัติ

## อินพุตและเอาต์พุต

- `output: "text"` (ค่าเริ่มต้น) ถือว่า stdout เป็นการตอบกลับสุดท้าย
- `output: "json"` พยายามแยกวิเคราะห์ JSON และดึงข้อความพร้อมรหัสเซสชัน
- `output: "jsonl"` แยกวิเคราะห์สตรีม JSONL และดึงข้อความสุดท้ายของเอเจนต์พร้อมตัวระบุเซสชันเมื่อมี
- สำหรับเอาต์พุต JSON ของ Gemini CLI OpenClaw จะอ่านข้อความตอบกลับจาก `response` และข้อมูลการใช้งานจาก `stats` เมื่อ `usage` ไม่มีอยู่หรือว่างเปล่า ค่าเริ่มต้นของ Gemini CLI ที่มาพร้อมระบบใช้ `stream-json`; การเขียนทับ `--output-format json` แบบเก่ายังคงใช้ตัวแยกวิเคราะห์ JSON

โหมดอินพุต:

- `input: "arg"` (ค่าเริ่มต้น) ส่งพรอมต์เป็นอาร์กิวเมนต์ CLI ตัวสุดท้าย
- `input: "stdin"` ส่งพรอมต์ผ่าน stdin
- หากพรอมต์ยาวมากและตั้งค่า `maxPromptArgChars` ไว้ ระบบจะใช้ stdin แทน

## ค่าเริ่มต้นที่ Plugin เป็นเจ้าของ

ค่าเริ่มต้นของแบ็กเอนด์ CLI เป็นส่วนหนึ่งของพื้นผิว Plugin:

- Plugin ลงทะเบียนค่าเหล่านี้ด้วย `api.registerCliBackend(...)`
- `id` ของแบ็กเอนด์จะกลายเป็นคำนำหน้าผู้ให้บริการในการอ้างอิงโมเดล
- การกำหนดค่าของผู้ใช้ใน `agents.defaults.cliBackends.<id>` ยังคงแทนที่ค่าเริ่มต้นของ Plugin
- การล้างการกำหนดค่าเฉพาะแบ็กเอนด์ยังคงเป็นความรับผิดชอบของ Plugin ผ่านฮุก `normalizeConfig` ซึ่งเป็นทางเลือก

Anthropic เป็นเจ้าของ `claude-cli` และ Google เป็นเจ้าของ `google-gemini-cli` การรันเอเจนต์ OpenAI Codex ใช้ชุดทดสอบ app-server ของ Codex ผ่าน `openai/*`; OpenClaw จะไม่ลงทะเบียนแบ็กเอนด์ `codex-cli` ที่รวมมาให้อีกต่อไป

Plugin Anthropic ที่รวมมาให้จะลงทะเบียนสำหรับ `claude-cli`:

| คีย์                   | ค่า                                                                                                                                                                                                         |
| --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `command`             | `claude`                                                                                                                                                                                                      |
| `args`                | `-p --output-format stream-json --include-partial-messages --verbose --setting-sources user --allowedTools mcp__openclaw__* --disallowedTools ScheduleWakeup,CronCreate,Bash(run_in_background:true),Monitor` |
| `output`              | `jsonl`                                                                                                                                                                                                       |
| `input`               | `stdin`                                                                                                                                                                                                       |
| `modelArg`            | `--model`                                                                                                                                                                                                     |
| `sessionArg`          | `--session-id`                                                                                                                                                                                                |
| `sessionMode`         | `always`                                                                                                                                                                                                      |
| `imageArg`            | `@`                                                                                                                                                                                                           |
| `imagePathScope`      | `workspace`                                                                                                                                                                                                   |
| `systemPromptFileArg` | `--append-system-prompt-file`                                                                                                                                                                                 |
| `systemPromptMode`    | `append`                                                                                                                                                                                                      |

Plugin Google ที่รวมมาให้จะลงทะเบียนสำหรับ `google-gemini-cli`:

| คีย์                       | ค่า                                                                                  |
| ------------------------- | -------------------------------------------------------------------------------------- |
| `command`                 | `gemini`                                                                               |
| `args`                    | `--skip-trust --approval-mode auto_edit --output-format stream-json --prompt {prompt}` |
| `resumeArgs`              | เหมือนกัน โดยมี `--resume {sessionId}`                                                      |
| `output` / `resumeOutput` | `jsonl`                                                                                |
| `jsonlDialect`            | `gemini-stream-json`                                                                   |
| `imageArg`                | `@`                                                                                    |
| `imagePathScope`          | `workspace`                                                                            |
| `modelArg`                | `--model`                                                                              |
| `sessionMode`             | `existing`                                                                             |
| `sessionIdFields`         | `["session_id", "sessionId"]`                                                          |

ข้อกำหนดเบื้องต้น: ต้องติดตั้ง Gemini CLI ในเครื่องและมีอยู่ใน `PATH` ในชื่อ `gemini` (`brew install gemini-cli` หรือ `npm install -g @google/gemini-cli`)

หมายเหตุเกี่ยวกับเอาต์พุต Gemini CLI:

- ตัวแยกวิเคราะห์ `stream-json` เริ่มต้นจะอ่านเหตุการณ์ `message` ของผู้ช่วย เหตุการณ์เครื่องมือ การใช้งาน `result` ขั้นสุดท้าย และเหตุการณ์ข้อผิดพลาดร้ายแรงของ Gemini
- หากแทนที่อาร์กิวเมนต์ Gemini เป็น `--output-format json` OpenClaw จะปรับแบ็กเอนด์นั้นกลับเป็น `output: "json"` และอ่านข้อความตอบกลับจากฟิลด์ `response` ใน JSON
- การใช้งานจะย้อนกลับไปใช้ `stats` เมื่อไม่มี `usage` หรือค่าว่าง; `stats.cached` จะถูกปรับเป็น `cacheRead` ของ OpenClaw และหากไม่มี `stats.input` ระบบจะคำนวณโทเค็นอินพุตจาก `stats.input_tokens - stats.cached`

แทนที่ค่าเริ่มต้นเฉพาะเมื่อจำเป็นเท่านั้น (กรณีที่พบบ่อยที่สุดคือพาธสัมบูรณ์ของ `command`)

## โอเวอร์เลย์การแปลงข้อความ

Plugin ที่ต้องใช้ชิมความเข้ากันได้ขนาดเล็กสำหรับพรอมต์/ข้อความสามารถประกาศการแปลงข้อความแบบสองทิศทางได้ โดยไม่ต้องแทนที่ผู้ให้บริการหรือแบ็กเอนด์ CLI:

```typescript
api.registerTextTransforms({
  input: [{ from: /red basket/g, to: "blue basket" }],
  output: [{ from: /blue basket/g, to: "red basket" }],
});
```

`input` จะเขียนพรอมต์ระบบและพรอมต์ผู้ใช้ที่ส่งให้ CLI ใหม่ `output` จะเขียนข้อความผู้ช่วยที่สตรีมและข้อความสุดท้ายที่แยกวิเคราะห์แล้วใหม่ ก่อนที่ OpenClaw จะจัดการมาร์กเกอร์ควบคุมและการส่งไปยังช่องทางของตนเอง สำหรับการเรียกโมเดลผ่านผู้ให้บริการ ระบบยังคืนค่าสตริงภายในอาร์กิวเมนต์การเรียกเครื่องมือแบบมีโครงสร้างหลังจากซ่อมแซมสตรีมและก่อนเรียกใช้เครื่องมือด้วย ส่วนย่อย JSON ดิบจากผู้ให้บริการจะไม่ถูกเปลี่ยนแปลง ผู้ใช้ข้อมูลควรใช้เพย์โหลดบางส่วน สิ้นสุด หรือผลลัพธ์แบบมีโครงสร้าง

สำหรับ CLI ที่ปล่อยเหตุการณ์ JSONL เฉพาะผู้ให้บริการ ให้ตั้งค่า `jsonlDialect` ในการกำหนดค่าของแบ็กเอนด์นั้น: `claude-stream-json` สำหรับสตรีมที่เข้ากันได้กับ Claude Code และ `gemini-stream-json` สำหรับเหตุการณ์ `stream-json` ของ Gemini CLI

## ความเป็นเจ้าของ Compaction แบบเนทีฟ

แบ็กเอนด์ CLI บางตัวรันเอเจนต์ที่ทำ Compaction ทรานสคริปต์ของตนเอง ดังนั้น OpenClaw ต้องไม่เรียกใช้ตัวสรุปเพื่อป้องกันของตนกับแบ็กเอนด์เหล่านั้น เพราะจะขัดแย้งกับ Compaction ของแบ็กเอนด์เองและอาจทำให้รอบการทำงานล้มเหลวอย่างรุนแรง

`claude-cli` ไม่มีปลายทางชุดทดสอบ (Claude Code ทำ Compaction ภายใน) จึงประกาศ `ownsNativeCompaction: true` และเส้นทาง Compaction ของ OpenClaw จะคืนรายการเซสชันโดยไม่เปลี่ยนแปลง OpenClaw ส่งงบประมาณบริบทที่มีผลของการรันผ่าน [`CLAUDE_CODE_AUTO_COMPACT_WINDOW`](https://code.claude.com/docs/en/env-vars) ที่ Claude Code จัดทำเอกสารไว้ เพื่อให้ Compaction อัตโนมัติแบบเนทีฟสอดคล้องกับขีดจำกัด `contextTokens` ของ Anthropic ที่กำหนดค่าไว้ ส่วนเซสชันชุดทดสอบเนทีฟ เช่น Codex ยังคงกำหนดเส้นทางไปยังปลายทาง Compaction ของชุดทดสอบนั้นแทน

```typescript
api.registerCliBackend({ id: "my-cli", ownsNativeCompaction: true /* ... */ });
```

ประกาศ `ownsNativeCompaction` เฉพาะสำหรับแบ็กเอนด์ที่เป็นเจ้าของ Compaction อย่างแท้จริงเท่านั้น โดยต้องจำกัดทรานสคริปต์ของตนเองให้อยู่ใกล้กรอบบริบทได้อย่างเชื่อถือได้ และคงสถานะเซสชันที่ดำเนินการต่อได้ (เช่น `--resume` / `--session-id`) มิฉะนั้นเซสชันที่เลื่อนออกไปอาจยังคงเกินงบประมาณ

## โอเวอร์เลย์ MCP ของบันเดิล

แบ็กเอนด์ CLI ไม่ได้รับการเรียกเครื่องมือ OpenClaw โดยตรง แต่แบ็กเอนด์สามารถเลือกใช้โอเวอร์เลย์การกำหนดค่า MCP ที่สร้างขึ้นด้วย `bundleMcp: true` พฤติกรรมที่รวมมาให้ในปัจจุบัน:

- `claude-cli`: ไฟล์การกำหนดค่า MCP แบบเข้มงวดที่สร้างขึ้น
- `google-gemini-cli`: ไฟล์การตั้งค่าระบบ Gemini ที่สร้างขึ้น

เมื่อเปิดใช้ MCP ของบันเดิล OpenClaw จะ:

- สร้างเซิร์ฟเวอร์ HTTP MCP แบบลูปแบ็กที่เปิดเผยเครื่องมือ Gateway แก่กระบวนการ CLI โดยยืนยันตัวตนด้วยสิทธิ์บริบทเฉพาะการรัน (`OPENCLAW_MCP_TOKEN`) ซึ่งใช้งานได้เฉพาะความพยายามดำเนินการปัจจุบัน
- ผูกการเข้าถึงเครื่องมือกับบริบทเซสชัน บัญชี และช่องทางที่ Gateway เลือก แทนที่จะเชื่อถือเฮดเดอร์ของกระบวนการลูก
- โหลดเซิร์ฟเวอร์ MCP ของบันเดิลที่เปิดใช้สำหรับพื้นที่ทำงานปัจจุบัน และผสานเข้ากับรูปแบบการกำหนดค่า/การตั้งค่า MCP ของแบ็กเอนด์ที่มีอยู่
- เขียนการกำหนดค่าการเริ่มทำงานใหม่โดยใช้โหมดการผสานรวมที่แบ็กเอนด์เป็นเจ้าของจาก Plugin เจ้าของ

หากไม่มีเซิร์ฟเวอร์ MCP ที่เปิดใช้ OpenClaw ยังคงแทรกการกำหนดค่าแบบเข้มงวดเมื่อแบ็กเอนด์เลือกใช้ MCP ของบันเดิล เพื่อให้การรันเบื้องหลังยังคงแยกออกจากกัน

รันไทม์ MCP ที่รวมมาให้และมีขอบเขตระดับเซสชันจะถูกแคชเพื่อนำกลับมาใช้ภายในเซสชัน จากนั้นจะถูกเก็บกวาดหลังไม่มีการใช้งานเป็นเวลา `mcp.sessionIdleTtlMs` มิลลิวินาที (ค่าเริ่มต้น 10 นาที; ตั้งค่า `0` เพื่อปิดใช้งาน) การรันแบบฝังครั้งเดียว เช่น การตรวจสอบการยืนยันตัวตน การสร้าง slug และการเรียกคืน Active Memory จะร้องขอการล้างเมื่อสิ้นสุดการรัน เพื่อไม่ให้กระบวนการลูก stdio และสตรีม Streamable HTTP/SSE ทำงานนานกว่าการรัน

## ขีดจำกัดประวัติสำหรับการเพาะเซสชันใหม่

เมื่อเพาะเซสชัน CLI ใหม่จากทรานสคริปต์ OpenClaw ก่อนหน้า (ตัวอย่างเช่น หลังลอง `session_expired` ใหม่) บล็อก `<conversation_history>` ที่เรนเดอร์จะถูกจำกัดเพื่อป้องกันไม่ให้พรอมต์สำหรับเพาะเซสชันใหม่มีขนาดพุ่งสูง ค่าเริ่มต้นคือ 12,288 อักขระ (ประมาณ 3,000 โทเค็น)

แบ็กเอนด์ Claude CLI จะปรับขีดจำกัดนี้ตามกรอบบริบท Claude ที่คำนวณได้แทน กล่าวคือกรอบบริบทที่ใหญ่กว่าจะได้รับส่วนประวัติก่อนหน้าที่ใหญ่ขึ้นจนถึงเพดานคงที่ ส่วนแบ็กเอนด์ CLI อื่นยังคงใช้ค่าเริ่มต้นแบบระมัดระวัง ขีดจำกัดนี้ควบคุมเฉพาะบล็อกประวัติก่อนหน้าของพรอมต์สำหรับเพาะเซสชันใหม่เท่านั้น โดยขีดจำกัดเอาต์พุตของเซสชันสดจะปรับแยกต่างหากภายใต้ `reliability.outputLimits` (ดู [เซสชัน](#sessions))

## ข้อจำกัด

- ไม่มีการเรียกเครื่องมือ OpenClaw โดยตรง: OpenClaw จะไม่แทรกการเรียกเครื่องมือลงในโปรโตคอลแบ็กเอนด์ CLI แบ็กเอนด์จะเห็นเครื่องมือ Gateway เฉพาะเมื่อเลือกใช้ `bundleMcp: true`
- การสตรีมขึ้นอยู่กับแต่ละแบ็กเอนด์: บางแบ็กเอนด์สตรีม JSONL ส่วนบางแบ็กเอนด์บัฟเฟอร์ไว้จนกว่าจะสิ้นสุด
- เอาต์พุตแบบมีโครงสร้างขึ้นอยู่กับรูปแบบ JSON ของ CLI เอง

## การแก้ไขปัญหา

| อาการ               | วิธีแก้ไข                                                               |
| --------------------- | ----------------------------------------------------------------- |
| ไม่พบ CLI         | ตั้งค่า `command` เป็นพาธแบบเต็ม                                     |
| ชื่อโมเดลไม่ถูกต้อง      | ใช้ `modelAliases` เพื่อแมป `provider/model` กับรหัสโมเดลของ CLI |
| เซสชันไม่ต่อเนื่อง | ตรวจสอบว่าได้ตั้งค่า `sessionArg` และ `sessionMode` ไม่ใช่ `none`       |
| รูปภาพถูกละเว้น        | ตั้งค่า `imageArg` และตรวจสอบว่า CLI รองรับพาธไฟล์            |
