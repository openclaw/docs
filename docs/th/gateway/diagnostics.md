OpenClaw สามารถสร้างไฟล์ zip การวินิจฉัยภายในเครื่องสำหรับรายงานบั๊กได้ โดยรวมสถานะ Gateway ที่ผ่านการล้างข้อมูลแล้ว สุขภาพระบบ บันทึก รูปแบบการกำหนดค่า และเหตุการณ์เสถียรภาพล่าสุดที่ไม่มีเพย์โหลด

ให้ปฏิบัติกับชุดการวินิจฉัยเหมือนความลับจนกว่าคุณจะตรวจสอบแล้ว ชุดเหล่านี้ออกแบบมาเพื่อเว้นหรือปกปิดเพย์โหลดและข้อมูลรับรอง แต่ยังคงสรุปบันทึก Gateway ภายในเครื่องและสถานะรันไทม์ระดับโฮสต์

## เริ่มต้นอย่างรวดเร็ว

```bash
openclaw gateway diagnostics export
```

คำสั่งจะแสดงพาธไฟล์ zip ที่เขียนไว้ หากต้องการเลือกพาธ:

```bash
openclaw gateway diagnostics export --output openclaw-diagnostics.zip
```

สำหรับระบบอัตโนมัติ:

```bash
openclaw gateway diagnostics export --json
```

## คำสั่งแชต

เจ้าของสามารถใช้ `/diagnostics [note]` ในแชตเพื่อขอส่งออก Gateway ภายในเครื่องได้ ใช้สิ่งนี้เมื่อบั๊กเกิดขึ้นในบทสนทนาจริง และคุณต้องการรายงานเดียวที่คัดลอกวางได้สำหรับฝ่ายสนับสนุน:

1. ส่ง `/diagnostics` ในบทสนทนาที่คุณสังเกตเห็นปัญหา เพิ่มบันทึกสั้น ๆ หากช่วยได้ เช่น `/diagnostics bad tool choice`
2. OpenClaw ส่งคำนำการวินิจฉัยและขอการอนุมัติ exec ที่ชัดเจนหนึ่งครั้ง การอนุมัติจะเรียกใช้ `openclaw gateway diagnostics export --json` อย่าอนุมัติการวินิจฉัยผ่านกฎที่อนุญาตทั้งหมด
3. หลังอนุมัติ OpenClaw จะตอบกลับด้วยรายงานที่วางได้ ซึ่งมีพาธชุดข้อมูลภายในเครื่อง สรุปแมนิเฟสต์ หมายเหตุด้านความเป็นส่วนตัว และ ID เซสชันที่เกี่ยวข้อง

ในแชตกลุ่ม เจ้าของยังคงเรียกใช้ `/diagnostics` ได้ แต่ OpenClaw จะไม่โพสต์รายละเอียดการวินิจฉัยกลับเข้าไปในแชตร่วม โดยจะส่งคำนำ พรอมป์ขออนุมัติ ผลการส่งออก Gateway และรายละเอียดเซสชัน/เธรดของ Codex ให้เจ้าของผ่านเส้นทางอนุมัติส่วนตัว กลุ่มจะได้รับเพียงประกาศสั้น ๆ ว่ากระบวนการวินิจฉัยถูกส่งแบบส่วนตัวแล้ว หาก OpenClaw หาเส้นทางส่วนตัวของเจ้าของไม่พบ คำสั่งจะปิดแบบปลอดภัยและขอให้เจ้าของเรียกใช้จาก DM

เมื่อเซสชัน OpenClaw ที่ใช้งานอยู่ใช้ฮาร์เนส OpenAI Codex แบบเนทีฟ การอนุมัติ exec เดียวกันจะครอบคลุมการอัปโหลดฟีดแบ็ก OpenAI สำหรับเธรดรันไทม์ Codex ที่ OpenClaw รู้จักด้วย การอัปโหลดนั้นแยกจากไฟล์ zip Gateway ภายในเครื่อง และจะปรากฏเฉพาะสำหรับเซสชันฮาร์เนส Codex เท่านั้น ก่อนอนุมัติ พรอมป์จะอธิบายว่าการอนุมัติการวินิจฉัยจะส่งฟีดแบ็ก Codex ด้วย แต่จะไม่แสดง ID เซสชันหรือเธรด Codex หลังอนุมัติ คำตอบในแชตจะแสดงช่องทาง ID เซสชัน OpenClaw, ID เธรด Codex และคำสั่ง resume ภายในเครื่องสำหรับเธรดที่ส่งไปยังเซิร์ฟเวอร์ OpenAI หากคุณปฏิเสธหรือเพิกเฉยต่อการอนุมัติ OpenClaw จะไม่เรียกใช้การส่งออก ไม่ส่งฟีดแบ็ก Codex และไม่พิมพ์ ID Codex

สิ่งนี้ทำให้ลูปดีบัก Codex ทั่วไปสั้นลง: สังเกตพฤติกรรมที่ไม่ถูกต้องใน Telegram, Discord หรือช่องทางอื่น เรียกใช้ `/diagnostics` อนุมัติหนึ่งครั้ง แชร์รายงานกับฝ่ายสนับสนุน จากนั้นเรียกใช้คำสั่ง `codex resume <thread-id>` ที่พิมพ์ออกมาภายในเครื่อง หากคุณต้องการตรวจสอบเธรด Codex แบบเนทีฟด้วยตนเอง ดู [ฮาร์เนส Codex](/th/plugins/codex-harness#inspect-a-codex-thread-from-the-cli) สำหรับเวิร์กโฟลว์การตรวจสอบนั้น

## สิ่งที่การส่งออกมี

ไฟล์ zip ประกอบด้วย:

- `summary.md`: ภาพรวมที่มนุษย์อ่านได้สำหรับฝ่ายสนับสนุน
- `diagnostics.json`: สรุปที่เครื่องอ่านได้ของการกำหนดค่า บันทึก สถานะ สุขภาพระบบ และข้อมูลเสถียรภาพ
- `manifest.json`: เมตาดาตาการส่งออกและรายการไฟล์
- รูปแบบการกำหนดค่าที่ผ่านการล้างข้อมูลแล้ว และรายละเอียดการกำหนดค่าที่ไม่ใช่ความลับ
- สรุปบันทึกที่ผ่านการล้างข้อมูลแล้ว และบรรทัดบันทึกล่าสุดที่ปกปิดแล้ว
- สแนปชอตสถานะและสุขภาพ Gateway แบบดีที่สุดเท่าที่ทำได้
- `stability/latest.json`: ชุดเสถียรภาพที่คงอยู่ล่าสุด เมื่อมี

การส่งออกมีประโยชน์แม้เมื่อ Gateway ไม่สมบูรณ์ หาก Gateway ไม่สามารถตอบคำขอสถานะหรือสุขภาพระบบได้ บันทึกภายในเครื่อง รูปแบบการกำหนดค่า และชุดเสถียรภาพล่าสุดจะยังคงถูกรวบรวมเมื่อมี

## โมเดลความเป็นส่วนตัว

การวินิจฉัยออกแบบมาให้แชร์ได้ การส่งออกเก็บข้อมูลปฏิบัติการที่ช่วยในการดีบัก เช่น:

- ชื่อระบบย่อย, Plugin ID, ID ผู้ให้บริการ, ID ช่องทาง และโหมดที่กำหนดค่าไว้
- รหัสสถานะ ระยะเวลา จำนวนไบต์ สถานะคิว และค่าการอ่านหน่วยความจำ
- เมตาดาตาบันทึกที่ผ่านการล้างข้อมูลแล้ว และข้อความปฏิบัติการที่ปกปิดแล้ว
- รูปแบบการกำหนดค่าและการตั้งค่าฟีเจอร์ที่ไม่ใช่ความลับ

การส่งออกจะเว้นหรือปกปิด:

- ข้อความแชต พรอมป์ คำสั่ง เนื้อหา Webhook และเอาต์พุตเครื่องมือ
- ข้อมูลรับรอง คีย์ API โทเค็น คุกกี้ และค่าความลับ
- เนื้อหาคำขอหรือคำตอบดิบ
- ID บัญชี, ID ข้อความ, ID เซสชันดิบ, ชื่อโฮสต์ และชื่อผู้ใช้ภายในเครื่อง

เมื่อข้อความบันทึกดูเหมือนข้อความผู้ใช้ แชต พรอมป์ หรือเพย์โหลดเครื่องมือ การส่งออกจะเก็บไว้เพียงว่ามีข้อความถูกเว้นไว้และจำนวนไบต์

## ตัวบันทึกเสถียรภาพ

Gateway จะบันทึกสตรีมเสถียรภาพแบบจำกัดขนาดและไม่มีเพย์โหลดโดยค่าเริ่มต้นเมื่อเปิดใช้งานการวินิจฉัย สิ่งนี้มีไว้สำหรับข้อเท็จจริงด้านปฏิบัติการ ไม่ใช่เนื้อหา

Heartbeat การวินิจฉัยเดียวกันจะบันทึกคำเตือนความมีชีวิตเมื่อ Gateway ยังทำงานอยู่ แต่ลูปเหตุการณ์ Node.js หรือ CPU ดูอิ่มตัว เหตุการณ์ `diagnostic.liveness.warning` เหล่านี้ประกอบด้วยดีเลย์ของลูปเหตุการณ์ การใช้ลูปเหตุการณ์ อัตราส่วนคอร์ CPU และจำนวนเซสชันที่ใช้งานอยู่/รออยู่/เข้าคิวอยู่ เหตุการณ์เหล่านี้จะไม่รีสตาร์ต Gateway ด้วยตัวเอง

ตรวจสอบตัวบันทึกสด:

```bash
openclaw gateway stability
openclaw gateway stability --type payload.large
openclaw gateway stability --json
```

ตรวจสอบชุดเสถียรภาพที่คงอยู่ล่าสุดหลังการออกแบบร้ายแรง การหมดเวลาปิดระบบ หรือการเริ่มต้นหลังรีสตาร์ตล้มเหลว:

```bash
openclaw gateway stability --bundle latest
```

สร้างไฟล์ zip การวินิจฉัยจากชุดที่คงอยู่ล่าสุด:

```bash
openclaw gateway stability --bundle latest --export
```

ชุดที่คงอยู่จะอยู่ใต้ `~/.openclaw/logs/stability/` เมื่อมีเหตุการณ์

## ตัวเลือกที่มีประโยชน์

```bash
openclaw gateway diagnostics export \
  --output openclaw-diagnostics.zip \
  --log-lines 5000 \
  --log-bytes 1000000
```

- `--output <path>`: เขียนไปยังพาธ zip ที่ระบุ
- `--log-lines <count>`: จำนวนบรรทัดบันทึกที่ผ่านการล้างข้อมูลแล้วสูงสุดที่จะรวม
- `--log-bytes <bytes>`: จำนวนไบต์บันทึกสูงสุดที่จะตรวจสอบ
- `--url <url>`: URL WebSocket ของ Gateway สำหรับสแนปชอตสถานะและสุขภาพระบบ
- `--token <token>`: โทเค็น Gateway สำหรับสแนปชอตสถานะและสุขภาพระบบ
- `--password <password>`: รหัสผ่าน Gateway สำหรับสแนปชอตสถานะและสุขภาพระบบ
- `--timeout <ms>`: เวลาหมดอายุของสแนปชอตสถานะและสุขภาพระบบ
- `--no-stability-bundle`: ข้ามการค้นหาชุดเสถียรภาพที่คงอยู่
- `--json`: พิมพ์เมตาดาตาการส่งออกที่เครื่องอ่านได้

## ปิดใช้งานการวินิจฉัย

การวินิจฉัยเปิดใช้งานโดยค่าเริ่มต้น หากต้องการปิดใช้งานตัวบันทึกเสถียรภาพและการรวบรวมเหตุการณ์การวินิจฉัย:

```json5
{
  diagnostics: {
    enabled: false,
  },
}
```

การปิดใช้งานการวินิจฉัยจะลดรายละเอียดรายงานบั๊ก แต่ไม่กระทบการบันทึก Gateway ตามปกติ
