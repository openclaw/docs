`scripts/` เก็บสคริปต์ช่วยเหลือสำหรับเวิร์กโฟลว์ภายในเครื่องและงานปฏิบัติการ ใช้สคริปต์เหล่านี้เมื่องานเกี่ยวข้องกับสคริปต์อย่างชัดเจน มิฉะนั้นให้เลือกใช้ CLI

## หลักเกณฑ์

- สคริปต์เป็นสิ่งที่ **ไม่บังคับ** เว้นแต่จะมีการอ้างถึงในเอกสารหรือรายการตรวจสอบการเผยแพร่
- เลือกใช้ส่วนติดต่อ CLI เมื่อมีให้ใช้งาน (ตัวอย่าง: `openclaw models status --check`)
- ให้ถือว่าสคริปต์ขึ้นอยู่กับโฮสต์โดยเฉพาะ โปรดอ่านสคริปต์ก่อนเรียกใช้บนเครื่องใหม่

## สคริปต์ตรวจสอบการยืนยันตัวตน

การยืนยันตัวตนของโมเดลโดยทั่วไปครอบคลุมอยู่ใน [การยืนยันตัวตน](/th/gateway/authentication) สคริปต์ด้านล่างเป็นระบบแยกต่างหากที่ไม่บังคับ สำหรับตรวจสอบ **โทเค็นการสมัครใช้งาน Claude Code CLI** บนโฮสต์ระยะไกล/ไม่มีส่วนติดต่อผู้ใช้ และยืนยันตัวตนอีกครั้งจากโทรศัพท์:

- `scripts/setup-auth-system.sh` - การตั้งค่าครั้งเดียว: ตรวจสอบการยืนยันตัวตนปัจจุบัน ช่วยสร้าง `claude setup-token` ที่มีอายุใช้งานยาวนาน และแสดงขั้นตอนการติดตั้ง systemd/Termux
- `scripts/claude-auth-status.sh [full|json|simple]` - ตรวจสอบสถานะการยืนยันตัวตนของ Claude Code และ OpenClaw
- `scripts/auth-monitor.sh` - ตรวจสอบสถานะเป็นระยะและส่งการแจ้งเตือน (ผ่านการส่งข้อความของ OpenClaw และ/หรือ ntfy.sh) เมื่อโทเค็นใกล้หมดอายุ ตัวแปรสภาพแวดล้อม: `WARN_HOURS` (ค่าเริ่มต้น `2`), `NOTIFY_PHONE`, `NOTIFY_NTFY` เรียกใช้ตามกำหนดเวลาด้วย `scripts/systemd/openclaw-auth-monitor.{service,timer}` ที่รวมมาให้ (ทุก 30 นาที)
- `scripts/mobile-reauth.sh` - เรียกใช้ `claude setup-token` อีกครั้งและแสดง URL สำหรับเปิดบนโทรศัพท์ เพื่อใช้งานผ่าน SSH จาก Termux
- `scripts/termux-quick-auth.sh`, `scripts/termux-auth-widget.sh`, `scripts/termux-sync-widget.sh` - สคริปต์ Termux:Widget ที่เชื่อมต่อ SSH ไปยังโฮสต์ แสดงข้อความสถานะแบบป๊อปอัป และเปิดคอนโซล/คำแนะนำสำหรับการยืนยันตัวตนอีกครั้งเมื่อการยืนยันตัวตนหมดอายุ

## ตัวช่วยอ่านข้อมูลจาก GitHub

ใช้ `scripts/gh-read` เมื่อต้องการให้ `gh` ใช้โทเค็นการติดตั้ง GitHub App สำหรับการเรียกอ่านข้อมูลที่จำกัดขอบเขตเฉพาะรีโพซิทอรี ขณะที่ `gh` ปกติยังคงใช้บัญชีส่วนตัวของคุณสำหรับการดำเนินการเขียนข้อมูล

ตัวแปรสภาพแวดล้อมที่จำเป็น:

- `OPENCLAW_GH_READ_APP_ID`
- `OPENCLAW_GH_READ_PRIVATE_KEY_FILE`

ตัวแปรสภาพแวดล้อมที่ไม่บังคับ:

- `OPENCLAW_GH_READ_INSTALLATION_ID` เมื่อต้องการข้ามการค้นหาการติดตั้งตามรีโพซิทอรี
- `OPENCLAW_GH_READ_PERMISSIONS` ใช้กำหนดแทนชุดย่อยของสิทธิ์อ่านที่จะร้องขอ โดยคั่นแต่ละค่าด้วยจุลภาค

ลำดับการระบุรีโพซิทอรี:

- `gh ... -R owner/repo`
- `GH_REPO`
- `git remote origin`

ตัวอย่าง:

- `scripts/gh-read pr view 123`
- `scripts/gh-read run list -R openclaw/openclaw`
- `scripts/gh-read api repos/openclaw/openclaw/pulls/123`
