OpenClaw ดึงตัวแปรสภาพแวดล้อมจากหลายแหล่ง กฎคือ **ห้ามแทนที่ค่าที่มีอยู่แล้ว**

## ลำดับความสำคัญ (สูงสุด → ต่ำสุด)

1. **สภาพแวดล้อมของโปรเซส** (สิ่งที่โปรเซส Gateway มีอยู่แล้วจากเชลล์หรือดีมอนแม่)
2. **`.env` ในไดเรกทอรีทำงานปัจจุบัน** (ค่าเริ่มต้นของ dotenv; ไม่แทนที่ค่าเดิม)
3. **`.env` ส่วนกลาง** ที่ `~/.openclaw/.env` (หรือ `$OPENCLAW_STATE_DIR/.env`; ไม่แทนที่ค่าเดิม)
4. **บล็อก `env` ของการกำหนดค่า** ใน `~/.openclaw/openclaw.json` (ใช้เฉพาะเมื่อยังไม่มีค่า)
5. **การนำเข้าจากเชลล์ล็อกอินแบบเลือกได้** (`env.shellEnv.enabled` หรือ `OPENCLAW_LOAD_SHELL_ENV=1`) ใช้เฉพาะกับคีย์ที่คาดไว้และยังไม่มีค่า

ในการติดตั้งใหม่บน Ubuntu ที่ใช้ไดเรกทอรีสถานะเริ่มต้น OpenClaw ยังถือว่า `~/.config/openclaw/gateway.env` เป็นตัวสำรองเพื่อความเข้ากันได้หลัง `.env` ส่วนกลางด้วย หากทั้งสองไฟล์มีอยู่และค่าขัดแย้งกัน OpenClaw จะใช้ `~/.openclaw/.env` ต่อไปและแสดงคำเตือน

หากไฟล์การกำหนดค่าหายไปทั้งหมด ขั้นตอนที่ 4 จะถูกข้ามไป; การนำเข้าจากเชลล์ยังคงทำงานหากเปิดใช้ไว้

## บล็อก `env` ของการกำหนดค่า

มีสองวิธีที่เทียบเท่ากันในการตั้งค่าตัวแปรสภาพแวดล้อมแบบอินไลน์ (ทั้งสองวิธีไม่แทนที่ค่าเดิม):

```json5
{
  env: {
    OPENROUTER_API_KEY: "sk-or-...",
    vars: {
      GROQ_API_KEY: "gsk-...",
    },
  },
}
```

## การนำเข้าสภาพแวดล้อมจากเชลล์

`env.shellEnv` เรียกใช้เชลล์ล็อกอินของคุณและนำเข้าเฉพาะคีย์ที่คาดไว้ซึ่งยัง **ไม่มีค่า**:

```json5
{
  env: {
    shellEnv: {
      enabled: true,
      timeoutMs: 15000,
    },
  },
}
```

ตัวแปรสภาพแวดล้อมที่เทียบเท่า:

- `OPENCLAW_LOAD_SHELL_ENV=1`
- `OPENCLAW_SHELL_ENV_TIMEOUT_MS=15000`

## ตัวแปรสภาพแวดล้อมที่ฉีดเข้าในระหว่างรันไทม์

OpenClaw ยังฉีดเครื่องหมายบริบทเข้าไปในโปรเซสลูกที่สร้างขึ้น:

- `OPENCLAW_SHELL=exec`: ตั้งค่าสำหรับคำสั่งที่รันผ่านเครื่องมือ `exec`
- `OPENCLAW_SHELL=acp`: ตั้งค่าสำหรับการสร้างโปรเซสแบ็กเอนด์รันไทม์ ACP (เช่น `acpx`)
- `OPENCLAW_SHELL=acp-client`: ตั้งค่าสำหรับ `openclaw acp client` เมื่อสร้างโปรเซสสะพาน ACP
- `OPENCLAW_SHELL=tui-local`: ตั้งค่าสำหรับคำสั่งเชลล์ `!` ของ TUI ในเครื่อง

สิ่งเหล่านี้เป็นเครื่องหมายรันไทม์ (ไม่ใช่การกำหนดค่าผู้ใช้ที่จำเป็น) สามารถใช้ในตรรกะของเชลล์หรือโปรไฟล์
เพื่อใช้กฎเฉพาะตามบริบทได้

## ตัวแปรสภาพแวดล้อมของ UI

- `OPENCLAW_THEME=light`: บังคับใช้พาเลต TUI แบบสว่างเมื่อเทอร์มินัลของคุณมีพื้นหลังสว่าง
- `OPENCLAW_THEME=dark`: บังคับใช้พาเลต TUI แบบมืด
- `COLORFGBG`: หากเทอร์มินัลของคุณส่งออกค่านี้ OpenClaw จะใช้คำใบ้สีพื้นหลังเพื่อเลือกพาเลต TUI อัตโนมัติ

## การแทนค่าตัวแปรสภาพแวดล้อมในการกำหนดค่า

คุณสามารถอ้างอิงตัวแปรสภาพแวดล้อมโดยตรงในค่าสตริงของการกำหนดค่าได้โดยใช้ไวยากรณ์ `${VAR_NAME}`:

```json5
{
  models: {
    providers: {
      "vercel-gateway": {
        apiKey: "${VERCEL_GATEWAY_API_KEY}",
      },
    },
  },
}
```

ดูรายละเอียดทั้งหมดที่ [การกำหนดค่า: การแทนค่าตัวแปรสภาพแวดล้อม](/th/gateway/configuration-reference#env-var-substitution)

## การอ้างอิงความลับเทียบกับสตริง `${ENV}`

OpenClaw รองรับรูปแบบที่ขับเคลื่อนด้วยสภาพแวดล้อมสองแบบ:

- การแทนค่าสตริง `${VAR}` ในค่าการกำหนดค่า
- ออบเจ็กต์ SecretRef (`{ source: "env", provider: "default", id: "VAR" }`) สำหรับฟิลด์ที่รองรับการอ้างอิงความลับ

ทั้งสองแบบแก้ค่าได้จากสภาพแวดล้อมของโปรเซสในเวลาที่เปิดใช้งาน รายละเอียด SecretRef มีบันทึกไว้ใน [การจัดการความลับ](/th/gateway/secrets)

## ตัวแปรสภาพแวดล้อมที่เกี่ยวกับเส้นทาง

| ตัวแปร               | วัตถุประสงค์                                                                                                                                                                          |
| ---------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `OPENCLAW_HOME`        | แทนที่ไดเรกทอรีบ้านที่ใช้สำหรับการแก้เส้นทางภายในทั้งหมด (`~/.openclaw/`, ไดเรกทอรีเอเจนต์, เซสชัน, ข้อมูลประจำตัว) มีประโยชน์เมื่อรัน OpenClaw เป็นผู้ใช้บริการเฉพาะ |
| `OPENCLAW_STATE_DIR`   | แทนที่ไดเรกทอรีสถานะ (ค่าเริ่มต้น `~/.openclaw`)                                                                                                                            |
| `OPENCLAW_CONFIG_PATH` | แทนที่เส้นทางไฟล์การกำหนดค่า (ค่าเริ่มต้น `~/.openclaw/openclaw.json`)                                                                                                             |

## การบันทึกล็อก

| ตัวแปร             | วัตถุประสงค์                                                                                                                                                                                      |
| -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `OPENCLAW_LOG_LEVEL` | แทนที่ระดับล็อกสำหรับทั้งไฟล์และคอนโซล (เช่น `debug`, `trace`) มีความสำคัญเหนือกว่า `logging.level` และ `logging.consoleLevel` ในการกำหนดค่า ค่าที่ไม่ถูกต้องจะถูกละเว้นพร้อมคำเตือน |

### `OPENCLAW_HOME`

เมื่อตั้งค่าแล้ว `OPENCLAW_HOME` จะแทนที่ไดเรกทอรีบ้านของระบบ (`$HOME` / `os.homedir()`) สำหรับการแก้เส้นทางภายในทั้งหมด วิธีนี้ช่วยให้แยกระบบไฟล์ได้เต็มรูปแบบสำหรับบัญชีบริการแบบไม่มีส่วนติดต่อผู้ใช้

**ลำดับความสำคัญ:** `OPENCLAW_HOME` > `$HOME` > `USERPROFILE` > `os.homedir()`

**ตัวอย่าง** (macOS LaunchDaemon):

```xml
<key>EnvironmentVariables</key>
<dict>
  <key>OPENCLAW_HOME</key>
  <string>/Users/user</string>
</dict>
```

`OPENCLAW_HOME` ยังสามารถตั้งเป็นเส้นทางที่มีเครื่องหมายทิลเดอได้ (เช่น `~/svc`) ซึ่งจะถูกขยายโดยใช้ `$HOME` ก่อนใช้งาน

## ผู้ใช้ nvm: ความล้มเหลว TLS ของ web_fetch

หากติดตั้ง Node.js ผ่าน **nvm** (ไม่ใช่ตัวจัดการแพ็กเกจของระบบ) `fetch()` ในตัวจะใช้
คลัง CA ที่มากับ nvm ซึ่งอาจไม่มี CA รากสมัยใหม่ (ISRG Root X1/X2 สำหรับ Let's Encrypt,
DigiCert Global Root G2 เป็นต้น) สิ่งนี้ทำให้ `web_fetch` ล้มเหลวด้วย `"fetch failed"` บนเว็บไซต์ HTTPS ส่วนใหญ่

บน Linux OpenClaw จะตรวจพบ nvm โดยอัตโนมัติและใช้การแก้ไขในสภาพแวดล้อมเริ่มต้นจริง:

- `openclaw gateway install` เขียน `NODE_EXTRA_CA_CERTS` ลงในสภาพแวดล้อมของบริการ systemd
- จุดเริ่มต้น CLI `openclaw` เรียกใช้งานตัวเองใหม่โดยตั้งค่า `NODE_EXTRA_CA_CERTS` ก่อนที่ Node จะเริ่มต้น

**การแก้ไขด้วยตนเอง (สำหรับเวอร์ชันเก่าหรือการเริ่มด้วย `node ...` โดยตรง):**

ส่งออกตัวแปรก่อนเริ่ม OpenClaw:

```bash
export NODE_EXTRA_CA_CERTS=/etc/ssl/certs/ca-certificates.crt
openclaw gateway run
```

อย่าพึ่งพาการเขียนตัวแปรนี้ลงใน `~/.openclaw/.env` เพียงอย่างเดียว; Node อ่าน
`NODE_EXTRA_CA_CERTS` ตอนเริ่มโปรเซส

## ตัวแปรสภาพแวดล้อมเดิม

OpenClaw อ่านเฉพาะตัวแปรสภาพแวดล้อม `OPENCLAW_*` เท่านั้น คำนำหน้าเดิม
`CLAWDBOT_*` และ `MOLTBOT_*` จากรุ่นก่อนหน้าจะถูกละเว้นอย่างเงียบๆ

หากยังมีตัวแปรใดตั้งค่าอยู่ในโปรเซส Gateway ตอนเริ่มต้น OpenClaw จะส่ง
คำเตือนการเลิกใช้งาน Node รายการเดียว (`OPENCLAW_LEGACY_ENV_VARS`) ที่แสดง
คำนำหน้าที่ตรวจพบและจำนวนรวม เปลี่ยนชื่อแต่ละค่าโดยแทนที่
คำนำหน้าเดิมด้วย `OPENCLAW_` (เช่น `CLAWDBOT_GATEWAY_TOKEN` →
`OPENCLAW_GATEWAY_TOKEN`); ชื่อเก่าไม่มีผลใดๆ
