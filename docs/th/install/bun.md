<Warning>
Bun ไม่สามารถเรียกใช้ OpenClaw CLI หรือ Gateway ได้ เนื่องจากไม่มี API `node:sqlite` ที่จำเป็น โปรดติดตั้ง Node เวอร์ชันที่รองรับสำหรับคำสั่งรันไทม์ทั้งหมดของ OpenClaw
</Warning>

ยังคงสามารถเลือกใช้ Bun เป็นตัวติดตั้งการขึ้นต่อกันและตัวเรียกใช้สคริปต์แพ็กเกจได้ ตัวจัดการแพ็กเกจเริ่มต้นยังคงเป็น `pnpm` ซึ่งรองรับอย่างสมบูรณ์และใช้โดยเครื่องมือเอกสาร Bun ไม่สามารถใช้ `pnpm-lock.yaml` และจะไม่สนใจรายการนี้

## การติดตั้ง

<Steps>
  <Step title="ติดตั้งการขึ้นต่อกัน">
    ```sh
    bun install
    ```

    `bun.lock` / `bun.lockb` ถูกละเว้นโดย Git จึงไม่ทำให้เกิดการเปลี่ยนแปลงที่ไม่จำเป็นในรีโพ หากต้องการข้ามการเขียนไฟล์ล็อกทั้งหมด:

    ```sh
    bun install --no-save
    ```

  </Step>
  <Step title="บิลด์และทดสอบ">
    ```sh
    bun run build
    bun run vitest run
    ```

    คำสั่งที่เรียกใช้ OpenClaw โดยตรงยังคงต้องทำงานผ่าน Node

  </Step>
</Steps>

## สคริปต์วงจรชีวิต

Bun จะบล็อกสคริปต์วงจรชีวิตของการขึ้นต่อกัน เว้นแต่จะได้รับความเชื่อถืออย่างชัดเจน สำหรับรีโพนี้ สคริปต์ที่มักถูกบล็อกต่อไปนี้ไม่จำเป็น:

- `baileys` `preinstall`: ตรวจสอบว่าเวอร์ชันหลักของ Node >= 20 (OpenClaw ต้องใช้ Node 22.22.3+, 24.15+ หรือ 25.9+ โดยแนะนำ Node 24)
- `protobufjs` `postinstall`: แสดงคำเตือนเกี่ยวกับรูปแบบเวอร์ชันที่เข้ากันไม่ได้ (ไม่มีอาร์ติแฟกต์จากการบิลด์)

หากพบปัญหารันไทม์ที่จำเป็นต้องใช้สคริปต์เหล่านี้ ให้กำหนดความเชื่อถืออย่างชัดเจน:

```sh
bun pm trust baileys protobufjs
```

## ข้อควรระวัง

สคริปต์แพ็กเกจบางรายการกำหนด `pnpm` ไว้ตายตัวภายใน (เช่น `check:docs`, `ui:*`, `protocol:check`) การเรียกใช้ผ่าน `bun run` จะยังคงเรียก `pnpm` ผ่านเชลล์ ดังนั้นให้เรียกใช้รายการเหล่านั้นผ่าน `pnpm` โดยตรง
