ผู้ให้บริการย้ายข้อมูล Hermes ที่รวมมาในชุดจะตรวจหาสถานะที่ `~/.hermes` แสดงตัวอย่างการเปลี่ยนแปลงทั้งหมดก่อนนำไปใช้ ปกปิดข้อมูลลับในแผนและรายงาน และเขียนข้อมูลสำรอง OpenClaw ที่ผ่านการตรวจสอบก่อนแตะต้องสิ่งใด

<Note>
การนำเข้าต้องใช้การตั้งค่า OpenClaw ใหม่ หากคุณมีสถานะ OpenClaw ภายในเครื่องอยู่แล้ว ให้รีเซ็ตการกำหนดค่า ข้อมูลประจำตัว เซสชัน และพื้นที่ทำงานก่อน หรือใช้ `openclaw migrate apply hermes` โดยตรงพร้อม `--overwrite` หลังจากตรวจสอบแผนแล้ว
</Note>

## วิธีนำเข้าสองแบบ

<Tabs>
  <Tab title="ตัวช่วยสร้างการเริ่มต้นใช้งาน">
    ตรวจพบ Hermes ที่ `~/.hermes` และแสดงตัวอย่างก่อนนำไปใช้

    ```bash
    openclaw onboard --flow import
    ```

    หรือระบุแหล่งที่มาโดยเฉพาะ:

    ```bash
    openclaw onboard --import-from hermes --import-source ~/.hermes
    ```

  </Tab>
  <Tab title="CLI">
    ใช้ `openclaw migrate` สำหรับการเรียกใช้ด้วยสคริปต์หรือการเรียกใช้ซ้ำ ดูข้อมูลอ้างอิงฉบับเต็มที่ [`openclaw migrate`](/th/cli/migrate)

    ```bash
    openclaw migrate hermes --dry-run    # แสดงตัวอย่างเท่านั้น
    openclaw migrate apply hermes --yes  # นำไปใช้โดยข้ามการยืนยัน
    ```

    เพิ่ม `--from <path>` เมื่อ Hermes อยู่ภายนอก `~/.hermes`

  </Tab>
</Tabs>

## สิ่งที่คงไว้เฉพาะในไฟล์เก็บถาวร

ผู้ให้บริการจะคัดลอกรายการต่อไปนี้ไปยังไดเรกทอรีรายงานการย้ายข้อมูลเพื่อให้ตรวจสอบด้วยตนเอง แต่จะ **ไม่** โหลดรายการเหล่านี้เข้าในการกำหนดค่าหรือข้อมูลประจำตัว OpenClaw ที่ใช้งานอยู่:

- `plugins/`
- `sessions/`
- `logs/`
- `cron/`
- `mcp-tokens/`
- `state.db`

OpenClaw ปฏิเสธที่จะเรียกใช้หรือเชื่อถือสถานะนี้โดยอัตโนมัติ เนื่องจากรูปแบบและสมมติฐานด้านความเชื่อถืออาจแตกต่างกันระหว่างระบบ หลังจากตรวจสอบไฟล์เก็บถาวรแล้ว ให้ย้ายสิ่งที่ต้องการด้วยตนเอง

## ขั้นตอนที่แนะนำ

<Steps>
  <Step title="แสดงตัวอย่างแผน">
    ```bash
    openclaw migrate hermes --dry-run
    ```

    แผนจะแสดงทุกสิ่งที่จะเปลี่ยนแปลง รวมถึงข้อขัดแย้ง รายการที่ข้าม และรายการที่ละเอียดอ่อน คีย์แบบซ้อนที่ดูเหมือนข้อมูลลับจะถูกปกปิดในผลลัพธ์

  </Step>
  <Step title="นำไปใช้พร้อมข้อมูลสำรอง">
    ```bash
    openclaw migrate apply hermes --yes
    ```

    OpenClaw จะสร้างและตรวจสอบข้อมูลสำรองก่อนนำไปใช้ ตัวอย่างแบบไม่โต้ตอบนี้จะนำเข้าเฉพาะสถานะที่ไม่ใช่ข้อมูลลับ เรียกใช้โดยไม่มี `--yes` เพื่อตอบคำถามเกี่ยวกับข้อมูลประจำตัวแบบโต้ตอบ หรือเพิ่ม `--include-secrets` เพื่อรวมข้อมูลประจำตัวที่รองรับในการเรียกใช้แบบไม่มีผู้ดูแล

  </Step>
  <Step title="เรียกใช้ doctor">
    ```bash
    openclaw doctor
    ```

    [Doctor](/th/gateway/doctor) จะนำการย้ายการกำหนดค่าที่ยังค้างอยู่ไปใช้อีกครั้ง และตรวจหาปัญหาที่เกิดขึ้นระหว่างการนำเข้า

  </Step>
  <Step title="เริ่มการทำงานใหม่และตรวจสอบ">
    ```bash
    openclaw gateway restart
    openclaw status
    ```

    ยืนยันว่า Gateway อยู่ในสถานะปกติ และโมเดล หน่วยความจำ และ Skills ที่นำเข้าได้รับการโหลดแล้ว

  </Step>
</Steps>

## การจัดการข้อขัดแย้ง

การนำไปใช้จะปฏิเสธที่จะดำเนินการต่อเมื่อแผนรายงานข้อขัดแย้ง (มีไฟล์หรือค่าการกำหนดค่าอยู่แล้วที่ปลายทาง)

<Warning>
เรียกใช้อีกครั้งด้วย `--overwrite` เฉพาะเมื่อตั้งใจแทนที่ปลายทางที่มีอยู่ ผู้ให้บริการอาจยังคงเขียนข้อมูลสำรองระดับรายการสำหรับไฟล์ที่ถูกเขียนทับไว้ในไดเรกทอรีรายงานการย้ายข้อมูล
</Warning>

ข้อขัดแย้งพบได้ไม่บ่อยในการติดตั้งใหม่ โดยทั่วไปจะปรากฏเมื่อคุณเรียกใช้การนำเข้าซ้ำกับการตั้งค่าที่มีการแก้ไขโดยผู้ใช้อยู่แล้ว

หากเกิดข้อขัดแย้งระหว่างการนำไปใช้ (ตัวอย่างเช่น เกิดสภาวะแข่งขันที่ไม่คาดคิดกับไฟล์การกำหนดค่า) Hermes จะทำเครื่องหมายรายการการกำหนดค่าที่เหลือซึ่งขึ้นต่อกันเป็น `skipped` พร้อมเหตุผล `blocked by earlier apply conflict` แทนการเขียนเพียงบางส่วน รายงานการย้ายข้อมูลจะบันทึกรายการที่ถูกบล็อกแต่ละรายการ เพื่อให้คุณแก้ไขข้อขัดแย้งต้นเหตุและเรียกใช้การนำเข้าอีกครั้งได้

## ข้อมูลลับ

`openclaw migrate` แบบโต้ตอบจะถามว่าจะนำเข้าข้อมูลประจำตัวสำหรับการยืนยันตัวตนที่ตรวจพบหรือไม่ โดยเลือกใช่เป็นค่าเริ่มต้น

- เมื่อยอมรับ ระบบจะนำเข้ารายการ OpenCode OpenAI OAuth และ GitHub Copilot จาก `auth.json` ของ OpenCode รวมถึง[คีย์ `.env` ที่รองรับ](/th/cli/migrate#supported-env-keys) ส่วนรายการ OAuth ใน `auth.json` ของ Hermes เองจะถูกรายงานเพื่อให้ยืนยันตัวตนกับ OpenAI ใหม่ด้วยตนเองหรือซ่อมแซมด้วย doctor แทน
- ใช้ `--no-auth-credentials` หรือตอบไม่ที่คำถาม เพื่อนำเข้าเฉพาะสถานะที่ไม่ใช่ข้อมูลลับ
- ใช้ `--include-secrets` เพื่อนำเข้าข้อมูลประจำตัวในการเรียกใช้ `--yes` แบบไม่มีผู้ดูแล
- ใช้แฟล็ก `--import-secrets` ของตัวช่วยสร้างการเริ่มต้นใช้งาน เพื่อนำเข้าข้อมูลประจำตัวจากตัวช่วยสร้าง

## ผลลัพธ์ JSON สำหรับระบบอัตโนมัติ

```bash
openclaw migrate hermes --dry-run --json
openclaw migrate apply hermes --json --yes
```

เมื่อใช้ `--json` โดยไม่มี `--yes` การนำไปใช้จะแสดงแผนและไม่เปลี่ยนแปลงสถานะ ซึ่งเป็นโหมดที่ปลอดภัยที่สุดสำหรับ CI และสคริปต์ที่ใช้ร่วมกัน
