ผู้ให้บริการการย้ายข้อมูล Hermes ที่รวมมาในชุดจะปฏิบัติตาม `HERMES_HOME` และโปรไฟล์ Hermes ที่ใช้งานอยู่ โดยใช้ `~/.hermes` เป็นทางเลือกสำรองบน macOS/Linux หรือ `%LOCALAPPDATA%\hermes` บน Windows ระบบจะแสดงตัวอย่างการเปลี่ยนแปลงทั้งหมดก่อนนำไปใช้ ปกปิดข้อมูลลับในแผนและรายงาน และเขียนข้อมูลสำรอง OpenClaw ที่ผ่านการตรวจสอบก่อนดำเนินการใดๆ พาธ `--from` ที่ระบุอย่างชัดเจนจะมีลำดับความสำคัญสูงสุดเสมอ

<Note>
การนำเข้าต้องใช้การตั้งค่า OpenClaw ใหม่ หากมีสถานะ OpenClaw ภายในเครื่องอยู่แล้ว ให้รีเซ็ตการกำหนดค่า ข้อมูลประจำตัว เซสชัน และพื้นที่ทำงานก่อน หรือใช้ `openclaw migrate apply hermes` โดยตรงกับ `--overwrite` หลังจากตรวจสอบแผนแล้ว
</Note>

## สิ่งที่คงไว้เฉพาะในที่เก็บถาวร

ผู้ให้บริการจะคัดลอกรายการเหล่านี้ไปยังไดเรกทอรีรายงานการย้ายข้อมูลเพื่อตรวจสอบด้วยตนเอง แต่จะ **ไม่** โหลดลงในการกำหนดค่าหรือข้อมูลประจำตัว OpenClaw ที่ใช้งานจริง:

- `plugins/`
- `sessions/`
- `logs/`
- `cron/`
- `mcp-tokens/`
- `plans/`, `workspace/`, `skins/` และ `kanban/`
- พื้นที่จัดเก็บ `pairing/` และ `platforms/` รวมถึงสถานะการกำหนดเส้นทาง/กระบวนการของ Gateway
- `state.db`, `hermes_state.db`, `projects.db`, `response_store.db`, `memory_store.db`, `verification_evidence.db`, `kanban.db` และ `retaindb_queue.db`

OpenClaw ปฏิเสธที่จะดำเนินการหรือเชื่อถือสถานะนี้โดยอัตโนมัติ เนื่องจากรูปแบบและสมมติฐานด้านความเชื่อถืออาจแตกต่างกันไปในแต่ละระบบ หลังจากตรวจสอบที่เก็บถาวรแล้ว ให้ย้ายสิ่งที่ต้องการด้วยตนเอง

## ขั้นตอนที่แนะนำ

<Steps>
  <Step title="แสดงตัวอย่างแผน">
    ```bash
    openclaw migrate hermes --dry-run
    ```

    แผนจะแสดงรายการทุกอย่างที่จะเปลี่ยนแปลง รวมถึงข้อขัดแย้ง รายการที่ข้าม และรายการที่ละเอียดอ่อน คีย์ซ้อนที่ดูเหมือนข้อมูลลับจะถูกปกปิดในผลลัพธ์

  </Step>
  <Step title="นำไปใช้พร้อมข้อมูลสำรอง">
    ```bash
    openclaw migrate apply hermes --yes
    ```

    OpenClaw จะสร้างและตรวจสอบข้อมูลสำรองก่อนนำไปใช้ ตัวอย่างแบบไม่โต้ตอบนี้จะนำเข้าเฉพาะสถานะที่ไม่ใช่ข้อมูลลับ เรียกใช้โดยไม่มี `--yes` เพื่อตอบพรอมต์ข้อมูลประจำตัวแบบโต้ตอบ หรือเพิ่ม `--include-secrets` เพื่อรวมข้อมูลประจำตัวที่รองรับในการเรียกใช้แบบไม่มีผู้ดูแล

  </Step>
  <Step title="เรียกใช้ Doctor">
    ```bash
    openclaw doctor
    ```

    [Doctor](/th/gateway/doctor) จะนำการย้ายการกำหนดค่าที่ยังค้างอยู่ไปใช้อีกครั้ง และตรวจหาปัญหาที่เกิดขึ้นระหว่างการนำเข้า

  </Step>
  <Step title="รีสตาร์ตและตรวจสอบ">
    ```bash
    openclaw gateway restart
    openclaw status
    ```

    ยืนยันว่า Gateway ทำงานเป็นปกติและโมเดล หน่วยความจำ และ Skills ที่นำเข้าได้รับการโหลดแล้ว

  </Step>
</Steps>

## การจัดการข้อขัดแย้ง

การนำไปใช้จะปฏิเสธที่จะดำเนินการต่อเมื่อแผนรายงานข้อขัดแย้ง (มีไฟล์หรือค่าการกำหนดค่าอยู่ที่เป้าหมายแล้ว)

<Warning>
เรียกใช้อีกครั้งด้วย `--overwrite` เฉพาะเมื่อตั้งใจแทนที่เป้าหมายที่มีอยู่ ผู้ให้บริการอาจยังคงเขียนข้อมูลสำรองระดับรายการสำหรับไฟล์ที่ถูกเขียนทับไว้ในไดเรกทอรีรายงานการย้ายข้อมูล
</Warning>

ข้อขัดแย้งเกิดขึ้นไม่บ่อยในการติดตั้งใหม่ โดยทั่วไปจะปรากฏเมื่อเรียกใช้การนำเข้าซ้ำกับการตั้งค่าที่มีการแก้ไขโดยผู้ใช้อยู่แล้ว

หากข้อขัดแย้งเกิดขึ้นระหว่างการนำไปใช้ (ตัวอย่างเช่น เกิดภาวะแข่งขันที่ไม่คาดคิดในไฟล์การกำหนดค่า) รายการนั้นจะถูกรายงานว่าเป็นข้อขัดแย้ง ขณะที่ไฟล์ Skills ข้อมูลประจำตัว ที่เก็บถาวร และรายการการกำหนดค่าอื่นที่เป็นอิสระจะดำเนินการต่อ แก้ไขรายการที่ขัดแย้งแล้วเรียกใช้การนำเข้าอีกครั้ง การนำเข้าหน่วยความจำที่เหมือนกันสามารถดำเนินการซ้ำได้โดยให้ผลลัพธ์เดิม

## ข้อมูลลับ

`openclaw migrate` แบบโต้ตอบจะถามว่าจะนำเข้าข้อมูลประจำตัวการยืนยันตัวตนที่ตรวจพบหรือไม่ โดยเลือกใช่เป็นค่าเริ่มต้น

- การยอมรับจะนำเข้ารายการ OAuth ของ Hermes OpenAI Codex ในปัจจุบัน รายการ OpenCode OpenAI OAuth และ GitHub Copilot ตลอดจน[คีย์ `.env` ที่รองรับ](/th/cli/migrate#supported-env-keys)
- ใช้ `--no-auth-credentials` หรือตอบไม่ที่พรอมต์ เพื่อนำเข้าเฉพาะสถานะที่ไม่ใช่ข้อมูลลับ
- ใช้ `--include-secrets` เพื่อนำเข้าข้อมูลประจำตัวในการเรียกใช้ `--yes` แบบไม่มีผู้ดูแล
- ใช้แฟล็ก `--import-secrets` ของตัวช่วยสร้างการเริ่มต้นใช้งานเพื่อนำเข้าข้อมูลประจำตัวจากตัวช่วยสร้าง

## เอาต์พุต JSON สำหรับระบบอัตโนมัติ

```bash
openclaw migrate hermes --dry-run --json
openclaw migrate apply hermes --json --yes
```

เมื่อใช้ `--json` โดยไม่มี `--yes` การนำไปใช้จะพิมพ์แผนและไม่เปลี่ยนแปลงสถานะ ซึ่งเป็นโหมดที่ปลอดภัยที่สุดสำหรับ CI และสคริปต์ที่ใช้ร่วมกัน
