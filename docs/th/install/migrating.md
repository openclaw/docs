OpenClaw รองรับเส้นทางการย้ายข้อมูลสามแบบ: นำเข้าจากระบบเอเจนต์อื่น ย้ายการติดตั้งที่มีอยู่ไปยังเครื่องใหม่ และอัปเกรด Plugin ในตำแหน่งเดิม

## นำเข้าจากระบบเอเจนต์อื่น

ใช้ผู้ให้บริการการย้ายข้อมูลที่รวมมาเพื่อดึงคำสั่ง MCP servers, Skills, การกำหนดค่าโมเดล และ API keys (เลือกเข้าร่วม) เข้าสู่ OpenClaw แผนจะถูกแสดงตัวอย่างก่อนมีการเปลี่ยนแปลงใดๆ ความลับจะถูกปกปิดในรายงาน และการนำไปใช้จะมีข้อมูลสำรองที่ผ่านการตรวจสอบรองรับ

  <Card title="ย้ายข้อมูลจาก Claude" href="/th/install/migrating-claude" icon="brain">
    นำเข้าสถานะ Claude Code และ Claude Desktop รวมถึง `CLAUDE.md`, MCP servers, Skills และคำสั่งโปรเจกต์
  </Card>

  </Card>

จุดเข้าใช้งาน CLI คือ [`openclaw migrate`](/th/cli/migrate) Onboarding ยังสามารถเสนอการย้ายข้อมูลเมื่อพบแหล่งที่รู้จัก (`openclaw onboard --flow import`)

## ย้าย OpenClaw ไปยังเครื่องใหม่

คัดลอก **ไดเรกทอรีสถานะ** (`~/.openclaw/` โดยค่าเริ่มต้น) และ **พื้นที่ทำงาน** ของคุณเพื่อเก็บรักษา:

- **การกำหนดค่า** — `openclaw.json` และการตั้งค่า Gateway ทั้งหมด
- **การยืนยันตัวตน** — `auth-profiles.json` ต่อเอเจนต์ (API keys รวมถึง OAuth) พร้อมสถานะของช่องทางหรือผู้ให้บริการใดๆ ภายใต้ `credentials/`
- **เซสชัน** — ประวัติการสนทนาและสถานะเอเจนต์
- **สถานะช่องทาง** — การเข้าสู่ระบบ WhatsApp, เซสชัน Telegram และสิ่งที่คล้ายกัน
- **ไฟล์พื้นที่ทำงาน** — `MEMORY.md`, `USER.md`, Skills และพรอมป์

<Tip>
เรียกใช้ `openclaw status` บนเครื่องเก่าเพื่อยืนยันเส้นทางไดเรกทอรีสถานะของคุณ โปรไฟล์แบบกำหนดเองใช้ `~/.openclaw-<profile>/` หรือเส้นทางที่ตั้งผ่าน `OPENCLAW_STATE_DIR`
</Tip>

### ขั้นตอนการย้ายข้อมูล

<Steps>
  <Step title="หยุด Gateway และสำรองข้อมูล">
    บนเครื่อง **เก่า** ให้หยุด Gateway เพื่อไม่ให้ไฟล์เปลี่ยนแปลงระหว่างการคัดลอก จากนั้นบีบอัดเก็บถาวร:

    ```bash
    openclaw gateway stop
    cd ~
    tar -czf openclaw-state.tgz .openclaw
    ```

    หากคุณใช้หลายโปรไฟล์ (เช่น `~/.openclaw-work`) ให้บีบอัดเก็บถาวรแต่ละโปรไฟล์แยกกัน

  </Step>

  <Step title="ติดตั้ง OpenClaw บนเครื่องใหม่">
    [ติดตั้ง](/th/install) CLI (และ Node หากจำเป็น) บนเครื่องใหม่ ไม่มีปัญหาหาก Onboarding สร้าง `~/.openclaw/` ใหม่ขึ้นมา คุณจะเขียนทับในขั้นตอนถัดไป
  </Step>

  <Step title="คัดลอกไดเรกทอรีสถานะและพื้นที่ทำงาน">
    โอนย้ายไฟล์บีบอัดผ่าน `scp`, `rsync -a` หรือไดรฟ์ภายนอก จากนั้นแตกไฟล์:

    ```bash
    cd ~
    tar -xzf openclaw-state.tgz
    ```

    ตรวจสอบให้แน่ใจว่ารวมไดเรกทอรีที่ซ่อนอยู่แล้ว และเจ้าของไฟล์ตรงกับผู้ใช้ที่จะเรียกใช้ Gateway

  </Step>

  <Step title="เรียกใช้ Doctor และตรวจสอบ">
    บนเครื่องใหม่ ให้เรียกใช้ [Doctor](/th/gateway/doctor) เพื่อใช้การย้ายการกำหนดค่าและซ่อมแซมบริการ:

    ```bash
    openclaw doctor
    openclaw gateway restart
    openclaw status
    ```

  </Step>
</Steps>

### ข้อผิดพลาดที่พบบ่อย

<AccordionGroup>
  <Accordion title="โปรไฟล์หรือ state-dir ไม่ตรงกัน">
    หาก Gateway เก่าใช้ `--profile` หรือ `OPENCLAW_STATE_DIR` แต่เครื่องใหม่ไม่ได้ใช้ ช่องทางจะแสดงเหมือนออกจากระบบแล้วและเซสชันจะว่างเปล่า ให้เปิด Gateway ด้วยโปรไฟล์หรือ state-dir **เดียวกัน** ที่คุณย้ายมา จากนั้นเรียกใช้ `openclaw doctor` อีกครั้ง
  </Accordion>

  <Accordion title="คัดลอกเฉพาะ openclaw.json">
    ไฟล์การกำหนดค่าเพียงอย่างเดียวไม่เพียงพอ โปรไฟล์การยืนยันตัวตนของโมเดลอยู่ภายใต้ `agents/<agentId>/agent/auth-profiles.json` และสถานะช่องทางกับผู้ให้บริการอยู่ภายใต้ `credentials/` ให้ย้ายไดเรกทอรีสถานะ **ทั้งหมด** เสมอ
  </Accordion>

  <Accordion title="สิทธิ์และเจ้าของ">
    หากคุณคัดลอกในฐานะ root หรือเปลี่ยนผู้ใช้ Gateway อาจอ่านข้อมูลรับรองไม่สำเร็จ ตรวจสอบให้แน่ใจว่าไดเรกทอรีสถานะและพื้นที่ทำงานเป็นของผู้ใช้ที่เรียกใช้ Gateway
  </Accordion>

  <Accordion title="โหมดระยะไกล">
    หาก UI ของคุณชี้ไปที่ Gateway **ระยะไกล** โฮสต์ระยะไกลจะเป็นเจ้าของเซสชันและพื้นที่ทำงาน ให้ย้ายโฮสต์ Gateway เอง ไม่ใช่แล็ปท็อป local ของคุณ ดู [FAQ](/th/help/faq#where-things-live-on-disk)
  </Accordion>

  <Accordion title="ความลับในข้อมูลสำรอง">
    ไดเรกทอรีสถานะมีโปรไฟล์การยืนยันตัวตน ข้อมูลรับรองช่องทาง และสถานะผู้ให้บริการอื่นๆ เก็บข้อมูลสำรองแบบเข้ารหัส หลีกเลี่ยงช่องทางโอนย้ายที่ไม่ปลอดภัย และหมุนเวียนคีย์หากคุณสงสัยว่ามีการเปิดเผย
  </Accordion>
</AccordionGroup>

### รายการตรวจสอบการยืนยัน

บนเครื่องใหม่ ให้ยืนยันว่า:

- [ ] `openclaw status` แสดงว่า Gateway กำลังทำงาน
- [ ] ช่องทางยังคงเชื่อมต่ออยู่ (ไม่ต้องจับคู่อีกครั้ง)
- [ ] แดชบอร์ดเปิดได้และแสดงเซสชันที่มีอยู่
- [ ] ไฟล์พื้นที่ทำงาน (หน่วยความจำ การกำหนดค่า) มีอยู่

## อัปเกรด Plugin ในตำแหน่งเดิม

การอัปเกรด Plugin ในตำแหน่งเดิมจะเก็บรักษา plugin id และ config keys เดิมไว้ แต่อาจย้ายสถานะบนดิสก์เข้าสู่เลย์เอาต์ปัจจุบัน คู่มือการอัปเกรดเฉพาะ Plugin อยู่เคียงข้างช่องทางของตน:

- [การย้าย Matrix](/th/channels/matrix-migration): ข้อจำกัดการกู้คืนสถานะที่เข้ารหัส พฤติกรรมสแนปชอตอัตโนมัติ และคำสั่งกู้คืนด้วยตนเอง

## ที่เกี่ยวข้อง

- [`openclaw migrate`](/th/cli/migrate): อ้างอิง CLI สำหรับการนำเข้าข้ามระบบ
- [ภาพรวมการติดตั้ง](/th/install): วิธีการติดตั้งทั้งหมด
- [Doctor](/th/gateway/doctor): การตรวจสุขภาพหลังการย้ายข้อมูล
- [ถอนการติดตั้ง](/th/install/uninstall): ลบ OpenClaw อย่างเรียบร้อย
