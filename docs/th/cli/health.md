# `openclaw health`

ดึงสแนปช็อตสถานะความพร้อมใช้งานจาก Gateway ที่กำลังทำงานผ่าน WebSocket RPC (CLI จะไม่เชื่อมต่อซ็อกเก็ตของช่องทางโดยตรง)

## ตัวเลือก

| แฟล็ก            | ค่าเริ่มต้น | คำอธิบาย                                                                                     |
| ---------------- | ----------- | -------------------------------------------------------------------------------------------- |
| `--json`         | `false`     | แสดงผล JSON ที่เครื่องอ่านได้แทนข้อความ                                                     |
| `--timeout <ms>` | `10000`     | ระยะหมดเวลาการเชื่อมต่อเป็นมิลลิวินาที                                                      |
| `--verbose`      | `false`     | บังคับตรวจสอบแบบสดและขยายผลลัพธ์ให้ครอบคลุมบัญชีและเอเจนต์ที่กำหนดค่าทั้งหมด             |
| `--debug`        | `false`     | ชื่อแทนสำหรับ `--verbose`                                                                    |

ตัวอย่าง:

```bash
openclaw health
openclaw health --json
openclaw health --timeout 2500
openclaw health --verbose
openclaw health --debug
```

## ลักษณะการทำงาน

- เมื่อไม่มี `--verbose` Gateway สามารถส่งคืนสแนปช็อตที่แคชไว้ (มีความใหม่ไม่เกิน 60 วินาทีและไม่เปลี่ยนแปลงจากสถานะรันไทม์แบบสดของช่องทาง) และรีเฟรชในเบื้องหลังสำหรับผู้เรียกครั้งถัดไป
- `--verbose` บังคับตรวจสอบแบบสด (ตรวจสอบบัญชีแยกตามช่องทาง) แสดงรายละเอียดการเชื่อมต่อ Gateway และขยายผลลัพธ์ที่มนุษย์อ่านได้ให้ครอบคลุมบัญชีและเอเจนต์ที่กำหนดค่าทั้งหมด แทนที่จะแสดงเฉพาะเอเจนต์เริ่มต้น
- `--json` ส่งคืนสแนปช็อตฉบับเต็มเสมอ ได้แก่ ช่องทาง ผลการตรวจสอบแยกตามบัญชี สถานะการโหลด Plugin สถานะการกักกันของกลไกบริบท สถานะแคชราคาของโมเดล สถานะความพร้อมใช้งานของลูปเหตุการณ์ และที่เก็บเซสชันแยกตามเอเจนต์
