# `openclaw health`

ดึงสแนปช็อตสถานะความพร้อมใช้งานจาก Gateway ที่กำลังทำงานผ่าน WebSocket RPC (CLI จะไม่เชื่อมต่อซ็อกเก็ตของช่องทางโดยตรง)

## ตัวเลือก

| แฟล็ก             | ค่าเริ่มต้น | คำอธิบาย                                                                       |
| ---------------- | ------- | --------------------------------------------------------------------------------- |
| `--json`         | `false` | แสดงผลเป็น JSON ที่เครื่องอ่านได้แทนข้อความ                                      |
| `--timeout <ms>` | `10000` | ระยะหมดเวลาการเชื่อมต่อเป็นมิลลิวินาที                                               |
| `--verbose`      | `false` | บังคับให้ตรวจสอบแบบสดและขยายผลลัพธ์ให้ครอบคลุมบัญชีและเอเจนต์ทั้งหมดที่กำหนดค่าไว้ |
| `--debug`        | `false` | ชื่อแทนสำหรับ `--verbose`                                                            |

ตัวอย่าง:

```bash
openclaw health
openclaw health --json
openclaw health --timeout 2500
openclaw health --verbose
openclaw health --debug
```

## ลักษณะการทำงาน

- หากไม่มี `--verbose` Gateway สามารถส่งคืนสแนปช็อตที่แคชไว้ (มีความสดใหม่สูงสุด 60 วินาทีและไม่เปลี่ยนแปลงจากสถานะรันไทม์ของช่องทางแบบสด) และรีเฟรชในเบื้องหลังสำหรับผู้เรียกถัดไป
- `--verbose` บังคับให้ตรวจสอบแบบสด (ตรวจสอบบัญชีแยกตามช่องทาง) แสดงรายละเอียดการเชื่อมต่อ Gateway และขยายผลลัพธ์ที่มนุษย์อ่านได้ให้ครอบคลุมบัญชีและเอเจนต์ทั้งหมดที่กำหนดค่าไว้ แทนที่จะแสดงเฉพาะเอเจนต์เริ่มต้น
- `--json` ส่งคืนสแนปช็อตฉบับเต็มเสมอ ได้แก่ ช่องทาง การตรวจสอบแยกตามบัญชี สถานะการโหลด Plugin สถานะการกักกันของเอนจินบริบท สถานะแคชราคาของโมเดล สถานะความพร้อมใช้งานของลูปเหตุการณ์ รายการที่ประมวลผลไม่สำเร็จในคิวการนำส่ง และที่เก็บเซสชันแยกตามเอเจนต์
- เมื่อการนำส่งขาออกหรือเหตุการณ์ช่องทางขาเข้าถูกย้ายไปยังรายการที่ประมวลผลไม่สำเร็จ ผลลัพธ์แบบข้อความจะรายงานจำนวนและระยะเวลาตั้งแต่เกิดความล้มเหลวที่เก่าที่สุด จำนวนขาเข้าจะจัดกลุ่มตามบัญชีช่องทาง โดยสามารถตรวจสอบหรือกู้คืนแต่ละเหตุการณ์ได้ด้วย [`openclaw channels dead-letters`](/th/cli/channels#inbound-dead-letters)
