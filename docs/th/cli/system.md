# `openclaw system`

ตัวช่วยระดับระบบสำหรับ Gateway: จัดคิวเหตุการณ์ระบบ ควบคุม Heartbeat และดูสถานะการปรากฏตัว

คำสั่งย่อย `system` ทั้งหมดใช้ RPC ของ Gateway และรองรับแฟล็กไคลเอนต์ร่วมกันดังนี้:

| แฟล็ก              | ค่าเริ่มต้น                              | คำอธิบาย                                                                                                                                                                                            |
| ----------------- | ------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `--url <url>`     | `gateway.remote.url` เมื่อตั้งค่าไว้ | URL WebSocket ของ Gateway                                                                                                                                                                                 |
| `--token <token>` | ไม่มี                                 | โทเค็นของ Gateway (หากจำเป็น)                                                                                                                                                                           |
| `--timeout <ms>`  | `30000`                              | ระยะหมดเวลาของ RPC หน่วยเป็นมิลลิวินาที                                                                                                                                                                           |
| `--expect-final`  | ปิด                                  | รอการตอบกลับสุดท้าย (เอเจนต์)                                                                                                                                                                       |
| `--json`          | ปิด                                  | ส่งออก JSON โดย `heartbeat last/enable/disable` และ `system presence` จะแสดงเพย์โหลด JSON ดิบของ RPC เสมอโดยไม่คำนึงถึงแฟล็กนี้ ส่วน `system event` ใช้แฟล็กนี้เพื่อสลับระหว่าง JSON กับบรรทัดข้อความธรรมดา `ok` |

## คำสั่งทั่วไป

```bash
openclaw system event --text "Check for urgent follow-ups" --mode now
openclaw system event --text "Check for urgent follow-ups" --url ws://127.0.0.1:18789 --token "$OPENCLAW_GATEWAY_TOKEN"
openclaw system heartbeat enable
openclaw system heartbeat last
openclaw system presence
```

## `system event`

โดยค่าเริ่มต้น จะจัดคิวเหตุการณ์ระบบไว้ในเซสชัน **หลัก** Heartbeat ครั้งถัดไปจะแทรกเหตุการณ์ดังกล่าวเป็นบรรทัด `System:` ในพรอมต์ ใช้ `--mode now` เพื่อทริกเกอร์ Heartbeat ทันที ส่วน `next-heartbeat` (ค่าเริ่มต้น) จะรอรอบการทำงานตามกำหนดครั้งถัดไป

ส่ง `--session-key` เพื่อระบุเซสชันเป้าหมาย เช่น เพื่อส่งต่อการเสร็จสิ้นของงานแบบอะซิงโครนัสกลับไปยังช่องทางที่เริ่มงานนั้น

<Note>
**ข้อยกเว้นด้านเวลากับ `--session-key`:** เมื่อระบุ `--session-key`
`--mode next-heartbeat` จะเปลี่ยนเป็นการปลุกเซสชันเป้าหมายทันทีแทนการรอรอบการทำงานตามกำหนดครั้งถัดไป การปลุกแบบระบุเป้าหมายใช้เจตนาของ Heartbeat เป็น
`immediate` จึงข้ามด่านตรวจสอบว่ายังไม่ถึงกำหนดของตัวรัน ซึ่งมิฉะนั้นจะเลื่อน (และทำให้สูญหายโดยพฤตินัย) การปลุกที่มีเจตนาเป็น `event` หากต้องการส่งแบบหน่วงเวลา
ให้ละเว้น `--session-key` เพื่อให้เหตุการณ์เข้าสู่เซสชันหลักและถูกส่งไปกับ Heartbeat ปกติครั้งถัดไป
</Note>

แฟล็ก:

- `--text <text>`: ข้อความเหตุการณ์ระบบที่จำเป็นต้องระบุ
- `--mode <mode>`: `now` หรือ `next-heartbeat` (ค่าเริ่มต้น)
- `--session-key <sessionKey>`: ไม่บังคับ ระบุเซสชันของเอเจนต์เป็นเป้าหมายแทนเซสชันหลักของเอเจนต์ คีย์ที่ไม่ได้เป็นของเอเจนต์ที่แก้ไขได้จะย้อนกลับไปใช้เซสชันหลักของเอเจนต์

## `system heartbeat last|enable|disable`

- `last`: แสดงเหตุการณ์ Heartbeat ล่าสุด
- `enable`: เปิด Heartbeat อีกครั้ง (ใช้เมื่อถูกปิดไว้)
- `disable`: หยุด Heartbeat ชั่วคราว

## `system presence`

แสดงรายการสถานะการปรากฏตัวของระบบปัจจุบันที่ Gateway รับรู้ (Node อินสแตนซ์ และบรรทัดสถานะที่คล้ายกัน)

## หมายเหตุ

- ต้องมี Gateway ที่กำลังทำงานและเข้าถึงได้ด้วยการกำหนดค่าปัจจุบันของคุณ (ภายในเครื่องหรือระยะไกล)
- เหตุการณ์ระบบเป็นข้อมูลชั่วคราวและจะไม่ถูกเก็บไว้เมื่อเริ่มระบบใหม่
