# `openclaw status`

การวินิจฉัยสำหรับช่องทาง + เซสชัน

```bash
openclaw status
openclaw status --all
openclaw status --deep
openclaw status --usage
```

หมายเหตุ:

- `--deep` เรียกใช้การตรวจสอบแบบสด (WhatsApp Web + Telegram + Discord + Slack + Signal)
- `openclaw status` แบบปกติจะอยู่บนเส้นทางอ่านอย่างเดียวที่รวดเร็ว และทำเครื่องหมายหน่วยความจำเป็น `not checked` แทนที่จะเป็นไม่พร้อมใช้งานเมื่อข้ามการตรวจสอบหน่วยความจำ การตรวจสอบความปลอดภัยแบบหนัก ความเข้ากันได้ของ Plugin และโพรบเวกเตอร์หน่วยความจำจะปล่อยให้ `openclaw status --all`, `openclaw status --deep`, `openclaw security audit` และ `openclaw memory status --deep` จัดการ
- `status --json --all` รายงานรายละเอียดหน่วยความจำจากรันไทม์ Plugin หน่วยความจำที่ใช้งานอยู่ ซึ่งเลือกโดย `plugins.slots.memory` Plugin หน่วยความจำแบบกำหนดเองสามารถปล่อยให้ `agents.defaults.memorySearch.enabled` ในตัวปิดอยู่ และยังรายงานไฟล์ ชังก์ เวกเตอร์ และสถานะ FTS ของตนเองได้
- `--usage` พิมพ์ช่วงเวลาการใช้งานของผู้ให้บริการที่ปรับมาตรฐานแล้วในรูปแบบ `X% left`
- เอาต์พุตสถานะเซสชันจะแยก `Execution:` ออกจาก `Runtime:` `Execution` คือเส้นทางแซนด์บ็อกซ์ (`direct`, `docker/*`) ส่วน `Runtime` บอกว่าเซสชันกำลังใช้ `OpenClaw Pi Default`, `OpenAI Codex`, แบ็กเอนด์ CLI หรือแบ็กเอนด์ ACP เช่น `codex (acp/acpx)` ดู [รันไทม์ของเอเจนต์](/th/concepts/agent-runtimes) สำหรับความแตกต่างระหว่างผู้ให้บริการ/โมเดล/รันไทม์
- ฟิลด์ดิบ `usage_percent` / `usagePercent` ของ MiniMax คือโควตาที่เหลืออยู่ ดังนั้น OpenClaw จะกลับค่าก่อนแสดงผล ฟิลด์แบบนับจำนวนจะมีสิทธิ์เหนือกว่าเมื่อมีอยู่ การตอบกลับ `model_remains` จะเลือกเอนทรีโมเดลแชต สร้างป้ายกำกับช่วงเวลาจากไทม์สแตมป์เมื่อจำเป็น และรวมชื่อโมเดลไว้ในป้ายกำกับแผน
- เมื่อสแนปช็อตเซสชันปัจจุบันมีข้อมูลแบบเบาบาง `/status` สามารถเติมค่าตัวนับโทเค็นและแคชย้อนหลังจากบันทึกการใช้งานทรานสคริปต์ล่าสุดได้ ค่าสดที่ไม่เป็นศูนย์ที่มีอยู่จะยังมีสิทธิ์เหนือค่าทางเลือกจากทรานสคริปต์
- ทางเลือกจากทรานสคริปต์ยังสามารถกู้คืนป้ายกำกับโมเดลรันไทม์ที่ใช้งานอยู่ได้เมื่อเอนทรีเซสชันสดไม่มีข้อมูลนี้ หากโมเดลในทรานสคริปต์นั้นต่างจากโมเดลที่เลือก status จะ resolve หน้าต่างบริบทเทียบกับโมเดลรันไทม์ที่กู้คืนมา แทนที่จะใช้โมเดลที่เลือก
- สำหรับการคำนวณขนาดพรอมต์ ทางเลือกจากทรานสคริปต์จะเลือกผลรวมที่เน้นพรอมต์ซึ่งมีค่ามากกว่า เมื่อเมทาดาทาของเซสชันหายไปหรือมีค่าน้อยกว่า เพื่อไม่ให้เซสชันผู้ให้บริการแบบกำหนดเองยุบเหลือการแสดงผลโทเค็นเป็น `0`
- เอาต์พุตรวมที่เก็บเซสชันต่อเอเจนต์เมื่อมีการกำหนดค่าเอเจนต์หลายตัว
- ภาพรวมรวมสถานะการติดตั้ง/รันไทม์ของ Gateway + บริการโฮสต์ Node เมื่อพร้อมใช้งาน
- ภาพรวมรวมช่องทางอัปเดต + SHA ของ git (สำหรับเช็กเอาต์จากซอร์ส)
- ข้อมูลอัปเดตจะแสดงในภาพรวม หากมีอัปเดตพร้อมใช้งาน status จะพิมพ์คำแนะนำให้เรียกใช้ `openclaw update` (ดู [การอัปเดต](/th/install/updating))
- พื้นผิวสถานะแบบอ่านอย่างเดียว (`status`, `status --json`, `status --all`) จะ resolve SecretRefs ที่รองรับสำหรับเส้นทางคอนฟิกเป้าหมายเมื่อทำได้
- หากกำหนดค่า SecretRef ของช่องทางที่รองรับไว้แต่ไม่พร้อมใช้งานในเส้นทางคำสั่งปัจจุบัน status จะยังคงอ่านอย่างเดียวและรายงานเอาต์พุตที่ลดระดับแทนที่จะล่ม เอาต์พุตสำหรับมนุษย์จะแสดงคำเตือน เช่น “โทเค็นที่กำหนดค่าไว้ไม่พร้อมใช้งานในเส้นทางคำสั่งนี้” และเอาต์พุต JSON จะรวม `secretDiagnostics`
- เมื่อการ resolve SecretRef เฉพาะคำสั่งสำเร็จ status จะเลือกสแนปช็อตที่ resolve แล้ว และล้างเครื่องหมายช่องทางชั่วคราว “secret unavailable” ออกจากเอาต์พุตสุดท้าย
- `status --all` รวมแถวภาพรวม Secrets และส่วนวินิจฉัยที่สรุปการวินิจฉัย secret (ตัดให้สั้นลงเพื่อให้อ่านง่าย) โดยไม่หยุดการสร้างรายงาน
