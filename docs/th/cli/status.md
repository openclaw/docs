การวินิจฉัยสำหรับช่องทางและเซสชัน

```bash
openclaw status
openclaw status --all
openclaw status --deep
openclaw status --usage
```

| แฟล็ก                    | คำอธิบาย                                                                                                     |
| ----------------------- | --------------------------------------------------------------------------------------------------------------- |
| `--all`                 | การวินิจฉัยแบบเต็ม (อ่านอย่างเดียวและนำไปวางได้) รวมการตรวจสอบความปลอดภัย ความเข้ากันได้ของ Plugin และการตรวจสอบเวกเตอร์หน่วยความจำ |
| `--deep`                | เรียกใช้การตรวจสอบแบบสด (WhatsApp Web + Telegram + Discord + Slack + Signal) และเปิดใช้การตรวจสอบความปลอดภัยด้วย         |
| `--usage`               | แสดงช่วงการใช้งานของผู้ให้บริการที่ปรับให้อยู่ในรูปแบบมาตรฐานเป็น `X% left`                                                          |
| `--json`                | เอาต์พุตที่เครื่องอ่านได้                                                                                        |
| `--verbose` / `--debug` | แสดงผลการระบุเป้าหมาย Gateway แบบดิบก่อนรายงานด้วย                                                 |

`openclaw status` แบบปกติจะยังคงใช้เส้นทางอ่านอย่างเดียวที่รวดเร็ว และทำเครื่องหมายหน่วยความจำเป็น
`not checked` แทนไม่พร้อมใช้งานเมื่อข้ามการตรวจสอบหน่วยความจำ ส่วนการ
ตรวจสอบความปลอดภัยแบบละเอียด ความเข้ากันได้ของ Plugin และการตรวจสอบเวกเตอร์หน่วยความจำจะดำเนินการโดย
`openclaw status --all`, `openclaw status --deep`, `openclaw security audit`
และ `openclaw memory status --deep`

## การระบุเซสชันและโมเดล

- เอาต์พุตสถานะเซสชันจะแยก `Execution:` ออกจาก `Runtime:` โดย `Execution`
  คือพาธแซนด์บ็อกซ์ (`direct`, `docker/*`) ขณะที่ `Runtime` ระบุว่า
  เซสชันกำลังใช้ `OpenClaw Default`, `OpenAI Codex`, แบ็กเอนด์
  CLI หรือแบ็กเอนด์ ACP เช่น `codex (acp/acpx)` ดูความแตกต่างระหว่าง
  ผู้ให้บริการ/โมเดล/รันไทม์ได้ที่ [รันไทม์ของเอเจนต์](/th/concepts/agent-runtimes)
- เมื่อสแนปช็อตของเซสชันปัจจุบันมีข้อมูลไม่ครบ `/status` สามารถเติมตัวนับโทเค็น
  และแคชจากบันทึกการใช้งานในทรานสคริปต์ล่าสุดได้ แต่ค่าปัจจุบันที่ไม่ใช่ศูนย์
  จะยังมีลำดับความสำคัญเหนือค่าทดแทนจากทรานสคริปต์
- การใช้ทรานสคริปต์เป็นข้อมูลทดแทนยังสามารถกู้คืนป้ายกำกับโมเดลรันไทม์ที่ใช้งานอยู่ได้เมื่อ
  รายการเซสชันแบบสดไม่มีข้อมูลดังกล่าว หากโมเดลจากทรานสคริปต์นั้นแตกต่าง
  จากโมเดลที่เลือก สถานะจะระบุขนาดหน้าต่างบริบทตาม
  โมเดลรันไทม์ที่กู้คืนมาแทนโมเดลที่เลือก
- สำหรับการคำนวณขนาดพรอมต์ การใช้ทรานสคริปต์เป็นข้อมูลทดแทนจะเลือกผลรวม
  ที่เน้นพรอมต์ซึ่งมีค่ามากกว่าเมื่อข้อมูลเมตาของเซสชันไม่มีหรือมีค่าน้อยกว่า เพื่อให้
  เซสชันของผู้ให้บริการแบบกำหนดเองไม่ยุบเหลือการแสดงโทเค็น `0`
- เมื่อเซสชันถูกตรึงกับโมเดลที่แตกต่างจากโมเดลหลักที่กำหนดค่าไว้
  สถานะจะแสดงทั้งสองค่า เหตุผล (`session override`) และ
  คำแนะนำ `/model default` โมเดลหลักที่กำหนดค่าไว้จะมีผลกับเซสชันใหม่หรือ
  เซสชันที่ไม่ได้ตรึง ส่วนเซสชันที่ตรึงอยู่แล้วจะคงการเลือกของเซสชันไว้
  จนกว่าจะล้าง
- เอาต์พุตจะรวมที่เก็บเซสชันของแต่ละเอเจนต์เมื่อ
  กำหนดค่าเอเจนต์หลายตัว

## การใช้งานและโควตา

- `--usage` จะแสดงช่วงการใช้งานของผู้ให้บริการที่ปรับให้อยู่ในรูปแบบมาตรฐานเป็น `X% left`
- ฟิลด์ดิบ `usage_percent` / `usagePercent` ของ MiniMax คือโควตาที่เหลือ
  ดังนั้น OpenClaw จึงกลับค่าก่อนแสดงผล โดยฟิลด์ที่อิงจำนวนจะมีลำดับความสำคัญเมื่อ
  มีอยู่ การตอบกลับ `model_remains` จะเลือกใช้รายการโมเดลแชต คำนวณ
  ป้ายกำกับช่วงเวลาจากการประทับเวลาเมื่อจำเป็น และใส่ชื่อโมเดลไว้ใน
  ป้ายกำกับแผน
- ความล้มเหลวในการรีเฟรชราคาโมเดลจะแสดงเป็นคำเตือนด้านราคาแบบไม่บังคับ
  ซึ่งไม่ได้หมายความว่า Gateway หรือช่องทางทำงานผิดปกติ

## ภาพรวมและสถานะการอัปเดต

- ภาพรวมจะแสดงสถานะการติดตั้ง/รันไทม์ของบริการโฮสต์ Gateway + Node เมื่อ
  มีข้อมูล พร้อมเวลาทำงานของกระบวนการ Gateway และเวลาทำงานของระบบโฮสต์แบบย่อ
- ภาพรวมจะแสดงช่องทางการอัปเดต + git SHA (สำหรับการเช็กเอาต์จากซอร์ส)
- ข้อมูลการอัปเดตจะแสดงในภาพรวม หากมีการอัปเดต สถานะ
  จะแสดงคำแนะนำให้เรียกใช้ `openclaw update` (ดู [การอัปเดต](/th/install/updating))

## ข้อมูลลับ

- เมื่อ Gateway ที่กำลังทำงานมีเจ้าของ SecretRef ที่ถูกแยกอย่างน้อยหนึ่งรายจากการเริ่มต้น การโหลดซ้ำ หรือการเขียนการกำหนดค่า สถานะจะรวม `degradedSecretOwners` ใน JSON และแถวภาพรวม **ข้อมูลลับเสื่อมสภาพ** ในเอาต์พุตสำหรับมนุษย์ แต่ละรายการจะระบุเจ้าของ สถานะการเสื่อมสภาพ (`cold` หรือ `stale`) พาธการกำหนดค่า และเหตุผลที่ปกปิดข้อมูล เจ้าของแบบเย็นจะไม่พร้อมใช้งาน ส่วนเจ้าของที่มีข้อมูลเก่าจะทำงานต่อด้วยค่าล่าสุดที่ทราบว่าใช้ได้
- พื้นผิวสถานะแบบอ่านอย่างเดียว (`status`, `status --json`, `status --all`)
  จะระบุ SecretRef ที่รองรับสำหรับพาธการกำหนดค่าเป้าหมายเมื่อ
  ทำได้
- หากกำหนดค่า SecretRef ของช่องทางที่รองรับไว้แต่ไม่พร้อมใช้งานใน
  เส้นทางคำสั่งปัจจุบัน สถานะจะยังคงเป็นแบบอ่านอย่างเดียวและรายงานเอาต์พุตที่เสื่อมสภาพ
  แทนที่จะขัดข้อง เอาต์พุตสำหรับมนุษย์จะแสดงคำเตือน เช่น "โทเค็นที่กำหนดค่าไว้
  ไม่พร้อมใช้งานในเส้นทางคำสั่งนี้" และเอาต์พุต JSON จะรวม
  `secretDiagnostics`
- เมื่อการระบุ SecretRef ภายในคำสั่งสำเร็จ สถานะจะเลือกใช้
  สแนปช็อตที่ระบุได้และล้างเครื่องหมายช่องทาง "ข้อมูลลับไม่พร้อมใช้งาน" ชั่วคราว
  ออกจากเอาต์พุตสุดท้าย
- `status --all` จะรวมแถวภาพรวมข้อมูลลับและส่วนการวินิจฉัย
  ที่สรุปการวินิจฉัยข้อมูลลับ (ตัดทอนเพื่อให้อ่านง่าย) โดยไม่
  หยุดการสร้างรายงาน

## หน่วยความจำ

`status --json --all` รายงานรายละเอียดหน่วยความจำจากรันไทม์ Plugin หน่วยความจำที่ใช้งานอยู่
ซึ่งเลือกโดย `plugins.slots.memory` Plugin หน่วยความจำแบบกำหนดเองสามารถปล่อยให้
`agents.defaults.memorySearch.enabled` ในตัวถูกปิดใช้งานและยังคงรายงาน
ไฟล์ ส่วนข้อมูล เวกเตอร์ และสถานะ FTS ของตนเองได้
