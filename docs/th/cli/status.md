การวินิจฉัยสำหรับช่องทาง + เซสชัน

```bash
openclaw status
openclaw status --all
openclaw status --deep
openclaw status --usage
```

หมายเหตุ:

- `--deep` เรียกใช้โพรบแบบสด (WhatsApp Web + Telegram + Discord + Slack + Signal)
- `openclaw status` แบบธรรมดายังคงใช้เส้นทางอ่านอย่างเดียวที่รวดเร็ว และทำเครื่องหมายหน่วยความจำเป็น `not checked` แทนการระบุว่าไม่พร้อมใช้งานเมื่อข้ามการตรวจสอบหน่วยความจำ โพรบตรวจสอบความปลอดภัยหนัก ความเข้ากันได้ของ Plugin และเวกเตอร์หน่วยความจำจะปล่อยไว้ให้ `openclaw status --all`, `openclaw status --deep`, `openclaw security audit` และ `openclaw memory status --deep`
- `status --json --all` รายงานรายละเอียดหน่วยความจำจากรันไทม์ Plugin หน่วยความจำที่ใช้งานอยู่ซึ่งเลือกโดย `plugins.slots.memory` Plugin หน่วยความจำแบบกำหนดเองสามารถปิดใช้งาน `agents.defaults.memorySearch.enabled` ที่มีมาในตัวไว้ และยังคงรายงานไฟล์ ชังก์ เวกเตอร์ และสถานะ FTS ของตัวเองได้
- `--usage` พิมพ์หน้าต่างการใช้งานของผู้ให้บริการที่ทำให้เป็นมาตรฐานในรูปแบบ `X% left`
- เอาต์พุตสถานะเซสชันแยก `Execution:` ออกจาก `Runtime:` `Execution` คือเส้นทางแซนด์บ็อกซ์ (`direct`, `docker/*`) ส่วน `Runtime` บอกคุณว่าเซสชันกำลังใช้ `OpenClaw Pi Default`, `OpenAI Codex`, แบ็กเอนด์ CLI หรือแบ็กเอนด์ ACP เช่น `codex (acp/acpx)` ดู [รันไทม์ของเอเจนต์](/th/concepts/agent-runtimes) สำหรับความแตกต่างระหว่างผู้ให้บริการ/โมเดล/รันไทม์
- ฟิลด์ดิบ `usage_percent` / `usagePercent` ของ MiniMax คือโควตาที่เหลืออยู่ ดังนั้น OpenClaw จะกลับค่าก่อนแสดงผล ฟิลด์แบบอิงจำนวนจะมีสิทธิ์เหนือกว่าเมื่อมีอยู่ การตอบกลับ `model_remains` จะเลือกเอนทรีโมเดลแชตก่อน อนุมานป้ายกำกับหน้าต่างจากไทม์สแตมป์เมื่อจำเป็น และรวมชื่อโมเดลไว้ในป้ายกำกับแผน
- เมื่อสแนปช็อตเซสชันปัจจุบันมีข้อมูลน้อย `/status` สามารถเติมตัวนับโทเค็นและแคชย้อนหลังจากบันทึกการใช้งานทรานสคริปต์ล่าสุด ค่าจริงแบบสดที่ไม่เป็นศูนย์ที่มีอยู่จะยังคงมีสิทธิ์เหนือกว่าค่าสำรองจากทรานสคริปต์
- `/status` รวมเวลาทำงานของกระบวนการ Gateway และเวลาทำงานของระบบโฮสต์แบบกระชับ
- การสำรองจากทรานสคริปต์ยังสามารถกู้คืนป้ายกำกับโมเดลรันไทม์ที่ใช้งานอยู่ได้เมื่อเอนทรีเซสชันสดไม่มีข้อมูลนั้น หากโมเดลทรานสคริปต์นั้นต่างจากโมเดลที่เลือก สถานะจะ resolve หน้าต่างบริบทกับโมเดลรันไทม์ที่กู้คืนมาแทนโมเดลที่เลือก
- สำหรับการนับขนาดพรอมป์ การสำรองจากทรานสคริปต์จะเลือกยอดรวมที่เน้นพรอมป์และมีค่ามากกว่าเมื่อเมทาดาทาเซสชันหายไปหรือมีค่าน้อยกว่า เพื่อให้เซสชันผู้ให้บริการแบบกำหนดเองไม่ยุบเป็นการแสดงโทเค็น `0`
- เอาต์พุตรวมที่เก็บเซสชันรายเอเจนต์เมื่อกำหนดค่าเอเจนต์หลายตัว
- ภาพรวมรวมสถานะการติดตั้ง/รันไทม์ของบริการโฮสต์ Gateway + node เมื่อมี
- ภาพรวมรวมช่องทางอัปเดต + git SHA (สำหรับการเช็กเอาต์จากซอร์ส)
- ข้อมูลอัปเดตแสดงในภาพรวม หากมีอัปเดตพร้อมใช้งาน สถานะจะพิมพ์คำแนะนำให้เรียกใช้ `openclaw update` (ดู [การอัปเดต](/th/install/updating))
- พื้นผิวสถานะแบบอ่านอย่างเดียว (`status`, `status --json`, `status --all`) จะ resolve SecretRefs ที่รองรับสำหรับเส้นทางการกำหนดค่าเป้าหมายเมื่อเป็นไปได้
- หากกำหนดค่า SecretRef ของช่องทางที่รองรับไว้แต่ไม่พร้อมใช้งานในเส้นทางคำสั่งปัจจุบัน สถานะจะยังคงเป็นอ่านอย่างเดียวและรายงานเอาต์พุตที่ลดระดับแทนการแครช เอาต์พุตสำหรับมนุษย์แสดงคำเตือน เช่น "โทเค็นที่กำหนดค่าไว้ไม่พร้อมใช้งานในเส้นทางคำสั่งนี้" และเอาต์พุต JSON รวม `secretDiagnostics`
- เมื่อการ resolve SecretRef เฉพาะคำสั่งสำเร็จ สถานะจะเลือกใช้สแนปช็อตที่ resolve แล้ว และล้างเครื่องหมายช่องทางชั่วคราว "ความลับไม่พร้อมใช้งาน" ออกจากเอาต์พุตสุดท้าย
- `status --all` รวมแถวภาพรวม Secrets และส่วนการวินิจฉัยที่สรุปการวินิจฉัยความลับ (ตัดให้สั้นเพื่อให้อ่านง่าย) โดยไม่หยุดการสร้างรายงาน
