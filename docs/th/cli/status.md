การวินิจฉัยสำหรับช่องทางและเซสชัน

```bash
openclaw status
openclaw status --all
openclaw status --deep
openclaw status --usage
```

| แฟล็ก                  | คำอธิบาย                                                                                                              |
| ----------------------- | --------------------------------------------------------------------------------------------------------------------- |
| `--all`                 | การวินิจฉัยแบบเต็ม (อ่านอย่างเดียวและสามารถวางเพื่อแชร์ได้) รวมการตรวจสอบความปลอดภัย ความเข้ากันได้ของ Plugin และการตรวจสอบเวกเตอร์หน่วยความจำ |
| `--deep`                | เรียกใช้การตรวจสอบแบบสด (WhatsApp Web + Telegram + Discord + Slack + Signal) และเปิดใช้การตรวจสอบความปลอดภัยด้วย              |
| `--usage`               | แสดงช่วงการใช้งานของผู้ให้บริการในรูปแบบมาตรฐานเป็น `เหลือ X%`                                                            |
| `--json`                | เอาต์พุตที่เครื่องอ่านได้                                                                                              |
| `--verbose` / `--debug` | แสดงการระบุเป้าหมาย Gateway แบบดิบก่อนรายงานด้วย                                                                         |

`openclaw status` แบบปกติจะใช้เส้นทางที่รวดเร็วและอ่านอย่างเดียวต่อไป และระบุหน่วยความจำเป็น
`ไม่ได้ตรวจสอบ` แทนที่จะระบุว่าใช้งานไม่ได้เมื่อข้ามการตรวจสอบหน่วยความจำ ส่วนการตรวจสอบ
ความปลอดภัยแบบหนัก การตรวจสอบความเข้ากันได้ของ Plugin และการตรวจสอบเวกเตอร์หน่วยความจำ
จะดำเนินการโดย `openclaw status --all`, `openclaw status --deep`, `openclaw security audit`
และ `openclaw memory status --deep`

## การระบุเซสชันและโมเดล

- เอาต์พุตสถานะเซสชันแยก `การดำเนินการ:` ออกจาก `รันไทม์:` โดย `การดำเนินการ`
  คือเส้นทางแซนด์บ็อกซ์ (`direct`, `docker/*`) ส่วน `รันไทม์` ระบุว่าเซสชัน
  กำลังใช้ `ค่าเริ่มต้นของ OpenClaw`, `OpenAI Codex`, แบ็กเอนด์ CLI
  หรือแบ็กเอนด์ ACP เช่น `codex (acp/acpx)` โปรดดู
  [รันไทม์ของเอเจนต์](/th/concepts/agent-runtimes) สำหรับความแตกต่างระหว่างผู้ให้บริการ โมเดล และรันไทม์
- เมื่อสแนปช็อตของเซสชันปัจจุบันมีข้อมูลไม่ครบ `/status` สามารถเติมตัวนับโทเค็น
  และแคชย้อนหลังจากบันทึกการใช้งานของทรานสคริปต์ล่าสุดได้ โดยค่าปัจจุบันที่ไม่เป็นศูนย์
  จะยังคงมีลำดับความสำคัญเหนือค่าทดแทนจากทรานสคริปต์
- การใช้ทรานสคริปต์เป็นข้อมูลทดแทนยังสามารถกู้คืนป้ายกำกับโมเดลรันไทม์ที่ใช้งานอยู่
  เมื่อรายการเซสชันแบบสดไม่มีข้อมูลดังกล่าว หากโมเดลในทรานสคริปต์นั้นแตกต่าง
  จากโมเดลที่เลือก สถานะจะระบุขนาดหน้าต่างบริบทตามโมเดลรันไทม์
  ที่กู้คืนมาแทนโมเดลที่เลือก
- สำหรับการคำนวณขนาดพรอมต์ การใช้ทรานสคริปต์เป็นข้อมูลทดแทนจะเลือกยอดรวม
  ที่เน้นพรอมต์ซึ่งมีค่ามากกว่า เมื่อข้อมูลเมตาของเซสชันไม่มีหรือมีค่าน้อยกว่า เพื่อไม่ให้
  เซสชันของผู้ให้บริการแบบกำหนดเองยุบเหลือการแสดงผล `0` โทเค็น
- เมื่อเซสชันถูกตรึงไว้กับโมเดลที่แตกต่างจากโมเดลหลักที่กำหนดค่าไว้ สถานะจะแสดง
  ทั้งสองค่า เหตุผล (`การแทนที่ของเซสชัน`) และคำแนะนำ `/model default`
  โมเดลหลักที่กำหนดค่าไว้จะใช้กับเซสชันใหม่หรือเซสชันที่ไม่ได้ตรึง ส่วนเซสชัน
  ที่ตรึงไว้แล้วจะคงการเลือกของเซสชันไว้จนกว่าจะล้าง
- เอาต์พุตรวมพื้นที่จัดเก็บเซสชันของแต่ละเอเจนต์เมื่อกำหนดค่าเอเจนต์หลายตัว

## การใช้งานและโควตา

- `--usage` แสดงช่วงการใช้งานของผู้ให้บริการในรูปแบบมาตรฐานเป็น `เหลือ X%`
- ฟิลด์ `usage_percent` / `usagePercent` แบบดิบของ MiniMax คือโควตาที่เหลือ
  ดังนั้น OpenClaw จึงกลับค่าก่อนแสดงผล โดยฟิลด์ที่อิงจำนวนจะมีลำดับความสำคัญ
  เมื่อมีอยู่ การตอบกลับ `model_remains` จะเลือกใช้รายการโมเดลแชท ระบุ
  ป้ายกำกับช่วงเวลาจากการประทับเวลาเมื่อจำเป็น และรวมชื่อโมเดลไว้ใน
  ป้ายกำกับแผน
- ความล้มเหลวในการรีเฟรชราคาโมเดลจะแสดงเป็นคำเตือนด้านราคาแบบไม่บังคับ
  ซึ่งไม่ได้หมายความว่า Gateway หรือช่องทางมีปัญหา

## ภาพรวมและสถานะการอัปเดต

- ภาพรวมรวมสถานะการติดตั้งและรันไทม์ของบริการโฮสต์ Gateway + Node เมื่อมีข้อมูล
  พร้อมระยะเวลาทำงานของกระบวนการ Gateway แบบย่อและระยะเวลาทำงานของระบบโฮสต์
- ภาพรวมรวมช่องทางการอัปเดต + git SHA (สำหรับเช็กเอาต์จากซอร์ส)
- ข้อมูลการอัปเดตจะแสดงในภาพรวม หากมีการอัปเดต สถานะจะแสดงคำแนะนำ
  ให้เรียกใช้ `openclaw update` (ดู [การอัปเดต](/th/install/updating))

## ข้อมูลลับ

- พื้นผิวสถานะแบบอ่านอย่างเดียว (`status`, `status --json`, `status --all`)
  จะระบุ SecretRef ที่รองรับสำหรับเส้นทางการกำหนดค่าเป้าหมายเมื่อทำได้
- หากกำหนดค่า SecretRef ของช่องทางที่รองรับไว้ แต่ไม่พร้อมใช้งานในเส้นทาง
  คำสั่งปัจจุบัน สถานะจะยังคงเป็นแบบอ่านอย่างเดียวและรายงานเอาต์พุตที่ลดทอน
  แทนที่จะขัดข้อง เอาต์พุตสำหรับมนุษย์จะแสดงคำเตือน เช่น "โทเค็นที่กำหนดค่าไว้
  ไม่พร้อมใช้งานในเส้นทางคำสั่งนี้" และเอาต์พุต JSON จะมี
  `secretDiagnostics`
- เมื่อการระบุ SecretRef เฉพาะคำสั่งสำเร็จ สถานะจะเลือกใช้สแนปช็อต
  ที่ระบุได้ และล้างเครื่องหมายช่องทางชั่วคราวที่ระบุว่า "ข้อมูลลับไม่พร้อมใช้งาน"
  ออกจากเอาต์พุตสุดท้าย
- `status --all` รวมแถวภาพรวมข้อมูลลับและส่วนการวินิจฉัย
  ที่สรุปการวินิจฉัยข้อมูลลับ (ตัดทอนเพื่อให้อ่านง่าย) โดยไม่หยุด
  การสร้างรายงาน

## หน่วยความจำ

`status --json --all` รายงานรายละเอียดหน่วยความจำจากรันไทม์ Plugin หน่วยความจำ
ที่ใช้งานอยู่ซึ่งเลือกโดย `plugins.slots.memory` Plugin หน่วยความจำแบบกำหนดเองสามารถ
ปิดใช้ `agents.defaults.memorySearch.enabled` ในตัวไว้ และยังคงรายงานไฟล์
ส่วนข้อมูล เวกเตอร์ และสถานะ FTS ของตนเองได้
