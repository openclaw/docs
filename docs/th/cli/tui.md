# `openclaw tui`

เปิด UI เทอร์มินัลที่เชื่อมต่อกับ Gateway หรือเรียกใช้ในโหมดฝังตัวภายในเครื่อง

คู่มือที่เกี่ยวข้อง: [TUI](/th/web/tui)

## ตัวเลือก

| แฟล็ก                        | ค่าเริ่มต้น                                | คำอธิบาย                                                                                       |
| ---------------------------- | ----------------------------------------- | ----------------------------------------------------------------------------------------------- |
| `--local`                    | `false`                                   | เรียกใช้กับรันไทม์เอเจนต์แบบฝังตัวภายในเครื่องแทน Gateway                                     |
| `--url <url>`                | `gateway.remote.url` จากการกำหนดค่า       | URL WebSocket ของ Gateway                                                                       |
| `--token <token>`            | (ไม่มี)                                   | โทเค็น Gateway หากจำเป็น                                                                        |
| `--password <pass>`          | (ไม่มี)                                   | รหัสผ่าน Gateway หากจำเป็น                                                                      |
| `--tls-fingerprint <sha256>` | `gateway.remote.tlsFingerprint`           | ลายนิ้วมือใบรับรอง TLS ที่คาดไว้สำหรับ Gateway แบบ `wss://` ที่ปักหมุดไว้                       |
| `--session <key>`            | `main` (หรือ `global` เมื่อขอบเขตเป็นแบบส่วนกลาง) | คีย์เซสชัน เมื่ออยู่ภายในพื้นที่ทำงานของเอเจนต์ ระบบจะเลือกเอเจนต์นั้นโดยอัตโนมัติ เว้นแต่จะระบุคำนำหน้า |
| `--deliver`                  | `false`                                   | ส่งการตอบกลับของผู้ช่วยผ่านช่องทางที่กำหนดค่าไว้                                                |
| `--thinking <level>`         | (ค่าเริ่มต้นของโมเดล)                     | เขียนทับระดับการคิด                                                                             |
| `--message <text>`           | (ไม่มี)                                   | ส่งข้อความเริ่มต้นหลังจากเชื่อมต่อ                                                              |
| `--timeout-ms <ms>`          | `agents.defaults.timeoutSeconds`          | ระยะหมดเวลาของเอเจนต์ ค่าที่ไม่ถูกต้องจะแสดงคำเตือนในบันทึกและถูกละเว้น                        |
| `--history-limit <n>`        | `200`                                     | จำนวนรายการประวัติที่จะโหลดเมื่อเชื่อมต่อ                                                       |

นามแฝง: `openclaw chat` และ `openclaw terminal` เรียกใช้คำสั่งนี้โดยถือว่าระบุ
`--local`

## หมายเหตุ

- ไม่สามารถใช้ `--local` ร่วมกับ `--url`, `--token`, `--password` หรือ `--tls-fingerprint` ได้
- `tui` จะแก้ค่า SecretRef สำหรับการยืนยันตัวตนของ Gateway ที่กำหนดค่าไว้ เพื่อใช้ยืนยันตัวตนด้วยโทเค็น/รหัสผ่าน
  เมื่อทำได้ (ผู้ให้บริการ `env`/`file`/`exec`)
- เมื่อไม่ได้ระบุ URL หรือพอร์ตอย่างชัดเจน `tui` จะใช้พอร์ต Gateway ภายในเครื่องที่ใช้งานอยู่
  ซึ่งบันทึกโดย Gateway ที่กำลังทำงานอยู่ โดย `--url`, `OPENCLAW_GATEWAY_URL`,
  `OPENCLAW_GATEWAY_PORT` และการกำหนดค่า Gateway ระยะไกลที่ระบุอย่างชัดเจนยังคงมีลำดับความสำคัญสูงกว่า
- เมื่อเรียกใช้จากภายในไดเรกทอรีพื้นที่ทำงานของเอเจนต์ที่กำหนดค่าไว้ TUI จะเลือก
  เอเจนต์นั้นเป็นค่าเริ่มต้นของคีย์เซสชันโดยอัตโนมัติ (เว้นแต่จะระบุ `--session`
  เป็น `agent:<id>:...` อย่างชัดเจน)
- หากต้องการแสดงชื่อโฮสต์ Gateway ในส่วนท้ายสำหรับการเชื่อมต่อที่ใช้ URL
  และไม่ใช่ภายในเครื่อง ให้เรียกใช้ `openclaw config set tui.footer.showRemoteHost true` ค่านี้ปิดอยู่
  โดยค่าเริ่มต้น และจะไม่แสดงสำหรับการเชื่อมต่อ local loopback หรือการเชื่อมต่อแบบฝังตัวภายในเครื่อง
- โหมดภายในเครื่องใช้รันไทม์เอเจนต์แบบฝังตัวโดยตรง เครื่องมือภายในเครื่องส่วนใหญ่ใช้งานได้
  แต่ฟีเจอร์ที่ใช้ได้เฉพาะกับ Gateway จะไม่พร้อมใช้งาน
- โหมดภายในเครื่องเพิ่ม `/auth [provider]` ลงในชุดคำสั่งของ TUI
- เกตการอนุมัติของ Plugin ยังคงมีผลในโหมดภายในเครื่อง: เครื่องมือที่ต้องได้รับการอนุมัติ
  จะแจ้งให้ตัดสินใจในเทอร์มินัล โดยไม่มีสิ่งใดได้รับการอนุมัติโดยอัตโนมัติอย่างเงียบ ๆ
- [เป้าหมาย](/th/tools/goal) ของเซสชันจะแสดงในส่วนท้ายและสามารถจัดการได้ด้วย
  `/goal`

## ตัวอย่าง

```bash
openclaw chat
openclaw tui --local
openclaw tui
openclaw tui --url ws://127.0.0.1:18789 --token <token>
openclaw tui --session main --deliver
openclaw chat --message "เปรียบเทียบการกำหนดค่าของฉันกับเอกสารและบอกฉันว่าต้องแก้ไขอะไร"
# เมื่อเรียกใช้ภายในพื้นที่ทำงานของเอเจนต์ ระบบจะอนุมานเอเจนต์นั้นโดยอัตโนมัติ
openclaw tui --session bugfix
```

## วงรอบการซ่อมแซมการกำหนดค่า

ใช้โหมดภายในเครื่องเพื่อให้เอเจนต์แบบฝังตัวตรวจสอบการกำหนดค่าปัจจุบัน เปรียบเทียบ
กับเอกสาร และช่วยซ่อมแซมจากเทอร์มินัลเดียวกัน

หาก `openclaw config validate` ล้มเหลวอยู่แล้ว ให้เรียกใช้ `openclaw configure` หรือ
`openclaw doctor --fix` ก่อน เนื่องจาก `openclaw chat` จะไม่ข้ามตัวป้องกัน
การกำหนดค่าที่ไม่ถูกต้อง

```bash
openclaw chat
```

จากนั้นภายใน TUI:

```text
!openclaw config file
!openclaw docs gateway auth token secretref
!openclaw config validate
!openclaw doctor
```

ใช้การแก้ไขแบบเจาะจงด้วย `openclaw config set` หรือ `openclaw configure` จากนั้น
เรียกใช้ `openclaw config validate` อีกครั้ง ดู [TUI](/th/web/tui) และ
[การกำหนดค่า](/th/cli/config)
