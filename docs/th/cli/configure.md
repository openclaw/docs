# `openclaw configure`

พรอมต์แบบโต้ตอบสำหรับตั้งค่าข้อมูลรับรอง อุปกรณ์ และค่าเริ่มต้นของเอเจนต์

<Note>
ส่วน **โมเดล** มีการเลือกหลายรายการสำหรับรายการอนุญาต `agents.defaults.models` (สิ่งที่แสดงใน `/model` และตัวเลือกโมเดล) ตัวเลือกการตั้งค่าที่อยู่ในขอบเขตของผู้ให้บริการจะรวมโมเดลที่เลือกเข้ากับรายการอนุญาตที่มีอยู่ แทนที่จะแทนที่ผู้ให้บริการที่ไม่เกี่ยวข้องซึ่งมีอยู่แล้วในการกำหนดค่า การเรียกใช้การยืนยันตัวตนของผู้ให้บริการจาก configure อีกครั้งจะคงค่า `agents.defaults.model.primary` ที่มีอยู่ไว้ ใช้ `openclaw models auth login --provider <id> --set-default` หรือ `openclaw models set <model>` เมื่อคุณตั้งใจต้องการเปลี่ยนโมเดลเริ่มต้น
</Note>

เมื่อ configure เริ่มจากตัวเลือกการยืนยันตัวตนของผู้ให้บริการ ตัวเลือกโมเดลเริ่มต้นและรายการอนุญาตจะเลือกผู้ให้บริการนั้นโดยอัตโนมัติ สำหรับผู้ให้บริการที่จับคู่กัน เช่น Volcengine และ BytePlus การตั้งค่าเดียวกันนี้จะจับคู่กับตัวแปรแผนการเขียนโค้ดของผู้ให้บริการเหล่านั้นด้วย (`volcengine-plan/*`, `byteplus-plan/*`) หากตัวกรองผู้ให้บริการที่ต้องการทำให้ได้รายการว่าง configure จะย้อนกลับไปใช้แค็ตตาล็อกที่ไม่ถูกกรองแทนการแสดงตัวเลือกว่าง

<Tip>
`openclaw config` ที่ไม่มีคำสั่งย่อยจะเปิดวิซาร์ดเดียวกัน ใช้ `openclaw config get|set|unset` สำหรับการแก้ไขแบบไม่โต้ตอบ
</Tip>

สำหรับการค้นหาเว็บ `openclaw configure --section web` ให้คุณเลือกผู้ให้บริการ
และกำหนดค่าข้อมูลรับรอง ผู้ให้บริการบางรายยังแสดงพรอมต์ติดตามผล
เฉพาะผู้ให้บริการด้วย:

- **Grok** สามารถเสนอการตั้งค่า `x_search` ที่เป็นทางเลือกพร้อม `XAI_API_KEY` เดียวกัน และ
  ให้คุณเลือกโมเดล `x_search`
- **Kimi** สามารถถามภูมิภาค Moonshot API (`api.moonshot.ai` เทียบกับ
  `api.moonshot.cn`) และโมเดลค้นหาเว็บ Kimi เริ่มต้น

ที่เกี่ยวข้อง:

- ข้อมูลอ้างอิงการกำหนดค่า Gateway: [การกำหนดค่า](/th/gateway/configuration)
- CLI การกำหนดค่า: [การกำหนดค่า](/th/cli/config)

## ตัวเลือก

- `--section <section>`: ตัวกรองส่วนที่ใช้ซ้ำได้

ส่วนที่มีให้ใช้งาน:

- `workspace`
- `model`
- `web`
- `gateway`
- `daemon`
- `channels`
- `plugins`
- `skills`
- `health`

หมายเหตุ:

- การเลือกตำแหน่งที่ Gateway ทำงานจะอัปเดต `gateway.mode` เสมอ คุณสามารถเลือก "ดำเนินการต่อ" โดยไม่ต้องมีส่วนอื่นได้ หากนั่นคือทั้งหมดที่คุณต้องการ
- บริการที่เน้นช่องทาง (Slack/Discord/Matrix/Microsoft Teams) จะขอรายการอนุญาตช่องทาง/ห้องระหว่างการตั้งค่า คุณสามารถป้อนชื่อหรือ ID ได้ วิซาร์ดจะแปลงชื่อเป็น ID เมื่อทำได้
- หากคุณเรียกใช้ขั้นตอนติดตั้ง daemon การยืนยันตัวตนด้วยโทเค็นต้องใช้โทเค็น และ `gateway.auth.token` อยู่ภายใต้การจัดการของ SecretRef configure จะตรวจสอบ SecretRef แต่จะไม่บันทึกค่าโทเค็นข้อความธรรมดาที่แก้ไขแล้วลงในข้อมูลเมตาสภาพแวดล้อมของบริการ supervisor
- หากการยืนยันตัวตนด้วยโทเค็นต้องใช้โทเค็นและ SecretRef ของโทเค็นที่กำหนดค่าไว้ยังแก้ไขไม่ได้ configure จะบล็อกการติดตั้ง daemon พร้อมคำแนะนำการแก้ไขที่ดำเนินการได้
- หากกำหนดค่าทั้ง `gateway.auth.token` และ `gateway.auth.password` แล้ว และไม่ได้ตั้งค่า `gateway.auth.mode` configure จะบล็อกการติดตั้ง daemon จนกว่าจะตั้งค่าโหมดอย่างชัดเจน

## ตัวอย่าง

```bash
openclaw configure
openclaw configure --section web
openclaw configure --section model --section channels
openclaw configure --section gateway --section daemon
```
