# `openclaw configure`

พรอมป์แบบโต้ตอบสำหรับการเปลี่ยนแปลงแบบเจาะจงกับการตั้งค่าที่มีอยู่: ข้อมูลรับรอง, อุปกรณ์, ค่าเริ่มต้นของเอเจนต์, Gateway, ช่องทาง, Plugin, Skills และการตรวจสอบสุขภาพ

ใช้ `openclaw onboard` สำหรับเส้นทางการใช้งานครั้งแรกแบบมีคำแนะนำเต็มรูปแบบ, `openclaw setup` สำหรับ config/workspace พื้นฐานเท่านั้น และ `openclaw channels add` เมื่อคุณต้องตั้งค่าบัญชีช่องทางเท่านั้น

<Note>
ส่วน **โมเดล** มีตัวเลือกหลายรายการสำหรับรายการอนุญาต `agents.defaults.models` (สิ่งที่แสดงใน `/model` และตัวเลือกโมเดล) ตัวเลือกการตั้งค่าที่จำกัดขอบเขตตามผู้ให้บริการจะผสานโมเดลที่เลือกเข้าในรายการอนุญาตที่มีอยู่ แทนที่จะแทนที่ผู้ให้บริการอื่นที่ไม่เกี่ยวข้องซึ่งมีอยู่แล้วใน config

การเรียกใช้การยืนยันตัวตนของผู้ให้บริการจาก configure อีกครั้งจะรักษา `agents.defaults.model.primary` ที่มีอยู่ไว้ แม้ว่าขั้นตอนการยืนยันตัวตนของผู้ให้บริการจะคืนค่าแพตช์ config พร้อมโมเดลเริ่มต้นที่แนะนำของตัวเองก็ตาม ซึ่งหมายความว่าการเพิ่มหรือยืนยันตัวตน xAI, OpenRouter หรือผู้ให้บริการอื่นอีกครั้งควรทำให้โมเดลใหม่พร้อมใช้งาน โดยไม่เข้ามาแทนที่โมเดลหลักปัจจุบันของคุณ ใช้ `openclaw models auth login --provider <id> --set-default` หรือ `openclaw models set <model>` เมื่อคุณตั้งใจต้องการเปลี่ยนโมเดลเริ่มต้น
</Note>

เมื่อ configure เริ่มจากตัวเลือกการยืนยันตัวตนของผู้ให้บริการ ตัวเลือกโมเดลเริ่มต้นและรายการอนุญาตจะให้ความสำคัญกับผู้ให้บริการนั้นโดยอัตโนมัติ สำหรับผู้ให้บริการแบบจับคู่ เช่น Volcengine และ BytePlus การตั้งค่านี้จะจับคู่กับตัวแปร coding-plan ของผู้ให้บริการเหล่านั้นด้วย (`volcengine-plan/*`, `byteplus-plan/*`) หากตัวกรองผู้ให้บริการที่ต้องการทำให้ได้รายการว่าง configure จะย้อนกลับไปใช้แค็ตตาล็อกแบบไม่กรองแทนการแสดงตัวเลือกว่าง

<Tip>
`openclaw config` โดยไม่มีคำสั่งย่อยจะเปิดตัวช่วยสร้างเดียวกัน ใช้ `openclaw config get|set|unset` สำหรับการแก้ไขแบบไม่โต้ตอบ
</Tip>

สำหรับการค้นหาเว็บ `openclaw configure --section web` ให้คุณเลือกผู้ให้บริการ
และกำหนดค่าข้อมูลรับรองของผู้ให้บริการนั้น ผู้ให้บริการบางรายยังแสดง
พรอมป์ติดตามผลเฉพาะผู้ให้บริการด้วย:

- **Grok** สามารถเสนอการตั้งค่า `x_search` แบบไม่บังคับด้วยโปรไฟล์ xAI OAuth
  หรือคีย์ API เดียวกัน และให้คุณเลือกโมเดล `x_search`
- **Kimi** สามารถถามภูมิภาค Moonshot API (`api.moonshot.ai` เทียบกับ
  `api.moonshot.cn`) และโมเดลค้นหาเว็บ Kimi เริ่มต้น

## ตัวเลือก

- `--section <section>`: ตัวกรองส่วนที่ระบุซ้ำได้

ส่วนที่พร้อมใช้งาน:

- `workspace`
- `model`
- `web`
- `gateway`
- `daemon`
- `channels`
- `plugins`
- `skills`
- `health`

หมายเหตุ:

- ตัวช่วยสร้างเต็มรูปแบบและส่วนที่เกี่ยวข้องกับ Gateway จะถามว่า Gateway ทำงานที่ไหนและอัปเดต `gateway.mode` ตัวกรองส่วนที่ไม่มี `gateway`, `daemon` หรือ `health` จะไปยังการตั้งค่าที่ร้องขอโดยตรง
- หลังจากเขียน config ในเครื่องแล้ว configure จะติดตั้ง Plugin ที่ดาวน์โหลดได้ที่เลือกไว้เมื่อเส้นทางการตั้งค่าที่เลือกต้องใช้ Plugin เหล่านั้น config ของ Gateway ระยะไกลจะไม่ติดตั้งแพ็กเกจ Plugin ในเครื่อง
- บริการที่เน้นช่องทาง (Slack/Discord/Matrix/Microsoft Teams) จะแจ้งให้ระบุรายการอนุญาตของช่องทาง/ห้องระหว่างการตั้งค่า คุณสามารถป้อนชื่อหรือ ID ได้ ตัวช่วยสร้างจะแปลงชื่อเป็น ID เมื่อทำได้
- หากคุณเรียกใช้ขั้นตอนติดตั้ง daemon การยืนยันตัวตนด้วยโทเค็นต้องมีโทเค็น และ `gateway.auth.token` ถูกจัดการด้วย SecretRef configure จะตรวจสอบ SecretRef แต่จะไม่บันทึกค่าโทเค็นแบบข้อความธรรมดาที่แก้ไขแล้วลงในข้อมูลเมทาดาทาสภาพแวดล้อมของบริการ supervisor
- หากการยืนยันตัวตนด้วยโทเค็นต้องมีโทเค็นและ SecretRef ของโทเค็นที่กำหนดค่าไว้ยังแก้ไขไม่ได้ configure จะบล็อกการติดตั้ง daemon พร้อมคำแนะนำการแก้ไขที่นำไปปฏิบัติได้
- หากมีการกำหนดค่าทั้ง `gateway.auth.token` และ `gateway.auth.password` และไม่ได้ตั้งค่า `gateway.auth.mode` configure จะบล็อกการติดตั้ง daemon จนกว่าจะตั้งค่าโหมดอย่างชัดเจน

## ตัวอย่าง

```bash
openclaw configure
openclaw configure --section web
openclaw configure --section model --section channels
openclaw configure --section gateway --section daemon
```
