# `openclaw configure`

พรอมต์แบบโต้ตอบสำหรับการเปลี่ยนแปลงเฉพาะจุดในการตั้งค่าที่มีอยู่ ได้แก่ ข้อมูลประจำตัว อุปกรณ์ ค่าเริ่มต้นของเอเจนต์ Gateway ช่องทาง Plugin Skills และการตรวจสอบสถานะ

ใช้ `openclaw onboard` หรือ `openclaw setup` สำหรับกระบวนการเริ่มใช้งานครั้งแรกแบบมีคำแนะนำอย่างครบถ้วน ใช้ `openclaw setup --baseline` เมื่อต้องการเฉพาะการกำหนดค่า/พื้นที่ทำงานพื้นฐาน และใช้ `openclaw channels add` เมื่อต้องการตั้งค่าบัญชีช่องทางเท่านั้น

<Tip>
`openclaw config` ที่ไม่มีคำสั่งย่อยจะเปิดตัวช่วยตั้งค่าเดียวกัน ใช้ `openclaw config get|set|unset` สำหรับการแก้ไขแบบไม่โต้ตอบ
</Tip>

## ตัวเลือก

`--section <section>`: ตัวกรองส่วนที่ระบุซ้ำได้ ส่วนที่ใช้ได้มีดังนี้:

`workspace`, `model`, `web`, `gateway`, `daemon`, `channels`, `plugins`, `skills`, `health`

```bash
openclaw configure
openclaw configure --section web
openclaw configure --section model --section channels
openclaw configure --section gateway --section daemon
```

การเลือก `gateway`, `daemon` หรือ `health` (หรือเรียกใช้ตัวช่วยตั้งค่าแบบเต็มโดยไม่มี `--section`) จะแสดงพรอมต์ถามว่า Gateway ทำงานที่ใดและอัปเดต `gateway.mode` ตัวกรองส่วนที่ข้ามทั้งสามส่วนจะไปยังการตั้งค่าที่ร้องขอโดยตรงโดยไม่มีพรอมต์โหมด Gateway การเลือกโหมด Gateway ระยะไกลจะเขียนการกำหนดค่าระยะไกลและออกทันที โดยจะไม่ดำเนินขั้นตอนที่ทำได้เฉพาะภายในเครื่อง เช่น การติดตั้ง Plugin

<Note>
`openclaw configure` ต้องใช้เทอร์มินัลแบบโต้ตอบ (ทั้ง stdin และ stdout ต้องเป็น TTY) หากไม่มี ระบบจะแสดงคำสั่ง `openclaw config get|set|patch|validate` แบบไม่โต้ตอบที่เทียบเท่ากัน และออกพร้อมข้อผิดพลาดแทนการทำงานเพียงบางส่วน
</Note>

## ส่วนโมเดล

<Note>
**โมเดล** มีตัวเลือกแบบหลายรายการสำหรับรายการอนุญาต `agents.defaults.models` (รายการที่แสดงใน `/model` และเครื่องมือเลือกโมเดล) ตัวเลือกการตั้งค่าที่จำกัดขอบเขตตามผู้ให้บริการจะผสานโมเดลที่เลือกเข้ากับรายการอนุญาตเดิม แทนที่จะแทนที่ผู้ให้บริการอื่นที่มีอยู่ในการกำหนดค่า

การเรียกใช้การยืนยันตัวตนของผู้ให้บริการอีกครั้งจาก configure จะรักษาค่า `agents.defaults.model.primary` ที่มีอยู่ แม้ว่าขั้นตอนการยืนยันตัวตนของผู้ให้บริการจะส่งคืนแพตช์การกำหนดค่าที่มีโมเดลเริ่มต้นซึ่งผู้ให้บริการแนะนำก็ตาม การเพิ่มหรือยืนยันตัวตนของผู้ให้บริการอีกครั้งจะทำให้โมเดลของผู้ให้บริการนั้นพร้อมใช้งานโดยไม่เข้ามาแทนที่โมเดลหลักปัจจุบันของคุณ ใช้ `openclaw models auth login --provider <id> --set-default` หรือ `openclaw models set <model>` เมื่อต้องการเปลี่ยนโมเดลเริ่มต้นโดยเจตนา
</Note>

เมื่อ configure เริ่มต้นจากตัวเลือกการยืนยันตัวตนของผู้ให้บริการ เครื่องมือเลือกโมเดลเริ่มต้นและรายการอนุญาตจะเลือกผู้ให้บริการนั้นเป็นค่าอ้างอิงโดยอัตโนมัติ สำหรับผู้ให้บริการที่จับคู่กัน เช่น Volcengine และ BytePlus การเลือกนี้จะตรงกับตัวแปรแผนการเขียนโค้ดของผู้ให้บริการเหล่านั้นด้วย (`volcengine-plan/*`, `byteplus-plan/*`) หากตัวกรองผู้ให้บริการที่ต้องการทำให้รายการว่าง configure จะย้อนกลับไปใช้แค็ตตาล็อกที่ไม่ผ่านการกรองแทนการแสดงเครื่องมือเลือกที่ว่างเปล่า

## ส่วนเว็บ

`openclaw configure --section web` จะเลือกผู้ให้บริการค้นหาเว็บและกำหนดค่าข้อมูลประจำตัว ผู้ให้บริการบางรายจะแสดงขั้นตอนติดตามผลเฉพาะผู้ให้บริการ:

- **Grok** สามารถเสนอตัวเลือกการตั้งค่า `x_search` ด้วยโปรไฟล์ xAI OAuth หรือคีย์ API เดียวกัน และให้คุณเลือกโมเดล `x_search`
- **Kimi** สามารถถามภูมิภาคของ Moonshot API (`api.moonshot.ai` เทียบกับ `api.moonshot.cn`) และโมเดลค้นหาเว็บ Kimi เริ่มต้น

## หมายเหตุอื่น ๆ

- หลังจากเขียนการกำหนดค่าภายในเครื่องแล้ว configure จะติดตั้ง Plugin ที่ดาวน์โหลดได้ซึ่งเลือกไว้ เมื่อเส้นทางการตั้งค่าที่เลือกจำเป็นต้องใช้ การกำหนดค่า Gateway ระยะไกลจะไม่ติดตั้งแพ็กเกจ Plugin ภายในเครื่อง
- บริการที่เน้นช่องทาง (Slack/Discord/Matrix/Microsoft Teams) จะแสดงพรอมต์ให้ระบุรายการอนุญาตของช่องทาง/ห้องระหว่างการตั้งค่า คุณสามารถป้อนชื่อหรือ ID ได้ และตัวช่วยตั้งค่าจะแปลงชื่อเป็น ID เมื่อทำได้
- หากคุณเรียกใช้ขั้นตอนการติดตั้ง daemon การยืนยันตัวตนด้วยโทเค็นจำเป็นต้องมีโทเค็น หาก `gateway.auth.token` จัดการโดย SecretRef configure จะตรวจสอบความถูกต้องของ SecretRef แต่จะไม่บันทึกค่าโทเค็นข้อความธรรมดาที่แก้ค่าแล้วลงในข้อมูลเมตาของสภาพแวดล้อมบริการ supervisor หากไม่สามารถแก้ค่า SecretRef ได้ configure จะระงับการติดตั้ง daemon พร้อมแสดงคำแนะนำการแก้ไขที่นำไปปฏิบัติได้
- หากกำหนดค่าทั้ง `gateway.auth.token` และ `gateway.auth.password` ไว้ แต่ยังไม่ได้ตั้งค่า `gateway.auth.mode` configure จะระงับการติดตั้ง daemon จนกว่าคุณจะตั้งค่าโหมดอย่างชัดเจน
