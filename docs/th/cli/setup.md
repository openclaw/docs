# `openclaw setup`

`openclaw setup` คือจุดเริ่มต้นของเอเจนต์ระบบ ในระบบที่กำหนดค่าแล้ว การเรียก
`openclaw setup` โดยไม่มีอาร์กิวเมนต์จะเปิดแชต OpenClaw แบบโต้ตอบ ส่วนในระบบใหม่
ระบบจะเข้าสู่การเริ่มต้นใช้งานแบบมีคำแนะนำ ใช้ `-m`/`--message` สำหรับคำขอหนึ่งรายการ หรือ
`--baseline` เพื่อเริ่มต้นโฟลเดอร์การกำหนดค่า/พื้นที่ทำงานโดยไม่ใช้วิซาร์ด

ลำดับการกำหนดเส้นทาง:

1. ตัวเลือกการเริ่มต้นใช้งานใดๆ (`--wizard`, `--baseline`, พื้นที่ทำงาน, รีเซ็ต,
   แบบไม่โต้ตอบ, โฟลว์, โหมด, Gateway, เดมอน, ข้าม, นำเข้า, ระยะไกล หรือตัวเลือกการยืนยันตัวตน)
   จะเรียกใช้การเริ่มต้นใช้งานเหมือนกับ `openclaw onboard` ทุกประการ
2. `-m`/`--message` หรือ `--yes` จะเรียกใช้เอเจนต์ระบบ
3. หากไม่มีตัวเลือกการกำหนดเส้นทาง ระบบแบบโต้ตอบที่กำหนดค่าแล้วจะเปิด OpenClaw
   ส่วนระบบใหม่จะเรียกใช้การเริ่มต้นใช้งาน ในระบบที่กำหนดค่าแล้ว `--json` จะแสดง
   ภาพรวมระบบแม้ไม่มี TTY ส่วนตัวเลือกการเริ่มต้นใช้งานจะคงข้อมูลสรุป JSON
   ของการเริ่มต้นใช้งานไว้

ในโหมดมีคำแนะนำ `--workspace <dir>` คือพื้นที่ทำงานที่เสนอต่อ OpenClaw
และจะบันทึกถาวรหลังจากอนุมัติข้อเสนอนั้นเท่านั้น การตั้งค่าแบบพื้นฐาน แบบคลาสสิก และ
แบบไม่โต้ตอบจะบันทึกพื้นที่ทำงานที่ระบุผ่านโฟลว์ปกติของแต่ละแบบ
ในการติดตั้งใหม่ เมื่อรายชื่อเอเจนต์ที่มีอยู่จะถูกแมปใหม่
วิซาร์ดแบบคลาสสิกจะขอการยืนยันอย่างชัดเจน ส่วนการตั้งค่าแบบไม่โต้ตอบจะคง
พื้นที่ทำงานของกลุ่มเอเจนต์ปัจจุบันไว้และแสดงคำเตือน

การตรวจหาการอนุมานแบบมีคำแนะนำทำงานบนโฮสต์ Gateway ใน macOS หรือ Linux โดย CLI
และแอป macOS จะเรียกใช้ตัวตรวจหาตัวเดียวกันซึ่งเป็นของ Gateway ตัวตรวจหานี้จะตรวจสอบ
โมเดลที่กำหนดค่าไว้ การเข้าสู่ระบบ CLI ที่รองรับ ตัวแปรสภาพแวดล้อมของคีย์ API และโมเดล
Ollama หรือ LM Studio ที่ติดตั้งไว้แล้ว ขั้นตอนอัตโนมัตินี้จะไม่ดาวน์โหลดโมเดลภายในเครื่อง
ระบบจะทดสอบรันไทม์ภายในเครื่องที่ตรวจพบโดยอัตโนมัติหลังจากตัวเลือก CLI และคีย์ API
เมื่อมีโมเดลภายในเครื่องหลายโมเดล OpenClaw จะเลือกตระกูลโมเดล instruct
ที่รองรับการเรียกใช้เครื่องมือได้ดีที่สุด ตัวเลือกที่เลือกต้องตอบคำขอ completion จริงได้
ก่อนที่จะบันทึกการกำหนดค่าผู้ให้บริการและโมเดล นอกจากนี้ ระบบจะรายงาน CLI ของ Gemini,
Antigravity, Pi และ OpenCode ที่ติดตั้งไว้ เมื่อไม่สามารถใช้เป็นเส้นทางการอนุมานแบบนำกลับมาใช้ใหม่
สำหรับการตั้งค่าแบบมีคำแนะนำได้

`setup` รองรับแฟล็กการเริ่มต้นใช้งานเดียวกับ `openclaw onboard` รวมถึง
การยืนยันตัวตน (`--auth-choice`, `--token`, แฟล็กคีย์ของผู้ให้บริการ), Gateway
(`--gateway-port`, `--gateway-bind`, `--gateway-auth`, `--install-daemon`),
Tailscale (`--tailscale`), การรีเซ็ต (`--reset`, `--reset-scope`), โฟลว์
(`--flow quickstart|advanced|manual|import`) และแฟล็กสำหรับข้าม
(`--skip-channels`, `--skip-skills`, `--skip-bootstrap`, `--skip-search`,
`--skip-health`, `--skip-ui`, `--skip-hooks`) ส่ง `--tui` เพื่อใช้
ช่องทางเทอร์มินัลเดียวกับ `openclaw onboard --tui` โปรดดู [การเริ่มต้นใช้งาน](/th/cli/onboard) และ
[การทำงานอัตโนมัติด้วย CLI](/th/start/wizard-cli-automation) สำหรับข้อมูลอ้างอิงแฟล็กทั้งหมดและ
ตัวอย่างแบบไม่โต้ตอบ `openclaw onboard --modern` ยังคงเป็นรายการเข้าที่รองรับความเข้ากันได้
สำหรับผู้ช่วย OpenClaw แบบเดียวกันซึ่งมีการตรวจสอบความพร้อมด้านการอนุมาน

<Note>
`openclaw setup` ใช้สำหรับการติดตั้งที่แก้ไขการกำหนดค่าได้ ในโหมด Nix (`OPENCLAW_NIX_MODE=1`) OpenClaw จะปฏิเสธการเขียนจากการตั้งค่า เนื่องจากไฟล์การกำหนดค่าจัดการโดย Nix ให้ใช้ [คู่มือเริ่มต้นฉบับย่อของ nix-openclaw](https://github.com/openclaw/nix-openclaw#quick-start) จากผู้พัฒนาโดยตรง หรือการกำหนดค่าต้นทางที่เทียบเท่าสำหรับแพ็กเกจ Nix อื่น
</Note>

## ตัวเลือก

| แฟล็ก                       | คำอธิบาย                                                                                          |
| -------------------------- | ---------------------------------------------------------------------------------------------------- |
| `-m, --message <text>`     | เรียกใช้คำขอ OpenClaw หนึ่งรายการ                                                                            |
| `--yes`                    | อนุมัติการเขียนการกำหนดค่าแบบถาวรสำหรับคำขอ `--message` หนึ่งรายการ                                        |
| `--workspace <dir>`        | ข้อเสนอพื้นที่ทำงาน กลุ่มเอเจนต์ที่มีอยู่ต้องยืนยันแบบคลาสสิก และจะถูกคงไว้ในโหมดไม่โต้ตอบ |
| `--baseline`               | สร้างโฟลเดอร์การกำหนดค่า/พื้นที่ทำงาน/เซสชันพื้นฐานโดยไม่เริ่มต้นใช้งาน                                 |
| `--wizard`                 | บังคับใช้การเริ่มต้นใช้งานแบบโต้ตอบ                                                                        |
| `--tui`                    | ใช้ช่องทางเทอร์มินัลแทนการส่งต่อไปยังเบราว์เซอร์                                               |
| `--non-interactive`        | เรียกใช้การเริ่มต้นใช้งานโดยไม่แสดงพรอมต์                                                                      |
| `--accept-risk`            | ยอมรับความเสี่ยงจากการให้เอเจนต์เข้าถึงทั้งระบบ ต้องใช้ร่วมกับ `--non-interactive`                        |
| `--mode <mode>`            | โหมดการเริ่มต้นใช้งาน: `local` หรือ `remote`                                                                |
| `--flow <flow>`            | โฟลว์การเริ่มต้นใช้งาน: `quickstart`, `advanced`, `manual` หรือ `import`                                       |
| `--reset`                  | รีเซ็ตการกำหนดค่า + ข้อมูลประจำตัว + เซสชันก่อนเริ่มต้นใช้งาน (รวมพื้นที่ทำงานเฉพาะเมื่อใช้ `--reset-scope full`)  |
| `--reset-scope <scope>`    | ขอบเขตการรีเซ็ต: `config`, `config+creds+sessions` หรือ `full`                                           |
| `--import-from <provider>` | ผู้ให้บริการการย้ายข้อมูลที่จะเรียกใช้ระหว่างการเริ่มต้นใช้งาน                                                         |
| `--import-source <path>`   | โฮมของเอเจนต์ต้นทางสำหรับ `--import-from`                                                               |
| `--import-secrets`         | นำเข้าข้อมูลลับที่รองรับระหว่างการย้ายข้อมูลในการเริ่มต้นใช้งาน                                                |
| `--remote-url <url>`       | URL WebSocket ของ Gateway ระยะไกล                                                                        |
| `--remote-token <token>`   | โทเค็น Gateway ระยะไกล (ไม่บังคับ)                                                                     |
| `--json`                   | ระบบที่กำหนดค่าแล้ว: ภาพรวม OpenClaw เส้นทางการเริ่มต้นใช้งาน: สรุปการเริ่มต้นใช้งาน                          |

`--classic` และ `--non-interactive` ใช้ร่วมกันไม่ได้ โดยแบบคลาสสิกจะเปิด
วิซาร์ดที่แสดงพรอมต์ ส่วนการตั้งค่าแบบไม่โต้ตอบจะใช้เส้นทางการทำงานอัตโนมัติ
ในการเริ่มต้นใช้งานแบบโต้ตอบ `--remote-url` และ `--remote-token` จะกรอกข้อมูลล่วงหน้าใน
ขั้นตอน Gateway ระยะไกล และมีลำดับความสำคัญเหนือค่าระยะไกลที่บันทึกไว้สำหรับการเรียกใช้ครั้งนั้น
การเปลี่ยน URL จะไม่นำข้อมูลประจำตัวที่บันทึกไว้กลับมาใช้ เว้นแต่จะส่งโทเค็นด้วย
โทเค็นจะยังคงถูกปกปิดและใช้โหมดจัดเก็บข้อความธรรมดาหรือ SecretRef
ที่เลือกไว้ในวิซาร์ด

### โหมดพื้นฐาน

`openclaw setup --baseline` คงลักษณะการทำงานแบบเก่าที่ใช้เฉพาะการตั้งค่าพื้นฐานไว้ โดยจะ
สร้างไดเรกทอรีการกำหนดค่า พื้นที่ทำงาน และเซสชัน จากนั้นออกโดยไม่
เรียกใช้การเริ่มต้นใช้งาน รองรับ `--workspace` และตัวควบคุมเอาต์พุตที่ไม่มีผลกระทบ แต่
จะปฏิเสธตัวเลือกการเริ่มต้นใช้งาน, Gateway, การยืนยันตัวตน, การรีเซ็ต หรือเดมอนที่ระบุอย่างชัดเจน
แทนที่จะเพิกเฉยโดยไม่แจ้ง หากการกำหนดค่าที่มีอยู่ไม่ถูกต้อง การตั้งค่าแบบพื้นฐานจะคง
การกำหนดค่านั้นไว้และขอให้เรียกใช้ `openclaw doctor` ก่อนลองอีกครั้ง

## ตัวอย่าง

```bash
openclaw setup
openclaw setup -m "status"
openclaw setup -m "restart gateway" --yes
openclaw setup --json
openclaw setup --wizard
openclaw setup --baseline
openclaw setup --workspace ~/.openclaw/workspace
openclaw setup --import-from hermes --import-source ~/.hermes
openclaw setup --non-interactive --accept-risk --mode remote --remote-url wss://gateway-host:18789 --remote-token <token>
```

## หมายเหตุ

- หลังการตั้งค่าแบบพื้นฐาน ให้เรียกใช้ `openclaw onboard` สำหรับกระบวนการแบบมีคำแนะนำทั้งหมด, `openclaw configure` สำหรับการเปลี่ยนแปลงเฉพาะจุด หรือ `openclaw channels add` เพื่อเพิ่มบัญชีช่องทาง
- หากตรวจพบสถานะของ Hermes การเริ่มต้นใช้งานแบบโต้ตอบสามารถเสนอการย้ายข้อมูลโดยอัตโนมัติ การเริ่มต้นใช้งานแบบนำเข้าต้องใช้การตั้งค่าใหม่ โปรดใช้ [การย้ายข้อมูล](/th/cli/migrate) สำหรับแผนการทดลองทำ การสำรองข้อมูล และโหมดเขียนทับภายนอกการเริ่มต้นใช้งาน
