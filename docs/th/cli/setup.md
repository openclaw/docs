# `openclaw setup`

`openclaw setup` เรียกใช้ขั้นตอนการเริ่มต้นใช้งานแบบมีคำแนะนำเดียวกับ `openclaw onboard`:
โดยจะตรวจสอบและบันทึกการอนุมานก่อน จากนั้นเริ่ม Crestodian เพื่อกำหนดค่า
พื้นที่ทำงาน, Gateway, ช่องทาง, Skills และสถานะระบบ ใช้ `--baseline` เมื่อคุณ
ต้องการเพียงเริ่มต้นโฟลเดอร์การกำหนดค่า/พื้นที่ทำงานโดยไม่ใช้ตัวช่วยสร้าง

ในโหมดแบบมีคำแนะนำ `--workspace <dir>` คือพื้นที่ทำงานที่เสนอให้ Crestodian;
ระบบจะบันทึกค่านี้หลังจากคุณอนุมัติข้อเสนอดังกล่าวเท่านั้น ส่วนการตั้งค่าแบบพื้นฐาน แบบคลาสสิก และ
แบบไม่โต้ตอบจะบันทึกพื้นที่ทำงานที่ระบุผ่านขั้นตอนปกติของแต่ละแบบ

`setup` รองรับแฟล็กการเริ่มต้นใช้งานเดียวกับ `openclaw onboard` รวมถึง
การยืนยันตัวตน (`--auth-choice`, `--token`, แฟล็กคีย์ของผู้ให้บริการ), Gateway
(`--gateway-port`, `--gateway-bind`, `--gateway-auth`, `--install-daemon`),
Tailscale (`--tailscale`), การรีเซ็ต (`--reset`, `--reset-scope`), ขั้นตอน
(`--flow quickstart|advanced|manual|import`) และแฟล็กข้ามขั้นตอน
(`--skip-channels`, `--skip-skills`, `--skip-bootstrap`, `--skip-search`,
`--skip-health`, `--skip-ui`, `--skip-hooks`) ดูรายการอ้างอิงแฟล็กฉบับเต็มและ
ตัวอย่างแบบไม่โต้ตอบได้ที่ [การเริ่มต้นใช้งาน](/th/cli/onboard) และ
[การทำงานอัตโนมัติด้วย CLI](/th/start/wizard-cli-automation) `openclaw onboard --modern` เป็นนามแฝง
เพื่อความเข้ากันได้สำหรับผู้ช่วย Crestodian ที่มีการตรวจสอบการอนุมาน และไม่มีคำสั่งที่เทียบเท่าใน `setup`

<Note>
`openclaw setup` ใช้สำหรับการติดตั้งการกำหนดค่าที่แก้ไขได้ ในโหมด Nix (`OPENCLAW_NIX_MODE=1`) OpenClaw จะปฏิเสธการเขียนจากการตั้งค่า เนื่องจากไฟล์การกำหนดค่าอยู่ภายใต้การจัดการของ Nix ใช้ [คู่มือเริ่มต้นฉบับย่อของ nix-openclaw](https://github.com/openclaw/nix-openclaw#quick-start) จากผู้พัฒนาโดยตรง หรือการกำหนดค่าต้นทางที่เทียบเท่าสำหรับแพ็กเกจ Nix อื่น
</Note>

## ตัวเลือก

| แฟล็ก                       | คำอธิบาย                                                                                           |
| -------------------------- | ----------------------------------------------------------------------------------------------------- |
| `--workspace <dir>`        | ข้อเสนอพื้นที่ทำงานในโหมดแบบมีคำแนะนำ; การตั้งค่าแบบพื้นฐาน แบบคลาสสิก และแบบไม่โต้ตอบจะบันทึกโดยตรง |
| `--baseline`               | สร้างโฟลเดอร์การกำหนดค่า/พื้นที่ทำงาน/เซสชันพื้นฐานโดยไม่เริ่มต้นใช้งาน                                  |
| `--wizard`                 | รองรับเพื่อความเข้ากันได้; การตั้งค่าจะเรียกใช้การเริ่มต้นใช้งานโดยค่าเริ่มต้น                                         |
| `--non-interactive`        | เรียกใช้การเริ่มต้นใช้งานโดยไม่มีพรอมต์                                                                       |
| `--accept-risk`            | ยอมรับความเสี่ยงจากการที่เอเจนต์เข้าถึงทั้งระบบ; จำเป็นเมื่อใช้ร่วมกับ `--non-interactive`                         |
| `--mode <mode>`            | โหมดการเริ่มต้นใช้งาน: `local` หรือ `remote`                                                                 |
| `--flow <flow>`            | ขั้นตอนการเริ่มต้นใช้งาน: `quickstart`, `advanced`, `manual` หรือ `import`                                        |
| `--reset`                  | รีเซ็ตการกำหนดค่า + ข้อมูลประจำตัว + เซสชันก่อนเริ่มต้นใช้งาน (รวมพื้นที่ทำงานเมื่อใช้ `--reset-scope full` เท่านั้น)   |
| `--reset-scope <scope>`    | ขอบเขตการรีเซ็ต: `config`, `config+creds+sessions` หรือ `full`                                            |
| `--import-from <provider>` | ผู้ให้บริการการย้ายข้อมูลที่จะเรียกใช้ระหว่างการเริ่มต้นใช้งาน                                                          |
| `--import-source <path>`   | โฮมของเอเจนต์ต้นทางสำหรับ `--import-from`                                                                |
| `--import-secrets`         | นำเข้าข้อมูลลับที่รองรับระหว่างการย้ายข้อมูลในขั้นตอนเริ่มต้นใช้งาน                                                 |
| `--remote-url <url>`       | URL WebSocket ของ Gateway ระยะไกล                                                                         |
| `--remote-token <token>`   | โทเค็นของ Gateway ระยะไกล (ไม่บังคับ)                                                                      |
| `--json`                   | แสดงผลข้อมูลสรุปแบบ JSON                                                                                |

`--classic` และ `--non-interactive` ใช้ร่วมกันไม่ได้: แบบคลาสสิกจะเปิด
ตัวช่วยสร้างที่มีพรอมต์ ส่วนการตั้งค่าแบบไม่โต้ตอบจะใช้ขั้นตอนการทำงานอัตโนมัติ

### โหมดพื้นฐาน

`openclaw setup --baseline` ยังคงรักษาลักษณะการทำงานเดิมที่ใช้เฉพาะการตั้งค่าพื้นฐาน:
โดยจะสร้างไดเรกทอรีการกำหนดค่า พื้นที่ทำงาน และเซสชัน จากนั้นออกโดยไม่
เรียกใช้การเริ่มต้นใช้งาน

## ตัวอย่าง

```bash
openclaw setup
openclaw setup --baseline
openclaw setup --workspace ~/.openclaw/workspace
openclaw setup --import-from hermes --import-source ~/.hermes
openclaw setup --non-interactive --accept-risk --mode remote --remote-url wss://gateway-host:18789 --remote-token <token>
```

## หมายเหตุ

- หลังจากตั้งค่าแบบพื้นฐานแล้ว ให้เรียกใช้ `openclaw setup` หรือ `openclaw onboard` สำหรับขั้นตอนแบบมีคำแนะนำฉบับเต็ม, `openclaw configure` สำหรับการเปลี่ยนแปลงเฉพาะจุด หรือ `openclaw channels add` เพื่อเพิ่มบัญชีช่องทาง
- หากตรวจพบสถานะของ Hermes การเริ่มต้นใช้งานแบบโต้ตอบสามารถเสนอการย้ายข้อมูลโดยอัตโนมัติได้ การเริ่มต้นใช้งานด้วยการนำเข้าจำเป็นต้องใช้การตั้งค่าใหม่; ใช้ [การย้ายข้อมูล](/th/cli/migrate) สำหรับแผนการทดลองทำ ข้อมูลสำรอง และโหมดเขียนทับนอกขั้นตอนเริ่มต้นใช้งาน
