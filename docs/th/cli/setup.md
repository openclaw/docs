# `openclaw setup`

`openclaw setup` คือจุดเริ่มต้นของเอเจนต์ระบบ ในระบบที่กำหนดค่าแล้ว การเรียก
`openclaw setup` โดยไม่มีอาร์กิวเมนต์จะเปิดแชต OpenClaw แบบโต้ตอบ ส่วนในระบบใหม่
คำสั่งจะเปลี่ยนไปใช้การเริ่มต้นใช้งานแบบมีคำแนะนำ ใช้ `-m`/`--message` สำหรับคำขอเดียว หรือ
`--baseline` เพื่อเริ่มต้นโฟลเดอร์การกำหนดค่า/เวิร์กสเปซโดยไม่ใช้วิซาร์ด

ลำดับการกำหนดเส้นทาง:

1. ตัวเลือกการเริ่มต้นใช้งานใดๆ (`--wizard`, `--baseline`, เวิร์กสเปซ, รีเซ็ต,
   ไม่โต้ตอบ, โฟลว์, โหมด, Gateway, เดมอน, ข้าม, นำเข้า, ระยะไกล หรือตัวเลือก
   การยืนยันตัวตน) จะเรียกใช้การเริ่มต้นใช้งานเหมือนกับ `openclaw onboard` ทุกประการ
2. `-m`/`--message` หรือ `--yes` จะเรียกใช้เอเจนต์ระบบ
3. เมื่อไม่มีตัวเลือกการกำหนดเส้นทาง ระบบแบบโต้ตอบที่กำหนดค่าแล้วจะเปิด OpenClaw ส่วน
   ระบบใหม่จะเรียกใช้การเริ่มต้นใช้งาน ในระบบที่กำหนดค่าแล้ว `--json` จะแสดง
   ภาพรวมระบบแม้ไม่มี TTY ส่วนตัวเลือกการเริ่มต้นใช้งานจะคงข้อมูลสรุป JSON
   ของการเริ่มต้นใช้งานไว้

ในโหมดมีคำแนะนำ `--workspace <dir>` คือเวิร์กสเปซที่เสนอให้ OpenClaw
โดยจะบันทึกถาวรหลังจากคุณอนุมัติข้อเสนอนั้นเท่านั้น การตั้งค่าแบบพื้นฐาน แบบคลาสสิก และ
แบบไม่โต้ตอบจะบันทึกเวิร์กสเปซที่ระบุผ่านโฟลว์ปกติของแต่ละแบบ

การตรวจหาการอนุมานแบบมีคำแนะนำทำงานบนโฮสต์ Gateway ใน macOS หรือ Linux โดย CLI
และแอป macOS เรียกใช้ตัวตรวจหาที่ Gateway เป็นเจ้าของร่วมกัน ซึ่งจะตรวจสอบ
โมเดลที่กำหนดค่าไว้ การเข้าสู่ระบบ CLI ที่รองรับ ตัวแปรสภาพแวดล้อมของคีย์ API และ
โมเดล Ollama หรือ LM Studio ที่ติดตั้งไว้แล้ว กระบวนการอัตโนมัตินี้จะไม่ดาวน์โหลด
โมเดลภายในเครื่องโดยเด็ดขาด โดยตัวเลือกที่เลือกต้องตอบการเติมข้อความจริงได้ก่อน
จึงจะบันทึกการกำหนดค่าผู้ให้บริการและโมเดล

`setup` รองรับแฟล็กการเริ่มต้นใช้งานเดียวกับ `openclaw onboard` รวมถึง
การยืนยันตัวตน (`--auth-choice`, `--token`, แฟล็กคีย์ผู้ให้บริการ), Gateway
(`--gateway-port`, `--gateway-bind`, `--gateway-auth`, `--install-daemon`),
Tailscale (`--tailscale`), การรีเซ็ต (`--reset`, `--reset-scope`), โฟลว์
(`--flow quickstart|advanced|manual|import`) และแฟล็กข้าม
(`--skip-channels`, `--skip-skills`, `--skip-bootstrap`, `--skip-search`,
`--skip-health`, `--skip-ui`, `--skip-hooks`) ดูข้อมูลอ้างอิงแฟล็กทั้งหมดและ
ตัวอย่างแบบไม่โต้ตอบได้ที่ [การเริ่มต้นใช้งาน](/th/cli/onboard) และ
[การทำงานอัตโนมัติของ CLI](/th/start/wizard-cli-automation) `openclaw onboard --modern` ยังคงเป็นรายการ
สำหรับความเข้ากันได้ของผู้ช่วย OpenClaw ที่มีเกตควบคุมด้วยการอนุมานแบบเดียวกัน

<Note>
`openclaw setup` ใช้สำหรับการติดตั้งที่แก้ไขการกำหนดค่าได้ ในโหมด Nix (`OPENCLAW_NIX_MODE=1`) OpenClaw จะปฏิเสธการเขียนข้อมูลตั้งค่า เนื่องจากไฟล์การกำหนดค่าจัดการโดย Nix โปรดใช้ [คู่มือเริ่มต้นฉบับย่อสำหรับ nix-openclaw ของผู้พัฒนาโดยตรง](https://github.com/openclaw/nix-openclaw#quick-start) หรือการกำหนดค่าต้นทางที่เทียบเท่าสำหรับแพ็กเกจ Nix อื่น
</Note>

## ตัวเลือก

| แฟล็ก                       | คำอธิบาย                                                                                           |
| -------------------------- | ----------------------------------------------------------------------------------------------------- |
| `-m, --message <text>`     | เรียกใช้คำขอ OpenClaw หนึ่งรายการ                                                                             |
| `--yes`                    | อนุมัติการเขียนการกำหนดค่าแบบถาวรสำหรับคำขอ `--message` หนึ่งรายการ                                         |
| `--workspace <dir>`        | ข้อเสนอเวิร์กสเปซในโหมดมีคำแนะนำ โดยการตั้งค่าแบบพื้นฐาน แบบคลาสสิก และแบบไม่โต้ตอบจะบันทึกโดยตรง |
| `--baseline`               | สร้างโฟลเดอร์การกำหนดค่า/เวิร์กสเปซ/เซสชันพื้นฐานโดยไม่เริ่มต้นใช้งาน                                  |
| `--wizard`                 | บังคับใช้การเริ่มต้นใช้งานแบบโต้ตอบ                                                                         |
| `--non-interactive`        | เรียกใช้การเริ่มต้นใช้งานโดยไม่แสดงพรอมต์                                                                       |
| `--accept-risk`            | ยอมรับความเสี่ยงจากการเข้าถึงระบบทั้งหมดของเอเจนต์ ซึ่งจำเป็นเมื่อใช้ร่วมกับ `--non-interactive`                         |
| `--mode <mode>`            | โหมดการเริ่มต้นใช้งาน: `local` หรือ `remote`                                                                 |
| `--flow <flow>`            | โฟลว์การเริ่มต้นใช้งาน: `quickstart`, `advanced`, `manual` หรือ `import`                                        |
| `--reset`                  | รีเซ็ตการกำหนดค่า + ข้อมูลประจำตัว + เซสชันก่อนเริ่มต้นใช้งาน (รวมเวิร์กสเปซเฉพาะเมื่อใช้ `--reset-scope full`)   |
| `--reset-scope <scope>`    | ขอบเขตการรีเซ็ต: `config`, `config+creds+sessions` หรือ `full`                                            |
| `--import-from <provider>` | ผู้ให้บริการการย้ายข้อมูลที่จะเรียกใช้ระหว่างการเริ่มต้นใช้งาน                                                          |
| `--import-source <path>`   | โฮมของเอเจนต์ต้นทางสำหรับ `--import-from`                                                                |
| `--import-secrets`         | นำเข้าข้อมูลลับที่รองรับระหว่างการย้ายข้อมูลในการเริ่มต้นใช้งาน                                                 |
| `--remote-url <url>`       | URL WebSocket ของ Gateway ระยะไกล                                                                         |
| `--remote-token <token>`   | โทเค็น Gateway ระยะไกล (ไม่บังคับ)                                                                      |
| `--json`                   | ระบบที่กำหนดค่าแล้ว: ภาพรวม OpenClaw เส้นทางการเริ่มต้นใช้งาน: สรุปการเริ่มต้นใช้งาน                           |

`--classic` และ `--non-interactive` ใช้ร่วมกันไม่ได้ โดยแบบคลาสสิกจะเปิด
วิซาร์ดที่มีพรอมต์ ส่วนการตั้งค่าแบบไม่โต้ตอบจะใช้เส้นทางการทำงานอัตโนมัติ

### โหมดพื้นฐาน

`openclaw setup --baseline` จะคงลักษณะการทำงานแบบพื้นฐานเท่านั้นรุ่นเก่าไว้ โดย
สร้างไดเรกทอรีการกำหนดค่า เวิร์กสเปซ และเซสชัน จากนั้นจึงออกโดยไม่
เรียกใช้การเริ่มต้นใช้งาน

## ตัวอย่าง

```bash
openclaw setup
openclaw setup -m "status"
openclaw setup -m "restart gateway" --yes
openclaw setup --json
openclaw setup --wizard
openclaw setup --baseline
openclaw setup --workspace ~/.openclaw/workspace
openclaw setup --import-from hermes --import-source ~/.hermes
openclaw setup --non-interactive --accept-risk --mode remote --remote-url wss://gateway-host:18789 --remote-token <token>
```

## หมายเหตุ

- หลังจากตั้งค่าแบบพื้นฐานแล้ว ให้เรียกใช้ `openclaw onboard` สำหรับขั้นตอนแบบมีคำแนะนำทั้งหมด, `openclaw configure` สำหรับการเปลี่ยนแปลงเฉพาะจุด หรือ `openclaw channels add` เพื่อเพิ่มบัญชีช่องทาง
- หากตรวจพบสถานะ Hermes การเริ่มต้นใช้งานแบบโต้ตอบอาจเสนอการย้ายข้อมูลโดยอัตโนมัติ การเริ่มต้นใช้งานแบบนำเข้าต้องเป็นการตั้งค่าใหม่ โปรดใช้ [การย้ายข้อมูล](/th/cli/migrate) สำหรับแผนการทดลองทำ การสำรองข้อมูล และโหมดเขียนทับภายนอกการเริ่มต้นใช้งาน
