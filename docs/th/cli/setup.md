# `openclaw setup`

`openclaw setup` คือจุดเริ่มต้นของเอเจนต์ระบบ ในระบบที่กำหนดค่าแล้ว การใช้
`openclaw setup` เพียงอย่างเดียวจะเปิดแชต OpenClaw แบบโต้ตอบ ส่วนในระบบใหม่
จะเข้าสู่กระบวนการเริ่มต้นใช้งานแบบมีคำแนะนำ ใช้ `-m`/`--message` สำหรับคำขอเดียว หรือ
`--baseline` เพื่อเริ่มต้นโฟลเดอร์การกำหนดค่า/พื้นที่ทำงานโดยไม่ใช้ตัวช่วยสร้าง

ลำดับการกำหนดเส้นทาง:

1. ตัวเลือกการเริ่มต้นใช้งานใดๆ (`--wizard`, `--baseline`, พื้นที่ทำงาน, รีเซ็ต,
   แบบไม่โต้ตอบ, โฟลว์, โหมด, Gateway, เดมอน, ข้าม, นำเข้า, ระยะไกล หรือตัวเลือก
   การยืนยันตัวตน) จะเรียกใช้การเริ่มต้นใช้งานเหมือนกับ `openclaw onboard` ทุกประการ
2. `-m`/`--message` หรือ `--yes` จะเรียกใช้เอเจนต์ระบบ
3. เมื่อไม่มีตัวเลือกการกำหนดเส้นทาง ระบบแบบโต้ตอบที่กำหนดค่าแล้วจะเปิด OpenClaw ส่วน
   ระบบใหม่จะเรียกใช้การเริ่มต้นใช้งาน ในระบบที่กำหนดค่าแล้ว `--json` จะแสดง
   ภาพรวมระบบแม้ไม่มี TTY ส่วนตัวเลือกการเริ่มต้นใช้งานจะคงข้อมูลสรุป JSON
   ของการเริ่มต้นใช้งานไว้

ในโหมดมีคำแนะนำ `--workspace <dir>` คือพื้นที่ทำงานที่เสนอให้ OpenClaw
โดยจะบันทึกถาวรหลังจากอนุมัติข้อเสนอนั้นเท่านั้น การตั้งค่าแบบพื้นฐาน แบบคลาสสิก และ
แบบไม่โต้ตอบจะบันทึกพื้นที่ทำงานที่ระบุผ่านโฟลว์ตามปกติของแต่ละแบบ

การตรวจหาการอนุมานแบบมีคำแนะนำจะทำงานบนโฮสต์ Gateway ที่ใช้ macOS หรือ Linux โดย CLI
และแอป macOS จะเรียกใช้ตัวตรวจหาที่ Gateway เป็นเจ้าของร่วมกัน ซึ่งตรวจสอบ
โมเดลที่กำหนดค่าไว้ การเข้าสู่ระบบ CLI ที่รองรับ ตัวแปรสภาพแวดล้อมของคีย์ API และโมเดล
Ollama หรือ LM Studio ที่ติดตั้งไว้แล้ว กระบวนการอัตโนมัตินี้จะไม่ดาวน์โหลดโมเดลภายในเครื่อง
โดยเด็ดขาด รันไทม์ภายในเครื่องที่ตรวจพบจะได้รับการทดสอบโดยอัตโนมัติหลังจากตัวเลือก CLI และคีย์ API
เมื่อมีโมเดลภายในเครื่องหลายรายการ OpenClaw จะเลือกตระกูลโมเดล instruct
ที่เรียกใช้เครื่องมือได้ดีที่สุด ตัวเลือกที่เลือกต้องตอบการเติมข้อความจริงได้
ก่อนบันทึกการกำหนดค่าผู้ให้บริการและโมเดล นอกจากนี้ ระบบจะรายงาน CLI ของ Gemini,
Antigravity, Pi และ OpenCode ที่ติดตั้งไว้ เมื่อ CLI เหล่านั้นไม่สามารถใช้เป็นเส้นทางการอนุมาน
ที่นำกลับมาใช้ซ้ำได้สำหรับการตั้งค่าแบบมีคำแนะนำ

`setup` รองรับแฟล็กการเริ่มต้นใช้งานเดียวกับ `openclaw onboard` รวมถึง
การยืนยันตัวตน (`--auth-choice`, `--token`, แฟล็กคีย์ของผู้ให้บริการ), Gateway
(`--gateway-port`, `--gateway-bind`, `--gateway-auth`, `--install-daemon`),
Tailscale (`--tailscale`), การรีเซ็ต (`--reset`, `--reset-scope`), โฟลว์
(`--flow quickstart|advanced|manual|import`) และแฟล็กข้าม
(`--skip-channels`, `--skip-skills`, `--skip-bootstrap`, `--skip-search`,
`--skip-health`, `--skip-ui`, `--skip-hooks`) โปรดดู [การเริ่มต้นใช้งาน](/th/cli/onboard) และ
[ระบบอัตโนมัติของ CLI](/th/start/wizard-cli-automation) สำหรับข้อมูลอ้างอิงแฟล็กทั้งหมดและ
ตัวอย่างแบบไม่โต้ตอบ `openclaw onboard --modern` ยังคงเป็นรายการเข้าที่รองรับความเข้ากันได้
สำหรับผู้ช่วย OpenClaw ที่ใช้เกตการอนุมานเดียวกัน

<Note>
`openclaw setup` ใช้สำหรับการติดตั้งที่แก้ไขการกำหนดค่าได้ ในโหมด Nix (`OPENCLAW_NIX_MODE=1`) OpenClaw จะปฏิเสธการเขียนข้อมูลจากการตั้งค่า เนื่องจากไฟล์การกำหนดค่าจัดการโดย Nix ให้ใช้ [คู่มือเริ่มต้นฉบับย่อของ nix-openclaw](https://github.com/openclaw/nix-openclaw#quick-start) จากผู้พัฒนาโดยตรง หรือการกำหนดค่าซอร์สที่เทียบเท่าสำหรับแพ็กเกจ Nix อื่น
</Note>

## ตัวเลือก

| แฟล็ก                       | คำอธิบาย                                                                                           |
| -------------------------- | ----------------------------------------------------------------------------------------------------- |
| `-m, --message <text>`     | เรียกใช้คำขอ OpenClaw หนึ่งรายการ                                                                             |
| `--yes`                    | อนุมัติการเขียนการกำหนดค่าแบบถาวรสำหรับคำขอ `--message` หนึ่งรายการ                                         |
| `--workspace <dir>`        | ข้อเสนอพื้นที่ทำงานในโหมดมีคำแนะนำ โดยการตั้งค่าแบบพื้นฐาน แบบคลาสสิก และแบบไม่โต้ตอบจะบันทึกโดยตรง |
| `--baseline`               | สร้างโฟลเดอร์การกำหนดค่า/พื้นที่ทำงาน/เซสชันพื้นฐานโดยไม่เริ่มต้นใช้งาน                                  |
| `--wizard`                 | บังคับใช้การเริ่มต้นใช้งานแบบโต้ตอบ                                                                         |
| `--non-interactive`        | เรียกใช้การเริ่มต้นใช้งานโดยไม่มีพรอมต์                                                                       |
| `--accept-risk`            | รับทราบความเสี่ยงจากการเข้าถึงระบบทั้งหมดของเอเจนต์ ซึ่งจำเป็นเมื่อใช้ร่วมกับ `--non-interactive`                         |
| `--mode <mode>`            | โหมดการเริ่มต้นใช้งาน: `local` หรือ `remote`                                                                 |
| `--flow <flow>`            | โฟลว์การเริ่มต้นใช้งาน: `quickstart`, `advanced`, `manual` หรือ `import`                                        |
| `--reset`                  | รีเซ็ตการกำหนดค่า + ข้อมูลประจำตัว + เซสชันก่อนเริ่มต้นใช้งาน (รวมพื้นที่ทำงานเมื่อใช้ `--reset-scope full` เท่านั้น)   |
| `--reset-scope <scope>`    | ขอบเขตการรีเซ็ต: `config`, `config+creds+sessions` หรือ `full`                                            |
| `--import-from <provider>` | ผู้ให้บริการการย้ายข้อมูลที่จะเรียกใช้ระหว่างการเริ่มต้นใช้งาน                                                          |
| `--import-source <path>`   | โฮมของเอเจนต์ต้นทางสำหรับ `--import-from`                                                                |
| `--import-secrets`         | นำเข้าข้อมูลลับที่รองรับระหว่างการย้ายข้อมูลในขั้นตอนเริ่มต้นใช้งาน                                                 |
| `--remote-url <url>`       | URL ของ WebSocket สำหรับ Gateway ระยะไกล                                                                         |
| `--remote-token <token>`   | โทเค็น Gateway ระยะไกล (ไม่บังคับ)                                                                      |
| `--json`                   | ระบบที่กำหนดค่าแล้ว: ภาพรวม OpenClaw เส้นทางการเริ่มต้นใช้งาน: ข้อมูลสรุปการเริ่มต้นใช้งาน                           |

`--classic` และ `--non-interactive` ใช้ร่วมกันไม่ได้ โดยแบบคลาสสิกจะเปิด
ตัวช่วยสร้างที่มีพรอมต์ ส่วนการตั้งค่าแบบไม่โต้ตอบจะใช้เส้นทางระบบอัตโนมัติ

### โหมดพื้นฐาน

`openclaw setup --baseline` จะคงพฤติกรรมเดิมที่ใช้เฉพาะแบบพื้นฐานไว้ โดยจะ
สร้างไดเรกทอรีการกำหนดค่า พื้นที่ทำงาน และเซสชัน จากนั้นออกโดยไม่
เรียกใช้การเริ่มต้นใช้งาน

## ตัวอย่าง

```bash
openclaw setup
openclaw setup -m "status"
openclaw setup -m "restart gateway" --yes
openclaw setup --json
openclaw setup --wizard
openclaw setup --baseline
openclaw setup --workspace ~/.openclaw/workspace
openclaw setup --import-from hermes --import-source ~/.hermes
openclaw setup --non-interactive --accept-risk --mode remote --remote-url wss://gateway-host:18789 --remote-token <token>
```

## หมายเหตุ

- หลังการตั้งค่าแบบพื้นฐาน ให้เรียกใช้ `openclaw onboard` สำหรับกระบวนการแบบมีคำแนะนำทั้งหมด, `openclaw configure` สำหรับการเปลี่ยนแปลงเฉพาะจุด หรือ `openclaw channels add` เพื่อเพิ่มบัญชีช่องทาง
- หากตรวจพบสถานะ Hermes การเริ่มต้นใช้งานแบบโต้ตอบสามารถเสนอการย้ายข้อมูลโดยอัตโนมัติได้ การเริ่มต้นใช้งานแบบนำเข้าต้องใช้การตั้งค่าใหม่ โปรดใช้ [ย้ายข้อมูล](/th/cli/migrate) สำหรับแผนทดลองดำเนินการ การสำรองข้อมูล และโหมดเขียนทับนอกกระบวนการเริ่มต้นใช้งาน
