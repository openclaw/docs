# `openclaw setup`

`openclaw setup` คือจุดเริ่มต้นของเอเจนต์ระบบ บนระบบที่กำหนดค่าแล้ว การใช้
`openclaw setup` โดยไม่มีอาร์กิวเมนต์จะเปิดแชต OpenClaw แบบโต้ตอบ ส่วนบนระบบใหม่
คำสั่งจะเปลี่ยนไปใช้การเริ่มต้นใช้งานแบบมีคำแนะนำ ใช้ `-m`/`--message` สำหรับคำขอเดียว หรือ
`--baseline` เพื่อเริ่มต้นโฟลเดอร์การกำหนดค่า/พื้นที่ทำงานโดยไม่ใช้วิซาร์ด

ลำดับการกำหนดเส้นทาง:

1. ตัวเลือกการเริ่มต้นใช้งานใดๆ (`--wizard`, `--baseline`, พื้นที่ทำงาน, รีเซ็ต,
   ไม่โต้ตอบ, โฟลว์, โหมด, Gateway, ดีมอน, ข้าม, นำเข้า, ระยะไกล หรือตัวเลือกการยืนยันตัวตน)
   จะเรียกใช้การเริ่มต้นใช้งานเหมือนกับ `openclaw onboard` ทุกประการ
2. `-m`/`--message` หรือ `--yes` จะเรียกใช้เอเจนต์ระบบ
3. เมื่อไม่มีตัวเลือกการกำหนดเส้นทาง ระบบแบบโต้ตอบที่กำหนดค่าแล้วจะเปิด OpenClaw ส่วน
   ระบบใหม่จะเรียกใช้การเริ่มต้นใช้งาน บนระบบที่กำหนดค่าแล้ว `--json` จะแสดง
   ภาพรวมระบบแม้ไม่มี TTY ส่วนตัวเลือกการเริ่มต้นใช้งานจะคงข้อมูลสรุป JSON
   ของการเริ่มต้นใช้งานไว้

ในโหมดมีคำแนะนำ `--workspace <dir>` คือพื้นที่ทำงานที่เสนอให้ OpenClaw
โดยจะบันทึกถาวรหลังจากคุณอนุมัติข้อเสนอนั้นเท่านั้น การตั้งค่าแบบพื้นฐาน แบบคลาสสิก และ
แบบไม่โต้ตอบจะบันทึกพื้นที่ทำงานที่ระบุผ่านโฟลว์ปกติของแต่ละแบบ

การตรวจหาการอนุมานแบบมีคำแนะนำทำงานบนโฮสต์ Gateway ที่ใช้ macOS หรือ Linux โดย CLI
และแอป macOS จะเรียกใช้ตัวตรวจหาตัวเดียวกันซึ่ง Gateway เป็นเจ้าของ ตัวตรวจหานี้จะตรวจสอบ
โมเดลที่กำหนดค่าไว้ การเข้าสู่ระบบ CLI ที่รองรับ ตัวแปรสภาพแวดล้อมของคีย์ API และโมเดล
Ollama หรือ LM Studio ที่ติดตั้งไว้แล้ว กระบวนการอัตโนมัตินี้จะไม่ดาวน์โหลดโมเดลภายในเครื่อง
ตัวรันไทม์ภายในเครื่องที่ตรวจพบจะได้รับการทดสอบโดยอัตโนมัติหลังจากตัวเลือก CLI และคีย์ API
เมื่อมีโมเดลภายในเครื่องหลายโมเดล OpenClaw จะเลือกตระกูล instruct ที่มีความสามารถในการเรียกใช้เครื่องมือ
สูงสุด ตัวเลือกที่เลือกต้องตอบคำขอ completion จริงได้ก่อนที่จะบันทึกการกำหนดค่าผู้ให้บริการและโมเดล
นอกจากนี้ ระบบจะรายงาน CLI ของ Gemini, Antigravity, Pi และ OpenCode ที่ติดตั้งไว้ด้วย เมื่อ
CLI เหล่านั้นไม่สามารถใช้เป็นเส้นทางการอนุมานที่นำกลับมาใช้ซ้ำได้สำหรับการตั้งค่าแบบมีคำแนะนำ

`setup` รองรับแฟล็กการเริ่มต้นใช้งานชุดเดียวกับ `openclaw onboard` รวมถึง
การยืนยันตัวตน (`--auth-choice`, `--token`, แฟล็กคีย์ผู้ให้บริการ), Gateway
(`--gateway-port`, `--gateway-bind`, `--gateway-auth`, `--install-daemon`),
Tailscale (`--tailscale`), การรีเซ็ต (`--reset`, `--reset-scope`), โฟลว์
(`--flow quickstart|advanced|manual|import`) และแฟล็กข้าม
(`--skip-channels`, `--skip-skills`, `--skip-bootstrap`, `--skip-search`,
`--skip-health`, `--skip-ui`, `--skip-hooks`) ส่ง `--tui` เพื่อใช้
ทางเลือกผ่านเทอร์มินัลแบบเดียวกับ `openclaw onboard --tui` ดูข้อมูลอ้างอิงแฟล็กทั้งหมดและ
ตัวอย่างแบบไม่โต้ตอบได้ที่ [การเริ่มต้นใช้งาน](/th/cli/onboard) และ
[การทำงานอัตโนมัติด้วย CLI](/th/start/wizard-cli-automation) `openclaw onboard --modern` ยังคงเป็นรายการ
ที่รองรับความเข้ากันได้สำหรับผู้ช่วย OpenClaw ที่มีการควบคุมด้วยการอนุมานแบบเดียวกัน

<Note>
`openclaw setup` ใช้สำหรับการติดตั้งที่แก้ไขการกำหนดค่าได้ ในโหมด Nix (`OPENCLAW_NIX_MODE=1`) OpenClaw จะปฏิเสธการเขียนข้อมูลการตั้งค่า เนื่องจากไฟล์การกำหนดค่าจัดการโดย Nix ให้ใช้ [คู่มือเริ่มต้นใช้งานฉบับย่อของ nix-openclaw](https://github.com/openclaw/nix-openclaw#quick-start) จากผู้พัฒนาโดยตรง หรือการกำหนดค่าต้นฉบับที่เทียบเท่าสำหรับแพ็กเกจ Nix อื่น
</Note>

## ตัวเลือก

| แฟล็ก                       | คำอธิบาย                                                                                           |
| -------------------------- | ----------------------------------------------------------------------------------------------------- |
| `-m, --message <text>`     | เรียกใช้คำขอ OpenClaw หนึ่งรายการ                                                                             |
| `--yes`                    | อนุมัติการเขียนการกำหนดค่าแบบถาวรสำหรับคำขอ `--message` หนึ่งรายการ                                         |
| `--workspace <dir>`        | ข้อเสนอพื้นที่ทำงานในโหมดมีคำแนะนำ โดยการตั้งค่าแบบพื้นฐาน แบบคลาสสิก และแบบไม่โต้ตอบจะบันทึกโดยตรง |
| `--baseline`               | สร้างโฟลเดอร์การกำหนดค่าพื้นฐาน/พื้นที่ทำงาน/เซสชันโดยไม่เริ่มต้นใช้งาน                                  |
| `--wizard`                 | บังคับใช้การเริ่มต้นใช้งานแบบโต้ตอบ                                                                         |
| `--tui`                    | ใช้ทางเลือกผ่านเทอร์มินัลแทนการส่งต่อไปยังเบราว์เซอร์                                                |
| `--non-interactive`        | เรียกใช้การเริ่มต้นใช้งานโดยไม่แสดงพรอมต์                                                                       |
| `--accept-risk`            | รับทราบความเสี่ยงจากการให้เอเจนต์เข้าถึงทั้งระบบ ซึ่งต้องใช้ร่วมกับ `--non-interactive`                         |
| `--mode <mode>`            | โหมดการเริ่มต้นใช้งาน: `local` หรือ `remote`                                                                 |
| `--flow <flow>`            | โฟลว์การเริ่มต้นใช้งาน: `quickstart`, `advanced`, `manual` หรือ `import`                                        |
| `--reset`                  | รีเซ็ตการกำหนดค่า + ข้อมูลประจำตัว + เซสชันก่อนเริ่มต้นใช้งาน (รวมพื้นที่ทำงานเฉพาะเมื่อใช้ `--reset-scope full`)   |
| `--reset-scope <scope>`    | ขอบเขตการรีเซ็ต: `config`, `config+creds+sessions` หรือ `full`                                            |
| `--import-from <provider>` | ผู้ให้บริการการย้ายข้อมูลที่จะเรียกใช้ระหว่างการเริ่มต้นใช้งาน                                                          |
| `--import-source <path>`   | โฮมของเอเจนต์ต้นทางสำหรับ `--import-from`                                                                |
| `--import-secrets`         | นำเข้าข้อมูลลับที่รองรับระหว่างการย้ายข้อมูลในการเริ่มต้นใช้งาน                                                 |
| `--remote-url <url>`       | URL WebSocket ของ Gateway ระยะไกล                                                                         |
| `--remote-token <token>`   | โทเค็น Gateway ระยะไกล (ไม่บังคับ)                                                                      |
| `--json`                   | ระบบที่กำหนดค่าแล้ว: ภาพรวม OpenClaw เส้นทางการเริ่มต้นใช้งาน: ข้อมูลสรุปการเริ่มต้นใช้งาน                           |

`--classic` และ `--non-interactive` ใช้ร่วมกันไม่ได้: แบบคลาสสิกจะเปิด
วิซาร์ดที่แสดงพรอมต์ ส่วนการตั้งค่าแบบไม่โต้ตอบจะใช้เส้นทางการทำงานอัตโนมัติ
ในการเริ่มต้นใช้งานแบบโต้ตอบ `--remote-url` และ `--remote-token` จะกรอกข้อมูลล่วงหน้าใน
ขั้นตอน Gateway ระยะไกล และมีลำดับความสำคัญเหนือค่าระยะไกลที่บันทึกไว้สำหรับการทำงานครั้งนั้น
การเปลี่ยน URL จะไม่นำข้อมูลประจำตัวที่บันทึกไว้กลับมาใช้ เว้นแต่คุณจะส่งโทเค็นด้วย
โทเค็นจะยังคงถูกปกปิดและใช้โหมดจัดเก็บข้อความธรรมดาหรือ SecretRef ที่เลือกในวิซาร์ด

### โหมดพื้นฐาน

`openclaw setup --baseline` จะรักษาลักษณะการทำงานแบบพื้นฐานเท่านั้นที่มีมาก่อน โดย
สร้างไดเรกทอรีการกำหนดค่า พื้นที่ทำงาน และเซสชัน จากนั้นออกโดยไม่
เรียกใช้การเริ่มต้นใช้งาน คำสั่งนี้รองรับ `--workspace` และตัวควบคุมเอาต์พุตที่ไม่มีผลกระทบ แต่
จะปฏิเสธตัวเลือกการเริ่มต้นใช้งาน, Gateway, การยืนยันตัวตน, การรีเซ็ต หรือดีมอนที่ระบุอย่างชัดเจน แทนที่จะ
ละเว้นตัวเลือกเหล่านั้นโดยไม่แจ้งให้ทราบ

## ตัวอย่าง

```bash
openclaw setup
openclaw setup -m "สถานะ"
openclaw setup -m "รีสตาร์ต Gateway" --yes
openclaw setup --json
openclaw setup --wizard
openclaw setup --baseline
openclaw setup --workspace ~/.openclaw/workspace
openclaw setup --import-from hermes --import-source ~/.hermes
openclaw setup --non-interactive --accept-risk --mode remote --remote-url wss://gateway-host:18789 --remote-token <token>
```

## หมายเหตุ

- หลังจากตั้งค่าแบบพื้นฐาน ให้เรียกใช้ `openclaw onboard` สำหรับขั้นตอนแบบมีคำแนะนำทั้งหมด, `openclaw configure` สำหรับการเปลี่ยนแปลงเฉพาะจุด หรือ `openclaw channels add` เพื่อเพิ่มบัญชีช่องทาง
- หากตรวจพบสถานะ Hermes การเริ่มต้นใช้งานแบบโต้ตอบสามารถเสนอการย้ายข้อมูลโดยอัตโนมัติได้ การเริ่มต้นใช้งานแบบนำเข้าต้องใช้การตั้งค่าใหม่ โปรดดู [การย้ายข้อมูล](/th/cli/migrate) สำหรับแผนการทดลองทำงาน การสำรองข้อมูล และโหมดเขียนทับนอกการเริ่มต้นใช้งาน
