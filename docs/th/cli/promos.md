# `openclaw promos`

ค้นหาและรับข้อเสนอโปรโมชันสำหรับโมเดลที่เผยแพร่บน ClawHub การรับโปรโมชันจะกำหนดค่าผู้ให้บริการ (การยืนยันตัวตนและ Plugin เมื่อจำเป็น) และลงทะเบียนโมเดลของโปรโมชัน โดยไม่ต้องดำเนินการเริ่มต้นใช้งานซ้ำ และไม่เปลี่ยนโมเดลเริ่มต้นของคุณ เว้นแต่คุณจะระบุให้เปลี่ยน

หัวข้อที่เกี่ยวข้อง:

- โมเดลเริ่มต้นและโมเดลสำรอง: [โมเดล](/th/cli/models)
- การตั้งค่าการยืนยันตัวตนของผู้ให้บริการ: [เริ่มต้นใช้งาน](/th/start/getting-started)

## คำสั่ง

```bash
openclaw promos list
openclaw promos claim <slug>
openclaw promos claim <slug> --api-key <key> --set-default
```

## `openclaw promos list`

แสดงรายการโปรโมชันที่ใช้งานอยู่ในขณะนี้ พร้อมโมเดล โมเดลเริ่มต้นที่แนะนำ เวลาที่เหลือ และคำสั่งรับสิทธิ์ที่ถูกต้องครบถ้วน `--json` จะแสดงเพย์โหลดดิบ

## `openclaw promos claim <slug>`

รับโปรโมชันที่ใช้งานอยู่:

1. ดึงข้อมูลโปรโมชันจาก ClawHub และตรวจสอบว่ายังอยู่ภายในช่วงเวลาที่กำหนด
2. ตรวจสอบผู้ให้บริการ ตัวเลือกการยืนยันตัวตน และแพ็กเกจ Plugin ที่โปรโมชันประกาศไว้เทียบกับ OpenClaw เวอร์ชันที่คุณติดตั้ง ระบบจะปฏิเสธรหัสที่ไม่รู้จักหรือแพ็กเกจที่ไม่ตรงกัน โปรโมชันจะไม่สามารถทำให้ CLI เรียกใช้สิ่งใดที่ CLI ยังไม่รู้วิธีดำเนินการได้
3. ใช้ข้อมูลประจำตัวของผู้ให้บริการที่มีอยู่แล้วเมื่อคุณมีข้อมูลดังกล่าว มิฉะนั้น ระบบจะนำคุณผ่านขั้นตอนการยืนยันตัวตนตามปกติของผู้ให้บริการ (โดยแสดง URL สมัครใช้งานของโปรโมชันเพื่อรับคีย์ฟรีก่อน) `--api-key <key>` จะดำเนินการยืนยันตัวตนด้วยคีย์ API ให้เสร็จสิ้นโดยไม่แสดงพรอมต์ ซึ่งสอดคล้องกับแฟล็กแบบไม่โต้ตอบของ `openclaw onboard` หากไม่ต้องการใส่คีย์ในบรรทัดคำสั่ง ให้ส่งออกตัวแปรสภาพแวดล้อมของผู้ให้บริการแทน (เช่น `OPENROUTER_API_KEY`) ระบบจะตรวจพบข้อมูลประจำตัวจากตัวแปรสภาพแวดล้อมที่มีอยู่โดยอัตโนมัติ และไม่จำเป็นต้องใช้แฟล็ก
4. ลงทะเบียนโมเดลของโปรโมชันพร้อมชื่อแทน โดยจะไม่เขียนทับชื่อแทนที่มีอยู่
5. เสนอให้ตั้งโมเดลที่โปรโมชันแนะนำเป็นโมเดลเริ่มต้นของคุณ โดย `--set-default` จะข้ามคำถามนี้ มิฉะนั้น การตั้งค่าเริ่มต้นของคุณจะไม่เปลี่ยนแปลง

เมื่อช่วงเวลาของโปรโมชันสิ้นสุดลง ผู้ให้บริการจะหยุดให้บริการโมเดลฟรี โดยการกำหนดค่าและข้อมูลประจำตัวของคุณจะไม่เปลี่ยนแปลง คุณสามารถสลับกลับได้ทุกเมื่อด้วย `openclaw models set <model>`

## การค้นหาแบบพาสซีฟใน `models list`

`openclaw models list` จะแสดงโปรโมชันด้วย แม้ว่าคุณจะไม่ได้สอบถาม ClawHub โดยตรง:

- ข้อเสนอที่ใช้งานอยู่ซึ่งมีโมเดลที่คุณยังไม่ได้กำหนดค่า จะปรากฏในกลุ่ม "พร้อมใช้งานผ่านโปรโมชัน" ใต้ตาราง โดยแต่ละรายการจะแสดงคำสั่งรับสิทธิ์
- โมเดลที่คุณลงทะเบียนผ่าน `promos claim` จะมีแท็ก `promo` ซึ่งจะเปลี่ยนเป็น `promo ended` เมื่อพ้นช่วงเวลาของข้อเสนอ
- เมื่อพบข้อเสนอใหม่เป็นครั้งแรก การแจ้งเตือนแบบครั้งเดียวจะแนะนำให้ใช้ `openclaw promos list` ข้อเสนอที่คุณเคยแสดงรายการหรือรับสิทธิ์แล้วจะไม่ถูกประกาศซ้ำ

การทำงานนี้จะอ่านสำเนาฟีดโปรโมชันที่โฮสต์โดย ClawHub จากแคชภายในเครื่อง (โดยปกติจะรีเฟรชวันละครั้งด้วยคำขอแบบมีเงื่อนไข หรือเร็วกว่านั้นเมื่อสแนปช็อตในแคชหมดอายุ และจะข้ามความล้มเหลวในการรีเฟรชโดยไม่แจ้งเตือน) การรีเฟรชข้อมูลที่ล้าสมัยจะรอไม่เกิน 2.5 วินาทีและจะไม่ทำให้การแสดงรายการล้มเหลว เอาต์พุต `--json` และ `--plain` จะคงรูปแบบที่สะอาดสำหรับการประมวลผลโดยเครื่อง โดยไม่มีส่วนโปรโมชันหรือการแจ้งเตือน การรับสิทธิ์จะตรวจสอบกับ API แบบสดของ ClawHub ซ้ำเสมอ ดังนั้นระบบจะปฏิเสธข้อเสนอที่ถูกถอนก่อนกำหนด แม้ว่าสำเนาในแคชจะยังคงแสดงข้อเสนอนั้นอยู่ก็ตาม
