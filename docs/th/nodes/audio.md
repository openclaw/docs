# บันทึกเสียง / ข้อความเสียง (2026-01-17)

## สิ่งที่ใช้งานได้

- **ความเข้าใจสื่อ (เสียง)**: หากเปิดใช้งานความเข้าใจเสียง (หรือตรวจพบอัตโนมัติ) OpenClaw จะ:
  1. ระบุไฟล์แนบเสียงไฟล์แรก (พาธในเครื่องหรือ URL) และดาวน์โหลดหากจำเป็น
  2. บังคับใช้ `maxBytes` ก่อนส่งไปยังแต่ละรายการโมเดล
  3. เรียกใช้รายการโมเดลรายการแรกที่เข้าเงื่อนไขตามลำดับ (ผู้ให้บริการหรือ CLI)
  4. หากล้มเหลวหรือถูกข้าม (ขนาด/หมดเวลา) จะลองรายการถัดไป
  5. เมื่อสำเร็จ จะแทนที่ `Body` ด้วยบล็อก `[Audio]` และตั้งค่า `{{Transcript}}`
- **การแยกวิเคราะห์คำสั่ง**: เมื่อถอดเสียงสำเร็จ `CommandBody`/`RawBody` จะถูกตั้งค่าเป็นข้อความถอดเสียง เพื่อให้คำสั่งสแลชยังคงทำงานได้
- **การบันทึกแบบละเอียด**: ใน `--verbose` เราจะบันทึกเมื่อการถอดเสียงทำงาน และเมื่อแทนที่เนื้อความ
- **การป้อนตามเสียงใน UI ควบคุม**: ตัวเขียนข้อความแชตสามารถส่งคลิปไมโครโฟนที่บันทึกจากเบราว์เซอร์ไปยัง `chat.transcribeAudio` ได้ Gateway RPC ดังกล่าวจะเขียนคลิปลงในไฟล์ชั่วคราวในเครื่อง เรียกใช้ไปป์ไลน์การถอดเสียงเดียวกันนี้ ส่งคืนข้อความร่างไปยังเบราว์เซอร์ และลบไฟล์ชั่วคราว โดยตัวมันเองจะไม่สร้างการรันของเอเจนต์

## การตรวจจับอัตโนมัติ (ค่าเริ่มต้น)

หากคุณ **ไม่ได้กำหนดค่าโมเดล** และ `tools.media.audio.enabled` **ไม่ได้** ตั้งเป็น `false`
OpenClaw จะตรวจจับอัตโนมัติตามลำดับนี้ และหยุดเมื่อพบตัวเลือกแรกที่ใช้งานได้:

1. **โมเดลตอบกลับที่ใช้งานอยู่** เมื่อผู้ให้บริการของโมเดลรองรับความเข้าใจเสียง
2. **CLI ในเครื่อง** (หากติดตั้งไว้)
   - `sherpa-onnx-offline` (ต้องใช้ `SHERPA_ONNX_MODEL_DIR` ที่มี encoder/decoder/joiner/tokens)
   - `whisper-cli` (จาก `whisper-cpp`; ใช้ `WHISPER_CPP_MODEL` หรือโมเดล tiny ที่รวมมาให้)
   - `whisper` (Python CLI; ดาวน์โหลดโมเดลโดยอัตโนมัติ)
3. **Gemini CLI** (`gemini`) โดยใช้ `read_many_files`
4. **การตรวจสอบสิทธิ์ผู้ให้บริการ**
   - รายการ `models.providers.*` ที่กำหนดค่าไว้และรองรับเสียงจะถูกลองก่อน
   - ลำดับสำรองที่รวมมาให้: OpenAI → Groq → xAI → Deepgram → Google → SenseAudio → ElevenLabs → Mistral

หากต้องการปิดการตรวจจับอัตโนมัติ ให้ตั้งค่า `tools.media.audio.enabled: false`
หากต้องการปรับแต่ง ให้ตั้งค่า `tools.media.audio.models`
หมายเหตุ: การตรวจจับไบนารีเป็นการพยายามให้ดีที่สุดบน macOS/Linux/Windows; ตรวจสอบให้แน่ใจว่า CLI อยู่ใน `PATH` (เราขยาย `~`) หรือตั้งค่าโมเดล CLI อย่างชัดเจนด้วยพาธคำสั่งแบบเต็ม

## ตัวอย่างการกำหนดค่า

### ผู้ให้บริการ + CLI สำรอง (OpenAI + Whisper CLI)

```json5
{
  tools: {
    media: {
      audio: {
        enabled: true,
        maxBytes: 20971520,
        models: [
          { provider: "openai", model: "gpt-4o-mini-transcribe" },
          {
            type: "cli",
            command: "whisper",
            args: ["--model", "base", "{{MediaPath}}"],
            timeoutSeconds: 45,
          },
        ],
      },
    },
  },
}
```

### เฉพาะผู้ให้บริการพร้อมการควบคุมด้วยขอบเขต

```json5
{
  tools: {
    media: {
      audio: {
        enabled: true,
        scope: {
          default: "allow",
          rules: [{ action: "deny", match: { chatType: "group" } }],
        },
        models: [{ provider: "openai", model: "gpt-4o-mini-transcribe" }],
      },
    },
  },
}
```

### เฉพาะผู้ให้บริการ (Deepgram)

```json5
{
  tools: {
    media: {

        enabled: true,
        models: [{ provider: "deepgram", model: "nova-3" }],
      },
    },
  },
}
```

### สะท้อนข้อความถอดเสียงกลับไปยังแชต (เลือกเปิดใช้)

```json5

  tools: {
    media: {
      audio: {

      },
    },
  },
}
```

## หมายเหตุและข้อจำกัด

- การตรวจสอบสิทธิ์ผู้ให้บริการเป็นไปตามลำดับการตรวจสอบสิทธิ์โมเดลมาตรฐาน (โปรไฟล์การตรวจสอบสิทธิ์ ตัวแปรสภาพแวดล้อม `models.providers.*.apiKey`)
- รายละเอียดการตั้งค่า Groq: [Groq](/th/providers/groq)
- Deepgram จะรับ `DEEPGRAM_API_KEY` เมื่อใช้ `provider: "deepgram"`
- รายละเอียดการตั้งค่า Deepgram: [Deepgram (การถอดเสียง)](/th/providers/deepgram)
- รายละเอียดการตั้งค่า Mistral: [Mistral](/th/providers/mistral)
- SenseAudio จะรับ `SENSEAUDIO_API_KEY` เมื่อใช้ `provider: "senseaudio"`
- รายละเอียดการตั้งค่า SenseAudio: [SenseAudio](/th/providers/senseaudio)
- ผู้ให้บริการเสียงสามารถแทนที่ `baseUrl`, `headers` และ `providerOptions` ผ่าน `tools.media.audio` ได้
- ขีดจำกัดขนาดเริ่มต้นคือ 20MB (`tools.media.audio.maxBytes`) เสียงที่เกินขนาดจะถูกข้ามสำหรับโมเดลนั้น และจะลองรายการถัดไป
- ไฟล์เสียงที่เล็กมาก/ว่างเปล่าต่ำกว่า 1024 ไบต์จะถูกข้ามก่อนการถอดเสียงผ่านผู้ให้บริการ/CLI
- ค่าเริ่มต้น `maxChars` สำหรับเสียงคือ **ไม่ได้ตั้งค่า** (ข้อความถอดเสียงแบบเต็ม) ตั้งค่า `tools.media.audio.maxChars` หรือ `maxChars` ต่อรายการเพื่อตัดผลลัพธ์
- ค่าเริ่มต้นอัตโนมัติของ OpenAI คือ `gpt-4o-mini-transcribe`; ตั้งค่า `model: "gpt-4o-transcribe"` เพื่อความแม่นยำที่สูงขึ้น
- ใช้ `tools.media.audio.attachments` เพื่อประมวลผลข้อความเสียงหลายรายการ (`mode: "all"` + `maxAttachments`)
- ข้อความถอดเสียงพร้อมให้เทมเพลตใช้เป็น `{{Transcript}}`
- `tools.media.audio.echoTranscript` ปิดไว้โดยค่าเริ่มต้น; เปิดใช้เพื่อส่งการยืนยันข้อความถอดเสียงกลับไปยังแชตต้นทางก่อนการประมวลผลของเอเจนต์
- `tools.media.audio.echoFormat` ปรับแต่งข้อความสะท้อนกลับ (ตัวยึดตำแหน่ง: `{transcript}`)
- stdout ของ CLI ถูกจำกัดขนาด (5MB); ให้ผลลัพธ์ CLI กระชับ
- `args` ของ CLI ควรใช้ `{{MediaPath}}` สำหรับพาธไฟล์เสียงในเครื่อง เรียกใช้ `openclaw doctor --fix` เพื่อย้ายตัวยึดตำแหน่ง `{input}` ที่เลิกใช้แล้วจากการกำหนดค่า `audio.transcription.command` รุ่นเก่า

### การรองรับสภาพแวดล้อมพร็อกซี

การถอดเสียงผ่านผู้ให้บริการรองรับตัวแปรสภาพแวดล้อมพร็อกซีขาออกมาตรฐาน:

- `HTTPS_PROXY`
- `HTTP_PROXY`
- `ALL_PROXY`
- `https_proxy`
- `http_proxy`
- `all_proxy`

หากไม่ได้ตั้งค่าตัวแปรสภาพแวดล้อมพร็อกซี จะใช้การออกสู่ภายนอกโดยตรง หากการกำหนดค่าพร็อกซีมีรูปแบบไม่ถูกต้อง OpenClaw จะบันทึกคำเตือนและถอยกลับไปใช้การดึงข้อมูลโดยตรง

## การตรวจจับการกล่าวถึงในกลุ่ม

เมื่อมีการตั้งค่า `requireMention: true` สำหรับแชตกลุ่ม ตอนนี้ OpenClaw จะถอดเสียง **ก่อน** ตรวจสอบการกล่าวถึง สิ่งนี้ช่วยให้ประมวลผลข้อความเสียงได้แม้จะมีการกล่าวถึงอยู่ภายใน

**วิธีการทำงาน:**

1. หากข้อความเสียงไม่มีเนื้อความแบบข้อความ และกลุ่มต้องการการกล่าวถึง OpenClaw จะทำการถอดเสียงแบบ "preflight"
2. ข้อความถอดเสียงจะถูกตรวจหารูปแบบการกล่าวถึง (เช่น `@BotName`, ทริกเกอร์อีโมจิ)
3. หากพบการกล่าวถึง ข้อความจะดำเนินต่อผ่านไปป์ไลน์การตอบกลับแบบเต็ม
4. ข้อความถอดเสียงถูกใช้สำหรับการตรวจจับการกล่าวถึง เพื่อให้ข้อความเสียงผ่านประตูการกล่าวถึงได้

**พฤติกรรมสำรอง:**

- หากการถอดเสียงล้มเหลวระหว่าง preflight (หมดเวลา ข้อผิดพลาด API เป็นต้น) ข้อความจะถูกประมวลผลตามการตรวจจับการกล่าวถึงจากข้อความเท่านั้น
- สิ่งนี้ช่วยให้มั่นใจว่าข้อความแบบผสม (ข้อความ + เสียง) จะไม่ถูกทิ้งอย่างไม่ถูกต้อง

**เลือกไม่ใช้ต่อกลุ่ม/หัวข้อ Telegram:**

- ตั้งค่า `channels.telegram.groups.<chatId>.disableAudioPreflight: true` เพื่อข้ามการตรวจสอบการกล่าวถึงจากข้อความถอดเสียง preflight สำหรับกลุ่มนั้น
- ตั้งค่า `channels.telegram.groups.<chatId>.topics.<threadId>.disableAudioPreflight` เพื่อแทนที่ต่อหัวข้อ (`true` เพื่อข้าม, `false` เพื่อบังคับเปิดใช้)
- ค่าเริ่มต้นคือ `false` (เปิดใช้ preflight เมื่อเงื่อนไขที่ต้องผ่านการกล่าวถึงตรงกัน)

**ตัวอย่าง:** ผู้ใช้ส่งข้อความเสียงที่พูดว่า "Hey @Claude, what's the weather?" ในกลุ่ม Telegram ที่มี `requireMention: true` ข้อความเสียงจะถูกถอดเสียง ตรวจพบการกล่าวถึง และเอเจนต์จะตอบกลับ

## ข้อควรระวัง

- กฎขอบเขตใช้หลักการรายการแรกที่ตรงกันเป็นฝ่ายชนะ `chatType` จะถูกปรับให้เป็น `direct`, `group` หรือ `room`
- ตรวจสอบให้แน่ใจว่า CLI ของคุณออกด้วยรหัส 0 และพิมพ์ข้อความธรรมดา; JSON ต้องถูกปรับแต่งผ่าน `jq -r .text`
- สำหรับ `parakeet-mlx` หากคุณส่ง `--output-dir` OpenClaw จะอ่าน `<output-dir>/<media-basename>.txt` เมื่อ `--output-format` เป็น `txt` (หรือไม่ได้ระบุ); รูปแบบผลลัพธ์ที่ไม่ใช่ `txt` จะถอยกลับไปใช้การแยกวิเคราะห์ stdout
- ตั้งค่าการหมดเวลาให้สมเหตุสมผล (`timeoutSeconds`, ค่าเริ่มต้น 60 วินาที) เพื่อหลีกเลี่ยงการบล็อกคิวตอบกลับ
- การถอดเสียง preflight จะประมวลผลเฉพาะไฟล์แนบเสียง **แรก** สำหรับการตรวจจับการกล่าวถึง เสียงเพิ่มเติมจะถูกประมวลผลระหว่างเฟสความเข้าใจสื่อหลัก
