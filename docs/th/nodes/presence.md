สถานะการใช้งานคอมพิวเตอร์บอก Gateway ว่า Node macOS ที่เชื่อมต่ออยู่เครื่องใดได้รับ
อินพุตจริงจากเมาส์หรือแป้นพิมพ์ล่าสุด OpenClaw ใช้สัญญาณดังกล่าวเพื่อ
กำหนด Mac เครื่องหนึ่งเป็น `active` ให้คำใบ้เกี่ยวกับ Node ที่ใช้งานอยู่ซึ่งคงที่แก่เอเจนต์ และส่ง
การแจ้งเตือนการเชื่อมต่อของ Node ไปยังคอมพิวเตอร์ที่มีแนวโน้มว่าคุณกำลังใช้งานอยู่มากที่สุด

ส่วนนี้แยกจาก [สถานะระบบ](/th/concepts/presence) ซึ่งเป็นรายการแบบเรียลไทม์
ของไคลเอนต์ Gateway และแยกจากบีคอน `node.presence.alive` แบบถาวร ซึ่ง
บันทึกว่า Node มือถือปลุกขึ้นมาครั้งล่าสุดเมื่อใดโดยไม่ถือว่าเชื่อมต่ออยู่

## ข้อกำหนด

- แอป OpenClaw สำหรับ macOS จับคู่และเชื่อมต่อในโหมด Node แล้ว
- เปิดใช้งาน **Settings -> Permissions -> Active computer detection** โดยค่าเริ่มต้นจะปิดอยู่
- ให้สิทธิ์ **Accessibility** แก่แอป OpenClaw ที่ลงนามแล้ว
- สำหรับการแจ้งเตือนการเชื่อมต่อ ต้องให้สิทธิ์ **Notifications** ด้วย และ
  Node Mac ต้องเปิดเผย `system.notify`

ปัจจุบันการรายงานกิจกรรมทำงานผ่าน Node macOS แบบเนทีฟ ส่วน iOS,
Android, watchOS และโฮสต์ Node แบบไม่มีส่วนติดต่อสามารถรายงานสถานะการเชื่อมต่อหรือ
สถานะที่พบล่าสุดในเบื้องหลังได้ แต่จะไม่ถูกนำมาใช้แข่งขันเพื่อกำหนดคอมพิวเตอร์ที่ใช้งานอยู่

## ตรวจสอบคอมพิวเตอร์ที่ใช้งานอยู่

1. ในแอป macOS ให้เปิด **Settings -> Permissions** เปิดใช้งาน
   **Active computer detection** และให้สิทธิ์ **Accessibility** ในการตั้งค่าระบบของ macOS
2. ยืนยันว่า Node Mac เชื่อมต่ออยู่:

   ```bash
   openclaw nodes status --connected
   ```

3. เลื่อนเมาส์หรือกดปุ่มบน Mac เครื่องนั้น แล้วเรียกใช้:

   ```bash
   openclaw nodes status
   openclaw nodes describe --node <node-id-or-name>
   ```

Mac ที่มีสิทธิ์และมีกิจกรรมล่าสุดจะถูกทำเครื่องหมายเป็น `active` ผลลัพธ์สถานะจะแสดงระยะเวลา
นับจากอินพุตล่าสุด ส่วน `describe` จะแสดง `active`, `lastActiveAtMs` และ `presenceUpdatedAtMs`
ระบบจงใจรวมการรายงานกิจกรรม ดังนั้นหลังจากรายงานล่าสุด จอแสดงผลอาจใช้เวลาสูงสุดประมาณ 15
วินาทีจึงจะแสดงอินพุตครั้งถัดไป

## วิธีที่กิจกรรมกลายเป็นสถานะการใช้งาน

ตัวรายงาน macOS จะสุ่มตรวจนาฬิกาเวลาว่างของระบบ HID ทุกสองวินาที โดย
รายงานหนึ่งครั้งเมื่อการเชื่อมต่อ Node พร้อมใช้งาน จากนั้นจะรายงานกิจกรรมจริงที่ใหม่กว่า
ไม่เกินหนึ่งครั้งต่อทุก 15 วินาที ระหว่างที่ไม่มีการใช้งาน ระบบจะส่งสัญญาณคงการเชื่อมต่อ
ทุกสามนาที ระยะเวลาว่างถูกจำกัดไว้ที่ 30 วัน เพื่อไม่ให้ตัวอย่างที่เก่ามาก
เลื่อนไปข้างหน้าและกลายเป็นคอมพิวเตอร์ล่าสุดอย่างไม่ถูกต้อง

การปิด **Active computer detection** จะหยุดการสุ่มตรวจและส่งเหตุการณ์ล้างข้อมูลที่ผ่านการตรวจสอบสิทธิ์
ผ่านการเชื่อมต่อ Node ปัจจุบัน Gateway จะลบ
การประทับเวลากิจกรรมที่เก็บไว้ของ Mac เครื่องนั้นทันทีและคำนวณคอมพิวเตอร์ที่ใช้งานอยู่ใหม่
ความสามารถอื่นของ Node และงานที่กำลังดำเนินการจะยังคงเชื่อมต่ออยู่ หาก Gateway ที่เชื่อมต่อ
เป็นเวอร์ชันก่อนรองรับการดำเนินการล้างข้อมูลนี้ Node Mac จะเชื่อมต่อใหม่หนึ่งครั้งเพื่อให้การล้างข้อมูล
เมื่อยกเลิกการเชื่อมต่อสามารถลบกิจกรรมที่เก็บไว้ได้แทน

Gateway จะยอมรับกิจกรรมต่อเมื่อเงื่อนไขทั้งหมดต่อไปนี้เป็นจริง:

- เหตุการณ์เป็นของการเชื่อมต่อปัจจุบันที่ผ่านการตรวจสอบสิทธิ์สำหรับรหัส Node นั้น
- Node มีสิทธิ์ `accessibility: true` ที่มีผล
- เพย์โหลดมีค่า `idleSeconds` ซึ่งเป็นจำนวนเต็มที่อยู่ภายในขอบเขต

Gateway จะลบ `idleSeconds` ออกจากเวลาที่ตนเองสังเกตการณ์เพื่อหา
`lastActiveAtMs` โดยจะไม่เชื่อถือการประทับเวลาจากนาฬิกาจริงที่ Node ส่งมา ในบรรดา
Mac ที่เชื่อมต่อและมีสิทธิ์ เครื่องที่มี `lastActiveAtMs` ใหม่ที่สุดจะชนะ หากเสมอกันจะใช้
การอัปเดตสถานะการใช้งานล่าสุด

สถานะการใช้งานเป็นข้อมูลเฉพาะโปรเซสและผูกกับการเชื่อมต่อ การยกเลิกการเชื่อมต่อ
เซสชันปัจจุบัน การแทนที่ด้วยเซสชันอื่นที่ใช้รหัส Node เดียวกัน หรือการเพิกถอน
สิทธิ์ Accessibility จะล้างสถานะกิจกรรมของ Node นั้นและคำนวณ Mac ที่ใช้งานอยู่ใหม่

## ความเป็นส่วนตัวและบริบทของโมเดล

การแชร์กิจกรรมจะปิดอยู่โดยค่าเริ่มต้นและแยกจากการให้สิทธิ์ Accessibility
ที่ใช้สำหรับการทำงานอัตโนมัติของ UI OpenClaw ส่งเฉพาะระยะเวลาว่าง ไม่ส่งเนื้อหาของอินพุต โดยจะไม่ส่งค่าปุ่ม
พิกัดเมาส์ ชื่อแอปพลิเคชัน ชื่อหน้าต่าง หรือเหตุการณ์อินพุตดิบ ตัวรายงาน
macOS อ่านสถานะ HID ของฮาร์ดแวร์ ดังนั้นเหตุการณ์ควบคุมคอมพิวเตอร์ที่จำลองขึ้น
จะไม่ทำให้ Mac ที่ทำงานอัตโนมัติดูเหมือนเป็นคอมพิวเตอร์ที่คุณใช้งานจริง

กิจกรรมต่อเนื่องจะไม่สร้างเหตุการณ์ระบบที่ส่งให้โมเดล บรรทัดรันไทม์
แบบไดนามิกจะมีเฉพาะรหัส Node ที่ผ่านการตรวจสอบสิทธิ์:

```text
active_node=<node-id>
```

การประทับเวลาที่แน่นอนและชื่อที่แสดงซึ่ง Node ควบคุมจะไม่ถูกใส่ในพรอมป์ เพื่อ
หลีกเลี่ยงการแทรกคำสั่งในพรอมป์และการเปลี่ยนแปลงแคชโดยไม่จำเป็น เมื่อเอเจนต์ต้องการรายละเอียดปัจจุบัน
เครื่องมือ `nodes` สามารถอ่าน `node.list` หรือ `node.describe` แทนได้

## วิธีส่งการแจ้งเตือนการเชื่อมต่อ

หลังจาก Node จับมือกับ Gateway สำเร็จเป็นครั้งแรกหลังได้รับการอนุมัติ
OpenClaw จะรอ 750 มิลลิวินาทีเพื่อให้ Mac ที่กำลังเชื่อมต่อส่งตัวอย่าง
กิจกรรมครั้งแรก จากนั้นจะลองส่งไปยัง Mac ที่เชื่อมต่ออยู่ รองรับการแจ้งเตือน และมี
กิจกรรมล่าสุด

- หากส่งผ่านช่องทางหลักสำเร็จ Mac เครื่องอื่นจะไม่ได้รับการแจ้งเตือน
- หากไม่มี Mac ที่ใช้งานอยู่หรือการส่งผ่านช่องทางหลักล้มเหลว OpenClaw จะรอห้า
  วินาที แล้วลองส่งไปยัง Mac ที่เชื่อมต่อเครื่องอื่นทั้งหมดซึ่งเปิดเผย `system.notify`
- การเชื่อมต่อใหม่ครั้งถัดไปจะไม่มีการแจ้งเตือน Gateway จะบันทึกการเชื่อมต่อที่สำเร็จ
  ไว้ในข้อมูลเมตาการจับคู่ ดังนั้นการเริ่ม Gateway ใหม่จะไม่ส่งการแจ้งเตือนซ้ำให้กับ
  Node ทุกเครื่องที่เคยเชื่อมต่อ

การแจ้งเตือนผูกกับข้อมูลประจำตัวของ Node ที่ผ่านการตรวจสอบสิทธิ์ เซสชันที่มาแทน
สำหรับ Node เดียวกันจะรับช่วงการแจ้งเตือนการเชื่อมต่อครั้งแรกที่รอดำเนินการ หาก Node นั้น
ไม่ได้เชื่อมต่ออยู่แล้วเมื่อเริ่มส่ง การแจ้งเตือนจะถูกยกเลิก

| อาการ                                   | สิ่งที่ต้องตรวจสอบ                                                                                                                                                                |
| ----------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| Mac ผิดเครื่องยังคงเป็นเครื่องที่ใช้งานอยู่              | ใช้งาน Mac เครื่องนั้นจริง รอให้พ้นช่วงเวลารวมการรายงาน แล้วเรียกใช้ `openclaw nodes status` อีกครั้ง การควบคุมคอมพิวเตอร์แบบจำลองจะไม่นับเป็นกิจกรรม                        |

| การแจ้งเตือนปรากฏบน Mac หลายเครื่อง         | การส่งผ่านช่องทางหลักไม่พร้อมใช้งานหรือล้มเหลว จึงเรียกใช้กลไกสำรองแบบหน่วงเวลา ตรวจสอบว่า Mac ที่ใช้งานอยู่เชื่อมต่ออยู่ อนุญาตการแจ้งเตือน และเปิดเผย `system.notify` |
