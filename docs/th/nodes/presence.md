สถานะการใช้งานคอมพิวเตอร์ที่กำลังทำงานอยู่จะแจ้งให้ Gateway ทราบว่า Node macOS ที่เชื่อมต่ออยู่เครื่องใดได้รับ
อินพุตจากเมาส์หรือแป้นพิมพ์จริงล่าสุด OpenClaw ใช้สัญญาณนี้เพื่อ
ทำเครื่องหมาย Mac หนึ่งเครื่องเป็น `active` ให้คำใบ้เกี่ยวกับ Node ที่กำลังทำงานอยู่ซึ่งคงที่แก่เอเจนต์ และกำหนดเส้นทาง
การแจ้งเตือนการเชื่อมต่อ Node ไปยังคอมพิวเตอร์ที่มีแนวโน้มว่าคุณกำลังใช้งานอยู่มากที่สุด

ส่วนนี้แยกจาก [สถานะของระบบ](/th/concepts/presence) ซึ่งเป็นรายการแบบเรียลไทม์
ของไคลเอนต์ Gateway และแยกจากบีคอน `node.presence.alive` แบบคงทน ซึ่ง
บันทึกเวลาที่ Node อุปกรณ์เคลื่อนที่ตื่นขึ้นล่าสุดโดยไม่ถือว่าเชื่อมต่ออยู่

## ข้อกำหนด

- แอป OpenClaw สำหรับ macOS จับคู่และเชื่อมต่อในโหมด Node แล้ว
- ให้สิทธิ์ **Accessibility** แก่แอป OpenClaw ที่ลงนามแล้ว
- สำหรับการแจ้งเตือนการเชื่อมต่อ ต้องให้สิทธิ์ **Notifications** ด้วย และ
  Node Mac ต้องเปิดให้ใช้ `system.notify`

ขณะนี้การรายงานกิจกรรมรองรับโดย Node macOS แบบเนทีฟ iOS,
Android, watchOS และโฮสต์ Node แบบไม่มีส่วนติดต่อผู้ใช้สามารถรายงานสถานะการเชื่อมต่อหรือ
สถานะการพบเห็นล่าสุดในเบื้องหลังได้ แต่จะไม่เข้าร่วมการพิจารณาว่าคอมพิวเตอร์ใดกำลังทำงานอยู่

## ตรวจสอบคอมพิวเตอร์ที่กำลังทำงานอยู่

1. ในแอป macOS ให้เปิด **Settings -> Permissions** และให้สิทธิ์
   **Accessibility** ใน macOS System Settings
2. ยืนยันว่า Node Mac เชื่อมต่ออยู่:

   ```bash
   openclaw nodes status --connected
   ```

3. ขยับเมาส์หรือกดแป้นบน Mac เครื่องนั้น แล้วเรียกใช้:

   ```bash
   openclaw nodes status
   openclaw nodes describe --node <node-id-or-name>
   ```

Mac ที่เข้าเกณฑ์และมีกิจกรรมล่าสุดที่สุดจะถูกทำเครื่องหมายเป็น `active` เอาต์พุตสถานะจะแสดง
ระยะเวลาตั้งแต่อินพุตล่าสุด ส่วน `describe` จะแสดง `active`, `lastActiveAtMs` และ `presenceUpdatedAtMs`
กิจกรรมจะถูกรวมเข้าด้วยกันโดยเจตนา ดังนั้นหลังจากรายงานล่าสุดแล้ว การแสดงผลอาจใช้เวลาสูงสุดประมาณ 15
วินาทีจึงจะแสดงอินพุตครั้งถัดไป

## กิจกรรมกลายเป็นสถานะการใช้งานได้อย่างไร

ตัวรายงาน macOS จะสุ่มตรวจสอบนาฬิกาวัดเวลาว่างของระบบ HID ทุกสองวินาที โดย
รายงานหนึ่งครั้งเมื่อการเชื่อมต่อ Node พร้อมใช้งาน จากนั้นรายงานกิจกรรมจริงที่ใหม่กว่า
ไม่เกินหนึ่งครั้งในทุก 15 วินาที ขณะไม่มีการใช้งาน ระบบจะส่งสัญญาณคงการเชื่อมต่อ
ทุกสามนาที ระยะเวลาที่ไม่มีการใช้งานจำกัดไว้สูงสุด 30 วัน เพื่อไม่ให้ตัวอย่างที่เก่ามาก
เลื่อนไปข้างหน้าและกลายเป็นคอมพิวเตอร์ล่าสุดอย่างไม่ถูกต้อง

Gateway จะยอมรับกิจกรรมต่อเมื่อเงื่อนไขทั้งหมดต่อไปนี้เป็นจริง:

- เหตุการณ์เป็นของการเชื่อมต่อปัจจุบันที่ผ่านการยืนยันตัวตนสำหรับรหัส Node นั้น
- Node มีสิทธิ์ `accessibility: true` ที่มีผล
- เพย์โหลดมีค่า `idleSeconds` ซึ่งเป็นจำนวนเต็มที่อยู่ภายในขอบเขต

Gateway จะลบ `idleSeconds` ออกจากเวลาที่ตนเองสังเกตเห็นเพื่อคำนวณ
`lastActiveAtMs` โดยจะไม่เชื่อถือการประทับเวลาตามนาฬิการะบบที่ Node ส่งมา ในบรรดา
Mac ที่เชื่อมต่อและเข้าเกณฑ์ เครื่องที่มี `lastActiveAtMs` ใหม่ที่สุดจะได้รับเลือก หากเสมอกันจะใช้
การอัปเดตสถานะการใช้งานล่าสุด

สถานะการใช้งานเป็นข้อมูลภายในโปรเซสและผูกกับการเชื่อมต่อ การยกเลิกการเชื่อมต่อเซสชัน
ปัจจุบัน การแทนที่ด้วยเซสชันอื่นที่ใช้รหัส Node เดียวกัน หรือการเพิกถอน
Accessibility จะล้างสถานะกิจกรรมของ Node นั้นและคำนวณ Mac ที่กำลังทำงานอยู่อีกครั้ง

## ความเป็นส่วนตัวและบริบทของโมเดล

OpenClaw ส่งระยะเวลาที่ไม่มีการใช้งาน ไม่ใช่เนื้อหาอินพุต ระบบจะไม่ส่งค่าของแป้น
พิกัดเมาส์ ชื่อแอปพลิเคชัน ชื่อหน้าต่าง หรือเหตุการณ์อินพุตดิบ
ตัวรายงาน macOS อ่านสถานะ HID ของฮาร์ดแวร์ ดังนั้นเหตุการณ์ควบคุมคอมพิวเตอร์
ที่สังเคราะห์ขึ้นจะไม่ทำให้ Mac อัตโนมัติดูเหมือนเป็นคอมพิวเตอร์ที่คุณใช้งานจริง

กิจกรรมต่อเนื่องจะไม่สร้างเหตุการณ์ระบบที่โมเดลมองเห็น บรรทัดรันไทม์
แบบไดนามิกมีเพียงรหัส Node ที่ผ่านการยืนยันตัวตน:

```text
active_node=<node-id>
```

การประทับเวลาที่แน่นอนและชื่อที่แสดงซึ่ง Node ควบคุมจะไม่ถูกใส่ในพรอมต์ เพื่อ
หลีกเลี่ยงการแทรกพรอมต์และการเปลี่ยนแปลงแคชโดยไม่จำเป็น เมื่อเอเจนต์ต้องการรายละเอียดปัจจุบัน
เครื่องมือ `nodes` สามารถอ่าน `node.list` หรือ `node.describe` แทนได้

## วิธีกำหนดเส้นทางการแจ้งเตือนการเชื่อมต่อ

หลังจาก Node ดำเนินการแฮนด์เชกกับ Gateway เสร็จสิ้น OpenClaw จะรอ 750 มิลลิวินาทีเพื่อให้
Mac ที่กำลังเชื่อมต่อส่งตัวอย่างกิจกรรมแรก จากนั้นระบบจะลองส่งไปยัง
Mac ที่เชื่อมต่อ รองรับการแจ้งเตือน และมีกิจกรรมล่าสุดที่สุด

- หากการส่งหลักสำเร็จ Mac เครื่องอื่นจะไม่ได้รับการแจ้งเตือน
- หากไม่มี Mac ที่กำลังทำงานอยู่หรือการส่งหลักล้มเหลว OpenClaw จะรอห้า
  วินาที แล้วลองส่งไปยัง Mac ที่เชื่อมต่ออยู่เครื่องอื่นทั้งหมดซึ่งเปิดให้ใช้ `system.notify`
- การแจ้งเตือนการเชื่อมต่อใหม่สำหรับ Node เดิมจะถูกระงับไว้ห้านาทีหลังจาก
  มีความพยายามส่งจริง เพื่อป้องกันไม่ให้การเชื่อมต่อใหม่ที่สลับไปมาสร้าง
  การแจ้งเตือนจำนวนมาก

การแจ้งเตือนผูกกับการเชื่อมต่อ Node ที่แน่นอน เซสชันต้นทางที่ยกเลิกการเชื่อมต่อ
หรือถูกแทนที่จะไม่สามารถดำเนินการแจ้งเตือนเดิมที่กำหนดเวลาไว้ให้เสร็จสิ้น และการเชื่อมต่อปลายทาง
ที่เข้ามาแทนยังสามารถเข้าร่วมการส่งสำรองได้

| ----------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| Mac ผิดเครื่องยังคงทำงานอยู่              | ใช้งาน Mac เครื่องนั้นจริง รอให้พ้นช่วงเวลารวมกิจกรรม แล้วเรียกใช้ `openclaw nodes status` อีกครั้ง การควบคุมคอมพิวเตอร์ที่สังเคราะห์ขึ้นจะไม่นับเป็นกิจกรรม                        |
| ข้อมูลอินพุตล่าสุดหายไป                | ตรวจสอบว่า Mac ยกเลิกการเชื่อมต่อ เซสชัน Node ถูกแทนที่ หรือ Accessibility ถูกเพิกถอนหรือไม่ แต่ละเงื่อนไขจะล้างกิจกรรมโดยเจตนา                       |
| การแจ้งเตือนปรากฏบน Mac หลายเครื่อง         | การส่งหลักไม่พร้อมใช้งานหรือล้มเหลว จึงมีการส่งสำรองหลังการหน่วงเวลา ตรวจสอบว่า Mac ที่กำลังทำงานอยู่เชื่อมต่ออยู่ อนุญาตการแจ้งเตือน และเปิดให้ใช้ `system.notify` |
| เอเจนต์ไม่กล่าวถึง Mac ที่กำลังทำงานอยู่ | เริ่มเทิร์นใหม่หลังจากกิจกรรมเปลี่ยนแปลง คำใบ้รันไทม์มีความคงที่และกระชับ ใช้เครื่องมือ `nodes` เพื่อดูข้อมูลเมตาปัจจุบันที่แน่นอน                                    |
