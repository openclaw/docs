การมีคอมพิวเตอร์ที่ใช้งานอยู่จะบอก Gateway ว่า Node macOS ใดที่เชื่อมต่อได้รับ
อินพุตจริงจากเมาส์หรือแป้นพิมพ์ล่าสุด OpenClaw ใช้สัญญาณนี้เพื่อ
ทำเครื่องหมาย Mac เครื่องหนึ่งเป็น `active` ให้คำใบ้เกี่ยวกับ Node ที่ใช้งานอยู่ที่เสถียรแก่เอเจนต์ และกำหนดเส้นทาง
การแจ้งเตือนการเชื่อมต่อ Node ไปยังคอมพิวเตอร์ที่คุณน่าจะกำลังใช้งานอยู่มากที่สุด

สิ่งนี้แยกจาก[สถานะการออนไลน์ของระบบ](/th/concepts/presence) ซึ่งเป็นรายการแบบเรียลไทม์
ของไคลเอนต์ Gateway และแยกจากบีคอน `node.presence.alive` แบบคงอยู่ ซึ่ง
บันทึกว่า Node มือถือเริ่มทำงานล่าสุดเมื่อใดโดยไม่ถือว่าเชื่อมต่ออยู่

## ข้อกำหนด

- แอป OpenClaw สำหรับ macOS จับคู่และเชื่อมต่อในโหมด Node แล้ว
- ให้สิทธิ์ **Accessibility** แก่แอป OpenClaw ที่ลงนามแล้ว
- สำหรับการแจ้งเตือนการเชื่อมต่อ ต้องให้สิทธิ์ **Notifications** ด้วย และ
  Node Mac ต้องเปิดเผย `system.notify`

ขณะนี้การรายงานกิจกรรมรองรับโดย Node macOS แบบเนทีฟ iOS,
Android, watchOS และโฮสต์ Node แบบไม่มีส่วนติดต่อผู้ใช้สามารถรายงานการเชื่อมต่อหรือสถานะ
การพบเห็นล่าสุดในเบื้องหลังได้ แต่จะไม่แข่งขันเพื่อรับสถานะคอมพิวเตอร์ที่ใช้งานอยู่

## ตรวจสอบคอมพิวเตอร์ที่ใช้งานอยู่

1. ในแอป macOS ให้เปิด **Settings -> Permissions** แล้วให้สิทธิ์
   **Accessibility** ในการตั้งค่าระบบของ macOS
2. ยืนยันว่า Node Mac เชื่อมต่ออยู่:

   ```bash
   openclaw nodes status --connected
   ```

3. ขยับเมาส์หรือกดปุ่มบน Mac เครื่องนั้น แล้วเรียกใช้:

   ```bash
   openclaw nodes status
   openclaw nodes describe --node <node-id-or-name>
   ```

Mac ที่มีสิทธิ์และมีกิจกรรมล่าสุดที่สุดจะถูกทำเครื่องหมายเป็น `active` เอาต์พุตสถานะจะแสดงระยะเวลา
นับจากอินพุตล่าสุด ส่วน `describe` จะแสดง `active`, `lastActiveAtMs` และ `presenceUpdatedAtMs`
ระบบจงใจรวมการรายงานกิจกรรม ดังนั้นหลังจากรายงานล่าสุด การแสดงผลอาจใช้เวลาสูงสุดประมาณ 15
วินาทีจึงจะแสดงอินพุตครั้งถัดไป

## กิจกรรมเปลี่ยนเป็นสถานะการออนไลน์ได้อย่างไร

ตัวรายงานของ macOS จะสุ่มตรวจนาฬิกาเวลาว่างของระบบ HID ทุกสองวินาที โดย
จะรายงานหนึ่งครั้งเมื่อการเชื่อมต่อ Node พร้อมใช้งาน จากนั้นจะรายงานกิจกรรมทางกายภาพที่ใหม่กว่า
ไม่เกินหนึ่งครั้งในทุก 15 วินาที ขณะไม่มีการใช้งาน ระบบจะส่ง keepalive
ทุกสามนาที ระยะเวลาที่ไม่มีการใช้งานถูกจำกัดสูงสุดที่ 30 วัน เพื่อไม่ให้ตัวอย่างที่เก่ามาก
เลื่อนไปข้างหน้าและกลายเป็นคอมพิวเตอร์ล่าสุดอย่างไม่ถูกต้อง

Gateway จะยอมรับกิจกรรมเฉพาะเมื่อเงื่อนไขทั้งหมดต่อไปนี้เป็นจริง:

- เหตุการณ์เป็นของการเชื่อมต่อที่ผ่านการยืนยันตัวตนปัจจุบันสำหรับรหัส Node นั้น
- Node มีสิทธิ์ `accessibility: true` ที่มีผล
- เพย์โหลดมีค่า `idleSeconds` ซึ่งเป็นจำนวนเต็มที่อยู่ภายในขอบเขต

Gateway จะลบ `idleSeconds` ออกจากเวลาที่ตนเองสังเกตเห็นเพื่อคำนวณ
`lastActiveAtMs` โดยจะไม่เชื่อถือการประทับเวลาแบบนาฬิกาจริงที่ Node ส่งมา ในบรรดา
Mac ที่มีสิทธิ์และเชื่อมต่ออยู่ `lastActiveAtMs` ที่ใหม่ที่สุดจะเป็นผู้ชนะ หากเสมอกันจะใช้
การอัปเดตสถานะการออนไลน์ล่าสุด

สถานะการออนไลน์มีผลเฉพาะภายในโปรเซสและผูกกับการเชื่อมต่อ การยกเลิกการเชื่อมต่อ
เซสชันปัจจุบัน การแทนที่ด้วยเซสชันอื่นที่ใช้รหัส Node เดียวกัน หรือการเพิกถอน
Accessibility จะล้างสถานะกิจกรรมของ Node นั้นและคำนวณ Mac ที่ใช้งานอยู่อีกครั้ง

## ความเป็นส่วนตัวและบริบทของโมเดล

OpenClaw ส่งระยะเวลาที่ไม่มีการใช้งาน ไม่ใช่เนื้อหาของอินพุต โดยจะไม่ส่งค่าปุ่ม
พิกัดเมาส์ ชื่อแอปพลิเคชัน ชื่อหน้าต่าง หรือเหตุการณ์อินพุตดิบ
ตัวรายงานของ macOS อ่านสถานะ HID ของฮาร์ดแวร์ ดังนั้นเหตุการณ์ควบคุมคอมพิวเตอร์
ที่สังเคราะห์ขึ้นจะไม่ทำให้ Mac อัตโนมัติดูเหมือนเป็นคอมพิวเตอร์ที่คุณใช้งานจริง

กิจกรรมต่อเนื่องจะไม่สร้างเหตุการณ์ระบบที่โมเดลมองเห็น บรรทัดรันไทม์
แบบไดนามิกมีเพียงรหัส Node ที่ผ่านการยืนยันตัวตน:

```text
active_node=<node-id>
```

การประทับเวลาที่แน่นอนและชื่อที่แสดงซึ่ง Node ควบคุมจะไม่รวมอยู่ในพรอมต์ เพื่อ
หลีกเลี่ยงการแทรกพรอมต์และการเปลี่ยนแปลงแคช เมื่อเอเจนต์ต้องการรายละเอียดปัจจุบัน
เครื่องมือ `nodes` สามารถอ่าน `node.list` หรือ `node.describe` แทนได้

## วิธีกำหนดเส้นทางการแจ้งเตือนการเชื่อมต่อ

หลังจาก Node ทำแฮนด์เชกกับ Gateway สำเร็จเป็นครั้งแรกภายหลังการอนุมัติ
OpenClaw จะรอ 750 มิลลิวินาทีเพื่อให้ Mac ที่กำลังเชื่อมต่อส่งตัวอย่างกิจกรรมแรก
จากนั้นจะลองใช้ Mac ที่เชื่อมต่ออยู่ รองรับการแจ้งเตือน และมีกิจกรรมล่าสุดที่สุด

- หากการส่งหลักสำเร็จ Mac เครื่องอื่นจะไม่ได้รับการแจ้งเตือน
- หากไม่มี Mac ที่ใช้งานอยู่หรือการส่งหลักล้มเหลว OpenClaw จะรอห้า
  วินาที แล้วลองส่งไปยัง Mac ที่เชื่อมต่ออยู่ที่เหลือทุกเครื่องซึ่งเปิดเผย `system.notify`
- การเชื่อมต่อใหม่ครั้งต่อ ๆ ไปจะไม่มีการแจ้งเตือน Gateway จะบันทึกการเชื่อมต่อที่สำเร็จ
  ไว้ในข้อมูลเมตาการจับคู่ ดังนั้นการรีสตาร์ต Gateway จะไม่ส่งการแจ้งเตือนซ้ำสำหรับ
  Node ทุก Node ที่เคยเชื่อมต่อ

การแจ้งเตือนผูกกับอัตลักษณ์ Node ที่ผ่านการยืนยันตัวตน เซสชันทดแทนของ
Node เดียวกันจะรับช่วงการแจ้งเตือนการเชื่อมต่อครั้งแรกที่รอดำเนินการ หาก Node นั้น
ไม่ได้เชื่อมต่ออีกต่อไปเมื่อเริ่มส่ง การแจ้งเตือนจะถูกยกเลิก
