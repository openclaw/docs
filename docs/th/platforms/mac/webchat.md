แอปแถบเมนู macOS ฝัง UI ของ WebChat เป็นมุมมอง SwiftUI แบบเนทีฟ โดยเชื่อมต่อกับ Gateway และใช้เซสชันหลักของเอเจนต์ที่เลือกเป็นค่าเริ่มต้น (`main` หรือ `global` เมื่อ `session.scope` เป็น `global`)

หน้าต่างแชตแบบเต็มเป็นมุมมองแยกแบบเนทีฟ:

- **แถบด้านข้างเซสชัน**: รายการเซสชันที่ค้นหาได้ พร้อมส่วนที่ปักหมุด ส่วนกลุ่มที่รองรับโดย Gateway และส่วนล่าสุด เซสชันลูกที่สร้างขึ้นจะซ้อนอยู่ใต้เซสชันแม่ภายในแต่ละส่วน โดยเซสชันแม่ที่ยุบอยู่จะแสดงสรุปเซสชันสืบทอดที่กำลังทำงาน ล้มเหลว และยังไม่ได้อ่าน เมนูบริบทรองรับข้อมูลเซสชัน การเปลี่ยนชื่อ การปักหมุด การแยกแขนง การทำเครื่องหมายว่าอ่านแล้ว/ยังไม่ได้อ่าน การเก็บถาวร/กู้คืน การคัดลอกคีย์เซสชัน และการลบ การดำเนินการหลักสำหรับเซสชันใหม่ (หรือ Shift-Cmd-N) จะสร้างทันทีผ่าน `sessions.create` ส่วนป๊อปโอเวอร์ตัวเลือกที่อยู่ติดกันสามารถเลือกเอเจนต์และขอเวิร์กทรีที่มีการจัดการพร้อม ref ฐานซึ่งระบุหรือไม่ก็ได้
- **แถบเครื่องมือหน้าต่าง**: วงแหวนการใช้บริบท (โทเค็นและค่าใช้จ่ายของเซสชัน พร้อมการดำเนินการแบบย่อ) ตัวควบคุมโมเดล และเมนูการดำเนินการของเซสชัน โมเดลจะจัดกลุ่มตามผู้ให้บริการโดยวางผู้ให้บริการเริ่มต้นไว้ก่อน ขณะที่โมเดลที่ปักหมุดและใช้ล่าสุดยังคงอยู่ด้านบน ตัวควบคุมสามารถรับช่วงหรือเขียนทับระดับการคิดของโมเดล เลือกระดับรายละเอียดของการเรียกใช้เครื่องมือ และเปิดหรือปิดการตอบกลับแบบเร็ว เมนูสามารถเปลี่ยนชื่อหรือแยกแขนงเซสชันปัจจุบัน และอัปเดตสถานะการปักหมุด การอ่าน หรือการเก็บถาวร **เซสชัน…** (Shift-Cmd-S) เปิดตัวจัดการเซสชันที่ใช้งานอยู่/เก็บถาวรแล้วสำหรับการค้นหาผ่าน Gateway การจัดการกลุ่ม การตรวจสอบเซสชัน การเปลี่ยนชื่อ การปักหมุด การเก็บถาวร และการกู้คืน โหมดเลือกใช้การปักหมุด เลิกปักหมุด เก็บถาวร หรือลบกับเซสชันที่ใช้งานอยู่หลายรายการ โดยยังคงแสดงความล้มเหลวของแต่ละรายการ เครื่องหมายถูกในเมนูแยกต่างหากใช้แสดงหรือซ่อนการให้เหตุผลของผู้ช่วยและกิจกรรมของเครื่องมือ ทั้งสองรายการเปิดไว้โดยค่าเริ่มต้นและระบบจะจดจำไว้ข้ามการเปิดแอป
- **บทสนทนาและช่องเขียนข้อความ**: ข้อความของผู้ช่วยแสดงเป็นข้อความธรรมดาพร้อมอวตาร ส่วนข้อความของผู้ใช้แสดงเป็นกรอบข้อความสีเน้น คำถามจากเอเจนต์ที่รอดำเนินการแสดงเป็นการ์ดแบบเนทีฟ พร้อมตัวเลือกแบบเลือกได้หนึ่งรายการหรือหลายรายการ คำตอบข้อความอิสระ **อื่นๆ** การนับถอยหลังจนหมดอายุ และสถานะปลายทางร่วมกัน แชตว่างจะแสดงพรอมต์เริ่มต้นสำหรับเดสก์ท็อป การพิมพ์ `/` จะเปิดการเติมคำสั่งแบบทับให้อัตโนมัติซึ่งรองรับโดย `commands.list` พร้อมการนำทางด้วยแป้นลูกศร/Tab/Return/Escape คลิกขวาที่ข้อความเพื่อคัดลอก Markdown ที่มองเห็นได้โดยไม่รวมการให้เหตุผลที่ซ่อนอยู่ ข้อความของผู้ช่วยที่ถูกตัดทอนยังมีตัวเลือก **เปิดข้อความฉบับเต็ม** ซึ่งจะโหลดตัวอ่าน Markdown ที่เลือกข้อความได้ ใช้ **ฟัง** สำหรับ TTS ผ่าน Gateway โดยมีการพูดในเครื่องเป็นตัวเลือกสำรอง
- **ตัวควบคุมเสียง**: ช่องเขียนข้อความสามารถเริ่มหรือหยุดโหมดสนทนาที่มีอยู่ของ macOS โดยไม่แทนที่โอเวอร์เลย์บนแถบเมนู ขณะที่โหมดสนทนาทำงาน ช่องเขียนข้อความจะแสดงสถานะกำลังฟัง/กำลังคิด/กำลังพูด กิจกรรมเสียงแบบสด และบทถอดเสียงต่อเนื่องที่ขยายได้ คลิกขวาที่ปุ่มสนทนาเพื่อเลือก **ค่าเริ่มต้นของระบบ** หรือไมโครโฟนที่เชื่อมต่ออยู่ การเลือกไมโครโฟนนี้เป็นรายการเดียวกับที่ Voice Wake และการกดเพื่อพูดใช้ หากไมโครโฟนที่เลือกตัดการเชื่อมต่อ เซสชันสนทนาที่ทำงานอยู่จะเปลี่ยนไปใช้ค่าเริ่มต้นของระบบ และลองใช้ตัวเลือกนั้นอีกครั้งเมื่อเริ่มโหมดสนทนาครั้งถัดไป การดำเนินการไมโครโฟนแยกต่างหากจะบันทึกข้อความเสียงเมื่อโหมดสนทนาไม่ได้ควบคุมการบันทึกเสียง

แผงแชตขนาดกะทัดรัดที่ยึดกับแถบเมนูยังคงใช้เค้าโครงคอลัมน์เดียวแบบกะทัดรัด พร้อมตัวควบคุมโมเดล การคิด ระดับรายละเอียด และแบบเร็วในบรรทัด รวมถึงพรอมต์เริ่มต้น โหมดสนทนา ข้อความเสียง และฟัง การให้เหตุผลของผู้ช่วยและกิจกรรมของเครื่องมือยังคงถูกซ่อนในส่วนติดต่อแบบกะทัดรัดนี้

## หน้าต่าง Gateway หลายหน้าต่าง

เปิด **Settings → Gateways** เพื่อเพิ่มหรือลบโปรไฟล์ Gateway ที่นำกลับมาใช้ใหม่ได้ แต่ละ
โปรไฟล์ประกอบด้วยปลายทาง `ws://` หรือ `wss://` และโทเค็นหรือ
รหัสผ่านซึ่งระบุหรือไม่ก็ได้ ข้อมูลประจำตัวจะจัดเก็บไว้ในพวงกุญแจของ macOS การลบโปรไฟล์
จะปิดหน้าต่างที่เปิดอยู่ของโปรไฟล์นั้นและยุติการเชื่อมต่อรองด้วย

เลือก **File → New Gateway Window…** หรือกด Cmd-N จากนั้นเลือกหนึ่งใน
โปรไฟล์ที่บันทึกไว้ ตัวเลือกจะจดจำโปรไฟล์ที่ใช้ล่าสุด ทุกครั้งที่
เลือกจะสร้างหน้าต่างอิสระใหม่ ดังนั้น Gateway เดียวกันจึงปรากฏใน
หลายหน้าต่างโดยมีเซสชันที่ใช้งานอยู่และสถานะการนำทางต่างกันได้

แต่ละโปรไฟล์ที่บันทึกไว้เป็นเจ้าของการเชื่อมต่อ Gateway ร่วมหนึ่งรายการ ขอบเขตการยืนยันตัวตนของอุปกรณ์
แคชบทสนทนา กล่องขาออกออฟไลน์ และสัญญาเช่าเส้นทาง หน้าต่างของโปรไฟล์นั้น
ใช้ทรัพยากรเหล่านี้ร่วมกัน แต่ยังคงนำทางแยกจากกันได้ หน้าต่างของ
โปรไฟล์ต่างกันจะคงการเชื่อมต่อและเรียกใช้แชตพร้อมกัน

Gateway ที่กำหนดค่าไว้ของแอปแถบเมนูยังคงเป็นเจ้าของความสามารถของ Node บน Mac
และโหมดสนทนา หน้าต่าง Gateway เพิ่มเติมมีไว้สำหรับผู้ปฏิบัติงานเท่านั้น ดังนั้น
Gateway ที่สองจึงไม่สามารถเปลี่ยนเป้าหมายของไมโครโฟนส่วนกลางหรือตัวควบคุมอุปกรณ์โดยไม่แจ้งให้ทราบ
ฟัง/TTS และการดำเนินการแชตปกติใช้การเชื่อมต่อ Gateway ของหน้าต่างนั้นเอง

## แถบแชตด่วน

กด Option-Space (⌥Space) หรือเลือก **แชตด่วน** จากเมนูแถบเมนูเพื่อเปิดช่องเขียนข้อความแบบลอยสำหรับเซสชันหลัก เปลี่ยนแป้นพิมพ์ลัดส่วนกลางด้วยตัวบันทึกใน **การตั้งค่า → ทั่วไป → แป้นพิมพ์ลัดแชตด่วน**

แชตด่วนแสดงเอเจนต์เป้าหมาย (อวตารหรืออีโมจิ โดยใช้ชื่อเอเจนต์เป็นข้อความตัวอย่าง) และส่งไปยังเซสชันหลักของเอเจนต์นั้น หลังจาก Return ยอมรับการส่ง แถบจะยังคงเปิดอยู่และขยายลงด้านล่างพร้อมการตอบกลับ Markdown ที่สตรีมมาและบทสนทนาล่าสุด ช่องป้อนข้อมูลของแถบยังคงเป็นช่องเขียนข้อความ กด Command-Return เพื่อส่งและเปิดเป้าหมายเดียวกันในหน้าต่างแชตแบบเต็ม กด Shift-Return เพื่อขึ้นบรรทัดใหม่ หรือ Escape เพื่อปิดทั้งแถบและพื้นที่ตอบกลับ การคลิกด้านนอกจะปิดเช่นกัน เมื่อขาดสิทธิ์ macOS ที่เกี่ยวข้อง แถบแนบจะแสดงการดำเนินการ **อนุญาต** และ **ไว้ภายหลัง**

ใช้ปุ่มไมโครโฟนเพื่อป้อนข้อความตามคำบอกลงในช่องเขียนข้อความ ผลลัพธ์คำพูดบางส่วนจะแทนที่ช่วงข้อความที่ป้อนตามคำบอกแบบสด โดยคงข้อความที่มีอยู่แล้วในช่องเขียนข้อความไว้ กดปุ่มอีกครั้ง Return หรือ Escape เพื่อหยุด การส่ง การซ่อน หรือการย้ายโฟกัสออกจากแชตด่วนจะปล่อยไมโครโฟนเช่นกัน การใช้งานครั้งแรกจะขอสิทธิ์เข้าถึงไมโครโฟนและการรู้จำเสียงพูดของ macOS

ตัวควบคุมโมเดลแบบกะทัดรัดแสดงโมเดลและระดับการให้เหตุผลปัจจุบันของเซสชันเป้าหมาย การเลือกโมเดลจะอัปเดตเซสชันนั้นและจึงคงอยู่ในเซสชัน ส่วนการเลือกระดับการให้เหตุผลจะมีผลเฉพาะกับแต่ละข้อความที่ส่งจากการแสดงแชตด่วนปัจจุบัน ตัวเลือกในเครื่องจะรีเซ็ตเมื่อซ่อนแถบ การสลับเอเจนต์หรือเลือกเซสชันล่าสุดจะคงตัวเลือกที่ระบุไว้อย่างชัดเจน แต่โหลดสถานะโมเดลพื้นฐานของเซสชันเป้าหมายใหม่

คลิกปุ่มประวัติเพื่อเลือกจากห้าเซสชันที่อัปเดตล่าสุดหรือกลับไปยัง **ข้อความใหม่ถึง &lt;agent&gt;** การเลือกรายการล่าสุดจะส่งไปยังเซสชันนั้นโดยตรงและเปลี่ยนข้อความตัวอย่างเป็น **ตอบกลับใน &lt;session&gt;** การซ่อนแชตด่วนจะรีเซ็ตเป้าหมายชั่วคราวนี้เป็นเซสชันหลักของเอเจนต์ที่เลือก การสลับเอเจนต์จากเมนูอวตารจะล้างเป้าหมายนี้ด้วย

Command-Return เปิดการสนทนาของเอเจนต์ที่ได้รับการส่ง รวมถึงเมื่อขอบเขตเซสชันเป็นแบบส่วนกลาง

ปุ่มกล้องเปิดเมนูสำหรับ **จับภาพหน้าต่าง…** หรือ **จับภาพพื้นที่…** การจับภาพหน้าต่างจะติดป้ายกำกับทุกหน้าต่างที่มองเห็นได้ ส่วนการจับภาพพื้นที่จะทำให้แต่ละจอมืดลงขณะลากเลือกบริเวณและแสดงขนาดแบบสด ภาพหน้าจอที่เลือกจะส่งไปยังเอเจนต์ที่เลือกโดยใช้ข้อความที่พิมพ์ไว้เป็นคำบรรยาย การใช้งานครั้งแรกจะขอสิทธิ์เข้าถึงการบันทึกหน้าจอของ macOS กด Escape คลิกพื้นที่ว่าง หรือคลิกโดยไม่ได้ลากเลือกพื้นที่อย่างมีนัยสำคัญเพื่อยกเลิก

ใช้ปุ่มข้อความเอกสารเพื่อแนบข้อความจากหน้าต่างที่มีโฟกัสของแอปที่มีโฟกัส แชตด่วนจะแสดงผลลัพธ์เป็นชิปบริบทที่นำออกได้แทนการวางข้อความที่จับมาไว้ในช่องเขียนข้อความ เมื่อส่ง ระบบจะผนวกข้อความของชิปเข้ากับข้อความขาออกแล้วล้างชิป การดำเนินการนี้ต้องใช้สิทธิ์การช่วยการเข้าถึงของ macOS ข้อความที่แนบจะถูกล้างทุกครั้งที่ปิดแชตด่วนด้วย เพื่อไม่ให้บริบทจากการแสดงครั้งหนึ่งรั่วไหลไปยังการส่งครั้งถัดไป

หลังจากการตอบกลับเสร็จสิ้น ให้เลือก **วางไปยัง &lt;app&gt;** เพื่อคัดลอกข้อความผู้ช่วยที่มองเห็นได้ โดยไม่รวมการให้เหตุผลที่ซ่อนอยู่ ไปยังคลิปบอร์ดทั่วไป และวางลงในแอปที่อยู่ด้านหน้าสุด การดำเนินการนี้ต้องใช้สิทธิ์การช่วยการเข้าถึงของ macOS การดำเนินการจะแทนที่เนื้อหาปัจจุบันของคลิปบอร์ดแล้วซ่อนแชตด่วน

ปิดใช้คุณสมบัตินี้ทั้งหมดด้วย **การตั้งค่า → ทั่วไป → แชตด่วน** ส่วนเดียวกันนี้มีตัวบันทึกแป้นพิมพ์ลัดด้วย

- **โหมดภายในเครื่อง**: เชื่อมต่อโดยตรงกับ WebSocket ของ Gateway ภายในเครื่อง
- **โหมดระยะไกล**: ส่งต่อพอร์ตควบคุม Gateway ผ่าน SSH และใช้อุโมงค์นั้นเป็นระนาบข้อมูล

## การเปิดใช้และการดีบัก

- ด้วยตนเอง: เมนู Lobster -> "เปิดแชต"
- เปิดอัตโนมัติสำหรับการทดสอบ:

  ```bash
  dist/OpenClaw.app/Contents/MacOS/OpenClaw --chat
  ```

  (`--webchat` ยังคงยอมรับเป็นนามแฝงแบบเดิม)

- บันทึก: `./scripts/clawlog.sh` (ระบบย่อย `ai.openclaw` หมวดหมู่ `WebChatSwiftUI`)

## การเชื่อมโยงระบบ

- ระนาบข้อมูล: เมธอด Gateway WS `chat.history`, `chat.message.get`, `chat.send`, `chat.abort`, `chat.inject` รวมถึง `question.list` และ `question.resolve` และเหตุการณ์ `chat`, `agent`, `presence`, `tick`, `health`; การ์ดคำถามจะติดตามเหตุการณ์ `question.requested` และ `question.resolved` และรีเฟรชจาก `question.list` หลังจากเชื่อมต่อใหม่
- `chat.history` ส่งคืนบทสนทนาที่ปรับรูปแบบเพื่อการแสดงผลแล้ว: แท็กคำสั่งในบรรทัดจะถูกนำออกจากข้อความที่มองเห็นได้ เพย์โหลด XML สำหรับการเรียกใช้เครื่องมือที่เป็นข้อความธรรมดา (`<tool_call>`, `<function_call>`, `<tool_calls>`, `<function_calls>` รวมถึงบล็อกที่ถูกตัดทอน) และโทเค็นควบคุมโมเดลที่รั่วไหลจะถูกนำออก แถวผู้ช่วยที่มีเพียงโทเค็นเงียบ เช่น `NO_REPLY`/`no_reply` ที่ตรงกันทุกประการจะถูกละเว้น และแถวที่มีขนาดใหญ่เกินไปอาจถูกแทนที่ด้วยข้อความตัวอย่างที่ถูกตัดทอน
- เซสชัน: ใช้เซสชันหลักตามที่กล่าวข้างต้นเป็นค่าเริ่มต้น UI สามารถสลับระหว่างเซสชันได้
- กลุ่มเซสชัน: `sessions.groups.list`, `sessions.groups.put`, `sessions.groups.rename` และ `sessions.groups.delete` เป็นเจ้าของแค็ตตาล็อกกลุ่ม การเป็นสมาชิกคือ `category` ของเซสชัน ซึ่งอัปเดตผ่าน `sessions.patch`
- สถานะยังไม่ได้อ่าน: หลังจากเปิดใช้งานเซสชันและโหลดประวัติแบบสดสำเร็จ แอปจะล้างเครื่องหมายยังไม่ได้อ่านของเซสชันนั้น การโหลดประวัติที่ล้มเหลวจะไม่ล้างเครื่องหมายดังกล่าว ส่วนความล้มเหลวชั่วคราวของแพตช์จะลองใหม่ในการเปิดใช้งานครั้งถัดไป
- การเริ่มต้นใช้งานใช้เซสชันเฉพาะเพื่อแยกการตั้งค่าครั้งแรกออกจากกัน
- แคชออฟไลน์: แอปเก็บแคชแบบอ่านอย่างเดียวขนาดเล็กของเซสชันแชตและบทสนทนาล่าสุดต่อ Gateway (`~/Library/Application Support/OpenClaw/chat-cache.sqlite`): เมื่อเปิดแบบ cold open ระบบจะแสดงบทสนทนาที่ทราบล่าสุดทันทีและรีเฟรชเมื่อ Gateway ตอบสนอง ส่วนแชตล่าสุดยังคงเรียกดูได้ขณะตัดการเชื่อมต่อ (การส่งจะยังคงปิดใช้งานจนกว่าการเชื่อมต่อจะกลับมา)
