แอปแถบเมนู macOS ฝัง UI ของ WebChat เป็นมุมมอง SwiftUI แบบเนทีฟ โดยเชื่อมต่อกับ Gateway และใช้เซสชันหลักของเอเจนต์ที่เลือกเป็นค่าเริ่มต้น (`main` หรือ `global` เมื่อ `session.scope` เป็น `global`)

หน้าต่างแชตแบบเต็มเป็นมุมมองแยกแบบเนทีฟ:

- **แถบด้านข้างเซสชัน**: รายการเซสชันที่ค้นหาได้ พร้อมส่วนปักหมุด ส่วนกลุ่มที่รองรับโดย Gateway และส่วนล่าสุด เซสชันลูกที่สร้างขึ้นจะซ้อนอยู่ใต้เซสชันแม่ภายในแต่ละส่วน โดยเซสชันแม่ที่ยุบอยู่จะแสดงสรุปเซสชันสืบทอดที่กำลังทำงาน ล้มเหลว และยังไม่ได้อ่าน เมนูบริบทรองรับข้อมูลเซสชัน การเปลี่ยนชื่อ การปักหมุด การแยกเซสชัน การทำเครื่องหมายว่าอ่านแล้ว/ยังไม่ได้อ่าน การเก็บถาวร/กู้คืน การคัดลอกคีย์เซสชัน และการลบ การดำเนินการหลักสำหรับสร้างเซสชันใหม่ (หรือ Cmd-N) จะสร้างทันทีผ่าน `sessions.create`; ป๊อปโอเวอร์ตัวเลือกที่อยู่ติดกันสามารถเลือกเอเจนต์และขอ worktree ที่มีการจัดการพร้อม base ref ซึ่งระบุหรือไม่ก็ได้
- **แถบเครื่องมือหน้าต่าง**: วงแหวนการใช้บริบท (โทเค็นและค่าใช้จ่ายของเซสชัน พร้อมการดำเนินการแบบกะทัดรัด) ตัวควบคุมโมเดล และเมนูการดำเนินการของเซสชัน โมเดลจัดกลุ่มตามผู้ให้บริการโดยแสดงผู้ให้บริการเริ่มต้นก่อน ส่วนโมเดลที่ปักหมุดและใช้ล่าสุดจะยังคงอยู่ด้านบน ตัวควบคุมสามารถสืบทอดหรือแทนที่ระดับการคิดของโมเดล เลือกระดับความละเอียดของการเรียกใช้เครื่องมือ และเปิดหรือปิดการตอบกลับแบบ Fast ได้ เมนูสามารถเปลี่ยนชื่อหรือแยกเซสชันปัจจุบัน และอัปเดตสถานะการปักหมุด การอ่าน หรือการเก็บถาวรได้ **เซสชัน…** (Shift-Cmd-S) เปิดตัวจัดการ Active/Archived สำหรับการค้นหาผ่าน Gateway การจัดการกลุ่ม การตรวจสอบเซสชัน การเปลี่ยนชื่อ การปักหมุด การเก็บถาวร และการกู้คืน โหมดเลือกใช้การปักหมุด เลิกปักหมุด เก็บถาวร หรือลบกับเซสชันที่ใช้งานอยู่หลายรายการ พร้อมแสดงความล้มเหลวของแต่ละรายการแยกกัน เครื่องหมายถูกในเมนูที่แยกจากกันใช้แสดงหรือซ่อนกระบวนการให้เหตุผลของผู้ช่วยและกิจกรรมของเครื่องมือ ทั้งสองรายการเปิดอยู่ตามค่าเริ่มต้นและระบบจะจดจำค่าไว้เมื่อเปิดแอปครั้งต่อไป
- **บทสนทนาและช่องเขียนข้อความ**: ข้อความของผู้ช่วยแสดงเป็นข้อความธรรมดาพร้อมอวตาร ส่วนข้อความของผู้ใช้แสดงเป็นฟองสีเน้น คำถามจากเอเจนต์ที่รอดำเนินการจะแสดงเป็นการ์ดแบบเนทีฟ พร้อมตัวเลือกแบบเลือกได้หนึ่งรายการหรือหลายรายการ คำตอบแบบข้อความอิสระ **อื่นๆ** การนับถอยหลังก่อนหมดอายุ และสถานะปลายทางที่ใช้ร่วมกัน แชตว่างจะแสดงข้อความเริ่มต้นสำหรับเดสก์ท็อป การพิมพ์ `/` จะเปิดการเติมคำสั่ง slash อัตโนมัติที่รองรับโดย `commands.list` พร้อมการนำทางด้วยแป้นลูกศร/Tab/Return/Escape คลิกขวาที่ข้อความเพื่อคัดลอก Markdown ที่มองเห็นได้โดยไม่รวมกระบวนการให้เหตุผลที่ซ่อนอยู่ ข้อความผู้ช่วยที่ถูกตัดทอนยังมีตัวเลือก **เปิดข้อความฉบับเต็ม** ซึ่งจะโหลดตัวอ่าน Markdown ที่เลือกข้อความได้ ใช้ **ฟัง** สำหรับ TTS ผ่าน Gateway โดยมีการสังเคราะห์เสียงในเครื่องเป็นตัวสำรอง
- **ตัวควบคุมเสียง**: ช่องเขียนข้อความสามารถเริ่มหรือหยุดโหมดสนทนาที่มีอยู่ของ macOS โดยไม่แทนที่โอเวอร์เลย์บนแถบเมนู ขณะที่โหมดสนทนาทำงาน ช่องเขียนข้อความจะแสดงสถานะกำลังฟัง/กำลังคิด/กำลังพูด กิจกรรมเสียงแบบสด และบทถอดเสียงต่อเนื่องที่ขยายได้ คลิกขวาที่ปุ่ม Talk เพื่อเลือก **System Default** หรือไมโครโฟนที่เชื่อมต่ออยู่ ซึ่งเป็นการเลือกไมโครโฟนเดียวกับที่ Voice Wake และการกดเพื่อพูดใช้ หากไมโครโฟนที่เลือกตัดการเชื่อมต่อ เซสชัน Talk ที่ทำงานอยู่จะเปลี่ยนไปใช้ค่าเริ่มต้นของระบบ และลองใช้ตัวเลือกเดิมอีกครั้งเมื่อเริ่มโหมดสนทนาในครั้งถัดไป การดำเนินการไมโครโฟนแยกต่างหากจะบันทึกโน้ตเสียงเมื่อโหมดสนทนาไม่ได้ควบคุมการบันทึกเสียง

แผงแชตแบบกะทัดรัดที่ยึดกับแถบเมนูยังคงใช้เค้าโครงคอลัมน์เดียวแบบกะทัดรัด โดยแสดงตัวควบคุมโมเดล การคิด ระดับความละเอียด และ Fast แบบอินไลน์ พร้อมข้อความเริ่มต้น โหมดสนทนา โน้ตเสียง และฟังก์ชันฟัง กระบวนการให้เหตุผลของผู้ช่วยและกิจกรรมของเครื่องมือยังคงซ่อนอยู่ในพื้นผิวแบบกะทัดรัดนี้

## แถบแชตด่วน

กด Option-Space (⌥Space) หรือเลือก **แชตด่วน** จากเมนูแถบเมนูเพื่อเปิดช่องเขียนข้อความแบบลอยสำหรับเซสชันหลัก เปลี่ยนแป้นพิมพ์ลัดส่วนกลางด้วยตัวบันทึกใน **การตั้งค่า → ทั่วไป → แป้นพิมพ์ลัดแชตด่วน**

แชตด่วนจะแสดงเอเจนต์เป้าหมาย (อวตารหรืออีโมจิ โดยใช้ชื่อเอเจนต์เป็นข้อความตัวอย่าง) และส่งไปยังเซสชันหลักของเอเจนต์นั้น หลังจาก Return ยืนยันการส่ง แถบจะยังคงเปิดอยู่และขยายลงด้านล่างพร้อมการตอบกลับ Markdown แบบสตรีมและบทสนทนาล่าสุด ช่องป้อนข้อมูลของแถบยังคงเป็นช่องเขียนข้อความ กด Command-Return เพื่อส่งและเปิดเป้าหมายเดียวกันในหน้าต่างแชตแบบเต็ม กด Shift-Return เพื่อขึ้นบรรทัดใหม่ หรือกด Escape เพื่อปิดทั้งแถบและพื้นที่ตอบกลับ การคลิกด้านนอกจะปิดด้วยเช่นกัน เมื่อไม่มีสิทธิ์อนุญาตของ macOS ที่เกี่ยวข้อง แถบที่แนบมาจะแสดงการดำเนินการ **อนุญาต** และ **ยังไม่ใช่ตอนนี้**

ใช้ปุ่มไมโครโฟนเพื่อป้อนข้อความตามคำบอกลงในช่องเขียนข้อความ ผลลัพธ์เสียงพูดบางส่วนจะแทนที่ช่วงข้อความที่ป้อนตามคำบอกแบบสด โดยคงข้อความที่มีอยู่แล้วในช่องเขียนข้อความไว้ กดปุ่มอีกครั้ง กด Return หรือกด Escape เพื่อหยุด การส่ง การซ่อน หรือการยกเลิกโฟกัสแชตด่วนจะปล่อยไมโครโฟนด้วย การใช้งานครั้งแรกจะขอสิทธิ์เข้าถึง Microphone และ Speech Recognition ของ macOS

ตัวควบคุมโมเดลแบบกะทัดรัดจะแสดงโมเดลและระดับการให้เหตุผลปัจจุบันของเซสชันเป้าหมาย การเลือกโมเดลจะอัปเดตเซสชันนั้นและจึงคงอยู่ในเซสชันดังกล่าว ส่วนการเลือกระดับการให้เหตุผลจะมีผลเฉพาะกับแต่ละข้อความที่ส่งจากการแสดงแชตด่วนปัจจุบัน ตัวเลือกภายในเครื่องจะรีเซ็ตเมื่อซ่อนแถบ การสลับเอเจนต์หรือเลือกเซสชันล่าสุดจะคงตัวเลือกที่กำหนดไว้อย่างชัดเจน แต่โหลดสถานะโมเดลพื้นฐานของเซสชันเป้าหมายใหม่

คลิกปุ่มประวัติเพื่อเลือกจากเซสชันที่อัปเดตล่าสุดห้ารายการ หรือกลับไปที่ **ข้อความใหม่ถึง &lt;agent&gt;** การเลือกรายการล่าสุดจะส่งไปยังเซสชันนั้นโดยตรงและเปลี่ยนข้อความตัวอย่างเป็น **ตอบกลับใน &lt;session&gt;** การซ่อนแชตด่วนจะรีเซ็ตเป้าหมายชั่วคราวนี้เป็นเซสชันหลักของเอเจนต์ที่เลือก การสลับเอเจนต์จากเมนูอวตารจะล้างเป้าหมายนี้ด้วย

Command-Return เปิดการสนทนาของเอเจนต์ที่ได้รับข้อความ รวมถึงเมื่อขอบเขตเซสชันเป็นแบบส่วนกลาง

ปุ่มกล้องจะเปิดเมนูสำหรับ **จับภาพหน้าต่าง…** หรือ **จับภาพพื้นที่…** การจับภาพหน้าต่างจะติดป้ายทุกหน้าต่างที่มองเห็น ส่วนการจับภาพพื้นที่จะทำให้แต่ละจอมืดลงขณะลากเลือกบริเวณและแสดงขนาดแบบสด ภาพหน้าจอที่เลือกจะถูกส่งไปยังเอเจนต์ที่เลือก โดยใช้ข้อความที่พิมพ์ไว้เป็นคำบรรยาย การใช้งานครั้งแรกจะขอสิทธิ์ Screen Recording ของ macOS การกด Escape การคลิกพื้นที่ว่าง หรือการคลิกโดยไม่ได้ลากเลือกพื้นที่อย่างมีนัยสำคัญจะยกเลิกการทำงาน

ใช้ปุ่มข้อความเอกสารเพื่อแนบข้อความจากหน้าต่างที่โฟกัสอยู่ของแอปที่โฟกัสอยู่ แชตด่วนจะแสดงผลลัพธ์เป็นชิปบริบทที่นำออกได้แทนการวางข้อความที่จับภาพไว้ในช่องเขียนข้อความ เมื่อส่ง ระบบจะผนวกข้อความของชิปเข้ากับข้อความขาออกแล้วล้างชิปออก การดำเนินการนี้ต้องใช้สิทธิ์ Accessibility ของ macOS ข้อความที่แนบจะถูกล้างทุกครั้งที่แชตด่วนปิดด้วย เพื่อไม่ให้บริบทจากการแสดงครั้งหนึ่งรั่วไหลไปยังการส่งในภายหลัง

หลังการตอบกลับเสร็จสิ้น ให้เลือก **วางลงใน &lt;app&gt;** เพื่อคัดลอกข้อความผู้ช่วยที่มองเห็นได้ โดยไม่รวมกระบวนการให้เหตุผลที่ซ่อนอยู่ ไปยังคลิปบอร์ดทั่วไป แล้ววางลงในแอปที่อยู่ด้านหน้าสุด การดำเนินการนี้ต้องใช้สิทธิ์ Accessibility ของ macOS การดำเนินการนี้จะแทนที่เนื้อหาปัจจุบันของคลิปบอร์ดแล้วซ่อนแชตด่วน

ปิดใช้งานคุณสมบัตินี้ทั้งหมดด้วย **การตั้งค่า → ทั่วไป → แชตด่วน** ส่วนเดียวกันนี้มีตัวบันทึกแป้นพิมพ์ลัดด้วย

- **โหมดภายในเครื่อง**: เชื่อมต่อโดยตรงกับ WebSocket ของ Gateway ภายในเครื่อง
- **โหมดระยะไกล**: ส่งต่อพอร์ตควบคุมของ Gateway ผ่าน SSH และใช้อุโมงค์ดังกล่าวเป็นระนาบข้อมูล

## การเปิดใช้และการแก้ไขข้อบกพร่อง

- ด้วยตนเอง: เมนู Lobster -> "Open Chat"
- เปิดอัตโนมัติสำหรับการทดสอบ:

  ```bash
  dist/OpenClaw.app/Contents/MacOS/OpenClaw --chat
  ```

  (`--webchat` ยังคงใช้ได้ในฐานะนามแฝงแบบเดิม)

- บันทึก: `./scripts/clawlog.sh` (ระบบย่อย `ai.openclaw`, หมวดหมู่ `WebChatSwiftUI`)

## การเชื่อมต่อภายใน

- ระนาบข้อมูล: เมธอด Gateway WS ได้แก่ `chat.history`, `chat.message.get`, `chat.send`, `chat.abort`, `chat.inject` รวมถึง `question.list` และ `question.resolve` และเหตุการณ์ `chat`, `agent`, `presence`, `tick`, `health`; การ์ดคำถามจะติดตามเหตุการณ์ `question.requested` และ `question.resolved` และรีเฟรชจาก `question.list` หลังการเชื่อมต่อใหม่
- `chat.history` ส่งคืนบทสนทนาที่ปรับรูปแบบสำหรับการแสดงผลแล้ว: แท็กคำสั่งแบบอินไลน์จะถูกตัดออกจากข้อความที่มองเห็นได้ เพย์โหลด XML ของการเรียกใช้เครื่องมือแบบข้อความธรรมดา (`<tool_call>`, `<function_call>`, `<tool_calls>`, `<function_calls>` รวมถึงบล็อกที่ถูกตัดทอน) และโทเค็นควบคุมโมเดลที่รั่วไหลจะถูกตัดออก แถวผู้ช่วยที่มีเพียงโทเค็นเงียบ เช่น `NO_REPLY`/`no_reply` แบบตรงทั้งหมดจะถูกละเว้น และแถวที่มีขนาดใหญ่เกินไปอาจถูกแทนที่ด้วยข้อความตัวอย่างที่ถูกตัดทอน
- เซสชัน: ใช้เซสชันหลักตามที่กล่าวไว้ข้างต้นเป็นค่าเริ่มต้น โดย UI สามารถสลับระหว่างเซสชันได้
- กลุ่มเซสชัน: `sessions.groups.list`, `sessions.groups.put`, `sessions.groups.rename` และ `sessions.groups.delete` เป็นเจ้าของแค็ตตาล็อกกลุ่ม การเป็นสมาชิกคือ `category` ของเซสชัน ซึ่งอัปเดตผ่าน `sessions.patch`
- สถานะยังไม่ได้อ่าน: หลังจากเซสชันทำงานและโหลดประวัติแบบสดสำเร็จ แอปจะล้างเครื่องหมายยังไม่ได้อ่านของเซสชันนั้น การโหลดประวัติที่ล้มเหลวจะไม่ล้างเครื่องหมายนี้ ส่วนความล้มเหลวชั่วคราวในการใช้แพตช์จะลองใหม่ในการเปิดใช้งานครั้งถัดไป
- การเริ่มต้นใช้งานใช้เซสชันเฉพาะเพื่อแยกการตั้งค่าครั้งแรกออกจากกัน
- แคชออฟไลน์: แอปเก็บแคชแบบอ่านอย่างเดียวขนาดเล็กสำหรับเซสชันแชตและบทสนทนาล่าสุดแยกตาม Gateway (`~/Library/Application Support/OpenClaw/chat-cache.sqlite`): เมื่อเปิดแบบ cold open ระบบจะแสดงบทสนทนาล่าสุดที่ทราบทันทีและรีเฟรชเมื่อ Gateway ตอบกลับ ส่วนแชตล่าสุดยังสามารถเปิดดูได้ขณะตัดการเชื่อมต่อ (การส่งยังคงปิดใช้งานจนกว่าจะเชื่อมต่ออีกครั้ง)

## พื้นผิวด้านความปลอดภัย

- โหมดระยะไกลส่งต่อเฉพาะพอร์ตควบคุม WebSocket ของ Gateway ผ่าน SSH

## ข้อจำกัดที่ทราบ

- UI ได้รับการปรับให้เหมาะสำหรับเซสชันแชต ไม่ใช่ sandbox ของเบราว์เซอร์แบบเต็มรูปแบบ
