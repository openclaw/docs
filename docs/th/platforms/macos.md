แอป macOS คือ **คู่หูบนแถบเมนู** ของ OpenClaw: UI ถาดระบบแบบเนทีฟ การแจ้งขอสิทธิ์ของ macOS การแจ้งเตือน WebChat การป้อนข้อมูลด้วยเสียง Canvas และ
เครื่องมือ Node ที่โฮสต์บน Mac เช่น `system.run`

ต้องการเพียง CLI และ Gateway ใช่ไหม เริ่มต้นที่ [เริ่มต้นใช้งาน](/th/start/getting-started)

## ดาวน์โหลด

ดาวน์โหลดบิลด์แอป macOS จาก [รุ่นเผยแพร่ OpenClaw บน GitHub](https://github.com/openclaw/openclaw/releases)
เมื่อรุ่นเผยแพร่มีไฟล์แอป macOS ให้มองหา:

- `OpenClaw-<version>.dmg` (แนะนำ)
- `OpenClaw-<version>.zip`

บางรุ่นเผยแพร่มีเฉพาะ CLI หลักฐาน หรือไฟล์สำหรับ Windows หากรุ่นเผยแพร่ล่าสุด
ไม่มีไฟล์แอป macOS ให้ใช้รุ่นล่าสุดที่มี หรือบิลด์จากซอร์สด้วย
[การตั้งค่าสภาพแวดล้อมพัฒนา macOS](/th/platforms/mac/dev-setup)

## การเรียกใช้ครั้งแรก

1. ติดตั้งและเปิดใช้ **OpenClaw.app**
2. เลือก **This Mac** สำหรับ Gateway ภายในเครื่อง หรือเชื่อมต่อกับ Gateway ระยะไกล
3. รอระหว่างที่แอปติดตั้งรันไทม์ CLI เวอร์ชันที่ตรงกัน ในโหมดภายในเครื่อง แอปจะ
   ติดตั้งและเริ่ม Gateway ด้วย
4. สร้างการเชื่อมต่อสำหรับการอนุมานด้วยการตรวจสอบโมเดลจริง เมื่อผ่านแล้ว OpenClaw
   จะจัดการการตั้งค่าที่เหลือ
5. ทำรายการตรวจสอบสิทธิ์ macOS ให้เสร็จและส่งข้อความทดสอบการเริ่มต้นใช้งาน

หากแอปเข้าถึง Gateway ที่มีอยู่ซึ่งเอเจนต์เริ่มต้นมีโมเดลที่กำหนดค่าไว้
แอปจะถือว่า Gateway นั้นตั้งค่าแล้ว ข้ามการเริ่มต้นใช้งานผู้ให้บริการและ
OpenClaw แล้วเปิดแดชบอร์ด หากเชื่อมต่อ Gateway ไม่ได้หรือ
เอเจนต์เริ่มต้นไม่มีโมเดล การเริ่มต้นใช้งานการอนุมานจะยังคงพร้อมสำหรับ
การกู้คืน

สำหรับเส้นทางการตั้งค่า CLI/Gateway ให้ใช้ [เริ่มต้นใช้งาน](/th/start/getting-started)
สำหรับการกู้คืนสิทธิ์ ให้ใช้ [สิทธิ์ของ macOS](/th/platforms/mac/permissions)

## การอัปเดต

การ์ดอัปเดตบนแดชบอร์ดระบุสิ่งที่แอปจะอัปเดต:

- **อัปเดตแอป Mac + Gateway** หมายความว่าแอปที่ลงนามแล้วเป็นผู้ดูแล Gateway
  ภายในเครื่องที่ทำงานผ่าน launchd Sparkle จะอัปเดตแอปก่อน หลังเปิดใหม่ แอปจะ
  อัปเดตและเริ่ม Gateway ของแอปใหม่โดยอัตโนมัติเป็นเวอร์ชันที่ตรงกัน แล้วตรวจสอบ
  การเชื่อมต่อ
- **อัปเดต Gateway** หมายความว่าแอปเชื่อมต่ออยู่กับ Gateway ระยะไกล Gateway ภายในเครื่อง
  ที่จัดการด้วยตนเอง หรือการติดตั้งอื่นที่แอปไม่ได้เป็นผู้ดูแล ปุ่มนี้
  จะเรียกใช้ขั้นตอนการอัปเดตตามปกติของ Gateway นั้นแทนการเปลี่ยนแอป Mac

การอัปเดตแบบประสานงานที่ล้มเหลวจะคงอยู่ในหน้าต่างลักษณะการตั้งค่า พร้อมตัวเลือกลองใหม่
[คู่มือการอัปเดต](/th/install/updating) และการดำเนินการของ Discord การซ่อมแซมอัตโนมัติจะไม่
ดาวน์เกรด Gateway ที่ใหม่กว่า หรือแทนที่การปักหมุดช่องทาง `extended-stable`

หลังอัปเดตสำเร็จ แอปจะค้นหาเซสชันโดยตรงระดับบนสุด
ที่มนุษย์ใช้งานล่าสุด และส่งเหตุการณ์อัปเดตแบบครั้งเดียวให้เอเจนต์นั้น กิจกรรม Heartbeat
และ Cron ไม่มีผลต่อการเลือกนี้ จากนั้นเอเจนต์สามารถต้อนรับกลับมา
จากบทสนทนาที่น่าจะใช้งานล่าสุด ในโหมดระยะไกล แอปจะ
อัปเดตเฉพาะรันไทม์ Node ของ Mac ภายในเครื่อง และข้ามการแจ้งเตือนเมื่อ
Gateway ระยะไกลเก่ากว่าแอป

Sparkle ทำตามการตั้งค่า `update.channel` ของ Gateway ค่า `beta` และ `dev` จะเลือกรับ
บิลด์แอปเบตา ส่วน `stable`, `extended-stable` และค่าที่ไม่มีหรือไม่รู้จัก
จะคงใช้บิลด์แอปเสถียร

## เปิดลิงก์ในแดชบอร์ด

ในแดชบอร์ดแบบฝังของแอป macOS การคลิกลิงก์เว็บภายนอกจะเปิดลิงก์นั้นในแถบด้านข้างของเบราว์เซอร์ที่ปรับขนาดได้ โดยมีความกว้างครึ่งหนึ่งของหน้าต่างและยังคงแสดงการนำทางของแดชบอร์ด ลากเส้นแบ่งเพื่อเลือกความกว้างอื่น แอปจะจดจำค่านั้น แต่ละลิงก์จะเปิดในแท็บของตนเอง แถบแท็บจะปรากฏเมื่อเปิดหลายหน้า และการคลิกลิงก์เดิมอีกครั้งจะใช้แท็บที่มีอยู่ ลากแท็บเพื่อจัดลำดับใหม่ ปิดด้วยปุ่มปิดแท็บหรือคลิกกลาง และคลิกขวาที่แท็บเพื่อใช้ **Open in Default Browser**, **Copy Link**, **Reload**, **Close Tab** และ **Close Other Tabs** ปุ่มย้อนกลับ/ไปข้างหน้าบนแถบชื่อหน้าต่างและการปัดแทร็กแพดใช้เพื่อนำทางประวัติของแดชบอร์ด ส่วนปุ่มย้อนกลับ/ไปข้างหน้าของแถบด้านข้างใช้เพื่อนำทางประวัติของแท็บที่ใช้งานอยู่ แถบด้านข้างยังมีปุ่มโหลดใหม่ เปิดในเบราว์เซอร์เริ่มต้น และปิด

ปุ่มบนแถบชื่อหน้าต่างจะปรับตามแถบด้านข้างของแอป: เมื่อแถบด้านข้างขยายอยู่ ปุ่มย้อนกลับ/ไปข้างหน้าจะอยู่ที่ขอบขวาถัดจากปุ่มสลับแถบด้านข้าง เมื่อยุบอยู่ ปุ่มเหล่านี้จะเปิดพื้นที่ให้ปุ่มค้นหา (เปิดชุดคำสั่ง) และปุ่มเซสชันใหม่

คลิกขวาที่ลิงก์ภายนอกเพื่อเลือก **Open in Sidebar**, **Open in Default Browser** หรือ **Copy Link** การคลิกพร้อมปุ่มปรับแต่งและลิงก์เปิดหน้าต่างใหม่ที่ผู้ใช้เรียกจากแดชบอร์ดจะยังคงเปิดในเบราว์เซอร์เริ่มต้น ส่วนลิงก์เปิดหน้าต่างใหม่ภายในแถบด้านข้างจะเปิดเป็นแท็บใหม่ในแถบด้านข้าง หน้า Control UI ปกติที่โฮสต์ในเบราว์เซอร์จะยังคงใช้ลิงก์และเมนูบริบทตามปกติของเบราว์เซอร์

## นำเข้าการเข้าสู่ระบบของเบราว์เซอร์

ครั้งแรกที่เปิดแถบด้านข้างของเบราว์เซอร์ขณะแอปทำงานกับ Gateway ภายในเครื่อง แดชบอร์ดจะแสดงแบนเนอร์ที่ปิดได้เมื่อ Mac มีโปรไฟล์ตระกูล Chrome ที่มีคุกกี้ แบนเนอร์นี้เสนอให้คัดลอกคุกกี้เหล่านั้นไปยังโปรไฟล์ที่มีการจัดการแบบแยกส่วนซึ่งเอเจนต์ใช้ท่องเว็บ เลือกโปรไฟล์จากตัวควบคุม **Import** (อาจต้องใช้ Touch ID) ความคืบหน้าและจำนวนคุกกี้ที่นำเข้าจะแสดงในบรรทัด และจะคัดลอกเฉพาะคุกกี้เท่านั้น — รหัสผ่านจะไม่ออกจากเบราว์เซอร์ต้นทาง การปิดแบนเนอร์จะบันทึกตัวเลือกนี้ไว้ **Settings → General → Browser login → Import…** จะเสนอตัวเลือกนี้อีกครั้งได้ทุกเมื่อ ดูขั้นตอนการนำเข้าพื้นฐานและเงื่อนไข `browser.allowSystemProfileImport` ได้ที่ [เบราว์เซอร์](/th/cli/browser)

## เลือกโหมด Gateway

| โหมด   | ใช้เมื่อ                                                                    | หน้ารายละเอียด                                        |
| ------ | ------------------------------------------------------------------------------ | -------------------------------------------------- |
| ภายในเครื่อง  | Mac เครื่องนี้ควรเรียกใช้ Gateway และให้ launchd ดูแลให้ทำงานอยู่                | [Gateway บน macOS](/th/platforms/mac/bundled-gateway) |
| ระยะไกล | โฮสต์อื่นเรียกใช้ Gateway และ Mac เครื่องนี้ควบคุมผ่าน SSH, LAN หรือ Tailnet | [การควบคุมระยะไกล](/th/platforms/mac/remote)            |

ทั้งสองโหมดต้องติดตั้ง CLI `openclaw` เพราะแอปใช้รันไทม์โฮสต์ Node
ของ CLI ซ้ำ บน Mac เครื่องใหม่ แอปจะติดตั้ง CLI เวอร์ชันที่ตรงกันโดยอัตโนมัติ จากนั้นโหมดภายในเครื่อง
จะเริ่มตัวช่วยตั้งค่า Gateway ส่วนโหมดระยะไกลจะเชื่อมต่อกับ Gateway ที่เลือก
โดยไม่เริ่ม Gateway ภายในเครื่องตัวที่สอง
ดูการกู้คืนด้วยตนเองที่ [Gateway บน macOS](/th/platforms/mac/bundled-gateway)

## สิ่งที่แอปดูแล

- สถานะแถบเมนู การแจ้งเตือน สุขภาพ และ WebChat
- การแจ้งขอสิทธิ์ของ macOS สำหรับหน้าจอ ไมโครโฟน คำพูด ระบบอัตโนมัติ และการช่วยการเข้าถึง
- Node ของ Mac หนึ่งตัวที่รวม Canvas แบบเนทีฟ การจับภาพจากกล้อง/หน้าจอ การแจ้งเตือน
  ตำแหน่ง และการควบคุมคอมพิวเตอร์ เข้ากับคำสั่งระบบ เบราว์เซอร์
  Plugin Skills และ MCP ของโฮสต์ Node ของ CLI
- การแจ้งขออนุมัติการดำเนินการสำหรับคำสั่งที่โฮสต์บน Mac
- การดำเนินการในบริบทของแอปสำหรับคำสั่งเชลล์ที่อนุมัติ โดยคงการระบุแหล่งที่มาของ
  สิทธิ์ macOS ของแอปไว้ ขณะที่รันไทม์ CLI เป็นผู้ดูแลนโยบาย Node ที่ใช้ร่วมกัน
- อุโมงค์ SSH ในโหมดระยะไกลหรือการเชื่อมต่อ Gateway โดยตรง

แอป **ไม่ได้** แทนที่เอกสาร Gateway หรือ CLI ทั่วไป การกำหนดค่า Gateway
ผู้ให้บริการ Plugin ช่องทาง เครื่องมือ และความปลอดภัยมีเอกสาร
แยกของตนเอง
