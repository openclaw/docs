แอป macOS คือ **แอปคู่หูบนแถบเมนู** ของ OpenClaw ซึ่งมี UI ถาดระบบแบบเนทีฟ การแจ้งขอสิทธิ์ของ macOS การแจ้งเตือน WebChat การป้อนข้อมูลด้วยเสียง Canvas และเครื่องมือโหนดที่โฮสต์บน Mac เช่น `system.run`

ใช้ **Quick Chat** เพื่อเขียนข้อความในเซสชันหลักแบบ Spotlight โดยไม่ต้องเปิดหน้าต่างเต็ม กด Option-Space (⌥Space) ซึ่งเป็นค่าเริ่มต้น เลือกจากเมนูบนแถบเมนู หรือบันทึกแป้นพิมพ์ลัดอื่นใน **Settings → General**

ต้องการเพียง CLI และ Gateway ใช่ไหม เริ่มต้นที่ [เริ่มต้นใช้งาน](/th/start/getting-started)

## ดาวน์โหลด

ดาวน์โหลดบิลด์ของแอป macOS จาก [รุ่นเผยแพร่ OpenClaw บน GitHub](https://github.com/openclaw/openclaw/releases)
เมื่อรุ่นเผยแพร่มีแอสเซ็ตแอป macOS ให้มองหารายการต่อไปนี้:

- `OpenClaw-<version>.dmg` (แนะนำ)
- `OpenClaw-<version>.zip`

บางรุ่นเผยแพร่มีเฉพาะ CLI หลักฐาน หรือแอสเซ็ต Windows หากรุ่นเผยแพร่ล่าสุด
ไม่มีแอสเซ็ตแอป macOS ให้ใช้รุ่นล่าสุดที่มี หรือสร้างจากซอร์สด้วย
[การตั้งค่าสภาพแวดล้อมพัฒนา macOS](/th/platforms/mac/dev-setup)

## การเรียกใช้ครั้งแรก

1. ติดตั้งและเปิด **OpenClaw.app**
2. เลือก **This Mac** สำหรับ Gateway ในเครื่อง หรือเชื่อมต่อกับ Gateway ระยะไกล
3. รอระหว่างที่แอปติดตั้งรันไทม์ CLI เวอร์ชันที่ตรงกัน ในโหมดภายในเครื่อง แอปจะ
   ติดตั้งและเริ่ม Gateway ด้วย
4. สร้างการเชื่อมต่อสำหรับการอนุมานด้วยการตรวจสอบโมเดลจริง เมื่อผ่านแล้ว OpenClaw
   จะดำเนินการตั้งค่าที่เหลือ
5. ทำรายการตรวจสอบสิทธิ์ของ macOS ให้เสร็จและส่งข้อความทดสอบการเริ่มต้นใช้งาน

หากแอปเชื่อมถึง Gateway ที่มีอยู่ซึ่งเอเจนต์เริ่มต้นได้กำหนดค่า
โมเดลไว้แล้ว แอปจะถือว่า Gateway นั้นตั้งค่าเสร็จแล้ว ข้ามการเริ่มต้นใช้งานผู้ให้บริการและ
OpenClaw แล้วเปิดแดชบอร์ด หากเชื่อมต่อ Gateway ไม่ได้หรือ
เอเจนต์เริ่มต้นไม่มีโมเดล ขั้นตอนเริ่มต้นใช้งานการอนุมานจะยังคงพร้อมใช้สำหรับ
การกู้คืน

สำหรับเส้นทางการตั้งค่า CLI/Gateway ให้ใช้ [เริ่มต้นใช้งาน](/th/start/getting-started)
สำหรับการกู้คืนสิทธิ์ ให้ใช้ [สิทธิ์ของ macOS](/th/platforms/mac/permissions)

## การอัปเดต

การ์ดอัปเดตบนแดชบอร์ดระบุสิ่งที่แอปจะอัปเดต:

- **Update Mac app + Gateway** หมายความว่าแอปที่ลงนามแล้วเป็นเจ้าของ Gateway
  ภายในเครื่องที่ทำงานผ่าน launchd โดย Sparkle จะอัปเดตแอปก่อน หลังจากเปิดแอปใหม่ แอปจะ
  อัปเดตและรีสตาร์ต Gateway เป็นเวอร์ชันที่ตรงกันโดยอัตโนมัติ จากนั้นตรวจสอบ
  การเชื่อมต่อ
- **Update Gateway** หมายความว่าแอปเชื่อมต่อกับ Gateway ระยะไกล Gateway ภายในเครื่อง
  ที่จัดการด้วยตนเอง หรือการติดตั้งอื่นที่แอปไม่ได้เป็นเจ้าของ ปุ่มนี้
  จะเรียกใช้ขั้นตอนอัปเดตตามปกติของ Gateway นั้นแทนการเปลี่ยนแอป Mac

การอัปเดตแบบประสานงานที่ล้มเหลวจะคงอยู่ในหน้าต่างแบบการตั้งค่า พร้อมตัวเลือกลองใหม่
[คู่มือการอัปเดต](/th/install/updating) และการดำเนินการผ่าน Discord การซ่อมแซมอัตโนมัติจะไม่
ดาวน์เกรด Gateway ที่ใหม่กว่า หรือเขียนทับการตรึงช่องทาง `extended-stable`

หลังจากอัปเดตสำเร็จ แอปจะค้นหาเซสชันโดยตรงระดับบนสุด
ที่มนุษย์ใช้งานล่าสุด และส่งเหตุการณ์อัปเดตแบบครั้งเดียวให้เอเจนต์นั้น กิจกรรม Heartbeat
และ Cron ไม่มีผลต่อการเลือกนี้ จากนั้นเอเจนต์สามารถต้อนรับกลับมา
จากบทสนทนาที่มีแนวโน้มว่าใช้งานล่าสุด ในโหมดระยะไกล แอปจะ
อัปเดตเฉพาะรันไทม์โหนด Mac ภายในเครื่องและข้ามการแจ้งเตือนเมื่อ
Gateway ระยะไกลเก่ากว่าแอป

Sparkle ใช้การตั้งค่า `update.channel` ของ Gateway ค่า `beta` และ `dev` จะเลือกใช้
บิลด์แอปเบตา ส่วน `stable`, `extended-stable` และค่าที่ไม่มีหรือไม่รู้จัก
จะใช้บิลด์แอปเสถียรต่อไป

## การเปิดลิงก์ในแดชบอร์ด

ในแดชบอร์ดแบบฝังของแอป macOS เมื่อคลิกลิงก์เว็บภายนอก ลิงก์จะเปิดในแถบด้านข้างของเบราว์เซอร์ที่ปรับขนาดได้ โดยมีความกว้างครึ่งหนึ่งของหน้าต่างและยังคงแสดงการนำทางของแดชบอร์ด ลากตัวแบ่งเพื่อเลือกความกว้างอื่น แอปจะจดจำค่านี้ แต่ละลิงก์จะเปิดในแท็บของตนเอง แถบแท็บจะปรากฏเมื่อเปิดหลายหน้า และการคลิกลิงก์เดิมอีกครั้งจะใช้แท็บเดิม ลากแท็บเพื่อจัดลำดับใหม่ ปิดด้วยปุ่มปิดแท็บหรือคลิกกลาง และคลิกขวาที่แท็บเพื่อเลือก **Open in Default Browser**, **Copy Link**, **Reload**, **Close Tab** และ **Close Other Tabs** ปุ่มย้อนกลับ/ถัดไปบนแถบชื่อหน้าต่างและการปัดแทร็กแพดใช้เพื่อนำทางประวัติของแดชบอร์ด ส่วนปุ่มย้อนกลับ/ถัดไปของแถบด้านข้างใช้เพื่อนำทางประวัติของแท็บที่ใช้งานอยู่ แถบด้านข้างยังมีปุ่มโหลดใหม่ เปิดในเบราว์เซอร์เริ่มต้น และปิด

ปุ่มบนแถบชื่อหน้าต่างจะเปลี่ยนตามแถบด้านข้างของแอป: เมื่อแถบด้านข้างขยายอยู่ ปุ่มย้อนกลับ/ถัดไปจะอยู่ที่ขอบขวาถัดจากปุ่มสลับแถบด้านข้าง เมื่อยุบแถบด้านข้าง ปุ่มเหล่านี้จะหลีกทางให้ปุ่มค้นหา (เปิดพาเล็ตคำสั่ง) และปุ่มเซสชันใหม่

คลิกขวาที่ลิงก์ภายนอกเพื่อเลือก **Open in Sidebar**, **Open in Default Browser** หรือ **Copy Link** การคลิกพร้อมปุ่มปรับแต่งและลิงก์เปิดหน้าต่างใหม่ที่ผู้ใช้เรียกใช้จากแดชบอร์ดจะยังคงเปิดในเบราว์เซอร์เริ่มต้น ส่วนลิงก์เปิดหน้าต่างใหม่ภายในแถบด้านข้างจะเปิดเป็นแท็บใหม่ในแถบด้านข้าง หน้า Control UI ปกติที่โฮสต์ในเบราว์เซอร์จะยังคงใช้พฤติกรรมลิงก์และเมนูบริบทตามปกติของเบราว์เซอร์

## นำเข้าข้อมูลเข้าสู่ระบบของเบราว์เซอร์

ครั้งแรกที่เปิดแถบด้านข้างของเบราว์เซอร์ขณะแอปทำงานกับ Gateway ภายในเครื่อง แดชบอร์ดจะแสดงแบนเนอร์ที่ปิดได้เมื่อมีโปรไฟล์ตระกูล Chrome พร้อมคุกกี้อยู่บน Mac แบนเนอร์จะเสนอให้คัดลอกคุกกี้เหล่านั้นไปยังโปรไฟล์ที่มีการจัดการและแยกออกจากกัน ซึ่งเอเจนต์ใช้สำหรับท่องเว็บ เลือกโปรไฟล์จากตัวควบคุม **Import** (อาจต้องใช้ Touch ID) ความคืบหน้าและจำนวนคุกกี้ที่นำเข้าจะแสดงในบรรทัดเดียวกัน และจะคัดลอกเฉพาะคุกกี้เท่านั้น — รหัสผ่านจะไม่ออกจากเบราว์เซอร์ต้นทาง การปิดแบนเนอร์จะบันทึกตัวเลือกไว้ และ **Settings → General → Browser login → Import…** จะแสดงตัวเลือกนี้อีกครั้งได้ทุกเมื่อ ดูขั้นตอนการนำเข้าพื้นฐานและเกต `browser.allowSystemProfileImport` ได้ที่ [เบราว์เซอร์](/th/cli/browser)

## เลือกโหมด Gateway

| โหมด   | ใช้เมื่อ                                                                    | หน้ารายละเอียด                                        |
| ------ | ------------------------------------------------------------------------------ | -------------------------------------------------- |
| ภายในเครื่อง  | Mac เครื่องนี้ควรเรียกใช้ Gateway และทำให้ทำงานต่อเนื่องด้วย launchd                | [Gateway บน macOS](/th/platforms/mac/bundled-gateway) |
| ระยะไกล | โฮสต์อื่นเรียกใช้ Gateway ส่วน Mac เครื่องนี้ควบคุมผ่าน SSH, LAN หรือ Tailnet | [การควบคุมระยะไกล](/th/platforms/mac/remote)            |

ทั้งสองโหมดต้องติดตั้ง CLI `openclaw` เนื่องจากแอปใช้รันไทม์โฮสต์โหนด
ของ CLI ซ้ำ บน Mac เครื่องใหม่ แอปจะติดตั้ง CLI เวอร์ชันที่ตรงกันโดยอัตโนมัติ จากนั้นโหมด
ภายในเครื่องจะเริ่มตัวช่วยสร้าง Gateway ส่วนโหมดระยะไกลจะเชื่อมต่อกับ Gateway ที่เลือก
โดยไม่เริ่ม Gateway ภายในเครื่องตัวที่สอง
ดูวิธีกู้คืนด้วยตนเองที่ [Gateway บน macOS](/th/platforms/mac/bundled-gateway)

## สิ่งที่แอปเป็นเจ้าของ

- สถานะบนแถบเมนู การแจ้งเตือน สถานะการทำงาน WebChat และแถบ Quick Chat แบบลอย
- การแจ้งขอสิทธิ์ของ macOS สำหรับหน้าจอ ไมโครโฟน เสียงพูด การทำงานอัตโนมัติ และการช่วยการเข้าถึง
- โหนด Mac หนึ่งโหนดที่รวม Canvas แบบเนทีฟ การจับภาพกล้อง/หน้าจอ การแจ้งเตือน
  ตำแหน่ง และการควบคุมคอมพิวเตอร์ เข้ากับคำสั่งระบบ เบราว์เซอร์
  Plugin, Skills และ MCP ของโฮสต์โหนด CLI
- การแจ้งขออนุมัติการดำเนินการสำหรับคำสั่งที่โฮสต์บน Mac
- การดำเนินการในบริบทของแอปสำหรับคำสั่งเชลล์ที่ได้รับอนุมัติ โดยคงการระบุแหล่งที่มาของ
  สิทธิ์ macOS ของแอป ขณะที่รันไทม์ CLI เป็นเจ้าของนโยบายโหนดที่ใช้ร่วมกัน
- อุโมงค์ SSH ในโหมดระยะไกลหรือการเชื่อมต่อ Gateway โดยตรง

ใน Control UI แบบฝัง **Settings → Notifications** จะแสดงสิทธิ์
การแจ้งเตือนแบบเนทีฟของแอปแทนการแจ้งเตือนแบบพุชของเบราว์เซอร์ เนื่องจากแอปส่งการแจ้งเตือนแบบเนทีฟ

แอป **ไม่ได้** แทนที่เอกสาร Gateway หรือ CLI ทั่วไป การกำหนดค่า Gateway
ผู้ให้บริการ Plugin ช่องทาง เครื่องมือ และความปลอดภัยมีเอกสาร
แยกของตนเอง
