# การแก้ไขปัญหา

## `clawhub login` เปิดเบราว์เซอร์แต่ไม่ดำเนินการจนเสร็จสมบูรณ์

CLI จะเริ่มเซิร์ฟเวอร์ callback ภายในที่มีอายุสั้นระหว่างการเข้าสู่ระบบผ่านเบราว์เซอร์

- ตรวจสอบให้แน่ใจว่าเบราว์เซอร์สามารถเข้าถึง `http://127.0.0.1:<port>/callback` ได้
- ตรวจสอบกฎของไฟร์วอลล์ภายใน, VPN และพร็อกซี หาก callback ไม่มาถึง
- ในสภาพแวดล้อมแบบ headless ให้สร้างโทเค็น API ในเว็บ UI ของ ClawHub แล้วเรียกใช้:

```bash
clawhub login --token clh_...
```

## `whoami` หรือ `publish` ส่งคืน `Unauthorized` (401)

- เข้าสู่ระบบอีกครั้งด้วย `clawhub login`
- หากใช้พาธการกำหนดค่าแบบกำหนดเอง ให้ยืนยันว่า `CLAWHUB_CONFIG_PATH` ชี้ไปยัง
  ไฟล์ที่มีโทเค็นปัจจุบัน
- หากใช้โทเค็น API ให้ยืนยันว่าโทเค็นนั้นไม่ได้ถูกเพิกถอนในเว็บ UI

## การค้นหาหรือการติดตั้งส่งคืน `Rate limit exceeded` (429)

อ่านข้อมูลการลองใหม่ในผลตอบกลับ:

- `Retry-After`: จำนวนวินาทีที่ต้องรอก่อนลองใหม่
- `RateLimit-Limit`: ขีดจำกัดที่ใช้กับคำขอนี้
- `RateLimit-Remaining`: โควตาคงเหลือที่แน่นอนเมื่อมีส่วนหัวนี้ สำหรับ `429` ค่านี้คือ `0`
- `RateLimit-Reset` หรือ `X-RateLimit-Reset`: เวลาที่รีเซ็ต

หากผู้ใช้หลายคนใช้ IP ขาออกร่วมกัน อาจถึงขีดจำกัด IP สำหรับผู้ใช้ที่ไม่ระบุตัวตนได้ แม้ว่าแต่ละ
คนจะส่งคำขอเพียงไม่กี่รายการ ให้เข้าสู่ระบบเมื่อทำได้และลองใหม่หลังจาก
ระยะเวลาหน่วงที่รายงาน

## การค้นหาหรือการติดตั้งล้มเหลวเมื่ออยู่หลังพร็อกซี

CLI รองรับตัวแปรพร็อกซีมาตรฐาน:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

ชื่อที่รองรับ ได้แก่ `HTTPS_PROXY`, `HTTP_PROXY`, `https_proxy` และ
`http_proxy`

## Skills ไม่ปรากฏในผลการค้นหา

- ตรวจสอบ slug ที่ตรงกันหรือหน้าของเจ้าของหากทราบ
- ยืนยันว่ารีลีสเป็นสาธารณะและไม่ได้ถูกระงับไว้เนื่องจากการสแกนหรือการกลั่นกรอง
- หากเป็นเจ้าของ Skills ให้เข้าสู่ระบบและตรวจสอบ:

```bash
clawhub inspect @openclaw/demo
```

ข้อมูลวินิจฉัยที่เจ้าของมองเห็นอาจอธิบายสถานะการสแกน ด่านการอัปโหลด หรือการกลั่นกรองได้

## การเผยแพร่ล้มเหลวเนื่องจากไม่มีข้อมูลเมตาที่จำเป็น

สำหรับ Skills ให้ตรวจสอบ frontmatter ของ `SKILL.md` ควรประกาศตัวแปรสภาพแวดล้อมและ
เครื่องมือที่จำเป็น เพื่อให้ผู้ใช้และตัวสแกนเข้าใจแพ็กเกจได้

สำหรับ Plugin ให้ตรวจสอบข้อมูลเมตาความเข้ากันได้ของ `package.json` การเผยแพร่
Plugin โค้ดต้องมีฟิลด์ความเข้ากันได้กับ OpenClaw เช่น `openclaw.compat.pluginApi` และ
`openclaw.build.openclawVersion`

ดูตัวอย่างเพย์โหลดการเผยแพร่ก่อน:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## การเผยแพร่ล้มเหลวเนื่องจากข้อผิดพลาดเกี่ยวกับเจ้าของหรือแหล่งที่มาของ GitHub

ClawHub ใช้อัตลักษณ์ GitHub และการระบุแหล่งที่มาเพื่อเชื่อมโยงแพ็กเกจกับ
ผู้เผยแพร่

- ตรวจสอบให้แน่ใจว่าเข้าสู่ระบบด้วยบัญชี GitHub ที่เป็นเจ้าของหรือสามารถเผยแพร่
  แพ็กเกจได้
- ตรวจสอบว่า URL แหล่งที่มาเป็นสาธารณะหรือ ClawHub สามารถเข้าถึงได้
- สำหรับแหล่งที่มาจาก GitHub ให้ใช้ `owner/repo`, `owner/repo@ref` หรือ URL GitHub แบบเต็ม

## การเผยแพร่ล้มเหลวเนื่องจากเนมสเปซถูกอ้างสิทธิ์หรือสงวนไว้

หากการเผยแพร่ล้มเหลวเนื่องจากแฮนเดิลเจ้าของ เนมสเปซขององค์กร ขอบเขตแพ็กเกจ slug ของ Skills
หรือชื่อแพ็กเกจถูกอ้างสิทธิ์หรือสงวนไว้แล้ว ก่อนอื่นให้ยืนยันว่า
กำลังเผยแพร่ด้วยเจ้าของที่ตรงกับเนมสเปซ สำหรับแพ็กเกจ Plugin
ชื่อที่มีขอบเขต เช่น `@example-org/example-plugin` ต้องเผยแพร่ในฐานะ
เจ้าของ `example-org` ที่ตรงกัน

หากเชื่อว่าองค์กร โปรเจกต์ หรือแบรนด์ของตนเป็นเจ้าของเนมสเปซโดยชอบธรรม แต่
ไม่สามารถจัดการเจ้าของ ClawHub ปัจจุบันได้ ให้เปิด
[ปัญหาการอ้างสิทธิ์องค์กร/เนมสเปซ](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
พร้อมหลักฐานสาธารณะที่ไม่ละเอียดอ่อน ดู
[การอ้างสิทธิ์องค์กรและเนมสเปซ](/th/clawhub/namespace-claims) สำหรับคำแนะนำด้านหลักฐานและสิ่งที่
ไม่ควรใส่ไว้ในปัญหาสาธารณะ

## `sync` แจ้งว่าไม่พบ Skills

`sync` จะค้นหาโฟลเดอร์ที่มี `SKILL.md` หรือ `skill.md`

ระบุรูทที่ต้องการสแกน:

```bash
clawhub sync --root /path/to/skills
```

ดูตัวอย่างก่อนหากไม่แน่ใจว่าจะเผยแพร่สิ่งใด:

```bash
clawhub sync --all --dry-run --no-input
```

## `update` ปฏิเสธเนื่องจากมีการเปลี่ยนแปลงภายในเครื่อง

ไฟล์ภายในเครื่องไม่ตรงกับเวอร์ชันใดที่ ClawHub รู้จัก ให้เลือกหนึ่งตัวเลือก:

- เก็บการแก้ไขภายในเครื่องไว้และข้ามการอัปเดต
- เขียนทับด้วยเวอร์ชันที่เผยแพร่แล้ว:

```bash
clawhub update @openclaw/demo --force
```

- เผยแพร่สำเนาที่แก้ไขแล้วเป็น slug ใหม่หรือ fork

## การติดตั้ง Plugin ล้มเหลวใน OpenClaw

- ใช้แหล่งที่มาของ ClawHub อย่างชัดเจน:

```bash
openclaw plugins install clawhub:<package>
```

- ตรวจสอบสถานะการสแกนและข้อมูลเมตาความเข้ากันได้ในหน้ารายละเอียดแพ็กเกจ
- ยืนยันว่าเวอร์ชัน OpenClaw เป็นไปตามช่วงความเข้ากันได้ที่แพ็กเกจ
  ประกาศไว้
- หากแพ็กเกจถูกซ่อน ระงับ หรือบล็อก แพ็กเกจนั้นอาจติดตั้งไม่ได้จนกว่า
  เจ้าของจะแก้ไขปัญหา

## คำขอ API สาธารณะล้มเหลว

- ปฏิบัติตามส่วนหัวการลองใหม่ของ `429` และแคชผลตอบกลับรายการ/การค้นหาสาธารณะ
- เชื่อมโยงผู้ใช้กลับไปยังรายการ ClawHub มาตรฐาน
- อย่ามิเรอร์เนื้อหาที่ซ่อน เป็นส่วนตัว ถูกระงับ หรือถูกบล็อกโดยการกลั่นกรองออกนอก
  พื้นผิว API สาธารณะ

ดูรายละเอียดปลายทางได้ที่ [HTTP API](/clawhub/http-api)
