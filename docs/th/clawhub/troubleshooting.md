# การแก้ไขปัญหา

## `clawhub login` เปิดเบราว์เซอร์แต่ไม่เสร็จสมบูรณ์

CLI จะเริ่มเซิร์ฟเวอร์เรียกกลับภายในเครื่องที่มีอายุสั้นระหว่างการเข้าสู่ระบบผ่านเบราว์เซอร์

- ตรวจสอบให้แน่ใจว่าเบราว์เซอร์ของคุณสามารถเข้าถึง `http://127.0.0.1:<port>/callback`
- หากไม่มีคำขอเรียกกลับเข้ามา ให้ตรวจสอบกฎของไฟร์วอลล์ภายในเครื่อง, VPN และพร็อกซี
- ในสภาพแวดล้อมแบบไม่มีส่วนติดต่อผู้ใช้ ให้สร้างโทเค็น API ในเว็บ UI ของ ClawHub แล้วเรียกใช้:

```bash
clawhub login --token clh_...
```

## `whoami` หรือ `publish` ส่งคืน `Unauthorized` (401)

- ลงชื่อเข้าใช้อีกครั้งด้วย `clawhub login`
- หากคุณใช้พาธการกำหนดค่าที่กำหนดเอง ให้ยืนยันว่า `CLAWHUB_CONFIG_PATH` ชี้ไปยังไฟล์ที่มีโทเค็นปัจจุบันของคุณ
- หากคุณใช้โทเค็น API ให้ยืนยันว่าโทเค็นดังกล่าวไม่ได้ถูกเพิกถอนในเว็บ UI

## การค้นหาหรือการติดตั้งส่งคืน `Rate limit exceeded` (429)

อ่านข้อมูลการลองใหม่ในผลตอบกลับ:

- `Retry-After`: จำนวนวินาทีที่ต้องรอก่อนลองใหม่
- `RateLimit-Limit`: ขีดจำกัดที่ใช้กับคำขอนี้
- `RateLimit-Remaining`: โควตาที่เหลืออยู่ของคุณอย่างแม่นยำเมื่อมีส่วนหัวนี้ สำหรับ `429` ค่าจะเป็น `0`
- `RateLimit-Reset` หรือ `X-RateLimit-Reset`: เวลาที่จะรีเซ็ต

หากผู้ใช้หลายคนใช้ IP ขาออกเดียวกัน อาจถึงขีดจำกัด IP แบบไม่ระบุตัวตนได้ แม้แต่ละคนจะส่งคำขอเพียงไม่กี่ครั้ง ลงชื่อเข้าใช้เมื่อทำได้ และลองใหม่หลังจากระยะเวลาหน่วงที่รายงาน

## การค้นหาหรือการติดตั้งล้มเหลวเมื่ออยู่หลังพร็อกซี

CLI รองรับตัวแปรพร็อกซีมาตรฐาน:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

ชื่อที่รองรับ ได้แก่ `HTTPS_PROXY`, `HTTP_PROXY`, `https_proxy` และ `http_proxy`

## Skills ไม่ปรากฏในผลการค้นหา

- ตรวจสอบ slug ที่ตรงกันทุกประการหรือหน้าของเจ้าของ หากคุณทราบ
- ยืนยันว่ารีลีสเป็นสาธารณะและไม่ได้ถูกระงับโดยการสแกนหรือการตรวจสอบเนื้อหา
- หากคุณเป็นเจ้าของ Skills ให้ลงชื่อเข้าใช้และตรวจสอบ:

```bash
clawhub inspect @openclaw/demo
```

ข้อมูลวินิจฉัยที่เจ้าของมองเห็นอาจอธิบายสถานะการสแกน ด่านตรวจการอัปโหลด หรือการตรวจสอบเนื้อหา

## การเผยแพร่ล้มเหลวเนื่องจากไม่มีข้อมูลเมตาที่จำเป็น

สำหรับ Skills ให้ตรวจสอบ frontmatter ของ `SKILL.md` ควรประกาศตัวแปรสภาพแวดล้อมและเครื่องมือที่จำเป็น เพื่อให้ผู้ใช้และระบบสแกนเข้าใจแพ็กเกจได้

สำหรับ Plugin ให้ตรวจสอบข้อมูลเมตาด้านความเข้ากันได้ใน `package.json` การเผยแพร่ Plugin โค้ดจำเป็นต้องมีฟิลด์ความเข้ากันได้ของ OpenClaw เช่น `openclaw.compat.pluginApi` และ `openclaw.build.openclawVersion`

แสดงตัวอย่างเพย์โหลดสำหรับการเผยแพร่ก่อน:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## การเผยแพร่ล้มเหลวเนื่องจากข้อผิดพลาดเกี่ยวกับเจ้าของหรือแหล่งที่มาของ GitHub

ClawHub ใช้ข้อมูลประจำตัว GitHub และการระบุแหล่งที่มาเพื่อเชื่อมโยงแพ็กเกจกับผู้เผยแพร่

- ตรวจสอบให้แน่ใจว่าคุณลงชื่อเข้าใช้ด้วยบัญชี GitHub ที่เป็นเจ้าของหรือสามารถเผยแพร่แพ็กเกจได้
- ตรวจสอบว่า URL แหล่งที่มาเป็นสาธารณะหรือ ClawHub สามารถเข้าถึงได้
- สำหรับแหล่งที่มาจาก GitHub ให้ใช้ `owner/repo`, `owner/repo@ref` หรือ URL GitHub แบบเต็ม

## การเผยแพร่ล้มเหลวเนื่องจากเนมสเปซถูกอ้างสิทธิ์หรือสงวนไว้

หากการเผยแพร่ล้มเหลวเนื่องจากชื่อผู้ใช้ของเจ้าของ เนมสเปซขององค์กร ขอบเขตแพ็กเกจ slug ของ Skills หรือชื่อแพ็กเกจถูกอ้างสิทธิ์หรือสงวนไว้แล้ว ให้ยืนยันก่อนว่าคุณกำลังเผยแพร่ด้วยเจ้าของที่ตรงกับเนมสเปซ สำหรับแพ็กเกจ Plugin ชื่อที่มีขอบเขต เช่น `@example-org/example-plugin` ต้องเผยแพร่โดยใช้เจ้าของ `example-org` ที่ตรงกัน

หากคุณเชื่อว่าองค์กร โปรเจกต์ หรือแบรนด์ของคุณเป็นเจ้าของเนมสเปซโดยชอบธรรม แต่คุณไม่สามารถจัดการเจ้าของ ClawHub ปัจจุบันได้ ให้เปิด[ปัญหาการอ้างสิทธิ์องค์กร / เนมสเปซ](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)พร้อมหลักฐานสาธารณะที่ไม่มีข้อมูลละเอียดอ่อน ดูคำแนะนำเกี่ยวกับหลักฐานและสิ่งที่ไม่ควรใส่ในปัญหาสาธารณะที่[การอ้างสิทธิ์องค์กรและเนมสเปซ](/clawhub/namespace-claims)

## `sync` แจ้งว่าไม่พบ Skills

`sync` จะค้นหาโฟลเดอร์ที่มี `SKILL.md` หรือ `skill.md`

กำหนดให้ชี้ไปยังไดเรกทอรีรากที่คุณต้องการสแกน:

```bash
clawhub sync --root /path/to/skills
```

แสดงตัวอย่างก่อน หากคุณไม่แน่ใจว่าจะเผยแพร่อะไร:

```bash
clawhub sync --all --dry-run --no-input
```

## `update` ปฏิเสธเนื่องจากมีการเปลี่ยนแปลงภายในเครื่อง

ไฟล์ภายในเครื่องไม่ตรงกับเวอร์ชันใดที่ ClawHub รู้จัก เลือกหนึ่งตัวเลือก:

- เก็บการแก้ไขภายในเครื่องไว้และข้ามการอัปเดต
- เขียนทับด้วยเวอร์ชันที่เผยแพร่แล้ว:

```bash
clawhub update @openclaw/demo --force
```

- เผยแพร่สำเนาที่แก้ไขแล้วโดยใช้ slug ใหม่หรือเป็น fork

## การติดตั้ง Plugin ล้มเหลวใน OpenClaw

- ใช้แหล่งที่มา ClawHub อย่างชัดเจน:

```bash
openclaw plugins install clawhub:<package>
```

- ตรวจสอบหน้ารายละเอียดแพ็กเกจเพื่อดูสถานะการสแกนและข้อมูลเมตาด้านความเข้ากันได้
- ยืนยันว่าเวอร์ชัน OpenClaw ของคุณเป็นไปตามช่วงความเข้ากันได้ที่แพ็กเกจระบุ
- หากแพ็กเกจถูกซ่อน ระงับ หรือบล็อก อาจไม่สามารถติดตั้งได้จนกว่าเจ้าของจะแก้ไขปัญหา

## คำขอ API สาธารณะล้มเหลว

- ปฏิบัติตามส่วนหัวการลองใหม่ของ `429` และแคชผลตอบกลับรายการ/การค้นหาสาธารณะ
- เชื่อมโยงผู้ใช้กลับไปยังรายการ ClawHub หลัก
- อย่าทำสำเนาเนื้อหาที่ถูกซ่อน เป็นส่วนตัว ถูกระงับ หรือถูกบล็อกโดยการตรวจสอบเนื้อหาไว้นอกพื้นผิว API สาธารณะ

ดูรายละเอียดปลายทางที่ [HTTP API](/clawhub/http-api)
