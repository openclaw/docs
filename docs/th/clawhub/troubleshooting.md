# การแก้ไขปัญหา

## `clawhub login` เปิดเบราว์เซอร์แต่ไม่เสร็จสมบูรณ์

CLI จะเริ่มเซิร์ฟเวอร์ callback ภายในเครื่องที่มีอายุสั้นระหว่างการเข้าสู่ระบบผ่านเบราว์เซอร์

- ตรวจสอบว่าเบราว์เซอร์สามารถเข้าถึง `http://127.0.0.1:<port>/callback` ได้
- ตรวจสอบกฎของไฟร์วอลล์ภายในเครื่อง, VPN และพร็อกซี หาก callback ไม่มาถึง
- ในสภาพแวดล้อมแบบไม่มีหน้าจอ ให้สร้างโทเค็น API ใน UI เว็บของ ClawHub แล้วเรียกใช้:

```bash
clawhub login --token clh_...
```

## `whoami` หรือ `publish` ส่งคืน `Unauthorized` (401)

- ลงชื่อเข้าใช้อีกครั้งด้วย `clawhub login`
- หากใช้พาธการกำหนดค่าแบบกำหนดเอง ให้ยืนยันว่า `CLAWHUB_CONFIG_PATH` ชี้ไปยัง
  ไฟล์ที่มีโทเค็นปัจจุบันของคุณ
- หากใช้โทเค็น API ให้ยืนยันว่าโทเค็นนั้นไม่ได้ถูกเพิกถอนใน UI เว็บ

## การค้นหาหรือการติดตั้งส่งคืน `Rate limit exceeded` (429)

อ่านข้อมูลการลองใหม่ในผลตอบกลับ:

- `Retry-After`: จำนวนวินาทีที่ต้องรอก่อนลองใหม่
- `RateLimit-Limit`: ขีดจำกัดที่ใช้กับคำขอนี้
- `RateLimit-Remaining`: โควตาที่เหลืออยู่จริงเมื่อมีส่วนหัวนี้ สำหรับ `429` ค่าคือ `0`
- `RateLimit-Reset` หรือ `X-RateLimit-Reset`: เวลาที่รีเซ็ต

หากผู้ใช้จำนวนมากใช้ IP ขาออกร่วมกัน อาจถึงขีดจำกัด IP แบบไม่ระบุตัวตนได้ แม้ว่าแต่ละ
คนจะส่งคำขอเพียงไม่กี่รายการ ลงชื่อเข้าใช้เมื่อทำได้ แล้วลองใหม่หลังจาก
ระยะเวลาหน่วงที่รายงาน

## การค้นหาหรือการติดตั้งล้มเหลวเมื่ออยู่หลังพร็อกซี

CLI รองรับตัวแปรพร็อกซีมาตรฐาน:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

ชื่อที่รองรับได้แก่ `HTTPS_PROXY`, `HTTP_PROXY`, `https_proxy` และ
`http_proxy`

## Skills ไม่ปรากฏในผลการค้นหา

- ตรวจสอบ slug ที่ตรงกันหรือหน้าของเจ้าของ หากทราบข้อมูลดังกล่าว
- ยืนยันว่ารีลีสเป็นสาธารณะและไม่ได้ถูกระงับไว้เนื่องจากการสแกนหรือการกลั่นกรอง
- หากคุณเป็นเจ้าของ Skills ให้ลงชื่อเข้าใช้และตรวจสอบ:

```bash
clawhub inspect @openclaw/demo
```

การวินิจฉัยที่เจ้าของมองเห็นอาจอธิบายสถานะการสแกน ด่านการอัปโหลด หรือการกลั่นกรองได้

## การเผยแพร่ล้มเหลวเนื่องจากไม่มีข้อมูลเมตาที่จำเป็น

สำหรับ Skills ให้ตรวจสอบ frontmatter ของ `SKILL.md` ควรประกาศตัวแปรสภาพแวดล้อมและ
เครื่องมือที่จำเป็น เพื่อให้ผู้ใช้และระบบสแกนเข้าใจแพ็กเกจได้

สำหรับ plugins ให้ตรวจสอบข้อมูลเมตาความเข้ากันได้ของ `package.json` การเผยแพร่ code-plugin
ต้องมีฟิลด์ความเข้ากันได้ของ OpenClaw เช่น `openclaw.compat.pluginApi` และ
`openclaw.build.openclawVersion`

ดูตัวอย่าง payload การเผยแพร่ก่อน:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## การเผยแพร่ล้มเหลวเนื่องจากข้อผิดพลาดเกี่ยวกับเจ้าของหรือแหล่งที่มาของ GitHub

ClawHub ใช้ข้อมูลประจำตัวของ GitHub และการระบุแหล่งที่มาเพื่อเชื่อมโยงแพ็กเกจกับ
ผู้เผยแพร่

- ตรวจสอบว่าคุณลงชื่อเข้าใช้ด้วยบัญชี GitHub ที่เป็นเจ้าของหรือสามารถเผยแพร่
  แพ็กเกจได้
- ตรวจสอบว่า URL แหล่งที่มาเป็นสาธารณะหรือ ClawHub สามารถเข้าถึงได้
- สำหรับแหล่งที่มาจาก GitHub ให้ใช้ `owner/repo`, `owner/repo@ref` หรือ URL GitHub แบบเต็ม

## การเผยแพร่ล้มเหลวเนื่องจาก namespace ถูกอ้างสิทธิ์หรือสงวนไว้

หากการเผยแพร่ล้มเหลวเนื่องจากแฮนเดิลเจ้าของ, namespace ขององค์กร, ขอบเขตแพ็กเกจ, slug ของ Skills
หรือชื่อแพ็กเกจถูกอ้างสิทธิ์หรือสงวนไว้แล้ว ให้ยืนยันก่อนว่าคุณกำลัง
เผยแพร่ด้วยเจ้าของที่ตรงกับ namespace สำหรับแพ็กเกจ Plugin
ชื่อแบบมีขอบเขต เช่น `@example-org/example-plugin` ต้องเผยแพร่ภายใต้เจ้าของ
`example-org` ที่ตรงกัน

หากเชื่อว่าองค์กร โปรเจกต์ หรือแบรนด์ของคุณเป็นเจ้าของ namespace โดยชอบธรรม แต่
ไม่สามารถจัดการเจ้าของ ClawHub ปัจจุบันได้ ให้เปิด
[ปัญหาการอ้างสิทธิ์องค์กร / Namespace](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
พร้อมหลักฐานสาธารณะที่ไม่ละเอียดอ่อน ดู
[การอ้างสิทธิ์องค์กรและ Namespace](/clawhub/namespace-claims) สำหรับคำแนะนำเกี่ยวกับหลักฐานและสิ่งที่
ไม่ควรใส่ในปัญหาสาธารณะ

## `sync` แจ้งว่าไม่พบ Skills

`sync` จะค้นหาโฟลเดอร์ที่มี `SKILL.md` หรือ `skill.md`

ระบุรากที่ต้องการสแกน:

```bash
clawhub sync --root /path/to/skills
```

ดูตัวอย่างก่อนหากไม่แน่ใจว่าจะเผยแพร่สิ่งใด:

```bash
clawhub sync --all --dry-run --no-input
```

## `update` ปฏิเสธเนื่องจากมีการเปลี่ยนแปลงภายในเครื่อง

ไฟล์ภายในเครื่องไม่ตรงกับเวอร์ชันใดที่ ClawHub รู้จัก เลือกหนึ่งตัวเลือก:

- เก็บการแก้ไขภายในเครื่องไว้และข้ามการอัปเดต
- เขียนทับด้วยเวอร์ชันที่เผยแพร่:

```bash
clawhub update @openclaw/demo --force
```

- เผยแพร่สำเนาที่แก้ไขแล้วเป็น slug ใหม่หรือ fork

## การติดตั้ง Plugin ล้มเหลวใน OpenClaw

- ใช้แหล่งที่มาของ ClawHub แบบระบุชัดเจน:

```bash
openclaw plugins install clawhub:<package>
```

- ตรวจสอบสถานะการสแกนและข้อมูลเมตาความเข้ากันได้ในหน้ารายละเอียดแพ็กเกจ
- ยืนยันว่าเวอร์ชัน OpenClaw ของคุณเป็นไปตามช่วงความเข้ากันได้
  ที่แพ็กเกจระบุ
- หากแพ็กเกจถูกซ่อน ระงับ หรือบล็อก อาจไม่สามารถติดตั้งได้จนกว่า
  เจ้าของจะแก้ไขปัญหา

## คำขอ API สาธารณะล้มเหลว

- ปฏิบัติตามส่วนหัวการลองใหม่ `429` และแคชผลตอบกลับรายการ/การค้นหาสาธารณะ
- เชื่อมโยงผู้ใช้กลับไปยังรายการ ClawHub ที่เป็นข้อมูลหลัก
- อย่าทำสำเนาเนื้อหาที่ซ่อน เป็นส่วนตัว ถูกระงับ หรือถูกบล็อกโดยการกลั่นกรองนอก
  พื้นผิว API สาธารณะ

ดูรายละเอียด endpoint ที่ [HTTP API](/clawhub/http-api)
