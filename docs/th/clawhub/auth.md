# การยืนยันตัวตน

ClawHub ใช้ GitHub สำหรับการลงชื่อเข้าใช้ผ่านเว็บ ส่วน CLI ใช้โทเค็น API ของ ClawHub ที่สร้างผ่านบัญชีซึ่งลงชื่อเข้าใช้แล้ว

## การลงชื่อเข้าใช้ผ่านเว็บ

ใช้ GitHub เพื่อลงชื่อเข้าใช้ที่ [clawhub.ai](https://clawhub.ai)

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานจะไม่สามารถลงชื่อเข้าใช้ ClawHub ตามขั้นตอนปกติได้ หากหลังจากลงชื่อเข้าใช้แล้วระบบนำคุณกลับไปยังสถานะที่ยังไม่ได้เข้าสู่ระบบ บัญชีของคุณอาจมีสถานะไม่เป็นปกติ หากบัญชีของคุณถูกแบนหรือปิดใช้งาน และคุณเชื่อว่าเกิดจากความผิดพลาด ให้ใช้[แบบฟอร์มอุทธรณ์ของ ClawHub](https://appeals.openclaw.ai/)

## การเข้าสู่ระบบ CLI

ขั้นตอนการเข้าสู่ระบบ CLI เริ่มต้นจะเปิดเบราว์เซอร์ของคุณ:

```bash
clawhub login
clawhub whoami
```

สิ่งที่จะเกิดขึ้น:

1. CLI เริ่มเซิร์ฟเวอร์เรียกกลับชั่วคราวบน `127.0.0.1`
2. เบราว์เซอร์ของคุณเปิดหน้าลงชื่อเข้าใช้ ClawHub
3. หลังจากลงชื่อเข้าใช้ด้วย GitHub แล้ว ClawHub จะสร้างโทเค็น API
4. เบราว์เซอร์เปลี่ยนเส้นทางกลับไปยังการเรียกกลับภายในเครื่อง
5. CLI จัดเก็บโทเค็นไว้ในไฟล์การกำหนดค่า ClawHub ของคุณ

หากเบราว์เซอร์ไม่สามารถเข้าถึงการเรียกกลับภายในเครื่องได้เนื่องจากกฎของไฟร์วอลล์ VPN หรือพร็อกซี ให้ใช้ขั้นตอนโทเค็นแบบไม่มีส่วนติดต่อผู้ใช้

## การเข้าสู่ระบบแบบไม่มีส่วนติดต่อผู้ใช้

สร้างโทเค็นใน UI เว็บของ ClawHub แล้วส่งโทเค็นให้ CLI:

```bash
clawhub login --token clh_...
```

ใช้ขั้นตอนนี้สำหรับเซิร์ฟเวอร์ งาน CI หรือสภาพแวดล้อมที่ใช้งานผ่านเทอร์มินัลเท่านั้น

สำหรับเชลล์ระยะไกลที่คุณสามารถเปิดเบราว์เซอร์จากที่อื่นได้ ให้เรียกใช้:

```bash
clawhub login --device
```

CLI จะแสดงรหัสแบบใช้ครั้งเดียวและรอระหว่างที่คุณอนุญาตสิทธิ์ที่
`https://clawhub.ai/cli/device`

## การจัดเก็บโทเค็น

พาธการกำหนดค่าเริ่มต้น:

- macOS: `~/Library/Application Support/clawhub/config.json`
- Linux/XDG: `$XDG_CONFIG_HOME/clawhub/config.json` หรือ `~/.config/clawhub/config.json`
- Windows: `%APPDATA%\\clawhub\\config.json`

กำหนดพาธเองด้วย:

```bash
export CLAWHUB_CONFIG_PATH=/path/to/config.json
```

แสดงโทเค็นที่จัดเก็บไว้สำหรับการตั้งค่า CI ด้วย:

```bash
clawhub token
```

## การเพิกถอน

คุณสามารถเพิกถอนโทเค็น API ได้ใน UI เว็บของ ClawHub

โทเค็นที่ถูกเพิกถอน ไม่ถูกต้อง หรือไม่มีอยู่จะส่งคืน `401 Unauthorized` ให้ลงชื่อเข้าใช้อีกครั้งด้วย `clawhub login` หรือระบุโทเค็นใหม่ด้วย `clawhub login --token`

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานจะไม่สามารถใช้โทเค็น API ที่มีอยู่ต่อได้ หากบัญชีของคุณถูกแบนหรือปิดใช้งาน และคุณเชื่อว่าเกิดจากความผิดพลาด ให้ใช้[แบบฟอร์มอุทธรณ์ของ ClawHub](https://appeals.openclaw.ai/)
