# การยืนยันตัวตน

ClawHub ใช้ GitHub สำหรับการลงชื่อเข้าใช้บนเว็บ ส่วน CLI ใช้โทเค็น API ของ ClawHub ที่สร้าง
ผ่านบัญชีที่ลงชื่อเข้าใช้นั้น

## การลงชื่อเข้าใช้บนเว็บ

ใช้ GitHub เพื่อลงชื่อเข้าใช้ที่ [clawhub.ai](https://clawhub.ai)

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถลงชื่อเข้าใช้ ClawHub ตามปกติได้
หากการลงชื่อเข้าใช้นำคุณกลับไปยังสถานะที่ออกจากระบบแล้ว บัญชีของคุณอาจไม่มี
สถานะที่ดี

## การเข้าสู่ระบบ CLI

โฟลว์การเข้าสู่ระบบ CLI เริ่มต้นจะเปิดเบราว์เซอร์ของคุณ:

```bash
clawhub login
clawhub whoami
```

สิ่งที่เกิดขึ้น:

1. CLI เริ่มเซิร์ฟเวอร์คอลแบ็กชั่วคราวบน `127.0.0.1`
2. เบราว์เซอร์ของคุณเปิดหน้าลงชื่อเข้าใช้ ClawHub
3. หลังจากลงชื่อเข้าใช้ GitHub แล้ว ClawHub จะสร้างโทเค็น API
4. เบราว์เซอร์เปลี่ยนเส้นทางกลับไปยังคอลแบ็กในเครื่อง
5. CLI จัดเก็บโทเค็นไว้ในไฟล์การกำหนดค่า ClawHub ของคุณ

หากเบราว์เซอร์ของคุณเข้าถึงคอลแบ็กในเครื่องไม่ได้เนื่องจากกฎของไฟร์วอลล์, VPN หรือ
พร็อกซี ให้ใช้โฟลว์โทเค็นแบบไม่มีเบราว์เซอร์

## การเข้าสู่ระบบแบบไม่มีเบราว์เซอร์

สร้างโทเค็นใน UI เว็บของ ClawHub แล้วส่งต่อให้ CLI:

```bash
clawhub login --token clh_...
```

ใช้โฟลว์นี้สำหรับเซิร์ฟเวอร์, งาน CI หรือสภาพแวดล้อมที่มีเฉพาะเทอร์มินัล

สำหรับเชลล์ระยะไกลที่คุณสามารถเปิดเบราว์เซอร์จากที่อื่นได้ ให้เรียกใช้:

```bash
clawhub login --device
```

CLI จะแสดงรหัสใช้ครั้งเดียวและรอขณะที่คุณอนุญาตสิทธิ์ที่
`https://clawhub.ai/cli/device`

## การจัดเก็บโทเค็น

เส้นทางการกำหนดค่าเริ่มต้น:

- macOS: `~/Library/Application Support/clawhub/config.json`
- Linux/XDG: `$XDG_CONFIG_HOME/clawhub/config.json` หรือ `~/.config/clawhub/config.json`
- Windows: `%APPDATA%\\clawhub\\config.json`

แทนที่เส้นทางด้วย:

```bash
export CLAWHUB_CONFIG_PATH=/path/to/config.json
```

## การเพิกถอน

คุณสามารถเพิกถอนโทเค็น API ได้ใน UI เว็บของ ClawHub

โทเค็นที่ถูกเพิกถอน ไม่ถูกต้อง หรือขาดหายจะส่งคืน `401 Unauthorized` ลงชื่อเข้าใช้อีกครั้ง
ด้วย `clawhub login` หรือระบุโทเค็นใหม่ด้วย `clawhub login --token`

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้โทเค็น API ที่มีอยู่ต่อไปได้
