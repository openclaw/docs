# เทเลเมทรี

ClawHub ใช้ **เทเลเมทรีขั้นต่ำ** เพื่อคำนวณ **จำนวนการติดตั้ง** (สิ่งที่ใช้งานอยู่จริง) และเพื่อช่วยให้การจัดเรียง/กรองดีขึ้น
สิ่งนี้อิงตามคำสั่ง CLI `clawhub sync`

## เมื่อมีการเก็บเทเลเมทรี

เทเลเมทรีจะถูกส่งเฉพาะเมื่อ:

- คุณ **เข้าสู่ระบบแล้ว** ใน CLI (เรากำหนดให้ต้องยืนยันตัวตนสำหรับโฟลว์ sync/publish อยู่แล้ว)
- คุณเรียกใช้ `clawhub sync`
- เทเลเมทรี **ไม่ได้ถูกปิดใช้งาน** (ดู “วิธีปิดใช้งาน” ด้านล่าง)

หากคุณไม่ได้เข้าสู่ระบบ จะไม่มีการรายงานอะไร

## สิ่งที่เราเก็บ

ในแต่ละครั้งที่เรียก `clawhub sync` CLI จะรายงาน **สแนปช็อตแบบเต็ม** ของสิ่งที่พบ โดยจัดกลุ่มตามรากการสแกน (“โฟลเดอร์/ราก”)

สำหรับแต่ละราก เราจัดเก็บ:

- `rootId`: **แฮช SHA-256** ของเส้นทางรากแบบมาตรฐาน (เซิร์ฟเวอร์ไม่เห็นเส้นทางดิบ)
- `label`: ป้ายกำกับที่มนุษย์อ่านได้ ซึ่งได้มาจากสองเซกเมนต์สุดท้ายของเส้นทาง (เส้นทางโฮมจะแสดงด้วย `~`)
- `firstSeenAt`, `lastSeenAt`, `expiredAt` ที่มีหรือไม่มีก็ได้

สำหรับแต่ละ Skill ที่พบใต้ราก เราจัดเก็บ:

- `skillId` (แก้จาก slug; ติดตามเฉพาะ Skills ที่มีอยู่ในรีจิสทรี)
- `firstSeenAt`, `lastSeenAt`
- `lastVersion` (ทำอย่างดีที่สุด; ปัจจุบันคือเวอร์ชันที่ตรงกับรีจิสทรี หากทราบ)
- `removedAt` ที่มีหรือไม่มีก็ได้ เมื่อการติดตั้งที่เคยรายงานไว้หายไปจากราก

### สิ่งที่เรา _ไม่_ เก็บ

- ไม่มีเส้นทางโฟลเดอร์แบบสัมบูรณ์ดิบ (มีเฉพาะ `rootId` ที่ถูกแฮช + ป้ายกำกับแสดงผลสั้นๆ)
- ไม่มีเนื้อหาไฟล์
- ไม่มีบันทึกต่อการรัน พรอมป์ หรือเอาต์พุต CLI อื่นๆ
- ไม่มีการติดตาม Skills ที่ไม่ได้อัปโหลดไปยังรีจิสทรี (slug ที่ไม่รู้จักจะถูกละเว้น)

## จำนวนการติดตั้ง

เราดูแลตัวนับสองตัวต่อ Skill:

- `installsCurrent`: ผู้ใช้ไม่ซ้ำที่ปัจจุบันมี Skill ติดตั้งอยู่ในรากที่ใช้งานอยู่อย่างน้อยหนึ่งราก
- `installsAllTime`: ผู้ใช้ไม่ซ้ำที่เคยรายงานว่ามี Skill ติดตั้งอยู่

### หลายราก

หากคุณ sync จากหลายโฟลเดอร์ เราจะถือว่าแต่ละรากการสแกนเป็นอิสระจากกัน Skill จะถือว่า “ติดตั้งอยู่ในปัจจุบัน” หากมีอยู่ในรากที่ใช้งานอยู่ **ใดๆ**

### การตรวจจับการถอนการติดตั้ง

เนื่องจาก `sync` รายงานชุดเต็มต่อราก:

- หาก Skill หายไปจากรากในการ sync ครั้งถัดไป เราจะทำเครื่องหมายว่าถูกนำออกสำหรับรากนั้น
- หาก Skill ถูกนำออกจากรากทั้งหมดของคุณ จะไม่นับรวมใน `installsCurrent` อีกต่อไป
- `installsAllTime` จะไม่ลดลง เว้นแต่คุณลบเทเลเมทรี (ดูด้านล่าง)

### ความเก่าเกินกำหนด (120 วัน)

รากที่ไม่รายงานเทเลเมทรีเป็นเวลา **120 วัน** จะถูกทำเครื่องหมายว่าเก่าเกินกำหนด และการติดตั้งของรากเหล่านั้นจะหยุดนับรวมใน `installsCurrent`
การประเมินนี้ทำแบบล่าช้า (ในการรายงานเทเลเมทรีครั้งถัดไป) เพื่อหลีกเลี่ยงงานเบื้องหลัง

## ความโปร่งใส + การควบคุมของผู้ใช้

ClawHub มีแท็บ “ติดตั้งแล้ว” แบบส่วนตัวบนโปรไฟล์ของคุณเอง:

- แสดงราก + Skills ที่ติดตั้งตามที่เราจัดเก็บไว้อย่างตรงกัน
- มีมุมมอง **ส่งออก JSON**
- มีการดำเนินการ **ลบเทเลเมทรี** เพื่อลบเทเลเมทรีที่จัดเก็บไว้ทั้งหมดสำหรับบัญชีของคุณ

คนอื่นจะเห็นเฉพาะ **ตัวนับการติดตั้งแบบรวม** เท่านั้น ไม่มีใครอื่นเห็นราก/โฟลเดอร์ของคุณได้

การลบบัญชีของคุณจะลบข้อมูลเทเลเมทรีของคุณด้วย

## วิธีปิดใช้งานเทเลเมทรี

ตั้งค่าตัวแปรสภาพแวดล้อม:

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

เมื่อตั้งค่านี้แล้ว CLI จะไม่ส่งเทเลเมทรีระหว่าง `clawhub sync`
