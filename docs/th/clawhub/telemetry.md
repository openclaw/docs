# เทเลเมทรี

ClawHub ใช้**เทเลเมทรีแบบน้อยที่สุด**เพื่อคำนวณ**จำนวนการติดตั้ง** (สิ่งที่ใช้งานอยู่จริง) และเพื่อช่วยให้การจัดเรียง/กรองดีขึ้น
สิ่งนี้อิงตามคำสั่ง CLI `clawhub sync`

## เมื่อมีการเก็บเทเลเมทรี

เทเลเมทรีจะถูกส่งเฉพาะเมื่อ:

- คุณ**เข้าสู่ระบบแล้ว**ใน CLI (เรากำหนดให้ต้องมีการยืนยันตัวตนสำหรับโฟลว์ sync/publish อยู่แล้ว)
- คุณเรียกใช้ `clawhub sync`
- เทเลเมทรี**ไม่ได้ถูกปิดใช้งาน** (ดู “วิธีปิดใช้งาน” ด้านล่าง)

หากคุณไม่ได้เข้าสู่ระบบ จะไม่มีการรายงานอะไร

## สิ่งที่เราเก็บ

ในการเรียกใช้ `clawhub sync` แต่ละครั้ง CLI จะรายงาน**สแนปช็อตทั้งหมด**ของสิ่งที่พบ โดยจัดกลุ่มตามรากการสแกน (“โฟลเดอร์/ราก”)

สำหรับแต่ละราก เราจัดเก็บ:

- `rootId`: **SHA-256 hash** ของพาธรากแบบบัญญัติ (เซิร์ฟเวอร์จะไม่เห็นพาธดิบ)
- `label`: ป้ายกำกับที่มนุษย์อ่านได้ ซึ่งได้มาจากสองเซกเมนต์สุดท้ายของพาธ (พาธ home จะแสดงด้วย `~`)
- `firstSeenAt`, `lastSeenAt`, `expiredAt` แบบไม่บังคับ

สำหรับแต่ละ skill ที่พบใต้ราก เราจัดเก็บ:

- `skillId` (แก้ไขจาก slug; จะติดตามเฉพาะ Skills ที่มีอยู่ในรีจิสทรีเท่านั้น)
- `firstSeenAt`, `lastSeenAt`
- `lastVersion` (ทำแบบ best-effort; ปัจจุบันคือเวอร์ชันที่ตรงกับรีจิสทรี หากทราบ)
- `removedAt` แบบไม่บังคับ เมื่อการติดตั้งที่เคยรายงานไว้หายไปจากราก

### สิ่งที่เรา _ไม่_ เก็บ

- ไม่มีพาธโฟลเดอร์แบบสัมบูรณ์ดิบ (มีเฉพาะ `rootId` ที่แฮชแล้ว + ป้ายกำกับสั้นสำหรับแสดงผล)
- ไม่มีเนื้อหาไฟล์
- ไม่มีบันทึกต่อการรัน พรอมป์ หรือเอาต์พุต CLI อื่น ๆ
- ไม่มีการติดตาม Skills ที่ไม่ได้อัปโหลดไปยังรีจิสทรี (slug ที่ไม่รู้จักจะถูกละเว้น)

## จำนวนการติดตั้ง

เราดูแลตัวนับสองตัวต่อ skill:

- `installsCurrent`: ผู้ใช้ที่ไม่ซ้ำกันซึ่งปัจจุบันมี skill ติดตั้งอยู่ในรากที่ใช้งานอยู่อย่างน้อยหนึ่งราก
- `installsAllTime`: ผู้ใช้ที่ไม่ซ้ำกันซึ่งเคยรายงานว่ามี skill ติดตั้งอยู่

### หลายราก

หากคุณ sync จากหลายโฟลเดอร์ เราจะถือว่าแต่ละรากการสแกนเป็นอิสระต่อกัน skill จะถือว่า “ติดตั้งอยู่ในปัจจุบัน” หากมีอยู่ในรากที่ใช้งานอยู่**ใด ๆ**

### การตรวจจับการถอนการติดตั้ง

เนื่องจาก `sync` รายงานชุดทั้งหมดต่อราก:

- หาก skill หายไปจากรากในการ sync ครั้งถัดไป เราจะทำเครื่องหมายว่า skill นั้นถูกนำออกสำหรับรากนั้น
- หาก skill ถูกนำออกจากรากทั้งหมดของคุณ skill นั้นจะไม่ถูกนับรวมใน `installsCurrent` อีกต่อไป
- `installsAllTime` จะไม่มีวันลดลง เว้นแต่คุณจะลบเทเลเมทรี (ดูด้านล่าง)

### ความเก่าเกินกำหนด (120 วัน)

รากที่ไม่รายงานเทเลเมทรีเป็นเวลา **120 วัน** จะถูกทำเครื่องหมายว่าเก่าเกินกำหนด และการติดตั้งของรากนั้นจะหยุดนับรวมใน `installsCurrent`
การประเมินนี้ทำแบบ lazy (ในการรายงานเทเลเมทรีครั้งถัดไป) เพื่อหลีกเลี่ยงงานเบื้องหลัง

## ความโปร่งใส + การควบคุมของผู้ใช้

ClawHub มีแท็บ “ติดตั้งแล้ว” แบบส่วนตัวบนโปรไฟล์ของคุณเอง:

- แสดงราก + Skills ที่ติดตั้งไว้ซึ่งเราจัดเก็บอย่างตรงไปตรงมา
- มีมุมมอง**ส่งออก JSON**
- มีการดำเนินการ**ลบเทเลเมทรี**เพื่อลบเทเลเมทรีที่จัดเก็บไว้ทั้งหมดสำหรับบัญชีของคุณ

คนอื่น ๆ จะเห็นเฉพาะ**ตัวนับการติดตั้งแบบรวม**เท่านั้น ไม่มีใครอื่นเห็นราก/โฟลเดอร์ของคุณได้

การลบบัญชีของคุณจะลบข้อมูลเทเลเมทรีของคุณด้วย

## วิธีปิดใช้งานเทเลเมทรี

ตั้งค่าตัวแปรสภาพแวดล้อม:

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

เมื่อตั้งค่านี้แล้ว CLI จะไม่ส่งเทเลเมทรีระหว่าง `clawhub sync`
