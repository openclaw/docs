# เทเลเมทรี

ClawHub ใช้**เทเลเมทรีขั้นต่ำ**เพื่อคำนวณ**จำนวนการติดตั้ง** (สิ่งที่ใช้งานอยู่จริง) และเพื่อช่วยให้การเรียงลำดับ/การกรองดีขึ้น
ข้อมูลนี้อิงตามคำสั่ง CLI `clawhub sync`

## เมื่อมีการเก็บเทเลเมทรี

เทเลเมทรีจะถูกส่งก็ต่อเมื่อ:

- คุณ**เข้าสู่ระบบ**ใน CLI แล้ว (เรากำหนดให้ต้องมีการยืนยันตัวตนสำหรับโฟลว์ sync/publish อยู่แล้ว)
- คุณเรียกใช้ `clawhub sync`
- เทเลเมทรี**ไม่ได้ถูกปิดใช้งาน** (ดู “วิธีปิดใช้งาน” ด้านล่าง)

หากคุณไม่ได้เข้าสู่ระบบ จะไม่มีการรายงานข้อมูลใดๆ

## สิ่งที่เราเก็บรวบรวม

ในการเรียกใช้ `clawhub sync` แต่ละครั้ง CLI จะรายงาน**สแนปช็อตเต็มรูปแบบ**ของสิ่งที่พบ โดยจัดกลุ่มตามรากการสแกน (“โฟลเดอร์/ราก”)

สำหรับแต่ละราก เราเก็บ:

- `rootId`: **แฮช SHA-256** ของพาธรากแบบ canonical (เซิร์ฟเวอร์จะไม่เห็นพาธดิบ)
- `label`: ป้ายกำกับที่มนุษย์อ่านได้ ซึ่งได้มาจากสองเซกเมนต์สุดท้ายของพาธ (พาธโฮมจะแสดงด้วย `~`)
- `firstSeenAt`, `lastSeenAt`, `expiredAt` ที่มีหรือไม่มีก็ได้

สำหรับแต่ละ Skill ที่พบภายใต้ราก เราเก็บ:

- `skillId` (แปลงจาก slug; ติดตามเฉพาะ Skills ที่มีอยู่ใน registry เท่านั้น)
- `firstSeenAt`, `lastSeenAt`
- `lastVersion` (พยายามให้ดีที่สุด; ปัจจุบันคือเวอร์ชันที่ตรงกับ registry หากทราบ)
- `removedAt` ที่มีหรือไม่มีก็ได้ เมื่อการติดตั้งที่เคยรายงานไว้หายไปจากราก

### สิ่งที่เรา _ไม่_ เก็บรวบรวม

- ไม่มีพาธโฟลเดอร์แบบ absolute ดิบ (มีเฉพาะ `rootId` ที่แฮชแล้ว + ป้ายกำกับสั้นๆ สำหรับแสดงผล)
- ไม่มีเนื้อหาไฟล์
- ไม่มีล็อกต่อการรัน พรอมป์ หรือเอาต์พุต CLI อื่นๆ
- ไม่มีการติดตาม Skills ที่ไม่ได้อัปโหลดไปยัง registry (slug ที่ไม่รู้จักจะถูกละเว้น)

## จำนวนการติดตั้ง

เราดูแลตัวนับสองตัวต่อ Skill:

- `installsCurrent`: ผู้ใช้ที่ไม่ซ้ำกันซึ่งปัจจุบันติดตั้ง Skill ไว้ในรากที่ใช้งานอยู่อย่างน้อยหนึ่งราก
- `installsAllTime`: ผู้ใช้ที่ไม่ซ้ำกันซึ่งเคยรายงานว่าติดตั้ง Skill

### หลายราก

หากคุณ sync จากหลายโฟลเดอร์ เราจะถือว่าแต่ละรากการสแกนเป็นอิสระต่อกัน Skill จะถือว่า “ติดตั้งอยู่ในปัจจุบัน” หากมีอยู่ในรากที่ใช้งานอยู่**ใดๆ**

### การตรวจจับการถอนการติดตั้ง

เนื่องจาก `sync` รายงานชุดข้อมูลเต็มรูปแบบต่อราก:

- หาก Skill หายไปจากรากในการ sync ครั้งถัดไป เราจะทำเครื่องหมายว่าถูกนำออกสำหรับรากนั้น
- หาก Skill ถูกนำออกจากรากทั้งหมดของคุณ จะไม่นับรวมใน `installsCurrent` อีกต่อไป
- `installsAllTime` จะไม่ลดลง เว้นแต่คุณจะลบเทเลเมทรี (ดูด้านล่าง)

### ความเก่าเกินกำหนด (120 วัน)

รากที่ไม่รายงานเทเลเมทรีเป็นเวลา **120 วัน** จะถูกทำเครื่องหมายว่าเก่าเกินกำหนด และการติดตั้งของรากนั้นจะหยุดนับรวมใน `installsCurrent`
การประเมินนี้ทำแบบ lazy (ในรายงานเทเลเมทรีครั้งถัดไป) เพื่อหลีกเลี่ยงงานเบื้องหลัง

## ความโปร่งใส + การควบคุมของผู้ใช้

ClawHub มีแท็บ “ติดตั้งแล้ว” แบบส่วนตัวบนโปรไฟล์ของคุณเอง:

- แสดงราก + Skills ที่ติดตั้งซึ่งเราเก็บไว้อย่างถูกต้องตรงกัน
- มีมุมมอง**ส่งออก JSON**
- มีการดำเนินการ**ลบเทเลเมทรี**เพื่อลบเทเลเมทรีที่จัดเก็บไว้ทั้งหมดสำหรับบัญชีของคุณ

คนอื่นจะเห็นเฉพาะ**ตัวนับการติดตั้งแบบรวม**เท่านั้น ไม่มีใครอื่นเห็นราก/โฟลเดอร์ของคุณได้

การลบบัญชีของคุณจะลบข้อมูลเทเลเมทรีของคุณด้วย

## วิธีปิดใช้งานเทเลเมทรี

ตั้งค่าตัวแปรสภาพแวดล้อม:

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

เมื่อตั้งค่านี้แล้ว CLI จะไม่ส่งเทเลเมทรีระหว่าง `clawhub sync`
