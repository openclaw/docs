# การส่งข้อมูลการใช้งาน

ClawHub ใช้ **การส่งข้อมูลการใช้งานขั้นต่ำ** เพื่อคำนวณ **จำนวนการติดตั้ง** (สิ่งที่มีการใช้งานจริง) และเพื่อช่วยให้การจัดเรียง/การกรองดีขึ้น
สิ่งนี้อิงตามคำสั่ง CLI `clawhub sync`

## เมื่อมีการเก็บข้อมูลการใช้งาน

ข้อมูลการใช้งานจะถูกส่งก็ต่อเมื่อ:

- คุณ **เข้าสู่ระบบแล้ว** ใน CLI (เรากำหนดให้ต้องยืนยันตัวตนสำหรับโฟลว์ sync/publish อยู่แล้ว)
- คุณรัน `clawhub sync`
- การส่งข้อมูลการใช้งาน **ไม่ได้ถูกปิดใช้งาน** (ดู “วิธีปิดใช้งาน” ด้านล่าง)

หากคุณไม่ได้เข้าสู่ระบบ จะไม่มีการรายงานสิ่งใด

## สิ่งที่เราเก็บ

ในแต่ละครั้งที่รัน `clawhub sync` CLI จะรายงาน **สแนปช็อตแบบเต็ม** ของสิ่งที่พบ โดยจัดกลุ่มตามรูทการสแกน (“โฟลเดอร์/รูท”)

สำหรับแต่ละรูท เราจัดเก็บ:

- `rootId`: **แฮช SHA-256** ของพาธรูทแบบ canonical (เซิร์ฟเวอร์จะไม่เห็นพาธดิบ)
- `label`: ป้ายกำกับที่อ่านได้โดยมนุษย์ ซึ่งได้มาจากเซกเมนต์พาธสองส่วนสุดท้าย (พาธโฮมจะแสดงด้วย `~`)
- `firstSeenAt`, `lastSeenAt`, `expiredAt` แบบไม่บังคับ

สำหรับ Skills แต่ละรายการที่พบใต้รูท เราจัดเก็บ:

- `skillId` (resolve จาก slug; ติดตามเฉพาะ Skills ที่มีอยู่ใน registry เท่านั้น)
- `firstSeenAt`, `lastSeenAt`
- `lastVersion` (พยายามให้ดีที่สุด; ปัจจุบันคือเวอร์ชันที่ตรงกับ registry หากทราบ)
- `removedAt` แบบไม่บังคับ เมื่อการติดตั้งที่เคยรายงานไว้หายไปจากรูท

### สิ่งที่เรา _ไม่_ เก็บ

- ไม่มีพาธโฟลเดอร์แบบสัมบูรณ์ดิบ (มีเฉพาะ `rootId` ที่ถูกแฮช + ป้ายกำกับแสดงผลแบบสั้น)
- ไม่มีเนื้อหาไฟล์
- ไม่มีบันทึกต่อการรัน พรอมป์ หรือเอาต์พุตอื่นจาก CLI
- ไม่มีการติดตาม Skills ที่ไม่ได้อัปโหลดไปยัง registry (slug ที่ไม่รู้จักจะถูกละเว้น)

## จำนวนการติดตั้ง

เราดูแลตัวนับสองรายการต่อ Skills:

- `installsCurrent`: ผู้ใช้ที่ไม่ซ้ำกันซึ่งปัจจุบันมี Skills ติดตั้งอยู่ในรูทที่ใช้งานอยู่ตั้งแต่หนึ่งรูทขึ้นไป
- `installsAllTime`: ผู้ใช้ที่ไม่ซ้ำกันซึ่งเคยรายงานว่ามี Skills ติดตั้งอยู่

### หลายรูท

หากคุณ sync จากหลายโฟลเดอร์ เราจะถือว่าแต่ละรูทการสแกนเป็นอิสระจากกัน Skills จะถือว่า “ติดตั้งอยู่ในปัจจุบัน” หากมีอยู่ในรูทที่ใช้งานอยู่ **ใดๆ**

### การตรวจจับการถอนการติดตั้ง

เนื่องจาก `sync` รายงานชุดข้อมูลเต็มต่อรูท:

- หาก Skills หายไปจากรูทในการ sync ครั้งถัดไป เราจะทำเครื่องหมายว่าถูกนำออกสำหรับรูทนั้น
- หาก Skills ถูกนำออกจากรูททั้งหมดของคุณ Skills นั้นจะไม่ถูกนับรวมใน `installsCurrent` อีกต่อไป
- `installsAllTime` จะไม่ลดลง เว้นแต่คุณจะลบข้อมูลการใช้งาน (ดูด้านล่าง)

### ความเก่าเกินกำหนด (120 วัน)

รูทที่ไม่รายงานข้อมูลการใช้งานเป็นเวลา **120 วัน** จะถูกทำเครื่องหมายว่าเก่าเกินกำหนด และการติดตั้งของรูทเหล่านั้นจะหยุดถูกนับรวมใน `installsCurrent`
การประเมินนี้ทำแบบ lazy (เมื่อมีรายงานข้อมูลการใช้งานครั้งถัดไป) เพื่อหลีกเลี่ยงงานเบื้องหลัง

## ความโปร่งใส + การควบคุมของผู้ใช้

ClawHub มีแท็บ “ติดตั้งแล้ว” แบบส่วนตัวบนโปรไฟล์ของคุณเอง:

- แสดงรูท + Skills ที่ติดตั้งแล้วซึ่งเราจัดเก็บไว้อย่างตรงตัว
- มีมุมมอง **ส่งออก JSON**
- มีการกระทำ **ลบข้อมูลการใช้งาน** เพื่อนำข้อมูลการใช้งานที่จัดเก็บทั้งหมดออกจากบัญชีของคุณ

คนอื่นจะเห็นเฉพาะ **ตัวนับการติดตั้งแบบรวม** เท่านั้น; ไม่มีใครนอกจากคุณที่เห็นรูท/โฟลเดอร์ของคุณได้

การลบบัญชีของคุณจะลบข้อมูลการใช้งานของคุณด้วย

## วิธีปิดใช้งานการส่งข้อมูลการใช้งาน

ตั้งค่าตัวแปรสภาพแวดล้อม:

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

เมื่อตั้งค่านี้แล้ว CLI จะไม่ส่งข้อมูลการใช้งานระหว่าง `clawhub sync`
