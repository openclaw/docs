# เทเลเมทรี

ClawHub ใช้ **เทเลเมทรีขั้นต่ำ** เพื่อคำนวณ **จำนวนการติดตั้ง** (สิ่งที่ใช้งานอยู่จริง) และช่วยให้การจัดเรียง/กรองดีขึ้น
สิ่งนี้อิงตามคำสั่ง CLI `clawhub sync`

## เมื่อมีการเก็บเทเลเมทรี

เทเลเมทรีจะถูกส่งเฉพาะเมื่อ:

- คุณ **เข้าสู่ระบบ** ใน CLI แล้ว (เรากำหนดให้ต้องมีการยืนยันตัวตนสำหรับโฟลว์ sync/publish อยู่แล้ว)
- คุณเรียกใช้ `clawhub sync`
- เทเลเมทรี **ไม่ได้ถูกปิดใช้งาน** (ดู “วิธีปิดใช้งาน” ด้านล่าง)

หากคุณไม่ได้เข้าสู่ระบบ จะไม่มีการรายงานข้อมูลใดๆ

## สิ่งที่เราเก็บ

ในแต่ละครั้งที่เรียกใช้ `clawhub sync` CLI จะรายงาน **สแนปช็อตแบบครบถ้วน** ของสิ่งที่พบ โดยจัดกลุ่มตามรากการสแกน (“โฟลเดอร์/ราก”)

สำหรับแต่ละราก เราจัดเก็บ:

- `rootId`: **แฮช SHA-256** ของเส้นทางรากแบบมาตรฐาน (เซิร์ฟเวอร์จะไม่เห็นเส้นทางดิบ)
- `label`: ป้ายกำกับที่มนุษย์อ่านได้ ซึ่งได้มาจากสองเซกเมนต์สุดท้ายของเส้นทาง (เส้นทาง home จะแสดงด้วย `~`)
- `firstSeenAt`, `lastSeenAt`, และ `expiredAt` แบบไม่บังคับ

สำหรับแต่ละ Skill ที่พบภายใต้ราก เราจัดเก็บ:

- `skillId` (แก้ไขจาก slug; ติดตามเฉพาะ Skills ที่มีอยู่ใน registry เท่านั้น)
- `firstSeenAt`, `lastSeenAt`
- `lastVersion` (พยายามให้ดีที่สุด; ปัจจุบันคือเวอร์ชันที่ตรงกับ registry หากทราบ)
- `removedAt` แบบไม่บังคับ เมื่อการติดตั้งที่เคยรายงานไว้หายไปจากราก

### สิ่งที่เรา _ไม่_ เก็บ

- ไม่มีเส้นทางโฟลเดอร์แบบสัมบูรณ์ดิบ (มีเฉพาะ `rootId` ที่ถูกแฮช + ป้ายกำกับสั้นสำหรับแสดงผล)
- ไม่มีเนื้อหาไฟล์
- ไม่มีบันทึกต่อการรัน พรอมป์ หรือเอาต์พุต CLI อื่นๆ
- ไม่มีการติดตาม Skills ที่ไม่ได้อัปโหลดไปยัง registry (slug ที่ไม่รู้จักจะถูกละเว้น)

## จำนวนการติดตั้ง

เราดูแลตัวนับสองรายการต่อ Skill:

- `installsCurrent`: ผู้ใช้ที่ไม่ซ้ำกันซึ่งปัจจุบันมี Skill ติดตั้งอยู่ในรากที่ใช้งานอยู่ อย่างน้อยหนึ่งราก
- `installsAllTime`: ผู้ใช้ที่ไม่ซ้ำกันซึ่งเคยรายงานว่ามี Skill ติดตั้งอยู่

### หลายราก

หากคุณ sync จากหลายโฟลเดอร์ เราจะถือว่าแต่ละรากการสแกนเป็นอิสระต่อกัน Skill จะถือว่า “ติดตั้งอยู่ในปัจจุบัน” หากมีอยู่ในรากที่ใช้งานอยู่ **ใดๆ**

### การตรวจจับการถอนการติดตั้ง

เนื่องจาก `sync` รายงานชุดข้อมูลทั้งหมดต่อราก:

- หาก Skill หายไปจากรากในการ sync ครั้งถัดไป เราจะทำเครื่องหมายว่าถูกลบออกสำหรับรากนั้น
- หาก Skill ถูกลบออกจากรากทั้งหมดของคุณ จะไม่นับรวมใน `installsCurrent` อีกต่อไป
- `installsAllTime` จะไม่ลดลง เว้นแต่คุณจะลบเทเลเมทรี (ดูด้านล่าง)

### ความเก่าเกินกำหนด (120 วัน)

รากที่ไม่รายงานเทเลเมทรีเป็นเวลา **120 วัน** จะถูกทำเครื่องหมายว่าเก่าเกินกำหนด และการติดตั้งของรากเหล่านั้นจะหยุดถูกนับรวมใน `installsCurrent`
สิ่งนี้จะถูกประเมินแบบ lazy (เมื่อมีรายงานเทเลเมทรีครั้งถัดไป) เพื่อหลีกเลี่ยงงานเบื้องหลัง

## ความโปร่งใส + การควบคุมของผู้ใช้

ClawHub มีแท็บ “ติดตั้งแล้ว” แบบส่วนตัวในโปรไฟล์ของคุณเอง:

- แสดงราก + Skills ที่ติดตั้งไว้ซึ่งเราจัดเก็บอย่างตรงไปตรงมา
- มีมุมมอง **ส่งออก JSON**
- มีการดำเนินการ **ลบเทเลเมทรี** เพื่อลบเทเลเมทรีที่จัดเก็บทั้งหมดสำหรับบัญชีของคุณ

ผู้อื่นทั้งหมดจะเห็นเฉพาะ **ตัวนับการติดตั้งแบบรวม**; ไม่มีใครอื่นสามารถเห็นราก/โฟลเดอร์ของคุณได้

การลบบัญชีของคุณจะลบข้อมูลเทเลเมทรีของคุณด้วย

## วิธีปิดใช้งานเทเลเมทรี

ตั้งค่าตัวแปรสภาพแวดล้อม:

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

เมื่อตั้งค่านี้แล้ว CLI จะไม่ส่งเทเลเมทรีระหว่าง `clawhub sync`
