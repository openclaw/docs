# ข้อมูลโทรมาตร

ClawHub ใช้**ข้อมูลโทรมาตรขั้นต่ำ**เพื่อคำนวณ**จำนวนการติดตั้ง** (สิ่งที่ใช้งานจริง) และเพื่อช่วยให้การเรียงลำดับ/การกรองดีขึ้น
สิ่งนี้อิงตามคำสั่ง CLI `clawhub sync`

## เมื่อมีการเก็บข้อมูลโทรมาตร

ข้อมูลโทรมาตรจะถูกส่งเฉพาะเมื่อ:

- คุณ**เข้าสู่ระบบแล้ว**ใน CLI (เรากำหนดให้ต้องยืนยันตัวตนสำหรับโฟลว์การซิงก์/เผยแพร่อยู่แล้ว)
- คุณเรียกใช้ `clawhub sync`
- ข้อมูลโทรมาตร**ไม่ได้ถูกปิดใช้งาน** (ดู “วิธีปิดใช้งาน” ด้านล่าง)

หากคุณไม่ได้เข้าสู่ระบบ จะไม่มีการรายงานข้อมูลใดๆ

## สิ่งที่เราเก็บ

ในแต่ละครั้งที่เรียกใช้ `clawhub sync` CLI จะรายงาน**สแนปช็อตแบบเต็ม**ของสิ่งที่พบ โดยจัดกลุ่มตามรากการสแกน (“โฟลเดอร์/ราก”)

สำหรับแต่ละราก เราเก็บ:

- `rootId`: **แฮช SHA-256** ของเส้นทางรากแบบบัญญัติ (เซิร์ฟเวอร์จะไม่เห็นเส้นทางดิบ)
- `label`: ป้ายกำกับที่มนุษย์อ่านได้ซึ่งได้มาจากสองส่วนสุดท้ายของเส้นทาง (เส้นทางโฮมจะแสดงด้วย `~`)
- `firstSeenAt`, `lastSeenAt`, `expiredAt` แบบไม่บังคับ

สำหรับแต่ละ Skill ที่พบใต้ราก เราเก็บ:

- `skillId` (แก้ไขจาก slug; ติดตามเฉพาะ Skills ที่มีอยู่ในรีจิสทรี)
- `firstSeenAt`, `lastSeenAt`
- `lastVersion` (พยายามให้ดีที่สุด; ปัจจุบันคือเวอร์ชันที่ตรงกับรีจิสทรี หากทราบ)
- `removedAt` แบบไม่บังคับ เมื่อการติดตั้งที่เคยถูกรายงานหายไปจากราก

### สิ่งที่เรา _ไม่_ เก็บ

- ไม่มีเส้นทางโฟลเดอร์แบบสัมบูรณ์ดิบ (มีเฉพาะ `rootId` ที่ถูกแฮช + ป้ายกำกับแสดงผลแบบสั้น)
- ไม่มีเนื้อหาไฟล์
- ไม่มีบันทึกต่อการเรียกใช้ พรอมป์ หรือเอาต์พุตอื่นๆ ของ CLI
- ไม่มีการติดตาม Skills ที่ไม่ได้อัปโหลดไปยังรีจิสทรี (slug ที่ไม่รู้จักจะถูกละเว้น)

## จำนวนการติดตั้ง

เรารักษาตัวนับสองรายการต่อ Skill:

- `installsCurrent`: ผู้ใช้ไม่ซ้ำที่ปัจจุบันมี Skill ติดตั้งอยู่ในรากที่ใช้งานอยู่อย่างน้อยหนึ่งราก
- `installsAllTime`: ผู้ใช้ไม่ซ้ำที่เคยรายงานว่ามี Skill ติดตั้งอยู่

### หลายราก

หากคุณซิงก์จากหลายโฟลเดอร์ เราจะถือว่าแต่ละรากการสแกนแยกจากกัน Skill จะถือว่า “ติดตั้งอยู่ในปัจจุบัน” หากมีอยู่ในรากที่ใช้งานอยู่**ใดๆ**

### การตรวจจับการถอนการติดตั้ง

เนื่องจาก `sync` รายงานชุดข้อมูลแบบเต็มต่อราก:

- หาก Skill หายไปจากรากในการซิงก์ครั้งถัดไป เราจะทำเครื่องหมายว่าถูกลบสำหรับรากนั้น
- หาก Skill ถูกลบออกจากรากทั้งหมดของคุณ Skill นั้นจะไม่นับรวมใน `installsCurrent` อีกต่อไป
- `installsAllTime` จะไม่ลดลง เว้นแต่คุณจะลบข้อมูลโทรมาตร (ดูด้านล่าง)

### ความเก่า (120 วัน)

รากที่ไม่รายงานข้อมูลโทรมาตรเป็นเวลา **120 วัน** จะถูกทำเครื่องหมายว่าเก่า และการติดตั้งของรากนั้นจะหยุดนับรวมใน `installsCurrent`
สิ่งนี้จะถูกประเมินแบบหน่วงเวลา (ในการรายงานข้อมูลโทรมาตรครั้งถัดไป) เพื่อหลีกเลี่ยงงานเบื้องหลัง

## ความโปร่งใส + การควบคุมของผู้ใช้

ClawHub มีแท็บ “ติดตั้งแล้ว” แบบส่วนตัวบนโปรไฟล์ของคุณเอง:

- แสดงราก + Skills ที่ติดตั้งซึ่งเราเก็บไว้อย่างถูกต้อง
- มีมุมมอง**ส่งออก JSON**
- มีการดำเนินการ**ลบข้อมูลโทรมาตร**เพื่อลบข้อมูลโทรมาตรที่จัดเก็บทั้งหมดสำหรับบัญชีของคุณ

คนอื่นจะเห็นเฉพาะ**ตัวนับการติดตั้งแบบรวม**เท่านั้น ไม่มีใครเห็นราก/โฟลเดอร์ของคุณได้

การลบบัญชีของคุณจะลบข้อมูลโทรมาตรของคุณด้วย

## วิธีปิดใช้งานข้อมูลโทรมาตร

ตั้งค่าตัวแปรสภาพแวดล้อม:

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

เมื่อตั้งค่านี้แล้ว CLI จะไม่ส่งข้อมูลโทรมาตรระหว่าง `clawhub sync`
