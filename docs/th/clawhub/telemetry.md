# เทเลเมทรี

ClawHub ใช้**เทเลเมทรีขั้นต่ำ**เพื่อคำนวณ**จำนวนการติดตั้ง** (สิ่งที่มีการใช้งานจริง) และเพื่อช่วยให้การจัดเรียง/การกรองดีขึ้น
สิ่งนี้อิงตามคำสั่ง CLI `clawhub sync`

## เมื่อมีการเก็บรวบรวมเทเลเมทรี

ระบบจะส่งเทเลเมทรีก็ต่อเมื่อ:

- คุณ**เข้าสู่ระบบแล้ว**ใน CLI (เรากำหนดให้ต้องมีการยืนยันตัวตนอยู่แล้วสำหรับโฟลว์ sync/publish)
- คุณเรียกใช้ `clawhub sync`
- เทเลเมทรี**ไม่ได้ถูกปิดใช้งาน** (ดู “วิธีปิดใช้งาน” ด้านล่าง)

หากคุณไม่ได้เข้าสู่ระบบ จะไม่มีการรายงานสิ่งใด

## สิ่งที่เราเก็บรวบรวม

ในแต่ละ `clawhub sync` CLI จะรายงาน**สแนปช็อตแบบเต็ม**ของสิ่งที่พบ โดยจัดกลุ่มตามรากการสแกน (“โฟลเดอร์/ราก”)

สำหรับแต่ละราก เราจัดเก็บ:

- `rootId`: **แฮช SHA-256** ของเส้นทางรากแบบมาตรฐาน (เซิร์ฟเวอร์จะไม่เห็นเส้นทางดิบ)
- `label`: ป้ายกำกับที่มนุษย์อ่านได้ ซึ่งได้มาจากสองเซกเมนต์สุดท้ายของเส้นทาง (เส้นทางโฮมจะแสดงด้วย `~`)
- `firstSeenAt`, `lastSeenAt`, `expiredAt` ที่เป็นตัวเลือก

สำหรับ Skills แต่ละรายการที่พบภายใต้ราก เราจัดเก็บ:

- `skillId` (แก้ไขจาก slug; ติดตามเฉพาะ Skills ที่มีอยู่ในรีจิสทรี)
- `firstSeenAt`, `lastSeenAt`
- `lastVersion` (พยายามอย่างดีที่สุด; ปัจจุบันคือเวอร์ชันที่ตรงกับรีจิสทรี หากทราบ)
- `removedAt` ที่เป็นตัวเลือก เมื่อการติดตั้งที่เคยรายงานไว้หายไปจากราก

### สิ่งที่เรา _ไม่_ เก็บรวบรวม

- ไม่มีเส้นทางโฟลเดอร์สัมบูรณ์แบบดิบ (มีเพียง `rootId` ที่ถูกแฮช + ป้ายกำกับแสดงผลสั้น ๆ)
- ไม่มีเนื้อหาไฟล์
- ไม่มีบันทึกต่อการเรียกใช้, prompt หรือเอาต์พุตอื่น ๆ ของ CLI
- ไม่มีการติดตาม Skills ที่ไม่ได้อัปโหลดไปยังรีจิสทรี (slug ที่ไม่รู้จักจะถูกละเว้น)

## จำนวนการติดตั้ง

เราดูแลตัวนับสองตัวต่อ Skills แต่ละรายการ:

- `installsCurrent`: ผู้ใช้ที่ไม่ซ้ำกันซึ่งขณะนี้มี Skills ติดตั้งอยู่ในรากที่ใช้งานอยู่แล้วอย่างน้อยหนึ่งราก
- `installsAllTime`: ผู้ใช้ที่ไม่ซ้ำกันซึ่งเคยรายงานว่าติดตั้ง Skills แล้ว

### หลายราก

หากคุณ sync จากหลายโฟลเดอร์ เราจะถือว่าแต่ละรากการสแกนเป็นอิสระจากกัน Skills จะถือว่า “ติดตั้งอยู่ในขณะนี้” หากมีอยู่ในรากที่ใช้งานอยู่**รากใดก็ได้**

### การตรวจจับการถอนการติดตั้ง

เนื่องจาก `sync` รายงานชุดข้อมูลแบบเต็มต่อราก:

- หาก Skills หายไปจากรากในการ sync ครั้งถัดไป เราจะทำเครื่องหมายว่าถูกลบสำหรับรากนั้น
- หาก Skills ถูกลบออกจากรากทั้งหมดของคุณ Skills นั้นจะไม่ถูกนับรวมใน `installsCurrent` อีกต่อไป
- `installsAllTime` จะไม่ลดลง เว้นแต่คุณจะลบเทเลเมทรี (ดูด้านล่าง)

### ความล้าสมัย (120 วัน)

รากที่ไม่รายงานเทเลเมทรีเป็นเวลา **120 วัน** จะถูกทำเครื่องหมายว่าล้าสมัย และการติดตั้งของรากนั้นจะหยุดนับรวมใน `installsCurrent`
สิ่งนี้จะถูกประเมินแบบหน่วงเวลา (ในรายงานเทเลเมทรีครั้งถัดไป) เพื่อหลีกเลี่ยงงานเบื้องหลัง

## ความโปร่งใส + การควบคุมของผู้ใช้

ClawHub มีแท็บ “ติดตั้งแล้ว” แบบส่วนตัวบนโปรไฟล์ของคุณเอง:

- แสดงราก + Skills ที่ติดตั้งแล้วซึ่งเราจัดเก็บไว้อย่างตรงกัน
- รวมมุมมอง**ส่งออก JSON**
- รวมการดำเนินการ**ลบเทเลเมทรี**เพื่อลบเทเลเมทรีทั้งหมดที่จัดเก็บไว้สำหรับบัญชีของคุณ

ผู้อื่นจะเห็นเพียง**ตัวนับการติดตั้งแบบรวม**เท่านั้น; ไม่มีใครอื่นที่สามารถเห็นราก/โฟลเดอร์ของคุณได้

การลบบัญชีของคุณจะลบข้อมูลเทเลเมทรีของคุณด้วย

## วิธีปิดใช้งานเทเลเมทรี

ตั้งค่าตัวแปรสภาพแวดล้อม:

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

เมื่อตั้งค่านี้แล้ว CLI จะไม่ส่งเทเลเมทรีระหว่าง `clawhub sync`
