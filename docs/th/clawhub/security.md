# ความปลอดภัย

สามารถรายงานปัญหาความปลอดภัยของ ClawHub ผ่าน GitHub Security Advisories สำหรับ
`openclaw/clawhub`

ใช้ GitHub Security Advisories สำหรับช่องโหว่ใน ClawHub เอง รายงาน advisory ของ
ClawHub ที่ดีรวมถึงบั๊กใน:

- เว็บไซต์, API หรือ CLI ของ ClawHub
- การเผยแพร่ registry, การดาวน์โหลด, การติดตั้ง หรือความถูกต้องสมบูรณ์ของ artifact
- การยืนยันตัวตน, การอนุญาตสิทธิ์ หรือ API tokens
- การสแกน, การดูแลกลั่นกรอง หรือการจัดการรายงาน

อย่าใช้ advisory ของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ Skills หรือ
Plugin บุคคลที่สามเอง ให้รายงานช่องโหว่เหล่านั้นโดยตรงไปยังผู้เผยแพร่หรือ
repository ต้นทางที่ลิงก์จากรายการใน ClawHub

## การเปิดเผยช่องโหว่

เนื่องจาก ClawHub เป็นแอปพลิเคชันคลาวด์แบบโฮสต์ ช่องโหว่ของบริการ ClawHub
จึงไม่ถูกเปิดเผยต่อสาธารณะโดยค่าเริ่มต้น ช่องโหว่จะถูกเปิดเผยต่อสาธารณะเมื่อมี
หลักฐานว่ามีผลกระทบต่อผู้ใช้จริง หรือเมื่อผู้ใช้จำเป็นต้องดำเนินการ

ตัวอย่างของผลกระทบต่อผู้ใช้จริง ได้แก่ การยืนยันว่ามีการโจมตีสำเร็จ, การเปิดเผย
ข้อมูลผู้ใช้หรือความลับ, เนื้อหาที่เป็นอันตรายเข้าถึงผู้ใช้เนื่องจากความล้มเหลว
ของแพลตฟอร์ม หรือปัญหาใดๆ ที่กำหนดให้ผู้ใช้ต้องหมุนเวียนข้อมูลประจำตัว, อัปเดต
ซอฟต์แวร์ในเครื่อง หรือดำเนินการป้องกันอื่นๆ

ช่องโหว่ในซอฟต์แวร์ที่ผู้ใช้ติดตั้งจะถูกเปิดเผยต่อสาธารณะ เช่น แพ็กเกจ CLI,
ไบนารี, ไลบรารี หรือ artifact รุ่นเผยแพร่อื่นๆ ของ ClawHub ที่ผู้ใช้ต้องอัปเดต
ในเครื่อง

## หน้าที่เกี่ยวข้อง

สำหรับป้ายกำกับการตรวจสอบขณะติดตั้ง, ระดับความเสี่ยง, สิ่งที่ตรวจพบ และการตีความ โปรดดู
[การตรวจสอบความปลอดภัย](/clawhub/security-audits)

สำหรับรายงานใน marketplace, การระงับเพื่อดูแลกลั่นกรอง, รายการที่ถูกซ่อน, การแบน และสถานะบัญชี โปรดดู [การดูแลกลั่นกรองและความปลอดภัยของบัญชี](/clawhub/moderation)
