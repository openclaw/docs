# ความปลอดภัย + การกำกับดูแล

ClawHub เปิดให้เผยแพร่ได้ แต่รายการสาธารณะยังคงผ่านการควบคุมด้านความน่าเชื่อถือ,
การสแกน, การรายงาน, และการกำกับดูแล เป้าหมายเป็นเชิงปฏิบัติ: ช่วยให้ผู้ใช้
ตรวจสอบสิ่งที่ติดตั้ง, ให้ผู้เผยแพร่มีเส้นทางกู้คืนสำหรับผลบวกลวง,
และกันแพ็กเกจที่มีพฤติกรรมไม่เหมาะสมออกจากการค้นพบแบบสาธารณะ

ดูเพิ่มเติมที่ [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## สิ่งที่ผู้ใช้ตรวจสอบได้

ก่อนติดตั้ง skill หรือ Plugin ให้ตรวจสอบรายการ ClawHub สำหรับ:

- เจ้าของและการระบุแหล่งที่มา
- เวอร์ชันล่าสุดและบันทึกการเปลี่ยนแปลง
- ตัวแปรสภาพแวดล้อมหรือสิทธิ์ที่จำเป็น
- เมตาดาต้าความเข้ากันได้สำหรับ Plugin
- สถานะการสแกนหรือการกำกับดูแล
- รายงาน, ความคิดเห็น, ดาว, การดาวน์โหลด, และสัญญาณการติดตั้งเมื่อมีแสดง

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการสแกน

ClawHub อาจแสดงผลลัพธ์การสแกนหรือการกำกับดูแลบนหน้าสาธารณะและข้อมูลวินิจฉัย
ที่เจ้าของมองเห็นได้

ผลลัพธ์ทั่วไปได้แก่:

- `clean`: ไม่พบปัญหาที่ต้องบล็อก
- `suspicious`: รีลีสนี้ต้องใช้ความระมัดระวังหรือการตรวจทาน
- `malicious`: รีลีสนี้ถือว่าไม่ปลอดภัย
- `pending`: การตรวจสอบยังไม่เสร็จสิ้น
- `held`, `quarantined`, `revoked`, หรือ `hidden`: รีลีสนี้ยังไม่พร้อมใช้งานอย่างเต็มรูปแบบ
  บนพื้นผิวการติดตั้งสาธารณะ

ถ้อยคำที่แน่นอนอาจแตกต่างกันไปตามพื้นผิว แต่ความหมายเชิงปฏิบัติเหมือนกัน: หาก
รีลีสถูกพักหรือถูกบล็อก ผู้ใช้ไม่ควรติดตั้งจนกว่าเจ้าของจะแก้ไขปัญหา
หรือการกำกับดูแลจะกู้คืนรีลีสนั้น

## Skills

การสแกน Skill จะดูที่บันเดิล Skill ที่เผยแพร่, เมตาดาต้า, ข้อกำหนดที่ประกาศไว้,
และคำสั่งที่น่าสงสัย

ClawHub ให้ความสำคัญเป็นพิเศษกับความไม่ตรงกันระหว่างสิ่งที่ Skill ประกาศกับ
สิ่งที่ดูเหมือนว่าทำอยู่ ตัวอย่างเช่น Skill ที่อ้างถึงคีย์ API ที่จำเป็น
ควรประกาศข้อกำหนดนั้นใน `SKILL.md` เพื่อให้ผู้ใช้เห็นได้ก่อนติดตั้ง

ผลการสแกนอิงตามอาร์ติแฟกต์ พฤติกรรมของผู้ให้บริการที่คาดไว้ เช่น ข้อมูลรับรอง API
ที่ประกาศไว้, การเรียกกลับ OAuth ของ localhost, การล้างข้อมูลเมื่อถอนการติดตั้งตามขอบเขต,
การเข้ารหัส Basic Auth, หรือการอัปโหลดไฟล์ที่ผู้ใช้เลือกไปยังผู้ให้บริการที่ระบุไว้
จะได้รับการปฏิบัติแตกต่างจากการส่งต่อข้อมูลรับรองแบบซ่อนเร้น, การเข้าถึงไฟล์ส่วนตัว
แบบกว้าง, ปลายทางเครือข่ายที่ไม่เกี่ยวข้อง, หรือการใช้งานเบราว์เซอร์ในทางที่ผิดแบบลอบทำ

ดู [รูปแบบ Skill](/th/clawhub/skill-format)

## Plugin

รีลีส Plugin มีเมตาดาต้าแพ็กเกจ, การระบุแหล่งที่มา, ฟิลด์ความเข้ากันได้,
และข้อมูลความสมบูรณ์ของอาร์ติแฟกต์

OpenClaw ตรวจสอบความเข้ากันได้ก่อนติดตั้ง Plugin ที่โฮสต์บน ClawHub ระเบียนแพ็กเกจ
อาจเปิดเผยเมตาดาต้า digest เพื่อให้ OpenClaw ตรวจสอบอาร์ติแฟกต์ที่ดาวน์โหลดได้ด้วย
ClawScan รวมเมตาดาต้า env/config ของแพ็กเกจ `openclaw.environment` ที่ประกาศไว้
เมื่อรีวิวรีลีส Plugin เพื่อเปรียบเทียบข้อกำหนดรันไทม์ที่ประกาศกับพฤติกรรมที่สังเกตได้

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน Skills, แพ็กเกจ, และความคิดเห็นได้

รายงานควรเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้ระบบรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

ตัวอย่างรายงาน:

- เมตาดาต้าที่ทำให้เข้าใจผิด
- ข้อกำหนดด้านข้อมูลรับรองหรือสิทธิ์ที่ไม่ได้ประกาศ
- คำสั่งติดตั้งที่น่าสงสัย
- ความคิดเห็นหลอกลวงหรือการแอบอ้างตัวตน
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## หมายเหตุ ClawScan สำหรับผู้เผยแพร่

ผู้เผยแพร่สามารถให้หมายเหตุ ClawScan แบบไม่บังคับเมื่อเผยแพร่ Skill หรือ
Plugin ได้ หมายเหตุนี้ให้บริบทแก่ ClawScan สำหรับพฤติกรรมที่อาจดูผิดปกติ
เช่น การเข้าถึงเครือข่าย, การเข้าถึงโฮสต์เนทีฟ, หรือข้อมูลรับรองเฉพาะผู้ให้บริการ

## การพักเพื่อกำกับดูแล

เมื่อสแกนเนอร์แบบสแตติกทำเครื่องหมาย Skill ที่อัปโหลดว่าเป็นอันตราย ผู้เผยแพร่จะถูก
วางไว้ภายใต้การพักเพื่อกำกับดูแลโดยอัตโนมัติ (`requiresModerationAt` ถูกตั้งบน
ผู้ใช้) สิ่งนี้จะซ่อน Skills ทั้งหมดของผู้เผยแพร่, ทำให้การเผยแพร่ในอนาคต
เริ่มต้นแบบซ่อน, และสร้างรายการบันทึกการตรวจสอบ `user.moderation.auto`

ผลการค้นพบแบบสแตติกที่น่าสงสัยจะถูกเก็บไว้เป็นหลักฐานไฟล์/บรรทัดสำหรับผู้กำกับดูแล
แต่ผลเหล่านี้จะไม่ซ่อนเนื้อหาหรือตัดสินคำตัดสินการสแกนสาธารณะด้วยตัวเอง
การอัปโหลดใหม่จะยังคงอยู่ในสถานะรีวิว/รอดำเนินการจนกว่าการรีวิวของ LLM จะได้ข้อสรุป
การสแกนแบบสแตติกจะบล็อกทันทีเฉพาะลายเซ็นที่เป็นอันตรายเท่านั้น การตรวจพบจากเอนจิน
VirusTotal ยังคงมองเห็นได้เป็นหลักฐานด้านความปลอดภัย แต่คำตัดสินของ VirusTotal
Code Insight/Palm เป็นเพียงคำแนะนำและไม่ซ่อน Skills ด้วยตัวเอง การรีวิว LLM ของ
ClawScan จะเก็บหมายเหตุที่สอดคล้องกับวัตถุประสงค์ไว้เป็นแนวทาง ผลการรีวิวระดับกลาง
ยังคงมองเห็นได้บนอาร์ติแฟกต์ ขณะที่ตัวกรองที่น่าสงสัยสงวนไว้สำหรับข้อกังวลของ LLM
ที่มีผลกระทบสูง, ผลการค้นพบที่เป็นอันตราย, หรือการตรวจพบจากเอนจิน AV ที่มีหลักฐานสนับสนุน

ผู้ดูแลระบบสามารถยกเลิกการพักที่เป็นผลบวกลวงได้:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

สิ่งนี้จะล้าง `requiresModerationAt` และ `requiresModerationReason`, กู้คืน
Skills ที่ถูกซ่อนโดยการพักระดับผู้ใช้, และเขียนรายการบันทึกการตรวจสอบ
`user.moderation.lift` Skills ที่ถูกซ่อนด้วยเหตุผลอื่น หรือ Skills ที่การสแกนแบบสแตติก
ของตัวเองยังคงเป็นอันตราย จะยังคงถูกซ่อนอยู่

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบาย ClawHub อาจสูญเสียสิทธิ์การเผยแพร่ การละเมิดร้ายแรง
อาจส่งผลให้บัญชีถูกแบน, เพิกถอนโทเค็น, ซ่อนเนื้อหา, หรือลบรายการ

บัญชีที่ถูกลบ, ถูกแบน, หรือถูกปิดใช้งานไม่สามารถใช้โทเค็น API ของ ClawHub ได้
หากการยืนยันตัวตน CLI เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้
UI เว็บเพื่อตรวจสอบสถานะบัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อก
ให้ติดต่อ security@openclaw.ai เพื่อขอรีวิวการกู้คืน

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลดผลบวกลวงและเพิ่มความเชื่อมั่นของผู้ใช้:

- รักษาชื่อ, สรุป, แท็ก, และบันทึกการเปลี่ยนแปลงให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- เพิ่มหมายเหตุ ClawScan สำหรับผู้เผยแพร่เมื่อรีลีสมีพฤติกรรมที่ผิดปกติแต่ตั้งใจไว้
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้อ่านยาก
- ลิงก์ไปยังซอร์สเมื่อเป็นไปได้
- ใช้ dry run ก่อนเผยแพร่ Plugin
- ตอบให้ชัดเจนหากผู้ใช้หรือผู้กำกับดูแลถามเกี่ยวกับพฤติกรรมของแพ็กเกจ
