# ความปลอดภัย + การกลั่นกรอง

ClawHub เปิดให้เผยแพร่ได้ แต่รายการสาธารณะยังต้องผ่านการควบคุมด้านความน่าเชื่อถือ
การสแกน การรายงาน และการกลั่นกรอง เป้าหมายคือการใช้งานจริง: ช่วยให้ผู้ใช้
ตรวจสอบสิ่งที่ติดตั้ง ให้ผู้เผยแพร่มีเส้นทางแก้ไขเมื่อเกิดผลบวกเท็จ
และกันแพ็กเกจที่ไม่เหมาะสมออกจากการค้นพบแบบสาธารณะ

ดูเพิ่มเติมที่ [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## สิ่งที่ผู้ใช้ตรวจสอบได้

ก่อนติดตั้ง skill หรือ Plugin ให้ตรวจสอบรายการ ClawHub ของรายการนั้นสำหรับ:

- เจ้าของและการระบุแหล่งที่มา
- เวอร์ชันล่าสุดและบันทึกการเปลี่ยนแปลง
- ตัวแปรสภาพแวดล้อมหรือสิทธิ์ที่จำเป็น
- เมตาดาต้าความเข้ากันได้สำหรับ Plugin
- สถานะการสแกนหรือการกลั่นกรอง
- รายงาน ความคิดเห็น ดาว จำนวนดาวน์โหลด และสัญญาณการติดตั้งเมื่อมีแสดง

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการสแกน

ClawHub อาจแสดงผลลัพธ์การสแกนหรือการกลั่นกรองบนหน้าสาธารณะและการวินิจฉัย
ที่เจ้าของมองเห็นได้

ผลลัพธ์ทั่วไปประกอบด้วย:

- `clean`: ไม่พบปัญหาที่ต้องบล็อก
- `suspicious`: รีลีสต้องใช้ความระมัดระวังหรือการตรวจสอบ
- `malicious`: รีลีสถูกพิจารณาว่าไม่ปลอดภัย
- `pending`: การตรวจสอบยังไม่เสร็จสิ้น
- `held`, `quarantined`, `revoked` หรือ `hidden`: รีลีสยังไม่พร้อมใช้งานอย่างสมบูรณ์
  บนพื้นผิวการติดตั้งสาธารณะ

ถ้อยคำที่แน่นอนอาจแตกต่างกันไปตามพื้นผิว แต่ความหมายเชิงปฏิบัติเหมือนกัน: หาก
รีลีสถูกระงับหรือบล็อก ผู้ใช้ไม่ควรติดตั้งจนกว่าเจ้าของจะแก้ไขปัญหา
หรือการกลั่นกรองจะกู้คืนรีลีสนั้น

## Skills

การสแกน skill จะตรวจสอบบันเดิล skill ที่เผยแพร่ เมตาดาต้า ข้อกำหนดที่ประกาศไว้
และคำสั่งที่น่าสงสัย

ClawHub ให้ความสำคัญเป็นพิเศษกับความไม่ตรงกันระหว่างสิ่งที่ skill ประกาศ
และสิ่งที่ดูเหมือนว่าทำอยู่ ตัวอย่างเช่น skill ที่อ้างถึงคีย์ API ที่จำเป็น
ควรประกาศข้อกำหนดนั้นใน `SKILL.md` เพื่อให้ผู้ใช้เห็นได้ก่อนติดตั้ง

ผลการสแกนอิงตามอาร์ติแฟกต์ พฤติกรรมของผู้ให้บริการที่คาดไว้ เช่น ข้อมูลรับรอง API
ที่ประกาศไว้, localhost OAuth callbacks, การล้างข้อมูลถอนการติดตั้งแบบกำหนดขอบเขต,
การเข้ารหัส Basic Auth หรือการอัปโหลดไฟล์ที่ผู้ใช้เลือกไปยังผู้ให้บริการที่ระบุไว้
จะถูกปฏิบัติแตกต่างจากการส่งต่อข้อมูลรับรองแบบซ่อน การเข้าถึงไฟล์ส่วนตัวอย่างกว้างขวาง
ปลายทางเครือข่ายที่ไม่เกี่ยวข้อง หรือการใช้เบราว์เซอร์ในทางที่ผิดแบบลอบซ่อน

ดู [รูปแบบ skill](/th/clawhub/skill-format)

## Plugin

รีลีสของ Plugin มีเมตาดาต้าแพ็กเกจ การระบุแหล่งที่มา ฟิลด์ความเข้ากันได้
และข้อมูลความสมบูรณ์ของอาร์ติแฟกต์

OpenClaw ตรวจสอบความเข้ากันได้ก่อนติดตั้ง Plugin ที่โฮสต์บน ClawHub ระเบียนแพ็กเกจ
อาจเปิดเผยเมตาดาต้าไดเจสต์ด้วย เพื่อให้ OpenClaw ตรวจสอบอาร์ติแฟกต์ที่ดาวน์โหลดได้
ClawScan รวมเมตาดาต้า env/config ของแพ็กเกจ `openclaw.environment` ที่ประกาศไว้
เมื่อรีวิวรีลีสของ Plugin เพื่อเปรียบเทียบข้อกำหนดรันไทม์ที่ประกาศไว้กับพฤติกรรมที่สังเกตพบ

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน skill, แพ็กเกจ และความคิดเห็นได้

รายงานควรเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้การรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

ตัวอย่างรายงาน:

- เมตาดาต้าที่ทำให้เข้าใจผิด
- ข้อกำหนดข้อมูลรับรองหรือสิทธิ์ที่ไม่ได้ประกาศ
- คำสั่งติดตั้งที่น่าสงสัย
- ความคิดเห็นหลอกลวงหรือการแอบอ้างตัวตน
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## หมายเหตุ ClawScan สำหรับผู้เผยแพร่

ผู้เผยแพร่สามารถให้หมายเหตุ ClawScan แบบไม่บังคับเมื่อเผยแพร่ skill หรือ
Plugin หมายเหตุนี้ให้บริบทแก่ ClawScan สำหรับพฤติกรรมที่อาจดูผิดปกติ
เช่น การเข้าถึงเครือข่าย การเข้าถึงโฮสต์เนทีฟ หรือข้อมูลรับรองเฉพาะผู้ให้บริการ

## การระงับเพื่อการกลั่นกรอง

เมื่อสแกนเนอร์แบบสแตติกระบุว่า skill ที่อัปโหลดเป็นอันตราย ผู้เผยแพร่จะถูก
นำเข้าสู่การระงับเพื่อการกลั่นกรองโดยอัตโนมัติ (`requiresModerationAt` ถูกตั้งบน
ผู้ใช้) การดำเนินการนี้ซ่อน skill ทั้งหมดของผู้เผยแพร่ ทำให้การเผยแพร่ในอนาคต
เริ่มต้นแบบซ่อน และสร้างรายการบันทึกการตรวจสอบ `user.moderation.auto`

ผลการตรวจพบแบบสแตติกที่น่าสงสัยจะถูกเก็บไว้เป็นหลักฐานไฟล์/บรรทัดสำหรับผู้กลั่นกรอง
แต่จะไม่ซ่อนเนื้อหาหรือตัดสินผลสแกนสาธารณะด้วยตัวเอง การอัปโหลดใหม่ยังคงอยู่ในสถานะ
รีวิว/รอดำเนินการจนกว่าการรีวิวของ LLM จะเสร็จสิ้น การสแกนแบบสแตติกจะบล็อกทันที
เฉพาะลายเซ็นที่เป็นอันตรายเท่านั้น ผลตรวจพบจากเอนจิน VirusTotal ยังคงเป็นหลักฐาน
ความปลอดภัยที่มองเห็นได้ แต่คำตัดสิน VirusTotal Code Insight/Palm เป็นเพียงคำแนะนำ
และจะไม่ซ่อน skill ด้วยตัวเอง การรีวิว LLM ของ ClawScan จะเก็บหมายเหตุที่สอดคล้องกับ
วัตถุประสงค์ไว้เป็นแนวทาง ผลการรีวิวระดับปานกลางยังคงมองเห็นได้บนอาร์ติแฟกต์
ขณะที่ตัวกรองน่าสงสัยสงวนไว้สำหรับข้อกังวลจาก LLM ที่มีผลกระทบสูง ผลการตรวจพบ
ที่เป็นอันตราย หรือการตรวจพบจากเอนจิน AV ที่มีหลักฐานสนับสนุน

ผู้ดูแลระบบสามารถยกเลิกการระงับที่เป็นผลบวกเท็จได้:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

การดำเนินการนี้ล้าง `requiresModerationAt` และ `requiresModerationReason`, กู้คืน
skill ที่ถูกซ่อนโดยการระงับระดับผู้ใช้ และเขียนรายการบันทึกการตรวจสอบ
`user.moderation.lift` skill ที่ถูกซ่อนด้วยเหตุผลอื่น หรือที่การสแกนแบบสแตติกของตัวเอง
ยังคงเป็นอันตราย จะยังคงถูกซ่อนอยู่

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบาย ClawHub อาจสูญเสียสิทธิ์การเผยแพร่ การละเมิดร้ายแรง
อาจส่งผลให้บัญชีถูกแบน โทเค็นถูกเพิกถอน เนื้อหาถูกซ่อน หรือรายการถูกลบออก

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้โทเค็น ClawHub API ได้ หากการยืนยันตัวตน
CLI เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ UI บนเว็บเพื่อตรวจสอบ
สถานะบัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อก ให้ติดต่อ
security@openclaw.ai เพื่อขอรีวิวการกู้คืน

## แนวทางสำหรับผู้เผยแพร่

เพื่อลดผลบวกเท็จและเพิ่มความเชื่อมั่นของผู้ใช้:

- ทำให้ชื่อ สรุป แท็ก และบันทึกการเปลี่ยนแปลงถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- เพิ่มหมายเหตุ ClawScan สำหรับผู้เผยแพร่เมื่อรีลีสมีพฤติกรรมที่ผิดปกติแต่ตั้งใจไว้
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้อ่านยาก
- ลิงก์ไปยังซอร์สเมื่อเป็นไปได้
- ใช้การทดลองรันก่อนเผยแพร่ Plugin
- ตอบอย่างชัดเจนหากผู้ใช้หรือผู้กลั่นกรองถามเกี่ยวกับพฤติกรรมของแพ็กเกจ
