# ความปลอดภัย + การดูแลตรวจสอบ

ClawHub เปิดให้เผยแพร่ได้ แต่รายการสาธารณะยังคงต้องผ่านการควบคุมด้านความเชื่อถือ
การสแกน การรายงาน และการดูแลตรวจสอบ เป้าหมายคือความใช้งานได้จริง: ช่วยให้ผู้ใช้
ตรวจสอบสิ่งที่ติดตั้ง มอบเส้นทางกู้คืนให้ผู้เผยแพร่เมื่อเกิดผลบวกลวง
และกันแพ็กเกจที่ไม่เหมาะสมออกจากการค้นพบแบบสาธารณะ

ดูเพิ่มเติมที่ [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## สิ่งที่ผู้ใช้ตรวจสอบได้

ก่อนติดตั้ง skill หรือ Plugin ให้ตรวจสอบรายการ ClawHub ของสิ่งนั้นสำหรับ:

- เจ้าของและการระบุแหล่งที่มา
- เวอร์ชันล่าสุดและ changelog
- ตัวแปรสภาพแวดล้อมหรือสิทธิ์ที่จำเป็น
- metadata ความเข้ากันได้สำหรับ Plugin
- สถานะการสแกนหรือการดูแลตรวจสอบ
- รายงาน ความคิดเห็น ดาว ดาวน์โหลด และสัญญาณการติดตั้งในจุดที่แสดง

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและไว้วางใจเท่านั้น

## สถานะการสแกน

ClawHub อาจแสดงผลลัพธ์การสแกนหรือการดูแลตรวจสอบบนหน้าสาธารณะและข้อมูลวินิจฉัย
ที่เจ้าของมองเห็นได้

ผลลัพธ์ทั่วไปประกอบด้วย:

- `clean`: ไม่พบปัญหาที่ต้องบล็อก
- `suspicious`: release นี้ต้องใช้ความระมัดระวังหรือการตรวจทาน
- `malicious`: release นี้ถือว่าไม่ปลอดภัย
- `pending`: การตรวจสอบยังไม่เสร็จสิ้น
- `held`, `quarantined`, `revoked`, หรือ `hidden`: release นี้ยังไม่พร้อมให้ใช้
  อย่างเต็มรูปแบบบนพื้นผิวการติดตั้งสาธารณะ

ถ้อยคำที่แน่นอนอาจแตกต่างกันไปตามพื้นผิว แต่ความหมายเชิงปฏิบัติเหมือนกัน: หาก
release ถูกพักหรือถูกบล็อก ผู้ใช้ไม่ควรติดตั้งจนกว่าเจ้าของจะแก้ปัญหา
หรือการดูแลตรวจสอบจะกู้คืนให้

## Skills

การสแกน skill จะดูที่บันเดิล skill ที่เผยแพร่ metadata ข้อกำหนดที่ประกาศไว้
และคำสั่งที่น่าสงสัย

ClawHub ให้ความสำคัญเป็นพิเศษกับความไม่ตรงกันระหว่างสิ่งที่ skill ประกาศไว้กับ
สิ่งที่ดูเหมือนจะทำจริง ตัวอย่างเช่น skill ที่อ้างอิง API key ที่จำเป็น
ควรประกาศข้อกำหนดนั้นใน `SKILL.md` เพื่อให้ผู้ใช้เห็นได้ก่อนติดตั้ง

ผลการสแกนอ้างอิงจาก artifact พฤติกรรม provider ที่คาดไว้ เช่น credential API
ที่ประกาศไว้ callback OAuth บน localhost การล้างข้อมูลเมื่อถอนการติดตั้งแบบมีขอบเขต การเข้ารหัส Basic Auth
หรือการอัปโหลดไฟล์ที่ผู้ใช้เลือกไปยัง provider ที่ระบุไว้ จะถูกปฏิบัติแตกต่างจาก
การส่งต่อ credential แบบซ่อน การเข้าถึงไฟล์ส่วนตัวแบบกว้าง
ปลายทางเครือข่ายที่ไม่เกี่ยวข้อง หรือการใช้เบราว์เซอร์อย่างลับ ๆ ในทางที่ผิด

ดู [รูปแบบ Skill](/th/clawhub/skill-format)

## Plugin

release ของ Plugin มี metadata ของแพ็กเกจ การระบุแหล่งที่มา ฟิลด์ความเข้ากันได้
และข้อมูลความสมบูรณ์ของ artifact

OpenClaw ตรวจสอบความเข้ากันได้ก่อนติดตั้ง Plugin ที่โฮสต์บน ClawHub ระเบียนแพ็กเกจ
อาจเปิดเผย metadata digest ด้วย เพื่อให้ OpenClaw ตรวจสอบ artifact ที่ดาวน์โหลดได้
ClawScan รวม metadata env/config ของแพ็กเกจ `openclaw.environment` ที่ประกาศไว้
เมื่อรีวิว release ของ Plugin เพื่อเปรียบเทียบข้อกำหนด runtime ที่ประกาศไว้กับพฤติกรรมที่สังเกตได้

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน Skills แพ็กเกจ และความคิดเห็นได้

รายงานควรเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้ระบบรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

ตัวอย่างรายงาน:

- metadata ที่ชวนเข้าใจผิด
- ข้อกำหนด credential หรือสิทธิ์ที่ไม่ได้ประกาศ
- คำสั่งติดตั้งที่น่าสงสัย
- ความคิดเห็นหลอกลวงหรือการแอบอ้าง
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## หมายเหตุ ClawScan สำหรับผู้เผยแพร่

ผู้เผยแพร่สามารถให้หมายเหตุ ClawScan เพิ่มเติมได้เมื่อเผยแพร่ skill หรือ
Plugin หมายเหตุนี้ให้บริบทแก่ ClawScan สำหรับพฤติกรรมที่อาจดูผิดปกติ
เช่น การเข้าถึงเครือข่าย การเข้าถึงโฮสต์ native หรือ credential
เฉพาะ provider

## การพักเพื่อการดูแลตรวจสอบ

เมื่อ static scanner ทำเครื่องหมาย skill ที่อัปโหลดว่าเป็นอันตราย ผู้เผยแพร่จะถูก
พักเพื่อการดูแลตรวจสอบโดยอัตโนมัติ (`requiresModerationAt` ถูกตั้งไว้บน
ผู้ใช้) การดำเนินการนี้จะซ่อน Skills ทั้งหมดของผู้เผยแพร่ ทำให้การเผยแพร่ในอนาคต
เริ่มต้นแบบซ่อน และสร้างรายการ audit log `user.moderation.auto`

ผลการตรวจพบแบบ static ที่น่าสงสัยจะถูกเก็บไว้เป็นหลักฐานไฟล์/บรรทัดสำหรับผู้ดูแล
แต่จะไม่ซ่อนเนื้อหาหรือตัดสินผลการสแกนสาธารณะด้วยตัวเอง
การอัปโหลดใหม่จะยังคงอยู่ในสถานะ review/pending จนกว่ารีวิว LLM จะได้ข้อสรุป การสแกนแบบ static
จะบล็อกทันทีเฉพาะลายเซ็นที่เป็นอันตรายเท่านั้น ผลตรวจจากเอนจิน VirusTotal
ยังคงมองเห็นได้เป็นหลักฐานความปลอดภัย แต่คำตัดสิน VirusTotal Code Insight/Palm
เป็นเพียงคำแนะนำและไม่ซ่อน Skills ด้วยตัวเอง รีวิว LLM ของ ClawScan
จะเก็บหมายเหตุที่สอดคล้องกับวัตถุประสงค์ไว้เป็นแนวทาง ผลการรีวิวระดับปานกลางยังคงมองเห็นได้บน
artifact ขณะที่ตัวกรอง suspicious สงวนไว้สำหรับข้อกังวลจาก LLM ที่มีผลกระทบสูง
ผลการตรวจพบที่เป็นอันตราย หรือการตรวจพบจากเอนจิน AV ที่มีหลักฐานสนับสนุน

ผู้ดูแลระบบสามารถยกเลิกการพักที่เป็นผลบวกลวงได้:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

การดำเนินการนี้จะล้าง `requiresModerationAt` และ `requiresModerationReason`, กู้คืน
Skills ที่ถูกซ่อนโดยการพักระดับผู้ใช้ และเขียนรายการ audit log `user.moderation.lift`
Skills ที่ถูกซ่อนด้วยเหตุผลอื่น หรือที่การสแกน static ของตัวเองยังคงเป็นอันตราย
จะยังคงถูกซ่อน

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบาย ClawHub อาจสูญเสียสิทธิ์เผยแพร่ การละเมิดร้ายแรง
อาจส่งผลให้ถูกแบนบัญชี เพิกถอนโทเค็น ซ่อนเนื้อหา หรือลบรายการ

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานจะไม่สามารถใช้โทเค็น ClawHub API ได้ หาก auth ของ CLI
เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ UI บนเว็บเพื่อตรวจสอบ
สถานะบัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อก ให้ติดต่อ
security@openclaw.ai เพื่อขอรีวิวการกู้คืน

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลดผลบวกลวงและเพิ่มความเชื่อมั่นของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และ changelog ให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- เพิ่มหมายเหตุ ClawScan ของผู้เผยแพร่เมื่อ release มีพฤติกรรมที่ผิดปกติแต่ตั้งใจให้เป็นเช่นนั้น
- หลีกเลี่ยงคำสั่งติดตั้งที่ถูกทำให้อ่านยาก
- ลิงก์ไปยัง source เมื่อทำได้
- ใช้ dry run ก่อนเผยแพร่ Plugin
- ตอบให้ชัดเจนหากผู้ใช้หรือผู้ดูแลถามเกี่ยวกับพฤติกรรมของแพ็กเกจ
