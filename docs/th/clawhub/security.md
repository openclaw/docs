# ความปลอดภัย + การดูแลตรวจสอบ

ClawHub เปิดให้เผยแพร่ได้ แต่รายการสาธารณะยังคงผ่านการควบคุมด้านความน่าเชื่อถือ
การสแกน การรายงาน และการดูแลตรวจสอบ เป้าหมายคือการใช้งานจริง: ช่วยให้ผู้ใช้
ตรวจสอบสิ่งที่ติดตั้ง ให้ผู้เผยแพร่มีเส้นทางกู้คืนเมื่อเกิดผลบวกลวง
และกันแพ็กเกจที่เป็นอันตรายออกจากการค้นพบแบบสาธารณะ

ดูเพิ่มเติมที่ [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## สิ่งที่ผู้ใช้ตรวจสอบได้

ก่อนติดตั้ง skill หรือ Plugin ให้ตรวจสอบรายการ ClawHub ของสิ่งนั้นสำหรับ:

- เจ้าของและการระบุแหล่งที่มา
- เวอร์ชันล่าสุดและ changelog
- ตัวแปรสภาพแวดล้อมหรือสิทธิ์ที่จำเป็น
- metadata ความเข้ากันได้สำหรับ plugins
- สถานะการสแกนหรือการดูแลตรวจสอบ
- รายงาน ความคิดเห็น ดาว การดาวน์โหลด และสัญญาณการติดตั้งเมื่อมีแสดง

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการสแกน

ClawHub อาจแสดงผลลัพธ์การสแกนหรือการดูแลตรวจสอบบนหน้าสาธารณะและการวินิจฉัย
ที่เจ้าของมองเห็นได้

ผลลัพธ์ทั่วไปมีดังนี้:

- `clean`: ไม่พบปัญหาที่ต้องบล็อก
- `suspicious`: release ต้องใช้ความระมัดระวังหรือต้องตรวจสอบ
- `malicious`: release ถือว่าไม่ปลอดภัย
- `pending`: การตรวจสอบยังไม่เสร็จสิ้น
- `held`, `quarantined`, `revoked`, หรือ `hidden`: release ยังไม่พร้อมใช้งานอย่างเต็มรูปแบบ
  บนพื้นผิวการติดตั้งสาธารณะ

ถ้อยคำที่แน่นอนอาจแตกต่างกันไปตามพื้นผิว แต่ความหมายในทางปฏิบัติเหมือนกัน: หาก
release ถูกระงับหรือถูกบล็อก ผู้ใช้ไม่ควรติดตั้งจนกว่าเจ้าของจะแก้ไข
ปัญหา หรือการดูแลตรวจสอบจะกู้คืน release นั้น

## Skills

การสแกน skill จะดูที่ชุด skill ที่เผยแพร่ metadata ข้อกำหนดที่ประกาศไว้
และคำสั่งที่น่าสงสัย

ClawHub ให้ความสำคัญเป็นพิเศษกับความไม่ตรงกันระหว่างสิ่งที่ skill ประกาศกับ
สิ่งที่ดูเหมือนว่ามันทำ ตัวอย่างเช่น skill ที่อ้างถึง API key ที่จำเป็น
ควรประกาศข้อกำหนดนั้นใน `SKILL.md` เพื่อให้ผู้ใช้เห็นได้ก่อน
ติดตั้ง

ผลการสแกนอิงจาก artifact พฤติกรรมผู้ให้บริการที่คาดหมาย เช่น ข้อมูลรับรอง API
ที่ประกาศไว้, localhost OAuth callbacks, การล้างข้อมูลเมื่อถอนการติดตั้งตามขอบเขต,
การเข้ารหัส Basic Auth, หรือการอัปโหลดไฟล์ที่ผู้ใช้เลือกไปยังผู้ให้บริการที่ระบุ
จะได้รับการปฏิบัติแตกต่างจากการส่งต่อข้อมูลรับรองแบบซ่อน การเข้าถึงไฟล์ส่วนตัว
แบบกว้าง ปลายทางเครือข่ายที่ไม่เกี่ยวข้อง หรือการใช้เบราว์เซอร์ในทางที่ผิดแบบลับๆ

ดู [รูปแบบ Skill](/th/clawhub/skill-format)

## Plugins

release ของ Plugin มี package metadata การระบุแหล่งที่มา ฟิลด์ความเข้ากันได้
และข้อมูลความถูกต้องสมบูรณ์ของ artifact

OpenClaw ตรวจสอบความเข้ากันได้ก่อนติดตั้ง plugins ที่โฮสต์บน ClawHub ระเบียน package
อาจแสดง digest metadata ด้วย เพื่อให้ OpenClaw ตรวจสอบ artifact ที่ดาวน์โหลดมาได้
ClawScan รวม metadata env/config ของ package `openclaw.environment` ที่ประกาศไว้
เมื่อตรวจสอบ release ของ plugin เพื่อเปรียบเทียบข้อกำหนด runtime ที่ประกาศไว้
กับพฤติกรรมที่สังเกตพบ

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน skills, packages และความคิดเห็นได้

รายงานควรเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้การรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

ตัวอย่างรายงาน:

- metadata ที่ทำให้เข้าใจผิด
- ข้อกำหนดด้านข้อมูลรับรองหรือสิทธิ์ที่ไม่ได้ประกาศ
- คำสั่งติดตั้งที่น่าสงสัย
- ความคิดเห็นหลอกลวงหรือการแอบอ้างบุคคล
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## หมายเหตุ ClawScan ของผู้เผยแพร่

ผู้เผยแพร่สามารถให้หมายเหตุ ClawScan เพิ่มเติมได้เมื่อเผยแพร่ skill หรือ
plugin หมายเหตุนี้ให้บริบทแก่ ClawScan สำหรับพฤติกรรมที่อาจดู
ผิดปกติ เช่น การเข้าถึงเครือข่าย การเข้าถึง native host หรือข้อมูลรับรอง
เฉพาะผู้ให้บริการ

## การระงับเพื่อดูแลตรวจสอบ

เมื่อ static scanner ทำเครื่องหมาย skill ที่อัปโหลดว่าเป็นอันตราย ผู้เผยแพร่จะถูก
วางไว้ภายใต้การระงับเพื่อดูแลตรวจสอบโดยอัตโนมัติ (`requiresModerationAt` ตั้งค่าไว้บน
user) สิ่งนี้จะซ่อน skills ทั้งหมดของผู้เผยแพร่ ทำให้การเผยแพร่ในอนาคต
เริ่มต้นเป็นแบบซ่อน และสร้างรายการ audit log `user.moderation.auto`

ผลการค้นพบแบบ static ที่น่าสงสัยจะถูกเก็บไว้เป็นหลักฐานระดับไฟล์/บรรทัดสำหรับผู้ดูแล
แต่ผลเหล่านั้นไม่ได้ซ่อนเนื้อหาหรือตัดสินผลการสแกนสาธารณะด้วยตัวเอง
การอัปโหลดใหม่จะยังอยู่ในสถานะ review/pending จนกว่า LLM review จะสรุปผล การสแกนแบบ static
จะบล็อกทันทีเฉพาะลายเซ็นที่เป็นอันตรายเท่านั้น การพบโดย engine ของ VirusTotal
ยังคงมองเห็นได้ในฐานะหลักฐานด้านความปลอดภัย แต่ verdicts ของ VirusTotal Code Insight/Palm
เป็นคำแนะนำและไม่ได้ซ่อน skills ด้วยตัวเอง รีวิว ClawScan LLM
จะเก็บหมายเหตุที่สอดคล้องกับวัตถุประสงค์ไว้เป็นแนวทาง ผลการตรวจสอบระดับ medium จะยังมองเห็นได้บน
artifact ขณะที่ตัวกรอง suspicious ถูกสงวนไว้สำหรับข้อกังวลจาก LLM
ที่มีผลกระทบสูง ผลการค้นพบที่เป็นอันตราย หรือการตรวจพบจาก AV-engine ที่มีหลักฐานสนับสนุน

ผู้ดูแลระบบสามารถยกเลิกการระงับที่เป็นผลบวกลวงได้:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

สิ่งนี้จะล้าง `requiresModerationAt` และ `requiresModerationReason`, กู้คืน
skills ที่ถูกซ่อนโดยการระงับระดับ user และเขียนรายการ audit log `user.moderation.lift`
skills ที่ถูกซ่อนด้วยเหตุผลอื่น หรือที่ static scan ของตัวเองยังคงเป็น
malicious จะยังคงถูกซ่อนอยู่

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบาย ClawHub อาจสูญเสียสิทธิ์การเผยแพร่ การละเมิดร้ายแรง
อาจส่งผลให้ถูกแบนบัญชี เพิกถอน token ซ่อนเนื้อหา หรือลบ
รายการ

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้ ClawHub API tokens ได้ หาก CLI auth
เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ web UI เพื่อตรวจสอบสถานะ
บัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อก ให้ติดต่อ
security@openclaw.ai เพื่อขอการตรวจสอบการกู้คืน

## แนวทางสำหรับผู้เผยแพร่

เพื่อลดผลบวกลวงและเพิ่มความเชื่อมั่นของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และ changelogs ให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- เพิ่มหมายเหตุ ClawScan ของผู้เผยแพร่เมื่อ release มีพฤติกรรมที่ผิดปกติแต่ตั้งใจไว้
- หลีกเลี่ยงคำสั่งติดตั้งที่ถูกทำให้อ่านยาก
- ลิงก์ไปยัง source เมื่อเป็นไปได้
- ใช้ dry runs ก่อนเผยแพร่ plugins
- ตอบกลับอย่างชัดเจนหากผู้ใช้หรือผู้ดูแลถามเกี่ยวกับพฤติกรรมของ package
