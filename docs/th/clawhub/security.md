# ความปลอดภัย

สามารถรายงานปัญหาด้านความปลอดภัยของ ClawHub ผ่าน GitHub Security Advisories สำหรับ
`openclaw/clawhub`

ใช้ GitHub Security Advisories สำหรับช่องโหว่ใน ClawHub เอง รายงานคำแนะนำด้านความปลอดภัยของ
ClawHub ที่ดีประกอบด้วยบั๊กใน:

- เว็บไซต์, API หรือ CLI ของ ClawHub
- การเผยแพร่รีจิสทรี, การดาวน์โหลด, การติดตั้ง หรือความสมบูรณ์ของอาร์ติแฟกต์
- การยืนยันตัวตน, การอนุญาต หรือโทเค็น API
- การสแกน, การกลั่นกรอง หรือการจัดการรายงาน

อย่าใช้คำแนะนำด้านความปลอดภัยของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ Skills หรือ
Plugin ของบุคคลที่สามเอง ให้รายงานช่องโหว่เหล่านั้นโดยตรงต่อผู้เผยแพร่หรือรีโพซิทอรีต้นทาง
ที่ลิงก์จากรายการใน ClawHub

## การเปิดเผยช่องโหว่

เนื่องจาก ClawHub เป็นแอปพลิเคชันคลาวด์แบบโฮสต์ ช่องโหว่ของบริการ ClawHub
จึงไม่ถูกเปิดเผยต่อสาธารณะโดยค่าเริ่มต้น ช่องโหว่จะถูกเปิดเผยต่อสาธารณะเมื่อมี
หลักฐานว่ามีผลกระทบจริงต่อผู้ใช้ หรือเมื่อผู้ใช้จำเป็นต้องดำเนินการ

ตัวอย่างของผลกระทบจริงต่อผู้ใช้ ได้แก่ การยืนยันว่ามีการโจมตีสำเร็จ, การเปิดเผยข้อมูลผู้ใช้
หรือข้อมูลลับ, เนื้อหาประสงค์ร้ายเข้าถึงผู้ใช้เนื่องจากความล้มเหลวของแพลตฟอร์ม
หรือปัญหาใด ๆ ที่กำหนดให้ผู้ใช้ต้องหมุนเวียนข้อมูลประจำตัว, อัปเดตซอฟต์แวร์ในเครื่อง
หรือดำเนินการป้องกันอื่น ๆ

ช่องโหว่ในซอฟต์แวร์ที่ผู้ใช้ติดตั้งจะถูกเปิดเผยต่อสาธารณะ เช่น
แพ็กเกจ ClawHub CLI, ไบนารี, ไลบรารี หรืออาร์ติแฟกต์เผยแพร่อื่น ๆ ที่ผู้ใช้
ต้องอัปเดตในเครื่อง
