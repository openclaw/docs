# ความปลอดภัย + การดูแลควบคุม

ClawHub เปิดให้เผยแพร่ได้ แต่รายการสาธารณะยังคงผ่านการควบคุมด้านความน่าเชื่อถือ
การสแกน การรายงาน และการดูแลควบคุม เป้าหมายคือความเป็นจริงในการใช้งาน: ช่วยให้ผู้ใช้
ตรวจสอบสิ่งที่ติดตั้ง ให้เส้นทางกู้คืนแก่ผู้เผยแพร่เมื่อเกิดผลบวกลวง
และกันแพ็กเกจที่เป็นอันตรายออกจากการค้นพบสาธารณะ

ดูเพิ่มเติมที่ [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## สิ่งที่ผู้ใช้ตรวจสอบได้

ก่อนติดตั้ง skill หรือ plugin ให้ตรวจสอบรายการ ClawHub ของรายการนั้นสำหรับ:

- การระบุเจ้าของและแหล่งที่มา
- เวอร์ชันล่าสุดและบันทึกการเปลี่ยนแปลง
- ตัวแปรสภาพแวดล้อมหรือสิทธิ์ที่ต้องใช้
- เมทาดาทาความเข้ากันได้สำหรับ plugins
- สถานะการสแกนหรือการดูแลควบคุม
- รายงาน ความคิดเห็น ดาว จำนวนดาวน์โหลด และสัญญาณการติดตั้งในจุดที่แสดง

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการสแกน

ClawHub อาจแสดงผลลัพธ์การสแกนหรือการดูแลควบคุมบนหน้าสาธารณะและข้อมูลวินิจฉัย
ที่เจ้าของมองเห็นได้

ผลลัพธ์ทั่วไปได้แก่:

- `clean`: ไม่พบปัญหาที่ต้องบล็อก
- `suspicious`: รีลีสนี้ต้องใช้ความระมัดระวังหรือการตรวจสอบ
- `malicious`: รีลีสนี้ถือว่าไม่ปลอดภัย
- `pending`: การตรวจสอบยังไม่เสร็จสิ้น
- `held`, `quarantined`, `revoked`, หรือ `hidden`: รีลีสนี้ยังไม่พร้อมใช้งานอย่างเต็มที่
  บนช่องทางติดตั้งสาธารณะ

ถ้อยคำที่แน่นอนอาจแตกต่างกันไปตามส่วนติดต่อ แต่ความหมายเชิงปฏิบัติเหมือนกัน: หาก
รีลีสถูกพักหรือถูกบล็อก ผู้ใช้ไม่ควรติดตั้งจนกว่าเจ้าของจะแก้ปัญหา
หรือการดูแลควบคุมจะกู้คืนรีลีสนั้น

## Skills

การสแกน skill จะตรวจดูบันเดิล skill ที่เผยแพร่ เมทาดาทา ข้อกำหนดที่ประกาศไว้
และคำสั่งที่น่าสงสัย

ClawHub ให้ความสนใจเป็นพิเศษกับความไม่ตรงกันระหว่างสิ่งที่ skill ประกาศไว้กับ
สิ่งที่ดูเหมือนว่าจะทำ ตัวอย่างเช่น skill ที่อ้างอิง API key ที่ต้องใช้
ควรประกาศข้อกำหนดนั้นใน `SKILL.md` เพื่อให้ผู้ใช้เห็นได้ก่อน
ติดตั้ง

ผลการสแกนอิงตามอาร์ติแฟกต์ พฤติกรรมที่คาดไว้ของผู้ให้บริการ เช่น ข้อมูลประจำตัว API
ที่ประกาศไว้, callback OAuth ของ localhost, การล้างข้อมูลเมื่อถอนการติดตั้งแบบจำกัดขอบเขต, การเข้ารหัส Basic Auth
หรือการอัปโหลดไฟล์ที่ผู้ใช้เลือกไปยังผู้ให้บริการที่ระบุไว้ จะถูกปฏิบัติ
แตกต่างจากการส่งต่อข้อมูลประจำตัวอย่างซ่อนเร้น การเข้าถึงไฟล์ส่วนตัวแบบกว้าง
ปลายทางเครือข่ายที่ไม่เกี่ยวข้อง หรือการใช้เบราว์เซอร์ในทางที่ผิดแบบแอบแฝง

ดู [รูปแบบ skill](/th/clawhub/skill-format)

## Plugins

รีลีส Plugin มีเมทาดาทาแพ็กเกจ การระบุแหล่งที่มา ฟิลด์ความเข้ากันได้
และข้อมูลความสมบูรณ์ของอาร์ติแฟกต์

OpenClaw ตรวจสอบความเข้ากันได้ก่อนติดตั้ง plugins ที่โฮสต์บน ClawHub ระเบียนแพ็กเกจ
อาจเปิดเผยเมทาดาทา digest ด้วย เพื่อให้ OpenClaw ตรวจสอบอาร์ติแฟกต์
ที่ดาวน์โหลดมาได้ ClawScan รวมเมทาดาทา env/config ของแพ็กเกจ `openclaw.environment`
ที่ประกาศไว้เมื่อตรวจสอบรีลีส plugin เพื่อเปรียบเทียบข้อกำหนดรันไทม์ที่ประกาศไว้
กับพฤติกรรมที่สังเกตพบ

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน skills, packages และความคิดเห็นได้

รายงานควรเจาะจงและนำไปดำเนินการได้ การใช้การรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

ตัวอย่างรายงาน:

- เมทาดาทาที่ทำให้เข้าใจผิด
- ข้อกำหนดข้อมูลประจำตัวหรือสิทธิ์ที่ไม่ได้ประกาศ
- คำสั่งติดตั้งที่น่าสงสัย
- ความคิดเห็นหลอกลวงหรือการแอบอ้างตัวตน
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## รายงานเรื่องความไม่สุจริตหรือเครื่องหมายการค้า

ClawHub ใช้กระบวนการรายงานและการดูแลควบคุมโดยเจ้าหน้าที่ชุดเดียวกันสำหรับการลงทะเบียน
โดยไม่สุจริต การแอบอ้างตัวตน และข้อพิพาทที่เกี่ยวข้องกับเครื่องหมายการค้า รายงานเหล่านี้ต้องมี
บริบทเพียงพอให้เจ้าหน้าที่ระบุผู้อ้างสิทธิ์ รายการที่มีข้อพิพาท และ
การดำเนินการที่ร้องขอได้

ให้ระบุ:

- URL skill หรือ package ตามมาตรฐานของ ClawHub และ handle ของเจ้าของ
- เครื่องหมายการค้า โปรเจกต์ บริษัท หรือชื่อผลิตภัณฑ์ที่เป็นประเด็น
- หลักฐานสาธารณะเกี่ยวกับความเป็นเจ้าของหรืออำนาจของผู้อ้างสิทธิ์
- เหตุผลที่เจ้าของปัจจุบันไม่ได้รับอนุญาตให้เผยแพร่ภายใต้ชื่อนั้น
- การดำเนินการที่ร้องขอ เช่น ซ่อนระหว่างรอการตรวจสอบ โอนความเป็นเจ้าของ เปลี่ยนชื่อ
  หรือลบ

อย่าใส่ความลับส่วนตัวหรือเอกสารทางกฎหมายที่ละเอียดอ่อนในรายงานสาธารณะ ให้เปิด
issue บน GitHub พร้อมหลักฐานที่ไม่ละเอียดอ่อน และขอเส้นทางส่งต่อแบบส่วนตัว
จากผู้ดูแลเมื่อจำเป็น

## การอุทธรณ์และการสแกนซ้ำ

เจ้าของสามารถขอการสแกนซ้ำเมื่อเชื่อว่า skill หรือ package ถูกพัก
หรือถูกตั้งค่าสถานะอย่างไม่ถูกต้อง ผู้ดูแลควบคุมแพลตฟอร์มและผู้ดูแลระบบสามารถขอสแกนซ้ำสำหรับ
skill หรือ package ใดก็ได้ขณะจัดการรายงานหรือคำขอสนับสนุน:

```bash
clawhub skill rescan <slug>
clawhub package rescan <name>
```

สำหรับเนื้อหาที่ถูกดูแลควบคุม เจ้าของอาจส่งอุทธรณ์ได้จากส่วนติดต่อ ClawHub
ที่เจ้าของมองเห็นได้ การอุทธรณ์ควรอธิบายว่าอะไรเปลี่ยนแปลง หรือเหตุใด
การตั้งค่าสถานะจึงไม่ถูกต้อง

## การพักเพื่อการดูแลควบคุม

เมื่อสแกนเนอร์แบบสแตติกตั้งค่าสถานะ skill ที่อัปโหลดว่าเป็นอันตราย ผู้เผยแพร่จะ
ถูกนำเข้าสู่การพักเพื่อการดูแลควบคุมโดยอัตโนมัติ (`requiresModerationAt` ถูกตั้งบน
ผู้ใช้) การดำเนินการนี้จะซ่อน skills ทั้งหมดของผู้เผยแพร่ ทำให้การเผยแพร่ในอนาคต
เริ่มต้นแบบซ่อน และสร้างรายการบันทึกการตรวจสอบ `user.moderation.auto`

ผลการค้นพบที่น่าสงสัยจากการสแกนแบบสแตติกจะถูกเก็บไว้เป็นหลักฐานไฟล์/บรรทัดสำหรับผู้ดูแล
แต่ผลดังกล่าวจะไม่ซ่อนเนื้อหาหรือตัดสินผลการสแกนสาธารณะด้วยตัวเอง
การอัปโหลดใหม่จะยังอยู่ในสถานะตรวจสอบ/รอดำเนินการจนกว่าการตรวจสอบ VirusTotal และ LLM
จะสรุปผล การสแกนแบบสแตติกจะบล็อกทันทีเฉพาะ signature ที่เป็นอันตรายเท่านั้น
การตรวจสอบ ClawScan LLM จะเก็บบันทึกที่สอดคล้องกับวัตถุประสงค์ไว้เป็นแนวทาง โดยจะคืน
คำตัดสิน Review/suspicious เฉพาะเมื่อการตรวจสอบแบบมีโครงสร้างมีข้อกังวลที่มีนัยสำคัญ

ผู้ดูแลระบบสามารถยกเลิกการพักที่เป็นผลบวกลวงได้:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

การดำเนินการนี้ล้าง `requiresModerationAt` และ `requiresModerationReason`, กู้คืน
skills ที่ถูกซ่อนโดยการพักระดับผู้ใช้ และเขียนรายการบันทึกการตรวจสอบ `user.moderation.lift`
skills ที่ถูกซ่อนด้วยเหตุผลอื่น หรือยังมีผลการสแกนแบบสแตติกของตัวเอง
เป็นอันตราย จะยังคงถูกซ่อนไว้

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบาย ClawHub อาจสูญเสียสิทธิ์การเผยแพร่ การใช้งานในทางที่ผิดอย่างรุนแรง
อาจทำให้บัญชีถูกแบน โทเค็นถูกเพิกถอน เนื้อหาถูกซ่อน หรือรายการ
ถูกลบได้

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้โทเค็น ClawHub API ได้ หากการยืนยันตัวตน CLI
เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ web UI เพื่อตรวจสอบสถานะ
บัญชี หรือติดต่อผู้ดูแลผ่านช่องทางสนับสนุนโปรเจกต์ที่คาดไว้

## แนวทางสำหรับผู้เผยแพร่

เพื่อลดผลบวกลวงและเพิ่มความเชื่อมั่นของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และบันทึกการเปลี่ยนแปลงให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่ต้องใช้
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้คลุมเครือ
- ลิงก์ไปยังแหล่งที่มาเมื่อเป็นไปได้
- ใช้การรันแบบ dry run ก่อนเผยแพร่ plugins
- ตอบอย่างชัดเจนหากผู้ใช้หรือผู้ดูแลควบคุมถามเกี่ยวกับพฤติกรรมของแพ็กเกจ
