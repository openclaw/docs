# ความปลอดภัย

สามารถรายงานปัญหาด้านความปลอดภัยของ ClawHub ผ่าน GitHub Security Advisories สำหรับ
`openclaw/clawhub`

ใช้ GitHub Security Advisories สำหรับช่องโหว่ในตัว ClawHub เอง รายงานคำแนะนำด้านความปลอดภัยของ
ClawHub ที่ดีควรรวมบั๊กใน:

- เว็บไซต์, API หรือ CLI ของ ClawHub
- การเผยแพร่รีจิสทรี, การดาวน์โหลด, การติดตั้ง หรือความสมบูรณ์ของอาร์ติแฟกต์
- การยืนยันตัวตน, การอนุญาต หรือโทเค็น API
- การสแกน, การดูแลกลั่นกรอง หรือการจัดการรายงาน

อย่าใช้คำแนะนำด้านความปลอดภัยของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของสกิลหรือ
plugin บุคคลที่สามเอง ให้รายงานโดยตรงต่อผู้เผยแพร่หรือที่เก็บซอร์ส
ที่ลิงก์จากรายการใน ClawHub

## การเปิดเผยช่องโหว่

เนื่องจาก ClawHub เป็นแอปพลิเคชันคลาวด์ที่โฮสต์อยู่ ช่องโหว่ของบริการ ClawHub
จึงจะไม่ถูกเปิดเผยต่อสาธารณะโดยค่าเริ่มต้น ช่องโหว่จะถูกเปิดเผยต่อสาธารณะเมื่อมี
หลักฐานว่ามีผลกระทบต่อผู้ใช้จริง หรือเมื่อผู้ใช้จำเป็นต้องดำเนินการ

ตัวอย่างของผลกระทบต่อผู้ใช้จริงรวมถึงการยืนยันว่าถูกโจมตีจริง, การเปิดเผยข้อมูลผู้ใช้
หรือความลับ, เนื้อหาที่เป็นอันตรายเข้าถึงผู้ใช้เนื่องจากความล้มเหลวของแพลตฟอร์ม
หรือปัญหาใดๆ ที่กำหนดให้ผู้ใช้ต้องหมุนเวียนข้อมูลประจำตัว, อัปเดตซอฟต์แวร์ภายในเครื่อง หรือ
ดำเนินการป้องกันอื่นๆ

ช่องโหว่ในซอฟต์แวร์ที่ผู้ใช้ติดตั้งจะถูกเปิดเผยต่อสาธารณะ เช่น
แพ็กเกจ CLI ของ ClawHub, ไบนารี, ไลบรารี หรืออาร์ติแฟกต์รุ่นเผยแพร่อื่นๆ ที่ผู้ใช้
จำเป็นต้องอัปเดตภายในเครื่อง
