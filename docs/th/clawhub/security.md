# ความปลอดภัย

สามารถรายงานปัญหาความปลอดภัยของ ClawHub ผ่าน GitHub Security Advisories สำหรับ
`openclaw/clawhub`

ใช้ GitHub Security Advisories สำหรับช่องโหว่ในตัว ClawHub เอง รายงาน advisory ของ
ClawHub ที่ดีรวมถึงบั๊กใน:

- เว็บไซต์, API หรือ CLI ของ ClawHub
- การเผยแพร่ registry, การดาวน์โหลด, การติดตั้ง หรือความถูกต้องสมบูรณ์ของ artifact
- การยืนยันตัวตน, การอนุญาต หรือโทเค็น API
- การสแกน, การกลั่นกรอง หรือการจัดการรายงาน

อย่าใช้ advisory ของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ skill หรือ
Plugin ของบุคคลที่สาม ให้รายงานโดยตรงไปยังผู้เผยแพร่หรือ repository ต้นทาง
ที่ลิงก์จากรายการใน ClawHub

## การเปิดเผยช่องโหว่

เนื่องจาก ClawHub เป็นแอปพลิเคชันคลาวด์ที่โฮสต์อยู่ ช่องโหว่ของบริการ ClawHub
จึงไม่ได้ถูกเปิดเผยต่อสาธารณะโดยค่าเริ่มต้น ช่องโหว่เหล่านี้จะถูกเปิดเผยต่อสาธารณะเมื่อมี
หลักฐานว่ามีผลกระทบต่อผู้ใช้จริง หรือเมื่อผู้ใช้จำเป็นต้องดำเนินการ

ตัวอย่างของผลกระทบต่อผู้ใช้จริง ได้แก่ การยืนยันว่ามีการโจมตีเกิดขึ้นแล้ว การเปิดเผยข้อมูลผู้ใช้
หรือความลับ เนื้อหาประสงค์ร้ายเข้าถึงผู้ใช้เนื่องจากความล้มเหลวของแพลตฟอร์ม
หรือปัญหาใดๆ ที่ทำให้ผู้ใช้ต้องหมุนเวียนข้อมูลประจำตัว อัปเดตซอฟต์แวร์ในเครื่อง หรือ
ดำเนินการป้องกันอื่นๆ

ช่องโหว่ในซอฟต์แวร์ที่ผู้ใช้ติดตั้งจะถูกเปิดเผยต่อสาธารณะ เช่น
แพ็กเกจ CLI, ไบนารี, ไลบรารี หรือ artifact รุ่นเผยแพร่อื่นๆ ของ ClawHub ที่ผู้ใช้
จำเป็นต้องอัปเดตในเครื่อง

## หน้าที่เกี่ยวข้อง

สำหรับป้ายกำกับการตรวจสอบระหว่างติดตั้ง ระดับความเสี่ยง ข้อค้นพบ และการตีความ โปรดดู
[การตรวจสอบความปลอดภัย](/clawhub/security-audits)

โปรดดู [การกลั่นกรองและความปลอดภัยของบัญชี](/clawhub/moderation)
