# ความปลอดภัย + การกลั่นกรอง

ClawHub เปิดให้เผยแพร่ได้ แต่รายการสาธารณะยังคงผ่านการควบคุมด้านความน่าเชื่อถือ
การสแกน การรายงาน และการกลั่นกรอง เป้าหมายเป็นเชิงปฏิบัติ: ช่วยให้ผู้ใช้
ตรวจสอบสิ่งที่ติดตั้ง ให้ผู้เผยแพร่มีเส้นทางกู้คืนสำหรับผลบวกเท็จ
และกันแพ็กเกจที่ละเมิดออกจากการค้นพบแบบสาธารณะ

ดูเพิ่มเติมที่ [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## สิ่งที่ผู้ใช้ตรวจสอบได้

ก่อนติดตั้ง skill หรือ plugin ให้ตรวจสอบรายการ ClawHub ของรายการนั้นสำหรับ:

- เจ้าของและการระบุแหล่งที่มา
- เวอร์ชันล่าสุดและบันทึกการเปลี่ยนแปลง
- ตัวแปรสภาพแวดล้อมหรือสิทธิ์ที่ต้องใช้
- เมตาดาต้าความเข้ากันได้สำหรับ Plugin
- สถานะการสแกนหรือการกลั่นกรอง
- รายงาน ความคิดเห็น ดาว ยอดดาวน์โหลด และสัญญาณการติดตั้งเมื่อมีแสดง

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการสแกน

ClawHub อาจแสดงผลลัพธ์การสแกนหรือการกลั่นกรองบนหน้าสาธารณะและ
การวินิจฉัยที่เจ้าของมองเห็นได้

ผลลัพธ์ทั่วไปได้แก่:

- `clean`: ไม่พบปัญหาที่ต้องบล็อก
- `suspicious`: รุ่นนี้ต้องใช้ความระมัดระวังหรือการตรวจสอบ
- `malicious`: รุ่นนี้ถือว่าไม่ปลอดภัย
- `pending`: การตรวจสอบยังไม่เสร็จสิ้น
- `held`, `quarantined`, `revoked`, หรือ `hidden`: รุ่นนี้ยังไม่พร้อมใช้งานเต็มรูปแบบ
  บนพื้นผิวการติดตั้งสาธารณะ

ข้อความที่แน่นอนอาจแตกต่างกันตามพื้นผิว แต่ความหมายเชิงปฏิบัติเหมือนกัน: หาก
รุ่นถูกพักไว้หรือถูกบล็อก ผู้ใช้ไม่ควรติดตั้งจนกว่าเจ้าของจะแก้ไข
ปัญหา หรือการกลั่นกรองจะคืนสถานะให้

## Skills

การสแกน skill จะดูบันเดิล skill ที่เผยแพร่ เมตาดาต้า ข้อกำหนดที่ประกาศไว้
และคำสั่งที่น่าสงสัย

ClawHub ให้ความสำคัญเป็นพิเศษกับความไม่ตรงกันระหว่างสิ่งที่ skill ประกาศกับ
สิ่งที่ดูเหมือนว่ามันทำ ตัวอย่างเช่น skill ที่อ้างถึง API key ที่จำเป็น
ควรประกาศข้อกำหนดนั้นใน `SKILL.md` เพื่อให้ผู้ใช้เห็นได้ก่อน
ติดตั้ง

ผลการสแกนอิงตามอาร์ติแฟกต์ พฤติกรรมของผู้ให้บริการที่คาดไว้ เช่น ข้อมูลประจำตัว
API ที่ประกาศไว้, localhost OAuth callbacks, การล้างข้อมูลเมื่อถอนการติดตั้งตามขอบเขต, การเข้ารหัส Basic Auth
หรือการอัปโหลดไฟล์ที่ผู้ใช้เลือกไปยังผู้ให้บริการที่ระบุไว้ จะได้รับการปฏิบัติ
ต่างจากการส่งต่อข้อมูลประจำตัวแบบซ่อนเร้น การเข้าถึงไฟล์ส่วนตัวอย่างกว้าง
ปลายทางเครือข่ายที่ไม่เกี่ยวข้อง หรือการใช้งานเบราว์เซอร์แบบลอบเร้น

ดู [รูปแบบ Skill](/th/clawhub/skill-format)

## Plugin

รุ่นของ Plugin จะรวมเมตาดาต้าแพ็กเกจ การระบุแหล่งที่มา ฟิลด์ความเข้ากันได้
และข้อมูลความสมบูรณ์ของอาร์ติแฟกต์

OpenClaw ตรวจสอบความเข้ากันได้ก่อนติดตั้ง Plugin ที่โฮสต์บน ClawHub ระเบียนแพ็กเกจ
อาจเปิดเผยเมตาดาต้า digest ด้วย เพื่อให้ OpenClaw ตรวจสอบอาร์ติแฟกต์
ที่ดาวน์โหลดมาได้ ClawScan รวมเมตาดาต้า env/config ของแพ็กเกจ `openclaw.environment` ที่ประกาศไว้
เมื่อรีวิวรุ่นของ Plugin เพื่อเปรียบเทียบข้อกำหนดรันไทม์ที่ประกาศไว้
กับพฤติกรรมที่สังเกตได้

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน Skills, แพ็กเกจ และความคิดเห็นได้

รายงานควรเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้การรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

ตัวอย่างรายงาน:

- เมตาดาต้าที่ทำให้เข้าใจผิด
- ข้อกำหนดด้านข้อมูลประจำตัวหรือสิทธิ์ที่ไม่ได้ประกาศ
- คำแนะนำการติดตั้งที่น่าสงสัย
- ความคิดเห็นหลอกลวงหรือการแอบอ้าง
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## รายงานโดยไม่สุจริตหรือเกี่ยวกับเครื่องหมายการค้า

ClawHub ใช้ไปป์ไลน์รายงานและการกลั่นกรองโดยทีมงานเดียวกันสำหรับการลงทะเบียน
โดยไม่สุจริต การแอบอ้าง และข้อพิพาทที่เกี่ยวข้องกับเครื่องหมายการค้า รายงานเหล่านี้ต้องมี
บริบทเพียงพอให้ทีมงานระบุผู้ร้องเรียน รายการที่มีข้อพิพาท และ
การดำเนินการที่ร้องขอได้

ระบุ:

- URL ของ skill หรือแพ็กเกจ ClawHub มาตรฐาน และ handle ของเจ้าของ
- เครื่องหมายการค้า โปรเจกต์ บริษัท หรือชื่อผลิตภัณฑ์ที่เป็นประเด็น
- หลักฐานสาธารณะเกี่ยวกับความเป็นเจ้าของหรืออำนาจของผู้ร้องเรียน
- เหตุผลที่เจ้าของปัจจุบันไม่ได้รับอนุญาตให้เผยแพร่ภายใต้ชื่อนั้น
- การดำเนินการที่ร้องขอ เช่น ซ่อนระหว่างรอรีวิว โอนความเป็นเจ้าของ เปลี่ยนชื่อ
  หรือลบ

อย่าใส่ความลับส่วนตัวหรือเอกสารทางกฎหมายที่ละเอียดอ่อนในรายงานสาธารณะ เปิด
GitHub issue พร้อมหลักฐานที่ไม่ละเอียดอ่อน และขอเส้นทางส่งต่อแบบส่วนตัวจากผู้ดูแล
เมื่อจำเป็น

## การอุทธรณ์และการสแกนซ้ำ

เจ้าของสามารถขอสแกนซ้ำเมื่อเชื่อว่า skill หรือแพ็กเกจถูกพักไว้
หรือถูกทำเครื่องหมายอย่างไม่ถูกต้อง ผู้กลั่นกรองและผู้ดูแลแพลตฟอร์มสามารถขอสแกนซ้ำสำหรับ
skill หรือแพ็กเกจใดก็ได้ระหว่างจัดการรายงานหรือคำขอสนับสนุน:

```bash
clawhub skill rescan <slug>
clawhub package rescan <name>
```

สำหรับเนื้อหาที่ถูกกลั่นกรอง เจ้าของอาจส่งอุทธรณ์ได้จากพื้นผิว ClawHub
ที่เจ้าของมองเห็นได้ การอุทธรณ์ควรอธิบายว่าสิ่งใดเปลี่ยนไป หรือเหตุใด
ธงจึงไม่ถูกต้อง

## การพักระงับเพื่อการกลั่นกรอง

เมื่อเครื่องสแกนแบบสแตติกทำเครื่องหมาย skill ที่อัปโหลดว่าเป็นอันตราย ผู้เผยแพร่จะถูก
วางไว้ภายใต้การพักระงับเพื่อการกลั่นกรองโดยอัตโนมัติ (`requiresModerationAt` ตั้งค่าไว้บน
ผู้ใช้) ซึ่งจะซ่อน Skills ทั้งหมดของผู้เผยแพร่ ทำให้การเผยแพร่ในอนาคต
เริ่มต้นแบบซ่อน และสร้างรายการ audit log `user.moderation.auto`

ผลการค้นพบแบบสแตติกที่น่าสงสัยจะถูกเก็บไว้เป็นหลักฐานไฟล์/บรรทัดสำหรับผู้กลั่นกรอง
แต่จะไม่ซ่อนเนื้อหาหรือตัดสินผลการสแกนสาธารณะด้วยตัวเอง
การอัปโหลดใหม่จะคงอยู่ในสถานะรีวิว/รอดำเนินการจนกว่าการรีวิว LLM จะเสร็จสิ้น การสแกนแบบสแตติก
จะบล็อกทันทีเฉพาะลายเซ็นที่เป็นอันตรายเท่านั้น ผลการตรวจพบจากเอนจิน VirusTotal
ยังคงแสดงเป็นหลักฐานความปลอดภัย แต่คำตัดสินของ VirusTotal Code Insight/Palm
เป็นเพียงคำแนะนำและไม่ซ่อน Skills ด้วยตัวเอง การรีวิว LLM ของ ClawScan
จะเก็บบันทึกที่สอดคล้องกับวัตถุประสงค์ไว้เป็นคำแนะนำ ผลการรีวิวระดับกลางยังคงแสดงบน
อาร์ติแฟกต์ ขณะที่ตัวกรองที่น่าสงสัยสงวนไว้สำหรับข้อกังวลจาก LLM
ที่มีผลกระทบสูง ผลการค้นพบที่เป็นอันตราย หรือการตรวจพบจากเอนจิน AV ที่มีหลักฐานสนับสนุน

ผู้ดูแลระบบสามารถยกเลิกการพักระงับที่เป็นผลบวกเท็จได้:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

คำสั่งนี้จะล้าง `requiresModerationAt` และ `requiresModerationReason`, คืนค่า
Skills ที่ถูกซ่อนโดยการพักระงับระดับผู้ใช้ และเขียนรายการ audit log `user.moderation.lift`
Skills ที่ถูกซ่อนด้วยเหตุผลอื่น หรือที่การสแกนแบบสแตติกของตัวเองยังคง
เป็นอันตราย จะยังคงถูกซ่อนอยู่

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบาย ClawHub อาจสูญเสียสิทธิ์การเผยแพร่ การละเมิดร้ายแรง
อาจส่งผลให้บัญชีถูกแบน token ถูกเพิกถอน เนื้อหาถูกซ่อน หรือรายการ
ถูกลบออก

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้ API tokens ของ ClawHub ได้ หากการยืนยันตัวตน CLI
เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ web UI เพื่อตรวจสอบสถานะ
บัญชี หรือติดต่อผู้ดูแลผ่านช่องทางสนับสนุนโปรเจกต์ที่คาดไว้

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลดผลบวกเท็จและปรับปรุงความเชื่อมั่นของผู้ใช้:

- รักษาชื่อ สรุป tags และบันทึกการเปลี่ยนแปลงให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่ต้องใช้
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้อ่านยาก
- ลิงก์ไปยังแหล่งที่มาเมื่อเป็นไปได้
- ใช้ dry runs ก่อนเผยแพร่ Plugin
- ตอบอย่างชัดเจนหากผู้ใช้หรือผู้กลั่นกรองถามเกี่ยวกับพฤติกรรมของแพ็กเกจ
