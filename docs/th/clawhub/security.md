# ความปลอดภัย

สามารถรายงานปัญหาความปลอดภัยของ ClawHub ผ่าน GitHub Security Advisories สำหรับ
`openclaw/clawhub`

ใช้ GitHub Security Advisories สำหรับช่องโหว่ในตัว ClawHub เอง รายงานคำแนะนำด้านความปลอดภัยของ
ClawHub ที่ดีควรรวมบั๊กใน:

- เว็บไซต์, API หรือ CLI ของ ClawHub
- การเผยแพร่ใน registry, การดาวน์โหลด, การติดตั้ง หรือความสมบูรณ์ของ artifact
- การยืนยันตัวตน, การอนุญาต หรือโทเค็น API
- การสแกน, การกลั่นกรอง หรือการจัดการรายงาน

อย่าใช้คำแนะนำด้านความปลอดภัยของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ Skills หรือ
Plugin ของบุคคลที่สามเอง ให้รายงานโดยตรงต่อผู้เผยแพร่หรือ repository ต้นทาง
ที่ลิงก์จากรายการ ClawHub

## การเปิดเผยช่องโหว่

เนื่องจาก ClawHub เป็นแอปพลิเคชันคลาวด์แบบโฮสต์ ช่องโหว่ของบริการ ClawHub
จึงจะไม่ถูกเปิดเผยต่อสาธารณะโดยค่าเริ่มต้น ช่องโหว่จะถูกเปิดเผยต่อสาธารณะเมื่อมี
หลักฐานของผลกระทบจริงต่อผู้ใช้ หรือเมื่อผู้ใช้จำเป็นต้องดำเนินการ

ตัวอย่างของผลกระทบจริงต่อผู้ใช้รวมถึงการยืนยันว่ามีการใช้ประโยชน์จากช่องโหว่,
การเปิดเผยข้อมูลผู้ใช้หรือความลับ, เนื้อหาที่เป็นอันตรายเข้าถึงผู้ใช้เนื่องจาก
ความล้มเหลวของแพลตฟอร์ม หรือปัญหาใดๆ ที่กำหนดให้ผู้ใช้ต้องหมุนเวียนข้อมูลรับรอง,
อัปเดตซอฟต์แวร์ในเครื่อง หรือดำเนินมาตรการป้องกันอื่นๆ

ช่องโหว่ในซอฟต์แวร์ที่ผู้ใช้ติดตั้งจะถูกเปิดเผยต่อสาธารณะ เช่น แพ็กเกจ CLI ของ
ClawHub, ไบนารี, ไลบรารี หรือ artifact รุ่นอื่นๆ ที่ผู้ใช้จำเป็นต้องอัปเดตในเครื่อง

สำหรับป้ายกำกับการตรวจสอบขณะติดตั้ง, ระดับความเสี่ยง, ผลการตรวจพบ และการตีความ โปรดดู
