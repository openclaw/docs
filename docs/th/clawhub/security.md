# ความปลอดภัย + การกลั่นกรอง

ClawHub เปิดให้เผยแพร่ได้ แต่รายการสาธารณะยังคงต้องผ่านการควบคุมด้านความน่าเชื่อถือ,
การสแกน, การรายงาน, และการกลั่นกรอง เป้าหมายคือการใช้งานจริง: ช่วยให้ผู้ใช้
ตรวจสอบสิ่งที่ติดตั้ง, ให้เส้นทางกู้คืนแก่ผู้เผยแพร่เมื่อเกิดผลบวกปลอม,
และกันแพ็กเกจที่ละเมิดออกจากการค้นพบสาธารณะ

ดูเพิ่มเติมที่ [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## สิ่งที่ผู้ใช้สามารถตรวจสอบได้

ก่อนติดตั้ง skill หรือ plugin ให้ตรวจสอบรายการ ClawHub ของสิ่งนั้นสำหรับ:

- เจ้าของและการระบุแหล่งที่มา
- เวอร์ชันล่าสุดและ changelog
- ตัวแปรสภาพแวดล้อมหรือสิทธิ์ที่จำเป็น
- เมตาดาต้าความเข้ากันได้สำหรับ plugins
- สถานะการสแกนหรือการกลั่นกรอง
- รายงาน, ความคิดเห็น, ดาว, การดาวน์โหลด, และสัญญาณการติดตั้งเมื่อมีแสดง

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือ

## สถานะการสแกน

ClawHub อาจแสดงผลลัพธ์การสแกนหรือการกลั่นกรองบนหน้าสาธารณะและ
การวินิจฉัยที่เจ้าของมองเห็นได้

ผลลัพธ์ทั่วไปประกอบด้วย:

- `clean`: ไม่พบปัญหาที่ต้องบล็อก
- `suspicious`: รีลีสนี้ต้องใช้ความระมัดระวังหรือการตรวจทาน
- `malicious`: รีลีสนี้ถือว่าไม่ปลอดภัย
- `pending`: การตรวจสอบยังไม่เสร็จสิ้น
- `held`, `quarantined`, `revoked`, หรือ `hidden`: รีลีสนี้ยังไม่พร้อมใช้งานอย่างเต็มที่
  บนพื้นผิวการติดตั้งสาธารณะ

ถ้อยคำที่แน่นอนอาจแตกต่างกันไปตามพื้นผิว แต่ความหมายในทางปฏิบัติเหมือนกัน: หาก
รีลีสถูกพักไว้หรือถูกบล็อก ผู้ใช้ไม่ควรติดตั้งจนกว่าเจ้าของจะแก้ไข
ปัญหาหรือการกลั่นกรองจะกู้คืนรีลีสนั้น

## Skills

การสแกน skill จะตรวจดูบันเดิล skill ที่เผยแพร่, เมตาดาต้า, ข้อกำหนดที่ประกาศไว้,
และคำสั่งที่น่าสงสัย

ClawHub ให้ความสนใจเป็นพิเศษกับความไม่สอดคล้องระหว่างสิ่งที่ skill ประกาศกับ
สิ่งที่ดูเหมือนว่ามันทำ ตัวอย่างเช่น skill ที่อ้างอิงคีย์ API ที่จำเป็น
ควรประกาศข้อกำหนดนั้นใน `SKILL.md` เพื่อให้ผู้ใช้มองเห็นก่อน
ติดตั้ง

ผลการสแกนอ้างอิงจาก artifact พฤติกรรมที่คาดหวังของ provider เช่น ข้อมูลรับรอง
API ที่ประกาศไว้, การเรียกกลับ OAuth ไปยัง localhost, การล้างข้อมูลเมื่อถอนการติดตั้งแบบจำกัดขอบเขต, การเข้ารหัส Basic Auth,
หรือการอัปโหลดไฟล์ที่ผู้ใช้เลือกไปยัง provider ที่ระบุไว้ จะได้รับการปฏิบัติ
ต่างจากการส่งต่อข้อมูลรับรองแบบซ่อน, การเข้าถึงไฟล์ส่วนตัวในวงกว้าง,
ปลายทางเครือข่ายที่ไม่เกี่ยวข้อง, หรือการใช้เบราว์เซอร์ในทางที่ผิดแบบซ่อนเร้น

ดู [รูปแบบ skill](/th/clawhub/skill-format)

## Plugins

รีลีส Plugin มีเมตาดาต้าแพ็กเกจ, การระบุแหล่งที่มา, ฟิลด์ความเข้ากันได้,
และข้อมูลความสมบูรณ์ของ artifact

OpenClaw ตรวจสอบความเข้ากันได้ก่อนติดตั้ง Plugin ที่โฮสต์บน ClawHub ระเบียนแพ็กเกจ
อาจเปิดเผยเมตาดาต้า digest ด้วย เพื่อให้ OpenClaw ตรวจสอบ
artifact ที่ดาวน์โหลดมาได้ ClawScan รวมเมตาดาต้า env/config ของแพ็กเกจ `openclaw.environment`
ที่ประกาศไว้เมื่อรีวิวรีลีส Plugin เพื่อเปรียบเทียบข้อกำหนด runtime ที่ประกาศไว้
กับพฤติกรรมที่สังเกตได้

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน skills, แพ็กเกจ, และความคิดเห็นได้

รายงานควรเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้ระบบรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

ตัวอย่างรายงาน:

- เมตาดาต้าที่ทำให้เข้าใจผิด
- ข้อกำหนดข้อมูลรับรองหรือสิทธิ์ที่ไม่ได้ประกาศ
- คำสั่งติดตั้งที่น่าสงสัย
- ความคิดเห็นหลอกลวงหรือการแอบอ้าง
- การจดทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## หมายเหตุ ClawScan สำหรับผู้เผยแพร่

ผู้เผยแพร่สามารถให้หมายเหตุ ClawScan แบบไม่บังคับเมื่อเผยแพร่ skill หรือ
Plugin หมายเหตุนี้ให้บริบทแก่ ClawScan สำหรับพฤติกรรมที่อาจดู
ผิดปกติ เช่น การเข้าถึงเครือข่าย, การเข้าถึง native host, หรือข้อมูลรับรอง
เฉพาะ provider

## การพักเพื่อกลั่นกรอง

เมื่อ static scanner ระบุว่า skill ที่อัปโหลดเป็นอันตราย ผู้เผยแพร่จะถูก
วางไว้ภายใต้การพักเพื่อกลั่นกรองโดยอัตโนมัติ (`requiresModerationAt` ถูกตั้งบน
ผู้ใช้) สิ่งนี้จะซ่อน skills ทั้งหมดของผู้เผยแพร่, ทำให้การเผยแพร่ในอนาคต
เริ่มต้นเป็นแบบซ่อน, และสร้างรายการ audit log `user.moderation.auto`

ผลการตรวจพบแบบ suspicious จาก static จะถูกเก็บไว้เป็นหลักฐานระดับไฟล์/บรรทัดสำหรับผู้กลั่นกรอง,
แต่ผลเหล่านั้นไม่ซ่อนเนื้อหาหรือตัดสิน verdict การสแกนสาธารณะด้วยตัวเอง
การอัปโหลดใหม่ยังคงอยู่ในสถานะ review/pending จนกว่าการรีวิว LLM จะสรุปผล Static
scanning จะบล็อกทันทีเฉพาะ signature ที่เป็น malicious เท่านั้น engine
hits ของ VirusTotal ยังคงมองเห็นเป็นหลักฐานด้านความปลอดภัย แต่ verdicts ของ VirusTotal Code Insight/Palm
เป็นคำแนะนำและไม่ซ่อน skills ด้วยตัวเอง การรีวิว LLM ของ ClawScan
เก็บหมายเหตุที่สอดคล้องกับวัตถุประสงค์ไว้เป็นแนวทาง ผลการรีวิวระดับ Medium ยังคงมองเห็นได้บน
artifact ขณะที่ตัวกรอง suspicious สงวนไว้สำหรับข้อกังวล LLM
ที่มีผลกระทบสูง, ผลการตรวจพบแบบ malicious, หรือการตรวจพบจาก AV-engine ที่มีหลักฐานสนับสนุน

ผู้ดูแลระบบสามารถยกเลิกการพักที่เป็นผลบวกปลอมได้:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

สิ่งนี้จะล้าง `requiresModerationAt` และ `requiresModerationReason`, กู้คืน
skills ที่ถูกซ่อนโดยการพักระดับผู้ใช้, และเขียนรายการ audit
log `user.moderation.lift` skills ที่ถูกซ่อนด้วยเหตุผลอื่น หรือที่ static scan ของตัวเองยังคงเป็น
malicious จะยังคงถูกซ่อนอยู่

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบาย ClawHub อาจสูญเสียสิทธิ์การเผยแพร่ การละเมิดร้ายแรง
อาจส่งผลให้บัญชีถูกแบน, โทเค็นถูกเพิกถอน, เนื้อหาถูกซ่อน, หรือรายการถูก
ลบออก

บัญชีที่ถูกลบ, ถูกแบน, หรือถูกปิดใช้งานไม่สามารถใช้โทเค็น ClawHub API ได้ หากการยืนยันตัวตน CLI
เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้เว็บ UI เพื่อตรวจสอบ
สถานะบัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อก ให้ติดต่อ
security@openclaw.ai เพื่อขอรีวิวการกู้คืน

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลดผลบวกปลอมและเพิ่มความไว้วางใจของผู้ใช้:

- รักษาชื่อ, สรุป, แท็ก, และ changelogs ให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- เพิ่มหมายเหตุ ClawScan สำหรับผู้เผยแพร่เมื่อรีลีสมีพฤติกรรมที่ผิดปกติแต่ตั้งใจไว้
- หลีกเลี่ยงคำสั่งติดตั้งที่ถูกทำให้อ่านยาก
- ลิงก์ไปยังแหล่งที่มาเมื่อเป็นไปได้
- ใช้ dry runs ก่อนเผยแพร่ Plugin
- ตอบกลับอย่างชัดเจนหากผู้ใช้หรือผู้กลั่นกรองถามเกี่ยวกับพฤติกรรมของแพ็กเกจ
