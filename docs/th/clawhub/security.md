# ความปลอดภัย + การควบคุมเนื้อหา

ClawHub เปิดให้เผยแพร่ได้ แต่รายการสาธารณะยังคงผ่านการควบคุมด้านความน่าเชื่อถือ
การสแกน การรายงาน และการควบคุมเนื้อหา เป้าหมายเป็นเรื่องเชิงปฏิบัติ: ช่วยให้ผู้ใช้
ตรวจสอบสิ่งที่ติดตั้ง ให้ผู้เผยแพร่มีเส้นทางกู้คืนเมื่อเกิดผลบวกลวง
และกันแพ็กเกจที่เป็นอันตรายออกจากการค้นพบแบบสาธารณะ

ดูเพิ่มเติมที่ [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## สิ่งที่ผู้ใช้ตรวจสอบได้

ก่อนติดตั้ง Skills หรือ Plugin ให้ตรวจสอบรายการ ClawHub ของรายการนั้นสำหรับ:

- เจ้าของและการระบุแหล่งที่มา
- เวอร์ชันล่าสุดและ changelog
- ตัวแปรสภาพแวดล้อมหรือสิทธิ์ที่จำเป็น
- metadata ความเข้ากันได้สำหรับ Plugin
- สถานะการสแกนหรือการควบคุมเนื้อหา
- รายงาน ความคิดเห็น ดาว ยอดดาวน์โหลด และสัญญาณการติดตั้งเมื่อมีแสดง

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการสแกน

ClawHub อาจแสดงผลลัพธ์การสแกนหรือการควบคุมเนื้อหาบนหน้าสาธารณะและ
diagnostics ที่เจ้าของมองเห็นได้

ผลลัพธ์ทั่วไปประกอบด้วย:

- `clean`: ไม่พบปัญหาที่ต้องบล็อก
- `suspicious`: release ต้องใช้ความระมัดระวังหรือได้รับการตรวจสอบ
- `malicious`: release ถือว่าไม่ปลอดภัย
- `pending`: การตรวจสอบยังไม่เสร็จสิ้น
- `held`, `quarantined`, `revoked` หรือ `hidden`: release ยังไม่พร้อมใช้งานอย่างเต็มรูปแบบ
  บนพื้นผิวการติดตั้งสาธารณะ

ถ้อยคำที่แน่นอนอาจแตกต่างกันไปตามพื้นผิว แต่ความหมายเชิงปฏิบัติเหมือนกัน: หาก
release ถูกระงับหรือถูกบล็อก ผู้ใช้ไม่ควรติดตั้งจนกว่าเจ้าของจะแก้ไข
ปัญหาหรือการควบคุมเนื้อหาจะกู้คืนรายการนั้น

## Skills

การสแกน Skills จะตรวจดูชุด Skills ที่เผยแพร่ metadata ข้อกำหนดที่ประกาศไว้
และคำสั่งที่น่าสงสัย

ClawHub ให้ความสำคัญเป็นพิเศษกับความไม่ตรงกันระหว่างสิ่งที่ Skills ประกาศกับ
สิ่งที่ดูเหมือนจะทำ ตัวอย่างเช่น Skills ที่อ้างอิง API key ที่จำเป็น
ควรประกาศข้อกำหนดนั้นใน `SKILL.md` เพื่อให้ผู้ใช้เห็นได้ก่อน
ติดตั้ง

ผลการสแกนอิงตาม artifact พฤติกรรมของ provider ที่คาดไว้ เช่น ข้อมูลรับรอง API
ที่ประกาศ, callback OAuth ของ localhost, การล้างข้อมูลเมื่อถอนการติดตั้งแบบมีขอบเขต, การเข้ารหัส Basic Auth
หรือการอัปโหลดไฟล์ที่ผู้ใช้เลือกไปยัง provider ที่ระบุไว้ จะถูกปฏิบัติ
ต่างจากการส่งต่อข้อมูลรับรองแบบซ่อนเร้น การเข้าถึงไฟล์ส่วนตัวอย่างกว้างขวาง
ปลายทางเครือข่ายที่ไม่เกี่ยวข้อง หรือการใช้เบราว์เซอร์อย่างลับ ๆ ในทางที่ผิด

ดู [รูปแบบ Skills](/th/clawhub/skill-format)

## Plugin

release ของ Plugin มี package metadata การระบุแหล่งที่มา ฟิลด์ความเข้ากันได้
และข้อมูลความถูกต้องสมบูรณ์ของ artifact

OpenClaw ตรวจสอบความเข้ากันได้ก่อนติดตั้ง Plugin ที่โฮสต์บน ClawHub ระเบียนแพ็กเกจ
อาจเปิดเผย digest metadata ด้วยเพื่อให้ OpenClaw ตรวจสอบ artifact
ที่ดาวน์โหลดมาได้ ClawScan รวม metadata env/config ของแพ็กเกจ `openclaw.environment`
ที่ประกาศไว้เมื่อรีวิว release ของ Plugin เพื่อให้ข้อกำหนด runtime ที่ประกาศไว้
ถูกเปรียบเทียบกับพฤติกรรมที่สังเกตได้

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้แล้วสามารถรายงาน Skills แพ็กเกจ และความคิดเห็นได้

รายงานควรเจาะจงและนำไปดำเนินการได้ การใช้การรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

ตัวอย่างรายงาน:

- metadata ที่ทำให้เข้าใจผิด
- ข้อกำหนดข้อมูลรับรองหรือสิทธิ์ที่ไม่ได้ประกาศ
- คำสั่งติดตั้งที่น่าสงสัย
- ความคิดเห็นหลอกลวงหรือการแอบอ้าง
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## หมายเหตุ ClawScan สำหรับผู้เผยแพร่

ผู้เผยแพร่สามารถใส่หมายเหตุ ClawScan แบบไม่บังคับเมื่อเผยแพร่ Skills หรือ
Plugin หมายเหตุนี้ให้บริบทแก่ ClawScan สำหรับพฤติกรรมที่อาจดู
ผิดปกติ เช่น การเข้าถึงเครือข่าย การเข้าถึง native host หรือข้อมูลรับรอง
เฉพาะ provider

## การระงับเพื่อควบคุมเนื้อหา

เมื่อ static scanner ทำเครื่องหมาย Skills ที่อัปโหลดว่าเป็นอันตราย ผู้เผยแพร่จะถูก
จัดให้อยู่ภายใต้การระงับเพื่อควบคุมเนื้อหาโดยอัตโนมัติ (`requiresModerationAt` ถูกตั้งค่าใน
ผู้ใช้) การดำเนินการนี้ซ่อน Skills ทั้งหมดของผู้เผยแพร่ ทำให้การเผยแพร่ในอนาคต
เริ่มต้นเป็น hidden และสร้างรายการ audit log `user.moderation.auto`

ผลการตรวจพบที่น่าสงสัยจาก static จะถูกเก็บไว้เป็นหลักฐาน file/line สำหรับผู้ควบคุมเนื้อหา
แต่ไม่ได้ซ่อนเนื้อหาหรือตัดสิน verdict การสแกนสาธารณะด้วยตัวเอง
การอัปโหลดใหม่จะยังอยู่ในสถานะ review/pending จนกว่าการรีวิว LLM จะสรุป Static
scanning จะบล็อกทันทีเฉพาะ signature ที่เป็นอันตรายเท่านั้น ผล hit จาก engine ของ VirusTotal
ยังคงมองเห็นเป็นหลักฐานด้านความปลอดภัย แต่ verdict ของ VirusTotal Code Insight/Palm
เป็นคำแนะนำและไม่ซ่อน Skills ด้วยตัวเอง รีวิว ClawScan LLM
จะเก็บหมายเหตุที่สอดคล้องกับวัตถุประสงค์ไว้เป็นแนวทาง ผลการรีวิวระดับ medium ยังคงมองเห็นได้บน
artifact ขณะที่ตัวกรอง suspicious สงวนไว้สำหรับข้อกังวล LLM
ที่มีผลกระทบสูง ผลการตรวจพบที่เป็นอันตราย หรือการตรวจพบจาก AV-engine ที่มีหลักฐานยืนยันร่วมกัน

ผู้ดูแลระบบสามารถยกเลิกการระงับจากผลบวกลวงได้:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

การดำเนินการนี้ล้าง `requiresModerationAt` และ `requiresModerationReason`, กู้คืน
Skills ที่ถูกซ่อนโดยการระงับระดับผู้ใช้ และเขียนรายการ audit log
`user.moderation.lift` Skills ที่ถูกซ่อนด้วยเหตุผลอื่น หรือที่ผล static scan ของตัวเองยังคง
เป็น malicious จะยังคงถูกซ่อนอยู่

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบาย ClawHub อาจสูญเสียสิทธิ์การเผยแพร่ การใช้งานในทางที่ผิดอย่างร้ายแรง
อาจส่งผลให้บัญชีถูกแบน token ถูกเพิกถอน เนื้อหาถูกซ่อน หรือ
รายการถูกลบ

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้ token ของ ClawHub API ได้ หาก CLI auth
เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ web UI เพื่อตรวจสอบ
สถานะบัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อก ให้ติดต่อ
security@openclaw.ai เพื่อขอรีวิวการกู้คืน

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลดผลบวกลวงและเพิ่มความไว้วางใจของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และ changelog ให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- เพิ่มหมายเหตุ ClawScan สำหรับผู้เผยแพร่เมื่อ release มีพฤติกรรมที่ผิดปกติแต่ตั้งใจให้เกิดขึ้น
- หลีกเลี่ยงคำสั่งติดตั้งที่ถูกทำให้อ่านยาก
- ลิงก์ไปยังซอร์สเมื่อเป็นไปได้
- ใช้ dry run ก่อนเผยแพร่ Plugin
- ตอบอย่างชัดเจนหากผู้ใช้หรือผู้ควบคุมเนื้อหาถามเกี่ยวกับพฤติกรรมของแพ็กเกจ
