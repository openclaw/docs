# ความปลอดภัย

สามารถรายงานปัญหาความปลอดภัยของ ClawHub ผ่าน GitHub Security Advisories สำหรับ
`openclaw/clawhub`

ใช้ GitHub Security Advisories สำหรับช่องโหว่ใน ClawHub เอง รายงาน advisory ของ
ClawHub ที่ดีรวมถึงบั๊กใน:

- เว็บไซต์, API หรือ CLI ของ ClawHub
- การเผยแพร่รีจิสทรี การดาวน์โหลด การติดตั้ง หรือความสมบูรณ์ของอาร์ติแฟกต์
- การตรวจสอบสิทธิ์ การอนุญาต หรือโทเค็น API
- การสแกน การกลั่นกรอง หรือการจัดการรายงาน

อย่าใช้ advisory ของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ Skills หรือ Plugin
ของบุคคลที่สามเอง ให้รายงานโดยตรงไปยังผู้เผยแพร่หรือรีโพซิทอรีต้นทางที่ลิงก์จากรายการใน ClawHub

## การเปิดเผยช่องโหว่

เนื่องจาก ClawHub เป็นแอปพลิเคชันคลาวด์แบบโฮสต์ ช่องโหว่ของบริการ ClawHub
จึงไม่ถูกเปิดเผยต่อสาธารณะโดยค่าเริ่มต้น ช่องโหว่เหล่านี้จะถูกเปิดเผยต่อสาธารณะเมื่อมีหลักฐานว่าผู้ใช้ได้รับผลกระทบจริง หรือเมื่อผู้ใช้จำเป็นต้องดำเนินการ

ตัวอย่างของผลกระทบจริงต่อผู้ใช้รวมถึงการยืนยันว่าถูกใช้โจมตี การเปิดเผยข้อมูลผู้ใช้หรือความลับ เนื้อหามุ่งร้ายที่เข้าถึงผู้ใช้เนื่องจากความล้มเหลวของแพลตฟอร์ม
หรือปัญหาใดๆ ที่กำหนดให้ผู้ใช้ต้องหมุนเวียนข้อมูลประจำตัว อัปเดตซอฟต์แวร์ในเครื่อง
หรือดำเนินการป้องกันอื่นๆ

ช่องโหว่ในซอฟต์แวร์ที่ผู้ใช้ติดตั้งจะถูกเปิดเผยต่อสาธารณะ เช่น
แพ็กเกจ CLI ของ ClawHub ไบนารี ไลบรารี หรืออาร์ติแฟกต์รุ่นอื่นๆ ที่ผู้ใช้
ต้องอัปเดตในเครื่อง
