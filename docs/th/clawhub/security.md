# ความปลอดภัย

ปัญหาด้านความปลอดภัยของ ClawHub สามารถรายงานผ่าน GitHub Security Advisories สำหรับ
`openclaw/clawhub`

ใช้ GitHub Security Advisories สำหรับช่องโหว่ในตัว ClawHub เอง รายงาน advisory ของ
ClawHub ที่ดีควรรวมบั๊กใน:

- เว็บไซต์, API หรือ CLI ของ ClawHub
- การเผยแพร่ registry, การดาวน์โหลด, การติดตั้ง หรือความสมบูรณ์ของอาร์ติแฟกต์
- การยืนยันตัวตน, การอนุญาต หรือโทเค็น API
- การสแกน, การกลั่นกรอง หรือการจัดการรายงาน

อย่าใช้ advisory ของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ Skills หรือ
Plugin บุคคลที่สาม ให้รายงานโดยตรงไปยังผู้เผยแพร่หรือ repository ต้นทาง
ที่ลิงก์จากรายการใน ClawHub

## การเปิดเผยช่องโหว่

เนื่องจาก ClawHub เป็นแอปพลิเคชันคลาวด์แบบโฮสต์ ช่องโหว่ของบริการ ClawHub
จะไม่ถูกเปิดเผยต่อสาธารณะโดยค่าเริ่มต้น ช่องโหว่จะถูกเปิดเผยต่อสาธารณะเมื่อมี
หลักฐานว่ากระทบต่อผู้ใช้จริง หรือเมื่อผู้ใช้จำเป็นต้องดำเนินการ

ตัวอย่างผลกระทบต่อผู้ใช้จริง ได้แก่ การยืนยันว่ามีการโจมตีสำเร็จ การเปิดเผยข้อมูล
ผู้ใช้หรือความลับ เนื้อหาที่เป็นอันตรายเข้าถึงผู้ใช้เนื่องจากความล้มเหลวของแพลตฟอร์ม
หรือปัญหาใด ๆ ที่กำหนดให้ผู้ใช้ต้องหมุนเวียนข้อมูลรับรอง อัปเดตซอฟต์แวร์ในเครื่อง
หรือดำเนินการป้องกันอื่น ๆ

ช่องโหว่ในซอฟต์แวร์ที่ผู้ใช้ติดตั้งจะถูกเปิดเผยต่อสาธารณะ เช่น แพ็กเกจ CLI ของ
ClawHub, ไบนารี, ไลบรารี หรืออาร์ติแฟกต์ release อื่น ๆ ที่ผู้ใช้จำเป็นต้องอัปเดต
ในเครื่อง
