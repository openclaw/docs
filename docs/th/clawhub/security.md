# ความปลอดภัย + การกลั่นกรอง

ClawHub เปิดให้เผยแพร่ได้ แต่รายการสาธารณะยังคงต้องผ่านการควบคุมด้านความน่าเชื่อถือ
การสแกน การรายงาน และการกลั่นกรอง เป้าหมายคือเชิงปฏิบัติ: ช่วยให้ผู้ใช้
ตรวจสอบสิ่งที่ติดตั้ง ให้ผู้เผยแพร่มีทางกู้คืนกรณีผลบวกลวง
และกันแพ็กเกจที่เป็นการละเมิดออกจากการค้นพบแบบสาธารณะ

ดูเพิ่มเติมที่ [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## สิ่งที่ผู้ใช้ตรวจสอบได้

ก่อนติดตั้ง Skills หรือ Plugin ให้ตรวจสอบรายการ ClawHub ของรายการนั้นสำหรับ:

- เจ้าของและการระบุแหล่งที่มา
- เวอร์ชันล่าสุดและบันทึกการเปลี่ยนแปลง
- ตัวแปรสภาพแวดล้อมหรือสิทธิ์ที่จำเป็น
- เมทาดาทาความเข้ากันได้สำหรับ Plugin
- สถานะการสแกนหรือการกลั่นกรอง
- รายงาน ความคิดเห็น ดาว การดาวน์โหลด และสัญญาณการติดตั้งในจุดที่แสดง

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการสแกน

ClawHub อาจแสดงผลลัพธ์การสแกนหรือการกลั่นกรองบนหน้าสาธารณะและ
การวินิจฉัยที่เจ้าของมองเห็นได้

ผลลัพธ์ทั่วไปได้แก่:

- `clean`: ไม่พบปัญหาที่ต้องบล็อก
- `suspicious`: รีลีสนี้ต้องใช้ความระมัดระวังหรือต้องตรวจทาน
- `malicious`: รีลีสนี้ถือว่าไม่ปลอดภัย
- `pending`: การตรวจสอบยังไม่เสร็จสิ้น
- `held`, `quarantined`, `revoked` หรือ `hidden`: รีลีสนี้ยังไม่พร้อมใช้งานครบถ้วน
  บนพื้นผิวการติดตั้งสาธารณะ

ถ้อยคำที่แน่นอนอาจแตกต่างกันไปตามพื้นผิว แต่ความหมายเชิงปฏิบัติเหมือนกัน: หาก
รีลีสถูกพักหรือถูกบล็อก ผู้ใช้ไม่ควรติดตั้งจนกว่าเจ้าของจะแก้ไข
ปัญหา หรือการกลั่นกรองจะกู้คืนรีลีสนั้น

## Skills

การสแกน Skills จะตรวจดูบันเดิล Skills ที่เผยแพร่ เมทาดาทา ข้อกำหนดที่ประกาศไว้
และคำสั่งที่น่าสงสัย

ClawHub ให้ความสำคัญเป็นพิเศษกับความไม่ตรงกันระหว่างสิ่งที่ Skills ประกาศกับ
สิ่งที่ดูเหมือนว่าจะทำ ตัวอย่างเช่น Skills ที่อ้างอิงคีย์ API ที่จำเป็น
ควรประกาศข้อกำหนดนั้นใน `SKILL.md` เพื่อให้ผู้ใช้มองเห็นได้ก่อน
ติดตั้ง

ผลการสแกนอิงตามอาร์ติแฟกต์ พฤติกรรมของผู้ให้บริการที่คาดไว้ เช่น ข้อมูลประจำตัว API
ที่ประกาศไว้ คอลแบ็ก OAuth ของ localhost การล้างข้อมูลตอนถอนการติดตั้งตามขอบเขต การเข้ารหัส Basic Auth
หรือการอัปโหลดไฟล์ที่ผู้ใช้เลือกไปยังผู้ให้บริการที่ระบุไว้ จะถูกพิจารณา
แตกต่างจากการส่งต่อข้อมูลประจำตัวแบบซ่อน การเข้าถึงไฟล์ส่วนตัวแบบกว้าง
ปลายทางเครือข่ายที่ไม่เกี่ยวข้อง หรือการใช้เบราว์เซอร์ในทางที่ผิดแบบลับ ๆ

ดู [รูปแบบ Skills](/th/clawhub/skill-format)

## Plugin

รีลีส Plugin มีเมทาดาทาแพ็กเกจ การระบุแหล่งที่มา ฟิลด์ความเข้ากันได้
และข้อมูลความสมบูรณ์ของอาร์ติแฟกต์

OpenClaw ตรวจสอบความเข้ากันได้ก่อนติดตั้ง Plugin ที่โฮสต์บน ClawHub ระเบียนแพ็กเกจ
อาจเปิดเผยเมทาดาทาไดเจสต์ด้วย เพื่อให้ OpenClaw ตรวจสอบอาร์ติแฟกต์
ที่ดาวน์โหลดมาได้ ClawScan รวมเมทาดาทา env/config ของแพ็กเกจ `openclaw.environment`
ที่ประกาศไว้เมื่อรีวิวรีลีส Plugin เพื่อให้ข้อกำหนดรันไทม์ที่ประกาศไว้
ถูกนำไปเปรียบเทียบกับพฤติกรรมที่สังเกตได้

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน Skills แพ็กเกจ และความคิดเห็นได้

รายงานควรระบุรายละเอียดและนำไปดำเนินการได้ การใช้ระบบรายงานในทางที่ผิดสามารถนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

ตัวอย่างรายงาน:

- เมทาดาทาที่ทำให้เข้าใจผิด
- ข้อกำหนดด้านข้อมูลประจำตัวหรือสิทธิ์ที่ไม่ได้ประกาศ
- คำสั่งติดตั้งที่น่าสงสัย
- ความคิดเห็นหลอกลวงหรือการแอบอ้าง
- การจดทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## หมายเหตุ ClawScan สำหรับผู้เผยแพร่

ผู้เผยแพร่สามารถระบุหมายเหตุ ClawScan เพิ่มเติมได้เมื่อเผยแพร่ Skills หรือ
Plugin หมายเหตุนี่ให้บริบทแก่ ClawScan สำหรับพฤติกรรมที่อาจดู
ผิดปกติ เช่น การเข้าถึงเครือข่าย การเข้าถึงโฮสต์แบบเนทีฟ หรือข้อมูลประจำตัว
เฉพาะผู้ให้บริการ

## การพักเพื่อกลั่นกรอง

เมื่อตัวสแกนแบบสแตติกระบุว่า Skills ที่อัปโหลดเป็นอันตราย ผู้เผยแพร่จะถูก
พักเพื่อการกลั่นกรองโดยอัตโนมัติ (`requiresModerationAt` ถูกตั้งค่าบน
ผู้ใช้) การดำเนินการนี้จะซ่อน Skills ทั้งหมดของผู้เผยแพร่ ทำให้การเผยแพร่ในอนาคต
เริ่มต้นแบบซ่อน และสร้างรายการบันทึกการตรวจสอบ `user.moderation.auto`

ผลการตรวจพบแบบสแตติกที่น่าสงสัยจะถูกเก็บไว้เป็นหลักฐานไฟล์/บรรทัดสำหรับผู้กลั่นกรอง
แต่ผลเหล่านี้ไม่ได้ซ่อนเนื้อหาหรือตัดสินผลการสแกนสาธารณะด้วยตัวเอง
รายการอัปโหลดใหม่จะยังอยู่ในสถานะรีวิว/รอดำเนินการจนกว่าการรีวิวของ LLM จะเสร็จสิ้น การสแกนแบบสแตติก
จะบล็อกทันทีเฉพาะลายเซ็นที่เป็นอันตรายเท่านั้น การตรวจพบจากเอนจิน VirusTotal
ยังคงแสดงเป็นหลักฐานความปลอดภัยที่มองเห็นได้ แต่คำตัดสินของ VirusTotal Code Insight/Palm
เป็นคำแนะนำและไม่ได้ซ่อน Skills ด้วยตัวเอง การรีวิว LLM ของ ClawScan
จะเก็บหมายเหตุที่สอดคล้องกับวัตถุประสงค์ไว้เป็นแนวทาง ผลการรีวิวระดับกลางยังคงมองเห็นได้บน
อาร์ติแฟกต์ ส่วนตัวกรองที่น่าสงสัยจะสงวนไว้สำหรับข้อกังวลจาก LLM
ที่มีผลกระทบสูง ผลการตรวจพบที่เป็นอันตราย หรือการตรวจพบจากเอนจิน AV ที่มีหลักฐานสนับสนุน

ผู้ดูแลระบบสามารถยกเลิกการพักจากผลบวกลวงได้:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

การดำเนินการนี้จะล้าง `requiresModerationAt` และ `requiresModerationReason`, กู้คืน
Skills ที่ถูกซ่อนโดยการพักระดับผู้ใช้ และเขียนรายการบันทึกการตรวจสอบ `user.moderation.lift`
Skills ที่ถูกซ่อนด้วยเหตุผลอื่น หรือ Skills ที่ผลสแกนแบบสแตติกของตัวเองยังคง
เป็นอันตราย จะยังคงถูกซ่อนอยู่

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบาย ClawHub อาจสูญเสียสิทธิ์การเผยแพร่ การละเมิดร้ายแรง
อาจทำให้บัญชีถูกแบน โทเค็นถูกเพิกถอน เนื้อหาถูกซ่อน หรือรายการ
ถูกนำออก

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้โทเค็น API ของ ClawHub ได้ หากการตรวจสอบสิทธิ์ CLI
เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ UI เว็บเพื่อตรวจสอบ
สถานะบัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ปกติถูกบล็อก โปรดติดต่อ
security@openclaw.ai เพื่อขอการรีวิวการกู้คืน

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลดผลบวกลวงและเพิ่มความน่าเชื่อถือของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และบันทึกการเปลี่ยนแปลงให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- เพิ่มหมายเหตุ ClawScan สำหรับผู้เผยแพร่เมื่อรีลีสมีพฤติกรรมที่ผิดปกติแต่ตั้งใจไว้
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้อ่านยาก
- ลิงก์ไปยังแหล่งที่มาเมื่อเป็นไปได้
- ใช้การทดสอบแบบไม่ลงมือเปลี่ยนแปลงจริงก่อนเผยแพร่ Plugin
- ตอบอย่างชัดเจนหากผู้ใช้หรือผู้กลั่นกรองถามเกี่ยวกับพฤติกรรมของแพ็กเกจ
