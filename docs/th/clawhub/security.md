# ความปลอดภัย

สามารถรายงานปัญหาความปลอดภัยของ ClawHub ผ่าน GitHub Security Advisories สำหรับ
`openclaw/clawhub`

ใช้ GitHub Security Advisories สำหรับช่องโหว่ใน ClawHub เอง รายงานคำแนะนำด้านความปลอดภัยของ
ClawHub ที่ดีรวมถึงบั๊กใน:

- เว็บไซต์, API หรือ CLI ของ ClawHub
- การเผยแพร่รีจิสทรี, การดาวน์โหลด, การติดตั้ง หรือความถูกต้องสมบูรณ์ของอาร์ติแฟกต์
- การยืนยันตัวตน, การอนุญาต หรือโทเค็น API
- การสแกน, การกลั่นกรอง หรือการจัดการรายงาน

อย่าใช้คำแนะนำด้านความปลอดภัยของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ Skills หรือ
Plugin ของบุคคลที่สามเอง ให้รายงานโดยตรงไปยังผู้เผยแพร่หรือรีโพซิทอรีต้นทาง
ที่ลิงก์จากรายการใน ClawHub

## การเปิดเผยช่องโหว่

เนื่องจาก ClawHub เป็นแอปพลิเคชันคลาวด์แบบโฮสต์ ช่องโหว่ของบริการ ClawHub
จึงไม่ถูกเปิดเผยต่อสาธารณะโดยค่าเริ่มต้น ช่องโหว่จะถูกเปิดเผยต่อสาธารณะเมื่อมี
หลักฐานว่าผู้ใช้ได้รับผลกระทบจริง หรือเมื่อผู้ใช้จำเป็นต้องดำเนินการ

ตัวอย่างของผลกระทบจริงต่อผู้ใช้รวมถึงการยืนยันว่ามีการใช้ช่องโหว่ในทางโจมตี, การเปิดเผยข้อมูลผู้ใช้
หรือความลับ, เนื้อหาที่เป็นอันตรายเข้าถึงผู้ใช้เนื่องจากความล้มเหลวของแพลตฟอร์ม,
หรือปัญหาใด ๆ ที่ทำให้ผู้ใช้ต้องหมุนเวียนข้อมูลประจำตัว, อัปเดตซอฟต์แวร์ในเครื่อง หรือ
ดำเนินการป้องกันอื่น ๆ

ช่องโหว่ในซอฟต์แวร์ที่ผู้ใช้ติดตั้งจะถูกเปิดเผยต่อสาธารณะ เช่น
แพ็กเกจ CLI ของ ClawHub, ไบนารี, ไลบรารี หรืออาร์ติแฟกต์รุ่นเผยแพร่อื่น ๆ ที่ผู้ใช้
ต้องอัปเดตในเครื่อง

สำหรับป้ายกำกับการตรวจสอบตอนติดตั้ง, ระดับความเสี่ยง, ข้อค้นพบ และการตีความ โปรดดู
