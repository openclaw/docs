# ความปลอดภัย + การกลั่นกรอง

ClawHub เปิดให้เผยแพร่ได้ แต่รายการสาธารณะยังคงผ่านการควบคุมด้านความน่าเชื่อถือ
การสแกน การรายงาน และการกลั่นกรอง เป้าหมายคือเชิงปฏิบัติ: ช่วยให้ผู้ใช้
ตรวจสอบสิ่งที่ติดตั้ง ให้ผู้เผยแพร่มีเส้นทางแก้ไขเมื่อเกิดผลบวกเท็จ
และกันแพ็กเกจที่ไม่เหมาะสมออกจากการค้นพบแบบสาธารณะ

ดูเพิ่มเติมที่ [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## สิ่งที่ผู้ใช้ตรวจสอบได้

ก่อนติดตั้ง skill หรือ plugin ให้ตรวจสอบรายการ ClawHub ของสิ่งนั้นเพื่อดู:

- เจ้าของและการระบุแหล่งที่มา
- เวอร์ชันล่าสุดและบันทึกการเปลี่ยนแปลง
- ตัวแปรสภาพแวดล้อมหรือสิทธิ์ที่จำเป็น
- เมตาดาต้าความเข้ากันได้สำหรับ plugin
- สถานะการสแกนหรือการกลั่นกรอง
- รายงาน ความคิดเห็น ดาว จำนวนดาวน์โหลด และสัญญาณการติดตั้งในจุดที่แสดง

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการสแกน

ClawHub อาจแสดงผลลัพธ์การสแกนหรือการกลั่นกรองบนหน้าสาธารณะและ
การวินิจฉัยที่เจ้าของมองเห็นได้

ผลลัพธ์ทั่วไปประกอบด้วย:

- `clean`: ไม่พบปัญหาที่ต้องบล็อก
- `suspicious`: รีลีสนี้ต้องใช้ความระมัดระวังหรือการตรวจสอบ
- `malicious`: รีลีสนี้ถือว่าไม่ปลอดภัย
- `pending`: การตรวจสอบยังไม่เสร็จสิ้น
- `held`, `quarantined`, `revoked` หรือ `hidden`: รีลีสนี้ยังไม่พร้อมใช้งานอย่างเต็มที่
  บนพื้นผิวการติดตั้งสาธารณะ

ถ้อยคำที่แน่นอนอาจแตกต่างกันไปตามพื้นผิว แต่ความหมายเชิงปฏิบัติเหมือนกัน: หาก
รีลีสถูกพักหรือถูกบล็อก ผู้ใช้ไม่ควรติดตั้งจนกว่าเจ้าของจะแก้ไข
ปัญหาได้ หรือการกลั่นกรองจะกู้คืนรีลีสนั้น

## Skills

การสแกน skill จะตรวจดูบันเดิล skill ที่เผยแพร่ เมตาดาต้า ข้อกำหนด
ที่ประกาศไว้ และคำสั่งที่น่าสงสัย

ClawHub ให้ความสำคัญเป็นพิเศษกับความไม่ตรงกันระหว่างสิ่งที่ skill ประกาศกับ
สิ่งที่ดูเหมือนว่ากำลังทำอยู่ ตัวอย่างเช่น skill ที่อ้างอิงคีย์ API ที่จำเป็น
ควรประกาศข้อกำหนดนั้นใน `SKILL.md` เพื่อให้ผู้ใช้เห็นได้ก่อน
ติดตั้ง

ผลการสแกนอิงตาม artifact พฤติกรรมของ provider ที่คาดหวัง เช่น ข้อมูลลับ API
ที่ประกาศไว้, callback OAuth บน localhost, การล้างข้อมูลเมื่อถอนการติดตั้งตามขอบเขต, การเข้ารหัส Basic Auth
หรือการอัปโหลดไฟล์ที่ผู้ใช้เลือกไปยัง provider ที่ระบุไว้ จะถูกปฏิบัติ
ต่างจากการส่งต่อข้อมูลลับแบบซ่อนเร้น การเข้าถึงไฟล์ส่วนตัวในวงกว้าง
ปลายทางเครือข่ายที่ไม่เกี่ยวข้อง หรือการใช้เบราว์เซอร์ในทางที่ผิดแบบลอบทำ

ดู [รูปแบบ Skill](/th/clawhub/skill-format)

## Plugin

รีลีส Plugin ประกอบด้วยเมตาดาต้าแพ็กเกจ การระบุแหล่งที่มา ฟิลด์ความเข้ากันได้
และข้อมูลความถูกต้องครบถ้วนของ artifact

OpenClaw ตรวจสอบความเข้ากันได้ก่อนติดตั้ง plugin ที่โฮสต์บน ClawHub ระเบียนแพ็กเกจ
อาจเปิดเผยเมตาดาต้า digest เพื่อให้ OpenClaw ตรวจสอบ artifact
ที่ดาวน์โหลดมาได้ด้วย ClawScan รวมเมตาดาต้า env/config ของแพ็กเกจ `openclaw.environment`
ที่ประกาศไว้เมื่อตรวจสอบรีลีส plugin เพื่อให้ข้อกำหนด runtime
ที่ประกาศไว้ถูกเปรียบเทียบกับพฤติกรรมที่สังเกตได้

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน Skills, แพ็กเกจ และความคิดเห็นได้

รายงานควรเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้การรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

ตัวอย่างรายงาน:

- เมตาดาต้าที่ทำให้เข้าใจผิด
- ข้อกำหนดด้านข้อมูลลับหรือสิทธิ์ที่ไม่ได้ประกาศ
- คำสั่งติดตั้งที่น่าสงสัย
- ความคิดเห็นหลอกลวงหรือการแอบอ้าง
- การจดทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## หมายเหตุ ClawScan สำหรับผู้เผยแพร่

ผู้เผยแพร่สามารถให้หมายเหตุ ClawScan เพิ่มเติมเมื่อเผยแพร่ skill หรือ
plugin หมายเหตุนั้นให้บริบทแก่ ClawScan สำหรับพฤติกรรมที่มิฉะนั้นอาจดู
ผิดปกติ เช่น การเข้าถึงเครือข่าย การเข้าถึง native host หรือข้อมูลลับ
เฉพาะ provider

## การพักเพื่อกลั่นกรอง

เมื่อเครื่องสแกนแบบ static ระบุว่า skill ที่อัปโหลดเป็นอันตราย ผู้เผยแพร่จะถูก
วางไว้ภายใต้การพักเพื่อกลั่นกรองโดยอัตโนมัติ (`requiresModerationAt` ถูกตั้งบน
ผู้ใช้) การดำเนินการนี้จะซ่อน Skills ทั้งหมดของผู้เผยแพร่ ทำให้การเผยแพร่ในอนาคต
เริ่มต้นแบบซ่อน และสร้างรายการบันทึกการตรวจสอบ `user.moderation.auto`

ผลการตรวจพบแบบ static ที่น่าสงสัยจะถูกเก็บไว้เป็นหลักฐานไฟล์/บรรทัดสำหรับผู้กลั่นกรอง
แต่ผลเหล่านั้นไม่ได้ซ่อนเนื้อหาหรือตัดสิน verdict การสแกนสาธารณะด้วยตัวเอง
การอัปโหลดใหม่จะยังคงอยู่ในสถานะ review/pending จนกว่าการตรวจสอบด้วย LLM จะเสร็จสิ้น การสแกนแบบ static
จะบล็อกทันทีเฉพาะ signature ที่เป็นอันตรายเท่านั้น ผลการพบจาก engine ของ VirusTotal
ยังคงเป็นหลักฐานด้านความปลอดภัยที่มองเห็นได้ แต่ verdict จาก VirusTotal Code Insight/Palm
เป็นเพียงคำแนะนำและไม่ได้ซ่อน Skills ด้วยตัวเอง การตรวจสอบ LLM ของ ClawScan
จะเก็บหมายเหตุที่สอดคล้องกับวัตถุประสงค์ไว้เป็นแนวทาง ผลการตรวจสอบระดับกลางยังคงมองเห็นได้บน
artifact ขณะที่ตัวกรอง suspicious ถูกสงวนไว้สำหรับข้อกังวลจาก LLM
ที่มีผลกระทบสูง ผลการตรวจพบที่เป็นอันตราย หรือการตรวจพบจาก AV-engine ที่มีหลักฐานสนับสนุน

ผู้ดูแลระบบสามารถยกเลิกการพักที่เป็นผลบวกเท็จได้:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

การดำเนินการนี้จะล้าง `requiresModerationAt` และ `requiresModerationReason`, กู้คืน
Skills ที่ถูกซ่อนโดยการพักระดับผู้ใช้ และเขียนรายการบันทึกการตรวจสอบ `user.moderation.lift`
Skills ที่ถูกซ่อนด้วยเหตุผลอื่น หรือที่การสแกนแบบ static ของตัวเองยังคง
เป็นอันตราย จะยังคงถูกซ่อนอยู่

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบาย ClawHub อาจสูญเสียสิทธิ์ในการเผยแพร่ การละเมิดที่ร้ายแรง
อาจส่งผลให้ถูกแบนบัญชี เพิกถอน token ซ่อนเนื้อหา หรือถอด
รายการออก

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้ token ของ ClawHub API ได้ หากการยืนยันตัวตน CLI
เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ web UI เพื่อตรวจสอบ
สถานะบัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อก ให้ติดต่อ
security@openclaw.ai เพื่อขอการตรวจสอบการกู้คืน

## แนวทางสำหรับผู้เผยแพร่

เพื่อลดผลบวกเท็จและเพิ่มความเชื่อมั่นของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และบันทึกการเปลี่ยนแปลงให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- เพิ่มหมายเหตุ ClawScan สำหรับผู้เผยแพร่เมื่อรีลีสมีพฤติกรรมที่ผิดปกติแต่ตั้งใจไว้
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้อ่านยาก
- ลิงก์ไปยังซอร์สเมื่อทำได้
- ใช้ dry runs ก่อนเผยแพร่ plugin
- ตอบกลับอย่างชัดเจนหากผู้ใช้หรือผู้กลั่นกรองถามเกี่ยวกับพฤติกรรมของแพ็กเกจ
