# ความปลอดภัย

สามารถรายงานปัญหาความปลอดภัยของ ClawHub ผ่าน GitHub Security Advisories สำหรับ
`openclaw/clawhub`

ใช้ GitHub Security Advisories สำหรับช่องโหว่ในตัว ClawHub เอง รายงานคำแนะนำด้านความปลอดภัยของ
ClawHub ที่ดีรวมถึงบั๊กใน:

- เว็บไซต์, API หรือ CLI ของ ClawHub
- การเผยแพร่รีจิสทรี, การดาวน์โหลด, การติดตั้ง หรือความสมบูรณ์ของอาร์ติแฟกต์
- การยืนยันตัวตน, การอนุญาต หรือโทเค็น API
- การสแกน, การดูแลเนื้อหา หรือการจัดการรายงาน

อย่าใช้คำแนะนำด้านความปลอดภัยของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ Skills หรือ
Plugin จากบุคคลที่สามเอง ให้รายงานโดยตรงไปยังผู้เผยแพร่หรือรีโพซิทอรีต้นทาง
ที่ลิงก์จากรายการใน ClawHub

## การเปิดเผยช่องโหว่

เนื่องจาก ClawHub เป็นแอปพลิเคชันคลาวด์แบบโฮสต์ ช่องโหว่ของบริการ ClawHub
จะไม่ถูกเปิดเผยต่อสาธารณะโดยค่าเริ่มต้น ช่องโหว่เหล่านี้จะถูกเปิดเผยต่อสาธารณะเมื่อมี
หลักฐานว่ามีผลกระทบต่อผู้ใช้จริง หรือเมื่อผู้ใช้จำเป็นต้องดำเนินการ

ตัวอย่างของผลกระทบต่อผู้ใช้จริงรวมถึงการยืนยันว่าถูกใช้โจมตี, การเปิดเผยข้อมูลผู้ใช้
หรือความลับ, เนื้อหาที่เป็นอันตรายเข้าถึงผู้ใช้เนื่องจากความล้มเหลวของแพลตฟอร์ม
หรือปัญหาใด ๆ ที่กำหนดให้ผู้ใช้ต้องหมุนเวียนข้อมูลประจำตัว, อัปเดตซอฟต์แวร์ในเครื่อง หรือ
ดำเนินการป้องกันอื่น ๆ

ช่องโหว่ในซอฟต์แวร์ที่ผู้ใช้ติดตั้งจะถูกเปิดเผยต่อสาธารณะ เช่น
แพ็กเกจ CLI ของ ClawHub, ไบนารี, ไลบรารี หรืออาร์ติแฟกต์รุ่นอื่น ๆ ที่ผู้ใช้
ต้องอัปเดตในเครื่อง

สำหรับป้ายกำกับการตรวจสอบขณะติดตั้ง, ระดับความเสี่ยง, ข้อค้นพบ และการตีความ โปรดดู
