# ความปลอดภัย + การดูแลเนื้อหา

ClawHub เปิดให้เผยแพร่ได้ แต่รายการสาธารณะยังต้องผ่านการควบคุมด้านความน่าเชื่อถือ
การสแกน การรายงาน และการดูแลเนื้อหา เป้าหมายนั้นใช้งานได้จริง: ช่วยให้ผู้ใช้
ตรวจสอบสิ่งที่ติดตั้ง ให้เส้นทางกู้คืนแก่ผู้เผยแพร่เมื่อเกิดผลบวกลวง
และกันแพ็กเกจที่ไม่เหมาะสมออกจากการค้นพบสาธารณะ

ดูเพิ่มเติมที่ [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## สิ่งที่ผู้ใช้ตรวจสอบได้

ก่อนติดตั้ง skill หรือ Plugin ให้ตรวจสอบรายการ ClawHub สำหรับ:

- เจ้าของและการระบุแหล่งที่มา
- เวอร์ชันล่าสุดและบันทึกการเปลี่ยนแปลง
- ตัวแปรสภาพแวดล้อมหรือสิทธิ์ที่จำเป็น
- เมทาดาทาความเข้ากันได้สำหรับ Plugin
- สถานะการสแกนหรือการดูแลเนื้อหา
- รายงาน ความคิดเห็น ดาว ดาวน์โหลด และสัญญาณการติดตั้งในที่ที่แสดง

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการสแกน

ClawHub อาจแสดงผลลัพธ์การสแกนหรือการดูแลเนื้อหาบนหน้าสาธารณะและข้อมูลวินิจฉัย
ที่เจ้าของมองเห็นได้

ผลลัพธ์ทั่วไปได้แก่:

- `clean`: ไม่พบปัญหาที่ต้องบล็อก
- `suspicious`: รีลีสนี้ต้องใช้ความระมัดระวังหรือต้องตรวจสอบ
- `malicious`: รีลีสนี้ถือว่าไม่ปลอดภัย
- `pending`: การตรวจสอบยังไม่เสร็จสิ้น
- `held`, `quarantined`, `revoked`, หรือ `hidden`: รีลีสนี้ยังไม่พร้อมใช้งานอย่างเต็มที่
  บนพื้นผิวการติดตั้งสาธารณะ

ถ้อยคำที่แน่นอนอาจแตกต่างกันไปตามพื้นผิว แต่ความหมายในทางปฏิบัติเหมือนกัน: หาก
รีลีสถูกระงับหรือถูกบล็อก ผู้ใช้ไม่ควรติดตั้งจนกว่าเจ้าของจะแก้ไขปัญหา
หรือการดูแลเนื้อหาจะคืนสถานะให้

## Skills

การสแกน skill จะดูที่บันเดิล skill ที่เผยแพร่ เมทาดาทา ข้อกำหนดที่ประกาศไว้
และคำสั่งที่น่าสงสัย

ClawHub ให้ความสำคัญเป็นพิเศษกับความไม่ตรงกันระหว่างสิ่งที่ skill ประกาศ
และสิ่งที่ดูเหมือนว่าจะทำ ตัวอย่างเช่น skill ที่อ้างอิงคีย์ API ที่จำเป็น
ควรประกาศข้อกำหนดนั้นใน `SKILL.md` เพื่อให้ผู้ใช้เห็นได้ก่อนติดตั้ง

ผลการสแกนอิงจากอาร์ติแฟกต์ พฤติกรรมของผู้ให้บริการที่คาดไว้ เช่น ข้อมูลประจำตัว API
ที่ประกาศไว้ คอลแบ็ก OAuth ของ localhost การล้างข้อมูลเมื่อถอนการติดตั้งแบบมีขอบเขต
การเข้ารหัส Basic Auth หรือการอัปโหลดไฟล์ที่ผู้ใช้เลือกไปยังผู้ให้บริการที่ระบุไว้
จะถูกปฏิบัติแตกต่างจากการส่งต่อข้อมูลประจำตัวแบบซ่อน การเข้าถึงไฟล์ส่วนตัวแบบกว้าง
ปลายทางเครือข่ายที่ไม่เกี่ยวข้อง หรือการใช้เบราว์เซอร์ในทางที่ผิดแบบแอบแฝง

ดู [รูปแบบ Skill](/th/clawhub/skill-format)

## Plugins

รีลีส Plugin มีเมทาดาทาแพ็กเกจ การระบุแหล่งที่มา ฟิลด์ความเข้ากันได้
และข้อมูลความสมบูรณ์ของอาร์ติแฟกต์

OpenClaw ตรวจสอบความเข้ากันได้ก่อนติดตั้ง Plugin ที่โฮสต์บน ClawHub ระเบียนแพ็กเกจ
อาจเปิดเผยเมทาดาทา digest ด้วย เพื่อให้ OpenClaw ตรวจสอบอาร์ติแฟกต์ที่ดาวน์โหลดได้
ClawScan รวมเมทาดาทา env/config ของแพ็กเกจ `openclaw.environment` ที่ประกาศไว้
เมื่อตรวจสอบรีลีส Plugin เพื่อเปรียบเทียบข้อกำหนดรันไทม์ที่ประกาศกับพฤติกรรมที่สังเกตได้

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน skill แพ็กเกจ และความคิดเห็นได้

รายงานควรเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้การรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

ตัวอย่างรายงาน:

- เมทาดาทาที่ทำให้เข้าใจผิด
- ข้อกำหนดข้อมูลประจำตัวหรือสิทธิ์ที่ไม่ได้ประกาศ
- คำแนะนำการติดตั้งที่น่าสงสัย
- ความคิดเห็นหลอกลวงหรือการแอบอ้าง
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## หมายเหตุ ClawScan สำหรับผู้เผยแพร่

ผู้เผยแพร่สามารถให้หมายเหตุ ClawScan เพิ่มเติมได้เมื่อเผยแพร่ skill หรือ
Plugin หมายเหตุนี้ให้บริบทแก่ ClawScan สำหรับพฤติกรรมที่อาจดูผิดปกติ
เช่น การเข้าถึงเครือข่าย การเข้าถึงโฮสต์เนทีฟ หรือข้อมูลประจำตัวเฉพาะผู้ให้บริการ

## การระงับโดยการดูแลเนื้อหา

เมื่อสแกนเนอร์แบบสแตติกตั้งค่าสถานะ skill ที่อัปโหลดว่าเป็นอันตราย ผู้เผยแพร่จะถูก
วางไว้ใต้การระงับโดยการดูแลเนื้อหาโดยอัตโนมัติ (`requiresModerationAt` ถูกตั้งค่าบน
ผู้ใช้) สิ่งนี้จะซ่อน skill ทั้งหมดของผู้เผยแพร่ ทำให้การเผยแพร่ในอนาคตเริ่มต้น
แบบซ่อน และสร้างรายการบันทึกตรวจสอบ `user.moderation.auto`

ผลการค้นพบแบบสแตติกที่น่าสงสัยจะถูกเก็บไว้เป็นหลักฐานไฟล์/บรรทัดสำหรับผู้ดูแล
แต่ไม่ได้ซ่อนเนื้อหาหรือตัดสินคำตัดสินการสแกนสาธารณะด้วยตัวเอง การอัปโหลดใหม่
จะยังอยู่ในสถานะตรวจสอบ/รอดำเนินการจนกว่าการตรวจสอบ LLM จะเสร็จสิ้น การสแกน
แบบสแตติกจะบล็อกทันทีเฉพาะลายเซ็นที่เป็นอันตรายเท่านั้น การพบจากเอนจิน VirusTotal
ยังคงเป็นหลักฐานความปลอดภัยที่มองเห็นได้ แต่คำตัดสินของ VirusTotal Code Insight/Palm
เป็นเพียงคำแนะนำและไม่ได้ซ่อน skill ด้วยตัวเอง การตรวจสอบ LLM ของ ClawScan
จะเก็บหมายเหตุที่สอดคล้องกับวัตถุประสงค์ไว้เป็นแนวทาง ผลการตรวจสอบระดับกลางยังคง
มองเห็นได้บนอาร์ติแฟกต์ ในขณะที่ตัวกรองที่น่าสงสัยสงวนไว้สำหรับข้อกังวล LLM
ที่มีผลกระทบสูง ผลการค้นพบที่เป็นอันตราย หรือการตรวจพบที่ได้รับการยืนยันจาก
เอนจิน AV

ผู้ดูแลระบบสามารถยกเลิกการระงับผลบวกลวงได้:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

สิ่งนี้จะล้าง `requiresModerationAt` และ `requiresModerationReason` คืนค่า
skill ที่ถูกซ่อนโดยการระงับระดับผู้ใช้ และเขียนรายการบันทึกตรวจสอบ
`user.moderation.lift` skill ที่ถูกซ่อนด้วยเหตุผลอื่น หรือที่การสแกนแบบสแตติก
ของตัวเองยังคงเป็นอันตราย จะยังคงถูกซ่อนไว้

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบาย ClawHub อาจสูญเสียสิทธิ์การเผยแพร่ การละเมิดร้ายแรง
อาจนำไปสู่การแบนบัญชี การเพิกถอนโทเค็น การซ่อนเนื้อหา หรือการลบรายการ

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้โทเค็น ClawHub API ได้ หากการยืนยันตัวตน
CLI เริ่มล้มเหลวหลังมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ UI บนเว็บเพื่อตรวจสอบ
สถานะบัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อก ให้ติดต่อ
security@openclaw.ai เพื่อขอการตรวจสอบเพื่อกู้คืน

## แนวทางสำหรับผู้เผยแพร่

เพื่อลดผลบวกลวงและเพิ่มความเชื่อมั่นของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และบันทึกการเปลี่ยนแปลงให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- เพิ่มหมายเหตุ ClawScan สำหรับผู้เผยแพร่เมื่อรีลีสมีพฤติกรรมที่ผิดปกติแต่ตั้งใจให้เกิดขึ้น
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้อ่านยาก
- ลิงก์ไปยังซอร์สเมื่อเป็นไปได้
- ใช้ dry run ก่อนเผยแพร่ Plugin
- ตอบอย่างชัดเจนหากผู้ใช้หรือผู้ดูแลถามเกี่ยวกับพฤติกรรมของแพ็กเกจ
