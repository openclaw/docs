# ความปลอดภัย

สามารถรายงานปัญหาความปลอดภัยของ ClawHub ผ่านคำแนะนำด้านความปลอดภัยของ GitHub สำหรับ
`openclaw/clawhub`

ใช้คำแนะนำด้านความปลอดภัยของ GitHub สำหรับช่องโหว่ในตัว ClawHub เอง รายงานคำแนะนำ
ClawHub ที่ดีรวมถึงบั๊กใน:

- เว็บไซต์, API หรือ CLI ของ ClawHub
- การเผยแพร่ในรีจิสทรี การดาวน์โหลด การติดตั้ง หรือความสมบูรณ์ของอาร์ติแฟกต์
- การยืนยันตัวตน การอนุญาต หรือโทเค็น API
- การสแกน การตรวจสอบเนื้อหา หรือการจัดการรายงาน

อย่าใช้คำแนะนำของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ Skills หรือ
Plugin ของบุคคลที่สามเอง ให้รายงานโดยตรงไปยังผู้เผยแพร่หรือรีโพซิทอรีต้นทาง
ที่ลิงก์จากรายการใน ClawHub

## การเปิดเผยช่องโหว่

เนื่องจาก ClawHub เป็นแอปพลิเคชันคลาวด์แบบโฮสต์ ช่องโหว่ของบริการ ClawHub
จึงไม่ถูกเปิดเผยต่อสาธารณะโดยค่าเริ่มต้น ช่องโหว่จะถูกเปิดเผยต่อสาธารณะเมื่อมี
หลักฐานว่ากระทบผู้ใช้จริง หรือเมื่อผู้ใช้จำเป็นต้องดำเนินการ

ตัวอย่างของผลกระทบต่อผู้ใช้จริง ได้แก่ การยืนยันว่ามีการใช้ช่องโหว่ ข้อมูลหรือความลับ
ของผู้ใช้ถูกเปิดเผย เนื้อหาที่เป็นอันตรายเข้าถึงผู้ใช้เนื่องจากความล้มเหลวของแพลตฟอร์ม
หรือปัญหาใดๆ ที่ทำให้ผู้ใช้ต้องหมุนเวียนข้อมูลรับรอง อัปเดตซอฟต์แวร์ภายในเครื่อง หรือ
ดำเนินการป้องกันอื่นๆ

ช่องโหว่ในซอฟต์แวร์ที่ผู้ใช้ติดตั้งจะถูกเปิดเผยต่อสาธารณะ เช่น
แพ็กเกจ ClawHub CLI, ไบนารี, ไลบรารี หรืออาร์ติแฟกต์รุ่นเผยแพร่อื่นๆ ที่ผู้ใช้
จำเป็นต้องอัปเดตภายในเครื่อง

สำหรับป้ายกำกับการตรวจสอบขณะติดตั้ง ระดับความเสี่ยง ผลการตรวจพบ และการตีความ โปรดดู

สถานะบัญชี โปรดดู [การตรวจสอบเนื้อหาและความปลอดภัยของบัญชี](/clawhub/moderation)
