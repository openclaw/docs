# ความปลอดภัย

สามารถรายงานปัญหาความปลอดภัยของ ClawHub ผ่าน GitHub Security Advisories สำหรับ
`openclaw/clawhub`

ใช้ GitHub Security Advisories สำหรับช่องโหว่ใน ClawHub เอง รายงาน Advisory ของ
ClawHub ที่ดีควรรวมบั๊กใน:

- เว็บไซต์, API หรือ CLI ของ ClawHub
- การเผยแพร่ใน registry, การดาวน์โหลด, การติดตั้ง หรือความถูกต้องสมบูรณ์ของอาร์ติแฟกต์
- การยืนยันตัวตน, การอนุญาต หรือโทเค็น API
- การสแกน, การกลั่นกรอง หรือการจัดการรายงาน

อย่าใช้ Advisory ของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของสกิลหรือ Plugin
จากบุคคลที่สามเอง ให้รายงานโดยตรงไปยังผู้เผยแพร่หรือคลังซอร์สที่ลิงก์จากรายการใน
ClawHub

## การเปิดเผยช่องโหว่

เนื่องจาก ClawHub เป็นแอปพลิเคชันคลาวด์แบบโฮสต์ ช่องโหว่ของบริการ ClawHub
จึงไม่ถูกเปิดเผยต่อสาธารณะโดยค่าเริ่มต้น ช่องโหว่เหล่านี้จะถูกเปิดเผยต่อสาธารณะเมื่อมีหลักฐานว่ามีผลกระทบต่อผู้ใช้จริง
หรือเมื่อผู้ใช้จำเป็นต้องดำเนินการ

ตัวอย่างของผลกระทบต่อผู้ใช้จริงรวมถึงการยืนยันว่ามีการใช้ประโยชน์จากช่องโหว่,
การเปิดเผยข้อมูลหรือความลับของผู้ใช้, เนื้อหาที่เป็นอันตรายเข้าถึงผู้ใช้เนื่องจากความล้มเหลวของแพลตฟอร์ม,
หรือปัญหาใดๆ ที่ทำให้ผู้ใช้ต้องหมุนเวียนข้อมูลรับรอง, อัปเดตซอฟต์แวร์ในเครื่อง หรือ
ดำเนินการป้องกันอื่นๆ

ช่องโหว่ในซอฟต์แวร์ที่ผู้ใช้ติดตั้งจะถูกเปิดเผยต่อสาธารณะ เช่น
แพ็กเกจ CLI ของ ClawHub, ไบนารี, ไลบรารี หรืออาร์ติแฟกต์รุ่นอื่นๆ ที่ผู้ใช้
ต้องอัปเดตในเครื่อง
