# ความปลอดภัย

สามารถรายงานปัญหาความปลอดภัยของ ClawHub ผ่าน GitHub Security Advisories สำหรับ
`openclaw/clawhub`

ใช้ GitHub Security Advisories สำหรับช่องโหว่ในตัว ClawHub เอง รายงานคำแนะนำด้านความปลอดภัยของ
ClawHub ที่ดีควรรวมบั๊กใน:

- เว็บไซต์, API หรือ CLI ของ ClawHub
- การเผยแพร่รีจิสทรี, การดาวน์โหลด, การติดตั้ง หรือความถูกต้องสมบูรณ์ของอาร์ติแฟกต์
- การยืนยันตัวตน, การอนุญาต หรือโทเค็น API
- การสแกน, การกลั่นกรอง หรือการจัดการรายงาน

อย่าใช้คำแนะนำด้านความปลอดภัยของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของสกิลหรือปลั๊กอินของบุคคลที่สามเอง ให้รายงานโดยตรงต่อผู้เผยแพร่หรือรีโพซิทอรีต้นทางที่ลิงก์จากรายการใน ClawHub

## การเปิดเผยช่องโหว่

เนื่องจาก ClawHub เป็นแอปพลิเคชันคลาวด์ที่โฮสต์ไว้ ช่องโหว่ของบริการ ClawHub
จะไม่ถูกเปิดเผยต่อสาธารณะโดยค่าเริ่มต้น ช่องโหว่จะถูกเปิดเผยต่อสาธารณะเมื่อมี
หลักฐานว่ามีผลกระทบต่อผู้ใช้จริง หรือเมื่อผู้ใช้จำเป็นต้องดำเนินการ

ตัวอย่างของผลกระทบต่อผู้ใช้จริง ได้แก่ การยืนยันว่ามีการแสวงหาประโยชน์, การเปิดเผยข้อมูลหรือความลับของผู้ใช้, เนื้อหาที่เป็นอันตรายไปถึงผู้ใช้เนื่องจากความล้มเหลวของแพลตฟอร์ม,
หรือปัญหาใด ๆ ที่กำหนดให้ผู้ใช้ต้องหมุนเวียนข้อมูลประจำตัว, อัปเดตซอฟต์แวร์ในเครื่อง หรือ
ดำเนินการป้องกันอื่น ๆ

ช่องโหว่ในซอฟต์แวร์ที่ผู้ใช้ติดตั้งจะถูกเปิดเผยต่อสาธารณะ เช่น
แพ็กเกจ CLI ของ ClawHub, ไบนารี, ไลบรารี หรืออาร์ติแฟกต์เผยแพร่อื่น ๆ ที่ผู้ใช้
จำเป็นต้องอัปเดตในเครื่อง

สำหรับป้ายกำกับการตรวจสอบขณะติดตั้ง, ระดับความเสี่ยง, ข้อค้นพบ และการตีความ โปรดดู
