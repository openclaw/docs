# ความปลอดภัย

สามารถรายงานปัญหาความปลอดภัยของ ClawHub ผ่านคำแนะนำด้านความปลอดภัยของ GitHub สำหรับ
`openclaw/clawhub`

ใช้คำแนะนำด้านความปลอดภัยของ GitHub สำหรับช่องโหว่ใน ClawHub เอง รายงานคำแนะนำ
ClawHub ที่ดีรวมถึงบั๊กใน:

- เว็บไซต์ API หรือ CLI ของ ClawHub
- การเผยแพร่รีจิสทรี การดาวน์โหลด การติดตั้ง หรือความสมบูรณ์ของอาร์ติแฟกต์
- การยืนยันตัวตน การอนุญาต หรือโทเค็น API
- การสแกน การดูแลจัดการ หรือการจัดการรายงาน

อย่าใช้คำแนะนำของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของสกิลบุคคลที่สามหรือ
Plugin เอง ให้รายงานโดยตรงไปยังผู้เผยแพร่หรือคลังซอร์สที่ลิงก์จากรายการใน ClawHub

## การเปิดเผยช่องโหว่

เนื่องจาก ClawHub เป็นแอปพลิเคชันคลาวด์แบบโฮสต์ ช่องโหว่ของบริการ ClawHub
จึงไม่ถูกเปิดเผยต่อสาธารณะโดยค่าเริ่มต้น ช่องโหว่จะถูกเปิดเผยต่อสาธารณะเมื่อมี
หลักฐานว่ามีผลกระทบต่อผู้ใช้จริง หรือเมื่อผู้ใช้จำเป็นต้องดำเนินการ

ตัวอย่างของผลกระทบต่อผู้ใช้จริง ได้แก่ การยืนยันว่ามีการโจมตีสำเร็จ การเปิดเผย
ข้อมูลผู้ใช้หรือความลับ เนื้อหาประสงค์ร้ายเข้าถึงผู้ใช้เพราะความล้มเหลวของแพลตฟอร์ม
หรือปัญหาใดๆ ที่ทำให้ผู้ใช้ต้องหมุนเวียนข้อมูลประจำตัว อัปเดตซอฟต์แวร์ในเครื่อง
หรือดำเนินการป้องกันอื่นๆ

ช่องโหว่ในซอฟต์แวร์ที่ผู้ใช้ติดตั้งจะถูกเปิดเผยต่อสาธารณะ เช่น แพ็กเกจ CLI
ของ ClawHub ไบนารี ไลบรารี หรืออาร์ติแฟกต์รุ่นเผยแพร่อื่นๆ ที่ผู้ใช้จำเป็นต้อง
อัปเดตในเครื่อง
