# ความปลอดภัย

สามารถรายงานปัญหาความปลอดภัยของ ClawHub ผ่าน GitHub Security Advisories สำหรับ
`openclaw/clawhub`

ใช้ GitHub Security Advisories สำหรับช่องโหว่ในตัว ClawHub เอง รายงาน advisory ของ
ClawHub ที่ดีรวมถึงบั๊กใน:

- เว็บไซต์ API หรือ CLI ของ ClawHub
- การเผยแพร่ registry, การดาวน์โหลด, การติดตั้ง หรือความถูกต้องสมบูรณ์ของ artifact
- การยืนยันตัวตน การอนุญาตสิทธิ์ หรือ API tokens
- การสแกน การดูแลเนื้อหา หรือการจัดการรายงาน

อย่าใช้ advisory ของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ skill หรือ
plugin ของบุคคลที่สาม ให้รายงานโดยตรงต่อผู้เผยแพร่หรือ repository ต้นทาง
ที่ลิงก์จากรายการใน ClawHub

## การเปิดเผยช่องโหว่

เนื่องจาก ClawHub เป็นแอปพลิเคชันคลาวด์แบบโฮสต์ ช่องโหว่ของบริการ ClawHub
จึงไม่ถูกเปิดเผยต่อสาธารณะโดยค่าเริ่มต้น ช่องโหว่จะถูกเปิดเผยต่อสาธารณะเมื่อมี
หลักฐานว่ากระทบผู้ใช้จริง หรือเมื่อผู้ใช้จำเป็นต้องดำเนินการ

ตัวอย่างของผลกระทบต่อผู้ใช้จริงรวมถึงการยืนยันว่ามีการโจมตีเกิดขึ้น การเปิดเผย
ข้อมูลหรือความลับของผู้ใช้ เนื้อหาที่เป็นอันตรายไปถึงผู้ใช้เพราะความล้มเหลวของแพลตฟอร์ม
หรือปัญหาใดๆ ที่ทำให้ผู้ใช้ต้องหมุนเวียนข้อมูลประจำตัว อัปเดตซอฟต์แวร์ในเครื่อง หรือ
ดำเนินมาตรการป้องกันอื่นๆ

ช่องโหว่ในซอฟต์แวร์ที่ผู้ใช้ติดตั้งจะถูกเปิดเผยต่อสาธารณะ เช่น
แพ็กเกจ CLI ของ ClawHub, ไบนารี, ไลบรารี หรือ artifact รุ่นเผยแพร่อื่นๆ ที่ผู้ใช้
ต้องอัปเดตในเครื่อง
