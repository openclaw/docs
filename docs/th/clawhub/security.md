# ความปลอดภัย + การกลั่นกรอง

ClawHub เปิดให้เผยแพร่ได้ แต่รายการสาธารณะยังคงต้องผ่านการควบคุมด้านความเชื่อถือ
การสแกน การรายงาน และการกลั่นกรอง เป้าหมายคือการใช้งานจริง: ช่วยให้ผู้ใช้
ตรวจสอบสิ่งที่ติดตั้ง มอบเส้นทางกู้คืนให้ผู้เผยแพร่เมื่อเกิดผลบวกลวง
และกันแพ็กเกจที่เป็นอันตรายออกจากการค้นพบสาธารณะ

ดูเพิ่มเติมที่ [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## สิ่งที่ผู้ใช้ตรวจสอบได้

ก่อนติดตั้ง Skills หรือ Plugin ให้ตรวจสอบรายการบน ClawHub สำหรับ:

- เจ้าของและการระบุแหล่งที่มา
- เวอร์ชันล่าสุดและบันทึกการเปลี่ยนแปลง
- ตัวแปรสภาพแวดล้อมหรือสิทธิ์ที่จำเป็น
- เมทาดาทาความเข้ากันได้สำหรับ Plugin
- สถานะการสแกนหรือการกลั่นกรอง
- รายงาน ความคิดเห็น ดาว การดาวน์โหลด และสัญญาณการติดตั้งเมื่อมีการแสดง

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการสแกน

ClawHub อาจแสดงผลลัพธ์การสแกนหรือการกลั่นกรองบนหน้าสาธารณะและข้อมูลวินิจฉัย
ที่เจ้าของมองเห็นได้

ผลลัพธ์ทั่วไปได้แก่:

- `clean`: ไม่พบปัญหาที่เป็นเหตุให้บล็อก
- `suspicious`: รุ่นนี้ต้องใช้ความระมัดระวังหรือการตรวจทาน
- `malicious`: รุ่นนี้ถือว่าไม่ปลอดภัย
- `pending`: การตรวจสอบยังไม่เสร็จสิ้น
- `held`, `quarantined`, `revoked` หรือ `hidden`: รุ่นนี้ยังไม่พร้อมให้ใช้งานอย่างเต็มที่
  บนพื้นผิวการติดตั้งสาธารณะ

ถ้อยคำที่แน่นอนอาจแตกต่างกันไปตามพื้นผิว แต่ความหมายในทางปฏิบัติเหมือนกัน: หาก
รุ่นถูกระงับหรือบล็อก ผู้ใช้ไม่ควรติดตั้งจนกว่าเจ้าของจะแก้ไขปัญหา
หรือการกลั่นกรองจะคืนสถานะให้

## Skills

การสแกน Skills จะตรวจดูบันเดิล Skills ที่เผยแพร่ เมทาดาทา ข้อกำหนดที่ประกาศไว้
และคำสั่งที่น่าสงสัย

ClawHub ให้ความสำคัญเป็นพิเศษกับความไม่ตรงกันระหว่างสิ่งที่ Skills ประกาศกับ
สิ่งที่ดูเหมือนว่าทำอยู่ ตัวอย่างเช่น Skills ที่อ้างอิงคีย์ API ที่จำเป็น
ควรประกาศข้อกำหนดนั้นใน `SKILL.md` เพื่อให้ผู้ใช้เห็นได้ก่อนติดตั้ง

ผลการสแกนอ้างอิงจากอาร์ติแฟกต์ พฤติกรรมของผู้ให้บริการที่คาดไว้ เช่น ข้อมูลรับรอง API
ที่ประกาศไว้ การเรียกกลับ OAuth ไปยัง localhost การล้างข้อมูลเมื่อถอนการติดตั้งแบบจำกัดขอบเขต
การเข้ารหัส Basic Auth หรือการอัปโหลดไฟล์ที่ผู้ใช้เลือกไปยังผู้ให้บริการที่ระบุ
จะได้รับการปฏิบัติต่างจากการส่งต่อข้อมูลรับรองที่ซ่อนอยู่ การเข้าถึงไฟล์ส่วนตัวในวงกว้าง
ปลายทางเครือข่ายที่ไม่เกี่ยวข้อง หรือการใช้งานเบราว์เซอร์อย่างลับ ๆ ในทางที่ผิด

ดู [รูปแบบ Skills](/th/clawhub/skill-format)

## Plugin

รุ่น Plugin รวมถึงเมทาดาทาแพ็กเกจ การระบุแหล่งที่มา ฟิลด์ความเข้ากันได้
และข้อมูลความถูกต้องของอาร์ติแฟกต์

OpenClaw ตรวจสอบความเข้ากันได้ก่อนติดตั้ง Plugin ที่โฮสต์บน ClawHub ระเบียนแพ็กเกจ
อาจเปิดเผยเมทาดาทาไดเจสต์ด้วย เพื่อให้ OpenClaw ตรวจสอบอาร์ติแฟกต์ที่ดาวน์โหลดได้
ClawScan รวมเมทาดาทา env/config ของแพ็กเกจ `openclaw.environment` ที่ประกาศไว้
เมื่อตรวจทานรุ่น Plugin เพื่อเปรียบเทียบข้อกำหนดรันไทม์ที่ประกาศกับพฤติกรรมที่สังเกตได้

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน Skills แพ็กเกจ และความคิดเห็นได้

รายงานควรเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้การรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

ตัวอย่างรายงาน:

- เมทาดาทาที่ทำให้เข้าใจผิด
- ข้อกำหนดด้านข้อมูลรับรองหรือสิทธิ์ที่ไม่ได้ประกาศ
- คำสั่งติดตั้งที่น่าสงสัย
- ความคิดเห็นหลอกลวงหรือการแอบอ้างตัวตน
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## บันทึก ClawScan ของผู้เผยแพร่

ผู้เผยแพร่สามารถระบุบันทึก ClawScan แบบไม่บังคับเมื่อเผยแพร่ Skills หรือ
Plugin ได้ บันทึกนี้ให้บริบทแก่ ClawScan สำหรับพฤติกรรมที่อาจดูผิดปกติ
เช่น การเข้าถึงเครือข่าย การเข้าถึงโฮสต์เนทีฟ หรือข้อมูลรับรองเฉพาะผู้ให้บริการ

## การระงับเพื่อการกลั่นกรอง

เมื่อสแกนเนอร์แบบสแตติกระบุว่า Skills ที่อัปโหลดเป็นอันตราย ผู้เผยแพร่จะถูก
จัดให้อยู่ภายใต้การระงับเพื่อการกลั่นกรองโดยอัตโนมัติ (`requiresModerationAt` ตั้งค่าไว้บน
ผู้ใช้) การดำเนินการนี้จะซ่อน Skills ทั้งหมดของผู้เผยแพร่ ทำให้การเผยแพร่ในอนาคต
เริ่มต้นแบบซ่อน และสร้างรายการบันทึกการตรวจสอบ `user.moderation.auto`

ผลการค้นพบแบบสแตติกที่น่าสงสัยจะถูกเก็บไว้เป็นหลักฐานระดับไฟล์/บรรทัดสำหรับผู้กลั่นกรอง
แต่ผลเหล่านี้ไม่ได้ซ่อนเนื้อหาหรือตัดสินผลสแกนสาธารณะด้วยตัวเอง
การอัปโหลดใหม่จะยังคงอยู่ในสถานะตรวจทาน/รอดำเนินการจนกว่าการตรวจทานของ LLM จะสิ้นสุด
การสแกนแบบสแตติกจะบล็อกทันทีเฉพาะลายเซ็นที่เป็นอันตรายเท่านั้น ผลตรวจพบจากเอนจิน
VirusTotal ยังคงแสดงเป็นหลักฐานด้านความปลอดภัยที่มองเห็นได้ แต่คำตัดสินจาก
VirusTotal Code Insight/Palm เป็นคำแนะนำและไม่ได้ซ่อน Skills ด้วยตัวเอง การตรวจทาน
LLM ของ ClawScan จะเก็บบันทึกที่สอดคล้องกับวัตถุประสงค์ไว้เป็นแนวทาง ผลการตรวจทานระดับกลาง
ยังคงมองเห็นได้บนอาร์ติแฟกต์ ขณะที่ตัวกรองน่าสงสัยสงวนไว้สำหรับข้อกังวลจาก LLM
ที่มีผลกระทบสูง ผลการค้นพบที่เป็นอันตราย หรือการตรวจพบจากเอนจิน AV ที่มีหลักฐานสนับสนุน

ผู้ดูแลระบบสามารถยกเลิกการระงับที่เป็นผลบวกลวงได้:

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

การดำเนินการนี้ล้าง `requiresModerationAt` และ `requiresModerationReason`, กู้คืน
Skills ที่ถูกซ่อนโดยการระงับระดับผู้ใช้ และเขียนรายการบันทึกการตรวจสอบ `user.moderation.lift`
Skills ที่ถูกซ่อนด้วยเหตุผลอื่น หรือที่การสแกนแบบสแตติกของตัวเองยังคงเป็นอันตราย
จะยังคงถูกซ่อนอยู่

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบายของ ClawHub อาจสูญเสียสิทธิ์การเผยแพร่ การละเมิดร้ายแรง
อาจทำให้บัญชีถูกแบน โทเคนถูกเพิกถอน เนื้อหาถูกซ่อน หรือรายการถูกลบออก

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้โทเคน API ของ ClawHub ได้ หากการยืนยันตัวตน CLI
เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้เว็บ UI เพื่อตรวจทาน
สถานะบัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อก ให้ติดต่อ
security@openclaw.ai เพื่อขอการตรวจทานการกู้คืน

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลดผลบวกลวงและเพิ่มความเชื่อมั่นของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และบันทึกการเปลี่ยนแปลงให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- เพิ่มบันทึก ClawScan ของผู้เผยแพร่เมื่อรุ่นมีพฤติกรรมที่ผิดปกติแต่ตั้งใจไว้
- หลีกเลี่ยงคำสั่งติดตั้งที่ถูกทำให้อ่านยาก
- ลิงก์ไปยังแหล่งที่มาเมื่อเป็นไปได้
- ใช้การทดลองรันก่อนเผยแพร่ Plugin
- ตอบอย่างชัดเจนหากผู้ใช้หรือผู้กลั่นกรองถามเกี่ยวกับพฤติกรรมของแพ็กเกจ
