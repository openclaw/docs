# ความปลอดภัย

สามารถรายงานปัญหาความปลอดภัยของ ClawHub ผ่าน GitHub Security Advisories สำหรับ
`openclaw/clawhub`

ใช้ GitHub Security Advisories สำหรับช่องโหว่ใน ClawHub เอง รายงาน advisory ของ
ClawHub ที่ดีรวมถึงบั๊กใน:

- เว็บไซต์, API หรือ CLI ของ ClawHub
- การเผยแพร่ในรีจิสทรี, การดาวน์โหลด, การติดตั้ง หรือความสมบูรณ์ของอาร์ติแฟกต์
- การยืนยันตัวตน, การอนุญาต หรือโทเค็น API
- การสแกน, การดูแลเนื้อหา หรือการจัดการรายงาน

อย่าใช้ advisory ของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ Skills หรือ
Plugin ของบุคคลที่สามเอง ให้รายงานโดยตรงต่อผู้เผยแพร่หรือที่เก็บซอร์ส
ที่ลิงก์จากรายการบน ClawHub

## การเปิดเผยช่องโหว่

เนื่องจาก ClawHub เป็นแอปพลิเคชันคลาวด์แบบโฮสต์ ช่องโหว่ของบริการ ClawHub
จะไม่ถูกเปิดเผยต่อสาธารณะโดยค่าเริ่มต้น ช่องโหว่เหล่านี้จะถูกเปิดเผยต่อสาธารณะเมื่อมี
หลักฐานว่ากระทบผู้ใช้จริง หรือเมื่อผู้ใช้จำเป็นต้องดำเนินการ

ตัวอย่างของผลกระทบต่อผู้ใช้จริงรวมถึงการยืนยันว่ามีการใช้ประโยชน์จากช่องโหว่,
การเปิดเผยข้อมูลหรือความลับของผู้ใช้, เนื้อหาประสงค์ร้ายเข้าถึงผู้ใช้เนื่องจากความล้มเหลวของแพลตฟอร์ม,
หรือปัญหาใดๆ ที่ทำให้ผู้ใช้ต้องหมุนเวียนข้อมูลประจำตัว, อัปเดตซอฟต์แวร์ภายในเครื่อง หรือ
ดำเนินการป้องกันอื่นๆ

ช่องโหว่ในซอฟต์แวร์ที่ผู้ใช้ติดตั้งจะถูกเปิดเผยต่อสาธารณะ เช่น
แพ็กเกจ CLI ของ ClawHub, ไบนารี, ไลบรารี หรืออาร์ติแฟกต์เผยแพร่อื่นๆ ที่ผู้ใช้
ต้องอัปเดตภายในเครื่อง

## หน้าที่เกี่ยวข้อง

สำหรับป้ายกำกับการตรวจสอบขณะติดตั้ง, ระดับความเสี่ยง, สิ่งที่พบ และการตีความ โปรดดู
