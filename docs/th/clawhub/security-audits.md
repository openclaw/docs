# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจได้ว่า Skills หรือ Plugin ปลอดภัยเพียงพอ
ที่จะติดตั้งหรือไม่ การตรวจสอบเหล่านี้แสดงว่ารีลีสทำอะไร ขออำนาจใดบ้าง และ
มีสิ่งใดที่ควรให้ความสนใจเป็นพิเศษก่อนที่จะเข้าถึงไฟล์ บัญชี
ข้อมูลประจำตัว โค้ด หรือบริการภายนอกหรือไม่

การตรวจสอบเป็นสัญญาณด้านความปลอดภัยที่หนักแน่น แต่ไม่ได้รับประกันว่ารีลีสจะ
ปราศจากความเสี่ยงโดยสิ้นเชิง โปรดใช้วิจารณญาณเสมอก่อนให้สิทธิ์เข้าถึงข้อมูลสำคัญ

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security), [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage),
และ [การกลั่นกรองและความปลอดภัยของบัญชี](/clawhub/moderation)

## สิ่งที่ควรตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจสอบ:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- ข้อค้นพบที่ระบุไว้
- ข้อมูลประจำตัว สิทธิ์ หรือ environment variables ที่จำเป็น
- เจ้าของ แหล่งที่มา เวอร์ชัน บันทึกการเปลี่ยนแปลง ยอดดาวน์โหลด ดาว และสัญญาณความน่าเชื่อถืออื่นๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและไว้วางใจเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกคุณว่าควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้ซึ่งสูงกว่าความเสี่ยงต่ำ                                |
| `Review`    | อ่านข้อค้นพบก่อนติดตั้ง รีลีสอาจยังคงถูกต้องตามวัตถุประสงค์ |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบข้อกังวลที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | ห้ามติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จสิ้น                                             |
| `Error`     | ไม่สามารถดำเนินการตรวจสอบให้เสร็จสมบูรณ์ได้                                         |

`Pass` ช่วยให้มั่นใจได้ แต่ไม่ได้แทนที่วิจารณญาณของคุณเอง สิ่งนี้สำคัญ
ที่สุดสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล รันคำสั่ง อ่านไฟล์ หรือ
เข้าถึงระบบโปรดักชันได้

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบาย blast radius: รีลีสดูเหมือนมีอำนาจมากเพียงใดหาก
คุณใช้งานตามที่ตั้งใจไว้

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบอำนาจที่ละเอียดอ่อนหรือผลกระทบต่อผู้ใช้น้อยมาก                          |
| `Medium`   | รีลีสมีอำนาจที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | รีลีสมีอำนาจที่มีผลกระทบสูง ข้อค้นพบร้ายแรง หรือสัญญาณมุ่งร้าย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคำถามคนละข้อกัน:

- ระดับความเสี่ยงถามว่า: "ที่นี่มีอำนาจมากเพียงใด?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอย่างไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น Skills สำหรับการเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยง `Medium` ซึ่ง
ไม่ได้หมายความว่ามุ่งร้าย หมายความว่า Skills ดูสอดคล้องกับวัตถุประสงค์ แต่สามารถ
ดำเนินการด้วยอำนาจบัญชีที่มีนัยสำคัญได้

## ข้อค้นพบ

ข้อค้นพบอธิบายว่าเหตุใดจึงแสดงผลการตรวจสอบนั้น ข้อค้นพบแต่ละรายการมักประกอบด้วย:

- ความหมายของข้อค้นพบ
- เหตุผลที่ถูกตั้งค่าสถานะ
- เนื้อหา Skills หรือ Plugin ที่เกี่ยวข้อง
- คำแนะนำ

ข้อค้นพบอาจติดป้ายกำกับเป็น `Info`, `Low`, `Medium`, `High` หรือ `Critical` ข้อค้นพบที่มี
ความรุนแรงสูงกว่าจะมีส่วนต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

ข้อค้นพบที่มีความเชื่อมั่นต่ำจะถูกซ่อนจากสรุปการตรวจสอบสาธารณะ เพื่อให้หน้า
มุ่งเน้นหลักฐานที่เป็นประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบอาร์ติแฟกต์รีลีสที่ส่งเข้ามา รวมถึง:

- คำสั่ง Skills หรือเมตาดาต้า Plugin
- environment variables และสิทธิ์ที่ประกาศไว้
- คำแนะนำการติดตั้งและเมตาดาต้าแพ็กเกจ
- ไฟล์ที่รวมไว้และรายการไฟล์
- เมตาดาต้าความเข้ากันได้และความสามารถ

คำถามหลักคือความสอดคล้อง: ชื่อ สรุป เมตาดาต้า อำนาจที่ร้องขอ
และเนื้อหาจริงสอดคล้องกับสิ่งที่ผู้ใช้คาดหวังได้อย่างสมเหตุสมผลหรือไม่?

พฤติกรรมที่ทรงพลังไม่ได้แย่โดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลประจำตัว
คำสั่งในเครื่อง API ของผู้ให้บริการ หรือการติดตั้งแพ็กเกจ การตรวจสอบจะตรวจสอบว่า
อำนาจนั้นเป็นสิ่งที่คาดหมาย เปิดเผยไว้ และได้สัดส่วนหรือไม่

หน้าอาร์ติแฟกต์ลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบรวม:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็น telemetry มัลแวร์ในชุดการตรวจสอบ VirusTotal เป็น
มาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับชื่อเสียงไฟล์และการสแกนมัลแวร์ และ
ความร่วมมือของเราช่วยให้ ClawHub เพิ่มข่าวกรองด้านความปลอดภัยที่กว้างขึ้นในการตรวจสอบ Skills และ Plugin

VirusTotal มีประโยชน์เป็นพิเศษสำหรับอาร์ติแฟกต์มุ่งร้ายที่รู้จักอยู่แล้ว การพบโดยเอนจิน และ
สัญญาณชื่อเสียงที่เสริมการตรวจสอบที่เข้าใจเอเจนต์ของ ClawHub เมื่อมีจำนวน
เอนจินจากผู้ให้บริการ การตรวจสอบจะสรุปเป็นภาษาธรรมดา เช่น:

```text
62/62 vendors flagged this skill as clean.
```

หรือ:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

เมื่อ ClawHub ไม่มี telemetry จำนวนผู้ให้บริการให้สรุป การตรวจสอบจะระบุว่า:

```text
No VirusTotal findings
```

VirusTotal ยังคงเป็น telemetry โดยไม่ได้แทนที่การวิเคราะห์ความเสี่ยง
ที่เข้าใจอาร์ติแฟกต์ของ ClawHub เอง

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัย
ของ ClawHub เอง ระบบจะตรวจสอบแต่ละรีลีสในฐานะอาร์ติแฟกต์ที่หันหน้าเข้าหาเอเจนต์: คำสั่ง
เมตาดาต้า สิทธิ์ที่ประกาศไว้ ไฟล์ สัญญาณความสามารถ สัญญาณการสแกนแบบสแตติก
ข้อค้นพบของ SkillSpector, telemetry ของ VirusTotal และบริบทที่ผู้เผยแพร่ให้ไว้
สัญญาณการสแกนแบบสแตติกเป็นบริบทภายในสำหรับการตรวจสอบนี้ ไม่ใช่
ส่วนการตรวจสอบสาธารณะแบบแยกต่างหากหรือคำตัดสินที่บล็อกการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นมุมมองสำหรับความเสี่ยงต่างๆ เช่น prompt injection การใช้เครื่องมือในทางที่ผิด การเปิดเผยข้อมูลประจำตัว
การดำเนินการที่ไม่ปลอดภัย การปนเปื้อนหน่วยความจำหรือบริบท และการมีอิสระในการกระทำมากเกินไป

ClawScan ไม่ถือว่าความสามารถที่ดูน่ากลัวเป็นมุ่งร้ายโดยอัตโนมัติ
แต่จะถามว่าความสามารถนั้นถูกเปิดเผย สอดคล้องกับวัตถุประสงค์ และได้รับการสนับสนุนโดย
กรณีการใช้งานที่รีลีสระบุไว้หรือไม่
