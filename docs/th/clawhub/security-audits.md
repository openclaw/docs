# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจได้ว่าทักษะหรือ Plugin ปลอดภัยพอ
ที่จะติดตั้งหรือไม่ การตรวจสอบจะแสดงว่ารุ่นเผยแพร่ทำอะไร ขอสิทธิ์อำนาจใด และ
มีสิ่งใดที่ควรได้รับความสนใจเพิ่มเติมก่อนที่จะเข้าถึงไฟล์ บัญชี
ข้อมูลลับ โค้ด หรือบริการภายนอกหรือไม่

การตรวจสอบเป็นสัญญาณความปลอดภัยที่หนักแน่น แต่ไม่ได้รับประกันว่ารุ่นเผยแพร่จะ
ปราศจากความเสี่ยงเสมอไป ใช้วิจารณญาณเสมอก่อนให้สิทธิ์เข้าถึงที่ละเอียดอ่อน

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security), [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage),
และ [การกลั่นกรองและความปลอดภัยของบัญชี](/clawhub/moderation)

## สิ่งที่ควรตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจสอบ:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- ผลการตรวจสอบที่ระบุไว้
- ข้อมูลลับ สิทธิ์อนุญาต หรือตัวแปรสภาพแวดล้อมที่จำเป็น
- เจ้าของ แหล่งที่มา เวอร์ชัน บันทึกการเปลี่ยนแปลง ยอดดาวน์โหลด ดาว และสัญญาณความน่าเชื่อถืออื่นๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกคุณว่าควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้เหนือความเสี่ยงต่ำ                                |
| `Review`    | อ่านผลการตรวจสอบก่อนติดตั้ง รุ่นเผยแพร่อาจยังถูกต้องชอบธรรมอยู่ |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบข้อกังวลที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | ห้ามติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จสิ้น                                             |
| `Error`     | ไม่สามารถดำเนินการตรวจสอบให้เสร็จสมบูรณ์ได้                                         |

`Pass` ทำให้อุ่นใจได้ แต่ไม่ได้แทนที่วิจารณญาณของคุณเอง สิ่งนี้สำคัญ
ที่สุดสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล รันคำสั่ง อ่านไฟล์ หรือ
เข้าถึงระบบโปรดักชัน

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบายขอบเขตผลกระทบ: รุ่นเผยแพร่ดูเหมือนมีอำนาจมากเพียงใดหาก
คุณใช้งานตามที่ตั้งใจไว้

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบอำนาจที่ละเอียดอ่อนหรือผลกระทบต่อผู้ใช้น้อย                          |
| `Medium`   | รุ่นเผยแพร่มีอำนาจที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | รุ่นเผยแพร่มีอำนาจที่มีผลกระทบสูง ผลการตรวจสอบร้ายแรง หรือสัญญาณมุ่งร้าย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคนละคำถามกัน:

- ระดับความเสี่ยงถามว่า: "มีอำนาจมากเพียงใดที่นี่?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอย่างไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น ทักษะสำหรับการเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยง `Medium` นั่น
ไม่ได้หมายความว่ามุ่งร้าย แต่หมายความว่าทักษะดูสอดคล้องกับวัตถุประสงค์ แต่สามารถ
ดำเนินการด้วยอำนาจระดับบัญชีที่มีนัยสำคัญได้

## ผลการตรวจสอบ

ผลการตรวจสอบอธิบายว่าทำไมผลการตรวจสอบจึงแสดงขึ้น ผลการตรวจสอบแต่ละรายการมักประกอบด้วย:

- หมายความว่าอะไร
- เหตุใดจึงถูกตั้งค่าสถานะ
- เนื้อหาของทักษะหรือ Plugin ที่เกี่ยวข้อง
- คำแนะนำ

ผลการตรวจสอบอาจติดป้าย `Info`, `Low`, `Medium`, `High` หรือ `Critical` ผลการตรวจสอบที่มี
ความรุนแรงสูงกว่าจะมีผลต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

ผลการตรวจสอบที่มีความมั่นใจต่ำจะถูกซ่อนจากสรุปการตรวจสอบสาธารณะ เพื่อให้หน้า
มุ่งเน้นหลักฐานที่เป็นประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบอาร์ติแฟกต์รุ่นเผยแพร่ที่ส่งเข้ามา รวมถึง:

- คำสั่งของทักษะหรือเมตาดาตาของ Plugin
- ตัวแปรสภาพแวดล้อมและสิทธิ์อนุญาตที่ประกาศไว้
- คำสั่งติดตั้งและเมตาดาตาแพ็กเกจ
- ไฟล์ที่รวมอยู่และแมนิเฟสต์ไฟล์
- เมตาดาตาความเข้ากันได้และความสามารถ

คำถามหลักคือความสอดคล้อง: ชื่อ สรุป เมตาดาตา อำนาจที่ร้องขอ
และเนื้อหาจริงสอดคล้องกับสิ่งที่ผู้ใช้คาดหวังได้อย่างสมเหตุสมผลหรือไม่?

พฤติกรรมที่ทรงพลังไม่ได้แย่โดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลลับ
คำสั่งในเครื่อง API ของผู้ให้บริการ หรือการติดตั้งแพ็กเกจ การตรวจสอบจะตรวจว่าพลังนั้น
เป็นสิ่งที่คาดหวัง เปิดเผยไว้ และได้สัดส่วนหรือไม่

หน้าอาร์ติแฟกต์ลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบรวม:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็นเทเลเมทรีมัลแวร์ในชุดการตรวจสอบ VirusTotal เป็น
มาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับชื่อเสียงของไฟล์และการสแกนมัลแวร์ และ
ความร่วมมือของเราช่วยให้ ClawHub เพิ่มข่าวกรองความปลอดภัยที่กว้างขึ้นในการรีวิวทักษะและ Plugin

VirusTotal มีประโยชน์เป็นพิเศษสำหรับอาร์ติแฟกต์มุ่งร้ายที่เป็นที่รู้จัก การตรวจพบจากเอนจิน และ
สัญญาณชื่อเสียงที่เสริมการรีวิวแบบเข้าใจเอเจนต์ของ ClawHub เมื่อมีจำนวนจากเอนจินของผู้ให้บริการ
การตรวจสอบจะสรุปเป็นภาษาธรรมดา เช่น:

```text
62/62 vendors flagged this skill as clean.
```

หรือ:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

เมื่อ ClawHub ไม่มีเทเลเมทรีจำนวนผู้ให้บริการให้สรุป การตรวจสอบจะระบุว่า:

```text
No VirusTotal findings
```

VirusTotal ยังคงเป็นเทเลเมทรี ไม่ได้แทนที่การวิเคราะห์ความเสี่ยงแบบเข้าใจอาร์ติแฟกต์
ของ ClawHub เอง

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ระบบตรวจสอบความปลอดภัยของ
ClawHub เอง ระบบจะรีวิวแต่ละรุ่นเผยแพร่ในฐานะอาร์ติแฟกต์ที่หันหน้าเข้าหาเอเจนต์: คำสั่ง
เมตาดาตา สิทธิ์อนุญาตที่ประกาศไว้ ไฟล์ สัญญาณความสามารถ สัญญาณสแกนแบบสถิต
ผลการตรวจสอบจาก SkillSpector เทเลเมทรีจาก VirusTotal และบริบทที่ผู้เผยแพร่ให้มา
สัญญาณสแกนแบบสถิตเป็นบริบทภายในสำหรับการรีวิวนี้ ไม่ใช่
ส่วนการตรวจสอบสาธารณะแบบสแตนด์อโลนหรือคำตัดสินที่บล็อกการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นเลนส์สำหรับความเสี่ยง เช่น การฉีดพรอมป์ การใช้เครื่องมือผิดวัตถุประสงค์ การเปิดเผยข้อมูลลับ
การดำเนินการที่ไม่ปลอดภัย การปนเปื้อนหน่วยความจำหรือบริบท และอำนาจกระทำการที่มากเกินไป

ClawScan ไม่ถือว่าความสามารถที่ดูน่ากลัวเป็นสิ่งมุ่งร้ายโดยอัตโนมัติ
ระบบจะถามว่าความสามารถนั้นถูกเปิดเผย สอดคล้องกับวัตถุประสงค์ และได้รับการสนับสนุนโดย
กรณีการใช้งานที่รุ่นเผยแพร่ระบุไว้หรือไม่
