# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจว่า skill หรือ plugin ปลอดภัยพอ
ที่จะติดตั้งหรือไม่ การตรวจสอบจะแสดงว่า release ทำอะไร ขอสิทธิ์อะไรบ้าง และ
มีสิ่งใดที่ควรพิจารณาเป็นพิเศษก่อนที่มันจะเข้าถึงไฟล์ บัญชี
ข้อมูลประจำตัว โค้ด หรือบริการภายนอกหรือไม่

การตรวจสอบเป็นสัญญาณด้านความปลอดภัยที่หนักแน่น แต่ไม่ใช่การรับประกันว่า release นั้น
ปราศจากความเสี่ยงเสมอ ใช้วิจารณญาณก่อนให้สิทธิ์เข้าถึงข้อมูลหรือระบบที่ละเอียดอ่อนทุกครั้ง

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security), [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage),
และ [การดูแลเนื้อหาและความปลอดภัยของบัญชี](/clawhub/moderation)

## สิ่งที่ต้องตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจทาน:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- findings ที่ระบุไว้
- ข้อมูลประจำตัว สิทธิ์ หรือ environment variables ที่ต้องใช้
- เจ้าของ แหล่งที่มา เวอร์ชัน changelog ยอดดาวน์โหลด ดาว และสัญญาณความน่าเชื่อถืออื่นๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกคุณว่าควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้ซึ่งสูงกว่าความเสี่ยงต่ำ                                |
| `Review`    | อ่าน findings ก่อนติดตั้ง release อาจยังถูกต้องตามเจตนาได้ |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบข้อกังวลที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | อย่าติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จสิ้น                                             |
| `Error`     | ไม่สามารถดำเนินการตรวจสอบให้เสร็จได้                                         |

`Pass` ช่วยให้มั่นใจได้ แต่ไม่แทนที่วิจารณญาณของคุณเอง เรื่องนี้สำคัญ
ที่สุดสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล รันคำสั่ง อ่านไฟล์ หรือ
เข้าถึงระบบ production ได้

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบาย blast radius: release ดูเหมือนมีอำนาจมากเพียงใดหาก
คุณใช้งานตามเจตนา

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบอำนาจที่ละเอียดอ่อนหรือผลกระทบต่อผู้ใช้น้อย                          |
| `Medium`   | release มีอำนาจที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | release มีอำนาจที่มีผลกระทบสูง findings ร้ายแรง หรือสัญญาณว่าเป็นอันตราย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคำถามต่างกัน:

- ระดับความเสี่ยงถามว่า: "ที่นี่มีอำนาจมากแค่ไหน?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอย่างไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น skill สำหรับเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยง `Medium` ซึ่ง
ไม่ได้หมายความว่ามันเป็นอันตราย แต่หมายความว่า skill ดูสอดคล้องกับวัตถุประสงค์
แต่สามารถดำเนินการด้วยอำนาจบัญชีที่มีนัยสำคัญได้

## Findings

Findings อธิบายว่าเหตุใดจึงแสดงผลการตรวจสอบนั้น โดยทั่วไปแต่ละ finding จะรวมถึง:

- หมายความว่าอะไร
- เหตุใดจึงถูกทำเครื่องหมาย
- เนื้อหา skill หรือ plugin ที่เกี่ยวข้อง
- คำแนะนำ

Findings อาจมีป้ายกำกับเป็น `Info`, `Low`, `Medium`, `High` หรือ `Critical` findings
ที่มีความรุนแรงสูงกว่าจะส่งผลต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

Findings ที่มีความเชื่อมั่นต่ำจะถูกซ่อนจากสรุปการตรวจสอบสาธารณะ เพื่อให้หน้า
มุ่งเน้นหลักฐานที่มีประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบ artifact ของ release ที่ส่งมา รวมถึง:

- คำสั่งของ skill หรือ metadata ของ plugin
- environment variables และสิทธิ์ที่ประกาศไว้
- คำแนะนำการติดตั้งและ metadata ของแพ็กเกจ
- ไฟล์ที่รวมมาและ file manifests
- metadata ด้านความเข้ากันได้และความสามารถ

คำถามหลักคือความสอดคล้อง: ชื่อ สรุป metadata อำนาจที่ขอ
และเนื้อหาจริงสอดคล้องกับสิ่งที่ผู้ใช้คาดหวังได้อย่างสมเหตุสมผลหรือไม่?

พฤติกรรมที่ทรงพลังไม่ได้แย่โดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลประจำตัว
คำสั่งในเครื่อง provider APIs หรือการติดตั้งแพ็กเกจ การตรวจสอบจะตรวจว่าพลังนั้น
เป็นสิ่งที่คาดหมาย เปิดเผยไว้ และได้สัดส่วนหรือไม่

หน้า artifact จะลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบรวม:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็น telemetry ด้านมัลแวร์ใน audit stack VirusTotal เป็น
มาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับชื่อเสียงของไฟล์และการสแกนมัลแวร์ และความร่วมมือของเรา
ทำให้ ClawHub เพิ่มข้อมูลข่าวกรองด้านความปลอดภัยที่กว้างขึ้นในการตรวจทาน skill และ plugin ได้

VirusTotal มีประโยชน์เป็นพิเศษสำหรับ artifacts ที่ทราบว่าเป็นอันตราย engine hits และ
สัญญาณชื่อเสียงที่เสริมการตรวจทานแบบเข้าใจ agent ของ ClawHub เมื่อมีจำนวนจาก vendor
engines การตรวจสอบจะสรุปเป็นภาษาธรรมดา เช่น:

```text
62/62 vendors flagged this skill as clean.
```

หรือ:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

เมื่อ ClawHub ไม่มี telemetry จำนวน vendor ให้สรุป การตรวจสอบจะระบุว่า:

```text
No VirusTotal findings
```

VirusTotal ยังคงเป็น telemetry ไม่ได้แทนที่การวิเคราะห์ความเสี่ยงของ ClawHub เอง
ที่เข้าใจ artifact

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัยของ
ClawHub เอง ระบบจะตรวจทานแต่ละ release ในฐานะ artifact ที่หันหน้าเข้าหา agent:
คำสั่ง metadata สิทธิ์ที่ประกาศ ไฟล์ สัญญาณความสามารถ สัญญาณ static scan
findings จาก SkillSpector, telemetry จาก VirusTotal และบริบทที่ผู้เผยแพร่ให้มา
สัญญาณ static scan เป็นบริบทภายในสำหรับการตรวจทานนี้ ไม่ใช่ส่วนการตรวจสอบสาธารณะ
แบบแยกเดี่ยวหรือคำตัดสินที่บล็อกการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นเลนส์สำหรับความเสี่ยง เช่น prompt injection การใช้เครื่องมือผิดวัตถุประสงค์ การเปิดเผยข้อมูลประจำตัว
การดำเนินการที่ไม่ปลอดภัย การปนเปื้อน memory หรือ context และการมีอำนาจดำเนินการมากเกินไป

ClawScan ไม่ถือว่าความสามารถที่ดูน่ากลัวเป็นอันตรายโดยอัตโนมัติ
แต่จะถามว่าความสามารถนั้นถูกเปิดเผย สอดคล้องกับวัตถุประสงค์ และได้รับการสนับสนุนโดย
กรณีการใช้งานที่ release ระบุไว้หรือไม่
