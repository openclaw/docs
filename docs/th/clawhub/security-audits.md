# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจได้ว่าทักษะหรือ Plugin ปลอดภัยพอที่จะติดตั้งหรือไม่ การตรวจสอบจะแสดงว่ารุ่นเผยแพร่ทำอะไร ขอสิทธิ์อะไร และมีสิ่งใดที่ควรให้ความสนใจเป็นพิเศษก่อนที่จะเข้าถึงไฟล์ บัญชี ข้อมูลรับรอง โค้ด หรือบริการภายนอกหรือไม่

การตรวจสอบเป็นสัญญาณความปลอดภัยที่สำคัญ แต่ไม่ได้รับประกันว่ารุ่นเผยแพร่จะปราศจากความเสี่ยง ใช้วิจารณญาณเสมอก่อนให้สิทธิ์เข้าถึงข้อมูลที่ละเอียดอ่อน

ดูเพิ่มเติมที่ [ความปลอดภัย](/th/clawhub/security), [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage) และ [การดูแลเนื้อหาและความปลอดภัยของบัญชี](/th/clawhub/moderation)

## สิ่งที่ควรตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจสอบ:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- findings ที่ระบุไว้
- ข้อมูลรับรอง สิทธิ์ หรือ environment variables ที่ต้องใช้
- เจ้าของ แหล่งที่มา เวอร์ชัน changelog ยอดดาวน์โหลด ดาว และสัญญาณความน่าเชื่อถืออื่น ๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกคุณว่าควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้ซึ่งสูงกว่าความเสี่ยงต่ำ                                |
| `Review`    | อ่าน findings ก่อนติดตั้ง รุ่นเผยแพร่อาจยังคงถูกต้องตามเจตนา |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบข้อกังวลที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | อย่าติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จสิ้น                                             |
| `Error`     | ไม่สามารถทำการตรวจสอบให้เสร็จสมบูรณ์ได้                                         |

`Pass` ช่วยให้มั่นใจได้ แต่ไม่แทนที่วิจารณญาณของคุณเอง เรื่องนี้สำคัญที่สุดสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล รันคำสั่ง อ่านไฟล์ หรือเข้าถึงระบบ production ได้

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบาย blast radius: รุ่นเผยแพร่ดูเหมือนจะมีอำนาจมากเพียงใดหากคุณใช้งานตามวัตถุประสงค์

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบอำนาจที่ละเอียดอ่อนหรือผลกระทบต่อผู้ใช้น้อย                          |
| `Medium`   | รุ่นเผยแพร่มีอำนาจที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | รุ่นเผยแพร่มีอำนาจที่มีผลกระทบสูง findings ร้ายแรง หรือสัญญาณที่เป็นอันตราย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคำถามคนละอย่าง:

- ระดับความเสี่ยงถามว่า: "ที่นี่มีอำนาจมากแค่ไหน?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอะไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น ทักษะสำหรับการเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยง `Medium` นั่นไม่ได้หมายความว่าทักษะนั้นเป็นอันตราย หมายความว่าทักษะนั้นดูสอดคล้องกับวัตถุประสงค์ แต่สามารถดำเนินการด้วยอำนาจบัญชีที่มีนัยสำคัญได้

## Findings

Findings อธิบายว่าทำไมจึงแสดงผลการตรวจสอบนั้น โดยปกติ finding แต่ละรายการจะมี:

- หมายความว่าอะไร
- เหตุใดจึงถูก flag
- เนื้อหาทักษะหรือ Plugin ที่เกี่ยวข้อง
- คำแนะนำ

Findings อาจมีป้ายกำกับ `Info`, `Low`, `Medium`, `High` หรือ `Critical` findings ที่มีความรุนแรงสูงกว่าจะมีผลต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

findings ที่มีความเชื่อมั่นต่ำจะถูกซ่อนจาก rollup การตรวจสอบสาธารณะ เพื่อให้หน้าดังกล่าวมุ่งเน้นหลักฐานที่มีประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบอาร์ติแฟกต์รุ่นเผยแพร่ที่ส่งเข้ามา รวมถึง:

- คำสั่งของทักษะหรือ metadata ของ Plugin
- environment variables และสิทธิ์ที่ประกาศไว้
- คำสั่งติดตั้งและ metadata ของแพ็กเกจ
- ไฟล์ที่รวมอยู่และ file manifests
- metadata ด้านความเข้ากันได้และความสามารถ

คำถามหลักคือความสอดคล้อง: ชื่อ สรุป metadata อำนาจที่ขอ และเนื้อหาจริงตรงกับสิ่งที่ผู้ใช้คาดหวังอย่างสมเหตุสมผลหรือไม่?

พฤติกรรมที่มีอำนาจสูงไม่ได้แย่โดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลรับรอง คำสั่งในเครื่อง provider APIs หรือการติดตั้งแพ็กเกจ การตรวจสอบจะดูว่าอำนาจนั้นเป็นสิ่งที่คาดหวัง เปิดเผยไว้ และได้สัดส่วนหรือไม่

หน้าอาร์ติแฟกต์จะลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบรวม:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็น telemetry ด้านมัลแวร์ใน audit stack VirusTotal เป็นมาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับชื่อเสียงของไฟล์และการสแกนมัลแวร์ และความร่วมมือของเราช่วยให้ ClawHub เพิ่มข้อมูลข่าวกรองด้านความปลอดภัยที่กว้างขึ้นในการรีวิวทักษะและ Plugin

VirusTotal มีประโยชน์เป็นพิเศษสำหรับอาร์ติแฟกต์ที่ทราบว่าเป็นอันตราย การตรวจพบจาก engine และสัญญาณชื่อเสียงที่เสริมการรีวิวที่เข้าใจเอเจนต์ของ ClawHub เมื่อมีจำนวน vendor engine การตรวจสอบจะสรุปเป็นภาษาธรรมดา เช่น:

```text
62/62 vendors flagged this skill as clean.
```

หรือ:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

เมื่อ ClawHub ไม่มี telemetry จำนวน vendor ให้สรุป การตรวจสอบจะระบุว่า:

```text
No VirusTotal findings
```

VirusTotal ยังคงเป็น telemetry ไม่ได้แทนที่การวิเคราะห์ความเสี่ยงของ ClawHub เองที่เข้าใจอาร์ติแฟกต์

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัยของ ClawHub เอง ระบบจะรีวิวรุ่นเผยแพร่แต่ละรายการในฐานะอาร์ติแฟกต์สำหรับเอเจนต์: คำสั่ง metadata สิทธิ์ที่ประกาศไว้ ไฟล์ สัญญาณความสามารถ สัญญาณการสแกนแบบ static findings จาก SkillSpector telemetry จาก VirusTotal และบริบทที่ผู้เผยแพร่ให้ไว้ สัญญาณการสแกนแบบ static เป็นบริบทภายในสำหรับการรีวิวนี้ ไม่ใช่ส่วนการตรวจสอบสาธารณะแบบ standalone หรือคำตัดสินที่บล็อกการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นกรอบมองความเสี่ยง เช่น prompt injection, tool misuse, credential exposure, unsafe execution, memory หรือ context poisoning และ excessive agency

ClawScan ไม่ถือว่าความสามารถที่ดูน่ากลัวเป็นอันตรายโดยอัตโนมัติ แต่จะถามว่าความสามารถนั้นถูกเปิดเผย สอดคล้องกับวัตถุประสงค์ และได้รับการสนับสนุนจากกรณีใช้งานที่รุ่นเผยแพร่ระบุไว้หรือไม่
