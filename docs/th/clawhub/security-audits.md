# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจได้ว่าสกิลหรือ Plugin ปลอดภัยพอที่จะติดตั้งหรือไม่ โดยจะแสดงว่ารีลีสทำอะไร ขอสิทธิ์อะไรบ้าง และมีสิ่งใดที่ควรให้ความสนใจเป็นพิเศษก่อนจะเข้าถึงไฟล์ บัญชี ข้อมูลประจำตัว โค้ด หรือบริการภายนอกหรือไม่.

การตรวจสอบเป็นสัญญาณด้านความปลอดภัยที่หนักแน่น แต่ไม่ได้รับประกันว่ารีลีสจะปราศจากความเสี่ยง ควรใช้วิจารณญาณเสมอก่อนให้สิทธิ์เข้าถึงข้อมูลที่ละเอียดอ่อน.

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security), [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage),
และ [การกลั่นกรองและความปลอดภัยของบัญชี](/clawhub/moderation).

## สิ่งที่ควรตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจสอบ:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- findings ที่ระบุไว้
- ข้อมูลประจำตัว สิทธิ์ หรือ environment variables ที่จำเป็น
- เจ้าของ แหล่งที่มา เวอร์ชัน changelog จำนวนดาวน์โหลด stars และสัญญาณความน่าเชื่อถืออื่นๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น.

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกว่าคุณควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้ซึ่งสูงกว่าความเสี่ยงต่ำ.                                |
| `Review`    | อ่าน findings ก่อนติดตั้ง รีลีสอาจยังคงถูกต้องตามวัตถุประสงค์. |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบข้อกังวลที่มีผลกระทบสูงหรือสัญญาณเตือน. |
| `Malicious` | อย่าติดตั้ง.                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จสิ้น.                                             |
| `Error`     | ไม่สามารถทำการตรวจสอบให้เสร็จได้.                                         |

`Pass` ช่วยให้อุ่นใจ แต่ไม่ทดแทนวิจารณญาณของคุณเอง สิ่งนี้สำคัญที่สุดสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล รันคำสั่ง อ่านไฟล์ หรือเข้าถึงระบบ production.

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบาย blast radius: รีลีสดูเหมือนจะมีอำนาจมากเพียงใดหากคุณใช้งานตามที่ตั้งใจไว้.

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบอำนาจที่ละเอียดอ่อนหรือผลกระทบต่อผู้ใช้น้อยมาก.                          |
| `Medium`   | รีลีสมีอำนาจที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล. |
| `High`     | รีลีสมีอำนาจที่มีผลกระทบสูง findings ที่รุนแรง หรือสัญญาณที่เป็นอันตราย. |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคำถามคนละข้อ:

- ระดับความเสี่ยงถามว่า: "ที่นี่มีอำนาจมากแค่ไหน?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอะไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น สกิลสำหรับการเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยง `Medium` นั่นไม่ได้หมายความว่ามันเป็นอันตราย แต่หมายความว่าสกิลดูสอดคล้องกับวัตถุประสงค์ แต่สามารถดำเนินการด้วยอำนาจบัญชีที่มีนัยสำคัญได้.

## Findings

Findings อธิบายว่าทำไมจึงแสดงผลการตรวจสอบนั้น finding แต่ละรายการมักประกอบด้วย:

- ความหมายของมัน
- เหตุผลที่ถูก flag
- เนื้อหาสกิลหรือ Plugin ที่เกี่ยวข้อง
- คำแนะนำ

Findings อาจมีป้ายกำกับเป็น `Info`, `Low`, `Medium`, `High`, หรือ `Critical` findings ที่มีความรุนแรงสูงกว่าจะส่งผลต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า.

Findings ที่มีความเชื่อมั่นต่ำจะถูกซ่อนจาก rollup การตรวจสอบสาธารณะ เพื่อให้หน้ายังคงโฟกัสกับหลักฐานที่มีประโยชน์.

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบ release artifacts ที่ส่งเข้ามา รวมถึง:

- คำสั่งของสกิลหรือ metadata ของ Plugin
- environment variables และสิทธิ์ที่ประกาศไว้
- คำแนะนำการติดตั้งและ package metadata
- ไฟล์ที่รวมอยู่และ file manifests
- metadata ด้านความเข้ากันได้และ capability

คำถามหลักคือความสอดคล้อง: ชื่อ สรุป metadata อำนาจที่ร้องขอ และเนื้อหาจริงสอดคล้องกับสิ่งที่ผู้ใช้คาดหวังได้อย่างสมเหตุสมผลหรือไม่?

พฤติกรรมที่ทรงพลังไม่ได้แย่โดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลประจำตัว คำสั่งภายในเครื่อง provider APIs หรือการติดตั้ง package การตรวจสอบจะดูว่าอำนาจนั้นเป็นสิ่งที่คาดหวัง เปิดเผยไว้ และได้สัดส่วนหรือไม่.

หน้า artifact จะลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบรวม:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็น malware telemetry ใน audit stack VirusTotal เป็นมาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับชื่อเสียงของไฟล์และการสแกนมัลแวร์ และความร่วมมือของเราช่วยให้ ClawHub เพิ่ม intelligence ด้านความปลอดภัยที่กว้างขึ้นในการรีวิวสกิลและ Plugin.

VirusTotal มีประโยชน์เป็นพิเศษสำหรับ artifacts ที่ทราบว่าเป็นอันตราย engine hits และสัญญาณชื่อเสียงที่เติมเต็มการรีวิวแบบ agent-aware ของ ClawHub เมื่อมีจำนวน vendor engine การตรวจสอบจะสรุปเป็นภาษาธรรมดา เช่น:

```text
62/62 vendors flagged this skill as clean.
```

หรือ:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

เมื่อ ClawHub ไม่มี vendor-count telemetry ให้สรุป การตรวจสอบจะระบุว่า:

```text
No VirusTotal findings
```

VirusTotal ยังคงเป็น telemetry มันไม่ได้แทนที่การวิเคราะห์ความเสี่ยงแบบ artifact-aware ของ ClawHub เอง.

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัยของ ClawHub เอง ระบบจะรีวิวแต่ละรีลีสในฐานะ artifact ที่หันเข้าหา agent: คำสั่ง metadata สิทธิ์ที่ประกาศ ไฟล์ สัญญาณ capability สัญญาณ static scan, findings ของ SkillSpector, telemetry ของ VirusTotal และบริบทที่ publisher ให้มา สัญญาณ static scan เป็นบริบทภายในสำหรับการรีวิวนี้ ไม่ใช่ส่วนการตรวจสอบสาธารณะแบบ standalone หรือคำตัดสินที่บล็อกการติดตั้ง.

การวิเคราะห์ความเสี่ยงใช้
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นกรอบมองความเสี่ยง เช่น prompt injection, tool misuse, credential exposure, unsafe execution, memory หรือ context poisoning และ excessive agency.

ClawScan ไม่ถือว่า capability ที่ดูน่ากลัวเป็นสิ่งอันตรายโดยอัตโนมัติ ระบบจะถามว่า capability นั้นถูกเปิดเผย สอดคล้องกับวัตถุประสงค์ และได้รับการสนับสนุนจาก use case ที่รีลีสระบุไว้หรือไม่.
