# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจว่า skill หรือ Plugin ปลอดภัยพอ
สำหรับการติดตั้งหรือไม่ การตรวจสอบจะแสดงว่า release ทำอะไร ขออำนาจใดบ้าง และ
มีสิ่งใดที่ควรได้รับความสนใจเพิ่มเติมก่อนที่มันจะเข้าถึงไฟล์ บัญชี
ข้อมูลประจำตัว โค้ด หรือบริการภายนอกหรือไม่

การตรวจสอบเป็นสัญญาณความปลอดภัยที่หนักแน่น แต่ไม่ได้รับประกันว่า release จะ
ปราศจากความเสี่ยง ใช้วิจารณญาณเสมอก่อนให้สิทธิ์เข้าถึงข้อมูลอ่อนไหว

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security), [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage),
และ [การดูแลเนื้อหาและความปลอดภัยของบัญชี](/clawhub/moderation)

## สิ่งที่ต้องตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจสอบ:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- findings ที่ระบุไว้
- ข้อมูลประจำตัว สิทธิ์ หรือ environment variables ที่จำเป็น
- เจ้าของ แหล่งที่มา เวอร์ชัน changelog ยอดดาวน์โหลด stars และสัญญาณความน่าเชื่อถืออื่นๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและไว้วางใจเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกคุณว่าควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้ซึ่งสูงกว่าความเสี่ยงต่ำ                                |
| `Review`    | อ่าน findings ก่อนติดตั้ง release อาจยังเป็นของถูกต้องตามวัตถุประสงค์ |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบข้อกังวลที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | อย่าติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จ                                             |
| `Error`     | ไม่สามารถดำเนินการตรวจสอบให้เสร็จได้                                         |

`Pass` ช่วยให้มั่นใจได้ แต่ไม่ได้แทนที่วิจารณญาณของคุณเอง เรื่องนี้สำคัญ
ที่สุดสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล รันคำสั่ง อ่านไฟล์ หรือ
เข้าถึงระบบ production ได้

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบาย blast radius: release ดูเหมือนมีอำนาจมากเพียงใดหาก
คุณใช้งานตามที่ตั้งใจไว้

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบอำนาจที่อ่อนไหวหรือผลกระทบต่อผู้ใช้น้อยมาก                          |
| `Medium`   | release มีอำนาจที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | release มีอำนาจที่มีผลกระทบสูง findings ร้ายแรง หรือสัญญาณประสงค์ร้าย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคำถามคนละข้อ:

- ระดับความเสี่ยงถามว่า: "มีอำนาจมากแค่ไหนอยู่ตรงนี้?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอย่างไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น publishing skill อาจแสดง `Review` พร้อมความเสี่ยง `Medium` นั่น
ไม่ได้หมายความว่ามันประสงค์ร้าย แต่หมายความว่า skill ดูสอดคล้องกับวัตถุประสงค์
แต่สามารถกระทำการด้วยอำนาจบัญชีที่มีนัยสำคัญได้

## Findings

Findings อธิบายว่าทำไมจึงแสดงผลการตรวจสอบนั้น แต่ละ finding มักประกอบด้วย:

- ความหมายของมัน
- เหตุผลที่ถูกระบุ
- เนื้อหา skill หรือ Plugin ที่เกี่ยวข้อง
- คำแนะนำ

Findings อาจติดป้าย `Info`, `Low`, `Medium`, `High` หรือ `Critical` findings
ที่มีความรุนแรงสูงกว่าจะส่งผลต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

Findings ที่มีความเชื่อมั่นต่ำจะถูกซ่อนจากสรุปการตรวจสอบสาธารณะ เพื่อให้หน้า
ยังคงมุ่งเน้นหลักฐานที่มีประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบ artifacts ของ release ที่ส่งเข้ามา รวมถึง:

- คำสั่งของ skill หรือ metadata ของ Plugin
- environment variables และสิทธิ์ที่ประกาศไว้
- คำสั่งติดตั้งและ metadata ของ package
- ไฟล์ที่รวมมาและ file manifests
- metadata ด้านความเข้ากันได้และความสามารถ

คำถามหลักคือความสอดคล้อง: ชื่อ สรุป metadata อำนาจที่ร้องขอ
และเนื้อหาจริงสอดคล้องกับสิ่งที่ผู้ใช้น่าจะคาดหวังอย่างสมเหตุสมผลหรือไม่?

พฤติกรรมที่ทรงพลังไม่ได้แย่โดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลประจำตัว
คำสั่งภายในเครื่อง provider APIs หรือการติดตั้ง package การตรวจสอบจะตรวจว่า
อำนาจนั้นเป็นสิ่งที่คาดหวังไว้ เปิดเผยแล้ว และได้สัดส่วนหรือไม่

หน้า artifact ลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบรวม:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็น telemetry ด้านมัลแวร์ในชุดการตรวจสอบ VirusTotal เป็น
มาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับชื่อเสียงของไฟล์และการสแกนมัลแวร์ และ
ความร่วมมือของเราช่วยให้ ClawHub เพิ่มข้อมูลความปลอดภัยที่กว้างขึ้นในการตรวจสอบ skill และ Plugin

VirusTotal มีประโยชน์เป็นพิเศษสำหรับ artifacts ที่ทราบว่าเป็นอันตราย engine hits และ
สัญญาณชื่อเสียงที่ช่วยเสริมการตรวจสอบแบบเข้าใจ agent ของ ClawHub เมื่อมีจำนวน
vendor engine ให้ใช้งาน การตรวจสอบจะสรุปเป็นภาษาง่ายๆ เช่น:

```text
62/62 vendors flagged this skill as clean.
```

หรือ:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

เมื่อ ClawHub ไม่มี telemetry แบบจำนวน vendor ให้สรุป การตรวจสอบจะระบุว่า:

```text
No VirusTotal findings
```

VirusTotal ยังคงเป็น telemetry และไม่ได้แทนที่การวิเคราะห์ความเสี่ยงที่เข้าใจ
artifact ของ ClawHub เอง

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัยของ
ClawHub เอง ระบบจะตรวจสอบแต่ละ release ในฐานะ artifact ที่ agent-facing: คำสั่ง,
metadata, สิทธิ์ที่ประกาศไว้, ไฟล์, สัญญาณความสามารถ, สัญญาณ static scan,
findings ของ SkillSpector, telemetry ของ VirusTotal และบริบทที่ publisher ให้มา
สัญญาณ static scan เป็นบริบทภายในสำหรับการตรวจสอบนี้ ไม่ใช่ส่วนการตรวจสอบสาธารณะ
แบบ standalone หรือ verdict ที่บล็อกการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นกรอบมองความเสี่ยง เช่น prompt injection, การใช้เครื่องมือผิดวัตถุประสงค์, การเปิดเผยข้อมูลประจำตัว,
การดำเนินการที่ไม่ปลอดภัย, การทำให้ memory หรือบริบทปนเปื้อน และ excessive agency

ClawScan ไม่ถือว่าความสามารถที่ดูน่ากลัวเป็นประสงค์ร้ายโดยอัตโนมัติ
ระบบจะถามว่าความสามารถนั้นถูกเปิดเผย สอดคล้องกับวัตถุประสงค์ และได้รับการรองรับโดย
กรณีการใช้งานที่ release ระบุไว้หรือไม่
