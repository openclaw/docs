# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจว่า Skills หรือ Plugin ปลอดภัยพอ
ที่จะติดตั้งหรือไม่ โดยจะแสดงว่ารุ่นเผยแพร่ทำอะไร ขอสิทธิ์อำนาจใดบ้าง และ
มีสิ่งใดที่ควรใส่ใจเป็นพิเศษก่อนที่มันจะเข้าถึงไฟล์ บัญชี
ข้อมูลลับ โค้ด หรือบริการภายนอกได้หรือไม่

การตรวจสอบเป็นสัญญาณความปลอดภัยที่แข็งแรง แต่ไม่ได้รับประกันว่ารุ่นเผยแพร่จะ
ไม่มีความเสี่ยงเลย ใช้วิจารณญาณเสมอก่อนให้สิทธิ์เข้าถึงข้อมูลที่ละเอียดอ่อน

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security), [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage),
และ [การกลั่นกรองและความปลอดภัยของบัญชี](/clawhub/moderation)

## สิ่งที่ควรตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจทาน:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- ผลการตรวจพบที่ระบุไว้
- ข้อมูลลับ สิทธิ์อนุญาต หรือตัวแปรสภาพแวดล้อมที่จำเป็น
- เจ้าของ แหล่งที่มา เวอร์ชัน บันทึกการเปลี่ยนแปลง ยอดดาวน์โหลด ดาว และสัญญาณความน่าเชื่อถืออื่นๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและไว้วางใจเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกคุณว่าควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้ซึ่งมีความเสี่ยงสูงกว่าระดับต่ำ                                |
| `Review`    | อ่านผลการตรวจพบก่อนติดตั้ง รุ่นเผยแพร่อาจยังคงถูกต้องตามเจตนา |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบข้อกังวลที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | อย่าติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จสิ้น                                             |
| `Error`     | ไม่สามารถดำเนินการตรวจสอบให้เสร็จสมบูรณ์ได้                                         |

`Pass` ช่วยให้มั่นใจได้ แต่ไม่ได้แทนที่วิจารณญาณของคุณเอง เรื่องนี้สำคัญ
ที่สุดสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล รันคำสั่ง อ่านไฟล์ หรือ
เข้าถึงระบบโปรดักชันได้

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบายขอบเขตผลกระทบ: รุ่นเผยแพร่ดูเหมือนมีอำนาจมากแค่ไหนหาก
คุณใช้งานตามเจตนา

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบสิทธิ์อำนาจที่ละเอียดอ่อนหรือผลกระทบต่อผู้ใช้น้อย                          |
| `Medium`   | รุ่นเผยแพร่มีสิทธิ์อำนาจที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | รุ่นเผยแพร่มีสิทธิ์อำนาจที่มีผลกระทบสูง ผลการตรวจพบที่รุนแรง หรือสัญญาณมุ่งร้าย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคำถามคนละข้อ:

- ระดับความเสี่ยงถามว่า: "ที่นี่มีอำนาจมากแค่ไหน?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอย่างไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น Skills สำหรับเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยง `Medium` ซึ่ง
ไม่ได้หมายความว่ามันมุ่งร้าย แต่หมายความว่า Skills นั้นดูสอดคล้องกับวัตถุประสงค์ แต่สามารถ
ดำเนินการด้วยสิทธิ์อำนาจของบัญชีที่มีนัยสำคัญได้

## ผลการตรวจพบ

ผลการตรวจพบอธิบายว่าทำไมจึงแสดงผลการตรวจสอบนั้น โดยแต่ละรายการมักมี:

- ความหมายของรายการนั้น
- เหตุผลที่ถูกตั้งค่าสถานะ
- เนื้อหา Skills หรือ Plugin ที่เกี่ยวข้อง
- คำแนะนำ

ผลการตรวจพบอาจมีป้ายกำกับ `Info`, `Low`, `Medium`, `High` หรือ `Critical` ความรุนแรง
ที่สูงกว่าจะมีผลต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

ผลการตรวจพบที่มีความเชื่อมั่นต่ำจะถูกซ่อนจากสรุปการตรวจสอบสาธารณะ เพื่อให้หน้า
มุ่งเน้นหลักฐานที่เป็นประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบอาร์ทิแฟกต์รุ่นเผยแพร่ที่ส่งเข้ามา รวมถึง:

- คำสั่งของ Skills หรือข้อมูลเมตาของ Plugin
- ตัวแปรสภาพแวดล้อมและสิทธิ์อนุญาตที่ประกาศไว้
- คำแนะนำการติดตั้งและข้อมูลเมตาของแพ็กเกจ
- ไฟล์ที่รวมมาและรายการไฟล์
- ข้อมูลเมตาด้านความเข้ากันได้และความสามารถ

คำถามหลักคือความสอดคล้อง: ชื่อ สรุป ข้อมูลเมตา สิทธิ์อำนาจที่ร้องขอ
และเนื้อหาจริงสอดคล้องกับสิ่งที่ผู้ใช้คาดหวังได้อย่างสมเหตุสมผลหรือไม่?

พฤติกรรมที่มีอำนาจสูงไม่ได้แย่โดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลลับ
คำสั่งภายในเครื่อง API ของผู้ให้บริการ หรือการติดตั้งแพ็กเกจ การตรวจสอบจะตรวจว่าอำนาจนั้น
เป็นสิ่งที่คาดหมายได้ มีการเปิดเผย และได้สัดส่วนหรือไม่

หน้าอาร์ทิแฟกต์ลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบรวม:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็นข้อมูลโทรมาตรมัลแวร์ในชุดการตรวจสอบ VirusTotal เป็น
มาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับชื่อเสียงของไฟล์และการสแกนมัลแวร์ และ
ความร่วมมือของเราช่วยให้ ClawHub เพิ่มข่าวกรองความปลอดภัยที่กว้างขึ้นให้กับการตรวจทาน
Skills และ Plugin

VirusTotal มีประโยชน์เป็นพิเศษสำหรับอาร์ทิแฟกต์มุ่งร้ายที่รู้จักแล้ว ผลตรวจจากเอนจิน และ
สัญญาณชื่อเสียงที่เสริมการตรวจทานของ ClawHub ซึ่งรับรู้บริบทของเอเจนต์ เมื่อมี
จำนวนผลจากเอนจินผู้ให้บริการ การตรวจสอบจะสรุปเป็นภาษาธรรมดา เช่น:

```text
62/62 vendors flagged this skill as clean.
```

หรือ:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

เมื่อ ClawHub ไม่มีข้อมูลโทรมาตรจำนวนผู้ให้บริการให้สรุป การตรวจสอบจะระบุว่า:

```text
No VirusTotal findings
```

VirusTotal ยังคงเป็นข้อมูลโทรมาตร ไม่ได้แทนที่การวิเคราะห์ความเสี่ยงของ ClawHub เอง
ที่รับรู้บริบทของอาร์ทิแฟกต์

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัยของ
ClawHub เอง ระบบจะตรวจทานแต่ละรุ่นเผยแพร่ในฐานะอาร์ทิแฟกต์ที่หันหน้าเข้าหาเอเจนต์: คำสั่ง,
ข้อมูลเมตา, สิทธิ์อนุญาตที่ประกาศไว้, ไฟล์, สัญญาณความสามารถ, สัญญาณการสแกนแบบสแตติก,
ผลการตรวจพบจาก SkillSpector, ข้อมูลโทรมาตรจาก VirusTotal และบริบทที่ผู้เผยแพร่ให้มา
สัญญาณการสแกนแบบสแตติกเป็นบริบทภายในสำหรับการตรวจทานนี้ ไม่ใช่
ส่วนการตรวจสอบสาธารณะแบบสแตนด์อโลนหรือคำตัดสินที่บล็อกการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นกรอบพิจารณาความเสี่ยง เช่น การฉีดพรอมต์ การใช้เครื่องมือในทางที่ผิด การเปิดเผยข้อมูลลับ
การดำเนินการที่ไม่ปลอดภัย การวางยาพิษหน่วยความจำหรือบริบท และอำนาจการดำเนินการที่มากเกินไป

ClawScan ไม่ถือว่าความสามารถที่ดูน่ากลัวเป็นสิ่งมุ่งร้ายโดยอัตโนมัติ
ระบบจะถามว่าความสามารถนั้นมีการเปิดเผย สอดคล้องกับวัตถุประสงค์ และได้รับการสนับสนุนจาก
กรณีการใช้งานที่รุ่นเผยแพร่ระบุไว้หรือไม่
