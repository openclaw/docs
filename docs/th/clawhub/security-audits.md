# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจได้ว่า skill หรือ plugin ปลอดภัยพอ
สำหรับติดตั้งหรือไม่ การตรวจสอบจะแสดงว่ารีลีสทำอะไร ขอสิทธิ์อะไรบ้าง และ
มีสิ่งใดที่ควรให้ความสนใจเป็นพิเศษก่อนจะเข้าถึงไฟล์ บัญชี
ข้อมูลรับรอง โค้ด หรือบริการภายนอกหรือไม่

การตรวจสอบเป็นสัญญาณความปลอดภัยที่สำคัญ แต่ไม่ได้รับประกันว่ารีลีสจะ
ปราศจากความเสี่ยง ใช้วิจารณญาณเสมอก่อนให้สิทธิ์เข้าถึงที่ละเอียดอ่อน

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security), [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage),
และ [การดูแลเนื้อหาและความปลอดภัยของบัญชี](/clawhub/moderation)

## สิ่งที่ควรตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจสอบ:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- findings ที่ระบุไว้
- ข้อมูลรับรอง สิทธิ์ หรือ environment variables ที่จำเป็น
- เจ้าของ แหล่งที่มา เวอร์ชัน changelog ยอดดาวน์โหลด ดาว และสัญญาณความน่าเชื่อถืออื่นๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกคุณว่าควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้ซึ่งสูงกว่าความเสี่ยงต่ำ                                |
| `Review`    | อ่าน findings ก่อนติดตั้ง รีลีสอาจยังเป็นรีลีสที่ถูกต้องได้ |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบประเด็นที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | อย่าติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จสิ้น                                             |
| `Error`     | ไม่สามารถดำเนินการตรวจสอบให้เสร็จสมบูรณ์ได้                                         |

`Pass` ช่วยให้มั่นใจขึ้น แต่ไม่สามารถแทนที่วิจารณญาณของคุณเองได้ เรื่องนี้สำคัญ
ที่สุดสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล รันคำสั่ง อ่านไฟล์ หรือ
เข้าถึงระบบ production ได้

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบายขอบเขตผลกระทบ: รีลีสดูเหมือนจะมีอำนาจมากเพียงใดหาก
คุณใช้งานตามวัตถุประสงค์

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบอำนาจที่ละเอียดอ่อนหรือผลกระทบต่อผู้ใช้น้อยมาก                          |
| `Medium`   | รีลีสมีอำนาจที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | รีลีสมีอำนาจที่มีผลกระทบสูง findings ร้ายแรง หรือสัญญาณประสงค์ร้าย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคำถามต่างกัน:

- ระดับความเสี่ยงถามว่า: "มีอำนาจมากแค่ไหนอยู่ตรงนี้?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอะไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น skill สำหรับเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยง `Medium` นั่น
ไม่ได้หมายความว่ามันประสงค์ร้าย แต่หมายความว่า skill ดูสอดคล้องกับวัตถุประสงค์ แต่สามารถ
ดำเนินการด้วยอำนาจบัญชีที่มีนัยสำคัญได้

## Findings

Findings อธิบายว่าเหตุใดจึงแสดงผลการตรวจสอบนั้น โดยแต่ละ finding มักประกอบด้วย:

- หมายความว่าอะไร
- เหตุใดจึงถูกแจ้งเตือน
- เนื้อหา skill หรือ plugin ที่เกี่ยวข้อง
- คำแนะนำ

Findings อาจติดป้ายกำกับเป็น `Info`, `Low`, `Medium`, `High` หรือ `Critical` findings ที่มี
ความรุนแรงสูงกว่าจะมีส่วนต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

Findings ที่มีความเชื่อมั่นต่ำจะถูกซ่อนจากสรุปการตรวจสอบสาธารณะ เพื่อให้หน้า
มุ่งเน้นหลักฐานที่เป็นประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบอาร์ติแฟกต์รีลีสที่ส่งเข้ามา รวมถึง:

- คำแนะนำของ skill หรือ metadata ของ plugin
- environment variables และสิทธิ์ที่ประกาศไว้
- คำแนะนำการติดตั้งและ package metadata
- ไฟล์ที่รวมมาและ file manifests
- metadata ด้านความเข้ากันได้และความสามารถ

คำถามหลักคือความสอดคล้อง: ชื่อ สรุป metadata อำนาจที่ขอ
และเนื้อหาจริงสอดคล้องกับสิ่งที่ผู้ใช้คาดหวังอย่างสมเหตุสมผลหรือไม่?

พฤติกรรมที่ทรงพลังไม่ได้แย่โดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลรับรอง
คำสั่ง local, provider APIs หรือการติดตั้ง package การตรวจสอบจะดูว่าอำนาจนั้น
เป็นสิ่งที่คาดหมายได้ มีการเปิดเผย และได้สัดส่วนหรือไม่

หน้าอาร์ติแฟกต์ลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบรวม:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็น telemetry มัลแวร์ในชุดการตรวจสอบ VirusTotal เป็น
มาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับชื่อเสียงของไฟล์และการสแกนมัลแวร์ และ
ความร่วมมือของเราช่วยให้ ClawHub เพิ่มข้อมูลข่าวกรองด้านความปลอดภัยที่กว้างขึ้นให้กับการรีวิว
skill และ plugin

VirusTotal มีประโยชน์เป็นพิเศษสำหรับอาร์ติแฟกต์ประสงค์ร้ายที่รู้จัก engine hits และ
สัญญาณชื่อเสียงที่เสริมการรีวิวที่เข้าใจ agent ของ ClawHub เมื่อจำนวน vendor
engine พร้อมใช้งาน การตรวจสอบจะสรุปเป็นภาษาธรรมดา เช่น:

```text
62/62 vendors flagged this skill as clean.
```

หรือ:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

เมื่อ ClawHub ไม่มี telemetry จำนวน vendor ให้สรุป การตรวจสอบจะระบุว่า:

```text
No VirusTotal findings
```

VirusTotal ยังคงเป็น telemetry ไม่ได้แทนที่การวิเคราะห์ความเสี่ยงของ ClawHub เอง
ที่เข้าใจอาร์ติแฟกต์

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัยของ ClawHub
เอง ระบบจะรีวิวแต่ละรีลีสในฐานะอาร์ติแฟกต์ที่ agent ต้องเผชิญ: คำแนะนำ
metadata สิทธิ์ที่ประกาศ ไฟล์ สัญญาณความสามารถ สัญญาณสแกนแบบ static
findings จาก SkillSpector, telemetry จาก VirusTotal และบริบทที่ผู้เผยแพร่ให้มา
สัญญาณสแกนแบบ static เป็นบริบทภายในสำหรับการรีวิวนี้ ไม่ใช่
ส่วนการตรวจสอบสาธารณะแบบ standalone หรือคำตัดสินที่บล็อกการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นเลนส์สำหรับความเสี่ยง เช่น prompt injection การใช้เครื่องมือผิดวัตถุประสงค์ การเปิดเผยข้อมูลรับรอง
การดำเนินการที่ไม่ปลอดภัย การปนเปื้อน memory หรือ context และ agency ที่มากเกินไป

ClawScan ไม่ถือว่าความสามารถที่ดูน่ากลัวเป็นสิ่งประสงค์ร้ายโดยอัตโนมัติ
ระบบจะถามว่าความสามารถนั้นถูกเปิดเผย สอดคล้องกับวัตถุประสงค์ และได้รับการสนับสนุนโดย
กรณีใช้งานที่รีลีสระบุไว้หรือไม่
