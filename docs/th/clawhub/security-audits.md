# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจว่าสกิลหรือ Plugin ปลอดภัยพอ
ที่จะติดตั้งหรือไม่ การตรวจสอบจะแสดงว่ารีลีสทำอะไร ขออำนาจใดบ้าง และ
มีสิ่งใดที่ควรให้ความสนใจเป็นพิเศษก่อนที่จะเข้าถึงไฟล์ บัญชี
ข้อมูลลับ โค้ด หรือบริการภายนอกหรือไม่

การตรวจสอบเป็นสัญญาณด้านความปลอดภัยที่หนักแน่น แต่ไม่ได้รับประกันว่ารีลีสจะ
ปราศจากความเสี่ยง ใช้วิจารณญาณเสมอก่อนให้สิทธิ์เข้าถึงข้อมูลหรือระบบที่ละเอียดอ่อน

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security), [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage),
และ [การดูแลเนื้อหาและความปลอดภัยของบัญชี](/clawhub/moderation)

## สิ่งที่ควรตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจสอบ:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- ข้อค้นพบที่แสดงไว้
- ข้อมูลลับ สิทธิ์ หรือ environment variables ที่ต้องใช้
- เจ้าของ แหล่งที่มา เวอร์ชัน บันทึกการเปลี่ยนแปลง ยอดดาวน์โหลด ดาว และสัญญาณความน่าเชื่อถืออื่นๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและไว้วางใจเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกว่าคุณควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้ซึ่งสูงกว่าความเสี่ยงต่ำ                                |
| `Review`    | อ่านข้อค้นพบก่อนติดตั้ง รีลีสอาจยังคงถูกต้องชอบธรรม |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบประเด็นที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | อย่าติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จ                                             |
| `Error`     | ไม่สามารถดำเนินการตรวจสอบให้เสร็จสมบูรณ์ได้                                         |

`Pass` ช่วยให้มั่นใจขึ้น แต่ไม่ได้แทนที่วิจารณญาณของคุณเอง สิ่งนี้สำคัญ
มากที่สุดสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล เรียกใช้คำสั่ง อ่านไฟล์ หรือ
เข้าถึงระบบโปรดักชันได้

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบายขอบเขตผลกระทบ: รีลีสดูเหมือนมีอำนาจมากเพียงใดหาก
คุณใช้งานตามเจตนาที่ออกแบบไว้

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบอำนาจที่ละเอียดอ่อนหรือผลกระทบต่อผู้ใช้น้อยมาก                          |
| `Medium`   | รีลีสมีอำนาจที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | รีลีสมีอำนาจที่มีผลกระทบสูง ข้อค้นพบรุนแรง หรือสัญญาณที่เป็นอันตราย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคนละคำถามกัน:

- ระดับความเสี่ยงถามว่า: "ที่นี่มีอำนาจมากเพียงใด?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอย่างไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น สกิลสำหรับการเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยง `Medium` นั่น
ไม่ได้หมายความว่าสกิลนั้นเป็นอันตราย แต่หมายความว่าสกิลดูสอดคล้องกับวัตถุประสงค์ แต่สามารถ
ดำเนินการด้วยอำนาจของบัญชีที่มีนัยสำคัญได้

## ข้อค้นพบ

ข้อค้นพบอธิบายว่าทำไมจึงแสดงผลการตรวจสอบนั้น ข้อค้นพบแต่ละรายการมักประกอบด้วย:

- หมายความว่าอย่างไร
- เหตุใดจึงถูกตั้งค่าสถานะ
- เนื้อหาสกิลหรือ Plugin ที่เกี่ยวข้อง
- คำแนะนำ

ข้อค้นพบอาจมีป้ายกำกับ `Info`, `Low`, `Medium`, `High` หรือ `Critical` ข้อค้นพบที่มี
ความรุนแรงสูงกว่าจะมีผลต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

ข้อค้นพบที่มีความเชื่อมั่นต่ำจะถูกซ่อนจากสรุปการตรวจสอบสาธารณะเพื่อให้หน้า
มุ่งเน้นหลักฐานที่เป็นประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบอาร์ติแฟกต์ของรีลีสที่ส่งเข้ามา ซึ่งรวมถึง:

- คำสั่งของสกิลหรือเมทาดาทาของ Plugin
- environment variables และสิทธิ์ที่ประกาศไว้
- คำแนะนำการติดตั้งและเมทาดาทาแพ็กเกจ
- ไฟล์ที่รวมมาและรายการไฟล์
- เมทาดาทาความเข้ากันได้และความสามารถ

คำถามหลักคือความสอดคล้อง: ชื่อ สรุป เมทาดาทา อำนาจที่ขอ
และเนื้อหาจริงสอดคล้องกับสิ่งที่ผู้ใช้คาดหวังได้อย่างสมเหตุสมผลหรือไม่?

พฤติกรรมที่มีอำนาจสูงไม่ได้แย่โดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลลับ
คำสั่งในเครื่อง API ของผู้ให้บริการ หรือการติดตั้งแพ็กเกจ การตรวจสอบจะดูว่าอำนาจนั้น
เป็นสิ่งที่คาดหมายได้ เปิดเผยไว้ และได้สัดส่วนหรือไม่

หน้าอาร์ติแฟกต์ลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบรวม:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็นข้อมูลโทรมาตรมัลแวร์ในชุดการตรวจสอบ VirusTotal เป็น
มาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับชื่อเสียงของไฟล์และการสแกนมัลแวร์ และ
ความร่วมมือของเราช่วยให้ ClawHub เพิ่มข่าวกรองด้านความปลอดภัยที่กว้างขึ้นให้กับการตรวจสอบสกิลและ Plugin

VirusTotal มีประโยชน์เป็นพิเศษสำหรับอาร์ติแฟกต์อันตรายที่รู้จัก การพบโดยเอนจิน และ
สัญญาณชื่อเสียงที่เสริมการตรวจสอบของ ClawHub ซึ่งเข้าใจบริบทของเอเจนต์ เมื่อมี
จำนวนจากเอนจินของผู้ให้บริการ การตรวจสอบจะสรุปเป็นภาษาธรรมดา เช่น:

```text
62/62 vendors flagged this skill as clean.
```

หรือ:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

เมื่อ ClawHub ไม่มีข้อมูลโทรมาตรจำนวนผู้ให้บริการให้สรุป การตรวจสอบจะระบุว่า:

```text
No VirusTotal findings
```

VirusTotal ยังคงเป็นข้อมูลโทรมาตร ไม่ได้แทนที่การวิเคราะห์ความเสี่ยงของ ClawHub เอง
ที่เข้าใจอาร์ติแฟกต์

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัยของ
ClawHub เอง ระบบนี้ตรวจสอบแต่ละรีลีสในฐานะอาร์ติแฟกต์ที่เอเจนต์ต้องพบ: คำสั่ง
เมทาดาทา สิทธิ์ที่ประกาศ ไฟล์ สัญญาณความสามารถ สัญญาณการสแกนแบบสถิต
ข้อค้นพบของ SkillSpector ข้อมูลโทรมาตรของ VirusTotal และบริบทที่ผู้เผยแพร่ให้มา
สัญญาณการสแกนแบบสถิตเป็นบริบทภายในสำหรับการตรวจสอบนี้ ไม่ใช่
ส่วนการตรวจสอบสาธารณะแบบแยกเดี่ยวหรือคำตัดสินที่บล็อกการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นกรอบมองความเสี่ยง เช่น prompt injection, การใช้เครื่องมือผิดวัตถุประสงค์, การเปิดเผยข้อมูลลับ,
การดำเนินการที่ไม่ปลอดภัย, การปนเปื้อนหน่วยความจำหรือบริบท และอำนาจการกระทำที่มากเกินไป

ClawScan ไม่ถือว่าความสามารถที่ดูน่ากลัวเป็นสิ่งอันตรายโดยอัตโนมัติ
ระบบจะถามว่าความสามารถนั้นถูกเปิดเผย สอดคล้องกับวัตถุประสงค์ และรองรับโดย
กรณีใช้งานที่รีลีสระบุไว้หรือไม่
