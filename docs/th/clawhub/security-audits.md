# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจได้ว่า skill หรือ Plugin ปลอดภัยพอที่จะติดตั้งหรือไม่ การตรวจสอบจะแสดงว่า release ทำอะไร ขอสิทธิ์อะไร และมีสิ่งใดที่ควรให้ความสนใจเพิ่มเติมก่อนที่จะเข้าถึงไฟล์ บัญชี ข้อมูลลับ โค้ด หรือบริการภายนอกหรือไม่

การตรวจสอบเป็นสัญญาณด้านความปลอดภัยที่แข็งแรง แต่ไม่ใช่การรับประกันว่า release จะไม่มีความเสี่ยง โปรดใช้วิจารณญาณเสมอก่อนมอบสิทธิ์เข้าถึงที่ละเอียดอ่อน

ดูเพิ่มเติมที่ [ความปลอดภัย](/th/clawhub/security), [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage) และ [การกลั่นกรองและความปลอดภัยของบัญชี](/th/clawhub/moderation)

## สิ่งที่ควรตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจสอบ:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- findings ที่ระบุไว้
- ข้อมูลลับ สิทธิ์ หรือ environment variables ที่ต้องใช้
- เจ้าของ แหล่งที่มา เวอร์ชัน บันทึกการเปลี่ยนแปลง ยอดดาวน์โหลด ดาว และสัญญาณความน่าเชื่อถืออื่น ๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกคุณว่าควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้เหนือระดับความเสี่ยงต่ำ                                |
| `Review`    | อ่าน findings ก่อนติดตั้ง release อาจยังคงถูกต้องตามเจตนา |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบข้อกังวลที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | อย่าติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จสิ้น                                             |
| `Error`     | ไม่สามารถดำเนินการตรวจสอบให้เสร็จสมบูรณ์ได้                                         |

`Pass` ช่วยให้มั่นใจขึ้น แต่ไม่แทนที่วิจารณญาณของคุณเอง สิ่งนี้สำคัญที่สุดสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล รันคำสั่ง อ่านไฟล์ หรือเข้าถึงระบบ production ได้

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบาย blast radius: release ดูเหมือนจะมีอำนาจมากเพียงใดหากคุณใช้งานตามที่ตั้งใจไว้

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบอำนาจที่ละเอียดอ่อนหรือผลกระทบต่อผู้ใช้น้อยมาก                          |
| `Medium`   | release มีอำนาจที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | release มีอำนาจที่มีผลกระทบสูง findings ที่ร้ายแรง หรือสัญญาณที่เป็นอันตราย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคำถามคนละข้อ:

- ระดับความเสี่ยงถามว่า: "มีอำนาจมากแค่ไหนอยู่ที่นี่?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอย่างไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น skill สำหรับเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยง `Medium` นั่นไม่ได้หมายความว่ามันเป็นอันตราย แต่หมายความว่า skill ดูสอดคล้องกับวัตถุประสงค์ แต่สามารถดำเนินการด้วยอำนาจบัญชีที่มีนัยสำคัญได้

## Findings

Findings อธิบายว่าเหตุใดจึงแสดงผลการตรวจสอบนั้น finding แต่ละรายการมักประกอบด้วย:

- ความหมายของมัน
- เหตุผลที่ถูก flag
- เนื้อหา skill หรือ Plugin ที่เกี่ยวข้อง
- คำแนะนำ

Findings อาจติดป้าย `Info`, `Low`, `Medium`, `High` หรือ `Critical` findings ที่มีความรุนแรงสูงกว่าจะส่งผลต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

findings ที่มีความเชื่อมั่นต่ำจะถูกซ่อนจากสรุปการตรวจสอบสาธารณะ เพื่อให้หน้ายังคงมุ่งเน้นหลักฐานที่เป็นประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบ release artifacts ที่ส่งเข้ามา รวมถึง:

- คำสั่งของ skill หรือ metadata ของ Plugin
- environment variables และสิทธิ์ที่ประกาศไว้
- คำแนะนำการติดตั้งและ package metadata
- ไฟล์และ file manifests ที่รวมอยู่
- metadata ด้านความเข้ากันได้และความสามารถ

คำถามหลักคือความสอดคล้อง: ชื่อ สรุป metadata อำนาจที่ร้องขอ และเนื้อหาจริงสอดคล้องกับสิ่งที่ผู้ใช้คาดหวังได้อย่างสมเหตุสมผลหรือไม่

พฤติกรรมที่มีอำนาจมากไม่ได้แย่โดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลลับ คำสั่ง local, provider APIs หรือการติดตั้งแพ็กเกจ การตรวจสอบจะตรวจว่าพลังนั้นเป็นสิ่งที่คาดหวังได้ มีการเปิดเผย และได้สัดส่วนหรือไม่

หน้า artifact ลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบรวม:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็น telemetry ด้านมัลแวร์ใน audit stack VirusTotal เป็นมาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับชื่อเสียงของไฟล์และการสแกนมัลแวร์ และความร่วมมือของเราช่วยให้ ClawHub เพิ่มข้อมูลข่าวกรองด้านความปลอดภัยที่กว้างขึ้นให้กับการตรวจสอบ skill และ Plugin

VirusTotal มีประโยชน์เป็นพิเศษสำหรับ artifacts ที่ทราบว่าเป็นอันตราย การพบจาก engine และสัญญาณชื่อเสียงที่เสริมการตรวจสอบของ ClawHub ซึ่งเข้าใจบริบทของ agent เมื่อมีจำนวน vendor engine ให้ใช้งาน การตรวจสอบจะสรุปเป็นภาษาธรรมดา เช่น:

```text
62/62 vendors flagged this skill as clean.
```

หรือ:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

เมื่อ ClawHub ไม่มี telemetry จำนวน vendor ให้สรุป การตรวจสอบจะระบุว่า:

```text
No VirusTotal findings
```

VirusTotal ยังคงเป็น telemetry ไม่ได้แทนที่การวิเคราะห์ความเสี่ยงของ ClawHub เองที่เข้าใจ artifact

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัยของ ClawHub เอง ระบบจะตรวจสอบแต่ละ release ในฐานะ artifact ที่มุ่งสู่ agent: คำสั่ง metadata สิทธิ์ที่ประกาศ ไฟล์ สัญญาณความสามารถ สัญญาณการสแกนแบบ static, findings ของ SkillSpector, telemetry ของ VirusTotal และบริบทที่ publisher ให้มา สัญญาณการสแกนแบบ static เป็นบริบทภายในสำหรับการตรวจสอบนี้ ไม่ใช่ส่วนการตรวจสอบสาธารณะแบบ standalone หรือ verdict ที่บล็อกการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นมุมมองสำหรับความเสี่ยง เช่น prompt injection, tool misuse, การเปิดเผยข้อมูลลับ, การดำเนินการที่ไม่ปลอดภัย, memory หรือ context poisoning และ agency ที่มากเกินไป

ClawScan ไม่ถือว่าความสามารถที่ดูน่ากลัวเป็นอันตรายโดยอัตโนมัติ แต่จะถามว่าความสามารถนั้นถูกเปิดเผย สอดคล้องกับวัตถุประสงค์ และได้รับการรองรับโดยกรณีใช้งานที่ release ระบุไว้หรือไม่
