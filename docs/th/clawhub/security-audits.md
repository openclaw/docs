# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจว่า Skills หรือ Plugin ปลอดภัยพอ
ที่จะติดตั้งหรือไม่ โดยแสดงว่ารุ่นเผยแพร่ทำอะไร ขอสิทธิ์ใดบ้าง และ
มีสิ่งใดที่ควรให้ความสนใจเป็นพิเศษก่อนที่จะเข้าถึงไฟล์ บัญชี
ข้อมูลรับรอง โค้ด หรือบริการภายนอกได้หรือไม่

การตรวจสอบเป็นสัญญาณด้านความปลอดภัยที่หนักแน่น แต่ไม่ได้รับประกันว่ารุ่นเผยแพร่จะ
ปราศจากความเสี่ยง ใช้วิจารณญาณเสมอก่อนให้สิทธิ์เข้าถึงข้อมูลสำคัญ

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security), [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage),
และ [การดูแลเนื้อหาและความปลอดภัยของบัญชี](/clawhub/moderation)

## สิ่งที่ควรตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจสอบ:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- ข้อค้นพบที่ระบุไว้
- ข้อมูลรับรอง สิทธิ์ หรือตัวแปรสภาพแวดล้อมที่จำเป็น
- เจ้าของ แหล่งที่มา เวอร์ชัน บันทึกการเปลี่ยนแปลง ยอดดาวน์โหลด ดาว และสัญญาณความน่าเชื่อถืออื่นๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกคุณว่าควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้ซึ่งสูงกว่าความเสี่ยงต่ำ                                |
| `Review`    | อ่านข้อค้นพบก่อนติดตั้ง รุ่นเผยแพร่อาจยังเป็นของถูกต้องตามเจตนา |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบข้อกังวลที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | ห้ามติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จสิ้น                                             |
| `Error`     | ไม่สามารถทำการตรวจสอบให้เสร็จสมบูรณ์ได้                                         |

`Pass` ช่วยให้มั่นใจได้ แต่ไม่แทนที่วิจารณญาณของคุณเอง สิ่งนี้สำคัญ
ที่สุดสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล รันคำสั่ง อ่านไฟล์ หรือ
เข้าถึงระบบที่ใช้งานจริง

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบายขอบเขตผลกระทบ: รุ่นเผยแพร่ดูเหมือนมีอำนาจมากเพียงใดหาก
คุณใช้งานตามที่ตั้งใจไว้

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบอำนาจที่เกี่ยวข้องกับข้อมูลสำคัญหรือผลกระทบต่อผู้ใช้น้อย                          |
| `Medium`   | รุ่นเผยแพร่มีอำนาจที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | รุ่นเผยแพร่มีอำนาจที่มีผลกระทบสูง ข้อค้นพบร้ายแรง หรือสัญญาณประสงค์ร้าย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคำถามคนละอย่าง:

- ระดับความเสี่ยงถามว่า: "ที่นี่มีอำนาจมากแค่ไหน?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอย่างไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น Skills สำหรับเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยง `Medium` ซึ่ง
ไม่ได้หมายความว่ามีเจตนาร้าย แต่หมายความว่า Skills ดูสอดคล้องกับวัตถุประสงค์ แต่สามารถ
ดำเนินการด้วยอำนาจบัญชีที่มีนัยสำคัญได้

## ข้อค้นพบ

ข้อค้นพบอธิบายว่าทำไมจึงแสดงผลการตรวจสอบนั้น โดยปกติข้อค้นพบแต่ละรายการจะรวมถึง:

- ความหมายของข้อค้นพบนั้น
- เหตุผลที่ถูกทำเครื่องหมาย
- เนื้อหา Skills หรือ Plugin ที่เกี่ยวข้อง
- คำแนะนำ

ข้อค้นพบอาจมีป้ายกำกับเป็น `Info`, `Low`, `Medium`, `High` หรือ `Critical` ข้อค้นพบที่มี
ความรุนแรงสูงกว่าจะมีผลต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

ข้อค้นพบที่มีความเชื่อมั่นต่ำจะถูกซ่อนจากสรุปการตรวจสอบสาธารณะ เพื่อให้หน้า
มุ่งเน้นหลักฐานที่เป็นประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบอาร์ติแฟกต์รุ่นเผยแพร่ที่ส่งเข้ามา รวมถึง:

- คำสั่งของ Skills หรือเมตาดาต้าของ Plugin
- ตัวแปรสภาพแวดล้อมและสิทธิ์ที่ประกาศไว้
- คำแนะนำการติดตั้งและเมตาดาต้าแพ็กเกจ
- ไฟล์ที่รวมมาและรายการไฟล์
- เมตาดาต้าความเข้ากันได้และความสามารถ

คำถามหลักคือความสอดคล้อง: ชื่อ สรุป เมตาดาต้า อำนาจที่ขอ
และเนื้อหาจริงสอดคล้องกับสิ่งที่ผู้ใช้คาดหวังได้อย่างสมเหตุสมผลหรือไม่?

พฤติกรรมที่มีอำนาจสูงไม่ได้แปลว่าไม่ดีโดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลรับรอง
คำสั่ง local, API ของผู้ให้บริการ หรือการติดตั้งแพ็กเกจ การตรวจสอบจะตรวจว่าพลังนั้น
เป็นสิ่งที่คาดหมายได้ เปิดเผยไว้ และสมส่วนหรือไม่

หน้าอาร์ติแฟกต์ลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบประกอบด้วย:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็นเทเลเมทรีมัลแวร์ในชุดการตรวจสอบ VirusTotal เป็น
มาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับชื่อเสียงของไฟล์และการสแกนมัลแวร์ และ
ความร่วมมือของเราทำให้ ClawHub เพิ่มข่าวกรองด้านความปลอดภัยที่กว้างขึ้นให้กับการรีวิว Skills และ Plugin ได้

VirusTotal มีประโยชน์เป็นพิเศษสำหรับอาร์ติแฟกต์ประสงค์ร้ายที่รู้จัก การตรวจพบจากเอนจิน และ
สัญญาณชื่อเสียงที่ช่วยเสริมการรีวิวที่เข้าใจเอเจนต์ของ ClawHub เมื่อมีจำนวนเอนจินของผู้ให้บริการ
การตรวจสอบจะสรุปเป็นภาษาธรรมดา เช่น:

```text
62/62 vendors flagged this skill as clean.
```

หรือ:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

เมื่อ ClawHub ไม่มีเทเลเมทรีจำนวนผู้ให้บริการให้สรุป การตรวจสอบจะระบุว่า:

```text
No VirusTotal findings
```

VirusTotal ยังคงเป็นเทเลเมทรี และไม่ได้แทนที่
การวิเคราะห์ความเสี่ยงที่เข้าใจอาร์ติแฟกต์ของ ClawHub เอง

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัยของ
ClawHub เอง ระบบนี้รีวิวแต่ละรุ่นเผยแพร่ในฐานะอาร์ติแฟกต์ที่เอเจนต์ต้องเผชิญ: คำสั่ง
เมตาดาต้า สิทธิ์ที่ประกาศไว้ ไฟล์ สัญญาณความสามารถ สัญญาณการสแกนแบบสแตติก
ข้อค้นพบของ SkillSpector เทเลเมทรี VirusTotal และบริบทที่ผู้เผยแพร่ให้มา
สัญญาณการสแกนแบบสแตติกเป็นบริบทภายในสำหรับการรีวิวนี้ ไม่ใช่
ส่วนการตรวจสอบสาธารณะแบบแยกเดี่ยวหรือคำตัดสินที่บล็อกการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นกรอบมองความเสี่ยง เช่น prompt injection, การใช้เครื่องมือในทางที่ผิด, การเปิดเผยข้อมูลรับรอง,
การดำเนินการที่ไม่ปลอดภัย, การปนเปื้อนหน่วยความจำหรือบริบท และอำนาจดำเนินการที่มากเกินไป

ClawScan ไม่ถือว่าความสามารถที่ดูน่ากลัวเป็นสิ่งประสงค์ร้ายโดยอัตโนมัติ
ระบบจะถามว่าความสามารถนั้นถูกเปิดเผย สอดคล้องกับวัตถุประสงค์ และมี
กรณีการใช้งานที่ระบุไว้ของรุ่นเผยแพร่รองรับหรือไม่
