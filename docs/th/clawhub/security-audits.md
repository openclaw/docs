# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้ตัดสินใจได้ว่า Skills หรือ Plugin มีความปลอดภัยเพียงพอ
สำหรับการติดตั้งหรือไม่ โดยแสดงว่ารีลีสทำอะไร ร้องขอสิทธิ์อำนาจใด และ
มีสิ่งใดที่ควรตรวจสอบเพิ่มเติมก่อนจะเข้าถึงไฟล์ บัญชี
ข้อมูลประจำตัว โค้ด หรือบริการภายนอก

การตรวจสอบเป็นสัญญาณด้านความปลอดภัยที่ชัดเจน แต่ไม่ได้รับประกันว่ารีลีสจะ
ปราศจากความเสี่ยง ควรใช้วิจารณญาณเสมอก่อนอนุญาตการเข้าถึงข้อมูลหรือระบบที่ละเอียดอ่อน

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security), [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)
และ [การกลั่นกรองและความปลอดภัยของบัญชี](/clawhub/moderation)

## สิ่งที่ควรตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจสอบ:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- ข้อค้นพบทั้งหมดที่ระบุไว้
- ข้อมูลประจำตัว สิทธิ์ หรือ environment variables ที่จำเป็น
- เจ้าของ แหล่งที่มา เวอร์ชัน บันทึกการเปลี่ยนแปลง จำนวนดาวน์โหลด ดาว และสัญญาณความน่าเชื่อถืออื่นๆ

ติดตั้งเฉพาะเนื้อหาที่เข้าใจและเชื่อถือเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกว่าควรดำเนินการอย่างไรต่อผลการตรวจสอบ:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้ซึ่งมีระดับสูงกว่าความเสี่ยงต่ำ                                |
| `Review`    | อ่านข้อค้นพบก่อนติดตั้ง รีลีสอาจยังคงถูกต้องและน่าเชื่อถือ |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบข้อกังวลที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | ห้ามติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จสิ้น                                             |
| `Error`     | ไม่สามารถดำเนินการตรวจสอบให้เสร็จสมบูรณ์ได้                                         |

`Pass` ช่วยให้มั่นใจได้ แต่ไม่สามารถทดแทนวิจารณญาณของตนเองได้ เรื่องนี้สำคัญ
อย่างยิ่งสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล เรียกใช้คำสั่ง อ่านไฟล์ หรือ
เข้าถึงระบบที่ใช้งานจริง

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบายขอบเขตผลกระทบ: รีลีสดูเหมือนจะมีอำนาจมากเพียงใดหาก
ใช้งานตามวัตถุประสงค์

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบสิทธิ์อำนาจที่ละเอียดอ่อนหรือผลกระทบต่อผู้ใช้เพียงเล็กน้อย                          |
| `Medium`   | รีลีสมีสิทธิ์อำนาจที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | รีลีสมีสิทธิ์อำนาจที่ส่งผลกระทบสูง ข้อค้นพบที่ร้ายแรง หรือสัญญาณที่เป็นอันตราย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคำถามที่แตกต่างกัน:

- ระดับความเสี่ยงถามว่า: "มีอำนาจมากเพียงใดในที่นี้?"
- สถานะการตรวจสอบถามว่า: "ควรทำอย่างไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น Skills สำหรับการเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยงระดับ `Medium` ซึ่ง
ไม่ได้หมายความว่าเป็นอันตราย แต่หมายความว่า Skills ดูสอดคล้องกับวัตถุประสงค์ แต่สามารถ
ดำเนินการโดยใช้สิทธิ์อำนาจของบัญชีที่มีนัยสำคัญ

## ข้อค้นพบ

ข้อค้นพบอธิบายสาเหตุที่แสดงผลการตรวจสอบ โดยทั่วไปข้อค้นพบแต่ละรายการประกอบด้วย:

- ความหมาย
- สาเหตุที่ถูกตั้งค่าสถานะ
- เนื้อหาของ Skills หรือ Plugin ที่เกี่ยวข้อง
- คำแนะนำ

ข้อค้นพบอาจมีป้ายกำกับ `Info`, `Low`, `Medium`, `High` หรือ `Critical` ข้อค้นพบที่มี
ความรุนแรงสูงกว่าจะส่งผลต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

ข้อค้นพบที่มีความเชื่อมั่นต่ำจะถูกซ่อนจากข้อมูลสรุปการตรวจสอบสาธารณะ เพื่อให้หน้า
มุ่งเน้นหลักฐานที่เป็นประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบอาร์ติแฟกต์ของรีลีสที่ส่งเข้ามา ซึ่งรวมถึง:

- คำสั่งของ Skills หรือข้อมูลเมตาของ Plugin
- environment variables และสิทธิ์ที่ประกาศไว้
- คำแนะนำการติดตั้งและข้อมูลเมตาของแพ็กเกจ
- ไฟล์และรายการไฟล์ที่รวมอยู่
- ข้อมูลเมตาด้านความเข้ากันได้และความสามารถ

คำถามหลักคือความสอดคล้อง: ชื่อ ข้อมูลสรุป ข้อมูลเมตา สิทธิ์อำนาจที่ร้องขอ
และเนื้อหาจริงสอดคล้องกับสิ่งที่ผู้ใช้คาดหวังอย่างสมเหตุสมผลหรือไม่?

พฤติกรรมที่มีอำนาจสูงไม่ได้เป็นสิ่งไม่ดีโดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากจำเป็นต้องใช้ข้อมูลประจำตัว
คำสั่งภายในเครื่อง API ของผู้ให้บริการ หรือการติดตั้งแพ็กเกจ การตรวจสอบจะพิจารณาว่า
อำนาจดังกล่าวเป็นไปตามความคาดหมาย มีการเปิดเผย และได้สัดส่วนหรือไม่

หน้าอาร์ติแฟกต์จะลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบประกอบด้วย:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็นข้อมูลเทเลเมทรีเกี่ยวกับมัลแวร์ในชุดระบบการตรวจสอบ VirusTotal เป็น
มาตรฐานอุตสาหกรรมที่ได้รับความเชื่อถือสำหรับการประเมินชื่อเสียงของไฟล์และการสแกนมัลแวร์ และ
ความร่วมมือของเราช่วยให้ ClawHub เพิ่มข้อมูลข่าวกรองด้านความปลอดภัยที่กว้างขึ้นในการตรวจสอบ Skills และ Plugin

VirusTotal มีประโยชน์อย่างยิ่งสำหรับอาร์ติแฟกต์ที่ทราบว่าเป็นอันตราย ผลตรวจพบจากเอนจิน และ
สัญญาณด้านชื่อเสียงที่ช่วยเสริมการตรวจสอบซึ่งตระหนักถึงเอเจนต์ของ ClawHub เมื่อมี
จำนวนเอนจินของผู้ให้บริการ หน้าการตรวจสอบจะสรุปเป็นภาษาที่เข้าใจง่าย เช่น:

```text
ผู้ให้บริการ 62/62 รายระบุว่า Skills นี้ปลอดภัย
```

หรือ:

```text
ผู้ให้บริการ 2/64 รายระบุว่า Skills นี้เป็นอันตราย 1/64 รายระบุว่าน่าสงสัย และ 61/64 รายระบุว่าปลอดภัย
```

เมื่อ ClawHub ไม่มีข้อมูลเทเลเมทรีจำนวนผู้ให้บริการให้สรุป หน้าการตรวจสอบจะระบุว่า:

```text
ไม่พบข้อค้นพบจาก VirusTotal
```

VirusTotal ยังคงเป็นข้อมูลเทเลเมทรี และไม่สามารถทดแทนการวิเคราะห์
ความเสี่ยงที่ตระหนักถึงอาร์ติแฟกต์ของ ClawHub เองได้

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัย
ของ ClawHub เอง ระบบนี้ตรวจสอบแต่ละรีลีสในฐานะอาร์ติแฟกต์สำหรับเอเจนต์ ได้แก่ คำสั่ง
ข้อมูลเมตา สิทธิ์ที่ประกาศไว้ ไฟล์ สัญญาณความสามารถ สัญญาณจากการสแกนแบบสแตติก
ข้อค้นพบจาก SkillSpector ข้อมูลเทเลเมทรีจาก VirusTotal และบริบทที่ผู้เผยแพร่ให้มา
สัญญาณจากการสแกนแบบสแตติกเป็นบริบทภายในสำหรับการตรวจสอบนี้ ไม่ใช่
ส่วนการตรวจสอบสาธารณะแบบแยกต่างหากหรือคำตัดสินที่ขัดขวางการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[10 ความเสี่ยงสูงสุดของ Skills สำหรับเอเจนต์โดย OWASP](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นกรอบพิจารณาความเสี่ยง เช่น การแทรกพรอมต์ การใช้เครื่องมือในทางที่ผิด การเปิดเผยข้อมูลประจำตัว
การดำเนินการที่ไม่ปลอดภัย การทำให้หน่วยความจำหรือบริบทเป็นพิษ และการให้อำนาจดำเนินการมากเกินไป

ClawScan ไม่ถือว่าความสามารถที่ดูน่ากลัวเป็นอันตรายโดยอัตโนมัติ
แต่จะพิจารณาว่าความสามารถนั้นได้รับการเปิดเผย สอดคล้องกับวัตถุประสงค์ และได้รับการสนับสนุนจาก
กรณีการใช้งานที่รีลีสระบุไว้หรือไม่
