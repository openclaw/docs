# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจได้ว่า Skills หรือ Plugin ปลอดภัยพอ
ที่จะติดตั้งหรือไม่ โดยจะแสดงว่ารุ่นเผยแพร่ทำอะไร ขออำนาจใดบ้าง และ
มีสิ่งใดที่ควรให้ความสนใจเป็นพิเศษก่อนที่จะเข้าถึงไฟล์ บัญชี
ข้อมูลประจำตัว โค้ด หรือบริการภายนอกหรือไม่

การตรวจสอบเป็นสัญญาณด้านความปลอดภัยที่หนักแน่น แต่ไม่ได้รับประกันว่ารุ่นเผยแพร่จะ
ปราศจากความเสี่ยง ใช้วิจารณญาณเสมอก่อนให้สิทธิ์เข้าถึงข้อมูลหรือระบบที่ละเอียดอ่อน

ดูเพิ่มเติมที่ [ความปลอดภัย](/th/clawhub/security), [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage),
และ [การดูแลเนื้อหาและความปลอดภัยของบัญชี](/th/clawhub/moderation)

## สิ่งที่ต้องตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจทาน:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- ข้อค้นพบที่ระบุไว้
- ข้อมูลประจำตัว สิทธิ์ หรือ environment variable ที่ต้องใช้
- เจ้าของ แหล่งที่มา เวอร์ชัน changelog ยอดดาวน์โหลด ดาว และสัญญาณความน่าเชื่อถืออื่นๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกว่าคุณควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้ซึ่งสูงกว่าความเสี่ยงต่ำ                                |
| `Review`    | อ่านข้อค้นพบก่อนติดตั้ง รุ่นเผยแพร่อาจยังคงถูกต้องตามเจตนา |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบข้อกังวลที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | อย่าติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จสิ้น                                             |
| `Error`     | ไม่สามารถดำเนินการตรวจสอบให้เสร็จสมบูรณ์ได้                                         |

`Pass` ช่วยให้มั่นใจได้ แต่ไม่แทนที่วิจารณญาณของคุณเอง สิ่งนี้สำคัญ
ที่สุดสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล รันคำสั่ง อ่านไฟล์ หรือ
เข้าถึงระบบ production ได้

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบายขอบเขตผลกระทบ: รุ่นเผยแพร่ดูเหมือนมีอำนาจมากเพียงใดหาก
คุณใช้งานตามที่ตั้งใจไว้

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบอำนาจที่ละเอียดอ่อนหรือผลกระทบต่อผู้ใช้น้อยมาก                          |
| `Medium`   | รุ่นเผยแพร่มีอำนาจที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | รุ่นเผยแพร่มีอำนาจที่มีผลกระทบสูง ข้อค้นพบรุนแรง หรือสัญญาณอันตราย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคำถามต่างกัน:

- ระดับความเสี่ยงถามว่า: "มีอำนาจมากแค่ไหนที่นี่?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอย่างไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น Skills สำหรับการเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยง `Medium` นั่น
ไม่ได้หมายความว่ามีเจตนาร้าย แต่หมายความว่า Skills ดูสอดคล้องกับวัตถุประสงค์ แต่สามารถ
ดำเนินการด้วยอำนาจของบัญชีที่มีนัยสำคัญได้

## ข้อค้นพบ

ข้อค้นพบอธิบายว่าทำไมจึงแสดงผลการตรวจสอบนั้น โดยปกติแต่ละข้อค้นพบจะมี:

- ความหมายของข้อค้นพบนั้น
- เหตุผลที่ถูกระบุ
- เนื้อหา Skills หรือ Plugin ที่เกี่ยวข้อง
- คำแนะนำ

ข้อค้นพบอาจมีป้ายกำกับ `Info`, `Low`, `Medium`, `High` หรือ `Critical` ข้อค้นพบที่มี
ความรุนแรงสูงกว่าจะมีส่วนต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

ข้อค้นพบที่มีความเชื่อมั่นต่ำจะถูกซ่อนจากสรุปการตรวจสอบสาธารณะ เพื่อให้หน้า
ยังคงเน้นหลักฐานที่เป็นประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบอาร์ติแฟกต์รุ่นเผยแพร่ที่ส่งเข้ามา รวมถึง:

- คำสั่ง Skills หรือ metadata ของ Plugin
- environment variable และสิทธิ์ที่ประกาศไว้
- คำแนะนำการติดตั้งและ metadata ของแพ็กเกจ
- ไฟล์ที่รวมไว้และ manifest ของไฟล์
- metadata ด้านความเข้ากันได้และความสามารถ

คำถามหลักคือความสอดคล้อง: ชื่อ สรุป metadata อำนาจที่ขอ
และเนื้อหาจริงสอดคล้องกับสิ่งที่ผู้ใช้คาดหวังอย่างสมเหตุสมผลหรือไม่

พฤติกรรมที่ทรงพลังไม่ได้แย่โดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลประจำตัว
คำสั่งในเครื่อง API ของผู้ให้บริการ หรือการติดตั้งแพ็กเกจ การตรวจสอบจะดูว่าอำนาจนั้น
เป็นสิ่งที่คาดไว้ เปิดเผยไว้ และได้สัดส่วนหรือไม่

หน้าอาร์ติแฟกต์ลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบรวม:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็น telemetry มัลแวร์ในชุดการตรวจสอบ VirusTotal เป็น
มาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับชื่อเสียงของไฟล์และการสแกนมัลแวร์ และ
ความร่วมมือของเราช่วยให้ ClawHub เพิ่มข้อมูลความปลอดภัยที่กว้างขึ้นให้กับการตรวจทาน Skills และ Plugin

VirusTotal มีประโยชน์เป็นพิเศษสำหรับอาร์ติแฟกต์ที่ทราบว่ามีเจตนาร้าย การตรวจพบจากเอนจิน และ
สัญญาณชื่อเสียงที่เสริมการตรวจทานแบบรับรู้เอเจนต์ของ ClawHub เมื่อมีจำนวนจากเอนจินของผู้ให้บริการ
การตรวจสอบจะสรุปเป็นภาษาธรรมดา เช่น:

```text
62/62 vendors flagged this skill as clean.
```

หรือ:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

เมื่อ ClawHub ไม่มี telemetry จำนวนผู้ให้บริการให้สรุป การตรวจสอบจะระบุว่า:

```text
No VirusTotal findings
```

VirusTotal ยังคงเป็น telemetry และไม่ได้แทนที่การวิเคราะห์ความเสี่ยงที่รับรู้อาร์ติแฟกต์
ของ ClawHub เอง

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัยของ
ClawHub เอง ระบบจะตรวจทานแต่ละรุ่นเผยแพร่ในฐานะอาร์ติแฟกต์สำหรับเอเจนต์: คำสั่ง
metadata สิทธิ์ที่ประกาศไว้ ไฟล์ สัญญาณความสามารถ สัญญาณการสแกนแบบคงที่
ข้อค้นพบของ SkillSpector, telemetry ของ VirusTotal และบริบทที่ผู้เผยแพร่ให้มา
สัญญาณการสแกนแบบคงที่เป็นบริบทภายในสำหรับการตรวจทานนี้ ไม่ใช่
ส่วนการตรวจสอบสาธารณะแบบสแตนด์อโลนหรือคำตัดสินที่บล็อกการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นเลนส์สำหรับความเสี่ยง เช่น prompt injection, การใช้เครื่องมือผิดวัตถุประสงค์, การเปิดเผยข้อมูลประจำตัว,
การดำเนินการที่ไม่ปลอดภัย, memory หรือการปนเปื้อนบริบท และการให้สิทธิ์กระทำการมากเกินไป

ClawScan ไม่ถือว่าความสามารถที่ดูน่ากลัวเป็นเจตนาร้ายโดยอัตโนมัติ
ระบบจะถามว่าความสามารถนั้นถูกเปิดเผยไว้ สอดคล้องกับวัตถุประสงค์ และได้รับการสนับสนุนโดย
กรณีการใช้งานที่รุ่นเผยแพร่ระบุไว้หรือไม่
