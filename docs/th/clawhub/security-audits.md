# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจได้ว่าสกิลหรือ Plugin ปลอดภัยเพียงพอ
สำหรับการติดตั้งหรือไม่ การตรวจสอบจะแสดงว่ารีลีสทำอะไร ขอสิทธิ์อำนาจใด และ
มีสิ่งใดที่ควรให้ความสนใจเพิ่มเติมก่อนจะเข้าถึงไฟล์ บัญชี
ข้อมูลรับรอง โค้ด หรือบริการภายนอกหรือไม่

การตรวจสอบเป็นสัญญาณความปลอดภัยที่สำคัญ แต่ไม่ใช่การรับประกันว่ารีลีสจะ
ไร้ความเสี่ยง ใช้วิจารณญาณเสมอก่อนให้สิทธิ์เข้าถึงข้อมูลที่ละเอียดอ่อน

ดูเพิ่มเติมที่ [ความปลอดภัย](/th/clawhub/security), [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage),
และ [การกลั่นกรองและความปลอดภัยของบัญชี](/th/clawhub/moderation)

## สิ่งที่ควรตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจสอบ:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- ข้อค้นพบที่ระบุไว้
- ข้อมูลรับรอง สิทธิ์อนุญาต หรือตัวแปรสภาพแวดล้อมที่จำเป็น
- เจ้าของ แหล่งที่มา เวอร์ชัน บันทึกการเปลี่ยนแปลง ยอดดาวน์โหลด ดาว และสัญญาณความน่าเชื่อถืออื่นๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกว่าคุณควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้เหนือความเสี่ยงต่ำ                                |
| `Review`    | อ่านข้อค้นพบก่อนติดตั้ง รีลีสอาจยังคงถูกต้องตามเจตนา |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบข้อกังวลที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | ห้ามติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จสิ้น                                             |
| `Error`     | ไม่สามารถดำเนินการตรวจสอบให้เสร็จสมบูรณ์ได้                                         |

`Pass` ช่วยให้มั่นใจได้ แต่ไม่ได้ทดแทนวิจารณญาณของคุณเอง สิ่งนี้สำคัญ
ที่สุดสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล รันคำสั่ง อ่านไฟล์ หรือ
เข้าถึงระบบ production

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบายขอบเขตผลกระทบ: รีลีสดูเหมือนมีอำนาจมากเพียงใดหาก
คุณใช้งานตามเจตนาที่ออกแบบไว้

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบอำนาจที่ละเอียดอ่อนหรือผลกระทบต่อผู้ใช้น้อย                          |
| `Medium`   | รีลีสมีอำนาจที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | รีลีสมีอำนาจที่มีผลกระทบสูง ข้อค้นพบรุนแรง หรือสัญญาณที่เป็นอันตราย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคำถามคนละข้อ:

- ระดับความเสี่ยงถามว่า: "มีอำนาจมากเพียงใดอยู่ที่นี่?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอย่างไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น สกิลสำหรับการเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยง `Medium` นั่น
ไม่ได้หมายความว่ามันเป็นอันตราย แต่หมายความว่าสกิลดูสอดคล้องกับวัตถุประสงค์ แต่สามารถ
ดำเนินการด้วยอำนาจระดับบัญชีที่มีนัยสำคัญได้

## ข้อค้นพบ

ข้อค้นพบอธิบายว่าทำไมจึงแสดงผลการตรวจสอบนั้น ข้อค้นพบแต่ละรายการมักมี:

- ความหมายของข้อค้นพบนั้น
- เหตุผลที่ถูกแจ้งเตือน
- เนื้อหาสกิลหรือ Plugin ที่เกี่ยวข้อง
- คำแนะนำ

ข้อค้นพบอาจมีป้ายกำกับเป็น `Info`, `Low`, `Medium`, `High` หรือ `Critical` ข้อค้นพบที่มี
ความรุนแรงสูงกว่าจะมีผลต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

ข้อค้นพบที่มีความเชื่อมั่นต่ำจะถูกซ่อนจากสรุปการตรวจสอบสาธารณะ เพื่อให้หน้า
ยังคงมุ่งเน้นหลักฐานที่เป็นประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบอาร์ติแฟกต์รีลีสที่ส่งมา รวมถึง:

- คำสั่งสกิลหรือเมทาดาทาของ Plugin
- ตัวแปรสภาพแวดล้อมและสิทธิ์อนุญาตที่ประกาศไว้
- คำแนะนำการติดตั้งและเมทาดาทาของแพ็กเกจ
- ไฟล์ที่รวมอยู่และแมนิเฟสต์ไฟล์
- เมทาดาทาความเข้ากันได้และความสามารถ

คำถามหลักคือความสอดคล้อง: ชื่อ สรุป เมทาดาทา
อำนาจที่ร้องขอ และเนื้อหาจริงสอดคล้องกับสิ่งที่ผู้ใช้ควรคาดหวังอย่างสมเหตุสมผลหรือไม่?

พฤติกรรมที่มีอำนาจสูงไม่ได้แปลว่าไม่ดีโดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลรับรอง
คำสั่งภายในเครื่อง API ของผู้ให้บริการ หรือการติดตั้งแพ็กเกจ การตรวจสอบจะตรวจว่าพลังดังกล่าว
เป็นสิ่งที่คาดหวัง เปิดเผยไว้ และได้สัดส่วนหรือไม่

หน้าอาร์ติแฟกต์ลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบรวม:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็นเทเลเมทรีมัลแวร์ในชุดการตรวจสอบ VirusTotal เป็น
มาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับชื่อเสียงของไฟล์และการสแกนมัลแวร์ และ
ความร่วมมือของเราช่วยให้ ClawHub เพิ่มข้อมูลข่าวกรองด้านความปลอดภัยที่กว้างขึ้นให้กับการรีวิวสกิลและ Plugin

VirusTotal มีประโยชน์เป็นพิเศษสำหรับอาร์ติแฟกต์ที่ทราบว่าเป็นอันตราย การตรวจพบของเอนจิน และ
สัญญาณชื่อเสียงที่ช่วยเสริมการรีวิวแบบเข้าใจเอเจนต์ของ ClawHub เมื่อมี
จำนวนการตรวจพบของเอนจินผู้ให้บริการ การตรวจสอบจะสรุปเป็นภาษาธรรมดา เช่น:

```text
62/62 vendors flagged this skill as clean.
```

หรือ:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

เมื่อ ClawHub ไม่มีเทเลเมทรีจำนวนผู้ให้บริการให้สรุป การตรวจสอบจะระบุว่า:

```text
No VirusTotal findings
```

VirusTotal ยังคงเป็นเทเลเมทรี ไม่ได้ทดแทนการวิเคราะห์ความเสี่ยงแบบเข้าใจอาร์ติแฟกต์ของ ClawHub เอง

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัยของ ClawHub เอง
ระบบจะรีวิวแต่ละรีลีสในฐานะอาร์ติแฟกต์ที่เอเจนต์ต้องใช้: คำสั่ง
เมทาดาทา สิทธิ์อนุญาตที่ประกาศไว้ ไฟล์ สัญญาณความสามารถ สัญญาณการสแกนแบบสถิต
ข้อค้นพบของ SkillSpector เทเลเมทรีของ VirusTotal และบริบทที่ผู้เผยแพร่ให้ไว้
สัญญาณการสแกนแบบสถิตเป็นบริบทภายในสำหรับการรีวิวนี้ ไม่ใช่
ส่วนการตรวจสอบสาธารณะแบบสแตนด์อโลนหรือคำตัดสินที่บล็อกการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นกรอบพิจารณาความเสี่ยง เช่น การแทรก prompt การใช้เครื่องมือผิดวัตถุประสงค์ การเปิดเผยข้อมูลรับรอง
การดำเนินการที่ไม่ปลอดภัย การปนเปื้อนของหน่วยความจำหรือบริบท และอำนาจการตัดสินใจที่มากเกินไป

ClawScan ไม่ถือว่าความสามารถที่ดูน่ากลัวเป็นสิ่งที่เป็นอันตรายโดยอัตโนมัติ
ระบบจะถามว่าความสามารถนั้นถูกเปิดเผย สอดคล้องกับวัตถุประสงค์ และได้รับการสนับสนุนโดย
กรณีการใช้งานที่รีลีสระบุไว้หรือไม่
