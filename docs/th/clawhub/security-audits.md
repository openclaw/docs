# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจได้ว่าสกิลหรือ Plugin ปลอดภัยพอ
สำหรับติดตั้งหรือไม่ การตรวจสอบจะแสดงว่ารีลีสทำอะไร ขอสิทธิ์อำนาจใดบ้าง และ
มีสิ่งใดควรได้รับความสนใจเป็นพิเศษก่อนที่จะเข้าถึงไฟล์ บัญชี
ข้อมูลประจำตัว โค้ด หรือบริการภายนอก

การตรวจสอบเป็นสัญญาณด้านความปลอดภัยที่หนักแน่น แต่ไม่ใช่การรับประกันว่ารีลีสจะ
ไม่มีความเสี่ยง ใช้วิจารณญาณเสมอก่อนให้สิทธิ์เข้าถึงที่อ่อนไหว

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security), [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage),
และ [การดูแลเนื้อหาและความปลอดภัยของบัญชี](/clawhub/moderation)

## สิ่งที่ควรตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจสอบ:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- ผลการตรวจพบที่ระบุไว้
- ข้อมูลประจำตัว สิทธิ์ หรือ Environment Variable ที่จำเป็น
- เจ้าของ แหล่งที่มา เวอร์ชัน บันทึกการเปลี่ยนแปลง ยอดดาวน์โหลด ดาว และสัญญาณความน่าเชื่อถืออื่นๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกคุณว่าควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้ซึ่งสูงกว่าความเสี่ยงต่ำ                                |
| `Review`    | อ่านผลการตรวจพบก่อนติดตั้ง รีลีสอาจยังเป็นของถูกต้องตามเจตนาได้ |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบข้อกังวลที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | ห้ามติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จ                                             |
| `Error`     | ไม่สามารถดำเนินการตรวจสอบให้เสร็จได้                                         |

`Pass` ช่วยให้มั่นใจขึ้น แต่ไม่ได้แทนที่วิจารณญาณของคุณเอง เรื่องนี้สำคัญที่สุด
สำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล รันคำสั่ง อ่านไฟล์ หรือ
เข้าถึงระบบ production

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบายขอบเขตผลกระทบ: รีลีสดูเหมือนมีอำนาจมากเพียงใดหาก
คุณใช้งานตามเจตนา

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบอำนาจที่อ่อนไหวหรือผลกระทบต่อผู้ใช้น้อย                          |
| `Medium`   | รีลีสมีอำนาจที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | รีลีสมีอำนาจที่มีผลกระทบสูง ผลการตรวจพบที่รุนแรง หรือสัญญาณที่เป็นอันตราย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคำถามคนละอย่าง:

- ระดับความเสี่ยงถามว่า: "มีอำนาจมากแค่ไหนอยู่ที่นี่?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอย่างไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น สกิลสำหรับเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยง `Medium` ซึ่ง
ไม่ได้หมายความว่าเป็นอันตราย แต่หมายความว่าสกิลดูสอดคล้องกับวัตถุประสงค์ แต่สามารถ
ดำเนินการด้วยอำนาจบัญชีที่มีนัยสำคัญได้

## ผลการตรวจพบ

ผลการตรวจพบอธิบายว่าทำไมจึงแสดงผลการตรวจสอบนั้น โดยปกติแต่ละรายการจะรวมถึง:

- ความหมายของรายการนั้น
- เหตุผลที่ถูกระบุ
- เนื้อหาสกิลหรือ Plugin ที่เกี่ยวข้อง
- คำแนะนำ

ผลการตรวจพบอาจติดป้าย `Info`, `Low`, `Medium`, `High` หรือ `Critical` ผลการตรวจพบที่มี
ความรุนแรงสูงกว่าจะส่งผลต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

ผลการตรวจพบที่มีความเชื่อมั่นต่ำจะถูกซ่อนจากสรุปการตรวจสอบสาธารณะ เพื่อให้หน้า
มุ่งเน้นหลักฐานที่มีประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบอาร์ติแฟกต์ของรีลีสที่ส่งเข้ามา รวมถึง:

- คำสั่งของสกิลหรือเมทาดาทาของ Plugin
- Environment Variable และสิทธิ์ที่ประกาศไว้
- คำแนะนำการติดตั้งและเมทาดาทาของแพ็กเกจ
- ไฟล์ที่รวมไว้และรายการไฟล์
- เมทาดาทาความเข้ากันได้และความสามารถ

คำถามหลักคือความสอดคล้อง: ชื่อ สรุป เมทาดาทา อำนาจที่ขอ
และเนื้อหาจริงสอดคล้องกับสิ่งที่ผู้ใช้คาดหวังได้อย่างสมเหตุสมผลหรือไม่?

พฤติกรรมที่ทรงพลังไม่ได้แย่โดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลประจำตัว
คำสั่งในเครื่อง API ของผู้ให้บริการ หรือการติดตั้งแพ็กเกจ การตรวจสอบจะดูว่าอำนาจนั้น
เป็นสิ่งที่คาดหมายได้ เปิดเผยไว้ และได้สัดส่วนหรือไม่

หน้าอาร์ติแฟกต์ลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบรวม:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็นข้อมูลโทรมาตรมัลแวร์ในชุดการตรวจสอบ VirusTotal เป็น
มาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับชื่อเสียงของไฟล์และการสแกนมัลแวร์ และ
ความร่วมมือของเราช่วยให้ ClawHub เพิ่มข้อมูลข่าวกรองด้านความปลอดภัยที่กว้างขึ้นในการรีวิวสกิลและ Plugin

VirusTotal มีประโยชน์เป็นพิเศษสำหรับอาร์ติแฟกต์อันตรายที่รู้จักอยู่แล้ว การตรวจพบของเอนจิน และ
สัญญาณชื่อเสียงที่เสริมการรีวิวของ ClawHub ซึ่งเข้าใจบริบทของเอเจนต์ เมื่อมีจำนวน
เอนจินจากผู้จำหน่ายพร้อมใช้งาน การตรวจสอบจะสรุปเป็นภาษาธรรมดา เช่น:

```text
62/62 vendors flagged this skill as clean.
```

หรือ:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

เมื่อ ClawHub ไม่มีข้อมูลโทรมาตรจำนวนผู้จำหน่ายให้สรุป การตรวจสอบจะระบุว่า:

```text
No VirusTotal findings
```

VirusTotal ยังคงเป็นข้อมูลโทรมาตร ไม่ได้แทนที่การวิเคราะห์ความเสี่ยงของ ClawHub เอง
ที่เข้าใจอาร์ติแฟกต์

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัยของ
ClawHub เอง ระบบนี้รีวิวแต่ละรีลีสในฐานะอาร์ติแฟกต์ที่หันหน้าเข้าหาเอเจนต์: คำสั่ง
เมทาดาทา สิทธิ์ที่ประกาศ ไฟล์ สัญญาณความสามารถ สัญญาณการสแกนแบบสถิต
ผลการตรวจพบจาก SkillSpector ข้อมูลโทรมาตรจาก VirusTotal และบริบทที่ผู้เผยแพร่ให้ไว้
สัญญาณการสแกนแบบสถิตเป็นบริบทภายในสำหรับการรีวิวนี้ ไม่ใช่
ส่วนการตรวจสอบสาธารณะแบบสแตนด์อโลนหรือคำตัดสินที่บล็อกการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นมุมมองสำหรับความเสี่ยง เช่น การแทรกพรอมป์ การใช้เครื่องมือผิดวัตถุประสงค์ การเปิดเผยข้อมูลประจำตัว
การดำเนินการที่ไม่ปลอดภัย การปนเปื้อนหน่วยความจำหรือบริบท และอำนาจการตัดสินใจที่มากเกินไป

ClawScan ไม่ถือว่าความสามารถที่ดูน่ากลัวเป็นอันตรายโดยอัตโนมัติ
ระบบจะถามว่าความสามารถนั้นถูกเปิดเผย สอดคล้องกับวัตถุประสงค์ และได้รับการสนับสนุนโดย
กรณีการใช้งานที่รีลีสระบุไว้หรือไม่
