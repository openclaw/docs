# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจได้ว่าทักษะหรือ Plugin ปลอดภัยพอที่จะติดตั้งหรือไม่ โดยจะแสดงว่ารุ่นเผยแพร่ทำอะไร ขออำนาจสิทธิ์ใด และมีสิ่งใดที่ควรให้ความสนใจเพิ่มเติมก่อนที่จะเข้าถึงไฟล์ บัญชี ข้อมูลประจำตัว โค้ด หรือบริการภายนอกหรือไม่

การตรวจสอบเป็นสัญญาณด้านความปลอดภัยที่เข้มแข็ง แต่ไม่ใช่การรับประกันว่ารุ่นเผยแพร่จะปราศจากความเสี่ยง ใช้วิจารณญาณเสมอก่อนอนุญาตการเข้าถึงที่ละเอียดอ่อน

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security), [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage) และ [การกลั่นกรองและความปลอดภัยของบัญชี](/clawhub/moderation)

## สิ่งที่ควรตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจสอบ:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- ข้อค้นพบที่ระบุไว้
- ข้อมูลประจำตัว สิทธิ์ หรือ environment variables ที่จำเป็น
- เจ้าของ แหล่งที่มา เวอร์ชัน บันทึกการเปลี่ยนแปลง ยอดดาวน์โหลด ดาว และสัญญาณความน่าเชื่อถืออื่นๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและไว้วางใจเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกคุณว่าควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้เหนือความเสี่ยงต่ำ                                |
| `Review`    | อ่านข้อค้นพบก่อนติดตั้ง รุ่นเผยแพร่อาจยังคงถูกต้องตามวัตถุประสงค์ |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบข้อกังวลที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | อย่าติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จสิ้น                                             |
| `Error`     | ไม่สามารถดำเนินการตรวจสอบให้เสร็จสมบูรณ์ได้                                         |

`Pass` ช่วยให้มั่นใจได้ แต่ไม่ได้แทนที่วิจารณญาณของคุณเอง สิ่งนี้สำคัญที่สุดสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล รันคำสั่ง อ่านไฟล์ หรือเข้าถึงระบบ production

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบายขอบเขตผลกระทบ: รุ่นเผยแพร่ดูเหมือนมีอำนาจมากเพียงใดหากคุณใช้งานตามวัตถุประสงค์

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบอำนาจสิทธิ์ที่ละเอียดอ่อนหรือผลกระทบต่อผู้ใช้เพียงเล็กน้อย                          |
| `Medium`   | รุ่นเผยแพร่มีอำนาจสิทธิ์ที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | รุ่นเผยแพร่มีอำนาจสิทธิ์ที่มีผลกระทบสูง ข้อค้นพบที่รุนแรง หรือสัญญาณที่เป็นอันตราย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคำถามคนละแบบ:

- ระดับความเสี่ยงถามว่า: "ที่นี่มีอำนาจมากเพียงใด?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอย่างไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น ทักษะสำหรับการเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยง `Medium` นั่นไม่ได้หมายความว่าเป็นอันตราย แต่หมายความว่าทักษะดูสอดคล้องกับวัตถุประสงค์ แต่สามารถดำเนินการด้วยอำนาจสิทธิ์บัญชีที่มีนัยสำคัญได้

## ข้อค้นพบ

ข้อค้นพบอธิบายว่าทำไมจึงแสดงผลการตรวจสอบนั้น ข้อค้นพบแต่ละรายการมักมี:

- ความหมายของข้อค้นพบ
- เหตุผลที่ถูกระบุ
- เนื้อหาทักษะหรือ Plugin ที่เกี่ยวข้อง
- คำแนะนำ

ข้อค้นพบอาจติดป้าย `Info`, `Low`, `Medium`, `High` หรือ `Critical` ข้อค้นพบที่มีความรุนแรงสูงกว่าจะมีผลต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

ข้อค้นพบที่มีความเชื่อมั่นต่ำจะถูกซ่อนจากสรุปการตรวจสอบสาธารณะ เพื่อให้หน้ายังคงโฟกัสที่หลักฐานที่เป็นประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบอาร์ติแฟกต์รุ่นเผยแพร่ที่ส่งเข้ามา รวมถึง:

- คำสั่งของทักษะหรือเมทาดาทาของ Plugin
- environment variables และสิทธิ์ที่ประกาศไว้
- คำสั่งติดตั้งและเมทาดาทาของแพ็กเกจ
- ไฟล์ที่รวมไว้และรายการไฟล์
- เมทาดาทาความเข้ากันได้และความสามารถ

คำถามหลักคือความสอดคล้อง: ชื่อ สรุป เมทาดาทา อำนาจสิทธิ์ที่ขอ และเนื้อหาจริง สอดคล้องกับสิ่งที่ผู้ใช้คาดหวังอย่างสมเหตุสมผลหรือไม่?

พฤติกรรมที่มีอำนาจสูงไม่ได้เป็นสิ่งไม่ดีโดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลประจำตัว คำสั่งภายในเครื่อง provider APIs หรือการติดตั้งแพ็กเกจ การตรวจสอบจะตรวจว่าพลังนั้นเป็นสิ่งที่คาดหมายได้ มีการเปิดเผย และได้สัดส่วนหรือไม่

หน้าอาร์ติแฟกต์ลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบรวม:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็น telemetry ของมัลแวร์ในชุดการตรวจสอบ VirusTotal เป็นมาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับชื่อเสียงของไฟล์และการสแกนมัลแวร์ และความร่วมมือของเราช่วยให้ ClawHub เพิ่มข้อมูลข่าวกรองด้านความปลอดภัยที่กว้างขึ้นในการตรวจสอบทักษะและ Plugin

VirusTotal มีประโยชน์อย่างยิ่งสำหรับอาร์ติแฟกต์อันตรายที่รู้จักอยู่แล้ว การตรวจพบจากเอนจิน และสัญญาณชื่อเสียงที่เสริมการตรวจสอบของ ClawHub ที่เข้าใจบริบทของเอเจนต์ เมื่อมีจำนวนเอนจินของผู้ให้บริการ การตรวจสอบจะสรุปเป็นภาษาง่ายๆ เช่น:

```text
62/62 vendors flagged this skill as clean.
```

หรือ:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

เมื่อ ClawHub ไม่มี telemetry จำนวนผู้ให้บริการให้สรุป การตรวจสอบจะระบุว่า:

```text
No VirusTotal findings
```

VirusTotal ยังคงเป็น telemetry ไม่ได้แทนที่การวิเคราะห์ความเสี่ยงของ ClawHub เองที่เข้าใจอาร์ติแฟกต์

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัยของ ClawHub เอง ระบบจะตรวจสอบแต่ละรุ่นเผยแพร่ในฐานะอาร์ติแฟกต์ที่หันหน้าเข้าหาเอเจนต์: คำสั่ง เมทาดาทา สิทธิ์ที่ประกาศไว้ ไฟล์ สัญญาณความสามารถ สัญญาณการสแกนแบบ static ข้อค้นพบของ SkillSpector, telemetry ของ VirusTotal และบริบทที่ผู้เผยแพร่ให้มา สัญญาณการสแกนแบบ static เป็นบริบทภายในสำหรับการตรวจสอบนี้ ไม่ใช่ส่วนการตรวจสอบสาธารณะแบบแยกเดี่ยวหรือคำตัดสินที่บล็อกการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นเลนส์สำหรับความเสี่ยง เช่น prompt injection, tool misuse, credential exposure, unsafe execution, memory or context poisoning และ excessive agency

ClawScan ไม่ถือว่าความสามารถที่ดูน่ากลัวเป็นอันตรายโดยอัตโนมัติ แต่จะถามว่าความสามารถนั้นถูกเปิดเผย สอดคล้องกับวัตถุประสงค์ และได้รับการสนับสนุนจากกรณีการใช้งานที่รุ่นเผยแพร่ระบุไว้หรือไม่
