# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจได้ว่า Skills หรือ Plugin ปลอดภัยพอที่จะติดตั้งหรือไม่ การตรวจสอบจะแสดงว่ารีลีสทำอะไร ขออำนาจใดบ้าง และมีสิ่งใดที่ควรให้ความสนใจเป็นพิเศษก่อนที่จะเข้าถึงไฟล์ บัญชี ข้อมูลประจำตัว โค้ด หรือบริการภายนอกหรือไม่

การตรวจสอบเป็นสัญญาณความปลอดภัยที่แข็งแรง แต่ไม่ใช่การรับประกันว่ารีลีสจะปราศจากความเสี่ยง ควรใช้วิจารณญาณเสมอก่อนให้สิทธิ์เข้าถึงข้อมูลที่ละเอียดอ่อน

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security), [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage) และ [การกลั่นกรองและความปลอดภัยของบัญชี](/clawhub/moderation)

## สิ่งที่ควรตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจสอบ:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- ข้อค้นพบใดๆ ที่ระบุไว้
- ข้อมูลประจำตัว สิทธิ์ หรือ Environment Variable ที่ต้องใช้
- เจ้าของ แหล่งที่มา เวอร์ชัน บันทึกการเปลี่ยนแปลง ยอดดาวน์โหลด ดาว และสัญญาณความน่าเชื่อถืออื่นๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและไว้วางใจเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกคุณว่าควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้ซึ่งมีความเสี่ยงสูงกว่าระดับต่ำ                                |
| `Review`    | อ่านข้อค้นพบก่อนติดตั้ง รีลีสอาจยังคงถูกต้องตามเจตนาได้ |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบประเด็นที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | ห้ามติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จสิ้น                                             |
| `Error`     | ไม่สามารถดำเนินการตรวจสอบให้เสร็จสมบูรณ์ได้                                         |

`Pass` ช่วยให้มั่นใจได้ แต่ไม่ได้แทนที่วิจารณญาณของคุณเอง สิ่งนี้สำคัญที่สุดสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล รันคำสั่ง อ่านไฟล์ หรือเข้าถึงระบบโปรดักชันได้

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบายขอบเขตผลกระทบ: รีลีสดูเหมือนมีอำนาจมากเพียงใดหากคุณใช้งานตามเจตนา

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบอำนาจที่ละเอียดอ่อนหรือผลกระทบต่อผู้ใช้น้อยมาก                          |
| `Medium`   | รีลีสมีอำนาจที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | รีลีสมีอำนาจที่มีผลกระทบสูง ข้อค้นพบร้ายแรง หรือสัญญาณที่เป็นอันตราย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคำถามต่างกัน:

- ระดับความเสี่ยงถามว่า: "ที่นี่มีอำนาจมากแค่ไหน?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอย่างไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น Skills สำหรับการเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยง `Medium` นั่นไม่ได้หมายความว่ามีเจตนาร้าย แต่หมายความว่า Skills ดูสอดคล้องกับวัตถุประสงค์ แต่สามารถดำเนินการด้วยอำนาจบัญชีที่มีนัยสำคัญได้

## ข้อค้นพบ

ข้อค้นพบอธิบายว่าทำไมผลการตรวจสอบจึงถูกแสดงขึ้น โดยปกติแต่ละข้อค้นพบจะมี:

- ความหมายของข้อค้นพบนั้น
- เหตุผลที่ถูกระบุ
- เนื้อหา Skills หรือ Plugin ที่เกี่ยวข้อง
- คำแนะนำ

ข้อค้นพบอาจมีป้ายกำกับเป็น `Info`, `Low`, `Medium`, `High` หรือ `Critical` ข้อค้นพบที่มีความรุนแรงสูงกว่าจะมีผลต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

ข้อค้นพบที่มีความมั่นใจต่ำจะถูกซ่อนจากสรุปการตรวจสอบสาธารณะ เพื่อให้หน้ายังคงเน้นหลักฐานที่เป็นประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบอาร์ติแฟกต์รีลีสที่ส่งเข้ามา รวมถึง:

- คำแนะนำของ Skills หรือ Metadata ของ Plugin
- Environment Variable และสิทธิ์ที่ประกาศไว้
- คำแนะนำการติดตั้งและ Metadata ของแพ็กเกจ
- ไฟล์ที่รวมมาและรายการไฟล์
- Metadata ด้านความเข้ากันได้และความสามารถ

คำถามหลักคือความสอดคล้อง: ชื่อ สรุป Metadata อำนาจที่ขอ และเนื้อหาจริงสอดคล้องกับสิ่งที่ผู้ใช้คาดหวังอย่างสมเหตุสมผลหรือไม่

พฤติกรรมที่มีอำนาจมากไม่ได้แย่โดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลประจำตัว คำสั่งในเครื่อง API ของผู้ให้บริการ หรือการติดตั้งแพ็กเกจ การตรวจสอบจะพิจารณาว่าอำนาจนั้นเป็นสิ่งที่คาดไว้ เปิดเผยไว้ และได้สัดส่วนหรือไม่

หน้าอาร์ติแฟกต์ลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบรวม:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็นข้อมูลโทรมาตรมัลแวร์ในชุดการตรวจสอบ VirusTotal เป็นมาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับชื่อเสียงของไฟล์และการสแกนมัลแวร์ และความร่วมมือของเราช่วยให้ ClawHub เพิ่มข้อมูลข่าวกรองด้านความปลอดภัยที่กว้างขึ้นในการตรวจสอบ Skills และ Plugin

VirusTotal มีประโยชน์เป็นพิเศษสำหรับอาร์ติแฟกต์ที่ทราบว่าเป็นอันตราย ผลตรวจจากเอนจิน และสัญญาณชื่อเสียงที่เสริมการตรวจสอบของ ClawHub ซึ่งเข้าใจบริบทของเอเจนต์ เมื่อมีจำนวนผลตรวจจากเอนจินของผู้ให้บริการ การตรวจสอบจะสรุปเป็นภาษาธรรมดา เช่น:

```text
62/62 vendors flagged this skill as clean.
```

หรือ:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

เมื่อ ClawHub ไม่มีข้อมูลโทรมาตรีจำนวนผู้ให้บริการให้สรุป การตรวจสอบจะระบุว่า:

```text
No VirusTotal findings
```

VirusTotal ยังคงเป็นข้อมูลโทรมาตรี ไม่ได้แทนที่การวิเคราะห์ความเสี่ยงของ ClawHub เองซึ่งเข้าใจอาร์ติแฟกต์

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัยของ ClawHub เอง ระบบจะตรวจสอบแต่ละรีลีสในฐานะอาร์ติแฟกต์ที่มุ่งให้เอเจนต์ใช้งาน: คำแนะนำ, Metadata, สิทธิ์ที่ประกาศ, ไฟล์, สัญญาณความสามารถ, สัญญาณการสแกนแบบสแตติก, ข้อค้นพบของ SkillSpector, โทรมาตรีของ VirusTotal และบริบทที่ผู้เผยแพร่ให้มา สัญญาณการสแกนแบบสแตติกเป็นบริบทภายในสำหรับการตรวจสอบนี้ ไม่ใช่ส่วนการตรวจสอบสาธารณะแบบแยกต่างหากหรือคำตัดสินที่บล็อกการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นกรอบมองความเสี่ยง เช่น Prompt Injection, การใช้เครื่องมือในทางที่ผิด, การเปิดเผยข้อมูลประจำตัว, การดำเนินการที่ไม่ปลอดภัย, การวางยาพิษหน่วยความจำหรือบริบท และอำนาจดำเนินการแทนผู้ใช้ที่มากเกินไป

ClawScan ไม่ถือว่าความสามารถที่ดูน่ากลัวเป็นสิ่งที่มีเจตนาร้ายโดยอัตโนมัติ ระบบจะถามว่าความสามารถนั้นถูกเปิดเผย สอดคล้องกับวัตถุประสงค์ และได้รับการสนับสนุนจากกรณีการใช้งานที่รีลีสระบุไว้หรือไม่
