# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจได้ว่าสกิลหรือ Plugin ปลอดภัยพอ
สำหรับการติดตั้งหรือไม่ การตรวจสอบจะแสดงว่ารุ่นเผยแพร่ทำอะไร ขออำนาจอะไรบ้าง และ
มีสิ่งใดที่ควรได้รับความสนใจเป็นพิเศษก่อนที่จะเข้าถึงไฟล์ บัญชี
ข้อมูลประจำตัว โค้ด หรือบริการภายนอกหรือไม่

การตรวจสอบเป็นสัญญาณด้านความปลอดภัยที่หนักแน่น แต่ไม่ได้รับประกันว่ารุ่นเผยแพร่นั้น
ปราศจากความเสี่ยง โปรดใช้วิจารณญาณเสมอก่อนให้สิทธิ์เข้าถึงข้อมูลอ่อนไหว

ดูเพิ่มเติมที่ [ความปลอดภัย](/th/clawhub/security), [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage),
และ [การกลั่นกรองและความปลอดภัยของบัญชี](/th/clawhub/moderation)

## สิ่งที่ควรตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจสอบ:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- รายการข้อค้นพบใดๆ
- ข้อมูลประจำตัว สิทธิ์ หรือตัวแปรสภาพแวดล้อมที่จำเป็น
- เจ้าของ แหล่งที่มา เวอร์ชัน บันทึกการเปลี่ยนแปลง ยอดดาวน์โหลด ดาว และสัญญาณความน่าเชื่อถืออื่นๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและไว้วางใจเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกว่าคุณควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้เหนือระดับความเสี่ยงต่ำ                                |
| `Review`    | อ่านข้อค้นพบก่อนติดตั้ง รุ่นเผยแพร่อาจยังคงถูกต้องตามวัตถุประสงค์ |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบข้อกังวลที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | อย่าติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จสิ้น                                             |
| `Error`     | ไม่สามารถดำเนินการตรวจสอบให้เสร็จสมบูรณ์ได้                                         |

`Pass` ทำให้อุ่นใจได้ แต่ไม่สามารถแทนที่วิจารณญาณของคุณเองได้ สิ่งนี้สำคัญ
ที่สุดสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล รันคำสั่ง อ่านไฟล์ หรือ
เข้าถึงระบบโปรดักชันได้

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบายขอบเขตผลกระทบ: รุ่นเผยแพร่ดูเหมือนจะมีอำนาจมากเพียงใดหาก
คุณใช้งานตามที่ตั้งใจไว้

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบอำนาจอ่อนไหวหรือผลกระทบต่อผู้ใช้น้อย                          |
| `Medium`   | รุ่นเผยแพร่มีอำนาจที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | รุ่นเผยแพร่มีอำนาจที่มีผลกระทบสูง ข้อค้นพบร้ายแรง หรือสัญญาณอันตราย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคำถามคนละข้อ:

- ระดับความเสี่ยงถามว่า: "ตรงนี้มีอำนาจมากเพียงใด?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอย่างไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น สกิลสำหรับเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยง `Medium` นั่นไม่ได้
หมายความว่าสกิลนั้นเป็นอันตราย แต่หมายความว่าสกิลดูสอดคล้องกับวัตถุประสงค์ แต่สามารถ
ดำเนินการด้วยอำนาจบัญชีที่มีนัยสำคัญได้

## ข้อค้นพบ

ข้อค้นพบอธิบายว่าทำไมจึงแสดงผลการตรวจสอบนั้น ข้อค้นพบแต่ละรายการมักประกอบด้วย:

- ความหมายของข้อค้นพบ
- เหตุผลที่ถูกทำเครื่องหมาย
- เนื้อหาสกิลหรือ Plugin ที่เกี่ยวข้อง
- คำแนะนำ

ข้อค้นพบอาจติดป้ายเป็น `Info`, `Low`, `Medium`, `High` หรือ `Critical` ข้อค้นพบที่มี
ความรุนแรงสูงกว่าจะมีส่วนต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

ข้อค้นพบที่มีความเชื่อมั่นต่ำจะถูกซ่อนจากสรุปการตรวจสอบสาธารณะ เพื่อให้หน้าเว็บ
ยังคงเน้นหลักฐานที่มีประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบอาร์ติแฟกต์ของรุ่นเผยแพร่ที่ส่งเข้ามา รวมถึง:

- คำสั่งของสกิลหรือเมตาดาต้าของ Plugin
- ตัวแปรสภาพแวดล้อมและสิทธิ์ที่ประกาศไว้
- คำสั่งติดตั้งและเมตาดาต้าของแพ็กเกจ
- ไฟล์ที่รวมอยู่และรายการไฟล์
- เมตาดาต้าด้านความเข้ากันได้และความสามารถ

คำถามหลักคือความสอดคล้อง: ชื่อ สรุป เมตาดาต้า อำนาจที่ร้องขอ
และเนื้อหาจริงสอดคล้องกับสิ่งที่ผู้ใช้ควรคาดหวังอย่างสมเหตุสมผลหรือไม่?

พฤติกรรมที่มีอำนาจมากไม่ได้แย่โดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลประจำตัว
คำสั่งภายในเครื่อง API ของผู้ให้บริการ หรือการติดตั้งแพ็กเกจ การตรวจสอบจะตรวจว่าพลังนั้น
เป็นสิ่งที่คาดหมายได้ เปิดเผยไว้ และได้สัดส่วนหรือไม่

หน้าอาร์ติแฟกต์ลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบประกอบด้วย:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็นเทเลเมทรีมัลแวร์ในสแต็กการตรวจสอบ VirusTotal เป็น
มาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับชื่อเสียงของไฟล์และการสแกนมัลแวร์ และ
ความร่วมมือของเราช่วยให้ ClawHub เพิ่มข่าวกรองด้านความปลอดภัยที่กว้างขึ้นให้กับการรีวิวสกิลและ Plugin

VirusTotal มีประโยชน์เป็นพิเศษสำหรับอาร์ติแฟกต์อันตรายที่รู้จัก การตรวจพบจากเอนจิน และ
สัญญาณชื่อเสียงที่เสริมการรีวิวของ ClawHub ซึ่งเข้าใจบริบทของเอเจนต์ เมื่อมีจำนวนการตรวจพบจาก
เอนจินของผู้ให้บริการ การตรวจสอบจะสรุปเป็นภาษาธรรมดา เช่น:

```text
62/62 vendors flagged this skill as clean.
```

หรือ:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

เมื่อ ClawHub ไม่มีเทเลเมทรีจำนวนผู้ให้บริการให้สรุป การตรวจสอบจะระบุว่า:

```text
No VirusTotal findings
```

VirusTotal ยังคงเป็นเทเลเมทรี ไม่ได้แทนที่การวิเคราะห์ความเสี่ยงของ ClawHub เอง
ที่เข้าใจอาร์ติแฟกต์

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัยของ
ClawHub เอง ระบบจะรีวิวแต่ละรุ่นเผยแพร่ในฐานะอาร์ติแฟกต์ที่เอเจนต์จะใช้งาน: คำสั่ง
เมตาดาต้า สิทธิ์ที่ประกาศไว้ ไฟล์ สัญญาณความสามารถ สัญญาณการสแกนแบบสแตติก
ข้อค้นพบจาก SkillSpector เทเลเมทรีจาก VirusTotal และบริบทที่ผู้เผยแพร่ให้ไว้
สัญญาณการสแกนแบบสแตติกเป็นบริบทภายในสำหรับการรีวิวนี้ ไม่ใช่
ส่วนการตรวจสอบสาธารณะแบบแยกเดี่ยวหรือคำตัดสินที่บล็อกการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นกรอบมองความเสี่ยง เช่น การแทรกพรอมป์ การใช้เครื่องมือในทางที่ผิด การเปิดเผยข้อมูลประจำตัว
การดำเนินการที่ไม่ปลอดภัย การปนเปื้อนหน่วยความจำหรือบริบท และการให้อำนาจดำเนินการมากเกินไป

ClawScan ไม่ถือว่าความสามารถที่ดูน่ากลัวเป็นอันตรายโดยอัตโนมัติ
ระบบจะถามว่าความสามารถนั้นถูกเปิดเผย สอดคล้องกับวัตถุประสงค์ และได้รับการสนับสนุนโดย
กรณีการใช้งานที่รุ่นเผยแพร่ระบุไว้หรือไม่
