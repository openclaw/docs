# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจได้ว่าทักษะหรือ Plugin ปลอดภัยพอที่จะติดตั้งหรือไม่ การตรวจสอบจะแสดงว่ารุ่นเผยแพร่ทำอะไร ขอสิทธิ์อำนาจใดบ้าง และมีสิ่งใดที่ควรพิจารณาเป็นพิเศษก่อนที่มันจะเข้าถึงไฟล์ บัญชี ข้อมูลลับ โค้ด หรือบริการภายนอกหรือไม่

การตรวจสอบเป็นสัญญาณด้านความปลอดภัยที่หนักแน่น แต่ไม่ได้รับประกันว่ารุ่นเผยแพร่จะปราศจากความเสี่ยงเสมอไป ใช้วิจารณญาณเสมอก่อนให้สิทธิ์เข้าถึงข้อมูลหรือระบบที่ละเอียดอ่อน

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security), [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage) และ [การกลั่นกรองและความปลอดภัยของบัญชี](/clawhub/moderation)

## สิ่งที่ต้องตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจสอบ:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- ผลการตรวจพบใดๆ ที่ระบุไว้
- ข้อมูลลับ สิทธิ์ หรือ environment variables ที่จำเป็น
- เจ้าของ แหล่งที่มา เวอร์ชัน changelog ยอดดาวน์โหลด ดาว และสัญญาณความน่าเชื่อถืออื่นๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกคุณว่าควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้ซึ่งมีความเสี่ยงสูงกว่าระดับต่ำ                                |
| `Review`    | อ่านผลการตรวจพบก่อนติดตั้ง รุ่นเผยแพร่อาจยังเป็นของถูกต้องตามวัตถุประสงค์ |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบข้อกังวลที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | อย่าติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จสิ้น                                             |
| `Error`     | ไม่สามารถทำการตรวจสอบให้เสร็จสมบูรณ์ได้                                         |

`Pass` ช่วยให้มั่นใจได้ แต่ไม่แทนที่วิจารณญาณของคุณเอง สิ่งนี้สำคัญที่สุดสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล รันคำสั่ง อ่านไฟล์ หรือเข้าถึงระบบ production

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบายขอบเขตผลกระทบ: รุ่นเผยแพร่ดูเหมือนจะมีอำนาจมากเพียงใดหากคุณใช้งานตามที่ตั้งใจไว้

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบสิทธิ์อำนาจที่ละเอียดอ่อนหรือผลกระทบต่อผู้ใช้น้อย                          |
| `Medium`   | รุ่นเผยแพร่มีสิทธิ์อำนาจที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | รุ่นเผยแพร่มีสิทธิ์อำนาจที่มีผลกระทบสูง ผลการตรวจพบที่รุนแรง หรือสัญญาณอันตราย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคำถามที่ต่างกัน:

- ระดับความเสี่ยงถามว่า: "ที่นี่มีอำนาจมากเพียงใด?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอะไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น ทักษะสำหรับการเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยง `Medium` นั่นไม่ได้หมายความว่ามันเป็นอันตราย แต่หมายความว่าทักษะดูสอดคล้องกับวัตถุประสงค์ แต่สามารถดำเนินการด้วยสิทธิ์อำนาจของบัญชีที่มีนัยสำคัญได้

## ผลการตรวจพบ

ผลการตรวจพบอธิบายว่าทำไมจึงแสดงผลการตรวจสอบนั้น ผลการตรวจพบแต่ละรายการมักมี:

- ความหมายของรายการนั้น
- เหตุผลที่ถูกตั้งธง
- เนื้อหาทักษะหรือ Plugin ที่เกี่ยวข้อง
- คำแนะนำ

ผลการตรวจพบอาจมีป้ายกำกับเป็น `Info`, `Low`, `Medium`, `High` หรือ `Critical` ผลการตรวจพบที่มีความรุนแรงสูงกว่าจะส่งผลต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

ผลการตรวจพบที่มีความเชื่อมั่นต่ำจะถูกซ่อนจากสรุปการตรวจสอบสาธารณะ เพื่อให้หน้ายังคงมุ่งเน้นหลักฐานที่เป็นประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบอาร์ติแฟกต์ของรุ่นเผยแพร่ที่ส่งมา รวมถึง:

- คำสั่งของทักษะหรือ metadata ของ Plugin
- environment variables และสิทธิ์ที่ประกาศไว้
- คำสั่งติดตั้งและ metadata ของแพ็กเกจ
- ไฟล์และ manifests ของไฟล์ที่รวมมา
- metadata ด้าน compatibility และ capability

คำถามหลักคือความสอดคล้อง: ชื่อ สรุป metadata สิทธิ์อำนาจที่ร้องขอ และเนื้อหาจริงสอดคล้องกับสิ่งที่ผู้ใช้คาดหวังได้อย่างสมเหตุสมผลหรือไม่?

พฤติกรรมที่ทรงพลังไม่ได้แปลว่าไม่ดีโดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลลับ คำสั่งในเครื่อง provider APIs หรือการติดตั้งแพ็กเกจ การตรวจสอบจะตรวจว่าพลังนั้นเป็นสิ่งที่คาดหมายได้ เปิดเผยไว้ และได้สัดส่วนหรือไม่

หน้าอาร์ติแฟกต์ลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบรวม:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็น telemetry ด้านมัลแวร์ในชุดการตรวจสอบ VirusTotal เป็นมาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับชื่อเสียงของไฟล์และการสแกนมัลแวร์ และความร่วมมือของเราช่วยให้ ClawHub เพิ่มข่าวกรองด้านความปลอดภัยที่กว้างขึ้นในการตรวจทานทักษะและ Plugin

VirusTotal มีประโยชน์เป็นพิเศษสำหรับอาร์ติแฟกต์ที่ทราบว่าเป็นอันตราย การพบของเอนจิน และสัญญาณชื่อเสียงที่เสริมการตรวจทานของ ClawHub ที่เข้าใจบริบทของเอเจนต์ เมื่อมีจำนวนจากเอนจินของผู้ให้บริการ การตรวจสอบจะสรุปเป็นภาษาธรรมดา เช่น:

```text
62/62 vendors flagged this skill as clean.
```

หรือ:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

เมื่อ ClawHub ไม่มี telemetry จำนวนผู้ให้บริการให้สรุป การตรวจสอบจะระบุว่า:

```text
No VirusTotal findings
```

VirusTotal ยังคงเป็น telemetry ไม่ได้แทนที่การวิเคราะห์ความเสี่ยงของ ClawHub เองที่เข้าใจอาร์ติแฟกต์

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัยของ ClawHub เอง ระบบจะตรวจทานแต่ละรุ่นเผยแพร่ในฐานะอาร์ติแฟกต์ที่เอเจนต์จะใช้งาน: คำสั่ง, metadata, สิทธิ์ที่ประกาศ, ไฟล์, สัญญาณ capability, สัญญาณจากการสแกนแบบ static, ผลการตรวจพบของ SkillSpector, telemetry ของ VirusTotal และบริบทที่ผู้เผยแพร่ให้มา สัญญาณจากการสแกนแบบ static เป็นบริบทภายในสำหรับการตรวจทานนี้ ไม่ใช่ส่วนการตรวจสอบสาธารณะแบบสแตนด์อโลนหรือ verdict ที่บล็อกการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นกรอบมองความเสี่ยง เช่น prompt injection, การใช้เครื่องมือในทางที่ผิด, การเปิดเผยข้อมูลลับ, การดำเนินการที่ไม่ปลอดภัย, การปนเปื้อน memory หรือ context และการมี agency มากเกินไป

ClawScan ไม่ถือว่า capability ที่ดูน่ากลัวเป็นอันตรายโดยอัตโนมัติ แต่จะถามว่า capability นั้นถูกเปิดเผย สอดคล้องกับวัตถุประสงค์ และรองรับโดยกรณีใช้งานที่รุ่นเผยแพร่ระบุไว้หรือไม่
