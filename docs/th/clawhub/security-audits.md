# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจได้ว่าทักษะหรือ Plugin ปลอดภัยพอ
ที่จะติดตั้งหรือไม่ การตรวจสอบจะแสดงว่ารุ่นเผยแพร่ทำอะไร ขออำนาจอะไรบ้าง และ
มีสิ่งใดที่ควรให้ความสนใจเป็นพิเศษก่อนที่จะเข้าถึงไฟล์ บัญชี
ข้อมูลประจำตัว โค้ด หรือบริการภายนอกหรือไม่

การตรวจสอบเป็นสัญญาณด้านความปลอดภัยที่แข็งแรง แต่ไม่ได้รับประกันว่ารุ่นเผยแพร่จะ
ปราศจากความเสี่ยงเสมอไป โปรดใช้วิจารณญาณก่อนให้สิทธิ์การเข้าถึงที่ละเอียดอ่อนเสมอ

ดูเพิ่มเติมที่ [ความปลอดภัย](/th/clawhub/security), [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage),
และ [การกลั่นกรองและความปลอดภัยของบัญชี](/th/clawhub/moderation)

## สิ่งที่ควรตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจทาน:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- ผลการตรวจพบที่แสดงไว้
- ข้อมูลประจำตัว สิทธิ์ หรือ Environment Variable ที่ต้องใช้
- เจ้าของ แหล่งที่มา เวอร์ชัน บันทึกการเปลี่ยนแปลง ยอดดาวน์โหลด ดาว และสัญญาณความน่าเชื่อถืออื่นๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกคุณว่าควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้ซึ่งมีความเสี่ยงสูงกว่าระดับต่ำ                                |
| `Review`    | อ่านผลการตรวจพบก่อนติดตั้ง รุ่นเผยแพร่อาจยังคงถูกต้องตามเจตนา |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบข้อกังวลที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | ห้ามติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จสิ้น                                             |
| `Error`     | ไม่สามารถทำการตรวจสอบให้เสร็จสมบูรณ์ได้                                         |

`Pass` ช่วยให้อุ่นใจได้ แต่ไม่ได้แทนที่วิจารณญาณของคุณเอง เรื่องนี้สำคัญ
ที่สุดสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล เรียกใช้คำสั่ง อ่านไฟล์ หรือ
เข้าถึงระบบโปรดักชันได้

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบายขอบเขตผลกระทบ: รุ่นเผยแพร่ดูเหมือนมีอำนาจมากเพียงใดหาก
คุณใช้งานตามเจตนา

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบอำนาจที่ละเอียดอ่อนหรือผลกระทบต่อผู้ใช้น้อยมาก                          |
| `Medium`   | รุ่นเผยแพร่มีอำนาจที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | รุ่นเผยแพร่มีอำนาจที่มีผลกระทบสูง ผลการตรวจพบที่ร้ายแรง หรือสัญญาณมุ่งร้าย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคำถามต่างกัน:

- ระดับความเสี่ยงถามว่า: "ที่นี่มีอำนาจมากแค่ไหน?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอะไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น ทักษะสำหรับการเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยง `Medium` ซึ่ง
ไม่ได้หมายความว่ามุ่งร้าย แต่หมายความว่าทักษะนี้ดูสอดคล้องกับวัตถุประสงค์ แต่สามารถ
กระทำการด้วยอำนาจของบัญชีที่มีนัยสำคัญได้

## ผลการตรวจพบ

ผลการตรวจพบอธิบายว่าเหตุใดจึงแสดงผลการตรวจสอบนั้น โดยแต่ละรายการมักมี:

- ความหมายของรายการนั้น
- เหตุผลที่ถูกแจ้งเตือน
- เนื้อหาทักษะหรือ Plugin ที่เกี่ยวข้อง
- คำแนะนำ

ผลการตรวจพบอาจถูกติดป้ายเป็น `Info`, `Low`, `Medium`, `High` หรือ `Critical` ผลการตรวจพบที่มี
ความรุนแรงสูงกว่าจะมีผลต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

ผลการตรวจพบที่มีความเชื่อมั่นต่ำจะถูกซ่อนจากสรุปการตรวจสอบสาธารณะ เพื่อให้หน้า
ยังคงมุ่งเน้นหลักฐานที่เป็นประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบอาร์ติแฟกต์ของรุ่นเผยแพร่ที่ส่งเข้ามา รวมถึง:

- คำสั่งของทักษะหรือเมตาดาตาของ Plugin
- Environment Variable และสิทธิ์ที่ประกาศไว้
- คำแนะนำการติดตั้งและเมตาดาตาของแพ็กเกจ
- ไฟล์ที่รวมมาและรายการไฟล์
- เมตาดาตาด้านความเข้ากันได้และความสามารถ

คำถามหลักคือความสอดคล้อง: ชื่อ สรุป เมตาดาตา อำนาจที่ขอ
และเนื้อหาจริงสอดคล้องกับสิ่งที่ผู้ใช้น่าจะคาดหวังอย่างสมเหตุสมผลหรือไม่?

พฤติกรรมที่ทรงพลังไม่ได้แย่โดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลประจำตัว
คำสั่งในเครื่อง API ของผู้ให้บริการ หรือการติดตั้งแพ็กเกจ การตรวจสอบจะดูว่าอำนาจนั้น
เป็นสิ่งที่คาดหมายได้ มีการเปิดเผย และได้สัดส่วนหรือไม่

หน้าอาร์ติแฟกต์ลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบรวม:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็นโทรมาตรมัลแวร์ในชุดการตรวจสอบ VirusTotal เป็น
มาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับชื่อเสียงของไฟล์และการสแกนมัลแวร์ และ
ความร่วมมือของเราช่วยให้ ClawHub เพิ่มข่าวกรองด้านความปลอดภัยที่กว้างขึ้นในการตรวจทานทักษะและ Plugin

VirusTotal มีประโยชน์เป็นพิเศษสำหรับอาร์ติแฟกต์มุ่งร้ายที่รู้จัก การตรวจพบจากเอนจิน และ
สัญญาณชื่อเสียงที่เสริมการตรวจทานแบบเข้าใจเอเจนต์ของ ClawHub เมื่อมีจำนวนจาก
เอนจินผู้ให้บริการ การตรวจสอบจะสรุปเป็นภาษาธรรมดา เช่น:

```text
62/62 vendors flagged this skill as clean.
```

หรือ:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

เมื่อ ClawHub ไม่มีโทรมาตรจำนวนผู้ให้บริการให้สรุป การตรวจสอบจะระบุว่า:

```text
No VirusTotal findings
```

VirusTotal ยังคงเป็นโทรมาตร ไม่ได้แทนที่การวิเคราะห์ความเสี่ยงแบบเข้าใจอาร์ติแฟกต์
ของ ClawHub เอง

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัย
ของ ClawHub เอง ระบบจะตรวจทานแต่ละรุ่นเผยแพร่ในฐานะอาร์ติแฟกต์สำหรับเอเจนต์: คำสั่ง
เมตาดาตา สิทธิ์ที่ประกาศไว้ ไฟล์ สัญญาณความสามารถ สัญญาณการสแกนแบบสถิต
ผลการตรวจพบจาก SkillSpector โทรมาตรจาก VirusTotal และบริบทที่ผู้เผยแพร่ให้มา
สัญญาณการสแกนแบบสถิตเป็นบริบทภายในสำหรับการตรวจทานนี้ ไม่ใช่
ส่วนการตรวจสอบสาธารณะแบบแยกต่างหากหรือคำตัดสินที่บล็อกการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นกรอบมองความเสี่ยง เช่น prompt injection การใช้เครื่องมือผิดวัตถุประสงค์ การเปิดเผยข้อมูลประจำตัว
การดำเนินการที่ไม่ปลอดภัย การปนเปื้อนหน่วยความจำหรือบริบท และการให้อำนาจเอเจนต์มากเกินไป

ClawScan ไม่ถือว่าความสามารถที่ดูน่ากลัวเป็นสิ่งมุ่งร้ายโดยอัตโนมัติ
ระบบจะถามว่าความสามารถนั้นได้รับการเปิดเผย สอดคล้องกับวัตถุประสงค์ และได้รับการสนับสนุนโดย
กรณีการใช้งานที่รุ่นเผยแพร่ระบุไว้หรือไม่
