# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจว่า Skills หรือ Plugin ปลอดภัยเพียงพอ
ที่จะติดตั้งหรือไม่ โดยจะแสดงว่ารีลีสทำอะไร ร้องขออำนาจใด และ
มีสิ่งใดที่ควรตรวจสอบเป็นพิเศษก่อนที่จะเข้าถึงไฟล์ บัญชี
ข้อมูลประจำตัว โค้ด หรือบริการภายนอก

การตรวจสอบเป็นสัญญาณด้านความปลอดภัยที่มีน้ำหนัก แต่ไม่ได้รับประกันว่ารีลีส
จะปราศจากความเสี่ยง โปรดใช้วิจารณญาณเสมอก่อนอนุญาตให้เข้าถึงข้อมูลหรือระบบที่ละเอียดอ่อน

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security), [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)
และ [การกำกับดูแลและความปลอดภัยของบัญชี](/clawhub/moderation)

## สิ่งที่ควรตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจสอบ:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- ข้อค้นพบที่ระบุไว้
- ข้อมูลประจำตัว สิทธิ์ หรือตัวแปรสภาพแวดล้อมที่จำเป็น
- เจ้าของ แหล่งที่มา เวอร์ชัน บันทึกการเปลี่ยนแปลง จำนวนดาวน์โหลด ดาว และสัญญาณความน่าเชื่อถืออื่น ๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกว่าคุณควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้ซึ่งมีความเสี่ยงสูงกว่าระดับต่ำ                                |
| `Review`    | อ่านข้อค้นพบก่อนติดตั้ง รีลีสอาจยังคงถูกต้องตามวัตถุประสงค์ |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบข้อกังวลที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | ห้ามติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จสิ้น                                             |
| `Error`     | ไม่สามารถดำเนินการตรวจสอบให้เสร็จสมบูรณ์ได้                                         |

`Pass` ช่วยเพิ่มความมั่นใจ แต่ไม่สามารถทดแทนวิจารณญาณของคุณเองได้ เรื่องนี้สำคัญ
ที่สุดสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล เรียกใช้คำสั่ง อ่านไฟล์ หรือ
เข้าถึงระบบที่ใช้งานจริง

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบายขอบเขตผลกระทบ: รีลีสดูเหมือนจะมีอำนาจมากเพียงใดหาก
คุณใช้งานตามวัตถุประสงค์

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบอำนาจที่ละเอียดอ่อนหรือผลกระทบต่อผู้ใช้น้อย                          |
| `Medium`   | รีลีสมีอำนาจที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | รีลีสมีอำนาจที่ส่งผลกระทบสูง ข้อค้นพบที่ร้ายแรง หรือสัญญาณประสงค์ร้าย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคำถามที่ต่างกัน:

- ระดับความเสี่ยงถามว่า: "มีอำนาจมากเพียงใด?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอย่างไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น Skills สำหรับเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยงระดับ `Medium` ซึ่ง
ไม่ได้หมายความว่าเป็นสิ่งประสงค์ร้าย แต่หมายความว่า Skills ดูสอดคล้องกับวัตถุประสงค์ ทว่า
สามารถดำเนินการโดยใช้อำนาจของบัญชีในระดับที่มีนัยสำคัญ

## ข้อค้นพบ

ข้อค้นพบอธิบายเหตุผลที่แสดงผลการตรวจสอบ โดยทั่วไปข้อค้นพบแต่ละรายการประกอบด้วย:

- ความหมายของข้อค้นพบ
- เหตุผลที่ถูกตั้งค่าสถานะ
- เนื้อหา Skills หรือ Plugin ที่เกี่ยวข้อง
- คำแนะนำ

ข้อค้นพบอาจมีป้ายกำกับ `Info`, `Low`, `Medium`, `High` หรือ `Critical` ข้อค้นพบที่มี
ความรุนแรงสูงกว่าจะส่งผลต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

ข้อค้นพบที่มีความเชื่อมั่นต่ำจะไม่แสดงในสรุปการตรวจสอบสาธารณะ เพื่อให้หน้า
มุ่งเน้นเฉพาะหลักฐานที่เป็นประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบอาร์ติแฟกต์ของรีลีสที่ส่งเข้ามา ซึ่งรวมถึง:

- คำสั่งของ Skills หรือข้อมูลเมตาของ Plugin
- ตัวแปรสภาพแวดล้อมและสิทธิ์ที่ประกาศไว้
- คำแนะนำการติดตั้งและข้อมูลเมตาของแพ็กเกจ
- ไฟล์และรายการไฟล์ที่รวมอยู่
- ข้อมูลเมตาด้านความเข้ากันได้และความสามารถ

คำถามหลักคือความสอดคล้องกัน: ชื่อ ข้อมูลสรุป ข้อมูลเมตา อำนาจที่ร้องขอ
และเนื้อหาจริงตรงกับสิ่งที่ผู้ใช้คาดหวังอย่างสมเหตุสมผลหรือไม่?

พฤติกรรมที่มีอำนาจสูงไม่ได้เป็นสิ่งไม่ดีโดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลประจำตัว
คำสั่งภายในเครื่อง API ของผู้ให้บริการ หรือการติดตั้งแพ็กเกจ การตรวจสอบจะพิจารณาว่า
อำนาจดังกล่าวเป็นสิ่งที่คาดหมาย มีการเปิดเผย และได้สัดส่วนหรือไม่

หน้าอาร์ติแฟกต์จะลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบประกอบด้วย:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็นข้อมูลเทเลเมทรีด้านมัลแวร์ในชุดระบบการตรวจสอบ VirusTotal เป็น
มาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับการประเมินชื่อเสียงของไฟล์และการสแกนมัลแวร์ และ
ความร่วมมือของเราช่วยให้ ClawHub เพิ่มข้อมูลเชิงลึกด้านความปลอดภัยที่ครอบคลุมยิ่งขึ้นในการตรวจสอบ Skills และ Plugin

VirusTotal มีประโยชน์เป็นพิเศษสำหรับอาร์ติแฟกต์ประสงค์ร้ายที่เป็นที่รู้จัก การตรวจพบจากเอนจิน และ
สัญญาณด้านชื่อเสียงที่ช่วยเสริมการตรวจสอบของ ClawHub ซึ่งคำนึงถึงบริบทของเอเจนต์ เมื่อมี
จำนวนผลจากเอนจินของผู้ให้บริการ การตรวจสอบจะสรุปด้วยภาษาที่เข้าใจง่าย เช่น:

```text
ผู้ให้บริการ 62/62 รายระบุว่า Skills นี้ปลอดภัย
```

หรือ:

```text
ผู้ให้บริการ 2/64 รายระบุว่า Skills นี้เป็นอันตราย 1/64 รายระบุว่าน่าสงสัย และ 61/64 รายระบุว่าปลอดภัย
```

เมื่อ ClawHub ไม่มีข้อมูลเทเลเมทรีจำนวนผู้ให้บริการให้สรุป การตรวจสอบจะแสดงว่า:

```text
ไม่มีข้อค้นพบจาก VirusTotal
```

VirusTotal ยังคงเป็นข้อมูลเทเลเมทรี และไม่สามารถทดแทนการวิเคราะห์ความเสี่ยงที่คำนึงถึง
อาร์ติแฟกต์ของ ClawHub เองได้

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัย
ของ ClawHub เอง ระบบจะตรวจสอบแต่ละรีลีสในฐานะอาร์ติแฟกต์สำหรับเอเจนต์ ได้แก่ คำสั่ง
ข้อมูลเมตา สิทธิ์ที่ประกาศไว้ ไฟล์ สัญญาณความสามารถ สัญญาณจากการสแกนแบบสถิต
ข้อค้นพบจาก SkillSpector ข้อมูลเทเลเมทรีจาก VirusTotal และบริบทที่ผู้เผยแพร่ให้ไว้
สัญญาณจากการสแกนแบบสถิตเป็นบริบทภายในสำหรับการตรวจสอบนี้ ไม่ใช่
ส่วนการตรวจสอบสาธารณะแบบแยกต่างหากหรือคำตัดสินที่ปิดกั้นการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[10 อันดับความเสี่ยงของ Skills สำหรับเอเจนต์จาก OWASP](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นกรอบในการพิจารณาความเสี่ยง เช่น การแทรกคำสั่งในพรอมต์ การใช้เครื่องมือในทางที่ผิด การเปิดเผยข้อมูลประจำตัว
การดำเนินการที่ไม่ปลอดภัย การวางยาพิษหน่วยความจำหรือบริบท และการให้อำนาจดำเนินการมากเกินไป

ClawScan ไม่ถือว่าความสามารถที่ดูน่ากลัวเป็นสิ่งประสงค์ร้ายโดยอัตโนมัติ
แต่จะพิจารณาว่าความสามารถดังกล่าวได้รับการเปิดเผย สอดคล้องกับวัตถุประสงค์ และรองรับโดย
กรณีการใช้งานที่รีลีสระบุไว้หรือไม่
