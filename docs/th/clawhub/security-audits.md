# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจได้ว่า Skills หรือ Plugin ปลอดภัยเพียงพอ
สำหรับติดตั้งหรือไม่ การตรวจสอบจะแสดงว่ารุ่นเผยแพร่ทำอะไร ขอสิทธิ์อะไรบ้าง และ
มีสิ่งใดที่ควรให้ความสนใจเป็นพิเศษก่อนที่จะเข้าถึงไฟล์ บัญชี
ข้อมูลลับ โค้ด หรือบริการภายนอกหรือไม่

การตรวจสอบเป็นสัญญาณด้านความปลอดภัยที่หนักแน่น แต่ไม่ได้รับประกันว่ารุ่นเผยแพร่จะ
ปราศจากความเสี่ยง โปรดใช้วิจารณญาณเสมอก่อนให้สิทธิ์เข้าถึงที่ละเอียดอ่อน

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security), [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage),
และ [การดูแลเนื้อหาและความปลอดภัยของบัญชี](/clawhub/moderation)

## สิ่งที่ควรตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจสอบ:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- ข้อค้นพบที่ระบุไว้
- ข้อมูลลับ สิทธิ์ หรือ environment variables ที่จำเป็น
- เจ้าของ แหล่งที่มา เวอร์ชัน changelog ยอดดาวน์โหลด ดาว และสัญญาณความน่าเชื่อถืออื่นๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกคุณว่าควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้ซึ่งมีความเสี่ยงสูงกว่าระดับต่ำ                                |
| `Review`    | อ่านข้อค้นพบก่อนติดตั้ง รุ่นเผยแพร่อาจยังเป็นของจริงและใช้งานได้อย่างถูกต้อง |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบข้อกังวลที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | ห้ามติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จสิ้น                                             |
| `Error`     | ไม่สามารถดำเนินการตรวจสอบให้เสร็จสมบูรณ์ได้                                         |

`Pass` ช่วยให้อุ่นใจได้ แต่ไม่ได้แทนที่วิจารณญาณของคุณเอง เรื่องนี้สำคัญ
ที่สุดสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล รันคำสั่ง อ่านไฟล์ หรือ
เข้าถึงระบบ production

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบายขอบเขตผลกระทบ: รุ่นเผยแพร่ดูเหมือนมีอำนาจมากแค่ไหนหาก
คุณใช้งานตามวัตถุประสงค์

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบสิทธิ์ที่ละเอียดอ่อนหรือผลกระทบต่อผู้ใช้น้อย                          |
| `Medium`   | รุ่นเผยแพร่มีสิทธิ์ที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | รุ่นเผยแพร่มีสิทธิ์ที่มีผลกระทบสูง ข้อค้นพบร้ายแรง หรือสัญญาณที่เป็นอันตราย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคำถามคนละข้อ:

- ระดับความเสี่ยงถามว่า: "ตรงนี้มีอำนาจมากแค่ไหน?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอย่างไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น Skills สำหรับเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยง `Medium`
ไม่ได้หมายความว่า Skills นั้นเป็นอันตราย แต่หมายความว่า Skills ดูสอดคล้องกับวัตถุประสงค์
แต่สามารถดำเนินการด้วยสิทธิ์บัญชีที่มีนัยสำคัญ

## ข้อค้นพบ

ข้อค้นพบอธิบายว่าทำไมจึงแสดงผลการตรวจสอบนั้น โดยปกติแต่ละข้อค้นพบจะรวมถึง:

- หมายความว่าอะไร
- เหตุใดจึงถูกแจ้งเตือน
- เนื้อหา Skills หรือ Plugin ที่เกี่ยวข้อง
- คำแนะนำ

ข้อค้นพบอาจมีป้ายกำกับ `Info`, `Low`, `Medium`, `High` หรือ `Critical` ข้อค้นพบที่มี
ความรุนแรงสูงกว่าจะส่งผลต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

ข้อค้นพบที่มีความมั่นใจต่ำจะถูกซ่อนจากสรุปการตรวจสอบสาธารณะ เพื่อให้หน้า
มุ่งเน้นหลักฐานที่เป็นประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบ artifact ของรุ่นเผยแพร่ที่ส่งเข้ามา รวมถึง:

- คำสั่งของ Skills หรือข้อมูลเมตาของ Plugin
- environment variables และสิทธิ์ที่ประกาศไว้
- คำแนะนำการติดตั้งและข้อมูลเมตาของแพ็กเกจ
- ไฟล์ที่รวมมาและ file manifests
- ข้อมูลเมตาด้านความเข้ากันได้และความสามารถ

คำถามหลักคือความสอดคล้อง: ชื่อ สรุป ข้อมูลเมตา สิทธิ์ที่ร้องขอ
และเนื้อหาจริงตรงกับสิ่งที่ผู้ใช้ควรคาดหวังอย่างสมเหตุสมผลหรือไม่

พฤติกรรมที่มีอำนาจสูงไม่ได้แย่โดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลลับ
คำสั่งในเครื่อง API ของผู้ให้บริการ หรือการติดตั้งแพ็กเกจ การตรวจสอบจะตรวจว่าพลังนั้น
เป็นสิ่งที่คาดหมายได้ เปิดเผยไว้ และได้สัดส่วนหรือไม่

หน้า artifact ลิงก์ไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบรวม:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็น telemetry ด้านมัลแวร์ในชุดการตรวจสอบ VirusTotal เป็น
มาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับชื่อเสียงของไฟล์และการสแกนมัลแวร์ และ
ความร่วมมือของเราช่วยให้ ClawHub เพิ่มข้อมูลข่าวกรองด้านความปลอดภัยที่กว้างขึ้นให้กับการตรวจสอบ Skills และ Plugin

VirusTotal มีประโยชน์เป็นพิเศษสำหรับ artifact อันตรายที่รู้จักอยู่แล้ว การพบโดยเอนจิน และ
สัญญาณชื่อเสียงที่เสริมการตรวจสอบแบบเข้าใจเอเจนต์ของ ClawHub เมื่อมีจำนวนเอนจินของผู้ให้บริการ
การตรวจสอบจะสรุปเป็นภาษาธรรมดา เช่น:

```text
62/62 vendors flagged this skill as clean.
```

หรือ:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

เมื่อ ClawHub ไม่มี telemetry จำนวนผู้ให้บริการให้สรุป การตรวจสอบจะแจ้งว่า:

```text
No VirusTotal findings
```

VirusTotal ยังคงเป็น telemetry ไม่ได้แทนที่การวิเคราะห์ความเสี่ยงแบบเข้าใจ artifact
ของ ClawHub เอง

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัยของ
ClawHub เอง ระบบจะตรวจสอบแต่ละรุ่นเผยแพร่ในฐานะ artifact สำหรับเอเจนต์: คำสั่ง,
ข้อมูลเมตา, สิทธิ์ที่ประกาศ, ไฟล์, สัญญาณความสามารถ, สัญญาณสแกนแบบ static,
ข้อค้นพบของ SkillSpector, telemetry ของ VirusTotal และบริบทที่ผู้เผยแพร่ให้ไว้
สัญญาณสแกนแบบ static เป็นบริบทภายในสำหรับการตรวจสอบนี้ ไม่ใช่
ส่วนการตรวจสอบสาธารณะแบบแยกเดี่ยวหรือคำตัดสินที่บล็อกการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นกรอบมองความเสี่ยง เช่น prompt injection, การใช้เครื่องมือผิดวัตถุประสงค์, การเปิดเผยข้อมูลลับ,
การดำเนินการที่ไม่ปลอดภัย, การปนเปื้อน memory หรือบริบท และอำนาจตัวแทนที่มากเกินไป

ClawScan ไม่ถือว่าความสามารถที่ดูน่ากลัวเป็นสิ่งที่เป็นอันตรายโดยอัตโนมัติ
ระบบจะถามว่าความสามารถนั้นถูกเปิดเผย สอดคล้องกับวัตถุประสงค์ และได้รับการสนับสนุนจาก
กรณีใช้งานที่รุ่นเผยแพร่ระบุไว้หรือไม่
