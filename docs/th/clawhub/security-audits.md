# การตรวจสอบความปลอดภัย

การตรวจสอบความปลอดภัยของ ClawHub ช่วยให้คุณตัดสินใจได้ว่าทักษะหรือ Plugin ปลอดภัยพอที่จะติดตั้งหรือไม่
การตรวจสอบจะแสดงว่ารีลีสทำอะไร ขอสิทธิ์อะไร และ
มีสิ่งใดที่ควรให้ความสนใจเป็นพิเศษก่อนที่จะเข้าถึงไฟล์ บัญชี
ข้อมูลลับ โค้ด หรือบริการภายนอกได้หรือไม่

การตรวจสอบเป็นสัญญาณด้านความปลอดภัยที่แข็งแรง แต่ไม่ใช่การรับประกันว่ารีลีสจะ
ปราศจากความเสี่ยงโดยสิ้นเชิง ใช้วิจารณญาณเสมอก่อนให้สิทธิ์เข้าถึงข้อมูลสำคัญ

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security), [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage),
และ [การดูแลและความปลอดภัยของบัญชี](/clawhub/moderation)

## สิ่งที่ต้องตรวจสอบก่อนติดตั้ง

ก่อนติดตั้ง ให้ตรวจสอบ:

- สถานะการตรวจสอบโดยรวม
- ระดับความเสี่ยง
- findings ที่ระบุไว้
- ข้อมูลลับ สิทธิ์ หรือ environment variables ที่จำเป็น
- เจ้าของ แหล่งที่มา เวอร์ชัน changelog ยอดดาวน์โหลด stars และสัญญาณความน่าเชื่อถืออื่นๆ

ติดตั้งเฉพาะเนื้อหาที่คุณเข้าใจและเชื่อถือเท่านั้น

## สถานะการตรวจสอบ

สถานะการตรวจสอบบอกคุณว่าควรตอบสนองต่อผลการตรวจสอบอย่างไร:

| สถานะ      | ความหมาย                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | ไม่พบปัญหาที่มองเห็นได้ซึ่งสูงกว่าความเสี่ยงต่ำ                                |
| `Review`    | อ่าน findings ก่อนติดตั้ง รีลีสอาจยังถูกต้องตามเจตนา |
| `Warn`      | ใช้ความระมัดระวังเป็นพิเศษ ClawHub พบข้อกังวลที่มีผลกระทบสูงหรือสัญญาณเตือน |
| `Malicious` | อย่าติดตั้ง                                                           |
| `Pending`   | การตรวจสอบยังไม่เสร็จสิ้น                                             |
| `Error`     | ไม่สามารถดำเนินการตรวจสอบให้เสร็จสมบูรณ์ได้                                         |

`Pass` ทำให้มั่นใจได้มากขึ้น แต่ไม่ได้แทนที่วิจารณญาณของคุณเอง เรื่องนี้สำคัญ
ที่สุดสำหรับเครื่องมือที่สามารถเผยแพร่เนื้อหา แก้ไขข้อมูล รันคำสั่ง อ่านไฟล์ หรือ
เข้าถึงระบบ production

## ระดับความเสี่ยง

ระดับความเสี่ยงอธิบาย blast radius: รีลีสดูเหมือนจะมีอำนาจมากเพียงใดหาก
คุณใช้งานตามที่ตั้งใจไว้

| ระดับความเสี่ยง | ความหมาย                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | พบอำนาจที่เกี่ยวข้องกับข้อมูลสำคัญหรือผลกระทบต่อผู้ใช้น้อย                          |
| `Medium`   | รีลีสมีอำนาจที่มีนัยสำคัญ เช่น การเข้าถึงบัญชีหรือการเปลี่ยนแปลงข้อมูล |
| `High`     | รีลีสมีอำนาจที่มีผลกระทบสูง findings รุนแรง หรือสัญญาณที่เป็นอันตราย |

ระดับความเสี่ยงและสถานะการตรวจสอบตอบคำถามคนละอย่าง:

- ระดับความเสี่ยงถามว่า: "ที่นี่มีอำนาจมากแค่ไหน?"
- สถานะการตรวจสอบถามว่า: "ฉันควรทำอย่างไรกับผลลัพธ์นี้?"

ตัวอย่างเช่น ทักษะสำหรับการเผยแพร่อาจแสดง `Review` พร้อมความเสี่ยง `Medium` นั่น
ไม่ได้หมายความว่ามันเป็นอันตราย หมายความว่าทักษะนั้นดูสอดคล้องกับวัตถุประสงค์ แต่สามารถ
ดำเนินการด้วยอำนาจบัญชีที่มีนัยสำคัญได้

## Findings

Findings อธิบายว่าทำไมจึงแสดงผลการตรวจสอบนั้น แต่ละ finding มักประกอบด้วย:

- หมายความว่าอะไร
- ทำไมจึงถูก flag
- เนื้อหาทักษะหรือ Plugin ที่เกี่ยวข้อง
- คำแนะนำ

Findings อาจติดป้าย `Info`, `Low`, `Medium`, `High` หรือ `Critical` findings ที่มี
ความรุนแรงสูงกว่าจะส่งผลต่อระดับความเสี่ยงและสถานะการตรวจสอบมากกว่า

Findings ที่มีความเชื่อมั่นต่ำจะถูกซ่อนจากสรุปการตรวจสอบสาธารณะ เพื่อให้หน้า
ยังคงมุ่งเน้นหลักฐานที่เป็นประโยชน์

## สิ่งที่ ClawHub ตรวจสอบ

ClawHub ตรวจสอบ release artifacts ที่ส่งเข้ามา รวมถึง:

- คำสั่งของทักษะหรือ metadata ของ Plugin
- environment variables และสิทธิ์ที่ประกาศไว้
- คำแนะนำการติดตั้งและ package metadata
- ไฟล์และ file manifests ที่รวมอยู่
- metadata ด้าน compatibility และ capability

คำถามหลักคือความสอดคล้อง: ชื่อ สรุป metadata อำนาจที่ขอ
และเนื้อหาจริงสอดคล้องกับสิ่งที่ผู้ใช้คาดหวังได้อย่างสมเหตุสมผลหรือไม่?

พฤติกรรมที่ทรงพลังไม่ได้แย่โดยอัตโนมัติ เครื่องมือที่มีประโยชน์จำนวนมากต้องใช้ข้อมูลลับ
คำสั่งในเครื่อง provider APIs หรือการติดตั้ง package การตรวจสอบจะตรวจว่า
อำนาจนั้นเป็นสิ่งที่คาดหมายได้ มีการเปิดเผย และได้สัดส่วนหรือไม่

หน้า artifact เชื่อมโยงไปยังการตรวจสอบฉบับเต็มที่:

```text
/<owner>/skills/<slug>/security-audit
```

หน้าการตรวจสอบรวม:

1. SkillSpector
2. VirusTotal
3. การวิเคราะห์ความเสี่ยง

## VirusTotal

ClawHub ใช้ VirusTotal เป็น malware telemetry ใน audit stack VirusTotal เป็น
มาตรฐานอุตสาหกรรมที่เชื่อถือได้สำหรับชื่อเสียงของไฟล์และการสแกนมัลแวร์ และ
ความร่วมมือของเราช่วยให้ ClawHub เพิ่มข้อมูลข่าวกรองความปลอดภัยที่กว้างขึ้นในการรีวิวทักษะและ Plugin

VirusTotal มีประโยชน์เป็นพิเศษสำหรับ artifacts ที่ทราบว่าเป็นอันตราย การตรวจพบจาก engine และ
สัญญาณชื่อเสียงที่ช่วยเสริมการรีวิวของ ClawHub ที่เข้าใจบริบทของเอเจนต์ เมื่อมีจำนวนการตรวจพบจาก
vendor engine การตรวจสอบจะสรุปเป็นภาษาธรรมดา เช่น:

```text
62/62 vendors flagged this skill as clean.
```

หรือ:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

เมื่อ ClawHub ไม่มี telemetry จำนวน vendor ให้สรุป การตรวจสอบจะระบุว่า:

```text
No VirusTotal findings
```

VirusTotal ยังคงเป็น telemetry ไม่ได้แทนที่
การวิเคราะห์ความเสี่ยงที่เข้าใจ artifact ของ ClawHub เอง

## การวิเคราะห์ความเสี่ยง

การวิเคราะห์ความเสี่ยงขับเคลื่อนภายในโดย ClawScan ซึ่งเป็นระบบตรวจสอบความปลอดภัยของ
ClawHub เอง ระบบจะรีวิวแต่ละรีลีสในฐานะ artifact ที่หันหน้าเข้าหาเอเจนต์: คำสั่ง
metadata สิทธิ์ที่ประกาศ ไฟล์ สัญญาณ capability สัญญาณ static scan
findings ของ SkillSpector, telemetry ของ VirusTotal และบริบทที่ผู้เผยแพร่ให้ไว้
สัญญาณ static scan เป็นบริบทภายในสำหรับการรีวิวนี้ ไม่ใช่
ส่วนการตรวจสอบสาธารณะแบบ standalone หรือ verdict ที่บล็อกการติดตั้ง

การวิเคราะห์ความเสี่ยงใช้
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
เป็นเลนส์สำหรับความเสี่ยง เช่น prompt injection, tool misuse, credential exposure,
unsafe execution, memory หรือ context poisoning และ excessive agency

ClawScan ไม่ถือว่า capability ที่ดูน่ากลัวเป็นอันตรายโดยอัตโนมัติ
แต่จะถามว่า capability นั้นมีการเปิดเผย สอดคล้องกับวัตถุประสงค์ และได้รับการสนับสนุนโดย
กรณีการใช้งานที่รีลีสระบุไว้หรือไม่
