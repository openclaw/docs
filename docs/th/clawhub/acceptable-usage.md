# การใช้งานที่ยอมรับได้

หน้านี้อธิบายประเภทของ Skills และเนื้อหาที่ ClawHub ยอมรับได้ รวมถึงเวิร์กโฟลว์การใช้งานในทางที่ผิดที่ ClawHub จะไม่โฮสต์

กฎเหล่านี้ตั้งใจให้ใช้งานได้จริง เราให้ความสำคัญสูงสุดกับเวิร์กโฟลว์การใช้งานในทางที่ผิดแบบครบวงจร ไม่ใช่แค่คำสำคัญที่แยกกันอยู่ หาก Skill ถูกสร้างขึ้นเพื่อหลบเลี่ยงการป้องกัน ใช้งานแพลตฟอร์มในทางที่ผิด หลอกลวงผู้คน ละเมิดความเป็นส่วนตัว หรือเปิดทางให้เกิดพฤติกรรมที่ไม่ได้รับความยินยอม Skill นั้นไม่เหมาะกับ ClawHub

## รูปแบบล่าสุดที่เรายอมรับอย่างชัดเจน

- งานส่วนหน้าและระบบออกแบบที่ใช้คอมโพเนนต์จริง โทเคนเชิงความหมาย สถานะที่เข้าถึงได้ และโฟลว์ผู้ใช้ที่ผ่านการทดสอบแล้ว
- การประกอบ shadcn/ui ที่ใช้คอมโพเนนต์ซอร์สที่ติดตั้งไว้ นามแฝงของโปรเจกต์ และตัวแปรที่มีเอกสาร แทนมาร์กอัปเฉพาะกิจ
- การแปลง UI5 จาก JavaScript เป็น TypeScript ที่คงความคิดเห็นไว้ ใช้ชนิด UI5 ที่เป็นรูปธรรม และทำให้อินเทอร์เฟซของคอนโทรลที่สร้างขึ้นตรวจทานได้
- การตรวจทานความปลอดภัยเชิงป้องกัน เครื่องมือมอดเดอเรชัน และพรอมป์ตรวจจับการใช้งานในทางที่ผิดที่แสดงหลักฐานและรักษาขอบเขตการอนุมัติโดยมนุษย์ให้ชัดเจน
- ระบบอัตโนมัติสำหรับเวิร์กโฟลว์ที่อิงความยินยอมสำหรับบัญชีส่วนตัวหรือบัญชีทีม โดยมีข้อมูลรับรองที่ชัดเจน การตั้งค่าที่โปร่งใส และโหมด dry-run หรือแสดงตัวอย่าง
- เอกสาร runbook การย้ายระบบ ยูทิลิตีสำหรับนักพัฒนา และ fixture สำหรับการทดสอบที่จำกัดขอบเขตตามซอฟต์แวร์ที่รองรับ

## ไม่ยอมรับ

- เวิร์กโฟลว์เลี่ยงความปลอดภัยหรือเข้าถึงโดยไม่ได้รับอนุญาต
  - ตัวอย่าง: การเลี่ยงการยืนยันตัวตน การยึดบัญชี การเลี่ยง CAPTCHA การหลบเลี่ยง Cloudflare หรือระบบป้องกันบอต การเลี่ยง rate limit การ scraping แบบซ่อนตัวที่ออกแบบมาเพื่อเอาชนะการป้องกัน การยึดสายสนทนาหรือเอเจนต์แบบสด การขโมยเซสชันที่นำกลับมาใช้ได้ การอนุมัติโฟลว์จับคู่โดยอัตโนมัติสำหรับผู้ใช้ที่ไม่ได้รับอนุมัติ

- การใช้งานแพลตฟอร์มในทางที่ผิดและการหลบเลี่ยงการแบน
  - ตัวอย่าง: บัญชีซ่อนตัวหลังถูกแบน การวอร์มหรือฟาร์มบัญชี การมีส่วนร่วมปลอม การปั้นคาร์มาหรือผู้ติดตาม ระบบอัตโนมัติหลายบัญชี การโพสต์จำนวนมาก บอตสแปม ระบบอัตโนมัติของมาร์เก็ตเพลสหรือโซเชียลที่สร้างมาเพื่อหลีกเลี่ยงการตรวจจับ

- การฉ้อโกง การหลอกลวง และเวิร์กโฟลว์ทางการเงินที่หลอกลวง
  - ตัวอย่าง: ใบรับรองปลอม ใบแจ้งหนี้ปลอม โฟลว์การชำระเงินที่หลอกลวง การติดต่อเพื่อหลอกลวง หลักฐานทางสังคมปลอม เครื่องมือที่เปิดทางให้ใช้จ่ายหรือเรียกเก็บเงินโดยไม่มีการอนุมัติจากมนุษย์ที่ชัดเจนและการควบคุมที่โปร่งใส หรือเวิร์กโฟลว์ตัวตนสังเคราะห์ที่สร้างขึ้นเพื่อสร้างบัญชีสำหรับการฉ้อโกง

- การ scraping การเสริมข้อมูล หรือการเฝ้าระวังที่ละเมิดความเป็นส่วนตัว
  - ตัวอย่าง: การ scraping รายละเอียดการติดต่อในวงกว้างเพื่อสแปม การ doxxing การสะกดรอยตาม การดึงลูกค้าเป้าหมายที่จับคู่กับการติดต่อที่ไม่ได้ร้องขอ การเฝ้าติดตามแบบลับ การค้นหาใบหน้าหรือการจับคู่ไบโอเมตริกที่ใช้โดยไม่มีความยินยอมชัดเจน หรือการซื้อ เผยแพร่ ดาวน์โหลด หรือนำข้อมูลที่รั่วไหลหรือชุดข้อมูลจากการละเมิดไปปฏิบัติการ

- การแอบอ้างโดยไม่ได้รับความยินยอมหรือการบิดเบือนตัวตนที่หลอกลวง
  - ตัวอย่าง: การสลับใบหน้า ฝาแฝดดิจิทัล บุคลิกปลอม อินฟลูเอนเซอร์ที่ถูกโคลน หรือเครื่องมือบิดเบือนตัวตนอื่นๆ ที่ใช้เพื่อแอบอ้างหรือทำให้เข้าใจผิด

- เนื้อหาทางเพศอย่างโจ่งแจ้งและการสร้างเนื้อหาผู้ใหญ่ที่ปิดการทำงานด้านความปลอดภัย
  - ตัวอย่าง: การสร้างรูปภาพ/วิดีโอ/เนื้อหา NSFW ตัวครอบเนื้อหาผู้ใหญ่รอบ API ของบุคคลที่สาม หรือ Skills ที่มีวัตถุประสงค์หลักเป็นเนื้อหาทางเพศอย่างโจ่งแจ้ง

- ข้อกำหนดการรันที่ซ่อนอยู่ ไม่ปลอดภัย หรือทำให้เข้าใจผิด
  - ตัวอย่าง: คำสั่งติดตั้งที่ทำให้อ่านยาก `curl | sh` ข้อกำหนด secret ที่ไม่ได้ประกาศ การใช้ private key ที่ไม่ได้ประกาศ การรัน `npx @latest` จากระยะไกลโดยไม่มีความสามารถในการตรวจทานที่ชัดเจน เมทาดาทาที่ทำให้เข้าใจผิดซึ่งซ่อนสิ่งที่ Skill ต้องใช้จริงเพื่อรัน

## รูปแบบล่าสุดที่เราไม่ยอมรับอย่างชัดเจน

- “สร้างบัญชีผู้ขายแบบซ่อนตัวหลังถูกแบนจากมาร์เก็ตเพลส”
- “แก้ไขการจับคู่ Telegram เพื่อให้ผู้ใช้ที่ไม่ได้รับอนุมัติได้รับรหัสจับคู่โดยอัตโนมัติ”
- “ปั้นบัญชี Reddit/Twitter ด้วยระบบอัตโนมัติที่ตรวจจับไม่ได้”
- “สร้างใบรับรองวิชาชีพหรือใบแจ้งหนี้สำหรับการใช้งานใดๆ”
- “สร้างเนื้อหา NSFW โดยปิดการตรวจสอบความปลอดภัย”
- “Scrape ลูกค้าเป้าหมาย เสริมข้อมูลผู้ติดต่อ และเปิดตัวการติดต่อแบบเย็นในวงกว้าง”
- “ซื้อ เผยแพร่ หรือดาวน์โหลดข้อมูลที่รั่วไหลหรือชุดข้อมูลจากการละเมิด”
- “สร้างบัญชีอีเมลหรือโซเชียลจำนวนมากด้วยตัวตนสังเคราะห์หรือการแก้ CAPTCHA”

## หมายเหตุสำหรับผู้ตรวจทาน

- บริบทมีความสำคัญ หัวข้อเดียวกันอาจชอบด้วยเหตุผลในสภาพแวดล้อมเชิงป้องกันหรืออิงความยินยอมที่แคบ และไม่ยอมรับได้เมื่อถูกจัดแพ็กเป็นเวิร์กโฟลว์การใช้งานในทางที่ผิด
- เราควรเอนเอียงไปทางการดำเนินการเมื่อ Skill ถูกปรับให้เหมาะสมอย่างชัดเจนสำหรับการหลบเลี่ยง การหลอกลวง หรือการใช้งานโดยไม่ได้รับความยินยอม
- การอัปโหลดซ้ำในหมวดหมู่เหล่านี้เป็นเหตุให้ซ่อนเนื้อหาและแบนบัญชีได้

## การบังคับใช้

- เราอาจซ่อน ลบ หรือ hard-delete Skills ที่ละเมิด
- เราอาจเพิกถอนโทเคน soft-delete เนื้อหาที่เกี่ยวข้อง และแบนผู้กระทำผิดซ้ำหรือร้ายแรง
- เราไม่รับประกันการบังคับใช้แบบเตือนก่อนสำหรับการใช้งานในทางที่ผิดที่ชัดเจน
