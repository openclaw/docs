# การใช้งานที่ยอมรับได้

หน้านี้อธิบายประเภทของ Skills และเนื้อหาที่ ClawHub ยอมรับได้ รวมถึงเวิร์กโฟลว์การละเมิดที่ ClawHub จะไม่โฮสต์

กฎเหล่านี้ตั้งใจให้ใช้งานได้จริง เราให้ความสำคัญสูงสุดกับเวิร์กโฟลว์การละเมิดแบบครบวงจร ไม่ใช่เพียงคีย์เวิร์ดที่แยกเป็นส่วน ๆ หาก Skill ถูกสร้างขึ้นเพื่อหลบเลี่ยงการป้องกัน ละเมิดแพลตฟอร์ม หลอกลวงผู้คน ละเมิดความเป็นส่วนตัว หรือเปิดทางให้เกิดพฤติกรรมที่ไม่ได้รับความยินยอม Skill นั้นไม่เหมาะสมบน ClawHub

## รูปแบบล่าสุดที่เรายอมรับอย่างชัดเจน

- งานฟรอนต์เอนด์และระบบออกแบบที่ใช้คอมโพเนนต์จริง โทเค็นเชิงความหมาย สถานะที่เข้าถึงได้ และโฟลว์ผู้ใช้ที่ผ่านการทดสอบแล้ว
- การประกอบ shadcn/ui ที่ใช้คอมโพเนนต์ซอร์สที่ติดตั้งแล้ว อลิอาสของโปรเจกต์ และตัวแปรที่มีเอกสารกำกับ แทนมาร์กอัปเฉพาะกิจ
- การแปลง UI5 จาก JavaScript เป็น TypeScript ที่คงคอมเมนต์ไว้ ใช้ชนิด UI5 ที่เป็นรูปธรรม และทำให้อินเทอร์เฟซคอนโทรลที่สร้างขึ้นตรวจทานได้
- การตรวจทานความปลอดภัยเชิงป้องกัน เครื่องมือมอดเดอเรชัน และพรอมป์ตรวจจับการละเมิดที่แสดงหลักฐานและคงขอบเขตการอนุมัติของมนุษย์ไว้อย่างชัดเจน
- ระบบอัตโนมัติของเวิร์กโฟลว์ตามความยินยอมสำหรับบัญชีส่วนบุคคลหรือทีม โดยมีข้อมูลประจำตัวที่ชัดเจน การตั้งค่าที่โปร่งใส และโหมด dry-run หรือพรีวิว
- เอกสาร runbook สำหรับการย้ายระบบ ยูทิลิตีสำหรับนักพัฒนา และฟิกซ์เจอร์ทดสอบที่มีขอบเขตตามซอฟต์แวร์ที่รองรับ

## ไม่ยอมรับ

- เวิร์กโฟลว์การข้ามความปลอดภัยหรือการเข้าถึงโดยไม่ได้รับอนุญาต
  - ตัวอย่าง: การข้ามการยืนยันตัวตน การยึดบัญชี การข้าม CAPTCHA การหลบเลี่ยง Cloudflare หรือระบบป้องกันบอต การข้าม rate limit การ scraping แบบซ่อนตัวที่ออกแบบมาเพื่อเอาชนะการป้องกัน การยึดสายสนทนาหรือ agent แบบสด การขโมยเซสชันที่นำกลับมาใช้ซ้ำได้ การอนุมัติโฟลว์จับคู่โดยอัตโนมัติสำหรับผู้ใช้ที่ไม่ได้รับอนุมัติ

- การละเมิดแพลตฟอร์มและการหลบเลี่ยงการแบน
  - ตัวอย่าง: บัญชีซ่อนตัวหลังถูกแบน การอุ่นเครื่องหรือฟาร์มบัญชี การมีส่วนร่วมปลอม การบ่มเพาะ karma หรือผู้ติดตาม ระบบอัตโนมัติหลายบัญชี การโพสต์จำนวนมาก บอตสแปม ระบบอัตโนมัติสำหรับมาร์เก็ตเพลสหรือโซเชียลที่สร้างมาเพื่อหลีกเลี่ยงการตรวจจับ

- การฉ้อโกง การหลอกลวง และเวิร์กโฟลว์ทางการเงินที่หลอกลวง
  - ตัวอย่าง: ใบรับรองปลอม ใบแจ้งหนี้ปลอม โฟลว์การชำระเงินที่หลอกลวง การติดต่อเพื่อหลอกลวง หลักฐานทางสังคมปลอม เครื่องมือที่เปิดทางให้ใช้จ่ายหรือเรียกเก็บเงินโดยไม่มีการอนุมัติจากมนุษย์ที่ชัดเจนและการควบคุมที่โปร่งใส หรือเวิร์กโฟลว์ตัวตนสังเคราะห์ที่สร้างขึ้นเพื่อสร้างบัญชีสำหรับการฉ้อโกง

- การ scraping การเสริมข้อมูล หรือการเฝ้าระวังที่ละเมิดความเป็นส่วนตัว
  - ตัวอย่าง: การ scraping รายละเอียดการติดต่อในวงกว้างเพื่อสแปม การเปิดเผยข้อมูลส่วนตัว การสะกดรอย การดึงข้อมูลลูกค้าเป้าหมายควบคู่กับการติดต่อที่ไม่ได้ร้องขอ การเฝ้าติดตามแบบลับ การค้นหาใบหน้าหรือการจับคู่ไบโอเมตริกที่ใช้โดยไม่มีความยินยอมที่ชัดเจน หรือการซื้อ เผยแพร่ ดาวน์โหลด หรือนำข้อมูลรั่วไหลหรือชุดข้อมูลจากการละเมิดไปใช้ปฏิบัติการ

- การแอบอ้างบุคคลโดยไม่ได้รับความยินยอมหรือการบิดเบือนตัวตนอย่างหลอกลวง
  - ตัวอย่าง: การสลับใบหน้า ฝาแฝดดิจิทัล ตัวตนปลอม อินฟลูเอนเซอร์ที่ถูกโคลน หรือเครื่องมือจัดการตัวตนอื่น ๆ ที่ใช้เพื่อแอบอ้างหรือทำให้เข้าใจผิด

- เนื้อหาทางเพศอย่างโจ่งแจ้งและการสร้างเนื้อหาผู้ใหญ่ที่ปิดระบบความปลอดภัย
  - ตัวอย่าง: การสร้างรูปภาพ/วิดีโอ/เนื้อหา NSFW wrapper เนื้อหาผู้ใหญ่รอบ API ของบุคคลที่สาม หรือ Skills ที่มีวัตถุประสงค์หลักเป็นเนื้อหาทางเพศอย่างโจ่งแจ้ง

- ข้อกำหนดการรันที่ซ่อนอยู่ ไม่ปลอดภัย หรือทำให้เข้าใจผิด
  - ตัวอย่าง: คำสั่งติดตั้งที่ทำให้อ่านยาก `curl | sh` ข้อกำหนดด้านความลับที่ไม่ประกาศไว้ การใช้ private key ที่ไม่ประกาศไว้ การรัน `npx @latest` จากระยะไกลโดยไม่มีความสามารถในการตรวจทานที่ชัดเจน เมทาดาทาที่ทำให้เข้าใจผิดและซ่อนสิ่งที่ Skill ต้องใช้จริงเพื่อรัน

## รูปแบบล่าสุดที่เราไม่ยอมรับอย่างชัดเจน

- “สร้างบัญชีผู้ขายแบบซ่อนตัวหลังถูกแบนจากมาร์เก็ตเพลส”
- “แก้ไขการจับคู่ Telegram เพื่อให้ผู้ใช้ที่ไม่ได้รับอนุมัติได้รับรหัสจับคู่โดยอัตโนมัติ”
- “บ่มเพาะบัญชี Reddit/Twitter ด้วยระบบอัตโนมัติที่ตรวจจับไม่ได้”
- “สร้างใบรับรองวิชาชีพหรือใบแจ้งหนี้สำหรับการใช้งานใดก็ได้”
- “สร้างเนื้อหา NSFW โดยปิดการตรวจสอบความปลอดภัย”
- “Scrape ลูกค้าเป้าหมาย เสริมข้อมูลผู้ติดต่อ และเริ่มการติดต่อแบบ cold outreach ในวงกว้าง”
- “ซื้อ เผยแพร่ หรือดาวน์โหลดข้อมูลรั่วไหลหรือชุดข้อมูลจากการละเมิด”
- “สร้างบัญชีอีเมลหรือโซเชียลจำนวนมากด้วยตัวตนสังเคราะห์หรือการแก้ CAPTCHA”

## หมายเหตุสำหรับผู้ตรวจทาน

- บริบทมีความสำคัญ หัวข้อเดียวกันอาจถูกต้องตามสมควรในบริบทเชิงป้องกันที่แคบหรืออิงความยินยอม และไม่ยอมรับได้เมื่อถูกบรรจุเป็นเวิร์กโฟลว์การละเมิด
- เราควรเอนเอียงไปทางการดำเนินการเมื่อ Skill ถูกปรับให้เหมาะกับการหลบเลี่ยง การหลอกลวง หรือการใช้งานที่ไม่ได้รับความยินยอมอย่างชัดเจน
- การอัปโหลดซ้ำในหมวดหมู่เหล่านี้เป็นเหตุให้ซ่อนเนื้อหาและแบนบัญชีได้

## การบังคับใช้

- เราอาจซ่อน ลบ หรือ hard-delete Skills ที่ละเมิด
- เราอาจเพิกถอนโทเค็น soft-delete เนื้อหาที่เกี่ยวข้อง และแบนผู้กระทำผิดซ้ำหรือร้ายแรง
- เราไม่รับประกันว่าจะมีการเตือนก่อนบังคับใช้สำหรับการละเมิดที่ชัดเจน
