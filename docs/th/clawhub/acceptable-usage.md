# การใช้งานที่ยอมรับได้

หน้านี้อธิบายประเภทของ Skills และเนื้อหาที่ ClawHub ยอมรับได้ รวมถึงเวิร์กโฟลว์การละเมิดที่จะไม่โฮสต์

กฎเหล่านี้ตั้งใจให้ใช้งานได้จริง เราให้ความสำคัญมากที่สุดกับเวิร์กโฟลว์การละเมิดแบบครบวงจร ไม่ใช่แค่คีย์เวิร์ดที่แยกจากกัน หาก Skills ถูกสร้างมาเพื่อหลบเลี่ยงการป้องกัน ใช้แพลตฟอร์มในทางที่ผิด หลอกลวงผู้คน ละเมิดความเป็นส่วนตัว หรือเปิดทางให้เกิดพฤติกรรมที่ไม่ได้รับความยินยอม Skills นั้นไม่เหมาะกับ ClawHub

## รูปแบบล่าสุดที่เรายอมรับอย่างชัดเจน

- งานฟรอนต์เอนด์และระบบออกแบบที่ใช้คอมโพเนนต์จริง โทเค็นเชิงความหมาย สถานะที่เข้าถึงได้ และโฟลว์ผู้ใช้ที่ผ่านการทดสอบแล้ว
- การประกอบ shadcn/ui ที่ใช้คอมโพเนนต์ซอร์สที่ติดตั้งแล้ว alias ของโปรเจกต์ และ variant ที่มีเอกสารกำกับ แทนมาร์กอัปเฉพาะกิจ
- การแปลง UI5 JavaScript เป็น TypeScript ที่คงความคิดเห็นไว้ ใช้ชนิด UI5 ที่เป็นรูปธรรม และทำให้อินเทอร์เฟซของคอนโทรลที่สร้างขึ้นตรวจทานได้
- การตรวจทานความปลอดภัยเชิงป้องกัน เครื่องมือมอดเรชัน และพรอมป์ตรวจจับการละเมิดที่แสดงหลักฐานและรักษาขอบเขตการอนุมัติโดยมนุษย์ให้ชัดเจน
- การทำงานอัตโนมัติของเวิร์กโฟลว์ตามความยินยอมสำหรับบัญชีส่วนบุคคลหรือทีม โดยมีข้อมูลรับรองอย่างชัดเจน การตั้งค่าที่โปร่งใส และโหมด dry-run หรือพรีวิว
- เอกสาร runbook สำหรับการย้ายระบบ ยูทิลิตีสำหรับนักพัฒนา และฟิกซ์เจอร์ทดสอบที่จำกัดขอบเขตตามซอฟต์แวร์ที่รองรับ

## ไม่ยอมรับ

- เวิร์กโฟลว์หลบเลี่ยงความปลอดภัยหรือเข้าถึงโดยไม่ได้รับอนุญาต
  - ตัวอย่าง: การข้ามการยืนยันตัวตน การยึดบัญชี การข้าม CAPTCHA การหลบเลี่ยง Cloudflare หรือระบบป้องกันบอต การข้าม rate limit การสแครปแบบซ่อนตัวที่ออกแบบมาเพื่อเอาชนะการป้องกัน การยึดสายสนทนาหรือ agent แบบสด การขโมยเซสชันที่นำกลับมาใช้ซ้ำได้ การอนุมัติโฟลว์ pairing อัตโนมัติสำหรับผู้ใช้ที่ไม่ได้รับอนุมัติ

- การใช้แพลตฟอร์มในทางที่ผิดและการหลบเลี่ยงการแบน
  - ตัวอย่าง: บัญชีซ่อนตัวหลังถูกแบน การอุ่นบัญชี/ฟาร์มบัญชี การมีส่วนร่วมปลอม การปลูกคะแนน karma หรือผู้ติดตาม การทำงานอัตโนมัติหลายบัญชี การโพสต์จำนวนมาก บอตสแปม การทำงานอัตโนมัติของมาร์เก็ตเพลสหรือโซเชียลที่สร้างมาเพื่อหลีกเลี่ยงการตรวจจับ

- การฉ้อโกง การหลอกลวง และเวิร์กโฟลว์ทางการเงินที่หลอกลวง
  - ตัวอย่าง: ใบรับรองปลอม ใบแจ้งหนี้ปลอม โฟลว์การชำระเงินที่หลอกลวง การติดต่อเพื่อหลอกลวง หลักฐานทางสังคมปลอม เครื่องมือที่เปิดทางให้ใช้จ่ายหรือเรียกเก็บเงินโดยไม่มีการอนุมัติโดยมนุษย์ที่ชัดเจนและการควบคุมที่โปร่งใส หรือเวิร์กโฟลว์ตัวตนสังเคราะห์ที่สร้างมาเพื่อสร้างบัญชีสำหรับการฉ้อโกง

- การสแครป การเพิ่มข้อมูล หรือการเฝ้าระวังที่ละเมิดความเป็นส่วนตัว
  - ตัวอย่าง: การสแครปรายละเอียดติดต่อในวงกว้างเพื่อสแปม การ doxxing การสะกดรอยตาม การดึงลีดพร้อมกับการติดต่อโดยไม่ได้รับเชิญ การเฝ้าติดตามแบบลับ การค้นหาใบหน้าหรือการจับคู่ไบโอเมตริกที่ใช้โดยไม่มีความยินยอมที่ชัดเจน หรือการซื้อ เผยแพร่ ดาวน์โหลด หรือนำข้อมูลรั่วไหลหรือชุดข้อมูลจากการละเมิดไปใช้งานจริง

- การปลอมตัวโดยไม่ได้รับความยินยอมหรือการจัดการตัวตนเพื่อหลอกลวง
  - ตัวอย่าง: face swap, digital twins, persona ปลอม, influencer ที่ถูกโคลน หรือเครื่องมือจัดการตัวตนอื่น ๆ ที่ใช้เพื่อปลอมตัวหรือทำให้เข้าใจผิด

- เนื้อหาทางเพศอย่างชัดเจนและการสร้างเนื้อหาผู้ใหญ่ที่ปิดการทำงานด้านความปลอดภัย
  - ตัวอย่าง: การสร้างรูปภาพ/วิดีโอ/เนื้อหา NSFW, wrapper เนื้อหาผู้ใหญ่รอบ API ของบุคคลที่สาม หรือ Skills ที่มีวัตถุประสงค์หลักเป็นเนื้อหาทางเพศอย่างชัดเจน

- ข้อกำหนดการรันที่ซ่อนอยู่ ไม่ปลอดภัย หรือทำให้เข้าใจผิด
  - ตัวอย่าง: คำสั่งติดตั้งที่ทำให้อ่านยาก, `curl | sh`, ข้อกำหนด secret ที่ไม่ประกาศ, การใช้ private key ที่ไม่ประกาศ, การรัน `npx @latest` ระยะไกลโดยไม่มีความสามารถในการตรวจทานที่ชัดเจน, metadata ที่ทำให้เข้าใจผิดซึ่งซ่อนสิ่งที่ Skills ต้องใช้จริงเพื่อรัน

## รูปแบบล่าสุดที่เราไม่ยอมรับอย่างชัดเจน

- “สร้างบัญชีผู้ขายแบบซ่อนตัวหลังถูกแบนจากมาร์เก็ตเพลส”
- “แก้ไขการ pairing ของ Telegram เพื่อให้ผู้ใช้ที่ไม่ได้รับอนุมัติได้รับรหัส pairing โดยอัตโนมัติ”
- “ปลูกบัญชี Reddit/Twitter ด้วยการทำงานอัตโนมัติที่ตรวจจับไม่ได้”
- “สร้างใบรับรองวิชาชีพหรือใบแจ้งหนี้สำหรับการใช้งานใดก็ได้”
- “สร้างเนื้อหา NSFW โดยปิดการตรวจสอบความปลอดภัย”
- “สแครปลีด เพิ่มข้อมูลผู้ติดต่อ และเริ่มการติดต่อแบบ cold outreach ในวงกว้าง”
- “ซื้อ เผยแพร่ หรือดาวน์โหลดข้อมูลรั่วไหลหรือชุดข้อมูลจากการละเมิด”
- “สร้างบัญชีอีเมลหรือโซเชียลจำนวนมากด้วยตัวตนสังเคราะห์หรือการแก้ CAPTCHA”

## หมายเหตุสำหรับผู้ตรวจทาน

- บริบทสำคัญ หัวข้อเดียวกันอาจถูกต้องได้ในบริบทเชิงป้องกันที่แคบหรือบริบทตามความยินยอม และยอมรับไม่ได้เมื่อถูกแพ็กเป็นเวิร์กโฟลว์การละเมิด
- เราควรเอนเอียงไปทางการดำเนินการเมื่อ Skills ถูกปรับให้เหมาะกับการหลบเลี่ยง การหลอกลวง หรือการใช้งานที่ไม่ได้รับความยินยอมอย่างชัดเจน
- การอัปโหลดซ้ำในหมวดหมู่เหล่านี้เป็นเหตุให้ซ่อนเนื้อหาและแบนบัญชีได้

## การบังคับใช้

- เราอาจซ่อน ลบ หรือ hard-delete Skills ที่ละเมิด
- เราอาจเพิกถอนโทเค็น soft-delete เนื้อหาที่เกี่ยวข้อง และแบนผู้กระทำผิดซ้ำหรือร้ายแรง
- เราไม่รับประกันว่าจะมีการเตือนก่อนบังคับใช้สำหรับการละเมิดที่ชัดเจน
