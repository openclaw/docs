# การใช้งานที่ยอมรับได้

ClawHub โฮสต์ Skills, Plugin, แพ็กเกจ และเมทาดาทาของมาร์เก็ตเพลสสำหรับ OpenClaw
ใช้หน้านี้เพื่อตัดสินว่าเนื้อหาหรือพฤติกรรมการเผยแพร่ควรอยู่บน
ClawHub หรือไม่

กฎเหล่านี้ใช้กับสิ่งที่รายการทำ สิ่งที่รายการขอให้ผู้ใช้เรียกใช้ วิธีที่รายการ
นำเสนอตัวเอง และวิธีที่ผู้เผยแพร่ใช้พื้นผิวการค้นพบ การติดตั้ง และความน่าเชื่อถือ
ของ ClawHub สำหรับสถานะการดูแลและสถานะบัญชี โปรดดู
[การดูแลและความปลอดภัยของบัญชี](/clawhub/moderation) สำหรับการอ้างสิทธิ์ด้านลิขสิทธิ์หรือสิทธิอื่น ๆ
โปรดดู [คำขอด้านสิทธิ์ในเนื้อหา](/clawhub/content-rights)

## เนื้อหาที่อนุญาต

ClawHub ยินดีต้อนรับเนื้อหาที่มีประโยชน์ เข้าใจได้ และเผยแพร่โดยสุจริต

| หมวดหมู่ | อนุญาตเมื่อ |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| ประสิทธิภาพการทำงานของนักพัฒนา | รายการช่วยให้ผู้ใช้สร้าง ทดสอบ ย้ายระบบ ดีบัก จัดทำเอกสาร หรือดำเนินงานซอฟต์แวร์ |
| เวิร์กโฟลว์ UI ข้อมูล และระบบอัตโนมัติ | ขอบเขตชัดเจน ระบุข้อมูลประจำตัวที่จำเป็นอย่างชัดเจน และการกระทำที่มีความเสี่ยงมีเส้นทางสำหรับการตรวจทาน การทดลองรัน การแสดงตัวอย่าง หรือการยืนยัน |
| การรักษาความปลอดภัยเชิงป้องกัน การดูแล และการตรวจทานการละเมิด | เครื่องมือถูกวางกรอบสำหรับการตรวจทานที่ได้รับอนุญาต เก็บรักษาหลักฐาน และทำให้ขอบเขตการอนุมัติของมนุษย์ชัดเจน |
| เวิร์กโฟลว์ส่วนบุคคลหรือทีม | เวิร์กโฟลว์ใช้บัญชีที่อิงความยินยอม การตั้งค่าที่โปร่งใส และสิทธิ์ที่ชัดเจน |
| แค็ตตาล็อกที่ได้รับการบำรุงรักษา | แต่ละรายการแตกต่าง มีประโยชน์ อธิบายอย่างถูกต้อง และได้รับการบำรุงรักษาอย่างสมเหตุสมผล |

บริบทมีความสำคัญ หัวข้อเดียวกันอาจยอมรับได้ในบริบทเชิงป้องกันที่แคบหรือ
อิงความยินยอม และยอมรับไม่ได้เมื่อบรรจุเป็นเวิร์กโฟลว์การละเมิด

## เนื้อหาที่ไม่อนุญาต

ClawHub ไม่โฮสต์เนื้อหาที่มีวัตถุประสงค์หลักเป็นการละเมิด การหลอกลวง การดำเนินการที่ไม่ปลอดภัย
หรือการละเมิดสิทธิ

| หมวดหมู่ | ไม่อนุญาต |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| การเข้าถึงโดยไม่ได้รับอนุญาตหรือการเลี่ยงการรักษาความปลอดภัย | การเลี่ยงการยืนยันตัวตน การยึดบัญชี การละเมิดขีดจำกัดอัตรา การยึดสายสดหรือเอเจนต์ การขโมยเซสชันที่นำกลับมาใช้ได้ หรือการอนุมัติโฟลว์การจับคู่ให้อัตโนมัติสำหรับผู้ใช้ที่ไม่ได้รับอนุมัติ |
| การละเมิดแพลตฟอร์มและการเลี่ยงการแบน | บัญชีซ่อนตัวหลังถูกแบน การอุ่นบัญชีหรือฟาร์มบัญชี การมีส่วนร่วมปลอม ระบบอัตโนมัติหลายบัญชี การโพสต์จำนวนมาก บอทสแปม หรือระบบอัตโนมัติที่สร้างมาเพื่อหลีกเลี่ยงการตรวจจับ |
| การฉ้อโกง การหลอกลวง และเวิร์กโฟลว์ทางการเงินที่ทำให้เข้าใจผิด | ใบรับรองหรือใบแจ้งหนี้ปลอม โฟลว์การชำระเงินที่ทำให้เข้าใจผิด การติดต่อหลอกลวง หลักฐานทางสังคมปลอม เวิร์กโฟลว์ตัวตนสังเคราะห์เพื่อการฉ้อโกง หรือเครื่องมือใช้จ่าย/เรียกเก็บเงินโดยไม่มีการอนุมัติของมนุษย์ที่ชัดเจน |
| การเพิ่มข้อมูลหรือการเฝ้าระวังที่รุกรานความเป็นส่วนตัว | การดึงข้อมูลผู้ติดต่อเพื่อสแปม การเปิดเผยข้อมูลส่วนบุคคล การสะกดรอย การดึงลีดที่จับคู่กับการติดต่อที่ไม่ได้ร้องขอ การเฝ้าติดตามลับ การจับคู่ชีวมิติที่ไม่ได้รับความยินยอม หรือการใช้ข้อมูลรั่วไหลหรือชุดข้อมูลจากการละเมิด |
| การแอบอ้างบุคคลโดยไม่ได้รับความยินยอมหรือการบิดเบือนตัวตน | การสลับใบหน้า ฝาแฝดดิจิทัล อินฟลูเอนเซอร์ที่ถูกโคลน ตัวตนปลอม หรือเครื่องมืออื่น ๆ ที่ใช้แอบอ้างหรือทำให้เข้าใจผิด |
| เนื้อหาทางเพศอย่างชัดแจ้งหรือการสร้างเนื้อหาผู้ใหญ่ที่ปิดกลไกความปลอดภัย | การสร้างรูปภาพ วิดีโอ หรือเนื้อหา NSFW; ตัวครอบเนื้อหาผู้ใหญ่รอบ API ของบุคคลที่สาม; หรือรายการที่มีวัตถุประสงค์หลักเป็นเนื้อหาทางเพศอย่างชัดแจ้ง |
| ข้อกำหนดการดำเนินการที่ซ่อนอยู่ ไม่ปลอดภัย หรือทำให้เข้าใจผิด | คำสั่งติดตั้งที่ทำให้อ่านยาก ตัวติดตั้งแบบ pipe-to-shell เช่น เนื้อหาที่ดาวน์โหลดแล้วรันด้วย `sh` หรือ `bash` โดยตรวจทานได้ไม่ชัดเจน ข้อกำหนดเกี่ยวกับความลับหรือคีย์ส่วนตัวที่ไม่ได้ประกาศ การเรียกใช้ `npx @latest` ระยะไกลโดยตรวจทานได้ไม่ชัดเจน หรือเมทาดาทาที่ซ่อนสิ่งที่รายการต้องใช้จริงเพื่อทำงาน |
| เนื้อหาที่ละเมิดลิขสิทธิ์หรือละเมิดสิทธิ | การเผยแพร่ Skills, Plugin, เอกสาร สินทรัพย์แบรนด์ หรือโค้ดกรรมสิทธิ์ของผู้อื่นซ้ำโดยไม่ได้รับอนุญาต การละเมิดเงื่อนไขสิทธิ์การใช้งาน หรือการแอบอ้างเป็นผู้เขียนหรือผู้เผยแพร่เดิม |

## พฤติกรรมในมาร์เก็ตเพลสที่ไม่อนุญาต

ClawHub ยังตรวจทานวิธีที่ผู้เผยแพร่ใช้มาร์เก็ตเพลสด้วย ห้ามใช้ ClawHub เพื่อ
บิดเบือนการค้นพบ เมตริก สัญญาณความน่าเชื่อถือ ระบบการดูแล หรือความสนใจ
ของผู้ใช้

พฤติกรรมในมาร์เก็ตเพลสที่ไม่อนุญาตรวมถึง:

- การเผยแพร่รายการจำนวนมากแบบเป็นกลุ่มที่ใช้ความพยายามต่ำ ซ้ำซ้อน เป็นตัวแทนชั่วคราว หรือ
  สร้างโดยเครื่อง ซึ่งดูเหมือนไม่มีคุณค่าจริงต่อผู้ใช้
- การท่วมพื้นผิวการค้นหาหรือหมวดหมู่ด้วย Skills หรือ Plugin ที่แทบเหมือนกัน
- การเผยแพร่รายการหลายร้อยรายการที่มีการใช้งาน การบำรุงรักษา ความชัดเจนของแหล่งที่มา
  หรือความแตกต่างที่มีความหมายเพียงเล็กน้อยหรือไม่มีเลย
- การเพิ่มจำนวนการติดตั้ง ดาวน์โหลด ดาว หรือเมตริกการมีส่วนร่วมอื่น ๆ
  อย่างไม่เป็นธรรมชาติผ่านระบบอัตโนมัติ วงจรการติดตั้งด้วยตนเอง บัญชีปลอม กิจกรรมประสานงาน
  การมีส่วนร่วมแบบจ่ายเงิน หรือพฤติกรรมอื่นที่ไม่เป็นธรรมชาติ
- การสร้างหรือหมุนเวียนบัญชีเพื่อเลี่ยงการดูแล การแบน ขีดจำกัดผู้เผยแพร่ หรือ
  การตรวจทานของมาร์เก็ตเพลส
- การทำให้ผู้ใช้เข้าใจผิดเกี่ยวกับความเป็นเจ้าของ แหล่งที่มา ความสามารถ ท่าทีด้านความปลอดภัย
  ข้อกำหนดการติดตั้ง หรือความเกี่ยวข้องกับโครงการหรือผู้เผยแพร่อื่น
- การอัปโหลดเนื้อหาที่ถูกซ่อน ลบ หรือบล็อกไปแล้วซ้ำ ๆ
  โดยไม่แก้ไขปัญหาต้นเหตุ

การเผยแพร่ปริมาณมากไม่ใช่การละเมิดโดยอัตโนมัติ แค็ตตาล็อกขนาดใหญ่ยอมรับได้
เมื่อรายการแตกต่างกันอย่างมีความหมาย อธิบายอย่างถูกต้อง ได้รับการบำรุงรักษา
และใช้งานโดยผู้ใช้จริง แค็ตตาล็อกขนาดใหญ่จะกลายเป็นปัญหาด้านความน่าเชื่อถือและความปลอดภัยเมื่อ
ปริมาณมาพร้อมกับรายการที่บาง ซ้ำซ้อน ทำให้เข้าใจผิด ไม่ได้รับการบำรุงรักษา หรือ
ได้รับการโปรโมตอย่างไม่เป็นธรรมชาติ

## สิทธิ์ในเนื้อหา

หากคุณเชื่อว่าเนื้อหาบน ClawHub ละเมิดลิขสิทธิ์หรือสิทธิอื่น ๆ ของคุณ ให้ใช้
[คำขอด้านสิทธิ์ในเนื้อหา](/clawhub/content-rights) อย่าใช้รายงานมาร์เก็ตเพลสปกติ
สำหรับการอ้างสิทธิ์ด้านลิขสิทธิ์หรือสิทธิ เว้นแต่รายการนั้นจะไม่ปลอดภัย
ประสงค์ร้าย หรือทำให้เข้าใจผิดด้วย

## การตรวจทานและการบังคับใช้

ClawHub อาจใช้การตรวจสอบอัตโนมัติ สัญญาณการละเมิดเชิงสถิติ รายงานของผู้ใช้ และ
การตรวจทานโดยเจ้าหน้าที่ เพื่อระบุเนื้อหาที่ไม่ปลอดภัยหรือพฤติกรรมการเผยแพร่ที่ละเมิด สัญญาณหนึ่ง
ไม่ได้พิสูจน์การละเมิดด้วยตัวเอง แต่ช่วยให้ ClawHub ตัดสินใจว่าสิ่งใดต้องได้รับการตรวจทาน

เราอาจ:

- ซ่อน พักไว้ ลบ ลบแบบกู้คืนได้ หรือหากชนิดทรัพยากรรองรับ
  ลบถาวรรายการที่ละเมิด
- บล็อกการดาวน์โหลดหรือการติดตั้งสำหรับรุ่นที่ไม่ปลอดภัย
- เพิกถอนโทเค็น API
- ลบเนื้อหาที่เกี่ยวข้องแบบกู้คืนได้
- จำกัดการเข้าถึงการเผยแพร่
- แบนผู้กระทำผิดซ้ำหรือร้ายแรง

เราไม่รับประกันการบังคับใช้แบบเตือนก่อนสำหรับการละเมิดที่ชัดเจน โปรดดู
[การดูแลและความปลอดภัยของบัญชี](/clawhub/moderation) สำหรับรายงาน การพักเพื่อดูแล
รายการที่ถูกซ่อน การแบน และสถานะบัญชี
