# การใช้งานที่ยอมรับได้

หน้านี้อธิบายประเภทของ Skills และเนื้อหาที่ ClawHub ยอมรับได้ รวมถึงเวิร์กโฟลว์การละเมิดที่ ClawHub จะไม่โฮสต์

กฎเหล่านี้ตั้งใจให้ใช้งานได้จริง เราให้ความสำคัญมากที่สุดกับเวิร์กโฟลว์การละเมิดแบบครบวงจร ไม่ใช่เพียงคีย์เวิร์ดที่แยกกันอยู่ หาก Skills ถูกสร้างมาเพื่อหลบเลี่ยงการป้องกัน ใช้แพลตฟอร์มในทางที่ผิด หลอกลวงผู้คน ละเมิดความเป็นส่วนตัว หรือเปิดทางให้เกิดพฤติกรรมที่ไม่ได้รับความยินยอม สิ่งนั้นไม่เหมาะอยู่บน ClawHub

## รูปแบบล่าสุดที่เราระบุชัดเจนว่ายอมรับได้

- งานฟรอนต์เอนด์และระบบดีไซน์ที่ใช้คอมโพเนนต์จริง โทเค็นเชิงความหมาย สถานะที่เข้าถึงได้ และโฟลว์ผู้ใช้ที่ผ่านการทดสอบแล้ว
- การประกอบ shadcn/ui ที่ใช้คอมโพเนนต์ซอร์สที่ติดตั้งแล้ว alias ของโปรเจกต์ และ variant ที่มีเอกสารกำกับ แทนมาร์กอัปเฉพาะกิจ
- การแปลง UI5 JavaScript เป็น TypeScript ที่คงความคิดเห็นไว้ ใช้ชนิด UI5 ที่เป็นรูปธรรม และทำให้อินเทอร์เฟซคอนโทรลที่สร้างขึ้นตรวจทานได้
- การตรวจทานความปลอดภัยเชิงป้องกัน เครื่องมือมอดเดอเรชัน และพรอมป์ต์ตรวจจับการละเมิดที่แสดงหลักฐานและทำให้ขอบเขตการอนุมัติของมนุษย์ชัดเจน
- เวิร์กโฟลว์อัตโนมัติที่อิงความยินยอมสำหรับบัญชีส่วนตัวหรือทีม โดยมีข้อมูลรับรองที่ชัดเจน การตั้งค่าที่โปร่งใส และโหมด dry-run หรือ preview
- เอกสาร runbook สำหรับการย้ายระบบ ยูทิลิตีสำหรับนักพัฒนา และ fixture ทดสอบที่มีขอบเขตตามซอฟต์แวร์ที่รองรับ

## ไม่ยอมรับ

- เวิร์กโฟลว์หลบเลี่ยงความปลอดภัยหรือเข้าถึงโดยไม่ได้รับอนุญาต
  - ตัวอย่าง: การข้ามการตรวจสอบสิทธิ์ การยึดบัญชี การข้าม CAPTCHA การหลบเลี่ยง Cloudflare หรือระบบป้องกันบอต การข้าม rate limit การสแครปแบบพรางตัวที่ออกแบบมาเพื่อเอาชนะการป้องกัน การยึดสายสนทนาสดหรือ agent การขโมยเซสชันที่นำกลับมาใช้ซ้ำได้ การอนุมัติโฟลว์จับคู่อัตโนมัติสำหรับผู้ใช้ที่ไม่ได้รับอนุมัติ

- การใช้แพลตฟอร์มในทางที่ผิดและการหลบเลี่ยงการแบน
  - ตัวอย่าง: บัญชีพรางตัวหลังถูกแบน การอุ่นบัญชีหรือฟาร์มบัญชี การสร้าง engagement ปลอม การเพาะ karma หรือผู้ติดตาม การทำงานอัตโนมัติหลายบัญชี การโพสต์จำนวนมาก บอตสแปม การทำงานอัตโนมัติใน marketplace หรือโซเชียลที่สร้างมาเพื่อหลีกเลี่ยงการตรวจจับ

- การฉ้อโกง กลโกง และเวิร์กโฟลว์ทางการเงินที่หลอกลวง
  - ตัวอย่าง: ใบรับรองปลอม ใบแจ้งหนี้ปลอม โฟลว์การชำระเงินที่หลอกลวง การติดต่อหลอกลวง หลักฐานทางสังคมปลอม เครื่องมือที่เปิดทางให้ใช้จ่ายหรือเรียกเก็บเงินโดยไม่มีการอนุมัติจากมนุษย์ที่ชัดเจนและการควบคุมที่โปร่งใส หรือเวิร์กโฟลว์อัตลักษณ์สังเคราะห์ที่สร้างมาเพื่อสร้างบัญชีสำหรับการฉ้อโกง

- การสแครป การเติมข้อมูล หรือการเฝ้าระวังที่ละเมิดความเป็นส่วนตัว
  - ตัวอย่าง: การสแครปรายละเอียดผู้ติดต่อในวงกว้างเพื่อสแปม การ doxxing การสะกดรอย การดึง lead ควบคู่กับการติดต่อโดยไม่ได้ร้องขอ การเฝ้าติดตามแบบแอบแฝง การค้นหาใบหน้าหรือการจับคู่ชีวมิติที่ใช้โดยไม่มีความยินยอมที่ชัดเจน หรือการซื้อ เผยแพร่ ดาวน์โหลด หรือนำข้อมูลรั่วไหลหรือชุดข้อมูลจากเหตุละเมิดไปใช้ในการปฏิบัติการ

- การสวมรอยโดยไม่ได้รับความยินยอมหรือการบิดเบือนอัตลักษณ์อย่างหลอกลวง
  - ตัวอย่าง: face swap, digital twins, ตัวตนปลอม, อินฟลูเอนเซอร์ที่ถูกโคลน หรือเครื่องมือบิดเบือนอัตลักษณ์อื่น ๆ ที่ใช้เพื่อสวมรอยหรือทำให้เข้าใจผิด

- เนื้อหาทางเพศอย่างโจ่งแจ้งและการสร้างเนื้อหาผู้ใหญ่ที่ปิดระบบความปลอดภัย
  - ตัวอย่าง: การสร้างรูปภาพ/วิดีโอ/เนื้อหา NSFW, wrapper เนื้อหาผู้ใหญ่รอบ API ของบุคคลที่สาม หรือ Skills ที่มีวัตถุประสงค์หลักคือเนื้อหาทางเพศอย่างโจ่งแจ้ง

- ข้อกำหนดการรันที่ซ่อนอยู่ ไม่ปลอดภัย หรือทำให้เข้าใจผิด
  - ตัวอย่าง: คำสั่งติดตั้งที่ทำให้อ่านยาก, `curl | sh`, ข้อกำหนดเกี่ยวกับ secret ที่ไม่ได้ประกาศ, การใช้ private key ที่ไม่ได้ประกาศ, การรัน `npx @latest` จากระยะไกลโดยไม่มีความสามารถในการตรวจทานที่ชัดเจน, metadata ที่ทำให้เข้าใจผิดซึ่งซ่อนสิ่งที่ Skills ต้องใช้จริงในการรัน

## รูปแบบล่าสุดที่เราระบุชัดเจนว่าไม่ยอมรับ

- “สร้างบัญชีผู้ขายแบบพรางตัวหลังถูกแบนจาก marketplace”
- “แก้ไขการจับคู่ Telegram เพื่อให้ผู้ใช้ที่ไม่ได้รับอนุมัติได้รับรหัสจับคู่โดยอัตโนมัติ”
- “เพาะบัญชี Reddit/Twitter ด้วยระบบอัตโนมัติที่ตรวจจับไม่ได้”
- “สร้างใบรับรองวิชาชีพหรือใบแจ้งหนี้สำหรับการใช้งานตามอำเภอใจ”
- “สร้างเนื้อหา NSFW โดยปิดการตรวจสอบความปลอดภัย”
- “สแครป lead, เติมข้อมูลผู้ติดต่อ และเริ่ม cold outreach ในวงกว้าง”
- “ซื้อ เผยแพร่ หรือดาวน์โหลดข้อมูลรั่วไหลหรือชุดข้อมูลจากเหตุละเมิด”
- “สร้างบัญชีอีเมลหรือโซเชียลจำนวนมากด้วยอัตลักษณ์สังเคราะห์หรือการแก้ CAPTCHA”

## หมายเหตุสำหรับผู้ตรวจทาน

- บริบทมีความสำคัญ หัวข้อเดียวกันอาจถูกต้องตามสมควรในบริบทเชิงป้องกันที่แคบหรืออิงความยินยอม และไม่อาจยอมรับได้เมื่อถูกบรรจุเป็นเวิร์กโฟลว์การละเมิด
- เราควรโน้มเอียงไปทางการดำเนินการเมื่อ Skills ได้รับการปรับให้เหมาะอย่างชัดเจนสำหรับการหลบเลี่ยง การหลอกลวง หรือการใช้งานโดยไม่ได้รับความยินยอม
- การอัปโหลดซ้ำในหมวดหมู่เหล่านี้เป็นเหตุให้ซ่อนเนื้อหาและแบนบัญชีได้

## การบังคับใช้

- เราอาจซ่อน ลบ หรือลบถาวร Skills ที่ละเมิด
- เราอาจเพิกถอน token, soft-delete เนื้อหาที่เกี่ยวข้อง และแบนผู้กระทำผิดซ้ำหรือร้ายแรง
- เราไม่รับประกันว่าจะเตือนก่อนเสมอในการบังคับใช้กับการละเมิดที่เห็นได้ชัด
