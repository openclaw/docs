# การใช้งานที่ยอมรับได้

ClawHub โฮสต์ Skills, Plugin, แพ็กเกจ และเมทาดาทาของมาร์เก็ตเพลซสำหรับ OpenClaw
ใช้หน้านี้เพื่อตัดสินใจว่าเนื้อหาหรือพฤติกรรมการเผยแพร่ควรอยู่บน
ClawHub หรือไม่

กฎเหล่านี้ใช้กับสิ่งที่รายการทำ สิ่งที่ขอให้ผู้ใช้เรียกใช้ วิธีที่รายการ
นำเสนอตัวเอง และวิธีที่ผู้เผยแพร่ใช้พื้นผิวการค้นพบ การติดตั้ง และ
ความน่าเชื่อถือของ ClawHub สำหรับสถานะการกลั่นกรองและสถานะบัญชี โปรดดู
[การกลั่นกรองและความปลอดภัยของบัญชี](/clawhub/moderation) สำหรับการร้องเรียนเรื่องลิขสิทธิ์หรือสิทธิอื่น ๆ
โปรดดู [คำขอด้านสิทธิ์เนื้อหา](/clawhub/content-rights)

## เนื้อหาที่อนุญาต

ClawHub ยินดีรับเนื้อหาที่เป็นประโยชน์ เข้าใจได้ และเผยแพร่ด้วยเจตนาดี

| หมวดหมู่                                         | อนุญาตเมื่อ                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| ประสิทธิภาพการทำงานของนักพัฒนา                           | รายการช่วยให้ผู้ใช้สร้าง ทดสอบ ย้ายระบบ ดีบัก จัดทำเอกสาร หรือดำเนินงานซอฟต์แวร์                                               |
| เวิร์กโฟลว์ UI, ข้อมูล และระบบอัตโนมัติ               | ขอบเขตชัดเจน ระบุข้อมูลรับรองที่จำเป็นไว้อย่างชัดเจน และการดำเนินการที่มีความเสี่ยงมีเส้นทางการตรวจทาน dry-run แสดงตัวอย่าง หรือยืนยัน |
| การรักษาความปลอดภัยเชิงป้องกัน การกลั่นกรอง และการตรวจทานการละเมิด | เครื่องมือถูกวางกรอบสำหรับการตรวจทานที่ได้รับอนุญาต เก็บรักษาหลักฐาน และทำให้ขอบเขตการอนุมัติโดยมนุษย์ชัดเจน                          |
| เวิร์กโฟลว์ส่วนบุคคลหรือทีม                       | เวิร์กโฟลว์ใช้บัญชีที่อิงตามความยินยอม การตั้งค่าที่โปร่งใส และสิทธิ์ที่ชัดเจน                                            |
| แคตตาล็อกที่ได้รับการดูแล                              | แต่ละรายการแตกต่างกัน มีประโยชน์ อธิบายอย่างถูกต้อง และได้รับการดูแลอย่างสมเหตุสมผล                                                |

บริบทมีความสำคัญ หัวข้อเดียวกันอาจยอมรับได้ในบริบทเชิงป้องกันที่แคบหรือ
อิงตามความยินยอม และยอมรับไม่ได้เมื่อถูกแพ็กเกจเป็นเวิร์กโฟลว์การละเมิด

## เนื้อหาที่ไม่อนุญาต

ClawHub ไม่โฮสต์เนื้อหาที่มีวัตถุประสงค์หลักเป็นการละเมิด การหลอกลวง การดำเนินการที่ไม่ปลอดภัย
หรือการละเมิดสิทธิ์

| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| การละเมิดแพลตฟอร์มและการเลี่ยงการแบน                              | บัญชีลับหลังถูกแบน การอุ่นหรือทำฟาร์มบัญชี การมีส่วนร่วมปลอม ระบบอัตโนมัติหลายบัญชี การโพสต์จำนวนมาก บอทสแปม หรือระบบอัตโนมัติที่สร้างขึ้นเพื่อหลีกเลี่ยงการตรวจจับ                                                                                                                                          |
| การฉ้อโกง การหลอกลวง และเวิร์กโฟลว์ทางการเงินที่ทำให้เข้าใจผิด             | ใบรับรองหรือใบแจ้งหนี้ปลอม โฟลว์การชำระเงินที่หลอกลวง การติดต่อหลอกลวง หลักฐานทางสังคมปลอม เวิร์กโฟลว์อัตลักษณ์สังเคราะห์เพื่อการฉ้อโกง หรือเครื่องมือใช้จ่าย/เรียกเก็บเงินที่ไม่มีการอนุมัติโดยมนุษย์อย่างชัดเจน                                                                                                                    |
| การเสริมข้อมูลหรือการเฝ้าระวังที่ละเมิดความเป็นส่วนตัว                 | การดึงข้อมูลผู้ติดต่อเพื่อสแปม การเปิดเผยข้อมูลส่วนบุคคล การสะกดรอย การดึงรายชื่อผู้สนใจที่จับคู่กับการติดต่อโดยไม่ได้ร้องขอ การเฝ้าติดตามแบบปกปิด การจับคู่ไบโอเมตริกโดยไม่ยินยอม หรือการใช้ข้อมูลรั่วไหลหรือชุดข้อมูลจากการละเมิด                                                                                                                  |
| การแอบอ้างหรือจัดการอัตลักษณ์โดยไม่ได้รับความยินยอม       | การสลับใบหน้า ฝาแฝดดิจิทัล อินฟลูเอนเซอร์ที่ถูกโคลน บุคลิกปลอม หรือเครื่องมืออื่นที่ใช้เพื่อแอบอ้างหรือทำให้เข้าใจผิด                                                                                                                                                                                                 |

## พฤติกรรมมาร์เก็ตเพลซที่ไม่อนุญาต

ClawHub ยังตรวจทานวิธีที่ผู้เผยแพร่ใช้มาร์เก็ตเพลซด้วย ห้ามใช้ ClawHub เพื่อ
จัดการการค้นพบ เมตริก สัญญาณความน่าเชื่อถือ ระบบกลั่นกรอง หรือความสนใจของผู้ใช้

พฤติกรรมมาร์เก็ตเพลซที่ไม่อนุญาตรวมถึง:

- เผยแพร่รายการจำนวนมากแบบเหมารวมที่ใช้ความพยายามต่ำ ซ้ำซ้อน เป็นตัวแทนที่ว่างเปล่า หรือ
  สร้างโดยเครื่อง ซึ่งดูเหมือนไม่มีคุณค่าจริงต่อผู้ใช้
- ถมพื้นผิวการค้นหาหรือหมวดหมู่ด้วย Skills หรือ Plugin ที่แทบเหมือนกัน
- เผยแพร่รายการหลายร้อยรายการที่มีการใช้งาน การดูแลรักษา ความชัดเจนของแหล่งที่มา
  หรือความแตกต่างที่มีความหมายเพียงเล็กน้อยหรือไม่มีเลย
- เพิ่มจำนวนการติดตั้ง ดาวน์โหลด ดาว หรือเมตริกการมีส่วนร่วมอื่น ๆ
  อย่างไม่เป็นธรรมชาติผ่านระบบอัตโนมัติ วงวนการติดตั้งด้วยตนเอง บัญชีปลอม กิจกรรมที่ประสานกัน
  การมีส่วนร่วมแบบจ่ายเงิน หรือพฤติกรรมอื่นที่ไม่เป็นธรรมชาติ
- สร้างหรือหมุนเวียนบัญชีเพื่อหลีกเลี่ยงการกลั่นกรอง การแบน ขีดจำกัดผู้เผยแพร่ หรือ
  การตรวจทานของมาร์เก็ตเพลซ
- ทำให้ผู้ใช้เข้าใจผิดเกี่ยวกับความเป็นเจ้าของ แหล่งที่มา ความสามารถ สถานะความปลอดภัย
  ข้อกำหนดการติดตั้ง หรือความเกี่ยวข้องกับโปรเจ็กต์หรือผู้เผยแพร่อื่น
- อัปโหลดเนื้อหาซ้ำ ๆ ที่เคยถูกซ่อน ลบ หรือบล็อกแล้ว
  โดยไม่แก้ไขปัญหาต้นเหตุ

การเผยแพร่ปริมาณมากไม่ได้เป็นการละเมิดโดยอัตโนมัติ แคตตาล็อกขนาดใหญ่ยอมรับได้
เมื่อรายการแตกต่างกันอย่างมีความหมาย อธิบายอย่างถูกต้อง ได้รับการดูแล
และถูกใช้งานโดยผู้ใช้จริง แคตตาล็อกขนาดใหญ่กลายเป็นปัญหาด้านความน่าเชื่อถือและความปลอดภัยเมื่อ
ปริมาณมาพร้อมกับรายการที่บาง ซ้ำซ้อน ทำให้เข้าใจผิด ไม่ได้รับการดูแล หรือ
ได้รับการโปรโมตอย่างไม่เป็นธรรมชาติ

## สิทธิ์เนื้อหา

หากคุณเชื่อว่าเนื้อหาบน ClawHub ละเมิดลิขสิทธิ์หรือสิทธิอื่น ๆ ของคุณ ให้ใช้
[คำขอด้านสิทธิ์เนื้อหา](/clawhub/content-rights) อย่าใช้รายงานมาร์เก็ตเพลซทั่วไป
สำหรับการร้องเรียนเรื่องลิขสิทธิ์หรือสิทธิ์ เว้นแต่รายการนั้นจะไม่ปลอดภัย
เป็นอันตราย หรือทำให้เข้าใจผิดด้วย

## การตรวจทานและการบังคับใช้

ClawHub อาจใช้การตรวจสอบอัตโนมัติ สัญญาณการละเมิดเชิงสถิติ รายงานจากผู้ใช้ และ
การตรวจทานโดยเจ้าหน้าที่เพื่อระบุเนื้อหาที่ไม่ปลอดภัยหรือพฤติกรรมการเผยแพร่ที่เป็นการละเมิด สัญญาณหนึ่ง
ไม่ได้พิสูจน์การละเมิดด้วยตัวเอง แต่ช่วยให้ ClawHub ตัดสินใจว่าสิ่งใดต้องได้รับการตรวจทาน

เราอาจ:

- ซ่อน ระงับ ลบ ลบแบบ soft-delete หรือในกรณีที่รองรับสำหรับประเภททรัพยากรนั้น
  ลบแบบ hard-delete รายการที่ละเมิด
- บล็อกการดาวน์โหลดหรือการติดตั้งสำหรับรุ่นที่ไม่ปลอดภัย
- เพิกถอนโทเค็น API
- ลบแบบ soft-delete เนื้อหาที่เกี่ยวข้อง
- จำกัดสิทธิ์การเผยแพร่
- แบนผู้กระทำผิดซ้ำหรือร้ายแรง

เราไม่รับประกันว่าจะบังคับใช้แบบเตือนก่อนสำหรับการละเมิดที่ชัดเจน โปรดดู
[การกลั่นกรองและความปลอดภัยของบัญชี](/clawhub/moderation) สำหรับรายงาน การระงับเพื่อกลั่นกรอง
รายการที่ถูกซ่อน การแบน และสถานะบัญชี
