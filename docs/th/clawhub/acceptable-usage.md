# การใช้งานที่ยอมรับได้

หน้านี้อธิบายประเภทของ Skills และเนื้อหาที่ ClawHub ยอมรับได้ รวมถึงเวิร์กโฟลว์การใช้งานในทางที่ผิดที่ ClawHub จะไม่โฮสต์

กฎเหล่านี้ตั้งใจให้ใช้ได้จริง เราให้ความสำคัญสูงสุดกับเวิร์กโฟลว์การใช้งานในทางที่ผิดแบบครบวงจร ไม่ใช่แค่คีย์เวิร์ดที่แยกออกมาโดด ๆ หาก Skill ถูกสร้างขึ้นเพื่อหลบเลี่ยงการป้องกัน ใช้แพลตฟอร์มในทางที่ผิด หลอกลวงผู้คน ละเมิดความเป็นส่วนตัว หรือเปิดทางให้เกิดพฤติกรรมที่ไม่ได้รับความยินยอม Skill นั้นไม่ควรอยู่บน ClawHub

## รูปแบบล่าสุดที่เรายืนยันชัดเจนว่ายอมรับได้

- งานฟรอนต์เอนด์และระบบออกแบบที่ใช้คอมโพเนนต์จริง semantic tokens สถานะที่เข้าถึงได้ และโฟลว์ผู้ใช้ที่ทดสอบแล้ว
- การประกอบ shadcn/ui ที่ใช้คอมโพเนนต์ซอร์สที่ติดตั้งแล้ว นามแฝงของโปรเจกต์ และตัวแปรที่มีเอกสารกำกับ แทนการใช้มาร์กอัปเฉพาะกิจ
- การแปลง UI5 JavaScript เป็น TypeScript ที่รักษาคอมเมนต์ไว้ ใช้ชนิด UI5 ที่ชัดเจน และทำให้อินเทอร์เฟซคอนโทรลที่สร้างขึ้นสามารถตรวจทานได้
- การตรวจทานความปลอดภัยเชิงป้องกัน เครื่องมือดูแลเนื้อหา และพรอมป์ตรวจจับการใช้งานในทางที่ผิดที่แสดงหลักฐานและรักษาขอบเขตการอนุมัติของมนุษย์ไว้อย่างชัดเจน
- การทำงานอัตโนมัติของเวิร์กโฟลว์ตามความยินยอมสำหรับบัญชีส่วนตัวหรือทีม โดยมีข้อมูลรับรองที่ชัดเจน การตั้งค่าที่โปร่งใส และโหมด dry-run หรือพรีวิว
- เอกสาร runbook สำหรับการย้ายระบบ ยูทิลิตีสำหรับนักพัฒนา และ fixture สำหรับทดสอบที่จำกัดขอบเขตอยู่กับซอฟต์แวร์ที่สิ่งเหล่านั้นสนับสนุน

## ไม่ยอมรับ

- เวิร์กโฟลว์หลบเลี่ยงความปลอดภัยหรือเข้าถึงโดยไม่ได้รับอนุญาต
  - ตัวอย่าง: การข้ามการยืนยันตัวตน การยึดบัญชี การข้าม CAPTCHA การหลบเลี่ยง Cloudflare หรือระบบป้องกันบอต การข้าม rate-limit การ scraping แบบแฝงตัวที่ออกแบบมาเพื่อเอาชนะการป้องกัน การยึดสายสนทนาสดหรือ agent การขโมยเซสชันที่นำกลับมาใช้ซ้ำได้ การอนุมัติโฟลว์จับคู่อัตโนมัติสำหรับผู้ใช้ที่ไม่ได้รับอนุมัติ

- การใช้แพลตฟอร์มในทางที่ผิดและการหลบเลี่ยงการแบน
  - ตัวอย่าง: บัญชีแฝงตัวหลังถูกแบน การอุ่นบัญชีหรือฟาร์มบัญชี การมีส่วนร่วมปลอม การปั้น karma หรือผู้ติดตาม การทำงานอัตโนมัติหลายบัญชี การโพสต์จำนวนมาก บอตสแปม การทำงานอัตโนมัติของมาร์เก็ตเพลสหรือโซเชียลที่สร้างมาเพื่อหลบเลี่ยงการตรวจจับ

- การฉ้อโกง การหลอกลวง และเวิร์กโฟลว์ทางการเงินที่ทำให้เข้าใจผิด
  - ตัวอย่าง: ใบรับรองปลอม ใบแจ้งหนี้ปลอม โฟลว์การชำระเงินที่หลอกลวง การติดต่อเพื่อหลอกลวง social proof ปลอม เครื่องมือที่เปิดทางให้ใช้จ่ายหรือเรียกเก็บเงินโดยไม่มีการอนุมัติจากมนุษย์อย่างชัดเจนและไม่มีการควบคุมที่โปร่งใส หรือเวิร์กโฟลว์ตัวตนสังเคราะห์ที่สร้างขึ้นเพื่อสร้างบัญชีสำหรับการฉ้อโกง

- การ scraping การเพิ่มข้อมูล หรือการสอดส่องที่ละเมิดความเป็นส่วนตัว
  - ตัวอย่าง: การ scraping รายละเอียดการติดต่อในปริมาณมากเพื่อสแปม การ doxxing การสะกดรอย การดึงรายชื่อลูกค้าเป้าหมายควบคู่กับการติดต่อที่ไม่ได้ร้องขอ การเฝ้าติดตามแบบลับ การค้นหาใบหน้าหรือการจับคู่ชีวมิติที่ใช้โดยไม่มีความยินยอมชัดเจน หรือการซื้อ เผยแพร่ ดาวน์โหลด หรือนำข้อมูลที่รั่วไหลหรือชุดข้อมูลจากการละเมิดไปใช้งานเชิงปฏิบัติการ

- การสวมรอยที่ไม่ได้รับความยินยอมหรือการบิดเบือนตัวตนอย่างหลอกลวง
  - ตัวอย่าง: face swap, digital twins, persona ปลอม, influencer ที่ถูกโคลน หรือเครื่องมือจัดการตัวตนอื่น ๆ ที่ใช้เพื่อสวมรอยหรือทำให้เข้าใจผิด

- เนื้อหาทางเพศอย่างโจ่งแจ้งและการสร้างเนื้อหาสำหรับผู้ใหญ่โดยปิดระบบความปลอดภัย
  - ตัวอย่าง: การสร้างภาพ/วิดีโอ/เนื้อหา NSFW, wrapper เนื้อหาสำหรับผู้ใหญ่รอบ API ของบุคคลที่สาม หรือ Skills ที่มีวัตถุประสงค์หลักเป็นเนื้อหาทางเพศอย่างโจ่งแจ้ง

- ข้อกำหนดการรันที่ซ่อนอยู่ ไม่ปลอดภัย หรือทำให้เข้าใจผิด
  - ตัวอย่าง: คำสั่งติดตั้งที่ทำให้อ่านยาก, `curl | sh`, ข้อกำหนดเรื่อง secret ที่ไม่ได้ประกาศ, การใช้ private-key ที่ไม่ได้ประกาศ, การรัน `npx @latest` จากระยะไกลโดยไม่มีความสามารถในการตรวจทานที่ชัดเจน, metadata ที่ทำให้เข้าใจผิดและซ่อนสิ่งที่ Skill จำเป็นต้องใช้จริง ๆ เพื่อรัน

## รูปแบบล่าสุดที่เรายืนยันชัดเจนว่าไม่ยอมรับ

- “สร้างบัญชีผู้ขายแบบแฝงตัวหลังถูกแบนจากมาร์เก็ตเพลส”
- “แก้ไขการจับคู่ Telegram เพื่อให้ผู้ใช้ที่ไม่ได้รับอนุมัติได้รับรหัสจับคู่โดยอัตโนมัติ”
- “ปั้นบัญชี Reddit/Twitter ด้วยระบบอัตโนมัติที่ตรวจจับไม่ได้”
- “สร้างใบรับรองวิชาชีพหรือใบแจ้งหนี้สำหรับการใช้งานตามอำเภอใจ”
- “สร้างเนื้อหา NSFW โดยปิดการตรวจสอบความปลอดภัย”
- “Scrape ลูกค้าเป้าหมาย เพิ่มข้อมูลติดต่อ และเริ่มการติดต่อแบบ cold outreach ในปริมาณมาก”
- “ซื้อ เผยแพร่ หรือดาวน์โหลดข้อมูลที่รั่วไหลหรือชุดข้อมูลจากการละเมิด”
- “สร้างบัญชีอีเมลหรือโซเชียลจำนวนมากด้วยตัวตนสังเคราะห์หรือการแก้ CAPTCHA”

## หมายเหตุสำหรับผู้ตรวจทาน

- บริบทมีความสำคัญ หัวข้อเดียวกันอาจชอบด้วยเหตุผลในกรอบเชิงป้องกันที่แคบหรือกรอบตามความยินยอม และไม่สามารถยอมรับได้เมื่อถูกแพ็กเป็นเวิร์กโฟลว์การใช้งานในทางที่ผิด
- เราควรเอนเอียงไปสู่การดำเนินการเมื่อ Skill ถูกปรับให้เหมาะกับการหลบเลี่ยง การหลอกลวง หรือการใช้งานที่ไม่ได้รับความยินยอมอย่างชัดเจน
- การอัปโหลดซ้ำในหมวดหมู่เหล่านี้เป็นเหตุให้ซ่อนเนื้อหาและแบนบัญชีได้

## การบังคับใช้

- เราอาจซ่อน ลบ หรือ hard-delete Skills ที่ละเมิด
- เราอาจเพิกถอน token, soft-delete เนื้อหาที่เกี่ยวข้อง และแบนผู้ละเมิดซ้ำหรือผู้ละเมิดร้ายแรง
- เราไม่รับประกันว่าจะเตือนก่อนบังคับใช้กับการใช้งานในทางที่ผิดที่ชัดเจน
