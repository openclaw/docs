# การใช้งานที่ยอมรับได้

หน้านี้อธิบายชนิดของ Skills และเนื้อหาที่ ClawHub ยอมรับได้ รวมถึงเวิร์กโฟลว์การใช้งานในทางที่ผิดที่ ClawHub จะไม่โฮสต์

กฎเหล่านี้ตั้งใจให้ใช้งานได้จริง เราให้ความสำคัญมากที่สุดกับเวิร์กโฟลว์การใช้งานในทางที่ผิดแบบครบวงจร ไม่ใช่เพียงคีย์เวิร์ดที่แยกออกมาเดี่ยว ๆ หาก Skill ถูกสร้างมาเพื่อหลบเลี่ยงการป้องกัน ใช้แพลตฟอร์มในทางที่ผิด หลอกลวงผู้คน ละเมิดความเป็นส่วนตัว หรือเปิดทางให้เกิดพฤติกรรมที่ไม่ได้รับความยินยอม Skill นั้นไม่ควรอยู่บน ClawHub

## รูปแบบล่าสุดที่เราระบุไว้อย่างชัดเจนว่ายอมรับได้

- งานฟรอนต์เอนด์และ design system ที่ใช้คอมโพเนนต์จริง โทเค็นเชิงความหมาย สถานะที่เข้าถึงได้ และโฟลว์ผู้ใช้ที่ผ่านการทดสอบแล้ว
- การประกอบ shadcn/ui ที่ใช้คอมโพเนนต์ซอร์สที่ติดตั้งแล้ว อะเลียสของโปรเจกต์ และตัวแปรที่มีเอกสารกำกับ แทนมาร์กอัปเฉพาะกิจ
- การแปลง UI5 JavaScript เป็น TypeScript ที่รักษาคอมเมนต์ ใช้ชนิด UI5 ที่เป็นรูปธรรม และทำให้อินเทอร์เฟซคอนโทรลที่สร้างขึ้นสามารถตรวจทานได้
- การตรวจทานความปลอดภัยเชิงป้องกัน เครื่องมือมอดเดอเรชัน และพรอมต์ตรวจจับการใช้งานในทางที่ผิดที่แสดงหลักฐานและคงขอบเขตการอนุมัติของมนุษย์ไว้อย่างชัดเจน
- การทำงานอัตโนมัติของเวิร์กโฟลว์ตามความยินยอมสำหรับบัญชีส่วนตัวหรือบัญชีทีม โดยมีข้อมูลประจำตัวที่ชัดเจน การตั้งค่าที่โปร่งใส และโหมด dry-run หรือพรีวิว
- เอกสาร runbook การย้ายข้อมูล ยูทิลิตีสำหรับนักพัฒนา และ fixture สำหรับทดสอบที่จำกัดขอบเขตอยู่ในซอฟต์แวร์ที่สิ่งเหล่านั้นสนับสนุน

## ไม่ยอมรับ

- เวิร์กโฟลว์หลบเลี่ยงความปลอดภัยหรือเข้าถึงโดยไม่ได้รับอนุญาต
  - ตัวอย่าง: การบายพาสการยืนยันตัวตน การยึดบัญชี การบายพาส CAPTCHA การหลบเลี่ยง Cloudflare หรือระบบป้องกันบอต การบายพาสขีดจำกัดอัตรา การสแครปแบบล่องหนที่ออกแบบมาเพื่อเอาชนะการป้องกัน การยึดสายสดหรือเอเจนต์ การขโมยเซสชันที่ใช้ซ้ำได้ การอนุมัติโฟลว์จับคู่อัตโนมัติสำหรับผู้ใช้ที่ไม่ได้รับอนุมัติ

- การใช้แพลตฟอร์มในทางที่ผิดและการหลบเลี่ยงการแบน
  - ตัวอย่าง: บัญชีล่องหนหลังถูกแบน การอุ่นหรือฟาร์มบัญชี การมีส่วนร่วมปลอม การปั้น karma หรือผู้ติดตาม การทำงานอัตโนมัติหลายบัญชี การโพสต์จำนวนมาก บอตสแปม ระบบอัตโนมัติของ marketplace หรือโซเชียลที่สร้างมาเพื่อหลีกเลี่ยงการตรวจจับ

- การฉ้อโกง การหลอกลวง และเวิร์กโฟลว์ทางการเงินที่หลอกลวง
  - ตัวอย่าง: ใบรับรองปลอม ใบแจ้งหนี้ปลอม โฟลว์การชำระเงินที่หลอกลวง การติดต่อเพื่อหลอกลวง social proof ปลอม เครื่องมือที่เปิดทางให้ใช้จ่ายหรือเรียกเก็บเงินโดยไม่มีการอนุมัติจากมนุษย์ที่ชัดเจนและการควบคุมที่โปร่งใส หรือเวิร์กโฟลว์ตัวตนสังเคราะห์ที่สร้างขึ้นเพื่อสร้างบัญชีสำหรับการฉ้อโกง

- การสแครป การเติมข้อมูล หรือการเฝ้าระวังที่ละเมิดความเป็นส่วนตัว
  - ตัวอย่าง: การสแครปรายละเอียดผู้ติดต่อในปริมาณมากเพื่อสแปม การเปิดเผยข้อมูลส่วนตัว การสะกดรอยตาม การดึงลีดที่จับคู่กับการติดต่อโดยไม่ได้รับเชิญ การติดตามแบบแอบแฝง การค้นหาใบหน้าหรือการจับคู่ไบโอเมตริกที่ใช้โดยไม่มีความยินยอมชัดเจน หรือการซื้อ เผยแพร่ ดาวน์โหลด หรือนำข้อมูลรั่วไหลหรือชุดข้อมูลจากเหตุละเมิดไปใช้ปฏิบัติการ

- การแอบอ้างโดยไม่ได้รับความยินยอมหรือการบิดเบือนตัวตนเพื่อหลอกลวง
  - ตัวอย่าง: การสลับใบหน้า digital twins บุคคลปลอม อินฟลูเอนเซอร์ที่ถูกโคลน หรือเครื่องมือบิดเบือนตัวตนอื่น ๆ ที่ใช้เพื่อแอบอ้างหรือทำให้เข้าใจผิด

- เนื้อหาทางเพศอย่างโจ่งแจ้งและการสร้างเนื้อหาผู้ใหญ่ที่ปิดระบบความปลอดภัย
  - ตัวอย่าง: การสร้างรูปภาพ/วิดีโอ/เนื้อหา NSFW wrapper เนื้อหาผู้ใหญ่รอบ API ของบุคคลที่สาม หรือ Skills ที่มีวัตถุประสงค์หลักเป็นเนื้อหาทางเพศอย่างโจ่งแจ้ง

- ข้อกำหนดการรันที่ซ่อนอยู่ ไม่ปลอดภัย หรือทำให้เข้าใจผิด
  - ตัวอย่าง: คำสั่งติดตั้งที่ทำให้อ่านยาก `curl | sh` ข้อกำหนด secret ที่ไม่ได้ประกาศ การใช้ private key ที่ไม่ได้ประกาศ การรัน `npx @latest` ระยะไกลโดยไม่มีความสามารถในการตรวจทานที่ชัดเจน เมตาดาตาที่ทำให้เข้าใจผิดและปิดบังสิ่งที่ Skill ต้องใช้จริงเพื่อรัน

## รูปแบบล่าสุดที่เราระบุไว้อย่างชัดเจนว่าไม่ยอมรับ

- “สร้างบัญชีผู้ขายแบบล่องหนหลังถูกแบนจาก marketplace”
- “แก้ไขการจับคู่ Telegram เพื่อให้ผู้ใช้ที่ไม่ได้รับอนุมัติได้รับรหัสจับคู่โดยอัตโนมัติ”
- “ปั้นบัญชี Reddit/Twitter ด้วยระบบอัตโนมัติที่ตรวจจับไม่ได้”
- “สร้างใบรับรองวิชาชีพหรือใบแจ้งหนี้สำหรับการใช้งานตามอำเภอใจ”
- “สร้างเนื้อหา NSFW โดยปิดการตรวจสอบความปลอดภัย”
- “สแครปลีด เติมข้อมูลผู้ติดต่อ และเริ่มการติดต่อแบบ cold outreach ในปริมาณมาก”
- “ซื้อ เผยแพร่ หรือดาวน์โหลดข้อมูลรั่วไหลหรือชุดข้อมูลจากเหตุละเมิด”
- “สร้างบัญชีอีเมลหรือโซเชียลจำนวนมากด้วยตัวตนสังเคราะห์หรือการแก้ CAPTCHA”

## หมายเหตุสำหรับผู้ตรวจทาน

- บริบทมีความสำคัญ หัวข้อเดียวกันอาจถูกต้องตามกฎในบริบทเชิงป้องกันที่แคบหรือบริบทตามความยินยอม และยอมรับไม่ได้เมื่อบรรจุเป็นเวิร์กโฟลว์การใช้งานในทางที่ผิด
- เราควรเอนเอียงไปทางการดำเนินการเมื่อ Skill ถูกปรับแต่งอย่างชัดเจนเพื่อการหลบเลี่ยง การหลอกลวง หรือการใช้งานที่ไม่ได้รับความยินยอม
- การอัปโหลดซ้ำในหมวดหมู่เหล่านี้เป็นเหตุให้ซ่อนเนื้อหาและแบนบัญชีได้

## การบังคับใช้

- เราอาจซ่อน ลบ หรือ hard-delete Skills ที่ละเมิดกฎ
- เราอาจเพิกถอนโทเค็น soft-delete เนื้อหาที่เกี่ยวข้อง และแบนผู้กระทำผิดซ้ำหรือผู้กระทำผิดร้ายแรง
- เราไม่รับประกันว่าจะมีการเตือนก่อนบังคับใช้กับการใช้งานในทางที่ผิดที่ชัดเจน
