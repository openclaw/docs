# การใช้งานที่ยอมรับได้

หน้านี้อธิบายประเภทของสกิลและเนื้อหาที่ ClawHub ยอมรับได้ รวมถึงเวิร์กโฟลว์การละเมิดที่ ClawHub จะไม่โฮสต์

กฎเหล่านี้ตั้งใจให้ใช้งานได้จริง เราให้ความสำคัญสูงสุดกับเวิร์กโฟลว์การละเมิดแบบครบวงจร ไม่ใช่แค่คีย์เวิร์ดที่แยกออกมาเป็นคำ ๆ หากสกิลถูกสร้างมาเพื่อหลบเลี่ยงการป้องกัน ละเมิดแพลตฟอร์ม หลอกลวงผู้คน บุกรุกความเป็นส่วนตัว หรือเปิดทางให้เกิดพฤติกรรมที่ไม่ได้รับความยินยอม สกิลนั้นไม่ควรอยู่บน ClawHub

## รูปแบบล่าสุดที่เรายอมรับอย่างชัดเจน

- งานฟรอนต์เอนด์และระบบดีไซน์ที่ใช้คอมโพเนนต์จริง โทเคนเชิงความหมาย สถานะที่เข้าถึงได้ และโฟลว์ผู้ใช้ที่ผ่านการทดสอบแล้ว
- การประกอบ shadcn/ui ที่ใช้คอมโพเนนต์ซอร์สที่ติดตั้งไว้ alias ของโปรเจกต์ และ variant ที่มีเอกสารกำกับ แทนมาร์กอัปเฉพาะครั้งเดียว
- การแปลง UI5 JavaScript เป็น TypeScript ที่คงคอมเมนต์ไว้ ใช้ชนิด UI5 ที่เป็นรูปธรรม และทำให้อินเทอร์เฟซของคอนโทรลที่สร้างขึ้นตรวจทานได้
- การรีวิวความปลอดภัยเชิงป้องกัน เครื่องมือมอดเดอเรชัน และพรอมป์ต์ตรวจจับการละเมิดที่แสดงหลักฐานและทำให้ขอบเขตการอนุมัติโดยมนุษย์ชัดเจน
- ระบบอัตโนมัติของเวิร์กโฟลว์ตามความยินยอมสำหรับบัญชีส่วนตัวหรือทีม โดยมีข้อมูลรับรองที่ชัดเจน การตั้งค่าที่โปร่งใส และโหมดทดลองรันหรือแสดงตัวอย่าง
- เอกสาร คู่มือการย้ายระบบ เครื่องมือสำหรับนักพัฒนา และฟิกซ์เจอร์ทดสอบที่จำกัดขอบเขตตามซอฟต์แวร์ที่รองรับ

## ไม่ยอมรับ

- เวิร์กโฟลว์ที่ข้ามระบบความปลอดภัยหรือเข้าถึงโดยไม่ได้รับอนุญาต
  - ตัวอย่าง: การข้ามการยืนยันตัวตน การยึดบัญชี การข้าม CAPTCHA การหลบเลี่ยง Cloudflare หรือระบบป้องกันบอต การข้าม rate limit การขูดข้อมูลแบบซ่อนตัวที่ออกแบบมาเพื่อเอาชนะการป้องกัน การยึดสายสนทนาหรือตัวแทนแบบสด การขโมยเซสชันที่นำกลับมาใช้ซ้ำได้ การอนุมัติโฟลว์การจับคู่ให้อัตโนมัติสำหรับผู้ใช้ที่ไม่ได้รับอนุมัติ

- การละเมิดแพลตฟอร์มและการหลบเลี่ยงการแบน
  - ตัวอย่าง: บัญชีซ่อนตัวหลังถูกแบน การวอร์มหรือฟาร์มบัญชี การสร้าง engagement ปลอม การเพาะ karma หรือผู้ติดตาม ระบบอัตโนมัติหลายบัญชี การโพสต์จำนวนมาก บอตสแปม ระบบอัตโนมัติสำหรับมาร์เก็ตเพลสหรือโซเชียลที่สร้างมาเพื่อหลีกเลี่ยงการตรวจจับ

- การฉ้อโกง การหลอกลวง และเวิร์กโฟลว์ทางการเงินที่ชวนให้เข้าใจผิด
  - ตัวอย่าง: ใบรับรองปลอม ใบแจ้งหนี้ปลอม โฟลว์การชำระเงินที่ชวนให้เข้าใจผิด การติดต่อเพื่อหลอกลวง หลักฐานทางสังคมปลอม เครื่องมือที่เปิดทางให้ใช้จ่ายหรือเรียกเก็บเงินโดยไม่มีการอนุมัติจากมนุษย์อย่างชัดเจนและไม่มีการควบคุมที่โปร่งใส หรือเวิร์กโฟลว์อัตลักษณ์สังเคราะห์ที่สร้างมาเพื่อเปิดบัญชีสำหรับการฉ้อโกง

- การขูดข้อมูล การเสริมข้อมูล หรือการเฝ้าระวังที่บุกรุกความเป็นส่วนตัว
  - ตัวอย่าง: การขูดข้อมูลติดต่อจำนวนมากเพื่อสแปม การเปิดเผยข้อมูลส่วนตัว การสะกดรอย การดึงรายชื่อลีดที่จับคู่กับการติดต่อโดยไม่ได้ร้องขอ การติดตามแบบลับ ๆ การค้นหาใบหน้าหรือจับคู่ข้อมูลชีวมิติที่ใช้โดยไม่มีความยินยอมชัดเจน หรือการซื้อ เผยแพร่ ดาวน์โหลด หรือทำให้ข้อมูลรั่วไหลหรือชุดข้อมูลจากการละเมิดพร้อมใช้งานเชิงปฏิบัติการ

- การแอบอ้างตัวตนหรือการจัดการอัตลักษณ์อย่างหลอกลวงโดยไม่ได้รับความยินยอม
  - ตัวอย่าง: การสลับใบหน้า ฝาแฝดดิจิทัล บุคลิกปลอม อินฟลูเอนเซอร์โคลน หรือเครื่องมือจัดการอัตลักษณ์อื่น ๆ ที่ใช้เพื่อแอบอ้างหรือทำให้เข้าใจผิด

- เนื้อหาทางเพศอย่างชัดเจนและการสร้างเนื้อหาผู้ใหญ่ที่ปิดระบบความปลอดภัย
  - ตัวอย่าง: การสร้างรูปภาพ/วิดีโอ/เนื้อหา NSFW ตัวครอบ API บุคคลที่สามสำหรับเนื้อหาผู้ใหญ่ หรือสกิลที่มีวัตถุประสงค์หลักเป็นเนื้อหาทางเพศอย่างชัดเจน

- ข้อกำหนดการรันที่ซ่อนอยู่ ไม่ปลอดภัย หรือชวนให้เข้าใจผิด
  - ตัวอย่าง: คำสั่งติดตั้งที่ทำให้เข้าใจยาก, `curl | sh`, ข้อกำหนด secret ที่ไม่ประกาศไว้, การใช้ private key ที่ไม่ประกาศไว้, การรัน `npx @latest` จากระยะไกลโดยไม่มีความสามารถในการตรวจทานที่ชัดเจน, เมตาดาต้าที่ทำให้เข้าใจผิดซึ่งซ่อนสิ่งที่สกิลต้องใช้จริงเพื่อรัน

## รูปแบบล่าสุดที่เราไม่ยอมรับอย่างชัดเจน

- “สร้างบัญชีผู้ขายแบบซ่อนตัวหลังถูกแบนจากมาร์เก็ตเพลส”
- “แก้ไขการจับคู่ Telegram เพื่อให้ผู้ใช้ที่ไม่ได้รับอนุมัติได้รับรหัสจับคู่โดยอัตโนมัติ”
- “เพาะบัญชี Reddit/Twitter ด้วยระบบอัตโนมัติที่ตรวจจับไม่ได้”
- “สร้างใบรับรองวิชาชีพหรือใบแจ้งหนี้สำหรับการใช้งานใด ๆ”
- “สร้างเนื้อหา NSFW โดยปิดการตรวจสอบความปลอดภัย”
- “ขูดลีด เสริมข้อมูลติดต่อ และเริ่มการติดต่อเย็นในวงกว้าง”
- “ซื้อ เผยแพร่ หรือดาวน์โหลดข้อมูลรั่วไหลหรือชุดข้อมูลจากการละเมิด”
- “สร้างบัญชีอีเมลหรือโซเชียลจำนวนมากด้วยอัตลักษณ์สังเคราะห์หรือการแก้ CAPTCHA”

## หมายเหตุสำหรับผู้รีวิว

- บริบทมีความสำคัญ หัวข้อเดียวกันอาจถูกต้องตามสมควรในบริบทเชิงป้องกันที่แคบหรืออิงความยินยอม และไม่ยอมรับได้เมื่อถูกจัดแพ็กเกจเป็นเวิร์กโฟลว์การละเมิด
- เราควรเอนเอียงไปทางการดำเนินการเมื่อสกิลถูกปรับให้เหมาะกับการหลบเลี่ยง การหลอกลวง หรือการใช้งานโดยไม่ได้รับความยินยอมอย่างชัดเจน
- การอัปโหลดซ้ำในหมวดหมู่เหล่านี้เป็นเหตุให้ซ่อนเนื้อหาและแบนบัญชีได้

## การบังคับใช้

- เราอาจซ่อน ลบ หรือ hard-delete สกิลที่ละเมิด
- เราอาจเพิกถอนโทเค็น soft-delete เนื้อหาที่เกี่ยวข้อง และแบนผู้กระทำผิดซ้ำหรือร้ายแรง
- เราไม่รับประกันการบังคับใช้แบบเตือนก่อนสำหรับการละเมิดที่ชัดเจน
