# การใช้งานที่ยอมรับได้

หน้านี้อธิบายประเภทของ Skills และเนื้อหาที่ ClawHub ยอมรับได้ รวมถึงเวิร์กโฟลว์การละเมิดที่ ClawHub จะไม่โฮสต์

กฎเหล่านี้ตั้งใจให้ใช้งานได้จริง เราให้ความสำคัญมากที่สุดกับเวิร์กโฟลว์การละเมิดแบบครบวงจร ไม่ใช่แค่คำสำคัญที่แยกจากบริบท หาก Skill ถูกสร้างขึ้นเพื่อหลบเลี่ยงการป้องกัน ละเมิดแพลตฟอร์ม หลอกลวงผู้คน บุกรุกความเป็นส่วนตัว หรือเปิดทางให้เกิดพฤติกรรมที่ไม่มีความยินยอม สิ่งนั้นไม่ควรอยู่บน ClawHub

## รูปแบบล่าสุดที่เรายืนยันอย่างชัดเจนว่ายอมรับได้

- งานฟรอนต์เอนด์และระบบดีไซน์ที่ใช้คอมโพเนนต์จริง โทเค็นเชิงความหมาย สถานะที่เข้าถึงได้ และโฟลว์ผู้ใช้ที่ผ่านการทดสอบแล้ว
- การประกอบ shadcn/ui ที่ใช้คอมโพเนนต์ซอร์สที่ติดตั้งไว้ นามแฝงของโปรเจกต์ และตัวแปรที่มีเอกสารกำกับ แทนมาร์กอัปเฉพาะครั้ง
- การแปลง UI5 จาก JavaScript เป็น TypeScript ที่คงคอมเมนต์ไว้ ใช้ชนิด UI5 ที่เป็นรูปธรรม และทำให้อินเทอร์เฟซคอนโทรลที่สร้างขึ้นสามารถตรวจทานได้
- การรีวิวความปลอดภัยเชิงป้องกัน เครื่องมือมอดเดอเรชัน และพรอมป์ตรวจจับการละเมิดที่แสดงหลักฐานและกำหนดขอบเขตการอนุมัติของมนุษย์ไว้อย่างชัดเจน
- การทำงานอัตโนมัติของเวิร์กโฟลว์ที่อิงความยินยอมสำหรับบัญชีส่วนบุคคลหรือทีม พร้อมข้อมูลรับรองที่ชัดเจน การตั้งค่าที่โปร่งใส และโหมดทดลองรันหรือแสดงตัวอย่าง
- เอกสาร คู่มือการย้ายระบบ ยูทิลิตีสำหรับนักพัฒนา และฟิกซ์เจอร์ทดสอบที่จำกัดขอบเขตอยู่กับซอฟต์แวร์ที่รองรับ

## ไม่ยอมรับ

- เวิร์กโฟลว์ที่เลี่ยงระบบความปลอดภัยหรือเข้าถึงโดยไม่ได้รับอนุญาต
  - ตัวอย่าง: การเลี่ยงผ่านการยืนยันตัวตน การยึดบัญชี การเลี่ยงผ่าน CAPTCHA การหลบเลี่ยง Cloudflare หรือระบบป้องกันบอต การเลี่ยงข้อจำกัดอัตรา การดึงข้อมูลแบบล่องหนที่ออกแบบมาเพื่อเอาชนะการป้องกัน การยึดสายสนทนาสดหรือเอเจนต์ การขโมยเซสชันที่ใช้ซ้ำได้ การอนุมัติอัตโนมัติในโฟลว์การจับคู่สำหรับผู้ใช้ที่ไม่ได้รับอนุมัติ

- การละเมิดแพลตฟอร์มและการหลบเลี่ยงการแบน
  - ตัวอย่าง: บัญชีล่องหนหลังถูกแบน การอุ่นเครื่องหรือฟาร์มบัญชี การมีส่วนร่วมปลอม การเพาะค่า karma หรือผู้ติดตาม การทำงานอัตโนมัติหลายบัญชี การโพสต์จำนวนมาก บอตสแปม การทำงานอัตโนมัติในมาร์เก็ตเพลสหรือโซเชียลที่สร้างมาเพื่อหลีกเลี่ยงการตรวจจับ

- การฉ้อโกง การหลอกลวง และเวิร์กโฟลว์การเงินที่หลอกลวง
  - ตัวอย่าง: ใบรับรองปลอม ใบแจ้งหนี้ปลอม โฟลว์การชำระเงินที่หลอกลวง การติดต่อเพื่อหลอกลวง หลักฐานทางสังคมปลอม เครื่องมือที่เปิดทางให้ใช้จ่ายหรือเรียกเก็บเงินโดยไม่มีการอนุมัติจากมนุษย์อย่างชัดเจนและการควบคุมที่โปร่งใส หรือเวิร์กโฟลว์ตัวตนสังเคราะห์ที่สร้างขึ้นเพื่อเปิดบัญชีสำหรับการฉ้อโกง

- การดึงข้อมูล การเพิ่มข้อมูล หรือการเฝ้าระวังที่บุกรุกความเป็นส่วนตัว
  - ตัวอย่าง: การดึงรายละเอียดติดต่อในปริมาณมากเพื่อสแปม การเปิดเผยข้อมูลส่วนตัว การสะกดรอย การสกัดรายชื่อว่าที่ลูกค้าที่จับคู่กับการติดต่อโดยไม่ได้ร้องขอ การติดตามแบบลับ การค้นหาใบหน้าหรือการจับคู่ชีวมิติที่ใช้โดยไม่มีความยินยอมชัดเจน หรือการซื้อ เผยแพร่ ดาวน์โหลด หรือนำข้อมูลรั่วไหลหรือชุดข้อมูลจากการละเมิดไปใช้ปฏิบัติการ

- การสวมรอยโดยไม่มีความยินยอมหรือการจัดการตัวตนที่หลอกลวง
  - ตัวอย่าง: การสลับใบหน้า ฝาแฝดดิจิทัล บุคคลปลอม อินฟลูเอนเซอร์ที่ถูกโคลน หรือเครื่องมือจัดการตัวตนอื่น ๆ ที่ใช้เพื่อสวมรอยหรือทำให้เข้าใจผิด

- เนื้อหาทางเพศอย่างชัดแจ้งและการสร้างเนื้อหาผู้ใหญ่ที่ปิดระบบความปลอดภัย
  - ตัวอย่าง: การสร้างรูปภาพ/วิดีโอ/เนื้อหา NSFW ตัวครอบเนื้อหาผู้ใหญ่สำหรับ API ของบุคคลที่สาม หรือ Skills ที่มีวัตถุประสงค์หลักเป็นเนื้อหาทางเพศอย่างชัดแจ้ง

- ข้อกำหนดการรันที่ซ่อนอยู่ ไม่ปลอดภัย หรือทำให้เข้าใจผิด
  - ตัวอย่าง: คำสั่งติดตั้งที่ทำให้อ่านยาก, `curl | sh`, ข้อกำหนดความลับที่ไม่ได้แจ้งไว้, การใช้คีย์ส่วนตัวที่ไม่ได้แจ้งไว้, การรัน `npx @latest` จากระยะไกลโดยไม่มีความสามารถในการตรวจทานที่ชัดเจน, เมตาดาต้าที่ทำให้เข้าใจผิดและซ่อนสิ่งที่ Skill ต้องใช้จริงเพื่อรัน

## รูปแบบล่าสุดที่เรายืนยันอย่างชัดเจนว่าไม่ยอมรับ

- “สร้างบัญชีผู้ขายแบบล่องหนหลังถูกแบนจากมาร์เก็ตเพลส”
- “แก้ไขการจับคู่ของ Telegram เพื่อให้ผู้ใช้ที่ไม่ได้รับอนุมัติได้รับรหัสจับคู่โดยอัตโนมัติ”
- “เพาะบัญชี Reddit/Twitter ด้วยระบบอัตโนมัติที่ตรวจจับไม่ได้”
- “สร้างใบรับรองวิชาชีพหรือใบแจ้งหนี้สำหรับการใช้งานใดก็ได้”
- “สร้างเนื้อหา NSFW โดยปิดการตรวจสอบความปลอดภัย”
- “ดึงรายชื่อลูกค้า เพิ่มข้อมูลติดต่อ และเริ่มการติดต่อแบบเย็นในปริมาณมาก”
- “ซื้อ เผยแพร่ หรือดาวน์โหลดข้อมูลรั่วไหลหรือชุดข้อมูลจากการละเมิด”
- “สร้างบัญชีอีเมลหรือโซเชียลจำนวนมากด้วยตัวตนสังเคราะห์หรือการแก้ CAPTCHA”

## หมายเหตุสำหรับผู้รีวิว

- บริบทมีความสำคัญ หัวข้อเดียวกันอาจชอบด้วยเหตุผลในกรอบเชิงป้องกันหรืออิงความยินยอมที่จำกัด และไม่อาจยอมรับได้เมื่อถูกบรรจุเป็นเวิร์กโฟลว์การละเมิด
- เราควรเอนเอียงไปทางการดำเนินการเมื่อ Skill ถูกปรับให้เหมาะกับการหลบเลี่ยง การหลอกลวง หรือการใช้งานที่ไม่มีความยินยอมอย่างชัดเจน
- การอัปโหลดซ้ำในหมวดหมู่เหล่านี้เป็นเหตุให้ซ่อนเนื้อหาและแบนบัญชีได้

## การบังคับใช้

- เราอาจซ่อน ลบ หรือ hard-delete Skills ที่ละเมิด
- เราอาจเพิกถอนโทเค็น soft-delete เนื้อหาที่เกี่ยวข้อง และแบนผู้กระทำผิดซ้ำหรือร้ายแรง
- เราไม่รับประกันว่าจะเตือนก่อนบังคับใช้กับการละเมิดที่ชัดเจน
