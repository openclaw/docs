# การใช้งานที่ยอมรับได้

ClawHub โฮสต์ Skills, Plugin, แพ็กเกจ และเมทาดาทาของ marketplace สำหรับ OpenClaw
ใช้หน้านี้เพื่อตัดสินว่าเนื้อหาหรือพฤติกรรมการเผยแพร่ควรอยู่บน
ClawHub หรือไม่

กฎเหล่านี้ใช้กับสิ่งที่รายการทำ สิ่งที่ขอให้ผู้ใช้รัน วิธีที่รายการ
แสดงตนเอง และวิธีที่ผู้เผยแพร่ใช้พื้นผิวการค้นพบ การติดตั้ง และความเชื่อถือ
ของ ClawHub สำหรับสถานะการดูแลและสถานะบัญชี โปรดดู
[การดูแลและความปลอดภัยของบัญชี](/clawhub/moderation) สำหรับการอ้างสิทธิ์ด้านลิขสิทธิ์หรือสิทธิอื่น ๆ
โปรดดู [คำขอเกี่ยวกับสิทธิ์ในเนื้อหา](/th/clawhub/content-rights)

## เนื้อหาที่อนุญาต

ClawHub ยินดีรับเนื้อหาที่มีประโยชน์ เข้าใจได้ และเผยแพร่ด้วยเจตนาดี

| หมวดหมู่                                         | อนุญาตเมื่อ                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| ประสิทธิภาพการทำงานของนักพัฒนา                           | รายการช่วยให้ผู้ใช้สร้าง ทดสอบ ย้ายระบบ ดีบัก จัดทำเอกสาร หรือดูแลการทำงานของซอฟต์แวร์                                               |
| เวิร์กโฟลว์ UI, ข้อมูล และอัตโนมัติ               | ขอบเขตชัดเจน ระบุข้อมูลรับรองที่จำเป็นอย่างชัดแจ้ง และการดำเนินการที่มีความเสี่ยงมีเส้นทางสำหรับการตรวจสอบ dry-run พรีวิว หรือยืนยัน |
| ความปลอดภัยเชิงป้องกัน การดูแล และการตรวจสอบการละเมิด | เครื่องมือถูกนำเสนอเพื่อการตรวจสอบที่ได้รับอนุญาต เก็บรักษาหลักฐาน และทำให้ขอบเขตการอนุมัติโดยมนุษย์ชัดเจน                          |
| เวิร์กโฟลว์ส่วนบุคคลหรือทีม                       | เวิร์กโฟลว์ใช้บัญชีที่อิงตามความยินยอม การตั้งค่าที่โปร่งใส และสิทธิ์ที่ชัดแจ้ง                                            |
| แคตตาล็อกที่ได้รับการดูแล                              | แต่ละรายการมีความแตกต่าง มีประโยชน์ อธิบายอย่างถูกต้อง และได้รับการดูแลอย่างสมเหตุสมผล                                                |

บริบทมีความสำคัญ หัวข้อเดียวกันอาจยอมรับได้ในบริบทเชิงป้องกันที่แคบหรือ
อิงตามความยินยอม และยอมรับไม่ได้เมื่อบรรจุเป็นเวิร์กโฟลว์การละเมิด

## เนื้อหาที่ไม่อนุญาต

ClawHub ไม่โฮสต์เนื้อหาที่มีวัตถุประสงค์หลักเพื่อการละเมิด การหลอกลวง การดำเนินการที่ไม่ปลอดภัย
หรือการละเมิดสิทธิ์

| หมวดหมู่                                                    | ไม่อนุญาต                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| การเข้าถึงที่ไม่ได้รับอนุญาตหรือการเลี่ยงผ่านความปลอดภัย                      | การเลี่ยงผ่านการยืนยันตัวตน การยึดบัญชี การละเมิด rate-limit การยึดสายเรียกสดหรือ agent การขโมยเซสชันที่นำกลับมาใช้ซ้ำได้ หรือการอนุมัติกระบวนการจับคู่โดยอัตโนมัติสำหรับผู้ใช้ที่ไม่ได้รับอนุมัติ                                                                                                                                                   |
| การละเมิดแพลตฟอร์มและการหลบเลี่ยงการแบน                              | บัญชีลับหลังถูกแบน การอุ่นหรือฟาร์มบัญชี การมีส่วนร่วมปลอม การทำอัตโนมัติหลายบัญชี การโพสต์จำนวนมาก บอตสแปม หรือระบบอัตโนมัติที่สร้างขึ้นเพื่อหลีกเลี่ยงการตรวจจับ                                                                                                                                          |
| การฉ้อโกง กลโกง และเวิร์กโฟลว์การเงินที่หลอกลวง             | ใบรับรองหรือใบแจ้งหนี้ปลอม โฟลว์การชำระเงินที่หลอกลวง การติดต่อหลอกลวง หลักฐานทางสังคมปลอม เวิร์กโฟลว์ตัวตนสังเคราะห์เพื่อการฉ้อโกง หรือเครื่องมือใช้จ่าย/เรียกเก็บเงินโดยไม่มีการอนุมัติจากมนุษย์อย่างชัดเจน                                                                                                                    |
| การเพิ่มข้อมูลหรือการเฝ้าระวังที่ละเมิดความเป็นส่วนตัว                 | การกวาดรายชื่อติดต่อเพื่อสแปม การเปิดเผยข้อมูลส่วนบุคคล การสะกดรอย การดึง lead ที่จับคู่กับการติดต่อโดยไม่ได้รับเชิญ การติดตามแบบลับ การจับคู่ชีวมิติที่ไม่ได้รับความยินยอม หรือการใช้ข้อมูลรั่วไหลหรือชุดข้อมูลจากการละเมิด                                                                                                                  |
| การแอบอ้างหรือการจัดการตัวตนโดยไม่ได้รับความยินยอม       | การสลับใบหน้า คู่แฝดดิจิทัล อินฟลูเอนเซอร์ที่ถูกโคลน persona ปลอม หรือเครื่องมืออื่น ๆ ที่ใช้เพื่อแอบอ้างหรือทำให้เข้าใจผิด                                                                                                                                                                                                 |
| เนื้อหาทางเพศอย่างชัดเจนหรือการสร้างเนื้อหาผู้ใหญ่ที่ปิดความปลอดภัย | การสร้างรูปภาพ วิดีโอ หรือเนื้อหา NSFW; wrapper เนื้อหาผู้ใหญ่รอบ API ของบุคคลที่สาม; หรือรายการที่มีวัตถุประสงค์หลักเป็นเนื้อหาทางเพศอย่างชัดเจน                                                                                                                                                       |
| ข้อกำหนดการดำเนินการที่ซ่อน ไม่ปลอดภัย หรือทำให้เข้าใจผิด        | คำสั่งติดตั้งที่ทำให้อ่านยาก ตัวติดตั้งแบบ pipe-to-shell เช่น เนื้อหาที่ดาวน์โหลดมาแล้วรันด้วย `sh` หรือ `bash` โดยไม่มีความสามารถในการตรวจสอบที่ชัดเจน ข้อกำหนด secret หรือ private key ที่ไม่ได้ประกาศ การดำเนินการ `npx @latest` ระยะไกลโดยไม่มีความสามารถในการตรวจสอบที่ชัดเจน หรือเมทาดาทาที่ซ่อนสิ่งที่รายการต้องใช้จริงเพื่อรัน |
| เนื้อหาที่ละเมิดลิขสิทธิ์หรือละเมิดสิทธิ์           | การเผยแพร่ Skills, Plugin, เอกสาร, สินทรัพย์แบรนด์ หรือโค้ดกรรมสิทธิ์ของผู้อื่นซ้ำโดยไม่ได้รับอนุญาต; การละเมิดเงื่อนไขใบอนุญาต; หรือการแอบอ้างเป็นผู้เขียนหรือผู้เผยแพร่ต้นฉบับ                                                                                                                            |

## พฤติกรรม marketplace ที่ไม่อนุญาต

ClawHub ยังตรวจสอบวิธีที่ผู้เผยแพร่ใช้ marketplace ด้วย ห้ามใช้ ClawHub เพื่อ
บิดเบือนการค้นพบ เมตริก สัญญาณความเชื่อถือ ระบบการดูแล หรือความสนใจของผู้ใช้

พฤติกรรม marketplace ที่ไม่อนุญาตรวมถึง:

- การเผยแพร่จำนวนมากของรายการที่ใช้ความพยายามต่ำ ซ้ำซ้อน เป็น placeholder หรือ
  สร้างโดยเครื่อง ซึ่งดูเหมือนไม่มีคุณค่าจริงต่อผู้ใช้
- การท่วมพื้นผิวการค้นหาหรือหมวดหมู่ด้วย Skills หรือ Plugin ที่แทบเหมือนกัน
- การเผยแพร่รายการหลายร้อยรายการที่มีการใช้งาน การดูแล ความชัดเจนของแหล่งที่มา
  หรือความแตกต่างที่มีความหมายเพียงเล็กน้อยหรือไม่มีเลย
- การเพิ่มจำนวนการติดตั้ง ดาวน์โหลด ดาว หรือเมตริกการมีส่วนร่วมอื่น ๆ
  อย่างเทียมผ่านระบบอัตโนมัติ ลูปติดตั้งด้วยตนเอง บัญชีปลอม กิจกรรมที่ประสานกัน
  การจ่ายเงินเพื่อการมีส่วนร่วม หรือพฤติกรรมอื่นที่ไม่เกิดขึ้นเองตามธรรมชาติ
- การสร้างหรือหมุนเวียนบัญชีเพื่อหลบเลี่ยงการดูแล การแบน ขีดจำกัดผู้เผยแพร่ หรือ
  การตรวจสอบ marketplace
- การทำให้ผู้ใช้เข้าใจผิดเกี่ยวกับความเป็นเจ้าของ แหล่งที่มา ความสามารถ สถานะความปลอดภัย
  ข้อกำหนดการติดตั้ง หรือความเกี่ยวข้องกับโปรเจกต์หรือผู้เผยแพร่อื่น
- การอัปโหลดเนื้อหาซ้ำ ๆ ที่เคยถูกซ่อน ลบ หรือบล็อกไปแล้ว
  โดยไม่แก้ไขปัญหาต้นเหตุ

การเผยแพร่ในปริมาณมากไม่ใช่การละเมิดโดยอัตโนมัติ แคตตาล็อกขนาดใหญ่ยอมรับได้
เมื่อรายการมีความแตกต่างอย่างมีความหมาย อธิบายอย่างถูกต้อง ได้รับการดูแล
และมีผู้ใช้จริงใช้งาน แคตตาล็อกขนาดใหญ่จะกลายเป็นปัญหาด้านความเชื่อถือและความปลอดภัยเมื่อ
ปริมาณมาพร้อมกับรายการที่บาง ซ้ำซ้อน ทำให้เข้าใจผิด ไม่ได้รับการดูแล หรือ
ถูกโปรโมตอย่างเทียม

## สิทธิ์ในเนื้อหา

หากคุณเชื่อว่าเนื้อหาบน ClawHub ละเมิดลิขสิทธิ์หรือสิทธิอื่น ๆ ของคุณ ให้ใช้
[คำขอเกี่ยวกับสิทธิ์ในเนื้อหา](/th/clawhub/content-rights) อย่าใช้รายงาน marketplace
ปกติสำหรับการอ้างสิทธิ์ด้านลิขสิทธิ์หรือสิทธิ์ เว้นแต่รายการนั้นไม่ปลอดภัย
เป็นอันตราย หรือทำให้เข้าใจผิดด้วย

## การตรวจสอบและการบังคับใช้

ClawHub อาจใช้การตรวจสอบอัตโนมัติ สัญญาณการละเมิดเชิงสถิติ รายงานจากผู้ใช้ และ
การตรวจสอบโดยเจ้าหน้าที่เพื่อระบุเนื้อหาที่ไม่ปลอดภัยหรือพฤติกรรมการเผยแพร่ที่ละเมิด สัญญาณหนึ่ง
ไม่ได้พิสูจน์การละเมิดด้วยตัวเอง แต่ช่วยให้ ClawHub ตัดสินใจว่าสิ่งใดต้องได้รับการตรวจสอบ

เราอาจ:

- ซ่อน พักไว้ ลบ ลบแบบ soft-delete หรือในกรณีที่ประเภททรัพยากรรองรับ
  ลบแบบ hard-delete รายการที่ละเมิด
- บล็อกการดาวน์โหลดหรือการติดตั้งสำหรับรีลีสที่ไม่ปลอดภัย
- เพิกถอนโทเค็น API
- ลบเนื้อหาที่เกี่ยวข้องแบบ soft-delete
- จำกัดการเข้าถึงการเผยแพร่
- แบนผู้กระทำผิดซ้ำหรือร้ายแรง

เราไม่รับประกันการบังคับใช้แบบเตือนก่อนสำหรับการละเมิดที่ชัดเจน โปรดดู
[การดูแลและความปลอดภัยของบัญชี](/clawhub/moderation) สำหรับรายงาน การพักการดูแล
รายการที่ถูกซ่อน การแบน และสถานะบัญชี
