# การใช้งานที่ยอมรับได้

หน้านี้อธิบายประเภทของ Skills และเนื้อหาที่ ClawHub ยอมรับได้ รวมถึงเวิร์กโฟลว์การละเมิดที่ ClawHub จะไม่โฮสต์

กฎเหล่านี้ตั้งใจให้ใช้งานได้จริง เราให้ความสำคัญมากที่สุดกับเวิร์กโฟลว์การละเมิดแบบครบวงจร ไม่ใช่เพียงคีย์เวิร์ดที่แยกออกมาโดด ๆ หาก Skill ถูกสร้างขึ้นเพื่อหลบเลี่ยงการป้องกัน ละเมิดแพลตฟอร์ม หลอกลวงผู้คน ละเมิดความเป็นส่วนตัว หรือเปิดทางให้เกิดพฤติกรรมที่ไม่ได้รับความยินยอม Skill นั้นไม่ควรอยู่บน ClawHub

## รูปแบบล่าสุดที่เราระบุชัดเจนว่ายอมรับได้

- งานฟรอนต์เอนด์และดีไซน์ซิสเต็มที่ใช้คอมโพเนนต์จริง โทเค็นเชิงความหมาย สถานะที่เข้าถึงได้ และโฟลว์ผู้ใช้ที่ผ่านการทดสอบแล้ว
- การประกอบ shadcn/ui ที่ใช้คอมโพเนนต์ซอร์สที่ติดตั้งแล้ว อะเลียสของโปรเจ็กต์ และเวอร์ชันแปรผันที่มีเอกสารกำกับ แทนมาร์กอัปเฉพาะกิจ
- การแปลง UI5 จาก JavaScript เป็น TypeScript ที่รักษาคอมเมนต์ไว้ ใช้ชนิดข้อมูล UI5 ที่ชัดเจน และทำให้อินเทอร์เฟซคอนโทรลที่สร้างขึ้นยังตรวจทานได้
- การรีวิวความปลอดภัยเชิงป้องกัน เครื่องมือมอดเดอเรชัน และพรอมป์ต์ตรวจจับการละเมิดที่แสดงหลักฐานและรักษาขอบเขตการอนุมัติโดยมนุษย์ให้ชัดเจน
- ระบบอัตโนมัติของเวิร์กโฟลว์ตามความยินยอมสำหรับบัญชีส่วนบุคคลหรือทีม โดยมีข้อมูลรับรองชัดเจน การตั้งค่าที่โปร่งใส และโหมดทดลองรันหรือดูตัวอย่าง
- เอกสาร รันบุ๊กการย้ายระบบ ยูทิลิตีสำหรับนักพัฒนา และฟิกซ์เจอร์ทดสอบที่จำกัดขอบเขตตามซอฟต์แวร์ที่รองรับ

## ไม่ยอมรับ

- เวิร์กโฟลว์การหลบเลี่ยงความปลอดภัยหรือการเข้าถึงโดยไม่ได้รับอนุญาต
  - ตัวอย่าง: การข้ามการยืนยันตัวตน การยึดบัญชี การข้าม CAPTCHA การหลบเลี่ยง Cloudflare หรือระบบป้องกันบอต การข้ามการจำกัดอัตรา การสเครปแบบซ่อนตัวที่ออกแบบมาเพื่อเอาชนะการป้องกัน การยึดสายสนทนาหรือเอเจนต์แบบสด การขโมยเซสชันที่นำกลับมาใช้ซ้ำได้ การอนุมัติโฟลว์จับคู่อัตโนมัติสำหรับผู้ใช้ที่ไม่ได้รับอนุมัติ

- การละเมิดแพลตฟอร์มและการหลบเลี่ยงการแบน
  - ตัวอย่าง: บัญชีซ่อนตัวหลังถูกแบน การอุ่นเครื่องหรือเพาะบัญชี การมีส่วนร่วมปลอม การสะสมคาร์มาหรือผู้ติดตาม ระบบอัตโนมัติหลายบัญชี การโพสต์จำนวนมาก บอตสแปม ระบบอัตโนมัติสำหรับมาร์เก็ตเพลซหรือโซเชียลที่สร้างมาเพื่อหลบเลี่ยงการตรวจจับ

- การฉ้อโกง การหลอกลวง และเวิร์กโฟลว์ทางการเงินที่หลอกลวง
  - ตัวอย่าง: ใบรับรองปลอม ใบแจ้งหนี้ปลอม โฟลว์การชำระเงินที่หลอกลวง การติดต่อหลอกลวง หลักฐานทางสังคมปลอม เครื่องมือที่เปิดทางให้ใช้จ่ายหรือเรียกเก็บเงินโดยไม่มีการอนุมัติจากมนุษย์อย่างชัดเจนและการควบคุมที่โปร่งใส หรือเวิร์กโฟลว์ตัวตนสังเคราะห์ที่สร้างขึ้นเพื่อสร้างบัญชีสำหรับการฉ้อโกง

- การสเครป การเพิ่มข้อมูล หรือการเฝ้าระวังที่ละเมิดความเป็นส่วนตัว
  - ตัวอย่าง: การสเครปรายละเอียดติดต่อจำนวนมากเพื่อสแปม การเปิดเผยข้อมูลส่วนตัว การสะกดรอย การดึงลีดที่จับคู่กับการติดต่อโดยไม่ได้รับเชิญ การติดตามแบบลับ การค้นหาใบหน้าหรือการจับคู่ข้อมูลไบโอเมตริกที่ใช้โดยไม่มีความยินยอมชัดเจน หรือการซื้อ เผยแพร่ ดาวน์โหลด หรือนำข้อมูลรั่วไหลหรือดัมป์จากการละเมิดข้อมูลไปใช้งานจริง

- การสวมรอยโดยไม่ได้รับความยินยอมหรือการบิดเบือนตัวตนแบบหลอกลวง
  - ตัวอย่าง: การสลับใบหน้า แฝดดิจิทัล บุคคลปลอม อินฟลูเอนเซอร์ที่ถูกโคลน หรือเครื่องมือจัดการตัวตนอื่น ๆ ที่ใช้เพื่อสวมรอยหรือทำให้เข้าใจผิด

- เนื้อหาทางเพศอย่างชัดเจนและการสร้างเนื้อหาผู้ใหญ่ที่ปิดระบบความปลอดภัย
  - ตัวอย่าง: การสร้างรูปภาพ/วิดีโอ/เนื้อหา NSFW แรปเปอร์เนื้อหาผู้ใหญ่รอบ API ของบุคคลที่สาม หรือ Skills ที่มีวัตถุประสงค์หลักเป็นเนื้อหาทางเพศอย่างชัดเจน

- ข้อกำหนดการรันที่ซ่อนอยู่ ไม่ปลอดภัย หรือทำให้เข้าใจผิด
  - ตัวอย่าง: คำสั่งติดตั้งที่ทำให้อ่านยาก, `curl | sh`, ข้อกำหนดความลับที่ไม่ประกาศไว้ การใช้คีย์ส่วนตัวที่ไม่ประกาศไว้ การรัน `npx @latest` จากระยะไกลโดยไม่มีความสามารถในการตรวจทานที่ชัดเจน เมทาดาทาที่ทำให้เข้าใจผิดซึ่งซ่อนว่า Skill ต้องการอะไรจริง ๆ เพื่อรัน

## รูปแบบล่าสุดที่เราระบุชัดเจนว่าไม่ยอมรับ

- “สร้างบัญชีผู้ขายแบบซ่อนตัวหลังถูกแบนจากมาร์เก็ตเพลซ”
- “ปรับเปลี่ยนการจับคู่ Telegram เพื่อให้ผู้ใช้ที่ไม่ได้รับอนุมัติได้รับโค้ดจับคู่โดยอัตโนมัติ”
- “เพาะบัญชี Reddit/Twitter ด้วยระบบอัตโนมัติที่ตรวจจับไม่ได้”
- “สร้างใบรับรองวิชาชีพหรือใบแจ้งหนี้สำหรับการใช้งานใดก็ได้”
- “สร้างเนื้อหา NSFW โดยปิดการตรวจสอบความปลอดภัย”
- “สเครปลีด เพิ่มข้อมูลผู้ติดต่อ และเริ่มการติดต่อแบบเย็นจำนวนมาก”
- “ซื้อ เผยแพร่ หรือดาวน์โหลดข้อมูลรั่วไหลหรือดัมป์จากการละเมิดข้อมูล”
- “สร้างบัญชีอีเมลหรือโซเชียลจำนวนมากด้วยตัวตนสังเคราะห์หรือการแก้ CAPTCHA”

## หมายเหตุสำหรับผู้รีวิว

- บริบทมีความสำคัญ หัวข้อเดียวกันอาจถูกต้องตามกฎในบริบทเชิงป้องกันที่แคบหรือบริบทตามความยินยอม และไม่อาจยอมรับได้เมื่อถูกบรรจุเป็นเวิร์กโฟลว์การละเมิด
- เราควรเอนเอียงไปทางการดำเนินการเมื่อ Skill ถูกปรับให้เหมาะกับการหลบเลี่ยง การหลอกลวง หรือการใช้งานโดยไม่ได้รับความยินยอมอย่างชัดเจน
- การอัปโหลดซ้ำในหมวดหมู่เหล่านี้เป็นเหตุให้ซ่อนเนื้อหาและแบนบัญชีได้

## การบังคับใช้

- เราอาจซ่อน ลบ หรือ hard-delete Skills ที่ละเมิด
- เราอาจเพิกถอนโทเค็น soft-delete เนื้อหาที่เกี่ยวข้อง และแบนผู้กระทำผิดซ้ำหรือร้ายแรง
- เราไม่รับประกันการบังคับใช้แบบเตือนก่อนสำหรับการละเมิดที่ชัดเจน
