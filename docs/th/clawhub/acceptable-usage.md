# การใช้งานที่ยอมรับได้

ClawHub โฮสต์ Skills, Plugin, แพ็กเกจ และเมทาดาทาของมาร์เก็ตเพลสสำหรับ OpenClaw
ใช้หน้านี้เพื่อตัดสินใจว่าเนื้อหาหรือพฤติกรรมการเผยแพร่ควรอยู่บน
ClawHub หรือไม่

กฎเหล่านี้ใช้กับสิ่งที่รายการทำ สิ่งที่รายการขอให้ผู้ใช้รัน วิธีที่รายการ
แสดงตัวตน และวิธีที่ผู้เผยแพร่ใช้พื้นผิวด้านการค้นพบ การติดตั้ง และ
ความน่าเชื่อถือของ ClawHub สำหรับสถานะการดูแลและสถานะบัญชี โปรดดู
[การดูแลและความปลอดภัยของบัญชี](/clawhub/moderation) สำหรับลิขสิทธิ์หรือการอ้างสิทธิ์อื่น
โปรดดู [คำขอสิทธิ์ในเนื้อหา](/clawhub/content-rights)

## เนื้อหาที่อนุญาต

ClawHub ยินดีรับเนื้อหาที่มีประโยชน์ เข้าใจได้ และเผยแพร่โดยสุจริต

| หมวดหมู่                                         | อนุญาตเมื่อ                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| ประสิทธิภาพการทำงานของนักพัฒนา                           | รายการช่วยให้ผู้ใช้สร้าง ทดสอบ ย้ายระบบ ดีบัก จัดทำเอกสาร หรือดำเนินงานซอฟต์แวร์                                               |
| เวิร์กโฟลว์ UI ข้อมูล และระบบอัตโนมัติ               | ขอบเขตชัดเจน ระบุข้อมูลรับรองที่จำเป็นอย่างชัดเจน และการกระทำที่มีความเสี่ยงมีเส้นทางสำหรับการตรวจทาน การทดลองรัน การแสดงตัวอย่าง หรือการยืนยัน |
| ความปลอดภัยเชิงป้องกัน การดูแล และการตรวจทานการละเมิด | เครื่องมือถูกนำเสนอเพื่อการตรวจทานที่ได้รับอนุญาต เก็บรักษาหลักฐาน และทำให้ขอบเขตการอนุมัติของมนุษย์ชัดเจน                          |
| เวิร์กโฟลว์ส่วนบุคคลหรือทีม                       | เวิร์กโฟลว์ใช้บัญชีตามความยินยอม การตั้งค่าที่โปร่งใส และสิทธิ์ที่ชัดเจน                                            |
| แคตตาล็อกที่มีการบำรุงรักษา                              | แต่ละรายการแตกต่างกัน มีประโยชน์ อธิบายอย่างถูกต้อง และได้รับการบำรุงรักษาอย่างสมเหตุสมผล                                                |

บริบทมีความสำคัญ หัวข้อเดียวกันอาจยอมรับได้ในบริบทเชิงป้องกันที่แคบหรือ
อิงตามความยินยอม และยอมรับไม่ได้เมื่อถูกจัดแพ็กเกจเป็นเวิร์กโฟลว์การละเมิด

## เนื้อหาที่ไม่อนุญาต

ClawHub ไม่โฮสต์เนื้อหาที่มีวัตถุประสงค์หลักเป็นการละเมิด การหลอกลวง การดำเนินการที่ไม่ปลอดภัย
หรือการละเมิดสิทธิ์

| หมวดหมู่                                                    | ไม่อนุญาต                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| การเข้าถึงโดยไม่ได้รับอนุญาตหรือการเลี่ยงความปลอดภัย                      | การเลี่ยงการยืนยันตัวตน การยึดบัญชี การละเมิดขีดจำกัดอัตรา การยึดการโทรสดหรือ agent การขโมยเซสชันที่นำกลับมาใช้ซ้ำได้ หรือโฟลว์การจับคู่ที่อนุมัติอัตโนมัติสำหรับผู้ใช้ที่ไม่ได้รับอนุมัติ                                                                                                                                                   |
| การละเมิดแพลตฟอร์มและการหลบเลี่ยงการแบน                              | บัญชีแอบแฝงหลังถูกแบน การอุ่นบัญชีหรือฟาร์มบัญชี การมีส่วนร่วมปลอม ระบบอัตโนมัติหลายบัญชี การโพสต์จำนวนมาก บอตสแปม หรือระบบอัตโนมัติที่สร้างขึ้นเพื่อหลีกเลี่ยงการตรวจจับ                                                                                                                                          |
| การฉ้อโกง การหลอกลวง และเวิร์กโฟลว์ทางการเงินที่หลอกลวง             | ใบรับรองหรือใบแจ้งหนี้ปลอม โฟลว์การชำระเงินที่หลอกลวง การติดต่อหลอกลวง หลักฐานทางสังคมปลอม เวิร์กโฟลว์ตัวตนสังเคราะห์เพื่อการฉ้อโกง หรือเครื่องมือใช้จ่าย/เรียกเก็บเงินที่ไม่มีการอนุมัติจากมนุษย์อย่างชัดเจน                                                                                                                    |
| การเพิ่มข้อมูลหรือการเฝ้าระวังที่รุกรานความเป็นส่วนตัว                 | การขูดข้อมูลผู้ติดต่อเพื่อสแปม การเปิดเผยข้อมูลส่วนบุคคล การสะกดรอย การดึงข้อมูลลูกค้าเป้าหมายที่จับคู่กับการติดต่อโดยไม่ได้ร้องขอ การเฝ้าติดตามแบบแอบแฝง การจับคู่ชีวมิติที่ไม่ได้รับความยินยอม หรือการใช้ข้อมูลที่รั่วไหลหรือชุดข้อมูลจากการละเมิด                                                                                                                  |
| การแอบอ้างหรือการจัดการตัวตนโดยไม่ได้รับความยินยอม       | การสลับใบหน้า ฝาแฝดดิจิทัล อินฟลูเอนเซอร์ที่ถูกโคลน ตัวตนปลอม หรือเครื่องมืออื่นที่ใช้เพื่อแอบอ้างหรือทำให้เข้าใจผิด                                                                                                                                                                                                 |
| เนื้อหาทางเพศอย่างโจ่งแจ้งหรือการสร้างเนื้อหาผู้ใหญ่ที่ปิดความปลอดภัย | การสร้างรูปภาพ วิดีโอ หรือเนื้อหา NSFW; ตัวครอบเนื้อหาผู้ใหญ่รอบ API ของบุคคลที่สาม; หรือรายการที่มีวัตถุประสงค์หลักเป็นเนื้อหาทางเพศอย่างโจ่งแจ้ง                                                                                                                                                       |
| ข้อกำหนดการดำเนินการที่ซ่อนอยู่ ไม่ปลอดภัย หรือทำให้เข้าใจผิด        | คำสั่งติดตั้งที่ทำให้อ่านยาก โปรแกรมติดตั้งแบบ pipe-to-shell เช่น เนื้อหาที่ดาวน์โหลดแล้วรันด้วย `sh` หรือ `bash` โดยไม่มีความสามารถในการตรวจทานที่ชัดเจน ข้อกำหนดความลับหรือคีย์ส่วนตัวที่ไม่ได้ประกาศ การดำเนินการ `npx @latest` ระยะไกลโดยไม่มีความสามารถในการตรวจทานที่ชัดเจน หรือเมทาดาทาที่ซ่อนว่ารายการต้องใช้อะไรจริง ๆ เพื่อรัน |

## พฤติกรรมในมาร์เก็ตเพลสที่ไม่อนุญาต

ClawHub ยังตรวจทานวิธีที่ผู้เผยแพร่ใช้มาร์เก็ตเพลสด้วย อย่าใช้ ClawHub เพื่อ
จัดการการค้นพบ เมตริก สัญญาณความน่าเชื่อถือ ระบบการดูแล หรือความสนใจของผู้ใช้

พฤติกรรมในมาร์เก็ตเพลสที่ไม่อนุญาตรวมถึง:

- การเผยแพร่จำนวนมากซึ่งรายการที่ใช้ความพยายามต่ำ ซ้ำซ้อน เป็นตัวแทนที่ว่างเปล่า หรือ
  สร้างด้วยเครื่อง และดูเหมือนไม่มีคุณค่าจริงต่อผู้ใช้
- การท่วมพื้นผิวการค้นหาหรือหมวดหมู่ด้วย Skills หรือ Plugin ที่แทบเหมือนกัน
- การเผยแพร่รายการหลายร้อยรายการที่มีการใช้งาน การบำรุงรักษา ความชัดเจนของแหล่งที่มา
  หรือความแตกต่างที่มีความหมายเพียงเล็กน้อยหรือไม่มีเลย
- การเพิ่มจำนวนการติดตั้ง การดาวน์โหลด ดาว หรือเมตริกการมีส่วนร่วมอื่น
  อย่างไม่เป็นธรรมผ่านระบบอัตโนมัติ วงจรติดตั้งด้วยตนเอง บัญชีปลอม กิจกรรมที่ประสานกัน
  การมีส่วนร่วมแบบจ่ายเงิน หรือพฤติกรรมอื่นที่ไม่เป็นธรรมชาติ
- การสร้างหรือหมุนเวียนบัญชีเพื่อหลบเลี่ยงการดูแล การแบน ขีดจำกัดผู้เผยแพร่ หรือ
  การตรวจทานของมาร์เก็ตเพลส
- การทำให้ผู้ใช้เข้าใจผิดเกี่ยวกับความเป็นเจ้าของ แหล่งที่มา ความสามารถ สถานะความปลอดภัย
  ข้อกำหนดการติดตั้ง หรือความเกี่ยวข้องกับโปรเจกต์หรือผู้เผยแพร่อื่น
- การอัปโหลดเนื้อหาซ้ำ ๆ ที่เคยถูกซ่อน ลบ หรือบล็อกไปแล้ว
  โดยไม่แก้ไขปัญหาพื้นฐาน

การเผยแพร่ปริมาณสูงไม่ใช่การละเมิดโดยอัตโนมัติ แคตตาล็อกขนาดใหญ่ยอมรับได้
เมื่อรายการแตกต่างกันอย่างมีความหมาย อธิบายอย่างถูกต้อง ได้รับการบำรุงรักษา
และถูกใช้โดยผู้ใช้จริง แคตตาล็อกขนาดใหญ่จะกลายเป็นปัญหาด้านความน่าเชื่อถือและความปลอดภัยเมื่อ
ปริมาณมาคู่กับรายการที่บาง ซ้ำซ้อน ทำให้เข้าใจผิด ไม่ได้รับการบำรุงรักษา หรือ
ถูกโปรโมตอย่างไม่เป็นธรรม

## สิทธิ์ในเนื้อหา

หากคุณเชื่อว่าเนื้อหาบน ClawHub ละเมิดลิขสิทธิ์หรือสิทธิ์อื่นของคุณ ให้ใช้
[คำขอสิทธิ์ในเนื้อหา](/clawhub/content-rights) อย่าใช้รายงานมาร์เก็ตเพลสปกติ
สำหรับการอ้างสิทธิ์ลิขสิทธิ์หรือสิทธิ์ เว้นแต่รายการนั้นจะไม่ปลอดภัย
เป็นอันตราย หรือทำให้เข้าใจผิดด้วย

## การตรวจทานและการบังคับใช้

ClawHub อาจใช้การตรวจสอบอัตโนมัติ สัญญาณการละเมิดเชิงสถิติ รายงานจากผู้ใช้ และ
การตรวจทานโดยเจ้าหน้าที่เพื่อระบุเนื้อหาที่ไม่ปลอดภัยหรือพฤติกรรมการเผยแพร่ที่เป็นการละเมิด สัญญาณหนึ่ง
ไม่ได้พิสูจน์การละเมิดด้วยตัวมันเอง แต่ช่วยให้ ClawHub ตัดสินใจว่าสิ่งใดต้องได้รับการตรวจทาน

เราอาจ:

- ซ่อน ระงับ ลบ ลบแบบ soft-delete หรือในกรณีที่ประเภทธรัพยากรรองรับ
  ลบแบบ hard-delete รายการที่ละเมิด
- บล็อกการดาวน์โหลดหรือการติดตั้งสำหรับรีลีสที่ไม่ปลอดภัย
- เพิกถอนโทเค็น API
- ลบแบบ soft-delete เนื้อหาที่เกี่ยวข้อง
- จำกัดสิทธิ์การเผยแพร่
- แบนผู้กระทำผิดซ้ำหรือร้ายแรง

เราไม่รับประกันการบังคับใช้แบบเตือนก่อนสำหรับการละเมิดที่ชัดเจน โปรดดู
[การดูแลและความปลอดภัยของบัญชี](/clawhub/moderation) สำหรับรายงาน การระงับเพื่อดูแล
รายการที่ถูกซ่อน การแบน และสถานะบัญชี
