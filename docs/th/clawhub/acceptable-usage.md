# การใช้งานที่ยอมรับได้

ClawHub โฮสต์ Skills, Plugin, แพ็กเกจ และเมทาดาทาของมาร์เก็ตเพลสสำหรับ OpenClaw
ใช้หน้านี้เพื่อตัดสินว่าเนื้อหาหรือพฤติกรรมการเผยแพร่ควรอยู่บน
ClawHub หรือไม่

กฎเหล่านี้ใช้กับสิ่งที่รายการประกาศทำ สิ่งที่ขอให้ผู้ใช้รัน วิธีที่รายการ
นำเสนอตนเอง และวิธีที่ผู้เผยแพร่ใช้พื้นผิวการค้นพบ การติดตั้ง และความน่าเชื่อถือ
ของ ClawHub สำหรับสถานะการดูแลและสถานะบัญชี โปรดดู
[การดูแลและความปลอดภัยของบัญชี](/clawhub/moderation) สำหรับการร้องเรียนลิขสิทธิ์หรือสิทธิ์อื่น ๆ
โปรดดู [คำขอด้านสิทธิ์ในเนื้อหา](/th/clawhub/content-rights)

## เนื้อหาที่อนุญาต

ClawHub ยินดีรับเนื้อหาที่มีประโยชน์ เข้าใจได้ และเผยแพร่โดยสุจริต

| หมวดหมู่                                         | อนุญาตเมื่อ                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| ประสิทธิภาพการทำงานของนักพัฒนา                           | รายการประกาศช่วยให้ผู้ใช้สร้าง ทดสอบ ย้ายระบบ ดีบัก จัดทำเอกสาร หรือดำเนินงานซอฟต์แวร์                                               |
| เวิร์กโฟลว์ UI ข้อมูล และระบบอัตโนมัติ               | ขอบเขตชัดเจน ระบุข้อมูลรับรองที่จำเป็นอย่างชัดเจน และการกระทำที่มีความเสี่ยงมีเส้นทางสำหรับการตรวจทาน การรันจำลอง การแสดงตัวอย่าง หรือการยืนยัน |
| การรักษาความปลอดภัยเชิงป้องกัน การดูแล และการตรวจสอบการใช้งานในทางที่ผิด | เครื่องมือถูกวางกรอบไว้สำหรับการตรวจสอบที่ได้รับอนุญาต เก็บรักษาหลักฐาน และทำให้ขอบเขตการอนุมัติของมนุษย์ชัดเจน                          |
| เวิร์กโฟลว์ส่วนบุคคลหรือทีม                       | เวิร์กโฟลว์ใช้บัญชีตามความยินยอม การตั้งค่าที่โปร่งใส และสิทธิ์ที่ชัดเจน                                            |
| แค็ตตาล็อกที่ได้รับการดูแลรักษา                              | แต่ละรายการประกาศแตกต่างกัน มีประโยชน์ อธิบายอย่างถูกต้อง และได้รับการดูแลรักษาอย่างสมเหตุสมผล                                                |

บริบทมีความสำคัญ หัวข้อเดียวกันอาจยอมรับได้ในบริบทเชิงป้องกันแบบแคบหรือ
ตามความยินยอม และยอมรับไม่ได้เมื่อถูกจัดแพ็กเกจเป็นเวิร์กโฟลว์สำหรับการใช้งานในทางที่ผิด

## เนื้อหาที่ไม่อนุญาต

ClawHub ไม่โฮสต์เนื้อหาที่มีวัตถุประสงค์หลักเป็นการใช้งานในทางที่ผิด การหลอกลวง การดำเนินการที่ไม่ปลอดภัย
หรือการละเมิดสิทธิ์

| หมวดหมู่                                                    | ไม่อนุญาต                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| การเข้าถึงโดยไม่ได้รับอนุญาตหรือการเลี่ยงการรักษาความปลอดภัย                      | การเลี่ยงการยืนยันตัวตน การยึดบัญชี การใช้อัตราจำกัดในทางที่ผิด การยึดสายสดหรือเอเจนต์ การขโมยเซสชันที่นำกลับมาใช้ซ้ำได้ หรือโฟลว์การจับคู่อัตโนมัติสำหรับผู้ใช้ที่ไม่ได้รับอนุมัติ                                                                                                                                                   |
| การใช้งานแพลตฟอร์มในทางที่ผิดและการเลี่ยงการแบน                              | บัญชีล่องหนหลังถูกแบน การวอร์มหรือฟาร์มบัญชี การมีส่วนร่วมปลอม ระบบอัตโนมัติหลายบัญชี การโพสต์จำนวนมาก บอตสแปม หรือระบบอัตโนมัติที่สร้างขึ้นเพื่อหลีกเลี่ยงการตรวจจับ                                                                                                                                          |
| การฉ้อโกง กลโกง และเวิร์กโฟลว์ทางการเงินที่หลอกลวง             | ใบรับรองหรือใบแจ้งหนี้ปลอม โฟลว์การชำระเงินที่หลอกลวง การติดต่อหลอกลวง หลักฐานทางสังคมปลอม เวิร์กโฟลว์อัตลักษณ์สังเคราะห์เพื่อการฉ้อโกง หรือเครื่องมือใช้จ่าย/เรียกเก็บเงินที่ไม่มีการอนุมัติจากมนุษย์อย่างชัดเจน                                                                                                                    |
| การเสริมข้อมูลหรือการเฝ้าระวังที่ละเมิดความเป็นส่วนตัว                 | การดึงข้อมูลติดต่อเพื่อสแปม การเปิดเผยข้อมูลส่วนบุคคล การสะกดรอย การดึงลูกค้าเป้าหมายควบคู่กับการติดต่อโดยไม่ได้ร้องขอ การติดตามแบบลับ การจับคู่ชีวมิติที่ไม่ได้รับความยินยอม หรือการใช้ข้อมูลรั่วไหลหรือชุดข้อมูลจากการละเมิด                                                                                                                  |
| การแอบอ้างหรือการจัดการอัตลักษณ์โดยไม่ได้รับความยินยอม       | การสลับใบหน้า ฝาแฝดดิจิทัล อินฟลูเอนเซอร์โคลน บุคคลปลอม หรือเครื่องมืออื่นที่ใช้เพื่อแอบอ้างหรือทำให้เข้าใจผิด                                                                                                                                                                                                 |

## พฤติกรรมในมาร์เก็ตเพลสที่ไม่อนุญาต

ClawHub ยังตรวจสอบวิธีที่ผู้เผยแพร่ใช้มาร์เก็ตเพลสด้วย อย่าใช้ ClawHub เพื่อ
ปั่นการค้นพบ เมตริก สัญญาณความน่าเชื่อถือ ระบบดูแล หรือความสนใจของผู้ใช้

พฤติกรรมในมาร์เก็ตเพลสที่ไม่อนุญาตรวมถึง:

- การเผยแพร่จำนวนมากของรายการประกาศที่ใช้ความพยายามต่ำ ซ้ำซ้อน เป็นเพียง placeholder หรือ
  สร้างโดยเครื่อง ซึ่งดูเหมือนไม่มีคุณค่าจริงต่อผู้ใช้
- การท่วมพื้นผิวการค้นหาหรือหมวดหมู่ด้วย Skills หรือ Plugin ที่เกือบเหมือนกัน
- การเผยแพร่รายการประกาศหลายร้อยรายการที่มีการใช้งาน การดูแลรักษา ความชัดเจนของแหล่งที่มา
  หรือความแตกต่างที่มีความหมายเพียงเล็กน้อยหรือไม่มีเลย
- การเพิ่มจำนวนการติดตั้ง ดาวน์โหลด ดาว หรือเมตริกการมีส่วนร่วมอื่น ๆ
  อย่างไม่เป็นธรรมผ่านระบบอัตโนมัติ วงจรติดตั้งด้วยตนเอง บัญชีปลอม กิจกรรมที่ประสานกัน
  การมีส่วนร่วมแบบจ่ายเงิน หรือพฤติกรรมอื่นที่ไม่เป็นธรรมชาติ
- การสร้างหรือหมุนเวียนบัญชีเพื่อเลี่ยงการดูแล การแบน ขีดจำกัดผู้เผยแพร่ หรือ
  การตรวจทานของมาร์เก็ตเพลส
- การทำให้ผู้ใช้เข้าใจผิดเกี่ยวกับความเป็นเจ้าของ แหล่งที่มา ความสามารถ สถานะความปลอดภัย
  ข้อกำหนดการติดตั้ง หรือความเกี่ยวข้องกับโครงการหรือผู้เผยแพร่อื่น
- การอัปโหลดเนื้อหาซ้ำ ๆ ที่ถูกซ่อน ลบ หรือบล็อกไปแล้ว
  โดยไม่ได้แก้ไขปัญหาต้นเหตุ

การเผยแพร่ในปริมาณมากไม่ใช่การใช้งานในทางที่ผิดโดยอัตโนมัติ แค็ตตาล็อกขนาดใหญ่ยอมรับได้
เมื่อรายการประกาศแตกต่างกันอย่างมีความหมาย อธิบายอย่างถูกต้อง ได้รับการดูแลรักษา
และถูกใช้โดยผู้ใช้จริง แค็ตตาล็อกขนาดใหญ่จะกลายเป็นปัญหาด้านความน่าเชื่อถือและความปลอดภัยเมื่อ
ปริมาณมาพร้อมกับรายการประกาศที่บาง ซ้ำซ้อน ทำให้เข้าใจผิด ไม่ได้รับการดูแลรักษา หรือ
ถูกโปรโมตอย่างไม่เป็นธรรม

## สิทธิ์ในเนื้อหา

หากคุณเชื่อว่าเนื้อหาบน ClawHub ละเมิดลิขสิทธิ์หรือสิทธิ์อื่นของคุณ ให้ใช้
[คำขอด้านสิทธิ์ในเนื้อหา](/th/clawhub/content-rights) อย่าใช้รายงานมาร์เก็ตเพลสทั่วไป
สำหรับการร้องเรียนลิขสิทธิ์หรือสิทธิ์ เว้นแต่รายการประกาศนั้นจะไม่ปลอดภัย
เป็นอันตราย หรือทำให้เข้าใจผิดด้วย

## การตรวจทานและการบังคับใช้

ClawHub อาจใช้การตรวจสอบอัตโนมัติ สัญญาณเชิงสถิติของการใช้งานในทางที่ผิด รายงานจากผู้ใช้ และ
การตรวจทานโดยเจ้าหน้าที่ เพื่อระบุเนื้อหาที่ไม่ปลอดภัยหรือพฤติกรรมการเผยแพร่ที่ไม่เหมาะสม สัญญาณหนึ่ง
ไม่ได้พิสูจน์การใช้งานในทางที่ผิดด้วยตัวเอง แต่ช่วยให้ ClawHub ตัดสินใจว่าสิ่งใดต้องได้รับการตรวจทาน

เราอาจ:

- ซ่อน ระงับ ลบ ลบแบบ soft-delete หรือในกรณีที่ประเภทรองรับ
  ลบแบบ hard-delete รายการประกาศที่ละเมิด
- บล็อกการดาวน์โหลดหรือการติดตั้งสำหรับรุ่นที่ไม่ปลอดภัย
- เพิกถอน API token
- ลบเนื้อหาที่เกี่ยวข้องแบบ soft-delete
- จำกัดสิทธิ์การเผยแพร่
- แบนผู้กระทำผิดซ้ำหรือร้ายแรง

เราไม่รับประกันการบังคับใช้แบบเตือนก่อนสำหรับการใช้งานในทางที่ผิดที่ชัดเจน โปรดดู
[การดูแลและความปลอดภัยของบัญชี](/clawhub/moderation) สำหรับรายงาน การระงับโดยการดูแล
รายการประกาศที่ถูกซ่อน การแบน และสถานะบัญชี
