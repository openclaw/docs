# การกลั่นกรองและความปลอดภัยของบัญชี

ClawHub เปิดให้เผยแพร่ได้ แต่พื้นผิวการค้นพบสาธารณะและการติดตั้งยังคง
ต้องมีมาตรการป้องกัน รายงาน การพักไว้เพื่อกลั่นกรอง รายการที่ถูกซ่อน และการดำเนินการกับบัญชี
ช่วยปกป้องผู้ใช้เมื่อรีลีสหรือบัญชีดูไม่ปลอดภัย ทำให้เข้าใจผิด หรืออยู่นอก
นโยบาย

หน้านี้ครอบคลุมการกลั่นกรองและสถานะบัญชี สำหรับป้ายกำกับการตรวจสอบ เช่น
`Pass`, `Review`, `Warn`, `Malicious` และระดับความเสี่ยง โปรดดู
[การตรวจสอบความปลอดภัย](/clawhub/security-audits)

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage) สำหรับข้อกังวลเรื่องลิขสิทธิ์หรือสิทธิ์เนื้อหาอื่น ๆ
ให้ใช้ [คำขอสิทธิ์เนื้อหา](/clawhub/content-rights)

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน Skills, Plugin และแพ็กเกจได้

ใช้รายงาน ClawHub เฉพาะกับเนื้อหา marketplace ที่ไม่ปลอดภัย เช่น:

- รายการที่เป็นอันตราย
- เมทาดาทาที่ทำให้เข้าใจผิด
- ข้อมูลประจำตัวหรือข้อกำหนดสิทธิ์ที่ไม่ได้ประกาศ
- คำแนะนำการติดตั้งที่น่าสงสัย
- การแอบอ้าง
- การจดทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

ใช้ปุ่ม **รายงาน Skills** บนหน้า Skills หรือคำสั่ง/API สำหรับรายงานแพ็กเกจ

อย่าใช้รายงาน ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ Skills หรือ
Plugin บุคคลที่สามเอง ให้รายงานสิ่งเหล่านั้นโดยตรงต่อผู้เผยแพร่หรือที่เก็บซอร์ส
ที่ลิงก์จากรายการ ClawHub ไม่ได้ดูแลหรือแพตช์โค้ด
Skills หรือ Plugin บุคคลที่สาม

GitHub Security Advisories สำหรับ `openclaw/clawhub` ใช้สำหรับช่องโหว่ใน
ClawHub เอง ตัวอย่างรวมถึงบั๊กในเว็บไซต์, API, CLI, รีจิสทรี, การตรวจสอบสิทธิ์,
การสแกน, การกลั่นกรอง หรือขอบเขตความเชื่อถือของการดาวน์โหลด/ติดตั้ง อย่าใช้ advisory ของ ClawHub
สำหรับช่องโหว่ใน Skills หรือ Plugin บุคคลที่สาม

รายงานที่ดีมีความเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้ระบบรายงานในทางที่ผิดก็อาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

## การอ้างสิทธิ์องค์กรและ namespace

ข้อพิพาทเรื่องความเป็นเจ้าขององค์กร แบรนด์ ขอบเขตแพ็กเกจ แฮนเดิลเจ้าของ หรือ namespace ควร
ใช้กระบวนการ [การอ้างสิทธิ์องค์กรและ Namespace](/clawhub/namespace-claims) ไม่ใช่
ขั้นตอนรายงานในผลิตภัณฑ์หรือแบบฟอร์มอุทธรณ์บัญชี

ใช้กระบวนการนั้นเมื่อคุณต้องการให้เจ้าหน้าที่ ClawHub ตรวจสอบหลักฐานที่ไม่อ่อนไหวว่า
namespace ควรถูกสงวน โอน เปลี่ยนชื่อ ซ่อน กักกัน ทำ alias
หรือรับการตรวจสอบในลักษณะอื่น อย่าใส่ความลับ เอกสารส่วนตัว ไฟล์กฎหมายส่วนตัว
เอกสารระบุตัวตนส่วนบุคคล โทเค็น API หรือโทเค็น DNS challenge ใน
issue สาธารณะ

## การพักไว้เพื่อกลั่นกรอง

ผลการตรวจพบที่รุนแรงหรือปัญหาด้านนโยบายบางอย่างอาจทำให้ผู้เผยแพร่หรือรายการถูก
พักไว้เพื่อกลั่นกรอง เมื่อเกิดกรณีนี้ เนื้อหาที่ได้รับผลกระทบอาจถูกซ่อนจากการค้นพบ
สาธารณะ หรือการเผยแพร่ในอนาคตอาจเริ่มต้นในสถานะซ่อนจนกว่าปัญหาจะได้รับการตรวจสอบ

การพักไว้เพื่อกลั่นกรองมีไว้เพื่อปกป้องผู้ใช้ในขณะที่ ClawHub แก้ไขกรณีที่มีความเสี่ยงสูง
และยังสามารถยกเลิกได้เมื่อยืนยันแล้วว่าเป็นผลบวกลวง

## รายการที่ถูกซ่อนหรือถูกบล็อก

รายการอาจถูกพักไว้ ซ่อน กักกัน เพิกถอน หรือไม่พร้อมใช้งานในลักษณะอื่นบน
พื้นผิวการติดตั้งสาธารณะ

หากคุณเห็นหนึ่งในสถานะเหล่านี้ อย่าติดตั้งรีลีสนั้น เว้นแต่เจ้าของ
จะแก้ไขปัญหาหรือการกลั่นกรองกู้คืนรายการนั้น

เจ้าของอาจยังเห็น diagnostics สำหรับรายการของตนเองที่ถูกพักไว้หรือถูกซ่อน
diagnostics เหล่านี้ช่วยอธิบายสิ่งที่เกิดขึ้นและสิ่งที่ต้องเปลี่ยนก่อนที่
รายการจะกลับสู่พื้นผิวสาธารณะได้

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบาย ClawHub อาจสูญเสียสิทธิ์การเผยแพร่ การละเมิดที่รุนแรงอาจ
ส่งผลให้บัญชีถูกแบน โทเค็นถูกเพิกถอน เนื้อหาถูกซ่อน หรือรายการถูกนำออก
สัญญาณแรงกดดันจากการละเมิดของผู้เผยแพร่จะถูกตรวจสอบทุกวัน สัญญาณที่ถึง
เกณฑ์อาจถูกแบนของ ClawHub สามารถเรียกคำเตือนอัตโนมัติได้ หากการสแกนที่มีสิทธิ์ครั้งถัดไป
หลังเส้นตายคำเตือนยังคงวางผู้เผยแพร่ไว้ในเกณฑ์อาจถูกแบน
ClawHub อาจใช้การดำเนินการกับบัญชีโดยอัตโนมัติ
สัญญาณการตรวจสอบเชิงเวลาแบบจำกัดและความเชื่อมั่นต่ำจะไม่อยู่ในการบังคับใช้
อัตโนมัติ

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้โทเค็น API ของ ClawHub ได้ หากการตรวจสอบสิทธิ์ CLI
เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้เว็บ UI เพื่อตรวจสอบ
สถานะบัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อกโดยการแบนหรือบัญชีที่ถูกปิดใช้งาน
ให้ใช้ [แบบฟอร์มอุทธรณ์ของ ClawHub](https://appeals.openclaw.ai/) เพื่อให้ตรวจสอบการกู้คืน

หากอีเมลที่ถูกเรียกโดยสแกนเนอร์ระบุว่าเวอร์ชันของ Skills หรือ Plugin เป็นอันตราย
ให้ดาวน์โหลดผลการสแกนที่จัดเก็บไว้สำหรับเวอร์ชันที่ส่งซึ่งถูกบล็อก:
`clawhub scan download <slug> --version <version>` สำหรับ Plugin ให้เพิ่ม
`--kind plugin` ตรวจสอบผลลัพธ์การสแกน แก้ไขรายการ เพิ่มหมายเลขเวอร์ชัน
และอัปโหลดเวอร์ชันที่แก้ไขแล้ว

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลดผลบวกลวงและเพิ่มความไว้วางใจของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และ changelog ให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้คลุมเครือ
- ลิงก์ไปยังซอร์สเมื่อเป็นไปได้
- ใช้ dry run ก่อนเผยแพร่ Plugin
- ตอบให้ชัดเจนหากผู้ใช้หรือผู้กลั่นกรองถามเกี่ยวกับพฤติกรรมของรีลีส
