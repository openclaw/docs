# การกลั่นกรองและความปลอดภัยของบัญชี

ClawHub เปิดให้เผยแพร่ได้ แต่พื้นที่การค้นพบและติดตั้งสาธารณะยังคง
ต้องมีมาตรการป้องกัน รายงาน การระงับเพื่อกลั่นกรอง รายการที่ซ่อน และมาตรการกับบัญชี
ช่วยปกป้องผู้ใช้เมื่อรีลีสหรือบัญชีดูไม่ปลอดภัย ทำให้เข้าใจผิด หรือ
ไม่เป็นไปตามนโยบาย

หน้านี้ครอบคลุมการกลั่นกรองและสถานะบัญชี สำหรับป้ายกำกับการตรวจสอบ เช่น
`Pass`, `Review`, `Warn`, `Malicious` และระดับความเสี่ยง โปรดดู
[การตรวจสอบความปลอดภัย](/th/clawhub/security-audits)

ดูเพิ่มเติมที่ [ความปลอดภัย](/th/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage) สำหรับข้อกังวลด้านลิขสิทธิ์หรือสิทธิ์
ในเนื้อหาอื่น ๆ ให้ใช้ [คำขอสิทธิ์เนื้อหา](/th/clawhub/content-rights)

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้แล้วสามารถรายงานสกิล Plugin และแพ็กเกจได้

ใช้รายงานของ ClawHub เฉพาะกับเนื้อหา marketplace ที่ไม่ปลอดภัย เช่น:

- รายการที่เป็นอันตราย
- metadata ที่ทำให้เข้าใจผิด
- ข้อมูลประจำตัวหรือข้อกำหนดสิทธิ์ที่ไม่ได้ประกาศ
- คำแนะนำการติดตั้งที่น่าสงสัย
- การแอบอ้างบุคคลหรือองค์กร
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

ใช้ปุ่ม **รายงานสกิล** บนหน้าสกิล หรือใช้คำสั่ง/API สำหรับรายงาน
แพ็กเกจ

อย่าใช้รายงานของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของสกิลหรือ
Plugin บุคคลที่สามเอง ให้รายงานโดยตรงต่อผู้เผยแพร่หรือคลังซอร์ส
ที่ลิงก์จากรายการ ClawHub ไม่ได้ดูแลหรือแพตช์
โค้ดสกิลหรือ Plugin บุคคลที่สาม

GitHub Security Advisories สำหรับ `openclaw/clawhub` ใช้สำหรับช่องโหว่ใน
ClawHub เอง ตัวอย่างได้แก่บั๊กในเว็บไซต์ API, CLI, registry, auth,
การสแกน การกลั่นกรอง หรือขอบเขตความน่าเชื่อถือของการดาวน์โหลด/ติดตั้ง อย่าใช้คำแนะนำของ ClawHub
สำหรับช่องโหว่ในสกิลหรือ Plugin บุคคลที่สาม

รายงานที่ดีต้องเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้การรายงานในทางที่ผิดอาจนำไปสู่
มาตรการกับบัญชีได้เช่นกัน

## การอ้างสิทธิ์องค์กรและ namespace

ข้อพิพาทเรื่องความเป็นเจ้าขององค์กร แบรนด์ package-scope, owner-handle หรือ namespace ควร
ใช้กระบวนการ [การอ้างสิทธิ์องค์กรและ Namespace](/th/clawhub/namespace-claims) ไม่ใช่
ขั้นตอนรายงานในผลิตภัณฑ์หรือแบบฟอร์มอุทธรณ์บัญชี

ใช้กระบวนการนั้นเมื่อคุณต้องการให้เจ้าหน้าที่ ClawHub ตรวจสอบหลักฐานที่ไม่อ่อนไหวว่า
namespace ควรถูกสำรอง โอน เปลี่ยนชื่อ ซ่อน กักกัน ทำ alias
หรือตรวจสอบด้วยวิธีอื่น อย่าใส่ความลับ เอกสารส่วนตัว ไฟล์กฎหมายส่วนตัว
เอกสารยืนยันตัวตนส่วนบุคคล API tokens หรือ DNS challenge tokens ใน
issue สาธารณะ

## การระงับเพื่อกลั่นกรอง

ผลการตรวจพบร้ายแรงหรือปัญหานโยบายบางอย่างอาจทำให้ผู้เผยแพร่หรือรายการอยู่ภายใต้
การระงับเพื่อกลั่นกรอง เมื่อเกิดเหตุการณ์นี้ เนื้อหาที่ได้รับผลกระทบอาจถูกซ่อนจากการค้นพบ
สาธารณะ หรือการเผยแพร่ในอนาคตอาจเริ่มต้นเป็นแบบซ่อนจนกว่าจะมีการตรวจสอบปัญหา

การระงับเพื่อกลั่นกรองมีไว้เพื่อปกป้องผู้ใช้ขณะที่ ClawHub แก้ไขกรณีที่มีความเสี่ยงสูง
และยังสามารถยกเลิกได้เมื่อยืนยันว่าเป็นผลบวกลวง

## รายการที่ซ่อนหรือถูกบล็อก

รายการอาจถูกระงับ ซ่อน กักกัน เพิกถอน หรือไม่พร้อมใช้งานด้วยวิธีอื่นบน
พื้นที่ติดตั้งสาธารณะ

หากคุณเห็นสถานะใดสถานะหนึ่งเหล่านี้ อย่าติดตั้งรีลีสนั้น เว้นแต่เจ้าของ
จะแก้ไขปัญหาแล้วหรือการกลั่นกรองคืนสถานะให้

เจ้าของอาจยังคงเห็นข้อมูลวินิจฉัยสำหรับรายการของตนเองที่ถูกระงับหรือซ่อนอยู่ ข้อมูลวินิจฉัยเหล่านี้
ช่วยอธิบายว่าเกิดอะไรขึ้นและต้องเปลี่ยนแปลงอะไรบ้างก่อนที่
รายการจะกลับสู่พื้นที่สาธารณะได้

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบายของ ClawHub อาจสูญเสียสิทธิ์การเผยแพร่ การละเมิดร้ายแรงอาจ
ทำให้ถูกแบนบัญชี เพิกถอน token ซ่อนเนื้อหา หรือลบรายการ
สัญญาณแรงกดดันจากการละเมิดของผู้เผยแพร่จะถูกตรวจสอบทุกวัน สัญญาณที่ถึง
เกณฑ์การอาจแบนของ ClawHub สามารถทริกเกอร์คำเตือนอัตโนมัติได้ หากการสแกนที่มีสิทธิ์ครั้งถัดไป
หลังเส้นตายคำเตือนยังคงจัดผู้เผยแพร่อยู่ใน
เกณฑ์การอาจแบน ClawHub อาจใช้มาตรการกับบัญชีโดยอัตโนมัติ
สัญญาณการตรวจสอบที่มีความเชื่อมั่นต่ำกว่าและจำกัดตามช่วงเวลาจะไม่เข้าสู่
การบังคับใช้อัตโนมัติ

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้ API tokens ของ ClawHub ได้ หากการยืนยันตัวตน CLI
เริ่มล้มเหลวหลังจากมีมาตรการกับบัญชี ให้ลงชื่อเข้าใช้ UI บนเว็บเพื่อตรวจสอบ
สถานะบัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อกโดยการแบนหรือบัญชีที่ถูกปิดใช้งาน
ให้ใช้ [แบบฟอร์มอุทธรณ์ ClawHub](https://appeals.openclaw.ai/) เพื่อให้ตรวจสอบการกู้คืน

หากอีเมลที่ทริกเกอร์โดย scanner ระบุเวอร์ชันของสกิลหรือ Plugin ว่าเป็นอันตราย
ให้ดาวน์โหลดผลการสแกนที่จัดเก็บไว้สำหรับเวอร์ชันที่ส่งและถูกบล็อก:
`clawhub scan download <slug> --version <version>` สำหรับ Plugin ให้เพิ่ม
`--kind plugin` ตรวจสอบผลลัพธ์การสแกน แก้ไขรายการ เพิ่มหมายเลขเวอร์ชัน
และอัปโหลดเวอร์ชันที่แก้ไขแล้ว

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลดผลบวกลวงและเพิ่มความไว้วางใจของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และ changelog ให้ถูกต้อง
- ประกาศ environment variables และสิทธิ์ที่จำเป็น
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้คลุมเครือ
- ลิงก์ไปยังซอร์สเมื่อเป็นไปได้
- ใช้ dry runs ก่อนเผยแพร่ Plugin
- ตอบอย่างชัดเจนหากผู้ใช้หรือผู้กลั่นกรองถามเกี่ยวกับพฤติกรรมของรีลีส
