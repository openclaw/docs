# การกลั่นกรองและความปลอดภัยของบัญชี

ClawHub เปิดให้เผยแพร่ได้ แต่พื้นผิวการค้นหาและติดตั้งแบบสาธารณะยังคง
ต้องมีมาตรการป้องกัน การรายงาน การระงับเพื่อตรวจสอบ การซ่อนรายการ และการดำเนินการกับบัญชี
ช่วยปกป้องผู้ใช้เมื่อรุ่นที่เผยแพร่หรือบัญชีดูไม่ปลอดภัย ชวนให้เข้าใจผิด หรือไม่เป็นไป
ตามนโยบาย

หน้านี้ครอบคลุมการกลั่นกรองและสถานะบัญชี สำหรับป้ายกำกับการตรวจสอบ เช่น
`Pass`, `Review`, `Warn`, `Malicious` และระดับความเสี่ยง โปรดดู
[การตรวจสอบความปลอดภัย](/clawhub/security-audits)

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage) สำหรับข้อกังวลเรื่องลิขสิทธิ์หรือสิทธิ
ในเนื้อหาอื่นๆ ให้ใช้ [คำขอเกี่ยวกับสิทธิในเนื้อหา](/clawhub/content-rights)

## การรายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน Skills, Plugin และแพ็กเกจได้

ใช้การรายงานของ ClawHub เฉพาะกับเนื้อหาในมาร์เก็ตเพลสที่ไม่ปลอดภัย เช่น:

- รายการที่เป็นอันตราย
- ข้อมูลเมตาที่ชวนให้เข้าใจผิด
- ข้อกำหนดด้านข้อมูลประจำตัวหรือสิทธิ์ที่ไม่ได้ประกาศ
- คำแนะนำการติดตั้งที่น่าสงสัย
- การแอบอ้างบุคคลอื่น
- การจดทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

ใช้ปุ่ม **Report skill** บนหน้าของ Skill หรือใช้คำสั่ง/API
การรายงานแพ็กเกจสำหรับแพ็กเกจ

อย่าใช้การรายงานของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ Skill หรือ
Plugin บุคคลที่สาม ให้รายงานโดยตรงไปยังผู้เผยแพร่หรือคลังซอร์ส
ที่ลิงก์ไว้จากรายการ ClawHub ไม่ได้ดูแลหรือแก้ไขโค้ด
ของ Skill หรือ Plugin บุคคลที่สาม

GitHub Security Advisories สำหรับ `openclaw/clawhub` ใช้สำหรับช่องโหว่ใน
ClawHub เอง ตัวอย่างได้แก่ข้อบกพร่องในเว็บไซต์, API, CLI, รีจิสทรี, การยืนยันตัวตน,
การสแกน, การกลั่นกรอง หรือขอบเขตความเชื่อถือของการดาวน์โหลด/ติดตั้ง อย่าใช้คำแนะนำด้านความปลอดภัยของ ClawHub
สำหรับช่องโหว่ใน Skills หรือ Plugin บุคคลที่สาม

รายงานที่ดีต้องเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้ระบบรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

## การอ้างสิทธิ์ในองค์กรและเนมสเปซ

ข้อพิพาทเกี่ยวกับความเป็นเจ้าขององค์กร แบรนด์ ขอบเขตแพ็กเกจ แฮนเดิลของเจ้าของ หรือเนมสเปซ ควร
ใช้กระบวนการ [การอ้างสิทธิ์ในองค์กรและเนมสเปซ](/clawhub/namespace-claims) ไม่ใช่
ขั้นตอนการรายงานในผลิตภัณฑ์หรือแบบฟอร์มอุทธรณ์บัญชี

ใช้กระบวนการดังกล่าวเมื่อต้องการให้เจ้าหน้าที่ ClawHub ตรวจสอบหลักฐานที่ไม่ละเอียดอ่อนว่า
ควรสงวน โอน เปลี่ยนชื่อ ซ่อน กักกัน สร้างนามแฝง
หรือตรวจสอบเนมสเปซในลักษณะอื่น อย่าใส่ความลับ เอกสารส่วนตัว ไฟล์ทางกฎหมาย
ส่วนตัว เอกสารระบุตัวตนส่วนบุคคล โทเค็น API หรือโทเค็นการยืนยัน DNS ใน
ปัญหาสาธารณะ

## การระงับเพื่อตรวจสอบ

ผลการตรวจสอบหรือปัญหาด้านนโยบายที่ร้ายแรงบางอย่างอาจทำให้ผู้เผยแพร่หรือรายการถูก
ระงับเพื่อตรวจสอบ เมื่อเกิดกรณีนี้ เนื้อหาที่ได้รับผลกระทบอาจถูกซ่อนจากการค้นหา
สาธารณะ หรือการเผยแพร่ในอนาคตอาจเริ่มต้นในสถานะซ่อนจนกว่าจะตรวจสอบปัญหาแล้ว

การระงับเพื่อตรวจสอบมีไว้เพื่อปกป้องผู้ใช้ระหว่างที่ ClawHub แก้ไขกรณีที่มีความเสี่ยงสูง
และสามารถยกเลิกได้เมื่อยืนยันว่าเป็นผลบวกลวง

## รายการที่ถูกซ่อนหรือบล็อก

รายการอาจถูกระงับ ซ่อน กักกัน เพิกถอน หรือไม่พร้อมใช้งานในลักษณะอื่นบน
พื้นผิวการติดตั้งสาธารณะ

หากพบสถานะใดสถานะหนึ่งเหล่านี้ อย่าติดตั้งรุ่นที่เผยแพร่จนกว่าเจ้าของ
จะแก้ไขปัญหาหรือฝ่ายกลั่นกรองจะคืนสถานะให้

เจ้าของอาจยังคงเห็นข้อมูลวินิจฉัยสำหรับรายการของตนที่ถูกระงับหรือซ่อน ข้อมูล
วินิจฉัยเหล่านี้ช่วยอธิบายสิ่งที่เกิดขึ้นและสิ่งที่ต้องเปลี่ยนแปลงก่อนที่
รายการจะกลับสู่พื้นผิวสาธารณะได้

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบายของ ClawHub อาจสูญเสียสิทธิ์ในการเผยแพร่ การใช้งานในทางที่ผิดอย่างร้ายแรงอาจ
ส่งผลให้บัญชีถูกแบน โทเค็นถูกเพิกถอน เนื้อหาถูกซ่อน หรือรายการถูกลบ
ระบบจะตรวจสอบสัญญาณแรงกดดันจากการใช้งานในทางที่ผิดของผู้เผยแพร่ทุกวัน สัญญาณที่ถึง
เกณฑ์การพิจารณาแบนของ ClawHub อาจทำให้เกิดคำเตือนอัตโนมัติ หากการสแกนครั้งถัดไป
ที่มีสิทธิ์หลังพ้นกำหนดเวลาคำเตือนยังคงจัดให้ผู้เผยแพร่อยู่ใน
เกณฑ์การพิจารณาแบน ClawHub อาจดำเนินการกับบัญชีโดยอัตโนมัติ
สัญญาณที่มีความเชื่อมั่นต่ำกว่าและสัญญาณการตรวจสอบตามช่วงเวลาที่มีขอบเขตจะไม่เข้าสู่
การบังคับใช้อัตโนมัติ

บัญชีที่ถูกลบ ถูกแบน หรือปิดใช้งานไม่สามารถใช้โทเค็น API ของ ClawHub ได้ หากการยืนยันตัวตนของ CLI
เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ UI บนเว็บเพื่อตรวจสอบสถานะ
บัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อกเนื่องจากการแบนหรือบัญชีถูกปิดใช้งาน
ให้ใช้ [แบบฟอร์มอุทธรณ์ของ ClawHub](https://appeals.openclaw.ai/) เพื่อขอรับการตรวจสอบการกู้คืน

หากอีเมลที่เกิดจากเครื่องสแกนระบุว่า Skill หรือ Plugin เวอร์ชันหนึ่งเป็นอันตราย
ให้ดาวน์โหลดผลการสแกนที่จัดเก็บไว้สำหรับเวอร์ชันที่ส่งและถูกบล็อก:
`clawhub scan download <slug> --version <version>` สำหรับ Plugin ให้เพิ่ม
`--kind plugin` ตรวจสอบผลลัพธ์การสแกน แก้ไขรายการ เพิ่มหมายเลขเวอร์ชัน
และอัปโหลดเวอร์ชันที่แก้ไขแล้ว

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลดผลบวกลวงและเพิ่มความไว้วางใจของผู้ใช้:

- รักษาชื่อ ข้อมูลสรุป แท็ก และบันทึกการเปลี่ยนแปลงให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้อ่านเข้าใจได้ยาก
- ลิงก์ไปยังซอร์สเมื่อทำได้
- ใช้การทดลองรันก่อนเผยแพร่ Plugin
- ตอบอย่างชัดเจนหากผู้ใช้หรือผู้กลั่นกรองสอบถามเกี่ยวกับพฤติกรรมของรุ่นที่เผยแพร่
