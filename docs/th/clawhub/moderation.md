# การกลั่นกรองและความปลอดภัยของบัญชี

ClawHub เปิดให้เผยแพร่ได้ แต่พื้นผิวสำหรับการค้นพบแบบสาธารณะและการติดตั้งยังคง
ต้องมีแนวป้องกัน รายงาน การพักไว้เพื่อตรวจสอบการกลั่นกรอง รายการที่ถูกซ่อน และการดำเนินการกับบัญชี
ช่วยปกป้องผู้ใช้เมื่อรีลีสหรือบัญชีดูไม่ปลอดภัย ทำให้เข้าใจผิด หรืออยู่นอก
นโยบาย

หน้านี้ครอบคลุมการกลั่นกรองและสถานะบัญชี สำหรับป้ายกำกับการตรวจสอบ เช่น
`Pass`, `Review`, `Warn`, `Malicious` และระดับความเสี่ยง โปรดดู
[การตรวจสอบความปลอดภัย](/clawhub/security-audits)

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage) สำหรับข้อกังวลด้านลิขสิทธิ์หรือสิทธิ์เนื้อหาอื่นๆ
ให้ใช้ [คำขอสิทธิ์เนื้อหา](/clawhub/content-rights)

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน Skills, Plugin และแพ็กเกจได้

ใช้รายงานของ ClawHub เฉพาะกับเนื้อหาในมาร์เก็ตเพลสที่ไม่ปลอดภัย เช่น:

- รายการที่เป็นอันตราย
- เมทาดาทาที่ทำให้เข้าใจผิด
- ข้อมูลประจำตัวหรือข้อกำหนดสิทธิ์ที่ไม่ได้ประกาศ
- คำแนะนำการติดตั้งที่น่าสงสัย
- การแอบอ้างตัวตน
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

ใช้ปุ่ม **รายงาน Skills** บนหน้า Skills หรือคำสั่ง/API สำหรับรายงานแพ็กเกจ
สำหรับแพ็กเกจ

อย่าใช้รายงานของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ Skills หรือ
Plugin ของบุคคลที่สาม ให้รายงานสิ่งเหล่านั้นโดยตรงไปยังผู้เผยแพร่หรือคลังซอร์ส
ที่ลิงก์จากรายการ ClawHub ไม่ได้ดูแลหรือแพตช์
โค้ด Skills หรือ Plugin ของบุคคลที่สาม

GitHub Security Advisories สำหรับ `openclaw/clawhub` ใช้สำหรับช่องโหว่ใน
ClawHub เอง ตัวอย่างได้แก่บั๊กในเว็บไซต์, API, CLI, รีจิสทรี, auth,
การสแกน, การกลั่นกรอง หรือขอบเขตความเชื่อถือของการดาวน์โหลด/ติดตั้ง อย่าใช้คำแนะนำของ ClawHub
สำหรับช่องโหว่ใน Skills หรือ Plugin ของบุคคลที่สาม

รายงานที่ดีต้องเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้ระบบรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

## การอ้างสิทธิ์องค์กรและเนมสเปซ

ข้อพิพาทเกี่ยวกับความเป็นเจ้าขององค์กร แบรนด์ ขอบเขตแพ็กเกจ แฮนเดิลเจ้าของ หรือเนมสเปซควร
ใช้กระบวนการ [การอ้างสิทธิ์องค์กรและเนมสเปซ](/clawhub/namespace-claims) ไม่ใช่
โฟลว์รายงานในผลิตภัณฑ์หรือแบบฟอร์มอุทธรณ์บัญชี

ใช้กระบวนการนั้นเมื่อคุณต้องการให้เจ้าหน้าที่ ClawHub ตรวจสอบหลักฐานที่ไม่ละเอียดอ่อนว่า
เนมสเปซควรถูกสงวนไว้ โอน เปลี่ยนชื่อ ซ่อน กักกัน ทำเป็นนามแฝง
หรือตรวจสอบด้วยวิธีอื่น อย่าใส่ความลับ เอกสารส่วนตัว ไฟล์กฎหมายส่วนตัว
เอกสารระบุตัวตนส่วนบุคคล โทเค็น API หรือโทเค็น DNS challenge ใน
issue สาธารณะ

## การพักไว้เพื่อตรวจสอบการกลั่นกรอง

ผลการตรวจพบที่ร้ายแรงบางอย่างหรือปัญหาด้านนโยบายอาจทำให้ผู้เผยแพร่หรือรายการถูก
พักไว้เพื่อตรวจสอบการกลั่นกรอง เมื่อเกิดกรณีนี้ เนื้อหาที่ได้รับผลกระทบอาจถูกซ่อนจาก
การค้นพบแบบสาธารณะ หรือการเผยแพร่ในอนาคตอาจเริ่มต้นแบบถูกซ่อนไว้จนกว่าปัญหาจะได้รับการตรวจสอบ

การพักไว้เพื่อตรวจสอบการกลั่นกรองมีไว้เพื่อปกป้องผู้ใช้ขณะที่ ClawHub จัดการกรณีที่มีความเสี่ยงสูง
และยังสามารถยกเลิกได้เมื่อยืนยันว่าเป็นผลบวกปลอม

## รายการที่ถูกซ่อนหรือถูกบล็อก

รายการอาจถูกพักไว้ ซ่อน กักกัน เพิกถอน หรือไม่พร้อมใช้งานด้วยวิธีอื่นบน
พื้นผิวการติดตั้งสาธารณะ

หากคุณเห็นสถานะเหล่านี้ อย่าติดตั้งรีลีสนั้น เว้นแต่เจ้าของ
จะแก้ไขปัญหาหรือการกลั่นกรองคืนสถานะให้

เจ้าของอาจยังเห็นข้อมูลวินิจฉัยสำหรับรายการของตนเองที่ถูกพักไว้หรือถูกซ่อน ข้อมูลวินิจฉัยเหล่านี้
ช่วยอธิบายว่าเกิดอะไรขึ้นและต้องเปลี่ยนแปลงอะไรบ้างก่อนที่
รายการจะกลับสู่พื้นผิวสาธารณะได้

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบายของ ClawHub อาจสูญเสียสิทธิ์การเผยแพร่ การละเมิดที่ร้ายแรงอาจ
นำไปสู่การแบนบัญชี การเพิกถอนโทเค็น เนื้อหาที่ถูกซ่อน หรือรายการที่ถูกลบ
สัญญาณแรงกดดันจากการละเมิดของผู้เผยแพร่จะถูกตรวจสอบทุกวัน สัญญาณที่ถึง
เกณฑ์อาจถูกแบนของ ClawHub อาจทริกเกอร์คำเตือนอัตโนมัติ หากการสแกนที่มีสิทธิ์ครั้งถัดไป
หลังเส้นตายคำเตือนยังคงจัดให้ผู้เผยแพร่อยู่ใน
เกณฑ์อาจถูกแบน ClawHub อาจใช้การดำเนินการกับบัญชีโดยอัตโนมัติ
สัญญาณการตรวจสอบที่มีความเชื่อมั่นต่ำกว่าและจำกัดตามช่วงเวลาจะไม่ถูกนำเข้า
การบังคับใช้อัตโนมัติ

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้โทเค็น API ของ ClawHub ได้ หาก CLI auth
เริ่มล้มเหลวหลังการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ UI บนเว็บเพื่อตรวจสอบ
สถานะบัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อกโดยการแบนหรือบัญชีที่ถูกปิดใช้งาน
ให้ใช้ [แบบฟอร์มอุทธรณ์ ClawHub](https://appeals.openclaw.ai/) เพื่อขอการตรวจสอบการกู้คืน

หากอีเมลที่ทริกเกอร์โดยสแกนเนอร์ระบุว่าเวอร์ชัน Skills หรือ Plugin เป็นอันตราย
ให้ดาวน์โหลดผลการสแกนที่จัดเก็บไว้สำหรับเวอร์ชันที่ส่งมาซึ่งถูกบล็อก:
`clawhub scan download <slug> --version <version>` สำหรับ Plugin ให้เพิ่ม
`--kind plugin` ตรวจสอบผลลัพธ์การสแกน แก้ไขรายการ เพิ่มหมายเลขเวอร์ชัน
และอัปโหลดเวอร์ชันที่แก้ไขแล้ว

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลดผลบวกปลอมและเพิ่มความเชื่อมั่นของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และบันทึกการเปลี่ยนแปลงให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- หลีกเลี่ยงคำสั่งติดตั้งที่ถูกทำให้อ่านยาก
- ลิงก์ไปยังซอร์สเมื่อเป็นไปได้
- ใช้ dry runs ก่อนเผยแพร่ Plugin
- ตอบอย่างชัดเจนหากผู้ใช้หรือผู้กลั่นกรองถามเกี่ยวกับพฤติกรรมของรีลีส
