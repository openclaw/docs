# การกลั่นกรองและความปลอดภัยของบัญชี

ClawHub เปิดให้เผยแพร่ได้ แต่พื้นผิวสำหรับการค้นพบแบบสาธารณะและการติดตั้งยังคง
ต้องมีรั้วป้องกัน รายงาน การพักไว้เพื่อการกลั่นกรอง รายการที่ถูกซ่อน และการดำเนินการกับบัญชี
ช่วยปกป้องผู้ใช้เมื่อรีลีสหรือบัญชีดูไม่ปลอดภัย ทำให้เข้าใจผิด หรือไม่เป็นไปตาม
นโยบาย

หน้านี้ครอบคลุมการกลั่นกรองและสถานะบัญชี สำหรับป้ายกำกับการตรวจสอบ เช่น
`Pass`, `Review`, `Warn`, `Malicious` และระดับความเสี่ยง โปรดดู
[การตรวจสอบความปลอดภัย](/clawhub/security-audits)

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage) สำหรับข้อกังวลด้านลิขสิทธิ์หรือสิทธิ์ในเนื้อหาอื่น ๆ
ให้ใช้ [คำขอเกี่ยวกับสิทธิ์ในเนื้อหา](/clawhub/content-rights)

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน Skills, Plugin และแพ็กเกจได้

ใช้รายงานของ ClawHub เฉพาะกับเนื้อหา marketplace ที่ไม่ปลอดภัย เช่น:

- รายการที่เป็นอันตราย
- เมตาดาต้าที่ทำให้เข้าใจผิด
- ข้อมูลรับรองหรือข้อกำหนดสิทธิ์ที่ไม่ได้ประกาศ
- คำแนะนำการติดตั้งที่น่าสงสัย
- การแอบอ้างตัวตน
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

ใช้ปุ่ม **รายงาน Skills** บนหน้า Skills หรือคำสั่ง/API สำหรับรายงานแพ็กเกจ
สำหรับแพ็กเกจ

อย่าใช้รายงานของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ Skills หรือ
Plugin ภายนอก ให้รายงานโดยตรงต่อผู้เผยแพร่หรือคลังซอร์ส
ที่ลิงก์จากรายการ ClawHub ไม่ได้ดูแลหรือแพตช์
โค้ด Skills หรือ Plugin ภายนอก

GitHub Security Advisories สำหรับ `openclaw/clawhub` ใช้สำหรับช่องโหว่ใน
ClawHub เอง ตัวอย่างรวมถึงบั๊กในเว็บไซต์, API, CLI, รีจิสทรี, การยืนยันตัวตน,
การสแกน, การกลั่นกรอง หรือขอบเขตความเชื่อถือในการดาวน์โหลด/ติดตั้ง อย่าใช้ ClawHub
advisories สำหรับช่องโหว่ใน Skills หรือ Plugin ภายนอก

รายงานที่ดีต้องเฉพาะเจาะจงและดำเนินการได้ การใช้ระบบรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

## การอ้างสิทธิ์องค์กรและเนมสเปซ

ข้อพิพาทเกี่ยวกับองค์กร แบรนด์ ขอบเขตแพ็กเกจ แฮนเดิลเจ้าของ หรือความเป็นเจ้าของเนมสเปซควร
ใช้กระบวนการ [การอ้างสิทธิ์องค์กรและเนมสเปซ](/clawhub/namespace-claims) ไม่ใช่
ขั้นตอนรายงานในผลิตภัณฑ์หรือแบบฟอร์มอุทธรณ์บัญชี

ใช้กระบวนการนั้นเมื่อคุณต้องการให้เจ้าหน้าที่ ClawHub ตรวจสอบหลักฐานที่ไม่ละเอียดอ่อนว่า
เนมสเปซควรถูกสงวน โอน เปลี่ยนชื่อ ซ่อน กักกัน ทำเป็นนามแฝง
หรือตรวจสอบด้วยวิธีอื่น อย่าใส่ความลับ เอกสารส่วนตัว ไฟล์กฎหมายส่วนตัว
เอกสารระบุตัวตนส่วนบุคคล โทเค็น API หรือโทเค็น DNS challenge ใน
issue สาธารณะ

## การพักไว้เพื่อการกลั่นกรอง

ผลการตรวจพบที่รุนแรงหรือปัญหาด้านนโยบายบางอย่างอาจทำให้ผู้เผยแพร่หรือรายการถูก
พักไว้เพื่อการกลั่นกรอง เมื่อเกิดขึ้น เนื้อหาที่ได้รับผลกระทบอาจถูกซ่อนจากการค้นพบแบบสาธารณะ
หรือการเผยแพร่ในอนาคตอาจเริ่มต้นเป็นแบบซ่อนจนกว่าปัญหาจะได้รับการตรวจสอบ

การพักไว้เพื่อการกลั่นกรองมีไว้เพื่อปกป้องผู้ใช้ขณะที่ ClawHub แก้ไขกรณี
ความเสี่ยงสูง นอกจากนี้ยังสามารถยกเลิกได้เมื่อยืนยันว่าเป็นผลบวกผิดพลาด

## รายการที่ถูกซ่อนหรือถูกบล็อก

รายการอาจถูกพักไว้ ซ่อน กักกัน เพิกถอน หรือไม่พร้อมใช้งานด้วยวิธีอื่นบน
พื้นผิวการติดตั้งสาธารณะ

หากคุณเห็นสถานะใดสถานะหนึ่งเหล่านี้ อย่าติดตั้งรีลีส เว้นแต่เจ้าของ
จะแก้ไขปัญหาหรือการกลั่นกรองจะกู้คืนรายการนั้น

เจ้าของอาจยังเห็นข้อมูลวินิจฉัยสำหรับรายการของตนเองที่ถูกพักไว้หรือถูกซ่อน ข้อมูลวินิจฉัยเหล่านี้
ช่วยอธิบายว่าเกิดอะไรขึ้นและต้องเปลี่ยนแปลงอะไรบ้างก่อนที่
รายการจะกลับสู่พื้นผิวสาธารณะได้

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบายของ ClawHub อาจสูญเสียสิทธิ์ในการเผยแพร่ การละเมิดที่รุนแรงอาจ
ส่งผลให้บัญชีถูกแบน โทเค็นถูกเพิกถอน เนื้อหาถูกซ่อน หรือรายการถูกนำออก
สัญญาณแรงกดดันจากการละเมิดของผู้เผยแพร่จะถูกตรวจสอบทุกวัน สัญญาณที่ถึง
เกณฑ์การแบนที่อาจเกิดขึ้นของ ClawHub สามารถทริกเกอร์คำเตือนอัตโนมัติได้ หากการสแกนที่เข้าเกณฑ์ครั้งถัดไป
หลังจากกำหนดเวลาคำเตือนยังคงจัดให้ผู้เผยแพร่อยู่ใน
เกณฑ์การแบนที่อาจเกิดขึ้น ClawHub อาจใช้การดำเนินการกับบัญชีโดยอัตโนมัติ
สัญญาณการตรวจสอบที่มีความเชื่อมั่นต่ำกว่าและจำกัดช่วงเวลาจะไม่ถูกนำไปใช้ในการบังคับใช้
อัตโนมัติ

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้โทเค็น API ของ ClawHub ได้ หากการยืนยันตัวตนของ CLI
เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ UI บนเว็บเพื่อตรวจสอบสถานะ
บัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อกโดยการแบนหรือบัญชีที่ถูกปิดใช้งาน
ให้ใช้ [แบบฟอร์มอุทธรณ์ ClawHub](https://appeals.openclaw.ai/) เพื่อขอการตรวจสอบกู้คืน

หากอีเมลที่ทริกเกอร์โดยสแกนเนอร์ระบุว่าเวอร์ชันของ Skills หรือ Plugin เป็นอันตราย
ให้ดาวน์โหลดผลการสแกนที่จัดเก็บไว้สำหรับเวอร์ชันที่ส่งเข้ามาซึ่งถูกบล็อก:
`clawhub scan download <slug> --version <version>` สำหรับ Plugin ให้เพิ่ม
`--kind plugin` ตรวจสอบผลลัพธ์การสแกน แก้ไขรายการ เพิ่มหมายเลขเวอร์ชัน
และอัปโหลดเวอร์ชันที่แก้ไขแล้ว

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลดผลบวกผิดพลาดและเพิ่มความเชื่อถือของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และบันทึกการเปลี่ยนแปลงให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้อ่านยาก
- ลิงก์ไปยังซอร์สเมื่อเป็นไปได้
- ใช้การรันจำลองก่อนเผยแพร่ Plugin
- ตอบอย่างชัดเจนหากผู้ใช้หรือผู้กลั่นกรองถามเกี่ยวกับพฤติกรรมของรีลีส
