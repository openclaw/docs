# การตรวจสอบเนื้อหาและความปลอดภัยของบัญชี

ClawHub เปิดให้เผยแพร่ได้ แต่พื้นผิวสำหรับการค้นพบสาธารณะและการติดตั้งยังคง
ต้องมีมาตรการป้องกัน รายงาน การระงับเพื่อตรวจสอบเนื้อหา รายการที่ถูกซ่อน และการดำเนินการกับบัญชี
ช่วยปกป้องผู้ใช้เมื่อรีลีสหรือบัญชีดูไม่ปลอดภัย ชวนให้เข้าใจผิด หรือไม่เป็นไปตามนโยบาย

หน้านี้ครอบคลุมการตรวจสอบเนื้อหาและสถานะบัญชี สำหรับป้ายกำกับการตรวจสอบ เช่น
`Pass`, `Review`, `Warn`, `Malicious` และระดับความเสี่ยง โปรดดู
[การตรวจสอบความปลอดภัย](/clawhub/security-audits)

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage) สำหรับข้อกังวลเรื่องลิขสิทธิ์หรือสิทธิ์ในเนื้อหาอื่น ๆ
ให้ใช้ [คำขอด้านสิทธิ์ในเนื้อหา](/clawhub/content-rights)

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน Skills, Plugin และแพ็กเกจได้

ใช้รายงานของ ClawHub สำหรับเนื้อหาในมาร์เก็ตเพลซที่ไม่ปลอดภัยเท่านั้น เช่น:

- รายการที่เป็นอันตราย
- เมทาดาทาที่ชวนให้เข้าใจผิด
- ข้อมูลรับรองหรือข้อกำหนดสิทธิ์ที่ไม่ได้ประกาศ
- คำแนะนำการติดตั้งที่น่าสงสัย
- การแอบอ้าง
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

ใช้ปุ่ม **รายงาน Skills** บนหน้า Skills หรือคำสั่ง/API สำหรับรายงานแพ็กเกจ

อย่าใช้รายงานของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ Skills หรือ
Plugin บุคคลที่สาม ให้รายงานโดยตรงต่อผู้เผยแพร่หรือรีโพซิทอรีซอร์ส
ที่ลิงก์จากรายการ ClawHub ไม่ได้ดูแลหรือแพตช์
โค้ด Skills หรือ Plugin ของบุคคลที่สาม

GitHub Security Advisories สำหรับ `openclaw/clawhub` ใช้สำหรับช่องโหว่ใน
ClawHub เอง ตัวอย่างเช่นบั๊กในเว็บไซต์, API, CLI, รีจิสทรี, การยืนยันตัวตน,
การสแกน, การตรวจสอบเนื้อหา หรือขอบเขตความน่าเชื่อถือของการดาวน์โหลด/ติดตั้ง อย่าใช้ advisory ของ ClawHub
สำหรับช่องโหว่ใน Skills หรือ Plugin บุคคลที่สาม

รายงานที่ดีควรเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้ระบบรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

## การอ้างสิทธิ์องค์กรและเนมสเปซ

ข้อพิพาทเรื่องสิทธิ์ความเป็นเจ้าขององค์กร แบรนด์ ขอบเขตแพ็กเกจ แฮนเดิลเจ้าของ หรือเนมสเปซควร
ใช้กระบวนการ [การอ้างสิทธิ์องค์กรและเนมสเปซ](/clawhub/namespace-claims) ไม่ใช่
ขั้นตอนรายงานในผลิตภัณฑ์หรือแบบฟอร์มอุทธรณ์บัญชี

ใช้กระบวนการนั้นเมื่อคุณต้องการให้เจ้าหน้าที่ ClawHub ตรวจสอบหลักฐานที่ไม่อ่อนไหวว่า
เนมสเปซควรถูกสงวนไว้ โอน เปลี่ยนชื่อ ซ่อน กักกัน ตั้งนามแฝง
หรือตรวจสอบในลักษณะอื่น อย่ารวมความลับ เอกสารส่วนตัว ไฟล์กฎหมายส่วนตัว
เอกสารยืนยันตัวตนส่วนบุคคล โทเค็น API หรือโทเค็น DNS challenge ไว้ใน
issue สาธารณะ

## การระงับเพื่อตรวจสอบเนื้อหา

ข้อค้นพบที่ร้ายแรงหรือประเด็นด้านนโยบายบางอย่างอาจทำให้ผู้เผยแพร่หรือรายการถูก
ระงับเพื่อตรวจสอบเนื้อหา เมื่อเกิดเหตุการณ์นี้ เนื้อหาที่ได้รับผลกระทบอาจถูกซ่อนจาก
การค้นพบสาธารณะ หรือการเผยแพร่ในอนาคตอาจเริ่มต้นแบบถูกซ่อนไว้จนกว่าประเด็นจะได้รับการตรวจสอบ

การระงับเพื่อตรวจสอบเนื้อหามีไว้เพื่อปกป้องผู้ใช้ระหว่างที่ ClawHub แก้ไขกรณีที่มีความเสี่ยงสูง
และยังสามารถยกเลิกได้เมื่อยืนยันแล้วว่าเป็นผลบวกลวง

## รายการที่ถูกซ่อนหรือถูกบล็อก

รายการอาจถูกระงับ ซ่อน กักกัน เพิกถอน หรือไม่พร้อมใช้งานในลักษณะอื่นบน
พื้นผิวการติดตั้งสาธารณะ

หากคุณเห็นสถานะเหล่านี้ อย่าติดตั้งรีลีสนั้นเว้นแต่เจ้าของ
จะแก้ไขปัญหาหรือฝ่ายตรวจสอบเนื้อหากู้คืนรายการแล้ว

เจ้าของอาจยังเห็นข้อมูลวินิจฉัยสำหรับรายการของตนเองที่ถูกระงับหรือถูกซ่อน ข้อมูลวินิจฉัยเหล่านี้
ช่วยอธิบายว่าเกิดอะไรขึ้นและต้องเปลี่ยนแปลงอะไรบ้างก่อนที่
รายการจะกลับสู่พื้นผิวสาธารณะได้

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบายของ ClawHub อาจสูญเสียสิทธิ์ในการเผยแพร่ การละเมิดที่ร้ายแรงอาจ
ส่งผลให้ถูกแบนบัญชี เพิกถอนโทเค็น ซ่อนเนื้อหา หรือลบรายการ
สัญญาณแรงกดดันจากการละเมิดของผู้เผยแพร่จะถูกตรวจสอบทุกวัน สัญญาณที่ถึง
เกณฑ์การอาจถูกแบนของ ClawHub สามารถเรียกคำเตือนอัตโนมัติได้ หากการสแกนที่เข้าเงื่อนไขครั้งถัดไป
หลังเส้นตายคำเตือนยังคงจัดให้ผู้เผยแพร่อยู่ใน
เกณฑ์การอาจถูกแบน ClawHub อาจใช้การดำเนินการกับบัญชีโดยอัตโนมัติ
สัญญาณการตรวจสอบที่มีความเชื่อมั่นต่ำกว่าและจำกัดตามช่วงเวลาจะไม่อยู่ในการบังคับใช้
อัตโนมัติ

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้โทเค็น API ของ ClawHub ได้ หากการยืนยันตัวตนของ CLI
เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ Web UI เพื่อตรวจสอบ
สถานะบัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อกเพราะบัญชีถูกแบนหรือถูกปิดใช้งาน
ให้ใช้ [แบบฟอร์มอุทธรณ์ ClawHub](https://appeals.openclaw.ai/) เพื่อขอการตรวจสอบเพื่อกู้คืน

หากอีเมลที่ถูกเรียกจากสแกนเนอร์ระบุเวอร์ชันของ Skills หรือ Plugin ว่าเป็นอันตราย
ให้ดาวน์โหลดผลการสแกนที่จัดเก็บไว้สำหรับเวอร์ชันที่ส่งเข้ามาและถูกบล็อก:
`clawhub scan download <slug> --version <version>` สำหรับ Plugin ให้เพิ่ม
`--kind plugin` ตรวจสอบผลลัพธ์การสแกน แก้ไขรายการ เพิ่มหมายเลขเวอร์ชัน
และอัปโหลดเวอร์ชันที่แก้ไขแล้ว

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลดผลบวกลวงและเพิ่มความเชื่อมั่นของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และ changelog ให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้เข้าใจยาก
- ลิงก์ไปยังซอร์สเมื่อเป็นไปได้
- ใช้ dry run ก่อนเผยแพร่ Plugin
- ตอบให้ชัดเจนหากผู้ใช้หรือผู้ตรวจสอบเนื้อหาถามเกี่ยวกับพฤติกรรมของรีลีส
