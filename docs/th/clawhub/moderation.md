# การกลั่นกรองและความปลอดภัยของบัญชี

ClawHub เปิดให้เผยแพร่ได้ แต่พื้นผิวสำหรับการค้นพบสาธารณะและการติดตั้งยังคง
ต้องมีมาตรการกำกับดูแล รายงาน การพักไว้เพื่อตรวจสอบ รายการที่ถูกซ่อน และการดำเนินการกับบัญชี
ช่วยปกป้องผู้ใช้เมื่อรุ่นเผยแพร่หรือบัญชีดูไม่ปลอดภัย ทำให้เข้าใจผิด หรืออยู่นอก
นโยบาย

หน้านี้ครอบคลุมการกลั่นกรองและสถานะบัญชี สำหรับป้ายกำกับการตรวจสอบ เช่น
`Pass`, `Review`, `Warn`, `Malicious` และระดับความเสี่ยง โปรดดู
[การตรวจสอบความปลอดภัย](/th/clawhub/security-audits)

ดูเพิ่มเติมที่ [ความปลอดภัย](/th/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage) สำหรับข้อกังวลด้านลิขสิทธิ์หรือสิทธิ์เนื้อหาอื่น ๆ
ให้ใช้ [คำขอสิทธิ์เนื้อหา](/th/clawhub/content-rights)

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน Skills, Plugin และแพ็กเกจได้

ใช้รายงานของ ClawHub เฉพาะกับเนื้อหา marketplace ที่ไม่ปลอดภัย เช่น:

- รายการที่เป็นอันตราย
- metadata ที่ทำให้เข้าใจผิด
- credentials หรือข้อกำหนดสิทธิ์ที่ไม่ได้ประกาศ
- คำแนะนำการติดตั้งที่น่าสงสัย
- การแอบอ้างตัวตน
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

ใช้ปุ่ม **รายงาน Skill** บนหน้า Skill หรือใช้คำสั่ง/API สำหรับรายงานแพ็กเกจ
สำหรับแพ็กเกจ

อย่าใช้รายงานของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ Skill หรือ
Plugin บุคคลที่สาม ให้รายงานโดยตรงไปยังผู้เผยแพร่หรือ repository ต้นทาง
ที่ลิงก์จากรายการ ClawHub ไม่ดูแลหรือแพตช์โค้ด Skill หรือ Plugin
ของบุคคลที่สาม

GitHub Security Advisories สำหรับ `openclaw/clawhub` มีไว้สำหรับช่องโหว่ใน
ClawHub เอง ตัวอย่างรวมถึงบั๊กในเว็บไซต์, API, CLI, registry, auth,
การสแกน, การกลั่นกรอง หรือขอบเขตความน่าเชื่อถือของการดาวน์โหลด/ติดตั้ง อย่าใช้ advisory ของ ClawHub
สำหรับช่องโหว่ใน Skills หรือ Plugin ของบุคคลที่สาม

รายงานที่ดีต้องเจาะจงและนำไปดำเนินการได้ การใช้ระบบรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

## การอ้างสิทธิ์ org และ namespace

ข้อพิพาทเกี่ยวกับความเป็นเจ้าของ org, แบรนด์, ขอบเขตแพ็กเกจ, owner-handle หรือ namespace ควร
ใช้กระบวนการ [การอ้างสิทธิ์ org และ namespace](/th/clawhub/namespace-claims) ไม่ใช่
ขั้นตอนรายงานในผลิตภัณฑ์หรือแบบฟอร์มอุทธรณ์บัญชี

ใช้กระบวนการนั้นเมื่อคุณต้องการให้เจ้าหน้าที่ ClawHub ตรวจสอบหลักฐานที่ไม่ละเอียดอ่อนว่า
ควรสงวน โอน เปลี่ยนชื่อ ซ่อน กักกัน สร้าง alias หรือทบทวน namespace
ในรูปแบบอื่น อย่าใส่ความลับ เอกสารส่วนตัว ไฟล์กฎหมายส่วนตัว
เอกสารยืนยันตัวตนส่วนบุคคล, API tokens หรือ DNS challenge tokens ใน
issue สาธารณะ

## การพักไว้เพื่อตรวจสอบ

การค้นพบร้ายแรงหรือปัญหานโยบายบางอย่างอาจทำให้ผู้เผยแพร่หรือรายการถูก
พักไว้เพื่อตรวจสอบ เมื่อเกิดเหตุการณ์นี้ เนื้อหาที่ได้รับผลกระทบอาจถูกซ่อนจาก
การค้นพบสาธารณะ หรือการเผยแพร่ในอนาคตอาจเริ่มต้นแบบถูกซ่อนจนกว่าปัญหาจะได้รับการตรวจสอบ

การพักไว้เพื่อตรวจสอบมีไว้เพื่อปกป้องผู้ใช้ขณะที่ ClawHub แก้ไขกรณีที่มีความเสี่ยงสูง
และสามารถยกเลิกได้เมื่อยืนยันว่าเป็น false positive

## รายการที่ถูกซ่อนหรือถูกบล็อก

รายการอาจถูกพักไว้ ซ่อน กักกัน เพิกถอน หรือไม่พร้อมใช้งานในรูปแบบอื่นบน
พื้นผิวการติดตั้งสาธารณะ

หากคุณเห็นสถานะเหล่านี้ อย่าติดตั้งรุ่นเผยแพร่นั้น เว้นแต่เจ้าของ
จะแก้ไขปัญหาแล้ว หรือการกลั่นกรองคืนสถานะให้

เจ้าของอาจยังเห็นข้อมูลวินิจฉัยสำหรับรายการของตนเองที่ถูกพักไว้หรือถูกซ่อน ข้อมูลวินิจฉัยเหล่านี้
ช่วยอธิบายว่าเกิดอะไรขึ้นและต้องเปลี่ยนแปลงอะไร ก่อนที่
รายการจะกลับสู่พื้นผิวสาธารณะได้

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบายของ ClawHub อาจสูญเสียสิทธิ์การเผยแพร่ การละเมิดร้ายแรงอาจ
นำไปสู่การแบนบัญชี การเพิกถอน token เนื้อหาที่ถูกซ่อน หรือรายการที่ถูกลบ
สัญญาณแรงกดดันจากการละเมิดของผู้เผยแพร่จะถูกตรวจสอบทุกวัน สัญญาณที่ถึง
เกณฑ์ potential-ban ของ ClawHub อาจทำให้เกิดคำเตือนอัตโนมัติ หากการสแกนที่มีสิทธิ์ครั้งถัดไป
หลังเส้นตายคำเตือนยังคงจัดผู้เผยแพร่ไว้ใน
เกณฑ์ potential-ban ClawHub อาจใช้การดำเนินการกับบัญชีโดยอัตโนมัติ
สัญญาณการตรวจสอบที่มีความมั่นใจต่ำกว่าและมีขอบเขตตามช่วงเวลาจะไม่เข้าสู่การบังคับใช้
อัตโนมัติ

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้ ClawHub API tokens ได้ หาก CLI auth
เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ web UI เพื่อตรวจสอบ
สถานะบัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อกโดยการแบนหรือบัญชีที่ถูกปิดใช้งาน
ให้ใช้ [แบบฟอร์มอุทธรณ์ ClawHub](https://appeals.openclaw.ai/) สำหรับการตรวจสอบเพื่อกู้คืน

หากอีเมลที่เกิดจาก scanner ระบุว่าเวอร์ชันของ Skill หรือ Plugin เป็นอันตราย
ให้ดาวน์โหลดผลการสแกนที่จัดเก็บไว้สำหรับเวอร์ชันที่ส่งซึ่งถูกบล็อก:
`clawhub scan download <slug> --version <version>` สำหรับ Plugin ให้เพิ่ม
`--kind plugin` ตรวจสอบ output ของการสแกน แก้ไขรายการ เพิ่มหมายเลขเวอร์ชัน
และอัปโหลดเวอร์ชันที่แก้ไขแล้ว

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลด false positives และเพิ่มความไว้วางใจของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และ changelog ให้ถูกต้อง
- ประกาศ environment variables และสิทธิ์ที่จำเป็น
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้เข้าใจยาก
- ลิงก์ไปยังซอร์สเมื่อเป็นไปได้
- ใช้ dry runs ก่อนเผยแพร่ Plugin
- ตอบอย่างชัดเจนหากผู้ใช้หรือผู้กลั่นกรองถามเกี่ยวกับพฤติกรรมของรุ่นเผยแพร่
