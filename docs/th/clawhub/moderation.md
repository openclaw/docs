# การดูแลเนื้อหาและความปลอดภัยของบัญชี

ClawHub เปิดให้เผยแพร่ได้ แต่พื้นผิวสำหรับการค้นพบและการติดตั้งแบบสาธารณะยังคง
ต้องมีมาตรการควบคุม รายงาน การพักไว้เพื่อดูแลเนื้อหา รายการที่ถูกซ่อน และการดำเนินการกับบัญชี
ช่วยปกป้องผู้ใช้เมื่อรีลีสหรือบัญชีดูไม่ปลอดภัย ทำให้เข้าใจผิด หรืออยู่นอก
นโยบาย

หน้านี้ครอบคลุมการดูแลเนื้อหาและสถานะบัญชี สำหรับป้ายกำกับการตรวจสอบ เช่น
`Pass`, `Review`, `Warn`, `Malicious` และระดับความเสี่ยง โปรดดู
[การตรวจสอบความปลอดภัย](/clawhub/security-audits)

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage) สำหรับข้อกังวลด้านลิขสิทธิ์หรือสิทธิ์ในเนื้อหาอื่น ๆ
ให้ใช้ [คำขอสิทธิ์ในเนื้อหา](/clawhub/content-rights)

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน Skills, Plugin และแพ็กเกจได้

ใช้รายงานของ ClawHub เฉพาะกับเนื้อหา Marketplace ที่ไม่ปลอดภัย เช่น:

- รายการที่เป็นอันตราย
- เมทาดาทาที่ทำให้เข้าใจผิด
- ข้อมูลประจำตัวหรือข้อกำหนดสิทธิ์ที่ไม่ได้ประกาศ
- คำแนะนำการติดตั้งที่น่าสงสัย
- การแอบอ้างตัวตน
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

ใช้ปุ่ม **รายงานสกิล** บนหน้าสกิล หรือคำสั่ง/API สำหรับรายงานแพ็กเกจ

อย่าใช้รายงานของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของสกิลหรือ
Plugin บุคคลที่สาม ให้รายงานโดยตรงไปยังผู้เผยแพร่หรือคลังซอร์ส
ที่ลิงก์จากรายการนั้น ClawHub ไม่ได้ดูแลหรือแพตช์
โค้ดสกิลหรือ Plugin บุคคลที่สาม

GitHub Security Advisories สำหรับ `openclaw/clawhub` มีไว้สำหรับช่องโหว่ใน
ClawHub เอง ตัวอย่างรวมถึงบั๊กในเว็บไซต์, API, CLI, registry, auth,
การสแกน, การดูแลเนื้อหา หรือขอบเขตความเชื่อถือในการดาวน์โหลด/ติดตั้ง อย่าใช้คำแนะนำของ ClawHub
สำหรับช่องโหว่ใน Skills หรือ Plugin บุคคลที่สาม

รายงานที่ดีต้องเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้ระบบรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

## การอ้างสิทธิ์องค์กรและ namespace

ข้อพิพาทเกี่ยวกับองค์กร แบรนด์ ขอบเขตแพ็กเกจ owner-handle หรือความเป็นเจ้าของ namespace ควร
ใช้กระบวนการ [การอ้างสิทธิ์องค์กรและ Namespace](/clawhub/namespace-claims) ไม่ใช่
โฟลว์รายงานในผลิตภัณฑ์หรือแบบฟอร์มอุทธรณ์บัญชี

ใช้กระบวนการดังกล่าวเมื่อคุณต้องการให้เจ้าหน้าที่ ClawHub ตรวจสอบหลักฐานที่ไม่ละเอียดอ่อนว่า
namespace ควรถูกสงวนไว้ โอน เปลี่ยนชื่อ ซ่อน กักกัน ทำ alias
หรือตรวจสอบด้วยวิธีอื่น อย่าใส่ความลับ เอกสารส่วนตัว ไฟล์กฎหมายส่วนตัว
เอกสารระบุตัวตนส่วนบุคคล โทเค็น API หรือโทเค็น DNS challenge ใน
issue สาธารณะ

## การพักไว้เพื่อดูแลเนื้อหา

ผลการตรวจพบที่รุนแรงหรือปัญหาด้านนโยบายบางอย่างอาจทำให้ผู้เผยแพร่หรือรายการถูก
พักไว้เพื่อดูแลเนื้อหา เมื่อเกิดกรณีนี้ เนื้อหาที่ได้รับผลกระทบอาจถูกซ่อนจาก
การค้นพบแบบสาธารณะ หรือการเผยแพร่ในอนาคตอาจเริ่มต้นแบบซ่อนจนกว่าปัญหาจะได้รับการตรวจสอบ

การพักไว้เพื่อดูแลเนื้อหามีไว้เพื่อปกป้องผู้ใช้ระหว่างที่ ClawHub แก้ไขกรณีที่มีความเสี่ยงสูง
และยังสามารถยกเลิกได้เมื่อยืนยันว่าเป็นผลบวกปลอม

## รายการที่ถูกซ่อนหรือถูกบล็อก

รายการอาจถูกพักไว้ ซ่อน กักกัน เพิกถอน หรือไม่พร้อมใช้งานด้วยวิธีอื่นบน
พื้นผิวการติดตั้งสาธารณะ

หากคุณเห็นสถานะใดสถานะหนึ่งเหล่านี้ อย่าติดตั้งรีลีสนั้น เว้นแต่เจ้าของ
จะแก้ไขปัญหาหรือการดูแลเนื้อหาจะกู้คืนรายการนั้น

เจ้าของยังอาจเห็นการวินิจฉัยสำหรับรายการของตนเองที่ถูกพักไว้หรือซ่อนอยู่ได้
การวินิจฉัยเหล่านี้ช่วยอธิบายว่าเกิดอะไรขึ้นและต้องเปลี่ยนแปลงอะไรบ้างก่อนที่
รายการจะกลับไปยังพื้นผิวสาธารณะได้

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบายของ ClawHub อาจเสียสิทธิ์การเผยแพร่ การละเมิดร้ายแรงอาจ
ส่งผลให้บัญชีถูกแบน โทเค็นถูกเพิกถอน เนื้อหาถูกซ่อน หรือรายการถูกลบ
สัญญาณแรงกดดันจากการละเมิดของผู้เผยแพร่จะถูกตรวจสอบทุกวัน สัญญาณที่ถึง
เกณฑ์การแบนที่เป็นไปได้ของ ClawHub อาจทริกเกอร์คำเตือนอัตโนมัติ หากการสแกนที่มีสิทธิ์ครั้งถัดไป
หลังเส้นตายของคำเตือนยังคงจัดผู้เผยแพร่ไว้ในเกณฑ์การแบนที่เป็นไปได้
ClawHub อาจดำเนินการกับบัญชีโดยอัตโนมัติ
สัญญาณการตรวจสอบที่มีความมั่นใจต่ำกว่าและจำกัดตามช่วงเวลาจะไม่ถูกรวมอยู่ในการบังคับใช้
อัตโนมัติ

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้โทเค็น API ของ ClawHub ได้ หากการยืนยันตัวตน CLI
เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ UI เว็บเพื่อตรวจสอบสถานะ
บัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อกโดยการแบนหรือบัญชีที่ถูกปิดใช้งาน
ให้ใช้ [แบบฟอร์มอุทธรณ์ ClawHub](https://appeals.openclaw.ai/) เพื่อขอการตรวจสอบการกู้คืน

หากอีเมลที่ถูกทริกเกอร์โดยสแกนเนอร์ระบุเวอร์ชันสกิลหรือ Plugin ว่าเป็นอันตราย
ให้ดาวน์โหลดผลการสแกนที่จัดเก็บไว้สำหรับเวอร์ชันที่ส่งและถูกบล็อก:
`clawhub scan download <slug> --version <version>` สำหรับ Plugin ให้เพิ่ม
`--kind plugin` ตรวจสอบเอาต์พุตการสแกน แก้ไขรายการ เพิ่มหมายเลขเวอร์ชัน
และอัปโหลดเวอร์ชันที่แก้ไขแล้ว

## แนวทางสำหรับผู้เผยแพร่

เพื่อลดผลบวกปลอมและเพิ่มความเชื่อมั่นของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และบันทึกการเปลี่ยนแปลงให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้อ่านยาก
- ลิงก์ไปยังซอร์สเมื่อเป็นไปได้
- ใช้ dry run ก่อนเผยแพร่ Plugin
- ตอบกลับอย่างชัดเจนหากผู้ใช้หรือผู้ดูแลเนื้อหาถามเกี่ยวกับพฤติกรรมของรีลีส
