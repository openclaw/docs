# การดูแลเนื้อหาและความปลอดภัยของบัญชี

ClawHub เปิดให้เผยแพร่ได้ แต่พื้นผิวสำหรับการค้นพบสาธารณะและการติดตั้งยังคง
ต้องมีมาตรการควบคุม รายงาน การพักไว้เพื่อตรวจสอบการดูแลเนื้อหา รายการที่ซ่อน และการดำเนินการกับบัญชี
ช่วยปกป้องผู้ใช้เมื่อรีลีสหรือบัญชีดูไม่ปลอดภัย ทำให้เข้าใจผิด หรืออยู่นอก
นโยบาย

หน้านี้ครอบคลุมการดูแลเนื้อหาและสถานะบัญชี สำหรับป้ายกำกับการตรวจสอบ เช่น
`Pass`, `Review`, `Warn`, `Malicious` และระดับความเสี่ยง โปรดดู
[การตรวจสอบความปลอดภัย](/th/clawhub/security-audits)

ดูเพิ่มเติมที่ [ความปลอดภัย](/th/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage) สำหรับข้อกังวลด้านลิขสิทธิ์หรือสิทธิ์ในเนื้อหาอื่นๆ
ให้ใช้ [คำขอสิทธิ์ในเนื้อหา](/th/clawhub/content-rights)

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน Skills, Plugins และแพ็กเกจได้

ใช้รายงานของ ClawHub เฉพาะกับเนื้อหา marketplace ที่ไม่ปลอดภัย เช่น:

- รายการที่เป็นอันตราย
- เมทาดาทาที่ทำให้เข้าใจผิด
- ข้อมูลประจำตัวหรือข้อกำหนดสิทธิ์ที่ไม่ได้ประกาศ
- คำแนะนำการติดตั้งที่น่าสงสัย
- การแอบอ้าง
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

ใช้ปุ่ม **รายงาน Skill** บนหน้า Skill หรือคำสั่ง/API การรายงานแพ็กเกจ
สำหรับแพ็กเกจ

อย่าใช้รายงานของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ Skill หรือ
Plugin ภายนอก ให้รายงานช่องโหว่เหล่านั้นโดยตรงต่อผู้เผยแพร่หรือ
ที่เก็บซอร์สที่ลิงก์จากรายการ ClawHub ไม่ได้ดูแลหรือแพตช์
โค้ด Skill หรือ Plugin ภายนอก

GitHub Security Advisories สำหรับ `openclaw/clawhub` ใช้สำหรับช่องโหว่ใน
ClawHub เอง ตัวอย่างรวมถึงบั๊กในเว็บไซต์, API, CLI, รีจิสทรี, การยืนยันตัวตน,
การสแกน, การดูแลเนื้อหา หรือขอบเขตความเชื่อถือในการดาวน์โหลด/ติดตั้ง อย่าใช้ advisories ของ ClawHub
สำหรับช่องโหว่ใน Skills หรือ Plugins ภายนอก

รายงานที่ดีต้องเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้การรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

## การอ้างสิทธิ์องค์กรและ namespace

ข้อพิพาทเกี่ยวกับองค์กร แบรนด์ ขอบเขตแพ็กเกจ owner-handle หรือความเป็นเจ้าของ namespace ควร
ใช้กระบวนการ [การอ้างสิทธิ์องค์กรและ Namespace](/th/clawhub/namespace-claims) ไม่ใช่
โฟลว์รายงานในผลิตภัณฑ์หรือแบบฟอร์มอุทธรณ์บัญชี

ใช้กระบวนการนั้นเมื่อคุณต้องการให้ทีม ClawHub ตรวจสอบหลักฐานที่ไม่อ่อนไหวว่า
namespace ควรถูกสงวนไว้ โอน เปลี่ยนชื่อ ซ่อน กักกัน ทำ alias
หรือได้รับการตรวจสอบอย่างอื่น อย่าใส่ความลับ เอกสารส่วนตัว ไฟล์กฎหมายส่วนตัว
เอกสารยืนยันตัวตนส่วนบุคคล โทเค็น API หรือโทเค็น DNS challenge ใน
issue สาธารณะ

## การพักไว้เพื่อตรวจสอบการดูแลเนื้อหา

ข้อค้นพบที่ร้ายแรงหรือปัญหาด้านนโยบายบางอย่างอาจทำให้ผู้เผยแพร่หรือรายการถูก
พักไว้เพื่อตรวจสอบการดูแลเนื้อหา เมื่อเกิดกรณีนี้ เนื้อหาที่ได้รับผลกระทบอาจถูกซ่อนจาก
การค้นพบสาธารณะ หรือการเผยแพร่ในอนาคตอาจเริ่มต้นเป็นแบบซ่อนจนกว่าปัญหาจะได้รับการตรวจสอบ

การพักไว้เพื่อตรวจสอบการดูแลเนื้อหามีไว้เพื่อปกป้องผู้ใช้ในขณะที่ ClawHub จัดการกรณีที่มีความเสี่ยงสูง
และยังสามารถยกเลิกได้เมื่อยืนยันว่าเป็น false positive

## รายการที่ซ่อนหรือถูกบล็อก

รายการอาจถูกพักไว้ ซ่อน กักกัน เพิกถอน หรือไม่พร้อมใช้งานด้วยเหตุผลอื่นบน
พื้นผิวการติดตั้งสาธารณะ

หากคุณเห็นสถานะเหล่านี้ อย่าติดตั้งรีลีสนั้น เว้นแต่เจ้าของ
จะแก้ไขปัญหาหรือการดูแลเนื้อหาจะกู้คืนรายการนั้น

เจ้าของอาจยังเห็นข้อมูลวินิจฉัยสำหรับรายการของตนเองที่ถูกพักไว้หรือซ่อน ข้อมูลวินิจฉัยเหล่านี้
ช่วยอธิบายว่าเกิดอะไรขึ้นและต้องเปลี่ยนอะไร ก่อนที่
รายการจะกลับสู่พื้นผิวสาธารณะได้

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบายของ ClawHub อาจเสียสิทธิ์การเผยแพร่ การละเมิดร้ายแรงอาจ
ส่งผลให้บัญชีถูกแบน โทเค็นถูกเพิกถอน เนื้อหาถูกซ่อน หรือรายการถูกลบ
สัญญาณแรงกดดันจากการละเมิดของผู้เผยแพร่จะถูกตรวจสอบทุกวัน สัญญาณที่ถึง
เกณฑ์ potential-ban ของ ClawHub อาจทริกเกอร์คำเตือนอัตโนมัติ หากการสแกนที่มีสิทธิ์ครั้งถัดไป
หลังพ้นกำหนดคำเตือนยังคงจัดให้ผู้เผยแพร่อยู่ใน
เกณฑ์ potential-ban ClawHub อาจใช้การดำเนินการกับบัญชีโดยอัตโนมัติ
สัญญาณการตรวจสอบที่มีความเชื่อมั่นต่ำกว่าและมีกรอบเวลาจำกัดจะไม่อยู่ในการบังคับใช้
อัตโนมัติ

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้โทเค็น API ของ ClawHub ได้ หากการยืนยันตัวตนของ CLI
เริ่มล้มเหลวหลังการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้เว็บ UI เพื่อตรวจสอบสถานะบัญชี
หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ปกติถูกบล็อกโดยการแบนหรือบัญชีที่ถูกปิดใช้งาน
ให้ใช้ [แบบฟอร์มอุทธรณ์ ClawHub](https://appeals.openclaw.ai/) เพื่อขอการตรวจสอบเพื่อกู้คืน

หากอีเมลที่ทริกเกอร์โดยสแกนเนอร์ระบุว่าเวอร์ชันของ Skill หรือ Plugin เป็นอันตราย
ให้ดาวน์โหลดผลการสแกนที่จัดเก็บไว้สำหรับเวอร์ชันที่ส่งและถูกบล็อก:
`clawhub scan download <slug> --version <version>` สำหรับ Plugins ให้เพิ่ม
`--kind plugin` ตรวจทานผลลัพธ์การสแกน แก้ไขรายการ เพิ่มหมายเลขเวอร์ชัน
และอัปโหลดเวอร์ชันที่แก้ไขแล้ว

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลด false positive และเพิ่มความไว้วางใจของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และ changelogs ให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้คลุมเครือ
- ลิงก์ไปยังซอร์สเมื่อทำได้
- ใช้ dry runs ก่อนเผยแพร่ Plugins
- ตอบให้ชัดเจนหากผู้ใช้หรือผู้ดูแลเนื้อหาถามเกี่ยวกับพฤติกรรมของรีลีส
