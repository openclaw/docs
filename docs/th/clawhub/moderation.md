# การดูแลและความปลอดภัยของบัญชี

ClawHub เปิดให้เผยแพร่ได้ แต่พื้นผิวการค้นพบสาธารณะและการติดตั้งยังคง
ต้องมีมาตรการป้องกัน รายงาน การพักไว้เพื่อตรวจสอบ รายการที่ถูกซ่อน และการดำเนินการกับบัญชี
ช่วยปกป้องผู้ใช้เมื่อรีลีสหรือบัญชีดูไม่ปลอดภัย ทำให้เข้าใจผิด หรืออยู่นอก
นโยบาย

หน้านี้ครอบคลุมการดูแลและสถานะบัญชี สำหรับป้ายกำกับการตรวจสอบ เช่น
`Pass`, `Review`, `Warn`, `Malicious` และระดับความเสี่ยง โปรดดู
[การตรวจสอบความปลอดภัย](/clawhub/security-audits)

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage) สำหรับข้อกังวลเรื่องลิขสิทธิ์หรือสิทธิ์เนื้อหา
อื่น ๆ ให้ใช้ [คำขอด้านสิทธิ์เนื้อหา](/clawhub/content-rights)

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน Skills, Plugin และแพ็กเกจได้

ใช้รายงานของ ClawHub เฉพาะกับเนื้อหา marketplace ที่ไม่ปลอดภัย เช่น:

- รายการที่เป็นอันตราย
- เมตาดาตาที่ทำให้เข้าใจผิด
- ข้อมูลประจำตัวหรือข้อกำหนดสิทธิ์ที่ไม่ได้ประกาศ
- คำแนะนำการติดตั้งที่น่าสงสัย
- การแอบอ้าง
- การจดทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

ใช้ปุ่ม **รายงาน skill** บนหน้า skill หรือคำสั่ง/API สำหรับรายงานแพ็กเกจ

อย่าใช้รายงานของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ Skills หรือ
Plugin บุคคลที่สามเอง ให้รายงานโดยตรงไปยังผู้เผยแพร่หรือคลังซอร์ส
ที่ลิงก์จากรายการ ClawHub ไม่ได้ดูแลหรือแพตช์
โค้ด Skills หรือ Plugin บุคคลที่สาม

GitHub Security Advisories สำหรับ `openclaw/clawhub` ใช้สำหรับช่องโหว่ใน
ClawHub เอง ตัวอย่างรวมถึงบั๊กในเว็บไซต์, API, CLI, registry, auth,
การสแกน, การดูแล หรือขอบเขตความน่าเชื่อถือของการดาวน์โหลด/ติดตั้ง อย่าใช้ advisory ของ ClawHub
สำหรับช่องโหว่ใน Skills หรือ Plugin บุคคลที่สาม

รายงานที่ดีต้องเฉพาะเจาะจงและดำเนินการได้ การใช้การรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

## การอ้างสิทธิ์ org และ namespace

ข้อพิพาทเรื่องความเป็นเจ้าของ org, แบรนด์, package-scope, owner-handle หรือ namespace ควร
ใช้กระบวนการ [การอ้างสิทธิ์ Org และ Namespace](/clawhub/namespace-claims) ไม่ใช่
โฟลว์รายงานในผลิตภัณฑ์หรือแบบฟอร์มอุทธรณ์บัญชี

ใช้กระบวนการนั้นเมื่อคุณต้องการให้เจ้าหน้าที่ ClawHub ตรวจสอบหลักฐานที่ไม่อ่อนไหวว่า
namespace ควรถูกสงวน โอน เปลี่ยนชื่อ ซ่อน กักกัน สร้าง alias
หรือตรวจสอบด้วยวิธีอื่น อย่าใส่ secrets, เอกสารส่วนตัว, ไฟล์กฎหมายส่วนตัว,
เอกสารยืนยันตัวตนส่วนบุคคล, API tokens หรือ DNS challenge tokens ใน
issue สาธารณะ

## การพักไว้เพื่อตรวจสอบ

ผลการตรวจพบที่รุนแรงหรือปัญหานโยบายบางอย่างสามารถทำให้ผู้เผยแพร่หรือรายการถูก
พักไว้เพื่อตรวจสอบได้ เมื่อเกิดกรณีนี้ เนื้อหาที่ได้รับผลกระทบอาจถูกซ่อนจาก
การค้นพบสาธารณะ หรือการเผยแพร่ในอนาคตอาจเริ่มต้นแบบซ่อนจนกว่าปัญหาจะได้รับการตรวจสอบ

การพักไว้เพื่อตรวจสอบมีไว้เพื่อปกป้องผู้ใช้ขณะที่ ClawHub แก้ไขกรณีที่มีความเสี่ยงสูง
และสามารถยกเลิกได้เมื่อยืนยันว่าเป็นผลบวกลวง

## รายการที่ถูกซ่อนหรือถูกบล็อก

รายการอาจถูกพักไว้ ซ่อน กักกัน เพิกถอน หรือไม่พร้อมใช้งานด้วยวิธีอื่นบน
พื้นผิวการติดตั้งสาธารณะ

หากคุณเห็นสถานะใดสถานะหนึ่งเหล่านี้ อย่าติดตั้งรีลีสนั้น เว้นแต่เจ้าของ
จะแก้ไขปัญหาหรือการดูแลจะกู้คืนให้

เจ้าของอาจยังคงเห็นการวินิจฉัยสำหรับรายการของตนเองที่ถูกพักไว้หรือซ่อนอยู่ การวินิจฉัยเหล่านี้
ช่วยอธิบายว่าเกิดอะไรขึ้นและต้องเปลี่ยนอะไรบ้างก่อนที่
รายการจะกลับสู่พื้นผิวสาธารณะได้

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบายของ ClawHub อาจสูญเสียสิทธิ์ในการเผยแพร่ การใช้งานในทางที่ผิดอย่างรุนแรงอาจ
ส่งผลให้บัญชีถูกแบน, โทเค็นถูกเพิกถอน, เนื้อหาถูกซ่อน หรือรายการถูกลบ
สัญญาณแรงกดดันจากการใช้งานผู้เผยแพร่ในทางที่ผิดจะถูกตรวจสอบทุกวัน สัญญาณที่ถึง
เกณฑ์ potential-ban ของ ClawHub อาจทริกเกอร์คำเตือนอัตโนมัติ หากการสแกนที่เข้าเกณฑ์ครั้งถัดไป
หลังเส้นตายคำเตือนยังคงจัดผู้เผยแพร่ให้อยู่ใน
เกณฑ์ potential-ban ClawHub อาจใช้การดำเนินการกับบัญชีโดยอัตโนมัติ
สัญญาณการตรวจสอบที่มีความมั่นใจต่ำกว่าและถูกจำกัดตามช่วงเวลาจะไม่อยู่ในการบังคับใช้
อัตโนมัติ

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้ API tokens ของ ClawHub ได้ หาก CLI auth
เริ่มล้มเหลวหลังการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ UI บนเว็บเพื่อตรวจสอบสถานะบัญชี
หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อกโดยการแบนหรือบัญชีที่ถูกปิดใช้งาน
ให้ใช้ [แบบฟอร์มอุทธรณ์ ClawHub](https://appeals.openclaw.ai/) สำหรับการตรวจสอบการกู้คืน

หากอีเมลที่ทริกเกอร์โดยสแกนเนอร์ตั้งชื่อเวอร์ชันของ Skills หรือ Plugin ว่าเป็นอันตราย
ให้ดาวน์โหลดผลการสแกนที่จัดเก็บไว้สำหรับเวอร์ชันที่ส่งและถูกบล็อก:
`clawhub scan download <slug> --version <version>` สำหรับ Plugin ให้เพิ่ม
`--kind plugin` ตรวจสอบเอาต์พุตการสแกน แก้ไขรายการ เพิ่มหมายเลขเวอร์ชัน
และอัปโหลดเวอร์ชันที่แก้ไขแล้ว

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลดผลบวกลวงและเพิ่มความน่าเชื่อถือของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และ changelog ให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้คลุมเครือ
- ลิงก์ไปยังซอร์สเมื่อเป็นไปได้
- ใช้ dry runs ก่อนเผยแพร่ Plugin
- ตอบให้ชัดเจนหากผู้ใช้หรือผู้ดูแลถามเกี่ยวกับพฤติกรรมของรีลีส
