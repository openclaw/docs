# การกลั่นกรองและความปลอดภัยของบัญชี

ClawHub เปิดให้เผยแพร่ได้ แต่พื้นผิวสำหรับการค้นพบสาธารณะและการติดตั้งยังคง
ต้องมีมาตรการกำกับดูแล รายงาน การระงับเพื่อตรวจสอบการกลั่นกรอง รายการที่ถูกซ่อน และการดำเนินการกับบัญชี
ช่วยปกป้องผู้ใช้เมื่อรีลีสหรือบัญชีดูไม่ปลอดภัย ชวนให้เข้าใจผิด หรืออยู่นอก
นโยบาย

หน้านี้ครอบคลุมการกลั่นกรองและสถานะบัญชี สำหรับป้ายกำกับการตรวจสอบ เช่น
`Pass`, `Review`, `Warn`, `Malicious` และระดับความเสี่ยง โปรดดู
[การตรวจสอบความปลอดภัย](/clawhub/security-audits)

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage) สำหรับข้อกังวลด้านลิขสิทธิ์หรือสิทธิ์ในเนื้อหาอื่น ๆ
ให้ใช้ [คำขอสิทธิ์ในเนื้อหา](/clawhub/content-rights)

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน Skills, Plugin และแพ็กเกจได้

ใช้รายงานของ ClawHub เฉพาะกับเนื้อหาในมาร์เก็ตเพลสที่ไม่ปลอดภัย เช่น:

- รายการที่เป็นอันตราย
- เมทาดาทาที่ชวนให้เข้าใจผิด
- ข้อมูลรับรองหรือข้อกำหนดสิทธิ์ที่ไม่ได้ประกาศ
- คำแนะนำการติดตั้งที่น่าสงสัย
- การแอบอ้างตัวตน
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

ใช้ปุ่ม **รายงาน Skill** บนหน้า Skill หรือคำสั่ง/API สำหรับรายงานแพ็กเกจ

อย่าใช้รายงานของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ Skill หรือ
Plugin ของบุคคลที่สามเอง ให้รายงานโดยตรงไปยังผู้เผยแพร่หรือที่เก็บซอร์ส
ที่ลิงก์จากรายการ ClawHub ไม่ได้ดูแลหรือแพตช์โค้ด Skill หรือ Plugin
ของบุคคลที่สาม

GitHub Security Advisories สำหรับ `openclaw/clawhub` ใช้สำหรับช่องโหว่ใน
ClawHub เอง ตัวอย่างรวมถึงบั๊กในเว็บไซต์, API, CLI, รีจิสทรี, auth,
การสแกน, การกลั่นกรอง หรือขอบเขตความเชื่อถือของการดาวน์โหลด/ติดตั้ง อย่าใช้ advisory ของ ClawHub
สำหรับช่องโหว่ใน Skills หรือ Plugin ของบุคคลที่สาม

รายงานที่ดีต้องเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้ระบบรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

## การอ้างสิทธิ์องค์กรและ namespace

ข้อพิพาทเกี่ยวกับความเป็นเจ้าขององค์กร แบรนด์ ขอบเขตแพ็กเกจ owner-handle หรือ namespace ควร
ใช้กระบวนการ [การอ้างสิทธิ์องค์กรและ Namespace](/clawhub/namespace-claims) ไม่ใช่
โฟลว์รายงานในผลิตภัณฑ์หรือแบบฟอร์มอุทธรณ์บัญชี

ใช้กระบวนการนั้นเมื่อคุณต้องการให้เจ้าหน้าที่ ClawHub ตรวจสอบหลักฐานที่ไม่ละเอียดอ่อนว่า
namespace ควรถูกสงวน โอน เปลี่ยนชื่อ ซ่อน กักกัน ทำเป็นนามแฝง
หรือได้รับการตรวจสอบด้วยวิธีอื่น อย่าใส่ข้อมูลลับ เอกสารส่วนตัว ไฟล์กฎหมายส่วนตัว
เอกสารยืนยันตัวตนส่วนบุคคล โทเค็น API หรือโทเค็น DNS challenge ใน
issue สาธารณะ

## การระงับเพื่อตรวจสอบการกลั่นกรอง

ผลการตรวจพบร้ายแรงหรือปัญหาด้านนโยบายบางอย่างอาจทำให้ผู้เผยแพร่หรือรายการอยู่ภายใต้
การระงับเพื่อตรวจสอบการกลั่นกรอง เมื่อเกิดเหตุการณ์นี้ เนื้อหาที่ได้รับผลกระทบอาจถูกซ่อนจากการค้นพบ
สาธารณะ หรือการเผยแพร่ในอนาคตอาจเริ่มต้นในสถานะซ่อนไว้จนกว่าปัญหาจะได้รับการตรวจสอบ

การระงับเพื่อตรวจสอบการกลั่นกรองมีไว้เพื่อปกป้องผู้ใช้ระหว่างที่ ClawHub แก้ไขกรณี
ความเสี่ยงสูง นอกจากนี้ยังสามารถยกเลิกได้เมื่อยืนยันว่าเป็นผลบวกลวง

## รายการที่ถูกซ่อนหรือถูกบล็อก

รายการอาจถูกระงับ ซ่อน กักกัน เพิกถอน หรือไม่พร้อมใช้งานด้วยวิธีอื่นบน
พื้นผิวการติดตั้งสาธารณะ

หากคุณเห็นสถานะเหล่านี้ อย่าติดตั้งรีลีสนั้น เว้นแต่เจ้าของ
จะแก้ไขปัญหาแล้ว หรือการกลั่นกรองคืนสถานะให้แล้ว

เจ้าของยังอาจเห็นการวินิจฉัยสำหรับรายการของตนเองที่ถูกระงับหรือซ่อนอยู่ได้ การวินิจฉัยเหล่านี้
ช่วยอธิบายว่าเกิดอะไรขึ้นและต้องเปลี่ยนแปลงอะไรบ้างก่อนที่
รายการจะกลับสู่พื้นผิวสาธารณะได้

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบายของ ClawHub อาจสูญเสียสิทธิ์การเผยแพร่ การละเมิดร้ายแรงอาจ
ส่งผลให้บัญชีถูกแบน โทเค็นถูกเพิกถอน เนื้อหาถูกซ่อน หรือรายการถูกลบ
สัญญาณแรงกดดันจากการละเมิดของผู้เผยแพร่จะถูกตรวจสอบทุกวัน สัญญาณที่ถึง
เกณฑ์อาจถูกแบนของ ClawHub สามารถทริกเกอร์คำเตือนอัตโนมัติได้ หากการสแกนที่มีสิทธิ์ครั้งถัดไป
หลังจากกำหนดเวลาคำเตือนยังคงจัดให้ผู้เผยแพร่อยู่ใน
เกณฑ์อาจถูกแบน ClawHub อาจใช้การดำเนินการกับบัญชีโดยอัตโนมัติ
สัญญาณการตรวจสอบตามช่วงเวลาที่มีความมั่นใจต่ำกว่าและมีขอบเขตจำกัดจะไม่รวมอยู่ในการ
บังคับใช้อัตโนมัติ

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้โทเค็น API ของ ClawHub ได้ หาก auth ของ CLI
เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ web UI เพื่อตรวจสอบสถานะ
บัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อกโดยการแบนหรือบัญชีที่ถูกปิดใช้งาน
ให้ใช้ [แบบฟอร์มอุทธรณ์ของ ClawHub](https://appeals.openclaw.ai/) เพื่อขอการตรวจสอบการกู้คืน

หากอีเมลที่ทริกเกอร์โดยสแกนเนอร์ระบุว่าเวอร์ชันของ Skill หรือ Plugin เป็นอันตราย
ให้ดาวน์โหลดผลการสแกนที่จัดเก็บไว้สำหรับเวอร์ชันที่ส่งซึ่งถูกบล็อก:
`clawhub scan download <slug> --version <version>` สำหรับ Plugin ให้เพิ่ม
`--kind plugin` ตรวจสอบเอาต์พุตการสแกน แก้ไขรายการ เพิ่มหมายเลขเวอร์ชัน
และอัปโหลดเวอร์ชันที่แก้ไขแล้ว

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลดผลบวกลวงและเพิ่มความไว้วางใจของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และ changelog ให้ถูกต้อง
- ประกาศ environment variables และสิทธิ์ที่จำเป็น
- หลีกเลี่ยงคำสั่งติดตั้งที่ถูกทำให้อ่านยาก
- ลิงก์ไปยังซอร์สเมื่อทำได้
- ใช้ dry runs ก่อนเผยแพร่ Plugin
- ตอบกลับอย่างชัดเจนหากผู้ใช้หรือผู้กลั่นกรองถามเกี่ยวกับพฤติกรรมของรีลีส
