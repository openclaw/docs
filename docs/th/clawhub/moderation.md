# การกลั่นกรองและความปลอดภัยของบัญชี

ClawHub เปิดให้เผยแพร่ได้ แต่พื้นที่สำหรับการค้นพบแบบสาธารณะและการติดตั้งยังคง
ต้องมีมาตรการป้องกัน รายงาน การระงับเพื่อตรวจสอบ รายการที่ถูกซ่อน และการดำเนินการกับบัญชี
ช่วยปกป้องผู้ใช้เมื่อรุ่นเผยแพร่หรือบัญชีดูเหมือนไม่ปลอดภัย ทำให้เข้าใจผิด หรือไม่เป็นไปตาม
นโยบาย

หน้านี้ครอบคลุมการกลั่นกรองและสถานะบัญชี สำหรับป้ายกำกับการตรวจสอบ เช่น
`Pass`, `Review`, `Warn`, `Malicious` และระดับความเสี่ยง โปรดดู
[การตรวจสอบความปลอดภัย](/clawhub/security-audits)

ดูเพิ่มเติมที่ [ความปลอดภัย](/th/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage) สำหรับข้อกังวลเรื่องลิขสิทธิ์หรือสิทธิ์ใน
เนื้อหาอื่น ๆ ให้ใช้ [คำขอเกี่ยวกับสิทธิ์ในเนื้อหา](/clawhub/content-rights)

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน Skills, Plugin และแพ็กเกจได้

ใช้รายงานของ ClawHub เฉพาะกับเนื้อหาในมาร์เก็ตเพลสที่ไม่ปลอดภัย เช่น:

- รายการที่เป็นอันตราย
- ข้อมูลเมตาที่ทำให้เข้าใจผิด
- ข้อมูลรับรองหรือข้อกำหนดด้านสิทธิ์ที่ไม่ได้ประกาศ
- คำแนะนำการติดตั้งที่น่าสงสัย
- การแอบอ้างบุคคลอื่น
- การจดทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

ใช้ปุ่ม **Report skill** บนหน้าของ Skill หรือใช้คำสั่ง/API สำหรับรายงาน
แพ็กเกจ

อย่าใช้รายงานของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ Skill หรือ
Plugin จากบุคคลที่สาม ให้รายงานโดยตรงต่อผู้เผยแพร่หรือคลังซอร์ส
ที่เชื่อมโยงจากรายการ ClawHub ไม่ได้ดูแลหรือแก้ไข
โค้ด Skill หรือ Plugin ของบุคคลที่สาม

GitHub Security Advisories สำหรับ `openclaw/clawhub` มีไว้สำหรับช่องโหว่ใน
ClawHub เอง ตัวอย่างรวมถึงข้อบกพร่องในเว็บไซต์, API, CLI, รีจิสทรี, การยืนยันตัวตน,
การสแกน, การกลั่นกรอง หรือขอบเขตความน่าเชื่อถือของการดาวน์โหลด/ติดตั้ง อย่าใช้คำแนะนำด้านความปลอดภัยของ ClawHub
สำหรับช่องโหว่ใน Skills หรือ Plugin ของบุคคลที่สาม

รายงานที่ดีต้องเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้ระบบรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

## การอ้างสิทธิ์ในองค์กรและเนมสเปซ

ข้อพิพาทเกี่ยวกับความเป็นเจ้าขององค์กร แบรนด์ ขอบเขตแพ็กเกจ ชื่อผู้ใช้ของเจ้าของ หรือเนมสเปซ ควร
ใช้กระบวนการ [การอ้างสิทธิ์ในองค์กรและเนมสเปซ](/clawhub/namespace-claims) ไม่ใช่
ขั้นตอนรายงานภายในผลิตภัณฑ์หรือแบบฟอร์มอุทธรณ์บัญชี

ใช้กระบวนการดังกล่าวเมื่อคุณต้องการให้เจ้าหน้าที่ ClawHub ตรวจสอบหลักฐานที่ไม่ละเอียดอ่อนว่า
ควรสงวน โอน เปลี่ยนชื่อ ซ่อน กักกัน สร้างนามแฝง หรือดำเนินการตรวจสอบเนมสเปซ
ในรูปแบบอื่น อย่าใส่ข้อมูลลับ เอกสารส่วนตัว ไฟล์ทางกฎหมายส่วนตัว
เอกสารระบุตัวตนส่วนบุคคล โทเค็น API หรือโทเค็นการยืนยัน DNS ใน
ปัญหาสาธารณะ

## การระงับเพื่อตรวจสอบ

ผลการตรวจสอบที่ร้ายแรงหรือปัญหาด้านนโยบายบางอย่างอาจทำให้ผู้เผยแพร่หรือรายการถูก
ระงับเพื่อตรวจสอบ เมื่อเกิดกรณีนี้ เนื้อหาที่ได้รับผลกระทบอาจถูกซ่อนจากการ
ค้นพบแบบสาธารณะ หรือการเผยแพร่ในอนาคตอาจเริ่มต้นในสถานะซ่อนจนกว่าจะมีการตรวจสอบปัญหา

การระงับเพื่อตรวจสอบมีไว้เพื่อปกป้องผู้ใช้ระหว่างที่ ClawHub แก้ไขกรณีที่มีความเสี่ยงสูง
และสามารถยกเลิกได้เมื่อยืนยันว่าเป็นผลบวกลวง

## รายการที่ถูกซ่อนหรือบล็อก

รายการอาจถูกระงับ ซ่อน กักกัน เพิกถอน หรือไม่พร้อมใช้งานในรูปแบบอื่นบน
พื้นที่ติดตั้งสาธารณะ

หากคุณเห็นสถานะใดสถานะหนึ่งเหล่านี้ อย่าติดตั้งรุ่นเผยแพร่ เว้นแต่เจ้าของจะ
แก้ไขปัญหาแล้ว หรือฝ่ายกลั่นกรองคืนสถานะให้

เจ้าของอาจยังคงเห็นข้อมูลวินิจฉัยสำหรับรายการของตนเองที่ถูกระงับหรือซ่อน ข้อมูล
วินิจฉัยเหล่านี้ช่วยอธิบายว่าเกิดอะไรขึ้นและต้องเปลี่ยนแปลงสิ่งใดก่อนที่
รายการจะกลับสู่พื้นที่สาธารณะได้

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบายของ ClawHub อาจสูญเสียสิทธิ์ในการเผยแพร่ การใช้งานในทางที่ผิดอย่างร้ายแรงอาจ
ส่งผลให้บัญชีถูกแบน โทเค็นถูกเพิกถอน เนื้อหาถูกซ่อน หรือรายการถูกนำออก
สัญญาณแรงกดดันจากการใช้งานในทางที่ผิดของผู้เผยแพร่จะได้รับการตรวจสอบทุกวัน สัญญาณที่ถึง
เกณฑ์อาจถูกแบนของ ClawHub สามารถทริกเกอร์คำเตือนอัตโนมัติได้ หากการสแกนครั้งถัดไป
ที่มีสิทธิ์หลังเส้นตายของคำเตือนยังคงจัดให้ผู้เผยแพร่อยู่ใน
เกณฑ์อาจถูกแบน ClawHub อาจดำเนินการกับบัญชีโดยอัตโนมัติ
สัญญาณการตรวจสอบที่มีความเชื่อมั่นต่ำกว่าและจำกัดอยู่ในช่วงเวลาจะไม่รวมอยู่ในการ
บังคับใช้อัตโนมัติ

บัญชีที่ถูกลบ ถูกแบน หรือปิดใช้งานไม่สามารถใช้โทเค็น API ของ ClawHub ได้ หากการยืนยันตัวตนของ CLI
เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ UI บนเว็บเพื่อตรวจสอบ
สถานะบัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อกเนื่องจากการแบนหรือบัญชีถูกปิดใช้งาน
ให้ใช้ [แบบฟอร์มอุทธรณ์ ClawHub](https://appeals.openclaw.ai/) เพื่อขอรับการตรวจสอบเพื่อกู้คืน

หากอีเมลที่ทริกเกอร์โดยเครื่องสแกนระบุว่าเวอร์ชันของ Skill หรือ Plugin เป็นอันตราย
ให้ดาวน์โหลดผลการสแกนที่จัดเก็บไว้สำหรับเวอร์ชันที่ส่งและถูกบล็อก:
`clawhub scan download <slug> --version <version>` สำหรับ Plugin ให้เพิ่ม
`--kind plugin` ตรวจสอบผลลัพธ์การสแกน แก้ไขรายการ เพิ่มหมายเลขเวอร์ชัน
และอัปโหลดเวอร์ชันที่แก้ไขแล้ว

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลดผลบวกลวงและเพิ่มความไว้วางใจของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และบันทึกการเปลี่ยนแปลงให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้อ่านหรือทำความเข้าใจได้ยาก
- เชื่อมโยงไปยังซอร์สเมื่อทำได้
- ใช้การทดลองรันก่อนเผยแพร่ Plugin
- ตอบอย่างชัดเจนหากผู้ใช้หรือผู้กลั่นกรองสอบถามเกี่ยวกับพฤติกรรมของรุ่นเผยแพร่
