# การดูแลเนื้อหาและความปลอดภัยของบัญชี

ClawHub เปิดให้เผยแพร่ได้ แต่พื้นผิวสำหรับการค้นพบสาธารณะและการติดตั้งยังคง
ต้องมีราวกั้น รายงาน การพักไว้เพื่อดูแลเนื้อหา รายการที่ถูกซ่อน และการดำเนินการกับบัญชี
ช่วยปกป้องผู้ใช้เมื่อรีลีสหรือบัญชีดูไม่ปลอดภัย ชวนเข้าใจผิด หรืออยู่นอก
นโยบาย

หน้านี้ครอบคลุมการดูแลเนื้อหาและสถานะบัญชี สำหรับป้ายกำกับการตรวจสอบ เช่น
`Pass`, `Review`, `Warn`, `Malicious` และระดับความเสี่ยง โปรดดู
[การตรวจสอบความปลอดภัย](/clawhub/security-audits)

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage) สำหรับข้อกังวลด้านลิขสิทธิ์หรือสิทธิ์ในเนื้อหาอื่น ๆ
ให้ใช้ [คำขอสิทธิ์ในเนื้อหา](/clawhub/content-rights)

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงานทักษะ Plugin และแพ็กเกจได้

ใช้รายงานของ ClawHub เฉพาะกับเนื้อหา marketplace ที่ไม่ปลอดภัย เช่น:

- รายการที่เป็นอันตราย
- เมทาดาทาที่ชวนเข้าใจผิด
- ข้อมูลประจำตัวหรือข้อกำหนดสิทธิ์ที่ไม่ได้ประกาศ
- คำแนะนำการติดตั้งที่น่าสงสัย
- การแอบอ้างบุคคลอื่น
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

ใช้ปุ่ม **รายงานทักษะ** บนหน้าทักษะ หรือคำสั่ง/API สำหรับรายงานแพ็กเกจ
สำหรับแพ็กเกจ

อย่าใช้รายงานของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของทักษะหรือ
Plugin ของบุคคลที่สามเอง ให้รายงานโดยตรงต่อผู้เผยแพร่หรือที่เก็บซอร์ส
ที่ลิงก์จากรายการ ClawHub ไม่ได้ดูแลหรือแพตช์
โค้ดทักษะหรือ Plugin ของบุคคลที่สาม

GitHub Security Advisories สำหรับ `openclaw/clawhub` มีไว้สำหรับช่องโหว่ใน
ClawHub เอง ตัวอย่างรวมถึงบั๊กในเว็บไซต์, API, CLI, รีจิสทรี, auth,
การสแกน, การดูแลเนื้อหา หรือขอบเขตความเชื่อถือของการดาวน์โหลด/ติดตั้ง อย่าใช้คำแนะนำของ ClawHub
สำหรับช่องโหว่ในทักษะหรือ Plugin ของบุคคลที่สาม

รายงานที่ดีต้องเฉพาะเจาะจงและดำเนินการได้ การใช้การรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

## คำร้องสิทธิ์ org และ namespace

ข้อพิพาทเกี่ยวกับความเป็นเจ้าของ org, แบรนด์, package-scope, owner-handle หรือ namespace ควร
ใช้กระบวนการ [คำร้องสิทธิ์ org และ namespace](/clawhub/namespace-claims) ไม่ใช่
โฟลว์รายงานในผลิตภัณฑ์หรือแบบฟอร์มอุทธรณ์บัญชี

ใช้กระบวนการนั้นเมื่อคุณต้องการให้เจ้าหน้าที่ ClawHub ตรวจสอบหลักฐานที่ไม่อ่อนไหวว่า
namespace ควรถูกสงวน โอน เปลี่ยนชื่อ ซ่อน กักกัน กำหนดนามแฝง
หรือได้รับการตรวจสอบในลักษณะอื่น อย่าใส่ความลับ เอกสารส่วนตัว ไฟล์กฎหมายส่วนตัว
เอกสารยืนยันตัวตนส่วนบุคคล โทเค็น API หรือโทเค็น DNS challenge ใน
issue สาธารณะ

## การพักไว้เพื่อดูแลเนื้อหา

ข้อค้นพบบางอย่างที่รุนแรงหรือปัญหาด้านนโยบายอาจทำให้ผู้เผยแพร่หรือรายการถูก
พักไว้เพื่อดูแลเนื้อหา เมื่อเกิดกรณีนี้ เนื้อหาที่ได้รับผลกระทบอาจถูกซ่อนจากการค้นพบสาธารณะ
หรือการเผยแพร่ในอนาคตอาจเริ่มต้นแบบถูกซ่อนไว้จนกว่าปัญหาจะได้รับการตรวจสอบ

การพักไว้เพื่อดูแลเนื้อหามีไว้เพื่อปกป้องผู้ใช้ระหว่างที่ ClawHub แก้ไขกรณี
ความเสี่ยงสูง นอกจากนี้ยังสามารถยกเลิกได้เมื่อยืนยันแล้วว่าเป็นผลบวกเทียม

## รายการที่ถูกซ่อนหรือถูกบล็อก

รายการอาจถูกพักไว้ ซ่อน กักกัน เพิกถอน หรือไม่พร้อมใช้งานในลักษณะอื่นบน
พื้นผิวการติดตั้งสาธารณะ

หากคุณเห็นสถานะเหล่านี้ อย่าติดตั้งรีลีส เว้นแต่เจ้าของ
จะแก้ไขปัญหาหรือการดูแลเนื้อหากู้คืนรายการนั้น

เจ้าของอาจยังเห็นการวินิจฉัยสำหรับรายการของตนเองที่ถูกพักไว้หรือถูกซ่อน
การวินิจฉัยเหล่านี้ช่วยอธิบายว่าเกิดอะไรขึ้นและต้องเปลี่ยนแปลงอะไรบ้างก่อนที่
รายการจะกลับสู่พื้นผิวสาธารณะได้

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบายของ ClawHub อาจสูญเสียสิทธิ์การเผยแพร่ การละเมิดอย่างรุนแรงอาจ
ส่งผลให้บัญชีถูกแบน โทเค็นถูกเพิกถอน เนื้อหาถูกซ่อน หรือรายการถูกลบ
สัญญาณแรงกดดันจากการละเมิดของผู้เผยแพร่จะถูกตรวจสอบทุกวัน สัญญาณที่ถึง
เกณฑ์อาจถูกแบนของ ClawHub สามารถทริกเกอร์คำเตือนอัตโนมัติได้ หากการสแกนที่มีสิทธิ์ครั้งถัดไป
หลังเส้นตายของคำเตือนยังคงจัดให้ผู้เผยแพร่อยู่ใน
เกณฑ์อาจถูกแบน ClawHub อาจใช้การดำเนินการกับบัญชีโดยอัตโนมัติ
สัญญาณการตรวจสอบที่มีความเชื่อมั่นต่ำกว่าและจำกัดตามช่วงเวลาจะไม่รวมอยู่ในการบังคับใช้
อัตโนมัติ

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้โทเค็น API ของ ClawHub ได้ หากการยืนยันตัวตน CLI
เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ web UI เพื่อตรวจสอบสถานะบัญชี
หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อกโดยการแบนหรือบัญชีที่ถูกปิดใช้งาน
ให้ใช้ [แบบฟอร์มอุทธรณ์ ClawHub](https://appeals.openclaw.ai/) เพื่อให้ตรวจสอบการกู้คืน

หากอีเมลที่ทริกเกอร์โดยสแกนเนอร์ระบุเวอร์ชันทักษะหรือ Plugin ว่าเป็นอันตราย
ให้ดาวน์โหลดผลการสแกนที่จัดเก็บไว้สำหรับเวอร์ชันที่ส่งแล้วซึ่งถูกบล็อก:
`clawhub scan download <slug> --version <version>` สำหรับ Plugin ให้เพิ่ม
`--kind plugin` ตรวจสอบผลลัพธ์การสแกน แก้ไขรายการ เพิ่มหมายเลขเวอร์ชัน
และอัปโหลดเวอร์ชันที่แก้ไขแล้ว

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลดผลบวกเทียมและเพิ่มความเชื่อถือของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และบันทึกการเปลี่ยนแปลงให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่ต้องใช้
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้อ่านยาก
- ลิงก์ไปยังซอร์สเมื่อทำได้
- ใช้ dry run ก่อนเผยแพร่ Plugin
- ตอบอย่างชัดเจนหากผู้ใช้หรือผู้ดูแลเนื้อหาถามเกี่ยวกับพฤติกรรมของรีลีส
