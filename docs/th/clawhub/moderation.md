# การกลั่นกรองและความปลอดภัยของบัญชี

ClawHub เปิดให้เผยแพร่ได้ แต่พื้นผิวการค้นพบสาธารณะและการติดตั้งยังคง
ต้องมีมาตรการป้องกัน รายงาน การพักไว้เพื่อการกลั่นกรอง รายการที่ซ่อน และการดำเนินการกับบัญชี
ช่วยปกป้องผู้ใช้เมื่อรีลีสหรือบัญชีดูไม่ปลอดภัย ชวนให้เข้าใจผิด หรืออยู่นอก
นโยบาย

หน้านี้ครอบคลุมการกลั่นกรองและสถานะบัญชี สำหรับป้ายกำกับการตรวจสอบ เช่น
`Pass`, `Review`, `Warn`, `Malicious` และระดับความเสี่ยง โปรดดู
[การตรวจสอบความปลอดภัย](/clawhub/security-audits)

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage) สำหรับข้อกังวลด้านลิขสิทธิ์หรือสิทธิ์ในเนื้อหาอื่น ๆ
ให้ใช้ [คำขอสิทธิ์ในเนื้อหา](/clawhub/content-rights)

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน Skills, Plugin และแพ็กเกจได้

ใช้รายงานของ ClawHub เฉพาะกับเนื้อหา marketplace ที่ไม่ปลอดภัย เช่น:

- รายการที่เป็นอันตราย
- เมทาดาทาที่ชวนให้เข้าใจผิด
- ข้อมูลประจำตัวหรือข้อกำหนดสิทธิ์ที่ไม่ได้ประกาศ
- คำแนะนำการติดตั้งที่น่าสงสัย
- การแอบอ้างบุคคลหรือองค์กร
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

ใช้ปุ่ม **รายงาน Skill** บนหน้า Skill หรือคำสั่ง/API สำหรับรายงานแพ็กเกจ

อย่าใช้รายงานของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ Skill หรือ
Plugin บุคคลที่สามเอง ให้รายงานโดยตรงต่อผู้เผยแพร่หรือคลังซอร์ส
ที่ลิงก์จากรายการ ClawHub ไม่ได้ดูแลหรือแพตช์
โค้ด Skill หรือ Plugin บุคคลที่สาม

GitHub Security Advisories สำหรับ `openclaw/clawhub` ใช้สำหรับช่องโหว่ใน
ClawHub เอง ตัวอย่างเช่น บั๊กในเว็บไซต์, API, CLI, รีจิสทรี, การยืนยันตัวตน,
การสแกน, การกลั่นกรอง หรือขอบเขตความเชื่อถือของการดาวน์โหลด/ติดตั้ง อย่าใช้คำแนะนำของ ClawHub
สำหรับช่องโหว่ใน Skills หรือ Plugin บุคคลที่สาม

รายงานที่ดีต้องเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้ระบบรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

## การอ้างสิทธิ์องค์กรและเนมสเปซ

ข้อพิพาทเรื่องความเป็นเจ้าขององค์กร แบรนด์ ขอบเขตแพ็กเกจ แฮนเดิลเจ้าของ หรือเนมสเปซ ควร
ใช้กระบวนการ [การอ้างสิทธิ์องค์กรและเนมสเปซ](/clawhub/namespace-claims) ไม่ใช่
ขั้นตอนรายงานในผลิตภัณฑ์หรือแบบฟอร์มอุทธรณ์บัญชี

ใช้กระบวนการนั้นเมื่อคุณต้องการให้เจ้าหน้าที่ ClawHub ตรวจสอบหลักฐานที่ไม่ละเอียดอ่อนว่า
เนมสเปซควรถูกสงวนไว้ โอน เปลี่ยนชื่อ ซ่อน กักกัน ทำเป็นนามแฝง
หรือได้รับการตรวจสอบด้วยวิธีอื่น อย่าใส่ความลับ เอกสารส่วนตัว ไฟล์กฎหมายส่วนตัว
เอกสารยืนยันตัวตนส่วนบุคคล โทเค็น API หรือโทเค็นท้าทาย DNS ใน
issue สาธารณะ

## การพักไว้เพื่อการกลั่นกรอง

ผลการตรวจพบที่รุนแรงบางอย่างหรือปัญหาด้านนโยบายอาจทำให้ผู้เผยแพร่หรือรายการถูก
พักไว้เพื่อการกลั่นกรอง เมื่อเกิดเหตุการณ์นี้ เนื้อหาที่ได้รับผลกระทบอาจถูกซ่อนจากการค้นพบ
สาธารณะ หรือการเผยแพร่ในอนาคตอาจเริ่มต้นเป็นแบบซ่อนจนกว่าปัญหาจะได้รับการตรวจสอบ

การพักไว้เพื่อการกลั่นกรองมีไว้เพื่อปกป้องผู้ใช้ขณะที่ ClawHub จัดการกรณี
ความเสี่ยงสูง และยังสามารถยกเลิกได้เมื่อยืนยันว่าเป็นผลบวกลวง

## รายการที่ซ่อนหรือถูกบล็อก

รายการอาจถูกพักไว้ ซ่อน กักกัน เพิกถอน หรือไม่พร้อมใช้งานบน
พื้นผิวการติดตั้งสาธารณะด้วยวิธีอื่น

หากคุณเห็นสถานะใดสถานะหนึ่งเหล่านี้ อย่าติดตั้งรีลีสนั้น เว้นแต่เจ้าของ
จะแก้ไขปัญหาหรือการกลั่นกรองจะคืนสถานะให้

เจ้าของอาจยังเห็นการวินิจฉัยสำหรับรายการของตนเองที่ถูกพักไว้หรือซ่อนอยู่ การวินิจฉัยเหล่านี้
ช่วยอธิบายว่าเกิดอะไรขึ้นและต้องเปลี่ยนอะไรบ้างก่อนที่
รายการจะกลับสู่พื้นผิวสาธารณะได้

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบายของ ClawHub อาจสูญเสียสิทธิ์การเผยแพร่ การใช้ในทางที่ผิดอย่างรุนแรงอาจ
ส่งผลให้บัญชีถูกแบน โทเค็นถูกเพิกถอน เนื้อหาถูกซ่อน หรือรายการถูกลบ
สัญญาณแรงกดดันจากการใช้ในทางที่ผิดของผู้เผยแพร่จะถูกตรวจสอบทุกวัน สัญญาณที่ถึง
เกณฑ์อาจถูกแบนของ ClawHub สามารถทริกเกอร์คำเตือนอัตโนมัติได้ หากการสแกนที่เข้าเกณฑ์ถัดไป
หลังพ้นกำหนดคำเตือนยังคงจัดให้ผู้เผยแพร่อยู่ใน
เกณฑ์อาจถูกแบน ClawHub อาจใช้การดำเนินการกับบัญชีโดยอัตโนมัติ
สัญญาณการตรวจสอบที่มีความมั่นใจต่ำกว่าและจำกัดตามช่วงเวลาจะไม่เข้าสู่
การบังคับใช้อัตโนมัติ

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้โทเค็น API ของ ClawHub ได้ หากการยืนยันตัวตน CLI
เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ UI บนเว็บเพื่อตรวจสอบ
สถานะบัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อกโดยการแบนหรือบัญชีที่ถูกปิดใช้งาน
ให้ใช้ [แบบฟอร์มอุทธรณ์ ClawHub](https://appeals.openclaw.ai/) เพื่อขอรับการตรวจสอบการกู้คืน

หากอีเมลที่ทริกเกอร์โดยสแกนเนอร์ระบุว่าเวอร์ชันของ Skill หรือ Plugin เป็นอันตราย
ให้ดาวน์โหลดผลการสแกนที่จัดเก็บไว้สำหรับเวอร์ชันที่ส่งมาซึ่งถูกบล็อก:
`clawhub scan download <slug> --version <version>` สำหรับ Plugin ให้เพิ่ม
`--kind plugin` ตรวจสอบเอาต์พุตการสแกน แก้ไขรายการ เพิ่มหมายเลขเวอร์ชัน
และอัปโหลดเวอร์ชันที่แก้ไขแล้ว

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลดผลบวกลวงและเพิ่มความเชื่อมั่นของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และบันทึกการเปลี่ยนแปลงให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้อ่านยากหรือคลุมเครือ
- ลิงก์ไปยังซอร์สเมื่อทำได้
- ใช้การรันแบบทดลองก่อนเผยแพร่ Plugin
- ตอบกลับให้ชัดเจนหากผู้ใช้หรือผู้กลั่นกรองถามเกี่ยวกับพฤติกรรมของรีลีส
