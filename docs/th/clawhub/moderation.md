# การกลั่นกรองและความปลอดภัยของบัญชี

ClawHub เปิดให้เผยแพร่ได้ แต่พื้นผิวการค้นพบและติดตั้งแบบสาธารณะยังคง
ต้องมีรั้วป้องกัน รายงาน การพักไว้เพื่อการกลั่นกรอง รายการที่ถูกซ่อน และการดำเนินการกับบัญชี
ช่วยปกป้องผู้ใช้เมื่อรีลีสหรือบัญชีดูไม่ปลอดภัย ทำให้เข้าใจผิด หรืออยู่นอก
นโยบาย

หน้านี้ครอบคลุมการกลั่นกรองและสถานะบัญชี สำหรับป้ายกำกับการตรวจสอบ เช่น
`Pass`, `Review`, `Warn`, `Malicious` และระดับความเสี่ยง โปรดดู
[การตรวจสอบความปลอดภัย](/clawhub/security-audits)

ดูเพิ่มเติมที่ [ความปลอดภัย](/th/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage) สำหรับข้อกังวลด้านลิขสิทธิ์หรือสิทธิ์
เนื้อหาอื่น ๆ ให้ใช้ [คำขอสิทธิ์เนื้อหา](/clawhub/content-rights)

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้แล้วสามารถรายงาน Skills, Plugins และแพ็กเกจได้

ใช้รายงาน ClawHub เฉพาะสำหรับเนื้อหา marketplace ที่ไม่ปลอดภัย เช่น:

- รายการที่เป็นอันตราย
- metadata ที่ทำให้เข้าใจผิด
- ข้อมูลประจำตัวหรือข้อกำหนดสิทธิ์ที่ไม่ได้ประกาศ
- คำแนะนำการติดตั้งที่น่าสงสัย
- การแอบอ้าง
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

ใช้ปุ่ม **รายงาน skill** บนหน้า skill หรือคำสั่ง/API รายงานแพ็กเกจ
สำหรับแพ็กเกจ

อย่าใช้รายงาน ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ skill หรือ
Plugin บุคคลที่สามเอง ให้รายงานโดยตรงต่อผู้เผยแพร่หรือที่เก็บซอร์ส
ที่ลิงก์จากรายการ ClawHub ไม่ได้ดูแลหรือแพตช์โค้ด
skill หรือ Plugin บุคคลที่สาม

GitHub Security Advisories สำหรับ `openclaw/clawhub` ใช้สำหรับช่องโหว่ใน
ClawHub เอง ตัวอย่างรวมถึงบั๊กในเว็บไซต์, API, CLI, registry, auth,
การสแกน, การกลั่นกรอง หรือขอบเขตความเชื่อถือของการดาวน์โหลด/ติดตั้ง อย่าใช้ advisory ของ ClawHub
สำหรับช่องโหว่ใน Skills หรือ Plugins บุคคลที่สาม

รายงานที่ดีต้องเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้การรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

## การอ้างสิทธิ์องค์กรและ namespace

ข้อพิพาทเรื่องความเป็นเจ้าขององค์กร แบรนด์ ขอบเขตแพ็กเกจ แฮนเดิลเจ้าของ หรือ namespace ควร
ใช้กระบวนการ [การอ้างสิทธิ์องค์กรและ Namespace](/clawhub/namespace-claims) ไม่ใช่
ขั้นตอนรายงานในผลิตภัณฑ์หรือแบบฟอร์มอุทธรณ์บัญชี

ใช้กระบวนการนั้นเมื่อคุณต้องการให้เจ้าหน้าที่ ClawHub ตรวจสอบหลักฐานที่ไม่อ่อนไหวว่า
namespace ควรถูกสงวน โอน เปลี่ยนชื่อ ซ่อน กักกัน ทำ alias
หรือได้รับการตรวจสอบด้วยเหตุผลอื่น อย่าใส่ความลับ เอกสารส่วนตัว ไฟล์กฎหมายส่วนตัว
เอกสารระบุตัวตนส่วนบุคคล โทเค็น API หรือโทเค็น DNS challenge ใน
issue สาธารณะ

## การพักไว้เพื่อการกลั่นกรอง

ผลการตรวจพบที่ร้ายแรงหรือประเด็นด้านนโยบายบางอย่างสามารถทำให้ผู้เผยแพร่หรือรายการอยู่ภายใต้
การพักไว้เพื่อการกลั่นกรองได้ เมื่อเกิดกรณีนี้ เนื้อหาที่ได้รับผลกระทบอาจถูกซ่อนจากการค้นพบ
สาธารณะ หรือการเผยแพร่ในอนาคตอาจเริ่มต้นแบบถูกซ่อนไว้จนกว่าประเด็นจะได้รับการตรวจสอบ

การพักไว้เพื่อการกลั่นกรองมีไว้เพื่อปกป้องผู้ใช้ระหว่างที่ ClawHub แก้ไขกรณีที่มีความเสี่ยงสูง
และยังสามารถยกเลิกได้เมื่อยืนยันว่าเป็นผลบวกเท็จ

## รายการที่ถูกซ่อนหรือถูกบล็อก

รายการอาจถูกพักไว้ ซ่อน กักกัน เพิกถอน หรือไม่พร้อมใช้งานด้วยเหตุผลอื่นบน
พื้นผิวการติดตั้งสาธารณะ

หากคุณเห็นสถานะเหล่านี้ อย่าติดตั้งรีลีสนั้น เว้นแต่เจ้าของ
จะแก้ไขประเด็นหรือการกลั่นกรองกู้คืนรายการนั้น

เจ้าของยังอาจเห็น diagnostics สำหรับรายการของตนเองที่ถูกพักไว้หรือถูกซ่อน diagnostics เหล่านี้
ช่วยอธิบายว่าเกิดอะไรขึ้นและต้องเปลี่ยนอะไร ก่อนที่
รายการจะกลับสู่พื้นผิวสาธารณะได้

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบาย ClawHub อาจสูญเสียสิทธิ์การเผยแพร่ การละเมิดที่ร้ายแรงอาจ
ส่งผลให้บัญชีถูกแบน โทเค็นถูกเพิกถอน เนื้อหาถูกซ่อน หรือรายการถูกลบ
สัญญาณแรงกดดันจากการละเมิดของผู้เผยแพร่จะถูกตรวจสอบทุกวัน สัญญาณที่ถึง
เกณฑ์ potential-ban ของ ClawHub อาจทริกเกอร์คำเตือนอัตโนมัติ หากการสแกนที่เข้าเกณฑ์ครั้งถัดไป
หลังเส้นตายคำเตือนยังคงจัดให้ผู้เผยแพร่อยู่ใน
เกณฑ์ potential-ban ClawHub อาจใช้การดำเนินการกับบัญชีโดยอัตโนมัติ
สัญญาณการตรวจสอบที่มีความเชื่อมั่นต่ำกว่าและจำกัดตามช่วงเวลาจะไม่เข้าสู่
การบังคับใช้อัตโนมัติ

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้โทเค็น API ของ ClawHub ได้ หาก auth ของ CLI
เริ่มล้มเหลวหลังการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ web UI เพื่อตรวจสอบ
สถานะบัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อกโดยการแบนหรือบัญชีที่ถูกปิดใช้งาน
ให้ใช้ [แบบฟอร์มอุทธรณ์ ClawHub](https://appeals.openclaw.ai/) สำหรับการตรวจสอบเพื่อกู้คืน

หากอีเมลที่ทริกเกอร์โดย scanner ระบุว่าเวอร์ชันของ skill หรือ Plugin เป็นอันตราย
ให้ดาวน์โหลดผลการสแกนที่เก็บไว้สำหรับเวอร์ชันที่ส่งและถูกบล็อก:
`clawhub scan download <slug> --version <version>` สำหรับ Plugins ให้เพิ่ม
`--kind plugin` ตรวจสอบผลลัพธ์การสแกน แก้ไขรายการ เพิ่มหมายเลขเวอร์ชัน
และอัปโหลดเวอร์ชันที่แก้ไขแล้ว

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลดผลบวกเท็จและเพิ่มความไว้วางใจของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และ changelog ให้ถูกต้อง
- ประกาศ environment variables และสิทธิ์ที่จำเป็น
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้อ่านยาก
- ลิงก์ไปยังซอร์สเมื่อเป็นไปได้
- ใช้ dry run ก่อนเผยแพร่ Plugins
- ตอบอย่างชัดเจนหากผู้ใช้หรือผู้กลั่นกรองถามเกี่ยวกับพฤติกรรมของรีลีส
