# การกลั่นกรองและความปลอดภัยของบัญชี

ClawHub เปิดให้เผยแพร่ได้ แต่พื้นผิวสำหรับการค้นพบสาธารณะและการติดตั้งยังคงต้องมีมาตรการป้องกัน รายงาน การพักไว้เพื่อตรวจสอบ การซ่อนรายการ และการดำเนินการกับบัญชีช่วยปกป้องผู้ใช้เมื่อ release หรือบัญชีดูไม่ปลอดภัย ชวนให้เข้าใจผิด หรือไม่เป็นไปตามนโยบาย

หน้านี้ครอบคลุมการกลั่นกรองและสถานะบัญชี สำหรับป้ายกำกับการตรวจสอบ เช่น `Pass`, `Review`, `Warn`, `Malicious` และระดับความเสี่ยง โปรดดู
[การตรวจสอบความปลอดภัย](/th/clawhub/security-audits)

ดูเพิ่มเติมที่ [ความปลอดภัย](/th/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage) สำหรับข้อกังวลเกี่ยวกับลิขสิทธิ์หรือสิทธิ์เนื้อหาอื่น ๆ ให้ใช้ [คำขอเกี่ยวกับสิทธิ์เนื้อหา](/th/clawhub/content-rights)

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน skill, plugin และแพ็กเกจได้

ใช้รายงานของ ClawHub เฉพาะกับเนื้อหา marketplace ที่ไม่ปลอดภัย เช่น:

- รายการที่เป็นอันตราย
- metadata ที่ชวนให้เข้าใจผิด
- ข้อมูลประจำตัวหรือข้อกำหนดด้านสิทธิ์ที่ไม่ได้ประกาศ
- คำแนะนำการติดตั้งที่น่าสงสัย
- การแอบอ้างบุคคลหรือองค์กร
- การจดทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

ใช้ปุ่ม **รายงาน skill** บนหน้า skill หรือคำสั่ง/API สำหรับรายงานแพ็กเกจ

อย่าใช้รายงานของ ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ skill หรือ plugin ของบุคคลที่สาม ให้รายงานโดยตรงต่อผู้เผยแพร่หรือ repository ต้นทางที่ลิงก์จากรายการ ClawHub ไม่ได้ดูแลหรือ patch โค้ด skill หรือ plugin ของบุคคลที่สาม

GitHub Security Advisories สำหรับ `openclaw/clawhub` ใช้สำหรับช่องโหว่ใน ClawHub เอง ตัวอย่างรวมถึงบั๊กในเว็บไซต์, API, CLI, registry, auth, การสแกน, การกลั่นกรอง หรือขอบเขตความน่าเชื่อถือของการดาวน์โหลด/ติดตั้ง อย่าใช้ advisory ของ ClawHub สำหรับช่องโหว่ใน skill หรือ plugin ของบุคคลที่สาม

รายงานที่ดีต้องเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้ระบบรายงานในทางที่ผิดอาจนำไปสู่การดำเนินการกับบัญชีได้เช่นกัน

## การอ้างสิทธิ์ org และ namespace

ข้อพิพาทเกี่ยวกับ org, แบรนด์, scope ของแพ็กเกจ, handle เจ้าของ หรือความเป็นเจ้าของ namespace ควรใช้กระบวนการ [การอ้างสิทธิ์ Org และ Namespace](/th/clawhub/namespace-claims) ไม่ใช่ flow รายงานในผลิตภัณฑ์หรือแบบฟอร์มอุทธรณ์บัญชี

ใช้กระบวนการนั้นเมื่อคุณต้องการให้เจ้าหน้าที่ ClawHub ตรวจสอบหลักฐานที่ไม่ละเอียดอ่อนว่า namespace ควรถูกจองไว้ โอน เปลี่ยนชื่อ ซ่อน กักกัน ทำ alias หรือตรวจสอบด้วยวิธีอื่น อย่าใส่ secret, เอกสารส่วนตัว, ไฟล์กฎหมายส่วนตัว, เอกสารยืนยันตัวตนส่วนบุคคล, API token หรือ DNS challenge token ใน issue สาธารณะ

## การพักไว้เพื่อตรวจสอบ

ผลการตรวจสอบร้ายแรงบางอย่างหรือปัญหาด้านนโยบายอาจทำให้ผู้เผยแพร่หรือรายการถูกพักไว้เพื่อตรวจสอบ เมื่อเกิดเหตุการณ์นี้ เนื้อหาที่ได้รับผลกระทบอาจถูกซ่อนจากการค้นพบสาธารณะ หรือการเผยแพร่ในอนาคตอาจเริ่มต้นเป็นสถานะซ่อนจนกว่าปัญหาจะได้รับการตรวจสอบ

การพักไว้เพื่อตรวจสอบมีไว้เพื่อปกป้องผู้ใช้ในขณะที่ ClawHub แก้ไขกรณีที่มีความเสี่ยงสูง และสามารถยกเลิกได้เมื่อยืนยันว่าเป็น false positive

## รายการที่ซ่อนหรือถูกบล็อก

รายการอาจถูกพักไว้ ซ่อน กักกัน เพิกถอน หรือไม่พร้อมใช้งานด้วยวิธีอื่นบนพื้นผิวการติดตั้งสาธารณะ

หากคุณเห็นสถานะเหล่านี้ อย่าติดตั้ง release นั้น เว้นแต่เจ้าของจะแก้ไขปัญหาหรือการกลั่นกรองจะกู้คืนรายการนั้น

เจ้าของอาจยังเห็น diagnostics สำหรับรายการของตนเองที่ถูกพักไว้หรือซ่อนอยู่ diagnostics เหล่านี้ช่วยอธิบายว่าเกิดอะไรขึ้นและต้องเปลี่ยนแปลงอะไร ก่อนที่รายการจะกลับสู่พื้นผิวสาธารณะได้

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบายของ ClawHub อาจสูญเสียสิทธิ์การเผยแพร่ การละเมิดร้ายแรงอาจส่งผลให้บัญชีถูกแบน token ถูกเพิกถอน เนื้อหาถูกซ่อน หรือรายการถูกลบ สัญญาณแรงกดดันจากการละเมิดของผู้เผยแพร่จะถูกตรวจสอบทุกวัน สัญญาณที่ถึง threshold การแบนที่เป็นไปได้ของ ClawHub อาจเรียกการเตือนอัตโนมัติ หากการสแกนครั้งถัดไปที่เข้าเกณฑ์หลัง deadline การเตือนยังคงจัดให้ผู้เผยแพร่อยู่ใน threshold การแบนที่เป็นไปได้ ClawHub อาจใช้การดำเนินการกับบัญชีโดยอัตโนมัติ สัญญาณการตรวจสอบที่มีความมั่นใจต่ำกว่าและจำกัดตามช่วงเวลาจะไม่ถูกนำไปบังคับใช้อัตโนมัติ

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้ API token ของ ClawHub ได้ หาก CLI auth เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ web UI เพื่อตรวจสอบสถานะบัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อกโดยการแบนหรือบัญชีที่ถูกปิดใช้งาน ให้ใช้ [แบบฟอร์มอุทธรณ์ ClawHub](https://appeals.openclaw.ai/) เพื่อขอการตรวจสอบการกู้คืน

หากอีเมลที่ถูกเรียกโดย scanner ระบุว่าเวอร์ชันของ skill หรือ plugin เป็นอันตราย ให้ดาวน์โหลดผลการสแกนที่จัดเก็บไว้สำหรับเวอร์ชันที่ส่งและถูกบล็อก:
`clawhub scan download <slug> --version <version>` สำหรับ plugin ให้เพิ่ม
`--kind plugin` ตรวจสอบผลลัพธ์การสแกน แก้ไขรายการ เพิ่มหมายเลขเวอร์ชัน และอัปโหลดเวอร์ชันที่แก้ไขแล้ว

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลด false positive และเพิ่มความไว้วางใจของผู้ใช้:

- รักษาชื่อ สรุป tags และ changelogs ให้ถูกต้อง
- ประกาศ environment variables และสิทธิ์ที่จำเป็น
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้เข้าใจยาก
- ลิงก์ไปยังซอร์สเมื่อเป็นไปได้
- ใช้ dry run ก่อนเผยแพร่ plugin
- ตอบอย่างชัดเจนหากผู้ใช้หรือผู้กลั่นกรองถามเกี่ยวกับพฤติกรรมของ release
