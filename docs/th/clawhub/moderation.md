# การดูแลและความปลอดภัยของบัญชี

ClawHub เปิดให้เผยแพร่ได้ แต่พื้นผิวการค้นพบและติดตั้งแบบสาธารณะยังคง
ต้องมีมาตรการป้องกัน รายงาน การพักไว้เพื่อการดูแล รายการที่ถูกซ่อน และการดำเนินการกับบัญชี
ช่วยปกป้องผู้ใช้เมื่อรีลีสหรือบัญชีดูไม่ปลอดภัย ทำให้เข้าใจผิด หรืออยู่นอกนโยบาย

หน้านี้ครอบคลุมการดูแลและสถานะบัญชี สำหรับป้ายกำกับการตรวจสอบ เช่น
`Pass`, `Review`, `Warn`, `Malicious` และระดับความเสี่ยง โปรดดู
[การตรวจสอบความปลอดภัย](/clawhub/security-audits)

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage) สำหรับข้อกังวลด้านลิขสิทธิ์หรือสิทธิ์เนื้อหาอื่น ๆ
ให้ใช้ [คำขอสิทธิ์เนื้อหา](/clawhub/content-rights)

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน Skills, Plugin และแพ็กเกจได้

ใช้รายงานของ ClawHub เฉพาะกับเนื้อหาตลาดที่ไม่ปลอดภัย เช่น:

- รายการที่เป็นอันตราย
- เมตาดาตาที่ทำให้เข้าใจผิด
- ข้อมูลประจำตัวหรือข้อกำหนดสิทธิ์ที่ไม่ได้ประกาศ
- คำแนะนำการติดตั้งที่น่าสงสัย
- การแอบอ้าง
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

ใช้ปุ่ม **รายงาน Skill** บนหน้า Skill หรือคำสั่ง/API สำหรับรายงานแพ็กเกจ

อย่าใช้รายงานของ ClawHub สำหรับช่องโหว่ในโค้ดต้นฉบับของ Skill หรือ
Plugin บุคคลที่สามเอง ให้รายงานโดยตรงต่อผู้เผยแพร่หรือที่เก็บซอร์ส
ที่ลิงก์จากรายการ ClawHub ไม่ได้บำรุงรักษาหรือแพตช์
โค้ด Skill หรือ Plugin ของบุคคลที่สาม

GitHub Security Advisories สำหรับ `openclaw/clawhub` ใช้สำหรับช่องโหว่ใน
ClawHub เอง ตัวอย่างเช่นบั๊กในเว็บไซต์, API, CLI, รีจิสทรี, auth,
การสแกน, การดูแล หรือขอบเขตความเชื่อถือของการดาวน์โหลด/ติดตั้ง อย่าใช้คำแนะนำของ ClawHub
สำหรับช่องโหว่ใน Skills หรือ Plugin ของบุคคลที่สาม

รายงานที่ดีต้องเฉพาะเจาะจงและนำไปดำเนินการได้ การใช้การรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

## การอ้างสิทธิ์ Org และ namespace

ข้อพิพาทเกี่ยวกับความเป็นเจ้าของ Org, แบรนด์, ขอบเขตแพ็กเกจ, owner-handle หรือ namespace ควร
ใช้กระบวนการ [การอ้างสิทธิ์ Org และ Namespace](/clawhub/namespace-claims) ไม่ใช่
โฟลว์รายงานในผลิตภัณฑ์หรือแบบฟอร์มอุทธรณ์บัญชี

ใช้กระบวนการนั้นเมื่อคุณต้องการให้เจ้าหน้าที่ ClawHub ตรวจสอบหลักฐานที่ไม่ละเอียดอ่อนว่า
namespace ควรถูกสงวนไว้ โอน เปลี่ยนชื่อ ซ่อน กักกัน ทำ alias
หรือตรวจสอบในรูปแบบอื่น อย่าใส่ความลับ เอกสารส่วนตัว ไฟล์กฎหมายส่วนตัว
เอกสารยืนยันตัวตนส่วนบุคคล, API tokens หรือ DNS challenge tokens ใน
issue สาธารณะ

## การพักไว้เพื่อการดูแล

ผลการตรวจพบที่ร้ายแรงบางอย่างหรือปัญหาด้านนโยบายอาจทำให้ผู้เผยแพร่หรือรายการถูก
พักไว้เพื่อการดูแล เมื่อเกิดเหตุการณ์นี้ เนื้อหาที่ได้รับผลกระทบอาจถูกซ่อนจาก
การค้นพบแบบสาธารณะ หรือการเผยแพร่ในอนาคตอาจเริ่มต้นแบบซ่อนจนกว่าปัญหาจะได้รับการตรวจสอบ

การพักไว้เพื่อการดูแลมีไว้เพื่อปกป้องผู้ใช้ในขณะที่ ClawHub แก้ไขกรณีที่มีความเสี่ยงสูง
นอกจากนี้ยังสามารถยกเลิกได้เมื่อยืนยันว่าเป็นผลบวกลวง

## รายการที่ถูกซ่อนหรือถูกบล็อก

รายการอาจถูกพักไว้ ซ่อน กักกัน เพิกถอน หรือไม่พร้อมใช้งานในรูปแบบอื่นบน
พื้นผิวการติดตั้งสาธารณะ

หากคุณเห็นสถานะใดสถานะหนึ่งเหล่านี้ อย่าติดตั้งรีลีสนั้น เว้นแต่เจ้าของ
จะแก้ไขปัญหาหรือฝ่ายดูแลกู้คืนรายการแล้ว

เจ้าของอาจยังเห็นการวินิจฉัยสำหรับรายการของตนเองที่ถูกพักไว้หรือถูกซ่อน การวินิจฉัยเหล่านี้
ช่วยอธิบายว่าเกิดอะไรขึ้นและต้องเปลี่ยนแปลงอะไรบ้างก่อนที่
รายการจะกลับสู่พื้นผิวสาธารณะได้

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบายของ ClawHub อาจสูญเสียสิทธิ์การเผยแพร่ การละเมิดที่ร้ายแรงอาจ
ส่งผลให้บัญชีถูกแบน, token ถูกเพิกถอน, เนื้อหาถูกซ่อน หรือรายการถูกลบ
สัญญาณแรงกดดันจากการละเมิดของผู้เผยแพร่จะถูกตรวจสอบทุกวัน สัญญาณที่ถึง
เกณฑ์อาจถูกแบนของ ClawHub อาจทริกเกอร์คำเตือนอัตโนมัติ หากการสแกนที่มีสิทธิ์ครั้งถัดไป
หลังจากกำหนดเวลาคำเตือนยังคงจัดผู้เผยแพร่ไว้ใน
เกณฑ์อาจถูกแบน ClawHub อาจใช้การดำเนินการกับบัญชีโดยอัตโนมัติ
สัญญาณการตรวจสอบที่มีความเชื่อมั่นต่ำกว่าและจำกัดตามช่วงเวลาจะไม่อยู่ในการบังคับใช้
อัตโนมัติ

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้ ClawHub API tokens ได้ หาก CLI auth
เริ่มล้มเหลวหลังจากการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ UI บนเว็บเพื่อตรวจสอบ
สถานะบัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อกโดยการแบนหรือบัญชีที่ถูกปิดใช้งาน
ให้ใช้ [แบบฟอร์มอุทธรณ์ ClawHub](https://appeals.openclaw.ai/) เพื่อขอการตรวจสอบการกู้คืน

หากอีเมลที่ทริกเกอร์โดยสแกนเนอร์ตั้งชื่อเวอร์ชัน Skill หรือ Plugin ว่าเป็นอันตราย
ให้ดาวน์โหลดผลการสแกนที่เก็บไว้สำหรับเวอร์ชันที่ส่งและถูกบล็อก:
`clawhub scan download <slug> --version <version>` สำหรับ Plugin ให้เพิ่ม
`--kind plugin` ตรวจสอบเอาต์พุตการสแกน แก้ไขรายการ เพิ่มหมายเลขเวอร์ชัน
และอัปโหลดเวอร์ชันที่แก้ไขแล้ว

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลดผลบวกลวงและเพิ่มความเชื่อมั่นของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และ changelogs ให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้อ่านยาก
- ลิงก์ไปยังซอร์สเมื่อทำได้
- ใช้ dry runs ก่อนเผยแพร่ Plugin
- ตอบอย่างชัดเจนหากผู้ใช้หรือผู้ดูแลถามเกี่ยวกับพฤติกรรมของรีลีส
