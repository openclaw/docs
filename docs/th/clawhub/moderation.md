# การกลั่นกรองและความปลอดภัยของบัญชี

ClawHub เปิดให้เผยแพร่ได้ แต่พื้นผิวสำหรับการค้นพบและติดตั้งแบบสาธารณะยังคง
ต้องมีมาตรการป้องกัน รายงาน การระงับเพื่อกลั่นกรอง รายการที่ถูกซ่อน และการดำเนินการกับบัญชี
ช่วยปกป้องผู้ใช้เมื่อรีลีสหรือบัญชีดูไม่ปลอดภัย ทำให้เข้าใจผิด หรืออยู่นอก
นโยบาย

หน้านี้ครอบคลุมการกลั่นกรองและสถานะบัญชี สำหรับป้ายกำกับการตรวจสอบ เช่น
`Pass`, `Review`, `Warn`, `Malicious` และระดับความเสี่ยง โปรดดู
[การตรวจสอบความปลอดภัย](/clawhub/security-audits)

ดูเพิ่มเติมที่ [ความปลอดภัย](/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage) สำหรับข้อกังวลเกี่ยวกับลิขสิทธิ์หรือสิทธิ์
ด้านเนื้อหาอื่น ๆ ให้ใช้ [คำขอสิทธิ์เนื้อหา](/clawhub/content-rights)

## รายงาน

ผู้ใช้ที่ลงชื่อเข้าใช้สามารถรายงาน Skills, Plugins และแพ็กเกจได้

ใช้รายงาน ClawHub เฉพาะสำหรับเนื้อหา marketplace ที่ไม่ปลอดภัย เช่น:

- รายการที่เป็นอันตราย
- เมทาดาทาที่ทำให้เข้าใจผิด
- ข้อมูลประจำตัวหรือข้อกำหนดสิทธิ์ที่ไม่ได้ประกาศ
- คำแนะนำการติดตั้งที่น่าสงสัย
- การแอบอ้าง
- การลงทะเบียนโดยไม่สุจริตหรือการใช้เครื่องหมายการค้าในทางที่ผิด
- เนื้อหาที่ละเมิด [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

ใช้ปุ่ม **รายงาน Skill** บนหน้า Skill หรือคำสั่ง/API สำหรับรายงานแพ็กเกจ
สำหรับแพ็กเกจ

อย่าใช้รายงาน ClawHub สำหรับช่องโหว่ในซอร์สโค้ดของ Skill หรือ
Plugin ของบุคคลที่สามเอง ให้รายงานโดยตรงต่อผู้เผยแพร่หรือ repository ซอร์ส
ที่ลิงก์จากรายการ ClawHub ไม่ได้บำรุงรักษาหรือแพตช์
โค้ด Skill หรือ Plugin ของบุคคลที่สาม

GitHub Security Advisories สำหรับ `openclaw/clawhub` มีไว้สำหรับช่องโหว่ใน
ClawHub เอง ตัวอย่างเช่นบั๊กในเว็บไซต์, API, CLI, registry, auth,
การสแกน, การกลั่นกรอง หรือขอบเขตความเชื่อถือของการดาวน์โหลด/ติดตั้ง อย่าใช้ advisory ของ ClawHub
สำหรับช่องโหว่ใน Skills หรือ Plugins ของบุคคลที่สาม

รายงานที่ดีต้องเจาะจงและนำไปดำเนินการได้ การใช้ระบบรายงานในทางที่ผิดอาจนำไปสู่
การดำเนินการกับบัญชีได้เช่นกัน

## การอ้างสิทธิ์องค์กรและ namespace

ข้อพิพาทเรื่องความเป็นเจ้าขององค์กร แบรนด์ ขอบเขตแพ็กเกจ handle เจ้าของ หรือ namespace ควร
ใช้กระบวนการ [การอ้างสิทธิ์องค์กรและ Namespace](/clawhub/namespace-claims) ไม่ใช่
โฟลว์รายงานในผลิตภัณฑ์หรือแบบฟอร์มอุทธรณ์บัญชี

ใช้กระบวนการดังกล่าวเมื่อคุณต้องการให้เจ้าหน้าที่ ClawHub ตรวจสอบหลักฐานที่ไม่อ่อนไหวว่า
namespace ควรถูกสงวนไว้ โอน เปลี่ยนชื่อ ซ่อน กักกัน ทำเป็น alias
หรือให้ตรวจสอบในลักษณะอื่น อย่าใส่ความลับ เอกสารส่วนตัว ไฟล์กฎหมายส่วนตัว
เอกสารระบุตัวตนส่วนบุคคล API tokens หรือ DNS challenge tokens ใน
issue สาธารณะ

## การระงับเพื่อกลั่นกรอง

ผลการตรวจพบที่รุนแรงหรือปัญหาด้านนโยบายบางอย่างอาจทำให้ผู้เผยแพร่หรือรายการถูก
ระงับเพื่อกลั่นกรอง เมื่อเกิดเหตุการณ์นี้ เนื้อหาที่ได้รับผลกระทบอาจถูกซ่อนจาก
การค้นพบสาธารณะ หรือการเผยแพร่ในอนาคตอาจเริ่มต้นในสถานะซ่อนจนกว่าจะมีการตรวจสอบปัญหา

การระงับเพื่อกลั่นกรองมีไว้เพื่อปกป้องผู้ใช้ระหว่างที่ ClawHub แก้ไขกรณี
ความเสี่ยงสูง และสามารถยกเลิกได้เมื่อยืนยันว่าเป็นผลบวกปลอม

## รายการที่ถูกซ่อนหรือถูกบล็อก

รายการอาจถูกระงับ ซ่อน กักกัน เพิกถอน หรือไม่พร้อมใช้งานในลักษณะอื่นบน
พื้นผิวการติดตั้งสาธารณะ

หากคุณเห็นสถานะเหล่านี้อย่างใดอย่างหนึ่ง อย่าติดตั้งรีลีสนั้น เว้นแต่เจ้าของ
จะแก้ไขปัญหาหรือการกลั่นกรองจะกู้คืนรายการนั้น

เจ้าของอาจยังคงเห็นข้อมูลวินิจฉัยสำหรับรายการของตนเองที่ถูกระงับหรือซ่อนอยู่ ข้อมูลวินิจฉัยเหล่านี้
ช่วยอธิบายว่าเกิดอะไรขึ้นและต้องเปลี่ยนแปลงอะไรบ้างก่อนที่
รายการจะกลับไปยังพื้นผิวสาธารณะได้

## การแบนและสถานะบัญชี

บัญชีที่ละเมิดนโยบาย ClawHub อาจเสียสิทธิ์การเผยแพร่ การละเมิดที่รุนแรงอาจ
ส่งผลให้บัญชีถูกแบน token ถูกเพิกถอน เนื้อหาถูกซ่อน หรือรายการถูกลบ
สัญญาณแรงกดดันจากการละเมิดของผู้เผยแพร่จะถูกตรวจสอบทุกวัน สัญญาณที่ถึง
เกณฑ์ potential-ban ของ ClawHub สามารถทริกเกอร์คำเตือนอัตโนมัติได้ หากการสแกนรอบถัดไป
ที่เข้าเกณฑ์หลังเส้นตายของคำเตือนยังคงจัดให้ผู้เผยแพร่อยู่ใน
เกณฑ์ potential-ban ClawHub อาจดำเนินการกับบัญชีโดยอัตโนมัติ
สัญญาณการตรวจสอบที่มีความเชื่อมั่นต่ำกว่าและมีกรอบเวลาจำกัดจะไม่ถูกนำเข้าสู่
การบังคับใช้อัตโนมัติ

บัญชีที่ถูกลบ ถูกแบน หรือถูกปิดใช้งานไม่สามารถใช้ ClawHub API tokens ได้ หากการยืนยันตัวตนของ CLI
เริ่มล้มเหลวหลังจากมีการดำเนินการกับบัญชี ให้ลงชื่อเข้าใช้ web UI เพื่อตรวจสอบ
สถานะบัญชี หากการลงชื่อเข้าใช้หรือการเข้าถึง CLI ตามปกติถูกบล็อกเนื่องจากการแบนหรือบัญชีถูกปิดใช้งาน
ให้ใช้ [แบบฟอร์มอุทธรณ์ ClawHub](https://appeals.openclaw.ai/) เพื่อขอการตรวจสอบการกู้คืน

หากอีเมลที่ทริกเกอร์โดย scanner ระบุว่าเวอร์ชันของ Skill หรือ Plugin เป็นอันตราย
ให้ดาวน์โหลดผลการสแกนที่จัดเก็บไว้สำหรับเวอร์ชันที่ส่งซึ่งถูกบล็อก:
`clawhub scan download <slug> --version <version>` สำหรับ Plugins ให้เพิ่ม
`--kind plugin` ตรวจสอบผลลัพธ์การสแกน แก้ไขรายการ เพิ่มหมายเลขเวอร์ชัน
และอัปโหลดเวอร์ชันที่แก้ไขแล้ว

## คำแนะนำสำหรับผู้เผยแพร่

เพื่อลดผลบวกปลอมและปรับปรุงความเชื่อถือของผู้ใช้:

- รักษาชื่อ สรุป แท็ก และ changelog ให้ถูกต้อง
- ประกาศตัวแปรสภาพแวดล้อมและสิทธิ์ที่จำเป็น
- หลีกเลี่ยงคำสั่งติดตั้งที่ทำให้คลุมเครือ
- ลิงก์ไปยังซอร์สเมื่อทำได้
- ใช้ dry runs ก่อนเผยแพร่ Plugins
- ตอบให้ชัดเจนหากผู้ใช้หรือผู้กลั่นกรองถามเกี่ยวกับพฤติกรรมของรีลีส
