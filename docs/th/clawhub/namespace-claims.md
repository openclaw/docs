# การอ้างสิทธิ์ในองค์กรและเนมสเปซ

ClawHub ใช้แฮนเดิลของเจ้าของ แฮนเดิลขององค์กร สลักของ Skills ชื่อแพ็กเกจ Plugin และ
ขอบเขตแพ็กเกจเป็นเนมสเปซสาธารณะ หากเนมสเปซดูเหมือนเป็นของ
โครงการ แบรนด์ ระบบนิเวศแพ็กเกจ หรือองค์กรในโลกจริง แต่ถูก
อ้างสิทธิ์ สงวนไว้ ทำให้เข้าใจผิด หรือมีข้อพิพาทบน ClawHub แล้ว ให้ขอให้เจ้าหน้าที่ตรวจสอบ
ผ่าน
[แบบฟอร์มปัญหาการอ้างสิทธิ์ในองค์กร/เนมสเปซ](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)

ใช้ช่องทางนี้สำหรับการตรวจสอบความเป็นเจ้าของแบบสาธารณะที่ไม่มีข้อมูลละเอียดอ่อน อย่าใช้รายงาน
ภายในผลิตภัณฑ์หรือแบบฟอร์มอุทธรณ์บัญชีสำหรับการอ้างสิทธิ์ในเนมสเปซ

## เมื่อใดควรยื่นคำร้อง

ยื่นคำร้องเกี่ยวกับเนมสเปซเมื่อคุณเชื่อว่าเจ้าหน้าที่ ClawHub ควรตรวจสอบว่า
ควรสงวน โอน เปลี่ยนชื่อ ซ่อน กักกัน ตั้งนามแฝง
หรือเปลี่ยนแปลงเนมสเปซในลักษณะอื่นเนื่องจากความเป็นเจ้าของในโลกจริงหรือไม่

ตัวอย่างได้แก่:

- แฮนเดิลขององค์กรที่ตรงกับองค์กร GitHub โครงการ บริษัท หรือชุมชนของคุณ
- ขอบเขตแพ็กเกจ เช่น `@example-org/*` ซึ่งควรเผยแพร่ภายใต้เจ้าของ
  ClawHub ที่ตรงกันเท่านั้น
- สลักของ Skills หรือชื่อแพ็กเกจ Plugin ที่ดูเหมือนแอบอ้างเป็นโครงการ
- ข้อพิพาทเกี่ยวกับแบรนด์ เครื่องหมายการค้า การเปลี่ยนชื่อโครงการ หรือประวัติแพ็กเกจ
- เจ้าของที่ถูกลบ ไม่ได้ใช้งาน หรือติดต่อไม่ได้ ซึ่งขัดขวางเจ้าของเนมสเปซ
  ที่มีสิทธิโดยชอบ

หากรายการดังกล่าวไม่ปลอดภัย เป็นอันตราย หรือทำให้เข้าใจผิดนอกเหนือจากข้อพิพาทด้านความเป็นเจ้าของ
ให้ปฏิบัติตามคำแนะนำด้านการกำกับดูแลหรือความปลอดภัยที่เกี่ยวข้องด้วย แบบฟอร์มคำร้องเกี่ยวกับเนมสเปซ
มีไว้สำหรับการตรวจสอบความเป็นเจ้าของ ไม่ใช่การเปิดเผยช่องโหว่ฉุกเฉิน

## ก่อนยื่นคำร้อง

ก่อนอื่น ให้ยืนยันว่าคุณกำลังเผยแพร่ด้วยเจ้าของที่ตรงกับเนมสเปซ
สำหรับแพ็กเกจ Plugin ชื่อที่มีขอบเขต เช่น `@example-org/example-plugin` ต้อง
เผยแพร่ในฐานะเจ้าของ `example-org` ที่ตรงกัน

หากคุณจัดการเจ้าของปัจจุบันได้ ให้แก้ไขเนมสเปซโดยตรงด้วยการเผยแพร่
เปลี่ยนชื่อ โอน ซ่อน หรือลบทรัพยากรที่ได้รับผลกระทบ ใช้คำร้อง
เมื่อคุณไม่สามารถจัดการเจ้าของปัจจุบันได้ หรือเมื่อเจ้าหน้าที่จำเป็นต้องระงับ
ข้อพิพาท

## หลักฐานที่ควรแนบ

ใช้หลักฐานสาธารณะที่ไม่มีข้อมูลละเอียดอ่อน หลักฐานที่เป็นประโยชน์ได้แก่:

- ประวัติองค์กร GitHub รีโพซิทอรี รุ่นเผยแพร่ หรือผู้ดูแล
- เอกสารทางการของโครงการที่ระบุชื่อเนมสเปซ
- หลักฐานจากโดเมนหรือโดเมนอีเมลทางการ
- การควบคุมขอบเขตใน npm, PyPI, crates.io หรือรีจิสทรีแพ็กเกจอื่น
- หลักฐานความเป็นเจ้าของเครื่องหมายการค้า แบรนด์ หรือโครงการที่สามารถอภิปราย
  ต่อสาธารณะได้อย่างปลอดภัย
- ประวัติรีโพซิทอรีต้นทาง ประวัติแพ็กเกจ หรือประกาศเปลี่ยนชื่อสาธารณะ
- ลิงก์ไปยังเจ้าของ Skills, Plugin, แพ็กเกจ หรือปัญหาบน ClawHub ที่มีข้อพิพาท

อธิบายว่าแต่ละลิงก์พิสูจน์อะไร เจ้าหน้าที่ควรสามารถเข้าใจ
ความสัมพันธ์ได้โดยไม่ต้องใช้ข้อมูลประจำตัวหรือข้อมูลลับส่วนตัว

## สิ่งที่ไม่ควรแนบ

อย่าใส่ข้อมูลลับหรือหลักฐานส่วนตัวในปัญหา GitHub สาธารณะ อย่าแนบ:

- โทเค็น API คีย์ลงนาม หรือข้อมูลประจำตัว
- โทเค็นการท้าทาย DNS
- เอกสารทางกฎหมายหรือสัญญาส่วนตัว
- เอกสารยืนยันตัวตนส่วนบุคคล
- อีเมลส่วนตัว รายงานความปลอดภัยส่วนตัว หรือข้อมูลลูกค้าที่เป็นความลับ

แบบฟอร์มคำร้องจะถามว่าหลักฐานละเอียดอ่อนจำเป็นต้องส่งผ่านช่องทางส่วนตัวของเจ้าหน้าที่หรือไม่
ให้ใช้ตัวเลือกนั้นแทนการโพสต์เนื้อหาละเอียดอ่อนต่อสาธารณะ

## ผลลัพธ์ที่อาจเกิดขึ้น

เจ้าหน้าที่ ClawHub อาจสงวนเนมสเปซ
โอนความเป็นเจ้าของ เปลี่ยนชื่อทรัพยากร ซ่อนหรือกักกันรายการที่มีอยู่
เพิ่มนามแฝงหรือการเปลี่ยนเส้นทาง ขอหลักฐานเพิ่มเติม หรือปฏิเสธคำขอ ทั้งนี้ขึ้นอยู่กับหลักฐานและความเสี่ยง

การตรวจสอบเนมสเปซไม่ได้รับประกันว่าจะมีการโอนชื่อที่ตรงกันทั้งหมด
เจ้าหน้าที่จะพิจารณาหลักฐานสาธารณะ การใช้งานที่มีอยู่ ความเสี่ยงด้านความปลอดภัย และผลกระทบต่อผู้ใช้
