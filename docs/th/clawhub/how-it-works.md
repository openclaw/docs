# ClawHub ทำงานอย่างไร

ClawHub คือชั้นรีจิสทรีสำหรับ Skills และ Plugin ของ OpenClaw โดยให้พื้นที่แก่ผู้ใช้ในการค้นพบแพ็กเกจ ให้พื้นที่แก่ผู้เผยแพร่ในการปล่อยเวอร์ชัน และให้เมทาดาทาแก่ OpenClaw เพียงพอสำหรับติดตั้งและอัปเดตแพ็กเกจเหล่านั้นอย่างปลอดภัย

## ระเบียนรีจิสทรี

รายการสาธารณะแต่ละรายการคือระเบียนรีจิสทรีที่มี:

- เจ้าของและ slug หรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่แล้วหนึ่งเวอร์ชันขึ้นไป
- เมทาดาทา สรุป ไฟล์ และการระบุแหล่งที่มา
- ข้อมูล changelog และแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง และการให้ดาว
- สถานะการสแกนความปลอดภัยและการตรวจสอบเนื้อหา

หน้ารายการคือที่อ้างอิงหลักสำหรับให้ผู้ใช้ตรวจสอบว่า Skills หรือ Plugin อ้างว่าสามารถทำอะไรได้บ้างก่อนติดตั้ง

## Skills

Skills คือชุดข้อความที่มีเวอร์ชัน โดยมีศูนย์กลางอยู่ที่ `SKILL.md` และสามารถรวมไฟล์สนับสนุน ตัวอย่าง เทมเพลต และสคริปต์ได้

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อ Skills คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และเมทาดาทา เมทาดาทาที่ถูกต้องมีความสำคัญเพราะช่วยให้ผู้ใช้ตัดสินใจว่าจะติดตั้ง Skills หรือไม่ และช่วยให้การสแกนอัตโนมัติตรวจพบความไม่ตรงกันระหว่างพฤติกรรมที่ประกาศไว้กับพฤติกรรมที่สังเกตได้

ดู [รูปแบบ Skills](/th/clawhub/skill-format)

## Plugin

Plugin คือส่วนขยาย OpenClaw ที่ถูกแพ็กเกจไว้ ClawHub จัดเก็บเมทาดาทาแพ็กเกจ ข้อมูลความเข้ากันได้ ลิงก์ซอร์ส อาร์ติแฟกต์ และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง Plugin จาก ClawHub ระบบจะตรวจสอบเมทาดาทาความเข้ากันได้ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจสามารถรวมความเข้ากันได้ของ API เวอร์ชัน Gateway ขั้นต่ำ เป้าหมายโฮสต์ ข้อกำหนดสภาพแวดล้อม และไดเจสต์ของอาร์ติแฟกต์ได้

ใช้แหล่งติดตั้ง ClawHub แบบระบุชัดเจนเมื่อคุณต้องการให้รีจิสทรีเป็นแหล่งข้อมูลอ้างอิงหลัก:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างบันทึกเวอร์ชันใหม่ที่แก้ไขไม่ได้ ผู้เผยแพร่ใช้ CLI `clawhub`
สำหรับเวิร์กโฟลว์รีจิสทรีที่ผ่านการยืนยันตัวตน:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้การทดลองรันเพื่อดูตัวอย่างเพย์โหลดที่ resolve แล้วก่อนอัปโหลด จากนั้นหน้าสาธารณะจะแสดง
เมตาดาต้าที่เผยแพร่ ไฟล์ การระบุแหล่งที่มา และสถานะการสแกน

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งที่มาของแพ็กเกจ:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึกเมตาดาต้าแหล่งที่มาของการติดตั้ง เพื่อให้การอัปเดตสามารถ resolve
แพ็กเกจรีจิสทรีเดียวกันได้ในภายหลัง CLI ของ ClawHub ยังรองรับเวิร์กโฟลว์ติดตั้งและ
อัปเดต skill โดยตรงสำหรับผู้ใช้ที่ต้องการให้โฟลเดอร์ skill อยู่ภายใต้การจัดการของรีจิสทรีนอก
เวิร์กสเปซ OpenClaw เต็มรูปแบบ

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่รีลีสยังคงอยู่ภายใต้เกตการอัปโหลด
การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้ดูแล

หน้าสาธารณะจะแสดงสรุปการสแกนเมื่อมี เนื้อหาที่ถูกระงับ ซ่อน
หรือบล็อกอาจหายไปจากการค้นหาสาธารณะและโฟลว์การติดตั้ง ขณะที่ยังคง
มองเห็นได้สำหรับเจ้าของเพื่อการวินิจฉัย

ดู [ความปลอดภัย](/clawhub/security), [การตรวจสอบความปลอดภัย](/clawhub/security-audits),
[การดูแลและความปลอดภัยของบัญชี](/th/clawhub/moderation), และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub เปิดเผย API อ่านแบบสาธารณะสำหรับการค้นพบ การค้นหา รายละเอียดแพ็กเกจ และ
การดาวน์โหลด แค็ตตาล็อกของบุคคลที่สามอาจใช้ API เหล่านี้ได้เมื่อเชื่อมโยงกลับไปยัง
รายการ ClawHub ที่เป็น canonical เคารพข้อจำกัดอัตราการใช้งาน และหลีกเลี่ยงการสื่อถึงการรับรอง

ดู [API สาธารณะ](/th/clawhub/api) และ [HTTP API](/clawhub/http-api)
