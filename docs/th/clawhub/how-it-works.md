# ClawHub ทำงานอย่างไร

ClawHub คือเลเยอร์ registry สำหรับ OpenClaw Skills และ Plugin โดยให้พื้นที่แก่ผู้ใช้เพื่อค้นพบแพ็กเกจ ให้พื้นที่แก่ผู้เผยแพร่เพื่อปล่อยเวอร์ชัน และให้ metadata แก่ OpenClaw อย่างเพียงพอสำหรับติดตั้งและอัปเดตแพ็กเกจเหล่านั้นอย่างปลอดภัย

## ระเบียน Registry

รายการสาธารณะแต่ละรายการคือระเบียน registry ที่มี:

- เจ้าของและ slug หรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่แล้วอย่างน้อยหนึ่งเวอร์ชัน
- metadata, สรุป, ไฟล์ และการระบุแหล่งที่มา
- changelog และข้อมูลแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง การติดดาว และความคิดเห็น
- สถานะการสแกนความปลอดภัยและการกลั่นกรอง

หน้ารายการคือที่ทางการสำหรับให้ผู้ใช้ตรวจสอบว่า Skills หรือ Plugin อ้างว่าจะทำอะไร ก่อนติดตั้ง

## Skills

Skills คือชุดข้อความแบบมีเวอร์ชันที่มีศูนย์กลางอยู่ที่ `SKILL.md` โดยอาจรวมไฟล์สนับสนุน ตัวอย่าง เทมเพลต และสคริปต์

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อ Skills คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และ metadata ความถูกต้องของ metadata สำคัญ เพราะช่วยให้ผู้ใช้ตัดสินใจว่าจะติดตั้ง Skills หรือไม่ และช่วยให้การสแกนอัตโนมัติตรวจจับความไม่ตรงกันระหว่างพฤติกรรมที่ประกาศกับพฤติกรรมที่สังเกตได้

ดู [รูปแบบ Skills](/th/clawhub/skill-format)

## Plugins

Plugin คือส่วนขยาย OpenClaw แบบแพ็กเกจ ClawHub จัดเก็บ metadata ของแพ็กเกจ ข้อมูลความเข้ากันได้ ลิงก์แหล่งที่มา artifacts และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง Plugin จาก ClawHub จะตรวจสอบ metadata ความเข้ากันได้ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจสามารถรวมความเข้ากันได้ของ API, เวอร์ชัน Gateway ขั้นต่ำ, เป้าหมายโฮสต์, ข้อกำหนดสภาพแวดล้อม และ digest ของ artifact

ใช้แหล่งติดตั้ง ClawHub ที่ระบุอย่างชัดเจนเมื่อคุณต้องการให้ registry เป็นแหล่งข้อมูลที่เชื่อถือได้:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างระเบียนเวอร์ชันใหม่ที่เปลี่ยนแปลงไม่ได้ ผู้เผยแพร่ใช้ CLI `clawhub` สำหรับเวิร์กโฟลว์ registry ที่ผ่านการยืนยันตัวตน:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้ dry run เพื่อดูตัวอย่าง payload ที่ resolve แล้วก่อนอัปโหลด จากนั้นหน้าสาธารณะจะแสดง metadata, ไฟล์, การระบุแหล่งที่มา และสถานะการสแกนที่เผยแพร่แล้ว

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งแพ็กเกจ:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึก metadata ของแหล่งติดตั้ง เพื่อให้การอัปเดตสามารถ resolve แพ็กเกจ registry เดิมได้ในภายหลัง ClawHub CLI ยังรองรับเวิร์กโฟลว์ติดตั้งและอัปเดต Skills โดยตรงสำหรับผู้ใช้ที่ต้องการโฟลเดอร์ Skills ที่จัดการโดย registry ภายนอกพื้นที่ทำงาน OpenClaw เต็มรูปแบบ

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่ release ยังคงอยู่ภายใต้ gate การอัปโหลด การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้ดูแล

หน้าสาธารณะแสดงสรุปการสแกนเมื่อมีให้ใช้งาน เนื้อหาที่ถูกพัก ซ่อน หรือบล็อกอาจหายไปจากการค้นหาสาธารณะและโฟลว์การติดตั้ง ขณะที่ยังคงมองเห็นได้สำหรับเจ้าของเพื่อการวินิจฉัย

ดู [ความปลอดภัย + การกลั่นกรอง](/th/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub เปิด public read API สำหรับการค้นพบ การค้นหา รายละเอียดแพ็กเกจ และการดาวน์โหลด แค็ตตาล็อกของบุคคลที่สามอาจใช้ API เหล่านี้ได้เมื่อเชื่อมโยงกลับไปยังรายการ ClawHub ทางการ เคารพ rate limit และหลีกเลี่ยงการสื่อว่ามีการรับรอง

ดู [Public API](/th/clawhub/api) และ [HTTP API](/th/clawhub/http-api)
