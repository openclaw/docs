# วิธีการทำงานของ ClawHub

ClawHub คือชั้นรีจิสทรีสำหรับ Skills และ Plugin ของ OpenClaw โดยให้ผู้ใช้มี
ที่สำหรับค้นพบแพ็กเกจ ให้ผู้เผยแพร่มีที่สำหรับปล่อยเวอร์ชัน และให้
OpenClaw มีเมทาดาทาเพียงพอสำหรับติดตั้งและอัปเดตแพ็กเกจเหล่านั้นอย่างปลอดภัย

## ระเบียนรีจิสทรี

รายการสาธารณะแต่ละรายการคือระเบียนรีจิสทรีที่มี:

- เจ้าของและ slug หรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่หนึ่งเวอร์ชันขึ้นไป
- เมทาดาทา สรุป ไฟล์ และการระบุแหล่งที่มา
- ข้อมูล changelog และแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง และดาว
- สถานะการสแกนความปลอดภัยและการดูแลเนื้อหา

หน้ารายการคือที่หลักอย่างเป็นทางการให้ผู้ใช้ตรวจสอบว่า Skills หรือ
Plugin อ้างว่าทำอะไรได้บ้างก่อนติดตั้ง

## Skills

Skills คือชุดข้อความที่มีเวอร์ชัน โดยมีศูนย์กลางอยู่ที่ `SKILL.md` และสามารถมี
ไฟล์สนับสนุน ตัวอย่าง เทมเพลต และสคริปต์ได้

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อ Skills,
คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และเมทาดาทา เมทาดาทาที่ถูกต้องมี
ความสำคัญ เพราะช่วยให้ผู้ใช้ตัดสินใจว่าจะติดตั้ง Skills หรือไม่ และช่วยให้
การสแกนอัตโนมัติตรวจพบความไม่ตรงกันระหว่างพฤติกรรมที่ประกาศไว้กับพฤติกรรมที่สังเกตได้

ดู [รูปแบบ Skill](/th/clawhub/skill-format)

## Plugins

Plugin คือส่วนขยาย OpenClaw ที่จัดแพ็กเกจแล้ว ClawHub เก็บเมทาดาทาของแพ็กเกจ
ข้อมูลความเข้ากันได้ ลิงก์แหล่งที่มา อาร์ติแฟกต์ และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง Plugin จาก ClawHub ระบบจะตรวจสอบเมทาดาทาความเข้ากันได้
ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจสามารถมีความเข้ากันได้ของ API,
เวอร์ชันขั้นต่ำของ Gateway, เป้าหมายโฮสต์, ข้อกำหนดสภาพแวดล้อม และไดเจสต์
ของอาร์ติแฟกต์

ใช้แหล่งติดตั้ง ClawHub ที่ระบุชัดเจนเมื่อคุณต้องการให้รีจิสทรีเป็น
แหล่งข้อมูลจริง:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างระเบียนเวอร์ชันใหม่ที่เปลี่ยนแปลงไม่ได้ ผู้เผยแพร่ใช้ CLI
`clawhub` สำหรับเวิร์กโฟลว์รีจิสทรีที่ยืนยันตัวตนแล้ว:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้การรันแบบทดลองเพื่อดูตัวอย่างเพย์โหลดที่แก้ไขค่าแล้วก่อนอัปโหลด จากนั้น
หน้าสาธารณะจะแสดงเมทาดาทาที่เผยแพร่ ไฟล์ การระบุแหล่งที่มา และสถานะการสแกน

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งแพ็กเกจ:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึกเมทาดาทาแหล่งติดตั้ง เพื่อให้การอัปเดตสามารถแก้ไปยัง
แพ็กเกจรีจิสทรีเดียวกันได้ในภายหลัง CLI ของ ClawHub ยังรองรับเวิร์กโฟลว์
ติดตั้งและอัปเดต Skills โดยตรงสำหรับผู้ใช้ที่ต้องการโฟลเดอร์ Skills ที่
จัดการโดยรีจิสทรีภายนอกพื้นที่ทำงาน OpenClaw แบบเต็ม

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่รุ่นที่ปล่อยยังอยู่ภายใต้ด่านอัปโหลด
การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้ดูแล

หน้าสาธารณะแสดงสรุปการสแกนเมื่อมีให้ใช้งาน เนื้อหาที่ถูกพักไว้ ซ่อน
หรือบล็อก อาจหายไปจากการค้นหาสาธารณะและโฟลว์การติดตั้ง ขณะที่ยังคง
มองเห็นได้สำหรับเจ้าของเพื่อการวินิจฉัย

ดู [ความปลอดภัย](/clawhub/security), [การตรวจสอบความปลอดภัย](/clawhub/security-audits),
[การดูแลเนื้อหาและความปลอดภัยของบัญชี](/th/clawhub/moderation) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub เปิดเผย API แบบอ่านสาธารณะสำหรับการค้นพบ การค้นหา รายละเอียดแพ็กเกจ
และการดาวน์โหลด แค็ตตาล็อกของบุคคลที่สามอาจใช้ API เหล่านี้ได้เมื่อเชื่อมโยง
กลับไปยังรายการ ClawHub อย่างเป็นทางการ เคารพขีดจำกัดอัตรา และหลีกเลี่ยง
การสื่อว่าได้รับการรับรอง

ดู [API สาธารณะ](/th/clawhub/api) และ [HTTP API](/clawhub/http-api)
