# การทำงานของ ClawHub

ClawHub คือเลเยอร์รีจิสทรีสำหรับทักษะและ Plugin ของ OpenClaw โดยมอบ
พื้นที่ให้ผู้ใช้ค้นพบแพ็กเกจ มอบพื้นที่ให้ผู้เผยแพร่ปล่อยเวอร์ชัน และ
มอบเมตาดาต้าให้ OpenClaw เพียงพอสำหรับติดตั้งและอัปเดตแพ็กเกจเหล่านั้นอย่างปลอดภัย

## ระเบียนรีจิสทรี

รายการสาธารณะแต่ละรายการคือระเบียนรีจิสทรีที่มี:

- เจ้าของและ slug หรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่แล้วหนึ่งเวอร์ชันหรือมากกว่า
- เมตาดาต้า สรุป ไฟล์ และการระบุแหล่งที่มา
- changelog และข้อมูลแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง การติดดาว และความคิดเห็น
- สถานะการสแกนความปลอดภัยและการดูแลกลั่นกรอง

หน้ารายการคือที่หลักสำหรับให้ผู้ใช้ตรวจสอบว่าทักษะหรือ
Plugin อ้างว่าทำอะไรได้บ้างก่อนติดตั้ง

## ทักษะ

ทักษะคือชุดข้อความแบบมีเวอร์ชันที่มี `SKILL.md` เป็นศูนย์กลาง โดยอาจมี
ไฟล์สนับสนุน ตัวอย่าง เทมเพลต และสคริปต์รวมอยู่ด้วย

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อทักษะ
คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และเมตาดาต้า เมตาดาต้าที่ถูกต้อง
มีความสำคัญ เพราะช่วยให้ผู้ใช้ตัดสินใจว่าจะติดตั้งทักษะหรือไม่ และ
ช่วยให้การสแกนอัตโนมัติตรวจพบความไม่ตรงกันระหว่างพฤติกรรมที่ประกาศกับพฤติกรรมที่สังเกตได้

ดู [รูปแบบทักษะ](/th/clawhub/skill-format)

## Plugin

Plugin คือส่วนขยาย OpenClaw ที่ถูกแพ็กเกจไว้ ClawHub จัดเก็บเมตาดาต้าแพ็กเกจ
ข้อมูลความเข้ากันได้ ลิงก์แหล่งที่มา artifact และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง Plugin จาก ClawHub ระบบจะตรวจสอบเมตาดาต้า
ความเข้ากันได้ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจสามารถมีความเข้ากันได้ของ API,
เวอร์ชัน Gateway ขั้นต่ำ, เป้าหมายโฮสต์, ข้อกำหนดสภาพแวดล้อม และ
digest ของ artifact

ใช้แหล่งติดตั้ง ClawHub แบบชัดเจนเมื่อคุณต้องการให้รีจิสทรีเป็น
แหล่งความจริง:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างระเบียนเวอร์ชันใหม่ที่เปลี่ยนแปลงไม่ได้ ผู้เผยแพร่ใช้ CLI `clawhub`
สำหรับเวิร์กโฟลว์รีจิสทรีที่ผ่านการยืนยันตัวตน:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้ dry run เพื่อดูตัวอย่าง payload ที่ resolve แล้วก่อนอัปโหลด จากนั้นหน้าสาธารณะจะ
แสดงเมตาดาต้าที่เผยแพร่ ไฟล์ การระบุแหล่งที่มา และสถานะการสแกน

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งแพ็กเกจ:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึกเมตาดาต้าแหล่งติดตั้ง เพื่อให้การอัปเดตสามารถ resolve
แพ็กเกจรีจิสทรีเดิมได้ในภายหลัง CLI ของ ClawHub ยังรองรับเวิร์กโฟลว์
ติดตั้งและอัปเดตทักษะโดยตรงสำหรับผู้ใช้ที่ต้องการโฟลเดอร์ทักษะแบบจัดการโดยรีจิสทรี
นอกเวิร์กสเปซ OpenClaw แบบเต็ม

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่ release ยังอยู่ภายใต้เกตการอัปโหลด
การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้ดูแล

หน้าสาธารณะแสดงสรุปการสแกนเมื่อมีให้ใช้งาน เนื้อหาที่ถูกพัก ซ่อน
หรือบล็อกอาจหายไปจากการค้นหาสาธารณะและโฟลว์การติดตั้ง ขณะที่ยังคง
มองเห็นได้สำหรับเจ้าของเพื่อการวินิจฉัย

ดู [ความปลอดภัย + การดูแลกลั่นกรอง](/th/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub เปิดเผย API แบบอ่านสาธารณะสำหรับการค้นพบ การค้นหา รายละเอียดแพ็กเกจ และ
การดาวน์โหลด แค็ตตาล็อกจากบุคคลที่สามอาจใช้ API เหล่านี้ได้เมื่อเชื่อมโยงกลับไปยัง
รายการ ClawHub หลัก เคารพ rate limit และหลีกเลี่ยงการสื่อว่าได้รับการรับรอง

ดู [API สาธารณะ](/th/clawhub/api) และ [HTTP API](/th/clawhub/http-api)
