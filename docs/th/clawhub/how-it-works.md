# ClawHub ทำงานอย่างไร

ClawHub เป็นชั้นรีจิสทรีสำหรับ Skills และ Plugin ของ OpenClaw โดยเป็นพื้นที่ให้ผู้ใช้
ค้นหาแพ็กเกจ ให้ผู้เผยแพร่เผยแพร่เวอร์ชัน และให้ข้อมูลเมตาแก่ OpenClaw อย่างเพียงพอ
เพื่อติดตั้งและอัปเดตแพ็กเกจเหล่านั้นได้อย่างปลอดภัย

## ระเบียนรีจิสทรี

รายการสาธารณะแต่ละรายการเป็นระเบียนรีจิสทรีที่ประกอบด้วย:

- เจ้าของและ slug หรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่แล้วอย่างน้อยหนึ่งเวอร์ชัน
- ข้อมูลเมตา สรุป ไฟล์ และการระบุแหล่งที่มา
- บันทึกการเปลี่ยนแปลงและข้อมูลแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง และการกดดาว
- สถานะการสแกนความปลอดภัยและการกลั่นกรอง

หน้ารายการเป็นแหล่งข้อมูลหลักที่ผู้ใช้ใช้ตรวจสอบว่า Skills หรือ
Plugin อ้างว่าสามารถทำอะไรได้บ้างก่อนติดตั้ง

## Skills

Skills คือชุดข้อความที่มีการกำหนดเวอร์ชันโดยมี `SKILL.md` เป็นศูนย์กลาง ซึ่งอาจมี
ไฟล์ประกอบ ตัวอย่าง เทมเพลต และสคริปต์รวมอยู่ด้วย

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อ Skills
คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และข้อมูลเมตา ข้อมูลเมตาที่ถูกต้อง
มีความสำคัญเพราะช่วยให้ผู้ใช้ตัดสินใจว่าจะติดตั้ง Skills หรือไม่ และ
ช่วยให้การสแกนอัตโนมัติตรวจพบความไม่สอดคล้องระหว่างพฤติกรรมที่ประกาศไว้กับพฤติกรรมที่สังเกตพบ

ดู[รูปแบบ Skills](/clawhub/skill-format)

## Plugin

Plugin คือส่วนขยาย OpenClaw ที่จัดทำเป็นแพ็กเกจ ClawHub จัดเก็บข้อมูลเมตาของแพ็กเกจ
ข้อมูลความเข้ากันได้ ลิงก์แหล่งที่มา อาร์ติแฟกต์ และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง Plugin จาก ClawHub ระบบจะตรวจสอบข้อมูลเมตา
ความเข้ากันได้ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจสามารถระบุความเข้ากันได้ของ API
เวอร์ชันขั้นต่ำของ Gateway โฮสต์เป้าหมาย ข้อกำหนดด้านสภาพแวดล้อม และไดเจสต์ของอาร์ติแฟกต์

ใช้แหล่งติดตั้ง ClawHub อย่างชัดเจนเมื่อต้องการให้รีจิสทรีเป็น
แหล่งข้อมูลจริงหลัก:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างระเบียนเวอร์ชันใหม่ที่แก้ไขไม่ได้ ผู้เผยแพร่ใช้ `clawhub`
CLI สำหรับเวิร์กโฟลว์รีจิสทรีที่ผ่านการยืนยันตัวตน:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้การทดลองรันเพื่อดูตัวอย่างเพย์โหลดที่ผ่านการรีโซลฟ์แล้วก่อนอัปโหลด จากนั้นหน้าสาธารณะจะ
แสดงข้อมูลเมตา ไฟล์ การระบุแหล่งที่มา และสถานะการสแกนที่เผยแพร่แล้ว

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งแพ็กเกจ:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึกข้อมูลเมตาของแหล่งติดตั้งเพื่อให้การอัปเดตสามารถรีโซลฟ์
แพ็กเกจรีจิสทรีเดิมได้ในภายหลัง CLI ของ ClawHub ยังรองรับเวิร์กโฟลว์การติดตั้งและ
อัปเดต Skills โดยตรง สำหรับผู้ใช้ที่ต้องการโฟลเดอร์ Skills ที่จัดการโดยรีจิสทรีนอก
เวิร์กสเปซ OpenClaw แบบเต็ม

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่รีลีสยังคงอยู่ภายใต้ด่านตรวจการอัปโหลด
การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้ดูแล

หน้าสาธารณะจะแสดงสรุปการสแกนเมื่อมีข้อมูล เนื้อหาที่ถูกพัก ซ่อน
หรือบล็อกอาจหายไปจากการค้นหาสาธารณะและขั้นตอนการติดตั้ง แต่ยังคง
ปรากฏแก่เจ้าของเพื่อการวินิจฉัย

ดู[ความปลอดภัย](/clawhub/security), [การตรวจสอบความปลอดภัย](/th/clawhub/security-audits),
[การกลั่นกรองและความปลอดภัยของบัญชี](/clawhub/moderation) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub มี API แบบอ่านสาธารณะสำหรับการค้นพบ การค้นหา รายละเอียดแพ็กเกจ และ
การดาวน์โหลด แค็ตตาล็อกของบุคคลที่สามอาจใช้ API เหล่านี้ได้เมื่อเชื่อมโยงกลับไปยัง
รายการ ClawHub หลัก เคารพขีดจำกัดอัตราการเรียกใช้ และหลีกเลี่ยงการสื่อเป็นนัยว่าได้รับการรับรอง

ดู[API สาธารณะ](/th/clawhub/api) และ [HTTP API](/clawhub/http-api)
