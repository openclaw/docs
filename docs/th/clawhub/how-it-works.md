# วิธีการทำงานของ ClawHub

ClawHub คือเลเยอร์รีจิสทรีสำหรับ Skills และ Plugin ของ OpenClaw โดยให้พื้นที่แก่ผู้ใช้เพื่อค้นพบแพ็กเกจ ให้พื้นที่แก่ผู้เผยแพร่เพื่อออกเวอร์ชัน และให้เมทาดาทาแก่ OpenClaw มากพอที่จะติดตั้งและอัปเดตแพ็กเกจเหล่านั้นได้อย่างปลอดภัย

## ระเบียนรีจิสทรี

รายการสาธารณะแต่ละรายการคือระเบียนรีจิสทรีที่มี:

- เจ้าของและ slug หรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่หนึ่งเวอร์ชันหรือมากกว่า
- เมทาดาทา สรุป ไฟล์ และการระบุแหล่งที่มา
- บันทึกการเปลี่ยนแปลงและข้อมูลแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง การติดดาว และความคิดเห็น
- สถานะการสแกนความปลอดภัยและการกลั่นกรอง

หน้ารายการคือที่อ้างอิงหลักสำหรับให้ผู้ใช้ตรวจสอบว่า Skill หรือ Plugin อ้างว่าจะทำอะไร ก่อนติดตั้ง

## Skills

Skill คือชุดข้อความแบบมีเวอร์ชันที่มีศูนย์กลางอยู่ที่ `SKILL.md` โดยสามารถรวมไฟล์สนับสนุน ตัวอย่าง เทมเพลต และสคริปต์ได้

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อ Skill คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และเมทาดาทา เมทาดาทาที่ถูกต้องมีความสำคัญ เพราะช่วยให้ผู้ใช้ตัดสินใจว่าจะติดตั้ง Skill หรือไม่ และช่วยให้การสแกนอัตโนมัติตรวจพบความไม่ตรงกันระหว่างพฤติกรรมที่ประกาศไว้กับพฤติกรรมที่สังเกตได้

ดู [รูปแบบ Skill](/th/clawhub/skill-format)

## Plugin

Plugin คือส่วนขยาย OpenClaw ที่ถูกแพ็กเกจไว้ ClawHub จัดเก็บเมทาดาทาแพ็กเกจ ข้อมูลความเข้ากันได้ ลิงก์แหล่งที่มา อาร์ติแฟกต์ และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง Plugin จาก ClawHub ระบบจะตรวจสอบเมทาดาทาความเข้ากันได้ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจสามารถรวมความเข้ากันได้ของ API เวอร์ชัน Gateway ขั้นต่ำ เป้าหมายโฮสต์ ข้อกำหนดสภาพแวดล้อม และ digest ของอาร์ติแฟกต์ได้

ใช้แหล่งติดตั้ง ClawHub แบบชัดเจนเมื่อคุณต้องการให้รีจิสทรีเป็นแหล่งความจริง:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างระเบียนเวอร์ชันใหม่ที่เปลี่ยนแปลงไม่ได้ ผู้เผยแพร่ใช้ CLI `clawhub` สำหรับเวิร์กโฟลว์รีจิสทรีที่ผ่านการยืนยันตัวตน:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้ dry run เพื่อดูตัวอย่าง payload ที่ resolve แล้วก่อนอัปโหลด จากนั้นหน้าสาธารณะจะแสดงเมทาดาทาที่เผยแพร่ ไฟล์ การระบุแหล่งที่มา และสถานะการสแกน

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งแพ็กเกจ:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึกเมทาดาทาแหล่งติดตั้ง เพื่อให้อัปเดตสามารถ resolve แพ็กเกจรีจิสทรีเดียวกันได้ในภายหลัง ClawHub CLI ยังรองรับเวิร์กโฟลว์ติดตั้งและอัปเดต Skill โดยตรงสำหรับผู้ใช้ที่ต้องการโฟลเดอร์ Skill ที่จัดการโดยรีจิสทรีภายนอกเวิร์กสเปซ OpenClaw แบบเต็ม

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่ release ยังคงอยู่ภายใต้เกตการอัปโหลด การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้กลั่นกรอง

หน้าสาธารณะจะแสดงสรุปการสแกนเมื่อมีให้ใช้งาน เนื้อหาที่ถูกพัก ซ่อน หรือบล็อก อาจหายไปจากการค้นหาสาธารณะและขั้นตอนการติดตั้ง แต่ยังคงมองเห็นได้สำหรับเจ้าของเพื่อการวินิจฉัย

ดู [ความปลอดภัย + การกลั่นกรอง](/th/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub เปิดเผย API แบบอ่านสาธารณะสำหรับการค้นพบ การค้นหา รายละเอียดแพ็กเกจ และการดาวน์โหลด แค็ตตาล็อกของบุคคลที่สามสามารถใช้ API เหล่านี้ได้เมื่อเชื่อมโยงกลับไปยังรายการ ClawHub ที่เป็นที่อ้างอิงหลัก เคารพ rate limit และหลีกเลี่ยงการสื่อว่าได้รับการรับรอง

ดู [API สาธารณะ](/th/clawhub/api) และ [HTTP API](/th/clawhub/http-api)
