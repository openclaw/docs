# วิธีการทำงานของ ClawHub

ClawHub คือเลเยอร์รีจิสทรีสำหรับ Skills และ Plugin ของ OpenClaw โดยให้พื้นที่แก่ผู้ใช้ในการค้นพบแพ็กเกจ ให้ผู้เผยแพร่มีที่สำหรับเผยแพร่เวอร์ชัน และให้ OpenClaw มีเมทาดาทาเพียงพอสำหรับติดตั้งและอัปเดตแพ็กเกจเหล่านั้นได้อย่างปลอดภัย

## ระเบียนรีจิสทรี

รายการสาธารณะแต่ละรายการเป็นระเบียนรีจิสทรีที่มี:

- เจ้าของและ slug หรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่แล้วหนึ่งเวอร์ชันหรือมากกว่า
- เมทาดาทา สรุป ไฟล์ และการระบุแหล่งที่มา
- ข้อมูลบันทึกการเปลี่ยนแปลงและแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง การติดดาว และความคิดเห็น
- สถานะการสแกนความปลอดภัยและการกลั่นกรอง

หน้ารายการคือที่มาตรฐานสำหรับให้ผู้ใช้ตรวจสอบว่า Skills หรือ Plugin อ้างว่าจะทำอะไร ก่อนติดตั้ง

## Skills

ทักษะคือชุดข้อความที่มีการกำหนดเวอร์ชัน โดยมี `SKILL.md` เป็นศูนย์กลาง อาจรวมไฟล์สนับสนุน ตัวอย่าง เทมเพลต และสคริปต์ไว้ด้วย

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อทักษะ คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และเมทาดาทา เมทาดาทาที่ถูกต้องมีความสำคัญ เพราะช่วยให้ผู้ใช้ตัดสินใจว่าจะติดตั้งทักษะหรือไม่ และช่วยให้การสแกนอัตโนมัติตรวจจับความไม่ตรงกันระหว่างพฤติกรรมที่ประกาศไว้กับพฤติกรรมที่สังเกตได้

ดู [รูปแบบทักษะ](/th/clawhub/skill-format)

## Plugins

Plugin คือส่วนขยายของ OpenClaw ที่ถูกแพ็กเกจไว้ ClawHub จัดเก็บเมทาดาทาของแพ็กเกจ ข้อมูลความเข้ากันได้ ลิงก์แหล่งที่มา อาร์ติแฟกต์ และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง Plugin จาก ClawHub ระบบจะตรวจสอบเมทาดาทาความเข้ากันได้ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจอาจรวมความเข้ากันได้ของ API เวอร์ชัน Gateway ขั้นต่ำ เป้าหมายโฮสต์ ข้อกำหนดสภาพแวดล้อม และ digest ของอาร์ติแฟกต์

ใช้แหล่งติดตั้ง ClawHub แบบชัดเจนเมื่อคุณต้องการให้รีจิสทรีเป็นแหล่งความจริง:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างระเบียนเวอร์ชันใหม่ที่เปลี่ยนแปลงไม่ได้ ผู้เผยแพร่ใช้ CLI `clawhub` สำหรับเวิร์กโฟลว์รีจิสทรีที่ผ่านการยืนยันตัวตน:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้ dry run เพื่อดูตัวอย่าง payload ที่ resolve แล้วก่อนอัปโหลด จากนั้นหน้าสาธารณะจะแสดงเมทาดาทา ไฟล์ การระบุแหล่งที่มา และสถานะการสแกนที่เผยแพร่แล้ว

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งแพ็กเกจ:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึกเมทาดาทาแหล่งติดตั้ง เพื่อให้การอัปเดตสามารถ resolve แพ็กเกจรีจิสทรีเดิมได้ในภายหลัง CLI ของ ClawHub ยังรองรับเวิร์กโฟลว์การติดตั้งและอัปเดตทักษะโดยตรง สำหรับผู้ใช้ที่ต้องการโฟลเดอร์ทักษะที่จัดการโดยรีจิสทรีภายนอกเวิร์กสเปซ OpenClaw แบบเต็ม

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่การเผยแพร่ยังคงอยู่ภายใต้เกตการอัปโหลด การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้กลั่นกรอง

หน้าสาธารณะแสดงสรุปการสแกนเมื่อมีให้ใช้งาน เนื้อหาที่ถูกพักไว้ ซ่อนไว้ หรือบล็อกไว้อาจหายไปจากการค้นหาสาธารณะและโฟลว์การติดตั้ง ขณะที่ยังคงมองเห็นได้สำหรับเจ้าของเพื่อการวินิจฉัยหรือการอุทธรณ์

ดู [ความปลอดภัย + การกลั่นกรอง](/th/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub เปิดเผย API แบบอ่านสาธารณะสำหรับการค้นพบ การค้นหา รายละเอียดแพ็กเกจ และการดาวน์โหลด แค็ตตาล็อกของบุคคลที่สามอาจใช้ API เหล่านี้ได้ เมื่อมีการลิงก์กลับไปยังรายการ ClawHub มาตรฐาน เคารพขีดจำกัดอัตราการใช้งาน และหลีกเลี่ยงการสื่อว่าได้รับการรับรอง

ดู [API สาธารณะ](/th/clawhub/api) และ [HTTP API](/th/clawhub/http-api)
