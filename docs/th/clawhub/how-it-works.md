# การทำงานของ ClawHub

ClawHub คือชั้นรีจิสทรีสำหรับ Skills และ Plugin ของ OpenClaw ซึ่งให้พื้นที่แก่ผู้ใช้ในการค้นหาแพ็กเกจ ให้พื้นที่แก่ผู้เผยแพร่ในการปล่อยเวอร์ชัน และให้เมตาดาต้าแก่ OpenClaw เพียงพอสำหรับติดตั้งและอัปเดตแพ็กเกจเหล่านั้นอย่างปลอดภัย

## ระเบียนรีจิสทรี

รายการสาธารณะแต่ละรายการคือระเบียนรีจิสทรีที่มี:

- เจ้าของและ slug หรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่แล้วหนึ่งรายการหรือมากกว่า
- เมตาดาต้า สรุป ไฟล์ และการระบุแหล่งที่มา
- ข้อมูลบันทึกการเปลี่ยนแปลงและแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง และดาว
- สถานะการสแกนความปลอดภัยและการกลั่นกรอง

หน้ารายการคือที่ทางการสำหรับให้ผู้ใช้ตรวจสอบว่าสกิลหรือ Plugin อ้างว่าทำอะไรได้ก่อนติดตั้ง

## Skills

สกิลคือชุดข้อความแบบมีเวอร์ชันที่มี `SKILL.md` เป็นศูนย์กลาง โดยสามารถรวมไฟล์สนับสนุน ตัวอย่าง เทมเพลต และสคริปต์ได้

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อสกิล คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และเมตาดาต้า เมตาดาต้าที่ถูกต้องมีความสำคัญ เพราะช่วยให้ผู้ใช้ตัดสินใจว่าจะติดตั้งสกิลหรือไม่ และช่วยให้การสแกนอัตโนมัติตรวจจับความไม่ตรงกันระหว่างพฤติกรรมที่ประกาศไว้กับพฤติกรรมที่สังเกตได้

ดู [รูปแบบสกิล](/th/clawhub/skill-format)

## Plugin

Plugin คือส่วนขยาย OpenClaw ที่ถูกจัดแพ็กเกจ ClawHub จัดเก็บเมตาดาต้าของแพ็กเกจ ข้อมูลความเข้ากันได้ ลิงก์แหล่งที่มา อาร์ติแฟกต์ และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง Plugin จาก ClawHub ระบบจะตรวจสอบเมตาดาต้าความเข้ากันได้ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจสามารถรวมความเข้ากันได้ของ API, เวอร์ชัน Gateway ขั้นต่ำ, เป้าหมายโฮสต์, ข้อกำหนดสภาพแวดล้อม และไดเจสต์ของอาร์ติแฟกต์ได้

ใช้แหล่งติดตั้ง ClawHub แบบชัดเจนเมื่อคุณต้องการให้รีจิสทรีเป็นแหล่งข้อมูลจริง:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างระเบียนเวอร์ชันใหม่ที่เปลี่ยนแปลงไม่ได้ ผู้เผยแพร่ใช้ CLI `clawhub` สำหรับเวิร์กโฟลว์รีจิสทรีที่ยืนยันตัวตนแล้ว:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้การทดลองรันเพื่อดูตัวอย่างเพย์โหลดที่ resolve แล้วก่อนอัปโหลด จากนั้นหน้าสาธารณะจะแสดงเมตาดาต้าที่เผยแพร่ ไฟล์ การระบุแหล่งที่มา และสถานะการสแกน

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งแพ็กเกจ:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึกเมตาดาต้าแหล่งติดตั้งเพื่อให้อัปเดตสามารถ resolve แพ็กเกจรีจิสทรีเดิมได้ในภายหลัง CLI ของ ClawHub ยังรองรับเวิร์กโฟลว์การติดตั้งและอัปเดตสกิลโดยตรงสำหรับผู้ใช้ที่ต้องการโฟลเดอร์สกิลที่จัดการโดยรีจิสทรีภายนอกเวิร์กสเปซ OpenClaw เต็มรูปแบบ

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่รีลีสยังอยู่ภายใต้ด่านอัปโหลด การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้ดูแล

หน้าสาธารณะจะแสดงสรุปการสแกนเมื่อมี เนื้อหาที่ถูกพัก ซ่อน หรือบล็อกอาจหายไปจากการค้นหาสาธารณะและโฟลว์การติดตั้ง ขณะที่ยังคงมองเห็นได้สำหรับเจ้าของเพื่อการวินิจฉัย

ดู [ความปลอดภัย](/th/clawhub/security), [การตรวจสอบความปลอดภัย](/th/clawhub/security-audits), [การกลั่นกรองและความปลอดภัยของบัญชี](/th/clawhub/moderation) และ [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub เปิดเผย API อ่านสาธารณะสำหรับการค้นพบ การค้นหา รายละเอียดแพ็กเกจ และการดาวน์โหลด แค็ตตาล็อกของบุคคลที่สามสามารถใช้ API เหล่านี้ได้เมื่อเชื่อมกลับไปยังรายการ ClawHub ทางการ เคารพขีดจำกัดอัตรา และหลีกเลี่ยงการสื่อว่าได้รับการรับรอง

ดู [API สาธารณะ](/th/clawhub/api) และ [HTTP API](/th/clawhub/http-api)
