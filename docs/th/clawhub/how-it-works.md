# ClawHub ทำงานอย่างไร

ClawHub คือเลเยอร์รีจิสทรีสำหรับ Skills และ plugins ของ OpenClaw โดยมอบ
พื้นที่ให้ผู้ใช้ค้นพบแพ็กเกจ ให้ผู้เผยแพร่เผยแพร่เวอร์ชัน และให้
OpenClaw มีเมทาดาทาเพียงพอสำหรับติดตั้งและอัปเดตแพ็กเกจเหล่านั้นได้อย่างปลอดภัย

## ระเบียนรีจิสทรี

รายการสาธารณะแต่ละรายการคือระเบียนรีจิสทรีที่มี:

- เจ้าของและ slug หรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่แล้วอย่างน้อยหนึ่งเวอร์ชัน
- เมทาดาทา สรุป ไฟล์ และการระบุแหล่งที่มา
- changelog และข้อมูลแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง และดาว
- สถานะการสแกนความปลอดภัยและการกลั่นกรอง

หน้ารายการคือที่มาตรฐานให้ผู้ใช้ตรวจสอบว่า skill หรือ
plugin อ้างว่าทำอะไรได้บ้างก่อนติดตั้ง

## Skills

skill คือบันเดิลข้อความแบบมีเวอร์ชันที่มีศูนย์กลางอยู่ที่ `SKILL.md` สามารถมี
ไฟล์สนับสนุน ตัวอย่าง เทมเพลต และสคริปต์ได้

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อ skill,
คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และเมทาดาทา เมทาดาทาที่ถูกต้อง
สำคัญเพราะช่วยให้ผู้ใช้ตัดสินใจว่าจะติดตั้ง skill หรือไม่ และ
ช่วยให้การสแกนอัตโนมัติตรวจพบความไม่ตรงกันระหว่างพฤติกรรมที่ประกาศไว้กับที่สังเกตพบ

ดู [รูปแบบ Skill](/th/clawhub/skill-format)

## Plugins

Plugins คือส่วนขยาย OpenClaw ที่จัดเป็นแพ็กเกจ ClawHub จัดเก็บเมทาดาทาแพ็กเกจ
ข้อมูลความเข้ากันได้ ลิงก์แหล่งที่มา อาร์ติแฟกต์ และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง plugin จาก ClawHub ระบบจะตรวจสอบเมทาดาทา
ความเข้ากันได้ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจสามารถมีความเข้ากันได้ของ API,
เวอร์ชัน Gateway ขั้นต่ำ เป้าหมายโฮสต์ ข้อกำหนดสภาพแวดล้อม และไดเจสต์
ของอาร์ติแฟกต์

ใช้แหล่งติดตั้ง ClawHub ที่ระบุอย่างชัดเจนเมื่อคุณต้องการให้รีจิสทรีเป็น
แหล่งข้อมูลที่เชื่อถือได้:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างระเบียนเวอร์ชันใหม่ที่เปลี่ยนแปลงไม่ได้ ผู้เผยแพร่ใช้ CLI `clawhub`
สำหรับเวิร์กโฟลว์รีจิสทรีที่ผ่านการยืนยันตัวตน:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้ dry run เพื่อดูตัวอย่างเพย์โหลดที่ resolve แล้วก่อนอัปโหลด จากนั้นหน้าสาธารณะจะ
แสดงเมทาดาทาที่เผยแพร่ ไฟล์ การระบุแหล่งที่มา และสถานะการสแกน

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งแพ็กเกจ:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึกเมทาดาทาแหล่งติดตั้งเพื่อให้การอัปเดตสามารถ resolve แพ็กเกจ
รีจิสทรีเดิมได้ในภายหลัง CLI ของ ClawHub ยังรองรับเวิร์กโฟลว์ติดตั้งและ
อัปเดต skill โดยตรงสำหรับผู้ใช้ที่ต้องการโฟลเดอร์ skill ที่จัดการโดยรีจิสทรีภายนอก
เวิร์กสเปซ OpenClaw แบบเต็ม

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่รีลีสยังคงอยู่ภายใต้ด่านอัปโหลด
การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้ดูแล

หน้าสาธารณะแสดงสรุปการสแกนเมื่อมี เนื้อหาที่ถูกพัก ซ่อน
หรือบล็อกอาจหายไปจากการค้นหาสาธารณะและโฟลว์การติดตั้ง ขณะยังคง
มองเห็นได้สำหรับเจ้าของเพื่อการวินิจฉัย

ดู [ความปลอดภัย](/clawhub/security), [การตรวจสอบความปลอดภัย](/clawhub/security-audits),
[การกลั่นกรองและความปลอดภัยของบัญชี](/th/clawhub/moderation), และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub เปิดเผย API แบบอ่านสาธารณะสำหรับการค้นพบ การค้นหา รายละเอียดแพ็กเกจ และ
การดาวน์โหลด แค็ตตาล็อกของบุคคลที่สามอาจใช้ API เหล่านี้ได้เมื่อมีลิงก์กลับไปยัง
รายการ ClawHub ที่เป็นมาตรฐาน เคารพขีดจำกัดอัตราการใช้งาน และหลีกเลี่ยงการสื่อว่าได้รับการรับรอง

ดู [API สาธารณะ](/th/clawhub/api) และ [HTTP API](/clawhub/http-api)
