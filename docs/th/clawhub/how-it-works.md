# วิธีการทำงานของ ClawHub

ClawHub คือเลเยอร์รีจิสทรีสำหรับ Skills และ Plugin ของ OpenClaw ซึ่งให้พื้นที่แก่ผู้ใช้ในการค้นหาแพ็กเกจ ให้พื้นที่แก่ผู้เผยแพร่ในการออกเวอร์ชัน และให้เมทาดาทาแก่ OpenClaw เพียงพอสำหรับติดตั้งและอัปเดตแพ็กเกจเหล่านั้นอย่างปลอดภัย

## ระเบียนรีจิสทรี

รายการสาธารณะแต่ละรายการคือระเบียนรีจิสทรีที่มี:

- เจ้าของและ slug หรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่แล้วตั้งแต่หนึ่งเวอร์ชันขึ้นไป
- เมทาดาทา สรุป ไฟล์ และการระบุแหล่งที่มา
- ข้อมูล changelog และแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง และดาว
- สถานะการสแกนความปลอดภัยและการกลั่นกรอง

หน้ารายการคือที่อ้างอิงหลักสำหรับให้ผู้ใช้ตรวจสอบว่า Skills หรือ Plugin ระบุว่าจะทำอะไรก่อนติดตั้ง

## Skills

Skills คือบันเดิลข้อความแบบมีเวอร์ชันที่มีศูนย์กลางอยู่ที่ `SKILL.md` ซึ่งอาจมีไฟล์สนับสนุน ตัวอย่าง เทมเพลต และสคริปต์รวมอยู่ด้วย

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อ Skills คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และเมทาดาทา เมทาดาทาที่ถูกต้องมีความสำคัญ เพราะช่วยให้ผู้ใช้ตัดสินใจว่าจะติดตั้ง Skills หรือไม่ และช่วยให้การสแกนอัตโนมัติตรวจพบความไม่ตรงกันระหว่างพฤติกรรมที่ประกาศไว้กับพฤติกรรมที่สังเกตได้

ดู [รูปแบบ Skills](/th/clawhub/skill-format)

## Plugins

Plugin คือส่วนขยาย OpenClaw ที่ถูกจัดแพ็กเกจไว้ ClawHub จัดเก็บเมทาดาทาแพ็กเกจ ข้อมูลความเข้ากันได้ ลิงก์ซอร์ส artifact และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง Plugin จาก ClawHub ระบบจะตรวจสอบเมทาดาทาความเข้ากันได้ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจอาจมีความเข้ากันได้ของ API, เวอร์ชัน Gateway ขั้นต่ำ, เป้าหมายโฮสต์, ข้อกำหนดสภาพแวดล้อม และ digest ของ artifact

ใช้แหล่งติดตั้ง ClawHub ที่ระบุอย่างชัดเจนเมื่อต้องการให้รีจิสทรีเป็นแหล่งข้อมูลจริง:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างระเบียนเวอร์ชันใหม่ที่เปลี่ยนแปลงไม่ได้ ผู้เผยแพร่ใช้ CLI `clawhub` สำหรับเวิร์กโฟลว์รีจิสทรีที่ผ่านการยืนยันตัวตน:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้การรันแบบทดลองเพื่อดูตัวอย่าง payload ที่ถูกแก้ไขแล้วก่อนอัปโหลด จากนั้นหน้าสาธารณะจะแสดงเมทาดาทา ไฟล์ การระบุแหล่งที่มา และสถานะการสแกนที่เผยแพร่แล้ว

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งแพ็กเกจ:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึกเมทาดาทาแหล่งติดตั้งเพื่อให้การอัปเดตสามารถแก้ไขไปยังแพ็กเกจรีจิสทรีเดียวกันได้ในภายหลัง CLI ของ ClawHub ยังรองรับเวิร์กโฟลว์การติดตั้งและอัปเดต Skills โดยตรงสำหรับผู้ใช้ที่ต้องการโฟลเดอร์ Skills ที่จัดการโดยรีจิสทรีนอกเวิร์กสเปซ OpenClaw แบบเต็ม

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่รีลีสยังคงอยู่ภายใต้ด่านการอัปโหลด การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้ดูแล

หน้าสาธารณะจะแสดงสรุปการสแกนเมื่อมี เนื้อหาที่ถูกพัก ซ่อน หรือบล็อกอาจหายไปจากการค้นหาสาธารณะและโฟลว์การติดตั้ง ขณะที่ยังคงมองเห็นได้สำหรับเจ้าของเพื่อการวินิจฉัย

ดู [ความปลอดภัย](/clawhub/security), [การตรวจสอบความปลอดภัย](/clawhub/security-audits), [การกลั่นกรองและความปลอดภัยของบัญชี](/th/clawhub/moderation) และ [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub เปิดเผย API อ่านแบบสาธารณะสำหรับการค้นพบ การค้นหา รายละเอียดแพ็กเกจ และการดาวน์โหลด แคตตาล็อกของบุคคลที่สามอาจใช้ API เหล่านี้ได้เมื่อเชื่อมโยงกลับไปยังรายการ ClawHub ที่เป็นที่อ้างอิงหลัก เคารพขีดจำกัดอัตราการใช้งาน และหลีกเลี่ยงการสื่อว่าได้รับการรับรอง

ดู [API สาธารณะ](/th/clawhub/api) และ [HTTP API](/clawhub/http-api)
