# ClawHub ทำงานอย่างไร

ClawHub คือเลเยอร์รีจิสทรีสำหรับ Skills และ Plugin ของ OpenClaw โดยให้พื้นที่แก่ผู้ใช้ในการค้นหาแพ็กเกจ ให้ผู้เผยแพร่มีพื้นที่สำหรับออกเวอร์ชัน และให้ OpenClaw มีเมตาดาต้าเพียงพอในการติดตั้งและอัปเดตแพ็กเกจเหล่านั้นอย่างปลอดภัย

## ระเบียนรีจิสทรี

รายการสาธารณะแต่ละรายการคือระเบียนรีจิสทรีที่มี:

- เจ้าของและ slug หรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่แล้วหนึ่งเวอร์ชันขึ้นไป
- เมตาดาต้า สรุป ไฟล์ และการระบุแหล่งที่มา
- changelog และข้อมูลแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง การติดดาว และความคิดเห็น
- สถานะการสแกนความปลอดภัยและการดูแลเนื้อหา

หน้ารายการคือที่อ้างอิงหลักให้ผู้ใช้ตรวจสอบว่า Skills หรือ Plugin อ้างว่าสามารถทำอะไรได้บ้างก่อนติดตั้ง

## Skills

Skill คือชุดข้อความแบบมีเวอร์ชันที่มีศูนย์กลางอยู่ที่ `SKILL.md` โดยสามารถรวมไฟล์สนับสนุน ตัวอย่าง เทมเพลต และสคริปต์ได้

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อ Skills คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และเมตาดาต้า เมตาดาต้าที่ถูกต้องมีความสำคัญ เพราะช่วยให้ผู้ใช้ตัดสินใจว่าจะติดตั้ง Skills หรือไม่ และช่วยให้การสแกนอัตโนมัติตรวจพบความไม่ตรงกันระหว่างพฤติกรรมที่ประกาศไว้กับพฤติกรรมที่สังเกตได้

ดู [รูปแบบ Skills](/th/clawhub/skill-format)

## Plugin

Plugin คือส่วนขยายของ OpenClaw ที่ถูกจัดแพ็กเกจไว้ ClawHub จัดเก็บเมตาดาต้าแพ็กเกจ ข้อมูลความเข้ากันได้ ลิงก์แหล่งที่มา อาร์ติแฟกต์ และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง Plugin จาก ClawHub ระบบจะตรวจสอบเมตาดาต้าความเข้ากันได้ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจสามารถรวมความเข้ากันได้ของ API เวอร์ชันขั้นต่ำของ Gateway เป้าหมายโฮสต์ ข้อกำหนดของสภาพแวดล้อม และ digest ของอาร์ติแฟกต์

ใช้แหล่งติดตั้ง ClawHub แบบระบุชัดเจนเมื่อคุณต้องการให้รีจิสทรีเป็นแหล่งข้อมูลจริง:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างระเบียนเวอร์ชันใหม่ที่เปลี่ยนแปลงไม่ได้ ผู้เผยแพร่ใช้ CLI `clawhub` สำหรับเวิร์กโฟลว์รีจิสทรีที่ผ่านการยืนยันตัวตน:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้ dry run เพื่อดูตัวอย่าง payload ที่ resolve แล้วก่อนอัปโหลด จากนั้นหน้าสาธารณะจะแสดงเมตาดาต้าที่เผยแพร่ ไฟล์ การระบุแหล่งที่มา และสถานะการสแกน

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งแพ็กเกจ:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึกเมตาดาต้าแหล่งติดตั้งเพื่อให้อัปเดตสามารถ resolve แพ็กเกจรีจิสทรีเดียวกันได้ในภายหลัง CLI ของ ClawHub ยังรองรับเวิร์กโฟลว์การติดตั้งและอัปเดต Skills โดยตรงสำหรับผู้ใช้ที่ต้องการโฟลเดอร์ Skills ที่จัดการโดยรีจิสทรีนอกพื้นที่ทำงาน OpenClaw แบบเต็ม

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่ release ยังคงอยู่ภายใต้ gate การอัปโหลด การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้ดูแล

หน้าสาธารณะจะแสดงสรุปการสแกนเมื่อมี เนื้อหาที่ถูกพัก ซ่อน หรือบล็อกอาจหายไปจากการค้นหาสาธารณะและโฟลว์การติดตั้ง ขณะที่ยังคงมองเห็นได้สำหรับเจ้าของเพื่อการวินิจฉัยหรืออุทธรณ์

ดู [ความปลอดภัย + การดูแลเนื้อหา](/th/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub เปิดเผย API สำหรับอ่านแบบสาธารณะสำหรับการค้นพบ การค้นหา รายละเอียดแพ็กเกจ และการดาวน์โหลด แคตตาล็อกของบุคคลที่สามสามารถใช้ API เหล่านี้ได้เมื่อมีลิงก์กลับไปยังรายการ ClawHub ที่เป็นที่อ้างอิงหลัก เคารพ rate limit และหลีกเลี่ยงการสื่อว่ามีการรับรอง

ดู [API สาธารณะ](/th/clawhub/api) และ [HTTP API](/th/clawhub/http-api)
