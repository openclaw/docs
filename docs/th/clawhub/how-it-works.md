# วิธีการทำงานของ ClawHub

ClawHub คือเลเยอร์ registry สำหรับ Skills และ Plugin ของ OpenClaw โดยให้ผู้ใช้มี
ที่สำหรับค้นหาแพ็กเกจ ให้ผู้เผยแพร่มีที่สำหรับปล่อยเวอร์ชัน และให้
OpenClaw มีเมตาดาต้าเพียงพอสำหรับติดตั้งและอัปเดตแพ็กเกจเหล่านั้นอย่างปลอดภัย

## ระเบียน registry

รายการสาธารณะแต่ละรายการคือระเบียน registry ที่มี:

- เจ้าของและ slug หรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่หนึ่งเวอร์ชันขึ้นไป
- เมตาดาต้า สรุป ไฟล์ และการระบุแหล่งที่มา
- ข้อมูล changelog และแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง และดาว
- สถานะการสแกนความปลอดภัยและการกลั่นกรอง

หน้ารายการคือที่มาตรฐานที่ผู้ใช้ใช้ตรวจสอบว่า Skills หรือ
Plugin อ้างว่าทำอะไรได้บ้างก่อนติดตั้ง

## Skills

Skills คือบันเดิลข้อความแบบมีเวอร์ชันที่มี `SKILL.md` เป็นศูนย์กลาง โดยอาจรวม
ไฟล์สนับสนุน ตัวอย่าง เทมเพลต และสคริปต์ไว้ด้วย

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อ Skills,
คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และเมตาดาต้า เมตาดาต้าที่ถูกต้อง
มีความสำคัญเพราะช่วยให้ผู้ใช้ตัดสินใจว่าจะติดตั้ง Skills หรือไม่ และ
ช่วยให้การสแกนอัตโนมัติตรวจพบความไม่ตรงกันระหว่างพฤติกรรมที่ประกาศไว้กับพฤติกรรมที่สังเกตได้

ดู [รูปแบบ Skills](/th/clawhub/skill-format)

## Plugin

Plugin คือส่วนขยาย OpenClaw ที่ถูกแพ็กเกจไว้ ClawHub จัดเก็บเมตาดาต้าแพ็กเกจ
ข้อมูลความเข้ากันได้ ลิงก์แหล่งที่มา อาร์ติแฟกต์ และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง Plugin จาก ClawHub ระบบจะตรวจสอบเมตาดาต้าความเข้ากันได้
ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจอาจมีความเข้ากันได้ของ API,
เวอร์ชัน Gateway ขั้นต่ำ เป้าหมายโฮสต์ ข้อกำหนดสภาพแวดล้อม และ digest
ของอาร์ติแฟกต์

ใช้แหล่งติดตั้ง ClawHub ที่ระบุชัดเจนเมื่อคุณต้องการให้ registry เป็น
แหล่งข้อมูลที่เชื่อถือได้:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างระเบียนเวอร์ชันใหม่ที่เปลี่ยนแปลงไม่ได้ ผู้เผยแพร่ใช้ CLI `clawhub`
สำหรับเวิร์กโฟลว์ registry ที่ผ่านการยืนยันตัวตน:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้ dry run เพื่อดูตัวอย่าง payload ที่ resolve แล้วก่อนอัปโหลด จากนั้นหน้าสาธารณะจะ
แสดงเมตาดาต้าที่เผยแพร่ ไฟล์ การระบุแหล่งที่มา และสถานะการสแกน

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งแพ็กเกจ:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึกเมตาดาต้าแหล่งติดตั้งเพื่อให้การอัปเดตสามารถ resolve แพ็กเกจ
registry เดิมได้ในภายหลัง CLI ของ ClawHub ยังรองรับเวิร์กโฟลว์ติดตั้งและ
อัปเดต Skills โดยตรงสำหรับผู้ใช้ที่ต้องการโฟลเดอร์ Skills ที่จัดการโดย registry
ภายนอก workspace OpenClaw แบบเต็ม

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่ release ยังอยู่ภายใต้เกตการอัปโหลด
การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้ดูแล

หน้าสาธารณะแสดงสรุปการสแกนเมื่อมีให้ใช้งาน เนื้อหาที่ถูกระงับ ซ่อน
หรือบล็อกอาจหายไปจากการค้นหาสาธารณะและโฟลว์การติดตั้ง ขณะที่ยังคง
มองเห็นได้สำหรับเจ้าของเพื่อใช้วินิจฉัยปัญหา

ดู [ความปลอดภัย](/clawhub/security), [การตรวจสอบความปลอดภัย](/clawhub/security-audits),
[การกลั่นกรองและความปลอดภัยของบัญชี](/th/clawhub/moderation) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub เปิดเผย API แบบอ่านสาธารณะสำหรับการค้นพบ การค้นหา รายละเอียดแพ็กเกจ และ
การดาวน์โหลด แค็ตตาล็อกของบุคคลที่สามอาจใช้ API เหล่านี้เมื่อเชื่อมโยงกลับไปยัง
รายการ ClawHub มาตรฐาน เคารพขีดจำกัดอัตราการเรียกใช้ และหลีกเลี่ยงการสื่อว่าได้รับการรับรอง

ดู [API สาธารณะ](/th/clawhub/api) และ [HTTP API](/clawhub/http-api)
