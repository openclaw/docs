# ClawHub ทำงานอย่างไร

ClawHub คือชั้นรีจิสทรีสำหรับ Skills และ Plugin ของ OpenClaw โดยมอบ
พื้นที่ให้ผู้ใช้ค้นพบแพ็กเกจ มอบพื้นที่ให้ผู้เผยแพร่ปล่อยเวอร์ชัน และ
มอบเมตาดาต้าที่เพียงพอให้ OpenClaw เพื่อติดตั้งและอัปเดตแพ็กเกจเหล่านั้นอย่างปลอดภัย

## ระเบียนรีจิสทรี

รายการสาธารณะแต่ละรายการคือระเบียนรีจิสทรีที่มี:

- เจ้าของและ slug หรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่หนึ่งเวอร์ชันหรือมากกว่า
- เมตาดาต้า สรุป ไฟล์ และการระบุแหล่งที่มา
- changelog และข้อมูลแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง และการติดดาว
- สถานะการสแกนความปลอดภัยและการกลั่นกรอง

หน้ารายการคือที่มาตรฐานสำหรับให้ผู้ใช้ตรวจสอบว่า skill หรือ
plugin อ้างว่าจะทำอะไรก่อนติดตั้ง

## Skills

skill คือชุดข้อความแบบมีเวอร์ชันที่มีศูนย์กลางอยู่ที่ `SKILL.md` โดยอาจมี
ไฟล์สนับสนุน ตัวอย่าง เทมเพลต และสคริปต์

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อ skill,
คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และเมตาดาต้า เมตาดาต้าที่ถูกต้อง
มีความสำคัญ เพราะช่วยให้ผู้ใช้ตัดสินใจว่าจะติดตั้ง skill หรือไม่ และ
ช่วยให้การสแกนอัตโนมัติตรวจพบความไม่ตรงกันระหว่างพฤติกรรมที่ประกาศกับพฤติกรรมที่สังเกตได้

ดู [รูปแบบ Skill](/th/clawhub/skill-format)

## Plugins

Plugins คือส่วนขยาย OpenClaw ที่ถูกแพ็กเกจไว้ ClawHub จัดเก็บเมตาดาต้าแพ็กเกจ
ข้อมูลความเข้ากันได้ ลิงก์ต้นทาง อาร์ติแฟกต์ และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง plugin จาก ClawHub ระบบจะตรวจสอบเมตาดาต้าความเข้ากันได้
ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจอาจรวมถึงความเข้ากันได้ของ API,
เวอร์ชัน Gateway ขั้นต่ำ เป้าหมายโฮสต์ ข้อกำหนดของสภาพแวดล้อม และ
ไดเจสต์ของอาร์ติแฟกต์

ใช้แหล่งติดตั้ง ClawHub แบบชัดเจนเมื่อคุณต้องการให้รีจิสทรีเป็น
แหล่งข้อมูลจริง:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างระเบียนเวอร์ชันใหม่ที่เปลี่ยนแปลงไม่ได้ ผู้เผยแพร่ใช้ CLI `clawhub`
สำหรับเวิร์กโฟลว์รีจิสทรีที่ผ่านการยืนยันตัวตน:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้ dry run เพื่อดูตัวอย่าง payload ที่ resolve แล้วก่อนอัปโหลด จากนั้นหน้าสาธารณะจะ
แสดงเมตาดาต้าที่เผยแพร่ ไฟล์ การระบุแหล่งที่มา และสถานะการสแกน

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งแพ็กเกจ:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึกเมตาดาต้าแหล่งติดตั้งเพื่อให้อัปเดตสามารถ resolve แพ็กเกจ
รีจิสทรีเดียวกันได้ในภายหลัง CLI ของ ClawHub ยังรองรับเวิร์กโฟลว์การติดตั้งและ
อัปเดต skill โดยตรงสำหรับผู้ใช้ที่ต้องการโฟลเดอร์ skill ที่จัดการโดยรีจิสทรีนอก
เวิร์กสเปซ OpenClaw เต็มรูปแบบ

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่ release ยังคงอยู่ภายใต้ด่านการอัปโหลด
การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้กลั่นกรอง

หน้าสาธารณะแสดงสรุปการสแกนเมื่อมีให้ใช้ เนื้อหาที่ถูกระงับ ซ่อน
หรือบล็อกอาจหายไปจากการค้นหาสาธารณะและโฟลว์การติดตั้ง ขณะที่ยังคง
มองเห็นได้สำหรับเจ้าของเพื่อการวินิจฉัย

ดู [ความปลอดภัย](/clawhub/security), [การตรวจสอบความปลอดภัย](/clawhub/security-audits),
[การกลั่นกรองและความปลอดภัยของบัญชี](/th/clawhub/moderation), และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub เปิด API อ่านแบบสาธารณะสำหรับการค้นพบ การค้นหา รายละเอียดแพ็กเกจ และ
การดาวน์โหลด แค็ตตาล็อกของบุคคลที่สามอาจใช้ API เหล่านี้ได้เมื่อเชื่อมโยงกลับไปยัง
รายการ ClawHub มาตรฐาน เคารพขีดจำกัดอัตราการเรียกใช้ และหลีกเลี่ยงการสื่อเป็นนัยถึงการรับรอง

ดู [API สาธารณะ](/clawhub/api) และ [HTTP API](/clawhub/http-api)
