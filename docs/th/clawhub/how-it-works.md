# ClawHub ทำงานอย่างไร

ClawHub เป็นชั้นรีจิสทรีสำหรับ Skills และ Plugin ของ OpenClaw โดยเป็นพื้นที่ให้ผู้ใช้ค้นพบแพ็กเกจ ให้ผู้เผยแพร่เผยแพร่เวอร์ชัน และให้ข้อมูลเมตาแก่ OpenClaw อย่างเพียงพอสำหรับติดตั้งและอัปเดตแพ็กเกจเหล่านั้นอย่างปลอดภัย

## ระเบียนรีจิสทรี

รายการสาธารณะแต่ละรายการเป็นระเบียนรีจิสทรีที่ประกอบด้วย:

- เจ้าของและสลักหรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่แล้วอย่างน้อยหนึ่งเวอร์ชัน
- ข้อมูลเมตา สรุป ไฟล์ และการระบุแหล่งที่มา
- บันทึกการเปลี่ยนแปลงและข้อมูลแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง และการติดดาว
- สถานะการสแกนความปลอดภัยและการกลั่นกรอง

หน้ารายการเป็นแหล่งข้อมูลหลักที่ผู้ใช้ใช้ตรวจสอบว่า Skills หรือ Plugin ระบุว่าสามารถทำสิ่งใดได้บ้างก่อนติดตั้ง

## Skills

Skills คือชุดข้อความที่มีการกำหนดเวอร์ชัน โดยมี `SKILL.md` เป็นศูนย์กลาง และสามารถรวมไฟล์ประกอบ ตัวอย่าง เทมเพลต และสคริปต์ได้

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อ คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และข้อมูลเมตาของ Skills ข้อมูลเมตาที่ถูกต้องมีความสำคัญ เพราะช่วยให้ผู้ใช้ตัดสินใจว่าจะติดตั้ง Skills หรือไม่ และช่วยให้การสแกนอัตโนมัติตรวจพบความไม่สอดคล้องระหว่างพฤติกรรมที่ประกาศไว้กับพฤติกรรมที่สังเกตพบ

ดู [รูปแบบ Skills](/clawhub/skill-format)

## Plugin

Plugin คือส่วนขยาย OpenClaw ที่จัดเป็นแพ็กเกจ ClawHub จัดเก็บข้อมูลเมตาของแพ็กเกจ ข้อมูลความเข้ากันได้ ลิงก์แหล่งที่มา อาร์ติแฟกต์ และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง Plugin จาก ClawHub ระบบจะตรวจสอบข้อมูลเมตาความเข้ากันได้ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจอาจประกอบด้วยความเข้ากันได้ของ API เวอร์ชัน Gateway ขั้นต่ำ เป้าหมายโฮสต์ ข้อกำหนดด้านสภาพแวดล้อม และไดเจสต์ของอาร์ติแฟกต์

ใช้แหล่งติดตั้ง ClawHub ที่ระบุอย่างชัดเจนเมื่อต้องการให้รีจิสทรีเป็นแหล่งข้อมูลจริงเพียงแหล่งเดียว:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างระเบียนเวอร์ชันใหม่ที่เปลี่ยนแปลงไม่ได้ ผู้เผยแพร่ใช้ CLI `clawhub` สำหรับเวิร์กโฟลว์รีจิสทรีที่ผ่านการยืนยันตัวตน:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้การทดลองรันเพื่อดูตัวอย่างเพย์โหลดที่แก้ค่าเรียบร้อยแล้วก่อนอัปโหลด จากนั้นหน้าสาธารณะจะแสดงข้อมูลเมตา ไฟล์ การระบุแหล่งที่มา และสถานะการสแกนที่เผยแพร่แล้ว

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งแพ็กเกจ:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึกข้อมูลเมตาของแหล่งติดตั้ง เพื่อให้การอัปเดตสามารถแก้ค่าไปยังแพ็กเกจรีจิสทรีเดียวกันได้ในภายหลัง นอกจากนี้ CLI ของ ClawHub ยังรองรับเวิร์กโฟลว์การติดตั้งและอัปเดต Skills โดยตรง สำหรับผู้ใช้ที่ต้องการให้รีจิสทรีจัดการโฟลเดอร์ Skills ภายนอกพื้นที่ทำงาน OpenClaw แบบเต็มรูปแบบ

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่รีลีสยังคงอยู่ภายใต้เกณฑ์ควบคุมการอัปโหลด การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้ดูแล

หน้าสาธารณะจะแสดงสรุปการสแกนเมื่อมีข้อมูล เนื้อหาที่ถูกระงับ ซ่อน หรือบล็อกอาจหายไปจากการค้นหาสาธารณะและขั้นตอนการติดตั้ง แต่ยังคงปรากฏแก่เจ้าของเพื่อใช้วินิจฉัยปัญหา

ดู [ความปลอดภัย](/th/clawhub/security), [การตรวจสอบความปลอดภัย](/clawhub/security-audits), [การกลั่นกรองและความปลอดภัยของบัญชี](/th/clawhub/moderation) และ [การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub ให้บริการ API แบบอ่านสาธารณะสำหรับการค้นพบ การค้นหา รายละเอียดแพ็กเกจ และการดาวน์โหลด แค็ตตาล็อกของบุคคลที่สามสามารถใช้ API เหล่านี้ได้ เมื่อเชื่อมโยงกลับมายังรายการ ClawHub หลัก ปฏิบัติตามขีดจำกัดอัตราการเรียกใช้ และหลีกเลี่ยงการสื่อเป็นนัยว่าได้รับการรับรอง

ดู [API สาธารณะ](/clawhub/api) และ [HTTP API](/clawhub/http-api)
