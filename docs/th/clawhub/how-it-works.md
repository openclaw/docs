# ClawHub ทำงานอย่างไร

ClawHub คือชั้นรีจิสทรีสำหรับ Skills และ Plugin ของ OpenClaw โดยให้ที่สำหรับผู้ใช้ในการค้นหาแพ็กเกจ ให้ที่สำหรับผู้เผยแพร่ในการออกเวอร์ชัน และให้เมทาดาทาแก่ OpenClaw เพียงพอสำหรับติดตั้งและอัปเดตแพ็กเกจเหล่านั้นอย่างปลอดภัย

## ระเบียนรีจิสทรี

รายการสาธารณะแต่ละรายการคือระเบียนรีจิสทรีที่มี:

- เจ้าของและ slug หรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่หนึ่งเวอร์ชันขึ้นไป
- เมทาดาทา สรุป ไฟล์ และการระบุแหล่งที่มา
- ข้อมูล changelog และแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง และดาว
- สถานะการสแกนความปลอดภัยและการดูแลเนื้อหา

หน้ารายการคือที่มาตรฐานสำหรับให้ผู้ใช้ตรวจสอบว่า Skills หรือ Plugin อ้างว่าทำอะไรได้บ้างก่อนติดตั้ง

## Skills

Skills คือชุดข้อความแบบมีเวอร์ชันที่มีศูนย์กลางอยู่ที่ `SKILL.md` โดยอาจมีไฟล์สนับสนุน ตัวอย่าง เทมเพลต และสคริปต์รวมอยู่ด้วย

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อ Skills คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และเมทาดาทา เมทาดาทาที่ถูกต้องมีความสำคัญ เพราะช่วยให้ผู้ใช้ตัดสินใจได้ว่าจะติดตั้ง Skills หรือไม่ และช่วยให้การสแกนอัตโนมัติตรวจพบความไม่ตรงกันระหว่างพฤติกรรมที่ประกาศไว้กับพฤติกรรมที่สังเกตได้

ดู [รูปแบบ Skills](/th/clawhub/skill-format)

## Plugins

Plugin คือส่วนขยาย OpenClaw ที่ถูกจัดแพ็กเกจ ClawHub จัดเก็บเมทาดาทาแพ็กเกจ ข้อมูลความเข้ากันได้ ลิงก์แหล่งที่มา อาร์ติแฟกต์ และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง Plugin จาก ClawHub ระบบจะตรวจสอบเมทาดาทาความเข้ากันได้ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจสามารถมีความเข้ากันได้ของ API, เวอร์ชัน Gateway ขั้นต่ำ, เป้าหมายโฮสต์, ข้อกำหนดสภาพแวดล้อม และไดเจสต์ของอาร์ติแฟกต์

ใช้แหล่งติดตั้ง ClawHub แบบชัดเจนเมื่อคุณต้องการให้รีจิสทรีเป็นแหล่งข้อมูลจริง:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างระเบียนเวอร์ชันใหม่ที่เปลี่ยนแปลงไม่ได้ ผู้เผยแพร่ใช้ CLI `clawhub` สำหรับเวิร์กโฟลว์รีจิสทรีที่ผ่านการยืนยันตัวตน:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้การทดลองรันเพื่อดูตัวอย่างเพย์โหลดที่แก้ไขแล้วก่อนอัปโหลด จากนั้นหน้าสาธารณะจะแสดงเมทาดาทาที่เผยแพร่ ไฟล์ การระบุแหล่งที่มา และสถานะการสแกน

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งแพ็กเกจ:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึกเมทาดาทาแหล่งติดตั้ง เพื่อให้การอัปเดตสามารถแก้ไปยังแพ็กเกจรีจิสทรีเดียวกันได้ในภายหลัง CLI ของ ClawHub ยังรองรับเวิร์กโฟลว์การติดตั้งและอัปเดต Skills โดยตรงสำหรับผู้ใช้ที่ต้องการโฟลเดอร์ Skills ที่จัดการโดยรีจิสทรีนอกเวิร์กสเปซ OpenClaw แบบเต็ม

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่รีลีสยังคงอยู่ภายใต้ด่านการอัปโหลด การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้ดูแล

หน้าสาธารณะแสดงสรุปการสแกนเมื่อมี เนื้อหาที่ถูกระงับ ซ่อน หรือบล็อกอาจหายไปจากการค้นหาสาธารณะและโฟลว์การติดตั้ง ขณะที่ยังคงมองเห็นได้สำหรับเจ้าของเพื่อการวินิจฉัย

ดู [ความปลอดภัย](/th/clawhub/security), [การตรวจสอบความปลอดภัย](/th/clawhub/security-audits), [การดูแลเนื้อหาและความปลอดภัยของบัญชี](/th/clawhub/moderation) และ [การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub เปิดเผย API แบบอ่านสาธารณะสำหรับการค้นพบ การค้นหา รายละเอียดแพ็กเกจ และการดาวน์โหลด แค็ตตาล็อกของบุคคลที่สามอาจใช้ API เหล่านี้ได้เมื่อเชื่อมโยงกลับไปยังรายการ ClawHub มาตรฐาน เคารพขีดจำกัดอัตราการใช้งาน และหลีกเลี่ยงการสื่อเป็นนัยว่าได้รับการรับรอง

ดู [API สาธารณะ](/th/clawhub/api) และ [HTTP API](/th/clawhub/http-api)
