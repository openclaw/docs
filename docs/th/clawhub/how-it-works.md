# วิธีการทำงานของ ClawHub

ClawHub คือชั้นรีจิสทรีสำหรับ Skills และ Plugin ของ OpenClaw โดยให้พื้นที่แก่ผู้ใช้ในการค้นพบแพ็กเกจ ให้พื้นที่แก่ผู้เผยแพร่ในการออกเวอร์ชัน และให้เมทาดาทาที่เพียงพอแก่ OpenClaw เพื่อให้ติดตั้งและอัปเดตแพ็กเกจเหล่านั้นได้อย่างปลอดภัย

## ระเบียนรีจิสทรี

รายการสาธารณะแต่ละรายการคือระเบียนรีจิสทรีที่มี:

- เจ้าของและสลักหรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่แล้วหนึ่งเวอร์ชันหรือมากกว่า
- เมทาดาทา สรุป ไฟล์ และการระบุแหล่งที่มา
- ข้อมูลบันทึกการเปลี่ยนแปลงและแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง การให้ดาว และความคิดเห็น
- สถานะการสแกนความปลอดภัยและการกลั่นกรอง

หน้ารายการคือที่ที่เป็นแหล่งอ้างอิงหลักให้ผู้ใช้ตรวจสอบว่า Skills หรือ
Plugin อ้างว่าทำอะไรได้บ้างก่อนติดตั้ง

## Skills

Skill คือชุดข้อความที่มีเวอร์ชัน โดยมี `SKILL.md` เป็นศูนย์กลาง สามารถมีไฟล์สนับสนุน ตัวอย่าง เทมเพลต และสคริปต์รวมอยู่ได้

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อ Skill
คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และเมทาดาทา เมทาดาทาที่ถูกต้องมีความสำคัญ เพราะช่วยให้ผู้ใช้ตัดสินใจว่าจะติดตั้ง Skill หรือไม่ และช่วยให้การสแกนอัตโนมัติตรวจพบความไม่ตรงกันระหว่างพฤติกรรมที่ประกาศไว้กับพฤติกรรมที่สังเกตได้

ดู [รูปแบบ Skill](/th/clawhub/skill-format)

## Plugins

Plugin คือส่วนขยาย OpenClaw ที่ถูกแพ็กเกจไว้ ClawHub เก็บเมทาดาทาแพ็กเกจ ข้อมูลความเข้ากันได้ ลิงก์แหล่งที่มา อาร์ติแฟกต์ และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง Plugin จาก ClawHub จะตรวจสอบเมทาดาทาความเข้ากันได้ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจอาจมีความเข้ากันได้ของ API เวอร์ชัน Gateway ขั้นต่ำ เป้าหมายโฮสต์ ข้อกำหนดสภาพแวดล้อม และไดเจสต์ของอาร์ติแฟกต์

ใช้แหล่งติดตั้ง ClawHub แบบชัดเจนเมื่อคุณต้องการให้รีจิสทรีเป็นแหล่งข้อมูลจริง:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างระเบียนเวอร์ชันใหม่ที่แก้ไขไม่ได้ ผู้เผยแพร่ใช้ CLI `clawhub`
สำหรับเวิร์กโฟลว์รีจิสทรีที่ผ่านการยืนยันตัวตน:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้การทดลองรันเพื่อดูตัวอย่างเพย์โหลดที่แก้ไขแล้วก่อนอัปโหลด จากนั้นหน้าสาธารณะจะแสดงเมทาดาทา ไฟล์ การระบุแหล่งที่มา และสถานะการสแกนที่เผยแพร่แล้ว

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งแพ็กเกจ:

```bash
openclaw skills install <skill-slug>
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึกเมทาดาทาแหล่งติดตั้งไว้ เพื่อให้อัปเดตสามารถ resolve แพ็กเกจรีจิสทรีเดียวกันได้ภายหลัง CLI ของ ClawHub ยังรองรับเวิร์กโฟลว์การติดตั้งและอัปเดต Skill โดยตรงสำหรับผู้ใช้ที่ต้องการโฟลเดอร์ Skill ที่จัดการโดยรีจิสทรีภายนอกเวิร์กสเปซ OpenClaw แบบเต็ม

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่การออกเวอร์ชันยังอยู่ภายใต้ด่านอัปโหลด การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้กลั่นกรอง

หน้าสาธารณะแสดงสรุปการสแกนเมื่อมีให้ใช้งาน เนื้อหาที่ถูกพัก ซ่อน หรือบล็อกอาจหายไปจากการค้นหาสาธารณะและโฟลว์การติดตั้ง ในขณะที่ยังคงมองเห็นได้สำหรับเจ้าของเพื่อการวินิจฉัย

ดู [ความปลอดภัย + การกลั่นกรอง](/th/clawhub/security) และ
[การใช้งานที่ยอมรับได้](/th/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub เปิดเผย API สำหรับอ่านแบบสาธารณะเพื่อการค้นพบ การค้นหา รายละเอียดแพ็กเกจ และการดาวน์โหลด แคตตาล็อกของบุคคลที่สามอาจใช้ API เหล่านี้ได้เมื่อเชื่อมโยงกลับไปยังรายการ ClawHub หลัก เคารพขีดจำกัดอัตราการเรียกใช้ และหลีกเลี่ยงการสื่อว่ามีการรับรอง

ดู [API สาธารณะ](/th/clawhub/api) และ [HTTP API](/th/clawhub/http-api)
