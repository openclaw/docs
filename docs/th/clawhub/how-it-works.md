# ClawHub ทำงานอย่างไร

ClawHub เป็นชั้นรีจิสทรีสำหรับ Skills และ Plugin ของ OpenClaw โดยเป็นพื้นที่ให้ผู้ใช้
ค้นหาแพ็กเกจ ให้ผู้เผยแพร่เปิดตัวเวอร์ชัน และให้ข้อมูลเมตาแก่ OpenClaw
อย่างเพียงพอเพื่อให้ติดตั้งและอัปเดตแพ็กเกจเหล่านั้นได้อย่างปลอดภัย

## ระเบียนรีจิสทรี

รายการสาธารณะแต่ละรายการเป็นระเบียนรีจิสทรีที่ประกอบด้วย:

- เจ้าของและ slug หรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่แล้วอย่างน้อยหนึ่งเวอร์ชัน
- ข้อมูลเมตา ข้อมูลสรุป ไฟล์ และการระบุแหล่งที่มา
- บันทึกการเปลี่ยนแปลงและข้อมูลแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง และการติดดาว
- สถานะการสแกนความปลอดภัยและการกลั่นกรอง

หน้ารายการเป็นแหล่งข้อมูลหลักที่ผู้ใช้ใช้ตรวจสอบว่า Skills หรือ
Plugin ระบุว่าสามารถทำสิ่งใดได้บ้างก่อนติดตั้ง

## Skills

Skills คือชุดข้อความที่มีการกำหนดเวอร์ชันและมี `SKILL.md` เป็นแกนหลัก โดยอาจมี
ไฟล์ประกอบ ตัวอย่าง เทมเพลต และสคริปต์รวมอยู่ด้วย

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อ Skills
คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และข้อมูลเมตา ข้อมูลเมตาที่ถูกต้อง
มีความสำคัญเพราะช่วยให้ผู้ใช้ตัดสินใจว่าจะติดตั้ง Skills หรือไม่ และ
ช่วยให้การสแกนอัตโนมัติตรวจพบความไม่ตรงกันระหว่างพฤติกรรมที่ประกาศไว้กับพฤติกรรมที่สังเกตพบ

ดู [รูปแบบ Skills](/th/clawhub/skill-format)

## Plugin

Plugin คือส่วนขยาย OpenClaw ที่จัดทำเป็นแพ็กเกจ ClawHub จัดเก็บข้อมูลเมตาของแพ็กเกจ
ข้อมูลความเข้ากันได้ ลิงก์แหล่งที่มา อาร์ติแฟกต์ และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง Plugin จาก ClawHub ระบบจะตรวจสอบข้อมูลเมตา
ความเข้ากันได้ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจอาจมีข้อมูลความเข้ากันได้ของ API
เวอร์ชัน Gateway ขั้นต่ำ เป้าหมายโฮสต์ ข้อกำหนดด้านสภาพแวดล้อม และไดเจสต์
ของอาร์ติแฟกต์

ใช้แหล่งติดตั้ง ClawHub ที่ระบุอย่างชัดเจนเมื่อต้องการให้รีจิสทรีเป็น
แหล่งข้อมูลที่เชื่อถือได้:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างระเบียนเวอร์ชันใหม่ที่แก้ไขไม่ได้ ผู้เผยแพร่ใช้ `clawhub`
CLI สำหรับเวิร์กโฟลว์รีจิสทรีที่ผ่านการยืนยันตัวตน:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้การทดลองรันเพื่อดูตัวอย่างเพย์โหลดที่ผ่านการแก้ค่าแล้วก่อนอัปโหลด จากนั้นหน้าสาธารณะ
จะแสดงข้อมูลเมตาที่เผยแพร่ ไฟล์ การระบุแหล่งที่มา และสถานะการสแกน

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งแพ็กเกจ:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึกข้อมูลเมตาของแหล่งติดตั้ง เพื่อให้การอัปเดตสามารถค้นหาแพ็กเกจ
รีจิสทรีเดียวกันได้ในภายหลัง นอกจากนี้ CLI ของ ClawHub ยังรองรับเวิร์กโฟลว์ติดตั้งและ
อัปเดต Skills โดยตรงสำหรับผู้ใช้ที่ต้องการให้รีจิสทรีจัดการโฟลเดอร์ Skills ภายนอก
พื้นที่ทำงาน OpenClaw แบบเต็มรูปแบบ

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่รุ่นที่เผยแพร่ยังคงอยู่ภายใต้เกตการอัปโหลด
การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้กลั่นกรอง

หน้าสาธารณะจะแสดงข้อมูลสรุปการสแกนเมื่อมี เนื้อหาที่ถูกระงับ ซ่อน
หรือบล็อกอาจหายไปจากการค้นหาสาธารณะและขั้นตอนการติดตั้ง แต่ยังคง
แสดงต่อเจ้าของเพื่อใช้วินิจฉัยปัญหา

ดู [ความปลอดภัย](/clawhub/security), [การตรวจสอบความปลอดภัย](/clawhub/security-audits),
[การกลั่นกรองและความปลอดภัยของบัญชี](/th/clawhub/moderation) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub มี API สำหรับอ่านแบบสาธารณะเพื่อใช้ค้นพบ ค้นหา ดูรายละเอียดแพ็กเกจ และ
ดาวน์โหลด แค็ตตาล็อกของบุคคลที่สามอาจใช้ API เหล่านี้ได้เมื่อเชื่อมโยงกลับมายัง
รายการ ClawHub หลัก เคารพขีดจำกัดอัตราการใช้งาน และไม่ทำให้เข้าใจว่าได้รับการรับรอง

ดู [API สาธารณะ](/clawhub/api) และ [HTTP API](/clawhub/http-api)
