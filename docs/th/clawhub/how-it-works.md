# วิธีการทำงานของ ClawHub

ClawHub คือชั้น registry สำหรับ Skills และ Plugin ของ OpenClaw โดยให้ผู้ใช้มี
ที่สำหรับค้นพบแพ็กเกจ ให้ผู้เผยแพร่มีที่สำหรับปล่อยเวอร์ชัน และ
ให้ OpenClaw มีข้อมูลเมตาเพียงพอสำหรับติดตั้งและอัปเดตแพ็กเกจเหล่านั้นอย่างปลอดภัย

## ระเบียน registry

รายการสาธารณะแต่ละรายการคือระเบียน registry ที่มี:

- เจ้าของและ slug หรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่หนึ่งรายการขึ้นไป
- ข้อมูลเมตา สรุป ไฟล์ และการระบุแหล่งที่มา
- changelog และข้อมูลแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง และดาว
- สถานะการสแกนความปลอดภัยและการกลั่นกรอง

หน้ารายการคือที่หลักสำหรับให้ผู้ใช้ตรวจสอบว่า skill หรือ
Plugin อ้างว่าสามารถทำอะไรได้ก่อนติดตั้ง

ดู [รูปแบบ Skill](/th/clawhub/skill-format)

## Skills

skill คือชุดข้อความแบบมีเวอร์ชันที่มีศูนย์กลางอยู่ที่ `SKILL.md` โดยสามารถมี
ไฟล์สนับสนุน ตัวอย่าง เทมเพลต และสคริปต์ได้

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อ skill,
คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และข้อมูลเมตา ข้อมูลเมตาที่ถูกต้อง
มีความสำคัญ เพราะช่วยให้ผู้ใช้ตัดสินใจว่าจะติดตั้ง skill หรือไม่ และ
ช่วยให้การสแกนอัตโนมัติตรวจพบความไม่ตรงกันระหว่างพฤติกรรมที่ประกาศไว้กับพฤติกรรมที่สังเกตได้

ดู [รูปแบบ Skill](/th/clawhub/skill-format)

## Plugin

Plugin คือส่วนขยายของ OpenClaw ที่จัดแพ็กเกจแล้ว ClawHub จัดเก็บข้อมูลเมตาของแพ็กเกจ,
ข้อมูลความเข้ากันได้ ลิงก์แหล่งที่มา อาร์ติแฟกต์ และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง Plugin จาก ClawHub จะตรวจสอบข้อมูลเมตาความเข้ากันได้
ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจสามารถรวมความเข้ากันได้ของ API,
เวอร์ชัน Gateway ขั้นต่ำ เป้าหมายโฮสต์ ข้อกำหนดสภาพแวดล้อม และไดเจสต์ของอาร์ติแฟกต์

ใช้แหล่งติดตั้ง ClawHub แบบชัดเจนเมื่อคุณต้องการให้ registry เป็น
แหล่งข้อมูลจริง:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างระเบียนเวอร์ชันใหม่ที่เปลี่ยนแปลงไม่ได้ ผู้เผยแพร่ใช้ CLI `clawhub`
สำหรับเวิร์กโฟลว์ registry ที่ยืนยันตัวตนแล้ว:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้ dry run เพื่อดูตัวอย่าง payload ที่ resolve แล้วก่อนอัปโหลด จากนั้นหน้าสาธารณะจะ
แสดงข้อมูลเมตาที่เผยแพร่ ไฟล์ การระบุแหล่งที่มา และสถานะการสแกน

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งแพ็กเกจ:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึกข้อมูลเมตาแหล่งติดตั้งเพื่อให้อัปเดตสามารถ resolve
แพ็กเกจ registry เดิมได้ในภายหลัง CLI ของ ClawHub ยังรองรับเวิร์กโฟลว์ติดตั้งและ
อัปเดต skill โดยตรงสำหรับผู้ใช้ที่ต้องการโฟลเดอร์ skill ที่จัดการโดย registry นอก
เวิร์กสเปซ OpenClaw แบบเต็ม

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่ release ยังอยู่ภายใต้เกตการอัปโหลด,
การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้กลั่นกรอง

หน้าสาธารณะแสดงสรุปการสแกนเมื่อมีให้ใช้ เนื้อหาที่ถูกพักไว้ ซ่อน
หรือบล็อก อาจหายไปจากการค้นหาสาธารณะและโฟลว์การติดตั้ง ขณะที่ยังคง
มองเห็นได้สำหรับเจ้าของเพื่อการวินิจฉัย

ดู [ความปลอดภัย](/clawhub/security), [การตรวจสอบความปลอดภัย](/clawhub/security-audits),
[การกลั่นกรองและความปลอดภัยของบัญชี](/th/clawhub/moderation), และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub เปิดเผย API อ่านสาธารณะสำหรับการค้นพบ การค้นหา รายละเอียดแพ็กเกจ และ
การดาวน์โหลด แคตตาล็อกของบุคคลที่สามอาจใช้ API เหล่านี้ได้เมื่อเชื่อมโยงกลับไปยัง
รายการ ClawHub หลัก เคารพขีดจำกัดอัตราการใช้งาน และหลีกเลี่ยงการสื่อว่ามีการรับรอง

ดู [API สาธารณะ](/th/clawhub/api) และ [HTTP API](/clawhub/http-api)
