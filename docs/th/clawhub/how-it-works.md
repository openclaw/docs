# วิธีการทำงานของ ClawHub

ClawHub คือเลเยอร์ registry สำหรับ Skills และ Plugin ของ OpenClaw โดยมอบ
พื้นที่ให้ผู้ใช้ค้นพบแพ็กเกจ มอบพื้นที่ให้ผู้เผยแพร่ออกเวอร์ชัน และมอบ
metadata ที่เพียงพอให้ OpenClaw ติดตั้งและอัปเดตแพ็กเกจเหล่านั้นได้อย่างปลอดภัย

## ระเบียน registry

รายการสาธารณะแต่ละรายการคือระเบียน registry ที่มี:

- เจ้าของและ slug หรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่แล้วอย่างน้อยหนึ่งเวอร์ชัน
- metadata, สรุป, ไฟล์ และการระบุแหล่งที่มา
- changelog และข้อมูลแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง และการติดดาว
- สถานะการสแกนความปลอดภัยและการดูแลตรวจสอบ

หน้ารายการคือพื้นที่มาตรฐานให้ผู้ใช้ตรวจสอบว่า skill หรือ
plugin อ้างว่าสามารถทำอะไรก่อนติดตั้ง

## Skills

skill คือชุดข้อความแบบมีเวอร์ชันที่มีศูนย์กลางอยู่ที่ `SKILL.md` โดยสามารถมี
ไฟล์สนับสนุน ตัวอย่าง เทมเพลต และสคริปต์ได้

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อ skill,
คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และ metadata metadata ที่ถูกต้องมีความสำคัญ
เพราะช่วยให้ผู้ใช้ตัดสินใจว่าจะติดตั้ง skill หรือไม่ และช่วยให้การสแกนอัตโนมัติ
ตรวจพบความไม่ตรงกันระหว่างพฤติกรรมที่ประกาศกับพฤติกรรมที่สังเกตได้

ดู [รูปแบบ skill](/th/clawhub/skill-format)

## Plugins

Plugin คือส่วนขยายของ OpenClaw ที่บรรจุเป็นแพ็กเกจ ClawHub จัดเก็บ metadata
ของแพ็กเกจ ข้อมูลความเข้ากันได้ ลิงก์แหล่งที่มา artifact และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง plugin จาก ClawHub ระบบจะตรวจสอบ metadata ความเข้ากันได้
ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจสามารถมีความเข้ากันได้ของ API,
เวอร์ชัน gateway ขั้นต่ำ เป้าหมาย host ข้อกำหนดสภาพแวดล้อม และ digest ของ artifact

ใช้แหล่งติดตั้ง ClawHub แบบชัดเจนเมื่อคุณต้องการให้ registry เป็น
แหล่งข้อมูลอ้างอิงหลัก:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างระเบียนเวอร์ชันใหม่ที่แก้ไขไม่ได้ ผู้เผยแพร่ใช้ CLI `clawhub`
สำหรับเวิร์กโฟลว์ registry ที่ผ่านการยืนยันตัวตน:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้การรันแบบ dry run เพื่อดูตัวอย่าง payload ที่ resolve แล้วก่อนอัปโหลด จากนั้น
หน้าสาธารณะจะแสดง metadata ที่เผยแพร่ ไฟล์ การระบุแหล่งที่มา และสถานะการสแกน

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งแพ็กเกจ:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึก metadata ของแหล่งติดตั้งเพื่อให้การอัปเดตสามารถ resolve
แพ็กเกจ registry เดิมได้ในภายหลัง CLI ของ ClawHub ยังรองรับเวิร์กโฟลว์
ติดตั้งและอัปเดต skill โดยตรงสำหรับผู้ใช้ที่ต้องการโฟลเดอร์ skill ที่จัดการโดย
registry ภายนอก workspace ของ OpenClaw แบบเต็ม

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่ release ยังคงอยู่ภายใต้ด่านการอัปโหลด
การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้ดูแล

หน้าสาธารณะแสดงสรุปการสแกนเมื่อมีให้ใช้งาน เนื้อหาที่ถูกระงับ ซ่อน
หรือบล็อกอาจหายไปจากการค้นหาสาธารณะและขั้นตอนการติดตั้ง ขณะที่ยังคง
มองเห็นได้สำหรับเจ้าของเพื่อใช้วินิจฉัยปัญหา

ดู [ความปลอดภัย](/clawhub/security), [การตรวจสอบความปลอดภัย](/clawhub/security-audits),
[การดูแลตรวจสอบและความปลอดภัยของบัญชี](/th/clawhub/moderation) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub เปิดเผย API อ่านสาธารณะสำหรับการค้นพบ การค้นหา รายละเอียดแพ็กเกจ และ
การดาวน์โหลด แค็ตตาล็อกของบุคคลที่สามอาจใช้ API เหล่านี้ได้เมื่อมีลิงก์กลับไปยัง
รายการ ClawHub มาตรฐาน เคารพ rate limit และหลีกเลี่ยงการสื่อเป็นนัยว่าได้รับการรับรอง

ดู [API สาธารณะ](/clawhub/api) และ [HTTP API](/clawhub/http-api)
