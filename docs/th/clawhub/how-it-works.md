# วิธีการทำงานของ ClawHub

ClawHub คือเลเยอร์รีจิสทรีสำหรับ Skills และ Plugin ของ OpenClaw โดยให้ผู้ใช้มี
ที่สำหรับค้นหาแพ็กเกจ ให้ผู้เผยแพร่มีที่สำหรับปล่อยเวอร์ชัน และให้
OpenClaw มีเมทาดาทาเพียงพอสำหรับติดตั้งและอัปเดตแพ็กเกจเหล่านั้นอย่างปลอดภัย

## ระเบียนรีจิสทรี

รายการสาธารณะแต่ละรายการคือระเบียนรีจิสทรีที่มี:

- เจ้าของและ slug หรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่แล้วหนึ่งเวอร์ชันขึ้นไป
- เมทาดาทา สรุป ไฟล์ และการระบุแหล่งที่มา
- ข้อมูลบันทึกการเปลี่ยนแปลงและแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง และการติดดาว
- สถานะการสแกนความปลอดภัยและการดูแลตรวจสอบ

หน้ารายการคือที่หลักอย่างเป็นทางการให้ผู้ใช้ตรวจสอบว่า Skills หรือ
Plugin อ้างว่าทำอะไรได้บ้างก่อนติดตั้ง

## Skills

Skills คือชุดข้อความแบบมีเวอร์ชันที่มี `SKILL.md` เป็นศูนย์กลาง โดยอาจมี
ไฟล์สนับสนุน ตัวอย่าง เทมเพลต และสคริปต์รวมอยู่ด้วย

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อ Skills,
คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และเมทาดาทา เมทาดาทาที่ถูกต้อง
มีความสำคัญ เพราะช่วยให้ผู้ใช้ตัดสินใจได้ว่าจะติดตั้ง Skills หรือไม่ และ
ช่วยให้การสแกนอัตโนมัติตรวจพบความไม่ตรงกันระหว่างพฤติกรรมที่ประกาศไว้กับพฤติกรรมที่สังเกตได้

ดู [รูปแบบ Skills](/th/clawhub/skill-format)

## Plugin

Plugin คือส่วนขยาย OpenClaw ที่แพ็กเกจแล้ว ClawHub จัดเก็บเมทาดาทาแพ็กเกจ
ข้อมูลความเข้ากันได้ ลิงก์แหล่งที่มา อาร์ติแฟกต์ และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง Plugin จาก ClawHub ระบบจะตรวจสอบเมทาดาทา
ความเข้ากันได้ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจอาจมีความเข้ากันได้ของ API,
เวอร์ชัน Gateway ขั้นต่ำ เป้าหมายโฮสต์ ข้อกำหนดสภาพแวดล้อม และ digest ของอาร์ติแฟกต์

ใช้แหล่งติดตั้ง ClawHub แบบชัดเจนเมื่อคุณต้องการให้รีจิสทรีเป็น
แหล่งข้อมูลหลักที่เชื่อถือได้:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างระเบียนเวอร์ชันใหม่ที่แก้ไขไม่ได้ ผู้เผยแพร่ใช้ CLI `clawhub`
สำหรับเวิร์กโฟลว์รีจิสทรีที่ผ่านการยืนยันตัวตน:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้ dry run เพื่อดูตัวอย่าง payload ที่ resolve แล้วก่อนอัปโหลด จากนั้นหน้า
สาธารณะจะแสดงเมทาดาทาที่เผยแพร่ ไฟล์ การระบุแหล่งที่มา และสถานะการสแกน

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งแพ็กเกจ:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึกเมทาดาทาแหล่งติดตั้งเพื่อให้อัปเดตสามารถ resolve ไปยัง
แพ็กเกจรีจิสทรีเดียวกันได้ในภายหลัง CLI ของ ClawHub ยังรองรับเวิร์กโฟลว์
ติดตั้งและอัปเดต Skills โดยตรงสำหรับผู้ใช้ที่ต้องการโฟลเดอร์ Skills ที่จัดการโดยรีจิสทรี
นอก workspace OpenClaw เต็มรูปแบบ

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่รีลีสยังคงอยู่ภายใต้เกตการอัปโหลด
การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้ดูแล

หน้าสาธารณะจะแสดงสรุปการสแกนเมื่อมีให้ใช้งาน เนื้อหาที่ถูกพักไว้ ซ่อนไว้
หรือบล็อกไว้อาจหายไปจากการค้นหาสาธารณะและโฟลว์ติดตั้ง ขณะที่ยังคง
มองเห็นได้สำหรับเจ้าของเพื่อการวินิจฉัย

ดู [ความปลอดภัย](/clawhub/security), [การตรวจสอบความปลอดภัย](/clawhub/security-audits),
[การดูแลตรวจสอบและความปลอดภัยของบัญชี](/th/clawhub/moderation) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub เปิดเผย API สำหรับอ่านแบบสาธารณะสำหรับการค้นพบ การค้นหา รายละเอียดแพ็กเกจ และ
การดาวน์โหลด แคตตาล็อกของบุคคลที่สามอาจใช้ API เหล่านี้ได้เมื่อมีลิงก์กลับไปยัง
รายการ ClawHub ที่เป็นต้นทางอย่างเป็นทางการ เคารพขีดจำกัดอัตราการใช้งาน และหลีกเลี่ยงการสื่อว่าได้รับการรับรอง

ดู [API สาธารณะ](/clawhub/api) และ [HTTP API](/clawhub/http-api)
