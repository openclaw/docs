# วิธีการทำงานของ ClawHub

ClawHub คือเลเยอร์รีจิสทรีสำหรับ Skills และ Plugin ของ OpenClaw ซึ่งมอบพื้นที่ให้ผู้ใช้ค้นพบแพ็กเกจ มอบพื้นที่ให้ผู้เผยแพร่ปล่อยเวอร์ชัน และมอบเมตาดาต้าที่เพียงพอให้ OpenClaw เพื่อติดตั้งและอัปเดตแพ็กเกจเหล่านั้นได้อย่างปลอดภัย

## ระเบียนรีจิสทรี

รายการสาธารณะแต่ละรายการคือระเบียนรีจิสทรีที่มี:

- เจ้าของและ slug หรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่แล้วหนึ่งเวอร์ชันหรือมากกว่า
- เมตาดาต้า สรุป ไฟล์ และการระบุแหล่งที่มา
- ข้อมูล changelog และแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง และดาว
- สถานะการสแกนความปลอดภัยและการดูแลเนื้อหา

หน้ารายการคือที่มาตรฐานสำหรับให้ผู้ใช้ตรวจสอบว่า skill หรือ plugin อ้างว่าทำอะไรได้บ้างก่อนติดตั้ง

## Skills

skill คือชุดข้อความแบบมีเวอร์ชันที่มีศูนย์กลางอยู่ที่ `SKILL.md` โดยอาจรวมไฟล์สนับสนุน ตัวอย่าง เทมเพลต และสคริปต์ได้

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อ skill คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และเมตาดาต้า เมตาดาต้าที่ถูกต้องมีความสำคัญ เพราะช่วยให้ผู้ใช้ตัดสินใจว่าจะติดตั้ง skill หรือไม่ และช่วยให้การสแกนอัตโนมัติตรวจพบความไม่ตรงกันระหว่างพฤติกรรมที่ประกาศไว้กับพฤติกรรมที่สังเกตได้

ดู [รูปแบบ Skill](/th/clawhub/skill-format)

## Plugins

Plugin คือส่วนขยาย OpenClaw ที่ถูกแพ็กเกจ ClawHub จัดเก็บเมตาดาต้าแพ็กเกจ ข้อมูลความเข้ากันได้ ลิงก์แหล่งที่มา อาร์ติแฟกต์ และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง plugin จาก ClawHub ระบบจะตรวจสอบเมตาดาต้าความเข้ากันได้ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจสามารถมีความเข้ากันได้ของ API เวอร์ชัน gateway ขั้นต่ำ เป้าหมายโฮสต์ ข้อกำหนดสภาพแวดล้อม และ digest ของอาร์ติแฟกต์

ใช้แหล่งติดตั้ง ClawHub แบบชัดเจนเมื่อคุณต้องการให้รีจิสทรีเป็นแหล่งข้อมูลที่เชื่อถือได้:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างระเบียนเวอร์ชันใหม่ที่แก้ไขไม่ได้ ผู้เผยแพร่ใช้ CLI `clawhub` สำหรับเวิร์กโฟลว์รีจิสทรีที่ผ่านการยืนยันตัวตน:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้ dry run เพื่อดูตัวอย่าง payload ที่แก้ไขแล้วก่อนอัปโหลด จากนั้นหน้าสาธารณะจะแสดงเมตาดาต้าที่เผยแพร่ ไฟล์ การระบุแหล่งที่มา และสถานะการสแกน

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งแพ็กเกจ:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึกเมตาดาต้าแหล่งติดตั้งเพื่อให้การอัปเดตสามารถ resolve แพ็กเกจรีจิสทรีเดียวกันได้ในภายหลัง CLI ของ ClawHub ยังรองรับเวิร์กโฟลว์ติดตั้งและอัปเดต skill โดยตรงสำหรับผู้ใช้ที่ต้องการโฟลเดอร์ skill ที่จัดการโดยรีจิสทรีภายนอก workspace OpenClaw แบบเต็ม

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่ release ยังอยู่ภายใต้เกตการอัปโหลด การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้ดูแล

หน้าสาธารณะแสดงสรุปการสแกนเมื่อมี เนื้อหาที่ถูกระงับ ซ่อน หรือบล็อกอาจหายไปจากการค้นหาสาธารณะและโฟลว์การติดตั้ง ในขณะที่ยังคงมองเห็นได้สำหรับเจ้าของเพื่อใช้วินิจฉัยปัญหา

ดู [ความปลอดภัย](/clawhub/security), [การตรวจสอบความปลอดภัย](/clawhub/security-audits),
[การดูแลเนื้อหาและความปลอดภัยของบัญชี](/th/clawhub/moderation) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub เปิด public read APIs สำหรับการค้นพบ การค้นหา รายละเอียดแพ็กเกจ และการดาวน์โหลด แค็ตตาล็อกของบุคคลที่สามอาจใช้ API เหล่านี้เมื่อเชื่อมโยงกลับไปยังรายการ ClawHub ที่เป็นมาตรฐาน เคารพ rate limit และหลีกเลี่ยงการสื่อว่าได้รับการรับรอง

ดู [Public API](/clawhub/api) และ [HTTP API](/clawhub/http-api)
