# วิธีการทำงานของ ClawHub

ClawHub คือเลเยอร์รีจิสทรีสำหรับ Skills และ Plugin ของ OpenClaw โดยให้ผู้ใช้มี
ที่สำหรับค้นหาแพ็กเกจ ให้ผู้เผยแพร่มีที่สำหรับออกเวอร์ชัน และให้
OpenClaw มีเมทาดาต้าเพียงพอสำหรับติดตั้งและอัปเดตแพ็กเกจเหล่านั้นอย่างปลอดภัย

## ระเบียนรีจิสทรี

รายการสาธารณะแต่ละรายการคือระเบียนรีจิสทรีที่มี:

- เจ้าของและสลักหรือชื่อแพ็กเกจ
- เวอร์ชันที่เผยแพร่หนึ่งเวอร์ชันขึ้นไป
- เมทาดาต้า สรุป ไฟล์ และการระบุแหล่งที่มา
- บันทึกการเปลี่ยนแปลงและข้อมูลแท็ก เช่น `latest`
- สัญญาณการดาวน์โหลด การติดตั้ง และการให้ดาว
- สถานะการสแกนความปลอดภัยและการกลั่นกรอง

หน้ารายการคือที่ canonical สำหรับให้ผู้ใช้ตรวจสอบว่า Skills หรือ
Plugin อ้างว่าทำอะไรได้ก่อนติดตั้ง

## Skills

Skills คือชุดข้อความที่มีเวอร์ชัน โดยมีศูนย์กลางอยู่ที่ `SKILL.md` และสามารถมี
ไฟล์สนับสนุน ตัวอย่าง เทมเพลต และสคริปต์ได้

ClawHub อ่าน frontmatter ของ `SKILL.md` เพื่อทำความเข้าใจชื่อ Skills,
คำอธิบาย ข้อกำหนด ตัวแปรสภาพแวดล้อม และเมทาดาต้า เมทาดาต้าที่ถูกต้อง
มีความสำคัญเพราะช่วยให้ผู้ใช้ตัดสินใจว่าจะติดตั้ง Skills หรือไม่ และ
ช่วยให้การสแกนอัตโนมัติตรวจพบความไม่ตรงกันระหว่างพฤติกรรมที่ประกาศไว้กับพฤติกรรมที่สังเกตได้

ดู [รูปแบบ Skills](/th/clawhub/skill-format)

## Plugin

Plugin คือส่วนขยาย OpenClaw ที่จัดแพ็กเกจแล้ว ClawHub จัดเก็บเมทาดาต้าแพ็กเกจ,
ข้อมูลความเข้ากันได้ ลิงก์แหล่งที่มา อาร์ติแฟกต์ และระเบียนเวอร์ชัน

เมื่อ OpenClaw ติดตั้ง Plugin จาก ClawHub ระบบจะตรวจสอบเมทาดาต้าความเข้ากันได้
ที่ประกาศไว้ก่อนติดตั้ง ระเบียนแพ็กเกจสามารถมีความเข้ากันได้ของ API,
เวอร์ชัน Gateway ขั้นต่ำ เป้าหมายโฮสต์ ข้อกำหนดด้านสภาพแวดล้อม และไดเจสต์ของอาร์ติแฟกต์

ใช้แหล่งติดตั้ง ClawHub แบบระบุชัดเจนเมื่อคุณต้องการให้รีจิสทรีเป็น
แหล่งข้อมูลจริง:

```bash
openclaw plugins install clawhub:<package>
```

## การเผยแพร่

การเผยแพร่จะสร้างระเบียนเวอร์ชันใหม่ที่เปลี่ยนแปลงไม่ได้ ผู้เผยแพร่ใช้ CLI `clawhub`
สำหรับเวิร์กโฟลว์รีจิสทรีที่ผ่านการยืนยันตัวตน:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

ใช้การทดลองรันเพื่อดูตัวอย่างเพย์โหลดที่แก้ไขแล้วก่อนอัปโหลด จากนั้นหน้าสาธารณะจะ
แสดงเมทาดาต้าที่เผยแพร่ ไฟล์ การระบุแหล่งที่มา และสถานะการสแกน

## การติดตั้งและการอัปเดต

คำสั่งติดตั้งของ OpenClaw ใช้ ClawHub เป็นแหล่งแพ็กเกจ:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw บันทึกเมทาดาต้าแหล่งติดตั้งเพื่อให้การอัปเดตสามารถแก้ไขไปยัง
แพ็กเกจรีจิสทรีเดิมได้ในภายหลัง CLI ของ ClawHub ยังรองรับเวิร์กโฟลว์ติดตั้งและ
อัปเดต Skills โดยตรงสำหรับผู้ใช้ที่ต้องการโฟลเดอร์ Skills ที่จัดการโดยรีจิสทรีนอก
เวิร์กสเปซ OpenClaw แบบเต็ม

## สถานะความปลอดภัย

ClawHub เปิดให้เผยแพร่ได้ แต่รีลิสยังคงอยู่ภายใต้เกตการอัปโหลด,
การตรวจสอบอัตโนมัติ รายงานจากผู้ใช้ และการดำเนินการของผู้กลั่นกรอง

หน้าสาธารณะแสดงสรุปการสแกนเมื่อมี เนื้อหาที่ถูกพักไว้ ซ่อนไว้
หรือบล็อกไว้อาจหายไปจากการค้นหาสาธารณะและโฟลว์การติดตั้ง แต่ยังคง
มองเห็นได้สำหรับเจ้าของเพื่อการวินิจฉัย

ดู [ความปลอดภัย](/clawhub/security), [การตรวจสอบความปลอดภัย](/clawhub/security-audits),
[การกลั่นกรองและความปลอดภัยของบัญชี](/th/clawhub/moderation) และ
[การใช้งานที่ยอมรับได้](/clawhub/acceptable-usage)

## การเข้าถึง API

ClawHub เปิดเผย API อ่านสาธารณะสำหรับการค้นพบ การค้นหา รายละเอียดแพ็กเกจ และ
การดาวน์โหลด แค็ตตาล็อกของบุคคลที่สามอาจใช้ API เหล่านี้ได้เมื่อเชื่อมโยงกลับไปยัง
รายการ ClawHub canonical เคารพขีดจำกัดอัตรา และหลีกเลี่ยงการสื่อว่าได้รับการรับรอง

ดู [API สาธารณะ](/clawhub/api) และ [HTTP API](/clawhub/http-api)
