# คู่มือเอกสาร

ไดเรกทอรีนี้เป็นเจ้าของการเขียนเอกสาร กฎลิงก์ของ Mintlify และนโยบาย i18n ของเอกสาร

## กฎของ Mintlify

- เอกสารถูกโฮสต์บน Mintlify (`https://docs.openclaw.ai`)
- ลิงก์เอกสารภายในใน `docs/**/*.md` ต้องคงเป็นแบบสัมพันธ์จากรูทโดยไม่มีส่วนต่อท้าย `.md` หรือ `.mdx` (ตัวอย่าง: `[Config](/gateway/configuration)`)
- การอ้างอิงข้ามส่วนควรใช้แองเคอร์บนพาธแบบสัมพันธ์จากรูท (ตัวอย่าง: `[Hooks](/gateway/configuration-reference#hooks)`)
- หัวข้อเอกสารควรหลีกเลี่ยงขีดยาวและอะพอสทรอฟี เพราะการสร้างแองเคอร์ของ Mintlify เปราะบางกับอักขระเหล่านั้น
- README และเอกสารอื่นที่เรนเดอร์บน GitHub ควรคง URL เอกสารแบบสัมบูรณ์ไว้ เพื่อให้ลิงก์ทำงานนอก Mintlify ได้
- เนื้อหาเอกสารต้องคงความทั่วไป: ไม่มีชื่ออุปกรณ์ส่วนตัว ชื่อโฮสต์ หรือพาธภายในเครื่อง; ใช้ตัวแทนค่าเช่น `user@gateway-host`

## กฎเนื้อหาเอกสาร

- สำหรับเอกสาร ข้อความ UI และรายการตัวเลือก ให้เรียงบริการ/ผู้ให้บริการตามตัวอักษร เว้นแต่ส่วนนั้นกำลังอธิบายลำดับรันไทม์หรือลำดับการตรวจจับอัตโนมัติอย่างชัดเจน
- รักษาการตั้งชื่อ Plugin ที่รวมมาให้สอดคล้องกับกฎคำศัพท์ Plugin ทั่วทั้งรีโปใน `AGENTS.md` ที่รูท

## เอกสารภายใน

- เอกสารผู้ปฏิบัติงานส่วนตัวที่ใช้งานระยะยาวควรอยู่ใน `~/Projects/manager/docs/`
- เอกสารร่าง/มิเรอร์ภายในที่อยู่ในรีโปอาจอยู่ใต้ `docs/internal/` ที่ถูกละเว้น
- ห้ามเพิ่มหน้า `docs/internal/**` ลงในระบบนำทางของ `docs/docs.json` หรือลิงก์ไปจากเอกสารสาธารณะ
- `scripts/docs-sync-publish.mjs` จะยกเว้นและลบ `docs/internal/**` ออกจากรีโปเผยแพร่สาธารณะ `openclaw/docs` หากมีการบังคับเพิ่มหน้าในภายหลัง
- เอกสารภายในอาจกล่าวถึงพาธของรีโป ชื่อแอปส่วนตัว ชื่อรายการ 1Password และ runbook ได้ แต่ห้ามใส่ค่าความลับ

## i18n ของเอกสาร

- เอกสารภาษาต่างประเทศไม่ได้รับการดูแลในรีโปนี้ เอาต์พุตสำหรับเผยแพร่ที่สร้างขึ้นอยู่ในรีโป `openclaw/docs` แยกต่างหาก (มักโคลนไว้ในเครื่องเป็น `../openclaw-docs`)
- อย่าเพิ่มหรือแก้ไขเอกสารที่แปลแล้วภายใต้ `docs/<locale>/**` ที่นี่
- ถือว่าเอกสารภาษาอังกฤษในรีโปนี้รวมถึงไฟล์อภิธานศัพท์เป็นแหล่งข้อมูลหลัก
- Pipeline: อัปเดตเอกสารภาษาอังกฤษที่นี่ อัปเดต `docs/.i18n/glossary.<locale>.json` ตามจำเป็น จากนั้นให้การซิงก์รีโปเผยแพร่และ `scripts/docs-i18n` ทำงานใน `openclaw/docs`
- ก่อนรัน `scripts/docs-i18n` อีกครั้ง ให้เพิ่มรายการอภิธานศัพท์สำหรับคำศัพท์ทางเทคนิค ชื่อหน้า หรือป้ายกำกับนำทางสั้น ๆ ใหม่ใด ๆ ที่ต้องคงเป็นภาษาอังกฤษหรือใช้คำแปลตายตัว
- `pnpm docs:check-i18n-glossary` เป็นตัวป้องกันสำหรับชื่อเอกสารภาษาอังกฤษที่เปลี่ยนแปลงและป้ายกำกับเอกสารภายในแบบสั้น
- หน่วยความจำการแปลอยู่ในไฟล์ `docs/.i18n/*.tm.jsonl` ที่สร้างขึ้นในรีโปเผยแพร่
- ดู `docs/.i18n/README.md`
