OpenClaw มีเส้นทางการเริ่มต้นใช้งานสองแบบ ทั้งสองแบบจะกำหนดค่า auth, Gateway และ
ช่องทางแชตที่เป็นตัวเลือก ต่างกันเพียงวิธีที่คุณโต้ตอบกับการตั้งค่า

## ฉันควรใช้เส้นทางใด?

|                | การเริ่มต้นใช้งานผ่าน CLI              | การเริ่มต้นใช้งานผ่านแอป macOS |
| -------------- | -------------------------------------- | ------------------------- |
| **แพลตฟอร์ม**  | macOS, Linux, Windows (แบบ native หรือ WSL2) | macOS เท่านั้น                |
| **อินเทอร์เฟซ**  | วิซาร์ดในเทอร์มินัล                        | UI แบบมีคำแนะนำในแอป      |
| **เหมาะที่สุดสำหรับ**   | เซิร์ฟเวอร์, headless, การควบคุมเต็มรูปแบบ        | Mac เดสก์ท็อป, การตั้งค่าแบบภาพ |
| **ระบบอัตโนมัติ** | `--non-interactive` สำหรับสคริปต์        | ทำด้วยตนเองเท่านั้น               |
| **คำสั่ง**    | `openclaw onboard`                     | เปิดแอป            |

ผู้ใช้ส่วนใหญ่ควรเริ่มด้วย **การเริ่มต้นใช้งานผ่าน CLI** เพราะใช้งานได้ทุกที่และให้
คุณควบคุมได้มากที่สุด

## สิ่งที่การเริ่มต้นใช้งานกำหนดค่า

ไม่ว่าคุณจะเลือกเส้นทางใด การเริ่มต้นใช้งานจะตั้งค่า:

1. **ผู้ให้บริการโมเดลและ auth** — คีย์ API, OAuth หรือโทเค็นการตั้งค่าสำหรับผู้ให้บริการที่คุณเลือก
2. **พื้นที่ทำงาน** — ไดเรกทอรีสำหรับไฟล์ของเอเจนต์ เทมเพลต bootstrap และหน่วยความจำ
3. **Gateway** — พอร์ต, ที่อยู่ bind, โหมด auth
4. **ช่องทาง** (ไม่บังคับ) — ช่องทางแชตในตัวและที่รวมมาด้วย เช่น
   iMessage, Discord, Feishu, Google Chat, Mattermost, Microsoft Teams,
   Telegram, WhatsApp และอื่น ๆ
5. **Daemon** (ไม่บังคับ) — บริการพื้นหลังเพื่อให้ Gateway เริ่มทำงานโดยอัตโนมัติ

## การเริ่มต้นใช้งานผ่าน CLI

เรียกใช้ในเทอร์มินัลใดก็ได้:

```bash
openclaw onboard
```

เพิ่ม `--install-daemon` เพื่อติดตั้งบริการพื้นหลังในขั้นตอนเดียวด้วย

เอกสารอ้างอิงฉบับเต็ม: [การเริ่มต้นใช้งาน (CLI)](/th/start/wizard)
เอกสารคำสั่ง CLI: [`openclaw onboard`](/th/cli/onboard)

## การเริ่มต้นใช้งานผ่านแอป macOS

เปิดแอป OpenClaw วิซาร์ดการเรียกใช้ครั้งแรกจะแนะนำคุณผ่านขั้นตอนเดียวกัน
ด้วยอินเทอร์เฟซแบบภาพ

เอกสารอ้างอิงฉบับเต็ม: [การเริ่มต้นใช้งาน (แอป macOS)](/th/start/onboarding)

## ผู้ให้บริการแบบกำหนดเองหรือไม่อยู่ในรายการ

หากผู้ให้บริการของคุณไม่อยู่ในรายการในการเริ่มต้นใช้งาน ให้เลือก **ผู้ให้บริการแบบกำหนดเอง** และ
ป้อน:

- โหมดความเข้ากันได้ของ API (เข้ากันได้กับ OpenAI, เข้ากันได้กับ Anthropic หรือ auto-detect)
- URL ฐานและคีย์ API
- ID โมเดลและนามแฝงที่เป็นตัวเลือก

ปลายทางแบบกำหนดเองหลายรายการสามารถอยู่ร่วมกันได้ โดยแต่ละรายการจะมี ID ปลายทางของตัวเอง
