OpenClaw มีการเริ่มต้นใช้งานผ่านเทอร์มินัลและแอป macOS ทั้งสองแบบจะตั้งค่าการอนุมานก่อน โดยจะตรวจหาการเข้าถึง AI ที่มีอยู่ กำหนดให้ทดสอบการตอบกลับจริงสำเร็จ และจากนั้นจึงเริ่ม Crestodian เพื่อกำหนดค่าการตั้งค่าที่เหลือ หาก Gateway ที่เข้าถึงได้และกำหนดค่าแล้วมีเอเจนต์เริ่มต้นที่กำหนดค่าโมเดลไว้เรียบร้อย ระบบจะข้ามการเริ่มต้นใช้งานและเปิด UI เอเจนต์ตามปกติ ขั้นตอนผ่านเทอร์มินัลยังมีตัวช่วยแบบคลาสสิกฉบับเต็มสำหรับการตั้งค่าโดยละเอียดอีกด้วย

## ควรใช้วิธีใด?

|                    | การเริ่มต้นใช้งานผ่าน CLI                    | การเริ่มต้นใช้งานผ่านแอป macOS       |
| ------------------ | -------------------------------------------- | ------------------------------------ |
| **แพลตฟอร์ม**      | macOS, Linux, Windows (แบบเนทีฟหรือ WSL2)    | macOS เท่านั้น                       |
| **อินเทอร์เฟซ**    | ตั้งค่าการอนุมาน แล้วจึงใช้ Crestodian       | ตั้งค่าการอนุมาน แล้วจึงใช้ Crestodian |
| **เหมาะสำหรับ**    | เซิร์ฟเวอร์ ระบบไร้หน้าจอ การควบคุมเต็มรูปแบบ | Mac เดสก์ท็อป การตั้งค่าแบบเห็นภาพ   |
| **การทำงานอัตโนมัติ** | `--non-interactive` สำหรับสคริปต์         | ทำด้วยตนเองเท่านั้น                  |
| **คำสั่ง**         | `openclaw onboard`                           | เปิดแอป                              |

ผู้ใช้ส่วนใหญ่ควรเริ่มด้วย **การเริ่มต้นใช้งานผ่าน CLI** เพราะทำงานได้ทุกที่และให้คุณควบคุมได้มากที่สุด

## การเริ่มต้นใช้งานกำหนดค่าอะไรบ้าง

ขั้นตอนการอนุมานแบบมีคำแนะนำจะตั้งค่าเฉพาะสิ่งต่อไปนี้:

1. **ผู้ให้บริการโมเดลและการยืนยันตัวตน** — การเข้าถึงที่ตรวจพบหรือคีย์ API ที่ตรวจสอบแล้ว
2. **การอนุมานที่ตรวจสอบแล้ว** — การตอบกลับจริงจากโมเดลที่มีผลใช้งานของเอเจนต์เริ่มต้น

หลังจากการตอบกลับนั้นผ่านแล้ว Crestodian สามารถกำหนดค่าพื้นที่ทำงาน, Gateway, บริการ Gateway, ช่องทาง, เอเจนต์, ปลั๊กอิน และคุณสมบัติเสริมอื่นๆ ได้

ตัวช่วย CLI แบบคลาสสิกสามารถกำหนดค่าเพิ่มเติมได้ดังนี้:

1. **ช่องทาง** (ไม่บังคับ) — ช่องทางแชตที่มีมาในตัวและรวมมาให้ เช่น Discord, Feishu, Google Chat, iMessage, Mattermost, Microsoft Teams, Telegram, WhatsApp และอื่นๆ
2. **การควบคุม Gateway ขั้นสูง** — โหมดระยะไกล การตั้งค่าเครือข่าย และตัวเลือกดีมอน

## การเริ่มต้นใช้งานผ่าน CLI

เรียกใช้ในเทอร์มินัลใดก็ได้:

```bash
openclaw onboard
```

ขั้นตอนแบบมีคำแนะนำจะตรวจหาการเข้าถึง AI ที่มีอยู่ ทดสอบตัวเลือกจริงตามลำดับ เปลี่ยนไปใช้ตัวเลือกถัดไปเมื่อไม่สำเร็จ และให้ป้อนคีย์ด้วยตนเองโดยซ่อนค่า ระบบจะบันทึกโมเดลและข้อมูลประจำตัวหลังจากการตอบกลับผ่านแล้วเท่านั้น จากนั้นจึงเริ่ม Crestodian เพื่อกำหนดค่าพื้นที่ทำงาน, Gateway, ช่องทาง, เอเจนต์, ปลั๊กอิน และคุณสมบัติเสริมอื่นๆ ไม่มี Crestodian ก่อนการอนุมาน ไม่มีเส้นทางข้าม AI และไม่มีการส่งต่อไปยังกระบวนการแบบคลาสสิกภายในขั้นตอนนี้ หากต้องการใช้ตัวช่วยแบบคลาสสิกแทน ให้ออกแล้วเรียกใช้ `openclaw onboard --classic`

หลังจากการอนุมานผ่านแล้ว Crestodian สามารถส่งต่อการตั้งค่าช่องทางไปยังตัวช่วยในเทอร์มินัลที่ซ่อนค่าละเอียดอ่อนได้ โดยจะไม่เปิดการตั้งค่าผู้ให้บริการแบบมีคำแนะนำหรือแบบคลาสสิก หากต้องการเปลี่ยนผู้ให้บริการโมเดลหรือการยืนยันตัวตนของผู้ให้บริการ ให้ออกจาก Crestodian แล้วเรียกใช้ `openclaw onboard`

ใช้ `openclaw onboard --classic` สำหรับการตั้งค่าโมเดล/การยืนยันตัวตน ช่องทาง Skills Gateway ระยะไกล หรือการนำเข้าโดยละเอียด การเพิ่ม `--install-daemon` จะเลือกขั้นตอนแบบคลาสสิกและติดตั้งบริการเบื้องหลังในขั้นตอนเดียวด้วย ใช้ `openclaw crestodian` สำหรับการตั้งค่าและซ่อมแซมที่ไม่เกี่ยวกับการอนุมานผ่านการสนทนา `openclaw onboard --modern` เป็นนามแฝงเพื่อความเข้ากันได้ซึ่งใช้ด่านตรวจสอบการอนุมานจริงเดียวกัน

ข้อมูลอ้างอิงฉบับเต็ม: [การเริ่มต้นใช้งาน (CLI)](/th/start/wizard)
เอกสารคำสั่ง CLI: [`openclaw onboard`](/th/cli/onboard)

## การเริ่มต้นใช้งานผ่านแอป macOS

เปิดแอป OpenClaw หาก Gateway ภายในเครื่องหรือระยะไกลที่กำหนดค่าไว้สามารถเข้าถึงได้ และเอเจนต์เริ่มต้นมีการกำหนดค่าโมเดลไว้แล้ว แอปจะข้ามการเริ่มต้นใช้งานและ Crestodian แล้วเปิด UI เอเจนต์ตามปกติทันที

สำหรับ Gateway ใหม่หรือที่กำหนดค่าไม่สมบูรณ์ ขั้นตอนการเรียกใช้ครั้งแรกจะตรวจหาการเข้าถึง AI ที่มีอยู่ (Claude Code, Codex หรือคีย์ API) ทดสอบตัวเลือกที่ดีที่สุดจริง และบันทึกหลังจากได้รับการตอบกลับจริงแล้วเท่านั้น โดยจะเปลี่ยนไปใช้ตัวเลือกสำรองโดยอัตโนมัติ และเสนอขั้นตอนป้อนคีย์ API ด้วยตนเองพร้อมการตรวจสอบเมื่อไม่พบตัวเลือกใด ข้อมูลประจำตัวที่ละเอียดอ่อนจะใช้ช่องป้อนข้อมูลที่ซ่อนค่า เมื่อการอนุมานผ่านแล้ว Crestodian จะเริ่มทำงานและช่วยกำหนดค่าส่วนที่เหลือ

Gemini CLI ยังคงใช้กับเอเจนต์ปกติได้หลังการตั้งค่า แต่จะไม่ถูกเสนอสำหรับด่านตรวจสอบการอนุมานนี้ เนื่องจากไม่สามารถบังคับใช้การตรวจสอบที่ไม่ใช้เครื่องมือได้

ข้อมูลอ้างอิงฉบับเต็ม: [การเริ่มต้นใช้งาน (แอป macOS)](/th/start/onboarding)

## ผู้ให้บริการแบบกำหนดเองหรือที่ไม่อยู่ในรายการ

หากผู้ให้บริการของคุณไม่อยู่ในรายการ ให้เรียกใช้ `openclaw onboard --classic` เลือก **ผู้ให้บริการแบบกำหนดเอง** แล้วป้อนข้อมูลต่อไปนี้:

- ความเข้ากันได้ของปลายทาง: เข้ากันได้กับ OpenAI (`/chat/completions`), เข้ากันได้กับ OpenAI Responses (`/responses`), เข้ากันได้กับ Anthropic (`/messages`) หรือไม่ทราบ (ตรวจสอบทั้งสามแบบและตรวจหาโดยอัตโนมัติ)
- URL ฐานและคีย์ API (คีย์ API ไม่บังคับ หากปลายทางไม่กำหนดให้ใช้)
- ID โมเดลและนามแฝงโมเดลซึ่งระบุหรือไม่ก็ได้

ปลายทางแบบกำหนดเองหลายรายการสามารถอยู่ร่วมกันได้ โดยแต่ละรายการจะมี ID ปลายทางของตนเอง
