OpenClaw มีขั้นตอนเริ่มต้นใช้งานผ่านเทอร์มินัลและแอป macOS ทั้งสองวิธีจะตั้งค่าการอนุมานก่อน:
โดยตรวจหาการเข้าถึง AI ที่มีอยู่ กำหนดให้ทดสอบการตอบกลับจริงสำเร็จ แล้วจึงเริ่ม
OpenClaw เพื่อกำหนดค่าการตั้งค่าที่เหลือ Gateway ที่เข้าถึงได้และกำหนดค่าแล้ว
ซึ่งเอเจนต์เริ่มต้นมีโมเดลที่กำหนดค่าไว้แล้ว จะข้ามขั้นตอนเริ่มต้นใช้งานและเปิด
UI ปกติของเอเจนต์ ขั้นตอนผ่านเทอร์มินัลยังมีวิซาร์ดแบบคลาสสิกฉบับเต็มสำหรับ
การตั้งค่าโดยละเอียดด้วย

## ควรใช้วิธีใด?

|                | การเริ่มต้นใช้งานผ่าน CLI                         | การเริ่มต้นใช้งานผ่านแอป macOS           |
| -------------- | -------------------------------------- | ------------------------------ |
| **แพลตฟอร์ม**  | macOS, Linux, Windows (แบบเนทีฟหรือ WSL2) | macOS เท่านั้น                     |
| **อินเทอร์เฟซ**  | ตั้งค่าการอนุมาน แล้วจึง OpenClaw         | ตั้งค่าการอนุมาน แล้วจึง OpenClaw |
| **เหมาะที่สุดสำหรับ**   | เซิร์ฟเวอร์, ระบบไร้หน้าจอ, การควบคุมเต็มรูปแบบ        | Mac แบบเดสก์ท็อป, การตั้งค่าแบบภาพ      |
| **ระบบอัตโนมัติ** | `--non-interactive` สำหรับสคริปต์        | ทำด้วยตนเองเท่านั้น                    |
| **คำสั่ง**    | `openclaw onboard`                     | เปิดแอป                 |

ผู้ใช้ส่วนใหญ่ควรเริ่มด้วย **การเริ่มต้นใช้งานผ่าน CLI** — ใช้งานได้ทุกที่และให้
การควบคุมมากที่สุด

## สิ่งที่ขั้นตอนเริ่มต้นใช้งานกำหนดค่า

ระยะการตั้งค่าการอนุมานแบบมีคำแนะนำจะกำหนดเฉพาะ:

1. **ผู้ให้บริการโมเดลและการยืนยันตัวตน** — การเข้าถึงที่ตรวจพบหรือการลงชื่อเข้าใช้ผู้ให้บริการที่ยืนยันแล้ว,
   คีย์ API หรือโทเค็น
2. **การอนุมานที่ยืนยันแล้ว** — การตอบกลับจริงด้วยโมเดลที่มีผลใช้งาน
   ของเอเจนต์เริ่มต้น

หลังจากการตอบกลับนั้นผ่าน OpenClaw จะสามารถกำหนดค่าพื้นที่ทำงาน, Gateway,
บริการ Gateway, ช่องทาง, เอเจนต์, Plugin และคุณสมบัติเสริมอื่นๆ ได้

วิซาร์ด CLI แบบคลาสสิกยังกำหนดค่าต่อไปนี้เพิ่มเติมได้:

1. **ช่องทาง** (ไม่บังคับ) — ช่องทางแชตในตัวและที่รวมมาให้ เช่น
   Discord, Feishu, Google Chat, iMessage, Mattermost, Microsoft Teams,
   Telegram, WhatsApp และอื่นๆ
2. **การควบคุม Gateway ขั้นสูง** — โหมดระยะไกล, การตั้งค่าเครือข่าย และตัวเลือกดีมอน

## การเริ่มต้นใช้งานผ่าน CLI

เรียกใช้ในเทอร์มินัลใดก็ได้:

```bash
openclaw onboard
```

ขั้นตอนแบบมีคำแนะนำจะตรวจหาการเข้าถึง AI ที่มีอยู่ ทดสอบตัวเลือกจริงตามลำดับ
และเปลี่ยนไปใช้ตัวเลือกถัดไปเมื่อทดสอบไม่ผ่าน หากตรวจหาจนครบแล้ว ระบบจะแสดง OpenAI,
Anthropic, xAI (Grok), Google และ OpenRouter ก่อน **More…** ประกอบด้วย
ผู้ให้บริการที่เหลือซึ่งจัดเป็นกลุ่มผู้ให้บริการ พร้อมภูมิภาค แผน และวิธีที่รองรับผ่าน
เบราว์เซอร์ อุปกรณ์ คีย์ API หรือโทเค็นในเมนูที่สอง ระบบจะบันทึกโมเดล
และข้อมูลประจำตัวหลังจากการตอบกลับผ่านแล้วเท่านั้น จากนั้นจึงเริ่ม OpenClaw เพื่อ
กำหนดค่าพื้นที่ทำงาน, Gateway, ช่องทาง, เอเจนต์, Plugin และคุณสมบัติเสริม
อื่นๆ **Skip for now** จะออกโดยไม่เริ่ม OpenClaw ไม่มี
การส่งต่อไปยังแบบคลาสสิกภายในขั้นตอนนี้ หากต้องการใช้
วิซาร์ดแบบคลาสสิกแทน ให้ออกแล้วเรียกใช้ `openclaw onboard --classic`

หลังจากการอนุมานผ่าน OpenClaw สามารถส่งต่อการตั้งค่าช่องทางไปยังวิซาร์ด
เทอร์มินัลที่ปิดบังข้อมูลได้ โดยจะไม่เปิดการตั้งค่าผู้ให้บริการแบบมีคำแนะนำหรือแบบคลาสสิก หากต้องการ
เปลี่ยนผู้ให้บริการโมเดลหรือการยืนยันตัวตน ให้ออกจาก OpenClaw แล้ว
เรียกใช้ `openclaw onboard`

ใช้ `openclaw onboard --classic` สำหรับการตั้งค่าโมเดล/การยืนยันตัวตน, ช่องทาง, Skills,
Gateway ระยะไกล หรือการนำเข้าโดยละเอียด การเพิ่ม `--install-daemon` จะเลือก
ขั้นตอนแบบคลาสสิกและติดตั้งบริการเบื้องหลังในขั้นตอนเดียวด้วย ใช้ `openclaw
openclaw` สำหรับการตั้งค่าและซ่อมแซมแบบสนทนาที่ไม่เกี่ยวกับการอนุมาน `openclaw
onboard --modern` เป็นนามแฝงเพื่อความเข้ากันได้ที่ใช้ด่าน
การอนุมานจริงเดียวกัน

เอกสารอ้างอิงฉบับเต็ม: [การเริ่มต้นใช้งาน (CLI)](/th/start/wizard)
เอกสารคำสั่ง CLI: [`openclaw onboard`](/th/cli/onboard)

## การเริ่มต้นใช้งานผ่านแอป macOS

เปิดแอป OpenClaw หาก Gateway ภายในเครื่องหรือระยะไกลที่กำหนดค่าไว้สามารถเข้าถึงได้
และเอเจนต์เริ่มต้นมีโมเดลที่กำหนดค่าไว้แล้ว แอปจะข้ามขั้นตอนเริ่มต้นใช้งาน
และ OpenClaw แล้วเปิด UI ปกติของเอเจนต์ทันที

สำหรับ Gateway ใหม่หรือที่ตั้งค่ายังไม่สมบูรณ์ ขั้นตอนการเรียกใช้ครั้งแรกจะตรวจหาการเข้าถึง
AI ที่มีอยู่ (Claude Code, Codex หรือคีย์ API) ทดสอบตัวเลือกที่ดีที่สุด
จริง และบันทึกหลังจากได้รับการตอบกลับจริงแล้วเท่านั้น — โดยเปลี่ยนไปใช้ตัวเลือกสำรองโดยอัตโนมัติและ
เสนอขั้นตอนป้อนคีย์ API ด้วยตนเองที่ผ่านการยืนยันเมื่อไม่พบตัวเลือกใด ข้อมูลประจำตัวที่ละเอียดอ่อน
จะป้อนผ่านช่องที่ปิดบังข้อมูล เมื่อการอนุมานผ่านแล้ว OpenClaw จะเริ่มทำงานและ
ช่วยกำหนดค่าส่วนที่เหลือ

Gemini CLI ยังคงใช้กับเอเจนต์ปกติได้หลังการตั้งค่า แต่จะไม่มี
ตัวเลือกนี้สำหรับด่านการอนุมาน เนื่องจากไม่สามารถบังคับใช้การตรวจสอบที่ไม่ใช้เครื่องมือได้

เอกสารอ้างอิงฉบับเต็ม: [การเริ่มต้นใช้งาน (แอป macOS)](/th/start/onboarding)

## ผู้ให้บริการแบบกำหนดเองหรือที่ไม่อยู่ในรายการ

หากผู้ให้บริการไม่อยู่ในรายการ ให้เรียกใช้ `openclaw onboard --classic` เลือก
**Custom Provider** แล้วป้อน:

- ความเข้ากันได้ของปลายทาง: เข้ากันได้กับ OpenAI (`/chat/completions`), เข้ากันได้กับ OpenAI Responses (`/responses`), เข้ากันได้กับ Anthropic (`/messages`) หรือไม่ทราบ (ตรวจสอบทั้งสามแบบและตรวจหาโดยอัตโนมัติ)
- URL ฐานและคีย์ API (คีย์ API เป็นตัวเลือกหากปลายทางไม่กำหนดให้ใช้)
- รหัสโมเดลและชื่อแทนโมเดลที่ไม่บังคับ

ปลายทางแบบกำหนดเองหลายรายการสามารถอยู่ร่วมกันได้ — แต่ละรายการจะมีรหัสปลายทางของตนเอง
