ขั้นตอนการเรียกใช้ครั้งแรกของแอป macOS: เลือกตำแหน่งที่ Gateway ทำงาน เชื่อมต่อแบ็กเอนด์ AI ที่ตรวจสอบแล้ว ให้สิทธิ์ และส่งต่อไปยังกระบวนการบูตสแตรปของเอเจนต์เอง
สำหรับการเริ่มต้นใช้งานผ่าน CLI และการเปรียบเทียบทั้งสองแนวทาง โปรดดู[ภาพรวมการเริ่มต้นใช้งาน](/th/start/onboarding-overview)

โมเดลความไว้วางใจด้านความปลอดภัย:

- โดยค่าเริ่มต้น OpenClaw เป็นเอเจนต์ส่วนบุคคล: มีขอบเขตผู้ดำเนินการที่เชื่อถือได้เพียงคนเดียว
- การตั้งค่าแบบใช้ร่วมกัน/หลายผู้ใช้จำเป็นต้องมีการจำกัดอย่างเข้มงวด: แยกขอบเขตความไว้วางใจ จำกัดการเข้าถึงเครื่องมือให้น้อยที่สุด และปฏิบัติตาม[ความปลอดภัย](/th/gateway/security)
- การเริ่มต้นใช้งานภายในจะกำหนดค่าเริ่มต้นของการกำหนดค่าใหม่เป็น `tools.profile: "coding"` เพื่อให้การตั้งค่าใหม่ยังคงใช้เครื่องมือระบบไฟล์/รันไทม์ได้โดยไม่ต้องใช้โปรไฟล์ `full` ที่ไม่มีข้อจำกัด
- หากเปิดใช้ฮุก/Webhook หรือฟีดเนื้อหาอื่นที่ไม่น่าเชื่อถือ ให้ใช้โมเดลสมัยใหม่ระดับประสิทธิภาพสูงและคงนโยบายเครื่องมือ/แซนด์บ็อกซ์ที่เข้มงวดไว้

</Step>
<Step title="ภายในเทียบกับระยะไกล">
<Frame>
<img src="/assets/macos-onboarding/04-choose-gateway.png" alt="" />
</Frame>

**Gateway** ทำงานที่ใด

- **Mac เครื่องนี้ (ภายในเท่านั้น):** การเริ่มต้นใช้งานจะกำหนดค่าการยืนยันตัวตนและบันทึกข้อมูลประจำตัวไว้ภายในเครื่อง
- **ระยะไกล (ผ่าน SSH/Tailnet):** การเริ่มต้นใช้งานจะ**ไม่**กำหนดค่าการยืนยันตัวตนภายในเครื่อง
  ข้อมูลประจำตัวต้องมีอยู่บนโฮสต์ Gateway แล้ว ช่องโทเค็นของ Gateway ระยะไกล
  จะจัดเก็บโทเค็นที่แอป macOS ใช้เชื่อมต่อกับ Gateway นั้น
  ค่า SecretRef ของ `gateway.remote.token` ที่มีอยู่จะถูกเก็บไว้จนกว่าจะ
  แทนที่ค่าเหล่านั้น
- **กำหนดค่าภายหลัง:** ข้ามการตั้งค่าและปล่อยให้แอปยังไม่ได้กำหนดค่า

<Tip>
**เคล็ดลับการยืนยันตัวตนของ Gateway:**

- โหมดการยืนยันตัวตนของ Gateway มีค่าเริ่มต้นเป็น `token` แม้จะผูกกับลูปแบ็ก ดังนั้นไคลเอ็นต์ WS ภายในจึงต้องยืนยันตัวตน
- การตั้งค่า `gateway.auth.mode: "none"` ทำให้ทุกโพรเซสภายในสามารถเชื่อมต่อได้ ใช้เฉพาะบนเครื่องที่เชื่อถือได้อย่างสมบูรณ์เท่านั้น
- ใช้โทเค็นสำหรับการเข้าถึงจากหลายเครื่องหรือการผูกกับอินเทอร์เฟซที่ไม่ใช่ลูปแบ็ก

</Tip>
</Step>
<Step title="CLI">
  การตั้งค่าภายในจะติดตั้ง CLI ส่วนกลาง `openclaw` ผ่าน npm, pnpm หรือ bun
  โดยเลือกใช้ npm ก่อน Node ยังคงเป็นรันไทม์ที่แนะนำสำหรับ Gateway
  การติดตั้งเดิมที่เข้ากันได้จะถูกนำกลับมาใช้
</Step>
<Step title="เชื่อมต่อ AI">
  Gateway ที่เชื่อมต่อและมีการกำหนดค่าโมเดลเอเจนต์แล้วจะข้าม
  หน้านี้ทั้งหมดและเปิด UI ปกติของเอเจนต์ การตั้งค่า OpenClaw และผู้ให้บริการ
  จะทำงานเฉพาะกับ Gateway ใหม่หรือที่กำหนดค่าไม่สมบูรณ์เท่านั้น

เมื่อ Gateway พร้อมแล้ว การเริ่มต้นใช้งานจะค้นหาการเข้าถึง AI ที่มีอยู่แล้ว:
การเข้าสู่ระบบ Claude Code หรือ Codex, `OPENAI_API_KEY` / `ANTHROPIC_API_KEY` หรือ
โมเดลที่ใช้เครื่องมือได้ซึ่งติดตั้งอยู่แล้วในเซิร์ฟเวอร์ Ollama หรือ LM Studio ที่เข้าถึงได้
การตรวจหาจะทำงานบนโฮสต์ Gateway รวมถึงเมื่อแอป macOS เชื่อมต่อกับ
Gateway บน Linux ตัวเลือกที่ดีที่สุดจะได้รับการทดสอบด้วยการเติมข้อความจริงและจะบันทึก
หลังจากตอบกลับแล้วเท่านั้น เมื่อการทดสอบล้มเหลว แอปจะลองใช้ตัวเลือกถัดไปโดยอัตโนมัติ
และแสดงสาเหตุที่ตัวเลือกก่อนหน้าล้มเหลว หากพบหลายตัวเลือก สามารถ
สลับระหว่างตัวเลือกเหล่านั้นก่อนดำเนินการต่อได้ การค้นหาภายในอัตโนมัติจะไม่ดึง
หรือดาวน์โหลดโมเดล

หากต้องการใช้การสมัครสมาชิก Claude เมื่อโฮสต์ Gateway ไม่มีการเข้าสู่ระบบ Claude CLI ให้เรียกใช้
`claude setup-token` บนเครื่องใดก็ได้ที่ติดตั้ง Claude Code จากนั้นวาง
โทเค็นที่พิมพ์ออกมาเป็น **Anthropic setup-token** ใต้ **Connect with an API key or
token**

CLI ของ Gemini, Antigravity, Pi และ OpenCode ที่ติดตั้งไว้จะแสดงเพื่อให้ข้อมูลประกอบ
เมื่อไม่สามารถเลือกเป็นเส้นทางการอนุมานสำหรับการตั้งค่าแบบมีคำแนะนำที่นำกลับมาใช้ได้
Gemini และ Antigravity ไม่สามารถบังคับใช้โพรบการอนุมานแบบไม่ใช้เครื่องมือได้ Pi และ
OpenCode เป็นชุดควบคุมเอเจนต์ทั้งระบบ ไม่ใช่เส้นทางการอนุมานสำหรับการตั้งค่า การผสานรวม
เซสชันของระบบเหล่านี้ต้องมีการตั้งค่ารันไทม์และ Plugin แยกต่างหาก

นอกจากนี้ยังสามารถลงชื่อเข้าใช้ผ่านขั้นตอน OAuth หรือการจับคู่อุปกรณ์ของผู้ให้บริการเองได้
ตัวเลือกในตัวประกอบด้วย OpenAI/ChatGPT, OpenRouter, GitHub Copilot, Google
Gemini CLI, xAI, MiniMax Global และ CN รวมถึง Chutes รายการนี้มาจาก
Plugin ผู้ให้บริการอนุมานข้อความที่ใช้งานอยู่ของ Gateway แทนที่จะเป็นรายการแอปแบบตายตัว
ผู้ให้บริการรายอื่นจึงสามารถเลือกเข้าร่วมได้โดยไม่ต้องเพิ่มโค้ด macOS เฉพาะสำหรับผู้ให้บริการ

ตัวเลือกคีย์/โทเค็นแบบกำหนดเองใช้รีจิสทรีผู้ให้บริการเดียวกัน ในทุกเส้นทาง
ผู้ให้บริการจะระบุโมเดลเริ่มต้นและการกำหนดค่าของตน OpenClaw จะตรวจสอบ
ข้อมูลประจำตัวด้วยการทดสอบจริงแบบเดียวกันก่อนจัดเก็บโปรไฟล์การยืนยันตัวตน ปุ่ม Next
จะยังคงถูกล็อกจนกว่าแบ็กเอนด์หนึ่งรายการจะผ่านการทดสอบ ดังนั้นแชตแรกของเอเจนต์จึงไม่สามารถ
เริ่มต้นได้หากการอนุมานยังไม่ทำงาน หลังจากการตรวจสอบจริงนั้นผ่านแล้ว OpenClaw จะ
พร้อมช่วยกำหนดค่าพื้นที่ทำงาน Gateway ช่องทาง และ
คุณสมบัติเสริมอื่น ๆ ที่เหลือ เมื่อ OpenClaw เสนอรายการตัวเลือกสั้น ๆ แอป
จะแสดงการ์ดตัวเลือกแบบเนทีฟ การเลือกหนึ่งรายการจะส่งตัวเลือกนั้น และ **Skip for
now** จะคงตัวเลือกไว้เป็นทางเลือกเสมอ นอกจากนี้ยังสามารถใช้ OpenClaw ภายหลังได้ที่
Settings → OpenClaw
</Step>
<Step title="นำเข้าความทรงจำ (แสดงเมื่อตรวจพบ)">
สำหรับ Gateway ภายใน การเริ่มต้นใช้งานจะตรวจสอบ Mac เพื่อหาความทรงจำจากเครื่องมือ AI
ที่รองรับ ได้แก่ ความทรงจำอัตโนมัติของ Claude Code ความทรงจำที่รวมแล้วของ Codex และไฟล์ความทรงจำ
ของ Hermes เมื่อพบ หน้านี้จะแสดงแหล่งที่มาแต่ละรายการพร้อมจำนวนความทรงจำ
และให้นำเข้าแหล่งที่มาที่เลือกไปยังพื้นที่ทำงานของเอเจนต์ภายใต้
`memory/imports/` เพื่อเรียกคืนด้วยดัชนี ไฟล์ที่นำเข้าแล้วจะถูกข้าม และ
หน้านี้จะไม่ปรากฏเมื่อไม่มีข้อมูลให้นำเข้า การข้ามสามารถทำได้อย่างปลอดภัย
หน้าการนำเข้าความทรงจำของแดชบอร์ดมีตัวเลือกนำเข้าแบบเดียวกันในภายหลัง พร้อมการควบคุม
แยกตามไฟล์
</Step>
<Step title="สิทธิ์">

<Frame caption="เลือกสิทธิ์ที่ต้องการมอบให้ OpenClaw">
<img src="/assets/macos-onboarding/05-permissions.png" alt="" />
</Frame>

การเริ่มต้นใช้งานจะขอสิทธิ์ TCC สำหรับ: การทำงานอัตโนมัติ (AppleScript), การแจ้งเตือน, การช่วยการเข้าถึง, การบันทึกหน้าจอ, ไมโครโฟน, การรู้จำเสียงพูด, กล้อง และตำแหน่งที่ตั้ง

</Step>
<Step title="เสร็จสิ้น">
  หลังจากการอนุมานผ่าน OpenClaw จะจัดการการตั้งค่าเสริมที่เหลือและสามารถ
  ส่งต่อไปยังแชตปกติของเอเจนต์ได้ เมื่อดำเนินขั้นตอนแนะนำการให้สิทธิ์เสร็จ
  แชตเดียวกันนั้นจะเปิดขึ้น แอปจะไม่สร้างพื้นที่ทำงานหรือเริ่มการสนทนา
  ตั้งค่าเอเจนต์แยกต่างหากก่อน OpenClaw โปรดดู
  [การบูตสแตรป](/th/start/bootstrapping) สำหรับสิ่งที่เกิดขึ้นบนโฮสต์ Gateway
  ระหว่างการโต้ตอบจริงครั้งแรกของเอเจนต์
</Step>
