เอกสารนี้อธิบายโฟลว์การตั้งค่าครั้งแรกแบบ**ปัจจุบัน** เป้าหมายคือประสบการณ์
"วันที่ 0" ที่ราบรื่น: เลือกตำแหน่งที่ Gateway ทำงาน เชื่อมต่อการยืนยันตัวตน เรียกใช้
วิซาร์ด และให้เอเจนต์เริ่มต้นระบบด้วยตัวเอง
สำหรับภาพรวมทั่วไปของเส้นทางการเริ่มใช้งาน โปรดดู [ภาพรวมการเริ่มใช้งาน](/th/start/onboarding-overview)

โมเดลความไว้วางใจด้านความปลอดภัย:

- ตามค่าเริ่มต้น OpenClaw เป็นเอเจนต์ส่วนบุคคล: ขอบเขตผู้ควบคุมที่เชื่อถือได้หนึ่งขอบเขต
- การตั้งค่าแบบแชร์/หลายผู้ใช้ต้องมีการล็อกดาวน์ (แยกขอบเขตความไว้วางใจ จำกัดการเข้าถึงเครื่องมือให้น้อยที่สุด และทำตาม [ความปลอดภัย](/th/gateway/security))
- ตอนนี้การเริ่มใช้งานกำหนดค่าเริ่มต้นของคอนฟิกใหม่เป็น `tools.profile: "coding"` เพื่อให้การตั้งค่าใหม่ภายในเครื่องยังคงมีเครื่องมือระบบไฟล์/รันไทม์ โดยไม่บังคับใช้โปรไฟล์ `full` ที่ไม่จำกัด
- หากเปิดใช้ hooks/webhooks หรือฟีดเนื้อหาอื่นที่ไม่น่าเชื่อถือ ให้ใช้ระดับโมเดลสมัยใหม่ที่แข็งแกร่ง และคงนโยบายเครื่องมือ/แซนด์บ็อกซ์ที่เข้มงวดไว้

<Step title="ภายในเครื่องเทียบกับระยะไกล">

<img src="/assets/macos-onboarding/04-choose-gateway.png" alt="" />
</Frame>

**Gateway** ทำงานที่ไหน?

- **Mac เครื่องนี้ (เฉพาะภายในเครื่อง):** การเริ่มใช้งานสามารถกำหนดค่าการยืนยันตัวตนและเขียนข้อมูลรับรอง
  ไว้ภายในเครื่องได้
- **ระยะไกล (ผ่าน SSH/Tailnet):** การเริ่มใช้งานจะ**ไม่**กำหนดค่าการยืนยันตัวตนภายในเครื่อง;
  ข้อมูลรับรองต้องมีอยู่บนโฮสต์ Gateway ช่องโทเค็น Gateway ระยะไกล
  จะเก็บโทเค็นที่แอป macOS ใช้เชื่อมต่อกับ Gateway นั้น; ค่า
  `gateway.remote.token` ที่ไม่ใช่ข้อความธรรมดาซึ่งมีอยู่เดิมจะถูกเก็บไว้จนกว่าคุณจะแทนที่
  ค่าเหล่านั้น
- **กำหนดค่าภายหลัง:** ข้ามการตั้งค่าและปล่อยให้แอปยังไม่ได้กำหนดค่า

<Tip>
**เคล็ดลับการยืนยันตัวตนของ Gateway:**

- ตอนนี้วิซาร์ดสร้าง**โทเค็น**แม้สำหรับ loopback ดังนั้นไคลเอนต์ WS ภายในเครื่องจึงต้องยืนยันตัวตน
- หากคุณปิดการยืนยันตัวตน กระบวนการภายในเครื่องใดๆ ก็สามารถเชื่อมต่อได้; ใช้เฉพาะบนเครื่องที่เชื่อถือได้อย่างเต็มที่เท่านั้น
- ใช้**โทเค็น**สำหรับการเข้าถึงแบบหลายเครื่องหรือการ bind ที่ไม่ใช่ loopback

</Tip>
</Step>

การเริ่มใช้งานร้องขอสิทธิ์ TCC ที่จำเป็นสำหรับ:

- ระบบอัตโนมัติ (AppleScript)
- การแจ้งเตือน
- การช่วยการเข้าถึง
- การบันทึกหน้าจอ
- ไมโครโฟน
- การรู้จำเสียงพูด
- กล้อง
- ตำแหน่งที่ตั้ง

</Step>
<Step title="CLI">
  <Info>ขั้นตอนนี้เป็นทางเลือก</Info>
  แอปสามารถติดตั้ง CLI `openclaw` แบบทั่วระบบผ่าน npm, pnpm หรือ bun ได้
  โดยจะเลือก npm ก่อน จากนั้น pnpm แล้วจึง bun หากนั่นเป็นตัวจัดการแพ็กเกจ
  เพียงตัวเดียวที่ตรวจพบ สำหรับรันไทม์ Gateway นั้น Node ยังคงเป็นเส้นทางที่แนะนำ
</Step>
<Step title="แชตการเริ่มใช้งาน (เซสชันเฉพาะ)">
  หลังการตั้งค่า แอปจะเปิดเซสชันแชตการเริ่มใช้งานเฉพาะ เพื่อให้เอเจนต์สามารถ
  แนะนำตัวเองและแนะนำขั้นตอนถัดไปได้ วิธีนี้ช่วยแยกคำแนะนำการใช้งานครั้งแรก
  ออกจากการสนทนาปกติของคุณ โปรดดู [การเริ่มต้นระบบ](/th/start/bootstrapping) สำหรับ
  สิ่งที่เกิดขึ้นบนโฮสต์ Gateway ระหว่างการรันเอเจนต์ครั้งแรก
</Step>
</Steps>
