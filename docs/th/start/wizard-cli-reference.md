หน้านี้ครอบคลุมลักษณะการทำงาน เอาต์พุต และการทำงานภายในของกระบวนการเริ่มต้นใช้งานแบบทีละขั้นตอน
สำหรับคำแนะนำแบบทีละขั้น โปรดดู [การเริ่มต้นใช้งาน (CLI)](/th/start/wizard) สำหรับเอกสารอ้างอิงแฟล็ก CLI
ฉบับเต็ม (`--flag` ทุกตัวอย่างแบบไม่โต้ตอบ คำสั่งเฉพาะผู้ให้บริการ)
โปรดดู [`openclaw onboard`](/th/cli/onboard)

## วิซาร์ดทำอะไรบ้าง

โหมดภายในเครื่อง (ค่าเริ่มต้น) จะแนะนำขั้นตอนต่อไปนี้:

- การตั้งค่าโมเดลและการยืนยันตัวตน (Anthropic, OAuth สำหรับการสมัครสมาชิก OpenAI Code, xAI, OpenCode, ปลายทางแบบกำหนดเอง และขั้นตอนการยืนยันตัวตนอื่นๆ ที่ผู้ให้บริการเป็นเจ้าของ)
- ตำแหน่งเวิร์กสเปซและไฟล์บูตสแตรป
- การตั้งค่า Gateway (พอร์ต การผูก การยืนยันตัวตน Tailscale)
- ช่องทางและผู้ให้บริการ (Discord, Feishu, Google Chat, iMessage, Mattermost, Microsoft Teams, QQ Bot, Signal, Slack, Telegram, WhatsApp และช่องทางอื่นๆ ที่รวมมาด้วยหรือเป็น Plugin)
- ผู้ให้บริการค้นหาเว็บ (ไม่บังคับ)
- การติดตั้งดีมอน (LaunchAgent, ยูนิตผู้ใช้ systemd หรืองานตามกำหนดเวลาของ Windows แบบเนทีฟพร้อมใช้โฟลเดอร์ Startup เป็นทางเลือกสำรอง)
- การตรวจสอบสถานะการทำงาน
- การตั้งค่า Skills

โหมดระยะไกลจะกำหนดค่าเครื่องนี้ให้เชื่อมต่อกับ Gateway ที่อยู่ที่อื่น โดยจะ
ไม่ติดตั้งหรือแก้ไขสิ่งใดบนโฮสต์ระยะไกล

## รายละเอียดขั้นตอนภายในเครื่อง

<Steps>
  <Step title="การตรวจหาการกำหนดค่าที่มีอยู่">
    - หากมี `~/.openclaw/openclaw.json` ให้เลือก **เก็บค่าปัจจุบันไว้**, **ตรวจสอบและอัปเดต** หรือ **รีเซ็ตก่อนตั้งค่า**
    - การเรียกใช้วิซาร์ดอีกครั้งจะไม่ล้างข้อมูลใดๆ เว้นแต่จะเลือกรีเซ็ตอย่างชัดเจน (หรือส่ง `--reset`)
    - CLI `--reset` มีค่าเริ่มต้นเป็น `config+creds+sessions`; ใช้ `--reset-scope full` เพื่อลบเวิร์กสเปซด้วย
    - หากการกำหนดค่าไม่ถูกต้องหรือมีคีย์แบบเดิม วิซาร์ดจะหยุดและแจ้งให้เรียกใช้ `openclaw doctor` ก่อนดำเนินการต่อ
    - การรีเซ็ตจะย้ายสถานะไปยังถังขยะ (ไม่ลบโดยตรงโดยเด็ดขาด) และมีขอบเขตให้เลือก:
      - เฉพาะการกำหนดค่า
      - การกำหนดค่า + ข้อมูลรับรอง + เซสชัน
      - รีเซ็ตทั้งหมด (ลบเวิร์กสเปซด้วย)

  </Step>
  <Step title="โมเดลและการยืนยันตัวตน">
    - ตารางตัวเลือกทั้งหมดอยู่ใน [ตัวเลือกการยืนยันตัวตนและโมเดล](#auth-and-model-options)

  </Step>
  <Step title="เวิร์กสเปซ">
    - ค่าเริ่มต้น `~/.openclaw/workspace` (กำหนดค่าได้)
    - สร้างไฟล์เวิร์กสเปซเริ่มต้นที่จำเป็นสำหรับการบูตสแตรปในการเรียกใช้ครั้งแรก
    - เมื่อเรียกใช้อีกครั้ง รายชื่อเอเจนต์ที่มีอยู่จะคงเวิร์กสเปซที่ใช้ร่วมกันทั้งฟลีตไว้ เว้นแต่
      จะยืนยันการย้ายอย่างชัดเจน การเรียกใช้อีกครั้งแบบไม่โต้ตอบจะแสดงคำเตือนและคง
      ค่าปัจจุบันไว้
    - โครงสร้างเวิร์กสเปซ: [เวิร์กสเปซของเอเจนต์](/th/concepts/agent-workspace)

  </Step>
  <Step title="Gateway">
    - แจ้งให้ระบุพอร์ต การผูก โหมดการยืนยันตัวตน และการเปิดให้เข้าถึงผ่าน Tailscale
    - คำแนะนำ: เปิดใช้การยืนยันตัวตนด้วยโทเค็นไว้แม้ใช้ลูปแบ็ก เพื่อให้ไคลเอ็นต์ WS ภายในเครื่องต้องยืนยันตัวตน
    - ในโหมดโทเค็น การตั้งค่าแบบโต้ตอบมีตัวเลือก:
      - **สร้าง/จัดเก็บโทเค็นเป็นข้อความธรรมดา** (ค่าเริ่มต้น)
      - **ใช้ SecretRef** (เลือกใช้)
    - ในโหมดรหัสผ่าน การตั้งค่าแบบโต้ตอบยังรองรับการจัดเก็บเป็นข้อความธรรมดาหรือ SecretRef
    - พาธ SecretRef สำหรับโทเค็นแบบไม่โต้ตอบ: `--gateway-token-ref-env <ENV_VAR>`
      - ต้องมีตัวแปรสภาพแวดล้อมที่ไม่ว่างในสภาพแวดล้อมของกระบวนการเริ่มต้นใช้งาน
      - ใช้ร่วมกับ `--gateway-token` ไม่ได้
    - ปิดใช้การยืนยันตัวตนเฉพาะเมื่อเชื่อถือทุกกระบวนการภายในเครื่องอย่างสมบูรณ์
    - การผูกที่ไม่ใช่ลูปแบ็กยังคงต้องใช้การยืนยันตัวตน

  </Step>
  <Step title="ช่องทาง">
    - [WhatsApp](/th/channels/whatsapp): การเข้าสู่ระบบด้วยคิวอาร์ที่ไม่บังคับ
    - [Telegram](/th/channels/telegram): โทเค็นบอต
    - [Discord](/th/channels/discord): โทเค็นบอต
    - [Google Chat](/th/channels/googlechat): JSON ของบัญชีบริการ + กลุ่มเป้าหมาย Webhook
    - [Mattermost](/th/channels/mattermost): โทเค็นบอต + URL ฐาน
    - [Signal](/th/channels/signal): การติดตั้ง `signal-cli` ที่ไม่บังคับ + การกำหนดค่าบัญชี
    - [iMessage](/th/channels/imessage): พาธ CLI ของ `imsg` + สิทธิ์เข้าถึงฐานข้อมูล Messages; ใช้ตัวครอบ SSH เมื่อ Gateway ทำงานนอก Mac
    - ความปลอดภัยของ DM: ค่าเริ่มต้นคือการจับคู่ DM แรกจะส่งรหัสมา ให้อนุมัติผ่าน
      `openclaw pairing approve <channel> <code>` หรือใช้รายการอนุญาต
  </Step>
  <Step title="การค้นหาเว็บ">
    - เลือกผู้ให้บริการ (Brave, DuckDuckGo, Exa, Firecrawl, Gemini, Grok, Kimi, MiniMax Search, Ollama Web Search, Perplexity, SearXNG, Tavily) หรือข้าม
    - ข้ามขั้นตอนนี้ด้วย `--skip-search`; กำหนดค่าใหม่ภายหลังด้วย `openclaw configure --section web`

  </Step>
  <Step title="การติดตั้งดีมอน">
    - macOS: LaunchAgent
      - ต้องมีเซสชันผู้ใช้ที่เข้าสู่ระบบอยู่ สำหรับระบบแบบไม่มีหน้าจอ ให้ใช้ LaunchDaemon แบบกำหนดเอง (ไม่มีให้มาด้วย)
    - Linux และ Windows ผ่าน WSL2: ยูนิตผู้ใช้ systemd
      - วิซาร์ดจะพยายามใช้ `loginctl enable-linger <user>` เพื่อให้ Gateway ทำงานต่อหลังออกจากระบบ
      - อาจแจ้งให้ใช้ sudo (เขียน `/var/lib/systemd/linger`); โดยจะลองโดยไม่ใช้ sudo ก่อน
    - Windows แบบเนทีฟ: ใช้งานตามกำหนดเวลาก่อน
      - หากไม่ได้รับอนุญาตให้สร้างงาน OpenClaw จะเปลี่ยนไปใช้รายการเข้าสู่ระบบในโฟลเดอร์ Startup สำหรับผู้ใช้แต่ละราย และเริ่ม Gateway ทันที
      - ยังคงแนะนำให้ใช้ Scheduled Tasks เนื่องจากแสดงสถานะตัวควบคุมดูแลได้ดีกว่า
    - การเลือกรันไทม์: ต้องใช้ Node เนื่องจากที่เก็บสถานะรันไทม์มาตรฐานของ OpenClaw ใช้ `node:sqlite`

  </Step>
  <Step title="การตรวจสอบสถานะการทำงาน">
    - เริ่ม Gateway (หากจำเป็น) และเรียกใช้ `openclaw health`
    - `openclaw status --deep` จะเพิ่มการตรวจสอบสถานะ Gateway ที่กำลังทำงานลงในเอาต์พุตสถานะ รวมถึงการตรวจสอบช่องทางเมื่อรองรับ

  </Step>
  <Step title="Skills">
    - อ่าน Skills ที่มีอยู่และตรวจสอบข้อกำหนด
    - ให้เลือกตัวจัดการ Node: npm, pnpm หรือ bun
    - ติดตั้งการขึ้นต่อกันที่ไม่บังคับสำหรับ Skills ที่เชื่อถือได้และรวมมาด้วย เมื่อมี
      โปรแกรมติดตั้งที่จำเป็น
    - ข้ามโปรแกรมติดตั้ง Homebrew, uv และ Go ที่ไม่พร้อมใช้งาน แล้วจัดกลุ่ม Skills ที่ได้รับผลกระทบ
      พร้อมคำแนะนำการตั้งค่าด้วยตนเอง เรียกใช้ `openclaw doctor` หลังจากติดตั้ง
      ข้อกำหนดเบื้องต้นที่ขาดหายไป

  </Step>
  <Step title="เสร็จสิ้น">
    - ข้อมูลสรุปและขั้นตอนถัดไป รวมถึงตัวเลือกแอปสำหรับ iOS, Android และ macOS

  </Step>
</Steps>

<Note>
หากตรวจไม่พบ GUI วิซาร์ดจะแสดงคำแนะนำการส่งต่อพอร์ต SSH สำหรับ Control UI แทนการเปิดเบราว์เซอร์
หากแอสเซ็ต Control UI ขาดหายไป วิซาร์ดจะพยายามสร้างแอสเซ็ตเหล่านั้น โดยมีทางเลือกสำรองคือ `pnpm ui:build` (ติดตั้งการขึ้นต่อกันของ UI โดยอัตโนมัติ)
</Note>

## รายละเอียดโหมดระยะไกล

โหมดระยะไกลจะกำหนดค่าเครื่องนี้ให้เชื่อมต่อกับ Gateway ที่อยู่ที่อื่น โดยจะ
ไม่ติดตั้งหรือแก้ไขสิ่งใดบนโฮสต์ระยะไกล

สิ่งที่ต้องตั้งค่า:

- URL ของ Gateway ระยะไกล (`ws://...` หรือ `wss://...`)
- โทเค็น รหัสผ่าน หรือไม่มีการยืนยันตัวตน ให้ตรงกับการกำหนดค่าของ Gateway ระยะไกล

<Steps>
  <Step title="การค้นหา (ไม่บังคับ)">
    หากมี `dns-sd` (macOS) หรือ `avahi-browse` (Linux) กระบวนการเริ่มต้นใช้งาน
    จะเสนอให้ค้นหาบีคอน Gateway ของ Bonjour/mDNS ก่อนเปลี่ยนไปใช้
    การป้อน URL ด้วยตนเอง นอกจากนี้จะพยายามค้นหาผ่าน DNS-SD แบบพื้นที่กว้างเมื่อ
    กำหนดค่าไว้ เอกสาร: [การค้นหา Gateway](/th/gateway/discovery), [Bonjour](/th/gateway/bonjour)
  </Step>
  <Step title="วิธีการเชื่อมต่อ">
    เมื่อเลือกบีคอนแล้ว ให้เลือก WebSocket โดยตรงหรืออุโมงค์ SSH:
    - **โดยตรง**: เชื่อมต่อผ่าน `wss://` และแจ้งให้เชื่อถือลายนิ้วมือ
      TLS ที่ค้นพบ (การปักหมุดแบบเชื่อถือเมื่อใช้งานครั้งแรก โดยจะปักหมุดเฉพาะเมื่อยอมรับ)
    - **อุโมงค์ SSH**: แสดงคำสั่ง `ssh -N -L 18789:127.0.0.1:18789 <user>@<host>`
      ที่ต้องเรียกใช้ก่อน แล้วจึงเชื่อมต่อกับปลายทางของอุโมงค์ภายในเครื่อง
  </Step>
  <Step title="การยืนยันตัวตน">
    เลือกโทเค็น (แนะนำ) รหัสผ่าน หรือไม่มีการยืนยันตัวตน จากนั้นเลือกได้ว่าจะจัดเก็บ
    เป็น SecretRef แทนข้อความธรรมดาหรือไม่
  </Step>
</Steps>

<Note>
หาก Gateway ใช้เฉพาะลูปแบ็กและค้นหาไม่พบ ให้ใช้อุโมงค์ SSH หรือ tailnet ด้วยตนเอง
ยอมรับ `ws://` แบบข้อความธรรมดาสำหรับลูปแบ็ก ลิเทอรัล IP ส่วนตัว `.local` และ URL `*.ts.net` ของ Tailnet; ชื่อ DNS ส่วนตัวอื่นๆ ต้องใช้ `OPENCLAW_ALLOW_INSECURE_PRIVATE_WS=1`
</Note>

## ตัวเลือกการยืนยันตัวตนและโมเดล

หากขั้นตอนการตั้งค่าผู้ให้บริการล้มเหลวระหว่างการเริ่มต้นใช้งานแบบโต้ตอบ (ตัวอย่างเช่น ตัวเลือกการใช้ CLI ซ้ำ
โดยไม่มีการเข้าสู่ระบบภายในเครื่อง) วิซาร์ดจะแสดงข้อผิดพลาดและย้อนกลับไปยังตัวเลือกผู้ให้บริการ
แทนที่จะออกจากโปรแกรม การเรียกใช้ `--auth-choice` อย่างชัดเจนยังคงล้มเหลวทันทีสำหรับระบบอัตโนมัติ

<AccordionGroup>
  <Accordion title="คีย์ API ของ Anthropic">
    ใช้ `ANTHROPIC_API_KEY` หากมีอยู่ หรือแจ้งให้ป้อนคีย์ จากนั้นบันทึกไว้สำหรับการใช้งานของดีมอน
  </Accordion>
  <Accordion title="Anthropic Claude CLI">
    พาธภายในเครื่องที่แนะนำในการเริ่มต้นใช้งาน/กำหนดค่าแบบโต้ตอบ โดยใช้การเข้าสู่ระบบ Claude CLI ที่มีอยู่ซ้ำเมื่อพร้อมใช้งาน
  </Accordion>
  <Accordion title="การสมัครสมาชิก OpenAI Code (OAuth)">
    ขั้นตอนผ่านเบราว์เซอร์ ให้วาง `code#state`

    ในการตั้งค่าใหม่ที่ยังไม่มีโมเดลหลัก จะตั้งค่า `agents.defaults.model` เป็น
    `openai/gpt-5.6-sol` ผ่านรันไทม์ Codex

ลักษณะการทำงานของโมเดล:

- เลือกโมเดลเริ่มต้นจากตัวเลือกที่ตรวจพบ หรือป้อนผู้ให้บริการและโมเดลด้วยตนเอง
- เมื่อการเริ่มต้นใช้งานเริ่มจากตัวเลือกการยืนยันตัวตนของผู้ให้บริการ ตัวเลือกโมเดลจะให้ความสำคัญกับ
  ผู้ให้บริการนั้นโดยอัตโนมัติ สำหรับ Volcengine และ BytePlus การกำหนดลักษณะเดียวกันนี้
  จะตรงกับตัวเลือก coding plan ของผู้ให้บริการเหล่านั้นด้วย (`volcengine-plan/*`,
  `byteplus-plan/*`)
- หากตัวกรองผู้ให้บริการที่ต้องการนั้นทำให้ไม่มีรายการ ตัวเลือกจะกลับไปใช้
  แค็ตตาล็อกทั้งหมดแทนการไม่แสดงโมเดลใดเลย
- ตัวช่วยสร้างจะตรวจสอบโมเดลและเตือนหากไม่รู้จักโมเดลที่กำหนดค่าไว้หรือขาดการยืนยันตัวตน

เส้นทางข้อมูลประจำตัวและโปรไฟล์:

- โปรไฟล์การยืนยันตัวตน (คีย์ API + OAuth): `~/.openclaw/agents/<agentId>/agent/auth-profiles.json`
- การนำเข้า OAuth แบบเดิม: `~/.openclaw/credentials/oauth.json`

โหมดการจัดเก็บข้อมูลประจำตัว:

- ลักษณะเริ่มต้นของการเริ่มต้นใช้งานจะบันทึกคีย์ API เป็นค่าข้อความธรรมดาในโปรไฟล์การยืนยันตัวตน
- `--secret-input-mode ref` เปิดใช้โหมดการอ้างอิงแทนการจัดเก็บคีย์เป็นข้อความธรรมดา
  ในการตั้งค่าแบบโต้ตอบ สามารถเลือกได้ดังนี้:
  - การอ้างอิงตัวแปรสภาพแวดล้อม (ตัวอย่างเช่น `keyRef: { source: "env", provider: "default", id: "OPENAI_API_KEY" }`)
  - การอ้างอิงผู้ให้บริการที่กำหนดค่าไว้ (`file` หรือ `exec`) พร้อมนามแฝงผู้ให้บริการ + ID
- โหมดการอ้างอิงแบบโต้ตอบจะดำเนินการตรวจสอบล่วงหน้าอย่างรวดเร็วก่อนบันทึก
  - การอ้างอิงตัวแปรสภาพแวดล้อม: ตรวจสอบชื่อตัวแปร + ค่าที่ไม่ว่างเปล่าในสภาพแวดล้อมการเริ่มต้นใช้งานปัจจุบัน
  - การอ้างอิงผู้ให้บริการ: ตรวจสอบการกำหนดค่าผู้ให้บริการและค้นหาค่าของ ID ที่ร้องขอ
  - หากการตรวจสอบล่วงหน้าล้มเหลว การเริ่มต้นใช้งานจะแสดงข้อผิดพลาดและให้ลองอีกครั้ง
- ในโหมดไม่โต้ตอบ `--secret-input-mode ref` รองรับผ่านตัวแปรสภาพแวดล้อมเท่านั้น
  - ตั้งค่าตัวแปรสภาพแวดล้อมของผู้ให้บริการในสภาพแวดล้อมของกระบวนการเริ่มต้นใช้งาน
  - แฟล็กคีย์แบบอินไลน์ (ตัวอย่างเช่น `--openai-api-key`) กำหนดให้ต้องตั้งค่าตัวแปรสภาพแวดล้อมนั้น มิฉะนั้นการเริ่มต้นใช้งานจะล้มเหลวทันที
  - สำหรับผู้ให้บริการแบบกำหนดเอง โหมด `ref` แบบไม่โต้ตอบจะจัดเก็บ `models.providers.<id>.apiKey` เป็น `{ source: "env", provider: "default", id: "CUSTOM_API_KEY" }`
  - ในกรณีผู้ให้บริการแบบกำหนดเองดังกล่าว `--custom-api-key` กำหนดให้ต้องตั้งค่า `CUSTOM_API_KEY` มิฉะนั้นการเริ่มต้นใช้งานจะล้มเหลวทันที
- ข้อมูลประจำตัวการยืนยันตัวตนของ Gateway รองรับตัวเลือกข้อความธรรมดาและ SecretRef ในการตั้งค่าแบบโต้ตอบ:
  - โหมดโทเค็น: **สร้าง/จัดเก็บโทเค็นแบบข้อความธรรมดา** (ค่าเริ่มต้น) หรือ **ใช้ SecretRef**
  - โหมดรหัสผ่าน: ข้อความธรรมดาหรือ SecretRef
- เส้นทาง SecretRef ของโทเค็นในโหมดไม่โต้ตอบ: `--gateway-token-ref-env <ENV_VAR>`
- การตั้งค่าแบบข้อความธรรมดาที่มีอยู่จะยังคงใช้งานได้โดยไม่มีการเปลี่ยนแปลง

<Note>
เคล็ดลับสำหรับระบบไร้ส่วนติดต่อและเซิร์ฟเวอร์: ดำเนินการ OAuth ให้เสร็จบนเครื่องที่มีเบราว์เซอร์ จากนั้นคัดลอก

</Note>

### คำแนะนำแอปที่ติดตั้ง

หลังจากการตรวจสอบสิทธิ์เข้าถึงโมเดลสำเร็จ การเริ่มต้นใช้งานแบบโต้ตอบคลาสสิกบน macOS จะสแกนชื่อแอปพลิเคชันและ bundle ID โดยไม่ขอสิทธิ์ความเป็นส่วนตัวของ macOS ระบบจะค้นหาแค็ตตาล็อก Plugin อย่างเป็นทางการและ ClawHub จากนั้นขอให้โมเดลที่กำหนดค่าไว้คัดชื่อที่ตรงกันแบบผิดพลาดออก และแนะนำ Plugin หรือ Skills ที่เกี่ยวข้อง รายการที่แนะนำจะถูกเลือกเป็นค่าเริ่มต้น ส่วนรายการเสริมต้องเลือกอย่างชัดเจน

หน้าจอผลลัพธ์จะแสดงรายการแอปพลิเคชันที่ตรวจพบและข้อความ: "ชื่อแอปได้รับการจับคู่โดยใช้โมเดลที่กำหนดค่าไว้และการค้นหา ClawHub" ตั้งค่า `wizard.appRecommendations` เป็น `false` เพื่อปิดใช้งานทั้งขั้นตอนการเริ่มต้นใช้งานนี้และการเข้าถึงรายการแอปของ Node ผ่าน Gateway การสแกนนี้ไม่ใช้ในการเริ่มต้นอย่างรวดเร็วหรือการเริ่มต้นใช้งานบนระบบที่ไม่ใช่ macOS

## การตั้งค่าแบบไม่โต้ตอบ

`--non-interactive` ต้องใช้ `--accept-risk` (ยอมรับว่าเอเจนต์
มีความสามารถสูงและการเข้าถึงระบบทั้งหมดมีความเสี่ยง):

```bash
openclaw onboard --non-interactive --accept-risk \
  --auth-choice apiKey \
  --anthropic-api-key "$ANTHROPIC_API_KEY"
```

ข้อมูลอ้างอิงแฟล็กทั้งหมดและตัวอย่างเฉพาะผู้ให้บริการ: [`openclaw onboard`](/th/cli/onboard), [การทำงานอัตโนมัติด้วย CLI](/th/start/wizard-cli-automation)

## ลักษณะการตั้งค่า Signal

- ดาวน์โหลดอาร์ติแฟกต์รุ่นที่เหมาะสมจาก GitHub releases อย่างเป็นทางการของ `signal-cli` (บิลด์เนทีฟ สำหรับ Linux x86-64 เท่านั้น)
- บนแพลตฟอร์มอื่น (macOS, Linux ที่ไม่ใช่ x64) จะติดตั้งผ่าน Homebrew แทน
- จัดเก็บการติดตั้งจากอาร์ติแฟกต์รุ่นไว้ภายใต้ `~/.openclaw/tools/signal-cli/<version>/`
- เขียน `channels.signal.cliPath` ในการกำหนดค่า
- ยังไม่รองรับ Windows แบบเนทีฟ; ให้เรียกใช้การเริ่มต้นใช้งานภายใน WSL2 เพื่อใช้เส้นทางการติดตั้งของ Linux
