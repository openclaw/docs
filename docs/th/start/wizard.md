การเริ่มต้นใช้งานผ่าน CLI เป็นเส้นทางการตั้งค่าในเทอร์มินัลที่**แนะนำ**สำหรับ OpenClaw บน
macOS, Linux หรือ Windows ผู้ใช้เดสก์ท็อป Windows ยังสามารถเริ่มด้วย
[Windows Hub](/th/platforms/windows) ได้ด้วย
ขั้นตอนนี้จะกำหนดค่า Gateway ภายในเครื่องหรือการเชื่อมต่อ Gateway ระยะไกล รวมถึงช่องทาง, skills,
และค่าเริ่มต้นของพื้นที่ทำงานในโฟลว์นำทางเดียว

```bash
openclaw onboard
```

QuickStart โดยปกติใช้เวลาเพียงไม่กี่นาที แต่การเริ่มต้นใช้งานแบบเต็มอาจใช้เวลานานกว่า
เมื่อการลงชื่อเข้าใช้ผู้ให้บริการ, การจับคู่ช่องทาง, การติดตั้ง daemon, การดาวน์โหลดผ่านเครือข่าย,
skills หรือ plugins เสริมต้องมีการตั้งค่าเพิ่มเติม ตัวช่วยจะแสดงกรอบเวลานี้ให้ทราบ
ตั้งแต่ต้น และสามารถข้ามขั้นตอนเสริมแล้วกลับมาทำภายหลังได้ด้วย
`openclaw configure`

## ภาษา

ตัวช่วย CLI จะแปลข้อความคงที่ของการเริ่มต้นใช้งาน โดยระบุภาษาจาก
`OPENCLAW_LOCALE` จากนั้น `LC_ALL` จากนั้น `LC_MESSAGES` จากนั้น `LANG` และจะ
ย้อนกลับไปใช้ภาษาอังกฤษ ภาษาของตัวช่วยที่รองรับคือ `en`, `zh-CN` และ `zh-TW`

```bash
OPENCLAW_LOCALE=zh-CN openclaw onboard
```

ชื่อและตัวระบุที่เสถียรจะคงไว้ตามตัวอักษร: `OpenClaw`, `Gateway`, `Tailscale`,
คำสั่ง, คีย์การกำหนดค่า, URL, ID ผู้ให้บริการ, ID โมเดล และป้ายชื่อ plugin/ช่องทาง
จะไม่ถูกแปล

<Info>
แชตแรกที่เร็วที่สุด: เปิด Control UI (ไม่ต้องตั้งค่าช่องทาง) เรียกใช้
`openclaw dashboard` แล้วแชตในเบราว์เซอร์ เอกสาร: [Dashboard](/th/web/dashboard)
</Info>

หากต้องการกำหนดค่าใหม่ภายหลัง:

```bash
openclaw configure
openclaw agents add <name>
```

<Note>
`--json` ไม่ได้หมายถึงโหมดไม่โต้ตอบ สำหรับสคริปต์ ให้ใช้ `--non-interactive`
</Note>

<Tip>
การเริ่มต้นใช้งานผ่าน CLI มีขั้นตอนค้นหาเว็บที่คุณเลือกผู้ให้บริการได้
เช่น Brave, DuckDuckGo, Exa, Firecrawl, Gemini, Grok, Kimi, MiniMax Search,
Ollama Web Search, Perplexity, SearXNG หรือ Tavily ผู้ให้บริการบางรายต้องใช้
คีย์ API ขณะที่บางรายไม่ต้องใช้คีย์ คุณยังสามารถกำหนดค่านี้ภายหลังได้ด้วย
`openclaw configure --section web` เอกสาร: [เครื่องมือเว็บ](/th/tools/web)
</Tip>

## QuickStart เทียบกับขั้นสูง

การเริ่มต้นใช้งานเริ่มด้วย **QuickStart** (ค่าเริ่มต้น) เทียบกับ **ขั้นสูง** (ควบคุมเต็มรูปแบบ)

<Tabs>
  <Tab title="QuickStart (defaults)">
    - Gateway ภายในเครื่อง (loopback)
    - ค่าเริ่มต้นของพื้นที่ทำงาน (หรือพื้นที่ทำงานที่มีอยู่)
    - พอร์ต Gateway **18789**
    - การยืนยันตัวตน Gateway **Token** (สร้างอัตโนมัติ แม้บน loopback)
    - ค่าเริ่มต้นของนโยบายเครื่องมือสำหรับการตั้งค่าภายในเครื่องใหม่: `tools.profile: "coding"` (โปรไฟล์ที่ตั้งไว้อย่างชัดเจนอยู่แล้วจะถูกคงไว้)
    - ค่าเริ่มต้นการแยก DM: การเริ่มต้นใช้งานภายในเครื่องจะเขียน `session.dmScope: "per-channel-peer"` เมื่อยังไม่ได้ตั้งค่า รายละเอียด: [อ้างอิงการตั้งค่า CLI](/th/start/wizard-cli-reference#outputs-and-internals)
    - การเปิดเผยผ่าน Tailscale **ปิด**
    - DM ของ Telegram + WhatsApp มีค่าเริ่มต้นเป็น **รายการที่อนุญาต** (คุณจะได้รับพร้อมท์ให้ใส่หมายเลขโทรศัพท์ของคุณ)

  </Tab>
  <Tab title="Advanced (full control)">
    - แสดงทุกขั้นตอน (โหมด, พื้นที่ทำงาน, gateway, ช่องทาง, daemon, skills)

  </Tab>
</Tabs>

## สิ่งที่การเริ่มต้นใช้งานกำหนดค่า

**โหมดภายในเครื่อง (ค่าเริ่มต้น)** จะพาคุณผ่านขั้นตอนเหล่านี้:

1. **โมเดล/การยืนยันตัวตน** — เลือกผู้ให้บริการ/โฟลว์การยืนยันตัวตนที่รองรับใดก็ได้ (คีย์ API, OAuth หรือการยืนยันตัวตนแบบแมนนวลเฉพาะผู้ให้บริการ) รวมถึงผู้ให้บริการแบบกำหนดเอง
   (เข้ากันได้กับ OpenAI, เข้ากันได้กับ Anthropic หรือการตรวจจับอัตโนมัติแบบไม่ทราบชนิด) เลือกโมเดลเริ่มต้น
   หมายเหตุด้านความปลอดภัย: หาก agent นี้จะเรียกใช้เครื่องมือหรือประมวลผลเนื้อหา webhook/hooks ให้เลือกโมเดลรุ่นล่าสุดที่แข็งแกร่งที่สุดเท่าที่มี และคงนโยบายเครื่องมือให้เข้มงวด ระดับที่อ่อนกว่า/เก่ากว่าจะถูก prompt-inject ได้ง่ายกว่า
   สำหรับการทำงานแบบไม่โต้ตอบ `--secret-input-mode ref` จะเก็บ ref ที่อิง env ในโปรไฟล์การยืนยันตัวตนแทนค่าคีย์ API แบบข้อความธรรมดา
   ในโหมด `ref` แบบไม่โต้ตอบ ต้องตั้งค่าตัวแปร env ของผู้ให้บริการไว้ หากส่งแฟล็กคีย์แบบ inline โดยไม่มีตัวแปร env นั้นจะล้มเหลวทันที
   ในการทำงานแบบโต้ตอบ การเลือกโหมดอ้างอิงความลับช่วยให้คุณชี้ไปยังตัวแปรสภาพแวดล้อมหรือ ref ผู้ให้บริการที่กำหนดค่าไว้ (`file` หรือ `exec`) พร้อมการตรวจสอบ preflight อย่างรวดเร็วก่อนบันทึก
   สำหรับ Anthropic การเริ่มต้นใช้งาน/กำหนดค่าแบบโต้ตอบจะเสนอ **Anthropic Claude CLI** เป็นเส้นทางภายในเครื่องที่ต้องการ และ **คีย์ API ของ Anthropic** เป็นเส้นทางสำหรับโปรดักชันที่แนะนำ Anthropic setup-token ยังคงพร้อมใช้งานในฐานะเส้นทาง token-auth ที่รองรับด้วย
2. **พื้นที่ทำงาน** — ตำแหน่งสำหรับไฟล์ agent (ค่าเริ่มต้น `~/.openclaw/workspace`) เติมไฟล์ bootstrap เริ่มต้น
3. **Gateway** — พอร์ต, bind address, โหมดการยืนยันตัวตน, การเปิดเผยผ่าน Tailscale
   ในโหมด token แบบโต้ตอบ ให้เลือกการเก็บ token แบบข้อความธรรมดาเริ่มต้น หรือเลือกใช้ SecretRef
   เส้นทาง token SecretRef แบบไม่โต้ตอบ: `--gateway-token-ref-env <ENV_VAR>`
4. **ช่องทาง** — ช่องทางแชตในตัวและ plugin ทางการ เช่น iMessage, Discord, Feishu, Google Chat, Mattermost, Microsoft Teams, QQ Bot, Signal, Slack, Telegram, WhatsApp และอื่น ๆ
5. **Daemon** — ติดตั้ง LaunchAgent (macOS), systemd user unit (Linux/WSL2) หรือ Windows Scheduled Task แบบเนทีฟพร้อม fallback เป็นโฟลเดอร์ Startup ต่อผู้ใช้
   หากการยืนยันตัวตนด้วย token ต้องใช้ token และ `gateway.auth.token` ถูกจัดการด้วย SecretRef การติดตั้ง daemon จะตรวจสอบความถูกต้อง แต่จะไม่คง token ที่ resolve แล้วไว้ใน metadata สภาพแวดล้อมของบริการ supervisor
   หากการยืนยันตัวตนด้วย token ต้องใช้ token และ token SecretRef ที่กำหนดค่าไว้ยัง resolve ไม่ได้ การติดตั้ง daemon จะถูกบล็อกพร้อมคำแนะนำที่นำไปปฏิบัติได้
   หากกำหนดค่าทั้ง `gateway.auth.token` และ `gateway.auth.password` และยังไม่ได้ตั้งค่า `gateway.auth.mode` การติดตั้ง daemon จะถูกบล็อกจนกว่าจะตั้งค่าโหมดอย่างชัดเจน
6. **ตรวจสุขภาพ** — เริ่ม Gateway และตรวจสอบว่ากำลังทำงานอยู่
7. **Skills** — ติดตั้ง skills ที่แนะนำและ dependency เสริม

<Note>
การเรียกใช้การเริ่มต้นใช้งานซ้ำจะ**ไม่**ล้างสิ่งใด เว้นแต่คุณจะเลือก **รีเซ็ต** อย่างชัดเจน (หรือส่ง `--reset`)
CLI `--reset` มีค่าเริ่มต้นเป็นการรีเซ็ตการกำหนดค่า, ข้อมูลรับรอง และเซสชัน ใช้ `--reset-scope full` เพื่อรวมพื้นที่ทำงาน
หากการกำหนดค่าไม่ถูกต้องหรือมีคีย์ legacy การเริ่มต้นใช้งานจะขอให้คุณเรียกใช้ `openclaw doctor` ก่อน
</Note>

**โหมดระยะไกล** จะกำหนดค่าเฉพาะไคลเอนต์ภายในเครื่องให้เชื่อมต่อกับ Gateway ที่อื่น
โดยจะ**ไม่**ติดตั้งหรือเปลี่ยนแปลงสิ่งใดบนโฮสต์ระยะไกล

## เพิ่ม agent อีกตัว

ใช้ `openclaw agents add <name>` เพื่อสร้าง agent แยกที่มีพื้นที่ทำงาน,
เซสชัน และโปรไฟล์การยืนยันตัวตนของตัวเอง การเรียกใช้โดยไม่มี `--workspace` จะเปิดการเริ่มต้นใช้งาน

สิ่งที่ตั้งค่า:

- `agents.list[].name`
- `agents.list[].workspace`
- `agents.list[].agentDir`

หมายเหตุ:

- พื้นที่ทำงานเริ่มต้นจะตามรูปแบบ `~/.openclaw/workspace-<agentId>`
- เพิ่ม `bindings` เพื่อกำหนดเส้นทางข้อความขาเข้า (การเริ่มต้นใช้งานทำสิ่งนี้ได้)
- แฟล็กแบบไม่โต้ตอบ: `--model`, `--agent-dir`, `--bind`, `--non-interactive`
