การเริ่มต้นใช้งานผ่าน CLI เป็นวิธีตั้งค่าผ่านเทอร์มินัลที่แนะนำบน macOS, Linux และ
Windows (แบบเนทีฟหรือ WSL2) โดยค่าเริ่มต้น ระบบจะตรวจหาการเข้าถึง AI ที่มีอยู่แล้วบน
เครื่อง ตรวจสอบด้วยการตอบให้เสร็จสมบูรณ์จริง และเริ่ม OpenClaw เพื่อ
กำหนดค่าพื้นที่ทำงาน, Gateway และฟีเจอร์เสริม `openclaw setup` เรียกใช้ขั้นตอนเดียวกัน ([การตั้งค่า](/th/cli/setup) ครอบคลุม
รูปแบบที่กำหนดค่าอย่างเดียวของ `--baseline`) ผู้ใช้เดสก์ท็อป Windows ยังสามารถเริ่มต้น
จาก [Windows Hub](/th/platforms/windows) ได้ด้วย

การเริ่มต้นใช้งานแบบมีคำแนะนำจะตั้งค่าการอนุมานก่อน โดยตรวจหาการเข้าถึง AI ที่พร้อมใช้งาน
กำหนดให้มีการตอบให้เสร็จสมบูรณ์จริง และหลังจากนั้นจึงเริ่ม [OpenClaw](/th/cli/openclaw)
เพื่อกำหนดค่าส่วนที่เหลือของ OpenClaw การเลือก **ข้ามไปก่อน** จะออกจากการเริ่มต้นใช้งาน
โดยไม่เริ่ม OpenClaw

ตัวช่วยแบบคลาสสิกยังคงพร้อมใช้งานสำหรับผู้ให้บริการแบบกำหนดเอง การตั้งค่า Gateway
ระยะไกล การจับคู่ช่องทาง การควบคุมดีมอน Skills และการนำเข้า เรียกใช้โดยระบุ
`openclaw onboard --classic` อย่างชัดเจน ตัวเลือกการอนุมานแบบมีคำแนะนำจะไม่ส่งต่อ
ไปยังตัวช่วยนี้ หลังจากการอนุมานผ่านแล้ว OpenClaw สามารถใช้ `open channel wizard for
<channel>` เพื่อส่งต่อการตั้งค่าช่องทางที่ต้องใช้ข้อมูลลับไปยังตัวช่วยในเทอร์มินัลที่ปกปิดข้อมูล
หากต้องการเปลี่ยนผู้ให้บริการโมเดลหรือการยืนยันตัวตน ให้ออกจาก OpenClaw แล้วเรียกใช้
`openclaw onboard` โดย OpenClaw จะไม่เปิดขั้นตอนของผู้ให้บริการทั้งแบบมีคำแนะนำหรือแบบคลาสสิก

<Info>
วิธีเริ่มแชตครั้งแรกที่เร็วที่สุด: ตั้งค่าแบบมีคำแนะนำให้เสร็จ เรียกใช้ `openclaw dashboard` แล้วแชตใน
เบราว์เซอร์ผ่าน Control UI เอกสาร: [แดชบอร์ด](/th/web/dashboard)
</Info>

## ภาษาและภูมิภาค

ตัวช่วยจะแปลข้อความคงที่ในการเริ่มต้นใช้งาน โดยใช้ค่าที่ไม่ว่างค่าแรกจาก
`OPENCLAW_LOCALE`, `LC_ALL`, `LC_MESSAGES` และ `LANG` ตามลำดับ จากนั้น
จึงใช้ภาษาอังกฤษเป็นค่าเริ่มต้น ภาษาที่รองรับ: `en`, `zh-CN`, `zh-TW`

```bash
OPENCLAW_LOCALE=zh-CN openclaw onboard
OPENCLAW_LOCALE=en openclaw onboard # บังคับใช้ภาษาอังกฤษอย่างชัดเจน
```

ชื่อผลิตภัณฑ์ คำสั่ง คีย์การกำหนดค่า URL, ID ผู้ให้บริการ, ID โมเดล และ
ป้ายกำกับ Plugin/ช่องทางจะยังคงเป็นภาษาอังกฤษไม่ว่าจะใช้ภาษาใด

หากต้องการกำหนดค่าการตั้งค่าที่ไม่เกี่ยวกับการอนุมานใหม่ในภายหลัง:

```bash
openclaw configure
openclaw agents add <name>
```

<Note>
`--json` ไม่ได้หมายความว่าจะเป็นโหมดไม่โต้ตอบ สำหรับสคริปต์ ให้ใช้ `--non-interactive` (ดู [ระบบอัตโนมัติของ CLI](/th/start/wizard-cli-automation))
</Note>

<Tip>
ตัวช่วยแบบคลาสสิกมีขั้นตอนค้นหาเว็บซึ่งคุณสามารถเลือกผู้ให้บริการได้: Brave,
DuckDuckGo, Exa, Firecrawl, Gemini, Grok, Kimi, MiniMax Search, Ollama Web
Search, Perplexity, SearXNG หรือ Tavily บางรายการต้องใช้คีย์ API ส่วนบางรายการ
ไม่ต้องใช้คีย์ กำหนดค่านี้ภายหลังได้ด้วย `openclaw configure --section web` เอกสาร:
[เครื่องมือเว็บ](/th/tools/web)
</Tip>

## ค่าเริ่มต้นแบบมีคำแนะนำ

การเรียก `openclaw onboard` โดยไม่มีตัวเลือกเพิ่มเติมจะทำตามขั้นตอนนี้:

1. ยอมรับประกาศด้านความปลอดภัย
2. ตรวจหาโมเดลที่กำหนดค่าไว้ ตัวแปรสภาพแวดล้อมของคีย์ API, CLI ของ AI ภายในเครื่องที่รองรับ
   และโมเดลที่รองรับเครื่องมือซึ่งติดตั้งไว้แล้วจากเซิร์ฟเวอร์ Ollama หรือ LM
   Studio ที่เข้าถึงได้บนโฮสต์ Gateway การตรวจสอบแบบอ่านอย่างเดียวนี้จะไม่ดาวน์โหลด
   โมเดล นอกจากนี้ยังรายงานการติดตั้ง Gemini CLI, Antigravity, Pi และ OpenCode
   เมื่อไม่สามารถใช้เป็นเส้นทางการอนุมานที่นำกลับมาใช้ได้สำหรับการตั้งค่าแบบมีคำแนะนำ
   Gemini และ Antigravity ไม่สามารถบังคับใช้การตรวจสอบแบบไม่มีเครื่องมือได้ ส่วน Pi และ OpenCode
   เป็นชุดควบคุมเอเจนต์ทั้งระบบ ไม่ใช่เส้นทางการอนุมานสำหรับการตั้งค่า
3. ทดสอบตัวเลือกแรกที่ตรวจพบด้วยการตอบให้เสร็จสมบูรณ์จริง หากล้มเหลว ให้แสดง
   สาเหตุและดำเนินการต่อไปยังตัวเลือกถัดไปที่ใช้งานได้
4. หากตรวจหาจนครบแล้ว ให้เลือก OpenAI, Anthropic, xAI (Grok), Google หรือ
   OpenRouter หรือเลือก **เพิ่มเติม…** สำหรับผู้ให้บริการที่เหลือ ภูมิภาค
   แผนบริการ และวิธีผ่านเบราว์เซอร์ อุปกรณ์ คีย์ API หรือโทเค็นที่รองรับของผู้ให้บริการแต่ละราย
   จะปรากฏในเมนูที่สองและได้รับการทดสอบด้วยการตอบให้เสร็จสมบูรณ์จริงแบบเดียวกัน
   เลือก **ข้ามไปก่อน** เพื่อออกโดยไม่เริ่ม OpenClaw
5. บันทึกเฉพาะเส้นทางโมเดลที่ตรวจสอบแล้วและสถานะข้อมูลประจำตัว/Plugin ใด ๆ ที่
   จำเป็นต้องใช้ การตั้งค่าพื้นที่ทำงานและ Gateway จะไม่ถูกเปลี่ยนแปลง
6. เริ่ม OpenClaw ด้วยโมเดลที่ตรวจสอบแล้ว เพื่อให้กำหนดค่าพื้นที่ทำงาน
   Gateway ช่องทาง เอเจนต์ Plugin และการตั้งค่าเสริมที่เหลือได้

การเรียกใช้คำสั่งซ้ำบนการติดตั้งที่กำหนดค่าแล้วจะทดสอบโมเดลเริ่มต้นปัจจุบัน
ก่อน ทำให้ขั้นตอนแบบมีคำแนะนำเป็นการตรวจสอบและซ่อมแซม การตรวจสอบที่ล้มเหลว
จะไม่แทนที่โมเดลที่กำหนดค่าไว้โดยอัตโนมัติ การเริ่มต้นใช้งานจะหยุดและ
ถามว่าจะดำเนินการต่ออย่างไร เรียกใช้ `openclaw channels add` หรือ `openclaw configure` สำหรับ
การเพิ่มส่วนที่ไม่เกี่ยวกับการอนุมานในภายหลัง ใช้ `openclaw onboard` สำหรับการเปลี่ยน
ผู้ให้บริการหรือเส้นทางการยืนยันตัวตน

## ตัวช่วยแบบคลาสสิก: QuickStart เทียบกับ Advanced

`--flow quickstart` หรือ `--flow advanced` (นามแฝง `manual`) เพื่อเลือกขั้นตอนแบบคลาสสิก
และข้ามคำถามนั้น

## สิ่งที่การเริ่มต้นใช้งานแบบคลาสสิกกำหนดค่า

โหมดภายในเครื่อง (ค่าเริ่มต้น) จะดำเนินการตามขั้นตอนเหล่านี้:

1. **โมเดล/การยืนยันตัวตน** - เลือกขั้นตอนการยืนยันตัวตนของผู้ให้บริการ (คีย์ API, OAuth หรือ
   การยืนยันตัวตนด้วยตนเองเฉพาะผู้ให้บริการ) รวมถึงผู้ให้บริการแบบกำหนดเอง
   (เข้ากันได้กับ OpenAI, เข้ากันได้กับ OpenAI Responses, เข้ากันได้กับ Anthropic หรือ
   ตรวจหาอัตโนมัติแบบไม่ทราบชนิด) เลือกโมเดลเริ่มต้น
   การตั้งค่าคีย์ API ของ OpenAI ใหม่จะใช้ `openai/gpt-5.6` เป็นค่าเริ่มต้น (ID ของ API โดยตรง
   ที่ไม่มีส่วนขยายจะถูกตีความเป็น Sol) การตั้งค่า ChatGPT/Codex ใหม่จะใช้
   `openai/gpt-5.6-sol` เป็นค่าเริ่มต้น การเรียกใช้การตั้งค่าซ้ำจะรักษาโมเดลที่ระบุไว้อย่างชัดเจนซึ่งมีอยู่แล้ว
   รวมถึง `openai/gpt-5.5` เลือก `openai/gpt-5.5` อย่างชัดเจนหาก
   บัญชีไม่มี GPT-5.6
   หมายเหตุด้านความปลอดภัย: หากเอเจนต์นี้จะเรียกใช้เครื่องมือหรือประมวลผลเนื้อหาจาก Webhook/ฮุก
   ให้เลือกโมเดลรุ่นล่าสุดที่มีประสิทธิภาพสูงสุดซึ่งพร้อมใช้งาน และรักษา
   นโยบายเครื่องมือให้เข้มงวด โมเดลระดับที่อ่อนกว่าหรือเก่ากว่าจะถูกโจมตีด้วยการแทรกพรอมต์ได้ง่ายกว่า
   สำหรับการเรียกใช้แบบไม่โต้ตอบ `--secret-input-mode ref` จะจัดเก็บการอ้างอิงที่อิงตัวแปรสภาพแวดล้อม
   แทนค่าคีย์ API แบบข้อความธรรมดา ตัวแปรสภาพแวดล้อมที่อ้างอิงต้องถูก
   ตั้งค่าไว้แล้ว มิฉะนั้นการเริ่มต้นใช้งานจะล้มเหลวทันที โหมดการอ้างอิงข้อมูลลับแบบโต้ตอบสามารถ
   ชี้ไปยังตัวแปรสภาพแวดล้อมหรือการอ้างอิงผู้ให้บริการที่กำหนดค่าไว้ (`file` หรือ
   `exec`) พร้อมการตรวจสอบล่วงหน้าอย่างรวดเร็วก่อนบันทึก หลังจากตั้งค่าโมเดล/การยืนยันตัวตนแล้ว
   ตัวช่วยจะเสนอการทดสอบการตอบให้เสร็จสมบูรณ์แบบสดเป็นตัวเลือก หากล้มเหลว สามารถกลับไปยัง
   การตั้งค่าโมเดล/การยืนยันตัวตนหนึ่งครั้ง หรือเพิกเฉยได้โดยไม่ขัดขวางส่วนที่เหลือของ
   ตัวช่วยแบบคลาสสิก การเพิกเฉยจะไม่ปลดล็อก OpenClaw การตั้งค่าแบบสนทนา
   ยังคงต้องผ่านการตรวจสอบการอนุมาน
2. **พื้นที่ทำงาน** - ไดเรกทอรีสำหรับไฟล์เอเจนต์ (ค่าเริ่มต้น `~/.openclaw/workspace`) สร้างไฟล์เริ่มต้น
3. **Gateway** - พอร์ต ที่อยู่สำหรับผูก โหมดการยืนยันตัวตน การเปิดให้เข้าถึงผ่าน Tailscale ใน
   โหมดโทเค็นแบบโต้ตอบ ให้เลือกจัดเก็บโทเค็นเป็นข้อความธรรมดา (ค่าเริ่มต้น) หรือเลือก
   ใช้ SecretRef เส้นทาง SecretRef แบบไม่โต้ตอบ: `--gateway-token-ref-env <ENV_VAR>`
4. **ช่องทาง** - ช่องทางแชตในตัวและของ Plugin ทางการ รวมถึง
   Discord, Feishu, Google Chat, iMessage, Mattermost, Microsoft Teams,
   QQ Bot, Signal, Slack, Telegram, WhatsApp และอื่น ๆ
5. **ดีมอน** - ติดตั้ง LaunchAgent (macOS), ยูนิตผู้ใช้ systemd
   (Linux/WSL2) หรือ Scheduled Task แบบเนทีฟของ Windows พร้อมวิธีสำรองผ่านโฟลเดอร์
   Startup เฉพาะผู้ใช้
   หากจำเป็นต้องยืนยันตัวตนด้วยโทเค็นและ `gateway.auth.token` อยู่ภายใต้การจัดการของ SecretRef
   การติดตั้งดีมอนจะตรวจสอบ แต่ไม่บันทึกโทเค็นที่แก้ค่าแล้วลงใน
   ข้อมูลเมตาสภาพแวดล้อมของบริการผู้ควบคุม SecretRef ที่แก้ค่าไม่ได้จะขัดขวาง
   การติดตั้งพร้อมแสดงคำแนะนำ หากตั้งค่าทั้ง `gateway.auth.token` และ
   `gateway.auth.password` ขณะที่ยังไม่ได้ตั้งค่า `gateway.auth.mode` การติดตั้ง
   จะถูกขัดขวางจนกว่าจะตั้งค่าโหมดอย่างชัดเจน
6. **การตรวจสอบสถานะ** - เริ่ม Gateway และตรวจสอบว่าสามารถเข้าถึงได้
7. **Skills** - ติดตั้ง Skills ที่แนะนำและการขึ้นต่อกันเสริม

<Note>
การเรียกใช้การเริ่มต้นใช้งานซ้ำจะ **ไม่** ล้างข้อมูลใด ๆ เว้นแต่จะเลือก
**รีเซ็ต** อย่างชัดเจน (หรือส่ง `--reset`) CLI `--reset` ใช้การกำหนดค่า ข้อมูลประจำตัว
และเซสชันเป็นค่าเริ่มต้น ใช้ `--reset-scope full` เพื่อลบพื้นที่ทำงานด้วย หาก
การกำหนดค่าไม่ถูกต้องหรือมีคีย์แบบเก่า การเริ่มต้นใช้งานจะขอให้เรียกใช้
`openclaw doctor` ก่อน
</Note>

`--flow import` เรียกใช้ขั้นตอนการย้ายข้อมูลที่ตรวจพบ (เช่น Hermes) ใน
ตัวช่วยแบบคลาสสิกแทนการตั้งค่าใหม่ ดู [ย้ายข้อมูล](/th/cli/migrate) และคู่มือการย้ายข้อมูลภายใต้
[ติดตั้ง](/th/install/migrating-hermes) `openclaw onboard --modern` เป็น
นามแฝงเพื่อความเข้ากันได้ของ [OpenClaw](/th/cli/openclaw) โดยใช้
เกตการอนุมานเดียวกับ `openclaw setup`: การอนุมานที่ตรวจสอบแล้วจะเริ่ม
ผู้ช่วย ส่วนความล้มเหลวแบบโต้ตอบจะกลับไปยังการตั้งค่าการอนุมานแบบมีคำแนะนำ

## เพิ่มเอเจนต์อีกตัว

ใช้ `openclaw agents add <name>` เพื่อสร้างเอเจนต์แยกต่างหากที่มี
พื้นที่ทำงาน เซสชัน และโปรไฟล์การยืนยันตัวตนของตนเอง การเรียกใช้โดยไม่มี `--workspace` จะเริ่ม
ขั้นตอนแบบโต้ตอบสำหรับชื่อ พื้นที่ทำงาน การยืนยันตัวตน ช่องทาง และการผูก โดย
ไม่ใช่ตัวช่วย `openclaw onboard` แบบเต็ม

ค่าที่ตั้ง:

- `agents.list[].name`
- `agents.list[].workspace`
- `agents.list[].agentDir`

หมายเหตุ:

- พื้นที่ทำงานเริ่มต้น: `~/.openclaw/workspace-<agentId>` (หรืออยู่ภายใต้
  `agents.defaults.workspace` หากมีการตั้งค่าไว้)
- เพิ่ม `bindings` เพื่อกำหนดเส้นทางข้อความขาเข้าไปยังเอเจนต์นี้ (การเริ่มต้นใช้งานสามารถดำเนินการให้ได้)
- แฟล็กแบบไม่โต้ตอบ: `--model`, `--agent-dir`, `--bind`, `--non-interactive`
