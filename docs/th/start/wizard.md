openclaw onboard

การเริ่มต้นใช้งานผ่าน CLI เป็นวิธีตั้งค่าผ่านเทอร์มินัลที่แนะนำบน macOS, Linux และ
Windows (แบบเนทีฟหรือ WSL2) โดยค่าเริ่มต้น ระบบจะตรวจหาการเข้าถึง AI ที่มีอยู่แล้วบน
เครื่อง ตรวจสอบด้วยการสร้างคำตอบจริง และเริ่ม OpenClaw เพื่อ
กำหนดค่าพื้นที่ทำงาน, Gateway และฟีเจอร์เสริม `openclaw setup` เรียกใช้ขั้นตอนเดียวกัน ([การตั้งค่า](/th/cli/setup) อธิบาย
ตัวเลือก `--baseline` ที่กำหนดค่าเพียงอย่างเดียว) ผู้ใช้เดสก์ท็อป Windows ยังสามารถเริ่มต้น
จาก [Windows Hub](/th/platforms/windows) ได้

การเริ่มต้นใช้งานแบบมีคำแนะนำจะตั้งค่าการอนุมานก่อน โดยตรวจหาการเข้าถึง AI ที่พร้อมใช้งาน
กำหนดให้สร้างคำตอบจริงสำเร็จ และหลังจากนั้นจึงเริ่ม [OpenClaw](/cli/openclaw)
เพื่อกำหนดค่าส่วนที่เหลือของ OpenClaw การเลือก **ข้ามไปก่อน** จะออกจากการเริ่มต้นใช้งาน
โดยไม่เริ่ม OpenClaw

วิซาร์ดแบบคลาสสิกยังคงพร้อมใช้งานสำหรับผู้ให้บริการแบบกำหนดเอง การตั้งค่า Gateway
ระยะไกล การจับคู่ช่องทาง การควบคุมดีมอน Skills และการนำเข้า เรียกใช้โดยระบุ
`openclaw onboard --classic` อย่างชัดเจน ตัวเลือกการอนุมานแบบมีคำแนะนำจะไม่ส่งต่อ
ไปยังวิซาร์ดนี้ หลังจากการอนุมานผ่านแล้ว OpenClaw สามารถใช้ `open channel wizard for
<channel>` เพื่อส่งต่อการตั้งค่าช่องทางที่ต้องใช้ข้อมูลลับให้แก่วิซาร์ดเทอร์มินัลที่ปกปิดข้อมูล
หากต้องการเปลี่ยนผู้ให้บริการโมเดลหรือการตรวจสอบสิทธิ์ ให้ออกจาก OpenClaw แล้วเรียกใช้
`openclaw onboard` โดย OpenClaw จะไม่เปิดขั้นตอนผู้ให้บริการแบบมีคำแนะนำหรือแบบคลาสสิก

<Info>
วิธีเริ่มแชตครั้งแรกที่เร็วที่สุด: ตั้งค่าแบบมีคำแนะนำให้เสร็จ เรียกใช้ `openclaw dashboard` แล้วแชตใน
เบราว์เซอร์ผ่าน UI ควบคุม เอกสาร: [แดชบอร์ด](/th/web/dashboard)
</Info>

## ภาษาและภูมิภาค

วิซาร์ดจะแปลข้อความคงที่สำหรับการเริ่มต้นใช้งาน ลำดับการพิจารณา: `OPENCLAW_LOCALE`,
`LC_ALL`, `LC_MESSAGES`, `LANG` แล้วจึงใช้ภาษาอังกฤษ ภาษาที่รองรับ: `en`,
`zh-CN`, `zh-TW`

```bash
OPENCLAW_LOCALE=zh-CN openclaw onboard
```

ชื่อผลิตภัณฑ์ คำสั่ง คีย์การกำหนดค่า URL, ID ผู้ให้บริการ, ID โมเดล และ
ป้ายกำกับ Plugin/ช่องทางจะยังคงเป็นภาษาอังกฤษไม่ว่าตั้งค่าภาษาใด

หากต้องการกำหนดค่าที่ไม่เกี่ยวกับการอนุมานใหม่ภายหลัง:

```bash
openclaw configure
openclaw agents add <name>
```

<Note>
`--json` ไม่ได้หมายถึงโหมดไม่โต้ตอบ สำหรับสคริปต์ ให้ใช้ `--non-interactive` (ดู [การทำงานอัตโนมัติด้วย CLI](/th/start/wizard-cli-automation))
</Note>

<Tip>
วิซาร์ดแบบคลาสสิกมีขั้นตอนค้นหาเว็บที่สามารถเลือกผู้ให้บริการได้ ได้แก่ Brave,
DuckDuckGo, Exa, Firecrawl, Gemini, Grok, Kimi, MiniMax Search, Ollama Web
Search, Perplexity, SearXNG หรือ Tavily บางรายต้องใช้คีย์ API ส่วนรายอื่น
ไม่ต้องใช้คีย์ กำหนดค่านี้ภายหลังด้วย `openclaw configure --section web` เอกสาร:
[เครื่องมือเว็บ](/th/tools/web)
</Tip>

## ค่าเริ่มต้นแบบมีคำแนะนำ

`openclaw onboard` แบบธรรมดาจะดำเนินตามเส้นทางนี้:

1. ยอมรับประกาศด้านความปลอดภัย
2. ตรวจหาโมเดลที่กำหนดค่าไว้ ตัวแปรสภาพแวดล้อมสำหรับคีย์ API, CLI ของ AI ภายในเครื่อง
   ที่รองรับ และโมเดลที่รองรับเครื่องมือซึ่งติดตั้งไว้แล้วจากเซิร์ฟเวอร์ Ollama หรือ LM
   Studio ที่เข้าถึงได้บนโฮสต์ Gateway การตรวจสอบแบบอ่านอย่างเดียวนี้จะไม่ดาวน์โหลด
   โมเดล ระบบจะรายงานการติดตั้ง Gemini CLI และ Antigravity แต่จะไม่ทดสอบ
   โดยอัตโนมัติ เนื่องจากไม่สามารถบังคับใช้การตรวจสอบที่ไม่ใช้เครื่องมือได้
3. ทดสอบตัวเลือกแรกที่ตรวจพบด้วยการสร้างคำตอบจริง หากล้มเหลว ให้แสดง
   สาเหตุและดำเนินการต่อไปยังตัวเลือกถัดไปที่ใช้งานได้
4. หากตรวจหาจนครบแล้ว ให้เลือก OpenAI, Anthropic, xAI (Grok), Google หรือ
   OpenRouter หรือเลือก **เพิ่มเติม…** สำหรับผู้ให้บริการที่เหลือ ภูมิภาค
   แผน และวิธีการผ่านเบราว์เซอร์ อุปกรณ์ คีย์ API หรือโทเค็นที่รองรับของผู้ให้บริการแต่ละราย
   จะปรากฏในเมนูที่สองและได้รับการทดสอบด้วยการสร้างคำตอบจริงแบบเดียวกัน
   เลือก **ข้ามไปก่อน** เพื่อออกโดยไม่เริ่ม OpenClaw
5. บันทึกเฉพาะเส้นทางโมเดลที่ผ่านการตรวจสอบและสถานะข้อมูลประจำตัว/Plugin
   ที่จำเป็นเท่านั้น การตั้งค่าพื้นที่ทำงานและ Gateway จะไม่ถูกเปลี่ยนแปลง
6. เริ่ม OpenClaw ด้วยโมเดลที่ผ่านการตรวจสอบ เพื่อให้สามารถกำหนดค่าพื้นที่ทำงาน,
   Gateway, ช่องทาง, เอเจนต์, Plugin และการตั้งค่าเสริมที่เหลือ

การเรียกใช้คำสั่งซ้ำในการติดตั้งที่กำหนดค่าแล้วจะทดสอบโมเดลเริ่มต้นปัจจุบัน
ก่อน ทำให้ขั้นตอนแบบมีคำแนะนำเป็นการตรวจสอบและซ่อมแซม การตรวจสอบที่ล้มเหลว
จะไม่แทนที่โมเดลที่กำหนดค่าไว้โดยอัตโนมัติ การเริ่มต้นใช้งานจะหยุดและ
ถามว่าจะดำเนินการต่ออย่างไร เรียกใช้ `openclaw channels add` หรือ `openclaw configure` สำหรับ
การเพิ่มส่วนที่ไม่เกี่ยวกับการอนุมานภายหลัง ใช้ `openclaw onboard` สำหรับการเปลี่ยน
ผู้ให้บริการหรือเส้นทางการตรวจสอบสิทธิ์

## วิซาร์ดแบบคลาสสิก: QuickStart เทียบกับ Advanced

เรียกใช้ `openclaw onboard --classic` เพื่อเปิดวิซาร์ดแบบเต็ม โดยเริ่มจาก
การเลือกระหว่าง **QuickStart** (ค่าเริ่มต้น) และ **Advanced** (ควบคุมทั้งหมด) ส่ง
`--flow quickstart` หรือ `--flow advanced` (ชื่อแทน `manual`) เพื่อเลือกขั้นตอนแบบคลาสสิก
และข้ามพรอมต์นั้น

<Tabs>
  <Tab title="QuickStart (ค่าเริ่มต้น)">
    - Gateway ภายในเครื่อง ผูกกับลูปแบ็ก
    - พื้นที่ทำงานเริ่มต้น (หรือพื้นที่ทำงานที่มีอยู่)
    - พอร์ต Gateway **18789**
    - การตรวจสอบสิทธิ์ Gateway **โทเค็น** (สร้างโดยอัตโนมัติ แม้ใช้ลูปแบ็ก)
    - นโยบายเครื่องมือ: `tools.profile: "coding"` สำหรับการตั้งค่าใหม่ (โปรไฟล์ที่ระบุไว้อย่างชัดเจนซึ่งมีอยู่แล้วจะถูกเก็บไว้)
    - การแยก DM: `session.dmScope: "per-channel-peer"` สำหรับการตั้งค่าใหม่ รายละเอียด: [ข้อมูลอ้างอิงการตั้งค่า CLI](/th/start/wizard-cli-reference#outputs-and-internals)
    - การเปิดให้เข้าถึงผ่าน Tailscale **ปิด**
    - DM ของ Telegram และ WhatsApp ใช้ **รายการอนุญาต** เป็นค่าเริ่มต้น: Telegram จะขอ ID ผู้ใช้ Telegram ที่เป็นตัวเลข ส่วน WhatsApp จะขอหมายเลขโทรศัพท์

  </Tab>
  <Tab title="Advanced (ควบคุมทั้งหมด)">
    - แสดงทุกขั้นตอน: โหมด พื้นที่ทำงาน Gateway ช่องทาง ดีมอน Skills

  </Tab>
</Tabs>

โหมดระยะไกล (`--mode remote`) จะใช้ขั้นตอนขั้นสูงเสมอ โดยจะ
กำหนดค่าเฉพาะเครื่องนี้ให้เชื่อมต่อกับ Gateway ที่อื่น และจะไม่ติดตั้ง
หรือเปลี่ยนแปลงสิ่งใดบนโฮสต์ระยะไกล

## สิ่งที่การเริ่มต้นใช้งานแบบคลาสสิกกำหนดค่า

โหมดภายในเครื่อง (ค่าเริ่มต้น) จะดำเนินตามขั้นตอนเหล่านี้:

1. **โมเดล/การตรวจสอบสิทธิ์** - เลือกขั้นตอนการตรวจสอบสิทธิ์ของผู้ให้บริการ (คีย์ API, OAuth หรือ
   การตรวจสอบสิทธิ์ด้วยตนเองเฉพาะผู้ให้บริการ) รวมถึงผู้ให้บริการแบบกำหนดเอง
   (เข้ากันได้กับ OpenAI, เข้ากันได้กับ OpenAI Responses, เข้ากันได้กับ Anthropic หรือ
   ตรวจหาอัตโนมัติแบบไม่ทราบประเภท) เลือกโมเดลเริ่มต้น
   การตั้งค่าคีย์ API ของ OpenAI ใหม่จะใช้ `openai/gpt-5.6` เป็นค่าเริ่มต้น (ID ของ API โดยตรง
   แบบไม่มีส่วนขยายจะถูกแปลงเป็น Sol) ส่วนการตั้งค่า ChatGPT/Codex ใหม่จะใช้
   `openai/gpt-5.6-sol` เป็นค่าเริ่มต้น การเรียกใช้การตั้งค่าซ้ำจะเก็บโมเดลที่ระบุไว้อย่างชัดเจนซึ่งมีอยู่แล้ว
   รวมถึง `openai/gpt-5.5` เลือก `openai/gpt-5.5` อย่างชัดเจนหาก
   บัญชีไม่ให้สิทธิ์เข้าถึง GPT-5.6
   หมายเหตุด้านความปลอดภัย: หากเอเจนต์นี้จะเรียกใช้เครื่องมือหรือประมวลผลเนื้อหา
   จาก webhook/hook ให้เลือกโมเดลรุ่นล่าสุดที่ทรงประสิทธิภาพที่สุดซึ่งมีให้ใช้ และคง
   นโยบายเครื่องมือไว้อย่างเข้มงวด เนื่องจากโมเดลระดับที่อ่อนกว่าหรือเก่ากว่าจะถูกแทรกพรอมต์ได้ง่ายกว่า
   สำหรับการเรียกใช้แบบไม่โต้ตอบ `--secret-input-mode ref` จะจัดเก็บข้อมูลอ้างอิง
   ที่อิงตัวแปรสภาพแวดล้อมแทนค่าคีย์ API แบบข้อความธรรมดา ตัวแปรสภาพแวดล้อมที่อ้างถึงต้องถูก
   ตั้งค่าไว้แล้ว มิฉะนั้นการเริ่มต้นใช้งานจะล้มเหลวทันที โหมดอ้างอิงข้อมูลลับแบบโต้ตอบสามารถ
   ชี้ไปยังตัวแปรสภาพแวดล้อมหรือข้อมูลอ้างอิงผู้ให้บริการที่กำหนดค่าไว้ (`file` หรือ
   `exec`) พร้อมการตรวจสอบล่วงหน้าอย่างรวดเร็วก่อนบันทึก หลังจากตั้งค่าโมเดล/การตรวจสอบสิทธิ์แล้ว
   วิซาร์ดจะเสนอการทดสอบการสร้างคำตอบจริงเป็นตัวเลือก หากล้มเหลว สามารถกลับไปยัง
   การตั้งค่าโมเดล/การตรวจสอบสิทธิ์หนึ่งครั้ง หรือเพิกเฉยโดยไม่ขัดขวางส่วนที่เหลือของ
   วิซาร์ดแบบคลาสสิก การเพิกเฉยไม่ได้ปลดล็อก OpenClaw การตั้งค่าผ่านการสนทนา
   ยังคงกำหนดให้การตรวจสอบการอนุมานผ่าน
2. **พื้นที่ทำงาน** - ไดเรกทอรีสำหรับไฟล์เอเจนต์ (ค่าเริ่มต้น `~/.openclaw/workspace`) สร้างไฟล์เริ่มต้น
3. **Gateway** - พอร์ต ที่อยู่สำหรับผูก โหมดการตรวจสอบสิทธิ์ การเปิดให้เข้าถึงผ่าน Tailscale ใน
   โหมดโทเค็นแบบโต้ตอบ ให้เลือกจัดเก็บโทเค็นแบบข้อความธรรมดา (ค่าเริ่มต้น) หรือเลือก
   ใช้ SecretRef เส้นทาง SecretRef แบบไม่โต้ตอบ: `--gateway-token-ref-env <ENV_VAR>`
4. **ช่องทาง** - ช่องทางแชตแบบในตัวและ Plugin ทางการ รวมถึง
   Discord, Feishu, Google Chat, iMessage, Mattermost, Microsoft Teams,
   QQ Bot, Signal, Slack, Telegram, WhatsApp และอื่น ๆ
5. **ดีมอน** - ติดตั้ง LaunchAgent (macOS), ยูนิตผู้ใช้ systemd
   (Linux/WSL2) หรือ Windows Scheduled Task แบบเนทีฟ พร้อมทางเลือกสำรอง
   เป็นโฟลเดอร์ Startup รายผู้ใช้
   หากจำเป็นต้องใช้การตรวจสอบสิทธิ์ด้วยโทเค็นและ `gateway.auth.token` จัดการด้วย SecretRef
   การติดตั้งดีมอนจะตรวจสอบความถูกต้อง แต่จะไม่บันทึกโทเค็นที่แปลงค่าแล้วลงใน
   ข้อมูลเมตาสภาพแวดล้อมของบริการผู้ควบคุม SecretRef ที่แปลงค่าไม่ได้จะขัดขวาง
   การติดตั้งพร้อมแสดงคำแนะนำ หากตั้งค่าทั้ง `gateway.auth.token` และ
   `gateway.auth.password` ขณะที่ยังไม่ได้ตั้งค่า `gateway.auth.mode` การติดตั้ง
   จะถูกขัดขวางจนกว่าจะตั้งค่าโหมดอย่างชัดเจน
6. **การตรวจสอบสถานะ** - เริ่ม Gateway และตรวจสอบว่าสามารถเข้าถึงได้
7. **Skills** - ติดตั้ง Skills ที่แนะนำและการขึ้นต่อกันเสริมของ Skills เหล่านั้น

<Note>
การเรียกใช้การเริ่มต้นใช้งานซ้ำจะ **ไม่** ลบข้อมูลใด เว้นแต่จะเลือก
**รีเซ็ต** อย่างชัดเจน (หรือส่ง `--reset`) CLI `--reset` จะใช้การกำหนดค่า ข้อมูลประจำตัว
และเซสชันเป็นค่าเริ่มต้น ใช้ `--reset-scope full` เพื่อลบพื้นที่ทำงานด้วย หาก
การกำหนดค่าไม่ถูกต้องหรือมีคีย์แบบเดิม การเริ่มต้นใช้งานจะขอให้เรียกใช้
`openclaw doctor` ก่อน
</Note>

`--flow import` จะเรียกใช้ขั้นตอนการย้ายข้อมูลที่ตรวจพบ (เช่น Hermes) ใน
วิซาร์ดแบบคลาสสิกแทนการตั้งค่าใหม่ ดู [ย้ายข้อมูล](/th/cli/migrate) และคู่มือการย้ายข้อมูลภายใต้
[ติดตั้ง](/th/install/migrating-hermes) `openclaw onboard --modern` เป็น
ชื่อแทนเพื่อความเข้ากันได้ของ [OpenClaw](/cli/openclaw) โดยใช้
ด่านตรวจการอนุมานเดียวกับ `openclaw setup`: การอนุมานที่ผ่านการตรวจสอบจะเริ่ม
ผู้ช่วย ส่วนความล้มเหลวแบบโต้ตอบจะกลับไปยังการตั้งค่าการอนุมานแบบมีคำแนะนำ

## เพิ่มเอเจนต์อีกหนึ่งตัว

ใช้ `openclaw agents add <name>` เพื่อสร้างเอเจนต์แยกต่างหากที่มี
พื้นที่ทำงาน เซสชัน และโปรไฟล์การตรวจสอบสิทธิ์ของตนเอง การเรียกใช้โดยไม่มี `--workspace` จะเริ่ม
ขั้นตอนแบบโต้ตอบสำหรับชื่อ พื้นที่ทำงาน การตรวจสอบสิทธิ์ ช่องทาง และการผูก โดย
ไม่ใช่วิซาร์ด `openclaw onboard` แบบเต็ม

สิ่งที่ระบบตั้งค่า:

- `agents.list[].name`
- `agents.list[].workspace`
- `agents.list[].agentDir`

หมายเหตุ:

- พื้นที่ทำงานเริ่มต้น: `~/.openclaw/workspace-<agentId>` (หรืออยู่ภายใต้
  `agents.defaults.workspace` หากมีการตั้งค่านี้)
- เพิ่ม `bindings` เพื่อกำหนดเส้นทางข้อความขาเข้าไปยังเอเจนต์นี้ (การเริ่มต้นใช้งานสามารถดำเนินการนี้ให้ได้)
- แฟล็กแบบไม่โต้ตอบ: `--model`, `--agent-dir`, `--bind`, `--non-interactive`
