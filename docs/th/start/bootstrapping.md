การบูตสแตรปเป็นขั้นตอนเริ่มต้นเมื่อเรียกใช้ครั้งแรก ซึ่งเตรียมพื้นที่ทำงานใหม่ของเอเจนต์และ
แนะนำเอเจนต์ตลอดกระบวนการเลือกอัตลักษณ์ โดยจะทำงานเพียงครั้งเดียวทันทีหลัง
การเริ่มต้นใช้งาน ในการโต้ตอบจริงครั้งแรกของเอเจนต์

## สิ่งที่เกิดขึ้น

เมื่อเรียกใช้ครั้งแรกกับพื้นที่ทำงานใหม่ทั้งหมด (ค่าเริ่มต้น `~/.openclaw/workspace`)
OpenClaw จะดำเนินการดังนี้:

- สร้าง `AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md` และ `BOOTSTRAP.md`
- ให้เอเจนต์ทำตามลำดับการถือกำเนิดที่จำกัดไว้สามจังหวะ ได้แก่ เสนอชื่อของตนเอง
  บอกประโยคสั้น ๆ หนึ่งประโยคเกี่ยวกับจิตวิญญาณ/บรรยากาศของตน และถามว่าต้องการชุด Plugin
  ขั้นต่ำที่แนะนำหรือความสะดวกสูงสุด
- บันทึกอัตลักษณ์ที่ตกลงกันไว้สองแห่ง ได้แก่ ใน `IDENTITY.md` และ `SOUL.md` (สิ่งที่
  เอเจนต์อ่านเกี่ยวกับตนเอง) และผ่าน `openclaw agents set-identity` (สิ่งที่ช่องทาง
  และ UI แสดง)
- อ่านคำแนะนำแอปที่บันทึกไว้แล้วระหว่างการเริ่มต้นใช้งานโดยไม่สแกนซ้ำ
  Plugin อย่างเป็นทางการใช้ `openclaw plugins install <id>`; Skills จาก ClawHub ของบุคคลที่สาม
  ยังคงต้องเลือกเข้าร่วมอย่างชัดแจ้ง หลังจากจัดการตัวเลือกแล้ว เอเจนต์
  จะรับทราบข้อเสนอที่บันทึกไว้ เพื่อไม่ให้ถามอีก
- ลบ `BOOTSTRAP.md` เมื่อพื้นที่ทำงานดูเหมือนได้รับการกำหนดค่าแล้ว เพื่อให้ขั้นตอนนี้ทำงานเพียงครั้งเดียว

พื้นที่ทำงานจะถือว่าได้รับการกำหนดค่าแล้วเมื่อ `SOUL.md`, `IDENTITY.md` หรือ `USER.md`
แตกต่างจากเทมเพลตเริ่มต้น หรือมีโฟลเดอร์ `memory/`

<Note>
`BOOTSTRAP.md` ครอบคลุมบทสนทนาเกี่ยวกับอัตลักษณ์ทั้งหมด ดูเนื้อหาได้ที่
[เทมเพลต BOOTSTRAP.md](/th/reference/templates/BOOTSTRAP)
</Note>

## การทำงานด้วยโมเดลแบบฝังตัวและโมเดลภายในเครื่อง

สำหรับการทำงานด้วยโมเดลแบบฝังตัวหรือโมเดลภายในเครื่อง OpenClaw จะไม่ใส่ `BOOTSTRAP.md` ไว้ใน
บริบทระบบที่มีสิทธิ์สูง ในการทำงานแบบโต้ตอบหลักครั้งแรก ระบบจะยังคง
ส่งเนื้อหาของไฟล์ผ่านพรอมต์ผู้ใช้ เพื่อให้โมเดลที่ไม่ได้เรียกใช้เครื่องมือ `read`
ได้อย่างเชื่อถือได้ยังสามารถดำเนินขั้นตอนนี้จนเสร็จสิ้น หากการทำงานปัจจุบัน
ไม่สามารถเข้าถึงพื้นที่ทำงานได้อย่างปลอดภัย เอเจนต์จะได้รับหมายเหตุการบูตสแตรป
แบบจำกัดสั้น ๆ แทนคำทักทายทั่วไป

## การข้ามการบูตสแตรป

หากต้องการข้ามขั้นตอนนี้ในพื้นที่ทำงานที่เตรียมข้อมูลไว้ล่วงหน้า ให้เรียกใช้:

```bash
openclaw onboard --skip-bootstrap
```

## ตำแหน่งที่ทำงาน

การบูตสแตรปจะทำงานบนโฮสต์ Gateway เสมอ หากแอป macOS เชื่อมต่อกับ
Gateway ระยะไกล พื้นที่ทำงานและไฟล์บูตสแตรปจะอยู่บนเครื่องระยะไกลดังกล่าว
ไม่ใช่บน Mac

<Note>
เมื่อ Gateway ทำงานบนเครื่องอื่น ให้แก้ไขไฟล์พื้นที่ทำงานบนโฮสต์ Gateway
(เช่น `user@gateway-host:~/.openclaw/workspace`)
</Note>
