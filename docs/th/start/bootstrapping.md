การเริ่มต้นระบบเป็นขั้นตอนแรกเมื่อใช้งาน ซึ่งสร้างไฟล์ตั้งต้นให้พื้นที่ทำงานใหม่ของเอเจนต์และ
แนะนำเอเจนต์ตลอดกระบวนการเลือกอัตลักษณ์ โดยจะทำงานเพียงครั้งเดียว ทันทีหลังจาก
การเริ่มใช้งาน ในรอบการทำงานจริงครั้งแรกของเอเจนต์

## สิ่งที่เกิดขึ้น

ในการทำงานครั้งแรกกับพื้นที่ทำงานใหม่ทั้งหมด (ค่าเริ่มต้นคือ `~/.openclaw/workspace`)
OpenClaw จะ:

- สร้างไฟล์ตั้งต้น `AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md` และ `BOOTSTRAP.md`
- ให้เอเจนต์ทำตาม `BOOTSTRAP.md` ซึ่งเป็นการสนทนาแบบอิสระ (ไม่ใช่แบบฟอร์มคำถามและคำตอบตายตัว) เพื่อกำหนดชื่อ บุคลิก และบรรยากาศโดยรวม
- บันทึกสิ่งที่เรียนรู้ลงใน `IDENTITY.md`, `USER.md` และ `SOUL.md`
- ลบ `BOOTSTRAP.md` เมื่อพื้นที่ทำงานดูเหมือนกำหนดค่าเรียบร้อยแล้ว เพื่อให้ขั้นตอนนี้ทำงานเพียงครั้งเดียว

พื้นที่ทำงานจะถือว่ากำหนดค่าแล้วเมื่อ `SOUL.md`, `IDENTITY.md` หรือ `USER.md`
แตกต่างจากเทมเพลตเริ่มต้น หรือมีโฟลเดอร์ `memory/` อยู่

<Note>
`BOOTSTRAP.md` ครอบคลุมการสนทนาเกี่ยวกับอัตลักษณ์ทั้งหมด ดูเนื้อหาได้ที่
[เทมเพลต BOOTSTRAP.md](/th/reference/templates/BOOTSTRAP)
</Note>

## การทำงานด้วยโมเดลแบบฝังและโมเดลภายในเครื่อง

สำหรับการทำงานด้วยโมเดลแบบฝังหรือโมเดลภายในเครื่อง OpenClaw จะไม่นำ `BOOTSTRAP.md`
เข้าสู่บริบทระบบที่มีสิทธิ์พิเศษ ในการทำงานแบบโต้ตอบหลักครั้งแรก ระบบยังคง
ส่งเนื้อหาของไฟล์ผ่านพรอมป์ของผู้ใช้ เพื่อให้โมเดลที่ไม่ได้เรียกใช้เครื่องมือ `read`
อย่างเชื่อถือได้ยังสามารถดำเนินขั้นตอนนี้จนเสร็จสมบูรณ์ได้ หากการทำงานปัจจุบัน
ไม่สามารถเข้าถึงพื้นที่ทำงานได้อย่างปลอดภัย เอเจนต์จะได้รับข้อความสั้น ๆ
สำหรับการเริ่มต้นระบบแบบจำกัดแทนคำทักทายทั่วไป

## การข้ามการเริ่มต้นระบบ

หากต้องการข้ามขั้นตอนนี้สำหรับพื้นที่ทำงานที่มีไฟล์ตั้งต้นอยู่แล้ว ให้เรียกใช้:

```bash
openclaw onboard --skip-bootstrap
```

## ตำแหน่งที่ทำงาน

การเริ่มต้นระบบจะทำงานบนโฮสต์ Gateway เสมอ หากแอป macOS เชื่อมต่อกับ
Gateway ระยะไกล พื้นที่ทำงานและไฟล์เริ่มต้นระบบจะอยู่บนเครื่องระยะไกลนั้น
ไม่ใช่บน Mac

<Note>
เมื่อ Gateway ทำงานบนเครื่องอื่น ให้แก้ไขไฟล์พื้นที่ทำงานบนโฮสต์ Gateway
(ตัวอย่างเช่น `user@gateway-host:~/.openclaw/workspace`)
</Note>
