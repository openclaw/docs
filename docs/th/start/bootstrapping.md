การเริ่มต้นระบบเป็นขั้นตอนครั้งแรกที่ใช้วางไฟล์ตั้งต้นในพื้นที่ทำงานใหม่ของเอเจนต์ และ
แนะนำเอเจนต์ตลอดกระบวนการเลือกอัตลักษณ์ ขั้นตอนนี้ทำงานเพียงครั้งเดียวทันทีหลังจาก
การเริ่มใช้งาน ในเทิร์นการทำงานจริงครั้งแรกของเอเจนต์

## สิ่งที่เกิดขึ้น

ในการทำงานครั้งแรกกับพื้นที่ทำงานใหม่ทั้งหมด (ค่าเริ่มต้น `~/.openclaw/workspace`)
OpenClaw จะดำเนินการดังนี้:

- สร้างไฟล์ตั้งต้น `AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md` และ `BOOTSTRAP.md`
- ให้เอเจนต์ทำตามลำดับการถือกำเนิดสามจังหวะแบบจำกัด: ถามว่าคุณต้องการ
  เรียกเอเจนต์ว่าอะไร บอกข้อความสั้นหนึ่งบรรทัดเกี่ยวกับตัวตน/บรรยากาศ และถามว่าคุณต้องการ
  ชุด Plugin ขั้นต่ำที่แนะนำหรือความสะดวกสูงสุด
- บันทึกอัตลักษณ์ที่ตกลงกันไว้สองส่วน: ลงใน `IDENTITY.md` และ `SOUL.md` (ข้อมูลที่
  เอเจนต์อ่านเกี่ยวกับตัวเอง) และผ่าน `openclaw agents set-identity` (ข้อมูลที่ช่องทางต่างๆ
  และ UI แสดง)
- อ่านคำแนะนำแอปที่จัดเก็บไว้แล้วระหว่างการเริ่มใช้งานโดยไม่สแกนซ้ำ
  Plugin อย่างเป็นทางการใช้ `openclaw plugins install <id>`; Skills จาก ClawHub ของบุคคลที่สาม
  ยังคงต้องเลือกเข้าร่วมอย่างชัดเจน หลังจากจัดการตัวเลือกแล้ว เอเจนต์จะ
  รับทราบข้อเสนอที่จัดเก็บไว้เพื่อไม่ให้ถามอีก
- ลบ `BOOTSTRAP.md` เมื่อพื้นที่ทำงานดูเหมือนกำหนดค่าเรียบร้อยแล้ว เพื่อให้ขั้นตอนนี้ทำงานเพียงครั้งเดียว

พื้นที่ทำงานจะถือว่ากำหนดค่าแล้วเมื่อ `SOUL.md`, `IDENTITY.md` หรือ `USER.md`
แตกต่างจากเทมเพลตเริ่มต้น หรือมีโฟลเดอร์ `memory/` อยู่

<Note>
`BOOTSTRAP.md` ครอบคลุมการสนทนาเรื่องอัตลักษณ์ทั้งหมด ดูเนื้อหาได้ที่
[เทมเพลต BOOTSTRAP.md](/th/reference/templates/BOOTSTRAP)
</Note>

## การทำงานด้วยโมเดลแบบฝังและโมเดลภายในเครื่อง

สำหรับการทำงานด้วยโมเดลแบบฝังหรือโมเดลภายในเครื่อง OpenClaw จะไม่นำ `BOOTSTRAP.md` เข้าไปใน
บริบทระบบที่มีสิทธิ์พิเศษ ในการทำงานครั้งแรกแบบโต้ตอบหลัก ระบบจะยังคง
ส่งเนื้อหาไฟล์ผ่านพรอมต์ของผู้ใช้ เพื่อให้โมเดลที่ไม่ได้เรียกใช้เครื่องมือ `read`
อย่างเชื่อถือได้ยังสามารถดำเนินขั้นตอนนี้จนเสร็จได้ หากการทำงานปัจจุบัน
ไม่สามารถเข้าถึงพื้นที่ทำงานได้อย่างปลอดภัย เอเจนต์จะได้รับหมายเหตุการเริ่มต้นระบบ
แบบจำกัดและกระชับแทนคำทักทายทั่วไป

## การข้ามการเริ่มต้นระบบ

หากต้องการข้ามขั้นตอนนี้ในพื้นที่ทำงานที่มีข้อมูลตั้งต้นอยู่แล้ว ให้เรียกใช้:

```bash
openclaw onboard --skip-bootstrap
```

## ตำแหน่งที่ทำงาน

การเริ่มต้นระบบจะทำงานบนโฮสต์ Gateway เสมอ หากแอป macOS เชื่อมต่อกับ
Gateway ระยะไกล พื้นที่ทำงานและไฟล์เริ่มต้นระบบจะอยู่บนเครื่องระยะไกลดังกล่าว
ไม่ใช่บน Mac

<Note>
เมื่อ Gateway ทำงานบนเครื่องอื่น ให้แก้ไขไฟล์พื้นที่ทำงานบนโฮสต์ Gateway
(ตัวอย่างเช่น `user@gateway-host:~/.openclaw/workspace`)
</Note>
