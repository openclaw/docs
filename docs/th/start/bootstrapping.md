การบูตสแตรปคือขั้นตอน**การรันครั้งแรก**ที่เตรียมพื้นที่ทำงานของเอเจนต์และ
รวบรวมรายละเอียดตัวตน ขั้นตอนนี้เกิดขึ้นหลังการเริ่มใช้งาน เมื่อเอเจนต์เริ่มทำงาน
เป็นครั้งแรก

## การบูตสแตรปทำอะไร

ในการรันเอเจนต์ครั้งแรก OpenClaw จะบูตสแตรปพื้นที่ทำงาน (ค่าเริ่มต้นคือ
`~/.openclaw/workspace`):

- เติมไฟล์เริ่มต้น `AGENTS.md`, `BOOTSTRAP.md`, `IDENTITY.md`, `USER.md`
- รันขั้นตอนถาม-ตอบสั้น ๆ (ทีละหนึ่งคำถาม)
- เขียนตัวตนและค่ากำหนดไปยัง `IDENTITY.md`, `USER.md`, `SOUL.md`
- ลบ `BOOTSTRAP.md` เมื่อเสร็จสิ้น เพื่อให้รันเพียงครั้งเดียว

สำหรับการรันโมเดลแบบฝังตัว/ภายในเครื่อง OpenClaw จะกัน `BOOTSTRAP.md` ออกจาก
บริบทระบบที่มีสิทธิ์พิเศษ ในการรันแบบโต้ตอบครั้งแรกหลัก ระบบยังคงส่ง
เนื้อหาไฟล์ในพรอมป์ผู้ใช้ เพื่อให้โมเดลที่ไม่ได้เรียกใช้เครื่องมือ `read`
อย่างเชื่อถือได้สามารถทำขั้นตอนนี้ให้เสร็จได้ หากการรันปัจจุบันไม่สามารถเข้าถึง
พื้นที่ทำงานได้อย่างปลอดภัย เอเจนต์จะได้รับหมายเหตุการบูตสแตรปแบบจำกัดแทน
คำทักทายทั่วไป

## การข้ามการบูตสแตรป

หากต้องการข้ามขั้นตอนนี้สำหรับพื้นที่ทำงานที่เตรียมไว้ล่วงหน้า ให้รัน `openclaw onboard --skip-bootstrap`

## ตำแหน่งที่รัน

การบูตสแตรปจะรันบน**โฮสต์ Gateway** เสมอ หากแอป macOS เชื่อมต่อกับ
Gateway ระยะไกล พื้นที่ทำงานและไฟล์การบูตสแตรปจะอยู่บนเครื่องระยะไกลนั้น

<Note>
เมื่อ Gateway รันอยู่บนเครื่องอื่น ให้แก้ไขไฟล์พื้นที่ทำงานบนโฮสต์ Gateway
(ตัวอย่างเช่น `user@gateway-host:~/.openclaw/workspace`)
</Note>
