### งานที่ตั้งเวลาไว้ (Cron) เทียบกับ Heartbeat

| มิติ | งานที่ตั้งเวลาไว้ (Cron) | Heartbeat |
| --------------- | ----------------------------------- | ------------------------------------- |
| เวลา | แน่นอน (นิพจน์ cron, ทำครั้งเดียว) | โดยประมาณ (ค่าเริ่มต้นทุก 30 นาที) |
| บริบทเซสชัน | ใหม่ (แยกอิสระ) หรือใช้ร่วมกัน | บริบทเซสชันหลักเต็มรูปแบบ |
| ระเบียนงาน | สร้างเสมอ | ไม่สร้างเลย |
| การส่งมอบ | ช่องทาง, webhook หรือแบบเงียบ | แทรกในเซสชันหลัก |
| เหมาะที่สุดสำหรับ | รายงาน การเตือน งานเบื้องหลัง | การตรวจกล่องขาเข้า ปฏิทิน การแจ้งเตือน |

ใช้งานที่ตั้งเวลาไว้ (Cron) เมื่อคุณต้องการเวลาที่แม่นยำหรือการดำเนินการแบบแยกอิสระ ใช้ Heartbeat เมื่องานได้ประโยชน์จากบริบทเซสชันเต็มรูปแบบและยอมรับเวลาประมาณได้

## แนวคิดหลัก

### งานที่ตั้งเวลาไว้ (cron)

Cron คือเครื่องมือตั้งเวลาในตัวของ Gateway สำหรับเวลาที่แม่นยำ มันเก็บงานถาวร ปลุกเอเจนต์ในเวลาที่ถูกต้อง และสามารถส่งเอาต์พุตไปยังช่องทางแชทหรือปลายทาง webhook ได้ รองรับการเตือนแบบครั้งเดียว นิพจน์ที่เกิดซ้ำ และทริกเกอร์ webhook ขาเข้า

ดู [งานที่ตั้งเวลาไว้](/th/automation/cron-jobs)

### งาน

บัญชีงานเบื้องหลังติดตามงานที่แยกออกมาทั้งหมด: การรัน ACP, การสร้างซับเอเจนต์, การดำเนินการ cron แบบแยกอิสระ และการดำเนินการ CLI งานคือระเบียน ไม่ใช่เครื่องมือตั้งเวลา ใช้ `openclaw tasks list` และ `openclaw tasks audit` เพื่อตรวจสอบ

ดู [งานเบื้องหลัง](/th/automation/tasks)

### ภาระผูกพันที่อนุมานได้

ภาระผูกพันคือหน่วยความจำติดตามผลแบบเลือกใช้และมีอายุสั้น OpenClaw อนุมานจากบทสนทนาปกติ จำกัดขอบเขตกับเอเจนต์และช่องทางเดียวกัน และส่งเช็กอินเมื่อครบกำหนดผ่าน Heartbeat การเตือนที่ผู้ใช้ร้องขอพร้อมเวลาที่แน่นอนยังคงเป็นหน้าที่ของ cron

ดู [ภาระผูกพันที่อนุมานได้](/th/concepts/commitments)

### Task Flow

Task Flow คือชั้นฐานการจัดการโฟลว์เหนือกว่างานเบื้องหลัง มันจัดการโฟลว์หลายขั้นตอนแบบทนทานด้วยโหมดซิงก์ที่จัดการและมิเรอร์ การติดตามรีวิชัน และ `openclaw tasks flow list|show|cancel` สำหรับการตรวจสอบ

ดู [Task Flow](/th/automation/taskflow)

### คำสั่งถาวร

คำสั่งถาวรมอบอำนาจการปฏิบัติงานแบบถาวรให้เอเจนต์สำหรับโปรแกรมที่กำหนดไว้ คำสั่งเหล่านี้อยู่ในไฟล์เวิร์กสเปซ (โดยทั่วไปคือ `AGENTS.md`) และถูกฉีดเข้าไปในทุกเซสชัน ใช้ร่วมกับ cron สำหรับการบังคับใช้ตามเวลา

ดู [คำสั่งถาวร](/th/automation/standing-orders)

### ฮุก

ฮุกภายในคือสคริปต์ที่ขับเคลื่อนด้วยเหตุการณ์ ซึ่งถูกทริกเกอร์โดยเหตุการณ์วงจรชีวิตของเอเจนต์ (`/new`, `/reset`, `/stop`), Compaction ของเซสชัน, การเริ่มทำงานของ Gateway และโฟลว์ข้อความ ฮุกเหล่านี้ถูกค้นพบโดยอัตโนมัติจากไดเรกทอรีและจัดการได้ด้วย `openclaw hooks` สำหรับการสกัดการเรียกเครื่องมือในกระบวนการ ให้ใช้ [ฮุกของ Plugin](/th/plugins/hooks)

ดู [ฮุก](/th/automation/hooks)

### Heartbeat

Heartbeat คือเทิร์นของเซสชันหลักที่เกิดเป็นระยะ (ค่าเริ่มต้นทุก 30 นาที) มันรวมการตรวจหลายอย่าง (กล่องขาเข้า ปฏิทิน การแจ้งเตือน) ไว้ในเทิร์นเอเจนต์เดียวพร้อมบริบทเซสชันเต็มรูปแบบ เทิร์นของ Heartbeat ไม่สร้างระเบียนงานและไม่ขยายความสดใหม่ของการรีเซ็ตเซสชันรายวัน/เมื่อไม่ได้ใช้งาน ใช้ `HEARTBEAT.md` สำหรับเช็กลิสต์ขนาดเล็ก หรือบล็อก `tasks:` เมื่อคุณต้องการการตรวจเป็นระยะเฉพาะรายการที่ครบกำหนดภายใน Heartbeat เอง ไฟล์ Heartbeat ว่างจะข้ามเป็น `empty-heartbeat-file`; โหมดงานเฉพาะรายการครบกำหนดจะข้ามเป็น `no-tasks-due` Heartbeat จะเลื่อนออกไปขณะงาน cron กำลังทำงานหรืออยู่ในคิว และ `heartbeat.skipWhenBusy` ยังสามารถเลื่อนออกไปได้เมื่อซับเอเจนต์หรือเลนซ้อนกำลังยุ่งอยู่

ดู [Heartbeat](/th/gateway/heartbeat)

## วิธีที่สิ่งเหล่านี้ทำงานร่วมกัน

- **Cron** จัดการกำหนดการที่แม่นยำ (รายงานประจำวัน การทบทวนรายสัปดาห์) และการเตือนแบบครั้งเดียว การดำเนินการ cron ทั้งหมดสร้างระเบียนงาน
- **Heartbeat** จัดการการตรวจติดตามตามปกติ (กล่องขาเข้า ปฏิทิน การแจ้งเตือน) ในเทิร์นที่รวมเป็นชุดเดียวทุก 30 นาที
- **ฮุก** ตอบสนองต่อเหตุการณ์เฉพาะ (การรีเซ็ตเซสชัน, Compaction, โฟลว์ข้อความ) ด้วยสคริปต์กำหนดเอง ฮุกของ Plugin ครอบคลุมการเรียกเครื่องมือ
- **คำสั่งถาวร** ให้บริบทถาวรและขอบเขตอำนาจแก่เอเจนต์
- **Task Flow** ประสานโฟลว์หลายขั้นตอนเหนือกว่างานแต่ละรายการ
- **งาน** ติดตามงานที่แยกออกมาทั้งหมดโดยอัตโนมัติ เพื่อให้คุณตรวจสอบและตรวจบัญชีได้
