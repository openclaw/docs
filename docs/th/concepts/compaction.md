ทุกโมเดลมีหน้าต่างบริบท: จำนวนโทเค็นสูงสุดที่สามารถประมวลผลได้ เมื่อการสนทนาเข้าใกล้ขีดจำกัดนั้น OpenClaw จะทำ **Compaction** กับข้อความเก่าให้เป็นสรุป เพื่อให้แชตดำเนินต่อได้

## วิธีทำงาน

1. เทิร์นการสนทนาเก่าจะถูกสรุปเป็นรายการแบบกระชับ
2. สรุปจะถูกบันทึกไว้ในทรานสคริปต์ของเซสชัน
3. ข้อความล่าสุดจะถูกเก็บไว้ครบถ้วน

เมื่อ OpenClaw แบ่งประวัติเป็นชังก์ Compaction จะคงการเรียกใช้เครื่องมือของผู้ช่วยให้จับคู่กับรายการ `toolResult` ที่ตรงกัน หากจุดแบ่งตกอยู่ภายในบล็อกเครื่องมือ OpenClaw จะเลื่อนขอบเขตเพื่อให้คู่ดังกล่าวอยู่ด้วยกัน และคงส่วนท้ายปัจจุบันที่ยังไม่ได้สรุปไว้

ประวัติการสนทนาเต็มยังคงอยู่บนดิสก์ Compaction เปลี่ยนเฉพาะสิ่งที่โมเดลเห็นในเทิร์นถัดไป

## Compaction อัตโนมัติ

Compaction อัตโนมัติเปิดไว้โดยค่าเริ่มต้น โดยจะทำงานเมื่อเซสชันใกล้ถึงขีดจำกัดบริบท หรือเมื่อโมเดลส่งคืนข้อผิดพลาดบริบทล้น ซึ่งในกรณีนั้น OpenClaw จะทำ Compaction แล้วลองใหม่

คุณจะเห็น:

- `🧹 Auto-compaction complete` ในโหมด verbose
- `/status` แสดง `🧹 Compactions: <count>`

<Info>
ก่อนทำ Compaction OpenClaw จะเตือนเอเจนต์โดยอัตโนมัติให้บันทึกโน้ตสำคัญไว้ในไฟล์ [memory](/th/concepts/memory) ซึ่งช่วยป้องกันการสูญเสียบริบท
</Info>

    - `request_too_large`

## Compaction ด้วยตนเอง

พิมพ์ `/compact` ในแชตใดก็ได้เพื่อบังคับทำ Compaction เพิ่มคำสั่งเพื่อกำหนดแนวทางให้สรุป:

```
/compact Focus on the API design decisions
```

เมื่อมีการตั้งค่า `agents.defaults.compaction.keepRecentTokens` Compaction ด้วยตนเองจะใช้จุดตัด Pi นั้น และคงส่วนท้ายล่าสุดไว้ในบริบทที่สร้างใหม่ หากไม่มีงบประมาณเก็บรักษาที่ระบุชัดเจน Compaction ด้วยตนเองจะทำงานเหมือนเช็กพอยต์แบบแข็ง และดำเนินต่อจากสรุปใหม่เพียงอย่างเดียว

## การกำหนดค่า

กำหนดค่า Compaction ภายใต้ `agents.defaults.compaction` ใน `openclaw.json` ของคุณ ตัวเลือกที่ใช้บ่อยที่สุดแสดงอยู่ด้านล่าง สำหรับข้อมูลอ้างอิงฉบับเต็ม โปรดดู [เจาะลึกการจัดการเซสชัน](/th/reference/session-management-compaction)

### ใช้โมเดลอื่น

โดยค่าเริ่มต้น Compaction ใช้โมเดลหลักของเอเจนต์ ตั้งค่า `agents.defaults.compaction.model` เพื่อมอบหมายการสรุปให้โมเดลที่มีความสามารถมากกว่าหรือเฉพาะทางกว่า ค่าแทนที่รับสตริง `provider/model-id` ใดก็ได้:

```json
{
  "agents": {
    "defaults": {
      "compaction": {
        "model": "openrouter/anthropic/claude-sonnet-4-6"
      }
    }
  }
}
```

วิธีนี้ใช้กับโมเดลโลคัลได้ด้วย เช่น โมเดล Ollama ตัวที่สองที่ใช้สำหรับการสรุปโดยเฉพาะ:

```json
{
  "agents": {
    "defaults": {
      "compaction": {
        "model": "ollama/llama3.1:8b"
      }
    }
  }
}
```

เมื่อไม่ได้ตั้งค่า Compaction จะใช้โมเดลหลักของเอเจนต์

### การเก็บรักษาตัวระบุ

การสรุปด้วย Compaction จะเก็บรักษาตัวระบุแบบทึบไว้โดยค่าเริ่มต้น (`identifierPolicy: "strict"`) แทนที่ด้วย `identifierPolicy: "off"` เพื่อปิดใช้ หรือใช้ `identifierPolicy: "custom"` พร้อม `identifierInstructions` สำหรับคำแนะนำแบบกำหนดเอง

### ตัวป้องกันจำนวนไบต์ของทรานสคริปต์ที่ใช้งานอยู่

เมื่อตั้งค่า `agents.defaults.compaction.maxActiveTranscriptBytes` OpenClaw จะทริกเกอร์ Compaction โลคัลตามปกติก่อนการรัน หาก JSONL ที่ใช้งานอยู่มีขนาดถึงค่านั้น สิ่งนี้มีประโยชน์สำหรับเซสชันที่ทำงานยาวนาน ซึ่งการจัดการบริบทฝั่งผู้ให้บริการอาจทำให้บริบทของโมเดลยังปกติ ในขณะที่ทรานสคริปต์โลคัลยังเติบโตต่อไป ฟีเจอร์นี้ไม่ได้แบ่งไบต์ JSONL ดิบ แต่จะขอให้ไปป์ไลน์ Compaction ตามปกติสร้างสรุปเชิงความหมาย

<Warning>
ตัวป้องกันจำนวนไบต์ต้องใช้ `truncateAfterCompaction: true` หากไม่มีการหมุนเวียนทรานสคริปต์ ไฟล์ที่ใช้งานอยู่จะไม่เล็กลง และตัวป้องกันจะยังไม่ทำงาน
</Warning>

### ทรานสคริปต์ตัวสืบทอด

เมื่อเปิดใช้ `agents.defaults.compaction.truncateAfterCompaction` OpenClaw จะไม่เขียนทับทรานสคริปต์เดิมในตำแหน่งเดิม แต่จะสร้างทรานสคริปต์ตัวสืบทอดที่ใช้งานอยู่ชุดใหม่จากสรุป Compaction สถานะที่เก็บรักษาไว้ และส่วนท้ายที่ยังไม่ได้สรุป จากนั้นเก็บ JSONL ก่อนหน้าไว้เป็นแหล่งเช็กพอยต์ที่เก็บถาวร
ทรานสคริปต์ตัวสืบทอดยังตัดเทิร์นผู้ใช้ยาวที่ซ้ำกันทุกประการซึ่งเข้ามา
ภายในหน้าต่าง retry สั้น ๆ เพื่อไม่ให้พายุการ retry ของช่องทางถูกพาเข้าไปใน
ทรานสคริปต์ที่ใช้งานอยู่ตัวถัดไปหลัง Compaction

เช็กพอยต์ก่อน Compaction จะถูกเก็บไว้เฉพาะเมื่อยังอยู่ต่ำกว่าขีดจำกัดขนาด
เช็กพอยต์ของ OpenClaw เท่านั้น ทรานสคริปต์ที่ใช้งานอยู่ขนาดใหญ่เกินไปยังคงทำ Compaction
แต่ OpenClaw จะข้ามสแนปช็อตดีบักขนาดใหญ่แทนการเพิ่มการใช้ดิสก์เป็นสองเท่า

### การแจ้งเตือน Compaction

โดยค่าเริ่มต้น Compaction จะทำงานแบบเงียบ ตั้งค่า `notifyUser` เพื่อแสดงข้อความสถานะสั้น ๆ เมื่อ Compaction เริ่มและเสร็จสมบูรณ์:

```json5
{
  agents: {
    defaults: {
      compaction: {
        notifyUser: true,
      },
    },
  },
}
```

### การฟลัชหน่วยความจำ

ก่อนทำ Compaction OpenClaw สามารถรันเทิร์น **การฟลัชหน่วยความจำแบบเงียบ** เพื่อเก็บโน้ตถาวรลงดิสก์ ตั้งค่า `agents.defaults.compaction.memoryFlush.model` เมื่อเทิร์นดูแลระบบนี้ควรใช้โมเดลโลคัลแทนโมเดลการสนทนาที่ใช้งานอยู่:

```json
{
  "agents": {
    "defaults": {
      "compaction": {
        "memoryFlush": {
          "model": "ollama/qwen3:8b"
        }
      }
    }
  }
}
```

ค่าแทนที่โมเดล memory-flush เป็นค่าแบบตรงตัว และไม่สืบทอดเชน fallback ของเซสชันที่ใช้งานอยู่ ดูรายละเอียดและการกำหนดค่าได้ที่ [หน่วยความจำ](/th/concepts/memory)

## ผู้ให้บริการ Compaction แบบเสียบใช้ได้

Plugins สามารถลงทะเบียนผู้ให้บริการ Compaction แบบกำหนดเองผ่าน `registerCompactionProvider()` บน API ของ Plugin ได้ เมื่อมีการลงทะเบียนและกำหนดค่าผู้ให้บริการแล้ว OpenClaw จะมอบหมายการสรุปให้ผู้ให้บริการนั้นแทนไปป์ไลน์ LLM ในตัว

หากต้องการใช้ผู้ให้บริการที่ลงทะเบียนไว้ ให้ตั้งค่า id ของผู้ให้บริการนั้นใน config ของคุณ:

```json
{
  "agents": {
    "defaults": {
      "compaction": {
        "provider": "my-provider"
      }
    }
  }
}
```

การตั้งค่า `provider` จะบังคับ `mode: "safeguard"` โดยอัตโนมัติ ผู้ให้บริการจะได้รับคำสั่ง Compaction และนโยบายการเก็บรักษาตัวระบุแบบเดียวกับเส้นทางในตัว และ OpenClaw ยังคงเก็บบริบทส่วนต่อท้ายของเทิร์นล่าสุดและเทิร์นที่ถูกแบ่งไว้หลังเอาต์พุตของผู้ให้บริการ

<Note>
หากผู้ให้บริการล้มเหลวหรือส่งคืนผลลัพธ์ว่าง OpenClaw จะ fallback ไปใช้การสรุปด้วย LLM ในตัว
</Note>

## Compaction เทียบกับการตัดทอน

|                  | Compaction                    | การตัดทอน                         |
| ---------------- | ----------------------------- | -------------------------------- |
| **ทำอะไร** | สรุปการสนทนาเก่า | ตัดผลลัพธ์เครื่องมือเก่า           |
| **บันทึกหรือไม่**       | ใช่ (ในทรานสคริปต์ของเซสชัน)   | ไม่ (อยู่ในหน่วยความจำเท่านั้น ต่อคำขอ) |
| **ขอบเขต**        | การสนทนาทั้งหมด           | เฉพาะผลลัพธ์เครื่องมือ                |

[การตัดทอนเซสชัน](/th/concepts/session-pruning) เป็นส่วนเสริมที่เบากว่า ซึ่งตัดเอาต์พุตเครื่องมือโดยไม่สรุป

## การแก้ไขปัญหา

**ทำ Compaction บ่อยเกินไปหรือไม่** หน้าต่างบริบทของโมเดลอาจเล็ก หรือเอาต์พุตเครื่องมืออาจมีขนาดใหญ่ ลองเปิดใช้ [การตัดทอนเซสชัน](/th/concepts/session-pruning)

**บริบทรู้สึกเก่าหลัง Compaction หรือไม่** ใช้ `/compact Focus on <topic>` เพื่อกำหนดแนวทางให้สรุป หรือเปิดใช้ [การฟลัชหน่วยความจำ](/th/concepts/memory) เพื่อให้โน้ตคงอยู่

**ต้องการเริ่มใหม่แบบสะอาดหรือไม่** `/new` เริ่มเซสชันใหม่โดยไม่ทำ Compaction

สำหรับการกำหนดค่าขั้นสูง เช่น โทเค็นสำรอง การเก็บรักษาตัวระบุ เอนจินบริบทแบบกำหนดเอง และ Compaction ฝั่งเซิร์ฟเวอร์ของ OpenAI โปรดดู [เจาะลึกการจัดการเซสชัน](/th/reference/session-management-compaction)
