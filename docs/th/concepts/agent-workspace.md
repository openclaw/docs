พื้นที่ทำงานคือโฮมของเอเจนต์: ไดเรกทอรีทำงานที่ใช้สำหรับเครื่องมือไฟล์
และบริบทของพื้นที่ทำงาน ให้เก็บเป็นส่วนตัวและถือว่าเป็นหน่วยความจำ

ส่วนนี้แยกจาก `~/.openclaw/` ซึ่งจัดเก็บการกำหนดค่า ข้อมูลรับรอง และเซสชัน

<Warning>
พื้นที่ทำงานคือ **cwd เริ่มต้น** ไม่ใช่แซนด์บ็อกซ์ที่บังคับใช้อย่างเคร่งครัด เครื่องมือจะแก้ไขพาธสัมพัทธ์โดยอิงจากพื้นที่ทำงาน แต่พาธสัมบูรณ์ยังคงเข้าถึงตำแหน่งอื่นบนโฮสต์ได้ เว้นแต่จะเปิดใช้แซนด์บ็อกซ์ หากต้องการการแยกส่วน ให้ใช้ [`agents.defaults.sandbox`](/th/gateway/sandboxing) (และ/หรือการกำหนดค่าแซนด์บ็อกซ์รายเอเจนต์)

เมื่อเปิดใช้แซนด์บ็อกซ์และ `workspaceAccess` ไม่ใช่ `"rw"` เครื่องมือจะทำงานภายในพื้นที่ทำงานของแซนด์บ็อกซ์ภายใต้ `~/.openclaw/sandboxes` ไม่ใช่พื้นที่ทำงานบนโฮสต์
</Warning>

## ตำแหน่งเริ่มต้น

- ค่าเริ่มต้น: `~/.openclaw/workspace`
- หากตั้งค่า `OPENCLAW_PROFILE` และค่าไม่ใช่ `"default"` ค่าเริ่มต้นจะเปลี่ยนเป็น `~/.openclaw/workspace-<profile>`
- เมื่อตั้งค่า `OPENCLAW_WORKSPACE_DIR` ค่านี้จะแทนที่ทั้งสองค่าข้างต้น
- เอเจนต์ที่ไม่ใช่ค่าเริ่มต้น (`agents.list[]`) ซึ่งไม่ได้ระบุพื้นที่ทำงานอย่างชัดเจน จะใช้ `<state-dir>/workspace-<agentId>` ไม่ใช่พื้นที่ทำงานเริ่มต้นที่ใช้ร่วมกัน

แทนที่ค่าใน `~/.openclaw/openclaw.json`:

```json5
{
  agents: {
    defaults: {
      workspace: "~/.openclaw/workspace",
    },
  },
}
```

การแทนที่ค่ารายเอเจนต์: `agents.list[].workspace`

`openclaw onboard`, `openclaw configure` หรือ `openclaw setup` จะสร้างพื้นที่ทำงานและเติมไฟล์เริ่มต้นหากยังไม่มี

<Note>
การคัดลอกไฟล์เริ่มต้นของแซนด์บ็อกซ์ยอมรับเฉพาะไฟล์ปกติภายในพื้นที่ทำงานเท่านั้น นามแฝงแบบ symlink/hardlink ที่แก้ไขพาธแล้วอยู่นอกพื้นที่ทำงานต้นทางจะถูกละเว้น
</Note>

หากจัดการไฟล์พื้นที่ทำงานด้วยตนเองอยู่แล้ว ให้ปิดการสร้างไฟล์เริ่มต้น:

```json5
{ agents: { defaults: { skipBootstrap: true } } }
```

## โฟลเดอร์พื้นที่ทำงานเพิ่มเติม

การติดตั้งรุ่นเก่าอาจสร้าง `~/openclaw` การเก็บไดเรกทอรีพื้นที่ทำงานหลายแห่งไว้อาจทำให้เกิดความสับสนด้านการยืนยันตัวตนหรือสถานะคลาดเคลื่อน เนื่องจากมีพื้นที่ทำงานที่ใช้งานอยู่ได้ครั้งละหนึ่งแห่งเท่านั้น

<Note>
**คำแนะนำ:** ให้เก็บพื้นที่ทำงานที่ใช้งานอยู่เพียงแห่งเดียว หากไม่ได้ใช้โฟลเดอร์เพิ่มเติมแล้ว ให้เก็บถาวรหรือย้ายไปยังถังขยะ (เช่น `trash ~/openclaw`) หากตั้งใจเก็บพื้นที่ทำงานหลายแห่งไว้ โปรดตรวจสอบว่า `agents.defaults.workspace` (หรือคีย์รายเอเจนต์ `workspace`) ชี้ไปยังพื้นที่ที่ใช้งานอยู่
</Note>

## สิ่งที่ไม่ได้อยู่ในพื้นที่ทำงาน

รายการต่อไปนี้อยู่ภายใต้ `~/.openclaw/` และไม่ควรคอมมิตไปยังรีโพซิทอรีพื้นที่ทำงาน:

- `~/.openclaw/openclaw.json` (การกำหนดค่า)
- `~/.openclaw/state/openclaw.sqlite` (สถานะการตั้งค่าพื้นที่ทำงานที่ใช้ร่วมกันและเอกสารรับรอง)
- `~/.openclaw/agents/<agentId>/agent/auth-profiles.json` (โปรไฟล์การยืนยันตัวตนของโมเดล: OAuth + คีย์ API)
- `~/.openclaw/agents/<agentId>/agent/openclaw-agent.sqlite` (แถวเซสชัน บันทึกบทสนทนา และสถานะรันไทม์รายเอเจนต์)
- `~/.openclaw/agents/<agentId>/agent/codex-home/` (บัญชีรันไทม์ Codex รายเอเจนต์ การกำหนดค่า Skills, Plugins และสถานะเธรดแบบเนทีฟ)
- `~/.openclaw/credentials/` (สถานะช่องทาง/ผู้ให้บริการ รวมถึงข้อมูลนำเข้า OAuth แบบเดิม)
- `~/.openclaw/agents/<agentId>/sessions/` (ต้นทางการย้ายข้อมูลแบบเดิมและอาร์ติแฟกต์สำหรับการเก็บถาวร/การสนับสนุน)
- `~/.openclaw/skills/` (Skills ที่มีการจัดการ)

หากต้องการย้ายเซสชันหรือการกำหนดค่า ให้คัดลอกแยกต่างหากและอย่านำเข้าไปอยู่ในการควบคุมเวอร์ชัน

OpenClaw รุ่นเก่าเขียนไฟล์ประกอบพื้นที่ทำงาน `openclaw-workspace-state.json`,
`.openclaw/workspace-state.json` และ `.attested` รันไทม์ปัจจุบัน
ใช้เฉพาะฐานข้อมูล SQLite ที่ใช้ร่วมกันสำหรับสถานะดังกล่าว หาก Doctor รายงาน
ไฟล์ใดไฟล์หนึ่งเหล่านี้ ให้เรียกใช้ `openclaw doctor --fix`; Doctor จะนำเข้าสถานะ
แบบเดิมที่ถูกต้องและลบต้นทางหลังจากตรวจสอบแถวในฐานข้อมูลแล้วเท่านั้น

## การสำรองข้อมูลด้วย Git (แนะนำ ให้เป็นส่วนตัว)

ถือว่าพื้นที่ทำงานเป็นหน่วยความจำส่วนตัว ให้นำไปไว้ในรีโพซิทอรี Git แบบ **ส่วนตัว** เพื่อให้มีข้อมูลสำรองและสามารถกู้คืนได้

เรียกใช้ขั้นตอนเหล่านี้บนเครื่องที่ Gateway ทำงานอยู่ (ซึ่งเป็นตำแหน่งที่พื้นที่ทำงานอยู่)

<Steps>
  <Step title="เริ่มต้นรีโพซิทอรี">
    หากติดตั้ง Git แล้ว พื้นที่ทำงานใหม่เอี่ยมจะถูกเริ่มต้นโดยอัตโนมัติ หากพื้นที่ทำงานนี้ยังไม่ใช่รีโพซิทอรี ให้เรียกใช้:

## อย่าคอมมิตข้อมูลลับ

<Warning>
แม้จะเป็นรีโพซิทอรีส่วนตัว ก็ควรหลีกเลี่ยงการเก็บข้อมูลลับในพื้นที่ทำงาน:

- คีย์ API, โทเค็น OAuth, รหัสผ่าน หรือข้อมูลรับรองส่วนตัว
- ทุกสิ่งภายใต้ `~/.openclaw/`
- ข้อมูลดิบจากการแชตหรือไฟล์แนบที่ละเอียดอ่อน

หากจำเป็นต้องเก็บข้อมูลอ้างอิงที่ละเอียดอ่อน ให้ใช้ตัวแทนค่าและเก็บข้อมูลลับจริงไว้ที่อื่น (ตัวจัดการรหัสผ่าน ตัวแปรสภาพแวดล้อม หรือ `~/.openclaw/`)
</Warning>

ตัวอย่างเริ่มต้นที่แนะนำสำหรับ `.gitignore`:

```gitignore
.DS_Store
.env
**/*.key
**/*.pem
**/secrets*
```

## การย้ายพื้นที่ทำงานไปยังเครื่องใหม่

<Steps>
  <Step title="โคลนรีโพซิทอรี">
    โคลนรีโพซิทอรีไปยังพาธที่ต้องการ (ค่าเริ่มต้น `~/.openclaw/workspace`)
  </Step>
  <Step title="อัปเดตการกำหนดค่า">
    ตั้งค่า `agents.defaults.workspace` เป็นพาธนั้นใน `~/.openclaw/openclaw.json`
  </Step>
  <Step title="เติมไฟล์ที่หายไป">
    เรียกใช้ `openclaw setup --workspace <path>` เพื่อเติมไฟล์ที่หายไป
  </Step>
  <Step title="คัดลอกเซสชัน (ไม่บังคับ)">
    หากต้องการเซสชัน ให้คัดลอก `~/.openclaw/agents/<agentId>/agent/openclaw-agent.sqlite`
    จากเครื่องเก่าแยกต่างหาก คัดลอก `~/.openclaw/agents/<agentId>/sessions/`
    เฉพาะเมื่อต้องการข้อมูลนำเข้าสำหรับการย้ายข้อมูลแบบเดิมหรืออาร์ติแฟกต์สำหรับการเก็บถาวร/การสนับสนุนด้วย
  </Step>
</Steps>

## หมายเหตุขั้นสูง

- การกำหนดเส้นทางแบบหลายเอเจนต์สามารถใช้พื้นที่ทำงานที่แตกต่างกันสำหรับแต่ละเอเจนต์ผ่าน `agents.list[].workspace` ดูการกำหนดค่าเส้นทางที่ [การกำหนดเส้นทางช่องทาง](/th/channels/channel-routing)
- หากเปิดใช้ `agents.defaults.sandbox` เซสชันที่ไม่ใช่เซสชันหลักสามารถใช้พื้นที่ทำงานแซนด์บ็อกซ์รายเซสชันภายใต้ `agents.defaults.sandbox.workspaceRoot`
