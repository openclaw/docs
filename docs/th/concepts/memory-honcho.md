[Honcho](https://honcho.dev) เพิ่มหน่วยความจำที่ออกแบบมาเพื่อ AI ให้กับ OpenClaw ผ่าน Plugin ภายนอก โดยจัดเก็บบทสนทนาอย่างถาวรในบริการเฉพาะ และสร้างโมเดลผู้ใช้กับโมเดลเอเจนต์อย่างต่อเนื่อง ทำให้เอเจนต์ของคุณมีบริบทข้ามเซสชันที่ครอบคลุมมากกว่าไฟล์ Markdown ในพื้นที่ทำงาน

## สิ่งที่มีให้

- **หน่วยความจำข้ามเซสชัน** - บทสนทนาจะถูกจัดเก็บอย่างถาวรหลังจากทุกเทิร์น ทำให้บริบทยังคงอยู่เมื่อรีเซ็ตเซสชัน ทำ Compaction และสลับช่องทาง
- **การสร้างแบบจำลองผู้ใช้** - Honcho ดูแลโปรไฟล์ของผู้ใช้แต่ละราย (ค่ากำหนด ข้อเท็จจริง รูปแบบการสื่อสาร) และของเอเจนต์ (บุคลิกภาพ พฤติกรรมที่เรียนรู้)
- **การค้นหาเชิงความหมาย** - ค้นหาจากข้อสังเกตในบทสนทนาที่ผ่านมา ไม่ใช่เฉพาะเซสชันปัจจุบัน
- **การรับรู้หลายเอเจนต์** - เอเจนต์แม่จะติดตามเอเจนต์ย่อยที่สร้างขึ้นโดยอัตโนมัติ โดยเพิ่มเอเจนต์แม่เป็นผู้สังเกตการณ์ในเซสชันของเอเจนต์ลูก

## เครื่องมือที่พร้อมใช้งาน

Honcho ลงทะเบียนเครื่องมือที่เอเจนต์สามารถใช้ระหว่างการสนทนา:

**การดึงข้อมูล (รวดเร็ว ไม่เรียกใช้ LLM):**

| เครื่องมือ                  | การทำงาน                                               |
| --------------------------- | ------------------------------------------------------ |
| `honcho_context`            | ข้อมูลแทนตัวผู้ใช้แบบครบถ้วนจากทุกเซสชัน               |
| `honcho_search_conclusions` | ค้นหาเชิงความหมายในข้อสรุปที่จัดเก็บไว้                |
| `honcho_search_messages`    | ค้นหาข้อความจากทุกเซสชัน (กรองตามผู้ส่งและวันที่)      |
| `honcho_session`            | ประวัติและข้อมูลสรุปของเซสชันปัจจุบัน                 |

**คำถามและคำตอบ (ขับเคลื่อนด้วย LLM):**

| เครื่องมือ   | การทำงาน                                                                           |
| ------------ | ---------------------------------------------------------------------------------- |
| `honcho_ask` | ถามเกี่ยวกับผู้ใช้ ใช้ `depth='quick'` สำหรับข้อเท็จจริง และ `'thorough'` สำหรับการสังเคราะห์ข้อมูล |

## เริ่มต้นใช้งาน

ติดตั้ง Plugin และเรียกใช้การตั้งค่า:

```bash
openclaw plugins install @honcho-ai/openclaw-honcho
openclaw honcho setup
openclaw gateway --force
```

คำสั่งตั้งค่าจะแจ้งให้ป้อนข้อมูลรับรอง API เขียนการกำหนดค่า และเสนอทางเลือกในการย้ายไฟล์หน่วยความจำที่มีอยู่ในพื้นที่ทำงาน

<Info>
Honcho สามารถทำงานภายในเครื่องทั้งหมด (โฮสต์เอง) หรือผ่าน API แบบมีผู้ดูแลที่ `api.honcho.dev` ตัวเลือกแบบโฮสต์เองไม่จำเป็นต้องใช้การพึ่งพาภายนอก
</Info>

## การกำหนดค่า

การตั้งค่าอยู่ภายใต้ `plugins.entries["openclaw-honcho"].config`:

```json5
{
  plugins: {
    entries: {
      "openclaw-honcho": {
        config: {
          apiKey: "your-api-key", // ไม่ต้องระบุเมื่อโฮสต์เอง
          workspaceId: "openclaw", // การแยกหน่วยความจำ
          baseUrl: "https://api.honcho.dev",
        },
      },
    },
  },
}
```

สำหรับอินสแตนซ์ที่โฮสต์เอง ให้กำหนด `baseUrl` ไปยังเซิร์ฟเวอร์ภายในเครื่องของคุณ (เช่น `http://localhost:8000`) และไม่ต้องระบุคีย์ API

## การย้ายหน่วยความจำที่มีอยู่

หากคุณมีไฟล์หน่วยความจำในพื้นที่ทำงานอยู่แล้ว (`USER.md`, `MEMORY.md`, `IDENTITY.md`, `memory/`, `canvas/`) คำสั่ง `openclaw honcho setup` จะตรวจพบและเสนอให้ย้ายไฟล์เหล่านั้น

<Info>
การย้ายจะไม่ทำลายข้อมูล โดยไฟล์จะถูกอัปโหลดไปยัง Honcho และไฟล์ต้นฉบับจะไม่ถูกลบหรือย้ายตำแหน่ง
</Info>

## วิธีการทำงาน

หลังจากทุกเทิร์นของ AI บทสนทนาจะถูกจัดเก็บอย่างถาวรใน Honcho ทั้งข้อความของผู้ใช้และเอเจนต์จะถูกสังเกตการณ์ ทำให้ Honcho สามารถสร้างและปรับปรุงโมเดลอย่างต่อเนื่อง

ระหว่างการสนทนา เครื่องมือของ Honcho จะสืบค้นบริการผ่านฮุก Plugin `before_prompt_build` ของ OpenClaw และแทรกบริบทที่เกี่ยวข้องก่อนที่โมเดลจะเห็นพรอมต์

## Honcho เทียบกับหน่วยความจำในตัว

|                    | ในตัว / QMD                         | Honcho                                  |
| ------------------ | ----------------------------------- | --------------------------------------- |
| **พื้นที่จัดเก็บ** | ไฟล์ Markdown ในพื้นที่ทำงาน        | บริการเฉพาะ (ภายในเครื่องหรือโฮสต์ไว้) |
| **ข้ามเซสชัน**     | ผ่านไฟล์หน่วยความจำ                  | อัตโนมัติและมีมาในตัว                   |
| **การสร้างแบบจำลองผู้ใช้** | ทำด้วยตนเอง (เขียนลงใน MEMORY.md) | โปรไฟล์อัตโนมัติ                        |
| **การค้นหา**       | เวกเตอร์ + คำสำคัญ (แบบผสม)         | เชิงความหมายจากข้อสังเกต                |
| **หลายเอเจนต์**    | ไม่มีการติดตาม                       | รับรู้ความสัมพันธ์ระหว่างแม่และลูก      |
| **การพึ่งพา**      | ไม่มี (มีมาในตัว) หรือไบนารี QMD     | ติดตั้ง Plugin                          |

Honcho และระบบหน่วยความจำในตัวสามารถทำงานร่วมกันได้ เมื่อกำหนดค่า QMD แล้ว จะมีเครื่องมือเพิ่มเติมสำหรับค้นหาไฟล์ Markdown ภายในเครื่องควบคู่กับหน่วยความจำข้ามเซสชันของ Honcho

## คำสั่ง CLI

```bash
openclaw honcho setup                        # กำหนดค่าคีย์ API และย้ายไฟล์
openclaw honcho status                       # ตรวจสอบสถานะการเชื่อมต่อ
openclaw honcho ask <question>               # สืบค้นข้อมูลเกี่ยวกับผู้ใช้จาก Honcho
openclaw honcho search <query> [-k N] [-d D] # ค้นหาเชิงความหมายในหน่วยความจำ
```
