OpenClaw สร้างพรอมต์ระบบแบบกำหนดเองสำหรับการรันเอเจนต์ทุกครั้ง พรอมต์นี้เป็น **ของ OpenClaw** และไม่ได้ใช้พรอมต์เริ่มต้นของ pi-coding-agent

พรอมต์ถูกประกอบโดย OpenClaw และฉีดเข้าไปในการรันเอเจนต์แต่ละครั้ง

Plugin ผู้ให้บริการสามารถเพิ่มคำแนะนำพรอมต์ที่คำนึงถึงแคชได้โดยไม่ต้องแทนที่
พรอมต์ทั้งหมดที่ OpenClaw เป็นเจ้าของ รันไทม์ของผู้ให้บริการสามารถ:

- แทนที่ส่วนหลักที่มีชื่อจำนวนเล็กน้อย (`interaction_style`,
  `tool_call_style`, `execution_bias`)
- ฉีด **คำนำหน้าที่เสถียร** เหนือขอบเขตแคชพรอมต์
- ฉีด **คำต่อท้ายแบบไดนามิก** ใต้ขอบเขตแคชพรอมต์

ใช้ส่วนที่ผู้ให้บริการเป็นเจ้าของสำหรับการปรับแต่งเฉพาะตระกูลโมเดล เก็บการกลายพันธุ์พรอมต์แบบเดิม
`before_prompt_build` ไว้เพื่อความเข้ากันได้หรือการเปลี่ยนพรอมต์แบบทั่วทั้งระบบจริง ๆ
ไม่ใช่พฤติกรรมปกติของผู้ให้บริการ

โอเวอร์เลย์ตระกูล OpenAI GPT-5 รักษากฎการดำเนินการหลักให้เล็กและเพิ่ม
คำแนะนำเฉพาะโมเดลสำหรับการยึดบุคลิกภาพ เอาต์พุตที่กระชับ วินัยในการใช้เครื่องมือ
การค้นหาแบบขนาน ความครอบคลุมของสิ่งส่งมอบ การตรวจสอบ บริบทที่ขาดหาย และ
สุขอนามัยของเครื่องมือเทอร์มินัล

## โครงสร้าง

พรอมต์ตั้งใจให้กระชับและใช้ส่วนคงที่:

- **เครื่องมือ**: ตัวเตือนแหล่งความจริงของ structured-tool พร้อมคำแนะนำการใช้เครื่องมือในรันไทม์
- **อคติในการดำเนินการ**: คำแนะนำการทำงานให้เสร็จแบบกระชับ: ลงมือในรอบเดียวกับคำขอ
  ที่ดำเนินการได้ ทำต่อจนเสร็จหรือถูกบล็อก ฟื้นตัวจากผลลัพธ์เครื่องมือที่อ่อน
  ตรวจสอบสถานะที่เปลี่ยนแปลงได้แบบสด และตรวจสอบก่อนสรุป
- **ความปลอดภัย**: ตัวเตือนขอบเขตป้องกันสั้น ๆ เพื่อหลีกเลี่ยงพฤติกรรมแสวงหาอำนาจหรือเลี่ยงการกำกับดูแล
- **Skills** (เมื่อพร้อมใช้งาน): บอกโมเดลว่าจะโหลดคำสั่งของ skill ตามต้องการอย่างไร
- **การอัปเดตตัวเองของ OpenClaw**: วิธีตรวจสอบ config อย่างปลอดภัยด้วย
  `config.schema.lookup`, แพตช์ config ด้วย `config.patch`, แทนที่ config ทั้งหมด
  ด้วย `config.apply`, และรัน `update.run` เฉพาะเมื่อผู้ใช้ร้องขออย่างชัดเจน
  เครื่องมือ `gateway` สำหรับเจ้าของเท่านั้นยังปฏิเสธการเขียนใหม่
  `tools.exec.ask` / `tools.exec.security` รวมถึง alias แบบเดิม `tools.bash.*`
  ที่ normalize ไปยังเส้นทาง exec ที่ได้รับการป้องกันเหล่านั้น
- **เวิร์กสเปซ**: ไดเรกทอรีทำงาน (`agents.defaults.workspace`)
- **เอกสาร**: เส้นทางภายในเครื่องไปยังเอกสาร OpenClaw (repo หรือแพ็กเกจ npm) และควรอ่านเมื่อใด
- **ไฟล์เวิร์กสเปซ (ฉีดเข้าไป)**: ระบุว่าไฟล์ bootstrap ถูกรวมไว้ด้านล่าง
- **แซนด์บ็อกซ์** (เมื่อเปิดใช้): ระบุรันไทม์แบบแซนด์บ็อกซ์ เส้นทางแซนด์บ็อกซ์ และ exec แบบยกระดับพร้อมใช้หรือไม่
- **วันที่และเวลาปัจจุบัน**: เวลาท้องถิ่นของผู้ใช้ เขตเวลา และรูปแบบเวลา
- **แท็กตอบกลับ**: ไวยากรณ์แท็กตอบกลับแบบไม่บังคับสำหรับผู้ให้บริการที่รองรับ
- **Heartbeat**: พรอมต์ Heartbeat และพฤติกรรม ack เมื่อเปิดใช้ Heartbeat สำหรับเอเจนต์เริ่มต้น
- **รันไทม์**: โฮสต์, OS, node, โมเดล, ราก repo (เมื่อตรวจพบ), ระดับการคิด (หนึ่งบรรทัด)
- **การให้เหตุผล**: ระดับการมองเห็นปัจจุบัน + คำใบ้สวิตช์ /reasoning

OpenClaw เก็บเนื้อหาเสถียรขนาดใหญ่ รวมถึง **บริบทโครงการ** ไว้เหนือ
ขอบเขตแคชพรอมต์ภายใน ส่วน channel/session ที่เปลี่ยนแปลงง่าย เช่น
คำแนะนำการฝัง Control UI, **การส่งข้อความ**, **เสียง**, **บริบทแชทกลุ่ม**,
**ปฏิกิริยา**, **Heartbeat**, และ **รันไทม์** จะถูกต่อท้ายใต้ขอบเขตนั้น
เพื่อให้แบ็กเอนด์ภายในเครื่องที่มี prefix cache สามารถนำคำนำหน้าเวิร์กสเปซที่เสถียรกลับมาใช้ซ้ำ
ข้ามรอบของ channel ได้ คำอธิบายเครื่องมือก็ควรหลีกเลี่ยงการฝังชื่อ channel ปัจจุบันเช่นกัน
เมื่อ schema ที่ยอมรับมีรายละเอียดรันไทม์นั้นอยู่แล้ว

ส่วนเครื่องมือยังมีคำแนะนำรันไทม์สำหรับงานที่รันนาน:

- ใช้ cron สำหรับการติดตามผลในอนาคต (`check back later`, การเตือนความจำ, งานที่เกิดซ้ำ)
  แทนลูป sleep ของ `exec`, ทริกหน่วงเวลา `yieldMs`, หรือการ polling `process`
  ซ้ำ ๆ
- ใช้ `exec` / `process` เฉพาะสำหรับคำสั่งที่เริ่มตอนนี้และยังคงรัน
  อยู่เบื้องหลัง
- เมื่อเปิดใช้การปลุกเมื่อเสร็จอัตโนมัติ ให้เริ่มคำสั่งครั้งเดียวและพึ่งพา
  เส้นทางปลุกแบบ push-based เมื่อมีเอาต์พุตหรือเกิดความล้มเหลว
- ใช้ `process` สำหรับล็อก สถานะ อินพุต หรือการแทรกแซงเมื่อคุณต้อง
  ตรวจสอบคำสั่งที่กำลังรันอยู่
- หากงานมีขนาดใหญ่กว่า ให้เลือกใช้ `sessions_spawn`; การเสร็จของ sub-agent เป็น
  แบบ push-based และประกาศกลับไปยังผู้ร้องขอโดยอัตโนมัติ
- อย่า poll `subagents list` / `sessions_list` ในลูปเพียงเพื่อรอให้
  เสร็จสิ้น

เมื่อเปิดใช้เครื่องมือทดลอง `update_plan` ส่วนเครื่องมือยังบอกให้
โมเดลใช้เฉพาะกับงานหลายขั้นตอนที่ไม่ธรรมดา รักษาให้มีขั้นตอน
`in_progress` เพียงหนึ่งขั้นตอนพอดี และหลีกเลี่ยงการทำซ้ำแผนทั้งหมดหลังการอัปเดตแต่ละครั้ง

ขอบเขตป้องกันความปลอดภัยในพรอมต์ระบบเป็นคำแนะนำเท่านั้น สิ่งเหล่านี้ชี้นำพฤติกรรมโมเดลแต่ไม่ได้บังคับใช้นโยบาย ใช้นโยบายเครื่องมือ การอนุมัติ exec แซนด์บ็อกซ์ และ allowlist ของ channel สำหรับการบังคับใช้ที่เข้มงวด ผู้ดำเนินการสามารถปิดสิ่งเหล่านี้ได้ตามการออกแบบ

บน channel ที่มีการ์ด/ปุ่มอนุมัติแบบ native ตอนนี้พรอมต์รันไทม์บอกให้
เอเจนต์พึ่งพา UI อนุมัติแบบ native นั้นก่อน ควรรวมคำสั่ง
`/approve` แบบ manual เฉพาะเมื่อผลลัพธ์เครื่องมือบอกว่าการอนุมัติผ่านแชทไม่พร้อมใช้งานหรือ
การอนุมัติแบบ manual เป็นเส้นทางเดียวเท่านั้น

## โหมดพรอมต์

OpenClaw สามารถเรนเดอร์พรอมต์ระบบที่เล็กลงสำหรับ sub-agent ได้ รันไทม์ตั้งค่า
`promptMode` สำหรับการรันแต่ละครั้ง (ไม่ใช่ config ที่แสดงต่อผู้ใช้):

- `full` (ค่าเริ่มต้น): รวมทุกส่วนด้านบน
- `minimal`: ใช้สำหรับ sub-agent; ตัด **Skills**, **การเรียกคืนหน่วยความจำ**, **การอัปเดตตัวเองของ OpenClaw
  **, **นามแฝงโมเดล**, **ตัวตนผู้ใช้**, **แท็กตอบกลับ**,
  **การส่งข้อความ**, **การตอบกลับแบบเงียบ**, และ **Heartbeat** ออก เครื่องมือ, **ความปลอดภัย**,
  เวิร์กสเปซ, แซนด์บ็อกซ์, วันที่และเวลาปัจจุบัน (เมื่อทราบ), รันไทม์, และบริบทที่ฉีดเข้าไป
  ยังคงพร้อมใช้งาน
- `none`: ส่งคืนเฉพาะบรรทัดตัวตนพื้นฐาน

เมื่อ `promptMode=minimal` พรอมต์เพิ่มเติมที่ฉีดเข้าไปจะติดป้ายว่า **บริบท Subagent
** แทน **บริบทแชทกลุ่ม**

สำหรับการรันตอบกลับอัตโนมัติของ channel OpenClaw สามารถละส่วนทั่วไป **การตอบกลับแบบเงียบ**
ได้เมื่อบริบทแชทโดยตรง/แชทกลุ่มมีพฤติกรรม `NO_REPLY` เฉพาะบทสนทนาที่ resolve แล้ว
อยู่แล้ว สิ่งนี้หลีกเลี่ยงการทำซ้ำกลไกโทเค็นทั้งในพรอมต์ระบบส่วนกลางและบริบท channel

## สแนปช็อตพรอมต์

OpenClaw เก็บสแนปช็อตพรอมต์เส้นทางปกติที่ commit แล้วสำหรับรันไทม์ Codex/message-tool
ไว้ใต้ `test/fixtures/agents/prompt-snapshots/happy-path/` สแนปช็อตเหล่านี้เรนเดอร์
พารามิเตอร์ thread/turn ของ app-server ที่เลือก พร้อมสแต็กเลเยอร์พรอมต์ที่ผูกกับโมเดลซึ่งสร้างขึ้นใหม่
สำหรับรอบ Telegram โดยตรง, กลุ่ม Discord, และ Heartbeat สแต็กนั้น
รวมฟิกซ์เจอร์พรอมต์โมเดล Codex `gpt-5.5` ที่ pin ไว้ ซึ่งสร้างจากรูปแบบ catalog/cache ของโมเดล Codex,
ข้อความ developer สิทธิ์เส้นทางปกติของ Codex,
คำสั่ง developer ของ OpenClaw, อินพุตรอบผู้ใช้, และการอ้างอิงไปยังสเปกเครื่องมือแบบไดนามิก

รีเฟรชฟิกซ์เจอร์พรอมต์โมเดล Codex ที่ pin ไว้ด้วย
`pnpm prompt:snapshots:sync-codex-model` โดยค่าเริ่มต้น สคริปต์จะมองหา
แคชรันไทม์ของ Codex ที่ `$CODEX_HOME/models_cache.json`, จากนั้น
`~/.codex/models_cache.json`, และจึงค่อย fallback ไปยัง convention ของ checkout Codex
สำหรับ maintainer ที่ `~/code/codex/codex-rs/models-manager/models.json` หาก
ไม่มีแหล่งข้อมูลเหล่านั้น คำสั่งจะออกโดยไม่เปลี่ยนฟิกซ์เจอร์ที่ commit แล้ว
ส่ง `--catalog <path>` เพื่อรีเฟรชจากไฟล์ `models_cache.json`
หรือ `models.json` ที่ระบุ

สแนปช็อตเหล่านี้ยังไม่ใช่การจับคำขอ OpenAI แบบ raw ที่ตรงกันทีละไบต์ Codex
สามารถเพิ่มบริบทเวิร์กสเปซที่รันไทม์เป็นเจ้าของ เช่น `AGENTS.md`, บริบทสภาพแวดล้อม,
หน่วยความจำ, คำสั่งของ app/plugin, และคำสั่งโหมดการทำงานร่วมกันในอนาคต
ภายในรันไทม์ Codex หลังจาก OpenClaw ส่งพารามิเตอร์ thread และ turn แล้ว

สร้างใหม่ด้วย `pnpm prompt:snapshots:gen` และตรวจสอบ drift ด้วย
`pnpm prompt:snapshots:check` CI รันการตรวจสอบ drift ใน shard ขอบเขตเพิ่มเติม
เพื่อให้การเปลี่ยนแปลงพรอมต์และการอัปเดตสแนปช็อตติดอยู่กับ PR เดียวกัน

## การฉีด Bootstrap ของเวิร์กสเปซ

ไฟล์ Bootstrap ถูกตัดและต่อท้ายใต้ **บริบทโครงการ** เพื่อให้โมเดลเห็นบริบทตัวตนและโปรไฟล์โดยไม่ต้องอ่านอย่างชัดเจน:

- `AGENTS.md`
- `SOUL.md`
- `TOOLS.md`
- `IDENTITY.md`
- `USER.md`
- `HEARTBEAT.md`
- `BOOTSTRAP.md` (เฉพาะเวิร์กสเปซใหม่เอี่ยม)
- `MEMORY.md` เมื่อมีอยู่

ไฟล์ทั้งหมดเหล่านี้ถูก **ฉีดเข้าไปในหน้าต่างบริบท** ในทุก turn เว้นแต่
จะมี gate เฉพาะไฟล์ใช้บังคับ `HEARTBEAT.md` จะถูกละไว้ในการรันปกติเมื่อ
ปิดใช้ Heartbeat สำหรับเอเจนต์เริ่มต้น หรือ
`agents.defaults.heartbeat.includeSystemPromptSection` เป็น false ทำให้ไฟล์ที่ฉีดเข้าไป
กระชับ โดยเฉพาะ `MEMORY.md` ซึ่งสามารถโตขึ้นตามเวลาและนำไปสู่
การใช้บริบทที่สูงเกินคาดและ Compaction ที่ถี่ขึ้น

ไฟล์รายวัน `memory/*.md` **ไม่ใช่** ส่วนหนึ่งของบริบทโครงการ bootstrap ปกติ ใน turn ทั่วไป ไฟล์เหล่านี้ถูกเข้าถึงตามต้องการผ่านเครื่องมือ `memory_search` และ `memory_get` ดังนั้นจึงไม่นับรวมกับหน้าต่างบริบท เว้นแต่โมเดลจะอ่านอย่างชัดเจน turn แบบ bare `/new` และ `/reset` เป็นข้อยกเว้น: รันไทม์สามารถเติมหน่วยความจำรายวันที่ล่าสุดไว้ด้านหน้าเป็นบล็อก startup-context แบบครั้งเดียวสำหรับ turn แรกนั้น
</Note>

ไฟล์ขนาดใหญ่จะถูกตัดด้วย marker ขนาดสูงสุดต่อไฟล์ควบคุมโดย
`agents.defaults.bootstrapMaxChars` (ค่าเริ่มต้น: 12000) เนื้อหา bootstrap ที่ฉีดเข้าไปทั้งหมด
ข้ามไฟล์ถูกจำกัดโดย `agents.defaults.bootstrapTotalMaxChars`
(ค่าเริ่มต้น: 60000) ไฟล์ที่ขาดหายจะฉีด marker ไฟล์ขาดหายสั้น ๆ เมื่อเกิดการตัด
OpenClaw สามารถฉีดบล็อกคำเตือนในบริบทโครงการ ควบคุมสิ่งนี้ด้วย
`agents.defaults.bootstrapPromptTruncationWarning` (`off`, `once`, `always`;
ค่าเริ่มต้น: `once`)

เซสชัน Sub-agent ฉีดเฉพาะ `AGENTS.md` และ `TOOLS.md` (ไฟล์ bootstrap อื่น
ถูกกรองออกเพื่อให้บริบท sub-agent เล็ก)

hook ภายในสามารถ intercept ขั้นตอนนี้ผ่าน `agent:bootstrap` เพื่อกลายพันธุ์หรือแทนที่
ไฟล์ bootstrap ที่ฉีดเข้าไป (เช่น สลับ `SOUL.md` เป็น persona ทางเลือก)

หากคุณต้องการทำให้เอเจนต์ฟังดู generic น้อยลง ให้เริ่มด้วย
[คู่มือบุคลิกภาพ SOUL.md](/th/concepts/soul)

เพื่อตรวจสอบว่าไฟล์ที่ฉีดเข้าไปแต่ละไฟล์มีส่วนร่วมมากเท่าใด (raw เทียบกับ injected, การตัด, รวมถึง overhead ของ schema เครื่องมือ) ให้ใช้ `/context list` หรือ `/context detail` ดู [บริบท](/th/concepts/context)

## การจัดการเวลา

พรอมต์ระบบมีส่วน **วันที่และเวลาปัจจุบัน** โดยเฉพาะเมื่อทราบ
เขตเวลาของผู้ใช้ เพื่อให้พรอมต์เสถียรต่อแคช ตอนนี้จึงรวมเฉพาะ
**เขตเวลา** (ไม่มีนาฬิกาไดนามิกหรือรูปแบบเวลา)

ใช้ `session_status` เมื่อเอเจนต์ต้องการเวลาปัจจุบัน การ์ดสถานะ
มีบรรทัด timestamp เครื่องมือเดียวกันยังสามารถตั้งค่า model override ต่อเซสชันแบบไม่บังคับ
ได้ (`model=default` ล้างค่านั้น)

กำหนดค่าด้วย:

- `agents.defaults.userTimezone`
- `agents.defaults.timeFormat` (`auto` | `12` | `24`)

ดู [วันที่และเวลา](/th/date-time) สำหรับรายละเอียดพฤติกรรมทั้งหมด

## Skills

เมื่อมี skills ที่มีสิทธิ์ OpenClaw จะฉีด **รายการ skills ที่พร้อมใช้งาน** แบบกระชับ
(`formatSkillsForPrompt`) ซึ่งรวม **เส้นทางไฟล์** สำหรับแต่ละ skill พรอมต์
สั่งให้โมเดลใช้ `read` เพื่อโหลด SKILL.md ที่ตำแหน่งที่ระบุไว้
(เวิร์กสเปซ, managed, หรือ bundled) หากไม่มี skills ที่มีสิทธิ์
ส่วน Skills จะถูกละไว้

สิทธิ์มีทั้ง gate metadata ของ skill, การตรวจสอบสภาพแวดล้อม/การกำหนดค่ารันไทม์,
และ allowlist skill ที่มีผลของเอเจนต์เมื่อกำหนดค่า `agents.defaults.skills` หรือ
`agents.list[].skills`

Skills ที่บันเดิลมากับ Plugin มีสิทธิ์เฉพาะเมื่อเปิดใช้ Plugin ที่เป็นเจ้าของ
สิ่งนี้ช่วยให้ Plugin เครื่องมือเปิดเผยคู่มือการปฏิบัติงานที่ลึกขึ้นได้โดยไม่ต้องฝัง
คำแนะนำทั้งหมดนั้นโดยตรงในคำอธิบายเครื่องมือทุกตัว

```
<available_skills>
  <skill>
    <name>...</name>
    <description>...</description>
    <location>...</location>
  </skill>
</available_skills>
```

สิ่งนี้ทำให้พรอมต์พื้นฐานเล็กอยู่เสมอ ในขณะที่ยังเปิดใช้การใช้งาน skill แบบเจาะจงได้

งบประมาณรายการ skills เป็นของระบบย่อย skills:

- ค่าเริ่มต้นส่วนกลาง: `skills.limits.maxSkillsPromptChars`
- การแทนที่ต่อเอเจนต์: `agents.list[].skillsLimits.maxSkillsPromptChars`

ข้อความตัดตอนรันไทม์แบบมีขอบเขตทั่วไปใช้พื้นผิวอื่น:

- `agents.defaults.contextLimits.*`
- `agents.list[].contextLimits.*`

การแยกนี้ทำให้การกำหนดขนาด skills แยกจากการกำหนดขนาดการอ่าน/การฉีดรันไทม์ เช่น
`memory_get`, ผลลัพธ์เครื่องมือสด, และการรีเฟรช AGENTS.md หลัง Compaction

## เอกสาร

พรอมป์ต์ระบบมีส่วน **เอกสารประกอบ** เมื่อมีเอกสารภายในเครื่องพร้อมใช้งาน ส่วนนี้จะชี้ไปยังไดเรกทอรีเอกสาร OpenClaw ภายในเครื่อง (`docs/` ในการเช็กเอาต์ Git หรือเอกสารของแพ็กเกจ npm ที่รวมมาให้) หากไม่มีเอกสารภายในเครื่อง ส่วนนี้จะถอยกลับไปใช้
[https://docs.openclaw.ai](https://docs.openclaw.ai)

ส่วนเดียวกันนี้ยังระบุตำแหน่งซอร์สของ OpenClaw ด้วย การเช็กเอาต์ Git จะเปิดเผยรูทซอร์สภายในเครื่องเพื่อให้เอเจนต์ตรวจสอบโค้ดได้โดยตรง การติดตั้งแพ็กเกจจะรวม URL ซอร์ส GitHub และบอกให้เอเจนต์ตรวจสอบซอร์สที่นั่นเมื่อเอกสารไม่สมบูรณ์หรือล้าสมัย พรอมป์ต์ยังระบุกระจกเอกสารสาธารณะ, Discord ชุมชน และ ClawHub
([https://clawhub.ai](https://clawhub.ai)) สำหรับการค้นพบ Skills โดยบอกให้โมเดลดูเอกสารก่อนสำหรับพฤติกรรม คำสั่ง การกำหนดค่า หรือสถาปัตยกรรมของ OpenClaw และให้รัน `openclaw status` เองเมื่อทำได้ (ถามผู้ใช้เฉพาะเมื่อไม่มีสิทธิ์เข้าถึง) สำหรับการกำหนดค่าโดยเฉพาะ พรอมป์ต์จะชี้เอเจนต์ไปที่การกระทำของเครื่องมือ `gateway`
`config.schema.lookup` สำหรับเอกสารและข้อจำกัดระดับฟิลด์ที่แม่นยำ จากนั้นจึงไปที่
`docs/gateway/configuration.md` และ `docs/gateway/configuration-reference.md`
สำหรับคำแนะนำที่กว้างขึ้น
