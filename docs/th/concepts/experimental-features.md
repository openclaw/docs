ฟีเจอร์ทดลองเป็นความสามารถรุ่นตัวอย่างที่ต้องเลือกเปิดใช้งานผ่านแฟล็กที่ระบุอย่างชัดเจน ฟีเจอร์เหล่านี้ต้องผ่านการใช้งานจริงมากขึ้นก่อนจะกำหนดเป็นค่าเริ่มต้นที่เสถียรหรือเป็นสัญญาระยะยาว

- ปิดใช้งานโดยค่าเริ่มต้น เว้นแต่เอกสารจะระบุให้คุณเปิดใช้งาน
- รูปแบบและพฤติกรรมอาจเปลี่ยนแปลงได้เร็วกว่าการกำหนดค่าที่เสถียร
- เลือกใช้แนวทางที่เสถียรหากมีอยู่แล้ว
- เปิดใช้งานในวงกว้างต่อเมื่อทดสอบในสภาพแวดล้อมขนาดเล็กกว่าก่อนแล้วเท่านั้น

## โหมดลดรูปสำหรับโมเดลภายใน

`agents.defaults.experimental.localModelLean: true` จะนำเครื่องมือเสริมที่ใช้ทรัพยากรมากออกจากชุดเครื่องมือโดยตรงของเอเจนต์ในทุกเทิร์น ได้แก่ `browser`, `cron`, `message`, `image_generate`, `music_generate`, `video_generate`, `tts` และ `pdf` เครื่องมือที่อนุญาตอย่างชัดเจนหรือจำเป็นต่อการส่งมอบจะยังคงใช้งานได้ แม้ว่าการค้นหาเครื่องมืออาจจัดเครื่องมือเหล่านั้นไว้ในแค็ตตาล็อกแทนการเปิดเผยโดยตรงก็ตาม โหมดลดรูปยังกำหนดให้แค็ตตาล็อก Plugin/MCP/ไคลเอนต์ใช้การค้นหาเครื่องมือแบบมีโครงสร้าง (`tool_search`, `tool_describe`, `tool_call`) โดยค่าเริ่มต้น เมื่อยังไม่ได้กำหนด `tools.toolSearch` ใช้ `agents.list[].experimental.localModelLean` เพื่อจำกัดการตั้งค่านี้ให้ใช้กับเอเจนต์เพียงตัวเดียว

หากคุณปรับแต่งการค้นหาเครื่องมือแบบส่วนกลางอยู่แล้ว OpenClaw จะไม่เปลี่ยนแปลงการกำหนดค่านั้น กำหนด `tools.toolSearch: false` เพื่อไม่ใช้ค่าเริ่มต้นของการค้นหาเครื่องมือในโหมดลดรูป

ในโหมด `tools` แบบมีโครงสร้าง การทำงานแบบลดรูปจะยังคงแสดง `exec` โดยตรงควบคู่กับส่วนควบคุมการค้นหาเครื่องมือ เพื่อให้โมเดลภายในที่ปรับแต่งมาสำหรับการเขียนโค้ดยังคงเลือกใช้เส้นทางเชลล์ที่คุ้นเคยได้ การตั้งค่านี้เปลี่ยนเฉพาะการมองเห็นสคีมาเท่านั้น นโยบายเครื่องมือ การทำแซนด์บ็อกซ์ และการอนุมัติการดำเนินการตามปกติยังคงมีผล โหมด `code` และ `directory` ที่ระบุอย่างชัดเจนจะยังคงใช้พฤติกรรม Compaction ตามปกติ

### เหตุผลที่เลือกเครื่องมือเหล่านี้

เครื่องมือเหล่านี้มีคำอธิบายยาวที่สุด มีรูปแบบพารามิเตอร์กว้างที่สุด หรือมีโอกาสสูงที่สุดที่จะทำให้โมเดลขนาดเล็กเสียสมาธิจากเส้นทางการเขียนโค้ดและการสนทนาตามปกติ สำหรับแบ็กเอนด์ที่มีบริบทขนาดเล็กหรือแบ็กเอนด์ที่เข้ากันได้กับ OpenAI ซึ่งเข้มงวดกว่า สิ่งนี้สร้างความแตกต่างระหว่าง:

- สคีมาของเครื่องมือพอดีกับพรอมต์ กับเบียดบังประวัติการสนทนา
- โมเดลเลือกเครื่องมือที่ถูกต้อง กับสร้างการเรียกเครื่องมือที่มีรูปแบบไม่ถูกต้องเนื่องจากมีสคีมาคล้ายกันมากเกินไป
- อะแดปเตอร์ Chat Completions อยู่ภายในขีดจำกัดของเอาต์พุตแบบมีโครงสร้าง กับเกิดข้อผิดพลาด 400 เนื่องจากขนาดเพย์โหลดการเรียกเครื่องมือ

การนำเครื่องมือเหล่านี้ออกเพียงทำให้รายการเครื่องมือโดยตรงสั้นลงเท่านั้น โมเดลยังคงมี `read`, `write`, `edit`, `exec`, `apply_patch`, ความสามารถในการทำความเข้าใจรูปภาพ, การค้นหา/ดึงข้อมูลจากเว็บ (เมื่อกำหนดค่าไว้), หน่วยความจำ และเครื่องมือเซสชัน/เอเจนต์ แค็ตตาล็อกเพิ่มเติมยังคงเข้าถึงได้ผ่านการค้นหาเครื่องมือ เว้นแต่คุณจะกำหนด `tools.toolSearch: false`; การอนุญาตเครื่องมืออย่างชัดเจนสามารถเพิ่มเครื่องมือกลับเข้าไปในเวิร์กโฟลว์แบบลดรูปของเอเจนต์ได้

### ควรเปิดใช้งานเมื่อใด

เปิดใช้งานโหมดลดรูปเมื่อคุณพิสูจน์แล้วว่าโมเดลสามารถสื่อสารกับ Gateway ได้ แต่เทิร์นเต็มรูปแบบของเอเจนต์ทำงานผิดปกติ:

1. `openclaw infer model run --gateway --model <ref> --prompt "Reply with exactly: pong"` ทำงานสำเร็จ
2. เทิร์นปกติของเอเจนต์ล้มเหลวเนื่องจากการเรียกเครื่องมือมีรูปแบบไม่ถูกต้อง พรอมต์มีขนาดใหญ่เกินไป หรือโมเดลเพิกเฉยต่อเครื่องมือ
3. การสลับ `localModelLean: true` ทำให้ข้อผิดพลาดหมดไป

### ควรปิดไว้เมื่อใด

หากแบ็กเอนด์ของคุณรองรับรันไทม์เริ่มต้นทั้งหมดได้อย่างราบรื่น ให้ปิดการตั้งค่านี้ไว้ นี่เป็นวิธีแก้ปัญหาชั่วคราวสำหรับสแต็กภายในที่ต้องการชุดเครื่องมือขนาดเล็กลง ไม่ใช่ค่าเริ่มต้นสำหรับโมเดลที่โฮสต์ไว้หรือระบบภายในที่มีทรัพยากรเพียงพอ

โหมดลดรูปไม่ได้ใช้แทน `tools.profile`, `tools.allow`/`tools.deny` หรือทางเลือกฉุกเฉิน `compat.supportsTools: false` ของโมเดล หากต้องการจำกัดชุดเครื่องมือของเอเจนต์หนึ่งอย่างถาวร ให้เลือกใช้ตัวควบคุมที่เสถียรเหล่านั้น

### เปิดใช้งาน

```json5
{
  agents: {
    defaults: {
      experimental: {
        localModelLean: true,
      },
    },
  },
}
```

สำหรับเอเจนต์เพียงตัวเดียว:

```json5
{
  agents: {
    list: [
      {
        id: "local",
        model: "lmstudio/gemma-4-e4b-it",
        experimental: {
          localModelLean: true,
        },
      },
    ],
  },
}
```

รีสตาร์ต Gateway หลังจากเปลี่ยนแฟล็ก การกรองแบบลดรูปจะนำ `browser`, `cron`, `message`, `image_generate`, `music_generate`, `video_generate`, `tts` และ `pdf` ออก เว้นแต่คุณจะคงเครื่องมือเหล่านั้นไว้อย่างชัดเจนด้วย `tools.allow` หรือ `tools.alsoAllow`; การค้นหาเครื่องมืออาจยังคงจัดเครื่องมือที่เก็บไว้เหล่านี้ลงในแค็ตตาล็อกแทนการเปิดเผยโดยตรง

## ฟีเจอร์ทดลองไม่ได้หมายถึงฟีเจอร์ที่ซ่อนอยู่

ฟีเจอร์ทดลองควรระบุสถานะดังกล่าวอย่างชัดเจนทั้งในเอกสารและในเส้นทางการกำหนดค่า ไม่ใช่ซ่อนไว้ภายใต้ตัวควบคุมค่าเริ่มต้นที่ดูเหมือนเสถียร
