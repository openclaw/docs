ฟีเจอร์ทดลองเป็นส่วนที่เปิดให้ทดลองใช้เบื้องต้นโดยต้องเปิดผ่านแฟล็กอย่างชัดเจน ฟีเจอร์เหล่านี้ต้องผ่านการใช้งานจริงมากขึ้นก่อนที่จะกำหนดค่าเริ่มต้นที่เสถียรหรือสัญญาระยะยาว

- ปิดไว้โดยค่าเริ่มต้น เว้นแต่เอกสารจะอธิบายกฎการตั้งค่าอัตโนมัติที่มีขอบเขตจำกัด
- รูปแบบและลักษณะการทำงานอาจเปลี่ยนแปลงได้เร็วกว่าการกำหนดค่าที่เสถียร
- ควรเลือกใช้แนวทางที่เสถียรหากมีอยู่แล้ว
- เปิดใช้งานในวงกว้างหลังจากทดสอบในสภาพแวดล้อมขนาดเล็กกว่าก่อนแล้วเท่านั้น

## ห้องทดลองของ Control UI

เปิด **Settings → Agents & Tools → Labs** เพื่อจัดการการทดลองที่มีสวิตช์ใน
Control UI การเปิดหรือปิดห้องทดลองจะปรับการกำหนดค่า Gateway มาตรฐาน
ทันที โดยหน้าจะแสดงคำแนะนำให้รีสตาร์ตเฉพาะเมื่อฟีเจอร์จำเป็นต้อง
รีสตาร์ต

Code Mode และ Swarm เป็นรายการ Labs ที่จัดส่งอยู่ในปัจจุบัน สวิตช์ทั้งสอง
จะเขียนคีย์การกำหนดค่าที่มีอยู่และผ่านการตรวจสอบแล้ว และโดยปกติจะมีผลกับการเรียกใช้เอเจนต์
ในอนาคตโดยไม่ต้องรีสตาร์ต Gateway

## โหมดประหยัดสำหรับโมเดลภายในเครื่อง

`agents.defaults.experimental.localModelLean: true` จะนำเครื่องมือเสริมที่ใช้ทรัพยากรมากออกจากส่วนการทำงานโดยตรงของเอเจนต์ในทุกเทิร์น ได้แก่ `browser`, `cron`, `message`, `image_generate`, `music_generate`, `video_generate`, `tts` และ `pdf` เครื่องมือที่อนุญาตอย่างชัดเจนหรือจำเป็นต่อการส่งมอบจะยังคงใช้งานได้ แม้ว่า Tool Search อาจจัดเครื่องมือเหล่านั้นไว้ในแค็ตตาล็อกแทนการเปิดให้ใช้โดยตรง นอกจากนี้ โหมดประหยัดจะกำหนดค่าเริ่มต้นให้แค็ตตาล็อก plugin/MCP/ไคลเอ็นต์ใช้ Tool Search แบบมีโครงสร้าง (`tool_search`, `tool_describe`, `tool_call`) เมื่อยังไม่ได้ตั้งค่า `tools.toolSearch` ใช้ `agents.list[].experimental.localModelLean` เพื่อจำกัดขอบเขตนี้ไว้ที่เอเจนต์เดียว

ระหว่างการเริ่มต้นใช้งาน เส้นทางการอนุมาน `ollama` หรือ `lmstudio` ที่ผ่านการตรวจสอบแล้วจะตั้งค่า `agents.defaults.experimental.localModelLean: true` โดยอัตโนมัติเมื่อไม่มีค่านั้น OpenClaw จะบันทึกว่าการตั้งค่านี้มาจากการเริ่มต้นใช้งาน ดังนั้นเส้นทางที่ไม่ใช่ภายในเครื่องซึ่งผ่านการตรวจสอบในภายหลังจะยกเลิกเฉพาะการตั้งค่าอัตโนมัตินี้เท่านั้น การกำหนดค่า `true` หรือ `false` อย่างชัดเจนจะยังคงอยู่ ผู้ให้บริการแบบโฮสต์เองและผู้ให้บริการที่เข้ากันได้กับ OpenAI รายอื่นจะไม่ถูกอนุมานจากชื่อโมเดลหรือ URL

หากปรับแต่ง Tool Search แบบส่วนกลางอยู่แล้ว OpenClaw จะไม่เปลี่ยนแปลงการกำหนดค่านั้น ตั้งค่า `tools.toolSearch: false` เพื่อปฏิเสธค่าเริ่มต้น Tool Search ของโหมดประหยัด

ในโหมด `tools` แบบมีโครงสร้าง การทำงานแบบประหยัดจะคง `exec` ให้มองเห็นได้โดยตรงข้างตัวควบคุม Tool Search เพื่อให้โมเดลภายในเครื่องที่ปรับแต่งมาสำหรับการเขียนโค้ดยังคงเลือกเส้นทางเชลล์ที่คุ้นเคยได้ การดำเนินการนี้เปลี่ยนเฉพาะการมองเห็นสคีมาเท่านั้น โดยนโยบายเครื่องมือปกติ การทำ sandbox และการอนุมัติ exec ยังคงมีผล โหมด `code` และ `directory` ที่ระบุอย่างชัดเจนจะยังคงลักษณะการทำงาน Compaction ตามปกติ

### เหตุผลที่เลือกเครื่องมือเหล่านี้

เครื่องมือเหล่านี้มีคำอธิบายยาวที่สุด รูปแบบพารามิเตอร์กว้างที่สุด หรือมีโอกาสสูงที่สุดที่จะทำให้โมเดลขนาดเล็กไขว้เขวจากเส้นทางการเขียนโค้ดและการสนทนาตามปกติ สำหรับแบ็กเอนด์ที่มีบริบทขนาดเล็กหรือเข้ากันได้กับ OpenAI แต่เข้มงวดกว่า สิ่งนี้คือความแตกต่างระหว่าง:

- สคีมาเครื่องมือพอดีกับพรอมต์ เทียบกับเบียดบังประวัติการสนทนา
- โมเดลเลือกเครื่องมือที่ถูกต้อง เทียบกับสร้างการเรียกเครื่องมือผิดรูปแบบเนื่องจากมีสคีมาคล้ายกันมากเกินไป
- อะแดปเตอร์ Chat Completions อยู่ภายในขีดจำกัดเอาต์พุตแบบมีโครงสร้าง เทียบกับได้รับ 400 เนื่องจากขนาดเพย์โหลดการเรียกเครื่องมือ

การนำเครื่องมือเหล่านี้ออกเพียงทำให้รายการเครื่องมือโดยตรงสั้นลง โมเดลยังคงมี `read`, `write`, `edit`, `exec`, `apply_patch`, ความสามารถในการเข้าใจภาพ, การค้นหา/ดึงข้อมูลจากเว็บ (เมื่อกำหนดค่าไว้), หน่วยความจำ และเครื่องมือเซสชัน/เอเจนต์ แค็ตตาล็อกเพิ่มเติมยังคงเข้าถึงได้ผ่าน Tool Search เว้นแต่จะตั้งค่า `tools.toolSearch: false`; การอนุญาตเครื่องมืออย่างชัดเจนสามารถเปิดให้เอเจนต์แบบประหยัดกลับมาใช้เวิร์กโฟลว์ที่ถูกตัดออกได้

### ควรเปิดใช้เมื่อใด

เปิดใช้โหมดประหยัดเมื่อพิสูจน์แล้วว่าโมเดลสามารถสื่อสารกับ Gateway ได้ แต่เทิร์นของเอเจนต์แบบเต็มทำงานผิดปกติ:

1. `openclaw infer model run --gateway --model <ref> --prompt "Reply with exactly: pong"` สำเร็จ
2. เทิร์นเอเจนต์ปกติล้มเหลวเนื่องจากการเรียกเครื่องมือผิดรูปแบบ พรอมต์มีขนาดใหญ่เกินไป หรือโมเดลเพิกเฉยต่อเครื่องมือ
3. การสลับ `localModelLean: true` ทำให้ข้อผิดพลาดหายไป

### ควรปิดไว้เมื่อใด

หากแบ็กเอนด์รองรับรันไทม์เริ่มต้นทั้งหมดได้อย่างราบรื่น ให้ปิดตัวเลือกนี้ไว้ นี่คือวิธีแก้ขัดสำหรับสแต็กภายในเครื่องที่ต้องการส่วนการทำงานของเครื่องมือขนาดเล็กกว่า ไม่ใช่ค่าเริ่มต้นสำหรับโมเดลที่โฮสต์หรือเครื่องภายในเครื่องที่มีทรัพยากรเพียงพอ

โหมดประหยัดไม่ได้ใช้แทน `tools.profile`, `tools.allow`/`tools.deny` หรือทางเลือกฉุกเฉิน `compat.supportsTools: false` ของโมเดล สำหรับส่วนการทำงานของเครื่องมือที่แคบลงอย่างถาวรในเอเจนต์หนึ่งโดยเฉพาะ ควรใช้ตัวควบคุมที่เสถียรเหล่านั้น

### เปิดใช้งาน

```json5
{
  agents: {
    defaults: {
      experimental: {
        localModelLean: true,
      },
    },
  },
}
```

สำหรับเอเจนต์เดียวเท่านั้น:

```json5
{
  agents: {
    list: [
      {
        id: "local",
        model: "lmstudio/gemma-4-e4b-it",
        experimental: {
          localModelLean: true,
        },
      },
    ],
  },
}
```

รีสตาร์ต Gateway หลังเปลี่ยนแฟล็ก การกรองแบบประหยัดจะนำ `browser`, `cron`, `message`, `image_generate`, `music_generate`, `video_generate`, `tts` และ `pdf` ออก เว้นแต่จะเก็บรักษาไว้โดยระบุอย่างชัดเจนด้วย `tools.allow` หรือ `tools.alsoAllow`; Tool Search อาจยังคงจัดเครื่องมือที่เก็บรักษาไว้ในแค็ตตาล็อกแทนการเปิดให้ใช้โดยตรง

## ทดลองไม่ได้หมายความว่าซ่อนไว้

ฟีเจอร์ทดลองควรระบุสถานะดังกล่าวอย่างชัดเจนในเอกสารและในเส้นทางการกำหนดค่าเอง ไม่ใช่ซ่อนอยู่หลังตัวควบคุมค่าเริ่มต้นที่ดูเหมือนเสถียร
