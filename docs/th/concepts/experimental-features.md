ฟีเจอร์ทดลองเป็นส่วนที่ให้ทดลองใช้เบื้องต้นภายใต้แฟล็กที่ต้องเปิดอย่างชัดเจน ฟีเจอร์เหล่านี้ต้องผ่านการใช้งานจริงมากขึ้นก่อนจะกำหนดเป็นค่าเริ่มต้นที่เสถียรหรือเป็นสัญญาที่รองรับในระยะยาว

- ปิดไว้โดยค่าเริ่มต้น เว้นแต่เอกสารจะอธิบายกฎการตั้งค่าอัตโนมัติในขอบเขตที่จำกัด
- รูปแบบและลักษณะการทำงานอาจเปลี่ยนแปลงเร็วกว่าการกำหนดค่าที่เสถียร
- เลือกใช้แนวทางที่เสถียรหากมีอยู่แล้ว
- เปิดใช้งานในวงกว้างหลังจากทดสอบในสภาพแวดล้อมที่เล็กกว่าก่อนเท่านั้น

## Control UI Labs

เปิด **Settings → Agents & Tools → Labs** เพื่อจัดการฟีเจอร์ทดลองที่มีสวิตช์ใน
Control UI การเปิดหรือปิดแล็บจะแก้ไขการกำหนดค่าหลักของ Gateway
ทันที โดยหน้าดังกล่าวจะแสดงคำแนะนำให้เริ่มระบบใหม่เฉพาะเมื่อฟีเจอร์นั้นจำเป็นต้อง
เริ่มระบบใหม่

ขณะนี้ Code Mode เป็นรายการ Labs เดียวที่เผยแพร่แล้ว ส่วน Swarm ยังไม่เปิดให้ใช้:
รูปแบบการกำหนดค่าของฟีเจอร์นี้ยังไม่ได้เผยแพร่ ดังนั้น Control UI จึงไม่เขียน
คีย์ที่คาดการณ์ไว้ซึ่งจะทำให้การกำหนดค่าของผู้ดูแลระบบใช้ไม่ได้

## โหมดประหยัดสำหรับโมเดลภายในเครื่อง

`agents.defaults.experimental.localModelLean: true` นำเครื่องมือเสริมขนาดใหญ่ออกจากส่วนที่เอเจนต์เข้าถึงโดยตรงในทุกเทิร์น ได้แก่ `browser`, `cron`, `message`, `image_generate`, `music_generate`, `video_generate`, `tts` และ `pdf` เครื่องมือที่อนุญาตไว้อย่างชัดเจนหรือจำเป็นต่อการส่งมอบยังคงใช้งานได้ แม้ Tool Search อาจจัดทำแค็ตตาล็อกเครื่องมือเหล่านั้นแทนการเปิดให้เข้าถึงโดยตรง นอกจากนี้โหมดประหยัดยังกำหนดให้แค็ตตาล็อก Plugin/MCP/ไคลเอ็นต์ใช้ Tool Search แบบมีโครงสร้าง (`tool_search`, `tool_describe`, `tool_call`) โดยค่าเริ่มต้น เมื่อยังไม่ได้ตั้งค่า `tools.toolSearch` ใช้ `agents.list[].experimental.localModelLean` เพื่อจำกัดขอบเขตนี้ไว้ที่เอเจนต์เดียว

ระหว่างการเริ่มต้นใช้งาน เส้นทางการอนุมาน `ollama` หรือ `lmstudio` ที่ผ่านการตรวจสอบแล้วจะตั้งค่า `agents.defaults.experimental.localModelLean: true` โดยอัตโนมัติหากยังไม่มีค่านี้ OpenClaw จะบันทึกว่าการตั้งค่าดังกล่าวมาจากการเริ่มต้นใช้งาน ดังนั้นเส้นทางที่ไม่ใช่ภายในเครื่องซึ่งผ่านการตรวจสอบในภายหลังจะยกเลิกเฉพาะการตั้งค่าอัตโนมัตินี้เท่านั้น ระบบจะคงค่า `true` หรือ `false` ที่กำหนดไว้อย่างชัดเจน ผู้ให้บริการแบบโฮสต์เองและผู้ให้บริการที่เข้ากันได้กับ OpenAI รายอื่นจะไม่ถูกอนุมานจากชื่อโมเดลหรือ URL

หากปรับแต่ง Tool Search แบบส่วนกลางไว้แล้ว OpenClaw จะไม่เปลี่ยนแปลงการกำหนดค่านั้น ตั้งค่า `tools.toolSearch: false` เพื่อเลือกไม่ใช้ค่าเริ่มต้นของ Tool Search สำหรับโหมดประหยัด

ในโหมด `tools` แบบมีโครงสร้าง การทำงานในโหมดประหยัดจะทำให้ `exec` มองเห็นได้โดยตรงข้างส่วนควบคุม Tool Search เพื่อให้โมเดลภายในเครื่องที่ปรับแต่งมาสำหรับการเขียนโค้ดยังคงเลือกเส้นทางเชลล์ที่คุ้นเคยได้ การตั้งค่านี้เปลี่ยนเฉพาะการมองเห็นสคีมาเท่านั้น โดยนโยบายเครื่องมือปกติ การทำ sandbox และการอนุมัติ exec ยังคงมีผล โหมด `code` และ `directory` ที่กำหนดไว้อย่างชัดเจนจะคงลักษณะการทำงานของ Compaction ตามปกติ

### เหตุผลที่เลือกเครื่องมือเหล่านี้

เครื่องมือเหล่านี้มีคำอธิบายยาวที่สุด มีรูปแบบพารามิเตอร์กว้างที่สุด หรือมีโอกาสสูงที่สุดที่จะทำให้โมเดลขนาดเล็กเสียสมาธิจากขั้นตอนการเขียนโค้ดและการสนทนาตามปกติ สำหรับแบ็กเอนด์ที่เข้ากันได้กับ OpenAI ซึ่งมีบริบทขนาดเล็กกว่าหรือเข้มงวดกว่า นี่คือความแตกต่างระหว่าง:

- สคีมาเครื่องมือพอดีกับพรอมต์ กับการเบียดบังประวัติการสนทนา
- โมเดลเลือกเครื่องมือที่ถูกต้อง กับการสร้างการเรียกใช้เครื่องมือที่มีรูปแบบไม่ถูกต้องเนื่องจากมีสคีมาคล้ายกันมากเกินไป
- อะแดปเตอร์ Chat Completions ทำงานภายในขีดจำกัดเอาต์พุตแบบมีโครงสร้าง กับการได้รับข้อผิดพลาด 400 เนื่องจากขนาดเพย์โหลดการเรียกใช้เครื่องมือ

การนำเครื่องมือเหล่านี้ออกจะเพียงลดความยาวของรายการเครื่องมือที่เข้าถึงโดยตรงเท่านั้น โมเดลยังคงมี `read`, `write`, `edit`, `exec`, `apply_patch`, การทำความเข้าใจรูปภาพ, การค้นหา/ดึงข้อมูลจากเว็บ (เมื่อกำหนดค่าไว้), หน่วยความจำ และเครื่องมือเซสชัน/เอเจนต์ แค็ตตาล็อกเพิ่มเติมยังคงเข้าถึงได้ผ่าน Tool Search เว้นแต่จะตั้งค่า `tools.toolSearch: false`; การอนุญาตเครื่องมืออย่างชัดเจนสามารถเพิ่มเวิร์กโฟลว์ที่ตัดทอนแล้วกลับมาให้เอเจนต์โหมดประหยัดได้

### เมื่อใดควรเปิดใช้

เปิดใช้โหมดประหยัดเมื่อพิสูจน์แล้วว่าโมเดลสื่อสารกับ Gateway ได้ แต่เทิร์นของเอเจนต์แบบเต็มทำงานผิดปกติ:

1. `openclaw infer model run --gateway --model <ref> --prompt "Reply with exactly: pong"` ทำงานสำเร็จ
2. เทิร์นปกติของเอเจนต์ล้มเหลวเนื่องจากการเรียกใช้เครื่องมือมีรูปแบบไม่ถูกต้อง พรอมต์มีขนาดใหญ่เกินไป หรือโมเดลละเลยเครื่องมือของตน
3. การสลับ `localModelLean: true` ทำให้ความล้มเหลวหายไป

### เมื่อใดควรปิดไว้

หากแบ็กเอนด์รองรับรันไทม์เริ่มต้นทั้งหมดได้อย่างถูกต้อง ให้ปิดฟีเจอร์นี้ไว้ ฟีเจอร์นี้เป็นวิธีแก้ปัญหาชั่วคราวสำหรับสแต็กภายในเครื่องที่ต้องการชุดเครื่องมือขนาดเล็กลง ไม่ใช่ค่าเริ่มต้นสำหรับโมเดลแบบโฮสต์หรือระบบภายในเครื่องที่มีทรัพยากรเพียงพอ

โหมดประหยัดไม่ได้ใช้แทน `tools.profile`, `tools.allow`/`tools.deny` หรือทางออกฉุกเฉิน `compat.supportsTools: false` ของโมเดล หากต้องการชุดเครื่องมือที่แคบลงอย่างถาวรสำหรับเอเจนต์เฉพาะ ให้เลือกใช้ตัวควบคุมที่เสถียรเหล่านั้น

### เปิดใช้

```json5
{
  agents: {
    defaults: {
      experimental: {
        localModelLean: true,
      },
    },
  },
}
```

สำหรับเอเจนต์เดียวเท่านั้น:

```json5
{
  agents: {
    list: [
      {
        id: "local",
        model: "lmstudio/gemma-4-e4b-it",
        experimental: {
          localModelLean: true,
        },
      },
    ],
  },
}
```

เริ่ม Gateway ใหม่หลังจากเปลี่ยนแฟล็ก การกรองในโหมดประหยัดจะนำ `browser`, `cron`, `message`, `image_generate`, `music_generate`, `video_generate`, `tts` และ `pdf` ออก เว้นแต่จะคงเครื่องมือเหล่านี้ไว้อย่างชัดเจนด้วย `tools.allow` หรือ `tools.alsoAllow`; Tool Search อาจยังคงจัดทำแค็ตตาล็อกเครื่องมือที่คงไว้แทนการเปิดให้เข้าถึงโดยตรง

## ทดลองไม่ได้หมายความว่าซ่อนอยู่

ฟีเจอร์ทดลองควรระบุสถานะดังกล่าวอย่างชัดเจนทั้งในเอกสารและในเส้นทางการกำหนดค่า ไม่ใช่ซ่อนไว้หลังตัวควบคุมเริ่มต้นที่ดูเหมือนเสถียร
