ระบบจะส่งตัวบ่งชี้การพิมพ์ไปยังช่องแชตขณะที่การเรียกใช้ยังทำงานอยู่ ใช้ `agents.defaults.typingMode` เพื่อควบคุมว่า **เมื่อใด** จะเริ่มแสดงการพิมพ์ และใช้ `typingIntervalSeconds` เพื่อควบคุมว่ารีเฟรช **บ่อยเพียงใด** (รอบการคงการเชื่อมต่อ ค่าเริ่มต้น 6 วินาที)

## ค่าเริ่มต้น

เมื่อ **ไม่ได้ตั้งค่า** `agents.defaults.typingMode`:

- **แชตโดยตรง**: เริ่มแสดงการพิมพ์ทันทีเมื่อวงรอบโมเดลเริ่มทำงาน
- **แชตกลุ่มที่มีการกล่าวถึง**: เริ่มแสดงการพิมพ์ทันที
- **แชตกลุ่มที่ไม่มีการกล่าวถึง**: เริ่มแสดงการพิมพ์เมื่อการเรียกใช้ที่ได้รับอนุญาตมีกิจกรรมที่ผู้ใช้มองเห็นได้ เช่น กิจกรรมการดำเนินการของชุดควบคุมหรือข้อความ
- **การเรียกใช้ Heartbeat**: เริ่มแสดงการพิมพ์เมื่อการเรียกใช้ Heartbeat เริ่มต้น หากเป้าหมาย Heartbeat ที่ได้รับการแก้ไขเป็นแชตที่รองรับการแสดงการพิมพ์และไม่ได้ปิดใช้งานการแสดงการพิมพ์

## โหมด

ตั้งค่า `agents.defaults.typingMode` เป็นค่าใดค่าหนึ่งต่อไปนี้:

- `never` - ไม่แสดงตัวบ่งชี้การพิมพ์ไม่ว่าในกรณีใด
- `instant` - เริ่มแสดงการพิมพ์ **ทันทีที่วงรอบโมเดลเริ่มทำงาน** แม้ว่าภายหลังการเรียกใช้จะส่งคืนเพียงโทเค็นตอบกลับแบบเงียบ
- `thinking` - เริ่มแสดงการพิมพ์เมื่อมี **เดลตาการให้เหตุผลรายการแรก** หรือเมื่อมีการดำเนินการของชุดควบคุมที่ทำงานอยู่หลังจากยอมรับรอบสนทนาแล้ว
- `message` - เริ่มแสดงการพิมพ์เมื่อมี **กิจกรรมการตอบกลับรายการแรกที่ผู้ใช้มองเห็นได้** เช่น การดำเนินการของชุดควบคุมที่ทำงานอยู่หรือเดลตาข้อความที่ไม่ใช่แบบเงียบ โทเค็นตอบกลับแบบเงียบ เช่น `NO_REPLY` จะไม่นับเป็นกิจกรรมข้อความ

ลำดับตามความเร็วในการทำงาน: `never` -> `message`/`thinking` -> `instant`

## การกำหนดค่า

ตั้งค่าค่าเริ่มต้นระดับเอเจนต์:

```json5
{
  agents: {
    defaults: {
      typingMode: "thinking",
      typingIntervalSeconds: 6,
    },
  },
}
```

เขียนทับโหมดหรือรอบการทำงานสำหรับแต่ละเซสชัน:

```json5
{
  session: {
    typingMode: "message",
    typingIntervalSeconds: 4,
  },
}
```

## หมายเหตุ

- โหมด `message` จะไม่เริ่มจากโทเค็นตอบกลับแบบเงียบ แต่การดำเนินการที่ทำงานอยู่ยังสามารถแสดงการพิมพ์ก่อนที่จะมีข้อความจากผู้ช่วยได้
- `thinking` ยังคงตอบสนองต่อการให้เหตุผลแบบสตรีม (`reasoningLevel: "stream"`) และยังสามารถเริ่มจากการดำเนินการที่ทำงานอยู่ก่อนที่เดลตาการให้เหตุผลจะมาถึง
- การแสดงการพิมพ์ของ Heartbeat เป็นสัญญาณบ่งชี้ว่าปลายทางการนำส่งที่ได้รับการแก้ไขยังทำงานอยู่ โดยจะเริ่มเมื่อการเรียกใช้ Heartbeat เริ่มต้น แทนที่จะอิงตามจังหวะสตรีมของ `message` หรือ `thinking` ตั้งค่า `typingMode: "never"` เพื่อปิดใช้งาน
- Heartbeat จะไม่แสดงการพิมพ์เมื่อเป้าหมาย Heartbeat คือ `"none"` เมื่อไม่สามารถแก้ไขเป้าหมายได้ เมื่อปิดใช้งานการนำส่งผ่านแชตสำหรับ Heartbeat หรือเมื่อช่องทางไม่รองรับการแสดงการพิมพ์
- `typingIntervalSeconds` ควบคุม **รอบการรีเฟรช** ไม่ใช่เวลาเริ่มต้น ค่าเริ่มต้น: 6 วินาที
