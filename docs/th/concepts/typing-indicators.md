ตัวบ่งชี้การพิมพ์จะถูกส่งไปยังช่องแชตขณะที่การรันกำลังทำงาน ใช้ `agents.defaults.typingMode` เพื่อควบคุมว่าเริ่มแสดงการพิมพ์ **เมื่อใด** และใช้ `typingIntervalSeconds` เพื่อควบคุมว่ารีเฟรช **บ่อยเพียงใด** (รอบการส่ง keepalive ค่าเริ่มต้น 6 วินาที)

## ค่าเริ่มต้น

เมื่อ **ไม่ได้ตั้งค่า** `agents.defaults.typingMode`:

- **แชตโดยตรง**: เริ่มแสดงการพิมพ์ทันทีที่ลูปโมเดลเริ่มทำงาน
- **แชตกลุ่มที่มีการกล่าวถึง**: เริ่มแสดงการพิมพ์ทันที
- **แชตกลุ่มที่ไม่มีการกล่าวถึง**: เริ่มแสดงการพิมพ์เมื่อการรันที่ได้รับอนุญาตมีกิจกรรมที่ผู้ใช้มองเห็นได้ เช่น กิจกรรมการดำเนินงานของ harness หรือข้อความ
- **การรัน Heartbeat**: เริ่มแสดงการพิมพ์เมื่อการรัน Heartbeat เริ่มขึ้น หากเป้าหมาย Heartbeat ที่ได้รับการระบุเป็นแชตที่รองรับการแสดงการพิมพ์และไม่ได้ปิดใช้งานการแสดงการพิมพ์

## โหมด

ตั้งค่า `agents.defaults.typingMode` เป็นค่าใดค่าหนึ่งต่อไปนี้:

- `never` - ไม่แสดงตัวบ่งชี้การพิมพ์ไม่ว่าในกรณีใด
- `instant` - เริ่มแสดงการพิมพ์ **ทันทีที่ลูปโมเดลเริ่มทำงาน** แม้ภายหลังการรันจะส่งคืนเพียงโทเค็นตอบกลับแบบเงียบ
- `thinking` - เริ่มแสดงการพิมพ์เมื่อมี **เดลตาการให้เหตุผลรายการแรก** หรือเมื่อ harness เริ่มดำเนินงานหลังจากระบบยอมรับเทิร์นแล้ว
- `message` - เริ่มแสดงการพิมพ์เมื่อมี **กิจกรรมการตอบกลับที่ผู้ใช้มองเห็นได้รายการแรก** เช่น การดำเนินงานของ harness หรือเดลตาข้อความที่ไม่ใช่แบบเงียบ โทเค็นตอบกลับแบบเงียบ เช่น `NO_REPLY` จะไม่นับเป็นกิจกรรมข้อความ

ลำดับตาม "ความเร็วในการเริ่มทำงาน": `never` -> `message`/`thinking` -> `instant`

## การกำหนดค่า

ตั้งค่าเริ่มต้นระดับเอเจนต์:

```json5
{
  agents: {
    defaults: {
      typingMode: "thinking",
      typingIntervalSeconds: 6,
    },
  },
}
```

เขียนทับโหมดสำหรับแต่ละเซสชัน:

```json5
{
  session: {
    typingMode: "message",
  },
}
```

## หมายเหตุ

- โหมด `message` จะไม่เริ่มจากโทเค็นตอบกลับแบบเงียบ แต่การดำเนินงานที่กำลังทำงานยังสามารถแสดงการพิมพ์ก่อนที่จะมีข้อความจากผู้ช่วย
- `thinking` ยังคงตอบสนองต่อการให้เหตุผลแบบสตรีม (`reasoningLevel: "stream"`) และยังสามารถเริ่มจากการดำเนินงานที่กำลังทำงานก่อนที่เดลตาการให้เหตุผลจะมาถึง
- การแสดงการพิมพ์ของ Heartbeat เป็นสัญญาณบ่งบอกความพร้อมทำงานสำหรับเป้าหมายการส่งที่ได้รับการระบุ โดยจะเริ่มเมื่อการรัน Heartbeat เริ่มขึ้น แทนที่จะอิงตามจังหวะเวลาของสตรีม `message` หรือ `thinking` ตั้งค่า `typingMode: "never"` เพื่อปิดใช้งาน
- Heartbeat จะไม่แสดงการพิมพ์เมื่อเป้าหมาย Heartbeat คือ `"none"` เมื่อไม่สามารถระบุเป้าหมายได้ เมื่อปิดใช้งานการส่งแชตสำหรับ Heartbeat หรือเมื่อช่องไม่รองรับการแสดงการพิมพ์
- `agents.defaults.typingIntervalSeconds` ควบคุม **รอบการรีเฟรช** ไม่ใช่เวลาเริ่มต้น ค่าเริ่มต้น: 6 วินาที
