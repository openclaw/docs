การอ้างอิงโมเดล (`provider/model`) ใช้เลือกผู้ให้บริการและโมเดล ไม่ใช่รันไทม์เอเจนต์ระดับล่าง
เมื่อไม่ได้ตั้งค่านโยบายรันไทม์หรือเป็น `auto` นโยบายเส้นทางที่ผู้ให้บริการ OpenAI เป็นเจ้าของ
อาจเลือก Codex เฉพาะสำหรับเส้นทาง Responses ของ Platform ผ่าน HTTPS อย่างเป็นทางการ
หรือเส้นทาง ChatGPT Responses ที่ตรงกันทุกประการและไม่มีการแทนที่คำขอที่ผู้ใช้กำหนดไว้
เพียงคำนำหน้า `openai/*` อย่างเดียวจะไม่เลือก Codex อะแดปเตอร์ Completions, endpoint
แบบกำหนดเอง และพฤติกรรมคำขอที่ผู้ใช้กำหนดไว้จะยังคงทำงานบน OpenClaw ส่วน endpoint HTTP
แบบข้อความธรรมดาที่เป็นทางการจะถูกปฏิเสธ ดู [รันไทม์เอเจนต์โดยนัยของ OpenAI](/th/providers/openai#implicit-agent-runtime)

การอ้างอิง Copilot แบบสมัครสมาชิก (`github-copilot/*`) สามารถเลือกใช้ Plugin รันไทม์เอเจนต์
GitHub Copilot ภายนอกได้ แต่เส้นทางนั้นต้องระบุอย่างชัดเจนเสมอ (จะไม่ถูกเลือกโดย
`auto`) การแทนที่รันไทม์ต้องอยู่ในนโยบายผู้ให้บริการ/โมเดล ไม่ใช่ในเอเจนต์
หรือเซสชันทั้งหมด การเลือกรันไทม์ไม่ได้กำหนดการเรียกเก็บเงิน: ข้อมูลประจำตัวของคีย์ API
OpenAI และข้อมูลประจำตัวการสมัครสมาชิก ChatGPT/Codex ยังคงแยกจากกัน ดู
[รันไทม์เอเจนต์](/th/concepts/agent-runtimes) และ
[รันไทม์เอเจนต์ GitHub Copilot](/th/plugins/copilot)

## ลำดับการเลือก

<Steps>
  <Step title="โมเดลหลัก">
    `agents.defaults.model.primary` (หรือ `agents.defaults.model` ในรูปแบบสตริงธรรมดา)
  </Step>
  <Step title="โมเดลสำรอง">
    `agents.defaults.model.fallbacks` โดยลองตามลำดับ
  </Step>
  <Step title="การสลับการยืนยันตัวตน">
    การหมุนเวียนโปรไฟล์การยืนยันตัวตนจะเกิดขึ้นภายในผู้ให้บริการ ก่อนที่ OpenClaw จะเปลี่ยนไปยังโมเดลสำรองถัดไป
  </Step>
</Steps>

ส่วนการกำหนดค่าโมเดลที่เกี่ยวข้อง:

- `agents.defaults.models` เก็บนามแฝงและการตั้งค่าเฉพาะโมเดล การเพิ่มรายการไม่ได้จำกัดการแทนที่โมเดล
- `agents.defaults.modelPolicy.allow` คือรายการอนุญาตสำหรับการแทนที่ซึ่งเป็นทางเลือก ใช้การอ้างอิงที่ตรงกันทุกประการหรือรายการ `provider/*`; ละไว้หรือตั้งเป็น `[]` เพื่ออนุญาตทุกโมเดล `agents.list[].modelPolicy.allow` รายเอเจนต์จะแทนที่นโยบายเริ่มต้นสำหรับเอเจนต์นั้น
- `agents.defaults.utilityModel` คือโมเดลต้นทุนต่ำกว่าซึ่งเป็นทางเลือกสำหรับงานภายในระยะสั้น เช่น ชื่อเซสชันแดชบอร์ดที่สร้างขึ้น ชื่อเธรด/หัวข้อของช่องทางที่รองรับ และคำบรรยายความคืบหน้า `agents.list[].utilityModel` รายเอเจนต์จะแทนที่ค่านี้ เมื่อไม่ได้ตั้งค่า OpenClaw จะใช้ค่าเริ่มต้นของโมเดลขนาดเล็กที่ผู้ให้บริการหลักประกาศไว้หากมี (OpenAI → `gpt-5.6-luna`, Anthropic → `claude-haiku-4-5`) มิฉะนั้นจะใช้โมเดลหลักของเอเจนต์ ตั้งเป็นสตริงว่างเพื่อปิดใช้งานการกำหนดเส้นทางงานอรรถประโยชน์ งานอรรถประโยชน์เป็นการเรียกโมเดลแยกต่างหากและอาจส่งเนื้อหางานที่มีขอบเขตจำกัดไปยังผู้ให้บริการโมเดลที่เลือก
- `agents.defaults.imageModel` ใช้เฉพาะเมื่อโมเดลหลักไม่สามารถรับภาพได้
- `agents.defaults.pdfModel` ใช้โดยเครื่องมือ `pdf` หากไม่ได้ตั้งค่า เครื่องมือจะใช้ `imageModel` เป็นตัวสำรอง แล้วจึงใช้โมเดลที่แก้ไขแล้วของเซสชัน/ค่าเริ่มต้น
- `agents.defaults.imageGenerationModel`, `musicGenerationModel` และ `videoGenerationModel` รองรับเครื่องมือสร้างสื่อที่ใช้ร่วมกัน หากไม่ได้ตั้งค่า เครื่องมือแต่ละรายการจะอนุมานค่าเริ่มต้นของผู้ให้บริการที่มีการยืนยันตัวตนรองรับ โดยเริ่มจากผู้ให้บริการเริ่มต้นปัจจุบัน แล้วตามด้วยผู้ให้บริการที่ลงทะเบียนรายอื่นสำหรับความสามารถนั้นตามลำดับรหัสผู้ให้บริการ ตั้ง `agents.defaults.mediaGenerationAutoProviderFallback: false` เพื่อปิดใช้งานการอนุมานข้ามผู้ให้บริการดังกล่าว โดยยังคงใช้โมเดลสำรองที่ระบุไว้อย่างชัดเจน
- `agents.list[].model` รายเอเจนต์ (รวมถึงการผูก) จะแทนที่ `agents.defaults.model` — ดู [การกำหนดเส้นทางแบบหลายเอเจนต์](/th/concepts/multi-agent)

ข้อมูลอ้างอิงคีย์ ค่าเริ่มต้น และตัวอย่าง JSON5 ฉบับสมบูรณ์: [ข้อมูลอ้างอิงการกำหนดค่า](/th/gateway/config-agents#agent-defaults)

## แหล่งที่มาของการเลือกและความเข้มงวดของโมเดลสำรอง

`provider/model` เดียวกันจะทำงานต่างกันตามแหล่งที่มา:

| แหล่งที่มา                                                                  | พฤติกรรม                                                                                                                                                                                                                                                       |
| ----------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ค่าเริ่มต้นที่กำหนดค่าไว้ (`agents.defaults.model.primary`, โมเดลหลักรายเอเจนต์) | จุดเริ่มต้นปกติ โดยใช้ `agents.defaults.model.fallbacks`                                                                                                                                                                                                 |
| โมเดลสำรองอัตโนมัติ                                                           | สถานะการกู้คืนชั่วคราว ซึ่งจัดเก็บเป็น `modelOverrideSource: "auto"` OpenClaw จะตรวจสอบโมเดลหลักเดิมซ้ำเป็นระยะ ล้างการเลือกอัตโนมัติเมื่อกู้คืนได้ และประกาศการเปลี่ยนผ่านไปยังโมเดลสำรอง/การกู้คืนหนึ่งครั้งต่อการเปลี่ยนสถานะ                              |
| การเลือกของผู้ใช้ในเซสชัน                                                  | ต้องตรงกันทุกประการและเข้มงวด `/model`, ตัวเลือกโมเดล, `session_status(model=...)` และ `sessions.patch` จะจัดเก็บ `modelOverrideSource: "user"` หากผู้ให้บริการ/โมเดลนั้นไม่สามารถเข้าถึงได้ การรันจะล้มเหลวอย่างชัดเจนแทนที่จะเปลี่ยนไปยังโมเดลอื่นที่กำหนดค่าไว้ |
| Cron `--model` / เพย์โหลด `model`                                        | โมเดลหลักรายงาน ยังคงใช้โมเดลสำรองที่กำหนดค่าไว้ เว้นแต่งานจะระบุเพย์โหลด `fallbacks` ของตนเอง (`fallbacks: []` บังคับให้รันแบบเข้มงวด)                                                                                                                    |

กฎการเลือกอื่นๆ:

- การเปลี่ยน `agents.defaults.model.primary` จะไม่เขียนทับการปักหมุดเซสชันที่มีอยู่ หากสถานะรายงาน `This session is pinned to X; config primary Y will apply to new/unpinned sessions.` ให้รัน `/model default` เพื่อล้างการปักหมุด
- ตัวเลือกโมเดลเริ่มต้นและรายการอนุญาตใน CLI จะเคารพ `models.mode: "replace"` โดยแสดงเฉพาะ `models.providers.*.models` แทนแค็ตตาล็อกในตัวทั้งหมด
- ตัวเลือกโมเดลใน Control UI จะขอมุมมองโมเดลที่กำหนดค่าไว้จาก Gateway `modelPolicy.allow` ที่ระบุไว้อย่างชัดเจนจะกรองมุมมองนี้ รวมถึงรายการไวลด์การ์ด `provider/*`; มิฉะนั้นจะแสดงโมเดลที่กำหนดค่าไว้ รวมทั้งผู้ให้บริการที่มีการยืนยันตัวตนที่ใช้งานได้ แค็ตตาล็อกในตัวทั้งหมดสงวนไว้สำหรับมุมมองเรียกดูที่ระบุไว้อย่างชัดเจน (`models.list` พร้อม `view: "all"` หรือ `openclaw models list --all`)
- UI คลังผู้ให้บริการใช้ `models.list` ร่วมกับ `view: "provider-config"` เพื่อแสดงแถว `models.providers.*.models` ที่กำหนดโดยแหล่งที่มา โดยไม่ใช้รายการอนุญาตของตัวเลือก

กลไกทั้งหมด: [การสลับไปใช้โมเดลสำรอง](/th/concepts/model-failover)

## นโยบายโมเดลแบบย่อ

- ตั้งโมเดลหลักเป็นโมเดลรุ่นล่าสุดที่มีประสิทธิภาพสูงสุดซึ่งใช้งานได้
- ใช้โมเดลสำรองสำหรับงานที่อ่อนไหวต่อต้นทุน/เวลาแฝงและการสนทนาที่มีความเสี่ยงต่ำกว่า
- สำหรับเอเจนต์ที่เปิดใช้เครื่องมือหรืออินพุตที่ไม่น่าเชื่อถือ ให้หลีกเลี่ยงระดับโมเดลที่เก่ากว่า/ด้อยกว่า

## การเริ่มต้นใช้งาน

```bash
openclaw onboard
```

ตั้งค่าโมเดลและการยืนยันตัวตนสำหรับผู้ให้บริการทั่วไปโดยไม่ต้องแก้ไขการกำหนดค่าด้วยตนเอง รวมถึง OAuth สำหรับการสมัครสมาชิก OpenAI Codex และ Anthropic (คีย์ API หรือใช้ Claude CLI ที่มีอยู่ซ้ำ)

เมื่อไม่ได้กำหนดค่าโมเดลหลัก การตั้งค่าคีย์ API ของ OpenAI ใหม่จะเลือก
`openai/gpt-5.6`; รหัส API โดยตรงแบบไม่มีคำนำหน้าจะถูกแก้ไขเป็นระดับ Sol การตั้งค่า
OAuth ของ ChatGPT/Codex ใหม่จะเลือกการอ้างอิงแค็ตตาล็อก `openai/gpt-5.6-sol` ที่ตรงกันทุกประการ
การยืนยันตัวตนใหม่จะรักษาโมเดลหลักที่ระบุไว้อย่างชัดเจนซึ่งมีอยู่แล้ว รวมถึง
`openai/gpt-5.5` หากบัญชีไม่สามารถใช้ GPT-5.6 ได้ ให้เลือก
`openai/gpt-5.5` อย่างชัดเจน OpenClaw จะไม่ลดระดับโดยไม่แจ้งให้ทราบ

## "ไม่อนุญาตให้ใช้โมเดล" (และเหตุใดการตอบกลับจึงหยุดลง)

หาก `agents.defaults.modelPolicy.allow` ไม่ว่าง ค่านี้จะกลายเป็นรายการอนุญาตสำหรับ `/model`, การแทนที่ในเซสชัน และ `--model` การเลือกโมเดลที่อยู่นอกรายการอนุญาตนั้นจะส่งผลลัพธ์กลับก่อนที่จะสร้างการตอบกลับตามปกติ `agents.list[].modelPolicy.allow` รายเอเจนต์จะแทนที่นโยบายเริ่มต้นสำหรับเอเจนต์นั้น

```text
การแทนที่โมเดล "provider/model" ไม่ได้รับอนุญาตโดย agents.defaults.modelPolicy.allow
เพิ่ม "provider/model" หรือ "provider/*" ลงใน agents.defaults.modelPolicy.allow หรือลบ/ทำให้รายการว่างเพื่ออนุญาตทุกโมเดล
```

แก้ไขโดยเพิ่มโมเดลหรือไวลด์การ์ดของผู้ให้บริการลงในคีย์ `modelPolicy.allow` ที่ระบุ ลบ/ทำให้รายการนั้นว่าง หรือเลือกโมเดลจาก `/model list` หากคำสั่งที่ถูกปฏิเสธมีการแทนที่รันไทม์ เช่น `/model openai/gpt-5.5 --runtime codex` ให้แก้ไขรายการอนุญาตก่อน แล้วลองใช้คำสั่งเดิมอีกครั้ง

สำหรับโมเดลภายในเครื่อง/GGUF รายการอนุญาตต้องมีการอ้างอิงแบบเต็มที่มีคำนำหน้าผู้ให้บริการ เช่น `ollama/gemma4:26b` หรือ `lmstudio/Gemma4-26b-a4-it-gguf` — ตรวจสอบสตริงที่ตรงกันทุกประการใน `openclaw models list --provider <provider>` ชื่อไฟล์หรือชื่อที่แสดงโดยไม่มีคำนำหน้าไม่เพียงพอเมื่อเปิดใช้รายการอนุญาตแล้ว

หากต้องการจำกัดผู้ให้บริการโดยไม่ต้องระบุทุกโมเดล ให้ใช้รายการไวลด์การ์ด `provider/*`:

```json5
{
  agents: {
    defaults: {
      modelPolicy: {
        allow: ["openai/*", "vllm/*"],
      },
    },
  },
}
```

จากนั้น `/model`, `/models` และตัวเลือกโมเดลจะแสดงแค็ตตาล็อกที่ค้นพบสำหรับผู้ให้บริการเหล่านั้นเท่านั้น และโมเดลใหม่สามารถปรากฏขึ้นได้โดยไม่ต้องแก้ไขรายการอนุญาต ผสมรายการ `provider/model` ที่ตรงกันทุกประการกับรายการ `provider/*` เพื่อรวมโมเดลเฉพาะหนึ่งรายการจากผู้ให้บริการรายอื่น

ตัวอย่างรายการอนุญาตที่มีนามแฝงและการตั้งค่าเฉพาะโมเดล:

```json5
{
  agents: {
    defaults: {
      model: { primary: "anthropic/claude-sonnet-4-6" },
      modelPolicy: {
        allow: ["anthropic/claude-sonnet-4-6", "anthropic/claude-opus-4-6"],
      },
      models: {
        "anthropic/claude-sonnet-4-6": { alias: "Sonnet" },
        "anthropic/claude-opus-4-6": { alias: "Opus" },
      },
    },
  },
}
```

<Accordion title="แก้ไขรายการอนุญาตอย่างชัดเจน">
ตั้งค่ารายการทั้งหมดโดยตรง:

```bash
openclaw config set agents.defaults.modelPolicy.allow '["openai/gpt-5.4","anthropic/*"]' --strict-json
```

`openclaw models set`, การตั้งค่าผู้ให้บริการ และ `openclaw models aliases add` สามารถเพิ่มรายการภายใต้ `agents.defaults.models` ได้ แต่จะไม่เปลี่ยน `modelPolicy.allow` วิธีนี้ช่วยให้ข้อมูลเมตาของโมเดลและนามแฝงเป็นอิสระจากนโยบายการแทนที่
</Accordion>

## `/model` ในแชต

```text
/model
/model list
/model 3
/model openai/gpt-5.4
/model default
/model status
```

- `/model` และ `/model list` แสดงตัวเลือกแบบมีหมายเลขขนาดกะทัดรัด (ตระกูลโมเดล + ผู้ให้บริการที่พร้อมใช้งาน); `/model <#>` ใช้เลือกจากรายการนี้ บน Discord การดำเนินการนี้จะเปิดรายการดรอปดาวน์ผู้ให้บริการ/โมเดลพร้อมขั้นตอน Submit; บน Telegram การเลือกจากตัวเลือกจะมีขอบเขตเฉพาะเซสชัน และจะไม่เขียนทับค่าเริ่มต้นถาวรของเอเจนต์ใน `openclaw.json` โดยเด็ดขาด `/models add` เลิกใช้แล้วและจะส่งข้อความกลับแทนการลงทะเบียนโมเดลจากแชต
- `/model` บันทึกการเลือกใหม่ของเซสชันทันที หากเอเจนต์ว่าง การรันครั้งถัดไปจะใช้ค่านี้ทันที; หากมีการรันอยู่แล้ว ระบบจะจัดคิวการสลับไว้สำหรับจุดลองใหม่ที่ปลอดภัยถัดไป (หรือจุดที่ช้ากว่านั้น หากกิจกรรมของเครื่องมือหรือการส่งคำตอบเริ่มขึ้นแล้ว)
- `/model default` ล้างการเลือกของเซสชันเพื่อให้กลับไปรับค่าหลักที่กำหนดไว้อีกครั้ง
- การอ้างอิง `/model` ที่ผู้ใช้เลือกจะมีผลแบบเคร่งครัดสำหรับเซสชันนั้น: หากไม่สามารถเข้าถึงได้ คำตอบจะแสดงข้อผิดพลาดอย่างชัดเจนแทนการย้อนกลับผ่าน `agents.defaults.model.fallbacks` โดยไม่แจ้งให้ทราบ ค่าเริ่มต้นที่กำหนดไว้และโมเดลหลักของงาน cron ยังคงใช้ลำดับการย้อนกลับ
- `/model status` คือมุมมองแบบละเอียด: ตัวเลือกการยืนยันตัวตนของผู้ให้บริการแต่ละราย และ (เมื่อกำหนดค่าไว้) ปลายทางผู้ให้บริการ `baseUrl` พร้อมโหมด `api`
- การอ้างอิงโมเดลจะถูกแยกโดยแบ่งที่ `/` ตัวแรก; ให้พิมพ์ `provider/model` หาก ID โมเดลมี `/` อยู่ภายใน (รูปแบบ OpenRouter) ให้ใส่คำนำหน้าผู้ให้บริการ เช่น `/model openrouter/moonshotai/kimi-k2` หากละเว้นผู้ให้บริการ OpenClaw จะลองตามลำดับ: (1) จับคู่ชื่อแทน (2) จับคู่ผู้ให้บริการที่กำหนดค่าไว้ซึ่งมีเพียงรายเดียวสำหรับ ID โมเดลที่ไม่มีคำนำหน้าตรงกันทุกประการ (3) ผู้ให้บริการเริ่มต้นที่กำหนดไว้ (การย้อนกลับที่เลิกใช้แล้ว) — และหากผู้ให้บริการนั้นไม่เปิดให้ใช้โมเดลเริ่มต้นที่กำหนดไว้อีกต่อไป ระบบจะใช้ผู้ให้บริการ/โมเดลคู่แรกที่กำหนดค่าไว้แทน เพื่อหลีกเลี่ยงการแสดงค่าเริ่มต้นของผู้ให้บริการที่ถูกลบซึ่งล้าสมัยแล้ว
- การอ้างอิงโมเดลจะถูกปรับให้เป็นตัวพิมพ์เล็ก; ส่วน ID ผู้ให้บริการต้องตรงตามตัวพิมพ์ทุกประการ ดังนั้นให้ใช้ ID ที่ Plugin ประกาศ

ลักษณะการทำงานของคำสั่งและการกำหนดค่าทั้งหมด: [คำสั่ง Slash](/th/tools/slash-commands)

## CLI

```bash
openclaw models status
openclaw models list
openclaw models set <provider/model>
openclaw models set-image <provider/model>
openclaw models scan
openclaw models aliases list|add|remove

openclaw models image-fallbacks list|add|remove|clear
openclaw models auth list|add|login|paste-api-key|paste-token|setup-token|order
```

`openclaw models` ที่ไม่มีคำสั่งย่อยเป็นทางลัดสำหรับ `models status` ซึ่งจะแสดงการหมดอายุของ OAuth สำหรับโปรไฟล์ในที่เก็บการยืนยันตัวตนด้วย (โดยค่าเริ่มต้นจะแจ้งเตือนภายใน 24h) แฟล็กทั้งหมด รูปแบบ JSON และคำสั่งย่อยของโปรไฟล์การยืนยันตัวตน: [ข้อมูลอ้างอิง CLI สำหรับโมเดล](/th/cli/models)

  <Accordion title="การสแกน (โมเดลฟรีของ OpenRouter)">
    `openclaw models scan` ตรวจสอบแค็ตตาล็อกโมเดลฟรีสาธารณะของ OpenRouter และสามารถทดสอบตัวเลือกที่เป็นไปได้แบบสดเพื่อดูการรองรับเครื่องมือและรูปภาพ ตัวแค็ตตาล็อกเป็นข้อมูลสาธารณะ ดังนั้นการสแกนเฉพาะข้อมูลเมตา (`--no-probe`) จึงไม่ต้องใช้คีย์; การทดสอบแบบสดและ `--set-default`/`--set-image` ต้องใช้คีย์ API ของ OpenRouter (โปรไฟล์การยืนยันตัวตนหรือ `OPENROUTER_API_KEY`) และหากไม่มีคีย์จะปิดการทำงานดังกล่าวโดยส่งออกเฉพาะข้อมูลเมตา

## รีจิสทรีโมเดล (`models.json`)

ผู้ให้บริการแบบกำหนดเองที่ตั้งค่าภายใต้ `models.providers` จะถูกเขียนลงใน `models.json` ภายใต้ไดเรกทอรีของเอเจนต์ (ค่าเริ่มต้นคือ `~/.openclaw/agents/<agentId>/agent/models.json`) แค็ตตาล็อกของ Plugin ผู้ให้บริการจะถูกจัดเก็บแยกกันเป็นส่วนย่อยของแค็ตตาล็อกที่สร้างขึ้นและเป็นของ Plugin โดยจะโหลดโดยอัตโนมัติ ไฟล์นี้จะถูกรวมกับการกำหนดค่าตามค่าเริ่มต้น; ตั้งค่า `models.mode: "replace"` เพื่อใช้เฉพาะผู้ให้บริการที่กำหนดค่าไว้

<AccordionGroup>
  <Accordion title="ลำดับความสำคัญของโหมดผสาน">
    สำหรับ ID ผู้ให้บริการที่ตรงกัน:

    - `baseUrl` ที่ไม่ว่างและมีอยู่แล้วใน `models.json` ของเอเจนต์จะมีลำดับความสำคัญสูงกว่า
    - `apiKey` ที่ไม่ว่างใน `models.json` จะมีลำดับความสำคัญสูงกว่าเฉพาะเมื่อผู้ให้บริการนั้นไม่ได้รับการจัดการด้วย SecretRef ในบริบทการกำหนดค่า/โปรไฟล์การยืนยันตัวตนปัจจุบัน
    - ค่า `apiKey` ที่จัดการด้วย SecretRef จะรีเฟรชจากมาร์กเกอร์แหล่งที่มาแทนการบันทึกข้อมูลลับที่แก้ไขค่าแล้ว: ใช้ชื่อของตัวแปรสภาพแวดล้อมสำหรับการอ้างอิง env และใช้ `secretref-managed` สำหรับการอ้างอิง file/exec
    - ค่าของส่วนหัวที่จัดการด้วย SecretRef จะรีเฟรชในลักษณะเดียวกัน โดยใช้ `secretref-env:ENV_VAR_NAME` สำหรับการอ้างอิง env
    - `apiKey`/`baseUrl` ที่ว่างหรือไม่มีอยู่ใน `models.json` จะย้อนกลับไปใช้ `models.providers` จากการกำหนดค่า
    - ฟิลด์อื่นของผู้ให้บริการจะรีเฟรชจากการกำหนดค่าและข้อมูลแค็ตตาล็อกที่ปรับให้เป็นมาตรฐานแล้ว

  </Accordion>
</AccordionGroup>

การบันทึกมาร์กเกอร์ถือว่าแหล่งที่มาเป็นข้อมูลหลัก: OpenClaw จะเขียนมาร์กเกอร์จากสแนปช็อตการกำหนดค่าแหล่งที่มาที่ใช้งานอยู่ (ก่อนแก้ไขค่า) ไม่ใช่จากค่าข้อมูลลับของรันไทม์ที่แก้ไขแล้ว เมื่อใดก็ตามที่สร้าง `models.json` ใหม่ — รวมถึงเส้นทางที่ขับเคลื่อนด้วยคำสั่ง เช่น `openclaw agent`
