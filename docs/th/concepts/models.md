การอ้างอิงโมเดล (`provider/model`) จะเลือกผู้ให้บริการและโมเดล ไม่ใช่รันไทม์เอเจนต์ระดับล่าง
เมื่อนโยบายรันไทม์ไม่ได้ตั้งค่าไว้หรือเป็น `auto` นโยบายเส้นทางที่ผู้ให้บริการ OpenAI เป็นเจ้าของ
อาจเลือก Codex เฉพาะสำหรับเส้นทาง Responses ของ Platform อย่างเป็นทางการผ่าน HTTPS
หรือเส้นทาง ChatGPT Responses ที่ตรงกันทุกประการและไม่มีการเขียนทับคำขอโดยผู้ใช้ ส่วนคำนำหน้า
`openai/*` เพียงอย่างเดียวจะไม่เลือก Codex ตัวปรับต่อ Completions, ปลายทาง
แบบกำหนดเอง และพฤติกรรมคำขอที่ผู้ใช้กำหนดยังคงทำงานบน OpenClaw ระบบจะปฏิเสธ
ปลายทาง HTTP แบบข้อความธรรมดาที่เป็นทางการ ดู [รันไทม์เอเจนต์โดยนัยของ OpenAI](/th/providers/openai#implicit-agent-runtime)

สามารถเลือกใช้การอ้างอิง Copilot แบบสมัครสมาชิก (`github-copilot/*`) กับ Plugin รันไทม์เอเจนต์
GitHub Copilot ภายนอกได้ แต่เส้นทางนั้นต้องระบุอย่างชัดเจนเสมอ (และจะไม่ถูกเลือกโดย
`auto`) การเขียนทับรันไทม์ต้องอยู่ในนโยบายของผู้ให้บริการ/โมเดล ไม่ใช่ใช้กับ
เอเจนต์หรือเซสชันทั้งหมด การเลือกรันไทม์ไม่ได้กำหนดการเรียกเก็บเงิน:
ข้อมูลประจำตัวของคีย์ API OpenAI และการสมัครสมาชิก ChatGPT/Codex ยังคงแยกจากกัน ดู
[รันไทม์เอเจนต์](/th/concepts/agent-runtimes) และ
[รันไทม์เอเจนต์ GitHub Copilot](/th/plugins/copilot)

## ลำดับการเลือก

<Steps>
  <Step title="โมเดลหลัก">
    `agents.defaults.model.primary` (หรือ `agents.defaults.model` เป็นสตริงธรรมดา)
  </Step>
  <Step title="โมเดลสำรอง">
    `agents.defaults.model.fallbacks` โดยลองตามลำดับ
  </Step>
  <Step title="การสลับการยืนยันตัวตนเมื่อเกิดข้อผิดพลาด">
    การหมุนเวียนโปรไฟล์การยืนยันตัวตนจะเกิดขึ้นภายในผู้ให้บริการ ก่อนที่ OpenClaw จะเปลี่ยนไปใช้โมเดลสำรองถัดไป
  </Step>
</Steps>

ส่วนการกำหนดค่าโมเดลที่เกี่ยวข้อง:

- `agents.defaults.models` คือรายการที่อนุญาต/แค็ตตาล็อกของโมเดลที่ OpenClaw ใช้งานได้ รวมถึงนามแฝง ใช้รายการ `provider/*` เพื่ออนุญาตทุกโมเดลที่ค้นพบจากผู้ให้บริการโดยไม่ต้องระบุทีละรายการ
- `agents.defaults.utilityModel` คือโมเดลต้นทุนต่ำกว่าที่เลือกใช้ได้สำหรับงานภายในระยะสั้น เช่น ชื่อเซสชันแดชบอร์ดที่สร้างขึ้น ชื่อเธรด/หัวข้อของช่องทางที่รองรับ และคำบรรยายความคืบหน้า โดย `agents.list[].utilityModel` ของแต่ละเอเจนต์จะเขียนทับค่านี้ เมื่อไม่ได้ตั้งค่า OpenClaw จะใช้ค่าเริ่มต้นของโมเดลขนาดเล็กที่ผู้ให้บริการหลักประกาศไว้ หากมี (OpenAI → `gpt-5.6-luna`, Anthropic → `claude-haiku-4-5`) มิฉะนั้นจะใช้โมเดลหลักของเอเจนต์ ตั้งค่าเป็นสตริงว่างเพื่อปิดการกำหนดเส้นทางงานอรรถประโยชน์ งานอรรถประโยชน์เป็นการเรียกโมเดลแยกต่างหากและอาจส่งเนื้อหางานที่จำกัดขอบเขตไปยังผู้ให้บริการโมเดลที่เลือก
- `agents.defaults.imageModel` จะใช้เฉพาะเมื่อโมเดลหลักไม่สามารถรับรูปภาพได้
- `agents.defaults.pdfModel` ใช้โดยเครื่องมือ `pdf` หากไม่ได้ตั้งค่า เครื่องมือจะย้อนกลับไปใช้ `imageModel` แล้วจึงใช้โมเดลที่ได้จากการแก้ค่าของเซสชัน/ค่าเริ่มต้น
- `agents.defaults.imageGenerationModel`, `musicGenerationModel` และ `videoGenerationModel` รองรับเครื่องมือสร้างสื่อที่ใช้ร่วมกัน หากไม่ได้ตั้งค่า แต่ละเครื่องมือจะอนุมานค่าเริ่มต้นของผู้ให้บริการที่มีการยืนยันตัวตนรองรับ โดยเริ่มจากผู้ให้บริการเริ่มต้นปัจจุบัน แล้วตามด้วยผู้ให้บริการที่ลงทะเบียนรายอื่นสำหรับความสามารถนั้นตามลำดับรหัสผู้ให้บริการ ตั้งค่า `agents.defaults.mediaGenerationAutoProviderFallback: false` เพื่อปิดการอนุมานข้ามผู้ให้บริการดังกล่าวโดยยังคงใช้โมเดลสำรองที่ระบุไว้อย่างชัดเจน
- `agents.list[].model` ของแต่ละเอเจนต์ (รวมถึงการผูก) จะเขียนทับ `agents.defaults.model` — ดู [การกำหนดเส้นทางแบบหลายเอเจนต์](/th/concepts/multi-agent)

เอกสารอ้างอิงคีย์ ค่าเริ่มต้น และตัวอย่าง JSON5 ฉบับสมบูรณ์: [เอกสารอ้างอิงการกำหนดค่า](/th/gateway/config-agents#agent-defaults)

## แหล่งที่มาของการเลือกและความเข้มงวดของโมเดลสำรอง

`provider/model` เดียวกันจะทำงานแตกต่างกันตามแหล่งที่มา:

| แหล่งที่มา                                                                  | พฤติกรรม                                                                                                                                                                                                                                                       |
| ----------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ค่าเริ่มต้นที่กำหนดไว้ (`agents.defaults.model.primary`, โมเดลหลักของแต่ละเอเจนต์) | จุดเริ่มต้นตามปกติ ใช้ `agents.defaults.model.fallbacks`                                                                                                                                                                                                 |
| โมเดลสำรองอัตโนมัติ                                                           | สถานะการกู้คืนชั่วคราว ซึ่งจัดเก็บเป็น `modelOverrideSource: "auto"` OpenClaw จะตรวจสอบโมเดลหลักเดิมซ้ำเป็นระยะ ล้างการเลือกอัตโนมัติเมื่อกู้คืนได้ และประกาศการเปลี่ยนไปใช้โมเดลสำรอง/การกู้คืนหนึ่งครั้งต่อการเปลี่ยนสถานะ                              |
| การเลือกเซสชันของผู้ใช้                                                  | ต้องตรงกันทุกประการและเข้มงวด `/model`, ตัวเลือกโมเดล, `session_status(model=...)` และ `sessions.patch` จะจัดเก็บ `modelOverrideSource: "user"` หากไม่สามารถเข้าถึงผู้ให้บริการ/โมเดลนั้นได้ การเรียกใช้จะล้มเหลวอย่างชัดเจนแทนที่จะเปลี่ยนไปใช้โมเดลอื่นที่กำหนดไว้ |
| Cron `--model` / เพย์โหลด `model`                                        | โมเดลหลักของแต่ละงาน ยังคงใช้โมเดลสำรองที่กำหนดไว้ เว้นแต่งานจะระบุเพย์โหลด `fallbacks` ของตนเอง (`fallbacks: []` บังคับให้เรียกใช้อย่างเข้มงวด)                                                                                                                    |

กฎการเลือกอื่นๆ:

- การเปลี่ยน `agents.defaults.model.primary` จะไม่เขียนพินของเซสชันที่มีอยู่ใหม่ หากสถานะรายงาน `This session is pinned to X; config primary Y will apply to new/unpinned sessions.` ให้เรียกใช้ `/model default` เพื่อล้างพิน
- ตัวเลือกโมเดลเริ่มต้นและรายการที่อนุญาตใน CLI จะปฏิบัติตาม `models.mode: "replace"` โดยแสดงเฉพาะ `models.providers.*.models` แทนแค็ตตาล็อกในตัวทั้งหมด
- ตัวเลือกโมเดลใน Control UI จะขอมุมมองโมเดลที่กำหนดค่าไว้จาก Gateway: ใช้ `agents.defaults.models` เมื่อตั้งค่าไว้ (รวมถึงรายการไวลด์การ์ด `provider/*`) มิฉะนั้นจะใช้ `models.providers.*.models` ร่วมกับผู้ให้บริการที่มีการยืนยันตัวตนพร้อมใช้งาน แค็ตตาล็อกในตัวทั้งหมดสงวนไว้สำหรับมุมมองเรียกดูที่ระบุอย่างชัดเจน (`models.list` พร้อม `view: "all"` หรือ `openclaw models list --all`)
- UI คลังผู้ให้บริการใช้ `models.list` พร้อม `view: "provider-config"` เพื่อแสดงแถว `models.providers.*.models` ที่ผู้เขียนแหล่งข้อมูลกำหนดไว้ โดยไม่ใช้รายการที่อนุญาตของตัวเลือก

กลไกฉบับเต็ม: [การสลับโมเดลเมื่อเกิดข้อผิดพลาด](/th/concepts/model-failover)

## นโยบายโมเดลฉบับย่อ

- ตั้งโมเดลหลักเป็นโมเดลรุ่นล่าสุดที่มีประสิทธิภาพสูงสุดซึ่งพร้อมให้ใช้งาน
- ใช้โมเดลสำรองสำหรับงานที่คำนึงถึงต้นทุน/เวลาแฝงและการสนทนาที่มีความเสี่ยงต่ำกว่า
- สำหรับเอเจนต์ที่เปิดใช้เครื่องมือหรืออินพุตที่ไม่น่าเชื่อถือ ให้หลีกเลี่ยงโมเดลระดับเก่ากว่า/ประสิทธิภาพต่ำกว่า

## การเริ่มต้นใช้งาน

```bash
openclaw onboard
```

ตั้งค่าโมเดลและการยืนยันตัวตนสำหรับผู้ให้บริการทั่วไปโดยไม่ต้องแก้ไขการกำหนดค่าด้วยตนเอง รวมถึง OAuth สำหรับการสมัครสมาชิก OpenAI Codex และ Anthropic (คีย์ API หรือใช้ข้อมูลจาก Claude CLI ซ้ำ)

เมื่อยังไม่ได้กำหนดค่าโมเดลหลัก การตั้งค่าคีย์ API OpenAI ใหม่จะเลือก
`openai/gpt-5.6` โดยรหัส API โดยตรงที่ไม่มีส่วนขยายจะถูกแก้เป็นระดับ Sol การตั้งค่า
OAuth ของ ChatGPT/Codex ใหม่จะเลือกการอ้างอิงแค็ตตาล็อก `openai/gpt-5.6-sol` ที่ตรงกันทุกประการ
การยืนยันตัวตนซ้ำจะคงโมเดลหลักที่ระบุไว้อย่างชัดเจนเดิมไว้ รวมถึง
`openai/gpt-5.5` หากบัญชีไม่สามารถใช้ GPT-5.6 ได้ ให้เลือก
`openai/gpt-5.5` อย่างชัดเจน OpenClaw จะไม่ลดระดับให้โดยอัตโนมัติ

## "Model is not allowed" (และสาเหตุที่การตอบกลับหยุดลง)

หากตั้งค่า `agents.defaults.models` ค่านี้จะกลายเป็นรายการที่อนุญาตสำหรับ `/model` และการเขียนทับของเซสชัน การเลือกโมเดลที่อยู่นอกรายการที่อนุญาตจะส่งคืนข้อความต่อไปนี้ก่อนที่จะสร้างการตอบกลับตามปกติ:

```text
ไม่อนุญาตโมเดล "provider/model" ใช้ /models เพื่อแสดงรายชื่อผู้ให้บริการ หรือ /models <provider> เพื่อแสดงรายชื่อโมเดล
เพิ่มโมเดลด้วย: openclaw config set agents.defaults.models '{"provider/model":{}}' --strict-json --merge
```

แก้ไขโดยเพิ่มโมเดลลงใน `agents.defaults.models` ล้างรายการที่อนุญาตทั้งหมด (นำคีย์ออก) หรือเลือกโมเดลจาก `/model list` หากคำสั่งที่ถูกปฏิเสธมีการเขียนทับรันไทม์ เช่น `/model openai/gpt-5.5 --runtime codex` ให้แก้ไขรายการที่อนุญาตก่อน แล้วลองใช้คำสั่ง `/model ... --runtime ...` เดิมอีกครั้ง

สำหรับโมเดลภายในเครื่อง/GGUF รายการที่อนุญาตต้องมีการอ้างอิงเต็มรูปแบบพร้อมคำนำหน้าผู้ให้บริการ เช่น `ollama/gemma4:26b` หรือ `lmstudio/Gemma4-26b-a4-it-gguf` — ตรวจสอบสตริงที่ถูกต้องจาก `openclaw models list --provider <provider>` เมื่อรายการที่อนุญาตทำงานอยู่ แค่ชื่อไฟล์หรือชื่อที่แสดงอย่างเดียวไม่เพียงพอ

หากต้องการจำกัดผู้ให้บริการโดยไม่ระบุโมเดลทุกตัว ให้ใช้รายการไวลด์การ์ด `provider/*`:

```json5
{
  agents: {
    defaults: {
      models: {
        "openai/*": {},
        "vllm/*": {},
      },
    },
  },
}
```

จากนั้น `/model`, `/models` และตัวเลือกโมเดลจะแสดงแค็ตตาล็อกที่ค้นพบสำหรับผู้ให้บริการเหล่านั้นเท่านั้น และโมเดลใหม่สามารถปรากฏขึ้นได้โดยไม่ต้องแก้ไขรายการที่อนุญาต ผสมรายการ `provider/model` แบบระบุแน่นอนกับรายการ `provider/*` เพื่อดึงโมเดลเฉพาะหนึ่งรายการจากผู้ให้บริการรายอื่นเข้ามา

ตัวอย่างรายการที่อนุญาตพร้อมนามแฝง:

```json5
{
  agents: {
    defaults: {
      model: { primary: "anthropic/claude-sonnet-4-6" },
      models: {
        "anthropic/claude-sonnet-4-6": { alias: "Sonnet" },
        "anthropic/claude-opus-4-6": { alias: "Opus" },
      },
    },
  },
}
```

<Accordion title="การแก้ไขรายการที่อนุญาตอย่างปลอดภัยจาก CLI">
ใช้ `--merge` สำหรับการเปลี่ยนแปลงแบบเพิ่มเติม:

```bash
openclaw config set agents.defaults.models '{"openai/gpt-5.4":{}}' --strict-json --merge
```

`openclaw config set` จะปฏิเสธการกำหนดออบเจ็กต์ธรรมดาให้กับ `agents.defaults.models`, `models.providers` หรือ `models.providers.<id>.models` เมื่อการกำหนดนั้นจะทำให้รายการที่มีอยู่หายไป ใช้ `--replace` เฉพาะเมื่อค่าใหม่ควรเป็นค่าเป้าหมายทั้งหมด การตั้งค่าผู้ให้บริการแบบโต้ตอบและ `openclaw configure --section model` จะผสานการเลือกที่มีขอบเขตตามผู้ให้บริการเข้ากับรายการที่อนุญาตอยู่แล้ว ดังนั้นการเพิ่มผู้ให้บริการจะไม่ทำให้รายการที่ไม่เกี่ยวข้องหายไป และการกำหนดค่าจะคง `agents.defaults.model.primary` ที่มีอยู่ไว้ คำสั่งที่ระบุอย่างชัดเจน เช่น `openclaw models auth login --provider <id> --set-default` และ `openclaw models set <model>` จะยังคงแทนที่โมเดลหลัก
</Accordion>

## `/model` ในการสนทนา

```text
/model
/model list
/model 3
/model openai/gpt-5.4
/model default
/model status
```

- `/model` และ `/model list` แสดงตัวเลือกแบบมีหมายเลขขนาดกะทัดรัด (ตระกูลโมเดล + ผู้ให้บริการที่พร้อมใช้งาน); `/model <#>` ใช้เลือกจากรายการนี้ บน Discord การดำเนินการนี้จะเปิดเมนูแบบเลื่อนลงสำหรับผู้ให้บริการ/โมเดลพร้อมขั้นตอน Submit; บน Telegram รายการที่เลือกจะมีขอบเขตเฉพาะเซสชันและจะไม่เขียนทับค่าเริ่มต้นถาวรของเอเจนต์ใน `openclaw.json` โดยเด็ดขาด `/models add` เลิกใช้แล้วและจะส่งข้อความตอบกลับแทนการลงทะเบียนโมเดลจากแชต
- `/model` บันทึกรายการที่เลือกใหม่สำหรับเซสชันทันที หากเอเจนต์ไม่ได้ทำงาน การรันครั้งถัดไปจะใช้รายการนั้นทันที; หากมีการรันอยู่แล้ว การสลับจะเข้าคิวสำหรับจุดลองใหม่ที่ไม่มีสถานะค้างครั้งถัดไป (หรือจุดหลังจากนั้น หากกิจกรรมของเครื่องมือหรือการส่งคำตอบเริ่มขึ้นแล้ว)
- `/model default` ล้างรายการที่เลือกสำหรับเซสชัน เพื่อให้กลับไปรับค่าหลักที่กำหนดไว้อีกครั้ง
- การอ้างอิง `/model` ที่ผู้ใช้เลือกมีผลอย่างเคร่งครัดสำหรับเซสชันนั้น: หากไม่สามารถเข้าถึงได้ การตอบกลับจะล้มเหลวอย่างชัดเจนแทนการย้อนกลับผ่าน `agents.defaults.model.fallbacks` โดยไม่มีการแจ้งเตือน ค่าเริ่มต้นที่กำหนดไว้และโมเดลหลักของงาน Cron ยังคงใช้ลำดับการย้อนกลับ
- `/model status` คือมุมมองโดยละเอียด: ตัวเลือกการยืนยันตัวตนของผู้ให้บริการแต่ละราย และ (เมื่อกำหนดค่าไว้) เอนด์พอยต์ของผู้ให้บริการ `baseUrl` พร้อมโหมด `api`
- การอ้างอิงโมเดลจะถูกแยกวิเคราะห์โดยแบ่งที่ `/` ตัวแรก; ให้พิมพ์ `provider/model` หาก ID โมเดลมี `/` อยู่ภายใน (รูปแบบ OpenRouter) ให้ใส่คำนำหน้าผู้ให้บริการ เช่น `/model openrouter/moonshotai/kimi-k2` หากไม่ระบุผู้ให้บริการ OpenClaw จะลองตามลำดับ: (1) จับคู่นามแฝง (2) จับคู่ผู้ให้บริการที่กำหนดค่าไว้เพียงรายเดียวสำหรับ ID โมเดลที่ไม่มีคำนำหน้าตรงกันนั้น (3) ผู้ให้บริการเริ่มต้นที่กำหนดไว้ (ทางเลือกย้อนกลับที่เลิกใช้แล้ว) — และหากผู้ให้บริการนั้นไม่ให้บริการโมเดลเริ่มต้นที่กำหนดไว้อีกต่อไป จะใช้ผู้ให้บริการ/โมเดลคู่แรกที่กำหนดค่าไว้แทน เพื่อหลีกเลี่ยงการแสดงค่าเริ่มต้นของผู้ให้บริการที่ถูกลบซึ่งล้าสมัย
- การอ้างอิงโมเดลจะถูกปรับเป็นตัวพิมพ์เล็ก; ส่วน ID ผู้ให้บริการต้องตรงตามที่ระบุทุกประการ ดังนั้นให้ใช้ ID ที่ Plugin ประกาศ

พฤติกรรมคำสั่งและการกำหนดค่าทั้งหมด: [คำสั่งแบบเครื่องหมายทับ](/th/tools/slash-commands)

## CLI

```bash
openclaw models status
openclaw models list
openclaw models set <provider/model>
openclaw models set-image <provider/model>
openclaw models scan
openclaw models aliases list|add|remove
openclaw models fallbacks list|add|remove|clear
openclaw models image-fallbacks list|add|remove|clear
openclaw models auth list|add|login|paste-api-key|paste-token|setup-token|order
```

`openclaw models` ที่ไม่มีคำสั่งย่อยเป็นทางลัดสำหรับ `models status` ซึ่งจะแสดงเวลาหมดอายุของ OAuth สำหรับโปรไฟล์ในที่จัดเก็บการยืนยันตัวตนด้วย (โดยค่าเริ่มต้นจะแจ้งเตือนภายใน 24h) แฟล็กทั้งหมด โครงสร้าง JSON และคำสั่งย่อยของโปรไฟล์การยืนยันตัวตน: [เอกสารอ้างอิง CLI สำหรับโมเดล](/th/cli/models)

<AccordionGroup>
  <Accordion title="การสแกน (โมเดลฟรีของ OpenRouter)">
    `openclaw models scan` ตรวจสอบแค็ตตาล็อกโมเดลฟรีสาธารณะของ OpenRouter และสามารถทดสอบตัวเลือกแบบสดว่ารองรับเครื่องมือและรูปภาพหรือไม่ ตัวแค็ตตาล็อกเป็นข้อมูลสาธารณะ ดังนั้นการสแกนเฉพาะข้อมูลเมตา (`--no-probe`) จึงไม่ต้องใช้คีย์; การทดสอบแบบสดและ `--set-default`/`--set-image` ต้องใช้คีย์ API ของ OpenRouter (โปรไฟล์การยืนยันตัวตนหรือ `OPENROUTER_API_KEY`) และหากไม่มีคีย์จะปฏิเสธการดำเนินการโดยแสดงเฉพาะผลลัพธ์จากข้อมูลเมตา

    ผลลัพธ์จะจัดอันดับตาม: การรองรับรูปภาพ ตามด้วยเวลาแฝงของเครื่องมือ ขนาดบริบท และจำนวนพารามิเตอร์ ใน TTY ผลลัพธ์ที่ผ่านการทดสอบจะแสดงข้อความให้เลือกทางเลือกย้อนกลับแบบโต้ตอบ; โหมดที่ไม่โต้ตอบต้องใช้ `--yes` เพื่อยอมรับค่าเริ่มต้น

</AccordionGroup>

## รีจิสทรีโมเดล (`models.json`)

ผู้ให้บริการแบบกำหนดเองที่กำหนดค่าไว้ภายใต้ `models.providers` จะถูกเขียนลงใน `models.json` ภายในไดเรกทอรีเอเจนต์ (ค่าเริ่มต้น `~/.openclaw/agents/<agentId>/agent/models.json`) แค็ตตาล็อกของ Plugin ผู้ให้บริการจะจัดเก็บแยกเป็นส่วนย่อยของแค็ตตาล็อกที่สร้างขึ้นและอยู่ภายใต้การดูแลของ Plugin พร้อมโหลดโดยอัตโนมัติ โดยค่าเริ่มต้น ไฟล์นี้จะถูกรวมกับการกำหนดค่า; ตั้งค่า `models.mode: "replace"` เพื่อใช้เฉพาะผู้ให้บริการที่กำหนดค่าไว้

<AccordionGroup>
  <Accordion title="ลำดับความสำคัญของโหมดรวม">
    สำหรับ ID ผู้ให้บริการที่ตรงกัน:

    - `baseUrl` ที่ไม่ว่างซึ่งมีอยู่แล้วใน `models.json` ของเอเจนต์จะมีลำดับความสำคัญสูงกว่า
    - `apiKey` ที่ไม่ว่างใน `models.json` จะมีลำดับความสำคัญสูงกว่าเฉพาะเมื่อผู้ให้บริการนั้นไม่ได้อยู่ภายใต้การจัดการของ SecretRef ในบริบทการกำหนดค่า/โปรไฟล์การยืนยันตัวตนปัจจุบัน
    - ค่า `apiKey` ที่อยู่ภายใต้การจัดการของ SecretRef จะรีเฟรชจากเครื่องหมายแหล่งที่มาแทนการบันทึกข้อมูลลับที่แก้ค่าแล้ว: ชื่อตัวแปรสภาพแวดล้อมสำหรับการอ้างอิง env และ `secretref-managed` สำหรับการอ้างอิงไฟล์/exec
    - ค่าส่วนหัวที่อยู่ภายใต้การจัดการของ SecretRef จะรีเฟรชในลักษณะเดียวกัน โดยใช้ `secretref-env:ENV_VAR_NAME` สำหรับการอ้างอิง env
    - `apiKey`/`baseUrl` ที่ว่างหรือไม่มีอยู่ใน `models.json` จะย้อนกลับไปใช้ `models.providers` จากการกำหนดค่า
    - ฟิลด์อื่นของผู้ให้บริการจะรีเฟรชจากการกำหนดค่าและข้อมูลแค็ตตาล็อกที่ปรับให้อยู่ในรูปแบบมาตรฐานแล้ว

  </Accordion>
</AccordionGroup>

การคงอยู่ของเครื่องหมายยึดแหล่งที่มาเป็นหลัก: OpenClaw จะเขียนเครื่องหมายจากสแนปช็อตการกำหนดค่าแหล่งที่มาที่ใช้งานอยู่ (ก่อนการแก้ค่า) ไม่ใช่จากค่าข้อมูลลับของรันไทม์ที่แก้ค่าแล้ว ทุกครั้งที่สร้าง `models.json` ใหม่ — รวมถึงเส้นทางที่ขับเคลื่อนด้วยคำสั่ง เช่น `openclaw agent`
