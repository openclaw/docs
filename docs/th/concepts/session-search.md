# การค้นหาเซสชัน

`sessions_search` ค้นหาข้อความของผู้ใช้และผู้ช่วยในเซสชันที่ผ่านมาของคุณเอง ผลลัพธ์แต่ละรายการ
ประกอบด้วย `sessionKey` การประทับเวลา บทบาท และข้อความส่วนที่ตรงกันแบบสั้น ส่ง
`sessionKey` ที่ได้คืนไปยัง `sessions_history` เมื่อต้องการดูบทสนทนาโดยรอบ

## การมองเห็นและผลลัพธ์

การค้นหาใช้กฎการมองเห็นเซสชันเดียวกับ `sessions_history` ผลลัพธ์ที่อยู่นอกโครงสร้างเซสชัน
ที่ผู้เรียกมองเห็นจะถูกนำออกก่อนใช้ขีดจำกัดจำนวนผลลัพธ์ เอเจนต์ในแซนด์บ็อกซ์ยังคงถูกจำกัด
ให้เข้าถึงเฉพาะเซสชันที่ตนสร้างขึ้น เมื่อเปิดใช้การมองเห็นเซสชันที่สร้างโดยเอเจนต์

ข้อความส่วนที่ยกมาจะถูกปกปิดข้อมูลก่อนส่งกลับไปยังโมเดล นอกจากนี้ ผลลัพธ์ยังถูกจำกัดตามจำนวน ความยาว
ของข้อความส่วนที่ยกมา และขนาดรวมของการตอบกลับ

## วงจรชีวิตของดัชนี

OpenClaw จัดเก็บดัชนีข้อความแบบเต็มไว้ข้างแถวบทถอดความในฐานข้อมูล SQLite ของแต่ละเอเจนต์
ข้อความใหม่จากผู้ใช้และผู้ช่วยจะได้รับการจัดทำดัชนีในธุรกรรมเดียวกับที่บันทึกข้อความนั้น ดังนั้น
ดัชนีจึงตามทันบทสนทนาปัจจุบันเสมอ โดยไม่รวมผลลัพธ์จากเครื่องมือ บล็อกการให้เหตุผล และรูปภาพ
ค้นหาได้เฉพาะสาขาที่ใช้งานอยู่ของบทถอดความเท่านั้น

บทถอดความที่มีอยู่ก่อนดัชนี (เช่น เซสชันที่นำเข้าด้วย `openclaw doctor`) และ
เซสชันที่มีการย้อนกลับสาขาที่ใช้งานอยู่ จะได้รับการจัดทำดัชนีใหม่โดยกระบวนการกระทบยอดเบื้องหลังซึ่งเริ่ม
พร้อมการค้นหาครั้งถัดไป ดังนั้นการตอบกลับที่มี `indexing: true` อาจไม่ครบถ้วน ให้ลองอีกครั้งหลัง
การจัดทำดัชนีเสร็จสิ้น การลบเซสชันจะนำรายการดัชนีของเซสชันนั้นออกในธุรกรรมเดียวกัน

ปัจจุบันการค้นหาใช้ตัวตัดคำ Unicode ของ SQLite พร้อมการลบเครื่องหมายกำกับเสียง การแบ่งโทเค็นแบบไตรแกรม
สำหรับการจับคู่ข้อความย่อยในภาษา CJK เป็นการปรับปรุงในอนาคต

## การค้นหาเซสชันเทียบกับการค้นหาหน่วยความจำ

ใช้ `sessions_search` สำหรับคำหรือวลีที่ตรงกันทุกประการจากบทถอดความเซสชันดิบ ใช้
[`memory_search`](/th/concepts/memory-search) สำหรับไฟล์หน่วยความจำถาวรและการเรียกคืนเชิงความหมาย คลังข้อมูล
หน่วยความจำเซสชันแบบทดลองเป็นส่วนเสริมเชิงความหมายของการค้นหาบทถอดความแบบตรงกันทุกประการนี้
