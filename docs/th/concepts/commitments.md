ข้อผูกพันคือความทรงจำสำหรับการติดตามผลที่มีอายุสั้น เมื่อเปิดใช้งาน OpenClaw สามารถ
สังเกตได้ว่าการสนทนาสร้างโอกาสให้กลับมาตรวจสอบในอนาคต และจดจำไว้
เพื่อนำกลับมาในภายหลัง

ตัวอย่าง:

- คุณพูดถึงการสัมภาษณ์ในวันพรุ่งนี้ OpenClaw อาจกลับมาตรวจสอบหลังจากนั้น
- คุณบอกว่าคุณหมดแรง OpenClaw อาจถามในภายหลังว่าคุณได้นอนหรือไม่
- เอเจนต์บอกว่าจะติดตามผลหลังจากมีบางอย่างเปลี่ยนแปลง OpenClaw อาจติดตาม
  วงจรที่ยังเปิดอยู่นั้น

ข้อผูกพันไม่ใช่ข้อเท็จจริงถาวรเหมือน `MEMORY.md` และไม่ใช่
การเตือนความจำแบบตรงตัว ข้อผูกพันอยู่กึ่งกลางระหว่างหน่วยความจำกับระบบอัตโนมัติ: OpenClaw จดจำ
ภาระผูกพันที่ผูกกับการสนทนา จากนั้น Heartbeat จะส่งมอบเมื่อถึงกำหนด

## เปิดใช้งานข้อผูกพัน

ข้อผูกพันปิดอยู่โดยค่าเริ่มต้น เปิดใช้งานใน config:

```bash
openclaw config set commitments.enabled true
openclaw config set commitments.maxPerDay 3
```

`openclaw.json` ที่เทียบเท่า:

```json
{
  "commitments": {
    "enabled": true,
    "maxPerDay": 3
  }
}
```

`commitments.maxPerDay` จำกัดจำนวนการติดตามผลที่อนุมานได้ซึ่งสามารถส่งมอบ
ต่อเซสชันเอเจนต์ในหนึ่งวันแบบเลื่อนเวลาได้ ค่าเริ่มต้นคือ `3`

## วิธีการทำงาน

หลังจากเอเจนต์ตอบกลับ OpenClaw อาจรันขั้นตอนการดึงข้อมูลพื้นหลังที่ซ่อนอยู่ใน
บริบทแยกต่างหาก ขั้นตอนนั้นจะมองหาเฉพาะข้อผูกพันในการติดตามผลที่อนุมานได้เท่านั้น ขั้นตอนนี้
จะไม่เขียนลงในการสนทนาที่มองเห็นได้ และจะไม่ขอให้เอเจนต์หลัก
ให้เหตุผลเกี่ยวกับการดึงข้อมูล

เมื่อพบผู้สมัครที่มีความมั่นใจสูง OpenClaw จะจัดเก็บข้อผูกพันพร้อมกับ:

- id ของเอเจนต์
- คีย์เซสชัน
- ช่องทางต้นทางและเป้าหมายการส่งมอบ
- ช่วงเวลาที่ถึงกำหนด
- ข้อความตรวจสอบสั้น ๆ ที่แนะนำ
- บริบทต้นทางเพียงพอให้ Heartbeat ตัดสินใจว่าจะส่งหรือไม่

การส่งมอบเกิดขึ้นผ่าน Heartbeat เมื่อข้อผูกพันถึงกำหนด Heartbeat
จะเพิ่มข้อผูกพันนั้นลงในรอบ Heartbeat สำหรับเอเจนต์และขอบเขตช่องทางเดียวกัน
โมเดลสามารถส่งข้อความตรวจสอบอย่างเป็นธรรมชาติหนึ่งครั้ง หรือตอบ `HEARTBEAT_OK` เพื่อยกเลิกได้

OpenClaw จะไม่ส่งมอบข้อผูกพันที่อนุมานได้ทันทีหลังจากเขียน
เวลาที่ถึงกำหนดจะถูกจำกัดให้ห่างออกไปอย่างน้อยหนึ่งช่วง Heartbeat หลังจากสร้างข้อผูกพัน
ดังนั้นการติดตามผลจึงไม่สามารถสะท้อนกลับมาในขณะเดียวกับที่ถูก
อนุมานได้

## ขอบเขต

ข้อผูกพันถูกจำกัดอยู่ในบริบทเอเจนต์และช่องทางที่แน่นอนซึ่งถูก
สร้างขึ้น การติดตามผลที่อนุมานระหว่างคุยกับเอเจนต์หนึ่งใน Discord จะไม่ถูก
ส่งมอบโดยเอเจนต์อื่น ช่องทางอื่น หรือเซสชันที่ไม่เกี่ยวข้อง

ขอบเขตนี้เป็นส่วนหนึ่งของฟีเจอร์ การตรวจสอบอย่างเป็นธรรมชาติควรรู้สึกเหมือนเป็น
การสนทนาเดิมที่ดำเนินต่อไป ไม่ใช่ระบบเตือนความจำแบบทั่วโลก

## ข้อผูกพันกับการเตือนความจำ

| ความต้องการ                                     | ใช้                                      |
| ----------------------------------------------- | ---------------------------------------- |
| "เตือนฉันตอน 15:00 น."                         | [งานที่ตั้งเวลาไว้](/th/automation/cron-jobs) |
| "ส่ง ping หาฉันในอีก 20 นาที"                  | [งานที่ตั้งเวลาไว้](/th/automation/cron-jobs) |
| "รันรายงานนี้ทุกวันธรรมดา"                    | [งานที่ตั้งเวลาไว้](/th/automation/cron-jobs) |
| "ฉันมีสัมภาษณ์พรุ่งนี้"                       | ข้อผูกพัน                              |
| "ฉันไม่ได้นอนทั้งคืน"                          | ข้อผูกพัน                              |
| "ติดตามผลถ้าฉันไม่ตอบเธรดที่ยังเปิดอยู่นี้" | ข้อผูกพัน                              |

คำขอของผู้ใช้แบบตรงตัวควรไปอยู่ในเส้นทางตัวตั้งเวลาอยู่แล้ว ข้อผูกพันมีไว้เฉพาะ
สำหรับการติดตามผลที่อนุมานได้: ช่วงเวลาที่ผู้ใช้ไม่ได้ขอการเตือนความจำ
แต่การสนทนาสร้างการตรวจสอบในอนาคตที่มีประโยชน์อย่างชัดเจน

## จัดการข้อผูกพัน

ใช้ CLI เพื่อตรวจสอบและล้างข้อผูกพันที่จัดเก็บไว้:

```bash
openclaw commitments
openclaw commitments --all
openclaw commitments --agent main
openclaw commitments --status snoozed
openclaw commitments dismiss cm_abc123
```

ดู [`openclaw commitments`](/th/cli/commitments) สำหรับเอกสารอ้างอิงคำสั่ง

## ความเป็นส่วนตัวและค่าใช้จ่าย

การดึงข้อมูลข้อผูกพันใช้ขั้นตอน LLM ดังนั้นการเปิดใช้งานจะเพิ่มการใช้โมเดล
พื้นหลังหลังจากรอบที่เข้าเงื่อนไข ขั้นตอนนี้ถูกซ่อนไว้จากการสนทนา
ที่ผู้ใช้มองเห็น แต่สามารถอ่านการแลกเปลี่ยนล่าสุดที่จำเป็นเพื่อพิจารณาว่า
มีการติดตามผลหรือไม่

ข้อผูกพันที่จัดเก็บเป็นสถานะในเครื่องของ OpenClaw เป็นหน่วยความจำเชิงปฏิบัติการ ไม่ใช่
หน่วยความจำระยะยาว ปิดใช้งานฟีเจอร์ด้วย:

```bash
openclaw config set commitments.enabled false
```

## การแก้ไขปัญหา

หากการติดตามผลที่คาดไว้ไม่ปรากฏ:

- ยืนยันว่า `commitments.enabled` เป็น `true`
- ตรวจสอบ `openclaw commitments --all` สำหรับระเบียนที่รอดำเนินการ ถูกยกเลิก เลื่อนเวลา หรือหมดอายุ
- ตรวจสอบให้แน่ใจว่า Heartbeat กำลังทำงานสำหรับเอเจนต์
- ตรวจสอบว่า `commitments.maxPerDay` ถึงขีดจำกัดสำหรับ
  เซสชันเอเจนต์นั้นแล้วหรือไม่
- จำไว้ว่าการเตือนความจำแบบตรงตัวจะถูกข้ามโดยการดึงข้อมูลข้อผูกพัน และควร
  ปรากฏภายใต้ [งานที่ตั้งเวลาไว้](/th/automation/cron-jobs) แทน
