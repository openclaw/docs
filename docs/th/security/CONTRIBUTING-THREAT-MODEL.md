# การมีส่วนร่วมในแบบจำลองภัยคุกคามของ OpenClaw

ขอบคุณที่ช่วยทำให้ OpenClaw ปลอดภัยยิ่งขึ้น แบบจำลองภัยคุกคามนี้เป็นเอกสารที่ปรับปรุงอย่างต่อเนื่อง และเรายินดีรับการมีส่วนร่วมจากทุกคน - คุณไม่จำเป็นต้องเป็นผู้เชี่ยวชาญด้านความปลอดภัย

## วิธีมีส่วนร่วม

### เพิ่มภัยคุกคาม

พบเวกเตอร์การโจมตีหรือความเสี่ยงที่เรายังไม่ได้ครอบคลุมหรือไม่? เปิด issue ใน [openclaw/trust](https://github.com/openclaw/trust/issues) แล้วอธิบายด้วยถ้อยคำของคุณเอง คุณไม่จำเป็นต้องรู้จักเฟรมเวิร์กใด ๆ หรือกรอกข้อมูลทุกช่อง - เพียงอธิบายสถานการณ์ก็พอ

**ข้อมูลที่มีประโยชน์หากใส่มาด้วย (แต่ไม่บังคับ):**

- สถานการณ์การโจมตีและวิธีที่อาจถูกนำไปใช้โจมตี
- ส่วนใดของ OpenClaw ที่ได้รับผลกระทบ (CLI, Gateway, ช่องทาง, ClawHub, เซิร์ฟเวอร์ MCP ฯลฯ)
- คุณคิดว่าระดับความรุนแรงเป็นอย่างไร (ต่ำ / กลาง / สูง / วิกฤต)
- ลิงก์ใด ๆ ไปยังงานวิจัยที่เกี่ยวข้อง, CVE หรือตัวอย่างจากโลกจริง

เราจะจัดการการแมป ATLAS, ID ภัยคุกคาม และการประเมินความเสี่ยงระหว่างการรีวิว หากคุณต้องการใส่รายละเอียดเหล่านั้นมาด้วยก็ยินดี - แต่ไม่ได้คาดหวังว่าต้องมี

> **สิ่งนี้มีไว้สำหรับเพิ่มข้อมูลลงในแบบจำลองภัยคุกคาม ไม่ใช่สำหรับรายงานช่องโหว่ที่ใช้งานโจมตีได้จริง** หากคุณพบช่องโหว่ที่สามารถถูกใช้โจมตีได้ โปรดดู [หน้า Trust](https://trust.openclaw.ai) ของเราสำหรับคำแนะนำเกี่ยวกับการเปิดเผยช่องโหว่อย่างรับผิดชอบ

### เสนอแนวทางบรรเทา

มีแนวคิดสำหรับจัดการภัยคุกคามที่มีอยู่หรือไม่? เปิด issue หรือ PR ที่อ้างอิงถึงภัยคุกคามนั้น แนวทางบรรเทาที่มีประโยชน์ควรเฉพาะเจาะจงและนำไปปฏิบัติได้ - ตัวอย่างเช่น "จำกัดอัตราต่อผู้ส่งที่ 10 ข้อความ/นาทีที่ Gateway" ดีกว่า "ใช้การจำกัดอัตรา"

### เสนอห่วงโซ่การโจมตี

ห่วงโซ่การโจมตีแสดงให้เห็นว่าภัยคุกคามหลายรายการรวมกันเป็นสถานการณ์การโจมตีที่สมจริงได้อย่างไร หากคุณเห็นการผสมผสานที่อันตราย ให้อธิบายขั้นตอนและวิธีที่ผู้โจมตีจะเชื่อมโยงขั้นตอนเหล่านั้นเข้าด้วยกัน คำบรรยายสั้น ๆ ว่าการโจมตีเกิดขึ้นจริงอย่างไรมีคุณค่ามากกว่าเทมเพลตที่เป็นทางการ

### แก้ไขหรือปรับปรุงเนื้อหาที่มีอยู่

คำผิด การชี้แจง ข้อมูลที่ล้าสมัย ตัวอย่างที่ดีกว่า - ยินดีรับ PR ไม่จำเป็นต้องเปิด issue

## สิ่งที่เราใช้

### MITRE ATLAS

แบบจำลองภัยคุกคามนี้สร้างบน [MITRE ATLAS](https://atlas.mitre.org/) (ภูมิทัศน์ภัยคุกคามจากฝ่ายตรงข้ามสำหรับระบบ AI) ซึ่งเป็นเฟรมเวิร์กที่ออกแบบมาโดยเฉพาะสำหรับภัยคุกคามด้าน AI/ML เช่น prompt injection, การใช้เครื่องมือในทางที่ผิด และการโจมตี agent คุณไม่จำเป็นต้องรู้จัก ATLAS เพื่อมีส่วนร่วม - เราจะแมปข้อเสนอเข้ากับเฟรมเวิร์กระหว่างการรีวิว

### ID ภัยคุกคาม

ภัยคุกคามแต่ละรายการจะได้รับ ID เช่น `T-EXEC-003` หมวดหมู่มีดังนี้:

| รหัส    | หมวดหมู่                                   |
| ------- | ------------------------------------------ |
| RECON   | การลาดตระเวน - การรวบรวมข้อมูล     |
| ACCESS  | การเข้าถึงเริ่มต้น - การได้สิทธิ์เข้าใช้งาน             |
| EXEC    | การดำเนินการ - การรันการกระทำที่เป็นอันตราย      |
| PERSIST | การคงอยู่ - การรักษาสิทธิ์เข้าถึง           |
| EVADE   | การหลบเลี่ยงการป้องกัน - การหลีกเลี่ยงการตรวจจับ       |
| DISC    | การค้นพบ - การเรียนรู้เกี่ยวกับสภาพแวดล้อม |
| EXFIL   | การลักลอบนำข้อมูลออก - การขโมยข้อมูล               |
| IMPACT  | ผลกระทบ - ความเสียหายหรือการหยุดชะงัก              |

ผู้ดูแลจะกำหนด ID ระหว่างการรีวิว คุณไม่จำเป็นต้องเลือกเอง

### ระดับความเสี่ยง

| ระดับ        | ความหมาย                                                           |
| ------------ | ----------------------------------------------------------------- |
| **วิกฤต** | ระบบถูกยึดครองทั้งหมด หรือมีความเป็นไปได้สูง + ผลกระทบขั้นวิกฤต      |
| **สูง**     | มีแนวโน้มเกิดความเสียหายอย่างมีนัยสำคัญ หรือมีความเป็นไปได้ปานกลาง + ผลกระทบขั้นวิกฤต |
| **กลาง**   | ความเสี่ยงปานกลาง หรือมีความเป็นไปได้ต่ำ + ผลกระทบสูง                    |
| **ต่ำ**      | ไม่น่าจะเกิดขึ้นและมีผลกระทบจำกัด                                       |

หากคุณไม่แน่ใจเกี่ยวกับระดับความเสี่ยง เพียงอธิบายผลกระทบมา แล้วเราจะประเมินให้

## กระบวนการรีวิว

1. **การคัดกรอง** - เรารีวิวข้อเสนอใหม่ภายใน 48 ชั่วโมง
2. **การประเมิน** - เรายืนยันความเป็นไปได้ กำหนดการแมป ATLAS และ ID ภัยคุกคาม และตรวจสอบระดับความเสี่ยง
3. **การจัดทำเอกสาร** - เราตรวจให้แน่ใจว่าทุกอย่างมีรูปแบบถูกต้องและครบถ้วน
4. **การผสาน** - เพิ่มลงในแบบจำลองภัยคุกคามและการแสดงผลด้วยภาพ
