# 组织与命名空间申诉

ClawHub 将所有者名称、组织名称、Skill 标识、插件包名称和包作用域用作公共命名空间。如果某个命名空间看起来属于现实中的项目、品牌、软件包生态系统或组织，但在 ClawHub 上已被占用、保留，或存在误导或争议，请使用
[组织/命名空间申诉问题表单](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
请求工作人员审核。

此流程适用于公开且不涉及敏感信息的所有权审核。请勿使用产品内举报功能或账号申诉表单提出命名空间申诉。

## 何时提出申诉

如果你认为 ClawHub 工作人员应审核某个命名空间是否因现实中的所有权而需要被保留、转让、重命名、隐藏、隔离、添加别名或以其他方式变更，请提出命名空间申诉。

示例包括：

- 与你的 GitHub 组织、项目、公司或社区名称相同的组织名称
- 类似 `@example-org/*` 的包作用域，其内容应只能由对应的 ClawHub 所有者发布
- 看起来冒充某个项目的 Skill 标识或插件包名称
- 与品牌、商标、项目重命名或软件包历史记录有关的争议
- 已删除、长期不活跃或无法联系的所有者阻碍合法的命名空间所有者使用该名称

如果相关条目除了所有权争议外还存在不安全、恶意或误导性问题，请同时遵循相关的内容审核或安全指南。命名空间申诉表单用于所有权审核，而非紧急漏洞披露。

## 提交前须知

首先确认你正在使用与该命名空间匹配的所有者身份发布内容。对于插件包，类似 `@example-org/example-plugin` 的限定作用域名称必须由对应的 `example-org` 所有者发布。

如果你能够管理当前所有者，请直接通过发布、重命名、转让、隐藏或删除受影响的资源来修正命名空间。如果你无法管理当前所有者，或需要工作人员解决争议，请提交申诉。

## 应提供的证据

请使用公开且不涉及敏感信息的证据。有效的证明包括：

- GitHub 组织、仓库、发布版本或维护者历史记录
- 明确提及该命名空间的官方项目文档
- 域名或官方邮箱域名的所有权证明
- 对 npm、PyPI、crates.io 或其他软件包注册表作用域的控制权
- 可以公开讨论的商标、品牌或项目所有权证据
- 源代码仓库历史记录、软件包历史记录或公开的重命名公告
- 指向存在争议的 ClawHub 所有者、Skill、插件、软件包或议题的链接

请说明每个链接能够证明什么。工作人员应当无需访问私人凭据或秘密信息，即可理解其中的关联。

## 不应提供的内容

请勿在公开的 GitHub 议题中提供秘密信息或非公开证明。请勿包含：

- API 令牌、签名密钥或凭据
- DNS 质询令牌
- 私人法律文件或合同
- 个人身份证明文件
- 私人电子邮件、非公开安全报告或客户机密数据

申诉表单会询问敏感证据是否需要通过工作人员的私密渠道提交。请使用该选项，不要公开发布敏感材料。

## 可能的处理结果

根据证据和风险，ClawHub 工作人员可能会保留命名空间、转让所有权、重命名资源、隐藏或隔离现有条目、添加别名或重定向、要求提供更多证据，或者拒绝该请求。

命名空间审核并不保证所有同名内容都会被转让。工作人员会综合衡量公开证据、现有使用情况、安全风险和用户影响。
