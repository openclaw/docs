# 组织和命名空间认领

ClawHub 将所有者用户名、组织用户名、技能短名称、插件包名和包作用域用作公共命名空间。如果某个命名空间看似属于现实中的项目、品牌、包生态系统或组织，但已在 ClawHub 上被认领、保留，或存在误导性或争议，请通过
[组织/命名空间认领问题表单](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
请求工作人员审核。

此渠道用于公开且不涉及敏感信息的所有权审核。请勿使用产品内举报功能或账号申诉表单提出命名空间认领。

## 何时提出认领

如果你认为 ClawHub 工作人员应审核某个命名空间是否应因现实所有权而被保留、转让、重命名、隐藏、隔离、设置别名或以其他方式更改，请提出命名空间认领。

示例包括：

- 与你的 GitHub 组织、项目、公司或社区匹配的组织用户名
- `@example-org/*` 之类应仅由匹配的 ClawHub 所有者发布的包作用域
- 看似冒充某个项目的技能短名称或插件包名
- 品牌、商标、项目重命名或包历史记录争议
- 已删除、不活跃或无法联系，且阻碍命名空间合法所有者的现有所有者

如果相关条目不仅存在所有权争议，还具有不安全、恶意或误导性，请同时遵循相关的审核或安全指南。命名空间认领表单用于所有权审核，而非紧急漏洞披露。

## 提交前

首先确认你使用的是与命名空间匹配的所有者身份。对于插件包，`@example-org/example-plugin` 之类的作用域名称必须以匹配的 `example-org` 所有者身份发布。

如果你能够管理当前所有者，请通过发布、重命名、转让、隐藏或删除受影响的资源，直接修正命名空间。如果你无法管理当前所有者，或需要工作人员解决争议，请提出认领。

## 应提供的证据

请使用公开且不涉及敏感信息的证据。有效的证明包括：

- GitHub 组织、仓库、发布版本或维护者历史记录
- 提及该命名空间的官方项目文档
- 域名或官方电子邮件域名证明
- 对 npm、PyPI、crates.io 或其他包注册表作用域的控制证明
- 可公开讨论的商标、品牌或项目所有权证据
- 源代码仓库历史记录、包历史记录或公开的重命名通知
- 指向有争议的 ClawHub 所有者、技能、插件、包或问题的链接

请说明每个链接能够证明什么。工作人员应能在不需要私密凭据或密钥的情况下理解其中的关系。

## 不应提供的内容

请勿在公开的 GitHub 问题中提供密钥或非公开证据。请勿包含：

- API 令牌、签名密钥或凭据
- DNS 验证令牌
- 非公开法律文件或合同
- 个人身份证明文件
- 私人电子邮件、非公开安全报告或机密客户数据

认领表单会询问是否需要通过私密的工作人员渠道提供敏感证据。请使用该选项，而不要公开发布敏感材料。

## 可能的处理结果

根据证据和风险，ClawHub 工作人员可能会保留命名空间、转让所有权、重命名资源、隐藏或隔离现有条目、添加别名或重定向、要求提供更多证明，或拒绝请求。

命名空间审核并不保证所有匹配的名称都会被转让。工作人员会综合考虑公开证据、现有使用情况、安全风险和用户影响。
