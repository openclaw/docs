# 组织和命名空间声明

ClawHub 使用所有者账号名、组织账号名、技能短名、插件包名和包作用域作为公共命名空间。如果某个命名空间看似属于现实中的项目、品牌、软件包生态系统或组织，但已在 ClawHub 上被占用、保留、存在误导性或争议，请通过 [Org / Namespace Claim issue form](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml) 请求工作人员审核。

此途径适用于公开且不涉及敏感信息的所有权审核。不要使用产品内举报功能或账号申诉表单处理命名空间声明。

## 何时提交声明

如果你认为 ClawHub 工作人员应审核某个命名空间是否应因现实所有权而被保留、转让、重命名、隐藏、隔离、设置别名或以其他方式更改，请提交命名空间声明。

示例包括：

- 与你的 GitHub 组织、项目、公司或社区相符的组织账号名
- 仅应由对应 ClawHub 所有者发布的包作用域，例如 `@example-org/*`
- 看似冒充某个项目的技能短名或插件包名
- 涉及品牌、商标、项目更名或软件包历史记录的争议
- 已删除、不活跃或无法联系，并阻碍正当命名空间所有者的现有所有者

如果相关条目除了所有权争议外还存在不安全、恶意或误导性问题，也请遵循相关的内容审核或安全指南。命名空间声明表单用于所有权审核，而非紧急漏洞披露。

## 提交前

首先确认你正以与该命名空间相符的所有者身份发布。对于插件包，`@example-org/example-plugin` 等带作用域的名称必须由对应的 `example-org` 所有者发布。

如果你可以管理当前所有者，请通过发布、重命名、转让、隐藏或删除受影响的资源直接修正命名空间。当你无法管理当前所有者，或需要工作人员解决争议时，请提交声明。

## 应提供的证据

请使用公开且不涉及敏感信息的证据。有效的证明包括：

- GitHub 组织、仓库、发布版本或维护者历史记录
- 明确指出该命名空间的官方项目文档
- 域名或官方电子邮件域名证明
- 对 npm、PyPI、crates.io 或其他软件包注册表作用域的控制证明
- 可公开讨论的商标、品牌或项目所有权证据
- 源代码仓库历史记录、软件包历史记录或公开的更名通知
- 指向存在争议的 ClawHub 所有者、技能、插件、软件包或议题的链接

请说明每个链接能够证明什么。工作人员应当无需私密凭证或机密信息，就能了解其中的关联。

## 不应提供的内容

不要在公开的 GitHub 议题中提供机密信息或私密证据。请勿包括：

- API 令牌、签名密钥或凭证
- DNS 验证令牌
- 私密法律文件或合同
- 个人身份证明文件
- 私密电子邮件、私密安全报告或机密客户数据

声明表单会询问敏感证据是否需要通过工作人员私密渠道提供。请使用该选项，不要公开发布敏感材料。

## 可能的处理结果

根据证据和风险，ClawHub 工作人员可能会保留命名空间、转让所有权、重命名资源、隐藏或隔离现有条目、添加别名或重定向、要求提供更多证明，或者拒绝请求。

命名空间审核并不保证所有匹配的名称都会被转让。工作人员会综合考虑公开证据、现有使用情况、安全风险和用户影响。
