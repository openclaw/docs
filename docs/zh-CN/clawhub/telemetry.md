# 遥测

ClawHub 使用**最少量遥测**来计算**安装数量**（实际正在使用的内容），并支持更好的排序/筛选。
这基于 CLI `clawhub sync` 命令。

## 何时收集遥测

仅在以下情况下发送遥测：

- 你已在 CLI 中**登录**（同步/发布流程本来就需要认证）。
- 你运行 `clawhub sync`。
- 遥测**未被禁用**（见下方“如何禁用”）。

如果你未登录，则不会上报任何内容。

## 我们收集什么

每次运行 `clawhub sync` 时，CLI 都会上报它找到内容的**完整快照**，并按扫描根目录（“文件夹/根目录”）分组。

对于每个根目录，我们会存储：

- `rootId`：规范根路径的 **SHA-256 哈希**（服务器永远不会看到原始路径）。
- `label`：从最后两个路径段派生的人类可读标签（主目录路径会显示为 `~`）。
- `firstSeenAt`、`lastSeenAt`、可选的 `expiredAt`。

对于在根目录下找到的每个技能，我们会存储：

- `skillId`（通过 slug 解析；仅跟踪注册表中存在的技能）。
- `firstSeenAt`、`lastSeenAt`。
- `lastVersion`（尽力而为；目前如果已知，则为与注册表匹配的版本）。
- 当先前上报过的安装从某个根目录消失时，可选的 `removedAt`。

### 我们_不_收集什么

- 不收集原始绝对文件夹路径（仅收集哈希后的 `rootId` + 简短显示标签）。
- 不收集文件内容。
- 不收集每次运行的日志、提示词或其他 CLI 输出。
- 不跟踪未上传到注册表的技能（未知 slug 会被忽略）。

## 安装数量

我们为每个技能维护两个计数器：

- `installsCurrent`：当前在至少一个活跃根目录中安装了该技能的唯一用户数。
- `installsAllTime`：曾经上报安装过该技能的唯一用户数。

### 多个根目录

如果你从多个文件夹同步，我们会独立处理每个扫描根目录。只要某个技能存在于**任意**活跃根目录中，就视为“当前已安装”。

### 卸载检测

因为 `sync` 会按根目录上报完整集合：

- 如果某个技能在下一次同步时从某个根目录消失，我们会将其标记为已从该根目录移除。
- 如果该技能已从你的所有根目录中移除，它将不再计入 `installsCurrent`。
- 除非你删除遥测（见下方），否则 `installsAllTime` 永远不会减少。

### 过期（120 天）

如果某个根目录连续 **120 天**未上报遥测，它会被标记为过期，其安装将不再计入 `installsCurrent`。
为避免后台任务，这会以惰性方式评估（在下一次遥测上报时）。

## 透明度 + 用户控制

ClawHub 会在你的个人资料中提供私有的“已安装”标签页：

- 显示我们存储的确切根目录 + 已安装技能。
- 包含 **JSON 导出**视图。
- 包含**删除遥测**操作，用于移除你账户下存储的所有遥测。

其他所有人只能看到**聚合后的安装计数器**；其他人无法看到你的根目录/文件夹。

删除你的账户也会删除你的遥测数据。

## 如何禁用遥测

设置环境变量：

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

设置后，CLI 在运行 `clawhub sync` 时不会发送遥测。
