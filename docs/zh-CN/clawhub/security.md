# 安全 + 审核

ClawHub 对发布保持开放，但公开列表仍会经过信任、扫描、举报和审核控制。目标很务实：帮助用户检查他们安装的内容，为发布者提供误报恢复路径，并阻止滥用包进入公开发现范围。

另请参阅[可接受使用](/zh-CN/clawhub/acceptable-usage)。

## 用户可以检查的内容

安装技能或插件前，请检查其 ClawHub 列表中的：

- 所有者和来源归属
- 最新版本和变更日志
- 必需的环境变量或权限
- 插件的兼容性元数据
- 扫描或审核状态
- 显示出来的举报、评论、星标、下载量和安装信号

只安装你理解并信任的内容。

## 扫描状态

ClawHub 可能会在公开页面和所有者可见的诊断中显示扫描或审核结果。

常见结果包括：

- `clean`：未发现阻塞问题。
- `suspicious`：该发布需要谨慎处理或复核。
- `malicious`：该发布被认为不安全。
- `pending`：检查尚未完成。
- `held`、`quarantined`、`revoked` 或 `hidden`：该发布未在公开安装界面上完全可用。

不同界面的具体措辞可能有所不同，但实际含义相同：如果某个发布被暂缓或阻止，用户不应安装它，直到所有者解决问题或审核恢复该发布。

## Skills

技能扫描会检查已发布的技能包、元数据、声明的要求以及可疑指令。

ClawHub 特别关注技能声明内容与其看起来实际执行内容之间的不匹配。例如，引用必需 API key 的技能应在 `SKILL.md` 中声明该要求，以便用户在安装前看到。

扫描发现基于构件。预期的提供商行为，例如已声明的 API 凭据、localhost OAuth 回调、有范围的卸载清理、Basic Auth 编码，或用户选择上传到所述提供商的文件，会与隐藏凭据转发、广泛的私有文件访问、无关网络目标或隐蔽浏览器滥用区别对待。

请参阅[技能格式](/zh-CN/clawhub/skill-format)。

## 插件

插件发布包含包元数据、来源归属、兼容性字段和构件完整性信息。

OpenClaw 会在安装 ClawHub 托管的插件前检查兼容性。包记录也可能公开摘要元数据，以便 OpenClaw 验证下载的构件。ClawScan 在审核插件发布时会包含已声明的包 `openclaw.environment` 环境/配置元数据，以便将已声明的运行时要求与观察到的行为进行比较。

## 举报

已登录用户可以举报技能、包和评论。

举报应具体且可执行。滥用举报本身也可能导致账号处理。

举报示例：

- 误导性元数据
- 未声明的凭据或权限要求
- 可疑的安装指令
- 诈骗评论或冒充行为
- 恶意注册或商标滥用
- 违反[可接受使用](/zh-CN/clawhub/acceptable-usage)的内容

## 恶意或商标举报

ClawHub 对恶意注册、冒充和商标相关争议使用同一套举报和工作人员审核流程。这些举报需要提供足够上下文，以便工作人员识别申诉方、有争议的列表以及请求的处理措施。

请包含：

- 标准 ClawHub 技能或包 URL 以及所有者 handle
- 涉及的商标、项目、公司或产品名称
- 申诉方所有权或授权的公开证据
- 当前所有者为何无权以该名称发布
- 请求的处理措施，例如在审核期间隐藏、转移所有权、重命名或移除

不要在公开举报中放入私人秘密或敏感法律文件。需要时，请创建一个包含非敏感证据的 GitHub issue，并请求维护者提供私下交接路径。

## 申诉和重新扫描

当所有者认为某个技能或包被错误暂缓或标记时，可以请求重新扫描。平台审核员和管理员在处理举报或支持请求时，可以为任何技能或包请求重新扫描：

```bash
clawhub skill rescan <slug>
clawhub package rescan <name>
```

对于已被审核处理的内容，所有者或许可以从所有者可见的 ClawHub 界面提交申诉。申诉应解释发生了什么变化，或说明该标记为何不正确。

## 审核暂缓

当静态扫描器将上传的技能标记为恶意时，发布者会被自动置于审核暂缓状态（在用户上设置 `requiresModerationAt`）。这会隐藏该发布者的所有技能，使未来发布从隐藏状态开始，并创建一条 `user.moderation.auto` 审计日志条目。

静态可疑发现会作为文件/行证据保留给审核员，但它们本身不会隐藏内容，也不会决定公开扫描结论。新上传内容会保持在审核/待处理状态，直到 VirusTotal 和 LLM 审核完成；静态扫描只会因恶意签名而立即阻止。ClawScan LLM 审核会保留与目的相符的备注作为指导；只有当结构化审核包含实质性问题时，它们才会返回 Review/suspicious 结论。

管理员可以解除误报导致的暂缓：

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

这会清除 `requiresModerationAt` 和 `requiresModerationReason`，恢复因用户级暂缓而隐藏的技能，并写入一条 `user.moderation.lift` 审计日志条目。因其他原因被隐藏的技能，或其自身静态扫描仍为恶意的技能，会保持隐藏。

## 封禁和账号状态

违反 ClawHub 政策的账号可能会失去发布权限。严重滥用可能导致账号封禁、令牌吊销、内容隐藏或列表移除。

已删除、被封禁或被停用的账号无法使用 ClawHub API 令牌。如果 CLI 认证在账号处理后开始失败，请登录 Web UI 查看账号状态，或通过预期的项目支持渠道联系维护者。

## 发布者指南

为了减少误报并提升用户信任：

- 保持名称、摘要、标签和变更日志准确
- 声明必需的环境变量和权限
- 避免混淆安装命令
- 尽可能链接到源代码
- 发布插件前使用 dry run
- 如果用户或审核员询问包行为，请清晰回应
