# 可接受使用规范

ClawHub 托管 OpenClaw 的 Skills、插件、软件包和市场元数据。
使用本页面判断内容或发布行为是否适合 ClawHub。

这些规则适用于条目的功能、要求用户运行的内容、自我描述方式，以及发布者使用 ClawHub 的发现、安装和信任机制的方式。有关审核状态和账号状态，请参阅[审核与账号安全](/clawhub/moderation)。有关版权或其他权利主张，请参阅[内容权利请求](/clawhub/content-rights)。

## 允许的内容

ClawHub 欢迎实用、易于理解且出于善意发布的内容。

| 类别 | 允许条件 |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 开发者生产力 | 条目帮助用户构建、测试、迁移、调试、记录或运维软件。 |
| UI、数据和自动化工作流 | 范围明确，清楚说明所需凭据，并且高风险操作包含审核、试运行、预览或确认流程。 |
| 防御性安全、审核与滥用审查 | 工具明确用于经授权的审查，保留证据，并清楚划定人工审批边界。 |
| 个人或团队工作流 | 工作流使用基于同意的账号、透明的设置流程和明确的权限。 |
| 持续维护的目录 | 每个条目均具有独特性和实用性、描述准确，并得到合理维护。 |

具体情境很重要。同一主题在范围有限的防御性或基于同意的场景中可能可以接受，但如果被包装为滥用工作流，则不可接受。

## 禁止的内容

ClawHub 不托管以滥用、欺骗、不安全执行或侵犯权利为主要目的的内容。

| 类别 | 不允许的内容 |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 未授权访问或绕过安全措施 | 绕过身份验证、接管账号、滥用速率限制、接管实时通话或智能体、窃取可重复使用的会话，或为未经批准的用户自动批准配对流程。 |
| 平台滥用和规避封禁 | 被封禁后使用隐蔽账号、账号养号或批量培育、虚假互动、多账号自动化、批量发帖、垃圾信息 Bot，或用于逃避检测的自动化。 |
| 欺诈、诈骗和欺骗性金融工作流 | 伪造证书或发票、欺骗性付款流程、诈骗推广、虚假社会认同、用于欺诈的合成身份工作流，或在没有明确人工审批的情况下支出或扣费的工具。 |
| 侵犯隐私的信息补充或监控 | 为发送垃圾信息抓取联系人、开盒、人肉跟踪、将潜在客户信息提取与未经请求的推广结合、秘密监控、未经同意的生物特征匹配，或使用泄露的数据或数据泄露转储。 |
| 未经同意的冒充或身份操纵 | 用于冒充或误导他人的换脸、数字孪生、克隆网红、虚假人设或其他工具。 |
| 露骨色情内容或禁用安全措施的成人内容生成 | NSFW 图像、视频或内容生成；对第三方 API 进行封装的成人内容工具；或主要用途为露骨色情内容的条目。 |
| 隐藏、不安全或具有误导性的执行要求 | 经过混淆的安装命令；将下载内容通过管道交给 `sh` 或 `bash` 运行等无法清晰审查的管道式 Shell 安装程序；未声明的密钥或私钥要求；无法清晰审查的远程 `npx @latest` 执行；或隐瞒条目实际运行需求的元数据。 |
| 侵犯版权或其他权利的材料 | 未经许可重新发布他人的 Skill、插件、文档、品牌资产或专有代码；违反许可条款；或冒充原作者或发布者。 |

## 禁止的市场行为

ClawHub 还会审查发布者使用市场的方式。不得使用 ClawHub 操纵发现机制、指标、信任信号、审核系统或用户注意力。

禁止的市场行为包括：

- 批量发布大量低投入、重复、占位或机器生成且看似没有实际用户价值的条目
- 使用近乎相同的 Skills 或插件淹没搜索或分类页面
- 发布数百个几乎没有使用量、维护、来源透明度或实质差异的条目
- 通过自动化、自安装循环、虚假账号、协同行为、付费互动或其他非自然行为，人为提高安装量、下载量、Star 数或其他互动指标
- 创建或轮换账号，以规避审核、封禁、发布者限制或市场审查
- 在所有权、来源、功能、安全状况、安装要求或与其他项目或发布者的关联关系方面误导用户
- 在未解决根本问题的情况下，反复上传已被隐藏、移除或屏蔽的内容

大规模发布本身并不自动构成滥用。如果条目之间存在实质差异、描述准确、得到维护且由真实用户使用，则大型目录可以接受。当发布量与内容单薄、重复、误导、缺乏维护或人为推广的条目相结合时，大型目录就会成为信任和安全问题。

## 内容权利

如果你认为 ClawHub 上的内容侵犯了你的版权或其他权利，请使用[内容权利请求](/clawhub/content-rights)。除非条目同时存在不安全、恶意或误导问题，否则请勿使用常规市场举报来提交版权或权利主张。

## 审查与执行

ClawHub 可能会使用自动检查、滥用行为统计信号、用户举报和工作人员审查来识别不安全内容或滥用性发布行为。单个信号本身不能证明存在滥用；它可以帮助 ClawHub 决定哪些内容需要审查。

我们可能会：

- 隐藏、暂扣、移除、软删除违规条目，或在资源类型支持的情况下将其彻底删除
- 阻止下载或安装不安全的版本
- 撤销 API 令牌
- 软删除相关内容
- 限制发布权限
- 封禁屡次违规者或严重违规者

对于明显的滥用行为，我们不保证会先发出警告再执行处罚。有关举报、审核暂扣、隐藏条目、封禁和账号状态，请参阅[审核与账号安全](/clawhub/moderation)。
