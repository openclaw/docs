# 可接受使用政策

ClawHub 托管 OpenClaw 的 Skills、插件、软件包和市场元数据。
请使用本页面判断内容或发布行为是否适合放在
ClawHub 上。

这些规则适用于条目执行的操作、要求用户运行的内容、条目如何
描述自身，以及发布者如何使用 ClawHub 的设备发现、安装和
信任机制。有关审核状态和账号状态，请参阅
[审核与账号安全](/clawhub/moderation)。有关版权或其他权利
主张，请参阅[内容权利请求](/clawhub/content-rights)。

## 允许的内容

ClawHub 欢迎实用、易于理解且本着诚信发布的内容。

| 类别                                         | 允许条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 开发者生产力                           | 条目帮助用户构建、测试、迁移、调试、记录或运维软件。                                               |
| UI、数据和自动化工作流               | 范围明确、清楚说明所需凭据，并且风险操作包含审核、试运行、预览或确认流程。 |
| 防御性安全、审核和滥用审查 | 工具明确用于授权审查，能够保留证据，并清晰划定人工审批边界。                          |
| 个人或团队工作流                       | 工作流使用基于同意的账号、透明的设置流程和明确的权限。                                            |
| 维护中的目录                              | 每个条目都各具特色、实用、描述准确且得到合理维护。                                                |

具体情境非常重要。同一主题在范围有限的防御性或
基于同意的场景中可能可以接受，但如果将其封装成滥用工作流，则不可接受。

## 禁止的内容

ClawHub 不托管以滥用、欺骗、不安全
执行或侵犯权利为主要目的的内容。

| 平台滥用和规避封禁                              | 被封禁后创建隐蔽账号、养号或批量培育账号、虚假互动、多账号自动化、批量发帖、垃圾信息机器人，或为逃避检测而构建的自动化。                                                                                                                                          |
| 欺诈、骗局和欺骗性金融工作流             | 伪造证书或发票、欺骗性付款流程、诈骗推广、虚假社会认同、用于欺诈的合成身份工作流，或缺乏明确人工审批的消费或收费工具。                                                                                                                    |
| 侵犯隐私的信息扩充或监控                 | 为发送垃圾信息而抓取联系人、曝光他人隐私、跟踪骚扰、将潜在客户信息提取与主动骚扰式联系相结合、秘密监控、未经同意的生物特征匹配，或使用泄露数据或数据泄露转储。                                                                                                                  |

## 禁止的市场行为

ClawHub 还会审核发布者如何使用市场。不得利用 ClawHub
操纵设备发现、指标、信任信号、审核系统或用户
注意力。

禁止的市场行为包括：

- 批量发布大量粗制滥造、重复、占位或
  机器生成且看起来不具备实际用户价值的条目
- 使用几乎相同的 Skills 或插件充斥搜索或分类界面
- 发布数百个几乎没有使用量、缺乏维护、来源
  不明确或没有实质差异的条目
- 通过自动化、自安装循环、虚假账号、协同行为、
  付费互动或其他非自然行为，人为夸大安装量、下载量、星标数或其他互动
  指标
- 创建或轮换账号以规避审核、封禁、发布者限制或
  市场审查
- 在所有权、来源、能力、安全状况、
  安装要求或与其他项目或发布者的关联方面误导用户
- 在未解决根本问题的情况下，反复上传已被隐藏、移除或屏蔽的
  内容

高频发布并不自动构成滥用。如果条目之间存在实质差异、描述准确、得到维护，
并且有真实用户使用，那么大型目录是可以接受的。当发布量与内容单薄、重复、误导、
缺乏维护或人为推广的条目相结合时，
大型目录就会成为信任与安全问题。

## 内容权利

如果你认为 ClawHub 上的内容侵犯了你的版权或其他权利，请使用
[内容权利请求](/clawhub/content-rights)。请勿使用普通市场
举报来提交版权或权利主张，除非该条目还存在不安全、
恶意或误导性问题。

## 审查与执行

ClawHub 可能会使用自动检查、统计性滥用信号、用户举报和
工作人员审查来识别不安全内容或滥用性发布行为。单个信号
本身并不能证明存在滥用；它可帮助 ClawHub 判断哪些内容需要审查。

我们可能会：

- 隐藏、暂扣、移除、软删除违规条目，或在资源类型支持的情况下
  将其彻底删除
- 阻止下载或安装不安全的版本
- 撤销 API 令牌
- 软删除相关内容
- 限制发布权限
- 封禁屡次违规或严重违规者

对于明显的滥用行为，我们不保证会先发出警告再执行处罚。有关举报、审核暂扣、
隐藏条目、封禁和账号状态，请参阅
[审核与账号安全](/clawhub/moderation)。
