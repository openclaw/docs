# 可接受使用

ClawHub 托管 OpenClaw 的 Skills、插件、包和市场元数据。
使用此页面判断内容或发布行为是否适合放在
ClawHub 上。

这些规则适用于列表的功能、它要求用户运行的内容、它如何
呈现自身，以及发布者如何使用 ClawHub 的发现、安装和
信任界面。有关审核状态和账号状态，请参阅
[审核和账号安全](/clawhub/moderation)。有关版权或其他权利
主张，请参阅 [内容权利请求](/clawhub/content-rights)。

## 允许的内容

ClawHub 欢迎有用、易懂且以善意发布的内容。

| 类别                                         | 允许条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 开发者生产力                           | 列表帮助用户构建、测试、迁移、调试、编写文档或运维软件。                                               |
| UI、数据和自动化工作流               | 范围清晰，所需凭证明确，并且高风险操作包含审查、试运行、预览或确认路径。 |
| 防御性安全、审核和滥用审查 | 工具定位为授权审查，保留证据，并清晰界定人工审批边界。                          |
| 个人或团队工作流                       | 工作流使用基于同意的账号、透明的设置和明确的权限。                                            |
| 维护中的目录                              | 每个列表都各不相同、有用、描述准确，并得到合理维护。                                                |

上下文很重要。同一主题在狭义的防御性或
基于同意的场景中可能可以接受，但作为滥用工作流打包时则不可接受。

## 禁止的内容

ClawHub 不托管主要目的为滥用、欺骗、不安全
执行或侵犯权利的内容。

| 类别                                                    | 不允许                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 未授权访问或安全绕过                      | 凭证绕过、账号接管、滥用速率限制、实时通话或智能体接管、可复用会话窃取，或为未获批准的用户自动批准配对流程。                                                                                                                                                   |
| 平台滥用和封禁规避                              | 封禁后的隐身账号、账号预热或养号、虚假互动、多账号自动化、批量发布、垃圾消息机器人，或为规避检测而构建的自动化。                                                                                                                                          |
| 欺诈、骗局和欺骗性金融工作流             | 伪造证书或发票、欺骗性支付流程、骗局外联、虚假社会证明、用于欺诈的合成身份工作流，或没有明确人工审批的支出/扣费工具。                                                                                                                    |
| 侵犯隐私的信息增强或监视                 | 为垃圾消息抓取联系人、开盒、跟踪、与未经请求的外联搭配的潜在客户提取、隐蔽监控、未经同意的生物识别匹配，或使用泄露数据或数据泄露转储。                                                                                                                  |
| 未经同意的冒充或身份操控       | 换脸、数字孪生、克隆网红、虚假人格，或其他用于冒充或误导的工具。                                                                                                                                                                                                 |
| 明示性性内容或禁用安全限制的成人内容生成 | NSFW 图像、视频或内容生成；围绕第三方 API 的成人内容包装器；或主要目的为明示性性内容的列表。                                                                                                                                                       |
| 隐藏的、不安全的或误导性的执行要求        | 混淆的安装命令、pipe-to-shell 安装器，例如在没有清晰可审查性的情况下使用 `sh` 或 `bash` 运行下载内容、未声明的密钥或私钥要求、没有清晰可审查性的远程 `npx @latest` 执行，或隐藏列表真正运行需求的元数据。 |
| 侵犯版权或违反权利的材料           | 未经许可重新发布他人的 skill、插件、文档、品牌资产或专有代码；违反许可条款；或冒充原作者或发布者。                                                                                                                            |

## 禁止的市场行为

ClawHub 也会审查发布者如何使用市场。不要使用 ClawHub 来
操纵发现、指标、信任信号、审核系统或用户
注意力。

禁止的市场行为包括：

- 批量发布大量低投入、重复、占位或
  机器生成的列表，且这些列表看起来没有真实用户价值
- 用近乎相同的 Skills 或插件淹没搜索或分类界面
- 发布数百个列表，却几乎没有使用量、维护、来源
  清晰度或有意义的差异化
- 通过自动化、自安装循环、虚假账号、协同行动、
  付费互动或其他非自然行为人为抬高安装量、下载量、星标或其他互动
  指标
- 创建或轮换账号以规避审核、封禁、发布者限制或
  市场审查
- 在所有权、来源、能力、安全状况、
  安装要求或与其他项目或发布者的关联方面误导用户
- 在没有修复根本问题的情况下，反复上传已经被隐藏、移除或阻止的
  内容

高频发布并不自动构成滥用。当列表具有有意义的差异、描述准确、得到维护，
并且由真实用户使用时，大型目录是可以接受的。
当数量与薄弱、重复、误导、无人维护或
人为推广的列表相伴时，大型目录就会成为信任和安全问题。

## 内容权利

如果你认为 ClawHub 上的内容侵犯了你的版权或其他权利，请使用
[内容权利请求](/clawhub/content-rights)。不要使用普通市场
报告处理版权或权利主张，除非该列表同时不安全、
恶意或具有误导性。

## 审查和执行

ClawHub 可以使用自动检查、统计滥用信号、用户报告和
工作人员审查来识别不安全内容或滥用性发布行为。单个信号
本身不能证明滥用；它帮助 ClawHub 判断哪些内容需要审查。

我们可能会：

- 隐藏、搁置、移除、软删除，或在资源类型支持的情况下
  硬删除违规列表
- 阻止不安全版本的下载或安装
- 撤销 API 令牌
- 软删除关联内容
- 限制发布访问权限
- 封禁重复或严重违规者

对于明显滥用，我们不保证先警告再执行。有关报告、审核搁置、
隐藏列表、封禁和账号状态，请参阅
[审核和账号安全](/clawhub/moderation)。
