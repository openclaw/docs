# 可接受使用

本页说明 ClawHub 允许的技能和内容类型，以及它不会托管的滥用工作流。

这些规则有意保持实用。我们最关注的是端到端滥用工作流，而不仅是孤立的关键词。如果某个技能被构建用于规避防御、滥用平台、诈骗他人、侵犯隐私，或促成非自愿行为，它就不适合出现在 ClawHub 上。

## 我们明确允许的近期模式

- 使用真实组件、语义化 token、无障碍状态和经过测试的用户流程的前端和设计系统工作。
- 使用已安装的源码组件、项目别名和文档化变体，而不是一次性标记的 shadcn/ui 组合。
- 保留注释、使用具体 UI5 类型，并让生成的控件接口可审查的 UI5 JavaScript 到 TypeScript 转换。
- 防御性安全审查、审核工具，以及展示证据并保持人工审批边界清晰的滥用检测提示。
- 面向个人或团队账户、基于同意的工作流自动化，包含明确凭证、透明设置，以及 dry-run 或预览模式。
- 作用域限定在其所支持软件内的文档、迁移运行手册、开发者实用工具和测试 fixture。

## 不允许

- 安全绕过或未授权访问工作流。
  - 示例：身份验证绕过、账户接管、CAPTCHA 绕过、Cloudflare 或反机器人规避、速率限制绕过、旨在击败保护机制的隐蔽抓取、实时通话或智能体接管、可复用会话窃取、为未获批准用户自动批准配对流程。

- 平台滥用和封禁规避。
  - 示例：被封禁后的隐蔽账户、账户养号/批量养号、虚假互动、karma 或粉丝培养、多账户自动化、批量发帖、垃圾消息机器人、旨在避免检测的市场或社交自动化。

- 欺诈、诈骗和欺骗性金融工作流。
  - 示例：假证书、假发票、欺骗性支付流程、诈骗外联、虚假社会证明、在没有清晰人工审批和透明控制的情况下促成支出或收费的工具，或为欺诈创建账户而构建的合成身份工作流。

- 侵犯隐私的抓取、信息增强或监控。
  - 示例：为垃圾消息大规模抓取联系方式、开盒、人身跟踪、与未经请求外联配套的潜在客户提取、隐蔽监控、未经明确同意使用的人脸搜索或生物识别匹配，或购买、发布、下载泄露数据或数据泄露转储，或将其操作化。

- 非自愿冒充或欺骗性身份操纵。
  - 示例：换脸、数字孪生、虚假人设、克隆网红，或其他用于冒充或误导的身份操纵工具。

- 明示性性内容和禁用安全机制的成人内容生成。
  - 示例：NSFW 图像/视频/内容生成、围绕第三方 API 的成人内容封装器，或主要目的为明示性性内容的技能。

- 隐藏的、不安全的或误导性的执行要求。
  - 示例：混淆的安装命令、`curl | sh`、未声明的密钥要求、未声明的私钥使用、缺少清晰可审查性的远程 `npx @latest` 执行、隐藏技能真实运行需求的误导性元数据。

## 我们明确不允许的近期模式

- “在市场封禁后创建隐蔽卖家账户。”
- “修改 Telegram 配对，让未获批准的用户自动收到配对码。”
- “用不可检测的自动化培养 Reddit/Twitter 账户。”
- “为任意用途生成专业证书或发票。”
- “在禁用安全检查的情况下生成 NSFW 内容。”
- “抓取潜在客户、增强联系人信息，并大规模发起冷外联。”
- “购买、发布或下载泄露数据或数据泄露转储。”
- “使用合成身份或 CAPTCHA 解题批量创建电子邮件或社交账户。”

## 给审查者的说明

- 上下文很重要。同一主题在狭义的防御性或基于同意的场景中可以是正当的，但在被打包为滥用工作流时则不可接受。
- 当某个技能明显针对规避、欺骗或非自愿使用进行优化时，我们应倾向于采取行动。
- 反复上传这些类别的内容，是隐藏内容并封禁账户的理由。

## 执行

- 我们可能会隐藏、移除或硬删除违规技能。
- 我们可能会撤销 token、软删除关联内容，并封禁重复违规者或严重违规者。
- 对明显滥用行为，我们不保证先警告再执行。
