# 可接受使用

本页说明 ClawHub 可以接受的技能和内容类型，以及它不会托管的滥用工作流。

这些规则有意保持务实。我们最关注端到端滥用工作流，而不只是孤立的关键词。如果某项技能被构建用来规避防护、滥用平台、诈骗他人、侵犯隐私，或促成未经同意的行为，它就不适合出现在 ClawHub 上。

## 我们明确允许的近期模式

- 使用真实组件、语义化 token、无障碍状态和经过测试的用户流程的前端和设计系统工作。
- 使用已安装源组件、项目别名和有文档说明的变体，而不是一次性标记的 shadcn/ui 组合。
- 保留注释、使用具体 UI5 类型，并使生成的控件接口便于评审的 UI5 JavaScript-to-TypeScript 转换。
- 防御性安全评审、审核工具，以及展示证据并保持人工批准边界清晰的滥用检测提示词。
- 面向个人或团队账户、基于同意的工作流自动化，具备明确凭证、透明设置，以及试运行或预览模式。
- 限定于其所支持软件范围内的文档、迁移运行手册、开发者实用工具和测试夹具。

## 不允许

- 安全绕过或未经授权访问的工作流。
  - 示例：认证绕过、账户接管、CAPTCHA 绕过、Cloudflare 或反机器人规避、速率限制绕过、旨在击败防护的隐蔽抓取、实时通话或智能体接管、可复用会话窃取、为未批准用户自动批准配对流程。

- 平台滥用和封禁规避。
  - 示例：封禁后的隐蔽账户、账户预热/养号、虚假互动、karma 或粉丝培养、多账户自动化、批量发帖、垃圾消息机器人、为规避检测而构建的市场或社交自动化。

- 欺诈、骗局和欺骗性金融工作流。
  - 示例：伪造证书、伪造发票、欺骗性付款流程、诈骗式外联、虚假社会认同、在没有清晰人工批准和透明控制的情况下促成支出或收费的工具，或为欺诈创建账户而构建的合成身份工作流。

- 侵犯隐私的抓取、扩充或监控。
  - 示例：为发送垃圾消息而大规模抓取联系方式、开盒、人身跟踪、与未经请求的外联搭配的线索提取、隐蔽监控、未经明确同意使用的人脸搜索或生物特征匹配，或购买、发布、下载泄露数据或数据泄露转储，或将其操作化。

- 未经同意的冒充或欺骗性身份操纵。
  - 示例：换脸、数字孪生、虚假人设、克隆网红，或其他用于冒充或误导的身份操纵工具。

- 露骨性内容和禁用安全措施的成人内容生成。
  - 示例：NSFW 图片/视频/内容生成、围绕第三方 API 的成人内容包装器，或主要用途是露骨性内容的技能。

- 隐藏、不安全或误导性的执行要求。
  - 示例：混淆的安装命令、`curl | sh`、未声明的密钥要求、未声明的私钥使用、缺少清晰可评审性的远程 `npx @latest` 执行、隐藏技能真实运行需求的误导性元数据。

## 我们明确不允许的近期模式

- “在市场封禁后创建隐蔽卖家账户。”
- “修改 Telegram 配对，让未批准用户自动收到配对码。”
- “用不可检测的自动化培养 Reddit/Twitter 账户。”
- “为任意用途生成专业证书或发票。”
- “在禁用安全检查的情况下生成 NSFW 内容。”
- “抓取线索、扩充联系人，并大规模发起冷外联。”
- “购买、发布或下载泄露数据或数据泄露转储。”
- “使用合成身份或 CAPTCHA 解决方案批量创建邮箱或社交账户。”

## 给审核者的说明

- 上下文很重要。同一主题在狭义的防御性或基于同意的场景中可能是合法的，但当它被包装成滥用工作流时则不可接受。
- 当一项技能明显针对规避、欺骗或未经同意的使用进行了优化时，我们应倾向于采取行动。
- 反复上传这些类别的内容，可作为隐藏内容并封禁账户的依据。

## 执行

- 我们可能会隐藏、移除或硬删除违规技能。
- 我们可能会撤销 token、软删除关联内容，并封禁重复违规或严重违规者。
- 对于明显滥用，我们不保证会先警告再执行。
