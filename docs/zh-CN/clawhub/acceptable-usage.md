# 可接受使用

本页说明 ClawHub 接受哪些类型的技能和内容，以及它不会托管哪些滥用工作流。

这些规则刻意保持务实。我们最关心的是端到端滥用工作流，而不只是孤立的关键词。如果某个技能的构建目的在于规避防护、滥用平台、诈骗他人、侵犯隐私，或促成非自愿行为，它就不适合出现在 ClawHub 上。

## 我们明确接受的近期模式

- 使用真实组件、语义化 token、无障碍状态和经过测试的用户流程的前端与设计系统工作。
- 使用已安装的源组件、项目别名和文档化变体，而不是一次性标记的 shadcn/ui 组合。
- 保留注释、使用具体 UI5 类型，并保持生成的控件接口可审查的 UI5 JavaScript-to-TypeScript 转换。
- 防御性安全审查、审核工具，以及展示证据并明确人工审批边界的滥用检测提示词。
- 面向个人或团队账户、基于同意的工作流自动化，且使用明确凭证、透明设置，以及试运行或预览模式。
- 范围限定于所支持软件的文档、迁移运行手册、开发者工具和测试夹具。

## 不接受

- 安全绕过或未授权访问工作流。
  - 示例：认证绕过、账户接管、CAPTCHA 绕过、Cloudflare 或 anti-bot 规避、速率限制绕过、旨在击败防护的隐蔽抓取、实时通话或智能体接管、可复用会话窃取、为未获批准用户自动批准配对流程。

- 平台滥用和封禁规避。
  - 示例：封禁后的隐蔽账户、账户预热/养号、虚假互动、karma 或粉丝培养、多账户自动化、批量发帖、垃圾机器人、为规避检测而构建的市场或社交自动化。

- 欺诈、诈骗和欺骗性金融工作流。
  - 示例：假证书、假发票、欺骗性支付流程、诈骗触达、虚假社会证明、在缺少明确人工审批和透明控制的情况下促成消费或收费的工具，或用于创建欺诈账户的合成身份工作流。

- 侵犯隐私的抓取、富集或监控。
  - 示例：大规模抓取联系方式用于垃圾信息、开盒、人身跟踪、与未经请求触达配套的线索提取、隐蔽监控、未经明确同意使用的人脸搜索或生物特征匹配，或购买、发布、下载、落地使用泄露数据或数据泄露转储。

- 非自愿冒充或欺骗性身份操纵。
  - 示例：换脸、数字孪生、虚假人格、克隆网红，或其他用于冒充或误导的身份操纵工具。

- 露骨性内容和关闭安全机制的成人内容生成。
  - 示例：NSFW 图像/视频/内容生成、围绕第三方 API 的成人内容封装，或主要目的为露骨性内容的技能。

- 隐藏的、不安全的或误导性的执行要求。
  - 示例：混淆的安装命令、`curl | sh`、未声明的密钥要求、未声明的私钥使用、缺少明确可审查性的远程 `npx @latest` 执行、隐藏技能真实运行需求的误导性元数据。

## 我们明确不接受的近期模式

- “在市场封禁后创建隐蔽卖家账户。”
- “修改 Telegram 配对，使未获批准用户自动收到配对码。”
- “用不可检测的自动化培养 Reddit/Twitter 账户。”
- “为任意用途生成专业证书或发票。”
- “在关闭安全检查的情况下生成 NSFW 内容。”
- “抓取线索、富集联系人，并大规模发起冷触达。”
- “购买、发布或下载泄露数据或数据泄露转储。”
- “使用合成身份或 CAPTCHA 求解批量创建电子邮件或社交账户。”

## 给审查者的说明

- 上下文很重要。同一个主题在狭窄的防御性或基于同意的场景中可能是合法的，但在被包装为滥用工作流时则不可接受。
- 当某个技能明显针对规避、欺骗或非自愿使用进行优化时，我们应倾向于采取行动。
- 反复上传这些类别的内容，足以成为隐藏内容和封禁账户的理由。

## 执行

- 我们可能隐藏、移除或硬删除违规技能。
- 我们可能吊销 token、软删除关联内容，并封禁重复违规者或严重违规者。
- 对明显滥用行为，我们不保证先警告再执行。
