# 可接受使用

本页面描述 ClawHub 接受的技能和内容类型，以及它不会托管的滥用工作流。

这些规则刻意保持务实。我们最关注端到端的滥用工作流，而不只是孤立的关键词。如果某个技能用于规避防御、滥用平台、诈骗他人、侵犯隐私，或促成非自愿行为，它就不属于 ClawHub。

## 我们明确接受的近期模式

- 使用真实组件、语义化 token、无障碍状态和经过测试的用户流程来完成前端和设计系统工作。
- 使用已安装的源组件、项目别名和有文档说明的变体来进行 shadcn/ui 组合，而不是一次性标记。
- UI5 JavaScript 到 TypeScript 的转换，保留注释，使用具体的 UI5 类型，并让生成的控件接口保持可审查。
- 防御性安全审查、审核工具，以及能够展示证据并清晰保留人工审批边界的滥用检测提示。
- 面向个人或团队账号、基于同意的工作流自动化，使用明确的凭据、透明的设置，以及 dry-run 或预览模式。
- 文档、迁移运行手册、开发者实用工具，以及限定在其所支持软件范围内的测试夹具。

## 不接受

- 安全绕过或未授权访问工作流。
  - 示例：认证绕过、账号接管、CAPTCHA 绕过、Cloudflare 或反机器人规避、速率限制绕过、旨在击败防护的隐蔽抓取、实时通话或智能体接管、可复用会话盗取、为未经批准的用户自动批准配对流程。

- 平台滥用和封禁规避。
  - 示例：封禁后的隐蔽账号、账号预热/养号、虚假互动、karma 或粉丝培养、多账号自动化、批量发帖、垃圾信息机器人、为避免检测而构建的市场平台或社交自动化。

- 欺诈、诈骗和欺骗性金融工作流。
  - 示例：虚假证书、虚假发票、欺骗性支付流程、诈骗外联、虚假社会证明、在缺少明确人工审批和透明控制的情况下促成支出或收费的工具，或为欺诈创建账号而构建的合成身份工作流。

- 侵犯隐私的抓取、丰富或监控。
  - 示例：大规模抓取联系信息用于垃圾信息、开盒、人身跟踪、将线索提取与未经请求的外联结合、隐蔽监控、在缺少明确同意的情况下使用人脸搜索或生物识别匹配，或购买、发布、下载泄露数据或数据泄露转储，或将其投入运营。

- 非自愿冒充或欺骗性身份操纵。
  - 示例：换脸、数字孪生、虚假人设、克隆网红，或其他用于冒充或误导的身份操纵工具。

- 显式性内容和禁用安全保护的成人内容生成。
  - 示例：NSFW 图像/视频/内容生成、围绕第三方 API 的成人内容封装，或主要目的为显式性内容的技能。

- 隐藏、不安全或误导性的执行要求。
  - 示例：混淆的安装命令、`curl | sh`、未声明的密钥要求、未声明的私钥使用、缺少清晰可审查性的远程 `npx @latest` 执行，或隐藏技能真实运行需求的误导性元数据。

## 我们明确不接受的近期模式

- “在市场平台封禁后创建隐蔽卖家账号。”
- “修改 Telegram 配对，使未经批准的用户自动收到配对码。”
- “用无法检测的自动化培养 Reddit/Twitter 账号。”
- “为任意用途生成专业证书或发票。”
- “在禁用安全检查的情况下生成 NSFW 内容。”
- “抓取线索、丰富联系人，并大规模发起冷外联。”
- “购买、发布或下载泄露数据或数据泄露转储。”
- “使用合成身份或 CAPTCHA 求解批量创建邮箱或社交账号。”

## 给审核者的说明

- 上下文很重要。同一个主题在狭窄的防御性或基于同意的场景中可能是正当的，但在被包装为滥用工作流时不可接受。
- 当某个技能明显针对规避、欺骗或非自愿使用进行优化时，我们应倾向于采取行动。
- 反复上传这些类别的内容，足以成为隐藏内容并封禁账号的理由。

## 执行

- 我们可能隐藏、移除或硬删除违规技能。
- 我们可能撤销 token、软删除关联内容，并封禁重复或严重违规者。
- 对于明显滥用，我们不保证先警告再执行。
