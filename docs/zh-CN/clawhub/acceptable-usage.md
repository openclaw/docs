# 可接受的使用方式

本页说明 ClawHub 可以接受的技能和内容类型，以及它不会托管的滥用工作流。

这些规则刻意保持实用。我们最关心的是端到端滥用工作流，而不只是孤立的关键词。如果某个技能被构建用来规避防御、滥用平台、诈骗他人、侵犯隐私，或促成非自愿行为，它就不适合出现在 ClawHub 上。

## 我们明确可以接受的近期模式

- 使用真实组件、语义化 token、可访问状态和经过测试的用户流程的前端和设计系统工作。
- 使用已安装源码组件、项目别名和已记录变体的 shadcn/ui 组合，而不是一次性标记。
- UI5 从 JavaScript 到 TypeScript 的转换，保留注释，使用具体的 UI5 类型，并让生成的控件接口保持可审查。
- 防御性安全审查、审核工具和滥用检测提示，它们展示证据并清晰保留人工审批边界。
- 面向个人或团队账号、基于同意的工作流自动化，包含明确凭证、透明设置，以及 dry-run 或预览模式。
- 文档、迁移运行手册、开发者工具和测试夹具，范围限定在它们所支持的软件内。

## 不可接受

- 安全绕过或未授权访问工作流。
  - 示例：身份验证绕过、账号接管、CAPTCHA 绕过、Cloudflare 或反机器人规避、速率限制绕过、旨在击败防护的隐蔽抓取、实时通话或智能体接管、可复用会话窃取、为未获批准用户自动批准配对流程。

- 平台滥用和封禁规避。
  - 示例：封禁后的隐蔽账号、账号预热或养号、虚假互动、karma 或粉丝培养、多账号自动化、批量发帖、垃圾信息机器人、为避免检测而构建的市场或社交自动化。

- 欺诈、诈骗和欺骗性金融工作流。
  - 示例：假证书、假发票、欺骗性支付流程、诈骗外联、虚假社会证明、在没有清晰人工审批和透明控制的情况下促成支出或扣款的工具，或为欺诈创建账号而构建的合成身份工作流。

- 侵犯隐私的抓取、信息丰富化或监视。
  - 示例：大规模抓取联系方式用于垃圾信息、开盒、人身跟踪、与未经请求外联配套的潜在客户提取、隐蔽监控、在没有明确同意的情况下使用的人脸搜索或生物特征匹配，或购买、发布、下载泄露数据或数据泄露转储，或将其投入操作使用。

- 非自愿冒充或欺骗性身份操纵。
  - 示例：换脸、数字孪生、虚假人设、克隆网红，或其他用于冒充或误导的身份操纵工具。

- 显式性内容和禁用安全措施的成人内容生成。
  - 示例：NSFW 图像、视频或内容生成，围绕第三方 API 的成人内容包装器，或主要目的为显式性内容的技能。

- 隐藏的、不安全的或误导性的执行要求。
  - 示例：混淆的安装命令、`curl | sh`、未声明的密钥要求、未声明的私钥使用、缺乏清晰可审查性的远程 `npx @latest` 执行、隐藏技能实际运行需求的误导性元数据。

## 我们明确不可接受的近期模式

- “在市场封禁后创建隐蔽卖家账号。”
- “修改 Telegram 配对，让未获批准的用户自动收到配对码。”
- “用不可检测的自动化培养 Reddit/Twitter 账号。”
- “为任意用途生成专业证书或发票。”
- “在禁用安全检查的情况下生成 NSFW 内容。”
- “抓取潜在客户、丰富联系人信息，并大规模发起冷外联。”
- “购买、发布或下载泄露数据或数据泄露转储。”
- “使用合成身份或 CAPTCHA 求解批量创建电子邮件或社交账号。”

## 给审查者的说明

- 上下文很重要。同一主题在狭义的防御性或基于同意的场景中可能是合法的，但当它被包装成滥用工作流时则不可接受。
- 当某个技能明显为规避、欺骗或非自愿使用而优化时，我们应倾向于采取行动。
- 在这些类别中重复上传，是隐藏内容和封禁账号的理由。

## 执行

- 我们可能隐藏、移除或硬删除违规技能。
- 我们可能撤销 token、软删除相关内容，并封禁重复违规者或严重违规者。
- 对于明显滥用行为，我们不保证先警告再执行。
