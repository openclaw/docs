# 可接受使用

本页说明 ClawHub 允许的技能和内容类型，以及它不会托管的滥用工作流。

这些规则刻意保持务实。我们最关注端到端的滥用工作流，而不只是孤立的关键词。如果某个技能是为了规避防护、滥用平台、诈骗他人、侵犯隐私或支持非自愿行为而构建，它就不适合出现在 ClawHub 上。

## 我们明确允许的近期模式

- 使用真实组件、语义化令牌、无障碍状态和经过测试的用户流程的前端与设计系统工作。
- 使用已安装的源组件、项目别名和文档化变体，而不是一次性标记的 shadcn/ui 组合。
- 保留注释、使用具体 UI5 类型，并让生成的控件接口可审查的 UI5 JavaScript 到 TypeScript 转换。
- 防御性安全审查、审核工具，以及能展示证据并保持清晰人工审批边界的滥用检测提示词。
- 面向个人或团队账户的基于同意的工作流自动化，具备明确凭证、透明设置，以及试运行或预览模式。
- 限定在其所支持软件范围内的文档、迁移运行手册、开发者实用工具和测试夹具。

## 不允许

- 安全绕过或未授权访问工作流。
  - 示例：身份验证绕过、账户接管、CAPTCHA 绕过、Cloudflare 或反机器人规避、速率限制绕过、旨在击败防护的隐蔽抓取、实时通话或智能体接管、可复用会话盗取、为未获批准用户自动批准配对流程。

- 平台滥用和封禁规避。
  - 示例：封禁后的隐蔽账户、账户养号/批量养号、虚假互动、karma 值或粉丝培养、多账户自动化、批量发帖、垃圾消息机器人、为规避检测而构建的市场或社交自动化。

- 欺诈、骗局和欺骗性金融工作流。
  - 示例：假证书、假发票、欺骗性支付流程、诈骗外联、虚假社会证明、在缺少明确人工审批和透明控制的情况下支持花费或扣款的工具，或为创建欺诈账户而构建的合成身份工作流。

- 侵犯隐私的抓取、扩充或监控。
  - 示例：为垃圾消息大规模抓取联系方式、开盒、人身跟踪、与未请求外联结合的潜在客户提取、隐蔽监控、未经明确同意使用的人脸搜索或生物识别匹配，或购买、发布、下载或操作化泄露数据或数据泄露转储。

- 非自愿冒充或欺骗性身份操纵。
  - 示例：换脸、数字孪生、虚假人设、克隆网红，或其他用于冒充或误导的身份操纵工具。

- 明示性内容和禁用安全机制的成人内容生成。
  - 示例：NSFW 图像/视频/内容生成、围绕第三方 API 的成人内容封装，或主要目的为明示性内容的技能。

- 隐藏、不安全或误导性的执行要求。
  - 示例：混淆的安装命令、`curl | sh`、未声明的密钥要求、未声明的私钥使用、缺少清晰可审查性的远程 `npx @latest` 执行，或隐藏技能真实运行需求的误导性元数据。

## 我们明确不允许的近期模式

- “在市场封禁后创建隐蔽卖家账户。”
- “修改 Telegram 配对，使未获批准用户自动收到配对码。”
- “用不可检测的自动化培养 Reddit/Twitter 账户。”
- “为任意用途生成专业证书或发票。”
- “在禁用安全检查的情况下生成 NSFW 内容。”
- “抓取潜在客户、扩充联系人，并大规模发起冷外联。”
- “购买、发布或下载泄露数据或数据泄露转储。”
- “使用合成身份或 CAPTCHA 解决方案批量创建电子邮件或社交账户。”

## 给审核者的说明

- 上下文很重要。同一个主题在狭窄的防御性或基于同意的场景中可能是正当的，但在被包装成滥用工作流时则不可接受。
- 当某个技能明显针对规避、欺骗或非自愿使用进行优化时，我们应倾向于采取行动。
- 反复上传这些类别的内容，构成隐藏内容和封禁账户的依据。

## 执行

- 我们可能会隐藏、移除或硬删除违规技能。
- 我们可能会吊销令牌、软删除关联内容，并封禁重复违规者或严重违规者。
- 对于明显滥用，我们不保证先警告后执行。
