# 可接受使用

ClawHub 托管 OpenClaw 的 Skills、插件、包和市场元数据。
使用本页面判断内容或发布行为是否适合放在
ClawHub 上。

这些规则适用于列表项的功能、它要求用户运行的内容、它如何
展示自身，以及发布者如何使用 ClawHub 的发现、安装和
信任界面。有关审核状态和账户信誉，请参阅
[审核与账户安全](/zh-CN/clawhub/moderation)。有关版权或其他权利
主张，请参阅 [内容权利请求](/zh-CN/clawhub/content-rights)。

## 允许的内容

ClawHub 欢迎有用、易懂且善意发布的内容。

| 类别                                         | 允许条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 开发者生产力                           | 该列表项帮助用户构建、测试、迁移、调试、记录或运维软件。                                               |
| UI、数据和自动化工作流               | 范围清晰，所需凭证明确，并且风险操作包含审查、试运行、预览或确认路径。 |
| 防御性安全、审核和滥用审查 | 该工具面向授权审查进行定位，保留证据，并明确保留人工审批边界。                          |
| 个人或团队工作流                       | 该工作流使用基于同意的账户、透明设置和明确权限。                                            |
| 维护中的目录                              | 每个列表项都彼此不同、有用、描述准确且得到合理维护。                                                |

上下文很重要。同一主题在狭义的防御性或
基于同意的场景中可能可以接受，但如果被打包为滥用工作流，则不可接受。

## 禁止的内容

ClawHub 不托管主要目的为滥用、欺骗、不安全
执行或侵犯权利的内容。

| 类别                                                    | 不允许                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 未授权访问或安全绕过                      | 凭证绕过、账户接管、速率限制滥用、实时通话或智能体接管、可复用会话窃取，或为未获批准用户自动批准配对流程。                                                                                                                                                   |
| 平台滥用和规避封禁                              | 封禁后的隐蔽账户、账户预热或养号、虚假互动、多账户自动化、批量发布、垃圾消息机器人，或为逃避检测而构建的自动化。                                                                                                                                          |
| 欺诈、骗局和欺骗性金融工作流             | 伪造证书或发票、欺骗性付款流程、诈骗外联、虚假社会证明、用于欺诈的合成身份工作流，或没有明确人工审批的消费/扣费工具。                                                                                                                    |
| 侵犯隐私的数据补充或监控                 | 为垃圾消息抓取联系人、开盒、人肉骚扰、与未请求外联配套的潜在客户提取、隐蔽监控、未经同意的生物特征匹配，或使用泄露数据或数据泄露转储。                                                                                                                  |
| 未经同意的冒充或身份操纵       | 换脸、数字孪生、克隆网红、虚假人格，或其他用于冒充或误导的工具。                                                                                                                                                                                                 |
| 明示性内容或禁用安全保护的成人内容生成 | NSFW 图像、视频或内容生成；第三方 API 的成人内容包装器；或主要目的为明示性内容的列表项。                                                                                                                                                       |
| 隐藏、不安全或误导性的执行要求        | 混淆的安装命令、管道到 shell 的安装器，例如在没有明确可审查性的情况下用 `sh` 或 `bash` 运行下载内容、未声明的密钥或私钥要求、没有明确可审查性的远程 `npx @latest` 执行，或隐藏列表项实际运行需求的元数据。 |
| 侵犯版权或违反权利的材料           | 未经许可重新发布他人的 Skill、插件、文档、品牌资产或专有代码；违反许可证条款；或冒充原作者或发布者。                                                                                                                            |

## 禁止的市场行为

ClawHub 还会审查发布者如何使用市场。不要使用 ClawHub 来
操纵发现、指标、信任信号、审核系统或用户
注意力。

禁止的市场行为包括：

- 批量发布大量低投入、重复、占位或
  机器生成的列表项，且这些列表项看起来没有真实用户价值
- 用近乎相同的 Skills 或插件淹没搜索或分类界面
- 发布数百个列表项，但几乎没有使用量、维护、来源
  清晰度或有意义的差异化
- 通过自动化、自安装循环、虚假账户、协同行动、
  付费互动或其他非自然行为，人为抬高安装量、下载量、星标或其他互动
  指标
- 创建或轮换账户以规避审核、封禁、发布者限制或
  市场审查
- 在所有权、来源、能力、安全状态、
  安装要求或与其他项目或发布者的从属关系方面误导用户
- 在未修复根本问题的情况下，反复上传已被隐藏、移除或阻止的
  内容

高频发布不一定自动构成滥用。大型目录在列表项具有有意义的差异、描述准确、得到维护
并被真实用户使用时是可以接受的。当数量与单薄、重复、误导、无人维护或
人为推广的列表项相结合时，大型目录就会成为信任与安全问题。

## 内容权利

如果你认为 ClawHub 上的内容侵犯了你的版权或其他权利，请使用
[内容权利请求](/zh-CN/clawhub/content-rights)。除非该列表项同时不安全、
恶意或误导，否则不要将普通市场报告用于版权或权利主张。

## 审查与执行

ClawHub 可能会使用自动检查、统计滥用信号、用户报告和
工作人员审查来识别不安全内容或滥用发布行为。单个信号
本身不能证明滥用；它帮助 ClawHub 判断哪些内容需要审查。

我们可能会：

- 隐藏、暂缓、移除、软删除，或在资源类型支持时
  硬删除违规列表项
- 阻止不安全版本的下载或安装
- 撤销 API 令牌
- 软删除相关内容
- 限制发布权限
- 封禁重复违规或严重违规者

对于明显滥用，我们不保证先警告再执行。有关报告、审核暂缓、
隐藏列表项、封禁和账户信誉，请参阅
[审核与账户安全](/zh-CN/clawhub/moderation)。
