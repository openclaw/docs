# 可接受使用

本页描述 ClawHub 接受的技能和内容类型，以及它不会托管的滥用工作流。

这些规则有意保持务实。我们最关注端到端的滥用工作流，而不只是孤立关键词。如果某个技能是为了规避防护、滥用平台、诈骗他人、侵犯隐私，或支持未经同意的行为而构建的，它就不属于 ClawHub。

## 近期我们明确接受的模式

- 使用真实组件、语义化 token、可访问状态和经过测试的用户流程的前端和设计系统工作。
- shadcn/ui 组合，使用已安装的源码组件、项目别名和文档化变体，而不是一次性标记。
- UI5 JavaScript 到 TypeScript 转换，保留注释，使用具体的 UI5 类型，并让生成的控件接口可审查。
- 防御性安全审查、审核工具和滥用检测提示，展示证据并清晰保留人工审批边界。
- 面向个人或团队账号、基于同意的工作流自动化，使用明确的凭据、透明的设置，以及空运行或预览模式。
- 文档、迁移运行手册、开发者工具和测试夹具，其范围限定于它们支持的软件。

## 不接受

- 安全绕过或未授权访问工作流。
  - 示例：身份验证绕过、账号接管、CAPTCHA 绕过、Cloudflare 或反机器人规避、速率限制绕过、旨在击败防护的隐蔽抓取、实时通话或智能体接管、可复用会话盗取、为未经批准的用户自动批准配对流程。

- 平台滥用和封禁规避。
  - 示例：封禁后的隐蔽账号、账号预热/养号、虚假互动、karma 或粉丝培育、多账号自动化、批量发帖、垃圾信息机器人、为避免检测而构建的市场或社交自动化。

- 欺诈、诈骗和欺骗性金融工作流。
  - 示例：假证书、假发票、欺骗性支付流程、诈骗外联、虚假社会证明、在没有明确人工批准和透明控制的情况下支持消费或扣款的工具，或为欺诈创建账号而构建的合成身份工作流。

- 侵犯隐私的抓取、丰富或监控。
  - 示例：大规模抓取联系方式用于垃圾信息、公开个人隐私信息、跟踪、与未经请求的外联配套的潜在客户提取、隐蔽监控、未经明确同意使用的人脸搜索或生物特征匹配，或购买、发布、下载或运营化泄露数据或数据泄露转储。

- 未经同意的冒充或欺骗性身份操纵。
  - 示例：换脸、数字孪生、虚假人设、克隆网红，或其他用于冒充或误导的身份操纵工具。

- 明确性内容和禁用安全机制的成人内容生成。
  - 示例：NSFW 图像/视频/内容生成、围绕第三方 API 的成人内容封装，或主要用途是明确性内容的技能。

- 隐藏、不安全或误导性的执行要求。
  - 示例：混淆的安装命令、`curl | sh`、未声明的密钥要求、未声明的私钥使用、没有清晰可审查性的远程 `npx @latest` 执行、隐藏技能真实运行需求的误导性元数据。

## 近期我们明确不接受的模式

- “在市场封禁后创建隐蔽卖家账号。”
- “修改 Telegram 配对，让未经批准的用户自动收到配对码。”
- “用不可检测的自动化培养 Reddit/Twitter 账号。”
- “生成可任意使用的专业证书或发票。”
- “在禁用安全检查的情况下生成 NSFW 内容。”
- “抓取潜在客户、丰富联系人，并大规模发起冷外联。”
- “购买、发布或下载泄露数据或数据泄露转储。”
- “使用合成身份或 CAPTCHA 求解批量创建邮箱或社交账号。”

## 给审核者的说明

- 上下文很重要。同一主题在狭义的防御性或基于同意的场景中可能是合法的，但当它被打包为滥用工作流时就不可接受。
- 当某个技能明显针对规避、欺骗或未经同意的用途进行优化时，我们应倾向于采取行动。
- 反复上传这些类别的内容，可作为隐藏内容并封禁账号的依据。

## 执行

- 我们可能隐藏、移除或硬删除违规技能。
- 我们可能吊销 token、软删除相关内容，并封禁重复或严重违规者。
- 对于明显滥用，我们不保证先警告再执行。
