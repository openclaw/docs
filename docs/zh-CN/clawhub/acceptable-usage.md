# 可接受使用

本页面说明 ClawHub 允许的技能和内容类型，以及它不会托管的滥用工作流。

这些规则刻意保持实用。我们最关注端到端的滥用工作流，而不是孤立的关键词。如果某个技能被设计用于规避防护、滥用平台、诈骗他人、侵犯隐私，或支持未经同意的行为，它就不应出现在 ClawHub 上。

## 我们明确接受的近期模式

- 使用真实组件、语义 token、无障碍状态和经过测试的用户流程进行前端和设计系统工作。
- shadcn/ui 组合使用已安装的源组件、项目别名和已记录的变体，而不是一次性标记。
- UI5 JavaScript 到 TypeScript 的转换会保留注释，使用具体的 UI5 类型，并保持生成的控件接口可审查。
- 防御性安全审查、审核工具，以及显示证据并保持人工批准边界清晰的滥用检测提示。
- 基于同意的个人或团队账号工作流自动化，具备明确凭证、透明设置，以及试运行或预览模式。
- 面向其所支持软件的文档、迁移运行手册、开发者实用工具和测试夹具。

## 不接受

- 安全绕过或未授权访问工作流。
  - 示例：认证绕过、账号接管、CAPTCHA 绕过、Cloudflare 或反机器人规避、速率限制绕过、旨在击败防护的隐蔽抓取、实时通话或智能体接管、可复用会话窃取、为未批准用户自动批准配对流程。

- 平台滥用和封禁规避。
  - 示例：封禁后的隐蔽账号、账号预热/养号、虚假互动、karma 或粉丝培养、多账号自动化、批量发帖、垃圾消息机器人、为规避检测而构建的市场平台或社交自动化。

- 欺诈、诈骗和欺骗性金融工作流。
  - 示例：虚假证书、虚假发票、欺骗性支付流程、诈骗外联、虚假社会证明、在缺少清晰人工批准和透明控制的情况下支持消费或收费的工具，或用于为欺诈创建账号的合成身份工作流。

- 侵犯隐私的抓取、信息补全或监控。
  - 示例：大规模抓取联系方式用于垃圾消息、开盒、跟踪骚扰、与未经请求外联配套的潜在客户提取、隐蔽监控、未经明确同意使用的人脸搜索或生物特征匹配，或购买、发布、下载或运营化泄露数据或泄露转储。

- 未经同意的冒充或欺骗性身份操纵。
  - 示例：换脸、数字分身、虚假人格、克隆网红，或其他用于冒充或误导的身份操纵工具。

- 显式色情内容和禁用安全限制的成人内容生成。
  - 示例：NSFW 图像/视频/内容生成、围绕第三方 API 的成人内容封装，或主要目的为显式色情内容的技能。

- 隐藏、不安全或误导性的执行要求。
  - 示例：混淆的安装命令、`curl | sh`、未声明的密钥要求、未声明的私钥使用、缺少清晰可审查性的远程 `npx @latest` 执行，或隐藏技能实际运行需求的误导性元数据。

## 我们明确不接受的近期模式

- “在市场平台封禁后创建隐蔽卖家账号。”
- “修改 Telegram 配对，让未批准用户自动接收配对代码。”
- “用不可检测的自动化培养 Reddit/Twitter 账号。”
- “为任意用途生成专业证书或发票。”
- “在禁用安全检查的情况下生成 NSFW 内容。”
- “抓取潜在客户、补全联系人信息，并大规模发起冷外联。”
- “购买、发布或下载泄露数据或泄露转储。”
- “使用合成身份或 CAPTCHA 求解批量创建电子邮件或社交账号。”

## 给审查者的备注

- 语境很重要。同一主题在狭义的防御性或基于同意的场景中可能是合法的，但当它被包装成滥用工作流时就不可接受。
- 当某个技能明显针对规避、欺骗或未经同意的使用进行优化时，我们应倾向于采取行动。
- 反复上传这些类别的内容，构成隐藏内容和封禁账号的理由。

## 执行

- 我们可能隐藏、移除或硬删除违规技能。
- 我们可能吊销 token、软删除关联内容，并封禁重复违规或严重违规者。
- 对明显滥用行为，我们不保证先警告再执行。
