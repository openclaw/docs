# 可接受使用规范

ClawHub 托管适用于 OpenClaw 的 Skills、插件、软件包和市场元数据。
请使用本页面判断内容或发布行为是否适合出现在 ClawHub 上。

这些规则适用于条目的功能、条目要求用户运行的内容、条目如何描述自身，以及发布者如何使用 ClawHub 的内容发现、安装和信任机制。有关审核状态和账号状况，请参阅[审核与账号安全](/clawhub/moderation)。有关版权或其他权利主张，请参阅[内容权利请求](/clawhub/content-rights)。

## 允许的内容

ClawHub 欢迎实用、易于理解且基于善意发布的内容。

| 类别 | 允许条件 |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 开发者生产力 | 条目帮助用户构建、测试、迁移、调试、记录或运维软件。 |
| UI、数据和自动化工作流 | 范围清晰，明确说明所需凭据，并且危险操作包含审核、试运行、预览或确认流程。 |
| 防御性安全、审核和滥用审查 | 工具定位为用于授权审查，能够保留证据，并明确维持人工审批边界。 |
| 个人或团队工作流 | 工作流使用基于同意的账号、透明的设置流程和明确的权限。 |
| 持续维护的目录 | 每个条目都各具特色、切实有用、描述准确，并得到合理维护。 |

具体情境很重要。同一主题在范围有限的防御性或基于同意的场景中可能可以接受，但如果被包装成滥用工作流，则不可接受。

## 禁止的内容

ClawHub 不托管以滥用、欺骗、不安全执行或侵犯权利为主要目的的内容。

| 类别 | 禁止内容 |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 未授权访问或绕过安全机制 | 绕过身份验证、接管账号、滥用速率限制、接管实时通话或智能体、窃取可重复使用的会话，或为未经批准的用户自动批准配对流程。 |
| 平台滥用和规避封禁 | 被封禁后使用隐蔽账号、养号或批量培育账号、制造虚假互动、多账号自动化、批量发帖、垃圾信息 Bot，或为逃避检测而构建的自动化。 |
| 欺诈、诈骗和欺骗性金融工作流 | 伪造证书或发票、欺骗性付款流程、诈骗式推广、虚假社会认同、用于欺诈的合成身份工作流，或在没有明确人工批准的情况下支出或收费的工具。 |
| 侵犯隐私的信息补充或监控 | 为发送垃圾信息抓取联系人、公开他人隐私、跟踪骚扰、将潜在客户信息提取与未经请求的推广结合、隐蔽监控、未经同意的生物特征匹配，或使用泄露的数据或数据泄露转储。 |
| 未经同意的冒充或身份操纵 | 换脸、数字孪生、克隆网红、虚假人设，或其他用于冒充或误导的工具。 |
| 露骨色情内容或禁用安全防护的成人内容生成 | NSFW 图像、视频或内容生成；封装第三方 API 的成人内容工具；或以露骨色情内容为主要目的的条目。 |
| 隐藏、不安全或具有误导性的执行要求 | 经过混淆的安装命令；通过管道传入 shell 的安装程序，例如在缺乏明确可审查性的情况下使用 `sh` 或 `bash` 运行下载内容；未声明的密钥或私钥要求；在缺乏明确可审查性的情况下远程执行 `npx @latest`；或使用元数据隐藏条目实际运行要求。 |
| 侵犯版权或其他权利的材料 | 未经许可重新发布他人的 Skill、插件、文档、品牌资产或专有代码；违反许可条款；或冒充原作者或发布者。 |

## 禁止的市场行为

ClawHub 还会审查发布者使用市场的方式。不得使用 ClawHub 操纵内容发现、指标、信任信号、审核系统或用户注意力。

禁止的市场行为包括：

- 批量发布大量低投入、重复、占位或机器生成的条目，且这些条目看起来并无实际用户价值
- 使用近乎相同的 Skills 或插件淹没搜索结果或分类页面
- 发布数百个几乎或完全没有使用量、维护、来源透明度或实质差异的条目
- 通过自动化、自我安装循环、虚假账号、协同行为、付费互动或其他非自然行为，人为夸大安装量、下载量、星标数或其他互动指标
- 创建或轮换账号，以规避审核、封禁、发布者限制或市场审查
- 在所有权、来源、能力、安全状况、安装要求或与其他项目或发布者的关联方面误导用户
- 在未解决根本问题的情况下，反复上传已被隐藏、移除或屏蔽的内容

高频发布本身并不一定构成滥用。如果各条目存在实质差异、描述准确、得到维护且有真实用户使用，则大型目录是可以接受的。当庞大的数量伴随着内容单薄、重复、误导、无人维护或人为推广的条目时，大型目录就会成为信任与安全问题。

## 内容权利

如果你认为 ClawHub 上的内容侵犯了你的版权或其他权利，请使用[内容权利请求](/clawhub/content-rights)。除非条目同时存在不安全、恶意或误导性问题，否则不要使用普通市场举报渠道提出版权或其他权利主张。

## 审查与执行措施

ClawHub 可能使用自动检查、滥用行为统计信号、用户举报和工作人员审查来识别不安全内容或滥用性发布行为。单个信号本身不能证明存在滥用；它用于帮助 ClawHub 判断哪些内容需要审查。

我们可能会：

- 隐藏、暂缓发布、移除或软删除违规条目；在相应资源类型支持的情况下，也可能永久删除
- 阻止下载或安装不安全的版本
- 撤销 API 令牌
- 软删除相关内容
- 限制发布权限
- 封禁屡次违规或严重违规者

对于明显的滥用行为，我们不保证先警告再执行措施。有关举报、审核暂缓、隐藏条目、封禁和账号状况的信息，请参阅[审核与账号安全](/clawhub/moderation)。
