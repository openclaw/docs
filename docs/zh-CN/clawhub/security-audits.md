# 安全审计

ClawHub 安全审计可帮助你判断某个技能或插件是否足够安全、可以安装。它们会展示某个发布版本会做什么、请求哪些权限，以及在访问文件、账号、凭证、代码或外部服务之前，是否有任何事项需要额外关注。

审计是很强的安全信号，但并不保证某个发布版本完全没有风险。在授予敏感访问权限之前，始终要自行判断。

另请参阅 [安全](/clawhub/security)、[可接受使用](/clawhub/acceptable-usage) 和 [审核与账号安全](/clawhub/moderation)。

## 安装前要检查什么

安装前，请检查：

- 整体审计状态
- 风险级别
- 所列出的任何发现
- 所需凭证、权限或环境变量
- 所有者、来源、版本、变更日志、下载量、星标数以及其他信任信号

只安装你理解并信任的内容。

## 审计状态

审计状态会告诉你应如何处理审计结果：

| 状态        | 含义                                                                      |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 未发现高于低风险的可见问题。                                              |
| `Review`    | 安装前请阅读发现。该发布版本可能仍然是合法的。                            |
| `Warn`      | 请格外谨慎。ClawHub 发现了高影响问题或警告信号。                          |
| `Malicious` | 不要安装。                                                                |
| `Pending`   | 审计尚未完成。                                                            |
| `Error`     | 无法完成审计。                                                            |

`Pass` 令人放心，但不能取代你自己的判断。对于能够发布内容、编辑数据、运行命令、读取文件或访问生产系统的工具，这一点最为重要。

## 风险级别

风险级别描述影响范围：如果你按预期使用该发布版本，它看起来拥有多大的权力。

| 风险级别 | 含义                                                                          |
| -------- | ----------------------------------------------------------------------------- |
| `Low`    | 未发现多少敏感权限或用户影响。                                                |
| `Medium` | 该发布版本拥有有实际意义的权限，例如账号访问或数据更改。                      |
| `High`   | 该发布版本拥有高影响权限、严重发现或恶意信号。                                |

风险级别和审计状态回答的是不同问题：

- 风险级别问的是：“这里有多大权力？”
- 审计状态问的是：“我应该如何处理这个结果？”

例如，一个发布技能可能显示为 `Review`，风险为 `Medium`。这并不意味着它是恶意的。这意味着该技能看起来与用途一致，但可以行使有实际意义的账号权限。

## 发现

发现会解释为什么显示某个审计结果。每条发现通常包括：

- 它意味着什么
- 为什么被标记
- 相关的技能或插件内容
- 建议

发现可能被标记为 `Info`、`Low`、`Medium`、`High` 或 `Critical`。严重程度越高，对风险级别和审计状态的影响越大。

低置信度发现会从公开审计汇总中隐藏，让页面专注于有用证据。

## ClawHub 检查什么

ClawHub 会审计提交的发布工件，包括：

- 技能说明或插件元数据
- 声明的环境变量和权限
- 安装说明和软件包元数据
- 包含的文件和文件清单
- 兼容性和能力元数据

核心问题是一致性：名称、摘要、元数据、请求的权限和实际内容，是否与用户的合理预期一致？

强大的行为并不自动意味着不好。许多有用工具需要凭证、本地命令、提供商 API 或软件包安装。审计会检查这些权力是否符合预期、是否已披露，以及是否相称。

工件页面会链接到完整审计：

```text
/<owner>/skills/<slug>/security-audit
```

审计页面结合了：

1. SkillSpector
2. VirusTotal
3. 风险分析

## VirusTotal

ClawHub 在审计栈中使用 VirusTotal 作为恶意软件遥测。VirusTotal 是文件声誉和恶意软件扫描方面值得信赖的行业标准，而我们的合作关系让 ClawHub 能够为技能和插件审核加入更广泛的安全情报。

VirusTotal 对已知恶意工件、引擎命中以及补充 ClawHub 面向智能体审核的声誉信号尤其有用。当供应商引擎数量可用时，审计会用通俗语言汇总它们，例如：

```text
62/62 vendors flagged this skill as clean.
```

或：

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

当 ClawHub 没有可汇总的供应商数量遥测时，审计会显示：

```text
No VirusTotal findings
```

VirusTotal 仍然是遥测。它不能取代 ClawHub 自己的工件感知风险分析。

## 风险分析

风险分析由 ClawHub 自己的安全审计系统 ClawScan 在内部提供支持。它会将每个发布版本作为面向智能体的工件来审核：说明、元数据、声明的权限、文件、能力信号、静态扫描信号、SkillSpector 发现、VirusTotal 遥测以及发布者提供的上下文。静态扫描信号是此审核的内部上下文；它们不是独立的公开审计章节，也不是阻止安装的判定。

风险分析使用 [OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/) 作为视角，审视提示注入、工具误用、凭证暴露、不安全执行、记忆或上下文污染，以及过度自主性等风险。

ClawScan 不会把看起来吓人的能力自动视为恶意。它会判断该能力是否已披露、是否与用途一致，以及是否受到该发布版本所述用例的支持。
