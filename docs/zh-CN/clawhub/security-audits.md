# 安全审计

ClawHub 安全审计可帮助你判断某个技能或插件是否足够安全，可以安装。审计会显示某个版本的作用、它请求的权限，以及在其访问文件、账户、凭据、代码或外部服务之前，是否有任何事项需要额外关注。

审计是强有力的安全信号，但并不保证某个版本毫无风险。授予敏感访问权限前，请始终自行判断。

另请参阅[安全](/clawhub/security)、[可接受使用政策](/zh-CN/clawhub/acceptable-usage)和[内容审核与账户安全](/clawhub/moderation)。

## 安装前需要检查的事项

安装前，请检查：

- 整体审计状态
- 风险级别
- 列出的所有发现
- 所需的凭据、权限或环境变量
- 所有者、来源、版本、变更日志、下载量、星标数及其他信任信号

仅安装你了解并信任的内容。

## 审计状态

审计状态告诉你应如何处理审计结果：

| 状态        | 含义                                                   |
| ----------- | ------------------------------------------------------ |
| `Pass`      | 未发现低风险以上的明显问题。                           |
| `Review`    | 安装前请阅读审计发现。该版本仍可能是合法的。           |
| `Warn`      | 请格外谨慎。ClawHub 发现了高影响问题或警告信号。       |
| `Malicious` | 请勿安装。                                             |
| `Pending`   | 审计尚未完成。                                         |
| `Error`     | 无法完成审计。                                         |

`Pass` 能让人更加放心，但不能代替你自己的判断。对于能够发布内容、编辑数据、运行命令、读取文件或访问生产系统的工具，这一点尤为重要。

## 风险级别

风险级别描述影响范围：按预期使用该版本时，它可能拥有多大的权限。

| 风险级别 | 含义                                                         |
| -------- | ------------------------------------------------------------ |
| `Low`    | 未发现多少敏感权限或用户影响。                               |
| `Medium` | 该版本拥有重要权限，例如访问账户或更改数据。                 |
| `High`   | 该版本拥有高影响权限、存在严重发现或恶意信号。               |

风险级别和审计状态回答的是不同问题：

- 风险级别回答：“这里有多大的权限？”
- 审计状态回答：“我应该如何处理这个结果？”

例如，某个发布内容的技能可能显示为 `Review`，风险级别为 `Medium`。这并不意味着它是恶意的，而是表示该技能的行为看起来符合其用途，但可以使用重要的账户权限执行操作。

## 审计发现

审计发现会解释为何显示该审计结果。每项发现通常包括：

- 它意味着什么
- 它为何被标记
- 相关的技能或插件内容
- 建议

审计发现可能标记为 `Info`、`Low`、`Medium`、`High` 或 `Critical`。严重性较高的发现对风险级别和审计状态的影响更大。

低置信度的发现不会显示在公开审计汇总中，以便页面专注于有用的证据。

## ClawHub 检查的内容

ClawHub 会审计提交的版本工件，包括：

- 技能说明或插件元数据
- 声明的环境变量和权限
- 安装说明和软件包元数据
- 包含的文件和文件清单
- 兼容性和能力元数据

核心问题是内容是否一致：名称、摘要、元数据、请求的权限和实际内容是否符合用户的合理预期？

强大的行为并不一定有害。许多实用工具都需要凭据、本地命令、提供商 API 或安装软件包。审计会检查这些权限是否符合预期、是否已披露以及是否适度。

工件页面会链接到以下路径中的完整审计：

```text
/<owner>/skills/<slug>/security-audit
```

审计页面结合了：

1. SkillSpector
2. VirusTotal
3. 风险分析

## VirusTotal

ClawHub 在审计体系中使用 VirusTotal 提供恶意软件遥测数据。VirusTotal 是文件信誉和恶意软件扫描领域广受信赖的行业标准；通过双方合作，ClawHub 可以在技能和插件审查中加入更广泛的安全情报。

VirusTotal 对于识别已知恶意工件、扫描引擎命中结果和信誉信号尤其有用，这些信息可以补充 ClawHub 针对智能体的审查。当供应商引擎数量可用时，审计会使用通俗语言进行汇总，例如：

```text
62/62 家供应商将此技能标记为安全。
```

或者：

```text
2/64 家供应商将此技能标记为恶意，1/64 家将其标记为可疑，61/64 家将其标记为安全。
```

当 ClawHub 没有可供汇总的供应商数量遥测数据时，审计会显示：

```text
无 VirusTotal 审计发现
```

VirusTotal 仍然只是遥测数据，不能取代 ClawHub 自身针对工件的风险分析。

## 风险分析

风险分析由 ClawHub 自有的安全审计系统 ClawScan 在内部提供支持。它会将每个版本作为面向智能体的工件进行审查，包括说明、元数据、声明的权限、文件、能力信号、静态扫描信号、SkillSpector 审计发现、VirusTotal 遥测数据以及发布者提供的背景信息。静态扫描信号是本次审查的内部背景信息；它们既不是独立的公开审计部分，也不是阻止安装的判定。

风险分析使用
[OWASP 智能体技能十大风险](https://owasp.org/www-project-agentic-skills-top-10/)
作为分析提示词注入、工具滥用、凭据泄露、不安全执行、记忆或上下文投毒以及过度自主权等风险的框架。

ClawScan 不会自动将看起来危险的能力视为恶意能力。它会判断该能力是否已披露、是否符合其用途，以及是否有该版本所述使用场景的支持。
