# 安全审核

ClawHub 安全审核可帮助你判断某个技能或插件是否足够安全、可以安装。它们会展示某个版本会做什么、请求什么权限，以及在它访问文件、账号、凭据、代码或外部服务前，是否有任何需要额外注意的内容。

审核是强安全信号，但不能保证某个版本完全没有风险。授予敏感访问权限前，请始终自行判断。

另请参阅 [安全](/clawhub/security)、[可接受使用](/clawhub/acceptable-usage) 和 [审核与账号安全](/clawhub/moderation)。

## 安装前要检查什么

安装前，请查看：

- 整体审核状态
- 风险级别
- 任何列出的发现
- 必需的凭据、权限或环境变量
- 所有者、来源、版本、更新日志、下载量、星标数和其他信任信号

只安装你理解并信任的内容。

## 审核状态

审核状态告诉你应如何处理审核结果：

| 状态        | 含义                                                                  |
| ----------- | --------------------------------------------------------------------- |
| `Pass`      | 未发现高于低风险的可见问题。                                          |
| `Review`    | 安装前请阅读发现。该版本仍可能是合法的。                              |
| `Warn`      | 请格外谨慎。ClawHub 发现了高影响问题或警告信号。                     |
| `Malicious` | 不要安装。                                                            |
| `Pending`   | 审核尚未完成。                                                        |
| `Error`     | 无法完成审核。                                                        |

`Pass` 令人安心，但不能取代你自己的判断。对于能够发布内容、编辑数据、运行命令、读取文件或访问生产系统的工具，这一点最为重要。

## 风险级别

风险级别描述影响范围：如果你按预期使用该版本，它看起来拥有多大的能力。

| 风险级别 | 含义                                                                  |
| -------- | --------------------------------------------------------------------- |
| `Low`    | 未发现多少敏感权限或用户影响。                                        |
| `Medium` | 该版本具有有意义的权限，例如账号访问或数据更改。                      |
| `High`   | 该版本具有高影响权限、严重发现或恶意信号。                            |

风险级别和审核状态回答的是不同问题：

- 风险级别问的是：“这里有多大能力？”
- 审核状态问的是：“我应该如何处理这个结果？”

例如，某个发布技能可能显示为 `Review`，风险为 `Medium`。这并不意味着它是恶意的。这意味着该技能看起来符合用途，但可以使用有意义的账号权限执行操作。

## 发现

发现解释为什么会显示某个审核结果。每条发现通常包括：

- 它意味着什么
- 为什么被标记
- 相关的技能或插件内容
- 一项建议

发现可能标记为 `Info`、`Low`、`Medium`、`High` 或 `Critical`。严重程度更高的发现会更强地影响风险级别和审核状态。

低置信度发现会从公开审核汇总中隐藏，以便页面专注于有用证据。

## ClawHub 检查什么

ClawHub 会审核提交的发布工件，包括：

- 技能说明或插件元数据
- 声明的环境变量和权限
- 安装说明和包元数据
- 包含的文件和文件清单
- 兼容性和能力元数据

核心问题是一致性：名称、摘要、元数据、请求的权限和实际内容，是否与用户合理预期相符？

强大的行为并不一定是坏事。许多有用工具需要凭据、本地命令、提供商 API 或包安装。审核会检查这些能力是否符合预期、是否已披露，以及是否与用途相称。

工件页面会链接到完整审核：

```text
/<owner>/skills/<slug>/security-audit
```

审核页面整合了：

1. SkillSpector
2. VirusTotal
3. 风险分析

## VirusTotal

ClawHub 在审核栈中使用 VirusTotal 作为恶意软件遥测。VirusTotal 是文件信誉和恶意软件扫描方面受信任的行业标准，我们的合作关系让 ClawHub 能够将更广泛的安全情报加入技能和插件审核。

VirusTotal 对已知恶意工件、引擎命中以及补充 ClawHub 智能体感知审核的信誉信号尤其有用。当供应商引擎计数可用时，审核会用直白语言总结它们，例如：

```text
62/62 vendors flagged this skill as clean.
```

或：

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

当 ClawHub 没有可总结的供应商计数遥测时，审核会显示：

```text
No VirusTotal findings
```

VirusTotal 仍然是遥测。它不能取代 ClawHub 自己的工件感知风险分析。

## 风险分析

风险分析由 ClawHub 自有安全审核系统 ClawScan 在内部提供支持。它会将每个版本作为面向智能体的工件进行审核：说明、元数据、声明的权限、文件、能力信号、静态扫描信号、SkillSpector 发现、VirusTotal 遥测以及发布者提供的上下文。静态扫描信号是本次审核的内部上下文；它们不是独立的公开审核部分，也不是阻止安装的结论。

风险分析使用 [OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/) 作为视角，审视提示注入、工具误用、凭据暴露、不安全执行、记忆或上下文投毒，以及过度自主性等风险。

ClawScan 不会把看起来可怕的能力自动视为恶意。它会判断该能力是否已披露、是否符合用途，以及是否受该版本声明的使用场景支持。
