# 安全审计

ClawHub 安全审计可帮助你判断某个技能或插件是否足够安全，可以安装。它们会显示某个版本会做什么、请求什么权限，以及在它能够访问文件、账户、凭证、代码或外部服务之前，是否有任何内容需要额外关注。

审计是强有力的安全信号，但并不能保证某个版本完全没有风险。授予敏感访问权限前，始终要自行判断。

另请参阅 [安全](/clawhub/security)、[可接受使用](/zh-CN/clawhub/acceptable-usage) 和 [审核与账户安全](/clawhub/moderation)。

## 安装前要检查什么

安装前，请检查：

- 整体审计状态
- 风险级别
- 所列出的任何发现
- 所需凭证、权限或环境变量
- 所有者、来源、版本、变更日志、下载量、星标数和其他信任信号

只安装你理解且信任的内容。

## 审计状态

审计状态告诉你应如何对审计结果做出反应：

| 状态        | 含义                                                                   |
| ----------- | ---------------------------------------------------------------------- |
| `Pass`      | 未发现低风险以上的可见问题。                                           |
| `Review`    | 安装前请阅读发现。该版本仍可能是合法的。                               |
| `Warn`      | 请格外谨慎。ClawHub 发现了高影响问题或警告信号。                      |
| `Malicious` | 不要安装。                                                             |
| `Pending`   | 审计尚未完成。                                                         |
| `Error`     | 审计无法完成。                                                         |

`Pass` 令人放心，但不能替代你自己的判断。对于能够发布内容、编辑数据、运行命令、读取文件或访问生产系统的工具，这一点尤其重要。

## 风险级别

风险级别描述影响范围：如果你按预期使用该版本，它看起来拥有多大的权限。

| 风险级别 | 含义                                                                       |
| -------- | -------------------------------------------------------------------------- |
| `Low`    | 未发现多少敏感权限或用户影响。                                             |
| `Medium` | 该版本拥有有意义的权限，例如账户访问或数据更改。                           |
| `High`   | 该版本拥有高影响权限、严重发现或恶意信号。                                 |

风险级别和审计状态回答的是不同问题：

- 风险级别问：“这里有多大权限？”
- 审计状态问：“我应该如何处理这个结果？”

例如，一个发布技能可能显示 `Review`，风险为 `Medium`。这并不意味着它是恶意的。这意味着该技能看起来与用途一致，但可以使用有意义的账户权限执行操作。

## 发现

发现会解释为什么显示某个审计结果。每条发现通常包括：

- 它意味着什么
- 它为什么被标记
- 相关的技能或插件内容
- 一项建议

发现可能被标记为 `Info`、`Low`、`Medium`、`High` 或 `Critical`。严重性越高的发现，对风险级别和审计状态的影响越强。

低置信度发现会从公开审计汇总中隐藏，以便页面专注于有用证据。

## ClawHub 检查什么

ClawHub 会审计提交的发布产物，包括：

- 技能说明或插件元数据
- 声明的环境变量和权限
- 安装说明和包元数据
- 包含的文件和文件清单
- 兼容性和能力元数据

核心问题是一致性：名称、摘要、元数据、请求的权限和实际内容，是否与用户可合理预期的内容一致？

强大的行为并不一定是坏事。许多有用工具都需要凭证、本地命令、提供商 API 或包安装。审计会检查这些权限是否符合预期、已披露且成比例。

产物页面会链接到完整审计，地址为：

```text
/<owner>/skills/<slug>/security-audit
```

审计页面组合了：

1. SkillSpector
2. VirusTotal
3. 风险分析

## VirusTotal

ClawHub 在审计栈中使用 VirusTotal 作为恶意软件遥测。VirusTotal 是文件信誉和恶意软件扫描领域值得信赖的行业标准，我们的合作关系让 ClawHub 能够为技能和插件审核加入更广泛的安全情报。

VirusTotal 对于已知恶意产物、引擎命中，以及补充 ClawHub 的智能体感知审核的信誉信号尤其有用。当供应商引擎计数可用时，审计会用通俗语言汇总它们，例如：

```text
62/62 vendors flagged this skill as clean.
```

或：

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

当 ClawHub 没有可汇总的供应商计数遥测时，审计会显示：

```text
No VirusTotal findings
```

VirusTotal 仍然是遥测。它不能替代 ClawHub 自己的产物感知风险分析。

## 风险分析

风险分析由 ClawHub 自己的安全审计系统 ClawScan 在内部驱动。它会将每个版本作为面向智能体的产物进行审核：说明、元数据、声明的权限、文件、能力信号、静态扫描信号、SkillSpector 发现、VirusTotal 遥测以及发布者提供的上下文。静态扫描信号是本次审核的内部上下文；它们不是独立的公开审计章节，也不是阻止安装的裁决。

风险分析使用 [OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/) 作为视角，审视提示注入、工具误用、凭证暴露、不安全执行、记忆或上下文污染，以及过度代理权等风险。

ClawScan 不会把看起来吓人的能力自动视为恶意。它会判断该能力是否已披露、与用途一致，并且是否由该版本声明的使用场景支持。
