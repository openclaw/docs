# 安全审计

ClawHub 安全审计可帮助你判断某项技能或插件是否足够安全，可以安装。审计会说明某个版本的用途、它请求的权限，以及在允许其访问文件、账户、凭据、代码或外部服务之前，是否有任何事项需要额外关注。

审计是有力的安全信号，但不能保证某个版本毫无风险。授予敏感访问权限前，务必自行判断。

另请参阅[安全](/clawhub/security)、[可接受使用政策](/zh-CN/clawhub/acceptable-usage)和[审核与账户安全](/clawhub/moderation)。

## 安装前的检查事项

安装前，请检查：

- 整体审计状态
- 风险级别
- 列出的所有发现
- 所需的凭据、权限或环境变量
- 所有者、来源、版本、变更日志、下载量、星标数和其他可信度信号

仅安装你了解并信任的内容。

## 审计状态

审计状态说明应如何处理审计结果：

| 状态      | 含义                                                                   |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 未发现高于低风险级别的明显问题。                                |
| `Review`    | 安装前请阅读相关发现。该版本仍可能是合法的。 |
| `Warn`      | 请格外谨慎。ClawHub 发现了影响较大的问题或警告信号。 |
| `Malicious` | 请勿安装。                                                           |
| `Pending`   | 审计尚未完成。                                             |
| `Error`     | 无法完成审计。                                         |

`Pass` 让人放心，但不能替代你自己的判断。对于能够发布内容、编辑数据、运行命令、读取文件或访问生产系统的工具，这一点尤为重要。

## 风险级别

风险级别描述影响范围：按预期使用该版本时，它看起来拥有多大权限。

| 风险级别 | 含义                                                                       |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | 未发现多少敏感权限或对用户的影响。                          |
| `Medium`   | 该版本拥有实质性权限，例如访问账户或更改数据。 |
| `High`     | 该版本拥有影响较大的权限、存在严重发现或恶意信号。 |

风险级别和审计状态回答的是不同问题：

- 风险级别问：“这里有多大权限？”
- 审计状态问：“我应如何处理此结果？”

例如，某项发布技能可能显示 `Review`，风险为 `Medium`。这并不意味着它是恶意的，而是表示该技能看起来符合其用途，但可以使用实质性的账户权限执行操作。

## 发现

发现会解释显示某项审计结果的原因。每项发现通常包括：

- 它表示什么
- 它被标记的原因
- 相关的技能或插件内容
- 建议

发现可能标记为 `Info`、`Low`、`Medium`、`High` 或 `Critical`。严重程度越高的发现，对风险级别和审计状态的影响越大。

低置信度的发现不会显示在公开审计汇总中，以便页面集中呈现有用的证据。

## ClawHub 的检查内容

ClawHub 会审计提交的版本工件，包括：

- 技能说明或插件元数据
- 声明的环境变量和权限
- 安装说明和软件包元数据
- 包含的文件和文件清单
- 兼容性和能力元数据

核心问题是内容是否一致：名称、摘要、元数据、请求的权限和实际内容是否符合用户的合理预期？

强大的行为并不一定有害。许多实用工具需要凭据、本地命令、提供商 API 或安装软件包。审计会检查这些权限是否符合预期、是否已披露且是否适度。

工件页面通过以下路径链接至完整审计：

```text
/<owner>/skills/<slug>/security-audit
```

审计页面综合呈现：

1. SkillSpector
2. VirusTotal
3. 风险分析

## VirusTotal

ClawHub 在审计体系中使用 VirusTotal 提供恶意软件遥测数据。VirusTotal 是文件信誉评估和恶意软件扫描领域值得信赖的行业标准。通过双方的合作，ClawHub 可以在技能和插件审查中加入更广泛的安全情报。

VirusTotal 特别适合识别已知恶意工件、扫描引擎命中和信誉信号，以补充 ClawHub 面向智能体的审查。如果能够获取供应商引擎的统计数量，审计会使用简明语言进行汇总，例如：

```text
62/62 家供应商将此技能标记为安全。
```

或：

```text
2/64 家供应商将此技能标记为恶意，1/64 家将其标记为可疑，61/64 家将其标记为安全。
```

如果 ClawHub 没有可供汇总的供应商数量遥测数据，审计会显示：

```text
无 VirusTotal 发现
```

VirusTotal 仍然只是遥测数据，不能替代 ClawHub 自身针对工件的风险分析。

## 风险分析

风险分析由 ClawHub 自有的内部安全审计系统 ClawScan 提供支持。它将每个版本视为面向智能体的工件进行审查，包括说明、元数据、声明的权限、文件、能力信号、静态扫描信号、SkillSpector 发现、VirusTotal 遥测数据和发布者提供的背景信息。静态扫描信号是此次审查使用的内部背景信息，并非独立的公开审计部分或阻止安装的结论。

风险分析以
[OWASP 智能体技能十大风险](https://owasp.org/www-project-agentic-skills-top-10/)
为框架，评估提示词注入、工具滥用、凭据泄露、不安全执行、记忆或上下文投毒以及过度自主权等风险。

ClawScan 不会因为某项能力看起来危险，就自动将其视为恶意能力。它会判断该能力是否已披露、是否符合其用途，以及是否与该版本声明的使用场景相符。
