# 安全审计

ClawHub 安全审计可帮助你判断某个技能或插件是否足够安全，可以安装。它们会展示某个发布版本会做什么、请求什么权限，以及在访问文件、账号、凭证、代码或外部服务之前，是否有任何内容需要额外注意。

审计是强安全信号，但并不保证某个发布版本完全没有风险。在授予敏感访问权限之前，请始终自行判断。

另请参阅[安全](/clawhub/security)、[可接受使用](/clawhub/acceptable-usage)和[审核与账号安全](/clawhub/moderation)。

## 安装前要检查什么

安装前，请查看：

- 整体审计状态
- 风险级别
- 任何列出的发现
- 所需的凭证、权限或环境变量
- 所有者、来源、版本、变更日志、下载量、星标数和其他信任信号

只安装你理解并信任的内容。

## 审计状态

审计状态会告诉你应如何应对审计结果：

| 状态        | 含义                                                                      |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 未发现高于低风险的可见问题。                                              |
| `Review`    | 安装前阅读发现。该发布版本仍可能是合法的。                                |
| `Warn`      | 需要格外谨慎。ClawHub 发现了高影响问题或警告信号。                        |
| `Malicious` | 不要安装。                                                                |
| `Pending`   | 审计尚未完成。                                                            |
| `Error`     | 无法完成审计。                                                            |

`Pass` 能让人放心，但不能取代你自己的判断。对于能够发布内容、编辑数据、运行命令、读取文件或访问生产系统的工具，这一点最重要。

## 风险级别

风险级别描述影响范围：如果你按预期使用该发布版本，它看起来拥有多大的权限。

| 风险级别 | 含义                                                                         |
| -------- | ---------------------------------------------------------------------------- |
| `Low`    | 未发现多少敏感权限或用户影响。                                               |
| `Medium` | 该发布版本拥有有意义的权限，例如账号访问或数据变更。                         |
| `High`   | 该发布版本拥有高影响权限、严重发现或恶意信号。                               |

风险级别和审计状态回答的是不同问题：

- 风险级别问：“这里有多大权限？”
- 审计状态问：“我应该如何处理这个结果？”

例如，一个发布技能可能显示为 `Review`，风险为 `Medium`。这并不意味着它是恶意的。它意味着该技能看起来与用途一致，但可以使用有意义的账号权限执行操作。

## 发现

发现会解释为什么显示某个审计结果。每个发现通常包括：

- 它意味着什么
- 为什么被标记
- 相关技能或插件内容
- 建议

发现可能标记为 `Info`、`Low`、`Medium`、`High` 或 `Critical`。严重级别越高，对风险级别和审计状态的影响越强。

低置信度发现会从公开审计汇总中隐藏，这样页面可以聚焦在有用证据上。

## ClawHub 检查什么

ClawHub 会审计提交的发布产物，包括：

- 技能说明或插件元数据
- 声明的环境变量和权限
- 安装说明和包元数据
- 包含的文件和文件清单
- 兼容性和能力元数据

核心问题是一致性：名称、摘要、元数据、请求的权限和实际内容，是否符合用户的合理预期？

强大的行为并不自动等于坏事。许多有用工具需要凭证、本地命令、提供商 API 或包安装。审计会检查这种权限是否符合预期、是否已披露、是否相称。

产物页面会链接到完整审计：

```text
/<owner>/skills/<slug>/security-audit
```

审计页面结合了：

1. SkillSpector
2. VirusTotal
3. 风险分析

## VirusTotal

ClawHub 在审计栈中使用 VirusTotal 作为恶意软件遥测。VirusTotal 是文件信誉和恶意软件扫描的可信行业标准，我们的合作让 ClawHub 能为技能和插件审核加入更广泛的安全情报。

VirusTotal 对已知恶意产物、引擎命中和信誉信号尤其有用，可补充 ClawHub 的 Agent 感知审核。当供应商引擎计数可用时，审计会用通俗语言汇总它们，例如：

```text
62/62 vendors flagged this skill as clean.
```

或：

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

当 ClawHub 没有可汇总的供应商计数遥测时，审计会显示：

```text
No VirusTotal findings
```

VirusTotal 仍然只是遥测。它不能取代 ClawHub 自身的产物感知风险分析。

## 风险分析

风险分析由 ClawHub 自己的安全审计系统 ClawScan 在内部提供支持。它会以面向 Agent 的产物来审核每个发布版本：说明、元数据、声明的权限、文件、能力信号、静态扫描信号、SkillSpector 发现、VirusTotal 遥测，以及发布者提供的上下文。静态扫描信号是本次审核的内部上下文；它们不是独立的公开审计部分，也不是阻止安装的裁定。

风险分析使用 [OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/) 作为风险视角，例如提示注入、工具误用、凭证暴露、不安全执行、记忆或上下文污染，以及过度代理能力。

ClawScan 不会把看起来吓人的能力自动视为恶意。它会判断该能力是否已披露、是否与用途一致，并且是否由发布版本声明的使用场景支持。
