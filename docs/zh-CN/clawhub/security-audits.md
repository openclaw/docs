# 安全审计

ClawHub 安全审计可帮助你判断某个 Skills 或插件是否足够安全，可以安装。审计会显示某个发布版本的作用、它请求哪些权限，以及在它能够访问文件、账户、凭据、代码或外部服务之前，是否有任何内容值得特别注意。

审计是重要的安全信号，但并不保证发布版本毫无风险。在授予敏感访问权限之前，始终应自行判断。

另请参阅[安全](/clawhub/security)、[可接受使用政策](/zh-CN/clawhub/acceptable-usage)和[审核与账户安全](/clawhub/moderation)。

## 安装前需要检查的内容

安装前，请检查：

- 整体审计状态
- 风险级别
- 列出的所有发现
- 所需的凭据、权限或环境变量
- 所有者、来源、版本、变更日志、下载量、星标数及其他可信度信号

仅安装你理解并信任的内容。

## 审计状态

审计状态说明你应如何应对审计结果：

| 状态        | 含义                                                               |
| ----------- | ------------------------------------------------------------------ |
| `Pass`      | 未发现高于低风险级别的明显问题。                                   |
| `Review`    | 安装前请阅读审计发现。该发布版本仍可能是合法的。                   |
| `Warn`      | 请格外谨慎。ClawHub 发现了高影响问题或警告信号。                   |
| `Malicious` | 请勿安装。                                                         |
| `Pending`   | 审计尚未完成。                                                     |
| `Error`     | 无法完成审计。                                                     |

`Pass` 能让人更放心，但不能取代你自己的判断。对于能够发布内容、编辑数据、运行命令、读取文件或访问生产系统的工具，这一点尤为重要。

## 风险级别

风险级别描述影响范围：按预期使用发布版本时，它看起来拥有多大的权限。

| 风险级别 | 含义                                                               |
| -------- | ------------------------------------------------------------------ |
| `Low`    | 未发现多少敏感权限或用户影响。                                     |
| `Medium` | 该发布版本拥有实质性权限，例如访问账户或更改数据。                 |
| `High`   | 该发布版本拥有高影响权限、存在严重发现或恶意信号。                 |

风险级别和审计状态回答的是不同问题：

- 风险级别问：“这里有多大权限？”
- 审计状态问：“我应该如何处理这个结果？”

例如，某个发布内容的 Skills 可能显示 `Review`，风险级别为 `Medium`。这并不意味着它是恶意的，而是意味着该 Skills 看起来符合其声明的用途，但能够使用实质性的账户权限执行操作。

## 审计发现

审计发现用于解释为什么会显示相应的审计结果。每项发现通常包括：

- 它意味着什么
- 为什么被标记
- 相关的 Skills 或插件内容
- 建议

发现可能标记为 `Info`、`Low`、`Medium`、`High` 或 `Critical`。严重级别越高，对风险级别和审计状态的影响越大。

低置信度的发现在公开审计汇总中会被隐藏，以使页面专注于有用的证据。

## ClawHub 检查的内容

ClawHub 会审计提交的发布工件，包括：

- Skills 指令或插件元数据
- 声明的环境变量和权限
- 安装说明和软件包元数据
- 包含的文件和文件清单
- 兼容性和能力元数据

核心问题是一致性：名称、摘要、元数据、请求的权限和实际内容是否与用户的合理预期一致？

强大的行为并不一定有害。许多实用工具都需要凭据、本地命令、提供商 API 或安装软件包。审计会检查这些权限是否符合预期、是否已披露，以及是否适度。

工件页面通过以下路径链接到完整审计：

```text
/<owner>/skills/<slug>/security-audit
```

审计页面汇总以下内容：

1. SkillSpector
2. VirusTotal
3. 风险分析

## VirusTotal

ClawHub 在审计技术栈中使用 VirusTotal 作为恶意软件遥测来源。VirusTotal 是文件信誉和恶意软件扫描领域广受信赖的行业标准；通过双方合作，ClawHub 可以在 Skills 和插件审查中加入更广泛的安全情报。

VirusTotal 特别适合识别已知恶意工件、扫描引擎命中结果和信誉信号，以补充 ClawHub 面向智能体的审查。当可以获取供应商引擎数量时，审计会使用通俗语言进行汇总，例如：

```text
62/62 家供应商将此 Skills 标记为安全。
```

或者：

```text
2/64 家供应商将此 Skills 标记为恶意，1/64 家将其标记为可疑，61/64 家将其标记为安全。
```

当 ClawHub 没有可供汇总的供应商数量遥测数据时，审计会显示：

```text
无 VirusTotal 发现
```

VirusTotal 仍然只是一种遥测来源，不能取代 ClawHub 自身面向工件的风险分析。

## 风险分析

风险分析由 ClawHub 自有的安全审计系统 ClawScan 在内部提供支持。它将每个发布版本作为面向智能体的工件进行审查，涵盖指令、元数据、声明的权限、文件、能力信号、静态扫描信号、SkillSpector 发现、VirusTotal 遥测数据以及发布者提供的上下文。静态扫描信号是此审查的内部上下文，并非独立的公开审计部分，也不是阻止安装的判定结果。

风险分析以
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
作为评估提示注入、工具滥用、凭据泄露、不安全执行、记忆或上下文投毒以及过度自主性等风险的视角。

ClawScan 不会将看起来危险的能力自动视为恶意能力。它会判断该能力是否已披露、是否符合其用途，以及是否有发布版本所声明的使用场景作为支持。
