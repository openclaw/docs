# 安全审计

ClawHub 安全审计可帮助你判断某项技能或插件是否足够安全，适合安装。审计会说明某个发布版本的功能、它请求的权限，以及在它能够访问文件、账户、凭据、代码或外部服务之前，是否有任何事项需要格外注意。

审计是重要的安全信号，但不能保证发布版本毫无风险。在授予敏感访问权限前，请始终谨慎判断。

另请参阅[安全](/clawhub/security)、[可接受使用政策](/clawhub/acceptable-usage)和[内容审核与账户安全](/clawhub/moderation)。

## 安装前需要检查的内容

安装前，请检查：

- 整体审计状态
- 风险级别
- 列出的所有发现
- 所需的凭据、权限或环境变量
- 所有者、来源、版本、变更日志、下载量、星标数和其他可信度信号

只安装你了解并信任的内容。

## 审计状态

审计状态说明你应如何应对审计结果：

| 状态        | 含义                                                       |
| ----------- | ---------------------------------------------------------- |
| `Pass`      | 未发现高于低风险级别的明显问题。                           |
| `Review`    | 安装前请阅读审计发现。该发布版本仍可能是合法的。           |
| `Warn`      | 请格外谨慎。ClawHub 发现了影响较大的问题或警告信号。       |
| `Malicious` | 请勿安装。                                                 |
| `Pending`   | 审计尚未完成。                                             |
| `Error`     | 无法完成审计。                                             |

`Pass` 能让人更放心，但不能代替你自己的判断。对于能够发布内容、编辑数据、运行命令、读取文件或访问生产系统的工具，这一点尤其重要。

## 风险级别

风险级别描述影响范围：如果你按预期方式使用该发布版本，它看起来拥有多大的权限。

| 风险级别 | 含义                                                       |
| -------- | ---------------------------------------------------------- |
| `Low`    | 未发现多少敏感权限或用户影响。                             |
| `Medium` | 该发布版本拥有实质性权限，例如访问账户或更改数据。         |
| `High`   | 该发布版本拥有高影响权限、存在严重发现或恶意信号。         |

风险级别和审计状态回答的是不同问题：

- 风险级别询问：“这里有多大的权限？”
- 审计状态询问：“我应如何处理这个结果？”

例如，某项发布类技能可能显示 `Review` 状态和 `Medium` 风险。这并不意味着它是恶意的，而是表示该技能的功能看起来符合其用途，但能够使用实质性的账户权限执行操作。

## 审计发现

审计发现解释为什么会显示相应的审计结果。每项发现通常包括：

- 它的含义
- 被标记的原因
- 相关的技能或插件内容
- 建议

审计发现可能标记为 `Info`、`Low`、`Medium`、`High` 或 `Critical`。严重程度越高的发现，对风险级别和审计状态的影响越大。

低置信度发现不会显示在公开审计汇总中，以便页面集中展示有用的证据。

## ClawHub 检查的内容

ClawHub 会审计提交的发布制品，包括：

- 技能说明或插件元数据
- 声明的环境变量和权限
- 安装说明和软件包元数据
- 包含的文件和文件清单
- 兼容性和能力元数据

核心问题是内容是否一致：名称、摘要、元数据、请求的权限和实际内容，是否符合用户的合理预期？

强大的功能并不一定有害。许多实用工具需要凭据、本地命令、提供商 API 或安装软件包。审计会检查这些权限是否符合预期、是否已披露，以及是否与用途相称。

制品页面通过以下路径链接到完整审计：

```text
/<owner>/skills/<slug>/security-audit
```

审计页面综合了：

1. SkillSpector
2. VirusTotal
3. 风险分析

## VirusTotal

ClawHub 在审计体系中使用 VirusTotal 提供恶意软件遥测数据。VirusTotal 是文件信誉评估和恶意软件扫描领域备受信赖的行业标准。通过双方合作，ClawHub 能够将更广泛的安全情报纳入技能和插件审查。

VirusTotal 对识别已知恶意制品、扫描引擎命中结果和信誉信号尤其有用，这些信息可作为 ClawHub 面向智能体的审查机制的补充。当能够获取供应商引擎统计数据时，审计会以通俗语言进行汇总，例如：

```text
62/62 vendors flagged this skill as clean.
```

或：

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

当 ClawHub 没有可供汇总的供应商统计遥测数据时，审计会显示：

```text
No VirusTotal findings
```

VirusTotal 仍然只是遥测数据，不能取代 ClawHub 自身针对制品的风险分析。

## 风险分析

风险分析由 ClawHub 自有的安全审计系统 ClawScan 在内部提供支持。它将每个发布版本作为面向智能体的制品进行审查，包括说明、元数据、声明的权限、文件、能力信号、静态扫描信号、SkillSpector 发现、VirusTotal 遥测数据和发布者提供的背景信息。静态扫描信号是这项审查的内部参考信息；它们不是独立的公开审计部分，也不是阻止安装的判定结果。

风险分析以
[OWASP Agentic Skills 十大风险](https://owasp.org/www-project-agentic-skills-top-10/)
为评估框架，关注提示词注入、工具滥用、凭据泄露、不安全执行、记忆或上下文投毒以及过度自主性等风险。

ClawScan 不会自动将看起来危险的能力视为恶意行为。它会判断该能力是否已披露、是否符合用途，以及是否有发布版本所声明的使用场景作为依据。
