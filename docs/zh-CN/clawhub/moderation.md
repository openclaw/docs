# 审核与账户安全

ClawHub 对发布保持开放，但公开发现和安装界面仍然需要保护措施。当某个版本或账户看起来不安全、具有误导性或不符合政策时，举报、审核暂缓、隐藏条目和账户处置有助于保护用户。

本页介绍审核和账户状态。有关 `Pass`、`Review`、`Warn`、`Malicious` 和风险级别等审计标签，请参阅[安全审计](/clawhub/security-audits)。

另请参阅[安全](/zh-CN/clawhub/security)和[可接受使用政策](/clawhub/acceptable-usage)。如对版权或其他内容权利有疑虑，请使用[内容权利请求](/clawhub/content-rights)。

## 举报

已登录的用户可以举报 Skills、插件和软件包。

ClawHub 举报仅用于不安全的市场内容，例如：

- 恶意条目
- 误导性元数据
- 未声明的凭据或权限要求
- 可疑的安装说明
- 冒充他人
- 恶意抢注或滥用商标
- 违反[可接受使用政策](/clawhub/acceptable-usage)的内容

在 Skill 页面上使用 **Report skill** 按钮，或对软件包使用软件包举报命令/API。

请勿使用 ClawHub 举报来报告第三方 Skill 或插件自身源代码中的漏洞。请直接向发布者或条目中链接的源代码仓库报告。ClawHub 不维护或修补第三方 Skill 或插件代码。

`openclaw/clawhub` 的 GitHub Security Advisories 用于报告 ClawHub 自身的漏洞，例如网站、API、CLI、注册表、身份验证、扫描、审核或下载/安装信任边界中的缺陷。请勿使用 ClawHub 安全公告来报告第三方 Skills 或插件中的漏洞。

有效的举报应具体且可操作。滥用举报功能本身也可能导致账户处置。

## 组织和命名空间归属声明

有关组织、品牌、软件包作用域、所有者用户名或命名空间所有权的争议，应使用[组织和命名空间归属声明](/clawhub/namespace-claims)流程，而不是产品内举报流程或账户申诉表单。

当你需要 ClawHub 工作人员审核非敏感证明，以确定是否应保留、转移、重命名、隐藏、隔离某个命名空间、为其设置别名或以其他方式进行审查时，请使用该流程。请勿在公开议题中包含密钥、私人文档、非公开法律文件、个人身份证明文件、API 令牌或 DNS 质询令牌。

## 审核暂缓

某些严重发现或政策问题可能导致发布者或条目被置于审核暂缓状态。发生这种情况时，受影响的内容可能会从公开发现界面中隐藏，或者后续发布的内容可能在问题经过审核前默认处于隐藏状态。

审核暂缓旨在 ClawHub 处理高风险事件期间保护用户。确认属于误报后，也可以解除审核暂缓。

## 隐藏或屏蔽的条目

条目可能处于暂缓、隐藏、隔离、撤销状态，或因其他原因无法在公开安装界面中使用。

如果看到上述任一状态，请勿安装该版本，除非所有者解决了问题或审核人员恢复了该版本。

所有者可能仍然可以查看自己被暂缓或隐藏条目的诊断信息。这些诊断信息有助于说明发生了什么，以及条目重新出现在公开界面之前需要进行哪些更改。

## 封禁和账户状态

违反 ClawHub 政策的账户可能失去发布权限。严重滥用可能导致账户封禁、令牌撤销、内容隐藏或条目删除。系统每日检查发布者滥用压力信号。达到 ClawHub 潜在封禁阈值的信号可能触发自动警告。如果警告截止日期后的下一次符合条件的扫描仍然判定发布者达到潜在封禁阈值，ClawHub 可能会自动执行账户处置。置信度较低且限定时间范围的审查信号不会用于自动执行处置。

已删除、被封禁或已停用的账户无法使用 ClawHub API 令牌。如果在账户处置后 CLI 身份验证开始失败，请登录 Web UI 查看账户状态。如果因账户被封禁或停用而无法登录或正常使用 CLI，请使用 [ClawHub 申诉表单](https://appeals.openclaw.ai/)申请恢复审查。

如果扫描器触发的电子邮件将某个 Skill 或插件版本标记为恶意，请下载被屏蔽的已提交版本所对应的已存储扫描结果：`clawhub scan download <slug> --version <version>`。对于插件，请添加 `--kind plugin`。检查扫描输出，修复条目，递增版本号，然后上传修复后的版本。

## 发布者指南

为减少误报并提高用户信任：

- 确保名称、摘要、标签和变更日志准确
- 声明所需的环境变量和权限
- 避免使用经过混淆的安装命令
- 尽可能链接到源代码
- 发布插件前使用试运行
- 如果用户或审核人员询问版本行为，请清晰回应
