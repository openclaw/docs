# 审核与账户安全

ClawHub 对发布开放，但公开发现和安装界面仍然需要防护措施。当某个发布版本或账户看起来不安全、具有误导性或违反政策时，举报、审核限制、隐藏列表和账户措施有助于保护用户。

本页介绍审核和账户状态。有关 `Pass`、`Review`、`Warn`、`Malicious` 和风险级别等审计标签，请参阅[安全审计](/clawhub/security-audits)。

另请参阅[安全性](/clawhub/security)和[可接受使用政策](/clawhub/acceptable-usage)。如有版权或其他内容权利方面的问题，请使用[内容权利请求](/clawhub/content-rights)。

## 举报

已登录的用户可以举报 Skills、插件和软件包。

ClawHub 举报仅用于不安全的市场内容，例如：

- 恶意列表
- 误导性元数据
- 未声明的凭据或权限要求
- 可疑的安装说明
- 冒充他人
- 恶意抢注或商标滥用
- 违反[可接受使用政策](/clawhub/acceptable-usage)的内容

在 Skill 页面上使用 **Report skill** 按钮；对于软件包，则使用软件包举报命令/API。

请勿使用 ClawHub 举报功能报告第三方 Skill 或插件自身源代码中的漏洞。请直接向发布者或列表中链接的源代码仓库报告。ClawHub 不维护或修补第三方 Skill 或插件代码。

`openclaw/clawhub` 的 GitHub Security Advisories 用于报告 ClawHub 本身的漏洞。例如网站、API、CLI、注册表、身份验证、扫描、审核或下载/安装信任边界中的缺陷。请勿使用 ClawHub 安全公告报告第三方 Skills 或插件中的漏洞。

有效的举报应具体且可采取行动。滥用举报功能本身也可能导致账户受到处理。

## 组织和命名空间声明

组织、品牌、软件包作用域、所有者用户名或命名空间所有权争议应使用[组织和命名空间声明](/clawhub/namespace-claims)流程，而不是产品内举报流程或账户申诉表单。

当你需要 ClawHub 工作人员审核非敏感证明，以确定某个命名空间是否应被保留、转让、重命名、隐藏、隔离、设置别名或以其他方式进行审核时，请使用该流程。请勿在公开议题中包含机密信息、私人文档、非公开法律文件、个人身份证明文件、API 令牌或 DNS 质询令牌。

## 审核限制

某些严重发现或政策问题可能导致发布者或列表受到审核限制。发生这种情况时，受影响的内容可能会从公开发现界面隐藏，或者在问题得到审核之前，后续发布的内容可能会默认为隐藏状态。

审核限制旨在 ClawHub 处理高风险事件期间保护用户。确认属于误报后，也可以解除限制。

## 隐藏或屏蔽的列表

列表可能被限制、隐藏、隔离、撤销，或以其他方式无法通过公开安装界面获取。

如果看到其中一种状态，除非所有者解决了问题或审核人员恢复了该发布版本，否则请勿安装。

所有者仍可能看到自己受到限制或被隐藏列表的诊断信息。这些诊断信息有助于解释发生了什么，以及列表恢复到公开界面之前需要进行哪些更改。

## 封禁和账户状态

违反 ClawHub 政策的账户可能会失去发布权限。严重滥用可能导致账户被封禁、令牌被撤销、内容被隐藏或列表被移除。系统每天检查发布者滥用压力信号。达到 ClawHub 潜在封禁阈值的信号可能会触发自动警告。如果警告截止时间后的下一次符合条件的扫描仍将发布者判定为达到潜在封禁阈值，ClawHub 可能会自动采取账户措施。置信度较低且时间范围有限的审核信号不会用于自动执行。

已删除、被封禁或已停用的账户无法使用 ClawHub API 令牌。如果采取账户措施后 CLI 身份验证开始失败，请登录 Web UI 查看账户状态。如果因账户被封禁或停用而无法登录或正常访问 CLI，请使用 [ClawHub 申诉表单](https://appeals.openclaw.ai/)申请恢复审核。

如果扫描程序触发的电子邮件将某个 Skill 或插件版本标记为恶意，请下载被屏蔽提交版本的已存储扫描结果：`clawhub scan download <slug> --version <version>`。对于插件，请添加 `--kind plugin`。查看扫描输出、修复列表、递增版本号，然后上传修复后的版本。

## 发布者指南

为了减少误报并提高用户信任度：

- 确保名称、摘要、标签和变更日志准确
- 声明所需的环境变量和权限
- 避免使用经过混淆的安装命令
- 尽可能链接到源代码
- 发布插件前使用试运行
- 如果用户或审核人员询问发布版本的行为，请清晰回应
