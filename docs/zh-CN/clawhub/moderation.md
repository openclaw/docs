# 审核与账号安全

ClawHub 对发布开放，但公共发现和安装界面仍然需要防护措施。举报、审核暂挂、隐藏条目和账号处置可以在某个版本或账号看起来不安全、具有误导性或违反政策时保护用户。

本页介绍审核和账号状态。关于 `Pass`、`Review`、`Warn`、`Malicious` 和风险级别等审计标签，请参阅 [安全审计](/zh-CN/clawhub/security-audits)。

另请参阅 [安全](/zh-CN/clawhub/security) 和 [可接受使用](/zh-CN/clawhub/acceptable-usage)。如有版权或其他内容权利问题，请使用 [内容权利请求](/zh-CN/clawhub/content-rights)。

## 举报

已登录用户可以举报技能、插件和包。

仅将 ClawHub 举报用于不安全的市场内容，例如：

- 恶意条目
- 误导性元数据
- 未声明的凭证或权限要求
- 可疑的安装说明
- 冒充
- 恶意注册或商标滥用
- 违反 [可接受使用](/zh-CN/clawhub/acceptable-usage) 的内容

在技能页面上使用 **报告技能** 按钮，或对包使用包举报命令/API。

不要将 ClawHub 举报用于第三方技能或插件自身源代码中的漏洞。请直接向条目中链接的发布者或源代码仓库报告这些问题。ClawHub 不维护或修补第三方技能或插件代码。

`openclaw/clawhub` 的 GitHub 安全公告用于报告 ClawHub 本身的漏洞。例如网站、API、CLI、注册表、身份验证、扫描、审核或下载/安装信任边界中的错误。不要将 ClawHub 公告用于第三方技能或插件中的漏洞。

好的举报应当具体且可执行。滥用举报本身也可能导致账号处置。

## 组织与命名空间声明

组织、品牌、包作用域、所有者用户名或命名空间所有权争议应使用 [组织与命名空间声明](/zh-CN/clawhub/namespace-claims) 流程，而不是产品内举报流程或账号申诉表单。

当你需要 ClawHub 工作人员审核非敏感证明，以确认某个命名空间应被保留、转移、重命名、隐藏、隔离、设置别名或以其他方式审核时，请使用该流程。不要在公开议题中包含密钥、私人文件、私人法律文件、个人身份证明文件、API 令牌或 DNS 质询令牌。

## 审核暂挂

一些严重发现或政策问题可能会使发布者或条目进入审核暂挂状态。发生这种情况时，受影响内容可能会从公共发现中隐藏，或未来发布可能会先以隐藏状态开始，直到问题得到审核。

审核暂挂旨在 ClawHub 解决高风险案例期间保护用户。确认误报时，也可以解除审核暂挂。

## 隐藏或阻止的条目

条目可能会被暂挂、隐藏、隔离、撤销，或以其他方式在公共安装界面上不可用。

如果你看到这些状态之一，在所有者解决问题或审核恢复该条目之前，不要安装该版本。

所有者仍然可以看到自己被暂挂或隐藏条目的诊断信息。这些诊断信息有助于解释发生了什么，以及条目返回公共界面前需要做出哪些更改。

## 封禁与账号状态

违反 ClawHub 政策的账号可能会失去发布权限。严重滥用可能导致账号封禁、令牌撤销、内容隐藏或条目移除。发布者滥用压力信号会每日检查。达到 ClawHub 潜在封禁阈值的信号可能会触发自动警告。如果警告截止日期后的下一次符合条件的扫描仍然将发布者置于潜在封禁阈值内，ClawHub 可能会自动应用账号处置。置信度较低和有界时间性审核信号不会进入自动执行。

已删除、被封禁或被禁用的账号无法使用 ClawHub API 令牌。如果 CLI 身份验证在账号处置后开始失败，请登录 Web UI 查看账号状态。如果登录或正常 CLI 访问因封禁或账号禁用而被阻止，请使用 [ClawHub 申诉表单](https://appeals.openclaw.ai/) 进行恢复审核。

如果由扫描器触发的电子邮件将某个技能或插件版本标记为恶意，请下载被阻止提交版本的已存储扫描结果：`clawhub scan download <slug> --version <version>`。对于插件，添加 `--kind plugin`。查看扫描输出，修复条目，递增版本号，然后上传修复后的版本。

## 发布者指南

为减少误报并提升用户信任：

- 保持名称、摘要、标签和更新日志准确
- 声明所需的环境变量和权限
- 避免使用混淆的安装命令
- 尽可能链接到源代码
- 发布插件前使用试运行
- 如果用户或审核人员询问版本行为，请清晰回应
