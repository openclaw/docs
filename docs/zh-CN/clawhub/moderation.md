# 审核和账号安全

ClawHub 对发布开放，但公共发现和安装界面仍需要防护措施。举报、审核搁置、隐藏条目和账号处置有助于在某个发布版本或账号看起来不安全、具有误导性或违反政策时保护用户。

本页介绍审核和账号状态。有关 `Pass`、`Review`、`Warn`、`Malicious` 以及风险级别等审计标签，请参阅[安全审计](/clawhub/security-audits)。

另请参阅[安全](/zh-CN/clawhub/security)和[可接受使用](/clawhub/acceptable-usage)。对于版权或其他内容权利问题，请使用[内容权利请求](/clawhub/content-rights)。

## 举报

已登录用户可以举报 Skills、插件和包。

仅在存在不安全的市场内容时使用 ClawHub 举报，例如：

- 恶意条目
- 具有误导性的元数据
- 未声明的凭证或权限要求
- 可疑的安装说明
- 冒充
- 恶意注册或商标滥用
- 违反[可接受使用](/clawhub/acceptable-usage)的内容

在技能页面上使用**举报技能**按钮，或对包使用包举报命令/API。

不要使用 ClawHub 举报来报告第三方技能或插件自身源代码中的漏洞。请直接向条目中链接的发布者或源码仓库报告这些问题。ClawHub 不维护或修补第三方技能或插件代码。

`openclaw/clawhub` 的 GitHub Security Advisories 用于 ClawHub 本身的漏洞。示例包括网站、API、CLI、注册表、认证、扫描、审核或下载/安装信任边界中的 bug。不要使用 ClawHub 安全公告来报告第三方 Skills 或插件中的漏洞。

高质量举报应具体且可操作。滥用举报本身也可能导致账号处置。

## 组织和命名空间申诉

组织、品牌、包作用域、所有者 handle 或命名空间所有权争议应使用[组织和命名空间申诉](/clawhub/namespace-claims)流程，而不是产品内举报流程或账号申诉表单。

当你需要 ClawHub 工作人员审核非敏感证明，以判断某个命名空间是否应被保留、转移、重命名、隐藏、隔离、设置别名或以其他方式审查时，请使用该流程。不要在公开 issue 中包含密钥、私人文档、私人法律文件、个人身份证明文件、API 令牌或 DNS 质询令牌。

## 审核搁置

某些严重发现或政策问题可能会让发布者或条目进入审核搁置状态。发生这种情况时，受影响内容可能会从公共发现中隐藏，或者后续发布可能会先以隐藏状态开始，直到问题完成审核。

审核搁置旨在 ClawHub 处理高风险案例期间保护用户。确认误报后，也可以解除审核搁置。

## 隐藏或阻止的条目

条目可能被搁置、隐藏、隔离、撤销，或以其他方式在公共安装界面上不可用。

如果你看到这些状态之一，除非所有者解决问题或审核恢复该版本，否则不要安装该发布版本。

所有者仍然可以看到其被搁置或隐藏条目的诊断信息。这些诊断信息有助于解释发生了什么，以及条目返回公共界面前需要做出哪些更改。

## 封禁和账号状态

违反 ClawHub 政策的账号可能会失去发布权限。严重滥用可能导致账号封禁、令牌撤销、内容隐藏或条目移除。发布者滥用压力信号会每日检查。达到 ClawHub 潜在封禁阈值的信号可能触发自动警告。如果警告截止时间后的下一次符合条件的扫描仍将发布者置于潜在封禁阈值内，ClawHub 可能会自动应用账号处置。较低置信度和有时间边界的审核信号不会进入自动执行流程。

已删除、被封禁或被禁用的账号无法使用 ClawHub API 令牌。如果 CLI 认证在账号处置后开始失败，请登录 Web UI 查看账号状态。如果封禁或禁用账号阻止了登录或正常 CLI 访问，请使用 [ClawHub 申诉表单](https://appeals.openclaw.ai/)进行恢复审核。

如果由扫描器触发的电子邮件将某个技能或插件版本标记为恶意，请下载被阻止的已提交版本的已存储扫描结果：`clawhub scan download <slug> --version <version>`。对于插件，请添加 `--kind plugin`。查看扫描输出，修复条目，递增版本号，并上传修复后的版本。

## 发布者指南

为减少误报并提升用户信任：

- 保持名称、摘要、标签和变更日志准确
- 声明所需的环境变量和权限
- 避免混淆的安装命令
- 尽可能链接到源码
- 发布插件前使用试运行
- 如果用户或审核人员询问发布行为，请清晰回应
