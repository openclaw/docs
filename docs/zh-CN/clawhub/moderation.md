# 内容审核和账户安全

ClawHub 开放发布，但公开发现和安装界面仍需要防护措施。举报、审核暂挂、隐藏列表和账户操作有助于在某个发布版本或账户看起来不安全、误导性或不符合政策时保护用户。

本页介绍内容审核和账户状态。关于 `Pass`、`Review`、`Warn`、`Malicious` 和风险级别等审计标签，请参阅 [安全审计](/clawhub/security-audits)。

另请参阅 [安全](/clawhub/security) 和 [可接受使用](/clawhub/acceptable-usage)。如涉及版权或其他内容权利问题，请使用 [内容权利请求](/clawhub/content-rights)。

## 举报

已登录用户可以举报 Skills、插件和包。

仅将 ClawHub 举报用于不安全的市场内容，例如：

- 恶意列表
- 误导性元数据
- 未声明的凭证或权限要求
- 可疑安装说明
- 冒充
- 恶意注册或商标滥用
- 违反 [可接受使用](/clawhub/acceptable-usage) 的内容

在 Skills 页面上使用 **举报 Skills** 按钮，或对包使用包举报命令/API。

不要使用 ClawHub 举报来报告第三方 Skills 或插件自身源代码中的漏洞。请直接向列表中链接的发布者或源代码仓库报告这些问题。ClawHub 不维护或修补第三方 Skills 或插件代码。

`openclaw/clawhub` 的 GitHub Security Advisories 用于 ClawHub 本身的漏洞。例如网站、API、CLI、注册表、身份验证、扫描、内容审核或下载/安装信任边界中的漏洞。不要使用 ClawHub advisories 来报告第三方 Skills 或插件中的漏洞。

好的举报应具体且可执行。滥用举报本身也可能导致账户操作。

## 组织和命名空间声明

组织、品牌、包作用域、所有者用户名或命名空间所有权争议应使用 [组织和命名空间声明](/clawhub/namespace-claims) 流程，而不是产品内举报流程或账户申诉表单。

当你需要 ClawHub 工作人员审核非敏感证明，以确定某个命名空间是否应被保留、转让、重命名、隐藏、隔离、别名化或以其他方式审核时，请使用该流程。不要在公开 issue 中包含密钥、私人文件、私人法律文件、个人身份证明文件、API 令牌或 DNS challenge 令牌。

## 审核暂挂

某些严重发现或政策问题可能会使发布者或列表进入审核暂挂状态。发生这种情况时，受影响内容可能会从公开发现中隐藏，或者未来发布可能会先以隐藏状态开始，直到问题完成审核。

审核暂挂旨在 ClawHub 处理高风险案例期间保护用户。确认误报后，也可以解除审核暂挂。

## 隐藏或被阻止的列表

列表可能会被暂挂、隐藏、隔离、撤销，或以其他方式无法在公开安装界面上使用。

如果你看到这些状态之一，除非所有者解决问题或内容审核恢复该发布版本，否则不要安装。

所有者仍可查看自己被暂挂或隐藏列表的诊断信息。这些诊断信息有助于解释发生了什么，以及列表重新回到公开界面前需要做出哪些更改。

## 封禁和账户状态

违反 ClawHub 政策的账户可能会失去发布权限。严重滥用可能导致账户封禁、令牌撤销、内容隐藏或列表移除。发布者滥用压力信号会每日检查。达到 ClawHub 潜在封禁阈值的信号可能触发自动警告。如果警告期限后的下一次合格扫描仍将该发布者判定为处于潜在封禁阈值，ClawHub 可能会自动执行账户操作。较低置信度和有时间边界的审核信号不会纳入自动执行。

已删除、被封禁或被停用的账户无法使用 ClawHub API 令牌。如果账户操作后 CLI 身份验证开始失败，请登录 Web UI 查看账户状态。如果登录或正常 CLI 访问因封禁或账户停用而被阻止，请使用 [ClawHub 申诉表单](https://appeals.openclaw.ai/) 进行恢复审核。

如果扫描器触发的电子邮件将某个 Skills 或插件版本标记为恶意，请下载被阻止提交版本的已存储扫描结果：`clawhub scan download <slug> --version <version>`。对于插件，请添加 `--kind plugin`。查看扫描输出，修复列表，递增版本号，然后上传修复后的版本。

## 发布者指南

为减少误报并提升用户信任：

- 保持名称、摘要、标签和变更日志准确
- 声明所需的环境变量和权限
- 避免混淆的安装命令
- 尽可能链接到源代码
- 发布插件前使用 dry runs
- 如果用户或审核人员询问发布行为，请清晰回应
