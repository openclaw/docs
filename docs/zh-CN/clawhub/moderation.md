# 审核和账号安全

ClawHub 对发布开放，但公开发现和安装界面仍然
需要护栏。举报、审核暂停、隐藏列表和账号操作
有助于在某个发布版本或账号看起来不安全、具有误导性或违反政策时保护用户。

本页涵盖审核和账号状态。对于 `Pass`、`Review`、`Warn`、`Malicious`
以及风险等级等审计标签，请参阅
[安全审计](/clawhub/security-audits)。

另请参阅[安全](/clawhub/security)和
[可接受使用](/clawhub/acceptable-usage)。对于版权或其他内容
权利问题，请使用[内容权利请求](/clawhub/content-rights)。

## 举报

已登录用户可以举报技能、插件和包。

仅将 ClawHub 举报用于不安全的市场内容，例如：

- 恶意列表
- 误导性元数据
- 未声明的凭证或权限要求
- 可疑的安装说明
- 冒充
- 恶意注册或商标滥用
- 违反[可接受使用](/clawhub/acceptable-usage)的内容

在技能页面使用**举报技能**按钮，或对包使用包举报
命令/API。

不要将 ClawHub 举报用于第三方技能或
插件自身源代码中的漏洞。请直接向发布者或列表中链接的源
仓库报告这些问题。ClawHub 不维护或修补
第三方技能或插件代码。

`openclaw/clawhub` 的 GitHub Security Advisories 用于
ClawHub 本身的漏洞。示例包括网站、API、CLI、注册表、认证、
扫描、审核或下载/安装信任边界中的 bug。不要将 ClawHub
安全公告用于第三方技能或插件中的漏洞。

好的举报应具体且可执行。滥用举报本身也可能导致
账号操作。

## 组织和命名空间声明

组织、品牌、包作用域、所有者用户名或命名空间所有权争议应
使用[组织和命名空间声明](/clawhub/namespace-claims)流程，而不是
产品内举报流程或账号申诉表。

当你需要 ClawHub 工作人员审核非敏感证明，以确定某个
命名空间应被保留、转让、重命名、隐藏、隔离、设置别名
或以其他方式审核时，请使用该流程。不要在
公开 issue 中包含密钥、私人文件、私人法律文件、个人身份证件、API 令牌
或 DNS challenge 令牌。

## 审核暂停

某些严重发现或政策问题可能会使发布者或列表进入
审核暂停状态。发生这种情况时，受影响内容可能会从公开
发现中隐藏，或未来发布可能默认隐藏，直到问题完成审核。

审核暂停旨在 ClawHub 处理高风险案例期间保护用户。
当确认是误报时，也可以解除暂停。

## 隐藏或阻止的列表

列表可能会被暂停、隐藏、隔离、撤销，或以其他方式在
公开安装界面上不可用。

如果你看到这些状态之一，请不要安装该发布版本，除非所有者
解决问题或审核恢复它。

所有者仍然可以看到自己被暂停或隐藏列表的诊断信息。这些
诊断有助于解释发生了什么，以及列表重新回到公开界面之前
需要做出哪些更改。

## 封禁和账号状态

违反 ClawHub 政策的账号可能会失去发布访问权限。严重滥用可能
导致账号封禁、令牌吊销、内容隐藏或列表移除。
发布者滥用压力信号每天都会检查。达到
ClawHub 潜在封禁阈值的信号可能触发自动警告。如果警告期限后
下一次符合条件的扫描仍将发布者置于
潜在封禁阈值内，ClawHub 可能会自动应用账号操作。
置信度较低和有时间边界的审核信号不会进入自动
执行。

已删除、被封禁或被禁用的账号无法使用 ClawHub API 令牌。如果 CLI 认证
在账号操作后开始失败，请登录 Web UI 查看账号
状态。如果封禁或禁用账号阻止了登录或正常 CLI 访问，
请使用 [ClawHub 申诉表](https://appeals.openclaw.ai/) 进行恢复审核。

如果扫描器触发的电子邮件将某个技能或插件版本标记为恶意，
请下载被阻止的已提交版本的存储扫描结果：
`clawhub scan download <slug> --version <version>`。对于插件，请添加
`--kind plugin`。查看扫描输出，修复列表，递增版本号，
并上传修复后的版本。

## 发布者指南

为减少误报并提升用户信任：

- 保持名称、摘要、标签和变更日志准确
- 声明所需的环境变量和权限
- 避免混淆的安装命令
- 尽可能链接到源代码
- 发布插件前使用 dry run
- 如果用户或审核员询问发布行为，请清晰回应
