# 审核与账户安全

ClawHub 开放发布，但公开发现和安装界面仍然需要防护措施。举报、审核暂停、隐藏列表和账户操作有助于在某个版本或账户看起来不安全、具有误导性或违反政策时保护用户。

本页介绍审核和账户状态。关于 `Pass`、`Review`、`Warn`、`Malicious` 等审核标签以及风险级别，请参阅 [安全审核](/zh-CN/clawhub/security-audits)。

另请参阅 [安全](/zh-CN/clawhub/security) 和 [可接受使用](/zh-CN/clawhub/acceptable-usage)。对于版权或其他内容权利问题，请使用 [内容权利请求](/zh-CN/clawhub/content-rights)。

## 举报

已登录用户可以举报技能、插件和软件包。

仅将 ClawHub 举报用于不安全的市场内容，例如：

- 恶意列表
- 误导性元数据
- 未声明的凭证或权限要求
- 可疑安装说明
- 冒充
- 恶意注册或商标滥用
- 违反 [可接受使用](/zh-CN/clawhub/acceptable-usage) 的内容

在技能页面上使用 **举报技能** 按钮，或对软件包使用软件包举报命令/API。

不要将 ClawHub 举报用于第三方技能或插件自身源代码中的漏洞。请直接向列表中链接的发布者或源码仓库报告这些问题。ClawHub 不维护或修补第三方技能或插件代码。

`openclaw/clawhub` 的 GitHub Security Advisories 用于 ClawHub 本身的漏洞。示例包括网站、API、CLI、注册表、身份验证、扫描、审核或下载/安装信任边界中的错误。不要将 ClawHub advisories 用于第三方技能或插件中的漏洞。

好的举报应具体且可执行。滥用举报本身也可能导致账户操作。

## 组织和命名空间声明

组织、品牌、软件包作用域、所有者用户名或命名空间所有权争议应使用 [组织和命名空间声明](/zh-CN/clawhub/namespace-claims) 流程，而不是产品内举报流程或账户申诉表单。

当你需要 ClawHub 工作人员审查非敏感证明，以确认某个命名空间应被保留、转让、重命名、隐藏、隔离、设置别名或以其他方式审查时，请使用该流程。不要在公开 issue 中包含密钥、私人文档、私人法律文件、个人身份证明文件、API 令牌或 DNS 质询令牌。

## 审核暂停

某些严重发现或政策问题可能会使发布者或列表进入审核暂停状态。发生这种情况时，受影响的内容可能会从公开发现中隐藏，或后续发布可能会在问题得到审查前默认隐藏。

审核暂停旨在 ClawHub 解决高风险案例期间保护用户。当确认是误报时，也可以解除审核暂停。

## 隐藏或阻止的列表

列表可能会被暂停、隐藏、隔离、撤销，或以其他方式在公开安装界面上不可用。

如果你看到这些状态之一，除非所有者解决了问题或审核恢复了该版本，否则不要安装该版本。

所有者可能仍会看到自己被暂停或隐藏列表的诊断信息。这些诊断信息有助于解释发生了什么，以及列表返回公开界面前需要做出哪些更改。

## 封禁和账户状态

违反 ClawHub 政策的账户可能会失去发布权限。严重滥用可能导致账户封禁、令牌吊销、内容隐藏或列表移除。发布者滥用压力信号会每日检查。达到 ClawHub 潜在封禁阈值的信号可能触发自动警告。如果警告截止时间之后的下一次符合条件的扫描仍然使发布者处于潜在封禁阈值内，ClawHub 可能会自动应用账户操作。置信度较低且有时间边界的审查信号不会进入自动执行。

已删除、被封禁或被禁用的账户无法使用 ClawHub API 令牌。如果 CLI 身份验证在账户操作后开始失败，请登录 Web UI 查看账户状态。如果封禁或禁用账户阻止了登录或正常 CLI 访问，请使用 [ClawHub 申诉表单](https://appeals.openclaw.ai/) 进行恢复审查。

如果扫描器触发的电子邮件将某个技能或插件版本标记为恶意，请下载被阻止提交版本的已存储扫描结果：
`clawhub scan download <slug> --version <version>`。对于插件，添加
`--kind plugin`。审查扫描输出，修复列表，递增版本号，然后上传修复后的版本。

## 发布者指南

为了减少误报并提升用户信任：

- 保持名称、摘要、标签和变更日志准确
- 声明所需的环境变量和权限
- 避免使用混淆的安装命令
- 尽可能链接到源码
- 发布插件前使用 dry runs
- 如果用户或审核人员询问发布行为，请清晰回应
