# 审核与账号安全

ClawHub 对发布开放，但公开发现和安装界面仍需设置防护措施。当某个版本或账号看起来不安全、具有误导性或违反政策时，举报、审核限制、隐藏列表和账号处置有助于保护用户。

本页介绍审核和账号状态。有关 `Pass`、`Review`、`Warn`、`Malicious` 等审计标签及风险级别，请参阅[安全审计](/clawhub/security-audits)。

另请参阅[安全](/clawhub/security)和[可接受使用政策](/clawhub/acceptable-usage)。对于版权或其他内容权利问题，请使用[内容权利请求](/clawhub/content-rights)。

## 举报

已登录用户可以举报技能、插件和软件包。

ClawHub 举报仅适用于不安全的市场内容，例如：

- 恶意列表
- 误导性元数据
- 未声明的凭据或权限要求
- 可疑的安装说明
- 冒充他人
- 恶意抢注或商标滥用
- 违反[可接受使用政策](/clawhub/acceptable-usage)的内容

在技能页面上使用 **举报技能** 按钮，或使用软件包举报命令/API 举报软件包。

请勿使用 ClawHub 举报功能报告第三方技能或插件自身源代码中的漏洞。请直接向发布者或列表中链接的源代码仓库报告。ClawHub 不维护或修补第三方技能或插件代码。

`openclaw/clawhub` 的 GitHub Security Advisories 用于报告 ClawHub 本身的漏洞。例如网站、API、CLI、注册表、身份验证、扫描、审核或下载/安装信任边界中的缺陷。请勿使用 ClawHub 安全公告报告第三方技能或插件中的漏洞。

高质量的举报应具体且可执行。滥用举报功能本身也可能导致账号处置。

## 组织和命名空间声明

组织、品牌、软件包作用域、所有者用户名或命名空间所有权争议应使用[组织和命名空间声明](/clawhub/namespace-claims)流程，而不是产品内举报流程或账号申诉表单。

当你需要 ClawHub 工作人员审核非敏感证据，以确定某个命名空间是否应被保留、转让、重命名、隐藏、隔离、设置别名或接受其他审核时，请使用该流程。请勿在公开议题中包含密钥、私人文档、非公开法律文件、个人身份证明文件、API 令牌或 DNS 质询令牌。

## 审核限制

某些严重发现或政策问题可能导致发布者或列表受到审核限制。发生这种情况时，受影响的内容可能会从公开发现界面中隐藏，或者后续发布的版本可能会默认隐藏，直至问题完成审核。

审核限制旨在 ClawHub 处理高风险事件期间保护用户。确认属于误报后，也可以解除限制。

## 隐藏或屏蔽的列表

列表可能被暂缓、隐藏、隔离、撤销，或因其他原因无法通过公开安装界面获取。

如果你看到这些状态之一，请勿安装该版本，除非所有者解决了问题或审核人员恢复了该版本。

所有者可能仍能查看自己受限或隐藏列表的诊断信息。这些诊断信息有助于解释发生了什么，以及列表重新出现在公开界面之前需要进行哪些更改。

## 封禁和账号状态

违反 ClawHub 政策的账号可能会失去发布权限。严重滥用可能导致账号封禁、令牌撤销、内容隐藏或列表移除。系统每日检查发布者滥用压力信号。达到 ClawHub 潜在封禁阈值的信号可能触发自动警告。如果警告截止时间后的下一次符合条件的扫描仍将发布者判定为达到潜在封禁阈值，ClawHub 可能会自动执行账号处置。置信度较低且时间范围有限的审核信号不会用于自动执行。

已删除、被封禁或已停用的账号无法使用 ClawHub API 令牌。如果 CLI 身份验证在账号处置后开始失败，请登录 Web UI 查看账号状态。如果因账号被封禁或停用而无法登录或正常使用 CLI，请使用 [ClawHub 申诉表单](https://appeals.openclaw.ai/)申请恢复审核。

如果由扫描器触发的电子邮件将某个技能或插件版本标记为恶意，请下载被屏蔽提交版本的已存储扫描结果：`clawhub scan download <slug> --version <version>`。对于插件，请添加 `--kind plugin`。检查扫描输出，修复列表，递增版本号，然后上传修复后的版本。

## 发布者指南

为减少误报并提升用户信任：

- 确保名称、摘要、标签和变更日志准确
- 声明所需的环境变量和权限
- 避免使用经过混淆的安装命令
- 尽可能链接到源代码
- 发布插件前使用试运行
- 如果用户或审核人员询问版本行为，请作出清晰回应
