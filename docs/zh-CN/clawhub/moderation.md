# 内容审核与账号安全

ClawHub 开放发布，但公共发现和安装界面仍然需要防护措施。当某个版本或账号看起来不安全、具有误导性或违反政策时，报告、审核保留、隐藏列表和账号处置有助于保护用户。

本页介绍内容审核和账号状态。对于 `Pass`、`Review`、`Warn`、`Malicious` 和风险等级等审计标签，请参阅 [安全审计](/zh-CN/clawhub/security-audits)。

另请参阅 [安全](/zh-CN/clawhub/security) 和 [可接受使用](/zh-CN/clawhub/acceptable-usage)。对于版权或其他内容权利相关问题，请使用 [内容权利请求](/zh-CN/clawhub/content-rights)。

## 报告

已登录用户可以报告技能、插件和软件包。

仅将 ClawHub 报告用于不安全的市场内容，例如：

- 恶意列表
- 具有误导性的元数据
- 未声明的凭据或权限要求
- 可疑的安装说明
- 冒充
- 恶意注册或商标滥用
- 违反 [可接受使用](/zh-CN/clawhub/acceptable-usage) 的内容

在技能页面上使用 **报告技能** 按钮，或对软件包使用软件包报告命令/API。

不要使用 ClawHub 报告来报告第三方技能或插件自身源代码中的漏洞。请直接向列表中链接的发布者或源码仓库报告。ClawHub 不维护或修补第三方技能或插件代码。

`openclaw/clawhub` 的 GitHub 安全公告用于 ClawHub 本身的漏洞。示例包括网站、API、CLI、注册表、身份验证、扫描、内容审核或下载/安装信任边界中的缺陷。不要将 ClawHub 公告用于第三方技能或插件中的漏洞。

好的报告应具体且可执行。滥用报告本身也可能导致账号处置。

## 组织和命名空间认领

组织、品牌、软件包作用域、所有者用户名或命名空间所有权争议应使用 [组织和命名空间认领](/zh-CN/clawhub/namespace-claims) 流程，而不是产品内报告流程或账号申诉表单。

当你需要 ClawHub 工作人员审查非敏感证明，以判断某个命名空间是否应被保留、转让、重命名、隐藏、隔离、别名化或以其他方式审查时，请使用该流程。不要在公开 issue 中包含密钥、私人文档、私人法律文件、个人身份证明文件、API 令牌或 DNS challenge 令牌。

## 审核保留

某些严重发现或政策问题可能会使发布者或列表进入审核保留状态。发生这种情况时，受影响内容可能会从公共发现中隐藏，或者未来发布可能会先以隐藏状态开始，直到问题完成审查。

审核保留旨在保护用户，同时 ClawHub 解决高风险案例。确认误报后，也可以解除审核保留。

## 隐藏或封锁的列表

列表可能会被保留、隐藏、隔离、撤销，或以其他方式在公共安装界面上不可用。

如果你看到这些状态之一，除非所有者解决问题或审核恢复该版本，否则不要安装该版本。

所有者仍然可以看到自己被保留或隐藏列表的诊断信息。这些诊断信息有助于解释发生了什么，以及列表返回公共界面前需要更改什么。

## 封禁与账号状态

违反 ClawHub 政策的账号可能会失去发布权限。严重滥用可能导致账号封禁、令牌撤销、内容隐藏或列表移除。发布者滥用风险信号每天都会检查。达到 ClawHub 潜在封禁阈值的信号可能会触发自动警告。如果警告截止时间后的下一次合格扫描仍将发布者置于潜在封禁阈值内，ClawHub 可能会自动应用账号处置。较低置信度和有时间范围限制的审查信号不会进入自动执行。

已删除、被封禁或被禁用的账号无法使用 ClawHub API 令牌。如果账号处置后 CLI 身份验证开始失败，请登录 Web 界面查看账号状态。如果封禁或账号禁用阻止登录或正常 CLI 访问，请使用 [ClawHub 申诉表单](https://appeals.openclaw.ai/) 进行恢复审查。

如果扫描器触发的电子邮件将某个技能或插件版本标记为恶意，请下载被封锁提交版本的已存储扫描结果：`clawhub scan download <slug> --version <version>`。对于插件，请添加 `--kind plugin`。查看扫描输出，修复列表，递增版本号，然后上传修复后的版本。

## 发布者指南

为减少误报并提升用户信任：

- 保持名称、摘要、标签和变更日志准确
- 声明所需的环境变量和权限
- 避免混淆的安装命令
- 尽可能链接到源码
- 发布插件前使用试运行
- 如果用户或审核人员询问发布行为，请清晰回应
