# 审核与账户安全

ClawHub 对发布开放，但公共发现和安装入口仍然需要防护措施。报告、审核冻结、隐藏列表和账户操作有助于在某个发布或账户看起来不安全、具有误导性或违反政策时保护用户。

本页介绍审核和账户状态。对于 `Pass`、`Review`、`Warn`、`Malicious` 等审计标签以及风险级别，请参阅 [安全审计](/zh-CN/clawhub/security-audits)。

另请参阅 [安全](/zh-CN/clawhub/security) 和 [可接受使用](/zh-CN/clawhub/acceptable-usage)。对于版权或其他内容权利问题，请使用 [内容权利请求](/zh-CN/clawhub/content-rights)。

## 报告

已登录用户可以报告技能、插件和包。

仅将 ClawHub 报告用于不安全的市场内容，例如：

- 恶意列表
- 误导性元数据
- 未声明的凭证或权限要求
- 可疑的安装说明
- 冒充
- 恶意注册或商标滥用
- 违反 [可接受使用](/zh-CN/clawhub/acceptable-usage) 的内容

在技能页面上使用 **报告技能** 按钮，或对包使用包报告命令/API。

不要使用 ClawHub 报告来报告第三方技能或插件自身源代码中的漏洞。请直接向列表中链接的发布者或源代码仓库报告这些问题。ClawHub 不维护或修补第三方技能或插件代码。

`openclaw/clawhub` 的 GitHub Security Advisories 用于 ClawHub 本身的漏洞。示例包括网站、API、CLI、注册表、身份验证、扫描、审核或下载/安装信任边界中的错误。不要使用 ClawHub advisories 来报告第三方技能或插件中的漏洞。

好的报告应当具体且可执行。滥用报告机制本身也可能导致账户操作。

## 组织和命名空间主张

组织、品牌、包作用域、所有者句柄或命名空间所有权争议应使用 [组织和命名空间主张](/zh-CN/clawhub/namespace-claims) 流程，而不是产品内报告流程或账户申诉表单。

当你需要 ClawHub 工作人员审核非敏感证明，以判断某个命名空间是否应被保留、转让、重命名、隐藏、隔离、设置别名或以其他方式审核时，请使用该流程。不要在公开 issue 中包含密钥、私人文件、私人法律文件、个人身份证明文件、API 令牌或 DNS challenge 令牌。

## 审核冻结

一些严重发现或政策问题可能会使发布者或列表进入审核冻结状态。发生这种情况时，受影响的内容可能会从公共发现中隐藏，或者未来发布可能会在问题审核完成前默认隐藏。

审核冻结旨在 ClawHub 解决高风险案例期间保护用户。当确认是误报时，也可以解除冻结。

## 隐藏或阻止的列表

列表可能会被冻结、隐藏、隔离、撤销，或以其他方式在公共安装入口上不可用。

如果你看到这些状态之一，除非所有者解决问题或审核恢复该版本，否则不要安装该发布。

所有者仍然可以看到自己被冻结或隐藏列表的诊断信息。这些诊断信息有助于说明发生了什么，以及列表返回公共入口前需要做出哪些更改。

## 封禁和账户状态

违反 ClawHub 政策的账户可能会失去发布权限。严重滥用可能导致账户封禁、令牌撤销、内容隐藏或列表移除。发布者滥用压力信号会每日检查。达到 ClawHub 潜在封禁阈值的信号可能会触发自动警告。如果警告截止时间之后的下一次符合条件的扫描仍然将发布者置于潜在封禁阈值内，ClawHub 可能会自动应用账户操作。置信度较低以及有时间范围限制的审核信号不会进入自动执行。

已删除、被封禁或被禁用的账户无法使用 ClawHub API 令牌。如果 CLI 身份验证在账户操作后开始失败，请登录 Web UI 查看账户状态。如果封禁或禁用账户阻止了登录或正常 CLI 访问，请使用 [ClawHub 申诉表单](https://appeals.openclaw.ai/) 进行恢复审核。

如果扫描器触发的邮件将某个技能或插件版本标记为恶意，请下载被阻止提交版本的已存储扫描结果：`clawhub scan download <slug> --version <version>`。对于插件，添加 `--kind plugin`。查看扫描输出，修复列表，递增版本号，然后上传修复后的版本。

## 发布者指南

为了减少误报并提升用户信任：

- 保持名称、摘要、标签和变更日志准确
- 声明必需的环境变量和权限
- 避免混淆的安装命令
- 尽可能链接到源代码
- 发布插件前使用 dry run
- 如果用户或审核员询问发布行为，请清晰回应
