# 语音唤醒与按住说话

## 模式

- **唤醒词模式**（默认）：始终开启的 Speech 识别器会等待触发词（`swabbleTriggerWords`）。匹配后，它会开始采集，显示带有部分文本的悬浮层，并在静音后自动发送。
- **按住说话（按住右 Option 键）**：按住右侧 Option 键可立即开始采集——无需触发词。按住期间会显示悬浮层；松开后会完成并转发，期间有一个短暂延迟，以便你微调文本。

## 运行时行为（唤醒词）

- Speech 识别器位于 `VoiceWakeRuntime` 中。
- 只有当唤醒词与下一个词之间存在**明显停顿**时，触发才会生效（约 `0.55s` 间隔）。悬浮层/提示音甚至可以在命令开始前的停顿时启动。
- 静音窗口：当语音持续流动时为 2.0 秒；如果只听到了触发词，则为 5.0 秒。
- 强制停止：120 秒，用于防止会话失控。
- 会话间去抖：350 毫秒。
- 悬浮层由 `VoiceWakeOverlayController` 驱动，并使用已提交/暂态着色。
- 发送后，识别器会干净地重新启动，以监听下一个触发。

## 生命周期不变量

- 如果语音唤醒已启用且权限已授予，那么唤醒词识别器应当始终处于监听状态（明确的按住说话采集期间除外）。
- 悬浮层可见性（包括通过 X 按钮手动关闭）绝不能阻止识别器恢复。

## 粘住的悬浮层故障模式（之前）

此前，如果悬浮层卡在可见状态，而你又手动将其关闭，语音唤醒可能会看起来“失效”，因为运行时的重启尝试可能会被悬浮层可见性阻止，而且之后也不会安排新的重启。

加固措施：

- 唤醒运行时重启不再受悬浮层可见性的阻碍。
- 悬浮层关闭完成后，会通过 `VoiceSessionCoordinator` 触发一次 `VoiceWakeRuntime.refresh(...)`，因此手动点击 X 关闭后总会恢复监听。

## 按住说话细节

- 热键检测使用针对**右侧 Option** 的全局 `.flagsChanged` 监视器（`keyCode 61` + `.option`）。我们只观察事件（不会吞掉事件）。
- 采集管线位于 `VoicePushToTalk`：会立即启动 Speech，将部分结果流式传给悬浮层，并在松开时调用 `VoiceWakeForwarder`。
- 当按住说话开始时，我们会暂停唤醒词运行时，以避免音频 tap 相互冲突；松开后会自动重启。
- 权限：需要麦克风 + Speech；要看到事件则需要辅助功能/Input Monitoring 授权。
- 外接键盘：某些设备可能不会按预期暴露右侧 Option——如果用户反馈漏检，请提供后备快捷键。

## 面向用户的设置

- **语音唤醒** 开关：启用唤醒词运行时。
- **按住 Cmd+Fn 说话**：启用按住说话监视器。在 macOS < 26 上禁用。
- 语言和麦克风选择器、实时电平表、触发词表、测试器（仅本地；不会转发）。
- 麦克风选择器会在设备断开时保留上次选择，显示断开提示，并在设备恢复前暂时回退到系统默认设备。
- **声音**：检测到触发词时和发送时会播放提示音；默认使用 macOS 的 “Glass” 系统声音。你可以为每个事件选择任意 `NSSound` 可加载文件（例如 MP3/WAV/AIFF），或选择 **No Sound**。

## 转发行为

- 启用语音唤醒后，转录内容会被转发到当前活动的 gateway 网关/智能体（使用与 mac 应用其余部分相同的本地或远程模式）。
- 回复会投递到**最近使用的主提供商**（WhatsApp/Telegram/Discord/WebChat）。如果投递失败，会记录错误日志，并且该次运行仍可通过 WebChat/会话日志查看。

## 转发负载

- `VoiceWakeForwarder.prefixedTranscript(_:)` 会在发送前为转录内容添加机器提示前缀。唤醒词和按住说话路径共用这一逻辑。

## 快速验证

- 打开按住说话，按住 Cmd+Fn，说话，然后松开：悬浮层应显示部分结果，然后发送。
- 按住期间，菜单栏耳朵图标应保持放大状态（使用 `triggerVoiceEars(ttl:nil)`）；松开后恢复。
