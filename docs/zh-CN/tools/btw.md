`/btw`（别名 `/side`）用于针对**当前会话**快速提出一个旁支问题，而不会将其添加到对话历史记录中。其设计参考了 Claude Code 的 `/btw`，并针对 OpenClaw 的 Gateway 网关和多渠道架构进行了调整。

```text
/btw 发生了什么变化？
/side 这个错误是什么意思？
```

## 功能说明

1. 将当前会话快照作为背景上下文（包括任何正在进行的主运行提示词）。
2. 运行一个独立的一次性旁支查询，指示模型仅回答旁支问题，不继续或引导主任务。
3. 将答案作为实时旁支结果交付，而不是普通的助手消息。
4. 绝不将问题或答案写入会话历史记录或 `chat.history`。

如果主运行处于活动状态，它将不受任何影响。

对于 Codex harness 会话，BTW 会将活动 Codex app-server 线程派生为一个临时子线程，而不是单独调用提供商。这样可以保持 Codex OAuth 以及原生工具和线程行为不变，派生线程也会保留父线程当前的审批策略、沙箱和原生工具表面。派生线程会收到一个边界提示词，告知模型边界之前的所有内容都是继承的参考上下文，而不是活动指令，只有边界之后的消息才是实时消息。`/btw` 要求已有 Codex 线程；请先发送一条普通消息。

对于 CLI 运行时别名，BTW 会以一次性旁支问题模式调用其所属的 CLI 后端：它会将经过清理的对话上下文植入一次全新的 CLI 调用，禁用工具捆绑和可复用会话状态，并添加后端支持的任何禁止继续和禁止工具标志。直接（非 CLI）运行时则改用直接的一次性提供商调用。

## 不会执行的操作

`/btw` 不会创建持久会话、继续未完成的主任务、将问题/答案数据持久化到对话历史记录，也无法在重新加载后保留。

## 交付模型

普通助手聊天使用 Gateway 网关的 `chat` 事件。BTW 使用独立的 `chat.side_result` 事件，因此客户端不会将其误认为常规对话历史记录。由于它不会通过 `chat.history` 重放，因此重新加载后会消失。

## 各界面行为

| 界面              | 行为                                                                                                                                                                                                                                                                                |
| ----------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| TUI               | 在聊天日志中内联呈现，与普通回复有明显区别，可按 `Enter` 或 `Esc` 将其关闭。                                                                                                                                                                                                         |
| 外部渠道          | 作为带有明确标签的一次性回复交付（Telegram、WhatsApp、Discord 没有本地临时浮层）。                                                                                                                                                                                                   |
| Control UI / Web  | 呈现为固定在线程上的浮动“旁支聊天”面板。答案以轮次形式累积，“继续提问”输入框用于提出下一个旁支问题。关闭（按 `Esc` 或点击 X）会保留对话，并在下一个答案到达时重新打开；垃圾桶按钮会丢弃该对话并停止待处理的运行。 |

## 文本选择弹出菜单（Control UI）

在 Control UI 的聊天消息中高亮文本时，会打开一个包含两个操作的小型文本选择弹出菜单：

- **更多详情**会立即发送一个隐式 `/btw` 问题，要求模型结合当前会话的上下文解释高亮文本。答案会显示在浮动旁支聊天面板中。
- **在旁支聊天中提问**会在编辑器中预填一个引用高亮文本的 `/btw` 草稿，供你输入自己的相关问题。

这两个操作都遵循正常的 `/btw` 语义：问题和答案不会进入会话历史记录，主运行也不会受到影响。

## 使用场景

当你需要快速澄清、在长时间运行仍在进行时获取一个事实性旁支答案，或获取一个不应进入未来会话上下文的临时答案时，可以使用 `/btw`。

```text
/btw 我们正在编辑哪个文件？
/btw 用一句话总结当前任务
/btw 17 * 19 等于多少？
```

对于任何你希望纳入会话未来工作上下文的内容，请改为在主会话中正常提问。
