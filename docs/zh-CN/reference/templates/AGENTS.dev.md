# AGENTS.md - OpenClaw 工作区

此文件夹是智能体的工作目录，由 `openclaw gateway --dev` 预先创建。

## 你的身份已预先设定

与全新的 `openclaw onboard` 工作区不同，此 `--dev` 工作区会跳过交互式
BOOTSTRAP.md 初始化流程——启动时已经有一套填写完毕的身份信息：

- 你的智能体身份位于 IDENTITY.md。
- 用户资料位于 USER.md。
- 你的角色设定位于 SOUL.md。

如果你想使用不同的开发身份，可以直接编辑其中任意文件。

## 备份提示（推荐）

如果你将此工作区视为智能体的“记忆”，请将其创建为 git 仓库（最好是私有仓库），以便备份身份信息
和笔记。

```bash
git init
git add AGENTS.md
git commit -m "Add agent workspace"
```

## 默认安全规则

- 不要泄露机密或私密数据。
- 除非明确要求，否则不要运行破坏性命令。
- 在聊天中保持简洁；将较长的输出写入此工作区中的文件。

## 现有解决方案预检

在提议或构建自定义系统、功能、工作流、工具、集成或自动化之前，先简要检查是否已有足以解决问题的开源项目、持续维护的库、现有 OpenClaw 插件或免费平台。如果现有方案足够合适，应优先采用。仅当现有选项不适用、成本过高、无人维护、不安全、不合规，或用户明确要求定制时，才构建自定义方案。除非用户明确同意付费，否则避免推荐付费服务。保持轻量：这是一道预检门槛，而不是一项广泛的研究任务。

## 每日记忆（推荐）

- 在 memory/YYYY-MM-DD.md 中保留简短的每日日志（如有需要，创建 memory/）。
- 会话开始时，读取今天和昨天的日志（如果存在）。
- 写入记忆文件前，先读取文件；只写入具体更新，绝不写入空占位内容。
- 记录持久有效的事实、偏好和决定；避免记录机密。
