# 目標

**目標**は、現在の OpenClaw セッションに紐づく永続的な目的を 1 つ表します。
長時間の作業について、エージェントとオペレーターが共有する対象を提供しますが、
その対象をバックグラウンドタスク、リマインダー、cron ジョブ、または
常設指示にはしません。

目標はセッション状態です。セッションキーとともに移動し、プロセスの再起動後も残り、
`/goal`、モデル向けの目標ツール、TUI フッターに表示されます。

## クイックスタート

```text
/goal start get CI green for PR 87469 and push the fix
/goal
/goal pause waiting for CI
/goal resume
/goal complete pushed and verified
/goal clear
```

`start` は任意です。`/goal get CI green for PR 87469` でも目標を作成します。
これは、既知のアクション語ではない `/goal` の後の任意のテキストが、
新しい目的として扱われるためです。

## 目標の用途

セッションに、多くのターンにわたって表示し続けるべき具体的な成果がある場合に目標を使います。

- PR の完了処理: 修正、検証、autoreview、push、PR の作成または更新。
- デバッグ実行: バグを再現し、所有するサーフェスを特定し、パッチを当て、
  修正を証明する。
- ドキュメント作業: 関連ドキュメントを読み、新しいページを書き、相互リンクし、
  ドキュメントビルドを検証する。
- メンテナンスタスク: 現在の状態を調査し、範囲を限定した変更を行い、
  適切なチェックを実行し、変更内容を報告する。

目標はタスクキューではありません。作業を切り離して実行する、スケジュールに従って繰り返す、
管理されたサブ作業に展開する、またはポリシーとして永続化する必要がある場合は、
[タスクフロー](/ja-JP/automation/taskflow)、[タスク](/ja-JP/automation/tasks)、
[cron ジョブ](/ja-JP/automation/cron-jobs)、または
[常設指示](/ja-JP/automation/standing-orders)を使います。

## コマンドリファレンス

引数なしの `/goal` は現在の目標サマリーを出力します。

```text
Goal
Status: active
Objective: get CI green for PR 87469 and push the fix
Tokens used: 12k
Token budget: 12k/50k

Commands: /goal pause, /goal complete, /goal clear
```

| コマンド                                            | 効果                                                                   |
| --------------------------------------------------- | ---------------------------------------------------------------------- |
| `/goal` または `/goal status`                       | 現在の目標を表示します。                                               |
| `/goal start <objective>`                           | 現在のセッションに新しい目標を作成します。                             |
| `/goal set <objective>`, `/goal create <objective>` | `start` のエイリアスです。                                             |
| `/goal <objective>`                                 | 新しい目標も作成します（認識済みのアクション語ではない任意のテキスト）。 |
| `/goal pause [note]`                                | アクティブな目標を一時停止します。                                     |
| `/goal resume [note]`                               | 一時停止、ブロック、使用量制限、または予算制限状態の目標を再開します。 |
| `/goal complete [note]`                             | 目標を達成済みにします。                                               |
| `/goal done [note]`                                 | `complete` のエイリアスです。                                          |
| `/goal block [note]`                                | 目標をブロック状態にします。                                           |
| `/goal blocked [note]`                              | `block` のエイリアスです。                                             |
| `/goal clear`                                       | セッションから目標を削除します。                                       |

1 つのセッションに存在できる目標は 1 つだけです。2 つ目の目標を開始すると、
現在の目標がクリアされるまで `Goal error: goal already exists` で失敗します。

`/goal start` はトークン予算フラグを取りません。予算はモデル向けの
`create_goal` ツールでのみ設定できます。

## ステータス

- `active`: セッションが目標を追っています。
- `paused`: オペレーターが目標を一時停止しました。`/goal resume` で再びアクティブになります。
- `blocked`: エージェントまたはオペレーターが実際のブロッカーを報告しました。
  新しい情報または状態が利用可能になったら、`/goal resume` で再びアクティブになります。
- `budget_limited`: 設定されたトークン予算に達しました。`/goal resume` により、
  同じ目的から新しい予算ウィンドウで追跡を再開します。
- `usage_limited`: 将来の使用量制限停止状態のために予約されています。`/goal
resume` は同じ方法で追跡を再開します。
- `complete`: 目標が達成されました。完了した目標は終端状態です。別の目標を開始する前に
`/goal clear` を使います。

`/new` と `/reset` は、意図的に新しいセッションコンテキストを開始するため、
現在のセッション目標をクリアします。

## トークン予算

目標には任意の正のトークン予算を設定できます。これは `create_goal` ツールの
`token_budget` パラメーターで設定します。予算は、目標作成時点のセッションの
新鮮なトークン数から測定されます。目標開始時にセッションに古い、または不明な
トークンスナップショットしかない場合、OpenClaw は次の新鮮なスナップショットを待ち、
それをベースラインとして使います。そのため、目標が存在する前に消費されたトークンは
目標に課金されません。

使用量が予算に達すると、目標は `budget_limited` に移行します。これは目標を削除したり、
目的を消去したりするものではありません。再開またはクリアされるまで、その目標が
アクティブには追跡されなくなったことをオペレーターとエージェントに伝えます。
再開すると、現在の新鮮なトークン数から新しい予算ウィンドウが開始されます。

トークン予算はセッション目標のガードレールであり、課金上限ではありません。
プロバイダーのクォータ、コストレポート、コンテキストウィンドウの動作は、
引き続き通常の OpenClaw 使用量およびモデル制御を使います。

## モデルツール

OpenClaw はエージェントハーネスに 3 つの目標ツールを公開します。

| ツール        | 目的                                                                                                                     |
| ------------- | ------------------------------------------------------------------------------------------------------------------------ |
| `get_goal`    | 現在のセッション目標を読み取ります: ステータス、目的、トークン使用量、トークン予算。                                    |
| `create_goal` | ユーザーまたはシステム指示が明示的に要求した場合にのみ目標を作成します。セッションに既に目標がある場合は失敗します。     |
| `update_goal` | 目標を `complete` または `blocked` にします。                                                                            |

モデルは目標を黙って一時停止、再開、クリア、または置換できません。それらは `/goal` と
リセットコマンドを通じたオペレーター/セッション制御として残ります。これにより、エージェントは
対象を密かに動かすことなく、達成または本物のブロッカーを報告できます。

`update_goal` は、目的が実際に達成された場合にのみ目標を `complete` にするべきです。
通常の難しさや仕上げ不足ではなく、同じブロック条件が少なくとも 3 回連続する目標ターンで
再発した後にのみ、目標を `blocked` にするべきです。

## TUI

TUI フッターは、アクティブなセッションの目標を、トークン/モードインジケーターの前で、
エージェント、セッション、モデルのフィールドの横に表示し続けます。

フッターの例:

- トークン予算付きのアクティブな目標では `Pursuing goal (12k/50k)`。
- 一時停止中の目標では `Goal paused (/goal resume)`。
- ブロック中の目標では `Goal blocked (/goal resume)`。
- 使用量制限中の目標では `Goal hit usage limits (/goal resume)`。
- 予算制限中の目標では `Goal unmet (50k/50k)`。
- 完了した目標では `Goal achieved (42k)`。

フッターは意図的にコンパクトです。完全な目的、メモ、トークン予算、利用可能なコマンドには
`/goal` を使います。

## チャネルの動作

`/goal` は、TUI やテキストコマンドを許可するチャットサーフェスを含む、
コマンド対応の OpenClaw セッションで機能します。目標状態はトランスポートではなく
セッションキーに紐づくため、セッションキーを共有する 2 つのサーフェスでは同じ目標が見えます。

目標状態は配信ディレクティブではありません。チャネル経由の返信を強制したり、
キューの動作を変えたり、ツールを承認したり、作業をスケジュールしたりしません。

## トラブルシューティング

| メッセージ                           | 意味                                                                                                                                           |
| ------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| `Goal error: goal already exists`    | セッションには既に目標があります。`/goal` で確認し、完了している場合は `/goal complete`、別の目的を開始する前には `/goal clear` を使います。 |
| `Goal error: goal not found`         | セッションにはまだ目標がありません。`/goal start <objective>` で開始します。                                                                   |
| `Goal error: goal is already complete` | 目標は終端状態です。別の目的を開始または再開する前にクリアします。                                                                             |

トークン使用量が `0` と表示される、または古く見える場合、アクティブなセッションにはまだ
新鮮なトークンスナップショットがない可能性があります。OpenClaw がセッション使用量と
トランスクリプト由来の合計を記録するにつれて、使用量は更新されます。
