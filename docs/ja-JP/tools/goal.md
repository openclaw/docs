# 目標

**目標**は、現在の OpenClaw セッションに関連付けられた、永続的な目的を1つ表します。
これは、長時間の作業に対してエージェントとオペレーターが共有するターゲットを与えますが、
そのターゲットをバックグラウンドタスク、リマインダー、Cron ジョブ、または
常駐指示にはしません。

目標はセッション状態です。セッションキーとともに移動し、プロセスの再起動後も保持され、
`/goal`、モデル向けの目標ツール、TUI フッターに表示されます。

## クイックスタート

```text
/goal start get CI green for PR 87469 and push the fix
/goal
/goal edit get CI green for PR 87469, push the fix, and update docs
/goal pause waiting for CI
/goal resume
/goal complete pushed and verified
/goal clear
```

`start` は省略可能です。`/goal get CI green for PR 87469` でも目標を作成します。
これは、既知のアクション語ではない `/goal` の後の任意のテキストが、
新しい目的として扱われるためです。

## 目標の用途

セッションに、多くのターンをまたいで表示され続けるべき具体的な成果がある場合に、
目標を使用します。

- PR の完了処理: 修正、検証、autoreview、push、PR の作成または更新。
- デバッグ実行: バグの再現、所有するサーフェスの特定、パッチ適用、
  修正の証明。
- ドキュメント対応: 関連ドキュメントの確認、新しいページの作成、相互リンク、
  ドキュメントビルドの検証。
- メンテナンスタスク: 現在の状態の確認、範囲を限定した変更、
  適切なチェックの実行、変更内容の報告。

目標はタスクキューではありません。作業を切り離して実行する、スケジュールで繰り返す、
管理されたサブ作業へ分岐する、またはポリシーとして永続化する必要がある場合は、
[タスクフロー](/ja-JP/automation/taskflow)、[タスク](/ja-JP/automation/tasks)、
[Cron ジョブ](/ja-JP/automation/cron-jobs)、または
[常駐指示](/ja-JP/automation/standing-orders)を使用します。

## コマンドリファレンス

引数なしの `/goal` は、現在の目標の概要を表示します。

```text
Goal
Status: active
Objective: get CI green for PR 87469 and push the fix
Tokens used: 12k
Token budget: 12k/50k

Commands: /goal edit <objective>, /goal pause, /goal complete, /goal clear
```

| コマンド                                            | 効果                                                                     |
| --------------------------------------------------- | ------------------------------------------------------------------------ |
| `/goal` または `/goal status`                       | 現在の目標を表示します。                                                 |
| `/goal start <objective>`                           | 現在のセッションに新しい目標を作成します。                               |
| `/goal set <objective>`, `/goal create <objective>` | `start` のエイリアスです。                                               |
| `/goal <objective>`                                 | 新しい目標も作成します（認識済みのアクション語ではない任意のテキスト）。 |
| `/goal edit <objective>`                            | 現在の目的を言い換えます。ステータスとトークン計上は維持されます。       |
| `/goal pause [note]`                                | アクティブな目標を一時停止します。                                       |
| `/goal resume [note]`                               | 一時停止、ブロック、使用量制限、または予算制限状態の目標を再開します。   |
| `/goal complete [note]`                             | 目標を達成済みにします。                                                 |
| `/goal done [note]`                                 | `complete` のエイリアスです。                                            |
| `/goal block [note]`                                | 目標をブロック状態にします。                                             |
| `/goal blocked [note]`                              | `block` のエイリアスです。                                               |
| `/goal clear`                                       | セッションから目標を削除します。                                         |

1つのセッションに存在できる目標は1つだけです。2つ目の目標を開始しようとすると、
現在の目標がクリアされるまで `Goal error: goal already exists` で失敗します。

`/goal start` はトークン予算フラグを受け取りません。予算は、モデル向けの
`create_goal` ツールを通じてのみ設定できます。

## ステータス

- `active`: セッションが目標を進めています。
- `paused`: オペレーターが目標を一時停止しました。`/goal resume` で再び
  アクティブになります。
- `blocked`: エージェントまたはオペレーターが実際のブロッカーを報告しました。
  新しい情報または状態が利用可能になったとき、`/goal resume` で再び
  アクティブになります。
- `budget_limited`: 設定されたトークン予算に達しました。`/goal resume` は、
  同じ目的から新しい予算ウィンドウで追跡を再開します。
- `usage_limited`: 将来の使用量制限停止状態のために予約されています。`/goal
resume` は同じ方法で追跡を再開します。
- `complete`: 目標が達成されました。完了済みの目標は終端状態です。別の目標を
  開始する前に `/goal clear` を使用してください。

`/new` と `/reset` は現在のセッション目標をクリアします。これらは意図的に
新しいセッションコンテキストを開始するためです。

## トークン予算

目標には、任意の正のトークン予算を設定できます。これは `create_goal` ツールの
`token_budget` パラメーターで設定します。予算は、目標作成時点のセッションの
新鮮なトークン数を基準に測定されます。目標開始時に、セッションに古い、または
不明なトークンスナップショットしかない場合、OpenClaw は次の新鮮なスナップショットを待ち、
それをベースラインとして使用します。そのため、目標が存在する前に消費されたトークンは
目標に計上されません。

使用量が予算に達すると、目標は `budget_limited` に移行します。これは目標を削除したり
目的を消去したりするものではありません。再開またはクリアされるまで、目標がアクティブに
追跡されなくなったことを、オペレーターとエージェントに伝えます。再開すると、現在の
新鮮なトークン数から新しい予算ウィンドウが開始されます。

トークン予算は、セッション目標のガードレールであり、課金上限ではありません。プロバイダーの
クォータ、コストレポート、コンテキストウィンドウの挙動は、通常の OpenClaw の使用量と
モデル制御を引き続き使用します。

## モデルツール

OpenClaw は、エージェントハーネスに3つの目標ツールを公開します。

| ツール        | 目的                                                                                                                     |
| ------------- | ------------------------------------------------------------------------------------------------------------------------ |
| `get_goal`    | 現在のセッション目標を読み取ります: ステータス、目的、トークン使用量、トークン予算。                                    |
| `create_goal` | ユーザーまたはシステム指示が明示的に要求した場合にのみ、目標を作成します。セッションにすでに目標がある場合は失敗します。 |
| `update_goal` | 目標を `complete` または `blocked` にします。                                                                            |

モデルは、目標を黙って一時停止、再開、クリア、置換することはできません。これらは `/goal` と
リセットコマンドを通じたオペレーター/セッション制御のままです。そのため、エージェントは
ターゲットを密かに動かすことなく、達成や本物のブロッカーを報告できます。

`update_goal` は、目的が実際に達成された場合にのみ目標を `complete` にするべきです。
通常の難しさや仕上げ不足ではなく、同じブロック条件が少なくとも3回連続する目標ターンで
再発した後にのみ、目標を `blocked` にするべきです。

## すべてのターンにおける目標コンテキスト

アクティブな目標があるすべてのユーザー/チャットターンには、このユーザーロールの
コンテキスト行が含まれます。

```text
Active goal: <objective> — advance it or update its status (get_goal/update_goal).
```

OpenClaw は長い目的を切り詰めることで、この行をコンパクトに保ちます。一時停止、
ブロック、予算制限、使用量制限、完了状態の目標は注入されないため、オペレーターの停止は
目標が再開されるまで有効なままです。

## コントロール UI

Web コントロール UI は、チャットコンポーザーの上にコンパクトなピルとして目標を表示します:
ステータスアイコン、ステータスラベル（例: `Pursuing goal`）、切り詰められた目的、
ライブの経過タイマーです。

ピルにはインラインコントロールがあります。

- **鉛筆** は、目的を言い換えて送信できるように、コンポーザーへ `/goal edit <objective>` を
  事前入力します。
- **一時停止 / 再開** は、現在のステータスに基づいて `/goal pause` と `/goal resume` を
  切り替えます。
- **ゴミ箱** は `/goal clear` を送信します。
- **シェブロン** はピルを展開し、完全な目的、最新のステータスノート、トークン使用量、
  経過時間を表示します。

アクションボタンは、コンポーザーが送信できない間（たとえば Gateway 接続がダウンしている場合）は
非表示になります。展開シェブロンは引き続き動作します。

## TUI

TUI フッターは、アクティブなセッションの目標を、エージェント、セッション、モデルのフィールドの
隣、トークン/モードインジケーターの前に表示し続けます。

フッターの例:

- トークン予算があるアクティブな目標の場合は `Pursuing goal (12k/50k)`。
- 一時停止された目標の場合は `Goal paused (/goal resume)`。
- ブロックされた目標の場合は `Goal blocked (/goal resume)`。
- 使用量制限状態の目標の場合は `Goal hit usage limits (/goal resume)`。
- 予算制限状態の目標の場合は `Goal unmet (50k/50k)`。
- 完了済みの目標の場合は `Goal achieved (42k)`。

フッターは意図的にコンパクトです。完全な目的、ノート、トークン予算、利用可能なコマンドには
`/goal` を使用してください。

## チャネルの挙動

`/goal` は、TUI やテキストコマンドを許可するチャットサーフェスなど、
コマンド対応の OpenClaw セッションで動作します。目標状態はトランスポートではなく
セッションキーに関連付けられるため、セッションキーを共有する2つのサーフェスには
同じ目標が表示されます。

目標状態は配信ディレクティブではありません。チャネル経由の返信を強制したり、
キューの挙動を変更したり、ツールを承認したり、作業をスケジュールしたりしません。

## トラブルシューティング

| メッセージ                           | 意味                                                                                                                                             |
| ------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| `Goal error: goal already exists`    | セッションにはすでに目標があります。別の目的を開始する前に、`/goal` で確認するか、完了していれば `/goal complete`、または `/goal clear` を使用します。 |
| `Goal error: goal not found`         | セッションにはまだ目標がありません。`/goal start <objective>` で開始してください。                                                               |
| `Goal error: goal is already complete` | 目標は終端状態です。別の目的を開始または再開する前にクリアしてください。                                                                         |

トークン使用量が `0` と表示される、または古く見える場合、アクティブなセッションにはまだ
新鮮なトークンスナップショットがない可能性があります。OpenClaw がセッション使用量と
トランスクリプト由来の合計を記録するにつれて、使用量は更新されます。
