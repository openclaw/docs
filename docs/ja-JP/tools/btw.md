`/btw` (エイリアス `/side`) は、**現在の
セッション**についての短い横道の質問を、会話履歴に追加せずに行います。これは
Claude Code の `/btw` をモデルにし、OpenClaw の Gateway とマルチチャネル
アーキテクチャに合わせて調整したものです。

```text
/btw what changed?
/side what does this error mean?
```

## 何をするか

1. 現在のセッションをバックグラウンドコンテキストとしてスナップショットします (進行中のメイン実行プロンプトを含む)。
2. 別個の 1 回限りの横道クエリを実行し、モデルに横道の質問だけに回答し、メインタスクを再開したり誘導したりしないよう指示します。
3. 通常のアシスタントメッセージではなく、ライブの横道結果として回答を配信します。
4. 質問や回答をセッション履歴や `chat.history` に書き込むことはありません。

メイン実行がアクティブな場合でも、それには手を加えません。

Codex ハーネスセッションでは、BTW は別個のプロバイダー呼び出しを実行する代わりに、アクティブな Codex app-server スレッドをエフェメラルな子スレッドにフォークします。これにより、Codex OAuth とネイティブのツール/スレッド動作がそのまま保たれ、フォークされたスレッドは親スレッドの現在の承認ポリシー、サンドボックス、ネイティブツールサーフェスを保持します。フォークされたスレッドには、それ以前のすべては継承された参照コンテキストであり、アクティブな指示ではなく、境界の後のメッセージだけがライブであることをモデルに伝える境界プロンプトが渡されます。`/btw` には既存の Codex スレッドが必要です。まず通常のメッセージを送信してください。

CLI ランタイムエイリアスでは、BTW は所有する CLI バックエンドを 1 回限りの横道質問モードで起動します。サニタイズ済みの会話コンテキストを新しい CLI 起動にシードし、ツールバンドリングと再利用可能なセッション状態を無効にして、バックエンドがサポートする no-resume/no-tools フラグを追加します。直接 (非 CLI) ランタイムでは、代わりに直接の 1 回限りのプロバイダー呼び出しを使用します。

## 何をしないか

`/btw` は、永続的なセッションを作成したり、未完了のメインタスクを続行したり、質問/回答データをトランスクリプト履歴に永続化したり、リロード後も残ったりしません。

## 配信モデル

通常のアシスタントチャットは Gateway の `chat` イベントを使用します。BTW は別個の
`chat.side_result` イベントを使用するため、クライアントが通常の会話履歴と誤認することはありません。`chat.history` から再生されないため、リロード後には消えます。

## サーフェス動作

| サーフェス        | 動作                                                                                                                                                    |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| TUI               | チャットログ内にインラインで表示され、通常の返信とは視覚的に区別され、`Enter` または `Esc` で閉じられます。                                          |
| 外部チャネル      | 明確にラベル付けされた 1 回限りの返信として配信されます (Telegram、WhatsApp、Discord には local ephemeral overlay がありません)。                    |
| Control UI / web  | Gateway は `chat.side_result` を正しく発行し、これは `chat.history` から除外されますが、Control UI にはブラウザーでライブ表示するコンシューマーがまだありません。 |

## いつ使うか

短い確認、長い実行がまだ進行中の間の事実確認の横道回答、または将来のセッションコンテキストに入れるべきではない一時的な回答には、`/btw` を使用します。

```text
/btw what file are we editing?
/btw summarize the current task in one sentence
/btw what is 17 * 19?
```

セッションの今後の作業コンテキストの一部にしたい内容は、代わりにメインセッションで通常どおり質問してください。
