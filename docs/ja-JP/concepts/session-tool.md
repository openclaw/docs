# セッションツール

OpenClaw は、エージェントがセッションをまたいで作業し、ステータスを確認し、サブエージェントをオーケストレーションするためのツールを提供します。

## 利用可能なツール

| ツール             | 説明                                                                          |
| ------------------ | ----------------------------------------------------------------------------- |
| `sessions_list`    | セッションを一覧表示します。オプションでフィルター（種類、ラベル、エージェント、最近の活動、プレビュー）を指定できます |
| `sessions_history` | 特定のセッションの transcript を読み取ります                                  |
| `sessions_send`    | 別のセッションにメッセージを送り、必要に応じて待機します                      |
| `sessions_spawn`   | バックグラウンド作業用に分離されたサブエージェントセッションを起動します      |
| `sessions_yield`   | 現在のターンを終了し、後続のサブエージェント結果を待ちます                    |
| `subagents`        | このセッションで起動されたサブエージェントの一覧表示、操作、停止を行います    |
| `session_status`   | `/status` 形式のカードを表示し、必要に応じてセッション単位のモデル上書きを設定します |

## セッションの一覧表示と読み取り

`sessions_list` は、セッションキー、agentId、種類、チャネル、モデル、トークン数、タイムスタンプを含むセッション一覧を返します。種類（`main`、`group`、`cron`、`hook`、`node`）、完全一致の `label`、完全一致の `agentId`、検索テキスト、または最近の活動（`activeMinutes`）でフィルターできます。メールボックスのようなトリアージが必要な場合は、可視性スコープ付きの派生タイトル、最後のメッセージのプレビュー断片、または各行の制限付き最近メッセージも要求できます。派生タイトルとプレビューは、設定されたセッションツールの可視性ポリシーの下で呼び出し元がすでに見えるセッションに対してのみ生成されるため、無関係なセッションは非表示のままです。

`sessions_history` は特定のセッションの会話 transcript を取得します。デフォルトではツール結果は除外されます。表示したい場合は `includeTools: true` を渡してください。返されるビューは意図的に制限され、安全フィルターが適用されています。

- アシスタントテキストはリコール前に正規化されます:
  - thinking タグは削除されます
  - `<relevant-memories>` / `<relevant_memories>` の足場ブロックは削除されます
  - `<tool_call>...</tool_call>`、`<function_call>...</function_call>`、`<tool_calls>...</tool_calls>`、`<function_calls>...</function_calls>` のようなプレーンテキストのツール呼び出し XML ペイロードブロックは削除されます。きれいに閉じていない切り捨てペイロードも含みます
  - `[Tool Call: ...]`、`[Tool Result ...]`、`[Historical context ...]` のような格下げされたツール呼び出し/結果の足場は削除されます
  - `<|assistant|>`、その他の ASCII `<|...|>` トークン、および全角の `<｜...｜>` 変種のような漏洩したモデル制御トークンは削除されます
  - `<invoke ...>` / `</minimax:tool_call>` のような不正な MiniMax ツール呼び出し XML は削除されます
- 資格情報/トークンらしいテキストは返却前に伏せ字化されます
- 長いテキストブロックは切り詰められます
- 非常に大きな履歴では、古い行が落とされるか、巨大すぎる行が `[sessions_history omitted: message too large]` に置き換えられることがあります
- このツールは `truncated`、`droppedMessages`、`contentTruncated`、`contentRedacted`、`bytes` などの要約フラグを報告します

どちらのツールも **session key**（`"main"` のようなもの）または前回の一覧取得から得た **session ID** のどちらでも受け付けます。

完全にバイト単位で一致する transcript が必要な場合は、`sessions_history` を生ダンプとして扱うのではなく、ディスク上の transcript ファイルを確認してください。

## クロスセッションメッセージの送信

`sessions_send` は別のセッションにメッセージを配信し、必要に応じて応答を待ちます。

- **Fire-and-forget:** `timeoutSeconds: 0` を設定すると、キューに入れて即座に返ります。
- **応答を待つ:** タイムアウトを設定すると、応答がその場で返ります。

ターゲットが応答した後、OpenClaw は **reply-back loop** を実行でき、エージェント同士が最大 5 ターンまで交互にメッセージをやり取りします。ターゲットエージェントは `REPLY_SKIP` を返して早期終了できます。

## ステータスとオーケストレーションの補助

`session_status` は、現在のセッションまたは他の可視セッションに対する軽量な `/status` 相当ツールです。使用状況、時刻、モデル/ランタイム状態、および存在する場合は関連するバックグラウンドタスクのコンテキストを報告します。`/status` と同様に、最新の transcript 使用量エントリから疎なトークン/キャッシュカウンターを補完でき、`model=default` でセッション単位の上書きをクリアします。

`sessions_yield` は、現在のターンを意図的に終了し、次のメッセージが待機中の後続イベントになるようにします。完了結果をポーリングループで組み立てるのではなく、次のメッセージとして受け取りたいときに、サブエージェント起動後に使ってください。

`subagents` は、すでに起動された OpenClaw サブエージェントの control-plane 補助です。次をサポートします。

- `action: "list"` でアクティブ/最近の実行を確認
- `action: "steer"` で実行中の子に追加ガイダンスを送信
- `action: "kill"` で 1 つの子、または `all` を停止

## サブエージェントの起動

`sessions_spawn` は、バックグラウンドタスク用の分離セッションを作成します。これは常にノンブロッキングで、`runId` と `childSessionKey` を即座に返します。

主なオプション:

- `runtime: "subagent"`（デフォルト）または外部ハーネスエージェント用の `"acp"`。
- 子セッション用の `model` と `thinking` 上書き。
- `thread: true` で、その起動をチャットスレッド（Discord、Slack など）にバインド。
- `sandbox: "require"` で、子に sandbox を強制。

デフォルトの leaf サブエージェントにはセッションツールは付与されません。`maxSpawnDepth >= 2` の場合、深さ 1 のオーケストレーターサブエージェントには追加で `sessions_spawn`、`subagents`、`sessions_list`、`sessions_history` が与えられ、自分の子を管理できます。leaf 実行には引き続き再帰的なオーケストレーションツールは付与されません。

完了後、announce ステップが要求元チャネルに結果を投稿します。完了配信では、利用可能な場合にバインドされたスレッド/トピックのルーティングが維持され、完了元がチャネルしか識別していない場合でも、OpenClaw は要求元セッションの保存済みルート（`lastChannel` / `lastTo`）を再利用して直接配信できます。

ACP 固有の動作については [ACP Agents](/ja-JP/tools/acp-agents) を参照してください。

## 可視性

セッションツールには、エージェントが見られる範囲を制限するスコープがあります。

| レベル  | スコープ                               |
| ------- | -------------------------------------- |
| `self`  | 現在のセッションのみ                   |
| `tree`  | 現在のセッション + 起動したサブエージェント |
| `agent` | このエージェントのすべてのセッション   |
| `all`   | すべてのセッション（設定されていればエージェント横断） |

デフォルトは `tree` です。sandbox 化されたセッションは、config に関係なく `tree` に制限されます。
