並列スペシャリストレーンを使用すると、1つの Gateway で異なるチャットやルームを
異なるエージェントに振り分けながら、高速なユーザー体験を維持できます。並列処理は
単なる「エージェントの増員」ではなく、希少なリソースの設計問題として扱ってください。

## 基本原則

スペシャリストレーンがスループットを向上させるのは、実際のボトルネックにおける
競合を軽減する場合に限られます。

- **セッションロック**: 特定のセッションを変更する実行は、一度に1つだけにする必要があります。
- **モデル全体のキャパシティ**: ユーザーに見えるすべてのチャット実行は、引き続きプロバイダーの制限を共有します。
- **ツールのキャパシティ**: シェル、ブラウザー、ネットワーク、リポジトリでの作業は、
  モデルのターン自体よりも時間がかかる場合があります。
- **コンテキスト予算**: 長いトランスクリプトは、以降のすべてのターンを遅くし、
  焦点をぼやけさせます。
- **所有権の曖昧さ**: 複数のエージェントが同じ作業を重複して行うと、キャパシティが無駄になります。

OpenClaw はすでにセッションごとに実行を直列化し、
[コマンドキュー](/ja-JP/concepts/queue)を通じて全体の並列処理数を制限しています。
スペシャリストレーンはその上に、どのエージェントがどの作業を所有するか、
チャット内に残す作業は何か、バックグラウンド作業に移すものは何かというポリシーを追加します。

## 推奨される導入手順

### フェーズ1: レーン契約と負荷の高いバックグラウンド作業

各レーンのワークスペースとシステムプロンプトに、明文化した契約を設定します。

- **目的**: このレーンが所有する作業。
- **対象外**: 自ら実行せず、引き渡すべき作業。
- **チャット予算**: 簡単な回答はチャット内で行います。時間のかかるタスクは簡潔に受領を伝え、
  バックグラウンドのサブエージェントまたはタスクで実行します。
- **引き渡しルール**: 別のレーンが作業を所有する場合は、送るべきレーンを示し、
  簡潔な引き渡し概要を提供します。
- **ツールリスクのルール**: 作業を実行できる最小限のツール範囲を優先します。

これは最も低コストなフェーズであり、処理停滞の大部分を解消します。1件のコーディング作業によって
調査レーンの動作が極端に遅くなることがなくなり、各チャットのコンテキストも
クリーンに保たれます。

### フェーズ2: 優先度と並行処理の制御

各レーンの業務価値に合わせて、キューとモデルのキャパシティを調整します。

```json5
{
  agents: {
    defaults: {
      maxConcurrent: 4,
      subagents: { maxConcurrent: 8, delegationMode: "prefer" },
    },
  },
  messages: {
    queue: {
      mode: "collect",
      debounceMs: 1000,
      cap: 20,
      drop: "summarize",
    },
  },
}
```

優先度の高い作業には、ダイレクトチャットや個人チャット、および本番運用エージェントを使用します。
システムがビジー状態のときは、調査、下書き作成、バッチコーディングを
バックグラウンドタスクに移します。

### フェーズ3: コーディネーター／トラフィックコントローラー

複数のレーンが稼働したら、小規模なコーディネーターパターンを追加します。

- 稼働中のレーンタスクと所有者を追跡します。
- グループ間で重複するリクエストを検出します。
- レーン間で引き渡し概要を振り分けます。
- ブロッカー、完了した結果、人間が判断すべき事項だけを提示します。

このフェーズから始めないでください。レーン契約のないコーディネーターは、
混乱を調整するだけです。

## 最小限のレーン契約テンプレート

```md
# レーン契約

## 担当範囲

- <このレーンが担当する作業>

## 担当範囲外

- <引き渡す作業>

## チャット予算

- 簡単な質問には直接回答する。
- 複数ステップ、時間がかかる、またはツールを多用する作業の場合: 簡潔に受領を伝え、
  作業を生成／バックグラウンド化し、完了後に結果を返す。

## 引き渡し

別のレーンがリクエストを所有する場合は、以下を含めて返信する。

- 対象レーン
- 目的
- 関連するコンテキスト
- 正確な次のアクション

## ツール利用方針

タスクを完了できる最小限のツール範囲を使用する。このレーンが明示的に所有していない限り、
広範なシェルまたはネットワーク作業は避ける。
```
