# OpenClaw v2026.6.11 リリースノート (2026-06-30)

フィードバックを受け止めました。v2026.6.11 では、OpenClaw の信頼性を損なっていた使いにくい点に重点を置き、返信先の誤り、送信の停止、再接続、モデル設定の失敗を修正し、管理者向けのデフォルト設定をより安全にしました。

## ハイライト

### チャネル配信の信頼性

[Telegram](/ja-JP/channels/telegram)、[WhatsApp](/ja-JP/channels/whatsapp)、[Matrix](/ja-JP/channels/matrix)、[Google Chat](/ja-JP/channels/googlechat)、[iMessage](/ja-JP/channels/imessage)、[Feishu](/ja-JP/channels/feishu)、[Mattermost](/ja-JP/channels/mattermost) 全体で、返信、コマンド、キューに入ったメッセージ、添付ファイルが欠落、重複、誤配送されたり、誤った会話に添付されたりしにくくなりました。

### プロバイダーとモデルの復旧

[モデル選択](/ja-JP/providers/models)とセットアップは、カタログ、認証情報、ストリーム、タイムアウト、Compaction、またはフォールバックで問題が発生した場合に、より明確に復旧できるようになりました。影響を受けていた[OpenAI](/ja-JP/providers/openai)、[OpenRouter](/ja-JP/providers/openrouter)、[OpenCode Go](/ja-JP/providers/opencode-go)の構成では、古い選択肢が残ったりリクエストが停止したりする代わりに、失敗の報告や復旧がより一貫して行われるようになりました。

### セッション、メモリ、信頼の継続性

[セッション](/ja-JP/concepts/session)、[Compaction](/ja-JP/concepts/compaction)、[メモリ](/ja-JP/concepts/memory)、[QMDを基盤とするメモリ](/ja-JP/concepts/memory-qmd)により、長時間の作業、再接続、アップグレード、トランスクリプトの修復を通じて、意図した会話と有用なコンテキストがより一貫して維持されます。また、エージェントが適切なコンテキストや機能を復元する必要がある場合も、[ツール検索](/ja-JP/tools/tool-search)がより確実に動作します。

暗号化された[Matrix](/ja-JP/channels/matrix)の復旧では、必要な鍵の状態を検証できない場合、安全に停止します。[ツールポリシー](/ja-JP/gateway/sandbox-vs-tool-policy-vs-elevated)、[承認](/ja-JP/cli/approvals)、[シークレットの取り扱い](/ja-JP/gateway/secrets)は意図したランタイム状態に関連付けられたままとなり、明示的に有効化されない限り、リスクの高い操作は無効のままです。

## Gateway、セキュリティ、信頼

### 再起動と準備状態の復旧

[Gateway の健全性](/ja-JP/gateway/health)と[トラブルシューティング](/ja-JP/gateway/troubleshooting)のシグナルが、OpenClaw が実際に準備完了しているか、再起動中か、または続行不能かという状態と、より一貫して対応するようになりました。[CLI](/ja-JP/cli/agent)から開始されたエージェント実行は、切断、シャットダウン、ルーティング変更、起動失敗の状態から、より確実に復旧します。

<Accordion title="情報源と貢献者">

- 設定済みまたは明示的に指定されたリモート Gateway が低速でも到達可能な場合、`openclaw gateway probe --timeout ...` は短い内部制限時間の経過後に到達不能と報告せず、指定されたタイムアウトまで待機するようになりました。[#89859](https://github.com/openclaw/openclaw/pull/89859) 関連: [#65355](https://github.com/openclaw/openclaw/issues/65355)。@hellocli、@mushuiyu886 に感謝します。
- 長時間または並列で実行される内部サブエージェントが不要なライブプレビュー処理を回避する一方、表示中のサブエージェントセッションでは引き続きライブ更新と最終応答が表示されるようになりました。[#91906](https://github.com/openclaw/openclaw/pull/91906) @lanzhi-lee、@vincentkoc に感謝します。
- ACP の会話、特に Kiro をバックエンドとするスレッドでは、バックエンドが古いセッションを再開できなくなった場合に新しいセッションを開始することで、最初の返信後も会話を続けられるようになりました。[#93547](https://github.com/openclaw/openclaw/pull/93547) 関連: [#87830](https://github.com/openclaw/openclaw/issues/87830)。@amersheeny、@chouzz に感謝します。

### ツールとスケジュールされた処理

[スケジュールされたジョブ](/ja-JP/cli/cron)と組み込みの[ツール](/ja-JP/tools/index)で、完了、再試行、失敗の報告、および意図した入力の保持が、より一貫して行われるようになりました。[Plugin SDK ランタイム](/ja-JP/plugins/sdk-runtime)の改善により、ツールを利用する拡張機能も、読み込み、結果の返却、スケジュールされた処理の実行時に、より確実に動作するようになりました。
