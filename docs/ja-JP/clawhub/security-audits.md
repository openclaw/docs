# セキュリティ監査

ClawHub のセキュリティ監査は、Skill や plugin をインストールしても十分安全かどうかを判断するのに役立ちます。リリースが何をするか、どの権限を求めるか、ファイル、アカウント、認証情報、コード、外部サービスへアクセスする前に特に注意すべき点があるかを示します。

監査は強力な安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、必ず自分で判断してください。

関連項目: [セキュリティ](/clawhub/security)、[利用許容ポリシー](/ja-JP/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/clawhub/moderation)。

## インストール前に確認すること

インストール前に、次を確認してください。

- 全体の監査ステータス
- リスクレベル
- 記載された findings
- 必要な認証情報、権限、環境変数
- オーナー、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し、信頼できるコンテンツだけをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 低リスクを上回る目に見える問題は見つかりませんでした。 |
| `Review`    | インストール前に findings を読んでください。リリースは依然として正当な可能性があります。 |
| `Warn`      | 特に注意してください。ClawHub が影響の大きい懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。 |
| `Pending`   | 監査はまだ完了していません。 |
| `Error`     | 監査を完了できませんでした。 |

`Pass` は安心材料ですが、自分自身の判断に代わるものではありません。これは、コンテンツを公開したり、データを編集したり、コマンドを実行したり、ファイルを読んだり、本番システムへアクセスしたりできるツールで特に重要です。

## リスクレベル

リスクレベルは、意図どおりに使用した場合にリリースが持つと思われる権限の範囲、つまり影響範囲を表します。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium`   | リリースには、アカウントアクセスやデータ変更などの意味のある権限があります。 |
| `High`     | リリースには、影響の大きい権限、重大な findings、または悪意のあるシグナルがあります。 |

リスクレベルと監査ステータスは異なる問いに答えます。

- リスクレベルが問うこと: 「ここにはどれだけの権限があるか」
- 監査ステータスが問うこと: 「この結果をどう扱うべきか」

たとえば、公開用の skill は `Medium` リスクで `Review` と表示される場合があります。これは悪意があるという意味ではありません。その skill は目的に沿っているように見えるものの、意味のあるアカウント権限で動作できるという意味です。

## Findings

Findings は、監査結果が表示された理由を説明します。各 finding には通常、次が含まれます。

- それが何を意味するか
- フラグされた理由
- 関連する skill または plugin の内容
- 推奨事項

Findings には `Info`、`Low`、`Medium`、`High`、`Critical` のラベルが付くことがあります。重大度の高い findings ほど、リスクレベルと監査ステータスに強く影響します。

信頼度の低い findings は、ページが有用な証拠に集中できるよう、公開監査ロールアップでは非表示になります。

## ClawHub が確認すること

ClawHub は提出されたリリース成果物を監査します。これには次が含まれます。

- skill の指示または plugin メタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と capability メタデータ

主な問いは一貫性です。名前、概要、メタデータ、要求された権限、実際の内容が、ユーザーが妥当に期待する内容と一致しているかどうかです。

強力な動作は自動的に悪いわけではありません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダー API、パッケージインストールが必要です。監査では、その権限が期待され、開示され、目的に見合っているかを確認します。

成果物ページは、次の完全な監査にリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は監査スタック内のマルウェアテレメトリとして VirusTotal を使用します。VirusTotal はファイルの評判とマルウェアスキャンに関する信頼された業界標準であり、私たちのパートナーシップにより、ClawHub は skill と plugin のレビューにより広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意ある成果物、エンジンの検出、ClawHub の agent-aware レビューを補完する評判シグナルに特に有用です。ベンダーエンジン数が利用できる場合、監査は次のように平易な言葉で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー数テレメトリがない場合、監査は次のように表示します。

```text
No VirusTotal findings
```

VirusTotal はあくまでテレメトリです。ClawHub 独自の成果物を考慮したリスク分析に代わるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に提供されます。各リリースを agent 向けの成果物としてレビューします。対象には、指示、メタデータ、宣言された権限、ファイル、capability シグナル、静的スキャンシグナル、SkillSpector findings、VirusTotal テレメトリ、公開者が提供したコンテキストが含まれます。静的スキャンシグナルはこのレビューの内部コンテキストであり、独立した公開監査セクションやインストールをブロックする判定ではありません。

リスク分析では、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリまたはコンテキストの汚染、過剰な agency などのリスクを見る視点として、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/) を使用します。

ClawScan は、怖く見える capability を自動的に悪意あるものとは扱いません。その capability が開示され、目的に沿っており、リリースが述べるユースケースによって裏付けられているかを確認します。
