# セキュリティ監査

ClawHub のセキュリティ監査は、スキルまたはプラグインをインストールしても十分に安全かどうかを判断するのに役立ちます。リリースが何をするか、どの権限を求めるか、ファイル、アカウント、認証情報、コード、外部サービスへアクセスする前に追加の注意が必要なものがあるかを示します。

監査は強い安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、必ず自分で判断してください。

関連項目: [セキュリティ](/ja-JP/clawhub/security)、[許容される利用](/ja-JP/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/ja-JP/clawhub/moderation)。

## インストール前に確認すること

インストール前に、次を確認してください。

- 全体的な監査ステータス
- リスクレベル
- 記載されている所見
- 必要な認証情報、権限、または環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し、信頼できるコンテンツのみをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 低リスクを超える目に見える問題は見つかりませんでした。 |
| `Review`    | インストール前に所見を読んでください。リリースは正当なものである可能性があります。 |
| `Warn`      | 追加の注意を払ってください。ClawHub が高影響の懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。 |
| `Pending`   | 監査はまだ完了していません。 |
| `Error`     | 監査を完了できませんでした。 |

`Pass` は安心材料になりますが、自分自身の判断に代わるものではありません。これは、コンテンツの公開、データの編集、コマンドの実行、ファイルの読み取り、本番システムへのアクセスが可能なツールで特に重要です。

## リスクレベル

リスクレベルは影響範囲を表します。意図された使い方をした場合に、そのリリースがどれだけの力を持つように見えるかです。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium`   | リリースには、アカウントアクセスやデータ変更など、意味のある権限があります。 |
| `High`     | リリースには、高影響の権限、重大な所見、または悪意のあるシグナルがあります。 |

リスクレベルと監査ステータスは、異なる問いに答えます。

- リスクレベルが問うこと: 「ここにはどれだけの権限があるか?」
- 監査ステータスが問うこと: 「この結果にどう対応すべきか?」

たとえば、公開用のスキルは `Medium` リスクで `Review` と表示されることがあります。これは悪意があるという意味ではありません。そのスキルが目的に沿っているように見える一方で、意味のあるアカウント権限で動作できることを意味します。

## 所見

所見は、監査結果が表示された理由を説明します。各所見には通常、次が含まれます。

- それが何を意味するか
- フラグ付けされた理由
- 関連するスキルまたはプラグインのコンテンツ
- 推奨事項

所見には `Info`、`Low`、`Medium`、`High`、または `Critical` のラベルが付くことがあります。重大度の高い所見ほど、リスクレベルと監査ステータスにより強く影響します。

信頼度の低い所見は公開監査の要約から非表示にされるため、ページは有用な証拠に集中した状態に保たれます。

## ClawHub が確認すること

ClawHub は、提出されたリリース成果物を監査します。対象には次が含まれます。

- スキルの指示またはプラグインのメタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能メタデータ

主な問いは一貫性です。名前、概要、メタデータ、要求される権限、実際のコンテンツが、ユーザーが合理的に期待する内容と一致しているかどうかです。

強力な動作が自動的に悪いわけではありません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダー API、パッケージインストールが必要です。監査は、その力が期待され、開示され、釣り合っているかを確認します。

成果物ページは、完全な監査へのリンクを次の場所に表示します。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は、監査スタック内のマルウェアテレメトリとして VirusTotal を使用します。VirusTotal はファイルレピュテーションとマルウェアスキャンの信頼された業界標準であり、このパートナーシップにより、ClawHub はスキルとプラグインのレビューにより広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意ある成果物、エンジンの検出、ClawHub のエージェント対応レビューを補完するレピュテーションシグナルに特に有用です。ベンダーエンジンの件数が利用できる場合、監査は次のように平易な言葉で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub が要約できるベンダー件数テレメトリを持っていない場合、監査は次のように表示します。

```text
No VirusTotal findings
```

VirusTotal は引き続きテレメトリです。ClawHub 独自の成果物認識型リスク分析を置き換えるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に提供されます。各リリースを、エージェント向けの成果物としてレビューします。対象には、指示、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpector の所見、VirusTotal テレメトリ、公開者が提供したコンテキストが含まれます。静的スキャンシグナルはこのレビューの内部コンテキストであり、独立した公開監査セクションやインストールをブロックする判定ではありません。

リスク分析は、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリまたはコンテキストの汚染、過剰なエージェンシーなどのリスクを見るための観点として、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/) を使用します。

ClawScan は、危険そうに見える機能を自動的に悪意あるものとして扱いません。その機能が開示され、目的に沿っており、リリースで示されたユースケースに支えられているかを確認します。
