# セキュリティ監査

ClawHub のセキュリティ監査は、スキルまたは Plugin をインストールしても十分に安全かどうかを判断するのに役立ちます。監査では、リリースが何をするか、どの権限を求めるか、ファイル、アカウント、認証情報、コード、外部サービスにアクセスできるようになる前に追加の注意が必要な点があるかどうかを示します。

監査は強い安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、必ず自身で判断してください。

[セキュリティ](/clawhub/security)、[許容される利用](/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/clawhub/moderation)も参照してください。

## インストール前に確認すること

インストールする前に、次を確認してください。

- 全体的な監査ステータス
- リスクレベル
- 記載されている検出事項
- 必要な認証情報、権限、環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し信頼できるコンテンツだけをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 低リスクを超える可視の問題は見つかりませんでした。 |
| `Review`    | インストール前に検出事項を読んでください。そのリリースは正当なものである可能性があります。 |
| `Warn`      | 追加の注意を払ってください。ClawHub は影響の大きい懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。 |
| `Pending`   | 監査はまだ完了していません。 |
| `Error`     | 監査を完了できませんでした。 |

`Pass` は安心材料になりますが、自身の判断に代わるものではありません。これは、コンテンツを公開する、データを編集する、コマンドを実行する、ファイルを読む、本番システムへアクセスする、といったことができるツールで特に重要です。

## リスクレベル

リスクレベルは影響範囲、つまり意図どおりに使用した場合にそのリリースが持つと思われる権限の大きさを表します。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium`   | そのリリースには、アカウントアクセスやデータ変更など、意味のある権限があります。 |
| `High`     | そのリリースには、影響の大きい権限、重大な検出事項、または悪意のあるシグナルがあります。 |

リスクレベルと監査ステータスは、異なる問いに答えます。

- リスクレベルが問うこと: 「ここにはどれほどの権限があるか?」
- 監査ステータスが問うこと: 「この結果に対して何をすべきか?」

たとえば、公開用のスキルは `Medium` リスクの `Review` と表示される場合があります。これは、そのスキルが悪意のあるものだという意味ではありません。そのスキルが目的に沿っているように見える一方で、意味のあるアカウント権限で動作できることを意味します。

## 検出事項

検出事項は、なぜその監査結果が表示されたのかを説明します。各検出事項には通常、次が含まれます。

- それが何を意味するか
- なぜフラグされたのか
- 関連するスキルまたは Plugin のコンテンツ
- 推奨事項

検出事項には、`Info`、`Low`、`Medium`、`High`、`Critical` のラベルが付く場合があります。重大度が高い検出事項ほど、リスクレベルと監査ステータスにより強く影響します。

信頼度の低い検出事項は、ページが有用な証拠に集中できるよう、公開監査ロールアップでは非表示になります。

## ClawHub が確認すること

ClawHub は、提出されたリリースアーティファクトを監査します。これには次が含まれます。

- スキルの指示または Plugin メタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能メタデータ

主な問いは一貫性です。名前、要約、メタデータ、要求された権限、実際のコンテンツが、ユーザーが合理的に期待するものと一致しているかどうかです。

強力な動作は、それだけで悪いものではありません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダー API、パッケージインストールが必要です。監査では、その権限が期待どおりで、開示され、釣り合っているかを確認します。

アーティファクトページは、完全な監査に次の場所でリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は、監査スタック内のマルウェアテレメトリとして VirusTotal を使用します。VirusTotal は、ファイルの評価とマルウェアスキャンにおける信頼された業界標準であり、このパートナーシップにより ClawHub はスキルと Plugin のレビューにより広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意あるアーティファクト、エンジンの検出、ClawHub のエージェント認識レビューを補完する評価シグナルに特に有用です。ベンダーエンジン数が利用できる場合、監査はそれらを次のような平易な表現で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー数テレメトリがない場合、監査は次のように表示します。

```text
No VirusTotal findings
```

VirusTotal は引き続きテレメトリです。ClawHub 独自のアーティファクト認識リスク分析に代わるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に動作します。各リリースを、エージェント向けアーティファクトとしてレビューします。対象には、指示、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpector の検出事項、VirusTotal テレメトリ、公開者が提供したコンテキストが含まれます。静的スキャンシグナルは、このレビューのための内部コンテキストです。独立した公開監査セクションやインストールをブロックする判定ではありません。

リスク分析では、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリまたはコンテキストの汚染、過剰なエージェンシーなどのリスクを見るための観点として、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/) を使用します。

ClawScan は、一見危険そうな機能を自動的に悪意あるものとは扱いません。その機能が開示され、目的に沿っており、リリースで明示されたユースケースによって裏付けられているかを確認します。
