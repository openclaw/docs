# セキュリティ監査

ClawHub のセキュリティ監査は、Skills または Plugin をインストールしても十分に安全かどうかを判断するのに役立ちます。監査では、リリースが何を行うか、どの権限を求めるか、ファイル、アカウント、認証情報、コード、外部サービスへアクセスする前に追加の注意が必要な点があるかどうかを示します。

監査は強力な安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機微なアクセスを許可する前に、常に自分で判断してください。

関連項目: [セキュリティ](/ja-JP/clawhub/security)、[許容される利用](/ja-JP/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/ja-JP/clawhub/moderation)。

## インストール前に確認すること

インストール前に、次を確認してください。

- 全体的な監査ステータス
- リスクレベル
- 記載されている所見
- 必要な認証情報、権限、または環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し、信頼できるコンテンツのみをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 低リスクを超える目に見える問題は見つかりませんでした。 |
| `Review`    | インストール前に所見を読んでください。リリースは正当なものである可能性があります。 |
| `Warn`      | 特に注意してください。ClawHub が影響の大きい懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。 |
| `Pending`   | 監査はまだ完了していません。 |
| `Error`     | 監査を完了できませんでした。 |

`Pass` は安心材料ですが、自分自身の判断に代わるものではありません。これは、コンテンツの公開、データの編集、コマンドの実行、ファイルの読み取り、本番システムへのアクセスが可能なツールで特に重要です。

## リスクレベル

リスクレベルは影響範囲を表します。つまり、意図されたとおりに使用した場合に、そのリリースがどれほどの力を持つように見えるかです。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | 機微な権限またはユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium`   | リリースには、アカウントアクセスやデータ変更など、意味のある権限があります。 |
| `High`     | リリースには、影響の大きい権限、重大な所見、または悪意のあるシグナルがあります。 |

リスクレベルと監査ステータスは異なる問いに答えます。

- リスクレベルが問うのは、「ここにはどれほどの力があるか」です。
- 監査ステータスが問うのは、「この結果をどう扱うべきか」です。

たとえば、公開用の Skills は `Medium` リスクで `Review` と表示される場合があります。これは悪意があるという意味ではありません。その Skills が目的に沿っているように見える一方で、意味のあるアカウント権限で動作できることを意味します。

## 所見

所見は、監査結果が表示された理由を説明します。各所見には通常、次が含まれます。

- それが何を意味するか
- フラグが付けられた理由
- 関連する Skills または Plugin のコンテンツ
- 推奨事項

所見には `Info`、`Low`、`Medium`、`High`、または `Critical` のラベルが付くことがあります。深刻度が高い所見ほど、リスクレベルと監査ステータスにより強く影響します。

信頼度の低い所見は公開監査の要約から非表示にされるため、ページは有用な証拠に集中できます。

## ClawHub が確認すること

ClawHub は、提出されたリリースアーティファクトを監査します。これには次が含まれます。

- Skills の指示または Plugin メタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能メタデータ

主な問いは一貫性です。名前、概要、メタデータ、要求される権限、実際のコンテンツが、ユーザーが合理的に期待する内容と一致しているかどうかです。

強力な動作は、自動的に悪いわけではありません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダー API、またはパッケージのインストールが必要です。監査では、その力が期待され、開示され、釣り合いが取れているかを確認します。

アーティファクトページは、次の完全な監査にリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は、監査スタック内のマルウェアテレメトリとして VirusTotal を使用します。VirusTotal はファイルの評判とマルウェアスキャンに関する信頼された業界標準であり、この提携により ClawHub は Skills と Plugin のレビューに、より広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意あるアーティファクト、エンジン検出、ClawHub のエージェント対応レビューを補完する評判シグナルに特に有用です。ベンダーエンジン数が利用可能な場合、監査はそれを次のような平易な表現で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー数テレメトリがない場合、監査には次のように表示されます。

```text
No VirusTotal findings
```

VirusTotal はあくまでテレメトリです。ClawHub 独自のアーティファクト対応リスク分析に代わるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に実行されます。各リリースをエージェント向けアーティファクトとしてレビューします。対象には、指示、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpector の所見、VirusTotal テレメトリ、公開者が提供した文脈が含まれます。静的スキャンシグナルは、このレビューの内部文脈です。独立した公開監査セクションでも、インストールをブロックする判定でもありません。

リスク分析では、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリまたはコンテキスト汚染、過剰な自律性といったリスクを見るための観点として、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/) を使用します。

ClawScan は、怖そうに見える機能を自動的に悪意あるものとして扱いません。その機能が開示され、目的に沿っており、リリースで述べられたユースケースに支えられているかを確認します。
