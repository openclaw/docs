# セキュリティ監査

ClawHubのセキュリティ監査は、スキルまたはプラグインをインストールしても十分に安全かどうかを判断するのに役立ちます。監査では、リリースが何を行うか、どの権限を要求するか、ファイル、アカウント、認証情報、コード、外部サービスにアクセスする前に追加の注意が必要な点があるかどうかを示します。

監査は強力な安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、常に自分で判断してください。

[セキュリティ](/clawhub/security)、[許容される利用](/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/clawhub/moderation)も参照してください。

## インストール前に確認すること

インストール前に、次を確認してください。

- 全体的な監査ステータス
- リスクレベル
- 一覧表示された指摘事項
- 必要な認証情報、権限、または環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し信頼できるコンテンツのみをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass` | 低リスクを超える目に見える問題は見つかりませんでした。 |
| `Review` | インストール前に指摘事項を読んでください。リリースは正当なものである可能性があります。 |
| `Warn` | 特に注意してください。ClawHubが影響の大きい懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。 |
| `Pending` | 監査はまだ完了していません。 |
| `Error` | 監査を完了できませんでした。 |

`Pass` は安心材料になりますが、自分自身の判断に代わるものではありません。これは、コンテンツの公開、データの編集、コマンドの実行、ファイルの読み取り、本番システムへのアクセスが可能なツールで特に重要です。

## リスクレベル

リスクレベルは影響範囲を示します。つまり、そのリリースを意図どおりに使用した場合に、どれほど大きな権限を持つように見えるかです。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low` | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium` | アカウントアクセスやデータ変更など、リリースには意味のある権限があります。 |
| `High` | リリースには影響の大きい権限、重大な指摘事項、または悪意のあるシグナルがあります。 |

リスクレベルと監査ステータスは、異なる問いに答えます。

- リスクレベルが問うこと: 「ここにはどれほどの権限があるか？」
- 監査ステータスが問うこと: 「この結果をどう扱うべきか？」

たとえば、公開用のスキルは `Medium` リスクで `Review` と表示されることがあります。これは悪意があるという意味ではありません。そのスキルが目的に沿っているように見える一方で、意味のあるアカウント権限で動作できることを意味します。

## 指摘事項

指摘事項は、監査結果が表示された理由を説明します。各指摘事項には通常、次が含まれます。

- それが何を意味するか
- なぜフラグ付けされたか
- 関連するスキルまたはプラグインのコンテンツ
- 推奨事項

指摘事項には `Info`、`Low`、`Medium`、`High`、または `Critical` のラベルが付くことがあります。重大度が高い指摘事項ほど、リスクレベルと監査ステータスにより強く影響します。

信頼度の低い指摘事項は、ページが有用な証拠に集中できるように、公開監査の集計からは非表示になります。

## ClawHubが確認すること

ClawHubは、提出されたリリースアーティファクトを監査します。これには次が含まれます。

- スキルの指示またはプラグインのメタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能メタデータ

主な問いは一貫性です。名前、概要、メタデータ、要求された権限、実際のコンテンツが、ユーザーが合理的に期待する内容と一致しているかどうかです。

強力な動作は、自動的に悪いものとは限りません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダーAPI、パッケージのインストールが必要です。監査では、その権限が期待され、開示され、目的に見合っているかを確認します。

アーティファクトページは、完全な監査へのリンクを次の場所に表示します。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHubは監査スタック内のマルウェアテレメトリとしてVirusTotalを使用します。VirusTotalはファイルのレピュテーションとマルウェアスキャンにおける信頼された業界標準であり、この提携によりClawHubはスキルおよびプラグインのレビューに、より広範なセキュリティインテリジェンスを追加できます。

VirusTotalは、既知の悪意あるアーティファクト、エンジンでの検出、ClawHubのエージェントを考慮したレビューを補完するレピュテーションシグナルに特に有用です。ベンダーエンジン数を利用できる場合、監査は次のように平易な言葉で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHubに要約できるベンダー数テレメトリがない場合、監査では次のように表示されます。

```text
No VirusTotal findings
```

VirusTotalはあくまでテレメトリです。ClawHub自身のアーティファクトを考慮したリスク分析に代わるものではありません。

## リスク分析

リスク分析は、ClawHub独自のセキュリティ監査システムであるClawScanによって内部的に実行されます。ClawScanは各リリースをエージェント向けアーティファクトとしてレビューします。対象には、指示、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpectorの指摘事項、VirusTotalテレメトリ、公開者が提供したコンテキストが含まれます。静的スキャンシグナルは、このレビューのための内部コンテキストです。独立した公開監査セクションや、インストールをブロックする判定ではありません。

リスク分析では、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリまたはコンテキストの汚染、過剰なエージェンシーなどのリスクを見るための観点として、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)を使用します。

ClawScanは、恐ろしく見える機能を自動的に悪意があるものとして扱うわけではありません。その機能が開示され、目的に沿っており、リリースに記載されたユースケースによって裏付けられているかを確認します。
