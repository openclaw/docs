# セキュリティ監査

ClawHub のセキュリティ監査は、Skill や Plugin がインストールするのに十分安全かどうかを判断するのに役立ちます。監査では、リリースが何を行うか、どのような権限を要求するか、ファイル、アカウント、認証情報、コード、外部サービスへのアクセスを許可する前に特に注意すべき点があるかどうかを確認できます。

監査は安全性を判断するうえで有力な指標ですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、必ずご自身で判断してください。

[セキュリティ](/clawhub/security)、[利用規定](/ja-JP/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/clawhub/moderation)も参照してください。

## インストール前の確認事項

インストールする前に、以下を確認してください。

- 監査の総合ステータス
- リスクレベル
- 記載されている検出事項
- 必要な認証情報、権限、環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼性指標

内容を理解し、信頼できるものだけをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどのように対応すべきかを示します。

| ステータス  | 意味                                                                      |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 低リスクを超える明らかな問題は見つかりませんでした。                     |
| `Review`    | インストール前に検出事項を確認してください。正当なリリースである可能性はあります。 |
| `Warn`      | 特に注意してください。ClawHub が重大な懸念事項または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。                                            |
| `Pending`   | 監査はまだ完了していません。                                              |
| `Error`     | 監査を完了できませんでした。                                              |

`Pass` は安心材料になりますが、ご自身の判断に代わるものではありません。これは、コンテンツの公開、データの編集、コマンドの実行、ファイルの読み取り、本番システムへのアクセスが可能なツールで特に重要です。

## リスクレベル

リスクレベルは影響範囲、つまりリリースを意図どおりに使用した場合に、どの程度の権限を持つと考えられるかを表します。

| リスクレベル | 意味                                                                         |
| ------------ | ---------------------------------------------------------------------------- |
| `Low`        | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。           |
| `Medium`     | アカウントへのアクセスやデータの変更など、重要な権限を持つリリースです。     |
| `High`       | 重大な影響を及ぼす権限、深刻な検出事項、または悪意を示すシグナルがあります。 |

リスクレベルと監査ステータスは、それぞれ異なる問いに答えます。

- リスクレベルの問い：「ここにはどの程度の権限があるか？」
- 監査ステータスの問い：「この結果を受けてどうすべきか？」

たとえば、公開用の Skill が `Medium` リスクの `Review` と表示される場合があります。これは悪意があるという意味ではありません。その Skill は目的に沿っていると考えられるものの、重要なアカウント権限を使用して操作できることを意味します。

## 検出事項

検出事項は、監査結果が表示された理由を説明します。各検出事項には通常、以下が含まれます。

- その意味
- フラグが付けられた理由
- 関連する Skill または Plugin の内容
- 推奨事項

検出事項には、`Info`、`Low`、`Medium`、`High`、`Critical` のラベルが付く場合があります。重大度の高い検出事項ほど、リスクレベルと監査ステータスに強く影響します。

確度の低い検出事項は公開監査の集計から除外されるため、ページには有用な証拠のみが表示されます。

## ClawHub の確認内容

ClawHub は、提出された以下のようなリリース成果物を監査します。

- Skill の指示または Plugin のメタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 同梱ファイルとファイルマニフェスト
- 互換性と機能のメタデータ

主な確認事項は整合性です。名前、概要、メタデータ、要求される権限、実際の内容が、ユーザーが合理的に期待する内容と一致しているかを確認します。

強力な動作が自動的に問題になるわけではありません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダー API、パッケージのインストールが必要です。監査では、その権限が想定内であり、開示され、相応であるかを確認します。

成果物ページでは、以下の完全な監査へのリンクが提供されます。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページでは、以下を組み合わせて表示します。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は、監査スタックでマルウェアのテレメトリとして VirusTotal を使用します。VirusTotal は、ファイルの評価とマルウェアスキャンにおいて信頼されている業界標準であり、この提携によって ClawHub は Skill と Plugin の審査に、より広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意ある成果物、エンジンによる検出、ClawHub のエージェント対応審査を補完する評価シグナルに特に有用です。ベンダーエンジンの件数を取得できる場合、監査では次のように平易な言葉で要約します。

```text
62/62 のベンダーが、この Skill をクリーンと判定しました。
```

または、

```text
2/64 のベンダーがこの Skill を悪意あり、1/64 が疑わしい、61/64 がクリーンと判定しました。
```

ClawHub に要約可能なベンダー件数のテレメトリがない場合、監査には次のように表示されます。

```text
VirusTotal の検出事項はありません
```

VirusTotal はあくまでテレメトリです。ClawHub 独自の成果物を考慮したリスク分析に代わるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に実行されます。各リリースをエージェント向け成果物として審査し、指示、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpector の検出事項、VirusTotal のテレメトリ、公開者が提供したコンテキストを確認します。静的スキャンシグナルはこの審査の内部コンテキストであり、独立した公開監査セクションでも、インストールをブロックする判定でもありません。

リスク分析では、
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
を基準として、プロンプトインジェクション、ツールの誤用、認証情報の漏えい、安全でない実行、メモリやコンテキストの汚染、過剰な自律性などのリスクを評価します。

ClawScan は、危険に見える機能を自動的に悪意のあるものとは判断しません。その機能が開示され、目的に沿っており、リリースで明示されたユースケースによって裏付けられているかを確認します。
