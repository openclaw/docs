# セキュリティ監査

ClawHub のセキュリティ監査は、スキルまたはPluginをインストールしても十分に安全かを判断するのに役立ちます。監査では、リリースが何を行うか、どの権限を要求するか、ファイル、アカウント、認証情報、コード、外部サービスへアクセスできるようになる前に追加で注意すべき点があるかを示します。

監査は強い安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、常に自分で判断してください。

関連項目: [セキュリティ](/clawhub/security)、[許容される利用](/ja-JP/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/clawhub/moderation)。

## インストール前に確認すること

インストール前に、以下を確認してください。

- 全体の監査ステータス
- リスクレベル
- 記載されている検出事項
- 必要な認証情報、権限、環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し、信頼できるコンテンツのみをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass` | 低リスクを超える明らかな問題は見つかりませんでした。 |
| `Review` | インストール前に検出事項を読んでください。リリースは正当なものかもしれません。 |
| `Warn` | 追加の注意が必要です。ClawHub が影響の大きい懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。 |
| `Pending` | 監査はまだ完了していません。 |
| `Error` | 監査を完了できませんでした。 |

`Pass` は安心材料になりますが、自分自身の判断の代わりにはなりません。これは、コンテンツを公開する、データを編集する、コマンドを実行する、ファイルを読む、本番システムにアクセスする、といったことができるツールで特に重要です。

## リスクレベル

リスクレベルは、想定どおりに使用した場合に、そのリリースがどれほどの権限を持つように見えるかという影響範囲を表します。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low` | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium` | リリースには、アカウントアクセスやデータ変更など、意味のある権限があります。 |
| `High` | リリースには、影響の大きい権限、重大な検出事項、または悪意のあるシグナルがあります。 |

リスクレベルと監査ステータスは、異なる問いに答えます。

- リスクレベルが問うのは「ここにどれほどの権限があるか」です。
- 監査ステータスが問うのは「この結果を受けて何をすべきか」です。

たとえば、公開用のスキルは `Medium` リスクで `Review` と表示されることがあります。これは悪意があるという意味ではありません。そのスキルが目的に沿っているように見える一方で、意味のあるアカウント権限で動作できることを意味します。

## 検出事項

検出事項は、監査結果が表示された理由を説明します。各検出事項には通常、以下が含まれます。

- それが何を意味するか
- フラグが立てられた理由
- 関連するスキルまたはPluginの内容
- 推奨事項

検出事項には `Info`、`Low`、`Medium`、`High`、`Critical` のラベルが付くことがあります。深刻度の高い検出事項ほど、リスクレベルと監査ステータスに強く影響します。

信頼度の低い検出事項は、ページが有用な証拠に集中できるように、公開監査の集計からは非表示になります。

## ClawHub が確認すること

ClawHub は提出されたリリース成果物を監査します。対象には以下が含まれます。

- スキルの指示またはPluginメタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能メタデータ

主な問いは一貫性です。名前、概要、メタデータ、要求される権限、実際の内容が、ユーザーが合理的に期待するものと一致しているかを確認します。

強力な動作は、それだけで悪いものではありません。多くの有用なツールは、認証情報、ローカルコマンド、プロバイダー API、パッケージインストールを必要とします。監査は、その権限が期待され、開示され、相応であるかを確認します。

成果物ページは、以下の完全な監査にリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは以下を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は、監査スタック内のマルウェアテレメトリとして VirusTotal を使用します。VirusTotal はファイルレピュテーションとマルウェアスキャンにおける信頼された業界標準であり、この提携により ClawHub はスキルとPluginのレビューに、より広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意ある成果物、エンジンの検出、ClawHub のエージェント認識レビューを補完するレピュテーションシグナルに特に有用です。ベンダーエンジン数が利用できる場合、監査ではそれらを次のような平易な言葉で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー数テレメトリがない場合、監査は次のように表示します。

```text
No VirusTotal findings
```

VirusTotal はあくまでテレメトリです。ClawHub 独自の成果物認識リスク分析の代わりにはなりません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に提供されます。各リリースを、エージェント向け成果物としてレビューします。対象は、指示、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpector の検出事項、VirusTotal テレメトリ、公開者が提供したコンテキストです。静的スキャンシグナルはこのレビューの内部コンテキストであり、独立した公開監査セクションやインストールをブロックする判定ではありません。

リスク分析は、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリまたはコンテキストの汚染、過剰な自律性などのリスクを見るための観点として、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/) を使用します。

ClawScan は、危険に見える機能を自動的に悪意あるものとして扱うことはありません。その機能が開示され、目的に沿っており、リリースで明記されたユースケースによって裏付けられているかを確認します。
