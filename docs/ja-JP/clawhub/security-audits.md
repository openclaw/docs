# セキュリティ監査

ClawHub のセキュリティ監査は、スキルやプラグインをインストールしても十分に安全かどうかを判断する助けになります。リリースが何をするのか、どの権限を要求するのか、ファイル、アカウント、認証情報、コード、外部サービスへアクセスできるようになる前に追加の注意が必要な点があるかを示します。

監査は強い安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、必ず自分で判断してください。

[セキュリティ](/ja-JP/clawhub/security)、[許容される利用](/ja-JP/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/ja-JP/clawhub/moderation)も参照してください。

## インストール前に確認すること

インストール前に、次を確認してください。

- 全体の監査ステータス
- リスクレベル
- 記載されている所見
- 必要な認証情報、権限、環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し信頼できるコンテンツだけをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass` | 低リスクを超える目に見える問題は見つかりませんでした。 |
| `Review` | インストール前に所見を読んでください。リリースは正当なものかもしれません。 |
| `Warn` | 追加の注意を払ってください。ClawHub は影響の大きい懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。 |
| `Pending` | 監査はまだ完了していません。 |
| `Error` | 監査を完了できませんでした。 |

`Pass` は安心材料になりますが、自分の判断に置き換わるものではありません。これは、コンテンツを公開する、データを編集する、コマンドを実行する、ファイルを読む、本番システムへアクセスする、といったことができるツールで特に重要です。

## リスクレベル

リスクレベルは影響範囲を表します。意図どおりに使用した場合に、そのリリースがどれだけの力を持つように見えるかを示します。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low` | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium` | アカウントアクセスやデータ変更など、意味のある権限をリリースが持っています。 |
| `High` | リリースに影響の大きい権限、重大な所見、または悪意のあるシグナルがあります。 |

リスクレベルと監査ステータスは別の問いに答えます。

- リスクレベルは問います: 「ここにはどれだけの力があるか?」
- 監査ステータスは問います: 「この結果に対して何をすべきか?」

たとえば、公開用のスキルは `Medium` リスクで `Review` と表示される場合があります。それは悪意があるという意味ではありません。そのスキルが目的に沿っているように見える一方で、意味のあるアカウント権限で動作できるという意味です。

## 所見

所見は、監査結果が表示された理由を説明します。各所見には通常、次が含まれます。

- それが何を意味するか
- フラグ付けされた理由
- 関連するスキルまたはプラグインの内容
- 推奨事項

所見には `Info`、`Low`、`Medium`、`High`、`Critical` のラベルが付く場合があります。重大度の高い所見ほど、リスクレベルと監査ステータスにより強く影響します。

信頼度の低い所見は公開監査の集約から隠されるため、ページは有用な証拠に集中した状態に保たれます。

## ClawHub が確認すること

ClawHub は提出されたリリース成果物を監査します。これには次が含まれます。

- スキルの手順またはプラグインのメタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能メタデータ

主な問いは整合性です。名前、概要、メタデータ、要求された権限、実際の内容が、ユーザーが合理的に期待するものと一致しているかどうかです。

強力な動作は自動的に悪いわけではありません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダー API、パッケージインストールが必要です。監査は、その力が期待され、開示され、釣り合っているかを確認します。

成果物ページは、次の完全な監査にリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせています。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は監査スタックのマルウェアテレメトリとして VirusTotal を使用します。VirusTotal はファイルの評判とマルウェアスキャンにおける信頼された業界標準であり、この提携により ClawHub はスキルとプラグインのレビューに、より広いセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意ある成果物、エンジンの検出、ClawHub のエージェントを考慮したレビューを補完する評判シグナルに特に有用です。ベンダーエンジン数が利用できる場合、監査はそれを次のような平易な表現で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー数テレメトリがない場合、監査は次のように表示します。

```text
No VirusTotal findings
```

VirusTotal はあくまでテレメトリです。ClawHub 独自の成果物を考慮したリスク分析に置き換わるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に支えられています。各リリースを、エージェント向け成果物としてレビューします。手順、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpector の所見、VirusTotal テレメトリ、公開者が提供したコンテキストが対象です。静的スキャンシグナルはこのレビューの内部コンテキストであり、独立した公開監査セクションやインストールをブロックする判定ではありません。

リスク分析では、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリやコンテキストの汚染、過剰なエージェンシーなどのリスクを見る観点として、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)を使用します。

ClawScan は、恐ろしく見える機能を自動的に悪意あるものとは扱いません。その機能が開示され、目的に沿っており、リリースで述べられているユースケースに支えられているかを問います。
