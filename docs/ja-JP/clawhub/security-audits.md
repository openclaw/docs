# セキュリティ監査

ClawHub のセキュリティ監査は、スキルまたはプラグインをインストールしても十分に安全かどうかを判断する助けになります。監査では、リリースが何を行うか、どの権限を要求するか、ファイル、アカウント、認証情報、コード、外部サービスへアクセスする前に追加の注意が必要な点があるかを示します。

監査は強い安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、必ず自分で判断してください。

[セキュリティ](/ja-JP/clawhub/security)、[許容される利用](/ja-JP/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/ja-JP/clawhub/moderation) も参照してください。

## インストール前に確認すること

インストール前に、次を確認してください。

- 全体的な監査ステータス
- リスクレベル
- 記載されている所見
- 必要な認証情報、権限、または環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し信頼できるコンテンツのみをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 低リスクを超える可視の問題は見つかりませんでした。                                |
| `Review`    | インストール前に所見を読んでください。そのリリースは正当なものの可能性があります。 |
| `Warn`      | 特に注意してください。ClawHub が影響度の高い懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。                                                           |
| `Pending`   | 監査はまだ完了していません。                                             |
| `Error`     | 監査を完了できませんでした。                                         |

`Pass` は安心材料になりますが、自分自身の判断に代わるものではありません。これは、コンテンツを公開したり、データを編集したり、コマンドを実行したり、ファイルを読んだり、本番システムへアクセスしたりできるツールで特に重要です。

## リスクレベル

リスクレベルは影響範囲を表します。つまり、そのリリースを意図どおりに使った場合に、どれほど大きな権限を持つように見えるかです。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。                          |
| `Medium`   | そのリリースには、アカウントアクセスやデータ変更など、意味のある権限があります。 |
| `High`     | そのリリースには、影響度の高い権限、重大な所見、または悪意あるシグナルがあります。 |

リスクレベルと監査ステータスは、異なる問いに答えるものです。

- リスクレベルが問うのは、「ここにはどれほどの権限があるか」です。
- 監査ステータスが問うのは、「この結果に対して何をすべきか」です。

たとえば、公開用のスキルは `Medium` リスクで `Review` と表示される場合があります。これは悪意があることを意味しません。そのスキルは目的に沿っているように見えるものの、意味のあるアカウント権限で動作できるという意味です。

## 所見

所見は、なぜその監査結果が表示されたのかを説明します。各所見には通常、次が含まれます。

- それが何を意味するか
- なぜフラグ付けされたか
- 関連するスキルまたはプラグインの内容
- 推奨事項

所見には `Info`、`Low`、`Medium`、`High`、または `Critical` のラベルが付く場合があります。深刻度が高い所見ほど、リスクレベルと監査ステータスにより強く影響します。

信頼度の低い所見は、ページが有用な証拠に集中できるよう、公開監査の集約表示では非表示になります。

## ClawHub が確認すること

ClawHub は、提出されたリリース成果物を監査します。対象には次が含まれます。

- スキルの指示またはプラグインのメタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能メタデータ

主な問いは一貫性です。名前、概要、メタデータ、要求された権限、実際の内容が、ユーザーが合理的に期待するものと一致しているかどうかです。

強力な動作は自動的に悪いものではありません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダー API、またはパッケージインストールが必要です。監査では、その権限が期待され、開示され、釣り合っているかを確認します。

成果物ページは、次の完全な監査へリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は、監査スタック内のマルウェアテレメトリとして VirusTotal を使用します。VirusTotal はファイルの評判とマルウェアスキャンに関する信頼された業界標準であり、私たちの提携により、ClawHub はスキルとプラグインのレビューにより広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意ある成果物、エンジンの検出、ClawHub のエージェント認識レビューを補完する評判シグナルに特に有用です。ベンダーエンジンの件数が利用できる場合、監査では次のように平易な言葉で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー件数テレメトリがない場合、監査では次のように表示されます。

```text
No VirusTotal findings
```

VirusTotal はあくまでテレメトリです。ClawHub 独自の成果物認識リスク分析に代わるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に支えられています。これは各リリースを、エージェント向けの成果物としてレビューします。対象は、指示、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpector の所見、VirusTotal テレメトリ、公開者が提供したコンテキストです。静的スキャンシグナルは、このレビューのための内部コンテキストです。独立した公開監査セクションでも、インストールをブロックする判定でもありません。

リスク分析は、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリまたはコンテキストの汚染、過剰な自律性などのリスクを見るためのレンズとして、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/) を使用します。

ClawScan は、見た目が危険そうな機能を自動的に悪意あるものとは扱いません。その機能が開示され、目的に沿っており、リリースが述べているユースケースによって裏付けられているかを確認します。
