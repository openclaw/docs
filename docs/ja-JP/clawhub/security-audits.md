# セキュリティ監査

ClawHubのセキュリティ監査は、スキルまたはPluginがインストールしても十分に安全かどうかを判断するのに役立ちます。監査では、リリースが何をするか、どの権限を求めるか、ファイル、アカウント、認証情報、コード、または外部サービスにアクセスする前に追加の注意が必要なものがあるかを示します。

監査は強力な安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、必ず自分で判断してください。

[セキュリティ](/clawhub/security)、[許容される利用](/ja-JP/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/clawhub/moderation)も参照してください。

## インストール前に確認すること

インストールする前に、以下を確認してください。

- 全体の監査ステータス
- リスクレベル
- 記載されている所見
- 必要な認証情報、権限、または環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し信頼できるコンテンツだけをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 低リスクを上回る明らかな問題は見つかりませんでした。 |
| `Review`    | インストール前に所見を読んでください。そのリリースは正当なものかもしれません。 |
| `Warn`      | 特に注意してください。ClawHubが影響の大きい懸念または警告シグナルを見つけました。 |
| `Malicious` | インストールしないでください。 |
| `Pending`   | 監査はまだ完了していません。 |
| `Error`     | 監査を完了できませんでした。 |

`Pass`は安心材料になりますが、自分自身の判断に置き換わるものではありません。これは、コンテンツを公開する、データを編集する、コマンドを実行する、ファイルを読む、または本番システムにアクセスするツールで最も重要です。

## リスクレベル

リスクレベルは影響範囲を表します。意図どおりに使用した場合に、そのリリースがどれほどの力を持つように見えるかを示します。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium`   | そのリリースは、アカウントアクセスやデータ変更など、意味のある権限を持ちます。 |
| `High`     | そのリリースは、影響の大きい権限、重大な所見、または悪意のあるシグナルを持ちます。 |

リスクレベルと監査ステータスは異なる質問に答えます。

- リスクレベルが問うのは、「ここにはどれほどの力があるか」です。
- 監査ステータスが問うのは、「この結果に対して何をすべきか」です。

たとえば、公開用スキルは`Medium`リスクで`Review`と表示されることがあります。これは、それが悪意のあるものだという意味ではありません。そのスキルが目的に沿っているように見える一方で、意味のあるアカウント権限で動作できることを意味します。

## 所見

所見は、なぜその監査結果が表示されたかを説明します。各所見には通常、以下が含まれます。

- それが何を意味するか
- なぜフラグが立てられたか
- 関連するスキルまたはPluginのコンテンツ
- 推奨事項

所見には`Info`、`Low`、`Medium`、`High`、または`Critical`のラベルが付く場合があります。深刻度の高い所見ほど、リスクレベルと監査ステータスにより強く影響します。

信頼度の低い所見は公開監査の集計から非表示にされるため、ページは有用な証拠に集中できます。

## ClawHubが確認すること

ClawHubは、提出されたリリース成果物を監査します。対象には以下が含まれます。

- スキル手順またはPluginメタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性とケイパビリティのメタデータ

主な問いは一貫性です。名前、概要、メタデータ、要求される権限、実際のコンテンツが、ユーザーが合理的に期待する内容と一致しているかを確認します。

強力な動作は自動的に悪いものではありません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダーAPI、またはパッケージインストールが必要です。監査では、その力が期待され、開示され、相応であるかを確認します。

成果物ページは、完全な監査へ以下でリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは以下を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHubは監査スタック内のマルウェアテレメトリとしてVirusTotalを使用します。VirusTotalはファイル評判とマルウェアスキャンの信頼された業界標準であり、このパートナーシップにより、ClawHubはスキルとPluginのレビューにより広範なセキュリティインテリジェンスを追加できます。

VirusTotalは、既知の悪意ある成果物、エンジン検出、ClawHubのエージェント対応レビューを補完する評判シグナルに特に有用です。ベンダーエンジン数が利用できる場合、監査はそれらを平易な表現で要約します。たとえば、次のようになります。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHubに要約できるベンダー数テレメトリがない場合、監査は次のように表示します。

```text
No VirusTotal findings
```

VirusTotalはあくまでテレメトリです。ClawHub独自の成果物対応リスク分析に置き換わるものではありません。

## リスク分析

リスク分析は、ClawHub独自のセキュリティ監査システムであるClawScanによって内部的に支えられています。各リリースをエージェント向け成果物としてレビューします。対象には、手順、メタデータ、宣言された権限、ファイル、ケイパビリティシグナル、静的スキャンシグナル、SkillSpectorの所見、VirusTotalテレメトリ、公開者が提供したコンテキストが含まれます。静的スキャンシグナルはこのレビューの内部コンテキストであり、独立した公開監査セクションやインストールをブロックする判定ではありません。

リスク分析は、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリまたはコンテキストのポイズニング、過剰な自律性などのリスクを見るレンズとして、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)を使用します。

ClawScanは、見た目が怖いケイパビリティを自動的に悪意あるものとは扱いません。そのケイパビリティが開示されているか、目的に沿っているか、リリースで示されたユースケースによって裏付けられているかを確認します。
