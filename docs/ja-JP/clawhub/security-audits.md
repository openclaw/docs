# セキュリティ監査

ClawHub のセキュリティ監査は、スキルまたは Plugin をインストールしても十分に安全かどうかを判断するのに役立ちます。リリースが何を行うか、どの権限を求めるか、ファイル、アカウント、認証情報、コード、外部サービスにアクセスできるようになる前に追加の注意が必要なものがあるかどうかを示します。

監査は強い安全性のシグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、必ず自分で判断してください。

[セキュリティ](/clawhub/security)、[許容される使用](/ja-JP/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/clawhub/moderation)も参照してください。

## インストール前に確認すること

インストールする前に、次を確認してください。

- 全体的な監査ステータス
- リスクレベル
- 記載されている検出事項
- 必要な認証情報、権限、または環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し、信頼できるコンテンツだけをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass` | 低リスクを超える可視の問題は見つかりませんでした。 |
| `Review` | インストール前に検出事項を読んでください。リリースは正当なものである可能性があります。 |
| `Warn` | 特に注意してください。ClawHub が影響の大きい懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。 |
| `Pending` | 監査はまだ完了していません。 |
| `Error` | 監査を完了できませんでした。 |

`Pass` は安心材料になりますが、自分自身の判断に代わるものではありません。これは、コンテンツを公開したり、データを編集したり、コマンドを実行したり、ファイルを読んだり、本番システムにアクセスしたりできるツールで特に重要です。

## リスクレベル

リスクレベルは、ブラスト半径、つまり意図どおりに使用した場合にリリースが持つと思われる力の大きさを表します。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low` | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium` | リリースには、アカウントアクセスやデータ変更などの意味のある権限があります。 |
| `High` | リリースには、影響の大きい権限、重大な検出事項、または悪意のあるシグナルがあります。 |

リスクレベルと監査ステータスは、異なる問いに答えます。

- リスクレベルは「ここにはどれだけの力があるか」を問います。
- 監査ステータスは「この結果をどう扱うべきか」を問います。

たとえば、公開用のスキルは `Medium` リスクで `Review` と表示されることがあります。これは悪意があるという意味ではありません。そのスキルが目的に沿っているように見える一方で、意味のあるアカウント権限で動作できることを意味します。

## 検出事項

検出事項は、監査結果が表示された理由を説明します。各検出事項には通常、次が含まれます。

- それが意味すること
- フラグされた理由
- 関連するスキルまたは Plugin のコンテンツ
- 推奨事項

検出事項には `Info`、`Low`、`Medium`、`High`、または `Critical` のラベルが付くことがあります。重大度の高い検出事項ほど、リスクレベルと監査ステータスに強く影響します。

信頼度の低い検出事項は公開監査ロールアップから非表示にされるため、ページは有用な証拠に集中できます。

## ClawHub が確認すること

ClawHub は、提出されたリリース成果物を監査します。これには次が含まれます。

- スキルの指示または Plugin メタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能メタデータ

主な問いは一貫性です。名前、概要、メタデータ、要求される権限、実際のコンテンツが、ユーザーが合理的に期待する内容と合っているかどうかです。

強力な動作は、それだけで悪いわけではありません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダー API、またはパッケージのインストールが必要です。監査は、その力が予期され、開示され、目的に見合っているかを確認します。

成果物ページは、次の完全な監査にリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は監査スタックのマルウェアテレメトリとして VirusTotal を使用します。VirusTotal はファイルレピュテーションとマルウェアスキャンの信頼された業界標準であり、このパートナーシップにより ClawHub はスキルと Plugin のレビューにより広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意ある成果物、エンジンのヒット、ClawHub のエージェント対応レビューを補完するレピュテーションシグナルに特に有用です。ベンダーエンジン数を利用できる場合、監査は次のように平易な表現で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー数テレメトリがない場合、監査は次のように表示します。

```text
No VirusTotal findings
```

VirusTotal はあくまでテレメトリです。ClawHub 独自の成果物対応リスク分析に代わるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に支えられています。各リリースをエージェント向け成果物としてレビューします。対象には、指示、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpector の検出事項、VirusTotal テレメトリ、公開者が提供したコンテキストが含まれます。静的スキャンシグナルは、このレビューのための内部コンテキストです。独立した公開監査セクションや、インストールをブロックする判定ではありません。

リスク分析は、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリまたはコンテキストの汚染、過剰なエージェンシーなどのリスクを捉える観点として、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)を使用します。

ClawScan は、見た目が危険そうな機能を自動的に悪意あるものとは扱いません。その機能が開示され、目的に沿っており、リリースで示されたユースケースによって裏付けられているかを確認します。
