# セキュリティ監査

ClawHub のセキュリティ監査は、スキルやプラグインがインストールするのに十分安全かどうかを判断するのに役立ちます。リリースが何を行うか、どの権限を求めるか、ファイル、アカウント、認証情報、コード、外部サービスにアクセスできるようになる前に追加の注意が必要な点があるかどうかを示します。

監査は強い安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、必ず自分で判断してください。

[セキュリティ](/ja-JP/clawhub/security)、[許容される利用](/ja-JP/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/ja-JP/clawhub/moderation)も参照してください。

## インストール前に確認すること

インストールする前に、次を確認してください。

- 全体的な監査ステータス
- リスクレベル
- 記載されている検出事項
- 必要な認証情報、権限、または環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し信頼できるコンテンツのみをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 低リスクを超える目に見える問題は見つかりませんでした。 |
| `Review`    | インストール前に検出事項を読んでください。リリースは正当なものである可能性があります。 |
| `Warn`      | 特に注意してください。ClawHub が影響の大きい懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。 |
| `Pending`   | 監査はまだ完了していません。 |
| `Error`     | 監査を完了できませんでした。 |

`Pass` は安心材料になりますが、自分自身の判断の代わりにはなりません。これは、コンテンツの公開、データの編集、コマンドの実行、ファイルの読み取り、本番システムへのアクセスが可能なツールで特に重要です。

## リスクレベル

リスクレベルは影響範囲を表します。つまり、意図どおりに使用した場合に、そのリリースがどの程度の力を持つように見えるかです。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium`   | リリースには、アカウントアクセスやデータ変更など、意味のある権限があります。 |
| `High`     | リリースには、影響の大きい権限、重大な検出事項、または悪意のあるシグナルがあります。 |

リスクレベルと監査ステータスは異なる問いに答えます。

- リスクレベルが問うのは、「ここにはどれだけの力があるか？」です。
- 監査ステータスが問うのは、「この結果をどう扱うべきか？」です。

たとえば、公開用スキルは `Medium` リスクで `Review` と表示されることがあります。これは悪意があるという意味ではありません。そのスキルは目的に沿っているように見えるものの、意味のあるアカウント権限で動作できるという意味です。

## 検出事項

検出事項は、監査結果が表示された理由を説明します。各検出事項には通常、次が含まれます。

- それが何を意味するか
- フラグが付けられた理由
- 関連するスキルまたはプラグインのコンテンツ
- 推奨事項

検出事項には `Info`、`Low`、`Medium`、`High`、`Critical` のラベルが付く場合があります。重大度の高い検出事項ほど、リスクレベルと監査ステータスに強く影響します。

信頼度の低い検出事項は、ページが有用な証拠に集中できるように、公開監査ロールアップでは非表示になります。

## ClawHub が確認すること

ClawHub は提出されたリリースアーティファクトを監査します。対象には次が含まれます。

- スキルの指示またはプラグインのメタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能メタデータ

主な問いは一貫性です。名前、概要、メタデータ、要求された権限、実際のコンテンツが、ユーザーが合理的に期待する内容と一致しているかどうかです。

強力な動作は自動的に悪いものではありません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダー API、またはパッケージのインストールが必要です。監査では、その力が期待され、開示され、釣り合っているかどうかを確認します。

アーティファクトページは、次の完全な監査にリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は、監査スタック内のマルウェアテレメトリとして VirusTotal を使用します。VirusTotal はファイルの評価とマルウェアスキャンにおける信頼された業界標準であり、この連携により ClawHub はスキルとプラグインのレビューに、より広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意あるアーティファクト、エンジンの検出、ClawHub のエージェントを考慮したレビューを補完する評価シグナルに特に有用です。ベンダーエンジン数が利用できる場合、監査はそれを次のような平易な表現で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または次のように表示します。

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー数テレメトリがない場合、監査は次のように表示します。

```text
No VirusTotal findings
```

VirusTotal はあくまでテレメトリです。ClawHub 独自の、アーティファクトを考慮したリスク分析の代わりにはなりません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に実行されます。各リリースをエージェント向けアーティファクトとしてレビューします。対象は、指示、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpector の検出事項、VirusTotal テレメトリ、公開者が提供したコンテキストです。静的スキャンシグナルは、このレビューの内部コンテキストです。単独の公開監査セクションや、インストールをブロックする判定ではありません。

リスク分析では、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリまたはコンテキストのポイズニング、過剰な自律性などのリスクを見るための観点として、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)を使用します。

ClawScan は、見た目が危険そうな機能を自動的に悪意あるものとは扱いません。その機能が開示され、目的に沿っており、リリースで示されたユースケースによって裏付けられているかを問います。
