# セキュリティ監査

ClawHub のセキュリティ監査は、スキルやプラグインをインストールしてよいほど安全かどうかを判断するのに役立ちます。監査では、リリースが何をするのか、どの権限を要求するのか、ファイル、アカウント、認証情報、コード、外部サービスにアクセスする前に追加の注意が必要な点があるかどうかが示されます。

監査は強力な安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、必ず自身で判断してください。

[セキュリティ](/ja-JP/clawhub/security)、[許容される利用](/ja-JP/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/ja-JP/clawhub/moderation)も参照してください。

## インストール前に確認すること

インストール前に、次を確認してください。

- 全体の監査ステータス
- リスクレベル
- 記載されている所見
- 必要な認証情報、権限、環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し信頼できるコンテンツだけをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 低リスクを超える目に見える問題は見つかりませんでした。 |
| `Review`    | インストール前に所見を読んでください。リリースは正当なものである可能性があります。 |
| `Warn`      | 追加の注意を払ってください。ClawHub が影響の大きい懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。 |
| `Pending`   | 監査はまだ完了していません。 |
| `Error`     | 監査を完了できませんでした。 |

`Pass` は安心材料になりますが、自身の判断に代わるものではありません。これは、コンテンツを公開する、データを編集する、コマンドを実行する、ファイルを読む、本番システムにアクセスするといったことができるツールで特に重要です。

## リスクレベル

リスクレベルは影響範囲を表します。つまり、意図どおりに使用した場合に、そのリリースがどれほどの力を持つように見えるかです。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium`   | そのリリースには、アカウントアクセスやデータ変更などの意味のある権限があります。 |
| `High`     | そのリリースには、影響の大きい権限、重大な所見、または悪意のあるシグナルがあります。 |

リスクレベルと監査ステータスは別の問いに答えます。

- リスクレベルが問うこと: 「ここにはどれほどの力があるか」
- 監査ステータスが問うこと: 「この結果をどう扱うべきか」

たとえば、公開用のスキルは `Medium` リスクで `Review` と表示される場合があります。これは悪意があるという意味ではありません。そのスキルが目的に沿っているように見える一方で、意味のあるアカウント権限で動作できるという意味です。

## 所見

所見は、監査結果が表示された理由を説明します。各所見には通常、次が含まれます。

- それが何を意味するか
- フラグが立てられた理由
- 関連するスキルまたはプラグインの内容
- 推奨事項

所見には `Info`、`Low`、`Medium`、`High`、`Critical` のラベルが付くことがあります。深刻度が高い所見ほど、リスクレベルと監査ステータスに強く影響します。

信頼度の低い所見は公開監査の集計から非表示になり、ページが有用な証拠に集中できるようにしています。

## ClawHub が確認すること

ClawHub は、提出されたリリース成果物を監査します。対象には次が含まれます。

- スキルの指示またはプラグインのメタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能のメタデータ

主な問いは一貫性です。名前、概要、メタデータ、要求される権限、実際の内容が、ユーザーが合理的に期待するものと一致しているかどうかです。

強力な動作が自動的に悪いわけではありません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダー API、パッケージのインストールが必要です。監査では、その力が期待され、開示され、相応であるかを確認します。

成果物ページは、次の完全な監査にリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は、監査スタック内のマルウェアテレメトリとして VirusTotal を使用します。VirusTotal はファイルレピュテーションとマルウェアスキャンの信頼された業界標準であり、このパートナーシップにより、ClawHub はスキルとプラグインのレビューにより広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意ある成果物、エンジンの検出、ClawHub のエージェント対応レビューを補完するレピュテーションシグナルに特に有用です。ベンダーエンジンの件数が利用できる場合、監査はそれらを次のような平易な言葉で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー件数テレメトリがない場合、監査は次のように表示します。

```text
No VirusTotal findings
```

VirusTotal はあくまでテレメトリです。ClawHub 独自の成果物対応リスク分析に代わるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に実行されます。各リリースをエージェント向け成果物としてレビューします。対象には、指示、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpector の所見、VirusTotal テレメトリ、公開者が提供したコンテキストが含まれます。静的スキャンシグナルはこのレビューのための内部コンテキストであり、独立した公開監査セクションやインストールをブロックする判定ではありません。

リスク分析では、プロンプトインジェクション、ツールの誤用、認証情報の漏えい、安全でない実行、メモリまたはコンテキストの汚染、過剰なエージェンシーなどのリスクを捉える視点として、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/) を使用します。

ClawScan は、見た目が危険そうな機能を自動的に悪意あるものとは扱いません。その機能が開示され、目的に沿っており、そのリリースで述べられたユースケースによって裏付けられているかを確認します。
