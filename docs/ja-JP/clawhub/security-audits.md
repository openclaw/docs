# セキュリティ監査

ClawHub のセキュリティ監査は、skill または plugin がインストールするのに十分安全かどうかを判断するのに役立ちます。監査では、リリースが何を行うか、どの権限を要求するか、ファイル、アカウント、認証情報、コード、外部サービスにアクセスする前に追加の注意が必要な点があるかどうかを示します。

監査は強力な安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、常に自分で判断してください。

[セキュリティ](/clawhub/security)、[許容される利用](/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/clawhub/moderation) も参照してください。

## インストール前に確認すること

インストール前に、次を確認してください。

- 全体的な監査ステータス
- リスクレベル
- 記載されているすべての所見
- 必要な認証情報、権限、環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し信頼できるコンテンツのみをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass` | 低リスクを超える目に見える問題は見つかりませんでした。 |
| `Review` | インストール前に所見を読んでください。リリースは正当なものかもしれません。 |
| `Warn` | 特に注意してください。ClawHub が影響の大きい懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。 |
| `Pending` | 監査はまだ完了していません。 |
| `Error` | 監査を完了できませんでした。 |

`Pass` は安心材料になりますが、自分自身の判断に代わるものではありません。これは、コンテンツを公開したり、データを編集したり、コマンドを実行したり、ファイルを読んだり、本番システムにアクセスしたりできるツールで特に重要です。

## リスクレベル

リスクレベルは影響範囲を示します。つまり、意図どおりに使用した場合に、そのリリースがどれほど大きな権限を持つように見えるかです。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low` | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium` | リリースには、アカウントアクセスやデータ変更など、意味のある権限があります。 |
| `High` | リリースには、影響の大きい権限、重大な所見、または悪意のあるシグナルがあります。 |

リスクレベルと監査ステータスは、異なる問いに答えます。

- リスクレベルが問うのは「ここにはどれほどの権限があるか」です。
- 監査ステータスが問うのは「この結果をどう扱うべきか」です。

たとえば、公開用の skill は `Medium` リスクで `Review` と表示されることがあります。これは悪意があるという意味ではありません。その skill が目的に沿っているように見える一方で、意味のあるアカウント権限で動作できることを意味します。

## 所見

所見は、監査結果が表示された理由を説明します。各所見には通常、次が含まれます。

- それが何を意味するか
- フラグ付けされた理由
- 関連する skill または plugin のコンテンツ
- 推奨事項

所見には `Info`、`Low`、`Medium`、`High`、`Critical` のラベルが付くことがあります。重大度が高い所見ほど、リスクレベルと監査ステータスにより強く影響します。

信頼度の低い所見は公開監査ロールアップから非表示になり、ページが有用な証拠に集中できるようになっています。

## ClawHub が確認すること

ClawHub は、提出されたリリース成果物を監査します。これには次が含まれます。

- skill の指示または plugin メタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能メタデータ

主な問いは一貫性です。名前、概要、メタデータ、要求された権限、実際のコンテンツが、ユーザーが合理的に期待する内容と一致しているかどうかです。

強力な動作が自動的に悪いわけではありません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダー API、パッケージインストールが必要です。監査では、その権限が期待され、開示され、相応であるかを確認します。

成果物ページは、次の完全な監査にリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせています。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は監査スタック内のマルウェアテレメトリとして VirusTotal を使用します。VirusTotal はファイルレピュテーションとマルウェアスキャンの信頼された業界標準であり、このパートナーシップにより、ClawHub は skill と plugin のレビューにより広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意ある成果物、エンジン検出、ClawHub のエージェント認識レビューを補完するレピュテーションシグナルに特に有用です。ベンダーエンジン数が利用できる場合、監査では次のように平易な表現で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー数テレメトリがない場合、監査では次のように表示されます。

```text
No VirusTotal findings
```

VirusTotal はあくまでテレメトリです。ClawHub 独自の成果物認識リスク分析に代わるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に提供されています。各リリースを、エージェント向け成果物としてレビューします。対象には、指示、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpector の所見、VirusTotal テレメトリ、公開者が提供したコンテキストが含まれます。静的スキャンシグナルはこのレビューの内部コンテキストであり、独立した公開監査セクションやインストールをブロックする判定ではありません。

リスク分析では、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリまたはコンテキスト汚染、過剰なエージェンシーなどのリスクを見るための観点として、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/) を使用します。

ClawScan は、危険に見える機能を自動的に悪意あるものとして扱うことはありません。その機能が開示され、目的に沿っており、リリースで述べられたユースケースによって裏付けられているかを確認します。
