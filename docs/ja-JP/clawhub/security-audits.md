# セキュリティ監査

ClawHub のセキュリティ監査は、スキルやプラグインをインストールしても十分に安全かどうかを判断するのに役立ちます。リリースが何を行うか、どの権限を求めるか、ファイル、アカウント、認証情報、コード、外部サービスへアクセスする前に追加の注意が必要なものがあるかを示します。

監査は強力な安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、必ず自分で判断してください。

[セキュリティ](/clawhub/security)、[許容される利用](/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/clawhub/moderation)も参照してください。

## インストール前に確認すること

インストール前に、次を確認してください。

- 全体の監査ステータス
- リスクレベル
- 記載されている所見
- 必要な認証情報、権限、環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し信頼できるコンテンツのみをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass` | 低リスクを超える目に見える問題は見つかりませんでした。 |
| `Review` | インストール前に所見を読んでください。リリースは正当なものかもしれません。 |
| `Warn` | 追加の注意を払ってください。ClawHub が影響の大きい懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。 |
| `Pending` | 監査はまだ完了していません。 |
| `Error` | 監査を完了できませんでした。 |

`Pass` は安心材料になりますが、自分自身の判断の代わりにはなりません。これは、コンテンツの公開、データの編集、コマンドの実行、ファイルの読み取り、本番システムへのアクセスができるツールで特に重要です。

## リスクレベル

リスクレベルはブラスト半径、つまり意図どおりに使用した場合に、そのリリースがどれだけの力を持つように見えるかを表します。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low` | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium` | そのリリースには、アカウントアクセスやデータ変更など、意味のある権限があります。 |
| `High` | そのリリースには、影響の大きい権限、重大な所見、または悪意のあるシグナルがあります。 |

リスクレベルと監査ステータスは異なる問いに答えます。

- リスクレベルが問うのは、「ここにはどれだけの力があるか？」
- 監査ステータスが問うのは、「この結果を受けて何をすべきか？」

たとえば、公開用のスキルは `Medium` リスクで `Review` と表示される場合があります。これは悪意があるという意味ではありません。そのスキルが目的に沿っているように見える一方で、意味のあるアカウント権限で動作できることを意味します。

## 所見

所見は、監査結果が表示された理由を説明します。各所見には通常、次が含まれます。

- それが何を意味するか
- フラグ付けされた理由
- 関連するスキルまたはプラグインのコンテンツ
- 推奨事項

所見には `Info`、`Low`、`Medium`、`High`、または `Critical` のラベルが付くことがあります。重大度が高い所見ほど、リスクレベルと監査ステータスにより強く影響します。

信頼度の低い所見は公開監査ロールアップから非表示にされるため、ページは有用な証拠に集中したままになります。

## ClawHub が確認すること

ClawHub は提出されたリリース成果物を監査します。対象には次が含まれます。

- スキルの手順またはプラグインのメタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能メタデータ

主な問いは一貫性です。名前、概要、メタデータ、要求される権限、実際のコンテンツが、ユーザーが合理的に期待する内容と一致しているかどうかです。

強力な動作は自動的に悪いものではありません。有用なツールの多くは、認証情報、ローカルコマンド、プロバイダー API、またはパッケージのインストールを必要とします。監査は、その力が想定され、開示され、釣り合いが取れているかを確認します。

成果物ページは、次の完全な監査にリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は監査スタック内のマルウェアテレメトリとして VirusTotal を使用します。VirusTotal はファイル評価とマルウェアスキャンにおける信頼された業界標準であり、この提携により ClawHub はスキルとプラグインのレビューに、より広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意ある成果物、エンジン検出、ClawHub のエージェント対応レビューを補完する評価シグナルに特に役立ちます。ベンダーエンジン数が利用可能な場合、監査は次のように平易な言葉で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー数テレメトリがない場合、監査には次のように表示されます。

```text
No VirusTotal findings
```

VirusTotal はテレメトリであり続けます。ClawHub 独自の成果物対応リスク分析の代わりにはなりません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に支えられています。各リリースをエージェント向け成果物としてレビューします。対象には、手順、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpector の所見、VirusTotal テレメトリ、公開者が提供したコンテキストが含まれます。静的スキャンシグナルは、このレビューの内部コンテキストです。独立した公開監査セクションや、インストールをブロックする判定ではありません。

リスク分析は、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリまたはコンテキストのポイズニング、過剰な自律性などのリスクを見るためのレンズとして、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)を使用します。

ClawScan は、見た目が危険そうな機能を自動的に悪意あるものとして扱いません。その機能が開示され、目的に沿っており、リリースの表明されたユースケースによって裏付けられているかを確認します。
