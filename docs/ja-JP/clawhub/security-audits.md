# セキュリティ監査

ClawHub のセキュリティ監査は、スキルやプラグインをインストールしても十分に安全かどうかを判断するのに役立ちます。監査では、リリースが何を行うか、どの権限を要求するか、ファイル、アカウント、認証情報、コード、外部サービスにアクセスする前に追加の注意が必要な点があるかどうかを示します。

監査は強い安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、常に自分で判断してください。

関連項目: [セキュリティ](/clawhub/security)、[許容される利用](/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/clawhub/moderation)。

## インストール前に確認すること

インストール前に、次を確認してください。

- 全体の監査ステータス
- リスクレベル
- 記載されている検出事項
- 必要な認証情報、権限、環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し、信頼できるコンテンツだけをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 低リスクを超える可視の問題は見つかりませんでした。                                |
| `Review`    | インストール前に検出事項を読んでください。リリースは正当なものである可能性があります。 |
| `Warn`      | 追加の注意を払ってください。ClawHub が高影響の懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。                                                           |
| `Pending`   | 監査はまだ完了していません。                                             |
| `Error`     | 監査を完了できませんでした。                                         |

`Pass` は安心材料になりますが、自分自身の判断に代わるものではありません。これは、コンテンツを公開する、データを編集する、コマンドを実行する、ファイルを読む、本番システムにアクセスする、といったことができるツールで特に重要です。

## リスクレベル

リスクレベルはブラスト半径、つまり意図された方法で使用した場合にそのリリースがどれだけの力を持つように見えるかを表します。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。                          |
| `Medium`   | そのリリースには、アカウントアクセスやデータ変更などの意味のある権限があります。 |
| `High`     | そのリリースには、高影響の権限、重大な検出事項、または悪意のあるシグナルがあります。 |

リスクレベルと監査ステータスは、別々の問いに答えます。

- リスクレベルが問うのは「ここにはどれだけの力があるか？」
- 監査ステータスが問うのは「この結果をどう扱うべきか？」

たとえば、公開用スキルは `Medium` リスクで `Review` と表示される場合があります。これは悪意があるという意味ではありません。そのスキルが目的に沿っているように見える一方で、意味のあるアカウント権限で動作できることを意味します。

## 検出事項

検出事項は、監査結果が表示された理由を説明します。各検出事項には通常、次が含まれます。

- それが何を意味するか
- なぜフラグ付けされたか
- 関連するスキルまたはプラグインの内容
- 推奨事項

検出事項には `Info`、`Low`、`Medium`、`High`、`Critical` のラベルが付く場合があります。重大度の高い検出事項ほど、リスクレベルと監査ステータスにより強く影響します。

信頼度の低い検出事項は公開監査ロールアップから非表示になり、ページが有用な証拠に集中できるようにしています。

## ClawHub が確認すること

ClawHub は、提出されたリリース成果物を監査します。これには次が含まれます。

- スキル指示またはプラグインメタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性とケイパビリティのメタデータ

主な問いは一貫性です。名前、概要、メタデータ、要求された権限、実際の内容が、ユーザーが合理的に期待するものと一致しているかどうかです。

強力な動作は、それだけで悪いわけではありません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダー API、パッケージインストールが必要です。監査では、その力が期待され、開示され、相応であるかどうかを確認します。

成果物ページは、次の完全な監査にリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は監査スタック内のマルウェアテレメトリとして VirusTotal を使用します。VirusTotal は、ファイル評価とマルウェアスキャンにおける信頼された業界標準であり、このパートナーシップにより、ClawHub はスキルとプラグインのレビューにより広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意ある成果物、エンジンによる検出、ClawHub のエージェント認識レビューを補完する評価シグナルに特に有用です。ベンダーエンジン数が利用できる場合、監査はそれを平易な言葉で要約します。たとえば次のようになります。

```text
62/62 vendors flagged this skill as clean.
```

または次のようになります。

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー数テレメトリがない場合、監査は次のように表示します。

```text
No VirusTotal findings
```

VirusTotal はあくまでテレメトリです。ClawHub 独自の成果物認識リスク分析に代わるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に支えられています。各リリースを、エージェント向け成果物としてレビューします。対象には、指示、メタデータ、宣言された権限、ファイル、ケイパビリティシグナル、静的スキャンシグナル、SkillSpector の検出事項、VirusTotal テレメトリ、公開者が提供したコンテキストが含まれます。静的スキャンシグナルは、このレビューの内部コンテキストです。単独の公開監査セクションや、インストールをブロックする判定ではありません。

リスク分析では、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリまたはコンテキストのポイズニング、過剰なエージェンシーといったリスクを見るレンズとして、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/) を使用します。

ClawScan は、恐ろしく見えるケイパビリティを自動的に悪意あるものとして扱いません。そのケイパビリティが開示されており、目的に沿っており、リリースに記載されたユースケースによって裏付けられているかどうかを確認します。
