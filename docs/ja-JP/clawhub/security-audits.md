# セキュリティ監査

ClawHubのセキュリティ監査は、スキルやプラグインをインストールしても十分に安全かどうかを判断するのに役立ちます。監査では、リリースが何を行うか、どの権限を求めるか、ファイル、アカウント、認証情報、コード、外部サービスにアクセスできるようになる前に追加の注意が必要な点があるかどうかを示します。

監査は強力な安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、必ず自分で判断してください。

[セキュリティ](/clawhub/security)、[許容される利用](/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/clawhub/moderation)も参照してください。

## インストール前に確認すること

インストール前に、次を確認してください。

- 全体的な監査ステータス
- リスクレベル
- 記載されている所見
- 必要な認証情報、権限、または環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し信頼できるコンテンツだけをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass` | 低リスクを超える目に見える問題は見つかりませんでした。 |
| `Review` | インストール前に所見を読んでください。リリースは正当なものかもしれません。 |
| `Warn` | 追加の注意を払ってください。ClawHubが影響の大きい懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。 |
| `Pending` | 監査はまだ完了していません。 |
| `Error` | 監査を完了できませんでした。 |

`Pass`は安心材料になりますが、自分自身の判断に置き換わるものではありません。これは、コンテンツを公開したり、データを編集したり、コマンドを実行したり、ファイルを読んだり、本番システムにアクセスしたりできるツールで特に重要です。

## リスクレベル

リスクレベルは爆発半径、つまり意図どおりに使用した場合にそのリリースがどれだけの力を持つように見えるかを表します。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low` | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium` | リリースには、アカウントアクセスやデータ変更などの意味のある権限があります。 |
| `High` | リリースには、影響の大きい権限、重大な所見、または悪意あるシグナルがあります。 |

リスクレベルと監査ステータスは別の問いに答えます。

- リスクレベルが問うこと: 「ここにはどれだけの力があるか？」
- 監査ステータスが問うこと: 「この結果をどう扱うべきか？」

たとえば、公開用スキルは`Medium`リスクで`Review`を示すことがあります。それは悪意があるという意味ではありません。そのスキルは目的に沿っているように見えるものの、意味のあるアカウント権限で動作できるという意味です。

## 所見

所見は、監査結果が表示された理由を説明します。各所見には通常、次が含まれます。

- それが何を意味するか
- フラグが立てられた理由
- 関連するスキルまたはプラグインのコンテンツ
- 推奨事項

所見には`Info`、`Low`、`Medium`、`High`、`Critical`のラベルが付くことがあります。重大度の高い所見ほど、リスクレベルと監査ステータスに強く影響します。

信頼度の低い所見は公開監査の集約表示から非表示にされるため、ページは有用な証拠に集中できます。

## ClawHubが確認すること

ClawHubは、提出されたリリースアーティファクトを監査します。対象には次が含まれます。

- スキルの指示またはプラグインのメタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能メタデータ

主な問いは一貫性です。名前、概要、メタデータ、要求された権限、実際のコンテンツが、ユーザーが合理的に期待する内容と一致しているかどうかです。

強力な動作は自動的に悪いものではありません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダーAPI、パッケージのインストールが必要です。監査では、その力が期待され、開示され、釣り合っているかどうかを確認します。

アーティファクトページは、次の完全な監査にリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHubは監査スタックでマルウェアテレメトリとしてVirusTotalを使用します。VirusTotalはファイルの評判とマルウェアスキャンにおける信頼された業界標準であり、このパートナーシップにより、ClawHubはスキルとプラグインのレビューにより広範なセキュリティインテリジェンスを追加できます。

VirusTotalは、既知の悪意あるアーティファクト、エンジンの検出、ClawHubのエージェント対応レビューを補完する評判シグナルに特に有用です。ベンダーエンジン数が利用可能な場合、監査では次のように平易な言葉で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHubに要約できるベンダー数テレメトリがない場合、監査には次のように表示されます。

```text
No VirusTotal findings
```

VirusTotalはあくまでテレメトリです。ClawHub独自のアーティファクト対応リスク分析に置き換わるものではありません。

## リスク分析

リスク分析は、ClawHub独自のセキュリティ監査システムであるClawScanによって内部的に提供されます。これは各リリースをエージェント向けアーティファクトとしてレビューします。対象には、指示、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpectorの所見、VirusTotalのテレメトリ、発行者が提供したコンテキストが含まれます。静的スキャンシグナルはこのレビューの内部コンテキストであり、独立した公開監査セクションやインストールをブロックする判断ではありません。

リスク分析では、プロンプトインジェクション、ツールの誤用、認証情報の漏えい、安全でない実行、メモリまたはコンテキストの汚染、過剰なエージェンシーなどのリスクを見るための視点として、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)を使用します。

ClawScanは、恐ろしく見える機能を自動的に悪意あるものとは扱いません。その機能が開示され、目的に沿っており、リリースで述べられたユースケースによって裏付けられているかどうかを問います。
