# セキュリティ監査

ClawHub のセキュリティ監査は、スキルまたは Plugin をインストールしても十分に安全かどうかを判断するのに役立ちます。監査では、リリースが何をするのか、どの権限を要求するのか、ファイル、アカウント、認証情報、コード、外部サービスにアクセスできるようになる前に追加の注意が必要なものがあるかどうかを示します。

監査は強力な安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前には、必ず自分で判断してください。

[セキュリティ](/clawhub/security)、[許容される使用](/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/clawhub/moderation)も参照してください。

## インストール前に確認すること

インストール前に、次を確認してください。

- 全体の監査ステータス
- リスクレベル
- 記載されている検出事項
- 必要な認証情報、権限、または環境変数
- オーナー、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し、信頼できるコンテンツだけをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass` | 低リスクを超える目に見える問題は見つかりませんでした。 |
| `Review` | インストール前に検出事項を読んでください。リリースは正当なものである可能性があります。 |
| `Warn` | 特に注意してください。ClawHub は影響の大きい懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。 |
| `Pending` | 監査はまだ完了していません。 |
| `Error` | 監査を完了できませんでした。 |

`Pass` は安心材料になりますが、自分自身の判断に代わるものではありません。これは、コンテンツを公開する、データを編集する、コマンドを実行する、ファイルを読む、または本番システムにアクセスするツールで特に重要です。

## リスクレベル

リスクレベルはブラスト半径を表します。つまり、意図されたとおりに使用した場合に、そのリリースがどれだけの力を持つように見えるかです。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low` | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium` | リリースには、アカウントアクセスやデータ変更など、意味のある権限があります。 |
| `High` | リリースには、影響の大きい権限、重大な検出事項、または悪意のあるシグナルがあります。 |

リスクレベルと監査ステータスは、別々の問いに答えます。

- リスクレベルが問うのは、「ここにはどれだけの力があるか?」です。
- 監査ステータスが問うのは、「この結果に対して何をすべきか?」です。

たとえば、公開用のスキルは `Medium` リスクで `Review` と表示される場合があります。これは悪意があるという意味ではありません。そのスキルが目的に沿っているように見える一方で、意味のあるアカウント権限で動作できることを意味します。

## 検出事項

検出事項は、監査結果が表示された理由を説明します。各検出事項には通常、次が含まれます。

- それが何を意味するか
- フラグが付けられた理由
- 関連するスキルまたは Plugin の内容
- 推奨事項

検出事項には `Info`、`Low`、`Medium`、`High`、または `Critical` のラベルが付く場合があります。重大度の高い検出事項ほど、リスクレベルと監査ステータスにより強く影響します。

信頼度の低い検出事項は公開監査の集計から非表示にされ、ページが有用な証拠に集中できるようにしています。

## ClawHub が確認すること

ClawHub は、提出されたリリース成果物を監査します。対象には次が含まれます。

- スキルの指示または Plugin メタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能メタデータ

主な問いは一貫性です。名前、概要、メタデータ、要求される権限、実際の内容が、ユーザーが合理的に期待するものと一致しているかどうかです。

強力な動作は自動的に悪いものではありません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダー API、またはパッケージインストールが必要です。監査では、その力が期待され、開示され、釣り合っているかどうかを確認します。

成果物ページは、完全な監査へ次の場所でリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は監査スタック内のマルウェアテレメトリとして VirusTotal を使用します。VirusTotal はファイルの評判とマルウェアスキャンにおける信頼された業界標準であり、このパートナーシップにより ClawHub はスキルと Plugin のレビューに、より広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意ある成果物、エンジン検出、ClawHub のエージェント認識型レビューを補完する評判シグナルに特に有用です。ベンダーエンジン数が利用可能な場合、監査では次のように平易な表現で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー数テレメトリがない場合、監査には次のように表示されます。

```text
No VirusTotal findings
```

VirusTotal はあくまでテレメトリです。ClawHub 独自の成果物認識型リスク分析に代わるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に支えられています。各リリースをエージェント向け成果物としてレビューします。対象には、指示、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpector の検出事項、VirusTotal テレメトリ、公開者が提供したコンテキストが含まれます。静的スキャンシグナルは、このレビューの内部コンテキストです。独立した公開監査セクションや、インストールをブロックする判定ではありません。

リスク分析では、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリまたはコンテキストの汚染、過剰な自律性といったリスクを見る観点として、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)を使用します。

ClawScan は、見た目が危険な機能を自動的に悪意あるものとして扱いません。その機能が開示され、目的に沿っており、リリースで明示されたユースケースによって裏付けられているかどうかを問います。
