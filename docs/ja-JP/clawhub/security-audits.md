# セキュリティ監査

ClawHub のセキュリティ監査は、スキルやプラグインをインストールして十分安全かどうかを判断するのに役立ちます。リリースが何を行うか、どの権限を要求するか、ファイル、アカウント、認証情報、コード、外部サービスへアクセスする前に追加の注意が必要な点があるかを示します。

監査は強力な安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、常に自分で判断してください。

[セキュリティ](/clawhub/security)、[許容される利用](/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/clawhub/moderation)も参照してください。

## インストール前に確認すること

インストール前に、次を確認してください。

- 全体的な監査ステータス
- リスクレベル
- 記載されている所見
- 必要な認証情報、権限、環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し信頼できるコンテンツだけをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass` | 低リスクを超える目に見える問題は見つかりませんでした。 |
| `Review` | インストール前に所見を読んでください。リリースは正当なものである可能性があります。 |
| `Warn` | 追加の注意を払ってください。ClawHub が影響の大きい懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。 |
| `Pending` | 監査はまだ完了していません。 |
| `Error` | 監査を完了できませんでした。 |

`Pass` は安心材料になりますが、自分自身の判断に置き換わるものではありません。これは、コンテンツを公開したり、データを編集したり、コマンドを実行したり、ファイルを読んだり、本番システムにアクセスしたりできるツールで特に重要です。

## リスクレベル

リスクレベルは影響範囲を表します。意図どおりに使用した場合、そのリリースがどれだけの権限を持つように見えるかを示します。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low` | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium` | リリースには、アカウントアクセスやデータ変更など、意味のある権限があります。 |
| `High` | リリースには、影響の大きい権限、重大な所見、または悪意のあるシグナルがあります。 |

リスクレベルと監査ステータスは、それぞれ異なる問いに答えます。

- リスクレベルは「ここにはどれだけの権限があるか」を問います。
- 監査ステータスは「この結果に対して何をすべきか」を問います。

たとえば、公開用スキルは `Medium` リスクで `Review` と表示されることがあります。これは悪意があるという意味ではありません。そのスキルが目的に沿っているように見える一方で、意味のあるアカウント権限で動作できることを意味します。

## 所見

所見は、監査結果が表示された理由を説明します。各所見には通常、次が含まれます。

- それが何を意味するか
- フラグ付けされた理由
- 関連するスキルまたはプラグインの内容
- 推奨事項

所見には `Info`、`Low`、`Medium`、`High`、`Critical` のラベルが付くことがあります。深刻度の高い所見ほど、リスクレベルと監査ステータスにより強く影響します。

信頼度の低い所見は公開監査の集約表示から非表示にされるため、ページは有用な証拠に集中できます。

## ClawHub が確認すること

ClawHub は、提出されたリリース成果物を監査します。これには次が含まれます。

- スキルの指示またはプラグインのメタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能メタデータ

主な問いは一貫性です。名前、要約、メタデータ、要求された権限、実際の内容が、ユーザーが合理的に期待するものと一致しているかです。

強力な動作は自動的に悪いものではありません。有用なツールの多くは、認証情報、ローカルコマンド、プロバイダー API、パッケージインストールを必要とします。監査では、その権限が期待され、開示され、釣り合っているかを確認します。

成果物ページは、次の完全な監査にリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は、監査スタック内のマルウェアテレメトリとして VirusTotal を使用します。VirusTotal はファイルの評判とマルウェアスキャンに関する信頼された業界標準であり、このパートナーシップにより、ClawHub はスキルとプラグインのレビューにより広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、ClawHub のエージェントを考慮したレビューを補完する、既知の悪意ある成果物、エンジン検出、評判シグナルに特に有用です。ベンダーエンジン数が利用できる場合、監査は次のように平易な言葉で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー数テレメトリがない場合、監査は次のように表示します。

```text
No VirusTotal findings
```

VirusTotal はあくまでテレメトリです。ClawHub 独自の成果物を考慮したリスク分析に置き換わるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に動作しています。各リリースをエージェント向け成果物としてレビューします。対象には、指示、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpector の所見、VirusTotal テレメトリ、公開者が提供したコンテキストが含まれます。静的スキャンシグナルはこのレビューの内部コンテキストであり、独立した公開監査セクションやインストールをブロックする判定ではありません。

リスク分析では、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリまたはコンテキスト汚染、過剰なエージェンシーなどのリスクを捉える観点として、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)を使用します。

ClawScan は、危険に見える機能を自動的に悪意あるものとは見なしません。その機能が開示され、目的に沿っており、リリースで述べられているユースケースによって裏付けられているかを確認します。
