# セキュリティ監査

ClawHub のセキュリティ監査は、スキルやプラグインをインストールしても十分に安全かどうかを判断するのに役立ちます。リリースが何をするのか、どの権限を求めるのか、ファイル、アカウント、認証情報、コード、外部サービスにアクセスできるようになる前に追加で注意すべき点があるかどうかを示します。

監査は強力な安全性のシグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、必ず自分で判断してください。

[セキュリティ](/clawhub/security)、[許容される利用](/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/clawhub/moderation)も参照してください。

## インストール前に確認すること

インストール前に、以下を確認してください。

- 全体的な監査ステータス
- リスクレベル
- 記載されている所見
- 必要な認証情報、権限、または環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し信頼できるコンテンツのみをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 低リスクを超える目に見える問題は見つかりませんでした。 |
| `Review`    | インストール前に所見を読んでください。リリースは正当なものかもしれません。 |
| `Warn`      | 追加の注意を払ってください。ClawHub は影響の大きい懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。 |
| `Pending`   | 監査はまだ完了していません。 |
| `Error`     | 監査を完了できませんでした。 |

`Pass` は安心材料になりますが、自分自身の判断に置き換わるものではありません。これは、コンテンツを公開したり、データを編集したり、コマンドを実行したり、ファイルを読んだり、本番システムにアクセスしたりできるツールで特に重要です。

## リスクレベル

リスクレベルはブラスト半径、つまり意図どおりに使用した場合にそのリリースがどれほどの力を持つように見えるかを表します。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium`   | そのリリースには、アカウントアクセスやデータ変更など、意味のある権限があります。 |
| `High`     | そのリリースには、影響の大きい権限、重大な所見、または悪意を示すシグナルがあります。 |

リスクレベルと監査ステータスは異なる問いに答えます。

- リスクレベルが問うこと: 「ここにはどれほどの力があるか」
- 監査ステータスが問うこと: 「この結果に対して何をすべきか」

たとえば、公開用のスキルは `Medium` リスクで `Review` と表示される場合があります。これは悪意があるという意味ではありません。そのスキルは目的に沿っているように見えるものの、意味のあるアカウント権限で動作できるという意味です。

## 所見

所見は、監査結果が表示された理由を説明します。各所見には通常、以下が含まれます。

- それが意味すること
- フラグが付けられた理由
- 関連するスキルまたはプラグインの内容
- 推奨事項

所見には `Info`、`Low`、`Medium`、`High`、または `Critical` のラベルが付く場合があります。重大度の高い所見ほど、リスクレベルと監査ステータスに強く影響します。

信頼度の低い所見は、ページが有用な証拠に集中できるように、公開監査ロールアップでは非表示になります。

## ClawHub が確認すること

ClawHub は提出されたリリースアーティファクトを監査します。対象には以下が含まれます。

- スキルの指示またはプラグインのメタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能メタデータ

主な問いは一貫性です。名前、概要、メタデータ、要求される権限、実際の内容が、ユーザーが合理的に期待する内容と一致しているかどうかです。

強力な挙動は自動的に悪いものではありません。有用なツールの多くは、認証情報、ローカルコマンド、プロバイダー API、またはパッケージのインストールを必要とします。監査では、その力が期待され、開示され、釣り合っているかどうかを確認します。

アーティファクトページは、完全な監査に以下でリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは以下を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は監査スタック内のマルウェアテレメトリとして VirusTotal を使用します。VirusTotal はファイルの評判とマルウェアスキャンにおける信頼された業界標準であり、このパートナーシップにより ClawHub はスキルとプラグインのレビューに、より広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意あるアーティファクト、エンジンの検出、ClawHub のエージェントを意識したレビューを補完する評判シグナルに特に有用です。ベンダーエンジンの件数が利用できる場合、監査はそれらを次のような平易な言葉で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー件数テレメトリがない場合、監査は次のように表示します。

```text
No VirusTotal findings
```

VirusTotal はあくまでテレメトリです。ClawHub 独自の、アーティファクトを意識したリスク分析に置き換わるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に支えられています。各リリースを、エージェント向けアーティファクトとしてレビューします。対象には、指示、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpector の所見、VirusTotal テレメトリ、公開者が提供したコンテキストが含まれます。静的スキャンシグナルはこのレビューの内部コンテキストであり、独立した公開監査セクションやインストールをブロックする判定ではありません。

リスク分析では、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリやコンテキストのポイズニング、過剰なエージェンシーなどのリスクを見るレンズとして、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)を使用します。

ClawScan は、見た目が恐ろしい機能を自動的に悪意あるものとして扱うことはありません。その機能が開示されているか、目的に沿っているか、リリースで述べられたユースケースによって裏付けられているかを確認します。
