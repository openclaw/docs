# セキュリティ監査

ClawHub のセキュリティ監査は、スキルまたはPluginがインストールするのに十分安全かどうかを判断するのに役立ちます。監査では、リリースが何をするか、どの権限を要求するか、ファイル、アカウント、認証情報、コード、外部サービスへアクセスする前に追加の注意が必要な点があるかどうかを示します。

監査は強い安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、常に自分で判断してください。

[セキュリティ](/ja-JP/clawhub/security)、[許容される使用](/ja-JP/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/ja-JP/clawhub/moderation)も参照してください。

## インストール前に確認すること

インストール前に、次を確認してください。

- 全体の監査ステータス
- リスクレベル
- 記載されている所見
- 必要な認証情報、権限、または環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し信頼できるコンテンツだけをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass` | 低リスクを超える目に見える問題は見つかりませんでした。 |
| `Review` | インストール前に所見を読んでください。リリースは正当なものである可能性があります。 |
| `Warn` | 特に注意してください。ClawHub が影響の大きい懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。 |
| `Pending` | 監査はまだ完了していません。 |
| `Error` | 監査を完了できませんでした。 |

`Pass` は安心材料ですが、自分自身の判断に代わるものではありません。これは、コンテンツを公開する、データを編集する、コマンドを実行する、ファイルを読む、本番システムへアクセスする、といったことができるツールで特に重要です。

## リスクレベル

リスクレベルは、想定どおりに使用した場合にそのリリースが持つと思われる権限の範囲、つまり影響範囲を表します。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low` | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium` | そのリリースには、アカウントアクセスやデータ変更など、意味のある権限があります。 |
| `High` | そのリリースには、影響の大きい権限、重大な所見、または悪意のあるシグナルがあります。 |

リスクレベルと監査ステータスは、異なる問いに答えます。

- リスクレベルが問うのは、「ここにはどれほどの権限があるか」です。
- 監査ステータスが問うのは、「この結果を受けて何をすべきか」です。

たとえば、公開用スキルは `Medium` リスクで `Review` と表示される場合があります。それは悪意があるという意味ではありません。そのスキルが目的に沿っているように見える一方で、意味のあるアカウント権限で動作できるという意味です。

## 所見

所見は、なぜその監査結果が表示されたのかを説明します。各所見には通常、次が含まれます。

- それが何を意味するか
- なぜフラグされたか
- 関連するスキルまたはPluginのコンテンツ
- 推奨事項

所見には `Info`、`Low`、`Medium`、`High`、または `Critical` のラベルが付く場合があります。重大度が高い所見ほど、リスクレベルと監査ステータスにより強く影響します。

信頼度の低い所見は、ページが有用な証拠に集中できるよう、公開監査の集計からは非表示になります。

## ClawHub が確認すること

ClawHub は、送信されたリリース成果物を監査します。これには次が含まれます。

- スキルの指示またはPluginメタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能メタデータ

主な問いは整合性です。名前、概要、メタデータ、要求された権限、実際のコンテンツは、ユーザーが合理的に期待する内容と一致しているか、ということです。

強力な動作が自動的に悪いわけではありません。有用なツールの多くは、認証情報、ローカルコマンド、プロバイダー API、またはパッケージインストールを必要とします。監査では、その権限が想定され、開示され、釣り合っているかを確認します。

成果物ページは、次の完全な監査にリンクしています。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は、監査スタックのマルウェアテレメトリとして VirusTotal を使用します。VirusTotal は、ファイル評価とマルウェアスキャンにおける信頼された業界標準であり、この提携により ClawHub はスキルとPluginのレビューに、より広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意ある成果物、エンジン検出、ClawHub のエージェント対応レビューを補完する評価シグナルに特に有用です。ベンダーエンジンの件数が利用できる場合、監査では次のように平易な言葉で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または、次のように表示されます。

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー件数テレメトリがない場合、監査は次のように表示します。

```text
No VirusTotal findings
```

VirusTotal はテレメトリのままです。ClawHub 独自の成果物対応リスク分析に代わるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に支えられています。これは各リリースを、エージェント向け成果物としてレビューします。対象には、指示、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpector の所見、VirusTotal テレメトリ、公開者が提供したコンテキストが含まれます。静的スキャンシグナルはこのレビューの内部コンテキストであり、独立した公開監査セクションでも、インストールをブロックする判定でもありません。

リスク分析では、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリまたはコンテキストの汚染、過剰な自律性といったリスクを見る観点として、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/) を使用します。

ClawScan は、怖く見える機能を自動的に悪意があるものとして扱いません。その機能が開示され、目的に沿っており、リリースで明示されたユースケースによって裏付けられているかを確認します。
