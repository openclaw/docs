# セキュリティ監査

ClawHub のセキュリティ監査は、Skill や Plugin がインストールするのに十分安全かどうかを判断する助けになります。監査では、リリースが何を行うか、どの権限を求めるか、ファイル、アカウント、認証情報、コード、外部サービスにアクセスする前に追加の注意が必要な点があるかどうかを示します。

監査は強力な安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、必ず自分で判断してください。

[セキュリティ](/clawhub/security)、[許容される利用](/clawhub/acceptable-usage)、[モデレーションとアカウント安全性](/clawhub/moderation)も参照してください。

## インストール前に確認すること

インストール前に、次を確認してください。

- 全体の監査ステータス
- リスクレベル
- 記載されている検出事項
- 必要な認証情報、権限、環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解して信頼できるコンテンツだけをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass` | 低リスクを超える目に見える問題は見つかりませんでした。 |
| `Review` | インストール前に検出事項を読んでください。リリースは正当なものの可能性があります。 |
| `Warn` | 特に注意してください。ClawHub が影響の大きい懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。 |
| `Pending` | 監査はまだ完了していません。 |
| `Error` | 監査を完了できませんでした。 |

`Pass` は安心材料になりますが、自分で判断することの代わりにはなりません。これは、コンテンツの公開、データの編集、コマンドの実行、ファイルの読み取り、本番システムへのアクセスが可能なツールで特に重要です。

## リスクレベル

リスクレベルは影響範囲を表します。つまり、意図どおりに使用した場合に、そのリリースがどれほど強い権限を持つように見えるかです。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low` | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium` | リリースには、アカウントアクセスやデータ変更など、意味のある権限があります。 |
| `High` | リリースには、影響の大きい権限、重大な検出事項、または悪意あるシグナルがあります。 |

リスクレベルと監査ステータスは、異なる問いに答えます。

- リスクレベルが問うこと: 「ここにはどれほどの権限があるか」
- 監査ステータスが問うこと: 「この結果にどう対応すべきか」

たとえば、公開用の Skill は `Medium` リスクで `Review` と表示される場合があります。それは悪意があるという意味ではありません。その Skill が目的に沿っているように見える一方で、意味のあるアカウント権限で動作できるという意味です。

## 検出事項

検出事項は、監査結果が表示された理由を説明します。各検出事項には通常、次が含まれます。

- それが何を意味するか
- フラグが付けられた理由
- 関連する Skill または Plugin の内容
- 推奨事項

検出事項には `Info`、`Low`、`Medium`、`High`、または `Critical` のラベルが付くことがあります。深刻度の高い検出事項ほど、リスクレベルと監査ステータスに強く影響します。

信頼度の低い検出事項は、ページが有用な証拠に集中できるように、公開監査の要約からは非表示になります。

## ClawHub が確認すること

ClawHub は、提出されたリリース成果物を監査します。これには次が含まれます。

- Skill の指示または Plugin メタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能メタデータ

主な問いは一貫性です。名前、概要、メタデータ、要求される権限、実際の内容が、ユーザーが合理的に期待する内容と一致しているかどうかです。

強力な動作は、自動的に悪いものとは限りません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダー API、またはパッケージインストールが必要です。監査では、その権限が期待され、開示され、釣り合っているかを確認します。

成果物ページは、次の完全な監査にリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は監査スタック内のマルウェアテレメトリとして VirusTotal を使用します。VirusTotal は、ファイルの評判とマルウェアスキャンに関する信頼された業界標準であり、この提携により ClawHub は Skill と Plugin のレビューに、より広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意ある成果物、エンジンの検出、ClawHub のエージェント対応レビューを補完する評判シグナルに特に有用です。ベンダーエンジン数が利用できる場合、監査は次のように平易な言葉で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー数テレメトリがない場合、監査では次のように表示されます。

```text
No VirusTotal findings
```

VirusTotal はあくまでテレメトリです。ClawHub 独自の成果物対応リスク分析を置き換えるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に支えられています。各リリースを、エージェント向けの成果物としてレビューします。対象には、指示、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpector の検出事項、VirusTotal テレメトリ、公開者が提供したコンテキストが含まれます。静的スキャンシグナルはこのレビューの内部コンテキストであり、単独の公開監査セクションやインストールをブロックする判定ではありません。

リスク分析では、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/) を、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリまたはコンテキストの汚染、過度な代理性などのリスクを見るための観点として使用します。

ClawScan は、見た目が怖い機能を自動的に悪意あるものとして扱いません。その機能が開示されているか、目的に沿っているか、リリースで述べられたユースケースに裏付けられているかを確認します。
