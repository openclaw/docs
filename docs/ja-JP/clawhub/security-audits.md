# セキュリティ監査

ClawHub のセキュリティ監査は、スキルまたはPluginをインストールしても十分に安全かどうかを判断するのに役立ちます。監査では、リリースが何を行うか、どの権限を求めるか、ファイル、アカウント、認証情報、コード、外部サービスへアクセスする前に追加の注意が必要な点があるかどうかを示します。

監査は強力な安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、必ず自分で判断してください。

[セキュリティ](/clawhub/security)、[許容される利用](/ja-JP/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/clawhub/moderation)も参照してください。

## インストール前に確認すること

インストール前に、次を確認してください。

- 全体的な監査ステータス
- リスクレベル
- 記載されている所見
- 必要な認証情報、権限、または環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し信頼できるコンテンツのみをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 低リスクを超える目に見える問題は見つかりませんでした。 |
| `Review`    | インストール前に所見を読んでください。リリースは正当なものである可能性があります。 |
| `Warn`      | さらに注意してください。ClawHub が影響の大きい懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。 |
| `Pending`   | 監査はまだ完了していません。 |
| `Error`     | 監査を完了できませんでした。 |

`Pass` は安心材料になりますが、自分自身の判断の代わりにはなりません。これは、コンテンツを公開する、データを編集する、コマンドを実行する、ファイルを読む、または本番システムにアクセスするツールで特に重要です。

## リスクレベル

リスクレベルはブラスト半径、つまり意図どおりに使用した場合にそのリリースがどれほどの力を持つように見えるかを表します。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium`   | リリースには、アカウントアクセスやデータ変更など、意味のある権限があります。 |
| `High`     | リリースには、影響の大きい権限、重大な所見、または悪意のあるシグナルがあります。 |

リスクレベルと監査ステータスは、異なる問いに答えます。

- リスクレベルは「ここにはどれほどの力があるか？」を問います。
- 監査ステータスは「この結果をどう扱うべきか？」を問います。

たとえば、公開用スキルは `Medium` リスクで `Review` と表示される場合があります。これは、そのスキルが悪意のあるものだという意味ではありません。目的に沿っているように見える一方で、意味のあるアカウント権限で動作できるという意味です。

## 所見

所見は、監査結果が表示された理由を説明します。各所見には通常、次が含まれます。

- それが何を意味するか
- フラグ付けされた理由
- 関連するスキルまたはPluginのコンテンツ
- 推奨事項

所見には `Info`、`Low`、`Medium`、`High`、または `Critical` のラベルが付く場合があります。重大度が高い所見ほど、リスクレベルと監査ステータスに強く影響します。

低信頼度の検出結果は公開監査ロールアップから非表示にされるため、ページは有用な証拠に集中した状態を保ちます。

## ClawHub が確認する内容

ClawHub は提出されたリリースアーティファクトを監査します。対象には以下が含まれます。

- スキルの指示またはプラグインメタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能メタデータ

主な問いは一貫性です。名前、要約、メタデータ、要求された権限、実際の内容が、ユーザーが合理的に期待するものと一致しているかを確認します。

強力な挙動が自動的に悪いわけではありません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダー API、またはパッケージのインストールが必要です。監査では、その力が期待され、開示され、相応であるかを確認します。

アーティファクトページは、完全な監査へ以下でリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは以下を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は監査スタック内のマルウェアテレメトリとして VirusTotal を使用します。VirusTotal はファイル評価とマルウェアスキャンにおいて信頼されている業界標準であり、このパートナーシップにより ClawHub はスキルとプラグインのレビューに、より広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意あるアーティファクト、エンジン検出、ClawHub のエージェント対応レビューを補完する評価シグナルに特に有用です。ベンダーエンジン数が利用可能な場合、監査は次のように平易な言葉で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー数テレメトリがない場合、監査には次のように表示されます。

```text
No VirusTotal findings
```

VirusTotal はあくまでテレメトリです。ClawHub 独自のアーティファクト対応リスク分析を置き換えるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に実行されます。各リリースを、エージェント向けアーティファクトとしてレビューします。対象には、指示、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpector の検出結果、VirusTotal テレメトリ、公開者が提供したコンテキストが含まれます。静的スキャンシグナルはこのレビューの内部コンテキストであり、単独の公開監査セクションやインストールをブロックする判定ではありません。

リスク分析では、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリまたはコンテキストのポイズニング、過剰なエージェンシーなどのリスクを見る観点として、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)を使用します。

ClawScan は、一見危険に見える機能を自動的に悪意あるものとは扱いません。その機能が開示され、目的に合致し、リリースで述べられたユースケースに支えられているかを確認します。
