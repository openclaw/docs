# セキュリティ監査

ClawHub のセキュリティ監査は、スキルまたはプラグインをインストールしても十分に安全かどうかを判断するのに役立ちます。監査では、リリースが何を行うか、どの権限を要求するか、ファイル、アカウント、認証情報、コード、外部サービスにアクセスできるようになる前に追加の注意が必要なものがあるかを示します。

監査は強力な安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、必ず自分で判断してください。

関連項目: [セキュリティ](/clawhub/security)、[許容される使用](/clawhub/acceptable-usage)、[モデレーションとアカウント安全性](/clawhub/moderation)。

## インストール前に確認すること

インストール前に、次を確認してください。

- 全体の監査ステータス
- リスクレベル
- 掲載されている検出事項
- 必要な認証情報、権限、環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解して信頼できるコンテンツだけをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass` | 低リスクを超える目に見える問題は見つかりませんでした。 |
| `Review` | インストール前に検出事項を読んでください。リリースが正当である可能性はあります。 |
| `Warn` | 追加の注意を払ってください。ClawHub が影響の大きい懸念または警告シグナルを見つけました。 |
| `Malicious` | インストールしないでください。 |
| `Pending` | 監査はまだ完了していません。 |
| `Error` | 監査を完了できませんでした。 |

`Pass` は安心材料になりますが、自分自身の判断に代わるものではありません。これは、コンテンツを公開する、データを編集する、コマンドを実行する、ファイルを読む、または本番システムにアクセスできるツールで特に重要です。

## リスクレベル

リスクレベルはブラスト半径、つまり意図どおりに使用した場合にリリースがどの程度の力を持つように見えるかを表します。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low` | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium` | リリースには、アカウントアクセスやデータ変更などの意味のある権限があります。 |
| `High` | リリースには、影響の大きい権限、重大な検出事項、または悪意のあるシグナルがあります。 |

リスクレベルと監査ステータスは、異なる問いに答えます。

- リスクレベルが問うこと: 「ここにはどれだけの力があるか」
- 監査ステータスが問うこと: 「この結果に対して何をすべきか」

たとえば、公開用のスキルは `Medium` リスクで `Review` と表示されることがあります。それは悪意があるという意味ではありません。そのスキルは目的に沿っているように見えるものの、意味のあるアカウント権限で動作できるという意味です。

## 検出事項

検出事項は、監査結果が表示された理由を説明します。各検出事項には通常、次が含まれます。

- それが意味すること
- フラグ付けされた理由
- 関連するスキルまたはプラグインのコンテンツ
- 推奨事項

検出事項には、`Info`、`Low`、`Medium`、`High`、または `Critical` のラベルが付けられることがあります。重大度が高い検出事項ほど、リスクレベルと監査ステータスに強く影響します。

信頼度の低い検出事項は公開監査ロールアップから非表示にされるため、ページは有用な証拠に集中できます。

## ClawHub が確認すること

ClawHub は、送信されたリリース成果物を監査します。これには次が含まれます。

- スキル手順またはプラグインメタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能メタデータ

主な問いは一貫性です。名前、概要、メタデータ、要求された権限、実際のコンテンツが、ユーザーが合理的に期待する内容と一致しているかどうかです。

強力な動作は、自動的に悪いわけではありません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダー API、またはパッケージインストールが必要です。監査は、その力が期待され、開示され、相応であるかを確認します。

成果物ページは、完全な監査へのリンクを次の場所に表示します。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は監査スタックのマルウェアテレメトリとして VirusTotal を使用します。VirusTotal はファイルレピュテーションとマルウェアスキャンにおける信頼された業界標準であり、このパートナーシップにより ClawHub はスキルとプラグインのレビューに、より広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意ある成果物、エンジンのヒット、ClawHub のエージェント認識型レビューを補完するレピュテーションシグナルに特に有用です。ベンダーエンジンの件数を利用できる場合、監査は次のように平易な言葉で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー件数テレメトリがない場合、監査には次のように表示されます。

```text
No VirusTotal findings
```

VirusTotal はあくまでテレメトリです。ClawHub 独自の成果物認識型リスク分析に代わるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に支えられています。各リリースを、エージェント向け成果物としてレビューします。対象には、手順、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpector の検出事項、VirusTotal テレメトリ、発行者が提供したコンテキストが含まれます。静的スキャンシグナルはこのレビューの内部コンテキストであり、独立した公開監査セクションやインストールをブロックする判定ではありません。

リスク分析では、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリまたはコンテキストの汚染、過剰なエージェンシーなどのリスクを見るためのレンズとして、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)を使用します。

ClawScan は、見た目が危険な機能を自動的に悪意があるものとして扱いません。その機能が開示されているか、目的に沿っているか、リリースで示されたユースケースによって裏付けられているかを確認します。
