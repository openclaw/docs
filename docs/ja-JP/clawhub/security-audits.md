# セキュリティ監査

ClawHub のセキュリティ監査は、スキルや Plugin をインストールするのに十分安全かどうかを判断する助けになります。監査では、リリースが何をするのか、どの権限を求めるのか、ファイル、アカウント、認証情報、コード、外部サービスへアクセスできるようになる前に追加の注意が必要な点があるかどうかを示します。

監査は強力な安全シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、必ず自分で判断してください。

[セキュリティ](/clawhub/security)、[許容される利用](/ja-JP/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/clawhub/moderation)も参照してください。

## インストール前に確認すること

インストール前に、次を確認してください。

- 全体的な監査ステータス
- リスクレベル
- 記載されている所見
- 必要な認証情報、権限、または環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し、信頼できるコンテンツだけをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 低リスクを超える明らかな問題は見つかりませんでした。 |
| `Review`    | インストール前に所見を読んでください。リリースが正当である可能性はあります。 |
| `Warn`      | 追加の注意を払ってください。ClawHub が影響の大きい懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。 |
| `Pending`   | 監査はまだ完了していません。 |
| `Error`     | 監査を完了できませんでした。 |

`Pass` は安心材料になりますが、自分自身の判断に代わるものではありません。これは、コンテンツの公開、データの編集、コマンドの実行、ファイルの読み取り、本番システムへのアクセスができるツールで特に重要です。

## リスクレベル

リスクレベルは、ブラスト半径、つまりリリースを意図どおりに使用した場合にどれだけ大きな力を持つように見えるかを表します。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium`   | リリースには、アカウントアクセスやデータ変更など、意味のある権限があります。 |
| `High`     | リリースには、影響の大きい権限、重大な所見、または悪意のあるシグナルがあります。 |

リスクレベルと監査ステータスは、別々の問いに答えます。

- リスクレベルが問うこと: 「ここにはどれだけの力があるか」
- 監査ステータスが問うこと: 「この結果に対して何をすべきか」

たとえば、公開用のスキルは `Medium` リスクで `Review` と表示される場合があります。それは悪意があるという意味ではありません。そのスキルは目的に合っているように見える一方で、意味のあるアカウント権限で動作できるという意味です。

## 所見

所見は、監査結果が表示された理由を説明します。各所見には通常、次が含まれます。

- それが何を意味するか
- フラグ付けされた理由
- 関連するスキルまたは Plugin の内容
- 推奨事項

所見には `Info`、`Low`、`Medium`、`High`、`Critical` のラベルが付く場合があります。重大度の高い所見ほど、リスクレベルと監査ステータスに強く影響します。

信頼度の低い所見は公開監査ロールアップから非表示にされるため、ページは有用な証拠に集中できます。

## ClawHub が確認すること

ClawHub は、提出されたリリース成果物を監査します。対象には次が含まれます。

- スキルの指示または Plugin メタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能メタデータ

主な問いは一貫性です。名前、概要、メタデータ、要求された権限、実際の内容が、ユーザーが合理的に期待するものと一致しているかどうかです。

強力な動作が自動的に悪いわけではありません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダー API、またはパッケージインストールが必要です。監査では、その力が期待され、開示され、釣り合っているかどうかを確認します。

成果物ページは、次の完全な監査へリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は監査スタック内のマルウェアテレメトリとして VirusTotal を使用します。VirusTotal はファイルレピュテーションとマルウェアスキャンにおける信頼された業界標準であり、この提携により ClawHub はスキルと Plugin のレビューに、より広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意ある成果物、エンジンでの検出、ClawHub のエージェント認識レビューを補完するレピュテーションシグナルに特に役立ちます。ベンダーエンジン数を利用できる場合、監査はそれを次のような平易な言葉で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー数テレメトリがない場合、監査は次のように表示します。

```text
No VirusTotal findings
```

VirusTotal はあくまでテレメトリです。ClawHub 独自の成果物認識リスク分析を置き換えるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に支えられています。各リリースを、エージェント向けの成果物としてレビューします。対象には、指示、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpector の所見、VirusTotal テレメトリ、公開者が提供したコンテキストが含まれます。静的スキャンシグナルはこのレビューの内部コンテキストであり、独立した公開監査セクションやインストールをブロックする判定ではありません。

リスク分析では、
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
をレンズとして使用し、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリまたはコンテキストの汚染、過剰な主体性などのリスクを捉えます。

ClawScan は、見た目が危険そうな機能を自動的に悪意あるものとは扱いません。その機能が開示され、目的に合っており、リリースが明記したユースケースによって裏付けられているかどうかを確認します。
