# セキュリティ監査

ClawHub のセキュリティ監査は、Skills や Plugin をインストールしても十分安全かを判断する助けになります。監査では、リリースが何をするか、どの権限を求めるか、ファイル、アカウント、認証情報、コード、外部サービスにアクセスする前に追加の注意が必要な点があるかを示します。

監査は強力な安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、必ず自分で判断してください。

関連項目: [セキュリティ](/clawhub/security)、[許容される利用](/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/clawhub/moderation)。

## インストール前に確認すること

インストール前に、次を確認してください。

- 全体的な監査ステータス
- リスクレベル
- 記載されている所見
- 必要な認証情報、権限、環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し信頼できるコンテンツだけをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス  | 意味                                                                      |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 低リスクを超える目に見える問題は見つかりませんでした。                  |
| `Review`    | インストール前に所見を読んでください。リリースは正当な可能性があります。 |
| `Warn`      | 特に注意してください。ClawHub が高影響の懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。                                            |
| `Pending`   | 監査はまだ完了していません。                                              |
| `Error`     | 監査を完了できませんでした。                                              |

`Pass` は安心材料になりますが、自分自身の判断に代わるものではありません。これは、コンテンツを公開する、データを編集する、コマンドを実行する、ファイルを読む、本番システムにアクセスする、といったことが可能なツールで特に重要です。

## リスクレベル

リスクレベルは影響範囲を表します。つまり、そのリリースを意図どおりに使った場合に、どれほど強い権限を持つように見えるかです。

| リスクレベル | 意味                                                                          |
| ---------- | ----------------------------------------------------------------------------- |
| `Low`      | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。           |
| `Medium`   | アカウントアクセスやデータ変更など、リリースには意味のある権限があります。     |
| `High`     | リリースには高影響の権限、重大な所見、または悪意を示すシグナルがあります。     |

リスクレベルと監査ステータスは、異なる問いに答えます。

- リスクレベルが問うこと: 「ここにはどれほどの権限があるか」
- 監査ステータスが問うこと: 「この結果に対して何をすべきか」

たとえば、公開用の skill が `Medium` リスクで `Review` と表示されることがあります。これは悪意があるという意味ではありません。その skill が目的に沿っているように見える一方で、意味のあるアカウント権限で動作できるという意味です。

## 所見

所見は、なぜその監査結果が表示されたのかを説明します。通常、各所見には次が含まれます。

- それが何を意味するか
- フラグ付けされた理由
- 関連する skill または Plugin のコンテンツ
- 推奨事項

所見には `Info`、`Low`、`Medium`、`High`、`Critical` のラベルが付く場合があります。重大度が高い所見ほど、リスクレベルと監査ステータスに強く影響します。

信頼度の低い所見は公開監査の集約表示からは非表示にされるため、ページは有用な証拠に集中できます。

## ClawHub が確認すること

ClawHub は、提出されたリリースアーティファクトを監査します。これには次が含まれます。

- skill の指示または Plugin メタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性とケイパビリティのメタデータ

主な問いは一貫性です。名前、概要、メタデータ、要求された権限、実際のコンテンツが、ユーザーが合理的に期待する内容と合っているか、ということです。

強力な動作は自動的に悪いものではありません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダー API、パッケージのインストールが必要です。監査では、その権限が期待され、開示され、釣り合っているかを確認します。

アーティファクトページは、完全な監査に次の場所でリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は、監査スタック内のマルウェアテレメトリとして VirusTotal を使用します。VirusTotal はファイルレピュテーションとマルウェアスキャンの信頼された業界標準であり、このパートナーシップにより、ClawHub は skill と Plugin のレビューにより広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意あるアーティファクト、エンジン検出、ClawHub のエージェント認識レビューを補完するレピュテーションシグナルに特に有用です。ベンダーエンジン数が利用できる場合、監査は次のような平易な表現で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー数テレメトリがない場合、監査は次のように表示します。

```text
No VirusTotal findings
```

VirusTotal はあくまでテレメトリです。ClawHub 独自のアーティファクト認識リスク分析に代わるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に実行されます。各リリースをエージェント向けアーティファクトとしてレビューします。対象には、指示、メタデータ、宣言された権限、ファイル、ケイパビリティシグナル、静的スキャンシグナル、SkillSpector の所見、VirusTotal テレメトリ、公開者が提供したコンテキストが含まれます。静的スキャンシグナルはこのレビューの内部コンテキストであり、単独の公開監査セクションやインストールをブロックする判定ではありません。

リスク分析では、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリまたはコンテキストの汚染、過剰な代理性といったリスクを見る観点として、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/) を使用します。

ClawScan は、危険に見えるケイパビリティを自動的に悪意あるものとは扱いません。そのケイパビリティが開示され、目的に沿っており、リリースで述べられたユースケースに支えられているかを確認します。
