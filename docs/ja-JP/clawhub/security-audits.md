# セキュリティ監査

ClawHub のセキュリティ監査は、skill や plugin をインストールしても十分に安全かどうかを判断するのに役立ちます。リリースが何を行うのか、どの権限を要求するのか、ファイル、アカウント、認証情報、コード、外部サービスにアクセスできるようになる前に追加の注意が必要なものがあるかを示します。

監査は強力な安全性シグナルですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、必ず自分で判断してください。

[セキュリティ](/clawhub/security)、[許容される利用](/ja-JP/clawhub/acceptable-usage)、[モデレーションとアカウントの安全性](/clawhub/moderation) も参照してください。

## インストール前に確認すること

インストール前に、次を確認してください。

- 全体の監査ステータス
- リスクレベル
- 記載されている所見
- 必要な認証情報、権限、環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼シグナル

理解し信頼できるコンテンツだけをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果にどう対応すべきかを示します。

| ステータス | 意味 |
| ----------- | ------------------------------------------------------------------------- |
| `Pass` | 低リスクを超える明らかな問題は見つかりませんでした。 |
| `Review` | インストール前に所見を読んでください。リリースはなお正当なものである可能性があります。 |
| `Warn` | 追加の注意を払ってください。ClawHub は影響の大きい懸念または警告シグナルを検出しました。 |
| `Malicious` | インストールしないでください。 |
| `Pending` | 監査はまだ完了していません。 |
| `Error` | 監査を完了できませんでした。 |

`Pass` は安心材料ですが、自分自身の判断に代わるものではありません。これは、コンテンツを公開したり、データを編集したり、コマンドを実行したり、ファイルを読み取ったり、本番システムにアクセスしたりできるツールで特に重要です。

## リスクレベル

リスクレベルはブラスト半径を説明します。つまり、意図されたとおりに使用した場合に、そのリリースがどれほどの力を持つように見えるかです。

| リスクレベル | 意味 |
| ---------- | ----------------------------------------------------------------------------- |
| `Low` | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。 |
| `Medium` | このリリースには、アカウントアクセスやデータ変更などの意味のある権限があります。 |
| `High` | このリリースには、影響の大きい権限、重大な所見、または悪意のあるシグナルがあります。 |

リスクレベルと監査ステータスは、異なる問いに答えます。

- リスクレベルが問うこと: 「ここにはどれほどの力があるか」
- 監査ステータスが問うこと: 「この結果をどう扱うべきか」

たとえば、公開用の skill は `Medium` リスクで `Review` と表示される場合があります。それは悪意があることを意味しません。その skill は目的に沿っているように見えるものの、意味のあるアカウント権限で動作できることを意味します。

## 所見

所見は、監査結果が表示された理由を説明します。各所見には通常、次が含まれます。

- それが意味すること
- フラグ付けされた理由
- 関連する skill または plugin のコンテンツ
- 推奨事項

所見には `Info`、`Low`、`Medium`、`High`、または `Critical` のラベルが付くことがあります。重大度の高い所見ほど、リスクレベルと監査ステータスに強く影響します。

信頼度の低い所見は、ページが有用な証拠に集中できるよう、公開監査の集計からは非表示になります。

## ClawHub が確認すること

ClawHub は、送信されたリリース成果物を監査します。対象には次が含まれます。

- skill の指示または plugin メタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能メタデータ

主な問いは一貫性です。名前、概要、メタデータ、要求された権限、実際のコンテンツが、ユーザーが合理的に期待する内容と一致しているかどうかです。

強力な動作は自動的に悪いものではありません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダー API、またはパッケージのインストールが必要です。監査では、その力が期待され、開示され、釣り合っているかを確認します。

成果物ページは、完全な監査へ次の場所でリンクします。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページは次を組み合わせます。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は監査スタックでマルウェアテレメトリとして VirusTotal を使用します。VirusTotal はファイルの評判とマルウェアスキャンにおける信頼された業界標準であり、この提携により ClawHub は skill と plugin のレビューに、より広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意ある成果物、エンジンの検出、ClawHub のエージェント認識型レビューを補完する評判シグナルに特に有用です。ベンダーエンジン数が利用できる場合、監査はそれを次のような平易な表現で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約できるベンダー数テレメトリがない場合、監査は次のように表示します。

```text
No VirusTotal findings
```

VirusTotal はテレメトリのままです。ClawHub 独自の成果物認識型リスク分析に代わるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に支えられています。各リリースをエージェント向け成果物としてレビューします。対象は、指示、メタデータ、宣言された権限、ファイル、機能シグナル、静的スキャンシグナル、SkillSpector の所見、VirusTotal テレメトリ、公開者が提供したコンテキストです。静的スキャンシグナルはこのレビューの内部コンテキストであり、独立した公開監査セクションやインストールをブロックする判定ではありません。

リスク分析では、プロンプトインジェクション、ツールの誤用、認証情報の露出、安全でない実行、メモリやコンテキストのポイズニング、過剰なエージェンシーなどのリスクを捉える観点として、[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/) を使用します。

ClawScan は、見た目が危険そうな機能を自動的に悪意あるものとして扱いません。その機能が開示され、目的に沿っており、リリースで述べられたユースケースに裏付けられているかを確認します。
