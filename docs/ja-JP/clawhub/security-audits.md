# セキュリティ監査

ClawHub のセキュリティ監査は、スキルや Plugin がインストールするのに十分安全かどうかを判断するのに役立ちます。監査には、リリースが何を行うか、どのような権限を要求するか、ファイル、アカウント、認証情報、コード、外部サービスにアクセスできるようになる前に特に注意すべき点があるかどうかが示されます。

監査は安全性を判断するための強力な指標ですが、リリースにリスクがないことを保証するものではありません。機密性の高いアクセスを許可する前に、必ず自身で判断してください。

[セキュリティ](/clawhub/security)、[許容される利用方法](/clawhub/acceptable-usage)、
[モデレーションとアカウントの安全性](/clawhub/moderation)も参照してください。

## インストール前に確認すること

インストール前に、以下を確認してください。

- 監査の総合ステータス
- リスクレベル
- 記載されているすべての検出事項
- 必要な認証情報、権限、環境変数
- 所有者、ソース、バージョン、変更履歴、ダウンロード数、スター数、その他の信頼性指標

内容を理解し、信頼できるものだけをインストールしてください。

## 監査ステータス

監査ステータスは、監査結果に対してどのように対応すべきかを示します。

| ステータス  | 意味                                                                      |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 低リスクを超える明らかな問題は見つかりませんでした。                     |
| `Review`    | インストール前に検出事項を確認してください。正当なリリースである可能性はあります。 |
| `Warn`      | 特に注意してください。ClawHub が影響の大きい懸念事項または警告の兆候を検出しました。 |
| `Malicious` | インストールしないでください。                                            |
| `Pending`   | 監査はまだ完了していません。                                              |
| `Error`     | 監査を完了できませんでした。                                              |

`Pass` は安心材料になりますが、自身の判断に代わるものではありません。これは、コンテンツの公開、データの編集、コマンドの実行、ファイルの読み取り、本番システムへのアクセスが可能なツールで特に重要です。

## リスクレベル

リスクレベルは影響範囲、つまりリリースを意図どおりに使用した場合に、どの程度の権限を持つと考えられるかを示します。

| リスクレベル | 意味                                                                         |
| ------------ | ---------------------------------------------------------------------------- |
| `Low`        | 機密性の高い権限やユーザーへの影響はほとんど見つかりませんでした。           |
| `Medium`     | アカウントへのアクセスやデータ変更など、重要な権限を持つリリースです。       |
| `High`       | 影響の大きい権限、重大な検出事項、または悪意のある兆候があるリリースです。   |

リスクレベルと監査ステータスは、それぞれ異なる問いに答えます。

- リスクレベルの問い：「ここにはどの程度の権限があるか？」
- 監査ステータスの問い：「この結果を受けて何をすべきか？」

たとえば、公開を行うスキルでは、`Medium` リスクで `Review` と表示される場合があります。これは悪意があることを意味しません。そのスキルは目的に沿っていると考えられるものの、重要なアカウント権限を使って操作できることを意味します。

## 検出事項

検出事項は、監査結果が表示された理由を説明します。通常、各検出事項には以下が含まれます。

- 何を意味するか
- 検出された理由
- 関連するスキルまたは Plugin の内容
- 推奨事項

検出事項には、`Info`、`Low`、`Medium`、`High`、`Critical` のラベルが付く場合があります。深刻度が高い検出事項ほど、リスクレベルと監査ステータスに強く影響します。

信頼度の低い検出事項は公開監査の集計に表示されないため、ページでは有用な証拠に焦点を当てられます。

## ClawHub が確認する内容

ClawHub は、提出された以下のようなリリース成果物を監査します。

- スキルの指示または Plugin のメタデータ
- 宣言された環境変数と権限
- インストール手順とパッケージメタデータ
- 含まれるファイルとファイルマニフェスト
- 互換性と機能のメタデータ

主な確認点は一貫性です。名前、概要、メタデータ、要求される権限、実際の内容が、ユーザーが合理的に期待するものと一致しているかを確認します。

強力な動作が自動的に問題となるわけではありません。多くの有用なツールには、認証情報、ローカルコマンド、プロバイダー API、パッケージのインストールが必要です。監査では、その権限が想定内であり、開示され、目的に対して適切かどうかを確認します。

成果物ページから、以下の完全な監査ページにアクセスできます。

```text
/<owner>/skills/<slug>/security-audit
```

監査ページでは、以下を組み合わせて表示します。

1. SkillSpector
2. VirusTotal
3. リスク分析

## VirusTotal

ClawHub は、監査スタックのマルウェアテレメトリとして VirusTotal を使用しています。VirusTotal は、ファイルの評判とマルウェアスキャンにおいて信頼されている業界標準であり、この提携により ClawHub はスキルと Plugin のレビューに、より広範なセキュリティインテリジェンスを追加できます。

VirusTotal は、既知の悪意のある成果物、エンジンによる検出、ClawHub のエージェント対応レビューを補完する評判指標に特に有用です。ベンダーエンジンの件数を利用できる場合、監査では次のように平易な表現で要約します。

```text
62/62 vendors flagged this skill as clean.
```

または、

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub に要約可能なベンダー件数のテレメトリがない場合、監査には次のように表示されます。

```text
No VirusTotal findings
```

VirusTotal はあくまでテレメトリです。ClawHub 独自の成果物を考慮したリスク分析に代わるものではありません。

## リスク分析

リスク分析は、ClawHub 独自のセキュリティ監査システムである ClawScan によって内部的に実行されます。各リリースを、エージェント向け成果物としてレビューします。対象には、指示、メタデータ、宣言された権限、ファイル、機能の兆候、静的スキャンの兆候、SkillSpector の検出事項、VirusTotal のテレメトリ、公開者が提供したコンテキストが含まれます。静的スキャンの兆候は、このレビューの内部コンテキストです。独立した公開監査セクションでも、インストールを阻止する判定でもありません。

リスク分析では、
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
を、プロンプトインジェクション、ツールの誤用、認証情報の漏えい、安全でない実行、メモリやコンテキストの汚染、過剰な自律性などのリスクを評価する観点として使用します。

ClawScan は、危険に見える機能を自動的に悪意のあるものとして扱いません。その機能が開示され、目的に沿っており、リリースに記載されたユースケースによって裏付けられているかを確認します。
