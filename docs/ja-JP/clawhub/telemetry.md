# テレメトリ

ClawHub は、**インストール数**（実際に使用されているもの）を計算し、より良い並べ替え/フィルタリングを支えるために、**最小限のテレメトリ**を使用します。
これは CLI の `clawhub sync` コマンドに基づいています。

## テレメトリが収集されるタイミング

テレメトリは次の場合にのみ送信されます。

- CLI で**ログイン済み**である（同期/公開フローにはすでに認証が必要です）。
- `clawhub sync` を実行する。
- テレメトリが**無効化されていない**（下の「無効化する方法」を参照）。

ログインしていない場合、何も報告されません。

## 収集するもの

各 `clawhub sync` で、CLI は検出した内容の**完全なスナップショット**を、スキャンルート（「フォルダー/ルート」）ごとにグループ化して報告します。

各ルートについて、次を保存します。

- `rootId`: 正規化されたルートパスの **SHA-256 ハッシュ**（サーバーは生のパスを一切見ません）。
- `label`: 最後の 2 つのパスセグメントから派生した、人間が読めるラベル（ホームパスは `~` で表示されます）。
- `firstSeenAt`、`lastSeenAt`、任意の `expiredAt`。

ルート配下で見つかった各 skill について、次を保存します。

- `skillId`（スラッグで解決されます。レジストリに存在する skills のみが追跡されます）。
- `firstSeenAt`、`lastSeenAt`。
- `lastVersion`（ベストエフォート。現在は、既知の場合はレジストリで一致したバージョン）。
- 以前に報告されたインストールがルートから消えた場合の任意の `removedAt`。

### _収集しない_ もの

- 生の絶対フォルダーパスは収集しません（ハッシュ化された `rootId` + 短い表示ラベルのみ）。
- ファイル内容は収集しません。
- 実行ごとのログ、プロンプト、その他の CLI 出力は収集しません。
- レジストリにアップロードされていない skills は追跡しません（不明なスラッグは無視されます）。

## インストール数

skill ごとに 2 つのカウンターを維持します。

- `installsCurrent`: 現在、少なくとも 1 つのアクティブなルートにその skill をインストールしている一意のユーザー数。
- `installsAllTime`: これまでにその skill のインストールを報告したことがある一意のユーザー数。

### 複数のルート

複数のフォルダーから同期する場合、各スキャンルートを個別に扱います。skill は、**いずれか**のアクティブなルートに存在する場合に「現在インストール済み」と見なされます。

### アンインストールの検出

`sync` はルートごとの完全なセットを報告するため、次のように処理します。

- 次回の同期で skill がルートから消えた場合、そのルートでは削除済みとしてマークします。
- その skill がすべてのルートから削除された場合、`installsCurrent` にはカウントされなくなります。
- テレメトリを削除しない限り（下記参照）、`installsAllTime` は減少しません。

### 古さ（120 日）

**120 日間**テレメトリを報告しないルートは stale とマークされ、そのインストールは `installsCurrent` にカウントされなくなります。
これはバックグラウンドジョブを避けるため、遅延評価されます（次回のテレメトリ報告時）。

## 透明性 + ユーザー管理

ClawHub は、自分のプロフィールにプライベートな「インストール済み」タブを提供します。

- 保存されている正確なルート + インストール済み skills を表示します。
- **JSON エクスポート**ビューを含みます。
- アカウントに保存されているすべてのテレメトリを削除するための **テレメトリを削除**アクションを含みます。

他の全員に表示されるのは**集計されたインストールカウンター**のみです。他の人があなたのルート/フォルダーを見ることはできません。

アカウントを削除すると、テレメトリデータも削除されます。

## テレメトリを無効化する方法

環境変数を設定します。

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

これを設定すると、CLI は `clawhub sync` 中にテレメトリを送信しません。
