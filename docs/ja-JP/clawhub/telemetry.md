# テレメトリ

ClawHub は、CLI の最小限のテレメトリを使用して、インストール数の集計値を算出します。

## テレメトリが収集される条件

テレメトリは、次のすべての条件を満たす場合にのみ送信されます。

- CLI でログインしている。
- `clawhub install <slug>` を実行する。
- テレメトリが**無効化されていない**（下記の「無効化する方法」を参照）。

ログインしていない場合、何も報告されません。

## 収集する情報

報告対象となる `clawhub install` の実行ごとに、CLI はベストエフォートでインストールイベントを 1 件送信します。

イベントには次の情報が含まれます。

- `slug`：インストールされたスキルのスラッグ。
- `version`：判明している場合、インストールされたバージョン。

### 収集しない情報

- フォルダーパスや、フォルダーから派生した識別子。
- ファイルの内容。
- 実行ごとのログ、プロンプト、その他の CLI 出力。

## インストール数

ClawHub はスキルごとに次の集計カウンターを保持します。

- `installsAllTime`：そのスキルについて、CLI でのインストールを 1 回以上報告したユニークユーザー数。
- `installsCurrent`：インストールを報告し、テレメトリを削除していないユニークユーザー数。

## 透明性とユーザーによる制御

すべてのユーザーに表示されるのは、**集計されたインストールカウンター**のみです。

アカウントを削除すると、テレメトリデータも削除されます。

## テレメトリを無効化する方法

次の環境変数を設定します。

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

この変数を設定すると、CLI はインストールテレメトリを送信しません。
