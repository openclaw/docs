# テレメトリー

ClawHub は、集計インストール数を算出するために最小限の CLI テレメトリーを使用します。

## テレメトリーが収集されるタイミング

テレメトリーは次の場合にのみ送信されます。

- CLI にログインしている。
- `clawhub install <slug>` を実行する。
- テレメトリーが**無効化されていない**（下記の「無効にする方法」を参照）。

ログインしていない場合、何も報告されません。

## 収集する内容

報告対象の各 `clawhub install` で、CLI はベストエフォートのインストールイベントを 1 件送信します。

イベントには次が含まれます。

- `slug`: インストールされたスキルの slug。
- `version`: 既知の場合、インストールされたバージョン。

### 収集_しない_内容

- フォルダー パスやフォルダー由来の識別子は収集しません。
- ファイルの内容は収集しません。
- 実行ごとのログ、プロンプト、その他の CLI 出力は収集しません。

## インストール数

ClawHub はスキルごとに集計カウンターを保持します。

- `installsAllTime`: そのスキルについて少なくとも 1 回の CLI インストールを報告したユニークユーザー。
- `installsCurrent`: インストールを報告し、まだ自分のテレメトリーを削除していないユニークユーザー。

## 透明性 + ユーザー制御

誰もが見るのは**集計されたインストールカウンター**のみです。

アカウントを削除すると、テレメトリーデータも削除されます。

## テレメトリーを無効にする方法

環境変数を設定します。

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

これを設定すると、CLI はインストールテレメトリーを送信しません。
