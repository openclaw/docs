# テレメトリ

ClawHub は **最小限のテレメトリ** を使用して **インストール数** (実際に使用されているもの) を計算し、より良い並べ替え/フィルタリングを実現します。
これは CLI の `clawhub sync` コマンドに基づいています。

## テレメトリが収集されるタイミング

テレメトリは次の場合にのみ送信されます。

- CLI に **ログイン済み** である (同期/公開フローではすでに認証を必須にしています)。
- `clawhub sync` を実行する。
- テレメトリが **無効化されていない** (下記の「無効化する方法」を参照)。

ログインしていない場合、何も報告されません。

## 収集する内容

各 `clawhub sync` で、CLI は検出した内容の **完全なスナップショット** を、スキャンルート (「フォルダー/ルート」) ごとにグループ化して報告します。

各ルートについて、次を保存します。

- `rootId`: 正規化されたルートパスの **SHA-256 ハッシュ** (サーバーが生のパスを見ることはありません)。
- `label`: パスの最後の2つのセグメントから派生した、人間が読めるラベル (ホームパスは `~` で表示されます)。
- `firstSeenAt`、`lastSeenAt`、任意の `expiredAt`。

ルート配下で見つかった各 skill について、次を保存します。

- `skillId` (slug で解決されます。レジストリに存在する skill のみが追跡されます)。
- `firstSeenAt`、`lastSeenAt`。
- `lastVersion` (ベストエフォート。現在は既知の場合、レジストリに一致したバージョン)。
- 以前に報告されたインストールがルートから消えた場合の任意の `removedAt`。

### 収集 _しない_ 内容

- 生の絶対フォルダーパスはありません (ハッシュ化された `rootId` + 短い表示ラベルのみ)。
- ファイル内容はありません。
- 実行ごとのログ、プロンプト、その他の CLI 出力はありません。
- レジストリにアップロードされていない skills は追跡しません (不明な slugs は無視されます)。

## インストール数

skill ごとに2つのカウンターを維持します。

- `installsCurrent`: 現在、少なくとも1つのアクティブなルートにその skill をインストールしているユニークユーザー数。
- `installsAllTime`: これまでにその skill のインストールを報告したユニークユーザー数。

### 複数のルート

複数のフォルダーから同期する場合、各スキャンルートを個別に扱います。skill が **いずれかの** アクティブなルートに存在する場合、その skill は「現在インストール済み」です。

### アンインストール検出

`sync` はルートごとに完全なセットを報告するため、次のようになります。

- 次回の同期で skill がルートから消えた場合、そのルートでは削除済みとしてマークします。
- その skill がすべてのルートから削除された場合、`installsCurrent` にはカウントされなくなります。
- テレメトリを削除しない限り、`installsAllTime` は減少しません (下記参照)。

### 古さ (120日)

**120日** にわたってテレメトリを報告しないルートは古いものとしてマークされ、そのインストールは `installsCurrent` にカウントされなくなります。
これはバックグラウンドジョブを避けるため、遅延評価されます (次回のテレメトリ報告時)。

## 透明性 + ユーザー制御

ClawHub は、自分のプロフィールに非公開の「インストール済み」タブを提供します。

- 保存している正確なルート + インストール済み skills を表示します。
- **JSON エクスポート** ビューが含まれます。
- アカウントに保存されているすべてのテレメトリを削除する **テレメトリを削除** アクションが含まれます。

他の全員には **集計されたインストールカウンター** のみが表示されます。他の人があなたのルート/フォルダーを見ることはできません。

アカウントを削除すると、テレメトリデータも削除されます。

## テレメトリを無効化する方法

環境変数を設定します。

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

これが設定されている場合、CLI は `clawhub sync` 中にテレメトリを送信しません。
