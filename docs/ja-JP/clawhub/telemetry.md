# テレメトリ

ClawHub は、**インストール数**（実際に使用されているもの）を算出し、よりよい並べ替え/フィルタリングを支えるために、**最小限のテレメトリ**を使用します。
これは CLI の `clawhub sync` コマンドに基づいています。

## テレメトリが収集されるタイミング

テレメトリは次の場合にのみ送信されます。

- CLI で**ログイン済み**である（sync/publish フローではすでに認証を必須にしています）。
- `clawhub sync` を実行する。
- テレメトリが**無効化されていない**（下の「無効化する方法」を参照）。

ログインしていない場合は、何も報告されません。

## 収集する内容

各 `clawhub sync` で、CLI は見つけた内容の**完全なスナップショット**を、スキャンルート（「フォルダー/ルート」）ごとにグループ化して報告します。

各ルートについて、次を保存します。

- `rootId`: 正規化されたルートパスの **SHA-256 ハッシュ**（サーバーが生のパスを見ることはありません）。
- `label`: パスの最後の 2 セグメントから派生した人間が読めるラベル（ホームパスは `~` で表示されます）。
- `firstSeenAt`、`lastSeenAt`、任意の `expiredAt`。

ルート配下で見つかった各スキルについて、次を保存します。

- `skillId`（slug で解決されます。レジストリに存在するスキルのみが追跡されます）。
- `firstSeenAt`、`lastSeenAt`。
- `lastVersion`（ベストエフォート。現在は既知の場合、レジストリで一致したバージョン）。
- 以前に報告されたインストールがルートから消えた場合の任意の `removedAt`。

### 収集_しない_内容

- 生の絶対フォルダーパスは収集しません（ハッシュ化された `rootId` と短い表示ラベルのみ）。
- ファイル内容は収集しません。
- 実行ごとのログ、プロンプト、その他の CLI 出力は収集しません。
- レジストリにアップロードされていないスキルは追跡しません（不明な slug は無視されます）。

## インストール数

スキルごとに 2 つのカウンターを維持します。

- `installsCurrent`: 少なくとも 1 つのアクティブなルートに現在そのスキルをインストールしているユニークユーザー数。
- `installsAllTime`: これまでにそのスキルのインストールを報告したことがあるユニークユーザー数。

### 複数のルート

複数のフォルダーから同期する場合、各スキャンルートを個別に扱います。スキルが**いずれかの**アクティブなルートに存在する場合、そのスキルは「現在インストール済み」と見なされます。

### アンインストールの検出

`sync` はルートごとの完全なセットを報告するため、次のように扱います。

- 次回の同期でスキルがルートから消えた場合、そのルートでは削除済みとしてマークします。
- すべてのルートからスキルが削除されると、`installsCurrent` にはカウントされなくなります。
- テレメトリを削除しない限り、`installsAllTime` は減りません（下記参照）。

### 古さ（120 日）

**120 日間**テレメトリを報告しないルートは古いものとしてマークされ、そのインストールは `installsCurrent` にカウントされなくなります。
バックグラウンドジョブを避けるため、これは遅延評価されます（次回のテレメトリ報告時）。

## 透明性 + ユーザー制御

ClawHub は、自分のプロフィールに非公開の「インストール済み」タブを提供します。

- 保存している正確なルート + インストール済みスキルを表示します。
- **JSON エクスポート**ビューを含みます。
- アカウントに保存されているすべてのテレメトリを削除する**テレメトリを削除**アクションを含みます。

それ以外の全員には、**集計されたインストールカウンター**のみが表示されます。他の人があなたのルート/フォルダーを見ることはできません。

アカウントを削除すると、テレメトリデータも削除されます。

## テレメトリを無効化する方法

環境変数を設定します。

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

これを設定すると、CLI は `clawhub sync` 中にテレメトリを送信しません。
