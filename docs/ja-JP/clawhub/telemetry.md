# テレメトリ

ClawHub は **最小限のテレメトリ** を使用して **インストール数**（実際に使用されているもの）を計算し、より良い並べ替え/フィルタリングに活用します。
これは CLI の `clawhub sync` コマンドに基づいています。

## テレメトリが収集されるタイミング

テレメトリが送信されるのは次の場合のみです。

- CLI で **ログイン済み** である（sync/publish フローではすでに認証を必須にしています）。
- `clawhub sync` を実行する。
- テレメトリが **無効化されていない**（下の「無効化する方法」を参照）。

ログインしていない場合、何も報告されません。

## 収集する内容

各 `clawhub sync` で、CLI は検出した内容の **完全なスナップショット** を、スキャンルート（「フォルダー/ルート」）ごとにグループ化して報告します。

各ルートについて、次を保存します。

- `rootId`: 正規化されたルートパスの **SHA-256 ハッシュ**（サーバーは生のパスを一切見ません）。
- `label`: 最後の 2 つのパスセグメントから派生した、人間が読めるラベル（ホームパスは `~` で表示されます）。
- `firstSeenAt`, `lastSeenAt`, 任意の `expiredAt`。

ルート配下で見つかった各スキルについて、次を保存します。

- `skillId`（slug で解決されます。レジストリに存在するスキルのみが追跡されます）。
- `firstSeenAt`, `lastSeenAt`。
- `lastVersion`（ベストエフォート。現在は、既知の場合はレジストリで一致したバージョン）。
- 以前に報告されたインストールがルートから消えた場合の任意の `removedAt`。

### 収集しない内容

- 生の絶対フォルダーパスは収集しません（ハッシュ化された `rootId` + 短い表示ラベルのみ）。
- ファイル内容は収集しません。
- 実行ごとのログ、プロンプト、その他の CLI 出力は収集しません。
- レジストリにアップロードされていないスキルは追跡しません（未知の slug は無視されます）。

## インストール数

スキルごとに 2 つのカウンターを維持します。

- `installsCurrent`: 現在、少なくとも 1 つのアクティブルートにそのスキルをインストールしているユニークユーザー数。
- `installsAllTime`: そのスキルがインストールされていることをこれまでに報告したユニークユーザー数。

### 複数のルート

複数のフォルダーから sync する場合、各スキャンルートを独立して扱います。スキルが **いずれか** のアクティブルートに存在する場合、そのスキルは「現在インストール済み」とみなされます。

### アンインストール検出

`sync` はルートごとに完全なセットを報告するため、次のようになります。

- 次回の sync でスキルがルートから消えた場合、そのルートでは削除済みとしてマークします。
- スキルがすべてのルートから削除された場合、`installsCurrent` にはカウントされなくなります。
- テレメトリを削除しない限り、`installsAllTime` は減少しません（下記参照）。

### 古さ（120 日）

**120 日間** テレメトリを報告しないルートは古くなったものとしてマークされ、そのインストールは `installsCurrent` にカウントされなくなります。
これはバックグラウンドジョブを避けるため、遅延評価されます（次回のテレメトリ報告時）。

## 透明性 + ユーザー制御

ClawHub は、自分のプロフィールに非公開の「インストール済み」タブを提供します。

- 保存している正確なルート + インストール済みスキルを表示します。
- **JSON エクスポート** ビューを含みます。
- アカウントに保存されたすべてのテレメトリを削除する **テレメトリを削除** アクションを含みます。

他のすべての人には **集計されたインストールカウンター** のみが表示されます。他の人があなたのルート/フォルダーを見ることはできません。

アカウントを削除すると、テレメトリデータも削除されます。

## テレメトリを無効化する方法

環境変数を設定します。

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

これを設定すると、CLI は `clawhub sync` 中にテレメトリを送信しません。
