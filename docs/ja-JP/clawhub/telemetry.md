# テレメトリー

ClawHub は、集計インストール数を算出するために最小限の CLI テレメトリーを使用します。

## テレメトリーが収集されるタイミング

テレメトリーは、次の場合にのみ送信されます。

- CLI にログインしている。
- `clawhub install <slug>` を実行する。
- テレメトリーが**無効化されていない**（下記の「無効化する方法」を参照）。

ログインしていない場合、何も報告されません。

## 収集する内容

報告対象の各 `clawhub install` で、CLI はベストエフォートのインストールイベントを 1 件送信します。

イベントには次が含まれます。

- `slug`: インストールされたスキルの slug。
- `version`: 判明している場合は、インストールされたバージョン。

### 収集_しない_内容

- フォルダーのパスやフォルダー由来の識別子はありません。
- ファイル内容はありません。
- 実行ごとのログ、プロンプト、その他の CLI 出力はありません。

## インストール数

ClawHub はスキルごとの集計カウンターを保持します。

- `installsAllTime`: そのスキルについて少なくとも 1 回の CLI インストールを報告したユニークユーザー。
- `installsCurrent`: インストールを報告しており、自分の
  テレメトリーを削除していないユニークユーザー。

## 透明性 + ユーザー制御

すべての人に表示されるのは**集計されたインストールカウンター**のみです。

アカウントを削除すると、テレメトリーデータも削除されます。

## テレメトリーを無効化する方法

環境変数を設定します。

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

これを設定すると、CLI はインストールテレメトリーを送信しません。
