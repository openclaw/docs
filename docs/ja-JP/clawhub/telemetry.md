# テレメトリ

ClawHub は **最小限のテレメトリ** を使用して **インストール数**（実際に使用されているもの）を計算し、より良い並べ替えやフィルタリングに活用します。
これは CLI の `clawhub sync` コマンドに基づいています。

## テレメトリが収集される場合

テレメトリは次の場合にのみ送信されます。

- CLI で **ログイン済み** である（同期/公開フローではすでに認証を必須にしています）。
- `clawhub sync` を実行する。
- テレメトリが **無効化されていない**（下の「無効化する方法」を参照）。

ログインしていない場合、何も報告されません。

## 収集する内容

各 `clawhub sync` で、CLI は検出した内容の **完全なスナップショット** を、スキャンルート（「フォルダー/ルート」）ごとにグループ化して報告します。

各ルートについて保存する内容:

- `rootId`: 正規化されたルートパスの **SHA-256 ハッシュ**（サーバーが生のパスを見ることはありません）。
- `label`: 最後の 2 つのパスセグメントから派生した、人間が読めるラベル（ホームパスは `~` で表示されます）。
- `firstSeenAt`、`lastSeenAt`、任意の `expiredAt`。

ルート配下で見つかった各スキルについて保存する内容:

- `skillId`（スラッグで解決。レジストリに存在するスキルのみ追跡されます）。
- `firstSeenAt`、`lastSeenAt`。
- `lastVersion`（ベストエフォート。現在は、既知であればレジストリで一致したバージョン）。
- 以前に報告されたインストールがルートから消えた場合の任意の `removedAt`。

### 収集 _しない_ 内容

- 生の絶対フォルダーパスは収集しません（ハッシュ化された `rootId` と短い表示ラベルのみ）。
- ファイル内容は収集しません。
- 実行ごとのログ、プロンプト、その他の CLI 出力は収集しません。
- レジストリにアップロードされていないスキルは追跡しません（未知のスラッグは無視されます）。

## インストール数

スキルごとに 2 つのカウンターを保持します。

- `installsCurrent`: 少なくとも 1 つのアクティブなルートに現在そのスキルをインストールしている一意のユーザー数。
- `installsAllTime`: これまでにそのスキルがインストールされていると報告した一意のユーザー数。

### 複数のルート

複数のフォルダーから同期する場合、各スキャンルートを個別に扱います。スキルは、**いずれか** のアクティブなルートに存在する場合に「現在インストール済み」とみなされます。

### アンインストールの検出

`sync` はルートごとの完全なセットを報告するため:

- 次回の同期でスキルがルートから消えた場合、そのルートでは削除済みとしてマークします。
- スキルがすべてのルートから削除されると、`installsCurrent` にはカウントされなくなります。
- テレメトリを削除しない限り、`installsAllTime` は減少しません（下記参照）。

### 鮮度切れ（120 日）

**120 日間** テレメトリを報告していないルートは古いものとしてマークされ、そのインストールは `installsCurrent` にカウントされなくなります。
バックグラウンドジョブを避けるため、これは遅延評価されます（次回のテレメトリ報告時）。

## 透明性 + ユーザー制御

ClawHub は、自分のプロフィールに非公開の「インストール済み」タブを提供します。

- 保存している正確なルート + インストール済みスキルを表示します。
- **JSON エクスポート** ビューを含みます。
- アカウントに保存されているすべてのテレメトリを削除する **テレメトリを削除** アクションを含みます。

他のユーザーに表示されるのは **集計されたインストールカウンター** のみです。他のユーザーがあなたのルート/フォルダーを見ることはできません。

アカウントを削除すると、テレメトリデータも削除されます。

## テレメトリを無効化する方法

環境変数を設定します。

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

これを設定すると、CLI は `clawhub sync` 中にテレメトリを送信しません。
