# テレメトリ

ClawHub は **最小限のテレメトリ**を使用して、**インストール数**（実際に使用されているもの）を計算し、より良い並べ替え/フィルタリングを実現します。
これは CLI の `clawhub sync` コマンドに基づいています。

## テレメトリが収集されるタイミング

テレメトリは次の場合にのみ送信されます。

- CLI で**ログイン済み**である（sync/publish フローではすでに認証が必要です）。
- `clawhub sync` を実行する。
- テレメトリが**無効化されていない**（下記の「無効にする方法」を参照）。

ログインしていない場合、何も報告されません。

## 収集する内容

各 `clawhub sync` で、CLI は検出した内容の**完全なスナップショット**を、スキャンルート（「フォルダー/ルート」）ごとにグループ化して報告します。

各ルートについて保存する内容:

- `rootId`: 正規化されたルートパスの **SHA-256 ハッシュ**（サーバーが生のパスを見ることはありません）。
- `label`: 最後の 2 つのパスセグメントから派生した、人間が読めるラベル（ホームパスは `~` で表示されます）。
- `firstSeenAt`, `lastSeenAt`, 任意の `expiredAt`。

ルート配下で見つかった各スキルについて保存する内容:

- `skillId`（スラッグで解決されます。レジストリに存在するスキルのみが追跡されます）。
- `firstSeenAt`, `lastSeenAt`。
- `lastVersion`（ベストエフォート。現在は、既知の場合はレジストリで一致したバージョン）。
- 以前に報告されたインストールがルートから消えた場合の任意の `removedAt`。

### 収集しない内容

- 生の絶対フォルダーパスは収集しません（ハッシュ化された `rootId` + 短い表示ラベルのみ）。
- ファイル内容は収集しません。
- 実行ごとのログ、プロンプト、その他の CLI 出力は収集しません。
- レジストリにアップロードされていないスキルは追跡しません（不明なスラッグは無視されます）。

## インストール数

スキルごとに 2 つのカウンターを維持します。

- `installsCurrent`: 少なくとも 1 つのアクティブなルートに現在そのスキルをインストールしている一意のユーザー数。
- `installsAllTime`: そのスキルがインストールされていると過去に報告した一意のユーザー数。

### 複数のルート

複数のフォルダーから同期する場合、各スキャンルートを個別に扱います。スキルが**いずれか**のアクティブなルートに存在する場合、「現在インストール済み」と見なされます。

### アンインストールの検出

`sync` はルートごとに完全なセットを報告するため:

- 次回の同期でスキルがルートから消えた場合、そのルートでは削除済みとしてマークします。
- そのスキルがすべてのルートから削除された場合、`installsCurrent` にはカウントされなくなります。
- `installsAllTime` は、テレメトリを削除しない限り減少しません（下記参照）。

### 古さ（120 日）

**120 日**間テレメトリを報告していないルートは古いものとしてマークされ、そのインストールは `installsCurrent` にカウントされなくなります。
これはバックグラウンドジョブを避けるため、遅延評価されます（次回のテレメトリ報告時）。

## 透明性 + ユーザーによる制御

ClawHub は、自分のプロフィールに非公開の「インストール済み」タブを提供します。

- 保存している正確なルート + インストール済みスキルを表示します。
- **JSON エクスポート**ビューを含みます。
- アカウントに保存されているすべてのテレメトリを削除するための **テレメトリを削除**アクションを含みます。

他の全員に表示されるのは**集計されたインストールカウンター**のみです。他の人があなたのルート/フォルダーを見ることはできません。

アカウントを削除すると、テレメトリデータも削除されます。

## テレメトリを無効にする方法

環境変数を設定します。

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

これを設定すると、CLI は `clawhub sync` 中にテレメトリを送信しません。
