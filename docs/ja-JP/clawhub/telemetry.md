# テレメトリ

ClawHub は **最小限のテレメトリ** を使用して **インストール数**（実際に使われているもの）を計算し、よりよいソート/フィルタリングを提供します。
これは CLI の `clawhub sync` コマンドに基づいています。

## テレメトリが収集されるタイミング

テレメトリは次の場合にのみ送信されます。

- CLI で **ログイン済み** である（sync/publish フローではすでに認証が必要です）。
- `clawhub sync` を実行する。
- テレメトリが **無効化されていない**（下の「無効化する方法」を参照）。

ログインしていない場合、何も報告されません。

## 収集するもの

各 `clawhub sync` で、CLI は見つけた内容の **完全なスナップショット** を、スキャンルート（「フォルダー/ルート」）ごとにグループ化して報告します。

各ルートについて保存するもの:

- `rootId`: 正規化されたルートパスの **SHA-256 ハッシュ**（サーバーは生のパスを一切見ません）。
- `label`: パスの最後の 2 セグメントから派生した、人間が読めるラベル（ホームパスは `~` で表示されます）。
- `firstSeenAt`, `lastSeenAt`, 任意の `expiredAt`。

ルート配下で見つかった各 Skill について保存するもの:

- `skillId`（slug で解決。レジストリに存在する Skills のみが追跡されます）。
- `firstSeenAt`, `lastSeenAt`。
- `lastVersion`（ベストエフォート。現在は、既知の場合はレジストリで一致したバージョン）。
- 以前に報告されたインストールがルートから消えた場合の任意の `removedAt`。

### 収集_しない_もの

- 生の絶対フォルダーパスは収集しません（ハッシュ化された `rootId` + 短い表示ラベルのみ）。
- ファイル内容は収集しません。
- 実行ごとのログ、プロンプト、その他の CLI 出力は収集しません。
- レジストリにアップロードされていない Skills は追跡しません（不明な slug は無視されます）。

## インストール数

Skill ごとに 2 つのカウンターを維持します。

- `installsCurrent`: 現在、少なくとも 1 つのアクティブなルートにその Skill をインストールしている一意のユーザー数。
- `installsAllTime`: これまでにその Skill のインストールを報告したことがある一意のユーザー数。

### 複数のルート

複数のフォルダーから sync する場合、各スキャンルートを個別に扱います。Skill は、**いずれか** のアクティブなルートに存在する場合、「現在インストール済み」と見なされます。

### アンインストール検出

`sync` はルートごとの完全なセットを報告するため:

- 次回の sync で Skill がルートから消えた場合、そのルートでは削除済みとしてマークします。
- すべてのルートから Skill が削除された場合、`installsCurrent` にはカウントされなくなります。
- テレメトリを削除しない限り、`installsAllTime` は減少しません（下記参照）。

### 古さ（120 日）

**120 日** 間テレメトリを報告しないルートは stale とマークされ、そのインストールは `installsCurrent` にカウントされなくなります。
これはバックグラウンドジョブを避けるため、遅延評価されます（次回のテレメトリ報告時）。

## 透明性 + ユーザーコントロール

ClawHub は、自分のプロフィールに非公開の「インストール済み」タブを提供します。

- 保存している正確なルート + インストール済み Skills を表示します。
- **JSON エクスポート** ビューを含みます。
- アカウントに保存されているすべてのテレメトリを削除する **テレメトリを削除** アクションを含みます。

他の全員には **集計されたインストールカウンター** のみが表示されます。他の人があなたのルート/フォルダーを見ることはできません。

アカウントを削除すると、テレメトリデータも削除されます。

## テレメトリを無効化する方法

環境変数を設定します。

```bash
export CLAWHUB_DISABLE_TELEMETRY=1
```

これを設定すると、CLI は `clawhub sync` 中にテレメトリを送信しません。
