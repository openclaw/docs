# モデレーションとアカウントの安全性

ClawHub は公開を受け付けていますが、公開ディスカバリーとインストール画面には
ガードレールが必要です。報告、モデレーション保留、非表示のリスティング、アカウント措置は、
リリースやアカウントが安全でない、誤解を招く、またはポリシー違反に見える場合に
ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウントの状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)と
[許容される使用](/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ権利に関する懸念は、
[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みのユーザーは、スキル、プラグイン、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 未申告の認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の不正使用
- [許容される使用](/clawhub/acceptable-usage)に違反するコンテンツ

スキルページの**スキルを報告**ボタン、またはパッケージ向けの報告
コマンド/API を使用してください。

サードパーティのスキルまたはプラグイン自身のソースコードにある脆弱性には、ClawHub の報告を使用しないでください。
それらは、リスティングからリンクされている公開者またはソースリポジトリへ直接報告してください。
ClawHub はサードパーティのスキルまたはプラグインのコードを保守またはパッチ適用しません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性向けです。
例として、Web サイト、API、CLI、レジストリ、認証、スキャン、モデレーション、またはダウンロード/インストールの信頼境界にあるバグが含まれます。
サードパーティのスキルまたはプラグインの脆弱性に ClawHub のアドバイザリを使用しないでください。

良い報告は具体的で実行可能です。報告の悪用自体が
アカウント措置につながることがあります。

## 組織と名前空間の主張

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争は、
製品内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の主張](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移転、名前変更、非表示、隔離、エイリアス化、またはその他の方法でレビューすべきだという
非機密の証拠を ClawHub スタッフに確認してもらう必要がある場合は、このプロセスを使用してください。
公開 issue に、シークレット、非公開文書、非公開の法的ファイル、本人確認書類、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

重大な検出事項やポリシー上の問題により、公開者またはリスティングが
モデレーション保留になることがあります。この場合、影響を受けるコンテンツは公開ディスカバリーから非表示になるか、
今後の公開がレビュー完了まで非表示で開始されることがあります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間にユーザーを保護するためのものです。
誤検知が確認された場合は解除されることもあります。

## 非表示またはブロックされたリスティング

リスティングは、公開インストール画面で、保留、非表示、隔離、失効、またはその他の理由で利用不可になることがあります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、モデレーションによって復元されるまで、
そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示のリスティングについて、引き続き診断情報を確認できる場合があります。
これらの診断情報は、何が起きたか、およびリスティングを公開画面に戻す前に何を変更する必要があるかを説明するのに役立ちます。

## 禁止措置とアカウントの状態

ClawHub ポリシーに違反するアカウントは、公開アクセスを失うことがあります。重大な悪用は、
アカウントの禁止、トークンの失効、コンテンツの非表示、またはリスティングの削除につながることがあります。
公開者の悪用圧力シグナルは毎日確認されます。シグナルが
ClawHub の潜在的な禁止しきい値に達すると、自動警告がトリガーされることがあります。警告期限後の次の
対象スキャンでも公開者が潜在的な禁止しきい値に入っている場合、
ClawHub はアカウント措置を自動的に適用することがあります。
信頼度が低いシグナルや時間的に限定されたレビューシグナルは、自動執行の対象外です。

削除、禁止、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が
失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。
禁止または無効化されたアカウントによってサインインや通常の CLI アクセスがブロックされている場合は、
復旧レビューに [ClawHub appeal form](https://appeals.openclaw.ai/) を使用してください。

スキャナーによってトリガーされたメールが、スキルまたはプラグインのバージョンを悪意ありとして示している場合は、
ブロックされた提出バージョンの保存済みスキャン結果をダウンロードしてください。
`clawhub scan download <slug> --version <version>`。プラグインの場合は
`--kind plugin` を追加します。スキャン出力を確認し、リスティングを修正し、バージョン番号を増やして、
修正済みバージョンをアップロードしてください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには、次のようにします。

- 名前、概要、タグ、変更履歴を正確に保つ
- 必須の環境変数と権限を申告する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- プラグインを公開する前にドライランを使用する
- ユーザーやモデレーターがリリースの動作について質問した場合は明確に回答する
