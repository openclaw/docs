# モデレーションとアカウント安全性

ClawHub は公開を受け付けていますが、公開ディスカバリーとインストール面にはなお
ガードレールが必要です。レポート、モデレーション保留、非表示のリスティング、アカウント措置は、
リリースやアカウントが安全でない、誤解を招く、またはポリシー違反に見える場合に
ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウント状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)と
[利用規定](/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ権利に関する懸念は、
[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## レポート

サインイン済みユーザーは、Skills、Plugin、パッケージを報告できます。

ClawHub のレポートは、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 未申告の認証情報または権限要件
- 疑わしいインストール手順
- なりすまし
- 悪意ある登録または商標の不正利用
- [利用規定](/clawhub/acceptable-usage)に違反するコンテンツ

Skills ページの **Skills を報告** ボタン、またはパッケージ用のパッケージ報告
コマンド/APIを使用してください。

サードパーティの Skills または
Plugin 自体のソースコードにある脆弱性には、ClawHub のレポートを使用しないでください。それらは、リスティングからリンクされている公開者またはソース
リポジトリに直接報告してください。ClawHub はサードパーティの Skills または Plugin コードを保守または修正しません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性を対象としています。例には、Webサイト、API、CLI、レジストリ、認証、
スキャン、モデレーション、またはダウンロード/インストールの信頼境界におけるバグが含まれます。サードパーティの Skills または Plugin の脆弱性に ClawHub の
アドバイザリを使用しないでください。

良いレポートは具体的で実行可能です。報告の濫用は、それ自体が
アカウント措置につながることがあります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争は、
製品内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移管、改名、非表示、隔離、エイリアス化、
またはその他の形でレビューすべきであることを示す非機密の証拠を ClawHub スタッフにレビューしてもらう必要がある場合は、そのプロセスを使用してください。公開 Issue に、シークレット、非公開文書、非公開の法的
ファイル、個人身分証明書、API トークン、または DNS チャレンジトークンを含めないでください。

## モデレーション保留

重大な検出事項やポリシー問題によって、公開者またはリスティングが
モデレーション保留になる場合があります。この場合、影響を受けるコンテンツが公開
ディスカバリーから非表示になったり、今後の公開が問題のレビューまで非表示で開始されたりすることがあります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間、
ユーザーを保護することを目的としています。誤検出が確認された場合には解除されることもあります。

## 非表示またはブロックされたリスティング

リスティングは、公開インストール面で保留、非表示、隔離、取り消し、またはその他の理由で利用不可になる場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するかモデレーションが復元するまで、
そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示のリスティングに関する診断を引き続き確認できる場合があります。これらの
診断は、何が起きたのか、リスティングが公開面に戻る前に何を変更する必要があるのかを説明するのに役立ちます。

## 禁止措置とアカウント状態

ClawHub ポリシーに違反するアカウントは、公開アクセスを失う場合があります。重大な濫用は、
アカウント禁止、トークン失効、コンテンツの非表示、またはリスティング削除につながる可能性があります。
公開者の濫用圧力シグナルは毎日チェックされます。
ClawHub の潜在的な禁止しきい値に達したシグナルは、自動警告をトリガーすることがあります。警告期限後の次の
対象スキャンでも公開者が
潜在的な禁止しきい値に該当する場合、ClawHub はアカウント措置を自動的に適用することがあります。
信頼度の低いシグナルや時間範囲が限定されたレビューシグナルは、自動
執行の対象外です。

削除、禁止、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が
失敗し始めた場合は、Web UI にサインインしてアカウント
状態を確認してください。禁止または無効化されたアカウントによってサインインや通常の CLI アクセスがブロックされている場合は、
復旧レビューに [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーによって送信されたメールで Skills または Plugin のバージョンが悪意ありと示されている場合は、
ブロックされた送信済みバージョンの保存済みスキャン結果をダウンロードしてください:
`clawhub scan download <slug> --version <version>`。Plugin の場合は、
`--kind plugin` を追加してください。スキャン出力をレビューし、リスティングを修正し、バージョン
番号を増やして、修正版をアップロードしてください。

## 公開者向けガイダンス

誤検出を減らし、ユーザーからの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を申告する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- Plugin を公開する前にドライランを使用する
- ユーザーまたはモデレーターからリリース動作について質問された場合は明確に回答する
