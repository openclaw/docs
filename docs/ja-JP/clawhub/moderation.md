# モデレーションとアカウントの安全性

ClawHub では公開が可能ですが、公開ディスカバリーとインストール面には引き続き
ガードレールが必要です。レポート、モデレーション保留、非表示リスティング、アカウント措置は、
リリースやアカウントが安全でない、誤解を招く、またはポリシー違反に見える場合に
ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウントの状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)と
[許容される使用](/clawhub/acceptable-usage)も参照してください。著作権またはその他のコンテンツ権利に関する懸念については、
[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## レポート

サインイン済みユーザーは、スキル、プラグイン、パッケージを報告できます。

ClawHub のレポートは、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 未宣言の認証情報または権限要件
- 疑わしいインストール手順
- なりすまし
- 悪意ある登録または商標の悪用
- [許容される使用](/clawhub/acceptable-usage)に違反するコンテンツ

スキルページの **スキルを報告** ボタン、またはパッケージ用の報告コマンド/APIを使用してください。

サードパーティのスキルまたはプラグイン自身のソースコードにある脆弱性には、ClawHub のレポートを使用しないでください。それらは、リスティングからリンクされている発行者またはソースリポジトリに直接報告してください。ClawHub はサードパーティのスキルまたはプラグインコードを保守またはパッチ適用しません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性を対象としています。例として、Web サイト、API、CLI、レジストリ、認証、スキャン、モデレーション、またはダウンロード/インストールの信頼境界にあるバグが含まれます。サードパーティのスキルまたはプラグインの脆弱性に ClawHub advisories を使用しないでください。

良いレポートは具体的で、対応可能です。報告の悪用は、それ自体がアカウント措置につながることがあります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権をめぐる紛争には、製品内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移転、名称変更、非表示、隔離、エイリアス化、またはその他の形でレビューすべきであることについて、ClawHub スタッフに非機密の証拠をレビューしてもらう必要がある場合は、そのプロセスを使用してください。公開 Issue にシークレット、非公開文書、非公開の法的ファイル、個人の身分証明書、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

重大な検出事項やポリシー問題によって、発行者またはリスティングがモデレーション保留になることがあります。この場合、影響を受けるコンテンツは公開ディスカバリーから非表示になることがあり、今後の公開も、問題がレビューされるまで非表示で開始されることがあります。

モデレーション保留は、ClawHub が高リスクのケースを解決している間にユーザーを保護するためのものです。誤検知が確認された場合には解除されることもあります。

## 非表示またはブロックされたリスティング

リスティングは、公開インストール面で保留、非表示、隔離、失効、またはその他の理由で利用できない状態になることがあります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、モデレーションによって復元されるまで、そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示のリスティングについて診断情報を引き続き確認できる場合があります。これらの診断情報は、何が起きたのか、リスティングを公開面に戻す前に何を変更する必要があるのかを説明するのに役立ちます。

## BAN とアカウントの状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失うことがあります。重大な悪用は、アカウントの BAN、トークン失効、コンテンツの非表示、またはリスティングの削除につながることがあります。
発行者の悪用圧力シグナルは毎日チェックされます。
ClawHub の潜在的 BAN しきい値に達したシグナルは、自動警告をトリガーすることがあります。警告期限後の次回の対象スキャンでも、発行者が潜在的 BAN しきい値にある場合、ClawHub はアカウント措置を自動的に適用することがあります。
信頼度が低いレビューシグナルや時間的に限定されたレビューシグナルは、自動執行の対象外です。

削除、BAN、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。BAN または無効化されたアカウントによってサインインや通常の CLI アクセスがブロックされている場合は、復旧レビューに [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーがトリガーしたメールで、スキルまたはプラグインのバージョンが悪意のあるものとして名指しされている場合は、ブロックされた送信済みバージョンの保存済みスキャン結果をダウンロードしてください。
`clawhub scan download <slug> --version <version>`。プラグインの場合は、
`--kind plugin` を追加します。スキャン出力を確認し、リスティングを修正し、バージョン番号を上げて、修正済みバージョンをアップロードしてください。

## 発行者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには、次のことを行ってください。

- 名前、概要、タグ、変更ログを正確に保つ
- 必要な環境変数と権限を宣言する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- プラグインを公開する前にドライランを使用する
- ユーザーまたはモデレーターがリリースの挙動について尋ねた場合は明確に回答する
