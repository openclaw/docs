# モデレーションとアカウント安全性

ClawHub は公開を受け付けていますが、公開ディスカバリーとインストール面には引き続きガードレールが必要です。報告、モデレーション保留、非表示のリスティング、アカウント措置は、リリースやアカウントが安全でない、誤解を招く、またはポリシーに反しているように見える場合にユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウント状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious` などの監査ラベルやリスクレベルについては、
[セキュリティ監査](/ja-JP/clawhub/security-audits)を参照してください。

[セキュリティ](/ja-JP/clawhub/security)と
[許容される使用](/ja-JP/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ権利に関する懸念については、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。

## 報告

サインイン済みのユーザーは、スキル、Plugin、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 未申告の認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の不正使用
- [許容される使用](/ja-JP/clawhub/acceptable-usage)に違反するコンテンツ

スキルページの **スキルを報告** ボタン、またはパッケージ向けのパッケージ報告コマンド/API を使用してください。

サードパーティのスキルまたはPlugin自身のソースコード内の脆弱性には、ClawHub の報告を使用しないでください。それらは、リスティングからリンクされている発行者またはソースリポジトリに直接報告してください。ClawHub はサードパーティのスキルまたはPluginコードを保守したりパッチしたりしません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性のためのものです。例として、ウェブサイト、API、CLI、レジストリ、認証、スキャン、モデレーション、またはダウンロード/インストールの信頼境界にあるバグが含まれます。サードパーティのスキルやPluginの脆弱性に ClawHub のアドバイザリを使用しないでください。

良い報告は具体的で、対応可能です。報告の悪用は、それ自体がアカウント措置につながる場合があります。

## 組織と名前空間の申請

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争は、製品内の報告フローやアカウント異議申し立てフォームではなく、[組織と名前空間の申請](/ja-JP/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移管、名前変更、非表示、隔離、エイリアス化、またはその他の方法でレビューすべきであることを示す非機密の証拠を ClawHub スタッフにレビューしてもらう必要がある場合は、このプロセスを使用してください。公開 issue には、シークレット、非公開文書、非公開の法務ファイル、個人身分証明書、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

一部の重大な検出事項やポリシー上の問題により、発行者またはリスティングがモデレーション保留になることがあります。これが発生すると、影響を受けるコンテンツが公開ディスカバリーから非表示になったり、今後の公開がレビュー完了まで非表示で開始されたりする場合があります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間にユーザーを保護することを目的としています。誤検知が確認された場合には解除することもできます。

## 非表示またはブロックされたリスティング

リスティングは、保留、非表示、隔離、取り消し、またはその他の理由で公開インストール面で利用できない状態になる場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、モデレーションによって復元されるまでは、そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示のリスティングについて診断情報を引き続き確認できる場合があります。これらの診断情報は、何が起きたのか、リスティングを公開面に戻す前に何を変更する必要があるのかを説明するのに役立ちます。

## 利用停止とアカウント状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な悪用は、アカウントの利用停止、トークンの取り消し、コンテンツの非表示、またはリスティングの削除につながることがあります。発行者の不正利用リスクを示すシグナルは毎日チェックされます。ClawHub の潜在的な利用停止しきい値に達したシグナルは、自動警告を引き起こす場合があります。警告期限後の次の対象スキャンでも発行者が潜在的な利用停止しきい値に該当する場合、ClawHub はアカウント措置を自動的に適用することがあります。信頼度の低いシグナルや時間的に限定されたレビューシグナルは、自動執行の対象外です。

削除済み、利用停止中、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。利用停止または無効化されたアカウントによってサインインや通常の CLI アクセスがブロックされている場合は、復旧レビューのために [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーが送信したメールで、スキルまたはPluginバージョンが悪意のあるものとして示されている場合は、ブロックされた送信バージョンの保存済みスキャン結果をダウンロードしてください:
`clawhub scan download <slug> --version <version>`。Plugin の場合は
`--kind plugin` を追加してください。スキャン出力を確認し、リスティングを修正し、バージョン番号を増やして、修正版をアップロードしてください。

## 発行者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、要約、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を申告する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- Plugin を公開する前にドライランを使用する
- ユーザーまたはモデレーターがリリースの挙動について質問した場合は、明確に回答する
