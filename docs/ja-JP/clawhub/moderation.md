# モデレーションとアカウントの安全性

ClawHub は公開に開かれていますが、公開ディスカバリーとインストール面には
引き続きガードレールが必要です。報告、モデレーション保留、非表示リスティング、アカウント措置は、
リリースやアカウントが安全でない、誤解を招く、またはポリシーに違反しているように見える場合に、
ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウント状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)および
[利用規約](/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ権利に関する懸念については、[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みのユーザーは、スキル、プラグイン、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 未申告の認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の不正使用
- [利用規約](/clawhub/acceptable-usage)に違反するコンテンツ

スキルページの**スキルを報告**ボタン、またはパッケージ向けの報告
コマンド/API を使用してください。

サードパーティのスキルまたはプラグイン自体のソースコードにある脆弱性については、ClawHub の報告を使用しないでください。それらは、リスティングからリンクされている公開者またはソースリポジトリに直接報告してください。ClawHub はサードパーティのスキルまたはプラグインのコードを保守したりパッチ適用したりしません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性を対象としています。例としては、ウェブサイト、API、CLI、レジストリ、認証、スキャン、モデレーション、ダウンロード/インストールの信頼境界にあるバグが含まれます。サードパーティのスキルまたはプラグインの脆弱性には、ClawHub のアドバイザリを使用しないでください。

良い報告は具体的で、対応可能です。報告の悪用は、それ自体がアカウント措置につながる可能性があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争は、
製品内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移管、名称変更、非表示、隔離、エイリアス化、またはその他の方法でレビューすべきであることを示す非機密の証拠を ClawHub スタッフに確認してもらう必要がある場合は、そのプロセスを使用してください。公開 issue には、シークレット、私的文書、私的な法的ファイル、本人確認書類、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

重大な検出事項やポリシー問題によって、公開者またはリスティングがモデレーション保留になる場合があります。これが発生すると、影響を受けるコンテンツが公開ディスカバリーから非表示になったり、今後の公開がレビュー完了まで非表示として開始されたりすることがあります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間にユーザーを保護することを目的としています。誤検出が確認された場合は解除されることもあります。

## 非表示またはブロックされたリスティング

リスティングは、保留、非表示、隔離、取り消し、またはその他の形で公開インストール面で利用できない状態になる場合があります。

これらの状態のいずれかが表示された場合は、所有者が問題を解決するか、モデレーションによって復元されるまで、そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示のリスティングについて診断情報を引き続き確認できる場合があります。これらの診断情報は、何が起きたのか、リスティングを公開面に戻す前に何を変更する必要があるのかを説明するのに役立ちます。

## 禁止とアカウント状態

ClawHub ポリシーに違反するアカウントは、公開アクセスを失う可能性があります。重大な悪用は、アカウントの禁止、トークンの取り消し、コンテンツの非表示、またはリスティングの削除につながる可能性があります。
公開者の悪用圧力シグナルは毎日チェックされます。
ClawHub の潜在的な禁止しきい値に達したシグナルは、自動警告を引き起こす場合があります。警告期限後の次回の対象スキャンでも、公開者が潜在的な禁止しきい値に該当する場合、ClawHub はアカウント措置を自動的に適用することがあります。
信頼度の低いシグナルおよび時間的に範囲が限定されたレビューシグナルは、自動執行の対象外に留まります。

削除、禁止、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。禁止または無効化されたアカウントによってサインインや通常の CLI アクセスがブロックされている場合は、復旧レビューに [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナー起因のメールでスキルまたはプラグインのバージョンが悪意ありと示されている場合は、
ブロックされた提出済みバージョンの保存済みスキャン結果をダウンロードしてください:
`clawhub scan download <slug> --version <version>`。プラグインの場合は、
`--kind plugin` を追加します。スキャン出力を確認し、リスティングを修正し、バージョン番号を増やして、修正済みバージョンをアップロードしてください。

## 公開者向けガイダンス

誤検出を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を申告する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- プラグインを公開する前にドライランを使用する
- ユーザーまたはモデレーターがリリースの挙動について質問した場合は明確に返答する
