# モデレーションとアカウントの安全性

ClawHub は公開に開かれていますが、公開ディスカバリーとインストール面には引き続き
ガードレールが必要です。報告、モデレーション保留、非表示の掲載、アカウント措置は、
リリースやアカウントが安全でない、誤解を招く、またはポリシーに反しているように見える場合に
ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウント状態について説明します。`Pass`、`Review`、
`Warn`、`Malicious` などの監査ラベルとリスクレベルについては、
[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)および
[許容される利用](/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ
権利に関する懸念については、[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みのユーザーは、スキル、Plugin、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のある掲載
- 誤解を招くメタデータ
- 宣言されていない認証情報または権限要件
- 疑わしいインストール手順
- なりすまし
- 悪意のある登録または商標の不正使用
- [許容される利用](/clawhub/acceptable-usage)に違反するコンテンツ

スキルページの**スキルを報告**ボタン、またはパッケージ向けの報告
コマンド/API を使用してください。

サードパーティのスキルまたは Plugin 自身のソースコードにある脆弱性には、
ClawHub の報告を使用しないでください。それらは、掲載からリンクされている公開者またはソース
リポジトリに直接報告してください。ClawHub はサードパーティのスキルや Plugin コードを
保守またはパッチ適用しません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性のためのものです。
例としては、Web サイト、API、CLI、レジストリ、認証、スキャン、モデレーション、
またはダウンロード/インストールの信頼境界におけるバグが含まれます。サードパーティのスキルや
Plugin の脆弱性には ClawHub のアドバイザリを使用しないでください。

良い報告は具体的で、対応可能です。報告の悪用は、それ自体がアカウント措置につながる場合があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争は、
製品内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移管、名前変更、非表示、隔離、エイリアス化、またはその他の形でレビューすべきかを
ClawHub スタッフに確認してもらうために、非機密の証拠をレビューする必要がある場合は、
そのプロセスを使用してください。公開 Issue には、シークレット、非公開文書、非公開の法的ファイル、
個人の本人確認書類、API トークン、または DNS チャレンジトークンを含めないでください。

## モデレーション保留

重大な検出結果やポリシー上の問題によって、公開者または掲載がモデレーション保留になる場合があります。
この場合、影響を受けるコンテンツは公開ディスカバリーから非表示になることがあり、また今後の公開は
問題がレビューされるまで非表示の状態で開始されることがあります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間、ユーザーを保護するためのものです。
誤検知が確認された場合には解除されることもあります。

## 非表示またはブロックされた掲載

掲載は、公開インストール面で保留、非表示、隔離、失効、またはその他の理由で利用不可になる場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、モデレーションによって復元されるまで、
そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示の掲載について診断情報を引き続き確認できる場合があります。
これらの診断情報は、何が起きたか、および掲載を公開面に戻す前に何を変更する必要があるかを
説明するのに役立ちます。

## 禁止とアカウント状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な悪用は、
アカウントの禁止、トークンの失効、コンテンツの非表示、または掲載の削除につながる可能性があります。
公開者の悪用圧力シグナルは毎日チェックされます。ClawHub の潜在的な禁止しきい値に達した
シグナルは、自動警告をトリガーする場合があります。警告期限後の次回対象スキャンでも公開者が
潜在的な禁止しきい値に入っている場合、ClawHub はアカウント措置を自動的に適用することがあります。
信頼度の低いレビューシグナルや時間範囲が限定されたレビューシグナルは、自動執行の対象外です。

削除、禁止、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置後に
CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。禁止または
無効化されたアカウントによってサインインや通常の CLI アクセスがブロックされている場合は、
復旧レビューに [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーによって送信されたメールで、スキルまたは Plugin バージョンが悪意のあるものとして示されている場合は、
ブロックされた送信済みバージョンの保存済みスキャン結果をダウンロードしてください。
`clawhub scan download <slug> --version <version>`。Plugin の場合は
`--kind plugin` を追加します。スキャン出力を確認し、掲載を修正し、バージョン番号を増やして、
修正済みバージョンをアップロードしてください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには、次のようにします。

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- Plugin を公開する前にドライランを使用する
- ユーザーまたはモデレーターからリリース動作について質問された場合は明確に回答する
