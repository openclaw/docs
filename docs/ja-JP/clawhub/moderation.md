# モデレーションとアカウント安全性

ClawHub は公開を受け付けていますが、公開ディスカバリーとインストール面には引き続きガードレールが必要です。報告、モデレーション保留、非表示のリスティング、アカウント措置は、リリースやアカウントが安全でない、誤解を招く、またはポリシー外に見える場合にユーザーを保護するために役立ちます。

このページでは、モデレーションとアカウントの状態について説明します。`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)と[利用可能な使用方法](/clawhub/acceptable-usage)も参照してください。著作権またはその他のコンテンツ権利に関する懸念については、[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みのユーザーは、Skills、Plugin、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 未申告の認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の不正使用
- [利用可能な使用方法](/clawhub/acceptable-usage)に違反するコンテンツ

Skills ページの **Skills を報告** ボタン、またはパッケージ用のパッケージ報告コマンド/APIを使用してください。

サードパーティの Skills または Plugin 独自のソースコード内の脆弱性について、ClawHub の報告を使用しないでください。リスティングからリンクされている公開元またはソースリポジトリに直接報告してください。ClawHub はサードパーティの Skills または Plugin コードを保守またはパッチ適用しません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性用です。例には、Web サイト、API、CLI、レジストリ、認証、スキャン、モデレーション、またはダウンロード/インストールの信頼境界におけるバグが含まれます。サードパーティの Skills または Plugin の脆弱性に ClawHub の advisory を使用しないでください。

良い報告は具体的で実行可能です。報告の悪用自体がアカウント措置につながる場合があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争は、プロダクト内の報告フローやアカウント異議申し立てフォームではなく、[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移管、名称変更、非表示、隔離、エイリアス化、またはその他のレビュー対象にすべきであることについて、ClawHub スタッフに非機密の証拠をレビューしてもらう必要がある場合に、このプロセスを使用してください。公開 Issue に、シークレット、非公開文書、非公開の法務ファイル、個人身分証明書、API トークン、または DNS チャレンジトークンを含めないでください。

## モデレーション保留

重大な所見やポリシー上の問題により、公開元またはリスティングがモデレーション保留になる場合があります。この場合、影響を受けるコンテンツは公開ディスカバリーから非表示になるか、以後の公開がレビュー完了まで非表示で開始される場合があります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間、ユーザーを保護するためのものです。誤検知が確認された場合には解除されることもあります。

## 非表示またはブロックされたリスティング

リスティングは、保留、非表示、隔離、取り消し、またはその他の理由で公開インストール面から利用できなくなる場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、モデレーションが復元するまで、そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示のリスティングについて診断情報を引き続き確認できる場合があります。これらの診断情報は、何が起きたか、およびリスティングが公開面に戻る前に何を変更する必要があるかを説明するのに役立ちます。

## 禁止とアカウント状態

ClawHub ポリシーに違反するアカウントは、公開アクセスを失う場合があります。重大な悪用は、アカウント禁止、トークン失効、コンテンツ非表示、またはリスティング削除につながる場合があります。公開元の悪用圧力シグナルは毎日チェックされます。ClawHub の潜在的禁止しきい値に達したシグナルは、自動警告をトリガーする場合があります。警告期限後の次回の対象スキャンでも公開元が潜在的禁止しきい値に該当する場合、ClawHub はアカウント措置を自動的に適用する場合があります。信頼度の低いレビューシグナルや期間が限定されたレビューシグナルは、自動適用の対象外のままです。

削除、禁止、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置の後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。禁止または無効化されたアカウントによりサインインや通常の CLI アクセスがブロックされている場合は、復旧レビューに [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーによってトリガーされたメールが Skills または Plugin のバージョンを悪意のあるものとして示している場合は、ブロックされた送信済みバージョンについて保存済みのスキャン結果をダウンロードしてください:
`clawhub scan download <slug> --version <version>`。Plugin の場合は `--kind plugin` を追加します。スキャン出力を確認し、リスティングを修正し、バージョン番号を増やして、修正済みバージョンをアップロードしてください。

## 公開元向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- Plugin を公開する前にドライランを使用する
- ユーザーまたはモデレーターからリリースの挙動について質問された場合は、明確に回答する
