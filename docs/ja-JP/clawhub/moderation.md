# モデレーションとアカウント安全性

ClawHub は公開に開かれていますが、公開ディスカバリーとインストール面には引き続き
ガードレールが必要です。報告、モデレーション保留、非表示のリスティング、アカウント措置は、
リリースやアカウントが安全でない、誤解を招く、またはポリシー違反に見える場合に
ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウントの状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/ja-JP/clawhub/security)と
[許容される利用](/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ
権利に関する懸念については、[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みユーザーは、Skills、plugins、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 宣言されていない認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意ある登録または商標の不正使用
- [許容される利用](/clawhub/acceptable-usage)に違反するコンテンツ

スキルページの**スキルを報告**ボタン、またはパッケージ用のパッケージ報告
コマンド/API を使用してください。

サードパーティのスキルまたは
plugin 自身のソースコード内の脆弱性に対して ClawHub の報告を使用しないでください。それらは、リスティングからリンクされている発行者またはソース
リポジトリに直接報告してください。ClawHub はサードパーティのスキルまたは plugin コードを保守またはパッチ適用しません。

`openclaw/clawhub` の GitHub Security Advisories は、
ClawHub 自体の脆弱性のためのものです。例には、Web サイト、API、CLI、レジストリ、認証、
スキャン、モデレーション、またはダウンロード/インストールの信頼境界におけるバグが含まれます。サードパーティの Skills または plugins の脆弱性に ClawHub
advisories を使用しないでください。

良い報告は具体的で実行可能です。報告の乱用は、それ自体が
アカウント措置につながる場合があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争は、
製品内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移管、名前変更、非表示、隔離、エイリアス化、またはその他の方法でレビューすべきであるという
機密でない証拠を ClawHub スタッフにレビューしてもらう必要がある場合は、そのプロセスを使用してください。公開 issue に、シークレット、非公開文書、非公開の法的ファイル、個人の本人確認書類、API トークン、または DNS チャレンジトークンを含めないでください。

## モデレーション保留

重大な検出事項やポリシー問題によって、発行者またはリスティングが
モデレーション保留になる場合があります。これが発生すると、影響を受けるコンテンツは公開
ディスカバリーから非表示になるか、今後の公開がレビューされるまで非表示で開始される場合があります。

モデレーション保留は、ClawHub が高リスクの
ケースを解決する間、ユーザーを保護することを目的としています。誤検知が確認された場合には解除されることもあります。

## 非表示またはブロックされたリスティング

リスティングは、公開インストール面で保留、非表示、隔離、取り消し、またはその他の理由で利用不可になる場合があります。

これらの状態のいずれかが表示された場合、所有者が
問題を解決するか、モデレーションが復元するまで、そのリリースをインストールしないでください。

所有者は、自分自身の保留中または非表示のリスティングについて診断を引き続き確認できる場合があります。これらの
診断は、何が起きたか、そしてリスティングが公開面に戻る前に何を変更する必要があるかを説明するのに役立ちます。

## 禁止とアカウントの状態

ClawHub ポリシーに違反したアカウントは公開アクセスを失う場合があります。重大な乱用は、
アカウントの禁止、トークンの取り消し、コンテンツの非表示、またはリスティングの削除につながる場合があります。
発行者の乱用圧力シグナルは毎日確認されます。
ClawHub の潜在的な禁止しきい値に達したシグナルは、自動警告をトリガーする場合があります。警告期限後の次の
対象スキャンでも発行者が潜在的な禁止しきい値に入っている場合、
ClawHub はアカウント措置を自動的に適用する場合があります。
信頼度が低いシグナルや時間範囲が限定されたレビューシグナルは、自動
執行の対象外のままです。

削除、禁止、または無効化されたアカウントは ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が
失敗し始めた場合は、Web UI にサインインしてアカウント
状態を確認してください。禁止または無効化されたアカウントによってサインインまたは通常の CLI アクセスがブロックされている場合は、
復旧レビューのために [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーがトリガーしたメールで、スキルまたは plugin のバージョンが悪意のあるものとして示されている場合は、
ブロックされた送信済みバージョンの保存済みスキャン結果をダウンロードしてください:
`clawhub scan download <slug> --version <version>`。plugins の場合は、
`--kind plugin` を追加します。スキャン出力をレビューし、リスティングを修正し、バージョン
番号を増やして、修正済みバージョンをアップロードしてください。

## 発行者向けガイダンス

誤検知を減らし、ユーザーの信頼を向上させるには:

- 名前、要約、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- plugins を公開する前にドライランを使用する
- ユーザーまたはモデレーターがリリースの動作について質問した場合は明確に回答する
