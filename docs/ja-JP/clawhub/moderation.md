# モデレーションとアカウント安全性

ClawHub は公開投稿を受け付けていますが、公開の発見およびインストール面には引き続きガードレールが必要です。報告、モデレーション保留、非表示リスティング、アカウント措置は、リリースやアカウントが安全でない、誤解を招く、またはポリシーに反しているように見える場合に、ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウントの状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/ja-JP/clawhub/security-audits)を参照してください。

[セキュリティ](/ja-JP/clawhub/security)および
[許容される使用](/ja-JP/clawhub/acceptable-usage)も参照してください。著作権またはその他のコンテンツ権利に関する懸念については、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。

## 報告

サインイン済みのユーザーは、スキル、プラグイン、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 未申告の認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意ある登録または商標の不正使用
- [許容される使用](/ja-JP/clawhub/acceptable-usage)に違反するコンテンツ

スキルページの**スキルを報告**ボタン、またはパッケージ向けの報告コマンド/API を使用してください。

サードパーティのスキルまたはプラグイン自体のソースコードにある脆弱性には、ClawHub の報告を使用しないでください。リスティングからリンクされている公開者またはソースリポジトリに直接報告してください。ClawHub はサードパーティのスキルやプラグインコードを保守または修正しません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性を対象とします。例には、Web サイト、API、CLI、レジストリ、認証、スキャン、モデレーション、またはダウンロード/インストールの信頼境界にあるバグが含まれます。サードパーティのスキルやプラグインの脆弱性には、ClawHub のアドバイザリを使用しないでください。

良い報告は具体的で、対応可能です。報告の悪用自体がアカウント措置につながる場合があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争では、製品内の報告フローやアカウント異議申し立てフォームではなく、[組織と名前空間の申し立て](/ja-JP/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移転、改名、非表示、隔離、エイリアス化、またはその他の形でレビューすべきであることを示す非機密の証拠を ClawHub スタッフにレビューしてもらう必要がある場合は、そのプロセスを使用してください。公開 issue には、シークレット、非公開文書、非公開の法的ファイル、本人確認書類、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

重大な検出事項やポリシー上の問題により、公開者またはリスティングがモデレーション保留の対象になる場合があります。この場合、影響を受けるコンテンツは公開の発見面から非表示になることがあり、今後の公開が問題のレビュー完了まで非表示で開始されることがあります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間にユーザーを保護するためのものです。誤検知が確認された場合には解除されることもあります。

## 非表示またはブロックされたリスティング

リスティングは、公開インストール面で保留、非表示、隔離、取り消し、またはその他の形で利用不可になる場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、モデレーションによって復元されるまで、そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示のリスティングについて診断を引き続き確認できる場合があります。これらの診断は、何が起きたのか、リスティングを公開面に戻す前に何を変更する必要があるのかを説明するのに役立ちます。

## 禁止措置とアカウント状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な悪用は、アカウント禁止、トークンの取り消し、コンテンツの非表示、またはリスティングの削除につながる場合があります。公開者の不正利用圧力シグナルは毎日チェックされます。ClawHub の潜在的な禁止しきい値に達したシグナルは、自動警告をトリガーする場合があります。警告期限後の次回の対象スキャンでも公開者が潜在的な禁止しきい値に該当する場合、ClawHub はアカウント措置を自動的に適用することがあります。信頼度の低いレビューシグナルや時間範囲が限定されたレビューシグナルは、自動執行の対象外です。

削除、禁止、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。禁止措置または無効化されたアカウントによってサインインや通常の CLI アクセスがブロックされている場合は、復旧レビューに [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーによって送信されたメールがスキルまたはプラグインのバージョンを悪意ありとして示している場合は、ブロックされた送信バージョンの保存済みスキャン結果をダウンロードしてください:
`clawhub scan download <slug> --version <version>`。プラグインの場合は
`--kind plugin` を追加します。スキャン出力を確認し、リスティングを修正し、バージョン番号を増やして、修正済みバージョンをアップロードしてください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- プラグインを公開する前にドライランを使用する
- ユーザーまたはモデレーターからリリースの挙動について質問された場合は明確に回答する
