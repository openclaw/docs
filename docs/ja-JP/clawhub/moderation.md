# モデレーションとアカウント安全性

ClawHub は公開を受け付けていますが、公開ディスカバリーとインストール画面には引き続きガードレールが必要です。報告、モデレーション保留、非表示リスティング、アカウント措置は、リリースやアカウントが安全でない、誤解を招く、またはポリシー違反に見える場合にユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウントの状態について説明します。`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)と[利用規定](/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ権利に関する懸念には、[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みユーザーは、Skills、Plugin、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 未申告の認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の不正使用
- [利用規定](/clawhub/acceptable-usage)に違反するコンテンツ

Skillsページの**Skillsを報告**ボタン、またはパッケージ向けの報告コマンド/APIを使用してください。

サードパーティのSkillsやPlugin自身のソースコードにある脆弱性には、ClawHub の報告を使用しないでください。リスティングからリンクされている公開者またはソースリポジトリに直接報告してください。ClawHub はサードパーティのSkillsやPluginコードを保守または修正しません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性を対象としています。例として、Webサイト、API、CLI、レジストリ、認証、スキャン、モデレーション、ダウンロード/インストールの信頼境界にあるバグが含まれます。サードパーティのSkillsやPluginの脆弱性に ClawHub advisories を使用しないでください。

よい報告は具体的で対応可能です。報告の悪用自体がアカウント措置につながることがあります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する異議は、製品内の報告フローやアカウント異議申し立てフォームではなく、[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移譲、名称変更、非表示、隔離、エイリアス化、またはその他の形でレビューすべきかどうかについて、ClawHub スタッフに機密でない証拠のレビューを依頼する必要がある場合は、そのプロセスを使用してください。公開 Issue にシークレット、非公開文書、非公開の法的ファイル、個人の身分証明書、APIトークン、DNSチャレンジトークンを含めないでください。

## モデレーション保留

一部の重大な検出事項やポリシー問題により、公開者またはリスティングがモデレーション保留になることがあります。これが発生すると、影響を受けるコンテンツが公開ディスカバリーから非表示になったり、以後の公開がレビュー完了まで非表示で開始されたりする場合があります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間、ユーザーを保護することを目的としています。誤検知が確認された場合は解除されることもあります。

## 非表示またはブロックされたリスティング

リスティングは、公開インストール画面で保留、非表示、隔離、取り消し、またはその他の理由で利用不可になる場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、モデレーションによって復元されるまで、そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示のリスティングについて、引き続き診断情報を確認できる場合があります。これらの診断情報は、何が起きたのか、リスティングを公開画面に戻す前に何を変更する必要があるのかを説明するのに役立ちます。

## 禁止とアカウント状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な悪用は、アカウントの禁止、トークンの失効、コンテンツの非表示、またはリスティングの削除につながることがあります。公開者の悪用圧力シグナルは毎日確認されます。ClawHub の潜在的な禁止しきい値に達したシグナルは、自動警告をトリガーすることがあります。警告期限後の次回対象スキャンでも公開者が潜在的な禁止しきい値に該当する場合、ClawHub はアカウント措置を自動的に適用することがあります。信頼度の低いレビューシグナルや、期間が限定されたレビューシグナルは、自動執行の対象外です。

削除、禁止、または無効化されたアカウントは、ClawHub APIトークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。禁止または無効化されたアカウントによってサインインや通常の CLI アクセスがブロックされている場合は、復旧レビューに [ClawHub appeal form](https://appeals.openclaw.ai/) を使用してください。

スキャナーによってトリガーされたメールでSkillsまたはPluginのバージョンが悪意ありと示されている場合は、ブロックされた提出済みバージョンの保存済みスキャン結果をダウンロードしてください: `clawhub scan download <slug> --version <version>`。Pluginの場合は `--kind plugin` を追加します。スキャン出力を確認し、リスティングを修正し、バージョン番号を増やして、修正済みバージョンをアップロードしてください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- Pluginを公開する前にドライランを使用する
- ユーザーまたはモデレーターからリリースの挙動について質問された場合は明確に回答する
