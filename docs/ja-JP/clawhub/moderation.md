# モデレーションとアカウントの安全性

ClawHub は公開に対して開かれていますが、公開ディスカバリーとインストール面には
依然としてガードレールが必要です。報告、モデレーション保留、非表示リスト、アカウント措置は、
リリースやアカウントが安全でない、誤解を招く、またはポリシー外であるように見える場合に、
ユーザーの保護に役立ちます。

このページでは、モデレーションとアカウントの状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)と
[許容される利用](/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ権利に関する懸念については、
[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みのユーザーは、スキル、プラグイン、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスト
- 誤解を招くメタデータ
- 未申告の認証情報または権限要件
- 疑わしいインストール手順
- なりすまし
- 悪意ある登録または商標の不正使用
- [許容される利用](/clawhub/acceptable-usage)に違反するコンテンツ

スキルページの**スキルを報告**ボタン、またはパッケージ用の報告コマンド/API を使用してください。

サードパーティのスキルまたはプラグイン自身のソースコード内の脆弱性には、ClawHub の報告を使用しないでください。
それらは、リストからリンクされている発行者またはソースリポジトリに直接報告してください。
ClawHub はサードパーティのスキルまたはプラグインコードを保守または修正しません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性を対象としています。
例として、Web サイト、API、CLI、レジストリ、認証、スキャン、モデレーション、ダウンロード/インストールの信頼境界におけるバグが含まれます。
サードパーティのスキルやプラグインの脆弱性に ClawHub のアドバイザリを使用しないでください。

よい報告は具体的で、対応可能なものです。報告の悪用は、それ自体がアカウント措置につながる可能性があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争は、
製品内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移管、名前変更、非表示、隔離、エイリアス化、またはその他の形でレビューすべきかについて、
ClawHub スタッフに非機密の証拠をレビューしてもらう必要がある場合は、このプロセスを使用してください。
公開 issue には、シークレット、非公開文書、非公開の法的ファイル、本人確認書類、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

重大な検出事項やポリシー上の問題によって、発行者またはリストがモデレーション保留になる場合があります。
この場合、影響を受けるコンテンツは公開ディスカバリーから非表示になることがあり、
今後の公開も問題がレビューされるまで非表示で開始されることがあります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間、ユーザーを保護することを目的としています。
誤検知が確認された場合には解除されることもあります。

## 非表示またはブロックされたリスト

リストは、公開インストール面で保留、非表示、隔離、取り消し、またはその他の理由で利用不可になる場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、モデレーションによって復元されるまで、
そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示のリストについて診断情報を引き続き確認できる場合があります。
これらの診断情報は、何が起きたのか、リストが公開面に戻る前に何を変更する必要があるのかを説明するために役立ちます。

## 禁止措置とアカウント状態

ClawHub ポリシーに違反するアカウントは、公開アクセスを失う場合があります。
重大な悪用は、アカウント禁止、トークン取り消し、コンテンツの非表示、またはリスト削除につながる可能性があります。
発行者の悪用圧力シグナルは毎日確認されます。
ClawHub の潜在的禁止しきい値に達したシグナルは、自動警告をトリガーする場合があります。
警告期限後の次の対象スキャンでも発行者が潜在的禁止しきい値に入っている場合、
ClawHub はアカウント措置を自動的に適用することがあります。
信頼度が低いシグナルや時間的に限定されたレビューシグナルは、自動執行の対象外です。

削除、禁止、または無効化されたアカウントは、ClawHub API トークンを使用できません。
アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。
禁止または無効化されたアカウントによってサインインや通常の CLI アクセスがブロックされている場合は、
復旧レビューに [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーによって送信されたメールが、スキルまたはプラグインのバージョンを悪意ありとして示している場合は、
ブロックされた送信バージョンの保存済みスキャン結果をダウンロードしてください:
`clawhub scan download <slug> --version <version>`。プラグインの場合は
`--kind plugin` を追加します。スキャン出力を確認し、リストを修正し、バージョン番号を増やして、
修正済みバージョンをアップロードしてください。

## 発行者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、要約、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を申告する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- プラグインを公開する前にドライランを使用する
- ユーザーまたはモデレーターからリリースの動作について質問された場合は、明確に回答する
