# モデレーションとアカウントの安全性

ClawHub は公開投稿に開かれていますが、公開ディスカバリーとインストールの画面には引き続き
ガードレールが必要です。報告、モデレーション保留、非表示の掲載、アカウント措置は、
リリースやアカウントが安全でない、誤解を招く、またはポリシー外であるように見える場合に
ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウントの状態について説明します。`Pass`、`Review`、
`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)と
[許容される利用](/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ
権利に関する懸念には、[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みのユーザーは、Skills、Plugin、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のある掲載
- 誤解を招くメタデータ
- 未申告の認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の不正使用
- [許容される利用](/clawhub/acceptable-usage)に違反するコンテンツ

Skills ページの **Skills を報告** ボタン、またはパッケージ用の報告
コマンド/API を使用してください。

サードパーティの Skills または Plugin 自身のソースコードにある脆弱性には、ClawHub の報告を
使用しないでください。掲載からリンクされている公開者またはソースリポジトリに直接報告してください。
ClawHub は、サードパーティの Skills または Plugin コードを保守またはパッチ適用しません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性を対象とします。
例として、Web サイト、API、CLI、レジストリ、認証、スキャン、モデレーション、または
ダウンロード/インストールの信頼境界にあるバグが含まれます。サードパーティの Skills または
Plugin の脆弱性に ClawHub のアドバイザリを使用しないでください。

良い報告は具体的で、対応可能です。報告機能の乱用は、それ自体がアカウント措置につながる
可能性があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、オーナーハンドル、または名前空間の所有権に関する紛争は、
製品内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移管、改名、非表示、隔離、エイリアス化、またはその他の方法でレビューすべきかを
ClawHub スタッフに確認してもらうために、機密性のない証拠をレビューしてもらう必要がある場合は、
そのプロセスを使用してください。公開 Issue には、シークレット、私的文書、非公開の法務ファイル、
本人確認書類、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

一部の重大な検出結果やポリシー問題により、公開者または掲載がモデレーション保留になることがあります。
この場合、影響を受けるコンテンツは公開ディスカバリーから非表示になるか、今後の公開がレビューされるまで
非表示の状態で開始される場合があります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間、ユーザーを保護することを目的としています。
誤検出が確認された場合は解除されることもあります。

## 非表示またはブロックされた掲載

掲載は、公開インストール画面で保留、非表示、隔離、取り消し、またはその他の理由で利用不可になる場合があります。

これらの状態のいずれかが表示された場合、オーナーが問題を解決するか、モデレーションによって復元されるまで、
そのリリースをインストールしないでください。

オーナーは、自分の保留中または非表示の掲載について診断情報を引き続き確認できる場合があります。これらの診断情報は、
何が起きたのか、掲載が公開画面に戻る前に何を変更する必要があるのかを説明するのに役立ちます。

## 禁止措置とアカウントの状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な乱用は、
アカウントの禁止、トークンの取り消し、コンテンツの非表示、または掲載の削除につながる可能性があります。
公開者の乱用圧力シグナルは毎日チェックされます。ClawHub の潜在的禁止しきい値に達したシグナルは、
自動警告をトリガーする場合があります。警告期限後の次の対象スキャンでも公開者が潜在的禁止しきい値に入っている場合、
ClawHub はアカウント措置を自動的に適用することがあります。信頼度の低いシグナルや時間的に限定されたレビューシグナルは、
自動執行の対象外です。

削除、禁止、または無効化されたアカウントは ClawHub API トークンを使用できません。アカウント措置の後に CLI 認証が
失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。禁止または無効化されたアカウントにより
サインインまたは通常の CLI アクセスがブロックされている場合は、復旧レビューに
[ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーによってトリガーされたメールで、Skills または Plugin のバージョンが悪意ありと名指しされている場合は、
ブロックされた提出バージョンの保存済みスキャン結果をダウンロードしてください:
`clawhub scan download <slug> --version <version>`。Plugin の場合は
`--kind plugin` を追加してください。スキャン出力を確認し、掲載を修正し、バージョン番号を増やして、
修正版をアップロードしてください。

## 公開者向けガイダンス

誤検出を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を申告する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- Plugin を公開する前にドライランを使用する
- ユーザーまたはモデレーターからリリースの挙動について質問された場合は明確に回答する
