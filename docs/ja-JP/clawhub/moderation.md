# モデレーションとアカウントの安全性

ClawHub は公開に開かれていますが、公開ディスカバリーとインストールサーフェスには依然として
ガードレールが必要です。報告、モデレーション保留、非表示リスティング、アカウント措置は、
リリースまたはアカウントが安全でない、誤解を招く、またはポリシー外であるように見える場合に
ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウント状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)および
[許容される利用](/clawhub/acceptable-usage)も参照してください。著作権またはその他のコンテンツ権利に関する懸念については、
[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みのユーザーは、Skills、Plugin、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 宣言されていない認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の不正使用
- [許容される利用](/clawhub/acceptable-usage)に違反するコンテンツ

Skills ページの **Skillsを報告** ボタン、またはパッケージ用の報告
コマンド/APIを使用してください。

サードパーティの Skills または
Plugin 自体のソースコードにある脆弱性には、ClawHub の報告を使用しないでください。リスティングからリンクされている公開者またはソース
リポジトリに直接報告してください。ClawHub はサードパーティの Skills または Plugin コードを保守またはパッチ適用しません。

`openclaw/clawhub` の GitHub Security Advisories は、
ClawHub 自体の脆弱性のためのものです。例には、Web サイト、API、CLI、レジストリ、auth、
スキャン、モデレーション、またはダウンロード/インストールの信頼境界におけるバグが含まれます。サードパーティの Skills または Plugin の脆弱性に ClawHub の
アドバイザリを使用しないでください。

良い報告は具体的で、対応可能です。報告機能の悪用自体も
アカウント措置につながる可能性があります。

## 組織と名前空間の申請

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争では、
製品内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申請](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移管、名称変更、非表示、隔離、エイリアス化、
またはその他の方法でレビューすべきであるという非機密の証拠を ClawHub スタッフにレビューしてもらう必要がある場合は、
このプロセスを使用してください。公開 issue には、シークレット、非公開文書、非公開の法的ファイル、
個人の本人確認書類、API トークン、または DNS チャレンジトークンを含めないでください。

## モデレーション保留

一部の重大な検出事項またはポリシー問題により、公開者またはリスティングが
モデレーション保留に置かれることがあります。この場合、影響を受けるコンテンツは公開ディスカバリーから非表示になるか、
今後の公開がレビューされるまで非表示で開始されることがあります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間、
ユーザーを保護するためのものです。誤検知が確認された場合には解除されることもあります。

## 非表示またはブロックされたリスティング

リスティングは、公開インストールサーフェス上で、保留、非表示、隔離、取り消し、
またはその他の形で利用不可になることがあります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、
モデレーションによって復元されるまで、そのリリースをインストールしないでください。

所有者は、自身の保留中または非表示のリスティングについて診断情報を引き続き確認できる場合があります。これらの
診断情報は、何が起きたのか、リスティングを公開サーフェスに戻す前に
何を変更する必要があるのかを説明するのに役立ちます。

## 禁止とアカウント状態

ClawHub ポリシーに違反するアカウントは、公開アクセスを失うことがあります。重大な悪用は、
アカウントの禁止、トークンの取り消し、コンテンツの非表示、またはリスティングの削除につながる可能性があります。
公開者の悪用圧力シグナルは毎日チェックされます。
ClawHub の潜在的な禁止しきい値に達したシグナルは、自動警告をトリガーすることがあります。警告期限後の次回の
対象スキャンでも、その公開者が潜在的な禁止しきい値に入っている場合、
ClawHub はアカウント措置を自動的に適用することがあります。
信頼度の低いシグナルや期間を限定したレビューシグナルは、自動
執行の対象外にとどまります。

削除、禁止、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置の後に CLI auth が
失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。
禁止または無効化されたアカウントによってサインインまたは通常の CLI アクセスがブロックされている場合は、
復旧レビューに [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーがトリガーしたメールで Skills または Plugin のバージョンが悪意のあるものとして示されている場合は、
ブロックされた送信バージョンの保存済みスキャン結果をダウンロードしてください。
`clawhub scan download <slug> --version <version>`。Plugin の場合は、
`--kind plugin` を追加してください。スキャン出力をレビューし、リスティングを修正し、バージョン
番号を増やして、修正済みバージョンをアップロードしてください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには、次のようにしてください。

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- Plugin を公開する前にドライランを使用する
- ユーザーまたはモデレーターからリリースの動作について質問された場合は明確に回答する
