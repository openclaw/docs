# モデレーションとアカウントの安全性

ClawHub は公開に対して開かれていますが、公開ディスカバリーとインストール面には引き続き
ガードレールが必要です。報告、モデレーション保留、非表示のリスティング、アカウント措置は、
リリースやアカウントが安全でない、誤解を招く、またはポリシー外であるように見える場合に
ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウント状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)と
[許容される利用](/clawhub/acceptable-usage)も参照してください。著作権またはその他のコンテンツ権利に関する懸念については、
[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みユーザーは、スキル、Plugin、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 未申告の認証情報または権限要件
- 疑わしいインストール手順
- なりすまし
- 悪意のある登録または商標の不正使用
- [許容される利用](/clawhub/acceptable-usage)に違反するコンテンツ

スキルページの **スキルを報告** ボタン、またはパッケージ用の報告コマンド/APIを使用してください。

サードパーティのスキルまたはPlugin自体のソースコードにある脆弱性については、ClawHub の報告を使用しないでください。
それらは、リスティングからリンクされている公開者またはソースリポジトリへ直接報告してください。
ClawHub はサードパーティのスキルまたはPluginコードを保守したりパッチしたりしません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性のためのものです。
例として、ウェブサイト、API、CLI、レジストリ、認証、スキャン、モデレーション、またはダウンロード/インストールの信頼境界にあるバグが含まれます。
サードパーティのスキルまたはPluginの脆弱性に ClawHub のアドバイザリを使用しないでください。

よい報告は具体的で実行可能です。報告機能の悪用自体がアカウント措置につながる場合があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する争いは、
製品内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移管、改名、非表示、隔離、エイリアス化、またはその他の形でレビューすべきことを示す
非機密の証拠を ClawHub スタッフにレビューしてもらう必要がある場合は、このプロセスを使用してください。
公開 Issue にシークレット、私的文書、私的な法的ファイル、個人識別書類、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

一部の重大な検出事項またはポリシー問題により、公開者またはリスティングがモデレーション保留になる場合があります。
これが発生すると、影響を受けたコンテンツは公開ディスカバリーから非表示になる場合があり、
また今後の公開は問題がレビューされるまで非表示で開始される場合があります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間にユーザーを保護するためのものです。
誤検知が確認された場合は解除されることもあります。

## 非表示またはブロックされたリスティング

リスティングは、公開インストール面で保留、非表示、隔離、失効、またはその他の形で利用不可になる場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、モデレーションが復元するまで、そのリリースをインストールしないでください。

所有者は、自分自身の保留中または非表示のリスティングについては診断情報を引き続き確認できる場合があります。
これらの診断情報は、何が起きたのか、リスティングを公開面に戻す前に何を変更する必要があるのかを説明するのに役立ちます。

## 禁止とアカウント状態

ClawHub ポリシーに違反するアカウントは、公開アクセスを失う場合があります。重大な悪用は、
アカウント禁止、トークン失効、コンテンツ非表示、またはリスティング削除につながる場合があります。
公開者の悪用圧力シグナルは毎日チェックされます。ClawHub の潜在的禁止しきい値に達したシグナルは、
自動警告を発生させる場合があります。警告期限後の次の対象スキャンでも公開者が潜在的禁止しきい値に該当する場合、
ClawHub はアカウント措置を自動的に適用する場合があります。
信頼度が低いシグナルや時間的に限定されたレビューシグナルは、自動執行の対象外のままです。

削除、禁止、または無効化されたアカウントは ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、
Web UI にサインインしてアカウント状態を確認してください。禁止または無効化されたアカウントによりサインインや通常の CLI アクセスがブロックされている場合は、
復旧レビューのために [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーが送信したメールでスキルまたはPluginバージョンが悪意のあるものとして指定されている場合は、
ブロックされた提出バージョンの保存済みスキャン結果をダウンロードしてください:
`clawhub scan download <slug> --version <version>`。Plugin の場合は
`--kind plugin` を追加してください。スキャン出力を確認し、リスティングを修正し、バージョン番号を増やして、修正済みバージョンをアップロードしてください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、要約、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を申告する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- Plugin を公開する前にドライランを使用する
- ユーザーまたはモデレーターがリリースの挙動について尋ねた場合は明確に応答する
