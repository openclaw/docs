# モデレーションとアカウントの安全性

ClawHubでは公開投稿が可能ですが、一般公開の検索・閲覧機能やインストール機能には、引き続き安全対策が必要です。リリースやアカウントが安全でない、誤解を招く、またはポリシーに違反していると思われる場合、報告、モデレーション保留、非表示の掲載情報、アカウント措置によってユーザーを保護します。

このページでは、モデレーションとアカウントの状態について説明します。`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/ja-JP/clawhub/security)と[利用規定](/clawhub/acceptable-usage)も参照してください。著作権その他のコンテンツに関する権利上の懸念については、[コンテンツ権利リクエスト](/clawhub/content-rights)を利用してください。

## 報告

サインイン済みのユーザーは、Skills、plugins、パッケージを報告できます。

ClawHubの報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のある掲載情報
- 誤解を招くメタデータ
- 明示されていない認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の不正使用
- [利用規定](/clawhub/acceptable-usage)に違反するコンテンツ

Skillページの**Skillを報告**ボタン、またはパッケージ用の報告コマンド/APIを使用してください。

サードパーティ製のSkillまたはPlugin自体のソースコードにある脆弱性を、ClawHubの報告で知らせないでください。掲載情報からリンクされている公開者またはソースリポジトリへ直接報告してください。ClawHubは、サードパーティ製のSkillまたはPluginのコードを保守または修正しません。

`openclaw/clawhub`のGitHub Security Advisoriesは、ClawHub自体の脆弱性を対象としています。対象例には、Webサイト、API、CLI、レジストリ、認証、スキャン、モデレーション、ダウンロードやインストールの信頼境界にあるバグが含まれます。サードパーティ製のSkillsまたはpluginsにある脆弱性には、ClawHubのアドバイザリを使用しないでください。

適切な報告は、具体的で対応可能なものです。報告機能の悪用自体が、アカウント措置につながる場合があります。

## 組織と名前空間の権利主張

組織、ブランド、パッケージスコープ、所有者ハンドル、名前空間の所有権に関する紛争には、製品内の報告フローやアカウント異議申し立てフォームではなく、[組織と名前空間の権利主張](/clawhub/namespace-claims)の手続きを使用してください。

名前空間の予約、移管、名前変更、非表示、隔離、エイリアス設定、その他の確認が必要であることを示す機密性のない証拠について、ClawHubスタッフによる審査が必要な場合は、この手続きを使用してください。公開Issueには、シークレット、非公開文書、非公開の法的文書、個人の身元証明書、APIトークン、DNSチャレンジトークンを含めないでください。

## モデレーション保留

一部の重大な検出結果やポリシー上の問題により、公開者または掲載情報がモデレーション保留になる場合があります。この場合、影響を受けるコンテンツが一般公開の検索・閲覧機能から非表示になることや、問題が審査されるまで今後の公開が非表示の状態で開始されることがあります。

モデレーション保留は、ClawHubが高リスクの事案を解決する間、ユーザーを保護することを目的としています。誤検知が確認された場合は、解除されることもあります。

## 非表示またはブロックされた掲載情報

掲載情報は、保留、非表示、隔離、取り消し、またはその他の理由により、一般公開のインストール機能で利用できなくなる場合があります。

これらの状態のいずれかが表示された場合は、所有者が問題を解決するか、モデレーションによって復元されるまで、そのリリースをインストールしないでください。

所有者は、保留または非表示になっている自身の掲載情報の診断情報を引き続き確認できる場合があります。この診断情報は、何が起きたのか、掲載情報を一般公開機能に戻す前に何を変更する必要があるのかを説明するのに役立ちます。

## アカウント停止とアカウントの状態

ClawHubのポリシーに違反したアカウントは、公開権限を失う場合があります。重大な悪用は、アカウント停止、トークンの取り消し、コンテンツの非表示、掲載情報の削除につながる可能性があります。公開者による悪用の可能性を示すシグナルは、毎日確認されます。ClawHubのアカウント停止候補しきい値に達したシグナルは、自動警告の契機となる場合があります。警告期限後の次回の対象スキャンでも、公開者がアカウント停止候補しきい値に該当する場合、ClawHubはアカウント措置を自動的に適用することがあります。信頼度の低いシグナルや、限定された期間のみを対象とする審査シグナルは、自動執行の対象外です。

削除、停止、または無効化されたアカウントでは、ClawHub APIトークンを使用できません。アカウント措置後にCLI認証が失敗し始めた場合は、Web UIにサインインしてアカウントの状態を確認してください。アカウント停止または無効化によりサインインや通常のCLIアクセスがブロックされている場合は、復旧審査のために[ClawHub異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーによって送信されたメールで、特定のSkillまたはPluginのバージョンが悪意のあるものとされている場合は、ブロックされた送信済みバージョンについて保存されているスキャン結果をダウンロードしてください。
`clawhub scan download <slug> --version <version>`。pluginsの場合は、`--kind plugin`を追加してください。スキャン出力を確認し、掲載情報を修正してバージョン番号を増やし、修正版をアップロードしてください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーからの信頼を高めるには、次の点に留意してください。

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を明記する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへのリンクを記載する
- pluginsを公開する前にドライランを使用する
- ユーザーやモデレーターからリリースの動作について質問された場合は、明確に回答する
