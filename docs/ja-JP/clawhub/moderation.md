# モデレーションとアカウントの安全性

ClawHub は公開に開かれていますが、公開ディスカバリーとインストール面には引き続き
ガードレールが必要です。報告、モデレーション保留、非表示の掲載、アカウント措置は、
リリースやアカウントが安全でない、誤解を招く、またはポリシーに反しているように見える場合に
ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウントの状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)と
[許容される利用](/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ権利に関する懸念には、
[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインインしたユーザーは、スキル、プラグイン、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のある掲載
- 誤解を招くメタデータ
- 宣言されていない認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の不正使用
- [許容される利用](/clawhub/acceptable-usage)に違反するコンテンツ

スキルページの**スキルを報告**ボタン、またはパッケージ向けの報告
コマンド/API を使用してください。

サードパーティのスキルまたはプラグイン自体のソースコードにある脆弱性には、
ClawHub の報告を使用しないでください。それらは、掲載からリンクされている公開者またはソース
リポジトリに直接報告してください。ClawHub はサードパーティのスキルまたはプラグインコードを
保守または修正しません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性のためのものです。
例として、Web サイト、API、CLI、レジストリ、認証、スキャン、モデレーション、またはダウンロード/インストールの信頼境界にあるバグが含まれます。サードパーティのスキルやプラグインの脆弱性に ClawHub の
アドバイザリを使用しないでください。

良い報告は具体的で、対応可能です。報告の悪用は、それ自体が
アカウント措置につながる場合があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争は、
製品内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移管、名前変更、非表示、隔離、エイリアス化、またはその他の方法でレビューすべきであることを
ClawHub スタッフに確認してもらうため、機密でない証拠をレビューする必要がある場合は、そのプロセスを使用してください。
公開 issue に、シークレット、非公開文書、非公開の法的ファイル、本人確認書類、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

重大な検出事項やポリシー上の問題によって、公開者または掲載が
モデレーション保留になる場合があります。この場合、影響を受けるコンテンツは公開ディスカバリーから非表示になるか、
今後の公開がレビューされるまで非表示で開始される場合があります。

モデレーション保留は、ClawHub が高リスクなケースを解決する間、ユーザーを保護することを目的としています。
誤検知が確認された場合には解除されることもあります。

## 非表示またはブロックされた掲載

掲載は、公開インストール面で保留、非表示、隔離、取り消し、またはその他の形で利用不可になる場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、モデレーションによって復元されるまで、
そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示の掲載について、引き続き診断情報を確認できる場合があります。
これらの診断情報は、何が起きたのか、掲載が公開面に戻る前に何を変更する必要があるのかを説明するのに役立ちます。

## 禁止措置とアカウント状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な悪用は、
アカウント禁止、トークン取り消し、コンテンツの非表示、または掲載の削除につながる場合があります。
公開者の悪用圧力シグナルは毎日確認されます。
ClawHub の潜在的禁止しきい値に達したシグナルは、自動警告を引き起こす場合があります。警告期限後の次回対象スキャンでも
公開者が潜在的禁止しきい値に該当する場合、ClawHub はアカウント措置を自動的に適用する場合があります。
信頼度の低いシグナルや、時間的範囲が限定されたレビューシグナルは、自動執行の対象外です。

削除、禁止、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が
失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。サインインまたは通常の CLI アクセスが、禁止または無効化されたアカウントによってブロックされている場合は、
回復レビューのために [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーが送信したメールで、スキルまたはプラグインのバージョンが悪意ありと示されている場合は、
ブロックされた送信済みバージョンの保存済みスキャン結果をダウンロードしてください。
`clawhub scan download <slug> --version <version>`。プラグインの場合は、
`--kind plugin` を追加します。スキャン出力を確認し、掲載を修正し、バージョン番号を増やして、
修正済みバージョンをアップロードしてください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには、次のことを行ってください。

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- プラグインを公開する前にドライランを使用する
- ユーザーまたはモデレーターがリリースの動作について質問した場合は、明確に回答する
