# モデレーションとアカウントの安全性

ClawHub は公開を受け付けていますが、公開ディスカバリーとインストール画面には引き続き
ガードレールが必要です。報告、モデレーション保留、非表示の掲載、アカウント措置は、
リリースまたはアカウントが安全でない、誤解を招く、またはポリシーに違反しているように見える場合に、
ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウント状態を扱います。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)と
[許容される使用](/clawhub/acceptable-usage)も参照してください。著作権またはその他のコンテンツ権利に関する懸念については、
[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みのユーザーは、Skills、plugins、packages を報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のある掲載
- 誤解を招くメタデータ
- 申告されていない認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意ある登録または商標の不正使用
- [許容される使用](/clawhub/acceptable-usage)に違反するコンテンツ

Skill ページの **Skill を報告** ボタン、または packages 用の報告
コマンド/API を使用してください。

サードパーティの skill または plugin 自身のソースコードにある脆弱性には、
ClawHub の報告を使用しないでください。それらは、掲載からリンクされている公開者またはソース
リポジトリに直接報告してください。ClawHub はサードパーティの skill または plugin コードを
保守または修正しません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性用です。
例には、ウェブサイト、API、CLI、レジストリ、認証、スキャン、モデレーション、またはダウンロード/インストールの信頼境界にあるバグが含まれます。
サードパーティの Skills または plugins の脆弱性に ClawHub の advisories を使用しないでください。

良い報告は具体的で、対応可能です。報告の悪用は、それ自体がアカウント措置につながる場合があります。

## 組織と名前空間の申請

組織、ブランド、package スコープ、所有者ハンドル、または名前空間の所有権に関する紛争は、
製品内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申請](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移管、改名、非表示、隔離、別名化、またはその他の方法でレビューすべきであることを示す
非機密の証拠を ClawHub スタッフに確認してもらう必要がある場合は、そのプロセスを使用してください。
公開 issue に、シークレット、非公開文書、非公開の法的ファイル、個人識別文書、API トークン、または DNS チャレンジトークンを含めないでください。

## モデレーション保留

一部の重大な検出事項またはポリシー問題により、公開者または掲載が
モデレーション保留になる場合があります。この場合、影響を受けるコンテンツは公開ディスカバリーから非表示になるか、
今後の公開がレビュー完了まで非表示で開始される場合があります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間にユーザーを保護するためのものです。
誤検知が確認された場合は解除されることもあります。

## 非表示またはブロックされた掲載

掲載は、保留、非表示、隔離、取り消し、またはその他の理由で
公開インストール画面で利用できない場合があります。

これらの状態のいずれかが表示された場合は、所有者が問題を解決するか、モデレーションが復元するまで、
そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示の掲載について診断情報を引き続き確認できる場合があります。
これらの診断情報は、何が起きたか、および掲載を公開画面に戻す前に何を変更する必要があるかを説明するのに役立ちます。

## 禁止措置とアカウント状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な悪用は、
アカウントの禁止、トークンの取り消し、コンテンツの非表示、または掲載の削除につながる場合があります。
公開者の悪用圧力シグナルは毎日確認されます。ClawHub の潜在的な禁止しきい値に達したシグナルは、
自動警告をトリガーする場合があります。警告期限後の次の対象スキャンでも、公開者が
潜在的な禁止しきい値に該当する場合、ClawHub はアカウント措置を自動的に適用することがあります。
信頼度の低いシグナルや、時間的に限定されたレビューシグナルは自動執行の対象外です。

削除、禁止、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が
失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。禁止または無効化されたアカウントにより
サインインまたは通常の CLI アクセスがブロックされている場合は、復旧レビューに
[ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーによって送信されたメールが、skill または plugin バージョンを悪意のあるものとして示している場合は、
ブロックされた送信バージョンの保存済みスキャン結果をダウンロードしてください:
`clawhub scan download <slug> --version <version>`。plugins の場合は
`--kind plugin` を追加します。スキャン出力を確認し、掲載を修正し、バージョン番号を増やして、
修正済みバージョンをアップロードしてください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を向上させるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を申告する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- plugins を公開する前にドライランを使用する
- ユーザーまたはモデレーターがリリースの動作について質問した場合は明確に回答する
