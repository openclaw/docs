# モデレーションとアカウント安全性

ClawHub は公開を受け付けていますが、公開ディスカバリーとインストール面には引き続き
ガードレールが必要です。報告、モデレーション保留、非表示の掲載、アカウント措置は、
リリースやアカウントが安全でない、誤解を招く、またはポリシー違反のように見える場合に
ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウント状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)および
[許容される使用](/clawhub/acceptable-usage)も参照してください。著作権またはその他のコンテンツ権利に関する懸念については、
[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みユーザーは、Skills、Plugin、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のある掲載
- 誤解を招くメタデータ
- 申告されていない認証情報または権限要件
- 疑わしいインストール手順
- なりすまし
- 悪意ある登録または商標の不正使用
- [許容される使用](/clawhub/acceptable-usage)に違反するコンテンツ

Skillページの**Skillを報告**ボタン、またはパッケージ用の報告
コマンド/APIを使用してください。

サードパーティのSkillまたは
Plugin自体のソースコードにある脆弱性について、ClawHub の報告を使用しないでください。それらは掲載からリンクされている公開者またはソース
リポジトリに直接報告してください。ClawHub はサードパーティのSkillまたはPluginコードを保守またはパッチ適用しません。

`openclaw/clawhub` の GitHub Security Advisories は、
ClawHub 自体の脆弱性用です。例として、Webサイト、API、CLI、レジストリ、認証、
スキャン、モデレーション、またはダウンロード/インストールの信頼境界にあるバグが含まれます。サードパーティのSkillsまたはPluginの脆弱性に ClawHub
アドバイザリを使用しないでください。

良い報告は具体的で実行可能です。報告の悪用は、それ自体が
アカウント措置につながる可能性があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争では、
製品内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移譲、名前変更、非表示、隔離、エイリアス化、またはその他の形でレビューすべきかについて、
ClawHub スタッフに機密性のない証拠をレビューしてもらう必要がある場合に、このプロセスを使用してください。公開イシューには、シークレット、非公開文書、非公開の法的ファイル、個人識別文書、APIトークン、DNSチャレンジトークンを含めないでください。

## モデレーション保留

重大な検出事項やポリシー問題によって、公開者または掲載が
モデレーション保留になる場合があります。この場合、影響を受けるコンテンツは公開
ディスカバリーから非表示になるか、以後の公開がレビューされるまで非表示で開始されることがあります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間にユーザーを保護するためのものです。誤検知が確認された場合は解除されることもあります。

## 非表示またはブロックされた掲載

掲載は、保留、非表示、隔離、失効、またはその他の理由で
公開インストール面で利用できない場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、モデレーションによって復元されるまで、そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示の掲載について、引き続き診断情報を確認できる場合があります。これらの診断情報は、
何が起きたか、および掲載を公開面に戻す前に何を変更する必要があるかを説明するのに役立ちます。

## 禁止措置とアカウント状態

ClawHub ポリシーに違反するアカウントは、公開アクセスを失う場合があります。重大な不正使用は、
アカウント禁止、トークン失効、コンテンツ非表示、または掲載削除につながる可能性があります。
公開者の不正使用圧力シグナルは毎日確認されます。
ClawHub の潜在的禁止しきい値に達したシグナルは、自動警告をトリガーする可能性があります。警告期限後の次回の対象スキャンでも公開者が
潜在的禁止しきい値に該当する場合、ClawHub はアカウント措置を自動的に適用することがあります。
信頼度の低いレビューシグナルや時間的に限定されたレビューシグナルは、自動執行の対象外のままです。

削除、禁止、または無効化されたアカウントは ClawHub APIトークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。禁止または無効化されたアカウントによってサインインや通常の CLI アクセスがブロックされている場合は、復旧レビューに
[ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーによってトリガーされたメールが、SkillまたはPluginのバージョンを悪意のあるものとして示している場合は、
ブロックされた送信バージョンの保存済みスキャン結果をダウンロードしてください。
`clawhub scan download <slug> --version <version>`。Pluginの場合は、
`--kind plugin` を追加します。スキャン出力を確認し、掲載を修正し、バージョン番号を増やして、修正済みバージョンをアップロードしてください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を申告する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- Pluginを公開する前にドライランを使用する
- ユーザーまたはモデレーターがリリースの動作について質問した場合は明確に回答する
