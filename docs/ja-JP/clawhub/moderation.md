# モデレーションとアカウント安全性

ClawHub は公開に開かれていますが、公開ディスカバリーとインストール面には引き続き
ガードレールが必要です。リリースやアカウントが安全でない、誤解を招く、または
ポリシー外であるように見える場合、報告、モデレーション保留、非表示リスティング、アカウント措置は
ユーザーの保護に役立ちます。

このページでは、モデレーションとアカウントの状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)と
[許容される使用](/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ権利に関する懸念は、
[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みユーザーは、skills、plugins、packages を報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 宣言されていない認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の悪用
- [許容される使用](/clawhub/acceptable-usage)に違反するコンテンツ

skill ページの **skill を報告** ボタン、または packages 向けの package 報告
コマンド/API を使用してください。

サードパーティ skill または
plugin 自体のソースコード内の脆弱性には、ClawHub の報告を使用しないでください。それらは、リスティングからリンクされている公開者またはソース
リポジトリに直接報告してください。ClawHub はサードパーティ skill または plugin のコードを保守またはパッチ適用しません。

`openclaw/clawhub` の GitHub Security Advisories は、
ClawHub 自体の脆弱性向けです。例には、Web サイト、API、CLI、レジストリ、認証、
スキャン、モデレーション、またはダウンロード/インストールの信頼境界におけるバグが含まれます。サードパーティ skills または plugins の脆弱性に ClawHub
advisories を使用しないでください。

良い報告は具体的で実行可能です。報告の悪用自体が
アカウント措置につながる場合があります。

## 組織と名前空間の申し立て

組織、ブランド、package スコープ、所有者ハンドル、または名前空間の所有権に関する紛争は、
製品内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移管、名前変更、非表示、隔離、エイリアス化、
またはその他の形でレビューすべきであることを示す非機密の証拠を ClawHub スタッフにレビューしてもらう必要がある場合は、そのプロセスを使用してください。
公開 issue に、シークレット、非公開文書、非公開の法的ファイル、
個人の身分証明書、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

一部の重大な検出事項またはポリシー問題により、公開者またはリスティングが
モデレーション保留下に置かれる場合があります。これが発生すると、影響を受けるコンテンツが公開
ディスカバリーから非表示になるか、今後の公開がレビュー完了まで非表示で開始される場合があります。

モデレーション保留は、ClawHub が高リスクの
ケースを解決する間、ユーザーを保護することを目的としています。誤検知が確認された場合には解除されることもあります。

## 非表示またはブロックされたリスティング

リスティングは、保留、非表示、隔離、取り消し、またはその他の理由で
公開インストール面で利用できない場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、
モデレーションが復元するまで、そのリリースをインストールしないでください。

所有者は、自分の保留または非表示のリスティングについて、引き続き診断を表示できる場合があります。これらの
診断は、何が起きたのか、およびリスティングが公開面に戻る前に何を変更する必要があるのかを説明するのに役立ちます。

## 禁止措置とアカウントの状態

ClawHub ポリシーに違反するアカウントは、公開アクセスを失う場合があります。重大な悪用は、
アカウント禁止、トークン失効、コンテンツ非表示、またはリスティング削除につながる場合があります。
公開者の悪用圧力シグナルは毎日確認されます。
ClawHub の潜在的禁止しきい値に達したシグナルは、自動警告をトリガーする場合があります。警告期限後の次の
対象スキャンでも公開者が
潜在的禁止しきい値に該当する場合、ClawHub はアカウント措置を自動的に適用する場合があります。
信頼度の低いシグナルや、時間的に限定されたレビューシグナルは、自動
執行の対象外です。

削除、禁止、または無効化されたアカウントは ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が
失敗し始めた場合は、Web UI にサインインしてアカウント
状態を確認してください。サインインまたは通常の CLI アクセスが、禁止または無効化されたアカウントによってブロックされている場合は、
復旧レビューのために [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーがトリガーしたメールで、skill または plugin のバージョンが悪意のあるものとして示されている場合は、
ブロックされた送信バージョンの保存済みスキャン結果をダウンロードしてください。
`clawhub scan download <slug> --version <version>`。plugins の場合は、
`--kind plugin` を追加してください。スキャン出力をレビューし、リスティングを修正し、バージョン
番号を上げ、修正済みバージョンをアップロードしてください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- plugins を公開する前にドライランを使用する
- ユーザーまたはモデレーターがリリースの動作について質問した場合は明確に回答する
