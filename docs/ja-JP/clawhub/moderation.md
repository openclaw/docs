# モデレーションとアカウントの安全性

ClawHub は公開に開かれていますが、公開ディスカバリーとインストール画面にはなおガードレールが必要です。報告、モデレーション保留、非表示の掲載、アカウント措置は、リリースやアカウントが安全でない、誤解を招く、またはポリシー外であるように見える場合にユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウントの状態について説明します。`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)と[許容される利用](/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ権利に関する懸念については、[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みユーザーは、スキル、Plugin、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のある掲載
- 誤解を招くメタデータ
- 申告されていない認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の不正利用
- [許容される利用](/clawhub/acceptable-usage)に違反するコンテンツ

スキルページの**スキルを報告**ボタン、またはパッケージ用のパッケージ報告コマンド/APIを使用してください。

サードパーティのスキルまたはPlugin自体のソースコードにある脆弱性には ClawHub の報告を使用しないでください。それらは掲載からリンクされている公開者またはソースリポジトリに直接報告してください。ClawHub はサードパーティのスキルまたはPluginコードを保守またはパッチ適用しません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性のためのものです。例には、ウェブサイト、API、CLI、レジストリ、認証、スキャン、モデレーション、またはダウンロード/インストールの信頼境界にあるバグが含まれます。サードパーティのスキルまたはPluginの脆弱性に ClawHub のアドバイザリを使用しないでください。

良い報告は具体的で、対応可能です。報告機能の悪用は、それ自体がアカウント措置につながる場合があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争には、製品内の報告フローやアカウント異議申し立てフォームではなく、[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移転、名前変更、非表示、隔離、エイリアス化、またはその他の形でレビューすべきかについて、ClawHub スタッフに非機密の証拠をレビューしてもらう必要がある場合は、そのプロセスを使用してください。公開 Issue には、シークレット、非公開文書、非公開の法務ファイル、個人識別文書、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

重大な検出事項やポリシー上の問題により、公開者または掲載がモデレーション保留になる場合があります。この場合、影響を受けるコンテンツは公開ディスカバリーから非表示になるか、今後の公開がレビュー完了まで非表示で開始される場合があります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間にユーザーを保護することを目的としています。誤検知が確認された場合には解除されることもあります。

## 非表示またはブロックされた掲載

掲載は、公開インストール画面で保留、非表示、隔離、取り消し、またはその他の理由で利用不可になる場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、モデレーションにより復元されるまで、そのリリースをインストールしないでください。

所有者は、自分が所有する保留中または非表示の掲載について、引き続き診断を確認できる場合があります。これらの診断は、何が起きたか、掲載を公開画面に戻す前に何を変更する必要があるかを説明するのに役立ちます。

## 禁止措置とアカウント状態

ClawHub ポリシーに違反するアカウントは、公開アクセスを失う場合があります。重大な不正利用は、アカウント禁止、トークン失効、コンテンツ非表示、または掲載削除につながる場合があります。公開者の不正利用圧力シグナルは毎日チェックされます。ClawHub の潜在的な禁止措置しきい値に達したシグナルは、自動警告をトリガーする場合があります。警告期限後の次の対象スキャンでも公開者が潜在的な禁止措置しきい値にある場合、ClawHub はアカウント措置を自動的に適用する場合があります。確信度の低いシグナルや時間範囲が限定されたレビューシグナルは、自動執行の対象外です。

削除、禁止、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。禁止または無効化されたアカウントによりサインインや通常の CLI アクセスがブロックされている場合は、復旧レビューに[ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーがトリガーしたメールでスキルまたはPluginのバージョンが悪意ありとして示されている場合は、ブロックされた送信済みバージョンの保存済みスキャン結果をダウンロードしてください: `clawhub scan download <slug> --version <version>`。Pluginの場合は、`--kind plugin` を追加してください。スキャン出力を確認し、掲載を修正し、バージョン番号を増やして、修正済みバージョンをアップロードしてください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を申告する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- Pluginを公開する前にドライランを使用する
- ユーザーまたはモデレーターがリリースの挙動について質問した場合は明確に回答する
