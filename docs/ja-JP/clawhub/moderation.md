# モデレーションとアカウントの安全性

ClawHub は公開に開かれていますが、公開ディスカバリとインストール面には引き続き
ガードレールが必要です。レポート、モデレーション保留、非表示リスティング、アカウント措置は、
リリースやアカウントが安全でない、誤解を招く、またはポリシー違反に見える場合に
ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウント状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/ja-JP/clawhub/security-audits)を参照してください。

[セキュリティ](/ja-JP/clawhub/security)および
[利用可能な用途](/ja-JP/clawhub/acceptable-usage)も参照してください。著作権またはその他のコンテンツ権利に関する懸念については、
[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。

## レポート

サインイン済みユーザーは、Skills、Plugin、パッケージを報告できます。

ClawHub レポートは、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 未申告の認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の不正使用
- [利用可能な用途](/ja-JP/clawhub/acceptable-usage)に違反するコンテンツ

Skill ページの **Skill を報告** ボタン、またはパッケージ用のパッケージ報告
コマンド/API を使用してください。

サードパーティの Skill または
Plugin 自身のソースコード内の脆弱性に ClawHub レポートを使用しないでください。それらは、リスティングからリンクされている公開者またはソース
リポジトリに直接報告してください。ClawHub はサードパーティの Skill または Plugin コードを保守またはパッチ適用しません。

`openclaw/clawhub` の GitHub Security Advisories は、
ClawHub 自体の脆弱性を対象としています。例として、Web サイト、API、CLI、レジストリ、認証、
スキャン、モデレーション、またはダウンロード/インストールの信頼境界にあるバグが含まれます。サードパーティの Skills または Plugin の脆弱性に ClawHub
アドバイザリを使用しないでください。

良いレポートは具体的で実行可能です。報告の乱用は、それ自体が
アカウント措置につながる可能性があります。

## 組織と名前空間の請求

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争は、
製品内レポートフローやアカウント異議申し立てフォームではなく、
[組織と名前空間の請求](/ja-JP/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移管、名称変更、非表示、隔離、エイリアス化、
またはその他のレビュー対象にすべきであることについて、ClawHub スタッフに非機密の証拠をレビューしてもらう必要がある場合に、そのプロセスを使用してください。公開 Issue には、シークレット、非公開文書、非公開の法的ファイル、
個人識別文書、API トークン、または DNS チャレンジトークンを含めないでください。

## モデレーション保留

重大な検出事項やポリシー問題によって、公開者またはリスティングが
モデレーション保留になる場合があります。この場合、影響を受けるコンテンツが公開
ディスカバリから非表示になる、または今後の公開がレビューされるまで非表示で開始されることがあります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間に
ユーザーを保護するためのものです。誤検知が確認された場合は解除されることもあります。

## 非表示またはブロックされたリスティング

リスティングは、公開インストール面で保留、非表示、隔離、取り消し、またはその他の理由で利用不可になる場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか
モデレーションが復元するまで、そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示のリスティングについて診断情報を引き続き確認できる場合があります。これらの
診断情報は、何が起きたか、およびリスティングを公開面に戻す前に
何を変更する必要があるかを説明するのに役立ちます。

## BAN とアカウント状態

ClawHub ポリシーに違反するアカウントは、公開アクセスを失う場合があります。重大な不正利用は、
アカウント BAN、トークン失効、コンテンツの非表示、またはリスティング削除につながる可能性があります。
公開者の不正利用圧力シグナルは毎日チェックされます。
ClawHub の潜在的 BAN しきい値に達したシグナルは、自動警告をトリガーする可能性があります。警告期限後の次の
対象スキャンでも公開者が
潜在的 BAN しきい値に該当する場合、ClawHub はアカウント措置を自動的に適用することがあります。
信頼度の低いシグナルや時間的に限定されたレビューシグナルは、自動
強制措置の対象外です。

削除、BAN、または無効化されたアカウントは ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が
失敗し始めた場合は、Web UI にサインインしてアカウント
状態を確認してください。BAN または無効化されたアカウントによってサインインまたは通常の CLI アクセスがブロックされている場合は、
復旧レビューに [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーがトリガーしたメールで、Skill または Plugin バージョンが悪意ありと示されている場合は、
ブロックされた送信バージョンの保存済みスキャン結果をダウンロードしてください。
`clawhub scan download <slug> --version <version>`。Plugin の場合は、
`--kind plugin` を追加します。スキャン出力を確認し、リスティングを修正し、バージョン
番号を増やして、修正版をアップロードしてください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには、次のようにしてください。

- 名前、要約、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- Plugin を公開する前にドライランを使用する
- ユーザーまたはモデレーターからリリース動作について質問された場合は明確に回答する
