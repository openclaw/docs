# モデレーションとアカウントの安全性

ClawHub は公開に開かれていますが、公開ディスカバリーとインストール面には
引き続きガードレールが必要です。報告、モデレーション保留、非表示リスティング、アカウント措置は、
リリースやアカウントが安全でない、誤解を招く、またはポリシー外であるように見える場合に
ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウントの状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)と
[利用規定](/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ権利に関する懸念がある場合は、
[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みのユーザーは、スキル、プラグイン、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 宣言されていない認証情報や権限要件
- 不審なインストール手順
- なりすまし
- 悪意ある登録や商標の不正使用
- [利用規定](/clawhub/acceptable-usage)に違反するコンテンツ

スキルページの **スキルを報告** ボタン、またはパッケージ用のパッケージ報告
コマンド/API を使用してください。

サードパーティのスキルやプラグイン自体のソースコードにある脆弱性には、ClawHub の報告を使用しないでください。
それらは、リスティングからリンクされている公開者またはソースリポジトリへ直接報告してください。
ClawHub はサードパーティのスキルやプラグインのコードを保守またはパッチ適用しません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性用です。
例として、Web サイト、API、CLI、レジストリ、認証、スキャン、モデレーション、
ダウンロード/インストールの信頼境界にあるバグが含まれます。サードパーティのスキルやプラグインの脆弱性に
ClawHub のアドバイザリを使用しないでください。

良い報告は具体的で、対応可能です。報告の悪用は、それ自体がアカウント措置につながる可能性があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争は、
製品内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移転、名前変更、非表示、隔離、エイリアス化、またはその他の形でレビューすべきかについて、
ClawHub スタッフに非機密の証拠をレビューしてもらう必要がある場合は、そのプロセスを使用してください。
公開 Issue には、シークレット、非公開文書、非公開の法的ファイル、個人の身分証明書、
API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

一部の重大な検出事項やポリシー問題により、公開者やリスティングがモデレーション保留になることがあります。
この場合、影響を受けるコンテンツは公開ディスカバリーから非表示になるか、今後の公開がレビュー完了まで
非表示で開始されることがあります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間、ユーザーを保護するためのものです。
誤検知が確認された場合は解除されることもあります。

## 非表示またはブロックされたリスティング

リスティングは、公開インストール面で保留、非表示、隔離、取り消し、またはその他の理由で利用不可になることがあります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、モデレーションによって復元されるまで、
そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示のリスティングについて診断情報を引き続き確認できる場合があります。
これらの診断情報は、何が起きたのか、リスティングが公開面に戻る前に何を変更する必要があるのかを説明するのに役立ちます。

## 禁止措置とアカウント状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う可能性があります。重大な悪用は、
アカウント禁止、トークン取り消し、コンテンツの非表示、またはリスティングの削除につながる可能性があります。
公開者の悪用圧力シグナルは毎日確認されます。ClawHub の潜在的禁止しきい値に達したシグナルは、
自動警告をトリガーすることがあります。警告期限後の次の対象スキャンでも公開者が
潜在的禁止しきい値に該当する場合、ClawHub はアカウント措置を自動的に適用することがあります。
信頼度の低いシグナルや時間的に限定されたレビューシグナルは、自動執行の対象外です。

削除、禁止、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が
失敗し始めた場合は、web UI にサインインしてアカウント状態を確認してください。禁止または無効化されたアカウントによって
サインインや通常の CLI アクセスがブロックされている場合は、復旧レビューのために
[ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーがトリガーしたメールで、スキルまたはプラグインのバージョンが悪意ありとされた場合は、
ブロックされた提出バージョンについて保存済みのスキャン結果をダウンロードしてください:
`clawhub scan download <slug> --version <version>`。プラグインの場合は
`--kind plugin` を追加します。スキャン出力をレビューし、リスティングを修正し、バージョン番号を増やして、
修正済みバージョンをアップロードしてください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- プラグインを公開する前にドライランを使用する
- ユーザーやモデレーターからリリースの動作について質問された場合は明確に回答する
