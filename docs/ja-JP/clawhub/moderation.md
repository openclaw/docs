# モデレーションとアカウントの安全性

ClawHub は公開に開かれていますが、公開ディスカバリーとインストール面にはなお
ガードレールが必要です。報告、モデレーション保留、非表示の掲載、アカウント措置は、
リリースやアカウントが安全でない、誤解を招く、またはポリシー違反に見える場合に、
ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウントの状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)と
[許容される利用](/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ権利に関する懸念は、
[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みユーザーは、Skills、plugins、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のある掲載
- 誤解を招くメタデータ
- 宣言されていない認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の不正使用
- [許容される利用](/clawhub/acceptable-usage)に違反するコンテンツ

Skills ページの**Skills を報告**ボタン、またはパッケージ用のパッケージ報告
コマンド/APIを使用してください。

サードパーティの Skills または
plugin 自身のソースコード内の脆弱性には、ClawHub の報告を使用しないでください。それらは掲載からリンクされている公開者またはソース
リポジトリに直接報告してください。ClawHub はサードパーティの Skills または plugin コードを保守またはパッチ適用しません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性を対象としています。例として、Web サイト、API、CLI、レジストリ、認証、
スキャン、モデレーション、またはダウンロード/インストールの信頼境界にあるバグが含まれます。サードパーティの Skills または plugins の脆弱性には ClawHub
advisories を使用しないでください。

良い報告は具体的で、対応可能です。報告の悪用自体が
アカウント措置につながる場合があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、オーナーハンドル、または名前空間の所有権紛争には、製品内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移管、改名、非表示、隔離、エイリアス化、またはその他の形でレビューすべきであることを示す非機密の証拠を ClawHub スタッフにレビューしてもらう必要がある場合に、そのプロセスを使用してください。公開 Issue に、シークレット、非公開文書、非公開の法的ファイル、個人の本人確認書類、API トークン、または DNS チャレンジトークンを含めないでください。

## モデレーション保留

重大な検出事項やポリシー問題によって、公開者または掲載が
モデレーション保留になる場合があります。この場合、影響を受けるコンテンツは公開ディスカバリーから非表示になるか、今後の公開がレビューされるまで非表示で開始される場合があります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間、ユーザーを保護することを目的としています。誤検知が確認された場合には解除されることもあります。

## 非表示またはブロックされた掲載

掲載は、公開インストール面で保留、非表示、隔離、取り消し、またはその他の理由で利用不能になる場合があります。

これらの状態のいずれかが表示された場合、オーナーが問題を解決するか、モデレーションによって復元されるまで、そのリリースをインストールしないでください。

オーナーは、自分の保留中または非表示の掲載について診断情報を引き続き確認できる場合があります。これらの診断情報は、何が起きたのか、掲載が公開面に戻る前に何を変更する必要があるのかを説明するのに役立ちます。

## BAN とアカウントの状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な悪用は、
アカウントの BAN、トークンの取り消し、コンテンツの非表示、または掲載の削除につながることがあります。
公開者の悪用圧力シグナルは毎日チェックされます。ClawHub の潜在的 BAN しきい値に達したシグナルは、自動警告をトリガーする場合があります。警告期限後の次回の対象スキャンでも公開者が潜在的 BAN しきい値に該当する場合、ClawHub はアカウント措置を自動的に適用することがあります。
信頼度の低いシグナルや、時間的に限定されたレビューシグナルは、自動執行の対象外です。

削除、BAN、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。BAN または無効化されたアカウントによってサインインや通常の CLI アクセスがブロックされている場合は、復旧レビューのために
[ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーがトリガーしたメールで Skills または plugin のバージョンが悪意のあるものとして示されている場合は、
ブロックされた提出バージョンの保存済みスキャン結果をダウンロードしてください:
`clawhub scan download <slug> --version <version>`。plugins の場合は
`--kind plugin` を追加します。スキャン出力を確認し、掲載を修正し、バージョン番号を増やして、修正済みバージョンをアップロードしてください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- plugins を公開する前にドライランを使用する
- ユーザーまたはモデレーターからリリースの挙動について質問された場合は、明確に応答する
