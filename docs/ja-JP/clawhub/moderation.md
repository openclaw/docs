# モデレーションとアカウント安全性

ClawHub は公開を受け付けていますが、公開ディスカバリーとインストール画面には引き続き
ガードレールが必要です。報告、モデレーション保留、非表示リスティング、アカウント措置は、
リリースやアカウントが安全でない、誤解を招く、またはポリシー外であるように見える場合に
ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウントの状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/ja-JP/clawhub/security-audits)を参照してください。

[セキュリティ](/ja-JP/clawhub/security)と
[利用規約](/ja-JP/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ権利に関する懸念については、
[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。

## 報告

サインイン済みのユーザーは、スキル、プラグイン、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 未申告の認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の悪用
- [利用規約](/ja-JP/clawhub/acceptable-usage)に違反するコンテンツ

スキルページの **スキルを報告** ボタン、またはパッケージ用の報告
コマンド/API を使用してください。

サードパーティのスキルまたはプラグイン自身のソースコードにある脆弱性には、ClawHub の報告を使用しないでください。
それらは、リスティングからリンクされている公開者またはソース
リポジトリに直接報告してください。ClawHub はサードパーティのスキルまたはプラグインのコードを保守したりパッチを適用したりしません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性を対象としています。
例には、Web サイト、API、CLI、レジストリ、認証、
スキャン、モデレーション、またはダウンロード/インストールの信頼境界にあるバグが含まれます。サードパーティのスキルやプラグインの脆弱性には ClawHub
アドバイザリを使用しないでください。

良い報告は具体的で、対応可能です。報告の悪用は、それ自体が
アカウント措置につながる可能性があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争は、
製品内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申し立て](/ja-JP/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移管、改名、非表示、隔離、エイリアス化、
またはその他の形でレビューすべきであることを、機密ではない証拠に基づいて ClawHub スタッフにレビューしてもらう必要がある場合は、そのプロセスを使用してください。公開 issue に、シークレット、非公開文書、非公開の法務ファイル、個人識別文書、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

重大な検出事項やポリシー上の問題によって、公開者またはリスティングが
モデレーション保留になることがあります。この場合、影響を受けるコンテンツは公開ディスカバリーから非表示になるか、
今後の公開がレビュー完了まで非表示で開始される場合があります。

モデレーション保留は、ClawHub が高リスクなケースを解決するまでユーザーを保護することを目的としています。
誤検知が確認された場合は、解除されることもあります。

## 非表示またはブロックされたリスティング

リスティングは、保留、非表示、隔離、失効、またはその他の理由で
公開インストール画面で利用できない場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、モデレーションが復元するまで
そのリリースをインストールしないでください。

所有者は、自分自身の保留中または非表示のリスティングについて、引き続き診断情報を確認できる場合があります。これらの
診断情報は、何が起きたのか、リスティングが公開画面に戻る前に何を変更する必要があるのかを説明するのに役立ちます。

## 禁止措置とアカウント状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う可能性があります。重大な悪用は、
アカウント禁止、トークン失効、コンテンツ非表示、またはリスティング削除につながる可能性があります。
公開者の悪用圧力シグナルは毎日チェックされます。
ClawHub の潜在的な禁止しきい値に達したシグナルは、自動警告を発生させることがあります。警告期限後の次回の
対象スキャンでも公開者が潜在的な禁止しきい値に該当する場合、ClawHub はアカウント措置を自動的に適用する場合があります。
信頼度の低いシグナルや時間的に限定されたレビューシグナルは、自動
執行の対象外です。

削除、禁止、または無効化されたアカウントは ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が
失敗し始めた場合は、Web UI にサインインしてアカウント
状態を確認してください。禁止または無効化されたアカウントによってサインインや通常の CLI アクセスがブロックされている場合は、
復旧レビューに [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーによって送信されたメールがスキルまたはプラグインのバージョンを悪意のあるものとして示している場合は、
ブロックされた送信バージョンの保存済みスキャン結果をダウンロードしてください:
`clawhub scan download <slug> --version <version>`。プラグインの場合は、
`--kind plugin` を追加してください。スキャン出力を確認し、リスティングを修正し、バージョン
番号を増やして、修正済みバージョンをアップロードしてください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を申告する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- プラグインを公開する前にドライランを使用する
- ユーザーまたはモデレーターからリリースの挙動について質問された場合は明確に回答する
