# モデレーションとアカウント安全性

ClawHub は公開を受け付けていますが、公開検索とインストールの画面には引き続き
ガードレールが必要です。報告、モデレーション保留、非表示の掲載、アカウント措置は、
リリースやアカウントが安全でない、誤解を招く、またはポリシー外に見える場合に
ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウント状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/ja-JP/clawhub/security-audits)を参照してください。

[セキュリティ](/ja-JP/clawhub/security)および
[許容される利用](/ja-JP/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ
権利に関する懸念については、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。

## 報告

サインイン済みユーザーは、Skills、Plugin、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のある掲載
- 誤解を招くメタデータ
- 未申告の認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の不正使用
- [許容される利用](/ja-JP/clawhub/acceptable-usage)に違反するコンテンツ

Skills ページの **Skills を報告** ボタン、またはパッケージ用の報告コマンド/APIを使用してください。

サードパーティの Skills または Plugin 自体のソースコード内の脆弱性には、ClawHub の報告を使用しないでください。
掲載からリンクされている発行者またはソースリポジトリに直接報告してください。ClawHub はサードパーティの
Skills または Plugin コードを保守またはパッチ適用しません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性のためのものです。
例には、Web サイト、API、CLI、レジストリ、認証、スキャン、モデレーション、またはダウンロード/インストールの
信頼境界におけるバグが含まれます。サードパーティの Skills または Plugin の脆弱性には ClawHub の
アドバイザリを使用しないでください。

良い報告は具体的で実行可能です。報告の悪用は、それ自体がアカウント措置につながる場合があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争は、
製品内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申し立て](/ja-JP/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移転、改名、非表示、隔離、エイリアス化、またはその他の方法でレビューすべきであることを示す
非機密の証拠を ClawHub スタッフにレビューしてもらう必要がある場合は、そのプロセスを使用してください。
公開 Issue には、シークレット、非公開文書、非公開の法的ファイル、本人確認書類、API トークン、
DNS チャレンジトークンを含めないでください。

## モデレーション保留

重大な検出事項やポリシー上の問題により、発行者または掲載がモデレーション保留になることがあります。
この場合、影響を受けるコンテンツは公開検索から非表示になるか、今後の公開がレビューされるまで
非表示の状態で開始されることがあります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間、ユーザーを保護するためのものです。
誤検知が確認された場合には解除することもできます。

## 非表示またはブロックされた掲載

掲載は、公開インストール画面で保留、非表示、隔離、失効、またはその他の理由で利用不可になる場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、モデレーションが復元するまで
そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示の掲載に関する診断を引き続き確認できる場合があります。
これらの診断は、何が起きたのか、掲載が公開画面に戻る前に何を変更する必要があるのかを説明するのに役立ちます。

## 禁止とアカウント状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な悪用は、
アカウント禁止、トークン失効、コンテンツの非表示、または掲載の削除につながることがあります。
発行者の悪用圧力シグナルは毎日確認されます。ClawHub の潜在的禁止しきい値に達したシグナルは、
自動警告を引き起こすことがあります。警告期限後の次の対象スキャンでも発行者が潜在的禁止しきい値に
該当する場合、ClawHub はアカウント措置を自動的に適用することがあります。
信頼度が低いレビューシグナルや時間範囲が限定されたレビューシグナルは、自動執行の対象外です。

削除、禁止、または無効化されたアカウントは ClawHub API トークンを使用できません。アカウント措置後に
CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。
禁止または無効化されたアカウントによってサインインや通常の CLI アクセスがブロックされている場合は、
復旧レビューのために [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーによって送信されたメールで Skills または Plugin バージョンが悪意ありとして名指しされている場合は、
ブロックされた送信済みバージョンの保存済みスキャン結果をダウンロードしてください:
`clawhub scan download <slug> --version <version>`。Plugin の場合は
`--kind plugin` を追加します。スキャン出力を確認し、掲載を修正し、バージョン番号を増やして、
修正済みバージョンをアップロードしてください。

## 発行者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を申告する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- Plugin を公開する前にドライランを使用する
- ユーザーまたはモデレーターがリリース動作について質問した場合は明確に回答する
