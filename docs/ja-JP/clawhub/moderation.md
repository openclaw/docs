# モデレーションとアカウントの安全性

ClawHub は公開投稿を受け付けていますが、公開ディスカバリーとインストール面には引き続き
ガードレールが必要です。報告、モデレーション保留、非表示リスティング、アカウント措置は、
リリースやアカウントが安全でない、誤解を招く、またはポリシー違反のように見える場合に、
ユーザーを保護するのに役立ちます。

このページではモデレーションとアカウントの状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/ja-JP/clawhub/security-audits)を参照してください。

[セキュリティ](/ja-JP/clawhub/security)と
[許容される使用](/ja-JP/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ権利に関する懸念には、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。

## 報告

サインイン済みのユーザーは、スキル、プラグイン、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 宣言されていない認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の悪用
- [許容される使用](/ja-JP/clawhub/acceptable-usage)に違反するコンテンツ

スキルページの **スキルを報告** ボタン、またはパッケージ用のパッケージ報告
コマンド/API を使用してください。

サードパーティのスキルまたはプラグイン自身のソースコードにある脆弱性については、ClawHub の報告を使用しないでください。それらは、リスティングからリンクされている公開元またはソース
リポジトリへ直接報告してください。ClawHub はサードパーティのスキルやプラグインのコードを保守またはパッチ適用しません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性のためのものです。例として、ウェブサイト、API、CLI、レジストリ、認証、
スキャン、モデレーション、またはダウンロード/インストールの信頼境界にあるバグが含まれます。サードパーティのスキルやプラグインの脆弱性には ClawHub
advisories を使用しないでください。

良い報告は具体的で実行可能です。報告機能の乱用は、それ自体が
アカウント措置につながる場合があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、オーナーハンドル、または名前空間の所有権紛争には、
製品内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申し立て](/ja-JP/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移転、改名、非表示、隔離、エイリアス化、またはその他の形でレビューすべきかを
ClawHub スタッフに確認してもらう必要がある場合は、そのプロセスを使用してください。公開 issue に、シークレット、非公開文書、非公開の法的ファイル、
個人身分証明書、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

一部の重大な検出結果やポリシー問題により、公開元またはリスティングが
モデレーション保留の対象になる場合があります。この場合、影響を受けるコンテンツは公開
ディスカバリーから非表示になるか、今後の公開がレビューされるまで非表示で開始されることがあります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間にユーザーを保護するためのものです。誤検知が確認された場合には解除されることもあります。

## 非表示またはブロックされたリスティング

リスティングは、公開インストール面で保留、非表示、隔離、取り消し、またはその他の形で利用不可になる場合があります。

これらの状態のいずれかが表示された場合、オーナーが問題を解決するかモデレーションが復元するまで、そのリリースをインストールしないでください。

オーナーは、自分の保留中または非表示のリスティングについて診断情報を引き続き確認できる場合があります。これらの診断情報は、何が起きたのか、またリスティングが公開面に戻る前に何を変更する必要があるのかを説明するのに役立ちます。

## 禁止措置とアカウントの状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な乱用は、
アカウントの禁止、トークンの取り消し、コンテンツの非表示、またはリスティングの削除につながることがあります。
公開元の乱用圧力シグナルは毎日確認されます。
ClawHub の潜在的な禁止しきい値に達したシグナルは、自動警告をトリガーする場合があります。警告期限後の次回の対象スキャンでも、公開元が
潜在的な禁止しきい値内にある場合、ClawHub はアカウント措置を自動的に適用することがあります。
信頼度が低いシグナルや時間的に限定されたレビューシグナルは、自動執行の対象外です。

削除、禁止、または無効化されたアカウントは ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。禁止または無効化されたアカウントによってサインインや通常の CLI アクセスがブロックされている場合は、
復旧レビューに [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーによってトリガーされたメールがスキルまたはプラグインのバージョンを悪意ありとして示している場合は、
ブロックされた送信済みバージョンの保存済みスキャン結果をダウンロードしてください。
`clawhub scan download <slug> --version <version>`。プラグインの場合は、
`--kind plugin` を追加してください。スキャン出力を確認し、リスティングを修正し、バージョン
番号を増やして、修正済みバージョンをアップロードしてください。

## 公開元向けガイダンス

誤検知を減らし、ユーザーからの信頼を高めるには、次のようにします。

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- プラグインを公開する前にドライランを使用する
- ユーザーやモデレーターがリリースの挙動について質問した場合は明確に回答する
