# モデレーションとアカウントの安全性

ClawHub は公開に開かれていますが、公開ディスカバリーとインストール面には引き続き
ガードレールが必要です。報告、モデレーション保留、非表示リスティング、アカウント措置は、
リリースまたはアカウントが安全でない、誤解を招く、またはポリシー違反の可能性がある場合に
ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウント状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)と
[許容される利用](/clawhub/acceptable-usage)も参照してください。著作権またはその他のコンテンツ権利に関する懸念は、
[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みユーザーは、スキル、Plugin、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 未申告の認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の不正使用
- [許容される利用](/clawhub/acceptable-usage)に違反するコンテンツ

スキルページの **スキルを報告** ボタン、またはパッケージ向けのパッケージ報告
コマンド/API を使用してください。

サードパーティのスキルまたはPlugin自体のソースコードにある脆弱性には、ClawHub の報告を使用しないでください。
それらは、リスティングからリンクされている公開元またはソース
リポジトリへ直接報告してください。ClawHub はサードパーティのスキルまたはPluginコードを保守またはパッチ適用しません。

`openclaw/clawhub` の GitHub Security Advisories は、
ClawHub 自体の脆弱性向けです。例として、ウェブサイト、API、CLI、レジストリ、認証、
スキャン、モデレーション、またはダウンロード/インストールの信頼境界にあるバグが含まれます。サードパーティのスキルまたはPluginの脆弱性には、ClawHub
アドバイザリを使用しないでください。

良い報告は具体的で、対応可能です。報告の悪用は、それ自体が
アカウント措置につながる場合があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争では、
製品内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移管、改名、非表示、隔離、エイリアス化、またはその他の形で審査すべきであることを、
ClawHub スタッフに非機密の証拠で審査してもらう必要がある場合に、そのプロセスを使用してください。
公開 issue には、シークレット、非公開文書、非公開の法的ファイル、個人の身分証明書、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

一部の重大な検出事項またはポリシー問題により、公開元またはリスティングが
モデレーション保留になる場合があります。この場合、影響を受けるコンテンツは公開
ディスカバリーから非表示になるか、以後の公開が問題の審査まで非表示で開始される場合があります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間にユーザーを保護することを目的としています。
誤検知が確認された場合は解除されることもあります。

## 非表示またはブロックされたリスティング

リスティングは、公開インストール面で保留、非表示、隔離、失効、またはその他の理由で利用できない場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するかモデレーションが復元するまで、
そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示のリスティングについて、引き続き診断情報を確認できる場合があります。
これらの診断情報は、何が起きたか、およびリスティングを公開面に戻す前に何を変更する必要があるかを説明するのに役立ちます。

## 禁止措置とアカウント状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な悪用は、
アカウント禁止、トークン失効、コンテンツ非表示、またはリスティング削除につながる場合があります。
公開元の悪用圧力シグナルは毎日確認されます。
ClawHub の潜在的な禁止しきい値に達したシグナルは、自動警告をトリガーする場合があります。警告期限後の次回の対象スキャンでも、その公開元が
潜在的な禁止しきい値に該当する場合、ClawHub はアカウント措置を自動的に適用することがあります。
信頼度の低いレビューシグナルや期限付きの一時的レビューシグナルは、自動執行の対象外です。

削除済み、禁止済み、または無効化済みのアカウントは、ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が
失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。
禁止または無効化済みアカウントによりサインインまたは通常の CLI アクセスがブロックされている場合は、
復旧審査に [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナー起点のメールで、スキルまたはPluginバージョンが悪意ありと示されている場合は、
ブロックされた提出バージョンの保存済みスキャン結果をダウンロードしてください:
`clawhub scan download <slug> --version <version>`。Pluginの場合は、
`--kind plugin` を追加します。スキャン出力を確認し、リスティングを修正し、バージョン番号を増やして、
修正済みバージョンをアップロードしてください。

## 公開元向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を申告する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- Pluginを公開する前にドライランを使用する
- ユーザーまたはモデレーターがリリース動作について尋ねた場合は明確に回答する
