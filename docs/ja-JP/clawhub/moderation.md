# モデレーションとアカウントの安全性

ClawHub は公開に開かれていますが、公開ディスカバリーとインストール面には引き続きガードレールが必要です。報告、モデレーション保留、非表示の掲載、アカウント措置は、リリースやアカウントが安全でない、誤解を招く、またはポリシー外であるように見える場合に、ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウントの状態について説明します。`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)と[許容される使用](/clawhub/acceptable-usage)も参照してください。著作権またはその他のコンテンツ権利に関する懸念については、[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みのユーザーは、skills、plugins、packages を報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のある掲載
- 誤解を招くメタデータ
- 未申告の認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の悪用
- [許容される使用](/clawhub/acceptable-usage)に違反するコンテンツ

skill ページの **skill を報告** ボタン、または packages 用の package 報告コマンド/API を使用してください。

サードパーティの skill または plugin 自体のソースコードにある脆弱性について、ClawHub の報告を使用しないでください。それらは、掲載からリンクされている公開者またはソースリポジトリに直接報告してください。ClawHub はサードパーティの skill または plugin コードを保守したりパッチを適用したりしません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性用です。例として、Web サイト、API、CLI、レジストリ、認証、スキャン、モデレーション、またはダウンロード/インストールの信頼境界にあるバグが含まれます。サードパーティの skills または plugins の脆弱性に ClawHub advisories を使用しないでください。

良い報告は具体的で実行可能です。報告の悪用自体がアカウント措置につながる場合があります。

## 組織と名前空間の申し立て

組織、ブランド、package スコープ、所有者ハンドル、または名前空間の所有権に関する紛争は、製品内の報告フローやアカウント異議申し立てフォームではなく、[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移転、名前変更、非表示、隔離、エイリアス化、またはその他の形でレビューすべきであることを示す非機密の証拠を ClawHub スタッフにレビューしてもらう必要がある場合は、そのプロセスを使用してください。公開 issue に、シークレット、非公開文書、非公開の法的ファイル、個人識別文書、API トークン、または DNS チャレンジトークンを含めないでください。

## モデレーション保留

重大な検出事項やポリシー問題によって、公開者または掲載がモデレーション保留の対象になる場合があります。これが発生すると、影響を受けるコンテンツが公開ディスカバリーから非表示になったり、今後の公開が問題のレビューまで非表示で開始されたりする場合があります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間にユーザーを保護することを目的としています。誤検出が確認された場合は解除されることもあります。

## 非表示またはブロックされた掲載

掲載は、公開インストール面で保留、非表示、隔離、取り消し、またはその他の形で利用できない場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、モデレーションが復元するまで、そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示の掲載について診断情報を引き続き確認できる場合があります。これらの診断情報は、何が起きたか、掲載が公開面に戻る前に何を変更する必要があるかを説明するのに役立ちます。

## 禁止とアカウントの状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な悪用は、アカウント禁止、トークン取り消し、コンテンツ非表示、または掲載削除につながる場合があります。公開者の悪用圧力シグナルは毎日確認されます。ClawHub の潜在的禁止しきい値に達したシグナルは、自動警告をトリガーする場合があります。警告期限後の次の対象スキャンでも公開者が潜在的禁止しきい値に該当する場合、ClawHub はアカウント措置を自動適用することがあります。信頼度の低いシグナルと時間的に限定されたレビューシグナルは、自動執行の対象外のままです。

削除、禁止、または無効化されたアカウントは ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。禁止または無効化されたアカウントによってサインインや通常の CLI アクセスがブロックされている場合は、復旧レビューに [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーによってトリガーされたメールが skill または plugin のバージョンを悪意ありとして指定している場合は、ブロックされた提出バージョンの保存済みスキャン結果をダウンロードしてください: `clawhub scan download <slug> --version <version>`。plugins の場合は、`--kind plugin` を追加します。スキャン出力をレビューし、掲載を修正し、バージョン番号を増やして、修正済みバージョンをアップロードしてください。

## 公開者向けガイダンス

誤検出を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- plugins を公開する前にドライランを使用する
- ユーザーまたはモデレーターからリリース動作について質問された場合は、明確に応答する
