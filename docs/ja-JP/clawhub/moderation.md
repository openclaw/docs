# モデレーションとアカウントの安全性

ClawHub では公開投稿が可能ですが、一般公開される検索・閲覧機能やインストール機能には引き続きガードレールが必要です。報告、モデレーション保留、非表示リスティング、アカウント措置は、リリースやアカウントが危険、誤解を招く、またはポリシー違反と思われる場合にユーザーを保護するためのものです。

このページでは、モデレーションとアカウントの状態について説明します。`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)および[許容される使用](/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ権利に関する懸念については、[コンテンツ権利に関するリクエスト](/clawhub/content-rights)を利用してください。

## 報告

サインイン済みのユーザーは、Skills、plugins、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 申告されていない認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意ある登録または商標の不正使用
- [許容される使用](/clawhub/acceptable-usage)に違反するコンテンツ

Skills のページでは **Report skill** ボタンを使用し、パッケージの場合はパッケージ報告用のコマンド/API を使用してください。

サードパーティ製 Skills または plugin 自体のソースコードに存在する脆弱性の報告には、ClawHub の報告機能を使用しないでください。リスティングからリンクされている発行者またはソースリポジトリに直接報告してください。ClawHub は、サードパーティ製 Skills または plugin のコードを保守したり、パッチを適用したりしません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性を対象としています。たとえば、Web サイト、API、CLI、レジストリ、認証、スキャン、モデレーション、ダウンロード/インストールの信頼境界にあるバグが該当します。サードパーティ製 Skills または plugins の脆弱性には、ClawHub のアドバイザリを使用しないでください。

適切な報告は、具体的で対応可能な内容になっています。報告機能の乱用自体がアカウント措置につながる場合があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争には、製品内の報告フローやアカウント異議申し立てフォームではなく、[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移管、名前変更、非表示、隔離、エイリアス化、またはその他の方法で審査すべきことを示す機密性のない証拠を ClawHub スタッフに審査してもらう必要がある場合は、このプロセスを使用してください。公開 Issue には、シークレット、非公開文書、非公開の法的ファイル、個人の身分証明書、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

一部の重大な検出結果やポリシー上の問題により、発行者またはリスティングがモデレーション保留になる場合があります。この場合、影響を受けるコンテンツは一般公開の検索・閲覧機能から非表示になることがあります。また、問題が審査されるまで、今後公開される内容が最初から非表示になる場合もあります。

モデレーション保留は、ClawHub が高リスクの事案を解決するまでユーザーを保護するためのものです。誤検出が確認された場合は、解除することもできます。

## 非表示またはブロックされたリスティング

リスティングは、保留、非表示、隔離、取り消し、またはその他の理由により、一般公開のインストール機能で利用できなくなる場合があります。

これらの状態のいずれかが表示された場合は、所有者が問題を解決するか、モデレーションによって復元されるまで、そのリリースをインストールしないでください。

所有者は、保留中または非表示になっている自身のリスティングについて、引き続き診断情報を確認できる場合があります。この診断情報は、何が起きたのか、およびリスティングを一般公開機能に戻す前に何を変更する必要があるのかを説明するためのものです。

## アカウント停止とアカウントの状態

ClawHub ポリシーに違反したアカウントは、公開権限を失う場合があります。重大な不正使用は、アカウント停止、トークンの取り消し、コンテンツの非表示、またはリスティングの削除につながる可能性があります。発行者による不正使用の兆候は毎日確認されます。ClawHub の潜在的なアカウント停止のしきい値に達した兆候は、自動警告を発生させる場合があります。警告期限後に実施される次回の対象スキャンでも、発行者が潜在的なアカウント停止のしきい値に該当する場合、ClawHub はアカウント措置を自動的に適用することがあります。信頼度が低い審査シグナルや期間が限定された審査シグナルは、自動執行の対象外です。

削除、停止、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗するようになった場合は、Web UI にサインインしてアカウントの状態を確認してください。アカウント停止または無効化によりサインインや通常の CLI アクセスがブロックされている場合は、復旧審査に [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーによって送信されたメールで、Skills または plugin のバージョンが悪意のあるものとして示されている場合は、ブロックされた提出バージョンについて保存済みのスキャン結果をダウンロードしてください。
`clawhub scan download <slug> --version <version>`。plugins の場合は、
`--kind plugin` を追加します。スキャン出力を確認し、リスティングを修正してバージョン番号を増やし、修正版をアップロードしてください。

## 発行者向けガイダンス

誤検出を減らし、ユーザーからの信頼を高めるには、次の事項に従ってください。

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を明記する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへのリンクを追加する
- plugins を公開する前にドライランを使用する
- リリースの動作についてユーザーやモデレーターから質問された場合は、明確に回答する
