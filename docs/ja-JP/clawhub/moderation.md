# モデレーションとアカウント安全性

ClawHub は公開に開かれていますが、公開ディスカバリーとインストールサーフェスには引き続きガードレールが必要です。報告、モデレーション保留、非表示リスティング、アカウント措置は、リリースやアカウントが安全でない、誤解を招く、またはポリシー外であると思われる場合にユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウントの状態について説明します。`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)と[許容される利用](/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ権利に関する懸念については、[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みのユーザーは、Skills、plugins、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 未申告の認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意ある登録または商標の悪用
- [許容される利用](/clawhub/acceptable-usage)に違反するコンテンツ

Skill ページの **Skill を報告** ボタン、またはパッケージ用のパッケージ報告コマンド/APIを使用してください。

サードパーティの Skill または plugin 自身のソースコードにある脆弱性については、ClawHub の報告を使用しないでください。リスティングからリンクされている発行者またはソースリポジトリに直接報告してください。ClawHub はサードパーティの Skill または plugin コードを保守またはパッチ適用しません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性を対象としています。例には、Web サイト、API、CLI、レジストリ、認証、スキャン、モデレーション、またはダウンロード/インストールの信頼境界におけるバグが含まれます。サードパーティの Skills や plugins の脆弱性に ClawHub アドバイザリを使用しないでください。

良い報告は具体的で実行可能です。報告の悪用は、それ自体がアカウント措置につながる場合があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権紛争には、製品内の報告フローやアカウント異議申し立てフォームではなく、[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移転、改名、非表示、隔離、エイリアス化、またはその他の形でレビューすべきことを示す非機密の証拠を ClawHub スタッフに確認してもらう必要がある場合は、そのプロセスを使用してください。公開 issue にシークレット、非公開文書、非公開の法的ファイル、個人の身分証明書、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

一部の重大な検出事項やポリシー問題により、発行者またはリスティングがモデレーション保留の対象になる場合があります。この場合、影響を受けるコンテンツは公開ディスカバリーから非表示になるか、以後の公開がレビュー完了まで非表示で開始される場合があります。

モデレーション保留は、ClawHub が高リスクのケースを解決している間にユーザーを保護するためのものです。誤検知が確認された場合には解除されることもあります。

## 非表示またはブロックされたリスティング

リスティングは、公開インストールサーフェスで保留、非表示、隔離、取り消し、またはその他の理由で利用不可になる場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するかモデレーションによって復元されるまで、そのリリースをインストールしないでください。

所有者は、自身の保留中または非表示のリスティングについて診断情報を引き続き確認できる場合があります。これらの診断情報は、何が起きたのか、リスティングが公開サーフェスに戻る前に何を変更する必要があるのかを説明するのに役立ちます。

## 禁止措置とアカウントの状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な悪用は、アカウントの禁止、トークンの取り消し、コンテンツの非表示、またはリスティングの削除につながる可能性があります。発行者の悪用圧力シグナルは毎日チェックされます。ClawHub の潜在的な禁止しきい値に達したシグナルは、自動警告をトリガーする場合があります。警告期限後の次回の対象スキャンでも発行者が潜在的な禁止しきい値に該当する場合、ClawHub はアカウント措置を自動的に適用することがあります。信頼度の低いシグナルや時間的に限定されたレビューシグナルは、自動執行の対象外です。

削除、禁止、または無効化されたアカウントは ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。禁止または無効化されたアカウントによりサインインや通常の CLI アクセスがブロックされている場合は、復旧レビューに [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーによって送信されたメールで Skill または plugin のバージョンが悪意のあるものとして示されている場合は、ブロックされた提出済みバージョンの保存済みスキャン結果をダウンロードしてください: `clawhub scan download <slug> --version <version>`。plugins の場合は `--kind plugin` を追加してください。スキャン出力を確認し、リスティングを修正し、バージョン番号を増やして、修正済みバージョンをアップロードしてください。

## 発行者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、要約、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- plugins を公開する前にドライランを使用する
- ユーザーやモデレーターからリリースの動作について質問された場合は明確に回答する
