# モデレーションとアカウントの安全性

ClawHub は公開に開かれていますが、公開ディスカバリーとインストールの画面には引き続きガードレールが必要です。報告、モデレーション保留、非表示のリスティング、アカウント措置は、リリースやアカウントが安全でない、誤解を招く、またはポリシーに違反しているように見える場合にユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウントの状態について説明します。`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)と[許容される利用](/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ権利に関する懸念については、[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みのユーザーは、スキル、Plugin、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 宣言されていない認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の不正使用
- [許容される利用](/clawhub/acceptable-usage)に違反するコンテンツ

スキルページの**スキルを報告**ボタン、またはパッケージ用のパッケージ報告コマンド/API を使用してください。

サードパーティのスキルまたはPlugin自身のソースコードにある脆弱性には、ClawHub の報告を使用しないでください。それらは、リスティングからリンクされているパブリッシャーまたはソースリポジトリに直接報告してください。ClawHub はサードパーティのスキルまたはPluginコードを保守またはパッチ適用しません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性のためのものです。例として、ウェブサイト、API、CLI、レジストリ、認証、スキャン、モデレーション、ダウンロード/インストールの信頼境界にあるバグが含まれます。サードパーティのスキルまたはPluginの脆弱性に ClawHub のアドバイザリを使用しないでください。

良い報告は具体的で対応可能です。報告機能の悪用自体がアカウント措置につながることがあります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、オーナーハンドル、または名前空間の所有権に関する紛争は、製品内の報告フローやアカウント異議申し立てフォームではなく、[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移管、名称変更、非表示、隔離、エイリアス化、またはその他の方法でレビューすべきであるという非機密の証拠を ClawHub スタッフにレビューしてもらう必要がある場合は、このプロセスを使用してください。公開 issue にシークレット、非公開文書、非公開の法的ファイル、本人確認書類、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

一部の重大な検出事項やポリシー問題により、パブリッシャーまたはリスティングがモデレーション保留になることがあります。これが発生すると、影響を受けるコンテンツは公開ディスカバリーから非表示になる場合があり、または今後の公開がレビュー完了まで非表示で開始される場合があります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間、ユーザーを保護することを目的としています。誤検知が確認された場合には解除されることもあります。

## 非表示またはブロックされたリスティング

リスティングは、保留、非表示、隔離、取り消し、またはその他の形で公開インストール画面で利用できない状態になることがあります。

これらの状態のいずれかが表示された場合、オーナーが問題を解決するか、モデレーションによって復元されるまで、そのリリースをインストールしないでください。

オーナーは、自分の保留または非表示のリスティングについて診断情報を引き続き確認できる場合があります。これらの診断情報は、何が起きたのか、リスティングが公開画面に戻る前に何を変更する必要があるのかを説明するのに役立ちます。

## 禁止とアカウントの状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失うことがあります。重大な悪用は、アカウント禁止、トークン失効、コンテンツの非表示、またはリスティングの削除につながることがあります。パブリッシャーの悪用圧力シグナルは毎日チェックされます。ClawHub の潜在的禁止しきい値に達したシグナルは、自動警告をトリガーすることがあります。警告期限後の次の対象スキャンでもパブリッシャーが潜在的禁止しきい値に該当する場合、ClawHub はアカウント措置を自動的に適用することがあります。信頼度の低いシグナルや期間が限定されたレビューシグナルは、自動執行の対象外です。

削除、禁止、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。禁止または無効化されたアカウントによってサインインまたは通常の CLI アクセスがブロックされている場合は、復旧レビューのために[ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーがトリガーしたメールで、スキルまたはPluginバージョンが悪意のあるものとして示されている場合は、ブロックされた提出バージョンの保存済みスキャン結果をダウンロードしてください: `clawhub scan download <slug> --version <version>`。Plugin の場合は、`--kind plugin` を追加してください。スキャン出力を確認し、リスティングを修正し、バージョン番号を上げて、修正版をアップロードしてください。

## パブリッシャー向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- Plugin を公開する前にドライランを使用する
- ユーザーまたはモデレーターからリリースの挙動について質問された場合は明確に回答する
