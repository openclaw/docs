# モデレーションとアカウントの安全性

ClawHub は公開投稿に開かれていますが、公開の発見面とインストール面には引き続き
ガードレールが必要です。レポート、モデレーション保留、非表示の掲載、アカウント措置は、
リリースやアカウントが安全でない、誤解を招く、またはポリシー外であるように見える場合に、
ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウントの状態を扱います。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/ja-JP/clawhub/security-audits)を参照してください。

[セキュリティ](/ja-JP/clawhub/security)と
[許容される利用](/ja-JP/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ権利に関する懸念については、
[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。

## レポート

サインイン済みユーザーは、Skills、plugins、パッケージをレポートできます。

ClawHub のレポートは、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のある掲載
- 誤解を招くメタデータ
- 申告されていない認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の不正使用
- [許容される利用](/ja-JP/clawhub/acceptable-usage)に違反するコンテンツ

Skill ページの **Skill をレポート** ボタン、またはパッケージ用のパッケージレポート
コマンド/API を使用してください。

サードパーティの Skill や
plugin 自体のソースコード内の脆弱性に ClawHub レポートを使用しないでください。それらは、掲載からリンクされている公開者またはソース
リポジトリに直接報告してください。ClawHub はサードパーティの Skill や plugin コードを保守またはパッチしません。

`openclaw/clawhub` の GitHub Security Advisories は、
ClawHub 自体の脆弱性用です。例として、Web サイト、API、CLI、レジストリ、認証、
スキャン、モデレーション、ダウンロード/インストールの信頼境界にあるバグが含まれます。サードパーティの Skills や plugins の脆弱性に ClawHub
advisories を使用しないでください。

良いレポートは具体的で実行可能です。レポートの悪用自体が
アカウント措置につながる可能性があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争は、
プロダクト内のレポートフローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申し立て](/ja-JP/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移転、改名、非表示、隔離、エイリアス化、
またはその他の方法でレビューすべきであることを示す非機密の証拠を ClawHub スタッフにレビューしてもらう必要がある場合は、そのプロセスを使用してください。公開 issue に、秘密情報、非公開文書、非公開の法務ファイル、
個人の本人確認書類、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

一部の重大な検出事項やポリシー問題により、公開者または掲載が
モデレーション保留に置かれる場合があります。これが発生すると、影響を受けるコンテンツは公開の
発見面から非表示になるか、今後の公開投稿がレビューされるまで非表示で開始される場合があります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間にユーザーを保護することを目的としています。
誤検知が確認された場合には解除されることもあります。

## 非表示またはブロックされた掲載

掲載は、保留、非表示、隔離、失効、またはその他の理由で
公開インストール面で利用できなくなる場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、モデレーションが復元するまで、
そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示の掲載について、診断を引き続き確認できる場合があります。これらの
診断は、何が起きたのか、掲載が公開面に戻る前に何を変更する必要があるのかを説明するのに役立ちます。

## BAN とアカウントの状態

ClawHub ポリシーに違反したアカウントは、公開投稿アクセスを失う場合があります。重大な悪用は、
アカウントの BAN、トークン失効、コンテンツの非表示、または掲載の削除につながる可能性があります。
公開者の悪用圧力シグナルは毎日チェックされます。
ClawHub の潜在的 BAN しきい値に達したシグナルは、自動警告をトリガーする可能性があります。警告期限後の次の
対象スキャンでも公開者が潜在的 BAN しきい値に入っている場合、ClawHub はアカウント措置を自動的に適用する場合があります。
信頼度が低いシグナルや時間範囲が限定されたレビューシグナルは、自動
執行の対象外です。

削除、BAN、または無効化されたアカウントは ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が
失敗し始めた場合は、Web UI にサインインしてアカウント
状態を確認してください。BAN または無効化されたアカウントによってサインインや通常の CLI アクセスがブロックされている場合は、
復旧レビューに [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーがトリガーしたメールで、Skill または plugin のバージョンが悪意のあるものとして示されている場合は、
ブロックされた提出バージョンの保存済みスキャン結果をダウンロードしてください:
`clawhub scan download <slug> --version <version>`。plugins の場合は、
`--kind plugin` を追加します。スキャン出力をレビューし、掲載を修正し、バージョン
番号を増やして、修正済みバージョンをアップロードしてください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を向上させるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を申告する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- plugins を公開する前にドライランを使用する
- ユーザーやモデレーターがリリースの挙動について質問した場合は明確に回答する
