# モデレーションとアカウント安全性

ClawHub は公開を受け付けていますが、公開ディスカバリーとインストール面には引き続き
ガードレールが必要です。レポート、モデレーション保留、非表示の掲載、アカウント措置は、
リリースやアカウントが安全でない、誤解を招く、またはポリシー外であるように見える場合に
ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウントの状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)と
[許容される利用](/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ権利に関する懸念については、
[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## レポート

サインイン済みのユーザーは、Skills、Plugin、パッケージを報告できます。

ClawHub のレポートは、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のある掲載
- 誤解を招くメタデータ
- 未申告の認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の不正使用
- [許容される利用](/clawhub/acceptable-usage)に違反するコンテンツ

Skills ページの **Skills を報告** ボタン、またはパッケージ用の報告
コマンド/API を使用してください。

サードパーティの Skills または
Plugin 自体のソースコードにある脆弱性には、ClawHub のレポートを使用しないでください。掲載からリンクされている公開者またはソース
リポジトリに直接報告してください。ClawHub はサードパーティの Skills または Plugin コードを保守または修正しません。

`openclaw/clawhub` の GitHub Security Advisories は、
ClawHub 自体の脆弱性のためのものです。例としては、ウェブサイト、API、CLI、レジストリ、認証、
スキャン、モデレーション、またはダウンロード/インストールの信頼境界にあるバグが含まれます。サードパーティの Skills や Plugin の脆弱性には、ClawHub
アドバイザリを使用しないでください。

良いレポートは具体的で実行可能です。報告機能の悪用は、それ自体が
アカウント措置につながる場合があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争は、
製品内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移転、名前変更、非表示、隔離、エイリアス設定、またはその他の方法でレビューすべきであることを示す非機密の証拠を
ClawHub スタッフに確認してもらう必要がある場合は、そのプロセスを使用してください。
公開 issue には、シークレット、非公開文書、非公開の法的ファイル、個人識別文書、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

一部の重大な検出結果やポリシー上の問題により、公開者または掲載が
モデレーション保留になる場合があります。これが発生すると、影響を受けるコンテンツが公開
ディスカバリーから非表示になる、または今後の公開がレビューされるまで非表示で開始される場合があります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間にユーザーを保護することを目的としています。
誤検知が確認された場合は解除されることもあります。

## 非表示またはブロックされた掲載

掲載は、公開インストール面で保留、非表示、隔離、取り消し、またはその他の理由で利用不可になる場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、モデレーションによって復元されるまで、
そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示の掲載に関する診断を引き続き確認できる場合があります。これらの
診断は、何が起きたのか、および掲載を公開面に戻す前に何を変更する必要があるのかを説明するのに役立ちます。

## 禁止措置とアカウントの状態

ClawHub ポリシーに違反するアカウントは、公開アクセスを失う場合があります。重大な悪用は、
アカウント禁止、トークン取り消し、コンテンツ非表示、または掲載削除につながる場合があります。
公開者の悪用圧力シグナルは毎日チェックされます。
ClawHub の潜在的な禁止しきい値に達したシグナルは、自動警告をトリガーする場合があります。警告期限後の次の対象スキャンでも
公開者が潜在的な禁止しきい値に入っている場合、ClawHub はアカウント措置を自動的に適用する場合があります。
信頼度の低いレビューシグナルや期間限定のレビューシグナルは、自動執行の対象外です。

削除、禁止、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置の後に CLI 認証が
失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。禁止または無効化されたアカウントによってサインインや通常の CLI アクセスがブロックされている場合は、
復旧レビューに [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーによって送信されたメールが、Skills または Plugin のバージョンを悪意のあるものとして示している場合は、
ブロックされた送信バージョンについて保存されたスキャン結果をダウンロードしてください。
`clawhub scan download <slug> --version <version>`。Plugin の場合は、
`--kind plugin` を追加します。スキャン出力を確認し、掲載を修正し、バージョン
番号を上げて、修正済みバージョンをアップロードしてください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには、次のようにします。

- 名前、要約、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を申告する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- Plugin を公開する前にドライランを使用する
- ユーザーやモデレーターがリリースの動作について尋ねた場合は明確に回答する
