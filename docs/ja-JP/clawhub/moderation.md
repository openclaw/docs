# モデレーションとアカウントの安全性

ClawHub は公開に開かれていますが、公開ディスカバリーとインストール面には引き続き
ガードレールが必要です。報告、モデレーション保留、非表示リスティング、アカウント措置は、
リリースやアカウントが安全でない、誤解を招く、またはポリシー外であるように見える場合に、
ユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウントの状態について説明します。
`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、
[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)および
[利用可能な範囲](/clawhub/acceptable-usage)も参照してください。著作権またはその他のコンテンツ権利に関する懸念については、
[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みユーザーは、Skills、plugins、packages を報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 宣言されていない認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意ある登録または商標の不正使用
- [利用可能な範囲](/clawhub/acceptable-usage)に違反するコンテンツ

Skill ページの **Skill を報告** ボタン、または packages 向けの package 報告
コマンド/APIを使用してください。

サードパーティの Skill または plugin 自身のソースコードの脆弱性には、ClawHub の報告を使用しないでください。
それらはリスティングからリンクされている公開者またはソースリポジトリに直接報告してください。
ClawHub はサードパーティの Skill または plugin コードを保守したりパッチしたりしません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性を対象としています。
例として、ウェブサイト、API、CLI、レジストリ、認証、スキャン、モデレーション、またはダウンロード/インストールの信頼境界におけるバグが含まれます。
サードパーティの Skills または plugins の脆弱性に ClawHub advisories を使用しないでください。

良い報告は具体的で、対応可能なものです。報告の乱用は、それ自体が
アカウント措置につながる場合があります。

## 組織と名前空間の申し立て

組織、ブランド、package スコープ、所有者ハンドル、または名前空間の所有権をめぐる紛争には、
プロダクト内の報告フローやアカウント異議申し立てフォームではなく、
[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移管、改名、非表示、隔離、エイリアス化、またはその他の形でレビューすべきであることを示す
機密でない証拠を ClawHub スタッフにレビューしてもらう必要がある場合は、そのプロセスを使用してください。
公開 issue には、シークレット、非公開文書、非公開の法的ファイル、個人識別文書、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

一部の重大な検出結果またはポリシー上の問題により、公開者またはリスティングが
モデレーション保留になる場合があります。この場合、影響を受けるコンテンツは公開ディスカバリーから非表示になるか、
今後の公開がレビューされるまで非表示として開始される場合があります。

モデレーション保留は、ClawHub が高リスクなケースを解決するまでユーザーを保護するためのものです。
誤検知が確認された場合には解除されることもあります。

## 非表示またはブロックされたリスティング

リスティングは、公開インストール面で保留、非表示、隔離、取り消し、またはその他の理由で利用不可になる場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、モデレーションによって復元されるまで、
そのリリースをインストールしないでください。

所有者は、自分が所有する保留中または非表示のリスティングについて診断情報を引き続き確認できる場合があります。
これらの診断情報は、何が起きたのか、リスティングを公開面に戻す前に何を変更する必要があるのかを説明するのに役立ちます。

## 禁止措置とアカウントの状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な乱用は、
アカウントの禁止、トークンの取り消し、コンテンツの非表示、またはリスティングの削除につながる場合があります。
公開者の乱用圧力シグナルは毎日確認されます。
ClawHub の潜在的な禁止しきい値に達したシグナルは、自動警告をトリガーする場合があります。
警告期限後の次回の対象スキャンでも公開者が潜在的な禁止しきい値に該当する場合、
ClawHub はアカウント措置を自動的に適用する場合があります。
信頼度の低いシグナルや、時間範囲が限定されたレビューシグナルは、自動執行の対象外です。

削除、禁止、または無効化されたアカウントは ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、
Web UI にサインインしてアカウント状態を確認してください。
禁止または無効化されたアカウントによってサインインまたは通常の CLI アクセスがブロックされている場合は、
復旧レビューに [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーがトリガーしたメールで Skill または plugin バージョンが悪意のあるものとして示されている場合は、
ブロックされた送信バージョンの保存済みスキャン結果をダウンロードしてください。
`clawhub scan download <slug> --version <version>`。plugins の場合は
`--kind plugin` を追加してください。スキャン出力を確認し、リスティングを修正し、バージョン番号を増やして、
修正済みバージョンをアップロードしてください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるために:

- 名前、要約、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- plugins を公開する前にドライランを使用する
- ユーザーまたはモデレーターがリリースの動作について質問した場合は、明確に回答する
