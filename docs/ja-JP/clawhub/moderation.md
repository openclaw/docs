# モデレーションとアカウントの安全性

ClawHub は公開を受け付けていますが、公開ディスカバリーとインストール面には引き続きガードレールが必要です。報告、モデレーション保留、非表示リスティング、アカウント措置は、リリースやアカウントが安全でない、誤解を招く、またはポリシー外であるように見える場合にユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウントの状態について説明します。`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)および[許容される使用](/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ権利に関する懸念については、[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みユーザーは、スキル、プラグイン、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 宣言されていない認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意のある登録または商標の不正使用
- [許容される使用](/clawhub/acceptable-usage)に違反するコンテンツ

スキルページの**スキルを報告**ボタン、またはパッケージ用のパッケージ報告コマンド/APIを使用してください。

サードパーティのスキルまたはプラグイン自体のソースコード内の脆弱性については、ClawHub の報告を使用しないでください。それらは、リスティングからリンクされている公開者またはソースリポジトリに直接報告してください。ClawHub はサードパーティのスキルまたはプラグインのコードを保守または修正しません。

`openclaw/clawhub` の GitHub セキュリティアドバイザリは、ClawHub 自体の脆弱性を対象としています。例には、Web サイト、API、CLI、レジストリ、認証、スキャン、モデレーション、またはダウンロード/インストールの信頼境界にあるバグが含まれます。サードパーティのスキルまたはプラグイン内の脆弱性には、ClawHub のアドバイザリを使用しないでください。

良い報告は具体的で、対応可能です。報告の悪用自体がアカウント措置につながることがあります。

## 組織と名前空間の申請

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争は、製品内の報告フローやアカウント異議申し立てフォームではなく、[組織と名前空間の申請](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移転、改名、非表示、隔離、エイリアス化、またはその他の形でレビューすべきであることを示す非機密の証拠を ClawHub スタッフに確認してもらう必要がある場合は、そのプロセスを使用してください。公開 issue には、シークレット、非公開文書、非公開の法的ファイル、本人確認書類、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

一部の重大な検出事項やポリシー上の問題により、公開者またはリスティングがモデレーション保留になることがあります。これが発生すると、影響を受けるコンテンツは公開ディスカバリーから非表示になる場合があり、また今後の公開は問題がレビューされるまで非表示で開始される場合があります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間にユーザーを保護するためのものです。誤検知が確認された場合には解除されることもあります。

## 非表示またはブロックされたリスティング

リスティングは、保留、非表示、隔離、取り消し、またはその他の理由で公開インストール面から利用できなくなることがあります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、モデレーションによって復元されるまでは、そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示のリスティングについて診断情報を引き続き確認できる場合があります。これらの診断情報は、何が起きたのか、リスティングを公開面に戻す前に何を変更する必要があるのかを説明するのに役立ちます。

## 禁止措置とアカウント状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失うことがあります。重大な悪用は、アカウントの禁止、トークンの取り消し、コンテンツの非表示、またはリスティングの削除につながることがあります。公開者の悪用圧力シグナルは毎日チェックされます。ClawHub の潜在的な禁止しきい値に達したシグナルは、自動警告をトリガーする場合があります。警告期限後の次の対象スキャンでも公開者が潜在的な禁止しきい値内にある場合、ClawHub はアカウント措置を自動的に適用することがあります。信頼度が低いシグナルや時間的に限定されたレビューシグナルは、自動執行の対象外です。

削除済み、禁止済み、または無効化済みのアカウントは、ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。禁止または無効化されたアカウントによってサインインや通常の CLI アクセスがブロックされている場合は、復旧レビューに [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーによって送信されたメールで、スキルまたはプラグインのバージョンが悪意ありとして指定されている場合は、ブロックされた提出済みバージョンの保存済みスキャン結果をダウンロードしてください:
`clawhub scan download <slug> --version <version>`。プラグインの場合は、`--kind plugin` を追加してください。スキャン出力を確認し、リスティングを修正し、バージョン番号を増やして、修正済みバージョンをアップロードしてください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには、次のようにします。

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- プラグインを公開する前にドライランを使用する
- ユーザーまたはモデレーターからリリースの動作について質問された場合は、明確に回答する
