# モデレーションとアカウントの安全性

ClawHub は公開に開かれていますが、公開ディスカバリーとインストールサーフェスには引き続きガードレールが必要です。報告、モデレーション保留、非表示リスティング、アカウント措置は、リリースやアカウントが安全でない、誤解を招く、またはポリシー外に見える場合にユーザーを保護するのに役立ちます。

このページでは、モデレーションとアカウントの状態について説明します。`Pass`、`Review`、`Warn`、`Malicious`、リスクレベルなどの監査ラベルについては、[セキュリティ監査](/clawhub/security-audits)を参照してください。

[セキュリティ](/clawhub/security)と[許容される利用](/clawhub/acceptable-usage)も参照してください。著作権やその他のコンテンツ権利に関する懸念については、[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。

## 報告

サインイン済みユーザーは、Skills、プラグイン、パッケージを報告できます。

ClawHub の報告は、次のような安全でないマーケットプレイスコンテンツにのみ使用してください。

- 悪意のあるリスティング
- 誤解を招くメタデータ
- 宣言されていない認証情報または権限要件
- 不審なインストール手順
- なりすまし
- 悪意ある登録または商標の不正使用
- [許容される利用](/clawhub/acceptable-usage)に違反するコンテンツ

Skill ページの **Skill を報告** ボタン、またはパッケージ用の報告コマンド/API を使用してください。

サードパーティの Skill またはプラグイン自体のソースコードにある脆弱性について、ClawHub の報告を使用しないでください。それらは、リスティングからリンクされている公開者またはソースリポジトリに直接報告してください。ClawHub はサードパーティの Skill またはプラグインのコードを保守またはパッチ適用しません。

`openclaw/clawhub` の GitHub Security Advisories は、ClawHub 自体の脆弱性を対象としています。例として、Web サイト、API、CLI、レジストリ、認証、スキャン、モデレーション、ダウンロード/インストールの信頼境界にあるバグが含まれます。サードパーティの Skills またはプラグインの脆弱性に ClawHub advisories を使用しないでください。

良い報告は具体的で実行可能です。報告の悪用は、それ自体がアカウント措置につながる可能性があります。

## 組織と名前空間の申し立て

組織、ブランド、パッケージスコープ、所有者ハンドル、または名前空間の所有権に関する紛争は、製品内の報告フローやアカウント異議申し立てフォームではなく、[組織と名前空間の申し立て](/clawhub/namespace-claims)プロセスを使用してください。

名前空間を予約、移転、名前変更、非表示、隔離、エイリアス化、またはその他の形でレビューすべきであることを示す、機密でない証拠を ClawHub スタッフにレビューしてもらう必要がある場合は、そのプロセスを使用してください。公開 Issue に、シークレット、非公開文書、非公開の法務ファイル、個人識別文書、API トークン、DNS チャレンジトークンを含めないでください。

## モデレーション保留

重大な検出事項やポリシー上の問題によって、公開者またはリスティングがモデレーション保留に置かれることがあります。この場合、影響を受けるコンテンツは公開ディスカバリーから非表示になるか、今後の公開がレビュー完了まで非表示で開始される可能性があります。

モデレーション保留は、ClawHub が高リスクのケースを解決する間にユーザーを保護するためのものです。誤検知が確認された場合には解除されることもあります。

## 非表示またはブロックされたリスティング

リスティングは、保留、非表示、隔離、取り消し、またはその他の理由で公開インストールサーフェス上で利用できない場合があります。

これらの状態のいずれかが表示された場合、所有者が問題を解決するか、モデレーションによって復元されるまで、そのリリースをインストールしないでください。

所有者は、自分の保留中または非表示のリスティングについて、引き続き診断情報を確認できる場合があります。これらの診断情報は、何が起きたのか、およびリスティングが公開サーフェスに戻る前に何を変更する必要があるのかを説明するのに役立ちます。

## 禁止措置とアカウントの状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う可能性があります。重大な悪用は、アカウントの禁止、トークンの取り消し、コンテンツの非表示、またはリスティングの削除につながる場合があります。公開者の悪用圧力シグナルは毎日チェックされます。ClawHub の潜在的な禁止しきい値に達したシグナルは、自動警告をトリガーする場合があります。警告期限後の次回の対象スキャンでも公開者が潜在的な禁止しきい値内にある場合、ClawHub はアカウント措置を自動的に適用することがあります。信頼度が低いシグナルや時間的に限定されたレビューシグナルは、自動執行の対象外です。

削除、禁止、または無効化されたアカウントは ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。禁止または無効化されたアカウントによりサインインや通常の CLI アクセスがブロックされている場合は、復旧レビューに [ClawHub 異議申し立てフォーム](https://appeals.openclaw.ai/)を使用してください。

スキャナーによってトリガーされたメールが、Skill またはプラグインのバージョンを悪意ありとして示している場合は、ブロックされた提出バージョンの保存済みスキャン結果をダウンロードしてください: `clawhub scan download <slug> --version <version>`。プラグインの場合は `--kind plugin` を追加してください。スキャン出力を確認し、リスティングを修正し、バージョン番号を増やして、修正済みバージョンをアップロードしてください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- プラグインを公開する前にドライランを使用する
- ユーザーやモデレーターからリリースの挙動について質問された場合は明確に回答する
