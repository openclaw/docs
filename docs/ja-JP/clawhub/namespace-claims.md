# 組織と名前空間の申立て

ClawHub は、所有者ハンドル、組織ハンドル、スキルスラッグ、Plugin パッケージ名、
パッケージスコープを公開名前空間として使用します。名前空間が実在する
プロジェクト、ブランド、パッケージエコシステム、または組織に属しているように見えるものの、
ClawHub 上ですでに取得済み、予約済み、誤解を招く状態、または係争中である場合は、
[組織／名前空間の申立て用 Issue フォーム](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
を使用してスタッフに審査を依頼してください。

この手順は、公開可能で機密性のない所有権審査に使用してください。名前空間の申立てに、
製品内の報告機能やアカウント異議申立てフォームを使用しないでください。

## 申立てを提出する場合

実在する主体の所有権を理由として、名前空間の予約、移管、名前変更、非表示、
隔離、エイリアス設定、またはその他の変更が必要かどうかを ClawHub スタッフが
審査すべきだと考える場合は、名前空間の申立てを提出してください。

例：

- GitHub の組織、プロジェクト、会社、またはコミュニティと一致する組織ハンドル
- 対応する ClawHub 所有者だけが公開すべき `@example-org/*` などの
  パッケージスコープ
- プロジェクトになりすましているように見えるスキルスラッグまたは Plugin パッケージ名
- ブランド、商標、プロジェクト名の変更、またはパッケージ履歴に関する紛争
- 正当な名前空間所有者による取得を妨げている、削除済み、非アクティブ、
  または連絡不能な所有者

掲載内容が所有権紛争の範囲を超えて危険、悪意がある、または誤解を招く場合は、
関連するモデレーションまたはセキュリティのガイダンスにも従ってください。名前空間の
申立てフォームは所有権審査用であり、緊急の脆弱性開示用ではありません。

## 提出前の確認

まず、名前空間と一致する所有者として公開していることを確認してください。
Plugin パッケージでは、`@example-org/example-plugin` などのスコープ付き名称は、
対応する `example-org` 所有者として公開する必要があります。

現在の所有者を管理できる場合は、影響を受けるリソースの公開、名前変更、移管、
非表示化、または削除により、名前空間を直接修正してください。現在の所有者を
管理できない場合や、紛争の解決にスタッフの対応が必要な場合は、申立てを使用してください。

## 提示する証拠

公開可能で機密性のない証拠を使用してください。有用な証拠には次のものがあります。

- GitHub の組織、リポジトリ、リリース、またはメンテナーの履歴
- 名前空間を明記した公式プロジェクトドキュメント
- ドメインまたは公式メールドメインの証明
- npm、PyPI、crates.io、またはその他のパッケージレジストリのスコープ管理権限
- 公開しても問題のない、商標、ブランド、またはプロジェクトの所有権を示す証拠
- ソースリポジトリの履歴、パッケージ履歴、または公開された名前変更のお知らせ
- 係争対象の ClawHub 所有者、スキル、Plugin、パッケージ、または Issue へのリンク

各リンクが何を証明するか説明してください。スタッフが非公開の認証情報やシークレットを
必要とせずに関係性を理解できるようにしてください。

## 提示してはいけないもの

公開 GitHub Issue にシークレットや非公開の証拠を投稿しないでください。
次のものを含めないでください。

- API トークン、署名鍵、または認証情報
- DNS チャレンジトークン
- 非公開の法的文書または契約書
- 個人の身分証明書
- 非公開メール、非公開のセキュリティ報告、または機密の顧客データ

申立てフォームでは、機密性の高い証拠についてスタッフとの非公開チャネルが
必要かどうかを尋ねます。機密情報を公開する代わりに、そのオプションを使用してください。

## 想定される結果

証拠とリスクに応じて、ClawHub スタッフは名前空間の予約、所有権の移管、
リソースの名前変更、既存の掲載の非表示化または隔離、エイリアスまたはリダイレクトの追加、
追加証拠の要求、あるいは申請の却下を行う場合があります。

名前空間の審査によって、一致するすべての名称が必ず移管されるわけではありません。
スタッフは、公開された証拠、既存の利用状況、セキュリティリスク、ユーザーへの影響を考慮します。
