# 組織と名前空間の申請

ClawHub は、所有者ハンドル、組織ハンドル、スキルのスラッグ、Plugin パッケージ名、パッケージスコープを公開名前空間として使用します。名前空間が実在するプロジェクト、ブランド、パッケージエコシステム、または組織に属しているように見えるものの、ClawHub 上ですでに申請済み、予約済み、誤解を招く状態、または係争中である場合は、スタッフに
[組織／名前空間の申請 Issue フォーム](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
から審査を依頼してください。

この手順は、公開可能かつ機密性のない所有権審査に使用してください。名前空間の申請には、製品内の報告機能やアカウント異議申し立てフォームを使用しないでください。

## 申請を開始する場合

実世界の所有権を理由として名前空間を予約、移管、改名、非表示、隔離、エイリアス化、またはその他の方法で変更すべきかを ClawHub スタッフが審査する必要があると考える場合は、名前空間の申請を開始してください。

次のような例があります。

- 自分の GitHub 組織、プロジェクト、会社、またはコミュニティと一致する組織ハンドル
- 対応する ClawHub 所有者の下でのみ公開されるべき `@example-org/*` などのパッケージスコープ
- プロジェクトになりすましているように見えるスキルのスラッグまたは Plugin パッケージ名
- ブランド、商標、プロジェクトの改名、またはパッケージ履歴に関する係争
- 正当な名前空間の所有者による使用を妨げている、削除済み、非アクティブ、または連絡不能な所有者

所有権の係争にとどまらず、掲載内容が安全でない、悪意がある、または誤解を招く場合は、関連するモデレーションまたはセキュリティのガイダンスにも従ってください。名前空間申請フォームは所有権審査のためのものであり、緊急の脆弱性開示のためのものではありません。

## 申請前の確認

まず、名前空間と一致する所有者として公開していることを確認してください。Plugin パッケージの場合、`@example-org/example-plugin` のようなスコープ付きの名前は、対応する `example-org` 所有者として公開する必要があります。

現在の所有者を管理できる場合は、影響を受けるリソースを公開、改名、移管、非表示、または削除して、名前空間を直接修正してください。現在の所有者を管理できない場合、またはスタッフによる係争の解決が必要な場合は、申請を使用してください。

## 提出する証拠

公開可能かつ機密性のない証拠を使用してください。有用な証拠には次のものがあります。

- GitHub 組織、リポジトリ、リリース、またはメンテナーの履歴
- 名前空間を明記した公式プロジェクトドキュメント
- ドメインまたは公式メールドメインの証明
- npm、PyPI、crates.io、またはその他のパッケージレジストリのスコープ管理権限
- 公開の場で安全に説明できる商標、ブランド、またはプロジェクトの所有権に関する証拠
- ソースリポジトリの履歴、パッケージの履歴、または公開された改名通知
- 係争対象の ClawHub 所有者、スキル、Plugin、パッケージ、または Issue へのリンク

各リンクが何を証明するのかを説明してください。スタッフが非公開の認証情報やシークレットを必要とせずに関係を理解できるようにしてください。

## 提出してはいけないもの

公開 GitHub Issue にシークレットや非公開の証拠を記載しないでください。以下のものを含めないでください。

- API トークン、署名鍵、または認証情報
- DNS チャレンジトークン
- 非公開の法的文書または契約書
- 個人の身分証明書
- 非公開メール、非公開のセキュリティ報告、または機密の顧客データ

申請フォームでは、機密性の高い証拠についてスタッフとの非公開チャネルが必要かどうかを尋ねます。機密情報を公開する代わりに、その選択肢を使用してください。

## 想定される結果

証拠とリスクに応じて、ClawHub スタッフは名前空間の予約、所有権の移管、リソースの改名、既存の掲載情報の非表示または隔離、エイリアスまたはリダイレクトの追加、追加証拠の依頼、または申請の却下を行う場合があります。

名前空間の審査によって、一致するすべての名前が必ず移管されるとは限りません。スタッフは、公開されている証拠、既存の利用状況、セキュリティリスク、ユーザーへの影響を考慮します。
