# Org と名前空間の申し立て

ClawHub は、owner ハンドル、org ハンドル、スキルスラッグ、Plugin パッケージ名、
パッケージスコープを公開名前空間として使用します。名前空間が現実世界のプロジェクト、
ブランド、パッケージエコシステム、または組織に属しているように見えるものの、ClawHub
ですでに申し立て済み、予約済み、誤解を招く、または争議中である場合は、
[Org / 名前空間の申し立て issue フォーム](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
でスタッフにレビューを依頼してください。

公開の非機密な所有権レビューにはこの手順を使用してください。名前空間の申し立てに、
製品内レポートやアカウント異議申し立てフォームを使用しないでください。

## 申し立てを開くタイミング

現実世界での所有権を理由に、名前空間を予約、移転、名前変更、非表示、隔離、エイリアス化、
またはその他の方法で変更すべきかを ClawHub スタッフがレビューする必要があると考える場合は、
名前空間の申し立てを開いてください。

例:

- 自分の GitHub org、プロジェクト、会社、またはコミュニティと一致する org ハンドル
- 対応する ClawHub owner の下でのみ公開されるべき `@example-org/*` のようなパッケージスコープ
- プロジェクトになりすましているように見えるスキルスラッグまたは Plugin パッケージ名
- ブランド、商標、プロジェクト名変更、またはパッケージ履歴に関する争議
- 正当な名前空間 owner を妨げている、削除済み、非アクティブ、または連絡不能な owner

リスティングが所有権の争議を超えて危険、悪意のあるもの、または誤解を招くものである場合は、
関連するモデレーションまたはセキュリティのガイダンスにも従ってください。名前空間の申し立て
フォームは所有権レビュー用であり、緊急の脆弱性開示用ではありません。

## 申請前に

まず、名前空間に一致する owner で公開していることを確認してください。Plugin パッケージでは、
`@example-org/example-plugin` のようなスコープ付き名前は、対応する `example-org` owner
として公開する必要があります。

現在の owner を管理できる場合は、影響を受けるリソースを公開、名前変更、移転、非表示、または削除して、
名前空間を直接修正してください。現在の owner を管理できない場合、またはスタッフが争議を解決する
必要がある場合に申し立てを使用してください。

## 含める証拠

公開の非機密な証拠を使用してください。有用な証拠には次のものがあります。

- GitHub org、リポジトリ、リリース、またはメンテナー履歴
- 名前空間を明記している公式プロジェクトドキュメント
- ドメインまたは公式メールドメインの証拠
- npm、PyPI、crates.io、またはその他のパッケージレジストリのスコープ管理
- 公開で議論しても安全な商標、ブランド、またはプロジェクト所有権の証拠
- ソースリポジトリ履歴、パッケージ履歴、または公開された名前変更通知
- 争議対象の ClawHub owner、スキル、Plugin、パッケージ、または issue へのリンク

各リンクが何を証明するかを説明してください。スタッフが非公開の認証情報やシークレットを必要とせずに、
関係を理解できるようにしてください。

## 含めてはいけないもの

公開 GitHub issue にシークレットや非公開の証拠を入れないでください。次のものを含めないでください。

- API トークン、署名キー、または認証情報
- DNS チャレンジトークン
- 非公開の法的ファイルまたは契約
- 個人の身分証明書
- 非公開メール、非公開のセキュリティレポート、または機密の顧客データ

申し立てフォームでは、機密の証拠に非公開のスタッフチャネルが必要かどうかを尋ねます。
機密素材を公開投稿する代わりに、そのオプションを使用してください。

## 想定される結果

証拠とリスクに応じて、ClawHub スタッフは名前空間の予約、所有権の移転、リソースの名前変更、
既存リスティングの非表示または隔離、エイリアスまたはリダイレクトの追加、追加証拠の依頼、
またはリクエストの却下を行う場合があります。

名前空間レビューは、一致するすべての名前が移転されることを保証するものではありません。
スタッフは公開証拠、既存の利用状況、セキュリティリスク、ユーザーへの影響を考慮します。
