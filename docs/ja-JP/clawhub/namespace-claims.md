# 組織と名前空間の申し立て

ClawHub は、オーナーハンドル、組織ハンドル、skill slug、plugin package name、package scope を公開名前空間として使用します。名前空間が現実世界のプロジェクト、ブランド、パッケージエコシステム、または組織に属しているように見えるものの、ClawHub 上ですでに申し立て済み、予約済み、誤解を招く、または係争中の場合は、[組織 / 名前空間の申し立て issue フォーム](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)でスタッフにレビューを依頼してください。

この経路は、公開の非機密な所有権レビューに使用します。名前空間の申し立てには、製品内レポートやアカウント異議申し立てフォームを使用しないでください。

## 申し立てを開くべき場合

現実世界の所有権を理由に、名前空間を予約、移管、名前変更、非表示、隔離、エイリアス化、またはその他の方法で変更すべきかを ClawHub スタッフがレビューする必要があると考える場合は、名前空間の申し立てを開いてください。

例:

- GitHub 組織、プロジェクト、会社、またはコミュニティと一致する組織ハンドル
- 一致する ClawHub オーナーの下でのみ公開されるべき `@example-org/*` のような package scope
- プロジェクトになりすましているように見える skill slug または plugin package name
- ブランド、商標、プロジェクト名変更、またはパッケージ履歴の係争
- 正当な名前空間オーナーを妨げている、削除済み、非アクティブ、または到達不能なオーナー

掲載内容が所有権係争を超えて安全でない、悪意がある、または誤解を招く場合は、関連するモデレーションまたはセキュリティのガイダンスにも従ってください。名前空間の申し立てフォームは所有権レビュー用であり、緊急の脆弱性開示用ではありません。

## 提出前に

まず、名前空間と一致するオーナーで公開していることを確認してください。plugin package の場合、`@example-org/example-plugin` のような scoped name は、一致する `example-org` オーナーとして公開する必要があります。

現在のオーナーを管理できる場合は、影響を受けるリソースを公開、名前変更、移管、非表示、または削除することで名前空間を直接修正してください。現在のオーナーを管理できない場合、またはスタッフが係争を解決する必要がある場合に申し立てを使用してください。

## 含める証拠

公開の非機密な証拠を使用してください。有用な証明には以下が含まれます。

- GitHub 組織、リポジトリ、リリース、またはメンテナー履歴
- 名前空間を明記している公式プロジェクトドキュメント
- ドメインまたは公式メールドメインの証明
- npm、PyPI、crates.io、またはその他のパッケージレジストリの scope 管理
- 公開で議論しても安全な商標、ブランド、またはプロジェクト所有権の証拠
- ソースリポジトリ履歴、パッケージ履歴、または公開の名前変更通知
- 係争中の ClawHub オーナー、skill、plugin、パッケージ、または issue へのリンク

各リンクが何を証明するかを説明してください。スタッフがプライベートな認証情報やシークレットを必要とせずに関係を理解できるようにしてください。

## 含めてはいけないもの

公開 GitHub issue にシークレットや非公開の証明を記載しないでください。以下を含めないでください。

- API トークン、署名キー、または認証情報
- DNS チャレンジトークン
- 非公開の法務ファイルまたは契約書
- 個人の身分証明書
- 非公開メール、非公開セキュリティレポート、または機密顧客データ

申し立てフォームでは、機密証拠に非公開のスタッフチャンネルが必要かどうかを尋ねます。機密資料を公開投稿する代わりに、そのオプションを使用してください。

## 考えられる結果

証拠とリスクに応じて、ClawHub スタッフは名前空間の予約、所有権の移管、リソースの名前変更、既存掲載の非表示または隔離、エイリアスまたはリダイレクトの追加、追加証明の依頼、またはリクエストの却下を行う場合があります。

名前空間レビューは、一致するすべての名前が移管されることを保証するものではありません。スタッフは公開証拠、既存の使用状況、セキュリティリスク、ユーザー影響を考慮します。
