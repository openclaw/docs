# 組織と名前空間の申し立て

ClawHub は、オーナーハンドル、組織ハンドル、skill スラッグ、plugin パッケージ名、package スコープを公開名前空間として使用します。名前空間が実在するプロジェクト、ブランド、パッケージエコシステム、または組織に属しているように見える一方で、ClawHub 上ですでに申し立て済み、予約済み、誤解を招く、または争議中である場合は、スタッフに
[組織 / 名前空間申し立て issue フォーム](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
でレビューを依頼してください。

この手順は、公開可能で機密性のない所有権レビューに使用します。名前空間の申し立てには、製品内レポートやアカウント異議申し立てフォームを使用しないでください。

## 申し立てを開始する場合

実世界での所有権を理由として、名前空間を予約、移転、名前変更、非表示、隔離、エイリアス化、またはその他の方法で変更すべきかを ClawHub スタッフがレビューする必要があると考える場合は、名前空間の申し立てを開始してください。

例:

- 自分の GitHub 組織、プロジェクト、会社、またはコミュニティと一致する組織ハンドル
- 一致する ClawHub オーナーの下でのみ公開されるべき `@example-org/*` などの package スコープ
- プロジェクトになりすましているように見える skill スラッグまたは plugin パッケージ名
- ブランド、商標、プロジェクト名変更、またはパッケージ履歴に関する争議
- 正当な名前空間オーナーを妨げている、削除済み、非アクティブ、または連絡不能なオーナー

リスティングが所有権争議を超えて危険、悪意がある、または誤解を招く場合は、関連するモデレーションまたはセキュリティのガイダンスにも従ってください。名前空間申し立てフォームは所有権レビュー用であり、緊急の脆弱性開示用ではありません。

## 提出前に

まず、その名前空間に一致するオーナーで公開していることを確認してください。plugin パッケージの場合、`@example-org/example-plugin` などのスコープ付き名前は、一致する `example-org` オーナーとして公開する必要があります。

現在のオーナーを管理できる場合は、影響を受けるリソースを公開、名前変更、移転、非表示、または削除して、名前空間を直接修正してください。現在のオーナーを管理できない場合、またはスタッフが争議を解決する必要がある場合に申し立てを使用してください。

## 含める証拠

公開可能で機密性のない証拠を使用してください。有用な証拠には次のものがあります。

- GitHub 組織、リポジトリ、リリース、またはメンテナー履歴
- 名前空間を明記している公式プロジェクトドキュメント
- ドメインまたは公式メールドメインの証拠
- npm、PyPI、crates.io、またはその他のパッケージレジストリのスコープ管理
- 公開で議論しても安全な商標、ブランド、またはプロジェクト所有権の証拠
- ソースリポジトリ履歴、パッケージ履歴、または公開された名前変更通知
- 争議対象の ClawHub オーナー、skill、plugin、パッケージ、または issue へのリンク

各リンクが何を証明するかを説明してください。スタッフがプライベートな認証情報やシークレットを必要とせずに関係性を理解できるようにしてください。

## 含めてはいけないもの

シークレットや非公開の証拠を公開 GitHub issue に記載しないでください。次のものを含めないでください。

- API トークン、署名キー、または認証情報
- DNS チャレンジトークン
- 非公開の法的文書または契約書
- 個人の本人確認書類
- 非公開メール、非公開セキュリティレポート、または機密の顧客データ

申し立てフォームでは、機密性の高い証拠にスタッフとの非公開チャネルが必要かどうかを尋ねます。機密資料を公開投稿する代わりに、その選択肢を使用してください。

## 起こり得る結果

証拠とリスクに応じて、ClawHub スタッフは名前空間の予約、所有権の移転、リソースの名前変更、既存リスティングの非表示または隔離、エイリアスまたはリダイレクトの追加、追加証拠の依頼、またはリクエストの却下を行う場合があります。

名前空間レビューは、一致するすべての名前が移転されることを保証するものではありません。スタッフは公開証拠、既存の使用状況、セキュリティリスク、ユーザーへの影響を総合的に判断します。
