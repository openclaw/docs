# 組織と名前空間の申し立て

ClawHub は、オーナー ハンドル、組織ハンドル、スキル スラッグ、Plugin パッケージ名、
パッケージ スコープを公開名前空間として使用します。名前空間が実在する
プロジェクト、ブランド、パッケージ エコシステム、または組織に属しているように見える一方で、
ClawHub 上ですでに申し立て済み、予約済み、誤解を招く、または係争中である場合は、
[組織 / 名前空間の申し立て issue フォーム](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)で
スタッフにレビューを依頼してください。

この手順は、公開可能で機密性のない所有権レビューに使用してください。名前空間の申し立てには、
製品内レポートやアカウント異議申し立てフォームを使用しないでください。

## 申し立てを開くタイミング

実世界での所有権を理由として、名前空間を予約、移管、名称変更、非表示、隔離、エイリアス化、
またはその他の方法で変更すべきかを ClawHub スタッフがレビューすべきだと考える場合に、
名前空間の申し立てを開いてください。

例:

- GitHub 組織、プロジェクト、会社、またはコミュニティと一致する組織ハンドル
- 一致する ClawHub オーナーの下でのみ公開されるべき `@example-org/*` のようなパッケージ スコープ
- プロジェクトになりすましているように見えるスキル スラッグまたは Plugin パッケージ名
- ブランド、商標、プロジェクト名変更、またはパッケージ履歴に関する係争
- 正当な名前空間オーナーを妨げている、削除済み、非アクティブ、または連絡不能なオーナー

掲載内容が所有権の係争を超えて危険、悪意あるもの、または誤解を招くものである場合は、
関連するモデレーションまたはセキュリティのガイダンスにも従ってください。名前空間の申し立て
フォームは所有権レビュー用であり、緊急の脆弱性開示用ではありません。

## 提出前に

まず、名前空間と一致するオーナーで公開していることを確認してください。Plugin パッケージでは、
`@example-org/example-plugin` のようなスコープ付き名前は、一致する `example-org` オーナーとして
公開する必要があります。

現在のオーナーを管理できる場合は、影響を受けるリソースを公開、名称変更、移管、非表示、
または削除して、名前空間を直接修正してください。現在のオーナーを管理できない場合、または
スタッフが係争を解決する必要がある場合に申し立てを使用してください。

## 含めるべき証拠

公開可能で機密性のない証拠を使用してください。有用な証拠には以下が含まれます。

- GitHub 組織、リポジトリ、リリース、またはメンテナー履歴
- 名前空間を明記している公式プロジェクト ドキュメント
- ドメインまたは公式メール ドメインの証明
- npm、PyPI、crates.io、またはその他のパッケージ レジストリのスコープ管理
- 公開の場で安全に議論できる商標、ブランド、またはプロジェクト所有権の証拠
- ソース リポジトリ履歴、パッケージ履歴、または公開された名称変更通知
- 係争対象の ClawHub オーナー、スキル、Plugin、パッケージ、または issue へのリンク

各リンクが何を証明するのかを説明してください。スタッフが、非公開の認証情報やシークレットを
必要とせずに関係性を理解できるようにしてください。

## 含めてはいけないもの

公開 GitHub issue にシークレットや非公開の証拠を記載しないでください。以下を含めないでください。

- API トークン、署名キー、または認証情報
- DNS チャレンジ トークン
- 非公開の法的ファイルまたは契約書
- 個人の身分証明書
- 非公開メール、非公開セキュリティ レポート、または機密の顧客データ

申し立てフォームでは、機密性の高い証拠に非公開のスタッフ チャネルが必要かを尋ねます。
機密資料を公開投稿する代わりに、そのオプションを使用してください。

## 想定される結果

証拠とリスクに応じて、ClawHub スタッフは名前空間の予約、所有権の移管、リソースの名称変更、
既存掲載の非表示または隔離、エイリアスまたはリダイレクトの追加、追加証拠の要求、
またはリクエストの却下を行う場合があります。

名前空間レビューは、一致するすべての名前が移管されることを保証するものではありません。
スタッフは公開証拠、既存の利用状況、セキュリティ リスク、ユーザーへの影響を考慮します。
