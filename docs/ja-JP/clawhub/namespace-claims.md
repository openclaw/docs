# 組織と名前空間の申請

ClawHub は、オーナーハンドル、組織ハンドル、Skills スラッグ、Plugin パッケージ名、
パッケージスコープを公開名前空間として使用します。名前空間が
現実のプロジェクト、ブランド、パッケージエコシステム、または組織に属しているように見えるものの、
ClawHub 上ですでに取得済み、予約済み、誤解を招く状態、または係争中の場合は、スタッフに
[組織 / 名前空間申請 issue フォーム](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)でレビューを依頼してください。

公開の、機密性のない所有権レビューにはこの経路を使用してください。名前空間の申請に、製品内
レポートやアカウント異議申し立てフォームを使用しないでください。

## 申請を開くタイミング

現実の所有権を理由に、名前空間を予約、移管、改名、非表示、隔離、エイリアス化、
またはその他の方法で変更すべきかを ClawHub スタッフがレビューする必要があると考える場合は、
名前空間申請を開いてください。

例:

- 自分の GitHub 組織、プロジェクト、会社、またはコミュニティと一致する組織ハンドル
- 一致する ClawHub オーナーの下でのみ公開されるべき `@example-org/*` のようなパッケージスコープ
- プロジェクトになりすましているように見える Skills スラッグまたは Plugin パッケージ名
- ブランド、商標、プロジェクト改名、またはパッケージ履歴に関する係争
- 正当な名前空間オーナーを妨げている、削除済み、非アクティブ、または連絡不能なオーナー

リスティングが所有権の係争を超えて安全でない、悪意がある、または誤解を招く場合は、
関連するモデレーションまたはセキュリティのガイダンスにも従ってください。名前空間申請
フォームは所有権レビュー用であり、緊急の脆弱性開示用ではありません。

## 提出前に

まず、名前空間と一致するオーナーで公開していることを確認してください。
Plugin パッケージの場合、`@example-org/example-plugin` のようなスコープ付き名は、
一致する `example-org` オーナーとして公開する必要があります。

現在のオーナーを管理できる場合は、影響を受けるリソースを公開、改名、移管、非表示、
または削除して、名前空間を直接修正してください。現在のオーナーを管理できない場合、
またはスタッフが係争を解決する必要がある場合に申請を使用してください。

## 含める証拠

公開の、機密性のない証拠を使用してください。有用な証拠には次のものがあります。

- GitHub 組織、リポジトリ、リリース、またはメンテナー履歴
- 名前空間を明記している公式プロジェクトドキュメント
- ドメインまたは公式メールドメインの証明
- npm、PyPI、crates.io、またはその他のパッケージレジストリのスコープ管理
- 公開で議論しても安全な商標、ブランド、またはプロジェクト所有権の証拠
- ソースリポジトリ履歴、パッケージ履歴、または公開された改名通知
- 係争中の ClawHub オーナー、Skills、Plugin、パッケージ、または issue へのリンク

各リンクが何を証明するのかを説明してください。スタッフが、非公開の認証情報やシークレットを必要とせずに
関係性を理解できるようにしてください。

## 含めてはいけないもの

公開 GitHub issue にシークレットや非公開の証拠を記載しないでください。次のものを含めないでください。

- API トークン、署名キー、または認証情報
- DNS チャレンジトークン
- 非公開の法的文書または契約
- 個人の本人確認書類
- 非公開メール、非公開セキュリティレポート、または機密の顧客データ

申請フォームでは、機密性の高い証拠に非公開のスタッフチャネルが必要かどうかを確認します。
機密資料を公開投稿する代わりに、そのオプションを使用してください。

## 考えられる結果

証拠とリスクに応じて、ClawHub スタッフは名前空間の予約、所有権の移管、
リソースの改名、既存リスティングの非表示または隔離、エイリアスまたはリダイレクトの追加、
追加証拠の依頼、またはリクエストの却下を行う場合があります。

名前空間レビューは、一致するすべての名前が移管されることを保証するものではありません。
スタッフは、公開証拠、既存の利用状況、セキュリティリスク、ユーザー影響を総合的に判断します。
