# 組織と名前空間の申請

ClawHub は owner ハンドル、組織ハンドル、skill スラッグ、plugin パッケージ名、
パッケージ scope を公開名前空間として使用します。名前空間が実在するプロジェクト、
ブランド、パッケージエコシステム、または組織に属しているように見えるものの、
ClawHub ですでに申請済み、予約済み、誤解を招く、または係争中である場合は、
[組織 / 名前空間申請 issue フォーム](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
でスタッフにレビューを依頼してください。

公開の、機密ではない所有権レビューにはこの手順を使用してください。名前空間の申請には、
製品内レポートやアカウント異議申し立てフォームを使用しないでください。

## 申請を開くタイミング

実世界での所有権を理由として、名前空間を予約、移管、名前変更、非表示、隔離、
エイリアス化、またはその他の変更をすべきかどうかを ClawHub スタッフがレビューする
必要があると考える場合に、名前空間申請を開いてください。

例:

- GitHub 組織、プロジェクト、会社、またはコミュニティと一致する組織ハンドル
- 一致する ClawHub owner の下でのみ公開されるべき `@example-org/*` のような
  パッケージ scope
- プロジェクトになりすましているように見える skill スラッグまたは plugin パッケージ名
- ブランド、商標、プロジェクト名の変更、またはパッケージ履歴に関する係争
- 正当な名前空間 owner を妨げている、削除済み、非アクティブ、または連絡不能な owner

掲載情報が所有権の係争を超えて安全でない、悪意がある、または誤解を招く場合は、
関連するモデレーションまたはセキュリティのガイダンスにも従ってください。名前空間申請
フォームは所有権レビューのためのものであり、緊急の脆弱性開示のためのものではありません。

## 申請前に

まず、名前空間と一致する owner で公開していることを確認してください。plugin パッケージでは、
`@example-org/example-plugin` のような scope 付きの名前は、一致する `example-org`
owner として公開する必要があります。

現在の owner を管理できる場合は、影響を受けるリソースを公開、名前変更、移管、非表示、
または削除して、名前空間を直接修正してください。現在の owner を管理できない場合、
またはスタッフが係争を解決する必要がある場合に申請を使用してください。

## 含めるべき証拠

公開の、機密ではない証拠を使用してください。役立つ証拠には次のものがあります。

- GitHub 組織、リポジトリ、リリース、またはメンテナー履歴
- 名前空間を明記している公式プロジェクトドキュメント
- ドメインまたは公式メールドメインの証拠
- npm、PyPI、crates.io、またはその他のパッケージレジストリの scope 管理
- 公開で議論しても安全な商標、ブランド、またはプロジェクト所有権の証拠
- ソースリポジトリ履歴、パッケージ履歴、または公開された名前変更通知
- 係争中の ClawHub owner、skill、plugin、パッケージ、または issue へのリンク

各リンクが何を証明するかを説明してください。スタッフは、非公開の認証情報やシークレットを
必要とせずに関係を理解できる必要があります。

## 含めてはいけないもの

シークレットや非公開の証拠を公開 GitHub issue に記載しないでください。次のものを
含めないでください。

- API トークン、署名キー、または認証情報
- DNS チャレンジトークン
- 非公開の法的文書または契約書
- 個人の本人確認書類
- 非公開メール、非公開セキュリティレポート、または機密の顧客データ

申請フォームでは、機密性の高い証拠に非公開のスタッフチャンネルが必要かどうかを尋ねます。
機密資料を公開投稿する代わりに、そのオプションを使用してください。

## 想定される結果

証拠とリスクに応じて、ClawHub スタッフは名前空間の予約、所有権の移管、リソースの名前変更、
既存の掲載情報の非表示または隔離、エイリアスまたはリダイレクトの追加、追加証拠の依頼、
またはリクエストの却下を行う場合があります。

名前空間レビューは、一致するすべての名前が移管されることを保証するものではありません。
スタッフは公開証拠、既存の利用状況、セキュリティリスク、ユーザーへの影響を比較検討します。
