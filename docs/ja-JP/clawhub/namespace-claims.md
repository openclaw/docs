# 組織と名前空間の申請

ClawHub は、オーナーのハンドル、組織のハンドル、スキルのスラッグ、Plugin パッケージ名、
パッケージスコープを公開名前空間として使用します。名前空間が実在するプロジェクト、
ブランド、パッケージエコシステム、または組織に属しているように見えるものの、
ClawHub 上ですでに申請済み、予約済み、誤解を招く、または係争中である場合は、
[組織 / 名前空間申請の issue フォーム](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
でスタッフにレビューを依頼してください。

この手順は、公開可能で機微ではない所有権レビューに使用します。名前空間の申請には、
製品内レポートやアカウント異議申し立てフォームを使用しないでください。

## 申請を開くべき場合

実世界での所有権を理由に、名前空間を予約、移譲、名称変更、非表示、隔離、エイリアス化、
またはその他の方法で変更すべきかどうかを ClawHub スタッフにレビューしてほしい場合は、
名前空間申請を開いてください。

例:

- GitHub 組織、プロジェクト、会社、またはコミュニティと一致する組織ハンドル
- 対応する ClawHub オーナーのみが公開すべき `@example-org/*` のようなパッケージスコープ
- プロジェクトになりすましているように見えるスキルのスラッグまたは Plugin パッケージ名
- ブランド、商標、プロジェクト名変更、またはパッケージ履歴に関する係争
- 正当な名前空間オーナーを妨げている、削除済み、非アクティブ、または連絡不能なオーナー

掲載内容が所有権の係争を超えて安全でない、悪意がある、または誤解を招く場合は、
関連するモデレーションまたはセキュリティのガイダンスにも従ってください。名前空間申請
フォームは所有権レビューのためのものであり、緊急の脆弱性開示のためのものではありません。

## 申請前に

まず、名前空間と一致するオーナーで公開していることを確認してください。
Plugin パッケージでは、`@example-org/example-plugin` のようなスコープ付き名前は、
対応する `example-org` オーナーとして公開する必要があります。

現在のオーナーを管理できる場合は、影響を受けるリソースを公開、名称変更、移譲、
非表示、または削除して、名前空間を直接修正してください。現在のオーナーを管理できない場合、
またはスタッフが係争を解決する必要がある場合に申請を使用してください。

## 含めるべき証拠

公開可能で機微ではない証拠を使用してください。有用な証拠には次のものがあります。

- GitHub の組織、リポジトリ、リリース、またはメンテナー履歴
- 名前空間を明示している公式プロジェクトドキュメント
- ドメインまたは公式メールドメインの証明
- npm、PyPI、crates.io、またはその他のパッケージレジストリのスコープ管理
- 公開で議論しても安全な商標、ブランド、またはプロジェクト所有権の証拠
- ソースリポジトリ履歴、パッケージ履歴、または公開された名称変更通知
- 係争対象の ClawHub オーナー、スキル、Plugin、パッケージ、または issue へのリンク

各リンクが何を証明するのかを説明してください。スタッフは、非公開の認証情報やシークレットを
必要とせずに関係を理解できる必要があります。

## 含めてはいけないもの

公開 GitHub issue にシークレットや非公開の証拠を載せないでください。次のものは含めないでください。

- API トークン、署名鍵、または認証情報
- DNS チャレンジトークン
- 非公開の法的ファイルまたは契約書
- 個人の本人確認書類
- 非公開メール、非公開セキュリティレポート、または機密顧客データ

申請フォームでは、機微な証拠に非公開のスタッフチャネルが必要かどうかを尋ねます。
機微な資料を公開投稿する代わりに、そのオプションを使用してください。

## 想定される結果

証拠とリスクに応じて、ClawHub スタッフは名前空間の予約、所有権の移譲、リソースの名称変更、
既存掲載の非表示または隔離、エイリアスまたはリダイレクトの追加、追加証拠の依頼、
またはリクエストの却下を行う場合があります。

名前空間レビューは、一致するすべての名前が移譲されることを保証するものではありません。
スタッフは、公開証拠、既存の利用状況、セキュリティリスク、ユーザーへの影響を考慮します。
