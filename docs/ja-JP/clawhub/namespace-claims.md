# Org と名前空間の申し立て

ClawHub は、所有者ハンドル、org ハンドル、skill スラッグ、plugin パッケージ名、
パッケージスコープを公開名前空間として使用します。名前空間が実在のプロジェクト、
ブランド、パッケージエコシステム、組織に属しているように見えるものの、ClawHub ですでに
申し立て済み、予約済み、誤解を招く、または争われている場合は、
[Org / Namespace Claim issue form](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
でスタッフにレビューを依頼してください。

公開の非機密な所有権レビューにはこの手順を使用してください。名前空間の申し立てに、
製品内レポートやアカウント異議申し立てフォームは使用しないでください。

## 申し立てを開くタイミング

ClawHub スタッフが、実世界の所有権を理由に名前空間を予約、移転、名前変更、非表示、
隔離、エイリアス化、またはその他の変更をすべきかレビューする必要があると考える場合は、
名前空間の申し立てを開いてください。

例:

- 自分の GitHub org、プロジェクト、会社、コミュニティと一致する org ハンドル
- 一致する ClawHub 所有者の下でのみ公開されるべき `@example-org/*` のようなパッケージスコープ
- プロジェクトになりすましているように見える skill スラッグまたは plugin パッケージ名
- ブランド、商標、プロジェクト名変更、またはパッケージ履歴の争議
- 正当な名前空間所有者を妨げている、削除済み、非アクティブ、または連絡不能な所有者

掲載内容が所有権の争議を超えて危険、悪意がある、または誤解を招く場合は、
関連するモデレーションまたはセキュリティのガイダンスにも従ってください。名前空間の申し立て
フォームは所有権レビュー用であり、緊急の脆弱性開示用ではありません。

## 提出前に

まず、その名前空間と一致する所有者で公開していることを確認してください。
plugin パッケージでは、`@example-org/example-plugin` のようなスコープ付き名前は、
一致する `example-org` 所有者として公開する必要があります。

現在の所有者を管理できる場合は、影響を受けるリソースを公開、名前変更、移転、非表示、
または削除して、名前空間を直接修正してください。現在の所有者を管理できない場合、
またはスタッフが争議を解決する必要がある場合に申し立てを使用してください。

## 含める証拠

公開可能で非機密の証拠を使用してください。有用な証拠には次が含まれます。

- GitHub org、リポジトリ、リリース、またはメンテナー履歴
- 名前空間を明記している公式プロジェクトドキュメント
- ドメインまたは公式メールドメインの証明
- npm、PyPI、crates.io、またはその他のパッケージレジストリのスコープ管理
- 公開の場で議論しても安全な商標、ブランド、またはプロジェクト所有権の証拠
- ソースリポジトリ履歴、パッケージ履歴、または公開された名前変更通知
- 争議対象の ClawHub 所有者、skill、plugin、パッケージ、または issue へのリンク

各リンクが何を証明するかを説明してください。スタッフが非公開の認証情報やシークレットなしで、
関係性を理解できる必要があります。

## 含めてはいけないもの

公開 GitHub issue にシークレットや非公開の証拠を載せないでください。次を含めないでください。

- API トークン、署名キー、または認証情報
- DNS チャレンジトークン
- 非公開の法的文書または契約書
- 個人の本人確認書類
- 非公開メール、非公開セキュリティレポート、または機密の顧客データ

申し立てフォームでは、機密証拠に非公開のスタッフチャネルが必要かどうかを尋ねます。
機密資料を公開投稿する代わりに、その選択肢を使用してください。

## 想定される結果

証拠とリスクに応じて、ClawHub スタッフは名前空間の予約、所有権の移転、リソースの名前変更、
既存掲載の非表示または隔離、エイリアスまたはリダイレクトの追加、追加証拠の依頼、
またはリクエストの却下を行う場合があります。

名前空間レビューは、一致するすべての名前が移転されることを保証するものではありません。
スタッフは公開証拠、既存の利用状況、セキュリティリスク、ユーザー影響を考慮します。
