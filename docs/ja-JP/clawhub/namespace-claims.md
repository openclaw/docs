# 組織と名前空間の申し立て

ClawHub は、所有者ハンドル、組織ハンドル、Skillsスラッグ、Pluginパッケージ名、
パッケージスコープを公開名前空間として使用します。名前空間が現実世界のプロジェクト、
ブランド、パッケージエコシステム、または組織に属しているように見える一方で、ClawHub 上で
すでに申し立て済み、予約済み、誤解を招く、または係争中である場合は、
[組織 / 名前空間の申し立て issue フォーム](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)で
スタッフにレビューを依頼してください。

公開の、機微ではない所有権レビューにはこの経路を使用してください。名前空間の申し立てに、
製品内レポートやアカウント異議申し立てフォームを使用しないでください。

## 申し立てを開くタイミング

現実世界の所有権を理由として、名前空間を予約、移転、名前変更、非表示、隔離、エイリアス化、
またはその他の方法で変更すべきかを ClawHub スタッフがレビューする必要があると考える場合は、
名前空間の申し立てを開いてください。

例:

- GitHub 組織、プロジェクト、会社、またはコミュニティと一致する組織ハンドル
- 対応する ClawHub 所有者の下でのみ公開されるべき `@example-org/*` のようなパッケージスコープ
- プロジェクトになりすましているように見える Skillsスラッグまたは Pluginパッケージ名
- ブランド、商標、プロジェクト名変更、またはパッケージ履歴の係争
- 正当な名前空間所有者を妨げている、削除済み、非アクティブ、または連絡不能な所有者

一覧が所有権の係争を超えて安全でない、悪意がある、または誤解を招く場合は、
関連するモデレーションまたはセキュリティガイダンスにも従ってください。名前空間申し立て
フォームは所有権レビュー用であり、緊急の脆弱性開示用ではありません。

## 提出前に

まず、名前空間と一致する所有者で公開していることを確認してください。
Pluginパッケージの場合、`@example-org/example-plugin` のようなスコープ付き名前は、
対応する `example-org` 所有者として公開する必要があります。

現在の所有者を管理できる場合は、影響を受けるリソースを公開、名前変更、移転、非表示、
または削除して、名前空間を直接修正してください。現在の所有者を管理できない場合、
または係争の解決にスタッフが必要な場合に申し立てを使用してください。

## 含める証拠

公開可能で機微ではない証拠を使用してください。有用な証拠には次のものがあります。

- GitHub 組織、リポジトリ、リリース、またはメンテナー履歴
- 名前空間を明記している公式プロジェクトドキュメント
- ドメインまたは公式メールドメインの証拠
- npm、PyPI、crates.io、またはその他のパッケージレジストリのスコープ管理
- 公開で議論しても安全な商標、ブランド、またはプロジェクト所有権の証拠
- ソースリポジトリ履歴、パッケージ履歴、または公開された名前変更通知
- 係争中の ClawHub 所有者、Skills、Plugin、パッケージ、または issue へのリンク

各リンクが何を証明するのか説明してください。スタッフが非公開の認証情報やシークレットを
必要とせずに関係性を理解できる必要があります。

## 含めてはいけないもの

公開 GitHub issue にシークレットや非公開の証拠を入れないでください。次のものを含めないでください。

- API トークン、署名キー、または認証情報
- DNS チャレンジトークン
- 非公開の法的ファイルまたは契約書
- 個人の身分証明書
- 非公開メール、非公開のセキュリティレポート、または機密の顧客データ

申し立てフォームでは、機微な証拠に非公開のスタッフチャネルが必要かどうかを尋ねます。
機微な資料を公開投稿する代わりに、そのオプションを使用してください。

## 想定される結果

証拠とリスクに応じて、ClawHub スタッフは名前空間の予約、所有権の移転、リソースの名前変更、
既存一覧の非表示または隔離、エイリアスまたはリダイレクトの追加、追加証拠の依頼、
またはリクエストの却下を行う場合があります。

名前空間レビューは、一致するすべての名前が移転されることを保証するものではありません。
スタッフは公開証拠、既存の利用状況、セキュリティリスク、ユーザー影響を考慮します。
