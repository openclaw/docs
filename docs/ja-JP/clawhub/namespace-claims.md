# 組織と名前空間の申請

ClawHub は、所有者ハンドル、組織ハンドル、スキルスラッグ、Plugin パッケージ名、および
パッケージスコープを公開名前空間として使用します。名前空間が
実在するプロジェクト、ブランド、パッケージエコシステム、または組織に属しているように見えるものの、ClawHub 上ですでに
申請済み、予約済み、誤解を招く、または係争中である場合は、
[組織 / 名前空間申請の issue フォーム](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)で
スタッフにレビューを依頼してください。

公開可能で機密性のない所有権レビューには、この手順を使用してください。名前空間の申請に、製品内
レポートやアカウント異議申し立てフォームを使用しないでください。

## 申請を作成するタイミング

実世界の所有権を理由として、名前空間を予約、移転、改名、非表示、隔離、エイリアス化、
またはその他の形で変更すべきかどうかを ClawHub スタッフがレビューする必要があると考える場合は、
名前空間の申請を作成してください。

例:

- GitHub 組織、プロジェクト、会社、またはコミュニティに一致する組織ハンドル
- 一致する ClawHub 所有者の下でのみ公開されるべき `@example-org/*` のようなパッケージスコープ
- プロジェクトになりすましているように見えるスキルスラッグまたは Plugin パッケージ名
- ブランド、商標、プロジェクト名変更、またはパッケージ履歴に関する係争
- 正当な名前空間所有者を妨げている、削除済み、非アクティブ、または連絡不能な所有者

リスティングが所有権の係争を超えて安全でない、悪意がある、または誤解を招く場合は、
関連するモデレーションまたはセキュリティのガイダンスにも従ってください。名前空間申請
フォームは所有権レビュー用であり、緊急の脆弱性開示用ではありません。

## 申請前に

まず、名前空間に一致する所有者で公開していることを確認してください。
Plugin パッケージの場合、`@example-org/example-plugin` のようなスコープ付き名前は、
一致する `example-org` 所有者として公開する必要があります。

現在の所有者を管理できる場合は、影響を受けるリソースを公開、改名、移転、非表示、または削除して、
名前空間を直接修正してください。現在の所有者を管理できない場合、またはスタッフによる係争の解決が必要な場合に
申請を使用してください。

## 含める証拠

公開可能で機密性のない証拠を使用してください。有用な証拠には次のものがあります。

- GitHub 組織、リポジトリ、リリース、またはメンテナー履歴
- 名前空間を明記している公式プロジェクトドキュメント
- ドメインまたは公式メールドメインの証拠
- npm、PyPI、crates.io、またはその他のパッケージレジストリのスコープ管理
- 公開で議論しても安全な商標、ブランド、またはプロジェクト所有権の証拠
- ソースリポジトリ履歴、パッケージ履歴、または公開された改名通知
- 係争中の ClawHub 所有者、スキル、Plugin、パッケージ、または issue へのリンク

各リンクが何を証明するのかを説明してください。スタッフが
非公開の認証情報やシークレットを必要とせずに関係性を理解できる必要があります。

## 含めてはいけないもの

公開 GitHub issue にシークレットや非公開の証拠を記載しないでください。次のものを含めないでください。

- API トークン、署名キー、または認証情報
- DNS チャレンジトークン
- 非公開の法的文書または契約書
- 個人の身分証明書
- 非公開メール、非公開セキュリティレポート、または機密顧客データ

申請フォームでは、機密性のある証拠に非公開のスタッフチャネルが必要かどうかを尋ねます。
機密資料を公開投稿する代わりに、そのオプションを使用してください。

## 考えられる結果

証拠とリスクに応じて、ClawHub スタッフは名前空間の予約、
所有権の移転、リソースの改名、既存リスティングの非表示または隔離、
エイリアスやリダイレクトの追加、追加証拠の要求、またはリクエストの却下を行う場合があります。

名前空間レビューは、一致するすべての名前が移転されることを保証するものではありません。
スタッフは公開証拠、既存の利用状況、セキュリティリスク、およびユーザーへの影響を考慮します。
