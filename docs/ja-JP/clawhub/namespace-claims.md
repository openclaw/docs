# Org と Namespace の申し立て

ClawHub は、所有者ハンドル、Org ハンドル、skill slug、Plugin パッケージ名、パッケージスコープを公開名前空間として使用します。名前空間が現実世界のプロジェクト、ブランド、パッケージエコシステム、組織に属しているように見えるものの、ClawHub ですでに申し立て済み、予約済み、誤解を招く、または係争中である場合は、[Org / Namespace Claim issue form](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml) でスタッフにレビューを依頼してください。

この手順は、公開可能で機密性のない所有権レビューに使用します。名前空間の申し立てには、製品内レポートやアカウント異議申し立てフォームを使用しないでください。

## 申し立てを開くタイミング

現実世界の所有権を理由に、名前空間を予約、譲渡、名前変更、非表示、隔離、エイリアス化、またはその他の形で変更すべきかどうかを ClawHub スタッフがレビューすべきだと考える場合は、名前空間の申し立てを開いてください。

例:

- あなたの GitHub Org、プロジェクト、会社、またはコミュニティと一致する Org ハンドル
- `@example-org/*` のように、一致する ClawHub 所有者の下でのみ公開されるべきパッケージスコープ
- プロジェクトになりすましているように見える skill slug または Plugin パッケージ名
- ブランド、商標、プロジェクト名変更、またはパッケージ履歴に関する係争
- 正当な名前空間所有者を妨げている、削除済み、非アクティブ、または連絡不能な所有者

リスティングが所有権の係争を超えて危険、悪意あるもの、または誤解を招くものである場合は、関連するモデレーションまたはセキュリティガイダンスにも従ってください。名前空間の申し立てフォームは所有権レビュー用であり、緊急の脆弱性開示用ではありません。

## 提出前に

まず、その名前空間に一致する所有者で公開していることを確認してください。Plugin パッケージの場合、`@example-org/example-plugin` のようなスコープ付き名前は、一致する `example-org` 所有者として公開する必要があります。

現在の所有者を管理できる場合は、影響を受けるリソースを公開、名前変更、譲渡、非表示、または削除して、名前空間を直接修正してください。現在の所有者を管理できない場合、またはスタッフが係争を解決する必要がある場合は、申し立てを使用してください。

## 含める証拠

公開可能で機密性のない証拠を使用してください。有用な証拠には次のものがあります。

- GitHub Org、リポジトリ、リリース、またはメンテナー履歴
- 名前空間を明記している公式プロジェクトドキュメント
- ドメインまたは公式メールドメインの証拠
- npm、PyPI、crates.io、またはその他のパッケージレジストリのスコープ管理
- 公開で議論しても安全な商標、ブランド、またはプロジェクト所有権の証拠
- ソースリポジトリ履歴、パッケージ履歴、または公開された名前変更通知
- 係争中の ClawHub 所有者、skill、Plugin、パッケージ、または issue へのリンク

各リンクが何を証明するかを説明してください。スタッフが、非公開の認証情報やシークレットを必要とせずに関係性を理解できるようにしてください。

## 含めてはいけないもの

公開 GitHub issue にシークレットや非公開の証拠を入れないでください。次のものを含めないでください。

- API トークン、署名キー、または認証情報
- DNS チャレンジトークン
- 非公開の法的ファイルまたは契約書
- 個人の身分証明書
- 非公開メール、非公開セキュリティレポート、または機密の顧客データ

申し立てフォームでは、機密証拠に非公開のスタッフチャネルが必要かどうかを尋ねます。機密資料を公開投稿する代わりに、そのオプションを使用してください。

## 想定される結果

証拠とリスクに応じて、ClawHub スタッフは名前空間の予約、所有権の譲渡、リソースの名前変更、既存リスティングの非表示または隔離、エイリアスまたはリダイレクトの追加、追加証拠の依頼、またはリクエストの却下を行う場合があります。

名前空間レビューは、一致するすべての名前が譲渡されることを保証するものではありません。スタッフは、公開証拠、既存の使用状況、セキュリティリスク、ユーザー影響を考慮します。
