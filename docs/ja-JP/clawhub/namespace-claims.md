# 組織と名前空間の申し立て

ClawHub は、所有者ハンドル、組織ハンドル、スキルスラッグ、Plugin パッケージ名、パッケージスコープを公開名前空間として使用します。名前空間が実在のプロジェクト、ブランド、パッケージエコシステム、または組織に属しているように見えるものの、ClawHub 上ですでに申し立て済み、予約済み、誤解を招く、または係争中である場合は、
[組織 / 名前空間の申し立て課題フォーム](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
でスタッフにレビューを依頼してください。

公開の非機密な所有権レビューにはこの手順を使用してください。名前空間の申し立てに、製品内レポートやアカウント異議申し立てフォームを使用しないでください。

## 申し立てを開くタイミング

実世界での所有権を理由として、名前空間を予約、移管、名前変更、非表示、隔離、エイリアス化、またはその他の方法で変更すべきかどうかを ClawHub スタッフがレビューする必要があると考える場合は、名前空間の申し立てを開いてください。

例:

- GitHub 組織、プロジェクト、会社、またはコミュニティと一致する組織ハンドル
- 一致する ClawHub 所有者の下でのみ公開されるべき `@example-org/*` のようなパッケージスコープ
- プロジェクトになりすましているように見えるスキルスラッグまたは Plugin パッケージ名
- ブランド、商標、プロジェクト名変更、またはパッケージ履歴に関する係争
- 正当な名前空間所有者をブロックしている、削除済み、非アクティブ、または到達不能な所有者

掲載内容が所有権の係争を超えて危険、悪意がある、または誤解を招く場合は、関連するモデレーションまたはセキュリティのガイダンスにも従ってください。名前空間の申し立てフォームは所有権レビューのためのものであり、緊急の脆弱性開示のためのものではありません。

## 提出前に

まず、名前空間と一致する所有者で公開していることを確認してください。Plugin パッケージの場合、`@example-org/example-plugin` のようなスコープ付き名は、一致する `example-org` 所有者として公開する必要があります。

現在の所有者を管理できる場合は、影響を受けるリソースを公開、名前変更、移管、非表示、または削除して、名前空間を直接修正してください。現在の所有者を管理できない場合、またはスタッフが係争を解決する必要がある場合に申し立てを使用してください。

## 含める証拠

公開可能で非機密の証拠を使用してください。有用な証明には次のものがあります。

- GitHub 組織、リポジトリ、リリース、またはメンテナー履歴
- 名前空間を明記している公式プロジェクトドキュメント
- ドメインまたは公式メールドメインの証明
- npm、PyPI、crates.io、またはその他のパッケージレジストリのスコープ管理
- 公開の場で議論しても安全な、商標、ブランド、またはプロジェクト所有権の証拠
- ソースリポジトリ履歴、パッケージ履歴、または公開された名前変更通知
- 係争中の ClawHub 所有者、スキル、Plugin、パッケージ、または課題へのリンク

各リンクが何を証明するかを説明してください。スタッフが非公開の認証情報やシークレットを必要とせずに関係性を理解できるようにしてください。

## 含めてはいけないもの

シークレットや非公開の証明を公開 GitHub 課題に記載しないでください。次のものを含めないでください。

- API トークン、署名キー、または認証情報
- DNS チャレンジトークン
- 非公開の法的ファイルまたは契約
- 個人の身分証明書
- 非公開メール、非公開のセキュリティレポート、または機密の顧客データ

申し立てフォームでは、機密性の高い証拠に非公開のスタッフチャネルが必要かどうかを尋ねます。機密資料を公開投稿する代わりに、そのオプションを使用してください。

## 考えられる結果

証拠とリスクに応じて、ClawHub スタッフは名前空間の予約、所有権の移管、リソースの名前変更、既存掲載の非表示または隔離、エイリアスまたはリダイレクトの追加、追加証拠の要求、またはリクエストの却下を行う場合があります。

名前空間レビューは、一致するすべての名前が移管されることを保証するものではありません。スタッフは公開証拠、既存の使用状況、セキュリティリスク、ユーザーへの影響を考慮します。
