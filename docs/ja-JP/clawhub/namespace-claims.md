# 組織と名前空間の申し立て

ClawHub は、所有者ハンドル、組織ハンドル、skill slug、plugin package names、および
package scopes を公開名前空間として使用します。名前空間が実在のプロジェクト、ブランド、パッケージエコシステム、または組織に属しているように見えるものの、ClawHub ですでに申し立て済み、予約済み、誤解を招く、または係争中である場合は、
[組織 / 名前空間の申し立て issue フォーム](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
でスタッフにレビューを依頼してください。

公開できる非機密の所有権レビューには、この経路を使用してください。名前空間の申し立てには、製品内レポートやアカウント異議申し立てフォームを使用しないでください。

## 申し立てを開くタイミング

実世界の所有権を理由に、名前空間を予約、移管、名称変更、非表示、隔離、エイリアス化、またはその他の方法で変更すべきかどうかを ClawHub スタッフがレビューする必要があると考える場合は、名前空間の申し立てを開いてください。

例:

- 自分の GitHub 組織、プロジェクト、会社、またはコミュニティと一致する組織ハンドル
- 対応する ClawHub 所有者の下でのみ公開されるべき `@example-org/*` のような package scope
- プロジェクトになりすましているように見える skill slug または plugin package name
- ブランド、商標、プロジェクト名変更、またはパッケージ履歴に関する紛争
- 正当な名前空間所有者を妨げている、削除済み、非アクティブ、または連絡不能な所有者

リスティングが所有権紛争を超えて危険、悪意がある、または誤解を招く場合は、関連するモデレーションまたはセキュリティのガイダンスにも従ってください。名前空間の申し立てフォームは所有権レビュー用であり、緊急の脆弱性開示用ではありません。

## 提出前に

まず、名前空間と一致する所有者で公開していることを確認してください。Plugin パッケージでは、`@example-org/example-plugin` のようなスコープ付き名前は、一致する `example-org` 所有者として公開する必要があります。

現在の所有者を管理できる場合は、影響を受けるリソースを公開、名称変更、移管、非表示、または削除することで、名前空間を直接修正してください。現在の所有者を管理できない場合、またはスタッフが紛争を解決する必要がある場合に申し立てを使用してください。

## 含める証拠

公開できる非機密の証拠を使用してください。有用な証拠には次のものがあります。

- GitHub 組織、リポジトリ、リリース、またはメンテナー履歴
- 名前空間を明記している公式プロジェクトドキュメント
- ドメインまたは公式メールドメインの証明
- npm、PyPI、crates.io、またはその他のパッケージレジストリでの scope 管理
- 公開で議論しても安全な商標、ブランド、またはプロジェクト所有権の証拠
- ソースリポジトリ履歴、パッケージ履歴、または公開された名称変更通知
- 係争中の ClawHub 所有者、skill、Plugin、パッケージ、または issue へのリンク

各リンクが何を証明するかを説明してください。スタッフが非公開の資格情報や秘密情報を必要とせずに関係を理解できるようにしてください。

## 含めてはいけないもの

公開 GitHub issue に秘密情報や非公開の証拠を入れないでください。次のものを含めないでください。

- API トークン、署名鍵、または認証情報
- DNS チャレンジトークン
- 非公開の法的ファイルまたは契約書
- 個人の本人確認書類
- 非公開メール、非公開セキュリティレポート、または機密の顧客データ

申し立てフォームでは、機密性の高い証拠に非公開のスタッフチャネルが必要かどうかを尋ねます。機密資料を公開投稿する代わりに、そのオプションを使用してください。

## 考えられる結果

証拠とリスクに応じて、ClawHub スタッフは名前空間の予約、所有権の移管、リソースの名称変更、既存リスティングの非表示または隔離、エイリアスまたはリダイレクトの追加、追加証拠の依頼、またはリクエストの却下を行う場合があります。

名前空間レビューは、一致するすべての名前が移管されることを保証するものではありません。スタッフは、公開証拠、既存の使用状況、セキュリティリスク、およびユーザーへの影響を考慮します。
