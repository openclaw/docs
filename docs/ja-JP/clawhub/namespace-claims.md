# 組織と名前空間の申請

ClawHub は、所有者ハンドル、組織ハンドル、スキルスラッグ、プラグインパッケージ名、パッケージスコープを公開名前空間として使用します。名前空間が実在するプロジェクト、ブランド、パッケージエコシステム、または組織に属しているように見えるものの、ClawHub 上ですでに申請済み、予約済み、誤解を招く、または争議中である場合は、[組織 / 名前空間申請の課題フォーム](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)でスタッフにレビューを依頼してください。

公開可能で機密性のない所有権レビューには、この手順を使用します。名前空間の申請に、製品内レポートやアカウント異議申し立てフォームを使用しないでください。

## 申請を開始するタイミング

実世界の所有権を理由として、名前空間を予約、移転、名前変更、非表示、隔離、エイリアス化、またはその他の変更を行うべきかを ClawHub スタッフがレビューすべきだと考える場合は、名前空間申請を開始してください。

例:

- GitHub 組織、プロジェクト、会社、またはコミュニティと一致する組織ハンドル
- 一致する ClawHub 所有者の下でのみ公開されるべき `@example-org/*` のようなパッケージスコープ
- プロジェクトを偽装しているように見えるスキルスラッグまたはプラグインパッケージ名
- ブランド、商標、プロジェクト名変更、またはパッケージ履歴に関する争議
- 正当な名前空間所有者を妨げている、削除済み、非アクティブ、または連絡不能な所有者

掲載内容が所有権争議の範囲を超えて安全でない、悪意がある、または誤解を招く場合は、関連するモデレーションまたはセキュリティのガイダンスにも従ってください。名前空間申請フォームは所有権レビュー用であり、緊急の脆弱性開示用ではありません。

## 提出前に

まず、名前空間と一致する所有者で公開していることを確認してください。プラグインパッケージの場合、`@example-org/example-plugin` のようなスコープ付き名は、一致する `example-org` 所有者として公開する必要があります。

現在の所有者を管理できる場合は、影響を受けるリソースを公開、名前変更、移転、非表示、または削除して、名前空間を直接修正してください。現在の所有者を管理できない場合、またはスタッフが争議を解決する必要がある場合に申請を使用してください。

## 含める証拠

公開可能で機密性のない証拠を使用してください。有用な証明には次のものがあります。

- GitHub 組織、リポジトリ、リリース、またはメンテナー履歴
- 名前空間を明示している公式プロジェクトドキュメント
- ドメインまたは公式メールドメインの証明
- npm、PyPI、crates.io、またはその他のパッケージレジストリのスコープ管理
- 公開で議論しても安全な商標、ブランド、またはプロジェクト所有権の証拠
- ソースリポジトリ履歴、パッケージ履歴、または公開された名前変更通知
- 争議中の ClawHub 所有者、スキル、プラグイン、パッケージ、または課題へのリンク

各リンクが何を証明するかを説明してください。スタッフは、非公開の資格情報やシークレットを必要とせずに関係を理解できる必要があります。

## 含めてはいけないもの

公開 GitHub 課題にシークレットや非公開の証明を載せないでください。次のものを含めないでください。

- API トークン、署名鍵、または資格情報
- DNS チャレンジトークン
- 非公開の法務ファイルまたは契約書
- 個人の身分証明書
- 非公開メール、非公開セキュリティレポート、または機密の顧客データ

申請フォームでは、機密性の高い証拠に非公開のスタッフチャネルが必要かどうかを尋ねます。機密資料を公開投稿する代わりに、そのオプションを使用してください。

## 想定される結果

証拠とリスクに応じて、ClawHub スタッフは名前空間の予約、所有権の移転、リソースの名前変更、既存掲載の非表示または隔離、エイリアスまたはリダイレクトの追加、追加証明の依頼、またはリクエストの却下を行う場合があります。

名前空間レビューは、一致するすべての名前が移転されることを保証するものではありません。スタッフは、公開証拠、既存の利用状況、セキュリティリスク、ユーザーへの影響を考慮します。
