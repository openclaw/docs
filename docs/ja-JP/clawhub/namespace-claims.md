# 組織と名前空間の申請

ClawHub は、所有者ハンドル、組織ハンドル、スキルスラッグ、Plugin パッケージ名、
パッケージスコープを公開名前空間として使用します。名前空間が実在のプロジェクト、
ブランド、パッケージエコシステム、または組織に属しているように見えるものの、
ClawHub 上ですでに申請済み、予約済み、誤解を招く、または係争中の場合は、
[組織 / 名前空間申請 issue フォーム](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
でスタッフにレビューを依頼してください。

公開の非機密な所有権レビューにはこの経路を使用してください。名前空間の申請に、
製品内レポートやアカウント異議申し立てフォームを使用しないでください。

## 申請を開くタイミング

実世界の所有権を理由に、名前空間を予約、移転、改名、非表示、隔離、エイリアス化、
またはその他の形で変更すべきかどうかを ClawHub スタッフがレビューすべきだと考える場合は、
名前空間申請を開いてください。

例:

- 自分の GitHub 組織、プロジェクト、会社、またはコミュニティと一致する組織ハンドル
- 一致する ClawHub 所有者の下でのみ公開されるべき `@example-org/*` のようなパッケージスコープ
- プロジェクトになりすましているように見えるスキルスラッグまたは Plugin パッケージ名
- ブランド、商標、プロジェクト改名、またはパッケージ履歴に関する係争
- 正当な名前空間所有者を妨げている、削除済み、非アクティブ、または連絡不能な所有者

一覧が所有権の係争を超えて安全でない、悪意がある、または誤解を招く場合は、
関連するモデレーションまたはセキュリティのガイダンスにも従ってください。名前空間申請フォームは
所有権レビュー用であり、緊急の脆弱性開示用ではありません。

## 申請前に

まず、名前空間と一致する所有者で公開していることを確認してください。
Plugin パッケージでは、`@example-org/example-plugin` のようなスコープ付き名前は、
一致する `example-org` 所有者として公開する必要があります。

現在の所有者を管理できる場合は、影響を受けるリソースを公開、改名、移転、非表示、
または削除して、名前空間を直接修正してください。現在の所有者を管理できない場合、
またはスタッフが係争を解決する必要がある場合に申請を使用してください。

## 含める証拠

公開の非機密な証拠を使用してください。有用な証拠には次が含まれます。

- GitHub 組織、リポジトリ、リリース、またはメンテナー履歴
- 名前空間を明記している公式プロジェクトドキュメント
- ドメインまたは公式メールドメインの証明
- npm、PyPI、crates.io、またはその他のパッケージレジストリのスコープ管理
- 公開で議論しても安全な商標、ブランド、またはプロジェクト所有権の証拠
- ソースリポジトリ履歴、パッケージ履歴、または公開の改名通知
- 係争対象の ClawHub 所有者、スキル、Plugin、パッケージ、または issue へのリンク

各リンクが何を証明するかを説明してください。スタッフが非公開の認証情報やシークレットを
必要とせずに関係性を理解できる必要があります。

## 含めないもの

公開 GitHub issue にシークレットや非公開の証拠を記載しないでください。次を含めないでください。

- API トークン、署名キー、または認証情報
- DNS チャレンジトークン
- 非公開の法的ファイルまたは契約書
- 個人の身分証明書
- 非公開メール、非公開セキュリティレポート、または機密の顧客データ

申請フォームでは、機密証拠に非公開のスタッフチャネルが必要かどうかを尋ねます。
機密情報を公開投稿する代わりに、その選択肢を使用してください。

## 考えられる結果

証拠とリスクに応じて、ClawHub スタッフは名前空間を予約する、所有権を移転する、
リソースを改名する、既存の一覧を非表示または隔離する、エイリアスまたはリダイレクトを追加する、
追加の証拠を求める、またはリクエストを却下することがあります。

名前空間レビューは、一致するすべての名前が移転されることを保証するものではありません。
スタッフは公開証拠、既存の利用状況、セキュリティリスク、ユーザーへの影響を考慮します。
