# セキュリティ + モデレーション

ClawHub では公開を受け付けていますが、公開リスティングは信頼性、
スキャン、報告、モデレーションの制御を通過します。目的は実用的です。ユーザーが
インストールするものを検査できるようにし、パブリッシャーには誤検知からの回復手段を提供し、
不正なパッケージを公開ディスカバリーから排除します。

[許容される使用](/ja-JP/clawhub/acceptable-usage) も参照してください。

## ユーザーが検査できる内容

Skill または Plugin をインストールする前に、その ClawHub リスティングで次を確認してください。

- オーナーとソースの帰属
- 最新バージョンと変更履歴
- 必要な環境変数または権限
- Plugin の互換性メタデータ
- スキャンまたはモデレーションの状態
- 表示される場合は、報告、コメント、スター、ダウンロード、インストールのシグナル

理解し、信頼できるコンテンツだけをインストールしてください。

## スキャン状態

ClawHub は公開ページとオーナーに表示される診断で、スキャンまたはモデレーションの結果を表示する場合があります。

一般的な結果は次のとおりです。

- `clean`: ブロック対象の問題は見つかりませんでした。
- `suspicious`: リリースには注意またはレビューが必要です。
- `malicious`: リリースは安全ではないと見なされています。
- `pending`: チェックがまだ完了していません。
- `held`、`quarantined`、`revoked`、または `hidden`: リリースは公開インストール面で完全には利用できません。

正確な文言は表示面によって異なる場合がありますが、実用上の意味は同じです。リリースが保留またはブロックされている場合、オーナーが問題を解決するか、モデレーションによって復元されるまで、ユーザーはインストールすべきではありません。

## Skills

Skill スキャンでは、公開された Skill バンドル、メタデータ、宣言された要件、不審な指示を確認します。

ClawHub は、Skill が宣言している内容と実際に行っているように見える内容の不一致に特に注意します。たとえば、必須の API キーを参照する Skill は、その要件を `SKILL.md` で宣言し、ユーザーがインストール前に確認できるようにする必要があります。

スキャン結果はアーティファクトに基づきます。宣言された API 認証情報、localhost OAuth コールバック、スコープ付きアンインストールクリーンアップ、Basic Auth エンコード、または明示されたプロバイダーへのユーザー選択ファイルアップロードなど、想定されるプロバイダー動作は、隠れた認証情報転送、広範な非公開ファイルアクセス、無関係なネットワーク宛先、または隠密なブラウザー悪用とは異なる扱いになります。

[Skill 形式](/ja-JP/clawhub/skill-format) を参照してください。

## Plugin

Plugin リリースには、パッケージメタデータ、ソースの帰属、互換性フィールド、アーティファクト整合性情報が含まれます。

OpenClaw は、ClawHub でホストされている Plugin をインストールする前に互換性を確認します。パッケージレコードは、OpenClaw がダウンロードしたアーティファクトを検証できるように、ダイジェストメタデータも公開する場合があります。ClawScan は Plugin リリースをレビューする際、宣言されたパッケージ `openclaw.environment` の env/config メタデータを含め、宣言された実行時要件を観測された動作と比較します。

## 報告

サインイン済みのユーザーは、Skills、パッケージ、コメントを報告できます。

報告は具体的で対応可能な内容にしてください。報告の悪用自体がアカウント措置につながる場合があります。

報告例:

- 誤解を招くメタデータ
- 宣言されていない認証情報または権限の要件
- 不審なインストール手順
- 詐欺コメントまたはなりすまし
- 悪意ある登録または商標の不正使用
- [許容される使用](/ja-JP/clawhub/acceptable-usage) に違反するコンテンツ

## パブリッシャーの ClawScan メモ

パブリッシャーは、Skill または Plugin を公開するときに任意の ClawScan メモを提供できます。このメモは、ネットワークアクセス、ネイティブホストアクセス、プロバイダー固有の認証情報など、通常とは異なって見える可能性のある動作について、ClawScan にコンテキストを提供します。

## モデレーション保留

静的スキャナーがアップロードされた Skill を malicious としてフラグ付けすると、パブリッシャーは自動的にモデレーション保留の対象になります（ユーザーに `requiresModerationAt` が設定されます）。これにより、パブリッシャーのすべての Skills が非表示になり、今後の公開は非表示で開始され、`user.moderation.auto` 監査ログエントリが作成されます。

静的な suspicious 検出結果は、モデレーター向けのファイル/行の証拠として保持されますが、それ単体ではコンテンツを非表示にしたり、公開スキャン判定を決定したりしません。新しいアップロードは、LLM レビューが確定するまでレビュー中/pending 状態のままです。静的スキャンは malicious シグネチャの場合のみ即時にブロックします。VirusTotal エンジンのヒットは表示可能なセキュリティ証拠として残りますが、VirusTotal Code Insight/Palm の判定は参考情報であり、それ単体で Skills を非表示にはしません。ClawScan LLM レビューでは、目的に沿ったメモをガイダンスとして保持します。中程度のレビュー結果はアーティファクト上で表示されたままになり、suspicious フィルターは影響の大きい LLM 上の懸念、malicious の検出、または AV エンジン検出による裏付けがある場合に限定されます。

管理者は誤検知による保留を解除できます。

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

これにより、`requiresModerationAt` と `requiresModerationReason` がクリアされ、ユーザーレベルの保留によって非表示にされた Skills が復元され、`user.moderation.lift` 監査ログエントリが書き込まれます。他の理由で非表示になっている Skills、またはその静的スキャン自体が引き続き malicious の Skills は、非表示のままです。

## 禁止措置とアカウントの状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な悪用は、アカウントの禁止、トークンの失効、コンテンツの非表示、またはリスティングの削除につながる可能性があります。

削除、禁止、または無効化されたアカウントは ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。サインインまたは通常の CLI アクセスがブロックされている場合は、回復レビューのために security@openclaw.ai に連絡してください。

## パブリッシャー向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- リリースに通常とは異なるが意図した動作がある場合は、パブリッシャー ClawScan メモを追加する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- Plugin を公開する前にドライランを使用する
- ユーザーまたはモデレーターがパッケージ動作について質問した場合は、明確に回答する
