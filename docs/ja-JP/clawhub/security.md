# セキュリティ + モデレーション

ClawHub は公開に開かれていますが、公開掲載は引き続き信頼性、
スキャン、報告、モデレーションの管理を通過します。目的は実用的です。ユーザーが
インストールするものを検査できるようにし、誤検知に対して公開者に回復手段を提供し、
不正なパッケージを公開検索から排除します。

[許容される利用](/ja-JP/clawhub/acceptable-usage)も参照してください。

## ユーザーが検査できる内容

Skill または Plugin をインストールする前に、ClawHub の掲載情報で次を確認してください。

- 所有者とソースの帰属
- 最新バージョンと変更履歴
- 必要な環境変数または権限
- Plugin の互換性メタデータ
- スキャンまたはモデレーションの状態
- 表示される場合は、報告、コメント、スター、ダウンロード、インストールシグナル

理解して信頼できるコンテンツだけをインストールしてください。

## スキャン状態

ClawHub は、公開ページと所有者に表示される診断に、スキャンまたはモデレーションの結果を表示することがあります。

一般的な結果は次のとおりです。

- `clean`: ブロック対象の問題は見つかっていません。
- `suspicious`: リリースには注意またはレビューが必要です。
- `malicious`: リリースは安全でないと判断されています。
- `pending`: チェックはまだ完了していません。
- `held`、`quarantined`、`revoked`、または `hidden`: リリースは公開インストール面で完全には
  利用できません。

正確な表現は表示面によって異なる場合がありますが、実用上の意味は同じです。リリースが保留またはブロックされている場合、所有者が問題を解決するか、モデレーションによって復元されるまで、ユーザーはそれをインストールすべきではありません。

## Skills

Skill のスキャンでは、公開された Skill バンドル、メタデータ、宣言された要件、不審な指示を確認します。

ClawHub は、Skill が宣言している内容と実際に行っているように見える内容の不一致に特に注意します。たとえば、必須の API キーを参照する Skill は、その要件を `SKILL.md` で宣言し、ユーザーがインストール前に確認できるようにする必要があります。

スキャン結果はアーティファクトに基づきます。宣言された API 認証情報、localhost OAuth コールバック、スコープ付きアンインストールクリーンアップ、Basic Auth エンコード、または指定されたプロバイダーへのユーザー選択ファイルアップロードなど、想定されるプロバイダー動作は、隠れた認証情報転送、広範なプライベートファイルアクセス、無関係なネットワーク宛先、またはステルス的なブラウザー悪用とは異なる扱いになります。

[Skill 形式](/ja-JP/clawhub/skill-format)を参照してください。

## Plugins

Plugin リリースには、パッケージメタデータ、ソースの帰属、互換性フィールド、アーティファクト整合性情報が含まれます。

OpenClaw は、ClawHub でホストされる Plugin をインストールする前に互換性を確認します。パッケージレコードは、OpenClaw がダウンロードしたアーティファクトを検証できるように digest メタデータを公開することもあります。ClawScan は、Plugin リリースをレビューするときに宣言済みパッケージ `openclaw.environment` の env/config メタデータを含めるため、宣言されたランタイム要件が観測された動作と比較されます。

## 報告

サインイン済みユーザーは、Skills、パッケージ、コメントを報告できます。

報告は具体的で対応可能なものにする必要があります。報告の悪用自体がアカウント措置につながる場合があります。

報告の例:

- 誤解を招くメタデータ
- 宣言されていない認証情報または権限要件
- 不審なインストール手順
- 詐欺コメントまたはなりすまし
- 悪意のある登録または商標の悪用
- [許容される利用](/ja-JP/clawhub/acceptable-usage)に違反するコンテンツ

## 公開者向け ClawScan ノート

公開者は、Skill または Plugin を公開するときに任意の ClawScan ノートを提供できます。このノートは、ネットワークアクセス、ネイティブホストアクセス、プロバイダー固有の認証情報など、通常とは異なって見える可能性のある動作について ClawScan に文脈を与えます。

## モデレーション保留

静的スキャナーがアップロードされた Skill を悪意ありと判定した場合、公開者は自動的にモデレーション保留の対象になります（ユーザーに `requiresModerationAt` が設定されます）。これにより公開者のすべての Skills が非表示になり、以後の公開は非表示状態で開始され、`user.moderation.auto` 監査ログエントリが作成されます。

静的な不審判定はモデレーター向けのファイル/行証拠として保持されますが、それだけでコンテンツを非表示にしたり、公開スキャン判定を決定したりすることはありません。新しいアップロードは、LLM レビューが確定するまでレビュー中/保留状態のままです。静的スキャンが即時にブロックするのは悪意あるシグネチャのみです。VirusTotal エンジンのヒットは表示可能なセキュリティ証拠として残りますが、VirusTotal Code Insight/Palm の判定は助言扱いであり、それだけで Skills を非表示にすることはありません。ClawScan LLM レビューは、目的に沿ったノートをガイダンスとして保持します。中程度のレビュー結果はアーティファクト上に表示されたままですが、不審フィルターは影響の大きい LLM の懸念、悪意ある判定、または裏付けのある AV エンジン検出に限定されます。

管理者は誤検知の保留を解除できます。

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

これにより `requiresModerationAt` と `requiresModerationReason` がクリアされ、ユーザーレベルの保留によって非表示になっていた Skills が復元され、`user.moderation.lift` 監査ログエントリが書き込まれます。別の理由で非表示になっている Skills、またはその静的スキャン自体が悪意ありのままである Skills は、非表示のままです。

## BAN とアカウント状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失うことがあります。重大な悪用は、アカウントの BAN、トークンの取り消し、コンテンツの非表示、または掲載削除につながる場合があります。

削除、BAN、または無効化されたアカウントは ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。サインインまたは通常の CLI アクセスがブロックされている場合は、回復レビューのために security@openclaw.ai に連絡してください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、要約、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- リリースに通常とは異なるが意図された動作がある場合は、公開者 ClawScan ノートを追加する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- Plugin を公開する前にドライランを使用する
- ユーザーまたはモデレーターがパッケージの動作について質問した場合は、明確に回答する
