# セキュリティ + モデレーション

ClawHub は公開に対して開かれていますが、公開リスティングは引き続き信頼、
スキャン、報告、モデレーションの管理を通過します。目的は実践的です。ユーザーが
インストールするものを確認できるようにし、公開者には誤検知からの回復経路を提供し、
悪用目的のパッケージを公開検索から除外します。

[許容される使用](/ja-JP/clawhub/acceptable-usage)も参照してください。

## ユーザーが確認できること

skill または plugin をインストールする前に、その ClawHub リスティングで次を確認してください。

- 所有者とソースの帰属
- 最新バージョンと変更履歴
- 必要な環境変数または権限
- plugins の互換性メタデータ
- スキャンまたはモデレーションの状態
- 表示されている場合は、報告、コメント、スター、ダウンロード、インストール信号

理解し信頼できるコンテンツのみをインストールしてください。

## スキャン状態

ClawHub は、公開ページと所有者に見える診断に、スキャンまたはモデレーションの結果を表示する場合があります。

一般的な結果は次のとおりです。

- `clean`: ブロック対象の問題は見つかりませんでした。
- `suspicious`: そのリリースには注意またはレビューが必要です。
- `malicious`: そのリリースは安全でないと見なされています。
- `pending`: チェックがまだ完了していません。
- `held`、`quarantined`、`revoked`、または `hidden`: そのリリースは公開インストール画面で完全には利用できません。

正確な文言は画面によって異なる場合がありますが、実用上の意味は同じです。リリースが保留またはブロックされている場合、所有者が問題を解決するか、モデレーションによって復元されるまで、ユーザーはインストールしないでください。

## Skills

Skill スキャンでは、公開された skill バンドル、メタデータ、宣言された要件、不審な指示を確認します。

ClawHub は、skill が宣言している内容と実際に行っているように見える内容の不一致に特に注意します。たとえば、必要な API キーに言及している skill は、ユーザーがインストール前に確認できるように、その要件を `SKILL.md` で宣言する必要があります。

スキャン結果はアーティファクトに基づきます。宣言された API 認証情報、localhost OAuth コールバック、スコープ付きのアンインストール時クリーンアップ、Basic Auth エンコード、明示されたプロバイダーへのユーザー選択ファイルアップロードなど、想定されるプロバイダー動作は、隠れた認証情報転送、広範なプライベートファイルアクセス、無関係なネットワーク宛先、またはステルス的なブラウザー悪用とは異なる扱いになります。

[Skill 形式](/ja-JP/clawhub/skill-format)を参照してください。

## Plugins

Plugin リリースには、パッケージメタデータ、ソースの帰属、互換性フィールド、アーティファクト整合性情報が含まれます。

OpenClaw は、ClawHub がホストする plugins をインストールする前に互換性を確認します。パッケージレコードでは、OpenClaw がダウンロード済みアーティファクトを検証できるように、ダイジェストメタデータも公開される場合があります。ClawScan は plugin リリースをレビューする際、宣言されたランタイム要件を観測された動作と比較できるように、宣言済みパッケージの `openclaw.environment` 環境変数/設定メタデータを含めます。

## 報告

サインインしたユーザーは、skills、パッケージ、コメントを報告できます。

報告は具体的で対応可能な内容にしてください。報告の悪用自体がアカウント措置につながる場合があります。

報告の例:

- 誤解を招くメタデータ
- 宣言されていない認証情報または権限の要件
- 不審なインストール手順
- 詐欺的なコメントまたはなりすまし
- 悪意ある登録または商標の不正使用
- [許容される使用](/ja-JP/clawhub/acceptable-usage)に違反するコンテンツ

## 公開者向け ClawScan 注記

公開者は、skill または plugin の公開時に任意の ClawScan 注記を提供できます。この注記は、ネットワークアクセス、ネイティブホストアクセス、プロバイダー固有の認証情報など、通常とは異なって見える可能性のある動作について、ClawScan にコンテキストを提供します。

## モデレーション保留

静的スキャナーがアップロードされた skill を悪意あるものとしてフラグ付けした場合、公開者は自動的にモデレーション保留（ユーザーに `requiresModerationAt` が設定される）に置かれます。これにより、その公開者のすべての skills が非表示になり、今後の公開は非表示状態で開始され、`user.moderation.auto` 監査ログエントリが作成されます。

静的な不審判定は、モデレーター向けのファイル/行の証拠として保持されますが、それ自体ではコンテンツを非表示にしたり、公開スキャン判定を決定したりしません。新しいアップロードは、LLM レビューが確定するまでレビュー中/pending 状態のままです。静的スキャンが即時にブロックするのは、悪意あるシグネチャの場合のみです。VirusTotal エンジンのヒットは引き続き可視のセキュリティ証拠として扱われますが、VirusTotal Code Insight/Palm の判定は助言的なものであり、それ自体では skills を非表示にしません。ClawScan LLM レビューは、目的に沿った注記をガイダンスとして保持します。中程度のレビュー結果はアーティファクト上で引き続き表示されますが、不審フィルターは、影響度の高い LLM の懸念、悪意ある判定、または AV エンジン検出によって裏付けられたものに限定されます。

管理者は誤検知による保留を解除できます。

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

これにより、`requiresModerationAt` と `requiresModerationReason` がクリアされ、ユーザーレベルの保留によって非表示になっていた skills が復元され、`user.moderation.lift` 監査ログエントリが書き込まれます。別の理由で非表示になっている skills、または自身の静的スキャンが引き続き悪意ある状態の skills は、非表示のままです。

## 禁止措置とアカウント状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な悪用は、アカウントの禁止、トークンの取り消し、コンテンツの非表示、またはリスティングの削除につながる可能性があります。

削除、禁止、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置の後に CLI 認証が失敗し始めた場合は、web UI にサインインしてアカウント状態を確認してください。サインインまたは通常の CLI アクセスがブロックされている場合は、復旧レビューのために security@openclaw.ai へ連絡してください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには、次を行ってください。

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- リリースに通常とは異なるが意図的な動作がある場合は、公開者 ClawScan 注記を追加する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- plugins を公開する前にドライランを使用する
- ユーザーまたはモデレーターからパッケージ動作について質問された場合は、明確に回答する
