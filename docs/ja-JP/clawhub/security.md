# セキュリティ + モデレーション

ClawHub は公開を受け付けていますが、公開リスティングは引き続き信頼、
スキャン、報告、モデレーションの管理を通過します。目的は実用的です。ユーザーが
インストールするものを検査できるようにし、公開者に誤検知からの復旧経路を提供し、
悪用的なパッケージを公開の検索対象から排除します。

[許容される使用](/ja-JP/clawhub/acceptable-usage) も参照してください。

## ユーザーが検査できる内容

Skill または plugin をインストールする前に、ClawHub のリスティングで次を確認してください。

- 所有者とソースの帰属
- 最新バージョンと変更履歴
- 必要な環境変数または権限
- plugins の互換性メタデータ
- スキャンまたはモデレーションのステータス
- 表示されている場合は、報告、コメント、スター、ダウンロード、インストールのシグナル

理解し信頼できるコンテンツのみをインストールしてください。

## スキャン状態

ClawHub は公開ページや所有者が確認できる診断に、スキャンまたはモデレーションの結果を表示する場合があります。

一般的な結果には次のものがあります。

- `clean`: ブロック対象の問題は見つかりませんでした。
- `suspicious`: リリースには注意またはレビューが必要です。
- `malicious`: リリースは安全ではないと見なされます。
- `pending`: チェックがまだ完了していません。
- `held`、`quarantined`、`revoked`、または `hidden`: リリースは公開インストール画面で完全には
  利用できません。

正確な文言は画面によって異なる場合がありますが、実際の意味は同じです。リリースが
保留またはブロックされている場合、所有者が問題を解決するかモデレーションが復旧するまで、
ユーザーはそれをインストールすべきではありません。

## Skills

Skill のスキャンでは、公開された skill バンドル、メタデータ、宣言された
要件、疑わしい指示を確認します。

ClawHub は、skill が宣言している内容と実際に行っているように見える内容の不一致に特に注意します。
たとえば、必須の API キーを参照する skill は、その要件を `SKILL.md` で宣言し、
ユーザーがインストール前に確認できるようにする必要があります。

スキャン結果は成果物に基づきます。宣言された
API 認証情報、localhost OAuth コールバック、スコープ付きアンインストールのクリーンアップ、Basic Auth
エンコード、または明示されたプロバイダーへのユーザー選択ファイルアップロードなど、想定される
プロバイダー動作は、隠れた認証情報転送、広範なプライベートファイルアクセス、
無関係なネットワーク宛先、またはステルスなブラウザー悪用とは異なる扱いになります。

[Skill 形式](/ja-JP/clawhub/skill-format) を参照してください。

## Plugins

Plugin リリースには、パッケージメタデータ、ソースの帰属、互換性
フィールド、成果物の整合性情報が含まれます。

OpenClaw は、ClawHub がホストする plugins をインストールする前に互換性を確認します。パッケージ
レコードは、OpenClaw がダウンロード済み成果物を検証できるように、ダイジェストメタデータを公開する場合もあります。
ClawScan は plugin リリースのレビュー時に、宣言されたパッケージ `openclaw.environment` env/config
メタデータを含め、宣言された実行時要件を
観測された動作と比較します。

## 報告

サインイン済みのユーザーは、skills、パッケージ、コメントを報告できます。

報告は具体的で対応可能な内容にしてください。報告の悪用自体が
アカウント措置につながる場合があります。

報告の例:

- 誤解を招くメタデータ
- 宣言されていない認証情報または権限要件
- 疑わしいインストール手順
- 詐欺コメントまたはなりすまし
- 悪意ある登録または商標の悪用
- [許容される使用](/ja-JP/clawhub/acceptable-usage) に違反するコンテンツ

## 公開者向け ClawScan メモ

公開者は、skill または plugin の公開時に任意の ClawScan メモを提供できます。
このメモは、ネットワークアクセス、ネイティブホストアクセス、プロバイダー固有の
認証情報など、通常は異常に見える可能性がある動作について ClawScan に文脈を提供します。

## モデレーション保留

静的スキャナーがアップロードされた skill を悪意ありとしてフラグした場合、公開者は
自動的にモデレーション保留の対象になります（ユーザーに `requiresModerationAt` が設定されます）。
これにより公開者のすべての skills が非表示になり、今後の公開は非表示で開始され、
`user.moderation.auto` 監査ログエントリが作成されます。

静的な疑わしい検出結果は、モデレーター向けのファイル/行の証拠として保持されますが、
それだけでコンテンツを非表示にしたり公開スキャン判定を決定したりすることはありません。
新しいアップロードは、LLM レビューが確定するまで review/pending 状態のままです。静的
スキャンは、悪意あるシグネチャの場合にのみ即時ブロックします。VirusTotal エンジンの
ヒットは表示されるセキュリティ証拠として残りますが、VirusTotal Code Insight/Palm
判定は参考情報であり、それだけで skills を非表示にすることはありません。ClawScan LLM レビューは、
目的に沿ったメモをガイダンスとして保持します。中程度のレビュー結果は成果物上で表示されたままになり、
一方で suspicious フィルターは影響の大きい LLM
懸念、悪意ある検出結果、または裏付けのある AV エンジン検出に限定されます。

管理者は誤検知の保留を解除できます。

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

これにより `requiresModerationAt` と `requiresModerationReason` がクリアされ、
ユーザーレベルの保留によって非表示になった skills が復元され、`user.moderation.lift` 監査
ログエントリが書き込まれます。他の理由で非表示になっている skills、または自身の静的スキャンが引き続き
悪意ありの skills は、非表示のままです。

## BAN とアカウント状態

ClawHub ポリシーに違反するアカウントは、公開アクセスを失う場合があります。重大な悪用は、
アカウント BAN、トークン失効、コンテンツの非表示、またはリスティング削除につながる可能性があります。

削除済み、BAN 済み、または無効化されたアカウントは ClawHub API トークンを使用できません。アカウント措置の後に CLI 認証が
失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。
サインインまたは通常の CLI アクセスがブロックされている場合は、復旧レビューのため
security@openclaw.ai に連絡してください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- リリースに通常とは異なるが意図的な動作がある場合は、公開者 ClawScan メモを追加する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- plugins を公開する前にドライランを使用する
- ユーザーまたはモデレーターからパッケージ動作について質問された場合は明確に回答する
