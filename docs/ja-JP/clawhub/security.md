# セキュリティ + モデレーション

ClawHub は公開に開かれていますが、公開リスティングは信頼、
スキャン、報告、モデレーションの制御を通過します。目的は実用的です。ユーザーが
インストールするものを確認できるようにし、公開者には誤検知からの回復経路を提供し、
悪用パッケージを公開ディスカバリーから排除します。

[許容される使用](/ja-JP/clawhub/acceptable-usage) も参照してください。

## ユーザーが確認できるもの

Skill または plugin をインストールする前に、ClawHub リスティングで次を確認してください。

- 所有者とソースの帰属
- 最新バージョンと変更履歴
- 必要な環境変数または権限
- plugin の互換性メタデータ
- スキャンまたはモデレーションの状態
- 表示されている場合は、報告、コメント、スター、ダウンロード、インストールシグナル

理解し信頼できるコンテンツだけをインストールしてください。

## スキャン状態

ClawHub は公開ページおよび所有者に表示される診断情報で、スキャンまたはモデレーションの結果を表示する場合があります。

一般的な結果には次が含まれます。

- `clean`: ブロック対象の問題は見つかりませんでした。
- `suspicious`: リリースには注意またはレビューが必要です。
- `malicious`: リリースは安全でないと見なされています。
- `pending`: チェックがまだ完了していません。
- `held`、`quarantined`、`revoked`、または `hidden`: リリースは公開インストール面で完全には利用できません。

正確な文言は面によって異なる場合がありますが、実用上の意味は同じです。リリースが保留またはブロックされている場合、所有者が問題を解決するか、モデレーションが復元するまで、ユーザーはインストールすべきではありません。

## Skills

Skill スキャンは、公開された skill バンドル、メタデータ、宣言された要件、不審な指示を確認します。

ClawHub は、skill が宣言している内容と実際に行っているように見える内容の不一致に特に注意します。たとえば、必要な API キーを参照する skill は、その要件を `SKILL.md` で宣言し、ユーザーがインストール前に確認できるようにする必要があります。

スキャン結果はアーティファクトに基づきます。宣言された API 認証情報、localhost OAuth コールバック、スコープを限定したアンインストール時のクリーンアップ、Basic Auth エンコード、または記載されたプロバイダーへのユーザー選択ファイルのアップロードなど、想定されるプロバイダー動作は、隠れた認証情報転送、広範なプライベートファイルアクセス、無関係なネットワーク宛先、またはステルス的なブラウザー悪用とは異なる扱いになります。

[Skill 形式](/ja-JP/clawhub/skill-format) を参照してください。

## Plugins

Plugin リリースには、パッケージメタデータ、ソースの帰属、互換性フィールド、アーティファクト整合性情報が含まれます。

OpenClaw は ClawHub でホストされる plugins をインストールする前に互換性を確認します。パッケージレコードは digest メタデータも公開する場合があり、OpenClaw はダウンロードされたアーティファクトを検証できます。ClawScan は plugin リリースをレビューする際に、宣言されたパッケージの `openclaw.environment` env/config メタデータを含めるため、宣言されたランタイム要件を観測された動作と比較できます。

## 報告

サインインしたユーザーは、skills、パッケージ、コメントを報告できます。

報告は具体的で対応可能な内容にしてください。報告の悪用自体がアカウント措置につながる場合があります。

報告の例:

- 誤解を招くメタデータ
- 未宣言の認証情報または権限要件
- 不審なインストール手順
- 詐欺コメントまたはなりすまし
- 悪意ある登録または商標の悪用
- [許容される使用](/ja-JP/clawhub/acceptable-usage) に違反するコンテンツ

## 公開者向け ClawScan ノート

公開者は、skill または plugin を公開する際に、任意の ClawScan ノートを提供できます。このノートは、ネットワークアクセス、ネイティブホストアクセス、プロバイダー固有の認証情報など、通常とは異なって見える可能性のある動作について ClawScan に文脈を提供します。

## モデレーション保留

静的スキャナーがアップロードされた skill を悪意ありとしてフラグ付けすると、公開者は自動的にモデレーション保留下に置かれます（ユーザーに `requiresModerationAt` が設定されます）。これにより、その公開者のすべての skills が非表示になり、以後の公開は非表示で開始され、`user.moderation.auto` 監査ログエントリが作成されます。

静的な不審検出はモデレーター向けのファイル/行の証拠として保持されますが、それ自体ではコンテンツを非表示にしたり、公開スキャン判定を決定したりしません。新しいアップロードは、LLM レビューが確定するまでレビュー中/pending 状態のままです。静的スキャンが即座にブロックするのは悪意あるシグネチャの場合のみです。VirusTotal エンジンのヒットは表示可能なセキュリティ証拠として残りますが、VirusTotal Code Insight/Palm の判定は参考情報であり、それ自体では skills を非表示にしません。ClawScan LLM レビューは、目的に沿ったノートをガイダンスとして保持します。中程度のレビュー結果はアーティファクト上に表示されたままになり、不審フィルターは影響の大きい LLM の懸念、悪意ある検出、または裏付けのある AV エンジン検出に限定されます。

管理者は誤検知による保留を解除できます。

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

これにより `requiresModerationAt` と `requiresModerationReason` がクリアされ、ユーザーレベルの保留により非表示になっていた skills が復元され、`user.moderation.lift` 監査ログエントリが書き込まれます。他の理由で非表示になっている skills、またはその静的スキャン自体が引き続き悪意ありの skills は、非表示のままです。

## 禁止措置とアカウント状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な悪用は、アカウントの禁止、トークンの失効、コンテンツの非表示、またはリスティングの削除につながる可能性があります。

削除、禁止、または無効化されたアカウントは ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。サインインまたは通常の CLI アクセスがブロックされている場合は、復旧レビューのため security@openclaw.ai に連絡してください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- リリースに通常とは異なるが意図された動作がある場合は、公開者 ClawScan ノートを追加する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- plugins を公開する前にドライランを使用する
- ユーザーまたはモデレーターがパッケージ動作について質問した場合は明確に回答する
