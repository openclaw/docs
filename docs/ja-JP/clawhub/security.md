# セキュリティ + モデレーション

ClawHub では公開が可能ですが、公開リストには信頼性、スキャン、報告、モデレーションの制御が適用されます。目的は実用的です。ユーザーがインストールするものを確認できるようにし、誤検知に対して公開者に復旧手段を提供し、不正なパッケージを公開ディスカバリーから排除することです。

関連項目: [許容される利用](/ja-JP/clawhub/acceptable-usage)。

## ユーザーが確認できるもの

Skills またはPluginをインストールする前に、ClawHub のリストで以下を確認してください。

- 所有者とソースの帰属
- 最新バージョンと変更履歴
- 必要な環境変数または権限
- Pluginの互換性メタデータ
- スキャンまたはモデレーションの状態
- 表示される場合は、報告、コメント、スター、ダウンロード数、インストールシグナル

理解し信頼できるコンテンツのみをインストールしてください。

## スキャン状態

ClawHub は公開ページおよび所有者に表示される診断で、スキャンまたはモデレーションの結果を表示することがあります。

一般的な結果は次のとおりです。

- `clean`: ブロック対象の問題は見つかりませんでした。
- `suspicious`: このリリースには注意またはレビューが必要です。
- `malicious`: このリリースは安全ではないと判断されています。
- `pending`: チェックはまだ完了していません。
- `held`、`quarantined`、`revoked`、または `hidden`: このリリースは公開インストール面で完全には利用できません。

正確な文言は画面によって異なる場合がありますが、実用上の意味は同じです。リリースが保留またはブロックされている場合、所有者が問題を解決するか、モデレーションによって復元されるまで、ユーザーはインストールすべきではありません。

## Skills

Skills のスキャンでは、公開されたSkillsバンドル、メタデータ、宣言された要件、不審な指示を確認します。

ClawHub は、Skills が宣言している内容と実際に行っているように見える内容との不一致に特に注意します。たとえば、必要な API キーを参照するSkillsは、その要件を `SKILL.md` で宣言し、ユーザーがインストール前に確認できるようにする必要があります。

スキャンの検出結果はアーティファクトに基づきます。宣言された API 認証情報、localhost OAuth コールバック、スコープ付きのアンインストール時クリーンアップ、Basic Auth エンコーディング、または明記されたプロバイダーへのユーザー選択ファイルアップロードなど、想定されるプロバイダーの動作は、隠れた認証情報の転送、広範なプライベートファイルアクセス、無関係なネットワーク宛先、またはステルス的なブラウザー悪用とは異なる扱いになります。

[Skills 形式](/ja-JP/clawhub/skill-format)を参照してください。

## Plugin

Pluginリリースには、パッケージメタデータ、ソースの帰属、互換性フィールド、アーティファクトの整合性情報が含まれます。

OpenClaw は、ClawHub でホストされているPluginをインストールする前に互換性を確認します。パッケージレコードでは、OpenClaw がダウンロード済みアーティファクトを検証できるようにダイジェストメタデータも公開される場合があります。ClawScan は、Pluginリリースをレビューする際に、宣言されたパッケージの `openclaw.environment` env/config メタデータを含め、宣言されたランタイム要件を観測された動作と比較します。

## 報告

サインイン済みのユーザーは、Skills、パッケージ、コメントを報告できます。

報告は具体的で対応可能なものである必要があります。報告機能の悪用は、それ自体がアカウント措置につながる場合があります。

報告の例:

- 誤解を招くメタデータ
- 宣言されていない認証情報または権限の要件
- 不審なインストール指示
- 詐欺的なコメントまたはなりすまし
- 悪意ある登録または商標の不正使用
- [許容される利用](/ja-JP/clawhub/acceptable-usage)に違反するコンテンツ

## 公開者向け ClawScan ノート

公開者は、Skills またはPluginを公開する際に任意の ClawScan ノートを提供できます。このノートは、ネットワークアクセス、ネイティブホストアクセス、プロバイダー固有の認証情報など、通常とは異なって見える可能性のある動作について、ClawScan にコンテキストを提供します。

## モデレーション保留

静的スキャナーがアップロードされたSkillsを悪意があると判定した場合、公開者は自動的にモデレーション保留の対象になります（ユーザーに `requiresModerationAt` が設定されます）。これにより公開者のすべてのSkillsが非表示になり、今後の公開は非表示状態で開始され、`user.moderation.auto` 監査ログエントリが作成されます。

静的な不審検出はモデレーター向けのファイル/行証拠として保持されますが、それだけでコンテンツを非表示にしたり、公開スキャン判定を決定したりすることはありません。新しいアップロードは、LLM レビューが確定するまでレビュー中/保留状態のままです。静的スキャンが即時ブロックするのは、悪意あるシグネチャの場合のみです。VirusTotal エンジンのヒットは表示可能なセキュリティ証拠として残りますが、VirusTotal Code Insight/Palm の判定は助言的なものであり、それだけでSkillsを非表示にすることはありません。ClawScan LLM レビューは、目的に沿ったノートをガイダンスとして保持します。中程度のレビュー検出結果はアーティファクト上に表示されたままですが、不審フィルターは影響の大きい LLM の懸念、悪意ある検出結果、または AV エンジンによって裏付けられた検出に限定されます。

管理者は誤検知による保留を解除できます。

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

これにより `requiresModerationAt` と `requiresModerationReason` がクリアされ、ユーザーレベルの保留によって非表示になっていたSkillsが復元され、`user.moderation.lift` 監査ログエントリが書き込まれます。他の理由で非表示になっているSkills、または自身の静的スキャンが引き続き悪意ありであるSkillsは、非表示のままです。

## BAN とアカウント状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な悪用は、アカウントBAN、トークン失効、コンテンツ非表示、またはリスト削除につながる可能性があります。

削除、BAN、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。サインインまたは通常の CLI アクセスがブロックされている場合は、復旧レビューのために security@openclaw.ai に連絡してください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには、次のようにしてください。

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- リリースに通常とは異なるが意図された動作がある場合は、公開者 ClawScan ノートを追加する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- Pluginを公開する前にドライランを使用する
- ユーザーまたはモデレーターからパッケージ動作について質問された場合は明確に回答する
