# セキュリティ + モデレーション

ClawHub は公開を受け付けていますが、公開リストは引き続き信頼性、
スキャン、報告、モデレーションの制御を通過します。目的は実用的です。ユーザーが
インストールするものを調査できるようにし、誤検知に対する復旧手段を公開者に提供し、
不正なパッケージを公開ディスカバリーから排除します。

[許容される利用](/ja-JP/clawhub/acceptable-usage)も参照してください。

## ユーザーが確認できること

Skill または Plugin をインストールする前に、その ClawHub リストで次を確認してください。

- 所有者とソースの帰属
- 最新バージョンと変更履歴
- 必要な環境変数または権限
- Plugin の互換性メタデータ
- スキャンまたはモデレーションのステータス
- 表示されている場合は、報告、コメント、スター、ダウンロード数、インストールシグナル

理解し信頼できるコンテンツだけをインストールしてください。

## スキャン状態

ClawHub は公開ページや所有者に表示される診断に、スキャンまたはモデレーションの結果を表示することがあります。

一般的な結果には次が含まれます。

- `clean`: ブロック対象の問題は見つかりませんでした。
- `suspicious`: リリースには注意またはレビューが必要です。
- `malicious`: リリースは安全でないと見なされています。
- `pending`: チェックはまだ完了していません。
- `held`、`quarantined`、`revoked`、または `hidden`: リリースは公開インストール面で完全には
  利用できません。

正確な文言は表示面によって異なる場合がありますが、実用上の意味は同じです。リリースが保留またはブロックされている場合、所有者が問題を解決するかモデレーションが復元するまで、ユーザーはそれをインストールすべきではありません。

## Skills

Skill スキャンでは、公開された Skill バンドル、メタデータ、宣言された
要件、不審な指示を確認します。

ClawHub は、Skill が宣言している内容と実際に行っているように見える内容の不一致に特に注意します。たとえば、必要な API キーを参照する Skill は、
ユーザーがインストール前に確認できるように、その要件を `SKILL.md` で宣言すべきです。

スキャン所見はアーティファクトベースです。宣言された
API 認証情報、localhost OAuth コールバック、スコープ付きアンインストールクリーンアップ、Basic Auth
エンコーディング、またはユーザーが選択して指定プロバイダーへアップロードするファイルなど、想定されるプロバイダーの動作は、隠れた認証情報転送、広範なプライベートファイルアクセス、
無関係なネットワーク宛先、または隠密なブラウザー悪用とは別に扱われます。

[Skill 形式](/ja-JP/clawhub/skill-format)を参照してください。

## Plugin

Plugin リリースには、パッケージメタデータ、ソースの帰属、互換性
フィールド、アーティファクト整合性情報が含まれます。

OpenClaw は、ClawHub でホストされる Plugin をインストールする前に互換性を確認します。パッケージ
レコードは、OpenClaw がダウンロード済み
アーティファクトを検証できるよう、ダイジェストメタデータを公開する場合もあります。ClawScan は Plugin リリースをレビューするときに、宣言されたパッケージ `openclaw.environment` env/config
メタデータを含めるため、宣言されたランタイム要件を
観測された動作と比較できます。

## 報告

サインイン済みユーザーは、Skills、パッケージ、コメントを報告できます。

報告は具体的で対応可能な内容にしてください。報告の悪用自体も
アカウント措置につながる可能性があります。

報告の例:

- 誤解を招くメタデータ
- 宣言されていない認証情報または権限要件
- 不審なインストール手順
- 詐欺コメントまたはなりすまし
- 悪意ある登録または商標の不正使用
- [許容される利用](/ja-JP/clawhub/acceptable-usage)に違反するコンテンツ

## 公開者の ClawScan ノート

公開者は Skill または
Plugin を公開するときに、任意で ClawScan ノートを提供できます。このノートは、ネットワークアクセス、ネイティブホストアクセス、プロバイダー固有の
認証情報など、通常とは異なって見える可能性のある動作について、ClawScan に文脈を提供します。

## モデレーション保留

静的スキャナーがアップロードされた Skill を悪意ありとしてフラグ付けすると、公開者は
自動的にモデレーション保留になります（ユーザーに `requiresModerationAt` が設定されます）。これにより公開者のすべての Skills が非表示になり、以後の公開は
非表示の状態で開始され、`user.moderation.auto` 監査ログエントリが作成されます。

静的な不審所見は、モデレーター向けのファイル/行エビデンスとして保持されますが、
それだけでコンテンツを非表示にしたり公開スキャン評決を決定したりすることはありません。
新規アップロードは、LLM レビューが確定するまでレビュー中/保留中の状態に残ります。静的
スキャンは、悪意あるシグネチャに対してのみ即時ブロックします。VirusTotal エンジンの
ヒットは目に見えるセキュリティエビデンスとして残りますが、VirusTotal Code Insight/Palm
の評決は助言扱いであり、それだけで Skills を非表示にすることはありません。ClawScan LLM レビューは、
目的に沿ったノートをガイダンスとして保持します。中程度のレビュー所見はアーティファクト上に表示されたままですが、不審フィルターは影響度の高い LLM
の懸念、悪意ある所見、または裏付けのある AV エンジン検出のために予約されています。

管理者は誤検知の保留を解除できます。

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

これにより `requiresModerationAt` と `requiresModerationReason` がクリアされ、
ユーザーレベルの保留によって非表示にされた Skills が復元され、`user.moderation.lift` 監査
ログエントリが書き込まれます。他の理由で非表示になっている Skills、または自身の静的スキャンが引き続き
悪意ありである Skills は非表示のままです。

## 禁止措置とアカウント状態

ClawHub ポリシーに違反したアカウントは公開アクセスを失う場合があります。重大な悪用は、
アカウントの禁止、トークン失効、コンテンツの非表示、またはリスト削除につながる可能性があります。

削除、禁止、または無効化されたアカウントは ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント
状態を確認してください。サインインまたは通常の CLI アクセスがブロックされている場合は、
復旧レビューのために security@openclaw.ai に連絡してください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- リリースに通常とは異なるが意図された動作がある場合は、公開者 ClawScan ノートを追加する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- Plugin を公開する前にドライランを使用する
- ユーザーまたはモデレーターがパッケージの動作について質問した場合は明確に回答する
