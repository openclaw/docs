# セキュリティ + モデレーション

ClawHub は公開に開かれていますが、公開リストは引き続き信頼、
スキャン、報告、モデレーション制御を通過します。目的は実用的です。ユーザーが
インストールするものを検査できるようにし、公開者には誤検知からの回復手段を提供し、
悪用的なパッケージを公開ディスカバリーから排除します。

[許容される利用](/ja-JP/clawhub/acceptable-usage) も参照してください。

## ユーザーが検査できるもの

skill または plugin をインストールする前に、その ClawHub リストで次を確認してください。

- 所有者とソースの帰属表示
- 最新バージョンと変更履歴
- 必要な環境変数または権限
- plugins の互換性メタデータ
- スキャンまたはモデレーションの状態
- 表示されている場合は、報告、コメント、スター、ダウンロード、インストールシグナル

理解し信頼できるコンテンツのみをインストールしてください。

## スキャン状態

ClawHub は、公開ページや所有者に表示される診断にスキャンまたはモデレーションの結果を表示する場合があります。

一般的な結果には次が含まれます。

- `clean`: ブロック対象の問題は見つかりませんでした。
- `suspicious`: このリリースには注意またはレビューが必要です。
- `malicious`: このリリースは安全ではないと見なされています。
- `pending`: チェックはまだ完了していません。
- `held`、`quarantined`、`revoked`、または `hidden`: このリリースは公開インストール面で完全には利用できません。

正確な文言は面によって異なる場合がありますが、実用上の意味は同じです。リリースが保留またはブロックされている場合、所有者が問題を解決するか、モデレーションが復元するまで、ユーザーはそれをインストールすべきではありません。

## Skills

Skill スキャンは、公開された skill バンドル、メタデータ、宣言された要件、疑わしい指示を確認します。

ClawHub は、skill が宣言している内容と実際に行っているように見える内容の不一致に特に注意します。たとえば、必須の API キーを参照する skill は、その要件を `SKILL.md` で宣言し、ユーザーがインストール前に確認できるようにする必要があります。

スキャン結果はアーティファクトに基づきます。宣言された API 認証情報、localhost OAuth コールバック、スコープ付きアンインストールクリーンアップ、Basic Auth エンコード、または明示されたプロバイダーへのユーザー選択ファイルアップロードなど、想定されるプロバイダー動作は、隠れた認証情報転送、広範なプライベートファイルアクセス、無関係なネットワーク宛先、またはステルスブラウザー悪用とは異なる扱いになります。

[Skill 形式](/ja-JP/clawhub/skill-format) を参照してください。

## Plugins

Plugin リリースには、パッケージメタデータ、ソースの帰属表示、互換性フィールド、アーティファクト整合性情報が含まれます。

OpenClaw は ClawHub ホストの plugins をインストールする前に互換性をチェックします。パッケージレコードは、OpenClaw がダウンロードしたアーティファクトを検証できるようにダイジェストメタデータを公開する場合もあります。ClawScan は plugin リリースをレビューする際に、宣言されたパッケージ `openclaw.environment` の env/config メタデータを含めるため、宣言されたランタイム要件が観測された動作と比較されます。

## 報告

サインイン済みのユーザーは、skills、パッケージ、コメントを報告できます。

報告は具体的で実行可能である必要があります。報告の悪用自体がアカウント措置につながる場合があります。

報告例:

- 誤解を招くメタデータ
- 宣言されていない認証情報または権限要件
- 疑わしいインストール手順
- 詐欺コメントまたはなりすまし
- 悪意ある登録または商標の不正使用
- [許容される利用](/ja-JP/clawhub/acceptable-usage) に違反するコンテンツ

## 悪意ある報告または商標報告

ClawHub は、悪意ある登録、なりすまし、商標関連の紛争に同じ報告およびスタッフモデレーションのパイプラインを使用します。これらの報告には、スタッフが申立人、争われているリスト、要求された措置を特定できる十分な文脈が必要です。

含めるもの:

- 正規の ClawHub skill またはパッケージ URL と所有者ハンドル
- 問題となっている商標、プロジェクト、会社、または製品名
- 申立人の所有権または権限を示す公開証拠
- 現在の所有者がその名前で公開する権限を持たない理由
- レビュー待ちの非表示、所有権の移転、名称変更、または削除など、要求する措置

公開報告にプライベートな秘密情報や機密性の高い法的文書を含めないでください。必要な場合は、機密ではない証拠を添えて GitHub issue を開き、非公開の引き継ぎ手段についてメンテナーに問い合わせてください。

## 異議申し立てと再スキャン

所有者は、skill またはパッケージが誤って保留またはフラグ付けされたと考える場合、再スキャンを要求できます。プラットフォームのモデレーターと管理者は、報告またはサポートリクエストの対応中に、任意の skill またはパッケージの再スキャンを要求できます。

```bash
clawhub skill rescan <slug>
clawhub package rescan <name>
```

モデレーション対象のコンテンツについては、所有者に表示される ClawHub 面から異議申し立てを提出できる場合があります。異議申し立てでは、何が変更されたのか、またはフラグが誤っている理由を説明する必要があります。

## モデレーション保留

静的スキャナーがアップロードされた skill を悪意あるものとしてフラグ付けすると、公開者は自動的にモデレーション保留状態になります（ユーザーに `requiresModerationAt` が設定されます）。これにより、公開者のすべての skills が非表示になり、以降の公開は非表示で開始され、`user.moderation.auto` 監査ログエントリが作成されます。

静的な疑わしい結果は、モデレーター向けのファイル/行の証拠として保持されますが、それだけでコンテンツを非表示にしたり、公開スキャン判定を決定したりすることはありません。新しいアップロードは、LLM レビューが確定するまでレビュー/保留状態のままです。静的スキャンが即時ブロックするのは、悪意あるシグネチャのみです。VirusTotal エンジンのヒットは表示可能なセキュリティ証拠として残りますが、VirusTotal Code Insight/Palm の判定は参考情報であり、それだけで skills を非表示にすることはありません。ClawScan LLM レビューでは、目的に沿ったメモをガイダンスとして保持します。中程度のレビュー結果はアーティファクト上に表示されたままになりますが、疑わしいフィルターは、影響の大きい LLM の懸念、悪意ある結果、または裏付けのある AV エンジン検出のために予約されています。

管理者は誤検知の保留を解除できます。

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

これにより `requiresModerationAt` と `requiresModerationReason` がクリアされ、ユーザーレベルの保留によって非表示になっていた skills が復元され、`user.moderation.lift` 監査ログエントリが書き込まれます。他の理由で非表示になっている skills、またはそれ自体の静的スキャンが引き続き悪意ありと判定される skills は、非表示のままです。

## 禁止措置とアカウント状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な悪用は、アカウント禁止、トークン失効、コンテンツ非表示、またはリスト削除につながる場合があります。

削除済み、禁止済み、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認するか、想定されるプロジェクトサポートチャネルを通じてメンテナーに連絡してください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- plugins を公開する前にドライランを使用する
- ユーザーまたはモデレーターからパッケージ動作について質問された場合は明確に応答する
