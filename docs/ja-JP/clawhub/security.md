# セキュリティ + モデレーション

ClawHub は公開に開かれていますが、公開リスティングは信頼、
スキャン、報告、モデレーションの制御を通過します。目的は実用的です。ユーザーが
インストールするものを確認できるようにし、公開者には誤検知からの復旧手段を提供し、
不正なパッケージを公開ディスカバリーから排除します。

[利用許容ポリシー](/ja-JP/clawhub/acceptable-usage)も参照してください。

## ユーザーが確認できること

スキルまたはPluginをインストールする前に、その ClawHub リスティングで次を確認してください。

- オーナーとソースの帰属
- 最新バージョンと変更履歴
- 必須の環境変数または権限
- Pluginの互換性メタデータ
- スキャンまたはモデレーションの状態
- 表示されている場合は、報告、コメント、スター、ダウンロード、インストールシグナル

理解し信頼できるコンテンツだけをインストールしてください。

## スキャン状態

ClawHub は公開ページやオーナーに表示される診断で、スキャンまたはモデレーションの結果を表示する場合があります。

一般的な結果は次のとおりです。

- `clean`: ブロック対象の問題は見つかりませんでした。
- `suspicious`: このリリースには注意またはレビューが必要です。
- `malicious`: このリリースは安全ではないと見なされています。
- `pending`: チェックがまだ完了していません。
- `held`、`quarantined`、`revoked`、または `hidden`: このリリースは公開インストール面で完全には利用できません。

正確な文言は画面によって異なる場合がありますが、実用上の意味は同じです。リリースが保留またはブロックされている場合、オーナーが問題を解決するか、モデレーションが復元するまで、ユーザーはそれをインストールすべきではありません。

## Skills

スキルのスキャンでは、公開されたスキルバンドル、メタデータ、宣言された要件、不審な指示を確認します。

ClawHub は、スキルが宣言している内容と、実際に行っているように見える内容の不一致に特に注意します。たとえば、必須の API キーを参照するスキルは、その要件を `SKILL.md` で宣言し、ユーザーがインストール前に確認できるようにする必要があります。

スキャン結果はアーティファクトベースです。宣言された API 認証情報、localhost OAuth コールバック、スコープ化されたアンインストール時のクリーンアップ、Basic Auth エンコーディング、または明示されたプロバイダーへのユーザー選択ファイルアップロードなど、想定されるプロバイダー動作は、隠れた認証情報転送、広範なプライベートファイルアクセス、無関係なネットワーク宛先、またはステルスなブラウザー悪用とは異なる扱いになります。

[スキル形式](/ja-JP/clawhub/skill-format)を参照してください。

## Plugins

Pluginリリースには、パッケージメタデータ、ソースの帰属、互換性フィールド、アーティファクト整合性情報が含まれます。

OpenClaw は、ClawHub でホストされているPluginをインストールする前に互換性を確認します。パッケージレコードは、OpenClaw がダウンロードされたアーティファクトを検証できるように、ダイジェストメタデータも公開する場合があります。ClawScan は、Pluginリリースをレビューする際に宣言済みのパッケージ `openclaw.environment` env/config メタデータを含めるため、宣言されたランタイム要件が観測された動作と比較されます。

## 報告

サインイン済みユーザーは、スキル、パッケージ、コメントを報告できます。

報告は具体的で対応可能な内容にしてください。報告の悪用自体もアカウント措置につながる可能性があります。

報告例:

- 誤解を招くメタデータ
- 未宣言の認証情報または権限要件
- 不審なインストール手順
- 詐欺コメントまたはなりすまし
- 悪意ある登録または商標の不正使用
- [利用許容ポリシー](/ja-JP/clawhub/acceptable-usage)に違反するコンテンツ

## 公開者の ClawScan ノート

公開者は、スキルまたはPluginを公開する際に、任意の ClawScan ノートを提供できます。このノートは、ネットワークアクセス、ネイティブホストアクセス、プロバイダー固有の認証情報など、通常とは異なって見える可能性のある動作について、ClawScan に文脈を提供します。

## モデレーション保留

静的スキャナーがアップロードされたスキルを悪意のあるものとしてフラグ付けすると、公開者は自動的にモデレーション保留状態になります（ユーザーに `requiresModerationAt` が設定されます）。これにより、公開者のすべてのスキルが非表示になり、以後の公開は非表示で開始され、`user.moderation.auto` 監査ログエントリが作成されます。

静的な不審検出は、モデレーター向けのファイル/行の証拠として保持されますが、それだけでコンテンツを非表示にしたり、公開スキャン判定を決定したりすることはありません。新しいアップロードは、LLM レビューが確定するまでレビュー中/保留状態のままです。静的スキャンが即時にブロックするのは、悪意のあるシグネチャの場合のみです。VirusTotal エンジンのヒットは表示可能なセキュリティ証拠として残りますが、VirusTotal Code Insight/Palm の判定は助言扱いであり、それだけでスキルを非表示にすることはありません。ClawScan LLM レビューは、目的に合ったノートをガイダンスとして保持します。中程度のレビュー結果はアーティファクト上に表示されたままですが、不審フィルターは影響の大きい LLM 上の懸念、悪意ある検出、または裏付けのある AVエンジン検出に限定されます。

管理者は誤検知による保留を解除できます。

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

これにより `requiresModerationAt` と `requiresModerationReason` がクリアされ、ユーザーレベルの保留によって非表示にされたスキルが復元され、`user.moderation.lift` 監査ログエントリが書き込まれます。他の理由で非表示になっているスキル、またはスキル自身の静的スキャンが引き続き悪意ありであるスキルは、非表示のままです。

## 禁止措置とアカウント状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な不正利用は、アカウント禁止、トークン失効、コンテンツ非表示、またはリスティング削除につながる可能性があります。

削除、禁止、または無効化されたアカウントは ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。サインインまたは通常の CLI アクセスがブロックされている場合は、復旧レビューのため security@openclaw.ai に連絡してください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必須の環境変数と権限を宣言する
- リリースに通常とは異なるが意図された動作がある場合は、公開者 ClawScan ノートを追加する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースへリンクする
- Pluginを公開する前にドライランを使用する
- ユーザーまたはモデレーターがパッケージの動作について質問した場合は、明確に回答する
