# セキュリティ + モデレーション

ClawHub は公開に開かれていますが、公開リスティングは引き続き信頼、
スキャン、報告、モデレーションの制御を通過します。目的は実用的です。ユーザーが
インストールするものを検査できるようにし、パブリッシャーに誤検知からの復旧経路を提供し、
悪用的なパッケージを公開ディスカバリーから排除します。

関連項目: [許容される利用](/ja-JP/clawhub/acceptable-usage)。

## ユーザーが検査できるもの

スキルまたはPluginをインストールする前に、その ClawHub リスティングで次を確認してください。

- 所有者とソースの帰属
- 最新バージョンと変更履歴
- 必要な環境変数または権限
- Pluginの互換性メタデータ
- スキャンまたはモデレーションの状態
- 表示されている場合は、報告、コメント、スター、ダウンロード、インストールのシグナル

理解し信頼できるコンテンツのみをインストールしてください。

## スキャン状態

ClawHub は公開ページと所有者に表示される診断情報に、スキャンまたはモデレーションの結果を表示することがあります。

一般的な結果は次のとおりです。

- `clean`: ブロック対象の問題は見つかりませんでした。
- `suspicious`: このリリースには注意またはレビューが必要です。
- `malicious`: このリリースは安全ではないと見なされます。
- `pending`: チェックはまだ完了していません。
- `held`、`quarantined`、`revoked`、または `hidden`: このリリースは公開インストール画面で完全には利用できません。

正確な文言は画面によって異なる場合がありますが、実用上の意味は同じです。リリースが保留またはブロックされている場合、所有者が問題を解決するかモデレーションによって復元されるまで、ユーザーはそれをインストールすべきではありません。

## Skills

スキルのスキャンでは、公開されたスキルバンドル、メタデータ、宣言された要件、不審な指示を確認します。

ClawHub は、スキルが宣言している内容と実際に行っているように見える内容の不一致に特に注意します。たとえば、必須の API キーを参照するスキルは、その要件を `SKILL.md` で宣言し、ユーザーがインストール前に確認できるようにする必要があります。

スキャン結果はアーティファクトに基づきます。宣言された API 認証情報、localhost OAuth コールバック、スコープされたアンインストール時のクリーンアップ、Basic Auth エンコーディング、または明示されたプロバイダーへのユーザー選択ファイルアップロードなど、想定されるプロバイダーの動作は、隠れた認証情報の転送、広範なプライベートファイルアクセス、無関係なネットワーク宛先、またはステルス的なブラウザー悪用とは異なるものとして扱われます。

[スキル形式](/ja-JP/clawhub/skill-format)を参照してください。

## Plugin

Pluginリリースには、パッケージメタデータ、ソースの帰属、互換性フィールド、アーティファクト整合性情報が含まれます。

OpenClaw は ClawHub でホストされるPluginをインストールする前に互換性を確認します。パッケージレコードはダイジェストメタデータも公開する場合があり、OpenClaw はそれを使ってダウンロード済みアーティファクトを検証できます。ClawScan はPluginリリースのレビュー時に、宣言されたパッケージの `openclaw.environment` 環境変数/設定メタデータを含めるため、宣言されたランタイム要件が観測された動作と比較されます。

## 報告

サインイン済みユーザーは、スキル、パッケージ、コメントを報告できます。

報告は具体的で対応可能な内容にしてください。報告機能の悪用自体がアカウント措置につながる可能性があります。

報告の例:

- 誤解を招くメタデータ
- 宣言されていない認証情報または権限要件
- 不審なインストール手順
- 詐欺的なコメントまたはなりすまし
- 悪意ある登録または商標の不正利用
- [許容される利用](/ja-JP/clawhub/acceptable-usage)に違反するコンテンツ

## 悪意ある登録または商標に関する報告

ClawHub は、悪意ある登録、なりすまし、商標関連の紛争に対して同じ報告およびスタッフモデレーションのパイプラインを使用します。これらの報告には、スタッフが申立人、係争中のリスティング、要求された措置を特定できるだけの十分なコンテキストが必要です。

含める内容:

- 正規の ClawHub スキルまたはパッケージ URL と所有者ハンドル
- 問題となっている商標、プロジェクト、会社、または製品名
- 申立人の所有権または権限を示す公開証拠
- 現在の所有者がその名前で公開する権限を持たない理由
- レビュー完了まで非表示、所有権の移転、名前変更、削除など、要求する措置

非公開のシークレットや機微な法的文書を公開報告に含めないでください。必要な場合は、機微でない証拠を添えて GitHub issue を開き、非公開の引き継ぎ経路をメンテナーに依頼してください。

## 異議申し立てと再スキャン

所有者は、スキルまたはパッケージが誤って保留またはフラグ付けされたと考える場合、再スキャンを要求できます。プラットフォームのモデレーターと管理者は、報告またはサポートリクエストへの対応中に、任意のスキルまたはパッケージについて再スキャンを要求できます。

```bash
clawhub skill rescan <slug>
clawhub package rescan <name>
```

モデレーション対象のコンテンツについては、所有者に表示される ClawHub 画面から異議申し立てを送信できる場合があります。異議申し立てでは、変更点またはフラグが誤っている理由を説明してください。

## モデレーション保留

静的スキャナーがアップロードされたスキルを悪意ありとフラグ付けすると、パブリッシャーは自動的にモデレーション保留の対象になります（ユーザーに `requiresModerationAt` が設定されます）。これにより、そのパブリッシャーのすべてのスキルが非表示になり、今後の公開は非表示で開始され、`user.moderation.auto` 監査ログエントリが作成されます。

静的な不審判定はモデレーター向けのファイル/行の証拠として保持されますが、それだけでコンテンツを非表示にしたり、公開スキャン判定を決定したりすることはありません。新しいアップロードは、VirusTotal と LLM レビューが確定するまでレビュー中/保留状態のままです。静的スキャンが即時にブロックするのは悪意あるシグネチャのみです。ClawScan LLM レビューは目的に沿ったメモをガイダンスとして保持します。構造化レビューに重大な懸念が含まれる場合にのみ、Review/suspicious 判定を返します。

管理者は誤検知の保留を解除できます。

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

これにより `requiresModerationAt` と `requiresModerationReason` がクリアされ、ユーザーレベルの保留によって非表示になっていたスキルが復元され、`user.moderation.lift` 監査ログエントリが書き込まれます。他の理由で非表示になっているスキル、またはそのスキル自身の静的スキャンが引き続き悪意ありのスキルは、非表示のままです。

## 禁止措置とアカウント状態

ClawHub ポリシーに違反するアカウントは、公開アクセスを失う可能性があります。重大な悪用は、アカウント禁止、トークン失効、コンテンツの非表示、またはリスティング削除につながる可能性があります。

削除済み、禁止済み、または無効化済みのアカウントは、ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認するか、想定されるプロジェクトサポートチャネルを通じてメンテナーに連絡してください。

## パブリッシャー向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- Pluginを公開する前にドライランを使用する
- ユーザーまたはモデレーターからパッケージ動作について質問された場合は明確に回答する
