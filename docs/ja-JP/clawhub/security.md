# セキュリティ + モデレーション

ClawHub は公開に対して開かれていますが、公開掲載は信頼、
スキャン、報告、モデレーションの管理を引き続き通過します。目的は実践的です。ユーザーが
インストールするものを確認できるようにし、公開者に誤検知からの回復経路を提供し、
不正なパッケージを公開ディスカバリーから排除します。

[利用規定](/ja-JP/clawhub/acceptable-usage)も参照してください。

## ユーザーが確認できること

Skill または Plugin をインストールする前に、その ClawHub 掲載で次を確認してください。

- オーナーとソースの帰属
- 最新バージョンと変更履歴
- 必要な環境変数または権限
- Plugin の互換性メタデータ
- スキャンまたはモデレーションの状態
- 表示される場合は、報告、コメント、スター、ダウンロード、インストールシグナル

理解し信頼できるコンテンツのみをインストールしてください。

## スキャン状態

ClawHub は、公開ページおよびオーナーに表示される診断で、スキャンまたはモデレーションの結果を表示する場合があります。

一般的な結果には次が含まれます。

- `clean`: ブロック対象の問題は見つかりませんでした。
- `suspicious`: このリリースには注意またはレビューが必要です。
- `malicious`: このリリースは安全でないと見なされています。
- `pending`: チェックはまだ完了していません。
- `held`、`quarantined`、`revoked`、または `hidden`: このリリースは公開インストール面で完全には利用できません。

正確な表現は表示面によって異なる場合がありますが、実用上の意味は同じです。リリースが保留またはブロックされている場合、オーナーが問題を解決するか、モデレーションによって復元されるまで、ユーザーはそれをインストールすべきではありません。

## Skills

Skill スキャンは、公開された Skill バンドル、メタデータ、宣言された要件、不審な指示を確認します。

ClawHub は、Skill が宣言している内容と、実際に行っているように見える内容の不一致に特に注意します。たとえば、必要な API キーを参照する Skill は、その要件を `SKILL.md` で宣言し、ユーザーがインストール前に確認できるようにする必要があります。

スキャン結果は成果物に基づきます。宣言された API 認証情報、localhost OAuth コールバック、スコープ指定されたアンインストール時のクリーンアップ、Basic Auth エンコード、または明示されたプロバイダーへのユーザー選択ファイルアップロードなどの想定されるプロバイダー動作は、隠れた認証情報転送、広範なプライベートファイルアクセス、無関係なネットワーク送信先、またはステルス的なブラウザー悪用とは異なる扱いになります。

[Skill 形式](/ja-JP/clawhub/skill-format)を参照してください。

## Plugin

Plugin リリースには、パッケージメタデータ、ソースの帰属、互換性フィールド、成果物の整合性情報が含まれます。

OpenClaw は、ClawHub でホストされる Plugin をインストールする前に互換性を確認します。パッケージレコードは、OpenClaw がダウンロードされた成果物を検証できるようにダイジェストメタデータを公開する場合もあります。ClawScan は、Plugin リリースをレビューする際に、宣言されたランタイム要件を観測された動作と比較できるよう、宣言済みのパッケージ `openclaw.environment` env/config メタデータを含めます。

## 報告

サインイン済みユーザーは、Skills、パッケージ、コメントを報告できます。

報告は具体的で対応可能な内容にしてください。報告の悪用は、それ自体がアカウント措置につながる場合があります。

報告例:

- 誤解を招くメタデータ
- 未宣言の認証情報または権限要件
- 不審なインストール手順
- 詐欺コメントまたはなりすまし
- 悪意ある登録または商標の不正使用
- [利用規定](/ja-JP/clawhub/acceptable-usage)に違反するコンテンツ

## 公開者向け ClawScan ノート

公開者は、Skill または Plugin を公開するときに任意の ClawScan ノートを提供できます。このノートは、ネットワークアクセス、ネイティブホストアクセス、プロバイダー固有の認証情報など、通常とは異なって見える可能性のある動作について ClawScan に文脈を与えます。

## モデレーション保留

静的スキャナーがアップロードされた Skill を悪意ありと判定した場合、公開者は自動的にモデレーション保留の対象になります（ユーザーに `requiresModerationAt` が設定されます）。これにより、その公開者のすべての Skills が非表示になり、今後の公開は非表示の状態で開始され、`user.moderation.auto` 監査ログエントリが作成されます。

静的な不審判定は、モデレーター向けのファイル/行単位の証拠として保持されますが、それだけでコンテンツを非表示にしたり、公開スキャンの判定を決定したりすることはありません。新しいアップロードは、LLM レビューが確定するまでレビュー中/保留状態のままになります。静的スキャンが即座にブロックするのは、悪意のあるシグネチャの場合のみです。VirusTotal エンジンのヒットは可視のセキュリティ証拠として残りますが、VirusTotal Code Insight/Palm の判定は参考情報であり、それだけで Skills を非表示にすることはありません。ClawScan LLM レビューでは、目的に沿ったノートをガイダンスとして保持します。中程度のレビュー結果は成果物上に表示され続けますが、不審フィルターは、影響の大きい LLM の懸念、悪意ありの判定、または裏付けのある AV エンジン検出に限定されます。

管理者は誤検知による保留を解除できます。

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

これにより、`requiresModerationAt` と `requiresModerationReason` がクリアされ、ユーザーレベルの保留によって非表示にされた Skills が復元され、`user.moderation.lift` 監査ログエントリが書き込まれます。他の理由で非表示になっている Skills、またはその Skill 自身の静的スキャンが引き続き悪意ありである場合は、非表示のままです。

## BAN とアカウント状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な不正利用は、アカウント BAN、トークン失効、コンテンツの非表示、または掲載削除につながる場合があります。

削除、BAN、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。サインインまたは通常の CLI アクセスがブロックされている場合は、復旧レビューのため security@openclaw.ai に連絡してください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を向上させるには:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必要な環境変数と権限を宣言する
- リリースに通常とは異なるが意図した動作がある場合は、公開者 ClawScan ノートを追加する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- Plugin を公開する前にドライランを使用する
- ユーザーまたはモデレーターからパッケージの動作について尋ねられた場合は、明確に回答する
