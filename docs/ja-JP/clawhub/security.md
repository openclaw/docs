# セキュリティ + モデレーション

ClawHub は公開を受け付けていますが、公開リストは引き続き信頼、スキャン、報告、モデレーションの管理を通過します。目的は実用的です。ユーザーがインストールするものを検査できるようにし、誤検知に対して公開者に回復手段を提供し、不正なパッケージを公開ディスカバリーから排除します。

[許容される利用](/ja-JP/clawhub/acceptable-usage) も参照してください。

## ユーザーが検査できること

Skills またはプラグインをインストールする前に、その ClawHub リストで以下を確認してください。

- 所有者とソースの帰属
- 最新バージョンと変更履歴
- 必須の環境変数または権限
- プラグインの互換性メタデータ
- スキャンまたはモデレーションの状態
- 表示されている場合は、報告、コメント、スター、ダウンロード、インストールシグナル

理解して信頼できるコンテンツだけをインストールしてください。

## スキャン状態

ClawHub は公開ページや所有者に表示される診断で、スキャンまたはモデレーションの結果を表示する場合があります。

一般的な結果には以下があります。

- `clean`: ブロック対象の問題は見つかりませんでした。
- `suspicious`: リリースには注意またはレビューが必要です。
- `malicious`: リリースは安全ではないと判断されています。
- `pending`: チェックはまだ完了していません。
- `held`、`quarantined`、`revoked`、または `hidden`: リリースは公開インストール面で完全には利用できません。

正確な文言は表示面によって異なる場合がありますが、実用上の意味は同じです。リリースが保留またはブロックされている場合、所有者が問題を解決するかモデレーションが復元するまで、ユーザーはインストールすべきではありません。

## Skills

Skills のスキャンでは、公開された Skills バンドル、メタデータ、宣言された要件、不審な指示を確認します。

ClawHub は、Skills が宣言している内容と実際に行っているように見える内容の不一致に特に注意します。たとえば、必須の API キーを参照する Skills は、その要件を `SKILL.md` で宣言し、ユーザーがインストール前に確認できるようにする必要があります。

スキャンの検出結果はアーティファクトに基づきます。宣言済み API 認証情報、localhost OAuth コールバック、スコープ付きアンインストール時クリーンアップ、Basic Auth エンコード、または指定されたプロバイダーへのユーザー選択ファイルアップロードといった、想定されるプロバイダーの動作は、隠れた認証情報の転送、広範なプライベートファイルアクセス、無関係なネットワーク宛先、またはステルス的なブラウザー悪用とは異なる扱いになります。

[Skills 形式](/ja-JP/clawhub/skill-format) を参照してください。

## プラグイン

プラグインリリースには、パッケージメタデータ、ソースの帰属、互換性フィールド、アーティファクト整合性情報が含まれます。

OpenClaw は、ClawHub でホストされているプラグインをインストールする前に互換性をチェックします。パッケージレコードは、OpenClaw がダウンロードしたアーティファクトを検証できるよう、ダイジェストメタデータも公開する場合があります。ClawScan は、プラグインリリースをレビューする際に、宣言されたパッケージの `openclaw.environment` env/config メタデータを含めるため、宣言されたランタイム要件が観測された動作と比較されます。

## 報告

サインイン済みユーザーは、Skills、パッケージ、コメントを報告できます。

報告は具体的で、対応可能な内容にしてください。報告機能の悪用自体がアカウント措置につながる場合があります。

報告の例:

- 誤解を招くメタデータ
- 未宣言の認証情報または権限要件
- 不審なインストール手順
- 詐欺コメントまたはなりすまし
- 悪意のある登録または商標の不正使用
- [許容される利用](/ja-JP/clawhub/acceptable-usage) に違反するコンテンツ

## 公開者の ClawScan メモ

公開者は、Skills またはプラグインを公開するときに任意の ClawScan メモを提供できます。このメモは、ネットワークアクセス、ネイティブホストアクセス、プロバイダー固有の認証情報など、通常とは異なって見える可能性がある動作について ClawScan に文脈を提供します。

## モデレーション保留

静的スキャナーがアップロードされた Skills を悪意ありとしてフラグした場合、公開者は自動的にモデレーション保留下に置かれます（ユーザーに `requiresModerationAt` が設定されます）。これにより、公開者のすべての Skills が非表示になり、今後の公開は非表示で開始され、`user.moderation.auto` 監査ログエントリが作成されます。

静的な不審検出はモデレーター向けのファイル/行の証拠として保持されますが、それ自体ではコンテンツを非表示にしたり、公開スキャン判定を決定したりしません。新しいアップロードは LLM レビューが確定するまでレビュー中/保留状態のままです。静的スキャンが即時にブロックするのは悪意あるシグネチャのみです。VirusTotal エンジンのヒットは表示可能なセキュリティ証拠として残りますが、VirusTotal Code Insight/Palm の判定は助言的なものであり、それ自体では Skills を非表示にしません。ClawScan LLM レビューでは、目的に沿ったメモをガイダンスとして保持します。中程度のレビュー検出はアーティファクト上で引き続き表示されますが、不審フィルターは影響度の高い LLM の懸念、悪意ある検出、または裏付けのある AV エンジン検出のために予約されています。

管理者は誤検知による保留を解除できます。

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

これにより `requiresModerationAt` と `requiresModerationReason` がクリアされ、ユーザーレベルの保留によって非表示になっていた Skills が復元され、`user.moderation.lift` 監査ログエントリが書き込まれます。他の理由で非表示になっている Skills、またはその Skills 自体の静的スキャンが悪意ありのままであるものは、非表示のままです。

## 禁止とアカウント状態

ClawHub ポリシーに違反したアカウントは、公開アクセスを失う場合があります。重大な悪用は、アカウント禁止、トークン失効、コンテンツ非表示、またはリスト削除につながる場合があります。

削除、禁止、または無効化されたアカウントは、ClawHub API トークンを使用できません。アカウント措置後に CLI 認証が失敗し始めた場合は、Web UI にサインインしてアカウント状態を確認してください。サインインまたは通常の CLI アクセスがブロックされている場合は、回復レビューのため security@openclaw.ai に連絡してください。

## 公開者向けガイダンス

誤検知を減らし、ユーザーの信頼を高めるために:

- 名前、概要、タグ、変更履歴を正確に保つ
- 必須の環境変数と権限を宣言する
- リリースに通常とは異なるが意図された動作がある場合は、公開者の ClawScan メモを追加する
- 難読化されたインストールコマンドを避ける
- 可能な場合はソースにリンクする
- プラグインを公開する前に dry run を使用する
- ユーザーまたはモデレーターがパッケージの動作について質問した場合は、明確に応答する
