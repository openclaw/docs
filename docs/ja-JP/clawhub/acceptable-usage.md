# 許容される利用

ClawHub は OpenClaw 向けの Skills、plugins、packages、マーケットプレイスメタデータをホストします。
コンテンツや公開時の挙動が ClawHub に属するかどうかを判断するために、このページを使用してください。

これらのルールは、掲載内容が何をするか、ユーザーに何の実行を求めるか、自身をどのように表現するか、公開者が ClawHub の発見、インストール、信頼の各サーフェスをどのように使用するかに適用されます。モデレーション状態とアカウントの状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。著作権またはその他の権利に関する請求については、[コンテンツ権利リクエスト](/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で、理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | 掲載内容が、ユーザーによるソフトウェアの構築、テスト、移行、デバッグ、ドキュメント作成、運用を支援する。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのある操作にレビュー、ドライラン、プレビュー、確認の経路が含まれる。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが承認済みレビュー向けとして位置付けられ、証拠を保持し、人による承認境界を明確に保つ。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明なセットアップ、明示的な権限を使用する。                                            |
| メンテナンスされているカタログ                              | 各掲載内容が明確に区別され、有用で、正確に説明され、合理的にメンテナンスされている。                                                |

文脈が重要です。同じトピックでも、限定的な防御的用途や同意に基づく設定では許容される一方、不正利用ワークフローとしてパッケージ化されると許容されない場合があります。

## 禁止されるコンテンツ

ClawHub は、主目的が不正利用、欺瞞、安全でない実行、または権利侵害であるコンテンツをホストしません。

| カテゴリ                                                    | 許可されないもの                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 不正アクセスまたはセキュリティ回避                      | 認証回避、アカウント乗っ取り、レート制限の不正利用、ライブ通話またはエージェントの乗っ取り、再利用可能なセッションの窃取、未承認ユーザー向けペアリングフローの自動承認。                                                                                                                                                   |
| プラットフォームの不正利用と禁止回避                              | 禁止後のステルスアカウント、アカウントのウォーミングまたはファーミング、偽のエンゲージメント、複数アカウントの自動化、大量投稿、スパムボット、検出回避を目的として構築された自動化。                                                                                                                                          |
| 詐欺、詐欺的行為、欺瞞的な金融ワークフロー             | 偽の証明書または請求書、欺瞞的な支払いフロー、詐欺的なアウトリーチ、偽のソーシャルプルーフ、詐欺のための合成アイデンティティワークフロー、明確な人による承認なしの支出または課金ツール。                                                                                                                    |
| プライバシー侵害的なエンリッチメントまたは監視                 | スパム目的の連絡先スクレイピング、晒し、ストーキング、未承諾アウトリーチと組み合わされたリード抽出、秘密裏の監視、同意のない生体照合、漏えいデータまたは侵害ダンプの使用。                                                                                                                  |
| 同意のないなりすましまたはアイデンティティ操作       | フェイススワップ、デジタルツイン、クローン化されたインフルエンサー、偽のペルソナ、またはなりすましや誤認を目的として使用されるその他のツール。                                                                                                                                                                                                 |
| 露骨な性的コンテンツまたは安全制限が無効化された成人向け生成 | NSFW 画像、動画、コンテンツ生成、サードパーティ API を包む成人向けコンテンツラッパー、または主目的が露骨な性的コンテンツである掲載内容。                                                                                                                                                       |
| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確にレビュー可能でないままダウンロードしたコンテンツを `sh` または `bash` で実行する pipe-to-shell インストーラー、未申告のシークレットまたは秘密鍵要件、明確にレビュー可能でないリモート `npx @latest` 実行、掲載内容の実行に本当に必要なものを隠すメタデータ。 |
| 著作権を侵害する、または権利に違反する素材           | 他者の skill、plugin、ドキュメント、ブランドアセット、またはプロプライエタリコードを許可なく再公開すること、ライセンス条件に違反すること、元の作者または公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイス上の挙動

ClawHub は、公開者がマーケットプレイスをどのように使用するかもレビューします。発見、指標、信頼シグナル、モデレーションシステム、またはユーザーの注意を操作するために ClawHub を使用しないでください。

禁止されるマーケットプレイス上の挙動には、次のものが含まれます。

- 実際のユーザー価値があるように見えない、低労力、重複、プレースホルダー、または機械生成の掲載内容を大量に公開すること
- ほぼ同一の skills または plugins で検索やカテゴリサーフェスを埋め尽くすこと
- 使用、メンテナンス、ソースの明確さ、または意味のある差別化がほとんど、あるいはまったくない掲載内容を数百件公開すること
- 自動化、自己インストールループ、偽アカウント、協調的な活動、有料エンゲージメント、またはその他の非オーガニックな挙動によって、インストール数、ダウンロード数、スター数、またはその他のエンゲージメント指標を人為的に膨らませること
- モデレーション、禁止、公開者制限、またはマーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有者、ソース、機能、セキュリティ態勢、インストール要件、または別のプロジェクトや公開者との関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は自動的に不正利用になるわけではありません。掲載内容が意味のある形で異なり、正確に説明され、メンテナンスされ、実際のユーザーに使用されている場合、大規模なカタログは許容されます。大量であることに、内容の薄さ、重複、誤解を招く内容、メンテナンス不足、または人為的な宣伝が組み合わさると、大規模なカタログは信頼と安全性の問題になります。

## コンテンツ権利

ClawHub 上のコンテンツが自分の著作権またはその他の権利を侵害していると思われる場合は、[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。掲載内容が安全でない、悪意がある、または誤解を招く場合を除き、著作権または権利に関する請求に通常のマーケットプレイス報告を使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツや不正な公開行為を特定するために、自動チェック、統計的な不正利用シグナル、ユーザー報告、スタッフレビューを使用する場合があります。シグナルはそれだけで不正利用を証明するものではありません。ClawHub がレビューの必要性を判断する助けになります。

当社は次の対応を行う場合があります。

- 違反する掲載内容を非表示、保留、削除、ソフトデリート、またはリソースタイプでサポートされている場合はハードデリートする
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツをソフトデリートする
- 公開アクセスを制限する
- 繰り返し違反する者、または重大な違反を行う者を禁止する

明らかな不正利用に対して、警告を先に行う執行を保証するものではありません。報告、モデレーション保留、非表示の掲載内容、禁止、アカウントの状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。
