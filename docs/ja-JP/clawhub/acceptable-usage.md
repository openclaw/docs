# 利用規定

ClawHub は OpenClaw 向けの Skills、Plugin、パッケージ、マーケットプレイスのメタデータをホストします。
コンテンツまたは公開時の挙動が ClawHub に適しているかを判断する際は、このページを参照してください。

これらの規則は、掲載項目の動作、ユーザーに実行を求める内容、掲載項目の表現方法、および公開者による ClawHub の検索、インストール、信頼性に関する機能の利用方法に適用されます。モデレーションの状態とアカウントのステータスについては、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。著作権またはその他の権利に関する申し立てについては、[コンテンツの権利に関する申請](/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用かつ理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性向上                           | 掲載項目が、ソフトウェアの構築、テスト、移行、デバッグ、文書化、または運用に役立つこと。                                               |
| UI、データ、自動化のワークフロー               | 適用範囲が明確で、必要な認証情報が明示され、リスクのある操作にレビュー、ドライラン、プレビュー、または確認の手順が含まれていること。 |
| 防御的セキュリティ、モデレーション、不正利用のレビュー | ツールが許可されたレビューを目的として位置付けられ、証拠を保全し、人間による承認の境界が明確に維持されていること。                          |
| 個人またはチームのワークフロー                       | ワークフローで、同意に基づくアカウント、透明性のあるセットアップ、明示的な権限が使用されていること。                                            |
| 保守されているカタログ                              | 各掲載項目が固有かつ有用で、正確に説明され、適切に保守されていること。                                                |

文脈が重要です。同じトピックでも、限定的な防御目的または同意に基づく状況では許容される一方、不正利用のワークフローとしてパッケージ化されている場合は許容されません。

## 禁止されるコンテンツ

ClawHub は、不正利用、欺瞞、安全でない実行、または権利侵害を主な目的とするコンテンツをホストしません。

| カテゴリ                                                    | 禁止されるもの                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 不正アクセスまたはセキュリティの回避                      | 認証の回避、アカウントの乗っ取り、レート制限の悪用、進行中の通話やエージェントの乗っ取り、再利用可能なセッションの窃取、または未承認ユーザーに対するペアリングフローの自動承認。                                                                                                                                                   |
| プラットフォームの悪用と禁止措置の回避                              | 禁止後のステルスアカウント、アカウントのウォーミングや大量育成、偽のエンゲージメント、複数アカウントの自動化、大量投稿、スパムボット、または検出を回避する目的で構築された自動化。                                                                                                                                          |
| 詐欺、不正行為、欺瞞的な金融ワークフロー             | 偽の証明書や請求書、欺瞞的な決済フロー、詐欺的な勧誘、偽の社会的証明、詐欺を目的とした合成 ID のワークフロー、または人間による明確な承認なしに支出や請求を行うツール。                                                                                                                    |
| プライバシーを侵害する情報拡充または監視                 | スパム目的の連絡先スクレイピング、個人情報の暴露、ストーキング、一方的な勧誘と組み合わせた見込み客情報の抽出、秘密裏の監視、同意のない生体認証照合、または漏えいデータや侵害データダンプの使用。                                                                                                                  |
| 同意のないなりすましまたは ID の操作       | フェイススワップ、デジタルツイン、インフルエンサーのクローン、偽のペルソナ、またはなりすましや誤認を目的として使用されるその他のツール。                                                                                                                                                                                                 |
| 露骨な性的コンテンツまたは安全機能を無効化した成人向け生成 | NSFW の画像、動画、コンテンツの生成、サードパーティ API を利用した成人向けコンテンツのラッパー、または露骨な性的コンテンツを主な目的とする掲載項目。                                                                                                                                                       |
| 隠蔽された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確にレビューできない状態でダウンロードしたコンテンツを `sh` や `bash` で実行するパイプ・トゥ・シェル形式のインストーラー、宣言されていないシークレットや秘密鍵の要件、明確にレビューできないリモートでの `npx @latest` 実行、または掲載項目の実行に実際に必要なものを隠すメタデータ。 |
| 著作権またはその他の権利を侵害する素材           | 他者の Skills、Plugin、ドキュメント、ブランド資産、またはプロプライエタリコードを許可なく再公開すること、ライセンス条件に違反すること、または元の作者や公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイスでの行為

ClawHub は、公開者によるマーケットプレイスの利用方法も審査します。検索結果、指標、信頼性を示すシグナル、モデレーションシステム、またはユーザーの注目を操作する目的で ClawHub を使用しないでください。

禁止されるマーケットプレイスでの行為には、以下が含まれます。

- 実際のユーザー価値があるように見えない、労力の少ない、重複した、プレースホルダーの、または
  機械生成された掲載項目を大量に公開すること
- ほぼ同一の Skills や Plugin で検索結果やカテゴリの表示領域を埋め尽くすこと
- 利用実績、保守、ソースの明確さ、または有意な差別化がほとんど、もしくはまったくない
  数百件の掲載項目を公開すること
- 自動化、自己インストールのループ、偽アカウント、組織的な
  活動、有償のエンゲージメント、またはその他の非オーガニックな行為により、インストール数、ダウンロード数、スター数、その他のエンゲージメント
  指標を人為的に水増しすること
- モデレーション、禁止措置、公開者の制限、または
  マーケットプレイスの審査を回避するために、アカウントを作成または切り替えること
- 所有権、ソース、機能、セキュリティ態勢、
  インストール要件、または別のプロジェクトや公開者との関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされた
  コンテンツを繰り返しアップロードすること

大量公開が自動的に不正利用となるわけではありません。掲載項目が有意に異なり、正確に説明され、保守され、実際のユーザーに利用されている場合、大規模なカタログは許容されます。大量であることに加えて、内容が乏しい、重複している、誤解を招く、保守されていない、または人為的に宣伝されている場合、大規模なカタログは信頼性と安全性の問題になります。

## コンテンツの権利

ClawHub 上のコンテンツが自身の著作権またはその他の権利を侵害していると思われる場合は、[コンテンツの権利に関する申請](/clawhub/content-rights)を利用してください。掲載項目が安全でない、悪意がある、または誤解を招くものでもある場合を除き、著作権またはその他の権利に関する申し立てに通常のマーケットプレイス報告を使用しないでください。

## 審査と執行

ClawHub は、安全でないコンテンツや不正な公開行為を特定するために、自動チェック、統計的な不正利用シグナル、ユーザーからの報告、スタッフによる審査を使用する場合があります。シグナルだけで不正利用が証明されるわけではなく、ClawHub が審査の必要な対象を判断するために役立ちます。

以下の措置を講じる場合があります。

- 違反する掲載項目を非表示、保留、削除、論理削除、またはリソースの種類でサポートされている場合は
  完全削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを失効させる
- 関連コンテンツを論理削除する
- 公開アクセスを制限する
- 違反を繰り返す者または重大な違反者を禁止する

明白な不正利用について、必ず事前に警告してから措置を講じることは保証しません。報告、モデレーションによる保留、非表示の掲載項目、禁止措置、アカウントのステータスについては、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。
