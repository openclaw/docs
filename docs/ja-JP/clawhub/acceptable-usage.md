# 利用許容範囲

ClawHub は OpenClaw 向けの Skills、Plugin、パッケージ、マーケットプレイスメタデータをホストします。
コンテンツや公開の振る舞いが ClawHub に属するかどうかを判断するために、このページを使用してください。

これらのルールは、リスティングが何をするか、ユーザーに何の実行を求めるか、どのように自身を表現するか、公開者が ClawHub のディスカバリー、インストール、信頼サーフェスをどのように使用するかに適用されます。モデレーション状態とアカウントの健全性については、[モデレーションとアカウント安全性](/clawhub/moderation)を参照してください。著作権またはその他の権利主張については、[コンテンツ権利リクエスト](/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で、理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される場合                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | リスティングが、ユーザーによるソフトウェアの構築、テスト、移行、デバッグ、文書化、または運用を支援する。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのある操作にレビュー、ドライラン、プレビュー、または確認の経路が含まれる。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが認可されたレビュー向けとして位置づけられ、証拠を保持し、人間による承認の境界を明確に保つ。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明なセットアップ、明示的な権限を使用する。                                            |
| メンテナンスされているカタログ                              | 各リスティングが明確に区別され、有用で、正確に説明され、合理的にメンテナンスされている。                                                |

コンテキストが重要です。同じトピックでも、狭い防御的または同意に基づく設定では許容される一方、不正利用ワークフローとしてパッケージ化されている場合は許容されません。

## 許可されないコンテンツ

ClawHub は、不正利用、欺瞞、安全でない実行、または権利侵害を主な目的とするコンテンツをホストしません。

| カテゴリ                                                    | 許可されない内容                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 認可されていないアクセスまたはセキュリティ回避                      | 認証バイパス、アカウント乗っ取り、レート制限の悪用、ライブ通話またはエージェントの乗っ取り、再利用可能なセッションの窃取、または未承認ユーザー向けペアリングフローの自動承認。                                                                                                                                                   |
| プラットフォーム不正利用と BAN 回避                              | BAN 後のステルスアカウント、アカウントのウォーミングまたはファーミング、偽のエンゲージメント、複数アカウント自動化、大量投稿、スパムボット、または検出を回避するために構築された自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書や請求書、欺瞞的な支払いフロー、スキャムのアウトリーチ、偽の社会的証明、詐欺目的の合成 ID ワークフロー、または明確な人間の承認なしでの支出/課金ツール。                                                                                                                    |
| プライバシーを侵害するエンリッチメントまたは監視                 | スパム目的の連絡先スクレイピング、ドクシング、ストーキング、未承諾アウトリーチと組み合わせたリード抽出、秘密監視、同意のない生体照合、または漏えいデータや侵害ダンプの使用。                                                                                                                  |
| 同意のないなりすましまたはアイデンティティ操作       | フェイススワップ、デジタルツイン、クローン化されたインフルエンサー、偽のペルソナ、またはなりすましや誤認誘導に使用されるその他のツール。                                                                                                                                                                                                 |
| 露骨な性的コンテンツまたは安全機能を無効化した成人向け生成 | NSFW の画像、動画、またはコンテンツ生成、サードパーティ API の成人向けコンテンツラッパー、または露骨な性的コンテンツを主な目的とするリスティング。                                                                                                                                                       |
| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確なレビュー可能性なしにダウンロードしたコンテンツを `sh` または `bash` で実行するような pipe-to-shell インストーラー、未申告のシークレットまたは秘密鍵要件、明確なレビュー可能性なしのリモート `npx @latest` 実行、またはリスティングの実行に本当に必要なものを隠すメタデータ。 |
| 著作権侵害または権利侵害にあたる素材           | 他者の skill、Plugin、ドキュメント、ブランドアセット、またはプロプライエタリコードを許可なく再公開すること、ライセンス条項に違反すること、または元の作者や公開者になりすますこと。                                                                                                                            |

## 許可されないマーケットプレイス上の振る舞い

ClawHub は、公開者がマーケットプレイスをどのように使用するかもレビューします。ClawHub を使用して、ディスカバリー、メトリクス、信頼シグナル、モデレーションシステム、またはユーザーの注意を操作しないでください。

許可されないマーケットプレイス上の振る舞いには、次が含まれます。

- 実際のユーザー価値があるように見えない、低労力、重複、プレースホルダー、または機械生成のリスティングを大量に一括公開すること
- ほぼ同一の Skills または Plugin で検索やカテゴリサーフェスをあふれさせること
- 利用、メンテナンス、ソースの明確さ、または意味のある差別化がほとんどない、あるいはまったくないリスティングを数百件公開すること
- 自動化、自己インストールループ、偽アカウント、協調的活動、有料エンゲージメント、またはその他の非オーガニックな振る舞いによって、インストール数、ダウンロード数、スター、またはその他のエンゲージメントメトリクスを人為的に水増しすること
- モデレーション、BAN、公開者制限、またはマーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有権、ソース、機能、セキュリティ態勢、インストール要件、または別のプロジェクトや公開者との関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は自動的に不正利用になるわけではありません。リスティングが意味のある違いを持ち、正確に説明され、メンテナンスされ、実際のユーザーによって使用されている場合、大規模なカタログは許容されます。量が薄い内容、重複、誤解を招く内容、未メンテナンス、または人為的に宣伝されたリスティングと組み合わさると、大規模なカタログは信頼と安全性の問題になります。

## コンテンツ権利

ClawHub 上のコンテンツがあなたの著作権またはその他の権利を侵害していると考える場合は、[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。リスティングが安全でない、悪意がある、または誤解を招く場合を除き、著作権または権利主張に通常のマーケットプレイスレポートを使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツや不正な公開行為を特定するために、自動チェック、統計的な不正利用シグナル、ユーザーレポート、スタッフレビューを使用する場合があります。シグナル単独で不正利用を証明するわけではありません。ClawHub がレビューを必要とする対象を判断する助けになります。

当社は次を行う場合があります。

- 違反リスティングを非表示、保留、削除、ソフト削除、またはリソース種別でサポートされている場合はハード削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツをソフト削除する
- 公開アクセスを制限する
- 繰り返し違反するユーザーまたは重大な違反者を BAN する

明白な不正利用に対して、警告を先に行う執行を保証するものではありません。レポート、モデレーション保留、非表示リスティング、BAN、アカウントの健全性については、[モデレーションとアカウント安全性](/clawhub/moderation)を参照してください。
