# 許容される利用

このページでは、ClawHub が受け入れ可能な Skills とコンテンツの種類、およびホストしない不正利用ワークフローについて説明します。

これらのルールは意図的に実用性を重視しています。私たちは、孤立したキーワードだけでなく、エンドツーエンドの不正利用ワークフローを最も重視します。防御の回避、プラットフォームの悪用、人への詐欺、プライバシー侵害、または非同意の行為を可能にする目的で Skills が作られている場合、それは ClawHub に置くべきではありません。

## 最近のパターンのうち、明示的に許容されるもの

- 実際のコンポーネント、セマンティックトークン、アクセシブルな状態、テスト済みのユーザーフローを使用するフロントエンドおよびデザインシステム作業。
- 一回限りのマークアップではなく、インストール済みのソースコンポーネント、プロジェクトエイリアス、文書化されたバリアントを使用する shadcn/ui の合成。
- コメントを保持し、具体的な UI5 型を使用し、生成されたコントロールインターフェースをレビュー可能な状態に保つ UI5 JavaScript から TypeScript への変換。
- 証拠を示し、人間による承認の境界を明確に保つ、防御的なセキュリティレビュー、モデレーションツール、不正利用検出プロンプト。
- 明示的な認証情報、透明なセットアップ、ドライランまたはプレビューモードを備えた、個人またはチームアカウント向けの同意に基づくワークフロー自動化。
- サポート対象のソフトウェアに範囲を限定したドキュメント、移行ランブック、開発者ユーティリティ、テストフィクスチャ。

## 許容されないもの

- セキュリティ回避または不正アクセスのワークフロー。
  - 例: 認証回避、アカウント乗っ取り、CAPTCHA 回避、Cloudflare またはボット対策の回避、レート制限回避、保護を破ることを目的としたステルススクレイピング、ライブ通話またはエージェントの乗っ取り、再利用可能なセッション窃取、承認されていないユーザー向けペアリングフローの自動承認。

- プラットフォーム悪用および禁止回避。
  - 例: 禁止後のステルスアカウント、アカウントのウォーミングまたはファーミング、偽のエンゲージメント、カルマまたはフォロワーの育成、複数アカウント自動化、大量投稿、スパムボット、検出回避を目的として作られたマーケットプレイスまたはソーシャル自動化。

- 詐欺、詐欺的行為、および欺瞞的な金融ワークフロー。
  - 例: 偽の証明書、偽の請求書、欺瞞的な支払いフロー、詐欺的なアウトリーチ、偽の社会的証明、明確な人間による承認と透明な制御なしに支出または課金を可能にするツール、または詐欺のためのアカウント作成を目的とした合成アイデンティティワークフロー。

- プライバシー侵害的なスクレイピング、エンリッチメント、または監視。
  - 例: スパム目的で連絡先情報を大規模にスクレイピングすること、晒し行為、ストーキング、未承諾アウトリーチと組み合わせたリード抽出、密かな監視、明確な同意なしに使用される顔検索または生体照合、または漏えいデータや侵害ダンプの購入、公開、ダウンロード、運用化。

- 非同意のなりすましまたは欺瞞的なアイデンティティ操作。
  - 例: なりすましや誤認を目的として使用される、顔交換、デジタルツイン、偽ペルソナ、クローン化されたインフルエンサー、またはその他のアイデンティティ操作ツール。

- 明示的な性的コンテンツおよび安全機能を無効化した成人向け生成。
  - 例: NSFW 画像、動画、コンテンツ生成、サードパーティ API を囲う成人向けコンテンツラッパー、または主目的が明示的な性的コンテンツである Skills。

- 隠された、安全でない、または誤解を招く実行要件。
  - 例: 難読化されたインストールコマンド、`curl | sh`、未申告のシークレット要件、未申告の秘密鍵使用、明確なレビュー可能性のないリモート `npx @latest` 実行、Skills の実行に実際に必要なものを隠す誤解を招くメタデータ。

## 最近のパターンのうち、明示的に許容されないもの

- 「マーケットプレイスで禁止された後にステルス販売者アカウントを作成する。」
- 「未承認ユーザーがペアリングコードを自動的に受け取れるように Telegram ペアリングを変更する。」
- 「検出不能な自動化で Reddit/Twitter アカウントを育成する。」
- 「任意の用途向けに専門的な証明書または請求書を生成する。」
- 「安全チェックを無効化して NSFW コンテンツを生成する。」
- 「リードをスクレイピングし、連絡先をエンリッチし、大規模にコールドアウトリーチを開始する。」
- 「漏えいデータまたは侵害ダンプを購入、公開、またはダウンロードする。」
- 「合成アイデンティティまたは CAPTCHA 解決を使用して、メールまたはソーシャルアカウントを一括作成する。」

## レビュー担当者向けメモ

- 文脈が重要です。同じトピックでも、狭い防御的または同意に基づく設定では正当であり、不正利用ワークフローとしてパッケージ化されている場合には許容されないことがあります。
- Skills が回避、欺瞞、または非同意の利用に明確に最適化されている場合は、対応する方向に判断すべきです。
- これらのカテゴリで繰り返しアップロードすることは、コンテンツの非表示およびアカウント禁止の理由になります。

## 執行

- 違反する Skills を非表示、削除、または完全削除する場合があります。
- トークンを取り消し、関連コンテンツをソフト削除し、反復的または重大な違反者を禁止する場合があります。
- 明白な不正利用に対して、警告を先に行う執行を保証しません。
