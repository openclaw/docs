# 許容される利用

このページでは、ClawHub が許容する Skills とコンテンツの種類、およびホストしない不正利用ワークフローについて説明します。

これらのルールは意図的に実用的なものにしています。単独のキーワードだけでなく、エンドツーエンドの不正利用ワークフローを最も重視します。防御の回避、プラットフォームの悪用、人をだますこと、プライバシー侵害、または非同意の行為を可能にすることを目的に Skill が作られている場合、それは ClawHub にふさわしくありません。

## 最近のパターンのうち、明示的に許容するもの

- 実際のコンポーネント、セマンティックトークン、アクセシブルな状態、テスト済みのユーザーフローを使用するフロントエンドおよびデザインシステムの作業。
- 一回限りのマークアップではなく、インストール済みのソースコンポーネント、プロジェクトエイリアス、文書化されたバリアントを使用する shadcn/ui の構成。
- コメントを保持し、具体的な UI5 型を使用し、生成されたコントロールインターフェースをレビュー可能に保つ UI5 JavaScript から TypeScript への変換。
- 証拠を示し、人間による承認境界を明確に保つ、防御的なセキュリティレビュー、モデレーションツール、不正利用検出プロンプト。
- 明示的な認証情報、透明なセットアップ、ドライランまたはプレビューモードを備えた、個人またはチームアカウント向けの同意ベースのワークフロー自動化。
- 支援対象のソフトウェアにスコープを限定したドキュメント、移行ランブック、開発者ユーティリティ、テストフィクスチャ。

## 許容されないもの

- セキュリティ回避または不正アクセスのワークフロー。
  - 例: 認証バイパス、アカウント乗っ取り、CAPTCHA 回避、Cloudflare またはボット対策の回避、レート制限のバイパス、保護の突破を目的に設計されたステルススクレイピング、ライブ通話またはエージェントの乗っ取り、再利用可能なセッションの窃取、未承認ユーザー向けのペアリングフローの自動承認。

- プラットフォームの悪用と BAN 回避。
  - 例: BAN 後のステルスアカウント、アカウントのウォーミング/ファーミング、偽のエンゲージメント、カルマやフォロワーの育成、複数アカウントの自動化、大量投稿、スパムボット、検出回避を目的に構築されたマーケットプレイスまたはソーシャル自動化。

- 詐欺、詐欺的行為、欺瞞的な金融ワークフロー。
  - 例: 偽の証明書、偽の請求書、欺瞞的な支払いフロー、詐欺的なアウトリーチ、偽の社会的証明、明確な人間の承認と透明な制御なしに支出または請求を可能にするツール、または詐欺用アカウントを作成するために構築された合成アイデンティティのワークフロー。

- プライバシーを侵害するスクレイピング、エンリッチメント、または監視。
  - 例: スパム目的で連絡先情報を大規模にスクレイピングすること、晒し、ストーキング、未承諾のアウトリーチと組み合わせたリード抽出、秘密裏の監視、明確な同意なしに使用される顔検索または生体照合、流出データまたは侵害ダンプの購入、公開、ダウンロード、または運用化。

- 非同意のなりすましまたは欺瞞的なアイデンティティ操作。
  - 例: 顔の入れ替え、デジタルツイン、偽のペルソナ、クローン化されたインフルエンサー、またはなりすましや誤認誘導に使用されるその他のアイデンティティ操作ツール。

- 明示的な性的コンテンツおよび安全機能を無効化した成人向け生成。
  - 例: NSFW の画像/動画/コンテンツ生成、サードパーティ API を囲い込む成人向けコンテンツラッパー、または主目的が明示的な性的コンテンツである Skills。

- 隠された、安全でない、または誤解を招く実行要件。
  - 例: 難読化されたインストールコマンド、`curl | sh`、未申告のシークレット要件、未申告の秘密鍵使用、明確なレビュー可能性のないリモート `npx @latest` 実行、Skill の実行に本当に必要なものを隠す誤解を招くメタデータ。

## 最近のパターンのうち、明示的に許容しないもの

- 「マーケットプレイスで BAN された後にステルス販売者アカウントを作成する。」
- 「未承認ユーザーがペアリングコードを自動的に受け取るように Telegram のペアリングを変更する。」
- 「検出不能な自動化で Reddit/Twitter アカウントを育成する。」
- 「任意の用途向けに専門的な証明書や請求書を生成する。」
- 「安全チェックを無効化して NSFW コンテンツを生成する。」
- 「リードをスクレイピングし、連絡先をエンリッチし、大規模なコールドアウトリーチを開始する。」
- 「流出データまたは侵害ダンプを購入、公開、またはダウンロードする。」
- 「合成アイデンティティまたは CAPTCHA 解決を使用して、メールまたはソーシャルアカウントを一括作成する。」

## レビュー担当者向けメモ

- 文脈が重要です。同じトピックでも、限定的な防御目的または同意ベースの設定では正当になり得ますが、不正利用ワークフローとしてパッケージ化されている場合は許容されません。
- Skill が回避、欺瞞、または非同意の利用に明確に最適化されている場合は、対応する方向に判断するべきです。
- これらのカテゴリで繰り返しアップロードされる場合、コンテンツの非表示およびアカウントの BAN の根拠になります。

## 執行

- 違反する Skills を非表示、削除、または完全削除する場合があります。
- トークンを取り消し、関連コンテンツをソフト削除し、繰り返しまたは重大な違反を行う利用者を BAN する場合があります。
- 明白な不正利用に対して、警告を先に行う執行を保証するものではありません。
