# 許容される利用

このページでは、ClawHub が許容するスキルやコンテンツの種類と、ホストしない不正利用ワークフローについて説明します。

これらのルールは意図的に実用性を重視しています。私たちが最も重視するのは、孤立したキーワードだけでなく、エンドツーエンドの不正利用ワークフローです。防御の回避、プラットフォームの不正利用、人への詐欺、プライバシーの侵害、または同意のない行為を可能にする目的でスキルが作られている場合、それは ClawHub に属しません。

## 最近のパターンのうち、明示的に許容するもの

- 実際のコンポーネント、セマンティックトークン、アクセシブルな状態、テスト済みのユーザーフローを使用するフロントエンドおよびデザインシステム作業。
- その場限りのマークアップではなく、インストール済みのソースコンポーネント、プロジェクトエイリアス、文書化されたバリアントを使用する shadcn/ui の構成。
- コメントを保持し、具体的な UI5 型を使用し、生成されたコントロールインターフェイスをレビュー可能な状態に保つ UI5 JavaScript-to-TypeScript 変換。
- 証拠を示し、人間の承認境界を明確に保つ、防御的なセキュリティレビュー、モデレーションツール、不正利用検出プロンプト。
- 明示的な認証情報、透明なセットアップ、ドライランまたはプレビューモードを備えた、個人またはチームアカウント向けの同意ベースのワークフロー自動化。
- サポート対象ソフトウェアにスコープを限定した、ドキュメント、移行ランブック、開発者ユーティリティ、テストフィクスチャ。

## 許容しないもの

- セキュリティ回避または不正アクセスのワークフロー。
  - 例: 認証バイパス、アカウント乗っ取り、CAPTCHA バイパス、Cloudflare またはボット対策の回避、レート制限バイパス、保護を破ることを目的に設計されたステルススクレイピング、ライブ通話またはエージェントの乗っ取り、再利用可能なセッションの窃取、未承認ユーザー向けペアリングフローの自動承認。

- プラットフォームの不正利用と BAN 回避。
  - 例: BAN 後のステルスアカウント、アカウントのウォーミングやファーミング、偽のエンゲージメント、カルマまたはフォロワーの育成、マルチアカウント自動化、大量投稿、スパムボット、検出を避けるために作られたマーケットプレイスまたはソーシャル自動化。

- 詐欺、スキャム、不正な金融ワークフロー。
  - 例: 偽の証明書、偽の請求書、欺瞞的な支払いフロー、スキャム outreach、偽のソーシャルプルーフ、明確な人間の承認と透明な制御なしに支出または請求を可能にするツール、または詐欺用アカウントを作成するために作られた合成 ID ワークフロー。

- プライバシーを侵害するスクレイピング、エンリッチメント、または監視。
  - 例: スパム目的で連絡先情報を大規模にスクレイピングすること、晒し、ストーキング、未承諾 outreach と組み合わされたリード抽出、秘密裏の監視、明確な同意なしに使用される顔検索または生体認証マッチング、漏洩データまたは侵害ダンプの購入、公開、ダウンロード、または運用化。

- 同意のないなりすまし、または欺瞞的な ID 操作。
  - 例: フェイススワップ、デジタルツイン、偽のペルソナ、クローン化されたインフルエンサー、またはなりすましや誤認を目的に使用されるその他の ID 操作ツール。

- 明示的な性的コンテンツおよび安全機能を無効化した成人向け生成。
  - 例: NSFW の画像、動画、コンテンツ生成、サードパーティ API をラップした成人向けコンテンツ、または主目的が明示的な性的コンテンツであるスキル。

- 隠された、安全でない、または誤解を招く実行要件。
  - 例: 難読化されたインストールコマンド、`curl | sh`、未申告のシークレット要件、未申告の秘密鍵使用、明確なレビュー可能性のないリモート `npx @latest` 実行、スキルが実際に実行に必要とするものを隠す誤解を招くメタデータ。

## 最近のパターンのうち、明示的に許容しないもの

- 「マーケットプレイスでの BAN 後にステルス販売者アカウントを作成する。」
- 「未承認ユーザーがペアリングコードを自動的に受け取れるように Telegram のペアリングを変更する。」
- 「検出されない自動化で Reddit/Twitter アカウントを育成する。」
- 「任意の用途向けに専門的な証明書や請求書を生成する。」
- 「安全性チェックを無効にして NSFW コンテンツを生成する。」
- 「リードをスクレイピングし、連絡先をエンリッチし、大規模なコールド outreach を開始する。」
- 「漏洩データまたは侵害ダンプを購入、公開、またはダウンロードする。」
- 「合成 ID または CAPTCHA 解決を使用してメールまたはソーシャルアカウントを一括作成する。」

## レビュアー向けの注記

- コンテキストが重要です。同じトピックでも、防御目的または同意ベースの狭い設定では正当であり、不正利用ワークフローとしてパッケージ化されている場合は許容できないことがあります。
- スキルが回避、欺瞞、または同意のない利用に明確に最適化されている場合は、対応する方向に寄せるべきです。
- これらのカテゴリで繰り返しアップロードが行われた場合、コンテンツの非表示およびアカウントの BAN の根拠になります。

## 執行

- 違反するスキルを非表示、削除、または完全削除する場合があります。
- トークンの取り消し、関連コンテンツの論理削除、繰り返し違反または重大な違反を行ったユーザーの BAN を行う場合があります。
- 明白な不正利用について、警告を先に行う執行を保証しません。
