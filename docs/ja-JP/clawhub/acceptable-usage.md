# 許容される利用

ClawHub は OpenClaw 向けのスキル、プラグイン、パッケージ、マーケットプレイスメタデータをホストします。
コンテンツや公開動作が ClawHub に属するかどうかを判断するために、このページを使用してください。

これらのルールは、リスティングが何を行うか、ユーザーに何の実行を求めるか、どのように自身を表現するか、公開者が ClawHub の発見、インストール、信頼のサーフェスをどのように使用するかに適用されます。モデレーション状態とアカウントの状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。著作権またはその他の権利に関する申し立てについては、[コンテンツ権利リクエスト](/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | リスティングが、ユーザーによるソフトウェアの構築、テスト、移行、デバッグ、文書化、運用に役立つ。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示されており、リスクのある操作にレビュー、ドライラン、プレビュー、確認の経路が含まれている。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが認可されたレビュー向けとして位置付けられ、証拠を保持し、人間による承認境界を明確に保っている。                          |
| 個人またはチームのワークフロー                       | ワークフローが同意に基づくアカウント、透明なセットアップ、明示的な権限を使用している。                                            |
| メンテナンスされているカタログ                              | 各リスティングが個別性を持ち、有用で、正確に説明され、合理的にメンテナンスされている。                                                |

文脈が重要です。同じトピックでも、狭い防御的または同意に基づく設定では許容される場合があり、不正利用ワークフローとしてパッケージ化されている場合は許容されません。

| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| プラットフォームの不正利用と禁止回避                              | 禁止後のステルスアカウント、アカウントのウォーミングまたはファーミング、偽のエンゲージメント、複数アカウントの自動化、大量投稿、スパムボット、または検出回避のために構築された自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書または請求書、欺瞞的な支払いフロー、スキャム勧誘、偽のソーシャルプルーフ、不正行為のための合成アイデンティティワークフロー、または明確な人間による承認なしに支出や請求を行うツール。                                                                                                                    |

| 同意のないなりすましまたはアイデンティティ操作       | フェイススワップ、デジタルツイン、複製されたインフルエンサー、偽のペルソナ、またはなりすましや誤解を招くために使用されるその他のツール。                                                                                                                                                                                                 |

| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確なレビュー可能性なしにダウンロードされたコンテンツを `sh` または `bash` で実行するような pipe-to-shell インストーラー、宣言されていないシークレットまたは秘密鍵の要件、明確なレビュー可能性なしのリモート `npx @latest` 実行、またはリスティングの実行に実際に必要なものを隠すメタデータ。 |
| 著作権侵害または権利侵害にあたる素材           | 許可なく他者のスキル、プラグイン、ドキュメント、ブランド資産、またはプロプライエタリコードを再公開すること、ライセンス条件に違反すること、または元の作者や公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイス行為

ClawHub は、公開者によるマーケットプレイスの利用方法もレビューします。発見、指標、信頼シグナル、モデレーションシステム、またはユーザーの注意を操作するために ClawHub を使用しないでください。

禁止されるマーケットプレイス行為には、次が含まれます。

- 実際のユーザー価値があるように見えない、低労力、重複、プレースホルダー、または機械生成のリスティングを大量公開すること
- ほぼ同一のスキルまたはプラグインで検索やカテゴリのサーフェスをあふれさせること
- 利用、メンテナンス、ソースの明確性、または意味のある差別化がほとんどない、またはまったくないリスティングを数百件公開すること
- 自動化、自己インストールループ、偽アカウント、協調活動、有料エンゲージメント、またはその他の非オーガニックな行動によって、インストール、ダウンロード、スター、またはその他のエンゲージメント指標を人為的に増加させること
- モデレーション、禁止、公開者制限、またはマーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有権、ソース、機能、セキュリティ姿勢、インストール要件、または別のプロジェクトや公開者との関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は自動的に不正利用になるわけではありません。リスティングが意味のある違いを持ち、正確に説明され、メンテナンスされ、実際のユーザーに使用されている場合、大規模なカタログは許容されます。大量であることが、内容の薄さ、重複、誤解を招く内容、メンテナンスされていない状態、または人為的に宣伝されたリスティングと組み合わさると、大規模なカタログは信頼と安全性の問題になります。

## コンテンツ権利

ClawHub 上のコンテンツがあなたの著作権またはその他の権利を侵害していると考える場合は、[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。リスティングが安全でない、悪意がある、または誤解を招くものでもある場合を除き、著作権または権利に関する申し立てに通常のマーケットプレイス報告を使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツや不正な公開行為を特定するために、自動チェック、統計的な不正利用シグナル、ユーザー報告、スタッフレビューを使用する場合があります。シグナルだけで不正利用が証明されるわけではありません。ClawHub がレビューすべき対象を判断する助けになります。

当社は次を行う場合があります。

- 違反しているリスティングを非表示、保留、削除、ソフトデリート、またはリソースタイプでサポートされている場合はハードデリートする
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツをソフトデリートする
- 公開アクセスを制限する
- 繰り返し違反する者または重大な違反者を禁止する

明白な不正利用に対して、警告を先行する執行を保証するものではありません。報告、モデレーション保留、非表示リスティング、禁止、アカウントの状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。
