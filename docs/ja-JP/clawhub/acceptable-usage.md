# 許容される利用

ClawHub は OpenClaw 向けの Skills、Plugin、パッケージ、マーケットプレイスメタデータをホストします。
このページは、コンテンツや公開時の動作が ClawHub に属するかどうかを判断するために使用します。

これらのルールは、掲載内容が何をするか、ユーザーに何の実行を求めるか、どのように自身を表現するか、公開者が ClawHub の発見、インストール、信頼のサーフェスをどのように使うかに適用されます。モデレーション状態とアカウントの状態については、[モデレーションとアカウントの安全性](/ja-JP/clawhub/moderation)を参照してください。著作権またはその他の権利侵害の申し立てについては、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | 掲載内容が、ユーザーによるソフトウェアの構築、テスト、移行、デバッグ、ドキュメント化、運用を支援する。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのある操作にレビュー、ドライラン、プレビュー、確認の経路が含まれている。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが認可されたレビュー向けとして位置づけられ、証拠を保持し、人間による承認境界を明確にしている。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明なセットアップ、明示的な権限を使用する。                                            |
| 保守されているカタログ                              | 各掲載内容が明確に区別され、有用で、正確に説明され、合理的に保守されている。                                                |

文脈が重要です。同じトピックでも、限定的な防御的または同意に基づく設定では許容される場合があり、不正利用ワークフローとしてパッケージ化されている場合は許容されません。

## 禁止されるコンテンツ

| カテゴリ                                                    | 許可されないもの                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 不正アクセスまたはセキュリティ回避                      | 認証回避、アカウント乗っ取り、レート制限の不正利用、ライブ通話またはエージェントの乗っ取り、再利用可能なセッションの窃取、未承認ユーザー向けのペアリングフローの自動承認。                                                                                                                                                   |
| プラットフォーム不正利用と禁止措置の回避                              | 禁止後のステルスアカウント、アカウントのウォーミングまたはファーミング、偽のエンゲージメント、複数アカウントの自動化、大量投稿、スパムボット、検出回避のために作られた自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書または請求書、欺瞞的な支払いフロー、スキャム目的のアウトリーチ、偽のソーシャルプルーフ、詐欺のための合成 ID ワークフロー、明確な人間の承認がない支出または課金ツール。                                                                                                                    |
| プライバシーを侵害するエンリッチメントまたは監視                 | スパム目的の連絡先スクレイピング、ドキシング、ストーキング、未承諾アウトリーチと組み合わせたリード抽出、秘密裏の監視、同意のない生体情報照合、漏えいデータまたは侵害ダンプの使用。                                                                                                                  |
| 同意のないなりすましまたは ID 操作       | フェイススワップ、デジタルツイン、複製されたインフルエンサー、偽のペルソナ、またはなりすましや誤誘導に使われるその他のツール。                                                                                                                                                                                                 |
| 明示的な性的コンテンツまたは安全機能を無効化した成人向け生成 | NSFW 画像、動画、コンテンツ生成、サードパーティ API を囲う成人向けコンテンツラッパー、または明示的な性的コンテンツを主目的とする掲載内容。                                                                                                                                                       |
| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確なレビュー可能性なしにダウンロードしたコンテンツを `sh` または `bash` で実行するような pipe-to-shell インストーラー、未申告のシークレットまたは秘密鍵の要件、明確なレビュー可能性のないリモート `npx @latest` 実行、または掲載内容の実行に本当に必要なものを隠すメタデータ。 |
| 著作権を侵害する、または権利に違反する素材           | 他者の Skills、Plugin、ドキュメント、ブランドアセット、またはプロプライエタリコードを許可なく再公開すること、ライセンス条件への違反、元の作者または公開者へのなりすまし。                                                                                                                            |

## 禁止されるマーケットプレイス上の行為

ClawHub は、公開者によるマーケットプレイスの使い方もレビューします。発見、指標、信頼シグナル、モデレーションシステム、ユーザーの注意を操作する目的で ClawHub を使用しないでください。

禁止されるマーケットプレイス上の行為には、次が含まれます。

- 実際のユーザー価値があるように見えない、低労力、重複、プレースホルダー、または機械生成の掲載内容を大量に公開すること
- ほぼ同一の Skills または Plugin で検索やカテゴリのサーフェスを埋め尽くすこと
- 利用、保守、ソースの明確性、意味のある差別化がほとんどまたはまったくない掲載内容を数百件公開すること
- 自動化、自己インストールループ、偽アカウント、協調的な活動、有償エンゲージメント、またはその他の非オーガニックな行為によって、インストール数、ダウンロード数、スター数、その他のエンゲージメント指標を人為的に水増しすること
- モデレーション、禁止措置、公開者の制限、マーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有権、ソース、機能、セキュリティ態勢、インストール要件、または別のプロジェクトや公開者との提携についてユーザーを誤解させること
- すでに非表示、削除、またはブロックされたコンテンツを、根本的な問題を修正せずに繰り返しアップロードすること

大量公開は自動的に不正利用とは限りません。掲載内容が意味のある違いを持ち、正確に説明され、保守され、実際のユーザーに利用されている場合、大規模なカタログは許容されます。量が薄い、重複している、誤解を招く、保守されていない、または人為的に宣伝された掲載内容と組み合わさると、大規模なカタログは信頼と安全性の問題になります。

## コンテンツ権利

ClawHub 上のコンテンツが著作権またはその他の権利を侵害していると思われる場合は、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。掲載内容が安全でない、悪意がある、または誤解を招く場合を除き、著作権または権利に関する申し立てに通常のマーケットプレイス報告を使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツや不正な公開行為を特定するために、自動チェック、統計的な不正利用シグナル、ユーザー報告、スタッフレビューを使用する場合があります。シグナルだけで不正利用が証明されるわけではありません。ClawHub がレビューの必要な対象を判断する助けになります。

当社は次を行う場合があります。

- 違反する掲載内容を非表示、保留、削除、ソフト削除、またはリソース種別でサポートされている場合は完全削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを失効させる
- 関連コンテンツをソフト削除する
- 公開アクセスを制限する
- 繰り返しの違反者または重大な違反者を禁止する

明らかな不正利用に対して、事前警告を保証するものではありません。報告、モデレーション保留、非表示の掲載内容、禁止措置、アカウントの状態については、[モデレーションとアカウントの安全性](/ja-JP/clawhub/moderation)を参照してください。
