# 許容される使用

ClawHub は OpenClaw 向けの Skills、Plugin、パッケージ、マーケットプレイスメタデータをホストします。
コンテンツや公開動作が ClawHub に属するかどうかを判断するために、このページを使用してください。

これらのルールは、リスティングが何を行うか、ユーザーに何の実行を求めるか、自身をどのように表現するか、公開者が ClawHub の発見、インストール、信頼のサーフェスをどのように使用するかに適用されます。モデレーション状態とアカウントの健全性については、[モデレーションとアカウント安全性](/clawhub/moderation)を参照してください。著作権またはその他の権利に関する申し立てについては、[コンテンツ権利リクエスト](/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者生産性                           | リスティングが、ユーザーによるソフトウェアの構築、テスト、移行、デバッグ、文書化、または運用を支援する。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのあるアクションにレビュー、ドライラン、プレビュー、または確認の経路が含まれる。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが認可されたレビュー向けとして位置付けられ、証拠を保持し、人間による承認境界を明確に保つ。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明なセットアップ、明示的な権限を使用する。                                            |
| メンテナンスされたカタログ                              | 各リスティングが明確に区別され、有用で、正確に説明され、合理的にメンテナンスされている。                                                |

文脈が重要です。同じトピックでも、狭い防御的または同意に基づく設定では許容される場合があり、不正利用ワークフローとしてパッケージ化されている場合は許容されません。

## 禁止されるコンテンツ

ClawHub は、不正利用、欺瞞、安全でない実行、または権利侵害を主目的とするコンテンツをホストしません。

| カテゴリ                                                    | 許可されないもの                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 不正アクセスまたはセキュリティ回避                      | 認証バイパス、アカウント乗っ取り、レート制限の不正利用、ライブ通話またはエージェントの乗っ取り、再利用可能なセッション窃取、または未承認ユーザー向けのペアリングフローの自動承認。                                                                                                                                                   |
| プラットフォーム不正利用とBAN回避                              | BAN後のステルスアカウント、アカウントのウォーミングまたはファーミング、偽のエンゲージメント、複数アカウント自動化、大量投稿、スパムボット、または検出回避を目的に構築された自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書または請求書、欺瞞的な支払いフロー、スキャム勧誘、偽の社会的証明、詐欺のための合成IDワークフロー、または明確な人間の承認がない支出/請求ツール。                                                                                                                    |
| プライバシー侵害的なエンリッチメントまたは監視                 | スパム目的の連絡先スクレイピング、晒し、ストーキング、未承諾のアウトリーチと組み合わせたリード抽出、 covert monitoring、同意のない生体照合、または漏えいデータや侵害ダンプの使用。                                                                                                                  |
| 同意のないなりすましまたはID操作       | フェイススワップ、デジタルツイン、クローン化されたインフルエンサー、偽ペルソナ、またはなりすましや誤認誘導に使用されるその他のツール。                                                                                                                                                                                                 |
| 明示的な性的コンテンツまたは安全機能を無効化した成人向け生成 | NSFW画像、動画、またはコンテンツ生成、サードパーティAPIを囲む成人向けコンテンツラッパー、または明示的な性的コンテンツを主目的とするリスティング。                                                                                                                                                       |
| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確なレビュー可能性なしにダウンロードしたコンテンツを `sh` または `bash` で実行するような pipe-to-shell インストーラー、未申告のシークレットまたは秘密鍵要件、明確なレビュー可能性のないリモート `npx @latest` 実行、またはリスティングの実行に本当に必要なものを隠すメタデータ。 |
| 著作権侵害または権利侵害にあたる素材           | 他者の Skills、Plugin、ドキュメント、ブランドアセット、またはプロプライエタリコードを許可なく再公開すること、ライセンス条件に違反すること、または元の作者や公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイス上の動作

ClawHub は、公開者によるマーケットプレイスの使い方もレビューします。発見、指標、信頼シグナル、モデレーションシステム、またはユーザーの注意を操作するために ClawHub を使用しないでください。

禁止されるマーケットプレイス上の動作には、次が含まれます。

- 実際のユーザー価値があるように見えない、低品質、重複、プレースホルダー、または機械生成のリスティングを大量に一括公開すること
- ほぼ同一の Skills または Plugin で検索やカテゴリのサーフェスをあふれさせること
- 利用、メンテナンス、ソースの明確さ、または意味のある差別化がほとんど、またはまったくない数百件のリスティングを公開すること
- 自動化、自己インストールループ、偽アカウント、協調的活動、有償エンゲージメント、またはその他の非オーガニックな行為を通じて、インストール、ダウンロード、スター、またはその他のエンゲージメント指標を人為的に膨らませること
- モデレーション、BAN、公開者制限、またはマーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有権、ソース、機能、セキュリティ姿勢、インストール要件、または別のプロジェクトや公開者との関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は自動的に不正利用になるわけではありません。リスティングが意味のある形で異なり、正確に説明され、メンテナンスされ、実際のユーザーに使用されている場合、大規模なカタログは許容されます。量が薄い内容、重複、誤解を招く内容、メンテナンス不足、または人為的に宣伝されたリスティングと組み合わさると、大規模なカタログは信頼と安全性の問題になります。

## コンテンツ権利

ClawHub 上のコンテンツが著作権またはその他の権利を侵害していると考える場合は、[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。リスティングが安全でない、悪意がある、または誤解を招く場合を除き、通常のマーケットプレイス報告を著作権または権利の申し立てに使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツや不正な公開動作を特定するために、自動チェック、統計的な不正利用シグナル、ユーザー報告、スタッフレビューを使用する場合があります。シグナルだけで不正利用が証明されるわけではありません。ClawHub がレビューの必要な対象を判断する助けになります。

当社は次を行う場合があります。

- 違反リスティングを非表示、保留、削除、ソフト削除、またはリソースタイプでサポートされている場合はハード削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- APIトークンを取り消す
- 関連コンテンツをソフト削除する
- 公開アクセスを制限する
- 繰り返しまたは重大な違反者をBANする

明白な不正利用に対して、事前警告を保証するものではありません。報告、モデレーション保留、非表示のリスティング、BAN、アカウントの健全性については、[モデレーションとアカウント安全性](/clawhub/moderation)を参照してください。
