# 利用規定

ClawHub は OpenClaw の Skills、Plugin、パッケージ、マーケットプレイスメタデータをホストします。
このページは、コンテンツや公開時の挙動が ClawHub に適しているかを判断するために使用してください。

これらのルールは、掲載項目が何をするか、ユーザーに何の実行を求めるか、自身をどう表現するか、公開者が ClawHub の発見、インストール、信頼のサーフェスをどう使うかに適用されます。モデレーション状態とアカウントの状態については、[モデレーションとアカウント安全性](/clawhub/moderation)を参照してください。著作権またはその他の権利に関する申し立てについては、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者生産性                           | 掲載項目が、ユーザーによるソフトウェアの構築、テスト、移行、デバッグ、文書化、運用に役立つ。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのある操作にレビュー、ドライラン、プレビュー、確認の経路が含まれている。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが承認済みレビュー向けとして位置付けられ、証拠を保持し、人間による承認境界を明確にしている。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明なセットアップ、明示的な権限を使用している。                                            |
| 保守されているカタログ                              | 各掲載項目が区別可能で、有用で、正確に説明され、合理的に保守されている。                                                |

文脈が重要です。同じトピックでも、限定的な防御的用途や同意に基づく設定では許容される一方で、不正利用ワークフローとしてパッケージ化されている場合は許容されないことがあります。

## 禁止されるコンテンツ

ClawHub は、不正利用、欺瞞、安全でない実行、権利侵害を主目的とするコンテンツをホストしません。

| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| プラットフォームの悪用とBAN回避                              | BAN 後のステルスアカウント、アカウントのウォーミングやファーミング、偽のエンゲージメント、複数アカウント自動化、大量投稿、スパムボット、検知回避を目的として構築された自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書や請求書、欺瞞的な決済フロー、スキャムのアウトリーチ、偽の社会的証明、詐欺のための合成アイデンティティワークフロー、明確な人間の承認なしに支出や課金を行うツール。                                                                                                                    |
| プライバシー侵害的なエンリッチメントまたは監視                 | スパム目的の連絡先スクレイピング、ドキシング、ストーキング、未承諾のアウトリーチと組み合わされたリード抽出、秘密裏の監視、同意のない生体照合、漏洩データや侵害ダンプの使用。                                                                                                                  |
| 同意のないなりすましまたはアイデンティティ操作       | フェイススワップ、デジタルツイン、クローン化されたインフルエンサー、偽の人格、その他なりすましや誤認誘導に使われるツール。                                                                                                                                                                                                 |
| 明示的な性的コンテンツまたは安全機能を無効化した成人向け生成 | NSFW 画像、動画、またはコンテンツ生成、サードパーティ API をラップする成人向けコンテンツラッパー、明示的な性的コンテンツを主目的とする掲載項目。                                                                                                                                                       |
| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確なレビュー可能性なしにダウンロードしたコンテンツを `sh` または `bash` で実行するような pipe-to-shell インストーラー、未申告のシークレットまたは秘密鍵要件、明確なレビュー可能性なしのリモート `npx @latest` 実行、掲載項目の実行に本当に必要なものを隠すメタデータ。 |
| 著作権侵害または権利侵害のある素材           | 他者の Skills、Plugin、ドキュメント、ブランド資産、プロプライエタリコードを許可なく再公開すること、ライセンス条件に違反すること、元の作者や公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイス行為

ClawHub は、公開者によるマーケットプレイスの利用方法もレビューします。発見、指標、信頼シグナル、モデレーションシステム、ユーザーの注意を操作するために ClawHub を使用しないでください。

禁止されるマーケットプレイス行為には次が含まれます。

- 実際のユーザー価値があるように見えない、低労力、重複、プレースホルダー、または機械生成の掲載項目を大量に公開すること
- ほぼ同一の Skills や Plugin で検索やカテゴリのサーフェスをあふれさせること
- 利用、保守、ソースの明確さ、意味のある差別化がほとんどない、またはまったくない掲載項目を数百件公開すること
- 自動化、自己インストールループ、偽アカウント、協調的活動、有償エンゲージメント、その他の非オーガニックな行為によって、インストール数、ダウンロード数、スター数、その他のエンゲージメント指標を人為的に水増しすること
- モデレーション、BAN、公開者制限、マーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有権、ソース、機能、セキュリティ態勢、インストール要件、他のプロジェクトや公開者との関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は自動的に不正利用とはみなされません。掲載項目が意味のある違いを持ち、正確に説明され、保守され、実際のユーザーに使われている場合、大規模なカタログは許容されます。大量であることが、薄い内容、重複、誤解を招く内容、未保守、または人為的に宣伝された掲載項目と組み合わさると、大規模カタログは信頼と安全性の問題になります。

## コンテンツの権利

ClawHub 上のコンテンツが自分の著作権またはその他の権利を侵害していると考える場合は、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。掲載項目が安全でない、悪意がある、または誤解を招く場合を除き、著作権または権利に関する申し立てに通常のマーケットプレイス報告を使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツや不正利用的な公開行為を特定するために、自動チェック、統計的な不正利用シグナル、ユーザー報告、スタッフレビューを使用することがあります。シグナル単独で不正利用が証明されるわけではありません。ClawHub がレビューの必要な対象を判断する助けになります。

当社は次を行うことがあります。

- 違反している掲載項目を非表示、保留、削除、ソフト削除、またはリソース種別でサポートされている場合はハード削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツをソフト削除する
- 公開アクセスを制限する
- 繰り返し違反する者または重大な違反者をBANする

明白な不正利用について、事前警告を行う執行を保証するものではありません。報告、モデレーション保留、非表示の掲載項目、BAN、アカウントの状態については、[モデレーションとアカウント安全性](/clawhub/moderation)を参照してください。
