# 許容される利用

ClawHub は OpenClaw 向けの Skills、Plugin、パッケージ、マーケットプレイスメタデータをホストします。
コンテンツや公開の挙動が ClawHub に属するかどうかを判断するために、このページを使用してください。

これらのルールは、リスティングが何を行うか、ユーザーに何の実行を求めるか、自身をどのように表現するか、公開者が ClawHub の発見、インストール、信頼のサーフェスをどのように使用するかに適用されます。モデレーション状態とアカウントの状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。著作権またはその他の権利に関する申し立てについては、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | リスティングが、ユーザーによるソフトウェアのビルド、テスト、移行、デバッグ、文書化、運用を支援する。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのあるアクションにレビュー、ドライラン、プレビュー、または確認の経路が含まれている。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが承認済みレビュー向けとして位置付けられ、証拠を保持し、人間の承認境界を明確に保つ。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明なセットアップ、明示的な権限を使用する。                                            |
| 保守されているカタログ                              | 各リスティングが明確に区別され、有用で、正確に説明され、妥当に保守されている。                                                |

文脈が重要です。同じトピックでも、狭い防御的な設定や同意に基づく設定では許容される一方、不正利用ワークフローとしてパッケージ化されている場合は許容されません。

| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| プラットフォーム不正利用とBAN回避                              | BAN後のステルスアカウント、アカウントのウォーミングまたはファーミング、偽のエンゲージメント、複数アカウント自動化、大量投稿、スパムボット、または検出回避のために構築された自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書や請求書、欺瞞的な支払いフロー、スキャム勧誘、偽の社会的証明、不正のための合成IDワークフロー、または明確な人間の承認がない支出・課金ツール。                                                                                                                    |
| プライバシー侵害的なエンリッチメントまたは監視                 | スパムのための連絡先スクレイピング、晒し、ストーキング、未承諾のアウトリーチと組み合わされたリード抽出、秘密の監視、同意のない生体照合、または漏えいデータや侵害ダンプの使用。                                                                                                                  |
| 同意のないなりすましまたはID操作       | フェイススワップ、デジタルツイン、クローン化されたインフルエンサー、偽のペルソナ、またはなりすましや誤認誘導に使用されるその他のツール。                                                                                                                                                                                                 |
| 明示的な性的コンテンツまたは安全制限を無効化した成人向け生成 | NSFW画像、動画、またはコンテンツ生成、サードパーティAPIを包む成人向けコンテンツラッパー、または明示的な性的コンテンツを主目的とするリスティング。                                                                                                                                                       |
| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、ダウンロードしたコンテンツを明確なレビュー可能性なしに `sh` または `bash` で実行するようなpipe-to-shellインストーラー、未申告のシークレットまたは秘密鍵要件、明確なレビュー可能性のないリモート `npx @latest` 実行、またはリスティングの実行に本当に必要なものを隠すメタデータ。 |
| 著作権侵害または権利侵害にあたる素材           | 他者のSkill、Plugin、ドキュメント、ブランドアセット、またはプロプライエタリコードを許可なく再公開すること、ライセンス条件に違反すること、または元の作者や公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイス上の挙動

ClawHub は、公開者がマーケットプレイスをどのように使用するかもレビューします。発見、指標、信頼シグナル、モデレーションシステム、またはユーザーの注意を操作する目的で ClawHub を使用しないでください。

禁止されるマーケットプレイス上の挙動には、次が含まれます。

- 実際のユーザー価値があるように見えない、低労力、重複、プレースホルダー、または機械生成のリスティングを大量に一括公開すること
- ほぼ同一のSkillsまたはPluginで検索やカテゴリのサーフェスを埋め尽くすこと
- 利用、保守、ソースの明確さ、または意味のある差別化がほとんど、またはまったくない数百件のリスティングを公開すること
- 自動化、自己インストールループ、偽アカウント、協調活動、有料エンゲージメント、またはその他の非オーガニックな挙動を通じて、インストール、ダウンロード、スター、またはその他のエンゲージメント指標を人為的につり上げること
- モデレーション、BAN、公開者制限、またはマーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有権、ソース、機能、セキュリティ体制、インストール要件、または別のプロジェクトや公開者との関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は自動的に不正利用になるわけではありません。リスティングが意味のある形で異なり、正確に説明され、保守され、実際のユーザーに使用されている場合、大規模なカタログは許容されます。大量であることが、内容の薄い、重複した、誤解を招く、保守されていない、または人為的に宣伝されたリスティングと組み合わさると、大規模なカタログは信頼と安全性の問題になります。

## コンテンツの権利

ClawHub 上のコンテンツが自分の著作権またはその他の権利を侵害していると思われる場合は、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。リスティングが同時に安全でない、悪意がある、または誤解を招く場合を除き、著作権または権利に関する申し立てには通常のマーケットプレイス報告を使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツや不正な公開挙動を特定するために、自動チェック、統計的な不正利用シグナル、ユーザー報告、スタッフレビューを使用する場合があります。シグナル自体が不正利用を証明するわけではありません。ClawHub がレビューの必要な対象を判断する助けになります。

実施する場合がある措置:

- 違反するリスティングを非表示、保留、削除、ソフト削除、またはリソース種別でサポートされている場合はハード削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- APIトークンを取り消す
- 関連コンテンツをソフト削除する
- 公開アクセスを制限する
- 繰り返し違反する、または重大な違反を行う者をBANする

明白な不正利用に対して、警告を先に行う執行は保証しません。報告、モデレーション保留、非表示リスティング、BAN、アカウントの状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。
