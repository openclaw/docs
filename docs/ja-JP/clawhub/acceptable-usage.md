# 許容される利用

ClawHub は OpenClaw の Skills、plugins、packages、marketplace メタデータをホストします。
このページを使って、コンテンツまたは公開動作が ClawHub に属するかどうかを判断してください。

これらのルールは、リスティングが何をするか、ユーザーに何の実行を求めるか、どのように自らを表現するか、そして公開者が ClawHub の発見、インストール、信頼のサーフェスをどのように使うかに適用されます。モデレーション状態とアカウントの状態については、[モデレーションとアカウントの安全性](/ja-JP/clawhub/moderation)を参照してください。著作権またはその他の権利侵害の申し立てについては、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | リスティングが、ユーザーによるソフトウェアのビルド、テスト、移行、デバッグ、文書化、運用を支援する。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのあるアクションにレビュー、ドライラン、プレビュー、確認の経路が含まれる。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが認可されたレビュー向けとして位置づけられ、証拠を保持し、人間の承認境界を明確に保つ。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明なセットアップ、明示的な権限を使用する。                                            |
| メンテナンスされているカタログ                              | 各リスティングが個別性を持ち、有用で、正確に説明され、合理的にメンテナンスされている。                                                |

文脈が重要です。同じトピックでも、狭い防御的な設定や同意に基づく設定では許容される場合があり、不正利用ワークフローとしてパッケージ化されている場合は許容されません。

## 禁止されるコンテンツ

ClawHub は、主な目的が不正利用、欺瞞、安全でない実行、または権利侵害であるコンテンツをホストしません。

| カテゴリ                                                    | 許可されないもの                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 不正アクセスまたはセキュリティ回避                      | 認証回避、アカウント乗っ取り、レート制限の不正利用、ライブ通話またはエージェントの乗っ取り、再利用可能なセッションの窃取、未承認ユーザー向けのペアリングフローの自動承認。                                                                                                                                                   |
| プラットフォームの不正利用と禁止回避                              | 禁止後のステルスアカウント、アカウントのウォーミングやファーミング、偽のエンゲージメント、複数アカウントの自動化、大量投稿、スパムボット、または検出回避を目的として作られた自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書や請求書、欺瞞的な支払いフロー、スキャムのアウトリーチ、偽の社会的証明、詐欺用の合成 ID ワークフロー、または明確な人間の承認なしに支出や請求を行うツール。                                                                                                                    |
| プライバシー侵害的なエンリッチメントまたは監視                 | スパム目的の連絡先スクレイピング、晒し、ストーキング、未承諾アウトリーチと組み合わされたリード抽出、秘密裏の監視、非同意の生体情報マッチング、または漏えいデータや侵害ダンプの使用。                                                                                                                  |
| 非同意のなりすましまたはアイデンティティ操作       | フェイススワップ、デジタルツイン、クローン化されたインフルエンサー、偽のペルソナ、またはなりすましや誤解を招くために使われるその他のツール。                                                                                                                                                                                                 |
| 露骨な性的コンテンツまたは安全機能を無効化した成人向け生成 | NSFW 画像、動画、またはコンテンツ生成、サードパーティ API の成人向けコンテンツラッパー、または主な目的が露骨な性的コンテンツであるリスティング。                                                                                                                                                       |
| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確にレビューできない状態でダウンロードしたコンテンツを `sh` や `bash` で実行するなどの pipe-to-shell インストーラー、未申告のシークレットまたは秘密鍵要件、明確にレビューできないリモート `npx @latest` 実行、またはリスティングの実行に本当に必要なものを隠すメタデータ。 |
| 著作権侵害または権利侵害にあたる素材           | 他者の skill、plugin、docs、ブランドアセット、またはプロプライエタリコードを許可なく再公開すること、ライセンス条件に違反すること、または元の作者や公開者になりすますこと。                                                                                                                            |

## 禁止されるmarketplace 行為

ClawHub は、公開者が marketplace をどのように使用するかもレビューします。ClawHub を使って、発見、メトリクス、信頼シグナル、モデレーションシステム、またはユーザーの注意を操作してはなりません。

禁止される marketplace 行為には以下が含まれます。

- 実際のユーザー価値があるように見えない、手間の少ない、重複した、プレースホルダーの、または機械生成のリスティングを大量に公開すること
- ほぼ同一の skills や plugins で検索またはカテゴリサーフェスを氾濫させること
- 利用、メンテナンス、ソースの明確さ、または意味のある差別化がほとんど、またはまったくない数百件のリスティングを公開すること
- 自動化、自己インストールループ、偽アカウント、協調的な活動、有料エンゲージメント、またはその他の非オーガニックな行為を通じて、インストール、ダウンロード、スター、その他のエンゲージメント指標を人為的に膨らませること
- モデレーション、禁止、公開者制限、または marketplace レビューを回避するためにアカウントを作成またはローテーションすること
- 所有権、ソース、機能、セキュリティ態勢、インストール要件、または別のプロジェクトや公開者との関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は自動的に不正利用とは見なされません。リスティングに意味のある違いがあり、正確に説明され、メンテナンスされ、実際のユーザーに使用されている場合、大規模なカタログは許容されます。大量であることが、内容の薄さ、重複、誤解を招く内容、メンテナンス不足、または人為的な宣伝と組み合わさると、大規模なカタログは信頼と安全性の問題になります。

## コンテンツの権利

ClawHub 上のコンテンツが著作権またはその他の権利を侵害していると考える場合は、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。リスティングが同時に安全でない、悪意がある、または誤解を招く場合を除き、著作権または権利侵害の申し立てに通常の marketplace レポートを使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツまたは不正な公開行為を特定するために、自動チェック、統計的な不正利用シグナル、ユーザーレポート、スタッフレビューを使用する場合があります。シグナルはそれだけで不正利用を証明するものではありません。ClawHub がレビューの必要な対象を判断する助けになります。

当社は以下を行う場合があります。

- 違反するリスティングを非表示、保留、削除、ソフト削除、またはリソースタイプでサポートされている場合はハード削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツをソフト削除する
- 公開アクセスを制限する
- 繰り返し違反する者または重大な違反者を禁止する

明白な不正利用に対して、必ず警告を先に行う執行を保証するものではありません。レポート、モデレーション保留、非表示のリスティング、禁止、アカウントの状態については、[モデレーションとアカウントの安全性](/ja-JP/clawhub/moderation)を参照してください。
