# 利用許容基準

ClawHub は、OpenClaw の Skills、Plugin、パッケージ、マーケットプレイスメタデータをホストします。
コンテンツや公開時の挙動が ClawHub に属するかどうかを判断するために、このページを使用してください。

これらのルールは、リスティングが何をするか、ユーザーに何の実行を求めるか、自身をどのように表現するか、公開者が ClawHub の発見、インストール、信頼のサーフェスをどのように使用するかに適用されます。モデレーション状態とアカウントの状態については、[モデレーションとアカウント安全性](/clawhub/moderation)を参照してください。著作権またはその他の権利に関する申し立てについては、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | リスティングが、ユーザーによるソフトウェアの構築、テスト、移行、デバッグ、ドキュメント化、運用に役立つ。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのある操作にレビュー、ドライラン、プレビュー、確認の経路が含まれる。 |
| 防御的セキュリティ、モデレーション、不正使用レビュー | ツールが承認済みレビュー向けとして位置付けられ、証拠を保持し、人間の承認境界を明確に保つ。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明なセットアップ、明示的な権限を使用する。                                            |
| メンテナンスされているカタログ                              | 各リスティングが個別性、有用性、正確な説明を備え、合理的にメンテナンスされている。                                                |

文脈が重要です。同じトピックでも、狭い防御的または同意に基づく設定では許容される場合があり、不正使用ワークフローとしてパッケージ化されている場合は許容されない場合があります。

## 禁止されるマーケットプレイス上の挙動

ClawHub は、公開者がマーケットプレイスをどのように使用するかもレビューします。発見、指標、信頼シグナル、モデレーションシステム、ユーザーの注目を操作するために ClawHub を使用しないでください。

禁止されるマーケットプレイス上の挙動には、次が含まれます。

- 実際のユーザー価値があるように見えない、低品質、重複、プレースホルダー、または機械生成のリスティングを大量に公開すること
- ほぼ同一の skills または plugins で検索やカテゴリのサーフェスをあふれさせること
- 使用状況、メンテナンス、ソースの明確性、意味のある差別化がほとんどまたはまったくないリスティングを数百件公開すること
- 自動化、自己インストールループ、偽アカウント、組織的な活動、有料エンゲージメント、その他の非オーガニックな挙動を通じて、インストール数、ダウンロード数、スター、その他のエンゲージメント指標を人為的に水増しすること
- モデレーション、BAN、公開者制限、マーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有者、ソース、機能、セキュリティ態勢、インストール要件、他のプロジェクトまたは公開者との関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は自動的に不正使用とはみなされません。大規模なカタログは、リスティングが意味のある違いを持ち、正確に説明され、メンテナンスされ、実際のユーザーに使用されている場合に許容されます。大量であることが、内容の薄い、重複した、誤解を招く、メンテナンスされていない、または人為的に宣伝されたリスティングと組み合わさると、大規模なカタログは信頼と安全性の問題になります。

## コンテンツ権利

ClawHub 上のコンテンツがあなたの著作権またはその他の権利を侵害していると考える場合は、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。リスティングが安全でない、悪意がある、または誤解を招く場合を除き、著作権または権利に関する申し立てに通常のマーケットプレイス報告を使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツまたは不正な公開行為を特定するために、自動チェック、統計的な不正使用シグナル、ユーザー報告、スタッフレビューを使用する場合があります。シグナルはそれ自体で不正使用を証明するものではありません。ClawHub がレビューの必要な対象を判断する助けになります。

当社は次を行う場合があります。

- 違反リスティングを非表示、保留、削除、ソフトデリート、またはリソースタイプでサポートされている場合はハードデリートする
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツをソフトデリートする
- 公開アクセスを制限する
- 繰り返し違反する者または重大な違反者を BAN する

明らかな不正使用に対して、警告を先に行う執行を保証するものではありません。報告、モデレーション保留、非表示リスティング、BAN、アカウントの状態については、[モデレーションとアカウント安全性](/clawhub/moderation)を参照してください。
