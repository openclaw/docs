# 利用規約上許容される使用

このページでは、ClawHub が許容する Skills とコンテンツの種類、およびホストしない不正利用ワークフローについて説明します。

これらのルールは意図的に実用性を重視しています。私たちが最も重視するのは、孤立したキーワードだけではなく、エンドツーエンドの不正利用ワークフローです。防御の回避、プラットフォームの悪用、人をだます行為、プライバシー侵害、または非同意の行為を可能にする目的で Skill が作られている場合、それは ClawHub に属しません。

## 最近のパターンで明示的に許容しているもの

- 実際のコンポーネント、セマンティックトークン、アクセシブルな状態、テスト済みのユーザーフローを使用するフロントエンドおよびデザインシステムの作業。
- 一回限りのマークアップではなく、インストール済みのソースコンポーネント、プロジェクトエイリアス、文書化されたバリアントを使用する shadcn/ui の構成。
- コメントを保持し、具体的な UI5 型を使用し、生成されたコントロールインターフェイスをレビュー可能な状態に保つ UI5 JavaScript から TypeScript への変換。
- 証拠を示し、人間による承認境界を明確に保つ、防御的なセキュリティレビュー、モデレーションツール、不正利用検出プロンプト。
- 明示的な認証情報、透明なセットアップ、ドライランまたはプレビューモードを備えた、個人またはチームアカウント向けの同意ベースのワークフロー自動化。
- サポート対象のソフトウェアにスコープを限定したドキュメント、移行ランブック、開発者ユーティリティ、テストフィクスチャ。

## 許容されないもの

- セキュリティ回避または不正アクセスのワークフロー。
  - 例: 認証バイパス、アカウント乗っ取り、CAPTCHA バイパス、Cloudflare またはボット対策の回避、レート制限バイパス、保護を打ち破るよう設計されたステルススクレイピング、ライブ通話またはエージェントの乗っ取り、再利用可能なセッション窃取、未承認ユーザー向けのペアリングフロー自動承認。

- プラットフォームの悪用とBAN回避。
  - 例: BAN 後のステルスアカウント、アカウントのウォーミング/ファーミング、偽のエンゲージメント、カルマやフォロワーの育成、複数アカウント自動化、大量投稿、スパムボット、検出を避けるために作られたマーケットプレイスまたはソーシャル自動化。

- 詐欺、スキャム、欺瞞的な金融ワークフロー。
  - 例: 偽の証明書、偽の請求書、欺瞞的な支払いフロー、スキャム目的のアウトリーチ、偽の社会的証明、明確な人間の承認と透明なコントロールなしに支出または課金を可能にするツール、または詐欺目的でアカウントを作成するために構築された合成アイデンティティのワークフロー。

- プライバシー侵害的なスクレイピング、エンリッチメント、または監視。
  - 例: スパム目的で連絡先情報を大規模にスクレイピングすること、晒し、ストーキング、未承諾アウトリーチと組み合わされたリード抽出、秘密裏の監視、明確な同意なしに使用される顔検索または生体認証マッチング、または漏えいデータや侵害ダンプの購入、公開、ダウンロード、運用化。

- 非同意のなりすまし、または欺瞞的なアイデンティティ操作。
  - 例: フェイススワップ、デジタルツイン、偽の人格、クローン化されたインフルエンサー、またはなりすましや誤認を目的として使われるその他のアイデンティティ操作ツール。

- 明示的な性的コンテンツ、および安全機能を無効化した成人向け生成。
  - 例: NSFW 画像/動画/コンテンツ生成、サードパーティ API を包む成人向けコンテンツラッパー、または明示的な性的コンテンツを主目的とする Skills。

- 隠された、安全でない、または誤解を招く実行要件。
  - 例: 難読化されたインストールコマンド、`curl | sh`、未申告のシークレット要件、未申告の秘密鍵使用、明確なレビュー可能性のないリモート `npx @latest` 実行、Skill が実行に本当に必要とするものを隠す誤解を招くメタデータ。

## 最近のパターンで明示的に許容していないもの

- 「マーケットプレイスでのBAN後にステルス販売者アカウントを作成する。」
- 「Telegram のペアリングを変更し、未承認ユーザーが自動的にペアリングコードを受け取れるようにする。」
- 「検出不能な自動化で Reddit/Twitter アカウントを育成する。」
- 「任意の用途向けに専門的な証明書や請求書を生成する。」
- 「安全チェックを無効にして NSFW コンテンツを生成する。」
- 「リードをスクレイピングし、連絡先をエンリッチし、大規模にコールドアウトリーチを開始する。」
- 「漏えいデータや侵害ダンプを購入、公開、またはダウンロードする。」
- 「合成アイデンティティまたは CAPTCHA 解決を使ってメールまたはソーシャルアカウントを一括作成する。」

## レビュアー向けメモ

- コンテキストが重要です。同じトピックでも、狭い防御的または同意ベースの設定では正当になり得ますが、不正利用ワークフローとしてパッケージ化されている場合は許容されません。
- Skill が回避、欺瞞、または非同意の使用に明確に最適化されている場合、私たちは対応する方向に傾くべきです。
- これらのカテゴリで繰り返しアップロードが行われた場合、コンテンツの非表示およびアカウントのBANの根拠となります。

## 執行

- 違反する Skills を非表示、削除、またはハード削除する場合があります。
- トークンを取り消し、関連コンテンツをソフト削除し、繰り返し違反または重大な違反を行うアカウントをBANする場合があります。
- 明白な不正利用に対して、警告を先に行う執行を保証するものではありません。
