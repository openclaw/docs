# 許容される利用

このページでは、ClawHub が許容するスキルとコンテンツの種類、およびホストしない不正利用ワークフローについて説明します。

これらのルールは意図的に実用性を重視しています。私たちが最も重視するのは、孤立したキーワードだけでなく、エンドツーエンドの不正利用ワークフローです。防御の回避、プラットフォームの悪用、人への詐欺、プライバシー侵害、または同意のない行為を可能にする目的でスキルが作られている場合、それは ClawHub にふさわしくありません。

## 明示的に許容する最近のパターン

- 実コンポーネント、セマンティックトークン、アクセシブルな状態、テスト済みのユーザーフローを使用するフロントエンドおよびデザインシステム作業。
- 単発のマークアップではなく、インストール済みのソースコンポーネント、プロジェクトエイリアス、文書化されたバリアントを使用する shadcn/ui の構成。
- コメントを保持し、具体的な UI5 型を使用し、生成されたコントロールインターフェースをレビュー可能に保つ UI5 JavaScript から TypeScript への変換。
- 証拠を示し、人間による承認境界を明確に保つ、防御的なセキュリティレビュー、モデレーションツール、不正利用検出プロンプト。
- 明示的な認証情報、透明なセットアップ、ドライランまたはプレビューモードを備えた、個人またはチームアカウント向けの同意に基づくワークフロー自動化。
- サポート対象ソフトウェアにスコープを限定した、ドキュメント、移行ランブック、開発者ユーティリティ、テストフィクスチャ。

## 許容されないもの

- セキュリティバイパスまたは不正アクセスのワークフロー。
  - 例: 認証バイパス、アカウント乗っ取り、CAPTCHA バイパス、Cloudflare またはボット対策の回避、レート制限バイパス、保護を破るよう設計されたステルススクレイピング、ライブ通話またはエージェントの乗っ取り、再利用可能なセッション窃取、未承認ユーザー向けのペアリングフローの自動承認。

- プラットフォームの悪用とBAN回避。
  - 例: BAN後のステルスアカウント、アカウントのウォーミング/ファーミング、偽のエンゲージメント、カルマまたはフォロワーの育成、複数アカウントの自動化、大量投稿、スパムボット、検出回避を目的として作られたマーケットプレイスまたはソーシャル自動化。

- 詐欺、スキャム、欺瞞的な金融ワークフロー。
  - 例: 偽の証明書、偽の請求書、欺瞞的な支払いフロー、スキャムアウトリーチ、偽のソーシャルプルーフ、明確な人間の承認と透明な制御なしに支出または課金を可能にするツール、または詐欺用アカウント作成を目的とした合成IDワークフロー。

- プライバシーを侵害するスクレイピング、エンリッチメント、または監視。
  - 例: スパム目的で連絡先情報を大規模にスクレイピングすること、晒し行為、ストーキング、未承諾アウトリーチと組み合わせたリード抽出、秘密裏の監視、明確な同意なしに使用される顔検索または生体認証照合、流出データまたは侵害ダンプの購入、公開、ダウンロード、運用化。

- 同意のないなりすまし、または欺瞞的なID操作。
  - 例: なりすましや誤認を目的として使用される、フェイススワップ、デジタルツイン、偽のペルソナ、クローン化されたインフルエンサー、その他のID操作ツール。

- 露骨な性的コンテンツ、および安全機能が無効化された成人向け生成。
  - 例: NSFW 画像/動画/コンテンツ生成、サードパーティ API を成人向けコンテンツ用にラップするもの、または主目的が露骨な性的コンテンツであるスキル。

- 隠された、安全でない、または誤解を招く実行要件。
  - 例: 難読化されたインストールコマンド、`curl | sh`、未申告のシークレット要件、未申告の秘密鍵使用、明確なレビュー可能性のないリモート `npx @latest` 実行、スキルの実行に本当に必要なものを隠す誤解を招くメタデータ。

## 明示的に許容しない最近のパターン

- 「マーケットプレイスでのBAN後にステルス販売者アカウントを作成する。」
- 「未承認ユーザーが自動的にペアリングコードを受け取るよう Telegram のペアリングを変更する。」
- 「検出不能な自動化で Reddit/Twitter アカウントを育成する。」
- 「任意の用途向けに専門的な証明書または請求書を生成する。」
- 「安全チェックを無効にして NSFW コンテンツを生成する。」
- 「リードをスクレイピングし、連絡先をエンリッチし、大規模にコールドアウトリーチを開始する。」
- 「流出データまたは侵害ダンプを購入、公開、またはダウンロードする。」
- 「合成IDまたは CAPTCHA 解除を使ってメールアカウントやソーシャルアカウントを一括作成する。」

## レビュアー向けメモ

- 文脈が重要です。同じトピックでも、狭い範囲の防御的または同意に基づく設定では正当であり、不正利用ワークフローとしてパッケージ化されている場合は許容されないことがあります。
- 回避、欺瞞、または同意のない利用に明確に最適化されたスキルについては、対応を優先するべきです。
- これらのカテゴリでの繰り返しアップロードは、コンテンツの非表示およびアカウントBANの根拠となります。

## 執行

- 違反スキルを非表示、削除、またはハード削除する場合があります。
- トークンを取り消し、関連コンテンツをソフト削除し、反復的または重大な違反者をBANする場合があります。
- 明白な不正利用について、警告を先に行う執行を保証しません。
