# 許容される利用

ClawHub は OpenClaw 向けの Skills、Plugin、パッケージ、マーケットプレイスメタデータをホストします。
このページは、コンテンツや公開時の挙動が ClawHub に属するかどうかを判断するために使用します。

これらのルールは、リスティングが行うこと、ユーザーに実行を求めること、リスティング自体の表現方法、および公開者が ClawHub の発見、インストール、信頼サーフェスをどのように使うかに適用されます。モデレーション状態とアカウントの状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。著作権またはその他の権利の申し立てについては、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | リスティングが、ユーザーによるソフトウェアの構築、テスト、移行、デバッグ、ドキュメント化、運用を支援する。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのある操作にレビュー、ドライラン、プレビュー、または確認のパスが含まれる。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが承認されたレビュー向けとして位置付けられ、証拠を保持し、人間による承認の境界を明確に保つ。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明なセットアップ、明示的な権限を使用する。                                            |
| メンテナンスされているカタログ                              | 各リスティングが明確に区別され、有用で、正確に説明され、合理的にメンテナンスされている。                                                |

文脈が重要です。同じトピックでも、狭い防御的または同意に基づく設定では許容される一方、不正利用ワークフローとしてパッケージ化されている場合は許容されません。

## 禁止されるコンテンツ

ClawHub は、主な目的が不正利用、欺瞞、安全でない実行、または権利侵害であるコンテンツをホストしません。

| カテゴリ                                                    | 許可されないもの                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 不正アクセスまたはセキュリティ回避                      | 認証回避、アカウント乗っ取り、レート制限の悪用、ライブ通話またはエージェントの乗っ取り、再利用可能なセッションの窃取、または未承認ユーザー向けペアリングフローの自動承認。                                                                                                                                                   |
| プラットフォームの不正利用と禁止回避                              | 禁止後のステルスアカウント、アカウントのウォーミングまたはファーミング、偽のエンゲージメント、複数アカウントの自動化、大量投稿、スパムボット、または検出回避を目的として構築された自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書または請求書、欺瞞的な支払いフロー、スキャムのアウトリーチ、偽のソーシャルプルーフ、詐欺のための合成 ID ワークフロー、または明確な人間の承認なしに支出や課金を行うツール。                                                                                                                    |
| プライバシー侵害的なエンリッチメントまたは監視                 | スパム目的の連絡先スクレイピング、ドクシング、ストーキング、未承諾のアウトリーチと組み合わされたリード抽出、秘密裏の監視、同意のない生体照合、または漏えいデータや侵害ダンプの使用。                                                                                                                  |
| 同意のないなりすましまたは ID 操作       | フェイススワップ、デジタルツイン、クローン化されたインフルエンサー、偽のペルソナ、またはなりすましや誤認を目的として使用されるその他のツール。                                                                                                                                                                                                 |
| 明示的な性的コンテンツまたは安全機能を無効にした成人向け生成 | NSFW 画像、動画、またはコンテンツ生成、サードパーティ API の周りの成人向けコンテンツラッパー、または主な目的が明示的な性的コンテンツであるリスティング。                                                                                                                                                       |
| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確なレビュー可能性なしに `sh` または `bash` でダウンロードコンテンツを実行するような pipe-to-shell インストーラー、未申告のシークレットまたは秘密鍵の要件、明確なレビュー可能性のないリモート `npx @latest` 実行、またはリスティングの実行に本当に必要なものを隠すメタデータ。 |
| 著作権を侵害する、または権利を侵害する素材           | 他者の Skills、Plugin、ドキュメント、ブランドアセット、またはプロプライエタリコードを許可なく再公開すること、ライセンス条件に違反すること、または元の著者や公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイス上の挙動

ClawHub は、公開者がマーケットプレイスをどのように使用するかもレビューします。発見、メトリクス、信頼シグナル、モデレーションシステム、またはユーザーの注意を操作するために ClawHub を使用しないでください。

禁止されるマーケットプレイス上の挙動には以下が含まれます。

- 実際のユーザー価値があるように見えない、低労力、重複、プレースホルダー、または機械生成のリスティングを大量に公開すること
- ほぼ同一の Skills または Plugin で検索やカテゴリサーフェスを埋め尽くすこと
- 使用状況、メンテナンス、ソースの明確さ、または意味のある差別化がほとんどまたはまったくないリスティングを数百件公開すること
- 自動化、自己インストールループ、偽アカウント、協調的活動、有料エンゲージメント、またはその他の非オーガニックな挙動を通じて、インストール、ダウンロード、スター、またはその他のエンゲージメントメトリクスを人為的に膨らませること
- モデレーション、禁止、公開者制限、またはマーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有権、ソース、機能、セキュリティ態勢、インストール要件、または別のプロジェクトや公開者との提携についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は自動的に不正利用になるわけではありません。リスティングが意味のある違いを持ち、正確に説明され、メンテナンスされ、実際のユーザーに使用されている場合、大規模なカタログは許容されます。大量であることが、内容の薄さ、重複、誤解を招く表現、メンテナンス不足、または人為的な宣伝と組み合わさると、大規模なカタログは信頼と安全性の問題になります。

## コンテンツの権利

ClawHub 上のコンテンツがあなたの著作権またはその他の権利を侵害していると思われる場合は、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。リスティングが安全でない、悪意がある、または誤解を招く場合を除き、著作権または権利の申し立てに通常のマーケットプレイス報告を使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツまたは不正な公開行為を特定するために、自動チェック、統計的な不正利用シグナル、ユーザー報告、スタッフレビューを使用する場合があります。シグナル自体が不正利用を証明するわけではありません。ClawHub がレビュー対象を判断する助けになります。

当社は以下を行う場合があります。

- 違反リスティングを非表示、保留、削除、ソフト削除、またはリソースタイプでサポートされている場合は物理削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツをソフト削除する
- 公開アクセスを制限する
- 繰り返しまたは重大な違反者を禁止する

明白な不正利用に対して、警告を先行する執行を保証するものではありません。報告、モデレーション保留、非表示のリスティング、禁止、アカウントの状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。
