# 利用可能な用途

ClawHub は、OpenClaw 向けの Skills、Plugin、パッケージ、マーケットプレイスのメタデータをホストします。
コンテンツや公開の振る舞いが ClawHub に属するかどうかを判断するために、このページを使用してください。

これらのルールは、掲載が何をするか、ユーザーに何を実行するよう求めるか、自身をどのように表現するか、公開者が ClawHub の発見、インストール、信頼の面をどのように使用するかに適用されます。モデレーション状態とアカウントの状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。著作権またはその他の権利の申し立てについては、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | 掲載が、ユーザーによるソフトウェアのビルド、テスト、移行、デバッグ、ドキュメント化、運用を支援する。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示されており、リスクのある操作にレビュー、ドライラン、プレビュー、確認の経路が含まれている。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが認可されたレビュー向けとして位置付けられ、証拠を保持し、人間の承認境界を明確に保つ。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明なセットアップ、明示的な権限を使用している。                                            |
| 保守されているカタログ                              | 各掲載が明確に区別され、有用で、正確に説明され、妥当に保守されている。                                                |

文脈が重要です。同じトピックでも、限定的な防御的設定や同意に基づく設定では許容される場合があり、不正利用ワークフローとしてパッケージ化されている場合は許容されません。

| プラットフォーム不正利用と禁止回避                              | 禁止後のステルスアカウント、アカウントの育成またはファーミング、偽のエンゲージメント、複数アカウントの自動化、大量投稿、スパムボット、検出回避を目的として作られた自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書または請求書、欺瞞的な支払いフロー、スキャム勧誘、偽のソーシャルプルーフ、詐欺のための合成アイデンティティワークフロー、明確な人間の承認なしでの支出または課金ツール。                                                                                                                    |

| 同意のないなりすましまたはアイデンティティ操作       | フェイススワップ、デジタルツイン、クローン化されたインフルエンサー、偽のペルソナ、またはなりすましや誤認誘導に使用されるその他のツール。                                                                                                                                                                                                 |
| 明示的な性的コンテンツまたは安全機能を無効化した成人向け生成 | NSFW 画像、動画、またはコンテンツ生成、サードパーティ API を囲む成人向けコンテンツラッパー、または主目的が明示的な性的コンテンツである掲載。                                                                                                                                                       |
| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、ダウンロードしたコンテンツを明確なレビュー可能性なしに `sh` または `bash` で実行する pipe-to-shell インストーラー、未申告のシークレットまたは秘密鍵要件、明確なレビュー可能性のないリモート `npx @latest` 実行、掲載の実行に本当に必要なものを隠すメタデータ。 |
| 著作権侵害または権利侵害にあたる素材           | 他者の Skill、Plugin、ドキュメント、ブランド資産、またはプロプライエタリコードを許可なく再公開すること、ライセンス条件に違反すること、元の作者または公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイス上の振る舞い

ClawHub は、公開者がマーケットプレイスをどのように使用するかも審査します。ClawHub を使用して、発見、メトリクス、信頼シグナル、モデレーションシステム、またはユーザーの注意を操作しないでください。

禁止されるマーケットプレイス上の振る舞いには、次が含まれます。

- 実際のユーザー価値があるように見えない、手間の少ない、重複した、プレースホルダー的な、または機械生成の掲載を大量に一括公開すること
- ほぼ同一の Skills または Plugin で検索面やカテゴリ面をあふれさせること
- 利用、保守、ソースの明確さ、または意味のある差別化がほとんどない、またはまったくない掲載を数百件公開すること
- 自動化、自己インストールのループ、偽アカウント、協調的活動、有償エンゲージメント、またはその他の非オーガニックな振る舞いを通じて、インストール、ダウンロード、スター、またはその他のエンゲージメントメトリクスを人為的に水増しすること
- モデレーション、禁止、公開者制限、またはマーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有権、ソース、機能、セキュリティ態勢、インストール要件、または別のプロジェクトや公開者との関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は自動的に不正利用とはみなされません。掲載が意味のある形で異なり、正確に説明され、保守され、実際のユーザーに使用されている場合、大規模なカタログは許容されます。大量性が、内容の薄い、重複した、誤解を招く、保守されていない、または人為的に宣伝された掲載と組み合わさると、大規模カタログは信頼と安全性の問題になります。

## コンテンツ権利

ClawHub 上のコンテンツが著作権またはその他の権利を侵害していると思われる場合は、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。掲載が安全でない、悪意がある、または誤解を招く場合を除き、著作権または権利の申し立てに通常のマーケットプレイスレポートを使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツまたは不正な公開行為を特定するために、自動チェック、統計的な不正利用シグナル、ユーザーレポート、スタッフレビューを使用する場合があります。シグナルはそれ自体で不正利用を証明するものではありません。ClawHub がレビューの必要な対象を判断するのに役立ちます。

当社は次を行う場合があります。

- 違反している掲載を非表示、保留、削除、ソフト削除、またはリソースタイプでサポートされている場合はハード削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツをソフト削除する
- 公開アクセスを制限する
- 繰り返しまたは重大な違反者を禁止する

明らかな不正利用に対して、警告を先に行う執行を保証するものではありません。レポート、モデレーション保留、非表示の掲載、禁止、アカウントの状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。
