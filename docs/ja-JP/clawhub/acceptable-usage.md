# 許容される利用

このページでは、ClawHub が許容するスキルとコンテンツの種類、およびホストしない不正利用ワークフローについて説明します。

これらのルールは、意図的に実用面を重視しています。私たちは、孤立したキーワードだけでなく、エンドツーエンドの不正利用ワークフローを最も重視します。防御の回避、プラットフォームの不正利用、人をだますこと、プライバシーの侵害、または非同意の行動を可能にするためにスキルが作られている場合、それは ClawHub に属しません。

## 明示的に許容する最近のパターン

- 実際のコンポーネント、セマンティックトークン、アクセシブルな状態、テスト済みのユーザーフローを使用するフロントエンドおよびデザインシステム作業。
- 一度限りのマークアップではなく、インストール済みのソースコンポーネント、プロジェクトエイリアス、文書化されたバリアントを使用する shadcn/ui の構成。
- コメントを保持し、具体的な UI5 型を使用し、生成されたコントロールインターフェイスをレビュー可能に保つ UI5 の JavaScript から TypeScript への変換。
- 証拠を示し、人間の承認境界を明確に保つ、防御的なセキュリティレビュー、モデレーションツール、不正利用検出プロンプト。
- 明示的な認証情報、透明なセットアップ、ドライランまたはプレビューモードを備えた、個人またはチームアカウント向けの同意ベースのワークフロー自動化。
- サポート対象のソフトウェアにスコープを限定した、ドキュメント、移行ランブック、開発者ユーティリティ、テストフィクスチャ。

## 許容されないもの

- セキュリティ回避または不正アクセスのワークフロー。
  - 例: 認証バイパス、アカウント乗っ取り、CAPTCHA バイパス、Cloudflare またはボット対策の回避、レート制限バイパス、保護を打ち破る目的で設計されたステルススクレイピング、ライブ通話またはエージェントの乗っ取り、再利用可能なセッション窃取、未承認ユーザー向けペアリングフローの自動承認。

- プラットフォームの不正利用と BAN 回避。
  - 例: BAN 後のステルスアカウント、アカウントのウォーミングまたはファーミング、偽のエンゲージメント、カルマまたはフォロワーの育成、複数アカウントの自動化、大量投稿、スパムボット、検出を避けるために構築されたマーケットプレイスまたはソーシャル自動化。

- 詐欺、スキャム、欺瞞的な金融ワークフロー。
  - 例: 偽の証明書、偽の請求書、欺瞞的な支払いフロー、スキャムのアウトリーチ、偽の社会的証明、明確な人間の承認と透明な制御なしに支出または請求を可能にするツール、または詐欺用アカウントを作成するために構築された合成アイデンティティのワークフロー。

- プライバシーを侵害するスクレイピング、エンリッチメント、監視。
  - 例: スパム目的で連絡先情報を大規模にスクレイピングすること、ドクシング、ストーキング、未承諾アウトリーチと組み合わせたリード抽出、秘密裏の監視、明確な同意なく使用される顔検索または生体照合、漏えいデータや侵害ダンプの購入、公開、ダウンロード、または運用化。

- 非同意のなりすましまたは欺瞞的なアイデンティティ操作。
  - 例: 顔の入れ替え、デジタルツイン、偽のペルソナ、クローン化されたインフルエンサー、またはなりすましや誤認させるために使用されるその他のアイデンティティ操作ツール。

- 明示的な性的コンテンツおよび安全機能を無効化した成人向け生成。
  - 例: NSFW の画像、動画、コンテンツ生成、サードパーティ API を囲む成人向けコンテンツラッパー、または主目的が明示的な性的コンテンツであるスキル。

- 隠された、安全でない、または誤解を招く実行要件。
  - 例: 難読化されたインストールコマンド、`curl | sh`、未申告のシークレット要件、未申告の秘密鍵使用、明確なレビュー可能性がないリモート `npx @latest` 実行、スキルが実行に本当に必要とするものを隠す誤解を招くメタデータ。

## 明示的に許容しない最近のパターン

- 「マーケットプレイスで BAN された後にステルス出品者アカウントを作成する。」
- 「未承認ユーザーがペアリングコードを自動的に受け取るよう Telegram のペアリングを変更する。」
- 「検出されない自動化で Reddit/Twitter アカウントを育成する。」
- 「任意の用途向けに専門的な証明書または請求書を生成する。」
- 「安全チェックを無効化して NSFW コンテンツを生成する。」
- 「リードをスクレイピングし、連絡先をエンリッチし、大規模なコールドアウトリーチを開始する。」
- 「漏えいデータまたは侵害ダンプを購入、公開、またはダウンロードする。」
- 「合成アイデンティティまたは CAPTCHA 解決を使ってメールまたはソーシャルアカウントを一括作成する。」

## レビュー担当者向けの注意

- コンテキストは重要です。同じトピックでも、狭い防御的または同意ベースの設定では正当になり得ますが、不正利用ワークフローとしてパッケージ化されている場合は許容されません。
- スキルが回避、欺瞞、または非同意の使用に明らかに最適化されている場合は、対応を優先すべきです。
- これらのカテゴリで繰り返しアップロードが行われた場合、コンテンツの非表示やアカウントの BAN の根拠になります。

## 執行

- 違反するスキルを非表示、削除、または完全削除する場合があります。
- トークンを取り消し、関連コンテンツをソフト削除し、繰り返しまたは重大な違反を行うユーザーを BAN する場合があります。
- 明白な不正利用について、警告を先行する執行は保証しません。
