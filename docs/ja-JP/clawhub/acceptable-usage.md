# 許容される利用

このページでは、ClawHub が許容するスキルとコンテンツの種類、および ClawHub がホストしない不正利用ワークフローについて説明します。

これらのルールは、意図的に実用性を重視しています。単独のキーワードだけでなく、エンドツーエンドの不正利用ワークフローを最も重視します。防御の回避、プラットフォームの悪用、人をだますこと、プライバシーの侵害、または同意のない行動を可能にすることを目的としてスキルが作られている場合、それは ClawHub に適しません。

## 明示的に許容している最近のパターン

- 実際のコンポーネント、セマンティックトークン、アクセシブルな状態、テスト済みのユーザーフローを使用するフロントエンドおよびデザインシステムの作業。
- 単発のマークアップではなく、インストール済みのソースコンポーネント、プロジェクトエイリアス、文書化されたバリアントを使用する shadcn/ui の構成。
- コメントを保持し、具体的な UI5 型を使用し、生成されたコントロールインターフェイスをレビュー可能な状態に保つ UI5 JavaScript-to-TypeScript 変換。
- 証拠を示し、人間の承認境界を明確に保つ、防御的なセキュリティレビュー、モデレーションツール、不正利用検出プロンプト。
- 明示的な認証情報、透明なセットアップ、ドライランまたはプレビューモードを備えた、個人またはチームアカウント向けの同意ベースのワークフロー自動化。
- サポート対象のソフトウェアに範囲を限定したドキュメント、移行ランブック、開発者ユーティリティ、テストフィクスチャ。

## 許容されないこと

- セキュリティ回避または不正アクセスのワークフロー。
  - 例: 認証バイパス、アカウント乗っ取り、CAPTCHA バイパス、Cloudflare またはボット対策の回避、レート制限のバイパス、保護を破ることを目的としたステルススクレイピング、ライブ通話またはエージェントの乗っ取り、再利用可能なセッション窃取、承認されていないユーザー向けのペアリングフローの自動承認。

- プラットフォームの悪用および禁止回避。
  - 例: 禁止後のステルスアカウント、アカウントのウォーミング/ファーミング、偽のエンゲージメント、カルマまたはフォロワーの育成、複数アカウント自動化、大量投稿、スパムボット、検出回避を目的として作られたマーケットプレイスまたはソーシャル自動化。

- 詐欺、スキャム、および欺瞞的な金融ワークフロー。
  - 例: 偽の証明書、偽の請求書、欺瞞的な支払いフロー、スキャムの勧誘、偽の社会的証明、明確な人間の承認と透明な制御なしに支出や課金を可能にするツール、または詐欺用アカウントの作成を目的とした合成 ID ワークフロー。

- プライバシーを侵害するスクレイピング、エンリッチメント、または監視。
  - 例: スパム目的で連絡先詳細を大規模にスクレイピングすること、晒し行為、ストーキング、未承諾のアウトリーチと組み合わせたリード抽出、秘密裏の監視、明確な同意なしに使用される顔検索または生体照合、漏えいデータや侵害ダンプの購入、公開、ダウンロード、または運用化。

- 同意のないなりすましまたは欺瞞的な ID 操作。
  - 例: 顔の入れ替え、デジタルツイン、偽のペルソナ、クローン化されたインフルエンサー、またはなりすましや誤認誘導に使用されるその他の ID 操作ツール。

- 露骨な性的コンテンツおよび安全機能を無効化した成人向け生成。
  - 例: NSFW 画像/動画/コンテンツ生成、サードパーティ API の成人向けコンテンツラッパー、または露骨な性的コンテンツを主目的とするスキル。

- 隠れた、安全でない、または誤解を招く実行要件。
  - 例: 難読化されたインストールコマンド、`curl | sh`、未申告のシークレット要件、未申告の秘密鍵使用、明確なレビュー可能性のないリモート `npx @latest` 実行、スキルの実行に本当に必要なものを隠す誤解を招くメタデータ。

## 明示的に許容しない最近のパターン

- 「マーケットプレイスで禁止された後にステルス販売者アカウントを作成する。」
- 「未承認ユーザーがペアリングコードを自動的に受け取れるように Telegram のペアリングを変更する。」
- 「検出不能な自動化で Reddit/Twitter アカウントを育成する。」
- 「任意の用途向けに専門的な証明書または請求書を生成する。」
- 「安全チェックを無効にして NSFW コンテンツを生成する。」
- 「リードをスクレイピングし、連絡先をエンリッチし、大規模にコールドアウトリーチを開始する。」
- 「漏えいデータまたは侵害ダンプを購入、公開、またはダウンロードする。」
- 「合成 ID または CAPTCHA 解決を使ってメールアカウントやソーシャルアカウントを一括作成する。」

## レビュアー向けメモ

- 文脈が重要です。同じトピックでも、限定的な防御目的または同意ベースの設定では正当になり得ますが、不正利用ワークフローとしてパッケージ化されている場合は許容されません。
- スキルが回避、欺瞞、または同意のない使用に明確に最適化されている場合は、措置を取る方向に寄せるべきです。
- これらのカテゴリで繰り返しアップロードすることは、コンテンツの非表示およびアカウントの禁止の根拠になります。

## 執行

- 違反するスキルを非表示、削除、または完全削除する場合があります。
- トークンの取り消し、関連コンテンツのソフト削除、繰り返し違反する利用者または重大な違反者の禁止を行う場合があります。
- 明白な不正利用に対して、警告を先に行う執行を保証しません。
