# 許容される利用

ClawHub は、OpenClaw 向けの Skills、Plugin、パッケージ、マーケットプレイスメタデータをホストします。
このページは、コンテンツまたは公開の振る舞いが ClawHub に属するかどうかを判断するために使用します。

これらのルールは、掲載が何を行うか、ユーザーに何の実行を求めるか、自身をどのように表現するか、公開者が ClawHub の発見、インストール、信頼のサーフェスをどのように使用するかに適用されます。モデレーション状態とアカウントの状態については、[モデレーションとアカウントの安全性](/ja-JP/clawhub/moderation)を参照してください。著作権またはその他の権利の申し立てについては、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | 掲載が、ユーザーによるソフトウェアの構築、テスト、移行、デバッグ、文書化、運用を支援する。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのある操作にレビュー、ドライラン、プレビュー、または確認の経路が含まれている。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが許可されたレビュー向けとして位置づけられ、証拠を保持し、人間による承認の境界を明確に保っている。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明なセットアップ、明示的な権限を使用している。                                            |
| 維持管理されているカタログ                              | 各掲載がそれぞれ異なり、有用で、正確に説明され、合理的に維持管理されている。                                                |

文脈が重要です。同じトピックでも、狭い防御的な設定や同意に基づく設定では許容される場合があり、不正利用ワークフローとしてパッケージ化されている場合は許容されません。

| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書または請求書、欺瞞的な支払いフロー、スキャムのアウトリーチ、偽のソーシャルプルーフ、詐欺のための合成アイデンティティワークフロー、または明確な人間の承認を伴わない支出・課金ツール。                                                                                                                    |

| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確なレビュー可能性がないままダウンロードされたコンテンツを `sh` または `bash` で実行するような pipe-to-shell インストーラー、宣言されていないシークレットまたは秘密鍵の要件、明確なレビュー可能性がないリモート `npx @latest` 実行、または掲載が実行に本当に必要とするものを隠すメタデータ。 |
| 著作権侵害または権利侵害にあたる素材           | 他者の Skills、Plugin、ドキュメント、ブランド資産、またはプロプライエタリコードを許可なく再公開すること、ライセンス条件に違反すること、または元の作者や公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイス上の振る舞い

ClawHub は、公開者がマーケットプレイスをどのように使用するかもレビューします。発見、指標、信頼シグナル、モデレーションシステム、またはユーザーの注意を操作するために ClawHub を使用しないでください。

禁止されるマーケットプレイス上の振る舞いには、次が含まれます。

- 実際のユーザー価値があるように見えない、低労力、重複的、プレースホルダー、または機械生成の掲載を大量に一括公開すること
- ほぼ同一の Skills または Plugin で検索やカテゴリのサーフェスを埋め尽くすこと
- 利用、維持管理、ソースの明確さ、または意味のある差別化がほとんど、あるいはまったくない掲載を数百件公開すること
- 自動化、自己インストールのループ、偽アカウント、協調的な活動、有償エンゲージメント、またはその他の非オーガニックな振る舞いによって、インストール数、ダウンロード数、スター数、またはその他のエンゲージメント指標を人為的に水増しすること
- モデレーション、禁止措置、公開者制限、またはマーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有権、ソース、機能、セキュリティ姿勢、インストール要件、または別のプロジェクトや公開者との関係についてユーザーに誤解を与えること
- すでに非表示、削除、またはブロックされたコンテンツを、根本的な問題を修正せずに繰り返しアップロードすること

大量公開は自動的に不正利用とはみなされません。掲載が意味のある違いを持ち、正確に説明され、維持管理され、実際のユーザーに使用されている場合、大規模なカタログは許容されます。大量であることが、内容の薄い、重複的な、誤解を招く、維持管理されていない、または人為的に宣伝された掲載と組み合わさると、大規模なカタログは信頼と安全性の問題になります。

## コンテンツ権利

ClawHub 上のコンテンツが著作権またはその他の権利を侵害していると考える場合は、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。掲載が安全でない、悪意がある、または誤解を招く場合を除き、著作権または権利の申し立てに通常のマーケットプレイス報告を使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツや不正利用的な公開の振る舞いを特定するために、自動チェック、統計的な不正利用シグナル、ユーザー報告、スタッフレビューを使用する場合があります。シグナルはそれ自体で不正利用を証明するものではありません。ClawHub が何をレビューすべきかを判断する助けになります。

当社は次を行う場合があります。

- 違反する掲載を非表示、保留、削除、ソフト削除、またはリソース種別でサポートされている場合はハード削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツをソフト削除する
- 公開アクセスを制限する
- 繰り返しの違反者または重大な違反者を禁止する

明白な不正利用に対して、必ず警告を先に行う執行を保証するものではありません。報告、モデレーション保留、非表示の掲載、禁止措置、アカウントの状態については、[モデレーションとアカウントの安全性](/ja-JP/clawhub/moderation)を参照してください。
