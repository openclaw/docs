# 利用許容ポリシー

ClawHub は、OpenClaw 向けの Skills、plugins、packages、マーケットプレイスメタデータをホストします。
コンテンツまたは公開時の挙動が ClawHub に属するかどうかを判断するために、このページを使用してください。

これらのルールは、掲載情報が何を行うか、ユーザーに何を実行するよう求めるか、どのように自身を表現するか、公開者が ClawHub の発見、インストール、信頼の各サーフェスをどのように使用するかに適用されます。モデレーション状態とアカウントの状態については、[モデレーションとアカウントの安全性](/ja-JP/clawhub/moderation)を参照してください。著作権またはその他の権利の申し立てについては、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | 掲載情報が、ユーザーによるソフトウェアの構築、テスト、移行、デバッグ、文書化、または運用を支援する。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのある操作にレビュー、ドライラン、プレビュー、または確認の経路が含まれる。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが承認済みレビュー向けとして位置付けられ、証拠を保持し、人間による承認の境界を明確に保つ。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明なセットアップ、明示的な権限を使用する。                                            |
| メンテナンスされているカタログ                              | 各掲載情報が明確に区別され、有用で、正確に説明され、適切にメンテナンスされている。                                                |

文脈が重要です。同じトピックでも、限定的な防御的または同意に基づく設定では許容される場合があり、不正利用ワークフローとしてパッケージ化されている場合は許容されません。

## 禁止されるコンテンツ

ClawHub は、不正利用、欺瞞、安全でない実行、または権利侵害を主目的とするコンテンツをホストしません。

| カテゴリ                                                    | 許可されないもの                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 不正アクセスまたはセキュリティ回避                      | 認証回避、アカウント乗っ取り、レート制限の悪用、ライブ通話またはエージェントの乗っ取り、再利用可能なセッションの窃取、または未承認ユーザー向けペアリングフローの自動承認。                                                                                                                                                   |
| プラットフォームの不正利用と禁止回避                              | 禁止後のステルスアカウント、アカウントのウォーミングまたはファーミング、偽のエンゲージメント、複数アカウント自動化、大量投稿、スパムボット、または検出回避を目的として構築された自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書または請求書、欺瞞的な支払いフロー、スキャムのアウトリーチ、偽の社会的証明、詐欺向けの合成 ID ワークフロー、または明確な人間の承認を伴わない支出・課金ツール。                                                                                                                    |
| プライバシーを侵害するエンリッチメントまたは監視                 | スパム目的の連絡先スクレイピング、ドクシング、ストーキング、未承諾アウトリーチと組み合わされたリード抽出、秘密裏の監視、非同意の生体照合、または漏えいデータや侵害ダンプの使用。                                                                                                                  |
| 非同意のなりすましまたは ID 操作       | フェイススワップ、デジタルツイン、クローン化されたインフルエンサー、偽のペルソナ、またはなりすましや誤解を招くために使用されるその他のツール。                                                                                                                                                                                                 |
| 明示的な性的コンテンツまたは安全制限を無効化した成人向け生成 | NSFW 画像、動画、またはコンテンツ生成、サードパーティ API の成人向けコンテンツラッパー、または主目的が明示的な性的コンテンツである掲載情報。                                                                                                                                                       |
| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確なレビュー可能性なしに `sh` または `bash` でダウンロードコンテンツを実行するような pipe-to-shell インストーラー、未申告のシークレットまたは秘密鍵要件、明確なレビュー可能性のないリモート `npx @latest` 実行、または掲載情報が実行に本当に必要とするものを隠すメタデータ。 |
| 著作権侵害または権利侵害にあたる素材           | 他者の Skill、plugin、docs、ブランドアセット、またはプロプライエタリコードを許可なく再公開すること、ライセンス条件に違反すること、または元の作者や公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイス上の挙動

ClawHub は、公開者がマーケットプレイスをどのように使用しているかもレビューします。ClawHub を使用して、発見、メトリクス、信頼シグナル、モデレーションシステム、またはユーザーの注意を操作してはなりません。

禁止されるマーケットプレイス上の挙動には、次が含まれます。

- 実際のユーザー価値があるように見えない、低労力、重複的、プレースホルダー、または機械生成の掲載情報を大量に公開すること
- ほぼ同一の Skills または plugins で検索やカテゴリのサーフェスをあふれさせること
- 利用、メンテナンス、ソースの明確性、または意味のある差別化がほとんど、またはまったくない掲載情報を数百件公開すること
- 自動化、自己インストールループ、偽アカウント、協調的活動、有料エンゲージメント、またはその他の非オーガニックな挙動を通じて、インストール数、ダウンロード数、スター数、またはその他のエンゲージメント指標を人為的に水増しすること
- モデレーション、禁止、公開者制限、またはマーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有権、ソース、機能、セキュリティ態勢、インストール要件、または別のプロジェクトや公開者との関係についてユーザーに誤解を与えること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は自動的に不正利用とはみなされません。掲載情報が意味のある形で異なり、正確に説明され、メンテナンスされ、実際のユーザーに使われている場合、大規模なカタログは許容されます。大量であることが、薄い、重複的、誤解を招く、メンテナンスされていない、または人為的に宣伝された掲載情報と組み合わさる場合、大規模なカタログは信頼と安全性の問題になります。

## コンテンツ権利

ClawHub 上のコンテンツが自分の著作権またはその他の権利を侵害していると考える場合は、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。掲載情報が安全でない、悪意がある、または誤解を招く場合を除き、著作権または権利の申し立てに通常のマーケットプレイスレポートを使用しないでください。

## レビューと執行

ClawHub は、自動チェック、統計的な不正利用シグナル、ユーザーレポート、スタッフレビューを使用して、安全でないコンテンツまたは不正な公開行動を特定する場合があります。シグナル単体では不正利用を証明しません。ClawHub がレビューの必要性を判断する助けになります。

当社は次を行う場合があります。

- 違反する掲載情報を非表示、保留、削除、ソフト削除、またはリソースタイプでサポートされている場合はハード削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツをソフト削除する
- 公開アクセスを制限する
- 繰り返しまたは重大な違反者を禁止する

明らかな不正利用について、警告を先に行う執行を保証するものではありません。レポート、モデレーション保留、非表示の掲載情報、禁止、アカウントの状態については、[モデレーションとアカウントの安全性](/ja-JP/clawhub/moderation)を参照してください。
