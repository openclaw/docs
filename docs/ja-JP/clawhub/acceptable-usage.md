# 許容される利用

ClawHub は、OpenClaw 向けの Skills、Plugin、パッケージ、マーケットプレイスのメタデータをホストします。
コンテンツや公開行為が ClawHub に適しているかどうかを判断する際は、このページを使用してください。

これらのルールは、リスティングの動作、ユーザーに実行を求める内容、自己の表現方法、および公開者による ClawHub の検索・インストール・信頼に関する機能の利用方法に適用されます。モデレーション状態とアカウントの健全性については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。著作権またはその他の権利に関する申し立てについては、[コンテンツの権利に関する申請](/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性向上                           | リスティングが、ソフトウェアの構築、テスト、移行、デバッグ、文書化、運用に役立つこと。                                               |
| UI、データ、自動化のワークフロー               | 適用範囲が明確で、必要な認証情報が明示され、リスクのある操作にレビュー、ドライラン、プレビュー、または確認の手順が含まれていること。 |
| 防御的セキュリティ、モデレーション、不正利用のレビュー | ツールが承認されたレビューを目的として位置付けられ、証拠を保持し、人による承認の境界を明確に保っていること。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明性のあるセットアップ、明示的な権限を使用していること。                                            |
| 保守されているカタログ                              | 各リスティングが互いに異なり、有用かつ正確に説明され、適切に保守されていること。                                                |

文脈が重要です。同じテーマでも、限定的な防御目的または同意に基づく状況では許容される一方、不正利用のワークフローとしてパッケージ化されている場合は許容されないことがあります。

ClawHub は、不正利用、欺瞞、安全でない実行、または権利侵害を主目的とするコンテンツをホストしません。

| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| プラットフォームの不正利用と禁止措置の回避                              | 禁止措置後の秘匿アカウント、アカウントの育成または量産、偽のエンゲージメント、複数アカウントの自動化、大量投稿、スパムボット、または検出を回避するために構築された自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽造証明書や請求書、欺瞞的な支払いフロー、スキャムの勧誘、偽の社会的証明、詐欺目的の合成 ID ワークフロー、または人による明確な承認なしに支出や課金を行うツール。                                                                                                                    |

| 同意のないなりすましまたは ID 操作       | フェイススワップ、デジタルツイン、クローン化されたインフルエンサー、偽のペルソナ、またはなりすましや誤認誘導に使用されるその他のツール。                                                                                                                                                                                                 |
| 露骨な性的コンテンツまたは安全機能を無効化した成人向け生成 | NSFW の画像、動画、またはコンテンツの生成、サードパーティ API を利用した成人向けコンテンツのラッパー、または露骨な性的コンテンツを主目的とするリスティング。                                                                                                                                                       |
| 隠蔽された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確にレビューできない状態でダウンロードしたコンテンツを `sh` または `bash` で実行するパイプ・トゥ・シェル形式のインストーラー、申告されていないシークレットまたは秘密鍵の要件、明確にレビューできないリモートでの `npx @latest` 実行、またはリスティングの実行に実際に必要なものを隠すメタデータ。 |
| 著作権を侵害する、または権利に違反する素材           | 他者の Skills、Plugin、ドキュメント、ブランドアセット、またはプロプライエタリコードを許可なく再公開すること、ライセンス条項に違反すること、または元の作者や公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイスでの行為

ClawHub は、公開者によるマーケットプレイスの利用方法も審査します。検索性、指標、信頼シグナル、モデレーションシステム、またはユーザーの注意を操作するために ClawHub を使用しないでください。

禁止されるマーケットプレイスでの行為には、以下が含まれます。

- 実際のユーザー価値があるとは思われない、低品質、重複、プレースホルダー、または機械生成のリスティングを大量に一括公開すること
- ほぼ同一の Skills または Plugin で検索結果やカテゴリの表示領域を埋め尽くすこと
- 利用実績、保守、ソースの明確性、または有意な差別化がほとんど、もしくはまったくないリスティングを数百件公開すること
- 自動化、自己インストールのループ、偽アカウント、組織的な活動、有料エンゲージメント、またはその他の非自然な行為によって、インストール数、ダウンロード数、スター数、またはその他のエンゲージメント指標を人為的に水増しすること
- モデレーション、禁止措置、公開者の制限、またはマーケットプレイスの審査を回避するために、アカウントを作成または切り替えること
- 所有権、ソース、機能、セキュリティ態勢、インストール要件、または別のプロジェクトや公開者との提携関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は、それ自体が自動的に不正利用となるわけではありません。リスティングが有意に異なり、正確に説明され、保守され、実際のユーザーに利用されている場合、大規模なカタログも許容されます。量の多さに、内容の乏しさ、重複、誤解を招く表現、保守不足、または人為的な宣伝が伴う場合、大規模なカタログは信頼と安全性の問題になります。

## コンテンツの権利

ClawHub 上のコンテンツが自身の著作権またはその他の権利を侵害していると思われる場合は、[コンテンツの権利に関する申請](/clawhub/content-rights)を利用してください。リスティングが安全でない、悪意がある、または誤解を招くものでもある場合を除き、著作権または権利に関する申し立てに通常のマーケットプレイス報告を使用しないでください。

## 審査と執行

ClawHub は、安全でないコンテンツや不正な公開行為を特定するために、自動チェック、統計的な不正利用シグナル、ユーザーからの報告、スタッフによる審査を使用する場合があります。シグナル単独では不正利用の証明にはなりませんが、ClawHub が審査の必要性を判断するために役立ちます。

以下の措置を講じる場合があります。

- 違反するリスティングを非表示、保留、削除、論理削除、またはリソースの種類でサポートされている場合は物理削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツを論理削除する
- 公開アクセスを制限する
- 違反を繰り返す者または重大な違反者を禁止する

明白な不正利用に対し、必ず警告を先に行うとは限りません。報告、モデレーションによる保留、非表示のリスティング、禁止措置、アカウントの健全性については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。
