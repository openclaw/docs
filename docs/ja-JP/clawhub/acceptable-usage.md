# 利用規定

ClawHub は OpenClaw 向けのスキル、プラグイン、パッケージ、マーケットプレイスのメタデータをホストします。
コンテンツや公開時の挙動が ClawHub に属するかどうかを判断するために、このページを使用してください。

これらのルールは、掲載内容が何をするか、ユーザーに何を実行するよう求めるか、自身をどのように表現するか、公開者が ClawHub の発見、インストール、信頼のサーフェスをどのように使用するかに適用されます。モデレーション状態とアカウントの状態については、[モデレーションとアカウント安全性](/clawhub/moderation)を参照してください。著作権またはその他の権利に関する申し立てについては、[コンテンツ権利リクエスト](/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | 掲載内容が、ユーザーによるソフトウェアの構築、テスト、移行、デバッグ、文書化、運用を支援する。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのある操作にレビュー、ドライラン、プレビュー、または確認の経路が含まれる。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが承認済みレビュー向けとして位置付けられ、証拠を保持し、人間による承認の境界を明確に保つ。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明なセットアップ、明示的な権限を使用する。                                            |
| メンテナンスされているカタログ                              | 各掲載内容が明確に区別され、有用で、正確に説明され、合理的にメンテナンスされている。                                                |

文脈が重要です。同じトピックでも、狭い防御的な設定や同意に基づく設定では許容され、悪用ワークフローとしてパッケージ化されている場合は許容されないことがあります。

| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| プラットフォームの悪用と禁止回避                              | 禁止後のステルスアカウント、アカウントのウォーミングまたはファーミング、偽のエンゲージメント、複数アカウントの自動化、大量投稿、スパムボット、または検出回避を目的として構築された自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書または請求書、欺瞞的な支払いフロー、スキャムのアウトリーチ、偽の社会的証明、不正行為のための合成アイデンティティワークフロー、または明確な人間の承認なしに支出や課金を行うツール。                                                                                                                    |

| 同意のないなりすましまたはアイデンティティ操作       | フェイススワップ、デジタルツイン、複製されたインフルエンサー、偽のペルソナ、またはなりすましや誤解を招くために使用されるその他のツール。                                                                                                                                                                                                 |

| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、ダウンロードされたコンテンツを明確なレビュー可能性なしに `sh` または `bash` で実行するようなパイプ・トゥ・シェルのインストーラー、宣言されていないシークレットまたは秘密鍵の要件、明確なレビュー可能性のないリモート `npx @latest` 実行、または掲載内容の実行に本当に必要なものを隠すメタデータ。 |

## 禁止されるマーケットプレイス上の行為

ClawHub は、公開者がマーケットプレイスをどのように使用するかもレビューします。発見性、指標、信頼シグナル、モデレーションシステム、またはユーザーの注意を操作するために ClawHub を使用しないでください。

禁止されるマーケットプレイス上の行為には、次が含まれます。

- 実際のユーザー価値があるように見えない、労力の低い、重複した、プレースホルダーの、または機械生成された掲載内容を大量に一括公開すること
- ほぼ同一のスキルやプラグインで検索またはカテゴリのサーフェスをあふれさせること
- 利用、メンテナンス、ソースの明確さ、または意味のある差別化がほとんどない、あるいはまったくない掲載内容を何百件も公開すること
- 自動化、自己インストールのループ、偽アカウント、協調的な活動、有料エンゲージメント、またはその他の非有機的な行為により、インストール数、ダウンロード数、スター、またはその他のエンゲージメント指標を人為的に水増しすること
- モデレーション、禁止、公開者制限、またはマーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有権、ソース、機能、セキュリティ態勢、インストール要件、または別のプロジェクトや公開者との関係についてユーザーに誤解を与えること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は自動的に不正利用とは見なされません。掲載内容が意味のある形で異なり、正確に説明され、メンテナンスされ、実際のユーザーに使用されている場合、大規模なカタログは許容されます。大量性が、内容の薄い、重複した、誤解を招く、メンテナンスされていない、または人為的に宣伝された掲載内容と組み合わさると、大規模なカタログは信頼と安全性の問題になります。

## コンテンツ権利

ClawHub 上のコンテンツが著作権またはその他の権利を侵害していると思われる場合は、[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。掲載内容が安全でない、悪意がある、または誤解を招く場合を除き、著作権または権利に関する申し立てに通常のマーケットプレイス報告を使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツや不正な公開行為を特定するために、自動チェック、統計的な不正利用シグナル、ユーザー報告、スタッフレビューを使用する場合があります。シグナルはそれ自体で不正利用を証明するものではありません。ClawHub がレビューが必要なものを判断するために役立ちます。

当社は次を行う場合があります。

- 違反している掲載内容を非表示、保留、削除、ソフト削除、またはリソースタイプでサポートされている場合はハード削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツをソフト削除する
- 公開アクセスを制限する
- 繰り返し違反する者または重大な違反者を禁止する

明白な不正利用に対して、警告を先に行う執行を保証するものではありません。報告、モデレーション保留、非表示の掲載内容、禁止、アカウントの状態については、[モデレーションとアカウント安全性](/clawhub/moderation)を参照してください。
