# 許容される利用

このページでは、ClawHub が許容するスキルとコンテンツの種類、およびホストしない悪用ワークフローについて説明します。

これらのルールは意図的に実用性を重視しています。私たちが最も重視するのは、孤立したキーワードだけではなく、エンドツーエンドの悪用ワークフローです。防御の回避、プラットフォームの悪用、人への詐欺、プライバシー侵害、または同意のない行動を可能にすることを目的にスキルが構築されている場合、それは ClawHub に属しません。

## 最近のパターンのうち、明示的に許容するもの

- 実際のコンポーネント、セマンティックトークン、アクセシブルな状態、テスト済みのユーザーフローを使用するフロントエンドおよびデザインシステム作業。
- 一度きりのマークアップではなく、インストール済みのソースコンポーネント、プロジェクトエイリアス、文書化されたバリアントを使用する shadcn/ui の構成。
- コメントを保持し、具体的な UI5 型を使用し、生成されたコントロールインターフェイスをレビュー可能な状態に保つ UI5 の JavaScript から TypeScript への変換。
- 証拠を示し、人間による承認の境界を明確に保つ、防御的なセキュリティレビュー、モデレーションツール、悪用検出プロンプト。
- 明示的な認証情報、透明なセットアップ、ドライランまたはプレビューモードを備えた、個人またはチームアカウント向けの同意ベースのワークフロー自動化。
- サポート対象のソフトウェアに範囲を限定した、ドキュメント、移行ランブック、開発者ユーティリティ、テストフィクスチャ。

## 許容しないもの

- セキュリティ回避または不正アクセスのワークフロー。
  - 例: 認証回避、アカウント乗っ取り、CAPTCHA 回避、Cloudflare またはボット対策の回避、レート制限の回避、保護機能を破ることを目的としたステルススクレイピング、ライブ通話またはエージェントの乗っ取り、再利用可能なセッションの窃取、未承認ユーザー向けペアリングフローの自動承認。

- プラットフォーム悪用と停止回避。
  - 例: 停止後のステルスアカウント、アカウントのウォーミング/ファーミング、偽のエンゲージメント、カルマまたはフォロワーの育成、複数アカウントの自動化、大量投稿、スパムボット、検出回避を目的として構築されたマーケットプレイスまたはソーシャル自動化。

- 不正行為、詐欺、欺瞞的な金融ワークフロー。
  - 例: 偽の証明書、偽の請求書、欺瞞的な支払いフロー、詐欺的なアウトリーチ、偽の社会的証明、明確な人間の承認と透明な制御なしに支出または課金を可能にするツール、または不正目的でアカウントを作成するために構築された合成アイデンティティのワークフロー。

- プライバシーを侵害するスクレイピング、エンリッチメント、監視。
  - 例: スパム目的で連絡先詳細を大規模にスクレイピングすること、ドクシング、ストーキング、未承諾アウトリーチと組み合わせたリード抽出、秘密裏の監視、明確な同意なしに使用される顔検索または生体認証照合、または漏えいデータや侵害データダンプの購入、公開、ダウンロード、運用化。

- 同意のないなりすまし、または欺瞞的なアイデンティティ操作。
  - 例: フェイススワップ、デジタルツイン、偽のペルソナ、クローン化されたインフルエンサー、またはなりすましや誤認誘導に使用されるその他のアイデンティティ操作ツール。

- 露骨な性的コンテンツおよび安全機能を無効化した成人向け生成。
  - 例: NSFW の画像/動画/コンテンツ生成、サードパーティ API を包む成人向けコンテンツラッパー、または主目的が露骨な性的コンテンツであるスキル。

- 隠された、安全でない、または誤解を招く実行要件。
  - 例: 難読化されたインストールコマンド、`curl | sh`、未申告のシークレット要件、未申告の秘密鍵使用、明確なレビュー可能性のないリモート `npx @latest` 実行、スキルの実行に本当に必要なものを隠す誤解を招くメタデータ。

## 最近のパターンのうち、明示的に許容しないもの

- 「マーケットプレイスで停止された後にステルス販売者アカウントを作成する。」
- 「未承認ユーザーがペアリングコードを自動的に受け取るように Telegram のペアリングを変更する。」
- 「検出不能な自動化で Reddit/Twitter アカウントを育成する。」
- 「任意の用途向けに専門的な証明書または請求書を生成する。」
- 「安全チェックを無効にして NSFW コンテンツを生成する。」
- 「リードをスクレイピングし、連絡先をエンリッチし、大規模なコールドアウトリーチを開始する。」
- 「漏えいデータまたは侵害データダンプを購入、公開、またはダウンロードする。」
- 「合成アイデンティティまたは CAPTCHA 解決を使って、メールまたはソーシャルアカウントを一括作成する。」

## レビュー担当者向けの注意

- 文脈が重要です。同じトピックでも、狭い防御的または同意ベースの設定では正当であり、悪用ワークフローとしてパッケージ化されている場合は許容できないことがあります。
- スキルが回避、欺瞞、または同意のない使用に明確に最適化されている場合は、対応を優先すべきです。
- これらのカテゴリで繰り返しアップロードすることは、コンテンツの非表示およびアカウント停止の根拠になります。

## 執行

- 違反するスキルを非表示、削除、または完全削除する場合があります。
- トークンを取り消し、関連コンテンツをソフト削除し、繰り返しまたは重大な違反を行うユーザーを停止する場合があります。
- 明らかな悪用に対して、警告を先に行う執行を保証しません。
