# 適正利用

ClawHub は、OpenClaw 向けのスキル、プラグイン、パッケージ、およびマーケットプレイスのメタデータをホストします。
コンテンツや公開時の挙動が ClawHub に適しているかどうかを判断するには、このページを参照してください。

これらのルールは、掲載項目の機能、ユーザーに実行を求める内容、自らをどのように表現するか、および公開者が ClawHub の検索、インストール、信頼性に関する機能をどのように利用するかに適用されます。モデレーション状態とアカウントの健全性については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。著作権またはその他の権利に関する申し立てについては、[コンテンツの権利に関する申請](/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用かつ理解しやすく、誠意をもって公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性向上                           | 掲載項目が、ソフトウェアの構築、テスト、移行、デバッグ、文書化、または運用に役立つ。                                               |
| UI、データ、および自動化のワークフロー               | 適用範囲が明確で、必要な認証情報が明示され、リスクのある操作にレビュー、ドライラン、プレビュー、または確認の手順が含まれている。 |
| 防御的セキュリティ、モデレーション、および不正利用のレビュー | ツールが許可されたレビュー用として位置付けられ、証拠を保持し、人間による承認の境界を明確に維持している。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明性のあるセットアップ、および明示的な権限を使用している。                                            |
| メンテナンスされているカタログ                              | 各掲載項目が明確に区別され、有用かつ正確に説明され、適切にメンテナンスされている。                                                |

状況が重要です。同じ主題でも、限定的な防御目的または同意に基づく環境では許容される一方、不正利用のワークフローとしてパッケージ化されている場合は許容されないことがあります。

## 禁止されるコンテンツ

ClawHub は、不正利用、欺瞞、安全でない実行、または権利侵害を主な目的とするコンテンツをホストしません。

| カテゴリ                                                    | 禁止されるもの                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 不正アクセスまたはセキュリティの回避                      | 認証の回避、アカウントの乗っ取り、レート制限の悪用、進行中の通話またはエージェントの乗っ取り、再利用可能なセッションの窃取、または未承認ユーザー向けのペアリングフローの自動承認。                                                                                                                                                   |
| プラットフォームの悪用および禁止措置の回避                              | 禁止措置後の隠密アカウント、アカウントのウォーミングや量産、偽のエンゲージメント、複数アカウントの自動化、大量投稿、スパムボット、または検出回避を目的とした自動化。                                                                                                                                          |
| 詐欺、スキャム、および欺瞞的な金融ワークフロー             | 偽造証明書や請求書、欺瞞的な決済フロー、詐欺的な勧誘、偽のソーシャルプルーフ、詐欺目的の合成 ID ワークフロー、または人間による明確な承認なしに支出や課金を行うツール。                                                                                                                    |
| プライバシーを侵害する情報付加または監視                 | スパム目的の連絡先スクレイピング、個人情報の暴露、ストーキング、未承諾の勧誘と組み合わせた見込み客情報の抽出、秘密裏の監視、同意のない生体認証照合、または漏えいデータや侵害データダンプの使用。                                                                                                                  |
| 同意のないなりすましまたは ID の操作       | フェイススワップ、デジタルツイン、クローン化されたインフルエンサー、偽のペルソナ、または他者へのなりすましや誤認を目的として使用されるその他のツール。                                                                                                                                                                                                 |
| 露骨な性的コンテンツまたは安全機能を無効化した成人向け生成 | NSFW の画像、動画、またはコンテンツ生成、第三者 API を利用した成人向けコンテンツのラッパー、または露骨な性的コンテンツを主目的とする掲載項目。                                                                                                                                                       |
| 隠蔽された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確にレビューできない状態でダウンロードしたコンテンツを `sh` や `bash` で実行するパイプ・トゥ・シェル形式のインストーラー、申告されていないシークレットまたは秘密鍵の要件、明確にレビューできないリモートでの `npx @latest` 実行、または掲載項目の実行に実際に必要なものを隠すメタデータ。 |
| 著作権またはその他の権利を侵害する素材           | 他者のスキル、プラグイン、ドキュメント、ブランド資産、またはプロプライエタリコードを許可なく再公開すること、ライセンス条項に違反すること、または元の作者や公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイス上の行為

ClawHub は、公開者によるマーケットプレイスの利用方法も審査します。検索結果、指標、信頼性シグナル、モデレーションシステム、またはユーザーの注意を操作する目的で ClawHub を使用しないでください。

禁止されるマーケットプレイス上の行為には、以下が含まれます。

- 実際のユーザー価値がないと見られる、低品質、重複、プレースホルダー、または
  機械生成の掲載項目を大量に一括公開すること
- ほぼ同一のスキルやプラグインで検索結果やカテゴリの表示領域を埋め尽くすこと
- 利用実績、メンテナンス、ソースの明確性、または意味のある差別化がほとんど、あるいは
  まったくない掲載項目を数百件公開すること
- 自動化、自己インストールのループ、偽アカウント、組織的な活動、
  有償エンゲージメント、またはその他の非自然な行為によって、インストール数、ダウンロード数、スター数、その他のエンゲージメント
  指標を人為的に増加させること
- モデレーション、禁止措置、公開者の制限、または
  マーケットプレイスの審査を回避するために、アカウントを作成または切り替えること
- 所有権、ソース、機能、セキュリティ体制、
  インストール要件、または他のプロジェクトや公開者との関係についてユーザーを誤解させること
- すでに非表示、削除、またはブロックされたコンテンツを、根本的な問題を修正せずに
  繰り返しアップロードすること

大量公開が自動的に不正利用となるわけではありません。掲載項目に意味のある違いがあり、正確に説明され、メンテナンスされ、実際のユーザーに利用されている場合、大規模なカタログも許容されます。大量の掲載項目が、内容の薄さ、重複、誤解を招く説明、メンテナンス不足、または人為的な宣伝を伴う場合、大規模なカタログは信頼性と安全性の問題になります。

## コンテンツの権利

ClawHub 上のコンテンツが自身の著作権またはその他の権利を侵害していると思われる場合は、[コンテンツの権利に関する申請](/clawhub/content-rights)を利用してください。掲載項目が安全でない、悪意がある、または誤解を招くものでもある場合を除き、著作権または権利に関する申し立てに通常のマーケットプレイス報告を使用しないでください。

## 審査と措置

ClawHub は、安全でないコンテンツや不正な公開行為を特定するために、自動チェック、統計的な不正利用シグナル、ユーザーからの報告、およびスタッフによる審査を使用する場合があります。シグナルだけで不正利用が証明されるわけではありません。これは、ClawHub が審査の必要な対象を判断するために役立ちます。

当社は、以下の措置を講じる場合があります。

- 違反している掲載項目を非表示、保留、削除、論理削除、またはリソースの種類で対応している場合は
  物理削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを無効化する
- 関連コンテンツを論理削除する
- 公開権限を制限する
- 違反を繰り返す者または重大な違反者を禁止する

明白な不正利用に対して、警告を先に行う措置を保証するものではありません。報告、モデレーションによる保留、非表示の掲載項目、禁止措置、およびアカウントの健全性については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。
