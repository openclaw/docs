# 利用規定

ClawHub は OpenClaw 向けの Skills、Plugin、パッケージ、マーケットプレイスメタデータをホストします。
このページは、コンテンツや公開時の挙動が ClawHub に適しているかを判断するために使用してください。

これらのルールは、リスティングが何を行うか、ユーザーに何の実行を求めるか、どのように自己表現するか、公開者が ClawHub の発見、インストール、信頼のサーフェスをどのように使用するかに適用されます。モデレーション状態とアカウントの状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。著作権またはその他の権利の申し立てについては、[コンテンツ権利リクエスト](/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | リスティングが、ユーザーによるソフトウェアのビルド、テスト、移行、デバッグ、ドキュメント作成、運用を支援する。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのある操作にレビュー、ドライラン、プレビュー、確認の経路が含まれている。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが認可されたレビュー向けとして位置づけられ、証拠を保持し、人間による承認の境界を明確に保つ。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明なセットアップ、明示的な権限を使用する。                                            |
| 保守されているカタログ                              | 各リスティングが個別性と有用性を持ち、正確に説明され、適切に保守されている。                                                |

文脈が重要です。同じトピックでも、狭い防御的または同意に基づく設定では許容される一方で、不正利用ワークフローとしてパッケージ化されている場合は許容されません。

ClawHub は、不正利用、欺瞞、安全でない実行、権利侵害を主な目的とするコンテンツをホストしません。

| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| プラットフォームの不正利用と禁止回避                              | 禁止後のステルスアカウント、アカウントのウォーミングや量産、偽のエンゲージメント、複数アカウントの自動化、大量投稿、スパムボット、または検出回避を目的として構築された自動化。                                                                                                                                          |
| 詐欺、詐取、欺瞞的な金融ワークフロー             | 偽の証明書や請求書、欺瞞的な支払いフロー、詐欺的なアウトリーチ、偽の社会的証明、不正目的の合成 ID ワークフロー、または明確な人間の承認を伴わない支出・課金ツール。                                                                                                                    |
| プライバシー侵害的なエンリッチメントまたは監視                 | スパム目的の連絡先スクレイピング、ドクシング、ストーキング、未承諾アウトリーチと組み合わせたリード抽出、秘密裏の監視、同意のない生体照合、漏洩データや侵害ダンプの使用。                                                                                                                  |
| 同意のないなりすましまたは ID 操作       | 顔の差し替え、デジタルツイン、クローン化されたインフルエンサー、偽の人格、またはなりすましや誤認を目的として使用されるその他のツール。                                                                                                                                                                                                 |
| 明示的な性的コンテンツまたは安全機能を無効化した成人向け生成 | NSFW 画像、動画、またはコンテンツ生成、サードパーティ API を囲む成人向けコンテンツラッパー、または明示的な性的コンテンツを主目的とするリスティング。                                                                                                                                                       |
| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確なレビュー可能性なしにダウンロードした内容を `sh` または `bash` で実行するような、シェルへパイプするインストーラー、未申告のシークレットまたは秘密鍵の要件、明確なレビュー可能性のないリモート `npx @latest` 実行、またはリスティングの実行に本当に必要なものを隠すメタデータ。 |
| 著作権侵害または権利侵害のある素材           | 他者の Skill、Plugin、ドキュメント、ブランド資産、またはプロプライエタリコードを許可なく再公開すること、ライセンス条項に違反すること、または元の作者や公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイス上の行為

ClawHub は、公開者がマーケットプレイスをどのように使用しているかも審査します。発見、指標、信頼シグナル、モデレーションシステム、ユーザーの注意を操作する目的で ClawHub を使用しないでください。

禁止されるマーケットプレイス上の行為には、次が含まれます。

- 実際のユーザー価値があるように見えない、低労力、重複的、プレースホルダー的、または機械生成のリスティングを大量に公開すること
- ほぼ同一の Skills や Plugin で検索やカテゴリのサーフェスを埋め尽くすこと
- 利用、保守、ソースの明確さ、または意味のある差別化がほとんど、あるいはまったくないリスティングを数百件公開すること
- 自動化、自己インストールのループ、偽アカウント、協調的な活動、有料エンゲージメント、またはその他の非オーガニックな行為によって、インストール数、ダウンロード数、スター数、その他のエンゲージメント指標を人為的に膨らませること
- モデレーション、禁止、公開者制限、またはマーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有者、ソース、機能、セキュリティ状態、インストール要件、または別のプロジェクトや公開者との関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は自動的に不正利用とは見なされません。リスティングが意味のある違いを持ち、正確に説明され、保守され、実際のユーザーに利用されている場合、大規模なカタログは許容されます。大量公開が、薄い内容、重複、誤解を招く内容、未保守、または人為的に宣伝されたリスティングと組み合わさる場合、大規模なカタログは信頼と安全性の問題になります。

## コンテンツ権利

ClawHub 上のコンテンツがあなたの著作権またはその他の権利を侵害していると考える場合は、[コンテンツ権利リクエスト](/clawhub/content-rights)を使用してください。リスティングが安全でない、悪意がある、または誤解を招く場合を除き、著作権または権利の申し立てに通常のマーケットプレイス報告を使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツや不正な公開行為を特定するために、自動チェック、統計的な不正利用シグナル、ユーザー報告、スタッフレビューを使用する場合があります。シグナル単体で不正利用を証明するものではありません。ClawHub がレビューの必要な対象を判断する助けになります。

当社は次を行う場合があります。

- 違反するリスティングを非表示、保留、削除、ソフト削除、またはリソースタイプでサポートされている場合はハード削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツをソフト削除する
- 公開アクセスを制限する
- 繰り返し違反する者、または重大な違反者を禁止する

明白な不正利用に対して、必ず警告を先行する執行を行うとは保証しません。報告、モデレーション保留、非表示リスティング、禁止、アカウントの状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。
