# 許容される利用

このページでは、ClawHub が許容するスキルとコンテンツの種類、およびホストしない悪用ワークフローについて説明します。

これらのルールは意図的に実用性を重視しています。単独のキーワードだけでなく、エンドツーエンドの悪用ワークフローを最も重視します。スキルが防御策の回避、プラットフォームの悪用、人への詐欺、プライバシーの侵害、または同意のない行為の実現を目的に構築されている場合、それは ClawHub にふさわしくありません。

## 最近のパターンで明示的に許容しているもの

- 実際のコンポーネント、セマンティックトークン、アクセシブルな状態、テスト済みのユーザーフローを使用するフロントエンドおよびデザインシステム作業。
- その場限りのマークアップではなく、インストール済みのソースコンポーネント、プロジェクトエイリアス、文書化されたバリアントを使用する shadcn/ui の構成。
- コメントを保持し、具体的な UI5 型を使用し、生成されたコントロールインターフェイスをレビュー可能な状態に保つ UI5 JavaScript から TypeScript への変換。
- 証拠を示し、人間による承認の境界を明確に保つ、防御的なセキュリティレビュー、モデレーションツール、不正利用検出プロンプト。
- 明示的な認証情報、透明なセットアップ、ドライランまたはプレビューモードを備えた、個人またはチームアカウント向けの同意ベースのワークフロー自動化。
- サポート対象のソフトウェアに範囲を限定したドキュメント、移行ランブック、開発者ユーティリティ、テストフィクスチャ。

## 許容されないもの

- セキュリティ回避または不正アクセスのワークフロー。
  - 例: 認証回避、アカウント乗っ取り、CAPTCHA 回避、Cloudflare またはボット対策の回避、レート制限回避、保護を打ち破ることを目的に設計されたステルススクレイピング、ライブ通話またはエージェントの乗っ取り、再利用可能なセッション窃取、未承認ユーザー向けのペアリングフロー自動承認。

- プラットフォーム悪用と ban 回避。
  - 例: ban 後のステルスアカウント、アカウントのウォーミングやファーミング、偽のエンゲージメント、karma やフォロワーの育成、複数アカウントの自動化、大量投稿、スパムボット、検出回避を目的に構築されたマーケットプレイスまたはソーシャル自動化。

- 詐欺、スキャム、欺瞞的な金融ワークフロー。
  - 例: 偽の証明書、偽の請求書、欺瞞的な支払いフロー、スキャムのアウトリーチ、偽の社会的証明、明確な人間の承認と透明な制御なしに支出または課金を可能にするツール、または詐欺目的のアカウント作成のために構築された合成アイデンティティワークフロー。

- プライバシーを侵害するスクレイピング、エンリッチメント、監視。
  - 例: スパム目的で連絡先詳細を大規模にスクレイピングすること、doxxing、ストーキング、未承諾アウトリーチと組み合わせたリード抽出、秘密裏の監視、明確な同意なしに使用される顔検索または生体照合、漏えいデータまたは侵害ダンプの購入、公開、ダウンロード、または運用化。

- 同意のないなりすまし、または欺瞞的なアイデンティティ操作。
  - 例: なりすましや誤認誘導に使用されるフェイススワップ、デジタルツイン、偽のペルソナ、複製されたインフルエンサー、その他のアイデンティティ操作ツール。

- 明示的な性的コンテンツおよび安全機能を無効にした成人向け生成。
  - 例: NSFW 画像、動画、コンテンツの生成、サードパーティ API の成人向けコンテンツラッパー、または明示的な性的コンテンツを主目的とするスキル。

- 隠された、安全でない、または誤解を招く実行要件。
  - 例: 難読化されたインストールコマンド、`curl | sh`、未申告のシークレット要件、未申告の秘密鍵使用、明確なレビュー可能性のないリモート `npx @latest` 実行、スキルが実行に本当に必要とするものを隠す誤解を招くメタデータ。

## 最近のパターンで明示的に許容していないもの

- 「マーケットプレイスの ban 後にステルス販売者アカウントを作成する。」
- 「Telegram のペアリングを変更し、未承認ユーザーがペアリングコードを自動的に受け取れるようにする。」
- 「検出不能な自動化で Reddit/Twitter アカウントを育成する。」
- 「任意の用途向けに専門的な証明書や請求書を生成する。」
- 「安全チェックを無効にして NSFW コンテンツを生成する。」
- 「リードをスクレイピングし、連絡先をエンリッチし、大規模なコールドアウトリーチを開始する。」
- 「漏えいデータまたは侵害ダンプを購入、公開、またはダウンロードする。」
- 「合成アイデンティティまたは CAPTCHA 解決を使って、メールまたはソーシャルアカウントを一括作成する。」

## レビュアー向けの注記

- コンテキストが重要です。同じトピックでも、範囲の狭い防御的または同意ベースの設定では正当になり得ますが、悪用ワークフローとしてパッケージ化されている場合は許容できません。
- スキルが回避、欺瞞、または同意のない使用に明確に最適化されている場合は、対応する方向に判断するべきです。
- これらのカテゴリで繰り返しアップロードされた場合、コンテンツの非表示およびアカウントの ban の根拠になります。

## 執行

- 違反するスキルを非表示、削除、または完全削除する場合があります。
- トークンを取り消し、関連コンテンツをソフト削除し、繰り返し違反する者または重大な違反者を ban する場合があります。
- 明白な悪用に対して、警告を先に行う執行を保証しません。
