# 利用許容ポリシー

ClawHub は OpenClaw の Skills、Plugin、パッケージ、マーケットプレイスメタデータをホストします。
コンテンツや公開の挙動が ClawHub に属するかどうかを判断するために、このページを使用してください。

これらのルールは、リスティングが何をするか、ユーザーに何の実行を求めるか、自身をどう表現するか、公開者が ClawHub の発見、インストール、信頼のサーフェスをどう使用するかに適用されます。モデレーション状態とアカウントの状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。著作権またはその他の権利に関する申し立てについては、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | リスティングが、ユーザーによるソフトウェアのビルド、テスト、移行、デバッグ、ドキュメント作成、運用を支援する。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのあるアクションにレビュー、ドライラン、プレビュー、または確認の経路が含まれている。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが承認済みレビューを目的として位置づけられ、証拠を保持し、人間による承認の境界を明確にしている。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明なセットアップ、明示的な権限を使用している。                                            |
| メンテナンスされているカタログ                              | 各リスティングが個別性を持ち、有用で、正確に説明され、合理的にメンテナンスされている。                                                |

文脈が重要です。同じトピックでも、狭い防御的または同意に基づく設定では許容される一方、不正利用ワークフローとしてパッケージ化されると許容されません。

| プラットフォームの不正利用と禁止回避                              | 禁止後のステルスアカウント、アカウントのウォーミングまたはファーミング、偽のエンゲージメント、複数アカウントの自動化、大量投稿、スパムボット、または検出回避を目的として作られた自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書または請求書、欺瞞的な支払いフロー、スキャムのアウトリーチ、偽のソーシャルプルーフ、詐欺のための合成アイデンティティワークフロー、または明確な人間の承認がない支出/課金ツール。                                                                                                                    |

| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確なレビュー可能性がないままダウンロードされたコンテンツを `sh` または `bash` で実行する pipe-to-shell インストーラー、未宣言のシークレットまたは秘密鍵要件、明確なレビュー可能性がないリモート `npx @latest` 実行、またはリスティングの実行に本当に必要なものを隠すメタデータ。 |
| 著作権侵害または権利侵害にあたる素材           | 他者の Skills、Plugin、ドキュメント、ブランドアセット、またはプロプライエタリコードを許可なく再公開すること、ライセンス条項に違反すること、または元の作者や公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイスでの挙動

ClawHub は、公開者がマーケットプレイスをどう使用するかもレビューします。発見、指標、信頼シグナル、モデレーションシステム、またはユーザーの注意を操作するために ClawHub を使用しないでください。

禁止されるマーケットプレイスでの挙動には、次のものが含まれます。

- 実際のユーザー価値があるように見えない、低労力、重複、プレースホルダー、または機械生成のリスティングを大量に一括公開すること
- ほぼ同一の Skills または Plugin で検索やカテゴリのサーフェスをあふれさせること
- 使用実績、メンテナンス、ソースの明確さ、または意味のある差別化がほとんど、またはまったくないリスティングを数百件公開すること
- 自動化、自己インストールのループ、偽アカウント、協調的活動、有料エンゲージメント、またはその他の非オーガニックな挙動を通じて、インストール、ダウンロード、スター、またはその他のエンゲージメント指標を人為的に膨らませること
- モデレーション、禁止、公開者制限、またはマーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有者、ソース、機能、セキュリティ態勢、インストール要件、または別のプロジェクトや公開者との関係についてユーザーに誤解を与えること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開が自動的に不正利用になるわけではありません。大規模なカタログは、リスティングが意味のある違いを持ち、正確に説明され、メンテナンスされ、実際のユーザーに使用されている場合は許容されます。大規模なカタログは、量が薄い内容、重複、誤解を招く内容、メンテナンス不足、または人為的に宣伝されたリスティングと組み合わさると、信頼と安全性の問題になります。

## コンテンツ権利

ClawHub 上のコンテンツが自分の著作権またはその他の権利を侵害していると考える場合は、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。リスティングが安全でない、悪意がある、または誤解を招く場合を除き、著作権または権利に関する申し立てに通常のマーケットプレイス報告を使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツや不正な公開行為を特定するために、自動チェック、統計的な不正利用シグナル、ユーザー報告、スタッフレビューを使用する場合があります。シグナル単体で不正利用を証明するものではありません。ClawHub がレビューを必要とするものを判断する助けになります。

当社は次を行う場合があります。

- 違反するリスティングを非表示、保留、削除、ソフト削除、またはリソースタイプでサポートされている場合はハード削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツをソフト削除する
- 公開アクセスを制限する
- 繰り返し違反する者、または重大な違反者を禁止する

明白な不正利用について、警告を先行する執行を保証するものではありません。報告、モデレーション保留、非表示のリスティング、禁止、アカウントの状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。
