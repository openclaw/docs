# 許容される使用

ClawHub は OpenClaw 向けの Skills、Plugin、パッケージ、マーケットプレイスメタデータをホストします。
このページは、コンテンツや公開時の挙動が ClawHub に属するかどうかを判断するために使用します。

これらのルールは、リスティングが行うこと、ユーザーに実行を求めること、自身をどう表現するか、公開者が ClawHub の発見、インストール、信頼サーフェスをどう使用するかに適用されます。モデレーション状態とアカウントの状態については、[モデレーションとアカウントの安全性](/ja-JP/clawhub/moderation)を参照してください。著作権またはその他の権利に関する申し立てについては、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | リスティングが、ユーザーによるソフトウェアの構築、テスト、移行、デバッグ、文書化、または運用に役立つ。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのある操作にレビュー、ドライラン、プレビュー、または確認の経路が含まれる。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが認可されたレビュー向けとして位置付けられ、証拠を保持し、人間による承認境界を明確に保つ。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明なセットアップ、明示的な権限を使用する。                                            |
| メンテナンスされているカタログ                              | 各リスティングが明確に区別され、有用で、正確に説明され、妥当にメンテナンスされている。                                                |

文脈が重要です。同じトピックでも、狭い防御的または同意に基づく設定では許容される一方で、不正利用ワークフローとしてパッケージ化されている場合は許容されないことがあります。

## 禁止されるコンテンツ

ClawHub は、主な目的が不正利用、欺瞞、安全でない実行、または権利侵害であるコンテンツをホストしません。

| カテゴリ                                                    | 許可されないもの                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 不正アクセスまたはセキュリティ回避                      | 認証回避、アカウント乗っ取り、レート制限の不正利用、ライブ通話またはエージェントの乗っ取り、再利用可能なセッションの窃取、未承認ユーザー向けペアリングフローの自動承認。                                                                                                                                                   |
| プラットフォームの不正利用と禁止回避                              | 禁止後のステルスアカウント、アカウントの育成またはファーミング、偽のエンゲージメント、複数アカウントの自動化、大量投稿、スパムボット、または検知を回避するために構築された自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書または請求書、欺瞞的な支払いフロー、スキャムのアウトリーチ、偽の社会的証明、詐欺向けの合成 ID ワークフロー、または明確な人間の承認を伴わない支出/課金ツール。                                                                                                                    |
| プライバシー侵害的なエンリッチメントまたは監視                 | スパム目的の連絡先スクレイピング、ドキシング、ストーキング、未承諾アウトリーチと組み合わせたリード抽出、秘密裏の監視、同意のない生体情報照合、または漏えいデータや侵害ダンプの使用。                                                                                                                  |
| 同意のないなりすましまたは ID 操作       | フェイススワップ、デジタルツイン、複製されたインフルエンサー、偽のペルソナ、またはなりすましや誤認誘導に使用されるその他のツール。                                                                                                                                                                                                 |
| 明示的な性的コンテンツまたは安全機能が無効化された成人向け生成 | NSFW 画像、動画、またはコンテンツ生成、サードパーティ API を囲い込む成人向けコンテンツラッパー、または主な目的が明示的な性的コンテンツであるリスティング。                                                                                                                                                       |
| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確なレビュー可能性なしにダウンロードしたコンテンツを `sh` または `bash` で実行するような pipe-to-shell インストーラー、未申告のシークレットまたは秘密鍵要件、明確なレビュー可能性なしのリモート `npx @latest` 実行、またはリスティングが実行に本当に必要とするものを隠すメタデータ。 |
| 著作権侵害または権利侵害にあたる素材           | 他者の skill、plugin、ドキュメント、ブランドアセット、またはプロプライエタリコードを許可なく再公開すること、ライセンス条項に違反すること、または元の作者や公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイス上の挙動

ClawHub は、公開者がマーケットプレイスをどのように使用するかもレビューします。ClawHub を使用して、発見、指標、信頼シグナル、モデレーションシステム、またはユーザーの注意を操作しないでください。

禁止されるマーケットプレイス上の挙動には、次が含まれます。

- 実際のユーザー価値があるように見えない、低労力、重複、プレースホルダー、または機械生成のリスティングを大量に一括公開すること
- ほぼ同一の skills または plugins で検索やカテゴリのサーフェスをあふれさせること
- 使用、メンテナンス、ソースの明確さ、または意味のある差別化がほとんど、またはまったくないリスティングを数百件公開すること
- 自動化、自己インストールループ、偽アカウント、協調的な活動、有料エンゲージメント、またはその他の非オーガニックな挙動により、インストール数、ダウンロード数、スター数、またはその他のエンゲージメント指標を人為的に膨らませること
- モデレーション、禁止、公開者制限、またはマーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有権、ソース、機能、セキュリティ態勢、インストール要件、または別のプロジェクトや公開者との関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は自動的に不正利用になるわけではありません。リスティングが意味のある形で異なり、正確に説明され、メンテナンスされ、実際のユーザーに使用されている場合、大規模なカタログは許容されます。量が薄い内容、重複、誤解を招く内容、メンテナンス不足、または人為的に宣伝されたリスティングと組み合わさると、大規模なカタログは信頼と安全性の問題になります。

## コンテンツ権利

ClawHub 上のコンテンツが著作権またはその他の権利を侵害していると考える場合は、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。リスティングが同時に安全でない、悪意がある、または誤解を招くものでない限り、著作権または権利に関する申し立てに通常のマーケットプレイス報告を使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツや不正な公開挙動を特定するために、自動チェック、統計的な不正利用シグナル、ユーザー報告、スタッフレビューを使用する場合があります。シグナル単独で不正利用が証明されるわけではありません。ClawHub が何をレビューすべきか判断する助けになります。

ClawHub は次を行う場合があります。

- 違反リスティングを非表示、保留、削除、ソフト削除、またはリソース種別でサポートされている場合はハード削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツをソフト削除する
- 公開アクセスを制限する
- 繰り返しまたは重大な違反者を禁止する

明白な不正利用に対して、警告を先に行う執行を保証するものではありません。報告、モデレーション保留、非表示リスティング、禁止、アカウントの状態については、[モデレーションとアカウントの安全性](/ja-JP/clawhub/moderation)を参照してください。
