# 許容される利用

ClawHub は OpenClaw 向けの Skills、plugins、パッケージ、マーケットプレイスメタデータをホストします。
コンテンツや公開時の挙動が ClawHub に属するかどうかを判断するために、このページを使用してください。

これらのルールは、掲載が何を行うか、ユーザーに何を実行するよう求めるか、自身をどのように表現するか、公開者が ClawHub の発見、インストール、信頼に関するサーフェスをどのように使用するかに適用されます。モデレーション状態とアカウントの状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。著作権またはその他の権利に関する申し立てについては、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | 掲載が、ユーザーによるソフトウェアのビルド、テスト、移行、デバッグ、文書化、運用に役立つ。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのある操作にレビュー、ドライラン、プレビュー、または確認の経路が含まれる。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが認可されたレビュー向けとして位置付けられ、証拠を保持し、人間による承認の境界を明確に保つ。                          |
| 個人またはチームのワークフロー                       | ワークフローが同意に基づくアカウント、透明なセットアップ、明示的な権限を使用する。                                            |
| メンテナンスされているカタログ                              | 各掲載が個別性を持ち、有用で、正確に説明され、合理的にメンテナンスされている。                                                |

文脈が重要です。同じトピックでも、限定的な防御的または同意に基づく設定では許容される場合があり、不正利用ワークフローとしてパッケージ化されている場合は許容されない場合があります。

## 禁止されるコンテンツ

ClawHub は、不正利用、欺瞞、安全でない実行、または権利侵害を主な目的とするコンテンツをホストしません。

| カテゴリ                                                    | 許可されないもの                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 不正アクセスまたはセキュリティ回避                      | 認証回避、アカウント乗っ取り、レート制限の不正利用、ライブ通話またはエージェントの乗っ取り、再利用可能なセッションの窃取、または未承認ユーザー向けのペアリングフローの自動承認。                                                                                                                                                   |
| プラットフォーム不正利用とBAN回避                              | BAN後のステルスアカウント、アカウントのウォーミングまたはファーミング、偽のエンゲージメント、複数アカウントの自動化、大量投稿、スパムボット、または検出回避のために構築された自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書または請求書、欺瞞的な支払いフロー、スキャム勧誘、偽の社会的証明、詐欺目的の合成アイデンティティワークフロー、または明確な人間の承認を伴わない支出/課金ツール。                                                                                                                    |
| プライバシーを侵害するエンリッチメントまたは監視                 | スパム目的の連絡先スクレイピング、ドキシング、ストーキング、未承諾のアウトリーチと組み合わせたリード抽出、秘密裏の監視、同意のない生体照合、または漏えいデータや侵害ダンプの使用。                                                                                                                  |
| 同意のないなりすましまたはアイデンティティ操作       | 顔交換、デジタルツイン、クローン化されたインフルエンサー、偽のペルソナ、またはなりすましや誤認誘導に使用されるその他のツール。                                                                                                                                                                                                 |
| 露骨な性的コンテンツまたは安全機能を無効化した成人向け生成 | NSFWの画像、動画、またはコンテンツ生成。サードパーティ API を囲む成人向けコンテンツラッパー。または主な目的が露骨な性的コンテンツである掲載。                                                                                                                                                       |
| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確なレビュー可能性なしにダウンロードしたコンテンツを `sh` または `bash` で実行するような pipe-to-shell インストーラー、未申告のシークレットまたは秘密鍵要件、明確なレビュー可能性のないリモート `npx @latest` 実行、または掲載の実行に本当に必要なものを隠すメタデータ。 |
| 著作権を侵害する、または権利を侵害する素材           | 他者の skill、plugin、ドキュメント、ブランドアセット、またはプロプライエタリコードを許可なく再公開すること。ライセンス条件に違反すること。または元の作者や公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイス上の挙動

ClawHub は、公開者がマーケットプレイスをどのように使用するかもレビューします。発見、指標、信頼シグナル、モデレーションシステム、またはユーザーの注意を操作するために ClawHub を使用しないでください。

禁止されるマーケットプレイス上の挙動には、次が含まれます。

- 実際のユーザー価値があるように見えない、手間のかかっていない、重複した、プレースホルダーの、または機械生成の掲載を大量に一括公開すること
- 検索またはカテゴリサーフェスを、ほぼ同一の skills または plugins であふれさせること
- 利用、メンテナンス、ソースの明確さ、または意味のある差別化がほとんど、またはまったくない数百件の掲載を公開すること
- 自動化、自己インストールループ、偽アカウント、協調活動、有償エンゲージメント、またはその他の非自然な行動を通じて、インストール数、ダウンロード数、スター数、またはその他のエンゲージメント指標を人為的に水増しすること
- モデレーション、BAN、公開者制限、またはマーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有権、ソース、機能、セキュリティ態勢、インストール要件、または別のプロジェクトや公開者との関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は自動的に不正利用になるわけではありません。掲載が意味のある違いを持ち、正確に説明され、メンテナンスされ、実際のユーザーに利用されている場合、大規模なカタログは許容されます。掲載量が、内容の薄さ、重複、誤解を招く内容、未メンテナンス、または人為的な宣伝と組み合わさると、大規模なカタログは信頼と安全性の問題になります。

## コンテンツ権利

ClawHub 上のコンテンツがあなたの著作権またはその他の権利を侵害していると思われる場合は、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。掲載が安全でない、悪意がある、または誤解を招く場合を除き、著作権または権利に関する申し立てに通常のマーケットプレイスレポートを使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツや不正な公開行為を特定するために、自動チェック、統計的な不正利用シグナル、ユーザーレポート、スタッフレビューを使用する場合があります。シグナル単独で不正利用が証明されるわけではありません。ClawHub がレビューが必要な対象を判断する助けになります。

当社は次を行う場合があります。

- 違反する掲載を非表示、保留、削除、ソフトデリート、またはリソースタイプでサポートされている場合はハードデリートする
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツをソフトデリートする
- 公開アクセスを制限する
- 繰り返し違反する者、または重大な違反を行う者をBANする

明白な不正利用に対して、警告を先に行う執行を保証するものではありません。レポート、モデレーション保留、非表示の掲載、BAN、アカウントの状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。
