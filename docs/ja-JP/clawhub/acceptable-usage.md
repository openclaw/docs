# 許容される利用

ClawHub は、OpenClaw 向けの Skills、Plugin、パッケージ、マーケットプレイスメタデータをホストします。
コンテンツや公開動作が ClawHub に属するかどうかを判断するために、このページを使用してください。

これらのルールは、掲載情報が何をするか、ユーザーに何の実行を求めるか、自身をどう表現するか、公開者が ClawHub の発見、インストール、信頼のサーフェスをどう使うかに適用されます。モデレーション状態とアカウントの状態については、[モデレーションとアカウント安全性](/clawhub/moderation)を参照してください。著作権またはその他の権利の申し立てについては、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で、理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | 掲載情報が、ユーザーによるソフトウェアの構築、テスト、移行、デバッグ、文書化、または運用を支援する。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのあるアクションにレビュー、ドライラン、プレビュー、または確認の経路が含まれる。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが承認済みレビュー向けとして位置付けられ、証拠を保持し、人による承認境界を明確に保つ。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明なセットアップ、明示的な権限を使用する。                                            |
| メンテナンスされているカタログ                              | 各掲載情報が個別性を持ち、有用で、正確に説明され、適切にメンテナンスされている。                                                |

文脈が重要です。同じトピックでも、限定的な防御的または同意に基づく設定では許容され、不正利用ワークフローとしてパッケージ化されている場合は許容されないことがあります。

## 禁止されるコンテンツ

ClawHub は、不正利用、欺瞞、安全でない実行、または権利侵害を主な目的とするコンテンツをホストしません。

| カテゴリ                                                    | 許可されないもの                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 不正アクセスまたはセキュリティ回避                      | 認証バイパス、アカウント乗っ取り、レート制限の悪用、ライブ通話またはエージェントの乗っ取り、再利用可能なセッションの窃取、未承認ユーザー向けのペアリングフローの自動承認。                                                                                                                                                   |
| プラットフォームの不正利用と禁止回避                              | 禁止後のステルスアカウント、アカウントのウォーミングまたは量産、偽のエンゲージメント、複数アカウントの自動化、大量投稿、スパムボット、または検出回避を目的に作られた自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書または請求書、欺瞞的な支払いフロー、スキャム勧誘、偽の社会的証明、詐欺向けの合成 ID ワークフロー、または明確な人間の承認がない支出・課金ツール。                                                                                                                    |
| プライバシーを侵害するエンリッチメントまたは監視                 | スパム目的の連絡先スクレイピング、晒し、ストーキング、未承諾の勧誘と組み合わせたリード抽出、秘密裏の監視、同意のない生体照合、または漏えいデータや侵害ダンプの使用。                                                                                                                  |
| 同意のないなりすましまたは ID 操作       | 顔交換、デジタルツイン、クローン化されたインフルエンサー、偽の人格、またはなりすましや誤認誘導に使われるその他のツール。                                                                                                                                                                                                 |
| 露骨な性的コンテンツまたは安全制限を無効にした成人向け生成 | NSFW 画像、動画、またはコンテンツ生成、サードパーティ API を囲い込む成人向けコンテンツラッパー、または露骨な性的コンテンツを主目的とする掲載情報。                                                                                                                                                       |
| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確なレビュー可能性なしにダウンロード内容を `sh` または `bash` で実行するような pipe-to-shell インストーラー、未申告のシークレットまたは秘密鍵要件、明確なレビュー可能性のないリモート `npx @latest` 実行、または掲載情報が実行に本当に必要とするものを隠すメタデータ。 |
| 著作権侵害または権利侵害にあたる素材           | 他者の Skills、Plugin、ドキュメント、ブランドアセット、またはプロプライエタリコードを許可なく再公開すること、ライセンス条件に違反すること、または元の作者や公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイス上の行動

ClawHub は、公開者がマーケットプレイスをどう使うかもレビューします。発見、指標、信頼シグナル、モデレーションシステム、またはユーザーの注意を操作するために ClawHub を使用しないでください。

禁止されるマーケットプレイス上の行動には、次が含まれます。

- 実際のユーザー価値があるように見えない、低労力、重複、プレースホルダー、または機械生成の掲載情報を大量に一括公開すること
- ほぼ同一の Skills または Plugin で検索やカテゴリのサーフェスを氾濫させること
- 利用、メンテナンス、ソースの明確性、または意味のある差別化がほとんど、またはまったくない掲載情報を数百件公開すること
- 自動化、自己インストールループ、偽アカウント、協調的活動、有料エンゲージメント、またはその他の非オーガニックな行動によって、インストール数、ダウンロード数、スター、またはその他のエンゲージメント指標を人為的に水増しすること
- モデレーション、禁止、公開者制限、またはマーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有権、ソース、機能、セキュリティ態勢、インストール要件、または別のプロジェクトや公開者との関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は自動的に不正利用になるわけではありません。掲載情報が意味のある違いを持ち、正確に説明され、メンテナンスされ、実際のユーザーに使われている場合、大規模なカタログは許容されます。大量であることが、内容の薄い、重複した、誤解を招く、メンテナンスされていない、または人為的に宣伝された掲載情報と組み合わさると、大規模カタログは信頼と安全性の問題になります。

## コンテンツ権利

ClawHub 上のコンテンツが著作権またはその他の権利を侵害していると思われる場合は、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。掲載情報が安全でない、悪意がある、または誤解を招く場合を除き、著作権または権利の申し立てに通常のマーケットプレイス報告を使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツや不正な公開行動を特定するために、自動チェック、統計的な不正利用シグナル、ユーザー報告、スタッフレビューを使用することがあります。シグナル単独で不正利用を証明するものではありません。ClawHub がレビューを必要とするものを判断する助けになります。

ClawHub は次を行うことがあります。

- 違反する掲載情報を非表示、保留、削除、ソフト削除、またはリソースタイプでサポートされている場合はハード削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツをソフト削除する
- 公開アクセスを制限する
- 繰り返し違反する、または重大な違反を行う者を禁止する

明らかな不正利用に対して、警告を先に行う執行を保証するものではありません。報告、モデレーション保留、非表示の掲載情報、禁止、アカウントの状態については、[モデレーションとアカウント安全性](/clawhub/moderation)を参照してください。
