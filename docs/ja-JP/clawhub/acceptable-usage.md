# 許容される利用

ClawHub は、OpenClaw 向けのスキル、Plugin、パッケージ、マーケットプレイスのメタデータをホストします。
コンテンツや公開の仕方が ClawHub に適しているかどうかを判断するには、
このページを使用してください。

これらの規則は、掲載内容の動作、ユーザーに実行を求めるもの、掲載内容の
表現方法、および公開者による ClawHub の検索、インストール、信頼性に関する
機能の使用方法に適用されます。モデレーション状態とアカウントの健全性については、
[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。著作権その他の権利に関する
申し立てについては、[コンテンツの権利に関するリクエスト](/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で理解しやすく、誠意をもって公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | 掲載内容が、ソフトウェアの構築、テスト、移行、デバッグ、文書化、運用に役立つこと。                                               |
| UI、データ、自動化のワークフロー               | 適用範囲が明確で、必要な認証情報が明示され、リスクのある操作にレビュー、ドライラン、プレビュー、確認の手順が含まれていること。 |
| 防御的セキュリティ、モデレーション、不正利用のレビュー | ツールが承認済みのレビュー用として位置付けられ、証拠を保持し、人による承認の境界が明確に保たれていること。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明性のあるセットアップ、明示的な権限を使用すること。                                            |
| 保守されているカタログ                              | 各掲載内容が固有かつ有用で、正確に説明され、合理的に保守されていること。                                                |

コンテキストが重要です。同じトピックでも、限定的な防御目的や
同意に基づく状況では許容される一方、不正利用のワークフローとして
パッケージ化されている場合は許容されません。

## 禁止されるコンテンツ

ClawHub は、不正利用、欺瞞、安全でない
実行、権利侵害を主な目的とするコンテンツをホストしません。

| カテゴリ                                                    | 禁止されるもの                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 不正アクセスまたはセキュリティ回避                      | 認証の回避、アカウントの乗っ取り、レート制限の不正利用、ライブ通話またはエージェントの乗っ取り、再利用可能なセッションの窃取、未承認ユーザーに対するペアリングフローの自動承認。                                                                                                                                                   |
| プラットフォームの不正利用とBAN回避                              | BAN後のステルスアカウント、アカウントの育成や量産、偽のエンゲージメント、複数アカウントの自動化、大量投稿、スパムボット、検出を回避するための自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書や請求書、欺瞞的な決済フロー、スキャム目的の働きかけ、偽の社会的証明、詐欺を目的とする合成IDのワークフロー、または人による明確な承認なしに支出や課金を行うツール。                                                                                                                    |
| プライバシーを侵害する情報拡充または監視                 | スパム目的の連絡先スクレイピング、個人情報の暴露、ストーキング、未承諾の働きかけと組み合わせた見込み客情報の抽出、秘密裏の監視、同意のない生体情報照合、流出データや侵害データダンプの使用。                                                                                                                  |
| 同意のないなりすましまたはID操作       | 顔の入れ替え、デジタルツイン、クローン化されたインフルエンサー、偽の人物像、その他なりすましや誤認を目的として使用されるツール。                                                                                                                                                                                                 |
| 露骨な性的コンテンツまたは安全機能を無効にした成人向け生成 | NSFW画像、動画、コンテンツの生成、第三者APIを利用した成人向けコンテンツのラッパー、または露骨な性的コンテンツを主目的とする掲載内容。                                                                                                                                                       |
| 隠蔽された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、`sh` または `bash` でダウンロードしたコンテンツを実行するなど、明確にレビューできないパイプ・トゥ・シェル型インストーラー、申告されていないシークレットや秘密鍵の要件、明確にレビューできないリモートでの `npx @latest` 実行、または掲載内容の実行に実際に必要なものを隠すメタデータ。 |
| 著作権を侵害する、または権利に違反する素材           | 他者のスキル、Plugin、ドキュメント、ブランド資産、プロプライエタリコードを許可なく再公開すること、ライセンス条件への違反、または原作者や公開者へのなりすまし。                                                                                                                            |

## 禁止されるマーケットプレイス上の行為

ClawHub は、公開者によるマーケットプレイスの使用方法も審査します。ClawHub を使用して、
検索、指標、信頼シグナル、モデレーションシステム、またはユーザーの
注意を操作しないでください。

禁止されるマーケットプレイス上の行為には、次のものが含まれます。

- 実際のユーザー価値がないと思われる、労力の少ない、重複した、プレースホルダー、または
  機械生成の掲載内容を大量に公開すること
- ほぼ同一のスキルやPluginで検索またはカテゴリの表示領域を埋め尽くすこと
- 利用実績、保守、ソースの
  明確性、意味のある差別化がほとんど、またはまったくない掲載内容を数百件公開すること
- 自動化、自己インストールのループ、偽アカウント、組織的な
  活動、有償エンゲージメント、その他の非オーガニックな行為によって、インストール数、ダウンロード数、スター数、その他のエンゲージメント
  指標を人為的に水増しすること
- モデレーション、BAN、公開者の制限、または
  マーケットプレイスの審査を回避するためにアカウントを作成または切り替えること
- 所有権、ソース、機能、セキュリティ態勢、
  インストール要件、または別のプロジェクトや公開者との関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされた
  コンテンツを繰り返しアップロードすること

大量公開が自動的に不正利用となるわけではありません。掲載内容に意味のある違いがあり、正確に説明され、保守され、
実際のユーザーに利用されている場合、大規模なカタログは許容されます。大量であることに加えて、内容が薄い、重複している、誤解を招く、保守されていない、または
人為的に宣伝されている場合、大規模なカタログは信頼性と安全性の問題になります。

## コンテンツの権利

ClawHub 上のコンテンツが自分の著作権その他の権利を侵害していると思われる場合は、
[コンテンツの権利に関するリクエスト](/clawhub/content-rights)を使用してください。掲載内容が安全でない、
悪意がある、または誤解を招くものでもある場合を除き、著作権や権利に関する申し立てに通常のマーケットプレイスの
報告機能を使用しないでください。

## 審査と措置

ClawHub は、安全でないコンテンツや不正な公開行為を特定するために、自動チェック、統計的な不正利用シグナル、ユーザーからの報告、
スタッフによる審査を使用する場合があります。シグナルだけで
不正利用が証明されるわけではなく、ClawHub が審査の必要な対象を判断するために役立ちます。

以下の措置を講じる場合があります。

- 違反する掲載内容を非表示、保留、削除、ソフトデリート、またはリソースの種類で対応している場合は、
  ハードデリートする
- 安全でないリリースのダウンロードまたはインストールをブロックする
- APIトークンを失効させる
- 関連コンテンツをソフトデリートする
- 公開アクセスを制限する
- 違反を繰り返した者、または重大な違反者をBANする

明白な不正利用に対して、事前警告を必ず行うとは限りません。報告、モデレーションによる保留、
非表示の掲載内容、BAN、アカウントの健全性については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。
