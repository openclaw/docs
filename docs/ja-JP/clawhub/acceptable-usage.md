# 許容される利用

ClawHub は OpenClaw の Skills、plugins、packages、marketplace metadata をホストします。
このページは、コンテンツや公開時の挙動が ClawHub に属するかどうかを判断するために使用します。

これらのルールは、listing が何をするか、ユーザーに何の実行を求めるか、自身をどのように表現するか、publishers が ClawHub の discovery、install、trust surfaces をどのように使用するかに適用されます。モデレーション状態とアカウントの standing については、[Moderation and Account Safety](/ja-JP/clawhub/moderation) を参照してください。著作権またはその他の権利に関する申し立てについては、[Content Rights Requests](/ja-JP/clawhub/content-rights) を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で、理解しやすく、善意で公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | listing が、ユーザーによるソフトウェアのビルド、テスト、移行、デバッグ、ドキュメント作成、運用を支援する。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのあるアクションにレビュー、dry-run、プレビュー、または確認パスが含まれる。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが承認済みレビュー向けとして位置付けられ、証拠を保持し、人間による承認の境界を明確に保つ。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明なセットアップ、明示的な権限を使用する。                                            |
| メンテナンスされているカタログ                              | 各 listing が明確に区別でき、有用で、正確に説明され、合理的にメンテナンスされている。                                                |

文脈が重要です。同じトピックでも、狭い防御的または同意に基づく設定では許容される場合があり、不正利用ワークフローとしてパッケージ化されている場合は許容されません。

## 禁止されるコンテンツ

ClawHub は、不正利用、欺瞞、安全でない実行、または権利侵害を主な目的とするコンテンツをホストしません。

| カテゴリ                                                    | 許可されないもの                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 未承認アクセスまたはセキュリティ回避                      | 認証バイパス、アカウント乗っ取り、レート制限の不正利用、live call または agent の乗っ取り、再利用可能なセッションの窃取、または未承認ユーザー向けのペアリングフローの自動承認。                                                                                                                                                   |
| プラットフォーム不正利用と ban 回避                              | ban 後のステルスアカウント、アカウントの warming または farming、偽のエンゲージメント、複数アカウント自動化、大量投稿、スパム bot、または検出回避を目的に作られた自動化。                                                                                                                                          |
| 詐欺、scam、欺瞞的な金融ワークフロー             | 偽の証明書または請求書、欺瞞的な支払いフロー、scam outreach、偽の social proof、詐欺のための synthetic-identity ワークフロー、または明確な人間の承認なしに支出や課金を行うツール。                                                                                                                    |
| プライバシー侵害的な enrich または監視                 | スパム目的の連絡先 scraping、doxxing、stalking、未承諾 outreach と組み合わせたリード抽出、 covert monitoring、同意のない生体情報照合、または漏えいデータや breach dumps の使用。                                                                                                                  |
| 同意のないなりすましまたはアイデンティティ操作       | face swap、digital twins、複製された influencers、偽の personas、またはなりすましや誤認誘導に使われるその他の tooling。                                                                                                                                                                                                 |
| 明示的な性的コンテンツまたは安全機能を無効化した成人向け生成 | NSFW 画像、動画、またはコンテンツ生成、サードパーティ API を囲む成人向けコンテンツ wrapper、または明示的な性的コンテンツを主目的とする listing。                                                                                                                                                       |
| 隠された、安全でない、または誤解を招く実行要件        | 難読化された install commands、明確なレビュー可能性なしにダウンロードしたコンテンツを `sh` または `bash` で実行する pipe-to-shell installers、未申告の secret または private-key 要件、明確なレビュー可能性のない remote `npx @latest` 実行、または listing が実行に本当に必要とするものを隠す metadata。 |
| 著作権侵害または権利侵害のある素材           | 他者の skill、plugin、docs、brand assets、または proprietary code を許可なく再公開すること、ライセンス条項への違反、または元の作者や publisher になりすますこと。                                                                                                                            |

## 禁止される marketplace の挙動

ClawHub は、publishers が marketplace をどのように使用するかもレビューします。ClawHub を使用して discovery、metrics、trust signals、moderation systems、またはユーザーの注意を操作しないでください。

禁止される marketplace の挙動には以下が含まれます。

- 実際のユーザー価値があるように見えない、手間の少ない、重複した、placeholder、または機械生成の listings を大量に一括公開すること
- ほぼ同一の skills または plugins で検索やカテゴリ surfaces をあふれさせること
- 使用状況、メンテナンス、ソースの明確さ、または意味のある差別化がほとんどない listings を数百件公開すること
- 自動化、自己インストール loop、偽アカウント、協調活動、有料エンゲージメント、またはその他の非オーガニックな挙動によって、installs、downloads、stars、またはその他のエンゲージメント指標を人為的につり上げること
- モデレーション、ban、publisher limits、または marketplace review を回避するためにアカウントを作成またはローテーションすること
- 所有権、ソース、機能、セキュリティ体制、install requirements、または別のプロジェクトや publisher との関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は自動的に不正利用になるわけではありません。listings が意味のある差異を持ち、正確に説明され、メンテナンスされ、実際のユーザーに利用されている場合、大規模なカタログは許容されます。大量であることが、薄い、重複した、誤解を招く、メンテナンスされていない、または人為的に宣伝された listings と組み合わさると、大規模なカタログは trust and safety の問題になります。

## コンテンツの権利

ClawHub 上のコンテンツがあなたの著作権またはその他の権利を侵害していると考える場合は、[Content Rights Requests](/ja-JP/clawhub/content-rights) を使用してください。listing が同時に安全でない、悪意がある、または誤解を招く場合を除き、著作権または権利の申し立てに通常の marketplace reports を使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツや不正な公開挙動を特定するために、自動チェック、統計的な不正利用シグナル、ユーザーレポート、スタッフレビューを使用する場合があります。シグナルはそれ自体で不正利用を証明するものではありません。ClawHub がレビューが必要なものを判断する助けになります。

実施する場合がある措置:

- 違反している listings を非表示にする、保留する、削除する、soft-delete する、または resource type でサポートされている場合は hard-delete する
- 安全でない releases の downloads または installs をブロックする
- API tokens を取り消す
- 関連コンテンツを soft-delete する
- 公開アクセスを制限する
- 繰り返し違反する者または重大な違反者を ban する

明白な不正利用に対して、警告を先に行う執行を保証するものではありません。reports、moderation holds、hidden listings、bans、account standing については、[Moderation and Account Safety](/ja-JP/clawhub/moderation) を参照してください。
