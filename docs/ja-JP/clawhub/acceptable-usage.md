# 許容される使用

ClawHub は OpenClaw の Skills、plugins、パッケージ、マーケットプレイスメタデータをホストします。
このページは、コンテンツまたは公開動作が ClawHub に属するかどうかを判断するために使用してください。

これらのルールは、掲載が行うこと、ユーザーに実行を求めること、自身をどのように表現するか、公開者が ClawHub の発見、インストール、信頼のサーフェスをどのように使用するかに適用されます。モデレーション状態とアカウントの状態については、[モデレーションとアカウント安全性](/clawhub/moderation) を参照してください。著作権またはその他の権利の申し立てについては、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights) を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で、理解しやすく、善意で公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される場合                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | 掲載が、ユーザーによるソフトウェアの構築、テスト、移行、デバッグ、ドキュメント作成、運用を支援する。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクの高いアクションにレビュー、ドライラン、プレビュー、確認の経路が含まれる。 |
| 防御的セキュリティ、モデレーション、不正使用レビュー | ツールが承認されたレビュー向けとして位置付けられ、証拠を保持し、人間による承認境界を明確に保つ。                          |
| 個人またはチームのワークフロー                       | ワークフローが同意に基づくアカウント、透明なセットアップ、明示的な権限を使用する。                                            |
| 維持管理されているカタログ                              | 各掲載がそれぞれ明確に異なり、有用で、正確に説明され、合理的に維持管理されている。                                                |

文脈が重要です。同じトピックでも、狭い防御的または同意に基づく設定では許容される一方で、不正使用ワークフローとしてパッケージ化されている場合は許容されません。

## 禁止されるコンテンツ

ClawHub は、不正使用、欺瞞、安全でない実行、または権利侵害を主目的とするコンテンツをホストしません。

| カテゴリ                                                    | 許可されないもの                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 不正アクセスまたはセキュリティ回避                      | 認証バイパス、アカウント乗っ取り、レート制限の悪用、ライブ通話またはエージェント乗っ取り、再利用可能なセッションの窃取、未承認ユーザー向けのペアリングフローの自動承認。                                                                                                                                                   |
| プラットフォームの不正使用と禁止回避                              | 禁止後のステルスアカウント、アカウントのウォーミングまたはファーミング、偽のエンゲージメント、複数アカウント自動化、大量投稿、スパムボット、または検出を避けるために作られた自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書または請求書、欺瞞的な支払いフロー、スキャムのアウトリーチ、偽の社会的証明、詐欺のための合成アイデンティティワークフロー、または明確な人間の承認なしに支出や請求を行うツール。                                                                                                                    |
| プライバシー侵害的なエンリッチメントまたは監視                 | スパムのための連絡先スクレイピング、晒し、ストーキング、未承諾アウトリーチと組み合わせたリード抽出、秘密裏の監視、非同意の生体照合、または流出データや侵害ダンプの使用。                                                                                                                  |
| 非同意のなりすましまたはアイデンティティ操作       | フェイススワップ、デジタルツイン、クローンされたインフルエンサー、偽のペルソナ、またはなりすましや誤解を招く目的で使用されるその他のツール。                                                                                                                                                                                                 |
| 明示的な性的コンテンツまたは安全機能を無効化した成人向け生成 | NSFW 画像、動画、またはコンテンツ生成、サードパーティ API を囲む成人向けコンテンツラッパー、または明示的な性的コンテンツを主目的とする掲載。                                                                                                                                                       |
| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確なレビュー可能性なしにダウンロードされたコンテンツを `sh` または `bash` で実行するような pipe-to-shell インストーラー、宣言されていないシークレットまたは秘密鍵要件、明確なレビュー可能性なしのリモート `npx @latest` 実行、または掲載が実行に本当に必要とするものを隠すメタデータ。 |
| 著作権を侵害する、または権利に違反する素材           | 許可なく他者の skill、plugin、ドキュメント、ブランドアセット、またはプロプライエタリコードを再公開すること、ライセンス条項に違反すること、または元の作者や公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイス動作

ClawHub は、公開者がマーケットプレイスをどのように使用するかもレビューします。発見、指標、信頼シグナル、モデレーションシステム、またはユーザーの注意を操作するために ClawHub を使用しないでください。

禁止されるマーケットプレイス動作には次が含まれます。

- 実際のユーザー価値があるように見えない、労力の低い、重複した、プレースホルダーの、または機械生成された掲載を大量に一括公開すること
- ほぼ同一の skills または plugins で検索やカテゴリサーフェスをあふれさせること
- 利用、維持管理、ソースの明確さ、または意味のある差別化がほとんどないかまったくない数百件の掲載を公開すること
- 自動化、自己インストールループ、偽アカウント、協調的活動、有料エンゲージメント、またはその他の非オーガニックな行動を通じて、インストール数、ダウンロード数、スター、またはその他のエンゲージメント指標を人為的に水増しすること
- モデレーション、禁止、公開者制限、またはマーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有者、ソース、機能、セキュリティ体制、インストール要件、または別のプロジェクトや公開者との関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は自動的に不正使用になるわけではありません。掲載が意味のある形で異なり、正確に説明され、維持管理され、実際のユーザーに使用されている場合、大規模なカタログは許容されます。量が、内容の薄い、重複した、誤解を招く、維持管理されていない、または人為的に宣伝された掲載と組み合わさると、大規模なカタログは信頼と安全性の問題になります。

## コンテンツ権利

ClawHub 上のコンテンツがあなたの著作権またはその他の権利を侵害していると思う場合は、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights) を使用してください。掲載が安全でない、悪意がある、または誤解を招くものでもある場合を除き、著作権または権利の申し立てに通常のマーケットプレイス報告を使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツまたは不正な公開動作を特定するために、自動チェック、統計的な不正使用シグナル、ユーザー報告、スタッフレビューを使用する場合があります。シグナル単体では不正使用を証明しません。ClawHub がレビューの必要なものを判断する助けになります。

当社は次を行う場合があります。

- 違反している掲載を非表示、保留、削除、ソフト削除、またはリソースタイプでサポートされている場合はハード削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツをソフト削除する
- 公開アクセスを制限する
- 繰り返し違反する、または重大な違反をする者を禁止する

明白な不正使用に対して、必ず警告を先に行う執行を保証するものではありません。報告、モデレーション保留、非表示の掲載、禁止、アカウントの状態については、[モデレーションとアカウント安全性](/clawhub/moderation) を参照してください。
