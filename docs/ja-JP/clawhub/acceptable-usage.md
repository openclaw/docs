# 許容される利用

このページでは、ClawHub が許容するスキルやコンテンツの種類と、ホストしない不正利用ワークフローについて説明します。

これらのルールは、意図的に実用性を重視しています。私たちは、単独のキーワードだけでなく、エンドツーエンドの不正利用ワークフローを最も重視します。防御の回避、プラットフォームの悪用、人をだますこと、プライバシーの侵害、または同意のない行為を可能にすることを目的にスキルが作られている場合、そのスキルは ClawHub にふさわしくありません。

## 明示的に許容する最近のパターン

- 実際のコンポーネント、セマンティックトークン、アクセシブルな状態、テスト済みのユーザーフローを使用するフロントエンドおよびデザインシステム作業。
- 一回限りのマークアップではなく、インストール済みのソースコンポーネント、プロジェクトエイリアス、文書化されたバリアントを使用する shadcn/ui の構成。
- コメントを保持し、具体的な UI5 型を使用し、生成されたコントロールインターフェイスをレビュー可能に保つ UI5 の JavaScript から TypeScript への変換。
- 証拠を示し、人間による承認の境界を明確に保つ、防御的なセキュリティレビュー、モデレーションツール、不正利用検出プロンプト。
- 明示的な認証情報、透明なセットアップ、ドライランまたはプレビューモードを備えた、個人またはチームアカウント向けの同意に基づくワークフロー自動化。
- サポート対象のソフトウェアにスコープを限定したドキュメント、移行ランブック、開発者ユーティリティ、テストフィクスチャ。

## 許容されないこと

- セキュリティ回避または不正アクセスのワークフロー。
  - 例: 認証バイパス、アカウント乗っ取り、CAPTCHA バイパス、Cloudflare またはボット対策の回避、レート制限バイパス、保護を破ることを目的としたステルススクレイピング、ライブ通話またはエージェントの乗っ取り、再利用可能なセッションの窃取、未承認ユーザー向けのペアリングフローの自動承認。

- プラットフォームの悪用および禁止回避。
  - 例: 禁止後のステルスアカウント、アカウントのウォーミングやファーミング、偽のエンゲージメント、カルマやフォロワーの育成、複数アカウントの自動化、大量投稿、スパムボット、検出を避けるために作られたマーケットプレイスまたはソーシャルの自動化。

- 詐欺、スキャム、欺瞞的な金融ワークフロー。
  - 例: 偽の証明書、偽の請求書、欺瞞的な支払いフロー、スキャムのアウトリーチ、偽の社会的証明、明確な人間の承認と透明な制御なしに支出や課金を可能にするツール、または詐欺用のアカウント作成を目的とした合成アイデンティティのワークフロー。

- プライバシーを侵害するスクレイピング、エンリッチメント、または監視。
  - 例: スパム目的で連絡先詳細を大規模にスクレイピングすること、晒し、ストーキング、未承諾アウトリーチと組み合わせたリード抽出、密かな監視、明確な同意なしに使用される顔検索または生体認証マッチング、漏えいデータや侵害ダンプの購入、公開、ダウンロード、または運用化。

- 同意のないなりすまし、または欺瞞的なアイデンティティ操作。
  - 例: なりすましや誤解を招く目的で使用される、顔交換、デジタルツイン、偽のペルソナ、クローン化されたインフルエンサー、またはその他のアイデンティティ操作ツール。

- 明示的な性的コンテンツおよび安全機能を無効にした成人向け生成。
  - 例: NSFW 画像、動画、コンテンツ生成、サードパーティ API を囲む成人向けコンテンツのラッパー、または明示的な性的コンテンツを主目的とするスキル。

- 隠された、安全でない、または誤解を招く実行要件。
  - 例: 難読化されたインストールコマンド、`curl | sh`、未申告のシークレット要件、未申告の秘密鍵使用、明確なレビュー可能性のないリモート `npx @latest` 実行、スキルの実行に本当に必要なものを隠す誤解を招くメタデータ。

## 明示的に許容しない最近のパターン

- 「マーケットプレイスで禁止された後にステルス販売者アカウントを作成する。」
- 「未承認ユーザーがペアリングコードを自動的に受け取るように Telegram のペアリングを変更する。」
- 「検出不能な自動化で Reddit/Twitter アカウントを育成する。」
- 「任意の用途向けに専門的な証明書または請求書を生成する。」
- 「安全確認を無効にして NSFW コンテンツを生成する。」
- 「リードをスクレイピングし、連絡先をエンリッチし、大規模なコールドアウトリーチを開始する。」
- 「漏えいデータや侵害ダンプを購入、公開、またはダウンロードする。」
- 「合成アイデンティティまたは CAPTCHA 解決を使って、メールまたはソーシャルアカウントを一括作成する。」

## レビュアー向けの注意

- 文脈が重要です。同じトピックでも、狭い防御的または同意に基づく場面では正当であり、不正利用ワークフローとしてパッケージ化されている場合は許容されないことがあります。
- スキルが回避、欺瞞、または同意のない使用に明確に最適化されている場合は、対応を優先すべきです。
- これらのカテゴリで繰り返しアップロードすることは、コンテンツの非表示およびアカウント禁止の根拠になります。

## 執行

- 違反するスキルを非表示、削除、または完全削除する場合があります。
- トークンの取り消し、関連コンテンツのソフト削除、繰り返しまたは重大な違反者の禁止を行う場合があります。
- 明白な不正利用に対して、警告を先に行う執行は保証しません。
