# 許容される利用

ClawHub は OpenClaw 向けの Skills、プラグイン、パッケージ、マーケットプレイスメタデータをホストします。
コンテンツや公開動作が ClawHub に属するかどうかを判断するために、このページを使用してください。

これらのルールは、掲載が何をするか、ユーザーに何の実行を求めるか、自身をどう表現するか、公開者が ClawHub の発見、インストール、信頼のサーフェスをどう使うかに適用されます。モデレーション状態とアカウント状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。著作権またはその他の権利に関する申し立てについては、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で、理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | 掲載が、ユーザーによるソフトウェアのビルド、テスト、移行、デバッグ、文書化、運用を支援する。                                               |
| UI、データ、自動化ワークフロー               | 範囲が明確で、必要な認証情報が明示され、リスクのある操作にレビュー、ドライラン、プレビュー、確認のパスが含まれている。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが承認済みレビュー向けとして位置付けられ、証拠を保持し、人間による承認境界を明確に保っている。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明なセットアップ、明示的な権限を使用している。                                            |
| 維持管理されているカタログ                              | 各掲載が個別性と有用性を持ち、正確に説明され、合理的に維持管理されている。                                                |

文脈は重要です。同じトピックでも、狭い防御的または同意に基づく設定では許容される一方、不正利用ワークフローとしてパッケージ化されている場合は許容されません。

| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| プラットフォームの不正利用と禁止回避                              | 禁止後のステルスアカウント、アカウントのウォーミングまたはファーミング、偽のエンゲージメント、複数アカウントの自動化、大量投稿、スパムボット、または検出を避けるために構築された自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書や請求書、欺瞞的な支払いフロー、スキャムのアウトリーチ、偽の社会的証明、不正行為のための合成アイデンティティワークフロー、または明確な人間の承認なしに支出や請求を行うツール。                                                                                                                    |

| 同意のないなりすましまたはアイデンティティ操作       | フェイススワップ、デジタルツイン、複製されたインフルエンサー、偽のペルソナ、またはなりすましや誤解を招くために使用されるその他のツール。                                                                                                                                                                                                 |

| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確なレビュー可能性なしにダウンロードしたコンテンツを `sh` または `bash` で実行するようなパイプからシェルへのインストーラー、未宣言のシークレットまたは秘密鍵要件、明確なレビュー可能性なしのリモート `npx @latest` 実行、または掲載が実行に本当に必要とするものを隠すメタデータ。 |
| 著作権侵害または権利侵害にあたる素材           | 他者の Skill、プラグイン、ドキュメント、ブランドアセット、またはプロプライエタリコードを許可なく再公開すること、ライセンス条件に違反すること、または元の作者や公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイス上の行為

ClawHub は、公開者がマーケットプレイスをどう使用するかもレビューします。発見、指標、信頼シグナル、モデレーションシステム、またはユーザーの注意を操作するために ClawHub を使用しないでください。

禁止されるマーケットプレイス上の行為には以下が含まれます。

- 実際のユーザー価値があるように見えない、低労力、重複、プレースホルダー、または機械生成の掲載を大量に公開すること
- ほぼ同一の Skills やプラグインで検索またはカテゴリサーフェスをあふれさせること
- 利用、メンテナンス、ソースの明確さ、または意味のある差別化がほとんど、あるいはまったくない掲載を数百件公開すること
- 自動化、自己インストールループ、偽アカウント、協調活動、有償エンゲージメント、またはその他の非オーガニックな行為によって、インストール、ダウンロード、スター、またはその他のエンゲージメント指標を人為的に膨らませること
- モデレーション、禁止、公開者制限、またはマーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有者、ソース、機能、セキュリティ態勢、インストール要件、または別のプロジェクトや公開者との関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は自動的に不正利用とは見なされません。掲載が意味のある違いを持ち、正確に説明され、維持管理され、実際のユーザーに使われている場合、大規模なカタログは許容されます。量が、薄い、重複した、誤解を招く、維持管理されていない、または人為的に宣伝された掲載と組み合わさると、大規模なカタログは信頼と安全性の問題になります。

## コンテンツ権利

ClawHub 上のコンテンツがあなたの著作権またはその他の権利を侵害していると思われる場合は、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。掲載が安全でない、悪意がある、または誤解を招く場合を除き、著作権または権利の申し立てに通常のマーケットプレイス報告を使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツまたは不正な公開行為を特定するために、自動チェック、統計的な不正利用シグナル、ユーザー報告、スタッフレビューを使用することがあります。シグナル単体では不正利用を証明しませんが、ClawHub がレビューを必要とする対象を判断する助けになります。

当社は以下を行う場合があります。

- 違反する掲載を非表示、保留、削除、論理削除、またはリソースタイプでサポートされている場合は物理削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツを論理削除する
- 公開アクセスを制限する
- 繰り返し違反する者または重大な違反者を禁止する

明らかな不正利用に対して、警告を先に行う執行を保証するものではありません。報告、モデレーション保留、非表示の掲載、禁止、アカウント状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。
