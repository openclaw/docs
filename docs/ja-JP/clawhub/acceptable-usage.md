# 許容される利用

ClawHub は OpenClaw 向けの Skills、plugins、packages、およびマーケットプレイスメタデータをホストします。
コンテンツや公開時の挙動が ClawHub に属するかどうかを判断するために、このページを使用してください。

これらのルールは、リスティングが何をするか、ユーザーに何を実行するよう求めるか、自身をどのように表現するか、公開者が ClawHub の発見、インストール、信頼サーフェスをどのように使うかに適用されます。モデレーション状態とアカウントの状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。著作権またはその他の権利に関する申し立てについては、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で、理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | リスティングが、ユーザーによるソフトウェアの構築、テスト、移行、デバッグ、ドキュメント作成、運用に役立つ。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのある操作にレビュー、ドライラン、プレビュー、または確認の経路が含まれる。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが承認済みレビュー向けとして位置付けられ、証拠を保持し、人間による承認境界を明確に保つ。                          |
| 個人またはチームのワークフロー                       | ワークフローが同意に基づくアカウント、透明なセットアップ、明示的な権限を使用する。                                            |
| メンテナンスされているカタログ                              | 各リスティングが明確に区別でき、有用で、正確に説明され、合理的にメンテナンスされている。                                                |

文脈が重要です。同じトピックでも、狭い防御的または同意に基づく設定では許容される場合があり、不正利用ワークフローとしてパッケージ化されている場合は許容されません。

## 許可されないコンテンツ

ClawHub は、不正利用、欺瞞、安全でない実行、または権利侵害を主目的とするコンテンツをホストしません。

| カテゴリ                                                    | 許可されないもの                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 不正アクセスまたはセキュリティ回避                      | 認証バイパス、アカウント乗っ取り、レート制限の不正利用、ライブ通話またはエージェントの乗っ取り、再利用可能なセッションの窃取、または未承認ユーザー向けペアリングフローの自動承認。                                                                                                                                                   |
| プラットフォーム不正利用と禁止回避                              | 禁止後のステルスアカウント、アカウントのウォーミングまたはファーミング、偽のエンゲージメント、複数アカウント自動化、大量投稿、スパムボット、または検出回避のために構築された自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書または請求書、欺瞞的な支払いフロー、スキャム勧誘、偽の社会的証明、詐欺のための合成IDワークフロー、または明確な人間の承認なしに支出や課金を行うツール。                                                                                                                    |
| プライバシー侵害的なエンリッチメントまたは監視                 | スパム目的の連絡先スクレイピング、晒し、ストーキング、未承諾のアウトリーチと組み合わせたリード抽出、秘密裏の監視、同意のない生体照合、または漏えいデータや侵害ダンプの使用。                                                                                                                  |
| 同意のないなりすましまたはアイデンティティ操作       | 顔交換、デジタルツイン、クローン化されたインフルエンサー、偽のペルソナ、またはなりすましや誤認を目的として使用されるその他のツール。                                                                                                                                                                                                 |
| 明示的な性的コンテンツまたは安全制限を無効化した成人向け生成 | NSFW 画像、動画、またはコンテンツ生成、サードパーティ API を包む成人向けコンテンツラッパー、または明示的な性的コンテンツを主目的とするリスティング。                                                                                                                                                       |
| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確なレビュー可能性なしにダウンロードしたコンテンツを `sh` または `bash` で実行するような pipe-to-shell インストーラー、未申告のシークレットまたは秘密鍵要件、明確なレビュー可能性なしのリモート `npx @latest` 実行、またはリスティングの実行に本当に必要なものを隠すメタデータ。 |
| 著作権侵害または権利侵害にあたる素材           | 他者の skill、plugin、docs、ブランドアセット、またはプロプライエタリコードを許可なく再公開すること、ライセンス条項に違反すること、または元の作者や公開者になりすますこと。                                                                                                                            |

## 許可されないマーケットプレイス上の挙動

ClawHub は、公開者がマーケットプレイスをどのように使用するかもレビューします。発見、指標、信頼シグナル、モデレーションシステム、またはユーザーの注意を操作するために ClawHub を使用しないでください。

許可されないマーケットプレイス上の挙動には、次が含まれます。

- 実際のユーザー価値があるように見えない、低労力、重複、プレースホルダー、または機械生成のリスティングを大量公開すること
- ほぼ同一の skills または plugins で検索やカテゴリサーフェスをあふれさせること
- 利用、メンテナンス、ソースの明確さ、または意味のある差別化がほとんどないかまったくないリスティングを数百件公開すること
- 自動化、自己インストールのループ、偽アカウント、協調的な活動、有料エンゲージメント、またはその他の非オーガニックな挙動によって、インストール、ダウンロード、スター、またはその他のエンゲージメント指標を人為的に膨らませること
- モデレーション、禁止、公開者制限、またはマーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有権、ソース、機能、セキュリティ態勢、インストール要件、または別のプロジェクトや公開者との関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は自動的に不正利用とは見なされません。リスティングが意味のある形で異なり、正確に説明され、メンテナンスされ、実際のユーザーに使用されている場合、大規模なカタログは許容されます。量が、内容の薄い、重複した、誤解を招く、メンテナンスされていない、または人為的に宣伝されたリスティングと組み合わさると、大規模なカタログは信頼と安全性の問題になります。

## コンテンツ権利

ClawHub 上のコンテンツが自身の著作権またはその他の権利を侵害していると考える場合は、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。リスティングが同時に安全でない、悪意がある、または誤解を招く場合を除き、著作権または権利に関する申し立てに通常のマーケットプレイス報告を使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツや不正な公開行動を特定するために、自動チェック、統計的な不正利用シグナル、ユーザー報告、スタッフレビューを使用する場合があります。シグナルはそれ自体で不正利用を証明するものではありません。ClawHub がレビューの必要な対象を判断する助けになります。

当社は次を行う場合があります。

- 違反リスティングを非表示、保留、削除、ソフト削除、またはリソースタイプでサポートされている場合はハード削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツをソフト削除する
- 公開アクセスを制限する
- 繰り返し違反する、または重大な違反を行う者を禁止する

明白な不正利用に対して、警告を先に行う執行を保証するものではありません。報告、モデレーション保留、非表示のリスティング、禁止、アカウントの状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。
