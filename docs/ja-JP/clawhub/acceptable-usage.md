# 許容される利用

ClawHub は OpenClaw 向けの Skills、plugins、パッケージ、マーケットプレイスメタデータをホストします。
このページは、コンテンツや公開時の動作が ClawHub に属するかどうかを判断するために使用します。

これらのルールは、リスティングが何をするか、ユーザーに何の実行を求めるか、自身をどのように表現するか、公開者が ClawHub の発見、インストール、信頼のサーフェスをどのように使用するかに適用されます。モデレーション状態とアカウントの状態については、[モデレーションとアカウント安全性](/clawhub/moderation) を参照してください。著作権またはその他の権利に関する申し立てについては、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights) を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | リスティングが、ユーザーによるソフトウェアの構築、テスト、移行、デバッグ、ドキュメント化、運用を支援する。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのある操作にレビュー、ドライラン、プレビュー、確認のパスが含まれている。 |
| 防御的セキュリティ、モデレーション、不正利用レビュー | ツールが承認済みレビュー向けとして位置付けられ、証拠を保持し、人間による承認の境界を明確に保つ。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明なセットアップ、明示的な権限を使用する。                                            |
| 保守されているカタログ                              | 各リスティングが個別で、有用で、正確に説明され、適切に保守されている。                                                |

文脈が重要です。同じトピックでも、限定的な防御的設定や同意に基づく設定では許容される一方、不正利用ワークフローとしてパッケージ化されている場合は許容されないことがあります。

## 禁止されるコンテンツ

| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| プラットフォーム不正利用と禁止回避                              | 禁止後のステルスアカウント、アカウントのウォーミングまたはファーミング、偽のエンゲージメント、複数アカウントの自動化、大量投稿、スパムボット、検出回避のために構築された自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書または請求書、欺瞞的な支払いフロー、スキャムのアウトリーチ、偽の社会的証明、不正行為のための合成アイデンティティワークフロー、明確な人間の承認なしに支出または課金を行うツール。                                                                                                                    |

| 同意のないなりすましまたはアイデンティティ操作       | フェイススワップ、デジタルツイン、クローン化されたインフルエンサー、偽のペルソナ、またはなりすましや誤認誘導に使用されるその他のツール。                                                                                                                                                                                                 |

| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確なレビュー可能性なしに `sh` または `bash` で実行されるダウンロード済みコンテンツなどの pipe-to-shell インストーラー、宣言されていないシークレットまたは秘密鍵の要件、明確なレビュー可能性なしのリモート `npx @latest` 実行、リスティングの実行に本当に必要なものを隠すメタデータ。 |
| 著作権侵害または権利侵害にあたる素材           | 他者の Skills、plugin、ドキュメント、ブランドアセット、またはプロプライエタリコードを許可なく再公開すること。ライセンス条項に違反すること。元の作者または公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイス行為

ClawHub は、公開者がマーケットプレイスをどのように使用するかもレビューします。ClawHub を使用して、発見、指標、信頼シグナル、モデレーションシステム、ユーザーの注意を操作してはなりません。

禁止されるマーケットプレイス行為には以下が含まれます。

- 実際のユーザー価値があるように見えない、低労力、重複、プレースホルダー、または
  機械生成のリスティングを大量に公開すること
- ほぼ同一の Skills または plugins で検索やカテゴリのサーフェスをあふれさせること
- 利用、保守、ソースの明確性、意味のある差別化がほとんどまたはまったくないリスティングを数百件公開すること
- 自動化、自己インストールループ、偽アカウント、組織的活動、有料エンゲージメント、またはその他の非オーガニックな行為を通じて、インストール、ダウンロード、スター、その他のエンゲージメント指標を人為的に膨らませること
- モデレーション、禁止、公開者制限、またはマーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有権、ソース、機能、セキュリティ態勢、インストール要件、または別のプロジェクトや公開者との提携についてユーザーを誤解させること
- すでに非表示、削除、またはブロックされたコンテンツを、根本的な問題を修正せずに繰り返しアップロードすること

大量公開は自動的に不正利用とはみなされません。リスティングが意味のある違いを持ち、正確に説明され、保守され、実際のユーザーに使用されている場合、大規模なカタログは許容されます。大量であることが、内容の薄い、重複した、誤解を招く、保守されていない、または人為的に宣伝されたリスティングと組み合わさると、大規模なカタログは信頼と安全性の問題になります。

## コンテンツ権利

ClawHub 上のコンテンツが著作権またはその他の権利を侵害していると思われる場合は、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights) を使用してください。リスティングが同時に安全でない、悪意がある、または誤解を招く場合を除き、著作権または権利に関する申し立てに通常のマーケットプレイス報告を使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツまたは不正利用的な公開行為を特定するために、自動チェック、統計的な不正利用シグナル、ユーザー報告、スタッフレビューを使用することがあります。シグナルだけで不正利用が証明されるわけではありません。ClawHub がレビューの必要なものを判断する助けになります。

当社は以下を行う場合があります。

- 違反しているリスティングを非表示、保留、削除、ソフトデリート、またはリソースタイプでサポートされている場合は
  ハードデリートする
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツをソフトデリートする
- 公開アクセスを制限する
- 繰り返し違反する者または重大な違反者を禁止する

明らかな不正利用に対して、警告を先に行う執行を保証するものではありません。報告、モデレーション保留、非表示リスティング、禁止、アカウント状態については、[モデレーションとアカウント安全性](/clawhub/moderation) を参照してください。
