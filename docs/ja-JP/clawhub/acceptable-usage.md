# 許容される利用

ClawHub は OpenClaw 向けの Skills、Plugin、パッケージ、マーケットプレイスメタデータをホストします。
コンテンツや公開時の挙動が ClawHub に属するかどうかを判断するために、このページを使用してください。

これらのルールは、掲載項目が何をするか、ユーザーに何の実行を求めるか、自身をどのように表現するか、公開者が ClawHub の発見、インストール、信頼のサーフェスをどのように使用するかに適用されます。モデレーション状態とアカウントの状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。著作権またはその他の権利に関する申し立てについては、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | 掲載項目が、ユーザーによるソフトウェアの構築、テスト、移行、デバッグ、ドキュメント化、または運用に役立つ。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのあるアクションにレビュー、ドライラン、プレビュー、または確認の経路が含まれている。 |
| 防御的セキュリティ、モデレーション、不正使用レビュー | ツールが許可されたレビュー向けとして位置付けられ、証拠を保持し、人間による承認の境界を明確に保っている。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明なセットアップ、明示的な権限を使用している。                                            |
| メンテナンスされているカタログ                              | 各掲載項目が区別可能で、有用で、正確に説明され、合理的にメンテナンスされている。                                                |

文脈が重要です。同じトピックでも、狭い防御的な用途や同意に基づく設定では許容され、悪用ワークフローとしてパッケージ化されている場合は許容されないことがあります。

## 禁止されるコンテンツ

ClawHub は、主な目的が悪用、欺瞞、安全でない実行、または権利侵害であるコンテンツをホストしません。

| カテゴリ                                                    | 許可されないもの                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 不正アクセスまたはセキュリティ回避                      | 認証回避、アカウント乗っ取り、レート制限の悪用、ライブ通話またはエージェントの乗っ取り、再利用可能なセッションの窃取、または未承認ユーザー向けのペアリングフローの自動承認。                                                                                                                                                   |
| プラットフォーム悪用と禁止回避                              | 禁止後のステルスアカウント、アカウントのウォーミングまたはファーミング、偽のエンゲージメント、複数アカウント自動化、大量投稿、スパムボット、または検出回避を目的として構築された自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書や請求書、欺瞞的な支払いフロー、スキャムのアウトリーチ、偽の社会的証明、詐欺目的の合成 ID ワークフロー、または明確な人間の承認なしに支出や課金を行うツール。                                                                                                                    |
| プライバシー侵害的なエンリッチメントまたは監視                 | スパム目的の連絡先スクレイピング、晒し、ストーキング、未承諾アウトリーチと組み合わせたリード抽出、秘匿監視、同意のない生体照合、または漏えいデータや侵害ダンプの使用。                                                                                                                  |
| 同意のないなりすましまたは ID 操作       | フェイススワップ、デジタルツイン、クローン化されたインフルエンサー、偽のペルソナ、またはなりすましや誤認を目的として使用されるその他のツール。                                                                                                                                                                                                 |
| 明示的な性的コンテンツまたは安全機能が無効化された成人向け生成 | NSFW 画像、動画、またはコンテンツ生成、サードパーティ API の周囲に構築された成人向けコンテンツラッパー、または主な目的が明示的な性的コンテンツである掲載項目。                                                                                                                                                       |
| 隠された、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確なレビュー可能性なしにダウンロードされたコンテンツを `sh` または `bash` で実行する pipe-to-shell インストーラー、宣言されていないシークレットまたは秘密鍵の要件、明確なレビュー可能性なしのリモート `npx @latest` 実行、または掲載項目の実行に本当に必要なものを隠すメタデータ。 |
| 著作権侵害または権利侵害のある素材           | 他者の Skill、Plugin、ドキュメント、ブランドアセット、またはプロプライエタリコードを許可なく再公開すること、ライセンス条件に違反すること、または元の作者や公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイス上の挙動

ClawHub は、公開者がマーケットプレイスをどのように使用するかもレビューします。発見、指標、信頼シグナル、モデレーションシステム、またはユーザーの注意を操作するために ClawHub を使用しないでください。

禁止されるマーケットプレイス上の挙動には、次が含まれます。

- 実際のユーザー価値があるように見えない、低労力、重複、プレースホルダー、または機械生成の掲載項目を大量に一括公開すること
- ほぼ同一の Skills または Plugin で検索やカテゴリのサーフェスをあふれさせること
- 利用、メンテナンス、ソースの明確性、または意味のある差別化がほとんどない、またはまったくない掲載項目を数百件公開すること
- 自動化、自己インストールループ、偽アカウント、協調的な活動、有料エンゲージメント、またはその他の非オーガニックな挙動を通じて、インストール数、ダウンロード数、スター数、またはその他のエンゲージメント指標を人為的に膨らませること
- モデレーション、禁止、公開者制限、またはマーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有者、ソース、機能、セキュリティ姿勢、インストール要件、または別のプロジェクトや公開者との関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされたコンテンツを繰り返しアップロードすること

大量公開は自動的に悪用とはみなされません。掲載項目が意味のある形で異なり、正確に説明され、メンテナンスされ、実際のユーザーに使用されている場合、大規模なカタログは許容されます。量が、内容の薄い、重複した、誤解を招く、メンテナンスされていない、または人為的に宣伝された掲載項目と組み合わさる場合、大規模なカタログは信頼と安全性の問題になります。

## コンテンツの権利

ClawHub 上のコンテンツが自分の著作権またはその他の権利を侵害していると考える場合は、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights)を使用してください。掲載項目が安全でない、悪意がある、または誤解を招く場合を除き、著作権または権利に関する申し立てに通常のマーケットプレイス報告を使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツや悪用的な公開行動を特定するために、自動チェック、統計的な悪用シグナル、ユーザー報告、スタッフレビューを使用することがあります。シグナルそれ自体が悪用を証明するわけではありません。ClawHub がレビューを必要とするものを判断する助けになります。

当社は次を行う場合があります。

- 違反している掲載項目を非表示、保留、削除、ソフト削除、またはリソースタイプでサポートされている場合は完全削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツをソフト削除する
- 公開アクセスを制限する
- 繰り返しの違反者または重大な違反者を禁止する

明らかな悪用に対して、警告を先に行う執行を保証するものではありません。報告、モデレーション保留、非表示の掲載項目、禁止、アカウントの状態については、[モデレーションとアカウントの安全性](/clawhub/moderation)を参照してください。
