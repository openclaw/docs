# 許容される利用

このページでは、ClawHub が許容するスキルとコンテンツの種類、およびホストしない不正利用ワークフローについて説明します。

これらのルールは意図的に実用性を重視しています。私たちは、単独のキーワードだけでなく、エンドツーエンドの不正利用ワークフローを最も重視します。防御を回避する、プラットフォームを悪用する、人をだます、プライバシーを侵害する、または同意のない行為を可能にする目的でスキルが作られている場合、それは ClawHub に属しません。

## 明示的に許容している最近のパターン

- 実際のコンポーネント、セマンティックトークン、アクセシブルな状態、テスト済みのユーザーフローを使用するフロントエンドおよびデザインシステムの作業。
- 一回限りのマークアップではなく、インストール済みのソースコンポーネント、プロジェクトエイリアス、文書化されたバリアントを使用する shadcn/ui の構成。
- コメントを保持し、具体的な UI5 型を使用し、生成されたコントロールインターフェイスをレビュー可能な状態に保つ UI5 JavaScript-to-TypeScript 変換。
- 証拠を示し、人間の承認境界を明確に保つ、防御的なセキュリティレビュー、モデレーションツール、不正利用検出プロンプト。
- 明示的な認証情報、透明性のあるセットアップ、ドライランまたはプレビューモードを備えた、個人またはチームアカウント向けの同意ベースのワークフロー自動化。
- サポート対象ソフトウェアに範囲を限定したドキュメント、移行ランブック、開発者ユーティリティ、テストフィクスチャ。

## 許容されないもの

- セキュリティ回避または不正アクセスのワークフロー。
  - 例: 認証回避、アカウント乗っ取り、CAPTCHA 回避、Cloudflare またはボット対策の回避、レート制限回避、保護を破ることを目的としたステルススクレイピング、ライブ通話またはエージェントの乗っ取り、再利用可能なセッション窃取、未承認ユーザー向けペアリングフローの自動承認。

- プラットフォームの悪用およびBAN回避。
  - 例: BAN 後のステルスアカウント、アカウントのウォーミング/ファーミング、偽のエンゲージメント、カルマまたはフォロワーの育成、複数アカウントの自動化、大量投稿、スパムボット、検出を避けるために構築されたマーケットプレイスまたはソーシャル自動化。

- 詐欺、詐欺的行為、および欺瞞的な金融ワークフロー。
  - 例: 偽の証明書、偽の請求書、欺瞞的な決済フロー、詐欺的なアウトリーチ、偽の社会的証明、明確な人間の承認と透明な制御なしに支出または課金を可能にするツール、または詐欺目的でアカウントを作成するために構築された合成IDワークフロー。

- プライバシーを侵害するスクレイピング、エンリッチメント、または監視。
  - 例: スパム目的で連絡先詳細を大規模にスクレイピングすること、晒し、ストーキング、未承諾のアウトリーチと組み合わせたリード抽出、秘密裏の監視、明確な同意なしに使用される顔検索または生体照合、または漏洩データや侵害ダンプの購入、公開、ダウンロード、運用化。

- 同意のないなりすまし、または欺瞞的なアイデンティティ操作。
  - 例: 顔交換、デジタルツイン、偽ペルソナ、クローン化されたインフルエンサー、またはなりすましや誤認誘導に使用されるその他のアイデンティティ操作ツール。

- 露骨な性的コンテンツおよび安全機能を無効にした成人向け生成。
  - 例: NSFW 画像/動画/コンテンツ生成、サードパーティAPIを包む成人向けコンテンツラッパー、または主目的が露骨な性的コンテンツであるスキル。

- 隠された、安全でない、または誤解を招く実行要件。
  - 例: 難読化されたインストールコマンド、`curl | sh`、未申告のシークレット要件、未申告の秘密鍵使用、明確なレビュー可能性のないリモート `npx @latest` 実行、スキルが実行に本当に必要とするものを隠す誤解を招くメタデータ。

## 明示的に許容していない最近のパターン

- 「マーケットプレイスでBANされた後にステルス販売者アカウントを作成する。」
- 「未承認ユーザーがペアリングコードを自動的に受け取るように Telegram のペアリングを変更する。」
- 「検出不能な自動化で Reddit/Twitter アカウントを育成する。」
- 「任意の用途向けに専門的な証明書または請求書を生成する。」
- 「安全チェックを無効にして NSFW コンテンツを生成する。」
- 「リードをスクレイピングし、連絡先をエンリッチし、大規模なコールドアウトリーチを開始する。」
- 「漏洩データまたは侵害ダンプを購入、公開、またはダウンロードする。」
- 「合成IDまたは CAPTCHA 解決を使用して、メールまたはソーシャルアカウントを一括作成する。」

## レビュー担当者向けメモ

- コンテキストが重要です。同じトピックでも、範囲の狭い防御的または同意ベースの設定では正当であり、不正利用ワークフローとしてパッケージ化されている場合は許容されません。
- スキルが回避、欺瞞、または同意のない使用に明確に最適化されている場合は、対応に傾けるべきです。
- これらのカテゴリで繰り返しアップロードされた場合、コンテンツの非表示およびアカウントのBANの根拠になります。

## 適用措置

- 違反するスキルを非表示、削除、または完全削除する場合があります。
- トークンを取り消し、関連コンテンツをソフト削除し、繰り返し違反する、または重大な違反を行うユーザーをBANする場合があります。
- 明白な不正利用に対して、警告を先行する適用措置を保証しません。
