# 許容される使用

ClawHub は OpenClaw のスキル、プラグイン、パッケージ、マーケットプレイスメタデータをホストします。
このページを使って、コンテンツや公開動作が ClawHub に属するかどうかを判断してください。

これらのルールは、リスティングが何をするか、ユーザーに何を実行するよう求めるか、どのように
自分自身を表現するか、そして公開者が ClawHub の発見、インストール、信頼のサーフェスを
どのように使用するかに適用されます。モデレーション状態とアカウントの状態については、
[モデレーションとアカウントの安全性](/ja-JP/clawhub/moderation) を参照してください。著作権またはその他の権利
の申し立てについては、[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights) を参照してください。

## 許可されるコンテンツ

ClawHub は、有用で、理解しやすく、誠実に公開されたコンテンツを歓迎します。

| カテゴリ                                         | 許可される条件                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 開発者の生産性                           | リスティングが、ユーザーによるソフトウェアのビルド、テスト、移行、デバッグ、ドキュメント化、または運用を支援する。                                               |
| UI、データ、自動化ワークフロー               | スコープが明確で、必要な認証情報が明示され、リスクのあるアクションにレビュー、ドライラン、プレビュー、または確認の経路が含まれている。 |
| 防御的セキュリティ、モデレーション、不正使用レビュー | ツールが認可されたレビュー向けに位置付けられ、証拠を保持し、人間による承認の境界を明確に保っている。                          |
| 個人またはチームのワークフロー                       | ワークフローが、同意に基づくアカウント、透明性のあるセットアップ、明示的な権限を使用している。                                            |
| メンテナンスされているカタログ                              | 各リスティングが個別で、有用で、正確に説明され、妥当にメンテナンスされている。                                                |

文脈が重要です。同じトピックでも、狭い防御的または
同意に基づく設定では許容される一方で、不正使用ワークフローとしてパッケージ化されている場合は許容されません。

## 禁止されるコンテンツ

ClawHub は、主な目的が不正使用、欺瞞、安全でない
実行、または権利侵害であるコンテンツをホストしません。

| カテゴリ                                                    | 許可されないもの                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 認可されていないアクセスまたはセキュリティ回避                      | 認証バイパス、アカウント乗っ取り、レート制限の不正使用、ライブ通話またはエージェントの乗っ取り、再利用可能なセッションの窃取、または未承認ユーザー向けのペアリングフローの自動承認。                                                                                                                                                   |
| プラットフォームの不正使用と禁止回避                              | 禁止後のステルスアカウント、アカウントのウォーミングやファーミング、偽のエンゲージメント、複数アカウントの自動化、大量投稿、スパムボット、または検出回避のために構築された自動化。                                                                                                                                          |
| 詐欺、スキャム、欺瞞的な金融ワークフロー             | 偽の証明書や請求書、欺瞞的な支払いフロー、スキャムのアウトリーチ、偽の社会的証明、不正行為のための合成IDワークフロー、または明確な人間の承認なしに支出や課金を行うツール。                                                                                                                    |
| プライバシー侵害的なエンリッチメントまたは監視                 | スパム目的の連絡先スクレイピング、晒し、ストーキング、未承諾アウトリーチと組み合わせたリード抽出、隠れた監視、非同意の生体照合、または漏洩データや侵害ダンプの使用。                                                                                                                  |
| 非同意のなりすましまたはアイデンティティ操作       | フェイススワップ、デジタルツイン、複製されたインフルエンサー、偽の人格、またはなりすましや誤解を招くために使用されるその他のツール。                                                                                                                                                                                                 |
| 明示的な性的コンテンツまたは安全設定を無効にした成人向け生成 | NSFW 画像、動画、またはコンテンツ生成。サードパーティ API の成人向けコンテンツラッパー。または主な目的が明示的な性的コンテンツであるリスティング。                                                                                                                                                       |
| 隠れた、安全でない、または誤解を招く実行要件        | 難読化されたインストールコマンド、明確なレビュー可能性なしに `sh` または `bash` で実行されるダウンロード済みコンテンツなどの pipe-to-shell インストーラー、未申告のシークレットまたは秘密鍵要件、明確なレビュー可能性なしのリモート `npx @latest` 実行、またはリスティングの実行に本当に必要なものを隠すメタデータ。 |
| 著作権を侵害する、または権利を侵害する素材           | 他者のスキル、プラグイン、ドキュメント、ブランドアセット、またはプロプライエタリコードを許可なく再公開すること、ライセンス条項に違反すること、または元の作者や公開者になりすますこと。                                                                                                                            |

## 禁止されるマーケットプレイス上の行動

ClawHub は、公開者がマーケットプレイスをどのように使用するかもレビューします。ClawHub を使用して、
発見、指標、信頼シグナル、モデレーションシステム、またはユーザーの
注意を操作しないでください。

禁止されるマーケットプレイス上の行動には以下が含まれます。

- 実際のユーザー価値があるように見えない、低労力、重複、プレースホルダー、または
  機械生成のリスティングを大量に公開すること
- ほぼ同一のスキルやプラグインで検索またはカテゴリサーフェスをあふれさせること
- 利用、メンテナンス、ソースの明確さ、または意味のある差別化がほとんどない、またはまったくない
  数百件のリスティングを公開すること
- 自動化、自己インストールループ、偽アカウント、協調的
  活動、有料エンゲージメント、またはその他の非オーガニックな行動を通じて、インストール数、ダウンロード数、スター数、またはその他のエンゲージメント
  指標を人為的に増やすこと
- モデレーション、禁止、公開者制限、または
  マーケットプレイスレビューを回避するためにアカウントを作成またはローテーションすること
- 所有権、ソース、機能、セキュリティ態勢、
  インストール要件、または他のプロジェクトや公開者との関係についてユーザーを誤解させること
- 根本的な問題を修正せずに、すでに非表示、削除、またはブロックされた
  コンテンツを繰り返しアップロードすること

大量公開は自動的に不正使用とはなりません。リスティングが意味のある形で異なり、正確に説明され、メンテナンスされ、
実際のユーザーに使用されている場合、大規模なカタログは許容されます。大量であることが、薄い、重複した、誤解を招く、メンテナンスされていない、または
人為的に宣伝されたリスティングと組み合わさると、大規模なカタログは信頼と安全性の問題になります。

## コンテンツ権利

ClawHub 上のコンテンツがあなたの著作権またはその他の権利を侵害していると思われる場合は、
[コンテンツ権利リクエスト](/ja-JP/clawhub/content-rights) を使用してください。リスティングが安全でない、悪意がある、
または誤解を招く場合を除き、著作権または権利の申し立てに通常のマーケットプレイス
レポートを使用しないでください。

## レビューと執行

ClawHub は、安全でないコンテンツや不正な公開行動を特定するために、自動チェック、統計的な不正使用シグナル、ユーザーレポート、
スタッフレビューを使用する場合があります。シグナルはそれ自体で不正使用を証明するものではありません。
ClawHub がレビューを必要とするものを判断する助けになります。

ClawHub は以下を行う場合があります。

- 違反リスティングを非表示、保留、削除、ソフト削除、またはリソースタイプでサポートされる場合は
  ハード削除する
- 安全でないリリースのダウンロードまたはインストールをブロックする
- API トークンを取り消す
- 関連コンテンツをソフト削除する
- 公開アクセスを制限する
- 繰り返し違反する者または重大な違反者を禁止する

明白な不正使用に対して、警告を先に行う執行を保証するものではありません。レポート、モデレーション保留、
非表示リスティング、禁止、アカウントの状態については、[モデレーションとアカウントの安全性](/ja-JP/clawhub/moderation) を参照してください。
