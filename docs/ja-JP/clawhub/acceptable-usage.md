# 利用許容範囲

このページでは、ClawHub が許容する Skills とコンテンツの種類、およびホストしない不正利用ワークフローについて説明します。

これらのルールは意図的に実用性を重視しています。私たちが最も重視するのは、孤立したキーワードだけではなく、エンドツーエンドの不正利用ワークフローです。防御の回避、プラットフォームの不正利用、人をだます行為、プライバシー侵害、または同意のない行動を可能にすることを目的として構築されたスキルは、ClawHub には適しません。

## 明示的に許容する最近のパターン

- 実際のコンポーネント、セマンティックトークン、アクセシブルな状態、テスト済みのユーザーフローを使用するフロントエンドおよびデザインシステム作業。
- 一回限りのマークアップではなく、インストール済みのソースコンポーネント、プロジェクトエイリアス、文書化されたバリアントを使用する shadcn/ui の構成。
- コメントを保持し、具体的な UI5 型を使用し、生成されたコントロールインターフェイスをレビュー可能な状態に保つ UI5 JavaScript から TypeScript への変換。
- 証拠を示し、人間による承認境界を明確に保つ、防御的セキュリティレビュー、モデレーションツール、不正利用検出プロンプト。
- 明示的な認証情報、透明性のあるセットアップ、ドライランまたはプレビューモードを備えた、個人またはチームアカウント向けの同意に基づくワークフロー自動化。
- サポート対象ソフトウェアに範囲を限定した、ドキュメント、移行ランブック、開発者ユーティリティ、テストフィクスチャ。

## 許容しないもの

- セキュリティ回避または不正アクセスのワークフロー。
  - 例: 認証回避、アカウント乗っ取り、CAPTCHA 回避、Cloudflare またはボット対策の回避、レート制限回避、保護を破ることを目的としたステルススクレイピング、ライブ通話またはエージェントの乗っ取り、再利用可能なセッション窃取、未承認ユーザー向けペアリングフローの自動承認。

- プラットフォームの不正利用および禁止回避。
  - 例: 禁止後のステルスアカウント、アカウントのウォーミング/ファーミング、偽のエンゲージメント、カルマまたはフォロワーの育成、複数アカウントの自動化、大量投稿、スパムボット、検出を避けるために構築されたマーケットプレイスまたはソーシャル自動化。

- 不正、詐欺、欺瞞的な金融ワークフロー。
  - 例: 偽の証明書、偽の請求書、欺瞞的な支払いフロー、詐欺的なアウトリーチ、偽のソーシャルプルーフ、明確な人間の承認と透明な制御なしに支出または請求を可能にするツール、または不正目的でアカウントを作成するために構築された合成アイデンティティのワークフロー。

- プライバシーを侵害するスクレイピング、エンリッチメント、または監視。
  - 例: スパム目的で連絡先詳細を大規模にスクレイピングすること、晒し、ストーキング、未承諾アウトリーチと組み合わせたリード抽出、 covert な監視、明確な同意なしに使用される顔検索または生体認証マッチング、または漏えいデータや侵害ダンプの購入、公開、ダウンロード、運用化。

- 同意のないなりすましまたは欺瞞的なアイデンティティ操作。
  - 例: フェイススワップ、デジタルツイン、偽のペルソナ、クローン化されたインフルエンサー、またはなりすましや誤認誘導に使用されるその他のアイデンティティ操作ツール。

- 露骨な性的コンテンツおよび安全機能を無効化した成人向け生成。
  - 例: NSFW 画像/動画/コンテンツ生成、サードパーティ API をラップした成人向けコンテンツ、または露骨な性的コンテンツを主目的とするスキル。

- 隠された、安全でない、または誤解を招く実行要件。
  - 例: 難読化されたインストールコマンド、`curl | sh`、未申告のシークレット要件、未申告の秘密鍵使用、明確なレビュー可能性のないリモート `npx @latest` 実行、スキルの実行に本当に必要なものを隠す誤解を招くメタデータ。

## 明示的に許容しない最近のパターン

- 「マーケットプレイスで禁止された後にステルス出品者アカウントを作成する。」
- 「未承認ユーザーがペアリングコードを自動的に受け取るように Telegram のペアリングを変更する。」
- 「検出不能な自動化で Reddit/Twitter アカウントを育成する。」
- 「任意の用途向けに専門的な証明書や請求書を生成する。」
- 「安全チェックを無効化して NSFW コンテンツを生成する。」
- 「リードをスクレイピングし、連絡先をエンリッチし、大規模なコールドアウトリーチを開始する。」
- 「漏えいデータや侵害ダンプを購入、公開、またはダウンロードする。」
- 「合成アイデンティティまたは CAPTCHA 解決を使ってメールアカウントやソーシャルアカウントを一括作成する。」

## レビュアー向けの注意

- 文脈が重要です。同じトピックでも、狭い防御的または同意に基づく設定では正当であり得る一方、不正利用ワークフローとしてパッケージ化されている場合は許容できません。
- スキルが回避、欺瞞、または同意のない使用に明確に最適化されている場合は、対応する方向に判断すべきです。
- これらのカテゴリで繰り返しアップロードすることは、コンテンツの非表示およびアカウント禁止の根拠になります。

## 執行

- 違反するスキルを非表示、削除、または完全削除する場合があります。
- トークンを取り消し、関連コンテンツをソフト削除し、繰り返し違反する者または重大な違反者を禁止する場合があります。
- 明白な不正利用に対して、警告を先に行う執行を保証しません。
