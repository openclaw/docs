# ドキュメントガイド

このディレクトリは、ドキュメントの作成、Mintlify のリンク規則、およびドキュメント i18n ポリシーを管理します。

## Mintlify のルール

- ドキュメントは Mintlify（`https://docs.openclaw.ai`）でホストされています。
- `docs/**/*.md` 内の内部ドキュメントリンクは、`.md` または `.mdx` の接尾辞なしのルート相対を維持する必要があります（例: `[Config](/gateway/configuration)`）。
- セクションの相互参照では、ルート相対パス上のアンカーを使用する必要があります（例: `[Hooks](/gateway/configuration-reference#hooks)`）。
- ドキュメント見出しでは、Mintlify のアンカー生成がそれらに対して脆弱なため、em ダッシュやアポストロフィを避けてください。
- README およびその他の GitHub で表示されるドキュメントでは、GitHub 外でもリンクが機能するように、絶対ドキュメント URL を維持してください。
- ドキュメントの内容は汎用的に保つ必要があります。個人のデバイス名、ホスト名、またはローカルパスは使用せず、`user@gateway-host` のようなプレースホルダーを使用してください。

## ドキュメント内容のルール

- ドキュメント、UI コピー、およびピッカーリストでは、そのセクションが実行順序または自動検出順序を明示的に説明している場合を除き、サービス/プロバイダーをアルファベット順に並べてください。
- バンドルされた plugin の命名は、ルート `AGENTS.md` にあるリポジトリ全体の plugin 用語ルールと一貫させてください。

## ドキュメント i18n

- 外国語ドキュメントはこのリポジトリでは保守されません。生成された公開出力は、別の `openclaw/docs` リポジトリ（多くの場合、ローカルでは `../openclaw-docs` としてクローンされています）にあります。
- ここでは `docs/<locale>/**` 配下のローカライズされたドキュメントを追加または編集しないでください。
- このリポジトリ内の英語ドキュメントと用語集ファイルを信頼できる情報源として扱ってください。
- パイプライン: ここで英語ドキュメントを更新し、必要に応じて `docs/.i18n/glossary.<locale>.json` を更新し、その後、公開リポジトリの同期と `openclaw/docs` 内での `scripts/docs-i18n` 実行に任せてください。
- `scripts/docs-i18n` を再実行する前に、英語のまま維持する必要がある、または固定訳を使う必要がある新しい技術用語、ページタイトル、短いナビゲーションラベルについて、用語集エントリを追加してください。
- `pnpm docs:check-i18n-glossary` は、変更された英語ドキュメントタイトルと短い内部ドキュメントラベルに対するガードです。
- 翻訳メモリは、公開リポジトリ内の生成された `docs/.i18n/*.tm.jsonl` ファイルにあります。
- `docs/.i18n/README.md` を参照してください。
