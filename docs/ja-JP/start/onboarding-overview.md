OpenClaw には、ターミナルと macOS アプリのオンボーディングがあります。どちらも最初に推論を確立します。
既存の AI アクセスを検出し、実際の補完を必須としたうえで、その後にのみ
OpenClaw を起動して残りのセットアップを構成します。到達可能で構成済みの Gateway で、
デフォルトエージェントに構成済みのモデルがすでにある場合は、オンボーディングをスキップして
通常のエージェント UI を開きます。ターミナルフローでは、詳細なセットアップ用に
完全な従来のウィザードも利用できます。

## どの方法を使用すればよいですか？

|                | CLI オンボーディング                         | macOS アプリのオンボーディング           |
| -------------- | -------------------------------------- | ------------------------------ |
| **プラットフォーム**  | macOS、Linux、Windows（ネイティブまたは WSL2） | macOS のみ                     |
| **インターフェース**  | 推論のセットアップ後、OpenClaw         | 推論のセットアップ後、OpenClaw |
| **最適な用途**   | サーバー、ヘッドレス環境、完全な制御        | デスクトップ Mac、視覚的なセットアップ      |
| **自動化** | スクリプト向けの `--non-interactive`        | 手動のみ                    |
| **コマンド**    | `openclaw onboard`                     | アプリを起動                 |

ほとんどのユーザーは **CLI オンボーディング** から始めることを推奨します。どの環境でも動作し、
最も詳細に制御できます。

## オンボーディングで構成される内容

ガイド付き推論フェーズで確立されるのは、次の項目のみです。

1. **モデルプロバイダーと認証** — 検出されたアクセス、または検証済みのプロバイダーへのサインイン、
   API キー、トークン
2. **検証済みの推論** — デフォルトエージェントの実効モデルで実行される実際の補完

この補完が成功すると、OpenClaw でワークスペース、Gateway、
Gateway サービス、チャンネル、エージェント、Plugin、その他のオプション機能を構成できます。

従来の CLI ウィザードでは、さらに次の項目を構成できます。

1. **チャンネル**（オプション）— Discord、Feishu、Google Chat、iMessage、Mattermost、Microsoft Teams、
   Telegram、WhatsApp などの組み込みおよび同梱チャットチャンネル
2. **Gateway の高度な制御** — リモートモード、ネットワーク設定、デーモンの選択

## CLI オンボーディング

任意のターミナルで実行します。

```bash
openclaw onboard
```

ガイド付きフローでは、既存の AI アクセスを検出し、候補を順番にライブテストして、
失敗した場合は次の候補に進みます。検出候補をすべて試しても見つからない場合は、まず OpenAI、
Anthropic、xAI（Grok）、Google、OpenRouter が表示されます。**その他…** には、
残りのプロバイダーがプロバイダーグループ別に表示され、2 番目のメニューには地域、プラン、および対応する
ブラウザ、デバイス、API キー、トークンの各方式が表示されます。補完に成功した後にのみモデルと
認証情報を保存し、その後 OpenClaw を起動して、
ワークスペース、Gateway、チャンネル、エージェント、Plugin、その他のオプション機能を構成します。
**今はスキップ** を選ぶと、OpenClaw を起動せずに終了します。
フロー内で従来のウィザードへ移行することはできません。代わりに従来のウィザードを使用する場合は、終了して
`openclaw onboard --classic` を実行してください。

推論が成功すると、OpenClaw はチャンネルのセットアップを入力内容がマスクされるターミナル
ウィザードに引き継ぐことができます。ガイド付きまたは従来のプロバイダーセットアップは開きません。モデルプロバイダーや
その認証を変更するには、OpenClaw を終了して `openclaw onboard` を実行してください。

モデル／認証、チャンネル、Skills、
リモート Gateway、インポートを詳細にセットアップするには `openclaw onboard --classic` を使用します。`--install-daemon` を追加すると、
従来のフローも選択され、バックグラウンドサービスが 1 回の手順でインストールされます。推論以外のセットアップや修復を対話形式で行うには `openclaw
openclaw` を使用します。`openclaw
onboard --modern` は、同じライブ推論
ゲートを使用する互換性エイリアスです。

完全なリファレンス：[オンボーディング（CLI）](/ja-JP/start/wizard)
CLI コマンドのドキュメント：[`openclaw onboard`](/ja-JP/cli/onboard)

## macOS アプリのオンボーディング

OpenClaw アプリを開きます。構成されたローカルまたはリモートの Gateway に到達でき、
デフォルトエージェントに構成済みのモデルがすでにある場合、アプリはオンボーディングと
OpenClaw をスキップし、通常のエージェント UI をすぐに開きます。

新規または構成が不完全な Gateway では、初回実行フローが既存の AI
アクセス（Claude Code、Codex、または API キー）を検出し、最適な
選択肢をライブテストして、実際の応答を得た後にのみ保存します。見つからない場合は自動的にフォールバックし、
検証済みの手動 API キー入力手順を提供します。機密性の高い
認証情報はマスク入力を使用します。推論が成功すると、OpenClaw が起動し、
残りの構成を支援します。

Gemini CLI はセットアップ後も通常のエージェントで使用できますが、
ツールを使用しないプローブを強制できないため、この推論ゲートでは提供されません。

完全なリファレンス：[オンボーディング（macOS アプリ）](/ja-JP/start/onboarding)

## カスタムまたは一覧にないプロバイダー

プロバイダーが一覧にない場合は、`openclaw onboard --classic` を実行して
**カスタムプロバイダー** を選択し、次を入力します。

- エンドポイントの互換性：OpenAI 互換（`/chat/completions`）、OpenAI Responses 互換（`/responses`）、Anthropic 互換（`/messages`）、または不明（3 種類すべてをプローブして自動検出）
- ベース URL と API キー（エンドポイントで不要な場合、API キーは省略可能）
- モデル ID とオプションのモデルエイリアス

複数のカスタムエンドポイントを共存させることができ、それぞれに固有のエンドポイント ID が割り当てられます。
