[Control UI](/ja-JP/web/control-ui) の **ワークスペース** タブは、あなたとエージェントが共同で構成する画面です。タブ、ウィジェット、12 列グリッド上の位置、データバインディングは、すべて 1 つのドキュメントに保存されます。そのドキュメントを編集できるものなら、あなた、`openclaw workspaces` CLI、または `workspace_*` ツールを呼び出すエージェントのいずれでもワークスペースを構成できます。

すべての書き込みは同じ検証済みの経路を通るため、人間のレイアウトとエージェントのレイアウトが食い違うことはありません。受理された書き込みごとにバージョンが更新され、`plugin.workspaces.changed` がブロードキャストされるため、エージェントによる編集は、すでに開いているブラウザーにも再読み込みなしで反映されます。

## ワークスペースを有効にする

同梱の Workspaces Plugin はデフォルトで無効です。Control UI で **プラグイン** を開き、**ワークスペース** を見つけて **有効化** を選択します。CLI から有効にすることもできます。

```sh
openclaw plugins enable workspaces
```

Plugin を有効にすると **ワークスペース** タブが追加され、`openclaw workspaces` CLI と `workspace_*` エージェントツールが利用可能になります。無効にすると、ワークスペースのデータベースやウィジェットアセットを削除せずに、それらの機能が取り除かれます。

## デフォルトのワークスペース

初回読み込み時には **概要** ワークスペースが表示されます。これには、コストとトークンのカード、インスタンスの正常性、セッション、Cron のステータス、アクティビティフィードが含まれます。これは通常のワークスペースコンテンツなので、ドラッグ、折りたたみ、非表示、削除が可能です。

## 組み込みウィジェット

Plugin には 9 個の信頼済みウィジェットが同梱され、ファーストパーティ UI としてレンダリングされます。

`stat-card`、`markdown`、`table`、`iframe-embed`、`sessions`、`usage`、`cron`、
`instances`、`activity`。

ウィジェットは **バインディング** を通じてデータを宣言し、自ら取得することはありません。

| バインディング | 解決される値                                                                                                      |
| -------------- | ----------------------------------------------------------------------------------------------------------------- |
| `static`       | ドキュメント内に保存されたリテラル値（最大 8 KB）。                                                               |
| `file`         | `<stateDir>/workspaces/data/` 配下の JSON、Markdown、または CSV ファイル。必要に応じて JSON ポインターで範囲を絞り込みます。 |
| `rpc`          | 固定の許可リストに含まれる読み取り専用 Gateway メソッドの 1 つ。信頼済みの Control UI によって解決されます。       |

`file` バインディングは、独自の数値をワークスペースに表示する最も簡単な方法です。JSON ファイルをデータディレクトリに書き込み、`stat-card` からそのファイルを参照します。

## 来歴

タブとウィジェットには、書き込みを行った主体に基づいて設定される `createdBy` スタンプ（`user`、`system`、または `agent:<id>`）が付与されます。この値は呼び出し元から指定できないため、エージェントが自身の作業をあなたの作業として表示することはできません。また、エージェントが作成したウィジェットの「AI」チップは、常にその表示どおりの意味を持ちます。

## カスタムウィジェット

エージェントは `workspace_widget_scaffold` を使用して実際の HTML ウィジェットを作成できます（`openclaw workspaces widget-scaffold <name>` を使用すれば、あなたも作成できます）。エージェントが作成したコードは信頼できないものとして扱われます。

- スキャフォールドされたウィジェットは、**保留中** としてレジストリに登録されます。オペレーターが承認するまで iframe は作成されず、アセットルートはそのファイルに対して 404 を返します。
- 承認はレイアウトの編集とは別の判断です。`workspaces.widget.approve` には、実行承認を保護するものと同じ `operator.approvals` スコープが必要です。
- 承認済みのウィジェットは `<iframe sandbox="allow-scripts">` 内でレンダリングされます。`allow-same-origin` は決して使用されません。そのため、オリジンは不透明になり、親の DOM、ストレージ、Cookie にはアクセスできません。
- アセットは `connect-src 'none'` を指定して配信されるため、`fetch`、XHR、WebSocket などのスクリプトによるネットワーク通信がブロックされます。ウィジェットは認証情報を保持せず、Gateway と通信することもありません。
- データは、バージョン管理された `postMessage` ブリッジのみを通じてウィジェットに渡されます。カスタムコードは、宣言済みの `static` バインディングを受信できます。これらは、エージェントまたはオペレーターがすでに作成したワークスペース値です。RPC バインディングとファイルバインディングは、信頼済みの組み込みウィジェット内に留まります。ブラウザーでは、サンドボックス化された子フレームが自身のフレームを移動できるため、特権データがエージェント作成の HTML に送信されることはありません。

ウィジェットからチャットへプロンプトを送信するには、さらにマニフェストのケイパビリティ、正確なテキストを引用する呼び出しごとの確認が必要であり、レート制限も適用されます。

## CLI

```sh
openclaw workspaces tabs list
openclaw workspaces tabs create --title Financials
openclaw workspaces widget-scaffold revenue-chart --title "Revenue Chart"
openclaw workspaces widget-approve revenue-chart
```

`widget-approve` には、`operator.approvals` スコープでペアリングされたデバイスが必要です。Control UI から承認する場合は、ブラウザーがすでにこのスコープを保持しているため不要です。

## ストレージ

ワークスペースドキュメント、カスタムウィジェットレジストリ、および 20 エントリの元に戻すリングは、`<stateDir>/workspaces/workspaces.sqlite` に保存されます。エージェントが作成したウィジェットアセットは `<stateDir>/workspaces/widgets/<name>/` 配下に、ファイルバインディングのデータは `<stateDir>/workspaces/data/` 配下に、それぞれディスク上で保持されます。これは、エージェントが通常のファイルツールを使用してそれらを作成し、ウィジェットルートがそのバイト列を配信するためです。
