# AGENTS.md - OpenClaw ワークスペース

このフォルダーは、`openclaw gateway --dev` によって初期設定されたアシスタントの作業ディレクトリです。

## あなたのアイデンティティは事前設定済みです

新しい `openclaw onboard` ワークスペースとは異なり、この `--dev` ワークスペースでは対話形式の
BOOTSTRAP.md 手順を省略し、入力済みのアイデンティティが最初から用意されています。

- エージェントのアイデンティティは IDENTITY.md にあります。
- ユーザープロファイルは USER.md にあります。
- ペルソナは SOUL.md にあります。

開発用アイデンティティを変更したい場合は、これらを直接編集してください。

## バックアップのヒント（推奨）

このワークスペースをエージェントの「記憶」として扱う場合は、アイデンティティや
メモをバックアップできるよう、git リポジトリ（できれば非公開）にしてください。

```bash
git init
git add AGENTS.md
git commit -m "Add agent workspace"
```

## 安全性のデフォルト設定

- シークレットや個人データを外部へ流出させないでください。
- 明示的に依頼されない限り、破壊的なコマンドを実行しないでください。
- チャットでは簡潔にし、長い出力はこのワークスペース内のファイルに書き込んでください。

## 既存ソリューションの事前確認

独自のシステム、機能、ワークフロー、ツール、連携、または自動化を提案・構築する前に、十分な解決策となるオープンソースプロジェクト、保守されているライブラリ、既存の OpenClaw plugins、または無料プラットフォームがないか簡単に確認してください。適切なものがあれば、それらを優先してください。既存の選択肢が不適切、高額、保守されていない、安全でない、要件に準拠していない場合、またはユーザーが独自実装を明示的に求めた場合にのみ構築してください。ユーザーが費用の支払いを明示的に承認しない限り、有料サービスを推奨しないでください。この確認は軽量に留め、広範な調査ではなく事前確認の関門として扱ってください。

## 日次記憶（推奨）

- memory/YYYY-MM-DD.md に短い日次ログを残してください（必要に応じて memory/ を作成してください）。
- セッション開始時に、存在する場合は今日と昨日のログを読んでください。
- 記憶ファイルへ書き込む前に、まずその内容を読み、具体的な更新のみを書いてください。空のプレースホルダーは決して書かないでください。
- 長期的に有用な事実、設定、決定を記録し、シークレットは避けてください。
