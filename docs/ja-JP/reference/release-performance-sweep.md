このページでは、2026年5月のOpenClawにおけるパフォーマンス、
パッケージサイズ、依存関係、shrinkwrap整理の根拠をまとめています。これは公開ブログ記事の
技術的な補足資料です。

ここでは、次の2つの監査を組み合わせています。

- **リリースパフォーマンス調査:** `v2026.5.28`から安定版`v2026.4.23`までの
  GitHub Releasesを対象に、`OpenClaw Performance`ワークフロー、
  `profile=smoke`、モックプロバイダーレーンを使用しました。ほとんどのタグ行は1サンプルです。
  `v2026.5.27`と`v2026.5.28`の行では、最新の3回反復リリースブランチ
  アーティファクトを使用しています。
- **4月前半のコンテキスト:** `v2026.4.1`から`v2026.5.2`までの、公開済み
  `clawgrit-reports`モックプロバイダーベースラインです。4月後半の不具合のあるリリースを
  公開パフォーマンスのベースラインとして扱わないためにのみ使用します。
- **インストールフットプリント調査:** 一時パッケージへ
  `npm install --ignore-scripts`で新規インストールし、サイズには
  `du -sk node_modules`、パッケージインスタンス数には`node_modules`の走査を
  使用しました。
- **npmパッケージサイズ調査:** 公開済みリリースに対して
  `npm pack openclaw@<version> --dry-run --json`を実行し、圧縮済みtarballサイズ、
  展開後サイズ、ファイル数を記録しました。

<Warning>
主要なパフォーマンス調査では、タグごとに1つのスモークサンプルを使用しています。ただし、
`v2026.5.27`と`v2026.5.28`の行では、最新の3回反復
リリースブランチアーティファクトを使用しています。4月前半のコンテキストでは、
`clawgrit-reports`で公開済みの3回反復の中央値を使用しています。これらの数値は
リリースゲートの統計値ではなく、傾向を示す根拠およびリグレッション調査のシグナルとして
扱ってください。
</Warning>

### Shrinkwrap の境界

`2026.5.20` はルート shrinkwrap がなく、大規模なネストされた OpenClaw
依存関係ツリーもない状態でリリースされました。`2026.5.22` でルート shrinkwrap が導入され、
ネストされた `openclaw/node_modules` 配下に 911.8MB がインストールされました。
`2026.5.28` は shrinkwrap を維持し、ネストされた `openclaw/node_modules` 配下に
引き続き 259.7MiB をインストールしますが、ローカルの新規インストール監査では
`@napi-rs/canvas` パッケージがインストールされなくなりました。

公開済み tarball の検査により、この境界が確認されています。

| バージョン  | stable として公開済みか | ルート `npm-shrinkwrap.json` | 注記                                          |
| ----------- | ------------------------ | ---------------------------- | --------------------------------------------- |
| `2026.5.20` | はい                     | なし                         | shrinkwrap 導入前の最後の stable リリース     |
| `2026.5.21` | いいえ                   | 該当なし                     | stable npm リリースなし                       |
| `2026.5.22` | はい                     | あり                         | shrinkwrap を導入                             |
| `2026.5.23` | いいえ                   | 該当なし                     | stable npm リリースなし                       |
| `2026.5.24` | いいえ                   | 該当なし                     | stable npm リリースなし                       |
| `2026.5.25` | いいえ                   | 該当なし                     | stable npm リリースなし                       |
| `2026.5.26` | はい                     | あり                         | ネストされた依存関係ツリーが引き続き存在      |
| `2026.5.27` | はい                     | あり                         | ネストされた依存関係ツリーが引き続き存在      |
| `2026.5.28` | はい                     | あり                         | ネストされた依存関係ツリーが大幅に縮小        |

重要な違いは、**shrinkwrap 自体が問題なのではない**という点です。
`v2026.5.28` でもルート shrinkwrap は引き続き同梱されています。問題だったのは、npm に
大規模なネストされた OpenClaw 依存関係ツリーと、12個すべての
`@napi-rs/canvas` プラットフォームパッケージを実体化させるパッケージ構成です。
`v2026.5.28` ではネストされたツリーが縮小され、Canvas プラットフォームの
大量展開はローカル監査で検出されなくなりました。

shrinkwrap の平易な説明とメンテナーレベルのパッケージ検査については、
[npm shrinkwrap](/ja-JP/gateway/security/shrinkwrap)を参照してください。

## サプライチェーン上の解釈

依存関係数は、インストールサイズの指標であるだけでなく、運用上のセキュリティ指標でもあります。
各パッケージによって、運用者が信頼しなければならないメンテナー、tarball、推移的更新、
オプションのネイティブバイナリ、インストール時の動作の範囲が広がります。

クリーンアップの方針は次のとおりです。

- 重量級およびオプションの機能をデフォルトのコアインストールから分離する
- Plugin パッケージが自身のランタイム依存関係グラフを所有するようにする
- Gateway 起動時のランタイムパッケージマネージャーによる修復を避ける
- 全プラットフォーム向けネイティブパッケージを実体化させずに、決定論的なインストールを維持する
- パッケージ受け入れおよび測定経路ではインストールスクリプトを無効に保つ
- 公開前に、ネストされた依存関係ツリーとネイティブのオプション依存関係の爆発的増加を検出する
