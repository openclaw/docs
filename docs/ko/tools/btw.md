`/btw`(별칭 `/side`)는 대화 기록에 추가하지 않고 **현재
세션**에 관해 간단한 부가 질문을 합니다. Claude Code의 `/btw`를 모델로 하되,
OpenClaw의 Gateway 및 다중 채널 아키텍처에 맞게 조정했습니다.

```text
/btw 무엇이 변경되었나요?
/side 이 오류는 무엇을 의미하나요?
```

## 수행하는 작업

1. 현재 세션을 배경 컨텍스트로 스냅샷합니다(진행 중인 기본 실행 프롬프트
   포함).
2. 모델에 부가 질문에만 답하고 기본 작업을 재개하거나 조정하지 말라고
   지시하여 별도의 일회성 부가 쿼리를 실행합니다.
3. 답변을 일반 어시스턴트 메시지가 아닌 실시간 부가 결과로 전달합니다.
4. 질문이나 답변을 세션 기록 또는 `chat.history`에 절대 기록하지 않습니다.

활성 상태인 기본 실행이 있더라도 그대로 유지됩니다.

Codex 하네스 세션에서 BTW는 별도의 제공자 호출을 실행하는 대신 활성 Codex
앱 서버 스레드를 일시적인 자식 스레드로 포크합니다. 이렇게 하면 Codex OAuth와
네이티브 도구/스레드 동작이 그대로 유지되며, 포크된 스레드는 상위 스레드의
현재 승인 정책, 샌드박스 및 네이티브 도구 표면을 유지합니다. 포크된 스레드에는
그 앞의 모든 내용이 활성 지침이 아닌 상속된 참조 컨텍스트이고 경계 뒤의
메시지만 활성 상태임을 모델에 알리는 경계 프롬프트가 제공됩니다. `/btw`를
사용하려면 기존 Codex 스레드가 필요하므로 먼저 일반 메시지를 보내십시오.

CLI 런타임 별칭의 경우 BTW는 소유 CLI 백엔드를 일회성 부가 질문 모드로
호출합니다. 즉, 정리된 대화 컨텍스트를 새로운 CLI 호출에 시드하고 도구 번들링과
재사용 가능한 세션 상태를 비활성화하며, 백엔드에서 지원하는 재개 금지/도구 금지
플래그를 추가합니다. 직접(비 CLI) 런타임은 대신 직접적인 일회성 제공자 호출을
사용합니다.

## 수행하지 않는 작업

`/btw`는 영구 세션을 생성하거나, 완료되지 않은 기본 작업을 계속하거나,
질문/답변 데이터를 트랜스크립트 기록에 보존하거나, 다시 로드한 후에도 유지되지
않습니다.

## 전달 모델

일반 어시스턴트 채팅은 Gateway `chat` 이벤트를 사용합니다. BTW는 별도의
`chat.side_result` 이벤트를 사용하므로 클라이언트가 이를 일반 대화 기록으로
오인할 수 없습니다. `chat.history`에서 재생되지 않으므로 다시 로드하면
사라집니다.

## 표면별 동작

| 표면              | 동작                                                                                                                                                                                                                                                                                |
| ----------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| TUI               | 채팅 로그에 인라인으로 렌더링되며 일반 답변과 명확히 구별되고 `Enter` 또는 `Esc`로 닫을 수 있습니다.                                                                                                                                                                                |
| 외부 채널         | 명확한 레이블이 있는 일회성 답변으로 전달됩니다(Telegram, WhatsApp, Discord에는 로컬 임시 오버레이가 없습니다).                                                                                                                                                                      |
| Control UI / 웹   | 스레드에 고정된 부동 "부가 채팅" 패널로 렌더링됩니다. 답변은 턴 단위로 누적되며 "후속 질문" 입력란에서 다음 부가 질문을 할 수 있습니다. 닫기(`Esc` 또는 X)를 사용하면 대화가 유지되고 다음 답변이 도착할 때 다시 열립니다. 휴지통 버튼은 대화를 삭제하고 대기 중인 실행을 중지합니다. |

## 선택 팝업(Control UI)

Control UI의 채팅 메시지 안에서 텍스트를 강조 표시하면 다음 두 가지 작업이
있는 작은 선택 팝업이 열립니다.

- **자세한 내용**은 현재 세션의 컨텍스트에서 강조 표시된 텍스트를 설명하도록
  모델에 요청하는 암시적 `/btw` 질문을 즉시 보냅니다. 답변은 부동 부가 채팅
  패널에 표시됩니다.
- **부가 채팅에서 질문**은 강조 표시된 텍스트를 인용하는 `/btw` 초안을 작성기에
  미리 채워 넣으므로 해당 텍스트에 관한 질문을 직접 입력할 수 있습니다.

두 작업 모두 일반 `/btw` 의미 체계를 따릅니다. 질문과 답변은 세션 기록에
포함되지 않으며 기본 실행은 그대로 유지됩니다.

## 사용 시점

간단한 설명이 필요할 때, 긴 실행이 아직 진행 중인 동안 사실에 관한 부가 답변이
필요할 때, 또는 향후 세션 컨텍스트에 포함되지 않아야 하는 임시 답변이 필요할 때
`/btw`를 사용하십시오.

```text
/btw 어떤 파일을 편집하고 있나요?
/btw 현재 작업을 한 문장으로 요약해 주세요
/btw 17 * 19는 얼마인가요?
```

향후 세션의 작업 컨텍스트에 포함하려는 내용은 대신 기본 세션에서 일반적인
방식으로 질문하십시오.
