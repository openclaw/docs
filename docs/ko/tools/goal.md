# 목표

**목표**는 현재 OpenClaw 세션에 연결된 하나의 지속적인 목적입니다.
에이전트와 운영자가 장기 실행 작업에 대해 공동의 지향점을 가질 수 있게 하면서도,
그 지향점을 백그라운드 작업, 알림, Cron 작업 또는
상시 명령으로 전환하지 않습니다.

목표는 세션 상태입니다. 세션 키와 함께 이동하고, 프로세스가
재시작되어도 유지되며, `/goal`, 모델에 제공되는 목표 도구 및 TUI
바닥글에 표시됩니다.

## 빠른 시작

```text
/goal start PR 87469의 CI를 통과시키고 수정 사항을 푸시하기
/goal
/goal edit PR 87469의 CI를 통과시키고, 수정 사항을 푸시하고, 문서를 업데이트하기
/goal pause CI 대기 중
/goal resume
/goal complete 푸시 및 검증 완료
/goal clear
```

`start`는 선택 사항입니다. `/goal get CI green for PR 87469`도 목표를 생성합니다.
이는 `/goal` 뒤에 오는 텍스트가 알려진 작업 단어가 아닌 경우
새로운 목적으로 처리되기 때문입니다.

## 목표의 용도

여러 차례의 대화에 걸쳐 계속 표시되어야 하는 구체적인 결과가 세션에 있을 때 목표를 사용하십시오.

- PR 마무리: 수정하고, 검증하고, 자동 리뷰를 수행하고, 푸시한 후 PR을 열거나 업데이트합니다.
- 디버그 실행: 버그를 재현하고, 담당 영역을 식별하고, 패치한 후 수정 사항을 입증합니다.
- 문서 작업: 관련 문서를 읽고, 새 페이지를 작성하고, 상호 링크를 추가한 후 문서 빌드를 검증합니다.
- 유지관리 작업: 현재 상태를 점검하고, 범위가 제한된 변경을 수행하고, 적절한 검사를 실행한 후 변경 사항을 보고합니다.

목표는 작업 대기열이 아닙니다. 작업을 분리된 상태로 실행하거나, 일정에 따라 반복하거나, 관리되는 하위 작업으로 분산하거나, 정책으로 유지해야 할 때는 [Task Flow](/ko/automation/taskflow),
[작업](/ko/automation/tasks), [Cron 작업](/ko/automation/cron-jobs) 또는
[상시 지시](/ko/automation/standing-orders)를 사용하십시오.

## 명령어 참조

인수 없이 `/goal`을 실행하면 현재 목표 요약이 출력됩니다.

```text
목표
상태: active
목적: PR 87469의 CI를 통과시키고 수정 사항을 푸시합니다
사용한 토큰: 12k
토큰 예산: 12k/50k

명령어: /goal edit <objective>, /goal pause, /goal complete, /goal clear
```

| 명령어                                              | 효과                                                                     |
| --------------------------------------------------- | ------------------------------------------------------------------------ |
| `/goal` 또는 `/goal status`                         | 현재 목표를 표시합니다.                                                  |
| `/goal start <objective>`                           | 현재 세션에 새 목표를 생성합니다.                                        |
| `/goal set <objective>`, `/goal create <objective>` | `start`의 별칭입니다.                                                     |
| `/goal <objective>`                                 | 새 목표를 생성하기도 합니다(인식되는 작업 단어가 아닌 모든 텍스트).       |
| `/goal edit <objective>`                            | 현재 목적을 다시 작성합니다. 상태와 토큰 집계는 그대로 유지됩니다.        |
| `/goal pause [note]`                                | 활성 목표를 일시 중지합니다.                                             |
| `/goal resume [note]`                               | 일시 중지되거나, 차단되거나, 사용량 또는 예산이 제한된 목표를 재개합니다. |
| `/goal complete [note]`                             | 목표를 달성한 것으로 표시합니다.                                         |
| `/goal done [note]`                                 | `complete`의 별칭입니다.                                                  |
| `/goal block [note]`                                | 목표를 차단된 것으로 표시합니다.                                         |
| `/goal blocked [note]`                              | `block`의 별칭입니다.                                                     |
| `/goal clear`                                       | 세션에서 목표를 제거합니다.                                              |

한 세션에는 한 번에 하나의 목표만 존재할 수 있습니다. 현재 목표를 지우기 전에 두 번째 목표를 시작하면
`Goal error: goal already exists` 오류가 발생합니다.

`/goal start`에는 토큰 예산 플래그를 사용할 수 없습니다. 예산은 모델용 `create_goal`
도구를 통해서만 설정할 수 있습니다.

## 상태

- `active`: 세션이 목표를 추진하고 있습니다.
- `paused`: 운영자가 목표를 일시 중지했습니다. `/goal resume`을 사용하면 목표가 다시
  활성화됩니다.
- `blocked`: 에이전트 또는 운영자가 실제 차단 요인을 보고했습니다. 새로운 정보나 상태를
  사용할 수 있게 되면 `/goal resume`을 사용하여 다시 활성화합니다.
- `budget_limited`: 구성된 토큰 예산에 도달했습니다. `/goal resume`은 동일한 목적을
  새 예산 구간으로 다시 추진합니다.
- `usage_limited`: 향후 사용량 제한으로 인한 중지 상태를 위해 예약되어 있습니다. `/goal
resume`은 동일한 방식으로 목표 추진을 다시 시작합니다.
- `complete`: 목표를 달성했습니다. 완료된 목표는 종료 상태입니다. 다른 목표를 시작하기 전에 `/goal
clear`를 사용하십시오.

`/new`와 `/reset`은 의도적으로 새로운 세션 컨텍스트를 시작하므로 현재 세션 목표를
지웁니다.

## 토큰 예산

목표에는 선택적으로 양의 토큰 예산을 설정할 수 있으며, `create_goal` 도구의
`token_budget` 매개변수를 통해 설정합니다. 예산은 목표 생성 시점의 세션 최신 토큰
수부터 측정됩니다. 목표가 시작될 때 세션에 오래되었거나 알 수 없는 토큰 스냅샷만
있는 경우, OpenClaw는 다음 최신 스냅샷을 기다린 후 이를 기준값으로 사용하므로 목표가
생성되기 전에 사용된 토큰은 예산에 포함되지 않습니다.

사용량이 예산에 도달하면 목표는 `budget_limited` 상태로 전환됩니다. 이는 목표를
삭제하거나 목적을 지우지 않습니다. 목표가 재개되거나 지워질 때까지 더 이상 적극적으로
추진되지 않음을 운영자와 에이전트에게 알립니다. 재개하면 현재 최신 토큰 수를 기준으로
새 예산 구간을 시작합니다.

토큰 예산은 세션 목표의 가드레일이며, 결제 한도가 아닙니다. 제공자 할당량, 비용 보고,
컨텍스트 창 동작에는 계속해서 일반적인 OpenClaw 사용량 및 모델 제어가 적용됩니다.

## 모델 도구

OpenClaw는 에이전트 하네스에 세 가지 목표 도구를 제공합니다:

| 도구          | 목적                                                                                                                  |
| ------------- | ------------------------------------------------------------------------------------------------------------------------ |
| `get_goal`    | 현재 세션 목표의 상태, 목적, 토큰 사용량 및 토큰 예산을 확인합니다.                                         |
| `create_goal` | 사용자 또는 시스템 지침에서 명시적으로 요청한 경우에만 목표를 생성합니다. 세션에 이미 목표가 있으면 실패합니다. |
| `update_goal` | 목표를 `complete` 또는 `blocked`로 표시합니다.                                                                                   |

모델은 목표를 암묵적으로 일시 중지하거나, 재개하거나, 지우거나, 교체할 수 없습니다. 이러한 작업은
`/goal` 및 재설정 명령을 통해 운영자/세션 제어로 유지되므로, 에이전트가
목표를 조용히 변경하지 않고도 달성 여부 또는 실제 차단 요인을
보고할 수 있습니다.

`update_goal`은 목적을 실제로 달성한 경우에만 목표를 `complete`로
표시해야 합니다. 일반적인 어려움이나 완성도 부족 때문이 아니라, 동일한
차단 조건이 연속된 목표 턴에서 최소 세 번 반복된 후에만 목표를 `blocked`로
표시해야 합니다.

## 모든 턴의 목표 컨텍스트

활성 목표가 있는 모든 사용자/채팅 턴에는 다음 사용자 역할 컨텍스트 줄이 포함됩니다.

```text
활성 목표: <objective> — 목표를 진전시키거나 상태를 업데이트하십시오(get_goal/update_goal).
```

OpenClaw은 긴 목표를 잘라내어 줄을 간결하게 유지합니다. 일시 중지되었거나,
차단되었거나, 예산 제한 또는 사용량 제한에 도달했거나, 완료된 목표는 주입되지 않으므로,
운영자가 중지한 상태는 목표가 재개될 때까지 유지됩니다.

## 제어 UI

웹 제어 UI는 채팅 작성기 위에 목표를 간결한 필 형태로 표시합니다.
상태 아이콘, 상태 레이블(예: `Pursuing goal`), 줄임 처리된
목표, 실시간 경과 시간 타이머로 구성됩니다.

필에는 인라인 컨트롤이 포함됩니다.

- **연필**은 작성기에 `/goal edit <objective>`를 미리 입력하여 목표를
  다시 표현한 후 제출할 수 있게 합니다.
- **일시 중지 / 재개**는 현재 상태에 따라 `/goal pause`와 `/goal resume`
  사이를 전환합니다.
- **휴지통**은 `/goal clear`를 전송합니다.
- **갈매기형 화살표**는 필을 펼쳐 전체 목표, 최신 상태 메모, 토큰 사용량,
  경과 시간을 표시합니다.

작성기에서 전송할 수 없는 동안(예: Gateway 연결이 끊어진 경우)에는
작업 버튼이 숨겨지지만, 펼치기 갈매기형 화살표는 계속 작동합니다.

## TUI

TUI 바닥글은 토큰/모드 표시기 앞에서 활성 세션의 목표를 에이전트,
세션 및 모델 필드 옆에 계속 표시합니다.

바닥글 예시:

- 토큰 예산이 있는 활성 목표는 `Pursuing goal (12k/50k)`로 표시됩니다.
- 일시 중지된 목표는 `Goal paused (/goal resume)`로 표시됩니다.
- 차단된 목표는 `Goal blocked (/goal resume)`로 표시됩니다.
- 사용량 제한에 도달한 목표는 `Goal hit usage limits (/goal resume)`로 표시됩니다.
- 예산 제한으로 달성하지 못한 목표는 `Goal unmet (50k/50k)`로 표시됩니다.
- 완료된 목표는 `Goal achieved (42k)`로 표시됩니다.

바닥글은 의도적으로 간결하게 구성되어 있습니다. 전체 목표,
메모, 토큰 예산 및 사용 가능한 명령을 확인하려면 `/goal`을 사용하십시오.

## 채널 동작

`/goal`은 TUI와 텍스트 명령을 허용하는
채팅 화면을 포함하여 명령을 지원하는 OpenClaw 세션에서 작동합니다. 목표 상태는
전송 수단이 아니라 세션 키에 연결되므로, 동일한 세션 키를 공유하는 두 화면에는
동일한 목표가 표시됩니다.

목표 상태는 전달 지시문이 아닙니다. 채널을 통해 응답을 강제하거나,
대기열 동작을 변경하거나, 도구를 승인하거나, 작업을 예약하지 않습니다.

## 문제 해결

| 메시지                                 | 의미                                                                                                                                                              |
| -------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Goal error: goal already exists`      | 세션에 이미 목표가 있습니다. `/goal`로 확인하고, 완료했다면 `/goal complete`를 사용하거나, 다른 목표를 시작하기 전에 `/goal clear`를 사용하십시오.                 |
| `Goal error: goal not found`           | 세션에 아직 목표가 없습니다. `/goal start <objective>`로 목표를 시작하십시오.                                                                                     |
| `Goal error: goal is already complete` | 목표가 종료 상태입니다. 다른 목표를 시작하거나 재개하기 전에 기존 목표를 지우십시오.                                                                              |

토큰 사용량이 `0`으로 표시되거나 오래된 값처럼 보인다면 활성 세션에
아직 최신 토큰 스냅샷이 없을 수 있습니다. OpenClaw가 세션 사용량과
트랜스크립트에서 산출된 합계를 기록하면 사용량이 갱신됩니다.
