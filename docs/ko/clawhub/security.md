# 보안 + 검토

ClawHub는 게시에 열려 있지만, 공개 목록은 여전히 신뢰,
검사, 신고, 검토 제어를 거칩니다. 목표는 실용적입니다. 사용자가
설치하는 항목을 검토할 수 있도록 돕고, 게시자에게 오탐에 대한 복구 경로를
제공하며, 악성 패키지가 공개 탐색에 노출되지 않도록 하는 것입니다.

[허용되는 사용](/ko/clawhub/acceptable-usage)도 참조하세요.

## 사용자가 검토할 수 있는 항목

skill 또는 plugin을 설치하기 전에 ClawHub 목록에서 다음을 확인하세요.

- 소유자 및 소스 출처
- 최신 버전 및 변경 로그
- 필요한 환경 변수 또는 권한
- plugin의 호환성 메타데이터
- 검사 또는 검토 상태
- 표시되는 경우 신고, 댓글, 별점, 다운로드 및 설치 신호

이해하고 신뢰하는 콘텐츠만 설치하세요.

## 검사 상태

ClawHub는 공개 페이지와 소유자에게 표시되는 진단 정보에 검사 또는 검토 결과를
표시할 수 있습니다.

일반적인 결과는 다음과 같습니다.

- `clean`: 차단할 문제가 발견되지 않았습니다.
- `suspicious`: 릴리스에 주의 또는 검토가 필요합니다.
- `malicious`: 릴리스가 안전하지 않은 것으로 간주됩니다.
- `pending`: 검사가 아직 완료되지 않았습니다.
- `held`, `quarantined`, `revoked` 또는 `hidden`: 릴리스가 공개 설치 표면에서
  완전히 사용할 수 없습니다.

정확한 문구는 표면마다 다를 수 있지만, 실질적인 의미는 같습니다. 릴리스가
보류되거나 차단된 경우, 소유자가 문제를 해결하거나 검토를 통해 복원될 때까지
사용자는 설치하지 않아야 합니다.

## Skills

Skill 검사는 게시된 skill 번들, 메타데이터, 선언된
요구 사항 및 의심스러운 지침을 확인합니다.

ClawHub는 skill이 선언한 내용과 실제로 수행하는 것으로 보이는 내용 사이의 불일치에
특히 주의를 기울입니다. 예를 들어, 필요한 API 키를 참조하는 skill은
사용자가 설치하기 전에 확인할 수 있도록 `SKILL.md`에 해당 요구 사항을
선언해야 합니다.

검사 결과는 아티팩트 기반입니다. 선언된 API 자격 증명, localhost OAuth 콜백,
범위가 지정된 제거 정리, Basic Auth 인코딩 또는 명시된 제공자로 사용자가 선택한
파일 업로드와 같은 예상 제공자 동작은 숨겨진 자격 증명 전달, 광범위한 비공개 파일 접근,
관련 없는 네트워크 대상 또는 은밀한 브라우저 악용과 다르게 취급됩니다.

[Skill 형식](/ko/clawhub/skill-format)을 참조하세요.

## Plugins

Plugin 릴리스에는 패키지 메타데이터, 소스 출처, 호환성
필드 및 아티팩트 무결성 정보가 포함됩니다.

OpenClaw는 ClawHub 호스팅 plugin을 설치하기 전에 호환성을 확인합니다. 패키지
레코드는 OpenClaw가 다운로드된 아티팩트를 검증할 수 있도록 digest 메타데이터도
노출할 수 있습니다. ClawScan은 plugin 릴리스를 검토할 때 선언된 패키지
`openclaw.environment` env/config 메타데이터를 포함하여 선언된 런타임 요구 사항을
관찰된 동작과 비교합니다.

## 신고

로그인한 사용자는 skill, 패키지 및 댓글을 신고할 수 있습니다.

신고는 구체적이고 실행 가능해야 합니다. 신고 기능 남용 자체도 계정 조치로
이어질 수 있습니다.

신고 예시는 다음과 같습니다.

- 오해의 소지가 있는 메타데이터
- 선언되지 않은 자격 증명 또는 권한 요구 사항
- 의심스러운 설치 지침
- 사기성 댓글 또는 사칭
- 악의적인 등록 또는 상표 오용
- [허용되는 사용](/ko/clawhub/acceptable-usage)을 위반하는 콘텐츠

## 게시자 ClawScan 참고 사항

게시자는 skill 또는 plugin을 게시할 때 선택적 ClawScan 참고 사항을 제공할 수
있습니다. 이 참고 사항은 네트워크 접근, native host 접근 또는 제공자별
자격 증명처럼 그렇지 않으면 비정상적으로 보일 수 있는 동작에 대한 맥락을
ClawScan에 제공합니다.

## 검토 보류

정적 스캐너가 업로드된 skill을 악성으로 표시하면, 게시자는 자동으로 검토 보류
상태가 됩니다(사용자에 `requiresModerationAt` 설정). 이는 게시자의 모든 skill을
숨기고, 향후 게시가 숨김 상태로 시작되도록 하며, `user.moderation.auto` 감사 로그
항목을 생성합니다.

정적 의심 결과는 검토자를 위한 파일/줄 증거로 보존되지만, 그 자체만으로 콘텐츠를
숨기거나 공개 검사 판정을 결정하지는 않습니다. 새 업로드는 LLM 검토가 확정될 때까지
검토/대기 상태로 유지됩니다. 정적 검사는 악성 시그니처에 대해서만 즉시 차단합니다.
VirusTotal 엔진 적중은 표시되는 보안 증거로 남지만, VirusTotal Code Insight/Palm
판정은 참고용이며 그 자체만으로 skill을 숨기지는 않습니다. ClawScan LLM 검토는
목적에 부합하는 참고 사항을 지침으로 유지합니다. 중간 수준 검토 결과는 아티팩트에
계속 표시되며, 의심 필터는 영향도가 큰 LLM 우려 사항, 악성 결과 또는 뒷받침된
AV 엔진 탐지에 한해 사용됩니다.

관리자는 오탐 보류를 해제할 수 있습니다.

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

이는 `requiresModerationAt` 및 `requiresModerationReason`을 지우고, 사용자 수준
보류로 숨겨진 skill을 복원하며, `user.moderation.lift` 감사 로그 항목을 작성합니다.
다른 이유로 숨겨진 skill 또는 자체 정적 검사가 여전히 악성인 skill은 숨김 상태로
유지됩니다.

## 차단 및 계정 상태

ClawHub 정책을 위반하는 계정은 게시 접근 권한을 잃을 수 있습니다. 심각한 남용은
계정 차단, 토큰 철회, 콘텐츠 숨김 또는 목록 제거로 이어질 수 있습니다.

삭제, 차단 또는 비활성화된 계정은 ClawHub API 토큰을 사용할 수 없습니다. 계정 조치
이후 CLI 인증이 실패하기 시작하면 웹 UI에 로그인하여 계정 상태를 검토하세요.
로그인 또는 정상적인 CLI 접근이 차단된 경우 복구 검토를 위해
security@openclaw.ai로 문의하세요.

## 게시자 지침

오탐을 줄이고 사용자 신뢰를 높이려면 다음을 따르세요.

- 이름, 요약, 태그 및 변경 로그를 정확하게 유지하세요.
- 필요한 환경 변수와 권한을 선언하세요.
- 릴리스에 비정상적이지만 의도된 동작이 있는 경우 게시자 ClawScan 참고 사항을 추가하세요.
- 난독화된 설치 명령을 피하세요.
- 가능한 경우 소스에 연결하세요.
- plugin을 게시하기 전에 dry run을 사용하세요.
- 사용자 또는 검토자가 패키지 동작에 대해 질문하면 명확하게 응답하세요.
