# 보안 + 모더레이션

ClawHub은 게시에 열려 있지만, 공개 목록은 여전히 신뢰,
스캔, 신고, 모더레이션 제어를 거칩니다. 목표는 실용적입니다. 사용자가
설치할 항목을 검토하도록 돕고, 게시자에게 오탐 복구 경로를 제공하며,
악성 패키지가 공개 탐색 영역에 노출되지 않도록 하는 것입니다.

[허용되는 사용](/ko/clawhub/acceptable-usage)도 참고하세요.

## 사용자가 검토할 수 있는 항목

Skill 또는 Plugin을 설치하기 전에 ClawHub 목록에서 다음을 확인하세요.

- 소유자 및 출처 표시
- 최신 버전 및 변경 로그
- 필요한 환경 변수 또는 권한
- Plugin 호환성 메타데이터
- 스캔 또는 모더레이션 상태
- 표시되는 경우 신고, 댓글, 별표, 다운로드, 설치 신호

이해하고 신뢰하는 콘텐츠만 설치하세요.

## 스캔 상태

ClawHub은 공개 페이지와 소유자에게 보이는 진단 정보에 스캔 또는
모더레이션 결과를 표시할 수 있습니다.

일반적인 결과는 다음과 같습니다.

- `clean`: 차단할 문제가 발견되지 않았습니다.
- `suspicious`: 릴리스에 주의 또는 검토가 필요합니다.
- `malicious`: 릴리스가 안전하지 않은 것으로 간주됩니다.
- `pending`: 검사가 아직 완료되지 않았습니다.
- `held`, `quarantined`, `revoked` 또는 `hidden`: 릴리스가 공개 설치
  표면에서 완전히 사용할 수 없습니다.

정확한 문구는 표면에 따라 다를 수 있지만, 실질적인 의미는 같습니다. 릴리스가
보류되었거나 차단된 경우, 소유자가 문제를 해결하거나 모더레이션이 복원할 때까지
사용자는 설치하지 않아야 합니다.

## Skills

Skill 스캔은 게시된 Skill 번들, 메타데이터, 선언된 요구 사항,
의심스러운 지침을 살펴봅니다.

ClawHub은 Skill이 선언한 내용과 실제로 수행하는 것으로 보이는 내용 사이의
불일치에 특히 주의를 기울입니다. 예를 들어, 필요한 API 키를 참조하는 Skill은
사용자가 설치 전에 이를 볼 수 있도록 `SKILL.md`에 해당 요구 사항을 선언해야
합니다.

스캔 결과는 아티팩트 기반입니다. 선언된 API 자격 증명, localhost OAuth 콜백,
범위가 지정된 제거 정리, Basic Auth 인코딩, 명시된 제공자로의 사용자 선택 파일
업로드와 같은 예상 제공자 동작은 숨겨진 자격 증명 전달, 광범위한 비공개 파일
접근, 관련 없는 네트워크 대상, 은밀한 브라우저 악용과 다르게 취급됩니다.

[Skill 형식](/ko/clawhub/skill-format)을 참고하세요.

## Plugin

Plugin 릴리스에는 패키지 메타데이터, 출처 표시, 호환성 필드,
아티팩트 무결성 정보가 포함됩니다.

OpenClaw은 ClawHub에서 호스팅되는 Plugin을 설치하기 전에 호환성을 확인합니다.
패키지 레코드는 OpenClaw이 다운로드된 아티팩트를 검증할 수 있도록 다이제스트
메타데이터를 노출할 수도 있습니다. ClawScan은 Plugin 릴리스를 검토할 때 선언된
패키지 `openclaw.environment` env/config 메타데이터를 포함하여 선언된 런타임
요구 사항을 관찰된 동작과 비교합니다.

## 신고

로그인한 사용자는 Skill, 패키지, 댓글을 신고할 수 있습니다.

신고는 구체적이고 실행 가능해야 합니다. 신고 남용 자체도 계정 조치로 이어질 수
있습니다.

신고 예시:

- 오해의 소지가 있는 메타데이터
- 선언되지 않은 자격 증명 또는 권한 요구 사항
- 의심스러운 설치 지침
- 사기 댓글 또는 사칭
- 악의적 등록 또는 상표 오용
- [허용되는 사용](/ko/clawhub/acceptable-usage)을 위반하는 콘텐츠

## 악의적 또는 상표 관련 신고

ClawHub은 악의적 등록, 사칭, 상표 관련 분쟁에 대해 동일한 신고 및 직원
모더레이션 파이프라인을 사용합니다. 이러한 신고에는 직원이 청구인, 분쟁 대상
목록, 요청된 조치를 식별할 수 있을 만큼 충분한 맥락이 필요합니다.

포함할 내용:

- 표준 ClawHub Skill 또는 패키지 URL과 소유자 핸들
- 문제가 되는 상표, 프로젝트, 회사 또는 제품 이름
- 청구인의 소유권 또는 권한에 대한 공개 증거
- 현재 소유자가 해당 이름으로 게시할 권한이 없는 이유
- 검토 중 숨김, 소유권 이전, 이름 변경 또는 제거와 같은 요청된 조치

공개 신고에 비공개 비밀 정보나 민감한 법률 문서를 넣지 마세요. 민감하지 않은
증거로 GitHub 이슈를 열고, 필요한 경우 Maintainer에게 비공개 인계 경로를
요청하세요.

## 이의 제기 및 재스캔

소유자는 Skill 또는 패키지가 잘못 보류되었거나 플래그되었다고 생각하는 경우
재스캔을 요청할 수 있습니다. 플랫폼 모더레이터와 관리자는 신고 또는 지원 요청을
처리하는 동안 모든 Skill 또는 패키지에 대해 재스캔을 요청할 수 있습니다.

```bash
clawhub skill rescan <slug>
clawhub package rescan <name>
```

모더레이션된 콘텐츠의 경우, 소유자는 소유자에게 보이는 ClawHub 표면에서 이의
제기를 제출할 수 있을 수 있습니다. 이의 제기에는 무엇이 변경되었는지 또는 해당
플래그가 왜 잘못되었는지를 설명해야 합니다.

## 모더레이션 보류

정적 스캐너가 업로드된 Skill을 악성으로 플래그하면 게시자는 자동으로
모더레이션 보류 상태가 됩니다(사용자에 `requiresModerationAt` 설정). 이로 인해
게시자의 모든 Skill이 숨겨지고, 이후 게시가 숨김 상태로 시작되며,
`user.moderation.auto` 감사 로그 항목이 생성됩니다.

정적 의심 결과는 모더레이터를 위한 파일/라인 증거로 유지되지만, 그 자체만으로
콘텐츠를 숨기거나 공개 스캔 판정을 결정하지는 않습니다. 새 업로드는 LLM 검토가
결정될 때까지 검토/대기 상태로 남습니다. 정적 스캔은 악성 시그니처에 대해서만
즉시 차단합니다. VirusTotal 엔진 히트는 계속 보이는 보안 증거로 유지되지만,
VirusTotal Code Insight/Palm 판정은 참고용이며 그 자체만으로 Skill을 숨기지는
않습니다. ClawScan LLM 검토는 목적에 부합하는 메모를 지침으로 유지합니다. 중간
수준 검토 결과는 아티팩트에 계속 표시되며, 의심 필터는 영향이 큰 LLM 우려,
악성 결과 또는 뒷받침되는 AV 엔진 탐지에만 사용됩니다.

관리자는 오탐 보류를 해제할 수 있습니다.

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

이 작업은 `requiresModerationAt` 및 `requiresModerationReason`을 지우고,
사용자 수준 보류로 숨겨진 Skill을 복원하며, `user.moderation.lift` 감사 로그
항목을 작성합니다. 다른 이유로 숨겨진 Skill 또는 자체 정적 스캔이 여전히 악성인
Skill은 숨김 상태로 유지됩니다.

## 금지 및 계정 상태

ClawHub 정책을 위반하는 계정은 게시 접근 권한을 잃을 수 있습니다. 심각한 남용은
계정 금지, 토큰 폐기, 콘텐츠 숨김 또는 목록 제거로 이어질 수 있습니다.

삭제, 금지 또는 비활성화된 계정은 ClawHub API 토큰을 사용할 수 없습니다. 계정
조치 후 CLI 인증이 실패하기 시작하면 웹 UI에 로그인하여 계정 상태를 검토하거나
예상되는 프로젝트 지원 채널을 통해 Maintainer에게 문의하세요.

## 게시자 지침

오탐을 줄이고 사용자 신뢰를 높이려면 다음을 따르세요.

- 이름, 요약, 태그, 변경 로그를 정확하게 유지하세요
- 필요한 환경 변수와 권한을 선언하세요
- 난독화된 설치 명령을 피하세요
- 가능한 경우 소스에 링크하세요
- Plugin을 게시하기 전에 dry run을 사용하세요
- 사용자 또는 모더레이터가 패키지 동작에 대해 질문하면 명확하게 답변하세요
