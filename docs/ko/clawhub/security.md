# 보안 + 모더레이션

ClawHub는 게시에 열려 있지만, 공개 목록은 여전히 신뢰,
스캔, 신고, 모더레이션 제어를 거칩니다. 목표는 실용적입니다. 사용자가
설치하는 항목을 검사할 수 있게 하고, 게시자에게 오탐 복구 경로를 제공하며,
악성 패키지가 공개 검색에 노출되지 않도록 하는 것입니다.

[허용되는 사용](/ko/clawhub/acceptable-usage)도 참조하세요.

## 사용자가 검사할 수 있는 항목

Skills 또는 Plugin을 설치하기 전에 ClawHub 목록에서 다음을 확인하세요.

- 소유자 및 출처 표시
- 최신 버전 및 변경 로그
- 필요한 환경 변수 또는 권한
- Plugin의 호환성 메타데이터
- 스캔 또는 모더레이션 상태
- 표시되는 경우 신고, 댓글, 별표, 다운로드 및 설치 신호

이해하고 신뢰하는 콘텐츠만 설치하세요.

## 스캔 상태

ClawHub는 공개 페이지와 소유자에게 보이는 진단 정보에 스캔 또는 모더레이션
결과를 표시할 수 있습니다.

일반적인 결과는 다음과 같습니다.

- `clean`: 차단할 문제가 발견되지 않았습니다.
- `suspicious`: 릴리스에 주의 또는 검토가 필요합니다.
- `malicious`: 릴리스가 안전하지 않은 것으로 간주됩니다.
- `pending`: 검사가 아직 완료되지 않았습니다.
- `held`, `quarantined`, `revoked` 또는 `hidden`: 릴리스가 공개 설치 표면에서
  완전히 제공되지 않습니다.

정확한 문구는 표면마다 다를 수 있지만, 실질적인 의미는 같습니다. 릴리스가
보류되거나 차단된 경우, 소유자가 문제를 해결하거나 모더레이션이 복원할 때까지
사용자는 설치하지 않아야 합니다.

## Skills

Skills 스캔은 게시된 Skills 번들, 메타데이터, 선언된 요구 사항,
의심스러운 지침을 살펴봅니다.

ClawHub는 Skills가 선언한 내용과 실제로 수행하는 것으로 보이는 내용 사이의
불일치에 특히 주의합니다. 예를 들어, 필수 API 키를 참조하는 Skills는 사용자가
설치 전에 볼 수 있도록 `SKILL.md`에 해당 요구 사항을 선언해야 합니다.

스캔 결과는 아티팩트 기반입니다. 선언된 API 자격 증명, localhost OAuth 콜백,
범위가 지정된 제거 정리, Basic Auth 인코딩, 또는 명시된 제공자로 사용자가
선택한 파일을 업로드하는 것과 같은 예상 제공자 동작은 숨겨진 자격 증명 전달,
광범위한 개인 파일 접근, 관련 없는 네트워크 대상, 또는 은밀한 브라우저 남용과
다르게 취급됩니다.

[Skills 형식](/ko/clawhub/skill-format)을 참조하세요.

## Plugins

Plugin 릴리스에는 패키지 메타데이터, 출처 표시, 호환성 필드,
아티팩트 무결성 정보가 포함됩니다.

OpenClaw는 ClawHub에서 호스팅되는 Plugin을 설치하기 전에 호환성을 확인합니다.
패키지 기록은 OpenClaw가 다운로드된 아티팩트를 검증할 수 있도록 다이제스트
메타데이터도 노출할 수 있습니다. ClawScan은 Plugin 릴리스를 검토할 때 선언된
패키지 `openclaw.environment` env/config 메타데이터를 포함하여 선언된 런타임
요구 사항을 관찰된 동작과 비교합니다.

## 신고

로그인한 사용자는 Skills, 패키지, 댓글을 신고할 수 있습니다.

신고는 구체적이고 실행 가능해야 합니다. 신고 남용은 그 자체로 계정 조치로
이어질 수 있습니다.

신고 예시:

- 오해의 소지가 있는 메타데이터
- 선언되지 않은 자격 증명 또는 권한 요구 사항
- 의심스러운 설치 지침
- 사기 댓글 또는 사칭
- 악의적 등록 또는 상표 오용
- [허용되는 사용](/ko/clawhub/acceptable-usage)을 위반하는 콘텐츠

## 악의적 또는 상표 관련 신고

ClawHub는 악의적 등록, 사칭, 상표 관련 분쟁에 동일한 신고 및 직원 모더레이션
파이프라인을 사용합니다. 이러한 신고에는 직원이 청구인, 분쟁 대상 목록,
요청된 조치를 식별할 수 있을 만큼 충분한 맥락이 필요합니다.

다음을 포함하세요.

- 표준 ClawHub Skills 또는 패키지 URL 및 소유자 핸들
- 문제가 되는 상표, 프로젝트, 회사 또는 제품 이름
- 청구인의 소유권 또는 권한에 대한 공개 증거
- 현재 소유자가 해당 이름으로 게시할 권한이 없는 이유
- 검토 중 숨김, 소유권 이전, 이름 변경, 제거와 같은 요청 조치

비공개 비밀 정보나 민감한 법적 문서를 공개 신고에 넣지 마세요. 민감하지 않은
증거로 GitHub 이슈를 열고, 필요한 경우 유지관리자에게 비공개 인계 경로를
요청하세요.

## 이의 제기 및 재스캔

소유자는 Skills 또는 패키지가 잘못 보류되었거나 플래그가 지정되었다고 생각할
때 재스캔을 요청할 수 있습니다. 플랫폼 모더레이터와 관리자는 신고 또는 지원
요청을 처리하는 동안 모든 Skills 또는 패키지에 대해 재스캔을 요청할 수
있습니다.

```bash
clawhub skill rescan <slug>
clawhub package rescan <name>
```

모더레이션된 콘텐츠의 경우, 소유자는 소유자에게 보이는 ClawHub 표면에서 이의
제기를 제출할 수 있을 수 있습니다. 이의 제기는 무엇이 변경되었는지 또는
플래그가 왜 잘못되었는지를 설명해야 합니다.

## 모더레이션 보류

정적 스캐너가 업로드된 Skills를 악성으로 플래그하면, 게시자는 자동으로
모더레이션 보류 상태가 됩니다(사용자에 `requiresModerationAt` 설정). 이렇게
하면 게시자의 모든 Skills가 숨겨지고, 이후 게시물은 숨김 상태로 시작되며,
`user.moderation.auto` 감사 로그 항목이 생성됩니다.

정적 의심 결과는 모더레이터를 위한 파일/라인 증거로 보존되지만, 그 자체로
콘텐츠를 숨기거나 공개 스캔 판정을 결정하지는 않습니다. 새 업로드는 VirusTotal
및 LLM 검토가 확정될 때까지 검토/대기 상태로 유지됩니다. 정적 스캔은 악성
시그니처에 대해서만 즉시 차단합니다. ClawScan LLM 검토는 목적에 부합하는
메모를 지침으로 유지합니다. 구조화된 검토에 중대한 우려가 포함된 경우에만
Review/suspicious 판정을 반환합니다.

관리자는 오탐 보류를 해제할 수 있습니다.

```bash
npx convex run users:liftModerationHold '{"userId": "<user-id>", "reason": "False positive from security tool scanning"}'
```

이 작업은 `requiresModerationAt` 및 `requiresModerationReason`을 지우고,
사용자 수준 보류로 숨겨진 Skills를 복원하며, `user.moderation.lift` 감사 로그
항목을 작성합니다. 다른 이유로 숨겨졌거나 자체 정적 스캔이 여전히 악성인
Skills는 숨김 상태로 유지됩니다.

## 차단 및 계정 상태

ClawHub 정책을 위반한 계정은 게시 접근 권한을 잃을 수 있습니다. 심각한 남용은
계정 차단, 토큰 폐기, 콘텐츠 숨김 또는 목록 제거로 이어질 수 있습니다.

삭제, 차단 또는 비활성화된 계정은 ClawHub API 토큰을 사용할 수 없습니다. 계정
조치 후 CLI 인증이 실패하기 시작하면 웹 UI에 로그인하여 계정 상태를 검토하거나
예상되는 프로젝트 지원 채널을 통해 유지관리자에게 문의하세요.

## 게시자 지침

오탐을 줄이고 사용자 신뢰를 높이려면 다음을 따르세요.

- 이름, 요약, 태그, 변경 로그를 정확하게 유지
- 필요한 환경 변수 및 권한 선언
- 난독화된 설치 명령 피하기
- 가능하면 소스에 연결
- Plugin을 게시하기 전에 드라이 런 사용
- 사용자 또는 모더레이터가 패키지 동작에 대해 질문하면 명확하게 응답
