# 허용되는 사용

ClawHub는 OpenClaw용 Skills, Plugin, 패키지 및 마켓플레이스 메타데이터를 호스팅합니다.
콘텐츠나 게시 행위가 ClawHub에 적합한지 판단할 때 이 페이지를 사용하세요.

이 규칙은 목록이 수행하는 작업, 사용자에게 실행하도록 요구하는 작업, 목록이 자신을 표현하는 방식, 게시자가 ClawHub의 검색, 설치 및 신뢰 기능을 사용하는 방식에 적용됩니다. 검토 상태와 계정 상태는 [검토 및 계정 안전](/clawhub/moderation)을 참조하세요. 저작권 또는 기타 권리 침해 신고는 [콘텐츠 권리 요청](/clawhub/content-rights)을 참조하세요.

## 허용되는 콘텐츠

ClawHub는 유용하고 이해하기 쉬우며 선의로 게시된 콘텐츠를 환영합니다.

| 범주                                             | 허용 조건                                                                                                                        |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 개발자 생산성                                    | 목록이 사용자의 소프트웨어 빌드, 테스트, 마이그레이션, 디버깅, 문서화 또는 운영을 지원합니다.                                    |
| UI, 데이터 및 자동화 워크플로                    | 범위가 명확하고 필요한 자격 증명이 명시되어 있으며, 위험한 작업에 검토, 시험 실행, 미리보기 또는 확인 절차가 포함되어 있습니다. |
| 방어적 보안, 검토 및 악용 검토                   | 도구가 승인된 검토용으로 명시되고, 증거를 보존하며, 사람의 승인 경계를 명확히 유지합니다.                                       |
| 개인 또는 팀 워크플로                            | 워크플로가 동의 기반 계정, 투명한 설정 및 명시적인 권한을 사용합니다.                                                           |
| 유지 관리되는 카탈로그                           | 각 목록이 서로 구별되고 유용하며 정확히 설명되어 있고 합리적으로 유지 관리됩니다.                                              |

맥락이 중요합니다. 같은 주제라도 제한적인 방어 목적이나 동의 기반 환경에서는 허용될 수 있지만, 악용 워크플로로 패키징되면 허용되지 않습니다.

## 금지되는 콘텐츠

ClawHub는 악용, 기만, 안전하지 않은 실행 또는 권리 침해를 주목적으로 하는 콘텐츠를 호스팅하지 않습니다.

| 범주                                                        | 허용되지 않는 항목                                                                                                                                                                                                                                                                                                      |
| ----------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 무단 액세스 또는 보안 우회                                  | 인증 우회, 계정 탈취, 속도 제한 악용, 실시간 통화 또는 에이전트 탈취, 재사용 가능한 세션 도용, 승인되지 않은 사용자의 페어링 흐름 자동 승인.                                                                                                                                                                             |
| 플랫폼 악용 및 차단 회피                                    | 차단 후 은밀한 계정 생성, 계정 워밍 또는 파밍, 가짜 참여, 다중 계정 자동화, 대량 게시, 스팸 봇 또는 탐지를 피하도록 만들어진 자동화.                                                                                                                                                                                     |
| 사기, 기만 및 사기성 금융 워크플로                          | 가짜 인증서 또는 청구서, 기만적인 결제 흐름, 사기성 홍보, 가짜 사회적 증거, 사기 목적의 합성 신원 워크플로 또는 사람의 명확한 승인 없이 지출하거나 요금을 청구하는 도구.                                                                                                                                                 |
| 개인정보를 침해하는 정보 보강 또는 감시                     | 스팸 목적의 연락처 스크래핑, 신상 털기, 스토킹, 원치 않는 홍보와 결합된 잠재 고객 정보 추출, 은밀한 모니터링, 비동의 생체정보 대조 또는 유출된 데이터나 침해 데이터 덤프 사용.                                                                                                                                             |
| 비동의 사칭 또는 신원 조작                                  | 사칭하거나 오도하는 데 사용되는 얼굴 바꾸기, 디지털 트윈, 복제된 인플루언서, 가짜 페르소나 또는 기타 도구.                                                                                                                                                                                                             |
| 노골적인 성적 콘텐츠 또는 안전 기능이 비활성화된 성인물 생성 | NSFW 이미지, 동영상 또는 콘텐츠 생성, 서드 파티 API를 감싼 성인 콘텐츠 래퍼 또는 노골적인 성적 콘텐츠를 주목적으로 하는 목록.                                                                                                                                                                                          |
| 숨겨지거나 안전하지 않거나 오해를 유발하는 실행 요구 사항   | 난독화된 설치 명령, 명확하게 검토할 수 없는 상태에서 다운로드한 콘텐츠를 `sh` 또는 `bash`로 실행하는 파이프-투-셸 설치 프로그램, 공개되지 않은 비밀 정보 또는 개인 키 요구 사항, 명확하게 검토할 수 없는 원격 `npx @latest` 실행 또는 목록 실행에 실제로 필요한 사항을 숨기는 메타데이터. |
| 저작권을 침해하거나 권리를 위반하는 자료                     | 허가 없이 다른 사람의 Skills, Plugin, 문서, 브랜드 자산 또는 독점 코드를 재게시하는 행위, 라이선스 조건을 위반하는 행위 또는 원저자나 게시자를 사칭하는 행위.                                                                                                                                                             |

## 금지되는 마켓플레이스 행위

ClawHub는 게시자가 마켓플레이스를 사용하는 방식도 검토합니다. 검색 노출, 지표, 신뢰 신호, 검토 시스템 또는 사용자의 관심을 조작하는 데 ClawHub를 사용하지 마세요.

금지되는 마켓플레이스 행위는 다음과 같습니다.

- 실질적인 사용자 가치가 없어 보이는 저품질, 중복, 자리표시자 또는
  기계 생성 목록을 대량으로 게시하는 행위
- 거의 동일한 Skills 또는 Plugin으로 검색이나 범주 영역을 도배하는 행위
- 사용 실적, 유지 관리, 출처의 명확성 또는 의미 있는 차별성이 거의 없거나
  전혀 없는 수백 개의 목록을 게시하는 행위
- 자동화, 자기 설치 반복, 가짜 계정, 조직적인 활동, 유료 참여 또는 기타
  비유기적 행위를 통해 설치 수, 다운로드 수, 별 수 또는 기타 참여 지표를
  인위적으로 부풀리는 행위
- 검토, 차단, 게시자 제한 또는 마켓플레이스 심사를 회피하기 위해 계정을
  생성하거나 교체하는 행위
- 소유권, 출처, 기능, 보안 태세, 설치 요구 사항 또는 다른 프로젝트나
  게시자와의 제휴 관계에 대해 사용자를 오도하는 행위
- 근본적인 문제를 해결하지 않고 이미 숨김, 삭제 또는 차단된 콘텐츠를
  반복해서 업로드하는 행위

대량 게시는 자동으로 악용으로 간주되지 않습니다. 목록이 의미 있게 서로 다르고, 정확히 설명되어 있으며, 유지 관리되고, 실제 사용자가 사용한다면 대규모 카탈로그도 허용됩니다. 대규모 카탈로그는 그 규모가 부실하거나 중복되거나 오해를 유발하거나 유지 관리되지 않거나 인위적으로 홍보된 목록과 결합될 때 신뢰 및 안전 문제가 됩니다.

## 콘텐츠 권리

ClawHub의 콘텐츠가 저작권 또는 기타 권리를 침해한다고 생각되면 [콘텐츠 권리 요청](/clawhub/content-rights)을 이용하세요. 해당 목록이 안전하지 않거나 악의적이거나 오해를 유발하는 경우가 아니라면, 저작권 또는 권리 침해 신고에 일반 마켓플레이스 신고 기능을 사용하지 마세요.

## 검토 및 집행

ClawHub는 안전하지 않은 콘텐츠 또는 악의적인 게시 행위를 식별하기 위해 자동 검사, 통계적 악용 신호, 사용자 신고 및 직원 검토를 사용할 수 있습니다. 신호 하나만으로 악용이 입증되지는 않으며, ClawHub가 검토가 필요한 대상을 판단하는 데 도움을 줍니다.

당사는 다음과 같은 조치를 취할 수 있습니다.

- 위반 목록을 숨기거나, 보류하거나, 삭제하거나, 소프트 삭제하거나,
  리소스 유형에서 지원되는 경우 영구 삭제
- 안전하지 않은 릴리스의 다운로드 또는 설치 차단
- API 토큰 취소
- 관련 콘텐츠 소프트 삭제
- 게시 액세스 제한
- 반복적으로 또는 심각하게 위반하는 사용자 차단

명백한 악용에 대해 항상 먼저 경고한다고 보장하지 않습니다. 신고, 검토 보류, 숨겨진 목록, 차단 및 계정 상태에 대한 자세한 내용은 [검토 및 계정 안전](/clawhub/moderation)을 참조하세요.
