# 허용 가능한 사용

이 페이지는 ClawHub가 허용하는 스킬과 콘텐츠의 종류, 그리고 호스팅하지 않을 악용 워크플로를 설명합니다.

이 규칙은 의도적으로 실용적입니다. 우리는 고립된 키워드보다 엔드투엔드 악용 워크플로를 가장 중요하게 봅니다. 스킬이 방어를 회피하거나, 플랫폼을 악용하거나, 사람을 속이거나, 프라이버시를 침해하거나, 동의 없는 행동을 가능하게 하도록 만들어졌다면 ClawHub에 적합하지 않습니다.

## 명시적으로 허용하는 최근 패턴

- 실제 컴포넌트, 시맨틱 토큰, 접근 가능한 상태, 테스트된 사용자 흐름을 사용하는 프론트엔드 및 디자인 시스템 작업.
- 일회성 마크업 대신 설치된 소스 컴포넌트, 프로젝트 별칭, 문서화된 변형을 사용하는 shadcn/ui 구성.
- 주석을 보존하고, 구체적인 UI5 타입을 사용하며, 생성된 컨트롤 인터페이스를 검토 가능하게 유지하는 UI5 JavaScript-to-TypeScript 변환.
- 증거를 제시하고 사람의 승인 경계를 명확히 유지하는 방어적 보안 검토, 모더레이션 도구, 악용 탐지 프롬프트.
- 명시적 자격 증명, 투명한 설정, 드라이런 또는 미리보기 모드를 갖춘 개인 또는 팀 계정용 동의 기반 워크플로 자동화.
- 지원하는 소프트웨어 범위로 한정된 문서, 마이그레이션 런북, 개발자 유틸리티, 테스트 픽스처.

## 허용되지 않음

- 보안 우회 또는 무단 접근 워크플로.
  - 예: 인증 우회, 계정 탈취, CAPTCHA 우회, Cloudflare 또는 안티봇 회피, 속도 제한 우회, 보호 장치를 무력화하도록 설계된 은밀한 스크래핑, 실시간 통화 또는 에이전트 탈취, 재사용 가능한 세션 절도, 승인되지 않은 사용자를 위한 페어링 흐름 자동 승인.

- 플랫폼 악용 및 차단 회피.
  - 예: 차단 후 은밀한 계정 생성, 계정 워밍/파밍, 가짜 참여, 카르마 또는 팔로워 육성, 다중 계정 자동화, 대량 게시, 스팸 봇, 탐지를 피하도록 만들어진 마켓플레이스 또는 소셜 자동화.

- 사기, 스캠, 기만적 금융 워크플로.
  - 예: 가짜 인증서, 가짜 송장, 기만적 결제 흐름, 스캠 홍보, 가짜 사회적 증거, 명확한 사람의 승인과 투명한 제어 없이 지출 또는 청구를 가능하게 하는 도구, 사기 목적으로 계정을 만들도록 설계된 합성 신원 워크플로.

- 프라이버시를 침해하는 스크래핑, 보강, 감시.
  - 예: 스팸을 위한 대규모 연락처 세부 정보 스크래핑, 신상털이, 스토킹, 원치 않는 홍보와 결합된 리드 추출, 은밀한 모니터링, 명확한 동의 없이 사용되는 얼굴 검색 또는 생체 매칭, 유출 데이터 또는 침해 덤프의 구매, 게시, 다운로드, 운영화.

- 동의 없는 사칭 또는 기만적 신원 조작.
  - 예: 얼굴 교체, 디지털 트윈, 가짜 페르소나, 복제된 인플루언서, 또는 사칭하거나 오도하는 데 사용되는 기타 신원 조작 도구.

- 노골적인 성적 콘텐츠 및 안전 기능이 비활성화된 성인물 생성.
  - 예: NSFW 이미지/동영상/콘텐츠 생성, 타사 API를 감싼 성인 콘텐츠 래퍼, 또는 주요 목적이 노골적인 성적 콘텐츠인 스킬.

- 숨겨져 있거나 안전하지 않거나 오해를 유발하는 실행 요구 사항.
  - 예: 난독화된 설치 명령, `curl | sh`, 선언되지 않은 시크릿 요구 사항, 선언되지 않은 개인 키 사용, 명확한 검토 가능성 없는 원격 `npx @latest` 실행, 스킬 실행에 실제로 필요한 것을 숨기는 오해의 소지가 있는 메타데이터.

## 명시적으로 허용하지 않는 최근 패턴

- “마켓플레이스 차단 후 은밀한 판매자 계정 생성.”
- “승인되지 않은 사용자가 페어링 코드를 자동으로 받도록 Telegram 페어링 수정.”
- “탐지되지 않는 자동화로 Reddit/Twitter 계정 육성.”
- “임의 사용을 위한 전문 인증서 또는 송장 생성.”
- “안전 검사가 비활성화된 NSFW 콘텐츠 생성.”
- “리드를 스크래핑하고, 연락처를 보강하며, 대규모 콜드 아웃리치 시작.”
- “유출 데이터 또는 침해 덤프 구매, 게시 또는 다운로드.”
- “합성 신원 또는 CAPTCHA 풀이로 이메일 또는 소셜 계정 대량 생성.”

## 검토자를 위한 참고 사항

- 맥락이 중요합니다. 동일한 주제라도 좁은 방어적 또는 동의 기반 설정에서는 정당할 수 있지만, 악용 워크플로로 패키징되면 허용될 수 없습니다.
- 스킬이 회피, 기만, 동의 없는 사용에 명확히 최적화되어 있다면 조치 쪽으로 판단해야 합니다.
- 이러한 범주의 반복 업로드는 콘텐츠 숨김 및 계정 차단의 근거가 됩니다.

## 집행

- 위반 스킬을 숨기거나, 제거하거나, 완전히 삭제할 수 있습니다.
- 토큰을 취소하고, 관련 콘텐츠를 소프트 삭제하며, 반복적이거나 심각한 위반자를 차단할 수 있습니다.
- 명백한 악용에 대해 경고 우선 집행을 보장하지 않습니다.
