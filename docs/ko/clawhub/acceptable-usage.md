# 허용 가능한 사용

ClawHub는 OpenClaw를 위한 Skills, plugins, packages 및 marketplace 메타데이터를 호스팅합니다.
이 페이지를 사용하여 콘텐츠 또는 게시 동작이 ClawHub에 속하는지 판단하세요.

이 규칙은 listing이 수행하는 작업, 사용자에게 실행하도록 요청하는 내용, 자신을 나타내는 방식, 그리고 게시자가 ClawHub의 발견, 설치 및 신뢰 표면을 사용하는 방식에 적용됩니다. 조정 상태와 계정 상태는 [조정 및 계정 안전](/clawhub/moderation)을 참조하세요. 저작권 또는 기타 권리 주장에 대해서는 [콘텐츠 권리 요청](/clawhub/content-rights)을 참조하세요.

## 허용되는 콘텐츠

ClawHub는 유용하고 이해하기 쉬우며 선의로 게시된 콘텐츠를 환영합니다.

| 범주 | 허용되는 경우 |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 개발자 생산성 | listing이 사용자가 소프트웨어를 빌드, 테스트, 마이그레이션, 디버그, 문서화 또는 운영하는 데 도움이 되는 경우 |
| UI, 데이터 및 자동화 워크플로 | 범위가 명확하고, 필요한 자격 증명이 명시되어 있으며, 위험한 작업에 검토, dry-run, 미리 보기 또는 확인 경로가 포함된 경우 |
| 방어적 보안, 조정 및 악용 검토 | 도구가 승인된 검토를 위해 구성되어 있고, 증거를 보존하며, 인간 승인 경계를 명확하게 유지하는 경우 |
| 개인 또는 팀 워크플로 | 워크플로가 동의 기반 계정, 투명한 설정 및 명시적 권한을 사용하는 경우 |
| 유지 관리되는 카탈로그 | 각 listing이 구별되고, 유용하며, 정확하게 설명되어 있고, 합리적으로 유지 관리되는 경우 |

맥락이 중요합니다. 같은 주제라도 좁은 방어적 또는 동의 기반 설정에서는 허용될 수 있지만, 악용 워크플로로 패키징되면 허용되지 않을 수 있습니다.

## 허용되지 않는 콘텐츠

ClawHub는 주된 목적이 악용, 기만, 안전하지 않은 실행 또는 권리 침해인 콘텐츠를 호스팅하지 않습니다.

| 범주 | 허용되지 않음 |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 무단 액세스 또는 보안 우회 | 인증 우회, 계정 탈취, rate-limit 악용, live call 또는 agent 탈취, 재사용 가능한 session 탈취, 또는 승인되지 않은 사용자를 위한 pairing flow 자동 승인 |
| 플랫폼 악용 및 차단 회피 | 차단 후 stealth account, 계정 warming 또는 farming, fake engagement, multi-account automation, mass posting, spam bots, 또는 탐지를 피하도록 구축된 자동화 |
| 사기, 스캠 및 기만적 금융 워크플로 | 가짜 인증서 또는 송장, 기만적 결제 flow, scam outreach, 가짜 social proof, 사기를 위한 synthetic-identity workflows, 또는 명확한 인간 승인 없는 지출/청구 도구 |
| 프라이버시 침해 enrichment 또는 surveillance | spam을 위한 contact scraping, doxxing, stalking, unsolicited outreach와 결합된 lead extraction, covert monitoring, 동의 없는 biometric matching, 또는 leaked data나 breach dumps 사용 |
| 동의 없는 사칭 또는 신원 조작 | face swap, digital twins, cloned influencers, fake personas, 또는 사칭하거나 오도하는 데 사용되는 기타 도구 |
| 명시적 성적 콘텐츠 또는 안전장치가 비활성화된 성인용 생성 | NSFW 이미지, 동영상 또는 콘텐츠 생성; 타사 API를 둘러싼 adult-content wrappers; 또는 주된 목적이 명시적 성적 콘텐츠인 listing |
| 숨겨진, 안전하지 않은 또는 오도하는 실행 요구 사항 | 난독화된 설치 명령, 명확한 검토 가능성 없이 다운로드된 콘텐츠를 `sh` 또는 `bash`로 실행하는 pipe-to-shell 설치 프로그램, 선언되지 않은 secret 또는 private-key 요구 사항, 명확한 검토 가능성 없는 원격 `npx @latest` 실행, 또는 listing이 실제 실행에 필요한 것을 숨기는 메타데이터 |
| 저작권 침해 또는 권리 위반 자료 | 허가 없이 다른 사람의 skill, plugin, 문서, 브랜드 자산 또는 proprietary code를 재게시하는 행위; 라이선스 조건 위반; 또는 원저자나 게시자를 사칭하는 행위 |

## 허용되지 않는 marketplace 동작

ClawHub는 게시자가 marketplace를 사용하는 방식도 검토합니다. 발견, 지표, 신뢰 신호, 조정 시스템 또는 사용자 주의를 조작하기 위해 ClawHub를 사용하지 마세요.

허용되지 않는 marketplace 동작에는 다음이 포함됩니다.

- 실제 사용자 가치가 있는 것으로 보이지 않는, 노력이 적고 중복적이며 placeholder이거나 machine-generated인 listing을 대량 게시하는 행위
- 거의 동일한 skills 또는 plugins로 검색이나 카테고리 표면을 범람시키는 행위
- 사용량, 유지 관리, source 명확성 또는 의미 있는 차별화가 거의 없거나 전혀 없는 수백 개의 listing을 게시하는 행위
- 자동화, self-install loops, fake accounts, coordinated activity, paid engagement 또는 기타 비유기적 동작을 통해 설치, 다운로드, stars 또는 기타 engagement 지표를 인위적으로 부풀리는 행위
- 조정, 차단, 게시자 제한 또는 marketplace 검토를 회피하기 위해 계정을 만들거나 순환 사용하는 행위
- 소유권, source, 기능, 보안 태세, 설치 요구 사항 또는 다른 프로젝트나 게시자와의 제휴에 대해 사용자를 오도하는 행위
- 근본적인 문제를 해결하지 않고 이미 숨김, 제거 또는 차단된 콘텐츠를 반복적으로 업로드하는 행위

대량 게시가 자동으로 악용인 것은 아닙니다. listing이 의미 있게 다르고, 정확하게 설명되어 있으며, 유지 관리되고, 실제 사용자가 사용하는 경우 대형 카탈로그는 허용됩니다. 대형 카탈로그는 규모가 얄팍하고, 중복적이며, 오도하거나, 유지 관리되지 않거나, 인위적으로 홍보된 listing과 결합될 때 신뢰 및 안전 문제가 됩니다.

## 콘텐츠 권리

ClawHub의 콘텐츠가 귀하의 저작권 또는 기타 권리를 침해한다고 생각하는 경우 [콘텐츠 권리 요청](/clawhub/content-rights)을 사용하세요. listing이 안전하지 않거나, 악의적이거나, 오도하는 경우가 아니라면 저작권 또는 권리 주장에 일반 marketplace 신고를 사용하지 마세요.

## 검토 및 집행

ClawHub는 안전하지 않은 콘텐츠 또는 악의적 게시 동작을 식별하기 위해 자동 검사, 통계적 악용 신호, 사용자 신고 및 직원 검토를 사용할 수 있습니다. 신호 자체만으로 악용을 증명하지는 않으며, ClawHub가 검토가 필요한 대상을 결정하는 데 도움이 됩니다.

당사는 다음을 수행할 수 있습니다.

- 위반 listing을 숨기거나, 보류하거나, 제거하거나, soft-delete하거나, 리소스 유형에서 지원되는 경우 hard-delete할 수 있습니다.
- 안전하지 않은 release에 대한 다운로드 또는 설치 차단
- API tokens 취소
- 관련 콘텐츠 soft-delete
- 게시 액세스 제한
- 반복적이거나 심각한 위반자 차단

명백한 악용에 대해 먼저 경고하는 집행을 보장하지 않습니다. 신고, 조정 보류, 숨겨진 listing, 차단 및 계정 상태에 대해서는 [조정 및 계정 안전](/clawhub/moderation)을 참조하세요.
