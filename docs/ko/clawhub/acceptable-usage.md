# 허용 가능한 사용

ClawHub는 OpenClaw용 Skills, Plugin, 패키지, 마켓플레이스 메타데이터를 호스팅합니다.
이 페이지를 사용해 콘텐츠나 게시 동작이 ClawHub에 적합한지 판단하세요.

이 규칙은 등록 항목이 무엇을 하는지, 사용자에게 무엇을 실행하도록 요구하는지, 자신을 어떻게
표현하는지, 게시자가 ClawHub의 탐색, 설치, 신뢰 표면을 어떻게 사용하는지에 적용됩니다.
검토 상태와 계정 상태는 [검토 및 계정 안전](/clawhub/moderation)을 참조하세요.
저작권 또는 기타 권리 침해 주장은 [콘텐츠 권리 요청](/ko/clawhub/content-rights)을 참조하세요.

## 허용되는 콘텐츠

ClawHub는 유용하고, 이해하기 쉬우며, 선의로 게시된 콘텐츠를 환영합니다.

| 범주                                             | 허용되는 경우                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 개발자 생산성                                   | 등록 항목이 사용자가 소프트웨어를 빌드, 테스트, 마이그레이션, 디버그, 문서화 또는 운영하는 데 도움을 줍니다.                   |
| UI, 데이터, 자동화 워크플로                     | 범위가 명확하고, 필요한 자격 증명이 명시되어 있으며, 위험한 작업에 검토, dry-run, 미리보기 또는 확인 경로가 포함됩니다.        |
| 방어적 보안, 검토, 남용 검토                    | 도구가 승인된 검토용으로 설명되고, 증거를 보존하며, 사람의 승인 경계를 명확히 유지합니다.                                      |
| 개인 또는 팀 워크플로                           | 워크플로가 동의 기반 계정, 투명한 설정, 명시적 권한을 사용합니다.                                                               |
| 유지 관리되는 카탈로그                          | 각 등록 항목이 서로 구별되고, 유용하며, 정확하게 설명되어 있고, 합리적으로 유지 관리됩니다.                                    |

맥락이 중요합니다. 같은 주제라도 좁은 방어적 또는 동의 기반 설정에서는 허용될 수 있지만,
남용 워크플로로 패키징되면 허용되지 않을 수 있습니다.

## 금지되는 콘텐츠

ClawHub는 주된 목적이 남용, 기만, 안전하지 않은 실행 또는 권리 침해인 콘텐츠를 호스팅하지 않습니다.

| 범주                                                        | 허용되지 않음                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 무단 접근 또는 보안 우회                                   | 인증 우회, 계정 탈취, rate-limit 남용, 실시간 호출 또는 에이전트 탈취, 재사용 가능한 세션 탈취, 승인되지 않은 사용자를 위한 페어링 흐름 자동 승인.                                                                                                                                                           |
| 플랫폼 남용 및 차단 회피                                   | 차단 후 은밀한 계정 사용, 계정 워밍 또는 파밍, 가짜 참여, 다중 계정 자동화, 대량 게시, 스팸 봇, 탐지를 피하도록 만들어진 자동화.                                                                                                                                                                             |
| 사기, 스캠, 기만적 금융 워크플로                           | 가짜 인증서 또는 청구서, 기만적 결제 흐름, 스캠 홍보, 가짜 사회적 증거, 사기를 위한 합성 신원 워크플로, 명확한 사람의 승인 없는 지출/청구 도구.                                                                                                                                                              |
| 개인정보 침해성 보강 또는 감시                             | 스팸 목적의 연락처 스크래핑, doxxing, 스토킹, 원치 않는 홍보와 결합된 리드 추출, 은밀한 모니터링, 비동의 생체 정보 매칭, 유출 데이터 또는 침해 덤프 사용.                                                                                                                                                    |
| 비동의 사칭 또는 신원 조작                                 | Face swap, 디지털 트윈, 복제된 인플루언서, 가짜 페르소나, 또는 사칭하거나 오도하는 데 사용되는 기타 도구.                                                                                                                                                                                                    |
| 노골적인 성적 콘텐츠 또는 안전장치가 비활성화된 성인 생성 | NSFW 이미지, 동영상 또는 콘텐츠 생성, 서드파티 API를 둘러싼 성인 콘텐츠 래퍼, 또는 주된 목적이 노골적인 성적 콘텐츠인 등록 항목.                                                                                                                                                                             |
| 숨겨진, 안전하지 않은, 또는 오해를 부르는 실행 요구 사항  | 난독화된 설치 명령, 명확한 검토 가능성 없이 다운로드된 콘텐츠를 `sh` 또는 `bash`로 실행하는 pipe-to-shell 설치 프로그램, 선언되지 않은 secret 또는 private-key 요구 사항, 명확한 검토 가능성 없는 원격 `npx @latest` 실행, 또는 등록 항목이 실제 실행에 필요한 것을 숨기는 메타데이터. |
| 저작권 침해 또는 권리 위반 자료                            | 허가 없이 다른 사람의 skill, plugin, 문서, 브랜드 자산 또는 독점 코드를 재게시하는 행위, 라이선스 조건 위반, 원저자 또는 게시자 사칭.                                                                                                                                                                         |

## 금지되는 마켓플레이스 동작

ClawHub는 게시자가 마켓플레이스를 어떻게 사용하는지도 검토합니다. ClawHub를 사용해
탐색, 지표, 신뢰 신호, 검토 시스템 또는 사용자 주의를 조작하지 마세요.

금지되는 마켓플레이스 동작에는 다음이 포함됩니다.

- 실제 사용자 가치가 없어 보이는 저품질, 중복, 자리표시자 또는
  기계 생성 등록 항목을 대량 게시하는 행위
- 거의 동일한 Skills 또는 Plugin으로 검색 또는 범주 표면을 넘치게 하는 행위
- 사용량, 유지 관리, 소스 명확성 또는 의미 있는 차별화가 거의 없거나 전혀 없는
  수백 개의 등록 항목을 게시하는 행위
- 자동화, 자기 설치 루프, 가짜 계정, 조직적 활동, 유료 참여 또는 기타 비유기적 동작을 통해
  설치, 다운로드, 별표 또는 기타 참여 지표를 인위적으로 부풀리는 행위
- 검토, 차단, 게시자 제한 또는 마켓플레이스 검토를 회피하기 위해
  계정을 만들거나 교체하는 행위
- 소유권, 소스, 기능, 보안 태세, 설치 요구 사항 또는 다른 프로젝트나 게시자와의 관계에 대해
  사용자를 오도하는 행위
- 근본 원인을 해결하지 않은 채 이미 숨김, 제거 또는 차단된 콘텐츠를
  반복적으로 업로드하는 행위

대량 게시는 자동으로 남용으로 간주되지 않습니다. 등록 항목이 의미 있게 다르고,
정확하게 설명되어 있으며, 유지 관리되고, 실제 사용자가 사용한다면 대규모 카탈로그도 허용됩니다.
대규모 카탈로그는 규모가 얕고, 중복되고, 오해를 부르며, 유지 관리되지 않거나,
인위적으로 홍보된 등록 항목과 결합될 때 신뢰 및 안전 문제가 됩니다.

## 콘텐츠 권리

ClawHub의 콘텐츠가 귀하의 저작권 또는 기타 권리를 침해한다고 생각한다면
[콘텐츠 권리 요청](/ko/clawhub/content-rights)을 사용하세요. 등록 항목이 안전하지 않거나,
악의적이거나, 오해를 부르는 경우가 아니라면 저작권 또는 권리 침해 주장에 일반 마켓플레이스
신고를 사용하지 마세요.

## 검토 및 집행

ClawHub는 안전하지 않은 콘텐츠 또는 남용성 게시 동작을 식별하기 위해 자동화된 검사,
통계적 남용 신호, 사용자 신고, 직원 검토를 사용할 수 있습니다. 신호 하나만으로 남용이
입증되지는 않습니다. 이는 ClawHub가 검토가 필요한 대상을 판단하는 데 도움을 줍니다.

당사는 다음을 수행할 수 있습니다.

- 위반 등록 항목을 숨기기, 보류, 제거, soft-delete하거나, 리소스 유형에서 지원되는 경우
  hard-delete하기
- 안전하지 않은 릴리스의 다운로드 또는 설치 차단
- API 토큰 취소
- 관련 콘텐츠 soft-delete
- 게시 접근 제한
- 반복적이거나 심각한 위반자 차단

명백한 남용에 대해 항상 경고 우선 집행을 보장하지는 않습니다. 신고, 검토 보류,
숨겨진 등록 항목, 차단, 계정 상태에 대해서는
[검토 및 계정 안전](/clawhub/moderation)을 참조하세요.
