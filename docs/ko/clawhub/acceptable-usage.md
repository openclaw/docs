# 허용 가능한 사용

ClawHub는 OpenClaw를 위한 Skills, Plugin, 패키지, 마켓플레이스 메타데이터를 호스팅합니다.
콘텐츠나 게시 동작이 ClawHub에 적합한지 판단할 때 이 페이지를 사용하세요.

이 규칙은 목록이 수행하는 작업, 사용자에게 실행을 요구하는 내용, 자신을 나타내는 방식, 그리고 게시자가 ClawHub의 탐색, 설치, 신뢰 영역을 사용하는 방식에 적용됩니다. 조정 상태와 계정 상태는 [조정 및 계정 안전](/clawhub/moderation)을 참조하세요. 저작권 또는 기타 권리 청구는 [콘텐츠 권리 요청](/ko/clawhub/content-rights)을 참조하세요.

## 허용되는 콘텐츠

ClawHub는 유용하고 이해하기 쉬우며 선의로 게시된 콘텐츠를 환영합니다.

| 범주                                         | 허용되는 경우                                                                                                                      |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| 개발자 생산성                           | 목록이 사용자가 소프트웨어를 빌드, 테스트, 마이그레이션, 디버그, 문서화 또는 운영하는 데 도움이 되는 경우.                                               |
| UI, 데이터, 자동화 워크플로               | 범위가 명확하고, 필요한 자격 증명이 명시되어 있으며, 위험한 작업에 검토, 드라이런, 미리 보기 또는 확인 경로가 포함된 경우. |
| 방어적 보안, 조정, 악용 검토 | 도구가 승인된 검토를 위한 것으로 설명되고, 증거를 보존하며, 사람의 승인 경계를 명확히 유지하는 경우.                          |
| 개인 또는 팀 워크플로                       | 워크플로가 동의 기반 계정, 투명한 설정, 명시적 권한을 사용하는 경우.                                            |
| 유지관리되는 카탈로그                              | 각 목록이 서로 구별되고, 유용하며, 정확하게 설명되어 있고, 합리적으로 유지관리되는 경우.                                                |

맥락이 중요합니다. 같은 주제라도 좁은 방어적 또는 동의 기반 환경에서는 허용될 수 있지만, 악용 워크플로로 패키징되면 허용되지 않을 수 있습니다.

## 허용되지 않는 콘텐츠

ClawHub는 주된 목적이 악용, 기만, 안전하지 않은 실행 또는 권리 침해인 콘텐츠를 호스팅하지 않습니다.

| 범주                                                    | 허용되지 않는 경우                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 무단 접근 또는 보안 우회                      | 인증 우회, 계정 탈취, 속도 제한 악용, 실시간 호출 또는 에이전트 탈취, 재사용 가능한 세션 탈취, 승인되지 않은 사용자를 위한 페어링 흐름 자동 승인.                                                                                                                                                   |
| 플랫폼 악용 및 차단 회피                              | 차단 후 은닉 계정 사용, 계정 예열 또는 양산, 가짜 참여, 다중 계정 자동화, 대량 게시, 스팸 봇, 또는 탐지를 피하도록 만들어진 자동화.                                                                                                                                          |
| 사기, 스캠, 기만적 금융 워크플로             | 가짜 인증서 또는 송장, 기만적 결제 흐름, 스캠 연락, 가짜 사회적 증거, 사기를 위한 합성 신원 워크플로, 또는 명확한 사람의 승인 없는 지출/청구 도구.                                                                                                                    |
| 개인정보 침해적 보강 또는 감시                 | 스팸을 위한 연락처 스크래핑, 신상 털기, 스토킹, 원치 않는 연락과 결합된 리드 추출, 은밀한 모니터링, 비동의 생체 인식 매칭, 또는 유출 데이터나 침해 덤프 사용.                                                                                                                  |
| 비동의 사칭 또는 신원 조작       | 얼굴 교체, 디지털 트윈, 복제된 인플루언서, 가짜 페르소나, 또는 사칭하거나 오도하는 데 사용되는 기타 도구.                                                                                                                                                                                                 |
| 명시적 성적 콘텐츠 또는 안전장치가 비활성화된 성인 콘텐츠 생성 | NSFW 이미지, 비디오 또는 콘텐츠 생성, 타사 API를 둘러싼 성인 콘텐츠 래퍼, 또는 명시적 성적 콘텐츠를 주된 목적으로 하는 목록.                                                                                                                                                       |
| 숨겨진, 안전하지 않은, 또는 오해의 소지가 있는 실행 요구 사항        | 난독화된 설치 명령, 명확한 검토 가능성 없이 다운로드한 콘텐츠를 `sh` 또는 `bash`로 실행하는 pipe-to-shell 설치 프로그램, 선언되지 않은 비밀값 또는 개인 키 요구 사항, 명확한 검토 가능성 없는 원격 `npx @latest` 실행, 또는 목록이 실제로 실행에 필요한 내용을 숨기는 메타데이터. |
| 저작권 침해 또는 권리 위반 자료           | 허가 없이 다른 사람의 Skill, Plugin, 문서, 브랜드 자산 또는 독점 코드를 재게시하는 행위, 라이선스 조건 위반, 또는 원저자나 게시자를 사칭하는 행위.                                                                                                                            |

## 허용되지 않는 마켓플레이스 동작

ClawHub는 게시자가 마켓플레이스를 사용하는 방식도 검토합니다. ClawHub를 사용해 탐색, 지표, 신뢰 신호, 조정 시스템 또는 사용자 관심을 조작하지 마세요.

허용되지 않는 마켓플레이스 동작에는 다음이 포함됩니다.

- 실제 사용자 가치가 없어 보이는 낮은 노력의 중복적, 자리표시자성 또는
  기계 생성 목록을 대량으로 게시하는 행위
- 거의 동일한 Skills 또는 Plugin으로 검색이나 범주 영역을 넘치게 하는 행위
- 사용량, 유지관리, 출처 명확성 또는 의미 있는 차별점이 거의 없거나 전혀 없는 목록을 수백 개 게시하는 행위
- 자동화, 자체 설치 루프, 가짜 계정, 조직적 활동, 유료 참여 또는 기타 비유기적 동작을 통해 설치, 다운로드, 별표 또는 기타 참여 지표를 인위적으로 부풀리는 행위
- 조정, 차단, 게시자 제한 또는 마켓플레이스 검토를 회피하기 위해 계정을 생성하거나 순환 사용하는 행위
- 소유권, 출처, 기능, 보안 태세, 설치 요구 사항 또는 다른 프로젝트나 게시자와의 제휴 관계에 대해 사용자를 오도하는 행위
- 근본 원인을 고치지 않고 이미 숨김 처리, 제거 또는 차단된 콘텐츠를 반복해서 업로드하는 행위

대량 게시가 자동으로 악용인 것은 아닙니다. 목록이 의미 있게 다르고, 정확하게 설명되어 있으며, 유지관리되고, 실제 사용자가 사용하는 경우 대형 카탈로그는 허용됩니다. 대형 카탈로그는 규모가 얇고, 중복적이며, 오해의 소지가 있고, 유지관리되지 않거나, 인위적으로 홍보된 목록과 결합될 때 신뢰와 안전 문제가 됩니다.

## 콘텐츠 권리

ClawHub의 콘텐츠가 귀하의 저작권 또는 기타 권리를 침해한다고 생각한다면 [콘텐츠 권리 요청](/ko/clawhub/content-rights)을 사용하세요. 해당 목록이 안전하지 않거나, 악의적이거나, 오해의 소지가 있는 경우가 아니라면 저작권 또는 권리 청구에 일반 마켓플레이스 신고를 사용하지 마세요.

## 검토 및 집행

ClawHub는 안전하지 않은 콘텐츠나 악의적 게시 동작을 식별하기 위해 자동 검사, 통계적 악용 신호, 사용자 신고, 직원 검토를 사용할 수 있습니다. 신호 자체만으로 악용이 입증되지는 않지만, ClawHub가 무엇을 검토해야 하는지 결정하는 데 도움이 됩니다.

당사는 다음을 수행할 수 있습니다.

- 위반 목록을 숨김 처리, 보류, 제거, 소프트 삭제하거나, 리소스 유형에서 지원되는 경우 하드 삭제
- 안전하지 않은 릴리스의 다운로드 또는 설치 차단
- API 토큰 취소
- 관련 콘텐츠 소프트 삭제
- 게시 접근 제한
- 반복적이거나 심각한 위반자 차단

명백한 악용에 대해 경고 우선 집행을 보장하지 않습니다. 신고, 조정 보류, 숨겨진 목록, 차단, 계정 상태는 [조정 및 계정 안전](/clawhub/moderation)을 참조하세요.
