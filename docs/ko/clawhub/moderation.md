# 조정 및 계정 안전

ClawHub는 게시에 열려 있지만, 공개 검색 및 설치 표면에는 여전히
보호 장치가 필요합니다. 신고, 조정 보류, 숨김 목록, 계정 조치는
릴리스나 계정이 안전하지 않거나, 오해를 유발하거나, 정책을 벗어난 것으로 보일 때
사용자를 보호하는 데 도움이 됩니다.

이 페이지는 조정 및 계정 상태를 다룹니다. `Pass`, `Review`, `Warn`,
`Malicious` 및 위험 수준과 같은 감사 라벨은
[보안 감사](/clawhub/security-audits)를 참조하세요.

[보안](/clawhub/security) 및
[허용되는 사용](/clawhub/acceptable-usage)도 참조하세요. 저작권 또는 기타 콘텐츠
권리 관련 우려 사항은 [콘텐츠 권리 요청](/clawhub/content-rights)을 사용하세요.

## 신고

로그인한 사용자는 Skills, Plugin, 패키지를 신고할 수 있습니다.

ClawHub 신고는 다음과 같은 안전하지 않은 마켓플레이스 콘텐츠에만 사용하세요.

- 악성 목록
- 오해를 유발하는 메타데이터
- 선언되지 않은 자격 증명 또는 권한 요구 사항
- 의심스러운 설치 지침
- 사칭
- 악의적 등록 또는 상표 오용
- [허용되는 사용](/clawhub/acceptable-usage)을 위반하는 콘텐츠

Skills 페이지의 **Skills 신고** 버튼이나 패키지용 패키지 신고
명령/API를 사용하세요.

타사 Skills 또는 Plugin 자체 소스 코드의 취약점에는 ClawHub 신고를 사용하지 마세요.
해당 취약점은 목록에 연결된 게시자나 소스 저장소에 직접 신고하세요.
ClawHub는 타사 Skills 또는 Plugin 코드를 유지 관리하거나 패치하지 않습니다.

`openclaw/clawhub`의 GitHub Security Advisories는 ClawHub 자체의 취약점을
위한 것입니다. 예에는 웹사이트, API, CLI, 레지스트리, 인증, 스캔, 조정 또는
다운로드/설치 신뢰 경계의 버그가 포함됩니다. 타사 Skills 또는 Plugin의 취약점에는
ClawHub 권고를 사용하지 마세요.

좋은 신고는 구체적이고 실행 가능해야 합니다. 신고 기능 남용은 그 자체로
계정 조치로 이어질 수 있습니다.

## 조직 및 네임스페이스 클레임

조직, 브랜드, 패키지 범위, 소유자 핸들 또는 네임스페이스 소유권 분쟁은
제품 내 신고 흐름이나 계정 이의 제기 양식이 아니라
[조직 및 네임스페이스 클레임](/clawhub/namespace-claims) 절차를 사용해야 합니다.

네임스페이스를 예약, 이전, 이름 변경, 숨김, 격리, 별칭 지정 또는 기타 방식으로 검토해야 한다는
비민감 증거를 ClawHub 직원이 검토해야 할 때 이 절차를 사용하세요.
공개 이슈에는 비밀, 비공개 문서, 비공개 법률 파일, 개인 신원 문서,
API 토큰 또는 DNS 챌린지 토큰을 포함하지 마세요.

## 조정 보류

일부 심각한 발견 사항이나 정책 문제는 게시자 또는 목록을 조정 보류 상태로 둘 수 있습니다.
이 경우 영향을 받는 콘텐츠가 공개 검색에서 숨겨지거나, 향후 게시가 문제가 검토될 때까지
숨김 상태로 시작될 수 있습니다.

조정 보류는 ClawHub가 고위험 사례를 해결하는 동안 사용자를 보호하기 위한 것입니다.
오탐이 확인되면 해제될 수도 있습니다.

## 숨김 또는 차단된 목록

목록은 보류, 숨김, 격리, 취소되거나 공개 설치 표면에서 다른 방식으로 사용할 수 없게 될 수 있습니다.

이러한 상태 중 하나가 보이면 소유자가 문제를 해결하거나 조정이 복원할 때까지
해당 릴리스를 설치하지 마세요.

소유자는 보류 또는 숨김 상태인 자신의 목록에 대한 진단 정보를 계속 볼 수 있습니다.
이 진단 정보는 무슨 일이 있었는지와 목록이 공개 표면으로 돌아가기 전에
무엇을 변경해야 하는지 설명하는 데 도움이 됩니다.

## 차단 및 계정 상태

ClawHub 정책을 위반하는 계정은 게시 접근 권한을 잃을 수 있습니다. 심각한 남용은
계정 차단, 토큰 취소, 콘텐츠 숨김 또는 목록 제거로 이어질 수 있습니다.
게시자 남용 위험 신호는 매일 확인됩니다. ClawHub의 잠재적 차단 임계값에 도달하는
신호는 자동 경고를 트리거할 수 있습니다. 경고 기한 이후 다음 적격 스캔에서도
게시자가 여전히 잠재적 차단 임계값에 해당하면, ClawHub는 계정 조치를 자동으로 적용할 수 있습니다.
신뢰도가 낮거나 시간 범위가 제한된 검토 신호는 자동 집행에서 제외됩니다.

삭제, 차단 또는 비활성화된 계정은 ClawHub API 토큰을 사용할 수 없습니다. 계정 조치 후
CLI 인증이 실패하기 시작하면 웹 UI에 로그인하여 계정 상태를 검토하세요.
차단 또는 비활성화된 계정으로 인해 로그인이나 정상적인 CLI 접근이 차단된 경우,
복구 검토를 위해 [ClawHub 이의 제기 양식](https://appeals.openclaw.ai/)을 사용하세요.

스캐너가 트리거한 이메일에서 Skills 또는 Plugin 버전을 악성으로 지목하는 경우,
차단된 제출 버전에 대한 저장된 스캔 결과를 다운로드하세요:
`clawhub scan download <slug> --version <version>`. Plugin의 경우
`--kind plugin`을 추가하세요. 스캔 출력을 검토하고, 목록을 수정하고, 버전
번호를 올린 다음, 수정된 버전을 업로드하세요.

## 게시자 지침

오탐을 줄이고 사용자 신뢰를 높이려면:

- 이름, 요약, 태그, 변경 로그를 정확하게 유지하세요
- 필요한 환경 변수와 권한을 선언하세요
- 난독화된 설치 명령을 피하세요
- 가능하면 소스에 연결하세요
- Plugin을 게시하기 전에 드라이런을 사용하세요
- 사용자나 조정자가 릴리스 동작에 대해 묻는 경우 명확하게 응답하세요
