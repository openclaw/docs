# 조정 및 계정 안전

ClawHub에서는 누구나 게시할 수 있지만, 공개 검색 및 설치 화면에는 여전히
안전장치가 필요합니다. 신고, 조정 보류, 숨김 목록 및 계정 조치는
릴리스나 계정이 안전하지 않거나 오해를 유발하거나 정책을 위반하는 것으로
보일 때 사용자를 보호하는 데 도움이 됩니다.

이 페이지에서는 조정 및 계정 상태를 다룹니다. `Pass`,
`Review`, `Warn`, `Malicious`, 위험 수준과 같은 감사 레이블은
[보안 감사](/clawhub/security-audits)를 참조하십시오.

[보안](/clawhub/security) 및
[허용되는 사용](/clawhub/acceptable-usage)도 참조하십시오. 저작권 또는 기타 콘텐츠
권리에 관한 우려 사항은 [콘텐츠 권리 요청](/clawhub/content-rights)을 이용하십시오.

## 신고

로그인한 사용자는 Skills, plugins 및 패키지를 신고할 수 있습니다.

ClawHub 신고는 다음과 같이 안전하지 않은 마켓플레이스 콘텐츠에만 사용하십시오.

- 악성 목록
- 오해를 유발하는 메타데이터
- 명시되지 않은 자격 증명 또는 권한 요구 사항
- 의심스러운 설치 지침
- 사칭
- 악의적인 등록 또는 상표 오용
- [허용되는 사용](/clawhub/acceptable-usage)을 위반하는 콘텐츠

Skills 페이지의 **Skills 신고** 버튼 또는 패키지 신고
명령/API를 사용하십시오.

타사 Skills 또는
Plugin 자체 소스 코드의 취약점을 신고하는 데 ClawHub 신고를 사용하지 마십시오. 목록에서 연결된 게시자 또는 소스
저장소에 직접 신고하십시오. ClawHub는
타사 Skills 또는 Plugin 코드를 유지 관리하거나 패치하지 않습니다.

`openclaw/clawhub`용 GitHub Security Advisories는
ClawHub 자체의 취약점을 위한 것입니다. 예로는 웹사이트, API, CLI, 레지스트리, 인증,
검사, 조정 또는 다운로드/설치 신뢰 경계의 버그가 있습니다. 타사 Skills 또는 plugins의
취약점에 ClawHub 권고를 사용하지 마십시오.

좋은 신고는 구체적이며 조치가 가능합니다. 신고 기능을 악용하면 그 자체로
계정 조치가 이루어질 수 있습니다.

## 조직 및 네임스페이스 소유권 주장

조직, 브랜드, 패키지 범위, 소유자 핸들 또는 네임스페이스 소유권 분쟁에는
제품 내 신고 절차나 계정 이의 제기 양식이 아닌
[조직 및 네임스페이스 소유권 주장](/clawhub/namespace-claims) 절차를 사용해야 합니다.

네임스페이스를 예약, 이전, 이름 변경, 숨김, 격리, 별칭 지정하거나
그 밖의 검토를 해야 한다는 민감하지 않은 증거를 ClawHub 직원이 검토해야 하는 경우 이 절차를
사용하십시오. 공개 이슈에 비밀, 비공개 문서, 비공개 법률
파일, 개인 신원 문서, API 토큰 또는 DNS 챌린지 토큰을 포함하지 마십시오.

## 조정 보류

일부 심각한 조사 결과 또는 정책 문제로 인해 게시자나 목록에
조정 보류가 적용될 수 있습니다. 이 경우 영향을 받는 콘텐츠가 공개
검색에서 숨겨지거나, 문제가 검토될 때까지 향후 게시물이 숨김 상태로 시작될 수 있습니다.

조정 보류는 ClawHub가 고위험
사례를 해결하는 동안 사용자를 보호하기 위한 것입니다. 오탐이 확인되면 해제될 수도 있습니다.

## 숨겨지거나 차단된 목록

목록은 보류, 숨김, 격리, 취소되거나 기타 이유로
공개 설치 화면에서 사용할 수 없게 될 수 있습니다.

이러한 상태 중 하나가 표시되면 소유자가
문제를 해결하거나 조정을 통해 복원될 때까지 해당 릴리스를 설치하지 마십시오.

소유자는 보류되거나 숨겨진 자체 목록의 진단 정보를 계속 볼 수 있습니다. 이러한
진단 정보는 발생한 상황과 목록이 공개 화면에 다시 표시되기 전에
변경해야 할 사항을 설명하는 데 도움이 됩니다.

## 차단 및 계정 상태

ClawHub 정책을 위반한 계정은 게시 권한을 잃을 수 있습니다. 심각한 악용은
계정 차단, 토큰 취소, 콘텐츠 숨김 또는 목록 삭제로
이어질 수 있습니다. 게시자 악용 압력 신호는 매일 확인됩니다.
ClawHub의 잠재적 차단 임계값에 도달하는 신호는 자동 경고를 트리거할 수 있습니다. 경고 기한 후
다음 적격 검사에서도 게시자가
잠재적 차단 임계값에 해당하면 ClawHub가 계정 조치를 자동으로 적용할 수 있습니다.
신뢰도가 낮고 시간 범위가 제한된 검토 신호에는 자동
집행이 적용되지 않습니다.

삭제, 차단 또는 비활성화된 계정은 ClawHub API 토큰을 사용할 수 없습니다. 계정 조치 후 CLI 인증이
실패하기 시작하면 웹 UI에 로그인하여 계정
상태를 검토하십시오. 차단 또는 비활성화된 계정으로 인해 로그인이나 정상적인 CLI 접근이 막힌 경우
복구 검토를 위해 [ClawHub 이의 제기 양식](https://appeals.openclaw.ai/)을 사용하십시오.

스캐너가 트리거한 이메일에서 특정 Skills 또는 Plugin 버전을 악성으로 지목한 경우
차단된 제출 버전에 대해 저장된 검사 결과를 다운로드하십시오.
`clawhub scan download <slug> --version <version>`. plugins의 경우
`--kind plugin`을 추가하십시오. 검사 출력을 검토하고 목록을 수정한 후 버전
번호를 증가시키고 수정된 버전을 업로드하십시오.

## 게시자 지침

오탐을 줄이고 사용자 신뢰를 높이려면 다음을 수행하십시오.

- 이름, 요약, 태그 및 변경 로그를 정확하게 유지하십시오
- 필수 환경 변수와 권한을 명시하십시오
- 난독화된 설치 명령을 피하십시오
- 가능하면 소스에 연결하십시오
- plugins를 게시하기 전에 시험 실행을 사용하십시오
- 사용자나 조정 담당자가 릴리스 동작에 관해 질문하면 명확하게 답변하십시오
