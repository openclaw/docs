# 중재 및 계정 안전

ClawHub는 게시에 열려 있지만, 공개 검색 및 설치 영역에는 여전히
가드레일이 필요합니다. 신고, 중재 보류, 숨김 등록 항목, 계정 조치는
릴리스나 계정이 안전하지 않거나, 오해를 일으키거나, 정책을 벗어난 것으로
보일 때 사용자를 보호하는 데 도움이 됩니다.

이 페이지는 중재와 계정 상태를 다룹니다. `Pass`, `Review`, `Warn`,
`Malicious` 같은 감사 라벨과 위험 수준은
[보안 감사](/ko/clawhub/security-audits)를 참조하세요.

[보안](/ko/clawhub/security) 및
[허용 가능한 사용](/ko/clawhub/acceptable-usage)도 참조하세요. 저작권 또는 기타 콘텐츠
권리 관련 문제는 [콘텐츠 권리 요청](/ko/clawhub/content-rights)을 사용하세요.

## 신고

로그인한 사용자는 Skills, Plugin, 패키지를 신고할 수 있습니다.

다음과 같은 안전하지 않은 마켓플레이스 콘텐츠에만 ClawHub 신고를 사용하세요.

- 악성 등록 항목
- 오해를 일으키는 메타데이터
- 선언되지 않은 자격 증명 또는 권한 요구 사항
- 의심스러운 설치 지침
- 사칭
- 악의적인 등록 또는 상표 오용
- [허용 가능한 사용](/ko/clawhub/acceptable-usage)을 위반하는 콘텐츠

Skills 페이지의 **Skills 신고** 버튼을 사용하거나, 패키지에는 패키지 신고
명령/API를 사용하세요.

타사 Skills 또는 Plugin 자체 소스 코드의 취약점에는 ClawHub 신고를 사용하지
마세요. 해당 취약점은 등록 항목에 연결된 게시자 또는 소스 저장소에 직접
신고하세요. ClawHub는 타사 Skills 또는 Plugin 코드를 유지관리하거나 패치하지
않습니다.

`openclaw/clawhub`의 GitHub Security Advisories는 ClawHub 자체의 취약점을
위한 것입니다. 예로는 웹사이트, API, CLI, 레지스트리, 인증, 스캔, 중재 또는
다운로드/설치 신뢰 경계의 버그가 있습니다. 타사 Skills 또는 Plugin의 취약점에
ClawHub 권고를 사용하지 마세요.

좋은 신고는 구체적이고 실행 가능합니다. 신고 남용 자체도 계정 조치로 이어질
수 있습니다.

## 조직 및 네임스페이스 클레임

조직, 브랜드, 패키지 범위, 소유자 핸들 또는 네임스페이스 소유권 분쟁은
제품 내 신고 흐름이나 계정 이의 제기 양식이 아니라
[조직 및 네임스페이스 클레임](/ko/clawhub/namespace-claims) 절차를 사용해야
합니다.

네임스페이스를 예약, 이전, 이름 변경, 숨김, 격리, 별칭 지정 또는 기타 검토해야
한다는 민감하지 않은 증거를 ClawHub 직원이 검토해야 할 때 이 절차를 사용하세요.
공개 이슈에 비밀 정보, 비공개 문서, 비공개 법률 파일, 개인 신원 문서, API 토큰
또는 DNS 챌린지 토큰을 포함하지 마세요.

## 중재 보류

일부 심각한 발견 사항이나 정책 문제는 게시자 또는 등록 항목을 중재 보류 상태로
둘 수 있습니다. 이 경우 영향을 받는 콘텐츠가 공개 검색에서 숨겨지거나, 문제가
검토될 때까지 이후 게시가 숨김 상태로 시작될 수 있습니다.

중재 보류는 ClawHub가 고위험 사례를 해결하는 동안 사용자를 보호하기 위한
것입니다. 오탐이 확인되면 해제될 수도 있습니다.

## 숨겨지거나 차단된 등록 항목

등록 항목은 보류, 숨김, 격리, 취소되거나 기타 이유로 공개 설치 영역에서 사용할
수 없을 수 있습니다.

이러한 상태 중 하나가 보이면, 소유자가 문제를 해결하거나 중재가 복원하기 전까지
해당 릴리스를 설치하지 마세요.

소유자는 자신이 소유한 보류 또는 숨김 등록 항목의 진단 정보를 계속 볼 수
있습니다. 이러한 진단 정보는 무슨 일이 있었는지와 등록 항목이 공개 영역으로
돌아가기 전에 무엇을 변경해야 하는지 설명하는 데 도움이 됩니다.

## 차단 및 계정 상태

ClawHub 정책을 위반한 계정은 게시 접근 권한을 잃을 수 있습니다. 심각한 남용은
계정 차단, 토큰 취소, 콘텐츠 숨김 또는 등록 항목 제거로 이어질 수 있습니다.
게시자 남용 압력 신호는 매일 확인됩니다. ClawHub의 잠재적 차단 임계값에 도달한
신호는 자동 경고를 트리거할 수 있습니다. 경고 기한 이후 다음 적격 스캔에서도
게시자가 잠재적 차단 임계값에 계속 해당하면, ClawHub가 계정 조치를 자동으로
적용할 수 있습니다. 신뢰도가 낮거나 시간 범위가 제한된 검토 신호는 자동 집행에서
제외됩니다.

삭제, 차단 또는 비활성화된 계정은 ClawHub API 토큰을 사용할 수 없습니다. 계정
조치 후 CLI 인증이 실패하기 시작하면 웹 UI에 로그인해 계정 상태를 검토하세요.
차단 또는 비활성화된 계정으로 인해 로그인이나 정상 CLI 접근이 막힌 경우,
복구 검토를 위해 [ClawHub 이의 제기 양식](https://appeals.openclaw.ai/)을
사용하세요.

스캐너가 트리거한 이메일에서 Skills 또는 Plugin 버전을 악성으로 지정한 경우,
차단된 제출 버전의 저장된 스캔 결과를 다운로드하세요:
`clawhub scan download <slug> --version <version>`. Plugin의 경우
`--kind plugin`을 추가하세요. 스캔 출력을 검토하고, 등록 항목을 수정하고,
버전 번호를 올린 다음, 수정된 버전을 업로드하세요.

## 게시자 지침

오탐을 줄이고 사용자 신뢰를 높이려면:

- 이름, 요약, 태그, 변경 로그를 정확하게 유지하세요
- 필요한 환경 변수와 권한을 선언하세요
- 난독화된 설치 명령을 피하세요
- 가능하면 소스에 연결하세요
- Plugin 게시 전에 드라이 런을 사용하세요
- 사용자나 중재자가 릴리스 동작에 대해 질문하면 명확하게 응답하세요
