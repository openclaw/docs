# 보안 감사

ClawHub 보안 감사는 Skills 또는 Plugin을 설치해도 충분히 안전한지 판단하는 데 도움이 됩니다. 감사는 릴리스가 수행하는 작업, 요구하는 권한, 그리고 파일, 계정, 자격 증명, 코드 또는 외부 서비스에 접근하기 전에 특별히 주의해야 할 사항이 있는지를 보여 줍니다.

감사는 강력한 안전성 지표이지만, 릴리스에 위험이 전혀 없음을 보장하지는 않습니다. 민감한 접근 권한을 부여하기 전에 항상 신중하게 판단하세요.

[보안](/clawhub/security), [허용되는 사용](/clawhub/acceptable-usage), [검토 및 계정 안전](/clawhub/moderation)도 참조하세요.

## 설치 전 확인 사항

설치하기 전에 다음을 검토하세요.

- 전반적인 감사 상태
- 위험 수준
- 나열된 모든 발견 사항
- 필요한 자격 증명, 권한 또는 환경 변수
- 소유자, 출처, 버전, 변경 로그, 다운로드 수, 별 수 및 기타 신뢰 지표

이해하고 신뢰하는 콘텐츠만 설치하세요.

## 감사 상태

감사 상태는 감사 결과에 어떻게 대응해야 하는지를 나타냅니다.

| 상태        | 의미                                                                  |
| ----------- | --------------------------------------------------------------------- |
| `Pass`      | 낮은 위험을 초과하는 명확한 문제가 발견되지 않았습니다.              |
| `Review`    | 설치 전에 발견 사항을 읽어 보세요. 릴리스는 여전히 정당할 수 있습니다. |
| `Warn`      | 각별히 주의하세요. ClawHub가 영향이 큰 우려 사항이나 경고 신호를 발견했습니다. |
| `Malicious` | 설치하지 마세요.                                                      |
| `Pending`   | 감사가 아직 완료되지 않았습니다.                                     |
| `Error`     | 감사를 완료할 수 없었습니다.                                         |

`Pass`는 안심할 만한 결과이지만, 사용자의 자체 판단을 대신하지는 않습니다. 콘텐츠를 게시하거나, 데이터를 편집하거나, 명령을 실행하거나, 파일을 읽거나, 프로덕션 시스템에 접근할 수 있는 도구에서는 특히 중요합니다.

## 위험 수준

위험 수준은 영향 범위, 즉 릴리스를 의도대로 사용할 때 얼마나 강력한 권한을 가지는 것으로 보이는지를 설명합니다.

| 위험 수준 | 의미                                                                        |
| --------- | --------------------------------------------------------------------------- |
| `Low`     | 민감한 권한이나 사용자에게 미치는 영향이 거의 발견되지 않았습니다.         |
| `Medium`  | 릴리스에 계정 접근이나 데이터 변경 같은 상당한 권한이 있습니다.            |
| `High`    | 릴리스에 영향이 큰 권한, 심각한 발견 사항 또는 악성 신호가 있습니다.       |

위험 수준과 감사 상태는 서로 다른 질문에 답합니다.

- 위험 수준의 질문: "여기에 얼마나 강력한 권한이 있는가?"
- 감사 상태의 질문: "이 결과를 보고 어떻게 해야 하는가?"

예를 들어 게시 Skills은 `Medium` 위험과 함께 `Review`로 표시될 수 있습니다. 이것이 악성이라는 의미는 아닙니다. 해당 Skills이 목적에는 부합하는 것으로 보이지만, 상당한 계정 권한으로 작업할 수 있다는 의미입니다.

## 발견 사항

발견 사항은 감사 결과가 표시된 이유를 설명합니다. 각 발견 사항에는 일반적으로 다음이 포함됩니다.

- 발견 사항의 의미
- 표시된 이유
- 관련 Skills 또는 Plugin 콘텐츠
- 권장 사항

발견 사항에는 `Info`, `Low`, `Medium`, `High` 또는 `Critical` 레이블이 지정될 수 있습니다. 심각도가 높은 발견 사항일수록 위험 수준과 감사 상태에 더 큰 영향을 미칩니다.

신뢰도가 낮은 발견 사항은 페이지가 유용한 증거에 집중할 수 있도록 공개 감사 요약에서 숨겨집니다.

## ClawHub의 검사 항목

ClawHub는 다음을 포함하여 제출된 릴리스 산출물을 감사합니다.

- Skills 지침 또는 Plugin 메타데이터
- 선언된 환경 변수 및 권한
- 설치 지침 및 패키지 메타데이터
- 포함된 파일 및 파일 매니페스트
- 호환성 및 기능 메타데이터

핵심 질문은 일관성입니다. 이름, 요약, 메타데이터, 요청한 권한 및 실제 콘텐츠가 사용자가 합리적으로 예상할 내용과 일치하는지를 확인합니다.

강력한 동작이 자동으로 나쁜 것은 아닙니다. 유용한 도구 중 상당수에는 자격 증명, 로컬 명령, 제공자 API 또는 패키지 설치가 필요합니다. 감사에서는 이러한 권한이 예상 가능하고, 공개되어 있으며, 목적에 비례하는지 확인합니다.

산출물 페이지는 다음 경로의 전체 감사로 연결됩니다.

```text
/<owner>/skills/<slug>/security-audit
```

감사 페이지는 다음을 결합합니다.

1. SkillSpector
2. VirusTotal
3. 위험 분석

## VirusTotal

ClawHub는 감사 체계에서 VirusTotal을 악성 코드 원격 측정 정보로 사용합니다. VirusTotal은 파일 평판 및 악성 코드 검사에 관한 신뢰받는 업계 표준이며, 양사의 협력을 통해 ClawHub는 Skills 및 Plugin 검토에 더 폭넓은 보안 인텔리전스를 추가할 수 있습니다.

VirusTotal은 알려진 악성 산출물, 검사 엔진 탐지 결과 및 ClawHub의 에이전트 인식 검토를 보완하는 평판 신호에 특히 유용합니다. 공급업체 엔진 수를 확인할 수 있는 경우 감사는 다음과 같이 이를 이해하기 쉬운 문장으로 요약합니다.

```text
62/62개 공급업체가 이 Skills을 안전한 것으로 판정했습니다.
```

또는:

```text
2/64개 공급업체가 이 Skills을 악성으로, 1/64개 공급업체가 의심스러운 것으로, 61/64개 공급업체가 안전한 것으로 판정했습니다.
```

ClawHub에 요약할 공급업체 수 원격 측정 정보가 없는 경우 감사에는 다음과 같이 표시됩니다.

```text
VirusTotal 발견 사항 없음
```

VirusTotal은 원격 측정 정보로 사용됩니다. ClawHub 자체의 산출물 인식 위험 분석을 대신하지 않습니다.

## 위험 분석

위험 분석은 내부적으로 ClawHub 자체 보안 감사 시스템인 ClawScan을 기반으로 합니다. ClawScan은 각 릴리스를 에이전트 대상 산출물로 검토합니다. 검토 대상에는 지침, 메타데이터, 선언된 권한, 파일, 기능 신호, 정적 검사 신호, SkillSpector 발견 사항, VirusTotal 원격 측정 정보 및 게시자가 제공한 맥락이 포함됩니다. 정적 검사 신호는 이 검토를 위한 내부 맥락이며, 독립적인 공개 감사 섹션이나 설치 차단 판정이 아닙니다.

위험 분석은 프롬프트 인젝션, 도구 오용, 자격 증명 노출, 안전하지 않은 실행, 메모리 또는 컨텍스트 오염, 과도한 자율성 등의 위험을 평가하는 관점으로 [OWASP 에이전트형 Skills 상위 10개 위험](https://owasp.org/www-project-agentic-skills-top-10/)을 사용합니다.

ClawScan은 위험해 보이는 기능을 자동으로 악성으로 간주하지 않습니다. 해당 기능이 공개되어 있고 목적에 부합하며 릴리스에 명시된 사용 사례로 뒷받침되는지를 확인합니다.
