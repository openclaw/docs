# 보안 감사

ClawHub 보안 감사는 Skills 또는 Plugin을 설치해도 충분히 안전한지 판단하는 데 도움이 됩니다. 감사는 릴리스가 무엇을 하는지, 어떤 권한을 요청하는지, 파일, 계정, 자격 증명, 코드 또는 외부 서비스에 접근하기 전에 추가로 주의해야 할 사항이 있는지를 보여 줍니다.

감사는 강력한 안전 신호이지만, 릴리스에 위험이 전혀 없다는 보장은 아닙니다. 민감한 접근 권한을 부여하기 전에는 항상 신중하게 판단하세요.

[보안](/clawhub/security), [허용되는 사용](/clawhub/acceptable-usage), [모더레이션 및 계정 안전](/clawhub/moderation)도 참고하세요.

## 설치 전에 확인할 사항

설치하기 전에 다음을 검토하세요.

- 전체 감사 상태
- 위험 수준
- 나열된 발견 사항
- 필요한 자격 증명, 권한 또는 환경 변수
- 소유자, 소스, 버전, 변경 로그, 다운로드 수, 별 수 및 기타 신뢰 신호

이해하고 신뢰할 수 있는 콘텐츠만 설치하세요.

## 감사 상태

감사 상태는 감사 결과에 어떻게 대응해야 하는지 알려 줍니다.

| 상태        | 의미                                                                      |
| ----------- | ------------------------------------------------------------------------- |
| `Pass`      | 낮은 위험을 초과하는 눈에 띄는 문제가 발견되지 않았습니다.               |
| `Review`    | 설치하기 전에 발견 사항을 읽어 보세요. 릴리스가 여전히 정상일 수 있습니다. |
| `Warn`      | 특히 주의하세요. ClawHub가 영향이 큰 우려 사항 또는 경고 신호를 발견했습니다. |
| `Malicious` | 설치하지 마세요.                                                          |
| `Pending`   | 감사가 아직 완료되지 않았습니다.                                          |
| `Error`     | 감사를 완료할 수 없었습니다.                                              |

`Pass`는 안심할 수 있는 신호이지만, 사용자의 판단을 대체하지는 않습니다. 이는 콘텐츠를 게시하거나, 데이터를 편집하거나, 명령을 실행하거나, 파일을 읽거나, 프로덕션 시스템에 접근할 수 있는 도구에서 가장 중요합니다.

## 위험 수준

위험 수준은 영향 범위를 설명합니다. 즉, 의도한 대로 사용할 때 릴리스가 어느 정도의 권한을 가진 것으로 보이는지를 나타냅니다.

| 위험 수준 | 의미                                                                           |
| ---------- | ------------------------------------------------------------------------------ |
| `Low`      | 민감한 권한이나 사용자 영향이 거의 발견되지 않았습니다.                       |
| `Medium`   | 릴리스가 계정 접근 또는 데이터 변경과 같은 의미 있는 권한을 가집니다.         |
| `High`     | 릴리스가 영향이 큰 권한, 심각한 발견 사항 또는 악성 신호를 가집니다.          |

위험 수준과 감사 상태는 서로 다른 질문에 답합니다.

- 위험 수준은 묻습니다. "여기에 얼마나 큰 권한이 있는가?"
- 감사 상태는 묻습니다. "이 결과를 어떻게 처리해야 하는가?"

예를 들어, 게시용 Skills는 `Medium` 위험과 함께 `Review`로 표시될 수 있습니다. 이것이 악성이라는 뜻은 아닙니다. 해당 Skills가 목적에 부합하는 것으로 보이지만, 의미 있는 계정 권한으로 작업할 수 있다는 뜻입니다.

## 발견 사항

발견 사항은 감사 결과가 표시된 이유를 설명합니다. 각 발견 사항에는 보통 다음이 포함됩니다.

- 의미
- 플래그된 이유
- 관련 Skills 또는 Plugin 콘텐츠
- 권장 사항

발견 사항에는 `Info`, `Low`, `Medium`, `High`, `Critical` 레이블이 붙을 수 있습니다. 심각도가 높은 발견 사항은 위험 수준과 감사 상태에 더 강하게 반영됩니다.

신뢰도가 낮은 발견 사항은 공개 감사 요약에서 숨겨 페이지가 유용한 증거에 집중하도록 합니다.

## ClawHub가 확인하는 항목

ClawHub는 제출된 릴리스 산출물을 감사하며, 여기에는 다음이 포함됩니다.

- Skills 지침 또는 Plugin 메타데이터
- 선언된 환경 변수와 권한
- 설치 지침과 패키지 메타데이터
- 포함된 파일과 파일 매니페스트
- 호환성 및 기능 메타데이터

핵심 질문은 일관성입니다. 이름, 요약, 메타데이터, 요청된 권한, 실제 콘텐츠가 사용자가 합리적으로 기대할 내용과 일치하는가?

강력한 동작이 자동으로 나쁜 것은 아닙니다. 유용한 많은 도구에는 자격 증명, 로컬 명령, 제공자 API 또는 패키지 설치가 필요합니다. 감사는 그 권한이 예상 가능한지, 공개되었는지, 목적에 비례하는지를 확인합니다.

산출물 페이지는 전체 감사로 연결됩니다.

```text
/<owner>/skills/<slug>/security-audit
```

감사 페이지는 다음을 결합합니다.

1. SkillSpector
2. VirusTotal
3. 위험 분석

## VirusTotal

ClawHub는 감사 스택에서 VirusTotal을 멀웨어 텔레메트리로 사용합니다. VirusTotal은 파일 평판과 멀웨어 스캔에 대한 신뢰받는 업계 표준이며, 이 파트너십을 통해 ClawHub는 Skills 및 Plugin 검토에 더 폭넓은 보안 인텔리전스를 추가할 수 있습니다.

VirusTotal은 ClawHub의 에이전트 인식 검토를 보완하는 알려진 악성 산출물, 엔진 탐지, 평판 신호에 특히 유용합니다. 벤더 엔진 수를 사용할 수 있는 경우, 감사는 이를 다음과 같이 쉬운 표현으로 요약합니다.

```text
62/62 vendors flagged this skill as clean.
```

또는:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

ClawHub에 요약할 벤더 수 텔레메트리가 없으면 감사는 다음과 같이 표시합니다.

```text
No VirusTotal findings
```

VirusTotal은 여전히 텔레메트리입니다. ClawHub 자체의 산출물 인식 위험 분석을 대체하지 않습니다.

## 위험 분석

위험 분석은 내부적으로 ClawHub 자체 보안 감사 시스템인 ClawScan으로 구동됩니다. 이는 각 릴리스를 에이전트 대상 산출물로 검토합니다. 여기에는 지침, 메타데이터, 선언된 권한, 파일, 기능 신호, 정적 스캔 신호, SkillSpector 발견 사항, VirusTotal 텔레메트리, 게시자가 제공한 맥락이 포함됩니다. 정적 스캔 신호는 이 검토를 위한 내부 맥락이며, 독립적인 공개 감사 섹션이나 설치 차단 판정이 아닙니다.

위험 분석은 프롬프트 인젝션, 도구 오용, 자격 증명 노출, 안전하지 않은 실행, 메모리 또는 컨텍스트 오염, 과도한 대리 권한과 같은 위험을 바라보는 관점으로 [OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)을 사용합니다.

ClawScan은 무섭게 보이는 기능을 자동으로 악성으로 간주하지 않습니다. 해당 기능이 공개되었는지, 목적에 부합하는지, 릴리스에 명시된 사용 사례로 뒷받침되는지를 묻습니다.
