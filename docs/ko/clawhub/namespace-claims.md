# 조직 및 네임스페이스 클레임

ClawHub는 소유자 핸들, 조직 핸들, Skills 슬러그, Plugin 패키지 이름, 패키지
스코프를 공개 네임스페이스로 사용합니다. 어떤 네임스페이스가 실제 프로젝트,
브랜드, 패키지 생태계 또는 조직에 속하는 것처럼 보이지만 ClawHub에서 이미
클레임되었거나, 예약되었거나, 오해를 일으키거나, 분쟁 중이라면, 스태프에게
[조직 / 네임스페이스 클레임 이슈 양식](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)으로 검토를 요청하세요.

공개적이고 민감하지 않은 소유권 검토에는 이 경로를 사용하세요. 네임스페이스
클레임에는 제품 내 신고나 계정 이의제기 양식을 사용하지 마세요.

## 클레임을 열어야 할 때

실제 소유권 때문에 네임스페이스를 예약, 이전, 이름 변경, 숨김, 격리, 별칭 지정
또는 기타 방식으로 변경해야 하는지 ClawHub 스태프가 검토해야 한다고 생각하면
네임스페이스 클레임을 여세요.

예시는 다음과 같습니다.

- 사용자의 GitHub 조직, 프로젝트, 회사 또는 커뮤니티와 일치하는 조직 핸들
- 일치하는 ClawHub 소유자 아래에서만 게시되어야 하는 `@example-org/*` 같은 패키지 스코프
- 프로젝트를 사칭하는 것처럼 보이는 Skills 슬러그 또는 Plugin 패키지 이름
- 브랜드, 상표, 프로젝트 이름 변경 또는 패키지 이력 분쟁
- 정당한 네임스페이스 소유자를 막고 있는 삭제되었거나, 비활성 상태이거나, 연락할 수 없는 소유자

목록이 소유권 분쟁을 넘어 안전하지 않거나, 악의적이거나, 오해를 일으킨다면
관련 조정 또는 보안 지침도 따르세요. 네임스페이스 클레임 양식은 소유권
검토용이며, 긴급 취약점 공개용이 아닙니다.

## 제출하기 전에

먼저 네임스페이스와 일치하는 소유자로 게시하고 있는지 확인하세요. Plugin
패키지의 경우 `@example-org/example-plugin` 같은 스코프가 있는 이름은 일치하는
`example-org` 소유자로 게시되어야 합니다.

현재 소유자를 관리할 수 있다면, 영향을 받는 리소스를 게시, 이름 변경, 이전,
숨김 또는 삭제하여 네임스페이스를 직접 수정하세요. 현재 소유자를 관리할 수
없거나 스태프가 분쟁을 해결해야 할 때 클레임을 사용하세요.

## 포함할 증거

공개적이고 민감하지 않은 증거를 사용하세요. 도움이 되는 증거는 다음과 같습니다.

- GitHub 조직, 저장소, 릴리스 또는 유지관리자 이력
- 네임스페이스를 명시하는 공식 프로젝트 문서
- 도메인 또는 공식 이메일 도메인 증거
- npm, PyPI, crates.io 또는 기타 패키지 레지스트리 스코프 제어
- 공개적으로 논의해도 안전한 상표, 브랜드 또는 프로젝트 소유권 증거
- 소스 저장소 이력, 패키지 이력 또는 공개 이름 변경 공지
- 분쟁 중인 ClawHub 소유자, Skills, Plugin, 패키지 또는 이슈 링크

각 링크가 무엇을 증명하는지 설명하세요. 스태프는 비공개 자격 증명이나 비밀
정보 없이도 관계를 이해할 수 있어야 합니다.

## 포함하지 말아야 할 것

공개 GitHub 이슈에 비밀 정보나 비공개 증거를 넣지 마세요. 다음을 포함하지
마세요.

- API 토큰, 서명 키 또는 자격 증명
- DNS 챌린지 토큰
- 비공개 법적 파일 또는 계약서
- 개인 신분 문서
- 비공개 이메일, 비공개 보안 보고서 또는 기밀 고객 데이터

클레임 양식은 민감한 증거에 비공개 스태프 채널이 필요한지 묻습니다. 민감한
자료를 공개적으로 게시하는 대신 해당 옵션을 사용하세요.

## 가능한 결과

증거와 위험에 따라 ClawHub 스태프는 네임스페이스를 예약하거나, 소유권을
이전하거나, 리소스 이름을 변경하거나, 기존 목록을 숨기거나 격리하거나, 별칭
또는 리디렉션을 추가하거나, 추가 증거를 요청하거나, 요청을 거절할 수 있습니다.

네임스페이스 검토가 일치하는 모든 이름의 이전을 보장하지는 않습니다. 스태프는
공개 증거, 기존 사용 현황, 보안 위험, 사용자 영향을 함께 고려합니다.
