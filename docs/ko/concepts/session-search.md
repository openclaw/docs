# 세션 검색

`sessions_search`는 사용자의 과거 세션에서 사용자와 어시스턴트의 텍스트를 검색합니다. 각 결과에는 `sessionKey`, 타임스탬프, 역할, 짧은 일치 발췌문이 포함됩니다. 주변 대화가 필요하면 반환된 `sessionKey`를 `sessions_history`에 전달하십시오.

## 가시성 및 출력

검색에는 `sessions_history`와 동일한 세션 가시성 규칙이 적용됩니다. 호출자에게 표시되는 세션 트리 외부의 결과는 결과 제한을 적용하기 전에 제거됩니다. 생성된 세션의 가시성이 활성화된 경우 샌드박스 에이전트는 자신이 생성한 세션으로 계속 제한됩니다.

발췌문은 모델에 반환되기 전에 민감 정보가 삭제됩니다. 결과는 개수, 발췌문 길이, 전체 응답 크기로도 제한됩니다.

## 인덱스 수명 주기

OpenClaw는 각 에이전트의 SQLite 데이터베이스에서 트랜스크립트 행과 함께 전체 텍스트 인덱스를 저장합니다. 새 사용자 및 어시스턴트 메시지는 이를 저장하는 동일한 트랜잭션에서 인덱싱되므로 인덱스가 실시간 대화보다 뒤처지지 않습니다. 도구 결과, 추론 블록, 이미지는 제외됩니다. 트랜스크립트의 활성 브랜치만 검색할 수 있습니다.

인덱스보다 오래된 트랜스크립트(예: `openclaw doctor`로 가져온 세션)와 활성 브랜치가 되감긴 세션은 다음 검색 시 시작되는 백그라운드 조정 작업을 통해 다시 인덱싱됩니다. 따라서 `indexing: true`가 포함된 응답은 불완전할 수 있습니다. 인덱싱이 완료된 후 다시 시도하십시오. 세션을 삭제하면 동일한 트랜잭션에서 해당 인덱스 항목도 제거됩니다.

현재 검색에는 분음 부호 제거 기능이 있는 SQLite의 유니코드 단어 토크나이저를 사용합니다. CJK 하위 문자열 일치를 위한 트라이그램 토큰화는 향후 개선 사항입니다.

## 세션 검색과 메모리 검색 비교

원시 세션 트랜스크립트에서 정확한 단어나 구문을 찾으려면 `sessions_search`를 사용하십시오. 영구 메모리 파일과 의미 기반 회상에는 [`memory_search`](/ko/concepts/memory-search)를 사용하십시오. 실험적인 세션 메모리 코퍼스는 이러한 정확한 트랜스크립트 검색을 의미적으로 보완합니다.
