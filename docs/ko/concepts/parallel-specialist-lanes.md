병렬 전문 레인을 사용하면 하나의 Gateway가 사용자 경험을 빠르게 유지하면서 서로 다른 채팅이나 대화방을 서로 다른 에이전트로 라우팅할 수 있습니다. 병렬 처리를 단순히 "더 많은 에이전트"가 아니라 희소 자원 설계 문제로 다루세요.

## 기본 원칙

전문 레인은 실제 병목 지점의 경합을 줄일 때만 처리량을 개선합니다.

- **세션 잠금**: 특정 세션은 한 번에 하나의 실행만 변경해야 합니다.
- **전역 모델 용량**: 사용자에게 표시되는 모든 채팅 실행은 여전히 제공자 한도를 공유합니다.
- **도구 용량**: 셸, 브라우저, 네트워크 및 저장소 작업은 모델 턴 자체보다 느릴 수 있습니다.
- **컨텍스트 예산**: 긴 대화 기록은 이후의 모든 턴을 더 느리고 산만하게 만듭니다.
- **소유권 모호성**: 동일한 작업을 수행하는 중복 에이전트는 용량을 낭비합니다.

OpenClaw는 이미 세션별로 실행을 직렬화하고 [명령 큐](/ko/concepts/queue)를 통해 전역 병렬 처리에 상한을 둡니다. 전문 레인은 그 위에 정책을 추가합니다. 즉, 어떤 에이전트가 어떤 작업을 소유하는지, 어떤 작업을 채팅에 유지하는지, 어떤 작업을 백그라운드 작업으로 전환하는지를 정합니다.

## 권장 도입 절차

### 1단계: 레인 계약 + 무거운 작업의 백그라운드 처리

각 레인의 작업 공간과 시스템 프롬프트에 문서화된 계약을 제공하세요.

- **목적**: 이 레인이 소유하는 작업입니다.
- **비목표**: 이 레인이 직접 시도하지 않고 인계해야 하는 작업입니다.
- **채팅 예산**: 빠른 답변은 채팅에서 처리하고, 긴 작업은 간단히 접수한 후 백그라운드 하위 에이전트나 작업으로 실행합니다.
- **인계 규칙**: 다른 레인이 작업을 소유하는 경우 작업을 보낼 위치를 알리고 간결한 인계 요약을 제공합니다.
- **도구 위험 규칙**: 작업을 수행할 수 있는 가장 작은 도구 범위를 우선 사용합니다.

이 단계는 비용이 가장 적게 들면서 대부분의 정체를 해결합니다. 하나의 코딩 작업 때문에 연구 레인이 더 이상 극도로 느려지지 않으며, 각 채팅의 컨텍스트도 깔끔하게 유지됩니다.

### 2단계: 우선순위 및 동시 실행 제어

각 레인의 비즈니스 가치에 맞춰 큐와 모델 용량을 조정하세요.

```json5
{
  agents: {
    defaults: {
      maxConcurrent: 4,
      subagents: { maxConcurrent: 8, delegationMode: "prefer" },
    },
  },
  messages: {
    queue: {
      mode: "collect",
      debounceMs: 1000,
      cap: 20,
      drop: "summarize",
    },
  },
}
```

직접/개인 채팅과 프로덕션 운영 에이전트는 우선순위가 높은 작업에 사용하세요. 시스템이 사용 중일 때는 연구, 초안 작성 및 일괄 코딩을 백그라운드 작업으로 전환하세요.

### 3단계: 코디네이터/트래픽 컨트롤러

여러 레인이 활성화되면 소규모 코디네이터 패턴을 추가하세요.

- 활성 레인 작업과 소유자를 추적합니다.
- 그룹 간 중복 요청을 감지합니다.
- 레인 간에 인계 요약을 라우팅합니다.
- 차단 요인, 완료된 결과 및 사람이 결정해야 하는 사항만 표시합니다.

이 단계부터 시작하지 마세요. 레인 계약이 없는 코디네이터는 혼란을 조정할 뿐입니다.

## 최소 레인 계약 템플릿

```md
# 레인 계약

## 소유하는 작업

- <이 레인이 담당하는 작업>

## 소유하지 않는 작업

- <인계할 작업>

## 채팅 예산

- 간단한 질문에는 직접 답변합니다.
- 여러 단계가 필요하거나 느리거나 도구 사용이 많은 작업의 경우: 간단히 접수하고,
  작업을 하위 에이전트에서 실행하거나 백그라운드로 전환한 후 완료되면 결과를 반환합니다.

## 인계

다른 레인이 요청을 소유하는 경우 다음 항목을 포함해 응답합니다.

- 대상 레인
- 목표
- 관련 컨텍스트
- 정확한 다음 작업

## 도구 사용 방침

작업을 완료할 수 있는 가장 작은 도구 범위를 사용합니다. 이 레인이 명시적으로
소유하는 작업이 아니라면 광범위한 셸 또는 네트워크 작업을 피합니다.
```
