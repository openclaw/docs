채널 및 세션 진단.

```bash
openclaw status
openclaw status --all
openclaw status --deep
openclaw status --usage
```

| 플래그                  | 설명                                                                                                       |
| ----------------------- | ---------------------------------------------------------------------------------------------------------- |
| `--all`                 | 전체 진단(읽기 전용, 붙여넣기 가능). 보안 감사, Plugin 호환성 및 메모리 벡터 검사를 포함합니다.            |
| `--deep`                | 실시간 검사(WhatsApp Web + Telegram + Discord + Slack + Signal)를 실행합니다. 보안 감사도 활성화합니다.    |
| `--usage`               | 정규화된 제공자 사용량 기간을 `X% 남음` 형식으로 출력합니다.                                               |
| `--json`                | 기계 판독 가능 출력을 생성합니다.                                                                          |
| `--verbose` / `--debug` | 보고서에 앞서 원시 Gateway 대상 결정 결과도 출력합니다.                                                    |

일반 `openclaw status`는 빠른 읽기 전용 경로를 유지하며, 메모리 검사를
건너뛸 때 메모리를 사용할 수 없음으로 표시하지 않고 `검사하지 않음`으로
표시합니다. 무거운 보안 감사, Plugin 호환성 및 메모리 벡터 검사는
`openclaw status --all`, `openclaw status --deep`, `openclaw security audit`,
`openclaw memory status --deep`에서 수행합니다.

## 세션 및 모델 결정

- 세션 상태 출력은 `실행:`과 `런타임:`을 구분합니다. `실행`은 샌드박스
  경로(`direct`, `docker/*`)이며, `런타임`은 세션에서 `OpenClaw 기본값`,
  `OpenAI Codex`, CLI 백엔드 또는 `codex (acp/acpx)` 같은 ACP 백엔드 중
  무엇을 사용하는지 알려 줍니다. 제공자/모델/런타임의 차이는
  [에이전트 런타임](/ko/concepts/agent-runtimes)을 참조하세요.
- 현재 세션 스냅샷에 정보가 부족한 경우 `/status`는 가장 최근의 대화 기록
  사용량 로그에서 토큰 및 캐시 카운터를 보충할 수 있습니다. 0이 아닌 기존
  실시간 값은 대화 기록의 대체 값보다 계속 우선합니다.
- 실시간 세션 항목에 활성 런타임 모델 레이블이 누락된 경우 대화 기록의
  대체 데이터를 통해 이를 복구할 수도 있습니다. 해당 대화 기록 모델이
  선택된 모델과 다르면 상태는 선택된 모델이 아닌 복구된 런타임 모델을
  기준으로 컨텍스트 창을 결정합니다.
- 프롬프트 크기를 계산할 때 세션 메타데이터가 누락되었거나 더 작은 경우,
  대화 기록의 대체 데이터는 프롬프트 중심의 더 큰 합계를 우선하므로 사용자
  지정 제공자 세션의 토큰 표시가 `0`으로 축소되지 않습니다.
- 세션이 구성된 기본 모델과 다른 모델로 고정된 경우 상태는 두 값과 이유
  (`세션 재정의`) 및 안내 `/model default`를 함께 출력합니다. 구성된 기본
  모델은 신규 세션이나 고정되지 않은 세션에 적용되며, 기존 고정 세션은
  해제할 때까지 해당 세션의 선택을 유지합니다.
- 여러 에이전트가 구성된 경우 출력에 에이전트별 세션 저장소가 포함됩니다.

## 사용량 및 할당량

- `--usage`는 정규화된 제공자 사용량 기간을 `X% 남음` 형식으로 출력합니다.
- MiniMax의 원시 `usage_percent` / `usagePercent` 필드는 남은 할당량이므로,
  OpenClaw는 표시하기 전에 이를 반전합니다. 카운트 기반 필드가 있으면
  우선합니다. `model_remains` 응답은 채팅 모델 항목을 우선하며, 필요한 경우
  타임스탬프에서 기간 레이블을 도출하고 요금제 레이블에 모델 이름을
  포함합니다.
- 모델 가격 새로 고침 실패는 선택적 가격 경고로 표시됩니다. 이는 Gateway나
  채널이 비정상이라는 의미가 아닙니다.

## 개요 및 업데이트 상태

- 개요에는 가능한 경우 Gateway와 Node 호스트 서비스의 설치/런타임 상태뿐
  아니라 간결한 Gateway 프로세스 가동 시간과 호스트 시스템 가동 시간이
  포함됩니다.
- 개요에는 업데이트 채널과 git SHA(소스 체크아웃의 경우)가 포함됩니다.
- 업데이트 정보는 개요에 표시됩니다. 업데이트가 있으면 상태는
  `openclaw update`를 실행하라는 안내를 출력합니다
  ([업데이트](/ko/install/updating) 참조).

## 비밀 정보

- 읽기 전용 상태 화면(`status`, `status --json`, `status --all`)은 가능한
  경우 대상 구성 경로에서 지원되는 SecretRef를 확인합니다.
- 지원되는 채널 SecretRef가 구성되어 있지만 현재 명령 경로에서 사용할 수
  없는 경우 상태는 읽기 전용을 유지하며 중단되는 대신 성능 저하 출력을
  보고합니다. 사람이 읽는 출력에는 "이 명령 경로에서 구성된 토큰을 사용할
  수 없음"과 같은 경고가 표시되며, JSON 출력에는 `secretDiagnostics`가
  포함됩니다.
- 명령 로컬 SecretRef 확인에 성공하면 상태는 확인된 스냅샷을 우선하며 최종
  출력에서 일시적인 "비밀 정보를 사용할 수 없음" 채널 표시를 제거합니다.
- `status --all`에는 비밀 정보 개요 행과 비밀 정보 진단을 요약하는 진단
  섹션이 포함됩니다. 가독성을 위해 일부 내용은 잘리지만 보고서 생성은
  중단되지 않습니다.

## 메모리

`status --json --all`은 `plugins.slots.memory`에서 선택한 활성 메모리 Plugin
런타임의 메모리 세부 정보를 보고합니다. 사용자 지정 메모리 Plugin은 기본
제공 `agents.defaults.memorySearch.enabled`를 비활성화된 상태로 두면서도 자체
파일, 청크, 벡터 및 FTS 상태를 보고할 수 있습니다.
