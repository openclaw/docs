# `openclaw worker`

`openclaw worker`은 준비된 워커 환경 내에서 클라우드 워커
오케스트레이터가 실행하기 위한 제한된 런타임 진입점입니다. 수동 워커 등록을 위한
범용 명령이 아닙니다.

Gateway는 일치하는 OpenClaw 번들을 설치하고 호스트 키가 고정된
역방향 SSH 터널을 엽니다. 워커 런처는 준비된 할당으로 이 명령을 시작합니다.
이 명령은 터널을 통해 전달된 로컬 소켓으로 연결하고 전용 `worker` 역할로
승인됩니다.

## 실행 계약

이 명령은 표준 입력에서 크기가 제한된 JSON 실행 엔벌로프 하나만 읽습니다.
엔벌로프에는 로컬 소켓 위치, 발급된 워커 자격 증명, 번들 및 프로토콜 ID,
소유자 에포크, 그리고 할당된 단일 세션과 턴이 포함됩니다.
자격 증명은 명령줄 인수를 통해 절대 허용되지 않으며, 이 페이지에서는 의도적으로
자격 증명이나 수동으로 작성한 엔벌로프 예시를 제공하지 않습니다.

엔벌로프가 유효하지 않거나, 자격 증명이 거부되거나, 번들 또는 프로토콜 기능이
일치하지 않거나, 세션 및 소유자 에포크가 더 이상 최신 상태가 아니면 승인이
실패 시 차단됩니다. 운영자는 이 진입점을 직접 호출하지 말고 클라우드 워커
오케스트레이터를 통해 워커를 시작해야 합니다.

## 런타임 경계

프로세스는 제한된 백엔드로 일반적인 내장 에이전트 루프를 실행합니다.

- `read`, `write`, `edit`, `apply_patch`, `exec`, `process` 코딩 도구는
  워커 작업 공간에서 로컬로 실행됩니다.
- 모델 호출은 Gateway 추론 프록시를 사용합니다. 로컬 모델 인증 프로필은
  로드되지 않습니다.
- 트랜스크립트 쓰기는 Gateway 트랜스크립트 커밋 RPC를 사용합니다.
- 스트리밍 및 도구 수명 주기 업데이트는 Gateway 라이브 이벤트 RPC를 사용합니다.
- 할당된 세션과 턴만 허용됩니다.

워커 모드는 할당된 세션 도구 세트 외의 채널, Gateway HTTP 표면 또는 Plugin 자동 시작을
실행하지 않습니다. 일회용 상태 디렉터리를 사용하며 상시 유지되는 공급자 또는
포지 자격 증명이 없습니다.

이 모드에서는 워커 간 세션 디스패치가 노출되지 않습니다. 배치 및
디스패치는 계속 Gateway가 소유합니다. 운영자는 기존의 로컬
관리형 작업 트리 세션을 Gateway를 통해 디스패치할 수 있지만, 워커 프로세스는
자기 자신이나 다른 워커를 디스패치할 수 없습니다.

준비된 할당에는 트랜스크립트 컨텍스트, 승인된 기준 리프,
커밋 시퀀스 및 라이브 이벤트 커서가 포함됩니다. 터널이 다시 연결되면 프로세스는
동일한 자격 증명과 소유자 에포크로 다시 승인되고, 승인된
트랜스크립트 기준을 유지하며, 확인되지 않은 라이브 이벤트 후미를 재생하고,
진행 중인 추론 턴에 동일한 ID로 다시 연결됩니다. 스트리밍된 델타가 누락된 경우
최종 추론 메시지가 권위 있는 기준입니다. 대체하는 소유자 에포크는
프로세스를 차단하고 정상 종료를 유발합니다.

`stale-base-leaf` 트랜스크립트 거부가 발생하면 현재 실행이 즉시 중단됩니다. 워커
모드는 거부된 시퀀스를 다른 리프에 대해 재시도하지 않으므로
중복 커밋이 생성되지 않습니다. 해당 실행에서 아직 커밋되지 않은 메모리 내 후미는
모두 손실됩니다. 재실행은 마일스톤 3 배치 소유자의 책임이며, 이 소유자는
Gateway의 권위 있는 트랜스크립트와 커밋 원장에서 새로운 할당을
생성해야 합니다. 마찬가지로 Gateway 프로세스가 다시 시작되면 대기 중인
추론 턴이 공급자 오류로 종료됩니다. 활성 상태인 동일 프로세스의 추론 스트림에는
터널 또는 워커 WebSocket 재연결만 다시 연결할 수 있습니다.

폐쇄형 워커 RPC 표면은 [Gateway 프로토콜](/ko/gateway/protocol#worker-role-and-closed-protocol)을,
아키텍처 및 보안 모델은 [클라우드 워커 계획](/ko/plan/cloud-workers)을 참조하십시오.
