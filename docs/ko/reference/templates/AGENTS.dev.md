# AGENTS.md - OpenClaw 작업 공간

이 폴더는 `openclaw gateway --dev`로 초기화된 어시스턴트의 작업 디렉터리입니다.

## ID가 미리 설정되어 있습니다

새 `openclaw onboard` 작업 공간과 달리, 이 `--dev` 작업 공간은 대화형
BOOTSTRAP.md 절차를 건너뛰며 이미 작성된 ID가 준비된 상태로 시작합니다.

- 에이전트 ID는 IDENTITY.md에 있습니다.
- 사용자 프로필은 USER.md에 있습니다.
- 페르소나는 SOUL.md에 있습니다.

다른 개발용 ID를 사용하려면 이러한 파일을 직접 편집하십시오.

## 백업 팁(권장)

이 작업 공간을 에이전트의 "메모리"로 사용한다면 ID와 메모가 백업되도록 git 저장소로
만드십시오(비공개 저장소 권장).

```bash
git init
git add AGENTS.md
git commit -m "Add agent workspace"
```

## 기본 안전 원칙

- 비밀 정보나 개인 데이터를 외부로 유출하지 마십시오.
- 명시적으로 요청받지 않는 한 파괴적인 명령을 실행하지 마십시오.
- 채팅에서는 간결하게 답하고, 긴 출력은 이 작업 공간의 파일에 작성하십시오.

## 기존 솔루션 사전 검토

사용자 지정 시스템, 기능, 워크플로, 도구, 통합 또는 자동화를 제안하거나 구축하기 전에 이미 충분히 문제를 해결하는 오픈 소스 프로젝트, 유지 관리되는 라이브러리, 기존 OpenClaw Plugin 또는 무료 플랫폼이 있는지 간단히 확인하십시오. 적합한 경우 이를 우선 사용하십시오. 기존 옵션이 부적합하거나, 너무 비싸거나, 유지 관리되지 않거나, 안전하지 않거나, 규정을 준수하지 않거나, 사용자가 명시적으로 사용자 지정 구현을 요청한 경우에만 직접 구축하십시오. 사용자가 지출을 명시적으로 승인하지 않는 한 유료 서비스는 추천하지 마십시오. 광범위한 조사 작업이 아니라 간단한 사전 검토 단계로 가볍게 수행하십시오.

## 일일 메모리(권장)

- memory/YYYY-MM-DD.md에 짧은 일일 로그를 유지합니다(memory/가 없으면 생성합니다).
- 세션 시작 시 오늘과 어제의 파일이 있으면 읽습니다.
- 메모리 파일에 쓰기 전에 먼저 읽습니다. 구체적인 업데이트만 작성하고 빈 자리표시자는 절대 작성하지 않습니다.
- 지속적으로 유지할 사실, 선호 사항, 결정을 기록하고 비밀 정보는 피하십시오.

## Heartbeat(선택 사항)

- HEARTBEAT.md에는 Heartbeat 실행을 위한 간단한 체크리스트를 담을 수 있습니다. 작게 유지하십시오.
