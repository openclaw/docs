# OpenClaw v2026.6.11 릴리스 노트 (2026-06-30)

여러분의 피드백을 반영했습니다. v2026.6.11은 잘못 전달되는 답장, 멈춘 전송, 재연결, 모델 설정 실패를 수정하고 관리자 기본값을 더 안전하게 개선하여 OpenClaw의 안정성을 떨어뜨리던 문제에 중점을 둡니다.

## 주요 내용

### 채널 전송 안정성

[Telegram](/ko/channels/telegram), [WhatsApp](/ko/channels/whatsapp), [Matrix](/ko/channels/matrix), [Google Chat](/ko/channels/googlechat), [iMessage](/ko/channels/imessage), [Feishu](/ko/channels/feishu), [Mattermost](/ko/channels/mattermost) 전반에서 답장, 명령, 대기열의 메시지 및 첨부 파일이 누락되거나 중복되거나 잘못 전달되거나 엉뚱한 대화에 첨부될 가능성이 줄었습니다.

### 제공자 및 모델 복구

카탈로그, 자격 증명, 스트림, 시간 초과, Compaction 또는 대체 경로에 문제가 생겼을 때 [모델 선택](/ko/providers/models) 및 설정 과정에서 더 명확하게 복구할 수 있습니다. 이제 영향을 받는 [OpenAI](/ko/providers/openai), [OpenRouter](/ko/providers/openrouter), [OpenCode Go](/ko/providers/opencode-go) 구성은 사용자가 오래된 선택 항목을 그대로 사용하거나 요청이 중단된 상태로 남지 않도록 오류를 더 일관되게 보고하거나 복구합니다.

[고속 모드](/ko/tools/thinking#fast-mode-fast)의 후속 수정으로 영향을 받는 제공자 및 대체 경로에서의 동작이 개선되었습니다. 자동 고속 모드 자체는 이번 릴리스에 새로 추가된 기능이 아닙니다.

### 세션, 메모리 및 신뢰 연속성

[세션](/ko/concepts/session), [Compaction](/ko/concepts/compaction), [메모리](/ko/concepts/memory), [QMD 기반 메모리](/ko/concepts/memory-qmd)는 장시간 실행되는 작업, 재연결, 업그레이드 및 대화 기록 복구 과정에서도 의도한 대화와 유용한 컨텍스트를 더욱 일관되게 보존합니다. 또한 에이전트가 적절한 컨텍스트나 기능을 복구해야 할 때 [도구 검색](/ko/tools/tool-search)이 더욱 안정적으로 작동합니다.

암호화된 [Matrix](/ko/channels/matrix) 복구는 필수 키 상태를 확인할 수 없으면 안전하게 중단됩니다. [도구 정책](/ko/gateway/sandbox-vs-tool-policy-vs-elevated), [승인](/ko/cli/approvals), [비밀 정보 처리](/ko/gateway/secrets)는 의도된 런타임 상태에 계속 연결되며, 위험도가 높은 작업은 명시적으로 활성화하지 않는 한 비활성화된 상태로 유지됩니다.

### 공식 Plugin 설치 및 복구

이제 [Plugin 관리](/ko/plugins/manage-plugins)는 일반적인 외부 패키지 설치 및 복구 흐름을 통해 더 많은 공식 통합을 처리합니다. [Plugin 인벤토리](/ko/plugins/plugin-inventory)와 관련 설정 검사를 통해 패키지가 누락되었거나, 호환되지 않거나, 다시 설치해야 할 때 사용자에게 더 명확한 결과를 제공합니다.

- `plugins.allow`에서 실제 Plugin ID 대신 채널 또는 패키지 이름을 사용하는 경우, 이제 시작 안내에서 일치하지 않는 항목을 식별하고 구성을 수정하는 데 필요한 검색된 Plugin ID를 표시합니다. [#68389](https://github.com/openclaw/openclaw/pull/68389) 관련 항목: [#68352](https://github.com/openclaw/openclaw/issues/68352). @aym9999, @jirboy, @lyfuci, @pahuchi-joe, @zmxccxy 님께 감사드립니다.

[#78105](https://github.com/openclaw/openclaw/pull/78105) 관련 항목: [#68780](https://github.com/openclaw/openclaw/issues/68780). @jirboy, @pahuchi-joe 님께 감사드립니다.

## Gateway, 보안 및 신뢰

### 재시작 및 준비 상태 복구

이제 [Gateway 상태](/ko/gateway/health) 및 [문제 해결](/ko/gateway/troubleshooting) 신호는 OpenClaw가 실제로 준비되었는지, 재시작 중인지 또는 계속 진행할 수 없는지와 더 일관되게 부합합니다. [CLI](/ko/cli/agent)를 통해 시작된 에이전트 실행은 연결 해제, 종료, 라우팅 변경 및 시작 실패 상태에서 더 원활하게 복구됩니다.

<Accordion title="출처 및 기여자">

- 구성되었거나 명시적으로 지정된 원격 Gateway가 느리지만 연결 가능한 경우, 이제 `openclaw gateway probe --timeout ...`는 더 짧은 내부 제한 시간 후 연결 불가로 보고하지 않고 요청된 제한 시간 동안 대기합니다. [#89859](https://github.com/openclaw/openclaw/pull/89859) 관련 항목: [#65355](https://github.com/openclaw/openclaw/issues/65355). @hellocli, @mushuiyu886 님께 감사드립니다.
- 장시간 또는 병렬로 실행되는 내부 하위 에이전트 작업은 이제 불필요한 실시간 미리 보기 처리를 피하는 한편, 표시되는 하위 에이전트 세션에는 계속 실시간 업데이트와 최종 응답이 나타납니다. [#91906](https://github.com/openclaw/openclaw/pull/91906) @lanzhi-lee, @vincentkoc 님께 감사드립니다.
- ACP 대화, 특히 Kiro 기반 스레드는 백엔드가 기존 세션을 더 이상 재개할 수 없을 때 새 세션을 시작하여 이제 첫 번째 응답 이후에도 계속됩니다. [#93547](https://github.com/openclaw/openclaw/pull/93547) 관련 항목: [#87830](https://github.com/openclaw/openclaw/issues/87830). @amersheeny, @chouzz 님께 감사드립니다.
- Linux의 메모리 압박으로 하위 명령이나 세션이 종료될 때, 이제 systemd가 관리하는 OpenClaw Gateway는 계속 실행되고 채널 연결을 유지하면서 하위 프로세스 실패를 보고합니다. [#93585](https://github.com/openclaw/openclaw/pull/93585) @snowzlm 님께 감사드립니다.
- 도구 작업 중 OpenClaw 실행을 취소하면 이제 다른 모델 턴을 시작하거나 세션을 잠긴 상태로 두지 않고 즉시 종료됩니다. [#94412](https://github.com/openclaw/openclaw/pull/94412) @szsip239, @vincentkoc 님께 감사드립니다.

- 에이전트가 시작되기 전에 WebChat 메시지가 실패하면 이제 WebChat과 Control UI는 세션이 계속 실행 중인 것처럼 보이게 두지 않고 실패한 것으로 표시합니다. [#84352](https://github.com/openclaw/openclaw/pull/84352) @jesse-merhi 님께 감사드립니다.

### 도구 및 예약 작업

[예약 작업](/ko/cli/cron)과 기본 제공 [도구](/ko/tools/index)는 이제 완료, 재시도, 실패 보고 및 의도된 입력 보존을 더욱 일관되게 처리합니다. [Plugin SDK 런타임](/ko/plugins/sdk-runtime)의 개선으로 도구 기반 확장 기능이 로드되거나 결과를 반환하거나 예약 작업을 실행할 때의 안정성도 향상되었습니다.

<Accordion title="출처 및 기여자">

- `deleteAfterRun`을 사용하는 격리된 Cron 작업은 이제 전달이 비활성화된 실행을 포함해 완료 후 임시 세션과 대화 기록을 제거하여 오래된 파일, 누적된 컨텍스트 및 수동 정리 작업을 줄입니다. [#84794](https://github.com/openclaw/openclaw/pull/84794) 관련 항목: [#84707](https://github.com/openclaw/openclaw/issues/84707). @bottenbenny, @turbotheturtle 님께 감사드립니다.
- 이제 개별 예약 작업에서 자체 대체 모델을 사용하거나, 대체 모델을 비활성화한 상태로 실행하거나, 운영자가 하위 수준의 페이로드 데이터를 편집할 필요 없이 CLI를 통해 일반적인 대체 모델 상속으로 돌아갈 수 있습니다. [#93369](https://github.com/openclaw/openclaw/pull/93369) 관련 항목: [#90302](https://github.com/openclaw/openclaw/issues/90302). @849261680, @walliiee 님께 감사드립니다.
- 이제 Cron 기록은 작업 ID 앞뒤에 불필요한 공백이 포함된 항목도 안정적으로 찾으며, 안전하게 다시 읽을 수 없는 로그 레코드가 생성되기 전에 중첩되거나 비어 있는 ID를 거부합니다. [#93567](https://github.com/openclaw/openclaw/pull/93567) @alix-007, @vincentkoc 님께 감사드립니다.
- Cron 작업을 추가하거나 제거해도 이미 실행 시점에 도달한 다른 반복 작업의 대기 중인 실행이 더 이상 사라지지 않습니다. [#94323](https://github.com/openclaw/openclaw/pull/94323) @yetval 님께 감사드립니다.
- 이제 Word, PowerPoint 및 Excel 문서 읽기와 쓰기에서 임의로 만들어진 확장자로 인해 실패하는 대신 의도된 `.docx`, `.pptx` 또는 `.xlsx` 경로를 사용합니다. [#95805](https://github.com/openclaw/openclaw/pull/95805) 관련 항목: [#93326](https://github.com/openclaw/openclaw/issues/93326). @bhnan, @lzyyzznl, @vincentkoc, @xzh-icenter 님께 감사드립니다.
- 브라우저 자동화 사용자는 스냅샷 생성 중 불필요한 처리는 줄이면서도, 간결한 결과에 포함된 유용한 분기를 비롯해 참조 정보가 풍부한 동일한 스냅샷을 계속 사용할 수 있습니다. [#96072](https://github.com/openclaw/openclaw/pull/96072) @vincentkoc 님께 감사드립니다.
- Windows에서 시간 초과된 명령과 중단된 핵심 업데이트가 때때로 자식 프로세스를 실행 상태로 남기던 문제를 수정하여, 취소, 시간 초과 또는 업데이트 정리 후 OpenClaw가 전체 프로세스 트리를 더 안정적으로 종료할 수 있습니다. [a192b2e](https://github.com/openclaw/openclaw/commit/a192b2ea52b3166a7d190bf5f60f3feb030306bb) @vincentkoc 님께 감사드립니다.
