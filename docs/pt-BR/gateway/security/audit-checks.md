`openclaw security audit` emite constatações estruturadas identificadas por `checkId`. Esta
página é o catálogo de referência desses IDs. Para consultar o modelo de ameaças em alto nível
e as orientações de proteção, consulte [Segurança](/pt-BR/gateway/security).

Algumas verificações são executadas somente com `openclaw security audit --deep`: varreduras de código
de plugins/Skills (`plugins.code_safety*`, `skills.code_safety*`) e verificações de sondagem
do Gateway em execução (`gateway.probe_*`). Todas as demais verificações desta tabela são executadas com um simples
`openclaw security audit`.

Uma gravidade como `warn/critical` significa que o mesmo `checkId` pode ser emitido em
qualquer um dos níveis, dependendo da configuração (por exemplo, se o Gateway está exposto
remotamente). Valores de alta relevância que você provavelmente verá em implantações reais (lista

| `security.trust_model.multi_user_heuristic`                     | aviso               | A configuração parece ser multiusuário, enquanto o modelo de confiança do Gateway é de assistente pessoal                 | separe os limites de confiança ou reforce a segurança para usuários compartilhados (`sandbox.mode`, negação de ferramentas/escopo do espaço de trabalho)       | não       |
| `tools.profile_minimal_overridden`                              | aviso               | Substituições do agente ignoram o perfil mínimo global                                           | `agents.list[].tools.profile`                                                                        | não       |
| `plugins.tools_reachable_permissive_policy`                     | aviso               | Ferramentas de extensão acessíveis em contextos permissivos                                        | `tools.profile` + permissão/negação de ferramentas                                                                    | não       |
| `models.legacy`                                                 | aviso               | Famílias de modelos legadas ainda estão configuradas                                              | seleção de modelo                                                                                      | não       |
| `models.weak_tier`                                              | aviso               | Os modelos configurados estão abaixo dos níveis atualmente recomendados                                   | seleção de modelo                                                                                      | não       |
| `models.small_params`                                           | crítico/informativo      | Modelos pequenos + superfícies de ferramentas inseguras aumentam o risco de injeção                                | escolha de modelo + política de sandbox/ferramentas                                                                   | não       |
| `channels.<provider>.dm.open`                                   | crítico           | A política de DM de `<provider>` é `"open"`; qualquer pessoa pode enviar uma DM ao bot                               | `channels.<provider>.dmPolicy`, `.allowFrom`                                                         | não       |
| `channels.<provider>.dm.open_invalid`                           | aviso               | `dmPolicy="open"` sem `"*"` em `allowFrom` é inconsistente                          | `channels.<provider>.allowFrom`                                                                      | não       |
| `channels.<provider>.dm.scope_main_multiuser`                   | aviso               | Vários remetentes de DM compartilham atualmente a sessão principal                                    | `session.dmScope`                                                                                    | não       |
| `channels.<provider>.allowFrom.dangerous_name_matching_enabled` | informativo               | `dangerouslyAllowNameMatching` reativa a correspondência de remetentes por nome/e-mail/tag mutáveis        | desative `dangerouslyAllowNameMatching`, use IDs de remetentes estáveis                                        | não       |
| `channels.<provider>.account.read_only_resolution`              | aviso               | Não foi possível resolver completamente uma conta de canal para auditoria (segredo/Gateway ausente)        | garanta que os segredos referenciados possam ser resolvidos ou execute com um snapshot ativo do Gateway                     | não       |
| `channels.<provider>.warning.<n>`                               | informativo/aviso/crítico | Aviso de segurança específico do provedor, classificado a partir de texto livre do Plugin               | consulte os detalhes da constatação                                                                                   | não       |
| `summary.attack_surface`                                        | info               | Resumo consolidado da postura de autenticação, canal, ferramenta e exposição            | várias chaves (consulte os detalhes da constatação)                                                   | não      |

Os checkIds `channels.<provider>.*` e `tools.elevated.allowFrom.<provider>.*` são
gerados por canal/provedor configurado, portanto `<provider>` é um id de canal real
(por exemplo, `telegram`, `discord`) na saída efetiva, e não uma string literal.
