Stellen Sie OpenClaw mit einer Ein-Klick-Vorlage auf Northflank bereit und greifen Sie über die webbasierte Steuerungsoberfläche darauf zu. Dies ist der einfachste Weg „ohne Terminal auf dem Server“: Northflank betreibt den Gateway für Sie.

## Erste Schritte

1. Klicken Sie auf [OpenClaw bereitstellen](https://northflank.com/stacks/deploy-openclaw), um die Vorlage zu öffnen.
2. Erstellen Sie ein [Konto bei Northflank](https://app.northflank.com/signup), falls Sie noch keines haben.
3. Klicken Sie auf **Deploy OpenClaw now**.
4. Legen Sie die erforderliche Umgebungsvariable fest: `OPENCLAW_GATEWAY_TOKEN` (verwenden Sie einen starken Zufallswert).
5. Klicken Sie auf **Deploy stack**, um die OpenClaw-Vorlage zu erstellen und auszuführen.
6. Warten Sie, bis die Bereitstellung abgeschlossen ist, und klicken Sie dann auf **View resources**.
7. Öffnen Sie den OpenClaw-Dienst.
8. Öffnen Sie die öffentliche OpenClaw-URL unter `/openclaw` und stellen Sie mit dem konfigurierten gemeinsamen Geheimnis eine Verbindung her. Diese Vorlage verwendet standardmäßig `OPENCLAW_GATEWAY_TOKEN`. Wenn Sie dies durch eine Passwortauthentifizierung ersetzen, verwenden Sie stattdessen dieses Passwort.

## Bereitgestellte Komponenten

- Gehosteter OpenClaw Gateway und Steuerungsoberfläche
- Persistenter Speicher über ein Northflank-Volume (`/data`), damit `openclaw.json`, agentenspezifische `auth-profiles.json`, Kanal-/Provider-Status, Sitzungen und Arbeitsbereich erneute Bereitstellungen überstehen

## Einen Kanal verbinden

Verwenden Sie die Steuerungsoberfläche unter `/openclaw` oder führen Sie `openclaw onboard` über SSH aus, um Anweisungen zur Kanaleinrichtung zu erhalten:

- [Telegram](/de/channels/telegram) (am schnellsten, nur ein Bot-Token erforderlich)
- [Discord](/de/channels/discord)
- [Alle Kanäle](/de/channels)
