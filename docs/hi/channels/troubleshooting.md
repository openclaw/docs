जब कोई चैनल कनेक्ट हो, लेकिन उसका व्यवहार गलत हो, तब इस पेज का उपयोग करें।

## कमांड क्रम

पहले इन्हें इसी क्रम में चलाएँ:

```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
openclaw channels status --probe
```

स्वस्थ आधार-स्थिति:

- `Runtime: running`
- `Connectivity probe: ok`
- `Capability: read-only`, `write-capable`, या `admin-capable`
- चैनल जाँच दिखाती है कि ट्रांसपोर्ट कनेक्ट है और जहाँ समर्थित हो, वहाँ `works` या `audit ok`

## अपडेट के बाद

जब अपडेट करने के बाद Telegram, iMessage, BlueBubbles-युग के कॉन्फ़िगरेशन या कोई अन्य Plugin चैनल गायब हो जाए, तब इसका उपयोग करें।

```bash
openclaw status --all
openclaw doctor --fix
openclaw gateway restart
openclaw status --all
```

`openclaw
status --all` में `plugin load failed: dependency tree corrupted; run openclaw doctor --fix` खोजें। इसका अर्थ है कि चैनल कॉन्फ़िगर किया गया है, लेकिन Plugin सेटअप/लोड ने चैनल को पंजीकृत करने के बजाय दूषित डिपेंडेंसी ट्री का सामना किया। `openclaw doctor --fix` पुराने Plugin-रनटाइम डिपेंडेंसी सिमलिंक और पुराने प्रमाणीकरण शैडो हटाता है, फिर `openclaw gateway restart` स्वच्छ स्थिति को दोबारा लोड करता है।
