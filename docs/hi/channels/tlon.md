Tlon, Urbit पर निर्मित एक विकेंद्रीकृत मैसेंजर है। OpenClaw आपके Urbit शिप से जुड़ता है और
DM तथा समूह चैट संदेशों का उत्तर देता है। समूह में उत्तर देने के लिए डिफ़ॉल्ट रूप से @ उल्लेख आवश्यक है, जिसके
ऊपर प्राधिकरण नियम और स्वामी-अनुमोदन प्रवाह लागू होते हैं।

स्थिति: बंडल किया गया Plugin। DM, समूह उल्लेख, थ्रेड, रिच टेक्स्ट, इमेज अपलोड/डाउनलोड और
स्वामी अनुमोदन प्रणाली समर्थित हैं। प्रतिक्रियाएँ और पोल समर्थित नहीं हैं।

## बंडल किया गया Plugin

Tlon वर्तमान OpenClaw रिलीज़ में बंडल होकर आता है; पैकेज किए गए बिल्ड के लिए अलग इंस्टॉलेशन आवश्यक नहीं है।

किसी पुराने बिल्ड या इसे शामिल न करने वाले कस्टम इंस्टॉलेशन पर npm से इंस्टॉल करें:

```bash
openclaw plugins install @openclaw/tlon
```

वर्तमान रिलीज़ टैग को ट्रैक करने के लिए केवल पैकेज नाम का उपयोग करें। केवल पुनरुत्पाद्य इंस्टॉलेशन के लिए संस्करण (`@openclaw/tlon@x.y.z`)
पिन करें।

स्थानीय चेकआउट से:

```bash
openclaw plugins install ./path/to/local/tlon-plugin
```

विवरण: [Plugins](/hi/tools/plugin)

## सेटअप

```bash
openclaw channels add --channel tlon --ship ~sampel-palnet --url https://your-ship-host --code lidlut-tabwed-pillex-ridrup
```

या कॉन्फ़िगरेशन को सीधे संपादित करें:

```json5
{
  channels: {
    tlon: {
      enabled: true,
      ship: "~sampel-palnet",
      url: "https://your-ship-host",
      code: "lidlut-tabwed-pillex-ridrup",
      ownerShip: "~your-main-ship", // अनुशंसित: आपका शिप, हमेशा अधिकृत
    },
  },
}
```

कॉन्फ़िगरेशन को सीधे संपादित करने के बाद Gateway पुनः आरंभ करें। फिर बॉट को DM करें या समूह
चैनल में उसका @ उल्लेख करें।

## इनबाउंड स्थायित्व

OpenClaw, एजेंट डिस्पैच से पहले स्वीकार किए गए Tlon DM और समूह-चैट इवेंट को स्थायी रूप से संग्रहीत करता है। लंबित या पुनः प्रयास योग्य टर्न Gateway के पुनः आरंभ होने के बाद भी बने रहते हैं और कार्य प्रत्येक समूह चैनल या प्रत्यक्ष पीयर के लिए क्रमबद्ध रहता है। स्थिर Urbit संदेश ID भी किसी पुनः डिलीवर किए गए इवेंट को तब तक दबाते हैं, जब तक उसका क्यू रिकॉर्ड या सुरक्षित रखा गया पूर्णता रिकॉर्ड मौजूद रहता है।

क्यू-से-एजेंट सीमा पर डिलीवरी कम-से-कम-एक-बार होती है: हैंडऑफ़ के दौरान क्रैश होने पर टर्न फिर से चल सकता है। इसलिए जहाँ व्यावहारिक हो, बाहरी दुष्प्रभाव उत्पन्न करने वाली एजेंट कार्रवाइयाँ इडेम्पोटेंट रहनी चाहिए।

## निजी/LAN शिप

SSRF सुरक्षा के लिए OpenClaw डिफ़ॉल्ट रूप से निजी/आंतरिक होस्टनाम और IP रेंज को ब्लॉक करता है। यदि आपका
शिप किसी निजी नेटवर्क (localhost, LAN IP, आंतरिक होस्टनाम) पर चलता है, तो स्पष्ट रूप से ऑप्ट इन करें:

```json5
{
  channels: {
    tlon: {
      url: "http://localhost:8080",
      network: {
        dangerouslyAllowPrivateNetwork: true,
      },
    },
  },
}
```

यह `http://localhost:8080`, `http://192.168.x.x:8080` और
`http://my-ship.local:8080` जैसे लक्ष्यों पर लागू होता है। इसे केवल उस शिप URL के लिए सक्षम करें जिस पर आपको भरोसा है; यह उस
अकाउंट के HTTP अनुरोधों के लिए SSRF सुरक्षा अक्षम कर देता है।

<Note>
`channels.tlon.allowPrivateNetwork` (फ़्लैट कुंजी) को हटा दिया गया है। `openclaw doctor --fix` इसे
स्वचालित रूप से `channels.tlon.network.dangerouslyAllowPrivateNetwork` में स्थानांतरित करता है।
</Note>

## समूह चैनल

चैनलों को मैन्युअल रूप से पिन करें या स्वचालित खोज चालू करें:

```json5
{
  channels: {
    tlon: {
      groupChannels: ["chat/~host-ship/general", "chat/~host-ship/support"],
      autoDiscoverChannels: true,
    },
  },
}
```

कॉन्फ़िगरेशन में सेट न होने पर `autoDiscoverChannels` का डिफ़ॉल्ट `false` होता है; सेटअप विज़ार्ड
प्रॉम्प्ट को डिफ़ॉल्ट रूप से हाँ पर रखता है और `true` को स्पष्ट रूप से लिखता है। इसके चालू रहने पर OpenClaw स्टार्टअप के समय जुड़े हुए समूहों को स्क्राइ करता है,
समूह आमंत्रण स्वीकार होते ही नए चैनलों की निगरानी करता है और हर 2 मिनट में दोबारा जाँच करता है।

## अभिगम नियंत्रण

DM अनुमति-सूची (खाली = प्रेषक के `ownerShip` होने के अलावा किसी DM की अनुमति नहीं):

```json5
{
  channels: {
    tlon: {
      dmAllowlist: ["~zod", "~nec"],
    },
  },
}
```

समूह प्राधिकरण का डिफ़ॉल्ट प्रत्येक चैनल के लिए `restricted` है। आधाररेखा के लिए `defaultAuthorizedShips` सेट करें
और प्रत्येक चैनल नेस्ट के अनुसार ओवरराइड करें:

```json5
{
  channels: {
    tlon: {
      defaultAuthorizedShips: ["~zod"],
      authorization: {
        channelRules: {
          "chat/~host-ship/general": {
            mode: "restricted",
            allowedShips: ["~zod", "~nec"],
          },
          "chat/~host-ship/announcements": {
            mode: "open",
          },
        },
      },
    },
  },
}
```

बॉट द्वारा किसी थ्रेड में उत्तर देने के बाद, वह उस थ्रेड के बाद के संदेशों का उत्तर
बिना किसी अन्य उल्लेख की आवश्यकता के देता रहता है।

उन फ़ॉलो-अप के लिए नया स्पष्ट उल्लेख आवश्यक बनाने हेतु `channels.tlon.implicitMentions.threadParticipation: false` सेट करें।
अकाउंट ओवरराइड `channels.tlon.accounts.<id>.implicitMentions` का उपयोग करते हैं। Tlon वर्तमान में
`replyToBot` या `quotedBot` तथ्य उत्पन्न नहीं करता, इसलिए इन फ़्लैग का यहाँ कोई प्रभाव नहीं पड़ता।

## स्वामी और अनुमोदन प्रणाली

```json5
{
  channels: {
    tlon: {
      ownerShip: "~your-main-ship",
    },
  },
}
```

स्वामी शिप हर जगह अधिकृत होता है: DM आमंत्रण हमेशा स्वचालित रूप से स्वीकार किए जाते हैं, समूह आमंत्रण
हमेशा स्वचालित रूप से स्वीकार किए जाते हैं और चैनल संदेश हमेशा प्राधिकरण पास करते हैं। स्वामी को
`dmAllowlist`, `defaultAuthorizedShips` या `groupInviteAllowlist` में होने की आवश्यकता नहीं है।

`ownerShip` सेट होने पर अनधिकृत अनुरोध केवल छोड़े नहीं जाते—वे एक लंबित
अनुमोदन को क्यू में डालते हैं और स्वामी को DM करते हैं:

- `dmAllowlist` में शामिल न होने वाले शिप से DM अनुरोध
- उन चैनलों में उल्लेख जहाँ प्रेषक प्राधिकरण में विफल रहता है
- `groupInviteAllowlist` में शामिल न होने वाले शिप से समूह आमंत्रण (जब स्वचालित स्वीकृति बंद हो, या चालू हो लेकिन
  आमंत्रण भेजने वाला अनुमति-सूची में न हो)

स्वामी किसी अनुरोध पर कार्रवाई करने के लिए DM में उत्तर देता है:

| स्वामी का उत्तर                  | प्रभाव                                               |
| ---------------------------- | ---------------------------------------------------- |
| `approve` / `deny` / `block` | सबसे हाल के लंबित अनुमोदन पर कार्रवाई करता है             |
| `approve <id>` / `deny <id>` | ID के आधार पर किसी विशिष्ट अनुमोदन पर कार्रवाई करता है                    |
| `block`                      | शिप को मूल रूप से ब्लॉक भी करता है, ताकि वह पुनः कनेक्ट न कर सके |
| `unblock ~ship`              | मूल ब्लॉक को उलटता है                              |
| `blocked`                    | वर्तमान में ब्लॉक किए गए शिप सूचीबद्ध करता है                        |
| `pending`                    | लंबित अनुमोदन अनुरोध सूचीबद्ध करता है                      |

`ownerShip` कॉन्फ़िगर न होने पर अनधिकृत DM और चैनल उल्लेख केवल छोड़ दिए जाते हैं और लॉग किए जाते हैं;
कोई अनुमोदन प्रॉम्प्ट नहीं होता।

## स्वचालित स्वीकृति सेटिंग

पहले से `dmAllowlist` में शामिल शिप के DM आमंत्रण स्वचालित रूप से स्वीकार करें (इस फ़्लैग से स्वतंत्र रूप से स्वामी को हमेशा
स्वचालित रूप से स्वीकार किया जाता है):

```json5
{
  channels: {
    tlon: {
      autoAcceptDmInvites: true,
    },
  },
}
```

अनुमति-सूची से समूह आमंत्रण स्वचालित रूप से स्वीकार करें (विफल होने पर बंद: `autoAcceptGroupInvites: true` और
खाली `groupInviteAllowlist` के साथ, किसी गैर-स्वामी का आमंत्रण स्वीकार नहीं किया जाता):

```json5
{
  channels: {
    tlon: {
      autoAcceptGroupInvites: true,
      groupInviteAllowlist: ["~zod"],
    },
  },
}
```

## Urbit सेटिंग स्टोर के माध्यम से हॉट-रीलोड

ऊपर की अधिकांश सेटिंग (`dmAllowlist`, `groupInviteAllowlist`, `groupChannels`,
`defaultAuthorizedShips`, `autoDiscoverChannels`, `autoAcceptDmInvites`,
`autoAcceptGroupInvites`, `ownerShip`, `showModelSignature`) पहली बार चलने पर शिप के
`%settings` एजेंट (डेस्क `moltbot`, बकेट `tlon`) में मिरर की जाती हैं और फिर वहीं से लाइव पढ़ी जाती हैं,
इसलिए Landscape क्लाइंट या बंडल किए गए स्किल के सेटिंग कमांड के माध्यम से किए गए परिवर्तन
Gateway को पुनः आरंभ किए बिना लागू हो जाते हैं। `channelRules` और लंबित अनुमोदन भी वहाँ JSON के रूप में स्थायी रूप से संग्रहीत होते हैं। सेटिंग
स्टोर में कभी न लिखे गए मानों के लिए फ़ाइल कॉन्फ़िगरेशन सत्य का स्रोत बना रहता है।

## बंडल किया गया स्किल

Plugin में सीधे Urbit संचालन के लिए CLI [`@tloncorp/tlon-skill`](https://github.com/tloncorp/tlon-skill) बंडल किया गया है,
जो Plugin इंस्टॉल होने के बाद स्वचालित रूप से उपलब्ध होता है:

- **गतिविधि**: उल्लेख, उत्तर, अपठित
- **चैनल**: सूचीबद्ध करना, बनाना, नाम बदलना
- **संपर्क**: प्रोफ़ाइल सूचीबद्ध करना/प्राप्त करना/अपडेट करना
- **समूह**: बनाना, जुड़ना, आमंत्रण/अनुरोध प्रवाह, भूमिकाएँ
- **हुक**: चैनल हुक प्रबंधित करना
- **संदेश**: इतिहास, खोज
- **DM**: भेजना, प्रतिक्रिया देना, स्वीकार/अस्वीकार करना
- **पोस्ट**: प्रतिक्रिया देना, हटाना
- **नोटबुक**: डायरी चैनलों पर पोस्ट करना
- **सेटिंग**: ऊपर दिए गए सेटिंग स्टोर के माध्यम से Plugin कॉन्फ़िगरेशन हॉट-रीलोड करना

## समस्या निवारण

```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
```

सामान्य विफलताएँ:

- **DM अनदेखे किए गए**: प्रेषक `dmAllowlist` में नहीं है और अनुमोदन प्रवाह के लिए कोई `ownerShip` कॉन्फ़िगर नहीं है।
- **समूह संदेश अनदेखे किए गए**: चैनल खोजा/पिन नहीं किया गया है, या प्रेषक प्राधिकरण में विफल रहता है और अनुमोदन को क्यू में डालने के लिए कोई
  `ownerShip` नहीं है।
- **कनेक्शन त्रुटियाँ**: जाँचें कि शिप URL पहुँच योग्य है; स्थानीय शिप के लिए
  `network.dangerouslyAllowPrivateNetwork` सेट करें।
- **प्रमाणीकरण त्रुटियाँ**: लॉगिन कोड बदलते रहते हैं—अपने शिप से वर्तमान कोड कॉपी करें।

| `channels.tlon.enabled`                                | चैनल स्टार्टअप सक्षम/अक्षम करें।                                |
| `channels.tlon.ship`                                   | बॉट का Urbit शिप नाम (उदा. `~sampel-palnet`)।                 |
| `channels.tlon.url`                                    | शिप URL (उदा. `https://sampel-palnet.tlon.network`)।          |
| `channels.tlon.code`                                   | शिप लॉगिन कोड।                                               |
| `channels.tlon.network.dangerouslyAllowPrivateNetwork` | localhost/LAN शिप URL की अनुमति दें (SSRF ऑप्ट-इन)।                   |
| `channels.tlon.ownerShip`                              | स्वामी शिप: हमेशा अधिकृत, अनुमोदन अनुरोध प्राप्त करता है।     |
| `channels.tlon.dmAllowlist`                            | DM की अनुमति वाले शिप (खाली = स्वामी के अलावा कोई नहीं)।              |
| `channels.tlon.autoAcceptDmInvites`                    | `dmAllowlist` में शामिल शिप से DM स्वचालित रूप से स्वीकार करें।                   |
| `channels.tlon.autoAcceptGroupInvites`                 | `groupInviteAllowlist` से समूह आमंत्रण स्वचालित रूप से स्वीकार करें।         |
| `channels.tlon.groupInviteAllowlist`                   | वे शिप जिनके समूह आमंत्रण स्वचालित रूप से स्वीकार किए जाते हैं।                   |
| `channels.tlon.autoDiscoverChannels`                   | जुड़े हुए समूह चैनल स्वचालित रूप से खोजें (डिफ़ॉल्ट: `false`)।        |
| `channels.tlon.implicitMentions.threadParticipation`   | सहभागिता वाले थ्रेड के फ़ॉलो-अप को उल्लेख नियंत्रण बायपास करने दें।      |
| `channels.tlon.groupChannels`                          | मैन्युअल रूप से पिन किए गए चैनल नेस्ट।                                 |
| `channels.tlon.defaultAuthorizedShips`                 | सभी चैनलों के लिए अधिकृत शिप (जब कोई नियम मेल न खाए तब उपयोग होता है)। |
| `channels.tlon.authorization.channelRules`             | प्रत्येक चैनल नेस्ट का प्रमाणीकरण मोड + अनुमति-सूची।                        |
| `channels.tlon.showModelSignature`                     | उत्तरों में `_[Generated by <model>]_` जोड़ें।                  |
| `channels.tlon.responsePrefix`                         | आउटबाउंड उत्तरों के आगे जोड़ा गया स्थिर उपसर्ग।                   |
| `channels.tlon.accounts.<id>`                          | अतिरिक्त नामित अकाउंट (मल्टी-शिप सेटअप)।                 |

## टिप्पणियाँ

- समूह में उत्तर देने के लिए @ उल्लेख (जैसे `~your-bot-ship`) आवश्यक है, जब तक कि बॉट पहले से उस थ्रेड में शामिल न हो।
- थ्रेड के उत्तर उसी थ्रेड में पहुँचते हैं; बॉट को एजेंट के लिए थ्रेड के संदर्भ के अंतिम 10 संदेश भी
  पहले जोड़कर मिलते हैं।
- रिच टेक्स्ट (बोल्ड, इटैलिक, कोड, हेडर, सूचियाँ) Tlon के मूल प्रारूप में परिवर्तित हो जाता है।
- चैनल के सारांश का अनुरोध करने वाला इनबाउंड संदेश भेजने पर (उदाहरण के लिए "इस
  चैनल का सारांश दें") सामान्य उत्तर प्रवाह के बजाय अंतर्निहित इतिहास सारांशीकरण शुरू हो जाता है।
