OpenClaw उन चैनलों पर अन्य बॉट द्वारा लिखे गए संदेश स्वीकार कर सकता है जो `allowBots` का समर्थन करते हैं। जब यह पथ सक्षम होता है, तो जोड़ी लूप सुरक्षा दो बॉट पहचानों को अनिश्चित काल तक एक-दूसरे को उत्तर देने से रोकती है।

यह सुरक्षा मुख्य इनबाउंड उत्तर रनर द्वारा लागू की जाती है। प्रत्येक समर्थित चैनल अपने इनबाउंड इवेंट को सामान्य तथ्यों में मैप करता है: अकाउंट या स्कोप, वार्तालाप आईडी, प्रेषक बॉट आईडी और प्राप्तकर्ता बॉट आईडी। कोर प्रतिभागियों की जोड़ी को दोनों दिशाओं में ट्रैक करता है (A से B और B से A को एक ही जोड़ी माना जाता है), एक स्लाइडिंग-विंडो बजट लागू करता है और बजट पार होने के बाद कूलडाउन अवधि के दौरान उस जोड़ी को दबा देता है।

## डिफ़ॉल्ट

जब भी कोई चैनल बॉट द्वारा लिखे गए संदेशों को डिस्पैच तक पहुँचने देता है, जोड़ी लूप सुरक्षा सक्रिय रहती है। अंतर्निहित डिफ़ॉल्ट:

| कुंजी                  | डिफ़ॉल्ट | अर्थ                                             |
| -------------------- | ------- | --------------------------------------------------- |
| `enabled`            | `true`  | इसका समर्थन करने वाले चैनलों के लिए सुरक्षा सक्रिय है।          |
| `maxEventsPerWindow` | `20`    | विंडो के भीतर बॉट की जोड़ी द्वारा आदान-प्रदान किए जा सकने वाले इवेंट।   |
| `windowSeconds`      | `60`    | स्लाइडिंग विंडो की अवधि।                              |
| `cooldownSeconds`    | `60`    | जोड़ी द्वारा बजट पार करने के बाद दमन की अवधि। |

यह सुरक्षा मनुष्यों द्वारा लिखे गए संदेशों, एकल-बॉट डिप्लॉयमेंट, स्वयं के संदेशों की फ़िल्टरिंग या बजट के भीतर रहने वाले बॉट उत्तरों को प्रभावित नहीं करती।

## साझा डिफ़ॉल्ट कॉन्फ़िगर करें

प्रत्येक समर्थित चैनल को समान आधाररेखा देने के लिए `channels.defaults.botLoopProtection` को एक बार सेट करें। चैनल अधिक सीमित ओवरराइड भी उपलब्ध करा सकते हैं; Feishu जानबूझकर केवल इसी साझा आधाररेखा का उपयोग करता है।

```json5
{
  channels: {
    defaults: {
      botLoopProtection: {
        maxEventsPerWindow: 20,
        windowSeconds: 60,
        cooldownSeconds: 60,
      },
    },
  },
}
```

`enabled: false` को केवल तभी सेट करें, जब आपकी चैनल नीति स्वचालित दमन के बिना बॉट-से-बॉट वार्तालापों को जानबूझकर अनुमति देती हो।

## प्रत्येक चैनल, अकाउंट या रूम के लिए ओवरराइड करें

समर्थित चैनल अपनी कॉन्फ़िगरेशन को कुंजी-दर-कुंजी साझा डिफ़ॉल्ट के ऊपर लागू करते हैं। प्राथमिकता, सबसे सीमित पहले:

1. `channels.<channel>.<room-or-space>.botLoopProtection`, जब चैनल प्रति-वार्तालाप ओवरराइड का समर्थन करता है
2. `channels.<channel>.accounts.<account>.botLoopProtection`, जब चैनल अकाउंट का समर्थन करता है
3. `channels.<channel>.botLoopProtection`, जब चैनल शीर्ष-स्तरीय डिफ़ॉल्ट का समर्थन करता है
4. `channels.defaults.botLoopProtection`
5. अंतर्निहित डिफ़ॉल्ट

```json5
{
  channels: {
    defaults: {
      botLoopProtection: {
        maxEventsPerWindow: 20,
      },
    },
    discord: {
      botLoopProtection: {
        maxEventsPerWindow: 8,
      },
      accounts: {
        secondary: {
          allowBots: true,
          botLoopProtection: {
            maxEventsPerWindow: 5,
            cooldownSeconds: 90,
          },
        },
      },
    },
    googlechat: {
      allowBots: true,
      groups: {
        "spaces/AAAA": {
          botLoopProtection: {
            maxEventsPerWindow: 5,
          },
        },
      },
    },
    matrix: {
      allowBots: "mentions",
      groups: {
        "!roomid:example.org": {
          botLoopProtection: {
            maxEventsPerWindow: 5,
          },
        },
      },
    },
    slack: {
      allowBots: "mentions",
      botLoopProtection: {
        maxEventsPerWindow: 8,
      },
    },
  },
}
```

## चैनल समर्थन

- Discord: मूल `author.bot` तथ्य, जिन्हें Discord अकाउंट, चैनल और बॉट जोड़ी के अनुसार कुंजीबद्ध किया जाता है।
- Feishu: स्वीकृत बॉट-लिखित समूह संदेशों के मूल `sender_type=bot` तथ्य, जिन्हें Feishu अकाउंट, चैट और बॉट जोड़ी के अनुसार कुंजीबद्ध किया जाता है। Feishu केवल `channels.defaults.botLoopProtection` का उपयोग करता है।
- Google Chat: स्वीकृत बॉट-लिखित संदेशों के मूल `sender.type=BOT` तथ्य, जिन्हें अकाउंट, स्पेस और बॉट जोड़ी के अनुसार कुंजीबद्ध किया जाता है।
- Matrix: कॉन्फ़िगर किए गए Matrix बॉट अकाउंट, जिन्हें Matrix अकाउंट, रूम और कॉन्फ़िगर की गई बॉट जोड़ी के अनुसार कुंजीबद्ध किया जाता है।
- Slack: स्वीकृत बॉट-लिखित संदेशों के मूल `bot_id` तथ्य, जिन्हें Slack अकाउंट, चैनल और बॉट जोड़ी के अनुसार कुंजीबद्ध किया जाता है।

जो चैनल विश्वसनीय इनबाउंड बॉट पहचान उपलब्ध नहीं कराते, वे स्वयं के संदेश और पहुँच-नीति के अपने सामान्य फ़िल्टर का उपयोग जारी रखते हैं। जब तक वे बॉट जोड़ी के दोनों प्रतिभागियों की पहचान नहीं कर सकते, उन्हें इस सुरक्षा को सक्षम नहीं करना चाहिए।
