OpenClaw API प्रदाताओं के अनुपलब्ध, दर-सीमित या गलत व्यवहार करने पर केवल-पाठ फ़ॉलबैक के रूप में स्थानीय AI CLI चला सकता है। इसे जानबूझकर सतर्क रखा गया है:

- OpenClaw टूल सीधे इंजेक्ट नहीं किए जाते, लेकिन `bundleMcp: true` वाला बैकएंड लूपबैक MCP ब्रिज के माध्यम से Gateway टूल प्राप्त कर सकता है।
- इसका समर्थन करने वाले CLI के लिए JSONL स्ट्रीमिंग।
- सत्र समर्थित हैं, इसलिए बाद की प्रतिक्रियाओं में संदर्भ सुसंगत रहता है।
- यदि CLI इमेज पथ स्वीकार करता है, तो इमेज पास की जाती हैं।

इसे "हमेशा काम करने वाली" पाठ प्रतिक्रियाओं के लिए सुरक्षा-जाल के रूप में उपयोग करें, प्राथमिक पथ के रूप में नहीं। ACP सत्र नियंत्रण, पृष्ठभूमि कार्यों, थ्रेड/वार्तालाप बाइंडिंग और स्थायी बाहरी कोडिंग सत्रों वाले पूर्ण हार्नेस रनटाइम के लिए इसके बजाय [ACP एजेंट](/hi/tools/acp-agents) का उपयोग करें; CLI बैकएंड ACP नहीं हैं।

<Tip>
  नया बैकएंड Plugin बना रहे हैं? [CLI बैकएंड Plugin](/hi/plugins/cli-backend-plugins) देखें। यह पृष्ठ पहले से पंजीकृत बैकएंड को कॉन्फ़िगर और संचालित करने के बारे में है।
</Tip>

## त्वरित शुरुआत

बंडल किया गया Anthropic Plugin एक डिफ़ॉल्ट `claude-cli` बैकएंड पंजीकृत करता है, इसलिए Claude Code इंस्टॉल और लॉग इन होने के अतिरिक्त किसी कॉन्फ़िगरेशन के बिना यह काम करता है:

```bash
openclaw agent --agent main --message "hi" --model claude-cli/claude-sonnet-4-6
```

जब कोई स्पष्ट एजेंट सूची कॉन्फ़िगर नहीं होती, तब `main` डिफ़ॉल्ट एजेंट आईडी होती है; अन्यथा इसे अपनी एजेंट आईडी से बदलें।

यदि Gateway न्यूनतम `PATH` वाले launchd/systemd के अंतर्गत चलता है, तो बाइनरी को स्पष्ट रूप से निर्दिष्ट करें:

```json5
{
  agents: {
    defaults: {
      cliBackends: {
        "claude-cli": {
          command: "/opt/homebrew/bin/claude",
        },
      },
    },
  },
}
```

यदि आप Gateway होस्ट पर बंडल किए गए CLI बैकएंड को प्राथमिक संदेश प्रदाता के रूप में उपयोग करते हैं, तो आपके कॉन्फ़िगरेशन द्वारा मॉडल रेफ़रेंस में या `agents.defaults.cliBackends` के अंतर्गत उस बैकएंड का संदर्भ दिए जाने पर OpenClaw स्वामित्व वाले बंडल किए गए Plugin को स्वतः लोड करता है।

## इसे फ़ॉलबैक के रूप में उपयोग करना

CLI बैकएंड को अपनी फ़ॉलबैक सूची में जोड़ें, ताकि यह केवल प्राथमिक मॉडल विफल होने पर चले:

```json5
{
  agents: {
    defaults: {
      model: {
        primary: "anthropic/claude-opus-4-6",
        fallbacks: ["claude-cli/claude-sonnet-4-6"],
      },
      models: {
        "anthropic/claude-opus-4-6": { alias: "Opus" },
        "claude-cli/claude-sonnet-4-6": {},
      },
    },
  },
}
```

प्राथमिक प्रदाता के विफल होने पर (प्रमाणीकरण, दर सीमाएँ, टाइमआउट) कॉन्फ़िगर किए गए फ़ॉलबैक पात्र बने रहते हैं, भले ही वे `agents.defaults.modelPolicy.allow` में न हों। CLI बैकएंड मॉडल को उस नीति में केवल तभी जोड़ें, जब उपयोगकर्ताओं को `/model`, सत्र ओवरराइड या `--model` के माध्यम से उसे सीधे चुनने में भी सक्षम होना चाहिए। `agents.defaults.models` केवल प्रति-मॉडल उपनामों, पैरामीटरों और मेटाडेटा का स्वामी है।

## कॉन्फ़िगरेशन

सभी CLI बैकएंड `agents.defaults.cliBackends` के अंतर्गत होते हैं और प्रदाता आईडी (जैसे `claude-cli`, `my-cli`) से कुंजीबद्ध होते हैं। प्रदाता आईडी मॉडल रेफ़रेंस का बायाँ भाग बनती है: `<provider>/<model>`।

```json5
{
  agents: {
    defaults: {
      cliBackends: {
        "my-cli": {
          command: "my-cli",
          args: ["--json"],
          output: "json",
          input: "arg",
          modelArg: "--model",
          modelAliases: {
            "claude-opus-4-6": "opus",
            "claude-sonnet-4-6": "sonnet",
          },
          sessionArg: "--session",
          sessionMode: "existing",
          sessionIdFields: ["session_id", "conversation_id"],
          systemPromptArg: "--system",
          // समर्पित प्रॉम्प्ट-फ़ाइल फ़्लैग:
          // systemPromptFileArg: "--system-file",
          // इसके बजाय Codex-शैली का कॉन्फ़िगरेशन-ओवरराइड फ़्लैग:
          // systemPromptFileConfigArg: "-c",
          // systemPromptFileConfigKey: "model_instructions_file",
          systemPromptWhen: "first",
          imageArg: "--image",
          imageMode: "repeat",
          // केवल तभी विकल्प चुनें, जब यह बैकएंड अमान्य किए गए सत्रों को
          // Compaction से पहले के सीमित कच्चे OpenClaw ट्रांसक्रिप्ट इतिहास से दोबारा सीड कर सकता हो।
          reseedFromRawTranscriptWhenUncompacted: true,
          serialize: true,
        },
      },
    },
  },
}
```

## यह कैसे काम करता है

1. प्रदाता प्रीफ़िक्स (`claude-cli/...`) से बैकएंड चुनता है।
2. उसी OpenClaw प्रॉम्प्ट और कार्यस्थान संदर्भ का उपयोग करके सिस्टम प्रॉम्प्ट बनाता है।
3. CLI को सत्र आईडी के साथ निष्पादित करता है (यदि समर्थित हो), ताकि इतिहास सुसंगत रहे। बंडल किया गया `claude-cli` बैकएंड प्रत्येक OpenClaw सत्र के लिए Claude stdio प्रक्रिया को चालू रखता है और बाद की प्रतिक्रियाएँ stream-json stdin पर भेजता है।
4. आउटपुट (JSON या सादा पाठ) को पार्स करता है और अंतिम पाठ लौटाता है।
5. प्रत्येक बैकएंड के लिए सत्र आईडी बनाए रखता है, ताकि बाद की प्रतिक्रियाएँ उसी CLI सत्र का पुनः उपयोग करें।

## टाइमआउट और लंबे समय तक चलने वाला कार्य

CLI बैकएंड की दो स्वतंत्र सीमाएँ होती हैं:

- `agents.defaults.timeoutSeconds` पूरे एजेंट टर्न को सीमित करता है। सामान्य Gateway टर्न को 48 घंटे की डिफ़ॉल्ट सीमा मिलती है; `0` टर्न बजट को असीमित बनाता है। `600` जैसा संग्रहीत ओवरराइड उस डिफ़ॉल्ट को बदल देता है।
- CLI का आउटपुट-रहित वॉचडॉग मौन रहने वाली उपप्रक्रिया को रोक देता है। यह `agents.defaults.cliBackends.<id>.reliability.watchdog` के अंतर्गत अलग-अलग नए/पुनः आरंभ प्रोफ़ाइलों का उपयोग करता है और समग्र टर्न बजट असीमित होने पर भी सक्रिय रहता है।

48 घंटे के डिफ़ॉल्ट पर लौटने के लिए छोटा समग्र-टाइमआउट ओवरराइड हटाएँ, या 12 घंटे जैसा स्पष्ट बजट सेट करें:

```bash
# 48 घंटे के डिफ़ॉल्ट पर लौटें:
openclaw config unset agents.defaults.timeoutSeconds

# या 12 घंटे की स्पष्ट सीमा चुनें:
openclaw config set agents.defaults.timeoutSeconds 43200
```

CLI के भीतर शुरू किया गया पृष्ठभूमि कार्य अब भी उसी CLI उपप्रक्रिया का हिस्सा होता है। यदि पैरेंट टर्न अपनी समग्र सीमा तक पहुँच जाता है, तो OpenClaw उपप्रक्रिया और उसके CLI-आंतरिक पृष्ठभूमि कार्यों को एक साथ रोक देता है। टिकाऊ लंबे कार्य के लिए अलग किया गया OpenClaw [उप-एजेंट](/hi/tools/subagents) या [ACP एजेंट](/hi/tools/acp-agents) उपयोग करें; अलग किए गए उप-एजेंट के लिए डिफ़ॉल्ट रूप से कोई रन टाइमआउट नहीं होता।

`openclaw agent` कमांड की अपनी अनुरोध समय-सीमा भी होती है। उसका 600-सेकंड का फ़ॉलबैक डिफ़ॉल्ट केवल उस कमांड आह्वान पर लागू होता है, सामान्य Gateway टर्न पर नहीं; [`openclaw agent`](/hi/cli/agent) देखें।

### Claude CLI की विशिष्टताएँ

बंडल किया गया `claude-cli` बैकएंड Claude Code के मूल स्किल रिज़ॉल्वर को प्राथमिकता देता है। जब वर्तमान स्किल स्नैपशॉट में वास्तविक पथ वाला कम से कम एक चयनित स्किल होता है, तो OpenClaw `--plugin-dir` के माध्यम से अस्थायी Claude Code Plugin पास करता है और जोड़े गए सिस्टम प्रॉम्प्ट से डुप्लिकेट OpenClaw स्किल कैटलॉग हटा देता है। वास्तविक Plugin स्किल के बिना, OpenClaw प्रॉम्प्ट कैटलॉग को फ़ॉलबैक के रूप में बनाए रखता है। स्किल परिवेश/API कुंजी ओवरराइड रन के लिए चाइल्ड प्रक्रिया परिवेश पर अब भी लागू होते हैं।

Claude CLI का अपना गैर-इंटरैक्टिव अनुमति मोड है; OpenClaw Claude-विशिष्ट कॉन्फ़िगरेशन जोड़ने के बजाय उसे मौजूदा निष्पादन नीति से मैप करता है। OpenClaw द्वारा प्रबंधित Claude लाइव सत्रों के लिए प्रभावी निष्पादन नीति प्रामाणिक होती है: YOLO (`tools.exec.security: "full"` और `tools.exec.ask: "off"`) सामान्यतः Claude को `--permission-mode bypassPermissions` के साथ लॉन्च करता है, जबकि प्रतिबंधात्मक नीति उसे `--permission-mode default` के साथ लॉन्च करती है। रूट द्वारा चलाए गए Gateway भी `default` का उपयोग करते हैं, क्योंकि Claude Code रूट के लिए बायपास मोड अस्वीकार करता है; OpenClaw फिर भी कॉन्फ़िगर की गई निष्पादन नीति के अनुसार Claude के stdio टूल-नियंत्रण अनुरोधों का उत्तर देता है। प्रति-एजेंट `agents.list[].tools.exec` सेटिंग उस एजेंट के लिए वैश्विक `tools.exec` को ओवरराइड करती है। कच्चे बैकएंड आर्ग्युमेंट में अब भी `--permission-mode` शामिल हो सकता है, लेकिन लाइव Claude लॉन्च उस फ़्लैग को प्रभावी नीति और होस्ट प्रतिबंध से मेल खाने के लिए सामान्यीकृत करते हैं।

बैकएंड OpenClaw के `/think` स्तरों को Claude Code के मूल `--effort` फ़्लैग से भी मैप करता है: `minimal`/`low` -> `low`, `medium` -> `medium`, और `high`/`xhigh`/`max` सीधे पास किए जाते हैं। इससे सदस्यता-समर्थित Claude CLI और API-कुंजी मार्गों के लिए समर्थित Fable 5 प्रयास स्तर समान रहते हैं। `adaptive` कॉन्फ़िगर किए गए `--effort` फ़्लैग हटाता है और कोई प्रतिस्थापन नहीं देता, इसलिए Claude Code अपने परिवेश, सेटिंग और मॉडल डिफ़ॉल्ट से प्रभावी प्रयास निर्धारित करता है। `/think` द्वारा उत्पन्न CLI को प्रभावित करने से पहले, अन्य CLI बैकएंड के स्वामित्व वाले Plugin को समकक्ष argv मैपर घोषित करना होगा।

OpenClaw द्वारा `claude-cli` उपयोग किए जाने से पहले, Claude Code को उसी होस्ट पर लॉग इन होना चाहिए:

```bash
claude auth login
claude auth status --text
openclaw models auth login --provider anthropic --method cli --set-default
```

Docker इंस्टॉलेशन के लिए Claude Code को केवल होस्ट पर नहीं, बल्कि स्थायी कंटेनर होम के भीतर इंस्टॉल और लॉग इन होना चाहिए; [Docker में Claude CLI बैकएंड](/hi/install/docker#claude-cli-backend-in-docker) देखें।

`agents.defaults.cliBackends.claude-cli.command` को केवल तभी सेट करें, जब `claude` बाइनरी पहले से `PATH` पर न हो।

## सत्र

- यदि CLI सत्रों का समर्थन करता है, तो `sessionArg` (जैसे `--session-id`) सेट करें, या जब आईडी को कई फ़्लैग में रखना हो तब `sessionArgs` (प्लेसहोल्डर `{sessionId}`) सेट करें।
- यदि CLI अलग फ़्लैग वाला पुनः आरंभ उपकमांड उपयोग करता है, तो `resumeArgs` सेट करें (पुनः आरंभ करते समय `args` को बदलता है) और गैर-JSON पुनः आरंभ के लिए वैकल्पिक रूप से `resumeOutput` सेट करें।
- `sessionMode`:
  - `always`: हमेशा सत्र आईडी भेजें (यदि कोई संग्रहीत न हो, तो नया UUID)।
  - `existing`: केवल तभी सत्र आईडी भेजें, जब कोई पहले से संग्रहीत हो।
  - `none`: कभी भी सत्र आईडी न भेजें।
- `claude-cli` डिफ़ॉल्ट रूप से `liveSession: "claude-stdio"`, `output: "jsonl"` और `input: "stdin"` होता है, इसलिए बाद के टर्न सक्रिय रहने के दौरान लाइव Claude प्रक्रिया का पुनः उपयोग करते हैं, इसमें वे कस्टम कॉन्फ़िगरेशन भी शामिल हैं जिनमें ट्रांसपोर्ट फ़ील्ड नहीं हैं। यदि Gateway पुनः आरंभ होता है या निष्क्रिय प्रक्रिया बंद हो जाती है, तो OpenClaw संग्रहीत Claude सत्र आईडी से पुनः आरंभ करता है। पुनः आरंभ करने से पहले संग्रहीत सत्र आईडी को पठनीय प्रोजेक्ट ट्रांसक्रिप्ट के विरुद्ध सत्यापित किया जाता है; ट्रांसक्रिप्ट न मिलने पर `--resume` के अंतर्गत चुपचाप नया सत्र शुरू करने के बजाय बाइंडिंग साफ़ कर दी जाती है (इसे `reason=transcript-missing` के रूप में लॉग किया जाता है)।
- Claude लाइव सत्र सीमित JSONL आउटपुट सुरक्षा बनाए रखते हैं: प्रति टर्न 8 MiB और 20,000 कच्ची JSONL पंक्तियाँ।
- संग्रहीत CLI सत्र प्रदाता-स्वामित्व वाली निरंतरता हैं। स्वतः रीसेट डिफ़ॉल्ट रूप से अक्षम है; `/reset` और स्पष्ट दैनिक या निष्क्रिय `session.reset` नीतियाँ अब भी उन्हें समाप्त करती हैं।
- नए CLI सत्र सामान्यतः केवल OpenClaw के Compaction सारांश और Compaction के बाद के अंतिम भाग से दोबारा सीड होते हैं। Compaction से पहले अमान्य हुए छोटे सत्रों को पुनर्प्राप्त करने के लिए बैकएंड `reseedFromRawTranscriptWhenUncompacted: true` विकल्प चुन सकता है। कच्चे ट्रांसक्रिप्ट से दोबारा सीड करना सीमित रहता है और केवल सुरक्षित अमान्यकरणों तक सीमित होता है, जैसे अनुपस्थित CLI ट्रांसक्रिप्ट, अनाथ टूल-उपयोग अंतिम भाग, संदेश-नीति/सिस्टम-प्रॉम्प्ट/cwd/MCP परिवर्तन या सत्र-समाप्त पुनः प्रयास; प्रमाणीकरण प्रोफ़ाइल या क्रेडेंशियल-युग परिवर्तन कभी भी कच्चे ट्रांसक्रिप्ट इतिहास को दोबारा सीड नहीं करते।

क्रमबद्धता: `serialize: true` समान-लेन रन को क्रम में रखता है (अधिकांश CLI एक प्रदाता लेन पर क्रमबद्ध होते हैं)। चयनित प्रमाणीकरण पहचान बदलने पर OpenClaw संग्रहीत CLI सत्र का पुनः उपयोग भी बंद कर देता है, जिसमें बदली हुई प्रमाणीकरण प्रोफ़ाइल आईडी, स्थिर API कुंजी, स्थिर टोकन या CLI द्वारा उजागर की गई OAuth खाता पहचान शामिल है; केवल OAuth एक्सेस/रीफ़्रेश टोकन रोटेशन सत्र को समाप्त नहीं करता। यदि CLI के पास कोई स्थिर OAuth खाता आईडी नहीं है, तो OpenClaw उस CLI को अपनी पुनः आरंभ अनुमतियाँ लागू करने देता है।

## claude-cli सत्रों से फ़ॉलबैक प्रस्तावना

जब `claude-cli` प्रयास [`agents.defaults.model.fallbacks`](/hi/concepts/model-failover) में किसी गैर-CLI उम्मीदवार पर फ़ेलओवर होता है, तो OpenClaw अगले प्रयास को Claude Code के स्थानीय JSONL ट्रांसक्रिप्ट से प्राप्त संदर्भ प्रस्तावना के साथ सीड करता है (`~/.claude/projects/` के अंतर्गत, प्रत्येक कार्यस्थान के लिए कुंजीबद्ध)। इस सीड के बिना फ़ॉलबैक प्रदाता बिना संदर्भ के शुरू होता है, क्योंकि `claude-cli` रन के लिए OpenClaw का अपना सत्र ट्रांसक्रिप्ट खाली होता है।

- प्रस्तावना नवीनतम `/compact` सारांश या `compact_boundary` चिह्नक को प्राथमिकता देती है, फिर वर्ण बजट तक सीमा के बाद के सबसे हालिया संवाद क्रम जोड़ती है। सीमा से पहले के संवाद क्रम हटा दिए जाते हैं क्योंकि सारांश पहले ही उनका प्रतिनिधित्व करता है।
- प्रॉम्प्ट बजट को सटीक बनाए रखने के लिए टूल ब्लॉक को संक्षिप्त `(tool call: name)` और `(tool result: …)` संकेतों में समेकित किया जाता है; बहुत बड़े सारांश को छोटा करके `(truncated)` लेबल दिया जाता है।
- समान-प्रदाता `claude-cli` से `claude-cli` फ़ॉलबैक Claude के अपने `--resume` पर निर्भर करते हैं और प्रस्तावना को छोड़ देते हैं।
- सीड मौजूदा Claude सत्र-फ़ाइल पथ सत्यापन का पुनः उपयोग करता है, इसलिए मनमाने पथ नहीं पढ़े जा सकते।

## छवियाँ

यदि आपका CLI छवि पथ स्वीकार करता है, तो `imageArg` सेट करें:

```json5
imageArg: "--image",
imageMode: "repeat"
```

OpenClaw base64 छवियों को अस्थायी फ़ाइलों में लिखता है। यदि `imageArg` सेट है, तो वे पथ CLI आर्ग्युमेंट के रूप में दिए जाते हैं; अन्यथा OpenClaw फ़ाइल पथों को प्रॉम्प्ट में जोड़ता है (पथ इंजेक्शन), जो उन CLI के लिए काम करता है जो सादे पथों से स्थानीय फ़ाइलें स्वतः लोड करते हैं।

## इनपुट और आउटपुट

- `output: "text"` (डिफ़ॉल्ट) stdout को अंतिम प्रतिक्रिया मानता है।
- `output: "json"` JSON पार्स करके टेक्स्ट और सत्र आईडी निकालने का प्रयास करता है।
- `output: "jsonl"` JSONL स्ट्रीम पार्स करता है और उपलब्ध होने पर अंतिम एजेंट संदेश तथा सत्र पहचानकर्ता निकालता है।
- Gemini CLI JSON आउटपुट के लिए, जब `usage` अनुपस्थित या खाली हो, तब OpenClaw उत्तर टेक्स्ट को `response` से और उपयोग को `stats` से पढ़ता है। बंडल किया गया Gemini CLI डिफ़ॉल्ट `stream-json` का उपयोग करता है; पुराने `--output-format json` ओवरराइड अब भी JSON पार्सर का उपयोग करते हैं।

इनपुट मोड:

- `input: "arg"` (डिफ़ॉल्ट) प्रॉम्प्ट को अंतिम CLI आर्ग्युमेंट के रूप में देता है।
- `input: "stdin"` प्रॉम्प्ट को stdin के माध्यम से भेजता है।
- यदि प्रॉम्प्ट बहुत लंबा है और `maxPromptArgChars` सेट है, तो इसके बजाय stdin का उपयोग किया जाता है।

## Plugin-स्वामित्व वाले डिफ़ॉल्ट

CLI बैकएंड डिफ़ॉल्ट Plugin सतह का हिस्सा हैं:

- Plugins उन्हें `api.registerCliBackend(...)` के साथ पंजीकृत करते हैं।
- बैकएंड `id` मॉडल संदर्भों में प्रदाता उपसर्ग बन जाता है।
- `agents.defaults.cliBackends.<id>` में उपयोगकर्ता कॉन्फ़िगरेशन अब भी Plugin डिफ़ॉल्ट को ओवरराइड करता है।
- बैकएंड-विशिष्ट कॉन्फ़िगरेशन सफ़ाई वैकल्पिक `normalizeConfig` हुक के माध्यम से Plugin के स्वामित्व में रहती है।

Anthropic के स्वामित्व में `claude-cli` है और Google के स्वामित्व में `google-gemini-cli` है। OpenAI Codex एजेंट रन `openai/*` के माध्यम से Codex ऐप-सर्वर हार्नेस का उपयोग करते हैं; OpenClaw अब बंडल किया गया `codex-cli` बैकएंड पंजीकृत नहीं करता।

बंडल किया गया Anthropic Plugin `claude-cli` के लिए पंजीकरण करता है:

| कुंजी                   | मान                                                                                                                                                                                                         |
| --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `command`             | `claude`                                                                                                                                                                                                      |
| `args`                | `-p --output-format stream-json --include-partial-messages --verbose --setting-sources user --allowedTools mcp__openclaw__* --disallowedTools ScheduleWakeup,CronCreate,Bash(run_in_background:true),Monitor` |
| `output`              | `jsonl`                                                                                                                                                                                                       |
| `input`               | `stdin`                                                                                                                                                                                                       |
| `modelArg`            | `--model`                                                                                                                                                                                                     |
| `sessionArg`          | `--session-id`                                                                                                                                                                                                |
| `sessionMode`         | `always`                                                                                                                                                                                                      |
| `imageArg`            | `@`                                                                                                                                                                                                           |
| `imagePathScope`      | `workspace`                                                                                                                                                                                                   |
| `systemPromptFileArg` | `--append-system-prompt-file`                                                                                                                                                                                 |
| `systemPromptMode`    | `append`                                                                                                                                                                                                      |

बंडल किया गया Google Plugin `google-gemini-cli` के लिए पंजीकरण करता है:

| कुंजी                       | मान                                                                                  |
| ------------------------- | -------------------------------------------------------------------------------------- |
| `command`                 | `gemini`                                                                               |
| `args`                    | `--skip-trust --approval-mode auto_edit --output-format stream-json --prompt {prompt}` |
| `resumeArgs`              | समान, `--resume {sessionId}` के साथ                                                      |
| `output` / `resumeOutput` | `jsonl`                                                                                |
| `jsonlDialect`            | `gemini-stream-json`                                                                   |
| `imageArg`                | `@`                                                                                    |
| `imagePathScope`          | `workspace`                                                                            |
| `modelArg`                | `--model`                                                                              |
| `sessionMode`             | `existing`                                                                             |
| `sessionIdFields`         | `["session_id", "sessionId"]`                                                          |

पूर्वापेक्षा: स्थानीय Gemini CLI स्थापित होना चाहिए और `PATH` पर `gemini` के रूप में उपलब्ध होना चाहिए (`brew install gemini-cli` या `npm install -g @google/gemini-cli`)।

Gemini CLI आउटपुट संबंधी टिप्पणियाँ:

- डिफ़ॉल्ट `stream-json` पार्सर सहायक `message` इवेंट, टूल इवेंट, अंतिम `result` उपयोग और घातक Gemini त्रुटि इवेंट पढ़ता है।
- यदि आप Gemini आर्ग्युमेंट को `--output-format json` में ओवरराइड करते हैं, तो OpenClaw उस बैकएंड को वापस `output: "json"` में सामान्यीकृत करता है और JSON के `response` फ़ील्ड से उत्तर टेक्स्ट पढ़ता है।
- जब `usage` अनुपस्थित या खाली हो, तो उपयोग `stats` पर फ़ॉलबैक करता है; `stats.cached` को OpenClaw `cacheRead` में सामान्यीकृत किया जाता है, और यदि `stats.input` अनुपस्थित हो, तो इनपुट टोकन `stats.input_tokens - stats.cached` से प्राप्त किए जाते हैं।

केवल आवश्यकता होने पर डिफ़ॉल्ट ओवरराइड करें (सबसे सामान्यतः एक निरपेक्ष `command` पथ)।

## टेक्स्ट रूपांतरण ओवरले

जिन Plugins को छोटे प्रॉम्प्ट/संदेश संगतता शिम की आवश्यकता होती है, वे प्रदाता या CLI बैकएंड को बदले बिना द्विदिश टेक्स्ट रूपांतरण घोषित कर सकते हैं:

```typescript
api.registerTextTransforms({
  input: [{ from: /red basket/g, to: "blue basket" }],
  output: [{ from: /blue basket/g, to: "red basket" }],
});
```

`input` CLI को दिए गए सिस्टम प्रॉम्प्ट और उपयोगकर्ता प्रॉम्प्ट को पुनर्लिखता है। OpenClaw द्वारा अपने नियंत्रण चिह्नकों और चैनल डिलीवरी को संभालने से पहले `output` स्ट्रीम किए गए सहायक टेक्स्ट और पार्स किए गए अंतिम टेक्स्ट को पुनर्लिखता है; प्रदाता-समर्थित मॉडल कॉल के लिए, यह स्ट्रीम सुधार के बाद और टूल निष्पादन से पहले संरचित टूल-कॉल आर्ग्युमेंट में स्ट्रिंग मान भी पुनर्स्थापित करता है। अपरिष्कृत प्रदाता JSON खंड अपरिवर्तित रहते हैं; उपभोक्ताओं को संरचित आंशिक, समाप्ति या परिणाम पेलोड का उपयोग करना चाहिए।

प्रदाता-विशिष्ट JSONL इवेंट उत्सर्जित करने वाले CLI के लिए, उस बैकएंड के कॉन्फ़िगरेशन पर `jsonlDialect` सेट करें: Claude Code-संगत स्ट्रीम के लिए `claude-stream-json`, Gemini CLI `stream-json` इवेंट के लिए `gemini-stream-json`।

## मूल Compaction स्वामित्व

कुछ CLI बैकएंड ऐसा एजेंट चलाते हैं जो अपने ट्रांसक्रिप्ट का स्वयं Compaction करता है, इसलिए OpenClaw को उनके विरुद्ध अपना सुरक्षा-सारांशकर्ता नहीं चलाना चाहिए—ऐसा करने से बैकएंड के अपने Compaction में टकराव होता है और संवाद क्रम पूरी तरह विफल हो सकता है।

`claude-cli` का कोई हार्नेस एंडपॉइंट नहीं है (Claude Code आंतरिक रूप से Compaction करता है), इसलिए यह `ownsNativeCompaction: true` घोषित करता है और OpenClaw का Compaction पथ सत्र प्रविष्टि को अपरिवर्तित लौटाता है। OpenClaw रन के प्रभावी संदर्भ बजट को Claude Code के दस्तावेज़ीकृत [`CLAUDE_CODE_AUTO_COMPACT_WINDOW`](https://code.claude.com/docs/en/env-vars) के माध्यम से देता है, जिससे मूल स्वचालित Compaction कॉन्फ़िगर की गई Anthropic `contextTokens` सीमाओं के अनुरूप रहता है। Codex जैसे मूल-हार्नेस सत्र इसके बजाय अपने हार्नेस Compaction एंडपॉइंट पर रूट होते रहते हैं।

```typescript
api.registerCliBackend({ id: "my-cli", ownsNativeCompaction: true /* ... */ });
```

`ownsNativeCompaction` केवल ऐसे बैकएंड के लिए घोषित करें जो वास्तव में Compaction का स्वामी हो: उसे संदर्भ विंडो के निकट अपने ट्रांसक्रिप्ट को विश्वसनीय रूप से सीमित करना और पुनः आरंभ किए जा सकने वाला सत्र बनाए रखना चाहिए (जैसे `--resume` / `--session-id`), अन्यथा स्थगित सत्र बजट से अधिक रह सकता है।

## बंडल MCP ओवरले

CLI बैकएंड सीधे OpenClaw टूल कॉल प्राप्त नहीं करते, लेकिन कोई बैकएंड `bundleMcp: true` के साथ जनरेट किए गए MCP कॉन्फ़िगरेशन ओवरले को चुन सकता है। वर्तमान बंडल व्यवहार:

- `claude-cli`: जनरेट की गई सख्त MCP कॉन्फ़िगरेशन फ़ाइल।
- `google-gemini-cli`: जनरेट की गई Gemini सिस्टम सेटिंग्स फ़ाइल।

बंडल MCP सक्षम होने पर, OpenClaw:

- एक लूपबैक HTTP MCP सर्वर आरंभ करता है जो CLI प्रक्रिया को Gateway टूल उपलब्ध कराता है और केवल वर्तमान निष्पादन प्रयास के लिए सक्रिय प्रति-रन संदर्भ अनुदान (`OPENCLAW_MCP_TOKEN`) से प्रमाणीकृत होता है;
- चाइल्ड-प्रोसेस हेडर पर भरोसा करने के बजाय टूल पहुँच को Gateway द्वारा चयनित सत्र, खाते और चैनल संदर्भ से बाँधता है;
- वर्तमान कार्यस्थान के लिए सक्षम बंडल-MCP सर्वर लोड करता है और उन्हें किसी भी मौजूदा बैकएंड MCP कॉन्फ़िगरेशन/सेटिंग्स संरचना के साथ मर्ज करता है;
- स्वामी Plugin के बैकएंड-स्वामित्व वाले एकीकरण मोड का उपयोग करके लॉन्च कॉन्फ़िगरेशन को पुनर्लिखता है।

यदि कोई MCP सर्वर सक्षम नहीं है, तब भी किसी बैकएंड द्वारा बंडल MCP चुने जाने पर OpenClaw एक सख्त कॉन्फ़िगरेशन इंजेक्ट करता है, जिससे पृष्ठभूमि रन पृथक रहते हैं।

सत्र-स्कोप वाले बंडल MCP रनटाइम को सत्र के भीतर पुनः उपयोग के लिए कैश किया जाता है, फिर 10 मिनट निष्क्रिय रहने के बाद हटा दिया जाता है। प्रमाणीकरण जाँच, स्लग जनरेशन और Active Memory रिकॉल जैसे एकबारगी एम्बेडेड रन, रन समाप्त होने पर सफ़ाई का अनुरोध करते हैं ताकि stdio चाइल्ड और स्ट्रीम-योग्य HTTP/SSE स्ट्रीम रन से अधिक समय तक सक्रिय न रहें।

## रीसीड इतिहास सीमा

जब किसी नए CLI सत्र को पिछले OpenClaw ट्रांसक्रिप्ट से सीड किया जाता है (उदाहरण के लिए, `session_expired` पुनः प्रयास के बाद), तो रेंडर किए गए `<conversation_history>` ब्लॉक की सीमा तय होती है, ताकि रीसीड प्रॉम्प्ट अत्यधिक बड़े न हो जाएँ। डिफ़ॉल्ट सीमा 12,288 वर्ण (लगभग 3,000 टोकन) है।

इसके बजाय, Claude CLI बैकएंड इस सीमा को निर्धारित Claude संदर्भ विंडो के अनुसार समायोजित करते हैं: बड़ी संदर्भ विंडो को पिछले इतिहास का बड़ा अंश मिलता है, जिसकी एक निश्चित अधिकतम सीमा होती है; अन्य CLI बैकएंड सीमित डिफ़ॉल्ट बनाए रखते हैं। यह सीमा केवल रीसीड प्रॉम्प्ट के पिछले-इतिहास वाले ब्लॉक को नियंत्रित करती है।

## सीमाएँ

- OpenClaw टूल को सीधे कॉल नहीं किया जा सकता: OpenClaw, CLI बैकएंड प्रोटोकॉल में टूल कॉल इंजेक्ट नहीं करता। बैकएंड को Gateway टूल केवल तभी दिखाई देते हैं, जब वे `bundleMcp: true` को चुनते हैं।
- स्ट्रीमिंग बैकएंड-विशिष्ट है: कुछ बैकएंड JSONL स्ट्रीम करते हैं, जबकि अन्य बाहर निकलने तक बफ़र करते हैं।
- संरचित आउटपुट CLI के अपने JSON प्रारूप पर निर्भर करते हैं।

## समस्या निवारण

| लक्षण                  | समाधान                                                                    |
| --------------------- | ----------------------------------------------------------------- |
| CLI नहीं मिला          | `command` को पूर्ण पथ पर सेट करें।                                |
| गलत मॉडल नाम           | `provider/model` को CLI की मॉडल आईडी से मैप करने के लिए `modelAliases` का उपयोग करें। |
| सत्र की निरंतरता नहीं  | सुनिश्चित करें कि `sessionArg` सेट है और `sessionMode`, `none` नहीं है। |
| छवियाँ अनदेखी हुईं     | `imageArg` सेट करें और सत्यापित करें कि CLI फ़ाइल पथों का समर्थन करता है। |
