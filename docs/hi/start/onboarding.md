macOS ऐप का पहली बार चलने वाला प्रवाह: चुनें कि Gateway कहाँ चलता है, किसी
सत्यापित AI बैकएंड से कनेक्ट करें, अनुमतियाँ दें, और नियंत्रण एजेंट की अपनी
बूटस्ट्रैप प्रक्रिया को सौंप दें।
CLI ऑनबोर्डिंग और दोनों मार्गों की तुलना के लिए, [ऑनबोर्डिंग अवलोकन](/hi/start/onboarding-overview) देखें।

सुरक्षा विश्वास मॉडल:

- डिफ़ॉल्ट रूप से, OpenClaw एक व्यक्तिगत एजेंट है: एक विश्वसनीय ऑपरेटर सीमा।
- साझा/बहु-उपयोगकर्ता सेटअप को कड़ा सुरक्षित करना आवश्यक है: विश्वास सीमाएँ अलग करें, टूल की पहुँच न्यूनतम रखें, और [सुरक्षा](/hi/gateway/security) का पालन करें।
- स्थानीय ऑनबोर्डिंग नई कॉन्फ़िगरेशन को डिफ़ॉल्ट रूप से `tools.profile: "coding"` पर सेट करती है, ताकि नए सेटअप में अप्रतिबंधित `full` प्रोफ़ाइल के बिना फ़ाइल सिस्टम/रनटाइम टूल उपलब्ध रहें।
- यदि हुक/Webhook या अविश्वसनीय सामग्री के अन्य फ़ीड सक्षम हैं, तो आधुनिक और सशक्त मॉडल स्तर का उपयोग करें तथा टूल नीति/सैंडबॉक्सिंग को सख्त रखें।

</Step>
<Step title="स्थानीय बनाम रिमोट">
<Frame>
<img src="/assets/macos-onboarding/04-choose-gateway.png" alt="" />
</Frame>

**Gateway** कहाँ चलता है?

- **यह Mac (केवल स्थानीय):** ऑनबोर्डिंग प्रमाणीकरण कॉन्फ़िगर करती है और क्रेडेंशियल स्थानीय रूप से लिखती है।
- **रिमोट (SSH/Tailnet के माध्यम से):** ऑनबोर्डिंग स्थानीय प्रमाणीकरण कॉन्फ़िगर **नहीं** करती;
  क्रेडेंशियल Gateway होस्ट पर पहले से मौजूद होने चाहिए। रिमोट Gateway टोकन
  फ़ील्ड उस टोकन को संग्रहीत करता है जिसका उपयोग macOS ऐप उस Gateway से कनेक्ट होने के लिए करता है;
  मौजूदा `gateway.remote.token` SecretRef मान तब तक सुरक्षित रखे जाते हैं, जब तक आप
  उन्हें बदल नहीं देते।
- **बाद में कॉन्फ़िगर करें:** सेटअप छोड़ें और ऐप को अकॉन्फ़िगर रखा रहने दें।

<Tip>
**Gateway प्रमाणीकरण सुझाव:**

- लूपबैक बाइंड के लिए भी Gateway प्रमाणीकरण मोड डिफ़ॉल्ट रूप से `token` होता है, इसलिए स्थानीय WS क्लाइंट को प्रमाणीकरण करना होगा।
- `gateway.auth.mode: "none"` सेट करने से कोई भी स्थानीय प्रक्रिया कनेक्ट कर सकती है; इसका उपयोग केवल पूरी तरह विश्वसनीय मशीनों पर करें।
- बहु-मशीन पहुँच या गैर-लूपबैक बाइंड के लिए टोकन का उपयोग करें।

</Tip>
</Step>
<Step title="CLI">
  स्थानीय सेटअप npm, pnpm या bun के माध्यम से वैश्विक `openclaw` CLI इंस्टॉल करता है,
  जिसमें पहले npm को प्राथमिकता दी जाती है। Gateway के लिए स्वयं Node अनुशंसित
  रनटाइम बना रहता है। मौजूदा संगत इंस्टॉलेशन का पुनः उपयोग किया जाता है।
</Step>
<Step title="अपना AI कनेक्ट करें">
  ऐसा कनेक्टेड Gateway, जिसमें एजेंट मॉडल पहले से कॉन्फ़िगर है, इस
  पृष्ठ को पूरी तरह छोड़ देता है और सामान्य एजेंट UI खोलता है। OpenClaw और प्रदाता सेटअप
  केवल नए या अपूर्ण Gateway के लिए चलते हैं।

Gateway तैयार होने के बाद, ऑनबोर्डिंग आपके पास पहले से उपलब्ध AI पहुँच खोजती है:
Claude Code या Codex लॉगिन, `OPENAI_API_KEY` / `ANTHROPIC_API_KEY`, या किसी
पहुँच योग्य Ollama अथवा LM Studio सर्वर में पहले से इंस्टॉल किया गया
टूल-सक्षम मॉडल। पहचान Gateway होस्ट पर चलती है, उस स्थिति में भी जब macOS ऐप किसी
Linux Gateway से कनेक्ट होता है। सर्वोत्तम विकल्प को वास्तविक कम्प्लीशन से परखा जाता है और
उत्तर मिलने के बाद ही सहेजा जाता है; परीक्षण विफल होने पर ऐप अपने आप अगला विकल्प
आजमाता है और दिखाता है कि पिछला विकल्प क्यों विफल हुआ। यदि कई विकल्प मिलते हैं, तो
आगे बढ़ने से पहले आप उनके बीच स्विच कर सकते हैं। स्वचालित स्थानीय खोज कभी किसी
मॉडल को पुल या डाउनलोड नहीं करती।

जब Gateway होस्ट पर Claude CLI लॉगिन न हो, तब Claude सदस्यता का उपयोग करने के लिए,
Claude Code इंस्टॉल वाली किसी भी मशीन पर `claude setup-token` चलाएँ, फिर प्रिंट किए गए
टोकन को **Connect with an API key or token** के अंतर्गत **Anthropic setup-token**
के रूप में पेस्ट करें।

इंस्टॉल किए गए Gemini CLI, Antigravity, Pi और OpenCode CLI को संदर्भ के लिए दिखाया
जाता है, जब उन्हें पुनः उपयोग योग्य निर्देशित-सेटअप इनफ़रेंस मार्ग के रूप में नहीं चुना जा सकता।
Gemini और Antigravity टूल-रहित इनफ़रेंस जाँच लागू नहीं कर सकते। Pi और
OpenCode सेटअप इनफ़रेंस मार्गों के बजाय पूर्ण-एजेंट हार्नेस हैं; उनके
सेशन एकीकरणों के लिए अलग रनटाइम और Plugin सेटअप आवश्यक है।

आप प्रदाता के अपने OAuth या डिवाइस-पेयरिंग प्रवाह के माध्यम से भी साइन इन कर सकते हैं।
अंतर्निहित विकल्पों में OpenAI/ChatGPT, OpenRouter, GitHub Copilot, Google
Gemini CLI, xAI, MiniMax Global और CN, तथा Chutes शामिल हैं। यह सूची किसी
निश्चित ऐप सूची के बजाय Gateway के सक्रिय टेक्स्ट-इनफ़रेंस प्रदाता Plugins से आती है,
इसलिए कोई अन्य प्रदाता प्रदाता-विशिष्ट macOS कोड जोड़े बिना इसमें शामिल हो सकता है।

मैन्युअल कुंजी/टोकन चयनकर्ता उसी प्रदाता रजिस्ट्री का उपयोग करता है। प्रत्येक मार्ग में,
प्रदाता अपना प्रारंभिक मॉडल और कॉन्फ़िगरेशन उपलब्ध कराता है; OpenClaw अपने प्रमाणीकरण
प्रोफ़ाइल को संग्रहीत करने से पहले उसी लाइव परीक्षण से क्रेडेंशियल सत्यापित करता है। जब तक
एक बैकएंड सफल नहीं हो जाता, Next लॉक रहता है, इसलिए कार्यशील इनफ़रेंस के बिना
पहली एजेंट चैट शुरू नहीं हो सकती। उस लाइव जाँच के सफल होने के बाद, OpenClaw शेष
वर्कस्पेस, Gateway, चैनल और अन्य वैकल्पिक सुविधाएँ कॉन्फ़िगर करने में सहायता के लिए
उपलब्ध हो जाता है। जब OpenClaw विकल्पों की एक छोटी सूची प्रस्तुत करता है, तो ऐप
नेटिव विकल्प कार्ड दिखाता है; किसी एक को चुनने पर चयन भेजा जाता है, और **Skip for
now** हमेशा विकल्प को वैकल्पिक ही रहने देता है। OpenClaw बाद में
Settings → OpenClaw के अंतर्गत भी उपलब्ध रहता है।
</Step>
<Step title="स्मृतियाँ आयात करें (पहचान होने पर दिखाया जाता है)">
स्थानीय Gateway के लिए, ऑनबोर्डिंग समर्थित AI टूल की स्मृतियों हेतु Mac की जाँच करती है:
Claude Code ऑटो-मेमोरी, Codex समेकित स्मृतियाँ और Hermes मेमोरी फ़ाइलें।
कोई भी मिलने पर, यह पृष्ठ प्रत्येक स्रोत को उसकी स्मृति संख्या के साथ सूचीबद्ध करता है
और अनुक्रमित पुनर्स्मरण के लिए चयनित स्रोतों को एजेंट वर्कस्पेस में
`memory/imports/` के अंतर्गत आयात करने देता है। पहले से आयात की गई फ़ाइलें छोड़ दी जाती हैं और
जब आयात करने के लिए कुछ न हो तो यह पृष्ठ कभी दिखाई नहीं देता। इसे छोड़ना सुरक्षित है;
डैशबोर्ड का Memory import पृष्ठ बाद में प्रति-फ़ाइल नियंत्रण के साथ वही आयात उपलब्ध कराता है।
</Step>
<Step title="अनुमतियाँ">

<Frame caption="चुनें कि आप OpenClaw को कौन-सी अनुमतियाँ देना चाहते हैं">
<img src="/assets/macos-onboarding/05-permissions.png" alt="" />
</Frame>

ऑनबोर्डिंग इनके लिए TCC अनुमतियाँ माँगती है: Automation (AppleScript), Notifications, Accessibility, Screen Recording, Microphone, Speech Recognition, Camera और Location।

</Step>
<Step title="समाप्त करें">
  इनफ़रेंस सफल होने के बाद, शेष वैकल्पिक सेटअप OpenClaw संभालता है और
  आपको सामान्य एजेंट चैट पर भेज सकता है। अनुमति प्रक्रिया पूरी करने पर
  वही चैट खुलती है; ऐप OpenClaw से पहले वर्कस्पेस नहीं बनाता या कोई अलग
  एजेंट सेटअप वार्तालाप शुरू नहीं करता। एजेंट के पहले वास्तविक टर्न के दौरान
  Gateway होस्ट पर क्या होता है, इसके लिए
  [बूटस्ट्रैपिंग](/hi/start/bootstrapping) देखें।
</Step>
</Steps>
