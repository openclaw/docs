macOS ऐप का पहली बार चलने वाला प्रवाह: चुनें कि Gateway कहाँ चलता है, किसी
सत्यापित AI बैकएंड से कनेक्ट करें, अनुमतियाँ दें, और नियंत्रण एजेंट की अपनी
बूटस्ट्रैप प्रक्रिया को सौंपें।
CLI ऑनबोर्डिंग और दोनों तरीकों की तुलना के लिए, [ऑनबोर्डिंग अवलोकन](/hi/start/onboarding-overview) देखें।

सुरक्षा विश्वास मॉडल:

- डिफ़ॉल्ट रूप से, OpenClaw एक व्यक्तिगत एजेंट है: एक विश्वसनीय ऑपरेटर सीमा।
- साझा/बहु-उपयोगकर्ता सेटअप को सख्त सुरक्षा की आवश्यकता होती है: विश्वास सीमाएँ अलग करें, टूल एक्सेस न्यूनतम रखें, और [सुरक्षा](/hi/gateway/security) का पालन करें।
- स्थानीय ऑनबोर्डिंग नए कॉन्फ़िगरेशन को डिफ़ॉल्ट रूप से `tools.profile: "coding"` पर सेट करती है, ताकि नए सेटअप अप्रतिबंधित `full` प्रोफ़ाइल के बिना फ़ाइल-सिस्टम/रनटाइम टूल बनाए रखें।
- यदि हुक/Webhook या अन्य अविश्वसनीय सामग्री फ़ीड सक्षम हैं, तो एक सशक्त आधुनिक मॉडल स्तर का उपयोग करें और सख्त टूल नीति/सैंडबॉक्सिंग बनाए रखें।

</Step>
<Step title="स्थानीय बनाम रिमोट">
<Frame>
<img src="/assets/macos-onboarding/04-choose-gateway.png" alt="" />
</Frame>

**Gateway** कहाँ चलता है?

- **यह Mac (केवल स्थानीय):** ऑनबोर्डिंग प्रमाणीकरण कॉन्फ़िगर करती है और क्रेडेंशियल स्थानीय रूप से लिखती है।
- **रिमोट (SSH/Tailnet के माध्यम से):** ऑनबोर्डिंग स्थानीय प्रमाणीकरण कॉन्फ़िगर **नहीं** करती;
  क्रेडेंशियल पहले से Gateway होस्ट पर मौजूद होने चाहिए। रिमोट Gateway टोकन
  फ़ील्ड उस टोकन को संग्रहीत करती है जिसका उपयोग macOS ऐप उस Gateway से कनेक्ट करने के लिए करता है;
  मौजूदा `gateway.remote.token` SecretRef मान तब तक सुरक्षित रखे जाते हैं, जब तक आप
  उन्हें बदल नहीं देते।
- **बाद में कॉन्फ़िगर करें:** सेटअप छोड़ दें और ऐप को बिना कॉन्फ़िगर किए रहने दें।

<Tip>
**Gateway प्रमाणीकरण सुझाव:**

- Gateway प्रमाणीकरण मोड लूपबैक बाइंड के लिए भी डिफ़ॉल्ट रूप से `token` होता है, इसलिए स्थानीय WS क्लाइंट को प्रमाणीकरण करना आवश्यक है।
- `gateway.auth.mode: "none"` सेट करने से कोई भी स्थानीय प्रक्रिया कनेक्ट कर सकती है; इसका उपयोग केवल पूर्णतः विश्वसनीय मशीनों पर करें।
- एकाधिक मशीनों से एक्सेस या गैर-लूपबैक बाइंड के लिए टोकन का उपयोग करें।

</Tip>
</Step>
<Step title="CLI">
  स्थानीय सेटअप npm, pnpm, या bun के माध्यम से वैश्विक `openclaw` CLI इंस्टॉल करता है,
  जिसमें पहले npm को प्राथमिकता दी जाती है। Gateway के लिए स्वयं Node अनुशंसित रनटाइम
  बना रहता है। मौजूदा संगत इंस्टॉलेशन का पुनः उपयोग किया जाता है।
</Step>
<Step title="अपने AI को कनेक्ट करें">
  ऐसा कनेक्टेड Gateway जिसमें एजेंट मॉडल पहले से कॉन्फ़िगर है, इस
  पृष्ठ को पूरी तरह छोड़ देता है और सामान्य एजेंट UI खोलता है। OpenClaw और प्रदाता सेटअप
  केवल नए या अपूर्ण Gateway के लिए चलते हैं।

Gateway तैयार होने पर, ऑनबोर्डिंग आपके पास पहले से उपलब्ध AI एक्सेस खोजती है:
Claude Code या Codex लॉगिन, `OPENAI_API_KEY` / `ANTHROPIC_API_KEY`, या कम से कम
16K मापे गए प्रभावी कॉन्टेक्स्ट वाला टूल-सक्षम मॉडल, जो पहले से किसी पहुँच योग्य
Ollama या LM Studio सर्वर में इंस्टॉल हो। पहचान Gateway होस्ट पर होती है,
उस स्थिति में भी जब macOS ऐप किसी Linux Gateway से कनेक्ट होता है। सर्वोत्तम
विकल्प को वास्तविक कम्प्लीशन के साथ परखा जाता है और उत्तर देने के बाद ही
सहेजा जाता है; परीक्षण विफल होने पर ऐप स्वचालित रूप से अगला विकल्प आज़माता है
और दिखाता है कि पिछला विकल्प क्यों विफल हुआ। यदि कई विकल्प मिलते हैं, तो आगे
बढ़ने से पहले आप उनके बीच स्विच कर सकते हैं। स्वचालित स्थानीय खोज कभी भी कोई
मॉडल पुल या डाउनलोड नहीं करती।

जब Gateway होस्ट पर Claude CLI लॉगिन न हो, तब Claude सदस्यता का उपयोग करने के लिए,
Claude Code इंस्टॉल वाली किसी भी मशीन पर `claude setup-token` चलाएँ, फिर प्रिंट किया गया
टोकन **Connect with an API key or token** के अंतर्गत **Anthropic setup-token**
के रूप में पेस्ट करें।

इंस्टॉल किए गए Gemini CLI, Antigravity, Pi, और OpenCode CLI को संदर्भ के लिए
दिखाया जाता है, जब उन्हें पुनः उपयोग योग्य निर्देशित-सेटअप इन्फ़रेंस मार्ग के रूप में
नहीं चुना जा सकता। Gemini और Antigravity टूल-मुक्त इन्फ़रेंस जाँच को लागू नहीं कर सकते।
Pi और OpenCode सेटअप इन्फ़रेंस मार्गों के बजाय संपूर्ण-एजेंट हार्नेस हैं; उनके
सेशन एकीकरण के लिए अलग रनटाइम और Plugin सेटअप आवश्यक है।

आप प्रदाता के अपने OAuth या डिवाइस-पेयरिंग प्रवाह के माध्यम से भी साइन इन कर सकते हैं।
अंतर्निहित विकल्पों में OpenAI/ChatGPT, OpenRouter, GitHub Copilot, Google
Gemini CLI, xAI, MiniMax Global और CN, तथा Chutes शामिल हैं। यह सूची किसी निश्चित
ऐप सूची के बजाय Gateway के सक्रिय टेक्स्ट-इन्फ़रेंस प्रदाता Plugin से आती है,
इसलिए कोई अन्य प्रदाता प्रदाता-विशिष्ट macOS कोड जोड़े बिना इसका विकल्प चुन सकता है।

मैन्युअल कुंजी/टोकन चयनकर्ता उसी प्रदाता रजिस्ट्री का उपयोग करता है। प्रत्येक मार्ग में,
प्रदाता अपना आरंभिक मॉडल और कॉन्फ़िगरेशन उपलब्ध कराता है; OpenClaw अपनी प्रमाणीकरण
प्रोफ़ाइल संग्रहीत करने से पहले उसी लाइव परीक्षण से क्रेडेंशियल सत्यापित करता है। जब तक
एक बैकएंड सफल नहीं हो जाता, Next लॉक रहता है, इसलिए कार्यशील इन्फ़रेंस के बिना
पहली एजेंट चैट शुरू नहीं हो सकती। वह लाइव जाँच सफल होने के बाद, शेष कार्यस्थान,
Gateway, चैनल और अन्य वैकल्पिक सुविधाओं को कॉन्फ़िगर करने में सहायता के लिए OpenClaw
उपलब्ध हो जाता है। जब OpenClaw विकल्पों की छोटी सूची प्रस्तुत करता है, तो ऐप
नेटिव विकल्प कार्ड दिखाता है; किसी विकल्प को चुनने पर चयन भेजा जाता है, और **Skip for
now** विकल्प को हमेशा वैकल्पिक ही रखता है। OpenClaw बाद में
Settings → OpenClaw के अंतर्गत भी उपलब्ध रहता है।
</Step>
<Step title="स्मृतियाँ आयात करें (पहचान होने पर दिखाया जाता है)">
स्थानीय Gateway के लिए, ऑनबोर्डिंग समर्थित AI टूल की स्मृतियों के लिए Mac की जाँच करती है:
Claude Code ऑटो-मेमोरी, Codex समेकित स्मृतियाँ, और Hermes मेमोरी
फ़ाइलें। कोई भी मिलने पर, यह पृष्ठ प्रत्येक स्रोत को उसकी स्मृति संख्या के साथ सूचीबद्ध करता है
और आपको अनुक्रमित पुनर्स्मरण के लिए चयनित स्रोतों को एजेंट कार्यस्थान में
`memory/imports/` के अंतर्गत आयात करने देता है। पहले से आयात की गई फ़ाइलें छोड़ दी जाती हैं, और
जब आयात करने के लिए कुछ नहीं होता तो यह पृष्ठ कभी दिखाई नहीं देता। इसे छोड़ना सुरक्षित है;
डैशबोर्ड का Memory import पृष्ठ बाद में प्रति-फ़ाइल नियंत्रण के साथ यही आयात उपलब्ध कराता है।
</Step>
<Step title="अनुमतियाँ">

<Frame caption="चुनें कि आप OpenClaw को कौन-सी अनुमतियाँ देना चाहते हैं">
<img src="/assets/macos-onboarding/05-permissions.png" alt="" />
</Frame>

ऑनबोर्डिंग इनके लिए TCC अनुमतियों का अनुरोध करती है: Automation (AppleScript), Notifications, Accessibility, Screen Recording, Microphone, Speech Recognition, Camera, और Location।

</Step>
<Step title="समाप्त करें">
  इन्फ़रेंस सफल होने के बाद, शेष वैकल्पिक सेटअप OpenClaw संभालता है और
  आपको सामान्य एजेंट चैट पर भेज सकता है। अनुमति चरण-दर्शन पूरा करने पर
  वही चैट खुलती है; ऐप OpenClaw से पहले कोई कार्यस्थान नहीं बनाता या अलग
  एजेंट सेटअप वार्तालाप शुरू नहीं करता। एजेंट के पहले वास्तविक टर्न के दौरान
  Gateway होस्ट पर क्या होता है, इसके लिए
  [बूटस्ट्रैपिंग](/hi/start/bootstrapping) देखें।
</Step>
</Steps>
