OpenClaw में दो ऑनबोर्डिंग पथ हैं। दोनों प्रमाणीकरण, Gateway, और
वैकल्पिक चैट चैनल कॉन्फ़िगर करते हैं — अंतर बस यह है कि आप सेटअप से कैसे इंटरैक्ट करते हैं।

## मुझे कौन-सा पथ इस्तेमाल करना चाहिए?

|                | CLI ऑनबोर्डिंग                         | macOS ऐप ऑनबोर्डिंग      |
| -------------- | -------------------------------------- | ------------------------- |
| **प्लैटफ़ॉर्म**  | macOS, Linux, Windows (नेटिव या WSL2) | केवल macOS                |
| **इंटरफ़ेस**  | टर्मिनल विज़ार्ड                        | ऐप में निर्देशित UI      |
| **इसके लिए श्रेष्ठ**   | सर्वर, हेडलेस, पूर्ण नियंत्रण        | डेस्कटॉप Mac, विज़ुअल सेटअप |
| **ऑटोमेशन** | स्क्रिप्ट के लिए `--non-interactive`        | केवल मैनुअल               |
| **कमांड**    | `openclaw onboard`                     | ऐप लॉन्च करें            |

अधिकांश उपयोगकर्ताओं को **CLI ऑनबोर्डिंग** से शुरू करना चाहिए — यह हर जगह काम करता है और
आपको सबसे अधिक नियंत्रण देता है।

## ऑनबोर्डिंग क्या कॉन्फ़िगर करती है

आप चाहे जो भी पथ चुनें, ऑनबोर्डिंग यह सेट अप करती है:

1. **मॉडल प्रदाता और प्रमाणीकरण** — आपके चुने हुए प्रदाता के लिए API कुंजी, OAuth, या सेटअप टोकन
2. **वर्कस्पेस** — एजेंट फ़ाइलों, बूटस्ट्रैप टेम्पलेटों, और मेमोरी के लिए डायरेक्टरी
3. **Gateway** — पोर्ट, बाइंड पता, प्रमाणीकरण मोड
4. **चैनल** (वैकल्पिक) — अंतर्निहित और बंडल किए गए चैट चैनल जैसे
   iMessage, Discord, Feishu, Google Chat, Mattermost, Microsoft Teams,
   Telegram, WhatsApp, और अन्य
5. **डेमन** (वैकल्पिक) — पृष्ठभूमि सेवा ताकि Gateway अपने-आप शुरू हो

## CLI ऑनबोर्डिंग

किसी भी टर्मिनल में चलाएँ:

```bash
openclaw onboard
```

एक ही चरण में पृष्ठभूमि सेवा भी इंस्टॉल करने के लिए `--install-daemon` जोड़ें।

पूर्ण संदर्भ: [ऑनबोर्डिंग (CLI)](/hi/start/wizard)
CLI कमांड दस्तावेज़: [`openclaw onboard`](/hi/cli/onboard)

## macOS ऐप ऑनबोर्डिंग

OpenClaw ऐप खोलें। फ़र्स्ट-रन विज़ार्ड आपको विज़ुअल इंटरफ़ेस के साथ उन्हीं चरणों से गुज़ारता है।

पूर्ण संदर्भ: [ऑनबोर्डिंग (macOS ऐप)](/hi/start/onboarding)

## कस्टम या असूचीबद्ध प्रदाता

यदि आपका प्रदाता ऑनबोर्डिंग में सूचीबद्ध नहीं है, तो **कस्टम प्रदाता** चुनें और
दर्ज करें:

- API संगतता मोड (OpenAI-संगत, Anthropic-संगत, या ऑटो-डिटेक्ट)
- बेस URL और API कुंजी
- मॉडल ID और वैकल्पिक उपनाम

कई कस्टम एंडपॉइंट साथ-साथ रह सकते हैं — प्रत्येक को अपना एंडपॉइंट ID मिलता है।
