OpenClaw में टर्मिनल और macOS ऐप ऑनबोर्डिंग उपलब्ध हैं। दोनों पहले इनफ़रेंस स्थापित करते हैं:
वे मौजूदा AI एक्सेस का पता लगाते हैं, लाइव कम्प्लीशन आवश्यक बनाते हैं, और उसके बाद ही
शेष सेटअप कॉन्फ़िगर करने के लिए OpenClaw शुरू करते हैं। यदि कोई पहुँच योग्य, कॉन्फ़िगर किया गया Gateway
हो, जिसके डिफ़ॉल्ट एजेंट में पहले से कॉन्फ़िगर किया गया मॉडल हो, तो ऑनबोर्डिंग छोड़ दी जाती है और
सामान्य एजेंट UI खुल जाता है। विस्तृत सेटअप के लिए टर्मिनल फ़्लो पूर्ण क्लासिक विज़ार्ड भी
प्रदान करता है।

## मुझे कौन-सा मार्ग उपयोग करना चाहिए?

|                | CLI ऑनबोर्डिंग                         | macOS ऐप ऑनबोर्डिंग           |
| -------------- | -------------------------------------- | ------------------------------ |
| **प्लेटफ़ॉर्म**  | macOS, Linux, Windows (नेटिव या WSL2) | केवल macOS                     |
| **इंटरफ़ेस**  | इनफ़रेंस सेटअप, फिर OpenClaw         | इनफ़रेंस सेटअप, फिर OpenClaw |
| **इनके लिए सर्वोत्तम**   | सर्वर, हेडलेस, पूर्ण नियंत्रण        | डेस्कटॉप Mac, विज़ुअल सेटअप      |
| **ऑटोमेशन** | स्क्रिप्ट के लिए `--non-interactive`        | केवल मैन्युअल                    |
| **कमांड**    | `openclaw onboard`                     | ऐप लॉन्च करें                 |

अधिकांश उपयोगकर्ताओं को **CLI ऑनबोर्डिंग** से शुरू करना चाहिए — यह हर जगह काम करती है और
आपको सर्वाधिक नियंत्रण देती है।

## ऑनबोर्डिंग क्या कॉन्फ़िगर करती है

निर्देशित इनफ़रेंस चरण केवल इन्हें स्थापित करता है:

1. **मॉडल प्रदाता और प्रमाणीकरण** — पता लगाया गया एक्सेस या सत्यापित प्रदाता साइन-इन,
   API कुंजी या टोकन
2. **सत्यापित इनफ़रेंस** — डिफ़ॉल्ट एजेंट के प्रभावी
   मॉडल पर वास्तविक कम्प्लीशन

वह कम्प्लीशन सफल होने के बाद, OpenClaw कार्यस्थान, Gateway,
Gateway सेवा, चैनल, एजेंट, Plugin और अन्य वैकल्पिक सुविधाएँ कॉन्फ़िगर कर सकता है।

क्लासिक CLI विज़ार्ड इसके अतिरिक्त इन्हें कॉन्फ़िगर कर सकता है:

1. **चैनल** (वैकल्पिक) — अंतर्निर्मित और बंडल किए गए चैट चैनल, जैसे
   Discord, Feishu, Google Chat, iMessage, Mattermost, Microsoft Teams,
   Telegram, WhatsApp और अन्य
2. **उन्नत Gateway नियंत्रण** — रिमोट मोड, नेटवर्क सेटिंग और डेमन विकल्प

## CLI ऑनबोर्डिंग

किसी भी टर्मिनल में चलाएँ:

```bash
openclaw onboard
```

निर्देशित फ़्लो मौजूदा AI एक्सेस का पता लगाता है, क्रम से उम्मीदवारों का लाइव परीक्षण
करता है और विफलता पर अगले विकल्प पर चला जाता है। यदि पता लगाने के सभी प्रयास समाप्त हो जाएँ, तो यह पहले OpenAI,
Anthropic, xAI (Grok), Google और OpenRouter दिखाता है। **More…** में
शेष प्रदाता, प्रदाता समूहों में होते हैं और दूसरे मेन्यू में क्षेत्र, प्लान तथा समर्थित
ब्राउज़र, डिवाइस, API-कुंजी या टोकन विधियाँ होती हैं। यह मॉडल
और क्रेडेंशियल केवल सफल कम्प्लीशन के बाद सहेजता है, फिर कार्यस्थान,
Gateway, चैनल, एजेंट, Plugin और अन्य वैकल्पिक सुविधाएँ कॉन्फ़िगर करने के लिए OpenClaw शुरू
करता है। **Skip for now** OpenClaw शुरू किए बिना बाहर निकलता है। फ़्लो के भीतर
क्लासिक फ़्लो को सौंपने की सुविधा नहीं है; इसके बजाय क्लासिक विज़ार्ड चाहिए तो बाहर निकलें और `openclaw onboard --classic`
चलाएँ।

इनफ़रेंस सफल होने के बाद, OpenClaw चैनल सेटअप को मास्क किए गए टर्मिनल
विज़ार्ड को सौंप सकता है। यह निर्देशित या क्लासिक प्रदाता सेटअप नहीं खोलता; मॉडल प्रदाता या उसके प्रमाणीकरण को
बदलने के लिए OpenClaw से बाहर निकलें और `openclaw onboard` चलाएँ।

विस्तृत मॉडल/प्रमाणीकरण, चैनल, स्किल,
रिमोट Gateway या आयात सेटअप के लिए `openclaw onboard --classic` उपयोग करें। `--install-daemon` जोड़ने पर
क्लासिक फ़्लो भी चुना जाता है और पृष्ठभूमि सेवा एक ही चरण में इंस्टॉल हो जाती है। संवादात्मक गैर-इनफ़रेंस सेटअप और सुधार के लिए `openclaw
openclaw` उपयोग करें। `openclaw
onboard --modern` एक संगतता उपनाम है, जो उसी लाइव-इनफ़रेंस
गेट का उपयोग करता है।

## macOS ऐप ऑनबोर्डिंग

OpenClaw ऐप खोलें। यदि उसका कॉन्फ़िगर किया गया स्थानीय या रिमोट Gateway पहुँच योग्य
है और डिफ़ॉल्ट एजेंट में पहले से कॉन्फ़िगर किया गया मॉडल है, तो ऐप ऑनबोर्डिंग
और OpenClaw को छोड़कर तुरंत सामान्य एजेंट UI खोलता है।

नए या अधूरे Gateway के लिए, पहली बार चलने वाला फ़्लो मौजूदा AI
एक्सेस (Claude Code, Codex या API कुंजियों) का पता लगाता है, सर्वोत्तम
विकल्प का लाइव परीक्षण करता है और उसे केवल वास्तविक उत्तर मिलने के बाद सहेजता है — आवश्यकता होने पर स्वतः अगले विकल्प पर जाता है और
कुछ न मिलने पर सत्यापित मैन्युअल API-कुंजी चरण प्रदान करता है। संवेदनशील
क्रेडेंशियल के लिए मास्क किया गया इनपुट उपयोग होता है। इनफ़रेंस सफल होने पर OpenClaw शुरू होता है और
शेष कॉन्फ़िगरेशन में सहायता करता है।

सेटअप के बाद Gemini CLI सामान्य एजेंटों के लिए उपलब्ध रहता है, लेकिन इसे
इस इनफ़रेंस गेट के लिए प्रस्तुत नहीं किया जाता, क्योंकि यह टूल-रहित प्रोब लागू नहीं कर सकता।

पूर्ण संदर्भ: [ऑनबोर्डिंग (macOS ऐप)](/hi/start/onboarding)

## कस्टम या असूचीबद्ध प्रदाता

यदि आपका प्रदाता सूचीबद्ध नहीं है, तो `openclaw onboard --classic` चलाएँ, **Custom Provider**
चुनें और ये दर्ज करें:

- एंडपॉइंट संगतता: OpenAI-संगत (`/chat/completions`), OpenAI Responses-संगत (`/responses`), Anthropic-संगत (`/messages`) या अज्ञात (तीनों की जाँच करके स्वतः पता लगाता है)
- बेस URL और API कुंजी (यदि एंडपॉइंट को इसकी आवश्यकता नहीं है, तो API कुंजी वैकल्पिक है)
- मॉडल ID और वैकल्पिक मॉडल उपनाम

कई कस्टम एंडपॉइंट एक साथ मौजूद रह सकते हैं — प्रत्येक को अपनी एंडपॉइंट ID मिलती है।
