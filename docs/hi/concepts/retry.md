## लक्ष्य

- प्रत्येक HTTP अनुरोध के लिए पुनः प्रयास करें, बहु-चरणीय प्रवाह के लिए नहीं।
- केवल वर्तमान चरण का पुनः प्रयास करके क्रम बनाए रखें।
- गैर-आइडेम्पोटेंट संक्रियाओं के दोहराव से बचें।

## डिफ़ॉल्ट

| सेटिंग                  | डिफ़ॉल्ट  |
| ----------------------- | --------- |
| प्रयास                  | 3         |
| अधिकतम विलंब सीमा       | 30000 ms  |
| जिटर                    | 0.1 (10%) |
| Telegram न्यूनतम विलंब | 400 ms    |
| Discord न्यूनतम विलंब  | 500 ms    |

## व्यवहार

### मॉडल प्रदाता

- OpenClaw सामान्य छोटे पुनः प्रयासों को प्रदाता SDK द्वारा संभालने देता है।
- Anthropic और OpenAI जैसे Stainless-आधारित SDK के लिए, पुनः प्रयास योग्य प्रतिक्रियाओं (`408`, `409`, `429`, और `5xx`) में `retry-after-ms` या `retry-after` शामिल हो सकता है। जब यह प्रतीक्षा 60 सेकंड से अधिक होती है, तो OpenClaw `x-should-retry: false` अंतःक्षेपित करता है, ताकि SDK त्रुटि को तुरंत उजागर करे और मॉडल फ़ेलओवर किसी अन्य प्रमाणीकरण प्रोफ़ाइल या फ़ॉलबैक मॉडल पर जा सके।
- सीमा को `OPENCLAW_SDK_RETRY_MAX_WAIT_SECONDS=<seconds>` से ओवरराइड करें। SDK को लंबी `Retry-After` प्रतीक्षाओं का आंतरिक रूप से पालन करने देने के लिए इसे `0`, `false`, `off`, `none`, या `disabled` पर सेट करें।

### Discord

- दर-सीमा त्रुटियों (HTTP 429), अनुरोध टाइमआउट, HTTP 5xx प्रतिक्रियाओं और DNS लुकअप विफलताओं, कनेक्शन रीसेट, सॉकेट बंद होने तथा फ़ेच विफलताओं जैसी अस्थायी ट्रांसपोर्ट विफलताओं पर पुनः प्रयास करता है।
- उपलब्ध होने पर Discord `retry_after` का उपयोग करता है, अन्यथा एक्सपोनेंशियल बैकऑफ़ का।

### Telegram

- अस्थायी त्रुटियों (429, टाइमआउट, कनेक्ट/रीसेट/बंद, अस्थायी रूप से अनुपलब्ध) पर पुनः प्रयास करता है।
- उपलब्ध होने पर `retry_after` का उपयोग करता है, अन्यथा एक्सपोनेंशियल बैकऑफ़ का।
- HTML/Markdown पार्सिंग त्रुटियों पर पुनः प्रयास नहीं किया जाता; पहले प्रयास में वे सादे टेक्स्ट पर फ़ॉलबैक करती हैं।

## कॉन्फ़िगरेशन

`~/.openclaw/openclaw.json` में प्रत्येक प्रदाता के लिए पुनः प्रयास नीति सेट करें:

```json5
{
  channels: {
    telegram: {
      retry: {
        attempts: 3,
        minDelayMs: 400,
        maxDelayMs: 30000,
        jitter: 0.1,
      },
    },
    discord: {
      retry: {
        attempts: 3,
        minDelayMs: 500,
        maxDelayMs: 30000,
        jitter: 0.1,
      },
    },
  },
}
```

## टिप्पणियाँ

- पुनः प्रयास प्रत्येक अनुरोध (संदेश भेजना, मीडिया अपलोड, प्रतिक्रिया, पोल, स्टिकर) पर लागू होते हैं।
- संयुक्त प्रवाह पूर्ण हो चुके चरणों का पुनः प्रयास नहीं करते।
