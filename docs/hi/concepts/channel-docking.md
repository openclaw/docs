चैनल डॉकिंग एक OpenClaw सत्र के लिए कॉल फ़ॉरवर्डिंग है। यह वही
वार्तालाप संदर्भ बनाए रखता है, लेकिन उस सत्र के भावी उत्तरों को पहुँचाने का
स्थान बदल देता है। डॉकिंग केवल सीधे चैट से काम करता है; यह समूह
चैट से नहीं चलता।

## उदाहरण

Alice Telegram और Discord पर OpenClaw को संदेश भेज सकती हैं:

```json5
{
  session: {
    identityLinks: {
      alice: ["telegram:123", "discord:456"],
    },
  },
}
```

यदि Alice इसे Telegram के सीधे चैट से भेजती हैं:

```text
/dock_discord
```

OpenClaw वर्तमान सत्र संदर्भ बनाए रखता है और उत्तर का मार्ग बदल देता है:

| डॉकिंग से पहले               | `/dock_discord` के बाद       |
| ---------------------------- | --------------------------- |
| उत्तर Telegram `123` पर जाते हैं | उत्तर Discord `456` पर जाते हैं |

सत्र को फिर से नहीं बनाया जाता। प्रतिलेख इतिहास उसी
सत्र से जुड़ा रहता है।

## इसका उपयोग क्यों करें

डॉकिंग का उपयोग तब करें, जब कोई कार्य एक चैट ऐप में शुरू होता है, लेकिन अगले उत्तर
किसी अन्य स्थान पर पहुँचने चाहिए।

सामान्य प्रवाह:

1. Telegram से एजेंट कार्य शुरू करें।
2. Discord पर जाएँ, जहाँ आप कार्य का समन्वय कर रहे हैं।
3. Telegram के सीधे चैट से `/dock_discord` भेजें।
4. वही OpenClaw सत्र बनाए रखें, लेकिन भावी उत्तर Discord में प्राप्त करें।

## आवश्यक कॉन्फ़िगरेशन

डॉकिंग के लिए `session.identityLinks` आवश्यक है। स्रोत प्रेषक और लक्षित पीयर
एक ही पहचान समूह में होने चाहिए:

```json5
{
  session: {
    identityLinks: {
      alice: ["telegram:123", "discord:456", "slack:U123"],
    },
  },
}
```

ये मान चैनल-उपसर्ग वाले पीयर आईडी हैं:

| मान          | अर्थ                      |
| -------------- | ---------------------------- |
| `telegram:123` | Telegram प्रेषक आईडी `123`     |
| `discord:456`  | Discord सीधे पीयर की आईडी `456` |
| `slack:U123`   | Slack उपयोगकर्ता आईडी `U123`         |

कैनोनिकल कुंजी (ऊपर `alice`) केवल साझा पहचान समूह का नाम है। डॉक
कमांड यह साबित करने के लिए चैनल-उपसर्ग वाले मानों का उपयोग करते हैं कि स्रोत प्रेषक और
लक्षित पीयर एक ही व्यक्ति हैं।

## कमांड

OpenClaw नेटिव कमांड का समर्थन करने वाले प्रत्येक लोड किए गए चैनल Plugin के लिए एक
`/dock-<channel>` कमांड जनरेट करता है, इसलिए Plugin जोड़े जाने पर सूची बढ़ती जाती है। वे बंडल किए गए
Plugin जो वर्तमान में इसका समर्थन करते हैं:

| लक्षित चैनल | कमांड            | उपनाम              |
| -------------- | ------------------ | ------------------ |
| Discord        | `/dock-discord`    | `/dock_discord`    |
| Mattermost     | `/dock-mattermost` | `/dock_mattermost` |
| Slack          | `/dock-slack`      | `/dock_slack`      |
| Telegram       | `/dock-telegram`   | `/dock_telegram`   |

अंडरस्कोर वाला रूप Telegram जैसे इंटरफ़ेस पर नेटिव कमांड का नाम भी है,
जो स्लैश कमांड को सीधे उपलब्ध कराते हैं।

## क्या बदलता है

डॉकिंग सक्रिय सत्र के डिलीवरी फ़ील्ड अपडेट करता है:

| सत्र फ़ील्ड   | `/dock_discord` के बाद उदाहरण            |
| --------------- | ---------------------------------------- |
| `lastChannel`   | `discord`                                |
| `lastTo`        | `456`                                    |
| `lastAccountId` | लक्षित चैनल खाता, या `default` |

वे फ़ील्ड सत्र स्टोर में स्थायी रूप से सहेजे जाते हैं और उस सत्र के बाद के उत्तरों की
डिलीवरी में उपयोग किए जाते हैं।

## क्या नहीं बदलता

डॉकिंग यह नहीं करता:

- चैनल खाते बनाना
- नया Discord, Telegram, Slack या Mattermost बॉट कनेक्ट करना
- किसी उपयोगकर्ता को पहुँच प्रदान करना
- चैनल अनुमति-सूचियों या DM नीतियों को दरकिनार करना
- प्रतिलेख इतिहास को किसी अन्य सत्र में ले जाना
- असंबंधित उपयोगकर्ताओं को एक सत्र साझा करवाना

यह केवल वर्तमान सत्र का डिलीवरी मार्ग बदलता है।

## समस्या निवारण

**कमांड बताता है कि प्रेषक लिंक नहीं है।**

वर्तमान प्रेषक और लक्षित पीयर, दोनों को एक ही
`session.identityLinks` समूह में जोड़ें। उदाहरण के लिए, यदि Telegram प्रेषक `123` को
Discord पीयर `456` पर डॉक करना है, तो `telegram:123` और `discord:456`, दोनों शामिल करें।

**कमांड बताता है कि डॉकिंग केवल सीधे चैट से उपलब्ध है।**

डॉक कमांड OpenClaw के साथ सीधे चैट से भेजें, समूह चैट से नहीं।

**कमांड बताता है कि कोई सक्रिय सत्र मौजूद नहीं है।**

किसी मौजूदा सीधे-चैट सत्र से डॉक करें। कमांड को सक्रिय सत्र
प्रविष्टि की आवश्यकता होती है, ताकि वह नया मार्ग स्थायी रूप से सहेज सके।

**उत्तर अभी भी पुराने चैनल पर जाते हैं।**

जाँचें कि कमांड ने सफलता संदेश के साथ उत्तर दिया है, और पुष्टि करें कि लक्षित
पीयर आईडी उस चैनल द्वारा उपयोग की गई आईडी से मेल खाती है। डॉकिंग केवल सक्रिय
सत्र मार्ग बदलता है; कोई अन्य सत्र अभी भी किसी अन्य स्थान पर रूट कर सकता है।

**मुझे वापस स्विच करना है।**

लिंक किए गए प्रेषक से मूल चैनल के लिए मेल खाता कमांड भेजें, जैसे `/dock_telegram` या
`/dock-telegram`।
