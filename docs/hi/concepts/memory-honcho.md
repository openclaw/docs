[Honcho](https://honcho.dev) एक
बाहरी Plugin के माध्यम से OpenClaw में AI-मूल स्मृति जोड़ता है। यह वार्तालापों को एक समर्पित सेवा में बनाए रखता है और
समय के साथ उपयोगकर्ता और एजेंट मॉडल बनाता है, जिससे आपके एजेंट को सत्रों के बीच ऐसा संदर्भ मिलता है जो
वर्कस्पेस की Markdown फ़ाइलों से आगे जाता है।

## यह क्या प्रदान करता है

- **सत्रों के बीच स्मृति** - वार्तालाप प्रत्येक संवाद के बाद बने रहते हैं, इसलिए
  सत्र रीसेट, Compaction और चैनल बदलने के बाद भी संदर्भ बना रहता है।
- **उपयोगकर्ता मॉडलिंग** - Honcho प्रत्येक उपयोगकर्ता के लिए (प्राथमिकताएँ,
  तथ्य, संचार शैली) और एजेंट के लिए (व्यक्तित्व, सीखे गए
  व्यवहार) एक प्रोफ़ाइल बनाए रखता है।
- **सिमेंटिक खोज** - केवल वर्तमान सत्र में नहीं, बल्कि पिछले वार्तालापों से प्राप्त
  अवलोकनों में खोज करें।
- **बहु-एजेंट जागरूकता** - पैरेंट एजेंट बनाए गए
  सब-एजेंटों को स्वचालित रूप से ट्रैक करते हैं और चाइल्ड सत्रों में पैरेंट को पर्यवेक्षक के रूप में जोड़ा जाता है।

## उपलब्ध टूल

Honcho ऐसे टूल पंजीकृत करता है जिनका एजेंट वार्तालाप के दौरान उपयोग कर सकता है:

**डेटा पुनर्प्राप्ति (तेज़, कोई LLM कॉल नहीं):**

| टूल                        | यह क्या करता है                                           |
| --------------------------- | ------------------------------------------------------ |
| `honcho_context`            | सभी सत्रों में उपयोगकर्ता का पूर्ण निरूपण               |
| `honcho_search_conclusions` | संग्रहीत निष्कर्षों में सिमेंटिक खोज                |
| `honcho_search_messages`    | सभी सत्रों में संदेश खोजें (प्रेषक, दिनांक के अनुसार फ़िल्टर करें) |
| `honcho_session`            | वर्तमान सत्र का इतिहास और सारांश                    |

**प्रश्नोत्तर (LLM-संचालित):**

| टूल         | यह क्या करता है                                                              |
| ------------ | ------------------------------------------------------------------------- |
| `honcho_ask` | उपयोगकर्ता के बारे में पूछें। तथ्यों के लिए `depth='quick'`, संश्लेषण के लिए `'thorough'` |

## शुरुआत करना

Plugin इंस्टॉल करें और सेटअप चलाएँ:

```bash
openclaw plugins install @honcho-ai/openclaw-honcho
openclaw honcho setup
openclaw gateway --force
```

सेटअप कमांड आपसे API क्रेडेंशियल माँगता है, कॉन्फ़िगरेशन लिखता है और
वैकल्पिक रूप से मौजूदा वर्कस्पेस स्मृति फ़ाइलों को माइग्रेट करता है।

<Info>
Honcho पूरी तरह स्थानीय रूप से (स्वयं होस्ट किया हुआ) या
`api.honcho.dev` पर प्रबंधित API के माध्यम से चल सकता है। स्वयं होस्ट किए गए
विकल्प के लिए किसी बाहरी निर्भरता की आवश्यकता नहीं है।
</Info>

## कॉन्फ़िगरेशन

सेटिंग्स `plugins.entries["openclaw-honcho"].config` के अंतर्गत होती हैं:

```json5
{
  plugins: {
    entries: {
      "openclaw-honcho": {
        config: {
          apiKey: "your-api-key", // स्वयं होस्ट किए गए इंस्टेंस के लिए छोड़ दें
          workspaceId: "openclaw", // मेमोरी पृथक्करण
          baseUrl: "https://api.honcho.dev",
        },
      },
    },
  },
}
```

स्वयं होस्ट किए गए इंस्टेंस के लिए, `baseUrl` को अपने स्थानीय सर्वर पर इंगित करें (उदाहरण के लिए
`http://localhost:8000`) और API कुंजी छोड़ दें।

## मौजूदा मेमोरी माइग्रेट करना

यदि आपके पास मौजूदा वर्कस्पेस मेमोरी फ़ाइलें (`USER.md`, `MEMORY.md`,
`IDENTITY.md`, `memory/`, `canvas/`) हैं, तो `openclaw honcho setup` उनका पता लगाता है और
उन्हें माइग्रेट करने का विकल्प देता है।

<Info>
माइग्रेशन गैर-विनाशकारी है - फ़ाइलें Honcho पर अपलोड की जाती हैं। मूल फ़ाइलें
कभी हटाई या स्थानांतरित नहीं की जातीं।
</Info>

## यह कैसे काम करता है

AI के प्रत्येक टर्न के बाद, बातचीत Honcho में सहेजी जाती है। उपयोगकर्ता और
एजेंट, दोनों के संदेशों का अवलोकन किया जाता है, जिससे Honcho समय के साथ अपने मॉडल
बना और परिष्कृत कर सकता है।

बातचीत के दौरान, Honcho टूल OpenClaw के
`before_prompt_build` Plugin हुक के दौरान सेवा से क्वेरी करते हैं और मॉडल द्वारा
प्रॉम्प्ट देखे जाने से पहले प्रासंगिक संदर्भ उसमें जोड़ते हैं।

## Honcho बनाम अंतर्निर्मित मेमोरी

|                   | अंतर्निर्मित / QMD                | Honcho                              |
| ----------------- | ---------------------------- | ----------------------------------- |
| **स्टोरेज**       | वर्कस्पेस Markdown फ़ाइलें     | समर्पित सेवा (स्थानीय या होस्ट की गई) |
| **सत्रों के बीच** | मेमोरी फ़ाइलों के माध्यम से             | स्वचालित, अंतर्निर्मित                 |
| **उपयोगकर्ता मॉडलिंग** | मैन्युअल (MEMORY.md में लिखें)  | स्वचालित प्रोफ़ाइल                  |
| **खोज**        | वेक्टर + कीवर्ड (हाइब्रिड)    | अवलोकनों पर सिमेंटिक खोज          |
| **बहु-एजेंट**   | ट्रैक नहीं किया जाता                  | पैरेंट/चाइल्ड जागरूकता              |
| **निर्भरताएँ**  | कोई नहीं (अंतर्निर्मित) या QMD बाइनरी | Plugin इंस्टॉलेशन                      |

Honcho और अंतर्निर्मित मेमोरी सिस्टम साथ मिलकर काम कर सकते हैं। QMD
कॉन्फ़िगर होने पर, Honcho की सत्रों के बीच की मेमोरी के साथ-साथ स्थानीय Markdown
फ़ाइलों को खोजने के लिए अतिरिक्त टूल उपलब्ध हो जाते हैं।

## CLI कमांड

```bash
openclaw honcho setup                        # API कुंजी कॉन्फ़िगर करें और फ़ाइलें माइग्रेट करें
openclaw honcho status                       # कनेक्शन की स्थिति जाँचें
openclaw honcho ask <question>               # उपयोगकर्ता के बारे में Honcho से क्वेरी करें
openclaw honcho search <query> [-k N] [-d D] # मेमोरी में सिमेंटिक खोज करें
```
