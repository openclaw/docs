एक **एजेंट रनटाइम** एक तैयार मॉडल लूप का स्वामी होता है: यह प्रॉम्प्ट प्राप्त करता है,
मॉडल आउटपुट संचालित करता है, नेटिव टूल कॉल संभालता है और पूर्ण हुई बारी
OpenClaw को लौटाता है।

रनटाइम को प्रदाताओं से समझने में भ्रम होना आसान है, क्योंकि दोनों मॉडल
कॉन्फ़िगरेशन के पास दिखाई देते हैं। ये अलग-अलग परतें हैं:

| परत          | उदाहरण                                      | अर्थ                                                                        |
| ------------- | -------------------------------------------- | --------------------------------------------------------------------------- |
| प्रदाता       | `anthropic`, `github-copilot`, `openai`      | OpenClaw कैसे प्रमाणीकरण करता है, मॉडल खोजता है और मॉडल रेफ़रेंस नामित करता है। |
| मॉडल          | `claude-opus-4-6`, `gpt-5.6-sol`             | एजेंट की बारी के लिए चुना गया मॉडल।                                         |
| एजेंट रनटाइम  | `claude-cli`, `codex`, `copilot`, `openclaw` | तैयार बारी को निष्पादित करने वाला निम्न-स्तरीय लूप या बैकएंड।                |
| चैनल          | Discord, Slack, Telegram, WhatsApp           | जहाँ से संदेश OpenClaw में प्रवेश करते हैं और बाहर जाते हैं।                 |

**हार्नेस** वह कार्यान्वयन है जो एजेंट रनटाइम उपलब्ध कराता है (कोड
शब्दावली)। उदाहरण के लिए, बंडल किया गया Codex हार्नेस `codex` रनटाइम लागू करता है।
सार्वजनिक कॉन्फ़िगरेशन प्रदाता या मॉडल प्रविष्टियों पर `agentRuntime.id` का उपयोग करता है; पूरे-एजेंट
के रनटाइम कुंजी-पिन विरासती हैं और अनदेखे किए जाते हैं। `openclaw doctor --fix` पुराने
पूरे-एजेंट रनटाइम पिन हटाता है और जहाँ आवश्यक हो, विरासती रनटाइम मॉडल रेफ़रेंस को कैनोनिकल
प्रदाता/मॉडल रेफ़रेंस तथा मॉडल-स्कोप वाली रनटाइम नीति में पुनर्लिखता है।

रनटाइम के दो परिवार:

- **एम्बेडेड हार्नेस** OpenClaw के तैयार एजेंट लूप के भीतर चलते हैं: अंतर्निहित
  `openclaw` रनटाइम और पंजीकृत Plugin हार्नेस, जैसे
  `codex` और `copilot`।
- **CLI बैकएंड** मॉडल रेफ़रेंस को कैनोनिकल रखते हुए एक स्थानीय CLI प्रक्रिया चलाते हैं।
  उदाहरण के लिए, मॉडल-स्कोप वाले `agentRuntime.id: "claude-cli"` के साथ
  `anthropic/claude-opus-4-8` का अर्थ है, "Anthropic मॉडल चुनें और
  Claude CLI के माध्यम से निष्पादित करें।" `claude-cli` कोई एम्बेडेड हार्नेस आईडी नहीं है और इसे
  AgentHarness चयन में पास नहीं किया जाना चाहिए।

`copilot` हार्नेस GitHub Copilot CLI के लिए एक अलग, ऑप्ट-इन बाहरी Plugin हार्नेस है;
PI, Codex और GitHub Copilot एजेंट रनटाइम के बीच उपयोगकर्ता-सामना करने वाले
निर्णय के लिए [GitHub Copilot एजेंट रनटाइम](/hi/plugins/copilot) देखें।

## Codex सतहें

कई सतहें Codex नाम साझा करती हैं:

| सतह                                             | OpenClaw नाम/कॉन्फ़िगरेशन              | यह क्या करती है                                                                                                      |
| ------------------------------------------------ | ------------------------------------ | -------------------------------------------------------------------------------------------------------------------- |
| नेटिव Codex ऐप-सर्वर रनटाइम                      | `openai/*` मॉडल रेफ़रेंस              | Codex ऐप-सर्वर के माध्यम से OpenAI की एम्बेडेड एजेंट बारी चलाता है। यह सामान्य ChatGPT/Codex सदस्यता सेटअप है।       |
| Codex OAuth प्रमाणीकरण प्रोफ़ाइल                  | `openai` OAuth प्रोफ़ाइल            | वह ChatGPT/Codex सदस्यता प्रमाणीकरण संग्रहीत करती हैं जिसका उपयोग Codex ऐप-सर्वर हार्नेस करता है।                    |
| Codex ACP अडैप्टर                                | `runtime: "acp"`, `agentId: "codex"` | बाहरी ACP/acpx नियंत्रण तल के माध्यम से Codex चलाता है। केवल तभी उपयोग करें जब ACP/acpx स्पष्ट रूप से माँगा गया हो। |
| नेटिव Codex चैट-नियंत्रण कमांड सेट               | `/codex ...`                         | चैट से Codex ऐप-सर्वर थ्रेड को बाइंड, पुनः आरंभ, निर्देशित, बंद और निरीक्षित करता है।                               |
| गैर-एजेंट सतहों के लिए OpenAI Platform API रूट   | `openai/*` तथा API-कुंजी प्रमाणीकरण   | सीधे OpenAI API, जैसे इमेज, एम्बेडिंग, स्पीच और रीयलटाइम।                                                            |

ये सतहें जानबूझकर एक-दूसरे से स्वतंत्र हैं। `codex` Plugin सक्षम करने पर
नेटिव ऐप-सर्वर सुविधाएँ उपलब्ध होती हैं; `openclaw doctor --fix`
विरासती Codex रूट सुधार और पुराने सत्र पिन की सफ़ाई का स्वामी है। किसी एजेंट मॉडल के लिए
`openai/*` चुनने का अब अर्थ है, "इसे Codex के माध्यम से चलाएँ", जब तक किसी गैर-एजेंट
OpenAI API सतह का उपयोग न हो रहा हो।

सामान्य ChatGPT/Codex सदस्यता सेटअप प्रमाणीकरण के लिए Codex OAuth का उपयोग करता है, लेकिन
मॉडल रेफ़रेंस को `openai/*` रखता है और `codex` रनटाइम चुनता है:

```json5
{
  agents: {
    defaults: {
      model: "openai/gpt-5.6-sol",
    },
  },
}
```

इसका अर्थ है कि OpenClaw एक OpenAI मॉडल रेफ़रेंस चुनता है और फिर Codex
ऐप-सर्वर रनटाइम से एम्बेडेड एजेंट बारी चलाने को कहता है। इसका अर्थ "API
बिलिंग का उपयोग करें" नहीं है, और न ही इसका अर्थ यह है कि चैनल, मॉडल प्रदाता कैटलॉग या
OpenClaw सत्र स्टोर Codex बन जाता है।

जब बंडल किया गया `codex` Plugin सक्षम हो, तो स्वाभाविक भाषा में Codex नियंत्रण के लिए
ACP के बजाय नेटिव `/codex` कमांड सतह (`/codex bind`, `/codex threads`, `/codex resume`, `/codex steer`,
`/codex stop`) का उपयोग करें। Codex के लिए ACP का उपयोग केवल तभी करें जब उपयोगकर्ता स्पष्ट रूप से
ACP/acpx माँगे या ACP अडैप्टर पथ का परीक्षण कर रहा हो। Claude Code, Gemini CLI,
OpenCode, Cursor और इसी तरह के बाहरी हार्नेस अब भी ACP का उपयोग करते हैं।

निर्णय-वृक्ष:

1. **Codex बाइंड/नियंत्रण/थ्रेड/पुनः आरंभ/निर्देशन/बंद करना** -> बंडल किया गया `codex` Plugin सक्षम होने पर नेटिव `/codex` कमांड सतह।
2. **एम्बेडेड रनटाइम के रूप में Codex** या सामान्य सदस्यता-समर्थित Codex एजेंट अनुभव -> `openai/<model>`।
3. **OpenAI मॉडल के लिए OpenClaw स्पष्ट रूप से चुना गया है** -> मॉडल रेफ़रेंस को `openai/<model>` रखें और प्रदाता/मॉडल रनटाइम नीति को `agentRuntime.id: "openclaw"` पर सेट करें। चुनी गई `openai` OAuth प्रोफ़ाइल को आंतरिक रूप से OpenClaw के Codex-प्रमाणीकरण ट्रांसपोर्ट के माध्यम से रूट किया जाता है।
4. **कॉन्फ़िगरेशन में विरासती Codex मॉडल रेफ़रेंस** -> `openclaw doctor --fix` से सुधारकर `openai/<model>` करें; जहाँ पुराने मॉडल रेफ़रेंस से इसका आशय निकलता था, वहाँ doctor प्रदाता/मॉडल-स्कोप वाला `agentRuntime.id: "codex"` जोड़कर Codex प्रमाणीकरण रूट बनाए रखता है। विरासती **`codex-cli/*`** मॉडल रेफ़रेंस उसी `openai/<model>` Codex ऐप-सर्वर रूट में सुधरते हैं; OpenClaw अब बंडल किया गया Codex CLI बैकएंड नहीं रखता।
5. **ACP, acpx या Codex ACP अडैप्टर स्पष्ट रूप से माँगा गया है** -> `runtime: "acp"` और `agentId: "codex"`।
6. **Claude Code, Gemini CLI, OpenCode, Cursor, Droid या कोई अन्य बाहरी हार्नेस** -> ACP/acpx, नेटिव उप-एजेंट रनटाइम नहीं।

| आपका आशय है...                         | उपयोग करें...                                 |
| --------------------------------------- | -------------------------------------------- |
| Codex ऐप-सर्वर चैट/थ्रेड नियंत्रण      | बंडल किए गए `codex` Plugin से `/codex ...` |
| Codex ऐप-सर्वर एम्बेडेड एजेंट रनटाइम   | `openai/*` एजेंट मॉडल रेफ़रेंस                |
| OpenAI Codex OAuth                      | `openai` OAuth प्रोफ़ाइल                    |
| Claude Code या अन्य बाहरी हार्नेस       | ACP/acpx                                     |

OpenAI-परिवार प्रीफ़िक्स विभाजन के लिए [OpenAI](/hi/providers/openai) और
[मॉडल प्रदाता](/hi/concepts/model-providers) देखें। Codex रनटाइम समर्थन
अनुबंध के लिए [Codex हार्नेस रनटाइम](/hi/plugins/codex-harness-runtime#v1-support-contract) देखें।

## रनटाइम का स्वामित्व

अलग-अलग रनटाइम लूप के अलग-अलग हिस्सों के स्वामी होते हैं:

| सतह                         | OpenClaw एम्बेडेड                              | Codex ऐप-सर्वर                                                                  |
| --------------------------- | ---------------------------------------------- | ------------------------------------------------------------------------------- |
| मॉडल लूप का स्वामी          | OpenClaw एम्बेडेड रनर के माध्यम से OpenClaw    | Codex ऐप-सर्वर                                                                  |
| कैनोनिकल थ्रेड स्थिति       | OpenClaw ट्रांसक्रिप्ट                          | Codex थ्रेड तथा OpenClaw ट्रांसक्रिप्ट मिरर                                     |
| OpenClaw डायनेमिक टूल       | नेटिव OpenClaw टूल लूप                         | Codex अडैप्टर के माध्यम से ब्रिज किए गए                                         |
| नेटिव शेल और फ़ाइल टूल      | OpenClaw पथ                                    | Codex-नेटिव टूल, जहाँ समर्थित हों वहाँ नेटिव हुक के माध्यम से ब्रिज किए गए      |
| संदर्भ इंजन                 | नेटिव OpenClaw संदर्भ संयोजन                   | OpenClaw संयोजित संदर्भ को Codex बारी में प्रक्षेपित करता है                    |
| Compaction                  | OpenClaw या चुना गया संदर्भ इंजन               | Codex-नेटिव Compaction, OpenClaw सूचनाओं और मिरर रखरखाव के साथ                  |
| चैनल वितरण                  | OpenClaw                                       | OpenClaw                                                                        |

डिज़ाइन नियम: यदि OpenClaw किसी सतह का स्वामी है, तो वह सामान्य Plugin हुक
व्यवहार उपलब्ध करा सकता है। यदि नेटिव रनटाइम सतह का स्वामी है, तो OpenClaw को रनटाइम
ईवेंट या नेटिव हुक की आवश्यकता होती है। यदि नेटिव रनटाइम कैनोनिकल थ्रेड स्थिति का स्वामी है,
तो OpenClaw असमर्थित आंतरिक भागों को पुनर्लिखने के बजाय संदर्भ को मिरर और प्रक्षेपित
करता है।

## रनटाइम चयन

OpenClaw प्रदाता और मॉडल समाधान के बाद निम्न क्रम में एक एम्बेडेड रनटाइम
निर्धारित करता है:

1. **मॉडल-स्कोप वाली रनटाइम नीति** को प्राथमिकता मिलती है। यह किसी कॉन्फ़िगर किए गए प्रदाता
   मॉडल प्रविष्टि में या `agents.defaults.models["provider/model"].agentRuntime`
   / `agents.list[].models["provider/model"].agentRuntime` में रहती है। `agents.defaults.models["vllm/*"].agentRuntime` जैसा प्रदाता
   वाइल्डकार्ड सटीक मॉडल नीति के बाद लागू होता है, ताकि डायनेमिक रूप से खोजे गए प्रदाता मॉडल
   सटीक प्रति-मॉडल अपवादों को ओवरराइड किए बिना एक रनटाइम साझा कर सकें।
2. **प्रदाता-स्कोप वाली रनटाइम नीति**: `models.providers.<provider>.agentRuntime`।
3. **`auto` मोड**: पंजीकृत Plugin रनटाइम समर्थित प्रदाता/मॉडल जोड़ियों पर दावा कर सकते हैं।
4. यदि `auto` मोड में कोई भी बारी पर दावा नहीं करता, तो OpenClaw संगतता रनटाइम के रूप में
   `openclaw` पर वापस जाता है। जब रन सख़्त होना आवश्यक हो,
   तब स्पष्ट रनटाइम आईडी का उपयोग करें।

पूरे-सत्र और पूरे-एजेंट रनटाइम पिन अनदेखे किए जाते हैं: `OPENCLAW_AGENT_RUNTIME`,
सत्र की `agentHarnessId`/`agentRuntimeOverride` स्थिति, `agents.defaults.agentRuntime`
और `agents.list[].agentRuntime`। पुराने पूरे-एजेंट रनटाइम कॉन्फ़िगरेशन को हटाने और जहाँ आशय
संरक्षित किया जा सके वहाँ विरासती रनटाइम मॉडल रेफ़रेंस को परिवर्तित करने के लिए
`openclaw doctor --fix` चलाएँ।

स्पष्ट प्रदाता/मॉडल Plugin रनटाइम विफलता पर बंद हो जाते हैं: प्रदाता या मॉडल पर
`agentRuntime.id: "codex"` का अर्थ Codex या स्पष्ट चयन/रनटाइम त्रुटि है—इसे कभी भी
चुपचाप वापस OpenClaw पर रूट नहीं किया जाता। केवल `auto` किसी बेमेल
बारी को OpenClaw पर रूट कर सकता है।

CLI बैकएंड उपनाम एम्बेडेड हार्नेस आईडी से अलग होते हैं। पसंदीदा Claude CLI रूप:

```json5
{
  agents: {
    defaults: {
      model: "anthropic/claude-opus-4-8",
      models: {
        "anthropic/claude-opus-4-8": {
          agentRuntime: { id: "claude-cli" },
        },
      },
    },
  },
}
```

`claude-cli/claude-opus-4-7` जैसे विरासती रेफ़रेंस संगतता के लिए
अब भी समर्थित हैं, लेकिन नए कॉन्फ़िगरेशन को प्रदाता/मॉडल कैनोनिकल रखना चाहिए और
निष्पादन बैकएंड को प्रदाता/मॉडल रनटाइम नीति में रखना चाहिए।

विरासती `codex-cli/*` रेफ़रेंस अलग हैं: doctor उन्हें `openai/*` में माइग्रेट करता है, ताकि
वे Codex CLI बैकएंड संरक्षित करने के बजाय Codex ऐप-सर्वर हार्नेस के माध्यम से चलें।

अधिकांश प्रदाताओं के लिए `auto` मोड जानबूझकर रूढ़िवादी है। OpenAI एजेंट
मॉडल अपवाद हैं: अनसेट रनटाइम और `auto` दोनों Codex हार्नेस में निर्धारित होते हैं।
स्पष्ट OpenClaw रनटाइम कॉन्फ़िगरेशन `openai/*` एजेंट बारियों के लिए ऑप्ट-इन संगतता
रूट बना रहता है; चुनी गई `openai` OAuth प्रोफ़ाइल के साथ जोड़े जाने पर, OpenClaw
सार्वजनिक मॉडल रेफ़रेंस को `openai/*` रखते हुए उस पथ को आंतरिक रूप से
Codex-प्रमाणीकरण ट्रांसपोर्ट के माध्यम से रूट करता है। पुराने OpenAI रनटाइम सत्र पिन
रनटाइम चयन द्वारा अनदेखे किए जाते हैं और `openclaw doctor --fix` से साफ़ किए जा सकते हैं।

यदि `openclaw doctor` चेतावनी देता है कि `codex` plugin सक्षम है, जबकि पुराने
Codex मॉडल संदर्भ अब भी कॉन्फ़िगरेशन में मौजूद हैं, तो इसे पुराने रूट की स्थिति मानें और
इसे Codex रनटाइम के साथ `openai/*` में फिर से लिखने के लिए
`openclaw doctor --fix` चलाएँ।

## GitHub Copilot एजेंट रनटाइम

बाहरी `@openclaw/copilot` plugin, GitHub Copilot CLI (`@github/copilot-sdk`) द्वारा समर्थित,
वैकल्पिक `copilot` रनटाइम पंजीकृत करता है। यह प्रामाणिक
सब्सक्रिप्शन `github-copilot` प्रदाता का दावा करता है और `auto` द्वारा
**कभी भी** चयनित नहीं किया जाता। `agentRuntime.id` के माध्यम से प्रत्येक मॉडल या प्रदाता के लिए इसे चुनें:

```json5
{
  agents: {
    defaults: {
      model: "github-copilot/gpt-5.5",
      models: {
        "github-copilot/gpt-5.5": {
          agentRuntime: { id: "copilot" },
        },
      },
    },
  },
}
```

हार्नेस `extensions/copilot/doctor-contract-api.ts` में अपने प्रदाता, रनटाइम, CLI सत्र कुंजी और प्रमाणीकरण प्रोफ़ाइल
उपसर्ग का दावा करता है, जिसे `openclaw doctor`
स्वतः लोड करता है। कॉन्फ़िगरेशन, प्रमाणीकरण, ट्रांसक्रिप्ट मिररिंग, Compaction,
घोषणात्मक डॉक्टर अनुबंध और व्यापक PI बनाम Codex बनाम Copilot SDK
निर्णय के लिए, [GitHub Copilot एजेंट रनटाइम](/hi/plugins/copilot) देखें।

## संगतता अनुबंध

जब कोई रनटाइम OpenClaw नहीं होता, तो उसके दस्तावेज़ों में यह बताया जाना चाहिए कि वह किन OpenClaw सतहों
का समर्थन करता है:

| प्रश्न                                  | यह क्यों महत्वपूर्ण है                                                                                         |
| -------------------------------------- | ------------------------------------------------------------------------------------------------- |
| मॉडल लूप का स्वामी कौन है?              | यह निर्धारित करता है कि पुनः प्रयास, टूल निरंतरता और अंतिम उत्तर के निर्णय कहाँ होते हैं।                              |
| प्रामाणिक थ्रेड इतिहास का स्वामी कौन है? | यह निर्धारित करता है कि OpenClaw इतिहास संपादित कर सकता है या केवल उसे मिरर कर सकता है।                              |
| क्या OpenClaw के डायनेमिक टूल काम करते हैं? | मैसेजिंग, सत्र, Cron और OpenClaw के स्वामित्व वाले टूल इस पर निर्भर करते हैं।                                      |
| क्या डायनेमिक टूल हुक काम करते हैं?      | Plugins, OpenClaw के स्वामित्व वाले टूल के आसपास `before_tool_call`, `after_tool_call` और मिडलवेयर की अपेक्षा करते हैं। |
| क्या नेटिव टूल हुक काम करते हैं?         | शेल, पैच और रनटाइम के स्वामित्व वाले टूल को नीति और अवलोकन के लिए नेटिव हुक समर्थन चाहिए।                          |
| क्या कॉन्टेक्स्ट इंजन जीवनचक्र चलता है?   | मेमोरी और कॉन्टेक्स्ट plugins, असेंबल, इंजेस्ट, टर्न के बाद और Compaction जीवनचक्र पर निर्भर करते हैं।                 |
| कौन-सा Compaction डेटा उपलब्ध कराया जाता है? | कुछ plugins को केवल सूचनाएँ चाहिए; अन्य को रखे गए/हटाए गए मेटाडेटा की आवश्यकता होती है।                           |
| जानबूझकर क्या असमर्थित है?               | जहाँ नेटिव रनटाइम अधिक स्थिति का स्वामी है, वहाँ उपयोगकर्ताओं को OpenClaw के समकक्षता की धारणा नहीं बनानी चाहिए।       |

Codex रनटाइम समर्थन अनुबंध का दस्तावेज़ीकरण
[Codex हार्नेस रनटाइम](/hi/plugins/codex-harness-runtime#v1-support-contract) में किया गया है।

## स्थिति लेबल

स्थिति आउटपुट में `Execution` और `Runtime` दोनों लेबल दिखाई दे सकते हैं। इन्हें
निदान के रूप में पढ़ें, प्रदाता नामों के रूप में नहीं:

- `openai/gpt-5.6-sol` जैसा मॉडल संदर्भ चयनित प्रदाता/मॉडल है।
- `codex` जैसी रनटाइम आईडी वह लूप है जो टर्न निष्पादित कर रहा है।
- Telegram या Discord जैसा चैनल लेबल वह स्थान है जहाँ बातचीत हो रही है।

यदि कोई रन अप्रत्याशित रनटाइम दिखाता है, तो पहले चयनित प्रदाता/मॉडल की
रनटाइम नीति की जाँच करें। पुराने सत्र रनटाइम पिन अब रूटिंग तय नहीं करते।
