## यह क्या है

- प्रत्येक प्रदाता के उपयोग एंडपॉइंट से सीधे प्रदाता का उपयोग/कोटा प्राप्त करता है। प्रदाता की बिलिंग का कोई अनुमान नहीं; केवल प्रदाता द्वारा रिपोर्ट किए गए प्लान के नाम, कोटा विंडो, शेष राशि, खर्च, बजट, दैनिक लागत इतिहास, टोकन/मॉडल श्रेयांकन या खाते की स्थिति के सारांश।
- मानव-पठनीय कोटा-विंडो आउटपुट को `X% left` में सामान्यीकृत किया जाता है, भले ही कोई प्रदाता उपभोग किया गया कोटा, शेष कोटा या केवल अपरिष्कृत संख्याएँ रिपोर्ट करे। रीसेट की जा सकने वाली कोटा विंडो के बिना प्रदाता इसके बजाय प्रदाता सारांश टेक्स्ट दिखाते हैं (उदाहरण के लिए, शेष राशि)।
- लाइव सत्र स्नैपशॉट में टोकन/मॉडल डेटा न होने पर सत्र-स्तरीय `/status` और `session_status` टूल सत्र के ट्रांसक्रिप्ट लॉग का फ़ॉलबैक के रूप में उपयोग करते हैं। यह फ़ॉलबैक अनुपलब्ध टोकन/कैश काउंटर भरता है, सक्रिय रनटाइम मॉडल लेबल पुनर्प्राप्त कर सकता है और सत्र मेटाडेटा अनुपलब्ध या छोटा होने पर प्रॉम्प्ट-उन्मुख बड़े योग को प्राथमिकता देता है (`totalTokensFresh !== true`, शून्य या ट्रांसक्रिप्ट से प्राप्त मान से कम)। शून्येतर लाइव मानों को हमेशा फ़ॉलबैक पर प्राथमिकता मिलती है।

## यह कहाँ दिखाई देता है

- चैट में `/status`: सत्र टोकन और अनुमानित लागत वाला स्थिति कार्ड (केवल API कुंजी मॉडल)। उपलब्ध होने पर **वर्तमान मॉडल प्रदाता** के लिए प्रदाता उपयोग, सामान्यीकृत `X% left` विंडो या प्रदाता सारांश टेक्स्ट के रूप में दिखाई देता है।
- चैट में `/usage off|tokens|full`: प्रत्येक प्रतिक्रिया के लिए उपयोग फ़ुटर।
- चैट में `/usage cost`: OpenClaw सत्र लॉग से एकत्रित स्थानीय लागत सारांश।
- CLI: `openclaw status --usage` प्रत्येक प्रदाता के उपयोग/कोटा का पूरा विवरण प्रिंट करता है।
- CLI: `openclaw models status` OAuth/टोकन प्रमाणीकरण प्रोफ़ाइल सूचीबद्ध करता है और प्रत्येक ऐसे प्रदाता के आगे उपयोग-विंडो सारांश दिखाता है जिसके लिए यह उपलब्ध है।
- कंट्रोल UI: **उपयोग** में OpenClaw के सत्र से प्राप्त टोकन और अनुमानित-लागत विश्लेषण के ऊपर प्रदाता के प्लान और बिलिंग कार्ड दिखाई देते हैं। Anthropic और OpenAI Admin API क्रेडेंशियल प्रदाता द्वारा रिपोर्ट किया गया आज का, 7-दिन और 30-दिन का खर्च, दैनिक रुझान, टोकन योग, शीर्ष मॉडल और लागत श्रेणियाँ जोड़ते हैं।
- कंट्रोल UI: चैट कंपोज़र के संदर्भ रिंग पॉपओवर में सदस्यता प्रदाताओं के लिए **प्लान उपयोग** दिखाई देता है—प्रत्येक विंडो के बार (5-घंटे, साप्ताहिक, मॉडल-सीमित) और उनके रीसेट समय, ज्ञात होने पर प्रदाता प्लान (उदाहरण के लिए `Max (20x)`) तथा अतिरिक्त-उपयोग क्रेडिट। प्लान के माध्यम से बिल किए गए सत्र प्रति-टोकन डॉलर अनुमान छिपाते हैं; API-बिल किए गए सत्र `Est. cost` और प्रकार के अनुसार लागत विवरण बनाए रखते हैं। Claude Code CLI (`claude-cli`) सेटअप उसी Anthropic सदस्यता उपयोग का पुनः उपयोग करते हैं।
- macOS मेनू बार: प्रदाता उपयोग स्नैपशॉट उपलब्ध होने पर Context के नीचे एक मूल "उपयोग" अनुभाग दिखाई देता है। [मेनू बार](/hi/platforms/mac/menu-bar) देखें।

`openclaw channels list` अब प्रदाता उपयोग प्रिंट नहीं करता; इसके बजाय यह उपयोगकर्ताओं को `openclaw status` या `openclaw models list` की ओर निर्देशित करता है।

## Anthropic और OpenAI का लागत इतिहास

सदस्यता कोटा और API बिलिंग प्रदाता के अलग-अलग इंटरफ़ेस हैं:

- Anthropic सदस्यता/सेटअप क्रेडेंशियल Claude कोटा विंडो और वैकल्पिक अतिरिक्त-उपयोग बजट दिखाना जारी रखते हैं। इसके बजाय संगठन का Usage and Cost API इतिहास दिखाने के लिए `ANTHROPIC_ADMIN_KEY` या `ANTHROPIC_ADMIN_API_KEY` सेट करें। `sk-ant-admin` से शुरू होने वाले Anthropic प्रदाता क्रेडेंशियल का स्वतः पता लगाया जाता है।
- OpenAI ChatGPT/Codex OAuth प्लान, कोटा विंडो और क्रेडिट शेष दिखाना जारी रखता है। इसके बजाय संगठन की लागत और completions-usage इतिहास दिखाने के लिए `OPENAI_ADMIN_KEY` सेट करें; इसे किसी एक प्रोजेक्ट तक सीमित करने के लिए वैकल्पिक रूप से `OPENAI_PROJECT_ID` सेट करें। OpenClaw कभी भी `OPENAI_API_KEY`, प्रदाता कॉन्फ़िगरेशन या प्रमाणीकरण प्रोफ़ाइल से अनुमान क्रेडेंशियल संगठन API को नहीं भेजता, क्योंकि वे कुंजियाँ कस्टम एंडपॉइंट से संबंधित हो सकती हैं।

Admin क्रेडेंशियल को प्राथमिकता मिलती है क्योंकि वे संगठन की वास्तविक बिलिंग प्रदान करते हैं। OpenClaw प्रदाता द्वारा रिपोर्ट किए गए इन योगों को अपने स्थानीय सत्र अनुमानों के साथ नहीं जोड़ता; दोनों अनुभाग जानबूझकर अलग-अलग प्रश्नों के उत्तर देते हैं।

## डिफ़ॉल्ट उपयोग फ़ुटर मोड

`/usage off|tokens|full` किसी सत्र के लिए फ़ुटर सेट करता है और उस
सत्र के लिए इसे याद रखा जाता है। जिन सत्रों ने कोई मोड नहीं चुना है, उनके लिए
`messages.responseUsage` उस मोड का आरंभिक मान निर्धारित करता है, ताकि हर बार `/usage` टाइप किए बिना फ़ुटर डिफ़ॉल्ट रूप से चालू रह सके।

प्रत्येक चैनल के लिए एक मोड या `default` फ़ॉलबैक वाला प्रति-चैनल मैप सेट करें:

```jsonc
{
  "messages": {
    "responseUsage": "tokens",
    // या: { "default": "off", "discord": "full" }
  },
}
```

स्वीकृत मान: `"off"`, `"tokens"`, `"full"` और पुराना उपनाम `"on"` (जिसे `"tokens"` माना जाता है)।

### सत्र की तीन अलग-अलग स्थितियाँ

किसी सत्र के `responseUsage` फ़ील्ड की तीन दर्शनीय स्थितियाँ होती हैं और प्रत्येक के
अर्थ अलग हैं:

| स्थिति               | संग्रहीत मान                    | प्रभावी मोड                                                        |
| ------------------- | ------------------------------- | --------------------------------------------------------------------- |
| **सेट नहीं / इनहेरिट** | `undefined` (अनुपस्थित)            | `messages.responseUsage` कॉन्फ़िगरेशन डिफ़ॉल्ट और फिर `off` का फ़ॉलबैक उपयोग करता है। |
| **स्पष्ट रूप से बंद**    | `"off"` (संग्रहीत)                | हमेशा बंद; बंद न होने वाला कॉन्फ़िगरेशन डिफ़ॉल्ट फ़ुटर को फिर से सक्षम नहीं कर सकता।     |
| **स्पष्ट रूप से चालू**     | `"tokens"` या `"full"` (संग्रहीत) | कॉन्फ़िगरेशन डिफ़ॉल्ट की परवाह किए बिना वही मोड।                              |

### प्राथमिकता

प्रभावी मोड = सत्र ओवरराइड → चैनल कॉन्फ़िगरेशन प्रविष्टि → `default` → `off`।

एक स्पष्ट `/usage off` सत्र में अक्षरशः `"off"` मान के रूप में **स्थायी रूप से संग्रहीत** होता है,
यह "सेट नहीं" के समान नहीं है। उपयोगकर्ता द्वारा फ़ुटर स्पष्ट रूप से अक्षम किए जाने के बाद बंद न होने वाला
`messages.responseUsage` डिफ़ॉल्ट उसे फिर से चालू नहीं कर सकता।

### रीसेट करना और बंद करना

- `/usage off` फ़ुटर को जबरन बंद करता है और उस विकल्प को स्थायी रूप से संग्रहीत करता है। कॉन्फ़िगर किया गया
  बंद न होने वाला डिफ़ॉल्ट इसे ओवरराइड नहीं कर सकता।
- `/usage reset` (उपनाम: `default`, `inherit`, `inherited`, `clear`, `unpin`) सत्र
  ओवरराइड साफ़ करता है। इसके बाद सत्र प्रभावी कॉन्फ़िगरेशन डिफ़ॉल्ट
  (`messages.responseUsage`) को **इनहेरिट** करता है। यदि कोई डिफ़ॉल्ट कॉन्फ़िगर नहीं है, तो फ़ुटर बंद रहता है।
- पूर्ण सत्र रीसेट (`/reset` या `/new`) या सत्र रोलओवर स्पष्ट उपयोग-मोड वरीयता को
  **बनाए रखता है**, ताकि उपयोगकर्ता का प्रदर्शन विकल्प सत्र रोलओवर के बाद भी बना रहे।
  केवल `/usage reset` (और उसके उपनाम) ओवरराइड साफ़ करते हैं।

### टॉगल व्यवहार

बिना आर्ग्युमेंट वाला `/usage` इस क्रम में चक्र चलाता है: बंद → टोकन → पूर्ण → बंद। चक्र का आरंभिक बिंदु
**प्रभावी** वर्तमान मोड होता है (सेट न होने पर सत्र ओवरराइड कॉन्फ़िगरेशन डिफ़ॉल्ट का
फ़ॉलबैक उपयोग करता है), इसलिए चक्र हमेशा उससे मेल खाता है जो उपयोगकर्ता
वर्तमान में फ़ुटर में देखता है।

### कॉन्फ़िगरेशन

कॉन्फ़िगरेशन न होने पर पिछला व्यवहार लागू रहता है (`/usage` तक फ़ुटर बंद)। सत्र ओवरराइड साफ़ करने और कॉन्फ़िगर किए गए डिफ़ॉल्ट को फिर से इनहेरिट करने के लिए
`/usage reset` का उपयोग करें।

## कस्टम `/usage full` फ़ुटर

`/usage tokens` हमेशा एक सामान्य `Usage: X in / Y out` पंक्ति प्रस्तुत करता है (उपलब्ध होने पर कैश और
अनुमानित-लागत प्रत्ययों सहित)। केवल `/usage full` नीचे वर्णित अधिक समृद्ध
फ़ुटर प्रस्तुत करता है।

`/usage full` उपलब्ध फ़ील्ड के आधार पर मॉडल, रीजनिंग, तेज़/धीमा,
संदर्भ विंडो और लागत वाला अंतर्निहित संक्षिप्त फ़ुटर दिखाता है। अंतर्निहित फ़ुटर के लिए
किसी टेम्पलेट फ़ाइल की आवश्यकता नहीं है।

`messages.usageTemplate` केवल उन्नत कस्टम लेआउट के लिए है। इसका मान
JSON फ़ाइल पथ (`~` समर्थित) या इनलाइन ऑब्जेक्ट होता है और मान्य होने पर यह अंतर्निहित
फ़ुटर को प्रतिस्थापित कर देता है। फ़ाइल पथ की निगरानी की जाती है और बदलाव होने पर उसे लाइव पुनः लोड किया जाता है।

```json
{
  "messages": {
    "usageTemplate": "~/.openclaw/usage-footer.json"
  }
}
```

अनुपलब्ध या खाली टेम्पलेट बिना किसी सूचना के अंतर्निहित फ़ुटर का फ़ॉलबैक उपयोग करते हैं। अपठनीय
या अमान्य कॉन्फ़िगर किए गए टेम्पलेट (खराब JSON या ऐसा आकार जिसमें प्रस्तुत करने योग्य आउटपुट
खंड न हों) भी अंतर्निहित फ़ुटर का फ़ॉलबैक उपयोग करते हैं और ऑपरेटर चेतावनी जारी करते हैं।

```jsonc
      { "when": "cost.turn_usd", "text": " 💰{cost.turn_usd|fixed:4}" },
    ],
    "surfaces": {
      "discord": [
        { "text": "-# -\n" },
        { "text": "-# {model.provider}{identity.emoji|🤖}{model.display_name|alias:models}" },
        { "map": "model.is_fallback", "cases": { "true": "🔄" } },
        { "map": "model.is_override", "cases": { "true": "📌" } },
        { "when": "model.reasoning", "text": "{model.reasoning|alias:reasoning}" },
        { "map": "state.fast_mode", "cases": { "true": "⚡️", "false": "🐌" } },
        {
          "when": "context.max_tokens",
          "text": " | 📚[{context.pct_used|meter:5:braille}]{context.max_tokens|num}",
        },
        { "when": "cost.turn_usd", "text": " 💰{cost.turn_usd|fixed:4}" },
      ],
    },
  },
}
```

### आकार

```jsonc
{
  "schema": "openclaw.usageBar.v1",
  "scales": { "<name>": "निम्न-से-उच्च ग्लिफ़" }, // स्ट्रिंग (1 ग्लिफ़/वर्ण) या सरणी
  "aliases": { "<table>": { "<value>": "<label>" } },
  "output": {
    "sep": "", // शेष खंडों को जोड़ता है
    "default": [/* pieces */], // किसी भी इंटरफ़ेस के लिए फ़ॉलबैक
    "surfaces": {
      "discord": [/* pieces */],
      "telegram": [/* pieces */],
    },
  },
}
```

प्रत्येक इंटरफ़ेस **खंडों** की एक क्रमबद्ध सूची है; इंजन प्रत्येक को प्रस्तुत करता है, खाली खंड
हटा देता है और शेष खंडों को `sep` से जोड़ता है। बिना प्रविष्टि वाला इंटरफ़ेस
`output.default` का उपयोग करता है।

### अनुबंध पथ

कोई खंड डॉट-पथ द्वारा प्रत्येक टर्न के अनुबंध से मान पढ़ता है। अनुपस्थित मान
खाली होते हैं (इसलिए `when` गार्ड या `|fallback` खंड को साफ़ रखता है)।

| पथ                                                                                | अर्थ                                                                                              |
| ----------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| `surface`                                                                           | चैनल आईडी (`discord`/`telegram`/आदि)                                                               |
| `agentId` / `chat_type`                                                             | स्वामी एजेंट आईडी / चैट सतह का प्रकार                                                                  |
| `model.id` / `model.display_name` / `model.provider`                                | मॉडल आईडी / प्रदर्शन नाम / प्रदाता आईडी                                                                |
| `model.actual`, `model.resolved_ref`                                                | टर्न के लिए वास्तव में उपयोग किया गया प्रदाता/मॉडल संदर्भ                                                        |
| `model.requested`                                                                   | अनुरोधित प्रदाता/मॉडल संदर्भ (फ़ॉलबैक से पहले)                                                       |
| `model.reasoning`                                                                   | प्रयास (`off` से `xhigh` तक)                                                                       |
| `model.is_fallback` / `model.is_override`                                           | बूलियन: फ़ॉलबैक उपयोग हुआ / मॉडल पिन किया गया                                                                   |
| `model.override_source` / `model.auth_mode`                                         | ओवरराइड स्रोत लेबल / क्रेडेंशियल मोड (`oauth`, `api-key`, `token`, `mixed`, `aws-sdk`, `unknown`) |
| `state.fast_mode`                                                                   | बूलियन: तेज़ बनाम धीमा                                                                                   |
| `state.compactions`                                                                 | सत्र के लिए Compaction की संख्या                                                                     |
| `context.max_tokens` / `context.used_tokens` / `context.pct_used`                   | विंडो बजट / अधिकृत टोकन / उपयोग का 0-100 प्रतिशत                                                         |
| `usage.input_tokens` / `usage.output_tokens` / `usage.total_tokens`                 | टर्न समुच्चय                                                                                       |
| `usage.cache_read_tokens` / `usage.cache_write_tokens`                              | टर्न के लिए कैश-पठन और कैश-लेखन टोकन                                                       |
| `usage.has_tokens` / `usage.has_split_tokens` / `usage.has_total_only_tokens`       | टोकन प्रदर्शन रक्षक                                                                                 |
| `usage.cache_hit_pct`                                                               | कुल प्रॉम्प्ट टोकन में कैश-पठन का हिस्सा                                                              |
| `usage.last.input_tokens` / `usage.last.output_tokens` / `usage.last.cache_hit_pct` | केवल अंतिम मॉडल कॉल (इसमें `cache_read_tokens`, `cache_write_tokens`, `total_tokens` भी हैं)           |
| `cost.turn_usd` / `cost.available`                                                  | अनुमानित टर्न लागत / क्या कोई लागत तालिका रिज़ॉल्व हुई                                                  |
| `timing.duration_ms`                                                                | वास्तविक समय के अनुसार टर्न की अवधि                                                                             |
| `identity.name` / `identity.emoji` / `identity.avatar`                              | एजेंट पहचान नाम / इमोजी / अवतार                                                                 |
| `session.id`                                                                        | सत्र आईडी                                                                                           |

(प्रदाता की दर-सीमा विंडो इस अनुबंध में **नहीं** हैं; आज कोई सरणी-मान वाला पथ नहीं है, इसलिए `each` अंश के पास पुनरावृत्ति के लिए कुछ नहीं है।)

### क्रियाएँ

किसी मान को क्रियाओं से बाएँ से दाएँ पाइप करें; जो खंड क्रिया नहीं है, वह फ़ॉलबैक होता है।

| क्रिया            | प्रभाव                                | उदाहरण                           |
| --------------- | ------------------------------------- | --------------------------------- |
| `num`           | संक्षिप्त संख्या                         | `272000 -> 272k`                  |
| `fixed:N`       | N दशमलव (`0..100`, डिफ़ॉल्ट 2)      | `0.0377`                          |
| `dur`           | सेकंड को अवधि में बदलना                   | `14820 -> 4h07m`                  |
| `pct`           | `%` जोड़ना                            | `96 -> 96%`                       |
| `inv`           | `100 - x`                             | उपयोग किए गए को शेष में बदलने के लिए             |
| `alias:TABLE`   | `aliases` में खोजें, सूचीबद्ध न होने पर यथावत लौटाएँ | `medium -> 🌗`                    |
| `meter:W:SCALE` | 0-100 मान पर W-सेल ग्लिफ़ बार   | `[⣿⣿⠐⠐⠐]` (`meter:1` = एक ग्लिफ़) |

`fixed:N` केवल 0 से 100 तक का पूर्ण दशमलव पूर्णांक स्वीकार करता है। अमान्य
परिशुद्धता तर्क उस इंटरपोलेशन को खाली कर देते हैं।

`meter:W:SCALE` केवल 1 से 100 तक की पूर्ण दशमलव पूर्णांक चौड़ाई स्वीकार करता है। डिफ़ॉल्ट 5 (`meter::braille`) का उपयोग करने के लिए चौड़ाई खाली छोड़ें; अमान्य
चौड़ाइयाँ उस इंटरपोलेशन को खाली कर देती हैं।

### अंश के रूप

- `{ "text": "📚 {context.max_tokens|num}" }`: लिटरल + इंटरपोलेशन।
- `{ "when": "<path>", "text": "..." }`: केवल तभी रेंडर करें जब पथ ट्रूथी हो।
- `{ "map": "<path>", "cases": { "true": "⚡", "false": "🐌" } }`: मान से ग्लिफ़ (एक `_default` केस बेमेल मानों को संभालता है)।
- `{ "each": "<array-path>", "item": "{label}" }`: सरणी-मान वाले पथ पर पुनरावृत्ति करें (वर्तमान अनुबंध का कोई भी पथ सरणी नहीं है)।

### उदाहरण

```jsonc
{
  "schema": "openclaw.usageBar.v1",
  "scales": { "braille": "⠐⡀⡄⡆⡇⣇⣧⣷⣿" },
  "aliases": { "reasoning": { "medium": "🌗", "high": "🌕" } },
  "output": {
    "surfaces": {
      "discord": [
        { "text": "{model.display_name}" },
        { "when": "model.reasoning", "text": " {model.reasoning|alias:reasoning}" },
        { "map": "state.fast_mode", "cases": { "true": " ⚡", "false": " 🐌" } },
        {
          "when": "context.max_tokens",
          "text": " | 📚 [{context.pct_used|meter:5:braille}]{context.max_tokens|num}",
        },
      ],
    },
  },
}
```

उदाहरण के लिए `claude-sonnet-4-6 🌗 🐌 | 📚 [⣿⣿⣿⣿⣧]272k` रेंडर करता है।

## प्रदाता + क्रेडेंशियल

जब उपयोग योग्य प्रदाता उपयोग प्रमाणीकरण रिज़ॉल्व नहीं किया जा सकता, तब उपयोग छिपा रहता है। OpenClaw
स्वचालित रूप से उन सक्षम प्रदाता plugins को खोजता है जो
`contracts.usageProviders` घोषित करते हैं और `resolveUsageAuth` तथा
`fetchUsageSnapshot` दोनों लागू करते हैं; कोर में कोई अलग प्रदाता अनुमति-सूची नहीं है। स्थिर
अनुबंध प्रत्येक प्रदाता plugin को आयात किए बिना खोज का दायरा सीमित रखता है। प्रत्येक
plugin अपने अपस्ट्रीम एंडपॉइंट और प्रतिक्रिया मैपिंग का स्वामी है।
साझा स्नैपशॉट योजना के नाम, कोटा विंडो, शेष राशियाँ, व्यय और बजट को
CLI, ऐप और Control UI उपभोक्ताओं के लिए प्रदाता-निरपेक्ष रखता है।

- **Anthropic (Claude)**: प्रमाणीकरण प्रोफ़ाइलों में OAuth टोकन। यदि OAuth टोकन में
  `user:profile` स्कोप नहीं है, तो सेट होने पर `claude.ai` वेब सत्र (`CLAUDE_AI_SESSION_KEY`,
  `CLAUDE_WEB_SESSION_KEY`, या `CLAUDE_WEB_COOKIE` में `sessionKey=` कुकी) पर फ़ॉलबैक करता है।
  जब Anthropic मॉडल-स्कोप सीमाएँ और सक्षम अतिरिक्त-उपयोग का मासिक व्यय/बजट रिपोर्ट करता है,
  तो उन्हें शामिल किया जाता है। इसके बजाय एक स्पष्ट Anthropic Admin API कुंजी या
  स्वतः-पहचानी गई `sk-ant-admin...` प्रदाता प्रोफ़ाइल, संगठन की 30-दिवसीय
  लागत और Messages API इतिहास दिखाती है।
- **ClawRouter**: API कुंजी (`CLAWROUTER_API_KEY`)। कॉन्फ़िगर होने पर मासिक बजट विंडो
  और प्रकारयुक्त USD बजट दिखाता है; अन्यथा समेकित व्यय और
  अनुरोध/टोकन/लागत सारांश दिखाता है।
- **DeepSeek**: env/config/auth स्टोर के माध्यम से API कुंजी (`DEEPSEEK_API_KEY`)।
  प्रदाता द्वारा रिपोर्ट की गई प्रत्येक मुद्रा की शेष राशि दिखाता है।
- **GitHub Copilot**: प्रमाणीकरण प्रोफ़ाइलों में OAuth टोकन।
- **Gemini CLI**: प्रमाणीकरण प्रोफ़ाइलों में OAuth टोकन।
- **MiniMax**: API कुंजी या MiniMax OAuth प्रमाणीकरण प्रोफ़ाइल। OpenClaw
  `minimax`, `minimax-cn`, और `minimax-portal` को एक ही MiniMax कोटा
  सतह मानता है, उपलब्ध होने पर संग्रहीत MiniMax OAuth को प्राथमिकता देता है, और अन्यथा
  `MINIMAX_CODE_PLAN_KEY`, `MINIMAX_CODING_API_KEY`, या `MINIMAX_API_KEY` पर फ़ॉलबैक करता है।
  उपयोग पोलिंग कॉन्फ़िगर होने पर `models.providers.minimax-portal.baseUrl`
  या `models.providers.minimax.baseUrl` से Coding Plan होस्ट प्राप्त करती है, और अन्यथा
  MiniMax CN होस्ट का उपयोग करती है।
  MiniMax के कच्चे `usage_percent` / `usagePercent` फ़ील्ड का अर्थ **शेष**
  कोटा है, इसलिए OpenClaw प्रदर्शन से पहले उन्हें उलटता है; उपलब्ध होने पर
  संख्या-आधारित फ़ील्ड को प्राथमिकता मिलती है।
  - विंडो लेबल उपलब्ध होने पर प्रदाता के घंटे/मिनट फ़ील्ड से आते हैं, फिर
    `start_time` / `end_time` अवधि पर फ़ॉलबैक करते हैं।
  - यदि coding-plan एंडपॉइंट `model_remains` लौटाता है, तो OpenClaw
    चैट-मॉडल प्रविष्टि को प्राथमिकता देता है, स्पष्ट
    `window_hours` / `window_minutes` फ़ील्ड अनुपस्थित होने पर टाइमस्टैम्प से विंडो लेबल प्राप्त करता है, और योजना
    लेबल में मॉडल का नाम शामिल करता है।
- **OpenAI (Codex/ChatGPT योजना)**: प्रमाणीकरण प्रोफ़ाइलों में OAuth टोकन (खाता आईडी मौजूद होने पर
  `ChatGPT-Account-Id` हेडर भेजा जाता है)। रिपोर्ट किए जाने पर ChatGPT योजना, रीसेट की जा सकने वाली
  Codex विंडो और क्रेडिट शेष दिखाता है। क्रेडिट प्रदाता
  क्रेडिट ही रहते हैं; OpenClaw उन्हें डॉलर के रूप में लेबल नहीं करता। जब कुंजी के पास Usage
  Dashboard पहुँच होती है, तब `OPENAI_ADMIN_KEY` संगठन की 30-दिवसीय लागत
  और completions-usage इतिहास जोड़ता है। इन्फ़रेंस क्रेडेंशियल कभी भी संगठन API को अग्रेषित नहीं किए जाते।
- **OpenRouter**: API कुंजी या OAuth-समर्थित API कुंजी (`OPENROUTER_API_KEY` या कोई प्रमाणीकरण
  प्रोफ़ाइल)। खाता क्रेडिट एंडपॉइंट को कुंजी कोटा एंडपॉइंट के साथ संयोजित करता है,
  ताकि क्रेडेंशियल द्वारा उन तक पहुँच संभव होने पर खाता शेष/व्यय, कुंजी बजट और दैनिक/साप्ताहिक/मासिक उपयोग
  दिखाई दें। दोनों में से कोई भी एंडपॉइंट स्वतंत्र रूप से स्नैपशॉट को
  समृद्ध कर सकता है।
- **Venice**: env/config/auth स्टोर के माध्यम से API कुंजी (`VENICE_API_KEY`)। रिपोर्ट किए जाने पर USD और
  DIEM शेष राशियाँ तथा DIEM युग आवंटन उपयोग दिखाता है।
- **Xiaomi MiMo**: दो अलग-अलग उपयोग सतहें। उपयोगानुसार भुगतान में API कुंजी
  (`XIAOMI_API_KEY`) का उपयोग होता है; Token Plan एक अलग कुंजी (`XIAOMI_TOKEN_PLAN_API_KEY`) का उपयोग करता है।
  वर्तमान में दोनों में से कोई भी कोटा विंडो रिपोर्ट नहीं करता।
- **z.ai**: env/config/auth स्टोर के माध्यम से API कुंजी (`ZAI_API_KEY` या `Z_AI_API_KEY`)।
