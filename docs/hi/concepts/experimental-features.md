प्रायोगिक सुविधाएँ स्पष्ट फ़्लैग के पीछे उपलब्ध पूर्वावलोकन सतहें हैं। स्थिर डिफ़ॉल्ट या दीर्घकालिक अनुबंध मिलने से पहले उन्हें वास्तविक दुनिया में अधिक उपयोग की आवश्यकता होती है।

- डिफ़ॉल्ट रूप से बंद, जब तक कोई दस्तावेज़ किसी सीमित स्वचालित सेटअप नियम का वर्णन न करे।
- इनकी संरचना और व्यवहार स्थिर कॉन्फ़िगरेशन की तुलना में अधिक तेज़ी से बदल सकते हैं।
- जहाँ स्थिर विकल्प पहले से मौजूद हो, उसे प्राथमिकता दें।
- पहले छोटे परिवेश में परीक्षण करने के बाद ही व्यापक रूप से लागू करें।

## Control UI लैब्स

ऐसे प्रयोगों को प्रबंधित करने के लिए **Settings → Agents & Tools → Labs** खोलें जिनके लिए
Control UI स्विच उपलब्ध है। किसी लैब को सक्षम या अक्षम करने पर प्रामाणिक Gateway
कॉन्फ़िगरेशन तुरंत पैच होता है; पृष्ठ केवल तभी पुनरारंभ संकेत दिखाता है जब किसी सुविधा को
इसकी आवश्यकता हो।

Code Mode वर्तमान में भेजी गई एकमात्र Labs प्रविष्टि है। Swarm अभी उपलब्ध नहीं है:
उसकी कॉन्फ़िगरेशन संरचना अभी भेजी नहीं गई है, इसलिए Control UI ऐसी काल्पनिक
कुंजी नहीं लिखता जो ऑपरेटर के कॉन्फ़िगरेशन को अमान्य कर दे।

## स्थानीय मॉडल लीन मोड

`agents.defaults.experimental.localModelLean: true` प्रत्येक टर्न पर एजेंट की प्रत्यक्ष सतह से भारी वैकल्पिक टूल हटा देता है: `browser`, `cron`, `message`, `image_generate`, `music_generate`, `video_generate`, `tts`, और `pdf`। स्पष्ट रूप से अनुमत या डिलीवरी के लिए आवश्यक टूल उपलब्ध रहते हैं, हालाँकि Tool Search उन्हें सीधे उपलब्ध कराने के बजाय कैटलॉग कर सकता है। जब `tools.toolSearch` पहले से सेट न हो, तो लीन मोड Plugin/MCP/क्लाइंट कैटलॉग के लिए संरचित Tool Search (`tool_search`, `tool_describe`, `tool_call`) को भी डिफ़ॉल्ट बनाता है। इसे एक एजेंट तक सीमित करने के लिए `agents.list[].experimental.localModelLean` का उपयोग करें।

ऑनबोर्डिंग के दौरान, सत्यापित `ollama` या `lmstudio` इनफ़रेंस रूट, मान अनुपस्थित होने पर, स्वचालित रूप से `agents.defaults.experimental.localModelLean: true` सेट करता है। OpenClaw दर्ज करता है कि सेटिंग ऑनबोर्डिंग से आई थी, इसलिए बाद में सत्यापित गैर-स्थानीय रूट केवल स्वचालित सेटिंग हटाता है। स्पष्ट रूप से कॉन्फ़िगर किया गया `true` या `false` सुरक्षित रखा जाता है। अन्य स्वयं-होस्ट किए गए और OpenAI-संगत प्रदाताओं का अनुमान मॉडल नामों या URL से नहीं लगाया जाता।

यदि आप Tool Search को पहले से वैश्विक रूप से समायोजित करते हैं, तो OpenClaw उस कॉन्फ़िगरेशन को यथावत छोड़ देता है। लीन-मोड Tool Search डिफ़ॉल्ट से बाहर निकलने के लिए `tools.toolSearch: false` सेट करें।

संरचित `tools` मोड में, लीन रन `exec` को Tool Search नियंत्रणों के साथ सीधे दृश्यमान रखते हैं, ताकि कोडिंग के लिए अनुकूलित स्थानीय मॉडल अभी भी अपना परिचित शेल पथ चुन सकें। इससे केवल स्कीमा दृश्यता बदलती है: सामान्य टूल नीति, सैंडबॉक्सिंग और exec अनुमोदन अब भी लागू होते हैं। स्पष्ट `code` और `directory` मोड अपना सामान्य Compaction व्यवहार बनाए रखते हैं।

### ये टूल क्यों

इन टूल के विवरण सबसे बड़े, पैरामीटर संरचनाएँ सबसे व्यापक हैं, या किसी छोटे मॉडल को सामान्य कोडिंग और वार्तालाप पथ से भटकाने की संभावना सबसे अधिक है। छोटे-कॉन्टेक्स्ट या अधिक सख़्त OpenAI-संगत बैकएंड पर इससे निम्न के बीच अंतर पड़ता है:

- टूल स्कीमा का प्रॉम्प्ट में समाना बनाम वार्तालाप इतिहास के लिए जगह कम करना।
- मॉडल द्वारा सही टूल चुनना बनाम बहुत अधिक समान स्कीमा के कारण विकृत टूल कॉल निकालना।
- Chat Completions अडैप्टर का संरचित-आउटपुट सीमाओं के भीतर रहना बनाम टूल-कॉल पेलोड आकार पर 400 त्रुटि।

उन्हें हटाने से केवल प्रत्यक्ष टूल सूची छोटी होती है। मॉडल के पास अब भी `read`, `write`, `edit`, `exec`, `apply_patch`, छवि समझ, वेब खोज/फ़ेच (कॉन्फ़िगर होने पर), मेमोरी और सत्र/एजेंट टूल होते हैं। जब तक आप `tools.toolSearch: false` सेट नहीं करते, अतिरिक्त कैटलॉग Tool Search के माध्यम से पहुँच योग्य रहते हैं; स्पष्ट टूल अनुमतियाँ किसी लीन एजेंट को सीमित वर्कफ़्लो में वापस शामिल कर सकती हैं।

### इसे कब चालू करें

यह सिद्ध होने के बाद लीन मोड सक्षम करें कि मॉडल Gateway से संवाद कर सकता है, लेकिन पूर्ण एजेंट टर्न ठीक से काम नहीं करते:

1. `openclaw infer model run --gateway --model <ref> --prompt "Reply with exactly: pong"` सफल होता है।
2. सामान्य एजेंट टर्न विकृत टूल कॉल, अत्यधिक बड़े प्रॉम्प्ट या मॉडल द्वारा अपने टूल की अनदेखी करने के कारण विफल होता है।
3. `localModelLean: true` को टॉगल करने से विफलता दूर हो जाती है।

### इसे कब बंद रखें

यदि आपका बैकएंड पूर्ण डिफ़ॉल्ट रनटाइम को ठीक से संभालता है, तो इसे बंद रखें। यह छोटे टूल सतह की आवश्यकता वाले स्थानीय स्टैक के लिए समाधान है, होस्ट किए गए मॉडल या पर्याप्त संसाधनों वाले स्थानीय सेटअप के लिए डिफ़ॉल्ट नहीं।

लीन मोड `tools.profile`, `tools.allow`/`tools.deny`, या मॉडल `compat.supportsTools: false` एस्केप हैच की जगह नहीं लेता। किसी विशिष्ट एजेंट पर स्थायी रूप से संकरी टूल सतह के लिए, उन स्थिर नियंत्रणों को प्राथमिकता दें।

### सक्षम करें

```json5
{
  agents: {
    defaults: {
      experimental: {
        localModelLean: true,
      },
    },
  },
}
```

केवल एक एजेंट के लिए:

```json5
{
  agents: {
    list: [
      {
        id: "local",
        model: "lmstudio/gemma-4-e4b-it",
        experimental: {
          localModelLean: true,
        },
      },
    ],
  },
}
```

फ़्लैग बदलने के बाद Gateway पुनरारंभ करें। लीन फ़िल्टरिंग `browser`, `cron`, `message`, `image_generate`, `music_generate`, `video_generate`, `tts`, और `pdf` को हटा देती है, जब तक आप उन्हें `tools.allow` या `tools.alsoAllow` के साथ स्पष्ट रूप से सुरक्षित न रखें; Tool Search सुरक्षित रखे गए टूल को सीधे उपलब्ध कराने के बजाय फिर भी कैटलॉग कर सकता है।

## प्रायोगिक का अर्थ छिपा हुआ नहीं है

किसी प्रायोगिक सुविधा को दस्तावेज़ों और कॉन्फ़िगरेशन पथ में स्वयं स्पष्ट रूप से ऐसा बताया जाना चाहिए, न कि स्थिर दिखने वाले डिफ़ॉल्ट नियंत्रण के पीछे छिपाया जाना चाहिए।
