प्रायोगिक सुविधाएँ स्पष्ट फ़्लैग के पीछे उपलब्ध ऑप्ट-इन पूर्वावलोकन सतहें हैं। स्थिर डिफ़ॉल्ट या दीर्घकालिक अनुबंध पाने से पहले उन्हें वास्तविक दुनिया में और अधिक उपयोग की आवश्यकता है।

- डिफ़ॉल्ट रूप से बंद, जब तक कोई दस्तावेज़ इसे सक्षम करने के लिए न कहे।
- इनका स्वरूप और व्यवहार स्थिर कॉन्फ़िगरेशन की तुलना में अधिक तेज़ी से बदल सकता है।
- जहाँ स्थिर मार्ग पहले से मौजूद हो, उसे प्राथमिकता दें।
- पहले छोटे परिवेश में परीक्षण करने के बाद ही व्यापक रूप से लागू करें।

## स्थानीय मॉडल लीन मोड

`agents.defaults.experimental.localModelLean: true` प्रत्येक बार एजेंट की प्रत्यक्ष सतह से भारी वैकल्पिक टूल हटा देता है: `browser`, `cron`, `message`, `image_generate`, `music_generate`, `video_generate`, `tts`, और `pdf`। स्पष्ट रूप से अनुमत या डिलीवरी के लिए आवश्यक टूल उपलब्ध रहते हैं, हालाँकि Tool Search उन्हें सीधे उपलब्ध कराने के बजाय सूचीबद्ध कर सकता है। जब `tools.toolSearch` पहले से सेट न हो, तो लीन मोड Plugin/MCP/क्लाइंट कैटलॉग के लिए संरचित Tool Search (`tool_search`, `tool_describe`, `tool_call`) को भी डिफ़ॉल्ट बनाता है। इसे केवल एक एजेंट तक सीमित करने के लिए `agents.list[].experimental.localModelLean` का उपयोग करें।

यदि आपने पहले से Tool Search को वैश्विक रूप से समायोजित किया है, तो OpenClaw उस कॉन्फ़िगरेशन को नहीं बदलता। लीन-मोड Tool Search डिफ़ॉल्ट से बाहर रहने के लिए `tools.toolSearch: false` सेट करें।

संरचित `tools` मोड में, लीन रन `exec` को Tool Search नियंत्रणों के साथ सीधे दृश्यमान रखते हैं, ताकि कोडिंग के लिए अनुकूलित स्थानीय मॉडल अब भी अपने परिचित शेल मार्ग को चुन सकें। इससे केवल स्कीमा की दृश्यता बदलती है: सामान्य टूल नीति, सैंडबॉक्सिंग और exec अनुमोदन अब भी लागू होते हैं। स्पष्ट `code` और `directory` मोड अपना सामान्य Compaction व्यवहार बनाए रखते हैं।

### ये टूल क्यों

इन टूल के विवरण सबसे बड़े, पैरामीटर संरचनाएँ सबसे व्यापक, या किसी छोटे मॉडल को सामान्य कोडिंग और वार्तालाप मार्ग से भटकाने की संभावना सबसे अधिक है। छोटे संदर्भ वाले या अधिक सख़्त OpenAI-संगत बैकएंड पर यही निम्न स्थितियों के बीच अंतर बनता है:

- टूल स्कीमा का प्रॉम्प्ट में समा जाना बनाम वार्तालाप इतिहास के लिए जगह कम कर देना।
- मॉडल द्वारा सही टूल चुनना बनाम बहुत अधिक समान स्कीमा के कारण विकृत टूल कॉल उत्पन्न करना।
- Chat Completions अडैप्टर का संरचित-आउटपुट सीमाओं के भीतर रहना बनाम टूल-कॉल पेलोड आकार के कारण 400 त्रुटि मिलना।

उन्हें हटाने से केवल प्रत्यक्ष टूल सूची छोटी होती है। मॉडल के पास फिर भी `read`, `write`, `edit`, `exec`, `apply_patch`, छवि समझ, वेब खोज/फ़ेच (कॉन्फ़िगर होने पर), मेमोरी और सत्र/एजेंट टूल रहते हैं। जब तक आप `tools.toolSearch: false` सेट नहीं करते, अतिरिक्त कैटलॉग Tool Search के माध्यम से उपलब्ध रहते हैं; स्पष्ट टूल अनुमतियाँ किसी लीन एजेंट को फिर से संक्षिप्त वर्कफ़्लो में शामिल कर सकती हैं।

### इसे कब चालू करें

यह सिद्ध करने के बाद लीन मोड सक्षम करें कि मॉडल Gateway से संचार कर सकता है, लेकिन पूर्ण एजेंट रन ठीक से काम नहीं करते:

1. `openclaw infer model run --gateway --model <ref> --prompt "Reply with exactly: pong"` सफल होता है।
2. सामान्य एजेंट रन विकृत टूल कॉल, अत्यधिक बड़े प्रॉम्प्ट या मॉडल द्वारा अपने टूल अनदेखा करने के कारण विफल होता है।
3. `localModelLean: true` को टॉगल करने से विफलता दूर हो जाती है।

### इसे कब बंद रखें

यदि आपका बैकएंड पूरे डिफ़ॉल्ट रनटाइम को बिना समस्या संभालता है, तो इसे बंद रखें। यह उन स्थानीय स्टैक के लिए एक समाधान है जिन्हें छोटी टूल सतह की आवश्यकता होती है, न कि होस्ट किए गए मॉडल या पर्याप्त संसाधनों वाले स्थानीय सेटअप के लिए डिफ़ॉल्ट।

लीन मोड `tools.profile`, `tools.allow`/`tools.deny`, या मॉडल के `compat.supportsTools: false` आपात विकल्प को प्रतिस्थापित नहीं करता। किसी विशिष्ट एजेंट के लिए स्थायी रूप से संकरी टूल सतह हेतु इन स्थिर नियंत्रणों को प्राथमिकता दें।

### सक्षम करें

```json5
{
  agents: {
    defaults: {
      experimental: {
        localModelLean: true,
      },
    },
  },
}
```

केवल एक एजेंट के लिए:

```json5
{
  agents: {
    list: [
      {
        id: "local",
        model: "lmstudio/gemma-4-e4b-it",
        experimental: {
          localModelLean: true,
        },
      },
    ],
  },
}
```

फ़्लैग बदलने के बाद Gateway को पुनः आरंभ करें। लीन फ़िल्टरिंग `browser`, `cron`, `message`, `image_generate`, `music_generate`, `video_generate`, `tts`, और `pdf` को हटा देती है, जब तक आप उन्हें `tools.allow` या `tools.alsoAllow` से स्पष्ट रूप से बनाए नहीं रखते; Tool Search संरक्षित टूल को सीधे उपलब्ध कराने के बजाय अब भी सूचीबद्ध कर सकता है।

## प्रायोगिक का अर्थ छिपा हुआ नहीं है

किसी प्रायोगिक सुविधा को दस्तावेज़ों और कॉन्फ़िगरेशन पथ में स्वयं स्पष्ट रूप से ऐसा बताना चाहिए, न कि स्थिर दिखने वाले डिफ़ॉल्ट नियंत्रण के पीछे छिपाना चाहिए।
