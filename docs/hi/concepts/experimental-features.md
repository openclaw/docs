प्रायोगिक सुविधाएँ स्पष्ट फ़्लैग के पीछे उपलब्ध पूर्वावलोकन सतहें हैं। स्थिर डिफ़ॉल्ट या दीर्घकालिक अनुबंध मिलने से पहले उन्हें वास्तविक दुनिया में और अधिक उपयोग की आवश्यकता है।

- डिफ़ॉल्ट रूप से बंद, जब तक कोई दस्तावेज़ सीमित स्वचालित सेटअप नियम का वर्णन न करे।
- इनकी संरचना और व्यवहार स्थिर कॉन्फ़िगरेशन की तुलना में अधिक तेज़ी से बदल सकते हैं।
- जब कोई स्थिर मार्ग पहले से मौजूद हो, तो उसे प्राथमिकता दें।
- पहले छोटे परिवेश में परीक्षण करने के बाद ही व्यापक रूप से लागू करें।

## Control UI लैब्स

Control UI स्विच वाले प्रयोगों को प्रबंधित करने के लिए **Settings → Agents & Tools → Labs** खोलें। किसी लैब को सक्षम या अक्षम करने से प्रामाणिक Gateway
कॉन्फ़िगरेशन तुरंत संशोधित होता है; पृष्ठ केवल तभी पुनः आरंभ करने का संकेत दिखाता है, जब किसी सुविधा को
उसकी आवश्यकता होती है।

Code Mode और Swarm वर्तमान में जारी किए गए लैब्स प्रविष्टियाँ हैं। दोनों स्विच
मौजूदा सत्यापित कॉन्फ़िगरेशन कुंजियाँ लिखते हैं और सामान्यतः Gateway को पुनः आरंभ किए बिना भविष्य के एजेंट
रन पर प्रभावी होते हैं।

## स्थानीय मॉडल लीन मोड

`agents.defaults.experimental.localModelLean: true` हर टर्न पर एजेंट की प्रत्यक्ष सतह से अधिक भार वाले वैकल्पिक टूल हटा देता है: `browser`, `cron`, `message`, `image_generate`, `music_generate`, `video_generate`, `tts`, और `pdf`। स्पष्ट रूप से अनुमत या डिलीवरी के लिए आवश्यक टूल उपलब्ध रहते हैं, हालाँकि Tool Search उन्हें सीधे उपलब्ध कराने के बजाय कैटलॉग कर सकता है। जब `tools.toolSearch` पहले से सेट न हो, तो लीन मोड Plugin/MCP/क्लाइंट कैटलॉग को संरचित Tool Search (`tool_search`, `tool_describe`, `tool_call`) पर भी डिफ़ॉल्ट करता है। इसे एक एजेंट तक सीमित करने के लिए `agents.list[].experimental.localModelLean` का उपयोग करें।

ऑनबोर्डिंग के दौरान, सत्यापित `ollama` या `lmstudio` इन्फ़रेंस मार्ग उस मान के अनुपस्थित होने पर स्वचालित रूप से `agents.defaults.experimental.localModelLean: true` सेट करता है। OpenClaw दर्ज करता है कि सेटिंग ऑनबोर्डिंग से आई थी, इसलिए बाद का सत्यापित गैर-स्थानीय मार्ग केवल स्वचालित सेटिंग हटाता है। स्पष्ट रूप से कॉन्फ़िगर किया गया `true` या `false` सुरक्षित रखा जाता है। अन्य स्वयं-होस्ट किए गए और OpenAI-संगत प्रदाताओं का अनुमान मॉडल नामों या URL से नहीं लगाया जाता।

यदि आप पहले से Tool Search को वैश्विक रूप से समायोजित करते हैं, तो OpenClaw उस कॉन्फ़िगरेशन को यथावत रखता है। लीन-मोड Tool Search डिफ़ॉल्ट से बाहर निकलने के लिए `tools.toolSearch: false` सेट करें।

संरचित `tools` मोड में, लीन रन `exec` को Tool Search नियंत्रणों के पास सीधे दृश्यमान रखते हैं, ताकि कोडिंग के लिए अनुकूलित स्थानीय मॉडल अब भी अपना परिचित शेल मार्ग चुन सकें। इससे केवल स्कीमा की दृश्यता बदलती है: सामान्य टूल नीति, सैंडबॉक्सिंग और exec अनुमोदन अब भी लागू होते हैं। स्पष्ट `code` और `directory` मोड अपना सामान्य Compaction व्यवहार बनाए रखते हैं।

### ये टूल क्यों

इन टूल के विवरण सबसे बड़े, पैरामीटर संरचनाएँ सबसे व्यापक या किसी छोटे मॉडल को सामान्य कोडिंग और वार्तालाप मार्ग से भटकाने की संभावना सबसे अधिक है। छोटे कॉन्टेक्स्ट या अधिक सख़्त OpenAI-संगत बैकएंड पर यही निम्न के बीच अंतर बनता है:

- टूल स्कीमा का प्रॉम्प्ट में समाना बनाम वार्तालाप इतिहास के लिए जगह कम करना।
- मॉडल द्वारा सही टूल चुनना बनाम बहुत अधिक समान स्कीमा के कारण विकृत टूल कॉल देना।
- Chat Completions अडैप्टर का संरचित-आउटपुट सीमाओं के भीतर रहना बनाम टूल-कॉल पेलोड आकार पर 400 त्रुटि।

उन्हें हटाने से केवल प्रत्यक्ष टूल सूची छोटी होती है। मॉडल के पास अब भी `read`, `write`, `edit`, `exec`, `apply_patch`, छवि समझ, वेब खोज/फ़ेच (कॉन्फ़िगर होने पर), मेमोरी और सत्र/एजेंट टूल रहते हैं। जब तक आप `tools.toolSearch: false` सेट नहीं करते, अतिरिक्त कैटलॉग Tool Search के माध्यम से सुलभ रहते हैं; स्पष्ट टूल अनुमतियाँ किसी लीन एजेंट को फिर से सीमित वर्कफ़्लो में शामिल कर सकती हैं।

### इसे कब चालू करें

जब आप प्रमाणित कर लें कि मॉडल Gateway से संचार कर सकता है, लेकिन पूर्ण एजेंट टर्न ठीक से काम नहीं करते, तब लीन मोड सक्षम करें:

1. `openclaw infer model run --gateway --model <ref> --prompt "Reply with exactly: pong"` सफल होता है।
2. सामान्य एजेंट टर्न विकृत टूल कॉल, अत्यधिक बड़े प्रॉम्प्ट या मॉडल द्वारा अपने टूल की अनदेखी के कारण विफल होता है।
3. `localModelLean: true` को टॉगल करने से विफलता दूर हो जाती है।

### इसे कब बंद रखें

यदि आपका बैकएंड पूरे डिफ़ॉल्ट रनटाइम को साफ़-सुथरे ढंग से संभालता है, तो इसे बंद रखें। यह उन स्थानीय स्टैक के लिए समाधान है जिन्हें छोटी टूल सतह की आवश्यकता होती है, न कि होस्ट किए गए मॉडल या पर्याप्त संसाधनों वाले स्थानीय सेटअप के लिए डिफ़ॉल्ट।

लीन मोड `tools.profile`, `tools.allow`/`tools.deny` या मॉडल के `compat.supportsTools: false` वैकल्पिक मार्ग को प्रतिस्थापित नहीं करता। किसी विशिष्ट एजेंट के लिए स्थायी रूप से संकरी टूल सतह हेतु उन स्थिर नियंत्रणों को प्राथमिकता दें।

### सक्षम करना

```json5
{
  agents: {
    defaults: {
      experimental: {
        localModelLean: true,
      },
    },
  },
}
```

केवल एक एजेंट के लिए:

```json5
{
  agents: {
    list: [
      {
        id: "local",
        model: "lmstudio/gemma-4-e4b-it",
        experimental: {
          localModelLean: true,
        },
      },
    ],
  },
}
```

फ़्लैग बदलने के बाद Gateway को पुनः आरंभ करें। लीन फ़िल्टरिंग `browser`, `cron`, `message`, `image_generate`, `music_generate`, `video_generate`, `tts`, और `pdf` को हटा देती है, जब तक आप उन्हें `tools.allow` या `tools.alsoAllow` के साथ स्पष्ट रूप से सुरक्षित न रखें; Tool Search सुरक्षित रखे गए टूल को सीधे उपलब्ध कराने के बजाय अब भी कैटलॉग कर सकता है।

## प्रायोगिक का अर्थ छिपा हुआ नहीं है

किसी प्रायोगिक सुविधा को दस्तावेज़ों और कॉन्फ़िगरेशन पथ में स्पष्ट रूप से ऐसा बताया जाना चाहिए, न कि स्थिर दिखाई देने वाले डिफ़ॉल्ट नियंत्रण के पीछे छिपाया जाना चाहिए।
