रन सक्रिय रहने के दौरान चैट चैनल पर टाइपिंग संकेतक भेजे जाते हैं। टाइपिंग **कब** शुरू हो, इसे नियंत्रित करने के लिए `agents.defaults.typingMode` और यह **कितनी बार** रीफ़्रेश हो, इसे नियंत्रित करने के लिए `typingIntervalSeconds` का उपयोग करें (कीपअलाइव अंतराल, डिफ़ॉल्ट 6 सेकंड)।

## डिफ़ॉल्ट

जब `agents.defaults.typingMode` **सेट नहीं हो**:

- **प्रत्यक्ष चैट**: मॉडल लूप शुरू होते ही टाइपिंग तुरंत शुरू हो जाती है।
- **उल्लेख वाली समूह चैट**: टाइपिंग तुरंत शुरू हो जाती है।
- **बिना उल्लेख वाली समूह चैट**: टाइपिंग तब शुरू होती है, जब स्वीकृत रन में उपयोगकर्ता को दिखाई देने वाली गतिविधि होती है, जैसे हार्नेस निष्पादन गतिविधि या संदेश टेक्स्ट।
- **Heartbeat रन**: Heartbeat रन शुरू होने पर टाइपिंग शुरू होती है, बशर्ते निर्धारित Heartbeat लक्ष्य टाइपिंग-समर्थ चैट हो और टाइपिंग अक्षम न हो।

## मोड

`agents.defaults.typingMode` को इनमें से किसी एक पर सेट करें:

- `never` - कभी भी कोई टाइपिंग संकेतक नहीं।
- `instant` - मॉडल लूप शुरू होते ही **तुरंत टाइपिंग शुरू करें**, भले ही रन बाद में केवल मौन उत्तर टोकन लौटाए।
- `thinking` - **पहले रीजनिंग डेल्टा** पर या टर्न स्वीकार होने के बाद सक्रिय हार्नेस निष्पादन पर टाइपिंग शुरू करें।
- `message` - **उपयोगकर्ता को दिखाई देने वाली पहली उत्तर गतिविधि** पर टाइपिंग शुरू करें, जैसे सक्रिय हार्नेस निष्पादन या गैर-मौन टेक्स्ट डेल्टा। `NO_REPLY` जैसे मौन उत्तर टोकन टेक्स्ट गतिविधि नहीं माने जाते।

"यह कितनी जल्दी सक्रिय होता है" का क्रम: `never` -> `message`/`thinking` -> `instant`।

## कॉन्फ़िगरेशन

एजेंट-स्तरीय डिफ़ॉल्ट सेट करें:

```json5
{
  agents: {
    defaults: {
      typingMode: "thinking",
      typingIntervalSeconds: 6,
    },
  },
}
```

प्रति सत्र मोड ओवरराइड करें:

```json5
{
  session: {
    typingMode: "message",
  },
}
```

## टिप्पणियाँ

- `message` मोड मौन उत्तर टोकन से शुरू नहीं होता, लेकिन सहायक का कोई टेक्स्ट उपलब्ध होने से पहले भी सक्रिय निष्पादन टाइपिंग दिखा सकता है।
- `thinking` अब भी स्ट्रीम की गई रीजनिंग (`reasoningLevel: "stream"`) पर प्रतिक्रिया करता है और रीजनिंग डेल्टा आने से पहले सक्रिय निष्पादन से भी शुरू हो सकता है।
- Heartbeat टाइपिंग निर्धारित डिलीवरी लक्ष्य के लिए सक्रियता संकेत है। यह `message` या `thinking` स्ट्रीम समय का अनुसरण करने के बजाय Heartbeat रन शुरू होते ही आरंभ होती है। इसे अक्षम करने के लिए `typingMode: "never"` सेट करें।
- जब Heartbeat लक्ष्य `"none"` हो, लक्ष्य निर्धारित न किया जा सके, Heartbeat के लिए चैट डिलीवरी अक्षम हो या चैनल टाइपिंग का समर्थन न करता हो, तब Heartbeat टाइपिंग नहीं दिखाते।
- `agents.defaults.typingIntervalSeconds` **रीफ़्रेश अंतराल** को नियंत्रित करता है, प्रारंभ समय को नहीं। डिफ़ॉल्ट: 6 सेकंड।
