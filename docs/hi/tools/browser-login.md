## मैन्युअल लॉगिन (अनुशंसित)

जब किसी साइट को लॉगिन की आवश्यकता हो, तो **host** ब्राउज़र प्रोफ़ाइल (openclaw ब्राउज़र) में **मैन्युअल रूप से साइन इन करें**।

मॉडल को अपने क्रेडेंशियल **न** दें। स्वचालित लॉगिन अक्सर एंटी-बॉट सुरक्षा को ट्रिगर करते हैं और खाते को लॉक कर सकते हैं।

मुख्य ब्राउज़र दस्तावेज़ों पर वापस जाएँ: [ब्राउज़र](/hi/tools/browser).

## कौन-सी Chrome प्रोफ़ाइल उपयोग की जाती है?

OpenClaw एक **समर्पित Chrome प्रोफ़ाइल** (`openclaw` नाम वाली, नारंगी रंगत वाले UI के साथ) को नियंत्रित करता है। यह आपकी दैनिक ब्राउज़र प्रोफ़ाइल से अलग है।

एजेंट ब्राउज़र टूल कॉल के लिए:

- डिफ़ॉल्ट विकल्प: एजेंट को अपने अलग-थलग `openclaw` ब्राउज़र का उपयोग करना चाहिए।
- `profile="user"` का उपयोग केवल तब करें जब मौजूदा लॉग-इन सत्र मायने रखते हों और उपयोगकर्ता किसी भी attach prompt पर क्लिक/स्वीकृति देने के लिए कंप्यूटर पर मौजूद हो।
- यदि आपके पास कई उपयोगकर्ता-ब्राउज़र प्रोफ़ाइल हैं, तो अनुमान लगाने के बजाय प्रोफ़ाइल को स्पष्ट रूप से निर्दिष्ट करें।

इसे एक्सेस करने के दो आसान तरीके:

1. **एजेंट से ब्राउज़र खोलने को कहें** और फिर स्वयं लॉग इन करें।
2. **इसे CLI के माध्यम से खोलें**:

```bash
openclaw browser start
openclaw browser open https://x.com
```

यदि आपके पास कई प्रोफ़ाइल हैं, तो `--browser-profile <name>` पास करें (डिफ़ॉल्ट `openclaw` है)।

## X/Twitter: अनुशंसित प्रवाह

- **पढ़ना/खोजना/थ्रेड्स:** **host** ब्राउज़र का उपयोग करें (मैन्युअल लॉगिन)।
- **अपडेट पोस्ट करना:** **host** ब्राउज़र का उपयोग करें (मैन्युअल लॉगिन)।

## सैंडबॉक्सिंग + host ब्राउज़र एक्सेस

सैंडबॉक्स किए गए ब्राउज़र सत्रों में बॉट पहचान ट्रिगर होने की संभावना **अधिक** होती है। X/Twitter (और अन्य सख्त साइटों) के लिए, **host** ब्राउज़र को प्राथमिकता दें।

यदि एजेंट सैंडबॉक्स किया गया है, तो ब्राउज़र टूल डिफ़ॉल्ट रूप से सैंडबॉक्स का उपयोग करता है। host नियंत्रण की अनुमति देने के लिए:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

फिर host ब्राउज़र स्वयं खोलें (CLI invocation हमेशा host ब्राउज़र के विरुद्ध चलते हैं):

```bash
openclaw browser open https://x.com --browser-profile openclaw
```

`sandbox.browser.allowHostControl: true` सेट होने के बाद एजेंट के `browser` टूल कॉल host को लक्षित कर सकते हैं। वैकल्पिक रूप से, अपडेट पोस्ट करने वाले एजेंट के लिए सैंडबॉक्सिंग अक्षम करें।
