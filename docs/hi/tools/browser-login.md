## मैन्युअल लॉगिन (अनुशंसित)

जब किसी साइट पर लॉगिन आवश्यक हो, तो होस्ट ब्राउज़र की `openclaw`
प्रोफ़ाइल में मैन्युअल रूप से साइन इन करें। मॉडल को अपने क्रेडेंशियल न दें: स्वचालित लॉगिन अक्सर
बॉट-रोधी सुरक्षा को सक्रिय कर देते हैं और अकाउंट लॉक कर सकते हैं।

X/Twitter और अन्य बॉट-संवेदनशील साइटों पर पढ़ने (खोज/थ्रेड) और
पोस्ट करने, दोनों के लिए होस्ट ब्राउज़र (मैन्युअल लॉगिन) का उपयोग करें। सैंडबॉक्स किए गए ब्राउज़र सत्रों में
बॉट पहचान सक्रिय होने की संभावना अधिक होती है।

मुख्य ब्राउज़र दस्तावेज़ पर वापस जाएँ: [ब्राउज़र](/hi/tools/browser)।

## किस Chrome प्रोफ़ाइल का उपयोग किया जाता है?

OpenClaw आपके दैनिक ब्राउज़र प्रोफ़ाइल से अलग, `openclaw` नामक एक समर्पित Chrome प्रोफ़ाइल
(नारंगी रंगत वाला UI) को नियंत्रित करता है।

एजेंट के ब्राउज़र टूल कॉल के लिए:

- डिफ़ॉल्ट विकल्प: एजेंट अपने पृथक `openclaw` ब्राउज़र का उपयोग करता है।
- `profile="user"` का उपयोग केवल तभी करें, जब मौजूदा लॉगिन किए हुए सत्र महत्वपूर्ण हों और आप
  किसी भी अटैच प्रॉम्प्ट पर क्लिक/स्वीकृति देने के लिए कंप्यूटर पर मौजूद हों।
- यदि आपके पास एकाधिक उपयोगकर्ता-ब्राउज़र प्रोफ़ाइल हैं, तो अनुमान लगाने के बजाय प्रोफ़ाइल को स्पष्ट रूप से
  निर्दिष्ट करें।

`openclaw` प्रोफ़ाइल तक पहुँचने के दो तरीके:

1. एजेंट से ब्राउज़र खोलने के लिए कहें, फिर स्वयं लॉगिन करें।
2. इसे CLI के माध्यम से खोलें:

```bash
openclaw browser start
openclaw browser open https://x.com
```

गैर-डिफ़ॉल्ट प्रोफ़ाइल के लिए, उपकमांड से पहले `--browser-profile <name>` रखें
(डिफ़ॉल्ट `openclaw` है):

```bash
openclaw browser --browser-profile <name> open https://x.com
```

## सैंडबॉक्सिंग: होस्ट ब्राउज़र की पहुँच की अनुमति दें

यदि एजेंट सैंडबॉक्स किया गया है, तो उसके `browser` टूल कॉल डिफ़ॉल्ट रूप से होस्ट के बजाय
सैंडबॉक्स ब्राउज़र को लक्षित करते हैं। एजेंट को इसके बजाय होस्ट ब्राउज़र लक्षित करने देने के लिए:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

CLI आह्वान हमेशा होस्ट ब्राउज़र को लक्षित करते हैं, सैंडबॉक्स को कभी नहीं, इसलिए इस सेटिंग की परवाह किए बिना आप
स्वयं होस्ट ब्राउज़र खोल सकते हैं:

```bash
openclaw browser --browser-profile openclaw open https://x.com
```

`sandbox.browser.allowHostControl: true` सेट होने के बाद, एजेंट के `browser`
टूल कॉल भी होस्ट को लक्षित कर सकते हैं। वैकल्पिक रूप से, अपडेट पोस्ट करने वाले
एजेंट के लिए सैंडबॉक्सिंग अक्षम करें।
