`openclaw agent` बिना किसी इनबाउंड चैट संदेश के कमांड लाइन से एजेंट का एक टर्न चलाता है। इसका उपयोग स्क्रिप्टेड कार्यप्रवाहों, परीक्षण और प्रोग्रामेटिक डिलीवरी के लिए करें। फ़्लैग और व्यवहार का पूरा संदर्भ:
[एजेंट CLI संदर्भ](/hi/cli/agent)।

## त्वरित शुरुआत

<Steps>
  <Step title="एजेंट का एक सरल टर्न चलाएँ">
    ```bash
    openclaw agent --agent main --message "आज मौसम कैसा है?"
    ```

    संदेश को Gateway के माध्यम से भेजता है और उत्तर प्रिंट करता है।

  </Step>

  <Step title="फ़ाइल से बहु-पंक्ति प्रॉम्प्ट भेजें">
    ```bash
    openclaw agent --agent ops --message-file ./task.md
    ```

    एजेंट संदेश के मुख्य भाग के रूप में एक मान्य UTF-8 फ़ाइल पढ़ता है।

  </Step>

  <Step title="किसी विशिष्ट एजेंट या सत्र को लक्षित करें">
    ```bash
    # किसी विशिष्ट एजेंट को लक्षित करें
    openclaw agent --agent ops --message "लॉग का सारांश दें"

    # किसी फ़ोन नंबर को लक्षित करें (सत्र कुंजी व्युत्पन्न करता है)
    openclaw agent --to +15555550123 --message "स्थिति अपडेट"

    # किसी मौजूदा सत्र का पुनः उपयोग करें
    openclaw agent --session-id abc123 --message "कार्य जारी रखें"

    # किसी सटीक सत्र कुंजी को लक्षित करें
    openclaw agent --session-key agent:ops:incident-42 --message "स्थिति का सारांश दें"
    ```

  </Step>

  <Step title="उत्तर को किसी चैनल पर डिलीवर करें">
    ```bash
    # WhatsApp पर डिलीवर करें (डिफ़ॉल्ट चैनल)
    openclaw agent --to +15555550123 --message "रिपोर्ट तैयार है" --deliver

    # Slack पर डिलीवर करें
    openclaw agent --agent ops --message "रिपोर्ट बनाएँ" \
      --deliver --reply-channel slack --reply-to "#reports"
    ```

  </Step>
</Steps>

## फ़्लैग

| फ़्लैग                       | विवरण                                                                  |
| --------------------------- | -------------------------------------------------------------------- |
| `--message <text>`          | भेजने के लिए इनलाइन संदेश                                               |
| `--message-file <path>`     | किसी मान्य UTF-8 फ़ाइल से संदेश पढ़ें (अधिकतम 4 MiB)                 |
| `--to <dest>`               | किसी लक्ष्य (फ़ोन, चैट आईडी) से सत्र कुंजी व्युत्पन्न करें                    |
| `--session-key <key>`       | स्पष्ट सत्र कुंजी का उपयोग करें                                          |
| `--agent <id>`              | कॉन्फ़िगर किए गए एजेंट को लक्षित करें (उसके `main` सत्र का उपयोग करता है)                  |
| `--session-id <id>`         | आईडी द्वारा किसी मौजूदा सत्र का पुनः उपयोग करें                                      |
| `--model <id>`              | इस रन के लिए मॉडल ओवरराइड (`provider/model` या मॉडल आईडी)           |
| `--local`                   | स्थानीय एम्बेडेड रनटाइम को बाध्य करें (Gateway को छोड़ें)                          |
| `--deliver`                 | उत्तर को किसी चैट चैनल पर भेजें                                     |
| `--channel <name>`          | डिलीवरी चैनल; `--agent` + `--to` के साथ, DM के दायरे पर भी लागू होता है     |
| `--reply-to <target>`       | डिलीवरी लक्ष्य ओवरराइड                                             |
| `--reply-channel <name>`    | डिलीवरी चैनल ओवरराइड                                            |
| `--reply-account <id>`      | डिलीवरी खाता आईडी ओवरराइड                                         |
| `--thinking <level>`        | चयनित मॉडल प्रोफ़ाइल के लिए चिंतन स्तर सेट करें                    |
| `--verbose <on\|full\|off>` | सत्र के लिए वर्बोज़ स्तर बनाए रखें (`full` टूल आउटपुट भी लॉग करता है) |
| `--timeout <seconds>`       | एजेंट टाइमआउट ओवरराइड करें (डिफ़ॉल्ट 600, या कॉन्फ़िग मान)                |
| `--json`                    | संरचित JSON आउटपुट करें                                               |

## व्यवहार

- डिफ़ॉल्ट रूप से, CLI **Gateway के माध्यम से** जाता है। वर्तमान मशीन पर एम्बेडेड रनटाइम को बाध्य करने के लिए `--local` जोड़ें।
- `--message` या `--message-file` में से ठीक एक पास करें। फ़ाइल संदेश वैकल्पिक UTF-8 BOM हटाने के बाद बहु-पंक्ति सामग्री को सुरक्षित रखते हैं। 4 MiB से बड़ी फ़ाइलें डिस्पैच से पहले अस्वीकार कर दी जाती हैं।
- क्षणिक हैंडशेक पुनः प्रयासों के बाद, Gateway टाइमआउट या बंद कनेक्शन के कारण कमांड stderr संकेत के साथ विफल हो जाता है; CLI कभी भी चुपचाप एम्बेडेड मोड में टर्न को दोबारा नहीं चलाता। Gateway किसी स्वीकृत टर्न को फिर भी पूरा कर सकता है, इसलिए `--local` के साथ पुनः प्रयास करने या दोबारा चलाने से पहले Gateway और सत्र की स्थिति सत्यापित करें।
- सत्र चयन: `--to` लक्ष्य से सत्र कुंजी व्युत्पन्न करता है (समूह/चैनल लक्ष्य अलगाव बनाए रखते हैं; प्रत्यक्ष चैट `main` में समाहित हो जाती हैं)। `--agent`, `--channel`, और `--to` का एक साथ उपयोग करने पर, रूटिंग चैनल के कैनोनिकल प्राप्तकर्ता और `session.dmScope` का अनुसरण करती है। केवल आउटबाउंड के लिए स्थिर पहचानें, एजेंट के मुख्य सत्र से अलग किए गए प्रदाता-स्वामित्व वाले सत्र का उपयोग करती हैं।
- `--session-key` एक स्पष्ट कुंजी चुनता है। एजेंट-प्रीफ़िक्स वाली कुंजियों को `agent:<agent-id>:<session-key>` का उपयोग करना आवश्यक है, और जब दोनों दिए जाएँ तो `--agent` को उस एजेंट आईडी से मेल खाना आवश्यक है। दिए जाने पर बिना प्रीफ़िक्स वाली गैर-सेंटिनल कुंजियाँ `--agent` के दायरे में आती हैं; उदाहरण के लिए, `--agent ops --session-key incident-42` को `agent:ops:incident-42` पर रूट किया जाता है। `--agent` के बिना, बिना प्रीफ़िक्स वाली गैर-सेंटिनल कुंजियाँ कॉन्फ़िगर किए गए डिफ़ॉल्ट एजेंट के दायरे में आती हैं। शाब्दिक `global` और `unknown` केवल तभी दायरे से बाहर रहते हैं, जब कोई `--agent` न दिया गया हो।
- `--reply-channel` और `--reply-account` केवल डिलीवरी को प्रभावित करते हैं।
- चिंतन और वर्बोज़ फ़्लैग सत्र स्टोर में बने रहते हैं।
- आउटपुट: डिफ़ॉल्ट रूप से सादा टेक्स्ट, या संरचित पेलोड + मेटाडेटा के लिए `--json`।
- `--json --deliver` के साथ, JSON में भेजी गई, दबाई गई, आंशिक और विफल डिलीवरी की स्थिति शामिल होती है। [JSON डिलीवरी स्थिति](/hi/cli/agent#json-delivery-status) देखें।

## उदाहरण

```bash
# JSON आउटपुट वाला सरल टर्न
openclaw agent --to +15555550123 --message "लॉग ट्रेस करें" --verbose on --json

# मॉडल ओवरराइड वाला टर्न
openclaw agent --agent ops --model openai/gpt-5.4 --message "लॉग का सारांश दें"

# चिंतन स्तर वाला टर्न
openclaw agent --session-id 1234 --message "इनबॉक्स का सारांश दें" --thinking medium

# फ़ाइल से बहु-पंक्ति प्रॉम्प्ट
openclaw agent --agent ops --message-file ./task.md

# सटीक सत्र कुंजी
openclaw agent --session-key agent:ops:incident-42 --message "स्थिति का सारांश दें"

# किसी एजेंट के दायरे में आने वाली पुरानी कुंजी
openclaw agent --agent ops --session-key incident-42 --message "स्थिति का सारांश दें"

# सत्र से भिन्न चैनल पर डिलीवर करें
openclaw agent --agent ops --message "चेतावनी" --deliver --reply-channel telegram --reply-to "@admin"
```
