`openclaw agent` कमांड लाइन से इनबाउंड चैट संदेश के बिना एक एजेंट टर्न चलाता है। इसका उपयोग स्क्रिप्टेड वर्कफ़्लो, परीक्षण और प्रोग्रामेटिक डिलीवरी के लिए करें। फ़्लैग और व्यवहार का पूरा संदर्भ:
[एजेंट CLI संदर्भ](/hi/cli/agent)।

## त्वरित शुरुआत

<Steps>
  <Step title="एक सामान्य एजेंट टर्न चलाएँ">
    ```bash
    openclaw agent --agent main --message "आज मौसम कैसा है?"
    ```

    संदेश को Gateway के माध्यम से भेजता है और उत्तर प्रिंट करता है।

  </Step>

  <Step title="फ़ाइल से बहु-पंक्ति प्रॉम्प्ट भेजें">
    ```bash
    openclaw agent --agent ops --message-file ./task.md
    ```

    मान्य UTF-8 फ़ाइल को एजेंट संदेश के मुख्य भाग के रूप में पढ़ता है।

  </Step>

  <Step title="किसी विशिष्ट एजेंट या सेशन को लक्षित करें">
    ```bash
    # किसी विशिष्ट एजेंट को लक्षित करें
    openclaw agent --agent ops --message "लॉग का सारांश दें"

    # किसी फ़ोन नंबर को लक्षित करें (सेशन कुंजी प्राप्त करता है)
    openclaw agent --to +15555550123 --message "स्थिति अपडेट"

    # किसी मौजूदा सेशन का पुनः उपयोग करें
    openclaw agent --session-id abc123 --message "कार्य जारी रखें"

    # किसी सटीक सेशन कुंजी को लक्षित करें
    openclaw agent --session-key agent:ops:incident-42 --message "स्थिति का सारांश दें"
    ```

  </Step>

  <Step title="उत्तर को किसी चैनल पर डिलीवर करें">
    ```bash
    # WhatsApp पर डिलीवर करें (डिफ़ॉल्ट चैनल)
    openclaw agent --to +15555550123 --message "रिपोर्ट तैयार है" --deliver

    # Slack पर डिलीवर करें
    openclaw agent --agent ops --message "रिपोर्ट बनाएँ" \
      --deliver --reply-channel slack --reply-to "#reports"
    ```

  </Step>
</Steps>

## फ़्लैग

| फ़्लैग                        | विवरण                                                          |
| --------------------------- | -------------------------------------------------------------------- |
| `--message <text>`          | भेजने के लिए इनलाइन संदेश                                               |
| `--message-file <path>`     | मान्य UTF-8 फ़ाइल से संदेश पढ़ें (अधिकतम 4 MiB)                 |
| `--to <dest>`               | लक्ष्य (फ़ोन, चैट आईडी) से सेशन कुंजी प्राप्त करें                    |
| `--session-key <key>`       | स्पष्ट सेशन कुंजी का उपयोग करें                                          |
| `--agent <id>`              | कॉन्फ़िगर किए गए एजेंट को लक्षित करें (इसके `main` सेशन का उपयोग करता है)                  |
| `--session-id <id>`         | आईडी द्वारा मौजूदा सेशन का पुनः उपयोग करें                                      |
| `--model <id>`              | इस रन के लिए मॉडल ओवरराइड (`provider/model` या मॉडल आईडी)           |
| `--local`                   | स्थानीय एम्बेडेड रनटाइम को बाध्य करें (Gateway को छोड़ें)                          |
| `--deliver`                 | उत्तर को चैट चैनल पर भेजें                                     |
| `--channel <name>`          | डिलीवरी चैनल; `--agent` + `--to` के साथ, DM का दायरा भी लागू करता है     |
| `--reply-to <target>`       | डिलीवरी लक्ष्य ओवरराइड                                             |
| `--reply-channel <name>`    | डिलीवरी चैनल ओवरराइड                                            |
| `--reply-account <id>`      | डिलीवरी खाता आईडी ओवरराइड                                         |
| `--thinking <level>`        | चयनित मॉडल प्रोफ़ाइल के लिए विचार स्तर सेट करें                    |
| `--verbose <on\|full\|off>` | सेशन के लिए वर्बोज़ स्तर बनाए रखें (`full` टूल आउटपुट भी लॉग करता है) |
| `--timeout <seconds>`       | एजेंट टाइमआउट ओवरराइड करें (डिफ़ॉल्ट 600 या कॉन्फ़िगरेशन मान)                |
| `--json`                    | संरचित JSON आउटपुट करें                                               |

## व्यवहार

- डिफ़ॉल्ट रूप से, CLI **Gateway के माध्यम से** जाता है। वर्तमान मशीन पर
  एम्बेडेड रनटाइम को बाध्य करने के लिए `--local` जोड़ें।
- `--message` या `--message-file` में से ठीक एक पास करें। फ़ाइल संदेश वैकल्पिक UTF-8 BOM
  हटाने के बाद बहु-पंक्ति सामग्री को सुरक्षित रखते हैं। 4 MiB से बड़ी फ़ाइलें
  डिस्पैच से पहले अस्वीकार कर दी जाती हैं।
- यदि Gateway अनुरोध विफल होता है, तो CLI स्थानीय एम्बेडेड रन पर **वापस लौटता है**;
  Gateway टाइमआउट मूल ट्रांसक्रिप्ट के साथ प्रतिस्पर्धा करने के बजाय नए सेशन के साथ
  वापस लौटता है।
- सेशन चयन: `--to` लक्ष्य से सेशन कुंजी प्राप्त करता है (समूह/चैनल लक्ष्य
  पृथक्करण बनाए रखते हैं; प्रत्यक्ष चैट `main` में समाहित हो जाती हैं)। `--agent`,
  `--channel` और `--to` को एक साथ उपयोग करने पर, रूटिंग चैनल के कैनोनिकल
  प्राप्तकर्ता और `session.dmScope` का अनुसरण करती है। स्थिर केवल-आउटबाउंड पहचानें,
  एजेंट के मुख्य सेशन से पृथक प्रदाता-स्वामित्व वाले सेशन का उपयोग करती हैं।
- `--session-key` एक स्पष्ट कुंजी चुनता है। एजेंट-उपसर्ग वाली कुंजियों को
  `agent:<agent-id>:<session-key>` का उपयोग करना आवश्यक है और दोनों दिए जाने पर `--agent` को
  उस एजेंट आईडी से मेल खाना आवश्यक है। दिए जाने पर बिना उपसर्ग वाली गैर-सेंटिनल कुंजियों का दायरा
  `--agent` तक होता है; उदाहरण के लिए, `--agent ops --session-key incident-42` को
  `agent:ops:incident-42` पर रूट किया जाता है। `--agent` के बिना, बिना उपसर्ग वाली गैर-सेंटिनल कुंजियों का दायरा
  कॉन्फ़िगर किए गए डिफ़ॉल्ट एजेंट तक होता है। लिटरल `global` और `unknown`
  केवल तभी बिना दायरे के रहते हैं जब कोई `--agent` न दिया गया हो; एम्बेडेड फ़ॉलबैक पथ
  उन सेंटिनल सेशन को कॉन्फ़िगर किए गए डिफ़ॉल्ट एजेंट पर रिज़ॉल्व करता है।
- `--reply-channel` और `--reply-account` केवल डिलीवरी को प्रभावित करते हैं।
- विचार और वर्बोज़ फ़्लैग सेशन स्टोर में बने रहते हैं।
- आउटपुट: डिफ़ॉल्ट रूप से सादा टेक्स्ट, या संरचित पेलोड + मेटाडेटा के लिए `--json`।
- `--json --deliver` के साथ, JSON में भेजे गए, रोके गए, आंशिक और विफल
  प्रेषणों की डिलीवरी स्थिति शामिल होती है। देखें
  [JSON डिलीवरी स्थिति](/hi/cli/agent#json-delivery-status)।

## उदाहरण

```bash
# JSON आउटपुट के साथ सामान्य टर्न
openclaw agent --to +15555550123 --message "लॉग ट्रेस करें" --verbose on --json

# मॉडल ओवरराइड के साथ टर्न
openclaw agent --agent ops --model openai/gpt-5.4 --message "लॉग का सारांश दें"

# विचार स्तर के साथ टर्न
openclaw agent --session-id 1234 --message "इनबॉक्स का सारांश दें" --thinking medium

# फ़ाइल से बहु-पंक्ति प्रॉम्प्ट
openclaw agent --agent ops --message-file ./task.md

# सटीक सेशन कुंजी
openclaw agent --session-key agent:ops:incident-42 --message "स्थिति का सारांश दें"

# एजेंट के दायरे वाली विरासती कुंजी
openclaw agent --agent ops --session-key incident-42 --message "स्थिति का सारांश दें"

# सेशन से भिन्न चैनल पर डिलीवर करें
openclaw agent --agent ops --message "चेतावनी" --deliver --reply-channel telegram --reply-to "@admin"
```
