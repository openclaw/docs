`screen` टूल एजेंट को ब्राउज़र-आधारित Control UI व्यवस्थित करने देता है। यह एक
टाइपयुक्त लेआउट और नेविगेशन सतह है, न कि स्क्रीनशॉट कैप्चर या ब्राउज़र
ऑटोमेशन।

टूल केवल तभी उपलब्ध होता है, जब आरंभ करने वाला क्लाइंट
`ui-commands` क्षमता घोषित करता है। टूल चलने के समय कम-से-कम एक सक्षम Control UI का
कनेक्ट होना अब भी आवश्यक है; अन्यथा Gateway `UNAVAILABLE` लौटाता है।

## कार्रवाइयाँ

| कार्रवाई                            | प्रभाव                                     | वैकल्पिक इनपुट                                |
| --------------------------------- | ------------------------------------------ | ---------------------------------------------- |
| `split_right`                     | लक्ष्य सेशन पेन को दाईं ओर विभाजित करें | `sessionKey` (डिफ़ॉल्ट रूप से वर्तमान सेशन) |
| `split_down`                      | लक्ष्य सेशन पेन को नीचे की ओर विभाजित करें     | `sessionKey` (डिफ़ॉल्ट रूप से वर्तमान सेशन) |
| `close_pane`                      | लक्ष्य सेशन पेन बंद करें              | `sessionKey` (डिफ़ॉल्ट रूप से वर्तमान सेशन) |
| `focus`                           | लक्ष्य सेशन पेन पर फ़ोकस करें              | `sessionKey` (डिफ़ॉल्ट रूप से वर्तमान सेशन) |
| `navigate`                        | लक्ष्य सेशन खोलें                    | `sessionKey` (डिफ़ॉल्ट रूप से वर्तमान सेशन) |
| `sidebar_show` / `sidebar_hide`   | मुख्य साइडबार दिखाएँ या छिपाएँ              | -                                              |
| `terminal_show` / `terminal_hide` | ऑपरेटर टर्मिनल पैनल दिखाएँ या छिपाएँ   | दिखाते समय `dock` (`bottom` या `right`)      |
| `browser_show` / `browser_hide`   | ब्राउज़र पैनल दिखाएँ या छिपाएँ             | दिखाते समय `dock` (`bottom` या `right`)      |

सफल कमांड, Gateway द्वारा टाइपयुक्त `ui.command` इवेंट प्रसारित करने के बाद,
`{ "ok": true }` लौटाता है।

## रूटिंग और सुरक्षा

प्रोटोकॉल v1 जानबूझकर कमांड को `ui-commands` घोषित करने वाले प्रत्येक कनेक्टेड Control UI को
भेजता है; यह किसी एक ब्राउज़र टैब को लक्षित नहीं करता। यह तब महत्वपूर्ण होता है, जब
एक ही ऑपरेटर के कई डैशबोर्ड खुले हों।

Gateway RPC के लिए `operator.write` आवश्यक है। टूल केवल प्रस्तुति की
स्थिति बदल सकता है: यह पिक्सेल नहीं पढ़ सकता, स्क्रीनशॉट नहीं ले सकता, पेज की मनमानी
सामग्री पर क्लिक नहीं कर सकता, या चयनित सेशन और ऑपरेटर
पैनलों की अनुमतियों को बायपास नहीं कर सकता।
