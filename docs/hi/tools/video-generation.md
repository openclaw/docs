OpenClaw एजेंट टेक्स्ट प्रॉम्प्ट, संदर्भ छवियों या
मौजूदा वीडियो से `video_generate` के माध्यम से वीडियो जनरेट करते हैं। सोलह प्रदाता बैकएंड
समर्थित हैं; एजेंट कॉन्फ़िगरेशन और उपलब्ध API कुंजियों के आधार पर
स्वचालित रूप से सही बैकएंड चुनता है।

<Note>
`video_generate` केवल तभी दिखाई देता है, जब कम-से-कम एक वीडियो-जनरेशन प्रदाता
उपलब्ध हो। यदि यह आपके एजेंट टूल में मौजूद नहीं है, तो प्रदाता API कुंजी सेट करें या
`agents.defaults.videoGenerationModel` कॉन्फ़िगर करें।
</Note>

`video_generate` के तीन रनटाइम मोड हैं, जिनका निर्धारण कॉल में दिए गए संदर्भ इनपुट
से होता है:

- `generate` - कोई संदर्भ मीडिया नहीं (टेक्स्ट-से-वीडियो)।
- `imageToVideo` - एक या अधिक संदर्भ छवियाँ।
- `videoToVideo` - एक या अधिक संदर्भ वीडियो।

प्रदाता इनमें से किसी भी मोड-उपसमूह का समर्थन कर सकते हैं। टूल सबमिशन से पहले
सक्रिय मोड को सत्यापित करता है और `action=list` में समर्थित मोड की जानकारी देता है।

## असिंक्रोनस जनरेशन कैसे काम करता है

वीडियो जनरेशन असिंक्रोनस है:

1. OpenClaw अनुरोध को प्रदाता के पास सबमिट करता है और तुरंत एक टास्क आईडी लौटाता है।
2. प्रदाता बैकग्राउंड में जॉब को प्रोसेस करता है (आमतौर पर प्रदाता और रिज़ॉल्यूशन के आधार पर 30 सेकंड से कई मिनट तक; धीमे कतार-आधारित प्रदाता कॉन्फ़िगर किए गए टाइमआउट तक चल सकते हैं)।
3. वीडियो तैयार होने पर OpenClaw उसी सेशन को आंतरिक पूर्णता इवेंट के साथ सक्रिय करता है।
4. एजेंट सेशन के सामान्य दृश्यमान-उत्तर मोड के माध्यम से इसकी जानकारी देता है:
   स्वचालित अंतिम उत्तर, या जब सेशन के लिए संदेश टूल आवश्यक हो तब `message(action="send")`।
   यदि अनुरोधकर्ता सेशन निष्क्रिय है, या उसे सक्रिय करना विफल रहता है और जनरेट किया गया
   मीडिया अभी भी पूर्णता उत्तर में मौजूद नहीं है, तो OpenClaw मीडिया के साथ
   एक इडेम्पोटेंट प्रत्यक्ष फ़ॉलबैक भेजता है।

जब कोई जॉब चल रहा हो, तो उसी सेशन में डुप्लिकेट `video_generate` कॉल
दूसरा जनरेशन शुरू करने के बजाय वर्तमान टास्क की स्थिति लौटाते हैं।
नया जनरेशन ट्रिगर किए बिना जाँचने के लिए `action: "status"`, या CLI से
`openclaw tasks list` / `openclaw tasks show <lookup>` का उपयोग करें
([बैकग्राउंड टास्क](/hi/automation/tasks) देखें)।

सेशन-समर्थित एजेंट रन के बाहर (उदाहरण के लिए, प्रत्यक्ष टूल आह्वान),
टूल इनलाइन जनरेशन पर वापस जाता है और उसी टर्न में अंतिम मीडिया पथ
लौटाता है।

जब प्रदाता बाइट्स लौटाता है, तो जनरेट की गई वीडियो फ़ाइलें OpenClaw-प्रबंधित
मीडिया स्टोरेज में सहेजी जाती हैं। डिफ़ॉल्ट सीमा 16MB (साझा वीडियो मीडिया
सीमा) है; बड़े रेंडर के लिए `agents.defaults.mediaMaxMb` इसे बढ़ाता है। जब कोई
प्रदाता होस्ट किया गया आउटपुट URL भी लौटाता है, तो स्थानीय स्थायित्व द्वारा अत्यधिक बड़ी फ़ाइल
अस्वीकार किए जाने पर टास्क विफल करने के बजाय OpenClaw वह URL डिलीवर करता है।

### टास्क जीवनचक्र

| स्थिति       | अर्थ                                                                                                |
| ----------- | ------------------------------------------------------------------------------------------------------ |
| `queued`    | टास्क बन गया है और प्रदाता द्वारा इसे स्वीकार किए जाने की प्रतीक्षा कर रहा है।                                                   |
| `running`   | प्रदाता प्रोसेस कर रहा है (आमतौर पर प्रदाता और रिज़ॉल्यूशन के आधार पर 30 सेकंड से कई मिनट तक)। |
| `succeeded` | वीडियो तैयार है; एजेंट सक्रिय होता है और इसे वार्तालाप में पोस्ट करता है।                                         |
| `failed`    | प्रदाता त्रुटि या टाइमआउट; एजेंट त्रुटि विवरण के साथ सक्रिय होता है।                                         |

CLI से स्थिति जाँचें:

```bash
openclaw tasks list
openclaw tasks show <lookup>
openclaw tasks cancel <lookup>
```

एकल-छवि से वीडियो के लिए भूमिका छोड़ दें या `first_frame` का उपयोग करें।
</Note>

### शैली नियंत्रण

<ParamField path="aspectRatio" type="string">
  पक्षानुपात संकेत, जैसे `1:1`, `16:9`, `9:16`, `adaptive`, या कोई प्रदाता-विशिष्ट मान। OpenClaw प्रत्येक प्रदाता के अनुसार असमर्थित मानों को सामान्यीकृत करता है या अनदेखा करता है।
</ParamField>
<ParamField path="resolution" type="string">रिज़ॉल्यूशन संकेत, जैसे `360P`, `480P`, `540P`, `720P`, `768P`, `1080P`, `4K`, या कोई प्रदाता-विशिष्ट मान। OpenClaw प्रत्येक प्रदाता के अनुसार असमर्थित मानों को सामान्यीकृत करता है या अनदेखा करता है।</ParamField>
<ParamField path="durationSeconds" type="number">
  सेकंड में लक्षित अवधि (निकटतम प्रदाता-समर्थित मान तक पूर्णांकित)।
</ParamField>
<ParamField path="size" type="string">जब प्रदाता इसका समर्थन करता हो, तब आकार संकेत।</ParamField>
<ParamField path="audio" type="boolean">
  समर्थित होने पर आउटपुट में जनरेट किया गया ऑडियो सक्षम करें। `audioRef*` (इनपुट) से अलग।
</ParamField>
<ParamField path="watermark" type="boolean">समर्थित होने पर प्रदाता की वॉटरमार्किंग चालू या बंद करें।</ParamField>

`adaptive` एक प्रदाता-विशिष्ट संकेतक है: इसे उन प्रदाताओं को ज्यों का त्यों
अग्रेषित किया जाता है जो अपनी क्षमताओं में `adaptive` घोषित करते हैं
(उदाहरण के लिए, BytePlus Seedance इनपुट छवि के आयामों से अनुपात का स्वतः पता
लगाने के लिए इसका उपयोग करता है)। जो प्रदाता इसे घोषित नहीं करते, वे टूल परिणाम
में `details.ignoredOverrides` के माध्यम से यह मान दिखाते हैं, ताकि इसे हटाया जाना दृश्यमान रहे।

### उन्नत

<ParamField path="action" type='"generate" | "status" | "list"' default="generate">
  `"status"` वर्तमान सत्र का कार्य लौटाता है; `"list"` प्रदाताओं की जाँच करता है।
</ParamField>
<ParamField path="model" type="string">प्रदाता/मॉडल ओवरराइड (उदाहरण के लिए, `runway/gen4.5`)।</ParamField>
<ParamField path="filename" type="string">आउटपुट फ़ाइल नाम का संकेत।</ParamField>
<ParamField path="timeoutMs" type="number">मिलीसेकंड में वैकल्पिक प्रदाता संचालन टाइमआउट। इसे छोड़ने पर OpenClaw कॉन्फ़िगर किया हुआ `agents.defaults.videoGenerationModel.timeoutMs` उपयोग करता है; अन्यथा, उपलब्ध होने पर Plugin द्वारा निर्धारित प्रदाता का डिफ़ॉल्ट मान उपयोग करता है।</ParamField>
<ParamField path="providerOptions" type="object">
  JSON ऑब्जेक्ट के रूप में प्रदाता-विशिष्ट विकल्प (उदाहरण के लिए, `{"seed": 42, "draft": true}`)।
  टाइप की गई स्कीमा घोषित करने वाले प्रदाता कुंजियों और प्रकारों को सत्यापित करते हैं;
  अज्ञात कुंजियाँ या असंगतियाँ फ़ॉलबैक के दौरान उस उम्मीदवार को छोड़ देती हैं।
  घोषित स्कीमा के बिना प्रदाता विकल्पों को ज्यों का त्यों प्राप्त करते हैं। प्रत्येक
  प्रदाता क्या स्वीकार करता है, यह देखने के लिए `video_generate action=list` चलाएँ।
</ParamField>

<Note>
सभी प्रदाता सभी पैरामीटर का समर्थन नहीं करते। OpenClaw अवधि को निकटतम
प्रदाता-समर्थित मान में सामान्यीकृत करता है और जब कोई फ़ॉलबैक प्रदाता अलग
नियंत्रण सतह प्रदान करता है, तब आकार-से-पक्षानुपात जैसे रूपांतरित ज्यामिति
संकेतों को पुनः मैप करता है। पूरी तरह असमर्थित ओवरराइड को यथासंभव प्रयास के
आधार पर अनदेखा किया जाता है और टूल परिणाम में चेतावनियों के रूप में रिपोर्ट
किया जाता है। कठोर क्षमता सीमाएँ (जैसे बहुत अधिक संदर्भ इनपुट) सबमिशन से पहले
विफल हो जाती हैं। टूल परिणाम लागू सेटिंग्स की रिपोर्ट करते हैं;
`details.normalization` अनुरोधित-से-लागू किसी भी रूपांतरण को दर्ज करता है।
</Note>

संदर्भ इनपुट रनटाइम मोड चुनते हैं:

- कोई संदर्भ मीडिया नहीं -> `generate`
- कोई भी छवि संदर्भ -> `imageToVideo`
- कोई भी वीडियो संदर्भ -> `videoToVideo`
- संदर्भ ऑडियो इनपुट समाधान किए गए मोड को **नहीं** बदलते; वे छवि/वीडियो
  संदर्भों द्वारा चुने गए किसी भी मोड के ऊपर लागू होते हैं और केवल उन
  प्रदाताओं के साथ काम करते हैं जो `maxInputAudios` घोषित करते हैं।

छवि और वीडियो संदर्भों का मिश्रण एक स्थिर साझा क्षमता सतह नहीं है।
प्रत्येक अनुरोध में एक ही संदर्भ प्रकार को प्राथमिकता दें।

#### फ़ॉलबैक और टाइप किए गए विकल्प

कुछ क्षमता जाँचें टूल सीमा के बजाय फ़ॉलबैक स्तर पर लागू होती हैं, इसलिए
प्राथमिक प्रदाता की सीमाएँ पार करने वाला अनुरोध भी किसी सक्षम फ़ॉलबैक पर
चल सकता है:

- जब अनुरोध में ऑडियो संदर्भ हों, तब `maxInputAudios` (या `0`) घोषित न
  करने वाले सक्रिय उम्मीदवार को छोड़ दिया जाता है; अगले उम्मीदवार को आज़माया
  जाता है। यही सुरक्षा जाँच छवि और वीडियो संदर्भों की संख्या पर
  `maxInputImages`/`maxInputVideos` के विरुद्ध लागू होती है।
- यदि सक्रिय उम्मीदवार का `maxDurationSeconds` अनुरोधित `durationSeconds` से कम है
  और कोई `supportedDurationSeconds` सूची घोषित नहीं है -> उसे छोड़ दिया जाता है।
- यदि अनुरोध में `providerOptions` है और सक्रिय उम्मीदवार स्पष्ट रूप से टाइप
  की गई `providerOptions` स्कीमा घोषित करता है -> दी गई कुंजियाँ स्कीमा में
  न होने या मान प्रकारों के मेल न खाने पर उसे छोड़ दिया जाता है। घोषित स्कीमा
  के बिना प्रदाता विकल्पों को ज्यों का त्यों प्राप्त करते हैं (पश्चगामी-संगत
  पास-थ्रू)। कोई प्रदाता रिक्त स्कीमा (`capabilities.providerOptions: {}`) घोषित करके सभी
  प्रदाता विकल्पों से बाहर हो सकता है, जिससे प्रकार असंगति के समान ही उसे
  छोड़ दिया जाता है।

किसी अनुरोध में छोड़ने का पहला कारण `warn` स्तर पर लॉग होता है, ताकि ऑपरेटर
देख सकें कि उनका प्राथमिक प्रदाता क्यों छोड़ा गया; लंबी फ़ॉलबैक शृंखलाओं को शांत
रखने के लिए बाद के कारण `debug` स्तर पर लॉग होते हैं। यदि प्रत्येक उम्मीदवार
छोड़ दिया जाता है, तो समेकित त्रुटि में प्रत्येक का छोड़ने का कारण शामिल होता है।

## कार्रवाइयाँ

| कार्रवाई     | यह क्या करती है                                                                                             |
| ---------- | -------------------------------------------------------------------------------------------------------- |
| `generate` | डिफ़ॉल्ट। दिए गए प्रॉम्प्ट और वैकल्पिक संदर्भ इनपुट से एक वीडियो बनाएँ।                             |
| `status`   | कोई अन्य जनरेशन शुरू किए बिना वर्तमान सत्र के प्रगतिरत वीडियो कार्य की स्थिति जाँचें। |
| `list`     | उपलब्ध प्रदाता, मॉडल और उनकी क्षमताएँ दिखाएँ।                                                |

## मॉडल चयन

OpenClaw मॉडल का समाधान इस क्रम में करता है:

1. **`model` टूल पैरामीटर** - यदि एजेंट कॉल में कोई मॉडल निर्दिष्ट करता है।
2. कॉन्फ़िगरेशन से **`videoGenerationModel.primary`**।
3. क्रमानुसार **`videoGenerationModel.fallbacks`**।
4. **स्वतः-पहचान** - वे प्रदाता जिनके पास मान्य प्रमाणीकरण है; वर्तमान
   डिफ़ॉल्ट प्रदाता से शुरू करके, फिर शेष प्रदाता वर्णानुक्रम में।

यदि कोई प्रदाता विफल होता है, तो अगला उम्मीदवार स्वतः आज़माया जाता है। यदि सभी
उम्मीदवार विफल होते हैं, तो त्रुटि में प्रत्येक प्रयास का विवरण शामिल होता है।

केवल स्पष्ट `model`, `primary`, और `fallbacks` प्रविष्टियों
का उपयोग करने के लिए `agents.defaults.mediaGenerationAutoProviderFallback: false` सेट करें।

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "google/veo-3.1-fast-generate-preview",
        fallbacks: ["runway/gen4.5", "qwen/wan2.6-t2v"],
        timeoutMs: 180000, // वैकल्पिक प्रति-टूल प्रदाता अनुरोध टाइमआउट ओवरराइड
      },
    },
  },
}
```

    चेतावनी के साथ अनदेखा किया जाता है।
  </Accordion>
  <Accordion title="OpenRouter">
    OpenRouter के एसिंक्रोनस `/videos` API का उपयोग करता है। OpenClaw
    जॉब सबमिट करता है, `polling_url` को पोल करता है और फिर या तो `unsigned_urls` या
    दस्तावेज़ीकृत जॉब सामग्री एंडपॉइंट से डाउनलोड करता है। बंडल किया गया `google/veo-3.1-fast` डिफ़ॉल्ट
    4/6/8 सेकंड की अवधियाँ, `720P`/`1080P` रिज़ॉल्यूशन और
    `16:9`/`9:16` आस्पेक्ट रेशियो उपलब्ध बताता है।
  </Accordion>
  <Accordion title="Qwen">
    Alibaba के समान DashScope बैकएंड। संदर्भ इनपुट दूरस्थ
    `http(s)` URL होने चाहिए; स्थानीय फ़ाइलें पहले ही अस्वीकार कर दी जाती हैं।
  </Accordion>
  <Accordion title="Runway">
    डेटा URI के माध्यम से स्थानीय फ़ाइलों का समर्थन करता है। वीडियो-टू-वीडियो के लिए
    `runway/gen4_aleph` आवश्यक है। केवल-टेक्स्ट रन में `16:9` और `9:16`
    आस्पेक्ट रेशियो उपलब्ध होते हैं।
  </Accordion>
  <Accordion title="Together">
    केवल एक इमेज संदर्भ।
  </Accordion>
  <Accordion title="Vydra">
    प्रमाणीकरण हटाने वाले रीडायरेक्ट से बचने के लिए सीधे `https://www.vydra.ai/api/v1` का
    उपयोग करता है। `veo3` केवल टेक्स्ट-टू-वीडियो के रूप में बंडल किया गया है; `kling` के लिए
    दूरस्थ इमेज URL आवश्यक है।
  </Accordion>
  <Accordion title="xAI">
    डिफ़ॉल्ट `grok-imagine-video` मॉडल टेक्स्ट-टू-वीडियो, एक
    प्रथम-फ़्रेम इमेज से इमेज-टू-वीडियो, xAI
    `reference_images` के माध्यम से अधिकतम 7 `reference_image` इनपुट और दूरस्थ वीडियो संपादन/विस्तार प्रवाहों का समर्थन करता है। जनरेशन का डिफ़ॉल्ट
    `480P` है; `aspectRatio` छोड़े जाने पर एकल-इमेज इमेज-टू-वीडियो स्रोत अनुपात प्राप्त करता है।
    वीडियो संपादन/विस्तार इनपुट की ज्यामिति प्राप्त करते हैं और
    आस्पेक्ट-रेशियो या रिज़ॉल्यूशन ओवरराइड स्वीकार नहीं करते। विस्तार 2-10
    सेकंड स्वीकार करता है।

    `grok-imagine-video-1.5` केवल इमेज-टू-वीडियो है: ठीक एक इमेज प्रदान करें।
    यह 1-15 सेकंड और `480P`, `720P` या `1080P` का समर्थन करता है, जिसका डिफ़ॉल्ट
    `480P` है; स्रोत इमेज अनुपात प्राप्त करने के लिए `aspectRatio` को छोड़ दें। प्रीव्यू
    और दिनांकित 1.5 आइडेंटिफ़ायर पर समान सत्यापन लागू होता है और उन्हें
    बिना बदलाव के अग्रेषित किया जाता है।

  </Accordion>
</AccordionGroup>

## प्रदाता क्षमता मोड

साझा वीडियो-जनरेशन अनुबंध केवल सपाट समग्र सीमाओं के बजाय
मोड-विशिष्ट क्षमताओं का समर्थन करता है। नए प्रदाता कार्यान्वयनों को
स्पष्ट मोड ब्लॉक को प्राथमिकता देनी चाहिए:

```typescript
capabilities: {
  generate: {
    maxVideos: 1,
    maxDurationSeconds: 10,
    supportsResolution: true,
  },
  imageToVideo: {
    enabled: true,
    maxVideos: 1,
    maxInputImages: 1,
    maxInputImagesByModel: { "provider/reference-to-video": 9 },
    maxDurationSeconds: 5,
  },
  videoToVideo: {
    enabled: true,
    maxVideos: 1,
    maxInputVideos: 1,
    maxDurationSeconds: 5,
  },
}
```

`maxInputImages` और `maxInputVideos` जैसे सपाट समग्र फ़ील्ड
ट्रांसफ़ॉर्म-मोड समर्थन उपलब्ध बताने के लिए **पर्याप्त नहीं** हैं। प्रदाताओं को
`generate`, `imageToVideo` और `videoToVideo` स्पष्ट रूप से घोषित करने चाहिए, ताकि लाइव
परीक्षण, अनुबंध परीक्षण और साझा `video_generate` टूल
मोड समर्थन का निर्धारणात्मक रूप से सत्यापन कर सकें।

जब किसी प्रदाता का एक मॉडल अन्य मॉडलों से अधिक व्यापक संदर्भ-इनपुट समर्थन देता हो,
तो पूरे मोड की सीमा बढ़ाने के बजाय `maxInputImagesByModel`, `maxInputVideosByModel` या
`maxInputAudiosByModel` का उपयोग करें।

## लाइव परीक्षण

साझा बंडल किए गए प्रदाताओं के लिए वैकल्पिक लाइव कवरेज:

```bash
OPENCLAW_LIVE_TEST=1 pnpm test:live -- extensions/video-generation-providers.live.test.ts
```

रेपो रैपर:

```bash
pnpm test:live:media video
```

यह लाइव फ़ाइल डिफ़ॉल्ट रूप से संग्रहीत प्रमाणीकरण
प्रोफ़ाइलों से पहले पहले से निर्यात किए गए प्रदाता एनवायरनमेंट वेरिएबल का उपयोग करती है और डिफ़ॉल्ट रूप से रिलीज़-सुरक्षित स्मोक चलाती है:

- `generate` स्वीप के प्रत्येक गैर-FAL प्रदाता के लिए।
- एक-सेकंड का लॉब्स्टर प्रॉम्प्ट।
- प्रत्येक प्रदाता की ऑपरेशन सीमा
  `OPENCLAW_LIVE_VIDEO_GENERATION_TIMEOUT_MS` से (`180000` डिफ़ॉल्ट रूप से)।

FAL वैकल्पिक है, क्योंकि प्रदाता-पक्ष की क्यू विलंबता रिलीज़
समय पर हावी हो सकती है:

```bash
pnpm test:live:media video --video-providers fal
```

घोषित ट्रांसफ़ॉर्म मोड भी चलाने के लिए `OPENCLAW_LIVE_VIDEO_GENERATION_FULL_MODES=1` सेट करें,
जिन्हें साझा स्वीप स्थानीय मीडिया के साथ सुरक्षित रूप से चला सकता है:

- `imageToVideo`, जब `capabilities.imageToVideo.enabled`।
- `videoToVideo`, जब `capabilities.videoToVideo.enabled` और
  प्रदाता/मॉडल साझा स्वीप में बफ़र-समर्थित स्थानीय वीडियो इनपुट स्वीकार करता हो।

वर्तमान में साझा `videoToVideo` लाइव लेन केवल तभी `runway` को कवर करती है, जब आप
`runway/gen4_aleph` चुनते हैं।

## कॉन्फ़िगरेशन

अपने OpenClaw कॉन्फ़िगरेशन में डिफ़ॉल्ट वीडियो-जनरेशन मॉडल सेट करें:

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "qwen/wan2.6-t2v",
        fallbacks: ["qwen/wan2.6-r2v-flash"],
      },
    },
  },
}
```

या CLI के माध्यम से:

```bash
openclaw config set agents.defaults.videoGenerationModel.primary "qwen/wan2.6-t2v"
```
