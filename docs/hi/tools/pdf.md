`pdf` एक या अधिक PDF दस्तावेज़ों का विश्लेषण करता है और टेक्स्ट लौटाता है। यह Anthropic और Google मॉडल पर नेटिव दस्तावेज़ इनपुट का उपयोग करता है और हर अन्य प्रदाता के लिए टेक्स्ट/इमेज निष्कर्षण पर वापस जाता है।

## उपलब्धता

यह टूल केवल तभी पंजीकृत होता है, जब OpenClaw एजेंट के लिए PDF-सक्षम मॉडल का समाधान कर सकता है। समाधान का क्रम:

1. `agents.defaults.pdfModel` (स्पष्ट प्राथमिक/फ़ॉलबैक)
2. `agents.defaults.imageModel` (स्पष्ट प्राथमिक/फ़ॉलबैक)
3. एजेंट का समाधान किया गया सत्र/डिफ़ॉल्ट मॉडल, यदि उसका प्रदाता नेटिव PDF इनपुट (Anthropic, Google) का समर्थन करता है या उसमें पहले से कॉन्फ़िगर किया गया विज़न मॉडल है
4. उपयोग योग्य प्रमाणीकरण वाले स्वतः पहचाने गए इमेज/विज़न-सक्षम प्रदाता, जिनमें पहले नेटिव-PDF प्रदाताओं को प्राथमिकता दी जाती है

उपयोग से पहले प्रत्येक फ़ॉलबैक उम्मीदवार का प्रमाणीकरण जाँचा जाता है, इसलिए कॉन्फ़िगर किया गया `provider/model` केवल तभी गिना जाता है, जब OpenClaw एजेंट के लिए उस प्रदाता को प्रमाणित कर सकता है। यदि कोई उपयोग योग्य मॉडल समाधान नहीं होता, तो `pdf` टूल उपलब्ध नहीं कराया जाता।

<ParamField path="prompt" type="string" default="इस PDF दस्तावेज़ का विश्लेषण करें।">

</ParamField>

<ParamField path="password" type="string">
एन्क्रिप्टेड PDF के लिए पासवर्ड। अनुरोध में प्रत्येक PDF पर लागू होता है; केवल निष्कर्षण फ़ॉलबैक मोड द्वारा उपयोग किया जाता है।
</ParamField>

<ParamField path="model" type="string">
`provider/model` प्रारूप में वैकल्पिक मॉडल ओवरराइड।
</ParamField>

<ParamField path="maxBytesMb" type="number">
प्रति-PDF आकार सीमा, MB में। डिफ़ॉल्ट `agents.defaults.pdfMaxBytesMb`, या सेट न होने पर `10`।
</ParamField>

टिप्पणियाँ:

- `pdf` और `pdfs` को लोड करने से पहले मर्ज करके डुप्लिकेट हटाए जाते हैं; कम-से-कम एक आवश्यक है।
- `pages` को 1-आधारित पेज संख्याओं के रूप में पार्स किया जाता है, डुप्लिकेट हटाकर क्रमबद्ध किया जाता है और `agents.defaults.pdfMaxPages` (डिफ़ॉल्ट `20`) तक सीमित किया जाता है। ऐसी रेंज जो सीमा के भीतर किसी पेज से मेल नहीं खाती, मॉडल कॉल से पहले त्रुटि देती है।

## समर्थित PDF संदर्भ

- स्थानीय फ़ाइल पथ (`~` विस्तार सहित)
- `file://` URL
- `http://` और `https://` URL
- OpenClaw द्वारा प्रबंधित इनबाउंड संदर्भ, जैसे `media://inbound/<id>`

अन्य URI स्कीम (उदाहरण के लिए `ftp://`) `details.error = "unsupported_pdf_reference"` लौटाती हैं। टूल के सैंडबॉक्स में चलने पर रिमोट `http(s)` URL अस्वीकार कर दिए जाते हैं। केवल-वर्कस्पेस फ़ाइल नीति सक्षम होने पर, अनुमत रूट से बाहर के स्थानीय पथ अस्वीकार कर दिए जाते हैं; OpenClaw के इनबाउंड मीडिया स्टोर के अंतर्गत प्रबंधित इनबाउंड संदर्भ और पुनः चलाए गए पथ फिर भी अनुमत हैं।

## निष्पादन मोड

### नेटिव प्रदाता मोड

प्रदाता `anthropic` और `google` के लिए उपयोग किया जाता है (वर्तमान में केवल यही प्रदाता नेटिव PDF दस्तावेज़ समर्थन घोषित करते हैं)। अपरिष्कृत PDF बाइट सीधे प्रदाता API को प्रत्येक फ़ाइल के लिए नेटिव दस्तावेज़/इनलाइन-PDF भाग के रूप में भेजे जाते हैं।

सीमाएँ:

- `pages` समर्थित नहीं है; सेट होने पर टूल `pages is not supported with native PDF providers` थ्रो करता है।
- `password` समर्थित नहीं है; सेट होने पर टूल `password is not supported with native PDF providers` थ्रो करता है। एन्क्रिप्टेड PDF के लिए गैर-नेटिव मॉडल का उपयोग करें।

### निष्कर्षण फ़ॉलबैक मोड

हर अन्य प्रदाता के लिए उपयोग किया जाता है।

1. बंडल किए गए `document-extract` Plugin के माध्यम से चुने गए पेजों (अधिकतम `agents.defaults.pdfMaxPages`, डिफ़ॉल्ट `20`) से टेक्स्ट निकालता है, जो टेक्स्ट और इमेज निष्कर्षण के लिए `clawpdf` पैकेज (PDFium WebAssembly) का उपयोग करता है।
2. यदि निकाला गया टेक्स्ट `200` वर्णों से छोटा है, तो उन्हीं पेजों को PNG इमेज में रेंडर करता है। रेंडर बजट कुल `4,000,000` पिक्सेल है, जिसे इमेज की आवश्यकता वाले सभी पेजों के बीच साझा किया जाता है (प्रति पेज नहीं, बल्कि प्रत्येक शेष पेज के लिए आनुपातिक रूप से आवंटित), इसलिए जिन टेक्स्ट पेजों में पहले से पर्याप्त टेक्स्ट है, वे रेंडरिंग को पूरी तरह छोड़ देते हैं।
3. निकाले गए टेक्स्ट (और रेंडर की गई किसी भी इमेज) को प्रॉम्प्ट के साथ चुने गए मॉडल को भेजता है।

विवरण:

- एन्क्रिप्टेड PDF शीर्ष-स्तरीय `password` पैरामीटर के साथ खुलती हैं।
- यदि मॉडल में इमेज इनपुट नहीं है और निकालने योग्य टेक्स्ट भी नहीं है, तो टूल त्रुटि देता है।
- यदि इमेज रेंडरिंग विफल होती है, तो OpenClaw इमेज को हटा देता है और निकाले गए टेक्स्ट के साथ जारी रखता है।
- यदि लक्ष्य मॉडल केवल-टेक्स्ट है और निष्कर्षण ने इमेज बनाई हैं, तो OpenClaw इमेज को हटा देता है और केवल टेक्स्ट भेजता है।

## कॉन्फ़िगरेशन

```json5
{
  agents: {
    defaults: {
      pdfModel: {
        primary: "anthropic/claude-opus-4-6",
        fallbacks: ["openai/gpt-5.4-mini"],
      },
      pdfMaxBytesMb: 10,
      pdfMaxPages: 20,
    },
  },
}
```

| कुंजी                             | डिफ़ॉल्ट | अर्थ                                                                                   |
| ------------------------------- | ------- | ----------------------------------------------------------------------------------------- |
| `agents.defaults.pdfModel`      | सेट नहीं   | स्पष्ट प्राथमिक/फ़ॉलबैक PDF मॉडल; `imageModel`, फिर सत्र मॉडल पर वापस जाता है। |
| `agents.defaults.pdfMaxBytesMb` | `10`    | प्रति-PDF आकार सीमा, MB में।                                                                   |
| `agents.defaults.pdfMaxPages`   | `20`    | प्रति PDF संसाधित किए जाने वाले अधिकतम पेज।                                                              |

फ़ील्ड के पूर्ण विवरण के लिए [कॉन्फ़िगरेशन संदर्भ](/hi/gateway/config-agents#agent-defaults) देखें।

## आउटपुट विवरण

टूल `content[0].text` में टेक्स्ट और `details` में संरचित मेटाडेटा लौटाता है।

सामान्य `details` फ़ील्ड:

- `model`: समाधान किया गया मॉडल संदर्भ (`provider/model`)
- `native`: नेटिव प्रदाता मोड के लिए `true`, फ़ॉलबैक के लिए `false`
- `attempts`: सफलता से पहले विफल हुए फ़ॉलबैक प्रयास

पथ फ़ील्ड:

- एकल PDF इनपुट: `details.pdf`
- एकाधिक PDF इनपुट: `pdf` प्रविष्टियों के साथ `details.pdfs[]`
- सैंडबॉक्स पथ पुनर्लेखन मेटाडेटा (जब लागू हो): `rewrittenFrom`

## त्रुटि व्यवहार

| स्थिति                         | परिणाम                                                         |
| --------------------------------- | -------------------------------------------------------------- |
| कोई PDF इनपुट नहीं                      | `pdf required: provide a path or URL to a PDF document` थ्रो करता है |
| 10 से अधिक PDF                 | `details.error = "too_many_pdfs"`                              |
| असमर्थित संदर्भ स्कीम      | `details.error = "unsupported_pdf_reference"`                  |
| नेटिव प्रदाता के साथ `pages`    | `pages is not supported with native PDF providers` थ्रो करता है      |
| नेटिव प्रदाता के साथ `password` | `password is not supported with native PDF providers` थ्रो करता है   |

## उदाहरण

एकल PDF:

```json
{
  "pdf": "/tmp/report.pdf",
  "prompt": "इस रिपोर्ट को 5 बुलेट में सारांशित करें"
}
```

एकाधिक PDF:

```json
{
  "pdfs": ["/tmp/q1.pdf", "/tmp/q2.pdf"],
  "prompt": "दोनों दस्तावेज़ों में जोखिमों और समय-सीमा के बदलावों की तुलना करें"
}
```

पेज-फ़िल्टर किया गया फ़ॉलबैक मॉडल:

```json
{
  "pdf": "https://example.com/report.pdf",
  "pages": "1-3,7",
  "model": "openai/gpt-5.4-mini",
  "prompt": "केवल ग्राहक को प्रभावित करने वाली घटनाएँ निकालें"
}
```

निष्कर्षण फ़ॉलबैक के साथ एन्क्रिप्टेड PDF:

```json
{
  "pdf": "/tmp/locked.pdf",
  "password": "example-password",
  "model": "openai/gpt-5.4-mini",
  "prompt": "इस अनुबंध को सारांशित करें"
}
```
