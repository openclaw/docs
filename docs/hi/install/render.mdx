रेपो के `render.yaml` Blueprint का उपयोग करके [Render](https://render.com) पर OpenClaw परिनियोजित करें। यह सेवा, डिस्क और एनवायरनमेंट वेरिएबल्स को एक ही फ़ाइल में घोषित करता है।

## पूर्वापेक्षाएँ

- एक [Render खाता](https://render.com) (निःशुल्क टियर उपलब्ध है)
- आपके पसंदीदा [मॉडल प्रोवाइडर](/hi/providers) की API कुंजी

## परिनियोजन

[Render पर परिनियोजित करें](https://render.com/deploy?repo=https://github.com/openclaw/openclaw)

यह `render.yaml` से एक Render सेवा बनाता है, Docker इमेज बिल्ड करता है और उसे परिनियोजित करता है। आपकी सेवा का URL `https://<service-name>.onrender.com` पैटर्न का अनुसरण करता है।

## Blueprint

```yaml
services:
  - type: web
    name: openclaw
    runtime: docker
    plan: starter
    healthCheckPath: /health
    envVars:
      - key: OPENCLAW_GATEWAY_PORT
        value: "8080"
      - key: OPENCLAW_STATE_DIR
        value: /data/.openclaw
      - key: OPENCLAW_WORKSPACE_DIR
        value: /data/workspace
      - key: OPENCLAW_GATEWAY_TOKEN
        generateValue: true # एक सुरक्षित टोकन अपने आप जनरेट करता है
    disk:
      name: openclaw-data
      mountPath: /data
      sizeGB: 1
```

| सुविधा               | उद्देश्य                                                    |
| --------------------- | ---------------------------------------------------------- |
| `runtime: docker`     | रेपो की Dockerfile से बिल्ड करता है                          |
| `healthCheckPath`     | Render `/health` की निगरानी करता है और अस्वस्थ इंस्टेंस को रीस्टार्ट करता है |
| `generateValue: true` | क्रिप्टोग्राफ़िक रूप से सुरक्षित मान अपने आप जनरेट करता है            |
| `disk`                | स्थायी स्टोरेज, जो पुनः परिनियोजन के बाद भी बना रहता है                 |

## प्लान चुनना

| प्लान      | बंद होना         | डिस्क          | इसके लिए सर्वोत्तम                      |
| --------- | ----------------- | ------------- | ----------------------------- |
| निःशुल्क      | 15 मिनट निष्क्रिय रहने के बाद | उपलब्ध नहीं | परीक्षण, डेमो                |
| Starter   | कभी नहीं             | 1GB+          | व्यक्तिगत उपयोग, छोटी टीमें     |
| Standard+ | कभी नहीं             | 1GB+          | प्रोडक्शन, कई चैनल |

Blueprint में डिफ़ॉल्ट रूप से `starter` होता है। निःशुल्क टियर का उपयोग करने के लिए, अपने फ़ोर्क की `render.yaml` में `plan: free` बदलें — ध्यान दें कि स्थायी डिस्क न होने पर OpenClaw की स्थिति प्रत्येक परिनियोजन के साथ रीसेट हो जाती है।

## परिनियोजन के बाद

### Control UI खोलें

वेब डैशबोर्ड `https://<your-service>.onrender.com/` पर उपलब्ध है। साझा सीक्रेट का उपयोग करके कनेक्ट करें: अपने आप जनरेट हुआ `OPENCLAW_GATEWAY_TOKEN` (इसे **Dashboard → your service → Environment** में खोजें), या यदि आपने पासवर्ड प्रमाणीकरण अपनाया है तो अपने पासवर्ड का उपयोग करें।

### लॉग

**Dashboard → your service → Logs** बिल्ड लॉग (Docker इमेज बनाना), परिनियोजन लॉग (सेवा शुरू होना) और रनटाइम लॉग (एप्लिकेशन आउटपुट) दिखाता है।

### शेल एक्सेस

**Dashboard → your service → Shell** एक शेल सेशन खोलता है। स्थायी डिस्क `/data` पर माउंट होती है।

### एनवायरनमेंट वेरिएबल्स

वेरिएबल्स को **Dashboard → your service → Environment** में संपादित करें। बदलाव अपने आप पुनः परिनियोजन ट्रिगर करते हैं।

### अपने आप परिनियोजन

कनेक्ट किए गए रेपो की ब्रांच में नया कमिट आने पर Render अपने आप पुनः परिनियोजन करता है। यदि आपने अपने फ़ोर्क के बजाय सीधे `openclaw/openclaw` से परिनियोजित किया है, तो इसे ट्रिगर करने के लिए आपके पास पुश एक्सेस नहीं होगा। इसलिए Dashboard से मैन्युअल Blueprint सिंक चलाकर अपडेट करें या सेवा को अपने फ़ोर्क की ओर इंगित करें।

## कस्टम डोमेन

1. **Dashboard → your service → Settings → Custom Domains**
2. अपना डोमेन जोड़ें
3. निर्देशानुसार DNS कॉन्फ़िगर करें (`*.onrender.com` के लिए CNAME)
4. Render अपने आप TLS प्रमाणपत्र जारी करता है

## स्केलिंग

- **वर्टिकल**: अधिक CPU/RAM के लिए प्लान बदलें। आमतौर पर OpenClaw के लिए यह पर्याप्त होता है।
- **हॉरिज़ॉन्टल**: इंस्टेंस की संख्या बढ़ाएँ (Standard प्लान और उससे ऊपर)। चूँकि OpenClaw रनटाइम स्थिति को स्थानीय डिस्क पर रखता है, इसलिए इसके लिए स्टिकी सेशन या बाहरी स्थिति प्रबंधन आवश्यक है।

## बैकअप और माइग्रेशन

Render Dashboard के शेल से किसी भी समय स्थिति, कॉन्फ़िगरेशन, प्रमाणीकरण प्रोफ़ाइल और वर्कस्पेस एक्सपोर्ट करें:

```bash
openclaw backup create
```

यह एक पोर्टेबल बैकअप आर्काइव बनाता है। [बैकअप](/hi/cli/backup) देखें।

## समस्या निवारण

### सेवा शुरू नहीं होती

Render Dashboard में परिनियोजन लॉग जाँचें। सामान्य समस्याएँ:

- `OPENCLAW_GATEWAY_TOKEN` मौजूद नहीं है — सत्यापित करें कि इसे **Dashboard → Environment** में सेट किया गया है
- पोर्ट मेल नहीं खाता — सुनिश्चित करें कि `OPENCLAW_GATEWAY_PORT=8080`, ताकि Gateway उस पोर्ट से बाइंड हो जिसकी Render अपेक्षा करता है

### धीमे कोल्ड स्टार्ट (निःशुल्क टियर)

निःशुल्क टियर की सेवाएँ 15 मिनट तक निष्क्रिय रहने के बाद बंद हो जाती हैं; बंद होने के बाद पहले अनुरोध पर कंटेनर शुरू होने में कुछ सेकंड लगते हैं। हमेशा चालू रखने के लिए Starter में अपग्रेड करें।

### पुनः परिनियोजन के बाद डेटा हानि

यह निःशुल्क टियर पर होता है (स्थायी डिस्क नहीं होती)। सशुल्क प्लान में अपग्रेड करें या Render शेल से `openclaw backup create` का उपयोग करके नियमित रूप से बैकअप एक्सपोर्ट करें।

### हेल्थ चेक विफलताएँ

यदि बिल्ड सफल होते हैं, लेकिन परिनियोजन विफल होते हैं, तो संभव है कि सेवा शुरू होने में बहुत अधिक समय ले रही हो या `/health` तक पहुँचा न जा सके। जाँचें:

- त्रुटियों के लिए बिल्ड लॉग
- क्या कंटेनर स्थानीय रूप से `docker build && docker run` के साथ चलता है
