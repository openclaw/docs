<Warning>
Bun OpenClaw CLI या Gateway नहीं चला सकता क्योंकि यह आवश्यक `node:sqlite` API प्रदान नहीं करता। सभी OpenClaw रनटाइम कमांड के लिए समर्थित Node संस्करण इंस्टॉल करें।
</Warning>

Bun का उपयोग वैकल्पिक डिपेंडेंसी इंस्टॉलर और पैकेज-स्क्रिप्ट रनर के रूप में किया जा सकता है। डिफ़ॉल्ट पैकेज मैनेजर `pnpm` ही है, जो पूरी तरह समर्थित है और दस्तावेज़ टूलिंग द्वारा उपयोग किया जाता है। Bun `pnpm-lock.yaml` का उपयोग नहीं कर सकता और इसे अनदेखा करता है।

## इंस्टॉल करना

<Steps>
  <Step title="डिपेंडेंसी इंस्टॉल करें">
    ```sh
    bun install
    ```

    `bun.lock` / `bun.lockb` को gitignore किया गया है, इसलिए रिपॉज़िटरी में कोई अनावश्यक बदलाव नहीं होता। लॉकफ़ाइल में लिखना पूरी तरह छोड़ने के लिए:

    ```sh
    bun install --no-save
    ```

  </Step>
  <Step title="बिल्ड और परीक्षण करें">
    ```sh
    bun run build
    bun run vitest run
    ```

    OpenClaw को स्वयं लॉन्च करने वाले कमांड अब भी Node के माध्यम से चलाए जाने चाहिए।

  </Step>
</Steps>

## लाइफ़साइकल स्क्रिप्ट

Bun डिपेंडेंसी लाइफ़साइकल स्क्रिप्ट को तब तक ब्लॉक करता है, जब तक उन पर स्पष्ट रूप से भरोसा न किया गया हो। इस रिपॉज़िटरी के लिए आम तौर पर ब्लॉक की जाने वाली स्क्रिप्ट आवश्यक नहीं हैं:

- `baileys` `preinstall`: जाँचता है कि Node का मेजर संस्करण >= 20 है (OpenClaw के लिए Node 22.22.3+, 24.15+, या 25.9+ आवश्यक है और Node 24 अनुशंसित है)
- `protobufjs` `postinstall`: असंगत संस्करण योजनाओं के बारे में चेतावनियाँ देता है (कोई बिल्ड आर्टिफ़ैक्ट नहीं)

यदि आपको कोई ऐसी रनटाइम समस्या आती है जिसके लिए इन स्क्रिप्ट की आवश्यकता है, तो उन पर स्पष्ट रूप से भरोसा करें:

```sh
bun pm trust baileys protobufjs
```

## सावधानियाँ

कुछ पैकेज स्क्रिप्ट आंतरिक रूप से `pnpm` को हार्डकोड करती हैं (उदाहरण के लिए `check:docs`, `ui:*`, `protocol:check`)। उन्हें `bun run` के माध्यम से चलाने पर भी शेल से `pnpm` चलाया जाता है, इसलिए उन्हें सीधे `pnpm` के माध्यम से चलाएँ।
