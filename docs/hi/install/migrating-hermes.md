बंडल किया गया Hermes माइग्रेशन प्रदाता `HERMES_HOME` और सक्रिय Hermes प्रोफ़ाइल का अनुसरण करता है, और उपलब्ध न होने पर macOS/Linux पर `~/.hermes` या Windows पर `%LOCALAPPDATA%\hermes` का उपयोग करता है। यह लागू करने से पहले प्रत्येक बदलाव का पूर्वावलोकन करता है, योजनाओं और रिपोर्टों में गोपनीय मानों को छिपाता है, और किसी भी चीज़ को छूने से पहले सत्यापित OpenClaw बैकअप लिखता है। स्पष्ट `--from` पथ को हमेशा प्राथमिकता मिलती है।

<Note>
आयात के लिए नया OpenClaw सेटअप आवश्यक है। यदि आपके पास पहले से स्थानीय OpenClaw स्थिति है, तो पहले कॉन्फ़िगरेशन, क्रेडेंशियल, सत्र और कार्यक्षेत्र रीसेट करें, या योजना की समीक्षा करने के बाद `--overwrite` के साथ सीधे `openclaw migrate apply hermes` का उपयोग करें।
</Note>

<AccordionGroup>
  <Accordion title="मॉडल कॉन्फ़िगरेशन">

  </Accordion>

## क्या केवल संग्रह में रहता है

प्रदाता मैन्युअल समीक्षा के लिए इन्हें माइग्रेशन रिपोर्ट निर्देशिका में कॉपी करता है, लेकिन इन्हें लाइव OpenClaw कॉन्फ़िगरेशन या क्रेडेंशियल में लोड **नहीं** करता:

- `plugins/`
- `sessions/`
- `logs/`
- `cron/`
- `mcp-tokens/`
- `plans/`, `workspace/`, `skins/`, और `kanban/`
- `pairing/` और `platforms/` स्टोर, साथ ही Gateway रूटिंग/प्रक्रिया स्थिति
- `state.db`, `hermes_state.db`, `projects.db`, `response_store.db`, `memory_store.db`, `verification_evidence.db`, `kanban.db`, और `retaindb_queue.db`

OpenClaw इस स्थिति को स्वतः निष्पादित करने या विश्वसनीय मानने से इनकार करता है, क्योंकि प्रणालियों के बीच प्रारूप और विश्वास-संबंधी मान्यताएँ बदल सकती हैं। संग्रह की समीक्षा करने के बाद अपनी आवश्यक सामग्री को मैन्युअल रूप से स्थानांतरित करें।

## अनुशंसित प्रवाह

<Steps>
  <Step title="योजना का पूर्वावलोकन करें">
    ```bash
    openclaw migrate hermes --dry-run
    ```

    योजना उन सभी चीज़ों को सूचीबद्ध करती है जो बदलेंगी, जिनमें टकराव, छोड़े गए आइटम और संवेदनशील आइटम शामिल हैं। आउटपुट में नेस्टेड गोपनीय-जैसी कुंजियाँ छिपाई जाती हैं।

  </Step>
  <Step title="बैकअप के साथ लागू करें">
    ```bash
    openclaw migrate apply hermes --yes
    ```

    OpenClaw लागू करने से पहले बैकअप बनाता और सत्यापित करता है। यह गैर-इंटरैक्टिव उदाहरण केवल गैर-गोपनीय स्थिति आयात करता है। क्रेडेंशियल प्रॉम्प्ट का इंटरैक्टिव रूप से उत्तर देने के लिए `--yes` के बिना चलाएँ, या स्वचालित रन में समर्थित क्रेडेंशियल शामिल करने के लिए `--include-secrets` जोड़ें।

  </Step>
  <Step title="डॉक्टर चलाएँ">
    ```bash
    openclaw doctor
    ```

    [डॉक्टर](/hi/gateway/doctor) किसी भी लंबित कॉन्फ़िगरेशन माइग्रेशन को फिर से लागू करता है और आयात के दौरान उत्पन्न समस्याओं की जाँच करता है।

  </Step>
  <Step title="पुनः प्रारंभ और सत्यापन करें">
    ```bash
    openclaw gateway restart
    openclaw status
    ```

    पुष्टि करें कि Gateway स्वस्थ है और आपके आयातित मॉडल, मेमोरी और Skills लोड हो गए हैं।

  </Step>
</Steps>

## टकराव प्रबंधन

जब योजना टकरावों की रिपोर्ट करती है, तब लागू करने की प्रक्रिया आगे बढ़ने से इनकार करती है (लक्ष्य पर कोई फ़ाइल या कॉन्फ़िगरेशन मान पहले से मौजूद है)।

<Warning>
`--overwrite` के साथ दोबारा केवल तभी चलाएँ, जब मौजूदा लक्ष्य को बदलना जानबूझकर किया जा रहा हो। प्रदाता ओवरराइट की गई फ़ाइलों के लिए माइग्रेशन रिपोर्ट निर्देशिका में आइटम-स्तरीय बैकअप फिर भी लिख सकते हैं।
</Warning>

नए इंस्टॉल में टकराव असामान्य हैं। वे आमतौर पर तब दिखाई देते हैं, जब आप ऐसे सेटअप पर आयात दोबारा चलाते हैं जिसमें पहले से उपयोगकर्ता के संपादन हैं।

यदि लागू करने के बीच में कोई टकराव सामने आता है (उदाहरण के लिए, किसी कॉन्फ़िगरेशन फ़ाइल पर अप्रत्याशित रेस), तो उस आइटम को टकराव के रूप में रिपोर्ट किया जाता है, जबकि स्वतंत्र फ़ाइलें, Skills, क्रेडेंशियल, संग्रह और कॉन्फ़िगरेशन प्रविष्टियाँ जारी रहती हैं। टकराव वाले आइटम को हल करें और आयात दोबारा चलाएँ; समान मेमोरी आयात आइडेम्पोटेंट होते हैं।

## गोपनीय मान

इंटरैक्टिव `openclaw migrate` पूछता है कि क्या पहचाने गए प्रमाणीकरण क्रेडेंशियल आयात करने हैं, जिसमें हाँ डिफ़ॉल्ट रूप से चुना होता है।

- स्वीकार करने पर वर्तमान Hermes OpenAI Codex OAuth प्रविष्टियाँ, OpenCode OpenAI OAuth और GitHub Copilot प्रविष्टियाँ, तथा [समर्थित `.env` कुंजियाँ](/hi/cli/migrate#supported-env-keys) आयात होती हैं।
- केवल गैर-गोपनीय स्थिति आयात करने के लिए `--no-auth-credentials` का उपयोग करें, या प्रॉम्प्ट पर नहीं उत्तर दें।
- स्वचालित `--yes` रन में क्रेडेंशियल आयात करने के लिए `--include-secrets` का उपयोग करें।
- विज़ार्ड से क्रेडेंशियल आयात करने के लिए ऑनबोर्डिंग विज़ार्ड के `--import-secrets` फ़्लैग का उपयोग करें।

## स्वचालन के लिए JSON आउटपुट

```bash
openclaw migrate hermes --dry-run --json
openclaw migrate apply hermes --json --yes
```

`--json` के साथ और `--yes` के बिना, लागू करने की प्रक्रिया योजना प्रिंट करती है और स्थिति में बदलाव नहीं करती—यह CI और साझा स्क्रिप्ट के लिए सबसे सुरक्षित मोड है।
