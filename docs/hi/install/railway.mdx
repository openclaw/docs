एक-क्लिक टेम्पलेट से Railway पर OpenClaw तैनात करें और वेब Control UI के माध्यम से इसे एक्सेस करें। यह सबसे आसान "सर्वर पर टर्मिनल की आवश्यकता नहीं" वाला तरीका है: Railway आपके लिए Gateway चलाता है।

## एक-क्लिक परिनियोजन

<a href="https://railway.com/deploy/clawdbot-railway-template" target="_blank" rel="noreferrer">
  Railway पर तैनात करें
</a>

<Steps>
  <Step title="टेम्पलेट तैनात करें">
    ऊपर **Deploy on Railway** पर क्लिक करें।
  </Step>

<Step title="वॉल्यूम जोड़ें">
  `/data` पर माउंट किया गया वॉल्यूम संलग्न करें (स्थायी स्थिति के लिए आवश्यक)।
</Step>

  <Step title="वेरिएबल सेट करें">
    सेवा पर आवश्यक **Variables** सेट करें:

    - `OPENCLAW_GATEWAY_PORT=8080` (आवश्यक -- Public Networking में पोर्ट से मेल खाना चाहिए)
    - `OPENCLAW_GATEWAY_TOKEN` (आवश्यक; इसे एडमिन सीक्रेट मानें)
    - `OPENCLAW_STATE_DIR=/data/.openclaw` (अनुशंसित)
    - `OPENCLAW_WORKSPACE_DIR=/data/workspace` (अनुशंसित)

  </Step>

<Step title="सार्वजनिक नेटवर्किंग सक्षम करें">
  **Public Networking** के अंतर्गत, पोर्ट `8080` पर सेवा के लिए **HTTP Proxy** सक्षम करें।
</Step>

  <Step title="कनेक्ट करें">
    अपना सार्वजनिक URL **Railway -> your service -> Settings -> Domains** में खोजें -- यह या तो जनरेट किया गया डोमेन (अक्सर `https://<something>.up.railway.app`) होगा या आपका संलग्न कस्टम डोमेन।

    `https://<your-railway-domain>/openclaw` खोलें और कॉन्फ़िगर किए गए साझा सीक्रेट का उपयोग करके कनेक्ट करें। टेम्पलेट डिफ़ॉल्ट रूप से `OPENCLAW_GATEWAY_TOKEN` का उपयोग करता है; यदि आप इसे पासवर्ड प्रमाणीकरण से बदलते हैं, तो इसके बजाय उस पासवर्ड का उपयोग करें।

  </Step>
</Steps>

## आपको क्या मिलता है

- होस्ट किया गया OpenClaw Gateway + Control UI
- Railway Volume (`/data`) के माध्यम से स्थायी स्टोरेज, ताकि `openclaw.json`, प्रति-एजेंट `auth-profiles.json`, चैनल/प्रदाता की स्थिति, सत्र और कार्यक्षेत्र पुनः परिनियोजन के बाद भी बने रहें

## चैनल कनेक्ट करें

चैनल सेटअप निर्देशों के लिए `/openclaw` पर Control UI का उपयोग करें या Railway के शेल के माध्यम से `openclaw onboard` चलाएँ:

- [Discord](/hi/channels/discord)
- [Telegram](/hi/channels/telegram) (सबसे तेज़ -- केवल एक बॉट टोकन)
- [सभी चैनल](/hi/channels)

## बैकअप और माइग्रेशन

अपनी स्थिति, कॉन्फ़िगरेशन, प्रमाणीकरण प्रोफ़ाइल और कार्यक्षेत्र निर्यात करें:

```bash
openclaw backup create
```

यह OpenClaw की स्थिति और किसी भी कॉन्फ़िगर किए गए कार्यक्षेत्र के साथ एक पोर्टेबल बैकअप आर्काइव बनाता है। विवरण के लिए [बैकअप](/hi/cli/backup) देखें।
