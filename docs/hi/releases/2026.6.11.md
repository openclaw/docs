### प्रदाता और मॉडल पुनर्प्राप्ति

[मॉडल चयन](/hi/providers/models) और सेटअप अब catalogs, credentials, streams, timeouts, Compaction, या fallbacks में गड़बड़ी होने पर अधिक स्पष्ट रूप से पुनर्प्राप्त होते हैं। प्रभावित [OpenAI](/hi/providers/openai), [OpenRouter](/hi/providers/openrouter), और [OpenCode Go](/hi/providers/opencode-go) कॉन्फ़िगरेशन अब उपयोगकर्ताओं को पुराने विकल्प या अटके हुए अनुरोध के साथ छोड़ने के बजाय विफलताओं की अधिक सुसंगत रूप से रिपोर्ट करते हैं या उनसे उबरते हैं।

[fast mode](/hi/tools/thinking#fast-mode-fast) के लिए follow-up fixes प्रभावित प्रदाता और fallback paths में इसके व्यवहार को सुधारते हैं; automatic fast mode स्वयं इस रिलीज़ में नया नहीं है।

### सत्र, मेमोरी, और भरोसे की निरंतरता

[सत्र](/hi/concepts/session), [compaction](/hi/concepts/compaction), [मेमोरी](/hi/concepts/memory), और [QMD-समर्थित मेमोरी](/hi/concepts/memory-qmd) लंबे समय तक चलने वाले काम, दोबारा कनेक्ट होने, अपग्रेड, और ट्रांसक्रिप्ट मरम्मत के दौरान अपेक्षित बातचीत और उपयोगी संदर्भ को अधिक निरंतरता से सुरक्षित रखते हैं। जब एजेंटों को सही संदर्भ या क्षमता वापस पाने की जरूरत होती है, तब [टूल खोज](/hi/tools/tool-search) भी अधिक भरोसेमंद तरीके से व्यवहार करती है।

एन्क्रिप्टेड [Matrix](/hi/channels/matrix) रिकवरी तब सुरक्षित रूप से रुक जाती है जब आवश्यक कुंजी स्थिति सत्यापित नहीं की जा सकती। [टूल नीतियां](/hi/gateway/sandbox-vs-tool-policy-vs-elevated), [अनुमोदन](/hi/cli/approvals), और [सीक्रेट हैंडलिंग](/hi/gateway/secrets) अपेक्षित रनटाइम स्थिति से जुड़ी रहती हैं, और अधिक जोखिम वाली कार्रवाइयां तब तक अक्षम रहती हैं जब तक उन्हें स्पष्ट रूप से सक्षम न किया जाए।

- Embedded agent sessions अब अत्यंत बड़े timeout के साथ भी किसी दूसरे session के file lock release करने की विश्वसनीय रूप से प्रतीक्षा करते हैं, timer overflow के जोखिम के बजाय। [4c736df](https://github.com/openclaw/openclaw/commit/4c736df975fed8e39f18db43bfaea9654b9ac0c7) धन्यवाद @vincentkoc.
- अत्यंत बड़े task timeout settings वाले queued commands अब विश्वसनीय रूप से time out होते हैं क्योंकि OpenClaw wait को runtime के safe maximum पर cap करता है। [1f6ae32](https://github.com/openclaw/openclaw/commit/1f6ae32cabb9d5ed308bb30715c287936ef483f1) धन्यवाद @vincentkoc.
- normalization-core द्वारा गलत string-coercion entry point expose करने और ACP sessions में outdated fast-mode value दिखाने को ठीक करता है, ताकि integrations को intended API मिले और users वास्तविक रूप से लागू mode देखें। [93ad397](https://github.com/openclaw/openclaw/commit/93ad39772590a34be3821d83709a5050c186fdf9) धन्यवाद @vincentkoc.
- Fast auto runs अब final replies को अधिक consistent रूप से deliver करते हैं, जिसमें progress-reset handling automatic mode तक सीमित है ताकि यह responses या forwarded callbacks में interfere न करे। [9e8ab08](https://github.com/openclaw/openclaw/commit/9e8ab083dd6b9df4dadf40ee523ff18ac1472bd3) धन्यवाद @vincentkoc.
- OpenAI Responses का उपयोग करने वाले Agent sessions अब replayed history में mismatched tool requests और results होने पर fail हुए या progress खोए बिना tool-based work resume करते हैं। [b4bc1f2](https://github.com/openclaw/openclaw/commit/b4bc1f20c9fca6d316561d42cbabdd793c67a6e7) धन्यवाद @vincentkoc.
- Completed plugin subagent और QA runs अब gateways द्वारा alternate completion envelope shapes लौटाने पर failures के रूप में misreport नहीं होते, जिससे plugin authors और operators के लिए successful handoffs अधिक reliable होते हैं। [d1b268f](https://github.com/openclaw/openclaw/commit/d1b268f7f7f7309dd5db99728019218b4d453e18) धन्यवाद @vincentkoc.
- Completed subagent tasks के कभी-कभी update के बिना ending होने को ठीक करता है, ताकि users result या parent agent का next step receive करें। [68a1e00](https://github.com/openclaw/openclaw/commit/68a1e00b73bd746f6fb577f4127fecb0ade9e228) धन्यवाद @vincentkoc.

## दस्तावेज़ और एडमिन टूल

### सेटअप और कमांड विश्वसनीयता

सामान्य [CLI commands](/hi/cli/index) अब configuration, paths, output, और failure cases को अधिक सुसंगत रूप से संभालते हैं। [Shell completion](/hi/cli/completion), [doctor](/hi/cli/doctor), [config commands](/hi/cli/config), और [gateway configuration](/hi/gateway/configuration) तब अधिक स्पष्ट मार्गदर्शन और सुरक्षित रिकवरी प्रदान करते हैं जब किसी installation या setting पर ध्यान देने की आवश्यकता होती है।

<Accordion title="स्रोत और योगदानकर्ता">

- OpenClaw का zsh टैब-completion मेनू अब `$` variables या backtick में लिपटे examples वाले option descriptions को shell input के रूप में evaluate करके मेनू को खराब करने के बजाय शाब्दिक रूप से दिखाता है। [#64490](https://github.com/openclaw/openclaw/pull/64490) धन्यवाद @edenkangdw.
- पुराने sandbox storage से upgrade करने के बाद, operators को अब बची हुई registry files के बारे में स्पष्ट `openclaw doctor` warning मिलती है और वे उन्हें migrate या clean up करने के लिए `openclaw doctor --fix` का उपयोग कर सकते हैं। [#84326](https://github.com/openclaw/openclaw/pull/84326) धन्यवाद @giodl73-repo.

- Cron इतिहास अब उन प्रविष्टियों को भरोसेमंद तरीके से ढूंढता है जिनकी जॉब ID में अतिरिक्त आसपास की स्पेस शामिल होती हैं, और नेस्टेड या खाली ID को इससे पहले अस्वीकार कर देता है कि वे ऐसे लॉग रिकॉर्ड बना सकें जिन्हें सुरक्षित रूप से वापस नहीं पढ़ा जा सकता। [#93567](https://github.com/openclaw/openclaw/pull/93567) धन्यवाद @alix-007, @vincentkoc।
- किसी cron जॉब को जोड़ने या हटाने से अब पहले से देय किसी अन्य आवर्ती जॉब की लंबित रन नहीं छूटती। [#94323](https://github.com/openclaw/openclaw/pull/94323) धन्यवाद @yetval।
- Word, PowerPoint, और Excel दस्तावेज़ पढ़ना और लिखना अब किसी बनावटी एक्सटेंशन पर विफल होने के बजाय अपेक्षित `.docx`, `.pptx`, या `.xlsx` पाथ का उपयोग करता है। [#95805](https://github.com/openclaw/openclaw/pull/95805) संबंधित [#93326](https://github.com/openclaw/openclaw/issues/93326)। धन्यवाद @bhnan, @lzyyzznl, @vincentkoc, @xzh-icenter।

- Windows उपयोगकर्ताओं को अब ripgrep जैसे ZIP डाउनलोड अनपैक करते समय एजेंट-प्रबंधित टूल इंस्टॉल विफल होने की संभावना कम है, क्योंकि OpenClaw अब PATH lookup पर निर्भर होने के बजाय अंतर्निहित Windows extraction programs का उपयोग करता है। [a5fde91](https://github.com/openclaw/openclaw/commit/a5fde9119c9c50685a392bff078a2f76a03d749d)
- Windows setup और runtime checks अब अनपेक्षित PATH entry पर निर्भर होने के बजाय trusted System32 resolver का उपयोग करके आवश्यक टूल्स को अधिक भरोसेमंद तरीके से ढूंढते हैं। [d3b4444](https://github.com/openclaw/openclaw/commit/d3b44442f6c8bedd765dc20a06316d3420b5d854) धन्यवाद @vincentkoc।
- `timeoutMs: 0` के साथ बनाई गई SDK runs अब अवांछित client-side watchdog के बिना अनुरोधित zero timeout बनाए रखती हैं। [2bdcc83](https://github.com/openclaw/openclaw/commit/2bdcc8314d3fce9ee2d0300759cd6a1d9bb45a7d) धन्यवाद @vincentkoc।
- रुके हुए OpenClaw commands अब तब भी भरोसेमंद तरीके से रुकते हैं जब callers अत्यधिक बड़े execution या idle-output timeouts देते हैं। [1425bb3](https://github.com/openclaw/openclaw/commit/1425bb3a03189813787194c23b8a38518166005a) धन्यवाद @vincentkoc।
- अत्यधिक बड़े timeout settings दिए गए commands अब इसलिए विफल होने के बजाय सुरक्षित अधिकतम का उपयोग करते हैं कि runtime अनुरोधित wait schedule नहीं कर सकता। [66b94ba](https://github.com/openclaw/openclaw/commit/66b94ba577b8836a4afa7cb59cd1a749bf2a8d68)
- Provider-specific tool allow/deny settings अब OpenClaw की doctor warnings के साथ अधिक सुसंगत रूप से मेल खाती हैं, जिसमें provider aliases, model-specific keys, OpenRouter-style model IDs, या malformed policy entries वाली configurations शामिल हैं। [8f2882f](https://github.com/openclaw/openclaw/commit/8f2882f94affbe5e89994ae175fdaf7304d7b392)
- SDK applications को अब terminal tools विफल होने या blocked होने पर भ्रमित करने वाले completion event के बजाय `tool.call.failed` मिलता है, ताकि मौजूदा failure handling सही ढंग से प्रतिक्रिया कर सके। [#95383](https://github.com/openclaw/openclaw/pull/95383) धन्यवाद @ly-wang19।
- पहचाने गए job fields के harmless trailing spaces के साथ आने पर cron add और update requests अस्वीकार हो रही थीं; यह ठीक किया गया है, ताकि ambiguous या unsafe input के checks को ढीला किए बिना schedules save किए जा सकें। [#95674](https://github.com/openclaw/openclaw/pull/95674) संबंधित [#95407](https://github.com/openclaw/openclaw/issues/95407)। धन्यवाद @nassiel, @zw-xysk।
- Codex subagent monitoring बड़े child agents और transcript files के sets को कम अनावश्यक scanning के साथ संभालता है, जबकि पुराने transcript filename formats पहले की तरह resolve होते रहते हैं। [#96085](https://github.com/openclaw/openclaw/pull/96085) धन्यवाद @vincentkoc।
- native Windows crabbox hydration runner और daemon के अलग-अलग home directories उपयोग करने पर अटक जाती थी या handoffs छूट जाते थे; यह ठीक किया गया है, ताकि दोनों समान job state और stop files ढूंढ सकें। [f354889](https://github.com/openclaw/openclaw/commit/f354889efa1c8bafca9304767afba2c270add549) धन्यवाद @vincentkoc।
