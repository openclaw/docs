OpenClaw **ट्रांसपोर्ट टाइमस्टैम्प के लिए होस्ट-स्थानीय समय** का उपयोग करता है और सिस्टम प्रॉम्प्ट में **केवल समय क्षेत्र** रखता है।
प्रदाता टाइमस्टैम्प सुरक्षित रखे जाते हैं, ताकि टूल अपने मूल अर्थ-विज्ञान बनाए रखें। जब एजेंट को वर्तमान
समय की आवश्यकता होती है, तो वह `session_status` टूल चलाता है।

## संदेश एनवेलप (डिफ़ॉल्ट रूप से स्थानीय)

इनबाउंड संदेशों को सप्ताह के दिन और सेकंड-सटीक टाइमस्टैम्प के साथ रैप किया जाता है:

```
[WhatsApp +1555 Mon 2026-01-05 16:26:34 PST] संदेश का टेक्स्ट
```

प्रदाता के समय क्षेत्र की परवाह किए बिना, एनवेलप टाइमस्टैम्प **डिफ़ॉल्ट रूप से होस्ट-स्थानीय** होता है।
`agents.defaults` के अंतर्गत इसे ओवरराइड करें:

```json5
{
  agents: {
    defaults: {
      envelopeTimezone: "local", // "utc" | "local" | "user" | IANA timezone
      envelopeTimestamp: "on", // "on" | "off"
      envelopeElapsed: "on", // "on" | "off"
    },
  },
}
```

| कुंजी                 | मान                                               | व्यवहार                                                                                                                                                                        |
| ------------------- | ---------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `envelopeTimezone`  | `local` (डिफ़ॉल्ट), `utc`, `user`, स्पष्ट IANA नाम | `user`, `agents.defaults.userTimezone` का उपयोग करता है (सेट न होने पर होस्ट का समय क्षेत्र)। स्पष्ट IANA नाम (उदा. `"America/Chicago"`) एक निश्चित क्षेत्र तय करता है; अपरिचित नाम UTC पर फ़ॉलबैक होते हैं। |
| `envelopeTimestamp` | `on` (डिफ़ॉल्ट), `off`                                | `off` एनवेलप हेडर, प्रत्यक्ष एजेंट प्रॉम्प्ट प्रीफ़िक्स और एम्बेड किए गए मॉडल-इनपुट प्रीफ़िक्स से निरपेक्ष टाइमस्टैम्प हटा देता है।                                                       |
| `envelopeElapsed`   | `on` (डिफ़ॉल्ट), `off`                                | `off` सत्र में पिछले संदेश के बाद से दिखाया गया बीता-समय प्रत्यय (`+30s` / `+2m` शैली) हटा देता है।                                                               |

### उदाहरण

**स्थानीय (डिफ़ॉल्ट):**

```
[WhatsApp +1555 Sun 2026-01-18 00:19:42 PST] नमस्ते
```

**उपयोगकर्ता का समय क्षेत्र:**

```
[WhatsApp +1555 Sun 2026-01-18 00:19:42 CST] नमस्ते
```

**`envelopeTimezone: "utc"` के साथ बीता समय:**

```
[WhatsApp +1555 +30s Sun 2026-01-18T05:19:00Z] अनुवर्ती संदेश
```

## सिस्टम प्रॉम्प्ट: वर्तमान दिनांक और समय

सिस्टम प्रॉम्प्ट में **वर्तमान दिनांक और समय** अनुभाग शामिल होता है, जिसमें **केवल समय क्षेत्र**
(घड़ी या समय प्रारूप नहीं) होता है, ताकि प्रॉम्प्ट कैशिंग स्थिर रहे:

```
समय क्षेत्र: America/Chicago
```

कॉन्फ़िगर होने पर क्षेत्र `agents.defaults.userTimezone` होता है, अन्यथा होस्ट का समय क्षेत्र।
प्रॉम्प्ट एजेंट को यह भी निर्देश देता है कि जब भी उसे वर्तमान दिनांक, समय या सप्ताह के दिन की
आवश्यकता हो, वह `session_status` टूल चलाए।

## सिस्टम इवेंट पंक्तियाँ (डिफ़ॉल्ट रूप से स्थानीय)

एजेंट संदर्भ में जोड़े गए कतारबद्ध सिस्टम इवेंट के आगे संदेश एनवेलप के समान
`envelopeTimezone` चयन का उपयोग करके टाइमस्टैम्प लगाया जाता है (डिफ़ॉल्ट: होस्ट-स्थानीय)।

```
सिस्टम: [2026-01-12 12:19:17 PST] मॉडल बदला गया।
```

### उपयोगकर्ता का समय क्षेत्र + प्रारूप कॉन्फ़िगर करें

```json5
{
  agents: {
    defaults: {
      userTimezone: "America/Chicago",
      timeFormat: "auto", // auto | 12 | 24
    },
  },
}
```

- `userTimezone` प्रॉम्प्ट संदर्भ (और `envelopeTimezone: "user"` के लिए) **उपयोगकर्ता-स्थानीय समय क्षेत्र** सेट करता है।
- `timeFormat` प्रॉम्प्ट में दिखाई देने वाले समय का **12h/24h प्रदर्शन** नियंत्रित करता है। `auto` OS की प्राथमिकताओं का पालन करता है।

## समय प्रारूप का पता लगाना (स्वचालित)

जब `timeFormat: "auto"` हो, तो OpenClaw OS की प्राथमिकता (macOS और Windows) की जाँच करता है
और अनुपलब्ध होने पर लोकेल फ़ॉर्मेटिंग का उपयोग करता है। बार-बार सिस्टम कॉल से बचने के लिए पता लगाया गया मान
**प्रति प्रोसेस कैश** किया जाता है।

## टूल पेलोड + कनेक्टर (अपरिष्कृत प्रदाता समय + सामान्यीकृत फ़ील्ड)

चैनल टूल **प्रदाता-मूल टाइमस्टैम्प** लौटाते हैं और एकरूपता के लिए सामान्यीकृत फ़ील्ड जोड़ते हैं:

- `timestampMs`: युग से मिलीसेकंड (UTC)
- `timestampUtc`: ISO 8601 UTC स्ट्रिंग

अपरिष्कृत प्रदाता फ़ील्ड सुरक्षित रखे जाते हैं, ताकि कुछ भी नष्ट न हो।

- Discord: UTC ISO टाइमस्टैम्प
- Slack: API से युग-जैसी स्ट्रिंग
- Telegram/WhatsApp: प्रदाता-विशिष्ट संख्यात्मक/ISO टाइमस्टैम्प

यदि आपको स्थानीय समय चाहिए, तो ज्ञात समय क्षेत्र का उपयोग करके इसे डाउनस्ट्रीम में रूपांतरित करें।
