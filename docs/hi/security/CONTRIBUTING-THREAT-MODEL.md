[खतरा मॉडल](/hi/security/THREAT-MODEL-ATLAS) एक सतत विकसित होने वाला दस्तावेज़ है। कोई भी योगदान कर सकता है; इसके लिए सुरक्षा या MITRE ATLAS की पृष्ठभूमि आवश्यक नहीं है।

<Note>
यह खतरा मॉडल में सामग्री जोड़ने के लिए है, सक्रिय कमजोरियों की रिपोर्ट करने के लिए नहीं। यदि आपको ऐसी कमजोरी मिली है जिसका दुरुपयोग किया जा सकता है, तो इसके बजाय [Trust पृष्ठ](https://trust.openclaw.ai) पर दिए गए उत्तरदायी प्रकटीकरण निर्देशों का पालन करें।
</Note>

## योगदान करने के तरीके

**कोई खतरा जोड़ें।** हमले के परिदृश्य का अपने शब्दों में वर्णन करते हुए [openclaw/trust](https://github.com/openclaw/trust/issues) पर एक इश्यू खोलें। निम्नलिखित जानकारी सहायक है, लेकिन आवश्यक नहीं:

- हमले का परिदृश्य और उसका दुरुपयोग कैसे किया जा सकता है।
- कौन-से घटक प्रभावित हैं (CLI, Gateway, चैनल, ClawHub, MCP सर्वर आदि)।
- गंभीरता का आपका अनुमान (निम्न / मध्यम / उच्च / अत्यंत गंभीर)।
- संबंधित शोध, CVE या वास्तविक उदाहरणों के लिंक।

समीक्षा के दौरान अनुरक्षक ATLAS मैपिंग, खतरा ID और जोखिम स्तर निर्धारित करते हैं।

**शमन उपाय सुझाएँ।** खतरे का संदर्भ देते हुए कोई इश्यू या PR खोलें। सुझाव विशिष्ट और कार्रवाई योग्य होना चाहिए: "Gateway पर प्रत्येक प्रेषक के लिए 10 संदेश/मिनट की दर सीमा" का सुझाव "दर सीमा लागू करें" से अधिक उपयोगी है।

**हमला शृंखला प्रस्तावित करें।** हमला शृंखलाएँ दिखाती हैं कि कई खतरे मिलकर किस प्रकार एक वास्तविक परिदृश्य बनाते हैं। चरणों और हमलावर द्वारा उन्हें शृंखलाबद्ध करने के तरीके का वर्णन करें; औपचारिक टेम्पलेट की तुलना में संक्षिप्त विवरण बेहतर है।

**मौजूदा सामग्री ठीक करें या सुधारें।** टंकण त्रुटियाँ, स्पष्टीकरण, पुरानी जानकारी और बेहतर उदाहरण: PR का स्वागत है, इश्यू आवश्यक नहीं है।

## फ़्रेमवर्क संदर्भ

खतरों को [MITRE ATLAS](https://atlas.mitre.org/) (AI प्रणालियों के लिए प्रतिकूल खतरा परिदृश्य) से मैप किया जाता है। यह प्रॉम्प्ट इंजेक्शन, टूल के दुरुपयोग और एजेंट के शोषण जैसे AI/ML-विशिष्ट खतरों के लिए एक फ़्रेमवर्क है। योगदान करने के लिए ATLAS का ज्ञान आवश्यक नहीं है; अनुरक्षक समीक्षा के दौरान प्रस्तुतियों को मैप करते हैं।

**खतरा ID।** प्रत्येक खतरे को `T-EXEC-003` जैसा एक ID मिलता है, जिसे समीक्षा के दौरान अनुरक्षक निर्धारित करते हैं।

| कोड    | श्रेणी                                   |
| ------- | ------------------------------------------ |
| RECON   | टोह लेना - जानकारी एकत्र करना     |
| ACCESS  | प्रारंभिक पहुँच - प्रवेश प्राप्त करना             |
| EXEC    | निष्पादन - दुर्भावनापूर्ण कार्रवाइयाँ चलाना      |
| PERSIST | स्थायित्व - पहुँच बनाए रखना           |
| EVADE   | सुरक्षा से बचाव - पहचान से बचना       |
| DISC    | खोज - परिवेश के बारे में जानना |
| EXFIL   | डेटा बाहर निकालना - डेटा चुराना               |
| IMPACT  | प्रभाव - क्षति या व्यवधान              |

**जोखिम स्तर।** यदि आप स्तर के बारे में अनिश्चित हैं, तो केवल प्रभाव का वर्णन करें; अनुरक्षक इसका आकलन करेंगे।

| स्तर        | अर्थ                                                           |
| ------------ | ----------------------------------------------------------------- |
| **अत्यंत गंभीर** | पूरी प्रणाली से समझौता, या उच्च संभावना + अत्यंत गंभीर प्रभाव      |
| **उच्च**     | महत्वपूर्ण क्षति की संभावना, या मध्यम संभावना + अत्यंत गंभीर प्रभाव |
| **मध्यम**   | मध्यम जोखिम, या कम संभावना + उच्च प्रभाव                    |
| **निम्न**      | कम संभावना और सीमित प्रभाव                                       |

## समीक्षा प्रक्रिया

1. **प्रारंभिक समीक्षा** - नई प्रस्तुतियों की 48 घंटों के भीतर समीक्षा की जाती है।
2. **आकलन** - अनुरक्षक व्यवहार्यता सत्यापित करते हैं, ATLAS मैपिंग और खतरा ID निर्धारित करते हैं तथा जोखिम स्तर को मान्य करते हैं।
3. **दस्तावेज़ीकरण** - स्वरूपण और पूर्णता की जाँच।
4. **मर्ज** - खतरा मॉडल और विज़ुअलाइज़ेशन में जोड़ा जाता है।
