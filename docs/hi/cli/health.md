# `openclaw health`

चल रहे Gateway से WebSocket RPC के माध्यम से स्वास्थ्य स्नैपशॉट प्राप्त करें (CLI से कोई प्रत्यक्ष चैनल सॉकेट नहीं)।

## विकल्प

| फ़्लैग             | डिफ़ॉल्ट | विवरण                                                                       |
| ---------------- | ------- | --------------------------------------------------------------------------------- |
| `--json`         | `false` | टेक्स्ट के बजाय मशीन-पठनीय JSON प्रिंट करें।                                      |
| `--timeout <ms>` | `10000` | कनेक्शन टाइमआउट मिलीसेकंड में।                                               |
| `--verbose`      | `false` | लाइव जाँच को बाध्य करता है और आउटपुट को सभी कॉन्फ़िगर किए गए खातों और एजेंटों के लिए विस्तृत करता है। |
| `--debug`        | `false` | `--verbose` का उपनाम।                                                            |

उदाहरण:

```bash
openclaw health
openclaw health --json
openclaw health --timeout 2500
openclaw health --verbose
openclaw health --debug
```

## व्यवहार

- `--verbose` के बिना, Gateway कैश किया हुआ स्नैपशॉट लौटा सकता है (जो अधिकतम 60 सेकंड तक नवीन रहता है और लाइव चैनल रनटाइम स्थिति से अपरिवर्तित होता है) और अगले कॉलर के लिए उसे पृष्ठभूमि में रीफ़्रेश कर सकता है।
- `--verbose` लाइव जाँच (प्रति-चैनल खाता जाँच) को बाध्य करता है, Gateway कनेक्शन विवरण प्रिंट करता है, और मानव-पठनीय आउटपुट को केवल डिफ़ॉल्ट एजेंट के बजाय सभी कॉन्फ़िगर किए गए खातों और एजेंटों के लिए विस्तृत करता है।
- `--json` हमेशा पूरा स्नैपशॉट लौटाता है: चैनल, प्रति-खाता जाँच, Plugin लोड स्थिति, कॉन्टेक्स्ट-इंजन क्वारंटीन स्थिति, मॉडल-मूल्य निर्धारण कैश स्थिति, इवेंट-लूप स्वास्थ्य और प्रति-एजेंट सत्र स्टोर।
