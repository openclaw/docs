# `openclaw health`

चल रहे Gateway से WebSocket RPC के माध्यम से स्वास्थ्य स्नैपशॉट प्राप्त करें (CLI से कोई प्रत्यक्ष चैनल सॉकेट नहीं)।

## विकल्प

| फ़्लैग             | डिफ़ॉल्ट | विवरण                                                                       |
| ---------------- | ------- | --------------------------------------------------------------------------------- |
| `--json`         | `false` | टेक्स्ट के बजाय मशीन-पठनीय JSON प्रिंट करें।                                      |
| `--timeout <ms>` | `10000` | कनेक्शन टाइमआउट, मिलीसेकंड में।                                               |
| `--verbose`      | `false` | लाइव जाँच को बाध्य करता है और सभी कॉन्फ़िगर किए गए खातों तथा एजेंटों के लिए आउटपुट विस्तृत करता है। |
| `--debug`        | `false` | `--verbose` का उपनाम।                                                            |

उदाहरण:

```bash
openclaw health
openclaw health --json
openclaw health --timeout 2500
openclaw health --verbose
openclaw health --debug
```

## व्यवहार

- `--verbose` के बिना, Gateway कैश किया हुआ स्नैपशॉट लौटा सकता है (अधिकतम 60 सेकंड तक ताज़ा और लाइव चैनल रनटाइम स्थिति से अपरिवर्तित) और अगले कॉलर के लिए इसे पृष्ठभूमि में रीफ़्रेश कर सकता है।
- `--verbose` लाइव जाँच (प्रति-चैनल खाता जाँच) को बाध्य करता है, Gateway कनेक्शन विवरण प्रिंट करता है और केवल डिफ़ॉल्ट एजेंट के बजाय सभी कॉन्फ़िगर किए गए खातों तथा एजेंटों के लिए मानव-पठनीय आउटपुट विस्तृत करता है।
- `--json` हमेशा पूरा स्नैपशॉट लौटाता है: चैनल, प्रति-खाता जाँच, Plugin लोड स्थिति, कॉन्टेक्स्ट-इंजन क्वारंटीन स्थिति, मॉडल-मूल्य निर्धारण कैश स्थिति, इवेंट-लूप स्वास्थ्य, डिलीवरी-कतार डेड लेटर और प्रति-एजेंट सत्र स्टोर।
- जब आउटबाउंड डिलीवरी या इनबाउंड चैनल इवेंट डेड-लेटर किए जाते हैं, तो टेक्स्ट आउटपुट उनकी संख्या और सबसे पुरानी विफलता की अवधि की रिपोर्ट करता है। इनबाउंड संख्या चैनल खाते के अनुसार समूहीकृत होती है; अलग-अलग इवेंट का निरीक्षण या पुनर्प्राप्ति [`openclaw channels dead-letters`](/hi/cli/channels#inbound-dead-letters) से करें।
