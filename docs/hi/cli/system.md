# `openclaw system`

Gateway के लिए सिस्टम-स्तरीय सहायक: सिस्टम इवेंट को कतार में लगाना, Heartbeat नियंत्रित करना और उपस्थिति देखना।

सभी `system` उपकमांड Gateway RPC का उपयोग करते हैं और साझा क्लाइंट फ़्लैग स्वीकार करते हैं:

| फ़्लैग              | डिफ़ॉल्ट                              | विवरण                                                                                                                                                                                            |
| ----------------- | ------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `--url <url>`     | कॉन्फ़िगर होने पर `gateway.remote.url` | Gateway WebSocket URL।                                                                                                                                                                                 |
| `--token <token>` | कोई नहीं                                 | Gateway टोकन (यदि आवश्यक हो)।                                                                                                                                                                           |
| `--timeout <ms>`  | `30000`                              | RPC टाइमआउट, मिलीसेकंड में।                                                                                                                                                                           |
| `--expect-final`  | बंद                                  | अंतिम प्रतिक्रिया (एजेंट) की प्रतीक्षा करें।                                                                                                                                                                       |
| `--json`          | बंद                                  | JSON आउटपुट करें। इस फ़्लैग से स्वतंत्र रूप से, `heartbeat last/enable/disable` और `system presence` हमेशा अपरिष्कृत RPC JSON पेलोड प्रिंट करते हैं; `system event` इसका उपयोग JSON और एक साधारण `ok` पंक्ति के बीच स्विच करने के लिए करता है। |

## सामान्य कमांड

```bash
openclaw system event --text "ज़रूरी फ़ॉलो-अप की जाँच करें" --mode now
openclaw system event --text "ज़रूरी फ़ॉलो-अप की जाँच करें" --url ws://127.0.0.1:18789 --token "$OPENCLAW_GATEWAY_TOKEN"
openclaw system heartbeat enable
openclaw system heartbeat last
openclaw system presence
```

## `system event`

डिफ़ॉल्ट रूप से **मुख्य** सत्र पर एक सिस्टम इवेंट कतार में लगाएँ। अगला Heartbeat इसे प्रॉम्प्ट में `System:` पंक्ति के रूप में प्रविष्ट करता है। Heartbeat को तुरंत ट्रिगर करने के लिए `--mode now` का उपयोग करें; `next-heartbeat` (डिफ़ॉल्ट) अगले निर्धारित टिक की प्रतीक्षा करता है।

किसी विशिष्ट सत्र को लक्षित करने के लिए `--session-key` दें, उदाहरण के लिए किसी एसिंक-कार्य की पूर्णता को उसे शुरू करने वाले चैनल पर वापस भेजने के लिए।

<Note>
**`--session-key` के साथ समय-निर्धारण अपवाद:** जब `--session-key` दिया जाता है, तो `--mode next-heartbeat` अगले निर्धारित टिक की प्रतीक्षा करने के बजाय तत्काल लक्षित वेक में बदल जाता है। लक्षित वेक Heartbeat इंटेंट `immediate` का उपयोग करते हैं, इसलिए वे रनर के नॉट-ड्यू गेट को बायपास करते हैं, जो अन्यथा `event`-इंटेंट वेक को स्थगित (और प्रभावी रूप से ड्रॉप) कर देता। यदि आप विलंबित डिलीवरी चाहते हैं, तो `--session-key` को छोड़ दें, ताकि इवेंट मुख्य सत्र पर पहुँचे और अगले नियमित Heartbeat के साथ वितरित हो।
</Note>

फ़्लैग:

- `--text <text>`: आवश्यक सिस्टम इवेंट टेक्स्ट।
- `--mode <mode>`: `now` या `next-heartbeat` (डिफ़ॉल्ट)।
- `--session-key <sessionKey>`: वैकल्पिक; एजेंट के मुख्य सत्र के बजाय किसी विशिष्ट एजेंट सत्र को लक्षित करें। जो कुंजियाँ निर्धारित एजेंट से संबंधित नहीं हैं, उनके लिए एजेंट का मुख्य सत्र उपयोग किया जाता है।

## `system heartbeat last|enable|disable`

- `last`: अंतिम Heartbeat इवेंट दिखाएँ।
- `enable`: Heartbeat फिर से चालू करें (यदि वे अक्षम कर दिए गए थे, तो इसका उपयोग करें)।
- `disable`: Heartbeat रोकें।

## `system presence`

Gateway को ज्ञात वर्तमान सिस्टम उपस्थिति प्रविष्टियों (Node, इंस्टेंस और समान स्थिति पंक्तियाँ) की सूची दिखाएँ।

## टिप्पणियाँ

- आपके वर्तमान कॉन्फ़िगरेशन से पहुँच योग्य, चालू Gateway (स्थानीय या रिमोट) आवश्यक है।
- सिस्टम इवेंट अस्थायी होते हैं और रीस्टार्ट के बाद बने नहीं रहते।
