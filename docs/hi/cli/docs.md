# `openclaw docs`

टर्मिनल से लाइव OpenClaw दस्तावेज़ अनुक्रमणिका खोजें।

## उपयोग

```bash
openclaw docs                       # दस्तावेज़ प्रवेश-बिंदु और खोज का उदाहरण प्रिंट करें
openclaw docs <query...>            # लाइव दस्तावेज़ अनुक्रमणिका खोजें
```

| आर्ग्युमेंट     | विवरण                                                                        |
| ------------ | ---------------------------------------------------------------------------------- |
| `[query...]` | मुक्त-रूप खोज क्वेरी। कई शब्दों वाली क्वेरी को रिक्त स्थानों से जोड़कर एक क्वेरी के रूप में भेजा जाता है। |

कोई क्वेरी न देने पर, `openclaw docs` खोज चलाने के बजाय दस्तावेज़ प्रवेश-बिंदु URL और खोज कमांड का एक नमूना प्रिंट करता है।

## उदाहरण

```bash
openclaw docs browser existing-session
openclaw docs sandbox allowHostControl
openclaw docs gateway token secretref
```

## यह कैसे काम करता है

`openclaw docs`, `https://docs.openclaw.ai/api/search` को कॉल करता है और JSON परिणामों को रेंडर करता है। खोज अनुरोध 30 सेकंड की निश्चित समय-सीमा का उपयोग करता है।

## आउटपुट

रिच (TTY) टर्मिनल में, परिणाम एक शीर्षक और उसके बाद बुलेट सूची के रूप में रेंडर होते हैं: पृष्ठ का शीर्षक, लिंक किया हुआ दस्तावेज़ URL, और अगली पंक्ति में एक छोटा अंश। खाली परिणामों के लिए "कोई परिणाम नहीं।" प्रिंट होता है।

नॉन-रिच आउटपुट (पाइप किया हुआ, `--no-color`, स्क्रिप्ट) में, यही डेटा Markdown के रूप में रेंडर होता है:

```markdown
# दस्तावेज़ खोज: <query>

- [शीर्षक](https://docs.openclaw.ai/...) - अंश
- [शीर्षक](https://docs.openclaw.ai/...) - अंश
```

## निकास कोड

| कोड | अर्थ                                                                  |
| ---- | ------------------------------------------------------------------------ |
| `0`  | खोज सफल हुई, जिसमें शून्य-परिणाम वाली प्रतिक्रियाएँ भी शामिल हैं।                       |
| `1`  | होस्ट किए गए दस्तावेज़ खोज API का कॉल विफल हुआ; stderr त्रुटि संदेश प्रिंट करता है। |
