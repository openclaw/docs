# `openclaw setup`

`openclaw setup` सिस्टम एजेंट का प्रवेश बिंदु है। कॉन्फ़िगर किए गए सिस्टम पर, केवल
`openclaw setup` एक इंटरैक्टिव OpenClaw चैट खोलता है। नए सिस्टम पर, यह
निर्देशित ऑनबोर्डिंग शुरू करता है। एक अनुरोध के लिए `-m`/`--message` या
विज़ार्ड के बिना कॉन्फ़िगरेशन/वर्कस्पेस फ़ोल्डर आरंभ करने के लिए `--baseline` का उपयोग करें।

रूटिंग क्रम:

1. कोई भी ऑनबोर्डिंग विकल्प (`--wizard`, `--baseline`, वर्कस्पेस, रीसेट,
   गैर-इंटरैक्टिव, प्रवाह, मोड, Gateway, डेमन, स्किप, आयात, रिमोट या प्रमाणीकरण
   विकल्प) ऑनबोर्डिंग को ठीक उसी तरह चलाता है जैसे `openclaw onboard` चलाता है।
2. `-m`/`--message` या `--yes` सिस्टम एजेंट चलाता है।
3. कोई रूटिंग विकल्प न होने पर, कॉन्फ़िगर किया गया इंटरैक्टिव सिस्टम OpenClaw खोलता है। नया
   सिस्टम ऑनबोर्डिंग चलाता है। कॉन्फ़िगर किए गए सिस्टम पर, `--json` TTY के बिना भी
   सिस्टम का अवलोकन प्रिंट करता है; ऑनबोर्डिंग विकल्प ऑनबोर्डिंग का
   JSON सारांश बनाए रखता है।

निर्देशित मोड में, `--workspace <dir>` वह वर्कस्पेस है जिसे OpenClaw के लिए प्रस्तावित किया जाता है;
इसे केवल आपके उस प्रस्ताव को स्वीकृत करने के बाद स्थायी किया जाता है। बेसलाइन, क्लासिक और
गैर-इंटरैक्टिव सेटअप नई स्थापना पर अपनी सामान्य प्रक्रिया के माध्यम से दिए गए वर्कस्पेस को स्थायी
करते हैं। जब मौजूदा एजेंट सूची को फिर से मैप किया जाएगा, तो
क्लासिक विज़ार्ड स्पष्ट पुष्टि मांगता है; गैर-इंटरैक्टिव सेटअप मौजूदा
फ़्लीट वर्कस्पेस बनाए रखता है और चेतावनी प्रिंट करता है।

निर्देशित इनफ़रेंस पहचान macOS या Linux पर Gateway होस्ट में चलती है। CLI
और macOS ऐप उसी Gateway-स्वामित्व वाले डिटेक्टर को कॉल करते हैं, जो कॉन्फ़िगर किए गए
मॉडल, समर्थित CLI लॉगिन, API-कुंजी पर्यावरण चर और पहले से
स्थापित Ollama या LM Studio मॉडल की जाँच करता है। इस स्वचालित
प्रक्रिया में स्थानीय मॉडल कभी डाउनलोड नहीं किए जाते। CLI और API-कुंजी
उम्मीदवारों के बाद पहचाने गए स्थानीय रनटाइम का स्वचालित परीक्षण किया जाता है; जब कई स्थानीय मॉडल
उपलब्ध हों, तो OpenClaw सबसे सक्षम टूल-कॉलिंग इंस्ट्रक्ट फ़ैमिली को प्राथमिकता देता है।
चयनित उम्मीदवार को उसके प्रोवाइडर और मॉडल कॉन्फ़िगरेशन को सहेजने से पहले
वास्तविक कम्प्लीशन का उत्तर देना आवश्यक है।
स्थापित Gemini, Antigravity, Pi और OpenCode CLI की भी सूचना दी जाती है, जब
वे निर्देशित सेटअप के लिए पुनः उपयोग योग्य इनफ़रेंस रूट के रूप में काम नहीं कर सकते।

`setup` वही ऑनबोर्डिंग फ़्लैग स्वीकार करता है जो `openclaw onboard`, जिसमें
प्रमाणीकरण (`--auth-choice`, `--token`, प्रोवाइडर कुंजी फ़्लैग), Gateway
(`--gateway-port`, `--gateway-bind`, `--gateway-auth`, `--install-daemon`),
Tailscale (`--tailscale`), रीसेट (`--reset`, `--reset-scope`), प्रवाह
(`--flow quickstart|advanced|manual|import`) और स्किप फ़्लैग
(`--skip-channels`, `--skip-skills`, `--skip-bootstrap`, `--skip-search`,
`--skip-health`, `--skip-ui`, `--skip-hooks`) शामिल हैं। `openclaw onboard --tui` जैसा
टर्मिनल निकास-पथ उपयोग करने के लिए `--tui` पास करें। फ़्लैग के पूर्ण संदर्भ और
गैर-इंटरैक्टिव उदाहरणों के लिए [ऑनबोर्ड](/hi/cli/onboard) और
[CLI स्वचालन](/hi/start/wizard-cli-automation) देखें। `openclaw onboard --modern` उसी
इनफ़रेंस-नियंत्रित OpenClaw सहायक के लिए संगतता
प्रवेश बना हुआ है।

<Note>
`openclaw setup` परिवर्तनशील कॉन्फ़िगरेशन वाली स्थापनाओं के लिए है। Nix मोड (`OPENCLAW_NIX_MODE=1`) में OpenClaw सेटअप द्वारा लिखने से इनकार करता है, क्योंकि कॉन्फ़िगरेशन फ़ाइल Nix द्वारा प्रबंधित होती है। प्रथम-पक्ष [nix-openclaw त्वरित आरंभ](https://github.com/openclaw/nix-openclaw#quick-start) या किसी अन्य Nix पैकेज के लिए समकक्ष स्रोत कॉन्फ़िगरेशन का उपयोग करें।
</Note>

## विकल्प

| फ़्लैग                       | विवरण                                                                                          |
| -------------------------- | ---------------------------------------------------------------------------------------------------- |
| `-m, --message <text>`     | एक OpenClaw अनुरोध चलाएँ।                                                                            |
| `--yes`                    | एक `--message` अनुरोध के लिए स्थायी कॉन्फ़िगरेशन लेखन स्वीकृत करें।                                        |
| `--workspace <dir>`        | वर्कस्पेस प्रस्ताव; मौजूदा फ़्लीट को क्लासिक पुष्टि चाहिए और गैर-इंटरैक्टिव रूप से संरक्षित रखा जाता है। |
| `--baseline`               | ऑनबोर्डिंग के बिना बेसलाइन कॉन्फ़िगरेशन/वर्कस्पेस/सत्र फ़ोल्डर बनाएँ।                                 |
| `--wizard`                 | इंटरैक्टिव ऑनबोर्डिंग अनिवार्य करें।                                                                        |
| `--tui`                    | ब्राउज़र हैंडऑफ़ के बजाय टर्मिनल निकास-पथ का उपयोग करें।                                               |
| `--non-interactive`        | संकेतों के बिना ऑनबोर्डिंग चलाएँ।                                                                      |
| `--accept-risk`            | पूरे सिस्टम तक एजेंट की पहुँच का जोखिम स्वीकार करें; `--non-interactive` के साथ आवश्यक।                        |
| `--mode <mode>`            | ऑनबोर्डिंग मोड: `local` या `remote`।                                                                |
| `--flow <flow>`            | ऑनबोर्ड प्रवाह: `quickstart`, `advanced`, `manual` या `import`।                                       |
| `--reset`                  | ऑनबोर्डिंग से पहले कॉन्फ़िगरेशन + क्रेडेंशियल + सत्र रीसेट करें (वर्कस्पेस केवल `--reset-scope full` के साथ)।  |
| `--reset-scope <scope>`    | रीसेट का दायरा: `config`, `config+creds+sessions` या `full`।                                           |
| `--import-from <provider>` | ऑनबोर्डिंग के दौरान चलाया जाने वाला माइग्रेशन प्रोवाइडर।                                                         |
| `--import-source <path>`   | `--import-from` के लिए स्रोत एजेंट होम।                                                               |
| `--import-secrets`         | ऑनबोर्डिंग माइग्रेशन के दौरान समर्थित सीक्रेट आयात करें।                                                |
| `--remote-url <url>`       | रिमोट Gateway WebSocket URL।                                                                        |
| `--remote-token <token>`   | रिमोट Gateway टोकन (वैकल्पिक)।                                                                     |
| `--json`                   | कॉन्फ़िगर किया गया सिस्टम: OpenClaw अवलोकन। ऑनबोर्डिंग रूट: ऑनबोर्डिंग सारांश।                          |

`--classic` और `--non-interactive` परस्पर अनन्य हैं: क्लासिक
संकेत देने वाला विज़ार्ड खोलता है, जबकि गैर-इंटरैक्टिव सेटअप स्वचालन पथ का उपयोग करता है।
इंटरैक्टिव ऑनबोर्डिंग में, `--remote-url` और `--remote-token`
रिमोट Gateway चरण को पहले से भरते हैं और उस रन के लिए संग्रहीत रिमोट मानों पर प्राथमिकता रखते हैं।
URL बदलने पर संग्रहीत क्रेडेंशियल का पुनः उपयोग नहीं होता, जब तक कि आप टोकन भी पास न करें।
टोकन छिपा रहता है और विज़ार्ड द्वारा चयनित सादे टेक्स्ट या SecretRef
संग्रहण मोड का उपयोग करता है।

### बेसलाइन मोड

`openclaw setup --baseline` पुराने केवल-बेसलाइन व्यवहार को बनाए रखता है: यह
कॉन्फ़िगरेशन, वर्कस्पेस और सत्र डायरेक्टरी बनाता है, फिर
ऑनबोर्डिंग चलाए बिना बाहर निकल जाता है। यह `--workspace` और अहानिकर आउटपुट नियंत्रण स्वीकार करता है, लेकिन
स्पष्ट ऑनबोर्डिंग, Gateway, प्रमाणीकरण, रीसेट या डेमन विकल्पों को
चुपचाप अनदेखा करने के बजाय अस्वीकार करता है। यदि कोई मौजूदा कॉन्फ़िगरेशन अमान्य है, तो बेसलाइन सेटअप
उसे सुरक्षित रखता है और दोबारा प्रयास करने से पहले `openclaw doctor` चलाने को कहता है।

## उदाहरण

```bash
openclaw setup
openclaw setup -m "status"
openclaw setup -m "restart gateway" --yes
openclaw setup --json
openclaw setup --wizard
openclaw setup --baseline
openclaw setup --workspace ~/.openclaw/workspace
openclaw setup --import-from hermes --import-source ~/.hermes
openclaw setup --non-interactive --accept-risk --mode remote --remote-url wss://gateway-host:18789 --remote-token <token>
```

## टिप्पणियाँ

- बेसलाइन सेटअप के बाद, पूर्ण निर्देशित यात्रा के लिए `openclaw onboard`, लक्षित परिवर्तनों के लिए `openclaw configure` या चैनल खाते जोड़ने के लिए `openclaw channels add` चलाएँ।
- यदि Hermes की स्थिति पहचानी जाती है, तो इंटरैक्टिव ऑनबोर्डिंग स्वचालित रूप से माइग्रेशन की पेशकश कर सकती है। आयात ऑनबोर्डिंग के लिए नया सेटअप आवश्यक है; ऑनबोर्डिंग के बाहर ड्राई-रन योजनाओं, बैकअप और ओवरराइट मोड के लिए [माइग्रेट करें](/hi/cli/migrate) का उपयोग करें।
