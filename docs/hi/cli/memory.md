# `openclaw memory`

सिमैंटिक मेमोरी इंडेक्सिंग, खोज और `MEMORY.md` में प्रोमोशन प्रबंधित करें।
बंडल किए गए `memory-core` Plugin द्वारा उपलब्ध कराया गया, और तब उपलब्ध होता है जब
`plugins.slots.memory`, `memory-core` (डिफ़ॉल्ट) चुनता है। अन्य मेमोरी
Plugins अपने स्वयं के CLI नेमस्पेस उपलब्ध कराते हैं।

संबंधित: [मेमोरी](/hi/concepts/memory) अवधारणा, [Dreaming](/hi/concepts/dreaming),
[मेमोरी कॉन्फ़िगरेशन संदर्भ](/hi/reference/memory-config), [मेमोरी विकी](/hi/plugins/memory-wiki),
[विकी](/hi/cli/wiki), [Plugins](/hi/tools/plugin)।

## `memory status`

```bash
openclaw memory status [--agent <id>] [--deep] [--index] [--fix] [--json] [--verbose]
```

`--agent` के बिना, `agents.list` में प्रत्येक एजेंट के लिए चलता है; यदि कोई एजेंट सूची
कॉन्फ़िगर नहीं है, तो डिफ़ॉल्ट एजेंट का उपयोग करता है।

| फ़्लैग        | प्रभाव                                                                                                                                                                                                                                                                                                    |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--deep`    | वेक्टर स्टोर, एम्बेडिंग प्रदाता और सिमैंटिक खोज की तत्परता की जाँच करें (इसमें अतिरिक्त प्रदाता कॉल शामिल होते हैं)। सामान्य `memory status` तेज़ रहता है और इसे छोड़ देता है; अज्ञात वेक्टर/सिमैंटिक स्थिति का अर्थ है कि उसकी जाँच नहीं की गई। QMD लेक्सिकल `searchMode: "search"`, `--deep` के साथ भी, हमेशा सिमैंटिक वेक्टर जाँच छोड़ देता है। |
| `--index`   | स्टोर में अप्रतिबिंबित बदलाव होने पर फिर से इंडेक्स करें। इसमें `--deep` निहित है।                                                                                                                                                                                                                                                          |
| `--fix`     | पुराने रिकॉल लॉक सुधारें और प्रोमोशन मेटाडेटा को सामान्यीकृत करें।                                                                                                                                                                                                                                               |
| `--json`    | JSON प्रिंट करें।                                                                                                                                                                                                                                                                                               |
| `--verbose` | प्रत्येक चरण के विस्तृत लॉग उत्सर्जित करें।                                                                                                                                                                                                                                                                             |

यदि `Dreaming` पंक्ति `dreaming.enabled: true` के साथ भी `off` ही रहती है, या
निर्धारित स्वीप कभी चलते हुए नहीं दिखते, तो प्रबंधित Dreaming Cron मिलान प्रक्रिया ट्रिगर करने के लिए
डिफ़ॉल्ट एजेंट के Heartbeat के सक्रिय होने पर निर्भर करता है। शेड्यूलिंग के विवरण के लिए
[Dreaming](/hi/concepts/dreaming) देखें।

स्थिति में `agents.defaults.memorySearch.extraPaths` के सभी अतिरिक्त खोज पथ भी सूचीबद्ध होते हैं।

## `memory index`

```bash
openclaw memory index [--agent <id>] [--force] [--verbose]
```

प्रति-एजेंट दायरा `status` के समान है। `--force` वृद्धिशील रीइंडेक्स के बजाय
पूर्ण रीइंडेक्स चलाता है। `--verbose` इंडेक्सिंग की प्रगति दिखाने से पहले प्रति-एजेंट प्रदाता,
मॉडल, स्रोत और अतिरिक्त पथों का विवरण प्रिंट करता है।

## `memory search`

```bash
openclaw memory search [query] [--query <text>] [--agent <id>] [--max-results <n>] [--min-score <n>] [--json]
```

- क्वेरी: स्थितीय `[query]` या `--query <text>`। यदि दोनों सेट हैं, तो `--query`
  प्रभावी होता है। यदि कोई भी सेट नहीं है, तो कमांड त्रुटि देता है।
- `--agent <id>`: डिफ़ॉल्ट एजेंट का उपयोग करता है (पूरी एजेंट सूची का नहीं)।
- `--max-results <n>`: परिणामों की संख्या सीमित करें (धनात्मक पूर्णांक)।
- `--min-score <n>`: इस स्कोर से कम मिलानों को फ़िल्टर करें।

## `memory promote`

`memory/YYYY-MM-DD.md` से अल्पकालिक प्रत्याशियों को रैंक करें और वैकल्पिक रूप से
शीर्ष प्रविष्टियाँ `MEMORY.md` में जोड़ें।

```bash
openclaw memory promote [--agent <id>] [--limit <n>] [--min-score <n>] \
  [--min-recall-count <n>] [--min-unique-queries <n>] [--apply] [--include-promoted] [--json]
```

| फ़्लैग                       | डिफ़ॉल्ट      | प्रभाव                                                            |
| -------------------------- | ------------ | ----------------------------------------------------------------- |
| `--limit <n>`              |              | लौटाने/लागू करने के लिए प्रत्याशियों की अधिकतम संख्या।                                   |
| `--min-score <n>`          | `0.75`       | न्यूनतम भारित प्रोमोशन स्कोर।                                 |
| `--min-recall-count <n>`   | `3`          | आवश्यक न्यूनतम रिकॉल संख्या।                                    |
| `--min-unique-queries <n>` | `2`          | आवश्यक अलग-अलग क्वेरियों की न्यूनतम संख्या।                            |
| `--apply`                  | केवल पूर्वावलोकन | चुने गए प्रत्याशियों को `MEMORY.md` में जोड़ें और उन्हें प्रोमोट किया हुआ चिह्नित करें। |
| `--include-promoted`       |              | पिछले चक्रों में पहले ही प्रोमोट किए गए प्रत्याशियों को शामिल करें।           |
| `--json`                   |              | JSON प्रिंट करें।                                                       |

ये CLI डिफ़ॉल्ट निर्धारित Dreaming स्वीप के गहन चरण की
सीमा से अलग हैं (नीचे [Dreaming](#dreaming) देखें); एकबारगी मैन्युअल रन में
स्वीप व्यवहार से मिलान करने के लिए स्पष्ट फ़्लैग पास करें।

रैंकिंग संकेत: रिकॉल आवृत्ति, पुनर्प्राप्ति प्रासंगिकता, क्वेरी विविधता,
समयगत नवीनता, अलग-अलग दिनों में समेकन और व्युत्पन्न अवधारणाओं की समृद्धि, जो
मेमोरी रिकॉल और दैनिक अंतर्ग्रहण पास—दोनों से लिए जाते हैं, साथ ही बार-बार Dreaming में
दोबारा आने पर हल्के/REM चरण से मिलने वाला एक छोटा सुदृढ़ीकरण बूस्ट। लिखने से पहले, प्रोमोशन
लाइव दैनिक नोट को दोबारा पढ़ता है, ताकि रैंकिंग के बाद अल्पकालिक स्निपेट में किए गए
संपादन या विलोपन का सम्मान हो और किसी पुराने स्नैपशॉट से प्रोमोशन न किया जाए।

## `memory promote-explain`

किसी एक प्रोमोशन प्रत्याशी के स्कोर का विवरण समझाएँ।

```bash
openclaw memory promote-explain <selector> [--agent <id>] [--include-promoted] [--json]
```

`<selector>` किसी प्रत्याशी की कुंजी (सटीक या उप-स्ट्रिंग), पथ या स्निपेट
टेक्स्ट से मिलान करता है।

## `memory rem-harness`

कुछ भी लिखे बिना REM चिंतन, प्रत्याशी सत्य और गहन चरण के प्रोमोशन आउटपुट का
पूर्वावलोकन करें।

```bash
openclaw memory rem-harness [--agent <id>] [--path <file-or-dir>] [--grounded] [--include-promoted] [--json]
```

- `--path <file-or-dir>`: लाइव वर्कस्पेस के बजाय ऐतिहासिक `YYYY-MM-DD.md`
  दैनिक फ़ाइलों से हार्नेस आरंभ करें।
- `--grounded`: ऐतिहासिक नोट्स से एक आधारयुक्त `What Happened` / `Reflections` /
  `Possible Lasting Updates` पूर्वावलोकन भी रेंडर करें।

## `memory rem-backfill`

UI समीक्षा के लिए आधारयुक्त ऐतिहासिक REM सारांश `DREAMS.md` में लिखें।
इसे पूर्ववत किया जा सकता है।

```bash
openclaw memory rem-backfill --path <file-or-dir> [--agent <id>] [--stage-short-term] [--json]
openclaw memory rem-backfill --rollback [--rollback-short-term] [--json]
```

- `--path <file-or-dir>`: जब तक `--rollback`/`--rollback-short-term`
  सेट न हो, आवश्यक है। वे ऐतिहासिक दैनिक मेमोरी फ़ाइलें या डायरेक्टरी, जिनसे बैकफ़िल करना है।
- `--stage-short-term`: लाइव अल्पकालिक प्रोमोशन स्टोर में आधारयुक्त टिकाऊ प्रत्याशी भी
  डालें, ताकि सामान्य गहन चरण उन्हें रैंक कर सके।
- `--rollback`: `DREAMS.md` से पहले लिखी गई आधारयुक्त डायरी प्रविष्टियाँ
  हटाएँ।
- `--rollback-short-term`: पहले से तैयार किए गए आधारयुक्त अल्पकालिक
  प्रत्याशी हटाएँ।

## Dreaming

Dreaming पृष्ठभूमि में चलने वाली मेमोरी समेकन प्रणाली है, जिसमें एक ही शेड्यूल पर क्रम से
चलने वाले तीन सहयोगी चरण हैं: **हल्का** (अल्पकालिक सामग्री को क्रमबद्ध/तैयार करना),
**REM** (चिंतन करना और विषय सामने लाना), **गहन** (टिकाऊ तथ्यों को
`MEMORY.md` में प्रोमोट करना)। केवल गहन चरण `MEMORY.md` में लिखता है।

- `plugins.entries.memory-core.config.dreaming.enabled: true` से सक्षम करें
  (डिफ़ॉल्ट `false`); `memory-core` स्वीप Cron जॉब को स्वतः प्रबंधित करता है, मैन्युअल
  `openclaw cron add` आवश्यक नहीं है।
- चैट से `/dreaming on|off` के साथ टॉगल करें; `/dreaming status`
  (या `/dreaming`/`/dreaming help`) से निरीक्षण करें। `on`/`off` के लिए चैनल स्वामी की स्थिति
  या Gateway `operator.admin` आवश्यक है; `status` और सहायता ऐसे किसी भी व्यक्ति के लिए उपलब्ध रहते हैं
  जो कमांड चला सकता है।
- मानव-पठनीय चरण आउटपुट `DREAMS.md` (या किसी मौजूदा `dreams.md`) में जाता है।
  डिफ़ॉल्ट रूप से (`dreaming.storage.mode: "separate"`) प्रत्येक चरण
  `memory/dreaming/<phase>/YYYY-MM-DD.md` में एक स्वतंत्र रिपोर्ट भी लिखता है; रिपोर्टों को दैनिक मेमोरी फ़ाइल में सम्मिलित करने के लिए `mode:
"inline"`, या दोनों के लिए `"both"`
  सेट करें।
- निर्धारित और मैन्युअल `memory promote` रन एक ही गहन चरण के
  रैंकिंग संकेत साझा करते हैं; केवल डिफ़ॉल्ट सीमाएँ अलग हैं (ऊपर दी गई तालिका की तुलना
  नीचे दिए गए निर्धारित डिफ़ॉल्ट से करें)।
- निर्धारित रन प्रत्येक कॉन्फ़िगर किए गए एजेंट के मेमोरी वर्कस्पेस में समानांतर रूप से फैलते हैं।

निर्धारित डिफ़ॉल्ट (`plugins.entries.memory-core.config.dreaming`):

| कुंजी                                    | डिफ़ॉल्ट     |
| -------------------------------------- | ----------- |
| `frequency`                            | `0 3 * * *` |
| `phases.deep.minScore`                 | `0.8`       |
| `phases.deep.minRecallCount`           | `3`         |
| `phases.deep.minUniqueQueries`         | `3`         |
| `phases.deep.recencyHalfLifeDays`      | `14`        |
| `phases.deep.maxAgeDays`               | `30`        |
| `phases.deep.maxPromotedSnippetTokens` | `160`       |

```json
{
  "plugins": {
    "entries": {
      "memory-core": {
        "config": {
          "dreaming": {
            "enabled": true
          }
        }
      }
    }
  }
}
```

कुंजियों की पूरी सूची और चरणों का विवरण: [Dreaming](/hi/concepts/dreaming),
[मेमोरी कॉन्फ़िगरेशन संदर्भ](/hi/reference/memory-config#dreaming)।

## SecretRef Gateway निर्भरता

यदि Active Memory के रिमोट API कुंजी फ़ील्ड SecretRefs के रूप में कॉन्फ़िगर किए गए हैं, तो `memory`
कमांड उन्हें सक्रिय Gateway स्नैपशॉट से हल करते हैं; यदि Gateway
अनुपलब्ध है, तो कमांड तुरंत विफल हो जाता है। इसके लिए `secrets.resolve` विधि का समर्थन करने वाला
Gateway आवश्यक है; पुराने Gateway अज्ञात-विधि त्रुटि लौटाते हैं।
