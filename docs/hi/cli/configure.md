# `openclaw configure`

मौजूदा सेटअप में लक्षित बदलावों के लिए इंटरैक्टिव प्रॉम्प्ट: क्रेडेंशियल, डिवाइस, एजेंट डिफ़ॉल्ट, Gateway, चैनल, plugins, skills और स्वास्थ्य जाँच।

पूरी निर्देशित प्रथम-रन प्रक्रिया के लिए `openclaw onboard` या `openclaw setup`, केवल आधारभूत कॉन्फ़िगरेशन/वर्कस्पेस के लिए `openclaw setup --baseline`, और जब केवल चैनल अकाउंट सेटअप की आवश्यकता हो तब `openclaw channels add` का उपयोग करें।

<Tip>
बिना किसी सबकमांड के `openclaw config` वही विज़ार्ड खोलता है। गैर-इंटरैक्टिव बदलावों के लिए `openclaw config get|set|unset` का उपयोग करें।
</Tip>

## विकल्प

`--section <section>`: दोहराने योग्य अनुभाग फ़िल्टर। उपलब्ध अनुभाग:

`workspace`, `model`, `web`, `gateway`, `daemon`, `channels`, `plugins`, `skills`, `health`

```bash
openclaw configure
openclaw configure --section web
openclaw configure --section model --section channels
openclaw configure --section gateway --section daemon
```

`gateway`, `daemon`, या `health` चुनने पर (या बिना `--section` के पूरा विज़ार्ड चलाने पर) यह पूछा जाता है कि Gateway कहाँ चलता है और `gateway.mode` को अपडेट किया जाता है। तीनों को छोड़ने वाले अनुभाग फ़िल्टर बिना Gateway-मोड प्रॉम्प्ट के सीधे अनुरोधित सेटअप पर जाते हैं। रिमोट Gateway मोड चुनने पर रिमोट कॉन्फ़िगरेशन लिखा जाता है और प्रक्रिया तुरंत समाप्त हो जाती है; यह plugin इंस्टॉलेशन जैसे केवल-स्थानीय चरण नहीं चलाता।

<Note>
`openclaw configure` के लिए इंटरैक्टिव टर्मिनल आवश्यक है (stdin और stdout, दोनों TTY होने चाहिए)। इसके बिना, यह आंशिक रूप से चलने के बजाय समकक्ष गैर-इंटरैक्टिव `openclaw config get|set|patch|validate` कमांड प्रिंट करता है और त्रुटि के साथ समाप्त हो जाता है।
</Note>

## मॉडल अनुभाग

<Note>
**मॉडल** में `agents.defaults.models` अनुमति-सूची के लिए बहु-चयन शामिल है (जो `/model` और मॉडल चयनकर्ता में दिखाई देता है)। प्रदाता-विशिष्ट सेटअप विकल्प, कॉन्फ़िगरेशन में पहले से मौजूद असंबंधित प्रदाताओं को बदलने के बजाय, अपने चयनित मॉडल को मौजूदा अनुमति-सूची में मिला देते हैं।

configure से प्रदाता प्रमाणीकरण दोबारा चलाने पर मौजूदा `agents.defaults.model.primary` सुरक्षित रहता है, भले ही प्रदाता का प्रमाणीकरण चरण अपने अनुशंसित डिफ़ॉल्ट मॉडल वाला कॉन्फ़िगरेशन पैच लौटाए। किसी प्रदाता को जोड़ने या उसका पुनः प्रमाणीकरण करने से उसके मॉडल उपलब्ध हो जाते हैं, लेकिन वे आपके मौजूदा प्राथमिक मॉडल का स्थान नहीं लेते। डिफ़ॉल्ट मॉडल को जानबूझकर बदलने के लिए `openclaw models auth login --provider <id> --set-default` या `openclaw models set <model>` का उपयोग करें।
</Note>

जब configure किसी प्रदाता प्रमाणीकरण विकल्प से शुरू होता है, तो डिफ़ॉल्ट-मॉडल और अनुमति-सूची चयनकर्ता स्वतः उस प्रदाता को प्राथमिकता देते हैं। Volcengine और BytePlus जैसे युग्मित प्रदाताओं के लिए यही प्राथमिकता उनके कोडिंग-प्लान प्रकारों (`volcengine-plan/*`, `byteplus-plan/*`) से भी मेल खाती है। यदि पसंदीदा-प्रदाता फ़िल्टर से सूची खाली हो जाती, तो रिक्त चयनकर्ता दिखाने के बजाय configure बिना फ़िल्टर वाले कैटलॉग पर वापस चला जाता है।

## वेब अनुभाग

`openclaw configure --section web` किसी वेब-खोज प्रदाता को चुनता है और उसके क्रेडेंशियल कॉन्फ़िगर करता है। कुछ प्रदाता, प्रदाता-विशिष्ट अनुवर्ती विकल्प दिखाते हैं:

- **Grok** उसी xAI OAuth प्रोफ़ाइल या API कुंजी के साथ वैकल्पिक `x_search` सेटअप की पेशकश कर सकता है और आपको `x_search` मॉडल चुनने देता है।
- **Kimi** Moonshot API क्षेत्र (`api.moonshot.ai` बनाम `api.moonshot.cn`) और डिफ़ॉल्ट Kimi वेब-खोज मॉडल पूछ सकता है।

## अन्य टिप्पणियाँ

- स्थानीय कॉन्फ़िगरेशन लिखे जाने के बाद, चुने गए सेटअप पथ द्वारा आवश्यक होने पर configure चयनित डाउनलोड योग्य plugins इंस्टॉल करता है। रिमोट Gateway कॉन्फ़िगरेशन स्थानीय plugin पैकेज इंस्टॉल नहीं करता।
- चैनल-केंद्रित सेवाएँ (Slack/Discord/Matrix/Microsoft Teams) सेटअप के दौरान चैनल/रूम अनुमति-सूचियाँ पूछती हैं। आप नाम या ID दर्ज कर सकते हैं; जहाँ संभव हो, विज़ार्ड नामों को ID में बदलता है।
- यदि आप डेमन इंस्टॉलेशन चरण चलाते हैं, तो टोकन प्रमाणीकरण के लिए टोकन आवश्यक है। यदि `gateway.auth.token` को SecretRef द्वारा प्रबंधित किया जाता है, तो configure SecretRef को सत्यापित करता है, लेकिन समाधान किए गए सादे-पाठ टोकन मानों को सुपरवाइज़र सेवा के परिवेश मेटाडेटा में स्थायी रूप से संग्रहीत नहीं करता; यदि SecretRef का समाधान नहीं होता, तो configure सुधार के लिए कार्रवाई योग्य मार्गदर्शन के साथ डेमन इंस्टॉलेशन को रोक देता है।
- यदि `gateway.auth.token` और `gateway.auth.password`, दोनों कॉन्फ़िगर किए गए हैं और `gateway.auth.mode` सेट नहीं है, तो configure तब तक डेमन इंस्टॉलेशन रोकता है जब तक आप मोड स्पष्ट रूप से सेट नहीं करते।
