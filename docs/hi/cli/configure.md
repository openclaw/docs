# `openclaw configure`

मौजूदा सेटअप में लक्षित बदलावों के लिए इंटरैक्टिव प्रॉम्प्ट: क्रेडेंशियल, डिवाइस, एजेंट डिफ़ॉल्ट, Gateway, चैनल, plugins, skills और स्वास्थ्य जाँच।

पूरी निर्देशित प्रथम-रन प्रक्रिया के लिए `openclaw onboard` या `openclaw setup`, केवल आधारभूत कॉन्फ़िगरेशन/वर्कस्पेस के लिए `openclaw setup --baseline`, और केवल चैनल अकाउंट सेटअप की आवश्यकता होने पर `openclaw channels add` का उपयोग करें।

<Tip>
बिना किसी सबकमांड के `openclaw config` वही विज़ार्ड खोलता है। गैर-इंटरैक्टिव बदलावों के लिए `openclaw config get|set|unset` का उपयोग करें।
</Tip>

## विकल्प

`--section <section>`: दोहराया जा सकने वाला सेक्शन फ़िल्टर। उपलब्ध सेक्शन:

`workspace`, `model`, `web`, `gateway`, `daemon`, `channels`, `plugins`, `skills`, `health`

```bash
openclaw configure
openclaw configure --section web
openclaw configure --section model --section channels
openclaw configure --section gateway --section daemon
```

`gateway`, `daemon` या `health` चुनने पर (या बिना `--section` के पूरा विज़ार्ड चलाने पर) यह पूछा जाता है कि Gateway कहाँ चलता है और `gateway.mode` को अपडेट किया जाता है। तीनों को छोड़ने वाले सेक्शन फ़िल्टर किसी Gateway-मोड प्रॉम्प्ट के बिना सीधे अनुरोधित सेटअप पर जाते हैं। रिमोट Gateway मोड चुनने पर रिमोट कॉन्फ़िगरेशन लिखा जाता है और प्रक्रिया तुरंत समाप्त हो जाती है; यह plugin इंस्टॉल जैसे केवल-लोकल चरण नहीं चलाता।

<Note>
`openclaw configure` के लिए इंटरैक्टिव टर्मिनल आवश्यक है (stdin और stdout, दोनों TTY होने चाहिए)। इसके बिना यह समकक्ष गैर-इंटरैक्टिव `openclaw config get|set|patch|validate` कमांड प्रिंट करता है और आंशिक रूप से चलने के बजाय त्रुटि के साथ बंद हो जाता है।
</Note>

## मॉडल सेक्शन

<Note>
**मॉडल** में स्पष्ट `agents.defaults.modelPolicy.allow` सूची के लिए बहु-चयन शामिल है (जो `/model` और मॉडल चयनकर्ता में दिखाई देता है)। प्रोवाइडर-स्कोप वाले सेटअप विकल्प अपने चुने हुए मॉडल को मौजूदा सूची में मर्ज करते हैं, न कि कॉन्फ़िगरेशन में पहले से मौजूद असंबंधित प्रोवाइडर को प्रतिस्थापित करते हैं। प्रत्येक मॉडल के उपनाम और पैरामीटर `agents.defaults.models` के अंतर्गत रहते हैं; वे प्रविष्टियाँ स्वयं मॉडल ओवरराइड को सीमित नहीं करतीं।

configure से प्रोवाइडर प्रमाणीकरण फिर से चलाने पर मौजूदा `agents.defaults.model.primary` सुरक्षित रहता है, भले ही प्रोवाइडर का प्रमाणीकरण चरण अपने अनुशंसित डिफ़ॉल्ट मॉडल वाला कॉन्फ़िगरेशन पैच लौटाए। किसी प्रोवाइडर को जोड़ने या फिर से प्रमाणित करने पर उसके मॉडल उपलब्ध हो जाते हैं, लेकिन वह आपके वर्तमान प्राथमिक मॉडल का स्थान नहीं लेता। डिफ़ॉल्ट मॉडल को जानबूझकर बदलने के लिए `openclaw models auth login --provider <id> --set-default` या `openclaw models set <model>` का उपयोग करें।
</Note>

जब configure किसी प्रोवाइडर प्रमाणीकरण विकल्प से शुरू होता है, तो डिफ़ॉल्ट-मॉडल और मॉडल-नीति चयनकर्ता स्वचालित रूप से उस प्रोवाइडर को प्राथमिकता देते हैं। Volcengine और BytePlus जैसे युग्मित प्रोवाइडर के लिए यही प्राथमिकता उनके कोडिंग-प्लान वेरिएंट (`volcengine-plan/*`, `byteplus-plan/*`) से भी मेल खाती है। यदि पसंदीदा-प्रोवाइडर फ़िल्टर से सूची खाली हो जाती, तो configure खाली चयनकर्ता दिखाने के बजाय बिना फ़िल्टर वाले कैटलॉग पर वापस चला जाता है।

## वेब सेक्शन

`openclaw configure --section web` वेब-सर्च प्रोवाइडर चुनता है और उसके क्रेडेंशियल कॉन्फ़िगर करता है। कुछ प्रोवाइडर, प्रोवाइडर-विशिष्ट अनुवर्ती विकल्प दिखाते हैं:

- **Grok** उसी xAI OAuth प्रोफ़ाइल या API कुंजी के साथ वैकल्पिक `x_search` सेटअप प्रस्तुत कर सकता है और आपको `x_search` मॉडल चुनने दे सकता है।
- **Kimi** Moonshot API क्षेत्र (`api.moonshot.ai` बनाम `api.moonshot.cn`) और डिफ़ॉल्ट Kimi वेब-सर्च मॉडल पूछ सकता है।

## अन्य टिप्पणियाँ

- लोकल कॉन्फ़िगरेशन लिखने के बाद, चुने गए सेटअप पथ के लिए आवश्यक होने पर configure चयनित डाउनलोड किए जा सकने वाले plugins इंस्टॉल करता है। रिमोट Gateway कॉन्फ़िगरेशन लोकल plugin पैकेज इंस्टॉल नहीं करता।
- चैनल-केंद्रित सेवाएँ (Slack/Discord/Matrix/Microsoft Teams) सेटअप के दौरान चैनल/रूम की अनुमति-सूचियाँ माँगती हैं। आप नाम या ID दर्ज कर सकते हैं; जहाँ संभव हो, विज़ार्ड नामों को ID में बदलता है।
- यदि आप डेमन इंस्टॉल चरण चलाते हैं, तो टोकन प्रमाणीकरण के लिए टोकन आवश्यक है। यदि `gateway.auth.token` SecretRef द्वारा प्रबंधित है, तो configure SecretRef को सत्यापित करता है, लेकिन हल किए गए सादे-पाठ टोकन मानों को सुपरवाइज़र सेवा के पर्यावरण मेटाडेटा में स्थायी रूप से संग्रहीत नहीं करता; यदि SecretRef हल नहीं हुआ है, तो configure कार्रवाई योग्य समाधान मार्गदर्शन के साथ डेमन इंस्टॉल को रोक देता है।
- यदि `gateway.auth.token` और `gateway.auth.password`, दोनों कॉन्फ़िगर हैं और `gateway.auth.mode` सेट नहीं है, तो configure आपको मोड स्पष्ट रूप से सेट करने तक डेमन इंस्टॉल को रोक देता है।
