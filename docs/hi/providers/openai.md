OpenClaw प्रत्यक्ष API-कुंजी प्रमाणीकरण और
ChatGPT/Codex सदस्यता प्रमाणीकरण, दोनों के लिए एक ही प्रदाता आईडी, `openai`, का उपयोग करता है। `openai/*` प्रामाणिक मॉडल रूट है।
रनटाइम नीति सेट न होने या `auto` होने पर एम्बेडेड एजेंट टर्न के लिए, OpenAI के रूट
तथ्य तय करते हैं कि OpenClaw बंडल किए गए Codex ऐप-सर्वर रनटाइम को
अंतर्निहित रूप से चुन सकता है या नहीं। केवल `openai/*` प्रीफ़िक्स रनटाइम नहीं चुनता।

- **एजेंट मॉडल** - स्पष्ट
  `agentRuntime` कॉन्फ़िगरेशन या OpenAI की अंतर्निहित रूट नीति द्वारा चुने गए रनटाइम के माध्यम से `openai/*`। ChatGPT/Codex सदस्यता के उपयोग के लिए Codex
  प्रमाणीकरण से साइन इन करें, या जब आप कुंजी-आधारित बिलिंग चाहते हों, तब API-कुंजी प्रमाणीकरण
  प्रोफ़ाइल कॉन्फ़िगर करें।
- **गैर-एजेंट OpenAI API** - `OPENAI_API_KEY` या किसी `openai` API-कुंजी प्रमाणीकरण प्रोफ़ाइल के
  माध्यम से प्रत्यक्ष OpenAI Platform पहुँच, जिसका शुल्क प्रति उपयोग लिया जाता है।
- **लेगेसी कॉन्फ़िगरेशन** - `codex/*` और `openai-codex/*` संदर्भों को
  `openclaw doctor --fix` द्वारा `openai/*` और मॉडल-स्कोप्ड `agentRuntime.id: "codex"` में
  सुधारा जाता है।

OpenAI, OpenClaw जैसे बाहरी टूल और
वर्कफ़्लो में सदस्यता OAuth के उपयोग का स्पष्ट रूप से समर्थन करता है।

## उपयोग और लागत ट्रैकिंग

OpenClaw सदस्यता कोटा और Platform API बिलिंग को अलग रखता है:

- ChatGPT/Codex OAuth सदस्यता प्लान, कोटा विंडो और क्रेडिट शेष दिखाता है।
- `OPENAI_ADMIN_KEY` Control UI के **उपयोग** में प्रदाता द्वारा रिपोर्ट की गई संगठन लागत और पूर्णता उपयोग के 30 दिन दिखाता है, जिसमें दैनिक खर्च, अनुरोध/टोकन के कुल योग, शीर्ष मॉडल और लागत श्रेणियाँ शामिल हैं।
- `OPENAI_PROJECT_ID` वैकल्पिक रूप से Admin API इतिहास को एक प्रोजेक्ट तक सीमित करता है।
- OpenClaw कभी भी `OPENAI_API_KEY` या किसी `openai` इनफ़रेंस प्रोफ़ाइल को संगठन API पर नहीं भेजता; वे क्रेडेंशियल कस्टम, Azure या एजेंट-स्थानीय एंडपॉइंट से संबंधित हो सकते हैं।

स्पष्ट Admin कुंजी को OAuth पर प्राथमिकता मिलती है। प्रदाता द्वारा रिपोर्ट किया गया इतिहास, OpenClaw की सत्र-व्युत्पन्न अनुमानित लागत के साथ मर्ज नहीं किया जाता; इसमें अन्य क्लाइंट की API गतिविधि और प्रदाता-पक्षीय बिलिंग समायोजन शामिल हो सकते हैं।

OpenAI के [API उपयोग डैशबोर्ड](https://help.openai.com/en/articles/10478918) दस्तावेज़, उपयोग डेटा के लिए संगठन-स्वामी और स्पष्ट Usage Dashboard अनुमति आवश्यकताओं का वर्णन करते हैं।

प्रदाता, मॉडल, रनटाइम और चैनल अलग-अलग परतें हैं। यदि इन लेबलों को
आपस में मिलाया जा रहा है, तो कॉन्फ़िगरेशन बदलने से पहले
[एजेंट रनटाइम](/hi/concepts/agent-runtimes) पढ़ें।

## अंतर्निहित एजेंट रनटाइम

जब प्रदाता/मॉडल `agentRuntime` नीति सेट नहीं होती या `auto` होती है, तब OpenAI की
प्रदाता-स्वामित्व वाली रूट नीति प्रभावी
एंडपॉइंट और अडैप्टर से अंतर्निहित रनटाइम चुनती है:

| प्रभावी रूट तथ्य                                                                                                                                                  | अंतर्निहित रनटाइम      |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------- |
| `openai-responses` वाला सटीक आधिकारिक Platform HTTPS एंडपॉइंट, या `openai-chatgpt-responses` वाला सटीक आधिकारिक ChatGPT HTTPS एंडपॉइंट; कोई लिखित अनुरोध ओवरराइड नहीं | Codex चुना जा सकता है |
| लिखित `openai-completions` अडैप्टर                                                                                                                                  | OpenClaw              |
| कस्टम एंडपॉइंट                                                                                                                                                        | OpenClaw              |
| HTTP का उपयोग करने वाला स्पष्ट सटीक आधिकारिक एंडपॉइंट                                                                                                                            | अस्वीकृत              |
| लिखित प्रदाता/मॉडल अनुरोध ओवरराइड वाला रूट                                                                                                                 | OpenClaw              |

स्पष्ट गैर-डिफ़ॉल्ट प्रदाता/मॉडल `agentRuntime.id` निर्णायक बना रहता है।
उदाहरण के लिए, `agentRuntime.id: "openclaw"` अन्यथा Codex-योग्य
रूट को OpenClaw पर रखता है, जबकि `agentRuntime.id: "codex"` को Codex की आवश्यकता होती है और जब
प्रभावी रूट को Codex-संगत घोषित नहीं किया गया हो, तब यह बंद होकर विफल होता है।
रनटाइम चयन क्रेडेंशियल प्रकार या बिलिंग नहीं बदलता: Platform API-कुंजी
प्रमाणीकरण और ChatGPT/Codex सदस्यता प्रमाणीकरण अलग रहते हैं।

`openclaw doctor --fix` लेगेसी `codex/*` और `openai-codex/*` मॉडल
संदर्भों, लेगेसी Codex प्रमाणीकरण प्रोफ़ाइल आईडी और लेगेसी Codex प्रमाणीकरण-क्रम प्रविष्टियों को
प्रामाणिक `openai` रूट में माइग्रेट करता है। माइग्रेट किए गए मॉडल संदर्भों को मॉडल-स्कोप्ड
`agentRuntime.id: "codex"` मिलता है; नए प्रमाणीकरण-क्रम कॉन्फ़िगरेशन के लिए `auth.order.openai` का उपयोग करें।

<Note>
नया OpenAI सेटअप GPT-5.6 को प्राथमिक मॉडल के रूप में केवल तभी लागू करता है, जब कोई प्राथमिक मॉडल
कॉन्फ़िगर नहीं हो। OpenAI प्रमाणीकरण जोड़ने या रीफ़्रेश करने पर मौजूदा स्पष्ट
चयन सुरक्षित रहता है, जिसमें `openai/gpt-5.5` भी शामिल है, जब तक कि आप स्पष्ट रूप से
`models auth login --set-default` या `models set` का उपयोग न करें। किसी एजेंट मॉडल के लिए API-कुंजी प्रमाणीकरण
केवल तभी उपयोग करें, जब आप API-कुंजी प्रमाणीकरण चाहते हों।
</Note>

## GPT-5.6 सीमित पूर्वावलोकन

OpenClaw सटीक `openai/gpt-5.6-sol`,
`openai/gpt-5.6-terra`, और `openai/gpt-5.6-luna` मॉडल आईडी को पहचानता है। वर्तमान कैटलॉग में ये तीनों
`xhigh` और `max` रीजनिंग प्रदान करते हैं। OpenAI Sol को
फ़्लैगशिप टियर, Terra को संतुलित टियर और Luna को तेज़,
कम-लागत वाला टियर बताता है। 
[GPT-5.6 लॉन्च घोषणा](https://openai.com/index/previewing-gpt-5-6-sol/)
और [पहुँच मार्गदर्शिका](https://help.openai.com/en/articles/20001325-a-preview-of-gpt-5-6-sol-terra-and-luna) देखें।

प्रत्यक्ष OpenAI API-कुंजी प्रमाणीकरण के साथ, केवल `openai/gpt-5.6` आईडी Sol का उपनाम है
और नए सेटअप का डिफ़ॉल्ट है। नेटिव Codex कैटलॉग उस प्रत्यक्ष-API उपनाम को
क्लाइंट-पक्ष पर लागू नहीं करता; वर्कस्पेस पहुँच के आधार पर, यह
सटीक Sol, Terra और Luna आईडी दिखा सकता है। इसलिए नया ChatGPT/Codex OAuth सेटअप
`openai/gpt-5.6-sol` का उपयोग करता है। वर्तमान खाते की जाँच इसके साथ करें:

```bash
openclaw models list --provider openai
```

API संगठन और Codex वर्कस्पेस की पहुँच अलग हो सकती है। यदि GPT-5.6
उपलब्ध नहीं है, तो GPT-5.5 को स्पष्ट रूप से चुनें:

```bash
openclaw models set openai/gpt-5.5
```

OpenClaw अपस्ट्रीम पहुँच त्रुटि दिखाता है और GPT-5.6 चयन को
चुपचाप GPT-5.5 से प्रतिस्थापित नहीं करता।

<Note>
पात्र सटीक आधिकारिक HTTPS रूट, रनटाइम नीति सेट न होने या `auto` होने पर बंडल किए गए Codex ऐप-सर्वर
Plugin को चुन सकते हैं; लिखित Completions रूट,
कस्टम एंडपॉइंट और अनुरोध-ट्रांसपोर्ट ओवरराइड OpenClaw पर बने रहते हैं। प्लेनटेक्स्ट
आधिकारिक HTTP एंडपॉइंट अस्वीकार किए जाते हैं। स्पष्ट प्रदाता/मॉडल रनटाइम कॉन्फ़िगरेशन
निर्णायक रहता है। पुराने लेगेसी Codex मॉडल
संदर्भों, `codex-cli/*` संदर्भों या स्पष्ट रनटाइम कॉन्फ़िगरेशन द्वारा सेट न किए गए पुराने रनटाइम सत्र पिन को सुधारने के लिए `openclaw doctor --fix` चलाएँ।
</Note>

API** के माध्यम से जाती है और इसके लिए Platform API कुंजी आवश्यक है। इसके बजाय Codex OAuth टोकन
ChatGPT Codex बैकएंड को प्रमाणित करते हैं; सार्वजनिक Realtime एंडपॉइंट के लिए वे Platform API
कुंजियों के साथ परस्पर विनिमेय नहीं हैं।

यदि API-कुंजी प्रमाणीकरण में बिलिंग अनुपलब्ध होने की सूचना मिले, तो API-कुंजी
प्रमाणीकरण का उपयोग करते समय आपके रीयलटाइम क्रेडेंशियल का समर्थन करने वाले संगठन के लिए
[platform.openai.com/account/billing](https://platform.openai.com/account/billing)
पर Platform क्रेडिट टॉप अप करें। रीयलटाइम वॉइस `openclaw onboard --auth-choice openai-api-key` द्वारा बनाए गए
`openai` API-कुंजी प्रमाणीकरण प्रोफ़ाइल, Control UI Talk के लिए
`talk.realtime.providers.openai.apiKey` के माध्यम से सेट की गई Platform API कुंजी, या Voice
Call के लिए `plugins.entries.voice-call.config.realtime.providers.openai.apiKey`, अथवा `OPENAI_API_KEY` पर्यावरण चर को स्वीकार करती है।

Control UI Video Talk में, OpenAI WebRTC माँग पर कैमरा संदर्भ प्राप्त करता है:
जब मॉडल `describe_view` को कॉल करता है, तो ब्राउज़र रीयलटाइम डेटा चैनल पर
एक सीमित JPEG भेजता है। OpenClaw, OpenAI सत्र में निरंतर कैमरा ट्रैक
संलग्न नहीं करता।
</Note>

## मेमोरी एम्बेडिंग

OpenClaw, `memory_search` इंडेक्सिंग और क्वेरी एम्बेडिंग के लिए
OpenAI या OpenAI-संगत एम्बेडिंग एंडपॉइंट का उपयोग कर सकता है:

```json5
{
  agents: {
    defaults: {
      memorySearch: {
        provider: "openai",
        model: "text-embedding-3-small",
      },
    },
  },
}
```

ऐसे OpenAI-संगत एंडपॉइंट के लिए जिन्हें असममित एम्बेडिंग लेबल की आवश्यकता होती है,
`memorySearch` के अंतर्गत `queryInputType` और `documentInputType` सेट करें। OpenClaw
इन्हें प्रदाता-विशिष्ट `input_type` अनुरोध फ़ील्ड के रूप में अग्रेषित करता है: क्वेरी
एम्बेडिंग `queryInputType` का उपयोग करती हैं; इंडेक्स किए गए मेमोरी खंड और बैच इंडेक्सिंग
`documentInputType` का उपयोग करते हैं। पूरे उदाहरण के लिए
[मेमोरी कॉन्फ़िगरेशन संदर्भ](/hi/reference/memory-config#provider-specific-config)
देखें।

    ### कॉन्फ़िगरेशन उदाहरण

    ```json5
    {
      plugins: { entries: { codex: { enabled: true } } },
      agents: {
        defaults: {
          model: { primary: "openai/gpt-5.6-sol" },
        },
      },
    }
    ```

    API-कुंजी बैकअप के साथ, चयनित मॉडल को `openai/*` के अंतर्गत रखें और
    प्रमाणीकरण क्रम को `openai` के अंतर्गत रखें। OpenClaw पहले सदस्यता, फिर
    API कुंजी आज़माता है और इस दौरान Codex हार्नेस पर बना रहता है:

    ```json5
    {
      plugins: { entries: { codex: { enabled: true } } },
      agents: {
        defaults: {
          model: { primary: "openai/gpt-5.6-sol" },
        },
      },
      auth: {
        order: {
          openai: [
            "openai:user@example.com",
            "openai:api-key-backup",
          ],
        },
      },
    }
    ```

    <Note>
    ऑनबोर्डिंग अब `~/.codex` से OAuth सामग्री आयात नहीं करती। ब्राउज़र
    OAuth (डिफ़ॉल्ट) या ऊपर दिए गए डिवाइस-कोड प्रवाह से साइन इन करें; OpenClaw परिणामी
    क्रेडेंशियल्स को अपने एजेंट प्रमाणीकरण स्टोर में प्रबंधित करता है।
    </Note>

    ### Codex OAuth रूटिंग की जाँच और पुनर्प्राप्ति

    ```bash
    openclaw models status
    openclaw models auth list --provider openai
    openclaw config get agents.defaults.model --json
    openclaw config get models.providers.openai.agentRuntime --json
    ```

    किसी विशिष्ट एजेंट के लिए, `--agent <id>` जोड़ें:

    ```bash
    openclaw models status --agent <id>
    openclaw models auth list --agent <id> --provider openai
    ```

    यदि किसी पुराने कॉन्फ़िगरेशन में अब भी विरासती Codex GPT संदर्भ हैं, या स्पष्ट रनटाइम
    कॉन्फ़िगरेशन के बिना कोई पुराना OpenAI रनटाइम सत्र पिन है, तो उसे सुधारें:

    ```bash
    openclaw doctor --fix
    openclaw config validate
    ```

    यदि `models auth list --provider openai` कोई उपयोग योग्य प्रोफ़ाइल नहीं दिखाता है, तो फिर से
    साइन इन करें:

    ```bash
    openclaw models auth login --provider openai
    openclaw models status --probe --probe-provider openai
    ```

    एक ही एजेंट में कई Codex OAuth लॉगिन के लिए `--profile-id` का उपयोग करें, फिर
    उन्हें प्रमाणीकरण क्रम या `/model ...@<profileId>` के माध्यम से नियंत्रित करें:

    ```bash
    openclaw models auth login --provider openai --profile-id openai:ritsuko
    openclaw models auth login --provider openai --profile-id openai:lain
    ```

    प्रोफ़ाइल क्रम पर निर्भर होने से पहले पुराने विरासती OpenAI Codex उपसर्ग वाले
    प्रोफ़ाइल आईडी और क्रम प्रविष्टियों को माइग्रेट करने के लिए `openclaw doctor --fix` चलाएँ।

    ### स्थिति संकेतक

    चैट `/status` दिखाता है कि वर्तमान सत्र के लिए कौन-सा मॉडल रनटाइम
    सक्रिय है। जब कोई पात्र अंतर्निहित रूट या स्पष्ट
    प्रदाता/मॉडल रनटाइम नीति इसे चुनती है, तो बंडल किया गया Codex ऐप-सर्वर हार्नेस
    `Runtime: OpenAI Codex` के रूप में दिखाई देता है।

    ### Doctor चेतावनी

    यदि विरासती Codex मॉडल संदर्भ या पुराने OpenAI रनटाइम पिन कॉन्फ़िगरेशन
    या सत्र स्थिति में बने रहते हैं, तो जब तक OpenClaw स्पष्ट रूप से कॉन्फ़िगर न हो,
    `openclaw doctor --fix` उन्हें Codex रनटाइम के साथ `openai/*` में पुनर्लिखता है।

    ### कॉन्टेक्स्ट विंडो सीमा

    OpenClaw मॉडल मेटाडेटा और रनटाइम कॉन्टेक्स्ट सीमा को अलग-अलग
    मानों के रूप में मानता है। Codex OAuth कैटलॉग के माध्यम से `openai/gpt-5.5` के लिए:

    - मूल `contextWindow`: `400000`
    - डिफ़ॉल्ट रनटाइम `contextTokens` सीमा: `272000`

    व्यवहार में छोटी डिफ़ॉल्ट सीमा बेहतर विलंबता और गुणवत्ता विशेषताएँ देती है।
    इसे `contextTokens` से ओवरराइड करें:

    ```json5
    {
      models: {
        providers: {
          openai: {
            models: [{ id: "gpt-5.5", contextTokens: 160000 }],
          },
        },
      },
    }
    ```

    <Note>
    मूल मॉडल मेटाडेटा घोषित करने के लिए `contextWindow` का उपयोग करें। रनटाइम
    कॉन्टेक्स्ट बजट सीमित करने के लिए `contextTokens` का उपयोग करें। प्रत्यक्ष OpenAI API-कुंजी रूट
    `gpt-5.5` के लिए बड़ा मूल `contextWindow` (`1000000`) रिपोर्ट करता है; दोनों
    रूट अलग-अलग ट्रैक किए जाते हैं क्योंकि अपस्ट्रीम कैटलॉग अलग हैं।
    </Note>

    ### कैटलॉग पुनर्प्राप्ति

    उपलब्ध होने पर OpenClaw `gpt-5.5` के लिए अपस्ट्रीम Codex कैटलॉग
    मेटाडेटा का उपयोग करता है। यदि खाता प्रमाणित होने के बावजूद लाइव Codex खोज
    `gpt-5.5` पंक्ति छोड़ देती है, तो OpenClaw उस OAuth मॉडल पंक्ति का निर्माण करता है ताकि Cron,
    उप-एजेंट और कॉन्फ़िगर किए गए डिफ़ॉल्ट-मॉडल रन
    `Unknown model` के साथ विफल न हों।

  </Tab>
</Tabs>

## मूल Codex ऐप-सर्वर प्रमाणीकरण

जब कोई पात्र सटीक आधिकारिक HTTPS रूट इसे अंतर्निहित रूप से चुनता है, या जब प्रदाता/मॉडल
`agentRuntime.id: "codex"` इसे स्पष्ट रूप से चुनता है, तब मूल Codex ऐप-सर्वर हार्नेस
`openai/*` मॉडल संदर्भों का उपयोग करता है। इसका प्रमाणीकरण अब भी
खाता-आधारित है। OpenClaw इस क्रम में प्रमाणीकरण चुनता है:

1. एजेंट के लिए क्रमबद्ध OpenAI प्रमाणीकरण प्रोफ़ाइल, जिन्हें प्राथमिकता से
   `auth.order.openai` के अंतर्गत रखा जाता है। पुराने विरासती
   Codex प्रमाणीकरण प्रोफ़ाइल आईडी और प्रमाणीकरण क्रम माइग्रेट करने के लिए `openclaw doctor --fix` चलाएँ।
2. ऐप-सर्वर का मौजूदा खाता, जैसे स्थानीय Codex CLI ChatGPT
   साइन-इन। डिफ़ॉल्ट पृथक एजेंट होम के लिए, OpenClaw उस मूल
   CLI खाते को उसके लॉगिन RPC के माध्यम से ऐप-सर्वर से जोड़ता है; यह
   CLI का कॉन्फ़िगरेशन, plugins या थ्रेड स्टोर साझा नहीं करता।
3. केवल स्थानीय stdio ऐप-सर्वर लॉन्च के लिए, और केवल तब जब ऐप-सर्वर
   कोई खाता रिपोर्ट नहीं करता: `CODEX_API_KEY`, फिर `OPENAI_API_KEY`।

स्थानीय ChatGPT/Codex सदस्यता साइन-इन केवल इसलिए प्रतिस्थापित नहीं होता क्योंकि
Gateway प्रक्रिया में प्रत्यक्ष OpenAI मॉडल या
एम्बेडिंग के लिए `OPENAI_API_KEY` भी है। env API-कुंजी फ़ॉलबैक केवल स्थानीय stdio बिना-खाता
पथ पर लागू होता है; इसे WebSocket ऐप-सर्वर कनेक्शन पर कभी नहीं भेजा जाता। जब कोई
सदस्यता-शैली की Codex प्रोफ़ाइल चुनी जाती है, तो OpenClaw
`CODEX_API_KEY` और `OPENAI_API_KEY` को उत्पन्न की गई stdio ऐप-सर्वर चाइल्ड प्रक्रिया से भी बाहर रखता है
और इसके बजाय ऐप-सर्वर लॉगिन RPC के माध्यम से चयनित क्रेडेंशियल्स भेजता है।

जब वह सदस्यता प्रोफ़ाइल Codex उपयोग सीमा के कारण अवरुद्ध होती है, तो OpenClaw
प्रोफ़ाइल को Codex द्वारा बताए गए रीसेट समय तक अवरुद्ध चिह्नित करता है और प्रमाणीकरण
क्रम को अगले `openai:*` प्रोफ़ाइल पर जाने देता है, बिना चयनित
मॉडल बदले या Codex हार्नेस से बाहर निकले। रीसेट समय बीतने के बाद,
सदस्यता प्रोफ़ाइल फिर से पात्र हो जाती है।

## छवि निर्माण

बंडल किया गया `openai` plugin
`image_generate` टूल के माध्यम से छवि निर्माण पंजीकृत करता है। यह समान
`openai/gpt-image-2` मॉडल संदर्भ के माध्यम से OpenAI API-कुंजी और Codex OAuth, दोनों छवि
निर्माण का समर्थन करता है।

| क्षमता                    | OpenAI API कुंजी                    | Codex OAuth                          |
| ------------------------- | ---------------------------------- | ------------------------------------ |
| मॉडल संदर्भ               | `openai/gpt-image-2`               | `openai/gpt-image-2`                 |
| प्रमाणीकरण                | `OPENAI_API_KEY`                   | OpenAI Codex OAuth साइन-इन           |
| परिवहन                    | OpenAI Images API                  | Codex Responses बैकएंड               |
| प्रति अनुरोध अधिकतम छवियाँ | 4                                  | 4                                    |
| संपादन मोड                | सक्षम (अधिकतम 5 संदर्भ छवियाँ)     | सक्षम (अधिकतम 5 संदर्भ छवियाँ)       |
| आकार ओवरराइड              | समर्थित, 2K/4K आकार सहित           | समर्थित, 2K/4K आकार सहित             |
| आस्पेक्ट अनुपात / रिज़ॉल्यूशन | OpenAI Images API को अग्रेषित नहीं | सुरक्षित होने पर समर्थित आकार में मैप किया जाता है |

```json5
{
  agents: {
    defaults: {
      imageGenerationModel: { primary: "openai/gpt-image-2" },
    },
  },
}
```

<Note>
साझा टूल पैरामीटर, प्रदाता चयन और फ़ेलओवर व्यवहार के लिए
[छवि निर्माण](/hi/tools/image-generation) देखें।
</Note>

OpenAI टेक्स्ट-से-छवि निर्माण और छवि संपादन के लिए `gpt-image-2` डिफ़ॉल्ट है।
`gpt-image-1.5`, `gpt-image-1`, और `gpt-image-1-mini` स्पष्ट मॉडल
ओवरराइड के रूप में उपयोग योग्य रहते हैं। पारदर्शी-पृष्ठभूमि PNG/WebP आउटपुट के लिए
`openai/gpt-image-1.5` का उपयोग करें; वर्तमान `gpt-image-2` API
`background: "transparent"` को अस्वीकार करता है।

पारदर्शी-पृष्ठभूमि अनुरोध के लिए, `image_generate` को
`model: "openai/gpt-image-1.5"`, `outputFormat: "png"` या `"webp"`, और
`background: "transparent"` के साथ कॉल करें; पुराना `openai.background` प्रदाता विकल्प
अब भी स्वीकार किया जाता है। OpenClaw डिफ़ॉल्ट `openai/gpt-image-2` पारदर्शी अनुरोधों को
`gpt-image-1.5` में पुनर्लिखकर सार्वजनिक OpenAI और OpenAI Codex OAuth
रूटों की भी सुरक्षा करता है; Azure और कस्टम OpenAI-संगत एंडपॉइंट अपने
कॉन्फ़िगर किए गए डिप्लॉयमेंट/मॉडल नाम बनाए रखते हैं।

यही सेटिंग हेडलेस CLI रन के लिए भी उपलब्ध है:

```bash
openclaw infer image generate \
  --model openai/gpt-image-1.5 \
  --output-format png \
  --background transparent \
  --prompt "पारदर्शी पृष्ठभूमि पर एक साधारण लाल वृत्त स्टिकर" \
  --json
```

किसी इनपुट फ़ाइल से शुरू करते समय `openclaw infer image edit` के साथ समान
`--output-format` और `--background` फ़्लैग का उपयोग करें।
`--openai-background` OpenAI-विशिष्ट उपनाम के रूप में उपलब्ध रहता है। OpenAI Images की गुणवत्ता और लागत
नियंत्रित करने के लिए `--quality low|medium|high|auto` का उपयोग करें।
`image generate` या `image edit` में से किसी से OpenAI का मॉडरेशन संकेत पास करने के लिए
`--openai-moderation low|auto` का उपयोग करें।

ChatGPT/Codex OAuth इंस्टॉल के लिए समान `openai/gpt-image-2` संदर्भ रखें। जब
कोई `openai` OAuth प्रोफ़ाइल कॉन्फ़िगर होती है, तो OpenClaw उस संग्रहीत OAuth
एक्सेस टोकन को रिज़ॉल्व करता है और Codex Responses बैकएंड के माध्यम से छवि अनुरोध भेजता है; यह
पहले `OPENAI_API_KEY` आज़माता नहीं है और न ही चुपचाप API कुंजी पर फ़ॉलबैक करता है।
जब आप इसके बजाय प्रत्यक्ष OpenAI Images API रूट चाहते हैं, तो API कुंजी, कस्टम आधार
URL या Azure एंडपॉइंट के साथ `models.providers.openai` को स्पष्ट रूप से कॉन्फ़िगर करें।
यदि वह कस्टम छवि एंडपॉइंट किसी विश्वसनीय LAN/निजी पते पर है, तो
`browser.ssrfPolicy.dangerouslyAllowPrivateNetwork: true` भी सेट करें; इस ऑप्ट-इन के बिना OpenClaw
निजी/आंतरिक OpenAI-संगत छवि एंडपॉइंट अवरुद्ध रखता है।

निर्माण:

```
/tool image_generate model=openai/gpt-image-2 prompt="macOS पर OpenClaw के लिए एक परिष्कृत लॉन्च पोस्टर" size=3840x2160 count=1
```

पारदर्शी PNG बनाएँ:

```
/tool image_generate model=openai/gpt-image-1.5 prompt="पारदर्शी पृष्ठभूमि पर एक साधारण लाल वृत्त स्टिकर" outputFormat=png background=transparent
```

संपादित करें:

```
/tool image_generate model=openai/gpt-image-2 prompt="वस्तु का आकार बनाए रखें, सामग्री को पारदर्शी काँच में बदलें" image=/path/to/reference.png size=1024x1536
```

## वीडियो निर्माण

बंडल किया गया `openai` plugin
`video_generate` टूल के माध्यम से वीडियो निर्माण पंजीकृत करता है।

| क्षमता             | मान                                                                                |
| ------------------ | ---------------------------------------------------------------------------------- |
| डिफ़ॉल्ट मॉडल      | `openai/sora-2`                                                                    |
| मोड                | टेक्स्ट-से-वीडियो, छवि-से-वीडियो, एकल-वीडियो संपादन                               |
| संदर्भ इनपुट       | 1 छवि या 1 वीडियो                                                                  |
| आकार ओवरराइड       | टेक्स्ट-से-वीडियो और छवि-से-वीडियो के लिए समर्थित                                  |
| आस्पेक्ट अनुपात    | निकटतम समर्थित आकार में परिवर्तित, अपरिवर्तित रूप में अग्रेषित नहीं                 |
| अन्य ओवरराइड       | `resolution`, `audio`, `watermark` असमर्थित हैं और टूल चेतावनी के साथ हटा दिए जाते हैं |

OpenAI इमेज-टू-वीडियो अनुरोध किसी इमेज के साथ `POST /v1/videos`
`input_reference` का उपयोग करते हैं। एकल-वीडियो संपादन `POST /v1/videos/edits` का उपयोग करते हैं, जिसमें
अपलोड किया गया वीडियो `video` फ़ील्ड में होता है।

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: { primary: "openai/sora-2" },
    },
  },
}
```

<Note>
साझा टूल पैरामीटर, प्रोवाइडर चयन और फ़ेलओवर व्यवहार के लिए [वीडियो जनरेशन](/hi/tools/video-generation) देखें।

OpenAI प्रोवाइडर `supportsSize` घोषित करता है, लेकिन `supportsAspectRatio` या
`supportsResolution` नहीं। OpenClaw की साझा नॉर्मलाइज़ेशन परत अनुरोधित
`aspectRatio` को अनुरोध के प्रोवाइडर तक पहुँचने से पहले सबसे निकट मेल खाने वाले OpenAI `size` में बदल देती है, इसलिए आस्पेक्ट-अनुपात अनुरोध सामान्यतः फिर भी काम करते हैं।
`resolution` के लिए कोई आकार फ़ॉलबैक नहीं है और इसे हटा दिया जाता है तथा कॉलर को
`Ignored unsupported overrides for openai/<model>: resolution=<value>` के रूप में दिखाया जाता है।
</Note>

## GPT-5 प्रॉम्प्ट योगदान

OpenClaw, `openai` प्रोवाइडर पर GPT-5-फ़ैमिली मॉडल के लिए एक साझा GPT-5 प्रॉम्प्ट योगदान जोड़ता है
(इसमें पुराने, सुधार-पूर्व Codex रेफ़रेंस भी शामिल हैं, जो
`openai/*` में नॉर्मलाइज़ होते हैं)। GPT-5-फ़ैमिली मॉडल आईडी भी प्रदान करने वाले अन्य प्रोवाइडर,
जैसे OpenRouter या opencode रूट, यह ओवरले प्राप्त नहीं करते; इसे
केवल मॉडल आईडी पर नहीं, बल्कि प्रोवाइडर आईडी `openai` पर नियंत्रित किया जाता है। पुराने GPT-4.x मॉडल इसे कभी
प्राप्त नहीं करते।

नेटिव Codex ऐप-सर्वर हार्नेस को डेवलपर निर्देशों के माध्यम से पर्सोना/टूल-
अनुशासन व्यवहार अनुबंध या मैत्रीपूर्ण इंटरैक्शन-शैली ओवरले प्राप्त नहीं होता;
नेटिव Codex, Codex के स्वामित्व वाला आधार, मॉडल और प्रोजेक्ट-दस्तावेज़ व्यवहार बनाए रखता है, और OpenClaw
नेटिव थ्रेड के लिए Codex का अंतर्निहित व्यक्तित्व अक्षम कर देता है, ताकि
एजेंट वर्कस्पेस की व्यक्तित्व फ़ाइलें प्रामाणिक रहें।
OpenClaw नेटिव Codex थ्रेड में केवल रनटाइम संदर्भ का योगदान करता है: चैनल
डिलीवरी, OpenClaw डायनेमिक टूल, ACP डेलिगेशन, वर्कस्पेस संदर्भ और
OpenClaw Skills। इसी योगदान का Heartbeat-मार्गदर्शन टेक्स्ट
एकमात्र अपवाद है: नेटिव Codex Heartbeat टर्न इसे प्राप्त करते हैं, जिसे
साझा प्रॉम्प्ट-योगदान हुक के बजाय समर्पित
सहयोग निर्देशों के रूप में इंजेक्ट किया जाता है।

GPT-5 योगदान मेल खाने वाले OpenClaw-संयोजित प्रॉम्प्ट में पर्सोना
स्थायित्व, निष्पादन सुरक्षा, टूल अनुशासन, आउटपुट स्वरूप, पूर्णता
जाँच और सत्यापन के लिए टैग किया गया व्यवहार अनुबंध जोड़ता है। चैनल-
विशिष्ट उत्तर और मौन-संदेश व्यवहार साझा OpenClaw सिस्टम
प्रॉम्प्ट और आउटबाउंड डिलीवरी नीति में रहता है। मैत्रीपूर्ण इंटरैक्शन-शैली परत
अलग और कॉन्फ़िगर करने योग्य है।

| मान                  | प्रभाव                                      |
| ---------------------- | ------------------------------------------- |
| `"friendly"` (डिफ़ॉल्ट) | मैत्रीपूर्ण इंटरैक्शन-शैली परत सक्षम करें |
| `"on"`                 | `"friendly"` का उपनाम                      |
| `"off"`                | केवल मैत्रीपूर्ण शैली परत अक्षम करें       |

<Tabs>
  <Tab title="कॉन्फ़िगरेशन">
    ```json5
    {
      agents: {
        defaults: {
          promptOverlays: {
            gpt5: { personality: "friendly" },
          },
        },
      },
    }
    ```
  </Tab>
  <Tab title="CLI">
    ```bash
    openclaw config set agents.defaults.promptOverlays.gpt5.personality off
    ```
  </Tab>
</Tabs>

<Tip>
रनटाइम पर मान केस-असंवेदी होते हैं, इसलिए `"Off"` और `"off"` दोनों
मैत्रीपूर्ण शैली परत को अक्षम करते हैं।
</Tip>

<Note>
जब साझा
`agents.defaults.promptOverlays.gpt5.personality` सेटिंग अनसेट हो, तब पुराने `plugins.entries.openai.config.personality` को अभी भी
संगतता फ़ॉलबैक के रूप में पढ़ा जाता है।
</Note>

## Azure OpenAI एंडपॉइंट

बंडल किया गया `openai` प्रदाता, बेस URL को ओवरराइड करके इमेज
जनरेशन के लिए Azure OpenAI संसाधन को लक्षित कर सकता है। इमेज-जनरेशन पथ पर, OpenClaw
`models.providers.openai.baseUrl` पर Azure होस्टनाम का पता लगाता है और
अपने-आप Azure के अनुरोध प्रारूप पर स्विच करता है।

<Note>
रीयलटाइम वॉइस एक अलग कॉन्फ़िगरेशन पथ
(`plugins.entries.voice-call.config.realtime.providers.openai.azureEndpoint`)
का उपयोग करती है और `models.providers.openai.baseUrl` से प्रभावित नहीं होती। इसकी Azure
सेटिंग के लिए [वॉइस और वाक्](#voice-and-speech) के अंतर्गत **रीयलटाइम
वॉइस** अकॉर्डियन देखें।
</Note>

Azure OpenAI का उपयोग तब करें, जब:

- आपके पास पहले से Azure OpenAI सदस्यता, कोटा या एंटरप्राइज़
  अनुबंध है
- आपको क्षेत्रीय डेटा निवास या Azure द्वारा उपलब्ध कराए गए अनुपालन नियंत्रण चाहिए
- आप ट्रैफ़िक को किसी मौजूदा Azure टेनेंसी के भीतर रखना चाहते हैं

### कॉन्फ़िगरेशन

बंडल किए गए `openai` प्रदाता के माध्यम से Azure इमेज जनरेशन के लिए,
`models.providers.openai.baseUrl` को अपने Azure संसाधन पर इंगित करें और `apiKey` को
Azure OpenAI कुंजी पर सेट करें (OpenAI Platform कुंजी पर नहीं):

```json5
{
  models: {
    providers: {
      openai: {
        baseUrl: "https://<your-resource>.openai.azure.com",
        apiKey: "<azure-openai-api-key>",
      },
    },
  },
}
```

OpenClaw, Azure इमेज-जनरेशन रूट के लिए इन Azure होस्ट प्रत्ययों को
पहचानता है:

- `*.openai.azure.com`
- `*.services.ai.azure.com`
- `*.cognitiveservices.azure.com`

किसी पहचाने गए Azure होस्ट पर इमेज-जनरेशन अनुरोधों के लिए, OpenClaw:

- `Authorization: Bearer` के बजाय `api-key` हेडर भेजता है
- डिप्लॉयमेंट-स्कोप किए गए पथों (`/openai/deployments/{deployment}/...`) का उपयोग करता है
- प्रत्येक अनुरोध में `?api-version=...` जोड़ता है
- Azure इमेज-जनरेशन कॉल के लिए 600s की डिफ़ॉल्ट अनुरोध समय-सीमा का उपयोग करता है।
  प्रति-कॉल `timeoutMs` मान फिर भी इस डिफ़ॉल्ट को ओवरराइड करते हैं।

अन्य बेस URL (सार्वजनिक OpenAI, OpenAI-संगत प्रॉक्सी) मानक
OpenAI इमेज अनुरोध प्रारूप बनाए रखते हैं।

<Note>
`openai` प्रदाता के इमेज-जनरेशन पथ के लिए Azure रूटिंग को
OpenClaw 2026.4.22 या बाद का संस्करण चाहिए। पुराने संस्करण किसी भी कस्टम
`openai.baseUrl` को सार्वजनिक OpenAI एंडपॉइंट की तरह मानते हैं और Azure इमेज
डिप्लॉयमेंट के साथ विफल हो जाते हैं।
</Note>

### API संस्करण

Azure इमेज-जनरेशन पथ के लिए किसी विशिष्ट Azure प्रीव्यू या GA संस्करण को
स्थिर करने हेतु `AZURE_OPENAI_API_VERSION` सेट करें:

```bash
export AZURE_OPENAI_API_VERSION="2024-12-01-preview"
```

वेरिएबल सेट न होने पर डिफ़ॉल्ट `2024-12-01-preview` है।

### मॉडल नाम डिप्लॉयमेंट नाम हैं

Azure OpenAI मॉडल को डिप्लॉयमेंट से संबद्ध करता है। बंडल किए गए
`openai` प्रदाता से रूट किए गए Azure इमेज-जनरेशन अनुरोधों के लिए, OpenClaw में
`model` फ़ील्ड वह **Azure डिप्लॉयमेंट नाम** होना चाहिए जिसे आपने Azure portal में
कॉन्फ़िगर किया है, न कि सार्वजनिक OpenAI मॉडल आईडी।

यदि आप `gpt-image-2` प्रदान करने वाला `gpt-image-2-prod` नामक डिप्लॉयमेंट बनाते हैं:

```
/tool image_generate model=openai/gpt-image-2-prod prompt="एक साफ़-सुथरा पोस्टर" size=1024x1024 count=1
```

यही डिप्लॉयमेंट-नाम नियम बंडल किए गए `openai` प्रदाता से रूट की गई
हर इमेज-जनरेशन कॉल पर लागू होता है।

### क्षेत्रीय उपलब्धता

Azure इमेज जनरेशन वर्तमान में केवल कुछ क्षेत्रों में उपलब्ध है
(उदाहरण के लिए `eastus2`, `swedencentral`, `polandcentral`, `westus3`,
`uaenorth`)। डिप्लॉयमेंट बनाने से पहले Microsoft की वर्तमान क्षेत्र सूची देखें
और पुष्टि करें कि विशिष्ट मॉडल आपके क्षेत्र में उपलब्ध है।

### पैरामीटर में अंतर

Azure OpenAI और सार्वजनिक OpenAI हमेशा समान इमेज पैरामीटर स्वीकार नहीं करते।
Azure उन विकल्पों को अस्वीकार कर सकता है जिन्हें सार्वजनिक OpenAI स्वीकार करता है (उदाहरण के लिए
`gpt-image-2` पर कुछ `background` मान), या उन्हें केवल विशिष्ट मॉडल
संस्करणों पर उपलब्ध करा सकता है। ये अंतर Azure और अंतर्निहित मॉडल से आते हैं,
OpenClaw से नहीं। यदि कोई Azure अनुरोध सत्यापन त्रुटि के साथ विफल होता है, तो
Azure portal में अपने विशिष्ट डिप्लॉयमेंट और API संस्करण द्वारा समर्थित
पैरामीटर सेट जाँचें।

<Note>
Azure OpenAI मूल ट्रांसपोर्ट और संगतता व्यवहार का उपयोग करता है, लेकिन उसे
OpenClaw के छिपे हुए एट्रिब्यूशन हेडर प्राप्त नहीं होते—[उन्नत कॉन्फ़िगरेशन](#advanced-configuration)
के अंतर्गत **मूल बनाम OpenAI-संगत
रूट** अकॉर्डियन देखें।

Azure पर चैट या Responses ट्रैफ़िक (इमेज जनरेशन से परे) के लिए,
ऑनबोर्डिंग प्रवाह या समर्पित Azure प्रदाता कॉन्फ़िगरेशन का उपयोग करें; केवल `openai.baseUrl`
Azure API/प्रमाणीकरण प्रारूप नहीं अपनाता। एक अलग
`azure-openai-responses/*` प्रदाता उपलब्ध है; नीचे Server-side compaction
अकॉर्डियन देखें।
</Note>

## उन्नत कॉन्फ़िगरेशन

नीचे दिए गए प्रति-मॉडल `params` उदाहरण OpenClaw के एम्बेडेड प्रदाता
अनुरोध को आकार देते हैं। इन्हें कॉन्फ़िगर करना लिखित अनुरोध व्यवहार है, इसलिए अन्यथा योग्य
`auto` रूट, Codex को निहित रूप से चुनने के बजाय OpenClaw पर ही रहता है। मूल
Codex app-server हार्नेस अपने ट्रांसपोर्ट और अनुरोध सेटिंग का स्वामी है; प्रभावी रूट के
Codex-संगत घोषित न होने पर स्पष्ट `agentRuntime.id: "codex"` बंद अवस्था में विफल होता है।

<AccordionGroup>
  <Accordion title="ट्रांसपोर्ट (WebSocket बनाम SSE)">
    OpenClaw, `openai/*` के लिए SSE फ़ॉलबैक (`"auto"`) के साथ WebSocket-प्रथम का उपयोग करता है।

    `"auto"` मोड में, OpenClaw:
    - SSE पर फ़ॉलबैक करने से पहले एक शुरुआती WebSocket विफलता पर पुनः प्रयास करता है
    - विफलता के बाद, WebSocket को 60 सेकंड के लिए अवनत चिह्नित करता है और
      कूल-डाउन के दौरान SSE का उपयोग करता है
    - पुनः प्रयासों और पुनः कनेक्शन के लिए स्थिर सत्र और टर्न पहचान हेडर
      संलग्न करता है
    - ट्रांसपोर्ट प्रकारों में उपयोग काउंटर (`input_tokens` / `prompt_tokens`) को
      सामान्यीकृत करता है

    | मान                | व्यवहार                          |
    | ---------------------- | ------------------------------------ |
    | `"auto"` (डिफ़ॉल्ट)   | पहले WebSocket, SSE फ़ॉलबैक     |
    | `"sse"`              | केवल SSE बाध्य करें                    |
    | `"websocket"`        | केवल WebSocket बाध्य करें              |

    ```json5
    {
      agents: {
        defaults: {
          models: {
            "openai/gpt-5.5": {
              params: { transport: "auto" },
            },
          },
        },
      },
    }
    ```

    संबंधित OpenAI दस्तावेज़:
    - [WebSocket के साथ Realtime API](https://platform.openai.com/docs/guides/realtime-websocket)
    - [स्ट्रीमिंग API प्रतिक्रियाएँ (SSE)](https://platform.openai.com/docs/guides/streaming-responses)

  </Accordion>

  <Accordion title="तेज़ मोड">
    OpenClaw, `openai/*` के लिए साझा तेज़-मोड टॉगल उपलब्ध कराता है:

    - **चैट/UI:** `/fast status|auto|on|off`
    - **कॉन्फ़िगरेशन:** `agents.defaults.models["<provider>/<model>"].params.fastMode`

    सक्षम होने पर, OpenClaw तेज़ मोड को OpenAI प्राथमिकता प्रोसेसिंग
    (`service_tier = "priority"`) से मैप करता है। मौजूदा `service_tier` मान
    संरक्षित रहते हैं और तेज़ मोड `reasoning` या
    `text.verbosity` को फिर से नहीं लिखता। `fastMode: "auto"` स्वतः कटऑफ़ तक नई मॉडल कॉल
    तेज़ी से शुरू करता है, फिर बाद की पुनः प्रयास, फ़ॉलबैक, टूल-परिणाम या
    निरंतरता कॉल तेज़ मोड के बिना शुरू करता है। कटऑफ़ का डिफ़ॉल्ट 60 सेकंड है;
    इसे बदलने के लिए सक्रिय मॉडल पर `params.fastAutoOnSeconds` सेट करें।

    ```json5
    {
      agents: {
        defaults: {
          models: {
            "openai/gpt-5.5": { params: { fastMode: "auto", fastAutoOnSeconds: 30 } },
          },
        },
      },
    }
    ```

    <Note>
    सत्र ओवरराइड को कॉन्फ़िगरेशन पर प्राथमिकता मिलती है। Sessions UI में
    सत्र ओवरराइड हटाने पर सत्र कॉन्फ़िगर किए गए डिफ़ॉल्ट पर लौट जाता है।
    </Note>

  </Accordion>

  <Accordion title="प्राथमिकता प्रोसेसिंग (service_tier)">
    OpenAI का API `service_tier` के माध्यम से प्राथमिकता प्रोसेसिंग उपलब्ध कराता है।
    इसे OpenClaw में प्रति मॉडल सेट करें:

    ```json5
    {
      agents: {
        defaults: {
          models: {
            "openai/gpt-5.5": { params: { serviceTier: "priority" } },
          },
        },
      },
    }
    ```

    समर्थित मान: `auto`, `default`, `flex`, `priority`।

    <Warning>
    `serviceTier` केवल मूल OpenAI एंडपॉइंट
    (`api.openai.com`) और मूल Codex एंडपॉइंट (`chatgpt.com/backend-api`) को अग्रेषित किया जाता है।
    यदि आप किसी भी प्रदाता को प्रॉक्सी के माध्यम से रूट करते हैं, तो OpenClaw
    `service_tier` को अपरिवर्तित छोड़ देता है।
    </Warning>

  </Accordion>

  <Accordion title="सर्वर-साइड Compaction (Responses API)">
    प्रत्यक्ष OpenAI Responses मॉडल (`api.openai.com` पर `openai/*`) के लिए,
    OpenAI Plugin का OpenClaw स्ट्रीम रैपर सर्वर-साइड
    Compaction को अपने-आप सक्षम करता है:

    - `store: true` को बाध्य करता है (जब तक मॉडल संगतता `supportsStore: false` सेट न करे)
    - `context_management: [{ type: "compaction", compact_threshold: ... }]` अंतःक्षेपित करता है
    - डिफ़ॉल्ट `compact_threshold`: `contextWindow` का 70% (या अनुपलब्ध होने पर
      `80000`)

    यह अंतर्निहित OpenClaw रनटाइम पथ और एम्बेडेड रन द्वारा उपयोग किए जाने वाले OpenAI प्रदाता
    हुक पर लागू होता है। मूल Codex app-server हार्नेस Codex के माध्यम से
    अपने संदर्भ का प्रबंधन करता है और इस सेटिंग से प्रभावित नहीं होता।

  <Accordion title="कठोर-एजेंटिक GPT मोड">
    OpenClaw के एम्बेडेड रनटाइम से चलाए गए `openai` प्रदाता के GPT-5-परिवार मॉडल के लिए,
    OpenClaw पहले से `strict-agentic` नामक अधिक कठोर निष्पादन अनुबंध को
    डिफ़ॉल्ट बनाता है। जब भी समाधान किया गया प्रदाता
    `openai` हो और मॉडल आईडी GPT-5 परिवार से मेल खाए, यह अपने-आप सक्रिय हो जाता है,
    जब तक कॉन्फ़िगरेशन स्पष्ट रूप से इससे बाहर न निकले:

    ```json5
    {
      agents: {
        defaults: {
          embeddedAgent: { executionContract: "default" },
        },
      },
    }
    ```

    `"strict-agentic"` को स्पष्ट रूप से सेट करने का समर्थित लेन पर कोई प्रभाव नहीं पड़ता (यह
    पहले से ही डिफ़ॉल्ट है) और असमर्थित प्रदाता/मॉडल युग्मों पर यह निष्क्रिय रहता है।

    `strict-agentic` सक्रिय होने पर, OpenClaw:
    - पर्याप्त कार्य के लिए `update_plan` को स्वतः सक्षम करता है
    - संरचनात्मक रूप से खाली या केवल तर्क वाले टर्न को दृश्यमान उत्तर की
      निरंतरता के साथ पुनः प्रयास करता है
    - चयनित हार्नेस द्वारा उपलब्ध कराए जाने पर स्पष्ट हार्नेस योजना इवेंट का
      उपयोग करता है

    OpenClaw यह तय करने के लिए सहायक के गद्य का वर्गीकरण नहीं करता कि कोई टर्न
    योजना, प्रगति अपडेट या अंतिम उत्तर है।

    <Note>
    यह अनुबंध पूरी तरह OpenClaw के एम्बेडेड एजेंट रनर में स्थित है। यह
    मूल Codex ऐप-सर्वर हार्नेस पर लागू नहीं होता, जो अपने टर्न और योजना व्यवहार को
    स्वयं प्रबंधित करता है; मूल Codex रन के लिए निष्पादन-अनुबंध सेटिंग की तुलना में
    हार्नेस का चयन अधिक महत्वपूर्ण है।
    </Note>

  </Accordion>

  <Accordion title="मूल बनाम OpenAI-संगत रूट">
    OpenClaw प्रत्यक्ष OpenAI, Codex और Azure OpenAI एंडपॉइंट को
    सामान्य OpenAI-संगत `/v1` प्रॉक्सी से अलग ढंग से संभालता है:

    **मूल रूट** (`openai/*`, Azure OpenAI):
    - `reasoning: { effort: "none" }` को केवल उन मॉडल के लिए रखता है जो
      OpenAI `none` प्रयास का समर्थन करते हैं
    - उन मॉडल या प्रॉक्सी के लिए अक्षम तर्क को छोड़ देता है जो
      `reasoning.effort: "none"` को अस्वीकार करते हैं
    - टूल स्कीमा को डिफ़ॉल्ट रूप से सख्त मोड में रखता है
    - केवल सत्यापित मूल होस्ट पर छिपे हुए एट्रिब्यूशन हेडर संलग्न करता है (Azure
      OpenAI को ये हेडर नहीं मिलते, भले ही वह एक मूल रूट है)
    - केवल OpenAI के अनुरोध आकार-निर्धारण (`service_tier`, `store`,
      तर्क-संगतता, प्रॉम्प्ट-कैश संकेत) को बनाए रखता है

    **प्रॉक्सी/संगत रूट:**
    - अधिक शिथिल संगतता व्यवहार का उपयोग करते हैं
    - गैर-मूल `openai-completions` पेलोड से Completions `store` को हटाते हैं
    - OpenAI-संगत Completions प्रॉक्सी के लिए उन्नत `params.extra_body`/`params.extraBody` पास-थ्रू JSON
      स्वीकार करते हैं
    - vLLM जैसे OpenAI-संगत Completions
      प्रॉक्सी के लिए `params.chat_template_kwargs` स्वीकार करते हैं
    - सख्त टूल स्कीमा या केवल-मूल हेडर लागू नहीं करते हैं

  </Accordion>
</AccordionGroup>
