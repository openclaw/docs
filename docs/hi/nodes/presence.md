कंप्यूटर की सक्रिय उपस्थिति Gateway को बताती है कि किस कनेक्टेड macOS Node को
सबसे हाल में माउस या कीबोर्ड से भौतिक इनपुट मिला था। OpenClaw उस संकेत का उपयोग
एक Mac को `active` के रूप में चिह्नित करने, एजेंट को सक्रिय Node का स्थिर संकेत देने, और
Node कनेक्शन अलर्ट उस कंप्यूटर पर भेजने के लिए करता है जहाँ आपके उपस्थित होने की सर्वाधिक संभावना है।

यह [सिस्टम उपस्थिति](/hi/concepts/presence) से अलग है, जो Gateway क्लाइंट की लाइव
सूची है, और स्थायी `node.presence.alive` बीकन से भी अलग है, जो
यह दर्ज करते हैं कि किसी मोबाइल Node को कनेक्टेड माने बिना वह पिछली बार कब सक्रिय हुआ था।

## आवश्यकताएँ

- OpenClaw macOS ऐप युग्मित है और Node मोड में कनेक्टेड है।
- **Settings -> Permissions -> Active computer detection** सक्षम है। यह डिफ़ॉल्ट रूप से बंद रहता है।
- हस्ताक्षरित OpenClaw ऐप को **Accessibility** अनुमति दी गई है।
- कनेक्शन अलर्ट के लिए **Notifications** अनुमति भी दी गई है और
  Mac Node `system.notify` उपलब्ध कराता है।

गतिविधि रिपोर्टिंग वर्तमान में नेटिव macOS Node द्वारा कार्यान्वित की गई है। iOS,
Android, watchOS और हेडलेस Node होस्ट कनेक्शन या बैकग्राउंड में अंतिम बार देखे जाने की
स्थिति रिपोर्ट कर सकते हैं, लेकिन वे सक्रिय कंप्यूटर के निर्धारण के लिए प्रतिस्पर्धा नहीं करते।

## सक्रिय कंप्यूटर जाँचें

1. macOS ऐप में **Settings -> Permissions** खोलें,
   **Active computer detection** सक्षम करें और macOS System Settings में **Accessibility** प्रदान करें।
2. पुष्टि करें कि Mac Node कनेक्टेड है:

   ```bash
   openclaw nodes status --connected
   ```

3. उस Mac पर माउस चलाएँ या कोई कुंजी दबाएँ, फिर चलाएँ:

   ```bash
   openclaw nodes status
   openclaw nodes describe --node <node-id-or-name>
   ```

सबसे नवीन योग्य Mac को `active` के रूप में चिह्नित किया जाता है। स्थिति आउटपुट उसके अंतिम इनपुट
के बाद बीता समय दिखाता है; `describe`, `active`, `lastActiveAtMs` और `presenceUpdatedAtMs` उपलब्ध कराता है।
गतिविधि को जानबूझकर समेकित किया जाता है, इसलिए हाल की रिपोर्ट के बाद हुए दूसरे इनपुट को
डिस्प्ले में दिखने में लगभग 15 सेकंड तक लग सकते हैं।

## गतिविधि उपस्थिति में कैसे बदलती है

macOS रिपोर्टर प्रत्येक दो सेकंड में HID सिस्टम की निष्क्रियता घड़ी का नमूना लेता है। यह
Node कनेक्शन के तैयार होने पर एक बार रिपोर्ट करता है, फिर नई भौतिक
गतिविधि को प्रत्येक 15 सेकंड में अधिकतम एक बार रिपोर्ट करता है। निष्क्रिय रहने पर यह
प्रत्येक तीन मिनट में एक कीपअलाइव भेजता है। निष्क्रियता अवधि 30 दिनों तक सीमित है, ताकि बहुत पुराना नमूना
आगे खिसककर गलती से सबसे नया कंप्यूटर न बन जाए।

**Active computer detection** अक्षम करने से नमूना लेना बंद हो जाता है और वर्तमान Node
कनेक्शन पर एक प्रमाणित क्लियर इवेंट भेजा जाता है। Gateway तुरंत
उस Mac के संरक्षित गतिविधि टाइमस्टैम्प हटा देता है और सक्रिय कंप्यूटर की पुनर्गणना करता है;
अन्य Node क्षमताएँ और जारी कार्य कनेक्टेड रहते हैं। यदि कनेक्टेड
Gateway इस क्लियर कार्रवाई से पुराना है, तो Mac Node एक बार फिर कनेक्ट होता है ताकि डिस्कनेक्ट
क्लीनअप संरक्षित गतिविधि को हटा सके।

Gateway गतिविधि केवल तभी स्वीकार करता है जब ये सभी बातें सत्य हों:

- इवेंट उस Node ID के वर्तमान प्रमाणित कनेक्शन से संबंधित हो;
- Node के पास प्रभावी `accessibility: true` अनुमति हो;
- पेलोड में सीमित पूर्णांक `idleSeconds` मान हो।

Gateway `lastActiveAtMs` प्राप्त करने के लिए अपने अवलोकन समय से
`idleSeconds` घटाता है। यह कभी भी Node द्वारा दिए गए वॉल-क्लॉक टाइमस्टैम्प पर भरोसा नहीं करता। कनेक्टेड
योग्य Mac में सबसे नया `lastActiveAtMs` विजयी होता है; बराबरी होने पर सबसे
हालिया उपस्थिति अपडेट का उपयोग किया जाता है।

उपस्थिति प्रोसेस-लोकल और कनेक्शन-बाउंड होती है। वर्तमान
सेशन को डिस्कनेक्ट करना, उसी Node ID वाले दूसरे सेशन से उसे बदलना, या
Accessibility निरस्त करना उस Node की गतिविधि स्थिति मिटा देता है और सक्रिय Mac की पुनर्गणना करता है।

## गोपनीयता और मॉडल संदर्भ

गतिविधि साझा करना डिफ़ॉल्ट रूप से बंद रहता है और UI ऑटोमेशन के लिए उपयोग किए जाने वाले
Accessibility अनुदान से अलग है। OpenClaw इनपुट की सामग्री नहीं, बल्कि निष्क्रियता अवधि भेजता है। यह कुंजी मान,
माउस निर्देशांक, एप्लिकेशन नाम, विंडो शीर्षक या रॉ इनपुट इवेंट नहीं भेजता। macOS
रिपोर्टर हार्डवेयर HID स्थिति पढ़ता है, इसलिए सिंथेटिक कंप्यूटर-नियंत्रण
इवेंट किसी स्वचालित Mac को वह कंप्यूटर नहीं दिखाते जिसका आपने भौतिक रूप से उपयोग किया था।

निरंतर गतिविधि मॉडल को दिखाई देने वाले सिस्टम इवेंट नहीं बनाती। डायनेमिक
रनटाइम पंक्ति में केवल प्रमाणित Node ID होता है:

```text
active_node=<node-id>
```

प्रॉम्प्ट इंजेक्शन और कैश में अनावश्यक बदलाव से बचने के लिए सटीक टाइमस्टैम्प और Node-नियंत्रित डिस्प्ले नाम
प्रॉम्प्ट से बाहर रहते हैं। जब एजेंट को वर्तमान विवरण चाहिए,
तो `nodes` टूल इसके बजाय `node.list` या `node.describe` पढ़ सकता है।

## कनेक्शन अलर्ट कैसे रूट किए जाते हैं

अनुमोदन के बाद Node द्वारा अपना पहला सफल Gateway हैंडशेक पूरा करने पर,
OpenClaw 750 मिलीसेकंड प्रतीक्षा करता है ताकि कनेक्ट हो रहा Mac अपना पहला
गतिविधि नमूना सबमिट कर सके। फिर यह सबसे नवीन गतिविधि वाले कनेक्टेड और सूचना-समर्थ Mac को आज़माता है।

- यदि प्राथमिक डिलीवरी सफल होती है, तो किसी अन्य Mac को अलर्ट नहीं मिलता।
- यदि कोई सक्रिय Mac उपलब्ध नहीं है या प्राथमिक डिलीवरी विफल होती है, तो OpenClaw पाँच
  सेकंड प्रतीक्षा करता है और `system.notify` उपलब्ध कराने वाले प्रत्येक शेष कनेक्टेड Mac को आज़माता है।
- बाद में होने वाले पुनः कनेक्शन मौन रहते हैं। Gateway सफल कनेक्शन को
  युग्मन मेटाडेटा में दर्ज करता है, इसलिए Gateway पुनरारंभ होने पर पहले
  कनेक्टेड रहे प्रत्येक Node के अलर्ट दोबारा नहीं चलाए जाते।

अलर्ट प्रमाणित Node पहचान से बंधे होते हैं। उसी Node का प्रतिस्थापन सेशन
उसके लंबित प्रथम-कनेक्शन अलर्ट को अपने अधीन ले लेता है; यदि डिलीवरी के समय वह Node
कनेक्टेड नहीं रहता, तो अलर्ट रद्द कर दिया जाता है।
