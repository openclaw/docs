सक्रिय कंप्यूटर उपस्थिति Gateway को बताती है कि किस कनेक्टेड macOS Node को
सबसे हाल का भौतिक माउस या कीबोर्ड इनपुट मिला। OpenClaw उस संकेत का उपयोग
एक Mac को `active` के रूप में चिह्नित करने, एजेंट को स्थिर सक्रिय-Node संकेत देने और
Node कनेक्शन अलर्ट उस कंप्यूटर पर भेजने के लिए करता है जहाँ आपके मौजूद होने की सबसे अधिक संभावना है।

यह [सिस्टम उपस्थिति](/hi/concepts/presence) से अलग है, जो Gateway क्लाइंट की लाइव
सूची है, और स्थायी `node.presence.alive` बीकन से भी अलग है, जो यह दर्ज करते हैं कि
कोई मोबाइल Node पिछली बार कब सक्रिय हुआ था, लेकिन उसे कनेक्टेड नहीं मानते।

## आवश्यकताएँ

- OpenClaw macOS ऐप युग्मित है और Node मोड में कनेक्टेड है।
- हस्ताक्षरित OpenClaw ऐप को **Accessibility** अनुमति दी गई है।
- कनेक्शन अलर्ट के लिए **Notifications** अनुमति भी दी गई है और
  Mac Node `system.notify` उपलब्ध कराता है।

गतिविधि रिपोर्टिंग वर्तमान में मूल macOS Node द्वारा कार्यान्वित है। iOS,
Android, watchOS और हेडलेस Node होस्ट कनेक्शन या पृष्ठभूमि में अंतिम बार देखे जाने की
स्थिति रिपोर्ट कर सकते हैं, लेकिन वे सक्रिय-कंप्यूटर पदनाम के लिए प्रतिस्पर्धा नहीं करते।

## सक्रिय कंप्यूटर जाँचें

1. macOS ऐप में **Settings -> Permissions** खोलें और macOS System Settings में
   **Accessibility** प्रदान करें।
2. पुष्टि करें कि Mac Node कनेक्टेड है:

   ```bash
   openclaw nodes status --connected
   ```

3. उस Mac पर माउस चलाएँ या कोई कुंजी दबाएँ, फिर चलाएँ:

   ```bash
   openclaw nodes status
   openclaw nodes describe --node <node-id-or-name>
   ```

सबसे हाल का पात्र Mac `active` के रूप में चिह्नित होता है। स्थिति आउटपुट उसके अंतिम इनपुट
की अवधि दिखाता है; `describe`, `active`, `lastActiveAtMs` और `presenceUpdatedAtMs` उपलब्ध कराता है।
गतिविधि को जानबूझकर समेकित किया जाता है, इसलिए हाल की रिपोर्ट के बाद किसी अन्य इनपुट को प्रदर्शित होने में
लगभग 15 सेकंड तक लग सकते हैं।

## गतिविधि उपस्थिति कैसे बनती है

macOS रिपोर्टर प्रत्येक दो सेकंड में HID सिस्टम की निष्क्रियता घड़ी का नमूना लेता है। वह
Node कनेक्शन तैयार होने पर एक बार रिपोर्ट करता है, फिर नई भौतिक
गतिविधि को प्रत्येक 15 सेकंड में अधिकतम एक बार रिपोर्ट करता है। निष्क्रिय रहते समय, वह
प्रत्येक तीन मिनट में एक कीपअलाइव भेजता है। निष्क्रियता अवधि 30 दिनों तक सीमित होती है, ताकि बहुत पुराना नमूना
आगे खिसककर गलत तरीके से सबसे नया कंप्यूटर न बन जाए।

Gateway गतिविधि केवल तभी स्वीकार करता है जब ये सभी बातें सत्य हों:

- ईवेंट उस Node id के वर्तमान प्रमाणीकृत कनेक्शन से संबंधित हो;
- Node के पास प्रभावी `accessibility: true` अनुमति हो;
- पेलोड में सीमाबद्ध पूर्णांक `idleSeconds` मान हो।

Gateway, `lastActiveAtMs` प्राप्त करने के लिए अपने अवलोकन समय से
`idleSeconds` घटाता है। वह कभी भी Node द्वारा दिए गए वॉल-क्लॉक टाइमस्टैम्प पर भरोसा नहीं करता। कनेक्टेड
पात्र Mac में, नवीनतम `lastActiveAtMs` विजयी होता है; बराबरी होने पर सबसे हाल का
उपस्थिति अपडेट उपयोग किया जाता है।

उपस्थिति प्रक्रिया-स्थानीय और कनेक्शन-बद्ध होती है। वर्तमान
सत्र को डिस्कनेक्ट करने, उसी Node id वाले किसी अन्य सत्र से उसे बदलने या
Accessibility निरस्त करने पर उस Node की गतिविधि स्थिति साफ हो जाती है और सक्रिय Mac की पुनर्गणना होती है।

## गोपनीयता और मॉडल संदर्भ

OpenClaw निष्क्रियता अवधि भेजता है, इनपुट सामग्री नहीं। वह कुंजी मान,
माउस निर्देशांक, ऐप्लिकेशन के नाम, विंडो शीर्षक या अपरिष्कृत इनपुट ईवेंट नहीं भेजता।
macOS रिपोर्टर हार्डवेयर HID स्थिति पढ़ता है, इसलिए कृत्रिम कंप्यूटर-नियंत्रण
ईवेंट के कारण कोई स्वचालित Mac ऐसा कंप्यूटर प्रतीत नहीं होता जिसे आपने भौतिक रूप से
उपयोग किया था।

निरंतर गतिविधि मॉडल के लिए सिस्टम ईवेंट नहीं बनाती। डायनेमिक
रनटाइम पंक्ति में केवल प्रमाणीकृत Node id होता है:

```text
active_node=<node-id>
```

प्रॉम्प्ट इंजेक्शन और कैश में बार-बार बदलाव से बचने के लिए सटीक टाइमस्टैम्प और Node-नियंत्रित प्रदर्शन नाम
प्रॉम्प्ट से बाहर रखे जाते हैं। जब एजेंट को वर्तमान विवरण चाहिए,
तो `nodes` टूल इसके बजाय `node.list` या `node.describe` पढ़ सकता है।

## कनेक्शन अलर्ट कैसे भेजे जाते हैं

अनुमोदन के बाद जब कोई Node अपना पहला सफल Gateway हैंडशेक पूरा कर लेता है,
OpenClaw 750 मिलीसेकंड प्रतीक्षा करता है, ताकि कनेक्ट हो रहा Mac अपना पहला
गतिविधि नमूना भेज सके। फिर वह सबसे हाल की गतिविधि वाले कनेक्टेड, सूचना-सक्षम Mac पर प्रयास करता है।

- यदि प्राथमिक डिलीवरी सफल होती है, तो किसी अन्य Mac को अलर्ट नहीं मिलता।
- यदि कोई सक्रिय Mac उपलब्ध नहीं है या प्राथमिक डिलीवरी विफल हो जाती है, तो OpenClaw पाँच
  सेकंड प्रतीक्षा करता है और `system.notify` उपलब्ध कराने वाले प्रत्येक शेष कनेक्टेड Mac पर प्रयास करता है।
- बाद के पुनः कनेक्शन मौन रहते हैं। Gateway सफल कनेक्शन को
  युग्मन मेटाडेटा में दर्ज करता है, इसलिए Gateway पुनः आरंभ होने पर पहले से
  कनेक्टेड प्रत्येक Node के लिए अलर्ट दोबारा नहीं भेजे जाते।

अलर्ट प्रमाणीकृत Node पहचान से बँधे होते हैं। उसी Node का प्रतिस्थापन सत्र
उसके लंबित प्रथम-कनेक्शन अलर्ट को अपने अधीन ले लेता है; यदि डिलीवरी के समय वह Node
अब कनेक्टेड नहीं है, तो अलर्ट रद्द कर दिया जाता है।
