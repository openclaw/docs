OpenClaw उत्तर पाइपलाइन चलने से पहले आने वाले मीडिया (इमेज/ऑडियो/वीडियो) का सारांश बना सकता है, ताकि कमांड पार्सिंग और रूटिंग रॉ बाइट्स के बजाय संक्षिप्त टेक्स्ट पर काम करें। समझने की सुविधा स्थानीय टूल या प्रोवाइडर कुंजियों का स्वतः पता लगाती है, या स्पष्ट मॉडल कॉन्फ़िगर किए जा सकते हैं। मूल मीडिया हमेशा की तरह मॉडल को दिया जाता है; समझने की प्रक्रिया विफल होने या अक्षम होने पर उत्तर प्रवाह बिना किसी बदलाव के जारी रहता है।

वेंडर plugins क्षमता मेटाडेटा पंजीकृत करते हैं (कौन-सा प्रोवाइडर किस मीडिया प्रकार का समर्थन करता है, डिफ़ॉल्ट मॉडल, प्राथमिकता)। OpenClaw कोर साझा `tools.media` कॉन्फ़िगरेशन, फ़ॉलबैक क्रम और उत्तर-पाइपलाइन एकीकरण का स्वामी है।

## यह कैसे काम करता है

  <Step title="अटैचमेंट एकत्र करें">

  </Step>
  <Step title="प्रत्येक क्षमता के अनुसार चुनें">

  </Step>
  <Step title="मॉडल चुनें">
    पहली योग्य मॉडल प्रविष्टि चुनें (आकार + क्षमता + उपलब्ध प्रमाणीकरण)।
  </Step>
  <Step title="विफलता पर फ़ॉलबैक करें">
    यदि किसी मॉडल में त्रुटि आती है, समय-सीमा समाप्त होती है या मीडिया `maxBytes` से अधिक है, तो अगली प्रविष्टि आज़माएँ।
  </Step>
  <Step title="सफलता पर लागू करें">
    `Body` एक `[Image]`, `[Audio]` या `[Video]` ब्लॉक बन जाता है। ऑडियो `{{Transcript}}` भी सेट करता है; कैप्शन टेक्स्ट मौजूद होने पर कमांड पार्सिंग उसका उपयोग करती है, अन्यथा ट्रांसक्रिप्ट का। कैप्शन ब्लॉक के भीतर `User text:` के रूप में सुरक्षित रखे जाते हैं।
  </Step>
</Steps>

## कॉन्फ़िगरेशन

`tools.media` में साझा मॉडल सूची और प्रत्येक क्षमता के लिए ओवरराइड होते हैं:

```json5
{
  tools: {
    media: {
      concurrency: 2, // समवर्ती क्षमता रन की अधिकतम संख्या (डिफ़ॉल्ट)
      models: [/* shared list, gate with capabilities */],
      image: {/* वैकल्पिक ओवरराइड */},
      audio: {
        /* वैकल्पिक ओवरराइड */
        echoTranscript: true,
        echoFormat: '📝 "{transcript}"',
      },
      video: {/* वैकल्पिक ओवरराइड */},
    },
  },
}
```

प्रत्येक क्षमता (`image`/`audio`/`video`) की कुंजियाँ:

| कुंजी                                             | प्रकार      | डिफ़ॉल्ट                                              | टिप्पणियाँ                                                                               |
| ----------------------------------------------- | --------- | ---------------------------------------------------- | ----------------------------------------------------------------------------------- |
| `enabled`                                       | `boolean` | स्वतः (`false` अक्षम करता है)                              | इस क्षमता के लिए स्वतः-पहचान बंद करने हेतु `false` सेट करें                             |
| `models`                                        | ऐरे     | कोई नहीं                                                 | साझा `tools.media.models` सूची से पहले वरीयता दी जाती है                               |
| `prompt`                                        | `string`  | `"Describe the {media}."` (+ maxChars मार्गदर्शन)      | डिफ़ॉल्ट रूप से केवल इमेज/वीडियो                                                         |
| `maxChars`                                      | `number`  | `500` (इमेज/वीडियो), सेट नहीं (ऑडियो)                   | मॉडल द्वारा अधिक लौटाए जाने पर आउटपुट छोटा कर दिया जाता है                                         |
| `maxBytes`                                      | `number`  | इमेज `10485760`, ऑडियो `20971520`, वीडियो `52428800` | अधिक आकार वाला मीडिया छोड़कर अगला मॉडल आज़माया जाता है                                             |
| `timeoutSeconds`                                | `number`  | `60` (इमेज/ऑडियो), `120` (वीडियो)                    |                                                                                     |
| `language`                                      | `string`  | सेट नहीं                                                | ऑडियो ट्रांसक्रिप्शन संकेत                                                            |
| `baseUrl`/`headers`/`providerOptions`/`request` | -         | -                                                    | प्रोवाइडर अनुरोध ओवरराइड; [टूल और कस्टम प्रोवाइडर](/hi/gateway/config-tools) देखें |
| `attachments`                                   | ऑब्जेक्ट    | `{ mode: "first", maxAttachments: 1 }`               | [अटैचमेंट नीति](#attachment-policy) देखें                                         |
| `scope`                                         | ऑब्जेक्ट    | सेट नहीं                                                | चैनल/chatType/keyPrefix के अनुसार गेट करें                                                  |
| `echoTranscript`                                | `boolean` | `false`                                              | केवल ऑडियो: एजेंट प्रोसेसिंग से पहले ट्रांसक्रिप्ट को चैट में वापस प्रतिध्वनित करें            |
| `echoFormat`                                    | `string`  | `'📝 "{transcript}"'`                                | केवल ऑडियो: `{transcript}` प्लेसहोल्डर                                              |

Deepgram-विशिष्ट विकल्प `providerOptions.deepgram` के अंतर्गत रखे जाते हैं (शीर्ष-स्तरीय `deepgram: { detectLanguage, punctuate, smartFormat }` फ़ील्ड अप्रचलित है, लेकिन अभी भी पढ़ा जाता है)।

### मॉडल प्रविष्टियाँ

प्रत्येक `models[]` प्रविष्टि एक **प्रोवाइडर** प्रविष्टि (डिफ़ॉल्ट) या **CLI** प्रविष्टि होती है:

<Tabs>
  <Tab title="प्रोवाइडर प्रविष्टि">
    ```json5
    {
      type: "provider", // छोड़े जाने पर डिफ़ॉल्ट
      provider: "openai",
      model: "gpt-5.6-sol",
      prompt: "इमेज का वर्णन <= 500 वर्णों में करें।",
      maxChars: 500,
      maxBytes: 10485760,
      timeoutSeconds: 60,
      capabilities: ["image"], // वैकल्पिक, मल्टी-मोडल साझा प्रविष्टियों के लिए
      profile: "vision-profile",
      preferredProfile: "vision-fallback",
    }
    ```
  </Tab>
  <Tab title="CLI प्रविष्टि">
    ```json5
    {
      type: "cli",
      command: "gemini",
      args: [
        "-m",
        "gemini-3-flash",
        "--allowed-tools",
        "read_file",
        "{{MediaPath}} पर स्थित मीडिया पढ़ें और उसका वर्णन <= {{MaxChars}} वर्णों में करें।",
      ],
      maxChars: 500,
      maxBytes: 52428800,
      timeoutSeconds: 120,
      capabilities: ["video", "image"],
    }
    ```

    CLI टेम्पलेट `{{MediaDir}}` (मीडिया फ़ाइल वाली डायरेक्टरी), `{{OutputDir}}` (इस रन के लिए बनाई गई स्क्रैच डायरेक्टरी) और `{{OutputBase}}` (स्क्रैच फ़ाइल का मूल पथ, बिना एक्सटेंशन) का भी उपयोग कर सकते हैं।

  </Tab>
</Tabs>

### प्रोवाइडर क्रेडेंशियल

प्रोवाइडर मीडिया समझने की सुविधा सामान्य मॉडल कॉल के समान प्रमाणीकरण समाधान का उपयोग करती है: प्रमाणीकरण प्रोफ़ाइल, पर्यावरण चर, फिर `models.providers.<providerId>.apiKey`। `tools.media.*.models[]` प्रविष्टियाँ इनलाइन `apiKey` फ़ील्ड स्वीकार नहीं करतीं।

```json5
{
  models: {
    providers: {
      openai: { apiKey: "<OPENAI_API_KEY>" },
      moonshot: { apiKey: "<MOONSHOT_API_KEY>" },
    },
  },
}
```

प्रोफ़ाइल, पर्यावरण चर और कस्टम आधार URL के लिए [टूल और कस्टम प्रोवाइडर](/hi/gateway/config-tools) देखें।

## नियम और व्यवहार

- `maxBytes` से अधिक आकार वाला मीडिया उस मॉडल को छोड़कर अगला मॉडल आज़माता है।
- 1024 बाइट्स से छोटी ऑडियो फ़ाइलों को खाली/दूषित मानकर ट्रांसक्रिप्शन से पहले छोड़ दिया जाता है; इसके बजाय एजेंट को एक नियतात्मक प्लेसहोल्डर ट्रांसक्रिप्ट मिलता है।
- यदि सक्रिय प्राथमिक इमेज मॉडल पहले से ही मूल रूप से विज़न का समर्थन करता है, तो OpenClaw `[Image]` सारांश ब्लॉक को छोड़कर मूल इमेज सीधे मॉडल में भेजता है। MiniMax इसका अपवाद है: `minimax`, `minimax-cn`, `minimax-portal` और `minimax-portal-cn` हमेशा इमेज समझने की प्रक्रिया को plugin-स्वामित्व वाले `MiniMax-VL-01` मीडिया प्रोवाइडर से रूट करते हैं, भले ही पुराना MiniMax M2.x चैट मेटाडेटा इमेज इनपुट का दावा करे (केवल `MiniMax-M3` और बाद के संस्करणों को मूल रूप से विज़न-सक्षम माना जाता है)।
- यदि Gateway/WebChat का प्राथमिक मॉडल केवल टेक्स्ट का समर्थन करता है, तो इमेज अटैचमेंट ऑफ़लोड किए गए `media://inbound/*` संदर्भों के रूप में सुरक्षित रखे जाते हैं, ताकि अटैचमेंट खोने के बजाय इमेज/PDF टूल या कॉन्फ़िगर किया गया इमेज मॉडल अब भी उनका निरीक्षण कर सके।
- स्पष्ट `openclaw infer image describe --file <path> --model <provider/model>` (उपनाम: `openclaw capability image describe`) उस इमेज-सक्षम प्रोवाइडर/मॉडल को सीधे चलाता है, जिसमें `ollama/qwen2.5vl:7b` जैसे Ollama संदर्भ भी शामिल हैं, जब `models.providers.ollama.models[]` के अंतर्गत मेल खाता इमेज-सक्षम मॉडल कॉन्फ़िगर किया गया हो।
- यदि `<capability>.enabled`, `false` नहीं है, लेकिन कोई मॉडल कॉन्फ़िगर नहीं है, तो OpenClaw सक्रिय उत्तर मॉडल को आज़माता है, बशर्ते उसका प्रोवाइडर उस क्षमता का समर्थन करता हो।

### स्वतः-पहचान (डिफ़ॉल्ट)

जब `tools.media.<capability>.enabled`, `false` नहीं है और कोई मॉडल कॉन्फ़िगर नहीं है, तो OpenClaw निम्न विकल्पों को क्रम से आज़माता है और पहले कार्यशील विकल्प पर रुक जाता है:

<Steps>
  <Step title="कॉन्फ़िगर किया गया इमेज मॉडल (केवल इमेज)">
    `agents.defaults.imageModel` प्राथमिक/फ़ॉलबैक संदर्भ, जब तक सक्रिय उत्तर मॉडल पहले से ही मूल रूप से विज़न का समर्थन न करता हो। `provider/model` संदर्भों को वरीयता दें; साधारण संदर्भ केवल तभी कॉन्फ़िगर की गई इमेज-सक्षम प्रोवाइडर मॉडल प्रविष्टियों से पूर्ण किए जाते हैं, जब मिलान अद्वितीय हो।
  </Step>
  <Step title="सक्रिय उत्तर मॉडल">
    सक्रिय उत्तर मॉडल, जब उसका प्रोवाइडर उस क्षमता का समर्थन करता हो।
  </Step>
  <Step title="प्रोवाइडर प्रमाणीकरण (केवल ऑडियो, स्थानीय CLI से पहले)">
    ऑडियो का समर्थन करने वाली कॉन्फ़िगर की गई `models.providers.*` प्रविष्टियाँ स्थानीय CLI से पहले आज़माई जाती हैं। बंडल किए गए प्रोवाइडर का प्राथमिकता क्रम (बराबरी होने पर प्रोवाइडर आईडी के वर्णक्रमानुसार निर्णय): Groq/OpenAI &rarr; xAI &rarr; Deepgram &rarr; OpenRouter &rarr; Google/SenseAudio &rarr; Deepinfra/ElevenLabs &rarr; Mistral।
  </Step>
  <Step title="स्थानीय CLI (केवल ऑडियो)">
    तैयार स्थानीय बाइनरी एक क्रमबद्ध फ़ॉलबैक सूची बन जाती हैं:
    - `whisper-cli` केवल तभी पहले, जब वर्तमान प्रोसेस में किसी पिछले मॉडल आह्वान ने Metal या CUDA देखा हो
    - CPU-डिफ़ॉल्ट `sherpa-onnx-offline` (`tokens.txt`/`encoder.onnx`/`decoder.onnx`/`joiner.onnx` के साथ `SHERPA_ONNX_MODEL_DIR` आवश्यक)
    - `whisper-cli`, जब त्वरण केवल बिल्ड-सक्षम हो या देखा न गया हो
    - Apple Silicon पर `parakeet-mlx` (MLX-सक्षम, डिवाइस उपयोग नहीं देखा गया)
    - `whisper` (Python CLI; डिफ़ॉल्ट रूप से `turbo` मॉडल का उपयोग करता है, स्वतः डाउनलोड करता है)

    बैकएंड क्षमता निरीक्षण कैश किया जाता है और कोई मॉडल लोड नहीं करता। बिल्ड क्षमता, अनुरोधित बैकएंड फ़्लैग और वास्तविक आह्वान से देखा गया बैकएंड अलग-अलग रहते हैं। स्वतः-पहचाना गया whisper.cpp मॉडल-रन लॉग सक्षम रखता है, ताकि अपस्ट्रीम की चयनित-बैकएंड पंक्ति रिकॉर्ड की जा सके। स्पष्ट CLI प्रविष्टियाँ अपना कॉन्फ़िगर किया गया क्रम, बैकएंड फ़्लैग और आउटपुट फ़्लैग बनाए रखती हैं।

  </Step>
  <Step title="प्रोवाइडर प्रमाणीकरण (इमेज/वीडियो)">
    क्षमता का समर्थन करने वाली कॉन्फ़िगर की गई `models.providers.*` प्रविष्टियाँ बंडल किए गए फ़ॉलबैक क्रम से पहले आज़माई जाती हैं। इमेज-सक्षम मॉडल वाले केवल-इमेज कॉन्फ़िगरेशन प्रोवाइडर मीडिया समझने की सुविधा के लिए स्वतः पंजीकृत हो जाते हैं, भले ही वे बंडल किए गए वेंडर plugin न हों।

    बंडल किए गए प्रोवाइडर का प्राथमिकता क्रम (बराबरी होने पर प्रोवाइडर आईडी के वर्णक्रमानुसार निर्णय):
    - इमेज: Anthropic/OpenAI &rarr; Google &rarr; MiniMax &rarr; Deepinfra &rarr; MiniMax Portal &rarr; Z.AI
    - वीडियो: Google &rarr; Qwen &rarr; Moonshot

  </Step>
  <Step title="Antigravity CLI (केवल इमेज/वीडियो)">
    पहला इंस्टॉल किया गया `agy` या `antigravity` बाइनरी (`OPENCLAW_ANTIGRAVITY_CLI` से ओवरराइड करें), जिसे मीडिया की डायरेक्टरी के विरुद्ध सैंडबॉक्स किया गया हो।
  </Step>
</Steps>

</Note>

### प्रॉक्सी समर्थन (ऑडियो/वीडियो प्रोवाइडर कॉल)

प्रदाता-आधारित **ऑडियो** और **वीडियो** समझ मानक आउटबाउंड प्रॉक्सी पर्यावरण चरों का पालन करती है, जिनमें `NO_PROXY`/`no_proxy` बायपास नियम शामिल हैं: `HTTPS_PROXY`, `HTTP_PROXY`, `ALL_PROXY`, `https_proxy`, `http_proxy`, `all_proxy`. लोअरकेस चर अपरकेस चरों पर प्राथमिकता लेते हैं। यदि कोई भी सेट नहीं है, तो मीडिया समझ सीधे निर्गमन का उपयोग करती है; यदि प्रॉक्सी मान विकृत है, तो OpenClaw चेतावनी लॉग करता है और सीधे फ़ेच पर वापस चला जाता है। इमेज समझ इस प्रॉक्सी पथ से नहीं गुजरती।

## क्षमताएँ

किसी `models[]` प्रविष्टि को विशिष्ट मीडिया प्रकारों तक सीमित करने के लिए उस पर `capabilities` सेट करें। साझा सूचियों के लिए, OpenClaw प्रत्येक बंडल किए गए प्रदाता के अनुसार डिफ़ॉल्ट निर्धारित करता है:

| प्रदाता                                                                 | क्षमताएँ              |
| ------------------------------------------------------------------------ | --------------------- |
| `openai`, `anthropic`, `minimax`                                         | इमेज                  |
| `minimax-portal`                                                         | इमेज                  |
| `moonshot`                                                               | इमेज + वीडियो         |
| `openrouter`                                                             | इमेज + ऑडियो          |
| `google` (Gemini API)                                                    | इमेज + ऑडियो + वीडियो |
| `qwen`                                                                   | इमेज + वीडियो         |
| `deepinfra`                                                              | इमेज + ऑडियो          |
| `mistral`                                                                | ऑडियो                 |
| `zai`                                                                    | इमेज                  |
| `groq`, `xai`, `deepgram`, `senseaudio`                                  | ऑडियो                 |
| इमेज-सक्षम मॉडल वाला कोई भी `models.providers.<id>.models[]` कैटलॉग | इमेज                  |

CLI प्रविष्टियों के लिए, अप्रत्याशित मिलान से बचने हेतु `capabilities` स्पष्ट रूप से सेट करें; यदि इसे छोड़ा जाता है, तो प्रविष्टि उस प्रत्येक क्षमता सूची के लिए पात्र होती है जिसमें वह दिखाई देती है।

## मॉडल चयन मार्गदर्शन

- जब गुणवत्ता और सुरक्षा महत्वपूर्ण हों, तो प्रत्येक मीडिया क्षमता के लिए वर्तमान पीढ़ी का सबसे शक्तिशाली मॉडल चुनें।
- अविश्वसनीय इनपुट संभालने वाले टूल-सक्षम एजेंटों के लिए पुराने/कमज़ोर मीडिया मॉडल से बचें।
- उपलब्धता के लिए प्रत्येक क्षमता हेतु कम-से-कम एक फ़ॉलबैक रखें (गुणवत्ता मॉडल + तेज़/सस्ता मॉडल)।
- प्रदाता API अनुपलब्ध होने पर CLI फ़ॉलबैक (`whisper-cli`, `whisper`, `gemini`) सहायक होते हैं।
- ज्ञात फ़ाइल-आउटपुट मोड प्रामाणिक होते हैं: खाली या अनुपस्थित अनुमानित ट्रांसक्रिप्ट फ़ाइल, CLI प्रगति आउटपुट पर वापस जाने के बजाय कोई ट्रांसक्रिप्ट उत्पन्न नहीं करती।
- `parakeet-mlx`: `--output-dir` और डिफ़ॉल्ट `{filename}` आउटपुट टेम्पलेट के साथ `--output-format txt` (या `all`) का उपयोग करें। अपस्ट्रीम `PARAKEET_OUTPUT_FORMAT` और `PARAKEET_OUTPUT_TEMPLATE` पर्यावरण चरों का भी पालन किया जाता है। OpenClaw `<output-dir>/<media-basename>.txt` पढ़ता है; डिफ़ॉल्ट `srt` प्रारूप, अन्य प्रारूप और कस्टम आउटपुट टेम्पलेट stdout का उपयोग जारी रखते हैं।

## अटैचमेंट नीति

प्रत्येक क्षमता का `attachments` नियंत्रित करता है कि किन अटैचमेंट को संसाधित किया जाए:

<ParamField path="mode" type='"first" | "all"' default="first">
  केवल पहले चयनित अटैचमेंट या सभी अटैचमेंट को संसाधित करें।
</ParamField>
<ParamField path="maxAttachments" type="number" default="1">
  संसाधित किए जाने वाले अटैचमेंट की संख्या सीमित करें।
</ParamField>
<ParamField path="prefer" type='"first" | "last" | "path" | "url"'>
  संभावित अटैचमेंट के बीच चयन की प्राथमिकता।
</ParamField>

जब `mode: "all"` हो, तो आउटपुट पर `[Image 1/2]`, `[Audio 2/2]`, आदि लेबल लगाए जाते हैं।

### फ़ाइल-अटैचमेंट निष्कर्षण

- निष्कर्षित फ़ाइल टेक्स्ट को मीडिया प्रॉम्प्ट में जोड़ने से पहले अविश्वसनीय बाहरी सामग्री के रूप में रैप किया जाता है, जिसमें `<<<EXTERNAL_UNTRUSTED_CONTENT id="...">>>` / `<<<END_EXTERNAL_UNTRUSTED_CONTENT id="...">>>` जैसे सीमा चिह्न और एक `Source: External` मेटाडेटा पंक्ति उपयोग होती है।
- मीडिया प्रॉम्प्ट को छोटा रखने के लिए इस पथ में लंबा `SECURITY NOTICE:` बैनर जानबूझकर छोड़ा गया है; सीमा चिह्न और मेटाडेटा फिर भी लागू होते हैं।
- जिस फ़ाइल में निष्कर्षण योग्य टेक्स्ट नहीं होता, उसे `[No extractable text]` मिलता है।
- यदि कोई PDF रेंडर किए गए पृष्ठों की इमेज पर वापस जाता है, तो OpenClaw उन इमेज को विज़न-सक्षम उत्तर मॉडल तक भेजता है और फ़ाइल ब्लॉक में प्लेसहोल्डर `[PDF content rendered to images]` बनाए रखता है।

## स्थिति आउटपुट

जब मीडिया समझ चलती है, तो `/status` में प्रत्येक क्षमता की सारांश पंक्ति शामिल होती है:

```
📎 मीडिया: इमेज सफल (openai/gpt-5.6-sol) · ऑडियो सफल (whisper-cli प्रेक्षित=metal)
```

प्रीफ़्लाइट इन्वेंट्री के लिए, `openclaw capability audio providers` चलाएँ। स्थानीय पंक्तियाँ वैश्विक प्रदाता चयन, तत्परता और अलग-अलग सक्षम/अनुरोधित/प्रेक्षित बैकएंड फ़ील्ड से अलग स्थानीय फ़ॉलबैक विजेता दिखाती हैं। यही स्थानीय चयन सूचनात्मक doctor निष्कर्ष के रूप में भी उपलब्ध है:

```bash
openclaw doctor --lint --only core/doctor/local-audio-acceleration --severity-min info
```

## टिप्पणियाँ

- समझ सर्वोत्तम-प्रयास आधारित है। त्रुटियाँ उत्तरों को अवरुद्ध नहीं करतीं।
- समझ अक्षम होने पर भी अटैचमेंट मॉडल को भेजे जाते हैं।
- समझ कहाँ चले, इसे सीमित करने के लिए `scope` का उपयोग करें (उदाहरण के लिए, केवल DM में)।
