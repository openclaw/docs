## त्वरित शुरुआत

### Gateway मोड

1. Gateway शुरू करें।

```bash
openclaw gateway
```

2. TUI खोलें।

```bash
openclaw tui
```

3. संदेश टाइप करें और Enter दबाएँ।

रिमोट Gateway:

```bash
openclaw tui --url ws://<host>:<port> --token <gateway-token>
```

यदि आपका Gateway पासवर्ड प्रमाणीकरण का उपयोग करता है, तो `--password` का उपयोग करें।

### स्थानीय मोड

Gateway के बिना TUI चलाएँ:

```bash
openclaw chat
# या
openclaw tui --local
```

- `openclaw chat` और `openclaw terminal`, `openclaw tui --local` के उपनाम हैं।
- `--local` को `--url`, `--token`, या `--password` के साथ संयोजित नहीं किया जा सकता।
- स्थानीय मोड सीधे एम्बेडेड एजेंट रनटाइम का उपयोग करता है। अधिकांश स्थानीय टूल काम करते हैं, लेकिन केवल-Gateway सुविधाएँ उपलब्ध नहीं होतीं।
- अकेला `openclaw` (बिना उपकमांड) स्वचालित रूप से लक्ष्य चुनता है: कॉन्फ़िगर न किया गया इंस्टॉलेशन इन्फ़रेंस ऑनबोर्डिंग चलाता है; अमान्य कॉन्फ़िगरेशन क्लासिक Doctor मार्गदर्शन खोलता है; पहुँच योग्य कॉन्फ़िगर किया गया Gateway इस TUI शेल को Gateway मोड में खोलता है; अन्यथा कॉन्फ़िगर किया गया स्थानीय मॉडल इसे स्थानीय मोड में खोलता है।

## आपको क्या दिखाई देता है

- हेडर: कनेक्शन URL, वर्तमान एजेंट, वर्तमान सत्र।
- चैट लॉग: उपयोगकर्ता संदेश, सहायक के उत्तर, सिस्टम सूचनाएँ, टूल कार्ड।
- स्थिति पंक्ति: कनेक्शन/रन की स्थिति (कनेक्ट हो रहा है, चल रहा है, स्ट्रीम हो रहा है, निष्क्रिय, त्रुटि)।
- फुटर: एजेंट + सत्र + मॉडल + लक्ष्य स्थिति + चिंतन/तेज़/विस्तृत/ट्रेस/तर्क + टोकन संख्या + डिलीवर।
- इनपुट: स्वतः-पूर्णता वाला टेक्स्ट एडिटर।

## मानसिक मॉडल: एजेंट + सत्र

- एजेंट अद्वितीय स्लग होते हैं (उदा. `main`, `research`)। Gateway उनकी सूची उपलब्ध कराता है।
- सत्र वर्तमान एजेंट से संबंधित होते हैं।
- सत्र कुंजियाँ `agent:<agentId>:<sessionKey>` के रूप में संग्रहीत होती हैं।
  - यदि आप `/session main` टाइप करते हैं, तो TUI इसे `agent:<currentAgent>:main` में विस्तारित करता है।
  - यदि आप `/session agent:other:main` टाइप करते हैं, तो आप स्पष्ट रूप से उस एजेंट सत्र पर स्विच करते हैं।
- सत्र का दायरा:
  - `per-sender` (डिफ़ॉल्ट): प्रत्येक एजेंट के कई सत्र होते हैं।
  - `global`: TUI हमेशा `global` सत्र का उपयोग करता है (चयनकर्ता खाली हो सकता है)।
- वर्तमान एजेंट + सत्र हमेशा फुटर में दिखाई देते हैं।
- यदि सत्र का कोई [लक्ष्य](/hi/tools/goal) है, तो फुटर उसकी संक्षिप्त स्थिति दिखाता है:
  `Pursuing goal`, `Goal paused (/goal resume)`, `Goal blocked (/goal resume)`, या `Goal achieved`।
- जब `--session` के बिना शुरू किया जाता है, तो Gateway-मोड TUI उसी Gateway, एजेंट और सत्र दायरे के लिए अंतिम चयनित सत्र को फिर से शुरू करता है, यदि वह सत्र अभी भी मौजूद हो। `--session`, `/session`, `/new`, या `/reset` देना स्पष्ट चयन बना रहता है।

## भेजना + डिलीवरी

- संदेश हमेशा Gateway (या स्थानीय मोड में एम्बेडेड रनटाइम) को भेजे जाते हैं; सहायक के उत्तर को वापस किसी चैट प्रदाता तक पहुँचाना एक अलग चरण है, जो डिफ़ॉल्ट रूप से बंद रहता है।
- TUI, WebChat की तरह एक आंतरिक स्रोत सतह है, सामान्य आउटबाउंड चैनल नहीं। जिन हार्नेस को दिखाई देने वाले उत्तरों के लिए `tools.message` की आवश्यकता होती है, वे लक्ष्य-रहित `message.send` से सक्रिय TUI टर्न को संतुष्ट कर सकते हैं; स्पष्ट प्रदाता डिलीवरी फिर भी सामान्य कॉन्फ़िगर किए गए चैनलों का उपयोग करती है और कभी भी `lastChannel` पर फ़ॉलबैक नहीं करती।
- लॉन्च के समय पूरे TUI सत्र के लिए डिलीवरी निर्धारित हो जाती है: इसे चालू करने के लिए `openclaw tui --deliver` के साथ शुरू करें। सत्र के बीच इसे बदलने के लिए कोई `/deliver` स्लैश कमांड या Settings टॉगल नहीं है; इसे बदलने के लिए TUI पुनः शुरू करें।

## चयनकर्ता + ओवरले

- मॉडल चयनकर्ता: उपलब्ध मॉडलों की सूची दिखाएँ और सत्र ओवरराइड सेट करें।
- एजेंट चयनकर्ता: कोई दूसरा एजेंट चुनें।
- सत्र चयनकर्ता: वर्तमान एजेंट के पिछले 7 दिनों में अपडेट किए गए अधिकतम 50 सत्र दिखाता है। किसी पुराने ज्ञात सत्र पर जाने के लिए `/session <key>` का उपयोग करें।
- सेटिंग्स (`/settings`): टूल आउटपुट विस्तार और चिंतन की दृश्यता टॉगल करें। यह पैनल डिलीवरी नियंत्रित नहीं करता।

## कीबोर्ड शॉर्टकट

- Esc: सक्रिय रन निरस्त करें
- Ctrl+C: इनपुट साफ़ करें (बाहर निकलने के लिए दो बार दबाएँ)
- Ctrl+D: बाहर निकलें
- Ctrl+L: मॉडल चयनकर्ता
- Ctrl+G: एजेंट चयनकर्ता
- Ctrl+P: सत्र चयनकर्ता
- Ctrl+O: टूल आउटपुट विस्तार टॉगल करें
- Ctrl+T: चिंतन की दृश्यता टॉगल करें (इतिहास पुनः लोड करता है)

## स्लैश कमांड

मुख्य:

- `/help`
- `/status` (Gateway को अग्रेषित; सत्र/मॉडल सारांश दिखाता है)
- `/gateway-status` (उपनाम `/gwstatus`; सीधे Gateway कनेक्शन की स्थिति दिखाता है)
- `/agent <id>` (या `/agents`)
- `/session <key>` (या `/sessions`)
- `/model <provider/model>` (या `/models`)

सत्र नियंत्रण:

- `/think <off|minimal|low|medium|high>` (मॉडल के आधार पर उच्चतर स्तर `xhigh`/`max` जैसे स्तर जोड़ सकते हैं)
- `/fast <status|auto|on|off>`
- `/verbose <on|full|off>`
- `/trace <on|off>`
- `/reasoning <on|off|stream>`
- `/usage <off|tokens|full|reset>` (`reset`/`inherit`/`clear`/`default` सत्र ओवरराइड साफ़ करता है)
- `/goal [status] | /goal start <objective> | /goal edit <objective> | /goal pause|resume|complete|block|clear`
- `/elevated <on|off|ask|full>` (उपनाम: `/elev`)
- `/activation <mention|always>`
- `/queue <steer|followup|collect|interrupt> [debounce:<duration>] [cap:<n>] [drop:<summarize|old|new>]`
- `/queue default` (या `/queue reset`) सत्र ओवरराइड साफ़ करता है

सत्र जीवनचक्र:

- `/new` (नई कुंजी के अंतर्गत नया, पृथक सत्र बनाएँ; पुराने सत्र पर मौजूद अन्य TUI क्लाइंट प्रभावित नहीं होते)
- `/reset` (वर्तमान सत्र कुंजी को उसी स्थान पर रीसेट करें)
- `/abort` (सक्रिय रन निरस्त करें)
- `/settings`
- `/exit` (या `/quit`)

केवल स्थानीय मोड:

- `/auth [provider]` TUI के भीतर प्रदाता प्रमाणीकरण/लॉगिन प्रवाह खोलता है।

स्थानीय मोड एम्बेडेड रनटाइम के भीतर उन्हीं कतार मोड को लागू करता है। रन के
बीच दिया गया प्रॉम्प्ट सत्र की `/queue` नीति का पालन करता है: जब
रनटाइम उसे स्वीकार कर सकता है, तब `steer` उसे शामिल करता है, `followup` अलग टर्न की प्रतीक्षा करता है, `collect`
लंबित प्रॉम्प्ट संयोजित करता है, और `interrupt` नया रन शुरू करने से पहले वर्तमान
रन रोकता है। स्पष्ट `/steer <message>` केवल Gateway के लिए है; स्थानीय मोड में `/queue steer` के साथ
सामान्य संदेश का उपयोग करें।

OpenClaw:

- `/openclaw [request]` सामान्य एजेंट TUI से [OpenClaw](#openclaw-setup-and-repair-helper) सेटअप/मरम्मत चैट पर लौटाता है और वैकल्पिक रूप से एक अनुरोध अग्रेषित करता है।

अन्य Gateway स्लैश कमांड (उदाहरण के लिए, `/context`) Gateway को अग्रेषित किए जाते हैं और सिस्टम आउटपुट के रूप में दिखाए जाते हैं। [स्लैश कमांड](/hi/tools/slash-commands) देखें।

## स्थानीय शेल कमांड

- TUI होस्ट पर स्थानीय शेल कमांड चलाने के लिए पंक्ति के आगे `!` लगाएँ।
- TUI स्थानीय निष्पादन की अनुमति के लिए प्रति सत्र एक बार पूछता है; अस्वीकार करने पर सत्र के लिए `!` अक्षम रहता है।
- कमांड TUI की कार्यशील डायरेक्टरी में नए, गैर-इंटरैक्टिव शेल में चलते हैं (कोई स्थायी `cd`/env नहीं)।
- स्थानीय शेल कमांड को उनके परिवेश में `OPENCLAW_SHELL=tui-local` मिलता है।
- अकेला `!` सामान्य संदेश के रूप में भेजा जाता है; आरंभिक स्पेस स्थानीय निष्पादन ट्रिगर नहीं करते।

## OpenClaw सेटअप और मरम्मत सहायक

OpenClaw रिंग-ज़ीरो सेटअप/मरम्मत सहायक है, जिसे कॉन्फ़िगर किया गया डिफ़ॉल्ट मॉडल लाइव इन्फ़रेंस जाँच पास करने के बाद `openclaw setup` के रूप में उपलब्ध कराया जाता है। यदि इन्फ़रेंस उपलब्ध नहीं है, तो इंटरैक्टिव आह्वान इन्फ़रेंस ऑनबोर्डिंग पर लौटता है और स्वचालन मरम्मत मार्गदर्शन के साथ विफल हो जाता है। यह `openclaw tui --local` वाले उसी स्थानीय TUI शेल में चलता है और OpenClaw के टाइप किए गए, अनुमोदन-नियंत्रित ऑपरेशन तक सीमित AI एजेंट द्वारा समर्थित है:

```bash
openclaw setup                       # इंटरैक्टिव रूप से शुरू करें
openclaw setup -m "status"           # एक अनुरोध चलाएँ और बाहर निकलें
openclaw setup -m "set default model openai/gpt-5.2" --yes   # कॉन्फ़िगरेशन लेखन लागू करें
```

- स्थायी कॉन्फ़िगरेशन लेखन के लिए अनुमोदन आवश्यक है: इंटरैक्टिव रूप से पुष्टि करें या `--yes` दें।
- `--json` चैट शुरू करने के बजाय स्टार्टअप अवलोकन को JSON के रूप में प्रिंट करता है।
- OpenClaw के भीतर से, `open-tui` अनुरोध (उदाहरण के लिए, सामान्य एजेंट से बात करने का अनुरोध) OpenClaw से बाहर निकलकर नियमित एजेंट TUI खोलता है; वापस आने के लिए वहाँ `/openclaw` का उपयोग करें।

जब वर्तमान कॉन्फ़िगरेशन पहले से मान्य हो और आप चाहते हों कि एम्बेडेड एजेंट उसी मशीन पर उसका निरीक्षण करे, उसकी तुलना दस्तावेज़ों से करे और चलते हुए Gateway पर निर्भर हुए बिना अंतर ठीक करने में सहायता करे, तब स्थानीय मोड का उपयोग करें।

यदि `openclaw config validate` पहले से विफल हो रहा है, तो पहले `openclaw configure` या `openclaw doctor --fix` से शुरू करें; `openclaw chat` को शुरू होने के लिए फिर भी लोड किए जा सकने वाले कॉन्फ़िगरेशन की आवश्यकता होती है।

सामान्य चक्र:

1. स्थानीय मोड शुरू करें:

```bash
openclaw chat
```

2. एजेंट को बताएँ कि आप क्या जाँचना चाहते हैं, उदाहरण के लिए:

```text
मेरे gateway प्रमाणीकरण कॉन्फ़िगरेशन की दस्तावेज़ों से तुलना करें और सबसे छोटा सुधार सुझाएँ।
```

3. सटीक प्रमाण और सत्यापन के लिए स्थानीय शेल कमांड का उपयोग करें:

```text
!openclaw config file
!openclaw docs gateway auth token secretref
!openclaw config validate
!openclaw doctor
```

4. `openclaw config set` या `openclaw configure` से सीमित बदलाव लागू करें, फिर `!openclaw config validate` दोबारा चलाएँ।
5. यदि Doctor स्वचालित माइग्रेशन या मरम्मत की अनुशंसा करता है, तो उसकी समीक्षा करें और `!openclaw doctor --fix` चलाएँ।

सुझाव:

- `openclaw.json` को हाथ से संपादित करने के बजाय `openclaw config set` या `openclaw configure` को प्राथमिकता दें।
- `openclaw docs "<query>"` उसी मशीन से लाइव दस्तावेज़ इंडेक्स खोजता है।
- जब आपको संरचित स्कीमा और SecretRef/समाधेयता त्रुटियाँ चाहिए हों, तब `openclaw config validate --json` उपयोगी है।

- टूल कॉल आर्ग्युमेंट + परिणाम वाले कार्ड के रूप में दिखाई देते हैं।
- Ctrl+O संक्षिप्त/विस्तृत दृश्यों के बीच टॉगल करता है।
- टूल चलने के दौरान आंशिक अपडेट उसी कार्ड में स्ट्रीम होते हैं।

## टर्मिनल के रंग

- TUI सहायक के मुख्य टेक्स्ट को आपके टर्मिनल के डिफ़ॉल्ट अग्रभूमि रंग में रखता है, ताकि गहरे और हल्के दोनों टर्मिनल पठनीय रहें।
- यदि आपका टर्मिनल हल्की पृष्ठभूमि का उपयोग करता है और स्वतः-पहचान गलत है, तो `openclaw tui` शुरू करने से पहले `OPENCLAW_THEME=light` सेट करें।
- इसके बजाय मूल गहरे रंग-पैलेट को बाध्य करने के लिए `OPENCLAW_THEME=dark` सेट करें।

## इतिहास + स्ट्रीमिंग

- कनेक्ट होने पर TUI नवीनतम इतिहास लोड करता है (डिफ़ॉल्ट 200 संदेश)।
- स्ट्रीमिंग उत्तर अंतिम होने तक उसी स्थान पर अपडेट होते हैं।
- अधिक समृद्ध टूल कार्ड के लिए TUI एजेंट टूल इवेंट भी सुनता है।

## कनेक्शन का विवरण

- TUI सामान्य `ui` क्लाइंट मोड के अंतर्गत क्लाइंट आईडी `openclaw-tui` से कनेक्ट होता है (वही मोड जिसे Control UI और WebChat Gateway नीति के लिए उपयोग करते हैं)।
- पुनः कनेक्शन सिस्टम संदेश दिखाते हैं; इवेंट में अंतराल लॉग में दिखाए जाते हैं।

## विकल्प

- `--local`: स्थानीय एम्बेडेड एजेंट रनटाइम के साथ चलाएँ
- `--url <url>`: Gateway WebSocket URL (डिफ़ॉल्ट रूप से कॉन्फ़िगरेशन से `gateway.remote.url`, या लूपबैक पर `ws://127.0.0.1:<port>`)
- `--token <token>`: Gateway टोकन (यदि आवश्यक हो)
- `--password <password>`: Gateway पासवर्ड (यदि आवश्यक हो)
- `--tls-fingerprint <sha256>`: पिन किए गए `wss://` Gateway के लिए अपेक्षित TLS प्रमाणपत्र फ़िंगरप्रिंट
- `--session <key>`: सत्र कुंजी (डिफ़ॉल्ट: `main`, या स्कोप वैश्विक होने पर `global`)
- `--deliver`: सहायक के उत्तर प्रदाता को भेजें (डिफ़ॉल्ट रूप से बंद)
- `--thinking <level>`: भेजते समय चिंतन स्तर को ओवरराइड करें
- `--message <text>`: कनेक्ट होने के बाद एक प्रारंभिक संदेश भेजें
- `--timeout-ms <ms>`: एजेंट टाइमआउट मिलीसेकंड में (डिफ़ॉल्ट रूप से `agents.defaults.timeoutSeconds`)
- `--history-limit <n>`: लोड की जाने वाली इतिहास प्रविष्टियाँ (डिफ़ॉल्ट `200`)

<Warning>
जब आप `--url` सेट करते हैं, तो TUI कॉन्फ़िगरेशन या परिवेश क्रेडेंशियल पर फ़ॉलबैक नहीं करता। `--token` या `--password` स्पष्ट रूप से पास करें, और जब लक्ष्य पिन किए गए प्रमाणपत्र का उपयोग करता हो, तब `--tls-fingerprint` भी पास करें। स्पष्ट क्रेडेंशियल का न होना एक त्रुटि है। स्थानीय मोड में, `--url`, `--token`, `--password`, या `--tls-fingerprint` पास न करें।
</Warning>

## समस्या निवारण

संदेश भेजने के बाद कोई आउटपुट नहीं:

- यह पुष्टि करने के लिए कि Gateway कनेक्टेड और निष्क्रिय/व्यस्त है, TUI में `/status` चलाएँ।
- Gateway लॉग जाँचें: `openclaw logs --follow`।
- पुष्टि करें कि एजेंट चल सकता है: `openclaw status` और `openclaw models status`।
- यदि आप किसी चैट चैनल में संदेशों की अपेक्षा करते हैं, तो पुष्टि करें कि TUI को `--deliver` के साथ शुरू किया गया था (पुनः आरंभ किए बिना इसे बाद में चालू नहीं किया जा सकता)।

## कनेक्शन संबंधी समस्या निवारण

- `disconnected`: सुनिश्चित करें कि Gateway चल रहा है और आपके `--url/--token/--password` सही हैं।
- चयनकर्ता में कोई एजेंट नहीं: `openclaw agents list` और अपना रूटिंग कॉन्फ़िगरेशन जाँचें।
- सत्र चयनकर्ता खाली है: हो सकता है कि आप वैश्विक स्कोप में हों या अभी कोई सत्र न हो।
