# सुरक्षा

ClawHub सुरक्षा समस्याओं की रिपोर्ट `openclaw/clawhub` के लिए GitHub Security Advisories के माध्यम से की जा सकती है।

ClawHub में ही मौजूद कमजोरियों के लिए GitHub Security Advisories का उपयोग करें। अच्छी ClawHub advisory रिपोर्टों में इनसे जुड़े बग शामिल होते हैं:

- ClawHub वेबसाइट, API, या CLI
- रजिस्ट्री प्रकाशन, डाउनलोड, इंस्टॉल, या आर्टिफैक्ट अखंडता
- प्रमाणीकरण, प्राधिकरण, या API टोकन
- स्कैनिंग, मॉडरेशन, या रिपोर्ट हैंडलिंग

किसी तृतीय-पक्ष skill या Plugin के अपने स्रोत कोड में मौजूद कमजोरियों के लिए ClawHub advisories का उपयोग न करें। उनकी रिपोर्ट सीधे उस प्रकाशक या स्रोत रिपॉजिटरी को करें जो ClawHub लिस्टिंग से लिंक है।

## कमजोरी का प्रकटीकरण

क्योंकि ClawHub एक होस्टेड क्लाउड एप्लिकेशन है, ClawHub सेवा कमजोरियों को डिफ़ॉल्ट रूप से सार्वजनिक रूप से प्रकट नहीं किया जाता। उन्हें तब सार्वजनिक रूप से प्रकट किया जाता है जब वास्तविक उपयोगकर्ता प्रभाव का प्रमाण हो या जब उपयोगकर्ताओं को कार्रवाई करनी हो।

वास्तविक उपयोगकर्ता प्रभाव के उदाहरणों में पुष्टि किया गया शोषण, उपयोगकर्ता डेटा या रहस्यों का उजागर होना, प्लेटफ़ॉर्म विफलता के कारण दुर्भावनापूर्ण सामग्री का उपयोगकर्ताओं तक पहुँचना, या कोई भी ऐसी समस्या शामिल है जिसके लिए उपयोगकर्ताओं को क्रेडेंशियल घुमाने, स्थानीय सॉफ़्टवेयर अपडेट करने, या अन्य सुरक्षात्मक कार्रवाई करने की आवश्यकता हो।

उपयोगकर्ता द्वारा इंस्टॉल किए गए सॉफ़्टवेयर में मौजूद कमजोरियों को सार्वजनिक रूप से प्रकट किया जाता है, जैसे ClawHub CLI पैकेज, बाइनरी, लाइब्रेरी, या अन्य रिलीज़ आर्टिफैक्ट जिन्हें उपयोगकर्ताओं को स्थानीय रूप से अपडेट करने की आवश्यकता होती है।

## संबंधित पृष्ठ

इंस्टॉल-समय ऑडिट लेबल, जोखिम स्तर, निष्कर्षों और व्याख्या के लिए, [सुरक्षा ऑडिट](/clawhub/security-audits) देखें।

मार्केटप्लेस रिपोर्ट, मॉडरेशन होल्ड, छिपी हुई लिस्टिंग, प्रतिबंध, और खाते की स्थिति के लिए, [मॉडरेशन और खाता सुरक्षा](/hi/clawhub/moderation) देखें।
