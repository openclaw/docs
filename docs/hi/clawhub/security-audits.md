# सुरक्षा ऑडिट

ClawHub सुरक्षा ऑडिट आपको यह तय करने में मदद करते हैं कि कोई skill या plugin इंस्टॉल करने के लिए पर्याप्त सुरक्षित है या नहीं। वे दिखाते हैं कि कोई रिलीज़ क्या करती है, वह कौन-सा अधिकार मांगती है, और फ़ाइलों, खातों, क्रेडेंशियल्स, कोड, या बाहरी सेवाओं तक पहुंच पाने से पहले किसी चीज़ पर अतिरिक्त ध्यान देने की ज़रूरत है या नहीं।

ऑडिट मजबूत सुरक्षा संकेत हैं, लेकिन वे यह गारंटी नहीं देते कि कोई रिलीज़ जोखिम-मुक्त है। संवेदनशील पहुंच देने से पहले हमेशा अपने विवेक का उपयोग करें।

[सुरक्षा](/clawhub/security), [स्वीकार्य उपयोग](/hi/clawhub/acceptable-usage), और [मॉडरेशन और खाता सुरक्षा](/hi/clawhub/moderation) भी देखें।

## इंस्टॉल करने से पहले क्या जांचें

इंस्टॉल करने से पहले, समीक्षा करें:

- समग्र ऑडिट स्थिति
- जोखिम स्तर
- सूचीबद्ध कोई भी निष्कर्ष
- आवश्यक क्रेडेंशियल्स, अनुमतियां, या environment variables
- मालिक, स्रोत, संस्करण, चेंजलॉग, डाउनलोड, स्टार, और अन्य भरोसे के संकेत

केवल वही सामग्री इंस्टॉल करें जिसे आप समझते और भरोसा करते हैं।

## ऑडिट स्थिति

ऑडिट स्थिति आपको बताती है कि ऑडिट परिणाम पर कैसे प्रतिक्रिया देनी है:

| स्थिति | अर्थ |
| ----------- | ------------------------------------------------------------------------- |
| `Pass` | कम जोखिम से ऊपर कोई दिखाई देने वाली समस्या नहीं मिली। |
| `Review` | इंस्टॉल करने से पहले निष्कर्ष पढ़ें। रिलीज़ अभी भी वैध हो सकती है। |
| `Warn` | अतिरिक्त सावधानी बरतें। ClawHub को उच्च-प्रभाव वाली चिंता या चेतावनी संकेत मिला। |
| `Malicious` | इंस्टॉल न करें। |
| `Pending` | ऑडिट अभी पूरे नहीं हुए हैं। |
| `Error` | ऑडिट पूरा नहीं किया जा सका। |

`Pass` आश्वस्त करता है, लेकिन यह आपके अपने विवेक की जगह नहीं लेता। यह उन टूल्स के लिए सबसे अधिक मायने रखता है जो सामग्री प्रकाशित कर सकते हैं, डेटा संपादित कर सकते हैं, कमांड चला सकते हैं, फ़ाइलें पढ़ सकते हैं, या उत्पादन प्रणालियों तक पहुंच सकते हैं।

## जोखिम स्तर

जोखिम स्तर blast radius का वर्णन करता है: रिलीज़ में कितनी शक्ति दिखती है, यदि आप उसे अपेक्षित रूप से उपयोग करते हैं।

| जोखिम स्तर | अर्थ |
| ---------- | ----------------------------------------------------------------------------- |
| `Low` | बहुत कम संवेदनशील अधिकार या उपयोगकर्ता प्रभाव मिला। |
| `Medium` | रिलीज़ के पास महत्वपूर्ण अधिकार हैं, जैसे खाता पहुंच या डेटा बदलाव। |
| `High` | रिलीज़ के पास उच्च-प्रभाव अधिकार, गंभीर निष्कर्ष, या दुर्भावनापूर्ण संकेत हैं। |

जोखिम स्तर और ऑडिट स्थिति अलग-अलग प्रश्नों का उत्तर देते हैं:

- जोखिम स्तर पूछता है: "यहां कितनी शक्ति है?"
- ऑडिट स्थिति पूछती है: "मुझे इस परिणाम के साथ क्या करना चाहिए?"

उदाहरण के लिए, कोई publishing skill `Medium` जोखिम के साथ `Review` दिखा सकती है। इसका मतलब यह नहीं है कि वह दुर्भावनापूर्ण है। इसका मतलब है कि skill उद्देश्य-संगत दिखती है, लेकिन महत्वपूर्ण खाता अधिकार के साथ कार्य कर सकती है।

## निष्कर्ष

निष्कर्ष बताते हैं कि कोई ऑडिट परिणाम क्यों दिखाया गया। प्रत्येक निष्कर्ष में आमतौर पर शामिल होता है:

- इसका क्या अर्थ है
- इसे क्यों फ़्लैग किया गया
- संबंधित skill या plugin सामग्री
- एक अनुशंसा

निष्कर्षों को `Info`, `Low`, `Medium`, `High`, या `Critical` लेबल किया जा सकता है। अधिक गंभीरता वाले निष्कर्ष जोखिम स्तर और ऑडिट स्थिति में अधिक मजबूत योगदान देते हैं।

कम-विश्वास वाले निष्कर्ष सार्वजनिक ऑडिट रोलअप से छिपाए जाते हैं ताकि पृष्ठ उपयोगी प्रमाण पर केंद्रित रहे।

## ClawHub क्या जांचता है

ClawHub सबमिट किए गए रिलीज़ आर्टिफैक्ट्स का ऑडिट करता है, जिनमें शामिल हैं:

- skill निर्देश या plugin मेटाडेटा
- घोषित environment variables और अनुमतियां
- इंस्टॉल निर्देश और पैकेज मेटाडेटा
- शामिल फ़ाइलें और फ़ाइल manifests
- संगतता और capability मेटाडेटा

मुख्य प्रश्न coherence है: क्या नाम, सारांश, मेटाडेटा, मांगा गया अधिकार, और वास्तविक सामग्री उस चीज़ से मेल खाते हैं जिसकी उपयोगकर्ता उचित रूप से अपेक्षा करेंगे?

शक्तिशाली व्यवहार अपने आप बुरा नहीं होता। कई उपयोगी टूल्स को क्रेडेंशियल्स, local commands, provider APIs, या package installs की आवश्यकता होती है। ऑडिट जांचता है कि वह शक्ति अपेक्षित, प्रकट, और अनुपातिक है या नहीं।

आर्टिफैक्ट पृष्ठ पूर्ण ऑडिट से यहां लिंक करते हैं:

```text
/<owner>/skills/<slug>/security-audit
```

ऑडिट पृष्ठ मिलाकर दिखाता है:

1. SkillSpector
2. VirusTotal
3. जोखिम विश्लेषण

## VirusTotal

ClawHub ऑडिट stack में VirusTotal को malware telemetry के रूप में उपयोग करता है। VirusTotal फ़ाइल प्रतिष्ठा और malware scanning के लिए एक भरोसेमंद industry standard है, और हमारी साझेदारी ClawHub को skill और plugin समीक्षा में व्यापक security intelligence जोड़ने देती है।

VirusTotal विशेष रूप से ज्ञात दुर्भावनापूर्ण आर्टिफैक्ट्स, engine hits, और reputation signals के लिए उपयोगी है, जो ClawHub की agent-aware समीक्षा को पूरक करते हैं। जब vendor engine counts उपलब्ध होते हैं, तो ऑडिट उन्हें सरल भाषा में सारांशित करता है, जैसे:

```text
62/62 vendors flagged this skill as clean.
```

या:

```text
2/64 vendors flagged this skill as malicious, 1/64 flagged it as suspicious, and 61/64 flagged it as clean.
```

जब ClawHub के पास सारांशित करने के लिए कोई vendor-count telemetry नहीं होती, तो ऑडिट कहता है:

```text
No VirusTotal findings
```

VirusTotal telemetry ही रहता है। यह ClawHub के अपने artifact-aware जोखिम विश्लेषण की जगह नहीं लेता।

## जोखिम विश्लेषण

जोखिम विश्लेषण आंतरिक रूप से ClawScan द्वारा संचालित है, जो ClawHub का अपना सुरक्षा ऑडिट सिस्टम है। यह प्रत्येक रिलीज़ को agent-facing artifact के रूप में समीक्षा करता है: निर्देश, मेटाडेटा, घोषित अनुमतियां, फ़ाइलें, capability signals, static scan signals, SkillSpector निष्कर्ष, VirusTotal telemetry, और publisher-provided context। Static scan signals इस समीक्षा के लिए आंतरिक context हैं; वे कोई स्वतंत्र सार्वजनिक ऑडिट अनुभाग या install-blocking verdict नहीं हैं।

जोखिम विश्लेषण
[OWASP Agentic Skills Top 10](https://owasp.org/www-project-agentic-skills-top-10/)
का उपयोग prompt injection, tool misuse, credential exposure, unsafe execution, memory या context poisoning, और excessive agency जैसे जोखिमों को देखने के lens के रूप में करता है।

ClawScan डरावनी दिखने वाली capability को अपने आप दुर्भावनापूर्ण नहीं मानता। यह पूछता है कि capability प्रकट की गई है या नहीं, उद्देश्य-संगत है या नहीं, और रिलीज़ के बताए गए use case से समर्थित है या नहीं।
