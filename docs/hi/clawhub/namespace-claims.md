# संगठन और नेमस्पेस दावे

ClawHub ओनर हैंडल, संगठन हैंडल, Skills स्लग, Plugin पैकेज नामों और
पैकेज स्कोपों को सार्वजनिक नेमस्पेस के रूप में उपयोग करता है। यदि कोई नेमस्पेस
किसी वास्तविक दुनिया के प्रोजेक्ट, ब्रांड, पैकेज इकोसिस्टम या संगठन से संबंधित
लगता है, लेकिन ClawHub पर पहले से दावा किया गया, आरक्षित, भ्रामक या विवादित है,
तो स्टाफ से
[संगठन / नेमस्पेस दावा इश्यू फॉर्म](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
के साथ उसकी समीक्षा करने के लिए कहें।

इस पथ का उपयोग सार्वजनिक, गैर-संवेदनशील स्वामित्व समीक्षा के लिए करें। नेमस्पेस
दावों के लिए इन-प्रोडक्ट रिपोर्ट या खाता अपील फॉर्म का उपयोग न करें।

## दावा कब खोलें

जब आपको लगे कि ClawHub स्टाफ को यह समीक्षा करनी चाहिए कि वास्तविक दुनिया के
स्वामित्व के कारण किसी नेमस्पेस को आरक्षित, स्थानांतरित, पुनर्नामित, छिपाया,
क्वारंटीन, उपनामित या अन्यथा बदला जाना चाहिए, तब नेमस्पेस दावा खोलें।

उदाहरणों में शामिल हैं:

- ऐसा संगठन हैंडल जो आपके GitHub संगठन, प्रोजेक्ट, कंपनी या समुदाय से मेल खाता हो
- `@example-org/*` जैसा पैकेज स्कोप, जिसे केवल मेल खाते ClawHub ओनर के तहत ही
  प्रकाशित होना चाहिए
- कोई Skills स्लग या Plugin पैकेज नाम जो किसी प्रोजेक्ट का प्रतिरूपण करता हुआ लगे
- कोई ब्रांड, ट्रेडमार्क, प्रोजेक्ट पुनर्नामकरण या पैकेज इतिहास विवाद
- कोई हटाया गया, निष्क्रिय या पहुंच से बाहर ओनर जो उचित नेमस्पेस ओनर को रोकता हो

यदि लिस्टिंग स्वामित्व विवाद से आगे बढ़कर असुरक्षित, दुर्भावनापूर्ण या भ्रामक
है, तो संबंधित मॉडरेशन या सुरक्षा मार्गदर्शन का भी पालन करें। नेमस्पेस दावा
फॉर्म स्वामित्व समीक्षा के लिए है, आपातकालीन भेद्यता प्रकटीकरण के लिए नहीं।

## फाइल करने से पहले

पहले पुष्टि करें कि आप उस ओनर के साथ प्रकाशित कर रहे हैं जो नेमस्पेस से मेल
खाता है। Plugin पैकेजों के लिए, `@example-org/example-plugin` जैसे स्कोप वाले
नामों को मेल खाते `example-org` ओनर के रूप में प्रकाशित किया जाना चाहिए।

यदि आप वर्तमान ओनर को प्रबंधित कर सकते हैं, तो प्रभावित संसाधन को प्रकाशित,
पुनर्नामित, स्थानांतरित, छिपाकर या हटाकर नेमस्पेस को सीधे ठीक करें। दावा तब
उपयोग करें जब आप वर्तमान ओनर को प्रबंधित नहीं कर सकते या जब स्टाफ को किसी विवाद
का समाधान करना हो।

## शामिल करने योग्य साक्ष्य

सार्वजनिक, गैर-संवेदनशील साक्ष्य का उपयोग करें। उपयोगी प्रमाणों में शामिल हैं:

- GitHub संगठन, रेपो, रिलीज़ या मेंटेनर इतिहास
- आधिकारिक प्रोजेक्ट दस्तावेज़ जो नेमस्पेस का नाम बताते हों
- डोमेन या आधिकारिक ईमेल-डोमेन प्रमाण
- npm, PyPI, crates.io या अन्य पैकेज-रजिस्ट्री स्कोप नियंत्रण
- ट्रेडमार्क, ब्रांड या प्रोजेक्ट स्वामित्व साक्ष्य जिन पर सार्वजनिक रूप से
  चर्चा करना सुरक्षित हो
- स्रोत रिपॉजिटरी इतिहास, पैकेज इतिहास या सार्वजनिक पुनर्नामकरण सूचनाएं
- विवादित ClawHub ओनर, Skills, Plugin, पैकेज या इश्यू के लिंक

समझाएं कि प्रत्येक लिंक क्या साबित करता है। स्टाफ को निजी क्रेडेंशियल या सीक्रेट
की आवश्यकता के बिना संबंध समझ में आना चाहिए।

## क्या शामिल न करें

सार्वजनिक GitHub इश्यू में सीक्रेट या निजी प्रमाण न डालें। ये शामिल न करें:

- API टोकन, साइनिंग कुंजियां या क्रेडेंशियल
- DNS चैलेंज टोकन
- निजी कानूनी फाइलें या अनुबंध
- व्यक्तिगत पहचान दस्तावेज़
- निजी ईमेल, निजी सुरक्षा रिपोर्ट या गोपनीय ग्राहक डेटा

दावा फॉर्म पूछता है कि क्या संवेदनशील साक्ष्य के लिए निजी स्टाफ चैनल की आवश्यकता
है। संवेदनशील सामग्री को सार्वजनिक रूप से पोस्ट करने के बजाय उस विकल्प का उपयोग
करें।

## संभावित परिणाम

साक्ष्य और जोखिम के आधार पर, ClawHub स्टाफ कोई नेमस्पेस आरक्षित कर सकता है,
स्वामित्व स्थानांतरित कर सकता है, किसी संसाधन का नाम बदल सकता है, मौजूदा
लिस्टिंग को छिपा या क्वारंटीन कर सकता है, कोई उपनाम या रीडायरेक्ट जोड़ सकता है,
अधिक प्रमाण मांग सकता है या अनुरोध अस्वीकार कर सकता है।

नेमस्पेस समीक्षा यह गारंटी नहीं देती कि हर मेल खाता नाम स्थानांतरित किया जाएगा।
स्टाफ सार्वजनिक साक्ष्य, मौजूदा उपयोग, सुरक्षा जोखिम और उपयोगकर्ता प्रभाव को
तौलता है।
