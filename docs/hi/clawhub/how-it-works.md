# ClawHub कैसे काम करता है

ClawHub, OpenClaw Skills और Plugin के लिए रजिस्ट्री परत है। यह उपयोगकर्ताओं को
पैकेज खोजने की जगह देता है, प्रकाशकों को संस्करण जारी करने की जगह देता है, और
OpenClaw को उन पैकेजों को सुरक्षित रूप से इंस्टॉल और अपडेट करने के लिए पर्याप्त मेटाडेटा देता है।

## रजिस्ट्री रिकॉर्ड

हर सार्वजनिक सूची एक रजिस्ट्री रिकॉर्ड होती है जिसमें शामिल हैं:

- मालिक और slug या पैकेज नाम
- एक या अधिक प्रकाशित संस्करण
- मेटाडेटा, सारांश, फाइलें, और स्रोत श्रेय
- changelog और tag जानकारी जैसे `latest`
- डाउनलोड, इंस्टॉल, और स्टार संकेत
- सुरक्षा स्कैन और मॉडरेशन स्थिति

सूची पेज उपयोगकर्ताओं के लिए वह प्रामाणिक जगह है जहां वे किसी Skill या
Plugin को इंस्टॉल करने से पहले यह देख सकते हैं कि वह क्या करने का दावा करता है।

## Skills

Skill एक संस्करणित टेक्स्ट बंडल है जिसका केंद्र `SKILL.md` होता है। इसमें
सहायक फाइलें, उदाहरण, टेम्पलेट, और स्क्रिप्ट शामिल हो सकते हैं।

ClawHub, Skill का नाम, विवरण, आवश्यकताएं, पर्यावरण चर, और मेटाडेटा समझने के लिए
`SKILL.md` frontmatter पढ़ता है। सटीक मेटाडेटा महत्वपूर्ण है क्योंकि यह उपयोगकर्ताओं को यह तय करने में मदद करता है कि Skill इंस्टॉल करना है या नहीं और
स्वचालित स्कैन को घोषित और देखे गए व्यवहार के बीच असंगतियां पहचानने में मदद करता है।

देखें [Skill प्रारूप](/hi/clawhub/skill-format)।

## Plugin

Plugin पैकेज किए गए OpenClaw एक्सटेंशन हैं। ClawHub पैकेज मेटाडेटा,
संगतता जानकारी, स्रोत लिंक, आर्टिफैक्ट, और संस्करण रिकॉर्ड संग्रहीत करता है।

जब OpenClaw, ClawHub से Plugin इंस्टॉल करता है, तो यह इंस्टॉल करने से पहले घोषित संगतता
मेटाडेटा की जांच करता है। पैकेज रिकॉर्ड में API संगतता,
न्यूनतम gateway संस्करण, host लक्ष्य, पर्यावरण आवश्यकताएं, और आर्टिफैक्ट
डाइजेस्ट शामिल हो सकते हैं।

जब आप चाहते हैं कि रजिस्ट्री प्रामाणिक स्रोत हो, तो स्पष्ट ClawHub इंस्टॉल स्रोत का उपयोग करें:

```bash
openclaw plugins install clawhub:<package>
```

## प्रकाशन

प्रकाशन एक नया अपरिवर्तनीय संस्करण रिकॉर्ड बनाता है। प्रकाशक प्रमाणीकृत रजिस्ट्री वर्कफ्लो के लिए `clawhub`
CLI का उपयोग करते हैं:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

अपलोड से पहले हल किए गए payload का पूर्वावलोकन करने के लिए dry run का उपयोग करें। इसके बाद सार्वजनिक पेज
प्रकाशित मेटाडेटा, फाइलें, स्रोत श्रेय, और स्कैन स्थिति दिखाते हैं।

## इंस्टॉल और अपडेट

OpenClaw इंस्टॉल कमांड ClawHub को पैकेज स्रोत के रूप में उपयोग करते हैं:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw इंस्टॉल स्रोत मेटाडेटा रिकॉर्ड करता है ताकि अपडेट बाद में उसी
रजिस्ट्री पैकेज को हल कर सकें। ClawHub CLI उन उपयोगकर्ताओं के लिए प्रत्यक्ष Skill इंस्टॉल और
अपडेट वर्कफ्लो का भी समर्थन करता है जो पूर्ण OpenClaw workspace के बाहर रजिस्ट्री-प्रबंधित Skill फोल्डर चाहते हैं।

## सुरक्षा स्थिति

ClawHub प्रकाशन के लिए खुला है, लेकिन रिलीज अभी भी अपलोड गेट,
स्वचालित जांच, उपयोगकर्ता रिपोर्ट, और मॉडरेटर कार्रवाई के अधीन हैं।

उपलब्ध होने पर सार्वजनिक पेज स्कैन सारांश दिखाते हैं। रोकी गई, छिपाई गई,
या अवरुद्ध सामग्री सार्वजनिक खोज और इंस्टॉल फ्लो से गायब हो सकती है, जबकि निदान के लिए
मालिक को दिखाई देती रहती है।

देखें [सुरक्षा](/clawhub/security), [सुरक्षा ऑडिट](/clawhub/security-audits),
[मॉडरेशन और खाता सुरक्षा](/hi/clawhub/moderation), और
[स्वीकार्य उपयोग](/clawhub/acceptable-usage)।

## API एक्सेस

ClawHub खोज, सर्च, पैकेज विवरण, और
डाउनलोड के लिए सार्वजनिक read API उपलब्ध कराता है। तृतीय-पक्ष कैटलॉग इन API का उपयोग कर सकते हैं जब वे
प्रामाणिक ClawHub सूची पर वापस लिंक करें, rate limit का सम्मान करें, और समर्थन का संकेत देने से बचें।

देखें [सार्वजनिक API](/hi/clawhub/api) और [HTTP API](/clawhub/http-api)।
