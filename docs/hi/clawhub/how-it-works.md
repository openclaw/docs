# ClawHub कैसे काम करता है

ClawHub OpenClaw Skills और Plugin के लिए रजिस्ट्री लेयर है। यह उपयोगकर्ताओं को
पैकेज खोजने की जगह देता है, प्रकाशकों को वर्शन रिलीज़ करने की जगह देता है, और
OpenClaw को उन पैकेजों को सुरक्षित रूप से इंस्टॉल और अपडेट करने के लिए पर्याप्त मेटाडेटा देता है।

## रजिस्ट्री रिकॉर्ड

हर सार्वजनिक लिस्टिंग एक रजिस्ट्री रिकॉर्ड है, जिसमें शामिल है:

- मालिक और स्लग या पैकेज नाम
- एक या अधिक प्रकाशित वर्शन
- मेटाडेटा, सारांश, फ़ाइलें, और स्रोत एट्रिब्यूशन
- चेंजलॉग और टैग जानकारी जैसे `latest`
- डाउनलोड, इंस्टॉल, और स्टार संकेत
- सुरक्षा स्कैन और मॉडरेशन स्थिति

लिस्टिंग पेज वह कैननिकल जगह है जहाँ उपयोगकर्ता इंस्टॉल करने से पहले देख सकते हैं कि कोई स्किल या
Plugin क्या करने का दावा करता है।

## Skills

स्किल `SKILL.md` पर केंद्रित एक वर्शनयुक्त टेक्स्ट बंडल है। इसमें
सहायक फ़ाइलें, उदाहरण, टेम्पलेट, और स्क्रिप्ट शामिल हो सकते हैं।

ClawHub स्किल नाम, विवरण, आवश्यकताएँ, एनवायरनमेंट वेरिएबल, और मेटाडेटा समझने के लिए
`SKILL.md` फ्रंटमैटर पढ़ता है। सटीक मेटाडेटा महत्वपूर्ण है क्योंकि यह उपयोगकर्ताओं को यह तय करने में मदद करता है कि
स्किल इंस्टॉल करनी है या नहीं, और स्वचालित स्कैन को घोषित और देखे गए व्यवहार के बीच अंतर पहचानने में
मदद करता है।

देखें [स्किल प्रारूप](/hi/clawhub/skill-format)।

## Plugin

Plugin पैकेज किए गए OpenClaw एक्सटेंशन हैं। ClawHub पैकेज मेटाडेटा,
संगतता जानकारी, स्रोत लिंक, आर्टिफ़ैक्ट, और वर्शन रिकॉर्ड संग्रहीत करता है।

जब OpenClaw ClawHub से कोई Plugin इंस्टॉल करता है, तो वह इंस्टॉल करने से पहले विज्ञापित संगतता
मेटाडेटा जाँचता है। पैकेज रिकॉर्ड में API संगतता,
न्यूनतम Gateway वर्शन, होस्ट लक्ष्य, एनवायरनमेंट आवश्यकताएँ, और आर्टिफ़ैक्ट
डाइजेस्ट शामिल हो सकते हैं।

जब आप चाहते हैं कि रजिस्ट्री सत्य का स्रोत हो, तो स्पष्ट ClawHub इंस्टॉल स्रोत का उपयोग करें:

```bash
openclaw plugins install clawhub:<package>
```

## प्रकाशन

प्रकाशन एक नया अपरिवर्तनीय वर्शन रिकॉर्ड बनाता है। प्रकाशक प्रमाणित रजिस्ट्री वर्कफ़्लो के लिए `clawhub`
CLI का उपयोग करते हैं:

```bash
clawhub skill publish ./my-skill
clawhub package publish <source> --family code-plugin --dry-run
clawhub package publish <source> --family code-plugin
```

अपलोड से पहले हल किए गए पेलोड का पूर्वावलोकन करने के लिए ड्राई रन का उपयोग करें। फिर सार्वजनिक पेज
प्रकाशित मेटाडेटा, फ़ाइलें, स्रोत एट्रिब्यूशन, और स्कैन स्थिति दिखाते हैं।

## इंस्टॉल और अपडेट

OpenClaw इंस्टॉल कमांड ClawHub को पैकेज स्रोत के रूप में उपयोग करते हैं:

```bash
openclaw skills install @openclaw/demo
openclaw plugins install clawhub:<package>
```

OpenClaw इंस्टॉल स्रोत मेटाडेटा रिकॉर्ड करता है ताकि अपडेट बाद में उसी
रजिस्ट्री पैकेज को हल कर सकें। ClawHub CLI उन उपयोगकर्ताओं के लिए सीधे स्किल इंस्टॉल और
अपडेट वर्कफ़्लो का भी समर्थन करता है जो पूर्ण OpenClaw वर्कस्पेस के बाहर रजिस्ट्री-प्रबंधित स्किल फ़ोल्डर चाहते हैं।

## सुरक्षा स्थिति

ClawHub प्रकाशन के लिए खुला है, लेकिन रिलीज़ फिर भी अपलोड गेट,
स्वचालित जाँच, उपयोगकर्ता रिपोर्ट, और मॉडरेटर कार्रवाई के अधीन हैं।

उपलब्ध होने पर सार्वजनिक पेज स्कैन सारांश दिखाते हैं। रोकी गई, छिपाई गई,
या ब्लॉक की गई सामग्री सार्वजनिक खोज और इंस्टॉल फ़्लो से गायब हो सकती है, जबकि डायग्नोस्टिक्स के लिए
मालिक को दिखाई देती रहती है।

देखें [सुरक्षा](/hi/clawhub/security), [सुरक्षा ऑडिट](/hi/clawhub/security-audits),
[मॉडरेशन और खाता सुरक्षा](/hi/clawhub/moderation), और
[स्वीकार्य उपयोग](/hi/clawhub/acceptable-usage)।

## API एक्सेस

ClawHub खोज, सर्च, पैकेज विवरण, और डाउनलोड के लिए सार्वजनिक रीड API उपलब्ध कराता है।
तृतीय-पक्ष कैटलॉग इन API का उपयोग कर सकते हैं, जब वे कैननिकल ClawHub लिस्टिंग पर वापस लिंक करते हैं,
रेट लिमिट का सम्मान करते हैं, और अनुमोदन का संकेत देने से बचते हैं।

देखें [सार्वजनिक API](/hi/clawhub/api) और [HTTP API](/hi/clawhub/http-api)।
