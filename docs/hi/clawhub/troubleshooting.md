# समस्या निवारण

## `clawhub login` ब्राउज़र खोलता है, लेकिन कभी पूरा नहीं होता

ब्राउज़र लॉगिन के दौरान CLI एक अल्पकालिक स्थानीय कॉलबैक सर्वर शुरू करता है।

- सुनिश्चित करें कि आपका ब्राउज़र `http://127.0.0.1:<port>/callback` तक पहुँच सकता है।
- यदि कॉलबैक कभी नहीं आता, तो स्थानीय फ़ायरवॉल, VPN और प्रॉक्सी नियमों की जाँच करें।
- हेडलेस परिवेशों में, ClawHub वेब UI में एक API टोकन बनाएँ और चलाएँ:

```bash
clawhub login --token clh_...
```

## `whoami` या `publish` से `Unauthorized` (401) मिलता है

- `clawhub login` से फिर से साइन इन करें।
- यदि आप कस्टम कॉन्फ़िग पथ का उपयोग करते हैं, तो पुष्टि करें कि `CLAWHUB_CONFIG_PATH` उस
  फ़ाइल की ओर संकेत करता है जिसमें आपका वर्तमान टोकन है।
- यदि आप API टोकन का उपयोग करते हैं, तो पुष्टि करें कि उसे वेब UI में निरस्त नहीं किया गया है।

## खोज या इंस्टॉल करने पर `Rate limit exceeded` (429) मिलता है

प्रतिक्रिया में पुनः प्रयास की जानकारी पढ़ें:

- `Retry-After`: पुनः प्रयास करने से पहले प्रतीक्षा के सेकंड।
- `RateLimit-Limit`: इस अनुरोध पर लागू सीमा।
- `RateLimit-Remaining`: हेडर मौजूद होने पर आपका सटीक शेष बजट। `429` पर यह `0` होता है।
- `RateLimit-Reset` या `X-RateLimit-Reset`: रीसेट का समय।

यदि कई उपयोगकर्ता एक ही एग्रेस IP साझा करते हैं, तो प्रत्येक व्यक्ति के केवल कुछ
अनुरोध भेजने पर भी अनाम IP सीमाएँ पूरी हो सकती हैं। जहाँ संभव हो, साइन इन करें और
बताए गए विलंब के बाद पुनः प्रयास करें।

## प्रॉक्सी के पीछे खोज या इंस्टॉल विफल होता है

CLI मानक प्रॉक्सी वेरिएबल का पालन करता है:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

समर्थित नामों में `HTTPS_PROXY`, `HTTP_PROXY`, `https_proxy` और
`http_proxy` शामिल हैं।

## कोई Skill खोज में दिखाई नहीं देती

- यदि आपको सटीक स्लग या स्वामी पृष्ठ पता है, तो उसकी जाँच करें।
- पुष्टि करें कि रिलीज़ सार्वजनिक है और स्कैन या मॉडरेशन के कारण रोकी नहीं गई है।
- यदि Skill आपकी है, तो साइन इन करके उसका निरीक्षण करें:

```bash
clawhub inspect @openclaw/demo
```

स्वामी को दिखाई देने वाले निदान स्कैन, अपलोड-गेट या मॉडरेशन स्थिति की व्याख्या कर सकते हैं।

## आवश्यक मेटाडेटा अनुपलब्ध होने के कारण प्रकाशन विफल होता है

Skills के लिए, `SKILL.md` फ्रंटमैटर की जाँच करें। आवश्यक परिवेश वेरिएबल और
टूल घोषित किए जाने चाहिए, ताकि उपयोगकर्ता और स्कैनर पैकेज को समझ सकें।

Plugins के लिए, `package.json` संगतता मेटाडेटा की जाँच करें। कोड-Plugin के प्रकाशन के लिए
`openclaw.compat.pluginApi` और `openclaw.build.openclawVersion` जैसे OpenClaw संगतता फ़ील्ड
आवश्यक होते हैं।

पहले प्रकाशन पेलोड का पूर्वावलोकन करें:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## GitHub स्वामी या स्रोत त्रुटि के कारण प्रकाशन विफल होता है

ClawHub पैकेजों को उनके प्रकाशकों से जोड़ने के लिए GitHub पहचान और स्रोत
एट्रिब्यूशन का उपयोग करता है।

- सुनिश्चित करें कि आपने उस GitHub खाते से साइन इन किया है जो पैकेज का स्वामी है या उसे
  प्रकाशित कर सकता है।
- जाँचें कि स्रोत URL सार्वजनिक है या ClawHub के लिए पहुँच योग्य है।
- GitHub स्रोतों के लिए, `owner/repo`, `owner/repo@ref` या पूर्ण GitHub URL का उपयोग करें।

## नेमस्पेस पर दावा या उसके आरक्षित होने के कारण प्रकाशन विफल होता है

यदि प्रकाशन इसलिए विफल होता है क्योंकि स्वामी हैंडल, संगठन नेमस्पेस, पैकेज स्कोप, Skill
स्लग या पैकेज नाम पर पहले से दावा किया गया है या वह आरक्षित है, तो पहले पुष्टि करें कि आप
नेमस्पेस से मेल खाने वाले स्वामी के साथ प्रकाशन कर रहे हैं। Plugin पैकेजों के लिए,
`@example-org/example-plugin` जैसे स्कोप किए गए नाम मेल खाने वाले
`example-org` स्वामी के रूप में प्रकाशित किए जाने चाहिए।

यदि आपको लगता है कि आपका संगठन, प्रोजेक्ट या ब्रांड उस नेमस्पेस का वास्तविक स्वामी है, लेकिन
आप वर्तमान ClawHub स्वामी को प्रबंधित नहीं कर सकते, तो सार्वजनिक और गैर-संवेदनशील प्रमाण के साथ
[संगठन / नेमस्पेस दावे की समस्या](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
खोलें। प्रमाण संबंधी मार्गदर्शन और सार्वजनिक समस्याओं में क्या शामिल नहीं करना चाहिए, इसके लिए
[संगठन और नेमस्पेस दावे](/hi/clawhub/namespace-claims) देखें।

## `sync` कहता है कि कोई Skill नहीं मिली

`sync` ऐसे फ़ोल्डर खोजता है जिनमें `SKILL.md` या `skill.md` हो।

इसे उन रूट की ओर इंगित करें जिन्हें आप स्कैन करना चाहते हैं:

```bash
clawhub sync --root /path/to/skills
```

यदि आप सुनिश्चित नहीं हैं कि क्या प्रकाशित होगा, तो पहले पूर्वावलोकन करें:

```bash
clawhub sync --all --dry-run --no-input
```

## स्थानीय परिवर्तनों के कारण `update` मना कर देता है

स्थानीय फ़ाइलें ClawHub को ज्ञात किसी भी संस्करण से मेल नहीं खातीं। इनमें से एक चुनें:

- स्थानीय संपादन रखें और अपडेट छोड़ दें।
- प्रकाशित संस्करण से अधिलेखित करें:

```bash
clawhub update @openclaw/demo --force
```

- अपनी संपादित प्रति को नए स्लग या फ़ोर्क के रूप में प्रकाशित करें।

## OpenClaw में Plugin इंस्टॉल विफल होता है

- स्पष्ट ClawHub स्रोत का उपयोग करें:

```bash
openclaw plugins install clawhub:<package>
```

- स्कैन स्थिति और संगतता मेटाडेटा के लिए पैकेज विवरण पृष्ठ की जाँच करें।
- पुष्टि करें कि आपका OpenClaw संस्करण पैकेज की घोषित
  संगतता सीमा को पूरा करता है।
- यदि पैकेज छिपा हुआ, रोका गया या ब्लॉक किया गया है, तो स्वामी द्वारा समस्या का समाधान किए जाने तक
  उसे इंस्टॉल करना संभव नहीं हो सकता है।

## सार्वजनिक API अनुरोध विफल होते हैं

- `429` पुनः प्रयास हेडर का पालन करें और सार्वजनिक सूची/खोज प्रतिक्रियाओं को कैश करें।
- उपयोगकर्ताओं को मानक ClawHub सूची से वापस लिंक करें।
- सार्वजनिक API सतह के बाहर छिपी, निजी, रोकी गई या मॉडरेशन द्वारा ब्लॉक की गई सामग्री को
  मिरर न करें।

एंडपॉइंट के विवरण के लिए [HTTP API](/clawhub/http-api) देखें।
