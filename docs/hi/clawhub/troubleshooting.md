# समस्या निवारण

## `clawhub login` ब्राउज़र खोलता है लेकिन कभी पूरा नहीं होता

CLI ब्राउज़र लॉगिन के दौरान एक अल्पकालिक स्थानीय कॉलबैक सर्वर शुरू करता है।

- सुनिश्चित करें कि आपका ब्राउज़र `http://127.0.0.1:<port>/callback` तक पहुँच सकता है।
- यदि कॉलबैक कभी नहीं आता है, तो स्थानीय फ़ायरवॉल, VPN, और प्रॉक्सी नियम जाँचें।
- हेडलेस वातावरणों में, ClawHub वेब UI में API टोकन बनाएँ और चलाएँ:

```bash
clawhub login --token clh_...
```

## `whoami` या `publish` `Unauthorized` (401) लौटाता है

- `clawhub login` से फिर से साइन इन करें।
- यदि आप कस्टम कॉन्फ़िग पथ का उपयोग करते हैं, तो पुष्टि करें कि `CLAWHUB_CONFIG_PATH` उस
  फ़ाइल की ओर इशारा करता है जिसमें आपका वर्तमान टोकन है।
- यदि आप API टोकन का उपयोग करते हैं, तो पुष्टि करें कि उसे वेब UI में निरस्त नहीं किया गया था।

## खोज या इंस्टॉल `Rate limit exceeded` (429) लौटाता है

प्रतिक्रिया में फिर से प्रयास करने की जानकारी पढ़ें:

- `Retry-After`: फिर से प्रयास करने से पहले प्रतीक्षा करने के सेकंड।
- `RateLimit-Limit`: इस अनुरोध पर लागू सीमा।
- `RateLimit-Remaining`: हेडर मौजूद होने पर आपका सटीक शेष बजट। `429` पर, यह `0` होता है।
- `RateLimit-Reset` या `X-RateLimit-Reset`: रीसेट समय।

यदि कई उपयोगकर्ता एक ही एग्रेस IP साझा करते हैं, तो अनाम IP सीमाएँ तब भी लग सकती हैं जब प्रत्येक
व्यक्ति केवल कुछ अनुरोध भेजता है। जहाँ संभव हो साइन इन करें और बताई गई देरी के बाद फिर प्रयास करें।

## खोज या इंस्टॉल प्रॉक्सी के पीछे विफल होता है

CLI मानक प्रॉक्सी वेरिएबल्स का सम्मान करता है:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

समर्थित नामों में `HTTPS_PROXY`, `HTTP_PROXY`, `https_proxy`, और
`http_proxy` शामिल हैं।

## कोई Skill खोज में दिखाई नहीं देता

- यदि आपको सटीक स्लग या स्वामी पेज पता है, तो उसे जाँचें।
- पुष्टि करें कि रिलीज़ सार्वजनिक है और स्कैन या मॉडरेशन द्वारा रोकी नहीं गई है।
- यदि आप Skill के स्वामी हैं, तो साइन इन करें और इसका निरीक्षण करें:

```bash
clawhub inspect @openclaw/demo
```

स्वामी-दृश्यमान निदान स्कैन, अपलोड-गेट, या मॉडरेशन स्थिति समझा सकते हैं।

## प्रकाशन विफल होता है क्योंकि आवश्यक मेटाडेटा अनुपस्थित है

Skills के लिए, `SKILL.md` फ्रंटमैटर जाँचें। आवश्यक एनवायरनमेंट वेरिएबल्स और
टूल्स घोषित होने चाहिए ताकि उपयोगकर्ता और स्कैनर पैकेज को समझ सकें।

Plugins के लिए, `package.json` संगतता मेटाडेटा जाँचें। Code-plugin प्रकाशनों को
OpenClaw संगतता फ़ील्ड चाहिए, जैसे `openclaw.compat.pluginApi` और
`openclaw.build.openclawVersion`।

पहले प्रकाशन पेलोड का पूर्वावलोकन करें:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## प्रकाशन GitHub स्वामी या स्रोत त्रुटि के साथ विफल होता है

ClawHub पैकेजों को उनके प्रकाशकों से जोड़ने के लिए GitHub पहचान और स्रोत श्रेय का उपयोग करता है।

- सुनिश्चित करें कि आप उस GitHub खाते से साइन इन हैं जो पैकेज का स्वामी है या उसे प्रकाशित कर सकता है।
- जाँचें कि स्रोत URL सार्वजनिक है या ClawHub के लिए पहुँच योग्य है।
- GitHub स्रोतों के लिए, `owner/repo`, `owner/repo@ref`, या पूरा GitHub URL उपयोग करें।

## प्रकाशन विफल होता है क्योंकि कोई नेमस्पेस दावा किया गया या आरक्षित है

यदि प्रकाशन इसलिए विफल होता है क्योंकि स्वामी हैंडल, संगठन नेमस्पेस, पैकेज स्कोप, Skill
स्लग, या पैकेज नाम पहले से दावा किया गया या आरक्षित है, तो पहले पुष्टि करें कि आप
उस स्वामी से प्रकाशित कर रहे हैं जो नेमस्पेस से मेल खाता है। Plugin पैकेजों के लिए,
`@example-org/example-plugin` जैसे स्कोप्ड नामों को मिलते-जुलते
`example-org` स्वामी के रूप में प्रकाशित किया जाना चाहिए।

यदि आपको लगता है कि आपका संगठन, प्रोजेक्ट, या ब्रांड वैध नेमस्पेस स्वामी है लेकिन
आप वर्तमान ClawHub स्वामी को प्रबंधित नहीं कर सकते, तो सार्वजनिक, गैर-संवेदनशील प्रमाण के साथ
[संगठन / नेमस्पेस दावा issue](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
खोलें। प्रमाण मार्गदर्शन और सार्वजनिक issues से क्या बाहर रखना है, इसके लिए
[संगठन और नेमस्पेस दावे](/clawhub/namespace-claims) देखें।

## `sync` कहता है कि कोई Skills नहीं मिलीं

`sync` `SKILL.md` या `skill.md` वाली फ़ोल्डर खोजता है।

इसे उन रूट्स की ओर निर्देशित करें जिन्हें आप स्कैन करना चाहते हैं:

```bash
clawhub sync --root /path/to/skills
```

यदि आप निश्चित नहीं हैं कि क्या प्रकाशित होगा, तो पहले पूर्वावलोकन करें:

```bash
clawhub sync --all --dry-run --no-input
```

## `update` स्थानीय बदलावों के कारण अस्वीकार करता है

स्थानीय फ़ाइलें ClawHub को ज्ञात किसी भी संस्करण से मेल नहीं खातीं। एक चुनें:

- स्थानीय संपादन रखें और अपडेट छोड़ें।
- प्रकाशित संस्करण से ओवरराइट करें:

```bash
clawhub update @openclaw/demo --force
```

- अपनी संपादित प्रति को नए स्लग या फ़ोर्क के रूप में प्रकाशित करें।

## OpenClaw में Plugin इंस्टॉल विफल होता है

- स्पष्ट ClawHub स्रोत उपयोग करें:

```bash
openclaw plugins install clawhub:<package>
```

- स्कैन स्थिति और संगतता मेटाडेटा के लिए पैकेज विवरण पेज जाँचें।
- पुष्टि करें कि आपका OpenClaw संस्करण पैकेज की विज्ञापित
  संगतता सीमा को पूरा करता है।
- यदि पैकेज छिपा हुआ, रोका गया, या अवरुद्ध है, तो स्वामी द्वारा समस्या हल किए जाने तक
  वह इंस्टॉल योग्य नहीं हो सकता।

## सार्वजनिक API अनुरोध विफल होते हैं

- `429` पुनः-प्रयास हेडर्स का सम्मान करें और सार्वजनिक सूची/खोज प्रतिक्रियाओं को कैश करें।
- उपयोगकर्ताओं को कैनॉनिकल ClawHub सूची से वापस लिंक करें।
- छिपी हुई, निजी, रोकी गई, या मॉडरेशन-अवरुद्ध सामग्री को सार्वजनिक API सतह के बाहर
  मिरर न करें।

एंडपॉइंट विवरणों के लिए [HTTP API](/clawhub/http-api) देखें।
