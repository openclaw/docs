# समस्या निवारण

## `clawhub login` ब्राउज़र खोलता है लेकिन कभी पूरा नहीं होता

CLI ब्राउज़र लॉगिन के दौरान एक अल्पकालिक स्थानीय कॉलबैक सर्वर शुरू करता है।

- सुनिश्चित करें कि आपका ब्राउज़र `http://127.0.0.1:<port>/callback` तक पहुंच सकता है।
- यदि कॉलबैक कभी नहीं आता है, तो स्थानीय फ़ायरवॉल, VPN, और प्रॉक्सी नियम जांचें।
- हेडलेस परिवेशों में, ClawHub वेब UI में एक API टोकन बनाएं और चलाएं:

```bash
clawhub login --token clh_...
```

## `whoami` या `publish` `Unauthorized` (401) लौटाता है

- `clawhub login` के साथ फिर से साइन इन करें।
- यदि आप कस्टम कॉन्फ़िग पथ का उपयोग करते हैं, तो पुष्टि करें कि `CLAWHUB_CONFIG_PATH` उस
  फ़ाइल की ओर इशारा करता है जिसमें आपका मौजूदा टोकन है।
- यदि आप API टोकन का उपयोग करते हैं, तो पुष्टि करें कि उसे वेब UI में निरस्त नहीं किया गया था।

## खोज या इंस्टॉल `Rate limit exceeded` (429) लौटाता है

प्रतिक्रिया में पुनः प्रयास की जानकारी पढ़ें:

- `Retry-After`: पुनः प्रयास करने से पहले प्रतीक्षा करने के सेकंड।
- `RateLimit-Limit`: इस अनुरोध पर लागू सीमा।
- `RateLimit-Remaining`: हेडर मौजूद होने पर आपका सटीक शेष बजट। `429` पर, यह `0` होता है।
- `RateLimit-Reset` या `X-RateLimit-Reset`: रीसेट समय।

यदि कई उपयोगकर्ता एक ही ईग्रेस IP साझा करते हैं, तो अनाम IP सीमाएं तब भी लग सकती हैं जब हर
व्यक्ति केवल कुछ अनुरोध भेजता है। जहां संभव हो साइन इन करें और बताई गई
देरी के बाद पुनः प्रयास करें।

## खोज या इंस्टॉल प्रॉक्सी के पीछे विफल होता है

CLI मानक प्रॉक्सी वेरिएबल्स का सम्मान करता है:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

समर्थित नामों में `HTTPS_PROXY`, `HTTP_PROXY`, `https_proxy`, और
`http_proxy` शामिल हैं।

## कोई Skill खोज में दिखाई नहीं देता

- यदि आपको सटीक स्लग या ओनर पेज पता है, तो उसे जांचें।
- पुष्टि करें कि रिलीज़ सार्वजनिक है और स्कैन या मॉडरेशन द्वारा रोकी नहीं गई है।
- यदि Skill आपकी है, तो साइन इन करें और उसका निरीक्षण करें:

```bash
clawhub inspect @openclaw/demo
```

ओनर-दृश्यमान डायग्नोस्टिक्स स्कैन, अपलोड-गेट, या मॉडरेशन स्थिति समझा सकते हैं।

## प्रकाशित करना विफल होता है क्योंकि आवश्यक मेटाडेटा अनुपस्थित है

Skills के लिए, `SKILL.md` frontmatter जांचें। आवश्यक परिवेश वेरिएबल्स और
टूल्स घोषित किए जाने चाहिए ताकि उपयोगकर्ता और स्कैनर पैकेज को समझ सकें।

Plugins के लिए, `package.json` संगतता मेटाडेटा जांचें। Code-plugin प्रकाशन के लिए
OpenClaw संगतता फ़ील्ड जैसे `openclaw.compat.pluginApi` और
`openclaw.build.openclawVersion` आवश्यक हैं।

पहले प्रकाशित पेलोड का पूर्वावलोकन करें:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## प्रकाशित करना GitHub ओनर या स्रोत त्रुटि के साथ विफल होता है

ClawHub पैकेजों को उनके प्रकाशकों से जोड़ने के लिए GitHub पहचान और स्रोत
एट्रिब्यूशन का उपयोग करता है।

- सुनिश्चित करें कि आप उस GitHub खाते से साइन इन हैं जो पैकेज का मालिक है या
  उसे प्रकाशित कर सकता है।
- जांचें कि स्रोत URL सार्वजनिक है या ClawHub के लिए पहुंच योग्य है।
- GitHub स्रोतों के लिए, `owner/repo`, `owner/repo@ref`, या पूरा GitHub URL उपयोग करें।

## प्रकाशित करना विफल होता है क्योंकि कोई नेमस्पेस क्लेम या रिज़र्व किया गया है

यदि प्रकाशन इसलिए विफल होता है क्योंकि ओनर हैंडल, संगठन नेमस्पेस, पैकेज स्कोप, Skill
स्लग, या पैकेज नाम पहले से क्लेम या रिज़र्व किया गया है, तो पहले पुष्टि करें कि आप
उस ओनर से प्रकाशित कर रहे हैं जो नेमस्पेस से मेल खाता है। Plugin पैकेजों के लिए,
`@example-org/example-plugin` जैसे स्कोप्ड नामों को मेल खाते
`example-org` ओनर के रूप में प्रकाशित किया जाना चाहिए।

यदि आपको लगता है कि आपका संगठन, प्रोजेक्ट, या ब्रांड सही नेमस्पेस ओनर है लेकिन
आप मौजूदा ClawHub ओनर को प्रबंधित नहीं कर सकते, तो सार्वजनिक, गैर-संवेदनशील प्रमाण के साथ
[संगठन / नेमस्पेस क्लेम मुद्दा](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
खोलें। प्रमाण मार्गदर्शन और सार्वजनिक मुद्दों से क्या बाहर रखना है, इसके लिए
[संगठन और नेमस्पेस क्लेम](/clawhub/namespace-claims) देखें।

## `sync` कहता है कि कोई Skills नहीं मिलीं

`sync` उन फ़ोल्डरों को खोजता है जिनमें `SKILL.md` या `skill.md` हो।

इसे उन रूट्स की ओर इंगित करें जिन्हें आप स्कैन करना चाहते हैं:

```bash
clawhub sync --root /path/to/skills
```

यदि आप अनिश्चित हैं कि क्या प्रकाशित होगा, तो पहले पूर्वावलोकन करें:

```bash
clawhub sync --all --dry-run --no-input
```

## `update` स्थानीय बदलावों के कारण इनकार करता है

स्थानीय फ़ाइलें ClawHub को ज्ञात किसी भी संस्करण से मेल नहीं खातीं। इनमें से एक चुनें:

- स्थानीय संपादन रखें और अपडेट छोड़ें।
- प्रकाशित संस्करण से ओवरराइट करें:

```bash
clawhub update @openclaw/demo --force
```

- अपनी संपादित कॉपी को नए स्लग या फ़ोर्क के रूप में प्रकाशित करें।

## OpenClaw में Plugin इंस्टॉल विफल होता है

- स्पष्ट ClawHub स्रोत का उपयोग करें:

```bash
openclaw plugins install clawhub:<package>
```

- स्कैन स्थिति और संगतता मेटाडेटा के लिए पैकेज विवरण पेज जांचें।
- पुष्टि करें कि आपका OpenClaw संस्करण पैकेज की घोषित
  संगतता रेंज को पूरा करता है।
- यदि पैकेज छिपा हुआ, रोका हुआ, या ब्लॉक किया हुआ है, तो ओनर के समस्या हल करने तक
  वह इंस्टॉल करने योग्य नहीं हो सकता।

## सार्वजनिक API अनुरोध विफल होते हैं

- `429` पुनः प्रयास हेडर्स का सम्मान करें और सार्वजनिक सूची/खोज प्रतिक्रियाओं को कैश करें।
- उपयोगकर्ताओं को canonical ClawHub लिस्टिंग पर वापस लिंक करें।
- छिपी, निजी, रोकी हुई, या मॉडरेशन-ब्लॉक की गई सामग्री को सार्वजनिक API सतह के बाहर
  मिरर न करें।

एंडपॉइंट विवरणों के लिए [HTTP API](/clawhub/http-api) देखें।
