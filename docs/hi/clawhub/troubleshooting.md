# समस्या निवारण

## `clawhub login` ब्राउज़र खोलता है लेकिन कभी पूरा नहीं होता

CLI ब्राउज़र लॉगिन के दौरान एक अल्पकालिक स्थानीय कॉलबैक सर्वर शुरू करता है।

- सुनिश्चित करें कि आपका ब्राउज़र `http://127.0.0.1:<port>/callback` तक पहुंच सकता है।
- यदि कॉलबैक कभी नहीं आता है, तो स्थानीय फ़ायरवॉल, VPN, और प्रॉक्सी नियमों की जांच करें।
- हेडलेस परिवेशों में, ClawHub वेब UI में एक API टोकन बनाएं और चलाएं:

```bash
clawhub login --token clh_...
```

## `whoami` या `publish` `Unauthorized` (401) लौटाता है

- `clawhub login` से फिर से साइन इन करें।
- यदि आप कोई कस्टम कॉन्फ़िग पथ उपयोग करते हैं, तो पुष्टि करें कि `CLAWHUB_CONFIG_PATH` उस
  फ़ाइल की ओर इशारा करता है जिसमें आपका मौजूदा टोकन है।
- यदि आप API टोकन उपयोग करते हैं, तो पुष्टि करें कि उसे वेब UI में निरस्त नहीं किया गया था।

## खोज या इंस्टॉल `Rate limit exceeded` (429) लौटाता है

प्रतिक्रिया में पुनः प्रयास की जानकारी पढ़ें:

- `Retry-After`: पुनः प्रयास करने से पहले प्रतीक्षा करने के सेकंड।
- `RateLimit-Limit`: इस अनुरोध पर लागू सीमा।
- `RateLimit-Remaining`: हेडर मौजूद होने पर आपका सटीक शेष बजट। `429` पर, यह `0` होता है।
- `RateLimit-Reset` या `X-RateLimit-Reset`: रीसेट समय।

यदि कई उपयोगकर्ता एक ईग्रेस IP साझा करते हैं, तो अनाम IP सीमाएं तब भी पूरी हो सकती हैं जब हर
व्यक्ति केवल कुछ अनुरोध भेजता हो। जहां संभव हो साइन इन करें और बताई गई
देरी के बाद पुनः प्रयास करें।

## प्रॉक्सी के पीछे खोज या इंस्टॉल विफल होता है

CLI मानक प्रॉक्सी वेरिएबल्स का सम्मान करता है:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

समर्थित नामों में `HTTPS_PROXY`, `HTTP_PROXY`, `https_proxy`, और
`http_proxy` शामिल हैं।

## कोई स्किल खोज में दिखाई नहीं देती

- यदि आपको सटीक स्लग या मालिक पेज पता है, तो उसे जांचें।
- पुष्टि करें कि रिलीज़ सार्वजनिक है और स्कैन या मॉडरेशन द्वारा रोकी नहीं गई है।
- यदि स्किल आपकी है, तो साइन इन करें और उसका निरीक्षण करें:

```bash
clawhub inspect @openclaw/demo
```

मालिक-दृश्यमान निदान स्कैन, अपलोड-गेट, या मॉडरेशन स्थिति समझा सकते हैं।

## प्रकाशन विफल होता है क्योंकि आवश्यक मेटाडेटा गायब है

स्किल्स के लिए, `SKILL.md` फ्रंटमैटर जांचें। आवश्यक पर्यावरण वेरिएबल्स और
टूल्स घोषित किए जाने चाहिए ताकि उपयोगकर्ता और स्कैनर पैकेज को समझ सकें।

Plugins के लिए, `package.json` संगतता मेटाडेटा जांचें। कोड-Plugin प्रकाशनों को
OpenClaw संगतता फ़ील्ड चाहिए जैसे `openclaw.compat.pluginApi` और
`openclaw.build.openclawVersion`।

पहले प्रकाशन पेलोड का पूर्वावलोकन करें:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## GitHub मालिक या स्रोत त्रुटि के कारण प्रकाशन विफल होता है

ClawHub पैकेजों को उनके प्रकाशकों से जोड़ने के लिए GitHub पहचान और स्रोत एट्रिब्यूशन का उपयोग करता है।

- सुनिश्चित करें कि आपने उस GitHub खाते से साइन इन किया है जो पैकेज का मालिक है या उसे प्रकाशित कर सकता है।
- जांचें कि स्रोत URL सार्वजनिक है या ClawHub के लिए पहुंच योग्य है।
- GitHub स्रोतों के लिए, `owner/repo`, `owner/repo@ref`, या पूरा GitHub URL उपयोग करें।

## प्रकाशन विफल होता है क्योंकि नेमस्पेस दावा किया गया या आरक्षित है

यदि कोई प्रकाशन इसलिए विफल होता है क्योंकि मालिक हैंडल, संगठन नेमस्पेस, पैकेज स्कोप, स्किल
स्लग, या पैकेज नाम पहले से दावा किया गया या आरक्षित है, तो पहले पुष्टि करें कि आप
उस मालिक से प्रकाशित कर रहे हैं जो नेमस्पेस से मेल खाता है। Plugin पैकेजों के लिए,
`@example-org/example-plugin` जैसे स्कोप्ड नामों को मेल खाते
`example-org` मालिक के रूप में प्रकाशित किया जाना चाहिए।

यदि आपको लगता है कि आपका संगठन, प्रोजेक्ट, या ब्रांड सही नेमस्पेस मालिक है लेकिन
आप मौजूदा ClawHub मालिक को प्रबंधित नहीं कर सकते, तो सार्वजनिक, गैर-संवेदनशील प्रमाण के साथ
[संगठन / नेमस्पेस दावा समस्या](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
खोलें। प्रमाण मार्गदर्शन और सार्वजनिक समस्याओं से क्या बाहर रखना है, इसके लिए
[संगठन और नेमस्पेस दावे](/clawhub/namespace-claims) देखें।

## `sync` कहता है कि कोई स्किल नहीं मिली

`sync` `SKILL.md` या `skill.md` वाली फ़ोल्डरों को खोजता है।

इसे उन रूट्स की ओर इंगित करें जिन्हें आप स्कैन करना चाहते हैं:

```bash
clawhub sync --root /path/to/skills
```

यदि आप अनिश्चित हैं कि क्या प्रकाशित होगा, तो पहले पूर्वावलोकन करें:

```bash
clawhub sync --all --dry-run --no-input
```

## स्थानीय बदलावों के कारण `update` इनकार करता है

स्थानीय फ़ाइलें ClawHub द्वारा ज्ञात किसी भी संस्करण से मेल नहीं खातीं। एक चुनें:

- स्थानीय संपादन रखें और अपडेट छोड़ दें।
- प्रकाशित संस्करण से ओवरराइट करें:

```bash
clawhub update @openclaw/demo --force
```

- अपनी संपादित कॉपी को नए स्लग या फोर्क के रूप में प्रकाशित करें।

## OpenClaw में Plugin इंस्टॉल विफल होता है

- एक स्पष्ट ClawHub स्रोत उपयोग करें:

```bash
openclaw plugins install clawhub:<package>
```

- स्कैन स्थिति और संगतता मेटाडेटा के लिए पैकेज विवरण पेज जांचें।
- पुष्टि करें कि आपका OpenClaw संस्करण पैकेज की विज्ञापित
  संगतता रेंज को संतुष्ट करता है।
- यदि पैकेज छिपा हुआ, रोका हुआ, या ब्लॉक किया गया है, तो मालिक द्वारा
  समस्या हल किए जाने तक यह इंस्टॉल करने योग्य नहीं हो सकता है।

## सार्वजनिक API अनुरोध विफल होते हैं

- `429` पुनः प्रयास हेडर का सम्मान करें और सार्वजनिक सूची/खोज प्रतिक्रियाओं को कैश करें।
- उपयोगकर्ताओं को कैनॉनिकल ClawHub लिस्टिंग पर वापस लिंक करें।
- छिपी हुई, निजी, रोकी हुई, या मॉडरेशन-ब्लॉक की गई सामग्री को सार्वजनिक
  API सतह के बाहर मिरर न करें।

एंडपॉइंट विवरणों के लिए [HTTP API](/clawhub/http-api) देखें।
