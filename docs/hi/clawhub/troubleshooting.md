# समस्या निवारण

## `clawhub login` ब्राउज़र खोलता है लेकिन कभी पूरा नहीं होता

CLI ब्राउज़र लॉगिन के दौरान एक अल्पकालिक स्थानीय कॉलबैक सर्वर शुरू करता है।

- सुनिश्चित करें कि आपका ब्राउज़र `http://127.0.0.1:<port>/callback` तक पहुंच सकता है।
- अगर कॉलबैक कभी नहीं आता है, तो स्थानीय फ़ायरवॉल, VPN, और प्रॉक्सी नियम जांचें।
- हेडलेस परिवेशों में, ClawHub वेब UI में एक API टोकन बनाएं और चलाएं:

```bash
clawhub login --token clh_...
```

## `whoami` या `publish` `Unauthorized` (401) लौटाता है

- `clawhub login` के साथ फिर से साइन इन करें।
- अगर आप कस्टम कॉन्फ़िग पथ इस्तेमाल करते हैं, तो पुष्टि करें कि `CLAWHUB_CONFIG_PATH` उस
  फ़ाइल की ओर संकेत करता है जिसमें आपका मौजूदा टोकन है।
- अगर आप API टोकन इस्तेमाल करते हैं, तो पुष्टि करें कि उसे वेब UI में रद्द नहीं किया गया है।

## खोज या इंस्टॉल `Rate limit exceeded` (429) लौटाता है

प्रतिक्रिया में पुनः प्रयास जानकारी पढ़ें:

- `Retry-After`: पुनः प्रयास करने से पहले प्रतीक्षा करने के सेकंड।
- `RateLimit-Limit`: इस अनुरोध पर लागू सीमा।
- `RateLimit-Remaining`: हेडर मौजूद होने पर आपका सटीक बचा हुआ बजट। `429` पर, यह `0` होता है।
- `RateLimit-Reset` या `X-RateLimit-Reset`: रीसेट समय।

अगर कई उपयोगकर्ता एक ही एग्रेस IP साझा करते हैं, तो हर व्यक्ति के केवल कुछ अनुरोध भेजने पर भी
अनाम IP सीमाएं पूरी हो सकती हैं। जहां संभव हो साइन इन करें और बताई गई देरी के बाद पुनः प्रयास करें।

## खोज या इंस्टॉल प्रॉक्सी के पीछे विफल होता है

CLI मानक प्रॉक्सी वेरिएबल्स का पालन करता है:

```bash
export HTTPS_PROXY=http://proxy.example.com:3128
clawhub search "my query"
```

समर्थित नामों में `HTTPS_PROXY`, `HTTP_PROXY`, `https_proxy`, और
`http_proxy` शामिल हैं।

## कोई Skill खोज में दिखाई नहीं देता

- अगर आपको सटीक स्लग या मालिक पेज पता है, तो उसे जांचें।
- पुष्टि करें कि रिलीज़ सार्वजनिक है और स्कैन या मॉडरेशन द्वारा रोकी नहीं गई है।
- अगर आप Skill के मालिक हैं, तो साइन इन करें और उसका निरीक्षण करें:

```bash
clawhub inspect @openclaw/demo
```

मालिक को दिखाई देने वाले निदान स्कैन, अपलोड-गेट, या मॉडरेशन स्थिति समझा सकते हैं।

## आवश्यक मेटाडेटा न होने के कारण प्रकाशन विफल होता है

Skills के लिए, `SKILL.md` frontmatter जांचें। आवश्यक पर्यावरण वेरिएबल्स और
टूल घोषित होने चाहिए ताकि उपयोगकर्ता और स्कैनर पैकेज को समझ सकें।

Plugins के लिए, `package.json` संगतता मेटाडेटा जांचें। code-plugin प्रकाशनों को
`openclaw.compat.pluginApi` और `openclaw.build.openclawVersion` जैसे OpenClaw संगतता फ़ील्ड चाहिए।

पहले प्रकाशन पेलोड का पूर्वावलोकन करें:

```bash
clawhub package publish <source> --family code-plugin --dry-run
```

## GitHub मालिक या स्रोत त्रुटि के कारण प्रकाशन विफल होता है

ClawHub पैकेजों को उनके प्रकाशकों से जोड़ने के लिए GitHub पहचान और स्रोत एट्रिब्यूशन का उपयोग करता है।

- सुनिश्चित करें कि आप उस GitHub खाते से साइन इन हैं जो पैकेज का मालिक है या उसे प्रकाशित कर सकता है।
- जांचें कि स्रोत URL सार्वजनिक है या ClawHub के लिए पहुंच योग्य है।
- GitHub स्रोतों के लिए, `owner/repo`, `owner/repo@ref`, या पूर्ण GitHub URL इस्तेमाल करें।

## namespace दावा किए जाने या आरक्षित होने के कारण प्रकाशन विफल होता है

अगर प्रकाशन इसलिए विफल होता है क्योंकि मालिक हैंडल, संगठन namespace, पैकेज स्कोप, Skill
स्लग, या पैकेज नाम पहले से दावा किया गया या आरक्षित है, तो पहले पुष्टि करें कि आप
namespace से मेल खाने वाले मालिक के साथ प्रकाशित कर रहे हैं। Plugin पैकेजों के लिए,
`@example-org/example-plugin` जैसे स्कोप्ड नामों को मेल खाते
`example-org` मालिक के रूप में प्रकाशित किया जाना चाहिए।

अगर आपको लगता है कि आपका संगठन, प्रोजेक्ट, या ब्रांड वैध namespace मालिक है लेकिन
आप मौजूदा ClawHub मालिक को प्रबंधित नहीं कर सकते, तो सार्वजनिक, गैर-संवेदनशील प्रमाण के साथ
[संगठन / namespace दावा issue](https://github.com/openclaw/clawhub/issues/new?template=org-namespace-claim.yml)
खोलें। प्रमाण मार्गदर्शन और सार्वजनिक issues से क्या बाहर रखना है, इसके लिए
[संगठन और namespace दावे](/clawhub/namespace-claims) देखें।

## `sync` कहता है कि कोई Skills नहीं मिले

`sync` `SKILL.md` या `skill.md` वाली फ़ोल्डर खोजता है।

उसे उन रूट्स की ओर निर्देशित करें जिन्हें आप स्कैन करना चाहते हैं:

```bash
clawhub sync --root /path/to/skills
```

अगर आपको निश्चित नहीं है कि क्या प्रकाशित होगा, तो पहले पूर्वावलोकन करें:

```bash
clawhub sync --all --dry-run --no-input
```

## स्थानीय बदलावों के कारण `update` मना करता है

स्थानीय फ़ाइलें ClawHub के ज्ञात किसी भी संस्करण से मेल नहीं खातीं। एक विकल्प चुनें:

- स्थानीय संपादन रखें और अपडेट छोड़ दें।
- प्रकाशित संस्करण से ओवरराइट करें:

```bash
clawhub update @openclaw/demo --force
```

- अपनी संपादित कॉपी को नए स्लग या fork के रूप में प्रकाशित करें।

## OpenClaw में Plugin इंस्टॉल विफल होता है

- स्पष्ट ClawHub स्रोत इस्तेमाल करें:

```bash
openclaw plugins install clawhub:<package>
```

- स्कैन स्थिति और संगतता मेटाडेटा के लिए पैकेज विवरण पेज जांचें।
- पुष्टि करें कि आपका OpenClaw संस्करण पैकेज की घोषित
  संगतता सीमा को पूरा करता है।
- अगर पैकेज छिपा हुआ, रोका हुआ, या ब्लॉक किया हुआ है, तो मालिक द्वारा समस्या हल किए जाने तक
  वह इंस्टॉल करने योग्य नहीं हो सकता।

## सार्वजनिक API अनुरोध विफल होते हैं

- `429` पुनः प्रयास हेडर्स का सम्मान करें और सार्वजनिक सूची/खोज प्रतिक्रियाओं को कैश करें।
- उपयोगकर्ताओं को canonical ClawHub सूची से वापस लिंक करें।
- छिपी, निजी, रोकी हुई, या मॉडरेशन-ब्लॉक की गई सामग्री को सार्वजनिक API सतह के बाहर
  मिरर न करें।

एंडपॉइंट विवरण के लिए [HTTP API](/clawhub/http-api) देखें।
