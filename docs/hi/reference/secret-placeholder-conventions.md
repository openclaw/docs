# सीक्रेट प्लेसहोल्डर परंपराएँ

ऐसे प्लेसहोल्डर का उपयोग करें जो मनुष्यों के लिए पठनीय हों, लेकिन वास्तविक सीक्रेट जैसे न दिखें।

## अनुशंसित शैली

- `example-openai-key-not-real` या `example-discord-bot-token` जैसे वर्णनात्मक मानों को प्राथमिकता दें।
- शेल स्निपेट के लिए, इनलाइन टोकन जैसी स्ट्रिंग के बजाय `${OPENAI_API_KEY}` को प्राथमिकता दें।
- उदाहरणों को स्पष्ट रूप से नकली और उद्देश्य (प्रोवाइडर, चैनल, प्रमाणीकरण प्रकार) तक सीमित रखें।

## दस्तावेज़ों में इन पैटर्न से बचें

- शाब्दिक PEM निजी-कुंजी हेडर या फुटर टेक्स्ट।
- ऐसे प्रीफ़िक्स जो सक्रिय क्रेडेंशियल जैसे दिखें, उदाहरण के लिए `sk-...`, `xoxb-...`, `AKIA...`।
- रनटाइम लॉग से कॉपी किए गए वास्तविक दिखने वाले बेयरर टोकन।

## उदाहरण

```bash
# अच्छा
export OPENAI_API_KEY="example-openai-key-not-real"

# बेहतर (जब दस्तावेज़ env वायरिंग के बारे में हो)
export OPENAI_API_KEY="${OPENAI_API_KEY}"
```
