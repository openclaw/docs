macOS अनुमति अनुदान नाज़ुक होते हैं। TCC किसी अनुमति अनुदान को ऐप के कोड हस्ताक्षर, बंडल पहचानकर्ता और डिस्क पर उसके पथ से संबद्ध करता है। यदि इनमें से कुछ भी बदलता है, तो macOS ऐप को नया मानता है और संकेतों को हटा या छिपा सकता है।

## स्थिर अनुमतियों के लिए आवश्यकताएँ

- समान पथ: ऐप को किसी निश्चित स्थान से चलाएँ (OpenClaw के लिए, `dist/OpenClaw.app`)।
- समान बंडल पहचानकर्ता: OpenClaw का बंडल ID `ai.openclaw.mac` है; इसे बदलने से एक नई अनुमति पहचान बनती है।
- हस्ताक्षरित ऐप: अहस्ताक्षरित या तदर्थ हस्ताक्षरित बिल्ड अनुमतियों को बनाए नहीं रखते।
- सुसंगत हस्ताक्षर: वास्तविक Apple Development या Developer ID प्रमाणपत्र का उपयोग करें, ताकि दोबारा बिल्ड करने पर भी हस्ताक्षर स्थिर रहे।

तदर्थ हस्ताक्षर प्रत्येक बिल्ड के साथ एक नई पहचान उत्पन्न करते हैं। macOS पिछले अनुदानों को भूल जाता है और पुरानी प्रविष्टियाँ हटाए जाने तक संकेत पूरी तरह गायब हो सकते हैं।

## Node और CLI रनटाइम के लिए Accessibility अनुदान

किसी सामान्य `node` बाइनरी के बजाय OpenClaw.app, Peekaboo.app या अपने अलग बंडल पहचानकर्ता वाले किसी अन्य हस्ताक्षरित सहायक को Accessibility देने को प्राथमिकता दें।

macOS TCC Accessibility उस प्रक्रिया की कोड पहचान को प्रदान करता है जिसे वह देखता है। यदि Homebrew, nvm, pnpm या npm कार्यप्रवाह के कारण किसी साझा `node` निष्पादन योग्य को Accessibility मिलती है, तो उसी निष्पादन योग्य के माध्यम से लॉन्च किया गया कोई भी JavaScript पैकेज GUI स्वचालन विशेषाधिकार प्राप्त कर सकता है।

System Settings में किसी `node` प्रविष्टि को उस Node रनटाइम के लिए व्यापक अनुमति मानें, न कि किसी एक npm पैकेज की अनुमति। `node` को Accessibility देने से बचें, जब तक कि आपको उस सटीक Node इंस्टॉलेशन के माध्यम से लॉन्च की जाने वाली प्रत्येक स्क्रिप्ट और पैकेज पर भरोसा न हो।

यदि आपने गलती से `node` को Accessibility दे दी है, तो System Settings -> Privacy & Security -> Accessibility से वह प्रविष्टि हटाएँ। फिर उस हस्ताक्षरित ऐप या सहायक को अनुमति दें जिसे UI स्वचालन का स्वामी होना चाहिए।

## संकेत गायब होने पर पुनर्प्राप्ति जाँच-सूची

1. ऐप बंद करें।
2. System Settings -> Privacy & Security में ऐप की प्रविष्टि हटाएँ।
3. ऐप को उसी पथ से फिर लॉन्च करें और अनुमतियाँ दोबारा प्रदान करें।
4. यदि संकेत फिर भी दिखाई नहीं देता, तो `tccutil` से TCC प्रविष्टियाँ रीसेट करें और फिर प्रयास करें।
5. कुछ अनुमतियाँ macOS को पूरी तरह रीस्टार्ट करने के बाद ही दोबारा दिखाई देती हैं।

रीसेट के उदाहरण (OpenClaw के बंडल ID, `ai.openclaw.mac`, का उपयोग करते हुए):

```bash
sudo tccutil reset Accessibility ai.openclaw.mac
sudo tccutil reset ScreenCapture ai.openclaw.mac
sudo tccutil reset AppleEvents
```

## फ़ाइलों और फ़ोल्डरों की अनुमतियाँ (Desktop/Documents/Downloads)

macOS टर्मिनल/पृष्ठभूमि प्रक्रियाओं के लिए Desktop, Documents और Downloads तक पहुँच को भी नियंत्रित कर सकता है। यदि फ़ाइल पढ़ना या डायरेक्टरी सूचीबद्ध करना अटक जाता है, तो उसी प्रक्रिया संदर्भ को पहुँच प्रदान करें जो फ़ाइल संचालन करता है (उदाहरण के लिए Terminal/iTerm, LaunchAgent से लॉन्च किया गया ऐप या SSH प्रक्रिया)।

वैकल्पिक उपाय: यदि आप प्रत्येक फ़ोल्डर के लिए अलग-अलग अनुमति अनुदान से बचना चाहते हैं, तो फ़ाइलों को OpenClaw कार्यस्थान (`~/.openclaw/workspace`) में ले जाएँ।

यदि आप अनुमतियों का परीक्षण कर रहे हैं, तो हमेशा वास्तविक प्रमाणपत्र से हस्ताक्षर करें। तदर्थ बिल्ड केवल त्वरित स्थानीय रन के लिए स्वीकार्य हैं, जहाँ अनुमतियाँ मायने नहीं रखतीं।
