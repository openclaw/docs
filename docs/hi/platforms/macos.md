OpenClaw **मेनू बार सहायक** के रूप में macOS ऐप: नेटिव ट्रे UI, macOS
अनुमति संकेत, सूचनाएँ, WebChat, वॉइस इनपुट, Canvas और
Mac पर होस्ट किए गए Node टूल, जैसे `system.run`।

पूरी विंडो खोले बिना Spotlight-शैली के मुख्य-सत्र कंपोज़र के लिए **त्वरित चैट** का उपयोग करें। डिफ़ॉल्ट रूप से Option-Space (⌥Space) दबाएँ, इसे मेनू बार मेनू से चुनें या **सेटिंग्स → सामान्य** में कोई दूसरा शॉर्टकट रिकॉर्ड करें।

केवल CLI और Gateway की आवश्यकता है? [आरंभ करना](/hi/start/getting-started) से शुरू करें।

## डाउनलोड

macOS ऐप बिल्ड [OpenClaw GitHub रिलीज़](https://github.com/openclaw/openclaw/releases) से प्राप्त करें।
जब किसी रिलीज़ में macOS ऐप एसेट शामिल हों, तो इन्हें देखें:

- `OpenClaw-<version>.dmg` (अनुशंसित)
- `OpenClaw-<version>.zip`

कुछ रिलीज़ में केवल CLI, साक्ष्य या Windows एसेट शामिल होते हैं। यदि नवीनतम रिलीज़
में macOS ऐप एसेट नहीं है, तो ऐसा एसेट शामिल करने वाली सबसे नई रिलीज़ का उपयोग करें या
[macOS डेवलपमेंट सेटअप](/hi/platforms/mac/dev-setup) के साथ स्रोत से बिल्ड करें।

## पहली बार चलाना

1. **OpenClaw.app** इंस्टॉल करके लॉन्च करें।
2. स्थानीय Gateway के लिए **This Mac** चुनें या किसी रिमोट Gateway से कनेक्ट करें।
3. ऐप द्वारा मेल खाने वाला CLI रनटाइम इंस्टॉल किए जाने तक प्रतीक्षा करें। स्थानीय मोड में यह
   Gateway को भी इंस्टॉल और प्रारंभ करता है।
4. लाइव मॉडल जाँच के साथ इनफ़रेंस स्थापित करें। जाँच सफल होने के बाद OpenClaw
   शेष सेटअप संभालता है।
5. macOS अनुमति चेकलिस्ट पूरी करें और ऑनबोर्डिंग परीक्षण संदेश भेजें।

यदि ऐप किसी मौजूदा Gateway तक पहुँचता है जिसके डिफ़ॉल्ट एजेंट में मॉडल कॉन्फ़िगर किया गया
है, तो वह उस Gateway को पहले से सेट अप मानता है, प्रदाता ऑनबोर्डिंग और
OpenClaw को छोड़ देता है और डैशबोर्ड खोलता है। यदि Gateway कनेक्ट नहीं हो सकता या उसके
डिफ़ॉल्ट एजेंट में कोई मॉडल नहीं है, तो पुनर्प्राप्ति के लिए इनफ़रेंस ऑनबोर्डिंग उपलब्ध
रहती है।

CLI/Gateway सेटअप पथ के लिए [आरंभ करना](/hi/start/getting-started) का उपयोग करें।
अनुमति पुनर्प्राप्ति के लिए [macOS अनुमतियाँ](/hi/platforms/mac/permissions) का उपयोग करें।

## अपडेट

डैशबोर्ड अपडेट कार्ड बताता है कि ऐप क्या अपडेट करेगा:

- **Mac ऐप + Gateway अपडेट करें** का अर्थ है कि साइन किया हुआ ऐप स्थानीय launchd
  Gateway का स्वामी है। Sparkle पहले ऐप को अपडेट करता है; पुनः लॉन्च होने के बाद ऐप स्वचालित रूप से
  अपने Gateway को मेल खाने वाले संस्करण पर अपडेट करके पुनः प्रारंभ करता है और फिर
  कनेक्शन सत्यापित करता है।
- **Gateway अपडेट करें** का अर्थ है कि ऐप किसी रिमोट Gateway, मैन्युअल रूप से
  प्रबंधित स्थानीय Gateway या किसी अन्य ऐसे इंस्टॉलेशन से कनेक्ट है जिसका ऐप स्वामी नहीं है। बटन
  Mac ऐप को बदलने के बजाय उस Gateway के सामान्य अपडेट प्रवाह को चलाता है।

विफल समन्वित अपडेट पुनः प्रयास,
[अपडेट मार्गदर्शिका](/hi/install/updating) और Discord कार्रवाइयों के साथ अपनी सेटअप-शैली की विंडो में बना रहता है। स्वचालित सुधार कभी भी
नए Gateway को पुराने संस्करण पर नहीं ले जाता या `extended-stable` चैनल पिन को ओवरराइड नहीं करता।

सफल अपडेट के बाद ऐप सबसे हाल में किसी व्यक्ति द्वारा उपयोग किए गए,
शीर्ष-स्तरीय प्रत्यक्ष सत्र को खोजता है और उस एजेंट को एक बार का अपडेट इवेंट देता है। Heartbeat
और Cron गतिविधि इस चयन को प्रभावित नहीं करती। इसके बाद एजेंट उस वार्तालाप से आपका फिर से स्वागत कर सकता है
जिसका उपयोग करने की संभावना सबसे अधिक थी। रिमोट मोड में ऐप
केवल स्थानीय Mac Node रनटाइम अपडेट करता है और जब रिमोट Gateway ऐप से पुराना हो, तो
सूचना छोड़ देता है।

Sparkle, Gateway की `update.channel` सेटिंग का अनुसरण करता है। `beta` और `dev` बीटा
ऐप बिल्ड के लिए सहमति देते हैं; `stable`, `extended-stable` और अनुपस्थित या अज्ञात मान
स्थिर ऐप बिल्ड पर बने रहते हैं।

## डैशबोर्ड लिंक खोलना

macOS ऐप के एम्बेडेड डैशबोर्ड में, किसी बाहरी वेब लिंक पर क्लिक करने से वह विंडो की आधी चौड़ाई वाले, आकार बदले जा सकने योग्य ब्राउज़र साइडबार में खुलता है, जबकि डैशबोर्ड नेविगेशन दिखाई देता रहता है। दूसरी चौड़ाई चुनने के लिए विभाजक को खींचें; ऐप इसे याद रखता है। प्रत्येक लिंक अपने अलग टैब में खुलता है, कई पेज खुले होने पर टैब पट्टी दिखाई देती है, और उसी लिंक पर दोबारा क्लिक करने से उसका मौजूदा टैब फिर से उपयोग होता है। टैब का क्रम बदलने के लिए उन्हें खींचें, टैब बंद करने वाले बटन या मध्य-क्लिक से उन्हें बंद करें, और **Open in Default Browser**, **Copy Link**, **Reload**, **Close Tab**, तथा **Close Other Tabs** के लिए किसी टैब पर राइट-क्लिक करें। विंडो के टाइटलबार के पीछे/आगे नियंत्रण और ट्रैकपैड स्वाइप डैशबोर्ड इतिहास में नेविगेट करते हैं; साइडबार के अपने पीछे/आगे नियंत्रण सक्रिय टैब के इतिहास में नेविगेट करते हैं। साइडबार में रीलोड, डिफ़ॉल्ट ब्राउज़र में खोलने और बंद करने के नियंत्रण भी हैं।

टाइटलबार नियंत्रण ऐप साइडबार का अनुसरण करते हैं: जब वह विस्तृत होता है, तो पीछे/आगे नियंत्रण साइडबार टॉगल के पास उसके दाएँ किनारे पर होते हैं; जब वह संकुचित होता है, तो उनकी जगह खोज बटन (कमांड पैलेट खोलता है) और नया-सत्र बटन आ जाते हैं।

**Open in Sidebar**, **Open in Default Browser**, या **Copy Link** चुनने के लिए किसी बाहरी लिंक पर राइट-क्लिक करें। संशोधक कुंजियों के साथ किए गए क्लिक और डैशबोर्ड से उपयोगकर्ता द्वारा सक्रिय किए गए नई-विंडो लिंक डिफ़ॉल्ट ब्राउज़र में ही खुलते रहते हैं; साइडबार के अंदर के नई-विंडो लिंक नए साइडबार टैब के रूप में खुलते हैं। ब्राउज़र में होस्ट किए गए सामान्य Control UI पेज ब्राउज़र के सामान्य लिंक और संदर्भ-मेन्यू व्यवहार को बनाए रखते हैं।

## ब्राउज़र लॉगिन आयात करना

जब ऐप स्थानीय Gateway के साथ चल रहा हो और ब्राउज़र साइडबार पहली बार खुले, तो Mac पर कुकी वाले Chrome-परिवार के किसी प्रोफ़ाइल के मौजूद होने पर डैशबोर्ड हटाया जा सकने वाला बैनर दिखाता है। बैनर उन कुकी को एजेंटों द्वारा ब्राउज़िंग के लिए उपयोग की जाने वाली अलग-थलग प्रबंधित प्रोफ़ाइल में कॉपी करने का विकल्प देता है। इसके **Import** नियंत्रण से कोई प्रोफ़ाइल चुनें (Touch ID आवश्यक हो सकता है); प्रगति और आयातित कुकी की संख्या उसी स्थान पर दिखाई देती है, और केवल कुकी कॉपी की जाती हैं—पासवर्ड कभी भी स्रोत ब्राउज़र से बाहर नहीं जाते। बैनर को हटाने पर यह चयन दर्ज हो जाता है; **Settings → General → Browser login → Import…** इसे किसी भी समय फिर से उपलब्ध कराता है। अंतर्निहित आयात प्रवाह और `browser.allowSystemProfileImport` गेट के लिए [ब्राउज़र](/hi/cli/browser) देखें।

दोनों मोड के लिए इंस्टॉल किया हुआ `openclaw` CLI आवश्यक है, क्योंकि ऐप उसके नोड-होस्ट
रनटाइम का पुनः उपयोग करता है। नए Mac पर, ऐप मेल खाने वाला CLI स्वचालित रूप से इंस्टॉल करता है; इसके बाद स्थानीय
मोड Gateway विज़ार्ड शुरू करता है, जबकि रिमोट मोड दूसरा स्थानीय Gateway शुरू किए बिना चुने गए
Gateway से कनेक्ट होता है।
मैन्युअल पुनर्प्राप्ति के लिए [macOS पर Gateway](/hi/platforms/mac/bundled-gateway) देखें।

## ऐप किन चीज़ों का स्वामी है

- मेन्यू बार स्थिति, सूचनाएँ, स्वास्थ्य, WebChat, और फ़्लोटिंग Quick Chat बार।
- स्क्रीन, माइक्रोफ़ोन, वाक्, ऑटोमेशन, और अभिगम्यता के लिए macOS अनुमति संकेत।
- एक Mac नोड, जो नेटिव Canvas, कैमरा/स्क्रीन कैप्चर, सूचनाएँ,
  स्थान, और कंप्यूटर नियंत्रण को CLI नोड होस्ट के सिस्टम, ब्राउज़र,
  Plugin, skill, और MCP कमांड के साथ संयोजित करता है।
- Mac पर होस्ट किए गए कमांड के लिए Exec अनुमोदन संकेत।
- अनुमोदित शेल कमांड के लिए ऐप-संदर्भ निष्पादन, जो ऐप के macOS
  अनुमति श्रेय को बनाए रखता है, जबकि CLI रनटाइम साझा नोड नीति का स्वामी होता है।
- रिमोट-मोड SSH टनल या प्रत्यक्ष Gateway कनेक्शन।

एम्बेडेड Control UI में, **Settings → Notifications** ब्राउज़र पुश के बजाय ऐप की नेटिव
सूचना अनुमति दिखाता है, क्योंकि ऐप सूचनाएँ नेटिव रूप से वितरित करता है।

ऐप Gateway या सामान्य CLI दस्तावेज़ों को प्रतिस्थापित **नहीं** करता। Gateway
कॉन्फ़िगरेशन, प्रदाता, plugins, चैनल, टूल, और सुरक्षा के अपने
अलग दस्तावेज़ हैं।
