OpenClaw अंतर्निहित एजेंट रनटाइम का स्वामी है। रनटाइम कोड `src/agents/` के अंतर्गत, मॉडल/प्रोवाइडर ट्रांसपोर्ट `src/llm/` के अंतर्गत रहता है, और Plugin-संबंधी अनुबंध `openclaw/plugin-sdk/*` बैरल के माध्यम से उपलब्ध कराए जाते हैं।

## रनटाइम संरचना

| पथ                                | स्वामित्व                                                                                                                                                                                                                      |
| ----------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `src/agents/embedded-agent-runner/` | अंतर्निहित प्रयास लूप (`run.ts`, `run/`), मॉडल चयन और प्रोवाइडर सामान्यीकरण (`model*.ts`), प्रत्येक प्रोवाइडर के अनुरोध पैरामीटर (`extra-params.*`), Compaction, ट्रांसक्रिप्ट और सत्र वायरिंग।                            |
| `src/agents/sessions/`              | सत्र स्थायित्व (`session-manager.ts`), संसाधन खोज (`package-manager.ts`, `resource-loader.ts`), सत्र के भीतर `extensions` लोडिंग, प्रॉम्प्ट टेम्पलेट, Skills, थीम और TUI-समर्थित टूल रेंडरर (`tools/`)। |
| `packages/agent-core/`              | पुनः उपयोग योग्य एजेंट कोर (`@openclaw/agent-core`): एजेंट लूप, हार्नेस प्रकार, संदेश, Compaction सहायक, प्रॉम्प्ट टेम्पलेट, Skills और सत्र भंडारण अनुबंध।                                                           |
| `src/agents/runtime/`               | OpenClaw फ़साड, जो `@openclaw/agent-core` को Plugin SDK LLM रनटाइम से जोड़ता है और स्थानीय प्रॉक्सी उपयोगिताओं सहित उसे पुनः निर्यात करता है।                                                                                             |
| `src/agents/agent-tools*.ts`        | OpenClaw-स्वामित्व वाली टूल परिभाषाएँ, पैरामीटर स्कीमा, टूल नीति, टूल-कॉल से पहले/बाद के अडैप्टर और होस्ट/सैंडबॉक्स संपादन टूल।                                                                                            |
| `src/agents/agent-hooks/`           | अंतर्निहित रनटाइम हुक: Compaction सुरक्षा उपाय, Compaction निर्देश, कॉन्टेक्स्ट छँटाई।                                                                                                                                   |
| `src/agents/harness/`               | अंतर्निहित और Plugin-पंजीकृत हार्नेस के लिए हार्नेस रजिस्ट्री, चयन नीति और जीवनचक्र।                                                                                                                       |
| `src/llm/`                          | मॉडल/प्रोवाइडर रजिस्ट्री, ट्रांसपोर्ट सहायक और प्रोवाइडर-विशिष्ट स्ट्रीम कार्यान्वयन (`src/llm/providers/`)।                                                                                                          |

## सीमाएँ

कोर, OpenClaw मॉड्यूल और SDK बैरल के माध्यम से अंतर्निहित रनटाइम को कॉल करता है; कोई बाहरी एजेंट फ़्रेमवर्क पैकेज शेष नहीं है। Plugins दस्तावेज़ीकृत `openclaw/plugin-sdk/*` एंट्रीपॉइंट का उपयोग करते हैं और `src/**` आंतरिक घटकों को आयात नहीं करते।

`@earendil-works/pi-tui` तृतीय-पक्ष निर्भरता बनी हुई है: स्थानीय TUI और सत्र टूल रेंडरर द्वारा उपयोग की जाने वाली एक टर्मिनल कॉम्पोनेंट टूलकिट। इसे आंतरिक बनाना एक अलग वेंडरिंग प्रयास होगा।

## मैनिफ़ेस्ट

संसाधन पैकेज `package.json` मेटाडेटा में OpenClaw संसाधन घोषित करते हैं। प्रविष्टियाँ पैकेज रूट के सापेक्ष फ़ाइल पथ या ग्लॉब होती हैं:

```json
{
  "openclaw": {
    "extensions": ["extensions/index.ts"],
    "skills": ["skills/*.md"],
    "prompts": ["prompts/*.md"],
    "themes": ["themes/*.json"]
  }
}
```

मैनिफ़ेस्ट में सूचीबद्ध नहीं किए गए संसाधन प्रकार पारंपरिक `extensions/`, `skills/`, `prompts/` और `themes/` निर्देशिकाओं की खोज पर वापस जाते हैं।

## रनटाइम चयन

- अंतर्निहित रनटाइम आईडी `openclaw` है। विरासती उपनाम `pi` को `openclaw` में सामान्यीकृत किया जाता है; `codex-app-server` को `codex` में सामान्यीकृत किया जाता है।
- Plugin हार्नेस अतिरिक्त रनटाइम आईडी पंजीकृत करते हैं (उदाहरण के लिए `codex`)।
- रनटाइम नीति मॉडल/प्रोवाइडर-स्कोप वाला `agentRuntime.id` कॉन्फ़िगरेशन है (मॉडल प्रविष्टि को प्रोवाइडर प्रविष्टि पर प्राथमिकता मिलती है)। सेट न किया गया मान या `default`, `auto` में रिज़ॉल्व होता है।
- `auto` एक पंजीकृत Plugin हार्नेस चुनता है जो प्रभावी प्रोवाइडर रूट का समर्थन करता हो; अन्यथा अंतर्निहित OpenClaw रनटाइम चुना जाता है। केवल प्रोवाइडर या मॉडल प्रीफ़िक्स कभी भी हार्नेस नहीं चुनता।
- OpenAI केवल बिना किसी लिखित अनुरोध ओवरराइड वाले सटीक आधिकारिक HTTPS Platform Responses या ChatGPT Responses रूट के लिए ही `codex` को अप्रत्यक्ष रूप से चुन सकता है। Completions अडैप्टर, कस्टम एंडपॉइंट और लिखित अनुरोध व्यवहार वाले रूट `openclaw` पर बने रहते हैं; प्लेनटेक्स्ट आधिकारिक HTTP एंडपॉइंट अस्वीकार कर दिए जाते हैं। [OpenAI का अप्रत्यक्ष एजेंट रनटाइम](/hi/providers/openai#implicit-agent-runtime) देखें।
