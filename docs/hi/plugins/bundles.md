OpenClaw तीन बाहरी पारितंत्रों से plugins इंस्टॉल कर सकता है: **Codex**, **Claude**,
और **Cursor**। इन्हें **बंडल** कहा जाता है—सामग्री और मेटाडेटा पैक जिन्हें
OpenClaw, Skills, hooks और MCP tools जैसी मूल सुविधाओं में मैप करता है।

<Info>
  बंडल मूल OpenClaw plugins के समान **नहीं** हैं। मूल plugins
  इन-प्रोसेस चलते हैं और किसी भी क्षमता को पंजीकृत कर सकते हैं। बंडल चुनिंदा
  सुविधा मैपिंग और अधिक सीमित विश्वास-सीमा वाले सामग्री पैक हैं।
</Info>

## बंडल क्यों मौजूद हैं

कई उपयोगी plugins Codex, Claude या Cursor प्रारूप में प्रकाशित किए जाते हैं। लेखकों
से उन्हें मूल OpenClaw plugins के रूप में फिर से लिखने की अपेक्षा करने के बजाय, OpenClaw
इन प्रारूपों का पता लगाता है और उनकी समर्थित सामग्री को मूल सुविधा
सेट में मैप करता है। आप Claude कमांड पैक या Codex स्किल बंडल इंस्टॉल करके उसका
तुरंत उपयोग कर सकते हैं।

## OpenClaw बंडल से क्या मैप करता है

आज प्रत्येक बंडल सुविधा OpenClaw में नहीं चलती। यहाँ बताया गया है कि क्या काम करता है और किसका
पता तो लगाया जाता है, लेकिन जिसे अभी जोड़ा नहीं गया है।

#### स्किल सामग्री

- बंडल स्किल रूट सामान्य OpenClaw स्किल रूट के रूप में लोड होते हैं।
- Claude `commands/` रूट को अतिरिक्त स्किल रूट माना जाता है।
- Cursor `.cursor/commands/` रूट को अतिरिक्त स्किल रूट माना जाता है।

Claude की Markdown कमांड फ़ाइलें और Cursor की कमांड Markdown, दोनों सामान्य
OpenClaw स्किल लोडर के माध्यम से काम करती हैं।

#### हुक पैक

बंडल हुक रूट **केवल** तब काम करते हैं, जब वे सामान्य OpenClaw हुक-पैक
लेआउट का उपयोग करते हैं: `HOOK.md` के साथ `handler.ts` या `handler.js`। आज यह मुख्य रूप से
Codex-संगत स्थिति है।

#### एम्बेडेड OpenClaw के लिए MCP

- सक्षम बंडल MCP सर्वर कॉन्फ़िग प्रदान कर सकते हैं।
- OpenClaw बंडल MCP कॉन्फ़िग को प्रभावी एम्बेडेड OpenClaw
  सेटिंग्स में `mcpServers` के रूप में मर्ज करता है।
- OpenClaw, stdio सर्वर लॉन्च करके या HTTP सर्वर से कनेक्ट करके, एम्बेडेड OpenClaw एजेंट
  टर्न के दौरान समर्थित बंडल MCP tools उपलब्ध कराता है।
- `coding` और `messaging` टूल प्रोफ़ाइल में बंडल MCP tools
  डिफ़ॉल्ट रूप से शामिल होते हैं; किसी एजेंट या Gateway के लिए इससे बाहर रहने हेतु `tools.deny: ["bundle-mcp"]` का उपयोग करें।
- प्रोजेक्ट-स्थानीय एम्बेडेड एजेंट सेटिंग्स बंडल डिफ़ॉल्ट के बाद भी लागू होती हैं, इसलिए
  आवश्यकता होने पर कार्यस्थान सेटिंग्स बंडल MCP प्रविष्टियों को ओवरराइड कर सकती हैं।
- पंजीकरण से पहले बंडल MCP टूल कैटलॉग को नियतात्मक रूप से क्रमबद्ध किया जाता है, ताकि
  अपस्ट्रीम `listTools()` क्रम में बदलाव से प्रॉम्प्ट-कैश टूल ब्लॉक बार-बार न बदलें।

##### ट्रांसपोर्ट

MCP सर्वर stdio या HTTP ट्रांसपोर्ट का उपयोग कर सकते हैं।

**Stdio** एक चाइल्ड प्रोसेस लॉन्च करता है:

```json
{
  "mcp": {
    "servers": {
      "my-server": {
        "command": "node",
        "args": ["server.js"],
        "env": { "PORT": "3000" }
      }
    }
  }
}
```

**HTTP** किसी चल रहे MCP सर्वर से कनेक्ट करता है और, जब तक
`streamable-http` का अनुरोध न किया गया हो, डिफ़ॉल्ट रूप से `sse` का उपयोग करता है:

```json
{
  "mcp": {
    "servers": {
      "my-server": {
        "url": "http://localhost:3100/mcp",
        "transport": "streamable-http",
        "headers": {
          "Authorization": "Bearer ${MY_SECRET_TOKEN}"
        },
        "connectionTimeoutMs": 30000
      }
    }
  }
}
```

- `transport`, `"streamable-http"` या `"sse"` स्वीकार करता है; छोड़े जाने पर डिफ़ॉल्ट `sse` होता है।
- `type: "http"` एक CLI-मूल डाउनस्ट्रीम आकार है; OpenClaw कॉन्फ़िग में `transport: "streamable-http"` का उपयोग करें। `openclaw mcp set` और `openclaw doctor --fix` सामान्य उपनाम को सामान्यीकृत करते हैं।
- केवल `http:` और `https:` URL स्कीम की अनुमति है।
- `headers` मान `${ENV_VAR}` इंटरपोलेशन का समर्थन करते हैं।
- `command` और `url`, दोनों वाली सर्वर प्रविष्टि अस्वीकार कर दी जाती है।
- URL क्रेडेंशियल (उपयोगकर्ता जानकारी और क्वेरी पैरामीटर) को टूल
  विवरण और लॉग में संशोधित करके छिपाया जाता है।
- `connectionTimeoutMs`, stdio और HTTP दोनों ट्रांसपोर्ट के लिए डिफ़ॉल्ट 30-सेकंड
  कनेक्शन टाइमआउट को ओवरराइड करता है। अनुरोध टाइमआउट डिफ़ॉल्ट रूप से 60 सेकंड है और
  इसे `requestTimeoutMs` से ओवरराइड किया जा सकता है।

##### टूल नामकरण

OpenClaw बंडल MCP tools को प्रदाता-सुरक्षित नामों के साथ
`serverName__toolName` प्रारूप में पंजीकृत करता है। उदाहरण के लिए, `"vigil-harbor"` कुंजी वाला सर्वर जो
`memory_search` टूल उपलब्ध कराता है, उसे `vigil-harbor__memory_search` के रूप में पंजीकृत किया जाता है।

- `A-Za-z0-9_-` से बाहर के वर्णों को `-` से बदल दिया जाता है।
- जिन खंडों की शुरुआत किसी गैर-अक्षर से होती, उन्हें अक्षर उपसर्ग मिलता है, इसलिए
  `12306` जैसी संख्यात्मक सर्वर कुंजियाँ प्रदाता-सुरक्षित टूल उपसर्ग बन जाती हैं।
- सर्वर उपसर्ग अधिकतम 30 वर्ण के होते हैं।
- पूरे टूल नाम अधिकतम 64 वर्ण के होते हैं।
- खाली सर्वर नामों के लिए फ़ॉलबैक `mcp` है।
- टकराने वाले स्वच्छीकृत नामों को संख्यात्मक प्रत्ययों से अलग किया जाता है।
- अंत में उपलब्ध कराया गया टूल क्रम सुरक्षित नाम के अनुसार नियतात्मक होता है, जिससे बार-बार होने वाले
  एम्बेडेड-एजेंट टर्न के दौरान कैश स्थिर रहता है।
- प्रोफ़ाइल फ़िल्टरिंग एक बंडल MCP सर्वर के प्रत्येक टूल को
  `bundle-mcp` के स्वामित्व वाला Plugin मानती है, ताकि प्रोफ़ाइल की अनुमति/अस्वीकृति सूचियाँ
  अलग-अलग उपलब्ध टूल नामों या `bundle-mcp` Plugin कुंजी, दोनों का संदर्भ दे सकें।

#### एम्बेडेड OpenClaw सेटिंग्स

बंडल सक्षम होने पर Claude `settings.json` को डिफ़ॉल्ट एम्बेडेड OpenClaw सेटिंग्स के रूप में
आयात किया जाता है। OpenClaw शेल ओवरराइड कुंजियों को लागू करने से पहले
स्वच्छीकृत करता है:

- `shellPath`
- `shellCommandPrefix`

#### एम्बेडेड OpenClaw LSP

- सक्षम Claude बंडल LSP सर्वर कॉन्फ़िग प्रदान कर सकते हैं।
- OpenClaw, `.lsp.json` के साथ मैनिफ़ेस्ट में घोषित सभी `lspServers` पथ लोड करता है।
- बंडल LSP कॉन्फ़िग को प्रभावी एम्बेडेड OpenClaw LSP
  डिफ़ॉल्ट में मर्ज किया जाता है।
- आज केवल समर्थित stdio-समर्थित LSP सर्वर चलाए जा सकते हैं; असमर्थित
  ट्रांसपोर्ट फिर भी `openclaw plugins inspect <id>` में दिखाई देते हैं।

### पहचाने गए, लेकिन निष्पादित नहीं

इन्हें पहचाना जाता है और निदान में दिखाया जाता है, लेकिन OpenClaw इन्हें नहीं चलाता:

- Claude `agents`, `hooks/hooks.json` स्वचालन, `outputStyles`
- Cursor `.cursor/agents`, `.cursor/hooks.json`, `.cursor/rules`
- क्षमता रिपोर्टिंग से परे Codex `.app.json` मेटाडेटा

## पहचान की प्राथमिकता

OpenClaw पहले मूल Plugin प्रारूप की जाँच करता है:

1. `openclaw.plugin.json` या `openclaw.extensions` वाला मान्य `package.json`—इसे **मूल Plugin** माना जाता है
2. बंडल मार्कर (`.codex-plugin/`, `.claude-plugin/` या डिफ़ॉल्ट Claude/Cursor लेआउट)—इसे **बंडल** माना जाता है

यदि किसी डायरेक्टरी में दोनों हैं, तो OpenClaw मूल पथ का उपयोग करता है। इससे
दोहरे-प्रारूप वाले पैकेज बंडल के रूप में आंशिक रूप से इंस्टॉल होने से बचते हैं।

## रनटाइम निर्भरताएँ और सफ़ाई

- तृतीय-पक्ष संगत बंडल को स्टार्टअप `npm install` सुधार नहीं मिलता। उन्हें
  `openclaw plugins install` के माध्यम से इंस्टॉल किया जाना चाहिए और इंस्टॉल की गई Plugin डायरेक्टरी में
  अपनी आवश्यक सभी चीज़ें शामिल करनी चाहिए।
- OpenClaw के स्वामित्व वाले बंडल plugins या तो कोर में हल्के रूप में शिप किए जाते हैं या
  Plugin इंस्टॉलर के माध्यम से डाउनलोड किए जा सकते हैं। Gateway स्टार्टअप उनके लिए कभी
  पैकेज मैनेजर नहीं चलाता।
- `openclaw doctor --fix` पुराने स्थानीय बंडल-Plugin इंस्टॉल रिकॉर्ड हटाता है
  और कॉन्फ़िग में अभी भी संदर्भित, लेकिन स्थानीय Plugin
  इंडेक्स से अनुपस्थित डाउनलोड करने योग्य plugins को पुनर्प्राप्त कर सकता है।

## सुरक्षा

बंडल की विश्वास-सीमा मूल plugins की तुलना में अधिक सीमित होती है:

- OpenClaw मनमाने बंडल रनटाइम मॉड्यूल को इन-प्रोसेस लोड **नहीं** करता।
- Skills और हुक-पैक पथ Plugin रूट के भीतर ही रहने चाहिए (सीमा-जाँच की जाती है)।
- सेटिंग्स फ़ाइलें समान सीमा जाँच के साथ पढ़ी जाती हैं।
- समर्थित stdio MCP सर्वर को सबप्रोसेस के रूप में लॉन्च किया जा सकता है।

इससे बंडल डिफ़ॉल्ट रूप से अधिक सुरक्षित होते हैं, लेकिन फिर भी आपको तृतीय-पक्ष
बंडल द्वारा उपलब्ध कराई जाने वाली सुविधाओं के लिए उन्हें विश्वसनीय सामग्री मानना चाहिए।
