OpenClaw में Google Meet, Microsoft Teams मीटिंग और Zoom के लिए अलग-अलग plugins हैं। ये तीनों Chrome के माध्यम से जुड़ सकते हैं, समान सहभागिता मोड का उपयोग करते हैं और Chrome को Gateway होस्ट या युग्मित node पर चला सकते हैं। इनके प्लेटफ़ॉर्म URL, इंस्टॉलेशन मॉडल और अतिरिक्त क्षमताएँ अलग-अलग हैं।

ये plugins मीटिंग में भाग लेते हैं। ये [Microsoft Teams चैनल](/hi/channels/msteams) जैसे मैसेजिंग चैनलों और [वॉइस कॉल plugin](/hi/plugins/voice-call) से अलग हैं।

## Plugin चुनें

| प्लेटफ़ॉर्म        | Plugin                                      | स्वीकृत मीटिंग लिंक                                                                                      | इंस्टॉलेशन                             | सहभागिता के मार्ग                                      | प्लेटफ़ॉर्म-विशिष्ट क्षमताएँ                                                                                |
| --------------- | ------------------------------------------- | ----------------------------------------------------------------------------------------------------------- | ---------------------------------------- | -------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| Google Meet     | [`google-meet`](/hi/plugins/google-meet)       | `meet.google.com/...`                                                                                       | npm या ClawHub से इंस्टॉल करें, फिर सक्षम करें | स्थानीय Chrome, युग्मित node पर Chrome या Twilio डायल-इन | Meet API या साइन-इन किए हुए ब्राउज़र के माध्यम से मीटिंग बना सकता है; OAuth के साथ समर्थित Meet आर्टिफ़ैक्ट पढ़ सकता है |
| Microsoft Teams | [`teams-meetings`](/hi/plugins/teams-meetings) | `teams.microsoft.com/l/meetup-join/...` के अंतर्गत कार्यस्थल लिंक और `teams.live.com/meet/...` के अंतर्गत उपभोक्ता लिंक | शामिल है; इसे सक्षम करें                      | स्थानीय Chrome या युग्मित node पर Chrome                  | कार्यस्थल और उपभोक्ता मीटिंग के लिए अतिथि के रूप में जुड़ना                                                                     |
| Zoom            | [`zoom-meetings`](/plugins/zoom-meetings)   | `zoom.us/j/...` और `example.zoom.us/j/...` जैसे अकाउंट सबडोमेन                                      | शामिल है; इसे सक्षम करें                      | स्थानीय Chrome या युग्मित node पर Chrome                  | Zoom Web App के माध्यम से अतिथि के रूप में जुड़ना                                                                           |

जब आपको मीटिंग बनानी हो, Google API आर्टिफ़ैक्ट चाहिए हों या Twilio फ़ोन मार्ग की आवश्यकता हो, तब Google Meet चुनें। उन प्लेटफ़ॉर्म पर सीधे ब्राउज़र से अतिथि के रूप में भाग लेने के लिए Teams या Zoom चुनें। Teams और Zoom plugins मीटिंग नहीं बनाते, डायल-इन नहीं करते, विक्रेता API को कॉल नहीं करते और मीटिंग रिकॉर्ड नहीं करते।

## मोड चुनें

तीनों plugins में समान मोड होते हैं:

| मोड         | व्यवहार                                                                                              | ऑडियो आवश्यकताएँ                                      |
| ------------ | ----------------------------------------------------------------------------------------------------- | ------------------------------------------------------- |
| `agent`      | रीयलटाइम ट्रांसक्रिप्शन कॉन्फ़िगर किए गए OpenClaw एजेंट को भेजा जाता है; सामान्य OpenClaw TTS उत्तर बोलता है।  | Chrome से प्रत्युत्तर बोलने के लिए BlackHole और SoX ब्रिज आवश्यक है। |
| `bidi`       | एक रीयलटाइम वॉइस मॉडल सुनता है और सीधे उत्तर देता है।                                                  | Chrome से प्रत्युत्तर बोलने के लिए BlackHole और SoX ब्रिज आवश्यक है। |
| `transcribe` | केवल अवलोकन के लिए जुड़ता है और प्लेटफ़ॉर्म द्वारा कैप्शन उपलब्ध कराने पर सीमित लाइव-कैप्शन ट्रांसक्रिप्ट प्रस्तुत करता है। | BlackHole या SoX प्रत्युत्तर ब्रिज की आवश्यकता नहीं है।                   |

जब एजेंट को केवल मीटिंग का टेक्स्ट चाहिए, तब `transcribe` का उपयोग करें। सामान्य OpenClaw तर्क और टूल के लिए `agent` का उपयोग करें। जब प्रत्येक संवाद को सामान्य एजेंट के माध्यम से रूट करने की तुलना में कम विलंबता वाली सीधी वॉइस अधिक महत्वपूर्ण हो, तब `bidi` का उपयोग करें।

कैप्शन ट्रांसक्रिप्ट सत्र-सीमित रनटाइम डेटा हैं, स्थायी मीटिंग रिकॉर्डिंग नहीं। कैप्शन की उपलब्धता अब भी मीटिंग प्लेटफ़ॉर्म, अकाउंट, भाषा और होस्ट नीति पर निर्भर करती है। इसकी ट्रांसक्रिप्ट सीमाओं और स्थिति फ़ील्ड के लिए प्लेटफ़ॉर्म मार्गदर्शिका देखें।

## Chrome और ऑडियो तैयार करें

Chrome को Gateway होस्ट या युग्मित node पर चलाया जा सकता है। रिमोट Chrome node को `browser.proxy` के साथ प्लेटफ़ॉर्म कमांड की अनुमति देनी होगी:

| Plugin          | Node कमांड           |
| --------------- | ---------------------- |
| Google Meet     | `googlemeet.chrome`    |
| Microsoft Teams | `teamsmeetings.chrome` |
| Zoom            | `zoommeetings.chrome`  |

Chrome के माध्यम से `agent` या `bidi` मोड के लिए, Chrome को macOS पर चलाएँ और उसी होस्ट पर साझा ऑडियो निर्भरताएँ इंस्टॉल करें:

```bash
brew install blackhole-2ch sox
sudo reboot
system_profiler SPAudioDataType | grep -i BlackHole
command -v sox
```

जब Chrome युग्मित node पर चलता है, तब भी Gateway होस्ट OpenClaw एजेंट और मॉडल क्रेडेंशियल का स्वामी रहता है। `agent` मोड के लिए रीयलटाइम ट्रांसक्रिप्शन प्रदाता और OpenClaw TTS कॉन्फ़िगर करें या `bidi` मोड के लिए रीयलटाइम वॉइस प्रदाता कॉन्फ़िगर करें। प्लेटफ़ॉर्म मार्गदर्शिकाओं में प्रदाता और ऑडियो-कमांड विकल्प दिए गए हैं।

## Plugin सक्षम करें

Google Meet को सक्षम करने से पहले इंस्टॉल करें। Teams मीटिंग और Zoom, OpenClaw में शामिल हैं और उन्हें केवल सक्षम करने की आवश्यकता है:

```bash
# केवल Google Meet
openclaw plugins install npm:@openclaw/google-meet

# केवल अपने उपयोग वाले मीटिंग plugins को सक्षम करें
openclaw plugins enable google-meet
openclaw plugins enable teams-meetings
openclaw plugins enable zoom-meetings
```

यदि आपका plugin-प्रबंधन मार्ग Gateway को स्वचालित रूप से पुनः आरंभ नहीं करता है, तो Gateway को पुनः आरंभ करें। फिर जुड़ने से पहले प्लेटफ़ॉर्म सेटअप जाँच चलाएँ।

## सत्यापित करें और जुड़ें

| प्लेटफ़ॉर्म        | सेटअप जाँच                    | जुड़ने का कमांड                                                                  |
| --------------- | ------------------------------ | ----------------------------------------------------------------------------- |
| Google Meet     | `openclaw googlemeet setup`    | `openclaw googlemeet join 'https://meet.google.com/abc-defg-hij'`             |
| Microsoft Teams | `openclaw teamsmeetings setup` | `openclaw teamsmeetings join 'https://teams.microsoft.com/l/meetup-join/...'` |
| Zoom            | `openclaw zoommeetings setup`  | `openclaw zoommeetings join 'https://zoom.us/j/1234567890'`                   |

किसी भी विफल सेटअप जाँच को उस ट्रांसपोर्ट और मोड के लिए अवरोधक मानें। केवल-अवलोकन स्मोक टेस्ट के लिए, `transcribe` मोड चुनें और कैप्शन टेक्स्ट की अपेक्षा करने से पहले पुष्टि करें कि स्थिति कॉल में सक्रिय सत्र दिखाती है।

## प्लेटफ़ॉर्म नीति प्रॉम्प्ट संभालें

ब्राउज़र ऑटोमेशन सामान्य अतिथि-नाम, जुड़ने-से-पहले कैमरा और माइक्रोफ़ोन, जुड़ने, कॉल के दौरान और कॉल छोड़ने के नियंत्रण संभालता है। यह प्लेटफ़ॉर्म या आयोजक की नीति को बायपास नहीं करता।

- Google Meet के लिए Google साइन-इन, होस्ट की स्वीकृति या ब्राउज़र अनुमति संबंधी निर्णय आवश्यक हो सकता है।
- Microsoft Teams के लिए टेनेंट साइन-इन, ईमेल सत्यापन या आयोजक की स्वीकृति आवश्यक हो सकती है।
- Zoom के लिए प्रमाणीकरण, ईमेल सत्यापन, पासकोड, CAPTCHA पूरा करना या होस्ट की स्वीकृति आवश्यक हो सकती है; अकाउंट ब्राउज़र से जुड़ना अक्षम भी कर सकता है।

जब जुड़ने या स्थिति का परिणाम `manualActionRequired` रिपोर्ट करे, तो पुनः प्रयास करने से पहले उसी OpenClaw Chrome प्रोफ़ाइल में रिपोर्ट किया गया चरण पूरा करें। बार-बार नए टैब खोलने से अकाउंट, टेनेंट, लॉबी या CAPTCHA अवरोध हल नहीं होता।

केवल उन्हीं मीटिंग में जुड़ें जहाँ ऑपरेटर एजेंट जोड़ने के लिए अधिकृत हो। जब स्थानीय नीति या सहमति नियमों के अनुसार स्वचालित सहभागिता, ट्रांसक्रिप्शन या संश्लेषित वाणी का खुलासा आवश्यक हो, तो प्रतिभागियों को बताएँ।

## Discord वॉइस चैट

[Discord वॉइस चैनल](/hi/channels/discord#voice-channels) ब्राउज़र मीटिंग ऑटोमेशन के बिना मूल, केवल-ऑडियो रीयलटाइम वार्तालाप प्रदान करते हैं। OpenClaw किसी वॉइस चैनल से जुड़ सकता है, सुन सकता है, संवादों को OpenClaw एजेंट या रीयलटाइम वॉइस मॉडल के माध्यम से रूट कर सकता है और उत्तर बोल सकता है। यह कैमरा वीडियो या स्क्रीन शेयरिंग भेजता या प्राप्त नहीं करता, भले ही लोग उसी Discord चैनल में वीडियो का उपयोग करें। इसलिए Discord वॉइस चौथे ब्राउज़र मीटिंग plugin के बजाय एक संबंधित लाइव-वार्तालाप माध्यम है।
